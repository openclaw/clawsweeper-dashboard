## Summary

Keep open. Current main still does not expose Feishu chat-history listing, so the contribution is relevant, but the PR branch has a blocking ID-type bug: it can pass a Feishu thread ID to the chat-history endpoint while declaring `container_id_type=chat`. It also needs tighter pagination normalization plus the required user-facing changelog/docs/tests before merge.

## What This Changes

This PR adds a Feishu shared message `list` action, a `listFeishuChatMessages` helper using Feishu `im.message.list` with `container_id_type=chat`, runtime wiring, action-discovery tests, and shared message-action target metadata.

## Best Possible Solution

Land a Feishu-owned message-history implementation that only sends chat IDs to `container_id_type=chat`, explicitly routes thread IDs to the existing thread-history helper or rejects them, clamps pagination inputs to Feishu's documented limits, returns pagination metadata predictably, documents the needed scopes, and adds focused unit coverage plus the required changelog entry.

## Reproduction Assessment

Yes. A static reproduction is enough: call the proposed `list` action with only `threadId`; the branch assigns that value to `chatId` and sends it to an API call using `container_id_type: "chat"`, while Feishu requires the ID type to match the selected container type.

## Solution Assessment

No. The feature direction is plausible, but the current patch is not the best implementation until the thread-ID fallback is removed or routed correctly, pagination inputs are normalized, and the user-facing release/docs/test coverage is added.

Full review comments:

- **[P1] Stop treating thread IDs as chat IDs:** `extensions/feishu/src/channel.ts:756-757`
  - body: The new `list` branch falls back to `ctx.params.threadId` when no chat ID is supplied, but `listFeishuChatMessages` calls `im.message.list` with `container_id_type: "chat"`. Feishu requires `container_id` to match the selected container type, so calls with only a thread ID will fail or query the wrong container; route thread IDs to the existing thread path or reject them.
  - confidence: 0.94

Normalize the public limit before calling `im.message.list` and cover the edge cases in the helper tests.
  - confidence: 0.84

  - body: This adds a user-facing Feishu message action, but the PR does not update `CHANGELOG.md`. OpenClaw policy requires user-facing `feat` changes to include a changelog entry before merge.
