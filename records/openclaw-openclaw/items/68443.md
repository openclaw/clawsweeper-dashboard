## Summary

Keep open. Current main still lacks a Feishu chat-history `list` action, so the PR is relevant, but the proposed branch has a blocking ID-type bug and is missing the required user-facing changelog entry. Open #41306 is related Feishu list work, but it targets chat listing and routing rather than reading message history by `container_id`, so it is not a clean superseding item.

## What This Changes

The branch adds a Feishu shared `message` action named `list`, a `listFeishuChatMessages` helper backed by `im.message.list` with `container_id_type=chat`, Feishu action-discovery tests, and shared action target metadata.

## Best Possible Solution

Land a Feishu-owned implementation that exposes chat-history reads only with correctly typed chat IDs, routes or rejects thread IDs explicitly, returns pagination metadata predictably, documents the required Feishu scopes, and includes focused unit tests plus the required changelog entry. Then decide whether any naming/routing overlap with #41306 should be consolidated.

## Reproduction Assessment

Yes. A static reproduction is enough: call the proposed `list` action with only `threadId`; the handler sends that thread ID to a helper using `container_id_type: "chat"`, while Feishu documents `chat` and `thread` as distinct container ID types.

## Solution Assessment

No. The feature direction is plausible for the Feishu plugin, but this patch is not the best implementation until it removes the `threadId` chat fallback or routes thread IDs to the existing thread helper, adds tests, and records the user-facing feature in the changelog.
