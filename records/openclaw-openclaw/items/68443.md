## Summary

The feature is not on current main and remains plausible, but this PR is not merge-ready because the branch can route thread IDs into a chat-history call, does not normalize numeric limits before calling Feishu, lacks inspectable real Feishu proof, and overlaps an open Feishu `list` PR that needs maintainer coordination.

## What This Changes

The branch adds a Feishu channel message `list` action for chat history by chat/container ID, wires it through runtime and shared action metadata, and updates Feishu action-surface tests.

## Best Possible Solution

Land a Feishu-owned message-history action that uses the existing chat-target resolver, rejects or separately routes thread IDs, normalizes pagination before the API call, adds focused tests/docs, and includes redacted live Feishu proof.

## Reproduction Assessment

Yes for the review findings: source inspection of the PR diff shows a thread ID can be passed to a chat-history endpoint and invalid numeric limits can flow into `page_size`. No live Feishu reproduction was run in this read-only review.

## Solution Assessment

No. The feature direction is plausible, but this branch bypasses existing Feishu chat-target normalization, mixes thread IDs with chat IDs, and needs pagination normalization plus real Feishu proof before it is the narrow maintainable solution.
