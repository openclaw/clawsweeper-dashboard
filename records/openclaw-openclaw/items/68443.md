## Summary

Keep open. The feature is plausible and current main does not provide this Feishu message-history action, but this PR still has blocking source-review findings and lacks inspectable real Feishu proof before merge.

## What This Changes

The branch adds a Feishu channel message `list` action for chat history by chat/container ID, wires it through the runtime and shared action metadata, and updates Feishu action-surface tests.

## Best Possible Solution

Land a Feishu-owned message-history action that uses the existing chat-target resolver, rejects or separately routes thread IDs, normalizes pagination before the API call, adds focused tests, documents required scopes, and includes redacted live Feishu proof.

## Reproduction Assessment

Yes for the review findings: source inspection of the PR head shows a thread ID can be passed to a chat-history endpoint and invalid numeric limits can flow into `page_size`. No live Feishu reproduction was run in this read-only review.

## Solution Assessment

No. The feature direction is plausible, but this branch bypasses existing Feishu chat-target normalization, mixes thread IDs with chat IDs, and needs pagination normalization plus real Feishu proof before it is the narrow maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Resolve only real Feishu chat IDs:** `extensions/feishu/src/channel.ts:758-760`
  - body: The new `list` branch falls back to `ctx.params.threadId` and passes that value to `listFeishuChatMessages`, whose request uses `container_id_type: "chat"`.   - confidence: 0.86
