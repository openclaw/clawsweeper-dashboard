## Summary

Keep open; this PR is useful but not merge-ready because the first cold status response can still format stale 200K fallback data before the async warmup completes, and the patch widens the agent context import graph on startup-adjacent paths.

## What This Changes

The PR adds background model context-window cache warming in status/Gateway startup, delegates status summary to the shared resolver, converts catalog loaders to static imports, and updates context/status/fast-path tests.

## Best Possible Solution

Keep one canonical status context-resolution path that gives deterministic first-cold output without widening `context.ts` import/startup cost, or explicitly approve background-only correction with latency proof before merge.

## Reproduction Assessment

Yes from source and PR-head tests. The added test confirms a first status-style lookup remains 200_000 until async warmup is flushed, although I did not run a live current-main repro in this read-only review.

## Solution Assessment

No. Delegating status to the shared resolver is the right direction, but fire-and-forget warmup plus static catalog imports is not the best fix for deterministic first-cold status output or startup safety.
