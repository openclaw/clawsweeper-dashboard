## Summary

## What This Changes

The branch removes `allowAsyncLoad:false` from status context lookups, delegates status-summary runtime context resolution to the shared resolver, adds Gateway startup cache warming, and updates focused tests.

## Best Possible Solution

Make status resolve the displayed context window deterministically from safe read-only metadata, or await a bounded read-only warmup before formatting, while keeping any new startup/status side effect explicit and covered by proof.

## Reproduction Assessment

Yes. Source inspection and the PR's own added test show first status-style reads return before async cache warmup completes, so a cold one-shot status render can still see `200_000`.

## Solution Assessment

No. Delegating CLI status to the shared resolver is a reasonable direction, but background-only warmup is not the best fix for a status command that needs the correct value before render.
