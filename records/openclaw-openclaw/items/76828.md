## Summary

Keep open: current main still has deliberate commentary-phase suppression and no clearly safe turn-end fallback for a no-final long tool loop. The report is credible, but the fallback policy needs maintainer judgment because broad commentary promotion could regress prior anti-leak behavior.

## What This Changes

The issue requests that final-only Telegram/agent delivery avoid silent user-visible drops when a long tool loop emits only commentary-phase assistant chunks and never emits a final-answer chunk.

## Best Possible Solution

Add a narrow turn-end no-final guard for final-only sessions that preserves commentary suppression by default while surfacing an explicit diagnostic or a maintainer-approved bounded fallback.

## Reproduction Assessment

Yes, at source level: commentary-phase message updates and message_end output are suppressed, and existing tests lock in that suppression. I did not run the full live Telegram/tool-loop scenario.

## Solution Assessment

Unclear. Auto-promoting the last commentary chunk would solve the silence but is not clearly the best fix because it can conflict with shipped anti-leak semantics; a terminal no-final diagnostic or tightly bounded fallback is safer.
