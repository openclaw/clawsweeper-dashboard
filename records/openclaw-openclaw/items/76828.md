## Summary

Keep open: current main has added useful Telegram topic final-delivery guardrails, but the core reported no-final path still exists at source level and the newer topic-thread `final_answer` evidence needs maintainer review against those recent changes.

## What This Changes

The issue asks OpenClaw to prevent Telegram turns from ending with no user-visible natural reply when final-only/commentary phase handling or topic-thread delivery suppresses the output.

## Best Possible Solution

Keep commentary/internal-text suppression, add a terminal no-visible-final guard that is explicit and observable, and separately ensure legitimate Telegram topic `final_answer` natural replies send to the originating topic with regression coverage.

## Reproduction Assessment

Yes, at source level: current main still suppresses commentary-phase assistant output and tests preserve commentary-only no-payload behavior. The live 15-minute Telegram topic run was not reproduced here.

## Solution Assessment

No clear single best fix yet. Broadly promoting the last commentary chunk could regress anti-leak behavior, so maintainers should choose a bounded terminal diagnostic/fallback policy and validate the separate Telegram topic final-delivery path after the recent #76764-era changes.
