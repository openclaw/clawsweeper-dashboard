## Summary

Keep open: current main still has a credible source-level path where commentary-only assistant output is intentionally suppressed before user-visible delivery, so the original final-only silent-turn class is not fully resolved. Nearby fixes now cover context-overflow replay, Telegram topic final delivery, and post-message-tool commentary, but they do not add a terminal no-final guard for this issue.

## What This Changes

The issue requests a safeguard so final-only delivery does not leave long tool-loop Telegram turns silent when the agent emits only commentary-phase assistant chunks.

## Best Possible Solution

Add a terminal no-final guard or explicit diagnostic for final-only source delivery that preserves commentary suppression, and handle topic-thread final_answer delivery as a separate Telegram visibility/config path.

## Reproduction Assessment

Yes, at source level: current main suppresses commentary-phase stream and message_end output, and tests assert commentary-only fallback payloads stay empty. I did not run a fresh live Telegram turn in this read-only review.

## Solution Assessment

Unclear. Auto-promoting the last commentary chunk would prevent silence but may leak intentionally suppressed progress text, so the safer path needs maintainer approval for a bounded diagnostic or fallback policy.
