## Summary

Close: current main already takes the safe diagnostic path for no-visible-output/incomplete tool turns, so final-only delivery no longer has to end as total silence; the later Telegram topic-thread observation matches the documented group/topic default where normal final replies stay private unless automatic visible replies are configured.

## What This Changes

The issue asks OpenClaw to avoid silently dropping tool-heavy Telegram/ACP turns when all persisted assistant text is commentary-phase or when topic-thread final delivery appears absent.

## Best Possible Solution

Keep the current diagnostic guard and documented group visible-reply policy; file a narrower follow-up only if `messages.groupChat.visibleReplies: "automatic"` still drops a `final_answer` topic reply on current main.

## Reproduction Assessment

No for current main: source inspection shows the no-payload terminal path now returns an explicit incomplete-turn error payload instead of ending silently. The released v2026.5.2 behavior described by the reporter is credible, but the current-main guard covers that failure class.

## Solution Assessment

Yes. The current implementation follows the safer suggested direction by surfacing a diagnostic rather than promoting arbitrary commentary into visible output, which preserves commentary-leak protections.
