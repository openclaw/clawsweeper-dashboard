## Summary

Current main no longer has the silent-final-only failure mode: commentary remains intentionally hidden, but a turn that produces no user-visible final payload is converted into a visible incomplete-turn error/diagnostic instead of completing silently; Telegram topic final delivery also has fallback paths that send a standard final message when preview finalization cannot prove delivery.

## What This Changes

Prevent final-only or Telegram-topic agent turns from silently dropping all user-visible output when only commentary or an unconfirmed topic final is produced.

## Best Possible Solution

Keep the current safer behavior: suppress commentary leaks, but surface a terminal diagnostic or verified final fallback whenever a turn would otherwise have no visible final output.

## Reproduction Assessment

Yes. The affected 2026.5.2 failure is reproducible from the reported transcript shape and current source still shows commentary-only output would be suppressed, while current main now routes the no-visible-output case through the incomplete-turn diagnostic path.

## Solution Assessment

Yes. The current fix direction is safer than auto-promoting arbitrary commentary because it preserves commentary suppression while preventing a silent user turn; Telegram topic finals also fall back to standard send when final delivery is not proven.
