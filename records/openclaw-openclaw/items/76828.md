## Summary

Keep open: current main still has a source-backed path where final-only/commentary-only turns can finish without user-visible delivery, and the newer Telegram topic evidence is credible but needs a config-controlled repro before changing group visible-reply semantics.

## What This Changes

Request that final-only Telegram/ACP turns never finish silently when only commentary-phase assistant chunks are emitted, and that Telegram topic final_answer natural replies dispatch or log a diagnostic.

## Best Possible Solution

Add a terminal no-final guard or bounded diagnostic/fallback for final-only turns that preserves commentary suppression, and separately verify topic natural-final delivery under explicit automatic visible replies.

## Reproduction Assessment

Yes, at source level: current main suppresses commentary-phase output and commentary-only fallback payloads, matching the silent final-only class. The topic-thread final_answer claim still needs a config-controlled repro with groupChat.visibleReplies set to automatic.

## Solution Assessment

Unclear: auto-promoting last commentary would fix silence but is not clearly the safest fix because current code and tests intentionally prevent commentary leaks. A maintainer-approved terminal diagnostic or bounded fallback is the safer direction.
