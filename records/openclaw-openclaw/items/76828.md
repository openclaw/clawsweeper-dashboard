## Summary

Keep open: current main still has a credible source-level path where final-only/channel delivery can end up with no user-visible text when the turn only produces commentary-phase assistant output, and the topic-thread addendum needs a config-controlled repro before it can be treated as the same bug.

## What This Changes

The issue asks OpenClaw to avoid silent final-only Telegram turns when long tool loops emit only commentary-phase assistant chunks, with additional evidence about Telegram topic final-answer delivery.

## Best Possible Solution

Add a terminal no-final guard or safe diagnostic/fallback that preserves commentary suppression, then separately cover Telegram topic final replies under an explicit automatic visible-replies configuration.

## Reproduction Assessment

Yes, at source level: current main suppresses commentary-phase assistant output and has no terminal no-final guard on the inspected final-only path. The topic-thread final_answer addendum is not yet a high-confidence repro because group/channel visible-reply policy must be pinned down.

## Solution Assessment

Unclear. Auto-promoting the last commentary chunk would address silence but conflicts with intentional anti-leak behavior, so the maintainable path is a maintainer-approved diagnostic or bounded fallback plus targeted topic-thread coverage.
