## Summary

Keep open: current main still has an intentional commentary-suppression path and no terminal no-final guard for final-only turns; the Telegram topic addendum overlaps the documented message-tool-only group policy and needs a config-controlled repro before changing routing behavior.

## What This Changes

Ensure Telegram/ACP final-only turns do not silently produce no user-visible output when an agent emits only commentary-phase assistant text, and clarify the topic-thread final-reply behavior.

## Best Possible Solution

Add a terminal no-final diagnostic or bounded fallback that preserves commentary suppression, then split the Telegram topic final_answer path into a separate fix only if it still fails with automatic group visible replies.

## Reproduction Assessment

Yes, at source level: current main suppresses commentary-phase assistant output and commentary-only fallback payloads, so a final-only turn with no final_answer text can complete silently. A deterministic live Telegram long-loop repro was not run in this read-only sweep.

## Solution Assessment

Unclear. Auto-promoting the last commentary chunk would address silence but conflicts with deliberate anti-leak behavior, so a maintainer-approved no-final guard or diagnostic is the narrower safe direction.
