## Summary

Keep open: current main still has a credible source-level path where commentary-phase assistant output is intentionally suppressed and a tool-heavy turn with no `final_answer` can end without a user-visible reply; choosing the fallback policy needs maintainer judgment because broad commentary promotion would conflict with shipped anti-leak behavior.

## What This Changes

The issue reports that `final_only` delivery can suppress an entire long Telegram-backed agent turn when all assistant chunks are tagged `phase=commentary` and no `final_answer` chunk is emitted.

## Best Possible Solution

Add a terminal no-final guard for final-only turns that surfaces a safe diagnostic or tightly bounded fallback, with regression coverage proving commentary remains hidden except under the approved no-final policy.

## Reproduction Assessment

Yes, at source level: current main suppresses commentary-phase message updates and message_end output, and signed commentary-only payload tests expect no user-visible payload. I did not establish a live 15-minute Telegram reproduction in this read-only review.

## Solution Assessment

Unclear. Auto-promoting the last commentary chunk would address silence but may regress shipped anti-leak semantics, so the narrower maintainable fix needs a maintainer-approved terminal diagnostic or bounded fallback.
