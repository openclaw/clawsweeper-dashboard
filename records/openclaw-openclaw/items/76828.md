## Summary

Keep open: current main still has a credible source-level path where commentary-phase assistant output is deliberately suppressed, while the newer Telegram topic evidence points to a second final-delivery path that needs diagnosis rather than cleanup closure.

## What This Changes

The issue asks OpenClaw to prevent Telegram/ACP turns from silently dropping user-visible replies when final-only delivery sees no final chunk, or when a Telegram topic final_answer chunk fails to dispatch.

## Best Possible Solution

Add a terminal no-final guard plus focused Telegram topic final-delivery diagnostics, with tests proving commentary remains hidden except under the maintainer-approved no-final fallback.

## Reproduction Assessment

Yes at source level: current main suppresses commentary-phase updates, commentary-only message_end output, and commentary-only final payload fallback. I did not establish the full live Telegram long-loop or topic-thread final_answer drop in this read-only review.

## Solution Assessment

Unclear. Auto-promoting the last commentary chunk would address silence but may undo anti-leak behavior, so maintainers should choose a bounded terminal diagnostic/fallback and separately verify the Telegram topic natural-delivery path.
