## Summary

Keep open: current main still intentionally suppresses commentary-only assistant output and has no verified terminal guard for final-only turns, so the silent-output class remains credible despite adjacent Telegram/topic fixes.

## What This Changes

The issue asks OpenClaw to avoid silent user-visible suppression when a final-only Telegram/ACP turn produces only commentary-phase assistant text and no final_answer chunk.

## Best Possible Solution

Add a maintainer-approved terminal no-final guard or bounded fallback for final-only turns that preserves commentary suppression while surfacing a user/operator-visible diagnostic instead of silence.

## Reproduction Assessment

Yes, at source level: current main suppresses commentary-phase stream and message_end output, and commentary-only fallback payloads are intentionally empty. I did not run a live Telegram turn in this read-only review.

## Solution Assessment

Unclear. Auto-promoting the last commentary chunk would address silence but is not clearly the safest fix because current tests enforce commentary suppression; a terminal diagnostic or bounded fallback needs maintainer policy approval.
