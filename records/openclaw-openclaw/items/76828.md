## Summary

Keep open: current main still has a credible source-level path where commentary-only assistant output is intentionally suppressed and a turn with no `final_answer` can remain user-silent. The later Telegram topic addendum may overlap with existing topic-delivery fixes and group visible-reply policy, but that does not close the primary no-final path.

## What This Changes

The issue asks OpenClaw to avoid silent Telegram/final-only turns when an agent emits only commentary-phase assistant text, preferably via a terminal diagnostic or bounded fallback.

## Best Possible Solution

Add a terminal no-final guard or explicit diagnostic for final-only turns that preserves commentary suppression, then separately verify Telegram topic `final_answer` delivery with automatic group visible replies enabled.

## Reproduction Assessment

Yes, at source level: current main suppresses commentary-phase output and refuses commentary-only fallback payloads, so a turn that never emits `final_answer` can finish without visible text. I did not run a live Telegram replay in this read-only review.

## Solution Assessment

Unclear. Auto-promoting the last commentary chunk would address silence but risks leaking intentionally hidden progress text; the narrower maintainable fix is a maintainer-approved terminal diagnostic or bounded fallback that does not weaken commentary suppression.
