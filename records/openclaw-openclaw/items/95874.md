## Summary

Current main and the latest release already provide the requested `openclaw cron edit --clear-model` path, including CLI wiring, `model: null` update payloads, backend delete semantics, tests, and docs.

## What This Changes

The issue asks for `openclaw cron edit` to add a supported `--clear-model` flag that removes an existing cron job `payload.model` override.

## Best Possible Solution

Keep the released `--clear-model` implementation and close this report as fixed; users on `2026.6.8` should upgrade to `v2026.6.9` or newer.

## Reproduction Assessment

No on current main: source inspection shows `cron edit --clear-model` is registered and emits `model: null`, and release-tag inspection shows the same behavior in `v2026.6.9`. The reported `2026.6.8` behavior is plausible, but it is no longer the current shipped behavior.

## Solution Assessment

Yes: the shipped fix is the narrowest maintainable path because it reuses the existing `cron.update` nullable clear contract instead of adding a second model-reset mechanism.
