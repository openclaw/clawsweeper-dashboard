## What This Changes

The PR adds half-block-art detection and styling for Control UI markdown code blocks and sidebar tool output so UTF-8 QR/block-art renders as tight black-and-white tiles.

## Best Possible Solution

Land a retargeted Control UI fix that applies half-block detection in the active markdown-it renderers and shared raw-output preprocessing paths, with consistent chat/sidebar CSS and focused visual proof.

## Reproduction Assessment

Yes, source-level: current main has no half-block handling in the active markdown-it code renderers, and raw chat/BTW text still goes directly through `toSanitizedMarkdownHtml`. I did not run a live browser repro in this read-only pass.

## Solution Assessment

No. The styling direction is reasonable, but this branch should be retargeted to current main's renderer rules and raw-output paths instead of the stale `htmlEscapeRenderer.code()` path and sidebar-only helper branch.

## Review Findings

Overall correctness: patch is incorrect

- **[P2] Retarget half-block detection to active renderers:** `ui/src/ui/markdown.ts:215-222`
  - body: This adds `.half-block-art` inside `htmlEscapeRenderer.code()`, but current main no longer uses that renderer; fenced and indented code render through `md.renderer.rules.fence` and `code_block`, so after conflicts are resolved QR fences would still render as normal code unless detection moves to those rules.

  - confidence: 0.78
