## Summary

Keep open: the reported QR rendering problem is valid and the PR has usable visual proof, but the branch still does not fix current main because it hooks an obsolete markdown renderer and misses raw transcript paths.

## What This Changes

The PR adds half-block-art detection and black-and-white tight-line-height styling for UTF-8 QR code blocks in the Control UI web chat and sidebar.

## Best Possible Solution

Land a current-main UI fix that detects half-block art in the active markdown-it code renderers and raw transcript/sidebar paths, preserves QR whitespace, keeps normal code blocks unchanged, and adds focused tests plus browser proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: current main has no half-block detection/styling in active code-block renderers, and unfenced raw chat text still flows through normal markdown parsing.

## Solution Assessment

No. The visual fix direction is reasonable, but this branch patches the retired renderer path and only partially handles raw output; the maintainable fix should be retargeted to current markdown-it and transcript/sidebar renderers.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Move half-block detection into markdown-it renderers:** `ui/src/ui/markdown.ts:215-219`
  - body: This adds `.half-block-art` from the old `htmlEscapeRenderer.code()` path, but current main renders fenced and indented code through markdown-it `fence` and `code_block` rules. Fenced QR blocks will still get the generic wrapper unless the detector moves to those active hooks.
  - confidence: 0.9
- **[P2] Fence raw QR transcript text before markdown parsing:** `ui/src/ui/chat/tool-helpers.ts:25-29`
  - body: The new fencing only runs in `formatToolOutputForSidebar()`. Main assistant/tool transcript text still goes straight through `toSanitizedMarkdownHtml(markdown)`, so unfenced multiline QR output remains paragraph text and never receives `.half-block-art`.
  - confidence: 0.88
- **[P3] Inherit the sidebar code font size:** `ui/src/styles/chat/sidebar.css:128-134`
  - body: The sidebar override inherits color and line-height but leaves the existing `pre code` font-size rule in effect, so the glyphs do not use the `1.3em` size set on the half-block `<pre>`. Add `font-size: inherit` for consistent QR tiling.
  - confidence: 0.78
