## Summary

Keep this PR open: the visual fix is useful and the screenshot proof is sufficient, but the branch is stale against current main's markdown-it renderer and still misses raw chat/BTW rendering paths, so it needs changes before merge.

## What This Changes

The branch adds half-block-art detection in markdown rendering, fences detected sidebar tool output, and adds chat/sidebar CSS for UTF-8 QR-style code blocks.

## Best Possible Solution

Land a current-main UI fix that recognizes half-block art in the markdown-it fence/code_block renderers and pre-fences raw chat, sidebar, and BTW text while preserving QR whitespace and overflow behavior.

## Reproduction Assessment

Yes, at source level: current main has no half-block handling in the active markdown-it renderers, and raw chat/BTW paths still call toSanitizedMarkdownHtml() directly. I did not run a live browser repro in this read-only review.

## Solution Assessment

No. The styling direction is reasonable, but this branch needs to be retargeted to current-main renderer rules and broadened beyond the sidebar helper before it is the maintainable fix.

- **[P2] Retarget half-block detection to active renderers:** `ui/src/ui/markdown.ts:216-222`

  - confidence: 0.78
