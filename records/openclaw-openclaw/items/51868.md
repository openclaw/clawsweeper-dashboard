## Summary

Keep open: current main still lacks the QR/block-art rendering fix, but this branch cannot merge as-is because it targets an obsolete renderer path and leaves raw transcript and side-result QR output uncovered.

## What This Changes

The branch adds half-block Unicode QR/block-art detection, sidebar raw-output fencing, and chat/sidebar CSS for tight black-on-white code-block rendering.

## Best Possible Solution

Implement shared half-block detection in the current markdown-it fence/code_block renderers and raw message paths, with CSS that preserves quiet-zone whitespace, inherited code sizing, and horizontal overflow.

## Reproduction Assessment

Yes. Source inspection shows current main has no half-block detector/styles, code blocks use generic markdown-it wrappers, and raw transcript/side-result text goes directly through markdown rendering; the PR screenshots show the affected web UI behavior before and after.

## Solution Assessment

No. The visual direction is right, but adding the class in `htmlEscapeRenderer.code()` is not the best current-main hook; the fix should move to the active markdown-it renderers and shared raw-output preprocessing.

  - confidence: 0.78
