## Summary

Keep open: the rendering gap is real and current main does not implement this, but the latest PR still misses the expanded inline tool-card output path and leaves a sidebar CSS inheritance issue that should be fixed before merge.

## What This Changes

This PR adds half-block-art detection to the UI markdown renderer and sidebar tool-output formatter, with black/white tight-line-height CSS for matching chat and sidebar code blocks.

## Best Possible Solution

Finish the UI fix across all affected render surfaces: markdown code blocks, sidebar markdown tool output, and expanded inline tool-card output, with regression coverage for detection, quiet-zone preservation, and CSS inheritance.

## Reproduction Assessment

Yes, there is a source-reproducible path: current main renders fenced markdown and expanded tool output through normal code/pre CSS, and the PR screenshots show the observed QR rendering breakage. I did not run a browser or scanner in this read-only pass.

## Solution Assessment

No. The PR is close, but it fixes markdown/sidebar paths while expanded inline tool-card output still uses a separate pre/code renderer and normal tool-card CSS; the maintainable fix is to share the half-block detection/styling across those UI paths.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Cover inline tool-card QR output:** `ui/src/ui/chat/tool-helpers.ts:28-31`
  - body: The new detection only reaches markdown code blocks and the sidebar formatter, but expanded tool results in the transcript still render `card.outputText` through `renderToolDataBlock()` as `<pre class="chat-tool-card__block-content"><code>...`. A tool-generated half-block QR therefore keeps the normal tool-card background, wrapping, and `line-height: 1.45` in the main chat output, which is the user-facing tool-output case this PR is meant to fix.
  - confidence: 0.91
- **[P2] Make sidebar QR glyphs inherit the art font size:** `ui/src/styles/chat/sidebar.css:130-136`
  - body: This selector resets color, line-height, background, and padding on the inner `<code>`, but it does not reset `font-size`. The existing `.sidebar-markdown :where(pre code)` rule still assigns `font-size: 12.5px`, so the characters inside sidebar half-block blocks ignore the `font-size: 1.3em` set on the parent `<pre>`. Add `font-size: inherit` here so the intended QR sizing applies.
  - confidence: 0.88
