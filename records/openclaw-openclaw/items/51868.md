## Summary

Keep open: the half-block QR rendering target is still useful and not implemented on current main, but this branch is not merge-ready because it patches a stale renderer path and misses current raw transcript/sidebar details.

## What This Changes

The PR adds half-block-art detection, code-block styling, sidebar tool-output fencing, and overflow handling for UTF-8 QR/block-art output in the Control UI.

## Best Possible Solution

Retarget QR rendering support to the active markdown-it code renderers and raw transcript/sidebar/side-result paths, preserve QR whitespace, keep normal code blocks unchanged, and add focused tests plus browser proof.

## Reproduction Assessment

Yes. Source inspection plus the supplied screenshots give a high-confidence path: current main has no half-block detector/styling and raw transcript text still flows through normal markdown rendering, though I did not run a live browser repro in this read-only pass.

## Solution Assessment

No. The visual direction is reasonable, but the PR targets the retired renderer path and only partially handles raw output; the safer fix belongs in the current markdown-it renderer and transcript/sidebar surfaces.

## Review Findings

Overall correctness: patch is incorrect

Full review comments:

- **[P2] Move half-block detection into active renderers:** `ui/src/ui/markdown.ts:215-219`
  - body: This adds `.half-block-art` from the old `htmlEscapeRenderer.code()` path, but current main renders fenced and indented code through markdown-it `fence` and `code_block` rules.   - confidence: 0.9

  - body: The new fencing only runs in `formatToolOutputForSidebar()`. Main assistant/tool transcript text still goes straight through `toSanitizedMarkdownHtml(markdown)`, so an unfenced multiline QR reply remains paragraph text and never receives `.half-block-art`.
  - confidence: 0.88
- **[P3] Inherit the sidebar code font size:** `ui/src/styles/chat/sidebar.css:128-134`
  - body: The sidebar override inherits color and line-height but leaves the existing `pre code` font-size rule in effect, so the glyphs do not use the larger half-block `<pre>` size. Add `font-size: inherit` for consistent QR tiling.
  - confidence: 0.78
