## Summary

Keep open: the reported QR rendering problem is still source-proven on current main, but this PR cannot merge as-is because it targets stale markdown renderer plumbing and misses raw transcript/BTW paths.

## What This Changes

Adds half-block-art detection, chat/sidebar CSS styling for UTF-8 QR code blocks, wrapper overflow handling, and sidebar tool-output fencing for detected half-block text.

## Best Possible Solution

Land a current-main UI fix that detects half-block art in the active markdown-it fence/code_block renderers and pre-fences raw chat, sidebar tool output, and BTW text while preserving QR quiet-zone whitespace.

## Reproduction Assessment

Yes, at source level: current main has no half-block handling in the active markdown-it code renderers, and raw transcript/BTW text still goes straight into `toSanitizedMarkdownHtml()`. I did not run a live browser repro in this read-only review.

## Solution Assessment

No. The styling direction is reasonable, but this branch needs to be retargeted to current main and broadened to the raw chat/BTW paths before it is the maintainable fix.

Overall correctness: patch is incorrect

Full review comments:

- **[P2] Retarget half-block detection to active renderers:** `ui/src/ui/markdown.ts:216-222`
  - body: This adds `.half-block-art` from `htmlEscapeRenderer.code()`, but current main renders fenced and indented code through `md.renderer.rules.fence` and `code_block`.   - confidence: 0.91

  - confidence: 0.88

  - body: The sidebar half-block `pre code` rule inherits color and line-height but leaves the existing `.sidebar-markdown :where(pre code)` font-size override active, so QR glyphs render smaller than the intended half-block `pre` size. Add `font-size: inherit`.
  - confidence: 0.78
