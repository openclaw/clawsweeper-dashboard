## Summary

Keep open: the web UI QR/block-art rendering bug is still present on current main, but this PR targets a removed markdown renderer and only partially covers raw output paths, so it needs a retargeted repair rather than cleanup closure.

## What This Changes

The PR adds half-block-art detection and CSS so UTF-8 QR/block-art code blocks render as tight black-and-white tiles in Control UI chat and sidebar output.

## Best Possible Solution

Land a retargeted Control UI fix on current main that covers active fenced/indented markdown renderers plus shared raw-output preprocessing, with consistent chat/sidebar CSS and focused visual proof.

## Reproduction Assessment

Yes, source-level: current main has no half-block handling in the active markdown-it code renderers, and raw chat/BTW text still goes directly through `toSanitizedMarkdownHtml`. I did not run a live browser repro in this read-only pass.

## Solution Assessment

No. The styling direction is useful, but this branch should be retargeted to current main's `md.renderer.rules.fence`/`code_block` paths and shared raw-output paths instead of the stale `htmlEscapeRenderer.code()` branch and sidebar-only helper path.
