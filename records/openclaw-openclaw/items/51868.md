## Summary

Keep open: the reported web UI QR rendering problem is valid and the visual proof is sufficient, but this branch is stale against current main and misses active renderer/raw-reply paths, so it needs a targeted repair before merge.

## What This Changes

The PR adds half-block-art detection and CSS styling for UTF-8 QR-style code blocks in web chat and sidebar Markdown rendering.

## Best Possible Solution

Land a current-main UI fix that classifies half-block art in markdown-it fence/code_block rendering and raw chat/sidebar/BTW text paths while preserving QR whitespace and narrow-pane overflow.

## Reproduction Assessment

Yes, at source level. Current main has no half-block handling in the active markdown-it code renderers, and raw chat/BTW text still goes straight through toSanitizedMarkdownHtml().

## Solution Assessment

No. The styling direction is reasonable, but this PR should be retargeted to current-main markdown-it renderer rules and raw reply preprocessing instead of the stale renderer hook.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Retarget half-block detection to markdown-it renderers:** `ui/src/ui/markdown.ts:216-222`
  - body: This adds the half-block wrapper from the old htmlEscapeRenderer.code path, but current main renders fenced and indented code through md.renderer.rules.fence and code_block. After rebasing, QR code fences would still get normal code-block markup unless the detector is wired into those active renderer rules.
  - confidence: 0.91
- **[P2] Fence raw half-block replies before Markdown parsing:** `ui/src/ui/chat/tool-helpers.ts:25-29`
  - body: The new raw half-block fencing only runs in formatToolOutputForSidebar(). Main transcript and BTW side-result text still call toSanitizedMarkdownHtml() directly, so an unfenced multiline QR reply is parsed as ordinary Markdown and never receives .half-block-art styling.
  - confidence: 0.88
- **[P3] Inherit the sidebar half-block code font size:** `ui/src/styles/chat/sidebar.css:128-134`
  - body: The sidebar half-block pre code rule inherits color and line-height but not font-size, leaving existing sidebar code font-size rules active for the glyphs. Add font-size: inherit so the intended QR sizing applies consistently.
  - confidence: 0.78
