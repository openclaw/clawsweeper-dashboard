## Summary

Keep open: the product problem is valid and the screenshot proof is sufficient, but the branch still misses current-main markdown-it and raw-output paths, so the patch would not fully fix QR rendering after retargeting.

## What This Changes

Adds half-block-art detection, sidebar fencing, and CSS for tight black-and-white rendering of UTF-8 QR/block-art code blocks in the web chat UI.

## Best Possible Solution

Retarget the UI fix to current main by detecting half-block art in markdown-it `fence`/`code_block` renderers plus raw transcript/sidebar/side-result paths, with focused regression tests and browser visual proof.

## Reproduction Assessment

Yes. The issue is source-reproducible because current main has no half-block detector/styling in active code-block renderers, and raw chat/sidebar text still flows directly through normal markdown sanitization.

## Solution Assessment

No. The visual direction is good, but the branch hooks an obsolete renderer path and only partially handles raw output; the safer fix is to wire the behavior into current markdown-it renderers and transcript/sidebar render paths.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Move detection into active markdown-it code renderers:** `ui/src/ui/markdown.ts:215-219`
  - body: This adds `.half-block-art` from the old `htmlEscapeRenderer.code()` path, but current main renders fenced and indented code through markdown-it `fence` and `code_block` rules. After retargeting, fenced QR blocks will still get the generic wrapper unless the detector moves to those active hooks.
  - confidence: 0.9
- **[P2] Fence raw QR transcript text before markdown parsing:** `ui/src/ui/chat/tool-helpers.ts:25-29`
  - body: The new fencing only runs in `formatToolOutputForSidebar()`. Main assistant/tool transcript text still goes straight through `toSanitizedMarkdownHtml(markdown)`, so unfenced multiline QR output remains paragraph text and never receives `.half-block-art`.
  - confidence: 0.88
- **[P3] Inherit the sidebar code font size for half-block art:** `ui/src/styles/chat/sidebar.css:128-134`
  - body: The sidebar override inherits color and line-height but leaves the existing `pre code` font-size rules in effect, so the glyphs do not use the `1.3em` size set on the half-block `<pre>`. Add `font-size: inherit` for consistent QR tiling.
  - confidence: 0.78
