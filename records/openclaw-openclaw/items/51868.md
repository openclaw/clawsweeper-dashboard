## Summary

Keep open: the reported WebChat QR rendering problem still appears valid on current main, but this branch is merge-conflicting and targets an obsolete Markdown renderer path, so it is not merge-ready as written.

## What This Changes

The PR adds half-block QR/block-art detection plus chat/sidebar CSS so Unicode QR code blocks render as tight black-and-white art in WebChat.

## Best Possible Solution

Land a current-main WebChat fix that classifies half-block QR/art in the active markdown-it code renderers and raw visible text paths, preserves quiet-zone whitespace, keeps normal code blocks unchanged, and proves scannable chat/sidebar rendering.

## Reproduction Assessment

Yes at source level: current main sends fenced code through markdown-it renderers and raw visible text through `toSanitizedMarkdownHtml`, with no half-block QR classifier on either path. I did not run a browser repro in this read-only review.

## Solution Assessment

No: the direction is sound, but the PR is not currently the best merge shape because it targets an old renderer hook and misses raw visible text paths. A small retargeted current-main repair is the safer solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Retarget half-block detection to markdown-it renderers:** `ui/src/ui/markdown.ts:216-222`
  - body: This hunk adds the `.half-block-art` wrapper inside the old `htmlEscapeRenderer.code()` path, but current main now renders fenced and indented code through `md.renderer.rules.fence` and `md.renderer.rules.code_block`. After resolving the merge conflict, leaving the detector on the old hook would mean fenced QR blocks still render as ordinary code blocks.
  - confidence: 0.94
- **[P2] Fence raw half-block replies before Markdown parsing:** `ui/src/ui/chat/tool-helpers.ts:25-29`
  - body: The new helper only fences sidebar tool output. Regular assistant/tool transcript text and side-result text still call `toSanitizedMarkdownHtml()` directly, so a raw multiline QR emitted without triple backticks is parsed as paragraphs with breaks and never receives `.half-block-art`.
  - confidence: 0.9
- **[P3] Inherit the sidebar half-block code font size:** `ui/src/styles/chat/sidebar.css:128-134`
  - body: The sidebar override resets color, line-height, background, and padding, but it does not reset `font-size`. The existing sidebar `pre code` rule can keep the rendered QR characters at the normal code size instead of the enlarged half-block-art size, so sidebar output can still differ from chat output.
  - confidence: 0.82
