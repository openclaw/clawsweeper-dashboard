## Summary

Keep open. Current main still lacks half-block QR/block-art handling, and the PR contains useful user-visible work, but it remains incorrect against current main because it targets a stale renderer hook and does not cover raw transcript/side-result QR output.

## What This Changes

The PR adds a half-block-art detector and black-on-white tight-line-height styling for QR/block-art code blocks in Control UI chat and sidebar tool output.

## Best Possible Solution

Land a retargeted UI fix that detects half-block art in the active markdown-it renderers and shared raw-output preprocessing while keeping chat/sidebar CSS consistent.

## Reproduction Assessment

Yes, source-level. Current main lacks half-block handling, renders code through markdown-it fence/code_block rules, and sends raw transcript/side-result text directly to the sanitizer; the PR screenshots show the visible before/after behavior.

## Solution Assessment

No. The visual direction is right, but the current branch targets the stale htmlEscapeRenderer path and only fences sidebar tool output, so the fix should move to the active renderers and shared raw-output paths.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Move detection into the active markdown-it renderers:** `ui/src/ui/markdown.ts:216-222`
  - body: This adds `.half-block-art` inside `htmlEscapeRenderer.code()`, but current main emits fenced and indented code through `md.renderer.rules.fence` and `md.renderer.rules.code_block`. After retargeting, QR code blocks will still use the generic wrapper unless the detector is applied in those active renderer rules.
  - confidence: 0.91
- **[P2] Fence raw half-block replies before markdown parsing:** `ui/src/ui/chat/tool-helpers.ts:25-29`
  - body: The new fencing only runs in `formatToolOutputForSidebar()`. Main assistant/tool transcript text and BTW side-result text still go straight to `toSanitizedMarkdownHtml()`, so an unfenced multiline QR reply remains paragraph text and never receives `.half-block-art`.
  - confidence: 0.88
- **[P3] Inherit the sidebar half-block code font size:** `ui/src/styles/chat/sidebar.css:128-134`
  - body: The sidebar override inherits color and line-height but leaves `.sidebar-markdown code { font-size: 0.88em; }` active on the inner `<code>`, shrinking the glyphs below the intended half-block `<pre>` size. Add `font-size: inherit` to keep sidebar rendering consistent.
  - confidence: 0.78
