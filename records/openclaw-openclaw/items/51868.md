## Summary

Keep open: the reported UI bug is plausible and the screenshot proof shows the desired visual result, but the branch is still incorrect against current main because it patches a stale renderer path and leaves raw transcript and side-result QR output uncovered.

## What This Changes

Adds a half-block art detector, sidebar output fencing, and chat/sidebar CSS intended to render UTF-8 QR/block art as tight black-on-white code blocks in the Control UI.

## Best Possible Solution

Port the detector into the current markdown-it code renderers and shared raw-output preprocessing, keep chat/sidebar CSS consistent, preserve QR quiet-zone whitespace, and verify scanable rendering in the browser.

## Reproduction Assessment

Yes, source-reproducible with high confidence: current main has no half-block handling, active code renderers return generic wrappers, and raw transcript/side-result text flows directly into markdown rendering. I did not run a live browser repro in this read-only pass.

## Solution Assessment

No. The visual direction is right, but the branch should target the current markdown-it renderers and shared raw-output paths instead of the stale `htmlEscapeRenderer.code()` path plus sidebar-only fencing.
