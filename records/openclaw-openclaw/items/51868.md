Keep open: the PR targets a real web UI rendering problem and has usable screenshot proof, but the current branch does not wire the fix into current main's active markdown-it renderers or raw transcript paths, so it needs repair before merge.

## What This Changes

The PR adds half-block-art detection, sidebar output fencing, and chat/sidebar CSS to render UTF-8 QR and block-art code blocks as tight black-and-white tiles.

## Best Possible Solution

Land a current-main UI fix that detects half-block art in markdown-it code renderers and raw chat/sidebar text paths, preserves QR quiet-zone whitespace, keeps normal code blocks unchanged, and includes focused tests plus browser proof.

## Reproduction Assessment

Yes. The issue is source-reproducible: current main has no half-block detector/styling in the active code renderers, and raw chat/sidebar text still enters `toSanitizedMarkdownHtml()` directly.

## Solution Assessment

No. The visual direction is reasonable, but this branch hooks an obsolete renderer path and only partially handles raw output; the narrower maintainable fix is to wire the behavior into current markdown-it and transcript/sidebar render paths.
