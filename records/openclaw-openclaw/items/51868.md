## Summary

Keep this PR open, but it still needs changes before merge: the proof and visual goal are valid, yet the branch targets an obsolete markdown renderer path and misses raw transcript rendering plus a sidebar font-size override on current main.

## What This Changes

The branch adds half-block-art detection for markdown code blocks, fences detected tool-output QR text for sidebar rendering, and adds chat/sidebar CSS for tight black-and-white QR rendering.

## Best Possible Solution

Land a current-main UI fix that detects half-block art in markdown-it fence/code_block renderers and raw transcript/sidebar paths, preserves QR quiet-zone whitespace, keeps normal code blocks unchanged, and includes focused tests plus browser proof.

## Reproduction Assessment

Yes. The issue is source-reproducible on current main because code blocks and raw chat/sidebar text still flow through generic markdown rendering with no half-block detector or QR-specific styling.

## Solution Assessment

No. The visual styling direction is good, but the proposed hook is stale; the maintainable fix is to wire detection into current markdown-it renderers and the raw transcript/side-result/sidebar paths before markdown parsing.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Retarget detection to the markdown-it renderers:** `ui/src/ui/markdown.ts:215-219`
  - body: This adds `.half-block-art` from `htmlEscapeRenderer.code()`, but current main renders fenced and indented code through markdown-it `fence` and `code_block` rules. After retargeting, QR code fences will still use the generic wrapper unless this detection moves to those active renderers.
  - confidence: 0.9
- **[P2] Fence raw QR transcript text before markdown parsing:** `ui/src/ui/chat/tool-helpers.ts:25-29`
  - body: The new half-block fencing only runs in `formatToolOutputForSidebar()`. Main assistant/tool transcript text and side-result text still go straight through `toSanitizedMarkdownHtml(...)`, so unfenced multiline QR output remains paragraph text and never receives the half-block styling.
  - confidence: 0.88
- **[P3] Inherit the sidebar code font size:** `ui/src/styles/chat/sidebar.css:128-134`
  - body: The sidebar override inherits color and line height but leaves the existing `pre code` font-size rule in effect. Add `font-size: inherit` so the glyphs actually use the `1.3em` size set on the half-block `<pre>`.
  - confidence: 0.78
