## Summary

Keep open: the bug is valid and the PR has usable after-fix proof, but the branch is not correct against current main because it hooks an obsolete markdown renderer path and misses raw transcript rendering paths.

## What This Changes

The PR adds half-block-art detection and CSS so UTF-8 QR/block-art code blocks render as tight black-and-white tiles in web chat and sidebar output.

## Best Possible Solution

Land a current-main UI fix that detects half-block art in markdown-it fence/code_block renderers and raw transcript/sidebar paths, preserves QR whitespace, keeps normal code blocks unchanged, and includes focused tests plus browser proof.

## Reproduction Assessment

Yes. The issue is source-reproducible because current main has no half-block detector/styling in active code-block renderers, and raw chat/sidebar text still flows directly through normal markdown sanitization.

## Solution Assessment

No. The visual direction is good, but the branch hooks an obsolete renderer path and only partially handles raw output; the safer fix is to wire the behavior into current markdown-it renderers and transcript/sidebar render paths.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Move detection into the markdown-it code renderers:** `ui/src/ui/markdown.ts:215-219`
  - body: This adds `.half-block-art` from `htmlEscapeRenderer.code()`, but current main renders fenced and indented code through markdown-it `fence` and `code_block` rules. After retargeting, fenced QR blocks will still get the generic wrapper unless the detector moves to those active hooks.
  - confidence: 0.9
- **[P2] Fence raw QR transcript text before markdown parsing:** `ui/src/ui/chat/tool-helpers.ts:25-29`
  - body: The new fencing only runs in `formatToolOutputForSidebar()`. Main assistant/tool transcript text still goes straight through `toSanitizedMarkdownHtml(markdown)`, so unfenced multiline QR output remains paragraph text and never receives `.half-block-art`.
  - confidence: 0.88
- **[P3] Inherit the sidebar code font size for half-block art:** `ui/src/styles/chat/sidebar.css:128-134`
  - body: The sidebar override inherits color and line-height but leaves the existing `pre code` font-size rules in effect, so the glyphs do not use the `1.3em` size set on the half-block `<pre>`. Add `font-size: inherit` for consistent QR tiling.
  - confidence: 0.78
