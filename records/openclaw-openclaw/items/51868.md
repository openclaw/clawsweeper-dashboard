## Summary

This PR should stay open because the QR rendering problem is valid and still useful to fix, but the branch targets an obsolete markdown renderer path and does not cover all current web chat render surfaces.

## What This Changes

The PR adds half-block-art detection plus tight black-and-white styling for UTF-8 QR/code-block art in web chat and sidebar tool output.

## Best Possible Solution

Move half-block detection into the current markdown-it fence/code_block renderers and add shared raw-output preprocessing for transcript, sidebar, and side-result text, with focused tests and browser proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main has generic markdown-it code-block rendering with no half-block styling, and raw multiline QR output can bypass code-block handling entirely; the PR screenshots show the same observable rendering failure and improvement.

## Solution Assessment

No. The visual direction is sound, but this branch is not the best current-main fix because it targets an obsolete renderer hook and only partially handles raw output paths.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Apply half-block handling in the active renderer:** `ui/src/ui/markdown.ts:216-220`
  - body: This adds the half-block wrapper in `htmlEscapeRenderer.code()`, but current main renders fenced and indented blocks through markdown-it `fence` and `code_block` rules. After rebasing, QR code blocks still take the generic code-block path unless the detector moves into those active hooks.
  - confidence: 0.91
- **[P2] Fence raw half-block replies before markdown parsing:** `ui/src/ui/chat/tool-helpers.ts:25-29`

  - confidence: 0.88
- **[P3] Inherit the sidebar half-block code font size:** `ui/src/styles/chat/sidebar.css:128-134`
  - body: The sidebar inner-code override inherits color and line-height but leaves the existing `pre code` font-size rule in force. Add `font-size: inherit` so sidebar QR glyphs use the larger half-block `<pre>` size consistently.
  - confidence: 0.8
