## Summary

Keep open: this is a real Control UI rendering bug with sufficient screenshot proof, but the PR is not safe to merge because it targets an obsolete markdown renderer path and misses raw transcript output on current main.

## What This Changes

The PR adds half-block-art detection, sidebar output fencing, and chat/sidebar CSS so UTF-8 QR/block-art code blocks render as tight black-and-white tiles.

## Best Possible Solution

Land a current-main UI fix that detects half-block art in the active markdown-it code renderers and raw chat/sidebar text paths, preserves QR quiet-zone whitespace, keeps normal code blocks unchanged, and includes focused tests plus browser proof.

## Reproduction Assessment

Yes. The issue is source-reproducible: current main has no half-block detector/styling in the active code-block renderers, and raw chat/sidebar text still flows into the normal markdown sanitizer path.

## Solution Assessment

No. The visual direction is good, but the branch hooks an obsolete renderer path and only partially handles raw output; the safer solution is a current-main replacement that wires the behavior into markdown-it and transcript/sidebar render paths.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Move half-block detection into the active markdown-it renderers:** `ui/src/ui/markdown.ts:215-219`
  - body: This adds `.half-block-art` from `htmlEscapeRenderer.code()`, but current main renders fenced and indented code through markdown-it `fence` and `code_block` rules. After retargeting, fenced QR blocks will still get the generic wrapper unless the detector moves to those active hooks.
  - confidence: 0.9
- **[P2] Wrap raw QR transcript text before markdown parsing:** `ui/src/ui/chat/tool-helpers.ts:25-29`
  - body: The new fencing only runs in `formatToolOutputForSidebar()`. Main assistant/tool transcript text still goes straight through `toSanitizedMarkdownHtml(markdown)`, so unfenced multiline QR output remains paragraph text and never receives `.half-block-art`.
  - confidence: 0.88
- **[P3] Inherit the sidebar half-block code font size:** `ui/src/styles/chat/sidebar.css:128-134`
  - body: The sidebar override inherits color and line-height but leaves the existing `pre code` font-size in effect, so the glyphs do not use the `1.3em` size set on the half-block `<pre>`. Add `font-size: inherit` for consistent QR tiling.
  - confidence: 0.78
