## Summary

Keep open. The PR addresses a real Web UI rendering gap, but current main still has no half-block QR handling and the proposed patch is stale/incomplete for the active markdown-it renderer, raw chat replies, and sidebar font inheritance.

## What This Changes

The PR adds half-block Unicode art detection, sidebar output fencing, and chat/sidebar CSS intended to make UTF-8 QR code blocks render as tight black-and-white tiles.

## Best Possible Solution

Rework this as a current-main Web UI fix that handles fenced, indented, sidebar, and raw assistant/tool half-block output while preserving QR quiet-zone whitespace and keeping sanitizer behavior unchanged.

## Reproduction Assessment

Yes. The PR body provides concrete web chat steps and screenshots, and source inspection on current main shows half-block QR text still flows through generic markdown/CSS paths without QR-safe styling.

## Solution Assessment

No. The direction is right, but the patch targets an obsolete renderer path and does not yet cover raw transcript output or the sidebar inner-code font override.

  - body: Current main no longer uses the old `htmlEscapeRenderer.code` path this branch modifies; code blocks are rendered by `md.renderer.rules.fence` and `md.renderer.rules.code_block`. An unfenced multiline QR in the main transcript, which is part of the reported path, will never receive `.half-block-art` styling.
  - confidence: 0.88

  - body: The sidebar half-block `pre` sets a larger font size, but the inner `code` rule still lacks `font-size: inherit`, so the existing sidebar `pre code` font-size can control the glyphs and make sidebar QR blocks render smaller than intended.
  - confidence: 0.78
