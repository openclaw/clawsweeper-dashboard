## What This Changes

The branch adds a half-block-art heuristic, sidebar tool-output fencing, and chat/sidebar CSS so UTF-8 QR/block art can render as tight black-and-white tiles.

## Best Possible Solution

Land a retargeted Control UI fix that applies half-block detection in the active markdown-it renderers and shared raw-output preprocessing paths, with consistent chat/sidebar CSS and focused visual proof.

## Reproduction Assessment

Yes, source-level: current main has no half-block handling in `md.renderer.rules.fence` or `code_block`, and raw chat/BTW text still flows straight into `toSanitizedMarkdownHtml`. I inspected contributor screenshots but did not run a live browser repro in this read-only pass.

## Solution Assessment

No. The styling direction is reasonable, but the branch should target current main’s markdown-it renderer rules and raw-output paths instead of the stale `htmlEscapeRenderer.code()` path and sidebar-only helper branch.

  - confidence: 0.8
