## Summary

Keep this PR open: current main still lacks the requested half-block QR rendering support, and the PR has useful proof, but the branch is stale against the active markdown-it renderer and still misses raw transcript paths that would keep some QR output unstyled.

## What This Changes

The PR adds half-block-art detection, sidebar tool-output fencing, and CSS for rendering UTF-8 QR/block-art code blocks in the Control UI chat and sidebar.

## Best Possible Solution

Land a retargeted Control UI fix that applies half-block detection in the active markdown-it renderers and shared raw-output preprocessing paths, with consistent chat/sidebar CSS and focused visual proof.

## Reproduction Assessment

Yes, source-level: current main has no half-block handling in the active markdown-it code renderers, and raw chat/BTW text still goes directly through `toSanitizedMarkdownHtml`. I did not run a live browser repro in this read-only pass.

## Solution Assessment

No. The styling direction is reasonable, but this branch should be retargeted to current main's renderer rules and raw-output paths instead of the stale `htmlEscapeRenderer.code()` path and sidebar-only helper branch.
