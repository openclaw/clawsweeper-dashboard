Keep open: the proof is useful and the direction is sound, but the latest diff still leaves raw transcript and BTW side-result half-block QR text outside the new styling path, and the sidebar code reset still misses the intended glyph-size inheritance.

## What This Changes

This PR adds half-block-art detection and styling for WebChat markdown code blocks and sidebar tool output, plus test mock updates for the exported markdown helper.

## Best Possible Solution

Land a WebChat fix that classifies half-block QR/art before markdown rendering for fenced, indented, raw transcript, sidebar, and side-result paths, while preserving quiet-zone whitespace and normal code rendering.

## Reproduction Assessment

Yes, at source level: current main lacks a half-block classifier, and the PR screenshots show the visual problem and intended improvement. I did not run a live browser repro in this read-only review.

## Solution Assessment

No, not yet. The code-block/sidebar approach is maintainable, but the fix needs to cover raw transcript and BTW side-result text and finish the sidebar font-size override before merge.

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Handle raw half-block text before Markdown parsing:** `ui/src/ui/markdown.ts:559-560`

  - confidence: 0.82
