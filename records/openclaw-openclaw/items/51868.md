## Summary

Keep open: the PR is a useful Web UI fix with screenshot proof, but the current branch still leaves raw transcript QR output outside the half-block path and the sidebar CSS can override the intended QR code font sizing.

## What This Changes

The PR adds half-block-art detection to the Web UI markdown renderer and sidebar tool-output formatting, plus chat/sidebar CSS for tightly rendered UTF-8 QR code blocks.

## Best Possible Solution

Extend the half-block wrapper to fenced, indented, and raw visible chat/sidebar text paths, make half-block code elements inherit the QR pre font sizing, add focused markdown/tool-helper coverage, and verify the browser rendering again.

## Reproduction Assessment

Yes at source level, but I did not run a browser repro in this read-only review. Current main lacks half-block classification/styling, and the inspected screenshots show the before/after Web UI rendering problem.

## Solution Assessment

No, not as currently patched. The renderer/CSS direction is right, but the fix needs to cover raw transcript text and inherit sidebar code sizing to be the narrow maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Handle raw half-block transcript text before Markdown parsing:** `ui/src/ui/markdown.ts:559-560`
  - body: The classifier now runs inside the markdown code renderers, plus the sidebar formatter, but regular assistant/tool transcript text still flows straight into `toSanitizedMarkdownHtml()`. If a tool emits multiline QR art without fences, markdown-it renders it as paragraph text with breaks, so `.half-block-art` is never applied and the main chat QR remains unscannable.
  - confidence: 0.86
- **[P3] Inherit font size for sidebar half-block code:** `ui/src/styles/chat/sidebar.css:431-436`
  - body: The sidebar half-block rule inherits color and line-height, but `.sidebar-markdown :where(pre code)` still supplies a fixed `font-size: 12.5px` for the actual `<code>` text. Add `font-size: inherit` here so the QR characters use the enlarged `pre` size this patch intends.
  - confidence: 0.82
