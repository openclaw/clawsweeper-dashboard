## Summary

Keep open: current main still lacks half-block QR-safe rendering, and this PR is useful source work but does not yet solve the active Web UI paths because it targets an obsolete renderer shape and misses raw transcript handling, sidebar sizing consistency, and the required changelog entry.

## What This Changes

The PR branch adds a half-block art detector, sidebar output fencing, code-block overflow handling, and chat/sidebar CSS so UTF-8 QR block art can render as tight black-and-white tiles.

## Best Possible Solution

Retarget the fix to current main with shared half-block detection in the active markdown-it code renderers, raw transcript handling that preserves QR quiet-zone whitespace, consistent chat/sidebar CSS, regression tests, and a single changelog entry.

## Reproduction Assessment

Yes, source-reproducible. The PR body provides concrete web-chat steps and screenshots, and current main still routes fenced code and raw chat markdown through generic render/CSS paths without QR-safe half-block styling.

## Solution Assessment

No. The direction is right, but this patch is not the best current-main solution because it hooks the removed `htmlEscapeRenderer.code` path and only adds sidebar fencing; the narrow maintainable fix should cover markdown-it fenced/indented blocks plus raw chat output without changing sanitization.
