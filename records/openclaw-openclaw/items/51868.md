## Summary

Keep open: current main still lacks half-block QR rendering support, but this PR is not merge-ready because it targets the old renderer shape and misses a main transcript path plus one sidebar CSS inheritance detail.

## What This Changes

The branch adds a half-block-art detector, wraps detected sidebar output in code fences, and adds web chat/sidebar CSS for tight black-and-white QR rendering.

## Best Possible Solution

Land a current-main UI fix that applies shared half-block detection in the active markdown-it renderers and raw chat/sidebar output paths, preserves QR quiet-zone whitespace, keeps normal code blocks unchanged, and includes focused tests plus browser proof.

## Reproduction Assessment

Yes, source-reproducible. Current main has no half-block detector/styling in the active code-block renderers, and the main chat transcript path sends raw markdown directly into the sanitizer/render pipeline.

## Solution Assessment

No. The intended direction is good, but this branch patches an obsolete renderer hook and only partially handles raw output, so the maintainable fix should be retargeted to current markdown-it and transcript/sidebar paths.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Retarget half-block wrapping to markdown-it renderers:** `ui/src/ui/markdown.ts:215-219`
  - body: This adds `.half-block-art` from `htmlEscapeRenderer.code()`, but current main renders fenced and indented code through `md.renderer.rules.fence` and `code_block`. After rebase, fenced QR blocks will keep the generic wrapper unless the detector moves to those active hooks.
  - confidence: 0.9

  - body: The new fencing only runs in `formatToolOutputForSidebar()`. Main assistant/tool transcript text still goes through `toSanitizedMarkdownHtml(markdown)`, where unfenced multiline QR art is parsed as ordinary markdown, so the web chat case remains unstyled.
  - confidence: 0.88
- **[P3] Inherit the sidebar half-block font size:** `ui/src/styles/chat/sidebar.css:128-134`
  - body: The sidebar override inherits color and line-height but leaves existing `pre code` font-size in effect, so the glyphs do not use the `1.3em` size set on the half-block `<pre>`. Add `font-size: inherit` for consistent QR tiling.
  - confidence: 0.78
