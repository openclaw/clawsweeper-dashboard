## Summary

Keep open: the requested web UI rendering improvement is useful and the screenshots show a credible after state, but this branch is stale against current main’s markdown renderer and still misses raw main-transcript QR output.

## What This Changes

The PR adds half-block Unicode art detection, sidebar tool-output fencing, and chat/sidebar CSS so UTF-8 QR-like block text renders as tight black-and-white tiles.

## Best Possible Solution

Implement shared half-block detection on current main’s markdown-it fenced/indented code renderers and raw transcript/sidebar paths, keep sanitization unchanged, and add focused regression plus visual QR verification.

## Reproduction Assessment

Yes, source-reproducible. Current main has generic code-block and raw markdown rendering with no half-block detector or styling, and the PR screenshots show the same broken-before and fixed-after visual behavior.

## Solution Assessment

No. The direction is right, but the branch is not the best current-main solution because it modifies the removed renderer path and only fences sidebar raw output, leaving normal transcript QR output uncovered.

  - body: The sidebar half-block `pre code` override does not set `font-size: inherit`, so the existing sidebar `pre code` font-size can still control the rendered glyphs and make QR blocks smaller than the surrounding half-block `pre` intends.
  - confidence: 0.8
