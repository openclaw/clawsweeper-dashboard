## Summary

Keep open: current main still lacks half-block QR/block-art handling, and the PR has useful proof and a plausible direction, but the branch is not correct against the current markdown-it renderer and raw transcript paths.

## What This Changes

The PR adds half-block Unicode QR/block-art detection plus chat/sidebar CSS so code output can render as tight black-and-white tiles in the web UI.

## Best Possible Solution

Land a retargeted web UI fix on current main with shared half-block detection in the active markdown-it renderers, raw-output handling before markdown parsing, consistent chat/sidebar CSS inheritance and overflow, focused regression tests, and browser proof.

## Reproduction Assessment

Yes, source-reproducible: current main lacks half-block detection in the active code-block renderers and raw transcript text goes directly through markdown parsing. I did not run a live browser repro in this read-only review.

## Solution Assessment

No. The PR's direction is right, but the current branch patches an old renderer seam and only partially handles raw output, so the fix should move to the current markdown-it and chat/sidebar paths.

Full review comments:

- **[P2] Retarget the wrapper to the active markdown-it renderers:** `ui/src/ui/markdown.ts:215-219`
  - body: This branch adds `.half-block-art` inside `htmlEscapeRenderer.code()`, but current main renders fenced and indented code through `md.renderer.rules.fence` and `md.renderer.rules.code_block`.   - confidence: 0.9

  - body: The new half-block fencing is limited to `formatToolOutputForSidebar()`. Regular assistant/tool message text still goes straight to `toSanitizedMarkdownHtml(markdown)`, so an unfenced multiline QR in the main transcript renders as normal markdown and never receives `.half-block-art` styling.
  - confidence: 0.88
- **[P3] Inherit the sidebar glyph font size:** `ui/src/styles/chat/sidebar.css:128-134`
  - body: The sidebar half-block `pre code` override does not set `font-size: inherit`, so existing sidebar code font sizing can still control the rendered QR glyphs instead of the intended enlarged half-block `pre` size.
  - confidence: 0.78
