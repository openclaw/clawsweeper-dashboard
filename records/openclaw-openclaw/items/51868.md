## What This Changes

The branch adds half-block-art detection and styling for WebChat markdown code blocks and sidebar tool output so UTF-8 QR block art renders without row gaps.

## Best Possible Solution

Land a WebChat fix that classifies half-block QR/art before markdown rendering for fenced code, indented code, raw transcript text, BTW side results, and sidebar tool output, while preserving QR quiet-zone whitespace and normal code rendering.

## Reproduction Assessment

Yes, at source level: current main has no half-block classifier and raw visible text paths flow directly into markdown-it rendering. I did not run a live browser repro in this read-only review, but the PR screenshots show the visual failure and intended improvement.

## Solution Assessment

No, not yet. The code-block and sidebar direction is maintainable, but the fix should also cover raw transcript and BTW side-result text and finish the sidebar font-size override before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Handle raw half-block text before Markdown parsing:** `ui/src/ui/markdown.ts:559-560`
  - body: This branch classifies half-block art only inside fenced/indented code blocks and sidebar formatting. Regular transcript text and BTW side results still call `toSanitizedMarkdownHtml()` directly, so a tool or model that emits the QR text without triple backticks will render as paragraphs/line breaks and never receive `.half-block-art`.
  - confidence: 0.9
- **[P3] Inherit the sidebar half-block code font size:** `ui/src/styles/chat/sidebar.css:431-437`
  - body: The sidebar half-block `pre code` override resets color, line-height, font family, background, and padding, but not `font-size`. Existing sidebar code rules can still size the glyphs independently, so sidebar QR output can render smaller than the half-block-art `pre` intends.
  - confidence: 0.82
