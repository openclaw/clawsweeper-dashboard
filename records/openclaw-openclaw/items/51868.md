## Summary

Keep this PR open: the user-visible fix is useful and supported by screenshots, but the current branch is not mergeable and still misses the active current-main rendering paths.

## What This Changes

The branch adds half-block Unicode QR/block-art detection, sidebar raw-output fencing, and chat/sidebar CSS for tight black-on-white code-block rendering.

## Best Possible Solution

Port the half-block detector into the current markdown-it fence/code_block renderers and shared raw-output preprocessing, then keep the chat/sidebar CSS consistent and visually verify QR scanability.

## Reproduction Assessment

Yes. Source inspection shows current main lacks half-block detection/styles and sends raw transcript and side-result text through generic markdown rendering; the PR screenshots show the affected web UI before and after.

## Solution Assessment

No. The visual direction is right, but this branch patches the stale `htmlEscapeRenderer.code()` path and only fences sidebar tool output, so the fix should move to the active renderers and raw message paths.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Move detection into the active markdown-it renderers:** `ui/src/ui/markdown.ts:216-222`
  - body: This adds `.half-block-art` inside `htmlEscapeRenderer.code()`, but current main emits fenced and indented code through markdown-it `fence` and `code_block` renderer rules. After conflict resolution, QR code blocks would still get the generic wrapper unless the detector moves to those active hooks.
  - confidence: 0.91

  - confidence: 0.88
- **[P3] Inherit the sidebar half-block code font size:** `ui/src/styles/chat/sidebar.css:128-134`
  - body: The sidebar override inherits color and line-height but leaves the existing sidebar code font sizing in effect. Add `font-size: inherit` so the rendered glyphs actually use the larger half-block `<pre>` size.
  - confidence: 0.78
