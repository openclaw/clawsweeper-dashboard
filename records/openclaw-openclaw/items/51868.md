## Summary

Keep open: current main still lacks half-block QR rendering support, but this PR is not merge-ready because it conflicts with the markdown-it renderer migration and still misses raw transcript and side-result paths. The PR screenshots are useful real behavior proof, and the remaining repair is narrow.

## What This Changes

The PR adds half-block QR/block-art detection, wraps detected sidebar tool output in code fences, and adds chat/sidebar CSS for tight black-and-white code-block rendering.

## Best Possible Solution

Implement half-block detection in the current markdown-it fence/code_block renderers plus raw transcript, sidebar, and side-result preprocessing, preserving QR whitespace with focused tests and browser proof.

## Reproduction Assessment

Yes. Source inspection shows current main uses generic markdown-it code-block rendering with no half-block styling, and the PR screenshots provide a clear before/after path for the broken QR rendering.

## Solution Assessment

No. The visual direction is sound, but the branch patches an obsolete renderer path and only partially handles raw QR output; the maintainable fix belongs in the current markdown-it renderer and all raw markdown surfaces.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Apply half-block detection in active renderers:** `ui/src/ui/markdown.ts:216-222`
  - body: This adds `.half-block-art` from the old `htmlEscapeRenderer.code()` path, but current main renders fenced and indented code through markdown-it `fence` and `code_block` rules. After rebase, QR code blocks still get the generic wrapper unless the detector moves to those active hooks.
  - confidence: 0.9
- **[P2] Handle raw QR replies before Markdown parsing:** `ui/src/ui/chat/tool-helpers.ts:25-29`
  - body: The new fencing only runs in the sidebar tool-output helper. Main assistant/tool transcript text and side-result text still go straight to `toSanitizedMarkdownHtml`, so an unfenced multiline QR reply remains paragraph text and never receives `.half-block-art`.
  - confidence: 0.88
- **[P3] Inherit the sidebar half-block code font size:** `ui/src/styles/chat/sidebar.css:128-134`
  - body: The sidebar override inherits color and line-height but leaves the current `pre code` font-size rule in effect, so the glyphs do not use the larger half-block `<pre>` size. Add `font-size: inherit` for consistent QR tiling.
  - confidence: 0.78
