## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Detect raw block art before Markdown parses it:** `ui/src/ui/markdown.ts:557-560`
  - body: The new half-block branch only runs after Markdown has produced a fenced or indented code token. Raw multiline QR text in normal assistant/tool messages is still parsed as paragraph text, so it never receives `.half-block-art` or the tight QR styling.
  - confidence: 0.84
- **[P2] Style expanded raw tool-card block art:** `ui/src/ui/chat/tool-helpers.ts:27-30`
  - body: Wrapping detected output in `formatToolOutputForSidebar()` helps sidebar Markdown, but expanded raw tool cards still render raw output directly as `<pre><code>`. Those expanded QR outputs keep the old spacing and remain unscannable.
  - confidence: 0.86
- **[P2] Inherit sidebar code font size for block art:** `ui/src/styles/chat/sidebar.css:429-436`
  - body: The sidebar half-block override resets color, line-height, background, and padding, but it does not reset `font-size`. The existing sidebar `pre code` font-size can still override the enlarged QR `pre`, making sidebar QR output smaller than intended.
  - confidence: 0.78
