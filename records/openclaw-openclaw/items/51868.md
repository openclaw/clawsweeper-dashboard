## Summary

Keep open. Current main still lacks half-block QR handling, and this PR is useful but not merge-ready: it handles fenced/code-block paths only, misses raw assistant/tool text in the main transcript, and needs to be ported to the current markdown-it renderer surface before landing.

## What This Changes

The PR adds half-block Unicode art detection, wraps detected markdown/code output with a half-block CSS class, and adds chat/sidebar/code-block styling intended to render UTF-8 QR blocks without row gaps.

## Best Possible Solution

Land a current-main implementation that detects half-block art before markdown parsing where needed, supports fenced and indented code blocks plus sidebar tool output, preserves QR quiet-zone whitespace, and applies explicit QR-safe styling with focused markdown/helper/style tests.

## Reproduction Assessment

Yes. The PR body gives concrete web chat steps and screenshots, and current-main source inspection shows half-block QR text still flows through generic markdown and CSS paths.

## Solution Assessment

No. The requested direction is appropriate, but this PR as written is incomplete for raw chat output and stale against the current renderer; the safer path is a small rework on the markdown-it and chat/sidebar helper surfaces.

  - confidence: 0.78
