## Summary

Keep open: current main still lacks half-block QR handling, and this PR needs retargeting to the current markdown-it and chat rendering paths before it can merge.

## What This Changes

The PR adds half-block Unicode art detection, sidebar output fencing, and chat/sidebar CSS so UTF-8 QR-like block text can render as tight black-and-white tiles.

## Best Possible Solution

Retarget the fix to current main with a shared half-block detector used by markdown-it fenced and indented code renderers, raw chat/sidebar output handling that preserves QR whitespace, consistent CSS overrides, focused regression tests, and visual QR proof.

## Reproduction Assessment

Yes, source-reproducible. Current main has no half-block detector or styling, and the active chat/sidebar paths still render QR-like text through generic markdown and code CSS; final browser scanability proof is still needed after the repair.

## Solution Assessment

No. The direction is right, but this branch targets the removed htmlEscapeRenderer path and only partially covers raw transcript/sidebar behavior, so the maintainable fix needs to land on the current markdown-it renderers and chat output paths.
