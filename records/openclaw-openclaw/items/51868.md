## Summary

Keep open: the visual bug is credible and the PR has useful proof, but the branch is now stale against current `main` and still misses active markdown-it/raw transcript paths, so it is not merge-ready as written.

## What This Changes

The PR adds half-block QR/block-art detection, code-block wrapping, and CSS styling for Control UI chat and sidebar output.

## Best Possible Solution

Retarget the QR rendering support to current markdown-it code renderers plus raw transcript/sidebar/side-result paths, preserve QR whitespace, keep normal code blocks unchanged, and add focused tests with browser proof.

## Reproduction Assessment

Yes, source inspection plus the PR screenshots give a high-confidence path: current `main` has no half-block detector or styling, and markdown/sidebar CSS still renders QR text through ordinary code-block rules. I did not run a live browser reproduction in this read-only pass.

## Solution Assessment

No. The visual direction is reasonable, but this branch patches an obsolete renderer path and only partially handles raw QR output; the narrow maintainable fix belongs in the current markdown-it renderer and transcript/sidebar surfaces.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Move half-block detection into active renderers:** `ui/src/ui/markdown.ts:215-219`
  - body: This adds `.half-block-art` from the old `htmlEscapeRenderer.code()` path, but current `main` renders fenced and indented code through markdown-it `fence` and `code_block` rules. After rebasing, QR code blocks would still get the generic wrapper unless the detector moves to those active hooks.
  - confidence: 0.9
- **[P2] Handle raw QR transcript text before Markdown parsing:** `ui/src/ui/chat/tool-helpers.ts:25-29`
  - body: The new fencing only runs in `formatToolOutputForSidebar()`. Main assistant/tool transcript text still goes straight through `toSanitizedMarkdownHtml(markdown)`, so an unfenced multiline QR reply remains paragraph text and never receives `.half-block-art`.
  - confidence: 0.88
- **[P3] Inherit the half-block font size in sidebar code:** `ui/src/styles/chat/sidebar.css:128-134`
  - body: The sidebar override inherits color and line-height but leaves the existing `pre code` font-size rule in effect, so the glyphs do not use the larger half-block `<pre>` size. Add `font-size: inherit` for consistent QR tiling.
  - confidence: 0.78
