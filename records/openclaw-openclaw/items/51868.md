## Summary

Keep open: this PR addresses a real web UI rendering problem and includes useful screenshot proof, but the branch is not merge-ready because current main renders code blocks through markdown-it renderer rules and raw chat/BTW half-block output still bypasses the sidebar-only fencing path.

## What This Changes

The PR adds half-block-art detection for UI markdown code blocks, chat/sidebar CSS for UTF-8 QR rendering, and sidebar tool-output fencing for detected half-block text.

## Best Possible Solution

Land a retargeted Control UI fix on current main that handles fenced, indented, and raw half-block QR output across chat, tool sidebar, and BTW surfaces while preserving sanitizer behavior and QR quiet-zone whitespace.

## Reproduction Assessment

Yes, at source level: current main has no half-block detector in the active markdown-it code renderers, and raw transcript/BTW text flows directly into `toSanitizedMarkdownHtml()`. I did not run a live browser repro in this read-only review.

## Solution Assessment

No. The styling direction is reasonable, but this branch should be retargeted to the current markdown-it renderer and raw-output preprocessing paths, with the sidebar code font-size override fixed.

  - confidence: 0.88

  - body: The sidebar half-block `pre code` rule inherits color and line-height but leaves the existing `.sidebar-markdown :where(pre code)` font-size override active, so QR glyphs render smaller than the intended half-block `pre` size. Add `font-size: inherit`.
  - confidence: 0.78
