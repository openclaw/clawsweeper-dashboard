## Summary

Keep open: the user-visible QR rendering problem is valid and the branch has useful proof, but the patch targets an obsolete markdown renderer path and still misses raw transcript QR output on current main.

## What This Changes

The branch adds half-block Unicode detection, sidebar tool-output fencing, and chat/sidebar CSS so QR/block art code blocks render as tight black-and-white tiles.

## Best Possible Solution

Land a retargeted current-main UI fix with shared half-block detection in the active markdown-it renderers, raw chat/sidebar output handling that preserves QR quiet-zone whitespace, consistent CSS inheritance/overflow, focused regression tests, and browser proof.

## Reproduction Assessment

Yes, source-reproducible. Current main has no half-block detector/styling in the active markdown-it code-block renderers and sends raw transcript markdown through the generic sanitizer path.

## Solution Assessment

No. The direction is right, but this branch patches the old renderer shape and only partially handles raw output, so the maintainable fix should move the detector to current markdown-it hooks and the main transcript/sidebar paths.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Retarget half-block wrapping to markdown-it renderers:** `ui/src/ui/markdown.ts:215-219`
  - body: This adds `.half-block-art` from `htmlEscapeRenderer.code()`, but current main renders fenced and indented code through `md.renderer.rules.fence` and `code_block`. After rebase, fenced QR blocks will keep the generic wrapper unless the detector moves to those active hooks.
  - confidence: 0.9
- **[P2] Handle raw transcript QR output before Markdown parsing:** `ui/src/ui/chat/tool-helpers.ts:25-29`
  - body: The new fencing only runs in `formatToolOutputForSidebar()`. Regular assistant/tool transcript text still goes directly through `toSanitizedMarkdownHtml(markdown)`, where markdown-it renders unfenced multiline QR art as paragraphs with `<br>`s, so the main chat case described in the PR remains unstyled.
  - confidence: 0.88
- **[P3] Inherit the sidebar half-block code font size:** `ui/src/styles/chat/sidebar.css:128-134`
  - body: The sidebar override inherits color and line-height but leaves the `<code>` font size controlled by existing sidebar code rules, so the glyphs do not use the `1.3em` size set on the half-block `<pre>`. Add `font-size: inherit` to keep sidebar QR tiling consistent with chat.
  - confidence: 0.78
