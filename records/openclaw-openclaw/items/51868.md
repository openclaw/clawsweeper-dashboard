## Summary

Keep open; this is a plausible UI bug with after-fix screenshot proof, but the branch still needs changes because it targets obsolete renderer plumbing, misses raw transcript output, and leaves a sidebar font-size inheritance bug.

## What This Changes

The PR adds a half-block-art detector, sidebar output fencing, and CSS intended to render UTF-8 QR/block-art code blocks as tight black-and-white tiles in the web chat and sidebar.

## Best Possible Solution

Land a current-main UI fix that detects half-block art in the markdown-it code renderers and raw chat/sidebar text paths, preserves QR quiet-zone whitespace, keeps normal code blocks unchanged, and includes focused tests plus browser proof.

## Reproduction Assessment

Yes, source-reproducible. Current main has no half-block detector/styling in the active code-block renderers, and raw chat transcript text still flows directly into the sanitizer/render pipeline.

## Solution Assessment

No. The visual direction is good, but the branch hooks an obsolete renderer path and only partially handles raw output; the safer solution is to retarget the same behavior to current markdown-it and transcript/sidebar paths.

- **[P2] Retarget half-block wrapping to active renderers:** `ui/src/ui/markdown.ts:215-219`
  - body: This adds `.half-block-art` from `htmlEscapeRenderer.code()`, but current main renders fenced and indented code through markdown-it `fence` and `code_block` rules. After retargeting, fenced QR blocks will still get the generic wrapper unless the detector moves to those active hooks.

  - body: The sidebar override inherits color and line-height but leaves the existing `pre code` font-size in effect, so the glyphs do not use the `1.3em` size set on the half-block `<pre>`. Add `font-size: inherit` for consistent QR tiling.
  - confidence: 0.78
