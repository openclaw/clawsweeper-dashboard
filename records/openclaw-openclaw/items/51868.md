## Summary

Keep open: the reported Web UI rendering problem is still valid on current main, but this PR is stale against the markdown-it renderer path and still leaves raw transcript and glyph-size gaps before it can merge.

## What This Changes

The PR adds a half-block-art detector, fences sidebar tool output, and adds chat/sidebar CSS intended to render UTF-8 QR-style blocks as tightly tiled black-and-white code blocks.

## Best Possible Solution

Land a small current-main Web UI fix that detects half-block art in the active markdown and raw transcript/sidebar paths, preserves QR quiet-zone whitespace, and keeps normal code-block sanitization/rendering unchanged.

## Reproduction Assessment

Yes, source-reproducible. The PR provides concrete web chat steps and screenshots, and current main still routes half-block text through generic markdown/CSS paths with no QR-safe wrapper or styling.

## Solution Assessment

No. The approach is directionally right, but the current patch is not the best current-main fix because it targets an obsolete renderer and does not cover unfenced transcript output or inner-code font inheritance.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Move QR detection into the active renderers:** `ui/src/ui/markdown.ts:215-219`
  - body: Current main no longer uses `htmlEscapeRenderer.code`; fenced and indented blocks are rendered by markdown-it `fence` and `code_block` rules. As written, this branch will not apply `.half-block-art` after rebase, so QR code blocks will still render generically.
  - confidence: 0.9
- **[P2] Handle raw chat QR output before Markdown parsing:** `ui/src/ui/chat/tool-helpers.ts:23-25`
  - body: The new helper only fences sidebar output. Regular assistant/tool text in the main transcript still goes straight through `toSanitizedMarkdownHtml(markdown)`, so unfenced multiline QR art remains a paragraph/line-break render with the old spacing.
  - confidence: 0.88
- **[P3] Inherit the chat QR glyph size:** `ui/src/styles/chat/text.css:169-175`
  - body: The half-block `<pre>` sets a larger font size, but the inner `<code>` still matches existing chat code font-size rules. Add `font-size: inherit` to this half-block inner-code rule so the rendered glyphs actually use the QR sizing.
  - confidence: 0.78
- **[P3] Inherit the sidebar QR glyph size:** `ui/src/styles/chat/sidebar.css:128-134`
  - body: The sidebar half-block `<pre>` also sets a larger font size, but the inner `<code>` rule does not override the sidebar `pre code` font size. Add `font-size: inherit` here to keep sidebar QR output consistent with the intended tiling.
  - confidence: 0.8
