## Summary

Keep open: current main still lacks half-block QR/block-art handling, and this PR is a useful narrow UI fix, but the branch is conflicting and its patch needs retargeting to current markdown-it/chat rendering paths before merge.

## What This Changes

The PR adds half-block-art detection and black-on-white tight-line-height styling for UTF-8 QR/block-art code blocks in Control UI chat and sidebar output.

## Best Possible Solution

Land a retargeted UI fix that applies half-block detection in the active markdown-it code renderers and shared raw-output preprocessing while keeping chat and sidebar CSS consistent.

## Reproduction Assessment

Yes, source-level: current main has no half-block handling, uses markdown-it fence/code_block renderers for code, and sends raw transcript and side-result text straight to the sanitizer. I did not run a live browser repro in this read-only review, but the PR screenshots show the visible before/after behavior.

## Solution Assessment

No. The visual direction is right, but this branch targets a stale renderer and only fences sidebar tool output, so the maintainable fix should move into current active renderers and shared raw-output paths.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Retarget half-block detection to markdown-it renderers:** `ui/src/ui/markdown.ts:216-222`
  - body: This adds `.half-block-art` inside the old `htmlEscapeRenderer.code()` path, but current main renders fenced and indented code through `md.renderer.rules.fence` and `md.renderer.rules.code_block`. After resolving the branch drift, QR code blocks still need the detector in those active renderer rules.
  - confidence: 0.91
- **[P2] Fence raw half-block replies before markdown parsing:** `ui/src/ui/chat/tool-helpers.ts:25-29`
  - body: The new fencing only runs in `formatToolOutputForSidebar()`. Main assistant/tool transcript text and BTW side-result text still go directly to `toSanitizedMarkdownHtml()`, so unfenced multiline QR output remains paragraph text and never receives `.half-block-art`.
  - confidence: 0.88
- **[P3] Inherit the sidebar half-block code font size:** `ui/src/styles/chat/sidebar.css:128-134`
  - body: The sidebar override inherits color and line-height but leaves `.sidebar-markdown code { font-size: 0.88em; }` active on the inner `<code>`, shrinking the glyphs below the intended half-block `<pre>` size. Add `font-size: inherit` to keep sidebar rendering consistent.
  - confidence: 0.78
