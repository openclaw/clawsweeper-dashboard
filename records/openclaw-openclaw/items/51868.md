## Summary

Keep open: current main still lacks QR-safe half-block rendering, but this PR is not merge-ready because the branch targets an obsolete markdown renderer path and still misses raw transcript output, sidebar font inheritance, and the required changelog entry.

## What This Changes

The PR adds half-block-art detection, sidebar tool-output fencing, and chat/sidebar CSS intended to render UTF-8 QR-style Unicode blocks as tight black-and-white tiles.

## Best Possible Solution

Land a rebased Web UI patch that handles fenced, indented, raw chat, and sidebar tool half-block output through the active markdown-it/rendering paths while preserving whitespace, sanitization, tests, and changelog coverage.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes: render a multiline UTF-8 half-block QR code in web chat or sidebar output; current main routes it through generic markdown/CSS paths with no `.half-block-art` handling, and the PR body includes concrete before/after screenshots.

## Solution Assessment

Is this the best way to solve the issue? No. The direction is right, but the patch must move detection into the active markdown-it renderers and cover raw transcript output plus the sidebar inner-code sizing before merge.
