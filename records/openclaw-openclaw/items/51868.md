## What This Changes

The PR adds half-block QR/block-art detection and styling for Control UI chat/sidebar markdown, plus sidebar tool-output fencing for detected half-block text.

## Best Possible Solution

Land a retargeted Control UI fix on current main that handles active fenced/indented renderers plus raw assistant/tool/BTW output, preserves QR quiet-zone whitespace, and keeps chat/sidebar CSS consistent.

## Reproduction Assessment

Yes, at source level: current main has no half-block handling in the active markdown-it code renderers, and raw transcript/BTW text still goes directly through toSanitizedMarkdownHtml. I did not run a live browser repro in this read-only pass.

## Solution Assessment

No. The styling direction is right, but this branch should be retargeted to current main's fence/code_block renderers and shared raw-output preprocessing instead of the stale renderer and sidebar-only path.

  - body: The sidebar half-block `pre code` rule inherits color and line-height but leaves the existing sidebar `pre code` font-size override active, so QR glyphs render smaller than the intended half-block `pre` size. Add `font-size: inherit`.
  - confidence: 0.78
