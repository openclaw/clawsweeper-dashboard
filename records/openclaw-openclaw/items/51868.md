## Summary

Keep open: the user-visible QR rendering problem still appears valid, but this PR head is not correct against current main because it patches the retired renderer path and misses active raw transcript rendering.

## What This Changes

The PR adds a half-block-art detector, fences sidebar tool output, and applies tight black/white code-block styling for UTF-8 QR rendering in the web UI.

## Best Possible Solution

Retarget the QR rendering support to the current markdown-it renderers and raw transcript/sidebar paths, preserve QR whitespace, keep ordinary code blocks unchanged, and add focused tests plus browser proof.

## Reproduction Assessment

Yes. Source inspection plus the supplied before screenshot give a high-confidence path: current main has no half-block detector/styling, and raw assistant/tool transcript text still flows directly through normal markdown parsing.

## Solution Assessment

No. The visual direction is useful, but this PR patches a retired renderer and only partially handles raw output; the maintainable fix should be implemented in the current markdown-it and transcript/sidebar surfaces.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Move half-block detection into markdown-it renderers:** `ui/src/ui/markdown.ts:215-219`
  - body: This adds `.half-block-art` from the old `htmlEscapeRenderer.code()` path, but current main renders fenced and indented code through markdown-it `fence` and `code_block` rules. Fenced QR blocks will still get the generic wrapper after rebasing unless the detector moves to those active hooks.
  - confidence: 0.9
- **[P2] Fence raw QR transcript text before markdown parsing:** `ui/src/ui/chat/tool-helpers.ts:25-29`
  - body: The new fencing only runs in `formatToolOutputForSidebar()`. Main assistant/tool transcript text still goes straight through `toSanitizedMarkdownHtml(markdown)`, so an unfenced multiline QR reply remains paragraph text and never receives `.half-block-art`.
  - confidence: 0.88
- **[P3] Inherit the sidebar code font size:** `ui/src/styles/chat/sidebar.css:128-134`
  - body: The sidebar override inherits color and line-height but leaves the existing `pre code` font-size rule in effect, so the glyphs do not use the `1.3em` size set on the half-block `<pre>`. Add `font-size: inherit` for consistent QR tiling.
  - confidence: 0.78
