## Summary

Keep open: this is a plausible focused web UI rendering fix with sufficient visual proof, but the current branch still has correctness blockers against current main because it targets the old markdown renderer path and misses raw transcript output.

## What This Changes

The branch adds half-block-art detection in the web markdown renderer, sidebar tool-output fencing, and CSS to style QR/block-art code blocks in chat and sidebar.

## Best Possible Solution

Retarget the QR rendering support to the current markdown-it fence/code_block renderers plus raw transcript/sidebar/side-result paths, preserve QR quiet-zone whitespace, and keep normal code blocks unchanged.

## Reproduction Assessment

Yes. Source inspection plus the PR screenshots give a high-confidence path: current main has no half-block detector or styling, and code blocks/raw transcript text still render through ordinary markdown/CSS rules.

## Solution Assessment

No. The visual direction is reasonable, but this branch patches an obsolete renderer path and only partially handles raw QR output; the maintainable fix belongs in the current markdown-it renderer and transcript/sidebar surfaces.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Move half-block detection into active renderers:** `ui/src/ui/markdown.ts:216-219`
  - body: This adds `.half-block-art` from the old `htmlEscapeRenderer.code()` path, but current main renders fenced and indented code through markdown-it `fence` and `code_block` rules. After rebasing, QR code blocks would still get the generic wrapper unless the detector moves to those active hooks.
  - confidence: 0.9
- **[P2] Handle raw QR transcript text before Markdown parsing:** `ui/src/ui/chat/tool-helpers.ts:25-29`
  - body: The new fencing only runs in `formatToolOutputForSidebar()`. Main assistant/tool transcript text and side-result text still go straight through `toSanitizedMarkdownHtml()`, so an unfenced multiline QR reply remains paragraph text and never receives `.half-block-art`.
  - confidence: 0.88
- **[P3] Inherit the sidebar half-block code font size:** `ui/src/styles/chat/sidebar.css:128-134`
  - body: The sidebar override inherits color and line-height but leaves the existing `pre code` font-size rule in effect, so the glyphs do not use the larger half-block `<pre>` size. Add `font-size: inherit` for consistent QR tiling.
  - confidence: 0.78
