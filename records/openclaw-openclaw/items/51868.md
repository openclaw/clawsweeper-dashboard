## Summary

This PR should stay open: it addresses a real web UI rendering problem, but the branch is currently conflicting/uneditable and the patch still misses current-main markdown rendering paths.

## What This Changes

The PR adds half-block QR/art detection in Control UI markdown rendering plus chat/sidebar CSS for tight black-on-white UTF-8 block-art code blocks.

## Best Possible Solution

Land a retargeted Control UI fix that detects half-block art in the active markdown-it code renderers and shared raw-output preprocessing paths, with consistent chat/sidebar CSS and browser proof.

## Reproduction Assessment

Yes, source-level: current main has no half-block handling in the active markdown-it code renderers, and raw transcript/BTW text flows directly into `toSanitizedMarkdownHtml`. The PR body also includes before/after screenshots showing the visible web UI rendering problem and intended improvement.

## Solution Assessment

No. The visual styling direction is reasonable, but the patch should target current main's markdown-it renderers and shared raw-output paths instead of the stale renderer/sidebar-only helper path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Retarget half-block detection to markdown-it renderers:** `ui/src/ui/markdown.ts:216-222`
  - body: This adds `.half-block-art` inside the old `htmlEscapeRenderer.code()` path, but current main renders fenced and indented code through `md.renderer.rules.fence` and `md.renderer.rules.code_block`. After resolving branch drift, the detector needs to run in those active renderer rules or fenced QR blocks will still render normally.
  - confidence: 0.91
- **[P2] Fence raw half-block replies before markdown parsing:** `ui/src/ui/chat/tool-helpers.ts:25-29`
  - body: The new fencing only runs in `formatToolOutputForSidebar()`. Main assistant/tool transcript text and BTW side-result text still go directly to `toSanitizedMarkdownHtml()`, so unfenced multiline QR output remains paragraph text and never receives `.half-block-art`.
  - confidence: 0.88
- **[P3] Inherit the sidebar half-block code font size:** `ui/src/styles/chat/sidebar.css:128-134`
  - body: The sidebar override inherits color and line-height but leaves the existing `.sidebar-markdown code` and `pre code` font sizing active on the inner `<code>`, shrinking the glyphs below the intended half-block `<pre>` size. Add `font-size: inherit` to keep sidebar rendering consistent.
  - confidence: 0.78
