## Summary

Keep this PR open: the reported web UI rendering gap is still present on current main, but the branch is conflicting and the patch still misses current render paths that would leave important QR cases unstyled.

## What This Changes

The PR adds web UI half-block QR/block-art detection, chat/sidebar CSS, and sidebar tool-output fencing for Unicode block QR output.

## Best Possible Solution

Implement half-block handling in the current markdown-it fence/code_block renderers and a shared raw-output preprocessing path for transcript, sidebar, and side-result surfaces, with CSS that preserves whitespace, inherited code sizing, and horizontal overflow.

## Reproduction Assessment

Yes. Source inspection shows current main renders code blocks and raw message text without half-block detection or styling, and the PR body provides visible before/after web UI screenshots for the same user flow.

## Solution Assessment

No. The visual direction is right, but the branch patches an obsolete renderer path and only handles the sidebar raw-output helper; the safer fix is to move detection into the current markdown-it and shared raw markdown surfaces.
