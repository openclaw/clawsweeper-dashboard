## Summary

Keep open. This is a valid Control UI rendering bug with sufficient screenshot proof, but the branch is not mergeable as-is: it targets an obsolete code-renderer path and still misses raw chat/BTW output plus one sidebar font-size detail.

## What This Changes

The PR adds Unicode half-block QR/art detection and styling for Control UI chat/sidebar code blocks, plus sidebar tool-output fencing for detected half-block text.

## Best Possible Solution

Land a retargeted Control UI fix that handles half-block art in the active markdown-it renderers and shared raw-output preprocessing paths, with consistent chat/sidebar CSS and visual browser proof.

## Reproduction Assessment

Yes, source-level. Current main's markdown-it renderers lack half-block handling and raw chat/BTW paths pass directly to `toSanitizedMarkdownHtml`, matching the broken before screenshots; I did not run a live browser repro in this read-only pass.

## Solution Assessment

No. The styling direction is reasonable, but the patch should target current main's markdown-it fence/code_block renderers and shared raw-output paths, then keep sidebar CSS inheritance consistent.

- **[P2] Retarget half-block detection to active renderers:** `ui/src/ui/markdown.ts:215-222`

  - confidence: 0.88

  - confidence: 0.78
