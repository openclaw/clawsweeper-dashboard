## Summary

Keep open. The QR rendering problem is credible and the screenshots show the intended UI outcome, but this branch is not ready because it targets the pre-current markdown renderer and still leaves raw transcript and sidebar font-size paths incomplete.

## What This Changes

The PR adds half-block-art detection for Unicode QR/block-art code blocks plus chat/sidebar CSS and sidebar tool-output fencing so those blocks render as tight black-and-white tiles.

## Best Possible Solution

Land a retargeted UI fix on current main with shared half-block detection, raw assistant/tool output handling, consistent chat/sidebar CSS inheritance, focused regression tests, and browser QR proof.

## Reproduction Assessment

Yes, source-reproducible. Current main renders code through generic markdown-it wrappers and raw transcript markdown through `toSanitizedMarkdownHtml`, while the PR screenshots show the before/after QR rendering problem.

## Solution Assessment

No. The direction is right, but the patch should move the detector to the current markdown-it fence/code_block renderers, handle raw transcript QR text before parsing, and make sidebar CSS inherit the intended glyph size.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Retarget half-block wrapping to markdown-it renderers:** `ui/src/ui/markdown.ts:215-219`
  - body: This adds `.half-block-art` inside `htmlEscapeRenderer.code()`, but current main renders fenced and indented code through `md.renderer.rules.fence` and `md.renderer.rules.code_block`. After rebase, QR code blocks will keep the generic wrapper unless the detector moves to those active hooks.
  - confidence: 0.9
- **[P2] Fence raw transcript QR text before markdown parsing:** `ui/src/ui/chat/tool-helpers.ts:25-29`
  - body: The new half-block fencing only runs in `formatToolOutputForSidebar()`. Regular assistant/tool message text still goes straight to `toSanitizedMarkdownHtml(markdown)`, so unfenced multiline QR output in the main transcript becomes ordinary markdown and never receives `.half-block-art` styling.
  - confidence: 0.88
- **[P3] Inherit the sidebar half-block code font size:** `ui/src/styles/chat/sidebar.css:128-134`
  - body: The sidebar `pre code` override does not set `font-size: inherit`. Current sidebar CSS gives `pre code` an absolute font size, so the QR glyphs can ignore the enlarged half-block `pre` size and render smaller than the chat surface.
  - confidence: 0.78
