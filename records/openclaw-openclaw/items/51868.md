## Summary

Keep open. Current main still lacks the requested half-block QR rendering behavior, and the PR contains useful source work, but the proposed patch does not fully cover raw assistant/tool transcript output and needs current-main maintainer follow-up.

## What This Changes

The PR adds half-block Unicode art detection in the UI markdown renderer, wraps detected sidebar tool output in a code fence, and adds chat/sidebar CSS intended to render UTF-8 QR blocks without row gaps.

## Best Possible Solution

Land or replace this with a current-main implementation that detects half-block art before markdown parsing for raw chat text, handles fenced/indented code blocks and sidebar tool output, preserves QR quiet-zone whitespace, applies explicit QR-safe styling in chat and sidebar, and adds focused markdown/tool-helper/style tests.

## Reproduction Assessment

Yes. The PR body includes concrete steps and screenshots, and source inspection confirms current main sends raw assistant/tool markdown through normal rendering while code blocks and sidebar text lack half-block-specific styling.

## Solution Assessment

No. The proposed direction is narrow and appropriate, but this patch is incomplete because raw assistant/tool QR text still bypasses detection and the diff needs rework for the current markdown-it renderer.

## Review Findings

  - confidence: 0.78
