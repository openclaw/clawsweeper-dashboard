## Summary

Keep open: this is a useful Web UI bug fix, but the branch is not correct against current main because it targets the old markdown renderer and misses raw chat/BTW paths. A narrow current-main repair or replacement PR is the right path.

## What This Changes

Adds half-block-art detection, sidebar tool-output fencing, wrapper overflow handling, and chat/sidebar CSS so UTF-8 half-block QR code blocks render as tight black/white art.

## Best Possible Solution

Land a current-main UI fix that detects half-block art in markdown-it fence/code_block renderers and pre-fences raw chat, sidebar tool output, and BTW text while preserving QR quiet-zone whitespace and normal code behavior.

## Reproduction Assessment

Yes, at source level: current main has no half-block-art handling in the active markdown-it code renderers, and raw transcript/BTW text still goes directly to toSanitizedMarkdownHtml(). I did not run a live browser repro.

## Solution Assessment

No. The styling direction is reasonable, but the PR should be retargeted to current-main renderers and broadened to raw chat/BTW paths before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Retarget half-block detection to active renderers:** `ui/src/ui/markdown.ts:216-222`
  - body: This adds `.half-block-art` from `htmlEscapeRenderer.code()`, but current main renders fenced and indented code through `md.renderer.rules.fence` and `code_block`. After conflicts are resolved, QR fences would still use normal code-block markup unless the detector is wired into those active renderer rules.
  - confidence: 0.91
- **[P2] Fence raw half-block replies before Markdown parsing:** `ui/src/ui/chat/tool-helpers.ts:25-29`
  - body: The new half-block fencing only runs in `formatToolOutputForSidebar()`. Main transcript and BTW side-result text still call `toSanitizedMarkdownHtml()` directly, so unfenced multiline QR output is parsed as ordinary markdown and never receives `.half-block-art`.
  - confidence: 0.88
- **[P3] Inherit the sidebar half-block code font size:** `ui/src/styles/chat/sidebar.css:128-134`
  - body: The sidebar half-block `pre code` rule inherits color and line-height but leaves the existing `.sidebar-markdown :where(pre code)` font-size override active, so QR glyphs render smaller than the intended half-block `pre` size. Add `font-size: inherit`.
  - confidence: 0.78
