## Summary

Keep open: the web UI QR rendering problem is valid, but this PR still targets a stale markdown renderer path and leaves raw chat/BTW half-block output uncovered on current main.

## What This Changes

This PR adds half-block-art detection/styling to Control UI markdown code blocks and wraps detected sidebar tool output in a code fence.

## Best Possible Solution

Land a current-main Control UI fix that handles fenced, indented, and raw half-block QR output across chat, tool sidebar, and BTW surfaces while preserving sanitizer behavior and QR quiet-zone whitespace.

## Reproduction Assessment

Yes, at source level: current main has no half-block detector in the active markdown-it code renderers, and raw transcript/BTW text still flows straight into `toSanitizedMarkdownHtml()`. I did not run a live browser repro in this read-only review.

## Solution Assessment

No. The visual styling approach is reasonable, but this branch needs to be retargeted to the active renderer and raw-output paths before it solves the reported web UI behavior consistently.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Retarget half-block detection to active renderers:** `ui/src/ui/markdown.ts:216-222`
  - body: This adds `.half-block-art` from `htmlEscapeRenderer.code()`, but current main renders fenced and indented code through `md.renderer.rules.fence` and `code_block`. After resolving the branch against main, QR fences would still use normal code-block markup unless the detector is wired into those active renderer rules.
  - confidence: 0.91
- **[P2] Fence raw half-block replies before Markdown parsing:** `ui/src/ui/chat/tool-helpers.ts:25-29`
  - body: The new half-block fencing only runs in `formatToolOutputForSidebar()`. Main transcript and BTW side-result text still call `toSanitizedMarkdownHtml()` directly, so unfenced multiline QR output is parsed as ordinary Markdown and never receives `.half-block-art`.
  - confidence: 0.88
- **[P3] Inherit the sidebar half-block code font size:** `ui/src/styles/chat/sidebar.css:128-134`
  - body: The sidebar half-block `pre code` rule inherits color and line-height but leaves the existing `.sidebar-markdown :where(pre code)` font-size override active, so QR glyphs render smaller than the intended half-block `pre` size. Add `font-size: inherit`.
  - confidence: 0.78
