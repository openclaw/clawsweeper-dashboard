## Summary

Keep open: current main still lacks half-block QR rendering support, and the PR is a useful focused UI fix, but it needs repair before merge because the branch is stale against the current markdown-it renderer and still misses raw transcript QR output.

## What This Changes

The PR adds half-block-art detection, wraps detected tool/sidebar output in code fences, and adds chat/sidebar CSS for tight black-and-white UTF-8 QR rendering.

## Best Possible Solution

Port the QR-art handling onto current main’s markdown-it renderer and transcript/sidebar paths, add focused regression coverage, and keep the sanitization and normal code-block behavior unchanged.

## Reproduction Assessment

Yes, by source inspection: render a multi-line UTF-8 half-block QR in the Control UI chat/sidebar and current main will route it through normal markdown/code styling with no half-block class. The PR screenshots also show the before/after visual behavior, but I did not run the browser locally in this read-only review.

## Solution Assessment

No, not as currently written. The direction is reasonable, but the fix should be moved to the current markdown-it fence/code_block renderers and should cover raw transcript QR art before markdown parsing.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Handle raw QR replies before Markdown parsing:** `ui/src/ui/markdown.ts:215-217`
  - body: The new detection only runs from the code-block renderer, so a model or tool reply that emits the multiline QR text without triple backticks still goes through normal `toSanitizedMarkdownHtml(markdown)` rendering in the transcript. That path produces ordinary paragraphs/line breaks, so the same raw QR output described in the PR can remain unstyled and unscannable; detect/wrap half-block art before parsing for the transcript surfaces as well.
  - confidence: 0.88
- **[P3] Reset the glyph font size inside half-block blocks:** `ui/src/styles/chat/text.css:171-176`
  - body: The half-block CSS increases the `<pre>` font size, but the actual glyphs are rendered by the nested `<code>` element and existing `.chat-text`/`.sidebar-markdown` code font-size rules still apply because this new override does not set `font-size: inherit`. Add the font-size reset in the half-block `pre code` rules so chat and sidebar QR blocks render at the intended size consistently.
  - confidence: 0.78
