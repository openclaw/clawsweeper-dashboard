## Summary

Keep open. The updated branch now targets the active markdown-it code renderers and has credible screenshot proof, but raw chat and side-result half-block output can still bypass the classifier, and the sidebar code-size override remains incomplete.

## What This Changes

The PR adds half-block-art detection to WebChat markdown code rendering and sidebar tool formatting, plus chat/sidebar CSS for QR-like code blocks and related test mock updates.

## Best Possible Solution

Land a current-main WebChat fix that classifies half-block QR/art in fenced code, indented code, raw visible text paths, and sidebar tool output while preserving quiet-zone whitespace and normal code rendering.

## Reproduction Assessment

Yes, at source level: current main has no half-block classifier and raw visible text paths flow directly into `toSanitizedMarkdownHtml`. I did not run a live browser repro in this read-only review, but the PR screenshots show the visual failure and intended improvement.

## Solution Assessment

No, not yet. The code-block and sidebar direction is maintainable, but the fix should also cover raw transcript and side-result text and finish the sidebar font-size override before merge.
