Keep open: this is a useful WebChat rendering fix with strong screenshot proof, but the current branch still misses raw transcript and BTW side-result text paths, so unfenced half-block QR output can remain unstyled and unscannable after merge.

## What This Changes

The branch adds half-block-art detection to WebChat markdown code rendering, QR-oriented chat/sidebar CSS, sidebar tool-output fencing for detected block art, and related test mock updates.

## Best Possible Solution

Extend the WebChat markdown flow so half-block art is classified before Markdown parsing for fenced code, indented code, raw transcript text, BTW side results, and sidebar tool output while preserving QR quiet-zone whitespace and normal code rendering.

## Reproduction Assessment

Yes, at source level: current main has no half-block classifier and raw visible text paths still flow directly into toSanitizedMarkdownHtml. I did not run a live browser repro in this read-only review, but the PR screenshots show the visual failure and intended improvement.

## Solution Assessment

No, not yet. The code-block and sidebar-tool direction is maintainable, but the patch should also handle raw transcript and BTW side-result text before Markdown turns multiline QR art into paragraphs.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Handle raw half-block text before Markdown parsing:** `ui/src/ui/markdown.ts:559-560`
  - body: This branch only assigns `.half-block-art` from the fenced and indented code renderers, but regular chat text and BTW side results still call `toSanitizedMarkdownHtml(...)` directly. With `breaks: true`, an unfenced multiline QR becomes paragraph text with `<br>`s and never receives the QR styling, so the reported live-tool-output path can remain unscannable. Please classify or wrap raw half-block text before these Markdown paths render it.
  - confidence: 0.88
- **[P3] Inherit the half-block code font size:** `ui/src/styles/chat/sidebar.css:431-437`
  - body: The half-block `pre` sets the intended QR size, but the inner `code` element still inherits the normal sidebar code font-size rule because this reset does not set `font-size: inherit`. Add the font-size reset, and mirror it for the chat rule if needed, so sidebar QR blocks render at the intended size consistently.
  - confidence: 0.78
