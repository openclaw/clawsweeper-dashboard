## Summary

Keep open: the reported web UI rendering problem is valid and the PR has useful visual proof, but the branch still needs to be retargeted to current main because the active markdown/chat paths differ from the old renderer it patches.

## What This Changes

The PR adds half-block Unicode QR/block-art detection and chat/sidebar CSS so QR-like code output renders as tight black-and-white tiles in the Control UI.

## Best Possible Solution

Land a retargeted UI fix on current main with shared half-block detection, raw-output handling, consistent CSS inheritance/overflow, focused regression tests, and after-fix browser QR proof.

## Reproduction Assessment

Yes, source-reproducible. Current main lacks half-block detection/styling in the active markdown-it renderer and sends raw transcript markdown through the generic sanitizer, so the reported QR rendering path is clear from source inspection.

## Solution Assessment

No. The direction is right, but this branch patches an old renderer path and only partially handles raw output, so the maintainable fix should move the detector to current markdown-it hooks and the main chat/sidebar output paths.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Retarget the wrapper to the active markdown-it renderers:** `ui/src/ui/markdown.ts:215-219`
  - body: This branch adds `.half-block-art` inside `htmlEscapeRenderer.code()`, but current main renders fenced and indented code through `md.renderer.rules.fence` and `md.renderer.rules.code_block`. After rebase, QR code blocks will keep the generic wrapper unless the detector moves to those active hooks.
  - confidence: 0.9
- **[P2] Fence raw transcript QR text before markdown parsing:** `ui/src/ui/chat/tool-helpers.ts:19-25`
  - body: The new half-block fencing is limited to `formatToolOutputForSidebar()`. Regular assistant/tool message text still goes straight to `toSanitizedMarkdownHtml(markdown)`, so an unfenced multiline QR in the main transcript renders as normal markdown and never receives `.half-block-art` styling.
  - confidence: 0.88
- **[P3] Inherit the sidebar glyph font size:** `ui/src/styles/chat/sidebar.css:128-134`
  - body: The sidebar half-block `pre code` override does not set `font-size: inherit`, so the existing sidebar `pre code` font-size can still control the rendered QR glyphs instead of the intended enlarged half-block `pre` size.
  - confidence: 0.78
