Keep this PR open: it addresses a real Control UI rendering problem and has useful visual proof, but the branch is now dirty, uneditable by maintainers, and its detector is wired to a stale renderer path while raw chat/BTW output remains uncovered.

## What This Changes

Adds half-block-art detection to UI markdown code blocks, chat/sidebar CSS for UTF-8 QR rendering, and sidebar tool-output fencing for detected half-block text.

## Best Possible Solution

Land a retargeted Control UI fix on current main that handles fenced, indented, and raw half-block QR output across chat, tool sidebar, and BTW surfaces while preserving sanitizer behavior and QR quiet-zone whitespace.

## Reproduction Assessment

Yes, at source level: current main has no half-block detector in the active markdown-it code renderers, and raw transcript/BTW text still flows straight into `toSanitizedMarkdownHtml()`. I did not run a live browser repro in this read-only review.

## Solution Assessment

No. The styling direction is reasonable, but this branch targets stale renderer plumbing and needs a current-main repair that also preprocesses raw chat/BTW output before Markdown parsing.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Retarget half-block detection to active renderers:** `ui/src/ui/markdown.ts:216-222`
  - body: This adds `.half-block-art` from `htmlEscapeRenderer.code()`, but current main renders fenced and indented code through `md.renderer.rules.fence` and `code_block`.   - confidence: 0.91

  - confidence: 0.88
- **[P3] Inherit the sidebar half-block code font size:** `ui/src/styles/chat/sidebar.css:128-134`
  - body: The sidebar half-block `pre code` rule inherits color and line-height but leaves the existing `.sidebar-markdown :where(pre code)` font-size override active, so QR glyphs render smaller than the intended half-block `pre` size. Add `font-size: inherit`.
  - confidence: 0.78
