## Summary

Keep open: the user-visible QR rendering problem still appears present on current main, and the PR has useful real UI proof, but the patch is not correct against the current markdown-it renderer and does not cover raw transcript QR output.

## What This Changes

The PR adds half-block-art detection and CSS for QR-like code blocks in web chat/sidebar markdown, plus sidebar tool-output fencing for detected block art.

## Best Possible Solution

Land a current-main UI change that applies shared half-block detection in the active markdown-it code renderers and raw chat/sidebar output paths, preserves QR quiet-zone whitespace, keeps normal code blocks unchanged, and includes focused tests plus browser proof.

## Reproduction Assessment

Yes, source-reproducible. Current main has no half-block detector/styling, and the active markdown-it renderers plus raw transcript calls would render QR text as generic code or paragraph markup rather than the tight QR mode.

## Solution Assessment

No. The visual direction is good, but the branch hooks an obsolete renderer path and only partially handles raw output; the safer solution is to retarget the same behavior to current markdown-it and transcript/sidebar paths.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Retarget half-block wrapping to active renderers:** `ui/src/ui/markdown.ts:215-219`
  - body: This adds `.half-block-art` from `htmlEscapeRenderer.code()`, but current main renders fenced and indented code through markdown-it `fence` and `code_block` rules. After retargeting, fenced QR blocks will still get the generic wrapper unless the detector moves to those active hooks.
  - confidence: 0.9
- **[P2] Wrap raw transcript QR text before Markdown parsing:** `ui/src/ui/chat/tool-helpers.ts:25-29`
  - body: The new fencing only runs in `formatToolOutputForSidebar()`. Main assistant/tool transcript text still goes straight through `toSanitizedMarkdownHtml(markdown)`, so unfenced multiline QR output remains paragraph text and never receives `.half-block-art`.
  - confidence: 0.88
- **[P3] Inherit the sidebar half-block code font size:** `ui/src/styles/chat/sidebar.css:128-134`
  - body: The sidebar override inherits color and line-height but leaves the existing `pre code` font-size in effect. Add `font-size: inherit` so the QR glyphs actually use the half-block `<pre>` size.
  - confidence: 0.78
