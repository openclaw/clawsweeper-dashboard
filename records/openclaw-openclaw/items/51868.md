## Summary

Keep open: the PR addresses a real, source-reproducible web UI rendering gap and includes visible proof, but the branch is stale against current main and does not wire the fix into the active Markdown rendering paths.

## What This Changes

The PR adds half-block-art detection and chat/sidebar CSS so UTF-8 QR-code-like code blocks render as tight black-and-white block art in the web UI.

## Best Possible Solution

Land a current-main UI fix that detects half-block art in the active markdown-it code renderers and pre-fences raw chat/sidebar/BTW text while preserving QR whitespace and normal code-block behavior.

## Reproduction Assessment

Yes, at source level: current main has no half-block handling in the active markdown-it code renderers, and raw chat/sidebar text paths still call `toSanitizedMarkdownHtml()` directly. I did not run a live browser repro in this read-only review.

## Solution Assessment

No. The styling direction is reasonable, but this PR needs to be retargeted to current-main renderer rules and broadened beyond the sidebar helper before it is the maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Retarget detection to markdown-it code renderers:** `ui/src/ui/markdown.ts:216-222`
  - body: This adds `.half-block-art` from `htmlEscapeRenderer.code()`, but current main renders fenced and indented code through `md.renderer.rules.fence` and `code_block`. After rebasing, QR fences would still use normal code-block markup unless the detector is wired into those active renderer rules.
  - confidence: 0.91
- **[P2] Fence raw half-block replies before Markdown parsing:** `ui/src/ui/chat/tool-helpers.ts:25-29`
  - body: The new half-block fencing only runs in `formatToolOutputForSidebar()`. Main transcript and BTW side-result text still call `toSanitizedMarkdownHtml()` directly, so unfenced multiline QR output is parsed as ordinary Markdown and never receives `.half-block-art`.
  - confidence: 0.88
- **[P3] Inherit the sidebar half-block code font size:** `ui/src/styles/chat/sidebar.css:128-134`
  - body: The sidebar half-block `pre code` rule resets color and line-height but leaves the existing sidebar `pre code` font-size override active, so QR glyphs can ignore the intended half-block `pre` size. Add `font-size: inherit` here.
  - confidence: 0.78
