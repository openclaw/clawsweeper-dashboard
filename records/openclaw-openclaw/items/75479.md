## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Export the new TTS helpers from speech-core:** `src/plugin-sdk/tts-runtime.ts:115-120`
  - body: `src/tts/tts.ts` reaches speech-core through the lazy `tts-runtime` facade. This PR adds facade entries for `setTtsVoice` and `getTtsVoiceByProvider`, but does not export those functions from `extensions/speech-core/runtime-api.ts`, so the runtime lookup will not find the new helpers when `tts.status` or `tts.setVoice` calls through the facade.
  - confidence: 0.94
- **[P2] Classify the new TTS RPCs as write methods:** `src/gateway/server-methods-list.ts:30-31`
  - body: The PR lists and implements `tts.setVoice` and `tts.preview`, but does not add them to `METHOD_SCOPE_GROUPS`. The existing classification test over `listGatewayMethods()` should fail, and Control UI callers with read/write scopes would be treated as missing admin scope for these new methods.
  - confidence: 0.92
- **[P3] Let config refresh finish after loading TTS state:** `ui/src/ui/app-settings.ts:341`
  - body: The new config-tab branch returns immediately after starting TTS status/provider loads. On current main, `refreshActiveTab` wraps the switch with `beginControlUiRefresh` and the shared success `finishControlUiRefresh` call after the switch, so this return would leave config refresh timings without their matching completion event.
  - confidence: 0.83
