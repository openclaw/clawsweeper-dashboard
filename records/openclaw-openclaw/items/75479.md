## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Export the new voice helpers from speech-core:** `src/plugin-sdk/tts-runtime.ts:115-120`
  - body: `src/tts/tts.ts` reaches speech-core through the lazy `tts-runtime` facade, but the PR adds `setTtsVoice` and `getTtsVoiceByProvider` to the facade without exporting them from `extensions/speech-core/runtime-api.ts`, so the real runtime lookup will fail.
  - confidence: 0.94
- **[P2] Classify the new TTS RPCs as write methods:** `src/gateway/server-methods-list.ts:30-31`
  - body: Adding `tts.setVoice` and `tts.preview` to the gateway method list without adding them to `METHOD_SCOPE_GROUPS` leaves them unclassified. The classification test should fail, and non-admin Control UI callers would be rejected instead of using the normal write scope.
  - confidence: 0.91
- **[P2] Disable fallback for explicit previews:** `src/gateway/server-methods/tts.ts:300-306`
  - body: When preview receives an explicit provider or voice, it builds overrides but calls `textToSpeech` without `disableFallback`. A failed selected provider can silently fall back and play a different provider or voice than the user selected.
  - confidence: 0.88
- **[P2] Return invalid request for bad preview overrides:** `src/gateway/server-methods/tts.ts:289-298`
  - body: `resolveExplicitTtsOverrides` can throw for unknown providers or unsupported voice overrides. `tts.convert` maps that to `INVALID_REQUEST`, but `tts.preview` lets it fall into the outer catch and reports `UNAVAILABLE`.
  - confidence: 0.86
- **[P2] Validate setVoice against the active provider registry:** `src/gateway/server-methods/tts.ts:265-275`
  - body: `tts.setProvider` canonicalizes with the runtime config and verifies the provider exists, but `tts.setVoice` only canonicalizes the raw id before persisting. It should reject provider ids that are not registered in the active config/plugin set.
  - confidence: 0.82
- **[P3] Replace the changelog placeholder:** `CHANGELOG.md:9`
  - body: The changelog entry would ship `Fixes #XXXX` and bot attribution. Replace the placeholder with the real related issue or PR reference, and avoid adding a `Thanks` line for an automation account.
  - confidence: 0.95
