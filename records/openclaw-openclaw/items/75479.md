## Summary

Keep open. This PR targets a real gap also tracked by #71487, and current main still lacks the Quick Settings TTS controls and voice-preview RPCs, but the proposed patch has blocking correctness issues around explicit preview fallback behavior and gateway method scope classification.

## What This Changes

The PR adds a Quick Settings TTS card, UI controller state, `tts.setVoice`/`tts.preview` gateway methods, per-provider voice persistence/application, SDK facade exports, tests, CSS, and a changelog entry.

## Best Possible Solution

Land a corrected version of this PR or a successor that keeps the first-class TTS settings UI, disables fallback for explicit provider/voice previews, classifies the new RPCs under the existing TTS write scope with tests, replaces the changelog placeholder with the real related issue/credit, and then closes #71487 when merged.

## Reproduction Assessment

Yes. The review findings reproduce from static code paths: `tts.preview` omits the explicit-override fallback guard used by `tts.convert`, and new advertised RPCs are absent from `METHOD_SCOPE_GROUPS`, which makes non-admin operator calls default to `operator.admin`.

## Solution Assessment

No. The product direction is appropriate, but the current implementation is not the best mergeable form until explicit previews cannot silently play fallback audio and the new gateway methods follow the existing TTS authorization contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Disable fallback for explicit TTS previews:** `src/gateway/server-methods/tts.ts:297-302`
  - body: `tts.preview` accepts a selected provider and voice, but this `textToSpeech` call omits `disableFallback`. Unlike `tts.convert`, a failed selected provider or invalid selected voice can synthesize from a fallback provider, and the UI will still play it as the requested sample.
  - confidence: 0.91
- **[P2] Classify the new TTS RPC methods:** `src/gateway/server-methods-list.ts:29-30`
  - body: Adding `tts.setVoice` and `tts.preview` to the advertised method list without adding them to `METHOD_SCOPE_GROUPS` leaves them unclassified. Non-admin operator clients with `operator.write` will get the default `missing scope: operator.admin`, unlike the existing TTS write RPCs.
  - confidence: 0.88
- **[P3] Replace the changelog placeholder reference:** `CHANGELOG.md:9`
  - body: The changelog entry contains `Fixes #XXXX`, so the release note would publish a placeholder issue reference and may imply a closing relationship that is not real. Replace it with the actual related issue or a neutral reference before merge.
  - confidence: 0.95
