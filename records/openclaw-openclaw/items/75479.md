## Summary

Keep open. The feature gap still exists on current main, and this PR is a useful implementation path for #71487, but the current patch is not merge-ready because it leaves new TTS RPC/runtime surfaces incompletely wired and incorrectly validated.

## What This Changes

The PR adds Quick Settings Text-to-Speech controls, per-provider voice preference persistence, new `tts.setVoice` and `tts.preview` gateway methods, speech-core default-voice application, UI controller/state wiring, tests, and a changelog entry.

## Best Possible Solution

Keep the feature direction and land a corrected implementation for #71487 that preserves the speech-core runtime facade, registers the new gateway methods with least-privilege scopes, validates provider and voice inputs like existing TTS writes, keeps explicit previews on the selected provider, updates docs/changelog metadata, and adds focused gateway/UI/method-scope coverage.

## Reproduction Assessment

Yes. Current-main reproduction is a source check: Quick Settings has no TTS card and the gateway method list/handlers do not expose `tts.setVoice`, `tts.preview`, or `voiceByProvider`; the PR defects reproduce by inspecting the patch against the runtime facade and method-scope contracts.

## Solution Assessment

No. A Quick Settings card plus narrow gateway methods is a good product direction, but this patch is not yet the best implementation because it leaves runtime exports, authorization scopes, input-error mapping, and explicit-preview fallback behavior inconsistent with existing TTS contracts.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Export the voice helpers from speech-core runtime:** `src/plugin-sdk/tts-runtime.ts:112-120`
  - body: The new facade calls load `setTtsVoice` and `getTtsVoiceByProvider` from `extensions/speech-core/runtime-api.ts`, but this patch does not export those names there. Real gateway calls through `src/tts/tts.ts` will therefore not reach the new speech-core helpers; add the runtime-api exports with matching coverage.
  - confidence: 0.94
- **[P2] Classify the new TTS RPCs as write methods:** `src/gateway/server-methods-list.ts:29-30`
  - body: `tts.setVoice` and `tts.preview` are added to the public method list, but not to `METHOD_SCOPE_GROUPS`. That leaves them unclassified/admin-only and should fail the gateway method classification contract; add them to the write scope next to the existing TTS writes.
  - confidence: 0.91
- **[P2] Keep explicit previews on the selected provider:** `src/gateway/server-methods/tts.ts:297-303`
  - body: When a provider or voice is supplied, preview builds explicit overrides but calls `textToSpeech` without `disableFallback`. If the selected provider fails, the preview can play a fallback provider instead of the selected voice/provider; pass `disableFallback` when explicit preview overrides are present, matching `tts.convert`.
  - confidence: 0.88
- **[P2] Return invalid request for bad preview overrides:** `src/gateway/server-methods/tts.ts:287-295`
  - body: `resolveExplicitTtsOverrides` can throw for an unknown provider or unsupported voice override. `tts.convert` maps that to `INVALID_REQUEST`, but `tts.preview` lets it fall into the outer catch and reports `UNAVAILABLE`, making caller input errors look like backend outages.
  - confidence: 0.86
- **[P2] Validate setVoice against the provider registry:** `src/gateway/server-methods/tts.ts:262-271`
  - body: `tts.setProvider` canonicalizes with the runtime config and rejects provider ids that are not registered, but `tts.setVoice` canonicalizes the raw string and persists it. Use the same registry-backed validation before saving a voice preference.
  - confidence: 0.83
- **[P3] Replace the changelog placeholder:** `CHANGELOG.md:9`
  - body: The changelog entry would ship `Fixes #XXXX` and bot attribution. Replace the placeholder with the real related issue/PR reference, and avoid forcing a `Thanks` line to an automation account when no human contributor attribution is available.
  - confidence: 0.96
