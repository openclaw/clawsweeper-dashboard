## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Export the new voice helpers from speech-core:** `src/plugin-sdk/tts-runtime.ts:115-120`
  - body: `src/tts/tts.ts` reaches speech-core through the lazy `tts-runtime` facade. This PR adds `setTtsVoice` and `getTtsVoiceByProvider` facade entries, but `extensions/speech-core/runtime-api.ts` still does not export those functions, so the real runtime lookup will fail.
  - confidence: 0.94
- **[P2] Classify the new TTS RPCs as write methods:** `src/gateway/server-methods-list.ts:30-31`
  - body: Adding `tts.setVoice` and `tts.preview` to the gateway method list without adding them to `METHOD_SCOPE_GROUPS` leaves them unclassified. The classification test should fail, and write-scoped Control UI callers would be rejected as requiring admin scope.
  - confidence: 0.91
- **[P2] Validate setVoice against the active registry:** `src/gateway/server-methods/tts.ts:270-275`
  - body: `tts.setProvider` canonicalizes with the runtime config and verifies the provider exists, but `tts.setVoice` only canonicalizes the raw id before persisting. It should reject provider ids that are not registered in the active config/plugin set.
  - confidence: 0.84
- **[P2] Map bad preview overrides to invalid request:** `src/gateway/server-methods/tts.ts:289-298`
  - body: `resolveExplicitTtsOverrides` can throw for unknown providers or unsupported voice overrides. `tts.convert` maps that path to `INVALID_REQUEST`, but `tts.preview` lets it fall into the outer catch and returns `UNAVAILABLE`.
  - confidence: 0.87
- **[P2] Disable fallback for explicit previews:** `src/gateway/server-methods/tts.ts:300-306`
  - body: When preview receives an explicit provider or voice, it builds overrides but calls `textToSpeech` without `disableFallback`. A failed selected provider can silently fall back and play a different provider or voice than the user selected.
  - confidence: 0.89
- **[P2] Remove the unused TTS controller imports:** `ui/src/ui/app-render.ts:115-122`
  - body: `app-render.ts` imports `loadTtsStatus` and `loadTtsProviders`, but the callbacks only use the setter and preview helpers. Leaving unused imports here is likely to fail the repo's lint/type gate.
  - confidence: 0.86
- **[P3] Do not return before finishing config refresh:** `ui/src/ui/app-settings.ts:341`
  - body: The new `case "config"` returns from `refreshActiveTab` before the shared `finishControlUiRefresh(..., "ok")` call runs. Config-tab refreshes would emit a start timing event but never the matching end event.
  - confidence: 0.82
- **[P3] Replace the changelog placeholder:** `CHANGELOG.md:9`
  - body: The new changelog entry would ship `Fixes #XXXX` and bot attribution. Replace the placeholder with the real related issue or PR reference, and avoid adding a `Thanks` line for an automation account.
  - confidence: 0.95
- **[P3] Show loading before the no-provider empty state:** `ui/src/ui/views/config-quick.ts:1123`
  - body: The TTS card has provider-loading state in app state, but the view decides between selectors and `No TTS providers configured` only from `configuredProviders.length`. Initial refreshes can briefly show the wrong empty state before `tts.providers` resolves.
  - confidence: 0.77
