## Summary

Keep open because current main still exposes shared poll creation fields on the generic `message` tool and still rejects `action: "send"` when non-empty poll-looking params are present; no target-repo PR has landed the fix.

## What This Changes

The issue asks for `message` tool `action: "send"` to avoid or ignore accidental poll creation fields so normal Feishu and other channel file sends are not rejected by poll validation.

## Best Possible Solution

Land a narrow shared message-tool/outbound fix that strips or scopes accidental poll creation params from `send`, preserves explicit `poll` behavior, and coordinates remaining schema-overexposure work with #43015.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: call `runMessageAction()` with `action: "send"`, a normal send target, and a non-empty poll creation field such as `pollQuestion`; current main throws before Feishu, Telegram, or plugin dispatch.

## Solution Assessment

Yes. The maintainable fix belongs in the shared message-tool/outbound boundary rather than Feishu-specific code, because the same schema and guard affect multiple channel send paths.
