## Summary

Keep open because current main still exposes shared poll fields on Feishu's multi-action message schema and still rejects `action="send"` when non-empty poll-looking params are present before channel dispatch. The issue also has an open linked repair PR in https://github.com/BingqingLyu/openclaw/pull/708 and a broader companion tracked at https://github.com/openclaw/openclaw/issues/43015, so closing it before that work lands would be premature.

## What This Changes

Requests that plain Feishu `message` sends with file attachments stop being rejected because poll-creation fields are exposed in the shared message tool schema.

## Best Possible Solution

Review, land, or replace the linked normalization fix so stray poll params do not block plain sends, while keeping explicit poll validation strict and tracking the broader schema-shape cleanup in https://github.com/openclaw/openclaw/issues/43015.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: call `runMessageAction()` with `action="send"`, a normal message or file payload, and a non-empty poll creation field such as `pollQuestion` or `pollDurationSeconds`; current main throws before channel dispatch.

## Solution Assessment

Yes. A narrow action-aware sanitization or schema-scoping fix is the best path for the central bug, while broader per-channel tool-shape work can remain in the companion issue.
