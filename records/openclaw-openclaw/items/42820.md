## Summary

Keep open: current main has narrowed the poll-intent guard enough to cover the latest metadata-default reproduction, but the generic message schema still exposes poll fields and the issue is still paired with an open, conflicting linked fix candidate plus a product-shape decision about action-specific schemas versus send-path sanitization.

The issue requests that generic `message` sends ignore or avoid poll-creation fields so normal Feishu/Telegram sends with attachments are not rejected by the poll guard.

## Best Possible Solution

Keep this issue open until maintainers choose the durable message-tool boundary and either land a clean fix or explicitly decide that current content-bearing poll intent detection is the intended contract.

## Solution Assessment

Unclear. Current main is a useful narrow mitigation for metadata defaults, but the issue asks for a broader schema/action design and the open linked PR is conflicting, so maintainers need to choose the best long-term path.
