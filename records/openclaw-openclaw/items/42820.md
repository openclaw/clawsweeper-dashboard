## Summary

Keep open because current main still exposes poll fields on mixed-action message schemas and still rejects `action="send"` when non-empty poll-looking params are present before channel dispatch. The discussion confirms the same guard affects Telegram as well as Feishu, and an open linked repair PR plus the broader companion issue are still unresolved.

## What This Changes

The issue asks for `message` `action="send"` to ignore or scope poll-creation fields so Feishu and other channel file sends are not rejected by the poll guard.

## Best Possible Solution

Land or replace the linked normalization fix so stray poll params do not block plain sends, while keeping explicit poll validation strict and tracking the broader schema split under https://github.com/openclaw/openclaw/issues/43015.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: call `runMessageAction()` with `action="send"`, a normal message or media payload, and a non-empty poll creation field such as `pollQuestion` or `pollDurationSeconds`; current main throws before channel dispatch.

## Solution Assessment

Yes. A narrow action-aware sanitization or schema-scoping fix is the best path; the issue should stay open until the linked repair PR is landed, replaced, or explicitly closed by maintainers.
