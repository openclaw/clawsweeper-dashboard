## Summary

Keep open: current `main` and the latest release contain the merged shared-modifier fix, but they still reject normal `send` calls when channel-extra poll fields such as `pollDurationSeconds` or `pollPublic` are present, and open linked repair branches remain unresolved.

## What This Changes

The issue asks OpenClaw to stop blocking Feishu and other generic `message` sends with files when schema-padded poll fields appear on an `action: "send"` call.

## Best Possible Solution

Keep this issue open and consolidate it with the broader schema-overexposure work so normal sends either never receive non-poll poll extras or sanitize them before the send guard, while explicit poll creation remains validated.

## Reproduction Assessment

Yes. Current source and focused tests reproduce the remaining path: `action: "send"` still rejects when channel-extra poll fields such as `pollDurationSeconds` or `pollPublic` are present.

## Solution Assessment

No. Current `main` is only a partial fix: shared schema-padded modifiers are accepted, but channel-extra poll fields still block sends, so maintainers still need the action-aware schema or sanitization decision.
