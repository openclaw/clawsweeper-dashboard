## Label Justifications

- `P1`: The bug can block real Feishu, Telegram, and other channel sends with attachments when models populate visible poll fields.
- `impact:message-loss`: Outbound messages can be rejected before delivery because stray poll-looking params trip the message tool guard.

## Summary

Keep open: current main and the latest release still reject ordinary `action="send"` calls before channel dispatch when non-empty poll-looking params are present, and an open linked repair PR has not landed.

## What This Changes

The issue asks for normal Feishu and other-channel `message` sends with file attachments to stop failing when stray poll schema fields are present on `action="send"`.

## Best Possible Solution

Land or replace https://github.com/BingqingLyu/openclaw/pull/708 with an action-aware fix that ignores stray poll creation params for ordinary sends while preserving explicit `action="poll"` validation; keep broader schema decomposition tracked in https://github.com/openclaw/openclaw/issues/43015.

## Reproduction Assessment

Yes, at source level. A `runMessageAction()` call with `action="send"`, normal message/file params, and a non-empty poll field such as `pollQuestion`, `pollOption`, or non-zero `pollDurationSeconds` reaches the existing throw before channel dispatch.

## Solution Assessment

Yes for the narrow bug. The maintainable fix is action-aware sanitization or a narrower poll-intent check on `send`, while the larger per-channel schema/tool-shape change belongs in https://github.com/openclaw/openclaw/issues/43015.
