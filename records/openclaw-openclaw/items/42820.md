## Summary

Keep open: current main and v2026.6.10 fix the metadata-default poll false positive, including `pollDurationSeconds`, but the broader shared-schema/content-field question remains unsettled and an open conflicting fork PR still claims the full fix path.

## What This Changes

The issue requests that plain `message` tool `action="send"` with Feishu or other channel attachments not be rejected because poll fields are exposed in the shared message schema.

## Best Possible Solution

Keep the shipped v2026.6.10 metadata fix, then have maintainers decide whether the remaining contract should be action-scoped schemas, send-path sanitization for content poll fields, or closing this as sufficiently fixed.

## Reproduction Assessment

Unclear: current main and v2026.6.10 no longer reproduce the metadata-default failure, but source still rejects non-empty poll content fields on `send`. A live current-main channel/model repro is still needed to prove ordinary file sends receive those content fields accidentally.
