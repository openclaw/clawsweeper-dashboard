- **[P1] Pass only one account context to Teams runtime actions:** `extensions/msteams/src/channel.ts:817-818`
  - body: For named accounts this call sends a config already collapsed to the account while also passing `accountId`. The runtime send/Graph helpers call `resolveMSTeamsAccountConfig` again; with no accounts map left, non-default identities are cleared and actions can fail with missing credentials. Pass the original config with `accountId`, or pass a scoped config without re-resolving.
  - confidence: 0.9
- **[P2] Preserve existing raw account keys during setup writes:** `extensions/msteams/src/setup-core.ts:80`
  - body: This normalizes the requested id and later reads/writes `accounts[accountId]`, but the resolver intentionally accepts display-style keys by normalized comparison. A config with `accounts: { "Support Bot": ... }` targeted as `support-bot` will get a new sibling entry instead of updating the existing account, leaving stale credentials/config behind.
  - confidence: 0.86
