## Summary

Keep open: current main still has a memory-core startup retry path that can emit the reported warn-level cron-unavailable message, and open PR #76099 is the linked unmerged fix candidate for this issue.

## What This Changes

The issue asks memory-core to avoid a misleading warn-level dreaming cron reconciliation message during normal gateway startup when the cron service is not ready yet.

## Best Possible Solution

Merge PR #76099 or an equivalent narrow memory-core change that keeps startup retry-window cron unavailability at debug level while preserving warn-level logging after retry exhaustion or during real runtime failures.

## Reproduction Assessment

Yes. Current main has a focused test path in extensions/memory-core/src/dreaming.test.ts that reproduces cron being unavailable at startup, advances the retry timer, and observes the reported warning before cron becomes available.

## Solution Assessment

Yes. Adjusting memory-core's existing bounded startup retry warning policy is narrower than adding a new core plugin-ready event and keeps genuine runtime failure logging intact.
