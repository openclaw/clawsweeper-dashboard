## Summary

Keep open: current main still has the reported warn-level path during deferred startup retries, and open PR #76099 is the unmerged implementation candidate that explicitly closes this issue.

## What This Changes

The issue asks memory-core to avoid a misleading warn-level dreaming cron unavailable message during the normal startup race before cron is ready.

## Best Possible Solution

Land PR #76099 or an equivalent memory-core change that keeps transient startup retry cron unavailability at debug level while preserving warn-level logging for post-window or runtime failures.

## Reproduction Assessment

Yes. Current `extensions/memory-core/src/dreaming.test.ts` reproduces cron being unavailable during startup retry, advances the retry timer, and observes the reported warning before cron becomes available.

## Solution Assessment

Yes. Adjusting memory-core's existing bounded startup retry warning policy is narrower than adding a new core plugin-ready event and preserves genuine runtime failure logging.
