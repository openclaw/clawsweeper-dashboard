Keep this PR open, but it is not merge-ready: the added PEM certificate path accepts printable `.pem` text through the generic host-read text validator, which would include private-key PEM blocks when expanded filesystem reads are enabled.

## What This Changes

The PR adds APK and PEM-style certificate MIME mappings, allows those types through host-local media validation, updates docs, and adds focused media/outbound tests.

## Best Possible Solution

Require a rebase plus a narrow PEM certificate validator or private-key rejection before owner/security acceptance of APK and certificate host-local sends.

## Reproduction Assessment

Yes for the review finding: source inspection shows `.pem` is added to the text fallback and current validation accepts printable text without checking certificate-only markers. The requested APK/certificate capability itself is a feature, not a current-main bug reproduction.

## Solution Assessment

No. Reusing the generic text gate is too broad for PEM because certificates and private keys have the same printable-text shape; the safer fix is to validate certificate PEM blocks or reject private-key blocks before allowing the alias.
