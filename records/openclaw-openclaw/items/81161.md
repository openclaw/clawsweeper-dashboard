## What This Changes

The PR changes OpenAI Codex browser OAuth to log TLS certificate preflight failures as advisory and rewrites a regression test to expect OAuth to proceed.

## Best Possible Solution

Keep the PR open, align the production change and regression test on the same OpenAI Codex OAuth runtime path, then require real CLI/browser proof for the linked TLS-preflight regression before merge.

## Reproduction Assessment

Yes for the blocking review finding: source inspection shows the rewritten test still calls `src/plugins/provider-openai-codex-oauth.ts`, whose `tls-cert` branch still throws. ## Solution Assessment

No. Treating the preflight as advisory is the right direction, but this patch needs to test the bundled runtime it changes or also update the legacy src helper that the rewritten test exercises.
