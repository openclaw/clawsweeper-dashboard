Full review comments:

- **[P2] Skip source checkouts before evaluating metadata:** `src/commands/doctor-rebuild-guard.ts:329`
  - body: `doctorCommand` passes the package root returned by `resolveOpenClawPackageRoot`, which also resolves source checkouts, and current doctor already detects those via `pnpm-workspace.yaml` plus `src/entry.ts`. Calling the rebuild guard unconditionally means source runs can warn as corrupt/rebuilt from missing or locally built `dist/build-info.json`, even though this diagnostic is about npm package installs.
  - confidence: 0.9
- **[P2] Bound the npm registry fetch:** `src/commands/doctor-rebuild-guard.ts:162`
  - body: This new normal doctor path awaits `fetch` with no timeout or abort signal. On a network that blackholes outbound HTTPS, a warn-only install note can stall `openclaw doctor`; use the existing timeout helper or equivalent and treat timeout as inconclusive.
  - confidence: 0.91
- **[P2] Do not revive expired registry cache entries:** `src/commands/doctor-rebuild-guard.ts:212`
  - body: `loadFreshCache` rejects stale cache entries, but the fallback then reads the same cache file unconditionally when fetch fails or offline mode is set. That can emit stale `version-unreleased` or rebuild warnings instead of staying silent when current registry data is unavailable.
  - confidence: 0.9
- **[P2] Validate package and build-info versions together:** `src/commands/doctor-rebuild-guard.ts:89-91`
  - body: The code reads `package.json.version` but never checks it against `dist/build-info.json.version`, then uses the build-info version for registry lookup. A partial update with mismatched files can be reported as pristine or unreleased instead of corrupt, which undermines the install-divergence signal.
  - confidence: 0.88
- **[P3] Report malformed metadata separately:** `src/commands/doctor-rebuild-guard.ts:67-72`
  - body: `readJsonFile` swallows all read and parse failures as `null`, so malformed JSON is reported as missing metadata. The doctor output should distinguish missing from malformed files so the warning points operators at the actual repair.
  - confidence: 0.84
- **[P3] Remove the release-owned changelog edit:** `CHANGELOG.md:7-10`
  - body: Root policy says `CHANGELOG.md` is release-owned and not edited for normal PRs. Keep the release-note context in the PR body or squash message and drop this Unreleased entry from the branch.
