## Summary

Keep open: current main still lacks an equivalent rebuild-vs-npm diagnostic, but the PR needs changes before merge because source inspection confirms unbounded registry fetch, expired-cache fallback, and local metadata validation defects; current main also moved doctor to lazy imports so the branch needs a rebase.

## What This Changes

The PR adds a warn-only openclaw doctor install-freshness check, docs, changelog text, and tests that compare local build metadata with npm registry publish times.

## Best Possible Solution

Land a rebased doctor diagnostic that remains warn-only, uses bounded registry fetching, treats stale registry data as inconclusive, validates package/build-info parity, reports corrupt metadata accurately, and preserves current lazy doctor loading.

## Reproduction Assessment

Yes for review purposes. Current main source lacks the requested doctor guard, and the PR defects are directly visible in the added source without needing a generated checkout run.

## Solution Assessment

No, not as submitted. The direction belongs in openclaw doctor, but the patch should be tightened around bounded network behavior, TTL-respecting cache use, local metadata parity, malformed metadata diagnostics, and current-main lazy imports.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Add a timeout around the npm registry fetch:** `src/commands/doctor-rebuild-guard.ts:162`
  - body: This doctor check calls the npm registry with a bare fetch. On networks that blackhole outbound HTTPS, a non-essential install note can stall openclaw doctor; use the existing timeout fetch helper or an abort signal so failures become inconclusive quickly.
  - confidence: 0.92
- **[P2] Do not revive expired registry cache entries:** `src/commands/doctor-rebuild-guard.ts:212`
  - body: After loadFreshCache rejects stale cache data, the fallback still reads the same cache file without checking age. That can emit false version-unreleased or rebuild warnings from expired registry data during offline or outage runs instead of staying silent when current registry data is unavailable.
  - confidence: 0.9
- **[P2] Validate build-info version against package.json:** `src/commands/doctor-rebuild-guard.ts:87-91`
  - body: package.json.version is only checked for existence while the registry lookup uses dist/build-info.json.version. A partial install with those files out of sync can evaluate the wrong version and suppress the corruption signal this guard is meant to provide.
  - confidence: 0.89
- **[P3] Distinguish malformed JSON from missing metadata:** `src/commands/doctor-rebuild-guard.ts:67-73`
  - body: readJsonFile collapses missing files and JSON parse failures into null, so malformed build-info.json is reported as build-info-missing. That points operators at the wrong remediation for a corrupt metadata file.
  - confidence: 0.86
