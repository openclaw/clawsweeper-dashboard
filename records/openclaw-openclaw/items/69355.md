## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.95

Full review comments:

- **[P1] Import the existing note module:** `src/commands/doctor-rebuild-guard.ts:4`
  - body: `../terminal/note.js` does not exist on current main; doctor code imports notes from `../../packages/terminal-core/src/note.js`. As written, the new guard cannot build or load once this module is resolved.
  - confidence: 0.96
- **[P2] Bound the npm registry fetch:** `src/commands/doctor-rebuild-guard.ts:162`
  - body: This runs in the normal `openclaw doctor` path, but the registry request has no abort signal or timeout. On networks that blackhole HTTPS, a nonessential warning check can stall doctor instead of staying quick and inconclusive.
  - confidence: 0.93
- **[P2] Keep stale cache from producing warnings:** `src/commands/doctor-rebuild-guard.ts:212`
  - body: After rejecting an expired cache entry, the fallback still loads the same cache file unconditionally when fetch is unavailable or offline. That can turn old registry data into fresh `version-unreleased` or rebuild warnings, violating the PR's silent-on-inconclusive behavior.
  - confidence: 0.92
- **[P2] Reject mismatched package metadata:** `src/commands/doctor-rebuild-guard.ts:89-91`
  - body: `package.json.version` is read but not compared with `dist/build-info.json.version`, so a partial update can evaluate the wrong version and report `pristine` or `unreleased` instead of corrupt local metadata.
  - confidence: 0.9
- **[P2] Skip source checkouts without build metadata:** `src/flows/doctor-health.ts:47`
  - body: `doctorCommand` passes the resolved package root for both source checkouts and package installs. In a normal source checkout with `package.json` but no `dist/build-info.json`, this emits a corrupt-install warning even though there is no npm package rebuild to compare.
  - confidence: 0.87
- **[P3] Remove the release-owned changelog entry:** `CHANGELOG.md:6-8`
  - body: Normal PRs should not edit `CHANGELOG.md`; release generation owns that file. Keep the release-note context in the PR body or squash message and leave the changelog unchanged.
  - confidence: 0.82
- **[P3] Distinguish malformed metadata from missing files:** `src/commands/doctor-rebuild-guard.ts:67-72`
  - body: `readJsonFile` returns `null` for both missing files and malformed JSON, so malformed `build-info.json` is reported as `build-info-missing`. Keep those reasons separate so the warning sends operators to the right repair path.
  - confidence: 0.78
