## Summary

Keep open. Current main still lacks the requested doctor install-freshness diagnostic, but the PR should not merge as submitted because the added guard can stall doctor on an unbounded registry fetch and can emit false or misleading warnings from stale or mismatched metadata.

## What This Changes

The PR adds a warn-only `openclaw doctor` check that compares local build metadata with npm publish times, plus docs, changelog text, and Vitest coverage.

## Best Possible Solution

Land a rebased doctor-owned diagnostic that stays warn-only, uses bounded registry I/O, treats expired registry data as inconclusive, validates local metadata parity, and keeps the docs aligned with that behavior.

## Reproduction Assessment

Yes. Current-main source inspection shows the doctor install-note path but no build-info/npm publish comparison, and targeted searches found no rebuild-guard symbols or install-freshness docs.

## Solution Assessment

No, not as submitted. The direction fits `openclaw doctor`, but the branch needs bounded network behavior, TTL-respecting cache fallback, package/build-info version parity, and clearer corrupt-metadata handling before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Add a timeout to the npm registry fetch:** `src/commands/doctor-rebuild-guard.ts:162`
  - body: This doctor check calls the npm registry with a bare `fetchFn(NPM_REGISTRY_URL)`. On networks that blackhole outbound HTTPS, a non-essential install note can stall `openclaw doctor`; use the existing timeout fetch helper or an abort signal so failures become inconclusive quickly.
  - confidence: 0.92
- **[P2] Honor the cache TTL when falling back:** `src/commands/doctor-rebuild-guard.ts:212`
  - body: After `loadFreshCache` rejects stale cache data, the fallback still reads the same cache file without checking age. That can emit false `version-unreleased` or rebuild warnings from expired registry data during offline or outage runs instead of staying silent when current registry data is unavailable.
  - confidence: 0.9
- **[P2] Validate build-info version against package.json:** `src/commands/doctor-rebuild-guard.ts:87-91`
  - body: `package.json.version` is only checked for existence while the registry lookup uses `dist/build-info.json.version`. A partial install with those files out of sync can evaluate the wrong version and suppress the corruption signal this guard is meant to provide.
  - confidence: 0.89
- **[P3] Distinguish malformed JSON from missing metadata:** `src/commands/doctor-rebuild-guard.ts:67-73`
  - body: `readJsonFile` collapses missing files and JSON parse failures into `null`, so malformed `build-info.json` is reported as `build-info-missing`. That points operators at the wrong remediation for a corrupt metadata file.
  - confidence: 0.86
