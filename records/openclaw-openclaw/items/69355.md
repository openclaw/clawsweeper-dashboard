## Summary

Keep open: the feature is useful and current main still lacks an equivalent install-divergence diagnostic, but this PR needs focused repairs before merge for source-checkout false positives, unbounded registry I/O, stale cache reuse, version parity, and malformed-metadata reporting.

## What This Changes

## Best Possible Solution

Land a rebased doctor-owned diagnostic that stays warn-only, skips ordinary source checkouts, uses bounded registry I/O, treats expired registry data as inconclusive, validates local metadata parity, and keeps docs aligned.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Skip source checkouts before corrupt metadata warnings:** `src/commands/doctor-rebuild-guard.ts:77-79`
  - body: In a source checkout, `resolveOpenClawPackageRoot` resolves the repo root and `dist/build-info.json` often does not exist before a build. This branch turns that normal state into an `Install` warning about malformed package metadata, so `pnpm openclaw doctor` would report a corrupt install for ordinary development runs; skip source roots or only treat missing build metadata as corrupt for packaged installs.
  - confidence: 0.9
- **[P2] Add a timeout to the npm registry fetch:** `src/commands/doctor-rebuild-guard.ts:162`
  - body: This doctor check calls the npm registry with a bare `fetchFn(NPM_REGISTRY_URL)`. On networks that blackhole outbound HTTPS, a non-essential install note can stall `openclaw doctor`; use the existing timeout helper or an abort signal so failures become inconclusive quickly.
  - confidence: 0.92
- **[P2] Honor cache TTL when registry lookup falls back:** `src/commands/doctor-rebuild-guard.ts:212`
  - body: After `loadFreshCache` rejects stale cache data, the fallback still reads the same cache file without checking age. That can emit false `version-unreleased` or rebuild warnings from expired registry data during offline or outage runs instead of staying silent when current registry data is unavailable.
  - confidence: 0.9
- **[P2] Validate build-info version against package.json:** `src/commands/doctor-rebuild-guard.ts:87-91`
  - body: `package.json.version` is only checked for existence while the registry lookup uses `dist/build-info.json.version`. A partial install with those files out of sync can evaluate the wrong version and suppress the corruption signal this guard is meant to provide.
  - confidence: 0.89
- **[P3] Distinguish malformed metadata from missing files:** `src/commands/doctor-rebuild-guard.ts:67-73`
  - body: `readJsonFile` collapses missing files and JSON parse failures into `null`, so malformed `build-info.json` is reported as `build-info-missing`. That points operators at the wrong remediation for a corrupt metadata file.
  - confidence: 0.86
