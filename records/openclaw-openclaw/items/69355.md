## Summary

Keep open: current main still lacks this doctor diagnostic, and the PR is useful, but the new registry/cache/metadata path has narrow correctness blockers that should be repaired before merge.

## What This Changes

The PR adds a warn-only `openclaw doctor` install freshness check that compares local build metadata with npm publish timestamps, plus tests, docs, and changelog text.

## Best Possible Solution

Land a rebased doctor-owned diagnostic that stays warn-only, uses bounded registry I/O, treats unavailable or expired registry data as inconclusive, validates package/build metadata parity, distinguishes corrupt metadata clearly, and keeps docs aligned.

## Reproduction Assessment

Yes. The review findings are source-reproducible from the PR diff: the new fetch lacks a timeout, stale cache can be reused after TTL rejection, and package/build-info versions are not compared; current main also confirms the requested diagnostic has not landed.

## Solution Assessment

No, not as submitted. The feature belongs in `openclaw doctor`, but the patch should be repaired so an optional diagnostic cannot stall doctor or warn from stale or mismatched metadata.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Bound the npm registry fetch:** `src/commands/doctor-rebuild-guard.ts:162`
  - body: This doctor check calls `fetchFn(NPM_REGISTRY_URL)` without a timeout or abort signal. On networks that blackhole outbound HTTPS, a warn-only check can stall `openclaw doctor`; use `fetchWithTimeout` or an abort signal and return inconclusive on timeout.
  - confidence: 0.92
- **[P2] Keep expired registry cache inconclusive:** `src/commands/doctor-rebuild-guard.ts:212`
  - body: After `loadFreshCache` rejects stale cache data, the fallback reloads the same cache file without checking age. Offline or outage runs can emit false rebuild or `version-unreleased` warnings from expired publish data instead of staying silent when current registry data is unavailable.
  - confidence: 0.9
- **[P2] Validate package and build-info versions:** `src/commands/doctor-rebuild-guard.ts:87-91`
  - body: `package.json.version` is only checked for existence while the registry lookup uses `dist/build-info.json.version`. If an update leaves those files out of sync, the guard can evaluate the wrong version and suppress the corruption signal it is meant to provide.
  - confidence: 0.89
- **[P3] Report malformed metadata separately:** `src/commands/doctor-rebuild-guard.ts:67-72`
  - body: `readJsonFile` collapses missing files and JSON parse failures into `null`, so malformed `build-info.json` is reported as `build-info-missing`. That sends operators toward the wrong remediation for a corrupt metadata file.
  - confidence: 0.86
