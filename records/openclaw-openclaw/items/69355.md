## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Skip source checkouts without build metadata:** `src/commands/doctor-rebuild-guard.ts:79-80`
  - body: `doctorCommand` passes the resolved OpenClaw package root for source checkouts too, and normal source trees have `package.json` plus `pnpm-workspace.yaml` but no `dist/build-info.json`. This branch would label those source installs as corrupt even though there is no npm package rebuild to compare, so the guard should first prove it is evaluating a packaged install or stay inconclusive.
  - confidence: 0.9
- **[P2] Bound the npm registry fetch:** `src/commands/doctor-rebuild-guard.ts:162`
  - body: This runs in the normal `openclaw doctor` path, but the registry request has no abort signal or timeout. On networks that blackhole HTTPS, a nonessential warning check can stall doctor for the transport timeout instead of staying quick and inconclusive.
  - confidence: 0.93
- **[P2] Keep expired cache from producing warnings:** `src/commands/doctor-rebuild-guard.ts:212`
  - body: After rejecting an expired cache entry, the fallback still loads the same cache file unconditionally when fetch is unavailable or offline. That can turn old registry data into fresh `version-unreleased` or rebuild warnings, violating the PR's silent-on-inconclusive behavior.
  - confidence: 0.92
- **[P2] Reject mismatched package metadata:** `src/commands/doctor-rebuild-guard.ts:87-91`
  - body: `package.json.version` is read but not compared with `dist/build-info.json.version`, so a partial update can evaluate the wrong version and report `pristine` or `unreleased` instead of corrupt local metadata. Treat mismatched versions as corrupt before looking up npm publish time.
  - confidence: 0.9
