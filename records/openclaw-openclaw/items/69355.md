## Summary

Keep open: current main still lacks the proposed doctor install-divergence diagnostic, and the PR is useful, but the submitted patch has narrow correctness blockers around unbounded registry I/O, stale cache reuse, and local metadata validation before it can merge.

## What This Changes

The PR adds a warn-only `openclaw doctor` install freshness check comparing local `dist/build-info.json` metadata with npm publish times, plus docs, changelog text, and focused tests.

## Best Possible Solution

Land a rebased doctor-owned diagnostic that stays warn-only, uses bounded registry I/O, treats expired registry data as inconclusive, validates local metadata parity, distinguishes corrupt metadata clearly, and keeps docs aligned.

## Reproduction Assessment

Yes for the review blockers by source inspection: the PR source shows a bare npm registry fetch, TTL-bypassing fallback, and missing package/build-info version parity check. Current-main source inspection also confirms the requested doctor diagnostic is not already present.

## Solution Assessment

No, not as submitted. The direction fits `openclaw doctor`, but the patch should be repaired to preserve current lazy loading and avoid slow or false-positive doctor warnings.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Add a timeout to the npm registry fetch:** `src/commands/doctor-rebuild-guard.ts:162`
  - body: This doctor check calls the npm registry with a bare `fetchFn(NPM_REGISTRY_URL)`. On networks that blackhole outbound HTTPS, a non-essential install note can stall `openclaw doctor`; use the existing timeout fetch helper or an abort signal so failures become inconclusive quickly.
  - confidence: 0.92
- **[P2] Honor cache TTL when registry lookup falls back:** `src/commands/doctor-rebuild-guard.ts:212`
  - body: After `loadFreshCache` rejects stale cache data, the fallback still reads the same cache file without checking age. That can emit false `version-unreleased` or rebuild warnings from expired registry data during offline or outage runs instead of staying silent when current registry data is unavailable.
  - confidence: 0.9
- **[P2] Validate build-info version against package.json:** `src/commands/doctor-rebuild-guard.ts:87-91`
  - body: `package.json.version` is only checked for existence while the registry lookup uses `dist/build-info.json.version`. A partial install with those files out of sync can evaluate the wrong version and suppress the corruption signal this guard is meant to provide.
  - confidence: 0.89
- **[P3] Distinguish malformed metadata from missing files:** `src/commands/doctor-rebuild-guard.ts:67-72`
  - body: `readJsonFile` collapses missing files and JSON parse failures into `null`, so malformed `build-info.json` is reported as `build-info-missing`. That points operators at the wrong remediation for a corrupt metadata file.
  - confidence: 0.86
