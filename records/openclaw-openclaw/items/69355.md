## What This Changes

The PR adds a doctor rebuild guard that reads local package build metadata, compares it with npm registry publish times using a cache, warns on rebuilt/unreleased/corrupt installs, and updates doctor wiring, tests, docs, and changelog.

## Best Possible Solution

Land a tightened doctor-owned diagnostic that fits the current lazy doctor flow, uses bounded npm registry fetching, treats expired registry data as inconclusive instead of warning, validates package/build-info version parity, distinguishes missing from malformed metadata, and updates docs/changelog/tests in repo style.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Add a timeout around the npm registry fetch:** `src/commands/doctor-rebuild-guard.ts:154`
  - body: This doctor path calls the npm registry with a bare `fetchFn(NPM_REGISTRY_URL)`. On networks that blackhole or severely delay outbound HTTPS, a non-essential install note can stall `openclaw doctor` instead of returning an inconclusive silent result; use the existing timeout fetch helper or an abort signal.
  - confidence: 0.92
- **[P2] Do not revive expired registry cache entries:** `src/commands/doctor-rebuild-guard.ts:212`
  - body: After rejecting stale cache entries with `loadFreshCache`, the fallback still reads the same cache file without checking age. That can emit false `version-unreleased` or rebuild warnings from old registry data during offline/outage runs, contrary to the PR's silent-on-inconclusive contract.
  - confidence: 0.9
- **[P2] Validate build-info version against package.json:** `src/commands/doctor-rebuild-guard.ts:87-91`
  - body: `package.json.version` is only checked for existence, while the registry lookup uses `dist/build-info.json.version`. A partial or interrupted install with those files out of sync can evaluate the wrong version and suppress the corruption signal this guard is meant to provide.
  - confidence: 0.89
- **[P3] Distinguish malformed JSON from missing metadata:** `src/commands/doctor-rebuild-guard.ts:67-73`
  - body: `readJsonFile` collapses read failures and JSON parse failures into `null`, so malformed `build-info.json` is reported as `build-info-missing`. That sends operators toward the wrong remediation for a corrupt metadata file.
  - confidence: 0.86
- **[P2] Fix the changelog section and attribution:** `CHANGELOG.md:7-9`
  - body: The added user-facing feature entry creates a `### Features` section with `#TBD` and no contributor credit. Current repo policy uses the active `### Changes`/`### Fixes` sections and requires real reference text plus `Thanks @...` attribution for user-facing entries.
  - confidence: 0.84
