Keep open: current main still lets `format` prompt failures reach shared auth-profile cooldown state, and #76866 plus #76963 are open implementation PRs with closing references that should be reviewed before this issue closes.

## What This Changes

Prevent malformed session transcript format errors from marking the shared provider auth profile in cooldown and blocking unrelated sessions.

## Best Possible Solution

Land one narrow auth-profile failure-policy fix that prevents `format` from mutating shared profile health while preserving true auth, billing, rate-limit, overload, and fallback behavior.

## Solution Assessment

Yes: the narrowest maintainable fix is at the shared auth-profile failure boundary, where session-local `format` failures should be filtered before state is persisted. The open PRs target that boundary and should be reviewed as the implementation path.
