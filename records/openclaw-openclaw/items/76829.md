## Summary

Keep open: current main still lets `format` prompt failures flow into shared auth-profile cooldown, and open PRs #76866 and #76963 are already attached with closing syntax.

## What This Changes

A corrupted Telegram embedded-agent transcript with blank user messages can produce provider 400 format errors that put the shared auth profile into cooldown and block unrelated requests.

## Best Possible Solution

Land one narrow fix that prevents session-local `format` failures from updating shared auth-profile health, while preserving existing transcript repair and true provider-health cooldowns.

## Reproduction Assessment

Yes at source level, but no live Telegram/Anthropic run was performed. A 400/422 request-shape error can classify as `format`, pass through `resolveAuthProfileFailureReason`, and be stored as a shared profile cooldown.

## Solution Assessment

Yes. The narrow maintainable fix is to exclude `format` request/transcript failures at the auth-profile failure-policy boundary, not to add new cooldown scope or config behavior.
