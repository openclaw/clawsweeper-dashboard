## Summary

Current main still allows `format` prompt failures to be recorded as shared auth-profile failures, which can put the profile into cooldown; this issue should stay open while the linked fixing PRs are reviewed.

## What This Changes

The issue reports that malformed session transcripts can surface as `format` failures that mark a shared auth profile in cooldown and block unrelated requests.

## Best Possible Solution

Review and land one of the linked narrow PRs, or an equivalent change that suppresses session-local `format` failures from shared auth-profile health while preserving real provider health cooldowns.

## Reproduction Assessment

Yes at source level: current main can route a `format` prompt failure through `resolveAuthProfileFailureReason`, into `maybeMarkAuthProfileFailure`, and then into profile-wide cooldown state. No live Telegram/Anthropic run was executed.

## Solution Assessment

Yes: filtering `format` at the shared auth-profile failure boundary is the narrowest maintainable fix. Resetting sessions globally or changing fallback classification more broadly would touch more behavior than this bug requires.
