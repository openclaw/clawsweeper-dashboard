## Summary

Current main still lets session-local `format` prompt failures reach shared auth-profile cooldown state, and two open PRs already reference this issue as fixes, so the issue should remain open until one canonical fix lands.

## What This Changes

Report that corrupted session transcripts can produce provider `format` errors which mark the shared auth profile in cooldown and block unrelated requests.

## Best Possible Solution

Land one narrow auth-profile failure-policy fix that suppresses session-local `format` failures from shared profile health while preserving real auth, billing, rate-limit, overload, and model-fallback behavior.

## Reproduction Assessment

Yes, at source level: current main can route a `format` prompt failure through `resolveAuthProfileFailureReason`, into `maybeMarkAuthProfileFailure`, and then into profile-wide cooldown state. No live Telegram/Anthropic run was executed in this read-only review.

## Solution Assessment

Yes: the narrowest maintainable fix is to filter `format` at the shared auth-profile failure boundary rather than changing provider error classification or resetting sessions globally. The linked PRs already target that boundary and should be reviewed as the implementation path.
