## Summary

Keep open: current main has repaired the blank-user transcript case, but a remaining source path still lets `format` request failures mark the shared auth profile in profile-wide cooldown and then skip the provider for unrelated sessions.

## What This Changes

Avoid a corrupted or invalid session transcript causing a format-classified provider error to put the shared auth profile into cooldown and block unrelated requests.

## Best Possible Solution

Keep the blank-user repair and adjust auth-profile failure policy so transcript/request-shape `format` failures stay session-local and cannot put the shared provider profile into cooldown.

## Reproduction Assessment

No high-confidence end-to-end reproduction was established. The source path is clear from the issue log plus current code: `format` still flows into shared auth-profile cooldown, while the exact blank-user transcript corruption path has already changed because repair now rewrites those turns.

## Solution Assessment

Yes, the narrow maintainable fix is to prevent `format` request/transcript failures from marking shared auth-profile health while preserving true `auth`, `billing`, and `rate_limit` cooldown behavior. Another transcript-only repair would not address the cascade if any future malformed local transcript reaches the provider boundary.
