## Summary

Keep open: current main still lets `format` prompt failures flow into shared auth-profile cooldown, and open fixing PRs #76866 and #76963 are already attached with closing syntax.

## What This Changes

Prevent corrupted or malformed session transcripts from putting the shared provider auth profile into cooldown and blocking unrelated requests.

## Best Possible Solution

Land one narrow auth-profile failure-policy change that filters `format` failures before shared profile health is marked, while preserving transcript repair and true provider-health cooldowns.

## Reproduction Assessment

Yes at source level; no live Telegram/Anthropic run was exercised. A classified `format` prompt failure resolves to an auth-profile failure and then persists as profile-wide cooldown on current main.

## Solution Assessment

Yes, the narrow maintainable fix is to return `null` for `format` at the auth-profile failure-policy boundary before `maybeMarkAuthProfileFailure`, without changing real auth, billing, rate-limit, overload, or transcript-repair behavior.
