## Summary

Keep open: current main still passes `format` prompt failures into shared auth-profile cooldown, and open fixing PRs #76866 and #76963 already reference this issue as their closing target.

## What This Changes

The issue reports that corrupted or blank session transcript turns can produce a `format` provider error that puts the shared auth profile into cooldown and blocks unrelated Telegram embedded-agent requests.

## Best Possible Solution

Land one narrow auth-profile policy fix that filters session-local `format` failures before shared profile health is marked, while preserving the existing transcript repair behavior.

## Reproduction Assessment

Yes at source level: a prompt failure classified as `format` passes through `resolveAuthProfileFailureReason` and can be persisted as shared auth-profile cooldown. I did not run a live Telegram/Anthropic reproduction.

## Solution Assessment

Yes, the narrow maintainable fix is to exclude `format` request/transcript failures at the auth-profile failure-policy boundary. Per-session cooldown isolation would be broader than needed for this bug.
