## Summary

Keep open: current main has repaired the blank-user transcript case, but `format` request failures still flow into shared auth-profile cooldown; #76866 is an open closing PR for the remaining policy fix, so this issue should stay open until that PR lands or is closed.

## What This Changes

The issue asks OpenClaw to keep corrupted transcript format errors session-local instead of placing the shared provider auth profile into cooldown and blocking unrelated requests.

## Best Possible Solution

Keep the blank-user transcript repair and land #76866 or an equivalent narrow change that prevents `format` request/transcript failures from marking shared auth-profile health, with regression coverage for the policy boundary.

## Reproduction Assessment

No end-to-end live reproduction was established in this read-only review. The current-main source path is high confidence: a `format` failover reason is still eligible for `markAuthProfileFailure()`, which computes shared cooldown state.

## Solution Assessment

Yes. The narrow maintainable fix is to filter `format` out of auth-profile health marking while preserving true auth, billing, rate-limit, and overload cooldown behavior; a transcript-only repair would not cover future local request-shape failures.
