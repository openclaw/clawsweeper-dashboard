## Summary

Keep open: current `main` still lets `format` failures flow into shared auth-profile cooldown, and two open unmerged PRs already reference this issue with closing syntax.

## What This Changes

The issue reports that a malformed embedded-agent transcript can produce a provider format error that marks the shared auth profile in cooldown and blocks unrelated requests.

## Best Possible Solution

Land one narrow auth-profile failure-policy fix that keeps transcript/request-shape `format` errors session-local while preserving shared cooldown behavior for real auth, billing, rate-limit, overload, and model-health failures.

## Reproduction Assessment

No high-confidence end-to-end reproduction was run. The source path is high-confidence: `format` still flows from prompt-error classification into shared auth-profile failure marking, and cooldown handling makes non-rate-limit failures profile-wide.

## Solution Assessment

Yes, the narrow maintainable fix is to exclude `format` from shared auth-profile health marking while preserving true provider-health failures. Another transcript-only repair would not cover future malformed local request shapes that reach the provider boundary.
