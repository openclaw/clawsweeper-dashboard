## Summary

Keep open: current main still lets `format` prompt failures reach shared auth-profile cooldown state, and two open fixing PRs are already attached with closing references.

## What This Changes

The report asks OpenClaw to prevent corrupted session transcripts or provider `format` errors from putting the shared provider auth profile into cooldown and blocking unrelated requests.

## Best Possible Solution

Land one narrow auth-profile failure-policy change that treats session-local `format` failures like `timeout` for shared profile health while preserving auth, billing, rate-limit, overload, and model-fallback behavior.

## Reproduction Assessment

Yes at source level: current main can route a `format` prompt failure through `resolveAuthProfileFailureReason`, into `maybeMarkAuthProfileFailure`, and then into profile-wide cooldown state. No live Telegram/Anthropic run was executed in this read-only review.

## Solution Assessment

Yes: the narrow maintainable fix is to filter `format` at the shared auth-profile failure boundary. Resetting sessions globally or broadening provider fallback classification would be a wider change than the observed cooldown cascade needs.
