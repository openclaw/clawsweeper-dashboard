## Summary

Keep open: current main still allows `format` prompt failures to reach shared auth-profile cooldown state, and open closing PRs #76866 and #76963 are already attached as implementation candidates.

## What This Changes

The issue reports that corrupted embedded-agent transcripts can trigger `format` provider errors that place a shared auth profile in cooldown and block unrelated requests.

## Best Possible Solution

Land one narrow auth-profile failure-policy fix that treats session-local `format` failures like `timeout` for shared profile health while preserving auth, billing, rate-limit, overload, and model-fallback behavior.

## Reproduction Assessment

Yes at source level: current main can route a `format` prompt failure through `resolveAuthProfileFailureReason`, into `maybeMarkAuthProfileFailure`, and then into profile-wide cooldown state. No live Telegram/Anthropic request was run.

## Solution Assessment

Yes: the narrowest maintainable fix is to filter `format` at the shared auth-profile failure boundary, not to reset sessions globally or change provider fallback classification broadly.
