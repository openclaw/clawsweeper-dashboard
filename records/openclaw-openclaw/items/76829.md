## Summary

Keep open: current main still lets session-local `format` prompt failures reach shared auth-profile cooldown, and open fixing PRs #76866 and #76963 already reference this issue with closing syntax.

## What This Changes

The report asks OpenClaw to prevent corrupted session transcripts and 400 format errors from putting the shared provider auth profile into cooldown and blocking unrelated sessions.

## Best Possible Solution

Land one narrow auth-profile failure-policy fix that filters `format` before shared profile health is marked, while preserving the existing blank-user transcript repair.

## Reproduction Assessment

Yes at source level, though not with a live Telegram/Anthropic run. A request-shape failure classified as `format` flows through the resolver and mark-helper into shared cooldown state on current main.

## Solution Assessment

Yes, the narrow maintainable fix is to exclude `format` at the auth-profile failure-policy boundary. Resetting or repairing transcripts alone would not protect future malformed request shapes that reach the provider boundary.
