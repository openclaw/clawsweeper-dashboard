## Summary

Keep open: current main still lets `format` prompt failures flow into shared auth-profile cooldown state, and open fixing PRs #76866 and #76963 already reference this issue.

## What This Changes

Prevent malformed transcript `format` errors from marking a shared auth profile unhealthy and blocking unrelated embedded-agent requests.

## Best Possible Solution

Land one narrow auth-profile failure-policy fix that treats session-local `format` failures like `timeout` for shared profile health while preserving true auth, billing, rate-limit, overload, and model-fallback behavior.

## Reproduction Assessment

Yes at source level: current main can route a `format` prompt failure through `resolveAuthProfileFailureReason`, into `maybeMarkAuthProfileFailure`, and then into profile-wide cooldown state. No live Telegram/Anthropic run was executed in this read-only review.

## Solution Assessment

Yes: the narrowest maintainable fix is to filter `format` at the shared auth-profile failure boundary, not to reset sessions globally or change provider fallback classification broadly. The two open PRs both target that boundary and should be reviewed as the implementation path.
