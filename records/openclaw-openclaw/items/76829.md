## Summary

Keep open: current main still lets `format` prompt failures reach shared auth-profile cooldown, and open fixing PRs #76866 and #76963 are already attached with closing syntax.

## What This Changes

A corrupted session transcript with blank user turns can trigger a provider `format` failure that marks the shared auth profile in cooldown and blocks unrelated requests.

## Best Possible Solution

Land one narrow auth-profile policy fix that filters session-local `format` failures before shared profile health is marked, while keeping the shipped blank-user transcript repair.

## Reproduction Assessment

No high-confidence live reproduction was exercised in this read-only pass. The source-level path is high-confidence: a `format` prompt failure is resolved to a profile failure and then stored as shared cooldown state.

## Solution Assessment

Yes, the narrow maintainable fix is to exclude `format` request/transcript failures at the auth-profile failure-policy boundary while preserving true auth, billing, rate-limit, and overload cooldown behavior.
