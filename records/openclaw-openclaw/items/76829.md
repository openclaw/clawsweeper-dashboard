## Summary

Keep open: current main still lets `format` prompt failures flow into shared auth-profile failure marking and profile-wide cooldown, while two linked fixing PRs are already open for maintainer review.

## What This Changes

The issue asks OpenClaw to prevent malformed session transcript format errors from putting a shared provider auth profile into cooldown and blocking unrelated requests.

## Best Possible Solution

Land one narrow auth-profile failure-policy fix that excludes session-local `format` failures from shared profile health while preserving auth, billing, rate-limit, overload, and model-not-found handling.

## Reproduction Assessment

Yes at source level: a prompt error classified as `format` goes through the auth-profile resolver into `maybeMarkAuthProfileFailure`, then writes profile-wide cooldown state. No live Telegram/Anthropic run was performed.

## Solution Assessment

Yes: the narrowest maintainable fix is to filter `format` at the shared auth-profile failure boundary rather than resetting sessions globally or changing provider fallback classification broadly.
