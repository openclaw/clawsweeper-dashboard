## Summary

Keep open: current main still lets `format` failures flow into shared auth-profile cooldown, and the linked open PR #76866 is the active implementation candidate rather than a reason to close this issue before merge.

## What This Changes

Prevent corrupted-session/request-format failures from placing the shared provider auth profile into cooldown and blocking unrelated requests.

## Best Possible Solution

Review and land #76866 or an equivalent narrow change that keeps request-shape `format` failures out of shared auth-profile cooldown while preserving true auth, billing, overload, and rate-limit cooldown behavior.

## Reproduction Assessment

No high-confidence end-to-end reproduction was established. The source path is clear on current main: prompt-side `format` failures still reach `markAuthProfileFailure`, which records a shared cooldown and lets fallback skip all profiles for that provider.

## Solution Assessment

Yes. The narrow maintainable fix is to exclude `format` request/transcript failures from shared auth-profile health marking; #76866 is already targeting that policy boundary.
