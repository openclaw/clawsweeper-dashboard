## Summary

Current `main` still has the failing boundary: `format` prompt failures can flow into auth-profile failure marking, so this issue should stay open while the linked fixing PRs are reviewed.

## What This Changes

The issue reports that malformed session transcripts can produce `format` prompt failures that poison shared provider auth-profile cooldown state and block unrelated requests.

## Best Possible Solution

Land one narrow auth-profile failure-policy fix that suppresses session-local `format` failures from shared profile health while preserving real auth, billing, rate-limit, overload, and model-fallback behavior.

## Reproduction Assessment

Yes at source level: current `main` can route a `format` prompt failure through `resolveAuthProfileFailureReason`, into `maybeMarkAuthProfileFailure`, and then into profile-wide cooldown state. No live Telegram/Anthropic run was executed in this read-only review.

## Solution Assessment

Yes, the narrowest maintainable solution is to filter `format` at the shared auth-profile failure boundary rather than resetting sessions globally or changing provider fallback classification broadly. The two open fixing PRs already target that boundary and should be reviewed as the implementation path.
