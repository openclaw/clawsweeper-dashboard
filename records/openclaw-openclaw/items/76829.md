## Summary

Keep open: current main still lets `format` failover reasons flow into shared auth-profile cooldown, and open PR #76866 explicitly targets this issue with closing syntax, so the issue should remain open until that PR lands or is replaced.

## What This Changes

The issue requests preventing corrupted or blank transcript format errors from putting a shared provider auth profile into cooldown and blocking unrelated sessions.

## Best Possible Solution

Review and land or replace #76866 so request-shape `format` failures stay out of shared auth-profile cooldown while preserving the existing blank-user transcript repair.

## Reproduction Assessment

No high-confidence end-to-end reproduction was run. Source inspection gives a clear current-main path: a `format` failover reason is resolved to an auth-profile failure and persisted as a profile-wide cooldown.

## Solution Assessment

Yes. The narrow maintainable fix is to exclude request/transcript `format` failures from shared auth-profile health marking, rather than adding session-specific cooldown config or broad fallback rewrites.
