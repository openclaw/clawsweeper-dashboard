## Summary

Keep open: current main still lets prompt-side `format` failures flow into shared auth-profile cooldown state, and #76866 plus #76963 are open implementation PRs with closing references. The issue should remain open until one fix lands or the PRs are closed.

## What This Changes

The issue reports that malformed session transcripts can produce provider `format` errors that mark a shared auth profile in cooldown and block unrelated requests.

## Best Possible Solution

Land one narrow auth-profile failure-policy fix that treats session-local `format` failures like `timeout` for shared profile health while preserving true auth, billing, rate-limit, overload, and model-fallback behavior.

## Reproduction Assessment

Yes at source level: current main can route a prompt-side `format` failure through `resolveAuthProfileFailureReason`, into `maybeMarkAuthProfileFailure`, and then into profile-wide cooldown state. I did not run a live Telegram/Anthropic reproduction in this read-only review.

## Solution Assessment

Yes: the narrowest maintainable solution is to filter `format` at the shared auth-profile failure boundary, not to reset sessions globally or alter provider fallback classification broadly. The open PRs already target that boundary and should be reviewed as the implementation path.
