## Summary

Keep open: current main still lets `format` failures flow into shared auth-profile cooldown state, and #76866 is an open closing PR for the remaining fix.

## What This Changes

Prevent local transcript/request-format failures from putting a shared provider auth profile into cooldown and blocking unrelated sessions.

## Best Possible Solution

Review and land or close #76866; the desired end state is that transcript/request-shape `format` failures remain session-local while real auth, billing, overload, and rate-limit failures keep their existing profile-health behavior.

## Reproduction Assessment

No high-confidence end-to-end reproduction was established here. The issue is source-reproducible on current main because `format` still flows from prompt failure classification into `markAuthProfileFailure`, and cooldowned profiles are then skipped by fallback.

## Solution Assessment

Yes. The narrow maintainable fix is to filter `format` out of shared auth-profile health marking, matching the existing timeout precedent without changing true provider-health cooldown behavior.
