## Summary

Keep open: current main still lets `format` failures reach shared auth-profile cooldown, and open PR #76866 is the active fix path for the remaining cascade.

## What This Changes

The issue reports that corrupted session transcript data can produce a provider `format` failure that is recorded as shared auth-profile cooldown, blocking unrelated requests.

## Best Possible Solution

Land or supersede PR #76866 with a focused change that keeps transcript/request-shape `format` failures session-local while preserving real auth, billing, overload, and rate-limit cooldown behavior.

## Reproduction Assessment

No high-confidence end-to-end reproduction was run. The source path is clear on current main: `format` is still returned by the auth-profile failure resolver and then recorded as a shared cooldown reason, while the exact blank-user transcript repair path has changed.

## Solution Assessment

Yes, the linked PR direction is the narrow maintainable solution: stop `format` failures from mutating shared auth-profile health. A transcript-only fix would not prevent future malformed local requests from poisoning the provider profile.
