## Summary

Keep open: current main still allows `format` request failures to flow into shared auth-profile cooldown, and the linked fix PR #76866 is open but unmerged.

## What This Changes

The issue reports that corrupted session transcripts can produce provider `format` errors that mark a shared auth profile in cooldown and block unrelated requests.

## Best Possible Solution

Land or replace the narrow #76866 fix so `format` failures remain request/session-local while preserving shared cooldown behavior for real provider-health failures.

## Reproduction Assessment

No high-confidence end-to-end reproduction was run. Source inspection shows a current-main path where a `format` failure is resolved into auth-profile failure state and then treated as provider cooldown.

## Solution Assessment

Yes, the best fix is the narrow policy change already proposed in #76866: keep `format` out of shared auth-profile cooldown without changing true auth, billing, rate-limit, or overload handling.
