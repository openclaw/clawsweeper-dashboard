## Summary

Keep open: current main still allows prompt-side `format` failures to reach shared auth-profile cooldown state, and #76866/#76963 are open closing PRs that should be reviewed instead of closing the issue now.

## What This Changes

Prevent malformed session transcript `format` failures from marking shared provider auth profiles in cooldown and blocking unrelated requests.

## Best Possible Solution

Land one narrow auth-profile failure-policy change that treats session-local `format` failures like `timeout` for shared profile health, with regression coverage, while keeping session-file repair separate.

## Reproduction Assessment

Yes at source level: current main can classify request-shape provider failures as `format`, resolve that reason for auth-profile failure, and persist shared cooldown state. I did not run a live Telegram/Anthropic scenario in this read-only pass.

## Solution Assessment

Yes: the narrow maintainable fix is to filter `format` at the auth-profile health boundary so it remains a request/failover signal without poisoning shared provider state. The open PRs #76866 and #76963 are the implementation candidates to review.
