## Summary

Keep open: current main still lets `format` prompt failures flow into shared auth-profile cooldown, and open fixing PRs #76866 and #76963 are already attached with closing syntax.

## What This Changes

The issue reports that malformed session transcripts can produce `format` provider errors that mark a shared auth profile in cooldown and block unrelated requests.

## Best Possible Solution

Land one narrow auth-profile policy fix that filters session-local `format` failures before shared profile health is marked, while preserving true auth, billing, rate-limit, and overload cooldown behavior.

## Reproduction Assessment

Yes for source-level reproduction: a prompt `format` failure is resolved to an auth-profile failure reason and then stored as shared cooldown state. No live Telegram/Anthropic run was executed in this read-only pass.

## Solution Assessment

Yes, the narrow maintainable solution is to exclude `format` request/transcript failures at the auth-profile failure-policy boundary while keeping real provider-health failures eligible for cooldown.
