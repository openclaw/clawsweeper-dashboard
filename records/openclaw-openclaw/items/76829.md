## Summary

Keep open: current main still lets `format` prompt failures flow into shared auth-profile cooldown, and two open fixing PRs are already linked with closing syntax.

## What This Changes

The issue reports that malformed session transcripts can produce provider `format` failures that are recorded as shared auth-profile cooldowns, blocking unrelated requests.

## Best Possible Solution

Land one narrow auth-profile policy fix that filters session-local `format` failures before shared profile health is marked, while preserving true auth, billing, rate-limit, overload, and model-health cooldown behavior.

## Reproduction Assessment

Yes at source level: a prompt failure classified as `format` reaches `resolveAuthProfileFailureReason`, then `maybeMarkAuthProfileFailure`, then profile-wide cooldown state. A live Telegram/Anthropic reproduction was not run in this read-only pass.

## Solution Assessment

Yes: the narrow maintainable fix is to exclude `format` at the auth-profile failure-policy boundary, not to change transcript repair, provider classification, or cooldown duration semantics broadly.
