## Summary

Keep open: current main fixes the logged blank-user transcript repair path, but `format` request-shape failures can still be recorded as shared auth-profile cooldowns, so the provider-wide cascade remains possible for unrepaired transcript/request corruption.

## What This Changes

The issue asks OpenClaw to recover from corrupted embedded-agent session transcripts without putting the shared provider auth profile into cooldown and blocking unrelated requests.

## Best Possible Solution

Keep the blank-user transcript repair and define a narrow policy that handles local transcript/request-shape `format` failures session-locally, with regression coverage showing unrelated sessions and `/new` are not blocked.

## Reproduction Assessment

No high-confidence end-to-end reproduction was established on current main. Source inspection shows the remaining cascade path: a `format` prompt failure still marks shared auth-profile cooldown state and fallback skips the provider while all profiles are unavailable.

## Solution Assessment

Unclear until a maintainer scopes `format` cooldown policy. The likely narrow fix belongs around auth-profile failure classification and transcript repair, not a broad fallback rewrite, because `format` also represents genuine provider request-schema failures.
