## Summary

Keep open: current main still lets `format` prompt failures flow into shared auth-profile cooldown, and the two linked implementation PRs are open but unmerged.

## What This Changes

The issue requests that malformed or repaired session transcripts stop poisoning shared provider cooldown state so unrelated sessions and `/new` remain usable.

## Best Possible Solution

Land one narrow auth-profile failure-policy change that treats `format` as session-local before marking shared profile health, while preserving transcript repair and true provider cooldown behavior.

## Reproduction Assessment

Yes, at source level: current main maps `format` through the resolver, then the prompt-error path marks auth-profile failure and usage records a profile-wide cooldown. I did not run a live Telegram/Anthropic reproduction, so this is source-reproducible rather than live-reproduced.

## Solution Assessment

Yes, the narrow maintainable fix is the linked PR shape: exclude `format` at the auth-profile failure-policy boundary. Resetting sessions or adding per-session cooldown isolation would be broader than needed for the remaining cascade.
