## Summary

Keep open: current main still allows `format` prompt failures to flow into shared auth-profile cooldown state, and two open unmerged PRs already reference this issue with closing syntax.

## What This Changes

The issue reports that a corrupted embedded-agent transcript can turn a local request-format failure into shared provider cooldown, blocking unrelated sessions and `/new` requests.

## Best Possible Solution

Land one narrow auth-profile failure-policy fix that excludes request-local `format` failures from shared profile health while preserving true auth, billing, rate-limit, overload, and model availability cooldown behavior.

## Reproduction Assessment

Yes at source level, but not live: a 400/request-format prompt failure can resolve to `format`, pass through the auth-profile failure resolver, and be stored as shared cooldown state. I did not run a live Telegram/Anthropic corrupted-transcript session.

## Solution Assessment

Yes: the linked PR direction that filters `format` at the auth-profile failure-policy boundary is the narrowest maintainable fix. It avoids changing transcript repair, provider classification, or cooldown duration semantics.
