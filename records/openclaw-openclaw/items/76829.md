## Summary

Keep open: current main appears to have fixed the reported blank-user transcript repair path, but the source still allows `format` provider/request-shape failures to mark the shared auth profile in cooldown, which is the cascade the issue is concerned about.

## What This Changes

The issue asks OpenClaw to prevent a corrupted Telegram embedded-agent transcript from producing invalid provider requests and globally cooling down the Anthropic auth profile.

## Best Possible Solution

Keep the blank-user repair, then narrow auth-profile cooldown policy so transcript/request-shape format failures are handled session-locally or by transcript reset without disabling unrelated sessions.

## Reproduction Assessment

No high-confidence end-to-end reproduction was established. The source path is clear, but the exact blank-user transcript failure has changed on current main because repair now rewrites blank user turns instead of dropping them.

## Solution Assessment

Unclear without maintainer policy input. The session repair direction is already good, but the remaining global cooldown behavior for `format` failures needs a scoped decision before an automated fix changes provider fallback semantics.
