## Summary

Keep open: current main repairs the blank-user transcript case, but `format` request-shape failures still flow into shared auth-profile cooldown; #76866 is the open closing PR for the remaining policy fix.

## What This Changes

Prevent corrupted session transcripts and resulting 400 format errors from putting the shared provider auth profile into cooldown and blocking unrelated sessions.

## Best Possible Solution

Review and land #76866 or an equivalent narrow change that keeps request/transcript `format` failures out of shared auth-profile cooldown while preserving true auth, billing, overload, and rate-limit cooldown behavior.

## Reproduction Assessment

No live end-to-end reproduction was established, but the current-main source path is high confidence: `format` is returned as an auth-profile failure reason, marked on the active profile, persisted as cooldown, and then used to skip the provider. The original blank-user transcript corruption path is partially changed by current repair coverage.

## Solution Assessment

Yes, the best narrow fix is to prevent `format` request/transcript failures from marking shared auth-profile health. Transcript repair alone would not prevent future malformed local request payloads from causing the same provider-wide cooldown cascade.
