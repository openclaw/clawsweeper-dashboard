## Summary

Keep open. Current main still has success-only JSON output for the affected Gateway-backed commands, and the active closing implementation at https://github.com/openclaw/openclaw/pull/79233 should land or be closed before this issue is resolved.

## What This Changes

The issue asks for Gateway-backed `--json` commands to emit structured stdout JSON, not only stderr text, when the Gateway transport connection fails.

## Best Possible Solution

Land one focused Gateway transport-error JSON implementation through https://github.com/openclaw/openclaw/pull/79233, then close this issue through the merge.

## Reproduction Assessment

Yes, source-level. Current main awaits Gateway RPC calls before stdout JSON writes in `health`, `gateway health`, and `devices list`, and the issue provides concrete unavailable-Gateway commands for the failing paths.

## Solution Assessment

Yes. The maintainable fix is a shared serializer for typed `GatewayTransportError` failures in JSON-mode command wrappers, while preserving existing human stderr behavior outside JSON mode.
