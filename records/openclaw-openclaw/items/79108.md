## Summary

Keep open. Current `main` still has the source-level JSON failure path, and two open fix PRs explicitly reference this issue, so the issue should stay open until one implementation lands or the PRs are resolved.

## What This Changes

The issue asks for gateway-backed `--json` CLI commands to emit structured JSON on gateway connection failure instead of leaving stdout empty.

## Best Possible Solution

Land one reviewed implementation that adds shared `GatewayTransportError` JSON serialization for the affected JSON-mode commands while preserving non-JSON stderr behavior.

## Reproduction Assessment

Yes, source-level. On current `main`, `health --json`, `gateway health --json`, and `devices list --json` all await gateway RPC work before writing stdout JSON, while typed transport errors can escape first.

## Solution Assessment

Yes. The narrow maintainable fix is a shared transport-error JSON formatter applied at the gateway-backed CLI command wrappers, plus focused regression tests; the existing open PRs are the right implementation candidates to review.
