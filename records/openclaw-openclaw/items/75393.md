Keep open. The PR targets the right provider-owned compatibility flag, but it does not fix the registered Copilot Claude models used by the linked reproduction because those models come from the manifest catalog, not the changed fallback helper; it also lacks the required changelog entry.

## What This Changes

The PR adds `compat.supportsEagerToolInputStreaming: false` to the GitHub Copilot model-default helper and the synthetic dynamic fallback model builder.

## Best Possible Solution

Land a provider-owned Copilot fix that sets `supportsEagerToolInputStreaming: false` on every Anthropic GitHub Copilot model path that can reach pi-ai, including the manifest/static catalog and dynamic fallback, with a focused regression assertion and a user-facing changelog entry.
