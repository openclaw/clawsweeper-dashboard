## Summary

Keep open: the PR uses the right compat flag, but the branch is incomplete because it only changes the defaults helper and synthetic fallback while registered Copilot Claude rows and live catalog rows still lack the flag; external real behavior proof is also still missing.

## What This Changes

The PR adds `compat.supportsEagerToolInputStreaming: false` to the Copilot model defaults helper and the forward-compatible fallback model builder.

## Best Possible Solution

Land a Copilot provider-owned fix that applies `supportsEagerToolInputStreaming: false` to every Anthropic Copilot model source, including static catalog rows, live `/models` rows, and dynamic fallback models, with regression coverage and redacted real behavior proof.

## Reproduction Assessment

Yes for source-level reproduction: the linked issue provides concrete v2026.4.29 logs, a later comment confirms v2026.5.4, and current `main` plus pi-ai 0.73.1 still show the same missing compat metadata path. I did not run a live Copilot account in this read-only review.
