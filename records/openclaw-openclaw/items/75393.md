## Summary

Keep open. The reported Copilot/pi-ai compatibility bug is valid and the compat flag is the right mechanism, but this PR does not fix the registered Copilot Claude models that hit the linked reproduction: it only patches the synthetic fallback path and an unused default-model helper, while the manifest catalog remains missing the flag and current catalog normalization would drop that compat key unless updated. The PR also needs the required user-facing changelog entry before merge.

## What This Changes

The PR adds `compat.supportsEagerToolInputStreaming: false` to the GitHub Copilot default-model helper and dynamic fallback model builder.
