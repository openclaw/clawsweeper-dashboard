## Summary

Keep open. Current main has global `before_tool_call` / `after_tool_call` hooks and a bundled-only trusted tool policy tier, but it does not implement tool-scoped hook registration or a bundled `fork-guard` plugin. The PR is also far too broad for its stated feature and needs maintainer review/splitting rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer judgment, but ask for a narrow rebased PR split into: (1) an additive, documented tool-scoped hook API with contract tests and no third-party breakage, and (2) a separate bundled fork-guard plugin or trusted tool policy with focused exec-command parsing, config docs, and security review. The current broad branch should not be merged as-is.
