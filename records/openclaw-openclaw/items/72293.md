## Summary

Keep this open. Current main still has a deterministic source-level path where a lifecycle `end` with `stopReason: "aborted"` is preserved but classified as successful by `agent.wait` and subagent completion, and the directly related fix work remains open rather than merged.

## What This Changes

The issue reports that aborted, empty subagent terminal turns are treated as successful completion and do not produce a parent-visible failure announcement.

## Best Possible Solution

Propagate terminal `stopReason: "aborted"` through existing failure/timeout lifecycle, `agent.wait`, subagent completion, and announcement paths with focused regression tests; review, land, or replace the open fix work instead of closing the report early.

## Reproduction Assessment

Yes. Emit a lifecycle event with `phase: "end"`, `stopReason: "aborted"`, and no `aborted: true`; current main preserves the stop reason but classifies `agent.wait` and subagent completion as success.

## Solution Assessment

Yes. The narrow maintainable fix is to normalize the existing abort signal into current non-success paths; adding retries or a new public `aborted` status would be broader product work and is not required to prevent the parent-visible failure gap.
