## Summary

Keep this open. Current main still treats a terminal lifecycle `end` with `stopReason: "aborted"` as a successful agent wait and subagent completion unless `aborted: true` is also present, and the linked fix work has not merged.

## What This Changes

The report asks OpenClaw to classify aborted empty-output subagent runs as failed or aborted and deliver a parent-visible completion instead of treating them as done.

## Best Possible Solution

Normalize terminal `stopReason: "aborted"` into the existing non-success lifecycle, `agent.wait`, subagent completion, and parent-visible announcement paths, with focused regression tests; land or replace the linked fix work rather than adding a new public status first.

## Reproduction Assessment

Yes. A deterministic source-level reproduction is to emit a lifecycle event with `phase: "end"`, `stopReason: "aborted"`, and no `aborted: true`; current main preserves the stop reason but classifies agent wait and subagent completion as success.

## Solution Assessment

Yes, the narrowest maintainable fix is to propagate the existing abort signal through current failure paths. Auto-retry or a new public `aborted` status would be broader product work and is not needed to prevent the parent-visible black-hole.
