## Summary

Keep this open. Current main still lets a terminal lifecycle `end` with `stopReason: "aborted"` but no `aborted: true` flow through `agent.wait` and subagent completion as a successful result, and the linked fix work has not merged.

## What This Changes

The issue asks OpenClaw to treat empty subagent terminal turns with `stopReason: "aborted"` as failed/aborted and surface a parent-visible completion instead of silently succeeding.

## Best Possible Solution

Normalize terminal `stopReason: "aborted"` into the existing non-success lifecycle, `agent.wait`, subagent completion, and parent-visible announcement paths, with focused regression tests; land or replace the linked fix work rather than adding a new public status first.

## Reproduction Assessment

Yes. Emit a lifecycle event with `phase: "end"`, `stopReason: "aborted"`, and no `aborted: true`; current main preserves the stop reason but maps wait and subagent completion to success.

## Solution Assessment

Yes for the narrow direction. Propagating the existing abort signal through current failure paths is maintainable; adding automatic retry or a new public status should be separate product work.
