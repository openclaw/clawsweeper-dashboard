## Summary

Keep this open. Current main still lets a terminal lifecycle `end` with `stopReason: "aborted"` flow through `agent.wait` and subagent completion as success, and the open closing-syntax fix work has not merged.

## What This Changes

The issue asks OpenClaw to treat empty aborted subagent terminal turns as parent-visible failures instead of successful silent completions.

## Best Possible Solution

Land or replace the linked fix with a narrow change that normalizes terminal `stopReason: "aborted"` into existing non-success lifecycle, `agent.wait`, and subagent completion outcomes, with focused regression coverage.

## Reproduction Assessment

Yes. A deterministic source-level reproduction is to emit a lifecycle event with `phase: "end"`, `stopReason: "aborted"`, and no `aborted: true`; current main preserves the stop reason but maps wait and subagent completion to success.

## Solution Assessment

Partly. Propagating the existing abort signal through current failure paths is the narrowest maintainable fix; adding auto-retry or a new public `aborted` status would be broader product work.
