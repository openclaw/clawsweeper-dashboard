## Summary

Keep this open. Current main still has a source-visible classification gap: a final assistant `stopReason: "aborted"` can be carried as lifecycle metadata, but the lifecycle producer and parent-visible consumers only treat `phase: "error"` or `aborted: true` as terminal failure/timeout paths. The issue is also paired with open related fix PRs, so it should remain open until a fix lands or maintainers choose a replacement.

## What This Changes

The issue asks OpenClaw to classify empty-content aborted subagent runs as parent-visible failures or aborted outcomes instead of silently treating them as successful completion.

## Best Possible Solution

Land or replace a narrow core fix that treats terminal assistant `stopReason: "aborted"` as a parent-visible failed, timeout, or killed outcome across lifecycle emission, `agent.wait`, subagent registry completion, and announce formatting, with focused regression tests. Avoid bundling auto-retry policy or a new public status union unless maintainers explicitly choose that broader behavior.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to emit lifecycle `phase: "end"` with `stopReason: "aborted"` and no `aborted: true`; current main preserves the stop reason but `agent.wait` and the subagent registry still classify that terminal event as successful.

## Solution Assessment

Yes. The best fix is a narrow classification and announcement repair in the core subagent lifecycle paths; auto-retry or a new public `aborted` status would be broader product behavior than the minimal bug fix needs.
