## Summary

Keep this open. Current main still preserves terminal `stopReason: "aborted"` in lifecycle metadata, but the embedded lifecycle producer, `agent.wait`, and subagent registry completion path do not consistently promote that stop reason into a parent-visible failure or killed outcome. The issue is also paired with an open linked fix PR using closing syntax, so it should remain open until that PR or an equivalent replacement lands.

## What This Changes

The issue asks OpenClaw to classify empty aborted subagent terminal turns as failed or killed and to surface a parent-visible completion/error instead of silently treating them as successful completion.

## Best Possible Solution

Land or replace a narrow core fix that normalizes a final assistant `stopReason: "aborted"` into a parent-visible failed or killed outcome across lifecycle emission, `agent.wait`, subagent registry completion, announce formatting, and focused tests, without adding a new public status union or provider retry policy unless maintainers explicitly choose that larger scope.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to emit lifecycle `phase: "end"` with `stopReason: "aborted"` and no `aborted: true`; current main keeps the stop reason but `agent.wait` and the subagent registry still classify that terminal event as successful.

## Solution Assessment

Yes. The narrow classification fix described by the linked PR is the maintainable path; auto-retry or a new public `aborted` status would be broader product behavior and should not be bundled into the minimal bug fix by default.
