## Summary

Keep this open. Current main still has the stopReason-only abort classification gap in the lifecycle, agent.wait, and subagent registry paths, and the issue is paired with open fix work that should be reviewed or deliberately replaced before cleanup.

## What This Changes

The issue requests that aborted subagent terminal assistant messages with empty content be classified as failed/aborted and surfaced to the parent instead of silently completing as done.

## Best Possible Solution

Review and land, replace, or close the open fix work with a narrow change that normalizes terminal stopReason "aborted" into existing non-success lifecycle, wait, and subagent completion outcomes, preserves sanitized error text, and adds focused regression coverage for the producer and both consumers.

## Reproduction Assessment

Yes. A deterministic reproduction is to emit a lifecycle phase "end" event with stopReason "aborted" and no aborted: true; current main preserves the stop reason but still maps agent.wait and subagent completion to success.

## Solution Assessment

Yes. The narrowest maintainable solution is to propagate the existing abort signal through current lifecycle consumers and tests; automatic provider retry or a new public status union would be broader product behavior.
