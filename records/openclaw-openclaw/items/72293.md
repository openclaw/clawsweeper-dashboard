- `impact:message-loss`: The reported failure is a missing or suppressed parent-visible subagent completion/failure announcement.

## Summary

Keep open: current main still has a source-visible path where lifecycle `end` events with `stopReason: "aborted"` but no `aborted: true` become successful `agent.wait` and subagent completion outcomes, even though session lifecycle persistence now maps that stop reason to killed.

## What This Changes

The issue reports that aborted subagent runs with empty assistant output are classified as successful completion and fail to produce a parent-visible failure announcement.

## Best Possible Solution

Normalize `stopReason: "aborted"` into the existing non-success subagent lifecycle, `agent.wait`, registry completion, and parent announce paths, while leaving auto-retry or a new public `aborted` status to separate product work.

## Reproduction Assessment

Yes, source-level with high confidence. Emit a lifecycle `end` event carrying `stopReason: "aborted"` and no `aborted: true`; current main preserves the stop reason but maps `agent.wait` and subagent registry completion to successful outcomes.

## Solution Assessment

Yes. The narrow maintainable fix is to map the existing abort signal into current killed, timeout, or error completion paths and add regression coverage; auto-retry or a new public status would be broader product work.
