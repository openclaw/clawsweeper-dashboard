## Summary

Keep open. Current main still maps terminal lifecycle `end` events carrying only `stopReason: "aborted"` to successful `agent.wait` and subagent completion outcomes, and the linked fix work has not landed.

## What This Changes

The issue asks OpenClaw to classify aborted empty-output subagent terminal states as failed or aborted and notify the parent instead of silently dropping completion.

## Best Possible Solution

Land or replace the linked fix with a narrow change that normalizes terminal `stopReason: "aborted"` into existing non-success `agent.wait` and subagent completion outcomes, with focused regression tests and no broader retry/status redesign unless maintainers choose it.

## Reproduction Assessment

Yes. Emit a lifecycle `phase: "end"` event with `stopReason: "aborted"` and no `aborted: true`, or end an embedded run from a final assistant message with `stopReason: "aborted"`; current main preserves the stop reason but still routes wait/registry completion as success.

## Solution Assessment

Yes for the core direction. The narrowest maintainable fix is to propagate the existing abort signal through lifecycle consumers and completion announcement handling; automatic retry or a new public `aborted` status is broader than needed for this bug.
