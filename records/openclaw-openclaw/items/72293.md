## Summary

Keep this open. Current main still treats a lifecycle terminal `end` carrying only `stopReason: "aborted"` as success for `agent.wait` and subagent completion, and linked fix work is still open rather than merged.

## What This Changes

The report asks OpenClaw to propagate aborted empty-output subagent terminal turns as failed or aborted so parent agents receive a visible completion/error instead of waiting silently.

## Best Possible Solution

Normalize terminal `stopReason: "aborted"` through lifecycle production, `agent.wait`, and subagent completion as an existing non-success outcome, with focused regression tests and no new public status union unless maintainers choose that broader API change.

## Reproduction Assessment

Yes. Emit a lifecycle `phase: "end"` event with `stopReason: "aborted"` and no `aborted: true`; current main preserves the stop reason but maps `agent.wait` and subagent completion to success.

## Solution Assessment

Yes. The narrow maintainable fix is to propagate the existing abort signal into current failure/timeout handling; auto-retry or a new public `aborted` status would be broader product work.
