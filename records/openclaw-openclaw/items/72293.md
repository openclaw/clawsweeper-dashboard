## Summary

Keep this open. Current main still preserves `stopReason: "aborted"` metadata but classifies a terminal lifecycle `end` without `aborted: true` as successful for `agent.wait` and subagent completion, and the linked fix work has not merged.

## What This Changes

Reports that empty-content aborted subagent turns are marked done/ok and can suppress the parent-visible completion announcement.

## Best Possible Solution

Land or replace a narrow fix that maps terminal `stopReason: "aborted"` into the existing non-success lifecycle, `agent.wait`, subagent completion, and parent announcement paths, with focused regression tests and no new public status or retry policy.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to emit a lifecycle `end` with `stopReason: "aborted"` and no `aborted: true`; current main preserves the stop reason but classifies `agent.wait` and subagent completion as success.

## Solution Assessment

Yes. The narrow maintainable fix is to normalize the existing abort signal into current failure/timeout paths; auto-retry or a new public `aborted` status would be broader product work.
