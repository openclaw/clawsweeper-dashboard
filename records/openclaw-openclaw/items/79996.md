## Summary

Keep open: current main still exposes WhatsApp `message_received` hooks only, with no typed `reaction_received` or `reaction_removed` plugin hook. This is a new public WhatsApp/plugin-SDK feature request with privacy and API semantics to settle, not an implemented cleanup candidate.

## What This Changes

Request a WhatsApp plugin hook surface for inbound reaction add/remove events, aligned with message-hook and listen-only semantics.

## Best Possible Solution

Define an additive maintained reaction-hook contract, emit it from the official WhatsApp listener without creating agent runs or outbound activity, and reconcile it with the listen-only work at https://github.com/openclaw/openclaw/issues/78963 and the broader reaction tracking at https://github.com/openclaw/openclaw/issues/11460.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a new public hook/API feature request. Source inspection gives high confidence that current main lacks the requested WhatsApp reaction hook surface.

## Solution Assessment

Unclear until maintainer API review: emitting from the official WhatsApp listener is the right ownership boundary, but the public hook names, payload contract, privacy gating, and listen-only interaction need agreement first.
