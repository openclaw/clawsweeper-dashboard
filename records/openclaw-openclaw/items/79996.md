## Summary

Keep open: current main still lacks `reaction_received`/`reaction_removed` hook names and WhatsApp reaction-update handling, and the request adds a public plugin/API/config surface rather than closing a stale cleanup item. The related reports are useful context, but https://github.com/openclaw/openclaw/issues/11460 is broader reaction querying, https://github.com/openclaw/openclaw/issues/78963 is listen-only mode, and https://github.com/openclaw/openclaw/issues/38714 is Discord-specific.

## What This Changes

Request adding WhatsApp `reaction_received` and optional `reaction_removed` plugin hooks for Baileys reaction updates without creating agent runs or outbound WhatsApp activity.

## Best Possible Solution

Define a maintained reaction-hook contract in the plugin SDK or WhatsApp plugin API, emit it from the official WhatsApp socket listener behind explicit account-scoped opt-in, and align no-agent/no-outbound behavior with https://github.com/openclaw/openclaw/issues/78963.
