## Summary

Keep open: current main still exposes only WhatsApp `message_received` plugin hooks and has no typed reaction add/remove hook or WhatsApp reaction listener. This is a new public plugin/API and WhatsApp privacy/config feature, not a cleanup item that can be safely closed.

## What This Changes

The issue asks to add WhatsApp `reaction_received` and `reaction_removed` plugin hooks emitted from the official WhatsApp listener without creating agent runs or outbound activity.

## Best Possible Solution

Define an additive reaction hook contract in the plugin SDK or WhatsApp plugin API, emit it from the official WhatsApp listener with explicit account-scoped privacy opt-in, and reconcile the design with https://github.com/openclaw/openclaw/issues/38714, https://github.com/openclaw/openclaw/issues/78963, and https://github.com/openclaw/openclaw/issues/11460.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/API request. Source inspection gives high confidence that current main lacks the requested WhatsApp reaction hook surface.

## Solution Assessment

Unclear until maintainer design review. Emitting from the official WhatsApp listener is the right owner boundary, but the exact hook names, payload, privacy gate, and Baileys event mapping need API/product agreement.
