## Summary

Keep open: current main still has no typed `reaction_received` or `reaction_removed` hook, and WhatsApp only exposes opt-in `message_received` hook delivery. This is a new plugin/channel API plus privacy/config decision rather than a cleanup-close candidate.

## What This Changes

Requests WhatsApp `reaction_received` and `reaction_removed` plugin hooks that observe add/remove reactions without creating agent runs or outbound WhatsApp activity.

## Best Possible Solution

Define an additive reaction observation hook contract, emit it from the official WhatsApp listener after the approved account/privacy gates, keep wake-up and outbound behavior out of scope, and align docs/tests with the broader reaction-hook design.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/API request. Source inspection gives high confidence that current main lacks the requested WhatsApp reaction hook surface.

## Solution Assessment

Unclear until product/API review: the official WhatsApp listener is the right owner boundary, but maintainers still need to choose hook names, payload shape, privacy opt-in, and how it composes with listen-only mode and the broader reaction-hook request.
