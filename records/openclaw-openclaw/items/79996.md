## Summary

Keep open: current main does not expose WhatsApp `reaction_received` or `reaction_removed` plugin hooks, and the broader WhatsApp reaction-tracking issue plus open agent-forwarding PRs do not fully cover this listen-only public plugin hook contract.

## What This Changes

Request to add WhatsApp reaction add/remove plugin hooks for inbound Baileys reaction events without creating agent runs or outbound WhatsApp activity.

## Best Possible Solution

Define a maintained WhatsApp reaction hook contract in the plugin SDK or WhatsApp plugin API, emit it from the official WhatsApp listener with privacy-scoped opt-in, and align no-agent/no-outbound behavior with https://github.com/openclaw/openclaw/issues/78963.
