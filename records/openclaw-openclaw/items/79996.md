## Summary

Keep open: current main still lacks a WhatsApp reaction listener and public plugin reaction hook contract; the request is useful but adds privacy-gated hook/config/API surface that overlaps open WhatsApp reaction and listen-only work.

## What This Changes

Add WhatsApp `reaction_received` and optionally `reaction_removed` plugin hooks for reaction add/remove events without agent runs or outbound WhatsApp activity.

## Best Possible Solution

Define an additive, privacy-gated reaction event contract, then implement it in the WhatsApp plugin-owned inbound monitor with tests, docs, no agent-run side effects, and live add/remove/reconnect proof while reconciling the related open reaction and listen-only threads.

## Reproduction Assessment

Not applicable: this is a feature/API request, not broken documented behavior. Source inspection gives high confidence that current main does not expose WhatsApp reactions through a plugin hook today.

## Solution Assessment

Unclear until maintainers choose the public API contract. An in-process WhatsApp-owned reaction hook is the plausible boundary given the report's socket-safety evidence, but final names, payload, opt-in config, and relation to the broader reaction trackers need product direction.
