## Summary

Keep open: current main lacks a WhatsApp reaction listener and public reaction hook contract, and the request would add a new privacy-sensitive plugin/API surface that needs maintainer scope selection.

## What This Changes

Add WhatsApp plugin hook events for reaction add/remove notifications so plugins can observe reaction feedback without opening a second WhatsApp socket or creating agent runs.

## Best Possible Solution

Design an additive reaction event contract, then implement it in the WhatsApp plugin's owned inbound monitor with explicit privacy opt-in, tests, docs, and live add/remove/reconnect proof.

## Reproduction Assessment

Not applicable: this is a feature/API request rather than broken documented behavior. Source inspection of current main shows the requested WhatsApp reaction listener and hook surface are absent.

## Solution Assessment

Unclear until maintainers choose the API contract. An in-process WhatsApp-owned hook is the right direction to avoid unsafe socket workarounds, but naming, payload, privacy gate, and relationship to the cross-channel reaction hook discussion need product/API selection first.
