## Summary

Keep open: current main and the latest shipped release still expose only the opt-in WhatsApp `message_received` hook plus outbound/status reaction support, with no WhatsApp `reaction_received`/`reaction_removed` hook or reaction listener. The request is a useful feature/API expansion with privacy-sensitive payload and config decisions, and the related open PRs/issues are implementation input rather than merged fixes.

## What This Changes

Request to add privacy-gated WhatsApp reaction add/remove plugin hooks without creating agent runs or outbound WhatsApp activity.

## Best Possible Solution

Define an additive WhatsApp-owned, privacy-gated reaction event hook contract, implement it in the official inbound listener, align it with listen-only and cross-channel hook design, and ship tests, docs, and live proof.

## Reproduction Assessment

Not applicable: this is a new feature/API request, not broken documented behavior. Source inspection on current main and `v2026.5.12` gives high confidence that WhatsApp reactions are not exposed through plugin hooks today.

## Solution Assessment

Unclear until maintainers choose the public API and privacy contract. An in-process WhatsApp-owned hook is the plausible boundary, but names, payload, removal semantics, opt-in config, and relation to the query/listen-only trackers need product review.
