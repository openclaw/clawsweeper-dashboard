## Summary

Keep open: current main exposes WhatsApp `message_received` hooks only and has no typed `reaction_received` or `reaction_removed` plugin hook; this is a new WhatsApp/plugin-SDK feature request, not an implemented cleanup candidate.

## What This Changes

Request adding WhatsApp `reaction_received` and optional `reaction_removed` plugin hooks emitted from the official WhatsApp channel listener without starting agent runs or sending outbound activity.

## Best Possible Solution

Define a maintained reaction-hook contract in the plugin SDK or WhatsApp plugin API, emit it from the official WhatsApp socket listener with account-scoped privacy opt-in, and align the no-agent/no-outbound semantics with https://github.com/openclaw/openclaw/issues/78963.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/API request. Source inspection gives high confidence that current main lacks the requested WhatsApp reaction hook surface.

## Solution Assessment
