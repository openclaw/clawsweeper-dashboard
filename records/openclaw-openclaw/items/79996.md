## Summary

Keep open. Current main and v2026.6.8 still expose WhatsApp `message_received` hooks and approval-specific reaction handling, but they do not expose ordinary WhatsApp reaction add/remove events through the public plugin hook API.

## What This Changes

Request to add observation-only WhatsApp plugin hooks for reaction add/remove events with payloads that plugins can consume without creating agent runs or outbound WhatsApp activity.

## Best Possible Solution

Approve a generic observation-only reaction hook contract, then implement WhatsApp add/remove emission inside the existing channel monitor with privacy gating, focused tests, docs, and proof that it creates no agent run or outbound WhatsApp activity.

## Reproduction Assessment

Yes, at source level: current main lacks a public reaction hook name, does not wire a WhatsApp reaction listener, and only handles reactionMessage for approval targets. No live WhatsApp account was exercised in this read-only review.

## Solution Assessment

Yes directionally: a channel-owned in-process observation hook is safer than plugins attaching to Baileys directly. The exact hook names, payload, privacy gating, and cross-channel alignment still need maintainer product/API approval.
