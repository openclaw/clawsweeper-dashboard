## Summary

Keep open: current main still exposes only WhatsApp `message_received` plugin hooks and outbound/ack reaction controls; there is no typed `reaction_received`/`reaction_removed` hook or WhatsApp reaction event listener. This is a valid new plugin/API feature request that needs maintainer design for hook names, payload, privacy opt-in, and overlap with related reaction/listen-only work.

## What This Changes

Request to add WhatsApp `reaction_received` and optionally `reaction_removed` plugin hooks for inbound Baileys reaction events without creating agent runs or outbound activity.

## Best Possible Solution

Define an additive, documented reaction hook contract, emit it from the official WhatsApp socket listener with explicit account/channel privacy opt-in, and align no-agent/no-outbound semantics with the WhatsApp listen-only work.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a plugin/API feature request. Source inspection gives high confidence that current main lacks the requested WhatsApp reaction hook surface.

## Solution Assessment

Unclear until maintainer API review: emitting from the official WhatsApp listener is the right ownership boundary, but hook naming, payload shape, privacy opt-in, and listen-only interaction need product/design agreement.
