## Summary

Keep open: current main still has opt-in WhatsApp `message_received` hooks and outbound `react`, but no WhatsApp reaction listener or reaction hook contract. The request is valid new plugin/API work, with useful production evidence from the reporter, but it needs maintainer scope selection before implementation.

## What This Changes

Add WhatsApp `reaction_received` and optional `reaction_removed` plugin hooks for inbound reaction add/remove events without agent runs or outbound activity.

## Best Possible Solution

Design an additive reaction event contract, then implement it in the WhatsApp plugin's owned inbound monitor with explicit privacy opt-in, tests, docs, and live add/remove/reconnect proof.
