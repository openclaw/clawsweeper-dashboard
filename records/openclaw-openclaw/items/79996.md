## Summary

Keep open: current main still has only the generic `message_received` hook and WhatsApp does not listen for or emit inbound reaction hook events. The related open WhatsApp reaction items are useful context, but they do not supersede this plugin-hook API request.

## What This Changes

Request to add WhatsApp `reaction_received` and `reaction_removed` plugin hooks for inbound reaction add/remove events without creating agent runs or outbound activity.

## Best Possible Solution

Define a maintained reaction hook contract in the plugin SDK, emit it from the official WhatsApp listener with account-scoped privacy opt-in, and align no-agent/no-outbound behavior with https://github.com/openclaw/openclaw/issues/78963.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a new plugin API feature request. Source inspection gives high confidence that current main lacks the requested WhatsApp reaction hook surface.

## Solution Assessment

Unclear until maintainer API review: emitting from the official WhatsApp listener is the right owner boundary, but the public hook contract, privacy/config gate, and listen-only semantics need an explicit product decision.
