## Summary

Keep open: current main still has opt-in WhatsApp `message_received` hooks and outbound `react`, but no WhatsApp reaction listener or reaction hook contract. This is narrower than the broad reaction-query tracker and needs maintainer API/product selection rather than cleanup closure.

## What This Changes

Request to add WhatsApp reaction add/remove plugin hooks that observe Baileys reaction events without starting agent runs or sending outbound WhatsApp activity.

## Best Possible Solution

Design an additive reaction observation hook contract, implement it in the WhatsApp plugin inbound monitor with explicit privacy opt-in, tests, docs, and live add/remove/reconnect proof; keep historical reaction query/storage scope aligned with the broader tracker.

## Reproduction Assessment

Not applicable: this is a feature/API request, not a broken documented behavior. Source inspection gives high confidence that current main lacks both a WhatsApp reaction listener and a public reaction hook surface.

## Solution Assessment

Unclear until maintainers choose the public API contract. A dedicated additive observation hook looks more maintainable than forwarding reactions as synthetic `message_received` or agent messages, but names, payload, opt-in, and relationship to broader reaction querying need owner judgment.
