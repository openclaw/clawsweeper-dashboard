## Summary

Keep open: current main still has opt-in WhatsApp `message_received` hooks and outbound `react` support, but no WhatsApp reaction listener or public reaction hook contract; adding one is a new privacy-sensitive plugin/API surface that needs maintainer direction.

## What This Changes

Request to add WhatsApp `reaction_received` and optional `reaction_removed` plugin hooks for inbound reaction add/remove events without agent runs or outbound WhatsApp activity.

## Best Possible Solution

Design an additive reaction event contract, then implement it in the WhatsApp plugin's inbound monitor with privacy opt-in, tests, docs, and live add/remove/reconnect proof.
