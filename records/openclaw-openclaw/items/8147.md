## Summary

Keep this issue open. Current main and the latest release still use plain Enter-to-send with no persisted send-shortcut setting or documented WebChat config option, and the open linked implementation PR should be reviewed before this tracker is closed.

## What This Changes

Request a configurable WebChat/Control UI send shortcut so Enter can be reserved for IME or multiline composition while Cmd/Ctrl+Enter sends.

## Best Possible Solution

Keep this issue as the canonical tracker until https://github.com/openclaw/openclaw/pull/73390 is reviewed, landed or closed, and then close this issue only after the chosen send-shortcut behavior ships.

## Reproduction Assessment

Yes. Source inspection on current main shows the composer still sends on plain Enter without Shift and has no send-shortcut setting, which reproduces the requested behavior gap even though this is a feature request rather than a broken existing contract.

## Solution Assessment

Unclear. A configurable modifier-enter mode is a plausible solution, but maintainers still need to decide whether the preference belongs in browser-local Control UI storage, gateway/openclaw.json config, or both before this is considered resolved.
