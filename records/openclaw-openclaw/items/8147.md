## Summary

Keep this issue open: current main and release v2026.6.5 still hardcode plain Enter-to-send, while the linked implementation remains open, unmerged, and conflicting.

## What This Changes

Add a persisted WebChat preference that lets users choose between plain Enter and Cmd/Ctrl+Enter for sending messages.

## Best Possible Solution

Preserve Enter-to-send as the upgrade-safe default, add a localized persisted Control UI preference for Cmd/Ctrl+Enter mode, and cover plain Enter, Shift+Enter, IME composition, and platform modifiers in focused tests.

## Reproduction Assessment

Yes, at source level: current main directly shows plain Enter-to-send with suppression only during active IME composition. This is a feature request rather than a failure of an existing configurable-shortcut contract.

## Solution Assessment

No, not as a new openclaw.json key; the narrower maintainable solution is a browser-local Control UI preference that preserves the current default and follows current i18n and settings conventions.
