## Summary

Keep this issue open: current main and release v2026.6.5 still use plain Enter-to-send, and the explicit closing implementation at https://github.com/openclaw/openclaw/pull/73390 remains open and unmerged.

## What This Changes

Requests a configurable WebChat send shortcut so users can choose Cmd/Ctrl+Enter instead of plain Enter.

## Best Possible Solution

Add a backward-compatible persisted Control UI setting for Enter versus Cmd/Ctrl+Enter sending, preserve the existing IME guards, update localized composer hinting, and test both modes across Enter, Shift+Enter, composition, and Cmd/Ctrl+Enter.

## Reproduction Assessment

Yes, at source level: dispatching plain Enter in the current composer invokes send unless Shift is held or IME composition is active, and a focused test codifies that behavior. No live browser reproduction was run during this read-only review.

## Solution Assessment

No to adding this to `openclaw.json`; a persisted browser-local Control UI setting is the narrower ownership boundary for a WebChat keyboard preference, subject to maintainer approval of the supported modes and default.
