## Summary

Keep open. Current main still resumes restart-aborted subagent sessions based on `abortedLastRun` without an age cutoff; the newer rapid re-wedge tombstone work is complementary and does not address the reported 24-hour or 16-day stale replay cases.

## What This Changes

Bound gateway restart recovery so stale interrupted subagent tasks become terminal instead of automatically replaying old prompts.

## Best Possible Solution

Add a configurable age gate in the restart-aborted subagent recovery path, terminalize stale matching runs/tasks as lost, and keep fresh restart recovery plus rapid re-wedge tombstoning intact.

## Reproduction Assessment

Yes. Persist an old unended subagent run with a child session entry marked `abortedLastRun: true`, restart the registry or gateway, and current main attempts recovery; the existing three-hour persistence test asserts that behavior.

## Solution Assessment

Yes. An age check at the orphan-recovery decision point, paired with task/session terminalization and config/docs/tests, is the narrowest maintainable fix; filtering only task-registry restore would leave the synthetic resume path intact.
