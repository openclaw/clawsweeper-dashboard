## Summary

Keep open. Current main still resumes restart-aborted subagent sessions from `abortedLastRun: true` without a stale-age cutoff; the existing guard only tombstones rapid repeated recoveries, and the open stale-task PR targets https://github.com/openclaw/openclaw/issues/79226 rather than this old-prompt replay path.

## What This Changes

Bound gateway restart subagent recovery so stale interrupted tasks are marked lost instead of replaying old prompts.

## Best Possible Solution

Add a stale-age policy before synthetic subagent orphan recovery, terminalize too-old restart-aborted work as lost, and align config, docs, and regression tests with the chosen guarantee.

## Reproduction Assessment

Yes, at source level. Seed an active restored subagent run older than the intended cutoff with a child session entry marked `abortedLastRun: true`; current code and tests keep it eligible for restart recovery instead of marking it lost.

## Solution Assessment

Unclear until maintainers choose the recovery policy. The narrow technical fix point is an age gate before `resumeOrphanedSession`, but the cutoff, config surface, and side-effectful-tool behavior need owner judgment.
