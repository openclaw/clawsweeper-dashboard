## Summary

Keep open. Current main still has no age bound for first-time restart-aborted subagent recovery, and existing tests still assert that stale aborted runs remain recoverable.

## What This Changes

The issue requests an age-bounded gateway recovery policy so old interrupted CLI subagent tasks are marked lost instead of automatically resumed.

## Best Possible Solution

Add a configurable age gate in the restart-aborted subagent recovery path so fresh interrupted work can recover, while stale restored runs and matching task records become lost without sending a synthetic resume.

## Reproduction Assessment

Yes. Persist an unended subagent run older than the intended stale window with a child session entry marked `abortedLastRun: true`, restart the registry/gateway, and current main will attempt recovery; the existing three-hour persistence test asserts this behavior.

## Solution Assessment

Yes. An age check at the orphan-recovery decision point, paired with task/session terminalization and config/docs/tests, is the narrowest maintainable fix; filtering task restore alone would leave the synthetic resume path intact.
