## Summary

Keep open. Current main still auto-recovers restart-aborted subagent sessions from `abortedLastRun: true` without a stale-age cutoff; the rapid re-wedge fix covers repeated near-term recovery loops, not old prompt replay after long gaps.

## What This Changes

The issue asks OpenClaw to bound restart recovery for interrupted subagent tasks by age and mark stale recoveries lost instead of re-dispatching their original prompts.

## Best Possible Solution

Add a stale-age cutoff at subagent orphan-recovery and task/session reconciliation, mark skipped ancient recovery work `lost`, and align config, docs, and regression tests with the chosen cutoff.

## Reproduction Assessment

Yes, at source level. Seed a persisted active subagent run older than the intended cutoff with a child session entry marked `abortedLastRun: true`, then restart/sweep recovery; current code and tests expect recovery instead of marking it lost.

## Solution Assessment

Yes, the narrow maintainable fix is an age gate before synthetic resume plus task-state reconciliation and focused tests. The exact cutoff/default and config surface still need maintainer selection.
