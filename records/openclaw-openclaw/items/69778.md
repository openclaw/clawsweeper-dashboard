## Summary

Keep open. Current main still lets restart-aborted subagent work replay solely from `abortedLastRun` plus the rapid re-wedge tombstone state, with no recovery-age cutoff or `agents.defaults.taskRecovery.maxAgeMinutes`; the closed wedge-loop fix is complementary but does not stop ancient prompt replay.

## What This Changes

Request that gateway restart recovery stop auto-resuming stale interrupted CLI/subagent tasks after an age threshold, ideally configurable, and mark them `lost` instead.

## Best Possible Solution

Add a stale-recovery max-age gate around subagent orphan recovery and task/session reconciliation, default it conservatively, document it, and leave side-effectful-tool policy as a separate follow-up.

## Reproduction Assessment

Yes, at source level. Current tests seed a three-hour-old restored run with `abortedLastRun: true` and expect recovery to call the gateway, matching the issue reports of 24-hour and 16-day-old replay.

## Solution Assessment

Yes. The narrow maintainable fix is an age check at the orphan-recovery decision point plus task-state reconciliation, config/docs, and regression coverage; the existing rapid re-wedge tombstone does not solve old prompt replay.
