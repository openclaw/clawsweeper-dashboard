## Summary

Keep open. Current main still auto-recovers restart-aborted subagent sessions based on `abortedLastRun` plus rapid re-wedge state, with no age cutoff or `agents.defaults.taskRecovery.maxAgeMinutes` config, and regression coverage still expects a three-hour stale aborted run to recover.

## What This Changes

Request an age-bound recovery guard so stale restart-aborted CLI/subagent tasks become lost instead of being auto-resumed after a gateway restart.

## Best Possible Solution

Add a configurable stale-recovery cutoff in the subagent orphan recovery and task/session reconciliation paths, mark skipped stale work lost without replaying the prompt, and document the behavior with regression coverage.

## Reproduction Assessment

Yes. Persist an unended subagent run older than the intended cutoff with a child session entry marked `abortedLastRun: true`, restart the registry/gateway, and current main will attempt recovery; the existing three-hour persistence test asserts this path.

## Solution Assessment

Yes. An age check at the orphan-recovery decision point, paired with terminalizing the matching run/task state and config/docs/tests, is the narrowest maintainable fix; filtering task restore alone would leave synthetic resume behavior intact.
