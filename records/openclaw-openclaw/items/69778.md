## Summary

Keep open. Current main fixed the related rapid re-wedge loop from #74864, but it still has no age bound for a first restart-aborted subagent recovery, so old interrupted prompts can still be synthetically resumed.

## What This Changes

Request an age-bounded gateway restart recovery policy so stale interrupted CLI/subagent tasks are marked lost instead of being auto-resumed long after interruption.

## Best Possible Solution

Add a bounded restart-aborted recovery policy in the core subagent/task recovery path: keep fresh interrupted work recoverable, mark stale restart-aborted runs and task records lost without synthetic redispatch, expose and document a default max-age setting, and preserve the #74864 rapid re-wedge tombstone behavior.

## Reproduction Assessment

Yes. Persist a subagent run older than the stale window with a child session marked `abortedLastRun: true`, restart the registry/gateway, and current main will schedule recovery; the existing persistence test already asserts this for a three-hour-old run.

## Solution Assessment

Yes. An age gate at the orphan-recovery decision point, with task/run terminalization plus config/docs/tests, is the narrowest maintainable fix; task restore filtering alone would leave the synthetic redispatch path intact.
