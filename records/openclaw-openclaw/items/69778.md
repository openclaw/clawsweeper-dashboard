## Summary

Keep open. Current main and the latest release still recover restart-aborted subagent sessions from `abortedLastRun: true` without a stale-age cutoff, and the discussion includes a second live report where a 16-day-old subagent sent a stale WhatsApp message after restart; the closed wedged-recovery fix bounds rapid re-wedges, not old-prompt replay.

## What This Changes

The issue requests an age-bounded gateway restart recovery policy so old interrupted CLI/subagent tasks are marked lost instead of being automatically re-dispatched.

## Best Possible Solution

Define a stale-age policy for restart-aborted subagent recovery, terminalize too-old work as lost before synthetic resume, and align config, docs, and regression tests with that guarantee.

## Reproduction Assessment

Yes, at source level. Seed an active restored subagent run older than the intended cutoff with a child session entry marked `abortedLastRun: true`; current code and tests keep it eligible for restart recovery instead of marking it lost.

## Solution Assessment

Unclear until maintainers choose the recovery policy. The narrow technical fix point is an age gate before `resumeOrphanedSession`, but the cutoff, config surface, and side-effectful-tool behavior need owner judgment.
