## Summary

Keep open. Current main still treats `abortedLastRun: true` as enough to auto-recover a restart-aborted subagent run without a stale-age cutoff; the merged https://github.com/openclaw/openclaw/issues/74864 fix bounds rapid re-wedge loops, but does not address old prompt replay after long gaps.

## What This Changes

The issue requests an age bound for gateway restart recovery so stale interrupted CLI/subagent tasks are marked `lost` instead of being automatically resumed.

## Best Possible Solution

Add a stale-age cutoff at the subagent orphan-recovery decision point and task/session reconciliation path, mark skipped ancient recovery work `lost`, and align config, docs, and regression tests with the chosen cutoff.

## Reproduction Assessment

Yes, at source level. Seed a persisted active subagent run older than the intended cutoff with a child session entry marked `abortedLastRun: true`, then restart/sweep recovery; current code and tests expect recovery instead of marking it lost.

## Solution Assessment

Yes, the narrow maintainable fix is an age gate before synthetic resume plus task-state reconciliation and focused tests. Side-effectful external-channel recovery should remain a separate follow-up unless maintainers choose to broaden this issue.
