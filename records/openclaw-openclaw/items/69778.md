## Summary

Keep open. Current main still auto-recovers restart-aborted subagent sessions from `abortedLastRun` without any stale-age cutoff, and the closed wedge-loop fix only bounds rapid repeated recovery, not old prompt replay.

## What This Changes

Bound gateway restart recovery so stale interrupted CLI/subagent tasks are marked lost or skipped by age instead of replaying old prompts automatically.

## Best Possible Solution

Add a stale-recovery cutoff in subagent orphan recovery and task/session reconciliation, mark skipped stale work lost without prompt replay, and update config docs plus regression coverage while preserving the rapid re-wedge tombstone behavior.

## Reproduction Assessment

Yes, at source level. Seed an active subagent run older than the intended cutoff with a child session entry marked `abortedLastRun: true`, then restart or schedule orphan recovery; current code and tests show that stale run is recovered instead of terminalized.

## Solution Assessment

Yes. The narrow maintainable fix is an age check at the orphan-recovery decision point plus task-state reconciliation, config/docs alignment, and focused tests; the closed wedge-loop fix is complementary but not sufficient.
