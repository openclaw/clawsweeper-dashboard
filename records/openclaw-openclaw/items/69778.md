## Summary

Keep open. Current main still treats restored subagent runs with `abortedLastRun: true` as recoverable without an age cutoff; #74864 only added a rapid repeated-recovery tombstone, and current tests still assert that a three-hour stale restart-aborted run remains recoverable.

## What This Changes

Request an age-bounded gateway restart recovery policy so stale restart-aborted subagent tasks are marked `lost` instead of auto-resumed.

## Best Possible Solution

Add a configurable stale-age cutoff at the subagent orphan-recovery decision point, terminalize matching stale runs/tasks as `lost`, and document the recovery window while preserving the existing #74864 rapid re-wedge tombstone.

## Reproduction Assessment

Yes. Seed a restored unended subagent run older than the intended recovery window with a child session entry marked `abortedLastRun: true`; the current persistence test uses a three-hour-old run and expects recovery work to be scheduled.

## Solution Assessment

Yes. An age check at orphan-recovery eligibility, paired with task/run/session reconciliation and config/docs/tests, is narrower and safer than filtering task restore alone because the synthetic resume path is what replays old work.
