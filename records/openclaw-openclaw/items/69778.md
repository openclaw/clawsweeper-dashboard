## Summary

Keep open. Current main still resumes stale restart-aborted subagent work solely from `abortedLastRun` plus the rapid-retry tombstone state, with no recovery-age cutoff or `agents.defaults.taskRecovery.maxAgeMinutes`; the closed related wedge-loop issue only fixed repeated rapid resurrection, not ancient prompt replay.

## What This Changes

Add an age-bound gateway restart recovery policy so stale interrupted CLI/subagent tasks are marked lost instead of replaying old prompts.

## Best Possible Solution

Add a configurable stale-recovery cutoff in the orphan recovery and task/session reconciliation paths, mark skipped stale work lost without replaying the prompt, and update docs plus regression coverage.

## Reproduction Assessment

Yes at source level. Seed an active subagent run older than the intended cutoff with a child session entry marked `abortedLastRun: true`, then restart/schedule orphan recovery; current tests already assert a three-hour stale aborted run is recovered.

## Solution Assessment

Yes. The narrow fix is an age check at the orphan-recovery decision point plus task-state reconciliation and config/docs/tests; the wedge-loop tombstone from https://github.com/openclaw/openclaw/issues/74864 is complementary but does not solve old prompt replay.
