## Summary

This PR should stay open for normal maintainer handling: current main still does not implement the requested OpenRouter native slash shorthand, but the branch has blocking provider-routing and allowlist-scope defects plus no real behavior proof.

## What This Changes

The PR changes model-selection parsing, provider inference, allowlist construction, and tests so selected minimax/qwen slash-style refs can resolve through configured OpenRouter allowlist entries.

## Best Possible Solution

Implement resolver-level matching of raw OpenRouter native IDs only against a unique exact configured OpenRouter allowlist or alias entry, keep provider-scoped allowlist keys, and update docs if this shorthand is accepted.

## Reproduction Assessment

Yes, for the PR defects by source inspection: a configured openrouter/qwen/... allowlist plus a real qwen provider would be widened or misrouted by the branch. I did not run tests because this was a read-only review.

## Solution Assessment

No. The branch uses parse-time vendor rewrites and bare allowlist keys; resolver-level exact configured matching with canonical keys is the narrower maintainable path.
