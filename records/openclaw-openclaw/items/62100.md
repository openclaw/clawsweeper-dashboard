## Summary

Keep this PR open for maintainer handling: the requested OpenRouter shorthand may be useful, but the branch currently weakens provider scoping, misroutes qualified refs, and lacks after-fix real behavior proof.

## What This Changes

The PR changes model-selection parsing, allowlist construction, provider inference, and tests so selected `minimax/...` and `qwen/...` slash-style refs can resolve through configured OpenRouter allowlist entries.

## Best Possible Solution

Add resolver-level exact matching only for accepted configured OpenRouter nested-id entries, keep allowlist keys provider-scoped, and document the shorthand if maintainers choose to support it.

## Reproduction Assessment

Yes, for the PR defects by source inspection: the branch rewrites qualified `qwen/...` and `minimax/...` refs to OpenRouter and adds nested model ids as bare allowed keys. I did not run tests because this was a read-only review.

## Solution Assessment

No. Parse-time vendor rewrites and bare allowlist keys are not the narrowest maintainable solution; resolver-level exact configured matching with canonical provider-scoped keys is safer.
