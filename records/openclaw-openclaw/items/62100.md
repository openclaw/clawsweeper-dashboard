The PR modifies agent model-selection parsing/resolution and tests to allow qwen/minimax-style OpenRouter native slash IDs to resolve from fully qualified OpenRouter allowlist entries.

## Best Possible Solution

If maintainers want shorthand support, implement exact configured OpenRouter nested-ID matching in the resolver while preserving explicit provider prefixes and provider-scoped allowlist keys; otherwise keep the documented explicit-prefix contract.

## Reproduction Assessment

Yes, by source inspection: current main parses slash-containing refs as provider-qualified and only infers configured providers for non-slash selections, while the PR diff shows the proposed shorthand path and allowlist widening. I did not run tests in this read-only checkout.

## Solution Assessment
