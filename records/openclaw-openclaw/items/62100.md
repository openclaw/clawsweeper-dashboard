## Summary

Keep open. The PR targets a real unresolved OpenRouter slash-ID gap, but the proposed patch widens provider-scoped allowlist behavior and can misroute explicit provider refs, so it is not merge-ready.

## What This Changes

The PR changes model selection parsing, allowlist resolution, and regression tests so selected qwen/minimax slash-style model IDs can resolve through allowlisted OpenRouter refs.

## Best Possible Solution

Implement resolver-level matching from raw vendor/model to configured openrouter/vendor/model only when that OpenRouter entry is explicit and unique, while preserving explicit provider refs and canonical allowedKeys.

## Reproduction Assessment

Yes. Configure an allowlist with openrouter/qwen/qwen3.6-plus:free plus a real qwen provider/model, then resolve qwen/qwen3.6-plus:free through the allowed-model path and verify it neither bypasses provider scoping nor gets rewritten away from the qwen provider.

## Solution Assessment

No. The PR targets the right gap, but parse-time vendor rewrites and bare allowed keys are broader than necessary; resolver-level matching against configured OpenRouter entries is the narrower maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Full review comments:

- **[P1] Preserve explicitly qualified provider refs:** `src/agents/model-selection.ts:149-155`
  - body: This branch rewrites qwen/<model> and minimax/<model> to provider openrouter whenever defaultProvider is openrouter. That breaks the existing provider/model contract for real qwen or minimax providers; provider-qualified selections should keep their stated provider unless the resolver proves an explicit OpenRouter allowlist match.
  - confidence: 0.89
- **[P1] Keep OpenRouter allowlist keys provider-scoped:** `src/agents/model-selection.ts:626`
  - body: Adding parsed.model to allowedKeys means an allowlist entry for openrouter/qwen/qwen3.6-plus:free can also admit qwen/qwen3.6-plus:free as a separate provider key. The gateway checks provider/model keys, so this widens the configured provider scope.
  - confidence: 0.93
- **[P2] Resolve all configured OpenRouter slash IDs:** `src/agents/model-selection.ts:149-155`
  - body: The new parseModelRef guard only recognizes minimax/ and qwen/, so the same OpenRouter-native pattern still fails for deepseek/, meta-llama/, mistralai/, google/, and other configured vendors. Resolve against configured openrouter/<raw> entries instead of maintaining a partial vendor list.
  - confidence: 0.84
