## Summary

Keep open: the PR targets a plausible model-resolution gap, but this branch weakens provider-scoped allowlists, can reroute explicit provider-qualified refs, and lacks after-fix real behavior proof.

## What This Changes

The PR changes model-selection parsing, allowlist building, and tests so selected qwen/minimax slash-style refs can resolve through configured OpenRouter allowlist entries.

## Best Possible Solution

Resolve raw vendor/model OpenRouter shorthand only through a unique exact configured OpenRouter allowlist or alias match, keep canonical provider-scoped allowlist keys, and document any relaxed shorthand behavior.

## Reproduction Assessment

Yes. Source inspection shows the PR would add bare OpenRouter nested model keys and rewrite selected provider-qualified refs; a focused regression would configure both openrouter/qwen/... and a real qwen provider and assert provider scoping is preserved.

## Solution Assessment

No. The PR aims at useful shorthand behavior, but parse-time vendor rewrites and bare allowed keys are broader than necessary; resolver-level matching against configured OpenRouter entries is safer.
