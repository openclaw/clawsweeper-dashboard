## Summary

Keep open: current main has related generic configured-model inference, but the resolver path still treats slash-containing raw refs as provider-qualified, while the submitted branch is conflicting and would weaken provider routing and allowlist boundaries if merged as-is.

## What This Changes

The PR changes model-selection parsing, allowlist construction, and tests so selected OpenRouter nested model refs such as `minimax/...` and `qwen/...` can resolve from fully qualified OpenRouter allowlist entries.

## Best Possible Solution

If maintainers want this shorthand, implement resolver-level exact matching for configured OpenRouter nested IDs in `src/agents/model-selection-shared.ts`, preserve provider-qualified refs and provider-scoped allowlist keys, then add tests and redacted real OpenRouter proof.

## Reproduction Assessment

Yes for the PR blockers: source inspection shows the submitted patch rewrites selected slash refs and adds bare allowlist keys, while current main keeps slash refs provider-qualified in `parseModelRef`. I did not run tests because this review is read-only.

## Solution Assessment

No. The submitted approach is not the best fix because parse-time vendor rewrites and bare allowlist keys create routing and policy drift; exact configured matching in the resolver would be safer if maintainers want the shorthand.
