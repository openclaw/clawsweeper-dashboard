## Summary

Keep open: the branch still has blocking correctness and security issues around provider-scoped allowlists and explicit provider routing, and it lacks after-fix real behavior proof.

## What This Changes

The PR changes model-selection parsing, provider inference, allowlist construction, and tests so selected minimax/qwen slash-style refs can resolve through configured OpenRouter allowlist entries.

## Best Possible Solution

Implement resolver-level exact matching only against unique configured OpenRouter allowlist or alias entries, keep allowlist keys provider-scoped, and document the shorthand if maintainers accept it.

## Reproduction Assessment

Yes, for the PR defects by source inspection: an openrouter/qwen/... allowlist plus a real qwen provider would be widened or misrouted by the branch. I did not run tests because this was a read-only review.

## Solution Assessment

No. Parse-time vendor rewrites and bare allowlist keys are not the narrowest maintainable solution; resolver-level exact configured matching with canonical keys is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Preserve provider-qualified qwen/minimax refs:** `src/agents/model-selection.ts:149-155`
  - body: This branch rewrites explicit refs like qwen/<model> and minimax/<model> to provider openrouter whenever defaultProvider is openrouter. If a user or plugin defines a real qwen or minimax provider, selecting that qualified ref is sent to OpenRouter instead of the stated provider.
  - confidence: 0.89
- **[P1] Keep OpenRouter allowlist keys provider-scoped:** `src/agents/model-selection.ts:626`
  - body: Adding parsed.model to allowedKeys widens an allowlist entry such as openrouter/qwen/qwen3.6-plus:free so it can also admit the separate provider key qwen/qwen3.6-plus:free. That bypasses the provider scope enforced by the configured allowlist.
  - confidence: 0.93
- **[P2] Resolve configured OpenRouter slash IDs generically:** `src/agents/model-selection.ts:149-155`
  - body: The special case only recognizes minimax/ and qwen/, so configured OpenRouter-native refs such as deepseek/..., meta-llama/..., mistralai/..., and google/... still miss this shorthand path. Match against configured OpenRouter entries instead of maintaining a partial vendor list.
  - confidence: 0.84
