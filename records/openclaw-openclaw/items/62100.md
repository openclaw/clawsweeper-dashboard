## Summary

Keep this PR open, but it is not merge-ready: the branch still introduces provider misrouting and allowlist-scope regressions, and it lacks after-fix real behavior proof for the external contributor gate.

## What This Changes

The PR changes model-selection parsing, configured-provider inference, allowlist key construction, and tests so selected `minimax/...` and `qwen/...` refs can resolve through allowlisted OpenRouter entries.

## Best Possible Solution

Implement resolver-level exact matching only for configured OpenRouter nested model IDs, keep provider-scoped allowlist keys, and document any accepted shorthand behavior.

## Reproduction Assessment

Yes, for the PR defects by source inspection: the branch rewrites explicit `qwen/...` and `minimax/...` refs to OpenRouter and adds nested OpenRouter model IDs as bare allowed keys. I did not run tests because this was a read-only review.

## Solution Assessment

No. Parse-time vendor rewrites and bare allowlist keys are not the narrowest maintainable solution; resolver-level exact configured matching with canonical provider-scoped keys is safer.
