The branch changes model selection to infer OpenRouter for qwen/minimax slash-style model IDs, widens allowed keys for nested OpenRouter models, and adds regression tests.

## Best Possible Solution

If maintainers want this shorthand, implement resolver-level exact matching for configured OpenRouter nested IDs while preserving explicit provider prefixes and provider-scoped allowlist keys, then align docs and tests.

Yes, by source inspection: current main parses slash-containing refs as provider-qualified and only infers configured providers for non-slash selections. The PR diff also shows the proposed shorthand path and allowlist widening.

## Solution Assessment

No. Parse-time vendor rewrites plus bare allowlist keys are not the narrowest maintainable solution; exact configured matching in the resolver would preserve provider intent and allowlist scope.
