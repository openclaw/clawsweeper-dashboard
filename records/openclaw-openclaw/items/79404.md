Full review comments:

- **[P2] Keep the install smoke root lane writable:** `scripts/docker/install-sh-smoke/Dockerfile:35`
  - body: This image is the root install smoke lane, and its runner still calls `npm install -g` for previous/baseline/update paths. Switching the container to `USER openclaw` leaves those global installs targeting the usual root-owned npm prefix and also stops testing the root install path that the separate nonroot lane does not cover.
  - confidence: 0.86
- **[P2] Keep the Deployment namespace selectable:** `scripts/k8s/manifests/deployment.yaml:5`
  - body: `scripts/k8s/deploy.sh` supports `OPENCLAW_NAMESPACE` and applies this kustomization with `kubectl apply -k ... -n "$NS"`. Adding `metadata.namespace: openclaw` makes custom-namespace deploys target the wrong namespace or fail with a namespace mismatch, while the sibling manifests remain namespace-neutral.
  - confidence: 0.91
- **[P2] Scope the Trivy suppression to the accepted resource:** `.trivyignore:4`
  - body: The comment accepts `KSV-0125` only for `scripts/k8s/manifests/deployment.yaml`, but this plain `.trivyignore` entry suppresses that rule for every repository scan target. If another manifest introduces the same issue later, the certification gate will silently hide it.
  - confidence: 0.9
- **[P2] Narrow the Gitleaks test allowlist:** `.gitleaks.toml:27-31`
  - body: This repo-wide allowlist disables several high-value secret rules for every `*.test.*` file, not only the known fake-token fixtures. A real private key or PAT committed in a new test would be accepted by the certification scan, so this needs exact fixture paths or narrower testdata patterns.
  - confidence: 0.82
