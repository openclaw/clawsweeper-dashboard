Full review comments:

- **[P2] Keep the install smoke root lane writable:** `scripts/docker/install-sh-smoke/Dockerfile:35`
  - body: Switching this image to USER openclaw changes the existing root/update smoke lanes, but scripts/test-install-sh-docker.sh still runs this image for those cases and run.sh calls npm install -g before any user-writable npm prefix is configured. Keep this image root for the root lane, or explicitly add a writable prefix and update the lane semantics so the smoke still exercises the intended install path.
  - confidence: 0.92

  - body: Adding metadata.namespace: openclaw conflicts with OPENCLAW_NAMESPACE because deploy.sh still applies the kustomize directory with -n "$NS". Custom-namespace installs can fail or split resources, so leave the Deployment namespace-neutral or generate namespace selection consistently across every manifest.
  - confidence: 0.94
- **[P2] Scope the Trivy suppression to the accepted manifest:** `.trivyignore:4`
  - body: The plain .trivyignore entry suppresses KSV-0125 for every scanned target, while the comment accepts it only for the OpenClaw Deployment manifest and the expiry is not machine-enforced. Use an inline ignore or .trivyignore.yaml with paths and expired_at so future occurrences are still reported.
  - confidence: 0.95
- **[P2] Narrow the Gitleaks test allowlist:** `.gitleaks.toml:16-29`
  - body: This global allowlist combines github-pat and private-key with the catch-all test-file path, so a real PAT or private key committed in any test file would be ignored. Restrict these high-risk rules to exact fixture files or placeholder regexes known to be false positives.
  - confidence: 0.88
