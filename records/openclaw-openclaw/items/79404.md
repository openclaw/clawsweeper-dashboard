- **[P1] Keep the Deployment namespace configurable:** `scripts/k8s/manifests/deployment.yaml:5`
  - body: Adding `metadata.namespace: openclaw` conflicts with `scripts/k8s/deploy.sh`, which supports `OPENCLAW_NAMESPACE` and applies the kustomize tree with `kubectl -n "$NS"`. Custom-namespace installs can fail with a namespace mismatch or split resources; leave the manifest namespace-neutral and let the deploy script choose it.
  - confidence: 0.95
- **[P1] Narrow the test secret allowlist:** `.gitleaks.toml:17-29`
  - body: This global allowlist suppresses high-value rules such as `github-pat` and `private-key` for every `*.test.*` file. A real token accidentally committed in a test would be ignored by the new certification gate; scope this to named fixture paths or fake-token patterns instead.
  - confidence: 0.86
- **[P2] Scope the Trivy suppression to the accepted manifest:** `.trivyignore:4`
  - body: A plain `KSV-0125` entry in `.trivyignore` suppresses that rule for every scan target, while the comment accepts the risk only for `scripts/k8s/manifests/deployment.yaml`. Use an inline ignore or `.trivyignore.yaml` with `paths` and expiry so future occurrences are still reported.
  - confidence: 0.93
