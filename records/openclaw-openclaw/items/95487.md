- **[P1] Gate default-on filtering behind approved policy:** `packages/speech-core/src/tts.ts:1213-1214`
  - body: This unconditionally rewrites text for every shared TTS provider. Existing users who expect emoji names or symbols in speech would lose content on upgrade while the canonical TTS spoken-output issue remains open and no opt-in config/docs/default decision is included.
  - confidence: 0.92
- **[P2] Cover regional flags and keycap emoji:** `packages/speech-core/src/tts.ts:1187`
  - body: The regex starts from `Extended_Pictographic`, so regional-indicator flags and keycap sequences are not stripped. Common emoji such as country flags and `1️⃣` can still reach TTS while the PR claims broad emoji cleanup.
  - confidence: 0.9
- **[P2] Skip emoji-only synthesis after stripping:** `packages/speech-core/src/tts.ts:1214`
  - body: When stripping leaves an empty string, this falls back to the original text, so explicit or direct emoji-only TTS requests still synthesize the emoji names. Return a no-op or explicit error after the policy is settled instead of reintroducing the content.
  - confidence: 0.88
- **[P2] Return the effective spoken text for metadata:** `packages/speech-core/src/tts.ts:1213-1214`
  - body: The provider can synthesize `inputText`, but downstream payloads still record the caller's original `textForAudio`. Audio metadata and transcripts can disagree with what was actually sent to the provider unless the effective spoken text is surfaced back to callers.
  - confidence: 0.8
