- **[P1] Put filtering behind a maintainer-approved policy:** `packages/speech-core/src/tts.ts:1213-1214`
  - body: This unconditionally rewrites text for every shared speech synthesis target. Existing users who expect emoji pronunciation would lose content on upgrade while the canonical TTS spoken-output issue remains open and no config/docs contract or explicit maintainer default decision is included.
  - confidence: 0.9
- **[P2] Cover regional flags and keycap emoji:** `packages/speech-core/src/tts.ts:1187`
  - body: The regex only starts from `Extended_Pictographic`, so regional-indicator flags and keycap sequences are not stripped. Common emoji such as country flags and keycaps can still reach TTS while the PR claims broad cleanup.
  - confidence: 0.88
- **[P2] Do not synthesize emoji-only text after filtering:** `packages/speech-core/src/tts.ts:1214`
  - body: When stripping leaves an empty string, this falls back to the original text, so an emoji-only TTS request can still be synthesized as emoji names. Return a no-op or explicit error after the policy is settled instead of reintroducing the content.
  - confidence: 0.86
- **[P2] Keep spoken-text metadata aligned with provider input:** `packages/speech-core/src/tts.ts:1213-1214`
  - body: The provider can synthesize `inputText`, but downstream payloads still record the original `textForAudio` because the prepared text is not surfaced through `TtsResult`. Audio metadata and transcripts can disagree with what was actually sent to the provider.
  - confidence: 0.78
