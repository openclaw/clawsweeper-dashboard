  - body: This unconditionally rewrites text for every shared speech synthesis target. Existing users who expect emoji pronunciation would lose content on upgrade while the canonical spoken-output policy remains open and no config/docs contract or explicit maintainer default decision is included.
  - confidence: 0.9

  - body: The regex starts from `Extended_Pictographic`, so regional-indicator flags and keycap sequences are not stripped; a read-only probe leaves `🇺🇸`, `🇨🇳`, and `1️⃣` in provider input while the PR claims broad emoji cleanup.
  - confidence: 0.88
- **[P2] Do not fall back to original emoji-only text:** `packages/speech-core/src/tts.ts:1212`
  - body: When stripping leaves an empty string, `inputText` falls back to the original text, so an emoji-only TTS request can still be synthesized as emoji names. Return a no-op/error after the policy is settled instead of reintroducing the content.
  - confidence: 0.86
- **[P2] Keep spoken-text metadata aligned with provider input:** `packages/speech-core/src/tts.ts:1212`
  - body: The provider can synthesize the stripped `inputText`, but downstream payloads and the explicit `tts` tool still record the original request text because the prepared text is not returned through `TtsResult`.
