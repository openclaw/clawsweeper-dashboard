## Summary

This PR should remain open but is not merge-ready: the append mode restores suffix-only CardKit writes that previously caused live Feishu cards to show only the latest fragment, and no exact-head real-environment proof supports the claimed tenant-specific exception.

## What This Changes

The PR adds a `channels.feishu.streamingContentMode` setting that optionally sends only newly appended suffix text to CardKit streaming updates while preserving full-content mode as the default.

## Best Possible Solution

Keep full accumulated content as the single CardKit payload contract and address latency through measured coalescing, update thresholds, or throttling that reduces request frequency without changing message semantics.

## Reproduction Assessment

Yes for the introduced defect: source, history, and related live evidence show that suffix-only payloads can leave a card displaying only its latest fragment. The claimed append-mode latency improvement has not been reproduced on this exact head.

## Solution Assessment

No; this restores a known-bad suffix behavior behind a configuration option instead of optimizing the supported full-content update path.
