## Summary

Keep this issue open. Current main has useful token-efficiency mitigations and several UX sub-items are now documented or implemented, but the central requests remain unresolved: default workspace bootstrap context is still a fixed file set with coarse filters/caps rather than declarative per-file load conditions, and compaction summaries still re-enter model-visible context instead of lazy/title-only references. The later production-use comment adds concrete evidence that conditional workspace loading is still a current pain point.

## What This Changes

The issue requests multiple agent-facing token-efficiency and UX improvements, led by conditional workspace file loading, collapsible compaction summaries, subagent UX clarity, cron health handling, heartbeat intervals, plugin discovery clarity, config schema access, usage tracking, and graceful gateway update behavior.

## Best Possible Solution

Keep the issue open, then split it into focused maintainer-track follow-ups: default per-bootstrap-file load policy or an equivalent default context-selection mechanism, lazy or bounded compaction-summary reintroduction, and separate closure notes for already-shipped sub-items. Any plugin/context-engine alternative should be documented as an advanced path without hiding the default-token-cost gap.

## Reproduction Assessment

Not applicable as a feature request. The missing capability is still verifiable on current main by inspecting the fixed bootstrap-file assembly path, config schema, docs, and compaction reseed behavior; no runtime test is required to see that the requested per-file load conditions and lazy summaries are absent.

## Solution Assessment

No single proposed patch is under review, and this issue should not be closed as implemented or ClawHub-only. The maintainable path is to split the umbrella into narrower core/context-engine decisions, because current mitigations do not fully solve the default agent-token UX problem.
