## Summary

Keep open. Current main still normalizes hosted custom-scheme origins such as `tauri://localhost` to the opaque `null` origin before exact allowlist matching, and open PR #73511 is the active closing implementation candidate.

## What This Changes

Allow `gateway.controlUi.allowedOrigins` to exact-match custom desktop-app URL schemes such as `tauri://localhost` and `electron://app`.

## Best Possible Solution

Land PR #73511 or an equivalent narrow parser-level fix with regression coverage for allowlisted and unlisted custom-scheme origins while preserving literal `Origin: null`, malformed-origin, host-header fallback, and loopback behavior.

## Reproduction Assessment

Yes. The source-reproducible path is `new URL("tauri://localhost").origin === "null"`, followed by exact comparison against `parsedOrigin.origin` in `checkBrowserOrigin`.

## Solution Assessment

Yes. The narrowest maintainable solution is to preserve hosted custom-scheme origins during parser normalization and cover that helper directly, rather than broadening wildcard or Host-header fallback behavior.
