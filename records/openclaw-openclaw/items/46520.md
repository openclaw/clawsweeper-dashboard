## Summary

Keep this open: current main and v2026.6.6 still normalize custom-scheme browser origins through `URL.origin`, and the open linked fix at https://github.com/openclaw/openclaw/pull/73511 specifically closes this issue.

## What This Changes

Fix Gateway Control UI origin validation so exact `gateway.controlUi.allowedOrigins` entries for custom URL schemes such as `tauri://localhost` and `electron://...` are accepted without falling back to `*`.

## Best Possible Solution

Land one consolidated gateway origin-normalization fix that preserves exact `scheme://host` matching for explicitly allowlisted non-special schemes while keeping fail-closed behavior for unsafe origins.

## Reproduction Assessment

Yes. Source inspection plus a Node URL probe show current main normalizes `tauri://localhost` to the opaque `"null"` origin before exact allowlist comparison.

## Solution Assessment

Yes. The narrow maintainable fix belongs in `src/gateway/origin-check.ts` with regression coverage in `src/gateway/origin-check.test.ts`, and the open linked PR already targets that layer.
