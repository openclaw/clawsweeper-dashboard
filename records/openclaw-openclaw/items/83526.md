## Summary

Keep open: current main still has a source-visible path where shared status rendering can promote aggregate transcript usage into the Context numerator when freshness is legacy or unknown, so the Web UI fix and prior stale-false guard do not fully cover this TUI/status-message report.

## What This Changes

The issue asks for TUI and `/status` Context percentage to stop dividing cumulative session/transcript usage by the model context window.

## Best Possible Solution

Make shared status-message render Context from a fresh latest prompt snapshot only, while keeping cumulative usage and cache counters on separate usage/cache surfaces.

## Reproduction Assessment

Yes, at source level: a legacy/unknown-freshness session with `includeTranscriptUsage` and multiple transcript usage lines can pass aggregate cache/input totals into `Context:`. I did not run a live Claude CLI/TUI session in this read-only review.

## Solution Assessment

Yes: the narrow maintainable fix is to require latest current prompt/context usage for `Context:` and stop using aggregate transcript/session totals for the context-window percentage.
