## Summary

Keep open: current main still leaves a plausible source path where the shared status formatter can promote transcript-derived aggregate usage into the Context numerator when freshness is legacy or unknown, and the related Web UI cleanup does not fully cover this TUI/status-message path.

## What This Changes

The issue asks for the TUI and channel status Context line to show current request prompt/context usage instead of cumulative session/cache usage that can exceed 100%.

## Best Possible Solution

Make the shared status-message runtime use only a fresh current prompt/context snapshot for the Context numerator, while keeping cumulative usage or cache information on separate usage/cache surfaces.

## Reproduction Assessment

No live high-confidence reproduction was run. Source inspection gives a credible reproduction path through `includeTranscriptUsage`, unknown `totalTokensFresh`, and aggregate transcript usage being promoted into the Context numerator.

## Solution Assessment

Yes: the maintainable fix is to require a fresh current prompt/context snapshot for Context and avoid using cumulative transcript/session totals for context-window percentages.
