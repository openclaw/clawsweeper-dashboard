The issue requests that TUI and channel `/status` Context percentages use the latest prompt/context usage rather than cumulative transcript cache totals.

## Best Possible Solution

Land one focused fix that renders Context from a fresh latest prompt/current-context snapshot while keeping cumulative cache/session usage on separate usage surfaces, then close this issue.
