## Summary

Keep open: current main and the reported v2026.5.12 release still have a source-reachable path where aggregate transcript cache usage can be promoted into the shared status Context numerator when session freshness is legacy or unknown.

## What This Changes

The issue requests that TUI and `/status` Context usage show the latest prompt/context size rather than cumulative session cache usage.

## Best Possible Solution

Make the shared status-message runtime use only a fresh latest prompt or last-call context snapshot for the Context numerator, while keeping cumulative cache/session usage on separate cache or usage surfaces.

## Reproduction Assessment

Yes, at source level: a status fixture with legacy/undefined `totalTokensFresh` and a recent transcript tail containing multiple usage lines with large cumulative `cacheRead` reaches the aggregate transcript fallback and can render an over-window Context percentage. I did not run a live Claude CLI/TUI session in this read-only review.

## Solution Assessment

Yes: the maintainable fix is to separate latest prompt/context accounting from aggregate transcript/session accounting in the shared status formatter. A display-only clamp would hide the symptom without fixing the stale source of the numerator.
