## Summary

Keep open: current main still has the shared status formatter path that can promote aggregate transcript/cache usage into the Context numerator, and two open PRs already explicitly target this issue, so the right cleanup is to review and land one fix rather than close the issue early.

## What This Changes

The issue reports that TUI and `/status` Context percent uses cumulative transcript/cache usage as the numerator instead of the latest prompt/context usage.

## Best Possible Solution

Land one narrow fix that prevents aggregate transcript/session totals from feeding the Context numerator while preserving separate Tokens and Cache reporting, then close this issue with regression coverage.

## Reproduction Assessment

Yes, from source inspection: a legacy or unknown-freshness session whose transcript has repeated cache-read usage and no fresh per-turn total can flow through the aggregate reader and be promoted into the Context numerator. I did not run a live long Claude CLI/TUI session in this review.

## Solution Assessment

Yes: the maintainable fix belongs in the shared status-message/transcript usage path, using a fresh latest-turn prompt snapshot or an explicit freshness gate instead of capping or hiding the displayed percentage.
