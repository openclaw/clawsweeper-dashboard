## Summary

Keep open: current `main` still lacks per-agent thinking metadata in the Agents panel path, and the linked open fix PR is the active implementation path for this issue.

## What This Changes

Control UI should show each agent's configured effective `thinkingDefault` when that agent uses a model different from the global default.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81795, or an equivalent focused fix, so the gateway supplies resolved per-agent thinking metadata and Control UI uses it as the source of truth.

## Reproduction Assessment

Yes, with source-level confidence. Current `main` does not expose `thinkingDefault` on `agents.list` rows, and the UI fallback paths can still fall back to global defaults when the active agent/session model differs.

## Solution Assessment

Yes, the linked PR direction is the narrow maintainable fix: expose resolved per-agent thinking metadata from the gateway and prevent global-default fallback when the active model differs.
