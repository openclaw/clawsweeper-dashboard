## Summary

Current main already sends row-scoped thinking metadata to the Control UI, so a session row for an agent on a non-default model uses that row's resolved model and agent id instead of blindly inheriting global defaults. This is current-main-only and is not in the latest v2026.5.7 release.

## What This Changes

The issue asks the Control UI Agents/Sessions surface to show each agent's actual thinking default when the agent uses a model different from the global default.

## Best Possible Solution

Keep the current row-scoped gateway session metadata path and let the next release carry it to users on v2026.5.7.

## Reproduction Assessment

No high-confidence failing reproduction remains on current main. The source path now computes Control UI row thinkingDefault from the row's agent/model metadata, and DeepSeek V4 Flash declares high as its provider default.

## Solution Assessment

Yes, the current implementation is the right shape: the UI consumes per-row thinkingDefault and falls back to session defaults only for matching default-model rows, avoiding the global-default leak reported here.
