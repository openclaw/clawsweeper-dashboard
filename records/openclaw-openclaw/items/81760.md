## Summary

Keep this issue open: current main still has the reported Control UI/gateway contract gap, and the open closing PR at https://github.com/openclaw/openclaw/pull/81795 is the active fix path but has not merged.

## What This Changes

Control UI should display each agent's effective configured thinking default for non-default models instead of deriving it from the global default model.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/81795, or an equivalent gateway/UI change, so agent rows expose resolved thinking metadata and the Control UI uses that value as the source of truth.

## Reproduction Assessment

Yes, source-level. Current main still omits per-agent thinking metadata from agents.list rows despite the documented per-agent thinkingDefault contract, and the UI has no field to render it.

## Solution Assessment

Yes. The narrow maintainable solution is to expose gateway-resolved per-agent thinking metadata and guard global-default fallbacks by model match, which is what the linked open PR attempts.
