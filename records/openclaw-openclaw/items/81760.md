## What This Changes

The issue reports that Control UI derives an agent thinking default from the global default model instead of the agent's configured model, causing wrong thinking-default display for non-default-model agents.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/81795 or an equivalent focused change that exposes resolved per-agent thinking metadata to agent rows and guards global-default fallbacks by model match.

## Reproduction Assessment

Yes. Source inspection on current main shows agents.list rows and the shared agent-row type still omit thinking metadata, while docs say per-agent thinkingDefault overrides the global default.

## Solution Assessment

Yes. The narrow maintainable fix is the linked gateway/shared/UI update that carries resolved per-agent thinking metadata and avoids using global defaults for different-model sessions; the issue should stay open until that PR or an equivalent merges.
