## What This Changes

The issue asks Control UI to show each agent's effective per-agent thinking default when the agent uses a model different from the global default.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/81795 or an equivalent gateway/UI fix so agent rows and chat status use resolved per-agent thinking metadata as the source of truth.

## Reproduction Assessment

Yes, source-level: current main omits thinking metadata from agents.list rows, and the chat plus /think fallback paths can still use the global default before model-specific resolution. I did not run the UI because this review was read-only.

## Solution Assessment

Yes. The narrow maintainable solution is to expose gateway-resolved per-agent thinking fields and avoid global-default fallbacks when the active session model differs, which is what the linked open PR attempts.
