## Summary

Keep open: the custom/proxy blocker from the earlier review is fixed and the PR now includes terminal runtime proof, but one credential-boundary defect remains because the new official-endpoint gate is hostname-only and would rotate global Gemini keys to an HTTP Gemini base URL.

## What This Changes

The branch routes Google Generative AI LLM stream setup through the shared API-key rotation helper and adds regression coverage for 429 recovery, OAuth JSON credentials, and custom/proxy endpoints.

## Best Possible Solution

Keep the provider-owned LLM rotation approach, but require an HTTPS official Gemini endpoint before collecting global keys and keep OAuth, Vertex, and custom-proxy exclusions covered by tests.

## Reproduction Assessment

Yes. The linked bug is source-reproducible on current main because the Google LLM stream path resolves one API key and does not collect the configured rotation key set; the remaining PR defect is source-reproducible with an HTTP Gemini base URL plus a 429 response stub.

## Solution Assessment

Not yet. Reusing the shared rotation helper at the pre-SSE boundary is the right fix shape, but the official-endpoint gate should require HTTPS before global Gemini keys are rotated.
