## Summary

Keep open: this is a focused candidate fix for the Google/Gemini LLM key-rotation bug, but the latest head still has a credential-boundary defect before merge because global Gemini keys can rotate to a non-TLS configured official-host URL.

## What This Changes

The PR routes Google Generative AI LLM pre-stream requests through the shared API-key rotation helper and adds regression tests for 429 recovery, OAuth JSON credentials, and custom Gemini endpoints.

## Best Possible Solution

Keep the provider-owned pre-SSE rotation approach, but require an HTTPS official Gemini endpoint before collecting global keys and preserve OAuth, Vertex, and custom-proxy exclusions with focused tests.

## Reproduction Assessment

Yes. The linked bug is source-reproducible on current main because the Google LLM stream path resolves one API key and does not collect the configured rotation key set; the remaining PR defect is source-reproducible with an HTTP Gemini base URL plus a 429 response stub.

## Solution Assessment

No, not yet. Reusing the shared rotation helper at the pre-SSE boundary is the right fix shape, but the official-endpoint gate should require HTTPS before global Gemini keys are rotated.
