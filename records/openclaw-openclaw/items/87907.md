## Summary

Keep open: this protected maintainer draft still has a merge-blocking compatibility defect because the new paused-index empty result can be consumed as an ordinary zero-hit search and trigger a forced full reindex.

## What This Changes

The branch adds memory index identity validation and CLI/status/search behavior for provider/model mismatches so memory-core can stop trusting stale SQLite vector indexes.

## Best Possible Solution

Keep the PR open and make invalid index identity a distinct paused/unavailable state for callers, or make the memory_search zero-hit retry skip forced sync when status.custom.indexIdentity is invalid, then add tool-level regression coverage.

## Reproduction Assessment

Yes, source-reproducible: PR head returns [] for invalid index identity, memory_search retries any zero-hit result with forced sync, and runSync maps forced search sync to full reindex. I did not run tests in this read-only review.

## Solution Assessment

No, not yet. The manager-level trust gate is plausible, but returning [] is not safe until callers distinguish paused identity from ordinary no matches or skip the existing forced zero-hit retry.
