## Summary

Keep open: this is a useful memory-core bug fix, but the draft still has a merge-blocking provider-cutover path where the agent memory_search tool can force a full reindex after the PR otherwise pauses mismatched indexes.

## What This Changes

The PR adds memory-core index identity validation for provider/model/settings metadata, pauses mismatched searches, updates CLI/status warnings, and adds focused memory tests plus an implementation plan.

## Best Possible Solution

Keep the shared identity gate, but make search-triggered forced sync respect dirty identity so only explicit indexing commands rebuild mismatched provider indexes, with a tool-level regression covering the memory_search path.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: make manager.search return zero on a mismatched index, then call the agent memory_search tool so its zero-hit retry invokes sync({ reason: "search", force: true }). That reaches the PR's full-reindex branch before the identity pause can return.

## Solution Assessment

No, not yet: the shared identity gate is the right direction, but the implementation is incomplete while the tool-level search retry can bypass the intended paused-search behavior. The safer fix is to restrict forced search sync to empty-index bootstrap or valid identity, leaving mismatched indexes for explicit indexing commands.
