## Summary

Keep open: current main and v2026.6.9 still serialize the relevant bundle-tools startup path, but this PR is not merge-ready because it suppresses whole-family initialization failures, weakens LSP failure diagnostics, and still lacks full MCP+LSP after-fix proof.

## What This Changes

The branch parallelizes bundled MCP catalog loading, bundled LSP server startup, and top-level MCP-versus-LSP initialization in the embedded attempt runner, with post-hoc MCP/LSP tool-name deduplication.

## Best Possible Solution

Parallelize independent startup while preserving the current whole-runtime failure contract, retaining per-server diagnostics, bounding or explicitly accepting fan-out, and proving the full MCP+LSP bundle-tools path with redacted timing output.
