## Summary

## What This Changes

The PR parallelizes bundled MCP catalog loading, bundled LSP server startup, and top-level MCP/LSP initialization in embedded agent attempts.

## Best Possible Solution

Parallelize independent startup while preserving whole-runtime failure propagation, retaining per-server diagnostics, and proving the full MCP+LSP bundle-tools path with redacted timing output.

No as submitted. Parallelizing independent startup is a plausible owner-boundary fix, but the fail-open attempt catches, weaker LSP diagnostics, and incomplete full-path proof should be fixed or explicitly accepted before merge.
