## Summary

Keep this PR open, but it is not merge-ready: the observe-only return is added after WhatsApp ack/status reaction setup, so disabled-policy messages can still produce visible outbound reactions despite the stated hook-only safety boundary.

## What This Changes

The PR changes WhatsApp inbound access control so disabled DM/group messages can be admitted as observe-only when `message_received` hooks are enabled, then stops processing after hook delivery.

## Best Possible Solution

Keep the existing opt-in hook direction, but make observe-only admissions hook-only across access control, processMessage, preflight, and broadcast paths before merge.

## Reproduction Assessment

Yes. Source inspection shows `messages.statusReactions.enabled` or `channels.whatsapp.ackReaction` can reach outbound reaction setup before the proposed observe-only return.

## Solution Assessment

No. The access-control admission idea is plausible, but the current return point is too late to preserve the hook-only/no-outbound-side-effects contract.
