## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Cancel the watchdog before in-process restarts continue:** `src/gateway/server-close.ts:879`
  - body: The close handler now arms the post-shutdown watchdog after every close and discards the cancel handle. During the existing SIGUSR1 in-process restart path, the same Node process starts the next gateway after close, so the old unref'd timer will still fire about 5 seconds later and call process.exit(0) against the restarted gateway. Gate this on non-restart closes or retain/cancel the handle from the restart startup path.
  - confidence: 0.96
- **[P1] Prove or narrow the live-probe 503 contract:** `src/gateway/server-http.ts:324-326`
  - body: This changes both live probes to return 503 whenever shutdown starts. Current main treats /health and /healthz as shallow live probes that answer 200, so external monitors and service managers may now see ordinary shutdown as a failed health check; either narrow this to the supervised-lock path or add explicit compatibility rationale and upgrade proof for the public probe contract.
  - confidence: 0.8
- **[P3] Deduplicate unhealthy-probe zombie traces:** `src/cli/gateway-cli/run.ts:462-466`
  - body: The new zombie_detected trace and warn log run on every supervised lock-recovery retry. A normal draining gateway can therefore emit duplicate alerts until timeout; guard the signal so the first unhealthy observation is logged once per recovery cycle, with any later retry detail summarized separately.
  - confidence: 0.88
