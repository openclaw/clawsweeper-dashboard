## Summary

Keep this open: current main and the latest release still have a source-level path where WhatsApp auto-replies can build quoted metadata for a bot-authored group message with `fromMe: false` and the replying user's participant after the bot's own send echo is filtered before quote metadata caching.

## What This Changes

The issue reports that WhatsApp Desktop drops OpenClaw's reply bubble when a user swipe-replies to a group message originally sent by the bot.

## Best Possible Solution

Fix this in the WhatsApp plugin by retaining or resolving bot-authored outbound message metadata for quoted replies while preserving the existing echo filter that prevents the bot from processing its own messages as inbound user input.

## Reproduction Assessment

Yes at source level, but not with a live Desktop run: current main filters bot-sent echoes before metadata caching and later falls back to user-authored quote metadata when the quoted ID is cache-missed.

## Solution Assessment

Yes in direction: the narrowest maintainable solution is a WhatsApp plugin fix that makes bot-authored outbound message keys available to quote construction, not a core reply-threading or config change.

Current main records bot-sent IDs only in the recent-outbound echo filter, filters those echoes before `cacheInboundMessageMeta`, and then `deliverWebReply` falls back to `fromMe: false` plus the replying group sender when a user replies to the bot's own previous message. Keep the fix inside `extensions/whatsapp`; do not add config, env vars, core reply-threading changes, or runtime fallback shims. Preserve existing behavior for quotes of user-authored messages and for the echo filter that prevents self-processing. Add focused regression coverage proving a bot-authored group message quote produces a Baileys quoted key with `fromMe: true` and the correct bot/self participant, and that cached inbound user-message quotes still keep their current metadata. Check Baileys 7.0.0-rc13 quoted-message behavior while implementing. No changelog edit.
