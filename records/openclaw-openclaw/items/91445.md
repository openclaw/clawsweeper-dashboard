## What This Changes

User reports that WhatsApp Desktop drops OpenClaw's reply bubble when a group member swipe-replies to a bot-authored WhatsApp message.

## Best Possible Solution

Fix this in the WhatsApp plugin by retaining bot-authored outbound message metadata for quote construction while preserving the existing self-echo filter, then prove it with focused regression tests and a WhatsApp Desktop round trip when feasible.

## Reproduction Assessment

Yes at source level, but not as a fresh live Desktop repro. Current main can still build quoted options with fallback `fromMe: false` and the replying user's participant when the quoted bot message was only remembered for echo suppression.

## Solution Assessment

Yes, the reported fix direction is the best shape: keep echo suppression, but make accepted bot-authored outbound sends available to quote metadata lookup. A core reply-threading change or config knob would be the wrong layer for this narrow WhatsApp plugin behavior.
