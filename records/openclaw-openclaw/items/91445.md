## Summary

Keep open: current main and the latest release still lack outbound quote metadata caching for bot-sent WhatsApp messages, and the open linked PR is the active fix candidate rather than a merged implementation.

## What This Changes

The issue asks OpenClaw’s WhatsApp plugin to build correct quoted-message metadata when a user swipe-replies to a bot-authored message so WhatsApp Desktop renders the bot’s reply bubble.

## Best Possible Solution

Review, refresh, and either land or close https://github.com/openclaw/openclaw/pull/94879; close this issue only after the fix merges or maintainers explicitly reject this behavior.

## Reproduction Assessment

Yes at source level: current main tracks bot-sent IDs for echo suppression, drops matching fromMe echoes before quote metadata caching, and then falls back to fromMe:false plus the replying participant when building the quoted reply. I did not establish a fresh live WhatsApp Desktop reproduction in this read-only review.

## Solution Assessment

Yes, the narrow maintainable direction is to cache bot-authored outbound quote metadata at the existing send-tracking boundary instead of guessing later in auto-reply delivery. The open linked PR targets that boundary, but it needs refresh/review before this issue can close.
