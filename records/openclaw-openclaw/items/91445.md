## Reproduction Assessment

Yes at source level: current main drops recent bot-sent echoes before caching quote metadata, then falls back to `fromMe:false` and the replying group participant when building the quoted reply. A fresh live WhatsApp Desktop reproduction is not attached here.

## Solution Assessment

Yes, the linked PR appears to target the right boundary by caching outbound bot message metadata at the send-tracking path instead of guessing later in reply delivery. The issue should remain open until that candidate merges or maintainers choose another outcome.
