## Summary

Keep open. Current main has not absorbed the WebChat ordering/reset work, and the PR is still draft with a verified reset-cleanup regression plus a missing changelog entry.

## What This Changes

The PR changes WebChat/Gateway chat history projection, reset/session-change reconciliation, stream/tool rendering, terminal error display, related tests, and two QA scenario timeouts.

## Best Possible Solution

Update this PR in place with immediate tool-stream cleanup on successful synchronous resets, add the active changelog entry, and keep the startup-greeting decision tracked separately.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a bare WebChat `/new` or `/reset` after live tool/stream output while `chat.history` is delayed or fails; `clearChatHistory` clears `chatMessages`, but `buildChatItems` can still render stale stream segments and tool messages.

## Solution Assessment

No, not as-is. The PR targets the right WebChat/Gateway surfaces, but the synchronous reset path should call the existing `resetToolStream` helper before any async reload can race or fail, and the user-facing fix needs a changelog entry.
