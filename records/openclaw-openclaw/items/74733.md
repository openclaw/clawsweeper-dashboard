This PR changes Control UI WebChat and Gateway chat-history projection to stabilize message ordering, reset/session refresh behavior, tool stream rendering, and visible terminal error handling, with focused UI/Gateway tests and QA timeout tweaks.

## Best Possible Solution

## Reproduction Assessment

Yes. A high-confidence source-level path exists: current main appends unrelated final messages during an active stream and can clear chatMessages while buildChatItems still renders stream/tool artifacts independently; this review did not run browser E2E because the task is read-only.

## Solution Assessment

No, not as-is. The PR targets the right WebChat/Gateway surfaces, but the new synchronous reset path needs resetToolStream cleanup before reload and the user-facing fix needs the required changelog entry.
