## Summary

Keep open: this is a protected maintainer draft PR, and the current head still leaves a skipped-prompt path where post-turn context-engine hooks can run while the coarse session write lock remains held.

## What This Changes

This PR narrows embedded PI session transcript write-lock scope, adds stale/max-hold lock configuration, propagates lock options through transcript paths, and adds focused regression coverage.

## Best Possible Solution

Keep the lock-controller/config direction, but release or fence the initial embedded lock before any post-turn finalizer work even when no provider stream is submitted, with regression coverage for skipped-prompt context-engine finalizers.

## Reproduction Assessment

Yes. Current main source shows the embedded lock spans the model turn, and the PR body includes concrete Blacksmith Testbox lock-span and contention-timeout proof; the remaining PR defect is source-reproducible on skipped-prompt paths.

## Solution Assessment

No. The overall approach is close, but the release point must cover paths where `activeSession.prompt()` is never called, not only the stream wrapper around provider submission.
