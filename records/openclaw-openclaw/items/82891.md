## Summary

Keep open: this protected maintainer PR addresses a real session-lock contention path and includes focused proof, but the patch still leaves post-turn context-engine work able to perform provider I/O while the reacquired transcript lock is held.

## What This Changes

This PR adds an embedded-attempt session lock controller, exposes session write-lock stale/max-hold config and env options, propagates those options through transcript paths, and updates docs, baselines, and regression tests.

## Best Possible Solution

Keep the lock-controller and config direction, but move context-engine afterTurn, ingest, and maintenance work outside the transcript lock and acquire the lock only around actual SessionManager or transcript rewrite/persistence operations.

## Reproduction Assessment

Yes. Current main source shows the embedded lock spans the model turn, and the PR body gives a concrete Blacksmith Testbox reproduction with lock-span timings and a same-process contention timeout.

## Solution Assessment

No. The approach is close, but the post-turn context-engine finalizer is still inside `withSessionWriteLock` even though that path can run plugin and provider/model work.
