## Label Justifications

- `P2`: The reported stall can disrupt channel delivery and Gateway availability, but the concrete fix is in an external plugin with limited repo-local action.
- `impact:message-loss`: The issue says the event-loop stall prevents other channels and model fallback mechanisms from working correctly.

## Summary

The report is plausible, but the blocking startup work is attributed to Tencent's external Weixin channel plugin source, not code that lives in this repository; OpenClaw current main and the latest release both document and catalog that boundary.

## What This Changes

Fix a Weixin channel startup stall where the external @tencent-weixin/openclaw-weixin plugin allegedly blocks the Node event loop for about 21 seconds during Gateway start.

## Best Possible Solution

Keep OpenClaw core channel-agnostic; Tencent's external Weixin plugin should replace blocking startup fs.*Sync work with async or off-thread behavior and publish an updated package.

## Reproduction Assessment

No. The liveness log is plausible, but the named startup files are not in this checkout and the external @tencent-weixin/openclaw-weixin package was not executed during this read-only repo review.

## Solution Assessment

No. A service-specific core startDelay or Weixin startup policy is not the best fit; the maintainable fix is for the external plugin to make its startup non-blocking through the existing async channel adapter.
