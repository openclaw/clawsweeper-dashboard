## Summary

Keep open. Current main still exposes Feishu `read`, but it is implemented only as a single `messageId` lookup; neither the broader open pagination issue #71452 nor the closed-unmerged Feishu chat-history PR #35517 implements this Feishu message-tool gap on main.

## What This Changes

The issue requests Feishu `message read` support for paginated chat-history reads using `chatId` plus limit/page-token/time filters instead of only single-message lookup by `messageId`.

## Best Possible Solution

Add a Feishu-owned paginated chat-container history path for `read` while preserving the existing single-message `messageId` behavior and documenting the supported parameters.
