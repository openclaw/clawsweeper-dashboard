## Repair Work Prompt

Fix the Feishu `message read` chat-history gap from https://github.com/openclaw/openclaw/issues/74729. Current main advertises Feishu `read`, but `extensions/feishu/src/channel.ts` only accepts `messageId` and calls `getMessageFeishu`; `chatId`/`limit`/`before`/`after`/`pageToken` cannot retrieve history. Add a Feishu-owned paginated chat-history read path that uses `client.im.message.list` with `container_id_type: "chat"` and `container_id` from `chatId`/target/channelId. Support bounded `limit`/`pageSize`, `pageToken`/`page_token`, and Feishu time filters (`startTime`/`start_time`, `endTime`/`end_time`, plus the shared `before`/`after` names if appropriate). Return multiple normalized messages plus `hasMore` and next `pageToken`. Preserve the existing single-message `messageId` read behavior. Keep core extension-agnostic; only add shared alias/normalization coverage if needed so Feishu `chatId` satisfies `read` target validation. Do not change Feishu dependency versions, credentials, onboarding, or startup migrations. Add/adjust focused tests in Feishu and shared message-action alias/spec coverage, update `docs/channels/feishu.md`, `docs/cli/message.md` if the CLI surface changes, and add a user-facing `CHANGELOG.md` fix entry.
