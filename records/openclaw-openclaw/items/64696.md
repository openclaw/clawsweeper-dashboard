## Repair Work Prompt

Fix the remaining memory-wiki bridge import and lint durability bugs tracked by https://github.com/openclaw/openclaw/issues/64696. Current main still has these source-proven issues: collectBrokenLinkIssues does not resolve native Markdown ./ or ../ targets relative to the containing page; generated reports call formatPageLink with page.relativePath so native report links are vault-root style; extractWikiLinks scans raw Markdown/fenced bridge content so reply directives can become broken-link targets; writeBridgeSourcePage renderFingerprint has no renderer contract/version token and renders raw source bytes; shouldSkipImportedSourceWrite skips when sync metadata matches and the destination exists without validating source-page frontmatter/provenance. Coordinate with open PRs https://github.com/openclaw/openclaw/pull/73722 and https://github.com/openclaw/openclaw/pull/78556: reuse or supersede their lint resolver work, but do not leave bridge render invalidation or malformed cached-page repair untested. Keep the fix inside extensions/memory-wiki; do not add config knobs, core policy, or plugin SDK surface. Add regression coverage in lint/markdown/compile/bridge/source-sync tests for the acceptance cases, and add a single-line CHANGELOG.md Fixes entry because wiki lint/import behavior is user-visible.
