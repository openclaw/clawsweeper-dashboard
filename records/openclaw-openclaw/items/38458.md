## Summary

Keep open. Current main still has only a partial global suppression setting: `messages.suppressToolErrors` exists, but mutating failures such as `message`, `write`, and `exec` still bypass it, and the requested per-channel or enum-style policy is not implemented.

## What This Changes

The issue requests a channel-level or broader config policy to suppress user-facing tool error output on chat surfaces while preserving tool failures in agent context.

## Best Possible Solution

Define one documented tool-error visibility policy for chat delivery, with a global default and any intended per-channel override, then apply it before user-facing payload delivery while preserving tool failures in agent context.

## Reproduction Assessment

Yes for the narrow observable problem: current source and tests show mutating tool failures still produce user-facing error payloads even when `messages.suppressToolErrors` is true. The broader per-channel/enum request is a feature-policy decision rather than a failing existing contract.

## Solution Assessment

Unclear. Reordering the existing global suppression check is the narrow fix for the current boolean gap, but adding per-channel or enum visibility needs maintainer policy direction and docs/config-schema alignment.
