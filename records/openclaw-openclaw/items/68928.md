## Summary

Keep open. The PR is a draft CSS-only visual polish pass that current main has not already implemented, and the hydrated diff still has two concrete merge-blocking regressions: light-mode semantic colors lose readable contrast and exec approval modal overflow guards are removed.

## What This Changes

The PR changes Control UI theme tokens toward a softer blue/yellow/red palette, updates overview attention item/link styling, and removes exec approval modal overflow constraints.

## Best Possible Solution

Keep the visual polish scoped to Control UI theme and dashboard CSS, preserve exec approval overflow protections, and choose semantic warning/danger foreground colors that stay readable in both light and dark modes across all token consumers before a maintainer design pass.

## Reproduction Assessment

Yes. The contrast regression reproduces by comparing the PR token values against the current light background, and the modal regression reproduces by inspecting the PR deletion against the current modal shell and exec approval scroll constraints.

## Solution Assessment

No. The CSS-only scope is appropriate, but the current diff is not the best mergeable form because it weakens semantic foreground contrast and removes required overflow containment; the safer fix is to restore those guards and use accessible semantic tokens.
