## Summary

Keep open. The CSS-only palette polish is a reasonable scope, but the current PR is not merge-ready because it regresses light-mode semantic color contrast and removes exec approval overflow containment that current modal behavior still needs.

## What This Changes

The PR changes `ui/src/styles/base.css` and `ui/src/styles/components.css` to soften accent, warning, and danger colors, adjust overview attention styling, and remove exec approval modal overflow constraints.

## Best Possible Solution

Keep the polish CSS-scoped, choose light-mode semantic foreground tokens with accessible contrast, preserve exec approval command scroll containment, then do the requested visual pass.

## Reproduction Assessment

Yes. A direct token contrast calculation reproduces the light-mode readability regression, and source inspection shows long exec/plugin approval content still depends on per-card overflow containment because the shared modal leaves slotted overflow visible.

## Solution Assessment

No. The CSS-only approach is appropriate, but the proposed branch is not the best mergeable form until it keeps semantic colors readable and preserves modal command scrolling.
