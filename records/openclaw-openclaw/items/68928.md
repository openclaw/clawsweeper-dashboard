## Summary

Keep open. Current main does not already implement the palette change, and this draft CSS-only PR is not merge-ready because it lowers light-mode semantic color contrast and removes overflow containment still needed by the exec approval modal.

## What This Changes

The PR changes Control UI accent, warning, danger, and overview attention styling while removing exec approval card and command overflow constraints.

## Best Possible Solution

Keep the palette polish scoped to Control UI CSS, restore exec approval scroll containment, and choose semantic foreground tokens that remain readable in light and dark modes before maintainer visual review.

## Reproduction Assessment

Yes. The contrast issue reproduces with a direct token contrast calculation, and the overflow issue follows from the PR deleting content scroll guards while the shared modal keeps overflow visible.

## Solution Assessment

No. The CSS-only scope is appropriate, but the current patch is not the best mergeable form until it restores modal containment and preserves accessible semantic colors.
