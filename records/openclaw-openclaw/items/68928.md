## Summary

This draft PR should stay open because the branch still has two concrete CSS regressions: light-mode warning/danger foreground contrast drops below readable text contrast, and long exec approval commands lose internal scroll containment.

## What This Changes

The PR changes Control UI theme tokens to a softer blue/amber/red palette, adjusts overview attention item/link styling, and removes exec approval modal overflow constraints.

## Best Possible Solution

Keep the palette polish CSS-only, choose light-mode semantic foreground tokens that remain readable, restore exec approval command scroll containment, and then complete a manual dashboard and approval-modal visual pass.

## Reproduction Assessment

Yes. A token contrast calculation reproduces the light-mode readability regression, and source inspection reproduces the overflow issue by comparing the PR deletions with current modal and approval CSS.

## Solution Assessment

No. The CSS-only direction is maintainable, but the proposed branch is not mergeable until it preserves readable semantic foreground colors and restores the existing long-command containment.
