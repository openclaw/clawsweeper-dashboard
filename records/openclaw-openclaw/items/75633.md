## Summary

Keep open. The specified current main still has reachable unguarded assistant-content block dereferences in the Anthropic Messages replay/conversion path, and the related fix PR was closed unmerged, so the crash class remains actionable in this repository.

## What This Changes

The issue reports that MiniMax's Anthropic-compatible provider path can crash when the Anthropic Messages adapter handles empty, thinking-only, or malformed assistant content before usable text is produced.

## Best Possible Solution

Harden assistant content normalization at the transport boundary before any `block.type` reads, cover empty, null, undefined, sparse, and thinking-only content with focused tests, and keep MiniMax thinking/reasoning visibility policy unchanged except where existing replay rules already convert safe text.

## Reproduction Assessment

Yes for the source-level crash class: a replayed assistant message with `content: [undefined]`, `content: [null]`, or a sparse slot reaches `transformTransportMessages` and can throw before failed turns are filtered. The live MiniMax intermittency remains provider-dependent.

## Solution Assessment

Yes, a narrow defensive guard/normalization fix in `transport-message-transform.ts` plus the Anthropic conversion path is the best maintainable approach. A broader fallback that surfaces hidden thinking as user-visible text by default would be riskier.
