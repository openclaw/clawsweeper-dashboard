## Summary

Keep open. Current main still has unguarded assistant content-block dereferences in the shared and Anthropic replay paths, and linked PR #75884 is open but not merged.

## What This Changes

The issue asks OpenClaw's Anthropic Messages transport to handle MiniMax thinking-only, empty, or malformed assistant content without crashing.

## Best Possible Solution

Review and land PR #75884 or an equivalent narrow transport-boundary guard that skips or normalizes malformed assistant content blocks in both replay paths, with focused regression tests and a changelog entry.

## Reproduction Assessment

Yes. A persisted assistant message with `content: [undefined]`, `content: [null]`, or a sparse content slot reaches the replay transform and can throw when `block.type` is read before filtering.

## Solution Assessment

Yes. The narrow maintainable fix is to harden the transport replay boundary before dereferencing content blocks, rather than closing this issue before the linked PR lands or broadly exposing hidden thinking as visible text.
