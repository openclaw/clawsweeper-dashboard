## Summary

Keep open. Current main still has unguarded `block.type` reads in the shared transport transform and Anthropic replay conversion, and open PR #75884 explicitly targets this issue but is not merged.

## What This Changes

The issue requests hardening the `anthropic-messages` adapter so MiniMax Anthropic-compatible responses with thinking-only, empty, or malformed assistant content do not crash before producing output.

## Best Possible Solution

Land PR #75884 or an equivalent narrow guard that skips or normalizes malformed assistant content blocks in both replay conversion paths, with focused regression tests and a changelog entry if user-visible behavior changes.

## Reproduction Assessment

Yes. A replayed assistant message with `content: [undefined]`, `content: [null]`, or a sparse content slot reaches `transformTransportMessages` and can throw on `block.type`; the MiniMax live failure rate was not rerun.

## Solution Assessment

Yes. The narrow maintainable fix is transport-boundary hardening in `transformTransportMessages` and `convertAnthropicMessages`, rather than exposing internal thinking as visible text by default or closing this issue before the linked PR merges.
