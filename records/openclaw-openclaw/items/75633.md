## Summary

Keep open. Current main still has unguarded `block.type` reads in the shared and Anthropic-specific transport replay loops, and open PR #75884 explicitly references this issue but is not merged.

## What This Changes

The issue requests defensive handling in the `anthropic-messages` MiniMax path when upstream responses have thinking-only, empty, or malformed content blocks so sessions do not crash.

## Best Possible Solution

Land PR #75884 or an equivalent narrow fix that normalizes or skips malformed assistant content blocks in both transport replay paths, with focused regression tests and a user-facing changelog entry if behavior changes.

## Reproduction Assessment

Yes. A replayed assistant message with `content: [undefined]`, `content: [null]`, or a sparse content slot reaches `transformTransportMessages` and can throw on `block.type` before failed turns are filtered; the provider’s intermittent live behavior was not rerun.

## Solution Assessment

Yes. The narrow maintainable fix is transport-boundary hardening in `transformTransportMessages` and `convertAnthropicMessages`, not exposing hidden thinking as visible text by default or closing this issue before the linked PR merges.
