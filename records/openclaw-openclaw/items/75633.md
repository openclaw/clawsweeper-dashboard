## Summary

Keep open. Current main still has the reported crash class in the Anthropic transport replay path, and open PR #75884 explicitly references this issue but has not merged.

## What This Changes

The issue requests defensive handling for MiniMax Anthropic-compatible responses or replayed assistant content that lacks a normal text block and can crash the anthropic-messages adapter.

## Best Possible Solution

Land PR #75884 or an equivalent narrow hardening that skips or normalizes malformed assistant content blocks in both replay transforms, with focused regression tests and a changelog entry if user-visible behavior changes.

## Reproduction Assessment

Yes. A replayed assistant message whose content array contains a null, undefined, or sparse slot reaches `transformTransportMessages` and can throw on `block.type` before the failed assistant turn is filtered.

## Solution Assessment

Yes. The maintainable fix belongs at the transport replay boundary in the shared and Anthropic-specific transforms, not by exposing hidden thinking as visible text by default or closing the issue before the linked PR merges.
