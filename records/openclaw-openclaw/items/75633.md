## Summary

Keep open. Current main still has unguarded assistant content-block dereferences in the shared transport transform and Anthropic payload conversion, and open PR #75884 explicitly references this issue but is not merged.

## What This Changes

The issue requests hardening the Anthropic Messages adapter so MiniMax thinking-only, empty, or malformed assistant content does not crash cron-driven sessions.

## Best Possible Solution

Review and land PR #75884, or an equivalent narrow fix that skips or normalizes nullish assistant content blocks in both transport replay paths with focused regression coverage.

## Reproduction Assessment

Yes. A replayed assistant message with `content: [undefined]`, `content: [null]`, or a sparse content slot can still reach `block.type` in current main before the failed-turn filter or Anthropic payload conversion can skip it.

## Solution Assessment

Yes. The narrow maintainable path is transport-boundary hardening while keeping the existing MiniMax default-thinking mitigation, not closing the issue before the linked PR merges.
