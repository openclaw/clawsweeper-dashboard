## What This Changes

The PR changes Feishu streaming card updates so CardKit content update requests send the accumulated full text instead of only the suffix delta, and updates the related unit expectations.

## Best Possible Solution

Merge the narrow Feishu streaming-card fix after redacted live Feishu/Lark or runtime CardKit evidence shows streaming updates retain the full accumulated response.

## Reproduction Assessment

Yes from source inspection: current main computes a suffix payload for follow-up streaming updates and sends that suffix to the CardKit content update endpoint, while the upstream schema says full text should be passed. I did not run a live Feishu bot in this read-only review.

## Solution Assessment

Yes, this appears to be the best narrow fix: the dispatcher already passes cumulative snapshots, and the bug is in the Feishu streaming session helper that converts those snapshots into CardKit content update payloads. Broader dispatcher or config changes would add unnecessary surface.
