## What This Changes

The PR adds a config-path suggestion helper and wires `Did you mean` hints into `openclaw config get` and `openclaw config unset` not-found errors.

## Best Possible Solution

Land the suggestion feature after adding the required changelog entry and deciding whether candidate paths should come from the runtime schema/UI-hint inventory, or explicitly documenting and testing that the feature is limited to static core config paths.

## Reproduction Assessment

Not applicable as a standalone bug report. The before/after path is clear: current main emits only `Config path not found`, while the PR would add hints for close static paths such as `gateway.prot`.

## Solution Assessment

Unclear. The get/unset integration is narrow and maintainable, but the static `FIELD_HELP` candidate list is narrower than OpenClaw's current runtime config schema surface for bundled channels and installed plugins.
