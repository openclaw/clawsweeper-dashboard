## Summary

## What This Changes

The PR adds a `FIELD_HELP`-backed config-path suggestion helper, unit tests, and `Did you mean` stderr hints for `openclaw config get` and `openclaw config unset` not-found errors.

## Best Possible Solution

Land the hint feature only after rebase, contributor-provided runtime proof, and a maintainer decision on whether suggestions should use runtime schema/UI-hint keys or clearly advertise static-only scope.

## Reproduction Assessment

Not applicable as a bug report. Source inspection shows current main only prints not-found recovery text, while the PR would add hints for close static paths such as `gateway.prot`.

## Solution Assessment

Unclear. The CLI hook and matcher are narrow, but maintainers should decide the exact-match/known-but-unset UX and whether the candidate inventory should come from runtime schema/UI hints instead of only `FIELD_HELP`.

## Review Findings

Overall correctness: patch is correct

Overall confidence: 0.82

Full review comments:

- **[P3] Avoid suggesting the same path as a correction:** `src/config/suggest-path.ts:117-118`
  - body: Because Phase 2 accepts distance 0, a valid-but-unset key can produce `Config path not found: gateway.port` followed by `Did you mean: gateway.port?`, especially in `config unset` where the raw resolved config is checked. Skip exact matches for `Did you mean` output or use separate copy for known-but-unset paths.
  - confidence: 0.78
- **[P3] Clarify static-only config suggestions:** `src/config/suggest-path.ts:76-78`
  - body: The helper only builds candidates from `FIELD_HELP`, while current schema responses merge plugin and channel UI hints. Deep channel paths such as Telegram streaming keys can therefore miss suggestions despite being part of the active config surface, so either use runtime schema/hint keys or make the static scope explicit in tests and copy.
  - confidence: 0.84
