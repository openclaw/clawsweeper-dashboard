The PR adds a FIELD_HELP-backed Levenshtein suggestion helper and CLI hint output for `openclaw config get` and `openclaw config unset` not-found errors, plus unit tests.

## Best Possible Solution

Build suggestions from the same runtime schema/UI-hint inventory used by config schema lookup, filter non-copyable placeholders, cover base plus channel/plugin paths, and refresh redacted terminal proof.

## Reproduction Assessment

Not applicable; this is a feature PR rather than a broken existing-behavior bug. Source inspection gives a high-confidence review path: the branch candidate set is static FIELD_HELP while the advertised channel-path proof depends on runtime channel metadata.

## Solution Assessment

No; FIELD_HELP-only suggestions do not cover channel/plugin paths that current main exposes through runtime schema UI hints. The safer path is to use that runtime inventory or deliberately scope the feature and proof to base config keys only.
