## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Resolve floor shares when no flush plan exists:** `src/auto-reply/reply/agent-runner-memory.ts:390-393`
  - body: When no memory-flush plan is configured, `memoryFlushPlan` is null and the preflight threshold falls back to raw `reserveTokensFloor` or `20_000`. A config that only sets `reserveTokensFloorShare` is ignored on this path, so large-window sessions compact later than the configured share intends.
  - confidence: 0.91
- **[P2] Preserve absolutes for fallback context windows:** `src/auto-reply/reply/memory-flush.ts:37-49`
  - body: `applyMemoryFlushSharesToPlan` treats every numeric `contextWindowTokens` as authoritative, but its caller can pass `DEFAULT_CONTEXT_TOKENS` when model metadata is unknown. That makes share thresholds override the plan's absolute values for custom or unknown models, contrary to the documented fallback behavior.
  - confidence: 0.86
- **[P2] Use authoritative context budgets for shares:** `src/agents/pi-embedded-runner/run/attempt.ts:916`
  - body: Passing `params.model.contextWindow` into share resolution can use a synthesized default window from fallback model construction rather than a real model limit. Unknown or custom small-window models can therefore compute share budgets from 200k instead of preserving absolute/default budgets.
  - confidence: 0.82
