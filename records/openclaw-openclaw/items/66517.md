## Summary

Keep this PR open. Current main still lacks the proposed ratio-based compaction budget fields, the branch is open and unmerged, and the proposed patch has unresolved correctness issues around preflight floor-share resolution and fallback context-window handling.

## What This Changes

The PR adds four ratio-based compaction `*Share` config fields, updates config type/schema/help/generated metadata, threads context-window values into Pi compaction and memory-flush paths, and adds focused tests.

## Best Possible Solution

Keep this PR open as an implementation candidate, but require a rebase onto current main, fixes for the context-window and preflight resolution bugs, aligned schema/help/docs/tests, and maintainer approval of the public config surface before merge.

## Reproduction Assessment

Yes. The main absence is reproduced by source/schema search, and the PR bugs are reproducible by static paths on the PR head: `reserveTokensFloorShare` is skipped when no flush plan exists, and unknown model windows can fall back to `DEFAULT_CONTEXT_TOKENS` before share resolution.

## Solution Assessment

No, not yet. Ratio sibling fields may be a reasonable direction, but the current patch is not the best merge shape until it distinguishes authoritative context windows from fallback guesses and applies the same floor-share resolver in preflight paths without memory-flush plans.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Resolve floor shares when no flush plan exists:** `src/auto-reply/reply/agent-runner-memory.ts:390-393`
  - body: When no memory flush plan is registered, `memoryFlushPlan` is null and the preflight threshold falls back to the raw absolute `reserveTokensFloor` or `20_000`. A config that only sets `reserveTokensFloorShare` is therefore ignored on this path, so large-window sessions compact later than requested.
  - confidence: 0.91
- **[P2] Preserve absolutes for fallback context windows:** `src/auto-reply/reply/memory-flush.ts:40-49`
  - body: `resolveMemoryFlushContextWindowTokens` returns `DEFAULT_CONTEXT_TOKENS` when the model window cannot be resolved, but `applyMemoryFlushSharesToPlan` treats any numeric window as authoritative. Unknown or custom small-window models can therefore apply share thresholds against the 200k fallback instead of preserving the absolute plan values promised by the PR.
  - confidence: 0.86
- **[P2] Do not pass synthesized model windows to share budgets:** `src/agents/pi-embedded-runner/run/attempt.ts:912-917`
  - body: The embedded runner now passes `params.model.contextWindow` into share resolution, but fallback model resolution can synthesize that field from `DEFAULT_CONTEXT_TOKENS` when no real model metadata exists. That makes share budgets override absolutes for unknown models, contrary to the documented "contextWindowTokens is unknown" fallback behavior; the direct compaction path adds the same pattern.
  - confidence: 0.82
