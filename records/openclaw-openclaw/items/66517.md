## Best Possible Solution

## Solution Assessment

  - body: When no memory-flush plan is configured, `memoryFlushPlan` is null and the preflight threshold falls back to raw `reserveTokensFloor` or `20_000`. A config that only sets `reserveTokensFloorShare` is ignored on this path, so large-window sessions compact later than configured.
  - confidence: 0.91

That lets share thresholds override the plan's absolute values even though the PR documents fallback to absolutes when the real window is unknown.
  - confidence: 0.86

  - body: Passing `params.model.contextWindow` into share resolution can use a synthesized default from model fallback construction rather than a real model limit. Unknown or custom small-window models can compute share budgets from 200k instead of preserving absolute/default budgets; the direct compaction path needs the same guard.
  - confidence: 0.82
