## Summary

Keep open: current main still has the Tool Search descriptor-read crash, and this maintainer-labeled draft is useful, but the branch is not merge-ready because its safe fallback wrapper drops channel tool metadata and it overlaps with several open descriptor-hardening drafts.

## What This Changes

The PR changes `src/agents/tool-search.ts` and its tests to guard Tool Search catalog descriptor reads, clone parameter schemas for catalog fingerprinting, and keep malformed entries from crashing catalog registration.

## Best Possible Solution

Refresh and land one coordinated descriptor-hardening path that snapshots descriptor reads once, preserves plugin/channel/before_tool_call metadata, and handles unreadable schemas consistently across Tool Search, Code Mode, and client catalog appends.

## Reproduction Assessment

Yes. Current main directly reads Tool Search descriptor fields during catalog construction, and a throwing getter can abort registration; the PR body also reports focused after-fix Node/Vitest proof, though I did not run tests in this read-only review.

## Solution Assessment

No, not as-is. Guarding the Tool Search catalog boundary is the right layer, but the fallback replacement tool must preserve channel metadata and should be coordinated with the sibling descriptor-hardening drafts before merge.
