## Summary

Keep open. Current main and v2026.5.22 still drop the direct `/hooks/agent` request-body flag before dispatch, and the remaining decision is whether direct hook-token callers may disable external-content wrapping at all.

## What This Changes

The issue requests that `allowUnsafeExternalContent: true` on direct `POST /hooks/agent` requests be honored instead of silently producing externally wrapped agent prompts.

## Best Possible Solution

Make the direct hook trust contract explicit: either reject and document direct request-body bypasses, or add a maintainer-approved gated direct-hook bypass with security docs and regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main and v2026.5.22 shows a direct `/hooks/agent` body can include the field, but `normalizeAgentPayload()` drops it before the isolated-agent wrapping decision.

## Solution Assessment

Unclear. Propagating the flag is the narrow mechanical change for the reporter's flow, but the maintainable fix needs an explicit decision on whether direct hook-token callers may disable external-content markers.
