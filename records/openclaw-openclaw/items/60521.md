## Summary

Keep open. Current main still drops `allowUnsafeExternalContent` from direct `/hooks/agent` request bodies, but the documented hook contract and adjacent security context do not currently define a raw HTTP-body opt-out, so this needs maintainer security/product judgment rather than cleanup closure.

## What This Changes

Request direct `POST /hooks/agent` payloads to honor `allowUnsafeExternalContent: true` so webhook-dispatched agent messages arrive without external-content wrapping.

## Best Possible Solution

Make the hook trust contract explicit: document and test that direct bodies cannot opt out, or add a deliberately gated direct-hook bypass with security docs and focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: a raw request body can include the field, `normalizeAgentPayload()` drops it, and wrapping only skips when the synthetic `agentTurn` payload receives the flag.

## Solution Assessment

Unclear. The proposed one-line propagation would satisfy the reporter's flow, but it broadens the external hook trust boundary and should not be merged without an explicit maintainer security/product decision.
