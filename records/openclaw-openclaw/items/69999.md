- **[P1] Use packaged-safe bundle lookup:** `apps/macos/Sources/OpenClaw/Wallboard/WallboardWindowController.swift:57-59`

Current main's `DeviceModelCatalog` avoids this exact packaged-app failure mode by checking `Bundle.main` first because `Bundle.module` can trap when the SwiftPM resource bundle is not at the generated path, so the wallboard placeholder should use the same main-bundle-first resolver before merge.
  - confidence: 0.9
- **[P3] Skip reloading an existing wallboard controller:** `apps/macos/Sources/OpenClaw/Wallboard/WallboardManager.swift:13-16`
  - body: When `open()` reuses an existing controller, it calls `present()` and then starts `loadBundledPlaceholder()` again. That triggers a fresh WKWebView navigation and can flash the blank/black load state even though the controller is already configured; re-presenting should return without reloading or should track whether the placeholder is already loaded.
  - confidence: 0.79
