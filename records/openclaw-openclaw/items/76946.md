## Summary

Keep open: current main still has a source-backed path where detailed bootstrap truncation warning text is appended to the effective prompt and can be carried as hidden runtime context when WebChat uses a separate transcript prompt. The current docs also describe agent-visible truncation warnings as intentional, so the desired default surface needs maintainer judgment.

## What This Changes

The issue asks OpenClaw to stop detailed bootstrap truncation warnings from being injected into WebChat runtime context for oversized startup files.

## Best Possible Solution

Define one canonical truncation diagnostic surface: keep only concise agent-facing guidance if maintainers want it, move raw counts/config names to `/context`, `/status`, doctor/logs, and align WebChat/runtime-context tests plus docs.

## Reproduction Assessment

Yes, from source inspection: use a workspace with a bootstrap file over `bootstrapMaxChars`, send a WebChat direct turn where `transcriptPrompt` differs from `effectivePrompt`, and the appended truncation warning is returned as runtime context. I did not run the live WebChat path because this review is read-only.

## Solution Assessment

Unclear: the code path is narrow, but the best fix depends on whether maintainers want any truncation warning agent-visible by default or only operator/debug visibility for raw counts and config knobs.
