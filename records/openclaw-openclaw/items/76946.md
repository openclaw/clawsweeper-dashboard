## Summary

Keep open: current main still has a source-backed path where detailed bootstrap truncation warning text can be appended to the model-facing prompt and then carried as hidden WebChat runtime context, while docs currently make an agent-visible warning the default. The remaining question is the intended diagnostic surface, so this needs maintainer direction rather than cleanup closure.

## What This Changes

The issue asks OpenClaw to stop carrying detailed bootstrap truncation diagnostics, including raw counts and config keys, in routine WebChat runtime context.

## Best Possible Solution

Define one canonical truncation diagnostic surface: keep any model-facing runtime guidance concise and non-sensitive, move raw counts/config knobs to `/context`, `/status`, doctor, or logs, then align WebChat/runtime-context tests and docs.

## Reproduction Assessment

Yes, from source inspection: use a WebChat direct turn with a workspace bootstrap file over `bootstrapMaxChars`; the warning appended to `effectivePrompt` can be separated from the transcript prompt into hidden runtime context. I did not run a live WebChat reproduction in this read-only review.

## Solution Assessment

Unclear: the narrow code path is identifiable, but the best fix depends on whether maintainers want detailed truncation diagnostics model-visible by default or only on operator/debug surfaces.
