## Summary

Keep open: current main still has a source-backed path that can place the bootstrap truncation warning, including raw/injected counts and config-key hints, into hidden WebChat runtime context, but the desired default warning surface needs maintainer judgment because current docs intentionally describe this warning as agent-visible.

## What This Changes

Request to keep detailed bootstrap truncation diagnostics out of normal WebChat/runtime context or move them to an operator/debug surface.

## Best Possible Solution

Define one canonical truncation diagnostic surface: keep only concise agent recovery guidance in runtime context if needed, move detailed counts and config knobs to operator/debug surfaces, then align WebChat/runtime-context tests and docs.

## Reproduction Assessment

Yes, from source inspection: a bootstrap file over `bootstrapMaxChars` produces warning lines, `runEmbeddedAttempt` appends them to `effectivePrompt`, and WebChat's separate `transcriptPrompt` path lets the splitter carry the added warning as hidden runtime context. I did not run a live WebChat reproduction because this review was read-only.

## Solution Assessment

Unclear: the narrow path is identifiable, but the best fix depends on whether maintainers want any default agent-visible truncation warning or only operator/debug visibility for raw counts and config knobs.
