## Summary

Keep open: current main still has a source-backed path where bootstrap truncation warning text can be separated from the visible WebChat prompt and carried as hidden runtime context, and the right default warning surface needs maintainer judgment.

## What This Changes

The issue asks OpenClaw to stop putting bootstrap truncation warning details into normal WebChat runtime context when startup files exceed bootstrap limits.

## Best Possible Solution

Define one canonical truncation diagnostic surface: keep only concise, non-sensitive agent guidance in runtime context or move detailed counts/config names to `/context`, `/status`, and logs, then align WebChat/runtime-context tests and docs.

## Reproduction Assessment

Yes, from source inspection: use a workspace with a bootstrap file over `bootstrapMaxChars`, send a WebChat `chat.send` turn, and the warning appended to `effectivePrompt` can be separated from `transcriptPrompt` into hidden runtime context. I did not run a live WebChat reproduction because this review is read-only.

## Solution Assessment

Unclear: the narrow code path is identifiable, but the best fix depends on whether maintainers want any agent-visible truncation warning by default or only operator/debug visibility for raw counts and config knobs.
