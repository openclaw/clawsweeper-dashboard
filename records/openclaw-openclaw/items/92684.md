## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/92684. Current `openclaw doctor` empty-allowlist scanning treats `channels.<channel>` as a standalone account before scanning `channels.<channel>.accounts.*`, so a parent fallback record with empty `groupAllowFrom` can warn that group messages will be dropped even when every enabled account has populated `groupAllowFrom` or `allowFrom`. Adjust the scanner/warning logic in `src/commands/doctor/shared/empty-allowlist-scan.ts` and related helpers so parent channel config is still used as fallback for account records but is not reported as a phantom account when account-scoped configs cover the active accounts. Add focused regression tests for at least one sender-based channel with `groupPolicy: "allowlist"`, empty top-level `groupAllowFrom`, and populated per-account lists; preserve existing warnings for true single-account/top-level configs and for accounts that actually lack effective allowlists. Do not add config options, change runtime channel authorization semantics, touch changelog, or move plugin/API boundaries.
