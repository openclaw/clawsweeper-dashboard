## Summary

Keep open: current main and the latest release still have Telegram inline buttons and exec-approval callbacks, but pairing approval remains command-only and needs a security/product contract before implementation.

## What This Changes

Request to let Telegram pairing notifications render approve/deny inline buttons that resolve pairing requests without copying CLI commands.

## Best Possible Solution

Define a typed, replay-safe pairing approval action contract, then render it as Telegram inline buttons while existing pairing stores and approval-time scope checks remain authoritative.

## Reproduction Assessment

Yes, source-level: current main and v2026.6.10 emit pairing approval instructions as text commands, and the Telegram callback handler has no pairing approve/deny branch.

## Solution Assessment

Unclear: inline buttons are plausible, but the best fix needs a typed approval/callback contract with explicit auth, expiry, replay, denial, and scope semantics instead of raw command-string callbacks.
