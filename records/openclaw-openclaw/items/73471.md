## Summary

Keep open. Current main still treats valid historical primary `.jsonl` transcripts as orphan candidates when they are absent from the current `sessions.json` reference set; the newer interactive-confirmation and archive-search mitigations do not fix the underlying session-history classification problem.

## What This Changes

The issue asks doctor/session maintenance to stop archiving valid historical session transcripts as orphans and to harden recovery, diagnostics, and doctor recommendation testing.

## Best Possible Solution

Define a shared semantic session-artifact classifier used by doctor and cleanup that preserves valid historical primary transcripts, then split restore UX, corrupt-store diagnostics, and doctor-recommendation CI hardening into focused follow-ups.

## Reproduction Assessment

Yes, at source level. A sessions directory with `sessions.json` pointing at current `A.jsonl` and a valid older `B.jsonl` will still flag `B.jsonl` because current main only excludes paths present in `referencedTranscriptPaths`.

## Solution Assessment

No, current main is not the best final solution. Interactive confirmation and archived-transcript indexing reduce blast radius, but the maintainable fix needs semantic classification and preservation tests, with recovery and CI hardening scoped deliberately.
