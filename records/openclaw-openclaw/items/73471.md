## Summary

Keep open. Current main fixed the worst non-interactive archive path, but the core orphan definition still treats valid historical primary transcripts as removable when they are absent from the current sessions.json pointer map, and the same assumption exists in disk-budget cleanup.

## What This Changes

The issue asks OpenClaw to stop treating historical primary session transcripts as orphaned state, improve recovery/wording, and harden state-mutating doctor recommendations.

## Best Possible Solution

Define a shared semantic transcript-retention rule for doctor and disk-budget cleanup that preserves valid historical primary transcripts, then add focused recovery/diagnostic and CI-audit follow-ups rather than treating every subrequest as one patch.

## Reproduction Assessment

Yes. A source-level reproduction is a sessions directory where sessions.json references current A.jsonl and the directory also contains valid older B.jsonl; current main flags B.jsonl because it is not in referencedTranscriptPaths.

## Solution Assessment

No. The current implementation is not the final fix because interactive confirmation reduces accidental damage but still relies on the wrong orphan definition; the safer path is a semantic classifier with preservation tests shared across doctor and cleanup.
