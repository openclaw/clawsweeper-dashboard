## Summary

Keep open. Current main mitigates some fallout by requiring interactive confirmation and indexing deleted/reset transcripts, but doctor and disk-budget cleanup still classify valid historical primary transcripts as removable solely because they are absent from the current `sessions.json` reference set.

## What This Changes

The issue asks OpenClaw to stop treating historical primary session transcripts as doctor orphans and to harden `openclaw doctor` repair recommendations with semantic safety tests.

## Best Possible Solution

Define a shared semantic orphan classifier for doctor and disk-budget cleanup that preserves valid historical primary transcripts, then split restore UX, corrupt-store diagnostics, and CI recommendation auditing into focused follow-ups.

## Reproduction Assessment

Yes. A source-level reproduction is a sessions directory where `sessions.json` references current `A.jsonl` and the same directory contains valid older `B.jsonl`; current main still flags `B.jsonl` because it is not in `referencedTranscriptPaths`.

## Solution Assessment

No, current main is not a complete solution. Interactive confirmation and deleted-transcript indexing reduce damage, but the maintainable fix is semantic classification plus preservation tests across doctor and disk-budget cleanup.
