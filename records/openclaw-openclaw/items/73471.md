## Summary

Keep open. Current main still classifies plain historical primary `.jsonl` transcripts as orphan candidates whenever they are absent from the current `sessions.json` reference set; interactive confirmation and archive search visibility are mitigations, not a fix for the underlying doctor/session-history semantics.

## What This Changes

The issue reports that `openclaw doctor --fix` can treat valid historical primary session transcripts as orphan files and asks for safer classification, recovery, diagnostics, and CI hardening around doctor recommendations.

## Best Possible Solution

Define a shared semantic session-artifact classifier used by doctor and cleanup that preserves valid historical primary transcripts, then split restore UX, corrupt-store diagnostics, and doctor-recommendation CI hardening into focused follow-ups.

## Reproduction Assessment

Yes, at source level. A sessions directory with `sessions.json` pointing at current `A.jsonl` and a valid older `B.jsonl` will still flag `B.jsonl` because current main only excludes paths present in `referencedTranscriptPaths`.

## Solution Assessment

No, current main is not the best final solution. Interactive confirmation and archive indexing reduce blast radius, but the maintainable fix needs semantic classification and preservation tests, with recovery and CI hardening scoped deliberately.
