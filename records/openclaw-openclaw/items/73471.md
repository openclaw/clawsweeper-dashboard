## Summary

Keep open. Current main still defines orphan primary transcripts by absence from the active `sessions.json` map, so valid older bare `.jsonl` transcript files can still be offered for `.deleted.<timestamp>` archival; newer interactive-confirmation, QMD-suppression, and deleted-transcript discovery mitigations reduce impact but do not fix the classifier described in the report.

## What This Changes

The issue requests that doctor stop treating valid historical primary session transcripts as orphan files and proposes broader hardening for doctor repair recommendations.

## Best Possible Solution

Define a real orphan transcript in the doctor state-integrity path instead of using absence from the active session-key map. The first maintainable fix should preserve valid historical primary transcripts and add regression coverage; prompt wording, restore support, corrupt-store diagnostics, and doctor recommendation CI hardening should be follow-up work with explicit scope.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is clear: create a sessions directory with one current `sessions.json` entry plus an older valid `<sessionId>.jsonl` absent from the store; current `noteStateIntegrity` will classify the older file as orphan because it is missing from `referencedTranscriptPaths`.

## Solution Assessment

No. The current interactive confirmation and archive-discovery mitigations are safer than silent repair, but they do not solve the definition bug; the narrow durable solution is to change the classifier and tests first.
