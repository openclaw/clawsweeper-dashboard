## Summary

Keep open. Current main mitigates the worst silent path by requiring interactive confirmation and by letting memory indexing include `.deleted` transcripts, but the core doctor classifier still defines orphan primary transcripts as ordinary `.jsonl` files absent from the current `sessions.json` entries. The session docs support the report's premise that `sessions.json` points at current sessions, not all historical chat transcripts, and the focused doctor tests still cover archive mechanics rather than preservation of valid older primary transcripts.

## What This Changes

The issue reports that `openclaw doctor --fix` can classify valid historical primary session transcripts as orphan candidates and asks for safer transcript classification plus broader hardening around doctor recommendations.

## Best Possible Solution

Define a real orphan transcript contract in the doctor state-integrity path and preserve valid historical primary transcripts even when they are not the current transcript for any session key. Add focused regression coverage for that preservation first, then split restore support, corrupt-store diagnostics, and doctor recommendation CI into separately scoped follow-ups.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is clear: create a sessions directory with one current `sessions.json` entry plus an older valid `<sessionId>.jsonl` absent from the store; current `noteStateIntegrity` will include the older file in `orphanTranscriptPaths` because it is not in `referencedTranscriptPaths`.

## Solution Assessment

No. The current interactive-confirmation mitigation is safer than blind `--fix`, and archived files are now more discoverable, but it is not the best final fix because valid historical primary transcripts can still be offered for tombstoning under the same false definition.
