## Summary

Keep open. Current main still defines orphan primary transcripts by absence from the active `sessions.json` map, so valid older bare `.jsonl` chat transcripts can still be offered for `.deleted.<timestamp>` archival; recent mitigations reduce the blast radius but do not resolve the classifier described here.

## What This Changes

The issue asks OpenClaw to stop `doctor --fix` from archiving valid historical session transcripts as orphan files and to harden doctor repair recommendations around user-content state.

## Best Possible Solution

Define a real orphan transcript in the doctor state-integrity path instead of using absence from the current session-key map. The first maintainable fix should preserve valid historical primary transcripts and add regression coverage; prompt wording, restore support, corrupt-store diagnostics, and doctor recommendation CI hardening should be scoped as explicit follow-ups.

## Reproduction Assessment

Yes. Create a sessions directory with a current `sessions.json` entry plus an older valid `<sessionId>.jsonl` absent from the store; current `noteStateIntegrity` will classify the older file as an orphan because it is missing from `referencedTranscriptPaths`.

## Solution Assessment

No. Current interactive-confirmation, QMD-suppression, and archive-discovery mitigations reduce harm, but they do not solve the definition bug; the safer first fix is to change the classifier and tests so valid historical transcripts are preserved.
