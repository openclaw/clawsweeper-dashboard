## Summary

Keep open. Current main has important mitigations, but it still treats ordinary historical primary transcripts as orphan/removable solely because they are absent from the current `sessions.json` reference set, so the central data-retention bug is not fixed.

## What This Changes

The issue asks OpenClaw to stop treating valid historical primary session transcripts as orphan files during doctor repair, improve recovery/diagnostics, and harden doctor recommendations in CI.

## Best Possible Solution

Define a shared semantic session-artifact classifier used by doctor and cleanup that preserves valid historical primary transcripts, then split restore UX, corrupt-store diagnostics, and doctor-recommendation CI hardening into focused follow-ups.

## Reproduction Assessment

Yes, at source level. A sessions directory with `sessions.json` pointing at current `A.jsonl` and a valid older `B.jsonl` will still flag `B.jsonl`, because current main only excludes files present in `referencedTranscriptPaths`.

## Solution Assessment

No, current main is not the final best solution. Interactive confirmation and archive indexing are useful mitigations, but the maintainable fix needs semantic classification and preservation tests, with recovery and CI hardening scoped deliberately.
