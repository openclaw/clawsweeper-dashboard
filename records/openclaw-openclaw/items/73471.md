## Summary

Keep open. Current main mitigates the silent/non-interactive archive damage, but the doctor and disk-budget cleanup paths still classify ordinary historical primary transcripts as orphan/removable solely because they are absent from the current `sessions.json` map.

## What This Changes

The issue asks doctor cleanup to stop treating valid historical primary transcripts as orphan archives and to harden the broader `openclaw doctor` recommendation surface.

## Best Possible Solution

Define semantic orphan classification shared by doctor and disk-budget cleanup, preserve valid historical primary transcripts even when they are not current in `sessions.json`, and split restore, corrupt-store diagnostics, and CI audit work into focused follow-ups.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a sessions directory where `sessions.json` references current `A.jsonl` and the same directory also contains valid older `B.jsonl`; current main flags `B.jsonl` because it is not in `referencedTranscriptPaths`.

## Solution Assessment

No for current main as the final solution. Interactive confirmation and `.deleted.*` discovery reduce damage, but the maintainable fix is semantic classification with preservation regression tests and matching disk-budget behavior.
