## Summary

Keep open. Current main has mitigations for the worst silent archive path, but the central classifier still treats ordinary historical primary transcript files as orphan candidates solely because they are absent from the current sessions.json map.

## What This Changes

The issue requests safer doctor orphan-transcript handling so valid historical session transcripts are not archived or hidden as orphans, plus follow-up hardening for doctor repair recommendations.

## Best Possible Solution

Define a real orphan transcript in the session state-integrity and disk-budget paths, preserve valid historical primary transcripts even when they are not the current session for any session key, and add regression coverage for that preservation before splitting restore, diagnostics, and doctor-recommendation CI into narrower follow-ups.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: create a sessions directory with sessions.json referencing current A.jsonl plus a valid older B.jsonl absent from the store; current noteStateIntegrity will flag B as an orphan because it is not in referencedTranscriptPaths.

## Solution Assessment

No. The current interactive-confirmation and deleted-transcript discovery mitigations reduce damage, but they are not the best final solution because the false orphan definition remains; the safer fix is semantic classification plus focused preservation tests.
