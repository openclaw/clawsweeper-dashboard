## Summary

Keep open. Current main mitigates the worst silent `--fix` path with an interactive confirmation and indexed `.deleted.*` transcripts, but doctor and disk-budget cleanup still treat ordinary historical primary transcripts as orphan/removable solely because they are absent from the current `sessions.json` map.

## What This Changes

The issue asks OpenClaw to stop `doctor --fix` from archiving valid historical session transcripts as orphan files and to harden the broader doctor-recommendation repair surface.

## Best Possible Solution

Add shared semantic orphan classification for doctor and disk-budget cleanup that preserves valid historical primary transcripts even when they are not the current `sessions.json` entry, with focused regression tests and separate follow-ups for restore, parse diagnostics, and CI audit work.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a sessions directory where `sessions.json` references current `A.jsonl` and a valid older `B.jsonl` remains on disk; current main flags `B.jsonl` because it is absent from `referencedTranscriptPaths`.

## Solution Assessment

No for current main as the final solution. Interactive confirmation and deleted-transcript discovery reduce damage, but the narrow maintainable fix is to stop using the current-session map as an all-history registry for primary transcript cleanup.
