## Summary

Keep open. Current main mitigates the original memory-search disappearance by indexing deleted/reset archives and making doctor archive confirmation interactive, but doctor and disk-budget cleanup still classify valid historical primary transcripts as orphan/removable solely because they are absent from the current `sessions.json` reference set.

## What This Changes

The issue asks OpenClaw to stop `doctor --fix` from treating valid historical session transcripts as orphan archives and to harden doctor recommendations with semantic tests.

## Best Possible Solution

Define a shared semantic session-artifact classifier used by doctor and disk-budget cleanup that preserves valid historical primary transcripts, then split restore UX, corrupt-store diagnostics, and doctor-recommendation CI hardening into focused follow-ups.

## Reproduction Assessment

Yes, at source level. A sessions directory with `sessions.json` pointing at current `A.jsonl` and a valid older `B.jsonl` will still flag `B.jsonl` because current main only checks whether the file is in the current reference set.

## Solution Assessment

No, current main is not the final best solution. Interactive confirmation and archive indexing are useful mitigations, but the maintainable fix needs semantic classification and preservation tests, with restore/diagnostics/CI policy scoped separately.
