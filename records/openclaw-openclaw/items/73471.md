## Summary

Keep open. Current main still uses the current `sessions.json` reference set as the orphan boundary, so valid older primary transcripts can still be treated as orphan candidates; the existing interactive-confirmation and memory-indexing mitigations reduce damage but do not resolve the underlying classifier semantics.

## What This Changes

The issue requests that `openclaw doctor --fix` stop archiving valid historical session transcripts as orphan files and proposes restore, diagnostics, and CI hardening around doctor repair recommendations.

## Best Possible Solution

Define a semantic orphan-transcript classifier for doctor and cleanup paths that preserves valid historical primary transcripts even when they are not the current session for any key, add regression coverage for that preservation, and split restore tooling, corrupt-store warnings, and doctor-recommendation CI into narrower follow-ups.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: create a sessions directory with `sessions.json` referencing current `A.jsonl` plus a valid older `B.jsonl` absent from the store; current `noteStateIntegrity` will flag `B.jsonl` because it is absent from `referencedTranscriptPaths`.

## Solution Assessment

No. The current interactive-confirmation and deleted-transcript discovery mitigations reduce silent damage, but they are not the best final fix because the false orphan definition remains; semantic classification plus preservation tests is the safer path.
