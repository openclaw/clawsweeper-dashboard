## Summary

Keep open. Current main has mitigations for the worst silent-archive path, but the central classifier still treats ordinary primary `.jsonl` transcripts as orphan candidates solely because they are absent from the current `sessions.json` map; docs and tests support that this map tracks current sessions, not all historical chat history.

## What This Changes

The issue requests that `openclaw doctor --fix` stop archiving valid historical session transcripts as orphan files, and proposes broader recovery, diagnostics, and CI hardening for doctor repair recommendations.

## Best Possible Solution

Define a semantic orphan-transcript classifier for doctor and cleanup paths that preserves valid historical primary transcripts even when they are not the current session for any key, add regression coverage for that preservation, and split restore tooling, corrupt-store warnings, disk-budget policy, and doctor-recommendation CI into narrower follow-ups.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: create `sessions.json` with a current `A.jsonl` entry plus a valid older `B.jsonl` in the same sessions directory; current `noteStateIntegrity` will flag `B.jsonl` because it is absent from `referencedTranscriptPaths`.

## Solution Assessment

No. The current interactive-confirmation and deleted-transcript discovery mitigations reduce silent damage, but they are not the best final fix because the false orphan definition remains; the safer solution is semantic classification plus preservation tests.
