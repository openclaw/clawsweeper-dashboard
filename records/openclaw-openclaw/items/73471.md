## Summary

Keep open. Current main and the latest release still classify any primary `.jsonl` transcript not referenced by `sessions.json` as an orphan and offer to rename it, while shipped archive indexing only mitigates memory-search visibility after the rename rather than fixing the doctor/session-history semantics.

## What This Changes

The issue reports that `openclaw doctor --fix` treats valid historical session transcripts as orphan files and asks for safer classification, recovery, and doctor-recommendation hardening.

## Best Possible Solution

Define a shared semantic transcript-state model that preserves valid historical primary transcripts, then handle restore UX, corrupt-store diagnostics, cleanup pruning, and doctor-recommendation CI hardening as scoped follow-ups.

## Reproduction Assessment

Yes, at source level. A sessions directory with `sessions.json` pointing at current `A.jsonl` and a valid older `B.jsonl` will still flag `B.jsonl` because current main only excludes paths present in `referencedTranscriptPaths`.

## Solution Assessment

No, not as one automated sweep. The narrow maintainable path is semantic transcript classification and preservation tests first, with restore UX and doctor-recommendation CI hardening deliberately scoped by maintainers.
