## Summary

Keep open. Current main and v2026.6.8 still classify ordinary historical primary `.jsonl` transcripts as orphan candidates when they are not the current `sessions.json` pointer; current mitigations only require interactive confirmation and improve some archive/search visibility, not the underlying classification.

## What This Changes

The issue requests safer doctor/session integrity behavior so valid historical transcripts are not archived as orphan files, plus recovery and CI hardening around doctor repair recommendations.

## Best Possible Solution

Define one semantic transcript-state model that preserves valid historical primary transcripts by default, then make orphan archive, restore, and doctor-recommendation audit behavior explicit and tested.

## Reproduction Assessment

Yes, at source level. A sessions directory with `sessions.json` pointing at current transcript `A.jsonl` and a valid older `B.jsonl` will still flag `B.jsonl` because current main only excludes paths in `referencedTranscriptPaths`.

## Solution Assessment

No, not as one automated sweep. The maintainable path is to fix semantic transcript classification and preservation tests first, then scope restore UX, corrupt-store diagnostics, and doctor-recommendation CI hardening with maintainer input.
