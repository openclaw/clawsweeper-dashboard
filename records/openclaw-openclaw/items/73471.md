## Summary

Keep open. Current main still treats ordinary historical primary transcript files as orphan/removable when they are absent from the current `sessions.json` pointers, so the central doctor safety bug remains unresolved even though archived transcript discovery now mitigates part of the memory-search loss.

## What This Changes

The issue asks doctor and related cleanup paths to stop tombstoning valid historical session transcripts as orphan files, improve warning/recovery behavior, and add CI coverage for doctor recommendations.

## Best Possible Solution

Define a shared semantic session-artifact classifier used by doctor and disk-budget cleanup that preserves valid historical primary transcripts, then track restore UX, corrupt-store diagnostics, and doctor-recommendation CI hardening as focused follow-ups.

## Reproduction Assessment

Yes, at source level. A sessions directory where `sessions.json` points at current `A.jsonl` while valid older `B.jsonl` remains on disk will still flag `B.jsonl`, because current main only checks membership in `referencedTranscriptPaths`; no live filesystem reproduction was run in this read-only review.

## Solution Assessment

No, current main is not the final best solution. Archive discovery and interactive confirmation are useful mitigations, but the narrow maintainable fix is semantic transcript classification plus preservation tests, with restore and CI hardening scoped separately.
