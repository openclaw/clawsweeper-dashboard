## Summary

Keep open. Current main still builds orphan transcript candidates by comparing primary session JSONL files against only the active `sessions.json` entries, so valid older transcripts can still be offered for archiving even though the docs define transcripts as the conversation-history layer.

## What This Changes

The issue asks OpenClaw to stop `doctor --fix` from classifying valid historical session transcripts as orphan files, improve archive/recovery and corrupt-store behavior, and harden doctor recommendations in CI.

## Best Possible Solution

Define a real orphan transcript in the doctor state-integrity path instead of using absence from the active session-key map. A focused first change should preserve valid historical primary transcripts with regression coverage; prompt wording, restore support, corrupt-store warnings, and doctor recommendation CI hardening should be split into scoped follow-ups.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is clear: create a sessions directory with one active `sessions.json` entry and an older valid `<sessionId>.jsonl` absent from the store, and current main will classify that older transcript as orphan because it is missing from `referencedTranscriptPaths`.

## Solution Assessment

No. The current interactive-confirmation and archived-discovery mitigations reduce damage, but they do not solve the definition bug; the narrow maintainable fix is to change the classifier and tests first, then handle UX and CI hardening separately.
