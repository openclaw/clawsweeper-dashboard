## Summary

Keep open. Current main reduces the original memory-search loss by indexing deleted/reset archives and requiring interactive confirmation, but doctor and disk-budget cleanup still classify valid historical primary transcripts as orphan/removable solely because they are absent from the current `sessions.json` reference set.

## What This Changes

The issue reports that `openclaw doctor --fix` can archive valid historical primary session transcripts as orphans and asks for safer classification, recovery, diagnostics, and CI coverage for doctor recommendations.

## Best Possible Solution

Define a shared semantic session-artifact classifier used by doctor and disk-budget cleanup that preserves valid historical primary transcripts, then split restore UX, corrupt-store diagnostics, and doctor-recommendation CI hardening into focused follow-ups.

## Reproduction Assessment

Yes, at source level. A sessions directory with `sessions.json` pointing at current `A.jsonl` and a valid older `B.jsonl` will still flag `B.jsonl` because current main only checks whether the file is in `referencedTranscriptPaths`; no live filesystem reproduction was run because this review is read-only.

## Solution Assessment

No, current main is not the final best solution. Interactive confirmation and archive indexing are useful mitigations, but the narrow maintainable fix is semantic classification plus preservation tests, with restore/diagnostics/CI policy scoped separately.
