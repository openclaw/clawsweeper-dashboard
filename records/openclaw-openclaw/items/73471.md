## Summary

Keep open. Current main and v2026.6.8 still classify ordinary historical primary `.jsonl` transcripts as orphan candidates when they are not the current `sessions.json` pointer, and the open visibility PR does not change doctor’s classification or archive prompt.

## What This Changes

The issue asks OpenClaw to stop `doctor --fix` from treating valid historical session transcripts as disposable orphans, add safer recovery semantics, and harden doctor recommendations.

## Best Possible Solution

Define a shared semantic transcript-state model that preserves valid historical primary transcripts, then scope restore UX and doctor-recommendation CI hardening as explicit follow-up work.

## Reproduction Assessment

Yes, at source level. A sessions directory with current `sessions.json` pointing at `A.jsonl` and a valid older `B.jsonl` will still flag `B.jsonl` because current main only excludes paths in `referencedTranscriptPaths`.

## Solution Assessment

No, not as one automated sweep. The narrow maintainable path is semantic transcript classification and preservation tests first, with restore UX and doctor-recommendation CI hardening scoped by maintainers.
