## Summary

Keep open. Current main still treats ordinary historical primary transcripts as orphan/removable when they are absent from the current `sessions.json` reference set; archive indexing now mitigates memory-search loss for `.deleted.*` files, but the underlying doctor and cleanup classification remains unsafe.

## What This Changes

The issue asks OpenClaw to stop `doctor --fix` from archiving valid historical session transcripts as orphans and to harden doctor repair recommendations with safer diagnostics, recovery, and CI coverage.

## Best Possible Solution

Define a shared semantic session-artifact classifier used by doctor and cleanup that preserves valid historical primary transcripts, then split restore UX, corrupt-store diagnostics, and doctor-recommendation CI hardening into focused follow-ups.

## Reproduction Assessment

Yes, at source level. A sessions directory with `sessions.json` pointing at current `A.jsonl` and a valid older `B.jsonl` will still flag `B.jsonl`, because current main only excludes paths present in `referencedTranscriptPaths`.

## Solution Assessment

No, current main is not the final best solution. Interactive confirmation and archive indexing are useful mitigations, but the maintainable fix needs semantic classification and preservation tests, with recovery and CI hardening scoped deliberately.
