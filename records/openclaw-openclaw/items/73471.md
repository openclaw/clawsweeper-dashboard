## Summary

Keep open. Current main mitigates the worst silent-loss path with interactive confirmation and deleted-transcript indexing, but doctor and disk-budget cleanup still treat valid historical primary transcripts as orphan/removable solely because they are absent from the current `sessions.json` reference set.

## What This Changes

The issue asks OpenClaw to stop `doctor --fix` from archiving valid historical session transcripts as orphans and to harden doctor-recommendation repair paths.

## Best Possible Solution

Define a shared semantic classifier for orphan/removable session artifacts that preserves valid historical primary transcripts, then add focused tests for doctor and disk-budget cleanup before separately scoping restore and CI-audit follow-ups.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a sessions directory where `sessions.json` references current `A.jsonl` and the same directory contains valid older `B.jsonl`; current main flags `B.jsonl` because it is not in `referencedTranscriptPaths`.

## Solution Assessment

No for current main as a final solution. Interactive confirmation and deleted-archive indexing reduce harm, but the narrow maintainable fix is semantic classification plus preservation tests across both doctor and disk-budget cleanup.
