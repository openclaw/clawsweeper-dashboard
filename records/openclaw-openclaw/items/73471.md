## Summary

Keep open. Current main and the latest release still classify ordinary historical primary JSONL transcripts as orphan candidates when they are not the current sessions.json pointer; related work only mitigates parts of memory visibility or native cleanup design, not the doctor classifier or archive prompt.

## What This Changes

The issue reports that doctor treats historical session transcript JSONL files as orphan archives, hiding chat history, and asks for safer doctor repair hardening.

## Best Possible Solution

Define the transcript-state contract first, then preserve valid historical primary transcripts by default while separately designing explicit restore/archive and doctor-recommendation audit behavior.

## Reproduction Assessment

Yes, at source level. A sessions directory with sessions.json pointing at current transcript A.jsonl and a valid older B.jsonl will still flag B.jsonl because current main only excludes paths in referencedTranscriptPaths.

## Solution Assessment

No, not as one automated sweep. The maintainable path is to fix semantic transcript classification and preservation tests first, then scope restore UX, corrupt-store diagnostics, and doctor-recommendation CI hardening with maintainer input.
