## Summary

Keep open. Current main has mitigations for silent `--fix` archival, but the doctor and disk-budget paths still classify ordinary historical primary transcripts as orphan/removable solely because they are absent from the current `sessions.json` map.

## What This Changes

Request safer doctor/session maintenance handling so valid historical primary transcripts are not treated as orphan archives, plus clearer recovery, diagnostics, and CI coverage for doctor recommendations.

## Best Possible Solution

Define semantic orphan classification for doctor and disk-budget cleanup, preserve valid historical primary transcripts even when they are not current in `sessions.json`, and split restore, parse diagnostics, and CI-audit work into focused follow-ups.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a sessions directory with `sessions.json` referencing current `A.jsonl` plus a valid older `B.jsonl`; current `noteStateIntegrity` flags `B.jsonl` because it is not in `referencedTranscriptPaths`.

## Solution Assessment

No for current main as the final solution. Interactive confirmation and archive discovery reduce damage, but the safer maintainable fix is a semantic classifier with preservation regression tests and matching disk-budget behavior.
