## Summary

Keep open. Current main has mitigations for the worst silent path, but the core classifier still treats valid older primary transcript files as orphan candidates solely because they are absent from the active sessions.json map, so the reported user-content preservation bug remains.

## What This Changes

The issue requests that doctor stop classifying valid historical primary session transcripts as orphan files, improve the archive prompt and recovery path, make corrupt session-store reads visible, and harden doctor recommendation coverage.

## Best Possible Solution

Define a real orphan transcript in the doctor state-integrity path and preserve valid historical primary transcripts even when they are not the current session for any session key. Add regression coverage for older valid transcripts, then handle prompt wording, restore support, corrupt-store diagnostics, and doctor recommendation CI as separate scoped follow-ups.

## Reproduction Assessment

Yes. A source-level reproduction is clear: create a sessions directory with one current sessions.json entry plus an older valid <sessionId>.jsonl that is absent from the store; current noteStateIntegrity will include the older file in orphanTranscriptPaths because it is not in referencedTranscriptPaths.

## Solution Assessment

No. The current interactive-confirmation mitigation is safer than silent --fix behavior, but it is not the best final fix because it leaves the false orphan definition intact; the safer narrow solution is to change classification semantics and test preservation first.
