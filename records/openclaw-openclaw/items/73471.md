## Summary

Keep open. Current main mitigates silent auto-archival and includes deleted/reset transcripts in memory indexing, but doctor and disk-budget cleanup still treat valid historical primary transcripts as orphan/removable solely because they are absent from the current `sessions.json` reference set.

## What This Changes

The issue reports unsafe doctor/session cleanup semantics for historical primary session transcripts and asks for safer classification, recovery, diagnostics, and CI coverage.

## Best Possible Solution

Define a shared semantic orphan classifier for doctor and disk-budget cleanup that preserves valid historical primary transcripts, then split restore UX, corrupt-store diagnostics, and doctor-recommendation CI hardening into focused follow-ups.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a sessions directory where `sessions.json` references current `A.jsonl` and the same directory contains valid older `B.jsonl`; current main flags `B.jsonl` because it is not in `referencedTranscriptPaths`.

## Solution Assessment

No for current main as a final solution. Interactive confirmation and deleted-transcript memory indexing reduce damage, but the maintainable fix is semantic classification plus preservation tests across doctor and disk-budget cleanup.
