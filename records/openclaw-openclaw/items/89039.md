## Summary

Keep open: the latest head fixes several real session-takeover paths, but it still has a blocking compatibility regression because the embedded retry-default wrapper injects `maxRetries: 0` before the session SDK can apply persisted provider retry settings.

## What This Changes

The PR wraps embedded steering writes in owned transcript writes, adds takeover-specific run failure handling, defaults embedded prompt SDK retries to zero, forwards `maxRetries` through native OpenAI transports, and adds regression/proof tests.

## Best Possible Solution

Preserve configured provider retry settings while only defaulting embedded prompt submissions to `maxRetries: 0` when neither the caller nor persisted provider settings supply a value, with a regression that drives the session SDK path through native OpenAI request options.
