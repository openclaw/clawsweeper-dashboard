## Label Justifications

- `P1`: The PR targets a live embedded agent/channel workflow where accepted user input can end in lost or generic failure output after session takeover.
- `merge-risk: 🚨 compatibility`: Existing setups with no configured provider retry value lose the OpenAI SDK's default embedded prompt-window retries in favor of OpenClaw-owned retry behavior.
- `merge-risk: 🚨 session-state`: The diff changes which queued transcript writes are trusted by the session fence while the prompt lock is released.
- `merge-risk: 🚨 availability`: Moving retry ownership out of the SDK in this window could make flaky provider calls fail sooner unless the surrounding fallback path covers the failure as intended.

## Summary

Keep open: this focused bug-fix PR is still useful and is not implemented on current main, but landing it needs maintainer acceptance of the retry and session-state compatibility tradeoffs.

## What This Changes

The PR wraps embedded steering writes as owned transcript writes, adds visible resend handling for embedded session takeover errors, scopes and forwards OpenAI SDK retry options, and adds regression proof/tests.

## Best Possible Solution

Have runtime/session owners explicitly accept the scoped retry and owned-steering contract, then land this PR or an equivalent maintainer patch with the same regression proof.

## Solution Assessment
