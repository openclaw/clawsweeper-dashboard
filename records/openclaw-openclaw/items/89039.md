## What This Changes

The PR wraps embedded steering writes as owned transcript writes, adds visible resend handling for embedded session takeover errors, scopes and forwards OpenAI SDK `maxRetries`, and adds regression proof/tests.

## Best Possible Solution

Land this PR only after the session/runtime owner accepts the scoped retry and owned-write boundary; then use it as the canonical fix path for the linked bug and related duplicates.

## Reproduction Assessment

Yes by source path and focused proof: current main can throw `EmbeddedAttemptSessionTakeoverError` on prompt-lock reacquire after an unowned session-file change, and the PR adds a real-fence/local-HTTP regression for that path. I did not independently run a live upstream ECONNRESET reproduction.

## Solution Assessment

Yes as an acceptable focused mitigation, not a full recovery design. Wrapping owned steering, forwarding scoped `maxRetries`, and surfacing resend guidance address the observed chain while avoiding riskier automatic replay/requeue semantics.
