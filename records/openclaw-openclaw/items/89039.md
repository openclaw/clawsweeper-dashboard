## Summary

Keep open for maintainer review. The patch addresses a real source-reproducible message-loss path and I found no concrete line-level defect, but it intentionally moves unconfigured embedded prompt calls away from the OpenAI SDK retry default and into OpenClaw-owned fallback/resend handling, so the compatibility and session-state policy needs maintainer acceptance before merge.

## What This Changes

The PR wraps embedded steering writes in owned session transcript writes, adds takeover-specific resend handling, scopes provider retry defaults during prompt-lock release, forwards maxRetries through OpenAI transports, and adds regression coverage.

## Best Possible Solution

Land a maintainer-approved version that keeps steering writes inside the owned transcript fence, forwards provider retry settings, and surfaces resend guidance while explicitly accepting the embedded retry-default compatibility tradeoff.
