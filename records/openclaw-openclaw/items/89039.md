## Summary

This PR is still necessary: current main still lets embedded steering writes bypass owned transcript publication, does not special-case pure EmbeddedAttemptSessionTakeoverError in the reply runner, and does not forward maxRetries through the native OpenAI transport. The patch looks like a plausible and internally tested fix, but it changes a session/message-delivery hot path and the unset embedded-provider retry default, so it should stay open for maintainer review and exact-head checks rather than be swept closed.

## What This Changes

The PR wraps embedded steering writes in owned transcript publication, adds user-facing resend handling for pure session-takeover errors, scopes embedded prompt-lock SDK retry defaults, forwards maxRetries through native OpenAI transports, and adds regression coverage.

## Best Possible Solution

Land a maintainer-approved version that preserves explicit retry settings, prevents owned in-process steering writes from tripping the prompt fence, and surfaces actionable resend guidance without broadening runtime fallback shims.
