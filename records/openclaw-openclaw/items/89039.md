## What This Changes

The PR wraps embedded steering writes in owned transcript-write context, adds session-takeover phase/error handling, and injects `maxRetries: 0` during embedded prompt streaming with regression tests.

## Best Possible Solution

Land a version that keeps owned steering writes and visible takeover handling while preserving configured provider retry behavior, with redacted real-run proof for the message-loss path.

## Reproduction Assessment

No live high-confidence reproduction was established in this review. Source inspection supports the race path on current main: embedded steering writes can advance the session file while the prompt lock is released, and the resulting takeover error is rethrown out of model fallback.

## Solution Assessment

No, not as submitted. The owned-write and visible-error pieces are plausible, but the retry defaulting is not the best fix because it shadows configured provider retries; preserve existing retry configuration before disabling SDK retries by default.
