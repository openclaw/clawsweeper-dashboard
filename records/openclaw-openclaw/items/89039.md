Full review comments:

- **[P1] Keep retry defaults fresh across installs:** `src/agents/embedded-agent-runner/run/attempt.session-lock.ts:1326-1327`
  - body: `installEmbeddedPromptRetryDefault` captures `defaultMaxRetries` in the wrapper and then returns on later installs. Since `runEmbeddedAttempt` calls this with `settingsManager.getProviderRetrySettings().maxRetries`, an active session that was first wrapped with unset or old retry settings will keep injecting that stale value and will shadow later `settings.retry.provider.maxRetries` changes. Please make the installed wrapper read/update the latest default on repeated installs, and add a regression where the second install changes the configured value.
  - confidence: 0.9
