## Summary

Keep open. Adds locked and hash-guarded exec-approvals writes across node-host, gateway, CLI, and allowlist metadata paths, plus regression tests for stale allowlist writeback.

## Best Possible Solution

Rebase onto current main, preserve the no-persistence and persistAllowAlwaysDecision behavior, move every exec-approvals policy or metadata write behind one stale-write-safe path or skip stale metadata, update focused CLI tests, and add redacted real behavior proof.

Yes. Source inspection shows current main resolves an approvals snapshot for node execution, admin writes use a base-hash freshness check, and later execution metadata persistence can still save the earlier approvals object.

No. The PR is a plausible mitigation, but it is not the best complete fix until initialization/defaulting writes are guarded, current-main conflicts are resolved, tests match the new write contract, and real behavior proof is supplied.
