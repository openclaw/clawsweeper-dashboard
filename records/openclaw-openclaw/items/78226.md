## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Guard the initialization write before resolving approvals:** `src/infra/exec-approvals.ts:786`
  - body: The new guarded writeback still starts from `resolveExecApprovals()`, but that call still runs `ensureExecApprovals()` before returning the hash, and `ensureExecApprovals()` saves a full approvals snapshot. If an admin revocation lands between that load and save, the old file can be restored before the later base-hash guard runs. Move initialization/defaulting behind the same lock/hash guard, or avoid saving when the file is already initialized.
  - confidence: 0.89
- **[P2] Update the CLI tests for locked saves:** `src/cli/exec-approvals-cli.ts:12`
  - body: The production CLIs now import `saveExecApprovalsLocked()`, but the focused CLI tests still mock and assert `saveExecApprovals()`. Those tests will no longer observe the local state mutation for the changed paths, so the approvals and exec-policy test expectations need to mock/assert the locked helper instead.
  - confidence: 0.84
