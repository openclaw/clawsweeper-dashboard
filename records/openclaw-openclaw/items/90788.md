Full review comments:

- **[P1] Gate the goal-plan tools behind the existing planning contract:** `src/agents/openclaw-tools.ts:470-481`
  - body: This adds `update_goal_plan` and `update_goal_plan_step` directly to every OpenClaw tool set, while current main only exposes the existing `update_plan` tool through explicit allow/config or strict-agentic mode. Because these new tools mutate persisted goal state, existing runs get a new planning source of truth by default; put them behind the existing gate or remove the parallel surface.
  - confidence: 0.91
- **[P1] Wire `cotPlanning` through the accepted config schema:** `src/config/types.agent-defaults.ts:354-359`
  - body: The PR adds `agents.defaults.cotPlanning` to the exported TypeScript type and runtime reads it, but `AgentDefaultsSchema` is strict and the key is absent from schema/help/labels/docs. Users cannot safely set `off`, `always`, or `minBudgetTokens` until the config contract and tests are added, or the config knob should be removed.
  - confidence: 0.89
- **[P2] Align prompt examples with the tool schema:** `src/agents/cot-planning-prompt.ts:110-117`
  - body: The prompt tells the model about `depends_on`/`estimated_tokens`, but the concrete example and parser still use `dependsOn`/`estimatedTokens` while the tool schema reads snake_case fields. A model that follows the shown example will not round-trip cleanly into `update_goal_plan` or `create_goal(plan: ...)`.
  - confidence: 0.9
- **[P2] Ask for a concise plan instead of chain-of-thought:** `src/agents/cot-planning-prompt.ts:89`
  - body: This prompt asks the model to produce a structured Chain-of-Thought plan inside tags before acting. OpenClaw should request a concise plan or checklist, not internal reasoning content that may leak into transcripts or tool payloads.
  - confidence: 0.84
- **[P2] Keep dynamic goal progress out of the stable prefix cache:** `src/agents/system-prompt.ts:1027`
  - body: `cotPlanningSection` depends on the active goal and step progress, then gets included in the stable prefix cache key and cached prefix. Every plan update can churn the stable-prefix cache and move per-session dynamic state above the cache boundary; render it below the boundary and exclude it from the stable key.
  - confidence: 0.86
- **[P3] Keep skipped and active steps consistent in progress output:** `src/agents/cot-planning-prompt.ts:160`
  - body: Progress counts only `done` steps, but dependency resolution treats `skipped` as completed and ignores an already `active` step when choosing the next action. This can under-report progress and point the model past the current in-progress step.
  - confidence: 0.82
