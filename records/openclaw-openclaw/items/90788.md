## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Gate goal-plan tools behind the existing plan contract:** `src/agents/openclaw-tools.ts:470-481`
  - body: These tools are added to every `createOpenClawTools` inventory, while current `update_plan` is gated by tool policy, `tools.experimental.planTool`, or strict-agentic GPT-5. Existing users would get two new mutating session-state tools by default.
  - confidence: 0.94
- **[P1] Wire cotPlanning through the strict config schema:** `src/config/types.agent-defaults.ts:353-359`
  - body: Adding the TypeScript field advertises `agents.defaults.cotPlanning`, but `AgentDefaultsSchema` is strict and does not accept that key. Users who configure it will hit an unknown-key config failure instead of controlling the feature.
  - confidence: 0.95
- **[P1] Keep step plans out of the documented goal contract:** `src/config/sessions/types.ts:218-219`
  - body: Goals are documented as one durable objective, not a task queue. Persisting a model-mutable `planSnapshot` of steps changes that contract and lets the agent silently replace subtask state across sessions and compaction.
  - confidence: 0.9
- **[P2] Keep dynamic goal progress below the cache boundary:** `src/agents/system-prompt.ts:1026-1028`
  - body: `cotPlanningSection` is derived from current goal state, included in the stable-prefix cache key, and rendered above the cache boundary. Plan status changes during execution will churn the stable prompt prefix instead of preserving cache identity.
  - confidence: 0.88
- **[P2] Use the tool schema field names consistently:** `src/agents/cot-planning-prompt.ts:110-117`
  - body: The prose now names snake_case fields, but the example and parser still use `dependsOn` and `estimatedTokens` while the tool schema reads `depends_on` and `estimated_tokens`. A model copying the example into tool args will lose dependency and token-estimate data.
  - confidence: 0.9
- **[P2] Reject plan-step updates after terminal goals:** `src/agents/tools/cot-planning-tools.ts:224-253`
  - body: `update_goal_plan_step` mutates `planSnapshot` without checking whether the goal is already `complete`, even though `update_goal_plan` rejects completed goals. This allows post-completion state drift.
  - confidence: 0.86
- **[P2] Ask for an execution plan, not chain-of-thought:** `src/agents/cot-planning-prompt.ts:89`
  - body: The prompt asks the model to produce a `Chain-of-Thought plan` inside visible tags. OpenClaw should request a concise execution plan/tool payload rather than assistant-visible internal reasoning.
  - confidence: 0.84
- **[P2] Count skipped steps consistently in progress:** `src/agents/cot-planning-prompt.ts:160-162`
  - body: Progress counts only `done` steps as complete, but dependency resolution treats `skipped` as satisfied. Rendered progress and compaction summaries will under-report completed work whenever a skipped step unblocks later work.
  - confidence: 0.82
- **[P2] Prefer the active step as the next step:** `src/agents/cot-planning-prompt.ts:347-350`
  - body: `resolveNextActionableStep` ignores an already active step, so the prompt can point the model past in-progress work to the next pending dependency-satisfied step.
  - confidence: 0.82
