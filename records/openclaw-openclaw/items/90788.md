## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Gate goal-plan tools behind existing plan policy:** `src/agents/openclaw-tools.ts:470-481`
  - body: These two mutating tools are added to every `createOpenClawTools` inventory, while current `update_plan` is gated by tool policy, `tools.experimental.planTool`, or strict-agentic GPT-5. Existing users would get new durable session-state mutation tools by default.
  - confidence: 0.94
- **[P1] Wire cotPlanning through the strict config schema:** `src/config/types.agent-defaults.ts:353-359`
  - body: Adding the TypeScript field advertises `agents.defaults.cotPlanning`, but `AgentDefaultsSchema` is strict and does not accept that key. Users who configure it will hit an unknown-key config failure instead of controlling the feature.
  - confidence: 0.95
- **[P1] Do not persist model-mutable plans on goals without a contract:** `src/config/sessions/types.ts:218-219`
  - body: Goals are documented as one durable objective, not a task queue. Persisting a model-mutable `planSnapshot` of steps changes that contract and lets the agent replace subtask state across sessions and compaction.
  - confidence: 0.9
- **[P2] Keep dynamic plan text out of the stable prompt cache:** `src/agents/system-prompt.ts:1026-1027`
  - body: `cotPlanningSection` is derived from current goal state, included in the stable-prefix cache key, and rendered above the cache boundary. Plan status changes during execution will churn the stable prompt prefix instead of preserving cache identity.
  - confidence: 0.88
- **[P2] Request an execution plan instead of Chain-of-Thought:** `src/agents/cot-planning-prompt.ts:89`
  - body: The prompt asks the model to produce a `Chain-of-Thought plan` inside visible tags. OpenClaw should request a concise execution plan or tool payload rather than assistant-visible internal reasoning.
  - confidence: 0.84
- **[P2] Use snake_case fields in the prompt example:** `src/agents/cot-planning-prompt.ts:110-117`
  - body: The prose names snake_case fields, but the example still uses `dependsOn` and `estimatedTokens` while the tool schema reads `depends_on` and `estimated_tokens`. A model copying the example into tool args will lose dependency and token-estimate data.
  - confidence: 0.9
- **[P2] Reject plan-step updates after completed goals:** `src/agents/tools/cot-planning-tools.ts:228-234`
  - body: `update_goal_plan_step` mutates `planSnapshot` without checking whether the goal is already `complete`, even though `update_goal_plan` rejects completed goals. This allows post-completion state drift.
  - confidence: 0.86
- **[P2] Count skipped steps as complete:** `src/agents/cot-planning-prompt.ts:160`
  - body: Progress counts only `done` steps as complete, but dependency resolution treats `skipped` as satisfied. Rendered progress and compaction summaries will under-report completed work whenever a skipped step unblocks later work.
  - confidence: 0.82
- **[P2] Prefer the active step when selecting next work:** `src/agents/cot-planning-prompt.ts:347-350`
  - body: `resolveNextActionableStep` ignores an already active step, so the prompt can point the model past in-progress work to the next pending dependency-satisfied step.
  - confidence: 0.82
