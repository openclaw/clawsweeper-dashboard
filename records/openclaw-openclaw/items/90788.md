- **[P1] Gate the goal-plan tools behind the existing plan contract:** `src/agents/openclaw-tools.ts:470-481`
  - body: These tools are added to every createOpenClawTools inventory, while current update_plan is gated by tool policy, tools.experimental.planTool, or strict-agentic GPT-5. That gives existing users two new mutating session-state tools by default and bypasses the documented planning-tool contract.
  - confidence: 0.93
- **[P1] Preserve planning-only retry accounting for goal-plan tools:** `src/agents/embedded-agent-runner/run/attempt-tool-construction-plan.ts:47-48`
  - body: update_goal_plan and update_goal_plan_step are added as OpenClaw factory names, but the strict-agentic no-stall guard only treats update_plan as non-progress. A turn that only writes the new plan can now avoid the act-now retry even though no real task step ran.
  - confidence: 0.91
- **[P1] Wire cotPlanning through the strict config schema:** `src/config/types.agent-defaults.ts:353-359`
  - body: Adding the TypeScript field advertises agents.defaults.cotPlanning, but AgentDefaultsSchema is strict and current schema/help/docs do not include it. Users who configure the new key will hit an unknown-key config failure instead of controlling the feature.
  - confidence: 0.94
- **[P1] Keep step plans out of the documented goal state contract:** `src/config/sessions/types.ts:218-219`
  - body: Goals are documented as one durable objective and not a task queue; adding a model-mutable planSnapshot of steps to SessionGoal changes that contract and lets the agent silently replace and persist subtask state across sessions and compaction.
  - confidence: 0.88
- **[P2] Use the tool schema field names in the prompt and parser:** `src/agents/cot-planning-prompt.ts:110-117`
  - body: The instructions now list snake_case fields, but the example and parser still use dependsOn and estimatedTokens while parseGoalPlan reads depends_on and estimated_tokens. Models copying the example into tool args will lose dependency and token-estimate data.
  - confidence: 0.9
- **[P2] Avoid asking the model to emit chain-of-thought:** `src/agents/cot-planning-prompt.ts:89`
  - body: The prompt asks for a Chain-of-Thought plan in assistant-visible cot_plan tags. OpenClaw should request a concise execution plan or tool payload, not user-visible internal reasoning, especially while channel reasoning/text leaks remain an active product risk.
  - confidence: 0.84
- **[P2] Keep dynamic goal progress below the prompt cache boundary:** `src/agents/system-prompt.ts:1026-1027`
  - body: The goal-derived cotPlanningSection is included in the stable prefix cache key and inserted above the cache boundary. Plan status changes during execution, so this will churn the stable prefix cache instead of preserving byte-identical cacheable text.
  - confidence: 0.87
- **[P2] Reject step updates after a goal completes:** `src/agents/tools/cot-planning-tools.ts:228-251`
  - body: update_goal_plan_step mutates planSnapshot without checking whether the goal is already complete, even though update_goal_plan rejects completed goals. This allows post-completion plan state drift after the goal should be terminal.
  - confidence: 0.86
