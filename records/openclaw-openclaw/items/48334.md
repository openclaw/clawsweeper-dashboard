## Summary

Keep open: the cron/session context-window bug is still source-reproducible on current main and the PR has useful direction, but the patch is not mergeable because it moves plain status onto an async discovery path, weakens stored context-token guards, and lacks inspectable after-fix proof.

## What This Changes

The PR adds an async context-token resolver for cron session persistence, status-side stale 200k display repair, per-agent contextTokens typing, and regression tests.

## Best Possible Solution

Land a rebased fix that blocks registry readiness only for cron persistence, keeps plain status on a cached/read-only path, preserves positive-value and explicit-cap guards, and includes focused regression coverage plus redacted real behavior proof.

## Reproduction Assessment

Yes, by source inspection: current main can persist DEFAULT_CONTEXT_TOKENS when finalizeCronRun calls lookupContextTokens with allowAsyncLoad false before the model registry cache is warm and no previous positive session value exists.

## Solution Assessment

No. The cron-side blocking resolver direction is reasonable, but the status-side repair should stay read-only/cached and preserve positive stored-value plus explicit-cap guards before this PR is mergeable.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Keep status summaries read-only:** `src/commands/status.summary.ts:181-187`
  - body: getStatusSummary now awaits resolveContextTokensForModelAsync, whose loader can prepare models.json and import provider discovery.   - confidence: 0.9

  - confidence: 0.9
- **[P2] Preserve explicit agent caps for legacy keys:** `src/commands/status.summary.ts:50`
  - body: hasExplicitContextTokensCap returns false when agentId is missing, but sessions.recent is rebuilt from raw store paths where legacy keys may not parse to an agent id. A legitimate 200000 per-agent cap can be upgraded to the registry window unless the store agent id is carried through or unknown-owner rows skip repair.
  - confidence: 0.84
