## Summary

Keep open: this is a useful focused bug fix with sufficient production proof, but the latest head still rebuilds stripped skill prompts without the original remote eligibility and can bypass the saved snapshot skill set on embedded resume.

## What This Changes

The PR stops persisting rendered skills prompt text in session snapshots, makes prompt fields optional, adds resume-time prompt rebuild logic, adjusts tests/fingerprinting, and updates the changelog.

## Best Possible Solution

Keep the store-size reduction, but make cold resume reconstruct the prompt from the saved snapshot skill set using the same eligibility context as original snapshot creation.

## Reproduction Assessment

Yes for source-level proof: current main persists `skillsSnapshot.prompt` while synchronously reading the session store, and the PR body supplies production terminal/log evidence of smaller stores after stripping it. I did not run a live gateway reproduction in this read-only review.

## Solution Assessment

No as currently written: stripping the persisted prompt is the right direction, but the resume rebuild is not equivalent to original snapshot construction. The narrow maintainable fix is to thread/recompute eligibility at resume call sites and filter rebuilt entries to the recorded snapshot skills.
