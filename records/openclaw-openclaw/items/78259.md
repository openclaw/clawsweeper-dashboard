## Summary

Keep open: the PR addresses a real session-store bloat bug and has sufficient production proof, but the latest branch still leaves stripped-prompt resume paths able to rebuild a different skills prompt than the original snapshot.

## What This Changes

The PR stops persisting `skillsSnapshot.prompt`, makes prompt fields optional, rebuilds stripped prompts from workspace skills, updates tests, and adds a changelog entry.

## Best Possible Solution

Land this optimization after stripped-prompt rebuilds use the saved skill set and original eligibility context across CLI, embedded run, and compaction paths, with the changelog aligned to measured proof.

## Reproduction Assessment

Yes, at source level: current main still persists the rendered prompt while synchronously reading the session store, and the PR body supplies production after-fix size and lock-stall evidence. I did not live-run the gateway in this read-only review.

## Solution Assessment

No as currently written: stripping persisted prompt text is the right fix direction, but the branch must make prompt rebuilds equivalent to the original snapshot construction before merge.
