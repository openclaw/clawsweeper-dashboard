## Summary

Keep open: the persistence fix is well targeted and real behavior proof is supplied, but latest head still has a source-reproducible cold-resume regression because the new fallback depends on remote eligibility that resume callers do not provide.

## What This Changes

The PR strips rendered skill prompt text from persisted session snapshots, makes prompt fields optional, adds fallback prompt rebuild on resume, updates affected session/skill tests, Claude fingerprinting, and changelog.

## Best Possible Solution

Keep the prompt persistence strip, but make cold resume rebuild the prompt with the same remote eligibility and filter semantics used when the original snapshot is created, then cover that path with a regression test.

## Reproduction Assessment

Yes for source-level reproduction: current main persists `skillsSnapshot.prompt` while synchronously reading the session store, and the PR's remaining cold-resume issue follows from eligibility-gated skill filtering with no eligibility supplied by resume callers.

## Solution Assessment

No as currently written: stripping the persisted prompt is the right direction, but the fallback is not yet equivalent to original snapshot construction. The narrower maintainable fix is to recompute or thread the same remote eligibility at the resume call sites before rebuilding the prompt.
