## Summary

Keep open: the persistence-size direction is useful, but the diff drops the persisted skills prompt without restoring it for CLI resume paths and the external PR still has only test-claim proof.

## What This Changes

The PR changes session persistence to drop `skillsSnapshot.prompt`, makes skill snapshot prompt fields optional, adjusts Claude live-session fingerprinting, and updates session-store stripping tests.

## Best Possible Solution

Keep the store-size reduction, but merge only after the prompt is rebuilt or otherwise preserved for every resume path, especially CLI backends, with regression coverage, a changelog entry, and real after-fix proof.

## Reproduction Assessment

Yes, at source level: current main persists `skillsSnapshot.prompt` and the PR removes it, while CLI resume preparation has no entries fallback to rebuild the prompt. I did not run a live gateway stress reproduction for the reported WebSocket timeout behavior.

## Solution Assessment

No. Stripping at the persistence chokepoint is plausible, but this patch is not complete because hydration only restores `resolvedSkills` and CLI preparation can fall through to an empty skills prompt after a cold load.
