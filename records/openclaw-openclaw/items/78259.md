## Summary

Keep open: the latest PR head preserves the useful store-size optimization, but two resume-path correctness issues from the prior review are still present and the changelog overstates the supplied production measurements.

## What This Changes

The branch stops persisting `skillsSnapshot.prompt` in session stores, makes prompt fields optional, rebuilds the skills prompt on resume, updates session/skills tests, and adds a changelog entry.

## Best Possible Solution

Keep the prompt-stripping optimization, but make prompt rebuilds equivalent to original snapshot construction by carrying remote eligibility and limiting rebuilds to recorded snapshot skills, then adjust the changelog and validate the exact head.

## Reproduction Assessment

Yes, source-level: current main persists `skillsSnapshot.prompt` while synchronously reading the session store, and the PR body supplies after-fix terminal/log evidence from a production gateway. I did not run live tests in this read-only review.

## Solution Assessment

No as currently written: stripping persisted prompt text is the right direction, but the latest rebuild path is not equivalent to original snapshot construction. The safer narrow fix is to pass or recompute remote eligibility at resume call sites and constrain every stripped-prompt rebuild to the recorded snapshot skill names.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Pass eligibility from cold-resume callers:** `src/agents/skills/workspace.ts:1009`
  - body: The new resolver parameter is still never supplied by CLI prepare, embedded run, or compaction. Cold resumes rebuild a stripped prompt by reloading entries without the remote macOS platform/bin context used by `buildWorkspaceSkillSnapshot`, so darwin-only or remote-bin-gated skills that were in the saved snapshot can disappear; pass the same eligibility context from these callers.
  - confidence: 0.89
- **[P2] Filter entry rebuilds by the saved snapshot:** `src/agents/skills/workspace.ts:1015-1020`
  - body: When `resolvedSkills` is absent, `resolveEmbeddedRunSkillEntries` supplies all currently loaded entries, and this branch runs before the snapshot-name fallback added below. That can rebuild an old session prompt with newly installed/plugin/bundled skills instead of only `skillsSnapshot.skills`; filter the entries with the saved skill names or prefer the saved-snapshot rebuild path when prompt is stripped.
  - confidence: 0.86
- **[P3] Use the measured shrink in the changelog:** `CHANGELOG.md:12`
  - body: The changelog says active session stores shrink about 30x, but the PR body's production table shows observed reductions of 2.3x to 6.9x after about 24 hours. Use the measured range or explicitly label the larger value as a projection so the release note does not overstate the supplied proof.
  - confidence: 0.81
