## Summary

Keep open: the store-size bug is well supported, but the latest PR head still rebuilds stripped skill prompts with non-equivalent skill eligibility and skill-set semantics, so it should not merge yet.

## What This Changes

The PR stops persisting rendered skills prompts in session snapshots, makes prompt fields optional, adds resume-time prompt rebuild logic, updates tests, and adds a changelog entry.

## Best Possible Solution

Keep the prompt-stripping optimization, but make cold resume rebuild prompts only from the recorded snapshot skills using the same remote eligibility context across CLI, embedded run, and compaction paths.

## Reproduction Assessment

Yes, source-level: current main persists skillsSnapshot.prompt while synchronously reading the session store, and the PR body supplies after-fix terminal/log evidence from a production gateway. I did not run a live gateway reproduction in this read-only review.

## Solution Assessment

No as currently written: stripping persisted prompt text is the right direction, but the rebuild path is not equivalent to original snapshot construction. The safer fix is to thread or recompute eligibility at resume call sites and constrain every rebuild to the recorded snapshot skill names.
