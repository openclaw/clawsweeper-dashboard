## Summary

Keep open: this is a useful session-store bloat fix with sufficient production proof, but the current head still has stripped-prompt resume semantics that can rebuild a different skills prompt than the original session had.

## What This Changes

The PR strips `skillsSnapshot.prompt` from persisted session stores, makes skill snapshot prompts optional, rebuilds missing skill prompts on resume, and updates related tests and changelog text.

## Best Possible Solution

Land the optimization after stripped-prompt rebuilds preserve the saved skill set and original eligibility context across CLI resume, embedded run, and compaction paths, with the release note aligned to measured proof.

## Reproduction Assessment

Yes, at source level: current main persists the rendered prompt while synchronously reading the full session store, and the PR supplies production after-fix size and lock-stall evidence. I did not live-run the gateway in this read-only sweep.

## Solution Assessment

No as currently written: stripping persisted prompt text is the right fix direction, but the rebuild path must remain equivalent to the original snapshot construction before merge.
