## Summary

Keep open: the PR addresses a real session-store bloat path and has sufficient production proof, but the new prompt rebuild fallback can change resumed skill prompts by ignoring eligibility-dependent skills.

## What This Changes

The branch strips `skillsSnapshot.prompt` from session persistence, makes prompt optional, rebuilds missing skill prompts on resume, updates Claude live-session fingerprinting, tests, and the changelog.

## Best Possible Solution

Keep the persistence strip, but rebuild missing prompts with the same skill eligibility and filter semantics used when snapshots are created, then land after targeted tests and changed-gate validation pass.

## Reproduction Assessment

Yes for the persistence-bloat path by source inspection: current main persists `skillsSnapshot.prompt` while synchronously reading the session store. The PR-introduced resume fallback regression is also source-reproducible because eligibility affects skill inclusion and the new rebuild path omits it.

## Solution Assessment

No as currently written: stripping the persisted prompt is the right direction, but the fallback rebuild is not yet equivalent to the original snapshot prompt. The safer fix is to preserve eligibility-aware rebuild semantics while still avoiding prompt persistence.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Preserve eligibility when rebuilding stripped skill prompts:** `src/agents/skills/workspace.ts:1026-1031`
  - body: This cold-resume fallback rebuilds the prompt with only config/filter/agentId. The original snapshots are built with an eligibility context, and `shouldIncludeSkill` uses that context for remote platforms and remote binaries; after `prompt` is no longer persisted, a resumed session can silently lose skills that were included because a paired node satisfied those requirements. Pass/recompute the same eligibility context, or rebuild from an eligibility-preserved snapshot path, before dropping the cached prompt.
