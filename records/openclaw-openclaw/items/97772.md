## Summary

Keep open: current main and v2026.6.10 still contain the stale Z.AI `enable_thinking` path, and the current GLM-5.2 plugin wrapper only adds `reasoning_effort` for enabled levels without adding the required `thinking` object. This is a valid, narrow provider payload bug.

## What This Changes

Fix Z.AI/GLM thinking request shaping so reasoning-on requests use Z.AI's current `thinking` object instead of the stale `enable_thinking` parameter.

## Best Possible Solution

Land a narrow Z.AI payload fix that sends `thinking.type` for enabled and disabled thinking, preserves the GLM-5.2 `reasoning_effort` mapping, removes the stale `enable_thinking` contract for Z.AI, and updates focused tests/docs/types.
