## Summary

## What This Changes

The issue asks OpenClaw to update Z.AI/GLM thinking payloads from stale `enable_thinking` to the current `thinking` object contract.

## Best Possible Solution

Land a narrow Z.AI provider/request-shaping fix that sends `thinking.type` for on/off, maps GLM-5.2 effort through `reasoning_effort`, removes the stale Z.AI `enable_thinking` contract, and updates the compat docs/types plus focused tests.
