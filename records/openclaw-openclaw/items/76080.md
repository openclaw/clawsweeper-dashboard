## Summary

Keep open. This PR is a focused, directionally correct fix for the linked Gemini idle-timeout bug: current main skips Google SSE parts that only carry `thoughtSignature`, while the idle watchdog only resets when the OpenClaw stream yields an event. The remaining blocker is a required `CHANGELOG.md` entry for this user-facing provider fix.

## What This Changes

The PR updates the Google transport to emit non-visible activity for `thoughtSignature`-only Gemini SSE parts and adds regression tests for signature-only thinking parts.

## Best Possible Solution

Land this transport-level liveness fix after adding the required changelog entry and keeping the existing green CI/changed-path proof intact.

## Reproduction Assessment

Yes for the code path. Current main can be reproduced with a deterministic Google SSE fixture containing a `thoughtSignature`-only part; the linked issue supplies detailed production steps, while the PR adds fixture coverage for the transport output shape.

## Solution Assessment

Yes, functionally. Emitting a non-visible activity event from the Google transport is a narrow fix that keeps hidden Gemini thinking/signature frames from tripping the generic idle watchdog without changing the core timeout policy.
