## Summary

Keep open. The PR addresses a plausible current-main Google transport liveness gap, but it should not merge as-is because it emits a new `thinking_signature` stream event that is not part of the public `AssistantMessageEvent` protocol.

## What This Changes

The PR adds Google transport handling and tests for Gemini `thoughtSignature`-only SSE parts by creating or updating thinking blocks and emitting a stream activity event.

## Best Possible Solution

Keep the Google-owned transport fix, but wake the idle iterator using existing typed stream events or add a deliberate typed SDK event seam with matching docs and tests.

## Reproduction Assessment

Yes for the transport liveness gap: a deterministic Google SSE fixture with `{ thoughtSignature: "sig" }` and no text/functionCall exercises a path that current main parses without yielding a stream event. The full production hang still needs live-provider confirmation if maintainers want end-to-end proof.

## Solution Assessment

No as written. The fix belongs in the Google transport, but the activity signal should stay within the existing `AssistantMessageEvent` protocol, such as `thinking_start` plus an empty `thinking_delta`, unless the stream contract is explicitly extended.
