## What This Changes

The PR changes the agent cron tool so omitted or invalid `runMode` sends `mode: "due"` instead of `mode: "force"`, and updates cron-tool expectations for that default.

## Best Possible Solution

Land the narrow agent/MCP default change after maintainer confirmation, with real runtime proof that omitted `runMode` respects schedule guards while explicit `runMode: "force"` still force-runs.

## Reproduction Assessment

Yes. Source inspection shows omitted agent-tool `runMode` becomes `mode: "force"`, and force mode bypasses enabled and schedule due checks; I did not run a live Gateway scenario because this review is read-only.

## Solution Assessment

Yes, with maintainer sign-off. Changing only the agent/MCP default is the narrowest fix for the linked unsafe default while preserving the documented CLI/Gateway force-default path; the broader alternative would carry a larger compatibility decision.
