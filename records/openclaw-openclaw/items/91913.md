## Summary

Keep this open: current main still has source-backed paths where an explicit active-memory thinking-off request can collapse to an omitted reasoning option, the OpenAI completions transport then defaults that omission to `high`, and LM Studio binary reasoning metadata does not map enabled canonical levels to a provider-accepted binary value.

## What This Changes

The issue asks OpenClaw to make OpenAI-compatible completions honor LM Studio binary reasoning compatibility maps and active-memory `thinking: "off"` instead of silently sending `reasoning_effort: "high"`.

## Best Possible Solution

Keep the issue open and fix the shared OpenAI-compatible reasoning path so explicit thinking-off remains disabled, LM Studio binary reasoning maps to accepted values, and the legacy completions builder no longer drifts from the documented compat-map contract.

## Reproduction Assessment

Yes from source inspection, though not from a live LM Studio run in this review. The current code path shows active-memory sends `thinkLevel: "off"`, the agent loop turns that into omitted reasoning, and OpenAI completions transport can then default the request to `reasoning_effort: "high"`.

## Solution Assessment

No, an old-path-only one-line compat-map fallback is too narrow for the reported behavior. The better fix should cover explicit off propagation, LM Studio binary mapping, and the legacy completions builder so all OpenAI-compatible completions paths share the same compatibility contract.
