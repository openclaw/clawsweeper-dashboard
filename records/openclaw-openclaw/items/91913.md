## What This Changes

The issue requests that active-memory and LM Studio OpenAI-compatible completions honor thinking-off and `compat.reasoningEffortMap` instead of sending unsupported `reasoning_effort: "high"`.

## Best Possible Solution

Land one focused fix that handles explicit thinking-off and LM Studio binary reasoning maps at the current OpenAI-compatible completions boundary, with active-memory and LM Studio regression coverage, then close this issue from the merged PR.

## Reproduction Assessment

Yes at source level: active-memory forwards `thinkLevel: off`, while the current OpenAI-compatible completions transport defaults missing reasoning effort to `high` and the off wrapper does not suppress the completions scalar. I did not run a live LM Studio server in this read-only review.

## Solution Assessment

No for a legacy-builder-only fix: current embedded runs use the boundary-aware transport path, so the maintainable fix belongs in the shared current completions reasoning boundary and LM Studio map behavior. The linked PRs should be reviewed or consolidated rather than opening another independent fix lane.
