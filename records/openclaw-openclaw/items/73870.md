## Summary

Keep open. Current main still has a single global `subagent` command lane plus per-session active-child rejection; it does not implement queued per-parent spawn admission, and the related open issues cover adjacent per-agent and multi-lane scheduling rather than this parent-keyed throttle.

## What This Changes

Request a configurable queued per-parent concurrency cap for subagent spawns to prevent one parent fanout from saturating gateway and WebSocket work.

## Best Possible Solution

Add a documented requester/parent-keyed queued spawn admission layer across native and ACP spawns, coordinated with the existing global lane, per-session child cap, and related per-agent/multi-lane proposals.

## Reproduction Assessment

Yes for the missing product behavior: source, docs, and tests on current main show global lane scheduling plus reject-only active-child caps, not queued per-parent throttling. No live large-fleet Slack timeout reproduction was attempted.

## Solution Assessment

Unclear. A queued per-parent cap plausibly targets same-parent fanout pressure, but the cap default, fairness model, queue UX, and lifecycle cleanup need maintainer policy before a safe implementation.
