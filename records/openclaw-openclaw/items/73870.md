## Summary

Keep open. Current main still has a global subagent lane plus reject-only active-child caps; it does not implement the requested queued per-parent spawn throttle, and related open issues cover different scheduling axes.

## What This Changes

Request a configurable per-parent subagent spawn concurrency cap so same-parent fanout queues beyond the limit instead of saturating gateway resources.

## Best Possible Solution

Add a documented requester/parent-keyed queued spawn throttle separate from the global subagent lane and active-child cap, covering native and ACP spawns with coherent status and lifecycle semantics.

## Reproduction Assessment

Yes for the missing product behavior: current source, docs, and tests show global lane scheduling plus reject-only active-child caps, not queued per-parent throttling. No live reproduction was attempted for the large-fleet Slack timeout symptom.

## Solution Assessment

Unclear. A queued per-parent throttle plausibly targets same-parent fanout pressure, but defaults, fairness, queued-state UX, and lifecycle cleanup need maintainer policy before a safe implementation.
