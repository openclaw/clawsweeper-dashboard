## Summary

Keep open. Current main still has a global subagent lane plus reject-only per-session child caps; it does not implement the requested queued per-parent spawn throttle, and the related open items cover adjacent per-agent and multi-lane scheduling axes rather than this parent-keyed admission policy.

## What This Changes

Request a configurable per-parent queued concurrency cap for subagent spawns to reduce large-fleet event-loop and WebSocket contention.

## Best Possible Solution

Add a documented requester/parent-keyed queued spawn admission layer across native and ACP spawns, with clear fairness, status, cancellation, and cleanup semantics aligned with existing global lane behavior.

## Reproduction Assessment

Yes for the missing product behavior: current source, docs, and tests show global lane scheduling plus reject-only active-child caps, not queued per-parent throttling. No live reproduction was attempted for the large-fleet Slack timeout symptom.

## Solution Assessment

Unclear. A queued per-parent cap plausibly targets same-parent fanout pressure, but the cap default, fairness model, queue UX, and lifecycle cleanup need maintainer policy before a safe implementation.
