## Summary

Keep open. Current main still has one global subagent execution lane plus reject-only per-session child caps; it does not implement the requested queued per-parent spawn throttle, and the related open issues cover adjacent per-agent and multi-lane scheduling work rather than this parent-keyed policy.

## What This Changes

Request a configurable queued per-parent concurrency limit for subagent spawns to reduce large-fleet event-loop and WebSocket contention.

## Best Possible Solution

Add a documented requester/parent-keyed queued spawn admission layer across native and ACP spawns, with clear fairness, status, cancellation, and cleanup semantics aligned with existing global lane behavior.

## Reproduction Assessment

Yes for the missing product behavior: source, docs, and tests on current main show global lane scheduling plus reject-only active-child caps, not queued per-parent throttling. No live reproduction was attempted for the large-fleet Slack timeout symptom.

## Solution Assessment

Unclear. A queued per-parent cap plausibly targets same-parent fanout pressure, but the cap default, fairness model, queue UX, and lifecycle cleanup need maintainer policy before a safe implementation.
