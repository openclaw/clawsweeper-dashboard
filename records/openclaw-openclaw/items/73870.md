## Summary

Keep open. Current main still has global subagent-lane scheduling plus reject-only per-requester child caps; it does not provide a queued parent-keyed spawn throttle, and the related open issues cover different concurrency axes.

## What This Changes

Request a configurable per-parent concurrency cap for subagent spawns so excess children queue instead of contending through the same gateway/WebSocket path.

## Best Possible Solution

Add a documented requester/parent-keyed spawn throttle that is distinct from the global subagent lane and the active-child admission cap, covering native and ACP child runs with coherent status and lifecycle semantics.

## Reproduction Assessment

Yes for the missing product behavior: current source, docs, and tests show global lane scheduling plus reject-only requester child caps, not queued per-parent throttling. The large-fleet Slack timeout symptom itself was not live-reproduced in this read-only pass.

## Solution Assessment

Unclear. A queued per-parent throttle plausibly addresses same-parent fanout pressure, but defaults, fairness policy, queued-state UX, and lifecycle behavior need maintainer design before implementation.
