## Summary

Keep open: the PR has useful live Telegram QA proof for the one-message final behavior, but the patch appears to silently drop inline buttons when the post-stop button edit fails, and the user-visible single-stream contract still needs Telegram maintainer acceptance.

## What This Changes

The PR rewrites Telegram text streaming so final text stops and reuses the active stream message, removes archived preview/fresh-final delivery paths, updates Telegram QA/tests/docs, and adds a changelog fix.

## Best Possible Solution

Preserve the single-stream final behavior only after restoring fallback for failed button edits, then land if the Telegram owner accepts the updated final-delivery contract.

## Reproduction Assessment

No high-confidence live before-fix reproduction was run in this read-only review. Current main source/docs show the old fresh-final paths, the PR body provides after-fix live output, and the inline-button regression is source-reproducible from the PR diff.

## Solution Assessment

No. The single-stream direction is reasonable once accepted, but swallowing failed inline-button edits is not the narrowest maintainable fix; failed required button edits should fall back to normal final delivery or leave the stream unfinalized.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Fall back when button edits fail:** `extensions/telegram/src/lane-delivery-text-deliverer.ts:172-176`

Treat a failed button edit as a fallback instead of returning `preview-finalized`.
