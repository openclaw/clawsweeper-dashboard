## Summary

Keep open. Current main can still suppress Matrix room final replies before `sendMessageMatrix` when `visibleReplies` is unset, and the likely code fix would override the current all-channel group visible-reply policy, so this needs maintainer policy review rather than cleanup closure.

## What This Changes

Restore or clearly explain Matrix room outbound replies that disappear before matrix-js-sdk `sendMessage` after 2026.5.x when `visibleReplies` is unset.

## Best Possible Solution

Maintainers should decide whether Matrix rooms inherit the current tool-only group default or preserve legacy automatic room replies, then implement either clearer Matrix diagnostics/docs or a Matrix-specific dispatch override with regression coverage.

## Reproduction Assessment

Yes, source-reproducible for the reported missing `sendMessage` symptom: a Matrix room with unset visible-reply config is treated as a channel turn and core suppresses automatic final delivery before `sendMessageMatrix`. I did not live-run a Matrix homeserver against current main.

## Solution Assessment

Unclear. The code shape from closed PR #78138 matches the observed suppression, but it changes the current group/channel default, so the safer path is a maintainer policy decision before code changes.
