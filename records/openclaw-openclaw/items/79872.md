## Summary

Keep open. The linked bug is still present on current main, but this PR is not ready because its canonicalizer would rewrite documented `OPENCLAW_WORKSPACE_DIR` defaults to `~/.openclaw/workspace` and the PR still lacks after-fix real behavior proof.

## What This Changes

The PR adds a default-workspace canonicalizer, threads it through setup/onboarding/configure config writes, exports it, and adds focused tests.

## Best Possible Solution

Keep write-time canonicalization for home/profile-derived defaults, preserve `OPENCLAW_WORKSPACE_DIR` backed defaults unchanged, and require redacted saved-config or CLI-flow proof before merge.

## Reproduction Assessment

Yes, at source level. Current main resolves default workspaces to host-specific absolute paths and setup/onboarding/configure persist those values directly; the PR regression is also source-reproducible because the new canonicalizer rewrites env-backed defaults.

## Solution Assessment

No, not as written. Write-time canonicalization is the right narrow approach, but it must skip or preserve `OPENCLAW_WORKSPACE_DIR` defaults instead of converting them to the home/profile portable path.

## Review Findings
