## Summary

Keep this PR open. It targets a real config-serialization bug, but the current branch would regress the documented `OPENCLAW_WORKSPACE_DIR` default after rebase and still lacks after-fix real behavior proof.

## What This Changes

The PR adds write-time canonicalization so setup, onboarding, and configure flows persist recognized default workspaces as portable `~/.openclaw/...` paths while preserving custom absolute paths.

## Best Possible Solution

Rebase the branch, keep canonicalization for home/profile-derived defaults, preserve env-backed `OPENCLAW_WORKSPACE_DIR` defaults unchanged, and add redacted saved-config or CLI flow proof.

## Reproduction Assessment

Yes, at source level. Current main resolves default workspaces to host-specific absolute paths and setup/onboarding/configure persist those values directly; the PR regression is also source-reproducible because the new canonicalizer rewrites any resolved default to a home/profile portable path.

## Solution Assessment

No, not as written. Write-time canonicalization is the right narrow approach, but it must not rewrite an env-backed default workspace to `~/.openclaw/workspace`.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88
