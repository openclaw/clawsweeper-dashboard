## Summary

Keep open. Current main still lacks Slack `ackReactionTiming`, but the PR branch is not mergeable as-is against the current Slack status-reaction lifecycle; the earlier scope-gate issue appears addressed by carrying `ackReactionAllowed`, while newer status-reaction, test-mock, generated-metadata, and changelog blockers remain.

## What This Changes

The PR adds Slack-scoped `ackReactionTiming` config, docs, and tests so Slack ack reactions can optionally be deferred until `onAgentRunStart`.

## Best Possible Solution

Land a rebased Slack implementation that keeps the default receipt-time behavior, accepts `received | run-start` through strict Slack config/schema/docs and generated metadata, starts the existing ack/status reaction lifecycle only at run start when configured, and preserves scope gates, `message_tool_only`, silent-success cleanup, terminal status reactions, and `removeAckAfterReply`.

## Reproduction Assessment

Yes. Static inspection of current main shows the requested config key is absent and Slack ack/status reactions are prepared before the agent run-start hook; the PR defects are also reproducible from the supplied diff against current imports and generated config surfaces.

## Solution Assessment

No, not as currently patched. The hook point is reasonable, but the branch needs to integrate with current Slack status reactions instead of a raw deferred reaction, then fix the test mock, regenerate config artifacts, and add changelog coverage.
