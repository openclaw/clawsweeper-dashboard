## Summary

Keep open: current main still lacks Slack ack-reaction timing, and the PR remains useful but not merge-ready. The earlier scope-gate issue was addressed with `ackReactionAllowed`, but the branch still needs to integrate with current Slack status reactions, update the test seam, refresh generated config metadata, and add changelog coverage.

## What This Changes

The PR adds Slack `channels.slack.ackReactionTiming` config, docs, and tests so Slack ack reactions can optionally be deferred until `onAgentRunStart`.

## Best Possible Solution

Land a rebased Slack implementation that keeps the default receipt-time behavior, accepts `received | run-start` through strict Slack config/schema/docs and generated metadata, starts the existing ack/status reaction lifecycle at run start when configured, and preserves scope gates, `message_tool_only`, terminal status reactions, silent-success restore, and `removeAckAfterReply`.

## Reproduction Assessment

Yes. Static inspection of current main shows the requested config key is absent and Slack ack/status reactions are prepared before the run-start hook; the PR defects are visible by comparing the diff to the current Slack status reaction lifecycle and runtime import seam.

## Solution Assessment

No, not as currently patched. The hook point is reasonable, but the implementation should defer the existing Slack status reaction lifecycle rather than send only a raw reaction, then refresh tests, metadata, and changelog.
