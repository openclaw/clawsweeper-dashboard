## Summary

Keep open: this is an active feature PR, but the current head still has a correctness bug in the disable path for absent experimental flags, and the supplied proof only exercises `--help` instead of an actual toggle/write path.

## What This Changes

The PR adds a schema-derived `openclaw experimental` CLI command plus gated `/experimental` chat/channel and TUI support, config schema metadata, docs, tests, and changelog coverage.

## Best Possible Solution

Keep the schema-derived shared registry approach, but make disable operations persist explicit `false` when absence has different runtime meaning, add regression coverage for that case, and provide redacted real toggle proof before maintainer approval.

## Reproduction Assessment

Yes for the review finding by source inspection: `/experimental off tools.experimental.planTool` reaches `writeExperimentalConfigFlagToFile`, applies the update to `snapshot.parsed`, and treats an absent key as already false. Current main treats undefined differently from explicit false for strict-agentic planTool auto-enable.

## Solution Assessment

No, not as-is. The schema-derived design is the right shape, but the implementation needs to distinguish absent from explicit false and the PR needs real toggle/write proof before merge.
