## Summary

Keep this PR open: the schema-derived, multi-surface direction addresses the maintainer concern, but the current patch still has a behavioral blocker for disabling an auto-enabled experimental flag and lacks real behavior proof for the actual toggle paths.

## What This Changes

The PR adds an `openclaw experimental` command plus gated `/experimental` chat/channel and TUI support backed by schema-derived experimental config flags, with docs and tests.

## Best Possible Solution

Keep the schema-derived shared registry approach, fix disable operations to persist explicit `false` where absence has different runtime behavior, then verify a real toggle path before merge.

## Reproduction Assessment

Yes for the review finding by source inspection: `/experimental off tools.experimental.planTool` reaches the new false-value helper, but an absent raw config value is treated as already false and no explicit `false` is written. Current main then still auto-enables the tool for strict-agentic runs when the config value is undefined.

## Solution Assessment

No, not as-is. The bounded schema-derived design is the right shape, but the implementation must distinguish absent from explicit false for disable operations and needs after-fix proof for actual toggling behavior.
