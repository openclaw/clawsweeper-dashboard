## Summary

Keep this PR open for revision. The shared schema-derived design is promising, but disabling an absent experimental flag can still become a no-op even when explicit false is the documented runtime opt-out, and the supplied real behavior proof only shows CLI help, not an actual toggle/write path.

## What This Changes

The PR adds a schema-derived experimental flag registry exposed through `openclaw experimental`, `/experimental`, TUI handling, native channel command registration, `commands.experimental` gating, docs, and tests.

## Best Possible Solution

Keep the shared schema-derived registry, make disabling persist explicit `false` when absence has runtime meaning, then require maintainer approval and redacted real toggle/write proof before merge.

## Reproduction Assessment

Yes. By source inspection, `off` for an absent `tools.experimental.planTool` produces no write, while current main treats undefined as strict-agentic auto-enable and explicit false as the opt-out.

## Solution Assessment

No, not as-is. The shared schema-derived approach is the right direction, but the absent-false write bug, proof gap, and new config-editing surface approval block merge.
