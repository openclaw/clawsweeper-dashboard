## Summary

Keep this PR open: it addresses a real Codex workspace-only policy gap, but the new policy resolver can incorrectly apply the default agent's explicit opt-out to a missing active agent, creating a workspace-only/native-tool mismatch.

## What This Changes

The PR disables Codex app-server native code/file mode when effective `tools.fs.workspaceOnly=true`, preserves explicit per-agent opt-out behavior, and adds Codex app-server regression coverage.

## Best Possible Solution

Align the Codex workspace-only check with the effective OpenClaw fs policy semantics, add the missing-agent/global-policy regression, then have maintainers accept the fail-closed compatibility tradeoff.

## Reproduction Assessment

## Solution Assessment

No as currently patched: disabling the native surface is the right short-term layer, but the resolver must follow the same effective-agent/global fallback semantics as OpenClaw-owned fs tools.
