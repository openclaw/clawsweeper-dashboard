## Best Possible Solution

Align the Codex check with the effective OpenClaw fs-policy semantics, keep the native Codex surface disabled whenever workspace-only is actually effective, and have maintainers explicitly accept or coordinate the fail-closed compatibility tradeoff.

## Solution Assessment

No as currently patched: disabling the native surface is the right short-term layer, but the resolver must follow the same effective fs-policy semantics as OpenClaw-owned tools before merge.
