## Summary

Keep open: the PR has strong live Codex app-server proof and the fail-closed direction is appropriate, but the current resolver can still let a missing or stale agent id inherit a default-agent `workspaceOnly:false` opt-out and reopen the Codex native filesystem surface under global workspace-only policy.

## What This Changes

The branch adds Codex app-server workspace-only fs policy detection, disables native code/file mode under that policy, and adds dynamic-tool/run-attempt regression coverage.

## Best Possible Solution

Disable native Codex code/file mode only when the effective OpenClaw fs policy is workspace-only, using the same agent/global fallback as core fs tools, then land only after maintainers accept the compatibility tradeoff.

## Reproduction Assessment

Yes, from source inspection: with global `tools.fs.workspaceOnly=true`, a default agent override of `workspaceOnly=false`, and a missing or stale active agent id, the PR resolver enables Codex native tools while core fs policy would fall back to global workspace-only.

## Solution Assessment

No as currently patched: disabling the native surface is the right short-term layer, but the resolver must follow the same effective-agent/global fallback semantics as OpenClaw-owned fs tools.
