## Summary

Keep open, but this PR is not merge-ready: it targets a real Codex filesystem-policy gap, yet the patch wires a filesystem-only setting into the broad native-surface switch and would disable unrelated Code Mode, MCP, and web-search behavior for workspace-only agents. The external PR also lacks after-fix real behavior proof.

## What This Changes

The PR disables the Codex app-server native tool surface when an effective agent or global `tools.fs.workspaceOnly` policy is true, and adds a focused unit test for mixed-agent config.

## Best Possible Solution

Resolve `tools.fs.workspaceOnly` through the canonical filesystem policy and enforce it only at the Codex filesystem or native-execution boundary, with explicit maintainer approval if the intended product behavior is to disable Code Mode, MCP, apps, and native search for workspace-only agents.
