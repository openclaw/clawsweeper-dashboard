## What This Changes

The PR changes diagnostics-otel span identity assignment so `gen_ai.request.model` uses the redacted full model id instead of the low-cardinality fallback.

## Best Possible Solution

Land the narrow span-attribute fix after maintainers decide whether the existing live proof is enough or whether to add the focused diagnostics-otel regression test first.

## Reproduction Assessment

Yes. Source inspection on current main shows slash-containing model ids fail `LOW_CARDINALITY_VALUE_RE` and therefore become `unknown` when assigned to the span model attribute.

## Solution Assessment

Yes. The shared span identity helper is the narrowest repair surface for the reported attribute loss because model usage and model call spans share it, while changing `lowCardinalityAttr` itself would also broaden metric labels.
