## Summary

Keep open. Current main and the latest release still send slash-containing model ids through the low-cardinality fallback for `gen_ai.request.model` on spans, while this PR is a narrow span-only repair with sufficient live after-fix proof and no blocking correctness finding.

## What This Changes

The PR changes diagnostics-otel span identity assignment so `gen_ai.request.model` on model usage and model call spans stores the redacted full model id instead of the low-cardinality fallback.

## Best Possible Solution

Land the narrow span-attribute fix, preferably with one focused diagnostics-otel regression test if maintainers want source-level coverage before merge.

## Reproduction Assessment

Yes. Source inspection on current main shows slash-containing model ids fail the low-cardinality regex and are passed through that helper for `gen_ai.request.model` on model usage/model call spans; I did not run a live repro in this read-only review.

## Solution Assessment

Yes. Updating the shared span identity helper is the narrowest maintainable fix because model usage and model call spans share it, while changing `lowCardinalityAttr` itself would broaden metric labels.
