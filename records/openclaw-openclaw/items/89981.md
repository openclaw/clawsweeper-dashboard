## Summary

Keep open: current main and the latest release still collapse slash-containing model ids to `unknown`, while this PR is a narrow, source-backed diagnostics-otel fix with sufficient live proof. The remaining maintainer choice is whether to require a focused regression test before merge and whether opt-in GenAI span names should be handled in the same change.

## What This Changes

The PR changes the diagnostics-otel GenAI span identity helper so `gen_ai.request.model` records the redacted trimmed model id instead of the low-cardinality fallback.

## Best Possible Solution

Land the span-attribute fix with a focused slash-id regression test if maintainers want the invariant locked before merge; leave GenAI metric labels bounded.

## Reproduction Assessment

Yes. Current main's low-cardinality regex rejects `/`, and the span helper assigns `gen_ai.request.model` through that fallback, so a provider/model id such as `anthropic/claude-sonnet-4.6` becomes `unknown` by source inspection.

## Solution Assessment

Yes. The shared span identity helper is the narrowest repair surface because both model usage and model call spans use it, while changing `lowCardinalityAttr` itself would also broaden metric labels.
