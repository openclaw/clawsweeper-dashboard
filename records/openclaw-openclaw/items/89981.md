## Summary

Keep open for maintainer review: current main and the latest release still collapse slash-containing GenAI span model attributes to `unknown`, and this PR fixes the shared span-attribute path without changing the bounded metric-label path; the main remaining gap is focused regression coverage.

## What This Changes

The PR changes diagnostics-otel so GenAI span attributes write the redacted full model id while GenAI metric attributes continue to use low-cardinality normalization.

## Best Possible Solution

Land the focused span-attribute fix with one diagnostics-otel regression test for a slash-containing model id on spans and unchanged bounded GenAI metric labels.

## Reproduction Assessment

Yes. Source inspection shows current main and `v2026.6.6` pass slash-containing model ids through `lowCardinalityAttr`, whose allowlist rejects `/` and returns `unknown`; the diagnostics test harness can reproduce this with a model usage or model call event using a provider/model id.

## Solution Assessment

Yes for the reported trace-backend attribution bug. The shared GenAI span identity helper is the narrowest maintainable fix surface, and metrics keep their separate low-cardinality builders; maintainers can separately decide whether latest-semconv span names should also use the full redacted model id.
