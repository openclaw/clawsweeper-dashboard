## Summary

Keep open: current main and the latest release still route `gen_ai.request.model` through the low-cardinality helper, while this PR is a narrow diagnostics-otel fix with sufficient live proof and no blocking code-review findings.

## What This Changes

The PR changes diagnostics-otel span identity assignment so `gen_ai.request.model` on spans uses the redacted trimmed model id instead of `lowCardinalityAttr`.

## Best Possible Solution

Land the shared span-attribute fix, preferably with one focused diagnostics-otel regression test for slash-containing model ids on spans and unchanged bounded GenAI metric labels.

## Reproduction Assessment

Yes. Source inspection shows current main sends slash-containing model ids through `lowCardinalityAttr`, whose allowlist rejects `/`; the existing diagnostics-otel test harness can exercise the path with a model usage or model call event.

## Solution Assessment

Yes. The shared span identity helper is the narrowest maintainable fix for the requested attribute because both usage and model-call spans use it, while the metric label builders stay on their separate low-cardinality path.
