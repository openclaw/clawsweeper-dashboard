## Summary

Keep open: current main and v2026.6.6 still collapse slash-containing model ids to `unknown` on GenAI span attributes. This focused PR fixes the shared span helper and has sufficient live proof; the only remaining review gap is a focused regression test.

## What This Changes

The PR changes `diagnostics-otel` span identity assignment so `gen_ai.request.model` receives the redacted full model id while metric label builders continue using `lowCardinalityAttr`.

## Best Possible Solution

Land the focused span-attribute fix with metrics still bounded, preferably with one regression test for a slash-containing model id before merge.

## Reproduction Assessment

Yes. Source inspection on current main shows the allowlist excludes `/` and `assignGenAiSpanIdentityAttrs` uses that helper for `gen_ai.request.model`; I did not run tests because this review is read-only, so the status is source-reproducible.

## Solution Assessment

Yes. The shared span identity helper is the narrowest maintainable fix because model usage and model-call span paths use it, while GenAI metric label paths remain separate and bounded.
