Keep open: the diagnostics bug is valid and the contributor supplied real live proof, but the merge result with current main leaves the opt-in latest GenAI model-call span-name path still collapsing slash-containing model ids to `unknown`.

## What This Changes

The PR changes diagnostics-otel span identity assignment to write the redacted full model id to `gen_ai.request.model` instead of the low-cardinality fallback.

## Best Possible Solution

Use one span-model identity path for GenAI span attributes and latest GenAI span names, while leaving GenAI metric labels on the existing low-cardinality path.

## Reproduction Assessment

Yes. Source inspection shows current main sends slash-containing model ids through `lowCardinalityAttr`, whose allowlist rejects `/`, and the PR body supplies a real gateway/Langfuse before-after proof for the default span attribute path.

## Solution Assessment

No. The attribute change is the right core repair direction, but the best fix also needs to cover the current-main latest GenAI span-name path or intentionally document and test why span names stay bounded.
