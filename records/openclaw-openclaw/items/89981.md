## What This Changes

The PR changes diagnostics OpenTelemetry span identity assignment so span attributes retain the redacted full model id instead of applying the low-cardinality fallback.

## Best Possible Solution

Land the focused span-attribute fix with metrics still using low-cardinality labels, preferably with one diagnostics-otel regression test for a slash-containing model id before merge.

## Reproduction Assessment

Yes. Source inspection shows current main sends slash-containing model ids through `lowCardinalityAttr`, whose allowlist rejects `/` and returns `unknown`; the existing diagnostics test harness can reproduce this by emitting a model usage or model call event with a provider/model id.

## Solution Assessment

Yes. The shared span identity helper is the narrowest maintainable fix surface because model usage and model call spans both use it, while metric label builders keep their separate low-cardinality path.
