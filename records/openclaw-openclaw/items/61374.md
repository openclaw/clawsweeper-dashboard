## What This Changes

The PR adds Slack block-action wake/thread routing by passing `threadTs` into session-key resolution, adding a wake hint to system-event enqueueing, and covering the path with focused tests.

## Best Possible Solution

Keep the shipped Slack-owned `requestHeartbeat` plus thread-aware session-resolution implementation; any broader end-to-end Slack button concerns should continue in the canonical Slack interaction tracker rather than this stale branch.

## Reproduction Assessment

## Solution Assessment

Yes for closing this PR: current main solves the requested behavior with a narrower Slack-owned heartbeat call after a successful enqueue, without adding the PR's now-obsolete generic `enqueueSystemEvent` wake option.
