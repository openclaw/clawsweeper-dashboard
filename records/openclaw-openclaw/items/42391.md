## Summary

Keep open: current main still lacks the requested structured numeric progress metrics on the public plugin hook event contract, and defining those semantics is a plugin API/product decision rather than conservative cleanup.

## What This Changes

Request adds optional numeric progress metrics such as step index, step total, percent, and ETA to existing plugin hook payloads for live progress-card rendering.

## Best Possible Solution

Define and document one stable optional numeric progress metrics contract, then emit it consistently through the relevant hook paths after maintainers settle the semantics.

## Reproduction Assessment

Not applicable: this is a feature/API contract request rather than broken documented behavior. Source inspection on current main confirms the requested metrics fields are absent from the hook event types and emitters.

## Solution Assessment

Unclear: enriching existing hook payloads is plausible and backward-compatible, but maintainers should first decide whether numeric progress belongs on hook events, progress drafts, diagnostics, or a shared contract.
