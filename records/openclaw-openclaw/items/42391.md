## Summary

Keep this open: current main and v2026.6.6 still do not expose structured step or percent metrics on the requested plugin hook events. The request fits OpenClaw's plugin-API direction, but it needs a maintainer decision on authoritative progress semantics before a safe implementation.

## What This Changes

Request to add optional structured progress metrics to `before_tool_call`, `after_tool_call`, and `agent_end` hook payloads for plugin-rendered live progress cards.

## Best Possible Solution

Define a narrow optional plugin-facing progress metric contract sourced only from authoritative producer events, with docs and tests for monotonicity, terminal success/failure behavior, and absent values when progress is unknown.

## Reproduction Assessment

Yes, at source-contract level: current main and v2026.6.6 define the relevant hook event types and producers without structured progress metrics. A live Feishu card is not required to verify that the requested API field is missing.

## Solution Assessment

No, not exactly as proposed: enriching hook payloads is plausible, but generic tool-call lifecycle events are not authoritative enough for totals or percentages. The safer path is producer-owned metrics with values omitted when progress is unknown.
