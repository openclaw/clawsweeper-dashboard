## Summary

Keep open: current main still lacks the requested structured metrics on plugin hook events, and adding them would define a new public hook contract that needs maintainer API design.

## What This Changes

The issue requests optional structured progress metrics on `before_tool_call`, `after_tool_call`, and `agent_end` hook payloads for plugin-rendered live progress cards.

## Best Possible Solution

Define, document, and version a stable optional hook metrics contract, then emit it consistently from shared tool and lifecycle hook paths with explicit monotonic and terminal-state semantics.

## Reproduction Assessment

Not applicable: this is a feature/API request rather than broken documented behavior. Source inspection confirms current main lacks the requested metrics field on the relevant hook types and emitters.

## Solution Assessment

Unclear: enriching hook payloads is plausibly backward-compatible, but maintainers need to settle the public field shape and progress semantics before implementation.
