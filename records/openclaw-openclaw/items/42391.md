## Summary

Keep open: current main and the latest release still lack the requested structured numeric progress metrics on the relevant plugin hook events, and the remaining work is a public plugin API/product-semantics decision rather than conservative cleanup.

## What This Changes

The issue requests optional structured progress metrics on `before_tool_call`, `after_tool_call`, and `agent_end` hook payloads so plugin-based live progress cards can render numeric completion.

## Best Possible Solution

Define one documented optional numeric progress metrics contract for plugin hooks, then emit it consistently through the relevant tool and agent-end paths after maintainers settle the semantics.

## Reproduction Assessment

Not applicable as a broken-behavior bug: this is a plugin API feature request. Source inspection on current main and `v2026.5.20` confirms the requested metric fields are absent from the hook event types and emitters.

## Solution Assessment

Unclear: enriching existing hook payloads is plausible and backward-compatible, but maintainers should first decide the numeric progress semantics across planning, retries, parallel tools, failures, and terminal state.
