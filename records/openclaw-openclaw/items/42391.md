## Summary

Keep open: current main still lacks structured numeric progress metrics on plugin hook events, and the remaining work is a public plugin/API contract plus progress-semantics decision rather than conservative cleanup.

## What This Changes

The issue asks OpenClaw to add optional `metrics` fields (`stepIndex`, `stepTotal`, `percent`, `etaSec`) to `before_tool_call`, `after_tool_call`, and `agent_end` hook payloads for plugin-rendered live progress cards.

## Best Possible Solution

Define and document one stable optional numeric progress contract, align it with shared agent progress/event surfaces, then emit it consistently through all relevant hook paths with explicit monotonic and terminal-state rules.

## Reproduction Assessment

Not applicable: this is a feature/API contract request rather than broken documented behavior. Source inspection confirms current main does not expose the requested metrics fields on the relevant hook events.

## Solution Assessment

Unclear: enriching existing hook payloads is plausibly backward-compatible, but maintainers should first decide whether the canonical numeric progress contract belongs on hook events, agent event streams, or both.
