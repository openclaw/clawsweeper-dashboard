## Summary

Keep this open: current main and v2026.6.6 still do not expose structured step or percent metrics on the requested plugin hook events. The request fits OpenClaw's plugin API direction, but it needs a maintainer decision on authoritative progress semantics before a safe implementation.

## What This Changes

The issue requests optional structured progress metrics on existing plugin hook payloads so live progress card plugins can render numeric progress.

## Best Possible Solution

Define a narrow optional plugin-facing progress metric contract sourced only from authoritative producer events, with values omitted when progress is unknown.

## Reproduction Assessment

Yes, at source-contract level: current main defines the requested hook event types and producers without the requested structured progress fields. A live Feishu card is not required to verify that the public API field is missing.

## Solution Assessment

No, not exactly as proposed: enriching hook payloads is plausible, but generic tool-call lifecycle hooks are not authoritative enough for totals or percentages. The safer path is producer-owned metrics, with values omitted when progress is unknown.
