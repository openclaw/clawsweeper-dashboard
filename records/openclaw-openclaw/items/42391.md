## Summary

Keep open: current main still lacks the requested structured numeric progress metrics on plugin hook events, and the remaining work is a public plugin/API contract plus progress-semantics decision rather than conservative cleanup.

## What This Changes

The issue requests optional structured progress metrics on existing plugin hook payloads so plugin live progress cards can render x/y or percent completion.

## Best Possible Solution

Define and document a stable optional numeric progress contract, preferably aligned with the shared agent event layer, then emit it consistently through the public hook/API surfaces with explicit monotonic and terminal-state rules.

## Reproduction Assessment

Not applicable: this is a feature/API request rather than broken documented behavior. Source inspection confirms current main lacks the requested metrics field on the relevant hook types and emitters.

## Solution Assessment

Unclear: enriching existing hook payloads is plausibly backward-compatible, but maintainers should decide whether the canonical numeric contract belongs on hook events, agent event streams, or both before implementation.
