## Summary

Keep open: current main and v2026.6.10 still omit structured progress metrics from the requested hook events. The remaining work is a public Plugin SDK progress-semantics decision, not stale cleanup.

## What This Changes

Request to add optional structured progress metrics such as step index, total, percent, and ETA to before_tool_call, after_tool_call, and agent_end hook payloads.

## Best Possible Solution

Define a producer-owned optional progress metric contract, update the hook types, every runtime producer, docs, and tests together, and omit unknown values instead of guessing percentages.

## Reproduction Assessment

Yes, at source-contract level: current main defines and emits the requested hook events without structured progress metric fields. A live Feishu card run is not needed to verify the missing public API field.

## Solution Assessment

Unclear as proposed: enriching hook payloads is plausible, but step totals, percent, ETA, and terminal success/failure semantics need maintainer agreement. The safer shape is producer-owned metrics with absent values when progress is unknown.
