## Summary

Keep open: current main still lacks the requested structured numeric progress metrics on plugin hook events, and the Feishu progress-card context points to a real plugin API gap rather than a cleanup-close candidate.

## What This Changes

The issue requests optional `metrics` fields such as `stepIndex`, `stepTotal`, `percent`, and `etaSec` on existing plugin hook events for live progress card rendering.

## Best Possible Solution

Define one documented optional numeric progress metrics contract for plugin hooks, then emit it consistently through the relevant tool and agent-end paths after maintainers settle the semantics.

## Reproduction Assessment

Not applicable: this is a plugin API feature request rather than broken documented behavior. Source inspection on current main confirms the requested metric fields are absent from the hook event types and emitters.

## Solution Assessment

Unclear: enriching existing hook payloads is plausible and backward-compatible, but maintainers should first decide the numeric progress semantics across planning, retries, parallel tool calls, failures, and terminal state.
