## Summary

Keep open: current main still exposes only textual/status-style hook data, and the requested structured progress metrics are a new public plugin SDK contract rather than an already-implemented or stale cleanup item.

## What This Changes

Request optional structured progress metrics on `before_tool_call`, `after_tool_call`, and `agent_end` hook payloads so plugins can render numeric live progress cards.

## Best Possible Solution

Define a documented optional progress metrics contract in the plugin SDK, then emit it consistently from the shared hook paths after agreeing on monotonic step and terminal-state semantics.

## Reproduction Assessment

Not applicable as a feature request. Source inspection confirms current hook payloads lack the requested metrics fields, but there is no broken existing documented contract to reproduce.

## Solution Assessment

Unclear. The optional `metrics` shape is plausibly backward-compatible, but the best implementation needs maintainer agreement on step and percent semantics before changing the public plugin hook contract.
