## Summary

Keep this open: current main and the latest release still do not expose step or percent metrics on the requested plugin hook events. The request is a valid Plugin SDK extension point, but it needs maintainer agreement on authoritative progress semantics before a fix PR should add public API fields.

## What This Changes

The issue requests optional structured progress metrics on existing hook payloads so plugin-rendered live progress cards can show step counts or percentages.

## Best Possible Solution

Define a narrow optional plugin-facing progress metric contract sourced from authoritative producer events, update SDK docs/types/tests, and omit unknown values rather than deriving misleading percentages.

## Reproduction Assessment

Yes, at source-contract level: current main defines and emits the requested hook events without structured progress metric fields. A live Feishu card is not needed to verify that the public API field is missing.

## Solution Assessment

No, not exactly as the generic step-counting proposal. Enriching plugin hook payloads is the right boundary, but the metrics should come from authoritative producer-owned progress facts and omit unknown values instead of guessing totals or percentages.
