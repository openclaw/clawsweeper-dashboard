## Summary

Current main and the latest release still omit structured progress from the public hook contract, so the plugin capability remains missing. The issue should stay open for a maintainer product decision because OpenClaw cannot truthfully derive total steps, percentage, or ETA from generic tool-call counts across arbitrary agent runs.

## What This Changes

Request optional structured step and percentage progress metrics on agent hook events so plugins can render numeric live progress.

## Best Possible Solution

Define a small optional producer-supplied progress type with explicit semantics, then forward it through generic hooks only when the active harness or tool owner has authoritative progress facts; keep unknown totals, percentages, and ETA absent rather than synthesizing them.

## Reproduction Assessment

Yes at the source-contract level: current main and v2026.6.5 define the named hook payloads without structured progress metrics. This is a missing capability rather than a regression in an existing documented contract.

## Solution Assessment

No, not exactly as proposed. Passing authoritative progress through the Plugin SDK is a suitable direction, but deriving totals, percentages, and ETA from generic tool-call lifecycle events would create false precision and inconsistent behavior.
