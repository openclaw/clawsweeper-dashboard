## Summary

Keep open: current main still lacks the requested structured numeric progress metrics on plugin hook payloads, and the remaining work is a public plugin API and progress-semantics decision rather than conservative cleanup.

## What This Changes

The issue requests optional structured progress metrics on existing plugin hook payloads so plugin-based live progress cards can render numeric completion.

## Best Possible Solution

Define and document one stable optional numeric progress contract, align it with existing progress draft/event surfaces, then emit it consistently through the relevant hook paths with focused contract tests.

## Reproduction Assessment

Not applicable: this is a feature/API contract request rather than broken documented behavior. Current-main source inspection confirms the requested metrics fields are not present on the relevant hook events.

## Solution Assessment

Unclear: enriching existing hook payloads is plausibly backward-compatible, but maintainers should decide whether numeric progress belongs on hook events, existing agent progress streams, or a shared contract before implementation.
