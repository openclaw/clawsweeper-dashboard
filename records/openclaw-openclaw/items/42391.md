## Summary

Keep open: current main and the latest shipped tag still do not expose structured numeric progress metrics on plugin hook payloads, and the remaining work is a plugin API/product-semantics decision rather than a narrow bug fix.

## What This Changes

The issue requests optional structured progress metrics on `before_tool_call`, `after_tool_call`, and `agent_end` hook events for plugin-rendered live progress cards.

## Best Possible Solution

Define one documented optional numeric progress metrics contract for plugin hooks, then emit it consistently through the tool and agent-end paths with focused contract and integration coverage.

## Reproduction Assessment

No high-confidence runtime bug reproduction is needed because this is a feature/API request. The source-level check is high-confidence: current main and `v2026.5.20` lack the requested metrics fields on the hook types and emitters.

## Solution Assessment

Unclear: enriching existing hook payloads is plausible and backward-compatible, but maintainers should first settle numeric progress semantics across planning, retries, parallel tools, failures, cancellation, and terminal state.
