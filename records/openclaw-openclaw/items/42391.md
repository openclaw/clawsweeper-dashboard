## Summary

Keep open: current main still does not expose the requested structured numeric progress metrics on the plugin hook event contract, and the next step is an API/product decision rather than conservative cleanup.

## What This Changes

The issue asks OpenClaw to add optional numeric progress metrics to existing `before_tool_call`, `after_tool_call`, and `agent_end` plugin hook payloads.

## Best Possible Solution

Define and document one stable optional progress metrics contract, then emit it consistently through the hook paths after maintainers settle the semantics.

## Reproduction Assessment

Not applicable: this is a feature/API contract request rather than broken documented behavior. Current-main source inspection confirms the requested metrics fields are absent from the relevant hook event types and emitters.

## Solution Assessment

Unclear: enriching existing hook payloads is plausible and backward-compatible, but maintainers should first decide whether numeric progress belongs on hook events, existing progress-draft streams, diagnostics, or a shared contract.
