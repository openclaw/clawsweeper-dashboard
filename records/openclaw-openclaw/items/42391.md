## Summary

Keep open: current main and the latest release still do not expose the requested structured progress metrics on the relevant plugin hook events, and defining monotonic numeric progress is a public plugin API/product-semantics decision.

## What This Changes

The issue requests optional structured numeric progress metrics on existing plugin hook payloads for live progress-card renderers.

## Best Possible Solution

Define one documented optional numeric progress metrics contract for plugin hooks, then emit it consistently through the tool and agent-end paths after maintainers settle the semantics.

## Reproduction Assessment

Not applicable as a broken-behavior bug, but source inspection gives a high-confidence check: current main and v2026.5.20 lack the requested metrics fields in the hook event types and emitters.

## Solution Assessment

Unclear: enriching existing hook payloads is plausible and backward-compatible, but maintainers should first define numeric progress semantics across planning, retries, parallel tools, failures, cancellation, and terminal state.
