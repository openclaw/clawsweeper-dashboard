## Summary

Keep open: current main and v2026.5.20 still lack the requested structured progress metrics on the plugin hook payloads, and adding monotonic numeric progress is a public plugin API/product-semantics decision.

## What This Changes

Expose optional numeric progress metrics on before_tool_call, after_tool_call, and agent_end hook payloads for plugin-rendered live progress cards.

## Best Possible Solution

Define one documented optional numeric progress metrics contract for plugin hooks, then emit it consistently through the tool and agent-end paths after maintainers settle the semantics.

## Reproduction Assessment

Not applicable as a broken-behavior bug, but the source-level check is high-confidence: current main and v2026.5.20 lack metrics fields on the requested hook types and emitters.

## Solution Assessment

Unclear: enriching existing hook payloads is plausible and backward-compatible, but maintainers should first define numeric progress semantics across planning, retries, parallel tools, failures, cancellation, and terminal state.
