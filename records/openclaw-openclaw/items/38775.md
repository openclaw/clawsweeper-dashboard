## Summary

Keep open: current main has reactive auth-profile rotation, cooldown/probe handling, and Anthropic usage display, but it does not implement proactive multi-account routing by utilization/headroom. The related open items overlap, but this issue preserves a specific Anthropic OAuth usage-routing design that is not fully superseded.

## What This Changes

The issue requests usage-aware routing across multiple auth profiles so OpenClaw can choose accounts by utilization/headroom instead of only reacting to cooldowns.

## Best Possible Solution

Design an opt-in provider-profile routing policy that reuses or extends the provider-usage surface for per-profile Anthropic headroom while preserving the current default order/cooldown behavior.

## Reproduction Assessment

Not applicable: this is a feature request, not a report of broken existing behavior. Source inspection confirms current main lacks the requested usage-aware routing mode, while docs describe the existing reactive fallback/cooldown behavior.

## Solution Assessment

No automated narrow fix is appropriate yet. The direction is plausible, but the scoring policy, config shape, cache semantics, and relation to the broader provider-pool routing issues need maintainer design before implementation.
