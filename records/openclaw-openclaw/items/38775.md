## Summary

Current main still does not implement usage-aware load balancing for multi-account auth profiles. Anthropic usage snapshots are available for status/reporting, and OpenAI Codex has a reactive WHAM cooldown probe after failures, but request-time auth-profile selection still uses explicit/configured order, credential type, last-used round-robin, and cooldown state rather than utilization headroom or proactive circuit breakers.

## What This Changes

The issue requests opt-in usage-aware routing for multi-account auth profiles, especially Anthropic OAuth profiles, so OpenClaw can choose accounts by remaining usage headroom before hitting provider rate limits.

## Best Possible Solution

Define an opt-in generic auth-profile routing policy that preserves current session-sticky/default behavior, lets provider plugins expose per-profile usage snapshots, supports proactive thresholds with fallback for profiles without usage data, records routing decisions/status, and updates config schema, docs, and focused auth/provider-usage tests before enabling Anthropic OAuth usage-aware balancing.
