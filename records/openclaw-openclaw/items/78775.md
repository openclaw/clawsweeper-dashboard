## What This Changes

The issue reports `openclaw update`/`gateway update.run` staging npm installs under `/usr/local` instead of the nvm-owned npm prefix, causing EACCES on nvm-only macOS installs.

## Best Possible Solution

Keep the current package-root-owned npm resolution and staging behavior; handle any setup without `<prefix>/bin/npm` as a separate follow-up only if a concrete repro appears.

## Reproduction Assessment

No, not as a current-main failure: the old EACCES path is described clearly, but current source and regression coverage route the reported nvm-prefix layout to the owning npm/global root.

## Solution Assessment

Yes: using the npm executable and global root implied by the resolved package root is the narrow maintainable fix for this staging bug; the missing-prefix-local-npm edge should remain separate unless someone supplies a fresh repro.
