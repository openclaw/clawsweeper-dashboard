## Summary

Keep this open: current main still has the npm staging target path that can follow the command-reported global root, and https://github.com/openclaw/openclaw/pull/78795 is an open linked fix that should close this issue only after merge.

## What This Changes

The issue reports that `openclaw update` / `gateway update.run` can stage npm package updates under `/usr/local/lib/node_modules` instead of the nvm-owned npm prefix for nvm-installed OpenClaw.

## Best Possible Solution

Land the narrow npm target-resolution fix from https://github.com/openclaw/openclaw/pull/78795 after required CI and maintainer review, then let the merged PR close this issue.

## Reproduction Assessment

Yes, source-level. Current main stages npm updates under the resolved install target's global root, and the issue includes a concrete `global install stage` EACCES log showing that target resolved to `/usr/local/lib/node_modules`.

## Solution Assessment

Yes for the linked PR direction. The maintainable fix is to keep npm staging anchored to the resolved package root's valid npm-prefix layout while preserving the existing pnpm custom-root path; the issue itself should remain open until that PR merges.
