## What This Changes

The issue reports `openclaw update` / `gateway update.run` staging under `/usr/local` instead of the nvm-owned npm prefix, causing EACCES on nvm-only macOS installs.

## Best Possible Solution

Land one narrow updater fix that keeps npm staging on the package-owning prefix while preserving pnpm custom-root behavior, then close this issue through the merged PR.

## Reproduction Assessment

Yes, source-level. Current main can pass a command-reported npm global root into staged install preparation, and the issue includes the matching `/usr/local/lib/node_modules/.openclaw-update-stage-*` EACCES log; I did not live-run macOS+nvm.

## Solution Assessment

Yes, the best fix shape is the narrow resolver change already proposed by the linked PRs: prefer a valid package-root-derived npm global root for npm staging while preserving the existing pnpm custom-root path.
