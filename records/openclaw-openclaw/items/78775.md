Keep open. Current main still has a source-level path where npm update staging follows the resolved npm global root into `.openclaw-update-stage-*`, and https://github.com/openclaw/openclaw/pull/78778 remains an open linked fix PR with closing syntax, so this issue should not be closed as cleanup before that PR is resolved.

## What This Changes

The issue reports that `openclaw update` / `gateway update.run` can stage under `/usr/local` instead of the nvm prefix that owns the running OpenClaw install, causing EACCES.

## Best Possible Solution

Resolve the linked fix path by rebasing or replacing it with a narrow npm package-root target selection change that preserves pnpm custom-root behavior and adds regression coverage for nvm-owned staging.
