## Summary

Keep open. The configurable WhatsApp retry surface is still not on current main, but this branch has a blocking retry-classifier gap, lacks after-fix real WhatsApp behavior proof, and needs maintainer review now that durable outbound queue recovery has shipped.

## What This Changes

The PR adds WhatsApp channel/account retry config and wraps text, media, and poll sends with a retry helper, plus schema, generated config docs, and retry tests.

## Best Possible Solution

If maintainers want immediate retry as a durable-queue complement, rebase it onto current main, keep it plugin-owned, explicitly retry safe pre-send/readiness failures, update the current config metadata/docs surface, and prove a real WhatsApp resend.

## Reproduction Assessment

Yes for the review finding via source inspection: current main throws a plain listener-unavailable Error before sending, while the PR classifier only retries code/name allowlists. I did not run a live WhatsApp repro in this read-only review.

## Solution Assessment

No, not as proposed. Configurable retry may still be useful, but it should be rebased onto the durable queue design and cover safe listener-readiness failures before enabling a default retry surface.
