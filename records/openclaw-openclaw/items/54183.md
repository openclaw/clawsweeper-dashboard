## What This Changes

The branch adds WhatsApp retry config, retry helpers, send-path wrappers, generated config docs, and tests for text, media, and poll outbound sends.

## Best Possible Solution

If maintainers still want configurable immediate retry, rebase it onto current main as a WhatsApp-plugin-owned complement to the durable queue, document the boundary, and classify only narrow pre-send/listener-readiness failures.

## Reproduction Assessment

Yes. From the proposed helper, a plain Error("No active WhatsApp Web listener") or Error("Connection Closed") has no allowlisted code/name and will return false from the retry classifier.

## Solution Assessment

No, not as-is. The safer path is a fresh current-main rebase that removes stale generated drift and decides whether this retry surface still belongs after the shipped durable queue.
