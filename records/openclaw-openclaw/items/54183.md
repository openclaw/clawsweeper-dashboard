## Summary

Keep open: current main still lacks the configurable WhatsApp send retry surface, and the PR remains adjacent rather than obsolete, but the patch has a blocking retry-classifier gap and still lacks required real behavior proof.

## What This Changes

The PR adds configurable WhatsApp outbound send retry for text, media, and poll sends, along with config schema/generated docs, shared retry helpers, and retry tests.

## Best Possible Solution

If maintainers want this surface, rebase it onto current main as a WhatsApp-plugin-owned complement to the durable queue, retry only safe pre-send/readiness failures, and document how it differs from queued recovery.

## Reproduction Assessment

Yes for the review finding. Source inspection shows current main throws `No active WhatsApp Web listener` as a plain message-only Error, while the PR classifier only retries allowlisted codes or names.

## Solution Assessment

No, not as proposed. Configurable retry may still be useful, but it should be rebased onto the durable queue design and classify only safe pre-send/readiness failures before adding a default config surface.
