## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Avoid retrying whole multi-send WhatsApp deliveries:** `extensions/whatsapp/src/channel.ts:132-137`
  - body: `wrapSendMessageWhatsApp` retries the entire logical send. `sendMessageWhatsApp` can issue more than one platform send for media cases such as voice plus visible caption text, so if the later send throws a retryable error after the first part was accepted, the retry resends the first part and duplicates visible WhatsApp content. Retry only before platform send starts or per part with receipt/unknown-send handling.
  - confidence: 0.82
- **[P1] Make WhatsApp send retry opt-in before changing defaults:** `extensions/whatsapp/src/outbound-retry.ts:15-18`
  - body: These defaults enable three immediate retry attempts for every upgraded WhatsApp user without a config change. Current main already has a durable delivery queue/reconnect drain, and WhatsApp does not advertise unknown-send reconciliation, so default-on retry is a compatibility and duplicate-delivery decision that needs maintainer approval or an opt-in default.
  - confidence: 0.78

Regenerate from current main and commit only the tracked current artifacts.
  - confidence: 0.9
