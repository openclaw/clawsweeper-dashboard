## Best Possible Solution

Keep the readable projection idea, but gate it on successful current-source delivery or an equivalent matching tool-result check, preserve search filtering, add failure/off-target coverage, and require real Control UI/WebChat proof before merge.

## Solution Assessment

No, not as written. Projecting readable text in the UI is a reasonable narrow fix, but it should require the same successful/current-source delivery semantics as delivery mirrors or an equivalent matching tool-result check.
