## Summary

## Best Possible Solution

Keep the readable UI projection idea, but align it with delivered current-source mirror semantics, apply search filtering to projected bubbles, cover failed/off-target/search cases, and require real Control UI/WebChat proof.

## Reproduction Assessment

Yes at source level: current `normalizeMessage` classifies assistant tool-call-only messages as `toolResult`, and current `buildChatItems` has no projection path for `message(action=send)` text. I did not run a live Control UI reproduction in this read-only review.

## Solution Assessment

No, not as written. A UI projection is a reasonable narrow fix, but it should require successful current-source delivery or equivalent mirror semantics and it must participate in chat search filtering.
