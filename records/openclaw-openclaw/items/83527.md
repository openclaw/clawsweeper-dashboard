The PR adds a Control UI `buildChatItems` projection that turns unmirrored `message(action=send)` tool-call text into synthetic assistant bubbles, plus unit tests for projection and duplicate suppression.

## Best Possible Solution

Keep the readable UI projection idea, but gate it on successful current-source delivery or an equivalent matching tool-result check, add failure/off-target coverage, and require real Control UI/WebChat proof before merge.

## Reproduction Assessment

I did not run a live Control UI reproduction in this read-only review.

## Solution Assessment

No, not as written. Projecting readable text in the UI is a reasonable narrow fix, but it should require the same successful/current-source delivery semantics as delivery mirrors or an equivalent matching tool-result check.
