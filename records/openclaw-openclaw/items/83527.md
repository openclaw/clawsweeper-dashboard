Full review comments:

- **[P2] Project before hiding tool-call-only turns:** `ui/src/ui/chat/build-chat-items.ts:622-634`
  - body: `normalizeMessage` reclassifies assistant messages that contain only tool-call blocks as `toolResult`, but this projection runs after the `!showToolCalls` skip. With readable mode enabled, the exact `message(action=send)` turn this PR targets is skipped before the synthetic assistant bubble can be added; move the projection before this guard or exempt raw assistant message sends and add `showToolCalls: false` coverage.
  - confidence: 0.94
- **[P2] Gate bubbles on a successful send result:** `ui/src/ui/chat/build-chat-items.ts:663-673`
  - body: The new bubble is created from tool-call arguments alone, without checking the paired `toolResult`. If a send fails, is aborted, or returns an error, Control UI will still show the text as a normal assistant reply even though the user did not receive it; pair by `toolCallId` and require a successful delivery result or mirror before projecting.
  - confidence: 0.9
- **[P3] Apply search filtering to projected bubbles:** `ui/src/ui/chat/build-chat-items.ts:663-673`
  - body: The projection pushes synthetic bubbles before the existing search filter runs. When chat search is active, an unrelated `message(action=send)` turn can leave a projected bubble in the results even though the source message is skipped; apply the same search predicate before injecting the bubble.
  - confidence: 0.82
