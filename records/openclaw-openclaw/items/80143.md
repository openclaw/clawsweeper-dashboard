## What This Changes

The PR adds browser config normalization so an existing-session default profile without its own endpoint inherits the top-level `browser.cdpUrl`, with regression tests for inheritance and profile-local precedence.

## Best Possible Solution

Land this narrow resolver fix if maintainers agree with preserving the selected existing-session profile, then let the linked issue close after merge.

## Reproduction Assessment

Yes. A high-confidence source reproduction on current main is `resolveBrowserConfig({ defaultProfile: "user", cdpUrl: "http://127.0.0.1:9222" })` followed by `resolveProfile`, which selects the built-in existing-session `user` profile with an empty endpoint and therefore reaches the Chrome MCP fallback path.

## Solution Assessment

Yes. Applying the normalized top-level endpoint only to the selected existing-session default profile when it lacks an explicit endpoint is a narrow browser-plugin fix that preserves profile-local precedence and avoids changing session caching, gateway routes, or Chrome MCP behavior.
