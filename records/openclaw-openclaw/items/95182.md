## Summary

Keep open: the latest head fixes the prior nullable-clear crash, but it still lets mixed `title`/`label` clear-and-set patches silently choose one side, and the public/durable `title` migration remains compatibility-sensitive without real behavior proof or maintainer acceptance of the naming contract.

## What This Changes

## Best Possible Solution

If maintainers want the `title` migration, land it as an explicit staged contract: mirror `title` and `label`, reject mixed clear/set pairs, preserve existing label clients, and require redacted real `/name` plus gateway round-trip proof before merge.

- **[P1] Reject mixed title/label clear-and-set patches:** `src/gateway/sessions-patch.ts:374-383`
