## Summary

Keep open: the PR targets a real source-backed compaction bug, but the latest clamp still undercounts a legitimate latest-usage plus post-usage-tail case that current main explicitly guards, and the external PR still lacks inspectable real behavior proof.

## What This Changes

The PR adds bounded stale-usage handling to preflight transcript token estimation and adds gateway `models.list` assertions for `openai/gpt-5.5` and `openai/gpt-5.5-mini`.

## Best Possible Solution

Revise the estimator to distinguish stale pre-compaction usage from legitimate latest usage plus trailing transcript pressure, preserve the conservative active-tail floor, keep the model-list assertion, and add after-fix real session or gateway proof.

## Reproduction Assessment

Yes for the PR regression at source level: current main has a combined-tail test that constructs the legitimate high-usage-plus-tail case and expects compaction. I did not run a live Discord/Pi reproduction of the production report.

## Solution Assessment

No. Bounding stale transcript usage is the right direction, but this implementation is still too broad because it can discard legitimate latest-usage pressure before adding only a small tail estimate.
