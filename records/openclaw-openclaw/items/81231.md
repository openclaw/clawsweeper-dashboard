The PR bounds stale transcript usage in preflight memory-compaction token estimation and adds regression coverage plus a gateway model-list assertion for new OpenAI IDs.

## Best Possible Solution

Revise the estimator so stale historical usage is bounded without losing legitimate latest usage plus post-usage tail pressure, keep the model-list assertion, and require real runtime proof before merge.

## Solution Assessment

No. Bounding stale usage is the right direction, but this implementation still replaces the latest prompt usage with a recent-message estimate when the clamp fires, so it can skip needed preflight compaction in an existing guarded case.
