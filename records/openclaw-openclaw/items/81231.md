## Summary

Keep open: the PR is a focused fix for a real source-backed compaction bug, but the revised clamp can still undercount valid latest-usage-plus-tail sessions, and the external real-behavior proof gate is still not satisfied.

## What This Changes

The branch bounds transcript-derived preflight token estimates in `agent-runner-memory.ts` and adds memory-compaction plus gateway `models.list` tests for new OpenAI allowlist IDs.

## Best Possible Solution

Keep the response-level model-list assertion, revise the estimator so only stale pre-compaction usage is capped while valid latest usage plus tail remains conservative, and add a regression using real message shapes that fails on current main.

## Reproduction Assessment

Yes at source level: current main reads the last nonzero transcript usage and uses it as a floor when `totalTokensFresh` is false, which matches the linked bug report. I did not run a live Discord/Pi reproduction in the read-only checkout.

## Solution Assessment

No. Bounding stale usage is the right direction, but this implementation can still discard legitimate latest usage pressure and the new regression fixture does not prove the real clamp path.
