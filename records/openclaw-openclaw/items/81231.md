## Summary

Keep open: this is a focused bug-fix PR, but the new clamp appears to drop legitimate post-usage tail pressure that current tests intentionally preserve, so it should not merge as written.

## What This Changes

The PR clamps transcript-derived preflight token estimates against recent message estimates and adds a gateway `models.list` allowlist test for OpenAI `gpt-5.5` IDs.

## Best Possible Solution

Revise the estimator so stale pre-compaction usage can be bounded without discarding legitimate latest-usage plus post-usage tail pressure, then add a focused stale-after-compaction regression test alongside the gateway model-list assertion.

## Reproduction Assessment

Yes for the PR regression at source level: the existing post-usage tail-pressure test gives a concrete path where the new clamp can suppress expected compaction. I did not run a live Discord/Pi reproduction of the linked production bug.

## Solution Assessment

No. The requested direction is appropriate, but this implementation is too broad because it can cap a legitimate latest usage record plus trailing tool output; the safer fix should distinguish stale pre-compaction usage from real tail pressure.
