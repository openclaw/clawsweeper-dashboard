## Summary

Keep open. This is a plausible focused feature PR, but it needs changes before merge because the new handler pulls the subagent spawn runtime into the eager slash-command path and the provided screenshot only proves spawn/list visibility, not final completion outside the watchdog.

## What This Changes

The PR adds a standard/native `/long [background|desktop] <task>` command, handler wiring, docs, and unit tests for spawning detached background subagents or passing desktop mode through.

## Best Possible Solution

Land the shortcut only after preserving the lazy subagent spawn boundary, adding convincing after-fix completion proof, and getting maintainer acceptance for the desktop pass-through semantics.

## Reproduction Assessment

Not applicable as a feature PR rather than a bug report. The review path is source, diff, CI metadata, and real-behavior proof inspection; the screenshot verifies spawn/list behavior but not final completion outside the watchdog.

## Solution Assessment

Unclear as-is. Reusing `spawnSubagentDirect` is the right implementation class, but the patch should keep that runtime lazy and the feature needs stronger real-environment proof before it is merge-ready.
