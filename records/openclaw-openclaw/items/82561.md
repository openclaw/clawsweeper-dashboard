Full review comments:

- **[P1] Honor sessions_spawn policy before spawning:** `src/auto-reply/reply/commands-long.ts:66-67`
  - body: This command reaches `spawnSubagentDirect` after only `rejectUnauthorizedCommand`, but current `sessions_spawn` availability is governed by the session's effective tool policy and can be removed by profile, channel, provider, sandbox, or per-agent policy. As written, any sender allowed to use text commands gets a new way to create detached agents even when `sessions_spawn` is intentionally unavailable, so route this through the existing tool-policy boundary or add an equivalent explicit gate before spawning.
  - confidence: 0.9
- **[P2] Import string helpers from the current package:** `src/auto-reply/reply/commands-long.ts:2`
  - body: Current main has no `src/shared/string-coerce.*`; nearby code imports `normalizeOptionalString` from `@openclaw/normalization-core/string-coerce`. This stale relative import will fail after rebase unless it is updated to the current helper package.
  - confidence: 0.95
- **[P2] Preserve multiline task payloads:** `src/auto-reply/reply/commands-long.ts:27`
  - body: The handler parses `task` from `commandBodyNormalized`, but the registry normalizer drops multiline tails for registered commands other than `skill` and `reset`. After `/long` is registered, a multi-line delegated task will silently spawn with only the first line, so parse from the raw command body or teach the normalizer to preserve `/long` payloads with coverage.
  - confidence: 0.92
- **[P2] Pass the active workspace into the subagent spawn:** `src/auto-reply/reply/commands-long.ts:66-82`
  - body: The existing `sessions_spawn` tool forwards `workspaceDir` into `spawnSubagentDirect`, but this command omits `params.workspaceDir` from the spawn context. That can make `/long` start detached work in a fallback/default workspace instead of the active chat workspace, which is especially bad for the refactor/test/deploy tasks this command targets.
  - confidence: 0.83
