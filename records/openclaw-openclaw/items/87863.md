## What This Changes

## Best Possible Solution

Repair the QQ Bot dispatcher so lane rejection or timeout emits one sanitized failure reply, makes timeout feedback terminal or suppresses late delivery, and adds focused regression coverage plus redacted real QQ Bot proof.

## Reproduction Assessment

Yes. Source inspection shows that a rejecting QQ Bot runtime.channel.inbound.run or a responseTimeoutMs expiry before any deliver callback reaches the catch path; current main only clears timeoutId and sends no user-facing text.

## Solution Assessment
