## Summary

Keep open: the direction is right, but the patch still misses packaged native-require failures where `MODULE_NOT_FOUND` is stored on `Error.cause`, so the intended doctor hint can still be omitted.

## What This Changes

The PR adds `openclaw doctor --fix` guidance to bundled-channel missing-module warnings and records the user-facing fix in `CHANGELOG.md`.

## Best Possible Solution

Keep dependency repair in doctor/plugin install flows, but update this PR to detect missing-module codes through the error cause chain and cover the emitted hint with focused regression tests.

## Reproduction Assessment

Yes. Source inspection shows current main emits bare bundled-channel missing-module warnings, and the PR helper still misses the packaged native-require path where `MODULE_NOT_FOUND` is nested under `cause`.

## Solution Assessment
