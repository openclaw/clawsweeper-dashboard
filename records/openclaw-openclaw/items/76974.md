## Summary

Keep open: the PR targets a real bundled-channel warning gap, but the latest diff still misses the packaged native-require path because it checks only the top-level error code while the module-not-found code can live on `cause`.

## What This Changes

The PR adds bundled-channel warning formatting for module-not-found errors in `src/channels/plugins/bundled.ts` and adds an Unreleased changelog entry.

## Best Possible Solution

Keep dependency repair in doctor/plugin install flows, but make bundled-channel warning formatting detect module-not-found codes through the error cause chain with focused regression coverage.

## Reproduction Assessment

Yes, from source inspection: current main emits bare bundled-channel missing-module warnings, and the proposed helper still misses nested `cause` codes from the built JavaScript loader path. I did not run the global-install scenario in this read-only pass.

## Solution Assessment

No, not yet: appending doctor guidance is the right narrow direction, but the implementation needs cause-chain detection so packaged load failures get the same hint.
