## Summary

Keep open: the PR addresses a real actionable-warning gap and follows the prior decision not to mirror plugin runtime deps into root, but the new formatter misses wrapped module-not-found errors on the bundled entry load path.

## What This Changes

The PR adds a bundled channel load-error formatter that appends `openclaw doctor --fix` guidance for module-not-found errors and adds a changelog entry.

## Best Possible Solution

Keep the centralized warning hint, but detect module-not-found codes through the error cause graph and add focused regression coverage for wrapped native-require failures.

## Reproduction Assessment

Yes, source inspection gives a clear current-main path: bundled channel load failures log bare formatted errors, while the docs expect missing dependency errors to point to an explicit fix. I did not live-run the npm/global reproduction because `pnpm` is unavailable here.

## Solution Assessment

No as written. The centralized helper is the right shape, but it should inspect nested `cause` errors so native `require` wrappers still get the doctor hint.
