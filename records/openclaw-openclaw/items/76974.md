## Summary

Keep open: the PR is directionally right, but the current patch still misses missing-module errors wrapped in the native-require loader, so the packaged bundled-channel path can continue to omit the doctor hint.

## What This Changes

The PR adds a bundled-channel load-error formatter that appends `openclaw doctor --fix` guidance for top-level missing-module error codes, plus an Unreleased changelog entry.

## Best Possible Solution

Keep doctor/plugin install flows as the repair surface, and make bundled-channel missing-module warning guidance detect missing-module codes through the error cause chain with focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main emits bare bundled-channel missing-module warnings, and the PR helper misses the packaged native-require path where the missing-module code is on `error.cause`.

## Solution Assessment

No. Appending doctor guidance is the narrow maintainable fix, but this implementation should walk nested causes before deciding whether to append the hint.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Traverse causes when detecting missing modules:** `src/channels/plugins/bundled.ts:239`
  - body: The formatter checks only `extractErrorCode(error)`. Built JavaScript channel modules are loaded through native require, and that loader wraps require failures in a new Error with `{ cause }`; in that path the `MODULE_NOT_FOUND` code remains on the nested cause, so the PR can still omit the doctor hint it is trying to add. Walk the cause chain before deciding whether to append the guidance.
  - confidence: 0.88
