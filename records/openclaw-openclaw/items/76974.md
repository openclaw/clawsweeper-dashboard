## Summary

Keep open: the direction is narrow and aligned with the existing repair model, but the PR still detects only top-level error codes, so packaged native-require failures wrapped in `cause` can miss the hint it is trying to add.

## What This Changes

The PR adds `openclaw doctor --fix` guidance to bundled-channel missing-module warnings in `src/channels/plugins/bundled.ts` and adds an Unreleased changelog entry.

## Best Possible Solution

Keep dependency repair in doctor/plugin install flows, but make the warning formatter detect missing-module codes through the error cause chain and cover that behavior with focused tests.

## Reproduction Assessment

Yes, from source inspection: current main emits bare bundled-channel missing-module warnings, and the PR helper misses the nested `cause` code from the packaged native-require path. I did not run the global-install scenario.

## Solution Assessment
