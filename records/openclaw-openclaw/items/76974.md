## Summary

Keep open: the direction is right and aligns with the prior #48803 repair-path decision, but the patch still misses the packaged native-require path because the missing-module code is stored on the wrapped error's `cause`.

## What This Changes

The PR appends `openclaw doctor --fix` guidance to bundled-channel missing-module warnings and adds an Unreleased changelog entry.

## Best Possible Solution

Keep the warning UX change, but detect missing-module codes through the error cause chain and add focused regression coverage before merge.

## Reproduction Assessment

Yes. From source inspection, packaged bundled JavaScript modules load through native `require`, `module-loader` wraps missing dependency failures in `cause`, and the PR checks only the top-level code; I did not run the global-install scenario.

## Solution Assessment

No, not yet. Appending doctor guidance is the narrow maintainable fix, but it needs cause-chain detection so the packaged missing-runtime-dependency path receives the hint.
