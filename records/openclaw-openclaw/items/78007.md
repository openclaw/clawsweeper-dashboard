Keep open: current main still has the same gateway-level Undici env-proxy dispatcher behavior as v2026.5.4, and the issue is already paired with open PR #78040 using closing syntax, so it should wait for that PR to be reviewed and either landed or rejected.

## What This Changes

The issue reports a 2026.5.4 regression where `fetch()` calls from the external `@tencent-weixin/openclaw-weixin` channel plugin fail inside the Gateway while standalone curl and Node fetch work.

## Best Possible Solution

Land a generic env-proxy dispatcher fix that preserves normal external plugin HTTPS requests under supported proxy configuration, then let the linked PR close the issue; avoid adding WeChat-specific core bypass logic.

## Reproduction Assessment

No, not as a live Weixin reproduction in this review. Source inspection does give a clear current-main path where Gateway startup installs a global Undici env-proxy dispatcher and passes ALL_PROXY fallback options into plugin-inherited fetch routing.
