## Summary

Keep open: the audit expansion is useful and the supplied CLI proof is convincing, but the PR is not merge-ready because valid JSON5 config backup/rejected artifacts can be marked scanned and then skipped by strict JSON parsing.

## What This Changes

The PR extends `openclaw secrets audit` to scan adjacent `openclaw.json` backup and rejected-payload artifacts, adds focused tests, and documents the new audit coverage.

## Best Possible Solution

Keep the audit expansion, but parse backup and rejected config artifacts with the same JSON5-aware config parser used for active config while still skipping backup-only SecretRef resolution.

## Reproduction Assessment

Yes. At PR head, a valid JSON5 `openclaw.json.bak` or `.rejected.*` file with a comment or trailing comma and a plaintext `models.providers.*.apiKey` is enumerated by `listConfigBackupPaths`, then rejected by `JSON.parse` before audit findings are collected.

## Solution Assessment

No. The scan scope and placement are the right direction, but config-shaped backup artifacts should use OpenClaw's JSON5 parser rather than the strict sidecar JSON helper.
