## Summary

Keep this PR open; it is a useful secrets-audit expansion with sufficient CLI proof, but the current patch still has a concrete false-negative for valid JSON5 config backup artifacts.

## What This Changes

The PR extends `openclaw secrets audit` to scan adjacent `openclaw.json` backup and rejected-payload artifacts, adds focused audit tests, and updates the secrets CLI docs.

## Best Possible Solution

Keep the audit expansion, but parse backup and rejected config artifacts with the same JSON5-aware config parser used for active config, while still skipping backup-only SecretRef resolution.

## Reproduction Assessment

Yes, source-reproducible. A valid JSON5 `openclaw.json.bak` with a comment or trailing comma and a plaintext `models.providers.*.apiKey` is enumerated by the PR but rejected by `JSON.parse` before audit findings are collected.

## Solution Assessment

No. The audit scope and scan placement are right, but config artifacts should reuse OpenClaw's JSON5 parser rather than the strict sidecar JSON helper.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Parse config backup artifacts as JSON5:** `src/secrets/audit.ts:276-280`
  - body: The new backup scan reads `.bak`/`.pre-update`/`.clobbered.*`/`.rejected.*` files through `readJsonObjectIfExists`, which uses `JSON.parse`. OpenClaw config is JSON5, and these artifacts are copies or rejected payloads of that authored config, so a valid backup with comments or trailing commas is marked scanned and then ignored before plaintext findings are collected. Parse these artifacts with the same JSON5 config parser while still skipping backup-only ref resolution.
