## Summary

Keep open: the PR usefully expands secrets-audit coverage and includes real CLI proof, but the new artifact scanner can silently skip valid JSON5 config backups, leaving a security-audit false negative before merge.

## What This Changes

The PR extends `openclaw secrets audit` to scan adjacent `openclaw.json` backup and rejected-payload artifacts for plaintext secret residues, adds focused tests, and documents that coverage.

## Best Possible Solution

Keep the audit expansion, but parse config backup artifacts with OpenClaw's JSON5-aware config parser while still suppressing backup-only SecretRef resolution.

## Reproduction Assessment

Yes, source-reproducible. A valid JSON5 `openclaw.json.bak` with a comment or trailing comma and a plaintext provider `apiKey` is enumerated by the PR, then rejected by `JSON.parse` before findings are collected.

## Solution Assessment

No. The scan scope and invalid-active-config placement are right, but copied `openclaw.json` artifacts should reuse the existing JSON5 config parser instead of the strict sidecar JSON helper.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Parse config backup artifacts as JSON5:** `src/secrets/audit.ts:276-279`
  - body: The new backup scan reads `.bak`/`.pre-update`/`.clobbered.*`/`.rejected.*` files through `readJsonObjectIfExists`, which uses `JSON.parse`. OpenClaw config is JSON5, and raw backup artifacts can preserve comments or trailing commas, so a valid backup can be marked scanned and then ignored before plaintext findings are collected. Parse these artifacts with the same JSON5 config parser while still skipping backup-only ref resolution.
  - confidence: 0.9
