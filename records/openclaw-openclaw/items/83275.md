## Summary

Keep this PR open: the direction is useful, but the patch still misses two audit cases and the external contributor has not provided after-fix real behavior proof.

## What This Changes

The PR extends `openclaw secrets audit` to scan adjacent config backup files, ignore backup-only SecretRefs for resolution checks, adds fixtures, and updates the secrets CLI docs.

## Best Possible Solution

Keep the audit expansion, but make config-artifact scanning independent of active-config validity and include every adjacent OpenClaw-owned config artifact that can preserve secret-shaped payloads.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: current main lacks backup enumeration, and the PR fixtures describe `.bak` plaintext detection. This read-only review did not execute the CLI or tests.

## Solution Assessment

No. `secrets audit` is the right surface, but this implementation should scan backup artifacts even when the active config is invalid and should include rejected config payload artifacts before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Scan backups when the active config is invalid:** `src/secrets/audit.ts:708`
  - body: Because this loop is inside `if (snapshot.valid)`, a malformed or clobbered active `openclaw.json` stops audit from checking adjacent backup files entirely. That misses plaintext residues in the recovery case this new scan is meant to cover, so move backup-artifact scanning out of the active-config-valid branch while still avoiding backup-only ref resolution.
  - confidence: 0.88
- **[P2] Include rejected config payload artifacts:** `src/secrets/storage-scan.ts:59-64`
  - body: OpenClaw saves failed config writes as `openclaw.json.rejected.*` beside the active config, and the troubleshooting docs direct operators to inspect those files. Since rejected payloads can contain the same secret-shaped fields, omitting them leaves `secrets audit` able to report clean while an adjacent rejected payload still contains plaintext credentials.
  - confidence: 0.82
