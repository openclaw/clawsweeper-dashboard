## Summary

Keep open: the PR is a plausible Feishu TTS delivery-policy implementation, but it is not mergeable yet because the new preserve-text path can resend fallback text on voice-media degradation/failure, and the external PR only provides mocked unit-test proof.

## What This Changes

The PR adds a Feishu `tts.preserveText` boolean config, changes reply delivery to send text alongside voice media when enabled, and adds focused dispatcher unit tests.

## Best Possible Solution

Land a backward-compatible TTS text-delivery policy only after fallback/degradation behavior is correct, the chosen config surface is documented, and real Feishu proof shows text plus voice without duplicate fallback messages.

## Reproduction Assessment

Yes, source-level: current main and tests show Feishu suppresses text for voice media, and the PR's duplicate-fallback edge can be traced from the changed `shouldDeliverText` branch into the unchanged `fallbackText` path. I did not run live Feishu or Vitest in this read-only review.

## Solution Assessment

No, not yet. A Feishu-specific opt-in may be acceptable, but the patch must avoid resending fallback text after preserveText already delivered the transcript, and maintainers should confirm whether this belongs in Feishu config or a generic TTS delivery policy.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Avoid resending fallback text when preserving voice text:** `extensions/feishu/src/reply-dispatcher.ts:555`
  - body: With `preserveText` enabled, this branch now sends the transcript before sending the voice media, but the later `sendMediaReplies(..., hasVoiceMedia && hasText ? { fallbackText: text } : undefined)` call is unchanged. If the Feishu voice upload degrades to a file or the media send falls into the existing error fallback, `sendMediaReplies` will send the same transcript again, producing duplicate text exactly in the fallback cases current tests protect. Gate the fallback on whether the text was already delivered, or mark the fallback text as already sent when preserveText is active.
