## What This Changes

The PR adds a Feishu `tts.preserveText` boolean, sends text alongside voice media when enabled, and adds dispatcher tests for enabled, default, and native-opus behavior.

## Best Possible Solution

Land a backward-compatible TTS text-preservation policy only after the fallback/degradation behavior is corrected and the selected config/docs surface is accepted.

## Reproduction Assessment

Yes, at source level: current main suppresses Feishu text for voice media, and the PR's changed text gate leaves the existing fallback transcript path active. I did not run live Feishu or Vitest in this read-only review.

## Solution Assessment

No, not as-is. The opt-in can be reasonable, but the patch must avoid treating already-delivered preserved text as skipped fallback text before it is safe to merge.
