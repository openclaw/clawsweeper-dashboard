## What This Changes

The PR changes `session_status` model override resolution to infer a unique configured provider for bare non-alias model names and adds a regression test for switching from `openai-codex` to `deepseek`.

## Best Possible Solution

## Reproduction Assessment

Yes, source-level. Current main resolves bare `session_status` model overrides with `defaultProvider: currentProvider`, then rejects the resulting wrong provider/model key against the configured allowlist.

## Solution Assessment

Yes in direction. Inferring a unique configured provider before falling back to the active provider is the narrow fix; after rebase, using the shared allowed-model resolver where feasible would reduce drift with the rest of model selection.
