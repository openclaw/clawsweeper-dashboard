## What This Changes

The PR changes `plugins update` so install-record-only npm plugin updates can persist the plugin install index without rewriting the authored config file, while adding freshness and validation rollback coverage.

## Best Possible Solution

Land this PR or an equivalent fix once maintainers accept the records-only config-persistence boundary and normal required checks are green, keeping config rewrites only for actual source-config mutations.

## Reproduction Assessment

Yes at source level: current main routes changed plugin install records through `commitPluginInstallRecordsWithConfig`, which delegates to `replaceConfigFile`, even when only the persisted plugin index changes. I did not run the Windows upgrade path in this read-only review.

## Solution Assessment

Yes. The narrowest maintainable fix is to persist install records without replacing `openclaw.json` only when the projected source config is unchanged, while preserving config writes for legacy cleanup, hook updates, and real config mutations.
