## Summary

Keep open. Current main still lacks the requested agent-visible pre-compaction warning: the config surface has no `visibleWarning` or `warningPrompt`, strict validation would reject those keys, and the memory-flush path still runs as a silent embedded memory trigger. The merged `notifyUser` work is adjacent user-channel UX, while recent discussion already keeps this in maintainer product/API judgment territory.

## What This Changes

The issue requests optional `memoryFlush.visibleWarning` and `warningPrompt` config that would inject a visible agent-facing warning before auto-compaction while preserving the existing silent memory flush.

## Best Possible Solution

Maintainers should decide the compaction lifecycle boundary, then implement either a validated, deduped agent-visible warning before the soft-threshold flush or a documented runtime/plugin extension point that provides the same capability while preserving silent memory flush and user-channel notices.

Not applicable as a feature request. There is a high-confidence verification path for the missing behavior by inspecting current main: the config keys are absent, validation is strict, and the runtime flush remains silent.
