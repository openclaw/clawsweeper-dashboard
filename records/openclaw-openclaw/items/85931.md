## Summary

Keep open. Current main and the latest release still have only process-local QMD update serialization, while this PR is a small, targeted fix with sufficient harness proof and no blocking code-review finding; the remaining decision is whether maintainers want a full real-QMD gateway-plus-CLI smoke before merge.

## What This Changes

The PR wraps QMD update writes in a per-agent cross-process file lock and adds regression coverage for that lock path.

## Best Possible Solution

Land the narrow update-lock fix after maintainers accept the bounded wait policy or verify it with a real-QMD gateway-plus-CLI concurrency smoke.

## Reproduction Assessment

Yes, with high confidence from source inspection and supplied terminal proof, though I did not rerun it locally in this read-only review. Current main has only process-local update serialization, and the PR body shows the two-process SQLite collision succeeding once wrapped with the same file-lock primitive.

## Solution Assessment

Yes. A per-agent cross-process withFileLock inside the existing QMD update queue is the narrowest maintainable fix and reuses the established embed-lock contract without changing config or search-result behavior.
