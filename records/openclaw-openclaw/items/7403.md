## Summary

Keep open. Current main still lacks an enforceable Private/Secret/Incognito session mode, and the issue discussion plus prior maintainer/Codex comments make this a cross-cutting privacy feature rather than a cleanup item.

## What This Changes

Request a Mission Control Private Mode that suppresses workspace context, avoids memory/history persistence, keeps real workspace files untouched, and shows an active private-state indicator.

## Best Possible Solution

Keep this as the canonical design tracker and land one additive ephemeral-session contract with tests proving private sessions neither read protected context nor write durable transcript, session-store, memory, plugin, media, compaction, or history artifacts.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high confidence current main cannot express or enforce the requested mode because closed schemas lack a private flag and normal session paths write JSONL/store state.

## Solution Assessment

Unclear for implementation details, but keeping this open for maintainer product/security design is the best current path. Existing context-injection controls can reduce injected workspace context, but they are not a hard no-read/no-write/no-history private-session contract.
