## Summary

Keep open. Current main still lacks an enforceable per-session Private/Secret/Incognito mode: the existing context-injection controls reduce prompt context globally, but there is no session privacy flag and session creation still writes durable store/transcript state.

## What This Changes

Requests a Mission Control Private Mode that suppresses workspace and memory context, uses temporary working memory, and prevents private sessions from reading/writing real workspace or history state.

## Best Possible Solution

Keep this as the canonical Private/Secret/Incognito tracker and design an additive core session privacy contract across protocol, config/UI, SessionEntry, transcripts, memory/compaction, plugin visibility, and channel activation before implementation.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: current main has no private/ephemeral/no-persist field in the session protocol, config schema, SessionEntry, or Control UI row, and sessions.create still creates a durable transcript file. I did not run a live private-mode flow because there is no current flag to exercise.

## Solution Assessment

No implementation is present yet. The best solution is a maintainer-approved privacy contract and enforcement boundary, not a narrow prompt instruction or config-only toggle.
