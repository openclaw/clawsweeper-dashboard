## Summary

Keep open: current main still lacks an enforceable Private/Secret/Incognito session contract, and the existing context-injection controls only cover part of the requested privacy guarantee while durable session metadata and JSONL transcript creation remain default behavior.

## What This Changes

Request for a Private Mode that suppresses workspace context and avoids persistent session, memory, and workspace writes for demos and sensitive conversations.

## Best Possible Solution

Keep this as the canonical tracker and design one additive private/ephemeral session contract across Gateway protocol, session storage/transcripts, agent context/memory, plugin enforcement, channel triggers, and Control UI.

## Reproduction Assessment

Yes. Source inspection shows the existing context-injection control is only a partial bootstrap suppression path, while current session creation still writes durable metadata and a JSONL transcript header.

## Solution Assessment

Yes for keeping this issue open. The best implementation is a maintainer-approved cross-layer privacy contract rather than a prompt-only, UI-only, or config-only toggle.
