## Summary

Keep open. Current main still lacks an enforceable per-session private/ephemeral contract across Gateway schemas, session config/store state, Control UI rows, and transcript persistence, while the discussion remains an active privacy/security design tracker.

## What This Changes

Request a Mission Control Private Mode toggle and/or per-session/per-channel ephemeral mode that suppresses workspace context, memory, and durable session persistence.

## Best Possible Solution

Keep this as the canonical design tracker and land an additive ephemeral-session contract across protocol, SessionEntry, transcript/session store, bootstrap, memory, tool/media, plugin, channel, and UI surfaces with regression tests proving no protected reads or durable writes.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence current main cannot express or enforce the requested mode because closed schemas lack a private flag and normal session paths write JSONL/store state.

## Solution Assessment

Unclear for implementation details; keeping this open for design-approved maintainer work is the best current outcome. Existing `contextInjection: "never"` only suppresses bootstrap injection and does not enforce no-read/no-write/no-history privacy.
