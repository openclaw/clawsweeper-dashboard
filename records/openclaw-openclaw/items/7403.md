## Summary

Keep open. Current main still lacks an enforceable private/ephemeral session contract; existing context-injection controls and unrelated ephemeral surfaces do not cover durable session metadata, JSONL transcripts, workspace/memory reads, or UI/channel semantics.

## What This Changes

Add a Mission Control private mode that suppresses workspace context and uses temporary/no-persist session state for demos, shared use, and public work.

## Best Possible Solution

Keep this as the canonical Private/Secret/Incognito tracker and design an additive core ephemeral-session contract with tests proving private sessions neither read protected workspace/memory context nor write durable history.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence missing-capability path because the create/patch schemas, session config, SessionEntry, and Control UI row model have no private/no-persist field while sessions.create writes a durable transcript.

## Solution Assessment

Yes, keeping this open is the best current outcome. A prompt instruction, contextInjection setting, BTW-style side query, or channel ephemeral reply cannot provide the requested program-level no-read/no-write/no-history guarantee.
