## Summary

Keep open as the canonical private-session tracker. Current main and v2026.6.5 still create durable session metadata/transcripts and expose no private or no-persist contract; the related implementation PR closed unmerged, while the discussion now requires a program-level privacy guarantee rather than a demo-only UI toggle.

## What This Changes

Add a per-session private mode that suppresses workspace and memory context, prevents durable session history or workspace writes, and exposes clear Control UI state.

## Best Possible Solution

Keep one canonical tracker and design an additive private-session contract with an explicit threat model, immutable or safely transitioned activation semantics, loss-on-restart behavior, and end-to-end enforcement across protocol, session state, transcripts, bootstrap/memory, tools/media/plugins, channels, history APIs, and Control UI.

## Reproduction Assessment

Yes at source level. Current main has a concrete sessions.create path, durable SessionEntry/sessionFile state, transcript-creation coverage, and no private/no-persist field; this demonstrates the missing capability, though it is a feature request rather than a broken existing contract.

## Solution Assessment

The requested privacy goal is valid, but a Mission Control toggle or config-validation-only phase is not the best complete solution. The maintainable fix starts with one canonical session privacy state and proves every relevant context read and durable write honors it before exposing UI or channel activation.
