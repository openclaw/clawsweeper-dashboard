## Summary

## What This Changes

Add a Private Mode for demos/content creation that suppresses workspace context and prevents private session state, memory, and history from being persisted.

## Best Possible Solution

Keep this as the canonical tracker for a design-approved private/ephemeral session contract across Gateway protocol, session storage/transcripts, agent context/memory, plugin enforcement, channel triggers, and Control UI.

## Reproduction Assessment

Yes, by source inspection: current main lacks a private/no-persist session flag in protocol, config, SessionEntry, and UI, while session creation still creates durable store metadata and transcript files. A live run was not necessary because there is no current private-mode control to exercise.

## Solution Assessment

Yes for keeping it open, but not for autonomous implementation yet. The maintainable solution is a single additive privacy contract across protocol, persistence, memory/compaction/plugin boundaries, and UI rather than a prompt-only or config-only toggle.
