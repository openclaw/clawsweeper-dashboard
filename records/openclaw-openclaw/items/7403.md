## Summary

Keep open. Current main still lacks an enforceable per-session Private/Secret/Incognito mode, and the live discussion keeps this as the canonical product/security tracker rather than a duplicate or shipped feature.

## What This Changes

The issue requests a Mission Control Private Mode that suppresses workspace context and provides per-session no-read/no-write/no-history behavior for demos and sensitive shared use.

## Best Possible Solution

Keep this as the canonical tracker and design one additive private/ephemeral session contract across Gateway protocol, SessionEntry, transcript/session storage, bootstrap and memory paths, plugin/channel visibility, and Control UI status.

## Reproduction Assessment

Yes, at source level. Current main has no private/ephemeral field in the session protocol, config, persisted session entry, or UI session row, while session creation and memory paths still use durable storage paths; I did not run a live session because that would create the persistence artifacts under review.

## Solution Assessment

No implementation is present yet. The best path is a design-approved cross-cutting privacy contract, not a prompt-only workaround or a narrow config toggle that leaves persistence paths active.
