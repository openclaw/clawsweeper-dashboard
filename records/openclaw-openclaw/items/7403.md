## Summary

Keep open. Current main still lacks an enforceable private/ephemeral session contract across Gateway schemas, session config/state, transcript persistence, bootstrap context, and Control UI state, and the issue discussion makes this a privacy/security-sensitive design item rather than a narrow bug fix.

## What This Changes

The issue requests a Private Mode toggle that suppresses workspace context, durable memory/history, and workspace writes for demo or shared-use sessions.

## Best Possible Solution

Keep this as the canonical Private/Secret/Incognito tracker and design an additive core ephemeral-session contract with regression tests proving private sessions neither read protected workspace/memory context nor write durable transcript, memory, plugin, media, or history artifacts.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence current main cannot express or enforce the requested mode because closed schemas lack a private flag and normal session paths write JSONL/store state.

## Solution Assessment

Unclear for implementation details, but keeping this open for design-approved maintainer work is the best current outcome. Existing contextInjection controls are useful mitigations, not a program-level no-read/no-write/no-history private-session contract.
