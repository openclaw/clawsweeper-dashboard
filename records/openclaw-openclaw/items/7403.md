## Summary

Keep open. Current main still lacks an enforceable per-session Private/Secret/Incognito mode: the protocol, session config, persisted session entry, transcript/store write paths, and UI session row model have no private or no-persist contract, and prior duplicate reports were intentionally consolidated here.

## What This Changes

The issue requests a Private Mode that suppresses workspace context, prevents durable session/memory reads and writes, and exposes a Mission Control toggle/status indicator.

## Best Possible Solution

Design one additive private/ephemeral session contract across Gateway protocol, SessionEntry, session store/transcripts, bootstrap/memory, compaction, plugin persistence hooks, channel triggers, and Control UI, with tests proving private sessions neither read private workspace context nor write durable history.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives high confidence current main cannot express or enforce the requested private/ephemeral session mode; I did not run a live session because that would create the persistence artifacts under review.

## Solution Assessment

Unclear until maintainers define the privacy contract. The requested direction is plausible, but a narrow config or UI toggle alone would not be the best solution without hard no-read/no-write enforcement across the persistence stack.
