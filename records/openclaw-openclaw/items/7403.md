## Summary

Keep open: current main and v2026.6.6 still lack an enforceable private/no-persist session contract. Existing context-injection controls can reduce bootstrap context, but they do not stop durable session metadata, transcript files, memory flush, memory indexing, or UI/session history paths, and related closed items point back to this issue as the canonical tracker.

## What This Changes

The issue requests a Mission Control Private Mode that suppresses workspace context and uses temporary non-persistent session memory for demos and content creation.

## Best Possible Solution

Keep this as the canonical tracker and design one additive private/ephemeral session contract across Gateway protocol, session storage/transcripts, agent context/memory, plugin enforcement, channel triggers, and Control UI with regression tests for no private context reads and no durable history writes.

## Reproduction Assessment

Yes, by source inspection: current main has no private/no-persist protocol, config, SessionEntry, or UI field, while sessions.create still creates durable store metadata and transcript files. A live run was not necessary because there is no current private-mode control to exercise.

## Solution Assessment

Yes for keeping this open, but no for autonomous implementation yet. The maintainable solution is a design-approved cross-layer privacy contract rather than a prompt-only or partial config toggle.
