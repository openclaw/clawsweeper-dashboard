## Summary

Keep open: current main still lacks an enforceable Private/Secret/Incognito session contract; existing context-injection controls do not prevent durable session metadata, JSONL transcript creation, memory/compaction/plugin persistence, or UI/channel enforcement.

## What This Changes

Add an enforceable Private Mode/private session contract that suppresses workspace context and prevents durable session and memory persistence for demos and sensitive sessions.

## Best Possible Solution

Keep this as the canonical tracker and design one additive private/ephemeral session contract across Gateway protocol, SessionEntry/session store/transcripts, bootstrap/memory/compaction/plugins, channel triggers, and Control UI, with tests proving no private-context reads or durable history writes.

## Reproduction Assessment

Yes by source inspection: current schemas and session creation paths have no private/no-persist field and still create a durable session entry plus JSONL header.

## Solution Assessment

Yes for triage: keeping this open is the best current outcome because prompt-only, config-only, or UI-only changes would falsely promise privacy without a cross-layer enforcement contract.
