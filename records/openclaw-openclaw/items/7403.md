## Summary

Keep open. Current main still lacks an enforceable private/ephemeral session contract; existing context-injection controls and ephemeral side-query surfaces do not cover session-store metadata, JSONL transcript persistence, workspace/memory reads, or durable tool/media/history boundaries.

## What This Changes

Add a Mission Control Private Mode that runs temporary, non-persistent sessions without workspace/persona/memory context or workspace reads and writes.

## Best Possible Solution

Keep this as the canonical Private/Secret/Incognito tracker and design an additive core ephemeral-session contract with tests proving private sessions do not read protected workspace/memory context or write durable history.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence missing-capability path because the create/patch schemas, session config, SessionEntry, and Control UI row model have no private/no-persist field while sessions.create creates a durable JSONL transcript.

## Solution Assessment

Yes, keeping this open is the best current outcome. A prompt instruction, ClawHub plugin, /btw-style side query, or global contextInjection setting cannot provide the requested program-level no-read/no-write/no-history guarantee.
