## Summary

Keep open. Current main still lacks an enforceable per-session private/no-persistence contract, and the related discussion shows this remains the canonical tracker for privacy, memory, transcript, and UI semantics.

## What This Changes

Request an enforceable Mission Control Private Mode/private session contract that suppresses workspace context and avoids durable session, history, and memory writes.

## Best Possible Solution

Design an additive private/ephemeral session contract across Gateway protocol, SessionEntry, transcript/session storage, bootstrap and memory paths, compaction, plugin/channel visibility, and Mission Control UI, with regression tests proving private sessions neither read private context nor write durable history.

## Reproduction Assessment

Yes, at source level: current main has no private/ephemeral field in the session protocol, config, persisted session entry, or UI/session row path, while session creation still writes store metadata and ensures a JSONL transcript. I did not run a live session because this read-only review should not create the persistence artifacts under review.

## Solution Assessment

Unclear until maintainers choose the privacy contract. A Mission Control toggle is useful only if it is backed by a shared no-read/no-write/no-history enforcement marker rather than prompt-only or UI-only behavior.
