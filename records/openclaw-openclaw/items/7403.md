## Summary

Keep open. Current main still lacks an enforceable Private/Secret/Incognito session mode, and related duplicate cleanup confirms this remains the canonical tracker rather than something to close.

## What This Changes

Request adds a Private Mode for demos/content creation that suppresses workspace and memory context, prevents durable session/history writes, and shows a Mission Control/UI indicator.

## Best Possible Solution

Keep this as the canonical design tracker and land one additive private/ephemeral session contract covering protocol, config, SessionEntry, transcripts, session store, memory, compaction, plugin visibility, media/tool persistence, channel triggers, and Mission Control status.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high confidence current main cannot express or enforce the requested mode because the closed schemas, UI row model, and persistence paths have no private/no-persist marker.

## Solution Assessment

Unclear for implementation details, but keeping this open for maintainer product/security design is the best current path. Existing context-injection controls reduce bootstrap context only and are not a hard no-read/no-write/no-history private-session contract.
