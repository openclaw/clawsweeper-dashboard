## Summary

Keep open. Current main still lacks an enforceable per-session Private/Secret/Incognito mode; `contextInjection: "never"` only suppresses bootstrap/context-file injection and does not cover durable transcripts, session metadata, memory writes, tool/media artifacts, channel triggers, or Control UI state.

## What This Changes

The issue requests a Mission Control Private Mode that suppresses workspace and memory context, prevents durable session history or workspace writes, and uses temporary session memory for demos or shared-screen work.

## Best Possible Solution

Keep this as the canonical Private/Secret/Incognito tracker and design an additive private/ephemeral session contract across Gateway protocol, session store, transcript/history APIs, bootstrap/memory, tool/media persistence, channel triggers, and Control UI, with regression tests proving no private context reads or durable writes.

## Reproduction Assessment

Not applicable as a feature/privacy request rather than a runtime bug. The absence check is high-confidence because current schemas, persisted session types, transcript creation, bootstrap-context handling, tests, docs, and UI session rows lack the requested private/no-persist contract.

## Solution Assessment

No current patch is the best complete solution. `contextInjection: "never"`, pruning, or a UI badge alone would not satisfy the no-read/no-write/no-history guarantee; the maintainable path needs a maintainer-approved cross-surface privacy contract.
