## Summary

Keep open. Current main still lacks an enforceable per-session private/no-persistence contract, and the related items show this remains the canonical tracker for the broader privacy mode rather than a stale duplicate.

## What This Changes

The issue requests a Mission Control Private Mode that suppresses workspace context and uses temporary, non-persisted session memory/history for demos and shared or public work.

## Best Possible Solution

Design an additive private/ephemeral session contract across Gateway protocol, SessionEntry, transcript/session storage, bootstrap and memory paths, compaction, plugin/channel visibility, and Control UI, with tests proving private sessions neither read private context nor write durable history.

## Reproduction Assessment

Yes, at source level: current main has no private/ephemeral field in the session protocol, config, persisted session entry, or UI row, while session creation still writes store metadata and a JSONL transcript. I did not run a live session because that would create the persistence artifacts under review.

## Solution Assessment

Unclear until maintainers choose the privacy contract. A Mission Control toggle is useful only if it is backed by a shared no-read/no-write/no-history enforcement marker rather than prompt-only or UI-only behavior.
