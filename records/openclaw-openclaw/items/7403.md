## Summary

Keep open. Current main still lacks an enforceable per-session Private/Secret/Incognito contract: the Gateway session protocol, session config, persisted session entry, transcript write path, bootstrap context flow, and Control UI session row model do not carry a private/no-persist bit. The discussion and closed duplicate make this a program-level privacy guarantee, not a UI-only enhancement or something solved by the narrower contextInjection setting.

## What This Changes

Add a Mission Control Private Mode or ephemeral session option that suppresses workspace and memory context, avoids durable session history, and prevents workspace reads/writes for demos and sensitive conversations.

## Best Possible Solution

Keep this as the canonical Private/Secret/Incognito tracker and design an additive private/ephemeral session contract across Gateway protocol, SessionEntry/store, transcript/history APIs, bootstrap/startup memory, tool/media persistence, channel triggers, and Control UI. Regression coverage should prove private sessions neither read private workspace/memory context nor write durable history or artifacts.

## Reproduction Assessment

Not applicable as a feature/privacy request rather than a runtime bug. The absence check is high-confidence: current schemas, persisted session types, transcript creation/appending, bootstrap context code, docs, and UI session rows all lack the requested private/no-persist contract.

## Solution Assessment

No current patch is the best complete solution. The requested direction is valid, but the maintainable fix needs a maintainer-approved cross-surface privacy contract rather than a UI-only toggle, global contextInjection setting, or validation-only scaffolding.
