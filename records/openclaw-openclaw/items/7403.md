## Summary

Keep open. Current main still lacks an enforceable private/ephemeral session mode, and the linked discussion keeps this as the canonical tracker rather than a stale or implemented request.

## What This Changes

The issue requests a Mission Control Private Mode that suppresses workspace/persona/memory context and prevents private sessions from reading or writing durable workspace/session data.

## Best Possible Solution

Keep this as the canonical Private/Secret/Incognito tracker and design an additive core ephemeral-session contract with tests proving private sessions neither read protected workspace/memory context nor write durable history.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence missing-capability path because the protocol schemas, session config, SessionEntry, tests, and Control UI row model lack a private/no-persist field while sessions.create still writes durable transcript state.

## Solution Assessment

Yes, keeping this open for design-approved maintainer work is the best current outcome. Existing contextInjection and BTW-style ephemeral side results are useful mitigations, but they are not a program-level no-read/no-write/no-history private-session contract.
