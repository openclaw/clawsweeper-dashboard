## Summary

Keep open. Current main still lacks an enforceable per-session Private/Secret/Incognito mode; the existing bootstrap suppression path only disables injected workspace context and does not provide the requested no-read/no-write/no-history guarantee.

## What This Changes

Add a Private Mode for demos and sensitive sessions that suppresses workspace context, memory reads/writes, and durable session history.

## Best Possible Solution

Keep this as the canonical Private/Secret/Incognito tracker and design an additive private/ephemeral session contract across Gateway protocol, session store, transcripts/history APIs, bootstrap/memory, tool/media persistence, channel triggers, and Control UI, with regression tests proving no private context reads or durable writes.

## Reproduction Assessment

Not applicable as a feature/privacy request rather than a runtime bug. The absence check is high-confidence because current schemas, session persistence types, transcript creation, bootstrap-context handling, docs, and UI session surfaces lack the requested private/no-persist contract.

## Solution Assessment

No current implementation is the best complete solution. A UI badge, pruning, or contextInjection:"never" alone would not satisfy the no-read/no-write/no-history guarantee; the maintainable path needs a maintainer-approved cross-surface privacy contract.
