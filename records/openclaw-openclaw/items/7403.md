## Summary

Keep open. Current main still lacks an enforceable per-session Private/Secret/Incognito mode; the existing `contextInjection: "never"` path only suppresses bootstrap/context-file injection and does not provide the requested no-read/no-write/no-history privacy guarantee.

## What This Changes

Request to add a Mission Control Private Mode that suppresses workspace/bootstrap/memory context and avoids durable session or workspace persistence for private sessions.

## Best Possible Solution

Keep this as the canonical Private/Secret/Incognito tracker and design an additive private/ephemeral session contract across Gateway protocol, session store, transcripts/history APIs, bootstrap/memory, tool/media persistence, channel triggers, and Control UI, with regression tests proving no private context reads or durable writes.

## Reproduction Assessment

Not applicable as a feature/privacy request rather than a runtime bug. The absence check is high-confidence because current schemas, persisted session types, transcript creation, bootstrap-context handling, docs, and UI session surfaces lack the requested private/no-persist contract.

## Solution Assessment

No current patch is the best complete solution. `contextInjection: "never"`, pruning, or a UI badge alone would not satisfy the no-read/no-write/no-history guarantee; the maintainable path needs a maintainer-approved cross-surface privacy contract.
