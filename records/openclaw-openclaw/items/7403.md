## Summary

Keep open. Current main and latest release still lack an enforceable per-session private/no-persistence contract, and the related private-mode PR is closed unmerged while this remains the canonical tracker for the privacy/session-state design.

## What This Changes

Request to add a Mission Control Private Mode that suppresses workspace context and uses non-persistent session memory/history.

## Best Possible Solution

Design a first-class private/ephemeral session contract enforced across Gateway protocol, SessionEntry, transcript/session storage, bootstrap and memory paths, compaction, plugins/channels, and Mission Control UI.

## Reproduction Assessment

Yes, at source level: current main has no private/ephemeral field in the session protocol, config, persisted session entry, or UI row path, while session creation and transcript paths still write durable session state. I did not run a live session because this read-only review should not create those artifacts.

## Solution Assessment

Unclear until maintainers choose the privacy contract. A Mission Control toggle is useful only if it is backed by a shared no-read/no-write/no-history enforcement marker rather than prompt-only or UI-only behavior.
