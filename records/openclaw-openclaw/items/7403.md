## Summary

Keep open. Current main still lacks an enforceable private/no-persistence session contract; existing context-injection controls only reduce prompt context, and related searches show this remains the canonical tracker for the broader privacy mode work.

## What This Changes

Add a Private Mode that suppresses workspace context reads and durable session or memory writes for demos, content creation, and sensitive conversations.

## Best Possible Solution

Design one additive private/ephemeral session contract across Gateway protocol, SessionEntry, transcript/session storage, bootstrap and memory paths, plugin/channel visibility, and Control UI status.

## Reproduction Assessment

Yes, at source level: current main has no private/ephemeral field in the session protocol, config, persisted session entry, or UI row, while session creation and memory flush paths still write durable state. I did not run a live session because that would create the persistence artifacts under review.

## Solution Assessment

No implementation is present yet. The best path is a design-approved cross-cutting privacy contract, not a prompt-only workaround or a narrow toggle that leaves persistence paths active.
