## Summary

Keep open. Current main still lacks an enforceable private/ephemeral session contract, and the discussion makes this a privacy-sensitive product/design request rather than a narrow bug fix.

## What This Changes

Request an enforceable Private Mode toggle that suppresses workspace and memory context while preventing persistent session history or workspace changes during demos and public content creation.

## Best Possible Solution

Keep this as the canonical design tracker and land one additive core ephemeral-session contract with tests proving private sessions neither read protected workspace/memory context nor write durable transcript, memory, plugin, media, or history artifacts.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence current main cannot express or enforce the requested mode because closed schemas lack a private flag and normal session paths write JSONL/store state.

## Solution Assessment

Unclear for implementation details, but keeping this open for design-approved maintainer work is the best current outcome. Existing contextInjection controls are useful mitigations, not a program-level no-read/no-write/no-history private-session contract.
