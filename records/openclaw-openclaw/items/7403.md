## Summary

Keep open. Current main still lacks an enforceable private/ephemeral session contract: Gateway session schemas, session config, SessionEntry, the Control UI row model, and transcript paths have no private/no-persist field, while session creation and transcript append paths still write durable state. The discussion and related items make this the canonical privacy/security design tracker rather than an implemented or stale request.

## What This Changes

Add a Mission Control Private Mode that suppresses workspace and memory context, avoids touching workspace files, and uses temporary non-persistent session memory for demos and content creation.

## Best Possible Solution

Keep this as the canonical Private/Secret/Incognito tracker and design an additive core ephemeral-session contract with tests proving private sessions neither read protected workspace or memory context nor write durable history, memory, plugin, media, or transcript artifacts.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence missing-capability path because the protocol schemas, session config, SessionEntry, Control UI row model, and transcript paths lack a private/no-persist field while current session creation and append paths write durable state.

## Solution Assessment

Yes, keeping this open for design-approved maintainer work is the best current outcome. Existing contextInjection controls are useful mitigations, but they are not a program-level no-read/no-write/no-history private-session contract.
