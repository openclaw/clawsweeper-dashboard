## Summary

Keep open. Current main still lacks an enforceable per-session Private/Secret/Incognito mode, and the related discussion keeps this as the canonical tracker rather than a duplicate or shipped feature.

## What This Changes

Request a Mission Control Private Mode and ephemeral session contract that suppresses workspace context and prevents durable session, memory, and transcript persistence.

## Best Possible Solution

Keep this as the canonical tracker and design one additive private/ephemeral session contract across Gateway protocol, SessionEntry, session store/transcripts, bootstrap/memory, plugin SDK visibility, channel defaults, and Control UI status.

## Reproduction Assessment

Yes, at source level. Current schemas have no private/ephemeral field, while sessions.create, transcript append, and memory-flush paths still persist durable state; I did not run a live repro because that would create the artifacts under review.

## Solution Assessment

No implementation is present yet. The best path is a design-approved cross-cutting contract, not reviving the closed partial config-only PR or adding a narrow prompt-level workaround.

## Security Review

Status: needs_attention

Summary: No patch was reviewed, but this feature defines a privacy boundary where partial enforcement could mislead users about what is actually persisted.
