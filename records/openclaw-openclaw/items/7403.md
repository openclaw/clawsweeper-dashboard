## Label Justifications

- `impact:security`: The request is about preventing sensitive workspace, memory, and transcript data from being exposed or persisted during demos and shared-use sessions.
- `impact:session-state`: The requested behavior changes session history, memory, transcript, and context persistence semantics.

## Summary

Keep open. Current main still lacks an enforceable private/ephemeral session mode: the protocol, config schema, persisted SessionEntry, transcript/history paths, and Control UI row model have no private or no-persist contract, while existing context-injection controls only suppress bootstrap context and do not stop durable session persistence.

## What This Changes

Request a Mission Control Private Mode that suppresses workspace/persona/memory context, prevents durable session history or memory writes, and shows a visible private-session state.

## Best Possible Solution

Design one additive private/ephemeral session contract across Gateway protocol, SessionEntry/store/transcripts, bootstrap and memory/compaction paths, plugin visibility, channel triggers, and Control UI, with regression tests proving private sessions neither read private workspace/memory context nor write durable private history.

## Reproduction Assessment

Not applicable as a runtime bug reproduction, but yes at source level: current main has no private/no-persist field in session protocol, config, SessionEntry, or UI row types, and session creation still persists transcript state.

## Solution Assessment

Unclear at implementation level because the best fix needs maintainer design first; a partial toggle is not sufficient, and the maintainable path is one shared enforcement contract across all persistence boundaries.
