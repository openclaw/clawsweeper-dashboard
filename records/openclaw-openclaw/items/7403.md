## Summary

Keep open. Current main still lacks an enforceable per-session private/no-persistence contract, and the live duplicate history confirms this remains the canonical tracker for privacy, memory, transcript, channel, plugin, and UI semantics.

## What This Changes

Add a Private Mode toggle or session contract that suppresses workspace context and prevents private sessions from reading or writing memory, transcripts, and durable history.

## Best Possible Solution

Keep this as the canonical tracker and design an additive private/ephemeral session contract enforced across Gateway protocol, SessionEntry, transcript/session storage, bootstrap and memory paths, compaction, plugins/channels, and Mission Control UI.

## Reproduction Assessment

Yes, at source level: current main has no private/ephemeral field in the session protocol, config, persisted session entry, or UI row path, while session creation still writes store metadata and ensures a JSONL transcript. I did not run a live session because this read-only review should not create those artifacts.

## Solution Assessment

Unclear until maintainers choose the privacy contract. A Mission Control toggle is useful only if it is backed by a shared no-read/no-write/no-history enforcement marker rather than prompt-only or UI-only behavior.
