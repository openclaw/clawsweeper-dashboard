## Summary

Keep open: current main still lacks an enforceable private/ephemeral session contract, and the issue remains the canonical tracker for a privacy-sensitive product design.

## What This Changes

The issue requests a Private Mode that suppresses workspace context, memory/history persistence, and real workspace writes while exposing a Mission Control toggle/status indicator.

## Best Possible Solution

Keep this as the canonical tracker and design an additive private/ephemeral session contract across Gateway protocol, config, SessionEntry, transcript/store writes, bootstrap/memory reads, compaction, plugin SDK visibility, channel triggers, and Control UI state before implementation.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main cannot express or enforce the requested private/no-persist mode, and a live run was not useful because no such flag exists to exercise.

## Solution Assessment

Yes for triage: keeping this open as the canonical product/security design tracker is the best current path. The implementation should wait for maintainer decisions on the threat model and enforcement boundary before adding config, protocol, UI, or plugin API surface.
