## Summary

Keep this issue open as the canonical private-session tracker. Current main and v2026.6.5 still expose no enforceable private/no-persist session contract, durable transcript creation remains expected behavior, and the related implementation PR closed unmerged.

## What This Changes

Requests an enforceable private session mode that suppresses workspace context and prevents durable session or memory persistence.

## Best Possible Solution

Design an additive private-session mode with one authoritative runtime state propagated through gateway protocol, session lifecycle, transcript/history, bootstrap and memory, plugin/tool persistence, channel controls, and Control UI, backed by tests proving no private context is read and no durable conversational state is written.

## Reproduction Assessment

Yes, at the source level: current main and v2026.6.5 expose no private/no-persist session field, and the existing sessions.create test requires durable metadata and transcript creation. This verifies the missing capability without needing a runtime failure reproduction.

## Solution Assessment

The issue is the right canonical problem statement, but the initial Mission Control toggle alone is not the best complete fix. The maintainable solution is a design-approved enforcement contract across every read and persistence boundary, with UI and channel triggers layered on that contract.
