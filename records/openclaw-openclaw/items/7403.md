## Summary

Keep open. Current main still lacks an enforceable per-session Private/Secret/Incognito mode: Gateway protocol, config, session metadata, and Control UI have no privacy marker, while session creation and transcript paths still persist by default; the related duplicate issues are closed into this canonical tracker and the partial private-mode PR was closed unmerged.

## What This Changes

Add a Mission Control Private Mode that suppresses workspace bootstrap context, avoids reading or writing memory and session history, uses temporary scratch state, and shows visible UI state for demos and content creation.

## Best Possible Solution

Keep this as the canonical Private/Secret/Incognito tracker and design one additive private-session contract across Gateway protocol, session storage/transcripts, memory/compaction/plugin visibility, channel triggers, and Control UI.

## Reproduction Assessment

No live bug reproduction is applicable because this is a feature request; source inspection gives high confidence current main cannot express or enforce a per-session private/no-persist mode because protocol, config, session row, and `SessionEntry` lack the marker and transcript creation writes a JSONL header.

## Solution Assessment

The existing `agents.defaults.contextInjection: "never"` workaround is not a complete solution because it only suppresses bootstrap context and does not gate session transcripts, memory flush/indexing, plugin persistence, tools/media, or UI/channel semantics.
