## Summary

Keep open. Current main still lacks an enforceable per-session Private/Secret/Incognito mode; the existing `contextInjection: "never"` support only suppresses bootstrap context and does not cover durable transcripts, session metadata, memory writes, tool/media artifacts, channel triggers, or Control UI state.

## What This Changes

Add a Mission Control Private Mode toggle that suppresses workspace context and prevents private sessions from reading or writing durable memory, workspace, or session history.

## Best Possible Solution

Keep this as the canonical Private/Secret/Incognito tracker and design an additive private/ephemeral session contract across Gateway protocol, session store, transcript/history APIs, bootstrap/memory, tool/media persistence, channel triggers, and Control UI, with regression tests proving no private context reads or durable writes.

## Reproduction Assessment

Not applicable as a feature/privacy request rather than a runtime bug. The absence check is high-confidence: current schemas, persisted session types, transcript creation, bootstrap context handling, docs search, and UI session rows lack the requested private/no-persist contract.

## Solution Assessment

No current patch is the best complete solution. The maintainable fix needs a maintainer-approved cross-surface privacy contract; `contextInjection: "never"`, pruning, or a UI badge alone would not satisfy the no-read/no-write/no-history guarantee.
