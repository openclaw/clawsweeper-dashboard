## Summary

Keep open: current main and v2026.6.8 still lack an enforceable Private/Secret/Incognito session contract, and the discussion makes this a program-level privacy guarantee rather than a prompt-only demo toggle.

## What This Changes

Add a Mission Control Private Mode that suppresses workspace context, avoids durable session and memory persistence, uses temporary scratch memory, and visibly marks private sessions.

## Best Possible Solution

Keep this as the canonical tracker and design one additive private/ephemeral session contract across Gateway protocol, session storage/transcripts, agent context/memory, plugin enforcement, channel triggers, and Control UI.

## Solution Assessment
