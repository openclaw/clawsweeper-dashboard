## Summary

Keep open. Current main still lacks an enforceable private/ephemeral session contract: the Gateway session schemas, session config, SessionEntry, Control UI session row, and transcript creation/append paths have no private/no-persist field or gate, while the discussion makes this the canonical privacy design tracker.

## What This Changes

Add a Private Mode for demos and public work that suppresses workspace context and prevents durable session history or memory writes.

## Best Possible Solution

Keep this as the canonical Private/Secret/Incognito tracker and design an additive core ephemeral-session contract with regression tests proving private sessions do not read protected workspace/memory context or write durable history, memory, plugin, media, or transcript artifacts.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that current main cannot express or enforce the requested mode because the session protocol, config, entry type, UI row, and transcript paths lack a private/no-persist field while normal session paths write durable state.

## Solution Assessment

Unclear for implementation details, but keeping this open for design-approved maintainer work is the best current outcome. Existing contextInjection controls are useful mitigations, not a program-level no-read/no-write/no-history private-session contract.
