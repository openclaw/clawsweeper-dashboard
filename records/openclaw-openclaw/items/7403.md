## Summary

Keep open. Current main and the latest release still lack an enforceable per-session private/no-persistence contract, and the discussion confirms this remains the canonical tracker for privacy, memory, transcript, channel, plugin, and UI semantics.

## What This Changes

Add a Private Mode that suppresses workspace context, memory/history persistence, and workspace reads/writes for demos, content creation, and private sessions.

## Best Possible Solution

Keep this as the canonical tracker and design an additive private/ephemeral session contract enforced across Gateway protocol, SessionEntry, transcript/session storage, bootstrap and memory paths, compaction, plugins/channels, and Mission Control UI.

## Reproduction Assessment

Yes, at source level: current main has no private/ephemeral field in the session protocol, config, persisted session entry, or UI row path, while session creation and memory paths still write or derive durable state. I did not run a live session because this read-only review should not create session artifacts.

## Solution Assessment

Unclear until maintainers choose the privacy contract. A Mission Control toggle is useful only if it is backed by a shared no-read/no-write/no-history enforcement marker rather than prompt-only or UI-only behavior.
