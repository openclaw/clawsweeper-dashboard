## Summary

Keep open. Current main and the latest release still lack an enforceable Private/Secret/Incognito session mode, while the issue discussion and closed duplicate history show this is the canonical tracker for the cross-cutting privacy contract.

## What This Changes

Add a Private Mode toggle and session privacy contract that suppresses workspace context, memory writes, and durable session history for private demos or conversations.

## Best Possible Solution

Keep this as the canonical design tracker and land one additive private/ephemeral session contract that covers protocol, config, SessionEntry, transcripts, session store, memory, compaction, plugin visibility, media/tool persistence, channel triggers, and Mission Control status.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high confidence current main cannot express or enforce the requested mode because the closed schemas and persisted session paths have no private/no-persist marker.

## Solution Assessment

Unclear for implementation details, but keeping this open for maintainer product/security design is the best current path. Existing context-injection controls reduce bootstrap context only and are not a hard no-read/no-write/no-history private-session contract.
