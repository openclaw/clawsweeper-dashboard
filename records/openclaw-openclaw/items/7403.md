## Summary

Keep open. Current main still lacks an enforceable per-session Private/Secret/Incognito mode, and the live discussion plus duplicate-search pass show this issue remains the canonical tracker for the broader privacy/no-persistence contract.

## What This Changes

Add a Private Mode or ephemeral-session control that suppresses workspace context and prevents durable session, transcript, and memory writes for demos or private shared-use sessions.

## Best Possible Solution

Keep this as the canonical tracker and design one additive private/ephemeral session contract across Gateway protocol, SessionEntry, transcript/session storage, bootstrap and memory paths, plugin/channel visibility, and Control UI status.

## Reproduction Assessment

Yes, at source level: current main has no private/ephemeral field in the session protocol, config, persisted session entry, or UI row, while session creation and memory flush paths still write durable state. I did not run a live session because that would create the persistence artifacts under review.

## Solution Assessment

No implementation is present yet. The best path is a design-approved cross-cutting privacy contract, not a prompt-only workaround or a narrow toggle that leaves persistence paths active.
