## Summary

Keep open. Current main still lacks an enforceable private/ephemeral session contract, and the discussion makes this a privacy-sensitive product design request rather than a narrow bug fix.

## What This Changes

Add a Mission Control Private Mode that suppresses workspace context and prevents private sessions from reading/writing durable workspace, memory, or history state.

## Best Possible Solution

Keep this as the canonical design tracker and land one additive core ephemeral-session contract with tests proving private sessions neither read protected workspace/memory context nor write durable transcript, session-store, memory, plugin, media, compaction, or history artifacts.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence current main cannot express or enforce the requested mode because closed schemas lack a private flag and normal session paths write JSONL/store state.

## Solution Assessment

Unclear for implementation details, but keeping this open for maintainer product/security design is the best current path. Existing contextInjection controls can reduce injected context, but they are not a hard no-read/no-write/no-history private-session contract.
