## Summary

Keep open. Current main still cannot express or enforce a private/ephemeral session mode, and the discussion makes this a cross-cutting privacy feature rather than a narrow cleanup item.

## What This Changes

Request a Control UI Private Mode toggle and session contract that suppresses workspace context, memory, durable history, and workspace writes for private sessions.

## Best Possible Solution

Keep this as the canonical design tracker and land one additive ephemeral-session contract with tests proving private sessions neither read protected context nor write durable transcript, session-store, memory, plugin, media, compaction, or history artifacts.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence current main cannot express or enforce the requested mode because closed schemas lack a private flag and normal session paths write JSONL/store state.

## Solution Assessment

Unclear for implementation details, but keeping this open for maintainer product/security design is the best current path. Existing contextInjection controls can reduce injected context, but they are not a hard no-read/no-write/no-history private-session contract.
