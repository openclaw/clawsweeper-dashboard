## Summary

Keep open: current main still lacks an enforceable private or no-persist session mode, and the live related reports point back to this issue as the canonical tracker for the privacy/session-state contract.

## What This Changes

Request a Mission Control Private Mode that suppresses workspace context and gives sessions temporary, non-persistent memory/history semantics for demos and privacy-sensitive work.

## Best Possible Solution

Design one additive private/ephemeral session contract across Gateway protocol, session store/transcripts, bootstrap/memory, plugins/hooks/media/diagnostics, channels, and Control UI, then implement it with regression tests for no private-context reads and no durable history writes.

## Reproduction Assessment

Yes, by source inspection: current main has no private/no-persist protocol, config, SessionEntry, or Control UI field, while session creation still writes durable store metadata and transcript files. A live run was not necessary because there is no current private-mode control to exercise.

## Solution Assessment

Yes for keeping this open as the canonical tracker; no for autonomous implementation yet. The maintainable solution needs a design-approved privacy contract rather than a prompt-only or partial config toggle.
