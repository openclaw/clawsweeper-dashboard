## Summary

Keep open. Current main still lacks an enforceable private/no-persistence session contract, and the issue discussion plus related cleanup shows this remains the canonical tracker rather than a stale duplicate.

## What This Changes

Request a Private Mode in Mission Control that suppresses workspace context and prevents session, memory, and workspace reads/writes during demos or sensitive conversations.

## Best Possible Solution

Design one additive private/ephemeral session contract across Gateway protocol, SessionEntry, transcript/session storage, bootstrap and memory paths, compaction, plugin/channel visibility, and Control UI status.

## Reproduction Assessment

Yes, at source level: current main has no private/ephemeral field in the session protocol, config, persisted session entry, or UI row, while session creation and memory-flush paths still write durable state. I did not run a live session because that would create the persistence artifacts under review.

## Solution Assessment

Unclear until maintainers choose the privacy contract. A Mission Control toggle is useful only if it is backed by a shared no-read/no-write/no-history enforcement marker rather than prompt-only or UI-only behavior.
