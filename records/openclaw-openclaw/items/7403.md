## Summary

Keep open: current main and the latest release still lack an enforceable Private/Secret/Incognito session contract, while session creation and transcript storage remain durable by default.

## What This Changes

The issue requests a Mission Control Private Mode that suppresses workspace context, prevents reads/writes to real workspace memory, uses temporary scratch state, and shows a private-session UI indicator.

## Best Possible Solution

Keep this as the canonical tracker and design one additive private/ephemeral session contract across Gateway protocol, session storage/transcripts, agent context/memory, plugin enforcement, channel triggers, and Control UI.

## Reproduction Assessment

Yes. Source inspection shows current main has no private/no-persist protocol, config, SessionEntry, or UI field, while sessions.create still creates durable store and transcript artifacts.

## Solution Assessment

Yes for keeping this open. The best fix is a maintainer-approved cross-layer privacy contract rather than a partial prompt-only, UI-only, or config-only toggle.
