## Label Justifications

- `P1`: The linked bug affects current live agent sessions by silently using the wrong model/provider after a user model switch and compaction.
- `merge-risk: 🚨 auth-provider`: The diff can route an attempt or compaction through a newly selected provider/model while retaining the previous auth profile and credential setup.

- `merge-risk: 🚨 session-state`: The patch clears the pending live-switch flag after partial local mutation, which can leave persisted session switch state and runtime state out of sync.

## Summary

Keep open: this PR targets a real current bug, but the submitted fix is not merge-ready because it applies the live switch only at the next loop entry while the timeout/overflow compaction call that needs the live selection still runs earlier in the prior iteration with stale routing state.

## What This Changes

The PR adds a live-model-switch check at the top of each `runEmbeddedAgentInternal` attempt-loop iteration and updates local `provider`, `modelId`, and `effectiveModel` values when a pending switch is found.

## Best Possible Solution

Feed the pending live selection as a single provider/model/auth-profile unit into the timeout and overflow compaction runtime contexts, add focused regression coverage, and require redacted real behavior proof before merge.

## Reproduction Assessment

Yes. Source inspection shows a pending `/model` switch can be detected after an attempt, but if the attempt cannot restart cleanly, timeout/overflow compaction still builds runtime context from stale `provider`/`modelId` before the next loop iteration.

## Solution Assessment

No. The submitted loop-entry mutation is a plausible retry-path mitigation, but the best fix needs to update the actual compaction call sites and keep provider, model, auth profile, and resolved model state together.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Feed the live selection into compaction calls:** `src/agents/embedded-agent-runner/run.ts:1912`
  - body: This new check runs only at the start of the next loop iteration. When a pending `/model` switch is detected after an attempt that cannot restart cleanly, timeout/overflow compaction still executes later in that same iteration with the old `provider`/`modelId`, so the compaction API call this PR is meant to fix can still use the previous model. Please thread the pending selection into both compaction runtime contexts before calling `compact()`.
  - confidence: 0.9
- **[P1] Keep auth and resolved model state together:** `src/agents/embedded-agent-runner/run.ts:1924-1939`
  - body: Inside this block, `provider` and `modelId` are changed before the new model resolution is proven, while auth profile, API key state, auth storage, model registry, and context-window state still come from the old selection. That can dispatch a retry with a new provider/model but stale credentials or stale model metadata; use the existing restart/full-resolution path or update the whole provider/model/auth unit before clearing the pending flag.
  - confidence: 0.86
