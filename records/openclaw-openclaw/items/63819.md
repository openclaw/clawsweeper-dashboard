## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/63819. Current main stores `pendingTerminalLifecycleErrors` as `Map<string, NodeJS.Timeout>`, so a lifecycle `error` for a non-aborted run schedules terminal cleanup, but a later non-error event for the same run clears and deletes the only pending timer before `finalizeLifecycleEvent` persists terminal session state. Preserve the deferred lifecycle event and options through the retry grace window and finalize it when a later non-retry/non-terminal event would otherwise clear the pending terminal cleanup. Keep fallback-success behavior intact: lifecycle error followed by fallback/end must still avoid premature `state:error`. Add focused regression coverage in `src/gateway/server-chat.agent-events.test.ts` for lifecycle error followed by a later non-error event with no lifecycle end, proving terminal cleanup/session finalization still occurs. Do not change Telegram-specific behavior, provider retry policy, public protocol shapes, or add a config option. Add a changelog entry because this is user-visible session recovery behavior.
