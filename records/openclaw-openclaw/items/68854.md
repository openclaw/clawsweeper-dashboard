## Summary

Keep open: the underlying gateway heredoc regression is still present on current main, but this PR is not merge-ready because it relaxes a host exec approval gate with incomplete heredoc detection, malformed tests, and no after-fix real behavior proof.

## What This Changes

The PR changes gateway exec allowlist heredoc detection so quoted and backslash-quoted heredocs avoid the extra heredoc approval gate while unquoted heredocs still prompt.

## Best Possible Solution

Rework the fix around the existing analyzer or a shared heredoc-classification helper, add gateway-level regression tests for quoted, backslash-quoted, unquoted strip-tab, and unsafe expansion cases, then attach redacted real CLI or gateway proof.

## Reproduction Assessment

Yes, source-reproducible. Current main still forces approval for any allowlist-satisfied heredoc token through `requiresHeredocApproval`; this review did not run a live gateway flow because the checkout is read-only.

## Solution Assessment

No. The intended regression fix is useful, but this implementation duplicates parser behavior with an incomplete raw regex and adds tests that do not compile or assert the wrong execution boundary.
