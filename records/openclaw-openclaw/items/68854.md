## Summary

Keep open. The regression is still source-reproducible on current main, but this external PR is not merge-ready because the PR-head regex misses unquoted strip-tab heredocs, the added tests are broken, and no after-fix real behavior proof was posted.

## What This Changes

The PR changes gateway exec allowlist heredoc handling to auto-approve quoted and backslash-quoted heredocs while still requiring explicit approval for unquoted heredocs, plus related warning and test updates.

## Best Possible Solution

Use a shared heredoc classifier or analyzer result for the gateway approval decision, cover quoted, backslash-quoted, strip-tab, and unsafe unquoted cases in gateway tests, then require redacted real CLI or gateway proof before merge.

## Reproduction Assessment

Yes. Source inspection of current main shows an allowlist-satisfied heredoc still sets `requiresHeredocApproval` in the gateway path whenever a segment argv token starts with `<<`; I did not run a live gateway flow in this read-only review.

## Solution Assessment

No. The regression fix is valid, but this branch duplicates only part of heredoc parsing with a raw regex, misses unquoted `<<-` and other delimiter forms, and needs real gateway proof before it is a safe merge path.
