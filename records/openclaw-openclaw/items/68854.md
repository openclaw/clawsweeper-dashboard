## Summary

Keep open: the regression is real, but this PR is not merge-ready because the raw heredoc classifier misses unquoted strip-tab forms, the added tests do not match the helper contract, and after-fix real behavior proof is still missing.

## What This Changes

The PR changes `processGatewayAllowlist` to inspect raw heredoc text so quoted and backslash-quoted heredocs skip the extra approval gate, updates the warning copy, and adds gateway allowlist tests.

## Best Possible Solution

Align the gateway decision with the existing heredoc analyzer semantics, add focused gateway tests for quoted, backslash-quoted, unquoted strip-tab, and unsafe expansion cases, then require redacted real CLI or gateway proof before merge.

## Reproduction Assessment

Yes, source-level: current main treats any allowlist-satisfied heredoc argv token starting with `<<` as requiring approval, while the analyzer already accepts quoted literal heredocs. I did not run a live gateway flow because this review is read-only.

## Solution Assessment

No. The intended regression fix is valid, but this implementation duplicates heredoc parsing with an incomplete raw regex and adds tests that do not prove the changed gateway contract.
