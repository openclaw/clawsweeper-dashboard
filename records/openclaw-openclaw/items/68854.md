## Summary

Keep open: the regression is still source-reproducible on current main, but this PR should not merge as-is because the new raw heredoc detector misses strip-tab heredocs and the added gateway tests are malformed. It also still needs external real behavior proof before merge.

## What This Changes

The branch changes gateway exec allowlist heredoc detection to bypass the extra approval gate for quoted and backslash-quoted heredocs while keeping unquoted heredocs on the explicit-approval path, and updates related tests and warning copy.

## Best Possible Solution

Reuse the existing heredoc analyzer or a shared heredoc-classification helper in the gateway approval path, cover quoted, backslash-quoted, unquoted strip-tab, and unsafe expansion cases, then attach redacted real CLI or gateway proof before merge.

## Reproduction Assessment

Yes, source-reproducible. Current main still forces approval for any allowlist-satisfied heredoc token through `requiresHeredocApproval`; I did not run a live gateway command in this read-only review.

## Solution Assessment

No. The intended regression fix is useful, but this implementation duplicates parser behavior with an incomplete raw regex and adds tests that either do not compile or assert the wrong execution boundary.
