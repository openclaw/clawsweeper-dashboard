## Summary

Keep this PR open, but do not merge it as submitted. Current main still has the quoted-heredoc allowlist regression this PR targets, while the proposed diff has a security-sensitive approval-policy gap, broken gateway tests, and no required changelog entry.

## What This Changes

This PR changes gateway exec allowlist heredoc approval logic to inspect raw segment text, updates the approval warning copy, and adds allowlist tests for quoted, unquoted, and backslash-quoted heredocs.

## Best Possible Solution

Rework the fix around a parser-backed heredoc classification helper or existing analyzer metadata, then pin the quoted, unquoted, and strip-tab cases with `processGatewayAllowlist` contract tests and a single-line changelog entry.

## Reproduction Assessment

Yes. The linked bug provides concrete allowlist-mode steps, and current main source still forces approval for any `<<` argv token after allowlist analysis succeeds; the PR defects are reproducible by inspecting the raw regex and added test assertions.

## Solution Assessment

No. The intended behavior is reasonable, but this implementation duplicates shell parsing with an incomplete regex and adds tests that do not compile or assert the actual auto-approved gateway path.
