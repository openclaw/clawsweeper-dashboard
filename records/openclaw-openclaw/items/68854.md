## Summary

Keep open: the heredoc allowlist regression is still present on current main and in the latest release, but this PR cannot merge as-is because it relaxes a host exec approval gate with an incomplete heredoc detector, malformed tests, and no after-fix real behavior proof.

## What This Changes

The PR changes gateway exec allowlist heredoc detection so quoted and backslash-quoted heredocs bypass the extra heredoc approval gate while unquoted heredocs still require approval.

## Best Possible Solution

Rework this around the existing analyzer or a shared heredoc-classification helper, add gateway coverage for quoted, backslash-quoted, unquoted strip-tab, and unsafe expansion cases, then attach redacted real CLI or gateway proof.

## Reproduction Assessment

Yes, source-reproducible. Current main and `v2026.5.12` still force approval for any allowlist-satisfied heredoc token through `requiresHeredocApproval`; I did not run a live gateway flow in this read-only review.

## Solution Assessment

No. The intended regression fix is useful, but this implementation duplicates parser behavior with an incomplete raw regex and adds tests that do not compile or assert the correct execution boundary.
