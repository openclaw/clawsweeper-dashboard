## Summary

Keep open: the underlying allowlist heredoc regression is still present on current main and the latest release, but this PR is not mergeable as-is because the new raw-text detector misses valid unquoted heredoc forms and the added tests do not match the current gateway contract.

## What This Changes

The PR changes gateway exec allowlist heredoc detection to exempt quoted and backslash-quoted heredocs, keeps explicit approval for unquoted heredocs, updates warning copy, and adds mocked gateway tests.

## Best Possible Solution

Rework the fix around the existing analyzer or a shared heredoc-classification helper, then add gateway-level tests for quoted, backslash-quoted, unquoted strip-tab, and unsafe expansion cases plus redacted real CLI/gateway proof.

## Reproduction Assessment

Yes. Source inspection on current main shows any `<<` argv token still forces approval after allowlist analysis succeeds, and the PR defects are visible from the proposed regex and mocked tests without mutating the checkout.

## Solution Assessment

No. The intended regression fix is useful, but this implementation duplicates parser behavior with an incomplete raw regex and adds tests that do not compile or assert the wrong execution boundary.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Recognize every unquoted heredoc delimiter:** `src/agents/bash-tools.exec-host-gateway.ts:165`
  - body: The detector only matches an identifier immediately after `<<`, so unquoted forms such as `<<-EOF` and other valid non-identifier delimiters do not set `requiresHeredocApproval` even though the PR body says unquoted heredocs still need explicit approval. Reuse the analyzer's delimiter parsing or match the same heredoc grammar before relaxing the gate.
  - confidence: 0.9
- **[P2] Fix the gateway allowlist test contract:** `src/agents/bash-tools.exec-host-gateway.test.ts:424-425`
  - body: The added tests pass `warnings: []` inline but then assert an undefined `warnings` variable, so the test file will not compile. The quoted-heredoc no-approval case should also assert the returned `execCommandOverride`, because `processGatewayAllowlist` returns work to the caller on that path instead of calling `runExecProcess` directly.
  - confidence: 0.95
