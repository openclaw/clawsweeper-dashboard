## Summary

Keep open: the reported gateway allowlist heredoc regression still exists on current main, but this PR is not merge-ready because its replacement heredoc gate misses an unquoted strip-tab form and the added tests do not match the gateway helper contract.

## What This Changes

The PR changes gateway exec allowlist heredoc handling so quoted and backslash-quoted heredocs bypass the extra approval gate while unquoted heredocs still require approval.

## Best Possible Solution

Rework the gateway decision around the existing heredoc analyzer semantics or a shared classifier, add focused gateway coverage for quoted, backslash-quoted, unquoted strip-tab, and unsafe expansion cases, then require redacted real CLI or gateway proof.

## Reproduction Assessment

Yes. Source inspection of current main shows any allowlist-satisfied heredoc token still triggers `requiresHeredocApproval` in the gateway path; I did not run a live gateway flow in this read-only review.

## Solution Assessment

No. The intended regression fix is valid, but this implementation duplicates parser behavior with an incomplete raw regex and adds tests that do not compile or assert the correct execution boundary.
