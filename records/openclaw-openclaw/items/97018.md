## What This Changes

## Best Possible Solution

Resolve LM Studio embedding preload context as `contextTokens` first, then `contextWindow`, cover the actual factory-to-load-helper call, and require redacted LM Studio load-request proof before merge.

## Reproduction Assessment

Yes. Source inspection shows current main omits `requestedContextLength` in the LM Studio embedding preload and the callee falls back to the default load context; I did not reproduce the CUDA OOM live.

## Solution Assessment

No. The provider-local layer is right, but the implementation should mirror `stream.ts` by preferring `contextTokens` before `contextWindow` and should test the actual preload call.
