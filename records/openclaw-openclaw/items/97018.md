## Summary

Keep open: the PR is a useful provider-local fix direction, but it is not merge-ready because it only honors `contextWindow` while existing LM Studio config and inference preload paths also support `contextTokens`, and the contributor proof is unit-test-only.

## What This Changes

The PR changes LM Studio memory embedding preload to pass provider model `contextWindow` as `requestedContextLength` and adds a new test file for that resolution.

## Best Possible Solution

Update the LM Studio embedding provider to resolve `contextTokens` before `contextWindow`, add a production-path mock test for the preload call, and require redacted LM Studio load-request proof before merge.

## Reproduction Assessment

Yes. Source inspection shows current main omits `requestedContextLength` in the LM Studio embedding preload and the callee falls back to the default load context; I did not reproduce the CUDA OOM live.

## Solution Assessment

No. The provider-local direction is right, but the implementation should mirror `stream.ts` by resolving `contextTokens` before `contextWindow` and should test the real factory call.
