## Summary

Keep open. Current main still contains the reported false-positive Raw-mode fallback path, and the latest release tag in this checkout still has the same behavior; the linked closing PRs are unmerged implementation candidates, not grounds to close the canonical issue.

## What This Changes

The issue requests re-enabling Control UI Raw mode by removing undefined safe-bin own properties and comparing raw redaction round trips against the authored/source config shape instead of the materialized runtime config.

## Best Possible Solution

Ship a narrow maintainer-approved fix that omits empty safe-bin fields, validates raw redaction against a JSON-serializable authored/source config shape, keeps fallback behavior for ambiguous redaction collisions, and adds regression tests for the reported safe-bin, materialization drift, and short sensitive-value cases.

## Reproduction Assessment

Yes. Current source shows a high-confidence path where restored redacted raw text is compared to `snapshot.config`, while the read path separately materializes runtime config and the UI disables Raw whenever redaction returns `raw: null`.

## Solution Assessment

Yes in direction, but the exact patch needs careful review. Comparing against the authored/source config and deleting empty safe-bin fields is the narrow fix, provided the redaction fallback still blocks ambiguous or unsafe raw text exposure.
