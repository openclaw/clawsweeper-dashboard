## What This Changes

The issue reports a Control UI regression where config Raw mode is forced off because redacted raw round-trip checks compare against materialized config and preserve undefined safe-bin keys.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/59336, https://github.com/BingqingLyu/openclaw/pull/1824, or a maintainer-owned replacement that compares against a JSON-serializable authored/source config shape, handles empty safe-bin fields without weakening exec policy semantics, preserves ambiguous-redaction fallback safety, and adds focused regression coverage.

## Reproduction Assessment

Yes, at source level: current main still creates undefined safe-bin own keys, compares restored raw against `snapshot.config`, and disables Raw when the fallback nulls `raw`. I did not run a live Control UI session in this read-only review.

## Solution Assessment

Unclear for the open PRs until the safe-bin semantics and redaction-safety concerns are resolved. The maintainable direction is to compare against the authored/source-shaped config while preserving strict fallback behavior for ambiguous secret redaction.
