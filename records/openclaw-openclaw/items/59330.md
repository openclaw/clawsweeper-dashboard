## What This Changes

The issue reports a Control UI regression where Raw mode is disabled because raw redaction round-trip checks compare source-shaped raw parsing against materialized runtime config with undefined safe-bin keys.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/59336 or a replacement that compares against a JSON-serializable source-shaped config, preserves safe-bin policy safety, handles ambiguous redaction collisions, and adds focused regression coverage.

## Reproduction Assessment

Yes, at source level: current main still assigns undefined safe-bin own fields and compares restored raw text against materialized `snapshot.config`; a focused Node proof confirms the JSON round-trip equality failure. I did not run a live Control UI session in this read-only review.

## Solution Assessment

Unclear for the existing PR until the safe-bin and redaction-safety review comments are addressed. The maintainable fix is narrow: compare against the authored/source config shape without weakening secret-redaction guarantees or exec safe-bin policy semantics.
