## Summary

Keep open: current main and v2026.5.7 still contain the reported false-positive Raw-mode fallback path, and the linked implementation PRs are open rather than merged.

## What This Changes

Fix Control UI Raw mode so safe raw snapshots are not disabled by undefined safe-bin fields or source-vs-runtime materialization drift.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/59336, https://github.com/BingqingLyu/openclaw/pull/1824, or a replacement that preserves redaction safety while restoring Raw mode for safe snapshots.

## Reproduction Assessment

Yes. Source inspection plus a focused Node proof show undefined own safe-bin fields fail the JSON round-trip equality check, and the current redaction/UI path turns that failure into `raw: null` and forced Form mode.

## Solution Assessment

Unclear until the open patch is reviewed. The maintainable direction is to omit undefined safe-bin fields, compare the raw redaction round trip against a JSON-serializable authored/source shape, and keep fallback protection for ambiguous redaction collisions.
