## Summary

Keep open: current main and the latest release still contain the reported false-positive Raw-mode fallback path, and the issue is already paired with open implementation PRs that have not merged.

## What This Changes

The issue requests restoring Control UI Raw mode by fixing config snapshot redaction round-trip checks that compare materialized runtime config against raw/source-shaped config and preserve undefined safe-bin keys.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/59336, https://github.com/BingqingLyu/openclaw/pull/1824, or a maintainer-owned replacement that compares against a JSON-serializable authored/source config shape, handles empty safe-bin fields without weakening safe-bin semantics, preserves ambiguous-redaction fallback safety, and adds focused regression coverage.

## Reproduction Assessment

Yes, at source level: current main still creates undefined safe-bin own keys, compares restored raw against the materialized runtime config, and disables Raw when the fallback nulls `raw`. I did not run a live Control UI session in this read-only review.

## Solution Assessment

Unclear for the open PRs until the safe-bin semantics and raw redaction-safety concerns are resolved. The maintainable fix should use the authored/source-shaped config for the round-trip check while keeping strict fallback behavior for ambiguous secret redaction.
