## Summary

Keep open for maintainer review. Current main has an explicit `canvas.a2ui.reset` path with capability refresh, but `canvas present` still intentionally preserves an explicit non-A2UI target, and the paired open report https://github.com/openclaw/openclaw/issues/81159 may affect the real macOS recovery story.

## What This Changes

The issue requests a macOS Canvas recovery path that restores A2UI after the WKWebView navigates away or a canvas capability expires.

## Best Possible Solution

Decide the intended macOS Canvas UX, then either document and validate `canvas.a2ui.reset` as the explicit recovery command or deliberately change `canvas present` semantics with matching tests and live macOS proof.

## Reproduction Assessment

Yes for the narrow `canvas present` behavior: current source and smoke coverage show it does not force A2UI after an explicit user target. No high-confidence live reproduction was established for the full token-expiry recovery failure on current main because the reset path now exists in source.

## Solution Assessment

Unclear. Forcing every `canvas present` back to A2UI is not clearly the narrowest maintainable fix because it would change target-preservation behavior; an explicit reset/recovery command may be the safer product direction if live proof confirms it works.
