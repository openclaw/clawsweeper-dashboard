## Summary

Keep open: current main has generic sessions, labels, cross-session tools, and plugin-owned session extension state, but it still lacks a first-class skill-facing project context API plus native project-scoped Control UI and history behavior.

## What This Changes

Request first-class project/thread awareness for skills, including project metadata, Control UI display/switching, project-scoped history, and cross-project queries.

## Best Possible Solution

Design an additive project-context contract if maintainers want first-class project semantics, starting from existing session and plugin extension seams and then wiring Gateway protocol, SDK or skill-facing access, Control UI display, and history filtering consistently.
