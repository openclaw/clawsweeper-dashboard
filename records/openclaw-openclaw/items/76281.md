## Summary

Keep open: current main has adjacent session labels, session tools, and plugin-owned session extension state, plus a related skill-only workaround that was directed to ClawHub, but it still lacks the requested skill-facing project context API, native UI project display/switching, project-scoped history filters, and cross-project query contract.

## What This Changes

Request first-class project/thread awareness for skills, including session project metadata, UI project display/switching, project-filtered history, and cross-project queries.

## Best Possible Solution

Define an additive project-context contract, choose the state and visibility owner, then wire Gateway schema, skill/SDK access, Control UI, and history filtering consistently.

## Reproduction Assessment

Not applicable: this is a feature request, not a bug report. Source and docs inspection gives a high-confidence comparison showing adjacent session/plugin primitives exist while the requested project contract does not.

## Solution Assessment

Unclear until maintainer design. A one-off session.set_project API would be premature before choosing whether project state belongs in plugin extensions, first-class session metadata, or a broader project model with access rules.
