## Summary

Keep open: current main has generic sessions, labels, session switching, cross-session tools, and plugin-owned session extensions, but it still lacks a first-class skill-facing project context API plus native project-scoped Control UI/history semantics for the requested workflow.

## What This Changes

Request first-class project/thread awareness for skills, including project metadata, Control UI display and switching, project-scoped history, and cross-project query support.

## Best Possible Solution

Define an additive, versioned project-context contract around sessions/skills if maintainers want first-class semantics, then wire Control UI display/filtering and history access consistently.

## Reproduction Assessment

Not applicable: this is a feature request, not a bug report. Source inspection gives a high-confidence current-main comparison showing generic session surfaces exist but the requested project metadata/API does not.

## Solution Assessment

Unclear. The proposed `session.set_project` shape is a useful sketch, but the maintainable path should first choose whether project context is plugin-owned extension state or a core session metadata contract.
