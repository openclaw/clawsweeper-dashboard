## Summary

Keep open: current main supports generic sessions, labels, cross-session tools, plugin-owned session extensions, and a chat session picker, but it still lacks the requested first-class skill-facing project API plus native project-scoped UI/history behavior.

## What This Changes

The issue requests first-class project/thread awareness for skills, including project metadata, Control UI display/switching, project-scoped history, and cross-project queries.

## Best Possible Solution

Design an additive project-context contract only if maintainers want first-class project semantics, starting from existing session and plugin extension seams and then wiring Gateway protocol, SDK or skill-facing access, Control UI display, and history filtering consistently.
