## Summary

Close as `clawhub`: current main already has the generic session, label, UI switching, cross-session tool, and plugin session-extension surfaces needed for a Project Atlas-style workflow, while the requested project manager behavior is optional skill/plugin packaging rather than a core built-in.

## What This Changes

Request first-class project metadata, project-thread switching, project-scoped history, and cross-project querying for skills and the Control UI.

## Best Possible Solution

Keep core on the generic session, label, and plugin-extension contracts, and implement Project Atlas or project-manager behavior as a ClawHub plugin/skill bundle unless a narrower missing API is identified.

## Reproduction Assessment

Not applicable; this is a feature request rather than a bug. I verified the current behavior by inspecting the checked-in docs, session schemas, session listing code, Control UI session selector, and plugin host-hook API.

## Solution Assessment

Yes. For this cleanup pass, ClawHub/plugin routing is the narrowest maintainable solution because current main already exposes generic session labels, session switching, cross-session tools, and plugin-owned session state without adding a core project abstraction.
