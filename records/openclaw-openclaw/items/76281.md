## Summary

Keep open: current main has generic sessions, labels, session switching, cross-session tools, and plugin-owned session extensions, but it still lacks a first-class skill-facing project id/name contract and native Control UI/history project scoping for the requested workflow.

## What This Changes

Add project/thread awareness for skills, including project declaration, UI project display and switching, project-scoped history, and cross-project lookup.

## Best Possible Solution

Define an additive, versioned project-context contract around sessions that plugins/skills can use, then wire consistent Control UI display, filtering, switching, and cross-session history access if maintainers choose first-class project semantics.

## Reproduction Assessment

Not applicable: this is a feature request, not a bug report. Source inspection gives a high-confidence current-main comparison showing generic session primitives exist but the requested skill project API and UI/history contract do not.

## Solution Assessment

Unclear. The proposed session.set_project shape is a useful sketch, but maintainers should first choose whether project context is core session metadata or a plugin-owned extension with explicit Control UI/history support.
