## Summary

## What This Changes

The branch updates gateway chat completion handling to broadcast `state: "error"` and cache an error dedupe result when delivered agent replies include `isError: true`, with regression tests for single and multiple error payloads.

## Best Possible Solution

Land the gateway error-event and error-dedupe behavior after preserving the existing transcript-update wait/catch semantics and adding real after-fix proof for the reported gateway flow.

## Reproduction Assessment

Yes. Source inspection shows current main collects returned `isError` replies, then the `agentRunStarted` post-dispatch branch reaches only the transcript update path and never broadcasts a chat error event; the linked issue includes matching JSONL prompt-error evidence.

## Solution Assessment

Not yet. The proposed gateway branch is the right ownership boundary for the fix, but it should preserve the existing transcript update wait/catch behavior and be backed by real gateway/ACP proof before merge.
