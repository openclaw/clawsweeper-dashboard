## Summary

Keep open: the updated proof now demonstrates the clean generated-path reset case, but the patch still preserves already-stale generated filenames from the linked upgrade scenario and is missing the required changelog entry.

## What This Changes

The PR extracts generated session-file rewriting into shared session helpers, applies it during Gateway session reset for direct session-store transcript paths, and adds helper/reset regression tests.

## Best Possible Solution

Make Gateway reset rotate any direct-store OpenClaw-generated transcript basename to the new session id, even when the embedded id differs from the current entry, while preserving explicit custom paths and documenting the user-visible fix.

## Reproduction Assessment

Yes. Current main is source-reproducible because sessions.reset generates a new session id but passes the prior sessionFile to a resolver that prefers it; the linked issue and PR body provide concrete TUI/Gateway evidence for the filename/header-id mismatch.

## Solution Assessment

No. The helper extraction is a narrow direction, but the reset path should classify direct-store generated filenames by their embedded id and rotate already-stale generated filenames too; it also needs the required changelog entry.
