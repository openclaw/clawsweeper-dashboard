## Summary

Keep open. Current main still lacks the requested additive agent-level prompt/rules config surface; the implemented config path is replacement-style `systemPromptOverride`, and the directly related implementation PR is closed unmerged while nearby open PRs cover narrower prompt controls.

## What This Changes

Request adds config-level `agents.defaults.systemPrompt`/`rules` and per-agent prompt/rules injection, including optional `$include`, so centrally managed rules can apply across sessions.

## Best Possible Solution

Define one additive agent prompt contribution contract that coexists with `systemPromptOverride`, merges defaults and per-agent text predictably, applies across CLI, embedded, cron, subagent, and channel-created sessions, and updates schema, generated docs, and focused prompt tests.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is source inspection on current main, which shows only replacement-style `systemPromptOverride` at the schema, generated schema, docs, and runner merge points.

## Solution Assessment

Unclear until maintainers choose the public API, include handling, and merge semantics. The current `systemPromptOverride` path is not the requested solution because it replaces the assembled prompt rather than appending centrally managed rules.
