## Summary

Keep open. Current main and v2026.6.10 still lack an agent-level `agents.defaults.systemPrompt` or `rules` contract, and this remains the canonical product/security decision for config-injected prompt rules.

## What This Changes

Request adding `agents.defaults.systemPrompt` or `rules`, plus per-agent overrides, so persistent agent behavior rules can be injected from `openclaw.json` instead of workspace bootstrap files.

## Best Possible Solution

Define one maintainer-approved agent-level append prompt/rules contract with explicit precedence, session-creation coverage, config schema/help, docs, tests, and security limits, while coordinating raw text includes and hard enforcement with their narrower issues.

## Reproduction Assessment

Not applicable as a bug reproduction, but yes at source level: current strict config schemas do not accept the requested fields, and prompt assembly does not resolve them.

## Solution Assessment

Unclear until maintainers decide the product/security boundary. The narrow maintainable direction is one append-style agent prompt/rules contract first, with `$include` and hard enforcement kept in their separate design tracks.
