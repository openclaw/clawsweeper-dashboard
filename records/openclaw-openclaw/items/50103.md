## Summary

Keep open. Current main still exposes only replacement-style `systemPromptOverride` at agent/default scope; the requested additive `systemPrompt`/`rules` contract is not in schema, generated config metadata, docs, or the CLI/embedded prompt assembly paths. The direct implementation PR is closed unmerged, and the open adjacent `systemPromptFile` PR covers only a file-based subset, so this still needs maintainer product/security judgment.

## What This Changes

Request adding config-managed additive agent prompt rules via `agents.defaults.systemPrompt`/`rules` and per-agent equivalents, with optional `$include`, across OpenClaw agent sessions.

## Best Possible Solution

Define one additive agent prompt contribution contract that coexists with `systemPromptOverride`, predictably appends defaults and per-agent contributions, uses the existing safe config include pipeline where appropriate, applies consistently across CLI, embedded, cron, subagent, and channel-started sessions, and updates schema/generated docs plus focused prompt-resolution tests.

## Reproduction Assessment

Not applicable as a feature request. The verification path is high-confidence source inspection: current schemas, docs, generated metadata, and runner code show replacement-only `systemPromptOverride`, not additive config prompt/rules behavior.

## Solution Assessment

Unclear until maintainers choose the public config contract and security semantics. The existing `systemPromptOverride` is not sufficient because it replaces the OpenClaw-built prompt, while PR #41807 is only a partial file-based approach rather than the requested inline/rules merge contract.
