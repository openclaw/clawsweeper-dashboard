## Summary

Keep this open: current main still lacks the requested append-style `agents.defaults.systemPrompt` / `agents.defaults.rules` contract, and the existing `systemPromptOverride` is documented and implemented as a full replacement. The duplicate feature issue was closed into this issue, while the exact implementation PR is closed unmerged and the open related PRs cover adjacent prompt controls rather than this canonical request.

## What This Changes

Request to add config-level `agents.defaults.systemPrompt` / `rules` and per-agent equivalents for append-style rule injection into agent system prompts.

## Best Possible Solution

Settle one canonical agent-level prompt-injection contract, then implement schema/types/docs/config metadata plus prompt resolution at session creation for main, cron, subagent, and channel-triggered runs.
