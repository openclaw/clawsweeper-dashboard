## Summary

Keep this open: current main still lacks the requested append-style `agents.defaults.systemPrompt` / `rules` config, and the existing `systemPromptOverride` is a full replacement rather than the same capability. The related duplicate was closed into this issue, while the exact implementation PR was closed unmerged and the remaining open PRs cover adjacent but different prompt controls.

## What This Changes

Request adds config-managed agent-level system prompt/rules under agent defaults and per-agent config so rules can be injected consistently across workspaces, channels, cron, and subagents.

## Best Possible Solution

Settle one canonical agent-level prompt-injection contract, then implement it through the strict config schema, per-agent resolution, session creation paths, docs, and regression tests for main, cron, subagent, and channel runs.

Not applicable. This is a feature request, and source inspection confirms current main only has the replacement-style `systemPromptOverride` contract rather than the requested append-style config fields.

## Solution Assessment
