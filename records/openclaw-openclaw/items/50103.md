## Summary

Keep open. Current main and v2026.6.10 still lack the requested agent-level `agents.defaults.systemPrompt`/`rules` and per-agent prompt-rule config contract, and the remaining work is a security-sensitive product/config decision rather than a narrow bug fix.

## What This Changes

The issue requests agent-level config fields for injecting default and per-agent system prompt rules from `openclaw.json`, including optional rules arrays and `$include` support.

## Best Possible Solution

Define one canonical, maintainer-approved agent-level prompt/rules contract with explicit precedence, session-creation coverage, docs, schema/help updates, and security limits; coordinate `$include` with the narrower include issue and hard enforcement with the pre-response gate issue.

## Reproduction Assessment

Yes for the feature gap, not as a bug reproduction: strict current-main schemas reject the requested fields and the prompt builders do not resolve config prompt/rules text. No runtime failure path is needed to show the requested capability is missing.

## Solution Assessment

Unclear until maintainers decide the product/security boundary. The narrow maintainable path is likely an append-style agent prompt/rules contract first, with raw `$include` and hard enforcement handled by their canonical related issues.
