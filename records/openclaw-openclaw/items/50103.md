## Summary

The issue requests global and per-agent config fields for injecting prompt/rules text into agent system prompts, plus optional file inclusion for those rules.

## Best Possible Solution

Define one canonical operator-owned prompt/rules injection contract with schema, docs, trust-boundary policy, deterministic precedence, and coverage across main, cron, subagent, CLI, embedded, and channel-triggered sessions.

Not applicable as a bug reproduction; this is a feature/config request. Current source, docs, tests, and v2026.6.6 release source give high-confidence proof that the requested append-style fields are not implemented.

## Solution Assessment

Unclear: a core config field is plausible, but maintainers still need to choose the prompt precedence, trust boundary, include semantics, and hard-versus-advisory behavior before implementation.
