## Summary

Keep open. Current main still only supports replacement-style `systemPromptOverride`; additive default/per-agent `systemPrompt` or `rules` injection and `$include` handling are absent from the schemas, docs, generated schema, and runner merge path. The direct implementation PR is closed unmerged, while the open related PRs cover adjacent but narrower prompt controls.

## What This Changes

The request adds centrally managed additive agent system prompt/rules config under `agents.defaults` and per-agent entries, with optional `$include` support.

## Best Possible Solution

Define one additive agent prompt contribution contract that coexists with `systemPromptOverride`, merges defaults and per-agent text predictably, applies across CLI, embedded, cron, subagent, and channel-created sessions, and updates schema/generated docs plus focused prompt tests.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is source inspection on current main: schemas, generated schema, docs, and runner code show only replacement-style `systemPromptOverride`, not additive config prompt/rules behavior.

## Solution Assessment

Unclear until maintainers choose the public API, include handling, and merge semantics. The current `systemPromptOverride` path is not the requested solution because it replaces the assembled prompt rather than appending centrally managed rules.
