## Summary

Keep open: current main and the latest release still provide whole-prompt replacement and plugin prompt mutation, not the requested append-style agent/default rules config. The feature remains product- and security-sensitive because it would define a new first-class prompt-injection contract.

## What This Changes

Request to add config-managed agent system prompt rules through `agents.defaults.systemPrompt` or `agents.defaults.rules`, with per-agent overrides and optional includes.

## Best Possible Solution

Settle the prompt-rule product and security boundary first; if core config is accepted, implement schema, types, docs, metadata, gateway mutation policy, and session-creation resolution across main, cron, subagent, and channel-triggered runs.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source, docs, release-tag inspection, and related GitHub state give high confidence that current main lacks the requested append-style agent prompt fields.

## Solution Assessment

Unclear. A core config field is plausible, but current replacement fields and plugin hooks are different surfaces, so maintainers need to choose the first-class config/UI contract and security boundary before implementation.
