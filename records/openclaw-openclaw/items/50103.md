## Summary

Keep open. Current `main` has whole-prompt replacement through `agents.defaults.systemPromptOverride`, but it does not implement the requested additive `agents.defaults.systemPrompt` / `rules` contract, per-agent additive overrides, or include/file semantics; the direct implementation PR is closed unmerged and the open related PRs are partial alternatives that still need maintainer API/security judgment.

## What This Changes

Request adds config-level agent system-prompt/rules injection under `agents.defaults` and per-agent config so rules can be centrally managed outside workspace bootstrap files.

## Best Possible Solution

Define one additive, schema-backed prompt-default contract with clear precedence, per-agent merge behavior, include/file trust boundaries, docs, generated config metadata, and runtime coverage for CLI, embedded, cron, subagent, and channel sessions.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source and docs inspection on current `main` gives high confidence that the requested `agents.defaults.systemPrompt`/`rules` and per-agent equivalents are absent, while only whole-prompt `systemPromptOverride` exists.

## Solution Assessment

Unclear. Additive config-level prompt rules are plausible, but maintainers should first settle the supported API shape, precedence, include/file semantics, and trust boundary rather than merging another partial prompt customization path.
