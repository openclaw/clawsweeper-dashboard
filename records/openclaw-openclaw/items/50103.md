## Summary

Keep this open: current `main` and the latest release still lack the requested append-style `agents.defaults.systemPrompt` / `agents.defaults.rules` contract, while the existing `systemPromptOverride` path is replacement-only. The exact implementation PR is closed unmerged, the duplicate feature issue was folded into this issue, and the remaining open PRs cover adjacent prompt controls rather than this canonical request.

## What This Changes

The issue requests config-level agent prompt injection via `agents.defaults.systemPrompt` or `agents.defaults.rules`, with optional per-agent overrides and include-file support.

## Best Possible Solution

Settle one canonical agent-level prompt-injection contract, then implement schema, types, docs/config metadata, config mutation policy, and prompt resolution at session creation for CLI, embedded, cron, subagent, and channel-triggered runs.
