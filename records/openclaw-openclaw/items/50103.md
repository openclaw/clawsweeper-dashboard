## Summary

Keep open. Current main still exposes replacement-style `systemPromptOverride`, while the requested additive `agents.defaults.systemPrompt` / `rules`, per-agent additive merge behavior, and `$include` handling are not implemented; the direct implementation PR is closed unmerged and the open adjacent prompt PRs do not cover the full requested contract.

## What This Changes

Add config-level agent prompt/rules fields that inject centrally managed instructions into all agent sessions, with default and per-agent merging plus optional includes.

## Best Possible Solution

Define one additive agent prompt contribution contract that coexists with `systemPromptOverride`, merges defaults and per-agent text predictably, applies across CLI, embedded, cron, subagent, and channel-created sessions, and updates schema/generated docs plus focused prompt tests.

## Reproduction Assessment

Not applicable as a feature request. The verification path is high-confidence source inspection: schemas, generated schema, docs, and runner code on current main show only replacement-style `systemPromptOverride`, not additive config prompt/rules behavior.

## Solution Assessment

Unclear until maintainers choose the public API and include/security semantics. The current `systemPromptOverride` path is not the best solution for this request because it replaces the built OpenClaw prompt instead of appending centrally managed rules.
