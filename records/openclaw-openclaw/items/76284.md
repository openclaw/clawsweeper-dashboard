## Summary

Keep open. Current main still lets the interactive `agents add` wizard perform uncached model catalog validation before committing the new agent, which matches the reported blocked wizard and the closed-but-unmerged related fix proposal.

## What This Changes

The issue asks for interactive `openclaw agents add` to create secondary agents like Jon and Atlas and show them in `openclaw agents list` instead of hanging or failing.

## Best Possible Solution

Skip catalog validation in the interactive `agents add` warning call, keep the auth warning behavior, add a regression proving no catalog load occurs, and document the user-facing fix in the changelog.

## Reproduction Assessment

Yes, from source inspection rather than a live CLI run. Interactive `openclaw agents add` reaches `warnIfModelConfigLooksOff` without `validateCatalog: false`, and that helper performs uncached catalog/provider discovery unless explicitly told not to.

## Solution Assessment

Yes. The narrowest maintainable fix is the same call-site policy already used by the setup wizard: pass `validateCatalog: false` in `agents add`, then cover it with a focused regression test.
