## Summary

Keep open. Current main and the latest release still run the interactive `agents add` wizard through `warnIfModelConfigLooksOff` without disabling catalog validation, while the helper defaults to an uncached model-catalog load; the linked fix PR was closed unmerged for queue policy, not because the fix was superseded.

## What This Changes

The issue reports that interactive `openclaw agents add` blocks before creating secondary agents such as Jon or Atlas.

## Best Possible Solution

Make the interactive `agents add` flow skip live catalog validation while preserving auth warnings and channel setup, then add a regression that proves the wizard passes `validateCatalog: false`.
