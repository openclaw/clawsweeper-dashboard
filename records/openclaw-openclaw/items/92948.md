## Summary

Keep open: the linked WhatsApp session-loss bug is still valid and current main still lacks a pre-logout session backup, but this PR is not merge-ready because backup copy failures are swallowed before credentials are deleted and the posted proof is only a simulated logout run.

## What This Changes

## Best Possible Solution

Keep the WhatsApp-owned backup-before-clear mitigation, but make backup success/failure explicit, cover both owned and legacy auth-dir branches, and require redacted real setup proof for the plugin-update-triggered logout path before merge.

## Reproduction Assessment

Yes, at source level: current main reaches `logoutWeb()` from logged-out/reconnect cleanup and deletes managed WhatsApp auth directories, and no current pre-logout backup exists. I did not run a live WhatsApp plugin update in this read-only review.

## Solution Assessment

No for the current patch shape. Backup-before-clear is an acceptable owner-local mitigation, but the implementation must not swallow copy failures and then log a successful backup before deleting credentials.
