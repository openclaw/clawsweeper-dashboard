## Summary

Keep open: this PR targets the right TTS summarization boundary, but the selected internal-scaffolding sanitizer preserves some reasoning/control text and the supplied proof does not exercise the patched summarizeText-to-speech handoff.

## What This Changes

The PR wraps TTS summarization output with `stripAssistantInternalScaffolding()` and adds two mocked `summarizeText` regression tests for closed `<thinking>` blocks.

## Best Possible Solution

Use a TTS/spoken-output sanitizer at `summarizeText` with strict reasoning/control cleanup and production-path proof before closing the linked TTS leak issue.

## Reproduction Assessment

Yes. Source inspection shows `summarizeText` can return model text containing reasoning/control markup, and `maybeApplyTtsToPayload` then sends `summary.summary` to `textToSpeech` and `spokenText`; I did not run live audio.

## Solution Assessment

No. `summarizeText` is the right owner boundary, but `stripAssistantInternalScaffolding()` is not the best spoken-output cleanup because its profile intentionally preserves some reasoning text; a strict TTS/delivery sanitizer is safer.
