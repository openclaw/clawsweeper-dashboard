## Best Possible Solution

Keep the `externalAbort` guard scoped to timeout-triggered compaction, preserve timeout error payload synthesis, and add redacted real Stop-button proof before merge.

## Reproduction Assessment

Yes by source inspection: current main can receive `timedOut=true`, `aborted=true`, and `externalAbort=true`, and the compaction guard does not check `externalAbort`. I did not run a live gateway Stop-button reproduction in this read-only review.

## Solution Assessment

No. The compaction guard is the right narrow fix, but applying the same guard to `timedOutDuringPrompt` can suppress the timeout error payload that the new test expects.
