## Summary

Keep open: this draft PR targets a valid timeout-compaction bug, but the diff still has a P2 correctness regression because it suppresses timeout payload synthesis along with compaction, and the external contributor proof gate is still unmet.

## What This Changes

The PR adds an `externalAbort` guard to timeout-triggered compaction and prompt-timeout handling, adds a regression test for user-aborted high-context timeouts, and adds a changelog entry.

## Best Possible Solution

Add `!externalAbort` only to the timeout-triggered compaction guard, leave timeout payload synthesis independent, update the regression test accordingly, and attach redacted real Stop-button proof.

## Reproduction Assessment

Yes, by source inspection. With `timedOut=true`, `aborted=true`, `externalAbort=true`, high prompt usage, and no messaging delivery, the patched `timedOutDuringPrompt` becomes false and skips the explicit timeout error branch.

## Solution Assessment

No. Skipping compaction on user aborts is a narrow fix direction, but gating `timedOutDuringPrompt` mixes compaction policy with terminal timeout error rendering.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Do not gate timeout payload synthesis on externalAbort:** `src/agents/pi-embedded-runner/run.ts:2514`
  - body: This predicate also controls the explicit timeout error return. With `timedOut=true`, `aborted=true`, and `externalAbort=true`, the patch skips compaction and then makes `timedOutDuringPrompt` false, so the timeout payload branch is bypassed; aborted payload building and incomplete-turn fallback do not restore that error. Keep this predicate unchanged and only gate the compaction trigger.
  - confidence: 0.92
