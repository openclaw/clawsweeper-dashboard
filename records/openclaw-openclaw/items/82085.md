## Summary

Keep open: the underlying abort/timeout-compaction bug is plausible from current source, but the PR still introduces a terminal timeout-payload regression and lacks after-fix real behavior proof.

## What This Changes

The PR adds an `externalAbort` guard to timeout-triggered compaction and `timedOutDuringPrompt`, adds a regression test, and adds a changelog entry.

## Best Possible Solution

Guard only the timeout-compaction retry decision with `externalAbort`, preserve prompt-timeout error payload synthesis, and require redacted real runtime proof of the Stop-button path before merge.

## Reproduction Assessment

Yes, by source inspection: current main can combine a caller abort marked `externalAbort` with timeout state, high prompt usage, and no messaging delivery, while the compaction gate ignores `externalAbort`. I did not run a live Control UI Stop-button reproduction in this read-only review.

## Solution Assessment

No. Adding `externalAbort` to the compaction gate is the right narrow direction, but adding the same guard to `timedOutDuringPrompt` suppresses the explicit timeout error payload path.
