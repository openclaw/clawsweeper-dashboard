## Summary

Keep this PR open: the reported bug is source-reproducible, but the current patch also changes timeout payload synthesis and can drop the explicit timeout error the new test expects.

## What This Changes

The PR adds an `externalAbort` guard to timeout-triggered compaction and `timedOutDuringPrompt`, adds a regression case, and adds a changelog entry.

## Best Possible Solution

Keep `externalAbort` out of timeout-triggered compaction only, preserve the prompt-timeout error payload predicate, and require redacted real Stop-button proof before merge.

## Reproduction Assessment

Yes, by source inspection: current main can reach `timedOut=true`, `aborted=true`, `externalAbort=true`, high prompt usage, and no messaging-tool delivery, causing timeout compaction to run. I did not run a live gateway Stop-button reproduction in this read-only review.

## Solution Assessment

No. Guarding timeout-triggered compaction is the narrow fix, but adding the same guard to `timedOutDuringPrompt` changes terminal error payload behavior and should be split or reverted.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:
