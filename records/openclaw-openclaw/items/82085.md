Keep open: the bug-fix direction is valid, but the current PR still couples timeout-compaction policy to timeout error payload synthesis and lacks required after-fix real behavior proof from a Stop-button run.

## What This Changes

## Best Possible Solution

Keep the `externalAbort` guard scoped to timeout-triggered compaction, preserve timeout error payload synthesis, restore the unrelated changelog entry, and require redacted real Stop-button proof before merge.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence current-main path with `timedOut=true`, `aborted=true`, `externalAbort=true`, high prompt usage, and no messaging-tool delivery; I did not execute the PR branch because this review is read-only.

## Solution Assessment

No. Adding `!externalAbort` to the compaction trigger is the narrow fix direction, but adding it to `timedOutDuringPrompt` mixes retry policy with terminal timeout error synthesis.
