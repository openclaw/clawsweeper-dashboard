## Summary

Keep open. The draft PR uses the right discriminator for skipping timeout-recovery compaction, but it also applies `externalAbort` to the timeout-payload classifier, which drops the explicit timeout error path and contradicts the added test; the external PR also lacks real behavior proof.

## What This Changes

This draft PR adds `externalAbort` gating to embedded-runner timeout compaction and timeout prompt classification, with a focused test and changelog entry.

## Best Possible Solution

Guard timeout-triggered compaction with `externalAbort`, keep prompt timeout payload synthesis intact unless maintainers explicitly choose a different Stop-button terminal UX, and cover both internal timeouts and user-aborted timeouts with focused runner tests.

## Reproduction Assessment

Yes, source-reproducible. In the PR head, `externalAbort` makes `timedOutDuringPrompt` false; current payload and incomplete-turn helpers then do not synthesize the timeout error that the new test expects, and I did not run tests because this review is read-only.

## Solution Assessment

No. `externalAbort` is the right narrow discriminator for skipping compaction, but applying it to `timedOutDuringPrompt` changes terminal payload handling and should either be removed or paired with an explicit, reviewed UX change.
