## Summary

This PR should stay open: it is an active external draft, and source review confirms a P2 regression in the proposed patch where the `externalAbort` guard suppresses prompt-timeout error payload synthesis along with timeout compaction. The PR body is still WIP and does not include after-fix real behavior proof.

## What This Changes

The PR changes PI embedded runner timeout handling to skip timeout-triggered compaction when `externalAbort` is true, adds a focused regression test, and adds a changelog entry.

## Best Possible Solution

Split the predicates so `externalAbort` suppresses only timeout-triggered compaction while prompt-timeout payload synthesis remains independent and still returns a clear terminal timeout message when needed.

## Reproduction Assessment

Yes for the PR-introduced regression by source inspection: an attempt with `timedOut=true`, `aborted=true`, `externalAbort=true`, high prompt usage, and no messaging-tool delivery makes the patched `timedOutDuringPrompt` false and bypasses timeout payload synthesis. The original real Stop-button behavior still needs contributor proof.

## Solution Assessment

No. `externalAbort` is the right discriminator for the compaction guard, but applying it to `timedOutDuringPrompt` mixes compaction policy with terminal response synthesis.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Do not gate timeout payload synthesis on externalAbort:** `src/agents/pi-embedded-runner/run.ts:2514`
  - body: This boolean also controls the explicit timeout error return. With `timedOut=true`, `aborted=true`, and `externalAbort=true`, the patch skips compaction and then makes `timedOutDuringPrompt` false, so the timeout payload branch is bypassed. Since aborted runs suppress assistant output and incomplete-turn fallback returns null for aborted or timed-out attempts, this can return with no error payload instead of the timeout text the new test expects.
