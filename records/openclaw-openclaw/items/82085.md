## Summary

Keep open: the reported Stop-button/timeout compaction bug is credible from source inspection, but the PR still changes timeout error payload synthesis and lacks real after-fix behavior proof, so it should not merge as-is.

## What This Changes

The PR adds an `externalAbort` guard to timeout-triggered compaction and `timedOutDuringPrompt`, plus a regression test and changelog entry for user Stop aborts.

## Best Possible Solution

Keep the `externalAbort` guard scoped to timeout-triggered compaction, preserve `timedOutDuringPrompt` timeout payload synthesis, and require redacted real Stop-button proof before merge.

## Reproduction Assessment

Yes, by source inspection: current main can carry `timedOut=true`, `aborted=true`, and `externalAbort=true` into a compaction predicate that does not check `externalAbort`. I did not run a live gateway Stop-button reproduction in this read-only review.

## Solution Assessment

No, not as currently proposed. Guarding timeout-triggered compaction is the narrow fix, but adding the same guard to `timedOutDuringPrompt` changes terminal timeout error payload behavior and should be removed or separately justified.
