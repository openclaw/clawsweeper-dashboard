## Summary

The latest head still has a blocking predicate regression: the compaction guard direction is valid, but the same `externalAbort` guard is also applied to timeout payload synthesis. The external contributor proof gate is also still missing real after-fix Stop-button evidence.

## What This Changes

The PR adds an `externalAbort` guard around timeout-triggered compaction and timeout-prompt handling, adds a timeout-compaction regression test, and edits the unreleased changelog.

## Best Possible Solution

Keep the compaction guard change scoped to timeout-triggered compaction, preserve timeout error payload synthesis for externally aborted prompt timeouts, restore the unrelated changelog text, and require redacted real Stop-button proof before merge.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path with `timedOut=true`, `aborted=true`, `externalAbort=true`, high prompt usage, and no messaging-tool delivery; the PR skips compaction but also disables the timeout payload predicate for that same path.

## Solution Assessment

No. Adding `!externalAbort` to the compaction trigger is the narrow direction, but adding it to `timedOutDuringPrompt` mixes compaction retry policy with terminal timeout error synthesis.
