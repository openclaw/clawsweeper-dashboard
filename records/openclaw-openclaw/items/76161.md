## Summary

Keep open. The direction is useful, but the PR is not merge-ready because the non-Linux `ps` argv probe can truncate the exact long LaunchAgent/Homebrew command line this fix targets, and the user-facing gateway fix still needs a changelog entry.

## What This Changes

The PR extends Unix stale gateway PID detection to recognize node/nodejs/bun listeners by reading process argv and adds a regression test for a node-launched OpenClaw gateway.

## Best Possible Solution

Keep the focused argv-based detection, but request unbounded `ps` output on non-Linux Unix, add a regression for that path plus the required changelog entry, and then merge after changed-lane validation.

## Reproduction Assessment

Yes. Current main can be reproduced by parsing lsof output where the listener command is `node` and the argv is an OpenClaw gateway; the existing parser ignores that listener, and the PR test models the Linux `/proc/<pid>/cmdline` form.

## Solution Assessment

No, not yet. Reusing `isGatewayArgv` is the right narrow approach, but the non-Linux implementation should request full-width `ps` output before this reliably solves the reported LaunchAgent/Homebrew case.
