## Summary

Keep open. The gateway bug is real and the argv-based direction is good, but the latest head is not merge-ready: non-Linux argv probing can still miss long LaunchAgent commands without wide `ps` output, the head now includes unrelated memory-core watcher behavior, changelog coverage is missing, and exact-head checks are not green.

## What This Changes

The PR extends stale gateway listener detection to inspect `node`/`nodejs`/`bun` process argv and, in a later commit, adds file-count caps for memory-core and QMD watchers.

## Best Possible Solution

Land a focused gateway detector that uses untruncated argv, has regression coverage and changelog, and split the memory watcher cap into its own reviewed change if desired.

## Reproduction Assessment

Yes. Source inspection of current main shows the Unix lsof parser only accepts command names containing `openclaw` while service installs can launch as `node <entrypoint> gateway`; the non-Linux PR regression is also source- and docs-reproducible through truncated `ps` output.

## Solution Assessment

No for the latest head. The argv-based gateway direction is the right narrow fix, but the implementation must request wide `ps` output and the unrelated memory watcher cap should be split or separately reviewed before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Request wide ps output for argv detection:** `src/infra/restart-stale-pids.ts:120-123`
  - body: On non-Linux platforms, the new detector parses `ps -p <pid> -o command=`. `ps` can truncate command output unless wide mode is requested, so long Homebrew/LaunchAgent paths can lose the `dist/index.js` or `gateway` token and the stale PID is still missed. Add `-ww` or the platform-compatible equivalent and cover a long command line.
  - confidence: 0.91
- **[P2] Honor the QMD collection glob when pre-counting:** `extensions/memory-core/src/memory/qmd-manager.ts:1631`
  - body: For glob collections, the new cap walks the whole collection root and only checks Markdown suffixes, ignoring the configured pattern. A collection such as `*.md` or `notes/**/*.md` can be disabled because unrelated Markdown elsewhere under the root exceeds the cap, even though chokidar would watch a smaller set.
  - confidence: 0.82
- **[P3] Add required changelog coverage:** `src/infra/restart-stale-pids.ts:120`
  - body: The PR now contains user-facing gateway and memory watcher fixes, but the head changelog has no matching Unreleased Fixes entry. Add a single-line entry, or split the memory commit and document whichever user-facing fixes remain.
  - confidence: 0.95
