## Summary

Keep open. The PR addresses a real current-main stale gateway listener gap, but the proposed non-Linux argv probe can still miss long LaunchAgent/Homebrew command lines because it calls BSD/macOS `ps` without wide output, and the required user-facing changelog entry is missing.

## What This Changes

The branch extends Unix stale gateway listener detection to inspect `node`/`nodejs`/`bun` process argv and adds a LaunchAgent-style regression test.

## Best Possible Solution

Keep the argv-based detector, request untruncated BSD/macOS `ps` output, add focused coverage for that path, and add one Unreleased Fixes changelog bullet before merge.

## Reproduction Assessment

Yes. Current main can be reproduced with lsof output whose command field is `node`; the parser only accepts command names containing `openclaw`, while service installs can launch the gateway as `node <entrypoint> gateway --port 18789`.

## Solution Assessment

No, not as currently written. The code direction is the narrow maintainable fix, but the non-Linux fallback should use wide `ps` output before this can reliably solve the LaunchAgent case.
