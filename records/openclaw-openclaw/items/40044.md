## Summary

Keep this PR open: current main still has no `session=` Runtime metadata, and the PR is still incomplete because it misses embedded prompts, can use the wrapper session for target-session commands, and lacks contributor-supplied real behavior proof.

## What This Changes

The branch adds optional `sessionId` Runtime prompt metadata, renders it as `session=<uuid>`, wires one command prompt path, and adds a runtime-line test assertion plus changelog entry.

## Best Possible Solution

Thread the opaque session UUID from the canonical current or target session through every real prompt runtime path, with focused coverage and redacted real behavior proof showing the resulting Runtime line.

## Reproduction Assessment

Not applicable as a bug reproduction: this is additive Runtime prompt metadata. Source inspection gives a high-confidence behavior check that current main lacks `session=` and that the PR still misses two real prompt paths.

## Solution Assessment

No, not yet. The `sessionId` direction is safer than exposing raw `sessionKey`, but the patch should also wire embedded prompts and use `targetSessionEntry?.sessionId` for command Runtime metadata before merge.
