## Summary

Keep open. Current main still lacks prompt-visible `session=` Runtime metadata, but this branch remains incomplete because it only renders the value when supplied, misses the normal embedded prompt path, and can use the wrapper session UUID for targeted command prompts.

## What This Changes

The PR adds optional `sessionId` Runtime prompt metadata, renders it as `session=<uuid>`, wires one command-prompt runtime value, and adds test/changelog coverage.

## Best Possible Solution

Thread the opaque session UUID through every real prompt runtime path from the canonical current or target session, with focused coverage for embedded prompts and target-session command prompts while keeping raw session keys out of provider-visible prompts.

## Reproduction Assessment

Not applicable as a bug reproduction: this is additive prompt metadata. Source inspection gives a high-confidence behavior check that current main lacks `session=` Runtime metadata and that the PR branch still misses or misattributes important prompt paths.

## Solution Assessment

No. `sessionId` is the safer direction versus raw `sessionKey`, but this patch should also thread the field through embedded prompts and use `targetSessionEntry?.sessionId` for command Runtime metadata.
