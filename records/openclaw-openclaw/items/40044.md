## Summary

Keep open: current main still lacks session UUID Runtime prompt metadata, and this PR is useful but incomplete because it misses embedded prompts, can use the wrapper session in command prompts, and lacks external real behavior proof.

## What This Changes

The PR adds optional `sessionId` Runtime prompt metadata, renders it as `session=<uuid>`, wires one command prompt value, adds a runtime-line assertion, and adds a changelog entry.

## Best Possible Solution

Thread the opaque session UUID from the canonical current or target session through every real prompt runtime path, with focused tests and contributor-supplied real behavior proof.

## Reproduction Assessment

Not applicable as a bug reproduction: this is additive Runtime prompt metadata. Source inspection gives a high-confidence behavior check that current main lacks `session=` metadata and that the PR still misses two real prompt paths.

## Solution Assessment

No. The `sessionId` direction is safer than exposing raw `sessionKey`, but the patch should also wire embedded prompts and use `targetSessionEntry?.sessionId` for command Runtime metadata before merge.

  - confidence: 0.88
