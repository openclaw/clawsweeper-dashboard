## Summary

Keep open: current main still lacks `session=` Runtime metadata, and this PR remains the active implementation path, but the patch should not merge until it covers the embedded prompt path, uses the target session for command prompts, and includes contributor-supplied real behavior proof.

## What This Changes

The branch adds optional `sessionId` Runtime prompt metadata, renders it as `session=<uuid>`, wires one command prompt runtime value, adds a runtime-line assertion, and records a changelog entry.

## Best Possible Solution

Thread the opaque session UUID from the canonical current or target session through every real prompt runtime path, with focused tests for embedded prompts and target-session command prompts while keeping raw session keys out of provider-visible prompts.

## Reproduction Assessment

Not applicable as a bug reproduction: this is additive Runtime prompt metadata. Source inspection gives a high-confidence behavior check that current main lacks `session=` metadata and that this PR still misses two real prompt paths.

## Solution Assessment

No. The `sessionId` direction is safer than exposing a raw `sessionKey`, but the patch should also wire embedded prompts and use `targetSessionEntry?.sessionId` for command Runtime metadata before merge.
