## Summary

Keep open: current main still lacks the requested Runtime session metadata, and this PR remains useful, but the branch is not merge-ready because it misses embedded-runner prompts, uses the wrapper session for targeted command prompts, and lacks after-fix real behavior proof.

## What This Changes

This PR adds optional `sessionId` Runtime prompt metadata, renders it as `session=<uuid>`, wires it into command prompt runtime metadata, updates a runtime-line assertion, and adds a changelog entry.

## Best Possible Solution

Thread the existing opaque session UUID through every real prompt runtime path from the canonical current or target session, while keeping raw session keys out of provider-visible prompts.

## Reproduction Assessment

Not applicable as a bug reproduction: this is an additive prompt metadata feature. Source inspection gives a high-confidence behavior check that current main lacks `session=` Runtime metadata and the PR branch still misses important prompt paths.

## Solution Assessment

No. The `sessionId` direction is the safer path versus raw `sessionKey`, but the current diff is incomplete because it does not thread the field through embedded prompts and uses the wrapper session entry for targeted command Runtime metadata.
