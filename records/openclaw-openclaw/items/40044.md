## Summary

Keep this PR open: the requested feature is not on current main, and the PR is useful but still incomplete because it does not wire the normal embedded run prompt and can use the wrong session entry in command-generated prompts.

## What This Changes

The PR adds optional `sessionId` Runtime metadata, renders it as `session=<uuid>`, passes it from the command system-prompt bundle, adds a runtime-line assertion, and adds a changelog entry.

## Best Possible Solution

Repair the PR so the current session UUID is threaded through every Runtime prompt builder that has the active session id, while preserving target-session semantics and keeping the change additive.

## Reproduction Assessment

Not applicable. This is an additive feature PR, and source inspection confirms current main does not render session metadata in the Runtime line.

## Solution Assessment

No. The direction is narrow, but the current patch is not the best implementation until it wires the normal run prompt and uses `targetSessionEntry?.sessionId` for command prompts.

  - body: The PR teaches `buildRuntimeLine` to render `session=...`, but the normal embedded run path still builds `runtimeInfo` without `params.sessionId`.   - confidence: 0.9

  - confidence: 0.92
