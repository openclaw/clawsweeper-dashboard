## Summary

Keep open: current main still lacks the requested `session=<uuid>` Runtime metadata, but this branch is incomplete and would expose the wrong session UUID in the commands path for target-session commands.

## What This Changes

This PR adds optional `sessionId` Runtime prompt metadata, renders it as `session=<uuid>`, wires it into the commands system prompt, updates a runtime-line test, and adds a changelog entry.

## Best Possible Solution

Thread the existing session UUID through every real prompt runtime path from the canonical current or target session, especially the embedded runner and commands target-session path, while keeping raw session keys out of provider-visible prompts.

## Reproduction Assessment

Not applicable as a bug reproduction: this is an additive feature PR. Source inspection gives a high-confidence behavioral check that current main lacks `session=` Runtime metadata and the PR branch still misses important prompt paths.

## Solution Assessment

No. The safer `sessionId` direction is appropriate, but the current diff is not complete because it misses the embedded-run prompt path and uses the wrapper session entry for command Runtime metadata.

  - confidence: 0.91

Commands targeting another session can therefore advertise the wrapper session UUID; use `targetSessionEntry?.sessionId` and assert the target-session case.
