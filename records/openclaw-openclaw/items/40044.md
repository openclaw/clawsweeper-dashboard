## Summary

This PR is useful and not implemented on current main, but the live branch still misses real prompt paths and lacks contributor real-behavior proof. The opaque `sessionId` direction is safer than the closed raw `sessionKey` alternative, but the implementation needs two narrow fixes before it is merge-ready.

## What This Changes

The PR adds optional `sessionId` Runtime prompt metadata, renders it as `session=<uuid>`, wires one command prompt path, adds a focused runtime-line assertion, and adds a changelog entry.

## Best Possible Solution

Thread the opaque session UUID from the canonical current or target session through every real prompt Runtime path, with focused coverage and redacted real-behavior proof showing the resulting Runtime line.

## Reproduction Assessment

Not applicable as a bug reproduction: this is additive Runtime prompt metadata, not a broken existing contract. Source inspection gives a high-confidence behavior check that current main lacks `session=` and that the PR branch still misses real prompt paths.

## Solution Assessment

No, not yet. The `sessionId` direction is safer than exposing raw `sessionKey`, but the patch should also wire embedded prompts and use `targetSessionEntry?.sessionId` for command Runtime metadata before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Wire sessionId through embedded prompts:** `src/agents/system-prompt.ts:1035`
  - body: Adding `session=...` in the renderer only helps paths that pass `runtimeInfo.sessionId`. The embedded runner's real prompt path builds runtime metadata without the available `params.sessionId`, so normal embedded sessions would still omit the advertised session UUID; thread it through that runtime object and cover it.
  - confidence: 0.91
- **[P2] Use the target session for Runtime metadata:** `src/auto-reply/reply/commands-system-prompt.ts:125`
  - body: This wires command Runtime metadata to `params.sessionEntry?.sessionId`, but this path already resolves `targetSessionEntry` for bootstrap and tool metadata. Commands aimed at another session can therefore show the wrapper session UUID; use `targetSessionEntry?.sessionId` and cover the target-session case.
  - confidence: 0.88
