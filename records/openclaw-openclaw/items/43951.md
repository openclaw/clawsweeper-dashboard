## Summary

Keep open: the reported bug is source-reproducible and the PR points in the right direction, but the branch is stale against current main, has blocking correctness/security issues, and only provides mocked/CI proof.

## What This Changes

The PR adds `accountId` parsing and propagation for `/hooks/agent`, routes hook-created jobs through cron normalization, changes cron message-tool policy, and adds tests plus a changelog entry.

## Best Possible Solution

Land a rebased, narrow gateway fix that parses and trims `accountId` in the active hook request handler, carries it through replay scope and cron delivery, preserves sanitized hook metadata and current message-tool policy, documents account selection, and includes focused tests plus redacted real behavior proof.

## Reproduction Assessment

Yes. Source inspection on current main is enough: `/hooks/agent` normalization omits `accountId`, and hook-created cron delivery contains only `mode`, `channel`, and `to`.

## Solution Assessment

No. Forwarding `accountId` is the right direction, but this branch should be rebased onto the current hook request handler, keep sanitized hook metadata, and preserve the current cron message-tool policy.
