## Summary

Keep this PR open: the underlying hook `accountId` delivery bug is still source-reproducible on current main, but this branch needs a rebase, real behavior proof, and fixes for the hook-name sanitization and cron message-tool policy regressions before merge.

## What This Changes

The branch adds `accountId` parsing and propagation for `/hooks/agent`, routes hook dispatch through cron normalization, changes shared cron message-tool policy, and adds tests plus a changelog entry.

## Best Possible Solution

Land a narrow rebased fix or replacement that threads trimmed `accountId` through hook normalization, replay scope, and `CronDelivery`, while preserving sanitized hook names, current cron message-tool behavior, tests, and public hook docs.

## Reproduction Assessment

Yes. Source inspection on current main shows `/hooks/agent` never reads `payload.accountId`, hook-created delivery only carries `mode`, `channel`, and `to`, and cron can only route by `accountId` when that field reaches `CronDelivery`.

## Solution Assessment

No. The direction is right, but this branch should be rebased and revised so it preserves sanitized hook names, current announce message-tool behavior, operator docs, and real behavior proof.
