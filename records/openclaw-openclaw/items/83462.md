## Summary

Keep this PR open, but it is not merge-ready: the latest commit covers the self-hosted path, yet the new guards still misclassify env-var credential references and disable existing no-auth local discovery behavior.

## What This Changes

This PR adds non-secret auth marker guards to shared provider catalog helpers and OpenAI-compatible self-hosted discovery so marker-backed auth skips `/models` catalog discovery.

## Best Possible Solution

Use a discovery-key-aware guard that skips true non-runnable local markers without rejecting env-var/SecretRef references or genuinely unauthenticated local discovery, and add regression coverage for all three cases.

## Reproduction Assessment

Yes for the PR regressions by source inspection: env-backed vLLM/SGLang discovery returns an env-var marker plus real `discoveryApiKey`, and the new guard would return `null`; current tests also demonstrate no-key local discovery that the new early return would skip. The linked 401 behavior has reporter logs, but no after-fix real proof yet.

## Solution Assessment

No. The direction is right, but the implementation should key off runnable discovery credentials or use a narrowed marker check instead of rejecting all known env-var markers and all missing API-key local discovery.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Preserve env-backed catalog auth markers:** `src/plugins/provider-catalog.ts:38`
  - body: `isNonSecretApiKeyMarker()` treats known env-var names as markers by default, while provider discovery can legitimately return `apiKey: "VLLM_API_KEY"` with `discoveryApiKey: "env-vllm-key"`. With this guard, vLLM/SGLang and other env-backed catalogs return `null` instead of using the resolved discovery key; narrow the check or key the decision off `discoveryApiKey`.
  - confidence: 0.94
- **[P1] Keep no-key local discovery working:** `src/plugins/provider-self-hosted-setup.ts:60`
  - body: The new `!actualKey` early return stops `discoverOpenAICompatibleLocalModels()` from fetching `/models` for local servers that do not require auth, which current tests cover by calling this helper without `apiKey` and expecting `headers: undefined`. Skip marker-derived calls upstream without disabling genuinely unauthenticated local discovery.
  - confidence: 0.9
