## Summary

Keep this PR open: it targets a real source-reproducible auth-provider catalog bug, but the patch as written would suppress valid env-var-backed catalogs, disables existing authless local discovery behavior, and still lacks after-fix real behavior proof.

## What This Changes

The PR imports `isNonSecretApiKeyMarker` into provider catalog and self-hosted setup helpers and skips catalog/model discovery when the resolved API key is missing or classified as a non-secret marker.

## Best Possible Solution

Preserve env-var and SecretRef catalog entries, skip only true local/non-runnable marker placeholders with discovery-key-aware logic, keep the public no-key local discovery helper working, add focused regression tests, and post redacted after-fix live output.

## Reproduction Assessment

Yes for the patch regressions by source inspection and existing contract tests: env-var marker names are expected to remain valid catalog `apiKey` values while real secrets ride in `discoveryApiKey`. The linked 401 symptom is also source-reproducible, but no live provider was run in this read-only review.

## Solution Assessment

No. The patch should narrow marker checks to true placeholders, preserve env-var references and no-key local discovery, and add regression coverage plus after-fix live proof.

- **[P1] Preserve env-var apiKey markers:** `src/plugins/provider-catalog.ts:35`

  - confidence: 0.9
