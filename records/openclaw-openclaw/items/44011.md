## Summary

Keep open: current main still lacks `ctx.agentId` and does not thread available session identity through outbound typed message hooks, while this PR needs a rebase, identity-source fix, docs, and real behavior proof before merge.

## What This Changes

The branch adds optional `sessionKey` and `agentId` to typed plugin message-hook contexts, threads them through inbound/outbound dispatch, and adds focused mapper and delivery tests.

## Best Possible Solution

Land one rebased additive hook-context contract that uses a single identity source, documents `ctx.agentId`, coordinates with related outbound work, and includes focused tests plus live plugin/runtime proof.

## Reproduction Assessment

Yes, at source level: current main lacks `agentId` in `PluginHookMessageContext`, and inbound/outbound message hook dispatch does not pass `agentId` into typed `ctx`. I did not run a live plugin reproduction in this read-only review.

## Solution Assessment

No, not as written. The additive API direction is plausible, but the branch should keep session identity fields from one source, update public docs, rebase onto current hook contract files, and provide real behavior proof before merge.
