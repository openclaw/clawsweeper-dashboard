## What This Changes

The branch adds optional `sessionKey` and `agentId` fields to typed plugin message hook context and threads them through inbound/outbound mappers, delivery, dispatch, and tests.

## Best Possible Solution

Rebase into one current-main hook-context change, or let https://github.com/openclaw/openclaw/pull/73706 land for `sessionKey` first, then add and document coherent `agentId` semantics with tests and real runtime proof.

## Reproduction Assessment

Yes, at source level: current main prepares outbound session identity for internal hooks but does not pass it into plugin `message_sending` or `message_sent`, and `agentId` is absent from the live typed message-hook context. I did not run a live hook scenario in this read-only review.

## Solution Assessment

No, not as written. The additive API direction is plausible, but the patch must target the current hook contract, keep identity fields from one source, coordinate with the newer outbound `sessionKey` work, and include real runtime proof.
