## Summary

Keep open: current main has some message-hook correlation fields, but typed message hooks still do not expose `ctx.agentId`, and this PR needs real behavior proof plus identity/docs fixes before merge.

## What This Changes

The PR adds optional `sessionKey` and `agentId` to typed plugin message hook contexts and threads them through inbound and outbound message hook delivery with targeted tests.

## Best Possible Solution

Land a rebased additive hook-contract update that exposes optional `ctx.agentId` alongside existing message-hook correlation fields, derives session identity from one selected source, and documents the public field.

## Reproduction Assessment

Yes, at source level: current main exposes `sessionKey` but not `agentId` in typed message hook context, inbound dispatch resolves an agent id without forwarding it, and outbound typed hooks omit session identity. This is an API capability gap rather than a broken documented behavior repro.

## Solution Assessment

No. The additive direction is maintainable, but this branch should be rebased to current main, update `src/plugins/hook-message.types.ts` and `docs/plugins/hooks.md`, and keep `sessionKey`/`agentId` from one identity source before merge.
