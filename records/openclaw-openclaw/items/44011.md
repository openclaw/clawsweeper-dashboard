## Summary

Keep open. Current main still does not expose `ctx.agentId` on typed message hooks and still omits session identity from outbound typed hook contexts, but this PR is not merge-ready because the branch targets the old hook type surface, can mix mirror and sender identities, and lacks public docs/changelog follow-through.

## What This Changes

The PR adds optional `sessionKey` and `agentId` to typed plugin message hook context plumbing for inbound and outbound message hooks, with mapper, dispatch, delivery, and regression test updates.

## Best Possible Solution

Land a rebased additive hook contract update in the current split type files that exposes optional `ctx.agentId` alongside `ctx.sessionKey`, derives both from the same selected session identity, and documents/releases the new plugin-facing field.

## Reproduction Assessment

Yes. Source inspection is enough: current main's typed message context lacks `agentId`, inbound mapping does not derive it, and outbound typed hook calls do not pass session identity even when delivery has a session or mirror.

## Solution Assessment

No. The additive API direction is maintainable, but this branch is not the best mergeable solution until it rebases to current main, keeps mirror/session identity internally consistent, and updates docs plus changelog.
