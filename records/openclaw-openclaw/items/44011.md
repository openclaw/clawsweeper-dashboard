## Best Possible Solution

Rebase onto the current hook-message.types.ts, message-hook-mappers.ts, and deliver.ts shape, preserve all existing correlation fields, and choose a single canonical source for sessionKey and any agent identity semantics in coordination with the newer outbound sessionKey work.

## Solution Assessment

No, not as written: the branch is conflicting, targets the old hook type file, and leaves agentId semantics ambiguous.
