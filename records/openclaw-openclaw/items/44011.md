## Summary

Keep open: the session-context gap is still present on current main, but this PR is not merge-ready because it can mix identity sources, lacks public hook documentation for the new field, and has only test-based proof.

## What This Changes

The PR adds optional sessionKey and agentId fields to typed message hook context, threads them through inbound and outbound hook dispatch, and adds focused mapper and delivery tests.

## Best Possible Solution

Land a rebased additive hook contract that derives sessionKey and agentId from one selected identity source, updates the live hook-message type/docs, and coordinates the outbound behavior with https://github.com/openclaw/openclaw/pull/63007.

## Reproduction Assessment

Yes, at source level: current main lacks ctx.agentId in the live message hook context and still omits available outbound session identity from message_sending/message_sent typed hook contexts. I did not establish a live plugin reproduction in this read-only review.

## Solution Assessment

No, not as written. The additive API direction is plausible, but the PR needs one identity source, docs for the new public field, a current-main rebase onto the live hook-message type surface, and real behavior proof.
