## Summary

Keep open: current main still lacks `ctx.agentId` on typed message hook contexts, but this branch is stale against the current hook type split and still has a merge-blocking mirror/session identity bug.

## What This Changes

The PR adds optional `sessionKey` and `agentId` to typed plugin message hook contexts and threads them through inbound and outbound message hook delivery with targeted tests.

## Best Possible Solution

Land a rebased additive hook-contract update that exposes optional `ctx.agentId` alongside `ctx.sessionKey` for typed message hooks, derives both from one selected session identity, and documents the public field.

## Reproduction Assessment

Yes. A source-level current-main check is enough: the typed message hook context exposes `sessionKey` but not `agentId`, inbound dispatch resolves the agent id without passing it to the hook context, and outbound typed hooks omit session identity.

## Solution Assessment

No. The additive API direction is maintainable, but this branch should not merge until it rebases to current main, keeps mirror/session identity internally consistent, updates the message hook docs, and includes real behavior proof.
