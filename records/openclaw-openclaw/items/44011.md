## Summary

Keep open: the PR still targets a useful additive hook-context direction, but current main only partially covers session identity, and this branch is conflicting, lacks real behavior proof, and has a concrete identity-mixing bug.

## What This Changes

This PR adds optional `sessionKey` and `agentId` to typed plugin message hook context and threads them through inbound and outbound message hook dispatch with focused tests.

## Best Possible Solution

Land a rebased additive hook contract that derives `sessionKey` and `agentId` from one selected identity source, updates the public hook docs, and coordinates outbound behavior with https://github.com/openclaw/openclaw/pull/63007.

## Reproduction Assessment

Yes, at source level: current main still lacks `ctx.agentId` in typed message hook context and does not pass available outbound session identity into `message_sending` or `message_sent`. I did not establish a live plugin reproduction in this read-only review.

## Solution Assessment

No, not as written. The additive API direction is plausible, but the branch mixes identity sources, misses public docs for the new field, and needs rebase plus real behavior proof before merge.
