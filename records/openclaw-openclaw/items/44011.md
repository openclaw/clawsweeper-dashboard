## Summary

Keep open: the API direction is plausible, but this PR still needs real behavior proof, a rebase onto the current hook-message type surface, and fixes for mixed outbound identity plus public hook docs before it can merge.

## What This Changes

The branch adds optional `sessionKey` and `agentId` to typed message hook context, threads them through inbound/outbound hook dispatch, and adds focused mapper and delivery tests.

## Best Possible Solution

Land a rebased additive hook contract that derives `sessionKey` and `agentId` from one selected identity source, updates current hook-message types and public docs, and coordinates outbound behavior with https://github.com/openclaw/openclaw/pull/63007.

## Reproduction Assessment

Yes, at source level: current main lacks `ctx.agentId` in typed message hook context and still omits available outbound session identity from `message_sending` / `message_sent` contexts. I did not establish a live plugin reproduction in this read-only review.

## Solution Assessment

No, not as written. The additive API direction is plausible, but the patch mixes identity sources, misses public docs for the new field, and needs a current-main rebase plus real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Keep hook session identity from one source:** `src/infra/outbound/deliver.ts:410-411`
  - body: When `mirror.sessionKey` is present but `mirror.agentId` is absent, these independent fallbacks expose the mirror session key with the sender session's agent id. Pick one identity source for both fields, or derive/omit `agentId` from the selected session key, so plugins do not see a mixed identity.
  - confidence: 0.87
- **[P3] Document ctx.agentId for message hooks:** `src/plugins/types.ts:565`
  - body: This makes `ctx.agentId` part of the public typed message-hook context, but `docs/plugins/hooks.md` still lists message hook context fields without it or its population rules. Document the contract so plugin authors do not have to infer it from source.
  - confidence: 0.74
