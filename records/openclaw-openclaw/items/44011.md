## Summary

Keep open. Current main still does not expose `ctx.agentId` on typed message hooks, and outbound typed hook calls still omit session identity; the PR is useful but needs rebase, identity-semantics, and docs fixes before merge.

## What This Changes

The PR adds optional `sessionKey` and `agentId` to typed plugin message hook context plumbing for inbound `message_received` and outbound `message_sending`/`message_sent`, with mapper, dispatch, delivery, and regression test updates.

## Best Possible Solution

Land a rebased additive hook contract update in the current split type files that exposes optional `ctx.agentId` alongside `ctx.sessionKey` for all typed message hooks, derives both fields from the same selected session identity, and documents the new plugin-facing field.

## Reproduction Assessment

Yes. A source-level reproduction is enough: current main lacks `agentId` in `PluginHookMessageContext`, the mapper cannot carry it, and outbound typed hook contexts do not receive session identity even when delivery has session or mirror metadata.

## Solution Assessment

No. The additive API direction is maintainable, but this branch is not the best mergeable solution until it rebases to current main, keeps mirror/session identity internally consistent, and updates public plugin hook documentation.
