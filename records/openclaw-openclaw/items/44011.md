## Summary

Keep open: the current main branch still lacks `ctx.agentId` for typed message hooks, and this PR is a plausible additive API fix, but the branch needs identity-source cleanup, docs, rebase work, and real runtime proof before merge.

## What This Changes

Adds optional `sessionKey` and `agentId` to typed plugin message-hook contexts and threads them through inbound and outbound hook mapping with targeted tests.

## Best Possible Solution

Land a rebased additive hook-contract change that exposes available session and agent identity from one consistent source, updates docs, and includes focused regression plus real plugin/runtime proof.

## Reproduction Assessment

Yes, at source level: current main has `sessionKey` but no `agentId` in `PluginHookMessageContext`, and inbound/outbound message hook dispatch does not pass `agentId` into `ctx`. I did not run a live plugin reproduction in this read-only review.

## Solution Assessment

No, not as written. The additive API direction is plausible, but the branch must use one session/agent identity source, update the public docs, rebase onto the current hook contract files, and provide real behavior proof before merge.
