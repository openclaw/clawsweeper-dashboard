## Summary

Keep open: current main still lacks `ctx.agentId` for typed message hooks and does not pass outbound session identity into `message_sending`/`message_sent`, but this PR has a blocking identity-source bug and still needs real behavior proof before merge.

## What This Changes

The PR adds optional `sessionKey` and `agentId` fields to typed plugin message hook contexts, threads them through inbound and outbound hook dispatch, and adds focused tests.

## Best Possible Solution

Land one rebased additive hook-context contract that exposes stable `sessionKey`/`agentId` semantics across inbound and outbound message hooks, documents those semantics, and coordinates outbound gateway behavior with https://github.com/openclaw/openclaw/pull/63007.

## Reproduction Assessment

Yes, at source level: current main lacks `ctx.agentId` in `PluginHookMessageContext`/`toPluginMessageContext`, and outbound typed hooks do not receive session identity. I did not establish a live plugin reproduction in this read-only review.

## Solution Assessment

No, not as written. The additive API direction is plausible, but the PR should use one identity source, update the public hook docs/current split type surface, coordinate with the newer outbound PR, and include real behavior proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

  - body: When `mirror.sessionKey` is present but `mirror.agentId` is absent, these independent fallbacks expose the mirror session key with the sender session's agent id. Pick one identity source for both fields, or derive/omit `agentId` from the selected session key, so plugins do not see a mixed identity.
  - confidence: 0.87

  - confidence: 0.74
