## Summary

Keep open. Current main still lacks `ctx.agentId` on typed message hooks, so this PR is not obsolete, but the branch has a blocking mirror/session identity mismatch plus missing public docs and changelog; a focused repair should rebase it onto the current split hook files.

## What This Changes

The PR adds optional `sessionKey` and `agentId` to typed plugin message hook context plumbing for inbound `message_received` and outbound `message_sending`/`message_sent`, with mapper and delivery tests.

## Best Possible Solution

Land a rebased additive hook contract update in the current split type files that exposes optional `ctx.agentId` alongside `ctx.sessionKey` for typed message hooks, derives both from the same selected session identity, and documents/releases the new field.

## Reproduction Assessment

Yes. Source inspection on current main shows typed message context and mappers lack `agentId`, inbound dispatch has a resolved session agent but does not pass it to the typed message hook context, and outbound typed hook calls omit session identity.

## Solution Assessment

No. The additive API direction is maintainable, but this branch is not the best mergeable solution until it rebases to current main, keeps mirror/session identity internally consistent, and updates docs plus changelog.
