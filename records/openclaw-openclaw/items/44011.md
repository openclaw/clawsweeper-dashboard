## Summary

Keep open but do not merge as-is: source inspection confirms a real hook-context gap on current main, but the branch still has a concrete mixed-identity bug, lacks after-fix real runtime proof, and overlaps active outbound hook identity work that should be coordinated.

## What This Changes

The PR adds optional `sessionKey` and `agentId` to typed plugin message-hook context and threads them through inbound and outbound hook dispatch with focused tests.

## Best Possible Solution

Land one rebased additive hook-context contract that selects a single coherent session identity source, derives or omits `agentId` consistently, documents the public semantics, and coordinates with the overlapping outbound identity work.

## Reproduction Assessment

Yes, at source level: current main still lacks `ctx.agentId` on typed message hook context and shared outbound `message_sending`/`message_sent` paths do not pass available session identity into plugin hook context. I did not establish a live plugin reproduction in this read-only review.

## Solution Assessment

No, not as written. The additive API direction is plausible, but the branch can mix mirror and sender identity, lacks public `agentId` semantics, needs a rebase onto current hook type files, and still needs real behavior proof.

Overall correctness: patch is incorrect
