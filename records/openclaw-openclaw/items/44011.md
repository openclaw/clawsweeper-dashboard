## Best Possible Solution

## Reproduction Assessment

Yes, at source level: current main prepares outbound session identity for internal hooks but does not pass it to plugin `message_sending` or `message_sent`, and `agentId` is not part of the live typed message-hook context. I did not run a live hook scenario in this read-only review.

## Solution Assessment

No, not as written. The additive API direction is plausible, but the patch must target the current hook contract, keep identity fields from one source, coordinate with the newer outbound `sessionKey` work, and include real runtime proof.
