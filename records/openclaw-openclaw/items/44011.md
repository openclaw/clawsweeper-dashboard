## Summary

Keep open: current main still lacks `ctx.agentId` and outbound session identity in typed message hooks, so a rebased additive hook-contract change could be useful, but this external PR is blocked by missing real behavior proof and concrete patch issues.

## What This Changes

This PR adds optional `sessionKey` and `agentId` to typed plugin message hook context, threads them through inbound and outbound message hook dispatch, and adds focused mapper and delivery tests.

## Best Possible Solution

Land a rebased additive hook contract that derives `sessionKey` and `agentId` from one selected identity source, documents the public message-hook semantics, and coordinates the outbound path with https://github.com/openclaw/openclaw/pull/63007.

## Reproduction Assessment

Yes, at source level: current main still lacks `ctx.agentId` in typed message hook context and does not pass available outbound session identity into `message_sending` or `message_sent`. I did not establish a live plugin reproduction in this read-only review.

## Solution Assessment

No, not as written. The additive API direction is plausible, but this branch mixes identity sources, misses public docs for the new field, conflicts with current main, and still needs real behavior proof.
