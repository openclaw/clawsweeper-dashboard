## Summary

Keep open: current main still lacks `ctx.agentId` and outbound session identity in typed message hooks, and this PR remains useful but needs rebase, identity-source cleanup, docs, and real behavior proof before merge.

## What This Changes

The PR adds optional `sessionKey` and `agentId` to typed plugin message-hook context, threads them through inbound/outbound hook mapping and delivery, and adds focused tests.

## Best Possible Solution

Land a rebased additive hook-contract change that exposes available session/agent identity consistently from one source across typed message hooks, updates docs, and includes regression plus real plugin/runtime proof.

## Reproduction Assessment

Yes, at source level: current main exposes some session correlation but not `ctx.agentId`, and shared outbound typed hooks still receive only routing context. This is an API capability gap rather than a broken documented behavior repro.

## Solution Assessment

No. The additive direction is maintainable, but this branch must be rebased, keep `sessionKey` and `agentId` from one identity source, and document the public field before merge.
