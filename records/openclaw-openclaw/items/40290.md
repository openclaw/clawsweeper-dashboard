## Summary

Keep open: current main still contradicts the sandbox docs by blocking credential-like `sandbox.docker.env` keys, but this PR is not merge-ready because its skill-filter bypass can re-enable skills that config explicitly disables.

## What This Changes

The PR adds `forceAllowKeys` support for Docker sandbox env sanitization and lets explicitly listed agent skills bypass normal eligibility checks.

## Best Possible Solution

Rebase and narrow the implementation so Docker sandbox env keys can be intentionally passed, explicit skill disables and allowlists remain authoritative, regression tests cover both paths, and the active changelog records the user-facing fix.

## Reproduction Assessment

Yes from source inspection. Current main sends `sandbox.docker.env` through the credential blocklist even though docs direct users to that config for sandbox skill API keys.

## Solution Assessment

No. The env bypass direction addresses the documented problem, but the current skill-filter change is too broad because it bypasses explicit disables; the safer fix preserves disable/allowlist checks and only relaxes runtime eligibility where intended.
