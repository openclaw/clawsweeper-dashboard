## Summary

Keep open: current main still blocks credential-shaped keys in configured Docker sandbox env, so the central bug is real, but this PR is not merge-ready because it drops caller sanitizer options, bypasses skill eligibility gates, and lacks contributor real-behavior proof.

## What This Changes

The PR adds `forceAllowKeys` to sandbox env sanitization, uses it for Docker config env, adds a sandbox env validator export, and changes explicit skill filters to bypass eligibility checks.

## Best Possible Solution

Land a narrow sandbox fix that force-allows only explicit Docker config env while preserving sanitizer options and skill eligibility gates, with regression tests and redacted real Docker proof.

## Reproduction Assessment

Yes, at source level. Current main sends configured Docker env through a credential blocklist while docs direct sandboxed skill API keys to agents.defaults.sandbox.docker.env; no live Docker run was performed in this read-only review.

## Solution Assessment

No. The Docker env exception is plausible, but this PR should preserve envSanitizationOptions, keep shouldIncludeSkill as the prerequisite gate, add focused tests, and provide real sandbox proof before merge.
