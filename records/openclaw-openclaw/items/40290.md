## Summary

Keep open: current main still has the documented Docker sandbox env/API-key blocklist conflict, but this PR is not mergeable as-is because it lacks real behavior proof and still weakens sanitizer and skill eligibility boundaries.

## What This Changes

The branch adds a sanitizer force-allow set for configured Docker env keys, adds a sandbox env validation helper, and lets explicit skill filters bypass eligibility checks.

## Best Possible Solution

Land a narrow fix that force-allows only explicit Docker config env at container creation while preserving sanitizer options and skill eligibility, with regression coverage and redacted real Docker proof.

## Reproduction Assessment

Yes, at source level. Current main sends configured Docker env through the credential blocklist while docs direct sandboxed skill API keys to agents.defaults.sandbox.docker.env; no live Docker run was performed in this read-only review.

## Solution Assessment

No. The Docker env exception is plausible, but this PR should preserve envSanitizationOptions, keep shouldIncludeSkill as the prerequisite gate, add focused tests, and provide real sandbox proof before merge.
