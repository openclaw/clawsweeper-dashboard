## Summary

Keep open: the underlying sandbox env bug is still source-reproducible, but the branch has blocking correctness/security issues and lacks contributor real behavior proof.

## What This Changes

The PR adds a force-allow path for Docker sandbox config env, adds a sandbox env validation helper, and changes explicit skill filters to bypass eligibility checks.

## Best Possible Solution

Land a narrow sandbox env fix that force-allows explicit Docker config env while preserving sanitizer options and skill eligibility gates, with focused regression tests and redacted real Docker proof.

## Reproduction Assessment

Yes, at source level. Current main sends configured Docker env through a credential blocklist while docs direct sandboxed skill API keys to agents.defaults.sandbox.docker.env; no live Docker run was performed in this read-only review.

## Solution Assessment

No. The Docker env exception is plausible, but this PR should preserve envSanitizationOptions, keep shouldIncludeSkill as the prerequisite gate, add focused tests, and provide real sandbox proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve skill eligibility before explicit filters:** `src/agents/skills/workspace.ts:79-82`
  - body: The explicit-name shortcut runs before shouldIncludeSkill, which is the gate for skills.entries.<key>.enabled === false, bundled allowlists, and runtime requires checks. A disabled or runtime-ineligible skill listed in an agent filter would become visible again; keep eligibility as the prerequisite and apply the explicit-name filter afterward.
  - confidence: 0.92
- **[P2] Preserve sanitizer options when force-allowing Docker env:** `src/agents/sandbox/docker.ts:370-372`
  - body: This call replaces params.envSanitizationOptions with a new object and force-allows every configured env key, so strict/custom sanitizer policy provided by callers can no longer constrain Docker env handling. Merge the existing options through and scope the bypass to the documented Docker env credential exception.
  - confidence: 0.88
