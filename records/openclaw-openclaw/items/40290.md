## Summary

Keep open: the underlying `docker.env` sanitizer problem is still source-reproducible on current `main`, but this PR should not merge as-is because it bypasses skill eligibility gates, drops caller sanitizer options, and lacks contributor real behavior proof.

## What This Changes

The branch adds `forceAllowKeys` sanitizer support for Docker config env, adds a new sandbox env validation helper, and lets explicitly named skills bypass eligibility filtering.

## Best Possible Solution

Revise this into a narrow `docker.env` credential exception that preserves caller sanitizer options, leaves `shouldIncludeSkill` as the prerequisite gate, adds focused regression tests, and includes redacted Docker sandbox proof.

## Reproduction Assessment

Yes, at source level. Current main sends configured Docker env through the credential blocklist while docs direct sandboxed skill API keys to `agents.defaults.sandbox.docker.env`; I did not run live Docker proof in this read-only review.

## Solution Assessment

No. The `docker.env` force-allow direction is plausible, but this PR should preserve `envSanitizationOptions`, keep skill eligibility gates intact, and provide real sandbox proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Full review comments:

- **[P1] Preserve skill eligibility before explicit filters:** `src/agents/skills/workspace.ts:79-82`
  - body: The explicit-name shortcut runs before `shouldIncludeSkill`, which is the gate for `skills.entries.<key>.enabled === false`, bundled allowlists, and runtime `requires` checks. A disabled or runtime-ineligible skill listed in an agent filter would become visible again; keep eligibility as the prerequisite and apply the explicit-name filter afterward.
  - confidence: 0.92

  - body: This call replaces `params.envSanitizationOptions` with a new object and force-allows every configured env key, so strict/custom sanitizer policy provided by callers can no longer constrain Docker env handling. Merge the existing options through and scope the bypass to the documented Docker env credential exception.
  - confidence: 0.88
