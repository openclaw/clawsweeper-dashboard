## Summary

Keep open. The Docker `docker.env` allow direction addresses a real source-level bug, but this PR still drops current-main sanitizer options, broadens skill visibility past existing eligibility gates, and lacks after-fix real behavior proof.

## What This Changes

The PR adds `forceAllowKeys` to sandbox env sanitization for admin-configured Docker env vars, adds an env validation helper, and lets explicit skill filters bypass skill eligibility.

## Best Possible Solution

Revise this into a narrow Docker sandbox env fix that force-allows explicitly configured `docker.env` keys while preserving sanitizer options and existing skill eligibility gates, then add focused regression coverage and redacted real behavior proof.

## Reproduction Assessment

Yes, at source level. Current docs direct users to `agents.defaults.sandbox.docker.env` for sandboxed skill API keys, while current main passes that map through a sanitizer that blocks API-key-shaped names.

## Solution Assessment

No. The `docker.env` force-allow direction is appropriate, but this patch should merge existing sanitizer options and keep `shouldIncludeSkill` as the prerequisite for skill visibility instead of broadening skill policy.
