## Summary

Keep this PR open. The latest head fixes the earlier `/root` blocklist regression, but it still bypasses skill eligibility, drops existing sandbox env sanitizer options, and lacks the required changelog entry.

## What This Changes

The PR adds a `forceAllowKeys` sandbox env sanitizer option for `docker.env`, wires it into Docker create args, adds an env validation helper, and changes explicit agent skill filters to bypass normal eligibility.

## Best Possible Solution

Revise this into a narrow sandbox env fix that preserves `shouldIncludeSkill` as the prerequisite for all skill visibility, merges forced Docker env keys with existing sanitizer options, adds focused tests, and records the user-facing fix in the changelog.

## Reproduction Assessment

Yes. Static review gives a high-confidence reproduction path: on the PR head, a disabled or runtime-ineligible skill still listed in an agent skill filter would be exposed, and any caller-provided `envSanitizationOptions` would be dropped by `buildSandboxCreateArgs`.

## Solution Assessment

No. The `docker.env` bypass direction is plausible, but this patch is not the best solution because it mixes in a skill-gate bypass and regresses the sanitizer options contract instead of extending it.
