## Summary

Keep open: the PR targets a real sandbox env bug that current main still appears to have, but the branch is not merge-ready because it lacks real Docker proof and the diff weakens skill eligibility and sanitizer-option handling.

## What This Changes

Adds `forceAllowKeys` to sandbox env sanitization for Docker config env, exports an env validation helper, restores `/root` after review feedback, and lets explicit skill filters bypass eligibility checks.

## Best Possible Solution

Land a narrow sandbox fix that force-allows only explicit Docker config env while preserving sanitizer options, skill eligibility gates, the OpenClaw exec marker, regression tests, and redacted Docker proof.

## Reproduction Assessment

Yes, at source level: current main sends configured Docker env through a credential-name blocklist while docs direct sandboxed skill API keys to `agents.defaults.sandbox.docker.env`. I did not run a live Docker sandbox in this read-only review.

## Solution Assessment

No. The Docker env exception is plausible, but this PR should preserve `envSanitizationOptions`, keep `shouldIncludeSkill` as the prerequisite gate, and refresh against current main before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve skill eligibility before explicit filters:** `src/agents/skills/workspace.ts:79`
  - body: The explicit-name shortcut runs before `shouldIncludeSkill`, which is the gate for `skills.entries.<key>.enabled === false`, bundled allowlists, and runtime `requires` checks. A disabled or runtime-ineligible skill listed in an agent filter would become visible again; keep eligibility as the prerequisite and apply the explicit-name filter afterward.
  - confidence: 0.92
- **[P2] Preserve sanitizer options when force-allowing Docker env:** `src/agents/sandbox/docker.ts:370-372`
  - body: This call replaces `params.envSanitizationOptions` with a new object and force-allows every configured env key, so strict or custom sanitizer policy provided by callers can no longer constrain Docker env handling. Merge the existing options through and scope the bypass to the documented Docker env credential exception.
  - confidence: 0.88
