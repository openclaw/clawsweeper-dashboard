## Summary

Keep open: the PR fixes the reported WebChat blank-on-switch path and has useful real proof, but the new cache-key aliasing is broader than the gateway session contract and can mix non-main `agent:main:*` sessions into configured default-agent sessions.

## What This Changes

This PR adds in-memory Control UI WebChat message caching per session, alias-aware restoration and reset invalidation, switched-away final-message caching, regression tests, and a changelog entry.

## Best Possible Solution

Land the per-session WebChat cache after narrowing cache-key equivalence to the gateway session-store contract and adding a regression that keeps non-main `agent:main:*` sessions separate from configured default-agent sessions.

## Reproduction Assessment

Yes, by source and supplied proof. Current main clears visible messages on switch and ignores switched-away finals, while inspected Crabbox screenshots show the blank baseline and preserved candidate; the review finding is source-reproducible with `defaultAgentId: "ops"` plus a non-main `agent:main:*` key.

## Solution Assessment

No, not as-is. The per-session cache is the right boundary, but its alias helper must match gateway canonicalization before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Restrict `agent:main` cache aliases to main sessions:** `ui/src/ui/chat/session-message-cache-keys.ts:98-99`
  - body: `resolveSessionStoreKey` only remaps legacy `agent:main` aliases when the rest is the configured main key; non-main keys like `agent:main:discord:direct:u1` stay separate. This helper normalizes any `agent:main:<rest>` to `<rest>` and the equivalent-key path writes both `agent:main` and default-agent keys, so with `defaultAgentId: "ops"` the UI cache can restore or append messages from the old main-agent channel into the ops/default session. Please restrict `agent:main` equivalence to main/mainKey aliases and add a regression for a non-main `agent:main:*` key.
  - confidence: 0.88
