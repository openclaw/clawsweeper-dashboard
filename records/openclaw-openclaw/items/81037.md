Keep open. This PR is a focused fix for the WebChat regression tracked at https://github.com/openclaw/openclaw/issues/80855: current main clears visible chat messages on session switch and ignores final events for non-selected sessions, while the patch adds a per-session visible-message cache with focused regression coverage and supplied terminal proof. I found no blocking correctness or security issue; it should proceed through normal maintainer review and required CI.

## What This Changes

This PR adds a per-session WebChat message cache in the Control UI, updates history and final-event handling, and adds regression tests for session switch restoration and switched-away final messages.

## Best Possible Solution
