## What This Changes

The PR flips message-tool same-provider cross-context sends to opt-in, updates config help/types/tests/changelog, adds Telegram topic handling, and also includes unrelated failover and UI scroll changes.

## Best Possible Solution

Refresh this into a narrow outbound-policy hardening patch that flips the default, preserves current per-agent config handling, keeps Telegram matching behind channel-owned threading contracts, blocks topic-less topic-bound escapes, drops unrelated edits, and includes redacted real chat proof.

## Reproduction Assessment

Yes. Source inspection on current main shows same-provider bound sends return early unless `allowWithinProvider` is explicitly `false`; I did not run tests because this review was read-only.

## Solution Assessment

No. The default-deny direction is sound, but this branch crosses the Telegram plugin boundary, permits a topic-bound escape, carries unrelated changes, and lacks real behavior proof.
