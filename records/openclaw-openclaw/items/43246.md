The PR flips bound same-provider message sends to explicit opt-in and updates config help/types/tests/changelog, while also carrying unrelated failover and UI scroll changes.

## Best Possible Solution

Refresh this into a narrow outbound-policy hardening patch that flips the default, keeps provider-specific chat/topic matching behind plugin threading contracts, blocks topic-bound topic-less escapes, drops unrelated edits, and includes redacted real chat proof.
