## Summary

Keep this PR open because current main still permits same-provider cross-context sends by default and the linked bug fix remains useful, but the branch is not merge-ready due to missing real behavior proof, a Telegram topic escape, a stale core-to-Telegram dependency, and unrelated drive-by changes.

## What This Changes

The branch flips `tools.message.crossContext.allowWithinProvider` to opt-in, updates config help/tests/changelog, and also includes unrelated failover retry and UI scroll fallback changes.

## Best Possible Solution

Land a refreshed, narrow outbound-policy hardening patch that keeps provider-specific topic identity behind channel/plugin contracts, requires exact topic preservation for topic-bound Telegram actions, removes unrelated changes, adds focused regression tests, and includes redacted real bound-chat proof.

## Solution Assessment

No. The desired default-deny end state is sound, but this branch should be narrowed and moved to a plugin-owned same-context/topic contract before merge.
