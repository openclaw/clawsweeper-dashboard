## Summary

Keep this PR open, but it is not merge-ready: the distributed-lock direction is useful, yet the current patch can skip default queued ACP turns, misses the `/acp steer` entrypoint, does not stop active turns after lost Redis renewal, and lacks real Redis-backed behavior proof.

## What This Changes

The PR adds an optional Redis-backed ACP session lock manager, fail-closed Redis behavior, ACP dispatch acquire/renew/release wiring, env documentation, and focused tests.

## Best Possible Solution

Move the lease to the shared ACP turn boundary, preserve existing queued-turn semantics, cover steer turns, fail active work on lost renewal, rebase on current main, and add redacted real Redis-backed ACP proof.

## Reproduction Assessment

Yes for the blocking PR regressions by source inspection: current main queues same-session `runTurn` calls, while PR head skips on lock contention, and `/acp steer` remains outside the lease. I did not run PR head or a live Redis-backed ACP setup.

## Solution Assessment

No. The best direction is a shared ACP turn lease that preserves current queue semantics and covers all turn entrypoints, not a dispatch-only lock that drops contended prompts.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve queued ACP turns when the lock is contended:** `src/auto-reply/reply/dispatch-acp.ts:239-247`
  - body: `AcpSessionManager.runTurn()` already serializes same-session turns and current tests verify both requests execute. This new branch returns `acp_execution_locked` before `runTurn`, so the default local-lock path drops a second prompt instead of queueing it behind the active turn.
  - confidence: 0.92
- **[P2] Apply the lease to every ACP turn entrypoint:** `src/auto-reply/reply/dispatch-acp.ts:233-234`
  - body: The new lease is acquired only in `tryDispatchAcpReply`, but `/acp steer` still calls `acpManager.runTurn()` directly. In Redis mode, a steer command can overlap with a normal prompt on the same session across workers, leaving the session-state race this feature is meant to prevent.
  - confidence: 0.88
- **[P2] Stop active turns when Redis renewal is lost:** `src/auto-reply/reply/dispatch-acp.ts:271-283`
  - body: Renewal failure is logged and ignored, so a long turn can keep running past the Redis TTL after an outage while another worker acquires the same session. The code needs to abort or otherwise fail the active turn when the lease can no longer be renewed.
  - confidence: 0.86
- **[P2] Close Redis sockets when handshake commands fail:** `src/auto-reply/reply/session-lock-manager.ts:283-286`
  - body: `connectRedis()` opens the socket and then awaits AUTH/SELECT before returning it to the caller that has the `finally` close. If AUTH or SELECT rejects, repeated ACP attempts leak TCP connections while still failing closed.
  - confidence: 0.84
