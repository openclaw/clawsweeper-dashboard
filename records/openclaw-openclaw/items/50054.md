## What This Changes

This PR adds optional Redis-backed ACP session locks, local/fail-closed lock managers, dispatch acquire/renew/release wiring, env examples, and focused lock/dispatch tests.

## Best Possible Solution

Move distributed leasing into the shared ACP turn boundary or an equivalent manager-level wrapper, preserve same-process queuing, define renewal-loss behavior, close Redis setup sockets, and require redacted Redis-backed runtime proof before merge.

## Reproduction Assessment

Yes for the PR regressions by source inspection: current main queues same-session runTurn calls, while the PR returns acp_execution_locked before that queue; current /acp steer calls runTurn outside the new lease; renewal loss is log-only. I did not run PR head or a Redis-backed ACP setup in this read-only review.

## Solution Assessment

No. The requested distributed locking direction may be useful, but this implementation should be moved to the shared ACP turn boundary with explicit renewal-loss handling and Redis cleanup instead of only wrapping tryDispatchAcpReply.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Preserve queued ACP turns when the lock is contended:** `src/auto-reply/reply/dispatch-acp.ts:240-247`
  - body: `AcpSessionManager.runTurn()` already serializes same-session turns and current tests exercise two same-session calls both executing. In the default no-Redis path, `LocalSessionLockManager` makes the second prompt take this branch and get marked skipped, so users can lose a normal queued turn instead of having it run after the active turn.
  - confidence: 0.94
- **[P2] Lease the shared ACP turn path:** `src/auto-reply/reply/dispatch-acp.ts:233-239`
  - body: This lock is only acquired in `tryDispatchAcpReply()`, while `/acp steer` still calls `acpManager.runTurn()` directly. In Redis mode, a steer command and normal prompt for the same session can overlap on different workers, so the distributed lock does not protect the session state it is meant to serialize.
  - confidence: 0.9
- **[P2] Stop the turn when Redis renewal is lost:** `src/auto-reply/reply/dispatch-acp.ts:271-283`
  - body: Renewal failure is logged but the turn continues; after TTL expiry another worker can acquire the same session while the first turn remains active. Distributed locking needs to abort/fail the turn or otherwise prevent lease loss from allowing concurrent session mutation.
  - confidence: 0.86
- **[P2] Close sockets when Redis setup fails:** `src/auto-reply/reply/session-lock-manager.ts:283-286`
  - body: `connectRedis()` creates the socket and then awaits `AUTH`/`SELECT` before returning the connection to the caller that closes it. If either setup command rejects or times out, the opened socket is never closed, so repeated fail-closed ACP attempts against a misconfigured Redis can leak connections.
  - confidence: 0.88
