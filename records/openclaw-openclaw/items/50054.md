## Summary

## Best Possible Solution

Move lease ownership to a shared ACP turn boundary that preserves same-session queueing, covers prompt and steer turns, aborts or fails closed when the lease is lost, guarantees Redis socket cleanup, and is proven with redacted real Redis-backed ACP output.

## Reproduction Assessment

Yes, for the blocking PR regressions by source inspection: PR head skips contended turns before the existing manager queue, while current main queues same-session runTurn calls; /acp steer also remains outside the lease. I did not run PR head or a live Redis-backed ACP setup.

## Solution Assessment

No. Wrapping only automatic prompt dispatch with a non-waiting lock is not the best boundary because ACP already has a shared manager queue and steer is another production runTurn entrypoint.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Preserve queued ACP turns when the lock is contended:** `src/auto-reply/reply/dispatch-acp.ts:239-247`
  - body: This returns `acp_execution_locked` whenever `acquire()` is contended. In the default no-Redis path the local lock manager is used, so a second same-session ACP prompt is skipped before `AcpSessionManager.withSessionActor()` can queue it; current main has coverage expecting both concurrent turns to execute serially.
  - confidence: 0.94
- **[P2] Apply the lease at the shared ACP turn boundary:** `src/auto-reply/reply/dispatch-acp.ts:233-239`
  - body: The lock is acquired only in `tryDispatchAcpReply()`, but `/acp steer` still calls `acpManager.runTurn()` directly. In Redis mode, a steer turn and an automatic prompt for the same session can run on different workers, so the distributed lock does not protect the shared ACP state boundary.
  - confidence: 0.9
- **[P2] Abort active turns after lease renewal fails:** `src/auto-reply/reply/dispatch-acp.ts:271-284`
  - body: The heartbeat only logs false or throwing `renew()` results and lets `runTurn` continue. If the Redis TTL expires after a missed renewal, another worker can acquire the same session while the first turn is still active, defeating the lock on long turns.
  - confidence: 0.88
- **[P2] Close Redis sockets when initialization fails:** `src/auto-reply/reply/session-lock-manager.ts:283-286`
  - body: `connectRedis()` returns only after AUTH/SELECT, while `createRedisCommandRunner()` installs its `close()` finally only after `connectRedis()` resolves. Bad credentials or DB selection errors can reject with an opened socket that is never closed.
  - confidence: 0.86
