## Summary

Keep open. Current main and v2026.6.10 still explicitly require `thread: true` for `mode="session"` in both native subagent and ACP spawn paths, and the remaining request is a product/contract decision rather than stale cleanup.

## What This Changes

The issue requests allowing persistent `sessions_spawn` session mode to work without requiring channel thread binding.

## Best Possible Solution

Define a first-class non-thread persistent-session contract, then update native spawn, ACP spawn, tool schema, follow-up targeting, lifecycle cleanup, docs, and focused tests around that contract.

## Reproduction Assessment

Yes, source-level. Current main and v2026.6.10 still reject both native and ACP `sessions_spawn` calls with `mode="session"` unless `thread: true` is set, and tests/docs preserve that contract.

## Solution Assessment

Unclear. Decoupling persistence from thread binding is a plausible direction, but the best fix needs an explicit non-thread ownership, delivery, and cleanup contract rather than direct guard removal.
