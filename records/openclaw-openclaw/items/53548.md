## Summary

Keep open. Current main still rejects unthreaded persistent `sessions_spawn` in both native subagent and ACP paths, and the tests/docs continue to encode `thread=true` as required; the open PR targeting this behavior is not merged or shipped.

## What This Changes

Allow `sessions_spawn` with `mode="session"` for native subagent and ACP runtimes without requiring `thread:true`, so persistent headless sessions can be continued via `sessions_send` on channels without thread binding.

## Best Possible Solution

Implement and document headless persistent native and ACP sessions while preserving existing `thread:true` behavior, returning stable continuation metadata for `sessions_send`, and adding regression coverage for non-thread channels plus ACP delivery semantics.
