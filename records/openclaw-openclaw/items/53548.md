## Summary

Keep open: current main still rejects unthreaded session mode for both native subagent and ACP spawns, and the exact open implementation path is unmerged with routing-contract and real-behavior-proof blockers.

## What This Changes

Allow `sessions_spawn` with `mode="session"` to create persistent native/ACP sessions without requiring `thread=true`, while keeping thread binding optional.

## Best Possible Solution

Land a maintainer-approved version of https://github.com/openclaw/openclaw/pull/39372 or a replacement that allows headless persistent sessions, gates ACP inline delivery on an explicit binding/relay contract, updates docs/tests, and proves both non-threaded and thread-bound behavior.

## Reproduction Assessment

Yes. Source inspection on current main shows native and ACP guards returning errors for `mode="session"` without `thread=true`, and the existing spawn-hooks test asserts the native rejection path.

## Solution Assessment

Unclear as a direct guard removal. Decoupling is plausible, but the implementation should first settle the ACP delivery contract so headless persistent sessions do not accidentally use thread-bound inline delivery semantics.
