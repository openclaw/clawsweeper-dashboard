## What This Changes

The issue requests allowing `sessions_spawn` `mode="session"` to create persistent native or ACP child sessions without requiring `thread: true`.

## Best Possible Solution

Land a rebased, maintainer-approved update to https://github.com/openclaw/openclaw/pull/39372 or a replacement that defines headless native/ACP session delivery, updates schema/docs/tests, and proves non-threaded plus thread-bound behavior.

Yes, source-level. Current main rejects `mode="session"` without `thread=true` in both native and ACP paths, and the schema/docs still encode the thread-bound contract.

## Solution Assessment

Unclear as a direct guard removal. Decoupling is plausible, but the maintainable solution must define headless native/ACP delivery, update the model-facing schema and docs, and prove both non-threaded and thread-bound behavior.
