## Label Justifications

- `P2`: The request addresses a real orchestration limitation with limited blast radius, but it is a feature/product-contract change rather than an urgent regression.
- `impact:session-state`: The issue is about whether persistent child sessions can exist and remain reachable without a bound channel thread.
- `impact:message-loss`: The requested behavior changes how child-session output and follow-up messages are routed when no channel thread binding exists.

## Summary

Keep open: current main and the latest release still reject unthreaded native and ACP session-mode spawns, while the open implementation path at https://github.com/openclaw/openclaw/pull/39372 is still blocked by ACP delivery semantics, schema/docs updates, conflicts, and real behavior proof.

## What This Changes

Request to allow `sessions_spawn` `mode="session"` without requiring `thread=true` so persistent native and ACP child sessions can be used without channel thread binding.

## Best Possible Solution

Land a rebased, maintainer-approved update to https://github.com/openclaw/openclaw/pull/39372 or a replacement that defines headless native/ACP session delivery, updates schema/docs/tests, and proves non-threaded plus thread-bound behavior.

## Reproduction Assessment

Yes. Source inspection on current main shows native and ACP guards rejecting `mode="session"` without `thread=true`, and the existing spawn-hooks test asserts the native rejection path.

## Solution Assessment

Unclear as a direct guard removal. Decoupling is plausible, but the maintainable solution must settle ACP headless delivery, update the tool schema and docs, rebase the linked implementation, and provide real behavior proof.
