## Summary

Keep open. Current main and v2026.6.10 still reject `sessions_spawn` with `mode="session"` unless `thread: true` is set in both native subagent and ACP paths, and related work covers narrower plugin, affinity, or thread-policy slices rather than the broad core contract requested here.

## What This Changes

Request to allow persistent native subagent and ACP `sessions_spawn` sessions without requiring a channel thread binding.

## Best Possible Solution

Define a first-class non-thread persistent-session contract, then update native spawn, ACP spawn, tool schema, follow-up targeting, lifecycle cleanup, docs, and focused tests around that contract.

## Reproduction Assessment

Yes, source-level. Current main still rejects both native and ACP `sessions_spawn` calls with `mode="session"` unless `thread: true` is set, and tests/docs preserve that contract.

## Solution Assessment

Unclear. Decoupling persistence from thread binding is a plausible direction, but the best fix needs an explicit non-thread ownership, delivery, and cleanup contract rather than direct guard removal.
