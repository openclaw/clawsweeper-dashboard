## Summary

Keep open: current main and the latest release still reject unthreaded native and ACP session-mode spawns, and the open implementation path needs maintainer decisions around delivery semantics, schema/docs, conflicts, and proof before this can close.

## What This Changes

The issue asks OpenClaw to allow persistent `sessions_spawn` session mode without requiring channel thread binding.

## Best Possible Solution

Land a rebased, maintainer-approved update to https://github.com/openclaw/openclaw/pull/39372 or a replacement that defines headless native/ACP session delivery, updates schema/docs/tests, and proves non-threaded plus thread-bound behavior.

## Reproduction Assessment

Yes, source-level. Current main rejects `mode="session"` without `thread=true` in both native and ACP paths, and existing schema/tests/docs still encode the thread-bound contract.

## Solution Assessment

Unclear as a direct guard removal. Decoupling is plausible, but the maintainable solution must define headless native/ACP delivery, update tool schema and docs, rebase the linked implementation, and provide real behavior proof.
