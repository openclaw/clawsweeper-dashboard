Keep open: current main and the latest release still reject unthreaded native and ACP session-mode spawns, and the exact open implementation path still needs maintainer agreement on ACP delivery semantics plus real behavior proof.

## What This Changes

Request to allow persistent native subagent and ACP session mode without requiring channel thread binding, so parent agents can relay follow-ups with `sessions_send` on channels without threads.

## Best Possible Solution

Land a maintainer-approved update to https://github.com/openclaw/openclaw/pull/39372 or a replacement that enables headless persistent sessions, makes ACP delivery explicit, updates docs/tests, and proves both headless and thread-bound behavior.

## Reproduction Assessment

Yes. Source inspection on current main shows native and ACP guards rejecting `mode="session"` without `thread=true`, and the existing spawn-hooks test asserts the native rejection path.

## Solution Assessment

Unclear as a direct guard removal. Decoupling is plausible, but the maintainable solution must also settle ACP headless-session delivery semantics and update docs and proof.
