## What This Changes

The PR makes main-session `systemEvent` cron jobs with `wakeMode: "now"` use the existing 60-minute agent-turn safety timeout and adds focused timeout-policy plus timer regression coverage.

## Best Possible Solution

Land the narrow timeout-policy fix with its regression coverage once maintainers accept the bounded 60-minute slot tradeoff and the PR is refreshed against any overlapping cron changes.

## Solution Assessment

Yes. Special-casing only the blocking main-session wake-now systemEvent path in timeout policy is narrower than adding a new configurable watchdog or making all system events long-running.
