## Summary

Keep open. Current main already fixes the stale-threshold side with interval-relative validation, but the restart-cap zero value remains an unresolved gateway config contract rather than a safe schema-only cleanup.

## What This Changes

The issue asks to change gateway config validation so stale channel thresholds cannot undercut the health-check interval and `channelMaxRestartsPerHour` can be set to `0`.

## Best Possible Solution

Keep the interval-relative stale-threshold rule, then decide the public zero-value restart-cap contract and update schema, runtime behavior, docs/help, generated metadata, and focused tests together.

## Reproduction Assessment

Yes for source-level reproduction of the remaining behavior: current main rejects `gateway.channelMaxRestartsPerHour: 0`, while the runtime would treat an accepted zero as zero allowed restarts. The stale-threshold part no longer reproduces because current validation rejects thresholds below the effective health-check interval.

## Solution Assessment

No, the literal schema-only suggestion is not the best fix. A static `.min(5)` is less precise than the current cross-field stale-threshold rule, and `.min(0)` needs a maintainer-approved zero-value contract plus runtime/docs/test updates.
