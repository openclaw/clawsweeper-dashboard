## Best Possible Solution

Land a maintainer-approved fix that keeps the direct fallback useful while ensuring internal runtime-context delimiters and queue persistence cannot expose OpenClaw scaffolding across external channel delivery.

A broader policy change to withhold all child output would be a separate product decision because current docs describe direct fallback from the captured child result.
