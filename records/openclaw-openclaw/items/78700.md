## Summary

Keep open: the reported bug is valid and the PR is focused, but the current head introduces an overbroad sanitizer unwrap that can remove literal prompt-data/untrusted-text lines from outbound content and queued retries.

## What This Changes

The PR rewrites subagent child-result prompt wrappers, strips internal runtime scaffolding from direct fallback and queue persistence, rebuilds sanitized queued batch plans, adds regression tests, and updates the changelog.

## Best Possible Solution

Keep the layered fallback and queue-sanitization fix, but narrow prompt-data unwrapping so only generated wrapper blocks are removed while literal user-authored tag examples are preserved.

## Reproduction Assessment

Yes, at source level. Current main returns raw `event.result` for completion fallback and persists original outbound payloads to the write-ahead queue before later normalization, which matches the related Telegram queue evidence.

## Solution Assessment

No, not as currently written. The layered fix is the right direction, but `unwrapPromptDataWrapperLines()` should gate tag removal to a generated header plus matching wrapper instead of deleting every standalone prompt-data/untrusted-text tag line.
