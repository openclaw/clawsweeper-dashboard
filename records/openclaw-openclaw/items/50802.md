## What This Changes

The PR extends session transcript repair and tool-call ID sanitization to recognize snake_case tool-call block types, adds regression tests for repair/result pairing/redaction/ID rewriting, and adds a changelog entry.

## Best Possible Solution

Land a rebased or replacement fix that uses one consistent tool-call type and id extractor across transcript repair, result pairing, signed-thinking reservation, and Cloud Code Assist ID rewriting, with real proof from a repaired strict-provider transcript.

## Reproduction Assessment

Yes, source-level. On current main, `repairToolUseResultPairing` calls `extractToolCallsFromAssistant`, but that helper only recognizes camelCase tool-call types, so snake_case assistant tool calls are invisible to result-pairing and ID-rewrite paths.

## Solution Assessment

No, not as-is. The direction is right, but the PR widens snake_case type recognition without extracting alternate id fields such as `call_id`, `tool_call_id`, or `tool_use_id`, and it still lacks real behavior proof.
