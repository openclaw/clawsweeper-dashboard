## Summary

Keep open: current main already has the sanitizer-side snake_case repair, but `tool-call-id.ts` still uses a camelCase-only tool-call type set, and this PR’s latest head adds the remaining helper alignment plus focused tests. It still needs human handling because the branch is conflicting and the PR lacks after-fix real behavior proof from a real repaired session path.

## What This Changes

The PR broadens transcript repair, tool-call extraction/ID rewriting, regression tests, and changelog text to handle `tool_call`, `tool_use`, and `function_call` blocks.

## Best Possible Solution

Rebase or replace this with a narrow patch that preserves current main’s sanitizer implementation, adds snake_case support to shared tool-call extraction/ID helpers with focused regression tests, and includes redacted real runtime proof before merge.

## Reproduction Assessment

Yes, at source level. On current main, a snake_case assistant block such as `tool_use` is recognized by transcript repair but is not extracted or rewritten by `tool-call-id.ts` because `TOOL_CALL_TYPES` is camelCase-only; I did not run a live Anthropic/Feishu interrupted-session path in this read-only review.

## Solution Assessment

Mostly yes. Aligning the shared tool-call recognizers on the same snake_case-aware type set is the narrow maintainable fix, but this branch should be rebased or trimmed around current main and backed by real behavior proof before merge.
