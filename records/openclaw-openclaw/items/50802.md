## Summary

Keep open: current main still has camelCase-only tool-call predicates in the transcript repair and ID-rewrite path, while the local Anthropic transport emits assistant tool calls as `tool_use`; the PR addresses that gap and the earlier review concern around `sessions_spawn` redaction coverage.

## What This Changes

The PR extends transcript repair and tool-call ID helpers to recognize `tool_call`, `tool_use`, and `function_call`, with regression tests and a changelog entry for linked #48915.

## Best Possible Solution

Land this PR, or an equivalent main-branch fix, after exact-head validation; keep linked #48915 open until the implementation merges.

## Reproduction Assessment

Yes, from source inspection. A persisted assistant block like `{ type: "tool_use", id: "call_1", name: "read" }` missing `input` and `arguments` is not recognized by current main's detector, so it bypasses the documented malformed-tool-call drop path before replay.

## Solution Assessment

Yes. The proposed fix is narrow and maintainable because it updates the detector, rebuild/redaction branch, result-pairing extraction, ID rewriting, tests, and changelog together while preserving existing provider-specific block shapes.
