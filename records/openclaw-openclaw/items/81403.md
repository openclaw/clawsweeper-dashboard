## Summary

Keep this PR open: it is a plausible host exec bug fix, but the current branch still lacks required after-fix real behavior proof and its new RC override test models the override after the wrapper, so it does not prove the promised PATH precedence.

## What This Changes

The PR threads `tools.exec.pathPrepend` into gateway exec runtime, wraps POSIX host commands with an `OPENCLAW_PREPEND_PATH` PATH export, adds helper/tests, and documents the new PATH precedence behavior.

## Best Possible Solution

Keep the host exec fix narrow: apply configured prepends after POSIX startup for gateway execution, back it with a real startup-file regression test, and require redacted macOS gateway proof showing the final PATH order.

## Reproduction Assessment

Unclear for the original macOS `.zshenv` bug because the PR body gives steps but no captured output or artifact. The added test flaw is reproducible from the shell command order: its simulated override runs after the wrapper and places `/evil/bin` first.

## Solution Assessment

No. The wrapper direction matches the existing sandbox pattern, but this PR is not yet the best mergeable solution because the regression test validates the wrong order and the external real behavior proof is missing.
