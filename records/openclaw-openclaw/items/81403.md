## Summary

Keep open: current main lacks a final shell-level PATH re-prepend, and the PR’s runtime direction is plausible, but the final head still leaves the RC-override regression test invalid because `BASH_ENV` is sanitized and the mock reads `PATH` from spawn env only.

## What This Changes

The PR threads `tools.exec.pathPrepend` into host exec runtime and wraps POSIX commands so configured PATH entries are re-applied after shell startup files run.

## Best Possible Solution

Keep the final-shell PATH prepend approach, but land it with a regression test that uses a real shell startup path or accurately models sanitized env plus wrapper execution.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path for the current-main gap: PATH is prepended before shell launch and no final wrapper runs after startup files. The PR body also supplies after-fix live output, but I did not run a live macOS/zsh reproduction in this read-only pass.
