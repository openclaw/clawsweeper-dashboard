## Summary

Keep open: the runtime direction is plausible and the PR has supplied real behavior proof, but the patch still has a source-level test blocker because the added RC override regression test does not actually model shell startup or the wrapper effect.

## What This Changes

The PR threads `tools.exec.pathPrepend` into host exec runtime, wraps POSIX commands with a final PATH export, adds path helper coverage, and documents the behavior.

## Best Possible Solution

Land the final-shell PATH prepend approach with regression coverage that actually exercises startup-file ordering and docs that describe the gateway behavior at the correct scope.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for the review finding: source inspection shows the new test can only read PATH from the mock spawn env while `BASH_ENV` is filtered, so it cannot observe the claimed RC-startup ordering.

## Solution Assessment

Is this the best way to solve the issue? Not yet; the runtime boundary is reasonable, but the regression test and docs placement need to be fixed before merge.
