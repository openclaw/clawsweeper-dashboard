## Summary

Keep open: current main and the latest release still require a concrete resolved delivery target before a successful Telegram message-tool send can verify hook delivery, so the targetless Gmail hook failure remains.

## What This Changes

The issue reports Gmail hook runs being marked error after a successful `message(action="send", channel="telegram", target=...)` send because hook delivery verification cannot match the target.

## Best Possible Solution

Define the hook delivery verification policy, then add regression coverage so same-channel explicit message-tool sends either satisfy the hook safely or fail with expected and observed target details.

## Reproduction Assessment

Yes from source inspection, though no live Gmail hook was run here: hook dispatch can pass `channel` with no `to`, and the verifier refuses to match a successful message-tool target unless `delivery.to` exists.

## Solution Assessment

Unclear until maintainers choose policy; the safest implementation should align hook dispatch, source-delivery verification, diagnostics, and regression tests rather than only loosening the matcher.
