Keep open for maintainer review: the patch is narrow and appears correct, but it intentionally changes bundled skill behavior by requiring resolved-preview approval before mutating actions.

## What This Changes

The branch tightens bundled `gog`, `imsg`, `oracle`, `ordercli`, and `wacli` `SKILL.md` instructions to require resolved previews and explicit approval before sends, reorders, calendar writes, and uploads.

## Best Possible Solution

Land the narrow wording if maintainers accept instruction-level safety as the right layer, and keep runtime enforcement, allowlists, audit logging, and bulk-send policy in https://github.com/openclaw/openclaw/issues/56898.

## Reproduction Assessment

Yes for the documentation gap: current `main` has broad confirmation wording and direct mutating examples, while the PR diff binds approval to resolved targets and previews. Live integration testing is not necessary for the prose-only behavior being changed.

## Solution Assessment

Yes for the stated scope: editing the bundled skill instructions is the narrowest maintainable way to improve model-followed guidance. If maintainers want non-bypassable guarantees, that should be separate runtime/tool enforcement rather than hidden in this prose-only PR.
