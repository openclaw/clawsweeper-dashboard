## Summary

Keep open. Current main still selects the `openclaw agent` session before channel/account delivery context is available to the resolver, and the resolver still has no scan for an existing Discord channel/thread session by normalized delivery context. The reporter’s dist-level workaround and the closed duplicate reports point to a source-level gap that is still present; the related open #42009 is adjacent but only targets `--agent` + `--to` precedence, not channel/account delivery-context lookup.

## Best Possible Solution

Implement generic delivery-context session routing for `openclaw agent` when a channel target is provided and no explicit selector should win. Normalize the target through the channel route parser, compare against persisted `deliveryContext` plus legacy `lastChannel`/`lastTo`/`lastAccountId`/`lastThreadId`, preserve explicit selector precedence, and update CLI docs/changelog to document the supported behavior.

## Repair Work Prompt

Fix the unresolved `openclaw agent --channel discord --to <target>` routing bug. Current main selects a session before channel/account delivery context is part of session resolution, so raw Discord channel/thread targets can collapse to `agent:*:main` instead of an existing live channel/thread session.

Implement a generic source-level fix, not dist patching and not Discord-specific core ids. Extend the agent session resolver path so when a channel target is provided and no explicit selector should win, it normalizes the target through the channel route parser and searches the relevant session store for an existing entry whose normalized `deliveryContext` or legacy `lastChannel`/`lastTo`/`lastAccountId`/`lastThreadId` matches. Preserve precedence for explicit `sessionKey`, `sessionId`, and explicit agent/main-session behavior. Keep channel/plugin boundary rules intact; use existing route parsing helpers or add a small generic seam if needed. Update regression tests, `docs/cli/agent.md`, `docs/tools/agent-send.md` if behavior changes there, and add a user-facing changelog fix entry with reporter attribution. Do not edit compiled `dist` bundles.

Related refs: this issue, #44450, #44451, #44452, and adjacent #42009. Coordinate with #42009’s `--agent` + `--to` precedence so both behaviors remain covered.
