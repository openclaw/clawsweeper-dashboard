## Summary

Keep open: current main still resolves `openclaw agent` sessions from raw `--to` before using Discord delivery context, so existing live channel/thread sessions can be bypassed.

## What This Changes

The issue asks `openclaw agent --channel discord --to <target>` to route to the live Discord channel/thread session instead of falling back to `agent:*:main`.

## Best Possible Solution

Add a generic delivery-context lookup in the source resolver for channel-targeted agent CLI calls, using existing channel target parsing while preserving explicit session selectors and agent scoping.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: with a stored `agent:main:discord:channel:<id>` session, `openclaw agent --channel discord --to <id>` resolves from raw `to` before Discord normalization and never scans stored delivery context.

## Solution Assessment

Yes. The narrow maintainable fix is source-level resolver and CLI wiring work; compiled `dist` patching is temporary, and #42009 is adjacent but does not solve delivery-context lookup for live Discord sessions.
