## Summary

Keep open. Current main and the latest release still expose only selected-row bulk delete in the Sessions view; archive visibility is partially present via a Show archived filter, but manual archive/unarchive remains tied to open archive and retention-policy work.

## What This Changes

The issue requests per-row delete/archive or dropdown actions in the Control UI Sessions table so individual sessions can be cleaned up without selecting rows first.

## Best Possible Solution

Split the narrow per-row Delete affordance from the broader Archive/Unarchive design, then land archive behavior through a reviewed sessions API and retention policy coordinated with https://github.com/openclaw/openclaw/pull/47090 and https://github.com/openclaw/openclaw/issues/56609.

## Reproduction Assessment

Yes for the current-state claim: source inspection on current main shows only selected-row bulk Delete wiring and no per-row action column or archive/unarchive callback. The Archive portion is a new feature/product contract rather than a broken existing behavior.

## Solution Assessment

No as one combined small fix. Per-row Delete can be a narrow UI change, but Archive/Unarchive needs the sessions API and retention semantics settled first.
