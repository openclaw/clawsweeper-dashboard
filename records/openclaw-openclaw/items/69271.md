## Summary

Keep open. Current main still lacks a per-row Sessions action and manual archive/unarchive API; the related archive work remains open and overlaps a broader retention-policy decision.

## What This Changes

Request a per-session row action in the Control UI Sessions view for quick delete/archive, optionally through an actions dropdown.

## Best Possible Solution

Split the narrow per-row Delete affordance from the broader Archive/Unarchive contract, then land archive behavior through a reviewed sessions API and retention policy coordinated with https://github.com/openclaw/openclaw/pull/47090 and https://github.com/openclaw/openclaw/issues/56609.

## Reproduction Assessment

Yes for the current-state claim: source inspection on current main and v2026.5.18 shows only selected-row bulk Delete wiring and no row-level action or archive/unarchive callback. The Archive part is a feature/product request rather than a broken existing contract.

## Solution Assessment

No for implementing the whole request as one small fix. Per-row Delete is narrow, but Archive/Unarchive needs a typed API, retention behavior, and coordination with the existing open archive and retention-policy threads.
