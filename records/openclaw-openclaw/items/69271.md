## Summary

Keep open. Current main still has selected-row bulk Delete only, plus archived-row visibility, but no per-row Delete/Archive action or manual Archive/Unarchive contract. Related work exists, especially https://github.com/openclaw/openclaw/pull/47090 and https://github.com/openclaw/openclaw/issues/54397, but nothing merged currently closes this focused UI request.

## What This Changes

The issue requests per-session row actions for deleting or archiving sessions in the Control UI Sessions table.

## Best Possible Solution

Split the immediate per-row Delete affordance from the broader Archive/Unarchive design, then route archive semantics through the existing session-management and retention discussions before exposing a manual UI action.

## Reproduction Assessment

Yes for the current-state claim: current main renders only selected-row bulk Delete and wires only onDeleteSelected, with no row action callback or Actions column. The Archive request is a feature/product design gap rather than a failing existing contract.

## Solution Assessment

No for implementing the full request as one small fix. Per-row Delete is narrow, but manual Archive/Unarchive should first align with the session lifecycle API, retention policy, and the active archive implementation candidate.
