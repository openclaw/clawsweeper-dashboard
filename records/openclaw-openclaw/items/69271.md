## Summary

Keep open. Current main still only supports checkbox selection plus the bulk delete bar for Control UI Sessions cleanup, with no per-row Delete/Actions affordance. A narrow per-row Delete repair is clear because the existing controller already accepts a one-key delete array, while first-class Archive remains broader Gateway/session API work.

## What This Changes

Add a per-session Delete and/or Archive action in the Control UI Sessions table so users can clean up one session without selecting it for bulk deletion.

## Best Possible Solution

Add a compact per-row Delete or Actions affordance in the Sessions table that delegates to the existing deletion controller for a single key, and track first-class Archive separately as Gateway/session API design work.

## Reproduction Assessment

Yes. This is a feature gap, and current-main source inspection shows the Sessions view has only checkbox selection plus a bulk Delete bar, while the existing controller can delete a one-key array.

## Solution Assessment

Yes for the narrow delete affordance: reusing deleteSessionsAndRefresh(state, [key]) is the smallest maintainable path because it preserves the existing confirmation, loading, refresh, and error behavior. No for adding Archive in the same small repair, because archived-session state/filter/restore semantics do not exist yet.
