## Summary

Keep open. Current main has archived-session filtering, but the Sessions table still only offers selected-row bulk delete and has no per-row action column, single-row delete callback, or manual Archive/Unarchive API.

## What This Changes

Request adds per-row Sessions table actions for deleting or archiving individual sessions, with a possible actions dropdown and archived-session view.

## Best Possible Solution

Split the narrow per-row Delete UI from the broader Archive/Unarchive contract, then land archive behavior through a reviewed sessions API and retention design coordinated with https://github.com/openclaw/openclaw/pull/47090 and https://github.com/openclaw/openclaw/issues/56609.

## Reproduction Assessment

Yes for the current-state claim: source inspection on current main shows only selected-row bulk Delete wiring and no per-row action column or callback. The Archive part is a feature/product request rather than a broken existing contract.

## Solution Assessment

No for implementing the whole request as one small fix. Per-row Delete is narrow, but Archive/Unarchive needs a settled API, retention behavior, and coordination with the existing open archive and retention-policy threads.
