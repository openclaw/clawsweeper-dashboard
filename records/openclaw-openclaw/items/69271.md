## Summary

Keep open. Current main and the latest stable release still expose selected-row bulk Delete plus an archived-row filter, but not a per-row Delete/Archive action; the Archive part is related to broader open session-retention work rather than already settled behavior.

## What This Changes

Request to add per-session Delete/Archive controls, or an actions menu, to each row in the Control UI Sessions table.

## Best Possible Solution

Split the low-risk row-level Delete affordance from the broader Archive/Unarchive contract, then land archive behavior through coordinated sessions API and retention-policy work.

## Reproduction Assessment

Yes for the current-state claim: source inspection shows selected-row bulk Delete and archived filtering, but no per-row Delete/Archive action callback or action column. The archive behavior is a feature/product contract, not a broken existing API.

## Solution Assessment

No as one combined small fix. Per-row Delete can be handled narrowly, but Archive/Unarchive should wait for the session archive and retention semantics to settle.
