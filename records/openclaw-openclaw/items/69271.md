## Summary

Keep open. Current main still lacks a per-row Control UI Sessions action affordance; the Delete part is narrow and can reuse the existing deletion controller, while Archive still needs explicit Gateway/session API design.

## What This Changes

Request to add per-session Delete and/or Archive controls to each row in the Control UI Sessions table.

## Best Possible Solution

Add a compact per-row Delete or Actions affordance that delegates to the existing one-key deletion controller path, and handle first-class Archive as separate Gateway/session API design.

## Reproduction Assessment

Yes. Source inspection shows the Sessions table still only supports checkbox selection plus the bulk Delete bar, while deleteSessionsAndRefresh already supports a one-key array.

## Solution Assessment

Yes for the narrow Delete affordance: reusing deleteSessionsAndRefresh(state, [key]) preserves existing confirmation, loading, refresh, and error handling. No for Archive in the same repair because the archive-only state and restore semantics are not defined.
