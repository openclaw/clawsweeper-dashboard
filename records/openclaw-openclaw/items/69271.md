## Summary

Keep open. Current main still requires selecting rows before deleting, and archive/unarchive remains an unsettled feature with an open related PR and retention-policy issue rather than an implemented Control UI action.

## What This Changes

Request adds a per-row Sessions table action for single-session delete and optionally archive/unarchive instead of relying on checkbox bulk delete.

## Best Possible Solution

Split the narrow per-row Delete UI from the broader Archive/Unarchive contract, then land archive behavior through a reviewed sessions API and retention design coordinated with https://github.com/openclaw/openclaw/pull/47090 and https://github.com/openclaw/openclaw/issues/56609.

## Reproduction Assessment

Yes for the current-state claim: source inspection on current main shows only selected-row bulk Delete wiring and no per-row action column or callback. The Archive part is a feature/product request rather than a broken existing contract.

## Solution Assessment

No for implementing the whole request as one small fix. Per-row Delete is narrow, but Archive/Unarchive needs a typed API, retention behavior, and coordination with the existing open archive and retention-policy threads.
