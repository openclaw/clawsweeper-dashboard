## Summary

Keep open. Current main and the latest release still provide selected-row bulk Delete only in the Sessions table, while Archive/Unarchive remains an unsettled feature/API/retention design that overlaps the open archive work.

## What This Changes

Request adding per-row delete/archive controls or an actions dropdown in Control UI Sessions for quick single-session cleanup.

## Best Possible Solution

Split the narrow per-row Delete affordance from Archive/Unarchive, then land archive behavior through a reviewed sessions API and retention contract coordinated with https://github.com/openclaw/openclaw/pull/47090 and https://github.com/openclaw/openclaw/issues/56609.

## Reproduction Assessment

Yes for the current-state gap: source inspection shows only selected-row bulk Delete wiring and no per-row action column or callback. The Archive part is a feature/product request rather than a broken existing contract.

## Solution Assessment

No for implementing the whole request as one small fix. Per-row Delete is narrow, but Archive/Unarchive needs a settled API, retention behavior, and coordination with the existing open archive and retention-policy threads.
