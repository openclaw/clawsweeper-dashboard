## Summary

Keep open. Current main and the latest release still expose selected-row bulk Delete, not a per-row Sessions action, and the Archive/Unarchive part overlaps unresolved archive and retention-policy work.

## What This Changes

Request adds a per-session row action in the Control UI Sessions table for quick Delete and optional Archive/Unarchive instead of relying only on selected-row bulk actions.

## Best Possible Solution

Split the row-level Delete affordance from the broader Archive/Unarchive contract, then land archive behavior through coordinated sessions API and retention work tracked by the related open items.

## Reproduction Assessment

Yes for the current-state claim: source inspection shows selected-row bulk Delete and archived filtering, but no per-row Delete/Archive action callback or action column. The archive behavior is a feature/product contract, not a broken existing API.

## Solution Assessment

No as one combined small fix. Per-row Delete can be handled narrowly, but Archive/Unarchive should wait for the session archive and retention semantics to settle.
