Keep open. Current main still supports only checkbox-selected bulk deletion in the Control UI Sessions table; archived-session visibility exists, but per-row Delete and a coherent manual Archive/Unarchive workflow are not merged.

## Best Possible Solution

Split the immediate per-row Delete affordance from the broader Archive/Unarchive design, then land Archive only through a reviewed session-archive contract with retention and recovery behavior settled.

## Solution Assessment

No for implementing the whole request as one small fix. Per-row Delete is narrow, but manual Archive/Unarchive needs a settled API and retention design, likely coordinated with the existing open archive PR.
