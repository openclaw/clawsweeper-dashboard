## Summary

Keep open. Current main still lacks a per-row Sessions action affordance; the existing bulk delete path can support a narrow single-row Delete repair, while Archive needs separate Gateway/session API design.

## What This Changes

Add a per-row delete/archive or actions affordance to the Control UI Sessions table for quick single-session cleanup.

## Best Possible Solution

Add a compact per-row Delete or Actions affordance that delegates to the existing deletion controller for one key, and track first-class Archive separately as Gateway/session API work.

## Reproduction Assessment

Yes. This is a feature gap, and current-main source inspection shows the Sessions view only has checkbox selection plus a bulk Delete bar while the deletion controller already accepts a one-key array.

## Solution Assessment

Yes for the narrow delete affordance: reusing deleteSessionsAndRefresh(state, [key]) preserves the existing confirmation, loading, refresh, and error behavior. No for Archive in the same repair, because archived-session state and restore semantics do not exist yet.
