## Summary

Keep open. Current main now has a `Show archived` Sessions filter, but it still has no per-row action affordance; the delete-only slice is narrow, while manual Archive remains a Gateway/session API and UX decision.

## What This Changes

Add per-row Control UI Sessions actions so one session can be deleted, and possibly archived, without using the selected-rows bulk bar.

## Best Possible Solution

Add a compact per-row Delete affordance that delegates to the existing deletion controller, and keep manual Archive as a separate session/API design discussion.

## Reproduction Assessment

Yes for the current-state claim. Source inspection of current main shows checkbox selection plus a selected-rows bulk Delete bar, but no row-level action callback, table column, or button.

## Solution Assessment

Yes for a narrow Delete-only repair: reusing `deleteSessionsAndRefresh(state, [key])` is the smallest maintainable path. No for adding Archive in the same repair, because current schemas do not define an archive mutation or restore/read policy.
