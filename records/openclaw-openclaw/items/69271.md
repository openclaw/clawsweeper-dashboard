## Summary

Keep open. Current main still has no per-row Sessions action column or single-row Delete/Archive callback; only the selected-row bulk Delete flow exists, while manual Archive/Unarchive remains an open product/API decision with an unmerged PR and a related retention-policy issue.

## What This Changes

Request adds per-row delete/archive controls, or an actions dropdown, to the Control UI Sessions table.

## Best Possible Solution

Split the narrow per-row Delete UX from the broader Archive/Unarchive contract, then land archive behavior through a reviewed API and retention design coordinated with https://github.com/openclaw/openclaw/pull/47090 and https://github.com/openclaw/openclaw/issues/56609.

## Reproduction Assessment

Yes for the current-state claim: source inspection on current main shows only checkbox selection plus a bulk Delete bar, with no per-row action column or callback. The Archive part is a feature/product request rather than a broken existing contract.

## Solution Assessment

No for implementing the whole request as one small fix. Per-row Delete is narrow, but Archive/Unarchive needs a settled API, retention behavior, and coordination with the existing open archive PR and retention-policy issue.
