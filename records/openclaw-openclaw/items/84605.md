## Summary

Current main has the generated-media completion handoff that the report needs: image-generation completions carry media attachments into the internal event, channel/group completions are forced through message-tool delivery, and delivery is rejected when the completion agent does not send the expected media. The fix is current-main-only and has not shipped in a tagged release after v2026.5.18 yet.

## What This Changes

The issue reports that background `image_generate` completion events contain an image attachment path but fail to deliver that attachment back to the originating Discord channel.

## Best Possible Solution

Keep the current main generated-media message-tool completion path and ship it in the next release; new reports should include gateway logs if Discord still drops attachments on a main build.

## Reproduction Assessment

No live Discord reproduction was run. The release-to-main source diff and current regression tests give a high-confidence source path for the fixed generated-media completion handoff.

## Solution Assessment

Yes. The maintainable fix is at the completion handoff boundary: preserve generated media on internal events, require message-tool delivery for channel/group media completions, and fail visibly when media delivery evidence is missing.
