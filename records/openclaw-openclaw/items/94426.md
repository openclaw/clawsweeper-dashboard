## Summary

Keep open: current main and the latest release still use the hostless `isPortBusy` probe on the `gateway --force` path, and open linked pull requests already target the exact fix.

## What This Changes

The issue reports that `gateway --force` can miss an IPv4-only listener because `isPortBusy` probes the port with a hostless listen.

## Best Possible Solution

Review and land one focused linked PR, preferably https://github.com/openclaw/openclaw/pull/94949 if maintainers accept its conservative `unknown` handling, then close this issue after the merge.

## Reproduction Assessment

Yes. Source inspection shows current `isPortBusy` still performs a hostless bind, and `forceFreePortAndWait` exits early when that probe reports not busy; I did not run a live dual-stack socket repro in this read-only sweep.

## Solution Assessment

Yes. Reusing the shared multi-endpoint `checkPortInUse` helper is the narrow maintainable fix, and the remaining decision is which linked implementation to land.
