## Summary

Keep open. Current main still matches the reported runtime gap: subagent announce delivery reaches the Gateway agent path with channel/session context but no provider/model override, and shared agent ingress model selection still never consults `channels.modelByChannel`.

## What This Changes

Announce delivery should honor `channels.modelByChannel` for the parent/requester channel instead of falling back to the agent default model.

## Best Possible Solution

Wire `resolveChannelModelOverride` into shared Gateway agent ingress model selection for channel-backed sessions, with focused regression coverage for direct subagent announce and override precedence.

## Reproduction Assessment

Yes. Static current-main inspection gives a high-confidence reproduction path: direct subagent announce enters `agentCommandFromIngress` without provider/model, and that path never checks `channels.modelByChannel` before model fallback.

## Solution Assessment

Yes. Fixing shared ingress model selection is the narrow maintainable solution because it covers announce delivery consistently without adding an announce-only model injection path that could drift from reply/status behavior.
