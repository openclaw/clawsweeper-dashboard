## Summary

Keep open: current main still has the reported model-routing gap. Direct subagent announce delivery delegates model selection to shared agent ingress without provider/model overrides, and that ingress path resolves the agent default, stored overrides, and explicit run overrides but does not apply the existing `channels.modelByChannel` resolver that normal reply and status paths use.

## What This Changes

The issue requests that subagent announce delivery use the parent session channel's `channels.modelByChannel` model pin instead of always starting from the agent default model.

## Best Possible Solution

Wire the existing channel model override resolver into shared agent ingress for channel-backed sessions, with focused regression coverage for direct subagent announce and override precedence.

## Reproduction Assessment

Yes, at source level. Direct subagent announce sends no provider/model override, and current shared agent ingress never calls `resolveChannelModelOverride` before selecting the model.

## Solution Assessment

Yes. The maintainable fix is to reuse the existing `channels.modelByChannel` resolver in shared agent ingress rather than adding an announce-only model workaround.
