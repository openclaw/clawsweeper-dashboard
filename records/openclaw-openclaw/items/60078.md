## Summary

Keep open: current main still has the reported model-selection gap. Direct subagent announce delivery enters shared agent ingress without a provider/model override, and that ingress path still does not apply the existing `channels.modelByChannel` resolver before running model fallback.

## What This Changes

The issue requests that subagent announce delivery honor `channels.modelByChannel` for the parent session channel instead of always starting from the agent default model.

## Best Possible Solution

Wire the existing channel model override resolver into shared agent ingress model selection for channel-backed sessions, with regression coverage for direct subagent announce and override precedence.

## Reproduction Assessment

Yes. Source inspection on current main shows direct subagent announce reaches Gateway agent ingress without provider/model overrides, and the ingress model-selection path never imports or calls `resolveChannelModelOverride`.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse the existing channel override resolver in shared agent ingress rather than adding an announce-only workaround.
