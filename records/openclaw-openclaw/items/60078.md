## Summary

Keep open. Current main still matches the reported runtime gap: subagent announce delivery forwards channel/session routing to Gateway, but the shared agent ingress model selection path only applies the agent default, stored session overrides, and explicit run overrides; it never consults `channels.modelByChannel`.

## What This Changes

Subagent announce delivery should select the parent channel's `channels.modelByChannel` model instead of starting from the agent default model.

## Best Possible Solution

Wire `resolveChannelModelOverride` into shared agent ingress model selection for channel-backed sessions, with regression coverage for announce delivery and override precedence.

## Reproduction Assessment

Yes. Static current-main inspection gives a high-confidence reproduction path: direct announce enters Gateway without provider/model, and agent ingress never reads `channels.modelByChannel` before starting model fallback.

## Solution Assessment

Yes. Fixing shared agent ingress selection is the narrow maintainable path because announce delivery already supplies the needed channel/session context and the resolver contract already exists.
