## Summary

Keep open: current main and the latest release still have the reported model-selection gap for direct subagent announce delivery. The existing `channels.modelByChannel` resolver is used by normal reply/status paths, but the announce path enters shared agent ingress without provider/model overrides and that ingress path still falls back to the agent default before stored or explicit overrides.

## What This Changes

The issue requests that subagent announce delivery honor `channels.modelByChannel` for the parent session's channel instead of always starting from `agents.defaults.model.primary`.

## Best Possible Solution

Wire the existing channel model override resolver into shared agent ingress for channel-backed sessions, with regression tests for direct subagent announce and override precedence.

## Reproduction Assessment

Yes, at source level. Direct subagent announce sends no provider/model override, and current shared agent ingress never calls `resolveChannelModelOverride` before selecting the model.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse the existing `channels.modelByChannel` resolver in shared agent ingress rather than adding an announce-only model workaround.
