## Summary

Keep open. Current main still has the reported runtime gap: subagent announce delivery delegates model choice to agent ingress, and agent ingress still resolves default/stored/explicit models without consulting `channels.modelByChannel`.

## What This Changes

The issue asks for subagent announce delivery to honor the parent session channel's `channels.modelByChannel` pin instead of starting from the agent default model.

## Best Possible Solution

Apply the channel model override in shared agent ingress selection for channel-backed sessions, with regression coverage for direct announce delivery and override precedence.

## Reproduction Assessment

Yes. Static current-main inspection gives a high-confidence reproduction path: a channel-pinned subagent announce enters Gateway/agent ingress without provider/model, and that ingress path never checks `channels.modelByChannel`.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse `resolveChannelModelOverride` in shared agent ingress selection rather than adding an announce-only model injection path that could drift from reply and status behavior.
