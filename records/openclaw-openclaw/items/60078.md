## Summary

Keep open: current main and the latest release still have the reported model-routing gap. Direct subagent announce delivery hands the run to shared agent ingress without a provider/model override, and that ingress path selects defaults, stored overrides, and explicit overrides without applying the existing `channels.modelByChannel` resolver used by normal replies and status output.

## What This Changes

The issue asks for subagent announce delivery to honor `channels.modelByChannel` for the parent session channel instead of always starting from the agent default model.

## Best Possible Solution

Reuse the existing `channels.modelByChannel` resolver in shared agent ingress for channel-backed sessions, preserving the current override precedence and adding focused regression coverage for subagent announce delivery.

## Reproduction Assessment

Yes, at source level. Direct subagent announce sends no provider/model override, and current shared agent ingress never calls `resolveChannelModelOverride` before selecting the model.

## Solution Assessment

Yes. The maintainable fix is to wire the existing channel model override resolver into shared agent ingress rather than adding an announce-only workaround.
