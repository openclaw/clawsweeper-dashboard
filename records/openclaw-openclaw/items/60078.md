## Summary

Keep open: current main still has the reported model-selection gap in the subagent announce path, and the existing channel-override contract is not wired into shared agent ingress execution.

## What This Changes

Subagent announce delivery should honor `channels.modelByChannel` for the parent channel instead of starting from the agent default model.

## Best Possible Solution

Wire `resolveChannelModelOverride` into shared agent ingress model selection for channel-backed sessions, with regression coverage for direct subagent announce and override precedence.

## Reproduction Assessment

Yes. Static current-main inspection gives a high-confidence reproduction path: direct subagent announce enters Gateway agent ingress without a provider/model override, and that model-selection path never checks `channels.modelByChannel`.

## Solution Assessment

Yes. Fixing shared agent ingress model selection is the narrow maintainable path because it covers announce delivery without adding an announce-only workaround that could drift from reply and status behavior.
