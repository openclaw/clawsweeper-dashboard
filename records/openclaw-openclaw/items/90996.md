## Summary

Keep this PR open, but do not merge it as-is: the narrow configured-agent bootstrap direction is useful, but the label fallback currently masks resolver errors such as ambiguous labels and can silently send to the configured agent main session instead.

## What This Changes

The PR adds configured-agent bare-id fallback for `sessions_send` `sessionKey` and `label`, creates missing configured-agent main sessions, updates docs/model-facing copy, and adds focused regression tests.

## Best Possible Solution

Keep the narrow configured-agent bootstrap, but preserve resolver precedence: existing key/label matches and ambiguity errors should win, and bare-agent fallback should run only after a confirmed no-match with focused regression coverage.
