The PR changes Matrix helper request URL construction so bare Matrix endpoints get a private client API prefix and relative URL resolution preserves homeserver path prefixes.

## Best Possible Solution

Land the narrowed transport-layer fix after adding a focused transport.test.ts regression for path-prefixed homeserver URLs, and keep public apiPrefix config out unless a Matrix owner reopens that product surface.

## Solution Assessment
