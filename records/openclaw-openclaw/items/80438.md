## Summary

Keep open: the admission-time guard is the right boundary and the latest head preserves provider-prefixed `last` routes, but it still rejects disabling an existing ambiguous cron job and the external real-behavior proof gate remains mock-only.

## What This Changes

The PR adds gateway cron validation to reject ambiguous cron announce delivery in multi-channel configurations while preserving deterministic routes and adding focused validation tests.

## Best Possible Solution

Keep the admission-time guard, skip announce-route validation when an update leaves the job disabled, add that regression, and require real configured multi-channel Gateway proof before merge.

## Reproduction Assessment

Yes, by source inspection. Current main resolves omitted isolated/current/session `agentTurn` delivery to announce `last`, and PR head source shows `{ enabled: false }` updates still pass through the new route validation.

## Solution Assessment

No. Admission-time validation is the right maintainable boundary, but the implementation should block creation or activation while still allowing operators to disable invalid legacy jobs.

## Review Findings
