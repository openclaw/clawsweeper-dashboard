## Summary

Keep open: the PR targets a real source-reproducible cron validation gap, but the current head still introduces two blocking validation regressions and the after-fix proof is limited to test/lint output rather than a real configured multi-channel Gateway cron run.

## What This Changes

The PR adds gateway cron admission validation and tests for ambiguous implicit `last` announce delivery, plus a generated Swift Gateway protocol model refresh.

## Best Possible Solution

Keep the admission-time guard, but make it mirror cron delivery/failure-plan resolution exactly, skip disabled outcomes, and require redacted multi-channel Gateway cron proof before merge.

## Reproduction Assessment

Yes. Source inspection shows omitted isolated/current/session `agentTurn` delivery resolves to announce `last`, then multi-channel selection throws when multiple configured channels exist without an explicit or fallback route; I did not run a live Gateway in this read-only review.

## Solution Assessment

No. Admission-time validation is the right boundary, but this implementation diverges from delivery-plan semantics and validates disabled jobs that cannot run.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Preserve prefixed targets with last channel:** `src/gateway/server-methods/cron.ts:77-78`
  - body: This rejects any primary or failure-destination `channel: "last"` before honoring a provider-prefixed `to`. Current delivery planning resolves the prefixed target before falling back to `last`, so routes like `channel: "last", to: "telegram:123"` are deterministic and should remain accepted.
  - confidence: 0.9
- **[P2] Allow disabling ambiguous legacy jobs:** `src/gateway/server-methods/cron.ts:240-245`
  - body: Because `enabled` is part of the validation trigger, `cron.update` with `{ enabled: false }` can still validate the resulting ambiguous delivery and reject the patch. Disabled jobs get no `nextRunAtMs` and are skipped by scheduling, so this blocks the safest remediation path for existing ambiguous jobs.
  - confidence: 0.88
