## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Include command payloads in implicit delivery detection:** `src/gateway/server-methods/cron.ts:175`
  - body: `isIsolatedLikeAgentTurn` only matches `agentTurn`, but current delivery planning treats isolated/current/session `command` jobs as detached output jobs that default to announce/last when delivery is omitted. A `cron.update` that changes a job to `sessionTarget: "isolated"` plus `payload.kind: "command"` without `patch.delivery` can still be admitted in a multi-channel config and fail later during delivery resolution; include command jobs in this guard and add the update regression.
  - confidence: 0.9
- **[P2] Start a process group before signaling one:** `scripts/lib/docker-e2e-logs.sh:58`
  - body: The helper still launches the command with `"$@" &`, and non-interactive bash background children do not normally get a PGID equal to `$!`; `kill -- -$command_pid` therefore usually targets no process group and falls back to the old single-pid kill. To fix the claimed Docker cleanup race, start the command in its own group or remove this unproven helper change from the cron PR.
  - confidence: 0.86
