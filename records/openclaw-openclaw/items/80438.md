## Summary

Keep open: the admission-time guard is useful, but the current branch still rejects deterministic provider-prefixed `last` routes and blocks disabling existing ambiguous jobs; the supplied proof is also test-only rather than a real configured Gateway cron run.

## What This Changes

The branch adds gateway cron admission validation and tests for ambiguous announce delivery, plus a generated shared Swift Gateway model refresh.

## Best Possible Solution

Keep an admission guard, but align it exactly with delivery-plan resolution, skip disabled non-schedulable outcomes, and require real multi-channel Gateway proof before merge.

## Reproduction Assessment

Yes. Source inspection on current main shows omitted isolated/current/session `agentTurn` delivery resolves to announce `last`, and multi-channel selection throws when no explicit or fallback channel exists.

## Solution Assessment

No. Admission-time validation is the right boundary, but this implementation diverges from delivery-plan semantics for provider-prefixed `last` routes and validates disabled outcomes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve provider-prefixed last routes:** `src/gateway/server-methods/cron.ts:77-78`
  - body: This rejects `delivery.channel: "last"` before considering `delivery.to`. Current delivery planning resolves a provider-prefixed target before falling back to `last`, so `channel: "last", to: "telegram:123"` and the failure-destination equivalent remain deterministic and should not be rejected.
  - confidence: 0.9
- **[P2] Skip the guard after disabling a job:** `src/gateway/server-methods/cron.ts:244-245`
  - body: Because `enabled` triggers validation unconditionally, `cron.update` with `{ enabled: false }` can reject an existing ambiguous delivery after applying the disabling patch. Disabled jobs are not schedulable, so this blocks the safest cleanup path for legacy jobs; skip the announce-route guard when the resulting job is disabled.
  - confidence: 0.87
