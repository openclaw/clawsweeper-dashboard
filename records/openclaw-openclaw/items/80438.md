## Summary

Keep open: the PR targets a real current-main cron delivery gap, but the latest head still blocks disabling legacy ambiguous jobs and the supplied proof is only focused tests/lint rather than a configured Gateway add/update run.

## What This Changes

The branch modifies gateway cron server-method validation and cron validation tests to reject implicit or unprefixed `last` announce routing in multi-channel cron add/update flows while accepting deterministic routes.

## Best Possible Solution

Keep the admission guard, allow remediation by disabling post-patch disabled jobs, preserve deterministic prefixed routes, and land only after redacted configured multi-channel Gateway proof.

## Reproduction Assessment

Yes, source-level. Current main resolves omitted isolated/current/session agent-turn delivery to implicit `last`, and the PR's tests model the admission guard, but I did not execute tests in this read-only review.

## Solution Assessment

No as written. Admission-time validation is the right seam, but the update guard must allow disabling legacy ambiguous jobs and the PR still needs real configured-Gateway behavior proof.
