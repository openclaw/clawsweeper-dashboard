## Summary

Keep open: the PR targets a source-reproducible cron delivery bug, but the current patch introduces two blocking semantic regressions and the supplied proof is still formatter/lint/Vitest output rather than a configured multi-channel Gateway cron run.

## What This Changes

The PR adds gateway cron add/update validation and tests to reject ambiguous implicit `last` announce delivery in multi-channel cron jobs while preserving selected deterministic routes.

## Best Possible Solution

Keep the admission-time guard, but mirror `resolveCronDeliveryPlan` and failure-destination semantics exactly, preserve deterministic provider-prefixed routes, and allow disabling ambiguous legacy jobs.

## Reproduction Assessment

Yes, by source inspection. Omitted delivery for isolated/current/session `agentTurn` jobs resolves to announce `last`, and multi-channel target selection can fail without an explicit or provider-prefixed route.

## Solution Assessment

No. Admission-time validation is the right boundary, but this implementation must preserve provider-prefixed `last` routes and skip delivery validation once an update disables the resulting job.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve prefixed targets when channel is last:** `src/gateway/server-methods/cron.ts:79-82`
  - body: Current delivery planning resolves a provider-prefixed `delivery.to` before falling back to `last`, and existing tests cover `channel: "last", to: "telegram:123"` as deterministic. This guard throws on any literal `last` channel, so accepted primary and failure-destination routes become invalid.
  - confidence: 0.93
- **[P2] Allow disabling invalid legacy cron jobs:** `src/gateway/server-methods/cron.ts:242-247`
  - body: Including `enabled` in `shouldValidateDeliveryOutcome` validates the patched job even when the patch sets `enabled: false`. Disabled cron jobs are unschedulable, so users should be able to disable an ambiguous legacy job instead of being blocked by the new admission guard.
  - confidence: 0.9
