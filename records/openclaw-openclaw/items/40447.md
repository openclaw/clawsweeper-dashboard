## Summary

Keep open: the README flag fix is already present on current main, but the provider-link change is not safe to merge as-is because it presents a fork as the canonical source while the npm package metadata still points elsewhere and the proposed Issues target is disabled.

## What This Changes

This PR changes the README quick-start `openclaw message send` example from `--to` to `--target` and changes the Claude Max API Proxy provider guide GitHub and Issues links from `atalovesyou` to `wende`.

## Best Possible Solution

Land or replace this with a docs-only update that keeps the already-correct README `--target` example and makes the provider guide internally consistent: either align the install command, source repository, package metadata, and support channel, or explicitly document maintained forks as community alternatives with provenance and a working support path.

## Reproduction Assessment

Yes. Static inspection is enough: current main already has the README `--target` example and CLI helper contract, while the provider guide still has old links; npm and GitHub metadata reproduce the PR's source/support mismatch.

## Solution Assessment

No. The README edit is correct but superseded on main, and the provider-link edit is not the narrowest maintainable fix because it changes the apparent canonical source without aligning package metadata or pointing to an enabled support tracker.
