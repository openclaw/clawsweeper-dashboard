## Summary

Keep open: the README part is already fixed on current main, but the provider-link change still needs maintainer judgment because it makes a different fork look canonical while the npm package metadata points elsewhere and the proposed Issues target is disabled.

## What This Changes

This PR updates the README `openclaw message send` example to use `--target` and changes the Claude Max API Proxy provider guide GitHub and Issues links from `atalovesyou` to `wende`.

## Best Possible Solution

Land or replace this with a docs-only update that keeps the already-correct README `--target` example and makes the provider guide internally consistent: either align install, source, package metadata, and support channel, or document maintained forks as community alternatives with clear provenance.

## Reproduction Assessment

Yes. Static inspection is enough: current main already has the README `--target` example and CLI helper contract, while npm and GitHub API checks reproduce the provider source/support mismatch in the PR.

## Solution Assessment

No. The README edit is correct but superseded on main, and the provider-link edit is not the narrowest maintainable fix because it changes the apparent canonical source without aligning package metadata or pointing to an enabled support tracker.
