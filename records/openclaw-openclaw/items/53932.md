## Summary

Keep open: current main still routes Slack outbound uploads through the optimized outbound-media path without a Slack config key or sender option that can set `optimizeImages: false`. The related implementation attempt, #53988, is closed and unmerged, so it does not supersede the remaining work.

## What This Changes

Add a Slack option to disable outbound image optimization so uploaded images can preserve their original bytes and format.

## Best Possible Solution

Implement a bundled Slack config option that defaults to today’s optimized behavior, wire it through an additive outbound-media option so Slack can request original image bytes, and document the setting alongside `mediaMaxMb` with regression coverage for both optimized default and preserve-original behavior.

## Reproduction Assessment

Yes. The source-level reproduction path is `uploadFile` or `sendSlackMessage` with `mediaUrl` -> `uploadSlackFile` -> `loadOutboundMediaFromUrl` -> `loadWebMedia`, where current main provides no Slack opt-out and the loader defaults `optimizeImages` to true.

## Solution Assessment

Yes, the narrow maintainable fix is a Slack-owned opt-out backed by an additive `loadOutboundMediaFromUrl` option. A direct raw-upload bypass would be less desirable because it could skip shared outbound media policy unless carefully re-threaded.
