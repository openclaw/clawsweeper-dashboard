## Summary

Keep open: current main still lacks a Slack-owned upload optimization opt-out. The only related implementation attempt was closed unmerged and added an inert config stub, while Slack uploads still reach the shared media loader with image optimization enabled by default.

## What This Changes

The issue requests a Slack configuration option or upload behavior change that preserves original image bytes instead of always optimizing uploaded images.

## Best Possible Solution

Add a default-compatible Slack config option, expose/pass `optimizeImages` through the existing outbound-media facade, preserve current size/local-read/SSRF guards, and document/test optimized-default versus preserve-original uploads.

## Reproduction Assessment

Yes, at source level. On current main, Slack `mediaUrl` sends go through `uploadSlackFile` to `loadOutboundMediaFromUrl`, and that facade does not accept or forward `optimizeImages: false` while the shared loader defaults optimization to true.

## Solution Assessment

Yes. A Slack-owned config option threaded through the existing outbound-media loader is the narrow maintainable path; bypassing the loader would risk losing current media size, local-read, and SSRF protections.
