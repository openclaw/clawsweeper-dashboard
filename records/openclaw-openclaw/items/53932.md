## Summary

Keep open: current main still has no Slack-owned config or outbound-media option that lets Slack uploads preserve original image bytes. The linked implementation attempt is closed and unmerged, while the merged upload-file action still routes through the same optimized media loader.

## What This Changes

The issue asks for a Slack upload setting that can opt out of image optimization so uploaded PNGs and other images are not degraded or converted unexpectedly.

## Best Possible Solution

Add a default-compatible Slack upload optimization config option, expose an additive outbound-media loader option for plugins, thread it through the Slack send/upload action path, and document it alongside Slack media upload limits with regression tests for both default optimized uploads and preserve-original uploads.

## Reproduction Assessment

Yes. The source-level reproduction path is Slack `uploadFile` or `sendSlackMessage` with `mediaUrl`, which reaches `uploadSlackFile`, then `loadOutboundMediaFromUrl`, then `loadWebMedia` where image optimization defaults to true and no Slack option can set it false.

## Solution Assessment

Yes. A Slack-owned opt-out plus an additive plugin SDK outbound-media option is the narrow maintainable fix; bypassing the shared media loader would risk skipping existing access, size, and SSRF policy checks.
