## Summary

Keep open: current main and v2026.6.6 partially preserve acceptable under-cap PNG/JPEG/WebP images, but Slack sends and explicit upload-file still route through the shared media loader with optimization enabled and no Slack/tool option for raw image delivery. The linked opt-out and PNG-preservation PRs are both closed and unmerged.

## What This Changes

The issue asks for Slack uploads to support lossless/original image delivery by disabling or opting out of image optimization.

## Best Possible Solution

Choose a maintainer-approved Slack/media contract, then add a channel-owned or shared preserve-original path that keeps caps/security checks and covers both normal media sends and explicit upload-file.

## Reproduction Assessment

Yes, at source level. Slack media sends and explicit upload-file converge on the shared outbound media loader, while no Slack-facing option disables optimization; I did not run a live Slack upload in this read-only review.

## Solution Assessment

Unclear until maintainers choose the public behavior boundary. A Slack opt-out is plausible, but a shared preserve-original option or explicit raw-delivery mode may be the more maintainable contract.
