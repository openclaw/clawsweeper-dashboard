## Summary

Keep open: current main still lacks a Slack-owned upload optimization opt-out, and Slack uploads still flow through the shared media loader with image optimization enabled by default. The linked implementation attempt was closed unmerged and only added an unused config stub, so no shipped or current-main fix covers the request.

## What This Changes

The issue requests a Slack option to preserve original image uploads instead of always using the optimized image path.

## Best Possible Solution

Add a default-compatible Slack config option, thread it through Slack upload into the existing outbound-media loader as `optimizeImages: false`, keep size/local-read/SSRF guards, and document/test both default optimized uploads and preserve-original uploads.

## Reproduction Assessment

Yes, at source level. Slack `sendSlackMessage` with `mediaUrl` reaches `uploadSlackFile`, then `loadOutboundMediaFromUrl`, then `loadWebMedia`, where optimization defaults to true and Slack has no current option to pass false.

## Solution Assessment

Yes. A Slack-owned opt-out threaded through the shared outbound-media loader is the narrow maintainable path; bypassing the loader would risk skipping existing size, local-read, and SSRF policy checks.
