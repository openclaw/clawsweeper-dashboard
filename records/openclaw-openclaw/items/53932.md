## Summary

Keep open: current main and v2026.6.8 still route Slack file uploads through the shared optimized media loader by default, with only partial original-image preservation and no Slack upload optimization opt-out. The remaining decision is what Slack file-fidelity policy maintainers want.

## What This Changes

Request to let Slack uploads bypass image optimization so original image files can be delivered without lossy conversion.

## Best Possible Solution

Choose a maintainer-approved Slack upload fidelity policy, then implement the narrow path in the Slack outbound/media boundary with tests for raw delivery, optimized fallback, and config or docs alignment.

## Reproduction Assessment

Yes, at source level: Slack uploads call loadOutboundMediaFromUrl, which calls optimizing loadWebMedia by default. I did not run a live Slack upload, so the exact reported artifact is not runtime-proven here.

## Solution Assessment

Unclear. A Slack config opt-out is plausible, but maintainers should first choose between raw upload by default, an explicit setting, message-tool steering, or a shared media-delivery API.
