## Summary

Keep open: current main and the latest release still do not expose or pass a Slack-owned image optimization opt-out. The related implementation attempt was closed unmerged and only added an inert stub, while Slack uploads still call the shared media loader on its default optimizing path.

## What This Changes

The issue requests a Slack configuration option to preserve original uploaded images instead of always using the shared image optimization path.

## Best Possible Solution

Add a default-compatible Slack config option, thread it through account resolution and the existing outbound-media facade, and preserve current size, local-read, and SSRF guardrails while documenting and testing both default optimized uploads and preserve-original uploads.

## Reproduction Assessment

Yes. At source level, Slack `mediaUrl` and `uploadFile` sends flow through `sendSlackMessage` and `uploadSlackFile` to `loadOutboundMediaFromUrl`, while current config and facade surfaces provide no way to pass `optimizeImages: false`.

## Solution Assessment

Yes. A Slack-owned config option threaded through the existing outbound-media loader is the narrow maintainable path; bypassing that loader would risk losing existing media size, local-read, and SSRF protections.
