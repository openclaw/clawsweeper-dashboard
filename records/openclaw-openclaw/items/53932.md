- `impact:data-loss`: The reported behavior can lose image format and fidelity when a recipient expects the original generated file.

## Summary

Keep open: current main and the latest release still route Slack uploads through the shared outbound media loader with image optimization enabled, and there is no Slack config or tool option to preserve original image bytes. The linked opt-out and PNG-preservation PRs are both closed and unmerged, so the remaining work needs a maintainer product decision rather than cleanup closure.

## What This Changes

Request to let Slack uploads preserve original image files or opt out of automatic image optimization so generated PNGs are not degraded to JPEG.

## Best Possible Solution

Choose the Slack/media contract, then implement preserve-original behavior through a channel-owned or shared media option with schema, docs, and focused coverage for normal media sends plus explicit `upload-file`.

## Reproduction Assessment

Yes, at source level. Slack media sends and explicit `upload-file` converge on `uploadSlackFile`, which calls the shared outbound-media loader while `loadWebMediaInternal` defaults image optimization on; I did not run a live Slack upload in this read-only review.

## Solution Assessment

Unclear until maintainers choose the public behavior boundary. A Slack opt-out is plausible, but a shared preserve-original option or narrower PNG-preservation behavior may be the more maintainable contract.
