## Summary

Keep open: current main still has no Slack opt-out or preserve-original path for uploaded images, and explicit Slack `upload-file` still flows through the shared media loader where image optimization defaults on. The closed related PR was inert, while the open shared-loader PR may address only part of the symptom and is not landed.

## What This Changes

The issue requests a way to prevent Slack image uploads from being optimized into degraded or format-changed files.

## Best Possible Solution

Choose the Slack/media contract, then implement preserve-original behavior through the shared media options with Slack config/schema/docs and focused coverage for normal sends plus explicit `upload-file`.

## Reproduction Assessment

Yes, at source level: Slack `sendMessage` and explicit `upload-file` both reach `uploadSlackFile`, which calls the shared outbound-media loader without an opt-out while `loadWebMediaInternal` defaults optimization on. I did not run a live Slack upload because this is a read-only review.

## Solution Assessment

Unclear until maintainers choose the public behavior boundary. A Slack-owned config option is plausible, but the open shared-loader PR shows that preserving under-cap PNGs globally may solve part of the symptom without adding a Slack-only setting.
