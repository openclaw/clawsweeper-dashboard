## Summary

Keep open: current main and v2026.5.20 still send Slack uploads through the shared image-optimizing media path without a Slack opt-out, and the related open PR is only a partial shared-media approach while the Slack-specific PR was an unmerged stub.

## What This Changes

Request to add a way to preserve original Slack-uploaded images instead of always applying outbound image optimization.

## Best Possible Solution

Choose the product boundary, then implement preservation through the existing media loader contract with Slack config/schema/docs and focused coverage for both normal Slack media sends and explicit upload-file behavior.

## Reproduction Assessment

Yes, at source level: Slack send and upload-file both reach uploadSlackFile, which calls loadOutboundMediaFromUrl without any optimization opt-out. I did not run a live Slack upload in this read-only review.

## Solution Assessment

Unclear until maintainers choose the public behavior boundary. A Slack-owned config option is plausible, but the related shared-media PR shows that preserving under-cap PNGs globally may solve part of the symptom without a Slack-only setting.
