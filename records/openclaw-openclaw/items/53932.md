## Summary

Keep open: current `main` and v2026.5.19 still route Slack file uploads through the shared media loader with image optimization enabled by default, and neither Slack config nor the plugin SDK outbound-media facade exposes an opt-out. The closed related PR at https://github.com/openclaw/openclaw/pull/53988 was an unmerged stub, while https://github.com/openclaw/openclaw/pull/40829 is an open partial shared-loader approach rather than a landed Slack option.

## What This Changes

The issue asks for Slack uploads to preserve original image files, or to offer a way to disable image optimization for Slack media delivery.

## Best Possible Solution

Choose the product boundary, then implement it through the existing media loader contract with Slack config/schema/docs and focused coverage for both normal Slack sends and explicit `upload-file` behavior.

## Reproduction Assessment

Yes, at source level: Slack `sendMessage` and explicit `upload-file` both reach `uploadSlackFile`, which calls the shared outbound-media loader without any path to set `optimizeImages: false`. I did not run a live Slack upload because this review is read-only.

## Solution Assessment

Unclear until maintainers choose the public behavior boundary. A Slack-owned config option is plausible, but the related shared-loader PR shows that preserving under-cap PNGs globally may solve part of the symptom without adding a Slack-only setting.
