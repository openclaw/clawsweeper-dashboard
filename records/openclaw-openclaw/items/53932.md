## Summary

Keep open: current main still lacks a Slack-owned upload optimization opt-out, and the only linked implementation attempt was closed unmerged with no runtime wiring. The remaining work is narrow and source-proven, but it is a feature/config addition rather than a broken documented contract.

## What This Changes

The issue requests a Slack-specific way to preserve original uploaded image bytes instead of always using the shared image optimization path.

## Best Possible Solution

Add a default-compatible Slack upload optimization config option, expose an additive plugin SDK outbound-media option, and cover default optimized uploads plus preserve-original uploads with docs and regression tests.

## Reproduction Assessment

Yes. The source-level path is Slack `uploadFile` or `sendSlackMessage` with `mediaUrl`, which reaches `uploadSlackFile`, then `loadOutboundMediaFromUrl`, then `loadWebMedia` where image optimization defaults to true and Slack cannot set it false.

## Solution Assessment

Yes. A Slack-owned opt-out threaded through the shared outbound-media loader is the narrow maintainable approach; bypassing the loader would risk skipping existing size, access, and SSRF policy checks.
