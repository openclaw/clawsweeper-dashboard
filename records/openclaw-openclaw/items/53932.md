## Summary

Keep open. Current main still lacks a Slack config or send-path option to preserve original image bytes on outbound Slack uploads; the underlying media layer can skip optimization, but Slack does not expose or forward that control. The related implementation attempt, #53988, is closed and unmerged, so it does not supersede the remaining work.

## What This Changes

The issue requests an opt-out for Slack outbound image optimization so generated/uploaded PNGs can be shared without lossy conversion or degradation.

## Best Possible Solution

Implement a Slack-owned, documented config option that defaults to the current optimized behavior and lets users preserve original image bytes for outbound Slack uploads, with regression tests proving PNG bytes/MIME are preserved when disabled and optimized delivery remains the default.
