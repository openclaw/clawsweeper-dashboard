## Summary

Keep open: current main still routes Slack outbound media through the shared optimizing media loader unless the agent explicitly uses the message-tool upload path, so the requested Slack fidelity policy remains unresolved rather than implemented or fully covered by the broader forced-document issue.

## What This Changes

The issue asks for a way to disable or bypass image optimization for media uploaded to Slack so original PNG/image files are delivered without lossy conversion.

## Best Possible Solution

Maintainers should choose one Slack media-fidelity contract, then implement it at the Slack outbound/message-tool boundary with tests for raw delivery, optimized fallback, and config or docs behavior.

## Reproduction Assessment

Yes, at source level: Slack uploads call loadOutboundMediaFromUrl, which uses the optimizing loadWebMedia path by default. I did not run a live Slack upload, so the exact reported artifact is not runtime-proven here.

## Solution Assessment

Unclear. A Slack opt-out or raw-upload default is plausible, but maintainers should first choose between enforcing message-tool upload, default raw delivery, an explicit channel setting, or a shared media-delivery contract.
