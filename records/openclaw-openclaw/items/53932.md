- `impact:data-loss`: The reported behavior can lose original image format and pixel fidelity when users expect generated media to transfer unchanged.

## Summary

Keep open: current main and v2026.6.10 still route Slack file uploads through the shared optimizing media loader, and Slack exposes no raw-upload or image-optimization opt-out; the remaining work needs a maintainer decision on the public Slack media-fidelity contract.

## What This Changes

Request to let Slack uploads bypass image optimization so original image files, especially PNGs, can be delivered without lossy conversion.

## Best Possible Solution

Choose and document a Slack media-fidelity contract, then implement it at the Slack outbound/message-tool boundary with focused tests and live Slack upload proof.

## Reproduction Assessment

Yes, at source level: Slack uploads call loadOutboundMediaFromUrl, which uses the optimizing loadWebMedia path by default, and Slack config has no upload-fidelity opt-out. I did not run a live Slack upload in this read-only review.

## Solution Assessment

Unclear. A Slack opt-out is plausible, but maintainers should first choose whether the right contract is raw-upload default behavior, a new config option, enforced message-tool behavior, or a shared media-delivery policy.
