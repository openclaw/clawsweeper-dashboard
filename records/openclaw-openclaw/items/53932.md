## Summary

Keep open: current main still routes Slack media uploads through the shared optimized media loader, and the linked opt-out PR was closed unmerged with no runtime effect. The remaining decision is what Slack file-fidelity policy maintainers want, not whether this is already implemented.

## What This Changes

Requested a Slack upload mode or opt-out so images can be sent without unconditional image optimization.

## Best Possible Solution

Choose a maintainer-approved Slack file-fidelity policy, then implement the narrow path in the Slack upload/media boundary with tests for raw upload behavior, optimized fallback behavior, and docs/config alignment.

## Reproduction Assessment

Yes, at source level: Slack media sends flow through loadOutboundMediaFromUrl into loadWebMedia, which optimizes images by default. I did not run a live Slack upload, so the exact reported PNG-to-JPG result is not runtime-proven here.

## Solution Assessment

Unclear. A config opt-out is plausible, but current main also has upload-file and partial original-preservation behavior, so maintainers should first choose whether the best fix is raw upload by default, explicit upload-file steering, or an account/channel setting.
