## Summary

Keep open: the latest PR head still leaves a raw-secret persistence path in oversized detail summaries because retained non-string fields are copied unchanged.

## What This Changes

The PR adds persistence-boundary redaction for persisted `toolResult.details`, passes logging config into the guard, extends structured field-key redaction, and adds regression tests.

## Best Possible Solution

Keep the persistence-boundary redaction approach and make all retained oversized-summary, fallback, and child-session fields recursively redacted or reduced to safe scalar metadata before session JSONL writes.

## Reproduction Assessment

Yes, at source level. Current main preserves small details unchanged, and latest PR head still copies retained non-string fields in oversized summary and fallback paths, so a structured `status` value containing a token can persist raw when another field makes the details oversized.

## Solution Assessment

No, not yet. The persistence boundary is the right place to solve this, but the summary, fallback, and child-session retained fields need the same recursive redaction guarantee before merge.
