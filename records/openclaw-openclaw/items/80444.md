## Summary

Keep open: this is a useful security fix direction, but the current PR head still has a P1 persisted-secret leak in oversized detail summaries because retained non-string fields bypass recursive redaction.

## What This Changes

The branch redacts persisted tool-result details, threads logging redaction config into the session guard, extends structured redaction helpers, and adds regression tests.

## Best Possible Solution

Keep the persistence-boundary redaction approach, but make every retained oversized-summary, fallback, and child-session field pass through recursive redaction or a safe summary before JSONL writes.

## Reproduction Assessment

Yes, at source level. Build oversized details with a large aggregated field and a retained non-string field such as status: { token: "..." }; PR head copies that object into the persisted summary.

## Solution Assessment

No, not yet. The boundary-redaction design is the right owner boundary, but the branch must recursively redact or omit retained non-string summary, fallback, and child-session fields before merge.
