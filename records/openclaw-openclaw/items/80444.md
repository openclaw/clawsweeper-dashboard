## Summary

Keep open: the PR addresses a real persisted-details secret leak, but latest head still copies retained non-string fields in oversized and fallback summaries, leaving a concrete raw nested-secret persistence path.

## What This Changes

The PR adds persistence-boundary redaction for persisted `toolResult.details`, passes logging redaction config into the guard, extends field-key redaction, and adds regression tests.

## Best Possible Solution

Keep the persistence-boundary redaction approach, but recursively redact or safely reduce every retained oversized-summary, fallback, and child-session field before session JSONL writes.

## Reproduction Assessment

Yes, at source level. Construct an oversized `toolResult.details` with a retained non-string field such as `status: { token: "..." }` plus a large `aggregated` field; latest PR head copies that object unchanged into the persisted summary.

## Solution Assessment

No, not yet. The persistence boundary is the right place to solve this, but the current patch must apply the same recursive redaction guarantee to retained summary and fallback fields before it is safe to merge.
