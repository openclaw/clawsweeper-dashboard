## Summary

Keep open. The PR fixes the small-details persistence gap, but the latest head still has a source-reproducible security leak where oversized detail summaries copy retained non-string fields unchanged.

## What This Changes

The PR redacts persisted `toolResult.details`, redacts selected oversized summary strings and detail keys, broadens env-style field-key masking, and adds focused agent/redaction regression tests.

## Best Possible Solution

Keep the persistence-boundary redaction approach, but make every retained oversized-summary and fallback field either recursively redacted or reduced to safe scalar metadata before session JSONL writes.

## Reproduction Assessment

Yes, at source level. Current main preserves small details unchanged, and the latest PR head still copies non-string allowlisted fields unchanged in the oversized-details summary path.

## Solution Assessment

No, not yet. The persistence boundary is the right place to fix this, but the oversized-summary and fallback branches need the same redaction guarantee for retained structured values before merge.
