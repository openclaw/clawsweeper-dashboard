## Summary

Keep this PR open: the latest head fixes the prior truncation-before-redaction problem and improves current main, but the oversized-details summary path can still persist raw secrets from non-string allowlisted fields.

## What This Changes

The branch redacts persisted `toolResult.details`, redacts oversized summary strings and detail keys, broadens env-style field-key masking, and adds focused agent/redaction regression tests.

## Best Possible Solution

Keep the persistence-boundary redaction approach, but make every retained oversized-summary field either recursively redacted or reduced to safe scalar metadata before session JSONL writes.

## Reproduction Assessment

Yes, at source level. Current main preserves small details unchanged, and the PR head still copies non-string allowlisted fields unchanged in the oversized-details summary path.

## Solution Assessment

No, not yet. The overall boundary is the right place to fix this, but the oversized-summary branch needs the same redaction guarantees for retained structured values before merge.
