## Summary

Keep this PR open. The latest head still has a concrete security blocker: the oversized-details summary path redacts strings but copies retained non-string fields unchanged, so structured secret values can still be persisted in session JSONL.

## What This Changes

The PR redacts secret-like values in persisted `toolResult.details`, threads logging redaction config into the session guard, and adds regression coverage for small and oversized detail payloads.

## Best Possible Solution

Keep the persistence-boundary redaction approach, but recursively redact or safely reduce every retained oversized-summary, fallback, and child-session field before session JSONL writes.

## Reproduction Assessment

Yes, at source level. Construct an oversized `toolResult.details` with a retained non-string field such as `status: { token: "..." }` plus a large `aggregated` field; PR head copies that object unchanged into the persisted summary.

## Solution Assessment

No, not yet. The persistence boundary is the right place to solve this, but the current patch must apply the same recursive redaction guarantee to retained summary, fallback, and child-session fields before merge.
