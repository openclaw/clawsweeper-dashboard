## Summary

## What This Changes

The PR adds delayed deletion of resolved Telegram approval messages in a legacy Telegram exec approval handler and bundles several docs cleanup edits.

## Best Possible Solution

Implement resolved-message deletion in the active Telegram native approval runtime using the existing final-action/delete transport seam, cover approve and deny flows, and keep the docs cleanup separate and source-only.

## Reproduction Assessment

Yes, source-level: current main's Telegram native runtime returns `clear-actions` for resolved approvals and callback handling calls `clearCallbackButtons()`, so the approval message remains. I did not run a live Telegram approval flow in this read-only review.

## Solution Assessment

No. This PR edits a removed legacy handler; the maintainable fix belongs in `extensions/telegram/src/approval-handler.runtime.ts` or shared final-action plumbing with owned or unref'd delayed deletion and real Telegram proof.
