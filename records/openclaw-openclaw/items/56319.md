## What This Changes

This PR adds delayed deletion of resolved Telegram approval messages in the legacy exec approval handler and includes several unrelated documentation cleanups.

## Best Possible Solution

Land or replace this with a focused change in `extensions/telegram/src/approval-handler.runtime.ts` that clears actions immediately, deletes resolved approval messages with managed timer/error behavior, and covers approve and deny flows.

## Reproduction Assessment

Yes, at source level. Current main resolves Telegram approvals to `clear-actions`, and both the native runtime and callback path clear inline buttons/reply markup rather than deleting the approval message.

## Solution Assessment

No. Deleting resolved approval messages is the right end state, but this PR edits a removed legacy handler, leaves timer/error behavior unmanaged, and mixes unrelated docs churn into the Telegram fix.
