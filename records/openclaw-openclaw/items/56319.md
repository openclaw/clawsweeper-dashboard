## Summary

Keep open: the linked Telegram approval bug still appears valid, but this PR is not mergeable as-is because it patches a removed handler, leaves delayed deletion unmanaged, carries stale docs edits, and lacks real Telegram proof.

## What This Changes

The PR adds delayed deletion of resolved Telegram approval messages to the legacy exec approval handler and includes several documentation cleanup edits.

## Best Possible Solution

Implement resolved-message deletion in the active Telegram native approval runtime using the existing final-action/delete transport seam, cover approve and deny flows, and keep docs changes separate and source-only.

## Reproduction Assessment

Yes, at source level: current main returns `clear-actions` for resolved Telegram approvals and callback handling clears inline buttons rather than deleting the message. No live Telegram run was performed in this read-only review.

## Solution Assessment

No. The desired behavior belongs in `extensions/telegram/src/approval-handler.runtime.ts` or shared final-action plumbing, not the removed `exec-approvals-handler.ts`, and delayed deletion needs owned or unref'd timer handling.
