Keep open: this PR targets a real linked Telegram approval cleanup, but the runtime edit is aimed at a deleted legacy handler, the active Telegram approval runtime still clears buttons instead of deleting the message, and the external PR still lacks required live Telegram proof.

## What This Changes

The PR adds delayed Telegram approval-message deletion to a legacy exec-approval handler and bundles several unrelated documentation cleanups.

## Best Possible Solution

Replace this with a focused active-runtime change in `extensions/telegram/src/approval-handler.runtime.ts` that clears actions immediately, deletes resolved approval messages through the existing Telegram delete seam, and covers approve and deny flows.

## Reproduction Assessment

Yes, at source level. Current main resolves Telegram approvals to `clear-actions`, and both the native runtime and callback path clear inline buttons or reply markup instead of deleting the approval message.

## Solution Assessment

No. Deleting resolved approval messages is the right end state, but this PR edits a stale handler, leaves timer behavior unmanaged, mixes unrelated docs churn into the Telegram fix, and lacks live proof.
