## Summary

Keep open: the linked Telegram bug is still source-reproducible, but this PR is not mergeable because it targets an obsolete handler, lacks real Telegram proof, and still includes unrelated docs issues.

## What This Changes

This PR adds delayed deletion of resolved Telegram exec-approval messages in a legacy handler and bundles several documentation cleanups.

## Best Possible Solution

Replace this with a focused change in `extensions/telegram/src/approval-handler.runtime.ts` that clears actions immediately, deletes resolved approval messages through the active runtime seam, and covers approve and deny flows.

## Reproduction Assessment

Yes, at source level. Current main resolves Telegram approvals to `clear-actions`, and both the native runtime and callback path clear inline buttons or reply markup instead of deleting the approval message.

## Solution Assessment

No. Deleting resolved approval messages is the right end state, but this PR edits a stale handler, leaves timer behavior unmanaged, and mixes unrelated docs churn into the Telegram fix.
