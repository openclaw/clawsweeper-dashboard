## Summary

Keep open. The bug this PR targets is still valid on current main, but the branch is no longer mergeable as-is because it patches the removed legacy Telegram approval handler while current Telegram approvals run through the shared native approval runtime.

## What This Changes

The PR adds delayed deletion of resolved Telegram exec approval messages in the legacy handler and bundles several small documentation cleanups.

## Best Possible Solution

Land a narrow current-main Telegram runtime fix that clears the inline buttons immediately, deletes the resolved approval message after a short visible delay, manages or unrefs the delayed timer with observable error handling, adds focused regression coverage, and keeps unrelated documentation edits out of the runtime bug fix.

## Reproduction Assessment

Yes. Trigger a native Telegram exec approval and approve or deny it; current-main source shows the resolved path returns `clear-actions`, so it removes the inline buttons through `editMessageReplyMarkupTelegram` and leaves the approval message in chat.

## Solution Assessment

No. The requested behavior is the right product outcome, but this PR applies it to a removed legacy handler and mixes in unrelated docs changes; the maintainable fix belongs in `extensions/telegram/src/approval-handler.runtime.ts` using the current native approval runtime contract.
