## Summary

Keep open. Current main has mitigations for the startup-blocking pricing regression, including deferred pricing refresh, manifest reuse, and a documented pricing opt-out, but the shutdown/hung-process part is still source-backed: in-flight pricing fetches are not tied to Gateway stop, and refresh completion can still schedule a referenced 24-hour timer after stop.

## What This Changes

The issue asks OpenClaw to fix a Windows npm-global regression where `openclaw gateway` hangs after LiteLLM/OpenRouter pricing fetch timeouts and cannot be interrupted cleanly with Ctrl+C.

## Best Possible Solution

Keep the existing startup mitigations, then make model-pricing refresh shutdown-aware: pass a Gateway lifecycle abort signal into both catalog fetches, abort in-flight work on stop, avoid cache mutation or refresh scheduling after stop, unref any long background timer, and add focused lifecycle tests.

## Reproduction Assessment

Yes. The original report has concrete released-package steps, and current main has a focused source-level reproduction path: start pricing refresh with hanging fetches, call the stop callback during the in-flight refresh, then assert the fetches abort and no post-stop refresh timer remains referenced.

## Solution Assessment

No. Current main is only a partial solution; the narrow maintainable fix is to make the existing pricing-refresh lifecycle honor Gateway stop rather than relying only on deferred startup, longer timeouts, or operator opt-outs.

## Repair Work Prompt

Fix the remaining pricing-refresh shutdown hang for https://github.com/openclaw/openclaw/issues/72208. Current main already defers pricing refresh and documents `models.pricing.enabled: false`; keep those mitigations. Make `src/gateway/model-pricing-cache.ts` lifecycle-aware instead: create/pass a Gateway-owned abort signal through `startGatewayModelPricingRefresh`, `refreshGatewayModelPricingCache`, `fetchOpenRouterPricingCatalog`, and `fetchLiteLLMPricingCatalog`; abort in-flight fetches when the returned stop callback runs; prevent cache replacement and `scheduleRefresh` after stop; unref the 24-hour refresh timer if it remains; preserve existing pricing semantics and manifest-registry behavior. Add focused tests in `src/gateway/model-pricing-cache.test.ts` and, if needed, `src/gateway/server-runtime-services.test.ts` or `src/gateway/server-close.test.ts` proving stop during an in-flight refresh aborts the fetches and does not leave a post-stop timer. Add a single-line user-facing `CHANGELOG.md` fix entry crediting `Thanks @rzcq`; do not change provider pricing data or plugin loading policy beyond this lifecycle repair.
