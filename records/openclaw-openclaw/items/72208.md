## Summary

Keep open. Current main has startup mitigations for the pricing-fetch regression, but the shutdown/hung-process part is still source-backed: model-pricing fetches are not lifecycle-abortable, and refresh completion can still schedule a referenced long timer after Gateway stop.

## What This Changes

The issue asks to fix a Windows npm-global Gateway regression where OpenRouter and LiteLLM pricing fetch timeouts after 2026.4.24 can leave startup hung, the dashboard unreachable, and Ctrl+C ineffective.

## Best Possible Solution

Keep the startup mitigations, then make model-pricing refresh shutdown-aware: pass a lifecycle abort signal through both catalog fetches, abort in-flight work on Gateway stop, prevent cache mutation or refresh scheduling after stop, unref long background timers, and add focused lifecycle tests.

## Reproduction Assessment

Yes. The issue has concrete released-package steps, and current main has a focused source-level reproduction: start a pricing refresh with hanging fetches, call the returned stop callback during the in-flight refresh, then assert the fetches abort and no post-stop timer is scheduled.

## Solution Assessment

No. Current main is only a partial mitigation; the narrow maintainable fix is to make the existing pricing-refresh lifecycle honor Gateway stop rather than only shortening timeouts or adding operator opt-outs.
