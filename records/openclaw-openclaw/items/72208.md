## Summary

Close as implemented on main. Current main has the missing pricing-refresh lifecycle fix and the later readiness-path deferral, so slow OpenRouter/LiteLLM pricing fetches should no longer block Gateway readiness or survive Gateway shutdown.

## What This Changes

The issue reports that OpenClaw 2026.4.24 global npm installs hang during Gateway startup when model-pricing fetches time out, leaving the dashboard unreachable and Ctrl+C ineffective.

## Best Possible Solution

Keep the current main lifecycle and readiness-path fixes, then let the next release carry the resolved behavior to npm users.

## Reproduction Assessment

Yes. The issue has concrete released-package steps, and current main has a focused source-level regression path: start pricing refreshes with hanging fetches, stop the Gateway refresh, and verify both fetches abort with no post-stop timer or cache mutation.

## Solution Assessment

Yes. The current fix is the narrow maintainable direction: make the optional pricing refresh stop-aware and move it off the Gateway readiness path instead of changing unrelated startup or provider logic.
