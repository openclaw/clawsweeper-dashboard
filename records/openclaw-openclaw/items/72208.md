## Summary

Keep open. Current main has startup mitigations for the pricing-fetch regression, but the shutdown/hung-process part remains source-backed: the pricing refresh cannot abort in-flight fetches and can schedule another referenced refresh after Gateway stop.

## What This Changes

The issue reports that the 2026.4.24 npm-global Gateway on Windows hangs during OpenRouter/LiteLLM pricing timeouts, leaving the dashboard unreachable and Ctrl+C ineffective.

## Best Possible Solution

Keep the existing startup mitigations, then make model-pricing refresh shutdown-aware: pass a lifecycle abort signal through both catalog fetches, abort in-flight work on Gateway stop, prevent cache mutation and refresh scheduling after stop, unref long background timers, and add focused lifecycle coverage.

## Reproduction Assessment

Yes. The issue has concrete released-package steps, and current main has a high-confidence source-level reproduction path: start a pricing refresh with hanging fetches, call the returned stop callback during the in-flight refresh, then assert the fetches abort and no post-stop timer is scheduled.

## Solution Assessment

No. Current main is only a partial mitigation; the narrow maintainable fix is to wire the existing pricing-refresh lifecycle into Gateway stop rather than relying only on timeout length or operator opt-outs.
