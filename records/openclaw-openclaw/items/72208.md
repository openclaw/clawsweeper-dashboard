## Best Possible Solution

Keep the existing startup mitigations, then make model-pricing refresh stop-aware: compose a Gateway-owned abort signal into both catalog fetches, suppress cache writes and scheduling after stop, unref background timers, and add focused lifecycle coverage plus a changelog entry.

## Repair Work Prompt

Fix the remaining Gateway model-pricing lifecycle bug tracked by https://github.com/openclaw/openclaw/issues/72208. Current main already defers pricing startup work and documents `models.pricing.enabled: false`, but an in-flight OpenRouter/LiteLLM pricing refresh is still timeout-only: stopping Gateway cannot abort the active fetches, refresh completion can mutate the cache after stop, and the next 24h timer can be scheduled after stop. Keep the fix scoped to the Gateway model-pricing lifecycle. Thread a Gateway-owned abort signal into both pricing catalog fetches, compose it with the existing timeout signal, make the stop callback abort active work, suppress cache replacement and follow-up scheduling after stop, and unref long background timers. Add focused tests that fail on current main: stopping before the microtask still skips bootstrap; stopping during hanging fetches aborts both sources; stopped refresh completion does not schedule another timer or mutate cache. Preserve the existing `models.pricing.enabled` config behavior and do not add a new env/config surface unless a maintainer asks. Add a single-line user-facing changelog entry. Coordinate context from #72904, #72971, closed PR #72781, and closed PR #73214, but do not revive the timeout-only approach as the main fix.
