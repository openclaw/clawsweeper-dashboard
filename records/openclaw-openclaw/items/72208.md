## Summary

Keep open. Current main has mitigations for startup cost and a documented config opt-out, but the remaining shutdown/lifecycle part of this Windows pricing-refresh hang is still actionable because in-flight pricing fetches are not cancelled by the Gateway stop path.

## What This Changes

The issue reports that `openclaw gateway` from npm global `2026.4.24` hangs on Windows when OpenRouter and LiteLLM pricing fetches time out, leaving the dashboard unreachable and Ctrl+C ineffective.

## Best Possible Solution

Make the model-pricing refresh stop-aware: compose a Gateway-owned abort signal into both pricing fetches, guard cache replacement and rescheduling after stop, unref background timers, and keep the documented `models.pricing.enabled` opt-out as the operator control.

## Reproduction Assessment

Yes. The issue has concrete released-package steps, and current main has a high-confidence source-level reproduction: start a pricing refresh with hanging fetches, call the returned stop callback while it is in flight, and observe that the fetches are not aborted by Gateway shutdown.

## Solution Assessment

No. Current main is a partial mitigation; the narrow maintainable fix is lifecycle cancellation in `src/gateway/model-pricing-cache.ts`, not only timeout-length changes or another operator opt-out.
