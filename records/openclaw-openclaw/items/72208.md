## Summary

Keep open. Current main has useful startup mitigations for the model-pricing timeout regression, but the remaining lifecycle bug is still present: Gateway stop only clears the queued timer and cannot abort an in-flight OpenRouter/LiteLLM pricing refresh or prevent refresh completion from scheduling later work.

## What This Changes

The issue asks for `openclaw gateway` to remain usable after npm-global upgrades when OpenRouter and LiteLLM model-pricing fetches time out during startup.

## Best Possible Solution

Keep the existing startup mitigations, then make model-pricing refresh lifecycle-aware: compose per-request timeout with a Gateway-owned abort signal, abort in-flight OpenRouter/LiteLLM fetches on stop, prevent cache mutation and refresh scheduling after stop, unref long background timers, and add focused lifecycle regression tests plus a changelog entry.

## Reproduction Assessment

Yes. The issue has concrete released-package steps, and current main has a high-confidence unit reproduction path: start a pricing refresh with hanging injected fetches, call the returned stop callback while fetches are in flight, and assert the requests abort and no post-stop cache write or timer scheduling occurs.

## Solution Assessment

No. Current main’s deferred bootstrap, manifest reuse, and documented opt-out are useful mitigations, but the narrow maintainable fix is to make the existing pricing refresh cancellable and stop-aware rather than relying on timeout length or operator disablement.
