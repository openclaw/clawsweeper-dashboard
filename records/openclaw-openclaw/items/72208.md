## Summary

Keep open. Current main has startup mitigations and a config opt-out, but the pricing refresh still is not stop-aware once in flight, so the shutdown/Ctrl+C hanging lifecycle part remains actionable.

## What This Changes

The issue reports that `openclaw gateway` hangs after OpenRouter/LiteLLM pricing fetch timeouts in the 2026.4.24 npm-global Windows install, leaving the dashboard unreachable and Ctrl+C ineffective.

## Best Possible Solution

Make the pricing refresh Gateway-lifecycle-owned: compose a stop signal into both remote fetches, suppress post-stop cache writes and scheduling, unref refresh timers, and preserve the existing config opt-out.

## Reproduction Assessment

Yes. The issue has concrete released-package steps, and current main has a focused source-level repro: start a pricing refresh with never-resolving fetches, call the returned stop callback while it is in flight, and assert the fetches are not aborted and post-stop scheduling is still possible.

## Solution Assessment

No. The current mitigations reduce startup work and allow manual opt-out, but the narrow maintainable fix is cancellation and stop guarding in the existing pricing refresh path.
