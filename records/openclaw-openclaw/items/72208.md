## Summary

Current main fixes both parts of the report: pricing refresh no longer runs in the Gateway readiness path, and in-flight pricing fetches are aborted on Gateway stop without post-stop cache writes or timers. The fix is also shipped in v2026.5.2.

## What This Changes

The issue reports that `openclaw gateway` in global npm install 2026.4.24 hangs during LiteLLM/OpenRouter pricing-fetch timeouts, leaving the dashboard unreachable and Ctrl+C ineffective.

## Best Possible Solution

Keep the shipped Gateway pricing lifecycle behavior in v2026.5.2; broader startup/network degradation should remain in narrower open trackers if still reproducible.

## Reproduction Assessment

No current-main failing reproduction remains. The historical 2026.4.24 path is covered by source inspection and focused regression tests for deferred startup, in-flight abort, and post-stop timer/cache guards.

## Solution Assessment

Yes. The shipped fix is the narrow maintainable solution: keep optional pricing work out of Gateway readiness and bind pricing fetches to Gateway shutdown without changing provider behavior.
