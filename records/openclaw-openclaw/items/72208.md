## Summary

Keep open. Current main has useful mitigations, including manifest reuse and a config opt-out, but the default pricing refresh still uses uncoupled 60s timeout signals and Gateway stop does not cancel in-flight pricing fetches, so the Ctrl+C/hanging lifecycle part remains actionable.

## What This Changes

Fix the Windows npm-global Gateway startup hang in 2026.4.24 when OpenRouter and LiteLLM pricing fetches time out, leaving the dashboard and shutdown unusable.

## Best Possible Solution

Keep the existing startup mitigations, then make model-pricing refresh stop-aware by composing a Gateway-owned abort signal into both catalog fetches, suppressing cache writes and scheduling after stop, unrefing background timers, and adding focused lifecycle coverage plus a changelog entry.

## Reproduction Assessment

Yes. The released steps are concrete, and current main has a source-level reproduction path: start a pricing refresh with hanging fetches, call the stop callback or Gateway close prelude while it is in flight, then assert the fetches abort and no post-stop timer/cache mutation occurs.

## Solution Assessment

No. Current main is a partial mitigation; the narrow maintainable fix is cancellation and stop guarding in the existing pricing refresh path rather than another broad startup refactor.

## Repair Work Prompt

Fix the remaining Gateway model-pricing lifecycle gap for this issue. Current main already has manifest reuse and `models.pricing.enabled: false`, but the default pricing refresh still uses only per-request `AbortSignal.timeout(60000)`, `startGatewayModelPricingRefresh` does not abort an in-flight refresh after stop, and `scheduleRefresh` can schedule work after shutdown. Implement a stop-aware pricing refresh in `src/gateway/model-pricing-cache.ts`: create or accept a Gateway-owned abort signal, compose it with the per-fetch timeout for both OpenRouter and LiteLLM fetches, abort on `stopModelPricingRefresh`, skip cache replacement and rescheduling after stop, and unref any long-lived background refresh timer. Keep provider pricing policy/model mapping behavior unchanged and do not lower the default timeout unless the code path requires it. Add focused tests that start a refresh with hanging fetches, stop it while in flight, and prove the fetches abort and no post-stop timer/cache mutation happens; cover the Gateway close seam if needed. Add a single-line user-facing `CHANGELOG.md` fix entry. Validate with the targeted Gateway pricing/runtime tests and formatter, then run `pnpm check:changed` in Testbox before maintainer handoff.
