## Summary

Current main now covers both parts of the report: optional pricing refresh no longer blocks the Gateway ready path, and shutdown now aborts in-flight pricing fetches without post-stop cache writes or timers.

## What This Changes

The issue reports OpenClaw 2026.4.24 on Windows hanging during `openclaw gateway` startup when OpenRouter/LiteLLM pricing fetches time out, leaving the dashboard unreachable and Ctrl+C ineffective.

## Best Possible Solution

Keep the current main implementation and let the next package release carry it; no additional source repair is needed for this report.

## Reproduction Assessment

Yes. The issue has concrete released-package steps, and current main has focused source-level coverage that reproduces the lifecycle shape with hanging pricing fetches followed by a stop call.

## Solution Assessment

Yes. Deferring the optional refresh until the ready path and making the refresh stop-aware is the narrow maintainable fix; disabling pricing entirely remains a config workaround, not the core behavior fix.
