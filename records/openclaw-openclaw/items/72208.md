## Summary

Keep open. Current main has useful mitigations for the 2026.4.24 pricing-startup regression, including deferred startup work, manifest reuse, and a documented pricing opt-out, but the pricing refresh itself is still not lifecycle-aware: in-flight OpenRouter/LiteLLM fetches use only per-request timeout signals, Gateway stop cannot abort them, and refresh completion can schedule the next timer after stop. The open related PR #73214 is also only a timeout/operator opt-out change for #72904, not the remaining shutdown/lifecycle repair for this issue.

## What This Changes

The issue reports a Windows/npm-global regression where `openclaw gateway` hangs on version 2026.4.24 when OpenRouter and LiteLLM pricing fetches time out, leaving the dashboard unreachable and Ctrl+C ineffective.

## Best Possible Solution

Keep the startup mitigations, then make model-pricing refresh lifecycle-aware: pass a Gateway-owned abort signal into both catalog fetches, abort in-flight work on stop, suppress cache mutation and refresh scheduling after stop, unref long background timers, and add focused lifecycle tests plus a user-facing changelog entry.

## Reproduction Assessment

Yes. The issue has concrete released-package steps, and current main has a high-confidence source-level reproduction path: start a pricing refresh with hanging fetches, call the returned stop callback while the refresh is in flight, then assert the fetches abort and no post-stop timer or cache write occurs.

## Solution Assessment

No. Current main is a partial mitigation, and #73214's shorter timeout/operator opt-out would be useful but still leaves the runtime lifecycle gap; the narrow maintainable fix is to make the existing pricing refresh cancellable and stop-aware.

## Repair Work Prompt

Fix the remaining gateway model-pricing lifecycle gap for https://github.com/openclaw/openclaw/issues/72208. Current main already defers pricing startup and has `models.pricing.enabled: false`; do not remove those mitigations or change provider/model pricing semantics. Make `startGatewayModelPricingRefresh` / `refreshGatewayModelPricingCache` stop-aware: create a Gateway-owned abort controller for the refresh, pass its signal into both OpenRouter and LiteLLM catalog fetches, combine it safely with the per-request timeout, abort in-flight fetches when the returned stop callback runs, avoid cache replacement and refresh scheduling after stop, and unref the long refresh timer. Add focused tests proving: early stop still prevents startup, stop during an in-flight fetch aborts both catalog requests, no post-stop timer is scheduled, and disabled pricing still skips imports/fetches. Coordinate with open PR #73214 before changing timeout or env opt-out behavior; if that PR lands first, rebase and preserve its documented knobs. Add a user-facing CHANGELOG.md fix entry with valid attribution. Do not broaden core/plugin ownership boundaries or edit unrelated Gateway startup behavior.
