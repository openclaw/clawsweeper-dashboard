## Summary

Keep open. Current main has useful startup mitigations for the pricing-fetch regression, but the remaining shutdown/lifecycle part is still source-backed: pricing fetches are not tied to Gateway stop, the stop callback cannot cancel an in-flight refresh, and refresh completion can still schedule a referenced 24-hour timer. The related open #73214 patch only shortens timeout behavior and adds an environment opt-out; it does not make the refresh lifecycle-abortable.

## What This Changes

The issue reports a Windows npm-global regression where OpenClaw 2026.4.24 gateway startup hangs after LiteLLM/OpenRouter model-pricing fetch timeouts, leaving the dashboard unreachable and Ctrl+C ineffective.

## Best Possible Solution

Keep the shipped startup mitigations, then make model-pricing refresh fully Gateway-lifecycle aware: thread a stop AbortSignal through both pricing catalog fetches, combine it with the per-fetch timeout, prevent cache mutation and refresh scheduling after stop, unref long refresh timers, and cover the in-flight stop case with focused Gateway pricing tests.

## Reproduction Assessment

Yes. The report has concrete released-package steps, and current main has a focused source-level reproduction path: start a pricing refresh with hanging fetches, call the returned stop callback while the fetches are in flight, then assert the fetches abort and no post-stop timer is scheduled.

## Solution Assessment

No. Current main is a partial mitigation; the narrow maintainable fix is lifecycle-aware pricing refresh cancellation rather than only deferring startup work, extending/shortening timeout constants, or adding operator opt-outs.
