## Summary

Keep open. Current main and the latest release still intentionally flatten remote Markdown images and block remote image fetches, so this is a Control UI product/security policy decision rather than a cleanup close.

## What This Changes

The issue asks Control UI chat Markdown to render remote http(s) image URLs inline instead of flattening them to alt text.

## Best Possible Solution

Decide the Control UI remote-media policy first; if approved, implement renderer, sanitizer attributes, CSP, docs, focused tests, and browser proof together, preferably with an explicit opt-in, allowlist, or proxy design.

## Reproduction Assessment

Yes, at source level: current main flattens non-data Markdown image URLs to escaped alt text, tests assert that behavior, and the served Control UI CSP excludes remote image sources. I did not run a live browser session in this read-only review.

## Solution Assessment

No for the renderer-only suggested fix. The maintainable path needs an approved remote-media policy and coordinated renderer, sanitizer/CSP, docs, tests, and browser proof changes.
