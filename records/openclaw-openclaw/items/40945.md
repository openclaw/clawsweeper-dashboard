## Summary

Keep this open for maintainer product and security review. Current main and the latest release still flatten remote markdown images by design, and the Control UI CSP/docs now explicitly preserve the no-remote-browser-fetch policy.

## What This Changes

The issue requests that Control UI chat render remote `http`/`https` markdown image URLs inline instead of flattening them to alt text.

## Best Possible Solution

Keep the current no-remote-fetch behavior until maintainers approve a remote-media policy, then implement the approved path across renderer, CSP, docs, tests, and browser proof.

## Reproduction Assessment

Yes. Source inspection on current main shows Control UI chat markdown goes through `toSanitizedMarkdownHtml`, and current tests assert remote markdown images flatten to alt text while `data:image` still renders inline.

## Solution Assessment

No for the suggested renderer-only fix. The maintainable solution needs an explicit remote-media trust policy and coordinated renderer, CSP, docs, tests, and browser proof rather than only allowing `http(s)` image `src` values.
