## Summary

Keep open. Current main still flattens remote Control UI markdown images to alt text, and the surrounding tests, CSP, docs, and latest release describe that as deliberate no-remote-fetch hardening. Open PR #41025 directly targets this issue with closing syntax, so the next step is maintainer product/security review rather than cleanup closure.

## What This Changes

The issue requests that Control UI chat markdown render remote `http`/`https` image URLs inline instead of flattening them to alt text.

## Best Possible Solution

Define the Control UI remote-media policy first. If remote images should be supported, implement it coherently across the markdown renderer, sanitizer expectations, CSP, docs, and tests with a reviewed control model such as opt-in, allowlist, or managed proxy; otherwise keep the current hardening and close the paired PR after maintainer review.

## Reproduction Assessment

Yes. The static reproduction is high-confidence: `toSanitizedMarkdownHtml("![Alt text](https://example.com/img.png)")` reaches the renderer branch that returns escaped alt text, and the current colocated test asserts no `<img>` output for that case.

## Solution Assessment

Unclear. The requested renderer-only direction is not the best standalone fix while current CSP, docs, and tests intentionally block remote image fetches; the maintainable solution needs an explicit remote-media policy decision.

## Security Review

Status: needs_attention

Summary: The requested behavior is security-sensitive because rendered chat content could cause the operator browser to fetch arbitrary remote image URLs.

Concerns:

- **[medium] Remote images would trigger browser fetches:** `docs/web/control-ui.md:348`
  - body: Allowing `http(s)` markdown image URLs inline would let user- or model-supplied chat content make the Control UI browser request third-party hosts, conflicting with the documented no-remote-fetch policy unless maintainers add an explicit control model.
  - confidence: 0.94
- **[medium] Renderer changes must align with CSP:** `src/gateway/control-ui-csp.ts:47`
  - body: Current CSP excludes remote image sources; enabling the renderer alone would leave broken images, while broadening `img-src` needs the same reviewed remote-media safety model.
  - confidence: 0.92
