## Summary

Keep open: current main and the latest release still intentionally flatten remote Markdown images, and the Control UI CSP/docs explicitly keep remote browser image fetches disabled, so this remains a maintainer product/security decision rather than cleanup.

## What This Changes

Request remote http/https Markdown image URLs render inline in Control UI chat instead of being flattened to alt text.

## Best Possible Solution

Define the approved Control UI remote-media policy first, then either keep the current no-remote-fetch behavior or update the renderer, CSP, docs, tests, and browser proof together.

## Reproduction Assessment

Yes, at source level: current main routes chat Markdown through a renderer that flattens non-data image URLs, and tests assert that behavior. I did not run live browser proof in this read-only review.

## Solution Assessment

No for the suggested renderer-only fix. The maintainable path needs an explicit remote-media trust policy and coordinated renderer, CSP, docs, tests, and browser proof rather than only allowing http(s) image src values.
