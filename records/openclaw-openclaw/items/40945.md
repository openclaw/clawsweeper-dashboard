## Summary

Keep open. Current main still deliberately flattens remote markdown image URLs in Control UI chat, the docs and CSP describe that no-remote-fetch policy, and an open closing PR already targets this issue but needs product/security review.

## What This Changes

Allow Control UI chat markdown image syntax with remote http/https URLs to render inline instead of falling back to alt text.

## Best Possible Solution

Make an explicit Control UI remote-media policy decision; if approved, land a coordinated renderer, sanitizer, CSP, docs, tests, privacy mitigation, and browser-proof change through the open PR or a replacement.

## Reproduction Assessment

Yes, at source level: current main returns escaped alt text for non-data markdown image URLs, tests assert that behavior, and the served Control UI CSP rejects remote image loads. I did not run a live browser repro in this read-only review.

## Solution Assessment

No for the suggested renderer-only fix. A maintainable solution must align rendering with sanitizer attributes, CSP, docs, tests, and an approved remote-fetch privacy policy.
