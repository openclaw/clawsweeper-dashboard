Keep open. Current main and the latest release intentionally flatten remote Markdown image URLs and block remote image fetches, while the linked open implementation PR remains the right place to resolve the product/security policy decision.

## What This Changes

The issue requests Control UI chat Markdown render safe remote http/https images inline instead of flattening them to alt text.

## Best Possible Solution

Make an explicit Control UI remote-media policy decision; if approved, implement it across the renderer, sanitizer, CSP, docs, tests, and real browser proof through the linked PR or a replacement.

## Reproduction Assessment

Yes, at source level: current main returns escaped alt text for non-data Markdown image URLs, tests assert that behavior, and the served Control UI CSP rejects remote image loads. I did not run a live browser session in this read-only review.

## Solution Assessment

No for the suggested renderer-only fix. A maintainable solution must align rendering with sanitizer attributes, CSP, docs, tests, real browser proof, and an approved remote-fetch privacy policy.
