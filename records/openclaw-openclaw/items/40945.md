## Summary

Keep this open: current main and v2026.6.6 still intentionally flatten remote Markdown images, and the Control UI CSP/docs preserve the no-remote-browser-fetch policy, so the remaining work is a maintainer product/security decision.

## What This Changes

Request to render remote http/https Markdown image URLs inline in Control UI chat instead of flattening them to alt text.

## Best Possible Solution

Define an approved Control UI remote-media policy first, then update the renderer, CSP, docs, tests, and browser proof together if remote images are allowed.

## Reproduction Assessment

Yes, at source level: current main routes chat Markdown through toSanitizedMarkdownHtml, the renderer flattens non-data image URLs, and tests assert the remote-image flattening behavior. I did not run live browser proof in this read-only review.

## Solution Assessment

No for the suggested renderer-only fix. The maintainable solution needs an explicit remote-media trust policy and coordinated renderer, CSP, docs, tests, and browser proof rather than only allowing http(s) image src values.
