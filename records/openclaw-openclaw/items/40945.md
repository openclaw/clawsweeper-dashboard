## Summary

Keep open. Current main still flattens remote markdown images to alt text, the Control UI CSP/docs now explicitly define a local/data/blob-only image policy, and the issue is paired with open closing PR #41025, so maintainers need to decide that policy path before cleanup.

## What This Changes

Request remote http/https markdown images to render inline in the Control UI chat instead of being flattened to alt text.

## Best Possible Solution

Make an explicit product/security decision: either keep the local/data-only image policy and close the remote-image PRs, or land a coordinated remote-image feature that updates renderer, CSP, docs, and regression tests together.

## Reproduction Assessment

Yes, from source inspection. Current main returns escaped alt text for non-data markdown image URLs, and tests assert that remote https markdown images do not emit img tags.

## Solution Assessment

No for a renderer-only fix. The maintainable solution must address the documented CSP policy and remote-fetch privacy tradeoff, not only allow http/https in the markdown renderer.
