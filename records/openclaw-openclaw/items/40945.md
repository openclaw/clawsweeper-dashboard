Keep open. Current main and the latest release still enforce a documented no-remote-image-fetch Control UI policy, and the open closing PR needs maintainer product/security judgment before this issue can be resolved.

## What This Changes

The issue requests that Control UI chat Markdown render remote `http`/`https` image URLs inline instead of flattening them to alt text.

## Best Possible Solution

Decide the Control UI remote-media policy first; if approved, land the linked PR or a replacement that updates the active renderer, sanitizer/CSP, docs, tests, and real browser proof together.

## Reproduction Assessment

Yes, at source level: current main flattens non-data Markdown image URLs to escaped alt text, tests assert that behavior, and the served Control UI CSP excludes remote image sources. I did not run a live browser session in this read-only review.

## Solution Assessment

No for the renderer-only suggested fix. The maintainable path needs an approved remote-media policy and coordinated changes across renderer behavior, sanitizer/CSP, docs, tests, and browser proof.
