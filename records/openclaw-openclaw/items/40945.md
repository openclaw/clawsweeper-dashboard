## Summary

Keep open. Current main and v2026.5.20 still intentionally flatten remote Markdown image URLs and enforce a documented no-remote-fetch Control UI policy, while https://github.com/openclaw/openclaw/pull/41025 remains the open implementation candidate that needs product/security judgment.

## What This Changes

The issue requests that Control UI chat render remote http(s) Markdown image URLs inline instead of flattening them to alt text.

## Best Possible Solution

Make an explicit Control UI remote-media policy decision; if approved, implement it through the linked PR or a replacement across the active renderer, sanitizer attributes, CSP, docs, tests, and real browser proof.

## Reproduction Assessment

Yes, at source level: current main returns escaped alt text for non-data Markdown image URLs, tests assert that behavior, and the served Control UI CSP rejects remote image loads. I did not run a live browser session in this read-only review.

## Solution Assessment

No for the suggested renderer-only fix. A maintainable solution must align rendering with sanitizer attributes, CSP, docs, tests, real browser proof, and an approved remote-fetch privacy policy.
