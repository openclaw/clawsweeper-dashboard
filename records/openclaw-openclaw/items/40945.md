## Summary

Keep open. Current main and the latest release still flatten remote Markdown image URLs to alt text, and the Control UI CSP still excludes remote image sources, so this remains a product/security decision rather than cleanup; the open related fix PR also means the issue should stay open until that path is reviewed or closed.

## What This Changes

The issue requests inline rendering for remote http/https Markdown image URLs in Control UI chat.

## Best Possible Solution

Define the remote-media policy first, then implement the chosen Control UI path across the markdown renderer, CSP, tests, and any docs, preferably with an explicit operator-controlled or click-to-load behavior rather than accidental default-on remote fetches.

## Reproduction Assessment

Yes. Source and tests on current main show that `![Alt](https://example.com/img.png)` is rendered as alt text, and the shipped CSP would block remote image loads even if the renderer emitted an img tag.

## Solution Assessment

No, not as an automatic renderer-only bug fix. The maintainable solution needs a product/security decision because the current renderer, tests, CSP, and VISION.md safe-defaults guidance all point to remote image loading as an explicit policy change.
