## Summary

Keep this open: current main and the latest release still intentionally flatten remote Markdown images, and the Control UI CSP/docs preserve a no-remote-browser-fetch policy, so this remains a maintainer product/security decision rather than stale cleanup.

## What This Changes

The issue requests rendering remote http/https Markdown images inline in Control UI chat instead of flattening them to alt text.

## Best Possible Solution

Define an approved Control UI remote-media policy first, then either keep the current no-remote-fetch behavior or update the renderer, CSP, docs, tests, and browser proof together.

## Reproduction Assessment

Yes, at source level: current main routes chat Markdown through a renderer that flattens non-data image URLs, and tests assert that behavior. I did not run live browser proof in this read-only review.

## Solution Assessment

No for the suggested renderer-only fix. The maintainable solution needs an explicit remote-media trust policy and coordinated renderer, CSP, docs, tests, and browser proof rather than only allowing http(s) image src values.
