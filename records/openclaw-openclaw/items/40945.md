## Summary

Keep this open: current main and the latest release still intentionally flatten remote Markdown images, while CSP and docs block remote browser image fetches, so the remaining change is a product/security decision rather than cleanup-closeable stale work.

## What This Changes

The issue asks Control UI chat markdown to render remote http/https image URLs inline instead of flattening them to alt text.

## Best Possible Solution

Define an approved Control UI remote-media policy first; if remote images are allowed, update the renderer, CSP, docs, tests, and real browser proof together.

## Reproduction Assessment

Yes, at source level: rendering `![Alt text](https://example.com/img.png)` through the current Control UI markdown path returns alt text, and tests assert that behavior. I did not run live browser proof because this was a read-only cleanup review.

## Solution Assessment

No for the suggested renderer-only fix. The maintainable path needs an explicit remote-media trust policy and coordinated renderer, CSP, docs, tests, and browser proof rather than only allowing http(s) image src values.
