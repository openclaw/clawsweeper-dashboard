## Summary

Keep open. Current main still intentionally flattens remote markdown images to alt text, and the requested change now conflicts with the documented Control UI CSP policy, so this needs the existing implementation PR and product/security decision handled before the issue closes.

## What This Changes

The issue asks Control UI chat markdown to render remote `http`/`https` image URLs inline instead of only rendering base64 `data:image/...` sources.

## Best Possible Solution

Make an explicit product/security decision: either keep the data/local-only image policy and close the remote-image PRs, or land a narrow remote-image implementation that updates renderer, CSP, docs, and regression tests together.

## Reproduction Assessment

Yes from source inspection. Current main returns escaped alt text for non-data markdown image URLs, and tests assert that remote `https://` markdown images do not emit `<img>` tags.

## Solution Assessment

No for a renderer-only fix. The maintainable solution must address the documented CSP policy and remote-fetch privacy tradeoff, not only allow `http`/`https` in the markdown renderer.
