## Summary

Keep open. Current main and v2026.5.22 intentionally flatten remote Markdown image URLs and the served Control UI CSP blocks remote image fetches, so the remaining work is a product/security decision rather than conservative cleanup. The open fork PR at https://github.com/BingqingLyu/openclaw/pull/254 is useful implementation context, and the broader media umbrella at https://github.com/openclaw/openclaw/issues/64607 is related, but neither settles this specific remote-image policy.

## What This Changes

The issue requests inline rendering of remote http(s) Markdown images in the Control UI chat instead of flattening them to alt text.

## Best Possible Solution

Decide the Control UI remote-media policy first; if approved, implement renderer, sanitizer/CSP, docs, focused tests, and browser proof together, preferably with an explicit opt-in, allowlist, or proxy design.

## Reproduction Assessment

Yes, at source level: current main flattens non-data Markdown image URLs to escaped alt text, tests assert that behavior, and the served Control UI CSP excludes remote image sources. I did not run a live browser session in this read-only review.

## Solution Assessment

No for the suggested renderer-only fix. The maintainable path needs an approved remote-media policy and coordinated renderer, sanitizer/CSP, docs, tests, and browser proof changes.
