## Summary

Keep open. Current main still intentionally flattens remote Markdown images and the served Control UI CSP/docs block remote image fetches, so this is a product/security policy decision rather than a cleanup close.

## What This Changes

Request to render remote http(s) Markdown image URLs inline in Control UI chat instead of flattening them to alt text.

## Best Possible Solution

Decide the Control UI remote-media policy first; if approved, implement renderer, sanitizer attributes, CSP, docs, tests, and browser proof together, preferably with an opt-in allowlist or proxy design.

## Reproduction Assessment

Yes, at source level: current main flattens non-data Markdown image URLs to escaped alt text, tests assert that behavior, and the served Control UI CSP excludes remote image sources. I did not run a live browser session in this read-only review.

## Solution Assessment

No for the renderer-only suggested fix. The maintainable path needs an approved remote-media policy and coordinated renderer, sanitizer/CSP, docs, tests, and browser proof changes.
