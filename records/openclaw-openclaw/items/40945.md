## Summary

Keep open. Current main still intentionally flattens remote Control UI markdown image URLs to alt text, while CSP, tests, and docs enforce the no-remote-image-fetch policy; the linked open implementation PR remains the right review path, but changing this behavior needs product and security judgment.

## What This Changes

Allow Control UI chat markdown `http(s)` image URLs to render inline instead of flattening to alt text.

## Best Possible Solution

Make an explicit Control UI remote-media policy decision; if approved, land a coordinated renderer, sanitizer, CSP, docs, tests, privacy mitigation, and browser-proof change through the linked PR or a replacement.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes, at source level: current main returns escaped alt text for non-data markdown image URLs, tests assert that behavior, and the served Control UI CSP rejects remote image loads.

## Solution Assessment

Is this the best way to solve the issue? No for the suggested renderer-only fix; a maintainable change must align rendering with sanitizer attributes, CSP, docs, tests, and an approved remote-fetch privacy policy.
