## Summary

Keep open for maintainer product and security review. Current main still reproduces the reported behavior, but that behavior is now backed by renderer tests, CSP policy, and docs that intentionally prevent remote browser image fetches from Control UI chat content.

## What This Changes

The issue asks for Control UI chat markdown images with remote http/https URLs to render inline instead of flattening to alt text.

## Best Possible Solution

Decide the Control UI remote-media trust policy first, then implement an approved path through structured media or a tightly scoped image pipeline with matching CSP, docs, tests, and browser proof.

## Reproduction Assessment

Yes. Source inspection shows Control UI chat text goes through toSanitizedMarkdownHtml, and current tests assert remote markdown images flatten to alt text while data-image URIs render inline.

## Solution Assessment

No for the suggested renderer-only fix. The maintainable solution needs an explicit remote-media policy and coordinated renderer, CSP, docs, tests, and real browser proof rather than merely allowing http(s) img src values.
