## Summary

Keep open. Current main and the latest release still intentionally flatten remote Markdown images and block remote image fetches via CSP, while the prior target-repo implementation PR was closed unmerged; this issue remains the focused tracker for the Control UI remote-image policy decision rather than a narrow bug-fix candidate.

## What This Changes

Control UI chat markdown should render remote http(s) image URLs inline instead of reducing them to alt text.

## Best Possible Solution

Keep this as the focused remote-image policy tracker; if maintainers approve the feature, implement an explicit opt-in or allowlisted renderer/CSP/docs/test/browser-proof change together.

## Reproduction Assessment

Yes, at source level: current main flattens non-data Markdown image URLs to escaped alt text, tests assert that behavior, and the served Control UI CSP excludes remote image sources. I did not run a live browser session in this read-only review.

## Solution Assessment

No for the renderer-only suggested fix. The maintainable path needs an approved remote-media policy and coordinated renderer, sanitizer/CSP, docs, tests, and browser proof, preferably behind an explicit opt-in or allowlist.
