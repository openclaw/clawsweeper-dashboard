## Summary

Keep open. Current main and v2026.5.19 still intentionally flatten remote Markdown image URLs and enforce a no-remote-fetch Control UI policy, while the open closing PR is the right place to resolve the product/security decision.

## What This Changes

Control UI chat should render remote http/https Markdown image URLs inline instead of flattening them to alt text.

## Best Possible Solution

Make an explicit Control UI remote-media policy decision; if approved, implement it through the linked PR or a replacement across renderer, sanitizer attributes, CSP, docs, tests, and real browser proof.

## Reproduction Assessment

Yes, at source level: current main returns escaped alt text for non-data Markdown image URLs, tests assert that behavior, and the served Control UI CSP rejects remote image loads. I did not run a live browser session in this read-only review.

## Solution Assessment

No for the suggested renderer-only fix. A maintainable solution must align rendering with sanitizer attributes, CSP, docs, tests, real browser proof, and an approved remote-fetch privacy policy.
