## Summary

Keep open. Current main and v2026.5.12 still flatten remote Control UI markdown images to alt text, while the current CSP, tests, and docs intentionally enforce a no-remote-image-fetch policy; the linked open implementation PR should stay the review path because changing this behavior needs product/security judgment.

## What This Changes

Request to render remote http/https markdown image URLs inline in Control UI chat instead of flattening them to alt text.

## Best Possible Solution

Make an explicit Control UI remote-media policy decision; if approved, land a coordinated renderer, sanitizer, CSP, docs, tests, privacy mitigation, and browser-proof change through the linked PR or a replacement.

## Reproduction Assessment

Yes, from source inspection. Current main returns escaped alt text for non-data markdown image URLs, tests assert that behavior, and the served Control UI CSP rejects remote image loads.

## Solution Assessment

No for the suggested renderer-only fix. The maintainable solution has to align rendering with CSP, sanitizer attributes, docs, tests, and an approved remote-fetch privacy policy.
