## Summary

Keep open: current main and the latest release still flatten remote Markdown images, while the Control UI CSP and docs intentionally block remote browser image fetches, so this remains a product/security decision rather than stale cleanup.

## What This Changes

The issue requests rendering remote http/https Markdown image URLs inline in Control UI chat instead of flattening them to alt text.

## Best Possible Solution

Define the approved Control UI remote-media policy first, then either keep the current no-remote-fetch behavior or update the renderer, CSP, docs, tests, and browser proof together.

## Reproduction Assessment

Yes, at source level: current main routes Markdown images through a renderer that flattens non-data URLs, and tests assert that behavior. I did not run a live browser reproduction in this read-only review.

## Solution Assessment

No for the suggested renderer-only fix. The maintainable path needs an explicit remote-media trust policy plus coordinated renderer, CSP, docs, tests, and browser proof.
