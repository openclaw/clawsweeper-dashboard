## Summary

Keep open. Current main still flattens remote Markdown images and the shipped Control UI policy deliberately blocks remote image fetches, so this needs a product/security decision rather than cleanup closure.

## What This Changes

The issue requests that Control UI chat render Markdown `http(s)` image URLs inline instead of reducing them to alt text.

## Best Possible Solution

Decide the Control UI remote-media policy first; if approved, implement renderer, sanitizer/CSP, docs, focused tests, and browser proof together, preferably with an explicit opt-in, allowlist, or proxy design.

## Reproduction Assessment

Yes, at source level: current main flattens non-data Markdown image URLs to escaped alt text, tests assert that behavior, and the served Control UI CSP excludes remote image sources. I did not run a live browser session in this read-only review.

## Solution Assessment

No for the renderer-only suggested fix. The maintainable path needs an approved remote-media policy and coordinated renderer, sanitizer/CSP, docs, tests, and browser proof changes.
