## Summary

Keep open. Current main intentionally keeps remote Markdown images inert and blocks browser fetches, while the linked implementation and broader inline-media issue leave the product/security decision unresolved rather than obsolete.

## What This Changes

Request inline rendering of remote HTTP(S) Markdown images in Control UI chat.

## Best Possible Solution

Keep remote Markdown images inert by default and resolve the linked implementation plus https://github.com/openclaw/openclaw/issues/64607 with one explicit privacy-safe design, such as click-to-load, operator opt-in or allowlists, or a guarded same-origin proxy integrated with structured media.

## Reproduction Assessment

Yes. On current main, the Markdown renderer flattens remote image syntax to alt text and the CSP independently blocks remote image requests; the latest-release context shows this is also shipped behavior.

## Solution Assessment

No. The suggested renderer-only HTTP(S) allowlist conflicts with the current CSP and does not address tracking privacy; the renderer, CSP, rich-output contract, tests, and user controls need one coordinated design.
