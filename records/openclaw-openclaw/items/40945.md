## Summary

Keep open. Current main and the latest release intentionally reject remote Markdown image URLs through the renderer, tests, CSP, and docs, so the requested behavior remains a product/security decision rather than conservative cleanup.

## What This Changes

Request remote http/https Markdown image URLs to render inline in Control UI chat instead of flattening to alt text.

## Best Possible Solution

Decide the Control UI remote-media policy first; if approved, update renderer, sanitizer/CSP, docs, focused tests, and browser proof together, preferably with an explicit opt-in, allowlist, or proxy design.

## Reproduction Assessment

Yes, at source level: current main flattens non-data Markdown image URLs to escaped alt text, tests assert that behavior, and the served Control UI CSP excludes remote image sources. I did not run a live browser session in this read-only review.

## Solution Assessment

No for the suggested renderer-only fix. The maintainable path needs an approved remote-media policy and coordinated renderer, sanitizer/CSP, docs, tests, and browser proof changes.
