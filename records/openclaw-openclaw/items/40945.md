## Summary

Keep open. Current main and the latest release still flatten remote Control UI markdown images to alt text, the shipped CSP/docs intentionally reject remote image fetches, and an open linked implementation PR still needs maintainer product/security judgment before this behavior changes.

## What This Changes

Request remote http/https Markdown image URLs in Control UI chat render inline instead of being flattened to alt text.

## Best Possible Solution

Make an explicit Control UI remote-media policy decision; if approved, land a coordinated renderer, sanitizer, CSP, docs, tests, privacy mitigation, and browser-proof change through the linked PR or a replacement, otherwise keep/document the no-remote-fetch behavior.

## Reproduction Assessment

Yes, from source inspection. Current main returns escaped alt text for non-data markdown image URLs, tests assert that behavior, and the Control UI CSP rejects remote image loads.

## Solution Assessment

No for the suggested renderer-only fix. The maintainable solution has to align rendering with CSP, sanitizer attributes, docs, tests, and an approved remote-fetch privacy policy.
