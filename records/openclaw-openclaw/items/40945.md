## Summary

Keep open. Current main still flattens remote Control UI markdown images to alt text, the served CSP and docs intentionally block remote image fetches, and open PR #41025 already targets this issue with closing syntax, so this needs maintainer product/security review rather than cleanup closure.

## What This Changes

Render safe remote http/https markdown image URLs inline in Control UI chat instead of flattening them to alt text.

## Best Possible Solution

Decide the Control UI remote-media policy first; if approved, update the renderer, CSP, docs, changelog, and tests together through #41025 or a replacement, otherwise keep the current hardening and route broader inline-media requests to a dedicated policy/design issue.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: `toSanitizedMarkdownHtml("![Alt text](https://example.com/img.png)")` reaches the branch that returns escaped alt text, and the current test asserts no `<img>` output.

## Solution Assessment

Unclear. The suggested renderer-only change is not the best standalone fix because current CSP, docs, and tests deliberately block remote image fetches; the maintainable solution needs an explicit policy decision and aligned implementation.

## Security Review

Status: needs_attention

Summary: Allowing remote markdown images would make operator browsers fetch arbitrary third-party URLs from rendered chat content.

Concerns:

- **[medium] Remote images would trigger browser fetches:** `docs/web/control-ui.md:349`
  - body: Enabling inline http(s) image URLs in model- or user-supplied chat markdown would let rendered content make the Control UI browser request third-party hosts, conflicting with the documented no-remote-fetch policy unless maintainers approve a reviewed control model.
  - confidence: 0.94
- **[medium] Renderer changes must align with CSP:** `src/gateway/control-ui-csp.ts:47`
  - body: The current CSP excludes remote image sources, so a renderer-only change would still leave broken images; broadening img-src is the same remote-media security decision and should be reviewed with the renderer change.
  - confidence: 0.92
