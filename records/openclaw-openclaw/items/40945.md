## Summary

Keep open. Current main and the latest release still flatten remote Control UI markdown image URLs to alt text, current main CSP/docs explicitly reject remote image fetches, and the issue is paired with an open implementation PR that uses closing syntax but is still unresolved.

## What This Changes

The issue asks Control UI chat markdown to render remote http/https image URLs inline instead of only rendering base64 data-image markdown images.

## Best Possible Solution

Make an explicit Control UI remote-media policy decision; if approved, land a coordinated renderer, sanitizer, CSP, docs, tests, and browser-proof change through the open PR or a replacement, otherwise document remote markdown images as unsupported and leave broader media work to the canonical umbrella.

## Reproduction Assessment

Yes, from source inspection. Current main returns escaped alt text for non-data markdown image URLs, tests assert that behavior, and the Control UI CSP rejects remote image loads.

## Solution Assessment

No for the suggested renderer-only fix. The maintainable solution has to align rendering with CSP, sanitizer attributes, docs, tests, and an approved remote-fetch privacy policy.
