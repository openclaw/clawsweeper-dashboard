## Summary

Keep open. Current main and the latest release still intentionally flatten remote Control UI markdown images to alt text, while CSP/docs reject remote image fetches; the linked implementation PR remains open, so this needs maintainer product/security handling before cleanup closure.

## What This Changes

Request to render remote http/https markdown image URLs inline in Control UI chat instead of flattening them to alt text.

## Best Possible Solution

Make an explicit Control UI remote-media policy decision; if approved, land a coordinated renderer, CSP, docs, tests, and privacy-mitigation change through the linked PR or a replacement, otherwise close remote-image rendering requests as unsupported.

## Reproduction Assessment

Yes, from source inspection. Current main returns escaped alt text for non-data markdown image URLs, tests assert that behavior, and v2026.5.7 ships the same renderer/CSP policy.

## Solution Assessment

No for a renderer-only change. The maintainable solution must align markdown rendering with Control UI CSP, docs, tests, and the remote-fetch privacy policy before enabling http/https images.
