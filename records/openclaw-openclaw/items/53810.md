## Summary

Keep open: current main and the latest release still do not normalize tagged tool-call sections when they arrive inside Qwen/ModelStudio reasoning or thinking content, so the reported premature subagent completion remains source-reproducible even though the Kimi text-block variant has a partial parser.

## What This Changes

The issue requests fixing subagent completion announcements when provider models emit non-standard tagged tool-call markup inside thinking/reasoning output instead of structured tool-call blocks.

## Best Possible Solution

Normalize only validated standalone tagged tool-call sections at the provider/OpenAI-compatible stream boundary, emit structured toolCall blocks, and set toolUse only after successful extraction with focused regression coverage.

## Reproduction Assessment

No live high-confidence reproduction was established because the reported path needs affected provider credentials and stochastic output. Source inspection does show the current-main path: tagged markers in reasoning/thinking content are not promoted to structured toolCall blocks.

## Solution Assessment

Yes. The best fix is provider or OpenAI-compatible stream normalization rather than changing the subagent announce queue, because the announce flow should react to a correctly normalized assistant message shape.
