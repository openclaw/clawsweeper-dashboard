Keep open: current main has partial text-block tool-call repair, but the reported marker-inside-thinking path remains unhandled for affected bundled Qwen/ModelStudio and Kimi-style model routes, and no canonical open replacement was found.

## What This Changes

The issue reports provider/model-specific tagged tool-call markers inside thinking/reasoning content causing subagent runs to stop without executing pending tools and announce incomplete work.

## Best Possible Solution

Add a narrowly gated provider/shared repair that promotes only standalone, validated tagged tool-call sections from affected thinking/reasoning content into structured `toolCall` blocks and updates `stopReason` only after successful extraction.

## Reproduction Assessment

No live high-confidence reproduction was run in this read-only review. Source inspection is enough to show the reported marker-in-thinking shape still bypasses current promotion because the repair paths only parse text blocks.

## Solution Assessment

Yes, the maintainable fix is provider/shared normalization before terminal tool-use decisions, not delaying announcements after the run has already been classified as stopped. The repair should stay allowed-tool-gated and avoid parsing mixed prose.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/53810 as a narrow provider/runtime bug. Current main promotes standalone plain-text tool calls from text blocks and parses Kimi tagged markers from text blocks, but markers embedded in thinking/reasoning content still bypass promotion and leave `stopReason: "stop"`. Add conservative normalization for the affected Kimi/Qwen/ModelStudio marker shape: only promote standalone validated `<|tool_calls_section_begin|>...<|tool_call_end|>` sections, only when the resolved tool names are allowed for the current request, and only update `stopReason` to `toolUse` after successful extraction. Do not add config/env flags, do not parse mixed prose, and do not broaden execution of arbitrary thinking text. Add focused regression coverage for marker-in-thinking content plus existing text-marker behavior, and keep user-visible sanitization behavior intact. If live Qwen/ModelStudio credentials are unavailable, state that clearly and include synthetic regression proof plus the exact live command/scenario a maintainer should run.
