## Summary

Keep open: the adjacent Kimi parser and parallel tool-call fixes are present, but current main still does not promote the reported Qwen/ModelStudio tagged tool-call sections from reasoning/thinking content into structured tool calls.

## What This Changes

The issue reports model/provider-specific subagent premature completion when tagged tool-call markup inside thinking content is not normalized into structured tool calls.

## Best Possible Solution

Add conservative provider-owned Qwen/ModelStudio normalization, or a narrowly gated shared OpenAI-compatible helper, that extracts only validated standalone tagged tool-call sections from reasoning/thinking content before subagent completion observes the turn.
