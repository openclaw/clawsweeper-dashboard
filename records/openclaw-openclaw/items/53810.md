## Summary

Keep open: current main still does not promote the reported Qwen/ModelStudio-style tagged tool calls when the markers live entirely inside thinking content, and a fix needs live provider proof plus maintainer security/product direction before executable tool calls are parsed out of reasoning text.

## What This Changes

The issue reports premature subagent completion announcements when some provider/model combinations emit tagged tool-call markers inside thinking blocks instead of structured `toolCall` blocks.

## Best Possible Solution

Add provider-owned Qwen/ModelStudio normalization, or a narrow shared OpenAI-compatible helper, that promotes only validated standalone tagged tool-call sections before stop-reason and announce logic, with live provider proof and allowed-tool gating.

## Reproduction Assessment

No high-confidence live reproduction is available. Source inspection shows the current path leaves marker text inside thinking blocks non-executable, but the reported provider/model behavior has not been replayed against current main.

## Solution Assessment

Unclear until maintainers choose the security boundary. Delaying announce alone is not the best fix; narrowly canonicalizing validated provider-owned markers before announce is the maintainable path if live proof confirms the behavior.
