## Best Possible Solution

Land the focused migration fix with its regression test after normal CI and maintainer review; keep v1 migration using one running ID set and no new config or compatibility mode.

## Reproduction Assessment

## Solution Assessment

Yes. Adding the generated ID to the existing running set immediately after generation is the narrowest maintainable fix, and sibling append/branch paths already keep their ID sources populated.
