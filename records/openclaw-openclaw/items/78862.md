## Summary

Current main already carries Telegram forum topic IDs from cron delivery target resolution into outbound Telegram sends, so the reported `:topic:` target no longer drops `message_thread_id`.

## What This Changes

The issue asks for Telegram cron delivery targets like `-100...:topic:12532` to send into that forum topic instead of the default topic.

## Best Possible Solution

Keep the current-main cron topic extraction and Telegram send-thread plumbing, and let the next release carry the main-only fix to users running v2026.5.6 or older.

## Reproduction Assessment

No: I do not have a high-confidence failing reproduction on current main because the current source path carries `:topic:` into `threadId` and then into Telegram `message_thread_id`. The failing behavior may still exist in older released builds or local deployments that predate the current main code.

## Solution Assessment

Yes: the current implementation is the narrow maintainable solution because it honors explicit `:topic:` intent without requiring an extra Telegram `getChat` lookup, while still passing explicit `--thread-id` through the same outbound path.
