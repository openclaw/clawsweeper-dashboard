## Summary

Keep open. Current main still has the source path that can render cumulative WebChat assistant text across tool boundaries, and the more complete implementation candidate remains open rather than merged.

## What This Changes

The issue requests a Control UI/WebChat fix so assistant text after tool-call boundaries renders as fresh cards instead of cumulative duplicate text.

## Best Possible Solution

Land a rebased UI-side committed-prefix reconciliation, such as https://github.com/openclaw/openclaw/pull/74498 or an equivalent patch, with focused regression tests while preserving full-snapshot gateway deltas for ACP, TUI, and generic subscribers.

## Reproduction Assessment

Yes, source-reproducible with high confidence. Current main assigns full-snapshot assistant deltas to `chatStream`, commits that stream into `chatStreamSegments` on tool start, and then renders committed segments plus the next active full snapshot separately.

## Solution Assessment

Yes for the current fix direction. The narrow maintainable fix is Control UI committed-prefix tracking/slicing, because changing gateway delta semantics would affect ACP, TUI, and other consumers that process full-snapshot message text.
