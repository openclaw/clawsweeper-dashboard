## Summary

Keep open. Current main still has the source-level Control UI path that can render cumulative assistant text across tool boundaries, and the focused UI-side fix remains open rather than merged.

## What This Changes

Investigate a Control UI/WebChat live-rendering bug where assistant text emitted between tool calls appears as cumulative duplicate message cards during streaming while persisted history remains correct.

## Best Possible Solution

Land or supersede the UI-side committed-prefix projection in https://github.com/openclaw/openclaw/pull/74498 while preserving full-snapshot gateway deltas for ACP, TUI, and generic subscribers.

## Reproduction Assessment

Yes. Current main is source-reproducible with high confidence: a full assistant snapshot is assigned to chatStream, a tool boundary commits that stream into chatStreamSegments, and the renderer later displays both the committed segment and the next active full snapshot.

## Solution Assessment

Yes. The best current direction is a Control UI committed-prefix/slicing fix like the active replacement PR, because changing gateway delta semantics would conflict with existing full-snapshot consumers and tests.
