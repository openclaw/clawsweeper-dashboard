## Summary

Keep open. Current main still has the live Control UI path that can render cumulative chat snapshots alongside already-committed tool-boundary stream cards, and the linked closing PR #74389 is still an unmerged implementation candidate rather than a shipped fix.

## What This Changes

The issue asks for Control UI WebChat live rendering to reset or delta assistant text at tool boundaries so each text-between-tools block appears as a fresh card instead of a cumulative duplicate.

## Best Possible Solution

Review and land #74389 or an equivalent narrow Control UI fix that tracks the committed stream prefix or otherwise deltas live assistant snapshots at tool boundaries, clears terminal live artifacts safely, and adds regression coverage for multi-tool text -> tool -> text turns plus the mirrored tool-bubble variant. A user-facing changelog entry should accompany the fix.

## Reproduction Assessment

Yes. The issue has concrete WebChat multi-tool steps, multiple independent confirmations, and a hard-refresh discriminator; current source shows cumulative server snapshots can still be rendered beside committed stream segments without prefix trimming.

## Solution Assessment

Unclear whether the linked PR is the complete best fix until normal review, but the best fix boundary is clear: Control UI live stream state/render reconciliation, not gateway transcript persistence, because history reload already displays the correct persisted blocks.
