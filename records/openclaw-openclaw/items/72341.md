## Summary

Keep open. Current main still has the reported live Control UI rendering shape: gateway chat deltas are cumulative, Control UI stores pre-tool stream text as committed segments, and the renderer appends the current cumulative live stream as another card without trimming the already committed segment text; the provided context also shows open PR #74389 explicitly targeting this issue, so the issue should stay open until that PR or an equivalent fix lands.

## What This Changes

Control UI/WebChat should render assistant text after each tool boundary as fresh live cards instead of cumulative duplicate cards during text/tool/text turns.

## Best Possible Solution

Land #74389 or an equivalent focused Control UI fix that reconciles cumulative gateway snapshots against already committed stream segments, clears terminal live artifacts safely, adds text-to-tool-to-text regression coverage, and includes the required user-facing changelog entry.

## Reproduction Assessment

Yes. The issue has concrete multi-tool WebChat steps, multiple independent confirmations, and a hard-refresh discriminator; current source also shows how cumulative gateway snapshots can be rendered alongside already committed stream segments.

## Solution Assessment

Unclear whether the linked PR is complete until normal PR review, but the correct fix boundary is clear: Control UI live stream reconciliation around tool boundaries and terminal events, not gateway persistence.
