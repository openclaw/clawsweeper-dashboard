## Summary

Keep open. Current main still has a source-reproducible Control UI/WebChat path where full-snapshot assistant stream text is committed and rendered around tool-card boundaries without subtracting already-rendered pre-tool text; the issue also has open implementation PRs rather than merged evidence.

## What This Changes

Control UI/WebChat should render assistant text around tool calls as fresh non-cumulative live cards.

## Best Possible Solution

Land the UI-side committed-prefix stream reconciliation in https://github.com/openclaw/openclaw/pull/74498 or an equivalent narrow Control UI fix, then close this issue and any superseded implementation PR.

## Reproduction Assessment

Yes. Source inspection on current main shows cumulative gateway snapshots are assigned to `chatStream`, pre-tool text is separately committed into `chatStreamSegments`, and the renderer emits both without subtracting the already-rendered prefix.

## Solution Assessment

Yes for the fix boundary. This should be solved in Control UI live-stream reconciliation while preserving the gateway full-snapshot contract used by other subscribers.
