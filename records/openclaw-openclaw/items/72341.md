## Summary

Keep open. Current main still has the source-level text -> tool -> cumulative full-snapshot path, and the open fix PRs have not merged; this issue should stay open until one lands or is superseded.

## What This Changes

The requested fix is to prevent live Control UI/WebChat text between tool calls from being rendered as cumulative duplicate cards during one assistant turn.

## Best Possible Solution

Land or replace a Control UI fix that uses deltaText or committed-prefix tracking in the WebChat live stream path while preserving the gateway's full-snapshot message contract.

## Reproduction Assessment

Yes. Source inspection on current main gives a deterministic text -> tool -> text path: payload.message sets the full chatStream, tool start commits the previous stream into chatStreamSegments, and buildChatItems renders the committed segment plus the active full snapshot.

## Solution Assessment

Yes for the fix direction. The narrow solution belongs in Control UI live-stream reconciliation, either by consuming deltaText or tracking the committed prefix, rather than changing the gateway full-snapshot message contract used by other consumers.
