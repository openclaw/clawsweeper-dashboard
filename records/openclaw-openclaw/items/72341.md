## Summary

Keep open. Current main still has the live WebChat mismatch: the gateway broadcasts cumulative merged assistant snapshots, while Control UI commits pre-tool text into stream segments and then renders the next live snapshot verbatim as another stream card. The open closing PR #74389 remains the right implementation path to review rather than closing this issue early.

## What This Changes

The issue requests a Control UI/WebChat fix so assistant text emitted between tool calls renders as fresh non-cumulative cards instead of cumulative duplicate bubbles.

## Best Possible Solution

Review and land #74389 or an equivalent narrow Control UI fix that tracks the committed stream prefix, deltas cumulative live snapshots after tool boundaries, and clears terminal live artifacts without hiding valid optimistic tails. The fix should add regression coverage for multi-tool text -> tool -> text turns and the mirrored tool-bubble variant, plus a user-facing changelog entry.

## Reproduction Assessment

Yes. The issue has concrete multi-tool WebChat steps, multiple independent confirmations, and a hard-refresh discriminator; current source also shows a source-level reproduction path where cumulative gateway snapshots are rendered beside already committed stream segments.

## Solution Assessment

Unclear whether the linked PR is the complete best fix until normal PR review, but the best fix boundary is clear: Control UI live stream reconciliation around tool boundaries. If #74389 only clears final artifacts, it should be checked against live in-progress cumulative cards as well as final duplicate bubbles.
