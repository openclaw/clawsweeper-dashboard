## Summary

Keep open. Current main still has the Control UI/WebChat source path that can render cumulative assistant text around tool boundaries, and the linked implementation pull requests remain open rather than merged.

## What This Changes

Fix Control UI/WebChat live rendering so assistant text after tool calls appears as fresh non-cumulative cards.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/74498 or an equivalent UI-side committed-prefix reconciliation with regression tests, while preserving gateway full-snapshot chat deltas for ACP, TUI, and other subscribers.

## Reproduction Assessment

Yes. The issue discussion gives concrete WebChat steps and current source inspection shows the deterministic path: delta assigns chatStream, tool start commits it into chatStreamSegments, the next full-snapshot delta overwrites chatStream, and buildChatItems renders both.

## Solution Assessment

Yes for the proposed direction. This should be fixed in Control UI stream-prefix reconciliation rather than by changing gateway delta semantics, because ACP/TUI and gateway tests rely on monotonic full-snapshot message text.
