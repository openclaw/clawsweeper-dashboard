## Summary

Keep open. Current main still has the Control UI live-render path where committed tool-boundary stream segments can be rendered alongside cumulative live chat snapshots, and #74389 is an open closing PR rather than merged evidence.

## What This Changes

The issue asks Control UI/WebChat to render assistant text between tool calls as fresh live cards instead of cumulative duplicate bubbles.

## Best Possible Solution

Review and land #74389 or an equivalent narrow Control UI live-stream reconciliation fix with regression coverage for text-tool-text turns.

## Reproduction Assessment

Yes. The reported text-to-tool-to-text path is source-reproducible: the gateway sends cumulative snapshots, the UI snapshots pre-tool text into segments, and the renderer appends segments plus the current stream without trimming already-rendered text.

## Solution Assessment

Yes for the fix boundary. This belongs in Control UI live stream/tool artifact reconciliation, not gateway persistence, because the report and source both show persisted history is correct while live render state duplicates.
