## Summary

Keep open. Current `origin/main` still has the source-level Control UI/WebChat path that can render already-committed pre-tool assistant text again after a tool boundary, and the focused UI-side fix remains open at https://github.com/openclaw/openclaw/pull/74498.

## What This Changes

Fix Control UI/WebChat live rendering so assistant text after tool calls appears as fresh, non-cumulative cards instead of repeating earlier text from the same turn.

## Best Possible Solution

Land a rebased UI-side committed-prefix projection, such as https://github.com/openclaw/openclaw/pull/74498 or an equivalent patch, with focused regression coverage while preserving gateway full-snapshot semantics for non-WebChat consumers.

## Reproduction Assessment

Yes. Current `origin/main` is source-reproducible: gateway deltas carry full merged assistant text, the UI commits pre-tool stream text into `chatStreamSegments`, then assigns the next full snapshot to `chatStream` and renders both surfaces separately.

## Solution Assessment

Yes for the active fix direction. The narrow maintainable solution is UI-side committed-prefix slicing, because changing gateway delta semantics would risk ACP, TUI, and other subscribers that expect monotonic full snapshots.
