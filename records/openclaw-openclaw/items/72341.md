## Summary

Keep open. Current main still has the source-level Control UI/WebChat render path that can replay already committed pre-tool assistant text, and the active UI-side fix remains open and conflicting rather than merged.

## What This Changes

The issue asks for Control UI/WebChat live rendering to show each assistant text block between tool calls as a fresh card instead of cumulative duplicate text.

## Best Possible Solution

Land or supersede the UI-only committed-prefix projection from https://github.com/openclaw/openclaw/pull/74498 after rebase and real Control UI proof, while preserving gateway full-snapshot deltas for ACP, TUI, and other subscribers.

## Reproduction Assessment

Yes. Current main is source-reproducible with high confidence: full-snapshot assistant text is assigned to chatStream, committed into chatStreamSegments at tool start, and then rendered again as the next active stream.

## Solution Assessment

Yes for the current direction. The narrow maintainable fix is Control UI committed-prefix tracking/slicing, because changing gateway delta semantics would risk ACP, TUI, and other full-snapshot consumers.
