## Summary

## What This Changes

The issue asks for Control UI/WebChat live streaming to render assistant text between tool calls as fresh cards instead of cumulative duplicate cards.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/74498 or an equivalent UI-only committed-prefix projection with focused regression tests, while preserving full-snapshot gateway deltas for ACP, TUI, and other subscribers.

## Reproduction Assessment

Yes. The issue is source-reproducible with high confidence: current main commits chatStream into chatStreamSegments at tool start, then assigns later full-snapshot assistant text back to chatStream, and buildChatItems renders both surfaces separately.

## Solution Assessment

Yes for the active direction. The best maintainable fix is a Control UI projection that slices already committed prefix text after tool boundaries, not a gateway delta contract change that could regress ACP, TUI, and generic websocket consumers.
