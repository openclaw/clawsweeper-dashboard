## What This Changes

Control UI/WebChat should render assistant text around tool calls as fresh message cards instead of cumulative duplicate cards.

## Best Possible Solution

Review and land #74389, or an equivalent narrow Control UI live-stream reconciliation fix with regression coverage for text/tool/text turns.

## Reproduction Assessment

Yes. Source inspection shows the gateway broadcasts cumulative assistant live snapshots while the UI renders earlier committed stream segments plus the current stream without trimming already-rendered text.

## Solution Assessment

Yes for the fix boundary. This belongs in Control UI live stream/tool artifact reconciliation, not gateway persistence, because the report and source both show persisted history is correct while live render state duplicates.
