## Summary

Keep open. Current main still has the Control UI path that can render cumulative assistant text across text/tool/text boundaries, a fresh user comment reports the same behavior on v2026.5.12, and the canonical UI-side fix remains open at https://github.com/openclaw/openclaw/pull/74498.

## What This Changes

The issue reports WebChat live rendering repeating cumulative assistant text cards after tool-call boundaries instead of showing each new assistant text block once.

## Best Possible Solution

Land a rebased UI-side committed-prefix projection like https://github.com/openclaw/openclaw/pull/74498, with focused text-to-tool-to-text regression coverage, while preserving full-snapshot gateway deltas for ACP, TUI, and generic subscribers.

## Reproduction Assessment

Yes. Current main is source-reproducible: a full-snapshot assistant message is assigned to `chatStream`, a tool event commits the prior stream into `chatStreamSegments`, and the renderer displays both the committed segment and active full snapshot.

## Solution Assessment

Yes. The best current direction is a UI-only committed-prefix offset/slice, because changing shared gateway delta semantics would risk ACP, TUI, and other subscribers that rely on monotonic full snapshots.
