## Summary

Keep open. Current main still has the Control UI/WebChat source path that can render cumulative text around tool boundaries, and the linked implementation PRs are open rather than merged proof.

## What This Changes

Fix Control UI/WebChat live rendering so assistant text after tool calls appears as fresh cards instead of cumulative duplicate cards.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/74498 or an equivalent UI-side committed-prefix reconciliation with regression tests, while preserving gateway full-snapshot chat deltas for other consumers.

## Reproduction Assessment

Yes. Source inspection on current main gives a deterministic text -> tool -> text path: delta sets `chatStream`, tool start copies it into `chatStreamSegments`, the next cumulative delta overwrites `chatStream`, and `buildChatItems` renders both.

## Solution Assessment

Yes for the fix direction. This should be solved in Control UI committed-prefix reconciliation rather than by changing gateway delta semantics, because gateway tests codify monotonic full-snapshot payloads.
