## What This Changes

The issue requests that Control UI/WebChat render assistant text after tool calls as fresh non-cumulative cards while keeping persisted transcript blocks unchanged.

## Best Possible Solution

Land or supersede https://github.com/openclaw/openclaw/pull/74498 with a UI-side committed-prefix projection and focused regression coverage while preserving full-snapshot gateway deltas for ACP, TUI, and generic subscribers.

## Reproduction Assessment

Yes. Current main is source-reproducible with high confidence: full-snapshot assistant text is assigned to `chatStream`, committed into `chatStreamSegments` on tool start, and rendered again as the next active stream.

## Solution Assessment

Yes for the current direction. A Control UI committed-prefix slice is the narrow maintainable fix because changing gateway delta semantics would affect ACP, TUI, and other full-snapshot consumers.
