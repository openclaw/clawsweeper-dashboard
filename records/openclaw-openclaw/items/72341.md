## Summary

Keep open. Current main still has the source-level path that can render cumulative Control UI/WebChat assistant text across tool boundaries, and the active UI-side fix remains unmerged.

## What This Changes

Control UI/WebChat should render assistant text segments around tool calls as fresh live cards instead of duplicating earlier text in later cards.

## Best Possible Solution

Land a rebased UI-side committed-prefix projection, such as https://github.com/openclaw/openclaw/pull/74498 or an equivalent patch, with focused regression tests while preserving full-snapshot gateway deltas for ACP, TUI, and generic subscribers.

## Reproduction Assessment

Yes. Current main is source-reproducible with high confidence: full-snapshot assistant text is assigned to chatStream, committed into chatStreamSegments on tool start, and rendered again as the next active stream.

## Solution Assessment

Yes for the current direction. The narrow maintainable fix is Control UI committed-prefix tracking/slicing, because changing gateway delta semantics would affect ACP, TUI, and other full-snapshot consumers.
