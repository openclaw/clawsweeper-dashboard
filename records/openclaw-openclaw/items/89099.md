## Summary

Keep open: current main still has the asymmetric `waitForAgentJob()` caller-timeout fallback, and this PR contains a useful unique repair because it forwards only hard timeout snapshots and adds hard/soft regression coverage. It should not be closed as superseded by the later related PRs because https://github.com/openclaw/openclaw/pull/92204 is a narrower unproven duplicate that closes the canonical issue, while https://github.com/openclaw/openclaw/pull/89367 is a stronger landing candidate but does not clearly cover the same `providerStarted`-based hard-timeout contract as this PR.

## What This Changes

The PR changes `waitForAgentJob()` to return pending hard-timeout snapshots from the caller-timeout fallback and adds two regression tests for hard-timeout preservation and soft-timeout retry grace.

## Best Possible Solution

Land the outcome-based wait-layer fix after real behavior proof or maintainer acceptance of bounded proof, while keeping the zero-output delivery work tracked in https://github.com/openclaw/openclaw/issues/89095 unless a separate fix covers it.
