## What This Changes

The PR changes `waitForAgentJob()` to return pending hard-timeout snapshots from the outer timer fallback and adds regression tests for hard-timeout preservation and soft-timeout retry grace.

## Best Possible Solution

## Solution Assessment

Yes for the scoped wait-layer bug: using the shared terminal-outcome hard-timeout classifier is a narrow maintainable fix and stronger than the phase-only gate in https://github.com/openclaw/openclaw/pull/89367 or the ungated fallback in https://github.com/openclaw/openclaw/pull/92204.
