## Summary

## What This Changes

Fix Weixin reconfiguration so gateway reload reconnects the saved account without reinstalling the tracked plugin or requiring another QR scan.

## Best Possible Solution

Land a narrow lifecycle fix that restarts or explicitly reloads the gateway only when the running process lacks the installed channel plugin, then prove saved Weixin auth reconnects without a second QR scan.

## Reproduction Assessment

Yes at source level with medium confidence: the exact CLI warning follows a failed `channels.start`, and current main rejects channels missing from the running registry. A live QR-authenticated Weixin reproduction was not performed.

## Solution Assessment

No, the suggested generic installer fallback is not the best fix because existing-target rejection is intentional and tested; the narrower gateway lifecycle fallback in https://github.com/openclaw/openclaw/pull/90779 is the better direction, pending live proof.
