## Summary

Keep open. Current main still has the source-level `channels.start` registry-miss failure, and the related fix candidate at https://github.com/openclaw/openclaw/pull/90779 is open and unmerged; the reporter’s suggested global installer fallback also conflicts with an intentional install-mode invariant.

## What This Changes

The issue reports that reconfiguring the Weixin channel triggers gateway reload but leaves the channel offline because the running gateway rejects `channels.start` for the plugin channel.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/90779 or a narrow replacement that recovers only when the running gateway lacks the installed channel plugin, with live Weixin proof that saved auth reconnects without a second QR scan.

## Reproduction Assessment

Yes at source level with medium confidence: current main still calls `channels.start` after local auth and the gateway rejects absent runtime registry entries. I did not run a live QR-authenticated Weixin reconfigure flow.

## Solution Assessment

No, the suggested generic installer fallback is not the best fix because existing-target rejection is intentional and tested. A narrower gateway lifecycle fallback or reload decision for unloaded channel plugins is the safer path.
