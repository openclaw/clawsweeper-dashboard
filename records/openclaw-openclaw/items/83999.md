## Summary

Keep open: current main still has generic unknown-command handling in both CLI parse-error formatting and the early unowned-root path, and no duplicate or existing helper appears to cover closest-command suggestions.

## What This Changes

Request adding CLI typo and alias suggestions, such as `upate` or `upgrade` pointing to `openclaw update`, for unknown root commands.

## Best Possible Solution

Add a shared pure command-suggestion helper used by both CLI parse errors and early unowned-root diagnostics, while leaving plugin ownership and allowlist messages authoritative.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source and existing tests show current main emits generic unknown-command output without closest-command hints.

## Solution Assessment

Yes, the proposed shared helper is the narrow maintainable solution if maintainers want this UX. It should append suggestions only after existing plugin ownership diagnostics are resolved.

## Repair Work Prompt

Implement closest-command suggestions for unknown OpenClaw CLI root commands. Add a small pure helper under `src/cli/program/` that supports explicit aliases such as `upgrade -> update`, computes close matches from core command names plus sub-CLI roots, and returns no suggestion for weak matches. Wire it into `formatCliParseErrorOutput()` and `resolveUnownedCliPrimaryMessage()` without auto-running commands and without weakening plugin ownership or `plugins.allow` diagnostics. Add regression coverage for `upate -> update`, `upgrade -> update`, weak non-matches, and preservation of existing plugin-policy diagnostics. Because this is user-visible CLI behavior, include a concise `CHANGELOG.md` entry under the active version. Do not add a new dependency for edit distance.
