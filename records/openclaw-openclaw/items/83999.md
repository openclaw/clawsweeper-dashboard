## Summary

Keep open: current main still reports unknown root commands with generic help/plugin diagnostics and has no closest-command suggestion path. The adjacent tool-name diagnostic work in https://github.com/openclaw/openclaw/issues/77214 and https://github.com/openclaw/openclaw/pull/77220 does not supersede this typo/alias feature.

## What This Changes

The issue requests CLI suggestions for unknown root commands, including typo matches like `upate` and semantic aliases like `upgrade` to `update`.

## Best Possible Solution

Add a small shared command-suggestion helper used by both unknown-root paths, sourcing candidates from core/sub-CLI descriptors while preserving plugin ownership diagnostics and never auto-running corrections.

## Repair Work Prompt

Implement unknown root command suggestions for OpenClaw CLI. Add a pure helper, likely under `src/cli/program/command-suggestions.ts`, that combines `getCoreCliCommandNames()` and `getSubCliEntries().map(entry => entry.name)`, supports a small explicit alias map such as `upgrade -> update`, computes close typo matches without adding a dependency, and returns only close suggestions. Wire it into both `formatCliParseErrorOutput()` and the early `resolveUnownedCliPrimaryMessage()` path, preserving existing plugin command, tool-owner, and `plugins.allow` diagnostics and never auto-running corrected commands. Add focused regression coverage for `upate -> update`, `upgrade -> update`, weak matches with no suggestion, and unchanged plugin-policy diagnostics. Do not add config options or new dependencies.
