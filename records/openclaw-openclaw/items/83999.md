## Repair Work Prompt

Implement closest-command suggestions for unknown root CLI commands. Add a small pure helper under `src/cli/program/` that uses an explicit conservative alias map such as `upgrade -> update`, computes close matches against `getCoreCliCommandNames()` plus `getSubCliEntries().map(entry => entry.name)`, and returns a small deterministic suggestion list without adding dependencies. Wire it into `formatCliParseErrorOutput()` and `resolveUnownedCliPrimaryMessage()` so `upate` and `upgrade` suggest `openclaw update`, weak matches do not suggest, no command is auto-run, and existing plugin allowlist/ownership diagnostics remain intact. Add or update focused tests in `src/cli/program/error-output.test.ts` and `src/cli/run-main.exit.test.ts`; do not change plugin policy semantics or add a config option. This is user-visible CLI UX, so include a changelog entry if the maintainer lane expects one for feature PRs.
