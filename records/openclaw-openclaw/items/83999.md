Add a small pure helper, likely under `src/cli/program/command-suggestions.ts`, that uses `getCoreCliCommandNames()` plus `getSubCliEntries().map(entry => entry.name)`, includes a tiny explicit alias map such as `upgrade -> update`, computes close matches without adding a dependency, and returns only high-confidence suggestions. Wire it into `formatCliParseErrorOutput()` and the early `resolveUnownedCliPrimaryMessage()` path without auto-running or changing command semantics. Preserve existing plugin ownership and `plugins.allow` diagnostics as authoritative; suggestions should only append where they do not obscure those messages. Add focused regression coverage for `upate -> update`, `upgrade -> update`, weak matches with no suggestion, and preservation of plugin-policy diagnostics. Include a single user-facing changelog entry for the CLI ergonomics improvement.
