## Best Possible Solution

## Reproduction Assessment

## Solution Assessment

Yes: the shared harness prompt-build helper is the narrowest maintainable fix point because Codex and Copilot harness callers already consume its returned prompt. A Codex protocol change would be the wrong layer, and duplicating the hook call in each harness caller would create lifecycle drift.
