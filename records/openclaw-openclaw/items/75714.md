## Summary

Keep this PR open. Current main still lets MCP `EmbeddedResource` tool-result blocks cross into Pi tool-result content unchanged, and the PR’s boundary-level normalization is the right narrow fix direction with focused tests and changelog coverage.

## What This Changes

The branch normalizes bundle-MCP tool-result `resource` blocks into Pi text content, adds materialization regression tests, and records the user-facing fix in `CHANGELOG.md`.

## Best Possible Solution

Land this boundary-level normalization after maintainer review and CI, keeping provider transports generic and tracking broader MCP block support separately if maintainers want it.

## Reproduction Assessment

Yes. A source-level reproduction is clear: a bundle-MCP `CallToolResult` with `content: [{ type: "resource", resource: { text: "..." } }]` is cast through unchanged on current main, then provider converters that only read top-level text/image content lose the payload.

## Solution Assessment

Yes. Normalizing at the bundle-MCP to Pi `AgentToolResult` boundary is the narrowest maintainable fix because it removes MCP-specific content before all provider transports and replay paths that expect Pi text/image content.
