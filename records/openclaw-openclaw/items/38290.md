## Summary

Keep open. Current main still rejects exact `chrome-extension://<id>` allowlist entries, and this PR is a focused fix with sufficient real behavior proof; the remaining work is maintainer review, conflict/rebase handling, and choosing between this PR and the overlapping custom-scheme PR.

## What This Changes

The PR changes Gateway browser-origin matching so exact opaque/custom-scheme origins normalize to `scheme://host` for allowlist comparison and adds unit plus WebSocket handshake tests.

## Best Possible Solution

Land this exact-origin normalization fix or the overlapping broader custom-scheme fix after rebase and current validation, while keeping scheme wildcard support in the separate policy discussion.
