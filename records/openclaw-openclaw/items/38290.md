## Summary

## What This Changes

The PR changes Gateway browser-origin matching so opaque schemes can normalize to `scheme://host`, normalizes configured `allowedOrigins` through that parser, and adds unit plus WebSocket handshake tests.

## Best Possible Solution

Land a refreshed narrow parser fix that applies `scheme://host` fallback only to incoming opaque origins, keeps configured allowlist entries as exact trimmed lowercase strings, adds fail-closed regression tests, and includes redacted runtime proof.

## Reproduction Assessment

Yes, source-reproducible: current main compares configured entries against `URL.origin`, and a Node probe shows the reported opaque/custom schemes serialize that origin as `null` before the exact allowlist check.

## Solution Assessment

No. The parser-local fallback direction is right, but this branch also reparses configured allowlist values and can authorize base origins that currently fail closed.
