## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Keep configured origins exact:** `src/gateway/origin-check.ts:47`
  - body: `normalizeAllowedOrigin()` reparses every configured `allowedOrigins` value, so a config like `https://trusted.example/app` silently becomes `https://trusted.example` and starts allowing an origin that currently fails closed. This fix only needs to normalize incoming opaque Origins; keep configured entries as exact trimmed/lowercased strings unless maintainers explicitly choose a config compatibility change.
  - confidence: 0.88
- **[P2] Reject empty-host opaque origins:** `src/gateway/origin-check.ts:17`
  - body: The fallback returns `protocol//host` even when `url.host` is empty, which makes values like `file://` newly matchable if configured. The reported extension and app origins all have hosts, so gate the opaque fallback on a non-empty host and keep empty-host schemes failing closed.
  - confidence: 0.78
