Overall correctness: patch is incorrect

Full review comments:

- **[P1] Keep configured origins exact:** `src/gateway/origin-check.ts:60`
  - body: `normalizeAllowedOrigin()` reparses every configured value, so an existing entry like `https://trusted.example/app` silently becomes `https://trusted.example` and starts authorizing the whole origin instead of failing closed. This fix only needs hosted opaque origins to become matchable when explicitly configured; preserve existing exact configured-string semantics unless maintainers intentionally approve a compatibility change.
  - confidence: 0.88
- **[P2] Reject empty-host opaque origins:** `src/gateway/origin-check.ts:17`
  - body: The fallback returns `${url.protocol}//${url.host}` even when `url.host` is empty, making values like `file://` newly matchable if configured. The reported extension and app origins all have hosts, so gate the opaque fallback on a non-empty host and keep empty-host schemes failing closed.
  - confidence: 0.8
