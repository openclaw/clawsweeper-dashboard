  - body: `normalizeAllowedOrigin()` reparses every configured `allowedOrigins` value, so a config like `https://trusted.example/app` silently becomes `https://trusted.example` and starts allowing an origin that current main fails closed.   - confidence: 0.88

  - body: The fallback returns `protocol//host` even when `url.host` is empty, which makes values like `file://` newly matchable if configured. The reported extension and app origins all have hosts, so gate the opaque fallback on a non-empty host and keep empty-host schemes failing closed.
