## Summary

Keep open: current main still has the exact-origin mismatch, and this PR remains a narrow, valid fix with sufficient real behavior proof and no blocking code or security findings.

## What This Changes

The branch adds symmetric normalization for non-standard Gateway browser origins and adds unit plus WebSocket handshake regression coverage.

## Best Possible Solution

Rebase and land this exact-origin normalization with the focused origin-check and WebSocket regression tests; keep scheme wildcards in the separate policy issue.

## Reproduction Assessment

Yes, at source level: current main compares allowlist entries against URL.origin, and Node returns "null" for chrome-extension:// origins. The WebSocket handshake routes through that checker, so an exact configured extension origin is rejected before this patch.

## Solution Assessment

Yes. Symmetric normalization of configured and incoming opaque exact origins is the narrow maintainable fix; broader chrome-extension://* matching should stay in https://github.com/openclaw/openclaw/issues/39739.
