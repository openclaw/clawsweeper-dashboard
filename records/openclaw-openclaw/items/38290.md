## Summary

Keep open. Current main and the latest release still reject exact custom-scheme browser origins, and this PR targets the right Gateway path, but the proposed allowlist normalization introduces a blocking exact-match regression that should be repaired before merge.

## What This Changes

The PR changes Gateway origin normalization and tests so exact `chrome-extension://<id>` browser origins can match `gateway.controlUi.allowedOrigins`.

## Best Possible Solution

Land one consolidated Gateway parser fix that preserves exact allowlist semantics, covers allowlisted and rejected hosted custom-scheme origins, adds a regression for path-containing allowlist entries, and leaves scheme wildcards to https://github.com/openclaw/openclaw/issues/39739.

## Reproduction Assessment

Yes, at source level. Current main parses `chrome-extension://...` through `URL.origin === "null"` and then compares that `null` string against explicit `gateway.controlUi.allowedOrigins` entries.

## Solution Assessment

No, not as written. The custom-scheme fallback is the right direction, but configured allowlist entries should not be normalized by stripping paths or other URL components from standard origins.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve exact allowlist semantics:** `src/gateway/origin-check.ts:44`
  - body: Parsing configured allowlist entries with `parseOrigin()` changes existing exact matching: `https://trusted.example/path` now normalizes to `https://trusted.example` and will allow that whole origin, where current main rejects the non-origin string. Keep the fallback for opaque incoming origins, but either require configured entries to already be exact origins or preserve the previous exact-string behavior for standard schemes.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: The diff fixes a real origin-allowlist bug but currently broadens misconfigured standard URL allowlist entries into full-origin grants.

Concerns:

This is a trust-boundary change in Gateway browser-origin policy and should be tightened before merge.
