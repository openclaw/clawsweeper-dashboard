## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Keep configured origins exact:** `src/gateway/origin-check.ts:47`
  - body: `normalizeAllowedOrigin()` reparses every configured `allowedOrigins` value and compares `URL.origin`, so a config such as `https://trusted.example/app` silently becomes `https://trusted.example` and starts allowing that browser origin even though the previous exact config failed closed. The fix only needs to normalize the incoming opaque Origin; keep configured entries as exact trimmed/lowercased strings, or add explicit validation/migration if path-bearing entries should change behavior.
  - confidence: 0.88
- **[P2] Reject empty-host opaque origins:** `src/gateway/origin-check.ts:17`
  - body: The fallback returns `protocol//host` even when `url.host` is empty, which makes configured values like `file://` or `data://` reachable through crafted Origin headers. The reported extension and app schemes all have hosts, so gate the fallback on a non-empty host and keep empty-host opaque origins failing closed.
  - confidence: 0.78
