  - body: `normalizeAllowedOrigin()` reparses every configured value, so an existing entry like `https://trusted.example/app` silently becomes `https://trusted.example` and starts authorizing a whole origin that current main fails closed.   - confidence: 0.88
- **[P2] Reject empty-host opaque origins:** `src/gateway/origin-check.ts:17-26`
  - body: The fallback returns `${url.protocol}//${url.host}` even when `url.host` is empty, making values like `file://` newly matchable if configured. The reported extension and app origins all have hosts, so gate the opaque fallback on a non-empty host and keep empty-host schemes failing closed.
  - confidence: 0.8
