## Summary

Keep open: current main still has the exact extension-origin allowlist bug, and this PR is the right narrow fix direction, but it is missing the required user-facing changelog entry before merge.

## What This Changes

The PR normalizes non-standard browser origins for exact `gateway.controlUi.allowedOrigins` matching and adds origin-check plus Gateway WebSocket handshake regression tests.

## Best Possible Solution

Land this exact-origin normalization with focused unit and handshake coverage, add the missing changelog entry, and keep scheme-level wildcard support tracked separately in #39739.

## Reproduction Assessment

Yes. Source inspection plus a Node URL check show current main compares configured origins against `URL.origin`, which is `"null"` for `chrome-extension://<id>`, so an exact allowlist entry cannot match.

## Solution Assessment

Yes. Normalizing both configured and incoming opaque-origin URLs to exact `scheme://host` matches is the narrowest maintainable fix; it preserves literal `Origin: null` rejection and avoids wildcard expansion.
