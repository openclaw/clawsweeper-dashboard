## Summary

The exact custom-scheme origin bug is still present on current main and this PR is useful, but it is not merge-ready because the patch broadens configured allowlist semantics, makes empty-host opaque schemes matchable, and is currently conflicting with main.

## What This Changes

The PR changes Gateway browser-origin matching so exact `chrome-extension://<id>` origins can be allowed in `gateway.controlUi.allowedOrigins`, with unit and WebSocket handshake tests.

## Best Possible Solution

Land a current-main origin-check fix that normalizes only incoming hosted opaque origins, keeps configured allowlist strings exact except `*`, rejects empty-host opaque origins, and covers the WebSocket and HTTP callers with focused tests.

## Reproduction Assessment

Yes, source-reproducible. Current main compares configured entries against `URL.origin`, and the Node probe shows hosted custom schemes serialize that origin as `null` before exact allowlist matching.

## Solution Assessment

No, not as written. The parser-local fallback is the right layer, but this PR should preserve exact configured allowlist strings and reject empty-host opaque origins before merge.

- **[P1] Keep configured origins exact:** `src/gateway/origin-check.ts:47`
  - body: `normalizeAllowedOrigin()` reparses every configured `allowedOrigins` value, so a config like `https://trusted.example/app` silently becomes `https://trusted.example` and starts allowing an origin that current main fails closed. This fix only needs to normalize incoming opaque `Origin` values; keep configured entries as exact trimmed/lowercased strings unless maintainers explicitly choose a compatibility change.
  - confidence: 0.88
- **[P2] Reject empty-host opaque origins:** `src/gateway/origin-check.ts:17`
  - body: The fallback returns `protocol//host` even when `url.host` is empty, which makes values like `file://` newly matchable if configured. The reported extension and app origins all have hosts, so gate the opaque fallback on a non-empty host and keep empty-host schemes failing closed.
  - confidence: 0.78
