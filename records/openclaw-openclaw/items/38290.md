## Summary

Keep open: this PR addresses a valid current and shipped Gateway origin-allowlist bug, and current main/latest release still normalize extension origins to "null" before allowlist comparison.

## What This Changes

The branch changes Gateway browser-origin matching to compare opaque extension origins by `scheme://host` and adds unit plus WebSocket handshake regression tests.

## Best Possible Solution

Rebase and land the exact opaque-origin normalization with focused origin-check and WebSocket handshake coverage, while keeping scheme-wildcard semantics in the separate policy issue.

## Reproduction Assessment

Yes, at source level: current main and `v2026.5.7` compare `allowedOrigins` against `url.origin`, and Node returns "null" for `chrome-extension://` origins. I did not run Gateway tests because this review is read-only.

## Solution Assessment

Yes. Symmetric exact-origin normalization for opaque schemes is the narrow maintainable fix; broader `chrome-extension://*` semantics should remain in https://github.com/openclaw/openclaw/issues/39739.
