## Summary

## What This Changes

The branch changes Gateway browser-origin allowlist normalization so exact `chrome-extension://<id>` origins can match `gateway.controlUi.allowedOrigins`, with origin-check and WebSocket handshake tests.

## Best Possible Solution

Land one refreshed narrow Gateway origin-check fix that preserves exact configured allowlist semantics, accepts only explicitly configured hosted non-special origins, rejects empty-host opaque origins, and keeps wildcard, auth, host-header fallback, and broader scheme-wildcard policy unchanged.

## Solution Assessment

No, not as written. The parser-local fallback is the right layer, but the PR should preserve exact configured allowlist strings and reject empty-host opaque origins before merge.
