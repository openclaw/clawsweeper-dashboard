## Summary

Keep open: current main still has the exact opaque-origin allowlist bug, and this PR is a narrow, reviewable fix with no blocking correctness findings. It still needs normal maintainer rebase, validation, and landing because the fix is not on main yet.

## What This Changes

The PR changes gateway browser-origin allowlist matching to normalize opaque origins with a protocol/host fallback and adds unit plus WebSocket auth tests for exact chrome-extension allowlisting.

## Best Possible Solution

Land this PR after rebase, or land an equivalent narrow gateway patch that symmetrically normalizes incoming origins and configured allowlist entries while keeping scheme wildcard support separate in #39739.

## Solution Assessment

Yes. The PR is the narrowest maintainable fix I see because it normalizes both sides consistently, preserves exact allowlist semantics, and does not add wildcard handling or change token/device auth.
