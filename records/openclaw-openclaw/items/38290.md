## Summary

## What This Changes

This PR changes Gateway browser-origin allowlist normalization to match exact opaque-scheme origins via `scheme://host` and adds unit plus WebSocket handshake tests.

## Best Possible Solution

Land this exact-origin normalization fix or the overlapping broader custom-scheme fix after rebase and current validation, while keeping scheme wildcard support in the separate policy discussion.

Yes. The source path is high-confidence: Node gives `chrome-extension://...` an opaque `URL.origin` of `"null"`, and current main compares that parsed value to exact `gateway.controlUi.allowedOrigins` entries.

## Solution Assessment

Yes. Normalizing opaque origins to exact `scheme://host` values on both sides preserves explicit allowlist enforcement without adding wildcard, host-header fallback, token, or device-auth changes.
