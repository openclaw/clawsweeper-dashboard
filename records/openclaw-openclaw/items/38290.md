Keep open. Current main and the latest release still collapse exact extension/custom-scheme origins to the opaque `URL.origin` value, while this branch is a narrow fix with real behavior proof; the remaining blocker is maintainer consolidation/rebase with overlapping exact-origin work.

The branch changes Gateway browser-origin matching to normalize opaque scheme origins for exact allowlist matching and adds unit plus WebSocket handshake coverage.

## Best Possible Solution

## Solution Assessment

Yes. Symmetric normalization of configured and incoming opaque origins is the narrow maintainable fix; broader wildcard/pattern matching should remain in the separate policy discussion.
