## Best Possible Solution

Land or replace the linked PR with a documented additive hook API that exposes current-turn tool calls behind `allowConversationAccess`, with focused tests and real hook-run proof.

## Solution Assessment

Broadly yes: an additive hook field behind the existing `allowConversationAccess` boundary is the right direction. The exact public field semantics should be settled in the linked PR before this issue closes.
