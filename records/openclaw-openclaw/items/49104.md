## Summary

Keep open: current main has Telegram HTML escaping hardening, but the remaining strict reasoning-tag path still intentionally drops unclosed `<think>` tails after visible text, and an open linked PR with closing syntax needs maintainer policy review before this issue can close.

## What This Changes

The issue asks OpenClaw to prevent Telegram `parse_mode=HTML` delivery from truncating agent replies that contain literal angle-bracket tags such as `<think>`.

## Best Possible Solution

Keep the shipped HTML escaping, then make an explicit strict unclosed-tail policy decision and land only a narrow shared-text plus Telegram regression fix if it prevents answer loss without leaking private reasoning.

## Reproduction Assessment

Yes, source-level. Current main routes Telegram final text through strict reasoning stripping, and current tests assert that visible text before an unclosed reasoning tail is preserved while the tail is dropped.

## Solution Assessment

No close is justified yet. The formatter-side HTML escape path is hardened, but the remaining unclosed-tail behavior requires a maintainer decision about reasoning privacy versus visible answer recovery.
