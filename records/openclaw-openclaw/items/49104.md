## Summary

Keep open: current main fixes the Telegram HTML escaping side, but it still deliberately drops text after an unclosed reasoning tag when visible text already exists, and https://github.com/BingqingLyu/openclaw/pull/1016 remains the open linked fix path that needs maintainer policy review.

## What This Changes

The issue asks Telegram delivery to preserve agent responses containing literal angle-bracket tags instead of truncating or suppressing text under HTML parse mode and strict reasoning-tag stripping.

## Best Possible Solution

Keep the landed Telegram HTML escaping, then make an explicit strict unclosed-tail policy and land a narrow upstream fix only if it prevents Telegram answer loss without leaking private reasoning.

## Reproduction Assessment

Yes, source-level. Current main routes Telegram final text through strict reasoning stripping, and current tests assert that visible text followed by an unclosed `<think>` tag drops the tail rather than recovering it.

## Solution Assessment

No close is justified yet. The HTML parse-mode escaping path is hardened, but the remaining unclosed-tail fix is a reasoning privacy policy decision rather than a purely mechanical bug fix.
