## Summary

Keep open: current main and the latest v2026.6.8 release still preserve the strict-mode behavior that drops text after an unclosed reasoning tag when visible text already exists. ## What This Changes

The issue requests that Telegram deliver full agent replies containing literal angle-bracket tags by escaping, sanitizing, or otherwise preventing HTML-mode truncation and strict-strip message loss.

## Best Possible Solution

Keep this issue as the canonical tracker, then land or replace the linked candidate with a narrow strict-tail recovery fix that preserves the strict privacy boundary and proves Telegram delivery with focused shared-text, Telegram splitter, formatter/delivery, and live Telegram evidence.
