## Summary

Keep open: current main and v2026.5.22 have fixed the general Telegram HTML escaping/fallback path, but strict reasoning-tag stripping still drops text after an unclosed reasoning tag when visible answer text already exists before it. The linked open fork PR owns that remaining policy-sensitive behavior, so this issue should stay open until maintainers decide and land or reject that path.

## What This Changes

Ensure Telegram delivers full agent responses containing literal angle-bracket tags by escaping unsupported HTML and avoiding strict reasoning-tag tail drops.

## Best Possible Solution

Keep the landed Telegram HTML escaping and fallback behavior, then make an explicit maintainer decision for strict unclosed reasoning-tag tails and land a narrow fix only if it prevents Telegram answer loss without exposing private reasoning.

## Reproduction Assessment

Yes, source-level: current Telegram answer splitting still uses strict reasoning stripping, and the current tests pin a visible-prefix plus unclosed `<think>` case that drops the tail. I did not run tests because this review is read-only.

## Solution Assessment

No close is justified yet. The HTML formatter side is addressed, but the remaining strict unclosed-tail behavior needs a maintainer privacy decision before a narrow fix should land.
