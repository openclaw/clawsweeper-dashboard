The issue requests preventing Telegram replies containing `<think>` or other angle-bracket reasoning tags from being truncated or suppressed during delivery.

## Best Possible Solution

Maintainers should choose the strict-mode privacy boundary, then land a narrow sanitizer or Telegram split fix that preserves true hidden reasoning while preventing visible answer loss, with focused shared-text and Telegram tests.

## Reproduction Assessment

Yes, at source level. Current Telegram answer splitting uses strict reasoning-tag stripping, and the current tests explicitly preserve the behavior where an unclosed `<think>` after visible text drops the tail.

## Solution Assessment

Unclear until maintainers decide the privacy boundary. The raw Telegram HTML escaping surface is partly hardened on main, but the remaining proposed tail-recovery fix changes strict-mode reasoning visibility semantics.
