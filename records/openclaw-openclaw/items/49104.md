## Summary

Keep open: current main has improved Telegram HTML escaping, but the remaining source-reproducible message-loss path is still present when strict reasoning stripping hides unclosed `<think>` tails after visible text. The linked open fix PR targets this issue but is currently conflicting and needs maintainer judgment on the strict-mode privacy contract.

## What This Changes

The issue asks Telegram delivery to stop losing agent reply text when literal angle-bracket tags such as `<think>` reach HTML-mode delivery or reasoning-tag stripping.

## Best Possible Solution

Decide the strict-mode privacy contract, then land a narrow fix or replacement for the linked PR that prevents visible Telegram answer loss while preserving actual reasoning privacy, with focused shared sanitizer and Telegram split tests.

## Reproduction Assessment

Yes, source-level. Current code routes Telegram answer text through strict reasoning-tag stripping, and the current test suite explicitly expects an unclosed `<think>` after visible text to drop the tail; I did not run tests because this was a read-only review.

## Solution Assessment

Unclear until maintainers decide the privacy boundary. The best technical location appears to be the shared reasoning-tag sanitizer or Telegram reasoning split path, not raw Telegram HTML formatting alone, but recovering every unclosed strict-mode tail may leak intended hidden reasoning.
