## Summary

Keep open. Current main covers raw Telegram HTML escaping and code-formatted literal tags, but it still intentionally drops unclosed reasoning-tag tails after visible text; the prior target-repo PRs were closed unmerged and the remaining behavior needs maintainer policy/design.

## What This Changes

The issue asks OpenClaw to prevent Telegram replies from being truncated when agent text contains literal angle-bracket tags such as `<think>`.

## Best Possible Solution

Keep this issue as the canonical tracker until maintainers choose a final-message policy, then land a narrow fix that preserves strict reasoning redaction while preventing user-visible Telegram answer loss for true literal-tag text.

## Reproduction Assessment

Yes. The source-level reproduction is `splitTelegramReasoningText("Before <think>unclosed content after")`, which reaches strict stripping and is pinned by current tests to keep only `Before`.

## Solution Assessment

No current close is justified. Raw HTML escaping is already handled, but the proposed broad/unconditional recovery approaches are not the best fix because they conflict with the documented strict reasoning-redaction policy.
