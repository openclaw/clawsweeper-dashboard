## Summary

Keep open: current main and the latest release already harden Telegram HTML escaping, but the remaining source path still drops an unclosed strict-mode `<think>` tail after visible answer text. The linked fork PR targets the issue, but it is open, conflicting, and changes the strict-mode privacy boundary, so this needs maintainer direction rather than cleanup closure.

## What This Changes

The issue asks Telegram delivery to preserve visible angle-bracket text instead of losing reply content when HTML parsing or reasoning-tag stripping encounters tags such as `<think>`.

## Best Possible Solution

Maintainers should decide the strict-mode privacy semantics, then either land a narrow regression-tested Telegram/shared-sanitizer fix that preserves visible answers without leaking genuine reasoning or explicitly document the current strict behavior as intentional.

## Reproduction Assessment

Yes, at source level. Current Telegram answer splitting uses strict reasoning-tag stripping, and checked-in tests explicitly preserve the unclosed-after-visible-text tail-drop behavior.

## Solution Assessment

Unclear until maintainers decide the privacy boundary. Raw Telegram HTML escaping is already in the right layer for non-reasoning tags, but the remaining `<think>` tail recovery would change current strict-mode reasoning visibility semantics.
