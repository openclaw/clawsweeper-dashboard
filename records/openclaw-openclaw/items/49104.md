## Summary

Keep open. Current main has fixed the plain Telegram HTML escaping path, but the remaining unclosed reasoning-tag tail-drop is still present and intentionally pinned by tests, so this needs a maintainer decision rather than cleanup closure.

## What This Changes

The issue asks OpenClaw to prevent Telegram replies from losing text when angle-bracket tags such as `<think>` appear in assistant output.

## Best Possible Solution

Keep this as the canonical tracker until maintainers choose the final-answer strict-mode policy, then land a narrow shared-text and Telegram regression fix that prevents message loss without leaking reasoning content.

## Reproduction Assessment

Yes, source-level. Current main routes Telegram final text through strict reasoning stripping, and the test for `Before <think>unclosed content after` pins the delivered answer text to `Before` rather than recovering the tail.

## Solution Assessment

No close is justified. Formatter-level HTML escaping is already in place, but the remaining strict-mode recovery changes a reasoning-redaction contract and needs maintainer policy before implementation.
