## Summary

Keep open: current main has Telegram formatter hardening, but the remaining strict reasoning-tag path still drops text after an unclosed `<think>` when visible text already exists, so this is not safe to close as implemented.

## What This Changes

The issue asks OpenClaw to prevent Telegram replies from losing response text when angle-bracket model output such as `<think>` interacts with HTML formatting and reasoning-tag stripping.

## Best Possible Solution

Keep the HTML escaping hardening, then make an explicit maintainer decision for strict-mode unclosed reasoning tails and land a clean narrow fix only if it can prevent visible answer loss without leaking private reasoning.

## Reproduction Assessment

Yes, source-level. Current main routes Telegram final text through strict reasoning stripping, and current tests assert that a visible-prefix unclosed reasoning tail is dropped rather than recovered.

## Solution Assessment

No close is justified. Current main is a partial fix: unsupported HTML is escaped, but the remaining unclosed-tail behavior is a reasoning-redaction policy choice that needs maintainer review.
