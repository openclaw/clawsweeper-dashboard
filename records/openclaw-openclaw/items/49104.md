## Summary

Keep open. Current main still has the strict-mode unclosed reasoning-tag tail-drop behavior on Telegram final text, and the issue is paired with open PR #49996 using closing syntax, so cleanup should wait for maintainer review, replacement, or closure of that implementation candidate.

## What This Changes

The issue asks Telegram delivery to preserve or escape literal angle-bracket tags such as `<think>` so responses are not truncated or dropped under HTML parse mode and reasoning-tag stripping.

## Best Possible Solution

Keep this issue open until maintainers decide the strict-mode final-message policy, then land or replace #49996 with a narrow patch that prevents user-visible answer drops while preserving hidden reasoning and the existing Telegram HTML escaping path.

## Reproduction Assessment

Yes. The source-level reproduction is `splitTelegramReasoningText("Before <think>unclosed content after")`, which reaches strict stripping and is pinned by tests to keep only `Before`.

## Solution Assessment

Unclear. Recovering every unclosed-tag tail would address the data loss, but because current docs/tests treat that tail as hidden reasoning, the safer solution is an explicit final-message policy with targeted Telegram and shared-text coverage.
