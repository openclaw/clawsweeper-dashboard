## Summary

Keep open: current main and the latest release include Telegram HTML escaping, but strict reasoning-tag stripping still drops visible trailing text after an unclosed reasoning tag, and an open closing PR is already carrying the policy-sensitive fix path.

## What This Changes

The issue asks OpenClaw to stop losing Telegram reply text when assistant output contains angle-bracket tags that interact with Telegram HTML mode and reasoning-tag stripping.

## Best Possible Solution

Keep the formatter hardening, then make an explicit maintainer decision for strict-mode unclosed reasoning tails and land a narrow shared-text plus Telegram regression fix that prevents visible answer loss without leaking private reasoning.

## Reproduction Assessment

Yes, source-level. Current main routes Telegram final text through strict reasoning stripping, and current tests pin visible-prefix unclosed reasoning tails to be dropped rather than recovered.

## Solution Assessment

No close is justified. Current main is a partial fix: unsupported HTML is escaped, but the remaining strict-mode recovery behavior changes a reasoning-redaction contract and needs maintainer policy review.
