## Summary

Keep open: current main and v2026.5.22 have hardened the Telegram HTML escaping path, but current strict reasoning-tag handling still intentionally drops a trailing unclosed tag tail after visible text, and an open linked PR already owns that policy-sensitive remaining behavior.

## What This Changes

The issue requests that Telegram delivery avoid silently losing agent response text when angle-bracket reasoning-like tags interact with HTML formatting or strict reasoning-tag stripping.

## Best Possible Solution

Keep the landed Telegram HTML escaping behavior, then make an explicit maintainer decision for strict unclosed reasoning-tag tails and land a narrow fix only if it prevents Telegram answer loss without leaking private reasoning.

## Reproduction Assessment

Yes, source-level: current Telegram answer splitting still calls strict reasoning stripping, and the current test suite explicitly expects visible text followed by an unclosed `<think>` tag to drop the tail. I did not run tests because this review is read-only.

## Solution Assessment

No close is justified yet. The formatter-level HTML escaping side is solved, but changing strict unclosed-tail recovery is a privacy-sensitive behavior choice rather than a purely mechanical bug fix.
