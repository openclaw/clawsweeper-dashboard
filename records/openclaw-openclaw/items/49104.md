## Summary

Keep open: current main has Telegram HTML escaping hardening, but the remaining strict reasoning-tag path can still drop visible Telegram answer text after an unclosed `<think>` tail, and the linked open fix needs maintainer policy review before this issue can close.

## What This Changes

The issue reports Telegram reply truncation when model output contains angle-bracket reasoning-like tags that interact with HTML delivery or strict reasoning-tag stripping.

## Best Possible Solution

Keep the shipped HTML escaping, then make an explicit strict unclosed-tail policy and land a narrow fix only if it prevents Telegram answer loss without leaking private reasoning.

## Reproduction Assessment

Yes, source-level. Current main routes Telegram final text through strict reasoning stripping, and current tests assert that a visible-prefix unclosed reasoning tail is dropped rather than recovered.

## Solution Assessment

No close is justified yet. The HTML parse-mode escape path is hardened, but the remaining unclosed-tail fix is a reasoning privacy policy decision rather than a purely mechanical bug fix.
