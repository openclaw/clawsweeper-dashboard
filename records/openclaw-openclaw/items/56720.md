## Best Possible Solution

Land this PR or a replacement after preserving default `SPAM,TRASH,DRAFT,SENT` exclusions, fixing explicit empty-list semantics, and adding redacted real gateway or `gog` proof.

No as submitted. Threading excluded labels through config is maintainable, but explicit `[]` must either pass `--exclude-labels` with an empty value or the new empty-list documentation must be removed.
