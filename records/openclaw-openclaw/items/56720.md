## Summary

Keep open. The PR fixes a real current-main Gmail watcher regression, but the latest diff still breaks its documented empty-list opt-out because it omits `--exclude-labels` when `excludeLabels` is `[]`, which lets upstream `gog` fall back to its own `SPAM,TRASH` default.

## What This Changes

The PR restores Gmail watcher excluded-label args, adds `hooks.gmail.excludeLabels` config/schema/docs plumbing, and updates Gmail tests plus generated config metadata.

## Best Possible Solution

Land one canonical Gmail watcher fix that restores default `SPAM,TRASH,DRAFT,SENT` exclusions, preserves explicit empty-list opt-out with an empty `--exclude-labels` value, and keeps schema, docs, generated metadata, and tests aligned.

## Reproduction Assessment

Yes. Source inspection of current main shows the resolved Gmail watcher config and `buildGogWatchServeArgs` have no `excludeLabels` field or `--exclude-labels` branch, and the gateway watcher uses that builder.

## Solution Assessment

No, not as-is. The typed config/runtime/docs approach is maintainable, but the PR must pass `--exclude-labels` with an empty value for `excludeLabels: []` to match its docs and upstream `gog` behavior.
