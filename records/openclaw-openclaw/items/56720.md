## Summary

Keep open: the Gmail watcher regression is still valid and this PR is the useful canonical fix path, but the current patch still breaks its documented empty-list semantics and lacks real gateway/gog behavior proof from the contributor.

## What This Changes

The PR adds `hooks.gmail.excludeLabels`, threads it through Gmail runtime config and `gog gmail watch serve` args, and updates related schema, docs, generated baselines, and Gmail tests.

## Best Possible Solution

Land this PR or a replacement after preserving explicit empty-list semantics, keeping the default `SPAM,TRASH,DRAFT,SENT` exclusions, and adding redacted real gateway/gog proof.

## Reproduction Assessment

Yes. Source inspection on current main shows the gateway watcher uses `buildGogWatchServeArgs`, the builder cannot emit `--exclude-labels`, the strict Gmail schema has no `excludeLabels`, and upstream `gogcli` defaults to excluding only `SPAM,TRASH` when OpenClaw omits the flag.

## Solution Assessment

No as submitted. Threading excluded labels through config is maintainable, but explicit `[]` must still pass `--exclude-labels` with an empty value or the new empty-list documentation must be removed.

## Review Findings
