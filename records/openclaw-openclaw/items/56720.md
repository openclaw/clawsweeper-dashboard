## Summary

Keep open. Current main and the latest release still omit Gmail excluded-label handling, while this PR is the canonical linked fix path but still has a blocking empty-list semantic bug and lacks real behavior proof.

## What This Changes

This PR adds configurable Gmail watcher excluded labels, passes them to `gog gmail watch serve`, and updates schema, generated config metadata, docs, and Gmail tests.

## Best Possible Solution

Land this PR or a replacement after preserving explicit empty-list semantics, keeping the default `SPAM,TRASH,DRAFT,SENT` exclusions, and adding redacted real gateway or `gog` proof.

## Reproduction Assessment

Yes, at source level. Current main cannot emit `--exclude-labels` from the shared Gmail watcher args builder, and upstream `gogcli` defaults to excluding only `SPAM,TRASH` when the flag is omitted.

## Solution Assessment

No as submitted. Threading excluded labels through config is a maintainable fix, but explicit `[]` must pass `--exclude-labels` with an empty value or the new empty-list documentation must be removed.
