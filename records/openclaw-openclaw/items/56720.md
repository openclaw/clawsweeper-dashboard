## Summary

Keep open: current main still lacks the Gmail watcher exclude-label fix, but this PR still has a blocking empty-list semantics bug and does not provide after-fix real behavior proof from a real gateway/gog run.

## What This Changes

This PR adds Gmail excluded-label config/schema/docs/tests and passes `--exclude-labels` to `gog gmail watch serve` through the shared watcher arg builder.

## Best Possible Solution

Fix the explicit empty-list behavior, then land this PR or an equivalent branch with the runtime default, config override, generated schema/docs, regression coverage, and redacted real gateway/gog proof intact.

## Reproduction Assessment

Yes. Source inspection on current main shows the gateway watcher uses `buildGogWatchServeArgs`, that builder cannot emit `--exclude-labels`, and upstream `gogcli` defaults to excluding only `SPAM,TRASH` when OpenClaw omits the flag.

## Solution Assessment

No as submitted. Threading excluded labels through config is a maintainable fix direction, but explicit `[]` must still pass `--exclude-labels` with an empty value or the new docs/tests must stop promising that every label will trigger.
