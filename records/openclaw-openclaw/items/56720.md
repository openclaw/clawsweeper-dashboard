## Summary

Keep open: the PR addresses a real current-main Gmail watcher regression, but the latest branch still mishandles the documented empty `excludeLabels` override by omitting the flag and falling back to `gogcli` defaults.

## What This Changes

The PR adds `hooks.gmail.excludeLabels` config/schema/docs/tests and threads default Gmail excluded labels into the `gog gmail watch serve` argument builder.

## Best Possible Solution

Land a corrected version that restores default `SPAM,TRASH,DRAFT,SENT` exclusions while preserving explicit empty-list semantics and aligned schema/docs/tests.

## Reproduction Assessment

Yes. Source inspection of current main shows the Gmail watcher builder and schema cannot emit or configure `--exclude-labels`, and the gateway watcher calls that builder.

## Solution Assessment

No, not as-is. The typed config approach is maintainable, but the builder must pass `--exclude-labels` with an empty value when `excludeLabels: []` is configured.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P2] Preserve empty exclude-label overrides:** `src/hooks/gmail.ts:255-257`
