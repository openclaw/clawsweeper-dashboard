## Best Possible Solution

Repair and land this canonical branch with default exclusions, config/docs/test alignment, and explicit empty-list opt-out by passing an empty `--exclude-labels` value.

## Reproduction Assessment

Yes. Current main source lacks both the runtime/config field and any builder branch that can emit `--exclude-labels`; the PR branch shows the empty-list path still omits the flag.

## Solution Assessment

No, not as-is. The typed config/runtime approach is maintainable, but the builder must preserve explicit empty arrays by passing `--exclude-labels` with an empty value.
