## Summary

Keep open: current main still has the Gmail watcher regression and this PR is the canonical open fix path, but the patch needs one correctness fix before merge because the documented empty-list override still lets gogcli apply its SPAM/TRASH default.

## What This Changes

The PR restores Gmail watcher `--exclude-labels`, adds `hooks.gmail.excludeLabels` config/schema/docs surfaces, and extends Gmail argument tests.

## Best Possible Solution

Fix the explicit empty-list behavior, then land this canonical PR or an equivalent branch with the runtime default, config override, generated schema/docs, and regression coverage intact.

## Reproduction Assessment

Yes. Source inspection on current main shows no `excludeLabels` config/runtime field and no `--exclude-labels` branch, while the gogcli contract defaults to only `SPAM,TRASH` unless OpenClaw passes a broader flag.

## Solution Assessment

No as submitted. Threading excluded labels through config is a maintainable fix direction, but explicit `[]` must pass an empty `--exclude-labels` value or the docs/tests must stop promising that every label will trigger.
