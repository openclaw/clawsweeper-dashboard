## What This Changes

The PR adds Gmail excluded-label config/schema/docs/test coverage and appends `--exclude-labels` to `gog gmail watch serve` when labels are configured.

## Best Possible Solution

Land a corrected canonical fix that restores default `SPAM,TRASH,DRAFT,SENT` exclusions, preserves explicit empty-list opt-out semantics, and keeps schema, generated config metadata, docs, and focused tests aligned.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to inspect current main's `resolveGmailHookRuntimeConfig` and `buildGogWatchServeArgs`: there is no `excludeLabels` runtime/config field and no `--exclude-labels` branch, while the gateway watcher calls that builder.

## Solution Assessment

No, not as-is. The typed config/runtime/docs approach is maintainable, but the builder must pass an explicit empty `--exclude-labels` value when `excludeLabels: []` is configured so OpenClaw overrides the upstream `gogcli` default.
