## Summary

Keep open because the PR targets a real current-main Gmail watcher regression, but the branch still has a blocking empty-list semantics bug and lacks external real behavior proof.

## What This Changes

This PR adds configurable Gmail excluded labels to config/schema/docs/tests and passes non-empty labels to `gog gmail watch serve`.

## Best Possible Solution

Land this PR or a replacement after preserving explicit empty-list semantics, keeping default `SPAM,TRASH,DRAFT,SENT` exclusions, and adding redacted real gateway or `gog` proof.

## Reproduction Assessment

Yes, at source level. Current main cannot emit `--exclude-labels` from the shared Gmail watcher args builder, and upstream `gogcli` defaults to excluding only `SPAM,TRASH` when the flag is omitted.

## Solution Assessment

No as submitted. Threading excluded labels through config is maintainable, but explicit `[]` must pass `--exclude-labels` with an empty value or the new empty-list documentation should be removed.
