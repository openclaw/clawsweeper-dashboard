## What This Changes

The PR adds two bounded process caches for ambient plugin setup manifest discovery and resolved setup registries, with clone isolation, environment/snapshot/scope keying, lifecycle invalidation, and regression tests.

## Best Possible Solution

Land the bounded owner-level cache with the current complete key, clone isolation, and lifecycle tests, preserving the documented restart or explicit reload/install/doctor boundary for plugin metadata changes.

## Solution Assessment

Yes. Caching at the setup-registry owner removes duplicate work for all ambient callers, matches the repository’s process-stable metadata policy, and is cleaner than adding separate `/models`-specific workarounds.
