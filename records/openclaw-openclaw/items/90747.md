## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.85

Full review comments:

- **[P2] Key setup-registry caches by ambient env:** `src/plugins/setup-registry.ts:367-377`
  - body: `loadPluginManifestRegistryForPluginRegistry` is sensitive to env inputs such as HOME, OPENCLAW_STATE_DIR, OPENCLAW_BUNDLED_PLUGINS_DIR, OPENCLAW_DISABLE_BUNDLED_PLUGINS, and OPENCLAW_COMPATIBILITY_HOST_VERSION, but this cache key treats every ambient `process.env` call as the same slot. Existing plugin snapshot memos include those env values, so after one setup lookup a profile/test/one-off script that changes the ambient plugin env can keep seeing old setup providers or CLI backends until a full metadata lifecycle clear. Include an env fingerprint, or skip this cache unless the env inputs are known stable, and make the explicit setup cache clear reset the new maps.
  - confidence: 0.86
