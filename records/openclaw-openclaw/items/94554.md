## Summary

Keep open: current main and the latest release still expose the bundled Cloudflare AI Gateway provider as an Anthropic Messages passthrough with one default Claude model, so the requested REST multi-provider bridge, upstream-provider mapping, and model catalog behavior remain unimplemented.

## What This Changes

Request the bundled `cloudflare-ai-gateway` provider to become a real AI Gateway bridge with REST `author/model` wire IDs, multi-provider model selection, and provider-aware catalog metadata.

## Best Possible Solution

Define a provider-owned Cloudflare bridge contract covering REST endpoint families, upstream-provider to `author/model` mapping, pinned-model metadata, dynamic-route limits, legacy passthrough preservation, auth/header migration, docs, and focused tests.

## Reproduction Assessment

Yes, source-level: current main and v2026.6.9 both generate a single Anthropic Messages Cloudflare provider with `claude-sonnet-4-6`, no upstream-provider mapping, and no REST `author/model` transform. I did not run a live Cloudflare request in this read-only cleanup review.

## Solution Assessment

Unclear: the user need is valid, but the best fix requires maintainer choices across Cloudflare REST, legacy passthrough compatibility, Workers AI onboarding, dynamic-route metadata, and whether any generic config normalization should exist.
