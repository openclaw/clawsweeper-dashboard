## Summary

Keep open: current main and v2026.6.9 still expose Cloudflare AI Gateway as an Anthropic Messages passthrough with one Claude model, so the requested multi-provider REST bridge, `author/model` wire-id mapping, and richer catalog contract remain unimplemented.

## What This Changes

The issue requests turning the bundled Cloudflare AI Gateway provider from a single Anthropic passthrough into a multi-provider AI Gateway bridge with REST wire-id mapping and richer catalog metadata.

## Best Possible Solution

Keep this issue open until maintainers define a provider-owned Cloudflare bridge contract covering REST endpoint families, upstream-provider to `author/model` mapping, pinned-model metadata, dynamic-route limits, legacy passthrough preservation, auth/header migration, docs, and focused tests.

## Reproduction Assessment

Yes. Source inspection shows current main and v2026.6.9 still generate a single Anthropic Messages Cloudflare provider with `claude-sonnet-4-6`, no upstream-provider map, and no REST `author/model` transform; I did not run a live Cloudflare request in this read-only cleanup review.

## Solution Assessment

Unclear. The requested direction is credible and source-backed, but the best fix requires maintainer choices across the Cloudflare REST, Workers AI onboarding, configurable model-id-normalization, legacy passthrough, and dynamic-route metadata directions before a bounded implementation should start.
