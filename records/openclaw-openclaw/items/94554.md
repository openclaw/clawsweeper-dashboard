## Summary

Keep open: current main and the latest release still implement Cloudflare AI Gateway as a single Anthropic Messages passthrough with one Claude model, while the requested REST bridge, multi-provider model selection, upstream-provider mapping, and catalog metadata behavior remain unimplemented and product-sensitive.

## What This Changes

Request to turn the bundled Cloudflare AI Gateway provider into a multi-provider AI Gateway bridge with REST `author/model` wire IDs and per-model catalog metadata instead of a single Anthropic passthrough.

## Best Possible Solution

Define a provider-owned Cloudflare bridge contract covering REST endpoints, upstream `author/model` mapping, pinned versus dynamic model metadata, legacy passthrough preservation, auth/header migration, docs, focused tests, and redacted live Cloudflare proof.

## Reproduction Assessment

Yes. Source inspection on current main reproduces the gap: Cloudflare setup and catalog generation still hard-code the Anthropic Messages endpoint with one `claude-sonnet-4-6` model and no upstream-provider mapping or REST `author/model` transform.

## Solution Assessment

Unclear. The requested direction is valid, but the best implementation needs maintainer decisions across REST defaults, legacy passthrough compatibility, Workers AI onboarding, dynamic-route metadata, and whether generic config normalization belongs in core or the plugin.
