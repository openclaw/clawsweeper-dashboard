## Summary

Keep open: current main and v2026.6.9 still implement Cloudflare AI Gateway as a single Anthropic Messages passthrough with one Claude default, while the requested REST multi-provider bridge, upstream-provider mapping, and catalog metadata behavior remain unimplemented and need maintainer product direction.

## What This Changes

The issue requests turning `cloudflare-ai-gateway` into a REST-capable multi-provider AI Gateway bridge with provider/model to `author/model` mapping and richer catalog metadata instead of a single Anthropic passthrough.

## Best Possible Solution

Define a provider-owned Cloudflare bridge contract covering REST endpoint families, upstream-provider to `author/model` mapping, pinned-model metadata, dynamic-route limits, legacy passthrough preservation, auth/header migration, docs, and focused tests.

## Reproduction Assessment

Yes. Current main and v2026.6.9 both show source-level reproduction: the Cloudflare AI Gateway provider builds one Anthropic Messages provider row with `claude-sonnet-4-6` and a `/anthropic` gateway URL, not a REST multi-provider bridge.

## Solution Assessment

Unclear. The requested direction is valid and aligned with model-provider support, but the best fix needs maintainer decisions across REST defaults, legacy passthrough compatibility, Workers AI onboarding, dynamic-route metadata, and whether any core config normalization seam is required.
