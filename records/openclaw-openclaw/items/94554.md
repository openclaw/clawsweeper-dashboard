## Summary

Keep open: current main and v2026.6.8 still expose the bundled Cloudflare AI Gateway plugin as an Anthropic Messages `/anthropic` passthrough with one default Claude model, while the requested multi-provider REST bridge, `author/model` mapping, and richer catalog contract remain unimplemented and need maintainer product direction.

## What This Changes

Request to turn the bundled `cloudflare-ai-gateway` provider from a single-model Anthropic passthrough into a multi-provider Cloudflare AI Gateway bridge with REST routing, upstream model mapping, and catalog metadata.

## Best Possible Solution

Define a provider-owned Cloudflare bridge contract covering REST endpoint families, upstream-provider to `author/model` mapping, pinned-model metadata, dynamic-route metadata limits, legacy passthrough preservation, auth/header migration, docs, and focused tests.

## Reproduction Assessment

Yes. Source inspection on current main and v2026.6.8 shows the Cloudflare plugin still generates a single Anthropic Messages provider with `claude-sonnet-4-6`, no upstream-provider map, and no REST `author/model` transform.

## Solution Assessment

Unclear. The user need is valid, but the best fix requires maintainer choices across Cloudflare REST defaults, Workers AI onboarding, configurable model-id normalization, auth migration, and dynamic-route metadata limits.
