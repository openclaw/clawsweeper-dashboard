## Summary

Keep open: current main still exposes the bundled Cloudflare AI Gateway provider as an Anthropic Messages passthrough with one default Claude model, so the requested multi-provider REST bridge, upstream-provider mapping, and catalog contract remain unimplemented. The related REST, Workers AI onboarding, and config-normalization threads overlap but do not supersede this broader provider decision.

## What This Changes

Request to make the bundled Cloudflare AI Gateway provider a multi-provider REST bridge with `author/model` wire-id mapping and richer catalog metadata instead of a single Anthropic default.

## Best Possible Solution

Define a provider-owned Cloudflare bridge contract in the Cloudflare plugin covering REST endpoint families, upstream-provider to `author/model` mapping, pinned-model metadata, dynamic-route limits, legacy passthrough preservation, docs, focused tests, and redacted live proof.

## Reproduction Assessment

Yes, at source level: current main and v2026.6.8 both generate a single Anthropic Messages Cloudflare provider with `claude-sonnet-4-6` and no upstream-provider mapping or REST `author/model` transform. I did not run a live Cloudflare request in this read-only review.

## Solution Assessment

Unclear. The user need is valid, but the best fix requires maintainer choices across REST defaults, legacy passthrough compatibility, model catalog ownership, config surface, and dynamic-route metadata limits.
