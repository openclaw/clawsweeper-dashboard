## Summary

Keep open: current main and v2026.6.8 still expose the bundled Cloudflare AI Gateway plugin as an Anthropic Messages provider with one default Claude model, while the requested multi-provider REST bridge and catalog mapping remain a product/API design item. The related open issues cover narrower REST, Workers AI onboarding, and configurable normalization facets rather than superseding this broader bridge request.

## What This Changes

Request to make the bundled Cloudflare AI Gateway provider a multi-provider AI Gateway bridge with REST `author/model` wire IDs and richer model metadata instead of one Anthropic default.

## Best Possible Solution

Define a provider-owned Cloudflare bridge contract that covers REST endpoint families, upstream-provider to `author/model` mapping, pinned-model metadata, dynamic-route metadata limits, legacy passthrough behavior, auth/header migration, docs, and focused plugin/onboarding tests.

## Reproduction Assessment

Yes, source-level: current main and v2026.6.8 both generate a single Anthropic Messages Cloudflare provider with `claude-sonnet-4-6` and no upstream-provider mapping or REST `author/model` transform. I did not run a live Cloudflare request in this read-only cleanup review.
