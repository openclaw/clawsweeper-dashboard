## Summary

Keep open: current main and v2026.6.9 still expose Cloudflare AI Gateway as a single Anthropic Messages passthrough with one Claude model, while the requested REST bridge, upstream-provider mapping, multi-provider model selection, and catalog metadata behavior remain unimplemented and product-sensitive.

## What This Changes

Make the bundled Cloudflare AI Gateway provider support multi-provider AI Gateway routing with correct REST author/model IDs and model catalog metadata instead of one Anthropic passthrough default.

## Best Possible Solution

Define a provider-owned Cloudflare bridge contract covering REST endpoints, upstream-provider to author/model mapping, pinned-model metadata, dynamic-route limits, legacy passthrough preservation, auth/header behavior, docs, focused tests, and live proof.

## Reproduction Assessment

Yes, at source level: current main and v2026.6.9 both generate a single Anthropic Messages Cloudflare provider with claude-sonnet-4-6 and no upstream-provider mapping or REST author/model transform. I did not run a credentialed Cloudflare request in this read-only review.

## Solution Assessment

Unclear: the user need is valid, but the best fix requires maintainer choices across REST defaults, legacy passthrough compatibility, Workers AI onboarding, dynamic-route metadata, and whether any generic config-normalization surface should exist.
