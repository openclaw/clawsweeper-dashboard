## Summary

Keep open: current main and v2026.6.8 still expose the bundled Cloudflare AI Gateway provider as a single Anthropic Messages provider with one default Claude model, so the requested multi-provider REST bridge and catalog mapping remain unimplemented.

## What This Changes

The issue requests turning the bundled Cloudflare AI Gateway provider into a multi-provider REST bridge with upstream-provider wire-id mapping and richer catalog metadata.

## Best Possible Solution

Define a provider-owned Cloudflare bridge contract covering REST endpoint families, upstream-provider wire-id mapping, pinned-model metadata, dynamic-route limits, legacy passthrough preservation, auth/header migration, docs, and focused tests.

## Reproduction Assessment

Yes, source-level: current main and v2026.6.8 both generate a single Anthropic Messages Cloudflare provider with `claude-sonnet-4-6` and no upstream-provider mapping or REST `author/model` transform. I did not run a live Cloudflare request in this read-only cleanup review.

## Solution Assessment

Unclear. The user need is valid, but the best fix requires maintainer choices across the Cloudflare REST, Workers AI onboarding, and configurable model-id-normalization directions before a bounded implementation should start.
