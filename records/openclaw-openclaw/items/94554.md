## Summary

Keep open: current main and v2026.6.9 still implement Cloudflare AI Gateway as a single Anthropic Messages passthrough with one default Claude model, while the requested REST bridge, upstream-provider mapping, multi-provider model selection, and catalog metadata contract remain unimplemented and product-sensitive.

## What This Changes

Request to turn the bundled Cloudflare AI Gateway provider from a single Anthropic passthrough into a REST bridge with multi-provider model selection, author/model wire IDs, and catalog metadata.

## Best Possible Solution

Define a provider-owned Cloudflare bridge contract covering REST endpoint families, upstream-provider to author/model mapping, pinned-model metadata, dynamic-route limits, legacy passthrough preservation, auth/header migration, docs, and focused tests.

## Reproduction Assessment

Yes, source-level: current main and v2026.6.9 both generate a single Anthropic Messages Cloudflare provider with `claude-sonnet-4-6` and no upstream-provider mapping or REST author/model transform. I did not run a credentialed Cloudflare request in this read-only review.

## Solution Assessment

Unclear. The user need is valid, but the best implementation requires maintainer choices across REST defaults, legacy passthrough compatibility, Workers AI onboarding, dynamic-route metadata, and whether generic config normalization belongs in core.
