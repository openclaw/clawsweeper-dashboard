## Summary

Keep open: current `main` and v2026.6.9 still expose Cloudflare AI Gateway as a single Anthropic Messages passthrough with one Claude model, while the requested REST bridge, multi-provider model selection, upstream-provider mapping, and catalog metadata behavior remain unimplemented and product-sensitive.

## What This Changes

Request to turn the bundled `cloudflare-ai-gateway` provider into a real Cloudflare AI Gateway bridge with REST `author/model` routing, multi-provider model selection, and correct catalog metadata.

## Best Possible Solution

Define a provider-owned Cloudflare bridge contract covering REST endpoint families, upstream-provider to `author/model` mapping, pinned-model metadata, dynamic-route limits, legacy passthrough preservation, auth/header migration, docs, and focused tests.

## Reproduction Assessment

Yes, at source level: current `main` and v2026.6.9 both generate a single Anthropic Messages Cloudflare provider with `claude-sonnet-4-6` and no upstream-provider mapping or REST `author/model` transform. I did not run a credentialed Cloudflare request in this read-only cleanup review.

## Solution Assessment

Unclear. The user need is valid, but the best implementation requires maintainer choices across REST defaults, legacy passthrough compatibility, Workers AI onboarding, dynamic-route metadata, and whether generic config normalization belongs in core.
