## Summary

Keep open: current main and v2026.6.8 still expose the bundled Cloudflare AI Gateway plugin as a single Anthropic Messages provider with one default Claude model, so the requested multi-provider REST bridge and catalog mapping remain unimplemented. The related Cloudflare REST, Workers AI onboarding, and model-id-normalization issues overlap but do not supersede this broader provider contract decision.

## What This Changes

Request to turn the bundled `cloudflare-ai-gateway` provider into a multi-provider Cloudflare AI Gateway REST bridge with correct upstream model-id and catalog handling.

## Best Possible Solution

Define a provider-owned Cloudflare bridge contract covering REST endpoint families, upstream-provider to `author/model` mapping, pinned-model metadata, dynamic-route limits, legacy passthrough preservation, auth/header migration, docs, and focused tests.

## Reproduction Assessment

Yes, source-level: current main and v2026.6.8 both generate a single Anthropic Messages Cloudflare provider with `claude-sonnet-4-6` and no upstream-provider mapping or REST `author/model` transform. I did not run a live Cloudflare request in this read-only cleanup review.

## Solution Assessment

Unclear. The user need is valid, but the best fix requires maintainer choices across the Cloudflare REST, Workers AI onboarding, and configurable model-id-normalization directions before a bounded implementation should start.
