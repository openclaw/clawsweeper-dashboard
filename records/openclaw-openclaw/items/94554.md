## Summary

Keep open: current main and v2026.6.9 still expose Cloudflare AI Gateway as one Anthropic Messages gateway-domain provider with one Claude Sonnet model, while the requested REST multi-provider bridge, author/model mapping, and richer catalog contract remain unimplemented. The related REST endpoint, Workers AI onboarding, and provider-config normalization items overlap but do not supersede this broader provider decision.

## What This Changes

The issue asks for the bundled Cloudflare AI Gateway provider to support multi-provider AI Gateway routing, REST author/model wire IDs, and per-model catalog metadata instead of a fixed Anthropic passthrough.

## Best Possible Solution

Define a provider-owned Cloudflare bridge contract covering REST endpoint families, upstream-provider to wire-model mapping, pinned-model metadata, dynamic-route limits, legacy passthrough preservation, auth/header behavior, docs, and focused tests.

## Reproduction Assessment

Yes, source-level: current main and v2026.6.9 both build Cloudflare AI Gateway as a single Anthropic Messages gateway-domain provider with `claude-sonnet-4-6` and no upstream-provider mapping or REST `author/model` transform. I did not run a live Cloudflare request in this read-only review.

## Solution Assessment

Unclear. The user need is valid, but the best implementation requires maintainer choices across the Cloudflare REST default, legacy passthrough compatibility, model catalog metadata, provider-config normalization, and dynamic route behavior.
