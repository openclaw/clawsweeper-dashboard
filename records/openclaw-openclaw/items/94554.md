## Summary

Keep open: current main and v2026.6.8 still expose `cloudflare-ai-gateway` as one Anthropic Messages passthrough with a single Claude model, while the requested REST bridge, upstream-provider mapping, and catalog metadata contract remain unimplemented.

## What This Changes

The issue requests turning the bundled Cloudflare AI Gateway provider into a REST bridge with multi-provider model selection, upstream `author/model` wire-id mapping, and catalog metadata instead of a single Anthropic passthrough.

## Best Possible Solution

Define a provider-owned Cloudflare bridge contract covering REST endpoint families, upstream-provider to `author/model` mapping, pinned-model metadata, dynamic-route limits, legacy passthrough preservation, auth/header migration, docs, tests, and live proof.

## Reproduction Assessment

Yes, at source level: current main builds `cloudflare-ai-gateway` as a single Anthropic Messages provider with the default Claude model, so the multi-provider REST bridge is absent. I did not run a credentialed live Cloudflare request during this read-only cleanup review.

## Solution Assessment

Unclear. The user need is valid, but the best fix requires maintainer choices across first-class Cloudflare bridge behavior, REST auth/header migration, Workers AI onboarding, and the narrower provider-config normalization direction.
