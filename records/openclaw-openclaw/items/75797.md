## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/75797 with a narrow packaged public-surface loader repair. Current `src/plugins/public-surface-loader.ts` opens bundled public artifacts with `rejectHardlinks: true`, so valid package-manager/global-install hardlinked artifacts can fail as `Unable to open bundled plugin public surface google/web-search-contract-api.js` even though direct Node import works. Update the loader to accept valid OpenClaw-owned bundled public artifacts after boundary-root and same-file-identity validation, while preserving hardlink rejection for untrusted external/custom plugin surfaces if such a path exists. Do not broaden plugin install/discovery trust policy and do not move Google-specific logic into core. Add a regression test in `src/plugins/public-surface-loader.test.ts` that loads a hardlinked bundled public artifact from an `OPENCLAW_BUNDLED_PLUGINS_DIR` fixture, and keep the existing post-validation identity race check. If the error wrapper remains, improve it enough that boundary validation cause is diagnosable without leaking paths/secrets. Add a single-line user-facing changelog entry under the active version, crediting the reporter if appropriate. Validate with targeted tests for the public-surface loader and the Google/Gemini secrets path.
