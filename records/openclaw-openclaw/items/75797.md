## Summary

Keep open: the prior ClawSweeper comment still matches current main. The Google/Gemini public-surface load still rejects hardlinked bundled artifacts, while the later diagnostics work only improves the crash report, not the startup failure path.

## What This Changes

The issue asks OpenClaw to stop Google/Gemini web-search contract loading from crash-looping gateway startup after v2026.4.29 and to report the real underlying failure.

## Best Possible Solution

Allow package-proven bundled public artifacts to load from hardlink-based installs while preserving strict hardlink rejection for external/custom plugin surfaces and keeping the improved redacted diagnostics.

## Reproduction Assessment

No for the full end-to-end crash loop; it still needs a packaged macOS/Homebrew LaunchAgent cold start after SIGTERM. Yes for the source-level error shape: current main rejects hardlinked bundled public artifacts before import and startup secret activation then fails fast.

## Solution Assessment

Yes for the narrow repair direction. The best first fix is in `src/plugins/public-surface-loader.ts` or a shared boundary helper, matching existing bundled-plugin hardlink policy without changing broader SecretRef startup semantics in the same PR.
