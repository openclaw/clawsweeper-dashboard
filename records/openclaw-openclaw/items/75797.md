## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/75797 with a narrow packaged public-surface loader repair. Current `src/plugins/public-surface-loader.ts` opens bundled public artifacts with `rejectHardlinks: true`, so valid package-manager/global-install hardlinked artifacts can fail as `Unable to open bundled plugin public surface google/web-search-contract-api.js` even though direct Node import works. Update the loader to accept valid OpenClaw-owned bundled public artifacts after boundary-root and same-file-identity validation, following the existing bundled-plugin policy that allows bundled hardlinked entry files while preserving hardlink rejection for untrusted external/custom plugin surfaces. Do not broaden plugin install/discovery trust policy, do not move Google-specific logic into core, and do not relax startup SecretRef fail-fast semantics. Add a regression test in `src/plugins/public-surface-loader.test.ts` that loads a hardlinked bundled public artifact from a trusted bundled-dir fixture and keep the post-validation identity race check. If the error wrapper remains, make the boundary validation cause diagnosable without leaking secrets. Add a single-line user-facing `CHANGELOG.md` fix entry crediting @ymebosma. Validate with the targeted loader/secrets tests and, if available, a macOS/Homebrew-style package smoke through Gateway restart after SIGTERM.
