## Summary

Keep open: current main still has the source-level failure path described in the report. The later diagnostics change records the failing contract in crash bundles, but it does not change the bundled public-surface loader or startup fail-fast behavior.

## What This Changes

The issue asks OpenClaw to stop a valid Google/Gemini-enabled packaged install from crash-looping when the Google web-search public contract loader reports `Unable to open bundled plugin public surface google/web-search-contract-api.js`.

## Best Possible Solution

Allow package-proven bundled public artifacts to load from hardlink-based installs while keeping strict hardlink rejection for external/custom plugin surfaces and preserving the improved redacted diagnostics.

## Reproduction Assessment

Partially yes. The full packaged macOS LaunchAgent crash loop was not live-reproduced, but current source still shows configured Google/Gemini reaching a bundled public artifact loader that rejects hardlinked files before startup secret activation fails fast.

## Solution Assessment

Yes for the narrow repair direction. The maintainable fix is to align `src/plugins/public-surface-loader.ts` with existing bundled/facade hardlink policy for package-proven artifacts, not to change broader SecretRef startup semantics in the same PR.
