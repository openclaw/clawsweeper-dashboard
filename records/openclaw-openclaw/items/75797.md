## Summary

Current main still matches the reported failure shape: Google/Gemini secret activation can reach the bundled web-search public artifact, and that loader still rejects hardlinked bundled artifacts before import. This should stay open as a narrow bug-fix candidate rather than be swept closed.

## What This Changes

The issue requests that Google/Gemini startup not crash-loop when the bundled `google/web-search-contract-api.js` public surface exists but OpenClaw reports it cannot open it.

## Best Possible Solution

Allow package-proven bundled public artifacts to load from hardlink-based installs while keeping external plugin hardlink checks strict and preserving the redacted diagnostic improvements.

## Reproduction Assessment

No for the full end-to-end LaunchAgent crash loop; this review did not run a packaged macOS/Homebrew cold start after SIGTERM. Yes for the source-level failure path: current main still routes configured Gemini to the public artifact loader, rejects hardlinked bundled artifacts, and fails startup secret activation fast.

## Solution Assessment

Yes for the narrow repair direction. The maintainable fix belongs in `src/plugins/public-surface-loader.ts` or a shared helper so bundled public surfaces follow the existing bundled hardlink policy without changing broader SecretRef startup semantics in the same PR.
