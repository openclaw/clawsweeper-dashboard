## Summary

Keep open: current main still has the same Google/Gemini bundled public-surface load path that can reject package-managed hardlinked artifacts before import, and the startup secrets path can still make that failure fatal.

## What This Changes

The issue asks OpenClaw to stop a Google/Gemini secrets-reloader public-surface load failure from crash-looping the gateway and to improve recovery/diagnostics around that path.

## Best Possible Solution

Allow package-proven bundled public artifacts to load from hardlink-based installs while preserving strict hardlink rejection for external/custom plugin surfaces and keeping the redacted startup diagnostics improvements.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for the source-level failure shape: current main still opens the Google bundled public artifact with hardlink rejection before import, and startup secret activation still fails fast on that error; no for the full LaunchAgent crash-loop without a packaged macOS/Homebrew cold-start run.

## Solution Assessment

Is this the best way to solve the issue? Yes for a narrow repair: change the bundled public-surface loader or shared boundary helper to follow the existing bundled hardlink policy, then add regression coverage without changing broader SecretRef startup semantics in the same PR.
