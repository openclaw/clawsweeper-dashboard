## What This Changes

The issue reports a v2026.4.29 gateway crash loop where Google/Gemini web-search secret preparation fails while opening the bundled `google/web-search-contract-api.js` public surface.

## Best Possible Solution

Land a bounded bundled public-surface hardlink fix that preserves external plugin containment, then open a narrower follow-up only if disable/uninstall recovery or secondary-provider degradation remains after that change.

## Reproduction Assessment

Yes for the source-level path: configured bundled Google/Gemini web-search preparation still reaches the public artifact loader, which still rejects bundled hardlinks and can fail startup secret activation. I did not live-reproduce the macOS/Homebrew LaunchAgent crash loop.

## Solution Assessment

Yes. The open linked PR targets the narrow loader boundary by allowing trusted bundled public-surface hardlinks while preserving external plugin containment; broader recovery UX should stay separate unless maintainers expand scope.
