## Summary

Keep open: current main still has the source-level failure path where bundled public artifacts are opened with hardlink rejection enabled, so a package-managed hardlinked Google web-search contract can fail before import and still make startup secret activation fatal. The later diagnostics work improves the crash bundle, but it does not remove the startup failure path.

## What This Changes

Prevent Google/Gemini startup secret resolution from crash-looping when the bundled Google web-search contract artifact is present but opened through a hardlink-based install.

## Best Possible Solution

Allow package-proven bundled public artifacts to load from hardlink-based installs while preserving strict hardlink checks for external/custom plugin surfaces and keeping the improved redacted diagnostics.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes at source level: current main still rejects hardlinked bundled public artifacts before import, and startup secret activation fails fast on that error. No live packaged macOS/Homebrew LaunchAgent crash loop was reproduced during this read-only pass.

## Solution Assessment

Is this the best way to solve the issue? Yes for the narrow repair direction: fix the bundled public-surface boundary to match existing package-proven hardlink policy, rather than changing Google provider semantics or broad startup SecretRef behavior in the same PR.
