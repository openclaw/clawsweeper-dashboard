## Summary

Keep open. Current main still manages only the direct Chromium `ChildProcess`, while the report describes normal-exit `chromium` and `chrome_crashpad` zombies accumulating under the live gateway process in PID-1 containers.

## What This Changes

The issue asks OpenClaw to reap managed Chromium and chrome_crashpad child processes after normal browser-use exits so containers do not exhaust their PID budget.

## Best Possible Solution

Fix the bundled browser plugin lifecycle so OpenClaw-owned Chromium is supervised and reaped across normal exits, explicit stops, restarts, startup retry cleanup, and crashpad/adopted descendants, with a focused Linux regression.

## Reproduction Assessment

Yes. The issue gives a high-confidence production diagnostic path in browser-heavy PID-1 gateway containers, and current source still matches the direct-child-only lifecycle gap; I did not run a live container repro in this read-only review.

## Solution Assessment

No current solution on main is sufficient. The narrow maintainable fix belongs in the browser plugin lifecycle, potentially with a generic SDK process helper if process-tree termination or reaping must be shared.

## Repair Work Prompt

Fix the linked Chromium zombie-process issue in the bundled browser plugin. Current main only tracks and stops the direct managed Chromium ChildProcess; implement owned-process supervision/reaping so normal browser exits, explicit stops, restarts, singleton-lock retry cleanup, and chrome_crashpad/adopted descendants do not leave defunct processes under the OpenClaw gateway in Linux PID-1 containers. Keep the fix in the browser plugin boundary; if reusing generic process-tree behavior, expose a proper `openclaw/plugin-sdk/process-runtime` seam instead of deep-importing core internals. Do not solve this by documenting Docker `--init`, adding a user config option, or changing unrelated browser/provider behavior. Add focused regression coverage, including a Linux/container-oriented check that repeatedly starts/stops managed Chromium and asserts no defunct chromium or chrome_crashpad descendants remain under the OpenClaw parent. Add a user-facing changelog fix entry.
