This older PR should close as mostly implemented: current main shipped the safer kickstart-first restart lifecycle, and the branch's remaining retry/force-kill pieces are either superseded, risky, or covered by narrower bootstrap-resilience work.

## What This Changes

The branch adds launchd PID wait escalation with SIGKILL, EIO-only bootstrap retry logic, install/restart/repair call-site changes, and focused launchd tests.

## Best Possible Solution

Keep the shipped kickstart-first restart behavior and handle any remaining bootstrap/install retry work as a narrow current-main patch that avoids PID-only SIGKILL and respects launchd domain policy.

## Reproduction Assessment

Yes for the PR review findings: PR head routes normal restart through bootout/bootstrap and PID-based escalation, while current main source and tests use kickstart-first restart with no bootout/bootstrap. The original EIO race has plausible pre-fix manual evidence in the PR body, but no after-fix macOS proof is present.

## Solution Assessment

No. The central restart problem is better solved by current main's kickstart-first lifecycle; any remaining EIO retry should be a smaller current-main change without PID-only SIGKILL.
