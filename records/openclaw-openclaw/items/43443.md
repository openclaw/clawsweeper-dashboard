## Summary

Keep open: the PR still contains a useful launchctl EIO retry idea, but it is not safe to merge because it regresses current kickstart-first restarts, adds PID-only SIGKILL escalation, and lacks real macOS proof.

## What This Changes

The PR adds launchd bootstrap EIO retry logic, PID wait escalation with SIGKILL, install/restart/repair call-site changes, and focused launchd tests.

## Best Possible Solution

Rework the useful bounded EIO retry onto current main while preserving kickstart-first restart behavior and avoiding force-kill unless the launchd job identity is verified.

## Reproduction Assessment

Yes for the blocking review findings: PR head routes restart through bootout/bootstrap and sends SIGKILL after only numeric PID liveness, while current main tests require kickstart-first restart. The original EIO race has plausible PR-body manual trap evidence, but no after-fix macOS proof.

## Solution Assessment

No. Bounded EIO retry may be useful, but this branch should be narrowed around the current kickstart-first lifecycle and should remove or job-verify force-kill.
