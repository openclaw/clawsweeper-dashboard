## Summary

Keep open. The PR targets a plausible macOS launchd bootout/bootstrap EIO race and adds relevant unit coverage, but current main has since moved the normal restart path to kickstart-first while install/repair bootstrap behavior still differs from the branch; the external PR also lacks after-fix real macOS proof and is currently dirty/failing checks.

## What This Changes

The PR adds launchd PID-exit escalation, EIO-only bootstrap retry, install/repair wiring, and unit coverage for macOS LaunchAgent restart/install races.

## Best Possible Solution

Rework the patch against current main: keep the kickstart-first restart design, add bounded retry/wait only to remaining bootout/bootstrap surfaces if live macOS proof confirms the race, and keep GUI-domain failures non-retriable.

## Reproduction Assessment

No high-confidence live reproduction was performed in this read-only review. Source inspection still shows raw bootstrap after bootout in install/repair paths, and the PR body describes a manual SIGTERM-trap root-cause test, so the race is plausible but not fully proven here.

## Solution Assessment

Unclear as-is. The retry/escalation idea is reasonable for remaining bootout/bootstrap race surfaces, but the branch predates current kickstart-first restart behavior and should be narrowed and rebased rather than merged wholesale.
