## Summary

Keep open: current main still lacks EIO-specific launchctl bootstrap retry, but this PR is not merge-ready because its force-kill path can target a reused PID and the main restart flow has since moved to a kickstart-first lifecycle.

## What This Changes

The PR adds EIO-specific launchctl bootstrap retry logic, SIGKILL escalation for saved restart PIDs, retry coverage for install/repair paths, and launchd unit tests.

## Best Possible Solution

Implement a shared bounded EIO-only bootstrap retry on current main across direct bootstrap, doctor repair, and launchd handoff fallback paths; add force-kill escalation only with explicit maintainer approval and launchd job identity verification.

## Reproduction Assessment

Yes for the blocking PR finding: the PR diff shows a numeric PID liveness probe followed by `SIGKILL` with no launchd job identity check. The original EIO race has a concrete manual macOS reproduction in the PR body, but I did not rerun it live.

## Solution Assessment

No. The retry helper is a narrower maintainable direction, while this PR mixes it with unsafe PID-based process killing and an older restart lifecycle that no longer matches current main.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Verify the saved PID before SIGKILL:** `src/daemon/launchd.ts:440`
  - body: The helper reads a numeric PID, waits, verifies only with `kill(pid, 0)`, then sends `SIGKILL`. If the LaunchAgent exits and the PID is reused before the final probe or between the probe and signal, this can kill an unrelated local process. Drop the force-kill path or verify the process still belongs to the expected launchd job before signaling.
  - confidence: 0.86
- **[P3] Add the required changelog entry:** `src/daemon/launchd.ts:227`
  - body: This is a user-facing fix to launchd restart/install/repair behavior, but the PR only changes daemon source and tests. OpenClaw policy requires a `CHANGELOG.md` entry for user-facing `fix` changes before merge.
  - confidence: 0.78
