## Summary

Keep this PR open, but it is not mergeable as-is: the bounded EIO retry idea still covers current direct bootstrap paths, while the branch also reintroduces an older bootout/bootstrap restart shape, adds PID-only SIGKILL escalation, and lacks after-fix macOS launchd proof.

## What This Changes

The PR adds EIO-specific launchctl bootstrap retries, PID wait/SIGKILL escalation, install/restart/repair call-site updates, and launchd unit tests.

## Best Possible Solution

Rework the fix as bounded EIO-only retry on actual bootstrap fallback, install, repair, and detached handoff paths while preserving the current kickstart restart lifecycle and avoiding force-kill unless the launchd job identity is verified.

## Reproduction Assessment

Yes for the blocking PR findings from source inspection: the branch still uses a bootout/bootstrap restart path and sends SIGKILL after PID-only liveness. The original EIO race is plausible from the PR's manual trap test, but no live after-fix macOS launchd run is shown.

## Solution Assessment

No. The EIO retry direction is useful, but the safer maintainable solution is a current-main rework that keeps `kickstart -k` restart behavior and drops or identity-verifies SIGKILL escalation.
