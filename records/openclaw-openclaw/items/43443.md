Rework the useful bounded EIO retry onto current main while preserving kickstart-first restart behavior and avoiding force-kill unless launchd job identity is verified.

## Reproduction Assessment

No. Retry on transient bootstrap EIO is useful, but this branch should be narrowed around current kickstart-first lifecycle and remove or job-verify force-kill.
