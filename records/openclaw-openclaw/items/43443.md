## Summary

Keep open. The PR addresses a plausible launchd bootstrap race and current main still lacks bounded EIO retry on direct bootstrap paths, but the branch is not mergeable as-is because it conflicts with the current kickstart-first restart lifecycle, introduces PID-only SIGKILL escalation, and lacks after-fix macOS launchd proof.

## What This Changes

The PR adds EIO-specific launchctl bootstrap retry, PID wait/SIGKILL escalation, install/restart/repair call-site changes, and launchd unit tests.

## Best Possible Solution

Rework the fix onto current main as bounded EIO-only retry on real bootstrap paths while preserving kickstart-first normal restart behavior and avoiding force-kill unless the launchd job identity is verified.

## Reproduction Assessment

Yes for the blocking PR findings from source inspection: the PR head uses a bootout/bootstrap restart path and sends SIGKILL after PID-only liveness. The original EIO race is plausible from the PR's manual trap test, but no after-fix live macOS launchd run is shown.

## Solution Assessment

No. The EIO retry direction is useful, but the proposed branch should be rebased and narrowed to current-main bootstrap fallback/repair/install paths while preserving kickstart restart behavior and removing or identity-verifying force-kill.
