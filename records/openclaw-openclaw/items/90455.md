## Summary

Keep open: current main and v2026.6.8 still route Matrix E2EE through fake-indexeddb 6.2.5 without finished-transaction pruning, and https://github.com/openclaw/openclaw/pull/94942 is the active linked fix candidate.

## What This Changes

The issue reports Matrix E2EE unbounded heap growth and gateway OOMs because fake-indexeddb retains finished transactions in its raw transaction list.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/94942, or an equivalent Matrix-owned pruner, then close this issue after the fix reaches main.

## Reproduction Assessment

## Solution Assessment
