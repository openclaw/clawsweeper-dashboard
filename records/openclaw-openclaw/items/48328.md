## Summary

Keep open, but do not merge as-is: current main still has the provider/model FTS filter, while the linked reporter later said their observed failure was stale index state, and this patch would expose non-current-model FTS rows that current cleanup intentionally preserves.

## What This Changes

The PR removes provider/model filtering from FTS keyword search and adds a focused regression test expecting keyword results across embedding models.

## Best Possible Solution

If maintainers want cross-model lexical recall, update the current memory-core plugin path together with FTS cleanup/dedup semantics and regression coverage; otherwise focus the linked issue on stale index sync behavior.

## Reproduction Assessment

No high-confidence runtime reproduction was established. Source inspection shows current main would filter out FTS rows whose model differs from the active provider, but the linked reporter later said sync.watch plus restart fixed the observed empty result.

## Solution Assessment

No. Removing the model predicate alone is not the safest fix because current cleanup preserves other-model FTS rows and hybrid merging keys by model-derived id; a complete fix needs cleanup or dedup semantics, or a stale-sync fix instead.

## Review Findings
