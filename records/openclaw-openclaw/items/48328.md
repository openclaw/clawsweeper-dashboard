## Summary

Keep open. Current main still has the active memory-core keyword search behavior this PR is trying to fix, but the branch is stale because it edits the old `src/memory/*` implementation instead of the current `extensions/memory-core/src/memory/*` code path.

## What This Changes

The PR removes the embedding provider/model predicate from lexical FTS keyword search and adds regression coverage for keyword matches indexed under different embedding models.

## Best Possible Solution

Land a current-main memory-core fix that removes provider/model filtering from lexical FTS keyword SQL and LIKE fallbacks, keeps vector search model filtering intact, and adds regression coverage proving old-model and new-model keyword hits both appear in hybrid search.

## Reproduction Assessment

Yes. Seed `chunks_fts` with the same keyword under two model values, then call `searchKeyword` with the current provider model; current main appends `AND model = ?`, so the old-model lexical row is excluded. The linked #48300 report also describes direct FTS `MATCH` returning the row while `memory_search` does not.

## Solution Assessment

The PR’s intended solution is the right narrow direction, but this branch is not the best current patch because it edits obsolete `src/memory/*` files. The maintainable fix is the same semantic change rebased onto `extensions/memory-core/src/memory/manager-search.ts`, `manager.ts`, and the active test file.
