## What This Changes

The branch removes provider/model filtering from FTS keyword search in old `src/memory` helpers and adds an in-memory regression test for cross-model keyword matches.

## Best Possible Solution

Retarget or replace the branch so active `extensions/memory-core` keyword search drops model filtering for MATCH, MATCH-failure LIKE fallback, and substring-only paths while preserving model-bound vector search, with regression coverage and redacted real `memory_search` proof.

## Reproduction Assessment

Yes for a source-level path: seed an FTS row under one model and call the active keyword helper with a different `providerModel`; current `main` appends `AND model = ?` before returning lexical rows. I did not run a full Ollama/gateway reproduction in this read-only review.

## Solution Assessment

No as written. Removing the provider/model predicate from lexical FTS is the right fix direction, but the patch must be ported from obsolete `src/memory` files to active `extensions/memory-core` paths before it solves the issue.
