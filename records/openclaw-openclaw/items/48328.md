## What This Changes

The PR removes provider/model filtering from the old `src/memory` FTS keyword search path and adds an in-memory regression test for cross-model keyword hits.

## Best Possible Solution

Retarget or replace this PR against `extensions/memory-core` so lexical FTS MATCH, MATCH-failure LIKE fallback, and substring-only search ignore provider/model while vector search remains model-bound, with focused regression coverage and redacted real `memory_search` proof.
