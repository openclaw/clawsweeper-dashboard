This PR addresses a real Microsoft Teams threading bug visible in current main, but it should stay open because the external contributor only provided unit-test/formatter output and no after-fix Teams runtime proof; it also appears to be missing the required user-facing changelog entry.

## What This Changes

The PR changes the Microsoft Teams proactive text send path to choose threaded replies for channel conversations with a stored thread root, and lets `sendMSTeamsMessages()` use proactive thread targeting when no live turn context exists.

## Best Possible Solution

Land a narrow Teams-threading fix that reuses the existing proactive `;messageid=` targeting path, with regression coverage, a changelog entry, and contributor-supplied real Teams proof.
