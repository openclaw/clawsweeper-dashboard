## Summary

Keep open: the bug is valid and the PR now supplies targeted gateway proof, but the new WebChat guard is broader than the source boundary and can drop Codex message-tool defaults for explicit external WebChat origins.

## What This Changes

The PR skips harness source-visible-reply defaults for WebChat contexts and adds dispatch plus gateway regression coverage for Codex WebChat final replies.

## Best Possible Solution

Keep automatic final delivery only for internal-only WebChat turns while preserving harness message-tool delivery for explicit external origins, with regression coverage for both paths.

## Reproduction Assessment

Yes by source inspection: current main selects the Codex `message_tool` default for direct chats with unset visibleReplies and maps it to `message_tool_only`, while WebChat is an internal/non-deliverable channel. I did not execute the regression because this review is read-only.

## Solution Assessment

No as written: the fix should target internal-only WebChat source delivery, but the current guard also matches WebChat turns that carry explicit external origin routing. A safer patch should distinguish internal WebChat from WebChat acting on behalf of an external channel.
