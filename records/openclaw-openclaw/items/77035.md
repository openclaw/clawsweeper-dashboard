## Summary

Keep open: current main does not already expose `diagnostics.pricing`, and this PR is still an active implementation path, but the patch should not merge as-is because it crosses the pricing cache lazy-loading boundary.

## What This Changes

The branch adds a read-scoped `diagnostics.pricing` Gateway RPC, method/scope registration, protocol docs, a changelog entry, and diagnostics tests for empty and populated pricing cache metadata.

## Best Possible Solution

Expose pricing cache metadata through a lightweight diagnostics handler that reuses cache state without loading the refresh runtime, then document the additive RPC and keep it under `operator.read`.

## Reproduction Assessment

Not applicable. This is an additive Gateway diagnostics feature; source inspection confirms current main lacks `diagnostics.pricing` while the pricing cache metadata seam exists.

## Solution Assessment

No, not as currently patched. The endpoint direction is reasonable, but the TTL should be shared from a lightweight state/constants module so the diagnostics handler does not statically import the pricing refresh runtime.
