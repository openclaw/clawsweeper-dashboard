## Summary

Keep open: the PR is useful, but it currently breaks the Gateway pricing import boundary by pulling the full pricing refresh module into the eagerly loaded diagnostics handler just to read the TTL.

## What This Changes

Adds a read-scoped `diagnostics.pricing` Gateway RPC with cache metadata, advertised method registration, docs, tests, and an Unreleased changelog entry.

## Best Possible Solution

Keep the new read-scoped diagnostic, but expose the TTL through a lightweight cache-state/constants module so `diagnostics.ts` does not statically import `model-pricing-cache.ts`.

## Reproduction Assessment

Yes for the review finding. Source inspection shows `server-methods.ts` eagerly imports `diagnosticsHandlers`, and the PR makes `diagnostics.ts` statically import `model-pricing-cache.ts` while current main relies on a dynamic import for that pricing module.

## Solution Assessment

No. The RPC direction is narrow, but the current implementation should move the shared TTL to a lightweight module instead of making diagnostics load the pricing refresh module at startup.
