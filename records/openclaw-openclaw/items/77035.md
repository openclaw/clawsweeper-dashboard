## Summary

Keep open: the PR addresses a real operator visibility gap from the pricing-timeout reports and includes scope/list/docs/changelog coverage, but the current patch should not merge because it pulls the heavy pricing cache module into the diagnostics/server-methods import path.

## What This Changes

The PR adds a `diagnostics.pricing` Gateway RPC for pricing cache metadata, advertises and scopes it for operator-read clients, documents it, tests empty/populated responses, and adds a changelog entry.

## Best Possible Solution

Land the diagnostic only after preserving the lazy pricing-cache boundary by moving shared TTL metadata into a lightweight state/constants module and keeping the method list, scopes, docs, tests, and changelog aligned.

## Reproduction Assessment

Yes for the blocking review finding: source inspection of the diff shows the new diagnostics import from `model-pricing-cache.js`, while current main imports that module dynamically from runtime services. No live test run was needed to establish the import-graph regression.

## Solution Assessment

No. The RPC shape and operator-read scope are reasonable, but the implementation should source TTL metadata from a lightweight constants/state module so diagnostics does not load the full pricing cache module.

## Review Findings
