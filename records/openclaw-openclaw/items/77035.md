## Summary

Keep open: the feature direction is coherent and PR checks are green, but the latest fix imports the full pricing refresh module into diagnostics just to reuse the TTL constant, which cuts across the existing lightweight cache-state split and Gateway import-boundary rules.

## What This Changes

The PR adds a READ-scoped `diagnostics.pricing` gateway RPC that reports pricing-cache `cachedAt`, `age`, `ttlMs`, and `size`, advertises and documents it, and adds diagnostics tests plus a changelog entry.

## Best Possible Solution

Keep the RPC, but move the pricing-cache TTL into a lightweight state/constants module shared by the scheduler and diagnostics so the public diagnostic shape lands without widening the core Gateway import path.

## Reproduction Assessment

Not applicable for the feature request itself; current main simply has no `diagnostics.pricing` RPC. The review finding is source-reproducible because `diagnosticsHandlers` is part of `coreGatewayHandlers` and the added runtime import loads `model-pricing-cache.ts`.

## Solution Assessment

No. The RPC shape is narrow, but sourcing `ttlMs` by importing the full pricing refresh module is not the best implementation; a lightweight shared constant keeps the same behavior with a smaller boundary.
