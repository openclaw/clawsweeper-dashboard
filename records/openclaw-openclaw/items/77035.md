## Summary

Keep open: current main now surfaces model-pricing degradation through health/status, but it does not implement this PR's separate cache-metadata RPC. The PR is not merge-ready because it lacks external real behavior proof and introduces an eager import of the pricing cache module from an always-loaded gateway diagnostics handler.

## What This Changes

The PR adds an operator-read `diagnostics.pricing` gateway RPC with scope and discovery registration, protocol docs, changelog text, and tests for model-pricing cache metadata.

## Best Possible Solution

If maintainers still want a separate cache-metadata RPC after the health/status work, implement it as a lightweight diagnostics handler backed only by leaf pricing state/constants and document/advertise it consistently.

## Reproduction Assessment

Not applicable as a feature PR rather than a broken existing behavior report. Source inspection does confirm current main lacks `diagnostics.pricing`, while health/status now cover degraded pricing failures through a different surface.

## Solution Assessment

No. The requested direction may still be useful, but this patch should not make the always-loaded diagnostics handler import the full pricing refresh module just to read a TTL constant; a leaf state/constants module would be safer, and maintainers should confirm the separate RPC remains desired after the health/status change.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Move pricing TTL out of the eager diagnostics import:** `src/gateway/server-methods/diagnostics.ts:5`
  - body: The new handler imports `../model-pricing-cache.js` only for `GATEWAY_MODEL_PRICING_CACHE_TTL_MS`. `server-methods.ts` eagerly loads diagnostics for gateway RPC handling, while current main deliberately loads `model-pricing-cache.js` on demand after pricing is enabled and has a regression test that disabled pricing does not import it. This makes disabled or minimal gateway paths load the pricing catalog/plugin-metadata module anyway; put the TTL in `model-pricing-cache-state.ts` or a small constants module shared by the scheduler and diagnostic.
  - confidence: 0.82
