## Summary

Keep open: current main and v2026.6.10 still route isolated cron setup timeouts into the Gateway restart path and scheduler restart-recovery pause, and the linked no-restart fix is open but not merged.

## What This Changes

The issue requests changing isolated cron setup-timeout handling from a Gateway restart/recovery pause to job-local failure, backoff, and safer escalation semantics.

## Best Possible Solution

Keep this issue open until https://github.com/openclaw/openclaw/pull/96396 or an equivalent maintainer-approved fix lands, then close it with release or main-commit proof.

## Solution Assessment

Mostly yes: the linked PR is a narrow fix for the destructive default by keeping setup-timeout handling at the cron-job boundary. The remaining policy question is what stronger signal should ever escalate setup failures to a Gateway restart.
