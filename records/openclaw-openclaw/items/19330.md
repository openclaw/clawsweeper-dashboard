## Summary

Keep open: current main and v2026.6.5 still use a shared managed inbound media bucket, flat managed-media references, and flat sandbox staging, so the cross-agent privacy/correctness gap remains. The closed unmerged implementation attempt and the narrower Feishu issue do not replace this broad tracker.

## What This Changes

Requests scoped inbound media storage so files received for one agent or account are not exposed through a shared managed inbound media pool.

## Best Possible Solution

Define a scoped managed inbound-media contract across storage, bundled/plugin save paths, managed URI/path resolution, local-root access checks, sandbox staging, cleanup, docs, regression tests, and explicit sharing policy.

## Reproduction Assessment

Yes, at source level: current main still saves representative inbound media into a shared bucket, rejects scoped managed inbound references, exposes global media roots, and flattens sandbox staging. I did not run live multi-agent channel traffic in this read-only review.

## Solution Assessment

No, the proposed save-time account subdirectory is not sufficient by itself. It can be part of the fix, but the maintainable solution needs a scoped managed-media contract covering read paths, access checks, staging, cleanup, compatibility, docs, and explicit sharing.
