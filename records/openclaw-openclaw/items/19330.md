## Summary

Keep open. Current main still has the reported shared inbound-media privacy gap across representative channel saves, managed references, sandbox staging, and media local roots; the prior implementation PR is closed unmerged and the open Feishu item is only a narrower slice, so this remains the canonical security/product tracker.

## What This Changes

Request to isolate inbound media by agent or account so media sent to one agent cannot be read or staged by another agent.

## Best Possible Solution

Define a backward-compatible scoped inbound-media contract across storage, SDK/channel callers, managed URI and absolute-path resolution, local-root policy, sandbox staging, cleanup, docs, and regression tests, with an explicit sharing/access policy.

## Reproduction Assessment

Yes, at source level. Current main still routes representative channel downloads into shared media/inbound while read-side references reject scoped paths and sandbox staging flattens media names; I did not run live multi-agent channel traffic in this read-only review.

## Solution Assessment

No, not as a save-time account subdirectory alone. That is a useful starting point, but the maintainable fix needs a scoped managed-media contract that also covers references, local access roots, staging, cleanup, compatibility, and explicit sharing.
