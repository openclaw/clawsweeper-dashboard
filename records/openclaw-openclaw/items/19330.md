## Label Justifications

- `P2`: This is a concrete multi-agent privacy and correctness gap, but it requires a deliberate cross-cutting contract rather than an emergency runtime repair.
- `impact:security`: Agents sharing filesystem access can inspect inbound media that crossed a different agent or user trust boundary.
- `impact:session-state`: Media belonging to one routed conversation can be discovered or staged into another agent's turn, associating content with the wrong session context.

## Summary

Current main and the latest release still use a shared flat inbound-media contract across representative channels, managed references, and sandbox staging, so the reported cross-agent privacy boundary remains unresolved; the closed implementation attempt did not merge, and the open Feishu item covers only one narrower channel slice.

## What This Changes

Introduce per-agent or per-account isolation for inbound media so one agent cannot discover, resolve, or stage media received for another agent.

## Best Possible Solution

Define a canonical owner-scoped inbound-media contract—preferably using resolved tenant/agent context where available—with authorization checks and explicit sharing, then update storage, references, staging, cleanup, bundled plugins, docs, and regression coverage together; handle shipped flat data through a bounded migration or clearly time-limited compatibility decision rather than permanent shared fallback.

## Reproduction Assessment

Yes, from source with high confidence: representative inbound downloads still use the global `inbound` bucket, managed references remain flat, and sandbox staging strips directory scope. I did not run a live multi-agent transport reproduction in this read-only review.

## Solution Assessment

No, the proposed account-ID subdirectory is not by itself the best complete solution because account and agent ownership are not universally identical and the reference, authorization, staging, cleanup, compatibility, and explicit-sharing surfaces must share one invariant.
