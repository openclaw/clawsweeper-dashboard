## Summary

Keep open. Current main still has the reported cross-agent media isolation gap: inbound media is saved to a shared flat `inbound` store by representative bundled channels, agent-scoped media roots still include the global media directory, managed inbound references reject scoped paths, and sandbox staging flattens files to `media/inbound/<basename>`. The existing maintainer review and related context also keep this as the broad tracker; #46162 was closed unmerged and #60154 is only a narrower Feishu slice.

## What This Changes

The issue requests per-agent or per-account isolation for inbound media files so multi-agent deployments do not share one flat media pool.

## Best Possible Solution

Keep this as the canonical cross-channel media-isolation tracker. A maintainer-quality fix should define a backward-compatible scoped inbound-media contract, thread sanitized account or resolved agent context through bundled and plugin media saves, support scoped managed-media URI and absolute-path resolution while preserving existing flat files, constrain agent media roots appropriately, preserve or intentionally remap scope through sandbox staging, update docs/config help, and add regression coverage.

## Reproduction Assessment

Yes. A source-level reproduction is clear: representative channel media downloads still pass `"inbound"` to `saveMediaBuffer()`, so two agents receiving attachments through those paths share the same managed media root; current tests also preserve first-level-only inbound behavior.

## Solution Assessment

No, the issue should not be closed and the proposed account-subdirectory-only fix is not sufficient by itself. The narrowest maintainable solution needs a scoped media contract that covers save, read/reference, access-root, staging, cleanup, docs, and compatibility behavior together.
