## Summary

Keep open: current `main` and the latest shipped release still preserve the shared inbound media model across representative channel downloads, managed media references, local media roots, and sandbox staging. The closed implementation attempt did not merge, and the open Feishu item is a narrower channel-specific slice rather than a replacement for this broad tracker.

## What This Changes

The issue requests scoped inbound media storage so media sent to one agent is not visible to other agents sharing the same OpenClaw instance.

## Best Possible Solution

Define a backward-compatible scoped inbound-media contract across storage, bundled and plugin save paths, managed URI/path resolution, local media roots, sandbox staging, cleanup, docs, regression tests, and explicit sharing/access rules.

## Reproduction Assessment

Yes, at source level: current `main` still writes representative inbound channel media to the shared `inbound` bucket, rejects scoped managed refs, keeps shared media roots, and flattens staged sandbox paths. I did not run live multi-agent channel traffic.

## Solution Assessment

No, the proposed save-time account subdirectory is not sufficient by itself. It can be part of the fix, but the maintainable solution needs a scoped managed-media contract that also covers read paths, staging, cleanup, local-root policy, compatibility, and explicit sharing.
