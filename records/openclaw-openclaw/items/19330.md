## Summary

Keep open. Current main still writes representative inbound channel media into a shared flat `inbound` bucket, keeps first-level-only managed reference and sandbox staging assumptions, and the earlier implementation PR was closed unmerged while the Feishu issue remains only a narrower slice.

## What This Changes

Request per-agent or per-account isolation for inbound media so channel attachments are not saved in one shared `media/inbound` pool across agents.

## Best Possible Solution

Define a backward-compatible scoped inbound-media contract covering writes, managed references, agent access roots, sandbox staging, cleanup, bundled/plugin callers, docs, and regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure two agents or channel accounts, receive media through representative channels, and the save paths still converge under the shared `media/inbound` store while reads and staging assume flat first-level files.

## Solution Assessment

No. The proposed account-subdirectory approach is a useful starting point, but the maintainable fix needs a full scoped managed-media contract rather than only changing save-time paths.
