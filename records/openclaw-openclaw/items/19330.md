## Label Justifications

## Summary

Keep this issue open as the canonical cross-channel tracker. Current main and v2026.6.6 still share inbound media across agents, while the only broad implementation attempt closed unmerged and the open Feishu issue covers only one plugin.

## What This Changes

Add per-agent or per-account isolation for inbound media so one agent cannot access or process another agent’s attachments.

## Best Possible Solution

Define one canonical scoped inbound-media contract that carries a sanitized, tenant-aware resolved agent or account scope through plugin saves, managed references, access checks, sandbox staging, cleanup, documentation, and regression coverage, with an explicit upgrade plan for existing flat files.

## Reproduction Assessment

Yes at source level: representative bundled channels save into one global inbound bucket, and the managed-reference and staging paths contain no agent scope. This read-only review did not run a live multi-agent deployment.

## Solution Assessment

No, save-time account subdirectories alone are not the best complete solution. The scope must be derived from the canonical routing identity and enforced through references, filesystem authorization, sandbox staging, cleanup, and upgrade handling.

## Risks / Open Questions

- Account identity is not necessarily equivalent to resolved agent or tenant identity, so account-only partitioning may leave cross-routing or multi-tenant exposure.
- Existing flat inbound files and plugin callers need an explicit migration or bounded compatibility contract; an indefinite runtime fallback would preserve the shared access path.
- Filesystem partitioning alone does not provide isolation when agents retain access to the parent media directory, so authorization and sandbox-root enforcement must be part of the design.
