## Summary

Keep open: current main still uses a shared inbound media store and flat staging/reference paths, while the latest release keeps the same behavior; the narrower Feishu issue does not replace this cross-channel media-isolation tracker.

## What This Changes

The issue requests per-agent or per-account isolation for inbound media so one agent cannot read media received by another agent.

## Best Possible Solution

Define a scoped managed inbound-media contract across storage, bundled and plugin save paths, managed URI/path resolution, local media roots, sandbox staging, cleanup, docs, regression tests, and explicit sharing policy.

## Reproduction Assessment

Yes, at source level: current main still writes representative inbound media into a shared bucket, rejects scoped managed inbound references, includes global media roots, and flattens sandbox staging. I did not run live multi-agent channel traffic in this read-only review.

## Solution Assessment

No, the proposed save-time account subdirectory is not sufficient by itself. It can be part of the fix, but the maintainable solution needs a scoped managed-media contract that also covers read paths, access checks, staging, cleanup, compatibility, docs, and explicit sharing.
