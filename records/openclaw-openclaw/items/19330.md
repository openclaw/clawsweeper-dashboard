## Summary

Keep open: current main still has the cross-agent inbound media privacy gap, and the latest release tag keeps the same flat write, reference, local-root, and sandbox-staging behavior. The linked Feishu issue is narrower, and the earlier implementation PR closed unmerged, so this remains the canonical broad tracker.

## What This Changes

Request scoped inbound media storage and staging so files received for one agent/account cannot be read or processed by another agent/account.

## Best Possible Solution

Define a backward-compatible scoped inbound-media contract across storage, bundled and plugin save paths, managed URI/path resolution, local media access roots, sandbox staging, cleanup, docs, regression tests, and explicit sharing/access policy.

## Reproduction Assessment

Yes, at source level: current main still writes representative inbound media into a shared bucket, rejects scoped managed inbound references, includes global media roots, and flattens sandbox staging. I did not run live multi-agent channel traffic in this read-only review.

## Solution Assessment

No, the proposed save-time account subdirectory is not sufficient by itself. It can be part of the fix, but the maintainable solution needs a scoped managed-media contract that also covers read paths, access checks, staging, cleanup, compatibility, docs, and explicit sharing.
