## Label Justifications

- `P2`: This is a credible multi-agent media privacy and correctness gap with a bounded but cross-cutting core/channel design surface.
- `impact:security`: The issue concerns sensitive inbound media crossing agent or account trust boundaries through a shared readable media store.

## Summary

Keep open: current main still has the shared inbound media behavior across the core store, representative channel save paths, managed media references, and sandbox staging, so the reported privacy boundary gap remains real and is not superseded by the narrower Feishu follow-up or the closed unmerged implementation attempt.

## What This Changes

The issue requests isolating inbound media so files received for one agent or channel account are not saved into a shared media pool readable by other agents.

## Best Possible Solution

Define one managed scoped inbound-media contract for core and bundled channels, covering save paths, managed references, local access checks, sandbox staging, cleanup, docs, and regression proof while preserving or intentionally migrating shipped flat inbound behavior.

## Reproduction Assessment

Yes, at source level: current main still saves representative inbound media into the shared managed inbound bucket, rejects scoped managed refs, and stages sandbox media into flat media/inbound paths. I did not run a live multi-agent channel deployment in this read-only review.

## Solution Assessment

Unclear as proposed: subdirectory-by-account is plausible, but the best fix needs a maintained scoped-media contract across save, resolve, staging, cleanup, docs, and compatibility rather than a save-path-only change.
