## Summary

Keep open. Current main still saves representative channel media into the shared `media/inbound` bucket, still exposes shared media roots to agent media access, and still has flat first-level managed-reference and sandbox-staging assumptions; the open Feishu item is narrower and the earlier implementation PR was closed unmerged.

## What This Changes

Request scoped inbound media storage so media received for one agent or account is not readable or staged as shared media for other agents.

## Best Possible Solution

Define a backward-compatible scoped inbound-media contract covering writes, managed references, agent access roots, sandbox staging, cleanup, bundled/plugin callers, docs, and regression coverage.

## Reproduction Assessment

Yes for source-level reproduction: configure multiple agents or channel accounts, receive media through representative bundled channels, and the current save paths still converge under the shared `media/inbound` store while reads and staging assume first-level flat files. I did not run a live multi-agent channel scenario in this read-only review.

## Solution Assessment

No. The account-subdirectory proposal is a useful starting point, but the maintainable fix needs a scoped managed-media contract across storage, references, access roots, staging, cleanup, and plugin/channel callers rather than only changing save-time paths.
