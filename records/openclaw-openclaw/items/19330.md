## Summary

Keep open. Current main still writes inbound media through a shared managed store, representative bundled/plugin channel paths still use the flat `inbound` bucket, managed references and sandbox staging still assume first-level inbound files, and the related duplicate/implementation history leaves this issue as the broad tracker.

## What This Changes

The issue requests per-agent or per-account scoped inbound media storage and staging so files sent to one agent are not visible through the shared media pool to another agent.

## Best Possible Solution

Define a backward-compatible scoped inbound-media contract that covers writes, managed references, allowed roots, sandbox staging, cleanup, bundled and plugin callers, docs, regression coverage, and any explicit cross-agent sharing model.

## Reproduction Assessment

Yes, at source level: configure multiple agents or channel accounts and receive media through representative channels, and current save paths still converge under the shared `media/inbound` store while read/staging paths assume first-level flat files. I did not run a live multi-agent channel scenario in this read-only review.

## Solution Assessment

No. The proposed account-subdirectory approach is a useful starting point, but the maintainable fix needs a scoped managed-media contract across storage, references, access roots, staging, cleanup, and channel/plugin callers rather than only changing save-time paths.
