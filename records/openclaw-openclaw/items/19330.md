## Summary

Keep open. Current main still writes representative inbound channel media to the shared `inbound` bucket, exposes shared media roots to agent-scoped media access, and keeps flat `media://inbound/<id>` and sandbox staging assumptions, so the broad privacy gap is still live.

## What This Changes

The issue requests scoped inbound media storage so media received for one agent/account is not visible through the shared inbound media pool to other agents.

## Best Possible Solution

Define a backward-compatible scoped inbound-media contract that covers writes, managed references, agent access roots, sandbox staging, cleanup, bundled/plugin callers, docs, and regression coverage.

## Reproduction Assessment

Yes. Source inspection shows multiple inbound media saves still write to the same shared `inbound` bucket, while current read and staging code assumes first-level flat inbound files.

## Solution Assessment

No. The proposed account-subdirectory approach is a useful starting point, but the maintainable fix needs a full scoped media contract rather than only changing save-time paths.
