## Summary

Keep open: current main and the latest release still save representative inbound media into a shared flat inbound store, reject scoped managed inbound references, and flatten sandbox staging, so the cross-agent privacy/correctness problem remains. The closed implementation PR was not merged, while the open Feishu and channel-namespacing reports are narrower slices rather than replacements.

## What This Changes

Request scoped inbound media storage so files received for one agent/account are isolated from other agents instead of sharing the global media/inbound pool.

## Best Possible Solution

Define a maintainer-approved scoped inbound-media contract across save paths, managed references, sandbox staging, cleanup/retention, docs, compatibility, and regression coverage, preserving or deliberately migrating existing flat media.

## Reproduction Assessment

Yes at source level: current main still saves representative inbound media into a shared inbound bucket, rejects scoped managed refs, and stages sandbox media into flat media/inbound/<file> paths. I did not run a live multi-agent deployment, so this is source-reproducible rather than live-reproduced.

## Solution Assessment

Unclear as proposed: account-id subdirectories are plausible, but a save-time-only change would leave reference resolution, sandbox staging, cleanup, compatibility, and explicit sharing semantics incomplete. The best fix is a maintainer-approved scoped media contract across the pipeline.
