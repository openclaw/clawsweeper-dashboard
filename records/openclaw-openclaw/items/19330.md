## Summary

Keep open. Current main still has a shared flat inbound media path across representative channel downloads, flat managed-media reference handling, and flat sandbox staging, so this remains a live privacy-sensitive isolation gap rather than a stale or superseded request.

## What This Changes

The issue requests scoped inbound media storage so attachments received for one agent or channel account are not visible to unrelated agents.

## Best Possible Solution

Define a backward-compatible scoped inbound-media contract that covers writes, managed references, agent access roots, sandbox staging, cleanup, bundled and plugin callers, docs, and regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence current-main path: configure multiple agents or channel accounts, receive media through representative channels, and current saves still converge under the shared media/inbound store while reads and staging assume flat first-level files.

## Solution Assessment

No. The issue's account-subdirectory proposal is a useful starting point, but the maintainable fix needs a full scoped managed-media contract rather than only changing save-time paths.
