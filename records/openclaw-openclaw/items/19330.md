## Summary

Keep open. Current main still has a shared managed inbound media bucket across representative channel/plugin downloads, read/staging code still assumes flat first-level inbound files, and the linked implementation attempt was closed unmerged while the remaining Feishu item is only a narrower slice.

## What This Changes

The issue requests scoped inbound media storage so media received by one agent or channel account is not visible to other agents.

## Best Possible Solution

Define a backward-compatible scoped inbound-media contract across storage, SDK/runtime callers, managed references, access checks, sandbox staging, cleanup, docs, and regression tests, with maintainer direction on scope and explicit sharing.

## Reproduction Assessment

Yes, at source level. Configure multiple agents or channel accounts that receive media, and current main still routes representative saves into shared `media/inbound` while read and staging paths assume first-level flat files.

## Solution Assessment

No, not as a save-time account-subdirectory patch alone. That approach is useful context, but the maintainable solution needs one scoped managed-media contract spanning save, read, access, staging, cleanup, and channel/plugin callers.
