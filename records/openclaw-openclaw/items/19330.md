## Summary

Keep open: current main and v2026.5.12 still route representative channel downloads into shared media/inbound, keep managed inbound references and sandbox staging flat, and keep shared media roots in agent-scoped access; the prior implementation PR is closed unmerged and the related Feishu issue is only a narrower slice.

## What This Changes

Request scoped inbound media storage and access so media received for one agent/account is not visible to other agents in multi-agent deployments.

## Best Possible Solution

Define a backward-compatible scoped inbound-media contract across storage, channel/plugin saves, managed URI and absolute-path resolution, local-root policy, sandbox staging, cleanup, docs, regression tests, and explicit sharing/access rules.

## Reproduction Assessment

Yes, at source level. Current main still routes representative channel downloads into shared media/inbound while read-side references reject scoped paths and sandbox staging flattens media names; I did not run live multi-agent traffic.

## Solution Assessment

No, not as the proposed save-time account subdirectory alone. That may be one input, but the maintainable fix needs a scoped managed-media contract covering references, local access roots, staging, cleanup, compatibility, and explicit sharing.
