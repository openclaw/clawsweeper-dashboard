## Summary

The issue remains necessary: current main still routes representative inbound downloads into the shared media bucket, rejects nested scoped managed references, and permits sandbox staging from the whole media root. The earlier cross-channel PR was closed unmerged, while the open Feishu report is only a narrower channel-specific slice.

## What This Changes

Add a canonical per-agent or equivalent trust-scope boundary for inbound media storage, resolution, access, and sandbox staging.

## Best Possible Solution

Define one route-resolved trust-scope media capability, carry it through plugin save APIs and managed references, authorize reads and sandbox staging against that scope, and handle existing flat inbound artifacts through an explicit doctor migration or documented compatibility decision.

## Reproduction Assessment

Yes at source level with high confidence: representative bundled downloads still use the shared inbound bucket, nested scoped references are rejected, and sandbox staging authorizes the global media root. A live multi-agent setup was not executed during this read-only review.

## Solution Assessment

No, save-time account subdirectories alone are not the best complete solution because account identity is not the universal authorization boundary; storage, managed references, resolved routing, filesystem access, sandbox staging, cleanup, and upgrade behavior must share one canonical scope.
