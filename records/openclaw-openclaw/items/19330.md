## Summary

Keep open. Current main still uses a shared flat inbound media bucket for representative bundled channel downloads, still exposes global media roots to agent-scoped media access, and still assumes first-level inbound files during managed-reference resolution and sandbox staging.

## What This Changes

Request per-agent or per-account isolation for inbound media saved by channel monitors, with corresponding sandbox staging support.

## Best Possible Solution

Define a backward-compatible scoped inbound-media contract that covers save, read/reference resolution, agent access roots, sandbox staging, cleanup, bundled/plugin callers, docs, and regression coverage.

## Reproduction Assessment

Yes. Source inspection shows two channel media saves that pass `"inbound"` to `saveMediaBuffer()` land in the same shared media root, and current read/staging code still assumes first-level flat inbound files.

## Solution Assessment

No. The proposed account-subdirectory approach is a useful starting point, but it is not sufficient without a full scoped managed-media contract across save, read, access-root, staging, cleanup, and compatibility behavior.
