## Summary

Keep open. Current main still has the reported multi-agent inbound-media isolation gap: representative bundled channel downloads write into the same `inbound` media bucket, agent-scoped media roots still include the global media directory, managed inbound references reject scoped paths, and sandbox staging flattens media back to `media/inbound/<basename>`.

## What This Changes

Request scoped inbound media storage so files received for one agent/account are not visible to other agents in multi-agent deployments.

## Best Possible Solution

Define and implement a backward-compatible scoped inbound-media contract that covers save, read/reference, access roots, sandbox staging, cleanup, docs, and regression coverage across bundled and plugin media callers.

## Reproduction Assessment

Yes. A source-level reproduction is clear: two channel media saves that pass `"inbound"` to `saveMediaBuffer()` land in the same global media root, and current read/staging code still assumes first-level flat inbound files.

## Solution Assessment

No. The issue should stay open because the proposed account-subdirectory-only approach would not by itself address managed URI resolution, local-root access, staging, cleanup, compatibility, and docs.
