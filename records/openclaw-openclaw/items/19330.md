## Summary

Keep open: current main and the latest release still do not provide per-agent inbound media isolation; representative channel media writes still land in the shared `inbound` bucket, scoped managed refs are rejected, and sandbox staging still flattens media into one `media/inbound` path.

## What This Changes

The issue requests per-agent or per-account isolation for inbound media so one agent cannot read media delivered to another agent.

## Best Possible Solution

Keep this as the canonical broad media-isolation tracker and define one scoped inbound-media contract that covers storage identity, managed refs, staging, channel/plugin callers, cleanup, docs, and regression proof.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: representative channel media still saves to `inbound`, scoped managed refs are rejected, and sandbox staging rewrites to a flat `media/inbound/<basename>` path. I did not run a live multi-agent channel setup in this read-only review.

## Solution Assessment

No, the proposed save-time account subdirectory is only part of the fix. The best solution needs a single scoped media contract that also updates URI/path resolution, staging, channel/plugin boundaries, cleanup, docs, and upgrade/security behavior.
