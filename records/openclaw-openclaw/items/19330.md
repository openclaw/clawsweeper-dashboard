## Summary

Keep open: current main and v2026.5.20 still use shared inbound media paths for representative channel downloads, still reject scoped managed inbound refs, and still flatten sandbox staging. The closed implementation at https://github.com/openclaw/openclaw/pull/46162 did not merge, and the open Feishu item at https://github.com/openclaw/openclaw/issues/60154 is a narrower channel-specific slice rather than a replacement for this cross-channel security decision.

## What This Changes

Request to isolate inbound media storage and staging by agent or account so one agent cannot read media received for another agent.

## Best Possible Solution

Define a backward-compatible scoped inbound-media contract covering storage, bundled/plugin save paths, managed URI/path resolution, local media roots, sandbox staging, cleanup, docs, regression tests, and explicit sharing/access policy.

## Reproduction Assessment

Yes, at source level: current main still writes representative channel media to shared inbound locations, rejects scoped managed refs, and flattens staged sandbox paths. I did not run live multi-agent channel traffic.

## Solution Assessment

No, the proposed save-time account subdirectory is not sufficient by itself. It can be part of the fix, but the maintainable solution needs a scoped managed-media contract that also covers read paths, staging, cleanup, local-root policy, compatibility, and explicit sharing.
