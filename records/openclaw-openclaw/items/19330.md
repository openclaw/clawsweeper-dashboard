## Summary

Keep open. Current main still stores representative inbound media in a shared flat `inbound` bucket, still allows shared managed media roots for agent media access, and still has flat first-level read/stage assumptions, so this remains a valid privacy-sensitive tracker.

## What This Changes

Request scoped inbound media storage so media received for one agent or account is not readable as another agent's inbound media.

## Best Possible Solution

Define a backward-compatible scoped inbound-media contract covering writes, managed references, agent access roots, sandbox staging, cleanup, bundled/plugin callers, docs, and regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure two agents or channel accounts, receive media through representative channels, and current save paths still converge under the shared `media/inbound` store while reads and staging assume first-level flat files.

## Solution Assessment

No. The account-subdirectory proposal is a useful starting point, but the maintainable fix needs a full scoped managed-media contract rather than only changing save-time paths.
