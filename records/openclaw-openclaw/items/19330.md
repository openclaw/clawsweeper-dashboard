## Summary

Keep open. Current main still matches the reported privacy gap: representative inbound media saves converge on the shared managed `inbound` bucket, scoped managed references are still rejected, sandbox staging still flattens to `media/inbound/<file>`, and the prior implementation attempt was closed unmerged while the open Feishu follow-up is only a narrower slice.

## What This Changes

The issue requests per-agent or per-account isolation for inbound media instead of saving all channel downloads into a shared inbound media directory.

## Best Possible Solution

Define a backward-compatible scoped inbound-media contract across storage, SDK/runtime callers, managed references, local access roots, sandbox staging, cleanup, docs, and regression tests, with an explicit sharing/access policy.

## Reproduction Assessment

Yes, at source level. Configure multiple agents or channel accounts that receive media, and current main still routes representative saves into shared `media/inbound` while read, local-root, and sandbox staging paths assume first-level flat files.

## Solution Assessment

No. The proposed account-subdirectory approach is a useful starting point, but the maintainable fix needs a scoped managed-media contract across storage, references, local access policy, cleanup, staging, and channel/plugin callers.
