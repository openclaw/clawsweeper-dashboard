## Summary

Keep open. Current main still saves representative inbound media into a shared flat `inbound` bucket, keeps flat managed-reference and sandbox-staging assumptions, and the prior implementation PR was closed unmerged while the open Feishu item is only a narrower slice.

## What This Changes

Request scoped inbound media storage so media received for one agent or account is not saved in a shared pool visible to other agents.

## Best Possible Solution

Define a backward-compatible scoped inbound-media contract covering writes, managed references, agent access roots, sandbox staging, cleanup, bundled/plugin callers, docs, and regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure two agents or channel accounts, receive media through representative channels, and their save paths still converge under the shared `media/inbound` store while reads and staging assume flat first-level files.

## Solution Assessment

No. The proposed account-subdirectory approach is a useful starting point, but the maintainable fix needs a full scoped media contract rather than only changing save-time paths.
