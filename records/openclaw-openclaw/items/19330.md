## Summary

Keep open: current `main` and the latest shipped release still use a shared inbound media model, and the related closed implementation did not merge. The open Feishu item is a narrower channel-specific slice, so this issue remains the broad tracker for the cross-agent media isolation/security decision.

## What This Changes

The issue requests scoped inbound media storage so media received for one agent cannot be read or staged for another agent.

## Best Possible Solution

Define a backward-compatible scoped inbound-media contract covering storage, bundled/plugin save paths, managed URI/path resolution, local media roots, sandbox staging, cleanup, docs, regression tests, and explicit sharing/access rules.

## Reproduction Assessment

Yes, at source level: current `main` still writes representative inbound channel media to the shared `inbound` bucket, rejects scoped managed refs, keeps shared media roots, and flattens staged sandbox paths. I did not run live multi-agent channel traffic.

## Solution Assessment

No, the proposed save-time account subdirectory is not sufficient by itself. It can be part of the fix, but the maintainable solution needs a scoped managed-media contract that also covers read paths, staging, cleanup, local-root policy, compatibility, and explicit sharing.
