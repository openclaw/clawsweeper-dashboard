## Summary

Keep open: current main and the latest release still use a shared flat inbound media contract for representative channel downloads, managed inbound refs, local media roots, and sandbox staging. The closed implementation at https://github.com/openclaw/openclaw/pull/46162 did not merge, and https://github.com/openclaw/openclaw/issues/60154 is a narrower Feishu slice rather than a replacement for this cross-channel security decision.

## What This Changes

The issue requests per-agent isolation for inbound media so files received by one agent or channel account are not readable or staged as another agent's media.

## Best Possible Solution

Define a backward-compatible scoped inbound-media contract covering storage, bundled/plugin save paths, managed URI/path resolution, agent-local roots, sandbox staging, cleanup, docs, regression tests, and explicit sharing/access policy.

## Reproduction Assessment

Yes, at source level: current main still writes representative channel media to shared inbound locations, rejects scoped managed refs, and flattens staged sandbox paths. I did not run live multi-agent channel traffic because this is a read-only review.

## Solution Assessment

No, the proposed save-time account subdirectory is not sufficient by itself. It can be part of the fix, but the maintainable solution needs a scoped managed-media contract that also covers read paths, local roots, staging, cleanup, compatibility, and explicit sharing.
