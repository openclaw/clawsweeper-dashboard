## Summary

Keep open. Current main and the latest release still use a shared inbound media bucket for representative channel downloads, flat managed inbound references, and flat sandbox staging; the prior implementation PR is closed unmerged and related discussion keeps this as the canonical cross-channel media-isolation tracker.

## What This Changes

Request to isolate inbound media storage and access by agent, account, or another scoped identity so one agent cannot read media intended for another.

## Best Possible Solution

Define a backward-compatible scoped inbound-media contract across storage, channel/plugin saves, managed URI and absolute-path resolution, local-root policy, sandbox staging, cleanup, docs, and regression tests, with an explicit sharing/access model.

## Reproduction Assessment

Yes, at source level. Current main still routes representative channel downloads into shared media/inbound while read-side references reject scoped paths and sandbox staging flattens media names; I did not run live multi-agent traffic.

## Solution Assessment

No, not as a save-time account subdirectory alone. That is a useful starting point, but the maintainable fix needs a scoped managed-media contract covering references, local access roots, staging, cleanup, compatibility, and explicit sharing.
