## Summary

Keep open. Current main still writes representative channel downloads into the shared flat `inbound` media bucket, still grants agent media access through shared media roots, and still assumes first-level flat inbound files for reference resolution and sandbox staging.

## What This Changes

Request to isolate inbound media storage so multi-agent deployments do not share one `media/inbound` directory across agents or channel accounts.

## Best Possible Solution

Define a backward-compatible scoped inbound-media contract covering writes, managed references, access roots, sandbox staging, cleanup, bundled/plugin callers, docs, and regression coverage.

## Reproduction Assessment

Yes, by source inspection. Configure two agents or channel accounts, receive media through representative channels, and current save paths still converge under shared `media/inbound` while reads and staging assume first-level flat files.

## Solution Assessment

No. The proposed account subdirectory is a useful starting point, but the maintainable fix needs a full scoped managed-media contract rather than only changing write-time paths.
