## Summary

Keep open. Current main still has the shared inbound media store and flat read/staging assumptions, while the only direct implementation attempt was closed unmerged and the open Feishu follow-up is narrower than this cross-channel isolation request.

## What This Changes

Request scoped inbound media storage so files received for one agent/account are not saved, read, or staged through a shared cross-agent inbound pool.

## Best Possible Solution

Define a backward-compatible scoped inbound-media contract covering writes, managed references, agent access roots, sandbox staging, cleanup, bundled/plugin callers, docs, regression coverage, and any explicit sharing model.

## Reproduction Assessment

Yes for source-level reproduction: configure multiple agents or channel accounts, receive media through representative bundled channels, and the current save paths still converge under the shared `media/inbound` store while reads and staging assume first-level flat files. I did not run a live multi-agent channel scenario in this read-only review.

## Solution Assessment

No. The account-subdirectory proposal is a useful starting point, but the maintainable fix needs a scoped managed-media contract across storage, references, access roots, staging, cleanup, and plugin/channel callers rather than only changing save-time paths.
