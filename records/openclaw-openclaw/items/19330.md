## Summary

Keep open: current main still has a shared flat inbound media path across representative channel downloads, while managed references, local-root access, and sandbox staging still assume flat `media/inbound` paths. The closed implementation PR was not merged, and the open Feishu item is a narrower channel-specific slice rather than a replacement for this broader security/design tracker.

## What This Changes

The issue requests scoped inbound media storage so one agent cannot read media sent to another agent in multi-agent deployments.

## Best Possible Solution

Define a backward-compatible scoped inbound-media contract across storage, bundled/plugin saves, managed URI/path resolution, local media roots, sandbox staging, cleanup, docs, regression tests, and explicit sharing/access rules.

## Reproduction Assessment

Yes, at source level: current main still writes representative channel media to the shared inbound bucket, rejects scoped managed refs, and flattens staged sandbox paths. I did not run live multi-agent channel traffic.

## Solution Assessment

No, the proposed save-time account subdirectory is not sufficient by itself. It can be part of the fix, but the maintainable solution needs a scoped managed-media contract that also covers read paths, staging, cleanup, local-root policy, compatibility, and explicit sharing.
