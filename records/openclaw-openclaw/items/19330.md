## Summary

Keep open. Current main still stores inbound channel media in a shared flat `inbound` bucket and keeps flat reference/staging assumptions, while the prior implementation PR was closed unmerged and the open Feishu item is only a narrower slice.

## What This Changes

Request per-agent or per-account isolation for inbound media saved by channel downloads so one agent cannot read another agent's attachments.

## Best Possible Solution

Define a backward-compatible scoped inbound-media contract covering writes, managed references, agent access roots, sandbox staging, cleanup, bundled/plugin callers, docs, and regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: configure two agents or channel accounts, receive media through representative channels, and the save paths still converge under the shared `media/inbound` store while reads and staging assume flat first-level files.

## Solution Assessment

No. The proposed account-subdirectory approach is a useful starting point, but the maintainable fix needs a full scoped media contract rather than only changing save-time paths.
