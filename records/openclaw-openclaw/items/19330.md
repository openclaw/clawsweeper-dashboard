## Summary

Keep open: current main and the latest shipped release still use a shared inbound media store for representative channel downloads, while managed references, agent media roots, and sandbox staging still assume flat `media/inbound` paths. This remains the broad canonical tracker; the direct implementation PR is closed unmerged, and the open Feishu item is only a narrower channel-specific slice.

## What This Changes

The issue requests scoped inbound media storage so media received by one agent is not visible to other agents in multi-agent deployments.

## Best Possible Solution

Define a backward-compatible scoped inbound-media contract across storage, bundled and plugin save paths, managed URI/path resolution, local media roots, sandbox staging, cleanup, docs, regression tests, and explicit sharing/access rules.

## Reproduction Assessment

Yes, at source level: current main still writes representative inbound channel media to the shared inbound bucket, rejects scoped managed refs, and flattens staged sandbox paths. I did not run live multi-agent channel traffic.

## Solution Assessment

No, the proposed save-time account subdirectory is not sufficient by itself. It can be part of the fix, but the maintainable solution needs a scoped managed-media contract that also covers read paths, staging, cleanup, local-root policy, compatibility, and explicit sharing.
