## Summary

Keep open: current main and v2026.5.20 still use a shared flat inbound-media contract, and the remaining work needs security/product design rather than cleanup closure.

## What This Changes

Request scoped inbound media storage so media received for one agent or account is not visible to other agents.

## Best Possible Solution

Define a backward-compatible scoped inbound-media contract covering storage, bundled/plugin save paths, managed URI/path resolution, agent-local roots, sandbox staging, cleanup, docs, regression tests, and explicit sharing/access policy.

## Reproduction Assessment

Yes, at source level: current main still writes representative channel media to shared inbound locations, rejects scoped managed refs, exposes global media roots, and flattens staged sandbox paths. I did not run live multi-agent channel traffic because this is a read-only review.

## Solution Assessment

No, the proposed save-time account subdirectory is not sufficient by itself. It can be part of the fix, but the maintainable solution needs a scoped managed-media contract that also covers read paths, local roots, staging, cleanup, compatibility, and explicit sharing.
