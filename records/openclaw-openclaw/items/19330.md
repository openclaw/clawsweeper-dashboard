## Summary

Keep open: current main and v2026.6.5 still use a flat shared inbound media contract, the narrower Feishu issue does not replace this cross-channel tracker, and the prior implementation PR was closed unmerged.

## What This Changes

Requests scoped inbound media storage and access so media received for one agent cannot be read or staged for another agent.

## Best Possible Solution

Define a scoped managed inbound-media contract across storage, bundled and plugin save paths, URI/path resolution, local access, sandbox staging, cleanup, docs, regression tests, and explicit sharing policy.

## Reproduction Assessment

Yes, at source level: current main and v2026.6.5 still save representative inbound media into a shared inbound bucket, reject scoped managed refs, expose global media roots, and flatten sandbox staging. I did not run live multi-agent channel traffic in this read-only review.

## Solution Assessment

No, the proposed save-time account subdirectory is not sufficient by itself. It can be part of the fix, but the maintainable solution needs a scoped managed-media contract that also covers read paths, access checks, staging, cleanup, compatibility, docs, and explicit sharing.
