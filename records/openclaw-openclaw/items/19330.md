## Summary

Keep this issue open: current main still uses a shared inbound media store across representative bundled channels, and the linked Feishu issue is narrower while the earlier implementation PR was closed unmerged.

## What This Changes

Requests scoped inbound media storage so media received by one agent or channel account is not saved into a shared pool visible to other agents.

## Best Possible Solution

Define a backward-compatible scoped inbound-media contract that isolates by the chosen security boundary, preserves legacy flat files, and updates channel saves, managed references, sandbox staging, cleanup, docs, and tests together.

## Reproduction Assessment

Yes, by source inspection: current main still writes representative inbound downloads to the flat `inbound` bucket, rejects nested managed inbound references, and stages sandbox media by basename. I did not run a live multi-agent deployment in this read-only review.

## Solution Assessment

No, the proposed account subdirectory alone is not sufficient. The maintainable fix needs a scoped media contract that covers write, read, staging, cleanup, compatibility, and cross-agent sharing semantics.
