## Summary

Keep open. Current main still only skips the duplicate-prone `sessions_send` A2A follow-up for parent-owned ACP background sessions, while this report is about persistent native sub-agent sessions that can still keep `deliveryContext`, return a waited reply inline, and then run the announce path against the same stored channel route.

## What This Changes

The issue asks OpenClaw to prevent persistent native sub-agent replies sent via `sessions_send` from appearing multiple times through overlapping inline reply and announce delivery paths.

## Best Possible Solution

Native parent-owned persistent sub-agent sessions contacted by their parent through `sessions_send` should have a single parent-visible result path, with peer A2A behavior preserved for unrelated sessions and stored delivery context preserved only where it is needed for legitimate thread-bound completion routing.

## Reproduction Assessment

Yes. The issue gives concrete steps, and current source shows a high-confidence reproduction path: a native persistent sub-agent can retain `deliveryContext`, the skip guard is ACP-only, and waited `sessions_send` can both return the child reply inline and start the announce flow.

## Solution Assessment

No, current main is not the best solution yet. The narrow repair should add a native parent-owned sub-agent/session-send single-delivery guard or route decision rather than globally clearing `deliveryContext` or disabling peer A2A.
