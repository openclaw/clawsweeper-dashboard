## Summary

Keep open. Current main still leaves the reported native persistent sub-agent path with two parent-visible delivery routes: waited `sessions_send` returns the child reply inline, then starts the A2A reply/announce flow unless the target is a parent-owned ACP session; native sub-agent sessions without `acp` metadata are not covered by that skip.

## What This Changes

Fix duplicate parent-visible replies when `sessions_send` targets a persistent native sub-agent session that retained announce delivery context.

## Best Possible Solution

Add a generic parent-owned native sub-agent ownership guard so parent-to-own-child `sessions_send` uses exactly one result path, while peer A2A, cross-agent sends, thread-bound user follow-ups, and the ACP-specific behavior continue to work.

## Reproduction Assessment

Yes. The issue gives concrete steps, and current source shows a high-confidence code-level path: native child sessions can retain delivery context, waited `sessions_send` returns inline, and the A2A announce flow can still run afterward.

## Solution Assessment

No. Current main only handles the adjacent parent-owned ACP case; the narrower maintainable fix should detect parent-owned native sub-agent targets and suppress only the duplicate A2A reinjection path.
