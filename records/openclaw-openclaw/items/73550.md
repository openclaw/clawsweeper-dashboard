## Summary

Keep open. Current main still has a plausible native sub-agent duplicate path: the `sessions_send` A2A skip is limited to parent-owned ACP sessions, while native sub-agent sessions can carry parent lineage and delivery context, and waited sends return the child reply inline before starting the announce delivery flow. Related #53145 is useful broader context, but it remains open and does not terminally supersede this narrower native sub-agent report.

## What This Changes

Prevent native persistent sub-agent replies sent through `sessions_send` from appearing multiple times through overlapping inline reply and announce/delivery-context paths.

## Best Possible Solution

Add a generic parent-owned native sub-agent/session-send guard so a parent sending to its own native persistent child has one parent-visible result path. The fix should avoid globally clearing delivery context, and should keep peer A2A, unrelated broad-visibility senders, ACP handling, and thread-bound completion delivery intact.

## Reproduction Assessment

Yes. The issue gives concrete steps, and current source provides a high-confidence code path: native children can have parent lineage and delivery context, waited `sessions_send` returns the child reply inline, and the A2A announce flow can deliver through the target session context.

## Solution Assessment

No, current main is not the best solution yet. The existing ACP-only guard fixes a related slice, but the narrower maintainable fix should detect the native parent-owned child case and choose a single delivery path without breaking peer session messaging.
