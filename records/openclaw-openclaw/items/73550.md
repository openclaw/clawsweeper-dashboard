## Summary

Keep open. Current main still has a plausible native persistent sub-agent duplicate path: the skip guard is ACP-only, waited `sessions_send` returns the child reply inline, and the A2A announce flow can still resolve a target from the child session's stored delivery context. Related #53145 overlaps but is broader and still open, so it does not terminally supersede this narrower native sub-agent report.

## What This Changes

The issue requests that parent-to-sub-agent `sessions_send` replies be delivered once instead of appearing 2-3 times through overlapping inline, announce, and delivery-context paths.

## Best Possible Solution

Parent-owned native child sends should have exactly one parent-visible result path, while retaining delivery context for legitimate thread-bound completion routing and preserving normal peer A2A behavior for unrelated sessions.

## Reproduction Assessment

Yes. The issue gives concrete steps, and current source shows a high-confidence code-level path: native child sessions can retain `deliveryContext`, the ACP-only skip does not apply, and waited `sessions_send` can both return inline and start A2A announce work.

## Solution Assessment

No, current main is not the best solution yet. The narrow maintainable fix should add a native parent-owned sub-agent/session-send guard or single-delivery-path decision, rather than globally clearing `deliveryContext` or disabling peer A2A.
