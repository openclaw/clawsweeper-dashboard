## Summary

Current main still leaves the reported native parent-owned sub-agent path open: the A2A skip is ACP-only, while waited `sessions_send` can both return the child reply inline and launch the A2A/announce flow against the child session's stored delivery context.

## What This Changes

The issue requests that replies from persistent sub-agent sessions sent through `sessions_send` reach the parent conversation once, without duplicate announce or delivery-context feedback paths.

## Best Possible Solution

Add a parent-owned native sub-agent `sessions_send` route decision so parent-to-own-child sends have one parent-visible result path, while preserving peer A2A and stored delivery context for legitimate channel/thread completion routing.

## Reproduction Assessment

Yes. The issue gives a concrete reproduction, and current source shows the same path remains reachable with a native spawned child that has `spawnedBy` and stored `deliveryContext`: the skip guard is ACP-only, while waited `sessions_send` still returns inline and starts A2A/announce.

## Solution Assessment

No. Current main is not the best solution yet; the narrow repair should extend single-delivery handling to native parent-owned sub-agent sends rather than globally clearing delivery context or disabling A2A.
