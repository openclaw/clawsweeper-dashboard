## Summary

Keep open. Current main still appears to have the reported native sub-agent echo path: the A2A skip is ACP-only, while native parent-owned child sessions can keep delivery context, return a waited `sessions_send` reply inline, and then run the A2A announce delivery path. Related #53145 is useful broader context, but it is still open and does not supersede this narrower persistent native sub-agent report.

## What This Changes

The report asks OpenClaw to prevent parent-visible duplicate replies when `sessions_send` targets a persistent native sub-agent session with stored delivery context.

## Best Possible Solution

Implement a single-delivery contract for parent-owned native persistent sub-agent targets: when the parent sends to its own native child, choose either the inline reply path or the appropriate internal completion/announce path, while preserving peer A2A behavior, unrelated broad-visibility sends, ACP behavior, and normal top-level completion delivery.

## Reproduction Assessment

Yes. The issue gives concrete steps, and current source provides a high-confidence code-level reproduction path: native spawned child sessions can retain `deliveryContext`, the ACP-only skip does not apply, and waited `sessions_send` both returns an inline reply and starts the A2A announce path.

## Solution Assessment

No. Current main only addresses the parent-owned ACP case; the narrower maintainable fix should detect parent-owned native sub-agent sends or otherwise enforce one parent-visible result path without disabling normal peer A2A or sub-agent completion delivery.
