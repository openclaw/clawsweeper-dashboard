## Summary

Keep open. Current main has narrowed the ACP side of this class of bug, including parent-owned persistent ACP sessions, but the reported native persistent sub-agent path is still not covered: native sub-agent rows can carry spawnedBy and deliveryContext without acp metadata, so sessions_send still treats them as normal peer sessions and can return the waited reply inline while also starting the A2A announce path.

## What This Changes

The issue reports that sessions_send to a persistent native sub-agent session can deliver the sub-agent reply multiple times to the parent conversation when the child keeps deliveryContext and parent-routing metadata.

## Best Possible Solution

Add a focused route decision for native parent-owned persistent sub-agent sessions so a parent sending to its own child has one parent-visible result path, while unrelated peer sessions and legitimate channel-bound A2A remain unchanged.

## Reproduction Assessment

Yes. The issue gives concrete steps, and current source shows the path: a native child can retain spawnedBy and deliveryContext without acp metadata, the skip guard returns false, and waited sessions_send can both return the reply inline and launch A2A announce delivery.

## Solution Assessment

No, current main is not the best solution yet. The narrow fix should extend the owned-child single-delivery decision to native sub-agent sessions instead of globally clearing deliveryContext or disabling announce/A2A for all persistent sessions.

## Repair Work Prompt

Fix the duplicate native parent-owned persistent sub-agent sessions_send path from openclaw/openclaw#73550. Current main skips A2A only for parent-owned ACP sessions; native runtime="subagent" mode="session" children can keep spawnedBy and deliveryContext without acp metadata, so a parent sessions_send can return the reply inline and still run A2A ping-pong/announce against the stored deliveryContext. Add a focused single-delivery guard or route decision for parent-to-owned native sub-agent sessions. Preserve normal peer A2A for unrelated sessions, broad tools.sessions.visibility senders, and legitimate external channel-bound sessions. Do not globally clear deliveryContext, do not disable announce for all persistent sessions, and keep streamTo documented as ACP-only. Add/adjust regression coverage around sessions_send to a native parent-owned persistent sub-agent with deliveryContext, plus existing ACP skip behavior. Add a user-facing CHANGELOG.md fix entry if behavior changes.
