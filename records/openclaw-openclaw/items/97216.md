## Summary

Keep open: current main still falls back to a generic subagent label when a sidebar row has no explicit label/displayName, and related session-title work is still open rather than merged.

## What This Changes

Request WebChat sidebar session names to prefer configured agent display names or ACP/session titles before the generic subagent fallback.

## Best Possible Solution

Define one session-name precedence contract for WebChat, then implement it through the gateway/session API and shared UI resolver so the sidebar and picker agree on title, label/displayName, derived title, and subagent fallback behavior.

## Reproduction Assessment

Yes, at source level: the sidebar calls resolveSessionDisplayName, which falls back to "Subagent:" unless label/displayName is present. I did not run a live WebChat browser session.

## Solution Assessment

Unclear as proposed: respecting title over kind is plausible, but the best fix needs maintainer alignment with canonical title, label/displayName, derived-title, and agent-identity precedence before implementation.
