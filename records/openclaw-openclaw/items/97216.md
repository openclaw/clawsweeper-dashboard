## Summary

Keep open: current main only shows the requested friendly name when a session row already carries `label` or `displayName`; the sidebar still falls back to generic `Subagent:` for unnamed subagent rows, and the canonical title/identity precedence is still being decided in overlapping open work.

## What This Changes

Request that WebChat sidebar session rows show the configured agent display name or ACP/session title before falling back to the generic subagent label.

## Best Possible Solution

Settle the canonical session-name precedence in the open title/display-name work, then make the sidebar use that single row field while preserving existing label compatibility.

## Reproduction Assessment

Yes, source-level: current main falls back to `Subagent:` when a sidebar row has no `label` or `displayName`, and the current sessions schema does not expose a canonical `title` field. I did not run a browser repro in this read-only cleanup review.

## Solution Assessment

Unclear until maintainers choose the naming contract: the UI should keep using one shared display field, but the source of that field across label/displayName/title/identity.name still needs product review.
