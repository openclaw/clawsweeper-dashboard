## Summary

Keep open: current main and the latest release still run a package-json-specific post-update smoke check against active tracked plugin records, so valid bundle-format plugins without `package.json` can still false-fail and abort before the gateway restart path.

## What This Changes

`openclaw update` should not fail post-core plugin validation or leave the gateway stopped when valid bundle-format plugins have no `package.json`.

## Best Possible Solution

Make the existing post-update plugin payload validation format-aware for bundle plugins while preserving fail-closed handling for truly corrupt code/npm plugin payloads.

## Reproduction Assessment

Yes, at source level: an active tracked install record with `format: "bundle"`, an existing install directory, and no `package.json` can reach `runPluginPayloadSmokeCheck` and produce `missing-package-json`. I did not run the full macOS update flow live in this read-only review.

## Solution Assessment

Yes: the narrowest maintainable fix is to make post-update payload validation respect bundle manifest contracts instead of weakening the whole post-core fail-closed policy for code/npm plugins.
