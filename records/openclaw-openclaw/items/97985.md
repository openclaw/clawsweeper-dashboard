## Summary

Keep open: current main still runs an npm/package-style payload smoke check against active tracked plugin records without a `format=bundle` exception, so the reported bundle-plugin false failure remains actionable and can leave the managed gateway stopped before the restart path runs.

## What This Changes

The issue reports that `openclaw update --yes` false-fails post-update plugin sync for active `format=bundle` plugins that have `.claude-plugin/plugin.json` but no `package.json`, then exits 1 with the gateway unloaded.

## Best Possible Solution

Teach post-update payload validation to handle active bundle-format install records through their bundle manifest contract, while leaving true package-plugin corruption fail-closed before restart.

## Reproduction Assessment

Yes, at source level: a tracked active install record whose directory exists but lacks `package.json` will produce `missing-package-json`, and current code does not exempt or validate `format=bundle` records. I did not run the full macOS update flow live in this read-only review.

## Solution Assessment

Yes, the best fix is narrow: make the existing update-time payload check format-aware for bundle records instead of changing the whole post-core fail-closed policy. A broader rule that restarts after any plugin-sync failure would conflict with the documented current safety contract.
