## Summary

Keep open. Current main still builds plugin-provided tools as a setup-time snapshot for embedded agent sessions: plugin install refreshes registry metadata and still points users at a gateway restart, while Pi sessions receive a fixed tool allowlist during session creation. The adjacent restart-event PR is related but does not implement active-session plugin tool re-resolution.

## What This Changes

Request dynamic refresh of plugin-provided tools into active agent sessions after `openclaw plugins install` so the next turn can use newly registered tools without a gateway/session restart.

## Best Possible Solution

Add a generic runtime lifecycle path that re-resolves plugin-provided tools after a successful install or registry refresh, applies the existing effective tool-policy pipeline, updates active sessions only between model turns, and invalidates affected tool inventory or prompt cache state without duplicating plugin runtime loads.

## Reproduction Assessment

Yes. The issue gives concrete install -> immediate use -> restart steps, and current-main source inspection confirms plugin tools and Pi allowlists are resolved during setup with no active-session refresh hook.

## Solution Assessment

Unclear as an implementation because this issue has no patch. The requested direction is valid, but the safer maintainable solution is turn-boundary re-resolution through existing policy filters rather than arbitrary mid-stream mutation of active session state.
