## Summary

Keep open. Current main still materializes plugin tools as a point-in-time tool array during agent-run setup, while plugin install only persists records and refreshes the registry snapshot; I found no active-session tool delta, turn-boundary tool re-resolution, or session allowlist update path after install.

## What This Changes

Request that successful plugin installs which add tools refresh those tools into already-active agent sessions before the next agent turn.

## Best Possible Solution

Add a generic runtime lifecycle path that re-resolves plugin-provided tools after a successful plugin install or registry refresh, applies the same effective tool-policy filters used during normal run setup, updates active sessions only at a safe turn boundary, and invalidates affected prompt/tool-cache state without duplicating plugin runtime loads.

## Reproduction Assessment

Yes. The issue gives concrete install -> immediate tool use -> gateway restart steps, and current-main source inspection confirms install refreshes the registry while active agent sessions keep a session-created tool allowlist snapshot.

## Solution Assessment

Unclear as a patch because this issue has no implementation. The requested direction is valid, but the safer maintainable shape is turn-boundary re-resolution through existing policy filters rather than arbitrary mid-stream mutation of active session state.
