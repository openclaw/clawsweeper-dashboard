## Summary

Keep open. Current main still resolves plugin-provided tools into a point-in-time tool array and Pi session allowlist during run setup, while plugin install persists records and refreshes the registry without an active-session tool refresh path.

## What This Changes

Request that tool-providing plugin installs refresh newly available tools into already-active agent sessions without requiring a gateway restart.

## Best Possible Solution

Add a generic runtime lifecycle path that re-resolves plugin-provided tools after successful plugin install or registry refresh, applies the same effective tool-policy filters used at normal run setup, updates active sessions only at a safe turn boundary, and invalidates affected prompt/tool inventory cache state without duplicating plugin runtime loads.

## Reproduction Assessment

Yes. The issue provides a concrete install -> immediate use -> gateway restart comparison, and current-main source inspection confirms plugin installs refresh the registry while active runs build and apply a fixed tool snapshot during session setup.

## Solution Assessment

Unclear as a patch because this issue has no implementation. The requested direction is valid, but the safer maintainable solution is turn-boundary re-resolution through existing policy filters rather than arbitrary mid-stream mutation of active session state.
