## Summary

Keep open. Current main still builds plugin tools into a point-in-time tool array during agent-run setup, while plugin install refreshes persisted registry metadata and continues to tell users to restart the Gateway; I found no active-session tool delta, turn-boundary re-resolution, or session allowlist update after install.

## What This Changes

The issue requests that successful plugin installs which add tools make those tools available to already-active agent sessions without requiring a Gateway restart.

## Best Possible Solution

Add a generic runtime lifecycle path that re-resolves plugin-provided tools after a successful plugin install or registry refresh, applies the normal effective-tool policy pipeline, updates active sessions only at a safe turn boundary, and invalidates affected prompt/tool cache state without duplicating plugin runtime loads.

## Reproduction Assessment

Yes. The issue gives concrete install -> immediate use -> Gateway restart steps, and current-main source inspection confirms plugin install refreshes registry metadata while agent sessions keep a session-created tool allowlist snapshot.

## Solution Assessment

Unclear as a patch because this issue has no implementation. The requested direction is valid, but the safer maintainable shape is turn-boundary re-resolution through existing policy filters rather than arbitrary mid-stream mutation of active session state.
