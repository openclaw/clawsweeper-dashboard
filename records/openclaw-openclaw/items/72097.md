## Summary

Keep open. Current main still builds plugin tools into setup-time tool arrays and Pi session allowlists, and the linked PR #75976 is open with closing syntax, so this issue should remain open pending that PR rather than be swept closed.

## What This Changes

Request live refresh of model-facing plugin tools in active agent sessions after `openclaw plugins install <pkg>` adds tool definitions.

## Best Possible Solution

Land or replace #75976 with a turn-boundary tool re-resolution path that refreshes plugin/runtime registries and reapplies existing tool policy before updating live session toolsets.

## Reproduction Assessment

Yes. The issue gives concrete install -> immediate use -> restart steps, and source inspection confirms current main refreshes registry metadata while active Pi sessions keep setup-time `customTools` and `sessionToolAllowlist` snapshots.

## Solution Assessment

Unclear because there is no merged patch on main. The requested direction is valid, but the maintainable fix should re-resolve tools through the existing policy pipeline at a safe turn boundary rather than mutate active session state mid-stream.

## Security Review

Status: needs_attention

Summary: The requested behavior is security-sensitive because newly installed plugin tools would become available to already-running sessions.

Concerns:

- **[medium] Reapply tool policy before live tool exposure:** `src/agents/pi-tools.ts:606`
  - body: Normal tool construction applies plugin allowlists, owner-only checks, provider/model filters, sandbox policy, group policy, subagent policy, schema normalization, and hook wrappers before tools reach the model; a live refresh path must reuse that pipeline.
  - confidence: 0.9
- **[medium] Refresh only at a safe turn boundary:** `src/agents/pi-embedded-runner/run/attempt.ts:1561`
  - body: The embedded runner creates `customTools`, `sessionToolAllowlist`, prompt state, and active tool names together during setup; updating one piece mid-stream could desynchronize prompt text, SDK allowlists, and tool-call handling.
  - confidence: 0.86
