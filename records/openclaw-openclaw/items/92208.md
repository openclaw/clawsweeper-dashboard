## Summary

This PR targets a real v2026.6.5 custom-provider auth regression, but it is not merge-ready: its marker-only snapshot guard can reuse a resolved credential across structurally different configurations, its proof is test-only, and it overlaps a second open implementation with stronger real-path evidence.

## What This Changes

The branch adds runtime-config snapshot fallback to the shared custom-provider API-key resolver and adds three tests for managed-marker resolution and limited snapshot isolation.

## Best Possible Solution

Keep one shared custom-provider resolver, gate runtime credential reuse through the canonical full-source snapshot applicability contract, add a different-config/same-marker isolation regression, and validate actual file-backed SecretRef preparation before landing one canonical PR.

## Reproduction Assessment

## Solution Assessment

No as written. The shared-resolver location is appropriate, but snapshot reuse must use the repository's full source-config applicability contract rather than provider-marker equality, and only one overlapping PR should land.
