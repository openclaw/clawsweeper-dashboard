## Summary

This should stay open: current main still has a source-reproducible gap where message-tool sends can be shown through live UI events but are not mirrored as durable assistant transcript messages for later `chat.history` reloads.

## What This Changes

The issue requests durable WebChat/Control UI transcript visibility for agent replies sent with `messages.visibleReplies: "message_tool"`.

## Best Possible Solution

Persist successful message-tool sends as transcript-visible assistant output on the active session branch, while keeping source-delivery dedupe intact.

## Reproduction Assessment

Yes, source-reproducible but not live-reproduced in this read-only pass. The code tracks successful `message(action=send)` text for live output and dedupe, then later `chat.history` rebuilds from persisted transcript where the visible message-tool text is not appended as an assistant message.

## Solution Assessment

Yes, the best fix is a narrow transcript-persistence repair, not changing `visibleReplies` policy or disabling duplicate suppression. The durable assistant display entry should be written once when message-tool delivery succeeds.
