## Summary

This should stay open: current main still has a source-reproducible message-tool transcript persistence gap, and the reporter’s follow-up says the same symptom may also affect automatic replies.

## What This Changes

The issue reports that WebChat/Control UI assistant replies disappear from the transcript after the next user turn, first with `visibleReplies: "message_tool"` and then also in automatic mode.

## Best Possible Solution

Persist or mirror each user-visible assistant reply into the active OpenClaw session transcript exactly once, including successful message-tool sends, while preserving source-delivery dedupe and validating whether automatic mode shares the same reload path.

## Reproduction Assessment

Yes for the `message_tool` path: source inspection shows successful tool sends can be surfaced live and deduped from final delivery without being appended as durable assistant transcript text. The automatic-mode add-on is plausible but still needs a focused reproduction to confirm it is the same bug.

## Solution Assessment

Yes, the narrow maintainable fix is transcript persistence on the active session branch, using the session manager rather than raw JSONL writes. Changing `visibleReplies` policy or only preserving more UI-local optimistic messages would leave history, search, and reload behavior inconsistent.

## Repair Work Prompt

Fix the WebChat/Control UI transcript persistence bug for source replies that are visible to the channel but missing from durable `chat.history`. Start from this issue and the proven `message_tool` path: successful `message(action=send)` text is committed to `messagingToolSentTexts`, used for live fallback/dedupe, and then lost on history reload because it is not appended as a visible assistant transcript message. Use `SessionManager.appendMessage(...)` or an existing wrapper, not raw JSONL writes, and keep source delivery dedupe intact. Also add a targeted check for the reporter's automatic-mode symptom; if automatic replies are already persisted on current main, keep the fix scoped to message-tool sends and document that in tests. Add regression coverage around the active session branch and UI history reload behavior. Include a user-facing changelog entry. Do not change `visibleReplies` defaults or add a new config option.
