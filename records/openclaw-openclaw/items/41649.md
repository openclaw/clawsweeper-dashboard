## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Keep handoff text out of system prompts:** `src/auto-reply/reply/get-reply-run.ts:274-281`
  - body: This appends archived user messages to `extraSystemPromptParts`, which gives old user-authored text system-prompt priority. A previous message containing instructions can now steer the replacement model as higher-priority context, defeating the goal of a clean handoff; pass this as escaped lower-priority user/context data instead.
  - confidence: 0.9
- **[P1] Use collision-resistant handoff paths:** `src/config/sessions/model-handoff.ts:130-132`
  - body: Sanitizing and truncating the session key maps distinct keys like `a:b` and `a_b` to the same JSON file. Since that file stores recent user messages, one session can overwrite or read another session's handoff context.
  - confidence: 0.91
- **[P1] Preserve fresh handoffs for all sessions:** `src/config/sessions/model-handoff.ts:287-292`
  - body: The cleanup loop keeps only the newest five handoff files across the entire sessions directory. A model switch affecting more than five sessions can delete fresh handoffs before those sessions send their first post-switch message.
  - confidence: 0.92
- **[P1] Archive transcripts on restart reload paths:** `src/gateway/server-startup.ts:77-78`
  - body: The startup path added for restart-based reloads only calls `clearSessionModelFields`, so restart-triggered model config changes still leave old transcripts in place and never write the handoff files this feature depends on.
  - confidence: 0.86
- **[P2] Write handoffs with private atomic storage:** `src/config/sessions/model-handoff.ts:163-165`
  - body: The handoff JSON contains recent user messages, but this direct `writeFileSync` call does not use the repo's private atomic write helper or set an explicit private mode. That increases both exposure and partial-write risk for sensitive session context.
  - confidence: 0.84
- **[P2] Consume handoffs only after a run will proceed:** `src/auto-reply/reply/get-reply-run.ts:274-275`
  - body: The one-shot handoff is consumed before later guards can return without running the model, so empty or unauthorized first-turn traffic can delete the handoff without ever delivering it to the replacement model.
  - confidence: 0.84
- **[P2] Clear context tokens with runtime model fields:** `src/config/sessions/store.ts:752-755`
  - body: `clearSessionModelFields` deletes the runtime model/provider but leaves `contextTokens` behind. The startup and fallback reset paths can therefore report the new model while retaining the old model's context window until another run rewrites the entry.
  - confidence: 0.82
