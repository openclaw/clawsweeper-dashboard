  - body: `consumeHandoffAsSystemPrompt` returns prior user messages and appends them to trusted system prompt parts. Current main keeps attacker-controlled chat text out of system metadata, so replaying pre-switch user text at system priority crosses the prompt boundary.
  - confidence: 0.91
- **[P1] Archive transcripts on restart reload paths:** `src/gateway/server-startup.ts:77-78`
  - body: The startup/restart path calls only `clearSessionModelFields`, so restart-based model config changes still leave transcripts in place and never create the handoff files this feature relies on.
  - confidence: 0.88
- **[P1] Use collision-resistant handoff paths:** `src/config/sessions/model-handoff.ts:131-132`
  - body: `resolveHandoffPath` derives filenames from sanitized and truncated session keys, so distinct keys can collide and overwrite or expose another session’s recent-message handoff.
  - confidence: 0.9
- **[P1] Keep handoffs per session until consumed:** `src/config/sessions/model-handoff.ts:291-292`
  - body: The cleanup loop keeps only the newest five handoff files across the whole sessions directory. A model switch affecting more than five sessions can delete fresh handoffs before users send their first post-switch message.
  - confidence: 0.92
- **[P2] Consume handoffs only when a run will proceed:** `src/auto-reply/reply/get-reply-run.ts:274-275`
  - body: The one-shot handoff is consumed before later early-return guards, so first-turn traffic that short-circuits can delete the handoff without ever sending it to the replacement model.
  - confidence: 0.86
- **[P2] Write handoffs with private atomic storage:** `src/config/sessions/model-handoff.ts:165`
  - body: The handoff JSON contains recent user messages but is written directly without the existing private atomic session-state write pattern, increasing sensitive-context exposure and partial-write risk.
  - confidence: 0.84
