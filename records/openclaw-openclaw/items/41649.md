## Summary

Keep open: current main still lacks a model-change transcript archive/handoff path, and this PR is plausible source work but the patch has blocking correctness and security issues plus a privacy/prompt contract that needs maintainer review.

## What This Changes

The PR archives old session transcripts on default-model changes, writes recent user-message handoff files, rotates session IDs, and injects handoff context into the next first-turn prompt.

## Best Possible Solution

Implement a maintained session model-change handoff with private atomic storage, per-session retention, one-shot consumption after execution is committed, persisted archive metadata, focused tests, docs/changelog, and an approved prompt/privacy contract.

## Reproduction Assessment

Yes. Source inspection of current main shows model-related config reload only refreshes heartbeat/catalog state and has no session handoff/reset path; the PR defects are also source-reproducible from the diff via more than five affected sessions and first-turn early-return events.

## Solution Assessment

No. The direction is plausible, but this patch should not merge until it preserves all fresh handoffs, delays one-shot consumption until a run proceeds, hardens sidecar writes, persists archive metadata, adds changelog coverage, and gets privacy/prompt review.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve every fresh model handoff:** `src/config/sessions/model-handoff.ts:291-292`
  - body: cleanupHandoffFiles keeps only the newest five handoff files in the whole sessions directory, and archiveAndHandoffOnModelChange calls it after creating handoffs. A default-model switch with more than five affected sessions will delete most fresh handoffs before those users send their first post-switch message.
  - confidence: 0.92
- **[P2] Consume handoffs only when the run will proceed:** `src/auto-reply/reply/get-reply-run.ts:275`
  - body: The PR reads and deletes the one-shot handoff before later early-return guards. If the first post-switch event is an unauthorized command, empty normalized body, or another pre-run rejection, the handoff is lost without ever reaching the model.
  - confidence: 0.86
- **[P3] Persist the archived transcript path:** `src/config/sessions/store.ts:838-840`
  - body: The handoff file is written before archiveSessionTranscripts runs, and the later archivePath assignment only mutates the returned object. The persisted JSON never gets the documented archivePath, so write the handoff after archiving or rewrite it atomically with that path.
  - confidence: 0.78
- **[P3] Add the required changelog entry:** `src/config/sessions/model-handoff.ts:139-147`
  - body: This is a user-facing feat(sessions) change, but the PR does not update CHANGELOG.md. Add a single-line entry under the active release section so the model-change handoff behavior is included in release notes.
  - confidence: 0.84
