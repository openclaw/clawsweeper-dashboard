## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Archive transcripts on restart reload paths:** `src/gateway/server-startup.ts:78`
  - body: The startup/restart path added here calls `clearSessionModelFields`, so restart-based model config changes still leave transcripts in place and never create the handoff files this feature relies on. Users whose config reload falls back to restart would keep exactly the old model transcript behavior the PR is meant to avoid.
  - confidence: 0.88
- **[P1] Use collision-resistant handoff paths:** `src/config/sessions/model-handoff.ts:131-132`
  - body: `resolveHandoffPath` derives the JSON filename from a sanitized and truncated session key, so distinct keys such as `a:b` and `a_b` can target the same file. Because these files contain recent user messages, this can overwrite or expose another session's handoff context.
  - confidence: 0.9
- **[P1] Keep handoffs per session until consumed:** `src/config/sessions/model-handoff.ts:291-292`
  - body: The cleanup loop keeps only the newest five handoff files across the session directory. A model switch affecting more than five sessions can delete fresh handoffs before those users send their first post-switch message, so most sessions may never receive the promised context.
  - confidence: 0.92
- **[P2] Consume handoffs only when a run will proceed:** `src/auto-reply/reply/get-reply-run.ts:275`
  - body: The first-turn path consumes and deletes the one-shot handoff before later guards can return without running the model. Empty or otherwise short-circuited first-turn traffic can discard the handoff without ever sending it to the replacement model.
  - confidence: 0.86
- **[P2] Write handoffs with private atomic storage:** `src/config/sessions/model-handoff.ts:165`
  - body: The handoff JSON contains recent user messages, but the new code writes it directly without the private atomic session-store helper or an explicit private mode. That increases sensitive-context exposure and partial-write risk compared with the existing session-state write path.
  - confidence: 0.82
