## Summary

Keep open: current main still does not implement automatic model-change transcript archival/handoff, and this external PR still has correctness, security-sensitive storage, product/privacy, and real-behavior-proof blockers before it can be merged.

## What This Changes

The PR adds default-model-change handling that archives old transcripts, writes recent-user-message handoff files, rotates session IDs, and injects handoff context into the first post-switch turn.

## Best Possible Solution

Implement a maintained session reset/handoff service that covers hot reload and restart paths consistently, uses private collision-resistant atomic storage, retains fresh per-session handoffs, consumes only when a run will proceed, and has approved prompt/privacy semantics.

## Reproduction Assessment

Yes, at source level. Current main lacks automatic model-change session handoff, and the PR defects are visible directly in the changed storage, cleanup, startup, and first-turn consumption paths.

## Solution Assessment

No. The direction may be useful, but this branch is not the best mergeable solution until restart coverage, private collision-resistant storage, retention, consumption timing, and privacy/prompt semantics are fixed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Archive transcripts when reload falls back to restart:** `src/gateway/server-startup.ts:78`
  - body: `gateway.reload.mode=restart` and mixed restart-triggering config changes skip the hot-reload handler, but this startup path only clears model fields. That leaves the old transcript in place with no handoff, so the new default model can still load the previous model's conversation history after restart.
  - confidence: 0.88
- **[P1] Make handoff filenames collision-resistant:** `src/config/sessions/model-handoff.ts:131-132`
  - body: Sanitizing and truncating the session key means distinct keys such as `a:b` and `a_b`, or long keys sharing the first 100 safe characters, resolve to the same handoff file. Because the file contains recent user messages, a collision can overwrite or expose one session's context to another.
  - confidence: 0.87
- **[P1] Keep all fresh handoffs for affected sessions:** `src/config/sessions/model-handoff.ts:291-292`
  - body: The cleanup keeps only the newest five handoff files in the entire sessions directory and runs immediately after creating handoffs. A default-model switch with more than five affected sessions will delete fresh handoffs before most users send their first post-switch message.
  - confidence: 0.92
- **[P2] Consume handoffs only when a run will proceed:** `src/auto-reply/reply/get-reply-run.ts:275`
  - body: The handoff is read and deleted before later early-return guards. If the first post-switch event is an unauthorized control command, an empty normalized body, or another pre-run rejection, the handoff is lost without ever reaching the model.
  - confidence: 0.86
- **[P2] Clear contextTokens in field-only model resets:** `src/config/sessions/store.ts:752-755`
  - body: `clearSessionModelFields` is used by startup and fallback paths, but it only deletes `model` and `modelProvider`. After a restart-based reset or handoff failure, the session can still carry the previous model's context window for status and budgeting until another run rewrites it.
  - confidence: 0.81
- **[P3] Persist the archive path before writing the handoff:** `src/config/sessions/store.ts:837-840`
  - body: The handoff JSON is written before `archiveSessionTranscripts` runs, and the later `archivePath` assignment only mutates the returned object. The on-disk handoff never contains the documented archived transcript path.
  - confidence: 0.78
