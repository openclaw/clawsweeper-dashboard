## Summary

Keep open: the PR addresses a real session/model-change gap that is still present on current main, but the proposed branch has blocking correctness and security issues plus no after-fix real behavior proof.

## What This Changes

The branch adds model-change session handoff plumbing that archives transcripts, writes recent-user-message handoff JSON, rotates session IDs, and injects handoff context on the next first turn.

## Best Possible Solution

Implement a maintained model-change handoff/reset flow with private atomic storage, per-session retention, delayed one-shot consumption, persisted archive metadata, focused regression coverage, and an approved prompt/privacy contract.

## Reproduction Assessment

Yes. Source inspection of current main shows model-related config reload only refreshes heartbeat/catalog state and has no session handoff/reset path; the PR defects are source-reproducible from the changed files.

## Solution Assessment

No. The direction is plausible, but this patch should not merge until it preserves all fresh handoffs, delays one-shot consumption until a run proceeds, hardens sidecar writes, persists archive metadata, and gets privacy/prompt review.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve all fresh handoffs:** `src/config/sessions/model-handoff.ts:291-292`
  - body: cleanupHandoffFiles keeps only the newest five handoff files in the whole sessions directory, and archiveAndHandoffOnModelChange calls it after creating handoffs. A default-model switch with more than five affected sessions will delete most fresh handoffs before those users send their first post-switch message.
  - confidence: 0.92
- **[P2] Consume handoffs only after the run will proceed:** `src/auto-reply/reply/get-reply-run.ts:275`
  - body: The handoff is read and deleted before later early-return guards. If the first post-switch event is an unauthorized command, empty normalized body, or another pre-run rejection, the handoff is lost without ever reaching the model.
  - confidence: 0.86
- **[P3] Persist the archived transcript path:** `src/config/sessions/store.ts:838-840`
  - body: The handoff file is written before archiveSessionTranscripts runs, and the later archivePath assignment only mutates the returned object. The persisted JSON never gets the documented archivePath, so write the handoff after archiving or rewrite it atomically with that path.
  - confidence: 0.78
