## Summary

Keep this PR open because current main still lacks the proposed model-change transcript handoff, but the patch is not ready to merge: it can drop fresh handoffs, consume one-shot context before an agent run proceeds, and writes sensitive handoff data without the private atomic storage pattern used by session artifacts.

## What This Changes

The PR adds a model-config-change session handoff flow that archives old transcripts, writes recent user-message handoff JSON, rotates session IDs, and injects handoff context on the next first turn.

## Best Possible Solution

Implement this as a maintained gateway/session feature with private atomic handoff storage, retention that preserves all fresh affected sessions, one-shot consumption only after a run is committed, explicit override preservation, contextTokens cleanup, tests, docs/changelog, and a reviewed prompt contract.

## Reproduction Assessment

Yes. Source inspection shows current main has no model-change handoff path, and the PR defects have clear source-level reproduction paths: switch models with more than five active sessions, or let a first-turn event consume a handoff before an early-return guard skips the run.

## Solution Assessment

No. The direction is plausible, but this patch should not merge until it preserves all fresh handoffs, delays consumption until execution is committed, hardens sidecar writes, persists the documented archive path, adds changelog coverage, and receives privacy/prompt review.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve every fresh model handoff:** `src/config/sessions/model-handoff.ts:291-292`
  - body: `cleanupHandoffFiles` keeps only the newest five handoff files across the whole sessions directory, and `archiveAndHandoffOnModelChange` calls it immediately after creating handoffs. A default-model switch with more than five affected sessions will delete most new handoffs before those users send their first post-switch message.
  - confidence: 0.92
- **[P2] Consume handoffs only when the run will proceed:** `src/auto-reply/reply/get-reply-run.ts:275`
  - body: This reads and deletes the one-shot handoff before later early-return guards run. If the first post-switch event is an unauthorized command, empty normalized body, or another pre-run rejection, the handoff is lost without ever reaching the model.
  - confidence: 0.86
- **[P3] Persist the archived transcript path:** `src/config/sessions/store.ts:838-840`
  - body: The handoff file is written before `archiveSessionTranscripts` runs, and this later assignment only mutates the returned object. The persisted JSON never gets the documented `archivePath`, so write the handoff after archiving or rewrite it atomically with that path.
  - confidence: 0.78
- **[P3] Add the required changelog entry:** `src/config/sessions/model-handoff.ts:139-147`
  - body: This is a user-facing `feat(sessions)` change, but the PR does not update `CHANGELOG.md`. Add a single-line entry under the active release section so the model-change handoff behavior is included in release notes.
  - confidence: 0.84
