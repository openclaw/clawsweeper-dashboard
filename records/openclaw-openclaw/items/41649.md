## Summary

Keep open: current main still lacks automatic model-change transcript archive/handoff behavior, but this PR is not mergeable as-is because it lacks real behavior proof and introduces privacy-sensitive storage/lifecycle defects.

## What This Changes

The PR adds model-change session handling that archives transcripts, writes recent-user-message handoff files, rotates session IDs, wires config reload actions, and injects handoff context into the next first turn.

## Best Possible Solution

Rework the feature on current session lifecycle code with private collision-resistant atomic handoff storage, per-session retention, hot-reload and restart parity, focused tests, and redacted real behavior proof.

## Reproduction Assessment

Yes, at source level: current main only refreshes heartbeat/catalog state for model config changes, while the PR head adds archive/handoff behavior with reviewable restart, storage, retention, and consumption defects. I did not run tests because this review was read-only.

## Solution Assessment

No. The direction may be useful, but this branch is not the best mergeable solution until it is rebased to current lifecycle code, fixes the storage/lifecycle defects, and gets real behavior proof plus privacy/prompt approval.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Archive transcripts when restart reloads run:** `src/gateway/server-startup.ts:78`
  - body: The restart/startup path added here calls clearSessionModelFields, so restart-based model config reloads still leave transcripts in place and never write the handoff files this feature relies on.
  - confidence: 0.88
- **[P1] Use collision-resistant handoff paths:** `src/config/sessions/model-handoff.ts:131-132`
  - body: resolveHandoffPath maps distinct session keys such as a:b and a_b to the same JSON file. Because the file contains recent user messages, this can overwrite or expose another session's context.
  - confidence: 0.9
- **[P1] Keep handoffs per session until consumed:** `src/config/sessions/model-handoff.ts:291-292`
  - body: The cleanup loop keeps only the newest five handoff files across the whole sessions directory. A model switch affecting more than five sessions can delete fresh handoffs before those users send their first post-switch message.
  - confidence: 0.92
- **[P2] Consume handoffs only when a run will proceed:** `src/auto-reply/reply/get-reply-run.ts:275`
  - body: This consumes and deletes the one-shot handoff before downstream early-return guards run, so unauthorized control-command traffic or an empty-body path can discard the handoff without ever sending it to the model.
  - confidence: 0.86
