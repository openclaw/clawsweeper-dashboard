## Summary

Keep open: current main and v2026.6.9 still only partially mitigate stale dist module rotation, and the issue has open linked PR candidates plus an explicit maintainer/product-decision label trail for choosing the right fix boundary.

## What This Changes

The issue reports that updating or rolling back an npm-global OpenClaw install without restarting the gateway can leave the running process holding old hashed dist import paths, silently dropping inbound Signal messages with ERR_MODULE_NOT_FOUND.

## Best Possible Solution

Land one maintainer-approved, live-proven fix at the chosen owner boundary that either prevents the old-process/new-files window or makes stale dist rotation visibly restart-required before inbound messages disappear.

## Reproduction Assessment

No live current-main reproduction was established. The report gives concrete rollback steps and current source supports the stale lazy-import failure class, but the exact package-swap plus inbound Signal flow still needs real environment proof.

## Solution Assessment

No complete merged solution exists yet. A warning can be a mitigation, but maintainers still need to choose between update lifecycle enforcement, postinstall/operator warning, and runtime stale-dist detection before closing the issue.
