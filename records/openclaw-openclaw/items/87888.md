## Summary

Keep open. This protected maintainer PR is a correct-looking construction-plan-level fix for a source-reproducible no-tools overhead path, but current main does not already implement it and https://github.com/openclaw/openclaw/pull/88142 is a duplicate open implementation choice.

## What This Changes

The PR passes model tool-support capability into embedded attempt tool-construction planning and adds planner plus runner tests for skipping local tool prep when `compat.supportsTools` is false.

## Best Possible Solution

Land one canonical no-tools guard, preferably at the construction-plan boundary so downstream code sees the correct no-tools state, then close or supersede the duplicate PR and track compaction separately only if maintainers want that surface covered too.

## Reproduction Assessment

Yes. Source inspection of current main shows the normal embedded attempt can plan and build local tools before `supportsModelTools` disables model-visible tools for `compat.supportsTools: false`.

## Solution Assessment

Yes for the primary attempt runner: moving the selected-model capability into `resolveEmbeddedAttemptToolConstructionPlan` is a cleaner owner-boundary fix than gating only at the later construction call site. The remaining maintainer choice is duplicate-PR selection and whether compact-runner coverage belongs in this PR or a follow-up.
