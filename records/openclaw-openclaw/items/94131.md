Close: current main already restores standard Telegram text delivery by default by gating Bot API rich messages behind `channels.telegram.richMessages: true`; the fix is main-only and the linked repair PR is not the canonical Telegram fix.

Telegram iOS users report that v2026.6.8 bot replies ignore the app font-size setting because outbound replies use the new rich-message transport.

## Best Possible Solution

Keep the current main behavior: standard Telegram text delivery by default, with Bot API 10.1 rich messages available only through explicit `channels.telegram.richMessages: true`.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: v2026.6.8 sends ordinary Telegram text through `sendRichMessage`, while current main no longer does unless `richMessages` is enabled. I did not rerun a live Telegram iOS font-size test.

Yes. Gating Bot API rich messages behind an explicit account setting is the narrowest maintainable fix for the reported client-rendering regression and avoids adding a separate parse-mode policy knob for this cleanup item.
