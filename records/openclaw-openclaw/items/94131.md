## Summary

## What This Changes

The issue reports a v2026.6.8 Telegram regression where bot-sent DM text ignores the iOS app font-size setting after rich-message delivery became the default.

## Best Possible Solution

Keep the current main behavior: standard Telegram text by default, rich messages only with `channels.telegram.richMessages: true`, and ship that fix in the next release.

## Reproduction Assessment

Yes, at source/report level for v2026.6.8: release code routed default Telegram text through rich-message delivery, and two users reported the iOS font-size setting being ignored. Current main no longer follows that default path, so I did not reproduce a current-main failure.

## Solution Assessment

Yes. The current main fix is broader and cleaner than a narrow iOS-only workaround because it restores the compatible text path by default and leaves rich messages as an explicit opt-in.
