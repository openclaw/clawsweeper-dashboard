## Summary

Keep open: current main and the latest release still treat `sessions_send` target turns as generic `inter_session` data with broad skip-token exits, and no merged or open canonical item fully covers the WhatsApp/group explicit-delivery suppression described here.

## What This Changes

The issue asks for explicit `inter_session` delivery requests into WhatsApp/group sessions to produce a normal outbound message instead of being suppressed as `REPLY_SKIP`, `ANNOUNCE_SKIP`, or `NO_REPLY`.

## Best Possible Solution

Add a narrow `sessions_send` inter-session delivery-intent guard in the existing prompt/runtime contract so explicit allowed channel-delivery requests produce visible messages while coordination chatter can still use skip tokens.

## Reproduction Assessment

Yes, at source level: current main marks `sessions_send` target turns as `inter_session`, offers broad skip/silent exits, and suppresses exact control tokens before visible delivery. I did not establish a live WhatsApp group reproduction in this read-only pass.

## Solution Assessment

Yes for keeping this open and routing a narrow repair. The first fix should refine the existing prompt/control-token contract with focused regression coverage; a typed delivery mode can remain a follow-up if maintainers want a larger public API contract.
