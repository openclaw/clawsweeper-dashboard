## What This Changes

The issue requests that explicit `sessions_send`/`inter_session` delivery requests to a WhatsApp group emit the normal channel message instead of being suppressed as `REPLY_SKIP`, `ANNOUNCE_SKIP`, or `NO_REPLY`.

## Best Possible Solution

Add a narrow `sessions_send` inter-session delivery-intent guard in prompt/runtime behavior so explicit allowed channel-delivery requests produce a normal visible message while coordination-only chatter can still use skip tokens.

Fix the `sessions_send` inter-session delivery precedence bug reported in https://github.com/openclaw/openclaw/issues/49223. On current main, `sessions_send` target turns are marked `inter_session`, explicit channel-delivery requests can be treated as coordination chatter, and exact `REPLY_SKIP` / `ANNOUNCE_SKIP` / `NO_REPLY` tokens suppress visible delivery. Add a narrow guard/prompt/runtime contract so an explicit allowed request to emit a normal target-channel message outranks generic A2A ping-pong or silent-token guidance, while coordination-only chatter can still use those tokens.
