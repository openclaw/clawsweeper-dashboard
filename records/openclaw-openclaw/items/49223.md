## Summary

Keep open: current main still has the reported source-level precedence path, with free-text `sessions_send` inter-session input, broad skip-token guidance, and runtime suppression before visible delivery.

## What This Changes

The issue asks OpenClaw to prefer normal channel-message delivery over `REPLY_SKIP`, `ANNOUNCE_SKIP`, or `NO_REPLY` when an authorized WhatsApp `inter_session` turn clearly requests outbound group delivery.

## Best Possible Solution

Add a narrow `sessions_send`/A2A prompt and runtime guard that preserves coordination silence tokens but tells recipients explicit channel-delivery requests should emit a normal message when policy allows, with focused regression coverage for the WhatsApp group shape.

## Reproduction Assessment

Yes, at source level: current main marks `sessions_send` turns as `inter_session`, teaches broad skip tokens, and drops exact control-token replies before gateway delivery. I did not establish a live WhatsApp group reproduction in this read-only pass.

## Solution Assessment

Yes for keeping this open and routing a narrow bugfix. The safer first fix is prompt/control-token guidance plus regression tests; a new typed delivery mode should be a follow-up only if that narrow fix cannot make explicit delivery intents reliable.
