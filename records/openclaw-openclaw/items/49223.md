- `P2`: The issue describes a bounded sessions_send delivery reliability bug that can suppress expected channel output without making the core runtime unusable.

## Summary

Keep open: current main and v2026.6.10 still route `sessions_send` as generic `inter_session` data, still invite exact skip/no-reply control tokens, and still suppress those tokens before channel delivery. The broader structured handoff issue overlaps, but it does not replace this distinct WhatsApp explicit-delivery symptom.

## What This Changes

The issue reports that WhatsApp inter-session delivery requests can be suppressed by skip/no-reply control-token guidance instead of posting the requested group message.

## Best Possible Solution

Define a maintainer-approved sessions_send/inter-session delivery-intent contract that preserves legitimate silent coordination while making explicit channel delivery requests produce normal outbound messages, with focused A2A and WhatsApp-like group regression coverage.

## Reproduction Assessment

Not as a high-confidence live WhatsApp reproduction. Source inspection gives a medium-confidence path: `sessions_send` creates an `inter_session` turn, current prompts still invite skip tokens, and exact non-deliverable replies still short-circuit delivery.

## Solution Assessment

No current implementation is ready to close this. The right fix needs maintainer choice between prompt narrowing, a typed delivery-intent signal, or folding this into the broader structured A2A handoff contract.
