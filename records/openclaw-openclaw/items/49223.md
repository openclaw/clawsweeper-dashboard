## Summary

Keep open: current `main` and the latest release still expose the reported source-level path where `sessions_send` target turns are marked as `inter_session`, broad skip/silent guidance remains available, and exact control tokens suppress visible delivery. The related open items cover adjacent dispatch/private-delivery semantics, but they are not merged and do not supersede this explicit delivery-precedence bug.

## What This Changes

The issue asks OpenClaw to prevent explicit WhatsApp group delivery requests arriving through `sessions_send` inter-session turns from being suppressed as skip/no-reply control tokens.

## Best Possible Solution

Add a narrow `sessions_send` inter-session delivery-intent guard in prompt/runtime behavior so explicit allowed channel-delivery requests produce a normal visible message while coordination-only chatter can still use skip tokens.

## Reproduction Assessment

Yes at source level: current main annotates `sessions_send` target turns as `inter_session`, gives broad skip/silent exits, and suppresses exact control tokens before visible delivery. I did not establish a live WhatsApp group repro in this read-only pass.

## Solution Assessment

Yes for keeping this open and routing a narrow bugfix. The first fix should refine the existing prompt/control-token contract with focused regression coverage; a typed delivery mode can remain a follow-up if maintainers want a larger public API contract.
