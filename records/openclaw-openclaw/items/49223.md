## Summary

Keep open: current main contains partial hardening for exact control-token echoes and same-session direct delivery, but the inter-session prompt and announce guidance can still make explicit delivery requests look like coordination data and still offers skip/silent responses without a delivery-intent distinction.

## What This Changes

The issue requests that `sessions_send`/`inter_session` delivery requests to WhatsApp group sessions emit the intended normal channel message instead of being suppressed as `REPLY_SKIP`, `ANNOUNCE_SKIP`, or `NO_REPLY`.

## Best Possible Solution

Keep this open and repair the existing sessions_send/inter_session contract so explicit channel-delivery requests are model-visible as delivery work, while exact control tokens remain suppressed for true coordination chatter.

## Reproduction Assessment

Yes at source level: current main still combines inter-session safety wording with skip-token announce guidance and lacks a delivery-intent distinction. I did not run a live WhatsApp group repro, so this is not marked as fully reproduced.

## Solution Assessment

Yes, a narrow fix in the sessions_send/inter_session prompt and tool-result metadata path is the best first repair. A new typed delivery mode should be deferred unless the narrow repair cannot make the intended behavior reliable.
