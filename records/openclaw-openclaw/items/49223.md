## Summary

Keep open: current main and the latest release still allow explicit `sessions_send` / `inter_session` delivery requests to pass through broad skip-token guidance and non-deliverable-token suppression, while the related open PRs address adjacent delivery or trust-boundary problems rather than a merged fix for this behavior.

## What This Changes

The issue asks for WhatsApp group `sessions_send` / `inter_session` delivery requests to emit explicit outbound messages instead of falling through to `REPLY_SKIP`, `ANNOUNCE_SKIP`, or `NO_REPLY`.

## Best Possible Solution

Land a narrow sessions_send/A2A fix that distinguishes explicit delivery intent from coordination chatter while preserving legitimate skip-token silence and the existing target-channel announce contract.

## Reproduction Assessment

No, not as a live WhatsApp reproduction in this read-only pass. The source path is clear: explicit `sessions_send` handoffs become `inter_session` turns, broad skip-token prompts remain, and non-deliverable token replies still short-circuit delivery.

## Solution Assessment

Yes for a narrow bug fix, not for closure. The maintainable path is to refine A2A intent handling and add regression coverage, rather than adding a public config knob or accepting the broader requester-delivery PRs as-is.
