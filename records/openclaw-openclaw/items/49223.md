## Label Justifications

- `P2`: The issue describes a bounded sessions_send delivery reliability bug that can suppress expected channel output without making the core runtime unusable.
- `impact:message-loss`: The reported failure mode silently drops an intended outbound group message.

## Summary

Keep open: current main and v2026.6.8 still route `sessions_send` turns as generic `inter_session` data, still offer broad skip/no-reply control-token guidance, and still suppress exact non-deliverable tokens without a delivery-intent override; related items cover adjacent handoff/routing problems but no merged fix resolves this report.

## What This Changes

The issue reports that WhatsApp inter-session delivery requests can be suppressed as `REPLY_SKIP`, `ANNOUNCE_SKIP`, or `NO_REPLY` instead of emitting the requested group message.

## Best Possible Solution

Define a default-compatible sessions_send/inter-session delivery-intent behavior that preserves legitimate skip-token silence but makes explicit channel delivery requests produce normal outbound messages, with focused regression coverage for the A2A and WhatsApp-like group path.

## Reproduction Assessment

No, not as a high-confidence live WhatsApp reproduction. The source path is clear enough to keep open: explicit `sessions_send` handoffs become `inter_session` turns, broad skip-token prompts remain, and non-deliverable token replies still short-circuit delivery.

## Solution Assessment

No for closure. The best fix is not the adjacent control-token leakage suppression; it needs an owner-approved delivery-intent rule or prompt/runtime guard before explicit delivery requests enter the generic skip path.
