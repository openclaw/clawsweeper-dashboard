## Summary

Keep open: current main has partial hardening for same-session and external-route delivery, but it still lets an explicit `sessions_send` handoff become an `inter_session` turn whose model-facing guidance and A2A control-token handling can suppress a real outbound group message.

## What This Changes

The issue requests that explicit WhatsApp `inter_session` delivery requests prefer normal outbound group-message emission over `REPLY_SKIP`, `ANNOUNCE_SKIP`, or `NO_REPLY`.

## Best Possible Solution

Keep the issue open and land a narrow sessions_send/A2A fix that marks or explains explicit delivery intent while preserving legitimate coordination silence.

## Reproduction Assessment

Not fully. The source path is clear on current main, but this read-only review did not run a live WhatsApp group repro against current main.

## Solution Assessment

Yes for the next repair shape. A bounded prompt/tool-result metadata fix plus regression tests is more maintainable than adding a public config knob or broad new mode first.
