## Summary

Keep open: current main still has the source-level precedence path where `sessions_send` inter-session turns can be guided toward skip/no-reply control tokens and those exact tokens are suppressed before channel delivery. The related Telegram/thread-routing fixes address adjacent announce delivery bugs, but they do not add an explicit-delivery exception for WhatsApp group handoffs.

## What This Changes

The issue reports that WhatsApp `inter_session` delivery requests can be suppressed as `REPLY_SKIP`, `ANNOUNCE_SKIP`, or `NO_REPLY` instead of posting the requested group message.

## Best Possible Solution

Add a narrow `sessions_send`/A2A prompt and runtime guard so explicit channel-delivery requests emit a normal message when policy allows, while preserving skip tokens for coordination-only ping-pong suppression.

## Reproduction Assessment

Yes, at source level: current main marks `sessions_send` turns as `inter_session`, gives broad skip-token guidance, and suppresses exact control-token replies before gateway delivery. I did not establish a live WhatsApp group reproduction in this read-only pass.

## Solution Assessment

Yes for keeping this open and routing a narrow bugfix. The best first fix is prompt/control-token guidance plus regression coverage; a new typed delivery mode should be a follow-up only if the narrow fix cannot make explicit delivery intents reliable.
