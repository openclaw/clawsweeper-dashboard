Keep open: current main and the latest release still label `sessions_send` target turns as `inter_session`, still provide broad skip/silent-token guidance, and still suppress exact control-token replies before visible delivery. ## What This Changes

Ensure explicit WhatsApp/inter-session `sessions_send` delivery requests prefer normal channel output instead of being suppressed by A2A skip or no-reply tokens.

## Best Possible Solution

Add a narrow `sessions_send` inter-session delivery-intent guard in prompt/runtime behavior so explicit allowed channel-delivery requests produce a normal visible message while coordination-only chatter can still use skip tokens.
