## Label Justifications

- `P2`: The issue describes a bounded sessions_send delivery reliability problem that can suppress expected channel output without making the core runtime unusable.
- `impact:message-loss`: The reported failure mode silently drops an intended outbound group message.

## Summary

Keep open: current main and the latest release still route `sessions_send` as generic `inter_session` data, still teach exact skip/no-reply control tokens, and still suppress those tokens before channel delivery. The open structured handoff issue overlaps, but it does not replace this distinct WhatsApp explicit-delivery symptom.

## What This Changes

The issue reports that WhatsApp inter-session delivery requests can be suppressed as skip/no-reply control output instead of emitting an explicit group message.

## Best Possible Solution

Define a maintainer-approved `sessions_send`/inter-session delivery-intent contract that preserves legitimate silent coordination while making explicit channel delivery requests produce normal outbound messages, with focused A2A and WhatsApp-like group regression coverage.

## Reproduction Assessment

No live high-confidence WhatsApp repro was established in this read-only review. Source inspection gives a medium-confidence path: current main routes `sessions_send` as `inter_session`, invites skip tokens, and suppresses exact non-deliverable replies before channel delivery.

## Solution Assessment

No current implementation is ready to close this. The best fix needs maintainer choice between prompt narrowing, a typed delivery-intent signal, or folding the behavior into the broader structured A2A handoff contract.
