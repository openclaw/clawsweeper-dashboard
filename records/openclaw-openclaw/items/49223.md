## Summary

Keep open: current main and the latest release still route `sessions_send` target turns as generic `inter_session` data, still expose broad skip-token exits, and do not give explicit allowed channel-delivery requests a higher-precedence path over `REPLY_SKIP`, `ANNOUNCE_SKIP`, or `NO_REPLY`.

## What This Changes

Requests that explicit WhatsApp group `sessions_send` / `inter_session` delivery intents emit normal outbound messages instead of being suppressed as skip or silent tokens.

## Best Possible Solution

Add a narrow `sessions_send` inter-session delivery-intent guard in the existing prompt/runtime contract so explicit allowed channel-delivery requests produce visible messages while coordination chatter can still use skip tokens.

## Reproduction Assessment

Yes at source level: current main annotates `sessions_send` target turns as `inter_session`, provides broad skip/silent exits, and suppresses exact control tokens before visible delivery. I did not establish a live WhatsApp group repro in this read-only pass.

## Solution Assessment

Yes for keeping this open and routing a narrow bugfix. The first fix should refine the existing prompt/control-token contract with focused regression coverage; a typed delivery mode can remain a follow-up if maintainers want a larger public API contract.
