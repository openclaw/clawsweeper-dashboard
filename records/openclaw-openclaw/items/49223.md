Keep open: current main and the latest release still expose the source-level path where `sessions_send` target turns are marked as `inter_session`, broad skip/silent guidance remains available, and exact control tokens suppress visible delivery. Related work such as https://github.com/openclaw/openclaw/pull/76490 and https://github.com/openclaw/openclaw/pull/50720 overlaps adjacent A2A delivery semantics, but neither is merged nor a complete superseding fix for this WhatsApp explicit-delivery precedence bug.

## What This Changes

The issue asks for `sessions_send` inter-session turns that explicitly request a normal WhatsApp or group channel message to deliver that message instead of being suppressed by skip or no-reply control tokens.

## Best Possible Solution

Add a narrow `sessions_send` inter-session delivery-intent guard in prompt/runtime behavior so explicit allowed channel-delivery requests produce a normal message while coordination-only chatter can still use skip tokens.
