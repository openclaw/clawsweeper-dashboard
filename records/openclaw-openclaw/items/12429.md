## Summary

Keep this issue open: current main now supports zero-touch first-time node enrollment from trusted CIDRs, but it still lacks Kubernetes ServiceAccount identity, TokenReview validation, autoscaled agent-worker architecture, and corresponding deployment assets.

## What This Changes

Requests secure headless authentication for dynamically autoscaled Kubernetes worker nodes using ServiceAccount identity, TokenReview validation, and Kubernetes deployment assets.

## Best Possible Solution

Define a generic, fail-closed workload-attestation contract for runtime registration—including identity claims, audience, allowlists, credential lifecycle, and pairing semantics—then implement Kubernetes TokenReview as a narrow adapter and keep deployment packaging separately owned.

## Reproduction Assessment

Not applicable: this requests a new workload-identity and distributed deployment capability rather than reporting broken behavior under an existing contract. Current source does provide a concrete CIDR-based node-enrollment alternative for comparison.

## Solution Assessment

No, the closed all-in-one Kubernetes PR is not the best maintainable solution because it combined protocol, authentication, runtime, deployment, documentation, and unrelated changes. The narrowest durable path is a generic workload-attestation boundary followed by an optional Kubernetes adapter.
