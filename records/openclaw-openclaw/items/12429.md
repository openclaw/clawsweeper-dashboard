Keep open. Current main still has no Kubernetes ServiceAccount Trust or TokenReview authentication path, and the related implementation PR is closed unmerged while the distributed runtime RFC is broader than this specific auth request.

## What This Changes

The issue requests opt-in Kubernetes ServiceAccount Trust authentication plus Kubernetes deployment assets so autoscaled worker/runtime pods can connect to a gateway without interactive pairing.

## Best Possible Solution

Keep this issue open until maintainers choose and review a concrete core design for K8s worker/runtime authentication, or explicitly fold this specific SA Trust requirement into a narrower distributed-runtime follow-up.
