## Summary

Keep open: current main and v2026.6.6 have the simpler Kubernetes install/token-auth baseline, but they still do not provide a Kubernetes ServiceAccount TokenReview credential, auth mode, or pairing-skip policy for autoscaled worker pods. The linked implementation PR was closed unmerged, and the broader runtime RFCs overlap without replacing this focused auth request.

## What This Changes

The issue requests an opt-in Kubernetes ServiceAccount TokenReview authentication path so autoscaled worker pods can pair with a gateway without interactive PIN/device pairing.

## Best Possible Solution

Keep this as the focused Kubernetes worker identity/authentication design item until maintainers either approve an SA-trust path or explicitly fold the remaining auth contract into a narrower distributed-runtime plan.
