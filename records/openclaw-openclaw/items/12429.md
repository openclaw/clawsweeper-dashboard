Keep open. Current main still lacks the requested Kubernetes ServiceAccount Trust / TokenReview authentication path, the linked implementation PR was closed unmerged, and the remaining work is a security-sensitive gateway auth/protocol feature rather than cleanup that can be closed.

## What This Changes

The issue requests an opt-in Kubernetes ServiceAccount Trust/TokenReview auth path, plus deployment assets, so autoscaled headless worker or node pods can connect to a gateway without interactive pairing.

## Best Possible Solution

Keep this open until maintainers approve a concrete opt-in Kubernetes worker identity design or explicitly fold the SA Trust requirement into a narrower distributed-runtime protocol plan.
