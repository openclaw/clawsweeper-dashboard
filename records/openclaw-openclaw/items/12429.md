## Summary

Keep open: current main still lacks an opt-in Kubernetes ServiceAccount Trust/TokenReview path for autoscaled node or worker pods, and the existing Kubernetes support is a single-pod Kustomize deployment using shared token auth. The linked implementation PR is closed unmerged, while the related distributed runtime RFC is broader and does not specifically replace this authentication request.

## What This Changes

Request an opt-in Kubernetes ServiceAccount Trust authentication method plus Kubernetes deployment assets so headless autoscaled worker nodes can authenticate to a gateway without interactive pairing.

## Best Possible Solution

Keep the issue open until maintainers either design and implement a secure opt-in Kubernetes ServiceAccount Trust path or fold the exact auth requirement into a narrower canonical distributed-runtime/Kubernetes deployment plan.
