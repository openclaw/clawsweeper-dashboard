## Summary

Keep open. Current main still does not implement Kubernetes ServiceAccount Trust / TokenReview authentication for headless autoscaled worker or runtime pods, and the remaining work is a security-sensitive gateway auth/protocol feature rather than cleanup that can be closed.

## What This Changes

Request an opt-in Kubernetes ServiceAccount Trust authentication path and deployment assets so worker nodes can pair with a gateway automatically in Kubernetes.

## Best Possible Solution

Keep this open until maintainers approve and implement an explicit Kubernetes worker identity design or fold the requirement into a narrower distributed-runtime auth plan.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection of current main gives high confidence that the requested config, protocol, runtime credential, and TokenReview surfaces are still absent.

## Solution Assessment

Unclear pending maintainer product and security design. Kubernetes SA Trust is plausible, but the maintainable solution needs reviewed gateway auth/config, protocol payloads, pairing-bypass policy, node-host token handling, TokenReview validation, tests, docs, and deployment assets.
