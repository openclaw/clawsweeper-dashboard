## Summary

Keep open: current main and v2026.6.8 still lack Kubernetes ServiceAccount TokenReview auth for autoscaled worker/node pairing. The merged Kubernetes install docs and trusted-CIDR node auto-approval path partially reduce setup friction, but they are not workload-identity authentication, and the linked SA Trust PR was closed unmerged rather than becoming a shipped or canonical replacement.

## What This Changes

The issue requests an opt-in Kubernetes ServiceAccount TokenReview authentication path so autoscaled worker pods can connect to a gateway without interactive device pairing.

## Best Possible Solution

Keep this as the canonical Kubernetes workload-identity pairing request until maintainers approve a focused SA Trust design or explicitly choose existing CIDR auto-approval plus the distributed-runtime roadmap as the supported path.
