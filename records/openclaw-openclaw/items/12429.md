Keep open. Current main still lacks the requested Kubernetes ServiceAccount Trust / TokenReview authentication path, and the remaining work is a security-sensitive gateway auth/protocol feature rather than stale cleanup or an already-shipped implementation.

## What This Changes

Request adds opt-in Kubernetes ServiceAccount Trust / TokenReview authentication so autoscaled Kubernetes worker or node pods can connect without interactive pairing, plus deployment assets.

## Best Possible Solution

Keep this open until maintainers approve a concrete, opt-in Kubernetes worker identity design or explicitly fold the same SA Trust requirement into a narrower distributed-runtime follow-up.

## Solution Assessment

Unclear pending maintainer security and product design.
