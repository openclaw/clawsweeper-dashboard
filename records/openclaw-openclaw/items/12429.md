## Summary

Keep open. Current main has a minimal Kubernetes/Kustomize deployment and an opt-in trusted-CIDR node auto-approval path, but it still has no Kubernetes ServiceAccount Trust / TokenReview auth path for headless autoscaled worker or node pods.

## What This Changes

Request to add opt-in Kubernetes ServiceAccount TokenReview authentication so autoscaled headless worker/node pods can connect without interactive pairing, plus Kubernetes deployment assets.

## Best Possible Solution

Keep this feature open until maintainers approve a Kubernetes worker identity design or explicitly fold the requirement into a narrower distributed-runtime auth plan.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main lacks the requested config, protocol, runtime credential, and TokenReview validation surfaces.

## Solution Assessment

Unclear pending maintainer product and security design. Kubernetes SA Trust is plausible, but the maintainable solution needs reviewed gateway auth/config, protocol payloads, pairing-bypass policy, node-host token handling, TokenReview validation, tests, docs, and deployment assets.
