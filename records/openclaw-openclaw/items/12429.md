## Summary

Keep open. Current main still does not implement the requested Kubernetes ServiceAccount Trust / TokenReview authentication path for headless autoscaled worker or node pods; the current Kubernetes assets are single-pod/token-auth install scaffolding, and the linked implementation PR was closed unmerged in favor of a simpler Kubernetes baseline.

## What This Changes

Request an opt-in Kubernetes ServiceAccount Trust authentication mode plus deployment assets so autoscaled worker pods can authenticate to the gateway without interactive pairing.

## Best Possible Solution

Keep this feature open until maintainers approve a Kubernetes worker identity design or explicitly fold the requirement into a focused distributed-runtime authentication plan.

## Solution Assessment

Unclear pending maintainer auth/security/product design. Kubernetes ServiceAccount Trust is plausible, but a maintainable solution needs reviewed config/schema, protocol payloads, token handling, pairing-skip policy, fail-closed TokenReview validation, tests, docs, and deployment assets.
