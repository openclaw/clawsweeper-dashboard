## Summary

Keep open. Current main and v2026.5.12 still lack the requested Kubernetes ServiceAccount/TokenReview authentication path; existing CIDR auto-approval is a different opt-in trust model, and the closed broad PR plus open distributed-runtime RFC do not supersede this focused security-sensitive request.

## What This Changes

Request an opt-in Kubernetes ServiceAccount Trust authentication path so autoscaled worker nodes can pair with a gateway without interactive PIN/device pairing.

## Best Possible Solution

Keep this as a focused auth/design item until maintainers either approve an opt-in Kubernetes ServiceAccount Trust design or explicitly document CIDR auto-approval or the distributed-runtime plan as the supported replacement.
