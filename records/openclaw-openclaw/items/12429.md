## Summary

Keep open. Current main still does not implement Kubernetes ServiceAccount Trust / TokenReview authentication for headless autoscaled worker or node pods, and the remaining work is a security-sensitive gateway auth/protocol feature rather than cleanup that can be closed.

## What This Changes

Request an opt-in Kubernetes ServiceAccount Trust auth path so autoscaled worker pods can authenticate to a gateway without interactive device pairing, plus deployment assets.

## Best Possible Solution

Keep this open until maintainers approve a concrete opt-in Kubernetes worker identity design or explicitly fold the SA Trust requirement into a narrower distributed-runtime protocol plan.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection of current main shows no config, protocol, node-host, or deployment surface for Kubernetes ServiceAccount TokenReview auth.

## Solution Assessment

Unclear pending maintainer security and product design. Kubernetes SA Trust is plausible, but the maintainable solution needs reviewed gateway auth, protocol, pairing-bypass policy, node-host credential handling, TokenReview validation, tests, docs, and deployment changes rather than a small autonomous patch.
