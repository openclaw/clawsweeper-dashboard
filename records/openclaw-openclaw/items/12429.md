Keep open. Current main still lacks the requested Kubernetes ServiceAccount Trust / TokenReview authentication path, and the remaining work is a security-sensitive gateway auth/protocol feature rather than an already-shipped implementation.

## What This Changes

Request adds opt-in Kubernetes ServiceAccount Trust / TokenReview auth so autoscaled node or worker pods can join a gateway headlessly, plus Kubernetes and Helm deployment assets.

## Best Possible Solution

Keep this open until maintainers approve a concrete opt-in Kubernetes worker identity design or explicitly fold the SA Trust requirement into a narrower distributed-runtime follow-up.

## Solution Assessment

Unclear pending maintainer security and product design.
