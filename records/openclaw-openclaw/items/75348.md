## Summary

Keep open: current main still leaves GitHub Copilot Claude/Anthropic-compatible model definitions without the compat override that stops pi-ai from sending the rejected `tools[].eager_input_streaming` field, and the linked closing PR remains open rather than merged.

## What This Changes

The issue asks OpenClaw to disable eager tool input streaming for GitHub Copilot Anthropic-compatible models so Copilot Claude tool calls no longer fail schema validation.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/75393 or a replacement that applies the provider-local Anthropic compat override across static manifest/default rows, live catalog rows, dynamic fallback rows, normalization/schema, and focused regression coverage.

## Reproduction Assessment

Yes. The issue and follow-up comment give concrete `400 tools.0.custom.eager_input_streaming` logs, and current source still resolves Copilot Claude models without the pi-ai compat flag that suppresses that field.

## Solution Assessment

Yes, the provider-local compat flag is the right mechanism, but it needs to be applied through the complete Copilot model catalog path rather than only the dynamic fallback helper.
