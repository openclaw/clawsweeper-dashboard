## Summary

Keep this issue open. Current main still lacks the Copilot-owned compat metadata needed to stop pi-ai from emitting `tools[].eager_input_streaming`, and the linked closing PR is open rather than merged proof.

## What This Changes

The issue requests that GitHub Copilot Claude model definitions disable pi-ai eager tool input streaming so Copilot’s Anthropic-compatible proxy stops rejecting tool payloads.

## Best Possible Solution

Land a provider-local GitHub Copilot fix that ensures every Copilot model using `anthropic-messages`, including manifest/default catalog rows and synthetic forward-compatible models, carries `compat.supportsEagerToolInputStreaming: false`, with focused regression coverage and the required changelog entry.

Yes. The issue provides a concrete v2026.4.29 CLI command and gateway logs showing Copilot rejecting `tools.0.custom.eager_input_streaming`, and source inspection confirms current main still allows pi-ai’s default behavior on Copilot Anthropic-compatible models.

## Solution Assessment
