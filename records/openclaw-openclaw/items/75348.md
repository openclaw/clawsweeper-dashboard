## Summary

Keep this open. Current main and the shipped v2026.4.29 tag still lack the Copilot-owned `compat.supportsEagerToolInputStreaming: false` metadata on Anthropic-compatible Copilot model paths, and PR #75393 is an open closing implementation candidate rather than merged evidence.

## What This Changes

The issue asks OpenClaw to stop Copilot Anthropic-compatible model requests from sending `tools[].eager_input_streaming` by setting the Copilot model compat override.

## Best Possible Solution

Land a provider-local Copilot fix that sets `compat.supportsEagerToolInputStreaming: false` for every Copilot model resolved with `anthropic-messages`, including helper-built defaults, synthetic Claude IDs, and manifest catalog rows that participate in runtime resolution.

## Reproduction Assessment

Yes. The issue gives a concrete v2026.4.29 CLI command and gateway logs, and current source plus the pi-ai contract confirm the same metadata gap still exists on current main.

## Solution Assessment

Yes. A narrow Copilot provider metadata fix is the maintainable path because the rejection is specific to Copilot's Anthropic-compatible proxy; changing core Anthropic transport behavior or pi-ai defaults would be broader than necessary.
