## Summary

Keep this open: current main still lacks the Copilot-owned compat metadata needed to stop pi-ai from adding `tools[].eager_input_streaming`, and PR #75393 is an open unmerged closing candidate rather than implementation evidence.

## What This Changes

The issue asks OpenClaw to set `compat.supportsEagerToolInputStreaming: false` for GitHub Copilot Anthropic-compatible model definitions so Copilot requests no longer send rejected tool payload fields.

## Best Possible Solution

Land PR #75393 or a replacement that applies the Copilot provider-local compat flag across all Anthropic-compatible Copilot model sources, with focused coverage for dynamic defaults and static catalog normalization if those rows feed runtime selection.

## Reproduction Assessment

Yes. The issue gives a concrete v2026.4.29 command and gateway log, and current main plus the pinned pi-ai 0.71.1 contract show the same metadata gap remains.

## Solution Assessment

Yes. A narrow Copilot provider metadata fix is the maintainable solution; broad core transport or dependency-default changes would affect providers that may legitimately support eager tool streaming.
