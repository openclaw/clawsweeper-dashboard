## Summary

Keep open: current main still has the Copilot-owned compat metadata gap, and the linked closing PR remains open and unmerged rather than implementation evidence.

## What This Changes

The issue asks OpenClaw to prevent GitHub Copilot Anthropic-compatible model requests from sending `tools[].eager_input_streaming`, which Copilot rejects.

## Best Possible Solution

Land #75393 or a replacement that applies the Copilot provider-local compat flag across every Anthropic-compatible Copilot model source, preserves it through catalog/config normalization where needed, and adds focused regression coverage plus a changelog entry.

## Reproduction Assessment

Yes. The issue gives a concrete v2026.4.29 command and gateway log, and current main source plus the pinned pi-ai contract show the same metadata gap; I did not run a live credentialed Copilot request in this read-only review.

## Solution Assessment

Yes. A narrow Copilot provider metadata fix is the maintainable direction; changing core transports or the dependency default would affect other Anthropic-compatible providers that may legitimately support eager tool input streaming.
