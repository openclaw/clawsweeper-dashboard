## Summary

Keep open: current main still lacks a comprehensive Copilot-owned compat override for Anthropic-message Copilot models, and the open closing PR is not merged implementation evidence.

## What This Changes

The issue requests disabling pi-ai eager tool input streaming for GitHub Copilot Anthropic model definitions so Copilot stops rejecting tool payloads.

## Best Possible Solution

Apply a Copilot-owned compat override for every Anthropic-message Copilot model source, including registry hits, generated/default/synthetic rows, and manifest catalog handling, then land #75393 or a replacement with regression coverage.

## Reproduction Assessment

Yes, at source level. The issue provides a concrete v2026.4.29 command and gateway log, and current main plus pi-ai 0.71.1 still show Anthropic Copilot paths that omit compat.supportsEagerToolInputStreaming false and therefore inherit eager_input_streaming emission.

## Solution Assessment

Yes, the narrow maintainable direction is provider-local Copilot compat metadata rather than changing core Anthropic transport defaults. The fix should cover all Copilot Anthropic model sources, not only the two source builders in the open PR.
