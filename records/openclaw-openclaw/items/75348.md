## Summary

Keep this open: current main still lacks the Copilot-owned eager tool streaming opt-out on Anthropic-compatible Copilot model paths, and PR #75393 is an open, unmerged closing implementation candidate rather than proof of a fix.

## What This Changes

The issue reports that GitHub Copilot Claude requests still fail because pi-ai emits `tools[].eager_input_streaming` unless Copilot model metadata opts out.

## Best Possible Solution

Land or amend PR #75393 so every Copilot model resolved with `anthropic-messages`, including static manifest rows if used at runtime, carries `compat.supportsEagerToolInputStreaming: false` with focused regression coverage.

## Reproduction Assessment

Yes. The issue gives a concrete v2026.4.29 CLI command and gateway log, and current main plus the pi-ai 0.71.1 contract show the same missing opt-out that would omit `tools[].eager_input_streaming`.

## Solution Assessment

Yes. A provider-local Copilot metadata opt-out is the narrowest maintainable fix; changing core Anthropic transport behavior or pi-ai defaults would be broader than necessary.
