## Summary

Keep open: current main still lacks the Copilot-owned compat metadata needed to stop pi-ai from adding `tools[].eager_input_streaming`, and the linked closing PR is still open rather than merged implementation evidence.

## What This Changes

The report asks OpenClaw to stop sending pi-ai's `tools[].eager_input_streaming` field for GitHub Copilot Claude models by setting Anthropic compat metadata in the Copilot provider.

## Best Possible Solution

Land #75393 or a replacement that applies the Copilot provider-local compat flag across static, default, and dynamic Anthropic-compatible Copilot model sources, with normalization/schema coverage where compat metadata is accepted.

## Reproduction Assessment

Yes. The issue provides a concrete v2026.4.29 command and gateway log, and current main source plus the pinned pi-ai contract show the same metadata gap remains; I did not run a live Copilot request.

## Solution Assessment

Yes. A narrow Copilot provider metadata fix is the maintainable path; changing core transports or pi-ai defaults would affect other Anthropic-compatible providers that may legitimately support eager tool input streaming.
