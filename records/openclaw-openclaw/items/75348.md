Keep open: current main still leaves Copilot Claude/Anthropic-compatible model definitions without the compat flag needed to stop pi-ai from sending `tools[].eager_input_streaming`, and the linked closing PR is still open rather than merged implementation evidence.

## What This Changes

The issue requests disabling pi-ai eager tool input streaming for GitHub Copilot Claude models so Copilot no longer rejects tool schemas.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/75393 or an equivalent provider-local fix after it covers static manifest rows, live catalog rows, dynamic fallback rows, compat normalization/schema, and focused regression coverage.
