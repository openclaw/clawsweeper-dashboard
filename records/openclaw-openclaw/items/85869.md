## Label Justifications

- `P2`: This is a concrete plugin installation compatibility bug with a confusing WhatsApp recovery failure, but its blast radius is limited to external or floating plugin installs.
- `impact:auth-provider`: The reported failure occurs during WhatsApp relink recovery and can leave channel auth recovery blocked by an incompatible plugin load.

## Summary

Keep open: current main still lets the direct npm plugin install path rely on `openclaw.install.minHostVersion` while WhatsApp declares newer OpenClaw/plugin API requirements elsewhere, so an older host can still accept a package that later fails to load.

## What This Changes

The issue requests that plugin installation select or reject WhatsApp package versions based on compatibility with the running OpenClaw core/plugin SDK.

## Best Possible Solution

Add a generic direct-npm package install compatibility gate, aligned with the ClawHub guard, and update WhatsApp metadata/tests so older hosts reject incompatible packages before loading them.

## Reproduction Assessment

Yes, from source inspection: a host like `2026.5.10-beta.1` satisfies WhatsApp's current `minHostVersion` floor while the package also declares `>=2026.5.22` for OpenClaw/plugin API compatibility. The direct npm validation path only enforces the lower min-host field.

## Solution Assessment

Yes, the narrow maintainable fix is a generic package-install compatibility check plus aligned metadata and regression tests. A WhatsApp-only recovery special case would leave the same version-skew bug for other external plugins.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/85869. Direct npm plugin installs such as `openclaw plugins install @openclaw/whatsapp` must not persist a package whose declared OpenClaw/plugin API compatibility is newer than the running host. Add a generic package-install compatibility check, preferably reusing or mirroring the ClawHub `pluginApiRange`/minimum-host fail-closed behavior, and align WhatsApp package metadata so the enforced install contract cannot be lower than the plugin API/OpenClaw peer requirement. Add regression tests for the reported version-skew shape where host `2026.5.10-beta.1` accepts `openclaw.install.minHostVersion >=2026.4.25` but must reject a package declaring `openclaw.compat.pluginApi` or `peerDependencies.openclaw` of `>=2026.5.22`. Do not change WhatsApp phone-code login, 515 recovery, or unrelated channel auth behavior. Add a changelog entry if the fix changes user-visible CLI/install behavior.
