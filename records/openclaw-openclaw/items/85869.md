## Summary

Close as duplicate/superseded: this WhatsApp report is a concrete instance of the open generic plugin/core API compatibility problem already tracked at https://github.com/openclaw/openclaw/issues/77868, and current source still confirms the same min-host-only gap.

## What This Changes

The issue requests an install/load guard so a floating WhatsApp plugin install cannot pair a newer plugin API requirement with an older OpenClaw core without a compatible version selection or clear error.

## Best Possible Solution

Keep the shared plugin/core compatibility guard work on https://github.com/openclaw/openclaw/issues/77868 and preserve this WhatsApp report as linked field evidence for that generic fix.

## Reproduction Assessment

Yes. Source inspection shows a host such as 2026.5.10-beta.1 can satisfy WhatsApp's minHostVersion floor while the package declares a newer OpenClaw/plugin API requirement, and the npm/local install plus registry paths only enforce minHostVersion.

## Solution Assessment

No unique WhatsApp-only fix should be attached here. The maintainable solution is the generic package-install and runtime plugin API compatibility gate already tracked at https://github.com/openclaw/openclaw/issues/77868.
