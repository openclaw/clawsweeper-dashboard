## Summary

Close as duplicate: the WhatsApp install failure is the same unresolved plugin/core compatibility gap already tracked by the older open issue covering Discord, MSTeams, and generic non-ClawHub plugin API range enforcement.

## What This Changes

The issue requests that floating WhatsApp plugin installs either select a host-compatible package version, refuse incompatible packages with a clear error, or avoid overriding bundled channel plugins implicitly.

## Best Possible Solution

Track the work in https://github.com/openclaw/openclaw/issues/77868 and fix it once with a shared plugin API/OpenClaw compatibility gate for npm/local install and non-bundled registry loading.

## Reproduction Assessment

Yes. Source inspection shows a host such as 2026.5.10-beta.1 can satisfy WhatsApp's current minHostVersion while the same package declares a newer plugin API/OpenClaw peer requirement that the npm install path does not enforce.

## Solution Assessment

Yes for routing, not as a standalone issue. The maintainable solution is the generic compatibility gate already tracked in the canonical issue, not a WhatsApp-only special case.
