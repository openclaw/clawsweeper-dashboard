## Summary

Close as duplicate/superseded: current main still shows the WhatsApp install/version-skew gap, but the remaining fix is the same generic non-ClawHub plugin API compatibility gate already tracked by https://github.com/openclaw/openclaw/issues/77868.

## What This Changes

Request that floating WhatsApp plugin installs avoid installing a package whose OpenClaw/plugin SDK requirements exceed the running core version.

## Best Possible Solution

Track the shared installer/runtime plugin API compatibility fix in https://github.com/openclaw/openclaw/issues/77868 and keep the eventual implementation generic rather than WhatsApp-specific.

## Reproduction Assessment

Yes, source-reproducible: a host such as 2026.5.10-beta.1 can satisfy WhatsApp's current minHostVersion floor while failing its peer/pluginApi requirement. The current npm install validation path only checks minHostVersion.

## Solution Assessment

No separate WhatsApp-only tracker is not the best endpoint because the missing behavior is the generic plugin API compatibility gate already tracked by the canonical open issue. The requested fail-closed install guard is still the right fix shape, but it should be implemented once across npm/local install and registry loading.
