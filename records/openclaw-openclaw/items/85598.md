## Summary

Keep this PR open: it is a focused replacement for the earlier memory-core ACL fix and targets a real current-main bug, but the new helper still has a security-boundary race around path-based chmod, and the supplied proof is targeted test output rather than real after-fix behavior proof.

## What This Changes

The PR adds a memory-core dreaming private-store helper, routes dreaming JSON state writes through it, and adds focused tests for ACL blast radius and symlink-swap behavior under `memory/.dreams`.

## Best Possible Solution

Keep the workspace-root `root()` containment approach, but harden `memory/.dreams` through a pinned/no-follow directory handle with inside-root identity checks, plus a regression for the chmod race and redacted real shared-workspace proof.

## Reproduction Assessment

Yes, by source and linked live evidence: current `main` writes memory-core dreaming JSON under `memory/.dreams` through `privateFileStore(workspaceDir)`, and the related Linux ACL repro shows that private-store path chmodding the shared workspace parents.

## Solution Assessment

No. The intended owner-boundary fix is right, but the implementation should not call path-based `fs.chmod` after `Root.resolve()`; the chmod needs the same pinned/no-follow boundary discipline as the write path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Avoid path-based chmod on resolved dreams directories:** `extensions/memory-core/src/dreaming-private-store.ts:13`
  - body: `Root.resolve()` only gives an absolute path; the fs-safe docs explicitly note it does not pin a later operation. Since this PR keeps `memory/` shared, a concurrent swap of `memory/.dreams` to a symlink between `resolve()` and `fs.chmod()` can make the chmod follow that symlink and change permissions outside the workspace before the later write rejects it. Please chmod an opened `O_DIRECTORY|O_NOFOLLOW` handle and verify the directory identity is still inside the root, or use an equivalent pinned fs-safe primitive, and add a regression for that race.
  - confidence: 0.86
