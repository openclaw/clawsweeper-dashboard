## What This Changes

The PR adds LiteLLM Claude model detection to Anthropic cache-retention family logic, enables Anthropic-style cache_control for those LiteLLM OpenAI-completions models, and adds unit tests for the detector/resolver behavior.

## Best Possible Solution

Enable explicit cacheRetention for LiteLLM Claude routes at the OpenAI-completions compatibility boundary, update the LiteLLM docs, and require redacted real LiteLLM-to-Anthropic proof before merge.

## Reproduction Assessment

Yes at source level: current main excludes the default LiteLLM openai-completions route from both cache-retention family resolution and Anthropic cache_control detection. This review did not run a live LiteLLM-to-Anthropic request.

## Solution Assessment

Mostly yes: touching both retention-family resolution and OpenAI-completions cache_control detection is the narrow maintainable fix. It is not merge-ready until docs and real behavior proof catch up, and the LiteLLM alias heuristic needs explicit maintainer acceptance.
