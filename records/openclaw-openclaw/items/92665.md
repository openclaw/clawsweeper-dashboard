## Summary

Keep open: this focused bug-fix PR is still useful and not obsolete, but it is not merge-ready because the explicit-cache opt-in can be lost through `model.compat` merging, public proxy docs still contradict the new LiteLLM exception, and the supplied proof is source/terminal-level rather than a redacted live LiteLLM-to-Anthropic run.

The PR adds LiteLLM Claude detection to Anthropic cache-retention resolution, enables Anthropic `cache_control` payload shaping in `openai-completions`, and adds resolver plus serialized-payload regression tests.

## Best Possible Solution

Carry the explicit-cache gate through compat merging, add the compat-override regression, align the proxy prompt-cache docs, and require redacted live LiteLLM-to-Anthropic proof before merge.
