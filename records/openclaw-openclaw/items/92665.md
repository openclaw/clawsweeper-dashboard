## Summary

Keep this PR open, but it is still not merge-ready. The patch is in the right provider/cache seams and now has serialized-payload tests, but the explicit-only LiteLLM cache gate is still lost when `model.compat` is merged, the LiteLLM docs would become stale, and the supplied proof still does not show a live LiteLLM-to-Anthropic request with cache usage.

## What This Changes

The PR makes LiteLLM-proxied Anthropic models honor explicit `cacheRetention` by resolving them as Anthropic-family cache targets and injecting Anthropic `cache_control` markers through `openai-completions`.

## Best Possible Solution

Carry the explicit-only gate through compat merging, add a compat-override regression, update LiteLLM cache docs, and require redacted live LiteLLM-to-Anthropic proof before merge.

## Reproduction Assessment

Yes at source level: current main and v2026.6.8 omit LiteLLM from both Anthropic-family cache-retention resolution and OpenAI-completions cache_control detection. I did not run a live LiteLLM-to-Anthropic cache-hit reproduction.
