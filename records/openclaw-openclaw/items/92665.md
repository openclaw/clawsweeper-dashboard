## Summary

Keep open. This PR targets a real linked LiteLLM cacheRetention bug, but it is not merge-ready because the explicit opt-in flag is lost when model.compat is merged, the docs still describe LiteLLM/proxy routes as having no prompt-cache hints, and the supplied proof stops short of a redacted live LiteLLM-to-Anthropic run.

## What This Changes

The branch adds LiteLLM Claude detection to Anthropic-family cache semantics, enables Anthropic cache_control payload shaping for explicit LiteLLM cacheRetention on OpenAI-completions models, and adds resolver/payload tests.

## Best Possible Solution

Carry the explicit-cache flag through compat merging, add the compat-override/no-cacheRetention regression, update the LiteLLM/proxy prompt-cache docs, and require redacted live LiteLLM-to-Anthropic proof before merge.
