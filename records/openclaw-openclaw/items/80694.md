## Summary

Current main implements the central memory-recall behavior through runtime-owned startup context: recent `memory/*.md` files, including date-prefixed `session-memory` artifacts, are loaded into the first startup/reset prompt. The exact `session:start` hook was not added, but the observable gap from the linked reports is covered on main and is not yet in a tagged release.

## What This Changes

Request to add a session-start memory recall hook/config so recent saved session-memory files are automatically injected into new sessions.

## Best Possible Solution

Keep `agents.defaults.startupContext` as the canonical native recall path; track semantic relevance ranking or a public `session:start` hook separately if maintainers still want that API surface.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source and regression tests on current main show recent memory files are now injected into startup/reset prompts.

## Solution Assessment

Yes. Current main solves the core continuity problem without adding a new hook API, and the implementation quotes workspace memory as untrusted runtime-loaded context with bounded size controls.
