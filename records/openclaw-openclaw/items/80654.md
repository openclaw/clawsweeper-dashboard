## Summary

Keep open: current main has partial lineage support, but it still rotates `sessionId` on `/new` and does not provide the requested stable session identity or per-agent session ledger for memory tools. The related continuity PR is still open and narrower than this session identity contract request.

## What This Changes

Request a stable OpenClaw session identity or per-agent session ledger so memory tools can map rotated session IDs to one durable thread with timestamps and kind metadata.

## Best Possible Solution

Define a single gateway-owned session lineage contract, exposed through session storage or Gateway APIs, that records stable family/thread identity plus rotated session IDs, start/end timestamps, reset reason, kind, and documented consumers for memory and cleanup tools.

## Reproduction Assessment

Not applicable: this is an enhancement request, not a report of broken documented behavior. Source inspection confirms current main rotates `sessionId` and only preserves partial lineage through usage-family metadata.

## Solution Assessment

Unclear: the proposed `memory/<agent>-thread.md` table is a useful sketch, but the more maintainable solution likely belongs in gateway-owned session storage or APIs rather than an ad hoc memory file.
