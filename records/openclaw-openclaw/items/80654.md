## Summary

Current main already has the stable session identity model the request asks for: conversations are keyed by stable sessionKey entries in sessions.json, while sessionId remains the rotating transcript id for reset/lifecycle boundaries, and reset archives remain indexable for memory search.

## What This Changes

Request stable session identity and memory lookup across resets instead of relying on opaque random transcript filenames.

## Best Possible Solution

Keep the current split: stable sessionKey rows in sessions.json for identity/routing, rotating sessionId transcript files for reset boundaries, and indexed reset archives for memory history.

## Reproduction Assessment

Not applicable as a feature request; the current-main source and docs show the storage contract rather than a failing behavior to reproduce. The old-version symptom is addressed by stable sessionKey rows, lifecycle metadata, and indexed reset archives.

## Solution Assessment

Yes. The current implementation is the narrower maintainable solution because it keeps stable external identity in sessions.json without changing /new and /reset transcript semantics.
