## Summary

Keep open: the linked Telegram album bug is real and this is a useful candidate fix, but the branch should not merge as-is because it can add skipped album messages to prompt context without a hydrated media mapping, letting raw `telegram:file/...` refs leak back into model-facing context.

## What This Changes

The PR keeps Telegram source message ids on downloaded album media, adds album sibling ids to prompt-context hydration, and tests that caption-first album siblings use `media://inbound/...` paths.

## Best Possible Solution

Keep this Telegram plugin fix, but only hydrate extra album context for successfully resolved media ids, cover the partial-failure case, and require redacted live Telegram album proof before merge.

## Reproduction Assessment

Yes at source level: current main records album messages with raw `telegram:file` refs and maps only the first/current media item into prompt-context hydration. I did not run a live Telegram album reproduction in this read-only review.

## Solution Assessment

No as submitted: the fix is in the right Telegram plugin layer, but extra context ids need to be limited to successfully hydrated album media so skipped photos cannot fall back to raw refs. The narrower repair is to derive extra ids from resolved media or require a media-map hit before adding an extra cached node.
