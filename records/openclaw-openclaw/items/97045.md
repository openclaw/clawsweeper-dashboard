## Summary

Keep open: this is a focused candidate fix for the linked Telegram album raw-reference bug, and the new live proof materially improves confidence, but the patch still adds every buffered album message id to prompt context even when a sibling media download was recoverably skipped.

## What This Changes

The PR keeps Telegram source message ids on downloaded inbound media, adds album sibling ids to Telegram prompt-context hydration, and adds a regression test for caption-first album siblings using `media://inbound/...` paths.

## Best Possible Solution

Keep the Telegram plugin fix, but derive extra album context ids only from successfully resolved media or require a media-map hit before adding an extra cached album node, with partial-failure regression coverage before merge.

## Reproduction Assessment

Yes at source level: current main records album messages with raw `telegram:file` refs and maps only the first/current media item into prompt-context hydration. I did not execute tests in this read-only review, and the live proof in the PR covers the all-success album path.

## Solution Assessment

No as submitted: the fix is in the right Telegram plugin layer, but extra context ids need to be limited to successfully hydrated album media so skipped photos cannot fall back to raw refs. The narrower repair is to derive extra ids from resolved media or require a media-map hit before adding an extra cached node.
