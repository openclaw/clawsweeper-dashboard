## Summary

Keep open. Current main still supports remote embedding endpoints and local/QMD reranking, but there is no typed config, resolved runtime state, HTTP call path, or docs for a Cohere-style remote reranker; the latest user comments add concrete demand and no canonical fix PR supersedes it.

## What This Changes

Request support for an operator-configured remote Cohere-style reranker endpoint for memory search, mirroring the existing remote embedding configuration.

## Best Possible Solution

Design an additive, default-off remote reranker contract with typed schema/help/docs, SecretInput-aware credentials, guarded HTTP, explicit timeout/fallback semantics, and clear builtin/QMD/plugin ownership boundaries.

## Reproduction Assessment

Yes, as a source-level capability gap. Current main has remote embedding config and local/QMD reranking paths, but no schema, resolved runtime state, provider API, or `/v1/rerank` call path for the proposed endpoint.

## Solution Assessment

Unclear. A Cohere-compatible endpoint is plausible, but the maintainable solution needs product and security decisions about memory egress, SecretInput handling, SSRF policy, fallback behavior, and builtin/QMD/plugin ownership.
