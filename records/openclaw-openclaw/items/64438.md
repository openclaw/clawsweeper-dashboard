## Summary

Keep open. Current main still supports remote memory embeddings and local/MMR/QMD ranking paths, but it does not expose a Cohere-style remote reranker config, runtime HTTP client, builtin integration, QMD integration, or plugin-facing reranker contract.

## What This Changes

Request support for Cohere-compatible remote reranker endpoints in memory search, including builtin/QMD integration and fallback behavior.

## Best Possible Solution

Design an additive, SecretRef-aware remote reranker contract with guarded Cohere-compatible HTTP, schema/help/docs, builtin post-retrieval fallback, and a clear QMD/plugin ownership boundary.

## Reproduction Assessment

Yes, as a source-level capability gap: the strict schema has no `memorySearch.query.hybrid.rerank.remote`, runtime config has no reranker fields, and current source has no `/v1/rerank` call path. I did not run a live reranker because current main has no configuration or runtime integration for it.

## Solution Assessment

Unclear. A Cohere-compatible endpoint is plausible, but the maintainable fix needs decisions about core config versus a plugin-facing reranker seam, private-memory outbound policy, and QMD ownership.
