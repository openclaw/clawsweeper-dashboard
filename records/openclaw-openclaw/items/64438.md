## Summary

Keep open: current main still lacks remote reranker endpoint support, and this issue is paired with an open closing implementation plus a competing generic rerank-provider API direction that needs maintainer product and security review.

## What This Changes

Request support for Cohere-compatible remote reranker endpoints in memory search so users can run reranking on external GPU or hosted services instead of local GGUF models.

## Best Possible Solution

Keep this issue open until maintainers approve a plugin/API direction and either land the selected implementation or explicitly reject remote reranking as an OpenClaw-supported memory capability.

Do we have a high-confidence way to reproduce the issue? Not applicable as a bug reproduction, but yes for the source-level feature gap: current main has remote embedding config, local MMR, and QMD rerank toggles, with no remote reranker endpoint or rerank-provider hook.

## Solution Assessment

Is this the best way to solve the issue? Unclear until maintainers choose the API direction; a plugin/API seam is more aligned than one-off core remote reranker config, but the concrete endpoint plugin and generic SDK seam are still competing open approaches.
