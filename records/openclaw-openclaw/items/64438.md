## Summary

Keep open. Current main still supports remote embedding endpoints and local/QMD reranking, but it has no schema, resolved runtime state, provider contract, or HTTP call path for a Cohere-style remote reranker endpoint.

## What This Changes

Request to add default-off remote reranker endpoint support for memory search, including Cohere-style API config, fallback, batching, timeout, and builtin/QMD behavior.

## Best Possible Solution

Design an additive, default-off remote reranker contract with typed schema/help/docs, SecretInput-aware credentials, guarded HTTP, explicit timeout/fallback semantics, and clear builtin/QMD/plugin ownership boundaries.

## Reproduction Assessment

Yes, as a source-level capability gap. Current main has remote embedding config and local/QMD reranking paths, but no schema, resolved state, provider API, or `/v1/rerank` call path for the proposed endpoint.

## Solution Assessment

Unclear. A Cohere-compatible endpoint is plausible, but the maintainable solution needs product and security decisions about memory egress, SecretInput handling, SSRF policy, fallback behavior, and QMD ownership.
