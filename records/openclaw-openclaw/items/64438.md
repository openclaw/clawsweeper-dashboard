## Summary

Keep open. Current main still has remote embedding support and local/MMR/QMD reranking paths, but no config schema, resolved runtime state, HTTP contract, QMD bridge, or plugin reranker API for a Cohere-style remote reranker endpoint.

## What This Changes

The issue requests support for configuring remote Cohere-style reranker HTTP endpoints for OpenClaw memory search.

## Best Possible Solution

Design an additive, default-off remote reranker contract with typed config/schema/help/docs, SecretInput-aware credentials, guarded HTTP, tested fallback semantics, and explicit builtin/QMD/plugin ownership boundaries.

## Reproduction Assessment

Yes, as a source-level capability gap. Current main has remote embedding support and local/MMR/QMD ranking paths, but no /v1/rerank call path, reranker provider contract, or config state that could carry the proposed endpoint.

## Solution Assessment

Unclear. A Cohere-compatible endpoint is plausible, but the maintainable solution needs product and security decisions about private memory egress, SecretInput handling, SSRF policy, fallback behavior, and whether QMD support belongs in core or a plugin/API seam.
