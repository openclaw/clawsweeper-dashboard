## Summary

Keep open. Current main still has remote memory embedding support and local/QMD ranking paths, but no Cohere-style remote reranker config, schema, HTTP client, builtin integration, QMD integration, or plugin-facing reranker provider contract; the prior ClawSweeper comment reached the same keep-open conclusion and no linked PR or duplicate now supersedes it.

## What This Changes

The issue requests remote Cohere-compatible reranker endpoint support for memory search, including config, builtin/QMD integration, fallback, batching, and timeout behavior.

## Best Possible Solution

Add an additive remote reranker design for memory search: typed config/schema/help/docs, SecretRef-aware credentials, guarded HTTP for a Cohere-compatible contract, builtin post-retrieval reranking with vector/hybrid fallback on failure, and a clear QMD strategy. If maintainers prefer plugin ownership, first expose a documented memory reranker provider API so Qwen/Cohere/Jina adapters can live outside core without replacing the whole memory backend.

## Reproduction Assessment

Yes. The missing feature has a high-confidence source-level reproduction path: the proposed `memorySearch.query.hybrid.rerank.remote` shape is absent from the strict schema, and current search code has no `/v1/rerank` call path.

## Solution Assessment

Unclear. The requested direction is plausible, but the best fix depends on maintainer decisions about remote-snippet security policy, QMD ownership, and whether rerankers belong in core memory config or a new plugin-facing provider contract.

## Risks / Open Questions

- The outbound reranker policy is not settled: memory snippets would be sent to arbitrary configured URLs, including possible LAN endpoints, so SSRF/private-network allowance, proxy behavior, SecretRef credentials, header redaction, and timeouts need maintainer design.
- The QMD integration boundary is unclear because QMD currently owns reranking and model selection through its own query pipeline and `QMD_RERANK_MODEL` environment variable.
- The current plugin API supports memory embeddings but not reranker providers, so maintainers need to decide whether to add core config directly or first expose a generic reranker capability.
