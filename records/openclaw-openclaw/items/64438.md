## Summary

Keep open. Current main still has remote embedding support and local/QMD reranking paths, but no config, runtime state, provider contract, or HTTP call path for a Cohere-style remote reranker endpoint.

## What This Changes

Add default-off support for Cohere-style remote reranker endpoints in memory search for builtin and QMD backends.

## Best Possible Solution

Design an additive, default-off remote reranker contract with typed config/schema/help/docs, SecretInput-aware credentials, guarded HTTP, tested fallback semantics, and explicit builtin/QMD/plugin ownership boundaries.

## Reproduction Assessment

Yes, as a source-level capability gap. Current main has remote embedding config and local/QMD ranking paths, but no schema/runtime state or /v1/rerank call path that could use the proposed endpoint.

## Solution Assessment

Unclear. A Cohere-compatible endpoint is plausible, but the maintainable solution needs product and security decisions about memory egress, SecretInput handling, SSRF policy, fallback behavior, and QMD ownership.
