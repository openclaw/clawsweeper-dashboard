## Summary

Keep open: current main and v2026.6.6 still lack remote reranker endpoint support, and an open linked implementation PR closes this issue, so cleanup should not close it or queue a competing fix lane.

## What This Changes

Request support for Cohere-compatible remote reranker endpoints in memory search so users can rerank candidates without local GGUF reranker models.

## Best Possible Solution

Keep this issue as the tracking item while maintainers choose one default-off reranker/plugin SDK contract, resolve egress and compatibility policy, and land either https://github.com/openclaw/openclaw/pull/92725, https://github.com/openclaw/openclaw/pull/89584, or a unified replacement.

## Reproduction Assessment

Yes, as a source-level capability gap: current config/schema/runtime paths only expose remote embeddings, builtin hybrid search only goes through merge/MMR/temporal decay, and QMD rerank remains QMD-side/local.

## Solution Assessment

Unclear until maintainers choose the API direction. A default-off plugin/API seam with explicit egress, SecretRef, fallback, and compatibility policy is safer than adding vendor-specific core config directly.
