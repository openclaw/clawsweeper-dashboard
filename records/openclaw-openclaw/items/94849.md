## Summary

Keep open: this is a focused fix attempt for a real cron delivery bug, but the patch is incomplete. It clears the fatal recovered-warning classification for preference-false channels while still leaving final-text payload selection channel-gated, so a Feishu-style run can reach delivery with the warning payload instead of the recovered final assistant report.

## What This Changes

The PR removes the `preferFinalAssistantVisibleText` guard from recovered cron tool-warning classification and adds one helper-level regression test.

## Best Possible Solution

Fix the helper so recovered tool-warning runs with `finalAssistantVisibleText` and no structured successful payload deliver the final assistant text, while preserving structured/media payloads and genuinely fatal error handling.

## Reproduction Assessment

Yes at source level. The linked issue gives a reduced scheduled Feishu repro, and current-main source shows Feishu does not opt into final-text preference while fatal structured payloads return before dispatch.

## Solution Assessment

No. Removing the fatal guard is only a partial fix because final-text payload selection remains channel-gated, so the recovered final report can still be replaced by the warning payload.
