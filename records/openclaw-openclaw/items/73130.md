## Summary

Keep open. Current main still lacks the credential blast-radius guard, and the PR has a concrete security-sensitive matcher bypass plus policy/documentation mismatch that should be resolved before merge.

## What This Changes

The PR adds a credential blast-radius classifier, tests, a PR-local audit note, and a `runBeforeToolCallHook` preflight that blocks or approval-routes destructive tool calls with names-only credential metadata.

## Best Possible Solution

Land a maintainer-reviewed version that tokenizes or parses `rm` arguments enough to require a real recursive flag before protected root/home targets, adds split-flag regression coverage, handles malformed approval decisions accurately, updates exec-policy docs and changelog, and removes PR-local audit artifacts from the source tree.

## Reproduction Assessment

Yes. The core defect is reproducible by evaluating the PR regex against `rm -r -f /`, `rm -v -r /`, `rm -r -v /*`, and `rm -f -R /`; these recursive root/root-glob deletion forms do not match the proposed `shell/rm-rf-root` pattern.

## Solution Assessment

No. The hardening direction is reasonable, but the current patch is not the best maintainable solution because a single regex misses common shell forms and the new global preflight is not reconciled with current exec-policy docs.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Catch split rm flags before root targets:** `src/agents/credential-blast-radius-classifier.ts:330`
  - body: The root/home deletion guard misses common catastrophic commands when a non-recursive flag is split from the recursive flag before the target. For example, `rm -r -f /`, `rm -v -r /`, `rm -r -v /*`, and `rm -f -R /` do not match this pattern, so the guard allows the class of root wipes it is meant to block. Parse the `rm` arguments or allow arbitrary flag groups while separately requiring at least one real recursive flag before the protected target.
  - confidence: 0.91
- **[P3] Treat undefined approval decisions as unavailable:** `src/agents/pi-tools.before-tool-call.ts:450-453`
  - body: If the approval request response includes a `decision` property with value `undefined`, this path skips the no-route branch and later reports a timeout. That is still fail-closed, but it gives the wrong operator/debug signal for an unavailable or malformed approval route; handle `null` and `undefined` together here.
  - confidence: 0.8
- **[P3] Document the new exec preflight:** `src/agents/pi-tools.before-tool-call.ts:613-619`
  - body: This hook now runs a heuristic destructive-action guard before the existing trusted/plugin hook pipeline, but the current exec approvals docs say YOLO mode has no separate heuristic command-obfuscation or script-preflight gate. Update the exec-policy docs and changelog, or make the guard policy-configured, so operator expectations match runtime behavior.
  - confidence: 0.78
- **[P3] Remove the PR-local audit artifact:** `shared/projects/system/audit/2026-04-27-1845-credential-blast-radius-quoted-home-root-revision.md:4`
  - body: The added audit note is a PR execution artifact with a machine-specific worktree path and stale draft-status text. Keep that proof in the PR discussion or an external artifact store instead of adding a new root `shared/projects/system/audit` tree to the product repository.
  - confidence: 0.74
