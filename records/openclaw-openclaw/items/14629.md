## Summary

Keep open: current main still strips `<final>` tags textually and only collapses exact duplicate blocks separated by blank lines, so the reported glued same-line duplicate remains possible on the user-facing reply path.

## What This Changes

The issue requests stronger output sanitization for same-line, single-newline, partial, and tag-boundary duplicate assistant text after final-tag stripping.

## Best Possible Solution

Land a conservative sanitizer repair for the glued final-tag and adjacent same-line exact duplicate cases, with regression tests and changelog, while leaving fuzzy partial-duplicate policy out of the narrow fix.
