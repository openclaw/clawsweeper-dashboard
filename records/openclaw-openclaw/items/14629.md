## Summary

Keep open: current main and the latest shipped tag still have the reported sanitizer gap, and the same-line duplicate fix attempts are closed unmerged while the open related PRs do not close this issue.

## What This Changes

Request is to extend user-facing output sanitization to catch same-line, single-newline, and tag-boundary duplicate text left after final-tag stripping.

## Best Possible Solution

Add a conservative sanitizer pass for the concrete glued `</final>` repeat and explicitly supported adjacent single-newline duplicate forms, with regression tests that preserve ordinary repetition, formatting, whitespace, and code indentation.

## Reproduction Assessment

Yes, from source inspection. The reported string reaches `stripFinalTagsFromText()`, becomes a glued same-line repeat, and then bypasses `collapseConsecutiveDuplicateBlocks()` because there is no blank-line boundary.

## Solution Assessment

Yes, if the repair is narrow. The maintainable path is to fix `sanitizeUserFacingText()` with conservative adjacent/tag-boundary duplicate handling and avoid fuzzy partial-duplicate removal unless maintainers choose that product behavior.
