## Summary

Keep open: current main and the latest release still strip final tags textually and only collapse exact blank-line duplicate blocks, so the glued same-line duplicate remains source-reproducible; the broader partial-duplicate behavior needs maintainer bounds before another repair lane runs.

## What This Changes

The issue asks OpenClaw to improve user-facing output sanitization for same-line duplicates, single-newline duplicates, partial duplicates, and final-tag-boundary repeats.

## Best Possible Solution

Define conservative delivery-sanitizer behavior for glued final-tag repeats and exact adjacent same-line or single-newline duplicates, while leaving fuzzy partial-duplicate cleanup to an approved visible-output normalizer design.

## Reproduction Assessment

Yes, at source level. The reported string loses the `</final>` marker through textual final-tag stripping and then remains a single block, so current duplicate collapse returns it unchanged.

## Solution Assessment

Partly. A narrow exact duplicate or final-tag-boundary pass is maintainable, but broad substring and near-duplicate cleanup needs maintainer product bounds to avoid dropping legitimate repeated text.
