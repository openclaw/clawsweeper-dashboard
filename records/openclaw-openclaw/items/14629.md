## Summary

Keep open: current main and the latest shipped tag still miss the reported same-line/tag-boundary duplicate sanitizer gap; prior targeted PRs were closed unmerged, and the open adjacent sanitizer PRs do not close this issue.

## What This Changes

Improve `sanitizeUserFacingText()` so final-tag stripping and duplicate collapse handle glued same-line/tag-boundary duplicates without harming normal formatting.

## Best Possible Solution

Add a conservative helper-level sanitizer pass for the concrete glued `</final>` repeat and any explicitly supported adjacent single-newline duplicate form, with regression coverage that preserves normal repetition and formatting.

## Reproduction Assessment

Yes, from source inspection. The reported input reaches `stripFinalTagsFromText()`, becomes a glued same-line repeat, then bypasses `collapseConsecutiveDuplicateBlocks()` because there is no blank-line separator.

## Solution Assessment

Yes, if the fix stays conservative. A narrow repair in `sanitizeUserFacingText()` is the maintainable path; broad fuzzy partial-duplicate removal should be avoided unless maintainers explicitly choose that behavior.
