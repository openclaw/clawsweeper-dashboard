## Summary

Keep open: current main and v2026.6.6 still strip final tags as plain removal and collapse only exact blank-line duplicate blocks, while the direct same-line duplicate PRs are closed unmerged and the broader visible-output PR is only overlapping work.

## What This Changes

Improve the user-facing text sanitizer so final-tag stripping and duplicate collapse catch same-line, single-newline, and conservative partial duplicate assistant output without damaging normal formatting.

## Best Possible Solution

Define a conservative core sanitizer boundary, then patch `sanitizeUserFacingText()` and focused tests to remove glued final-tag repeats and explicitly supported adjacent duplicates while preserving normal repeated text and formatting.

## Reproduction Assessment

Yes at source level, not with a live WhatsApp run: stripping `</final>` concatenates adjacent text, and duplicate collapse returns unchanged when the result has no blank-line-separated duplicate blocks.

## Solution Assessment

Unclear as requested: the narrow tag-boundary or adjacent same-line repair is the right direction, but broad partial/near-duplicate detection needs a maintainer boundary to avoid deleting intentional repeated output.
