## Summary

Keep open: current main and the latest release still strip final tags as plain deletion and only collapse exact blank-line duplicate blocks, so the reported glued same-line duplicate remains source-reproducible. The open related normalizer PR is draft and does not close this issue, so it is a possible implementation path rather than a reason to close.

## What This Changes

Improve user-facing output sanitization so glued final-tag repeats, same-line or single-newline repeats, and bounded partial duplicates do not reach channel users.

## Best Possible Solution

Define conservative delivery-sanitizer behavior for glued final-tag repeats and exact adjacent same-line or single-newline duplicates, while leaving fuzzy partial-duplicate cleanup to an approved visible-output normalizer design.

## Reproduction Assessment

Yes, at source level. The reported string loses the </final> marker through textual final-tag stripping and then remains a single block, so current duplicate collapse returns it unchanged.

## Solution Assessment

Unclear. A narrow exact duplicate or final-tag-boundary pass is maintainable, but broad substring and near-duplicate cleanup needs maintainer product bounds to avoid dropping legitimate repeated text.
