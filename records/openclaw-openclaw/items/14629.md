## Summary

Keep open: current main still strips `<final>` tags textually and only collapses exact duplicate blocks separated by blank lines, so the reported glued same-line duplicate can still pass through the user-facing reply sanitizer.

## What This Changes

Requests stronger user-facing output sanitization for same-line duplicate text, partial duplicates, and `</final>` tag-boundary repeats.

## Best Possible Solution

Add a conservative sanitizer pass for the concrete glued `</final>` repeat and any deliberately supported adjacent same-line or single-newline exact duplicate form, while preserving normal formatting and intentional repetition.

## Reproduction Assessment

Yes from source inspection: stripping `</final>` leaves adjacent text and the duplicate collapse returns unchanged unless it sees a blank-line separator. I did not run a live test in this read-only review, but the current source path is deterministic.

## Solution Assessment

Yes, a narrow sanitizer repair is the best path; broad near-duplicate or whole-response coherence heuristics should be avoided unless a maintainer explicitly wants that product behavior.
