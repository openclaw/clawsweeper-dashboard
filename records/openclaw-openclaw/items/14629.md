## Summary

Keep open: current main and v2026.6.6 still only collapse exact duplicate paragraph blocks separated by blank lines, and the open broader sanitizer PR is a dirty draft that needs maintainer review rather than closing this issue.

## What This Changes

The issue requests stronger user-facing output sanitization for same-line, tag-boundary, and partial duplicate assistant text after final/thinking tag cleanup.

## Best Possible Solution

Add or land a conservative shared sanitizer repair for concrete glued final-tag repeats and explicitly supported adjacent duplicate forms, with tests preserving intentional repetition, normal single-newline formatting, trailing whitespace, and indented code blocks.

## Reproduction Assessment

Yes, at source level. Current main strips final tags before duplicate collapse, then returns same-line or single-newline repeats unchanged because the duplicate pass only splits on blank-line paragraph boundaries.

## Solution Assessment

Yes for keeping this open, but the repair should be narrower than a broad fuzzy duplicate detector. The maintainable path is a conservative sanitizer rule for concrete adjacent/tag-boundary repeats plus regression tests for false positives.
