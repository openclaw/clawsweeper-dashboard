## Summary

Keep open: current main still strips `<final>` tags as plain text and only collapses exact duplicate blocks separated by blank lines on the outbound sanitizer path, while the related same-line duplicate PRs are closed unmerged and the broader visible-output PR is still an unmerged draft.

## What This Changes

The issue asks the user-facing output sanitizer to catch same-line, tag-boundary, and partial duplicate replies that survive current final-tag stripping and paragraph duplicate collapse.

## Best Possible Solution

Add narrow exact/tag-boundary duplicate cleanup in the existing sanitizer with regression coverage, and leave broad near-duplicate coherence heuristics for a separate design decision if maintainers want them.

## Reproduction Assessment

Yes, from source inspection. After `</final>` is stripped, the reported example becomes one glued block, and `collapseConsecutiveDuplicateBlocks()` returns it unchanged because it only splits on blank-line boundaries.

## Solution Assessment

Yes for a narrow sanitizer repair. The maintainable path is exact/tag-boundary duplicate handling in `sanitizeUserFacingText()` with preservation tests; broad partial-match cleanup is not the best default without separate product review.

Fix openclaw/openclaw#14629 on current main. The observable bug is that `sanitizeUserFacingText()` strips `<final>` tags textually, then `collapseConsecutiveDuplicateBlocks()` only handles exact blank-line duplicate paragraphs, so glued output like `Atlas here...?</final>Atlas here...?Fair point.` can still be delivered with the duplicated prefix. Implement a conservative repair in `src/agents/pi-embedded-helpers/sanitize-user-facing-text.ts`: handle the concrete glued final-tag/exact adjacent duplicate cases and any deliberately supported same-line or single-newline exact duplicate form, but do not add broad fuzzy partial-duplicate or whole-response coherence heuristics. Add regression tests in `src/agents/pi-embedded-helpers.sanitizeuserfacingtext.test.ts` for the reported glued final-tag repeat and supported adjacent duplicate forms, plus preservation tests for ordinary repeated wording, normal single-newline formatting, trailing whitespace, and code-block indentation. Check related refs first: #16813 and #19113 were closed unmerged, #41851 is code-block indentation, and draft #68986 may overlap but is not landed; stop or re-scope if a newer landed PR has already fixed this. Because this is user-facing output behavior, add a single-line `CHANGELOG.md` fix entry unless maintainers decide the change is test/internal only.
