## Summary

Keep open: current main and the latest release still strip final tags as plain markers and only collapse exact duplicate blocks separated by blank lines, so the same-line/tag-boundary duplicate report remains source-reproducible. The direct prior fix attempts are closed unmerged, and https://github.com/openclaw/openclaw/pull/68986 is an overlapping draft rather than a landed replacement.

## What This Changes

The issue requests stricter user-facing output sanitization for glued final-tag repeats, same-line duplicates, and conservative partial duplicates.

## Best Possible Solution

Define a conservative sanitizer boundary and land either a narrowed version of https://github.com/openclaw/openclaw/pull/68986 or a focused sanitizer PR that removes glued final-tag and adjacent exact duplicate leaks while preserving intentional repetition, single-newline formatting, trailing whitespace, and code indentation.

## Reproduction Assessment

Yes, at source level: stripping `</final>` leaves adjacent text glued together, and the current duplicate collapse returns unchanged text unless duplicates are separated by blank lines. I did not run tests because this review is read-only and the source path is sufficient to prove the sanitizer gap.

## Solution Assessment

Unclear as a full product direction: the concrete glued final-tag duplicate is best solved by a narrow sanitizer pass, but broad partial or near-duplicate removal needs maintainer-defined boundaries to avoid deleting intentional text.
