## Summary

Keep open: current main still has the reported same-line and final-tag-boundary duplicate gap at source level. The earlier focused same-line fix PRs were closed unmerged, and the broader visible-output normalizer is still an open draft needing changes rather than a merged superseding fix.

## What This Changes

The issue requests extending the user-facing text sanitizer to catch same-line, tag-boundary, and partial duplicate output after final-tag stripping.

## Best Possible Solution

Add conservative regression coverage and sanitizer handling for glued `</final>` repeats plus exact adjacent same-line or single-newline duplicates, while leaving broad fuzzy partial-dedupe behavior to a maintainer-approved visible-output design.

## Reproduction Assessment

Yes, at source level. The reported string loses `</final>` through final-tag stripping, then remains one block without `\n\n`, so duplicate collapse returns it unchanged; no live WhatsApp replay was run.

## Solution Assessment

Partly. A narrow exact duplicate or final-tag-boundary pass is maintainable, but broad substring and near-duplicate cleanup needs product direction to avoid dropping legitimate repeated text.
