## Summary

Keep open: current main still leaves the reported same-line/tag-boundary duplicate path source-reproducible, and the related same-line fix PRs are closed unmerged while the broader visible-output normalizer remains open and draft.

## What This Changes

Improve the user-facing output sanitizer so same-line, single-newline, tag-boundary, and partial duplicate assistant text is handled without removing legitimate content.

## Best Possible Solution

Add conservative regression coverage and sanitizer handling for glued `</final>` repeats plus exact adjacent same-line or single-newline duplicates, while leaving broad fuzzy partial-dedupe behavior to a maintainer-approved visible-output design.

## Reproduction Assessment

Yes, at source level. The reported string loses `</final>` through final-tag stripping, then remains one block without `\n\n`, so duplicate collapse returns it unchanged.

## Solution Assessment

Partly. A narrow exact duplicate or final-tag-boundary pass is maintainable, but the requested broad substring and near-duplicate cleanup needs product direction to avoid dropping legitimate repeated text.
