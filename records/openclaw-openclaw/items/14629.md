## Summary

Keep open: current main still strips final tags as plain text and only collapses exact duplicate blocks separated by blank lines, so the reported glued same-line duplicate remains source-reproducible. The older same-line fix PRs are closed unmerged, and the open visible-output normalizer is a broader draft branch rather than a landed terminal fix.

## What This Changes

The issue requests stronger user-facing output sanitization for glued same-line duplicates, partial duplicates, and final-tag boundary repeats.

## Best Possible Solution

Add a conservative regression and sanitizer pass for glued `</final>` repeats and deliberately supported adjacent same-line or single-newline exact duplicate shapes, while leaving broad fuzzy partial-dedupe behavior to a maintainer-approved visible-output design.

## Reproduction Assessment

Yes, at source level. The reported string reaches final-tag stripping, becomes glued same-line duplicate text, and bypasses duplicate collapse because the helper only splits on blank-line separators.

## Solution Assessment

Partly. A narrow tag-boundary or adjacent exact-duplicate sanitizer repair is maintainable, but broad partial or near-duplicate detection is not the best path without maintainer product approval.
