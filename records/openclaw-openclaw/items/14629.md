## Summary

Keep open: current main and v2026.6.10 still strip final tags as plain tag removal and only collapse exact duplicate blocks separated by blank lines, so the reported same-line/tag-boundary duplicate cleanup is not implemented. The direct fix PRs are closed unmerged, and the broader visible-output normalizer PR is open draft/overlapping rather than a landed canonical fix.

## What This Changes

The issue asks for the user-facing output sanitizer to catch glued same-line duplicates, partial duplicates, and final-tag boundary repeats that survive existing paragraph-level duplicate collapse.

## Best Possible Solution

Land a conservative sanitizer repair for the concrete glued final-tag repeat and any explicitly approved adjacent same-line or single-newline duplicate form, with regressions that preserve intentional repetition and formatting.

## Reproduction Assessment

Yes, at source level: current main removes final tags before a paragraph-only duplicate collapse that returns unchanged for same-line or single-newline duplicates. I did not run a live WhatsApp reproduction in this read-only review.

## Solution Assessment

No, current main is not the best end state. The narrow maintainable path is a tested sanitizer rule for concrete adjacent/tag-boundary duplicates; broader near-duplicate cleanup needs maintainer product bounds first.
