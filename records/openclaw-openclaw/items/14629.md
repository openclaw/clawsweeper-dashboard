## Summary

Keep open: current main still has the reported sanitizer gap for same-line and final-tag-boundary duplicates, and the broader visible-output normalizer remains an open draft needing changes rather than a merged superseding fix.

## What This Changes

The issue requests broader output-sanitizer duplicate detection for same-line repeats, partial/near-duplicates, and text duplicated across stripped final-tag boundaries.

## Best Possible Solution

Add conservative regression coverage and handling for glued final-tag repeats plus exact adjacent same-line or single-newline duplicates, while leaving fuzzy partial-dedupe behavior to a maintainer-approved visible-output normalization design.

## Reproduction Assessment

Yes, at source level. The reported string loses </final> through final-tag stripping, then remains one blank-line-free block, so the current duplicate collapse returns it unchanged; no live WhatsApp replay was run.

## Solution Assessment

Partly. A narrow exact duplicate or final-tag-boundary pass is maintainable, but broad substring and near-duplicate cleanup needs product direction to avoid dropping legitimate repeated text.
