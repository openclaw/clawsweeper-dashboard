## Summary

Keep open: current main still has the reported sanitizer gap for glued/tag-boundary same-line duplicates, and the latest release has the same blank-line-only duplicate collapse. The related same-line PRs are closed unmerged; the open related PRs are adjacent or broader draft work rather than a landed fix for this issue.

## What This Changes

The issue requests stronger user-facing text sanitization for same-line, tag-boundary, and partial duplicate assistant output.

## Best Possible Solution

Add a narrow sanitizer regression and implementation for concrete glued `</final>` repeats and any deliberately supported adjacent single-newline duplicate shape, while avoiding broad fuzzy partial-dedupe behavior without maintainer approval.

## Reproduction Assessment

Yes. Source inspection shows the reported string reaches final-tag stripping, becomes a glued same-line repeat, and then bypasses duplicate collapse because the helper only splits on blank-line separators.

## Solution Assessment

Yes for a narrow repair, but not for broad fuzzy duplicate detection. The maintainable fix is conservative tag-boundary or adjacent duplicate handling in the active sanitizer with tests that preserve normal repeats, single-newline formatting, trailing whitespace, and code indentation.
