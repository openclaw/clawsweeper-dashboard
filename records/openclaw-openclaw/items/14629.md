## Summary

Keep open: current main still strips `<final>` tags as plain text and only collapses exact duplicate blocks separated by blank lines, so the reported glued same-line repeat remains. The related same-line duplicate PRs are closed unmerged, and the open adjacent PRs are either narrower indentation work or broader draft sanitizer work that has not landed on main.

## What This Changes

The issue requests extending `sanitizeUserFacingText()` duplicate cleanup to catch same-line, single-newline, partial, and tag-boundary repeats after model scaffolding tags are stripped.

## Best Possible Solution

Add a conservative sanitizer pass for the concrete glued `</final>` repeat and explicitly supported adjacent same-line or single-newline exact duplicate forms, with regression tests preserving normal repeated wording, single-newline formatting, trailing whitespace, and code-block indentation.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: after `</final>` replacement the reported example becomes one glued block, and the current duplicate collapse returns single-block text unchanged because it only splits on blank lines.

## Solution Assessment

Yes. A narrow repair in the existing user-facing sanitizer path is the maintainable direction; broad partial/near-duplicate or whole-response coherence heuristics should be avoided unless separately designed and tightly tested.
