## Summary

Keep open: current main and the latest release still only collapse exact adjacent duplicate paragraph blocks, while the same-line/tag-boundary duplicate behavior requested here remains unimplemented and the related direct fix PRs are closed unmerged.

## What This Changes

The issue asks OpenClaw to make `sanitizeUserFacingText()` detect same-line, single-newline, tag-boundary, and limited partial duplicate assistant output that current paragraph-only duplicate collapse misses.

## Best Possible Solution

Add a maintainer-approved conservative sanitizer rule for glued final-tag repeats and explicitly supported adjacent same-line or single-newline duplicates, with regression tests that preserve intentional repetition, requested repeats, and code formatting.

## Reproduction Assessment

Yes, at source level: current main strips final tags and then returns unchanged when the text does not contain blank-line-separated duplicate blocks. I did not run a live WhatsApp/model reproduction in this read-only review.

## Solution Assessment

No, not as a broad partial-duplicate filter. The safer solution is a narrow, owner-approved sanitizer rule for concrete glued/tag-boundary or adjacent duplicate patterns, with tests proving ordinary repetition is preserved.
