## Summary

Keep open: current main and v2026.6.5 still strip final tags textually and only collapse blank-line duplicate blocks, so the reported same-line/tag-boundary duplicate can still pass through outbound reply sanitization.

## What This Changes

The issue requests stronger outbound text sanitization for same-line, tag-boundary, and partial duplicate assistant output.

## Best Possible Solution

Define a conservative sanitizer contract for adjacent same-line and tag-boundary duplicates, or fold that contract into the broader visible-output PR, with regression tests that preserve intentional repeats and formatting.

## Reproduction Assessment

Yes, from source inspection: final tags are removed before dedupe, the dedupe pass only splits on blank lines, and outbound replies call this sanitizer before delivery. I did not run a live reproduction because this review is read-only and tests could create artifacts.

## Solution Assessment

Unclear for the full requested heuristic. A narrow adjacent/tag-boundary sanitizer pass is likely the best fix for the concrete leak, while partial/near-duplicate detection needs a maintainer product boundary to avoid deleting intentional repetition.
