## Summary

Keep open: current main and the latest shipped tag still have the sanitizer gap for glued same-line duplicates after `</final>` stripping, and the related same-line PRs are closed unmerged while the open draft sanitizer work does not close this issue.

## What This Changes

Improve `sanitizeUserFacingText()` so final-tag stripping and duplicate collapse handle glued same-line and deliberately supported adjacent duplicate output without damaging normal formatting.

## Best Possible Solution

Add a conservative helper-level sanitizer pass for the concrete glued `</final>` repeat and any explicitly supported adjacent single-newline duplicate form, with regression coverage that preserves normal repetition, single-newline formatting, trailing whitespace, and code indentation.

## Reproduction Assessment

Yes, from source inspection. The reported input reaches `stripFinalTagsFromText()`, becomes a glued same-line repeat, then bypasses `collapseConsecutiveDuplicateBlocks()` because there is no blank-line separator.

## Solution Assessment

Yes, if the fix stays conservative. A narrow sanitizer repair in `sanitizeUserFacingText()` is the maintainable path; broad fuzzy partial-duplicate removal should be avoided unless maintainers explicitly choose that product behavior.
