## Summary

Keep open. Current main still lacks the requested `openclaw cron lint` command and contextual multi-account `delivery.accountId` validation; the linked logging issue is complementary, not a replacement.

## What This Changes

Request adds a read-only cron lint command and validation requiring announce-mode cron jobs to set `delivery.accountId` when the resolved channel has multiple configured accounts.

## Best Possible Solution

Add a shared cron lint helper using channel plugin account enumeration, expose `openclaw cron lint`, warn at gateway startup, reject invalid new/edited announce jobs, and keep the linked telemetry issue separate.

## Solution Assessment

Yes, with staged enforcement. A cron-owned lint/validation helper is the narrow maintainable fix; fail-closed startup rollout should remain a later maintainer-controlled step after warning-only migration.
