## Summary

Keep open: current main still lacks the requested `openclaw cron lint` surface and still validates cron announce delivery by channel/target only, not by missing `delivery.accountId` on multi-account channels; the related open PR is partial, conflicting, and does not cover the lint/startup rollout requested here.

## What This Changes

Request to add a cron lint command plus startup and add/edit validation requiring explicit `delivery.accountId` for announce-mode crons targeting multi-account channel configs.

## Best Possible Solution

Implement a shared cron-owned lint/account validation helper using read-only channel account metadata, expose warn-only CLI/startup reporting first, and promote fail-closed behavior only after an explicit rollout decision.

## Reproduction Assessment

Yes. Source inspection on current main shows no cron lint command/rule symbols, and the gateway add/update path validates announce channel/target compatibility before persistence without requiring missing `delivery.accountId` on multi-account channel configs.

## Solution Assessment

Partly yes. A shared cron lint helper plus mutation guard is the right boundary, but fail-closed startup behavior, deprecation timing, waivers, and opt-out policy should be decided before a full implementation lands.
