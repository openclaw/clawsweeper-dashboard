## Summary

Keep open. Current main still lacks the requested `openclaw cron lint` command and still allows multi-account announce cron jobs to be created or edited without an explicit `delivery.accountId`; the related logging and explicit-invalid-account threads are complementary rather than replacements.

## What This Changes

Request adds `openclaw cron lint` plus gateway and cron add/edit checks requiring explicit `delivery.accountId` for announce delivery to multi-account channels.

## Best Possible Solution

Add a cron-owned lint helper using channel configured-account metadata, expose `openclaw cron lint`, warn at gateway startup, reject new or edited multi-account announce jobs missing accountId, and leave strict startup failure plus delivery telemetry to separate follow-ups.

## Reproduction Assessment

Yes. Source inspection shows the requested command and rule symbols are absent, and a multi-account announce job with an explicit channel but no `delivery.accountId` reaches current gateway validation because that validation checks channel ambiguity only.

## Solution Assessment

Yes for the staged version. A shared cron lint/validation helper backed by the channel account-list contract is the narrow maintainable path; strict startup failure and waiver policy should be kept as a later maintainer-controlled rollout.
