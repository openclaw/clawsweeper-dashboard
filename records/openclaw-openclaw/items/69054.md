## Summary

Keep open: current main and the latest release still lack the requested cron lint surface and accountId rule, and the remaining work needs maintainer rollout decisions around warning-only versus fail-closed enforcement.

## What This Changes

The issue requests a cron lint command and validation paths requiring explicit delivery.accountId for announce-mode cron jobs on multi-account channels.

## Best Possible Solution

Add a shared cron-owned lint helper with warn-only CLI/startup reporting first, then add focused add/edit validation while treating fail-closed rollout and waivers as maintainer policy decisions.

## Reproduction Assessment

Yes, for the missing behavior. Source inspection shows current main accepts announce delivery without delivery.accountId, has no cron lint command/rule, and only passes accountId through when it is supplied.

## Solution Assessment

Yes directionally, but the full proposal needs maintainer choices. A shared lint helper plus mutation guard is the right boundary, while fail-closed startup timing, waivers, and opt-out behavior should be decided before implementation.
