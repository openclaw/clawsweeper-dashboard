## Summary

Keep open: current main and v2026.6.6 still lack default-account startup promotion, and this clean, proof-backed PR remains the canonical fix path; the remaining gate is maintainer acceptance of the intentional startup-order compatibility and availability risk.

## What This Changes

The PR changes Discord startup slot calculation so an eligible configured `channels.discord.defaultAccount` starts before secondary accounts, with regressions for default-first, duplicate-token, and unconfigured-account cases.

## Best Possible Solution

Land the Discord-local fix after maintainer acceptance that an eligible configured default account should own the zero-delay startup slot while secondary accounts keep the existing stagger.

## Reproduction Assessment

## Solution Assessment

Yes, as an implementation shape, with maintainer acceptance required for upgrade behavior. The PR stays inside the Discord plugin startup path and reuses existing default-account and runtime-enabled predicates instead of hard-coding account names or adding config.
