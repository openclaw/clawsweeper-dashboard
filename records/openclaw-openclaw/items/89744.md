## What This Changes

The PR changes Discord multi-account startup ordering so the configured runtime-eligible default account gets the zero-delay startup slot before secondary accounts.

## Best Possible Solution

Land a Discord-local startup-order fix that uses the existing runtime-enabled account predicate before promoting the configured default account, with secondary account staggering preserved.

## Reproduction Assessment

Yes. Source inspection shows current main sorts account ids before computing startup delay, so a configured default account that sorts later receives a nonzero delay.

## Solution Assessment

Yes. The PR keeps the fix inside the Discord plugin, reuses the existing runtime-enabled account predicate, and avoids hard-coded account ids or a new config option.
