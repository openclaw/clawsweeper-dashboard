- `merge-risk: 🚨 compatibility`: The patch changes startup ordering for existing Discord multi-account/defaultAccount configs and can treat duplicate-token aliases differently from the gateway enablement contract.
- `merge-risk: 🚨 availability`: A skipped duplicate-token default can take the first startup slot and delay the real Discord account during restart.

## Summary

Keep open: the PR fixes the main Discord default-account ordering bug and has strong real runtime proof, but the merge result still builds startup slots from enabled/configured accounts rather than Discord's runtime-enabled predicate, so a duplicate-token configured default can consume slot 0 even though gateway startup will skip it.

## What This Changes

The PR reorders Discord multi-account startup so the configured default account is first and adds a regression test for staggered secondary accounts.

## Best Possible Solution

Build Discord startup slots from the same runtime-enabled accounts that gateway startup will actually start, then promote the configured default within that filtered list and cover the duplicate-token default case in tests.

## Reproduction Assessment

Yes. Current source shows startup delay follows sorted `listDiscordAccountIds`, and the PR body includes redacted live Discord gateway logs showing the configured default starts first after the patch; the remaining blocker is source-reproducible from the duplicate-token runtime-enabled path.

## Solution Assessment

No, not as-is. The default-promotion approach is the right layer, but it should use the same runtime-enabled candidate set that gateway startup uses so disabled duplicate-token accounts cannot consume delay slots.

- **[P1] Use runtime-enabled accounts before promoting the default:** `extensions/discord/src/channel.ts:224-231`
  - body: This helper still builds the startup order from accounts that are merely enabled/configured, then moves the configured default to slot 0. Filter candidates with the same runtime-enabled predicate before promotion and add that regression.
