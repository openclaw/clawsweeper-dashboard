## Summary

Keep open: this PR addresses a real Discord multi-account startup bug and includes credible live runtime proof, but the new slot ordering can still promote a default account that the gateway will later skip as runtime-disabled, so it needs a narrow fix before merge.

## What This Changes

The PR promotes the resolved Discord default account to the first Discord gateway startup slot and adds a regression test for default-account-first staggering.

## Best Possible Solution

Filter Discord startup delay candidates with the same runtime-enabled predicate gateway startup uses, then keep the default-account-first behavior and add duplicate-token regression coverage.

## Reproduction Assessment

Yes. Source inspection shows current main builds delay slots from sorted Discord account IDs, so `main` after `billy` or `farber` can receive a nonzero delay despite `channels.discord.defaultAccount`; the PR body also includes live redacted Discord startup logs for the fixed path.

## Solution Assessment

No as submitted. Reusing the existing default-account resolver is the right layer, but the delay-slot list must use the same runtime-enabled filtering as gateway startup so skipped duplicate-token accounts cannot consume slots.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Filter disabled defaults before assigning startup slots:** `extensions/discord/src/channel.ts:232-239`
  - body: `resolveDiscordStartupAccountIds` promotes `defaultAccountId` whenever it survives the local `candidate.enabled`/token filter, but gateway startup filters accounts through `plugin.config.isEnabled`, and Discord uses `isDiscordAccountEnabledForRuntime` to skip duplicate-token accounts before `startAccount`. If `channels.discord.defaultAccount` names a duplicate-token account that the gateway will not start, this new promotion still gives that disabled account slot 0 and delays the actual token owner. Use the same runtime-enabled predicate, or the existing enabled-account helper, before assigning delay slots.
  - confidence: 0.84
