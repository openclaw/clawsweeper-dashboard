## Summary

Keep open: the PR is the current canonical fix path for the Discord default-account startup-order bug, with no concrete patch defects found. It still needs maintainer acceptance because it intentionally changes startup order for existing multi-account Discord configs.

## What This Changes

The PR changes Discord startup delay calculation to promote an eligible configured `channels.discord.defaultAccount` to the zero-delay slot after filtering runtime-enabled and configured accounts, with focused regression tests.

## Best Possible Solution

Land the Discord-local fix after maintainer acceptance that an eligible configured default account should own the zero-delay startup slot while secondary accounts keep the existing stagger.

## Reproduction Assessment

Yes. Source inspection on current main shows `resolveDiscordStartupDelayMs` uses the sorted account-list index without default-account promotion, so a configured default account that sorts after secondary accounts can receive a nonzero delay.

## Solution Assessment

Yes, with maintainer acceptance. The PR stays in the Discord plugin, reuses the existing runtime-enabled and configured predicates, and adds regressions for the prior predicate gaps.
