## What This Changes

The PR updates Discord account startup ordering so the configured default account gets the first startup slot before staggered secondary accounts.

## Best Possible Solution

Land this plugin-local startup ordering fix after normal maintainer and CI gates, then let it close the linked Discord startup issue at https://github.com/openclaw/openclaw/issues/77429.

## Reproduction Assessment

Yes. Source inspection of current main shows Discord startup delay is computed from the sorted enabled account list, so a configured default account like `main` can get a nonzero delay when `billy` or `farber` sorts earlier.

## Solution Assessment

Yes. The PR reuses the existing Discord default-account resolver inside the Discord startup-delay path, which is narrower than changing shared account-list ordering for every channel.
