## Summary

Keep this PR open, but it is not merge-ready: it addresses the Discord default-account startup bug for ordinary multi-account configs, yet the startup slot list still diverges from the gateway runtime-enabled predicate and can delay the actual enabled bot in duplicate-token configs.

## What This Changes

The PR changes Discord gateway startup delay calculation so the configured default account is promoted ahead of staggered secondary accounts and adds focused channel tests.

## Best Possible Solution

Land a version that promotes the configured default only after filtering startup slots through Discord's runtime-enabled predicate, with duplicate-token regression coverage and the existing live proof retained or refreshed.

## Reproduction Assessment

Yes. Source inspection shows current main delays Discord accounts by list index, and the PR body supplies redacted live logs where a configured default account that would have been index 3 starts immediately after the patch.

## Solution Assessment

No. The PR is a plausible fix for the ordinary case, but the best fix is to build the slot list from the same runtime-enabled account predicate the gateway uses, then promote defaultAccount within that filtered list.
