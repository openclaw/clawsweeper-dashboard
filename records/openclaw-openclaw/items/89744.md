## Summary

Keep open: the PR fixes the central Discord default-account startup ordering bug and has sufficient live proof, but it still needs one narrow repair before merge because the new startup slot helper can count Discord accounts that gateway startup skips as runtime-disabled.

## What This Changes

The PR promotes the configured Discord default account to the first Discord startup slot and adds a regression test for secondary account staggering.

## Best Possible Solution

Promote the configured default account after filtering to the same runtime-enabled and configured Discord account set used by gateway startup, with a duplicate-token regression test.

## Reproduction Assessment

Yes. Source inspection shows current main can assign a nonzero startup index to a configured Discord default account that sorts after secondary accounts; the linked issue and PR body provide concrete multi-account logs.

## Solution Assessment

No, not quite. Default-account promotion is the right layer, but the helper should derive slots from the same runtime-enabled predicate used by gateway startup before assigning stagger delays.
