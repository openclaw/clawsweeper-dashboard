## Summary

Keep open: current main still has the reported sorted-account delay behavior, and this PR addresses the central bug, but the patch should be tightened before merge because the promoted default can be a duplicate-token account that gateway startup already treats as disabled.

## What This Changes

This PR reorders Discord multi-account startup delay calculation so the resolved default account starts in the zero-delay slot and adds a focused channel test for that ordering.

## Best Possible Solution

Merge after the Discord startup delay order is derived from the same runtime-enabled account set the gateway actually starts, with duplicate-token default coverage added to the focused test.

## Reproduction Assessment

Yes. Current main source shows delay slots are computed from the sorted Discord account id list, and the linked issue/PR proof shows a configured default account can be delayed behind secondary accounts.

## Solution Assessment

No. The default-account promotion is the right fix direction, but the helper should use the runtime-enabled account set so disabled duplicate-token defaults cannot consume the first startup slot.
