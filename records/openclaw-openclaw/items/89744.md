## What This Changes

The PR promotes the configured Discord default account to the zero-delay startup slot and adds a regression test for multi-account startup staggering.

## Best Possible Solution

Land this plugin-local startup-order fix after required CI/maintainer gates pass, preserving the shared sorted account-list contract and closing the linked Discord startup bug through the merge.

## Reproduction Assessment

Yes. Source inspection shows enabled Discord accounts are sorted before delay calculation, so a configured default account like `main` can receive a later delay slot when other enabled account IDs sort before it.

## Solution Assessment

Yes. Reordering only Discord startup-delay candidates with the existing default-account resolver is the narrowest maintainable fix; changing shared account-list ordering would affect other callers unnecessarily.
