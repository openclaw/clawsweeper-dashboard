## Summary

Keep open for maintainer review: the patch appears to be the right Discord-local implementation, but it intentionally changes upgrade behavior for existing multi-account Discord configs that set a default account.

## What This Changes

The PR promotes the eligible configured Discord default account into the zero-delay startup slot and adds Discord startup-delay regressions for default, duplicate-token, and unconfigured-account cases.

## Best Possible Solution

Land the Discord-local fix after maintainer acceptance that an eligible configured default account should own the zero-delay startup slot while secondary accounts keep the existing stagger.

## Reproduction Assessment

Yes. Source inspection shows current main and v2026.6.10 compute Discord startup delay from the sorted filtered account list, so a configured default account that sorts later can receive a nonzero delay.

## Solution Assessment

Yes as an implementation shape, with maintainer acceptance required for upgrade behavior. The PR stays inside the Discord plugin startup path and reuses existing default-account, runtime-enabled, and configured-token predicates instead of hard-coding account names or adding config.
