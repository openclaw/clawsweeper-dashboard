## Summary

Keep open: current main and the latest release still compute Discord startup delay from the sorted account list, while this focused PR promotes the configured default Discord account to the first startup slot with sufficient live-log proof and no blocking findings.

## What This Changes

The PR moves the configured, enabled Discord default account to the first startup-delay slot and adds a regression test for secondary account staggering.

## Best Possible Solution

Land this plugin-local startup-order fix after normal maintainer and CI gates, then let it close the linked Discord startup bug at https://github.com/openclaw/openclaw/issues/77429.

## Reproduction Assessment

Yes. Current main sorts Discord account IDs before computing startup delay, so a configured default account such as main can receive a nonzero delay when enabled accounts like billy or farber sort earlier.

## Solution Assessment

Yes. Reordering only Discord startup-delay candidates with the existing default-account resolver is the narrowest maintainable fix; changing shared account-list ordering would affect other callers unnecessarily.
