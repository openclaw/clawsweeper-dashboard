## Summary

Keep open for maintainer review: current main and the latest release still use Discord account-list order for startup delay, while this PR is a focused, proof-backed fix with no blocking code findings. The remaining decision is whether maintainers accept the intentional startup-order change for existing Discord multi-account configs.

## What This Changes

The PR changes Discord startup delay calculation to promote an eligible configured default account ahead of secondary accounts and adds regressions for default-first, duplicate-token, and unconfigured-account cases.

## Best Possible Solution

Land this Discord-local startup-order fix after maintainer acceptance of the compatibility change, preserving runtime-enabled/configured filtering and secondary-account staggering.

## Reproduction Assessment

Yes. Source inspection shows current main and v2026.6.6 assign Discord startup delay from sorted account-list order, so a config with billy, farber, and main plus channels.discord.defaultAccount: "main" can delay main behind specialist accounts.

## Solution Assessment

Yes, with maintainer acceptance of the upgrade behavior. The PR keeps the fix inside the Discord plugin, reuses the existing default-account and runtime-enabled predicates, and avoids a new config knob or hard-coded account names.
