## Summary

Keep open: the fix direction is right, but the PR still ranks Discord startup candidates before applying Discord's runtime-enabled duplicate-token filter, so a skipped default account can delay the real gateway monitor.

## What This Changes

The PR changes Discord gateway startup delay ordering to move the configured default account ahead of secondary accounts and adds a regression test for distinct enabled accounts.

## Best Possible Solution

Rank only Discord accounts that are enabled for runtime startup, then promote the configured default account within that filtered set and cover duplicate-token ordering in the Discord channel test.

## Reproduction Assessment

Yes. Source inspection shows a duplicate-token Discord setup where the configured default account is disabled by plugin.config.isEnabled, while the PR helper can still rank that skipped account ahead of the real token owner.

## Solution Assessment

No, not yet. Promoting the configured default account is the right fix shape, but it must happen after Discord's existing runtime-enabled account filtering.
