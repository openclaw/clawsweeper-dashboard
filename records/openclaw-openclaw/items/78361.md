## Summary

Keep open: current main fixed the narrow cron equal-timeout propagation bug, but the requested user-facing idle-timeout config surfaces under agent defaults, per-agent config, and cron flags are still absent and need a maintainer product decision.

## What This Changes

Expose the model stream idle watchdog as configurable agent-default, per-agent, and per-cron settings instead of relying on the hardcoded/default provider-timeout behavior.

## Best Possible Solution

Define one supported idle-timeout contract, decide its precedence against existing provider and run timeouts, then update schema, CLI cron payloads, docs, doctor/config behavior, and focused regression tests around that contract.

## Reproduction Assessment

Yes for the missing config surface: current source, schema, CLI, and docs show no requested idleTimeoutMs controls. The live Gemini stall itself was not replayed in this read-only review.

## Solution Assessment

Unclear. Dedicated idle-timeout knobs could solve the reported operator need, but maintainers should first choose whether this belongs in a new agent/cron hierarchy or in the existing provider/request-timeout contract.
