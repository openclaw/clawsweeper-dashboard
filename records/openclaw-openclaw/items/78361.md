## Summary

Keep open: current main and the latest release still lack the requested user-facing idle-timeout controls, and the remaining work changes public config/CLI semantics rather than being a narrow bug fix.

## What This Changes

Expose model stream idle-timeout controls for agent defaults, per-agent config, and cron jobs so users can widen or disable the 120s idle watchdog.

## Best Possible Solution

Define one supported idle-timeout contract and precedence, then update the resolver, attempt wiring, config schema/types/help, cron payload/CLI, docs, doctor behavior, and focused timeout tests together.

## Reproduction Assessment

Yes for the missing configuration surface: current source, schema, CLI, and docs expose no requested idleTimeoutMs controls, and the cloud non-cron resolver still caps implicit idle windows at 120s. The exact Gemini preview stall was not replayed.

## Solution Assessment

Unclear. Dedicated idle-timeout knobs are plausible, but maintainers should first choose the supported public surface and precedence against provider, run, first-token, generation, and cron timeouts.
