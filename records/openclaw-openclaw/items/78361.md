## Summary

Keep open: current main still lacks a user-facing idle-watchdog config surface, and the requested global/per-agent/per-cron controls require a maintainer timeout-contract decision before implementation.

## What This Changes

Request to expose the embedded model-stream idle watchdog as configurable globally, per agent, and per cron job, with an optional disable mode.

## Best Possible Solution

Define one explicit timeout contract that separates provider request timeout, stream idle timeout, first-token timeout, and total generation timeout, then wire the chosen idle setting through config schema, model/run resolution, cron protocol/CLI, docs, and regression tests.

## Reproduction Assessment

Yes for the actionable missing config surface: source inspection shows the idle watchdog is resolved internally and the requested schema/cron fields are absent. No live Gemini preview stall was reproduced during this read-only review.

## Solution Assessment

Unclear until maintainers choose the timeout contract. Adding `idleTimeoutMs` exactly as requested may be maintainable, but it should be reconciled with provider `timeoutSeconds` and the existing generation-timeout work before implementation.
