## Summary

Keep open: current main and v2026.6.8 still have the source-backed `models.json` `$VAR`/`${VAR}` audit false-positive path, and the inactive-surface/provider-marker parts need security/product review rather than cleanup closure.

## What This Changes

The issue asks for `openclaw secrets audit --check` to stop flagging SecretRef/env marker shorthands and inactive or placeholder model credentials as unresolved/plaintext, plus clearer custom provider SecretRef behavior.

## Best Possible Solution

Land a security-reviewed audit fix that aligns generated `models.json` marker handling with the SecretInput/runtime contract, covers inactive-surface semantics explicitly, and preserves warnings for real plaintext credentials.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: put `apiKey: "$OPENAI_API_KEY"` or `"${OPENAI_API_KEY}"` in an audited `models.json`, and current `collectModelsJsonSecrets` emits `REF_UNRESOLVED` because that string is coerced as a SecretRef.

## Solution Assessment

No for closing or blind automation. The shorthand bug has a clear code path, but the best fix needs a narrow security-approved boundary for generated markers and inactive surfaces so the audit does not create false negatives.
