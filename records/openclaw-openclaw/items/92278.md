## Summary

This focused recovery change is useful but not merge-ready: the shared formatter can recommend unusable identifiers on the code-mode exact-ID path, and the supplied validation does not demonstrate real model recovery.

## What This Changes

The PR adds up to three Levenshtein-ranked catalog suggestions and a `tool_search` recovery hint to unknown Tool Search tool-ID errors.

## Best Possible Solution

Render the canonical `entry.id` in every suggestion—optionally alongside its human-readable name—so the hint is valid for both lookup contracts, add focused raw and code-mode regression tests, and demonstrate same-turn recovery in a redacted live run.

## Reproduction Assessment

Yes at the source level: current main returns a bare error for an unknown raw `tool_call` identifier, and the linked issue provides repeated production logs reaching that path. This review did not execute the full live memory-flush scenario.

## Solution Assessment

No in its current form. Closest-match guidance is a suitable narrow fix, but it should recommend canonical catalog IDs that every affected dispatch path accepts rather than names that code mode rejects.
