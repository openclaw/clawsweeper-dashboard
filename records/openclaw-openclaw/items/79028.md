## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.78

Full review comments:

- **[P2] Normalize GroupSpace before resolving toolsAllow:** `src/auto-reply/reply/agent-runner-utils.ts:344`
  - body: This new preflight policy lookup passes `params.sessionCtx.GroupSpace` through unnormalized, but the actual embedded run later resolves policy with `normalizeOptionalString(params.sessionCtx.GroupSpace)`. A padded or whitespace-only group space can make `resolveGroupToolPolicy` derive a different group-scoped allowlist for `toolsAllow`, so group-scoped plugin tools may still be dropped before the run even though the final embedded policy context would have allowed them. Normalize this value the same way as the embedded run call.
  - confidence: 0.78
- **[P3] Preserve allow-all policies when appending plugin entries:** `src/agents/tool-policy-pipeline.ts:176-190`
  - body: `allow: []` is treated as allow-all by the tool policy matcher, but `preserveExplicitPluginAllowlist()` will append preserved plugin names and turn that policy into a restrictive allowlist. Return early for empty allow arrays before appending preserved entries so existing allow-all semantics stay intact.
  - confidence: 0.72
