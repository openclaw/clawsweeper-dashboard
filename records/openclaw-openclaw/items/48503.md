## Summary

Keep open. Current main still exposes `before_tool_call` without the requested action classification or input provenance, and the related HELM OSS comment is useful prior art for a public Plugin SDK/security design rather than a reason to close this as stale or implemented.

## What This Changes

Request to add optional action classification and input provenance fields to the `before_tool_call` plugin hook event.

## Best Possible Solution

Define an additive, versioned Plugin SDK contract for host-owned action/capability metadata and redacted active input provenance, then thread it through trusted and ordinary `before_tool_call` hooks with docs and contract tests.

## Reproduction Assessment

Not applicable as a feature request. The gap is source-verifiable: current main’s hook type, runtime dispatch, and public docs omit action classification and `inputProvenance` while input provenance exists elsewhere in the session/runner pipeline.

## Solution Assessment

Unclear until maintainers choose the public contract. The safer direction is to align this with tool capability descriptors and provenance/trust-model work, then expose a redacted stable view rather than adding a one-off enum only to the event.
