Add optional `before_tool_call` hook fields for stable action classification and input provenance so security policy plugins can avoid brittle tool-name mapping.

Define a stable plugin hook policy context across tool capability descriptors, external-content provenance, and input provenance, then land the chosen optional fields with docs and regression tests.
