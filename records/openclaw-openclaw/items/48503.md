The issue requests optional action classification and input provenance fields on the `before_tool_call` plugin hook event.

## Best Possible Solution

Define a stable optional policy-context contract for `before_tool_call`, coordinate it with tool capability descriptors and external-content provenance work, then land SDK types, runtime propagation, docs, and focused hook tests after maintainer security review.

## Solution Assessment
