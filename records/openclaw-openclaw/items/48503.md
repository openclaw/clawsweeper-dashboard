## Summary

Keep open: current main and the latest release still lack the requested action-category and input-provenance fields on `before_tool_call`, while the overlapping open PR only covers a narrower `externalContent` signal and the public security taxonomy still needs maintainer approval.

## What This Changes

The issue asks OpenClaw to add optional action classification and input provenance to the plugin `before_tool_call` event so security policy plugins can avoid brittle tool-name maps.

## Best Possible Solution

Define a stable additive plugin hook policy context across tool classification, capability metadata, and provenance, then land the chosen optional fields with docs and regression tests.

## Reproduction Assessment

Yes at source level, but not as an end-user bug reproduction. Current main and `v2026.6.6` expose hook identity, params, ids, `toolKind`/`toolInputKind`, and `derivedPaths`, but not the requested action classification or input provenance fields.

## Solution Assessment

Unclear as proposed. The direction fits OpenClaw's plugin-security API goals, but maintainers still need to choose the stable taxonomy and provenance-disclosure shape before implementation.
