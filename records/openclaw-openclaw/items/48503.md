Keep open. Current main and the latest shipped tag still expose `before_tool_call` without action classification or input provenance, while related capability/provenance/security work remains open and needs maintainer API/security decisions.

## What This Changes

The issue requests enriching the plugin `before_tool_call` event with stable action classification and input provenance fields for security policy plugins.

## Best Possible Solution

Define an additive Plugin SDK contract for host-owned action/capability metadata plus redacted active input provenance, then thread it through trusted policies and ordinary `before_tool_call` hooks with docs, API baseline updates, and contract tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/API request. The gap is source-verifiable because current hook types, dispatch, docs, and the latest shipped tag omit action classification and `inputProvenance` while input provenance exists elsewhere.

## Solution Assessment

Unclear until maintainers choose the public contract. The safer direction is to coordinate this with capability descriptors, tool security classification, and provenance/trust-model work before exposing a redacted stable view.
