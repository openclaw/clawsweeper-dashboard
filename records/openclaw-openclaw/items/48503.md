## Summary

Keep open. Current main still exposes `before_tool_call` with tool name, params, run/tool IDs, and `derivedPaths`, but not the requested action classification or input provenance; the HELM OSS comment is useful prior-art context, and the remaining work is a public Plugin SDK/security design decision rather than stale cleanup.

## What This Changes

The issue requests optional action classification and input provenance fields on the `before_tool_call` plugin hook event.

## Best Possible Solution

Define an additive, versioned Plugin SDK contract for host-owned action/capability metadata and redacted active input provenance, thread it through `before_tool_call` and trusted policies, then document and test the trust model.

## Reproduction Assessment

Not applicable as a feature request. The gap is source-verifiable: current main’s hook type, runtime dispatch, and docs omit action classification and `inputProvenance` while input provenance exists elsewhere in the runner/session pipeline.

## Solution Assessment

Unclear until maintainers choose the public contract. The maintainable direction is to align this with tool capability descriptors and durable provenance work, then expose a redacted stable view rather than adding a one-off enum only to the event.
