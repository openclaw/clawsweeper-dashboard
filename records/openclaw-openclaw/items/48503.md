## Summary

Keep open: current main and `v2026.6.6` still lack the requested `before_tool_call` action classification and input-provenance hook contract. The related open work covers only pieces of the policy context and still needs maintainer API/security direction.

## What This Changes

The issue requests optional `before_tool_call` hook fields for stable tool-action classification and input provenance so security policy plugins do not have to infer policy context from tool names.

## Best Possible Solution

Define a stable optional policy-context contract for `before_tool_call`, coordinate the taxonomy with https://github.com/openclaw/openclaw/issues/62962 and https://github.com/openclaw/openclaw/pull/91800, then land docs and regression tests after maintainer security approval.

## Reproduction Assessment

Yes, at source level. Current main exposes hook identity, params, ids, `toolKind`/`toolInputKind`, and `derivedPaths`, while input provenance exists elsewhere and is not surfaced on the tool hook event.

## Solution Assessment

Unclear as proposed. The direction fits OpenClaw's plugin-security API goals, but maintainers still need to choose the stable taxonomy and provenance-disclosure shape before implementation.
