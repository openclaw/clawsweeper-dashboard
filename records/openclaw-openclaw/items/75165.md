## Summary

Keep open. Current main does not implement the termination algebra or GSAR scorer, and the PR is not merge-ready because the live-test expectation contradicts the new `MaxIterations` contract.

## What This Changes

The PR adds an agents termination-condition algebra, a GSAR grounding scorer with tests/live proof, a demo script, a deadcode allowlist entry, and an optional `termination` hook in `runSessionsSendA2AFlow`.

## Best Possible Solution

Land only after correcting the live-test contract and demo live-mode claim, then have agents maintainers decide whether this API belongs in core now, behind an SDK/plugin seam, or behind a production consumer.

## Reproduction Assessment

Yes for the review findings: `runLiveLoop` returns the `MaxIterations` stop reason at turn 5 while the flat live test asserts `null`, and the demo script advertises `--live` without any code path that reads that flag or calls a live model. Reproduction is not applicable to the feature request itself because this PR proposes new behavior.

## Solution Assessment

No; the PR is not merge-ready as written. The narrow repair is to expect `max_iterations` or test a real no-condition exhaustion path, and either implement the demo's live mode or remove the advertised live command.
