## Summary

Keep open: this is active feature work, and current head still has a blocking integration gap because the new termination hook is not reachable from the production `sessions_send` caller. The module/demo proof is useful, but agents maintainers still need to decide the API placement and fix the wiring before merge.

## What This Changes

The PR adds core agents termination and GSAR modules with tests and a demo, plus an optional `TerminationCondition` hook on the A2A session-send helper.

## Best Possible Solution

Have agents maintainers choose the API boundary, then either expose a typed `sessions_send` termination policy with production A2A tests or narrow the PR to a library-only algebra/GSAR addition whose docs and demo do not claim tool-level integration.

## Reproduction Assessment

Not applicable as a bug reproduction for the main feature request. Yes for the blocking review finding: source inspection of current PR head shows the new `termination` parameter is accepted only by the A2A helper and is never passed by the production `sessions_send` caller.

## Solution Assessment

Unclear for the full feature: the algebra and GSAR helpers are plausible, but the current PR is not the best complete product solution because the production caller cannot exercise the new condition. A safer path is a maintainer-approved typed API seam plus integration coverage, or a clearly library-only scope.
