## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Require trusted evidence before accepting grounding tags:** `src/agents/gsar.ts:247-253`
  - body: `buildHybridScorer()` treats any `[G]`, `[U]`, `[X]`, or `[K]` tags in the model reply as the grounding partition. A hallucinating or prompted model can self-label its own claims as grounded, letting `GroundednessCondition` stop the A2A loop without any trusted evidence check; make the scorer consume trusted retrieved evidence or remove this fast path from production-facing use.
  - confidence: 0.94
- **[P1] Wire the policy through real sessions_send callers:** `src/agents/tools/sessions-send-tool.a2a.ts:87-88`
  - body: The new parameter exists only on the internal A2A helper, while `sessions-send-tool.ts` still calls it without supplying any condition. Normal `sessions_send` runs therefore keep the fixed-turn behavior despite this PR's claimed runtime feature; add an owner-approved policy at the real entry point or scope the change as library-only groundwork.
  - confidence: 0.96
- **[P1] Check termination before scheduling the next A2A turn:** `src/agents/tools/sessions-send-tool.a2a.ts:187-188`
  - body: The first condition check happens after `runAgentStep()` has already sent `incomingMessage` to the other agent. If the initial target reply already contains `FOUND IT` or reaches GSAR proceed, the code still burns another A2A turn and may change `latestReply` before announce; evaluate the available reply before sending the next turn.
  - confidence: 0.9
