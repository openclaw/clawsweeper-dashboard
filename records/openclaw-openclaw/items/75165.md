## Summary

Keep open. The PR adds useful agent-loop machinery, but the current head is not merge-ready because one live test contradicts the new MaxIterations contract; current main also does not already implement the proposed termination algebra or GSAR scorer.

## What This Changes

The PR adds a composable agent termination API, a GSAR groundedness scorer, A2A loop integration via an optional termination condition, new unit/live tests, and a deadcode allowlist entry for the new GSAR module.

## Best Possible Solution

Fix the deterministic live-test mismatch first, then have agents maintainers decide whether the termination algebra and GSAR scorer should land in core, move behind a documented SDK/plugin contract, or wait for a production consumer.

## Reproduction Assessment

Not applicable for the feature itself because this PR proposes new behavior rather than reporting a current-main bug. Yes for the review finding: on the PR head, `runLiveLoop` returns `max_iterations` when `MaxIterations(MAX_TURNS)` fires, while the flat live test asserts `null`.

## Solution Assessment

No; the PR is not merge-ready as written. The narrowest repair is to expect `max_iterations` in the flat live test, or change that test to exercise a no-condition exhaustion path if `null` is the intended signal.
