## Summary

Keep this PR open. Current main does not already have the proposed termination algebra or GSAR surface, and the PR still has a concrete correctness blocker in its live test: the flat `MaxIterations` path returns a stop reason at the budget while the assertion expects no reason. The author's follow-up usefully clarifies that the live proof runs through OpenClaw's existing model/transport helpers, but that does not change the deterministic assertion mismatch.

## What This Changes

The PR adds a new agents termination algebra, a GSAR groundedness scorer and condition with unit/live tests, an optional `termination` hook in `runSessionsSendA2AFlow`, and a Knip unused-file allowlist entry for the currently test-only GSAR module.

## Best Possible Solution

Repair the deterministic live-test expectation first, then have maintainers decide whether the termination algebra belongs in core now and whether GSAR should land as test-only library code, move behind a documented evidence/scorer contract, or wait for a real non-test consumer.

## Reproduction Assessment

Not applicable for the feature request itself because this PR adds a new API rather than reporting a current-main bug. Yes for the review finding: on the PR head, `runLiveLoop` returns `max_iterations` when `MaxIterations(MAX_TURNS)` fires, while the flat live test asserts `null`.

## Solution Assessment

No, this is not merge-ready as written. The narrowest repair is to expect `max_iterations` for the flat `MaxIterations` live case, or change that test to exercise a no-condition exhaustion path if `null` is the intended signal.
