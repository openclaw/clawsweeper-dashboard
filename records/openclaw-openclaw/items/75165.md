## Summary

Keep open for maintainer review: the branch contains plausible new agent-runtime work, but the proposed production integration is incomplete and still needs an owner decision on the public termination-policy surface.

## What This Changes

The PR adds termination and GSAR agent modules, tests/live-demo coverage, a deadcode allowlist entry, and an optional termination hook on the A2A session-send helper.

## Best Possible Solution

Have agents owners choose a stable typed termination-policy surface, then either wire it through production sessions_send with integration docs/tests or narrow this branch to library-only helpers and remove production integration claims.

## Reproduction Assessment

Not applicable for the main request because this is new feature work, not a broken existing contract. The blocking review findings are source-reproducible from PR head: the production caller passes no condition, and the demo never selects a live provider path.

## Solution Assessment

No for the current PR as a complete solution: the helper and library pieces may be useful, but the production sessions_send path cannot exercise the new policy. The safer path is a maintainer-approved tool/config/API seam or an explicitly library-only scope.
