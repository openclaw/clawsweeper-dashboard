Full review comments:

- **[P1] Pass evidence into the judge scorer:** `src/agents/gsar.ts:195-205`
  - body: `buildLlmJudgeScorer` asks the judge to label claims as grounded or contradicted while only passing the reply text. Without retrieved evidence, citations, session facts, or tool outputs, this cannot distinguish a plausible false claim from a supported one, yet `GroundednessCondition` can terminate on the resulting score. Thread the evidence/provenance context through the scorer API or remove this helper until it has real grounding input.
  - confidence: 0.91
- **[P2] Wire GSAR into a reachable runtime path:** `scripts/deadcode-unused-files.allowlist.mjs:23`
  - body: The PR adds `src/agents/gsar.ts` to the unused-file allowlist, and the PR tree has no production imports of `GroundednessCondition` outside the new module itself. That means the advertised first-class GSAR behavior is only exercised by tests/demo code, not by the A2A runtime. Either add the real runtime/API wiring with docs/proof or split GSAR into a follow-up.
  - confidence: 0.88
- **[P2] Isolate termination scorer failures:** `src/agents/tools/sessions-send-tool.a2a.ts:129`
  - body: The new hook awaits `termination.check()` inside the outer A2A try block before the final announce step. Because the new API explicitly supports async scorers, a model timeout, parser error, or scorer throw will make the whole announce flow fall into the outer catch and skip delivery. Catch scorer failures and continue or fall back to the existing bounded loop behavior.
  - confidence: 0.84
