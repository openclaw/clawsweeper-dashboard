## Summary

Keep open: the PR is active and not implemented on main, but current head still leaves the advertised `sessions_send` termination behavior unreachable from the production tool path and has a misleading live-demo command.

## What This Changes

The branch adds agent termination and GSAR modules with tests/live proof/demo, plus an optional `termination` hook on `runSessionsSendA2AFlow`.

## Best Possible Solution

Have agents maintainers choose the API boundary, then either wire a typed production `sessions_send` termination policy with integration coverage/docs or scope this as library-only and remove tool-level/live-demo claims.

Not applicable as a bug reproduction for the feature itself. Yes for the blocking review findings: source inspection of PR head shows the helper-only termination hook is not reachable from the production `sessions_send` schema/caller, and the demo advertises a `--live` mode that is not implemented.

## Solution Assessment

No for the current PR as a complete product solution: the helpers are plausible, but the production caller cannot exercise them and the demo overstates live behavior. A narrower maintainable path is a maintainer-approved typed API seam with real A2A tests, or a clearly library-only addition.
