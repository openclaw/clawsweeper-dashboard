## Summary

Keep open. Current `main` does not implement the proposed termination/GSAR surface, and the PR is still not merge-ready because the live test expects no exit reason even though `MaxIterations` returns `max_iterations` when it fires.

## What This Changes

This PR adds a composable agent `TerminationCondition` algebra, a GSAR groundedness scorer, an optional A2A termination hook, new unit/live tests, and a deadcode allowlist entry for the new GSAR module.

## Best Possible Solution

Fix the deterministic live-test mismatch, then have agents maintainers decide whether the termination algebra and GSAR scorer should land in core, move behind a documented SDK/plugin contract, or wait for a production consumer.

## Reproduction Assessment

Not applicable for the feature itself because this PR proposes new behavior rather than reporting a current-main bug. Yes for the review finding: on the PR head, `runLiveLoop` returns `max_iterations` when `MaxIterations(MAX_TURNS)` fires, while the flat live test asserts `null`.

## Solution Assessment

No; the PR is not merge-ready as written. The narrowest repair is to expect `max_iterations` in the flat live test, or change that test to exercise a no-condition exhaustion path if `null` is the intended signal.
