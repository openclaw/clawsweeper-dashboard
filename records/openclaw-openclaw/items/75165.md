## Summary

Keep open: the feature is plausible, but the current PR is not merge-ready because its new termination hook is still unreachable from the production `sessions_send` path and the demo advertises a live mode it does not implement.

## What This Changes

The PR adds new agents termination and GSAR modules with tests/live-demo coverage, adds `src/agents/gsar.ts` to the deadcode allowlist, and adds an optional `termination` hook to the A2A session-send helper.

## Best Possible Solution

Have agents maintainers choose the API boundary, then either expose a typed `sessions_send` termination policy with production A2A tests/docs or narrow the PR to a library-only algebra/GSAR addition and remove tool-level integration claims.

## Reproduction Assessment

Not applicable for the main feature request because it is new behavior, not a broken existing contract. Yes for the blocking review findings: source inspection of PR head shows the new condition is never passed by the production caller, and the demo has no live-provider branch despite documenting `--live`.

## Solution Assessment

No for the current PR as a complete solution: the algebra and GSAR helpers may be useful, but the production `sessions_send` path cannot exercise them. The safer path is a maintainer-approved typed API/config seam with integration coverage, or an explicitly library-only scope.
