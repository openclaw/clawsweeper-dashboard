- **[P2] Export the new hook types from plugin-runtime:** `extensions/discord/src/monitor/message-handler.ts:6-9`
  - body: This file imports `PluginHookInboundActivityContext` and `PluginHookInboundActivityEvent` from `openclaw/plugin-sdk/plugin-runtime`, but that subpath only re-exports hook-runner-global and `plugins/types`; the new type exports were added to `src/plugins/hooks.ts`, which is not part of this SDK subpath. The Discord handler will fail type-checking with missing exported members until the types are exported through the subpath or imported from an exported source.
  - confidence: 0.94
- **[P2] Apply message controls after creating the buffer:** `src/auto-reply/inbound-debounce.ts:232-235`
  - body: `emitActivity` runs before the first debounce buffer is inserted into `buffers`, while `ctx.extend()` and `ctx.cancel()` call helpers that immediately return when `buffers.get(key)` is missing. That makes the controls silently no-op for the initial `MESSAGE_CREATE` activity even though the new hook exposes them for that activity key; create the buffer first or carry the requested control changes into buffer creation.
  - confidence: 0.84
