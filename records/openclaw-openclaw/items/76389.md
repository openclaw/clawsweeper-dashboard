## Summary

Keep open. The underlying exec SSRF gap is real, but the updated patch is not merge-ready because the new parser still skips common curl/wget URL forms, rejects ordinary non-network gateway exec syntax, and changes a security boundary that needs maintainer/security review.

## What This Changes

The PR adds a gateway-host exec SSRF filter for curl/wget/httpx/aria2c-style commands, integrates it into `createExecTool`, adds tests, and updates the changelog.

## Best Possible Solution

Land a security-owner-approved exec egress guard that preserves normal gateway exec syntax and either uses robust egress isolation/proxying or a fail-closed parser with complete curl/wget option coverage and regression tests.

## Reproduction Assessment

Yes, at source level. Current main has no gateway exec SSRF filter, and the PR head still permits bypass shapes such as `curl --compressed http://127.0.0.1/...` and `wget -d http://localhost/...` by skipping those URLs as option values.

## Solution Assessment

No. The direction addresses a valid gap, but this implementation is not the narrowest maintainable solution until the parser bypasses, global non-network exec regression, and security-boundary tradeoff are resolved.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Do not turn parse misses into global exec denials:** `src/infra/exec-ssrf-filter.ts:756-762`
  - body: This filter runs for every gateway exec command before it knows whether a network tool is present. `analyzeShellCommand` rejects ordinary shell syntax such as redirection (`>`), `;`, `&&`, or command substitution, so non-network commands would now fail with an SSRF error; this return also omits the `blockedHost`/`blockedUrl` fields required by the false result arm.
  - confidence: 0.94
- **[P1] Classify curl standalone flags before consuming values:** `src/infra/exec-ssrf-filter.ts:609-610`
  - body: The value-taking set is still checked before standalone flags, and it still contains no-value curl options such as `-S` and `--compressed`. `curl --compressed http://127.0.0.1/...` advances over the URL as an option value and is allowed.
  - confidence: 0.93
- **[P1] Classify wget standalone flags before consuming values:** `src/infra/exec-ssrf-filter.ts:656-657`
  - body: `WGET_OPTIONS_WITH_VALUE` is also checked before standalone flags and includes no-value wget flags such as `-d`, `-v`, `-r`, and `--spider`. `wget -d http://localhost/file` skips the URL and leaves the direct fallback unblocked.
  - confidence: 0.93
