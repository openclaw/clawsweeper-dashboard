## Summary

Keep open. The PR addresses a valid host-exec SSRF gap from #76260, but the diff is not merge-ready because the new parser skips common curl/wget URL forms and fails open on unsupported shell syntax in a security boundary.

## What This Changes

The PR adds a gateway-host exec SSRF filter for network-tool commands, wires it into `createExecTool`, adds tests, and adds a changelog entry.

## Best Possible Solution

Land a security-owner-approved exec egress policy that either uses robust sandbox/egress isolation or a fail-closed argv mitigation with regression tests for common curl/wget forms and parser failure paths.

## Reproduction Assessment

Yes, at source level. Current main has no host-exec SSRF check, and the PR branch filter skips commands such as `curl -L http://127.0.0.1:18760/status/500` and `wget -q http://localhost/file` because value-taking option sets are evaluated first.

## Solution Assessment

No, not as written. A parser-based mitigation may be useful, but this implementation skips direct URL forms and fails open on unsupported shell syntax; a fail-closed guard or stronger egress boundary is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Do not skip URLs after standalone curl flags:** `src/infra/exec-ssrf-filter.ts:623-625`
  - body: Because `CURL_OPTIONS_WITH_VALUE` is checked before standalone options and includes flags like `-L`, `-k`, `-I`, `-4`, and `--compressed`, `curl -L http://127.0.0.1:18760/status/500` advances past the URL as an option value and returns allowed.
  - confidence: 0.94
- **[P1] Validate URL-bearing option assignments:** `src/infra/exec-ssrf-filter.ts:634-637`
  - body: The blanket `--option=value` skip means direct curl URL forms such as `curl --url=http://127.0.0.1:18760/status/500` are never checked. Parse URL-bearing assignments explicitly before hostname validation.
  - confidence: 0.91
- **[P1] Classify wget flags before consuming values:** `src/infra/exec-ssrf-filter.ts:663-665`
  - body: `WGET_OPTIONS_WITH_VALUE` also contains standalone flags and is checked first, including `-q`. As a result, `wget -q http://localhost/file` skips the blocked URL and is allowed.
  - confidence: 0.93
- **[P2] Fail closed when network command parsing is unsupported:** `src/infra/exec-ssrf-filter.ts:762-765`
  - body: When `analyzeShellCommand` rejects syntax, the filter returns allowed even though gateway exec still runs the original shell command. Unsupported network-tool commands need a conservative scan or denial path.
  - confidence: 0.88
