## Summary

Keep open. The PR targets a real current-main host-exec SSRF gap, but it is not merge-ready because the new parser skips common curl/wget URL forms and fails open on unsupported shell syntax, leaving obvious localhost fallback paths unblocked.

## What This Changes

This PR adds a gateway-host exec SSRF filter for curl/wget/httpx/aria2c-style commands, wires it into `createExecTool`, adds tests, and updates the changelog.

## Best Possible Solution

Land a security-owner-approved exec egress policy that either provides a robust sandbox/egress boundary or a clearly documented, fail-closed argv mitigation with regression tests for common curl/wget forms and known bypass classes.

## Reproduction Assessment

Yes, at source level. Current main has no host-exec SSRF filter, and the PR branch source shows bypasses such as `curl -L http://127.0.0.1:18760/status/500` being skipped because `-L` is treated as value-taking before standalone flags.

## Solution Assessment

No, not as written. A parser-based mitigation may be useful, but this implementation must not skip common URL forms or fail open on shell syntax it cannot analyze; maintainers should also decide whether argv scanning is acceptable versus egress isolation.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Do not skip URLs after standalone curl flags:** `src/infra/exec-ssrf-filter.ts:623-625`
  - body: Because `CURL_OPTIONS_WITH_VALUE` is checked before the standalone set and includes flags such as `-L`, `-k`, `-I`, `-4`, and `--compressed`, `curl -L http://127.0.0.1:18760/status/500` advances past the URL as if it were an option value and returns allowed. That leaves a very common curl fallback path unblocked.
  - confidence: 0.94
- **[P1] Validate URL-bearing option assignments:** `src/infra/exec-ssrf-filter.ts:634-637`
  - body: The blanket `--option=value` skip means direct curl URL forms such as `curl --url=http://127.0.0.1:18760/status/500` are never checked. Parse URL-bearing options explicitly instead of discarding every long option assignment before hostname validation.
  - confidence: 0.91
- **[P1] Classify wget flags before consuming values:** `src/infra/exec-ssrf-filter.ts:663-665`
  - body: `WGET_OPTIONS_WITH_VALUE` also contains standalone flags that are checked first, including `-q`. As a result, `wget -q http://localhost/file` skips the blocked URL and is allowed, which leaves the obvious wget fallback path open.
  - confidence: 0.93
- **[P2] Fail closed when network command parsing is unsupported:** `src/infra/exec-ssrf-filter.ts:762-765`
  - body: When `analyzeShellCommand` rejects syntax such as command substitution, the filter returns allowed even though gateway exec still runs the original shell command. A command like `curl http://$(printf 127.0.0.1):18760/status/500` can bypass the new guard, so unsupported network-tool commands need a conservative scan or denial path.
  - confidence: 0.88
