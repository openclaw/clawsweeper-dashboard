## Summary

Keep open. The PR targets a real current-main exec/web_fetch SSRF gap, but the proposed extractor skips blocked URLs after common flag-only curl/wget options, preserving easy bypasses of the new guard.

## What This Changes

The PR adds a gateway exec SSRF filter for network-tool commands, hooks it into `createExecTool`, adds focused tests, and records the fix in the changelog.

## Best Possible Solution

Keep the PR open and fix the argument scanner so common network-tool flags cannot hide blocked destinations, then land it as a narrow partial mitigation for the exec fallback reported in #76260.

## Reproduction Assessment

Yes, from source inspection of the PR branch: `extractUrlsFromCurlArgs` and `extractUrlsFromWgetArgs` skip the URL following flag-only options before applying `isBlockedHostnameOrIp`. A focused regression test with `curl -L http://127.0.0.1` or `wget -q http://127.0.0.1` should reproduce the bypass.

## Solution Assessment

No. Applying a gateway exec filter is the right direction for the reported fallback, but the implementation should use accurate option parsing or fail closed for known network-tool URL extraction misses before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Stop skipping URLs after curl flag-only options:** `src/infra/exec-ssrf-filter.ts:134-136`
  - body: `extractUrlsFromCurlArgs` checks `CURL_OPTIONS_WITH_VALUE` first, but that set includes flag-only options such as `-L`/`--location`. As a result, `curl -L http://127.0.0.1:18760/status/500` skips the blocked URL entirely and preserves a common exec SSRF bypass path this PR is meant to close.
  - confidence: 0.94
- **[P1] Do not treat wget quiet flags as value-taking:** `src/infra/exec-ssrf-filter.ts:352-353`
  - body: `extractUrlsFromWgetArgs` also checks the value-taking set first, but `-q`/`--quiet` are standalone flags. `wget -q http://127.0.0.1/file` therefore advances past the private URL without validation, so wget still bypasses the new SSRF filter in a normal invocation.
  - confidence: 0.92
