## Summary

Keep open. Current main still has the warn-level `potential-exfiltration` audit path and no manifest-driven egress-intent/rationale contract; this is a new security-audit policy and public plugin manifest/config surface, not a broken existing behavior.

## What This Changes

The issue requests a plugin manifest capability/rationale field so legitimate file-read-plus-network-send plugins can annotate, downgrade, or suppress `plugins.code_safety` exfiltration warnings.

## Best Possible Solution

Define a small versioned manifest/audit contract that records declared egress intent and rationale while preserving operator visibility by default, then document any explicit acknowledgement path.
