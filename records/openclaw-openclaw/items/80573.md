## Summary

Keep open. Current main and the latest release still have the warn-level `potential-exfiltration` audit path and no manifest-backed egress-intent/rationale contract; this is a concrete feature request, but it changes security-audit trust semantics and public plugin/config surface.

## What This Changes

Request to let plugins declare legitimate egress capabilities and rationale in `openclaw.plugin.json` so audit can annotate, downgrade, or suppress expected `potential-exfiltration` findings.

## Best Possible Solution

Define a small versioned manifest/audit contract for declared egress intent and rationale, keep findings visible by default, and only downgrade or suppress through explicit operator policy.

## Reproduction Assessment

Not applicable: this is a feature request, not a report of broken current behavior. Source inspection confirms current main still emits the warn-level exfiltration path and lacks a manifest or acknowledgement contract.

## Solution Assessment

Unclear: the declaration-and-rationale direction is plausible, but maintainers need to choose annotate, downgrade, or suppress semantics and coordinate with the broader acknowledgement discussion at https://github.com/openclaw/openclaw/issues/17447.
