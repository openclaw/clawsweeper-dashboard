## Summary

Keep open. Current main still has the warn-level `potential-exfiltration` audit path and no manifest-backed egress intent/rationale contract; the related general suppression issue at https://github.com/openclaw/openclaw/issues/17447 is adjacent but does not supersede this distinct plugin-manifest trust-contract proposal.

## What This Changes

The issue requests a plugin manifest capability field and rationale so intentional backup/sync/export egress can be surfaced or acknowledged without permanent `potential-exfiltration` warning noise.

## Best Possible Solution

Define a small versioned manifest/audit contract that records declared egress intent and rationale while keeping findings visible by default, then document any explicit operator acknowledgement path and cover it in scanner, audit, manifest, and docs tests.

## Solution Assessment
