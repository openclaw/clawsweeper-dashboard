## Summary

Keep open for maintainer follow-up. Current main and the latest release still have the warn-level `potential-exfiltration` scanner/audit path with no plugin manifest egress declaration, rationale, or operator acknowledgement contract, and the requested change is a security-audit policy decision rather than a bug fix.

## What This Changes

Allow plugins to declare legitimate egress intent and rationale in `openclaw.plugin.json` so `plugins.code_safety` `potential-exfiltration` warnings can be annotated, downgraded, or acknowledged.

## Best Possible Solution

Define a documented manifest/audit contract for declared egress intent and rationale, keep findings visible by default, and require explicit operator acknowledgement before any suppression.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken current behavior report. Source inspection confirms the current warn path exists and the requested manifest/acknowledgement contract is absent.

## Solution Assessment

Unclear: the declaration-and-rationale direction is plausible, but maintainers need to choose whether audit should annotate, downgrade, or suppress only after explicit operator acknowledgement.
