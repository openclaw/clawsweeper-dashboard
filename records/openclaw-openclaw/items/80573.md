## Summary

Keep open. Current main still has the reported `potential-exfiltration` warning path and no plugin-manifest egress intent/rationale contract, but this is a new security-audit policy and public manifest feature rather than broken existing behavior.

## What This Changes

Request a plugin manifest egress-intent capability/rationale field so `plugins.code_safety` potential-exfiltration findings can be annotated, downgraded, or suppressed after acknowledgement.

## Best Possible Solution

Design a small versioned manifest/audit contract that exposes declared egress intent and rationale without silently hiding suspicious code, then document it and add focused scanner/audit coverage.
