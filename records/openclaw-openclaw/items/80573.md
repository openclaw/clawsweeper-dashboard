## Summary

Keep open. Current main still emits warn-level `potential-exfiltration` through `plugins.code_safety` and has no manifest egress-intent/rationale contract, but this is a security-sensitive feature and policy decision rather than a strict bug fix.

## What This Changes

Add a manifest-level egress intent and rationale contract so security audit can annotate, downgrade, or suppress legitimate plugin exfiltration warnings.

## Best Possible Solution

Define a small explicit manifest/audit contract that records declared egress intent and rationale while keeping warnings visible by default unless a maintainer-approved operator acknowledgement model is added.
