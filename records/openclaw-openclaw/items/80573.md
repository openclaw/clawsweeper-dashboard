## Summary

Keep open: current main still reports file-read plus network-send as a generic plugin code-safety warning, and it has operator suppressions but no plugin-declared egress-intent/rationale contract.

## What This Changes

The issue requests a plugin manifest capability/rationale field for intentional data egress so security audit can annotate, downgrade, or suppress potential-exfiltration findings.

## Best Possible Solution

Add a narrow, documented manifest egress-intent/rationale contract that annotates matching findings and only downgrades or suppresses through explicit operator-controlled acceptance.

## Reproduction Assessment

Not applicable as a feature request. Source inspection confirms current main has the warning and manual suppressions but lacks a plugin manifest egress-intent contract.

## Solution Assessment

Unclear. Plugin-declared intent is a plausible direction, but automatic suppression is not proven best; annotation plus explicit operator acknowledgement is the safer maintainable shape.
