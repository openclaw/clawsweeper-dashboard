## Summary

Keep open: current main and v2026.6.8 still have the generic `potential-exfiltration` plugin code-safety warning, and neither the plugin manifest contract nor audit path implements egress intent, rationale, downgrade, or operator acknowledgement semantics.

## What This Changes

The issue requests a plugin manifest egress-intent/rationale field that security audit can use to annotate, downgrade, or suppress legitimate potential-exfiltration findings.

## Best Possible Solution

Keep this open for a maintainer-approved security design: define an additive manifest egress-intent/rationale contract that annotates findings and only downgrades or suppresses after explicit operator-owned acknowledgement.

## Reproduction Assessment

Yes, source-level: current main still emits `potential-exfiltration` and aggregates it into a generic `plugins.code_safety` warning, while the plugin manifest contract lacks the proposed egress-intent/rationale fields.
