## Summary

Keep open. Current main still has the warn-level plugin `potential-exfiltration` scanner/audit path and generic operator suppressions, but no plugin manifest egress-intent/rationale contract or audit policy for annotating, downgrading, or capability-aware suppression; this is a new security-audit/product surface, not a broken existing behavior.

## What This Changes

Request to add plugin manifest egress-intent capabilities and rationale so legitimate backup/sync/export plugins can reduce false-positive `potential-exfiltration` audit noise.

## Best Possible Solution

Define a documented manifest/audit contract for declared egress intent and rationale, keep the finding visible by default, and only suppress or downgrade through explicit operator-controlled semantics that align with the broader audit-suppression policy.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken current behavior report. Source inspection confirms the current warn path exists and the requested manifest/rationale contract is absent on current main and in `v2026.5.12`.

## Solution Assessment

Unclear: the declaration-and-rationale direction is plausible, but maintainers need to choose whether audit should annotate, downgrade, or suppress only after explicit operator acknowledgement. The narrowest maintainable direction is a small public contract tied to existing `security.audit.suppressions` semantics rather than an unconditional plugin-controlled suppression.
