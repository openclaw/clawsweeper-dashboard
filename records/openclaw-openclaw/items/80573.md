## Summary

Keep this open: current main still emits the file-read-plus-network-send warning as a generic plugin code-safety finding, and the existing operator-level suppression mechanism is not the requested plugin-declared egress-intent contract.

## What This Changes

The issue requests a plugin-declared capability and rationale manifest field so security audit can distinguish intentional file-to-network export behavior from unexpected potential exfiltration.

## Best Possible Solution

Add a narrow documented plugin manifest egress-intent contract that annotates potential-exfiltration findings and only downgrades or suppresses through explicit operator acknowledgement, while retaining security.audit.suppressions as the manual escape hatch.

## Reproduction Assessment

Not applicable as a feature request. Source review confirms current main has the warning and global suppressions but no plugin manifest egress-intent contract.

## Solution Assessment

Unclear. Plugin-declared intent is a plausible fit, but automatic suppression is not proven best; the safer shape is manifest rationale plus operator-visible annotation or explicit acknowledgement.
