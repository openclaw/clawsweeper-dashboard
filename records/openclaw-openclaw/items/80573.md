## Summary

Keep open. Current main still has the warn-level `potential-exfiltration` scanner/audit path and no manifest or operator-acknowledgement contract for legitimate plugin egress intent, but this is a new security-audit policy and public manifest surface rather than a broken existing behavior.

## What This Changes

The issue requests a plugin manifest capability/rationale field so `openclaw security audit --deep` can distinguish intended backup/sync/export egress from suspicious exfiltration warnings.

## Best Possible Solution

Define a small, documented manifest/audit contract for declared egress intent and rationale, while keeping findings visible by default and requiring explicit operator semantics before any suppression.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken current behavior report. Source inspection confirms the current warn path exists and the requested manifest/acknowledgement contract is absent.

## Solution Assessment

Unclear: the declaration-and-rationale direction is plausible, but maintainers need to choose whether audit should annotate, downgrade, or suppress only after explicit operator acknowledgement.
