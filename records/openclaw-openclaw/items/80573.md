## Summary

Keep open. Current main still emits warn-level `potential-exfiltration` plugin code-safety findings and has no plugin manifest egress-intent/rationale contract, but the requested behavior is a new security-audit policy surface rather than a broken existing contract.

## What This Changes

The issue requests a plugin manifest capability/rationale field and audit handling for legitimate plugin egress so `potential-exfiltration` findings can be annotated, downgraded, or explicitly acknowledged.

## Best Possible Solution

Define a documented plugin egress-intent and rationale contract that annotates findings by default and only downgrades or suppresses them under explicit operator-controlled acknowledgement.
