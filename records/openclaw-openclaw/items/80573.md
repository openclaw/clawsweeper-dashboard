## Summary

Keep open: current main still treats file-read plus network-send as a generic plugin code-safety warning, and there is no plugin-declared egress intent or rationale contract to distinguish intended export plugins from suspicious code.

## What This Changes

Add a plugin manifest egress-intent capability/rationale contract so `openclaw security audit --deep` can annotate, downgrade, or suppress known intentional data-export findings.

## Best Possible Solution

Define a narrow, documented plugin manifest egress-intent/rationale contract that annotates matching plugin code-safety findings and only downgrades or suppresses through explicit operator-controlled acceptance.

## Reproduction Assessment

Not applicable as a feature request. Source inspection confirms current main has the warning and manual suppressions but lacks a plugin manifest egress-intent contract.

## Solution Assessment

Unclear. Plugin-declared intent is plausible, but automatic suppression is not proven best; annotation plus explicit operator acknowledgement is the safer maintainable shape.
