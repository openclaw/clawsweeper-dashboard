## Summary

The issue should stay open: current main still emits generic `plugins.code_safety` warnings for the `potential-exfiltration` rule, while the plugin manifest contract has no egress-intent/rationale field, so this is a real feature request that needs security-audit policy design rather than cleanup closure.

## What This Changes

Request a plugin manifest capability/rationale field that lets `openclaw security audit --deep` distinguish intentional remote state export from suspicious exfiltration-like code.

## Best Possible Solution

Design a small, documented manifest/audit contract that exposes declared egress intent and rationale while keeping suspicious code visible unless maintainers explicitly choose an acknowledgement flow.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing-behavior bug. Source inspection confirms the current warning path exists and lacks a plugin manifest egress-intent contract.

## Solution Assessment

Unclear: the requested direction is plausible, but maintainers need to choose whether declarations annotate, downgrade, or suppress findings and whether any operator acknowledgement belongs in config.
