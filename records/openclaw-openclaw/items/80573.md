## Summary

Keep open. Current main still emits warn-level plugin exfiltration findings and has no manifest-driven egress intent/rationale contract, but this is a new security-audit policy and public plugin manifest surface rather than a narrow bug fix.

## What This Changes

The request asks for a plugin manifest egress-intent capability and rationale that can annotate, downgrade, or suppress `potential-exfiltration` audit warnings.

## Best Possible Solution

Define a small versioned manifest and audit contract that records declared egress intent and rationale while preserving operator visibility by default, then document the manifest field, audit behavior, and any explicit acknowledgement path.

## Reproduction Assessment

Not applicable: this is a feature request, not broken existing behavior. Source inspection confirms the current warning path exists and lacks a manifest-based egress intent contract.

## Solution Assessment

Unclear: the declaration-and-rationale direction is plausible, but self-declaration alone is not the safest trust model. Maintainers should decide whether audit only annotates, downgrades, or suppresses after explicit operator acknowledgement.
