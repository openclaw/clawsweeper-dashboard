## Summary

Keep open. Current main still has the warn-level plugin code-safety path and no manifest/audit contract for declared egress intent, and implementing one would add a security-sensitive public policy surface.

## What This Changes

The issue requests plugin manifest fields for declared egress capability and rationale so audit can reduce false-positive potential-exfiltration noise for backup, sync, and export plugins.

## Best Possible Solution

Define a small versioned manifest/audit contract for declared egress intent and rationale, keep findings visible by default, and document any operator acknowledgement path explicitly.

## Solution Assessment

Unclear: the declaration-and-rationale direction is plausible, but maintainers need to decide whether audit should annotate, downgrade, or suppress only after explicit operator acknowledgement.
