## Risks / Open Questions

- Changing LAN/custom binding from IPv4-only to IPv6 or dual-stack can expose Gateway on additional non-loopback interfaces, including globally routable IPv6 addresses.
- A direct `lan` to `::` change is platform-sensitive because dual-stack behavior can differ across Linux sysctl settings, Windows, containers, and firewalls.
- Maintainers still need to choose whether existing `lan` changes semantics, `customBindHost` accepts IPv6 literals, or a distinct explicit dual-stack/IPv6 mode is required.
