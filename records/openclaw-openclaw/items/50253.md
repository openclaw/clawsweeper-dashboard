## Summary

Keep open: current main and the latest release still encode IPv4-only Gateway LAN/custom bind behavior, and related reports plus closed PR attempts show this is the canonical unresolved Gateway networking contract rather than cleanup.

## What This Changes

Support IPv6 or dual-stack listening for OpenClaw Gateway LAN/custom bind modes.

## Best Possible Solution

Define a core Gateway IPv6/dual-stack bind contract, then update bind resolution, listener expansion, custom host validation, status/control UI links, security/config docs, and focused regression plus live socket proof together.

## Reproduction Assessment

Yes, source-reproducible: current main resolves `gateway.bind=lan` to `0.0.0.0`, passes non-loopback hosts through without adding `::`, and rejects IPv6 `customBindHost` values. I did not run live sockets because this review is read-only.

## Solution Assessment

Unclear. Switching LAN to `::` may be right on some platforms, but the maintainable fix needs an explicit cross-platform Gateway bind contract for LAN, custom IPv6, fallback behavior, and exposure/security documentation.
