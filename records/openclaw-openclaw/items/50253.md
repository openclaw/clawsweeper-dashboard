## What This Changes

Request IPv6 and dual-stack Gateway listening for LAN/custom bind mode.

## Best Possible Solution

Define the public Gateway IPv6/dual-stack bind contract, then update bind resolution, listener expansion, custom host validation, status/control UI links, security/config docs, and socket proof together.

## Reproduction Assessment

Yes. Source inspection shows current main resolves gateway.bind=lan to 0.0.0.0, does not add :: for non-loopback listeners, and rejects IPv6 customBindHost values; no live socket run was performed in this read-only review.

## Solution Assessment

Unclear. Directly switching LAN to :: may solve IPv6-only deployments, but the safer maintainable solution needs an explicit cross-platform dual-stack policy and exposure documentation.
