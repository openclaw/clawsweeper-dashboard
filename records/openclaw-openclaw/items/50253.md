## What This Changes

Request IPv6 or dual-stack support for Gateway LAN/custom bind mode so IPv6-only clients can connect.

## Best Possible Solution

Define the public Gateway IPv6/dual-stack bind contract, then update bind resolution, listener expansion, custom host validation, status/control UI links, security/config docs, and socket proof together.

## Solution Assessment

Unclear. Switching LAN directly to :: may solve some IPv6-only deployments, but the safer maintainable path needs an explicit cross-platform dual-stack/IPv6 bind policy and exposure documentation.
