## Summary

Keep open: current `main` and `v2026.5.22` still implement and document Gateway LAN/custom BYOH bind as IPv4-only, while related duplicate reports and the docs-only workaround PR are closed. The request is valid, but enabling non-loopback IPv6 listening needs an explicit exposure and cross-platform dual-stack policy before implementation.

## What This Changes

Request Gateway bind modes, especially `lan` and custom bind, to support IPv6 or dual-stack listening instead of IPv4-only binding.

## Best Possible Solution

Define the public Gateway IPv6 and dual-stack bind contract, then update bind resolution, listener expansion, custom host validation, status/control UI links, security docs, and cross-platform socket proof together.

## Reproduction Assessment

Yes. Source inspection shows current `main` resolves `gateway.bind=lan` to `0.0.0.0`, does not add `::` for non-loopback listeners, and rejects IPv6 `customBindHost` values; no live socket run was performed in this read-only review.

## Solution Assessment

Unclear. Directly switching LAN to `::` could help IPv6-only deployments, but the maintainable path needs an explicit cross-platform dual-stack and exposure policy first.
