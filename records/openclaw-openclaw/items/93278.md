## Label Justifications

- `P2`: This is a bounded but security-sensitive Skill Workshop lifecycle improvement with real workflow impact and no emergency runtime outage.
- `impact:security`: The issue asks OpenClaw to trust chat-originated evidence for mutating generated skill proposals, which is an authorization boundary.
- `impact:session-state`: The reported gap leaves owner-decided proposals stuck in pending lifecycle state instead of recording rejected or quarantined state.

## Summary

Keep open: current main and v2026.6.6 still require the generic plugin approval route for Skill Workshop `apply`, `reject`, and `quarantine`, and there is still no first-class chat-originated owner approval evidence or redacted audit contract for those lifecycle actions. The related turn-source PR at https://github.com/openclaw/openclaw/pull/89590 may address part of the delivery route, but it is open/unmerged and does not by itself settle the symmetric Skill Workshop approval-evidence design.

## What This Changes

Request a documented or built-in owner-approval path so Skill Workshop `apply`, `reject`, and `quarantine` can safely consume verified chat-originated approval evidence.

## Best Possible Solution

Define a generic, redacted approval-evidence or approval-route contract, then wire Skill Workshop `apply`, `reject`, and `quarantine` through it consistently while preserving fail-closed pending approval by default.

## Reproduction Assessment

Yes. Source inspection shows pending Skill Workshop lifecycle calls return `requireApproval`, a no-route plugin approval returns `decision: null`, and the before-tool-call path reports `Plugin approval unavailable (no approval route)` before the proposal service can transition state.

## Solution Assessment

Unclear as requested. Symmetric approval evidence is the right product direction, but maintainers need to choose whether the durable contract belongs in generic plugin approval routing, Skill Workshop-specific policy, documentation for existing route registration, or a combination.
