## Summary

Keep open: current main and v2026.6.8 have symmetric generic plugin-approval gating and route-registration docs, but they still do not provide the requested first-class audited owner-approval evidence contract for Skill Workshop apply, reject, and quarantine.

## What This Changes

The issue requests a first-class or documented symmetric owner-approval path for Skill Workshop apply, reject, and quarantine lifecycle actions.

## Best Possible Solution

Keep the fail-closed default, then have maintainers choose whether the documented generic plugin-approval route is the official answer or whether Skill Workshop needs a bounded audited approval-evidence contract for all lifecycle actions.

## Reproduction Assessment

Yes, at source level. Current main still routes apply, reject, and quarantine through generic plugin approval and has no proposal approval-evidence fields or chat approval policy surface; I did not run a live chat workflow in this read-only review.

## Solution Assessment

Unclear until maintainers make the product/security decision. The generic approvals.plugin route is a real partial solution, but the requested first-class audited owner-evidence path needs an explicit authority and audit model before implementation.
