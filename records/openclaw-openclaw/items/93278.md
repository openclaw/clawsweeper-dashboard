## Summary

Keep open: current main and v2026.6.6 still have the reported Skill Workshop approval gap. Lifecycle actions are approval-gated, but the agent-side request does not carry a symmetric chat-originated owner-evidence/audit contract for apply, reject, and quarantine, and the related routing PR is open and unmerged.

## What This Changes

The issue asks for a first-class Skill Workshop owner-approval path that works symmetrically for apply, reject, and quarantine lifecycle transitions from chat-originated workflows.

## Best Possible Solution

Define one canonical lifecycle approval contract that binds verified owner evidence to pending Skill Workshop proposals, records redacted audit metadata, and applies the same authority model to apply, reject, and quarantine while preserving fail-closed behavior without evidence.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes at source level: pending Skill Workshop lifecycle actions require a plugin approval route, no-route approvals return the reported failure, and the current proposal/action models have no chat owner-evidence fields.

## Solution Assessment

Is this the best way to solve the issue? Unclear: the symmetry requirement is sound, but maintainers need to choose whether the permanent contract belongs in generic plugin approval routing, Skill Workshop-specific policy, documentation, or some combination of those.
