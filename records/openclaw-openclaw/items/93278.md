## Summary

Keep open: current main still has the reported Skill Workshop approval gap. The lifecycle gate covers apply, reject, and quarantine, but the agent-side approval request does not carry the requested chat-originated owner evidence or audit model; related open PRs may address generic turn-source routing, not the full symmetric lifecycle contract.

## What This Changes

Request a first-class or documented owner-approval path for Skill Workshop apply, reject, and quarantine when chat-originated workflows lack a generic plugin approval route.

## Best Possible Solution

Define a generic, redacted approval-evidence contract or explicit route-registration guidance, then apply it consistently to Skill Workshop apply, reject, and quarantine without weakening the pending approval default.

## Reproduction Assessment

Yes. Source inspection shows pending Skill Workshop lifecycle calls return requireApproval, a no-route plugin approval returns decision null, and the before-tool-call path reports Plugin approval unavailable while the proposal remains pending.

## Solution Assessment

Unclear as requested. The direction is plausible, but maintainers need to choose between documenting/configuring existing plugin approval routes and adding a built-in approval-evidence/audit contract.
