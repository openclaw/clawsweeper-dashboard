## Summary

Keep open: current main has the generic plugin-approval turn-source route, but the requested Skill Workshop approval-evidence and audit contract for all lifecycle actions is still not implemented.

## What This Changes

Request a first-class, symmetric owner-approval path for Skill Workshop apply, reject, and quarantine lifecycle actions from chat-originated workflows.

## Best Possible Solution

Define or explicitly reject a generic redacted approval-evidence contract, then document or implement it consistently for Skill Workshop apply, reject, and quarantine without weakening the default pending approval gate.

## Reproduction Assessment

Yes, at source level. Current main still has no persistent approval-evidence fields on Skill Workshop proposal records, and no-route plugin approval still fails closed when there is no approval client or turn-source route.

## Solution Assessment

Unclear as an implementation path. The merged turn-source route is a useful partial fix, but the best solution still needs maintainer product and security direction on whether approval evidence belongs in generic approval plumbing, Skill Workshop records, docs, or some combination.
