## Summary

Keep open: current main and v2026.6.6 still lack the requested symmetric owner-approval evidence path for Skill Workshop lifecycle actions. The existing code gates apply, reject, and quarantine through plugin approval and fails closed when no route is available, but it does not document or implement verified chat-originated approval evidence or audit metadata for all three actions.

## What This Changes

The issue requests a documented or built-in owner-approval path for Skill Workshop apply, reject, and quarantine from chat-originated workflows.

## Best Possible Solution

Define a generic, redacted approval-evidence or route-registration contract, then wire Skill Workshop apply, reject, and quarantine through it while preserving pending approval fail-closed behavior.

## Reproduction Assessment

Yes at source level, but not with a live chat workflow in this read-only review. Current main gates lifecycle actions through plugin approval, maps a no-route decision to the reported error, and proposal records have no owner-approval evidence field.

## Solution Assessment

Unclear without maintainer decision. Symmetric owner approval is the right end state, but maintainers need to choose route-registration docs versus a built-in approval-evidence and audit contract.
