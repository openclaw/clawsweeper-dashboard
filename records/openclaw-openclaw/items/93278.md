## Summary

Keep open: current main has symmetric generic plugin approval prompts and route documentation, but it still lacks the requested first-class, audited owner-approval evidence contract for Skill Workshop apply, reject, and quarantine.

## What This Changes

Skill Workshop should provide or document a symmetric owner-approval path for apply, reject, and quarantine lifecycle actions without requiring action-specific ad hoc bridges.

## Best Possible Solution

Keep the fail-closed default, then either bless and document the generic approvals.plugin route as the official answer or design one bounded, audited owner-evidence contract that applies to all Skill Workshop lifecycle actions.

## Reproduction Assessment

Yes, at source level. Current main can reproduce the generic no-route failure path and shows no Skill Workshop approval-evidence fields or chatApproval policy surface; I did not run a live chat approval session in this read-only review.

## Solution Assessment

Unclear as product direction. Generic approvals.plugin routing is a real partial solution, but a first-class audited owner-evidence path needs explicit maintainer security/product approval before implementation.
