## Summary

Keep open: current main still routes Skill Workshop apply, reject, and quarantine through the generic plugin approval prompt, and there is no first-class owner-approval evidence path or documentation that makes chat-originated decisions symmetric across all three lifecycle actions.

## What This Changes

The issue requests a symmetric owner-approval path for Skill Workshop apply, reject, and quarantine so verified chat-originated owner decisions can complete proposal lifecycle actions safely.

## Best Possible Solution

Define or document one approval-evidence contract that binds proposal id, lifecycle action, approving principal, originating conversation, replay window, and redacted audit metadata, then apply it consistently to Skill Workshop apply, reject, and quarantine.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: pending lifecycle calls require generic plugin approval, and the gateway/no-route branch returns a null decision that the tool wrapper reports as Plugin approval unavailable. I did not run a live chat/Gateway reproduction in this read-only review.

## Solution Assessment

Unclear as requested: a symmetric owner-approval path is the right product direction to evaluate, but the permanent design needs maintainer security and UX decisions before a safe implementation can be chosen.
