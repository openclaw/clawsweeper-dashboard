## Summary

Keep open: current main still lacks the requested symmetric Skill Workshop owner-approval and audit path. The lifecycle gate covers apply, reject, and quarantine, but lifecycle mutations do not consume or store verified chat-originated approval evidence; the related open turn-source PR only addresses route delivery, not the full approval-evidence contract.

## What This Changes

The issue requests symmetric owner-approved Skill Workshop lifecycle handling for apply, reject, and quarantine actions.

## Best Possible Solution

Design a generic, auditable approval-evidence contract bound to proposal, lifecycle action, trusted turn source, and authorized principal, then wire Skill Workshop apply/reject/quarantine to consume it while preserving fail-closed direct mutation.

## Reproduction Assessment

Yes at source level: current main requires approval for lifecycle actions, fails no-route approval closed, and has no action input or record fields for approval evidence; no live chat reproduction was run.

## Solution Assessment

Unclear as an immediate implementation path: generic plugin approval routing may help delivery, but maintainers still need to choose the approval authority, audit metadata, and privacy model.
