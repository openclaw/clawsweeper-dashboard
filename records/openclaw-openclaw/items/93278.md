## Summary

Keep open: current main still lacks a first-class symmetric owner-approval path for Skill Workshop lifecycle actions. The existing code gates apply, reject, and quarantine and fails closed when no approval route exists, while docs only describe the broad auto bypass and proposal records do not store chat approval evidence.

## What This Changes

The issue requests a symmetric, first-class owner-approval path for Skill Workshop apply, reject, and quarantine in chat-originated workflows.

## Best Possible Solution

Define one redacted approval-evidence or route-registration contract, then apply it consistently to Skill Workshop apply, reject, and quarantine while preserving fail-closed behavior when no verified route exists.

## Reproduction Assessment

Yes. Source inspection shows pending Skill Workshop lifecycle calls return requireApproval, a no-route plugin approval returns decision null, and the before-tool-call path reports Plugin approval unavailable while the proposal remains pending.

## Solution Assessment

Unclear as requested. The direction is plausible, but maintainers need to choose between documenting/configuring existing plugin approval routes and adding a built-in approval-evidence and audit contract.
