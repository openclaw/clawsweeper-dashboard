## Summary

Keep open: current main and v2026.6.8 have symmetric generic plugin approval routing for Skill Workshop lifecycle actions, but they still do not provide the requested first-class audited owner-approval evidence contract for apply, reject, and quarantine.

## What This Changes

Requests a documented or built-in symmetric owner-approval path for Skill Workshop apply, reject, and quarantine actions from chat-originated workflows.

## Best Possible Solution

Keep fail-closed defaults, then have maintainers choose whether the generic plugin approval route is sufficient or whether Skill Workshop should get a bounded audited approval-evidence contract for all lifecycle actions.

## Reproduction Assessment

Yes, at source level. Current main and v2026.6.8 show generic approval routing and the reported no-route failure path, while lacking approval-evidence fields or a chatApproval policy surface.

## Solution Assessment

Unclear until maintainers choose the product and security model. The generic approvals.plugin route is a real partial solution, but consuming prior chat approval evidence as durable Skill Workshop authority needs an explicit contract.
