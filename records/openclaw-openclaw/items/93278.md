## Summary

Keep open: current main still lacks the requested symmetric owner-approval evidence path for Skill Workshop lifecycle actions. The code gates `apply`, `reject`, and `quarantine`, but the agent approval request does not carry chat-originated approval evidence, and related open PRs only address lower-level routing.

## What This Changes

The issue requests a first-class symmetric owner-approval path for Skill Workshop `apply`, `reject`, and `quarantine` lifecycle actions from chat-originated workflows.

## Best Possible Solution

Define a generic, redacted approval-evidence contract or documented route-registration path, then apply it consistently to Skill Workshop `apply`, `reject`, and `quarantine` without weakening the pending approval default.

## Reproduction Assessment

Yes, at source level. Current main gates the lifecycle actions, sends a plugin approval request without the requested chat-originated evidence fields, and returns the reported no-route error when no approval route is available.

## Solution Assessment

Unclear as a final implementation path. The requested symmetry is maintainable, but maintainers must choose whether the solution is documented route registration, generic approval-evidence plumbing, or a Skill Workshop-specific policy surface.
