## Summary

Keep open: current main and v2026.6.9 have the generic plugin-approval route and symmetric fail-closed lifecycle gate, but they still do not provide the requested first-class owner-approval evidence and audit contract for Skill Workshop apply, reject, and quarantine.

## What This Changes

The issue requests a documented or built-in symmetric owner-approval path for Skill Workshop lifecycle actions, including verified chat-originated approval evidence and redacted audit metadata.

## Best Possible Solution

Keep the fail-closed default, then have maintainers choose whether the documented generic plugin-approval route is the official answer or whether Skill Workshop needs a bounded audited owner-evidence contract for all lifecycle actions.

## Reproduction Assessment

Yes, at source level. Current main can reproduce the generic no-route failure path and shows no proposal approval-evidence fields or chatApproval policy surface; I did not run a live chat approval session in this read-only review.

## Solution Assessment

Unclear as a product direction. The generic approvals.plugin route is a real partial solution, but the requested first-class audited owner-evidence path still needs a maintainer security/product decision before implementation.
