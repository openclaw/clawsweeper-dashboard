## Summary

Keep open: current main still has only the generic fail-closed plugin approval path for Skill Workshop lifecycle actions. The docs cover approvals.plugin routing and same-chat /approve, but they do not add the first-class audited owner-approval evidence contract requested here.

## What This Changes

The issue asks OpenClaw to document or add a symmetric owner-approval path for Skill Workshop apply, reject, and quarantine lifecycle actions so chat-originated owner decisions can be verified and audited without ad hoc bridges.

## Best Possible Solution

Keep the fail-closed default, then have maintainers choose whether documented generic approvals.plugin routing is the official product answer or whether Skill Workshop needs a bounded audited approval-evidence contract for all lifecycle actions.

## Reproduction Assessment

Yes, at source level. Current main can reproduce the generic no-route failure path and shows no proposal approval-evidence fields or chatApproval policy surface; I did not run a live chat approval session in this read-only review.

## Solution Assessment

Unclear as a product direction. Generic approvals.plugin routing is a maintainable partial solution, but a first-class chat-originated owner-evidence path needs a maintainer security/product decision before implementation.
