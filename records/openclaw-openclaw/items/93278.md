## Summary

Keep open: current main still lacks the requested symmetric, audited owner-approval path for Skill Workshop lifecycle actions; it fails closed safely when no route exists, but that leaves decided proposals pending without a first-class evidence/audit model.

## What This Changes

Request a documented or built-in approval route that covers Skill Workshop apply, reject, and quarantine with verified owner evidence and audit metadata.

## Best Possible Solution

Define a generic, redacted approval-evidence or route-registration contract, then apply it consistently to Skill Workshop apply, reject, and quarantine without weakening the fail-closed default.

## Reproduction Assessment

Yes. Source inspection shows pending Skill Workshop lifecycle calls require approval, no-route plugin approval returns a null decision, and the before-tool-call path reports the exact no-approval-route failure.

## Solution Assessment

Unclear. The requested direction is valid, but maintainers need to choose whether the durable fix is documented route registration, generic turn-source forwarding, or a built-in redacted approval-evidence/audit contract.
