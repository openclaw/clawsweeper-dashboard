## Summary

Keep open: current main and v2026.6.6 still lack a first-class owner-approval evidence and audit path for Skill Workshop lifecycle actions. The existing gate is symmetric for apply/reject/quarantine and fails closed on no route, but the agent-side approval request does not carry chat-originated owner evidence, and the open turn-source PR only addresses part of the routing problem.

## What This Changes

The issue requests a symmetric, first-class approval path for Skill Workshop apply, reject, and quarantine actions that can use verified owner approval evidence without weakening fail-closed defaults.

## Best Possible Solution

Define a generic redacted approval-evidence contract or explicit route-registration guidance, then apply it consistently to Skill Workshop apply, reject, and quarantine while preserving the pending fail-closed default.

## Reproduction Assessment

Yes, at source level: pending Skill Workshop lifecycle calls require plugin approval, no-route plugin approvals return decision null, and the before-tool-call path reports Plugin approval unavailable while the proposal remains pending. I did not run a live chat-originated workflow in this read-only review.

## Solution Assessment

Unclear as an implementation path: the direction is valid, but maintainers need to choose between explicit route-registration documentation and a built-in approval-evidence/audit contract. The best solution should avoid duplicating the open turn-source forwarding work and must not rely on approvalPolicy:auto.
