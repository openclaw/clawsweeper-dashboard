## Summary

Keep open: current main now has the generic same-chat/plugin approval routing needed for Skill Workshop lifecycle prompts, but the requested symmetric owner-approval evidence and durable audit contract for apply, reject, and quarantine is still not implemented.

## What This Changes

The issue asks for a first-class, symmetric owner-approval path for Skill Workshop apply, reject, and quarantine lifecycle actions, including verified chat-originated approval evidence and audit metadata.

## Best Possible Solution

Reuse the current generic plugin approval route, then add or document a narrow Skill Workshop lifecycle approval-evidence/audit contract that covers apply, reject, and quarantine without weakening the pending approval default.

## Reproduction Assessment

Yes, at source level. Current main has generic plugin approval routing, but SkillProposalRecord and the apply/reject/quarantine service paths still have no durable approval-evidence or approving-principal metadata.

## Solution Assessment

Unclear until maintainers choose the product/security contract. The generic turn-source approval route is the right prerequisite, but the remaining work should be an explicit evidence/audit design rather than an ad hoc Skill Workshop bypass.
