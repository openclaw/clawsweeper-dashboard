## Summary

Keep open: current main and v2026.6.6 still gate Skill Workshop apply, reject, and quarantine through generic plugin approval and fail closed when no route exists; there is no symmetric owner-evidence/audit contract for chat-originated lifecycle decisions, and the related open PRs cover lower-level routing or timeout symptoms rather than the full lifecycle design.

## What This Changes

The issue requests a first-class owner-approved Skill Workshop lifecycle path for apply, reject, and quarantine actions.

## Best Possible Solution

Define a generic or Skill Workshop-owned approval-evidence contract with redacted audit metadata, then apply it consistently to apply, reject, and quarantine while preserving the pending fail-closed default.

## Reproduction Assessment

Yes, at source level. Pending Skill Workshop lifecycle actions request plugin approval, the no-route path returns the exact reported failure, and proposal records cannot store the requested owner approval evidence.

## Solution Assessment

Unclear as an implementation path. The direction is valid, but maintainers need to choose whether the durable contract belongs in generic plugin approval routing, Skill Workshop policy/storage, documentation for route registration, or some combination.
