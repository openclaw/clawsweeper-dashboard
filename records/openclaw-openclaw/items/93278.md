## Summary

Keep open: current main and v2026.6.8 still route Skill Workshop apply, reject, and quarantine through a generic approval prompt, while proposal/action records lack verified owner-approval evidence and audit fields. The open related PR at https://github.com/openclaw/openclaw/pull/89590 covers turn-source delivery plumbing, not the full symmetric lifecycle approval contract.

## What This Changes

The issue requests a first-class, symmetric owner-approval path for Skill Workshop apply, reject, and quarantine lifecycle actions.

## Best Possible Solution

Define a security-reviewed approval-evidence contract for Skill Workshop lifecycle actions, reuse generic approval routing where possible, and keep direct lifecycle mutation fail-closed without verified owner evidence.

## Reproduction Assessment

Yes, at source level: current main has a generic lifecycle approval gate and proposal state transitions, but no owner-evidence fields or action input path for chat-originated approval. I did not run a live chat reproduction in this read-only review.

## Solution Assessment

Unclear as an implementation proposal: the desired end state is valid, but maintainers need to choose whether the contract belongs in generic plugin approval routing, Skill Workshop policy, docs for deployment approval routes, or a combination.
