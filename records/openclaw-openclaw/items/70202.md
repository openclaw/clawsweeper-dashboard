## Summary

Keep open. Current main still lacks the requested first-run multi-model onboarding flow, and the same-author follow-ups at https://github.com/openclaw/openclaw/issues/81960 and https://github.com/openclaw/openclaw/issues/81961 are open without a maintainer consolidation decision.

## What This Changes

Request to let onboarding configure multiple models/providers and choose one default so users can balance cost, latency, and capability.

## Best Possible Solution

Define a scoped interactive onboarding step that reuses the existing model allowlist, fallback, and primary-model contracts, while tracking Dashboard management and automatic routing separately.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection on current main gives high confidence that classic local onboarding exits after one provider/default-model selection.

## Solution Assessment

Unclear until maintainers scope the UX. The narrow maintainable direction is to reuse promptModelAllowlist, applyModelAllowlist, applyModelFallbacksFromSelection, and existing AgentDefaultsConfig fields rather than add a parallel config contract or automatic routing here.
