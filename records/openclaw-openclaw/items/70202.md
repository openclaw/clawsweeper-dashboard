## Summary

Keep open. Current main already has model allowlists, fallback chains, and configure-time multi-select, but first-run onboarding still only prompts for one default model and there is no open canonical PR or issue that owns the onboarding-specific gap.

## What This Changes

Request first-run onboarding support for selecting multiple available models plus one default model to reduce cost and model-switching friction.

## Best Possible Solution

Add a first-run onboarding step that reuses the existing allowlist and fallback helpers to seed agents.defaults.models and agents.defaults.model.fallbacks around the chosen primary; keep automatic routing separate.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. The source check is high-confidence: first-run onboarding calls promptDefaultModel/applyPrimaryModel and has no promptModelAllowlist path.

## Solution Assessment

Unclear until maintainers choose the onboarding UX. The narrow maintainable path is to reuse promptModelAllowlist, applyModelAllowlist, and applyModelFallbacksFromSelection rather than adding new config or automatic routing here.
