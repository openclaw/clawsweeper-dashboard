## Summary

Keep open. Current main still has a single-default first-run onboarding path, while multi-model allowlist/fallback selection exists in configure/model-picker surfaces; the newer same-author follow-up at https://github.com/openclaw/openclaw/issues/81960 may help scoping, but it is open and there is no maintainer signal to close this issue as consolidated.

## What This Changes

The issue requests first-run onboarding support for selecting multiple models while choosing one default model to reduce cost and switching friction.

## Best Possible Solution

Define a scoped optional onboarding step that reuses the existing model allowlist and fallback helpers to seed `agents.defaults.models` and `agents.defaults.model` around the chosen primary, while tracking Dashboard switching and automatic routing separately.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection gives a high-confidence current-main gap because setup still uses the single default-model picker while configure owns the multi-select allowlist path.

## Solution Assessment

Unclear until maintainers choose the onboarding UX and policy. The narrow maintainable direction is to reuse `promptModelAllowlist`, `applyModelAllowlist`, and `applyModelFallbacksFromSelection` rather than add automatic routing or a parallel config contract here.
