## Summary

Keep open. Current main already has configure-time multi-model selection, fallbacks, and chat/session switching, but the first-run onboarding wizard still records a single primary/default model and the requested onboarding UX needs maintainer judgment.

## What This Changes

The issue requests first-run onboarding support for selecting multiple LLM models while choosing one default to reduce model cost and switching friction.

## Best Possible Solution

Add a first-run onboarding step that reuses the existing allowlist and fallback helpers to seed `agents.defaults.models` and `agents.defaults.model.fallbacks` around the chosen primary model, while leaving automatic routing as a separate feature.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. The source check is high-confidence: the setup wizard still uses `promptDefaultModel`/`applyPrimaryModel`, while the multi-select allowlist path lives in configure/model-picker code.

## Solution Assessment

Unclear until maintainers choose the onboarding UX. The narrow maintainable path is to reuse `promptModelAllowlist`, `applyModelAllowlist`, and `applyModelFallbacksFromSelection` rather than add a new config contract or automatic routing here.
