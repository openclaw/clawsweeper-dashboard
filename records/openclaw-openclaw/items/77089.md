## Summary

Keep open: current main still has the reported session_expired-only retry gate, and the provided GitHub context shows open unmerged PR #77141 with closing syntax for the same bug.

## What This Changes

The issue asks OpenClaw to clear a persisted CLI session binding and retry fresh for any resumed-session FailoverError, not only session_expired.

## Best Possible Solution

Review and land #77141 or an equivalent narrow fix that broadens stale-binding recovery to any FailoverError from an active reused CLI binding while preserving the no-extra-retry behavior for fresh-session failures.

## Reproduction Assessment

Yes by source inspection: a resumed CLI attempt that throws FailoverError with reason timeout or unknown while a stored binding exists misses the session_expired-only branch and rethrows. I did not run tests because this review was required to keep the checkout read-only.

## Solution Assessment

Yes: reusing the existing clearCliSessionInStore plus one fresh retry is the narrowest maintainable fix, provided the guard stays limited to an active persisted binding and regression tests cover timeout and unknown.
