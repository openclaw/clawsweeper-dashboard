## Summary

Keep open: current main still gates stale CLI session clearing and fresh retry on `session_expired`, while real CLI timeout and unknown failures still fall through from a resumed binding; open PR #77141 is the active closing implementation candidate.

## What This Changes

The issue requests broadening CLI stale-session recovery so any `FailoverError` from a reused session clears the binding and retries once fresh before model fallback.

## Best Possible Solution

Review and land or revise #77141 so timeout and unknown resumed-session failures clear the persisted binding, retry once fresh, and add focused regression coverage without changing fresh-session failure behavior.

## Reproduction Assessment

Yes, by source inspection rather than live execution. A resumed CLI run with an active binding that throws `FailoverError` reason `timeout` or `unknown` bypasses the current `session_expired` recovery branch and rethrows.

## Solution Assessment

Yes. Broadening the existing stale-binding recovery path for active reused CLI sessions is the narrowest maintainable fix, but the issue should stay open while #77141 is reviewed, merged, or replaced.
