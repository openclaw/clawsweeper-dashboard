## Summary

Keep open: current main still gates stale CLI session clear-and-fresh-retry recovery on `session_expired`, and open unmerged PR #77141 explicitly tracks the requested fix with closing syntax.

## What This Changes

Retry a stale persisted CLI backend session with a fresh session for any resumed-session `FailoverError`, not only `session_expired`.

## Best Possible Solution

Review and land PR #77141, or a narrow replacement, with timeout and unknown regression coverage for the existing clear-and-retry-fresh path.

## Reproduction Assessment

Yes by source inspection: a resumed CLI binding that throws `FailoverError` with reason `timeout` or `unknown` reaches the catch block but fails the `session_expired` guard and is rethrown. I did not run tests because this review is read-only.

## Solution Assessment

Yes: broadening the existing stale-binding recovery branch while keeping the active persisted-binding guard and one fresh retry is the narrowest maintainable fix. The linked PR #77141 is the implementation candidate to review or replace.
