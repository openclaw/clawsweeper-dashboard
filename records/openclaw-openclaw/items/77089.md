## Summary

Keep open: current main still has `session_expired`-only recovery branches for resumed CLI `FailoverError` handling, and #77141 is an open, unmerged PR with closing syntax for this issue.

## What This Changes

The issue asks the CLI backend to clear a stale resumed session binding and retry once fresh after any `FailoverError`, not only `session_expired`.

## Best Possible Solution

Land #77141 or an equivalent patch that broadens the guarded stale-binding recovery to timeout and unknown `FailoverError` reasons, with regression coverage that preserves the single fresh retry only for active resumed sessions.

## Reproduction Assessment

Yes. By source inspection, a resumed CLI call that throws `FailoverError` with `reason: "timeout"` or `"unknown"` bypasses the current clear-and-retry branch and rethrows into fallback handling.

## Solution Assessment

Yes. Reusing the existing stale-binding clear plus one fresh retry is the narrowest maintainable fix, and the linked open PR is the current implementation candidate to review before closing this issue.
