## Summary

Keep open: current main still clears and retries a persisted CLI session binding only for `session_expired`, while `timeout` and `unknown` `FailoverError` paths can still rethrow from a resumed binding; open PR #77385 is the active closing implementation candidate.

## What This Changes

The issue asks OpenClaw to clear stale persisted CLI backend session bindings and retry fresh after any resumed-session `FailoverError`, not only `session_expired`.

## Best Possible Solution

Land or revise #77385 so stale CLI session bindings are cleared for any resumed-session `FailoverError`, retrying fresh only for recoverable reasons and preserving normal fallback behavior for fresh-session failures.

## Reproduction Assessment

Yes. Source inspection on current main shows a resumed CLI run with an active binding and `FailoverError.reason` other than `session_expired` bypasses the clear-and-retry branch and rethrows.

## Solution Assessment

Yes. Broadening the existing guarded stale-binding recovery path is the narrowest maintainable fix, and the already-linked PR targets that path with focused command-level coverage.
