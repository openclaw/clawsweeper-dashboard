## Summary

Keep open: current main still clears and retries a persisted CLI session binding only for `session_expired`, while `timeout` and `unknown` are real `FailoverError` reasons that still fall through with the stale binding intact. Open PR #77385 is the active closing implementation candidate, so the issue should remain open until that PR lands or is replaced.

## What This Changes

The issue requests that resumed CLI backend sessions clear stale bindings and retry fresh for FailoverError reasons beyond `session_expired`.

## Best Possible Solution

Land or revise #77385 so active CLI session bindings are cleared for all FailoverError reasons, with a fresh retry only for recoverable reasons and focused regression coverage.

## Reproduction Assessment

Yes. A source-level reproduction is clear: mock a resumed CLI run with an active binding that throws `FailoverError` reason `timeout` or `unknown`, and current main bypasses the clear-and-retry branch.

## Solution Assessment

Yes. Broadening the existing stale-binding recovery path is the narrowest maintainable fix; the issue should stay open while the linked closing PR is reviewed or replaced.
