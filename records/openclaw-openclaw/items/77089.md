## Summary

Keep open: current main still gates persisted CLI session clear-and-retry on `session_expired`, while `timeout` and `unknown` `FailoverError` reasons can still rethrow from a resumed binding. The linked closing PR #77385 is open and unmerged, so this issue should stay open until that implementation or an equivalent fix lands.

## What This Changes

The issue requests widening CLI backend stale-session recovery so any `FailoverError` from a resumed session clears the stored binding and retries fresh, not only `session_expired`.

## Best Possible Solution

Land or revise PR #77385 so persisted CLI bindings are cleared for all resumed-session `FailoverError` reasons, with one fresh retry only for recoverable reasons and focused regression coverage.

## Reproduction Assessment

Yes. Source inspection of current main shows a resumed CLI run with an active binding that throws `FailoverError` reason `timeout` or `unknown` bypasses the `session_expired` recovery branch and rethrows.

## Solution Assessment

Yes. Broadening the existing stale-binding recovery path is the narrowest maintainable direction, and the issue should remain open while the linked closing PR is reviewed or replaced.
