## Summary

Keep open: current main still has the reported `session_expired`-only recovery gate, and open PR #77141 is the active closing implementation candidate.

## What This Changes

The issue asks the CLI backend to clear a stale resumed session binding and retry once fresh for any `FailoverError`, not only `session_expired`.

## Best Possible Solution

Land or revise #77141 so resumed CLI session bindings clear and retry once fresh for timeout and unknown FailoverError paths without adding a retry loop for fresh-session failures.

## Reproduction Assessment

Yes. Source inspection on current main shows resumed CLI FailoverError reasons `timeout` and `unknown` bypass both existing fresh-retry branches and fall through to fallback handling instead of clearing the binding.

## Solution Assessment

Yes. Broadening the existing guarded stale-binding recovery path is the narrowest maintainable fix, but this issue should remain open while #77141 is reviewed, landed, or replaced.
