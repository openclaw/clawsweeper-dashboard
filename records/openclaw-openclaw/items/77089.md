## Summary

Keep open: current `main` still only clears and retries a persisted CLI session binding for `session_expired`, while `timeout` and `unknown` `FailoverError` paths still fall through to fallback; open PR #77385 is the active closing implementation candidate.

## What This Changes

The issue requests stale CLI backend session bindings be cleared on any resumed-session `FailoverError`, with one fresh retry for recoverable failures before API fallback.

## Best Possible Solution

Review and land or replace #77385 so active CLI session bindings clear on all `FailoverError` reasons, retry fresh once only for recoverable reasons, and keep focused regression coverage plus changelog coverage.

## Reproduction Assessment

Yes. By source inspection, a resumed CLI run with an active binding that throws `FailoverError` reason `timeout` or `unknown` bypasses the current `session_expired` branch and rethrows instead of clearing and retrying fresh.

## Solution Assessment

Yes. Broadening the existing guarded stale-binding recovery path is the narrow maintainable fix, and this issue should stay open while #77385 is reviewed, landed, or replaced.
