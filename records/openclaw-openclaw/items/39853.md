## Summary

Keep open: current main still lacks structured non-self Feishu mention metadata, but this PR is not ready because it drops supported non-`open_id` mention records, lacks after-fix real behavior proof, and overlaps the open channel-side arbitration work at https://github.com/openclaw/openclaw/pull/38939.

## What This Changes

The PR adds a `Mentions` template-context field, populates it from Feishu `event.message.mentions`, and serializes it into inbound conversation metadata.

## Best Possible Solution

Choose one Feishu mention-arbitration contract, ideally with a shared extractor that preserves `open_id`, `user_id`, and `union_id`, excludes the receiving bot only when identity is comparable, and emits bounded untrusted metadata only if maintainers approve that surface.

## Reproduction Assessment

Yes, at source level: current main receives Feishu `event.message.mentions`, normalizes mentions into message text, and only forwards `WasMentioned` into inbound metadata, so no structured non-self mention array reaches the agent. I did not run a live Feishu multi-bot group.

## Solution Assessment

No, not as submitted. The metadata direction may be useful, but the extractor is `open_id`-only and the final contract should be reconciled with the open Feishu channel-side arbitration PR.
