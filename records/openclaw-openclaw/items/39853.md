## Summary

Keep open: current main still lacks structured Feishu mention metadata, but this PR is blocked by missing after-fix real behavior proof and an open_id-only extractor that drops supported user_id/union_id mention records. Related channel-side arbitration work remains open at https://github.com/openclaw/openclaw/pull/38939.

## What This Changes

The PR adds a Feishu `Mentions` template-context field and serializes non-self mentions into inbound conversation metadata.

## Best Possible Solution

Choose one Feishu mention-arbitration contract, with a shared extractor that preserves supported ID shapes and bounded untrusted metadata only if maintainers approve that surface.

## Reproduction Assessment

Yes, at source level: current main receives Feishu `event.message.mentions`, normalizes some mention data into text, and only forwards `WasMentioned` into inbound metadata. I did not establish a live Feishu multi-bot group reproduction.

## Solution Assessment

No, not as submitted. The metadata direction may be useful, but extraction should preserve `open_id`, `user_id`, and `union_id` where available and be reconciled with the open channel-side arbitration PR.
