## Summary

Keep open. Current main still lacks structured non-self Feishu mention metadata, so the contribution targets a real multi-bot routing gap, but the submitted patch is not safe to merge as-is because it drops valid Feishu mention payloads that do not include `open_id`.

## What This Changes

The branch adds a `Mentions` field to inbound template context, populates it from Feishu message mentions, and serializes it as `mentions` in conversation info.

## Best Possible Solution

Revise this into a tested Feishu mention extraction contract that excludes the receiving bot, preserves stable identity across supported Feishu ID shapes, serializes the list only in the untrusted inbound conversation metadata block, and coordinates behavior with #38939.

## Reproduction Assessment

Yes. A Feishu group with multiple bot accounts and `requireMention=false` can dispatch a message addressed only to a different bot; current main passes only `WasMentioned` and has no structured non-self mention list for the agent to inspect.

## Solution Assessment

No, not as submitted. The direction is maintainable, but the implementation should use a shared extractor with `open_id`/`user_id`/`union_id` fallback and focused tests instead of an inline `open_id`-only filter.

## Review Findings

Overall correctness: patch is incorrect
