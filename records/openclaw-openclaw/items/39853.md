## Best Possible Solution

Adopt one Feishu mention handling path: either populate the existing generic mention metadata from Feishu facts or land an approved arbitration policy, with supported ID forms, broadcast filtering, focused coverage, and redacted live Feishu proof.

## Solution Assessment

No, not as submitted. The maintainable path should reuse the existing generic mention metadata contract or an accepted Feishu arbitration policy, while this branch creates a parallel schema and misses ID and broadcast semantics.

  - confidence: 0.91

Including broadcasts in `mentions` can make agents suppress replies even when no sibling bot was addressed; reuse the existing broadcast guard before mapping.
  - confidence: 0.84
