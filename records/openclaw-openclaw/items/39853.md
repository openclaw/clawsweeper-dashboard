## Summary

Keep open: current main still lacks Feishu structured non-self mention metadata, and this PR is not mergeable as submitted because it drops supported mention ID forms, does not reuse broadcast filtering, and lacks real Feishu proof.

## What This Changes

The PR adds Feishu non-self mention data to the template context and serializes it into inbound conversation metadata.

## Best Possible Solution

Use the existing generic mention metadata or an accepted Feishu arbitration contract, handling all Feishu ID forms and broadcast mentions with focused tests and redacted live proof.

## Reproduction Assessment

Yes, at source level: current main receives Feishu event.message.mentions, normalizes mention text, and forwards only WasMentioned into inbound metadata. I did not establish a live Feishu multi-bot reproduction in this read-only review.

## Solution Assessment

No, not as submitted. The metadata direction may be useful, but the patch should align with the current generic mention metadata or a reviewed arbitration policy and must preserve Feishu ID and broadcast semantics.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve mentions without open_id:** `extensions/feishu/src/bot.ts:1332-1335`
  - body: Filtering on `m.id.open_id` makes the new metadata empty for Feishu mention records that only carry `user_id` or `union_id`, which the current event type already supports. In those deliveries the agent still cannot see that another bot or user was addressed, so derive a stable ID from all supported fields before mapping.
  - confidence: 0.91
- **[P2] Filter broadcast mentions before serializing:** `extensions/feishu/src/bot.ts:1332-1335`
  - body: The new list treats any non-self `open_id` as an addressed user or bot. Feishu broadcast mentions can arrive as `@all`/`@_all` or ID `all`, and current main already has a guard for this; serializing broadcast as a normal mention can make agents suppress replies when no sibling bot was targeted.
  - confidence: 0.84
