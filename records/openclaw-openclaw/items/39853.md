## Summary

Keep open: current main still lacks the requested structured Feishu mention metadata, and the PR has a correctness issue in its `open_id`-only extraction plus a missing required changelog entry; it also overlaps the open #38939 arbitration work.

## What This Changes

The PR adds a Feishu `Mentions` template-context field from inbound message mentions and serializes it as `mentions` in inbound conversation info.

## Best Possible Solution

Choose one Feishu mention-arbitration contract, preferably sharing a tested extractor that preserves supported Feishu ID shapes, excludes the receiving bot where identity is known, emits bounded untrusted metadata only if still needed, and includes the required changelog entry.

## Reproduction Assessment

Yes. Source inspection shows current main has `event.message.mentions`, normalizes mentions into text, and passes only `WasMentioned` into inbound metadata, so a multi-bot Feishu group has no structured non-self mention list for the agent to inspect.

## Solution Assessment

No, not as submitted. The metadata direction may be useful, but the implementation drops valid non-`open_id` mention entries and should be coordinated with #38939's channel-side arbitration work.
