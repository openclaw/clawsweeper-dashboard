## Summary

Keep open: this PR targets a real Feishu multi-bot duplicate-reply gap, but the submitted extractor drops valid non-`open_id` mention records, lacks after-fix Feishu proof, and overlaps the still-open arbitration work at https://github.com/openclaw/openclaw/pull/38939.

## What This Changes

The PR adds a `Mentions` template-context field, populates it from Feishu `event.message.mentions`, and serializes it into inbound conversation metadata.

## Best Possible Solution

Define one Feishu mention-arbitration contract, preferably with a shared extractor that preserves supported Feishu ID forms and documents that mention names are untrusted.

## Reproduction Assessment

Yes, at source level: current main receives Feishu mention data and can normalize mentions into message text, but only `WasMentioned` reaches inbound metadata. I did not run a live Feishu multi-bot group.

## Solution Assessment

No, not as submitted. The metadata direction may be useful, but the extractor is `open_id`-only and maintainers should reconcile it with the open Feishu channel-side arbitration PR.
