## Summary

Keep open: current main still lacks structured non-self Feishu mention metadata, but this PR is not ready because its open_id-only extraction drops valid mention shapes and it overlaps the still-open Feishu arbitration work in #38939.

## What This Changes

The PR adds a `Mentions` template-context field, fills it from Feishu inbound mention metadata excluding the receiving bot, and serializes it as `mentions` in the inbound conversation-info block.

## Best Possible Solution

Coordinate with #38939 and land one Feishu mention-extraction/arbitration path that preserves supported ID shapes, avoids duplicate bot replies at the right channel boundary, and includes focused regression tests plus the required changelog entry.

## Reproduction Assessment

Yes, source-level. Current main receives `event.message.mentions`, normalizes them into text, and passes only `WasMentioned` into inbound context, so a multi-bot Feishu group has no structured non-self mention list for the agent to inspect.

## Solution Assessment

No, not as submitted. The metadata direction is plausible, but the extractor should preserve Feishu's supported ID shapes and the final direction should be coordinated with #38939's channel-side arbitration work.
