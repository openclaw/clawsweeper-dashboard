## Summary

Keep open: the PR addresses a real Feishu metadata gap, but the submitted extraction drops supported mention ID shapes and the user-facing feature lacks the required changelog entry; it also needs coordination with the open Feishu arbitration work in #38939.

## What This Changes

The PR adds a `Mentions` template-context field, populates it from Feishu inbound mention events excluding the receiving bot by `open_id`, and serializes it as `mentions` in inbound conversation info.

## Best Possible Solution

Revise or fold the work into #38939 so Feishu has one tested mention-extraction contract that handles `open_id`/`user_id`/`union_id`, excludes the receiving bot correctly, emits bounded untrusted metadata if maintainers still want it, and includes the required changelog entry.

## Reproduction Assessment

Yes. Source inspection shows Feishu inbound events can carry structured mentions, while current main only passes `WasMentioned` into the template context and does not expose a structured non-self mention list to the agent metadata path.

## Solution Assessment

No, not as submitted. The metadata direction is plausible, but the patch should reuse or introduce a shared Feishu mention extractor with ID fallback and coordinate with #38939 rather than adding duplicate `open_id`-only inline logic.
