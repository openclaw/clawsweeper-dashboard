## Summary

This PR addresses a real Feishu multi-bot mention gap, but it is not merge-ready: the patch drops valid non-`open_id` mention records, lacks inspectable real Feishu proof, and now overlaps the newer generic mention metadata contract plus the open arbitration work.

## What This Changes

The PR adds a Feishu-populated `Mentions` template-context field and serializes it as a `mentions` array in inbound conversation metadata.

## Best Possible Solution

Decide the Feishu mention strategy on current main, then implement it through the generic mention metadata contract or channel-side arbitration while preserving all supported Feishu mention IDs, broadcast/self filtering, focused regression tests, and redacted real multi-bot proof.

## Reproduction Assessment

Yes, at source level: current main receives Feishu `event.message.mentions`, normalizes non-self mentions into the body, and only forwards `WasMentioned` into Feishu inbound metadata. I did not establish a live Feishu multi-bot reproduction in this read-only review.

## Solution Assessment

No, not as submitted. The metadata direction may be useful, but current main now has a generic mention metadata contract and this PR adds a parallel `Mentions` shape while losing `user_id`/`union_id` mention records.

## Review Findings

Overall correctness: patch is incorrect
