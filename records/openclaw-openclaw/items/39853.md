## Summary

Keep this PR open. Current main still does not expose structured Feishu non-self mentions in inbound metadata, but the submitted implementation has a correctness bug around Feishu ID shapes and overlaps with the broader open Feishu mention-arbitration work in #38939.

## What This Changes

The PR adds a `Mentions` template-context field, populates it from Feishu message mentions excluding the receiving bot by `open_id`, and serializes it as `mentions` in inbound conversation info.

## Best Possible Solution

Revise or fold the work into the Feishu arbitration path so one tested mention extraction contract excludes the receiving bot, preserves supported Feishu ID shapes, writes only bounded untrusted inbound metadata, and includes the required changelog entry.

## Reproduction Assessment

Yes. A Feishu group with multiple bot accounts and `requireMention=false` can deliver a rich-text mention of another bot; current main passes only `WasMentioned` and gives the agent no structured non-self mention list to inspect.

## Solution Assessment

No, not as submitted. The metadata direction is reasonable, but the implementation should use a shared extractor with `open_id`/`user_id`/`union_id` fallback and coordinate with #38939 rather than relying on an inline `open_id`-only filter.
