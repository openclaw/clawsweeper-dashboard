## Summary

This PR addresses a well-supported Bedrock Mantle duplication bug, but its provider-agnostic heuristic can collapse legitimate adjacent Responses messages whenever their text is prefix-related and their phases match—or are both absent. Existing delivery coverage explicitly preserves such messages as separate replies, so the predicate must be narrowed before merge.

## What This Changes

The PR changes OpenAI Responses stream consumers to collapse adjacent same-phase cumulative message snapshots into one text block while preserving boundaries around reasoning and tool calls.

## Best Possible Solution

Narrow snapshot detection to a positively identified malformed-response pattern that cannot match legitimate adjacent messages, preserve normal message-item boundaries, and prove final text, block-streamed delivery, and next-turn replay.

Yes. The issue gives a concrete live Bedrock Mantle reproduction plus raw API captures, and the PR reproduces the malformed event sequence through a real local OpenClaw agent and session store.

## Solution Assessment

No. The shared compatibility seam is a reasonable owner, but prefix plus equal phase is not a safe discriminator because current behavior intentionally supports distinct adjacent prefix-related messages.
