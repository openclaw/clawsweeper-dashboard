## Summary

The PR addresses a well-supported provider regression and includes credible end-to-end proof, but its generic prefix heuristic can silently collapse independently identified assistant message items that the Responses protocol exposes as distinct output items.

## What This Changes

The PR adds shared logic that collapses adjacent same-phase cumulative OpenAI Responses message items in both stream consumers and completed-response backfill, while preserving the first item signature and adding regression tests.

## Best Possible Solution

Keep normalization in the shared Responses compatibility seam, but require a stronger malformed cumulative-snapshot invariant than text-prefix adjacency before changing canonical message-item boundaries.

## Reproduction Assessment

Yes. Raw streaming and non-streaming provider evidence establishes the malformed cumulative-item shape, and current main deterministically appends those items as separate text blocks.

## Solution Assessment

No, not in its current form. The shared compatibility boundary is appropriate, but text-prefix adjacency alone is broader than the demonstrated provider corruption and can erase distinct output items.
