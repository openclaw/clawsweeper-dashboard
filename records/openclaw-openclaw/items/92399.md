## Summary

The central fix is necessary and the shared adjacent-snapshot rule is well targeted, but the completed-response fallback does not reset collapse state when a reasoning item appears between messages. That one-sided boundary handling can suppress a legitimate post-reasoning message and produce a transcript/replay shape that differs from both streaming consumers, so the PR should remain open for a narrow repair and live-provider proof.

## What This Changes

The PR adds same-phase cumulative message-snapshot collapsing to both OpenAI Responses stream consumers and completed-response backfill, with regression tests for streaming boundaries.

## Best Possible Solution

Apply one canonical output-item boundary rule across streaming and completed-response parsing: collapse only truly adjacent same-phase message snapshots, reset on every non-message item, preserve the first snapshot signature, and prove the repaired path against live Bedrock Mantle with redacted terminal or transcript evidence.

## Reproduction Assessment

Yes from source and linked wire evidence: current main appends every completed message item, reproducing cumulative snapshot duplication, and the proposed backfill path also has a deterministic message → reasoning → prefix-message failure case. This review did not run the affected live provider.

## Solution Assessment

No, not yet. Shared adjacent same-phase collapse is the narrow maintainable direction, but completed-response backfill must enforce the same non-message boundaries as streaming before this is the best fix.
