## Label Justifications

## Summary

The reported duplication is real and the shared stream consumers are the right ownership boundary, but the proposed text-only collapse rule can silently discard legitimate distinct assistant message items, so this head is not safe to merge yet.

## What This Changes

The PR adds shared cumulative-message snapshot collapsing to both OpenAI Responses stream consumers and completed-response backfill, with regression and boundary tests.

## Best Possible Solution

Keep the shared consumer/backfill fix, but gate collapse on a provider-quirk or a stronger sequence-level signature that proves a cumulative snapshot run; preserve every independently identified message when that proof is absent.

## Reproduction Assessment

Yes. The linked issue provides a concrete live Bedrock Mantle command and raw streaming/non-streaming cumulative output, while current main clearly appends every completed message item.

## Solution Assessment

No. Fixing all shared consumers is the right boundary, but text-prefix similarity alone is not a safe identity rule for independent Responses output items; collapse needs an affected-provider or stronger sequence discriminator.
