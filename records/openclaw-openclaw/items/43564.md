## Summary

Keep open: current main still has no ACP skill-context injection path for `sessions_spawn`, and the remaining work needs maintainer product and security API design rather than conservative cleanup.

## What This Changes

Request to let `runtime="acp"` `sessions_spawn` calls inject selected OpenClaw skill context into the spawned ACP agent's initial prompt.

## Best Possible Solution

Define a maintainer-approved, explicit, allowlisted ACP skill-context contract first, with bounded prompt injection, no tool or env grant, persistence semantics, and focused tests/docs before considering automatic relevance or defaults.

## Reproduction Assessment

Yes, as a source-level feature gap: current `sessions_spawn`, ACP spawn params, ACP manager types, and ACPX adapter paths have no `skills` field or skill-context carrier. This is not a bug reproduction because the item requests a new capability.

## Solution Assessment

Unclear: the issue identifies a real gap, but automatic/default injection is not yet a safe implementation shape. The narrowest maintainable path is an explicit allowlisted skill-selection contract before relevance detection or default skill sets.
