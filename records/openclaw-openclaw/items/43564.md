## Summary

Keep open: current main still lacks ACP skill-context injection, and the request is a new ACP/session capability with API, prompt-budget, and trust-boundary decisions rather than a broken existing contract.

## What This Changes

Request to let `sessions_spawn(runtime="acp")` include selected, default, or automatically detected OpenClaw skill context in the spawned ACP harness prompt.

## Best Possible Solution

Add a documented ACP skill-context capability only after maintainers choose the API and safety policy; an explicit typed skill selector is the narrow first step, with automatic and default injection deferred.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection on current main shows the ACP spawn schema, params, dispatch path, and runtime turn input still have no skill-context field.

## Solution Assessment

Unclear for the full proposal. Explicit skill selection appears to be the narrow maintainable path, while automatic relevance detection and default injection need prompt-budget, trust-boundary, and configuration policy first.

## Security Review

Status: needs_attention

Summary: The issue is security-sensitive because it would move local skill prompt material across the ACP host-runtime boundary into external harness prompts.

Concerns:

- **[medium] Define the skill prompt boundary before injection:** `src/agents/acp-spawn.ts:1383`
  - body: ACP spawns currently send only `params.task` to the Gateway agent method. Any implementation should enforce explicit selection, eligibility filtering, prompt budgets, and path/content exposure rules before sending rendered skill content to an external ACP harness.
  - confidence: 0.88
- **[medium] Preserve skill secret isolation:** `src/acp/client.ts:120`
  - body: The ACP client strips active skill env keys from child process environments, and the skills docs warn that skill env/API key values must stay out of prompts and logs. Prompt-level skill injection must not reintroduce those secrets through rendered context.
  - confidence: 0.84
