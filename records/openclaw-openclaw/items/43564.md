## Summary

Keep open: current main and the latest release still lack an ACP skill-selection parameter or skill-context carrier, and the remaining work needs maintainer decisions across ACP prompt boundaries, session persistence, backend behavior, and skill-security rules.

## What This Changes

Request ACP `sessions_spawn` support for injecting selected OpenClaw skill context into spawned Codex/Pi/OpenCode/Gemini ACP sessions.

## Best Possible Solution

Define a maintainer-approved ACP skill-context contract first: explicit allowlisted selection, bounded rendered prompt text, no tool/env grants, clear resume persistence semantics, backend behavior, docs, and focused tests.

## Reproduction Assessment

Yes, as a source-level feature gap: current `sessions_spawn`, ACP spawn params, ACP runtime inputs, and ACP docs have no `skills` field or skill-context carrier. This is not a bug reproduction because the item requests a new capability.

## Solution Assessment

Unclear: the requested direction identifies a real gap, but automatic/default injection is not yet a safe implementation shape. The narrowest maintainable path is an explicit allowlisted ACP skill-selection contract before relevance detection or default skill sets.
