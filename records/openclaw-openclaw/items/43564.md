## Summary

Keep open: current main and v2026.6.6 still have ACP `sessions_spawn` as task/text plus attachments, with no `skills` parameter, skill snapshot carrier, or ACP manager/runtime path for selected skill context. The request is valid but needs maintainer product and security design before implementation.

## What This Changes

Request ACP `sessions_spawn` to optionally inject selected OpenClaw skill context into spawned external harness sessions.

## Best Possible Solution

Define a maintainer-approved ACP skill-context contract first: explicit allowlisted selection, bounded prompt size, no tool/env grants, clear persistence semantics, adapter-specific support where needed, and focused docs/tests before considering automatic relevance or defaults.

## Reproduction Assessment

Yes, as a source-level feature gap: current `sessions_spawn`, ACP spawn params, ACP manager inputs, and ACPX adapter paths have no `skills` field or skill-context carrier. This is not a bug reproduction because the item requests a new capability.

## Solution Assessment

Unclear: the request identifies a real gap, but automatic/default injection is not yet a safe implementation shape. The narrowest maintainable path is an explicit allowlisted skill-selection contract before relevance detection or default skill sets.
