## Summary

Keep open: current main and the latest release still lack an ACP skill-selection parameter or skill-context carrier, and the request crosses ACP prompt, session-state, and skill security boundaries that need maintainer design before implementation.

## What This Changes

Request optional injection of selected or rendered OpenClaw skill context into `sessions_spawn({ runtime: "acp" })` child sessions.

## Best Possible Solution

Define a maintainer-approved ACP skill-context contract first: explicit allowlisted selection, bounded prompt size, no env/tool grants, clear persistence/resume semantics, backend behavior, docs, and focused tests.

## Reproduction Assessment

Yes, as a source-level feature gap: the current `sessions_spawn` schema, ACP spawn params, ACP runtime inputs, and ACPX adapter path have no `skills` field or skill-context carrier. This is not a bug reproduction because the item requests a new capability.

## Solution Assessment

Unclear: the request identifies a real gap, but automatic/default injection is not yet a safe implementation shape. The narrowest maintainable path is an explicit allowlisted skill-selection contract before relevance detection or default skill sets.
