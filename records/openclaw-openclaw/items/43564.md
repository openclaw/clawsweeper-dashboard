## Summary

Keep open: current main and v2026.6.10 still lack a `sessions_spawn(runtime: "acp")` skill selector or ACP skill-context carrier, and the discussion adds security/audit requirements that need maintainer product direction before implementation.

## What This Changes

The issue requests optional OpenClaw skill context injection into ACP sessions spawned via `sessions_spawn(runtime: "acp")`, including explicit, automatic, or default skill selection.

## Best Possible Solution

Define a maintainer-approved ACP skill-injection contract covering selection mode, redaction, observability, persistence/resume behavior, and harness-specific delivery, then implement the schema/runtime/docs/tests against that contract.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence check: current `sessions_spawn` has no `skills` parameter, ACP spawn params carry no skill context, and ACP runtime turn input has no skill carrier.

## Solution Assessment

Unclear for implementation. The request is valid, but the best fix depends on maintainer choices about explicit/default/auto selection, redaction, observability, persistence, and whether new config should exist.
