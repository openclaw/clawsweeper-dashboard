## Summary

Keep open: current main and the latest v2026.6.6 release still expose ACP `sessions_spawn` as task/session controls plus attachments, with no `skills` parameter, skill snapshot carrier, or ACP runtime path for selected skill context. The request is valid, but it needs maintainer product and security design before implementation.

## What This Changes

Request to let `sessions_spawn` for `runtime: "acp"` optionally inject selected or rendered OpenClaw skill context into the spawned ACP session.

## Best Possible Solution

Define a maintainer-approved ACP skill-context contract first: explicit allowlisted selection, bounded prompt size, no tool or environment grants, clear persistence semantics, adapter-specific behavior where needed, and focused docs/tests before considering automatic relevance or default skill sets.

## Reproduction Assessment

Yes, as a source-level feature gap: current `sessions_spawn`, ACP spawn params, ACP manager inputs, ACPX adapter paths, and ACP docs have no `skills` field or skill-context carrier. This is not a bug reproduction because the item requests a new capability.

## Solution Assessment

Unclear: the request identifies a real gap, but automatic/default injection is not yet a safe implementation shape. The narrowest maintainable path is an explicit allowlisted skill-selection contract before relevance detection or default skill sets.
