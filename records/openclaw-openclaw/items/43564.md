## Summary

Keep open: current main and the latest release still expose ACP `sessions_spawn` as task/session controls plus attachments, with no `skills` parameter, selected-skill snapshot carrier, or ACP adapter path for injected skill context. The request is valid, but it changes a model-callable ACP session API and needs maintainer product/security design before implementation.

## What This Changes

Request optional injection of selected OpenClaw skill content into ACP `sessions_spawn` contexts for child agents.

## Best Possible Solution

Define a maintainer-approved ACP skill-context contract first: explicit allowlisted selection, bounded prompt size, no tool/env grants, clear persistence semantics, adapter support where needed, and focused docs/tests before considering automatic relevance or defaults.

## Reproduction Assessment

Yes, as a source-level feature gap: current `sessions_spawn`, ACP spawn params, ACP manager inputs, and ACPX adapter paths have no `skills` field or skill-context carrier. This is not a bug reproduction because the item requests a new capability.

## Solution Assessment

Unclear: the request identifies a real gap, but automatic/default injection is not yet a safe implementation shape. The narrowest maintainable path is an explicit allowlisted skill-selection contract before relevance detection or default skill sets.
