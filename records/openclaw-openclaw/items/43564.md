- `impact:session-state`: The requested behavior changes what context is injected into spawned ACP sessions and how that context persists across session turns.

## Summary

Keep open: current main and the latest release still lack a `sessions_spawn(runtime: "acp")` skill-selection parameter or ACP skill-context carrier, and the requested automatic/default injection crosses session-state and security boundaries that need maintainer design first.

## What This Changes

Request to let ACP session spawns receive selected OpenClaw skill context, either explicitly, automatically, or from defaults.

## Best Possible Solution

Define a bounded ACP skill-context contract with explicit allowlisting, snapshot timing, prompt-only versus env/tool separation, tests, and docs before adding any spawn parameter or default injection behavior.

## Reproduction Assessment

Yes, as a source-level feature gap: current `sessions_spawn`, ACP spawn params, manager inputs, and runtime turn types have no `skills` field or skill-context carrier. This is not a broken-current-behavior bug reproduction.

## Solution Assessment

Unclear: the issue identifies a real gap, but automatic or default injection is not the best confirmed solution without maintainer decisions on explicit selection, trust, prompt size, and session persistence. A narrow explicit allowlisted contract is the safer direction than implicit relevance injection.
