## Summary

Keep open: current main still lacks a `sessions_spawn(runtime: "acp")` skill-selection parameter or ACP skill-context carrier, while the requested capability crosses prompt, session-state, and security boundaries that need maintainer design first.

## What This Changes

The issue requests a mechanism for `sessions_spawn(runtime: "acp")` to inject selected OpenClaw skill context into spawned ACP session prompts.

## Best Possible Solution

Define a maintainer-approved ACP skill-context contract first: explicit allowlisted selection, bounded prompt size, no tool or env grants, clear persistence semantics, adapter-specific behavior, and focused docs/tests before automatic relevance or defaults.

## Reproduction Assessment

Yes, as a source-level feature gap: current `sessions_spawn`, ACP spawn params, ACP manager inputs, and ACPX adapter paths have no `skills` field or skill-context carrier. This is not a bug reproduction because the item requests a new capability.

## Solution Assessment

Unclear: the request identifies a real gap, but automatic/default injection is not yet a safe implementation shape. The narrowest maintainable path is an explicit allowlisted skill-selection contract before relevance detection or default skill sets.
