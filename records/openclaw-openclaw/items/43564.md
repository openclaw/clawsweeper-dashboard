## Summary

Keep open. Current main and v2026.5.12 still expose ACP `sessions_spawn` without a per-call skill selector or rendered skill-context injection path, and the request crosses session-context and external-harness security boundaries that need maintainer product scoping.

## What This Changes

The issue requests optional ACP `sessions_spawn` skill context injection through explicit skills, automatic relevance, or a default skill set.

## Best Possible Solution

Define a narrow opt-in ACP spawn skill-context contract that reuses existing skill snapshot/filtering, preserves ACP isolation, documents redaction and token behavior, and starts with explicit skill lists before broader auto/default injection.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main lacks a sessions_spawn skill field and that ACP dispatch sends only params.task.

## Solution Assessment

Unclear until maintainer product and security scoping is done. An explicit opt-in skill list looks like the safest starting point; automatic relevance and default injection should wait for a broader policy decision.
