## Summary

Keep open: current main still lacks ACP skill-context injection, and the request would add a new ACP/session API plus prompt-boundary policy rather than fix broken existing behavior.

## What This Changes

Request optional injection of selected OpenClaw skill context into ACP sessions spawned through `sessions_spawn(runtime: "acp")`.

## Best Possible Solution

Add a documented ACP skill-context capability only after maintainers settle the API and safety policy; an explicit typed skill selector is the narrow first step, with automatic/default injection deferred until trust-boundary and prompt-budget rules are defined.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Current-main source inspection shows the ACP spawn schema, params, dispatch path, and runtime turn input still have no skill-context field.

## Solution Assessment

Unclear for the full proposal. Explicit skill selection looks like the narrowest maintainable first implementation, while automatic relevance detection and default injection need product, prompt-budget, and host-runtime safety decisions first.

## Security Review

Status: needs_attention

Summary: The requested feature is security-sensitive because it would move local skill context across the ACP host-runtime boundary into external harness prompts.

Concerns:

- **[medium] Define skill prompt boundary before injection:** `src/agents/acp-spawn.ts:1388`
  - body: ACP spawns currently send only `params.task` as the Gateway message. Any implementation that injects rendered `SKILL.md` content should enforce explicit selection, eligibility filtering, prompt budgets, and path/secret exposure rules before sending it to an external ACP harness.
  - confidence: 0.88
- **[medium] Preserve skill secret isolation:** `src/acp/client.ts:120`
  - body: The ACP client strips active skill env keys from child process environments, and the skills docs warn that `skills.entries.*` secrets must stay out of prompts and logs. Prompt-level skill injection should not reintroduce those values through rendered skill context.
  - confidence: 0.84
