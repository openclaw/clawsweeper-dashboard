## Summary

Keep open: current main and the latest shipped release still lack an ACP `sessions_spawn` skill selector or rendered skill-context delivery path, and the remaining design crosses prompt/session-state and security boundaries.

## What This Changes

Request to let `sessions_spawn(runtime="acp")` inject selected, automatic, or default OpenClaw skill context into spawned ACP sessions.

## Best Possible Solution

Define a narrow opt-in ACP skill-context contract that reuses existing skill snapshots and filters, preserves ACP isolation, documents redaction/token behavior, and defers automatic or default injection until policy is settled.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main lacks a `sessions_spawn` skill field and that ACP dispatch currently sends only the task text into the ACP runtime path.

## Solution Assessment

Unclear until maintainer product and security scoping is done. An explicit opt-in skill list looks like the safest starting point; automatic relevance and default injection should wait for a broader policy decision.
