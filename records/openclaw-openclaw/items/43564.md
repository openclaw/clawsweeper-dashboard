## Summary

Keep this issue open. Current main still does not implement ACP skill-context injection: the public `sessions_spawn` schema has no skill selector, ACP spawn params and runtime turn input have no rendered skill-context field, and ACP dispatch forwards only the task text. The earlier ClawSweeper comment remains accurate after checking current `main`; this needs maintainer API and safety decisions rather than cleanup closure.

## What This Changes

The issue requests optional OpenClaw skill-context injection for `sessions_spawn` calls using `runtime: "acp"`, via explicit skill lists, automatic relevance, or default skill sets.

## Best Possible Solution

Add a documented ACP skill-context capability only after maintainers settle the public API and safety policy. A conservative first step would be an explicit typed skill selector that reuses existing skill allowlists, eligibility filtering, and prompt-budget logic, with clear behavior for missing skills, truncation, secrets, sandboxed requesters, and docs.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The current-main static check is high-confidence because the spawn schema, ACP params, dispatch call, runtime turn input, and docs all lack a skill-context path.

## Solution Assessment

Unclear for the full proposal. Explicit skill selection looks like the narrowest maintainable first implementation, while automatic relevance detection and default injection need product, prompt-budget, and host-runtime safety decisions first.

## Security Review

Status: needs_attention

Summary: The requested feature is security-sensitive because it would move local skill context across the ACP host-runtime boundary into external harness prompts.

Concerns:

- **[medium] Constrain skill context crossing the ACP boundary:** `src/agents/acp-spawn.ts:1387`
  - body: ACP spawns currently send only task text to the Gateway `agent` call. Any implementation that injects skill context should preserve skill allowlists, prompt budgets, path exposure expectations, and local configuration isolation before sending rendered skill content to an external harness.
  - confidence: 0.86
- **[medium] Preserve ACP skill-secret isolation:** `src/acp/client.ts:135`
  - body: The ACP client strips active skill env keys from spawned ACP server environments. Prompt-level skill injection should not reintroduce secret values or sensitive local configuration through rendered skill context.
  - confidence: 0.8
