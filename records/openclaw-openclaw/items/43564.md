## Summary

Keep open: current main has ACP spawn support and normal OpenClaw skill snapshots, but the ACP spawn path still sends only the task text to the external harness and does not expose a `skills` selection or injection contract. The request is a new user-facing API/config capability with security and product choices, not stale cleanup.

## What This Changes

The issue requests optional OpenClaw skill context injection for `sessions_spawn({ runtime: "acp" })` sessions.

## Best Possible Solution

Define an additive ACP skill-context contract that reuses existing OpenClaw skill snapshots and per-agent allowlists, then document the security semantics before wiring it into ACP spawn.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection does give high confidence that current main lacks a `skills` parameter or ACP skill-context injection path.

## Solution Assessment

Unclear for implementation because there is no patch yet. The maintainable direction is likely an additive tool/protocol field backed by the existing skill snapshot and allowlist system, but auto relevance and default injection need maintainer design.
