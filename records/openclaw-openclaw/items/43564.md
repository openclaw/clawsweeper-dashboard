## Summary

Keep open: current main still lacks ACP skill-context injection, and the request would add a new public ACP/session API plus prompt-boundary security policy rather than fix broken existing behavior.

## What This Changes

Request ACP `sessions_spawn` to accept skill selection or automatic/default skill injection so spawned ACP harness sessions receive relevant OpenClaw skill context.

## Best Possible Solution

Add a documented ACP skill-context capability only after maintainers settle the API and safety policy; an explicit typed skill selector is the narrow first step, with automatic/default injection deferred until trust-boundary and prompt-budget rules are defined.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Current-main source inspection shows the ACP spawn schema, params, dispatch path, and runtime turn input have no skill-context field.

## Solution Assessment

Unclear for the full proposal. Explicit skill selection looks like the narrowest maintainable first implementation, while automatic relevance detection and default injection need product, prompt-budget, and host-runtime safety decisions first.

## Security Review

Status: needs_attention

Summary: The requested feature is security-sensitive because it would move local skill context across the ACP host-runtime boundary into external harness prompts.

Concerns:

- **[medium] Constrain skill context crossing the ACP boundary:** `src/agents/acp-spawn.ts:1388`
  - body: ACP spawns currently send only task text to the Gateway `agent` call. Any implementation that injects skill context should preserve skill allowlists, eligibility filtering, prompt budgets, path exposure expectations, and local configuration isolation before sending rendered skill content to an external harness.
  - confidence: 0.86
- **[medium] Preserve ACP skill-secret isolation:** `src/acp/client.ts:120`
  - body: The ACP client strips active skill env keys from spawned ACP server environments, and skills docs warn to keep skill env/apiKey values out of prompts and logs. Prompt-level skill injection should not reintroduce secrets or sensitive local configuration through rendered skill context.
  - confidence: 0.82
