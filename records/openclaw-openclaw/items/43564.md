## Summary

Keep open. Current main still lacks an ACP spawn skill selector and an ACP skill prompt-injection path; implementing this needs ACP API and prompt-safety policy decisions rather than conservative cleanup.

## What This Changes

Allow `sessions_spawn({ runtime: "acp" })` to accept skill selection/default skill context and inject rendered skill content into ACP session prompts.

## Best Possible Solution

Define a narrow opt-in ACP spawn contract that reuses existing skill filtering/snapshot code, keeps prompt-unsafe content out, documents token/security behavior, and adds focused schema, dispatch, ACP-runtime, and docs coverage.

## Reproduction Assessment

## Solution Assessment

Unclear. An explicit opt-in skill list looks like the narrowest starting point, but maintainers should decide the ACP API and safety policy before implementation.
