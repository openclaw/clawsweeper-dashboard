## Summary

Keep open: current main still lacks a `sessions_spawn` skill selector or rendered skill prompt path for ACP, and the remaining work crosses session-context and prompt security boundaries that need maintainer product scoping.

## What This Changes

Request to add optional skill-context injection for ACP `sessions_spawn` calls, including explicit skill lists, automatic relevance, or default skill sets.

## Best Possible Solution

Define a narrow opt-in ACP skill-context contract that reuses existing skill snapshots and filters, preserves ACP isolation, documents redaction and token behavior, and starts with explicit skill lists before auto/default injection.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main lacks a `sessions_spawn` skill field and that ACP dispatch currently sends only `params.task`.

## Solution Assessment

Unclear until maintainer product and security scoping is done. An explicit opt-in skill list looks like the safest starting point; automatic relevance and default injection should wait for a broader policy decision.
