## Summary

Current `main` still does not expose a `sessions_spawn` skill selector or pass rendered skill context through ACP spawn, so this remains a real feature request rather than a cleanup-close candidate. The requested behavior crosses the tool schema, ACP runtime contract, and prompt-safety boundary, so it needs maintainer product/security scoping before implementation.

## What This Changes

Add an opt-in way for `sessions_spawn(runtime="acp")` to inject selected OpenClaw skill context into spawned ACP agent prompts.

## Best Possible Solution

Define a narrow opt-in ACP spawn skill-context contract that reuses existing skill filtering and snapshot rendering, documents token/safety behavior, and keeps secret or environment-bearing skill metadata out of ACP prompts.

## Reproduction Assessment

Not applicable. This is a feature request, and current-source inspection confirms there is no `sessions_spawn` skill-selection field or ACP prompt-injection contract to reproduce as existing behavior.

## Solution Assessment

Unclear. An explicit opt-in skill list looks like the narrowest maintainable starting point, but maintainers should decide the ACP API shape and prompt-safety policy before implementation.
