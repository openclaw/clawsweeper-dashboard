## Summary

Keep open: current main and v2026.6.9 still do not expose a `skills` selector or ACP skill-context carrier, and this remains a product/security contract decision across `sessions_spawn`, ACP runtime inputs, prompt boundaries, observability, and resume behavior.

## What This Changes

Request optional skill context injection for ACP `sessions_spawn` sessions using explicit, automatic, or default skill selection.

## Best Possible Solution

Define an explicit allowlisted ACP skill-context contract first, with bounded rendered content, no tool or environment authority transfer, observable selection metadata, resume semantics, and focused docs/tests before adding auto or default selection.

## Reproduction Assessment

Yes, as a source-level feature gap: current `sessions_spawn`, `SpawnAcpParams`, ACP runtime inputs, and ACP docs have no `skills` field or rendered skill-context carrier. This is not a bug reproduction because the item requests a new capability.

## Solution Assessment

Unclear: explicit allowlisted injection could be a maintainable first step, but automatic/default injection and cross-backend support need maintainer product and security decisions. The current ACP path deliberately forwards plain task text and attachments only.
