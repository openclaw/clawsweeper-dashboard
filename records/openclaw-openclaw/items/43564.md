## Summary

Keep open: current main still has no ACP skill-context injection path, and the request needs maintainer product/security API design rather than conservative cleanup.

## What This Changes

Request to let `sessions_spawn({ runtime: "acp" })` inject selected, automatic, or default OpenClaw skill context into spawned ACP harness sessions.

## Best Possible Solution

Define one maintainer-approved ACP skill-context contract first: preferably explicit allowlisted skill selection, no implicit tool/env grants, documented session persistence semantics, and focused tests across `sessions_spawn`, ACP runtime types, acpx/Codex behavior, and docs before considering automatic/default injection.

## Reproduction Assessment

Yes, for the source-level feature gap: current `sessions_spawn` and ACP runtime types accept task text, runtime options, and attachments, but no `skills` field or skill-context carrier. This is not a bug reproduction because the item requests a new capability.

## Solution Assessment

Unclear: the broad proposal is useful, but automatic/default injection is not yet safely specified. The narrowest maintainable path is a maintainer-approved explicit skill-selection contract before adding relevance detection or defaults.
