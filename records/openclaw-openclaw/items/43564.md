## What This Changes

The issue requests optional ACP session skill-context injection for spawned agents through explicit, automatic, or default skill selection.

## Best Possible Solution

Define a narrow opt-in ACP skill-context contract that reuses existing skill snapshots and filters, preserves ACP isolation, documents redaction and token behavior, and starts with explicit skill lists before auto/default injection.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main lacks a `sessions_spawn` skill field and that ACP dispatch currently sends only the task text into the ACP turn path.

## Solution Assessment

Unclear until maintainer product and security scoping is done. An explicit opt-in skill list looks like the safest starting point; automatic relevance and default injection should wait for a broader policy decision.
