## Summary

Keep open: current main and v2026.6.9 still lack an ACP skill-selection parameter or skill-context carrier, and the remaining work needs maintainer decisions for prompt lifetime, redaction/audit, persistence, and external harness authority.

## What This Changes

Request to let `sessions_spawn(runtime: "acp")` optionally inject selected OpenClaw skill context into external ACP session prompts.

## Best Possible Solution

Define a maintainer-approved ACP skill-context contract first: explicit allowlisted selection, bounded and redacted rendered context, observability fields, no OpenClaw-only tool/env grants, resume semantics, docs, and focused tests.

## Reproduction Assessment

Yes, as a source-level feature gap: current `sessions_spawn`, ACP spawn params, ACP runtime inputs, and ACP docs have no `skills` field or skill-context carrier. This is not a bug reproduction because the item requests a new capability.

## Solution Assessment

Unclear: the requested direction identifies a real gap, but automatic/default injection is not yet a safe implementation shape. The narrowest maintainable path is an explicit allowlisted ACP skill-selection contract before relevance detection or default skill sets.
