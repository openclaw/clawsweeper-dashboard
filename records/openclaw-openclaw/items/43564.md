## Summary

Keep open. Current main still has no `sessions_spawn` skill-selection field or ACP skill prompt-injection path: ACP turns are dispatched with the task text before the normal skill snapshot path runs. This is a new ACP/API and skill-policy feature, not conservative cleanup.

## What This Changes

Requested support for injecting selected, auto-detected, or default OpenClaw skill content into ACP-spawned agent sessions.

## Best Possible Solution

Define a narrow, opt-in ACP spawn contract that reuses existing skill filtering/snapshot code, injects only prompt-safe rendered skill content, documents token/security behavior, and adds focused schema, dispatch, and ACP-runtime tests.

## Reproduction Assessment

Not applicable. This is a feature request rather than broken current behavior; source inspection confirms current main lacks the requested skill-selection and ACP prompt-injection path.

## Solution Assessment

Unclear. The direction is plausible, but maintainers should choose the API and safety policy first; an explicit opt-in skill list is the narrowest likely starting point, while auto/default injection is broader product behavior.
