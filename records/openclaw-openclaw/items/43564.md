## Summary

Keep open: current main and v2026.6.10 still do not expose a `sessions_spawn(runtime: "acp")` skill selector or ACP skill-context carrier, and the request needs a maintainer decision because it would send local skill material into external harness prompts.

## What This Changes

Request to add an optional ACP skill-context injection mechanism for spawned sessions, with explicit, automatic, or default skill selection.

## Best Possible Solution

Define a maintainer-approved ACP skill-context contract first: explicit allowlisted selection, bounded and redacted rendered context, observability fields, no OpenClaw-only tool/env grants, resume semantics, docs, and focused tests.

## Reproduction Assessment

Yes, as a source-level feature gap: current main and v2026.6.10 expose no ACP `skills` selector, no skill-context field, and no ACP runtime skill carrier. No live bug reproduction applies because this is a feature request.

## Solution Assessment

Unclear: explicit skill-list injection is likely the safest first slice, but automatic/default injection is not the best path until maintainers approve security, observability, backend, and resume semantics.
