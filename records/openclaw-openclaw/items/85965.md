## Summary

Keep this PR open: the active-agent scoping fix is still needed, but the latest branch is not merge-ready because it can reuse a picker result loaded for a previous active agent after a session/agent switch.

## What This Changes

The PR scopes Control UI chat session picker `sessions.list` requests to the active agent, renders popover rows through the filtered session option projection, seeds/paginates scoped picker pages, and adds chat view regression tests.

## Best Possible Solution

Keep the scoped request and filtered row projection, but associate picker results with the active agent or clear/reload them on session switches before merging.

## Reproduction Assessment

Yes. Current main is source-reproducible because the picker request omits `agentId` and renders raw rows; the remaining PR bug is source-reproducible from a stale `chatSessionPickerResult` surviving a session/agent switch.

## Solution Assessment

No, not yet. The scoped request and filtered projection are the right direction, but the cached picker result needs active-agent ownership or invalidation before this is the narrow maintainable fix.
