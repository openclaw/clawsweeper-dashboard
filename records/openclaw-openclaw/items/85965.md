## Summary

Keep this PR open for a small fix. The latest head addresses the original active-agent scoping bug and the earlier agent-switch cache issue, but it still leaves a source-reproducible picker state regression when a user clears an active search while the popover stays open.

## What This Changes

The PR scopes Control UI chat session picker requests to the active agent, renders picker rows through filtered session options, resets picker state on session switches, and adds chat view regression tests.

## Best Possible Solution

Keep the active-agent scoped request and filtered-row rendering, but make clearing search reload or restore the scoped default picker page so search, clear, pagination, and agent-switch flows share one state boundary.

## Reproduction Assessment

Yes. Current main source omits agentId from picker sessions.list requests and renders raw result rows; the remaining PR blocker is also reproducible from source because clearing search nulls the picker result while the open picker refuses to fall back or reload.

## Solution Assessment

No. The main fix direction is right, but it is not complete until the clear-search path reloads or restores the active-agent scoped default page.
