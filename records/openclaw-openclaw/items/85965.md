## Summary

Keep this PR open, but it is not merge-ready yet: it fixes the main active-agent scoping path, but the picker result/request state is still not owned by the active agent, so switching agents can reuse stale scoped rows.

## What This Changes

The branch adds active `agentId` to Control UI chat session picker `sessions.list` calls, renders picker rows through filtered session option groups, seeds unsearched scoped picker pages, and adds regression tests.

## Best Possible Solution

Keep the scoped request and filtered projection, but tie picker results and in-flight requests to the active agent or clear/reload them on session switches before merging.

## Reproduction Assessment

Yes. Current main is source-reproducible because the picker request omits `agentId` and renders raw rows; the remaining PR bug is source-reproducible because scoped picker results survive active-agent/session switches.

## Solution Assessment

No, not yet. The scoped request and filtered projection are the right direction, but picker result and request ownership need to include the active agent or be invalidated on session switches.
