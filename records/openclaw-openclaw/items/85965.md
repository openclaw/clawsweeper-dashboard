## Summary

Keep open. This is a focused fix for a source-reproducible Control UI multi-agent session-picker leak, but the current head introduces a clear-search regression: clearing a picker search can leave the open picker empty instead of restoring a scoped active-agent list.

## What This Changes

The PR adds active-agent scoping and filtered-row projection to the Control UI chat session picker, resets picker state on session switch, and adds focused picker regression tests.

## Best Possible Solution

Keep the active-agent scoping approach, but reload or preserve a scoped default picker page after clearing search and add a regression test for that visible flow before landing.

## Reproduction Assessment

Yes. Current main is source-reproducible for the original bug because picker requests omit agentId and the open popover can render unscoped result rows; the remaining PR regression is also source-reproducible from the clear-search and open-picker state branches.

## Solution Assessment

No, not yet. The active-agent scoping and filtered-row projection are the right fix direction, but the patch should also restore a scoped default result after clearing picker search.
