## Summary

## What This Changes

The PR adds active-agent `agentId` scoping to Control UI chat session picker requests, filters picker rows through the session option projection, and adds focused chat view tests.

## Best Possible Solution

Keep the active-agent scoping, but make picker initialization and pagination derive from one consistently scoped result set before merge.

## Reproduction Assessment

Yes. Current main source omits `agentId` and renders raw picker rows, and the remaining PR issue is source-reproducible from the fallback to unscoped `state.sessionsResult` before scoped load-more.

## Solution Assessment

No, not yet. The scoping/filtering approach is maintainable, but pagination must use a consistently scoped result before this is the safest fix.
