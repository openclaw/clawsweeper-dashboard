## Summary

The PR should stay open: current main has the reported unscoped picker bug and this branch is mostly the right fix, but the final head still leaves the picker empty after clearing a non-empty search because the open-popover fallback is disabled and no reload runs.

## What This Changes

The PR scopes the Control UI chat session picker to the active agent, filters picker rows through existing session option projection, resets picker cache on session switches, and adds regression coverage.

## Best Possible Solution

Keep the active-agent scoping and filtered-row projection, but make clearing a picker search reload or restore the active-agent-scoped default page while the popover is open.

## Reproduction Assessment

Yes. Current main is source-reproducible for the original bug because picker requests omit agentId and the open popover renders unscoped result rows; the remaining PR regression is source-reproducible from the clear-search and open-picker state branches.

## Solution Assessment

No. The agent-scoping direction is the right narrow fix, but the clear-search state path needs to restore a scoped default page before this is the best merge shape.
