## Summary

Keep this PR open: it fixes the main unscoped request/raw-row rendering bug, but the picker result cache is still not tied to the active agent/session and can be reused after switching agents.

## What This Changes

## Best Possible Solution

Keep the active-agent scoped request and filtered-row rendering, but clear or key the picker result/query by active agent/session and add a regression test for switching agents after a loaded picker result.

## Reproduction Assessment

Yes: current main omits `agentId` from picker `sessions.list` requests and renders raw result rows, matching the linked multi-agent leakage report. The remaining PR blocker is also source-reproducible from the cached picker result surviving `switchChatSession`.

## Solution Assessment
