## Summary

Keep open: the OpenAI cache-retention passthrough bug is still source-reproducible on current main, but this PR is now stale against the current resolver path and lacks after-fix real behavior proof.

## What This Changes

This PR extracts cache-retention resolution into a provider-neutral helper, forwards explicit `cacheRetention` for OpenAI/OpenAI-Codex providers, and adds focused resolver tests.

## Best Possible Solution

Rebase the fix onto current `src/agents/pi-embedded-runner/prompt-cache-retention.ts`, allow explicit direct-OpenAI retention without adding defaults, preserve current modelApi/modelId cache semantics, and prove it with focused tests plus redacted real OpenAI behavior output.

## Reproduction Assessment

Yes, from source inspection. A direct OpenAI model with explicit `cacheRetention: "long"` reaches the current resolver, but the resolver returns `undefined` before stream options can carry the value downstream.

## Solution Assessment

No, not as-is. The intended behavior is the right fix, but this branch should update the current `prompt-cache-retention.ts` resolver instead of adding a provider-only helper from an obsolete file layout.
