## Summary

Keep open: the documented OpenAI cache-retention passthrough gap is still real on current main, but this PR should not merge as written because it replaces current model-aware cache routing with provider-only logic and still lacks contributor real behavior proof.

## What This Changes

The PR adds a provider-only cache-retention helper and tests to pass explicit `cacheRetention` through for OpenAI/OpenAI-Codex while re-exporting the helper from the Anthropic wrapper module.

## Best Possible Solution

Rebase the OpenAI passthrough into current `src/agents/pi-embedded-runner/prompt-cache-retention.ts`, preserve `modelApi`/`modelId` routing for Anthropic, Bedrock, and Google, add focused regression coverage, and collect redacted direct OpenAI runtime proof.
