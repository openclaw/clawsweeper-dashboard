## Summary

Keep open: the OpenAI cache-retention passthrough bug is still present on current main, but this PR is stale against the current resolver and would bypass model/API eligibility rules that now protect Bedrock, custom Anthropic, and Google prompt-cache behavior.

## What This Changes

The PR extracts cache-retention resolution into a new provider-neutral helper, re-exports the old Anthropic wrapper symbol, imports the helper from extra-params, and adds focused OpenAI/OpenAI-Codex passthrough tests.

## Best Possible Solution

Extend the existing prompt-cache-retention resolver to pass explicitly configured cacheRetention through for direct openai and openai-codex without adding an implicit default, while preserving current Anthropic-family, Bedrock model/ARN, custom anthropic-messages, and Google eligibility rules.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: resolveCacheRetention({ cacheRetention: 'long' }, 'openai', 'openai-responses', 'gpt-5') returns undefined on current main, and applyExtraParamsToAgent only forwards cacheRetention when that resolver returns a value.

## Solution Assessment

No. The requested behavior is correct, but this PR is not the best current-main implementation because the provider-only helper would discard current modelApi/modelId eligibility gates; extending the existing resolver in place is narrower and safer.
