## Summary

Keep open: the requested OpenAI passthrough is still valid on current main, but this PR is stale against the current model-aware prompt-cache-retention resolver and would regress existing provider eligibility rules if merged as-is.

## What This Changes

This PR extracts cache-retention resolution into a provider-neutral helper, re-exports it from the Anthropic wrapper module, imports it from extra-params, and adds focused tests for explicit OpenAI/OpenAI-Codex cacheRetention passthrough.

## Best Possible Solution

Extend the current prompt-cache-retention resolver to pass explicitly configured cacheRetention through for direct openai and openai-codex without adding an implicit default, while preserving the existing Anthropic-family, Bedrock model/ARN, custom anthropic-messages, and Google gates.

## Reproduction Assessment

## Solution Assessment

No. The requested behavior is correct, but this PR is not the best current-main implementation because its provider-only helper drops current modelApi/modelId eligibility gates; updating the existing resolver in place is narrower and safer.
