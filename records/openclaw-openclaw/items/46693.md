## Summary

Keep open: current main still filters explicit OpenAI cache retention before the OpenAI transport, but this branch is not merge-ready because its provider-only resolver predates the current model-aware resolver and lacks real behavior proof.

## What This Changes

The PR extracts cache-retention resolution into a new helper, passes explicit `cacheRetention` through for OpenAI/OpenAI-Codex providers, re-exports the helper from the old Anthropic wrapper, and adds focused resolver tests.

## Best Possible Solution

Implement the explicit OpenAI/OpenAI-Codex passthrough inside current `prompt-cache-retention.ts`, preserve existing model-aware routes, add regression coverage, and require redacted real runtime proof before merge.

## Reproduction Assessment

Yes, from source inspection. Current docs say direct OpenAI `cacheRetention: "long"` should become `prompt_cache_retention: "24h"`, but current main's resolver returns `undefined` for OpenAI before the OpenAI transport can map it.

## Solution Assessment

No for the PR as written. The direction is right, but the change should be rebased into current `prompt-cache-retention.ts` rather than adding the older provider-only resolver.
