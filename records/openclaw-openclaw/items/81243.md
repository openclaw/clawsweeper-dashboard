## Summary

Keep open: the feature is still useful and not implemented on current main, but this PR still has a blocking URL-only routing bug in the public message-tool path.

## What This Changes

The PR adds a Discord message `fetch` action for single-message retrieval by `channelId` plus `messageId` or by Discord message URL, with shared action metadata, schema updates, tests, skill docs, and a repro script.

## Best Possible Solution

Expose the `fetch` URL alias through the actual Discord plugin action surface, add public message-tool regression coverage for URL-only fetch, and then land this or a narrow equivalent after maintainers accept the shared public action contract.

## Reproduction Assessment

Yes. Current main lacks the public fetch action by source inspection, and the PR's remaining URL-only bug is source-reproducible because target normalization reads aliases from the exported plugin action wrapper, which does not expose the alias added in the PR.

## Solution Assessment

No, not as currently patched. The general direction is right, but the URL-only alias must be exposed through `extensions/discord/src/channel.ts` or another real plugin action surface before the public message tool can reliably route URL fetches.
