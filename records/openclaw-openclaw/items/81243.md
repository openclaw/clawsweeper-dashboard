## Summary

Keep open: this is a focused Discord feature PR with sufficient real behavior proof, but the latest head still misses the URL-only public message-tool path because the new target alias is added to the implementation object but not exposed by the Discord channel plugin wrapper.

## What This Changes

This PR adds a Discord `message` tool `fetch` action for retrieving one message by `channelId` plus `messageId` or by Discord message URL, with shared action metadata, schema/docs/tests, and a repro script.

## Best Possible Solution

Forward `messageActionTargetAliases` through the Discord channel wrapper and add focused normalization coverage for URL-only and target/direct-ID fetch, then land this PR or a narrow equivalent.

## Reproduction Assessment

Yes. Source inspection shows current main lacks the public Discord `fetch` action, and the PR's remaining URL-only gap is source-reproducible through the normalizer's bootstrap-plugin alias lookup.

## Solution Assessment

No, not yet. Exposing the existing Discord fetch backend is the right narrow direction, but the URL alias must be exposed through the channel wrapper before the public message-tool path is complete.
