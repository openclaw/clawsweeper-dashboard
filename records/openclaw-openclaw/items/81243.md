## Summary

Keep open: the feature is still useful and the PR mostly wires the existing Discord fetch runtime into the public message tool, but the current patch has a target-normalization regression that should be fixed before merge.

## What This Changes

The PR adds a Discord message `fetch` action for single-message lookup by direct IDs or Discord message URL, with shared action metadata, schema/docs updates, tests, and a live repro script.

## Best Possible Solution

Land this PR or a narrow equivalent after preserving URL-only fetch while also accepting standard `target` or `channelId` routing for direct message fetches.

## Reproduction Assessment

Yes for the feature gap: current main omits `fetch` from the shared action list and Discord discovery, while the existing internal runtime already supports `fetchMessage`. The target-mode regression is source-reproducible through `normalizeMessageActionInput` and `applyTargetToParams`.

## Solution Assessment

No as currently patched: exposing the existing Discord fetch runtime is the right direction, but `fetch: "none"` rejects standard `target + messageId` direct fetch calls. The safer fix is an optional-target path or equivalent normalization that supports both URL-only and channel-scoped fetch.

## Review Findings

Overall correctness: patch is incorrect

Direct fetch is channel-scoped, so calls using the standard `target + messageId` shape need to keep working while URL-only fetch remains targetless.
