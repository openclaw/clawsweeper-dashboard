## Summary

Keep open: current main still lacks the public Discord `fetch` action, but this draft PR has blocking public-runner/schema issues and incomplete real behavior proof for the promised URL path.

## What This Changes

The PR adds a Discord message `fetch` action for direct message IDs or Discord message URLs, exposes it in shared action metadata and Discord action discovery, adds tests, a Discord skill note, and a manual repro script.

## Best Possible Solution

Revise this PR, or a narrow replacement, to expose Discord fetch through the public message tool with targetless URL handling, `url`/`messageLink` schema coverage, focused runner tests, and redacted live public-path proof.

## Reproduction Assessment

Yes. Source inspection shows `runMessageAction` normalizes before Discord dispatch, so the PR's `fetch: "to"` setting makes URL-only fetch fail before the Discord handler can parse the URL.

## Solution Assessment

No. Reusing the existing Discord fetch runtime is the right direction, but the public contract needs targetless URL handling, URL schema fields, runner coverage, and proof of the public path before merge.

## Review Findings

That leaves the promised URL form undiscoverable through the public tool contract, so add those fields and schema/runner coverage with the action exposure.
