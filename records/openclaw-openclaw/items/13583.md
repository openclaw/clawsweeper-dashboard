## Summary

Keep open. Current main has adjacent input-gate, pre-tool, finalization, reply-dispatch, and outbound cancellation hooks, but it still lacks the requested fail-closed responseGate/two-phase validator surface with mandatory tool-call accounting before final answer delivery.

## What This Changes

The issue requests a fail-closed pre-response enforcement layer that blocks final output until mandatory tool calls and validators have run and passed.

## Best Possible Solution

Define a versioned core/plugin API for mandatory tool ledgers, typed response validators, fail-closed final-response gating, bounded retry/fallback behavior, and audit records, with domain-specific policy packs shipped as plugins or ClawHub packages.

## Reproduction Assessment

Not applicable. This is a feature/API request rather than a broken existing contract, and source/docs inspection on current main verifies the requested response-gate surface is absent.

## Solution Assessment

Yes for the triage direction. Keeping this open for explicit core/plugin API and security design is the narrowest maintainable path; treating existing input-gate, pre-tool, finalization, or message-send hooks as the requested hard gate would leave mandatory-tool accounting and fail-closed response delivery undefined.
