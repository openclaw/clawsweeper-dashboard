## Summary

Keep open. Current main and the latest release still only provide adjacent tool-execution gates and final-answer revision hooks; they do not provide the requested fail-closed response gate with mandatory tool-call accounting, typed validators, two-phase planning, bounded fallback, or audit records.

## What This Changes

The issue requests a pre-response enforcement layer that blocks final answers until configured mandatory tool calls and validators have run and passed.

## Best Possible Solution

Keep this as the canonical response-gate design item: define a versioned core/plugin API for mandatory tool-call accounting, typed validators, fail-closed final-response gating, bounded retry/fallback behavior, and audit records, with domain policies shipped as plugins or ClawHub packages.

## Reproduction Assessment

Not applicable as a conventional bug reproduction, but yes for the source-level gap: current main has no responseGate/mandatory-tool validator contract, and the closest finalize hook continues on missing or failed hooks.

## Solution Assessment

Yes for the keep-open direction. The maintainable solution is a reviewed core/plugin contract that composes with existing tool and delivery hooks, not another prompt rule or standalone policy plugin alone.
