## Summary

Keep open. Current main has adjacent pre-run, pre-tool, finalization, reply-dispatch, and outbound cancellation hooks, but it still lacks the requested fail-closed responseGate/two-phase validator surface for mandatory tool-call accounting before final answer delivery.

## What This Changes

Add a pre-response enforcement layer that blocks final answer delivery until configured mandatory tool calls and validators pass.

## Best Possible Solution

Define a versioned core/plugin API for mandatory tool ledgers, typed response validators, fail-closed final-response gating, bounded retry/fallback behavior, and audit records, with domain-specific policy packs shipped as plugins or ClawHub packages.

## Reproduction Assessment

Not applicable. This is a feature/API request rather than a broken existing contract; source and docs inspection verify the requested response-gate surface is absent on current main.

## Solution Assessment

Yes for the current triage direction. Keeping this open for a versioned core/plugin API design is the narrowest maintainable path; treating existing pre-run, pre-tool, finalization, or message-send hooks as the requested hard gate would leave response-delivery policy undefined.
