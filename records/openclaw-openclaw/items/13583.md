## Summary

Keep open. Current main still has adjacent pre-tool blocking, final-answer revision, reply-dispatch, and visible-reply suppression hooks, but it does not implement the requested fail-closed responseGate with mandatory tool-call accounting, typed validators, two-phase planning, retry/fallback policy, or audit records.

## What This Changes

Add a pre-response enforcement layer that blocks final answer delivery until configured validators and mandatory tool-call rules pass.

## Best Possible Solution

Keep this as the canonical response-gate policy-as-code design item and define a versioned core/plugin API for mandatory tool ledgers, typed response validators, fail-closed delivery gating, bounded retry/fallback behavior, and audit records.

## Reproduction Assessment

## Solution Assessment

Yes. Keeping this open for maintainer product/security design is the narrowest maintainable path; treating before_tool_call, before_agent_finalize, reply_dispatch, or message_sending as the requested hard gate would leave core policy semantics unspecified.
