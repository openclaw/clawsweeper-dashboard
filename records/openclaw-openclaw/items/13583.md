## Summary

Keep open: current main has adjacent pre-tool blocking, Codex Stop/before-finalize revision hooks, reply dispatch hooks, and outbound cancellation paths, but it still lacks the requested fail-closed responseGate with mandatory tool-call accounting, typed validators, two-phase planning, bounded fallback semantics, and audit records. The linked APort Guardrails, openclaw-governance, and claude-rule-enforcer comments are useful complementary plugin evidence, but they do not remove the core/plugin API gap.

## What This Changes

The issue requests a hard pre-response enforcement layer that blocks final answers or external/public actions until mandatory tool calls and policy validators have run and passed.

## Best Possible Solution

Keep this open as the canonical response-gate policy-as-code design item: core should expose a versioned fail-closed response-gate API only after maintainer product/security review, while domain-specific validators and policy packs should live in plugins or ClawHub packages.
