## Summary

Keep open. Current main has adjacent pre-tool, before-run, finalization, reply-dispatch, and message-delivery hooks, but it still lacks the requested fail-closed response gate with mandatory tool-call accounting, typed validators, two-phase planning, fallback/retry semantics, and audit records before final delivery.

## What This Changes

Request a fail-closed pre-response enforcement layer that gates final output on mandatory tool calls, validators, retries/fallbacks, and audit records.

## Best Possible Solution

Define a versioned core/plugin response-gate API for mandatory tool ledgers, typed validators, fail-closed final-response delivery, bounded retry/fallback behavior, and audit records, while keeping domain-specific policy packs in plugins or ClawHub packages.

## Reproduction Assessment

Not applicable. This is a feature/API request rather than a broken existing contract, and source/docs inspection shows the requested response-gate surface is absent on current main.

## Solution Assessment

Yes for keeping this open as a design item, but not for treating current hooks as the final solution. Existing pre-tool, before-run, finalization, and message-delivery hooks are adjacent controls; the requested hard gate needs an explicit core/plugin API and security review.
