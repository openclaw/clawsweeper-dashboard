## Summary

The central gap remains on current main: plugins can block tools, request final-answer revision, or cancel outbound messages, but there is no generic fail-closed contract that accounts for required tool completion and validator results before final delivery. The issue should remain the canonical product/security design item rather than be closed in favor of the narrower simulated-tool report.

## What This Changes

Requests a fail-closed pre-response policy layer that requires configured tools and validators to pass before a final answer or external action can be delivered.

## Best Possible Solution

Design a versioned generic core/plugin response-authorization contract with runtime-owned tool-result accounting, typed validator decisions, bounded retries, deterministic fail-closed recovery, and audit events, while leaving finance, security, and other domain rules in external plugins or ClawHub packages.

## Reproduction Assessment

Yes, at source level. Current main exposes pre-tool blocking, final-answer revision, and outbound cancellation, but no contract can require and verify configured tool completion before final delivery; discussion also includes a concrete OpenClaw 2026.5.28 external-action incident.

## Solution Assessment

The requested outcome is valid, but the proposed full two-phase protocol is not yet proven to be the narrowest solution. The best direction is a versioned, harness-neutral response-authorization API with runtime-owned accounting rather than embedding domain-specific validators or commands in core.

- Fail-closed behavior can suppress all assistant output or strand runs when a validator/plugin fails, so fallback, timeout, retry, and operator-recovery semantics require explicit product and security decisions.
- A new plugin API and configuration surface would become compatibility-sensitive and must work consistently across embedded, native/Codex, and outbound delivery paths.
- Mandatory tool accounting must distinguish attempted, denied, failed, timed-out, and successful calls without letting model-authored text or plugin state falsely satisfy a gate.
