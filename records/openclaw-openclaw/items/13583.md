## Summary

Keep open. Current main still has adjacent tool blocking, final-answer revision, reply dispatch, and outbound cancellation hooks, but it does not implement the requested fail-closed response gate with mandatory tool-call accounting, typed validators, two-phase planning, retry/fallback semantics, and audit records before final delivery.

## What This Changes

The issue requests a pre-response enforcement layer that blocks final answers until required tool calls and policy validators have run and passed.

## Best Possible Solution

Define a versioned core/plugin response-gate API for mandatory tool ledgers, typed validators, fail-closed final-response delivery, bounded retry/fallback behavior, and audit records, while keeping domain-specific policy packs in plugins or ClawHub packages.

## Reproduction Assessment

Not applicable. This is a feature/API request rather than a broken existing contract, and source/docs inspection shows the requested response-gate surface is absent on current main.

## Solution Assessment

Yes for the triage direction, but not yet for a concrete implementation. Keeping this open for explicit core/plugin API and security design is safer than treating existing finalization, pre-tool, or message-send hooks as the requested hard gate.
