## Summary

Keep open: current main and v2026.6.5 still have adjacent pre-tool, finalization-revision, reply-dispatch, and outbound-cancel hooks, but not the requested fail-closed response gate with mandatory tool-call accounting, typed validators, two-phase planning, fallback policy, and audit records.

## What This Changes

Request a pre-response enforcement layer that blocks final output until required tools and configured validators have run and passed.

## Best Possible Solution

Keep this open as the canonical response-gate policy-as-code design item, with a versioned core/plugin API for mandatory tool-call accounting, typed response validators, fail-closed final-response gating, bounded retry/fallback behavior, exact-content approval gates, and audit records.

## Reproduction Assessment

Yes for the source-level gap, not as a live rerun here. Current main exposes advisory prompts, pre-tool policy hooks, final-answer revision, and delivery-cancel hooks, but no fail-closed responseGate or validator/mandatory-tool ledger contract.

## Solution Assessment

No as a one-shot fix PR; the best path is a maintainer-approved, versioned core/plugin API before implementation. Domain-specific policies should layer on that API as plugins or ClawHub packages rather than hardcoding quant/security rules in core.
