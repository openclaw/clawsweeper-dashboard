## Summary

Keep open. Current main still has adjacent input, pre-tool, final-answer revision, reply-dispatch, and outbound-message hooks, but it does not provide the requested fail-closed response gate with mandatory tool-call accounting, typed validators, two-phase planning, bounded fallback, and audit records.

Request a pre-response enforcement layer that blocks final answer delivery until required tool calls and validators pass, with configuration, retries/fallbacks, and audit records.

## Best Possible Solution

Keep this open as the canonical response-gate policy-as-code design item, with a versioned core/plugin API for mandatory tool-call accounting, typed response validators, fail-closed final-response gating, bounded fallback behavior, and audit records.
