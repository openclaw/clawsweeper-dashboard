## Summary

Keep open. Current main has useful adjacent hooks for pre-run blocking, tool-call policy, final-answer revision, and outbound cancellation, but it still does not provide the requested fail-closed response gate with mandatory tool-call accounting, typed validators, two-phase planning, fallback policy, and audit records before final delivery.

## What This Changes

The issue requests a configurable pre-response enforcement layer that can block final output until required tools and validators have run and passed.

## Best Possible Solution

Keep this as the canonical response-gate policy-as-code design item: define a versioned core/plugin API for mandatory tool ledgers, typed response validators, fail-closed final-response delivery, bounded retry/fallback behavior, and audit records, while leaving domain policy packs to plugins or ClawHub packages.
