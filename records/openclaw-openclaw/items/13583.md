## Summary

Keep open. Current main has adjacent pre-tool, before-run, finalization, reply-dispatch, and outbound cancellation hooks, but it still lacks the requested fail-closed response gate with mandatory tool-call accounting, typed validators, two-phase planning, fallback/retry semantics, and audit records before final delivery. The discussion usefully links complementary plugin work while preserving the core gap that plugins cannot yet mechanically prevent final response delivery until required tools and validators pass.

## What This Changes

The issue requests a fail-closed pre-response enforcement layer that can require tool calls and validator success before final output delivery.

## Best Possible Solution

Define a versioned core/plugin response-gate API for mandatory tool ledgers, typed validators, fail-closed final-response delivery, bounded retry/fallback behavior, and audit records, while keeping domain-specific policy packs in plugins or ClawHub packages.

## Reproduction Assessment

Not applicable. This is a feature/API request rather than a broken existing contract, and source/docs inspection shows the requested response-gate surface is absent on current main.

## Solution Assessment

Yes for keeping this open as a design item, but no for treating current hooks as a complete solution. Existing pre-tool, before-run, finalization, and message-delivery hooks are adjacent controls; the requested hard gate needs an explicit core/plugin API and security review.
