## Summary

Keep open. Current main still has adjacent pre-tool, before-run, finalization, reply-dispatch, and outbound cancellation hooks, but it lacks the requested fail-closed response gate with mandatory tool-call accounting, typed validators, two-phase planning, retry/fallback policy, and audit records before final delivery.

## What This Changes

Add a configurable pre-response enforcement layer that prevents final answer delivery until mandatory tools and validators have run and passed.

## Best Possible Solution

Define a versioned core/plugin response-gate API for mandatory tool ledgers, typed validators, fail-closed final-response delivery, bounded retry/fallback behavior, and audit records, while keeping domain-specific policy packs in plugins or ClawHub packages.

## Reproduction Assessment

Not applicable. This is a feature/API request rather than a broken existing contract, and current-main source/docs inspection shows the requested response-gate surface is absent.

## Solution Assessment

Yes for keeping this open as a design item, but no for treating existing hooks as a complete solution. Existing pre-tool, before-run, finalization, and message-delivery hooks are adjacent controls; the requested hard gate needs explicit core/plugin API and security review.
