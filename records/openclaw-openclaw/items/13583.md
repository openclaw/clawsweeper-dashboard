## Summary

Keep open. Current main and the latest release still provide only adjacent pre-tool enforcement plus final-answer revision/cancellation hooks; the requested fail-closed response gate with mandatory tool-call accounting, validators, bounded fallback, and audit records remains a security-sensitive core/plugin API gap.

## What This Changes

The issue requests a pre-response enforcement layer that blocks final output until mandatory tools and policy validators have run and passed.

## Best Possible Solution

Keep this as the canonical design item for a versioned core/plugin response-gate API, with domain policy packs implemented as plugins or ClawHub packages after the runtime seam is approved.

## Reproduction Assessment

Yes, at source level. Current main has adjacent pre-tool and message/finalization hooks, but source and docs show no fail-closed responseGate, mandatory-tool ledger, or typed validator API; I did not run a live canary reproduction.

## Solution Assessment

No one-shot patch is the best solution yet. The maintainable path is a maintainer-reviewed core/plugin API design that separates generic response-gate mechanics from plugin-owned domain policies.
