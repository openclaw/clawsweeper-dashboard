## Summary

Keep open: current main and v2026.6.8 still have adjacent pre-tool blocking, final-answer revision, and outbound pre-send hooks, but not the requested fail-closed response gate with mandatory tool-call accounting, typed validators, two-phase planning, fallback semantics, and audit records.

## What This Changes

The issue requests a hard pre-response enforcement layer that prevents final answers until mandatory tool calls, validators, fallback policy, and audit rules pass.

## Best Possible Solution

Keep this open as the canonical response-gate policy-as-code design item: add a versioned core/plugin API for mandatory-tool accounting, typed validators, fail-closed final-response gating, bounded retry/fallback behavior, and audit records, with domain policy packs implemented as plugins or ClawHub packages.

## Reproduction Assessment

Yes, at source level: current main and v2026.6.8 lack the requested responseGate/require_tools/two_phase contract, and the closest before_agent_finalize path continues when hooks are absent or fail.

## Solution Assessment

Yes for keeping this open; no for a one-shot fix PR. The maintainable path is maintainer-reviewed core/plugin API design plus contract tests, not an automated patch that chooses fail-closed policy semantics by itself.
