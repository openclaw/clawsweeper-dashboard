## Summary

Keep open. Current main has adjacent pre-tool blocking, outbound message cancellation, final-answer revision, and Codex Stop relay hooks, but it still lacks the requested fail-closed response gate with mandatory tool-call accounting, typed validators, two-phase planning, retry/fallback policy, and audit records before final delivery.

## What This Changes

The issue requests a pre-response enforcement layer that blocks final output until required tools and validators have run and passed.

## Best Possible Solution

Keep this as the canonical response-gate policy-as-code design item: define a versioned core/plugin API for mandatory tool ledgers, typed response validators, fail-closed final-response delivery, bounded retry/fallback behavior, and audit records, while leaving domain policy packs to plugins or ClawHub packages.

## Reproduction Assessment

Not applicable. This is a feature/API request rather than a broken existing contract, and current-main source/docs inspection shows the requested response-gate surface is absent.

## Solution Assessment

Yes for keeping this open as a design item, but no for treating existing hooks as a complete solution. Existing pre-tool, before-run, finalization, and message-delivery hooks are adjacent controls; the requested hard gate needs explicit core/plugin API and security review.
