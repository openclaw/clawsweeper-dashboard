## Summary

Keep open. Current main and the latest release still provide adjacent pre-tool blocking, final-answer revision, reply-dispatch, and outbound cancellation hooks, but not the requested fail-closed responseGate with mandatory tool-call accounting, typed validators, two-phase planning, fallback policy, and audit records.

## What This Changes

The issue requests a hard pre-response enforcement layer that blocks final answers until mandatory tool calls and configured validators have passed.

## Best Possible Solution

Keep this as the canonical response-gate policy-as-code design item: define a versioned core/plugin API for mandatory tool-call accounting, typed response validators, fail-closed final-response gating, bounded retry/fallback behavior, and audit records, with domain-specific policies shipped as plugins or ClawHub packages.

## Reproduction Assessment

Yes for the source-level feature gap, but not as a conventional bug reproduction. Current main and v2026.5.20 have no responseGate or mandatory-tool validator contract, and the closest finalize hook continues on missing or failed hooks.

## Solution Assessment

Yes for the keep-open direction. The maintainable solution is a reviewed core/plugin contract that composes with existing tool, finalization, native hook relay, and delivery hooks, not another prompt rule or standalone policy plugin alone.
