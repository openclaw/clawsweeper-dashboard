## Summary

Keep open. Current main has adjacent input-gate, pre-tool, finalization, reply-dispatch, and outbound-cancel hooks, but it still lacks the requested fail-closed response gate with mandatory tool-call accounting, typed validators, two-phase planning, fallback/retry policy, and audit records before final answer delivery.

## What This Changes

The issue requests a pre-response enforcement layer that prevents final output until configured validators and mandatory tool calls have run and passed.

## Best Possible Solution

Define a versioned core/plugin API for mandatory tool ledgers, typed response validators, fail-closed final-response gating, bounded retry/fallback behavior, and audit records, with domain-specific policies shipped as plugins or ClawHub packages.

## Reproduction Assessment

Not applicable. This is a feature/API request rather than a broken existing contract, and source/docs inspection shows the requested response-gate surface is absent on current main.

## Solution Assessment

Yes for the triage direction. Keeping this open for explicit core/plugin API and security design is the narrowest maintainable path; treating existing input-gate, pre-tool, finalization, or message-send hooks as the requested hard gate would leave mandatory-tool accounting and fail-closed response delivery undefined.
