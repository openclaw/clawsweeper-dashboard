## Summary

Keep open: current main and the latest release still provide only adjacent pre-tool, final-answer revision, reply-dispatch, and message-cancellation hooks, not the requested fail-closed response gate with mandatory tool-call accounting, typed validators, two-phase planning, fallback policy, and audit records.

## What This Changes

Request a fail-closed pre-response enforcement layer that prevents final output until required tools and policy validators have run and passed.

## Best Possible Solution

Keep this as the canonical response-gate policy-as-code design item: define a versioned core/plugin API for mandatory tool-call accounting, typed response validators, fail-closed final-response gating, bounded retry/fallback behavior, and audit records, with domain-specific policies shipped as plugins or ClawHub packages.

## Reproduction Assessment

Yes, for the source-level feature gap rather than a conventional bug reproduction. Current main and v2026.5.22 lack a responseGate or mandatory-tool validator contract, and the closest finalize hook continues on missing or failed hooks.

## Solution Assessment

Yes for the keep-open direction, but the exact API needs maintainer design review. The maintainable solution is a reviewed core/plugin contract that composes with existing tool, finalization, native hook relay, and delivery hooks, not another prompt rule or standalone policy plugin alone.
