## Summary

Keep open. Current main still has adjacent input-gate, pre-tool, finalization, reply-dispatch, and outbound-message hooks, but not the requested fail-closed response gate with mandatory tool accounting, typed validators, two-phase planning, retry/fallback semantics, and audit records before final delivery.

## What This Changes

The issue requests a core/plugin API for pre-response hard gates that prevent final answers until configured mandatory tool calls and validators have passed.

## Best Possible Solution

Define a versioned core/plugin API for mandatory tool ledgers, typed response validators, fail-closed final-response gating, bounded retry/fallback behavior, and audit records, while keeping domain-specific policies in plugins or ClawHub packages.

## Reproduction Assessment

Not applicable. This is a feature/API request rather than a broken existing contract, and source/docs inspection shows the requested response-gate surface is absent on current main.

## Solution Assessment

Yes for the triage direction. Keeping this open for explicit core/plugin API and security design is the maintainable path; treating existing input-gate, pre-tool, finalization, or message-send hooks as the requested hard gate would leave mandatory-tool accounting and fail-closed response delivery undefined.
