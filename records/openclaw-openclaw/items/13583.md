## Summary

Keep open. Current main has adjacent pre-tool, pre-run, final-answer revision, message-sending, and Codex native hook relay mechanisms, but it still lacks the requested fail-closed response gate with mandatory tool-call accounting, typed response validators, two-phase planning, bounded retry/fallback policy, and audit records before final delivery.

## What This Changes

The issue requests a configurable pre-response enforcement layer that blocks final answers until mandatory tool calls and validators have run and passed.

## Best Possible Solution

Keep this as the canonical response-gate policy-as-code design item: define a versioned core/plugin API for mandatory tool ledgers, typed response validators, fail-closed final-response delivery, bounded retry/fallback behavior, and audit records, while leaving domain policy packs to plugins or ClawHub packages.

## Solution Assessment

Yes for keeping this open as a design item, but no for treating existing hooks as a complete solution. Existing pre-tool, pre-run, finalization, and message-delivery hooks are adjacent controls; the requested hard gate needs explicit core/plugin API and security review.
