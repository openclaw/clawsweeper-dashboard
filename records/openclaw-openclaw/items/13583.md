## Summary

Keep open: current main still has adjacent pre-tool blocking, final-answer revision, reply dispatch, and outbound-cancel hooks, but not the requested fail-closed response gate with mandatory tool accounting, typed validators, two-phase planning, fallback behavior, and audit records.

## What This Changes

Request to add a fail-closed pre-response enforcement layer that blocks final output until mandatory tool calls and configured validators pass.

## Best Possible Solution

Keep this as the canonical response-gate design item and define a versioned core/plugin API for mandatory tool ledgers, typed validators, fail-closed final-response authorization, bounded retry/fallback behavior, and audit records, with domain policy packs outside core.

## Reproduction Assessment

Yes, at source level rather than as a narrow live bug. Current main lacks a responseGate config/API, the closest finalization hook is fail-open revision-only behavior, and the issue discussion adds real-world examples.

## Solution Assessment

Unclear for the exact design, but a versioned generic core/plugin seam is the right layer. One-off domain validators in core or delivery-only cancellation hooks would not satisfy mandatory tool accounting and fail-closed response authorization.
