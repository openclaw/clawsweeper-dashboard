## Summary

Keep open. Current `main` has adjacent pre-model, pre-tool, final-answer revision, native relay, and outbound cancellation hooks, but it still lacks the requested fail-closed pre-response gate with mandatory tool-call accounting, typed validators, two-phase planning, fallback/retry policy, and audit records.

## What This Changes

The issue requests a configurable fail-closed response-gate layer that prevents final answer delivery until required tools and validators have run and passed.

## Best Possible Solution

Define a versioned core/plugin API for mandatory tool ledgers, typed response validators, fail-closed final-response gating, bounded retry/fallback behavior, and audit records, with domain-specific policy packs shipped as plugins or ClawHub packages.

## Reproduction Assessment

Not applicable. This is a feature/API request rather than a broken existing contract; source and docs inspection verify the requested response-gate surface is absent on current `main`.

## Solution Assessment

Yes for the current triage direction. Keeping this open for a versioned core/plugin API design is the narrowest maintainable path; treating existing pre-tool, pre-run, finalization, or message-send hooks as the requested hard gate would leave response-delivery policy undefined.
