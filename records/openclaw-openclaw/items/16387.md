## Summary

Keep open: current main still forwards configured provider headers as literal strings, the latest release has the same static-header behavior, Fireworks does not add session-affinity headers, and the two related implementation PRs are closed unmerged.

## What This Changes

Request support for allowlisted runtime placeholders in custom provider header config so provider requests can include session-, agent-, and run-specific values.

## Best Possible Solution

Add a documented allowlisted dispatch-time resolver for models.providers.*.headers and models.providers.*.request.headers covering {{session.key}}, {{agent.id}}, and {{run.id}} while preserving existing static header, SecretRef, auth, attribution, retry, and provider-plugin behavior.

## Reproduction Assessment

Yes for the source-level behavior: configured header values such as {{session.key}} are copied and merged as literal strings today. It is not a strict bug reproduction because the request is for a new config capability.

## Solution Assessment

Yes. A narrow allowlisted OpenClaw-level resolver is the maintainable path; a Fireworks-only plugin wrapper would not cover custom provider config or the LiteLLM/Langfuse use case, while arbitrary template evaluation would be too broad.
