Keep this as the canonical session-aware provider-header item until maintainers either land the typed affinity/schema companion PRs for the Fireworks path or define a narrow documented placeholder contract that preserves static headers, SecretRef resolution, auth injection, attribution headers, provider wrappers, and caller precedence.
