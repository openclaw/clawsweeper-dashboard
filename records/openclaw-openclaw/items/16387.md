## Summary

Keep open: current main still treats configured provider headers as static strings, and the requested session/run-aware header interpolation is not implemented by the Fireworks plugin, provider request config, or OpenAI transport paths.

## What This Changes

Request dynamic templating for custom LLM provider headers so per-session, per-agent, or per-run values can be injected into upstream provider requests.

## Best Possible Solution

Add a documented allowlisted dispatch-time resolver for `models.providers.*.headers` and `models.providers.*.request.headers` covering `{{session.key}}`, `{{agent.id}}`, and `{{run.id}}` while preserving current static-header behavior.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature/config request. Source inspection gives high confidence that a value like `{{session.key}}` would be forwarded literally today.

## Solution Assessment

Yes. An OpenClaw-level allowlisted resolver is the narrow maintainable direction for arbitrary user-configured provider headers; provider plugins can add provider-owned headers, but that does not solve dynamic values in existing custom provider config.
