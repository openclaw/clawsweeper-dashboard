## Summary

Keep open: current main still treats configured provider headers as literal strings, and the open related PRs only cover typed Fireworks/session-affinity and schema gaps rather than the broader header-templating request.

## What This Changes

Requests dynamic placeholder interpolation in custom model-provider headers so provider requests can include runtime values such as session key, agent id, or run id.

## Best Possible Solution

Maintainers should decide the narrow supported contract, then land coordinated provider-header behavior and docs that preserve existing static headers, SecretRef resolution, auth injection, retries, attribution headers, and provider-owned stream hooks.

## Reproduction Assessment

Yes for the reported limitation: current main sanitizes and merges configured header strings without expanding session, agent, or run values. The claimed Fireworks cache-hit and Langfuse tracing benefits are external outcomes, not reproduced as a live provider metric here.

## Solution Assessment

Unclear that raw string templating is the best final shape. The Fireworks use case overlaps an existing typed affinity seam, while broader tracing headers need a maintainer-approved placeholder or hook contract with explicit security and precedence rules.
