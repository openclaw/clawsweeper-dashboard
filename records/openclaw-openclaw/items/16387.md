## Summary

Keep open: current main still treats configured provider headers and provider request headers as literal strings, the Fireworks plugin does not add session-affinity itself, and the prior dynamic-templating PRs are closed unmerged. Adjacent open PRs for agent headers or per-agent/job static headers do not cover session/run placeholder interpolation for arbitrary configured headers.

## What This Changes

The issue requests dynamic interpolation for configured provider headers so runtime values like session, agent, or run identifiers can be sent to custom LLM providers.

## Best Possible Solution

Add a documented dispatch-time resolver for configured provider headers and request.headers with an explicit placeholder allowlist, while preserving existing static-header and provider-owned runtime behavior.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature/config request. Source inspection gives a high-confidence current-main proof that values such as `{{session.key}}` would pass through as literal header strings today.

## Solution Assessment

Yes. An OpenClaw-level resolver is the narrow maintainable path because OpenClaw owns the requested session, agent, and run identity values, while provider plugins can add provider-owned headers but cannot make arbitrary user-configured headers dynamic.
