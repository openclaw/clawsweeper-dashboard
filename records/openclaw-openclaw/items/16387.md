## Summary

Keep open. Current main still treats configured provider headers and request headers as static strings, while the prior dynamic-header implementation PRs are closed unmerged and the docs still describe the header config as static.

## What This Changes

Request dynamic interpolation for configured LLM provider headers so values like `{{session.key}}`, `{{agent.id}}`, and `{{run.id}}` can be attached per model request.

## Best Possible Solution

Add a small documented dispatch-time resolver for configured model-provider headers that supports only explicit session, agent, and run identity placeholders while preserving existing static header behavior.

## Reproduction Assessment

Yes for the missing behavior. Source inspection shows a header such as `x-session-affinity: {{session.key}}` is sanitized, merged, and sent as a literal string because the current header paths have no resolver or session/run context.

## Solution Assessment

Yes. A generic OpenClaw resolver is the narrowest maintainable direction because OpenClaw owns the requested identity values and the feature needs to work for arbitrary custom providers, not only a bundled Fireworks hook.
