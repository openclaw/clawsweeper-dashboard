## Summary

Keep open. Current main still treats configured model-provider headers as literal strings, the bundled Fireworks plugin does not add session-affinity routing, and the existing provider-plugin turn-state hook does not solve arbitrary user-configured header placeholders for custom providers.

## What This Changes

Support runtime interpolation for custom LLM provider headers so values such as `{{session.key}}`, `{{agent.id}}`, and `{{run.id}}` can be sent per session or run.

## Best Possible Solution

Land a documented dispatch-time resolver for configured model-provider headers that supports only explicit session, agent, and run identity placeholders across provider-level headers and `request.headers`, while keeping static header behavior and provider plugin hooks unchanged.

## Reproduction Assessment

Yes. Configure a custom provider header such as `x-session-affinity: {{session.key}}`; current main sanitizes and merges that value as a literal string, and the runtime path does not provide session key or run id to resolve it.

## Solution Assessment

Yes. A generic OpenClaw resolver at model-request dispatch is the narrowest maintainable direction because OpenClaw owns the requested session/run identity and the feature needs to work for arbitrary custom providers, not only the bundled Fireworks plugin.
