## Summary

Keep open. Current main still treats `models.providers.*.headers` and `models.providers.*.request.headers` as literal strings, the transport/plugin hooks do not resolve arbitrary configured placeholders, and the two linked implementation PRs are closed unmerged.

## What This Changes

Dynamic header templating for configured LLM provider headers is requested so runtime values like session, agent, and run identity can be used for provider-specific cache or tracing headers.

## Best Possible Solution

Add a documented dispatch-time resolver for configured provider headers that supports only explicit session, agent, and run identity placeholders across provider-level headers and `request.headers`.

## Reproduction Assessment

Yes, via source inspection. Configure a header such as `x-session-affinity: {{session.key}}`; current main sanitizes, merges, and sends it as a literal string, and the merge path lacks the requested runtime identity context.

## Solution Assessment

Yes. A generic OpenClaw dispatch-time resolver is the narrowest maintainable direction because OpenClaw owns the requested identity values and the feature must work for arbitrary custom providers, not only a Fireworks-specific plugin hook.
