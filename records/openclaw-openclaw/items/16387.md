## Summary

Keep open: current main and v2026.5.20 still treat configured provider headers as literal static strings, while the dynamic-header implementation attempts are closed unmerged. The remaining work is a narrow provider-config feature, not a stale or already-implemented item.

## What This Changes

Request dynamic placeholders in custom model-provider header config so provider requests can include per-session, per-agent, or per-run header values.

## Best Possible Solution

Add a documented allowlisted dispatch-time resolver for models.providers.*.headers and models.providers.*.request.headers covering {{session.key}}, {{agent.id}}, and {{run.id}} while preserving static headers, SecretRef, auth, attribution, retry, and provider-plugin behavior.

## Reproduction Assessment

Yes for source-level behavior: a configured header value such as {{session.key}} is copied and merged as a literal string today. I did not run a live provider request in this read-only review, and this remains a feature request rather than broken documented behavior.

## Solution Assessment

Yes. A narrow allowlisted OpenClaw-level resolver is the maintainable path; a Fireworks-only plugin hook can cover one vendor but not arbitrary custom provider config or LiteLLM/Langfuse-style placeholders.
