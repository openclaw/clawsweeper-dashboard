## Summary

Keep open. Current main still passes configured provider headers through as literal strings, provider-plugin turn-state hooks do not resolve arbitrary user-configured placeholders, and both prior implementation PRs for this feature are closed unmerged.

## What This Changes

Request dynamic interpolation for custom model-provider headers so values like `{{session.key}}`, `{{agent.id}}`, and `{{run.id}}` can be resolved per request for Fireworks affinity and Langfuse tracing.

## Best Possible Solution

Land a documented dispatch-time resolver for configured model-provider headers that supports only explicit session, agent, and run identity placeholders across provider-level headers and `request.headers`.

## Reproduction Assessment

Yes. Configure a custom provider header such as `x-session-affinity: {{session.key}}`; current main sanitizes and merges it as a literal string, and the merge/transport paths do not have the requested session/run context to resolve it.

## Solution Assessment

Yes. A generic OpenClaw resolver is the narrowest maintainable direction because OpenClaw owns the requested identity values and the feature needs to work for arbitrary custom providers, not only a bundled Fireworks hook.
