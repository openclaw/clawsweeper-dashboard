## Summary

Keep open: current main still treats configured provider headers as literal strings, the Fireworks plugin only applies a Kimi payload wrapper, and the prior exact implementation PRs were closed unmerged. The discussion provides concrete Fireworks and Langfuse/LiteLLM use cases, and no open PR currently closes this runtime interpolation gap.

## What This Changes

Request support for allowlisted runtime placeholders in custom provider header config so session, agent, and run identifiers can be sent to Fireworks, LiteLLM, and similar providers.

## Best Possible Solution

Add a documented allowlisted dispatch-time resolver for models.providers.*.headers and models.providers.*.request.headers covering {{session.key}}, {{agent.id}}, and {{run.id}} while preserving existing static header behavior.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature/config request. Source inspection on current main confirms placeholders such as {{session.key}} would pass through as literal header text.

## Solution Assessment

Yes. An OpenClaw-level allowlisted resolver is the narrow maintainable direction because OpenClaw owns the requested session, agent, and run identities while provider plugins cannot make arbitrary user-configured headers dynamic.
