## Summary

Keep open: current main still treats configured provider headers as static strings, and the related open PRs add adjacent static/header-specific behavior rather than the requested runtime placeholder interpolation.

## What This Changes

Request dynamic interpolation for custom LLM provider headers so values like session key, agent id, and run id can be sent to Fireworks, LiteLLM, and similar providers.

## Best Possible Solution

Add a documented allowlisted dispatch-time resolver for `models.providers.*.headers` and `models.providers.*.request.headers` that preserves existing static header, auth, SecretRef, attribution, retry, and provider-wrapper behavior.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature/config request. Source inspection confirms placeholders such as `{{session.key}}` would currently be sent as literal header text.

## Solution Assessment

Yes. An OpenClaw-level resolver is the narrow maintainable direction because OpenClaw owns the requested session, agent, and run identity values, while provider plugins cannot make arbitrary user-configured headers dynamic.
