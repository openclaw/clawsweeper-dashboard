## Summary

Keep open: current main still normalizes and merges configured provider headers as literal strings, and the newer provider turn-state hook is plugin-owned rather than a generic `{{session.key}}`/`{{agent.id}}`/`{{run.id}}` config placeholder contract. The exact implementation attempts are closed unmerged, and live search did not find an open canonical replacement PR.

## What This Changes

Requests dynamic placeholder interpolation in custom model provider headers for session, agent, and run identifiers used by Fireworks affinity and LiteLLM/Langfuse tracing.

## Best Possible Solution

Add a documented, allowlisted dispatch-time resolver for `models.providers.*.headers` and `models.providers.*.request.headers` covering `{{session.key}}`, `{{agent.id}}`, and `{{run.id}}`, while preserving static headers, SecretRef resolution, auth injection, retries, attribution, and provider-plugin semantics.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature/config request. Source inspection gives high confidence that a configured value like `{{session.key}}` would be forwarded literally today.

## Solution Assessment

Yes. A narrow allowlisted OpenClaw-level resolver is the maintainable path; a Fireworks-only plugin wrapper would not cover custom provider config or the LiteLLM/Langfuse use case, while arbitrary template evaluation would be too broad.
