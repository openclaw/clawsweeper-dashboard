## Summary

Keep open. Current main has structured heartbeat responses and plugin tool descriptors, but it still lacks a skill-declared, schema-enforced remote heartbeat invocation path; this remains security-sensitive API/product design work.

## What This Changes

Request a skill/plugin metadata contract that lets remote heartbeat endpoints invoke only predeclared, schema-validated functions instead of free-form prompt instructions.

## Best Possible Solution

Design a versioned skill/plugin heartbeat invocation contract that validates declared calls before model exposure, routes execution through existing tool and permission policy, and documents the threat model plus ClawHub trust signal.
