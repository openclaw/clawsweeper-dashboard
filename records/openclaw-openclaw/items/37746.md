## Summary

Keep open. Current main has the building blocks for skills, plugin tools, and external-content wrapping, but not the requested core enforcement model for skill-declared sandboxed heartbeat function calls; this is security-sensitive product/API design, not stale cleanup.

## What This Changes

Request adds skill-declared sandboxed functions so remote heartbeat endpoints return structured calls that OpenClaw validates and executes instead of free-form instructions.

## Best Possible Solution

Design a versioned skill/plugin capability that declares callable actions with JSON schemas, validates remote heartbeat payloads before they reach the model, executes through existing tool and permission policy, and documents the threat model plus ClawHub signal.

Not applicable. This is a feature/security design request, and source inspection shows current main lacks the requested sandboxed function declaration and enforcement path.

## Solution Assessment
