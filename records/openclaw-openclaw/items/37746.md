## Summary

Keep open. Current main has structured heartbeat responses and plugin tool descriptors, but it still lacks a skill-declared sandboxed remote heartbeat invocation path; this remains security-sensitive API/product design work.

## What This Changes

Add a skill/plugin metadata contract for sandboxed remote heartbeat function invocations, validating calls and params before execution instead of passing fetched instructions to the LLM.

## Best Possible Solution

Design a versioned skill/plugin capability that validates remote heartbeat payloads against declared schemas before model exposure, executes through existing tool and permission policy, and documents the threat model plus ClawHub signal.

## Reproduction Assessment

Not applicable. This is a security-oriented feature request, and source inspection of current main shows the requested declaration/enforcement path is absent rather than a broken existing contract.

## Solution Assessment

Unclear. Structured declared calls are a plausible direction, but the exact solution needs maintainer design across skill metadata, plugin manifests, MCP/tool policy, and heartbeat runtime security.
