## Summary

Keep open. Current main has output-side `heartbeat_respond` and plugin tool descriptor work, but it still lacks a skill-declared sandboxed remote heartbeat invocation path; this remains security-sensitive API/product design work rather than stale cleanup.

## What This Changes

Add a skill/plugin metadata-declared sandbox for remote heartbeat payloads so remote content can invoke only validated callable functions instead of reaching the model as free-form instructions.

## Best Possible Solution

Design a versioned skill/plugin capability that validates remote heartbeat payloads against declared schemas before model exposure, executes through existing tool and permission policy, and documents the threat model plus ClawHub signal.

## Reproduction Assessment

Not applicable. This is a security-oriented feature request rather than a broken existing contract, and source inspection shows the requested declaration/enforcement path is absent on current main.

## Solution Assessment

Unclear. Declared structured calls are a plausible direction, but the exact fix needs maintainer design across skill metadata, plugin manifests, MCP/tool policy, and heartbeat runtime security.
