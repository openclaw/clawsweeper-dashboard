## Summary

Keep open. Current main still treats heartbeat instructions and HEARTBEAT.md prose as model-readable prompt text, while the existing structured heartbeat tool only constrains the model's response; the requested skill-declared, schema-enforced remote invocation boundary remains absent and security-sensitive.

## What This Changes

Add a skill/plugin metadata contract for sandboxed remote heartbeat functions so remote endpoints can return structured, validated calls instead of free-form instructions.

## Best Possible Solution

Design a versioned plugin/skill heartbeat invocation capability that validates declared calls before LLM exposure, routes execution through existing tool permission policy, and documents the threat model plus ClawHub trust signal.

Not applicable. This is a security-oriented feature request, and source inspection shows the requested declaration/enforcement path is absent rather than a broken existing contract.

## Solution Assessment

Unclear. Declared structured calls are a plausible direction, but the exact contract needs maintainer design across skill metadata, plugin manifests, tool policy, MCP compatibility, and heartbeat runtime security.
