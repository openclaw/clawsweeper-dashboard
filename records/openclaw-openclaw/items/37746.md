## Summary

Keep open. Current main has typed plugin-tool building blocks and structured heartbeat responses, but it does not implement skill-declared sandboxed remote heartbeat calls; this remains security-sensitive product/API design rather than cleanup-ready work.

## What This Changes

The issue requests a skill/plugin contract where remote heartbeat endpoints return validated structured function calls instead of free-form instructions sent to the model.

## Best Possible Solution

Design a versioned skill/plugin capability that validates remote heartbeat payloads against declared schemas before model exposure, executes through existing tool and permission policy, and documents the threat model plus ClawHub signal.

## Reproduction Assessment

Not applicable. This is a feature/security design request, and source inspection on current main shows the requested declaration and enforcement path is absent.

## Solution Assessment

Unclear. Structured declared calls are a plausible direction, but the exact contract needs maintainer design across skill metadata, plugin manifests, MCP/tool policy, and heartbeat runtime security.
