## Summary

Keep open. Current main still exposes heartbeat instructions and HEARTBEAT.md prose to the model as prompt text, while the existing structured heartbeat tool only constrains the model's response; the requested skill-declared, schema-enforced remote invocation boundary is still absent and security-sensitive.

## What This Changes

Request a skill-declared schema for remote heartbeat invocations so fetched heartbeat content can only call allowed functions instead of becoming natural-language instructions.

## Best Possible Solution

Design a versioned skill/plugin heartbeat invocation capability that validates declared calls before LLM exposure, routes execution through existing tool permission policy, and documents the threat model plus any ClawHub trust signal.

## Reproduction Assessment

Not applicable. This is a security-oriented feature request, and source inspection shows the requested declaration/enforcement path is absent rather than a broken existing contract.

## Solution Assessment

Unclear. Declared structured calls are a plausible direction, but the exact contract needs maintainer design across skill metadata, plugin manifests, tool policy, MCP compatibility, and heartbeat runtime security.
