## Summary

Keep open. Current main still treats heartbeat run guidance and non-task HEARTBEAT.md content as model-readable prompt text, and there is no skill-declared, schema-enforced remote-call boundary; related open threads cover adjacent skill capability and prompt-injection work but do not supersede this heartbeat-specific security/API request.

## What This Changes

The issue requests skill-declared sandboxed heartbeat functions so remote heartbeat endpoints can invoke only schema-validated calls instead of sending free-form instructions to the model.

## Best Possible Solution

Design a versioned skill/plugin heartbeat invocation capability that validates declared calls before LLM exposure, routes execution through existing tool permission policy, and documents the threat model plus ClawHub trust signal.

## Reproduction Assessment

Not applicable as a bug. Source inspection gives high-confidence evidence that current main lacks the requested declaration/enforcement path and still exposes heartbeat guidance as prompt text.

## Solution Assessment

Unclear as a final design. Declared structured calls are a plausible direction, but the best implementation needs maintainer decisions across skill metadata, plugin manifests, heartbeat runtime, tool policy, MCP compatibility, and ClawHub trust wording.
