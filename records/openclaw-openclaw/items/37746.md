## Summary

Keep open. Current main has structured heartbeat responses and plugin tool descriptors, plus install-time skill scanning, but it still lacks a declared, schema-validated remote heartbeat invocation path that keeps fetched heartbeat content out of the model prompt.

## What This Changes

Request a new skill/plugin metadata contract for sandboxed heartbeat functions and runtime enforcement of structured remote heartbeat calls.

## Best Possible Solution

Design a versioned skill/plugin capability that validates remote heartbeat payloads against declared schemas before model exposure, executes through existing tool and permission policy, and documents the security model plus ClawHub trust signal.

## Reproduction Assessment

Not applicable. This is a security-oriented feature request, and current-main source inspection shows the requested declaration and runtime enforcement path is absent rather than a broken existing contract.

## Solution Assessment

Unclear. Declared structured calls are a plausible direction, but the exact solution needs maintainer design across skill metadata, plugin manifests, tool policy, heartbeat runtime, and ClawHub trust semantics.
