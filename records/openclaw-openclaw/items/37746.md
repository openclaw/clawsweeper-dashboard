## Summary

Keep this open: current main still lacks the requested declared, schema-validated heartbeat callable contract, and the remaining work is a security-sensitive core/plugin API design rather than stale cleanup or ClawHub-only work.

## What This Changes

Request a new skill/plugin metadata contract that lets remote heartbeat endpoints invoke only predeclared schema-validated functions instead of free-form instructions.

## Best Possible Solution

Add a versioned declared-callable contract, parse and document it, implement a structured remote heartbeat fetch/parse path, validate call names and params against declared schemas, execute only through existing tool/plugin capability boundaries, and add regression tests proving injected free-form text never reaches the model as heartbeat instructions.

## Reproduction Assessment

Not applicable as a feature/security design request. Source inspection gives high confidence that the requested contract is absent from skill metadata, plugin manifests, heartbeat config, and heartbeat execution.

## Solution Assessment

Unclear as an implementation, but the structured-invocation direction is the right security shape. The exact API and execution boundary need maintainer/security design before a narrow code fix is safe.
