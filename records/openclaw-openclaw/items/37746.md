## Summary

Keep open. Current main still lacks the requested core-enforced sandboxed function contract: skill metadata has no declared callable schema list, heartbeat config has no structured remote invocation mode, and heartbeat execution still sends resolved heartbeat text through the model path.

## What This Changes

The issue asks for a core-enforced skill metadata contract that lets remote heartbeat endpoints invoke only predeclared schema-validated functions instead of delivering free-form natural-language instructions to the model.

## Best Possible Solution

Keep this open for maintainer security/product design. A complete solution should add a versioned declared-callable contract for skills/plugins, parse and document it, add a structured remote heartbeat fetch/parse path, validate call names and params against declared schemas, execute only through existing capability/tool boundaries, and include regression coverage proving untrusted remote heartbeat text never reaches the model as instructions.

## Reproduction Assessment

Not applicable as a feature/security design request, but the absence is high-confidence. Current-main source and docs show heartbeat remains prompt-driven and no sandboxed remote invocation contract exists.

## Solution Assessment

No, current main is not the best way to solve the issue because it still relies on model handling of heartbeat prompt text. The safer path is a core-enforced structured contract before ClawHub trust indicators or platform-specific skills rely on this pattern.
