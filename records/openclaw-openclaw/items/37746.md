## Summary

Keep open. Current main has structured heartbeat responses and plugin tool descriptor caching, but it still lacks the requested skill-declared, schema-enforced remote heartbeat invocation path; this remains security-sensitive API/product design work.

## What This Changes

Add a declared sandboxed-function contract so remote heartbeat endpoints can invoke only pre-approved, schema-validated skill functions instead of sending free-form instructions to the model.

## Best Possible Solution

Design a versioned skill/plugin heartbeat invocation capability that validates declared calls before model exposure, routes execution through existing tool and permission policy, and documents the threat model plus ClawHub trust signal.

## Reproduction Assessment

Not applicable. This is a security-oriented feature request, and source inspection shows the requested declaration/enforcement path is absent rather than a broken existing contract.

## Solution Assessment

Unclear. Declared structured calls are a plausible direction, but the exact shape needs maintainer design across skill metadata, plugin manifests, tool policy, MCP compatibility, and heartbeat runtime security.
