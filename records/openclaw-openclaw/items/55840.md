## Summary

Keep open. Current main and the latest release still intentionally remove the Chrome Extension Relay, reject `driver: "extension"`, and migrate legacy relay config to Chrome MCP `existing-session`; restoring the old relay or replacing it with a plugin/backend API needs maintainer product and security direction.

## What This Changes

Restore the removed Chrome Extension Relay as an optional browser driver or migration path for existing-tab, video-control, chat-automation, and auto-reattach workflows.

## Best Possible Solution

Maintainers should choose a browser strategy: keep the relay retired with clearer migration-gap docs, or design a vetted optional relay/browser-backend path with security review, tests, packaging, docs, and migration behavior.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to configure or create a profile with `driver: "extension"`: current main rejects it, the relay helper is a no-op, and the contract test asserts the 400 error.

## Solution Assessment

Unclear. Directly restoring the old relay in core may not be the best maintainable path; a hardened optional browser plugin/backend API may be safer if maintainers want to support this workflow again.
