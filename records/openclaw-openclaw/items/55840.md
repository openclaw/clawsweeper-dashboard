## Summary

Keep open. Current main and the latest release still intentionally retire the Chrome Extension Relay, reject `driver: "extension"`, and migrate legacy relay config to Chrome MCP `existing-session`; the remaining request needs maintainer browser-strategy and security direction, not conservative cleanup closure.

## What This Changes

Restore the retired Chrome Extension Relay as an optional browser driver, or provide a migration path and documentation for users who depended on it.

## Best Possible Solution

Maintainers should explicitly choose the browser strategy: keep the legacy relay retired with clear migration-gap docs, or design a vetted optional browser-backend/plugin path and maintain a Chrome Extension Relay with security review, tests, packaging, docs, and migration behavior.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: current main rejects `driver: "extension"`, tests assert that rejection, and the relay lifecycle helper is a no-op.

## Solution Assessment

Unclear. Directly restoring the old relay in core may not be the narrowest maintainable solution because OpenClaw’s current direction favors optional capabilities through plugins, while the needed browser-backend plugin seam remains open in #19289.
