## Summary

Keep open. Current main and the latest release still intentionally retire the Chrome Extension Relay, reject `driver: "extension"`, and migrate legacy relay config to Chrome MCP `existing-session`; the remaining request needs maintainer browser-strategy and security direction, especially with the related browser-backend plugin-interface work still open in #19289.

## What This Changes

Restore the removed Chrome Extension Relay as an optional browser driver, including `driver: "extension"`, the port 18792 relay server, extension assets, and migration guidance.

## Best Possible Solution

Maintainers should choose the browser strategy explicitly: keep the legacy relay retired with clearer migration-gap docs, or design a vetted optional browser-backend/plugin path and maintain a Chrome Extension Relay with security review, tests, packaging, docs, and migration behavior.

## Reproduction Assessment

Yes. Source inspection on current main shows `ensureExtensionRelayForProfiles` is a no-op, `/profiles/create` rejects `driver: "extension"`, doctor migrates that driver to `existing-session`, and tests assert the rejection.

## Solution Assessment

Unclear. Directly restoring the old relay in core is not clearly the narrowest maintainable path because current project direction favors optional capabilities through plugins, while the browser-backend plugin seam remains open in #19289.
