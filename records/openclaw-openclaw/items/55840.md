## Summary

Keep open. Current main still intentionally removes the Chrome Extension Relay, rejects `driver: "extension"`, and migrates legacy relay config to Chrome MCP `existing-session`; that does not restore the reported existing-tab relay, auto-reattach, video-control, or chat-automation workflows. The request is concrete, but it needs maintainer product and security direction rather than cleanup closure or an automated repair lane.

## What This Changes

The issue asks to restore the retired Chrome Extension Relay as an optional browser driver, including legacy `driver: "extension"` compatibility, integrated relay startup on port 18792, and bundled extension assets.

## Best Possible Solution

Maintainers should make the browser strategy explicit: either keep the legacy relay retired and document the migration gaps clearly, or design a vetted browser-backend/plugin path and maintain an optional Chrome Extension Relay with security review, tests, docs, packaging, and legacy-config migration behavior.
