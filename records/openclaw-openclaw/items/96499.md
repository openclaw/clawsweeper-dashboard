## Summary

## What This Changes

The PR changes GitHub Copilot model discovery and embedding success-response parsing to use `readProviderJsonResponse`, updates setup-error matching, and adjusts related tests to use real `Response` bodies.

## Best Possible Solution

Keep model discovery on the generic bounded reader, but make the `/embeddings` success reader preserve the established embedding response budget and cover both a large valid batch and an oversized stream in tests.

## Reproduction Assessment

Yes. Current main uses unbounded `response.json()` on both successful Copilot JSON paths, and the PR body includes terminal proof that the old path reads the whole oversized stream while the bounded reader cancels near the cap.

## Solution Assessment

No, not as submitted. The shared bounded reader is the right direction, but `/embeddings` should preserve the established remote embedding response budget instead of inheriting the generic 16 MiB provider JSON cap.
