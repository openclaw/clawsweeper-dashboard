## Summary

Keep open. Current main still redacts text-bearing transcript messages before session writes and still forces Control UI tool-payload redaction, so the reported byte mutation is real; changing it safely requires a maintainer-owned redaction trust-boundary decision rather than cleanup closure.

## What This Changes

The issue reports that secret redaction of tool arguments/results mutates persisted session history and breaks byte-stable KV-cache reuse for local LLM providers.

## Best Possible Solution

Define a provider-aware model-facing prompt/redaction contract that preserves trusted local-provider prompt bytes while keeping persisted transcript, Control UI, diagnostics, support, approval, and tool-output safety surfaces redacted.

## Reproduction Assessment

Yes at source level: current main redacts text-bearing toolResult transcript content before persistence, and the default patterns match the reported ENV-style TOKEN strings. I did not run a live local LLM cache trace.

## Solution Assessment

Unclear. A provider-default-off toggle might restore cache bytes, but the safer maintainable direction is a maintainer-owned trust-boundary design or model-facing prompt copy that does not weaken security surfaces.
