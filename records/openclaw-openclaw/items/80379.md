## Summary

Keep open. Current main still redacts persisted tool-result transcript text before session writes, matching the reported mutation path, but fixing it would change a security-sensitive redaction boundary for local providers and needs maintainer product/security judgment.

## What This Changes

The issue asks OpenClaw to stop mutating model-facing session history when tool outputs contain secret-like strings, especially for local LLM providers that rely on byte-stable KV-cache prefixes.

## Best Possible Solution

Define a provider-aware model-facing prompt/transcript contract, or a separate prompt-build copy, that preserves trusted local-provider prefix bytes while keeping persisted transcript, Control UI, diagnostics, support, approval, and log redaction intact.

## Reproduction Assessment

Yes at source level: current main redacts toolResult transcript text before persistence, and the default patterns match the reported ENV-style TOKEN strings. I did not run a live local LLM KV-cache trace, so this is source-reproducible rather than fully reproduced.

## Solution Assessment

Unclear. A blanket provider default-off toggle would preserve cache bytes, but the safer maintainable direction is a maintainer-owned trust-boundary design or separate model-facing copy that leaves user-visible and diagnostic redaction surfaces intact.
