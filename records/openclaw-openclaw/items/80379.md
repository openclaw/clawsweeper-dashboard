## Summary

Keep open. Current main still redacts transcript/tool-result text before persistence and before UI tool-event emission, and the newer append-sink security fix reinforces that this is an intentional redaction boundary; changing it for local-provider KV-cache preservation needs maintainer security/product review, not cleanup closure.

## What This Changes

The issue asks OpenClaw to preserve model-facing session/tool-result bytes for trusted local providers so secret redaction does not invalidate local KV-cache prefix reuse.

## Best Possible Solution

Define a provider-aware redaction contract or separate model-facing prompt copy that preserves trusted local-provider prompt bytes while keeping persisted transcripts, Control UI, diagnostics, support, approval prompts, sessions_history, and logs protected according to policy.

## Reproduction Assessment

Yes at source level: current main redacts transcript messages before persistence, those session messages feed subsequent agent runs, and the default patterns match the reported secret-looking environment assignments. I did not establish a live local LLM KV-cache trace.

## Solution Assessment

Unclear. A blanket provider default-off toggle would preserve cache bytes, but the safer maintainable path is a maintainer-owned trust-boundary design or model-facing copy that leaves protected redaction surfaces intact.
