## Summary

Keep open. Current main still redacts persisted transcript text and Control UI tool payloads by default, and the remaining request is a local-provider redaction policy change with security implications rather than something already implemented.

## What This Changes

The issue requests provider-aware or non-mutating tool result secret redaction so local LLM KV-cache prefix matching remains stable.

## Best Possible Solution

Define a provider-aware transcript redaction contract that preserves prior prompt bytes for local prefix-cache providers without weakening documented UI, diagnostic, and persisted-secret protections.

## Reproduction Assessment

Yes at source level: guarded session writes run transcript redaction before persisting toolResult messages, and the default regexes match ENV-style TOKEN/KEY assignments. I did not run a live local LLM cache trace, so this is not a full runtime reproduction.

## Solution Assessment

Unclear. The requested provider-aware toggle or default-off behavior may be useful for local KV-cache performance, but the best fix needs a maintainer-owned security boundary rather than a blanket redaction bypass.
