## Summary

Keep open. Current main still redacts persisted transcript text before session writes and still forces UI tool-payload redaction; resolving the local-provider cache impact changes a security-sensitive redaction contract, so this needs maintainer policy review rather than cleanup closure.

## What This Changes

The issue requests a provider-aware or model-facing redaction path so tool-result secret redaction does not mutate prompt bytes persisted for local LLM cache reuse.

## Best Possible Solution

Define a provider-aware transcript redaction contract, or a separate model-facing prompt copy, that preserves trusted local-provider prompt bytes without weakening persisted transcript, Control UI, diagnostics, support, and approval redaction surfaces.

## Reproduction Assessment

Yes at source level: guarded session writes run transcript redaction before persisting toolResult messages, and the default patterns match ENV-style TOKEN/KEY assignments. I did not run a live local LLM KV-cache trace.

## Solution Assessment

Unclear. A blanket provider default-off toggle would preserve cache bytes, but the safer direction is a maintainer-owned redaction/trust-boundary design or a model-facing copy that leaves UI and diagnostic redaction intact.
