## Summary

Keep open. Current main still redacts tool-result/session transcript content before persisted session writes, and the requested fix would change a security-sensitive redaction boundary for local providers rather than being a cleanup close.

## What This Changes

The issue requests that tool argument/result secret redaction stop mutating model-facing session history for trusted local LLM providers because it invalidates prefix/KV caches.

## Best Possible Solution

Define a provider-aware model-facing prompt/session redaction contract that preserves trusted local-provider prompt bytes while keeping persisted transcript, Control UI, diagnostics, support, approval, and log redaction intact.

## Reproduction Assessment

Yes at source level: current main redacts toolResult/session transcript content before the persisted message is returned and reused, and default patterns match the reported TOKEN/KEY strings. I did not run a live exo/llama.cpp KV-cache trace.

## Solution Assessment

Unclear. A blanket provider default-off toggle would preserve cache bytes, but the safer maintainable solution is a maintainer-owned trust-boundary design or separate model-facing copy that does not weaken UI and diagnostic redaction.
