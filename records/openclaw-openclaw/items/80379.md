## Summary

Keep open. Current main still redacts tool-result and transcript text before it can be reused, and changing that for local providers would alter a security-sensitive redaction contract rather than a cleanup-closeable implementation gap.

## What This Changes

The issue asks OpenClaw to preserve model-facing tool-result bytes for trusted local providers so secret redaction does not break KV cache prefix reuse.

## Best Possible Solution

Define a provider-aware or model-facing-copy redaction contract that preserves trusted local-provider prompt bytes while keeping persisted transcript, UI, diagnostics, support, approval, and log surfaces protected.

## Reproduction Assessment

Yes at source level: current main redacts transcript/tool-result messages before storage and the default patterns match the reported TOKEN/KEY assignment shape. I did not run a live local LLM KV-cache trace.

## Solution Assessment

Unclear. A provider-default-off toggle could preserve cache bytes, but the safer maintainable direction is a maintainer-owned trust-boundary design or separate model-facing prompt copy that does not weaken safety surfaces.
