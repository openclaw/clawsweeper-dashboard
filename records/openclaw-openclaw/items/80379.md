## Summary

Keep open: current main still redacts transcript/tool-result payload text before persistence and UI emission, and changing that for local-provider cache reuse is a security/product boundary decision rather than cleanup closure.

## What This Changes

The issue asks OpenClaw to avoid mutating model-facing session history with secret redaction for trusted local LLM providers so KV-cache prefixes stay byte-stable.

## Best Possible Solution

Define a provider-aware model-facing redaction boundary, or a separate prompt copy, that preserves trusted local-provider prompt bytes while keeping persisted transcripts, Control UI, diagnostics, support bundles, approval prompts, and logs protected.

## Reproduction Assessment

Yes at source level: current main redacts matching TOKEN/KEY strings before transcript persistence and still deep-redacts tool payloads for UI events. I did not run a live local-provider KV-cache trace.

## Solution Assessment

Unclear. A blanket local-provider default-off toggle would preserve bytes, but the safer maintainable direction is a maintainer-owned trust-boundary design or model-facing copy that does not weaken other redaction surfaces.
