## Summary

Keep open. Current main and the reported release still redact persisted transcript text and Control UI tool payloads, so the source-reproducible mutation remains; changing it would alter a security-sensitive redaction contract and needs maintainer product/security judgment.

## What This Changes

The issue asks OpenClaw to stop secret-looking tool result redaction from mutating local-provider model-facing session history and invalidating KV cache prefix reuse.

## Best Possible Solution

Define a provider-aware redaction/transcript contract, or a separate model-facing prompt copy, that preserves trusted local-provider prompt bytes while keeping persisted transcript, Control UI, diagnostics, support, approval, and log surfaces protected.

## Reproduction Assessment

Yes at source level: toolResult messages pass through the before-write redaction hook before persistence, and the default patterns match the reported ENV-style TOKEN/KEY strings. I did not run a live exo/llama.cpp KV-cache trace.

## Solution Assessment

Unclear. A blanket local-provider redaction opt-out would preserve prompt bytes, but the safer maintainable direction is a maintainer-owned trust-boundary design or model-facing copy that leaves security surfaces redacted.
