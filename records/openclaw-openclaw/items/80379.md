## Summary

Keep open. Current main still redacts persisted transcript/tool-result text and forces Control UI tool-payload redaction, so the local-provider cache concern is source-reproducible; fixing it would change a documented secret-redaction boundary and needs maintainer security/product review.

## What This Changes

The issue asks OpenClaw to avoid mutating model-facing tool-result bytes for trusted local LLM providers, possibly by adding a redaction toggle or separating persisted/UI redaction from the prompt copy.

## Best Possible Solution

Define a maintainer-approved redaction/trust-boundary design that preserves stable model-facing bytes for trusted local providers while keeping persisted transcripts, Control UI events, diagnostics, support exports, approval prompts, and history tools protected.

## Reproduction Assessment

Yes at source level: current main redacts tool-result transcript content before persistence and the default patterns match the reported ENV-style token strings. I did not run a live local LLM KV-cache trace in this read-only review.

## Solution Assessment

Unclear. A simple provider default-off toggle would preserve prompt bytes, but the safer solution is a maintainer-owned model-facing copy or provider-aware redaction contract that does not weaken documented UI, transcript, diagnostic, support, and approval redaction surfaces.
