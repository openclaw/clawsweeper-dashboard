## Summary

Keep open. Current main still redacts text-bearing persisted transcript messages before session writes, and resolving the reported local-provider cache impact would change a security-sensitive redaction/trust-boundary contract rather than a cleanup-closeable stale issue.

## What This Changes

Preserve local-provider prompt/cache bytes by avoiding secret-redaction mutation of model-facing tool results while retaining required secret redaction on protected surfaces.

## Best Possible Solution

Define a maintainer-owned provider-aware transcript redaction contract, or separate model-facing prompt bytes from persisted/UI/diagnostic redacted copies, so trusted local providers keep stable prefixes without weakening protected secret surfaces.

## Reproduction Assessment

Yes at source level: current main redacts text-bearing toolResult messages before persistence, and the default patterns match ENV-style TOKEN/KEY assignments. I did not run a live local LLM KV-cache trace.

## Solution Assessment

Unclear. A blanket local-provider default-off toggle would preserve cache bytes, but the safer maintainable path is a security-reviewed split between model-facing bytes and redacted persistence/UI/diagnostic surfaces.
