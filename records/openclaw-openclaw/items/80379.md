## Summary

Keep open. Current main and the latest release still redact transcript text before session writes and force-redact UI tool payloads; fixing the local-provider cache impact means redesigning a security-sensitive redaction boundary, not doing cleanup closure.

## What This Changes

Avoid mutating model-facing/session history when tool-result secret redaction masks text for local LLM providers, so KV cache prefixes remain stable.

## Best Possible Solution

Keep transcript/UI/diagnostic redaction as protected surfaces while adding a separate trusted local-provider model-facing prompt copy or equivalent provider-aware contract that preserves byte-stable history.

## Reproduction Assessment

Yes at source level: current main redacts text-bearing toolResult messages before persistence, and the default patterns match ENV-style TOKEN/KEY assignments. I did not run a live exo/llama.cpp/LM Studio KV-cache trace.

## Solution Assessment

Unclear. A blanket local-provider toggle would preserve prompt bytes, but the safer fix is a maintainer-owned trust-boundary design or model-facing copy that leaves persisted transcript, UI, diagnostics, and support redaction intact.
