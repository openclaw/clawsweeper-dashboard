## Summary

Keep open. Current main still redacts persisted transcript text before session writes and still forces tool/UI payload redaction, so the reported local-provider cache mutation remains source-reproducible; changing it needs maintainer security/product policy rather than cleanup closure.

## What This Changes

The issue asks OpenClaw to avoid mutating model-facing session/tool-result bytes through secret redaction for trusted local LLM providers so KV/prefix caches remain stable.

## Best Possible Solution

Define a provider-aware model-facing transcript/redaction contract, or a separate model prompt copy, that preserves trusted local-provider prompt bytes without weakening persisted transcript, Control UI, diagnostics, support, approval, and sessions_history redaction surfaces.

## Reproduction Assessment

Yes at source level: guarded session writes run transcript redaction before persisting toolResult messages, and the default patterns match ENV-style TOKEN/KEY assignments. I did not run a live local LLM KV-cache trace.

## Solution Assessment

Unclear. A provider default-off toggle would preserve local cache bytes, but the safer direction is a maintainer-owned redaction/trust-boundary design or model-facing copy that leaves UI and diagnostic redaction intact.
