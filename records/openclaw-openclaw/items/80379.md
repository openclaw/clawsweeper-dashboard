## Summary

Keep open. Current main still redacts persisted session transcript/tool-result content by default before later replay, while the existing `logging.redactSensitive: "off"` escape hatch is broad enough to require maintainer security/product judgment rather than cleanup closure.

## What This Changes

The issue asks OpenClaw to avoid mutating model-visible session history with secret redaction for trusted local LLM providers so KV/prefix caches stay stable.

## Best Possible Solution

Add a maintainer-approved trusted-local/provider-aware redaction policy, or split persisted redacted transcripts from model-visible replay, so local cache stability improves without weakening secure defaults.

## Reproduction Assessment

No, not as a live end-to-end repro. Source inspection shows the default transcript write path still redacts and returns persisted message content, and tests cover default masking versus `redactSensitive: "off"`, but I did not run a local LLM KV-cache reproduction.

## Solution Assessment

Unclear. A blanket default-off policy for local providers may be too broad; the safer solution is an explicit trusted-local policy or a redacted-persisted/raw-replay split with security review and upgrade tests.
