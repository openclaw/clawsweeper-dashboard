## Summary

Keep open. Current main still redacts transcript/tool payload text before persistence and UI emission, while cache-stability docs make byte-stable prompts important; resolving the conflict needs maintainer security/product design rather than cleanup closure.

## What This Changes

The issue asks OpenClaw to avoid rewriting model-facing local-provider session/tool-result bytes when secret redaction would otherwise break KV/prefix cache reuse.

## Best Possible Solution

Define a provider-aware or model-facing transcript redaction boundary that preserves trusted local-provider prompt bytes while persisted transcripts, Control UI, diagnostics, support, session-history, and approval surfaces stay redacted.

## Reproduction Assessment

Yes for the source-level mutation path: current main redacts transcript messages before writes and tests prove matching secrets are removed. I did not establish a live local-provider KV-cache trace in this read-only review.

## Solution Assessment

Unclear. A blanket provider default-off toggle would preserve bytes, but the safer direction is a maintainer-owned trust-boundary design or separate model-facing copy that does not weaken persisted/UI/diagnostic redaction.
