## Summary

Keep open. Current main still redacts tool-result transcript text before persistence and still intentionally redacts Control UI tool payloads, so fixing the local-provider cache impact requires a maintainer-owned redaction trust-boundary decision rather than cleanup closure.

## What This Changes

The issue requests preserving local-provider model-facing session bytes when secret-looking tool results are redacted, so KV-cache prefix reuse is not broken.

## Best Possible Solution

Define a provider-aware transcript redaction contract, or a separate model-facing prompt copy, that preserves trusted local-provider prompt bytes without weakening persisted transcript, Control UI, diagnostics, support, approval, and session-history redaction surfaces.

## Reproduction Assessment

Yes at source level: guarded session writes run transcript redaction before persisting toolResult messages, and the built-in patterns match the reported ENV-style TOKEN strings. I did not establish a live local LLM KV-cache trace in this read-only review.

## Solution Assessment

Unclear. A provider default-off toggle would preserve cache bytes, but the safer solution is a maintainer-owned trust-boundary design or model-facing copy that leaves UI, diagnostics, support, and persisted transcript redaction intact.
