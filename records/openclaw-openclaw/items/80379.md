## Summary

Keep open. Current main and the shipped v2026.5.7 behavior still redact text-bearing session transcript content before persistence and still force Control UI/tool payload redaction, while changing that for local providers would alter a security-sensitive redaction contract.

## What This Changes

The issue asks OpenClaw to preserve model-facing session history bytes for local LLM KV-cache reuse when tool results contain secret-looking strings that currently get redacted.

## Best Possible Solution

Define a provider-aware model-facing prompt/redaction boundary that preserves trusted local-provider prompt bytes without weakening persisted transcript, Control UI, diagnostics, support, approval, log, and sessions_history redaction.

## Reproduction Assessment

Yes at source level: current main redacts text-bearing toolResult content before session persistence, and the default patterns match ENV-style TOKEN/KEY assignments. I did not run a live local LLM KV-cache trace.

## Solution Assessment

Unclear. A blanket local-provider redaction toggle would preserve cache bytes, but the safer maintainable direction is a maintainer-owned trust-boundary design or separate model-facing copy that keeps safety-boundary surfaces redacted.
