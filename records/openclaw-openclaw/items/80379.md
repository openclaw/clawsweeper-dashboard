## Summary

Keep open. Current main still redacts persisted transcript text and Control UI tool payloads, and the requested local-provider exception would change the secret-redaction trust boundary rather than apply an already-implemented fix.

## What This Changes

The issue asks OpenClaw to avoid mutating persisted tool-result/session history with secret redaction for local LLM providers so prefix/KV caches can keep matching prior prompt bytes.

## Best Possible Solution

Define a provider-aware transcript redaction contract that preserves prompt bytes for trusted local prefix-cache providers without weakening UI, diagnostics, support, and persisted-secret protections.

## Reproduction Assessment

Yes for the transcript mutation path: current main runs redaction before persisting toolResult messages, and the default regexes match ENV-style TOKEN/KEY assignments. I did not run a live local LLM cache trace, so the cache-prefill cost is supported by reporter evidence rather than directly reproduced here.

## Solution Assessment

Unclear. A provider-aware toggle or default-off behavior for local models could solve the performance problem, but the safest solution needs a maintainer-owned security boundary rather than a blanket redaction bypass.
