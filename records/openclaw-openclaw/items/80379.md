## Summary

Keep open. Current main still redacts persisted transcript text before session writes and still forces Control UI tool-payload redaction, so the remaining work is a security-sensitive local-provider redaction policy decision rather than an already-implemented fix.

## What This Changes

Avoid mutating stored tool-result transcript text through secret redaction for local LLM providers so prefix/KV caches can reuse prior prompt bytes.

## Best Possible Solution

Define a provider-aware transcript redaction contract that preserves model-facing prompt bytes for trusted local cache providers while keeping UI, diagnostic, support, and documented secret-protection surfaces intact.

## Reproduction Assessment

Yes at source level: guarded session writes run transcript redaction before persisting toolResult messages, and the default patterns match ENV-style TOKEN/KEY assignments. I did not run a live local LLM KV-cache trace.

## Solution Assessment

Unclear. A blanket provider default-off or config toggle may preserve cache bytes, but the safest solution needs maintainer-owned security/product policy or a separate model-facing copy that does not weaken redaction on UI and diagnostic surfaces.
