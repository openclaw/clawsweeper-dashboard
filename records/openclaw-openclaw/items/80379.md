## Summary

Keep open. Current main still redacts transcript/tool-result bytes before persistence and still forces Control UI tool payload redaction, so the reported mutation path remains source-proven; changing it requires security and product judgment about local-provider trust boundaries.

## What This Changes

Request provider-aware or configurable tool-result secret redaction so trusted local LLM providers keep stable prompt bytes for KV cache prefix reuse.

## Best Possible Solution

Define a maintainer-approved redaction contract that preserves stable model-facing bytes for explicitly trusted local providers while keeping persisted transcript, UI, diagnostic, support, approval, and log redaction intact.

## Reproduction Assessment

Yes at source level: current main redacts transcript/tool-result text before persistence, and the default patterns match the reported ENV-style TOKEN strings. I did not run a live local LLM KV-cache trace.

## Solution Assessment

Unclear. A provider default-off toggle could preserve cache bytes, but the safer solution is a maintainer-owned trust-boundary or separate model-facing prompt copy that does not weaken redaction on storage and UI surfaces.
