## Summary

Keep open. Current main still redacts tool-result transcript text before session writes, and the prior ClawSweeper comment is consistent with the current source: fixing the local-provider cache impact would change a security-sensitive redaction contract rather than qualify as cleanup closure.

## What This Changes

The issue requests preserving local-provider prompt/cache bytes when tool results contain secret-looking strings, without mutating the model-facing session history.

## Best Possible Solution

Define a provider-aware or model-facing transcript contract that preserves trusted local-provider prompt bytes while keeping persisted transcript, Control UI, diagnostics, support, approval, and `sessions_history` redaction intact.

## Reproduction Assessment

Yes at source level: default redaction patterns match the reported ENV-style token references, and guarded tool-result persistence writes the redacted text back into the session manager before later prompt assembly. I did not run a live exo/llama-server KV-cache trace.

## Solution Assessment

Unclear. A local-provider default-off toggle would preserve prompt bytes, but the safer maintainable direction is a maintainer-owned redaction/trust-boundary design or separate model-facing copy that does not weaken UI and diagnostic safety surfaces.
