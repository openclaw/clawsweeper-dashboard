## Summary

Keep open. Current main still mutates persisted transcript/tool-result text through the redaction path, and changing that for local providers would alter a security-sensitive redaction contract rather than being conservative cleanup.

## What This Changes

The issue asks OpenClaw to avoid redacting model-facing tool-result session history for trusted local LLM providers so KV-cache prefixes remain stable.

## Best Possible Solution

Define a provider-aware model-facing transcript contract, or a separate model prompt copy, that preserves trusted local-provider bytes without weakening persisted transcript, Control UI, diagnostics, support, approval, and history redaction surfaces.

## Reproduction Assessment

Yes at source level: current main redacts messages before transcript persistence, the tool event surface still forces tool-payload redaction, and the default patterns match the reported TOKEN/KEY strings. I did not run a live local LLM KV-cache trace.

## Solution Assessment

Unclear. A blanket provider default-off toggle would preserve cache bytes, but the safer path is a maintainer-owned trust-boundary design or model-facing copy that keeps UI, diagnostics, sessions_history, approval prompts, and persisted transcript redaction intact.
