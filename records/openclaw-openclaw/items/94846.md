## Summary

Keep open: current main and the latest release still classify the reported recovered tool-warning path as fatal before delivery dispatch, and the open closing PRs are candidate fixes rather than merged resolution.

## What This Changes

Ensure scheduled isolated agentTurn runs that recover from early non-terminal tool errors still dispatch final assistant output and record delivery state accurately.

## Best Possible Solution

Land one narrow cron payload-outcome fix that treats recovered tool warnings with final output as non-fatal for delivery while preserving true fatal structured-error suppression and genuine run-level failures.

## Reproduction Assessment

Yes at source level: current main still gates recovered tool-warning recovery on `runLevelError` absence and channel final-text preference, then returns before delivery dispatch when the warning remains fatal. I did not run the live WSL2/Feishu scheduled setup in this read-only review.

## Solution Assessment

Yes: the best solution is a narrow cron payload outcome classification/finalization fix, not a new config flag or broad cron status rewrite. The open linked PRs should be reviewed and consolidated into that shape.
