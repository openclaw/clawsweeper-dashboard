## What This Changes

The issue reports that scheduled isolated cron `agentTurn` delivery is skipped before dispatch when an earlier recovered tool error is later classified as the fatal run outcome.

## Best Possible Solution

Patch the cron payload outcome classification so final assistant visible text can recover plain `⚠️ 🛠️` tool-warning payloads for this direct-delivery path, while keeping genuinely fatal structured errors on the existing non-dispatch failure path.

## Repair Work Prompt

Fix the scheduled isolated cron `agentTurn` pre-dispatch delivery skip reported at https://github.com/openclaw/openclaw/issues/94846. On current main, `src/cron/isolated-agent/helpers.ts` only lets final assistant visible text recover plain `⚠️ 🛠️` tool-warning payloads when `preferFinalAssistantVisibleText === true`; Feishu direct delivery does not set that channel policy, so `src/cron/isolated-agent/run.ts` can enter `hasFatalStructuredErrorPayload && prepared.deliveryRequested` and return before `dispatchCronDelivery`. Adjust the outcome classification so an isolated run with final assistant visible text and a plain recovered tool warning can still reach completion delivery for the reported direct-delivery path, while genuinely fatal structured provider/tool errors remain out of normal announce delivery. Add focused regression coverage for a Feishu-like direct delivery channel with an early `isError: true` `⚠️ 🛠️` payload, final assistant visible output, and delivery requested, asserting the outcome is non-fatal and dispatch is reached. Compare with but do not duplicate https://github.com/openclaw/openclaw/pull/94116, which covers a later delivery-dispatch error propagation path. Do not add config flags, new status values, or changelog changes.
