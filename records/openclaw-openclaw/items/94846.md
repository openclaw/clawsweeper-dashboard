## Summary

Keep open: current main and the latest release still have the reported pre-dispatch fatal-classification path, and several open PRs already target the fix, so this should stay open until one canonical fix lands.

The issue reports scheduled isolated cron agent turns skipping delivery when an earlier recovered tool error is still classified as fatal despite final assistant output.

## Best Possible Solution

Land one canonical cron outcome fix that uses a narrow recovered-tool/non-terminal predicate for both fatality and final-text delivery selection, proves the no-preference delivery path, and then close this issue as implemented.
