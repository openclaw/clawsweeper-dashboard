## What This Changes

The issue requests that recovered isolated scheduled `agentTurn` tool errors not suppress final assistant output delivery before dispatch.

## Best Possible Solution

Land one reviewed narrow cron payload-outcome fix that lets recovered tool-warning runs with final output reach delivery dispatch while preserving the true fatal structured-error suppression path.

## Reproduction Assessment

Yes at source level: current main still gates recovered `⚠️ 🛠️` tool-warning recovery on `preferFinalAssistantVisibleText`, while the fatal structured-payload branch returns before `dispatchCronDelivery` when the warning remains fatal. I did not run the live WSL2/Feishu scheduled setup in this read-only review.

## Solution Assessment

Yes, a focused cron payload classification fix is the best path. It should avoid adding new config or broad status semantics and should preserve intentional suppression of true fatal structured-error announces.
