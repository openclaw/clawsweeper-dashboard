## Summary

Keep open: current `main` and `v2026.6.9` still have the source-level path where recovered tool-warning output can remain fatal before delivery dispatch, and multiple open closing PRs are still candidate fixes rather than merged implementation.

## What This Changes

The issue reports that isolated scheduled cron `agentTurn` runs can recover from early tool errors and produce final output, yet cron finalization treats the earlier error as fatal and skips delivery dispatch.

## Best Possible Solution

Review and land one narrow cron payload-outcome fix that treats recovered tool warnings with final output as non-fatal for delivery, promotes the recovered final text, and preserves suppression for genuine fatal structured errors.

## Reproduction Assessment

Yes, at source level: current `main` still gates recovered `⚠️ 🛠️` tool-warning recovery on `runLevelError` and channel final-text preference, then returns before delivery dispatch when the warning remains fatal. I did not run the live WSL2/Feishu scheduled setup in this read-only review.

## Solution Assessment

Yes, the best fix is a narrow cron payload-outcome/final-text selection repair, not a new config option or broad status rewrite. The issue should stay open until one linked PR or a replacement lands and proves that recovered final output reaches delivery while true fatal errors remain suppressed.
