## What This Changes

The issue reports scheduled isolated cron agent turns skipping delivery when an earlier recovered tool error is treated as fatal despite final assistant output.

## Best Possible Solution

Land one narrow cron payload-outcome fix that treats recovered tool-error runs with final assistant output as non-fatal for delivery while preserving genuine fatal structured-error and run-level failure handling.

## Reproduction Assessment

Yes, at source level: current main still gates recovered tool-warning recovery on `preferFinalAssistantVisibleText`, and the fatal structured-payload branch returns before dispatch when that warning remains fatal. I did not run the live WSL2/Feishu scheduled setup in this read-only review.

## Solution Assessment

Yes, the best direction is a narrow classifier fix in `resolveCronPayloadOutcome`. A new config flag or broad cron status rewrite is larger than this specific pre-dispatch skip requires.
