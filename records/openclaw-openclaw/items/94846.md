## Summary

Keep open: current `main` and `v2026.6.9` still gate recovered tool-warning recovery on channel final-text preference and still return before delivery dispatch when that warning remains fatal; open closing PRs are still the right path to resolve it.

## What This Changes

The issue reports that scheduled isolated `agentTurn` cron delivery is skipped when a recovered early tool error is still treated as fatal before dispatch.

## Best Possible Solution

Review and land one narrow cron payload-outcome fix that classifies recovered tool warnings with final output as non-fatal for delivery, while preserving true fatal structured-error suppression and genuine run-level failures.

## Reproduction Assessment

Yes at source level: current main still gates recovered `⚠️ 🛠️` tool-warning recovery on `preferFinalAssistantVisibleText`, and the fatal structured-payload branch returns before delivery dispatch when that warning remains fatal. I did not run the live WSL2/Feishu scheduled setup in this read-only review.

## Solution Assessment

Yes, the narrow maintainable solution is to fix cron payload outcome classification so recovered tool warnings with final output do not suppress dispatch. A new config flag or broad cron status rewrite would be larger than this specific bug requires.
