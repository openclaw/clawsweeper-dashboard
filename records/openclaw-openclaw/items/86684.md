## Summary

Keep open: current main and the latest release still contain the successful-turn overflow predicate that can trust inflated `usage.input`, and compaction entries still do not persist a trigger cause; the June provider-independent comments make this an active session-state bug rather than stale cleanup.

## What This Changes

The issue requests that subagent completion or `sessions_yield` wake-ups avoid compacting a low-usage parent branch unless real overflow or manual compaction is present, and asks for persisted compaction trigger metadata.

## Best Possible Solution

Make parent-wake compaction decisions use a trustworthy final-call or context snapshot, persist a closed trigger reason on compaction entries, and prove subagent-yield wake on current main with provider-independent coverage.
