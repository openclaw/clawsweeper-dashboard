## Summary

Keep open: current main and latest v2026.6.9 still allow successful-turn inflated usage to trigger overflow compaction, and compaction transcript entries still cannot record the trigger needed to distinguish parent wake, overflow, budget, or manual paths.

## What This Changes

The issue asks OpenClaw to stop subagent completion or `sessions_yield` wake-up handling from compacting the parent branch at low real context usage and to persist compaction trigger metadata.

## Best Possible Solution

Use a trustworthy last-call/context snapshot or provider-aware sanity-checked usage for parent-wake compaction decisions, persist a closed compaction trigger reason, and prove the subagent-yield wake path on current main.

## Reproduction Assessment

Yes at source level: a successful assistant message with inflated `usage.input + cacheRead` still trips `isContextOverflow` and starts overflow auto-compaction; the issue comments add real low-context subagent-wait examples, but no live current-main run was executed in this read-only pass.

## Solution Assessment

Unclear for automation: the requested direction is valid, but the best fix crosses provider/runtime usage attribution, overflow semantics, compaction trigger metadata, and live subagent wake proof.
