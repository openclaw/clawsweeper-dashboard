## Summary

Keep open: current main and v2026.6.8 still trust successful-turn `usage.input + cacheRead` for overflow compaction, while compaction records still lack trigger-cause metadata and recent provider-independent comments keep the low-context subagent-wake compaction report active.

## What This Changes

The issue requests that subagent completion or `sessions_yield` wake-ups stop compacting parent branches at low real context usage and record durable compaction trigger metadata.

## Best Possible Solution

Make parent-wake compaction decisions use a trustworthy last-call or sanity-checked context snapshot, persist a closed compaction trigger reason, and prove the subagent-yield wake path on current main with provider-independent coverage.

## Reproduction Assessment

Yes at source level: current main still has the successful-turn overflow predicate that can trust inflated `usage.input`, and recent comments provide concrete low-context compaction logs. I did not run a live current-main subagent-yield scenario in this read-only review.

## Solution Assessment

Unclear as a single automated patch: the desired direction is valid, but the best fix crosses usage attribution, overflow-provider semantics, compaction trigger metadata, and subagent wake proof.
