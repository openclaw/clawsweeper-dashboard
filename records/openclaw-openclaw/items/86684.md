- `impact:session-state`: The affected behavior changes parent transcript branch state during yielded subagent wake handling.
- `impact:data-loss`: Unexpected compaction can discard exact parent-session history even when visible context usage is low.

## Summary

Keep open: current main still has the successful-turn overflow decision that can compact from `usage.input + cacheRead`, while compaction entries still lack a trigger/cause field and are parented to the active transcript leaf. The June 13 provider-independent comment makes this a current session-state bug, not a stale or implemented report.

## What This Changes

The issue asks OpenClaw to stop compacting yielded parent sessions during subagent wake/direct-announce handling unless there is real context overflow, and to record the compaction trigger cause.

## Best Possible Solution

Define the maintainer-owned compaction trigger contract, then fix the shared session/runtime path so parent wake turns use reliable per-call context accounting and cannot compact at low real usage; add trigger metadata to future compaction entries.

## Reproduction Assessment

Partly: current source still has the implicated successful-turn overflow decision and hidden-yield compaction-parenting surface, and the issue now has two live-data signatures. I did not run a deterministic current-main repro, so this is source-reproducible rather than fully reproduced.

## Solution Assessment

Unclear as an automated repair: the requested end state is right, but maintainers should choose whether the primary fix is usage-accounting isolation, an overflow sanity guard, trigger metadata, or a combined repair across yielded wake paths.
