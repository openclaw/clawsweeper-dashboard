## Label Justifications

- `P1`: The issue reports a high-impact regression that can unexpectedly compact parent-session history during real subagent workflows.
- `impact:session-state`: The affected behavior is yielded parent transcript state, hidden branch leaves, and compaction parentage during subagent wake-up.
- `impact:data-loss`: Unexpected compaction can remove exact parent-session history even when visible context usage is far below the model window.

## Summary

Keep open: current main still has the source-backed path where inflated successful-turn usage can trigger overflow auto-compaction, the yielded-session hidden context can remain as the compaction leaf, and compaction entries still do not persist the requested trigger cause. The merged cron accounting fix is adjacent, but it does not cover this parent/subagent wake contract.

## What This Changes

The issue asks OpenClaw to stop compacting yielded parent sessions during subagent completion wake-up at low real context usage and to record compaction trigger metadata.

## Best Possible Solution

Fix the shared yielded-parent/subagent wake compaction contract: use reliable final-call/current-prompt usage for overflow decisions, ignore impossible usage snapshots, persist trigger cause on compaction entries, and prove the parent does not compact at low context.

## Reproduction Assessment

Yes, source-reproducible but not live-reproduced here: the issue has detailed live logs, a provider-independent follow-up, and current main still contains the overflow, hidden-yield, and compaction-entry behaviors that explain the reports.

## Solution Assessment

Yes on the requested end state, but the best fix should span overflow sanity, usage attribution, and compaction trigger metadata rather than patching only one caller.
