## Label Justifications

- `P1`: The issue reports a high-impact regression that can unexpectedly compact parent-session history during real subagent workflows.
- `impact:session-state`: The affected surface is yielded parent transcript state, hidden branch leaves, and compaction parentage during subagent wake-up.
- `impact:data-loss`: Unexpected compaction can remove exact parent-session history even when visible context usage is far below the model window.

## Summary

Keep open: current main still trusts successful-turn `usage.input` for overflow compaction, the shared session runtime still auto-compacts from that classification, and compaction transcript entries still lack trigger cause; the related cron accounting fix only addressed a narrower writer path.

## What This Changes

The issue asks OpenClaw to stop low-context parent-session compactions during `sessions_yield` subagent wake-ups and to persist compaction trigger metadata.

## Best Possible Solution

Implement a shared core fix that uses reliable final-call/current-prompt usage for parent overflow decisions, ignores impossible usage snapshots, records compaction cause on persisted entries or checkpoints, and proves yielded parents do not compact at low context.

## Reproduction Assessment

Yes, source-reproducible: the issue and follow-up comment provide live data, and current source still has the successful-turn overflow and missing trigger-metadata paths. I did not run a deterministic current-main live repro in this read-only review.

## Solution Assessment

Yes on the requested end state, but the best fix should span overflow sanity, usage attribution, and compaction trigger metadata rather than patching only `sessions_yield`. Maintainers should decide the successful-turn overflow contract before automation changes core session behavior.
