## Summary

Keep open: current main and v2026.6.8 still retain the successful-turn overflow predicate that can trust inflated `usage.input`, and compaction entries still do not persist trigger-cause metadata; recent provider-independent comments keep the report active rather than stale.

## What This Changes

The issue asks OpenClaw to stop subagent wake/direct-announce compactions at low real context usage and persist compaction trigger metadata for diagnosis.

## Best Possible Solution

Make parent-wake compaction decisions use a trustworthy context snapshot or sanity-checked usage, persist a closed trigger reason on compaction entries, and prove subagent-yield wake on current main with provider-independent coverage.

## Reproduction Assessment

Yes at source level: current main still has the successful-turn overflow predicate that can trust inflated `usage.input`, and comments provide concrete low-context compaction logs. I did not run a live current-main subagent-yield scenario in this read-only review.

Unclear for a single automated patch: the requested direction is valid, but the best fix crosses usage attribution, overflow-provider semantics, compaction trigger metadata, and subagent wake proof.
