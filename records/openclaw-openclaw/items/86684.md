## Label Justifications

- `P1`: The issue reports a high-impact regression that can corrupt or compact active parent-session history during real subagent workflows.
- `impact:session-state`: The affected behavior is yielded parent transcript state, hidden branch leaves, and compaction parentage during subagent wake-up.
- `impact:data-loss`: Unexpected compaction can remove exact parent-session history even when visible context usage is far below the model window.

## Summary

Keep open: current main still has a source-backed path where inflated successful-turn usage can trigger overflow compaction, hidden `sessions_yield` context can remain as a transcript leaf, and compaction entries still lack the requested trigger cause. The merged cron accounting PR is adjacent but does not fix this parent/subagent wake contract.

## What This Changes

The issue asks OpenClaw to stop compacting low-context yielded parent branches during subagent completion wake-ups and to persist compaction trigger metadata.

## Best Possible Solution

Fix the shared yielded-parent/subagent wake compaction contract: use reliable final-call/current-prompt usage for overflow decisions, ignore impossible usage snapshots, persist trigger cause on compaction entries, and prove the parent does not compact at low context.

## Reproduction Assessment

Yes, source-backed but not fully live-reproduced on current main: the issue has detailed live logs and current source still contains the overflow, hidden-yield, and compaction-entry behaviors that explain the reports.

## Solution Assessment

Yes on the end state, but the best fix should span overflow sanity, usage attribution, and compaction trigger metadata rather than patching only one caller.
