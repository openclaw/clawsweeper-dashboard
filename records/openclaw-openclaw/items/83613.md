## Summary

Keep open: current main still lets official Docker release builds prune the Codex plugin unless the release keep list includes it, and an open linked PR already proposes the focused fix. The issue should track review and landing of that PR rather than be closed or queued as a second repair.

## What This Changes

Official Docker images should retain the bundled `codex` plugin so the default OpenAI agent harness is present in runtime images.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/83626 or an equivalent fix that retains Codex in official Docker release images, preserves the PI fallback as defense, and adds image/runtime regression proof.

## Reproduction Assessment

Yes: source inspection gives a high-confidence path because current main excludes Codex from package dist, prunes excluded plugin dirs unless kept, and official release jobs keep only `diagnostics-otel`. I did not build the full image during this read-only review.

## Solution Assessment

Yes: retaining Codex in the official Docker release keep list is the narrow maintainable fix because the documented default OpenAI agent path is Codex-backed. The fallback from https://github.com/openclaw/openclaw/pull/82532 should remain defensive rather than becoming the normal official-image path.
