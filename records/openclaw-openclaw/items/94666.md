## What This Changes

The issue reports that a Control UI cleanup action can misclassify the active Gateway process tree as stale and trigger a systemd restart.

## Best Possible Solution

Route any Control UI cleanup affordance through safe restart/preflight or an ACPX orphan reaper, and make stale-port cleanup unable to terminate the active Gateway process tree.

## Solution Assessment
