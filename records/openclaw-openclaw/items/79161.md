## Summary

Keep open: this PR moves the fix into the right central metadata helper, but it still needs real behavior proof and appears to miss the npm config/~/.npmrc mirror setup described by the linked issue.

## What This Changes

Adds a registry-base resolver in `src/infra/update-check.ts` that uses `npm_config_registry`/`NPM_CONFIG_REGISTRY` for update metadata URLs, with focused unit coverage.

## Best Possible Solution

Keep the central metadata-fetch fix, but make the registry resolver cover the reported package-manager config path and preserve existing startup timeout/error behavior.

## Reproduction Assessment

Yes. Source inspection shows current main hardcodes `registry.npmjs.org`, and the proposed branch only handles env-var mirrors, leaving the reported npm config/~/.npmrc path unproven.

## Solution Assessment

No. The central helper is the right boundary, but env-only resolution is not the narrowest complete fix for the linked mirror-config bug; the safer path is a bounded shared resolver for env plus configured package-manager registry.
