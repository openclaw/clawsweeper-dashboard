  - body: The new background alive throttle stores one global `gateway.backgroundAlive.lastSuccessAtMs` value, so a successful beacon for one gateway can suppress beacon delivery for another active gateway profile for ten minutes while still connected.   - confidence: 0.89

  - body: The gateway throttle drops events whenever `receivedAtMs - lastPersistedAt` is below the interval, so a backward wall-clock jump makes the difference negative and suppresses legitimate alive events until time catches up. Only throttle when the new timestamp is at or after the stored timestamp, or reset stale entries on clock regression.
  - confidence: 0.78
