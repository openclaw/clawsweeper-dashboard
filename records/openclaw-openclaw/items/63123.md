Full review comments:

- **[P1] Normalize presence reasons before persistence:** `src/gateway/server-node-events.ts:695-707`
  - body: The branch persists `normalizeOptionalString(obj.trigger)` directly into node/device pairing metadata, so a compromised or buggy node can store arbitrarily large or unexpected `lastSeenReason` values. Use the closed reason normalization now present on main before writing durable state.
  - confidence: 0.9
- **[P1] Bound throttle keys after approved updates:** `src/gateway/server-node-events.ts:717`
  - body: `recentNodePresencePersistAt` is an unbounded module-level map and this branch records the key after pairing update calls that can no-op for unknown identities. High-cardinality node event traffic can grow memory and drive repeated pairing I/O; bound the map and only record approved updates.
  - confidence: 0.87
- **[P2] Scope recent-success throttling per gateway:** `apps/ios/Sources/Model/NodeAppModel.swift:3875-3885`
  - body: The iOS recent-success check reads one global `UserDefaults` timestamp, so a successful beacon for one gateway profile can suppress a beacon for another active gateway. Key the timestamp by stable gateway identity, as the shipped replacement effectively avoids this stale branch shape.
  - confidence: 0.78
