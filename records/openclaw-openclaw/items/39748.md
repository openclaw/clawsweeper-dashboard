## Best Possible Solution

Use one Nostr-local relay URL normalizer/validator across direct config and setup paths, preserving or explicitly migrating currently normalized aliases with focused tests and redacted real setup proof.

## Reproduction Assessment

## Solution Assessment

No. The direct-config-only schema is not the best merge shape because it can break normalized aliases and leaves setup validation on a separate contract; the safer fix is a shared validator/normalizer with upgrade proof.
