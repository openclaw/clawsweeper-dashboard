## Summary

Keep open: this PR addresses a real Nostr relay validation gap, but it is not merge-ready. Current main still accepts arbitrary direct relay strings, and the PR’s stricter schema does not cover the setup paths that can persist remote plaintext ws:// relays.

## What This Changes

The PR adds a Nostr relay URL schema that allows wss:// plus loopback ws://, applies it to relays, adds tests under the old types.test.ts file, and updates secrets baseline line numbers.

## Best Possible Solution

Update the PR to share one Nostr-local relay URL validator across direct config, guided setup, and dependency-light setup, with current-main regression coverage for setup persistence.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main direct config accepts arbitrary relay strings, and both setup parsers accept any ws: or wss: URL before persisting relays.

## Solution Assessment

No. The schema validator is the right building block, but the maintainable fix should reuse the same relay policy in direct config and both setup paths so setup cannot save a value the schema rejects.
