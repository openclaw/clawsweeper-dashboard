## Best Possible Solution

Use one Nostr-local relay URL normalizer/validator across direct config and both setup paths, with upgrade-safe handling for currently normalized aliases and proof for fresh and existing config.

## Reproduction Assessment

Yes. This is source-reproducible: current direct config accepts arbitrary relay strings, both setup parsers accept any `ws:` or `wss:` URL, and `nostr-tools@2.23.3` normalizes `https:` to `wss:` before connecting.

## Solution Assessment

No. The direct schema validator is useful, but the maintainable solution needs one shared validator/normalizer across all relay entry points plus an explicit upgrade policy for dependency-normalized aliases.
