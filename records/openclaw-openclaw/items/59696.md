## Summary

Keep open: the hardening direction is valid, but this external PR has no after-fix real behavior proof and it still leaves the same path-only Host-derived URL parser in the MCP loopback request path on current main.

## What This Changes

The PR replaces Host-derived URL constructor bases with the literal `http://localhost` in five Gateway HTTP handlers that parse request path or query data.

## Best Possible Solution

Land a focused Gateway hardening that covers every core path/query-only request URL parser, including MCP loopback, with regression coverage for hostile Host headers while preserving intentional Host/Origin trust checks.

## Reproduction Assessment

Yes. Source inspection on current main is a high-confidence path: the five target handlers and the adjacent MCP request validator still derive URL constructor bases from `req.headers.host` while consuming only `pathname` or `searchParams`.

## Solution Assessment

No, not quite as submitted. The literal localhost base is the right narrow fix, but the PR should also cover `src/gateway/mcp-http.request.ts`, add/confirm focused hostile-Host coverage, and provide real behavior proof before merge.
