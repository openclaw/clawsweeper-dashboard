The PR replaces Host-derived URL constructor bases with `"http://localhost"` in five Gateway HTTP handlers used for pathname and query parsing.

## Best Possible Solution

Land a rebased focused hardening that covers every core Gateway request URL parser which only consumes path/query data, including MCP loopback, while leaving intentional Host/Origin trust checks unchanged.

## Reproduction Assessment

Yes. Source inspection on current main is a high-confidence reproduction: the target handlers and the adjacent MCP loopback validator still derive URL constructor bases from `req.headers.host` while consuming only `pathname` or `searchParams`.

## Solution Assessment

No, not quite as submitted. The literal localhost base is the right narrow fix, but the PR should also cover `src/gateway/mcp-http.request.ts` and resolve the dirty branch state before landing.
