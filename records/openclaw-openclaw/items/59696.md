## Summary

Keep this PR open. Current main has not implemented the five Host-header URL-base replacements, and the proposed direction matches existing Gateway parsing patterns, but the PR still leaves one adjacent Gateway HTTP request validator using the same Host-derived base, so the standardization is not complete yet.

## What This Changes

The PR replaces Host-derived URL constructor bases with the literal "http://localhost" in five Gateway HTTP handler files used for pathname and query parsing.

## Best Possible Solution

Land a focused hardening that covers every core Gateway request URL parser which only consumes `pathname` or `searchParams`, including the MCP loopback validator, while leaving intentional Host/Origin trust checks unchanged.

## Reproduction Assessment

Yes. The high-confidence reproduction is source inspection on current main: `rg` finds the Host-derived URL bases in the five target files and the adjacent MCP request validator, and the affected code paths only read path/query data from those constructed URLs.

## Solution Assessment

No, not quite as submitted. The literal localhost base is the narrow maintainable fix, but the PR should also update `src/gateway/mcp-http.request.ts` or explicitly narrow its scope so maintainers do not treat the Gateway-wide sweep as complete.
