## Summary

Keep open: the PR is still useful, but current main only partially covers the hardening and the branch is not merge-ready. Main now uses a localhost base for several listed handlers, but `/tools/invoke` still derives the URL base from `Host`, the adjacent MCP path parser still has the same pattern, and the external PR still lacks real behavior proof.

## What This Changes

The PR replaces Host-derived `URL` constructor bases with the literal `http://localhost` in five Gateway HTTP handlers.

## Best Possible Solution

Land a refreshed focused Gateway hardening that keeps the current localhost-base fixes, updates `/tools/invoke` and MCP path-only parsers, and adds hostile-Host regression coverage while preserving intentional Host and Origin trust checks.

## Reproduction Assessment

Yes. Source inspection on current main plus a Node URL check gives a high-confidence path: Host-derived URL bases can throw on malformed Host input, while a localhost base preserves the path/query values these handlers consume.

## Solution Assessment

No, not as submitted. The literal localhost base is the right narrow direction, but the PR needs a branch refresh, real behavior proof, and coverage for the remaining path-only parsers before it is the best mergeable solution.
