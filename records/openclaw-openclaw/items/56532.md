## Summary

Keep open: the PR is useful but not merge-ready because the advertised retry budget is still bypassed on the timeout-bounded auto-recall path, the docs/fallback wording lag the new config, and the real-environment proof shows healthy deployed recall rather than the timeout/retry failure path.

## What This Changes

The PR adds memory-lancedb embedding timeout/retry config, schema, UI hints, runtime wiring for the direct OpenAI-compatible client, config tests, and a changelog entry.

## Best Possible Solution

Keep the config plugin-local, honor the retry budget on every direct OpenAI-compatible embedding path including auto-recall, document defaults/ranges/trust scope, and merge after fault-path proof plus rebase validation.

## Reproduction Assessment

Yes for the source-level config gap: current main lacks timeoutMs/maxRetries config and the PR adds them to parser/manifest/runtime. No for the live failure mode: this review did not reproduce a hung or 5xx embedding backend, and the PR proof does not fault-inject one.

## Solution Assessment

No, not as-is. The plugin-local config boundary is right, but the implementation must carry maxRetries through timeout-bounded auto-recall requests instead of letting the per-request maxRetries: 0 override the new setting.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Honor maxRetries on auto-recall embedding requests:** `extensions/memory-lancedb/index.ts:363`
  - body: The PR adds a client-level maxRetries default/config knob, but before_prompt_build calls embed() with a timeoutMs option, and embed() still posts with maxRetries: 0 whenever timeoutMs is present. OpenAI 6.37.0 uses request maxRetries over the client default, so the main auto-recall path advertised in the PR body cannot retry on 429/5xx even when maxRetries is configured. Carry the configured retry budget into the per-request options or stop overriding it for this path.
  - confidence: 0.91
- **[P3] Document the new embedding request budget settings:** `extensions/memory-lancedb/openclaw.plugin.json:41-50`
  - body: The manifest exposes embedding.timeoutMs and embedding.maxRetries, but the public memory-lancedb docs still only describe baseUrl and dimensions for the direct OpenAI-compatible path. Add the defaults, allowed ranges, and custom-endpoint scope so operators can tune the new behavior safely.
  - confidence: 0.86
- **[P3] Align fallback wording with runtime defaults:** `extensions/memory-lancedb/config.ts:207-217`
  - body: These comments say invalid timeout/retry values fall back to SDK defaults, and the changelog repeats that, but the parser returns undefined and the runtime then applies this PR’s 10s/1-retry OpenClaw defaults while the manifest rejects out-of-range config. Update the comments/changelog or change the fallback behavior.
  - confidence: 0.84
