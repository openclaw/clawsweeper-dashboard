## Summary

Keep open. The PR is still useful because current main does not expose `memory-lancedb` embedding timeout/retry config, but the latest head still needs maintainer review for the custom-baseUrl retry trust model plus small docs/wording cleanup before merge.

## What This Changes

This PR adds `embedding.timeoutMs` and `embedding.maxRetries` to the `memory-lancedb` config parser, manifest schema/UI hints, OpenAI-compatible embedding client construction, config tests, and changelog.

## Best Possible Solution

Land a scoped memory-lancedb update that keeps the direct OpenAI-compatible timeout/retry behavior, documents the new options and trust model, and makes fallback wording match the actual OpenClaw defaults.

## Reproduction Assessment

Yes. Source inspection on current main shows timeoutMs/maxRetries are rejected by both the runtime parser and manifest schema, and the direct OpenAI-compatible client lacks config-backed timeout/retry construction.

## Solution Assessment

Unclear. The PR targets the right plugin-owned direct client path and uses supported OpenAI SDK options, but it is not the best final shape until docs, fallback wording, and the custom-baseUrl retry trust model are resolved.
