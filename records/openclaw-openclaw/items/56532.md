## Summary

Keep open: the direct OpenAI-compatible embedding client change is plausible and the supplied terminal proof is useful, but the new docs still show a provider-auth configuration path where these knobs are not wired, and the invalid-value wording does not match parser/schema behavior.

## What This Changes

The PR adds opt-in `embedding.timeoutMs` and `embedding.maxRetries` config for memory-lancedb OpenAI-compatible embedding calls, plus config schema, UI metadata, docs, changelog, and focused config tests.

## Best Possible Solution

Align the user-facing surface with the actual runtime path: document these as direct `embedding.apiKey`/`baseUrl` OpenAI-compatible client knobs, or intentionally wire timeout/retry through the provider adapter contract with focused coverage.

## Reproduction Assessment

Yes, at source/dependency level: current main constructs the direct OpenAI embedding client without a configurable constructor timeout/retry budget, and OpenAI 6.38.0 defaults to a 600000ms timeout with 2 retries. I did not run a full Gateway scenario in this read-only review.

## Solution Assessment

Mostly yes: opt-in constructor options preserve upgrade compatibility for unset direct-client installs and match the OpenAI SDK contract. The remaining issue is that docs and UI wording must match the direct-client scope, or the provider adapter must be intentionally extended.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Make the budget example use a bounded runtime path:** `docs/plugins/memory-lancedb.md:255-258`
  - body: This example omits `embedding.apiKey`/`baseUrl`, so `memory-lancedb` routes through `ProviderAdapterEmbeddings` rather than the `OpenAiCompatibleEmbeddings` constructor this PR changes. That provider path calls `embedQuery(text)` and the provider contract only carries an `AbortSignal`, so users following the documented config will not get the advertised request budget. Either make the example use the direct OpenAI-compatible client path or wire these settings through provider adapters.
  - confidence: 0.9
- **[P3] Describe invalid budget values accurately:** `docs/plugins/memory-lancedb.md:242-245`
  - body: The new docs say out-of-range values are clamped, but the runtime parser falls back to `undefined` and the manifest schema rejects out-of-range values. Please change this wording so operators know whether a bad value is rejected by config validation or ignored back to SDK defaults, rather than expecting min/max clamping.
  - confidence: 0.88
