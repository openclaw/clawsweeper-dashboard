## Summary

Keep open: this is a useful memory-lancedb availability/config improvement, but the latest branch still exposes the new budget knobs more broadly than the runtime implements, especially for provider-auth embeddings and auto-recall wording.

## What This Changes

The PR adds optional `memory-lancedb` `embedding.timeoutMs` and `embedding.maxRetries` config, docs/manifest/changelog/tests, and passes configured values into the direct OpenAI-compatible embedding client.

## Best Possible Solution

Ship the direct-client timeout/retry knobs with docs and manifest text scoped to the implemented path, or deliberately extend the provider embedding contract with focused tests before presenting the knobs as generic provider controls.

## Reproduction Assessment

Yes, at source/dependency level. Current main lacks configurable direct-client constructor timeout/retry, and the remaining PR mismatches are visible from source because provider-auth embeddings ignore the fields and auto-recall still uses a fixed per-request timeout.

## Solution Assessment

Not yet. The direct-client implementation is a maintainable narrow path, but the public docs and manifest need to match that runtime scope unless maintainers intentionally extend the provider embedding contract.

Full review comments:

- **[P2] Do not promise auto-recall uses this configurable budget:** `docs/plugins/memory-lancedb.md:232-235`
  - body: The new docs say these knobs protect auto-recall, but auto-recall still calls `embeddings.embed()` with `DEFAULT_AUTO_RECALL_TIMEOUT_MS`; the direct client then sends that as a per-request timeout with `maxRetries: 0`, overriding the constructor values added here. A user setting `embedding.timeoutMs: 2000` will still get the fixed 15s auto-recall budget, so either wire this config into auto-recall with tests or scope the docs away from auto-recall.
  - confidence: 0.9
- **[P2] Make the budget example use the bounded client:** `docs/plugins/memory-lancedb.md:254-258`
  - body: This example omits `embedding.apiKey`, so `createEmbeddings()` takes the provider-auth route and returns `ProviderAdapterEmbeddings`; that path calls `embedQuery(text)` without the new timeout/retry values. Users copying the example can still hang on provider defaults, so make it use the direct OpenAI-compatible client shape or extend and test the provider contract.
  - confidence: 0.93
- **[P2] Scope budget UI hints to the direct client:** `extensions/memory-lancedb/openclaw.plugin.json:41-50`
  - body: These manifest hints are shown as generic embedding request controls, but provider-backed configs that omit `embedding.apiKey` ignore `timeoutMs` and `maxRetries`. Scope the help text to the direct OpenAI-compatible client path or wire the provider adapter path before exposing the fields as general controls.
  - confidence: 0.88
- **[P3] Describe invalid budget values as rejected or ignored:** `docs/plugins/memory-lancedb.md:244-245`
  - body: The docs say out-of-range values are clamped, but the parser sets invalid values to `undefined` and the manifest schema rejects out-of-range numeric values. Please change this wording so operators know invalid settings are rejected or fall back to SDK defaults rather than being coerced to the nearest boundary.
  - confidence: 0.9
