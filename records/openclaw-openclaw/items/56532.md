## Summary

Keep open. Current main does not implement configurable memory-lancedb embedding timeout/retry settings, and this PR is live source work, but it still needs maintainer review for the direct custom-endpoint retry trust model plus small docs/wording cleanup.

## What This Changes

The PR adds memory-lancedb embedding timeout/retry config parsing, manifest schema/UI hints, direct OpenAI-compatible client wiring, config tests, and a changelog entry.

## Best Possible Solution

Land a scoped memory-lancedb update that adds documented timeout/retry knobs for direct OpenAI-compatible embeddings and records the accepted custom-endpoint trust model.

## Reproduction Assessment

Yes. Source inspection of current main shows `embedding.timeoutMs` and `embedding.maxRetries` are rejected by both the runtime parser and manifest schema, and the direct OpenAI-compatible client has no config-backed timeout/retry constructor options.

## Solution Assessment

Unclear. The PR targets the owner plugin and uses supported OpenAI SDK options, but it is not the best final shape until public docs, fallback wording, and the custom-baseUrl retry policy are resolved.
