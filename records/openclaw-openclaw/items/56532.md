## What This Changes

The PR adds `embedding.timeoutMs` and `embedding.maxRetries` config, schema/manifest entries, direct OpenAI-compatible constructor wiring, config tests, and a changelog entry for `memory-lancedb`.

## Best Possible Solution

Ship the direct OpenAI-compatible timeout/retry knobs with manifest, UI, and changelog copy scoped to `embedding.apiKey`, or deliberately extend the provider embedding contract with tests before advertising provider-wide support.

## Reproduction Assessment

Yes. Source inspection plus the PR's terminal blackhole proof show the direct OpenAI-compatible client can fail fast with configured timeout/retry, and source inspection shows provider-backed embeddings do not receive those settings.

## Solution Assessment

No, not quite yet. The constructor wiring is a narrow maintainable solution for direct clients, but the public surface must either be scoped to that path or backed by a provider-contract extension.
