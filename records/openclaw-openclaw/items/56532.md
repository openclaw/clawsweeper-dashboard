## Summary

## What This Changes

The PR adds `embedding.timeoutMs` and `embedding.maxRetries` to memory-lancedb config/schema/manifest, wires them into the direct OpenAI-compatible client only when explicitly configured, adds config tests, and updates the changelog.

## Best Possible Solution

Define the config scope explicitly: either make the validated knobs affect every OpenAI-compatible memory-lancedb embedding path that advertises them with tests/proof, or narrow schema/help/changelog to direct plugin-apiKey requests while preserving the existing auto-recall guard.

## Reproduction Assessment

## Solution Assessment

Unclear: preserving SDK defaults for unset config is the right upgrade-safe direction, but the PR should either wire the configured budget into the paths it advertises or narrow the feature wording and tests to the direct OpenAI client path.
