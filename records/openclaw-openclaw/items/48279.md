## Summary

Keep open. Current main still advertises ACP session listing and pins `@agentclientprotocol/sdk` 0.21.0, but `AcpGatewayAgent` only implements `unstable_listSessions` while the SDK dispatches `session/list` to `agent.listSessions`. The latest release is still affected, the direct fix PR #48292 is closed unmerged, and open PR #73458 is related ACP metadata work rather than a replacement fix.

## What This Changes

The issue reports that ACP `session/list` fails because OpenClaw implements `unstable_listSessions` while ACP SDK v0.16+ dispatches to `listSessions`.

## Best Possible Solution

Expose `listSessions` on `AcpGatewayAgent` using the existing Gateway-backed `sessions.list` logic, keep `unstable_listSessions` as a thin compatibility alias if TypeScript permits it, and add dispatcher-level regression coverage for JSON-RPC `session/list`.

## Reproduction Assessment

Yes. The reporter’s stdin JSON-RPC command is a concrete reproduction path, and the static current-main path confirms it: `openclaw acp` constructs an SDK `AgentSideConnection`, SDK v0.21.0 dispatches `session/list` to `agent.listSessions`, and current `AcpGatewayAgent` does not define that method.

## Solution Assessment

Yes. The narrow maintainable fix is to add the SDK-required `listSessions` method backed by the existing implementation and test it through the dispatcher; a simple direct rename is close, but a delegating alias is safer around older callers and related ACP session work.
