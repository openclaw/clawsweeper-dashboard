## Summary

Keep open. Current main still has the ACP SDK v0.16+ handler mismatch: OpenClaw advertises session listing and depends on `@agentclientprotocol/sdk` 0.21.0, but `AcpGatewayAgent` only implements `unstable_listSessions`; the direct fix PR is closed unmerged and the related lineage PR does not replace this regression fix.

## What This Changes

The issue asks OpenClaw ACP to handle `session/list` through the SDK v0.16+ `listSessions` method instead of the old `unstable_listSessions` name.

## Best Possible Solution

Expose `listSessions` on `AcpGatewayAgent` using the existing Gateway-backed session-list logic, keep the old name as a delegating alias if practical, and add dispatcher-level regression coverage for `session/list`.
