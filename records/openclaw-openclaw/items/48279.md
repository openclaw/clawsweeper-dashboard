## Summary

Current main implements the SDK-required ACP `listSessions` handler with regression coverage, so the original `session/list` method-not-found regression is fixed on main. The fix is not in the latest tagged release, so users on `v2026.5.7` still need a newer build or the next release.

## What This Changes

The issue asks OpenClaw to update ACP session listing from the removed `unstable_listSessions` handler name to the stable `listSessions` handler required by ACP SDK v0.16+.

## Best Possible Solution

Keep the merged current-main ACP lifecycle implementation and let the next release ship it; handle the older open fix PR as superseded by the merged main implementation.

## Reproduction Assessment

Yes for affected released builds: the reporter's JSON-RPC stdin command and the SDK v0.21 dispatcher contract reproduce why `session/list` failed when only `unstable_listSessions` existed. Current main no longer reproduces the mismatch because `AcpGatewayAgent.listSessions` is present and covered by lifecycle tests.

## Solution Assessment

Yes. The merged fix uses the SDK-required stable method name, keeps the implementation Gateway-backed, adds bounded pagination and cwd validation, and covers the handler contract in tests; a compatibility alias is not needed for the pinned SDK 0.21.0 surface.
