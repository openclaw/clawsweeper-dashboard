## Summary

Keep open. Current main still matches the reported lifecycle bug: a non-pending, non-yield `agent.wait` result is treated as terminal subagent completion, freezes output from the child history immediately, and the later same-session refresh only helps before cleanup has completed. The related open PR #75462 is adjacent error-recovery work, not a compaction/resume completion transfer fix.

## What This Changes

The issue requests that internal subagent session compaction/resume not fire the parent-visible completion event or freeze `frozenResultText` until the child task actually ends.

## Best Possible Solution

Add a compaction-aware lifecycle contract so internal compaction/resume does not satisfy parent-visible subagent completion. The final behavior should announce exactly once, after the resumed child run reaches terminal state, while preserving existing `sessions_yield`, retry-grace, frozen-result, and cleanup semantics.

## Reproduction Assessment

Yes. The issue provides three timestamped compaction incidents where the parent completion preceded the worker's later final assistant message, plus a no-compaction control; current source still shows the matching wait/freeze path.

## Solution Assessment

Yes. Keeping the task lifecycle pending across compaction/resume is the narrowest maintainable solution; only extending the output-capture retry or refreshing text after cleanup would leave successful premature announces uncorrected.
