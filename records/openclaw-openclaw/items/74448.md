## Summary

Keep open. Current main still has a credible premature-completion path: compaction emits `stream: "compaction"` with `willRetry`, but the subagent registry only reacts to lifecycle events and still treats an `agent.wait` `ok` result as terminal, freezing whatever child output is visible at that moment.

## What This Changes

The issue reports that subagent completion is announced during session compaction instead of after the resumed child run produces its final handoff.

## Best Possible Solution

Make compaction retry nonterminal for subagent completion so the tracked task remains pending across compaction and announces exactly once from the resumed run's real terminal output.

## Reproduction Assessment

Yes. The issue gives concrete production reproductions, and current source provides a focused regression path: emit compaction `willRetry` plus a lifecycle `end`/`agent.wait` `ok` and assert that no parent completion is announced until the resumed terminal result is available.

## Solution Assessment

Yes. Fixing the lifecycle contract is the narrowest maintainable direction; extending only the output-capture retry window would still allow a successful completion event before the child task is actually done.

## Repair Work Prompt

Fix the subagent completion contract reported in https://github.com/openclaw/openclaw/issues/74448. On current main, compaction retry must not be treated as terminal completion for a subagent task: a parent completion event should be emitted exactly once, after the resumed child run produces its real terminal handoff. Keep the fix in the generic subagent/runtime lifecycle surfaces, not in a provider or channel-specific workaround. Likely areas are `src/agents/pi-embedded-subscribe.handlers.compaction.ts`, `src/agents/pi-embedded-subscribe.handlers.lifecycle.ts`, `src/gateway/server-methods/agent-job.ts`, `src/agents/subagent-registry.ts`, `src/agents/subagent-registry-run-manager.ts`, `src/agents/subagent-registry-lifecycle.ts`, and `src/agents/subagent-announce-output.ts`. Add focused regression coverage that models compaction `willRetry` plus an apparent lifecycle/agent.wait end before the resumed final answer, and assert that no mid-task frozen result is announced. Preserve existing yielded-run behavior and delivery retry behavior. Add a user-facing `CHANGELOG.md` fix entry unless the final patch is test-only. Do not add a config flag or broaden plugin/channel behavior.
