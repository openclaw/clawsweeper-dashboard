## Summary

Keep open. Current main still has a credible subagent lifecycle bug: completion can be driven by a terminal-looking wait/lifecycle result while compaction is tracked on a separate stream, and the registry can freeze and announce whatever child output is visible at that moment.

## What This Changes

The issue requests that subagent completion remain pending across mid-task session compaction and announce only after the resumed subagent actually finishes.

## Best Possible Solution

Add a generic compaction-aware subagent lifecycle contract: compaction/resume keeps the parent-visible run pending, preserves a single final announce after the resumed terminal state, and leaves sessions_yield, retry grace, cleanup retry, orphan recovery, and frozen-output caps intact.

## Reproduction Assessment

Yes. The issue gives three timestamped compaction incidents and a no-compaction control, and current source supports a focused regression that simulates compaction followed by an old-run wait/lifecycle terminal and asserts no parent announce until the resumed run ends.

## Solution Assessment

Yes. The preferred direction, not resolving parent-visible completion on compaction, is the narrowest maintainable fix; refreshing frozenResultText later would still allow an incorrect successful completion event to reach the parent.
