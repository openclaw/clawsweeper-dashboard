## Summary

Keep open. Current main still has a source-visible false-success path where compaction retry is tracked separately, but a later lifecycle end can still become `agent.wait` `ok` and subagent completion `ok` before the resumed child produces the real final handoff.

## What This Changes

The issue asks OpenClaw to prevent subagent completion from resolving or announcing during session compaction and freezing mid-task output as the final result.

## Best Possible Solution

Carry compaction-resume continuation through `agent.wait` and subagent registry outcome handling so the parent receives exactly one completion from the resumed run’s real terminal output, with regression tests for compaction retry and frozen-result capture.

## Reproduction Assessment

Yes, at source level. The reporter provides concrete production traces, and current main still maps a non-aborted lifecycle `end` to successful wait and subagent completion without treating compaction retry as a pending continuation.

## Solution Assessment

Yes. Fixing the lifecycle/wait contract is the maintainable solution; longer frozen-result polling or output heuristics would still allow a success event to be emitted before the child task actually ends.
