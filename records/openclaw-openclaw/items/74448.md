## Summary

Keep open. Current main still has a source-visible path where a compaction retry is emitted separately from lifecycle completion, while `agent.wait` and the subagent registry can still map the old run's lifecycle `end` to a successful completion before the resumed worker's final handoff is available.

## What This Changes

The issue asks OpenClaw to avoid treating subagent auto-compaction/resume as a completed child task and to deliver the real final handoff after resumed work ends.

## Best Possible Solution

Make compaction retry nonterminal for subagent completion: carry the resume chain through `agent.wait` and the registry so the parent receives exactly one completion from the resumed run's real terminal output.

## Reproduction Assessment

Yes at source level. The reporter gives concrete production traces, and current main still maps lifecycle `end`/`agent.wait` `ok` to successful subagent completion without treating compaction retry as a pending continuation.

## Solution Assessment

Yes, fixing the lifecycle/wait contract is the best direction. Output-only retries or longer frozen-result polling would still allow a successful completion event to fire before the child task actually ends.
