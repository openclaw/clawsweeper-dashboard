## Summary

Keep open. Current main still has a source-level path where an embedded compaction retry can emit a normal lifecycle terminal event for the old run id, and the subagent registry treats that `agent.wait` result as final completion before freezing output from the child transcript. The related open PR #75462 is adjacent completion-error recovery work and does not address compaction/resume lifecycle transfer.

## What This Changes

The issue asks OpenClaw to keep subagent completion pending across mid-task session compaction so the parent receives the worker's real final handoff instead of a pre-compaction assistant fragment.

## Best Possible Solution

Add a compaction-aware lifecycle contract so internal compaction/resume cannot satisfy parent-visible subagent completion. The run should announce exactly once, after the resumed child turn reaches a real terminal state, while preserving existing output-freeze and cleanup behavior for normal terminal runs.

## Reproduction Assessment

Yes. The issue provides three timestamped compaction incidents plus a no-compaction control, and current source still shows a deterministic reproduction path: emit `compaction_end` with `willRetry: true`, then an intermediate `agent_end`, and observe that `agent.wait` can resolve before the retry/final turn completes.

## Solution Assessment

No, current main is not the best solution because it waits for compaction retry inside the runner but still emits a terminal lifecycle event that `agent.wait` treats as final. The narrower maintainable fix is to suppress or explicitly mark intermediate compaction-retry lifecycle events so `agent.wait` and subagent completion remain pending until the final resumed turn ends.
