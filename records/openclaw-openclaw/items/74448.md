## Summary

Keep open. Current main still has a credible source-level premature-completion path: compaction retry is emitted on a separate stream, while `agent.wait` and the subagent registry still treat a lifecycle `end`/`ok` for the old run as terminal and freeze the currently visible child output.

## What This Changes

The issue asks OpenClaw to keep subagent completion pending across session compaction so the parent receives the actual final handoff, not a mid-task assistant fragment.

## Best Possible Solution

Make compaction retry nonterminal for subagent completion, preserving one pending tracked task across the resume chain and announcing exactly once from the resumed run's true terminal output.

## Reproduction Assessment

Yes, source-level. Emit a compaction `willRetry` event followed by a lifecycle `end`/`agent.wait` `ok` for a registered completion subagent, then assert no parent completion is delivered until the resumed terminal run has final output.

## Solution Assessment

Yes. Fixing the lifecycle/wait contract is the narrowest maintainable direction; only extending the output-capture retry window would still allow a successful completion event before the child task is actually done.

## Repair Work Prompt

Fix issue #74448: subagent completion must not fire when session compaction ends the old run before the resumed child task reaches its real terminal handoff. Keep the boundary in the core agent/gateway lifecycle code; do not add a config option or product mode. Make compaction retry nonterminal for `agent.wait`/subagent completion, or carry enough resume-chain state so the subagent registry suppresses the pre-compaction completion and announces exactly once after the resumed terminal output is available. Add regression coverage that registers a completion subagent, emits/observes compaction `willRetry` plus lifecycle `end`/`agent.wait ok`, verifies no parent announce/frozen result is produced from the mid-task assistant text, then verifies the resumed terminal output is announced once. Consider adjacent open PR #75462 as related diagnostics work, not a fix for this lifecycle contract. Add a user-facing changelog fix entry unless the final change is tests-only.
