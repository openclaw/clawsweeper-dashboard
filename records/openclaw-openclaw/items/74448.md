## Summary

Keep open. Current main still has a plausible premature subagent completion path: `agent.wait` terminal success is treated as final completion, the child result is frozen immediately from current child history, and compaction events are tracked on a separate stream without a wait-transfer or resume-chain contract. The related open PR #75462 is adjacent error-recovery work, not a closing fix for compaction-triggered early completion.

## What This Changes

The issue asks OpenClaw to keep subagent completion pending across session compaction/resume so the parent receives the actual final child handoff instead of a mid-task frozen result.

## Best Possible Solution

Add a compaction-aware lifecycle contract so internal compaction/resume does not satisfy parent-visible subagent completion. The subagent should announce exactly once, after the resumed child run reaches a real terminal state, while preserving existing `sessions_yield`, retry-grace, deferred-announce, and cleanup behavior.

## Reproduction Assessment

Yes. The issue provides three timestamped compacted-session incidents plus a no-compaction control, and current source still exposes the matching wait/freeze/announce path. A focused regression can synthesize lifecycle completion during compaction before the final child assistant message and assert no parent completion is emitted until the resumed run finishes.

## Solution Assessment

Yes. Carrying the task lifecycle across compaction/resume, or suppressing terminal completion while a compaction resume chain is active, is the narrow maintainable fix. Merely extending the frozen-output polling window would still allow a successful premature completion event to be delivered and cleaned up.
