## Summary

Keep open. Current main still has a source-visible false-success path: compaction retry is emitted separately, while lifecycle `end` can still become `agent.wait` `ok` and the subagent registry can still complete the old run before the resumed child has produced its final handoff.

## What This Changes

The issue reports that subagent completion is emitted during session compaction before the resumed child task has actually finished.

## Best Possible Solution

Carry compaction-resume continuation through `agent.wait` and subagent registry outcome handling so the parent receives exactly one completion from the resumed run’s real terminal output.

## Reproduction Assessment

Yes at source level. The reporter provides concrete production traces, and current main still maps a non-aborted lifecycle `end` to successful subagent completion without making compaction retry a pending continuation.

## Solution Assessment

Yes. Fixing the lifecycle/wait contract is the maintainable solution; longer frozen-result polling or output heuristics would still allow a success event to be emitted before the child task actually ends.
