## Summary

Keep open. Current main still has a credible premature-completion path: non-pending, non-yielded `agent.wait` results become terminal subagent completion, while compaction/retry events are emitted separately and are not used to keep the tracked run pending through resume.

## What This Changes

Ensure subagent completion waits through session compaction/resume so parents receive one final handoff instead of a mid-task `frozenResultText`.

## Best Possible Solution

Treat compaction/retry/resume as nonterminal for subagent runs, continue or remap the wait to the resumed run, and freeze/announce exactly once from the actual terminal resumed result.

## Reproduction Assessment

Yes. The issue includes timestamped production reproductions, and current code supports a focused regression that simulates `agent.wait` returning `ok` around compaction/resume and asserts no parent completion announce until the resumed run ends.

## Solution Assessment

Yes. Fixing the lifecycle contract is the narrowest maintainable solution; extending output-capture retries alone could still deliver a successful completion event before the child task is done.

## Repair Work Prompt

Fix the subagent completion contract for https://github.com/openclaw/openclaw/issues/74448. Observable bug: when a child session compacts mid-task, the parent can receive a successful completion event with a mid-task `frozenResultText` before the resumed child run finishes. Keep the fix scoped to subagent lifecycle/wait handling: compaction retry/resume must be nonterminal for completion-mode subagent runs, the tracked run should continue/remap to the resumed run or suppress the first completion, and `frozenResultText` must be captured once from the actual terminal resumed result. Preserve `sessions_yield` paused behavior, lifecycle error grace, orphan recovery, deferred announce retry, cleanup retry, and existing delivery fallback behavior. Treat PR #75462 and issue #75196 as adjacent context only; do not fold broad completion-error hardening into this PR unless directly needed. Add focused regression coverage that simulates the compaction/resume signal shape and proves no parent completion announce occurs until the resumed run ends. Add a single-line user-facing `CHANGELOG.md` fix entry for the active Unreleased Fixes section.
