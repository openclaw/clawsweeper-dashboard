## Summary

Keep open. Current main still has a credible premature-completion path: lifecycle `end` or `agent.wait` `ok` results are treated as terminal, the registry freezes the latest visible child text immediately, and later same-session refreshes only help while the original completion run remains pending and uncleaned.

## What This Changes

Ensure subagent completion waits across auto-compaction/resume and delivers the actual final child handoff instead of freezing mid-task output.

## Best Possible Solution

Make compaction/retry nonterminal for subagent completion by carrying the tracked task across the compaction resume chain and freezing or announcing exactly once from the resumed terminal result.

## Reproduction Assessment

Yes. The issue gives three timestamped production reproductions plus a no-compaction control, and a focused regression can simulate compaction retry/lifecycle end followed by a resumed terminal assistant handoff.

## Solution Assessment

Yes. Fixing the lifecycle contract is the narrowest maintainable direction; merely extending the output-capture retry would still allow a successful completion event before the child task actually finishes.

## Repair Work Prompt

Fix the current subagent compaction completion bug tracked by https://github.com/openclaw/openclaw/issues/74448. Current main can complete and announce a subagent when the original run emits lifecycle end / `agent.wait` ok during auto-compaction, freezing a mid-task latest assistant message before the resumed run produces its real final handoff. Keep the fix in the subagent lifecycle/wait boundary: make compaction/retry/resume nonterminal for completion, carry or remap the tracked subagent task across the resumed run as needed, and freeze/announce exactly once from the real terminal output. Do not solve this only by increasing the output retry window. Preserve `sessions_yield`, lifecycle error/timeout grace, descendant announce deferral, orphan recovery, cleanup retries, and announce idempotency. Add a regression that simulates compaction retry/lifecycle end followed by a resumed final assistant response and proves the parent does not receive the pre-compaction mid-task text. Include a user-facing `CHANGELOG.md` fix entry; do not touch unrelated plugin/channel behavior.
