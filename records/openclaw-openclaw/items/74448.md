## Summary

Keep open. Current main still has a credible premature-completion path: compaction/retry is emitted on a separate stream, while both `agent.wait` and the subagent registry treat a lifecycle `end`/`ok` result as terminal and freeze the current child output immediately.

## What This Changes

The issue reports that subagent completion is announced during session compaction instead of after the resumed child task actually finishes.

## Best Possible Solution

Make compaction/retry nonterminal for subagent completion by keeping the tracked child run pending, carrying or remapping the wait through the resumed turn, and freezing/announcing exactly once from the true terminal result.

## Reproduction Assessment

Yes. The issue gives timestamped production reproductions, and a focused regression can simulate `agent.wait`/lifecycle `ok` around a compaction `willRetry` event and assert no parent completion is announced until the resumed run reaches a real terminal end.

## Solution Assessment

Yes. Fixing the lifecycle contract is the narrowest maintainable direction; extending output-capture retries alone would still let parents receive a false successful completion before the task is done.
