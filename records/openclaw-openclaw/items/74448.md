## Summary

Keep open. Current main still has a credible premature-completion path: `agent.wait` results are treated as terminal unless explicitly `pending` or `yielded`, while compaction is emitted on a separate stream and does not keep the subagent registry pending through resume before freezing and announcing output.

## What This Changes

Keep parent-visible subagent completion pending across auto-compaction/resume so the parent receives the child’s actual final handoff.

## Best Possible Solution

Make compaction/resume nonterminal for subagent completion: keep the tracked task pending, remap or continue the wait to the resumed turn, and freeze/announce exactly once from the terminal resumed result.

## Reproduction Assessment

Yes. The issue gives timestamped production reproductions, and a focused test can simulate `agent.wait` returning `ok` around a compaction/resume sequence and assert the registry does not announce until the resumed run reaches a real terminal state.

## Solution Assessment

Yes. Fixing the lifecycle contract is narrower and safer than only extending the output-capture retry, because the parent must not receive any successful completion event before the child task is actually finished.
