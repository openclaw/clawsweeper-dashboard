## Summary

Keep open. Current main still has a credible premature-completion path: lifecycle `end` events can make `agent.wait` report `ok`, and the subagent registry then freezes and announces the currently visible child output without proving that a compaction retry/resume chain has reached the real terminal handoff.

## What This Changes

The issue reports that subagent completion is announced and `frozenResultText` is captured when session compaction ends the current run, before the resumed worker actually finishes.

## Best Possible Solution

Make compaction/retry nonterminal for subagent completion so the tracked task remains pending across compaction and freezes or announces exactly once from the resumed run’s real terminal output.

## Reproduction Assessment

Yes. The issue gives concrete production reproductions, and a focused regression can emit compaction retry plus lifecycle end events and assert that the registry does not announce until the resumed terminal result is available.

## Solution Assessment

Yes. Fixing the lifecycle contract is the narrowest maintainable direction; extending the output-capture retry alone would still allow a successful completion event before the child task actually finishes.

## Repair Work Prompt

Fix the subagent compaction/resume completion contract for issue #74448. Current main can treat an `agent.wait` `ok` result around session compaction retry/resume as terminal, then freeze `frozenResultText` from the child session before the resumed worker actually finishes. Keep the solution in the generic subagent lifecycle/runtime boundary, likely `src/agents/subagent-registry-run-manager.ts`, `src/agents/subagent-registry.ts`, `src/agents/subagent-registry-lifecycle.ts`, `src/agents/subagent-announce-output.ts`, `src/agents/pi-embedded-subscribe.handlers.lifecycle.ts`, and/or `src/gateway/server-methods/agent-job.ts`; do not add plugin-specific behavior. Preserve `sessions_yield` paused-state semantics, lifecycle error/timeout grace, orphan recovery, descendant announce deferral, cleanup retry paths, and the current completion announcement shape. Add a focused regression that models compaction retry making the original wait/lifecycle path look successful while the resumed run is still active, and assert the parent receives exactly one completion event with the resumed terminal output and real timing/stats. Add a user-facing `CHANGELOG.md` fix entry if runtime behavior changes.
