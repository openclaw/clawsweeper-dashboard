## Summary

Keep open. Current main still has a source-visible false-success path where a compaction retry can be followed by lifecycle completion of the old run, and both `agent.wait` and the subagent registry can still normalize that terminal signal to successful completion before the resumed child produces the real handoff.

## What This Changes

The issue reports that subagent completion is emitted when a worker session compacts mid-task, causing the parent to freeze an intermediate result instead of the real final handoff.

## Best Possible Solution

Carry compaction-resume continuation through `agent.wait` and subagent registry outcome handling so the parent receives exactly one completion from the resumed run's real terminal output.

## Reproduction Assessment

Yes at source level. The reporter provides concrete production traces, and current main still maps a non-aborted lifecycle `end` to successful subagent completion without making compaction retry a pending continuation for the parent-facing wait contract.

## Solution Assessment

Yes, fixing the lifecycle/wait contract is the best direction. Longer frozen-result polling or output heuristics would still allow a success event to be emitted before the child task actually ends.
