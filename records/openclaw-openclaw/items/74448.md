## Summary

Keep open. Current main still has a source-visible premature success path: compaction retry is emitted separately, lifecycle `end` and `agent.wait` can still become `ok`, and the subagent registry can complete the old run before the resumed child has produced its final handoff.

## What This Changes

The issue reports that subagent completion is emitted during session compaction before the resumed child task actually finishes.

## Best Possible Solution

Carry compaction-resume continuation through `agent.wait` and subagent registry outcome handling so the parent receives exactly one completion from the resumed run's real terminal output.

## Reproduction Assessment

Yes at source level. The reporter supplied concrete production traces, and current main still maps a non-aborted lifecycle `end` to `agent.wait`/registry success without making compaction retry a pending continuation.

## Solution Assessment

Yes, fixing the lifecycle/wait contract is the best direction. Frozen-result polling or completion-text heuristics would still allow an old-run success event to be emitted before the child task actually ends.
