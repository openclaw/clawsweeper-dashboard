## Summary

Keep open. Current main still has the same parent-visible premature completion shape: a lifecycle `end`/`agent.wait` success can finalize the subagent run and freeze the latest child transcript text without a compaction/resume-chain check, while compaction handling remains separate from the completion contract.

## What This Changes

The issue requests that subagent completion only be reported after the child task truly finishes, not when session compaction resumes the worker mid-task.

## Best Possible Solution

Add a compaction-aware lifecycle contract so internal compaction/retry does not satisfy parent-visible `agent.wait` or subagent completion. The final run should freeze and announce exactly once after the resumed child reaches a real terminal state, while preserving `sessions_yield` and existing retry-grace semantics.

## Reproduction Assessment

Yes. The issue includes a high-confidence reproduction pattern with three compaction incidents and timestamped parent/child divergence; current source inspection shows the matching wait/freeze path still exists.

## Solution Assessment

Yes. The narrowest maintainable fix is to keep `agent.wait` or the subagent registry pending across compaction/resume, rather than extending the 1.5s output-capture retry after the wrong terminal signal has already fired.
