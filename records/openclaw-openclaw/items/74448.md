## Summary

Keep open. Current main still has a credible premature-completion path: compaction retry events are tracked, but a retry-side `agent_end` still emits lifecycle `end`, `agent.wait` maps lifecycle `end` to `ok`, and the subagent registry treats that as terminal completion before the resumed/follow-up work has produced the real handoff.

## What This Changes

Make subagent completion and `agent.wait` treat compaction retry as nonterminal so the parent receives the real final handoff instead of an intermediate assistant message.

## Best Possible Solution

Make compaction/retry nonterminal for the public wait and subagent-completion contract, then announce exactly once from the resumed run's real terminal output.

## Reproduction Assessment

Yes. The issue gives three production reproductions, and current tests can be adapted into a focused regression by emitting `compaction_end` with `willRetry: true` plus `agent_end` and asserting that `agent.wait` and the subagent registry remain pending until the later terminal handoff.

## Solution Assessment

Yes. Fixing the lifecycle/wait contract is the narrowest maintainable solution; only extending frozen-output polling would still allow a false successful completion before the child task is actually done.
