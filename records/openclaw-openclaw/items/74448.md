## Summary

Keep open. Current main still has a source-visible path where compaction retry is emitted separately, while lifecycle and agent.wait completion paths can still turn a non-error terminal event into a successful subagent completion before the resumed work's final handoff is available.

## What This Changes

The issue reports that subagent completion can be announced during session compaction before the resumed child task actually finishes.

## Best Possible Solution

Make compaction retry nonterminal for subagent completion so a tracked subagent announces exactly once from the resumed run's real terminal output, while coordinating adjacent raw-output and blocked-liveness fixes.

## Reproduction Assessment

Yes at source level. The issue provides concrete production traces, and current main still has a focused path where compaction `willRetry` is separate from lifecycle while lifecycle/agent.wait/subagent completion can map the old run to `ok`.

## Solution Assessment

Yes, fixing the lifecycle/wait contract is the best direction. Extending only output capture or retry timing would still allow a successful completion event to be emitted before the child task is actually done.
