## Summary

Keep open. Current main still has a credible premature-completion path: compaction retry state is emitted on a non-lifecycle stream, while `agent.wait` and the subagent registry treat an ordinary lifecycle `end` as terminal and can freeze the latest visible child output before the resumed task handoff exists.

## What This Changes

The issue asks for subagent completion to remain pending across session compaction so the parent receives the resumed run's final handoff instead of a mid-task snapshot.

## Best Possible Solution

Make compaction/retry nonterminal for subagent completion, preserving one tracked task across compaction and announcing only from the resumed run's real terminal output.

## Reproduction Assessment

Yes. The issue provides concrete production reproductions, and a focused regression can emit `compaction_end` with `willRetry: true` followed by lifecycle `end` and assert that no parent completion is announced until the resumed terminal handoff is available.

## Solution Assessment

Yes. Fixing the lifecycle/wait contract is the narrowest maintainable direction; only extending the output-capture retry window would still allow a successful completion event before the child task is actually done.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/74448. The observable bug is that a subagent can announce successful completion during session compaction/retry, freezing a mid-task assistant line as `frozenResultText`, while the child continues and later produces the real handoff. Keep the fix in the subagent/runtime lifecycle boundary: make compaction retry nonterminal for `agent.wait` and subagent completion, or carry enough compaction-resume state so the registry suppresses the pre-retry lifecycle `end` and announces exactly once after the resumed terminal run. Do not solve this only by increasing the completion-output polling window. Likely files: `src/agents/pi-embedded-subscribe.handlers.compaction.ts`, `src/agents/pi-embedded-subscribe.handlers.lifecycle.ts`, `src/gateway/server-methods/agent-job.ts`, `src/agents/subagent-registry.ts`, `src/agents/subagent-registry-run-manager.ts`, `src/agents/subagent-announce-output.ts`, and focused colocated tests. Add a user-facing `CHANGELOG.md` fix entry. Preserve current behavior for true terminal `ok`, `error`, `timeout`, and `yielded` runs.
