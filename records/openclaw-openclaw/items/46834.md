## Summary

Keep open: the underlying ACP direct-chat result gap is still relevant, but this PR is not correct as-is because it bypasses the current task-completion delivery model and has concrete type-safety and output-corruption blockers.

## What This Changes

The PR adds a direct `routeReply` path in the ACP parent stream relay that buffers assistant output and routes final/error text to non-threaded parent chat delivery contexts, with relay tests.

## Best Possible Solution

Integrate final ACP result text into the parent-owned task completion path so requester routing, idempotency, threaded behavior, whitespace, fallback delivery, and tests are handled in one place.

## Reproduction Assessment

Yes for the PR defects by source inspection: `cfg` can be undefined at the `routeReply` call, whitespace-only deltas are possible from the current emitter, and current main already has task terminal delivery that this patch does not coordinate with. I did not run a live WhatsApp/ACP smoke for the original direct-chat scenario.

## Solution Assessment

No, not as-is. The direct-reply idea addresses the visible gap, but the maintainable fix should feed final child output into the current task-completion delivery model instead of adding an independent terminal send path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Require cfg before direct replies:** `src/agents/acp-spawn-parent-stream.ts:301`
  - body: `cfg` is declared as possibly `undefined` after the best-effort parent-session lookup, but this call passes it to `routeReply`, whose params require `cfg: OpenClawConfig`. Keep the config and target in one narrowed object, or skip direct delivery unless both are present.
  - confidence: 0.92
- **[P2] Append whitespace-only deltas before filtering:** `src/agents/acp-spawn-parent-stream.ts:364-365`
  - body: The direct-reply buffer is appended only after the existing `!delta.trim()` return, so standalone space/newline chunks are dropped. A stream split as `"hello"`, `" "`, `"world"` would be delivered as `helloworld`; buffer raw final-output deltas before the progress-only trim filter.
  - confidence: 0.9
- **[P2] Avoid double-sending terminal completion:** `src/agents/acp-spawn-parent-stream.ts:394-400`
  - body: Current main already creates a parent-owned ACP task and delivers a terminal requester notification. This new lifecycle-end direct reply sends raw child output without marking that task delivery handled or sharing its idempotency path, so non-threaded chats can receive both the raw answer and the task completion message.
  - confidence: 0.84
