## Summary

Keep open. Current main still does not implement this PR's raw final-output relay for non-threaded direct chats; it only has generic ACP task-completion delivery. The proposed branch has blocking correctness issues and overlaps newer parent-owned task delivery semantics, so it should not be closed or merged as-is.

## What This Changes

The PR adds a best-effort ACP parent-stream relay path that loads the parent delivery context, buffers child assistant deltas, and calls `routeReply` for non-threaded direct-chat completions and errors, with new relay tests.

## Best Possible Solution

Integrate the useful behavior into the current parent-owned ACP delivery model deliberately: preserve the requester route, deliver the final visible child result exactly once when product policy wants raw child output, avoid threaded-surface duplication, keep config and target narrowing together, preserve whitespace-only streamed deltas, and cover the task-notifier interaction in tests.

## Reproduction Assessment

Yes for the PR defects: the cfg issue is visible from the PR head source and the current `routeReply` contract, and the whitespace bug reproduces by streaming child deltas like `"hello"`, `" "`, `"world"` before lifecycle end. The original direct-chat report has concrete WhatsApp ACP steps, but this review did not run a live channel smoke.

## Solution Assessment

No, not as-is. The direct-reply idea may solve the original raw-output gap, but the patch should be rebased onto current ACP task-delivery semantics and fixed so direct replies are type-safe, whitespace-preserving, and not duplicative with terminal task notifications.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Require cfg before direct replies:** `src/agents/acp-spawn-parent-stream.ts:302`
  - body: `cfg` is declared as possibly `undefined` after the best-effort `loadSessionEntry` catch, but this call passes it to `routeReply`, whose params require `cfg: OpenClawConfig`. With strict TypeScript this should fail to typecheck; keep the config and target in one narrowed object or skip direct send unless both are present.
  - confidence: 0.92
- **[P2] Append whitespace-only deltas before filtering:** `src/agents/acp-spawn-parent-stream.ts:363-367`
  - body: The direct-reply buffer is appended only after the existing `!delta.trim()` return, so standalone spaces or newlines emitted as their own assistant chunks are dropped. A stream like `"hello"`, `" "`, `"world"` would be delivered as `helloworld`; buffer the raw final-output delta before the progress-only trim filter.
  - confidence: 0.9
- **[P2] Avoid double-sending terminal completion:** `src/agents/acp-spawn-parent-stream.ts:396-400`
  - body: Current main already creates a pending ACP task for `streamTo="parent"` runs and the task registry sends a terminal message to the requester origin. This new lifecycle-end direct reply sends raw child output without marking that task delivery handled or folding the text into the task path, so direct chats can receive both the raw answer and a generic background-task completion.
  - confidence: 0.82
