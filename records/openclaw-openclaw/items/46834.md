## Summary

Keep open. The PR targets a real ACP direct-chat final-delivery gap, but current main has moved parent-owned ACP delivery through the task registry/parent-notifier model, and this branch still has blocking cfg narrowing, whitespace preservation, and duplicate terminal-delivery risks.

## What This Changes

The PR adds direct `routeReply` delivery of accumulated ACP child output or error text from parent stream relays to non-threaded direct-chat parent sessions, with relay tests.

## Best Possible Solution

Integrate final ACP result delivery into the parent-owned task completion model deliberately, preserving exact-once delivery, requester routing, threaded-surface behavior, whitespace, and tests for notifier interaction.

## Reproduction Assessment

Yes for the PR defects by source inspection: the cfg narrowing issue follows from `routeReply`'s required `cfg`, whitespace loss follows from buffering after the trim filter, and duplicate delivery follows from the existing ACP task terminal path. The original WhatsApp scenario has concrete steps, but this review did not run a live channel smoke.

## Solution Assessment

No, not as-is. The direct-reply idea may address the raw-output gap, but it should be reconciled with current parent task delivery semantics and fixed so it is type-safe, whitespace-preserving, and exact-once.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Require cfg before direct replies:** `src/agents/acp-spawn-parent-stream.ts:302`
  - body: `cfg` is declared as possibly `undefined` after the best-effort session lookup, but this call passes it to `routeReply`, whose params require `cfg: OpenClawConfig`. With strict TypeScript this should fail; keep the config and target in one narrowed object or skip direct send unless both are present.
  - confidence: 0.92
- **[P2] Append whitespace-only deltas before filtering:** `src/agents/acp-spawn-parent-stream.ts:363-367`
  - body: The direct-reply buffer is appended only after the existing `!delta.trim()` return, so standalone spaces or newlines emitted as their own assistant chunks are dropped. A stream like `"hello"`, `" "`, `"world"` would be delivered as `helloworld`; buffer the raw final-output delta before the progress-only trim filter.
  - confidence: 0.9
- **[P2] Avoid double-sending terminal completion:** `src/agents/acp-spawn-parent-stream.ts:396-400`
  - body: Current main already creates a parent-owned ACP task and delivers a terminal requester notification. This new lifecycle-end direct reply sends raw child output without marking that task delivery handled or folding the text into that path, so direct chats can receive both the raw answer and the task completion message.
  - confidence: 0.82
