## Summary

Keep open for maintainer review: the reported direct-chat gap has real supporting context, but this PR is not mergeable as-is against the current ACP task-delivery model and has source-inspected correctness defects.

## What This Changes

The PR adds a direct-reply buffer in the ACP parent stream relay that sends child final/error output to non-threaded parent delivery contexts and extends relay tests.

## Best Possible Solution

Define the ACP final-result policy once and route successful child output through the parent-owned task completion/announce path with idempotency, sanitization, threaded behavior, and regression coverage handled together.

## Reproduction Assessment

Yes for the PR defects by source inspection: the optional cfg is passed to a required routeReply parameter, whitespace-only deltas are possible from the current emitter, and current main already has task terminal delivery that the PR bypasses. I did not establish a fresh live direct-chat reproduction for the original WhatsApp scenario.

## Solution Assessment

No, not as-is. The direct-route buffer addresses the visible complaint, but the narrower maintainable direction is to coordinate final ACP result delivery with the current task-completion model instead of adding an independent terminal send path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Require config before direct replies:** `src/agents/acp-spawn-parent-stream.ts:302`
  - body: `cfg` is optional after the best-effort parent-session lookup, but this call passes it to `routeReply`, whose params require `cfg: OpenClawConfig`. Keep the config and direct target in one narrowed object, or skip direct delivery unless both are present, so the patch type-checks under the repo's strict TS settings.
  - confidence: 0.92
- **[P2] Preserve whitespace-only direct reply deltas:** `src/agents/acp-spawn-parent-stream.ts:367`
  - body: The new direct-reply buffer appends only after `!delta.trim()` returns, so standalone space/newline chunks are dropped. Current assistant streaming can produce deltas from `cleanedText.slice(...)`, so a stream split as `"hello"`, `" "`, `"world"` would be delivered as `helloworld`.
  - confidence: 0.9
- **[P2] Avoid bypassing task terminal delivery:** `src/agents/acp-spawn-parent-stream.ts:396-400`
  - body: Current main already creates a parent-owned ACP task and delivers terminal updates with requester-origin routing and idempotency. Sending raw child output here on lifecycle end can double-send or disagree with the task completion message instead of sharing the same delivery state.
  - confidence: 0.84
