## Summary

Keep open: current main has ACP task-completion delivery, but it still does not implement this PR's intended raw final-output relay for non-threaded direct chats. The proposed branch also has blocking correctness issues around TypeScript narrowing and whitespace-preserving streamed output, so it needs maintainer review and a rebase rather than cleanup closure.

## What This Changes

The PR adds a best-effort ACP parent-stream relay path that loads the parent delivery context, buffers child assistant deltas, and calls `routeReply` for non-threaded direct-chat completions and errors, with new relay tests.

## Best Possible Solution

Integrate the useful behavior into the current parent-owned ACP delivery model deliberately: preserve the requester route, deliver the final visible child result exactly once when product policy wants raw child output, avoid threaded-surface duplication, keep config and target narrowing together, preserve whitespace-only streamed deltas, and cover the task-notifier interaction in tests.

## Reproduction Assessment

Yes for the PR defects: the cfg issue is visible from the PR head source and the current `routeReply` contract, and the whitespace bug reproduces by streaming child deltas like `"hello"`, `" "`, `"world"` before lifecycle end. The original direct-chat report has concrete WhatsApp ACP steps, but this review did not run a live channel smoke.

## Solution Assessment

No, not as-is. The direct-reply idea may solve the original gap, but the patch should first be rebased onto current ACP task-delivery semantics and fixed so direct replies are type-safe, whitespace-preserving, and not duplicative with terminal task notifications.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Require cfg before direct replies:** `src/agents/acp-spawn-parent-stream.ts:302`
  - body: `cfg` is declared as possibly `undefined` after the best-effort `loadSessionEntry` catch, but the new `routeReply` call passes it to a required `OpenClawConfig` parameter. With strict TypeScript this should fail to typecheck; keep the config and target in one narrowed object or skip direct send unless both are present.
  - confidence: 0.92
- **[P2] Append whitespace-only deltas before filtering:** `src/agents/acp-spawn-parent-stream.ts:363-367`
  - body: The direct-reply buffer is appended only after the existing `!delta.trim()` return, so standalone spaces or newlines emitted as their own assistant chunks are dropped. A stream such as `"hello"`, `" "`, `"world"` would be delivered as `helloworld`; buffer the raw final-output delta before the progress-only trim filter.
  - confidence: 0.9
