## Summary

Keep open: this PR targets a real P1 exec-approval trust bug, but it is protected/security-sensitive and still has a portable test failure, a doctor-diagnostic mismatch, and only test/check proof rather than real behavior proof.

## What This Changes

The branch changes exec allowlists, safe-bin trust, skill auto-allow, Allow Always persistence, approval audit metadata, tests, and changelog to use executable realpaths as the trust identity.

## Best Possible Solution

Land a corrected version that uses one realpath trust contract across runtime checks, persistence/audit metadata, doctor diagnostics, and regression coverage, with real behavior proof and maintainer/security signoff.

## Reproduction Assessment

Yes. Current main is source-reproducible because allowlist matching uses the PATH-resolved path while approved execution pins to the executable realpath, so a symlinked binary with a distinct realpath exercises the mismatch.

## Solution Assessment

No, not as-is. Realpath canonicalization is the right direction, but the branch needs the portable safe-bin test fixed, doctor diagnostics aligned with runtime trust, and real behavior proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Use an exact trusted safe-bin realpath in the test:** `src/infra/exec-approvals-safe-bins.test.ts:358`
  - body: This injected trustedDirs set is not normalized. On a case-sensitive platform, the resolved directory `/opt/homebrew/Cellar/...` will not equal lowercase `/opt/homebrew/cellar/...`, so this expectation fails; use the exact-cased realpath dir or construct the set through the normal trusted-dir normalization path.
  - confidence: 0.9
- **[P2] Align doctor safe-bin hints with realpath trust:** `src/infra/exec-approvals-allowlist.ts:109-110`
  - body: Runtime safe-bin trust now checks the executable trust path, but `scanExecSafeBinTrustedDirHints()` still checks only the PATH-resolved candidate and builds trusted dirs without the safeBins target expansion. A runtime-accepted realpath trust config can still produce a doctor warning, so the diagnostic should use the same trust-path contract.
  - confidence: 0.84
