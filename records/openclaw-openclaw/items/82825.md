## Summary

Keep open: this PR targets a valid P1 exec approval bug, but the latest head still has merge-blocking review findings, a protected maintainer label, and no real behavior proof beyond tests/checks.

## What This Changes

The PR changes exec approval allowlists, safe-bin trust, skill auto-allow, Allow Always persistence, approval audit metadata, tests, and changelog to use executable realpaths as the trust identity.

## Best Possible Solution

Land a corrected version that uses one realpath trust contract across runtime checks, persistence/audit metadata, doctor diagnostics, and existing-approval repair, with redacted real behavior proof and maintainer/security signoff.

## Reproduction Assessment

Yes. Source inspection of current main shows path-shaped allowlist entries are matched against resolvedPath, while node-host execution pins resolvedRealPath ?? resolvedPath, so a symlinked PATH binary with a distinct realpath exercises the mismatch.

## Solution Assessment

No, not as-is. Realpath canonicalization is the right direction, but the PR needs an upgrade path for existing symlink-path approvals, doctor/runtime alignment, and real behavior proof before merge.
