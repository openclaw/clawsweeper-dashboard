## Summary

Keep this open: current main still has the same source-level no-channel cron path that the prior review identified, and the related cron fix PRs are closed without merging. The latest release tag also still defaults implicit isolated agent-turn cron delivery to announce/last, so this is not implemented on main or shipped.

## What This Changes

Cron jobs in local/no-channel setups should complete and preserve file-output prompts instead of failing implicit announce delivery with a required-channel error.

## Best Possible Solution

Cron-owned implicit/no-channel isolated runs should avoid mandatory fallback announce delivery unless the user explicitly chooses a deliverable target, while explicit announce and webhook delivery remain strict.

## Reproduction Assessment

Yes, at source level. A default Control UI isolated agent-turn cron job still submits announce delivery on channel "last", and backend-created isolated agent-turn jobs still default to announce, which reaches the reported no-channel resolver error when there is no previous or configured channel.

## Solution Assessment

Not applicable as a patch review. The narrow maintainable fix is to handle implicit no-channel cron delivery in the UI/default path or target-resolution downgrade path while preserving strict behavior for explicit delivery targets.
