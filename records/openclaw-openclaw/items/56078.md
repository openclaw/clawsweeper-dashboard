## What This Changes

The issue reports that cron jobs in local/webchat-only OpenClaw 2026.3.24 runs execute but fail to save requested output because delivery channel resolution errors when no external channels are configured.

## Best Possible Solution

Cron-owned implicit/no-channel isolated runs should avoid mandatory fallback announce delivery unless the user explicitly chooses a deliverable target, while explicit announce and webhook delivery stay strict.

## Reproduction Assessment

Yes, source-level. A default Control UI or backend-created isolated agentTurn cron job still requests announce delivery to `last` with bestEffort disabled, then no-channel target resolution reaches the reported resolver error and strict dispatch treats it as fatal.

## Solution Assessment

Yes. The narrow maintainable fix is to handle implicit no-channel cron delivery in the default/UI/runtime path while preserving strict behavior for explicit delivery targets; current main and the closed related PRs do not yet do that.
