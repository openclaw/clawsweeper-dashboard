## Summary

Keep open: current main and v2026.6.10 still default detached cron output jobs to implicit announce/last delivery and hard-fail unresolved non-best-effort delivery. The open linked PR at https://github.com/openclaw/openclaw/pull/91399 is the implementation candidate, but it has not merged, so this issue is still necessary.

## What This Changes

Cron jobs in local/no-channel setups should not lose or fail output after successful execution because cron delivery cannot resolve a channel.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/91399 or an equivalent narrow fix that downgrades only unresolved implicit no-channel cron delivery while preserving explicit-target fail-closed behavior.

## Reproduction Assessment

Yes. Source inspection shows detached cron output jobs default to requested announce/last delivery, no-channel resolution returns ok:false, and non-best-effort dispatch hard-fails that unresolved target; I did not run the live Control UI scenario in this read-only review.
