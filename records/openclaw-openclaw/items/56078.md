## Summary

Keep open: current main and v2026.6.8 still default detached cron output jobs to implicit announce/last delivery, and unresolved non-best-effort delivery still hard-fails the run. The open closing PR at https://github.com/openclaw/openclaw/pull/91399 is the implementation candidate, but it has not merged, so this issue remains necessary.

## What This Changes

Restore cron job output/save behavior for local no-channel setups where a cron job executes successfully but implicit delivery resolution fails with a channel-required error.

## Best Possible Solution

Land a focused fix that downgrades only unresolved implicit no-channel cron delivery while preserving explicit-target fail-closed behavior, then close this issue with merge proof.

## Reproduction Assessment

Yes. Source inspection shows detached cron output jobs default to requested announce/last delivery, no-channel resolution returns ok:false, and non-best-effort dispatch hard-fails that unresolved target; I did not run the live Control UI scenario in this read-only review.
