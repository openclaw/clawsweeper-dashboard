The PR moves Telegram inbound `requireMention` resolution so `topicConfig.requireMention` is checked before `activationOverride`, and adds focused topic precedence tests.

## Best Possible Solution

Land or replace this with a focused Telegram plugin fix that makes explicit topic `requireMention` outrank persisted activation state while preserving activation fallback when no topic override exists.
