## Summary

Keep this PR open. Current main still resolves Telegram inbound mention gating with persisted activation state before explicit topic-level `requireMention`, so the PR's intended fix is not implemented on main; the proposed reorder is narrow and matches the documented topic override model, but it still needs maintainer review for the activation-vs-topic product rule and likely a rebase onto the current surrounding code.

## What This Changes

The PR reorders Telegram inbound `requireMention` precedence so topic config is checked before persisted activation state, and adds a dedicated regression test file for topic, group, and activation precedence cases.

## Best Possible Solution

Land or replace this with a focused Telegram plugin change that makes explicit `topicConfig.requireMention` outrank persisted activation state while preserving activation fallback when no topic override exists, with regression coverage for both `false` and `true` topic overrides against activation state.

## Reproduction Assessment

Yes. A focused harness case can set a forum topic with `topicConfig.requireMention=false` and `resolveGroupActivation` returning `true`; current main will resolve `requireMention` from activation first and then drop an unmentioned group message through the existing `no-mention` path.

## Solution Assessment

Yes, with a validation gap. Moving `topicConfig?.requireMention` ahead of `activationOverride` is the narrowest code change for the reported precedence bug, but the final patch should add the inverse `topicConfig.requireMention=true` over `activationOverride=false` regression test and be rebased onto current main.
