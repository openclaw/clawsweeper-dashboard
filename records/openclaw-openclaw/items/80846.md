## Summary

Keep open. The report describes a concrete current-release Telegram/MiniMax regression, and current main still has a plausible provider/runtime gap: Telegram can stream reasoning previews when it receives callbacks, but MiniMax's documented Anthropic-compatible `reasoning_content` chunks are not proven to become incremental native thinking events.

## What This Changes

The issue requests restoration of progressive Telegram `/reasoning stream` preview updates for MiniMax-M2.7 after v2026.5.7 started showing reasoning as a completed block.

## Best Possible Solution

Trace MiniMax reasoning chunks through the provider transport into `onReasoningStream` and keep Telegram on the existing `sendMessage`/`editMessageText` preview lifecycle, with focused transport and dispatch tests plus live Telegram proof when feasible.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection shows a plausible current-main path for MiniMax Anthropic-compatible `reasoning_content` chunks not becoming incremental native `thinking_delta` callbacks, but no real Telegram/MiniMax provider-event log was verified.

## Solution Assessment

Unclear. Restoring literal Telegram draft bubbles is not the maintainable target; the likely fix is to normalize MiniMax reasoning deltas into the existing native reasoning callback path while preserving the current Telegram preview transport.
