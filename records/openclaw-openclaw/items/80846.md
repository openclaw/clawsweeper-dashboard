## Summary

Keep open. This is a concrete current-release Telegram/MiniMax regression report, current docs still promise a transient `/reasoning stream` preview, and the source shows a real Telegram reasoning lane but does not prove the reported full-block behavior is fixed or obsolete.

## What This Changes

The issue reports that Telegram `/reasoning stream` on MiniMax-M2.7 stopped updating progressively in v2026.5.7 and now shows reasoning as one completed block before deleting it.

## Best Possible Solution

Keep the current Telegram send/edit preview design and trace MiniMax reasoning deltas through the runtime into the Telegram reasoning lane, with focused mocked provider-event coverage and live Telegram proof when feasible.

## Reproduction Assessment

No. The issue gives specific steps and current source shows the intended callback chain, but I did not establish a failing current-main run or inspect logs proving that v2026.5.7 emits only one reasoning callback for MiniMax-M2.7.

## Solution Assessment

Unclear. Restoring literal draft-bubble character streaming is not the best direction because current maintainer guidance keeps Telegram on sendMessage/editMessageText; the maintainable fix is incremental preview edits when provider reasoning deltas are actually available.
