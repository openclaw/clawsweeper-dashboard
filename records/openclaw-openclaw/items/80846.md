## Summary

Keep open. This is a fresh, concrete Telegram/MiniMax regression report, and current main still has a documented Telegram reasoning-preview path plus a MiniMax safety wrapper that can prevent native incremental reasoning from reaching that path unless MiniMax thinking is explicitly configured.

## What This Changes

Restore Telegram `/reasoning stream` for MiniMax-M2.7 so reasoning preview text updates progressively during generation instead of appearing as a completed block.

## Best Possible Solution

Keep the current Telegram send/edit preview design, then trace MiniMax provider reasoning events into `onReasoningStream` safely or document the explicit MiniMax thinking configuration required when incremental deltas are unavailable.

## Reproduction Assessment

No high-confidence live reproduction is established. Source inspection shows a plausible current-main path because Telegram can stream reasoning callbacks, while MiniMax Anthropic transport disables thinking by default and MiniMax emits non-native `reasoning_content` chunks when enabled.

## Solution Assessment

Unclear. Restoring literal Telegram draft-bubble streaming is not the right fix under current maintainer guidance; the maintainable path is to preserve send/edit previews and safely bridge provider reasoning deltas when MiniMax exposes them.
