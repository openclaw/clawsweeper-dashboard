## Summary

Keep open: this is a fresh external regression report against the latest release, and current main still has a source-level path that can emit Telegram reasoning only once at `message_end` when no incremental thinking/tagged stream arrives. MiniMax is especially plausible because current code explicitly disables MiniMax Anthropic `reasoning_content` streaming to avoid leaking hidden reasoning, and no canonical merged fix or duplicate owns this exact Telegram regression.

## What This Changes

The issue reports that Telegram `/reasoning stream` on MiniMax-M2.7 changed from incremental reasoning draft updates to a final full-block flash in v2026.5.7.

## Best Possible Solution

Keep the issue open and fix the provider-to-channel stream boundary so explicit Telegram reasoning stream sessions receive incremental reasoning updates for MiniMax without exposing hidden reasoning in normal sessions.

## Reproduction Assessment

Yes, at source level: current main emits `rawThinking` once at `message_end` when no earlier thinking/tagged reasoning stream arrives, and MiniMax has a documented wrapper that suppresses its non-native `reasoning_content` stream. I did not perform a live Telegram/MiniMax run, so this is not a fully reproduced current-main runtime failure.

## Solution Assessment

Unclear until live MiniMax stream evidence is captured. The likely maintainable fix is to normalize MiniMax reasoning chunks into the existing explicit `onReasoningStream` path or suppress the final-only flash, while preserving current hidden-reasoning gates and Telegram preview semantics.
