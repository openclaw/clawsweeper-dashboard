## Summary

Keep open. This is still a concrete current-release Telegram/MiniMax regression report: current docs promise transient `/reasoning stream` preview updates, current source has the Telegram reasoning lane and runtime callback path, but current `main` does not prove MiniMax-M2.7 reasoning arrives incrementally instead of as one final accumulated block.

## What This Changes

The issue reports a v2026.5.7 Telegram regression where `/reasoning stream` with MiniMax-M2.7 shows reasoning as one completed block instead of progressive preview updates.

## Best Possible Solution

Trace MiniMax-M2.7 reasoning events through the provider wrapper and embedded subscriber into Telegram, then preserve the current send/edit preview design while ensuring multiple provider reasoning events produce progressive preview edits before cleanup.

## Reproduction Assessment

No. The issue gives a concrete version, model, channel, and commands, and current source shows the intended callback path, but I did not establish a live current-main Telegram/MiniMax run or logs proving only one full-block reasoning callback.

## Solution Assessment

Unclear. Literal native draft-bubble streaming is not the right target under current Telegram maintainer guidance; the maintainable repair is incremental preview edits when MiniMax reasoning deltas are actually available.
