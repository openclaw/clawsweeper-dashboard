## Summary

Keep open. Current main still has the Telegram `/reasoning stream` preview path, but MiniMax still has a source-documented `reasoning_content` stream mismatch and no merged PR, release note, or live Telegram/MiniMax proof shows the reported full-block behavior is fixed. The adjacent stale-preview issue remains related but not a duplicate.

## What This Changes

The issue reports that Telegram `/reasoning stream` stopped showing MiniMax-M2.7 reasoning progressively and now shows the reasoning as one full block before deleting it.

## Best Possible Solution

Normalize MiniMax reasoning output into safe internal reasoning deltas for the existing Telegram preview lane, or explicitly document MiniMax progressive reasoning as unsupported until the provider contract can be handled without leaking reasoning into final text.

## Reproduction Assessment

No high-confidence live reproduction is available. Source inspection does show a current MiniMax reasoning stream mismatch: Telegram can only preview what reaches `onReasoningStream`, while MiniMax's current Anthropic-compatible path is documented in code as using OpenAI-style `reasoning_content` deltas and is guarded by disabling thinking by default.

## Solution Assessment

Unclear. Restoring literal character-by-character Telegram drafts is not the right target; the maintainable fix is provider-event normalization into the existing coalesced send/edit preview lane, with live or deterministic MiniMax proof.
