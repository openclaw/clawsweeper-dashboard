## Summary

Keep open. Current main still documents `/reasoning stream` as an incremental Telegram preview path, but the MiniMax path still carries an intentional thinking-disable wrapper for OpenAI-style `reasoning_content` deltas and I found no merged/live proof that MiniMax-M2.7 now emits multiple reasoning updates into Telegram.

## What This Changes

The issue reports that Telegram `/reasoning stream` with MiniMax-M2.7 regressed from incremental reasoning previews to one full-block reasoning preview after upgrading to v2026.5.7.

## Best Possible Solution

Normalize MiniMax reasoning deltas into the existing native reasoning stream path, or explicitly document the MiniMax limitation, while keeping Telegram's maintained coalesced send/edit preview behavior.

## Reproduction Assessment

No high-confidence reproduction yet. The issue has concrete v2026.5.7 Telegram/MiniMax steps and current source shows a plausible MiniMax reasoning-delta mismatch, but I did not establish a current-main live MiniMax event stream or Telegram run showing the full-block behavior.

## Solution Assessment

Unclear. The best fix is not a literal character-by-character Telegram promise; it is provider-event normalization so MiniMax reasoning reaches the existing `onReasoningStream` lane as incremental updates, with focused fixtures and live Telegram proof.
