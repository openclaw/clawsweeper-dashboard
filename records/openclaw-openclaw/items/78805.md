## Best Possible Solution

Split this into a maintainer-approved async or no-blocking credential path plus separate profiled event-loop performance fixes, preserving current no-keychain-prompt runtime behavior and validating Gateway/Telegram responsiveness under load.

No high-confidence live reproduction was run against current main for the exact macOS/WSL2/Windows stalls. Source inspection does confirm the synchronous keychain and shell-command paths still exist, and the discussion includes real liveness-delay logs from multiple users.

## Solution Assessment

No single broad async-I/O rewrite is the best fix. The safer path is a narrow maintainer-approved credential/auth boundary change, while broader prep and plugin discovery stalls continue under their own profiled trackers.
