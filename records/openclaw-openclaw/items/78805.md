## Summary

Keep open: current main still contains synchronous macOS keychain reads on credential paths that can block the Node event loop, and the linked implementation PR https://github.com/openclaw/openclaw/pull/78818 is still open rather than merged.

## What This Changes

The issue requests moving runtime-critical gateway credential and file I/O away from synchronous execSync/readFileSync paths to avoid event-loop stalls.

## Best Possible Solution

Keep this issue open while maintainers profile the macOS gateway workload and land a focused non-blocking credential/session I/O fix through the linked PR or a clean replacement.

## Reproduction Assessment

No high-confidence end-to-end reproduction is established. Source inspection confirms blocking sync calls and the gateway diagnostic path matches the reported event_loop_delay warning, but no macOS Google/Telegram trace ties those calls to the exact stall.

## Solution Assessment

Unclear. Moving confirmed hot keychain/file reads off the event loop is a plausible fix, but the maintainable path is a targeted profiling-backed change rather than replacing every synchronous read mechanically.
