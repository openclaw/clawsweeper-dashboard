## Summary

Keep open: current main still does not implement the proposed research-script import-template preflight, but the PR head changes the exec security boundary and still has concrete scanner bypasses plus missing real behavior proof.

## What This Changes

The PR adds sandbox-only Python/Node research-script import validation before direct exec, updates shell preflight wiring, adds focused tests, and documents the new sandbox behavior.

## Best Possible Solution

Keep the hardening goal, rebase onto current main, and replace the regex-only scanner with parser-like or multiline-aware validation that fails closed for direct sandboxed scripts while preserving host exec behavior.

## Reproduction Assessment

Yes for the review findings: source inspection of the PR head gives deterministic bypass inputs for multiline Node imports, whitespace before require/import parentheses, and bare relative Python imports. I did not run tests because this is a read-only cleanup review.

## Solution Assessment

No. The security-hardening direction is useful, but the current line-oriented regex scanner is not robust enough for a sandbox preflight guarantee at the exec boundary.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Parse multiline Node imports before allowing execution:** `src/agents/sandbox-execution-templates.ts:442-457`
  - body: `validateNodeImports` scans each trimmed line separately, so `import {\n  execSync\n} from "node:child_process"` never presents both the import keyword and module specifier to the validator. That lets denied Node modules pass the new sandbox preflight.
  - confidence: 0.95
- **[P1] Accept whitespace before require/import parentheses:** `src/agents/sandbox-execution-templates.ts:473-484`
  - body: The literal and dynamic-call checks only recognize `require(` and `import(`. JavaScript permits `require ("node:child_process")` and `import ("node:child_process")`, which bypasses both the literal import check and dynamic-call guard.
  - confidence: 0.92
- **[P2] Check imported names for bare relative Python imports:** `src/agents/sandbox-execution-templates.ts:287-290`
  - body: For `from . import subprocess`, the source specifier is `.`; stripping leading dots produces an empty root and returns before denied names are checked. A local wrapper with a denied name can therefore avoid the template's stated text-level block.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: Needs attention because the PR changes sandboxed exec preflight and the added import scanner has concrete bypasses.

  - confidence: 0.95
