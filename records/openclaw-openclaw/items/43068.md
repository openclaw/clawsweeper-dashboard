## Summary

Keep open: the security-hardening goal is useful, but this PR is not merge-ready because current main does not already contain the import-template validator, the proposed scanner has concrete bypasses, and the PR lacks after-fix real behavior proof.

## What This Changes

Adds sandbox-only Python/Node research-script import-template validation to direct exec preflight, with focused tests and sandboxing docs.

## Best Possible Solution

Rebase onto current main, preserve the fs-safe preflight path, use parser-like or multiline-aware import validation, add regressions for the bypasses, and include redacted real exec proof.

## Reproduction Assessment

Yes for the review findings: PR-head source inspection gives deterministic bypass inputs for multiline Node imports, spaced require/import calls, and Python bare-relative imports. I did not run tests because this cleanup review is read-only.

## Solution Assessment

No. The hardening direction is useful, but the line-oriented regex scanner is not robust enough for a sandbox preflight guarantee; a parser-like or multiline-aware validator after the safe file read is the safer path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Parse multiline Node imports before sandbox approval:** `src/agents/sandbox-execution-templates.ts:442-457`
  - body: `validateNodeImports` scans each trimmed line separately, so `import {\n  execSync\n} from "node:child_process"` never presents both the import keyword and module specifier to the validator. That lets denied Node modules pass the new sandbox preflight.
  - confidence: 0.95
- **[P1] Handle whitespace before require/import parentheses:** `src/agents/sandbox-execution-templates.ts:473-484`
  - body: The literal and dynamic-call checks only recognize `require(` and `import(`. JavaScript permits `require ("node:child_process")` and `import ("node:child_process")`, which bypasses both the literal import check and dynamic-call guard.
  - confidence: 0.92
- **[P2] Check bare-relative Python import names:** `src/agents/sandbox-execution-templates.ts:287-290`
  - body: For `from . import subprocess`, the source specifier is `.`, which becomes an empty root and returns before denied names are checked. A local wrapper with a denied name can therefore avoid the template's stated text-level block.
  - confidence: 0.82
