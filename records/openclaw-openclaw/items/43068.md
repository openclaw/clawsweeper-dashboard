## Summary

Keep open: current main still lacks sandbox research-script import-template validation, and this PR has blocking parser bypasses in a security-sensitive exec path plus missing real behavior proof.

## What This Changes

Adds sandbox-only Python/Node research-script import-template validation, focused tests, and sandboxing documentation for direct exec script runs.

## Best Possible Solution

Rework or replace the PR with sandbox-only import validation that uses parser-backed or multiline-aware Node extraction, explicit Python bare-relative denied-name checks, focused regressions, docs, and secops review.

## Reproduction Assessment

Yes. Source inspection of the PR head shows valid Node multiline imports and whitespace-before-call forms can evade the line scanner, and `from . import subprocess` reaches the Python bare-relative early return.

## Solution Assessment

No. The sandbox-only direction is plausible, but the PR's regex line scanner is not robust enough for command-execution hardening and should be replaced with parser-backed or multiline-aware extraction while preserving current main's fail-closed preflight behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Parse Node imports across multiline declarations:** `src/agents/sandbox-execution-templates.ts:436-471`
  - body: The validator scans each trimmed line independently, so a standard multiline import or re-export with `from "node:child_process"` on a later line never reaches `assertNodeImportAllowed`. That lets denied runtime modules bypass the new sandbox import template.
  - confidence: 0.95
- **[P1] Recognize spaced require and import calls:** `src/agents/sandbox-execution-templates.ts:473-484`
  - body: JavaScript permits whitespace before the call parenthesis, but these guards only recognize `require(` and `import(`. Calls like `require ("node:child_process")` are neither validated as literal imports nor rejected as dynamic calls.
  - confidence: 0.93
- **[P2] Check bare relative Python imported names:** `src/agents/sandbox-execution-templates.ts:287-289`
  - body: For `from . import subprocess`, the source specifier becomes empty after stripping dots and returns before checking the imported name. The template should reject denied-looking imported names even when the source is a bare relative marker.
  - confidence: 0.82
