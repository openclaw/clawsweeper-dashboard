## Summary

Keep open: the security-hardening direction is useful, but the current patch is not mergeable because the added import scanner has concrete bypasses and the external PR lacks real after-fix behavior proof.

## What This Changes

The PR adds sandbox-only Python/Node research-script import templates, wires them into direct exec preflight, adds focused tests, and documents the sandbox behavior.

## Best Possible Solution

Rebase onto current main and integrate sandbox import-template validation into the current fs-safe, fail-closed script preflight path with multiline/parser-aware import validation and regressions for the known bypasses.

## Reproduction Assessment

Yes for the review findings: PR-head source inspection gives deterministic bypass inputs for multiline Node imports, spaced require/import calls, and Python bare-relative imports. I did not run tests because this cleanup review is read-only.

## Solution Assessment

No. The security-hardening direction is useful, but the current line-oriented regex implementation and stale integration point are not robust enough for a sandbox preflight guarantee.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Parse multiline Node imports before allowing execution:** `src/agents/sandbox-execution-templates.ts:442-457`
  - body: `validateNodeImports` scans each trimmed line separately, so a valid import like `import {\n  execSync\n} from "node:child_process"` never presents both the import keyword and module specifier to the validator. That lets denied Node modules pass the new sandbox preflight.
  - confidence: 0.95
- **[P1] Accept whitespace before require/import parentheses:** `src/agents/sandbox-execution-templates.ts:473-484`
  - body: The literal and dynamic-call checks only recognize `require(` and `import(`. JavaScript permits `require ("node:child_process")` and `import ("node:child_process")`, which bypasses both the literal import check and dynamic-call guard.
  - confidence: 0.92
- **[P2] Check imported names for bare relative Python imports:** `src/agents/sandbox-execution-templates.ts:287-290`
  - body: For `from . import subprocess`, the source specifier is `.`; stripping leading dots produces an empty root and returns before denied names are checked. A local wrapper with a denied name can therefore avoid the template's stated text-level block.
  - confidence: 0.82
