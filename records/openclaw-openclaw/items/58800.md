## Summary

Keep open: current main still omits identity on several Slack edit paths, and this PR remains useful but is not mergeable because the head miswires preview finalization, lacks send-path fallback parity, and has no live Slack proof.

## What This Changes

The PR attempts to pass Slack agent identity through editSlackMessage, draft stream edits, preview finalization, status-final updates, and related regression tests.

## Best Possible Solution

Land or replace this with a Slack-plugin-only fix that preserves visible agent identity across edit helper, draft edit, preview finalization, and status-final updates, with missing-scope fallback parity and live/API proof.

## Reproduction Assessment

Yes, at source level. Configure a Slack route with a custom agent identity and trigger a draft/preview/status edit path; current main resolves identity for sends but omits it from edit helper and key edit callers.

## Solution Assessment

No for the current PR revision. Slack-plugin identity threading is the right boundary, but this head passes a timestamp as identity in one path, lacks send-path missing-scope fallback parity, and needs proof that Slack preserves authorship on edit.
