## Summary

Keep open. The Slack regression is still real on current main, but this PR remains incorrect because one edit path passes a thread timestamp as identity, edit calls lack the existing missing-scope fallback parity, the proposed `chat.update` authorship fields need Slack API proof or a supported alternative, and the user-facing fix lacks a changelog entry.

## What This Changes

The PR threads Slack custom sender identity through `editSlackMessage`, draft-stream, preview-finalize, and status-final update paths, with focused Slack regression tests.

## Best Possible Solution

Land a Slack-plugin-only fix that preserves visible agent identity on draft, preview-finalize, and status-final edit flows using Slack-supported behavior, keeps send/edit missing-scope fallback parity, and includes regression tests plus the required changelog entry.

## Reproduction Assessment

Yes. The linked bug gives concrete Slack steps, and current main source shows the edit helper and draft/finalize edit paths drop the resolved Slack identity while send paths already support it.

## Solution Assessment

No. The PR is directionally useful, but this revision wires one identity argument incorrectly, lacks the existing best-effort missing-scope retry on edits, and depends on `chat.update` authorship fields that are not present in the current Slack SDK type contract.
