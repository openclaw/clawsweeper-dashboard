## Summary

Keep open: this is a useful Slack bug-fix direction tied to an open regression report, but the current PR head still wires one edit path incorrectly, lacks missing-scope fallback parity, and has no live Slack proof that edited messages retain custom identity.

## What This Changes

The PR threads Slack custom identity through message edit, draft stream, preview-finalize, and status-final update paths and adds focused regression tests.

## Best Possible Solution

Land or replace this with a Slack-plugin-only fix that preserves visible agent identity across edit helper, draft edit, preview finalization, and status-final updates, with missing-scope fallback parity and real Slack proof.

## Reproduction Assessment

Yes, at source level. Configure a Slack route with a custom agent identity and trigger a draft/preview/status edit path; current main resolves identity for sends but omits it from the edit helper and key edit callers.

## Solution Assessment

No. Threading identity inside the Slack plugin is the right boundary, but this revision passes a timestamp as identity in one path, lacks send-path fallback parity, and needs Slack-visible proof for the `chat.update` authorship strategy.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Pass the resolved Slack identity to preview finalization:** `extensions/slack/src/monitor/message-handler/dispatch.ts:582`
  - body: This call sets `identity` to `finalThreadTs`, so `editSlackMessage` receives a timestamp string instead of the resolved Slack identity object. The normal preview-finalize edit path therefore still lacks `username` and `icon_*`, leaving the visible identity regression in place.
  - confidence: 0.94
- **[P2] Retry edits without custom identity on missing scope:** `extensions/slack/src/actions.ts:223-239`
  - body: When custom identity is present, the new `chat.update` calls can fail for workspaces missing `chat:write.customize`, while current sends catch that condition and retry without identity. Without the same fallback, previously successful edits can become failed edits instead of degrading to the bot default identity.
  - confidence: 0.89
- **[P2] Prove or replace the chat.update authorship strategy:** `extensions/slack/src/actions.ts:223-239`
  - body: The Slack SDK contract exposes `username` and `icon_*` for `chat.postMessage`, but not for `chat.update`, and the linked bug says update calls drop those fields. Payload-only unit tests do not prove the edited Slack message keeps the agent identity; add credible live/API proof for this method or switch to a supported strategy.
  - confidence: 0.82
