## Summary

Keep open. Current main still lacks Slack edit identity propagation, and this PR targets the right regression, but the latest diff is not merge-ready because one preview-finalization path passes a thread timestamp as the identity object, edit calls would lose the existing missing-scope fallback behavior, and Slack edit authorship support needs proof before payload-only tests can be trusted.

## What This Changes

The PR threads Slack custom sender identity through message edits, draft streaming, preview finalization, and status-final completion updates, with focused regression tests.

## Best Possible Solution

Land a Slack-plugin-only fix that preserves the resolved agent identity on every OpenClaw-authored Slack edit path, keeps edit reliability aligned with send reliability when `chat:write.customize` is missing, and proves the chosen Slack API behavior with tests plus docs or live/API evidence.

## Reproduction Assessment

Yes. The linked issue provides a concrete live path, and current-main source gives a high-confidence source-level reproduction: custom identity is resolved for sends but omitted from `editSlackMessage`, draft preview edits, preview finalization, and status-final updates.

## Solution Assessment

No. The proposed direction is narrow and in the right Slack-owned area, but this revision is not the best fix until it passes the resolved `slackIdentity`, adds edit-side fallback parity, and proves or avoids unsupported `chat.update` authorship fields.
