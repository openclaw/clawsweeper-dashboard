## What This Changes

The PR threads Slack agent identity through editSlackMessage, draft-stream edits, preview finalization, status-final updates, and related Slack regression tests.

## Best Possible Solution

Land or replace this with a Slack-plugin-only fix that preserves agent identity across edit helper, draft edit, preview finalization, and status-final updates, with missing-scope fallback parity and live/API proof.

## Reproduction Assessment

Yes, at source level. Current main resolves Slack identity for sends but omits it from editSlackMessage, draft edit, and preview finalization paths; I did not run a live Slack workspace repro in this read-only review.

## Solution Assessment

No for the current PR revision. Slack-plugin identity threading is the right boundary, but this head passes a timestamp as identity, lacks send-path missing-scope fallback parity, and needs proof that Slack preserves authorship on edit.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Pass the resolved Slack identity to preview finalization:** `extensions/slack/src/monitor/message-handler/dispatch.ts:582`
  - body: This call sets identity to finalThreadTs, a Slack timestamp string, instead of the resolved slackIdentity object. Normal preview finalization therefore still reaches editSlackMessage without username/icon fields, leaving the visible identity regression on that path.
  - confidence: 0.96
- **[P2] Retry edits without custom identity on missing scope:** `extensions/slack/src/actions.ts:217-239`
  - body: When identity is present, these chat.update calls can fail for workspaces missing chat:write.customize, while current sends retry without identity. That turns previously successful edits into hard failures instead of degrading to the bot default identity.
  - confidence: 0.9
- **[P2] Prove or replace the chat.update authorship strategy:** `extensions/slack/src/actions.ts:217-239`
  - body: Slack's documented and typed chat.update surface does not expose username/icon authorship fields like chat.postMessage does. Payload-only unit tests do not prove the edited message keeps identity, so this needs credible live/API proof or a supported Slack strategy.
  - confidence: 0.84
