## Summary

Keep open: the Slack identity regression is still present on current main and in the latest release, and this PR is a plausible fix path but still has correctness and real-behavior-proof blockers.

## What This Changes

The PR threads custom Slack agent identity through message edit, draft stream, preview-finalize, and status-final update paths, with focused regression tests.

## Best Possible Solution

Land or replace this with a Slack-plugin-only fix that preserves visible agent identity across edit helper, draft edit, preview finalization, and status-final updates, with missing-scope fallback parity and live Slack proof.

## Reproduction Assessment

Yes, at source level. Configure a Slack route with custom agent identity and trigger a draft, preview-finalize, or status edit path; current main resolves identity for sends but drops it in edit helpers and key edit callers.

## Solution Assessment

No for this PR revision. Slack-plugin identity threading is the right boundary, but the current head passes a timestamp as identity in one path, lacks send-path missing-scope fallback parity, and needs proof that Slack preserves authorship on edit.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Pass the resolved Slack identity to preview finalization:** `extensions/slack/src/monitor/message-handler/dispatch.ts:582`
  - body: This call sets identity to finalThreadTs, a Slack timestamp, instead of the resolved slackIdentity object. Normal preview finalization therefore still calls editSlackMessage without username/icon fields, leaving the visible identity regression on that edit path.
  - confidence: 0.94
- **[P2] Retry edits without custom identity on missing scope:** `extensions/slack/src/actions.ts:223-239`
  - body: When identity is present, these chat.update calls can fail for workspaces missing chat:write.customize, while current sends retry without identity. That turns previously successful edits into hard failures instead of degrading to the bot default identity.
  - confidence: 0.89
- **[P2] Prove or replace the chat.update authorship strategy:** `extensions/slack/src/actions.ts:223-239`
  - body: The locked Slack SDK request types expose username/icon authorship for chat.postMessage, not ChatUpdateArguments. Payload-only unit tests do not prove Slack will keep the edited message identity, so this needs credible live/API proof or a supported strategy.
  - confidence: 0.82
