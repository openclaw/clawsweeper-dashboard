## Summary

This should stay open: the linked Slack regression is still valid on current main, but this PR is not merge-ready because it wires the wrong value into preview finalization, lacks the existing missing-scope fallback parity for identity edits, and has no real Slack proof.

## What This Changes

The PR threads Slack agent identity into message edit/update paths and adds unit tests for edit, draft stream, and preview-finalize identity propagation.

## Best Possible Solution

Refresh the Slack fix on current main with one shared identity payload and fallback path for post/update, pass slackIdentity through every preview, draft, and status edit path, and prove the result in a real Slack workspace.

Yes, source-level reproduction is high confidence: current main resolves agent Slack identity for sends, but editSlackMessage and preview finalization do not carry identity into chat.update. I did not run a live Slack workspace repro in this read-only review.

## Solution Assessment

No. The fix direction is right, but this branch is not the best current solution until it passes slackIdentity, preserves the existing missing-scope fallback behavior, refreshes over current main, and includes real Slack proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Pass the resolved Slack identity to final edits:** `extensions/slack/src/monitor/message-handler/dispatch.ts:582`
  - body: This wires `finalThreadTs` into `identity`, but `finalThreadTs` is a thread timestamp string while `finalizeSlackPreviewEdit` forwards `identity` to `editSlackMessage` as the username/avatar object. In the normal preview-finalization path this either fails type checking or still omits `username`/`icon_*`, so the central edit flow remains unfixed; pass `slackIdentity` here instead.
  - confidence: 0.96
- **[P1] Retry edited messages without custom identity on missing scope:** `extensions/slack/src/actions.ts:223`
  - body: These new identity-bearing `chat.update` calls do not mirror the existing `postSlackMessageBestEffort` behavior that retries without identity when Slack reports missing `chat:write.customize`. That means a workspace that currently edits successfully with the bot default can start failing draft/status edits once an agent identity is configured; wrap the identity update path with the same fallback before returning.
  - confidence: 0.9
