This PR should stay open for normal maintainer review and landing: current main still has the Google Chat thread available in inbound context but message-tool reply inheritance falls back to the inbound message resource, and the proposed plugin-local threading context is the narrow fix with sufficient redacted real-behavior proof.

The PR adds a Google Chat `buildToolContext` so message-tool replies use the inbound thread resource, preserves per-account `replyToMode`, and adds a regression test for `[[reply_to_current]]`.

## Best Possible Solution

## Solution Assessment
