## What This Changes

## Best Possible Solution

Define the recovery semantics first: keep the signature poison root under https://github.com/openclaw/openclaw/issues/92201, and handle this issue with a focused session-recovery policy that prevents stale recovered work from silently re-owning a shared task while emitting an owner-visible handoff signal.
