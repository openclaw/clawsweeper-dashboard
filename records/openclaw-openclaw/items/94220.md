## Summary

Keep open: current main and the latest release still have Anthropic thinking recovery that retries and repairs the active session transcript, but I found no current behavior that fences stale detached task intent or coordinates sibling session ownership after a heal. The linked Anthropic issue remains the poison trigger, while this issue is the distinct post-heal task-lifecycle hazard.

## What This Changes

The issue reports that session medic recovery can resume stale detached task intent after Anthropic thinking-signature healing, causing multi-session coordination races and livelock.

## Best Possible Solution

Define a bounded recovery/task lifecycle rule that preserves Anthropic self-heal while requiring a fresh trigger, explicit owner signal, or task handoff before recovered sessions continue stale detached work; keep the signature creation investigation on the canonical Anthropic issue.

## Reproduction Assessment

No for a full live reproduction: current source shows recovery retries and repairs the same active session without a stale-work guard, but the reported multi-session artifact race still needs redacted incident data or a focused harness. The production report and current code make the missing guard source-reproducible enough to keep open.

## Solution Assessment

Unclear: suppressing stale detached-task continuation after a heal is plausible, but maintainers should choose whether the boundary is abort, fence, checkpoint, owner notification, or a narrower task handoff. The single-writer lock proposal is likely separate from the minimum recovery safety rule.
