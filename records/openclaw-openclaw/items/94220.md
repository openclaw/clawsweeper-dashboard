## What This Changes

The issue asks OpenClaw to prevent a medic-healed session from continuing stale detached work after Anthropic thinking-signature recovery, and to reduce the resulting multi-session race/livelock risk.

## Best Possible Solution

Define a bounded recovery/task lifecycle rule that preserves Anthropic self-heal while requiring a fresh trigger, explicit owner signal, or task handoff before recovered sessions continue stale detached work; keep signature creation work on the canonical Anthropic issue.

## Solution Assessment

Unclear: suppressing stale detached-task continuation after a heal is plausible, but maintainers should choose whether the boundary is abort, fence, checkpoint, owner notification, or a narrower task handoff.
