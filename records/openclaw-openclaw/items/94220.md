## Summary

Keep open: current main and the latest release still self-heal Anthropic thinking-signature failures by retrying and repairing the active transcript, but I found no shipped or current-main guard that stops recovered sessions from continuing stale detached work or fences sibling sessions from racing shared artifacts.

## What This Changes

The issue asks OpenClaw to change session-medic recovery so a healed session does not automatically resume stale detached work, with possible owner notification, single-writer guardrails, and reduced high-thinking Anthropic signature poison rates.

## Best Possible Solution

Define a bounded recovery/task lifecycle rule that keeps Anthropic self-heal, but requires a fresh trigger, explicit owner signal, or task handoff before recovered sessions continue work that may be stale.

## Reproduction Assessment

No, not as a high-confidence live reproduction. Source inspection shows recovery retries and repairs the same active session without a stale detached-task guard, but the reported multi-session race still needs redacted incident data or a focused harness.

## Solution Assessment

Unclear. Preventing stale detached work after a recovery event is plausible, but maintainers should choose the lifecycle boundary because always aborting healed turns could weaken the shipped self-heal behavior.
