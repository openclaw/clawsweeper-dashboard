## Summary

Keep open: current main has the Anthropic thinking-replay recovery mechanism, but I did not find a current-main guard that prevents a successfully healed session from continuing queued follow-up or detached work with stale intent. The linked recovery and create-side signature issues are real related context, but they do not own the distinct post-heal stale-task race reported here.

## What This Changes

The issue requests that session healing after invalid Anthropic thinking signatures restore session state without automatically re-driving stale detached or queued task intent, with related coordination and signature-rate mitigations.

## Best Possible Solution

Add a focused core recovery boundary that marks a thinking-replay heal as invalidating stale queued or detached task intent unless a fresh owner trigger revalidates it, while leaving the create-side signature work to https://github.com/openclaw/openclaw/issues/92201 and broad single-writer migration work to https://github.com/openclaw/openclaw/issues/88838.

## Reproduction Assessment

No high-confidence end-to-end reproduction was established in this read-only pass. Source inspection does show that recovery repairs thinking replay and normal follow-up drain can continue queued work, but the exact stale detached-task race still needs redacted trajectory samples or a targeted multi-session repro.

## Solution Assessment

No, the full proposal is broader than the best first fix. The maintainable path is a narrow post-recovery queue/task invalidation boundary, with signature creation and single-writer storage tracked by their existing canonical issues.
