## Summary

Keep open: current main and the latest local release tag still have the Anthropic thinking self-heal and transcript repair path, but I found no current behavior that prevents recovered sessions from continuing stale detached work or racing sibling sessions. The linked Anthropic signature issue tracks the poison trigger, while this issue remains the distinct recovery/task-lifecycle hazard.

## What This Changes

The issue asks for session-medic recovery to avoid resuming stale detached work after a thinking-signature heal and to reduce or fence the resulting multi-session race risk.

## Best Possible Solution

Define a bounded recovery/task lifecycle rule that preserves Anthropic self-heal while requiring a fresh trigger, explicit owner signal, or task handoff before recovered sessions continue stale detached work; keep the create-side signature investigation on the canonical Anthropic issue.

## Reproduction Assessment

No for a full live reproduction: current source shows the recovery path retries and repairs the same active session without a detached-task stale-work guard, but the multi-session race still needs redacted incident data or a focused harness. The reported production counts make the scenario plausible enough to keep open.

## Solution Assessment

Unclear: suppressing stale detached-task continuation after a heal is plausible, but maintainers should choose whether the boundary is abort, fence, checkpoint, owner notification, or a narrower task handoff. The single-writer lock proposal is likely separate from the minimum recovery safety rule.
