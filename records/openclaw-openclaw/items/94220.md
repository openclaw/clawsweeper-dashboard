## Summary

Keep open: current main and v2026.6.8 contain the Anthropic thinking-signature recovery path, but I found no current-main behavior that prevents a recovered session from continuing stale detached work or fences sibling sessions from racing shared artifacts.

## What This Changes

The issue asks OpenClaw to stop session recovery from auto-continuing stale detached work after a thinking-signature heal, and to consider stronger coordination signals or writer fencing.

## Best Possible Solution

Define a bounded recovery/task lifecycle rule that preserves thinking-signature self-heal while requiring a fresh trigger, explicit owner signal, or task handoff before recovered sessions continue stale detached work.

## Reproduction Assessment

No, not as a full live reproduction. Source inspection shows the recovery path retries and repairs the same active session without a detached-task stale-work guard, but the multi-session race still needs redacted incident data or a focused harness.

## Solution Assessment

Unclear. Suppressing stale detached-task continuation after a recovery event is plausible, but maintainers should choose the exact boundary because aborting every heal could weaken the shipped recovery behavior.
