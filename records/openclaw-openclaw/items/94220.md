## Summary

Keep open: #92201 owns the Anthropic signature-poisoning trigger, but current main still does not define or enforce a post-heal contract that prevents a repaired session from continuing stale queued or detached intent.

## What This Changes

The issue asks OpenClaw to prevent session-medic thinking-signature recovery from resuming stale detached work that can race sibling sessions, while separately reducing the Anthropic high-thinking poison rate.

## Best Possible Solution

Define an owner-approved post-heal session contract: repair poisoned thinking state, then require a fresh trigger or explicit ownership handoff before old detached or queued intent can act on shared artifacts; keep the signature creation root cause under #92201.

## Reproduction Assessment

No full live reproduction was run. The reporter supplied production evidence, and current source shows thinking recovery repairs transcript state while queue and diagnostic recovery paths preserve or resume queued work without a stale-intent guard.

## Solution Assessment

Unclear as an implementation path. Preventing auto-resume is plausible, but the best fix needs a maintainer decision on recovery semantics and shared-artifact ownership rather than only adding the optional config or only fixing #92201.
