## Summary

Keep open: current main and v2026.6.9 have Anthropic thinking replay recovery and transcript repair, but I found no current behavior that prevents a healed session from continuing stale detached-task intent or coordinating sibling ownership after the heal.

## What This Changes

The issue reports that session-medic healing can resume stale detached task intent, causing sibling-session races and livelock after Anthropic thinking-signature recovery.

## Best Possible Solution

Keep this issue open for a maintainer decision on the stale-work boundary after automatic transcript heal, with a fix that either fences/cancels stale detached task intent, adds an explicit owner handoff rule, or documents why healed sessions should continue prior work.

## Reproduction Assessment

Unclear: the production report is detailed and current source shows the recovery hook repairs transcript state without a stale-task fence, but I did not run a live multi-agent reproduction.

## Solution Assessment

Unclear: no-auto-resume may be the right end state, but deciding whether healed sessions should cancel, pause, notify, or require an explicit coordinator handoff is a product/lifecycle decision.
