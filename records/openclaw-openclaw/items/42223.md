## What This Changes

The PR removes the active-tab guard from sidebar group collapse class assignment in `ui/src/ui/app-render.ts` so collapsed groups hide children even when the active tab belongs to the group.

## Best Possible Solution

Port the same behavior to current `main` by making collapsed nav sections hide their children even when they contain the active tab, while preserving compact-sidebar behavior and adding regression coverage plus a changelog entry.

## Reproduction Assessment

Yes, source-reproducible: set the active tab to a tab in a collapsed group, such as `channels` with `navGroupsCollapsed.control = true`; `hasActiveTab` keeps `showItems` true, so `.nav-section--collapsed` is not applied and CSS does not hide the children.

## Solution Assessment

No for this PR as-is. The one-line intent is good, but the current branch changes an obsolete `nav-group--collapsed` path; the maintainable fix is the same logic ported to the current `showItems` calculation with focused coverage.
