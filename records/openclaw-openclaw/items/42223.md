## Summary

Keep open: current main still has the active-tab exception in the live sidebar renderer, while this PR changes obsolete `.nav-group` markup and would not fix the current `.nav-section` path.

## What This Changes

The PR removes the active-tab guard from the old `nav-group--collapsed` class logic in `ui/src/ui/app-render.ts` so collapsed groups hide children even when the active tab is inside the group.

## Best Possible Solution

Port the behavior to the current `.nav-section` renderer so expanded-sidebar group visibility follows `isGroupCollapsed` while preserving collapsed-rail behavior, then add focused browser coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows that `/channels` in the `control` group with `settings.navGroupsCollapsed.control = true` still sets `showItems` true, preventing `.nav-section--collapsed` from hiding the children.

## Solution Assessment

No. The intended behavior is correct, but this PR changes the old `.nav-group` path; the maintainable fix is to update the current `.nav-section` `showItems` logic and cover the active-tab collapsed-group case.
