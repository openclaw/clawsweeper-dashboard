## What This Changes

The PR removes the active-tab guard from the sidebar group collapsed class in `ui/src/ui/app-render.ts` so collapsed groups hide child items even when they contain the active tab.

## Best Possible Solution

Port the active-group collapse behavior to current main's `.nav-section` `showItems` logic, preserve the collapsed rail behavior, and add a focused browser regression test.

## Reproduction Assessment

Yes, at source level: set an active tab inside a group with `navGroupsCollapsed[group.label]` true and `navCollapsed` false, and current main keeps `showItems` true because `hasActiveTab` is true. That prevents `.nav-section--collapsed` from applying, so child items remain visible.

## Solution Assessment

No, not as currently based. The intended behavior change is right, but the branch edits stale `.nav-group` markup; the maintainable fix belongs in current main's `.nav-section` `showItems` path with focused browser coverage.
