## Summary

Keep open: the sidebar collapse bug is still source-reproducible on current main, but this PR edits stale `.nav-group` markup and does not patch the live `.nav-section` renderer. It also still needs inspectable real behavior proof before merge.

## What This Changes

The PR removes the active-tab condition from the sidebar group collapsed class in `ui/src/ui/app-render.ts`.

## Best Possible Solution

Port the predicate in current main's `.nav-section` renderer so user-collapsed groups hide children while the collapsed rail still shows icon-only items, then cover it in `navigation.browser.test.ts`.

## Reproduction Assessment

Yes, at source level: set an active tab inside a group with `navGroupsCollapsed[group.label]` true and `navCollapsed` false, and current main keeps `showItems` true because `hasActiveTab` is true. That prevents `.nav-section--collapsed` from applying, so the child items remain visible.

## Solution Assessment

No, not as currently based. The intended behavior change is right, but the submitted hunk targets stale `.nav-group` markup instead of current main's `.nav-section` `showItems` path and lacks focused browser coverage.
