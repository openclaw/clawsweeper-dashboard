## Summary

Keep open: the reported sidebar collapse bug still exists on current main, but this PR targets stale `.nav-group` markup while the live renderer now uses `.nav-section` and `showItems`; it also still lacks inspectable real behavior proof.

## What This Changes

The PR removes the active-tab guard from sidebar group collapse class application in `ui/src/ui/app-render.ts` so collapsed groups hide child items even when the active tab is inside the group.

## Best Possible Solution

Port the collapse predicate to the current `.nav-section` `showItems` path so user-collapsed groups hide children even when active, preserve collapsed-rail behavior, and add focused browser coverage plus redacted Control UI proof.

## Reproduction Assessment

Yes, at source level: with `navCollapsed` false, an active tab inside the group, and `navGroupsCollapsed[group.label]` true, current main still computes `showItems` as true. That prevents `.nav-section--collapsed` from applying, so child items remain visible.

## Solution Assessment

No, not as submitted. The intended behavior change is right, but the branch edits stale `.nav-group` markup; the maintainable fix belongs in current main's `.nav-section` `showItems` path with browser regression coverage.
