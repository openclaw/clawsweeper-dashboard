## Summary

## What This Changes

The PR removes the active-tab guard from sidebar group collapse class assignment so child items hide even when the active tab belongs to the collapsed group.

## Best Possible Solution

Port the collapse behavior to the current `nav-section` renderer by making collapsed groups hide their children even when active, then add focused regression coverage and a single-line changelog entry.

## Reproduction Assessment

Yes, source-reproducible: set the active tab to `channels` and `navGroupsCollapsed.control` to `true`; `hasActiveTab` keeps `showItems` true, so the collapsed CSS class is not applied and the children remain visible.

## Solution Assessment

No for this PR as-is. The one-line intent is right, but the branch changes an obsolete `nav-group--collapsed` path; the maintainable fix is to update the current `showItems` logic and cover the active-tab case.
