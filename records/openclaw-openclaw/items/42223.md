## Summary

Keep open. The linked bug still has a clear current-main source repro, and the intended behavior change is useful, but this PR is stale/conflicting and edits the old `.nav-group` renderer instead of current main's `.nav-section`/`showItems` path; external real behavior proof is also missing.

## What This Changes

The PR removes the active-tab guard from the old sidebar group collapsed class in `ui/src/ui/app-render.ts` so a collapsed group would hide child items even when it contains the active tab.

## Best Possible Solution

Port the collapse rule to current main's `.nav-section` visibility logic so expanded-sidebar groups hide when `isGroupCollapsed` even if active, while preserving collapsed-rail behavior and adding focused browser coverage.

## Reproduction Assessment

Yes, at source level: use an expanded sidebar with `state.tab` in the `control` group and `settings.navGroupsCollapsed.control = true`. Current main keeps `showItems` true because `hasActiveTab` is true, so `.nav-section--collapsed` is absent and child items remain visible.

## Solution Assessment

No. The intended behavior change is correct, but this branch edits stale `.nav-group` markup; the maintainable fix is to apply the same rule to current main's `.nav-section` `showItems` path and cover the active-tab collapsed-group case.
