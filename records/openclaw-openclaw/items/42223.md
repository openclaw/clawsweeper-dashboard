## Summary

Keep open: current main still has the source-level sidebar collapse bug, and the linked issue was closed in favor of this PR, but the branch edits stale sidebar markup and is currently conflicting, so it needs a current-main port plus contributor proof before merge.

## What This Changes

The PR removes the active-tab guard from the old Control UI sidebar group collapse class so child items hide when their group is collapsed.

## Best Possible Solution

Port the collapse rule to current main's .nav-section showItems logic so expanded-sidebar groups hide when collapsed even if active, while preserving collapsed-rail behavior and adding focused browser coverage.

## Reproduction Assessment

Yes, at source level: use an expanded sidebar with state.tab in the control group and settings.navGroupsCollapsed.control = true. Current main keeps showItems true because hasActiveTab is true, so .nav-section--collapsed is absent and child items remain visible.

## Solution Assessment

No, not as currently based. The intended behavior change is correct, but the branch edits stale .nav-group markup; the maintainable fix is to apply the same rule to current main's .nav-section showItems path and cover the active-tab collapsed-group case.
