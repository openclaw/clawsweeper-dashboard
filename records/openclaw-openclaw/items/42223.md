## Summary

Keep open: the underlying sidebar bug is still present on current main, and this remains the canonical open PR thread for the linked user report, but the branch no longer updates the live sidebar renderer.

## What This Changes

The branch changes `ui/src/ui/app-render.ts` so a collapsed sidebar group always receives the old `nav-group--collapsed` class instead of exempting the group that contains the active tab.

## Best Possible Solution

Update the current `.nav-section` sidebar renderer so group collapse hides children even when the active tab is inside that group, while preserving collapsed-rail behavior and adding focused browser coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `state.tab = "channels"` and `settings.navGroupsCollapsed.control = true` leave `showItems` true, so `.nav-section--collapsed` is not applied and the child items remain visible.

## Solution Assessment

No. The intended behavior is correct, but the PR changes obsolete `.nav-group` markup; the maintainable fix is to port the condition to the current `.nav-section` `showItems` logic and cover it in `ui/src/ui/navigation.browser.test.ts`.
