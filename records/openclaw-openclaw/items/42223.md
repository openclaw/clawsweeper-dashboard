## Summary

Keep open. The sidebar collapse bug is still source-reproducible on current main, but this PR targets the old `.nav-group` renderer while current main uses `.nav-section` and `showItems`, so it needs a current-main port/rebase plus inspectable real behavior proof before merge.

## What This Changes

The PR removes the active-tab guard so collapsing a sidebar group should hide child items even when the active tab is inside that group.

## Best Possible Solution

Port the collapse rule to current main's `.nav-section` renderer so expanded-sidebar group visibility follows `isGroupCollapsed` even for active tabs while preserving collapsed-rail behavior, then add focused browser coverage.

## Reproduction Assessment

Yes, at source level: use an expanded sidebar with `state.tab` in the `control` group and `settings.navGroupsCollapsed.control = true`. Current main keeps `showItems` true because `hasActiveTab` is true, so `.nav-section--collapsed` is absent and the children remain visible.

## Solution Assessment

No. The intended behavior change is correct, but the branch edits stale `.nav-group` markup; the maintainable fix is to apply the same rule to current main's `.nav-section` `showItems` logic and cover the active-tab collapsed-group case.
