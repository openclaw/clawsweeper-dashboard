## Summary

Keep open: the sidebar bug is still source-reproducible on current main, but this branch edits the pre-refactor `.nav-group` renderer and needs a current `.nav-section` port plus real behavior proof before merge.

## What This Changes

The PR removes the active-tab guard from the old `.nav-group--collapsed` class assignment in `ui/src/ui/app-render.ts` so collapsed groups hide child items even when the active tab is inside the group.

## Best Possible Solution

Port the active-tab collapse fix to the current `.nav-section` renderer while preserving collapsed rail behavior, then add focused browser coverage for an active tab inside a collapsed group.

## Reproduction Assessment

Yes, source-reproducible: set `state.tab` to `channels` and `settings.navGroupsCollapsed.control` to `true`; the current `showItems` expression remains true and prevents `.nav-section--collapsed` from hiding the children.

## Solution Assessment

No. The intended behavior is right, but the branch changes an obsolete `.nav-group` path; the maintainable fix is to update the current `.nav-section` visibility logic and cover it in `ui/src/ui/navigation.browser.test.ts`.
