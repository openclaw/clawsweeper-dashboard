## Summary

The reported Control UI bug still exists on current main, but the submitted patch targets older `nav-group` markup that no longer exists, and GitHub currently reports the branch as not mergeable; the fix needs to be adapted to the current `showItems` / `nav-section--collapsed` render path.

## What This Changes

The PR removes the active-tab guard from the sidebar group collapsed class in `ui/src/ui/app-render.ts` so collapsing a group hides its child items even when the active tab is inside that group.

## Best Possible Solution

Adapt the PR to current main by removing the active-tab exception from the `showItems` decision, while keeping `navCollapsed` as the icon-only rail override, then add browser coverage for collapsing a group that contains the active tab and a user-facing changelog entry.

## Reproduction Assessment

Yes. Set the active tab to a member of a collapsed group, for example `channels` with `navGroupsCollapsed.control = true` and `navCollapsed = false`; current main still makes `showItems` true through `hasActiveTab`, so `.nav-section--collapsed` is not applied and CSS leaves the children visible.

## Solution Assessment

No, not as submitted. The functional direction is right, but the patch must be rebased onto the current `nav-section` render path and should include a regression test and changelog entry.
