## Summary

Keep open. Current main and v2026.6.8 still have observed/pending channel history windows plus plugin prompt hooks, but not a native opt-in Discord/shared-channel scrollback preload at session start or after inactivity; this remains a product/API/security design item rather than conservative cleanup.

## What This Changes

The issue requests an opt-in native channel-history preload that fetches recent Discord or other channel scrollback at session start and after inactivity, then injects it into session context.

## Best Possible Solution

Make a maintainer product/API call first: keep this as plugin-owned channel context work, or define a narrow generic channel history-preload contract with budgets, throttling, untrusted-context marking, docs, and focused channel tests.

## Reproduction Assessment

Not applicable: this is a feature request, not broken existing behavior. Source, docs, release-tag, and test inspection show current OpenClaw records observed/pending channel history and has plugin prompt hooks, but no native platform scrollback preload for Discord/shared channel sessions.

## Solution Assessment

Unclear as requested: a core-wide preloadHistory knob is not obviously the best boundary. The maintainable path is likely either a provider-owned generic SDK/channel contract, possibly informed by Slack's narrow thread-history fetch, or an explicit decision to keep this as plugin work.

Status: needs_attention

Summary: No patch is under review, but the requested feature needs security review because it would fetch shared-channel history with channel credentials and inject it into model context.

Concerns:

- **[medium] Define scrollback trust and redaction boundaries:**
