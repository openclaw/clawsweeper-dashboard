## Summary

Keep open. Current main and v2026.6.8 still provide bounded observed-message/channel history plus plugin prompt hooks, but not native opt-in platform scrollback preload at session start; this remains a product/API/privacy decision rather than conservative cleanup.

## What This Changes

The issue requests a native opt-in channel-history preload that fetches recent shared-channel messages at session start or after inactivity and injects them into session context.

## Best Possible Solution

Make a maintainer product/API call first: either keep this as ClawHub/plugin-owned channel context work, or define a narrow generic channel history-preload contract with budgets, throttling, untrusted-context marking, docs, and focused channel tests.

## Reproduction Assessment

Not applicable: this is a feature request, not broken existing behavior. Source and docs inspection show current main only records observed or pending channel history and does not fetch native platform scrollback at session start.

## Solution Assessment

Unclear as requested: a core-wide preloadHistory knob is not obviously the best boundary. The maintainable path is likely either a provider-owned generic SDK/channel contract or an explicit decision to keep this as ClawHub/plugin work.
