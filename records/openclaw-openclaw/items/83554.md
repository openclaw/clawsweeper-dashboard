## Summary

Keep open. Current main has pending-only observed-message history, session lifecycle hooks, and prompt-build hooks, but I did not find a native opt-in platform scrollback preload for session start/resume or an open canonical duplicate that owns this exact Discord-and-other-channels request.

## What This Changes

Request an opt-in native channel-history preload that fetches recent platform scrollback at session start or after inactivity for Discord and other shared channels.

## Best Possible Solution

Adopt a maintainer-approved channel-owned or SDK-level opt-in preload contract with fetch budgets, throttling, pruning, untrusted-context formatting, contextVisibility enforcement, docs, and provider tests, or explicitly route the capability to ClawHub if existing hooks are considered sufficient.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing behavior report. The high-confidence check is source/docs comparison showing pending-only observed history and hooks, but no native session-start platform scrollback preload.

## Solution Assessment

Unclear as requested: a core-wide `preloadHistory` knob is not obviously the best boundary because current docs keep platform-side history fetch channel-local. The safer path is a maintainer design decision for a provider-owned implementation, a narrow SDK contract, or a ClawHub plugin direction.
