## Summary

Keep open. Current main still provides observed-message history windows and plugin prompt hooks, but not a native opt-in platform scrollback preload at session start; the requested behavior adds config/API, channel ownership, and security-policy decisions that need maintainer review.

## What This Changes

Request native opt-in session-start channel history preloading, initially for Discord and ideally generic across shared channels.

## Best Possible Solution

Make a maintainer decision between a ClawHub/plugin-owned capability and a narrow generic channel-history preload contract that providers implement opt-in with budgets, throttling, pruning, untrusted-context labeling, docs, and tests.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken current contract. Source and docs inspection show current behavior uses observed runtime history windows and plugin prompt hooks rather than native platform scrollback preload.

## Solution Assessment

Unclear as requested: a core-wide preloadHistory knob may not be the right ownership boundary. The maintainable path is either provider-owned channel implementation behind a generic contract or a ClawHub plugin if no new core API is needed.
