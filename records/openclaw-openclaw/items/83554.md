## Summary

Keep open. Current main has bounded observed-message history and prompt-build hooks, but not native opt-in provider scrollback preloading at session start/resume; related prior work either covered manual `message read` tooling or was closed stale without merging the requested automatic preload.

## What This Changes

Request native, opt-in channel history preloading for Discord and other shared channels so recent platform scrollback can be injected into session context on session start and after an inactivity window.

## Best Possible Solution

Make a maintainer product/API call first: either keep scrollback preload as a channel/plugin-owned ClawHub capability, or add a narrow generic channel-history preload contract that providers implement opt-in with budgets, throttling, untrusted-context formatting, docs, and tests.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken current contract. Source inspection shows current history support records observed runtime messages rather than fetching native platform scrollback when a session starts.

## Solution Assessment

Unclear as requested: a core-wide `preloadHistory` knob is not obviously the best boundary. The maintainable path is likely a provider-owned implementation behind a generic SDK/channel contract, or leaving the existing hook-based workaround to ClawHub if no core API is missing.
