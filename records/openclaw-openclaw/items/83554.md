## Summary

Keep open. Current main has observed-message group history windows and plugin prompt hooks, but I did not find a native opt-in session-start/platform-scrollback preload or a duplicate canonical issue that already owns this exact request.

## What This Changes

Request native opt-in preloading of recent channel scrollback into session context for Discord and other shared channels.

## Best Possible Solution

Make a maintainer product/API call first: either keep this as a channel/plugin-owned ClawHub capability, or add a narrow generic channel-history preload contract that providers implement opt-in with budgets, throttling, untrusted-context marking, docs, and tests.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken current contract. Source/docs inspection shows current history support records observed messages from runtime maps rather than fetching native platform scrollback when a session starts.

## Solution Assessment

Unclear as requested: a core-wide `preloadHistory` knob is not obviously the best boundary. The maintainable path is likely a provider-owned implementation behind a generic SDK/channel contract, or leaving the existing plugin workaround to ClawHub if no core API is missing.

- Prompt-build-time network fetches could add latency or availability risk unless budgeted, throttled, cached, and fail-open as the issue describes.
