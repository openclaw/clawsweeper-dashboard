## Summary

Keep open. Current main has bounded observed-message history and prompt-injection plugin hooks, but it does not provide a native opt-in platform scrollback preload at session start; the request is fresh, high-signal, and needs a maintainer product/API boundary decision before implementation.

## What This Changes

The issue requests an opt-in native channel-history preload that fetches recent platform scrollback for shared channels at session start or after an inactivity window.

## Best Possible Solution

Decide the boundary first: keep this as ClawHub/provider-owned plugin work, or add a narrow generic channel-history preload contract with budgets, throttling, untrusted-context marking, docs, and focused channel tests.

## Reproduction Assessment

Not applicable: this is a feature request, not broken existing behavior. Source inspection shows current Discord/group history is map-backed observed-message context and does not fetch native platform scrollback at session start.

## Solution Assessment

Unclear as requested: a core-wide `preloadHistory` knob may be useful, but the maintainable boundary needs a maintainer decision. A provider-owned SDK/channel contract or ClawHub plugin path may be safer than one-off Discord/core logic.

## Risks / Open Questions

- A native preload would inject untrusted historical channel content into session context, so permission scope, privacy, retention, and prompt-injection labeling need explicit design.
- A generic cross-channel feature may require a provider-owned SDK/config contract; implementing it only in core or only in Discord could duplicate ownership and drift across plugins.
- Prompt-build-time platform fetches can add latency, rate-limit pressure, or availability failures unless the design has budgets, throttling, pruning, and fail-open behavior.
