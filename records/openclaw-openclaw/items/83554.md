## Summary

Keep open. Current main still provides rolling, observed-message channel history plus plugin prompt hooks, but not a native opt-in platform scrollback preload at session start; the closest prior Discord implementation was stale-closed unmerged, so this needs maintainer product/API/security review rather than cleanup closure.

## What This Changes

The issue requests native opt-in preloading of recent platform channel history into agent session context at session start or after an inactivity window.

## Best Possible Solution

Make a maintainer product/API call first: either keep this as plugin-owned/ClawHub Discord capability with documented hooks, or add a narrow generic channel-history preload contract with budgets, permissions, untrusted-context labeling, throttling, pruning, docs, and focused channel tests.

## Reproduction Assessment

Not applicable: this is a feature request, not broken existing behavior. Source and docs inspection show current main lacks a native channel-history preload while retaining observed-message history and plugin prompt hooks.

## Solution Assessment

Unclear as requested: a core-wide `preloadHistory` knob is not obviously the best ownership boundary. The safer path is a maintainer decision between plugin-owned Discord behavior and a generic provider/channel SDK contract.

- A native preload would fetch and inject historical channel content with provider credentials, so permission scope, privacy, and redaction rules need explicit design.

- Prompt-build-time platform fetches could add latency, rate-limit pressure, or availability failures unless budgets, throttling, pruning, and fail-open behavior are part of the contract.
