## Risks / Open Questions

- A native preload would inject untrusted historical channel content into prompt context, so privacy, permission scope, and prompt-injection labeling need explicit design.
- A cross-channel implementation likely needs a provider-owned SDK/config contract; a Discord-only core shortcut could drift from Slack, Telegram, and other channel plugins.
- Prompt-build-time platform fetches can add latency, rate-limit pressure, and availability failures unless budgets, aborts, throttling, pruning, and fail-open behavior are part of the contract.
- This overlaps but is not identical to missed-inbound replay work, so maintainers should avoid merging session-start orientation with message-loss catch-up semantics by accident.
