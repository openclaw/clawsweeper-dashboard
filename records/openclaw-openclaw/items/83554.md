## Summary

Keep open. Current main and the latest release still provide observed-message history windows and plugin prompt hooks, but not a native opt-in channel scrollback preload at session start; this remains a maintainer product/API/privacy decision, not conservative cleanup.

## What This Changes

The issue asks for an opt-in native channel history preload that fetches recent Discord or other channel scrollback at session start and injects it into session context.

## Best Possible Solution

Make a maintainer product/API call first: either keep this as plugin-owned channel context work, or define a narrow generic channel-history preload contract with budgets, throttling, untrusted-context handling, docs, and focused channel tests.

## Reproduction Assessment

Not applicable: this is a feature request, not broken existing behavior. Source and docs inspection show current main only records observed or pending channel history and does not fetch native platform scrollback at session start.

## Solution Assessment

Unclear as requested: a core-wide preloadHistory knob is not obviously the best boundary. The maintainable path is likely either a provider-owned generic SDK/channel contract or an explicit decision to keep this as plugin-owned work.
