## Summary

Keep open. Current main and the latest release still provide bounded observed-message history plus plugin prompt hooks, but not native opt-in platform scrollback preload at session start; the remaining work needs a product/API/security boundary decision rather than a narrow bug-fix PR.

## What This Changes

Requests native opt-in channel history preloading at session start or after long idle periods, initially for Discord, with budgeted fetch, refresh throttling, and per-channel config.

## Best Possible Solution

Make a maintainer product/API call first: either keep this as ClawHub/plugin-owned channel context work, or define a narrow generic channel history-preload contract with budgets, throttling, untrusted-context marking, docs, and focused channel tests.

## Reproduction Assessment

Not applicable: this is a feature request, not broken existing behavior. Source and docs inspection show current main only records observed/pending channel history and does not fetch native platform scrollback at session start.

## Solution Assessment

Unclear as requested: a core-wide `preloadHistory` knob is not obviously the best boundary. The maintainable path is likely either a provider-owned generic SDK/channel contract or an explicit decision to keep this as ClawHub/plugin work.

- Prompt-build or session-start platform fetches can add latency, rate-limit pressure, and availability failures unless the design has hard budgets, throttling, pruning, and fail-open behavior.
