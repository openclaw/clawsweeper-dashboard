## Summary

Keep open. Current main has rolling observed/pending channel history, Discord read-message tooling, Slack thread seeding, and iMessage downtime catchup, but not a native opt-in scrollback preload for Discord/general channel sessions; this is a fresh feature/config/API boundary request rather than stale cleanup.

## What This Changes

The issue requests an opt-in native channel-history preload that fetches recent platform messages on session start or after inactivity and injects them before prompt build.

## Best Possible Solution

Make a maintainer product/API decision first: either document/promote a ClawHub plugin pattern for provider-owned history injection, or define an opt-in generic channel-history preload contract with budgets, throttling, security filters, docs, and focused provider tests.

## Reproduction Assessment

Not applicable: this is a feature request, not broken existing behavior. Source and docs inspection show current main has no native preload knob and only observed/pending or provider-specific history paths.

## Solution Assessment

Unclear as requested: a core-wide `preloadHistory` option is not obviously the best boundary. The maintainable path is either a provider-owned SDK/channel contract or an explicit decision to keep this as plugin/ClawHub work.
