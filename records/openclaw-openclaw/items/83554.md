## Summary

Keep open. Current main still provides bounded observed-message group history and plugin prompt hooks, but not a native opt-in platform scrollback preload at session start; the closest prior Discord implementation was closed stale rather than merged, and this needs a product/API boundary decision before implementation.

## What This Changes

Request a native, opt-in channel-history preload that fetches recent platform messages for Discord and eventually other shared channels at session start or after an inactivity window.

## Best Possible Solution

Make a maintainer product/API call first: keep this as ClawHub/plugin-owned Discord capability, or define a narrow generic provider history-preload contract with budgets, throttling, untrusted-context marking, docs, and focused channel tests.

## Reproduction Assessment

Not applicable: this is a feature request, not broken existing behavior. Source and docs inspection show current history support records observed messages from runtime maps rather than fetching native platform scrollback when a session starts.

## Solution Assessment

Unclear as requested: a core-wide `preloadHistory` knob is not obviously the best boundary. The maintainable path is likely either a provider-owned generic SDK/channel contract or an explicit decision to keep this as ClawHub/plugin work.

## Risks / Open Questions

- A native preload would inject untrusted historical channel content into session context, so privacy, permission scope, and prompt-injection labeling need explicit design.
- A generic cross-channel feature may need a provider-owned SDK/config contract; implementing only in core or only in Discord could duplicate channel ownership and drift across plugins.
- Prompt-build-time platform fetches could add latency, rate-limit pressure, or availability failures unless the design has budgets, throttling, pruning, and fail-open behavior.
