## Summary

Keep open: current main and the latest release still leave the central cleanup missing, because `cron.remove` only removes the cron job record and the cron reaper only prunes retention-expired `:run:` session keys. An open cross-repository closing PR already claims the issue, so the right path is maintainer review/adoption or a narrow replacement rather than closing or queueing a duplicate repair.

## What This Changes

The issue reports that deleting a cron job leaves stale session records and run-history artifacts behind, so removed cron jobs can still appear as ghost sessions in the Control UI.

## Best Possible Solution

Land or replace the linked cleanup work with a narrow cron-owned fix that removes deleted-job cron session rows while preserving active/named cron-session invariants and current SQLite run-history policy.

## Reproduction Assessment
