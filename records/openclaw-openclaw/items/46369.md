## Summary

Keep open: current main still removes only the cron job record, and the issue is already paired with open PR #46506 using closing syntax, so the issue should stay open until that PR or a replacement lands.

## What This Changes

Deleting a cron job via `cron.remove` should also remove matching cron session-store entries and the job run-log file so deleted jobs do not leave ghost sessions.

## Best Possible Solution

Land or replace the linked service-layer cleanup so successful cron deletion removes matching base/run cron session records and the job run-log file, with regression coverage for preserving unrelated sessions.

## Reproduction Assessment

Yes, by source inspection: the `cron.remove` RPC reaches `CronService.remove`, which only deletes the job from `jobs.json` and never updates `sessions.json` or unlinks the run log. I did not execute a live UI flow in this read-only review.

## Solution Assessment

Yes, fixing this in the cron service removal path is the narrow maintainable direction because both Control UI and tool/RPC deletion converge there. PR #46506 is the current implementation candidate and should be reviewed or replaced rather than closing the issue early.
