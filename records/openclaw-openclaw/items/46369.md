## Summary

Keep open. Current main still removes only the cron job record, while the linked fix PR is open, conflicting, and previously reviewed as needing changes.

## What This Changes

Delete cron job session-store entries and per-job run-log files when `cron.remove` removes a job so deleted cron jobs do not leave ghost sessions.

## Best Possible Solution

Land a narrow cron removal cleanup that preserves removed-event job snapshots, removes the deleted job's base and run session keys after active runs settle, drains pending run-log writes before unlinking, and adds focused regression coverage.
