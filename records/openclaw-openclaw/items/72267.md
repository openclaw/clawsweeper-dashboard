## What This Changes

Removes the leading newline before `#compdef` in the zsh completion generator so generated zsh completion output starts with the directive.

## Best Possible Solution

Land this narrow source fix after redacted zsh fpath/compinit proof is attached and current required checks are refreshed; keep the separate escaping and compinit-registration work out of scope.

## Solution Assessment

Yes. Starting the zsh template at `#compdef` is the narrowest maintainable fix for the reported autoload path; changing profile sourcing or docs would not correct generated files loaded through fpath.
