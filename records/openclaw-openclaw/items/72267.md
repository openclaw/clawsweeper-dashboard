## What This Changes

The PR removes the leading newline from generated zsh completion output so `#compdef` is the first line of `openclaw completion --shell zsh`.

## Best Possible Solution

Land the narrow generator fix after the contributor adds redacted zsh `fpath`/`compinit` proof and the required checks are refreshed green.

## Reproduction Assessment

Yes, source-level: current main and `v2026.6.8` emit a blank first line before `#compdef`, while zsh `compinit` reads only the first line to detect `#compdef`. I did not run a live zsh repro because `zsh` is not installed in this checkout environment.

## Solution Assessment

Yes, this is the best fix shape: the zsh generator should emit `#compdef` at the first byte of the cached completion file. Changing installer/source behavior would not fix fpath autoloading, and bash/fish/PowerShell do not share this invariant.
