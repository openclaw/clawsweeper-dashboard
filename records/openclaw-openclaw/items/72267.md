## Summary

Keep open: current main and the latest release still emit a leading blank line before zsh `#compdef`, and this PR is a focused fix for that contract. It is not merge-ready yet because contributor real behavior proof is not inspectable and the branch/check state still needs refresh.

## What This Changes

The PR removes the leading newline from the generated zsh completion template so `#compdef` is emitted as the first line.

## Best Possible Solution

Land the one-line zsh template fix after inspectable redacted fpath/compinit proof is added, the branch/check state is refreshed, and maintainers decide whether to require a first-line regression assertion.

## Reproduction Assessment

Yes, source-reproducible: current main and v2026.6.9 generate zsh completion text with a leading blank line before `#compdef`, while zsh `compinit` uses the first line of fpath files for completion tags. I did not run a live zsh fpath scenario in this read-only review.

## Solution Assessment

Yes as a code change: starting the zsh template directly at `#compdef` is the narrowest maintainable fix for this first-line autoload contract. The remaining gaps are proof, branch/check freshness, and optional regression coverage rather than a different implementation layer.
