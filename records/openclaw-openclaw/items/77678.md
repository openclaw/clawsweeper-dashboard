The PR adds a skipped QMD embedding-probe state for `searchMode=search`, renders `openclaw memory status --deep` as skipped instead of unavailable, updates memory SDK types, adds tests, and updates the changelog.

## Best Possible Solution

Keep the skipped embedding-probe state, but make the renderer type-safe, clean the changelog hunk, and require real QMD status output before merge.

## Reproduction Assessment

Yes, at source level: current main documents QMD lexical mode as intentionally skipping vector probes, but the current probe path maps that false result to the generic unavailable embeddings error. I did not run a live QMD setup during this read-only review.
