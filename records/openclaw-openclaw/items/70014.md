## Summary

Keep this issue open. Current main still treats configured dreaming workspaces as a replacement set: once `agents.list` resolves any workspaces, the runtime primary workspace is only used as an empty-list fallback, so the reported primary-workspace skip remains live.

## What This Changes

The issue reports that memory-core dreaming processes configured AGI sub-agent workspaces but skips the primary runtime workspace.

## Best Possible Solution

Merge the runtime primary/default workspace with configured agent workspaces in one shared dreaming workspace collection path, dedupe by normalized path, and keep runtime dreaming, doctor status, artifact listing, and tests aligned around that contract.

## Reproduction Assessment

Yes. A source-level reproduction is a config whose `agents.list` contains only sub-agent workspaces while the primary workspace arrives via `ctx.workspaceDir` or `agents.defaults.workspace`; current main processes only the configured list and never adds the primary unless that list is empty.

## Solution Assessment

Yes. The narrow maintainable fix is to merge the primary/runtime workspace into the configured dreaming workspace set with normalized dedupe, not to add AGI-specific IDs or change unrelated dreaming phase behavior.
