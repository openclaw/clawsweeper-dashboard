## Summary

Keep this issue open. Current main still treats configured agent workspaces as a replacement set for dreaming, and only falls back to the runtime primary workspace when that configured set is empty, which matches the reported AGI-subagent-only sweep.

## What This Changes

The issue asks memory-core dreaming to include the primary/default agent workspace when automatic dreaming fans out over configured agent workspaces.

## Best Possible Solution

Make the dreaming workspace collection contract include the runtime primary/default workspace alongside configured agent workspaces, deduped by normalized path, then have runtime dreaming and doctor status share that contract without adding AGI-specific IDs or changing dreaming phase thresholds.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a config whose `agents.list` contains only subagent workspaces while the primary workspace is supplied through the runtime `workspaceDir` or default workspace path; current main processes only the configured list and never adds the primary unless that list is empty.

## Solution Assessment

Yes. The narrow maintainable fix is to merge the primary/default workspace into the configured dreaming workspace set with normalized dedupe, rather than adding owner-specific agent IDs or changing unrelated memory-core scheduling behavior.
