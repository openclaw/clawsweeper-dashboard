## Summary

Current main still returns `terminate: true` after a qualifying `message.send` in `message_tool_only` mode, matching the reported premature turn end, and the explicitly paired fix remains open at https://github.com/openclaw/openclaw/pull/92180. This issue should stay open until that PR is validated and merged or replaced.

## What This Changes

Prevent embedded-runtime agents in `message_tool_only` delivery mode from ending a turn after an interstitial progress message before completing the promised tool work.

## Best Possible Solution

Land a focused fix that distinguishes delivery tracking from agent-loop termination, with coverage for both interstitial sends that must continue and final message-only replies that must not duplicate or spin.

## Reproduction Assessment

Yes at source level: current main deterministically returns `terminate: true` after the qualifying message-tool send, and the loop consumes that as terminal. A full live Anthropic/Slack reproduction was not run during this read-only review.

## Solution Assessment

Unclear whether removing termination unconditionally is the best complete fix; it addresses the reported path, but the final-reply case needs explicit proof that the model stops naturally without duplicate sends or extra turns.
