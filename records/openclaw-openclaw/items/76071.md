## Summary

Keep open. This is a fresh, detailed regression report and current main still has a plausible idle-activity gap: the agent idle watchdog only observes yielded stream events, while the Google SSE transport can consume Gemini chunks that contain metadata/signatures without pushing any stream event.

## What This Changes

The issue asks OpenClaw to prevent Gemini 3.1 Pro Preview agent runs from idling out when the provider is producing hidden or thinking stream activity.

## Best Possible Solution

Add a non-user-visible activity signal or lower-level idle reset for Google/Gemini SSE chunks so hidden/thought-signature frames count as model progress while preserving existing timeout semantics.

## Reproduction Assessment

Yes. The issue provides a concrete real-world setup, and current source supports a focused unit reproduction with a Google SSE stream containing delayed visible output plus thoughtSignature/usage-only chunks that currently do not yield transport events for the idle wrapper.

## Solution Assessment

Yes. The narrow maintainable direction is to count provider stream activity at the Google transport/shared stream boundary, not to raise global timeouts or alter failover policy.

## Repair Work Prompt

Fix issue #76071: Gemini 3.1 Pro Preview via the Google direct API can produce hidden/thought-signature SSE activity while OpenClaw's model idle watchdog only resets on yielded assistant stream events, causing agent runs to hit the full configured timeout. Work in the Google provider transport and/or the shared model stream idle boundary so provider SSE activity that carries thoughtSignature, usage, or other non-visible progress resets liveness without adding user-visible text, empty transcript blocks, provider-specific core policy, or broader timeout changes. Also verify the Google request is actually dispatched and model-call diagnostics can observe first provider activity where appropriate. Add a regression test that simulates Google SSE chunks with no visible text/functionCall before a delayed visible chunk and proves the run does not idle-timeout while preserving final content/signatures. Add a user-facing CHANGELOG.md fix entry crediting @marcoschierhorn if policy allows. Do not change bundled provider defaults, global timeout values, or unrelated provider behavior.
