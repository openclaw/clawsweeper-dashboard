## Summary

Keep open. The feature is still valid and current main lacks `groupScope`, but this PR is not merge-ready because the new scope is only threaded through inbound routing, misses outbound and Telegram topic session derivation, lacks privacy-facing config/docs updates, and carries IRC active-client work already tracked separately in #45662.

## What This Changes

This PR adds a per-route-binding `groupScope` option for routing matched group/channel peers to the agent main session and also adds an IRC active-client registry for proactive sends.

## Best Possible Solution

Ship the group-to-main behavior as a documented binding session-scope feature after settling the config shape, threading it through every session-key derivation path, adding focused regression tests, generated config metadata, changelog, and keeping the IRC active-client fix on its own PR path.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: current main rejects or omits `groupScope` and resolves group/channel peers to isolated keys, while the PR diff leaves outbound fallback and Telegram topic recomputation without the new scope.

## Solution Assessment

No. The per-binding opt-in direction is plausible, but this patch is not the best implementation yet because supported routing paths and user-facing privacy/config documentation are incomplete.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Propagate groupScope into outbound session derivation:** `src/routing/resolve-route.ts:673`
  - body: When a binding sets `groupScope: "main"`, inbound routing can collapse group/channel traffic onto the main session, but outbound helpers still derive group targets with only global `dmScope`. Sends that omit an explicit `sessionKey` can therefore fork the same configured conversation back into a per-channel session instead of the main session.
  - confidence: 0.89
- **[P2] Preserve groupScope in Telegram topic overrides:** `extensions/telegram/src/conversation-route.ts:65-72`
  - body: Telegram `topicAgentId` overrides rebuild the route `sessionKey` with `buildAgentSessionKey()` and only `dmScope`. A topic matching a `groupScope: "main"` binding would be overwritten back to a per-topic group session, so that config combination would not work.
  - confidence: 0.84
- **[P3] Fall back when active IRC sends fail:** `extensions/irc/src/send.ts:72-74`
  - body: When an active monitor client is found, the new IRC path calls `active.sendPrivmsg()` and returns. If the monitor connection drops after `isReady()` but before the write, proactive sends fail instead of trying the transient fallback preserved for missing active clients.
  - confidence: 0.72
