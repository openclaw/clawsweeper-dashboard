  - body: Current main already has a strict `bindings[].session` object for per-binding session overrides, including `dmScope`.   - confidence: 0.92

  - body: The new scope is only threaded through inbound route resolution. Outbound paths that omit an explicit `sessionKey` still call `buildOutboundBaseSessionKey()` with only `cfg.session.dmScope`, so messages mirrored from a merged group can fork back into a per-channel session instead of the configured main session.
  - confidence: 0.87
- **[P2] Preserve groupScope when Telegram topics override agents:** `extensions/telegram/src/conversation-route.ts:65-72`
  - body: Telegram topic-agent overrides rebuild the route's `sessionKey` with `buildAgentSessionKey()` and only pass `dmScope` and `identityLinks`. A matched Telegram group/topic binding configured for main would still be rewritten to a per-topic session in that path.
  - confidence: 0.8
- **[P2] Fall back when an active IRC send fails:** `extensions/irc/src/send.ts:72-74`
  - body: After `getActiveClient()` passes `isReady()`, the socket can still close before `active.sendPrivmsg()` writes. In that race this PR throws without trying the transient send path, which can regress proactive IRC delivery compared with making a fresh connection attempt.
  - confidence: 0.76
