## Summary

## What This Changes

This PR adds a top-level route-binding `groupScope` option for group/channel peers to route to the agent main session and bundles an IRC active-client registry for proactive sends.

## Best Possible Solution

Define an additive binding-session target under the existing `bindings[].session` contract, preserve default group/channel isolation, thread the target through inbound, outbound, Telegram topic routing, docs, tests, and security semantics, and keep the IRC proactive-send fix separately reviewed or redesigned.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config PR. Source inspection gives high confidence that current main still isolates non-direct peers and exposes no group-to-main config option.

## Solution Assessment

No. The per-binding opt-in direction is plausible, but this patch should align with the nested binding session contract, cover outbound and Telegram topic routing, split or harden IRC delivery, document security semantics, and include real behavior proof before merge.

- **[P1] Carry group scope through outbound routing:** `src/routing/resolve-route.ts:673-674`
  - body: This only threads `groupScope` through inbound `resolveAgentRoute()`. Outbound helpers still derive group targets from `dmScope` alone, so sends that omit an explicit `sessionKey` can mirror or write the same configured main-scoped group into a per-channel session.
  - confidence: 0.91
- **[P2] Use the binding session contract for group scope:** `src/config/types.agents.ts:45`
  - body: Current main already places route-binding session controls under `bindings[].session`. Adding `groupScope` at the binding top level creates a competing public config shape and makes `dmScope` plus `groupScope` inconsistent for users and docs.
  - confidence: 0.88
- **[P2] Preserve group scope for Telegram topic overrides:** `src/routing/session-key.ts:138-139`
  - body: Telegram topics with `topicAgentId` rebuild the route session key with `buildAgentSessionKey()` and only pass `dmScope`. A matching `groupScope: "main"` binding can still be rewritten into a per-topic group session for that configuration.
  - confidence: 0.86
- **[P2] Route IRC proactive sends through gateway ownership:** `extensions/irc/src/send.ts:72-74`
  - body: The new active-client map is module-local. `openclaw message send` runs in a CLI process while IRC outbound remains direct, so proactive channel sends can still fall back to the same transient unjoined connection this change is trying to avoid.
  - confidence: 0.91
- **[P2] Publish IRC clients after confirmed channel joins:** `extensions/irc/src/monitor.ts:140`
  - body: The monitor registers the client as soon as `connectIrcClient()` resolves, but readiness is based on numeric `001` and JOIN commands are only sent afterward. A proactive send during that window can still target a channel before confirmed membership.
  - confidence: 0.84
