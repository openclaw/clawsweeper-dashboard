## Summary

Keep open. Current main still lacks the requested group-session scope and the discussion shows a real Discord/IRC use case, but this PR is not merge-ready because the proposed routing option is incomplete across supported session derivation paths and needs a privacy/API shape decision before merge. Reviewed: https://github.com/openclaw/openclaw/pull/47234

## What This Changes

The PR adds an optional per-binding groupScope route setting for group/channel peers and bundles an IRC active-client registry so proactive sends reuse a monitor connection.

## Best Possible Solution

Ship a documented binding session-scope feature, preferably aligned with bindings[].session, with inbound routing, outbound derivation, Telegram topic overrides, generated config help, docs, changelog, and focused regression tests updated; handle the IRC active-client fix through #45662 or rebase after it lands.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: current main has no groupScope schema/type surface, non-direct peers derive isolated group/channel session keys, and the PR diff leaves outbound and Telegram recomputation paths without the new scope.

## Solution Assessment

No. The per-binding opt-in direction is plausible, but this patch is not the best implementation yet because it misses supported session derivation paths, lacks user-facing privacy documentation, and has an API-shape mismatch with existing bindings[].session.dmScope.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Propagate groupScope into outbound session derivation:** `src/routing/resolve-route.ts:673`
  - body: When a binding collapses a group/channel to the main session, outbound helpers that infer a target still build their base session from only global dmScope. Sends without an explicit sessionKey can therefore mirror the same configured conversation into a per-channel session instead of the main session.
  - confidence: 0.89
- **[P2] Preserve groupScope when Telegram overrides topic agents:** `extensions/telegram/src/conversation-route.ts:64-72`
  - body: Telegram topicAgentId overrides rebuild route.sessionKey with buildAgentSessionKey but do not carry the matched binding's group scope. A topic that matched the new main-session binding would be rewritten back to a per-topic group session.
  - confidence: 0.84
- **[P3] Fall back when active IRC sends fail:** `extensions/irc/src/send.ts:72-74`
  - body: After finding an active monitor client, the new path sends and returns without catching write errors. If the connection drops between readiness and sendPrivmsg, proactive channel sends fail instead of trying the transient fallback used when no active client exists.
  - confidence: 0.72
