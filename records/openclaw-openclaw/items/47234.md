## Summary

Keep open. Current main still has no groupScope surface, and this PR addresses a valid group-session fragmentation use case, but the patch misses outbound and Telegram session recomputation paths, lacks user-facing privacy documentation, and bundles separate IRC active-client work already tracked by #45662.

## What This Changes

This PR adds an optional per-binding groupScope route option for group/channel peers and also adds an IRC active-client registry for proactive sends.

## Best Possible Solution

Ship group session scope as a documented binding session-scope feature, likely aligned with bindings[].session, with inbound routing, outbound derivation, Telegram topic overrides, generated config help, docs, changelog, and focused regression tests all updated; keep the IRC active-client fix on its own PR path or rebase after #45662 lands.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: current main rejects or omits groupScope, non-direct peers derive isolated group/channel keys, and the PR diff leaves outbound and Telegram recomputation paths without the new scope. No tests were run because this review was read-only.

## Solution Assessment

No. The per-binding opt-in direction is plausible, but this patch is not the best implementation yet because it misses supported session derivation paths, lacks privacy-facing documentation, and uses a config shape that conflicts with the existing bindings[].session.dmScope pattern.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Propagate groupScope into outbound session derivation:** `src/routing/resolve-route.ts:673`
  - body: When a binding sets groupScope to main, inbound routing can collapse group/channel traffic onto the main session, but outbound helpers still derive inferred targets from only global dmScope. Sends that omit an explicit sessionKey can therefore fork the same configured conversation back into a per-channel session.
  - confidence: 0.89
- **[P2] Preserve groupScope in Telegram topic overrides:** `src/routing/resolve-route.ts:100`
  - body: Telegram topicAgentId overrides rebuild route.sessionKey with buildAgentSessionKey and only global dmScope. A topic matching a groupScope main binding would be overwritten back to a per-topic group session instead of staying on the main session.
  - confidence: 0.84
- **[P3] Fall back when active IRC sends fail:** `extensions/irc/src/send.ts:72-74`
  - body: When an active monitor client is found, the new IRC path calls active.sendPrivmsg and returns. If that client drops after the readiness check but before the write, proactive sends fail instead of trying the transient fallback that is still used when no active client exists.
  - confidence: 0.72
