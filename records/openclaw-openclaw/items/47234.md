## Summary

Keep this PR open. Current main still lacks a group-to-main session scope, so the work is not obsolete, but the patch is not ready because it misses outbound and Telegram session derivation paths, needs an API-shape decision against the newer nested binding session config, lacks required docs/changelog coverage, and bundles the still-open IRC active-client work from #45662.

## What This Changes

The PR adds a route-binding `groupScope` option to route selected group/channel peers into an agent main session and also adds IRC active-client reuse for proactive sends.

## Best Possible Solution

Ship a documented per-binding group session-scope feature through the approved binding session API, propagate it through inbound routing, outbound derivation, and Telegram topic overrides, add focused tests plus generated config docs/help and changelog, and keep the IRC active-client fix split through #45662 or a clean rebase.

## Reproduction Assessment

Yes. Source inspection on current main shows no `groupScope` surface and confirms non-direct peers still use isolated group/channel session keys, while the PR diff can be checked against outbound and Telegram recomputation paths without running tests.

## Solution Assessment

No. The per-binding opt-in direction is plausible and is supported by related user context in #7524, but this patch is not the best complete solution until the binding API shape, privacy docs, outbound routing, Telegram override path, changelog, and #45662 overlap are resolved.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Propagate groupScope into outbound session derivation:** `src/routing/resolve-route.ts:673`
  - body: When a binding collapses a group/channel to the main session, outbound helpers that infer a target still build their base session from only global `dmScope`. Sends without an explicit `sessionKey` can therefore mirror the configured conversation into a per-channel session instead of the main session.
  - confidence: 0.89
- **[P2] Preserve groupScope when Telegram overrides topic agents:** `extensions/telegram/src/conversation-route.ts:70-72`
  - body: Telegram topic-agent overrides rebuild `route.sessionKey` with `buildAgentSessionKey()` but do not carry the matched binding's group scope. A matching Telegram topic would be rewritten back to a per-topic group session instead of the topic agent's main session.
  - confidence: 0.84
- **[P3] Fall back when active IRC sends fail:** `extensions/irc/src/send.ts:72-74`
  - body: After finding an active monitor client, the new path sends and returns without catching write errors. If the connection drops between readiness and `sendPrivmsg`, proactive channel sends fail instead of trying the transient fallback used when no active client exists.
  - confidence: 0.72
- **[P3] Add the required changelog entry:** `src/config/zod-schema.agents.ts:40`
  - body: This PR adds a user-facing config feature and an IRC send fix, but it does not update `CHANGELOG.md`. Repo policy requires user-facing `feat` and `fix` changes to have a changelog entry before merge.
  - confidence: 0.82
