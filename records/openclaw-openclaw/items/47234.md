## Summary

Keep open: current main still lacks the requested group-session scope, and the PR is useful, but it is not merge-ready because it misses outbound and Telegram topic routing paths and needs the API/privacy documentation decision resolved.

## What This Changes

The PR adds a per-route-binding `groupScope` option for routing selected group/channel peers to the main agent session and also adds IRC active-client reuse for proactive sends.

## Best Possible Solution

Ship a documented per-binding group session-scope feature through the chosen binding session API, propagate it through inbound routing, outbound derivation, and Telegram topic overrides, add focused tests plus generated config docs and changelog, and handle the IRC active-client fix through #45662 or a clean rebase.

## Reproduction Assessment

Yes. Source inspection on current main shows no `groupScope` schema/type/docs and non-direct peers always derive isolated group/channel session keys; the PR diff can also be checked against outbound and Telegram recomputation paths without running tests.

## Solution Assessment

No. The per-binding opt-in direction is plausible, but this patch is not the best complete solution yet because it misses supported session derivation paths, lacks user-facing privacy/config documentation, and conflicts with the existing nested `bindings[].session.dmScope` shape.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Propagate groupScope into outbound session derivation:** `src/routing/resolve-route.ts:673`
  - body: When a binding collapses a group/channel to the main session, outbound helpers that infer a target still build their base session from only global `dmScope`. Sends without an explicit `sessionKey` can therefore mirror the configured conversation into a per-channel session instead of the main session.
  - confidence: 0.89
- **[P2] Preserve groupScope when Telegram overrides topic agents:** `extensions/telegram/src/conversation-route.ts:70-72`
  - body: Telegram topic-agent overrides rebuild `route.sessionKey` with `buildAgentSessionKey()` but do not carry the matched binding's group scope. A matching Telegram topic would be rewritten back to a per-topic group session instead of the topic agent's main session.
  - confidence: 0.84
- **[P3] Fall back when active IRC sends fail:** `extensions/irc/src/send.ts:72-74`
  - body: After finding an active monitor client, the new path sends and returns without catching write errors. If the connection drops between readiness and `sendPrivmsg`, proactive channel sends fail instead of trying the transient fallback used when no active client exists.
  - confidence: 0.72
- **[P3] Add the required changelog entry:** `src/config/zod-schema.agents.ts:40`
  - body: This PR adds a user-facing config feature and an IRC send fix, but it does not update `CHANGELOG.md`. Repo policy requires user-facing `feat` and `fix` changes to have a changelog entry before merge.
  - confidence: 0.82
