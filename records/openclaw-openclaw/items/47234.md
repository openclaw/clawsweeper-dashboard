## Summary

Keep open. Current main still lacks the requested group-to-main group session control, and the PR is useful, but it is not merge-ready because it misses outbound and Telegram session-key recomputation paths, lacks privacy-facing config/docs coverage, and bundles the still-open IRC proactive-send work from #45662.

## What This Changes

Adds a per-binding `groupScope` route option that sends matched group/channel conversations to the agent main session and adds an IRC active-client registry so proactive IRC sends can reuse the monitor connection.

## Best Possible Solution

Ship this as a first-class binding session-scope feature: settle the config shape alongside `bindings[].session.dmScope`, propagate it through inbound routing, outbound session derivation, Telegram topic overrides, config metadata, docs, changelog, and focused regression tests, while keeping the IRC active-client fix on the #45662 path.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: current main rejects/omits `groupScope`, group/channel peers still resolve to isolated session keys, and the PR diff leaves outbound and Telegram recomputation paths without the new scope.

## Solution Assessment

No. The per-binding opt-in direction is plausible, but this patch is not the best implementation yet because it misses supported session derivation paths, leaves a privacy-sensitive config undocumented, and mixes unrelated IRC work into the feature branch.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Propagate groupScope into outbound session derivation:** `src/routing/resolve-route.ts:673`
  - body: When a binding sets `groupScope: "main"`, inbound routing can collapse group/channel traffic onto the main session, but outbound helpers still derive group targets from only global `dmScope`. Sends that infer a target session can therefore fork the configured conversation back into a per-channel session instead of the main session.
  - confidence: 0.89
- **[P2] Preserve groupScope in Telegram topic overrides:** `src/routing/resolve-route.ts:100`
  - body: The new scope is not carried into Telegram forum-topic `topicAgentId` overrides. That path recomputes `route.sessionKey` with `buildAgentSessionKey()` and only global `dmScope`, so a matching topic would be overwritten back to a per-topic group session.
  - confidence: 0.84
- **[P2] Generate config help for groupScope:** `src/config/zod-schema.agents.ts:40`
  - body: This adds a user-facing, privacy-sensitive binding field but does not update the generated config metadata/help/docs surface. Users and config tooling would not get the warning or discoverability needed for a mode that merges group/channel context into the main session.
  - confidence: 0.82
- **[P3] Fall back when active IRC sends fail:** `extensions/irc/src/send.ts:72-74`
  - body: When an active monitor client is found, the new IRC path calls `active.sendPrivmsg()` and returns. If that connection drops after `isReady()` but before the write, proactive sends fail instead of trying the transient fallback preserved for missing active clients.
  - confidence: 0.72
