## What This Changes

The issue reports that the packaged browser plugin cannot lazy-load because a bundled browser file imports the private `ssrf-runtime-internal` Plugin SDK subpath that the package export map omits.

## Best Possible Solution

Restore packaged browser CLI/tool lazy-loading while preserving the owner-scoped managed-proxy helper, or intentionally promote a documented SDK contract with security review and package smoke coverage.

## Reproduction Assessment

Yes, source-level. Current main and v2026.6.10 import a package subpath that the package excludes and does not export; I did not execute the packaged Windows browser CLI path in this read-only review.

## Solution Assessment

No, the one-line import path suggested by one candidate is not the best fix as-is because the public `ssrf-runtime` module does not export the bypass helper. The best fix should either preserve owner-scoped private resolution or deliberately promote the SDK surface with maintainer approval.
