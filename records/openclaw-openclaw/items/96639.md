## Summary

Keep open: current main and the latest release still contain the package export mismatch, and several open closing PRs are candidate fixes. The remaining decision is which SDK/private-boundary repair to land, not issue cleanup.

## What This Changes

The issue reports that Browser lazy loading and `openclaw browser` can fail because bundled code imports `openclaw/plugin-sdk/ssrf-runtime-internal`, which the package does not export or ship.

## Best Possible Solution

Land one canonical fix that restores packaged Browser and Ollama lazy loading while preserving the private helper boundary, unless maintainers explicitly approve a documented public SDK promotion with replacement guardrails and package smoke proof.

## Reproduction Assessment

Yes. Source inspection shows current main and `v2026.6.10` import `openclaw/plugin-sdk/ssrf-runtime-internal` from bundled Browser/Ollama paths while package files exclude and do not export that subpath.

## Solution Assessment

No. A one-line Browser import move is not sufficient because public `ssrf-runtime` does not export the bypass helper; the best repair needs an owner-scoped package/load fix or explicit maintainer-approved SDK promotion.
