## Summary

Keep open. The linked Windows bug is valid and current main still has bare helper-command spawns, but this PR is not merge-ready because the patch misses the documented CLI `gog` path, can still directly spawn a resolved `.cmd` wrapper, introduces a command-resolution security concern, and changes the deprecated Plugin SDK barrel without the required SDK follow-through.

## What This Changes

The PR adds a Windows-only executable resolver, applies it to `gcloud` and `tailscale` in Gmail setup utilities and `gog` in the gateway watcher, adds resolver/setup tests, and exports `resolveSystemBin` from the deprecated Plugin SDK `infra-runtime` barrel.

## Best Possible Solution

Keep the branch focused on the Windows Gmail helper-command fix: resolve PATHEXT shims for all documented setup/run/watcher paths, use an existing Windows-safe spawn seam for long-running `.cmd`/`.bat` wrappers, avoid executing a helper resolver by bare name, and remove or intentionally complete the SDK export with baseline/docs updates.

## Reproduction Assessment

Yes. The linked bug provides native PowerShell reproduction, and current main still shows bare `gcloud`, `tailscale`, and `gog` calls on the documented Gmail setup/run/watcher paths; the PR-specific defects are statically reproducible from the diff and current code contracts.

## Solution Assessment

No. Resolving Windows PATHEXT helpers is the right direction, but this implementation misses `src/hooks/gmail-ops.ts`, leaves a direct `.cmd` spawn path, uses a bare `where.exe` lookup, and expands a public SDK surface outside the stated fix.
