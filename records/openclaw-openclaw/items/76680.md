## Summary

Keep open. Current main still has source-backed paths where embedded PI turns gate `before_prompt_build` and `agent_end` on the global hook runner, while loader tests intentionally preserve an older gateway hook runner across later default-mode registry activation; the related open hook issue overlaps, but this memory-lancedb report adds a selected-memory-slot and hot-reload angle that should not be swept as a duplicate yet.

## What This Changes

The issue reports that memory-lancedb auto-recall and auto-capture hooks register successfully but never fire in embedded PI, cron, and channel-driven runs.

## Best Possible Solution

Fix the generic plugin runtime boundary so selected memory-slot and explicitly enabled hook-capable plugins remain visible to embedded PI, cron, restart, and hot-reload hook dispatch without hardcoding memory-lancedb or a channel.

## Reproduction Assessment

Yes by source inspection, but not by live execution: embedded PI obtains the global runner and both reported hooks are skipped when that runner reports no matching hooks, while loader code and tests show a stale-runner path can exist. The reporter also supplied concrete runtime steps and logs for v2026.5.2.

## Solution Assessment

Yes, a generic loader/runtime fix is the best direction. A memory-lancedb-only workaround would miss the shared hook runner and startup-scoped registry behavior already implicated by the related embedded PI hook issue.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/76680 together with the related embedded PI hook visibility issue https://github.com/openclaw/openclaw/issues/76649. Current main gates `before_prompt_build` and `agent_end` on `getGlobalHookRunner().hasHooks(...)`; loader/runtime paths can leave the global runner bound to an older or startup-scoped registry, so memory-lancedb auto-recall/capture can silently skip after config/hook-policy reloads or cron/embedded PI runs. Implement a generic fix so selected memory-slot plugins and explicitly enabled hook-capable plugins with the required hook policy are visible to embedded PI/cron hook dispatch after gateway restart and hot reload. Do not hardcode memory-lancedb, Feishu, or any plugin id, and do not broaden runtime loading to every installed plugin if a manifest/config-derived scope can stay narrow. Add regression coverage for a non-bundled memory-style plugin or fixture registering `before_prompt_build` and `agent_end`, including the stale global-runner/default-mode activation case. Update CHANGELOG.md because this is user-facing plugin hook behavior.
