## Summary

Keep open. Current main still lacks the broad Anthropic server-tools request: no Anthropic server-tool config/schema, no direct `web_search_*` / `web_fetch_*` / `code_execution_*` payload injection, no preservation of server-tool result blocks, and no Anthropic web-search/fetch/server-tool manifest contract; the active related work only covers a narrower web-search slice.

## What This Changes

Request support for opt-in Anthropic native server-side tools, including web search, web fetch, code execution, programmatic tool calling, response block handling, and provider-specific config.

## Best Possible Solution

Keep this as the broad Anthropic server-tools tracker and implement the remaining work in the Anthropic plugin/transport boundary, while letting https://github.com/openclaw/openclaw/issues/49949 and https://github.com/openclaw/openclaw/pull/50692 handle only the narrower native web-search slice.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a provider feature request. Source inspection gives a high-confidence absence check for the requested config, payload injection, server content-block preservation, and Anthropic plugin contract.

## Solution Assessment

Unclear until maintainer product/security design is settled. The maintainable direction appears to be Anthropic plugin/transport-owned support split across config/schema, payload injection, response preservation/rendering, docs, tests, and least-privilege execution policy.

## Security Review

Status: needs_attention

Summary: No patch is under review, but the requested Anthropic code_execution/PTC and server web-fetch behavior is security-sensitive.

Concerns:

- **[medium] Define least-privilege server code execution policy:**
  - body: Allowing Anthropic server-side code execution to invoke OpenClaw custom tools via `allowed_callers` changes command/tool execution, data-access, audit, and transcript semantics; a future implementation needs explicit opt-in defaults, allowlists, rendering behavior, and regression coverage.
  - confidence: 0.9
- **[medium] Preserve web-fetch exfiltration controls:**
  - body: Anthropic documents data-exfiltration risk for server-side web fetch in trusted/untrusted-data mixes, so OpenClaw needs domain limits, max-use controls, and clear activation boundaries before exposing it as a provider-owned server tool.
  - confidence: 0.86
