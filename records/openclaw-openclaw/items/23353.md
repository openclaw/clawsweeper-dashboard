## Summary

Keep this issue open. Current main and v2026.6.5 still lack the broad Anthropic native server-tools feature: there is no Anthropic server-tool config, no provider-owned webSearch/webFetch contract, no `allowed_callers`/PTC surface, and the Anthropic Messages stream path still only turns ordinary `tool_use` blocks into OpenClaw tool calls; the narrower Anthropic web-search tracker at https://github.com/openclaw/openclaw/issues/49949 remains separate and open.

## What This Changes

The issue requests opt-in Anthropic native server-side tools for web search, web fetch, and code execution/PTC as alternatives to OpenClaw client-side tools.

## Best Possible Solution

Implement a typed Anthropic-plugin capability with explicit per-tool server-tool policy, model/version gating, caller authorization, retention disclosure, container/session ownership, and shared stream/transcript preservation; keep native web search coordinated with https://github.com/openclaw/openclaw/issues/49949.

## Reproduction Assessment

Yes, at source level: current main and v2026.6.5 lack the requested Anthropic tool IDs, caller authorization, server-result preservation, and provider contract wiring. I did not run a live Anthropic request in this read-only issue review.

## Solution Assessment

No, the raw `tools.serverTools` list proposed in the issue is not the best complete shape. The maintainable solution is provider-owned typed capability and policy wiring that covers authorization, retention, model compatibility, stream preservation, and session ownership.
