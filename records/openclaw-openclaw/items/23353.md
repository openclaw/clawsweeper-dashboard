## Summary

Keep open. Current main still lacks the broad Anthropic native server-tools request: config/schema does not define Anthropic server tools or PTC, the Anthropic transport only builds client-side tool definitions and only preserves client tool/text/thinking blocks, and the related open work is scoped to native web_search rather than web_fetch, code_execution/PTC, allowed_callers, and server-result handling.

## What This Changes

Request support for configurable Anthropic native server-side web_search, web_fetch, and code_execution/PTC tools, including allowed_callers and response-block handling.

## Best Possible Solution

Keep this as the broad Anthropic server-tools tracker, let the narrower web_search provider work proceed separately, and design the remaining web_fetch, code_execution/PTC, allowed_callers, server-block preservation, docs, tests, and least-privilege defaults in the Anthropic plugin/transport boundary.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a provider feature request. Source inspection gives a high-confidence absence check for the requested config/schema, Anthropic server-tool payload injection, response block preservation, and plugin capability contract.

## Solution Assessment

Yes for keeping this issue open. Closing it for the narrower native web_search work would drop the remaining web_fetch, code_execution/PTC, allowed_callers, response rendering, and security-policy requirements.

## Security Review

Status: needs_attention

Summary: The requested web_fetch and code_execution/PTC support expands network, execution, retention, and tool-invocation policy surface.

Concerns:

- **[medium] Define least-privilege code execution and PTC policy:**
  - body: Anthropic code execution can run Bash commands, manipulate files, and use code_execution_20260120 for programmatic tool calling through allowed_callers; OpenClaw needs explicit opt-in, allowlists, transcript/audit behavior, and tests before exposing that surface. ([platform.claude.com](https://platform.claude.com/docs/en/agents-and-tools/tool-use/code-execution-tool))
  - confidence: 0.9
- **[medium] Preserve web_fetch exfiltration controls:**
  - body: Anthropic's web fetch docs warn about data exfiltration when untrusted input and sensitive data mix, and recommend max_uses and allowed_domains controls; OpenClaw should map those controls into its web/tool policy before enabling provider-hosted fetch. ([platform.claude.com](https://platform.claude.com/docs/en/agents-and-tools/tool-use/web-fetch-tool))
  - confidence: 0.88
