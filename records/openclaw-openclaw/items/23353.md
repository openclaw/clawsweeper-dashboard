## Summary

Keep open. Current main still lacks the broad Anthropic native server-tools support requested here, and the open related PR only targets the narrower Anthropic native web_search provider path.

## What This Changes

The issue asks OpenClaw to opt into Anthropic server-side web_search, web_fetch, and code_execution/PTC, inject those server tools into Anthropic requests, and preserve/render the resulting server content blocks.

## Best Possible Solution

Keep this as the broad Anthropic native server-tools tracker; review the narrower #49949/#50692 web_search work separately, then design provider-owned support for web_fetch, code_execution/PTC, allowed_callers, server content blocks, docs, tests, and security defaults.

## Reproduction Assessment

Not applicable as a feature request, but current-main source inspection gives a high-confidence verification path: the requested config keys, payload injection, and streamed block preservation are absent.

## Solution Assessment

No as a one-shot cleanup or close. The maintainable path is phased provider-owned implementation with any needed generic transport seams and explicit security policy for PTC/tool invocation.
