## Security Review

Status: needs_attention

Summary: The requested feature is security-sensitive because it would add provider-hosted network fetch/search and provider-hosted code execution that can invoke selected OpenClaw tools.

Concerns:

- **[medium] Define least-privilege server web policy:** `src/config/types.tools.ts:571`
  - body: Anthropic-hosted web_search and web_fetch would move outbound access, domain controls, result handling, and cost controls into provider request parameters, so OpenClaw needs explicit opt-in policy and narrow defaults before enabling them.
  - confidence: 0.91
- **[medium] Constrain programmatic tool authority:** `src/agents/anthropic-transport-stream.ts:465`
  - body: PTC requires code_execution plus allowed_callers on custom tools, which would let provider-hosted code invoke selected OpenClaw tools; that needs explicit allowlists, audit behavior, and least-privilege defaults.
  - confidence: 0.9
- **[medium] Preserve server blocks without losing state:** `src/agents/anthropic-transport-stream.ts:74`
  - body: Server tool and result blocks need a first-class transcript representation so replay, citations, encrypted result continuity, caller metadata, and session state are not silently dropped or flattened.
  - confidence: 0.88
