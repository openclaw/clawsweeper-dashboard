## Summary

Keep open. Current main still lacks the broad Anthropic native server-tools request: config/schema do not expose Anthropic serverTools or PTC, the Anthropic transport still emits client tool definitions and only parses text/thinking/client tool_use blocks, and the linked open work is limited to a narrower native web_search provider slice.

## What This Changes

The issue requests opt-in Anthropic native server-side tools for web_search, web_fetch, and code_execution/PTC instead of OpenClaw client-side equivalents.

## Best Possible Solution

Keep this as the broad Anthropic server-tools tracker, let https://github.com/openclaw/openclaw/issues/49949 and https://github.com/openclaw/openclaw/pull/50692 handle the narrower web_search provider, and separately design web_fetch, code_execution/PTC, allowed_callers, server-block preservation/rendering, docs, tests, and least-privilege defaults in the Anthropic plugin/transport boundary.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a provider feature request. Source inspection gives a high-confidence absence check for the requested config/schema, Anthropic server-tool payload injection, server content-block preservation, and plugin contract.

## Solution Assessment

Yes for keeping this issue open. The narrow web_search implementation path should not close the broader web_fetch/code_execution/PTC request, and the remaining work needs maintainer product/security design before implementation.

## Security Review

Status: needs_attention

Summary: No patch is under review, but the requested Anthropic web_fetch and code_execution/PTC behavior expands network and execution policy surface.

Concerns:

- **[medium] Define least-privilege server code execution policy:**
  - body: Anthropic’s current code execution docs describe Bash command support and direct file manipulation in a provider-hosted sandbox, so OpenClaw should require explicit opt-in, allowlists, transcript/audit semantics, and regression coverage before exposing code_execution/PTC. ([docs.anthropic.com](https://docs.anthropic.com/en/docs/agents-and-tools/tool-use/code-execution-tool?utm_source=openai))
  - confidence: 0.9
- **[medium] Preserve web-fetch exfiltration controls:**
  - body: Anthropic’s web fetch docs warn about data-exfiltration risk with untrusted input plus sensitive data and describe domain and max_uses controls; OpenClaw should map those controls into its existing web/tool policy before enabling provider-hosted fetch. ([docs.anthropic.com](https://docs.anthropic.com/de/docs/agents-and-tools/tool-use/web-fetch-tool?utm_source=openai))
  - confidence: 0.88
