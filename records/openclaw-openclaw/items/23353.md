## Summary

Keep open. Current main still lacks the broad Anthropic native server-tools request: schema does not accept Anthropic serverTools/PTC config, the Anthropic transport still emits only client-side tool definitions and only parses text/thinking/client tool_use blocks, and the linked open work covers only the narrower native web_search provider slice.

## What This Changes

Request support for Anthropic native server-side tools, including web_search, web_fetch, code_execution/PTC, allowed_callers, response block handling, and provider-scoped configuration.

## Best Possible Solution

Keep this as the broad Anthropic server-tools tracker while the narrower web_search provider work proceeds separately, then design web_fetch, code_execution/PTC, allowed_callers, server result preservation/rendering, docs, tests, and least-privilege defaults in the Anthropic plugin/transport boundary.

## Security Review

Status: needs_attention

Summary: No patch is under review, but the requested web_fetch and code_execution/PTC support expands network and execution policy surface.

Concerns:

- **[medium] Define least-privilege code execution policy:**
  - body: Anthropic code execution can run Bash commands, manipulate files, and use provider-hosted sandbox state; PTC lets code_execution call custom tools through allowed_callers, so OpenClaw needs explicit opt-in, allowlists, transcript/audit behavior, and regression coverage before exposing it. ([docs.anthropic.com](https://docs.anthropic.com/en/docs/agents-and-tools/tool-use/code-execution-tool))
  - confidence: 0.9
- **[medium] Preserve web_fetch exfiltration controls:**
  - body: Anthropic's web fetch docs call out data-exfiltration risk with untrusted input plus sensitive data and recommend max_uses and allowed_domains controls, so OpenClaw should map those controls into its existing web/tool policy before enabling provider-hosted fetch. ([docs.anthropic.com](https://docs.anthropic.com/en/docs/agents-and-tools/tool-use/web-fetch-tool))
  - confidence: 0.88
