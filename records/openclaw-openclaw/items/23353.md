## Summary

Current main and v2026.6.5 still do not implement Anthropic native server tools. The existing Anthropic plugin can mutate outgoing payloads, but the broad request also needs typed server-result preservation, programmatic caller authorization, container/session lifecycle, retention disclosure, model gating, and live Anthropic validation; all related implementation PRs are closed unmerged, while the narrower native web-search request remains separately open.

## What This Changes

Add opt-in Anthropic-native web search, web fetch, code execution, programmatic tool calling, and first-class server content-block handling.

## Best Possible Solution

Implement a typed Anthropic-plugin capability with explicit per-tool caller authorization, model/version gating, retention disclosure, and container lifecycle, adding only the smallest shared stream/transcript contract needed to preserve server blocks and citations; coordinate native web search through https://github.com/openclaw/openclaw/issues/49949.

## Reproduction Assessment

Yes at source level: current main and v2026.6.5 lack the requested tool IDs, caller metadata, container lifecycle, and server-result handling, and no related implementation PR merged. A live Anthropic request was not run during this read-only issue review.

## Solution Assessment

No, the proposed raw `tools.serverTools` list is not the best complete solution. The maintainable path is a provider-owned typed capability that also defines authorization, retention, session ownership, response preservation, and model compatibility.

## Risks / Open Questions

- Anthropic-hosted code execution and programmatic tool calling are not eligible for Zero Data Retention, creating a privacy and retention boundary that must be disclosed and intentionally enabled.
- Broad `allowed_callers` could let provider-hosted code invoke powerful OpenClaw tools; defaults need least-privilege allowlisting and clear direct-versus-programmatic caller semantics.
- Reusable container IDs and expiry create session state that could be lost, retried incorrectly, or associated with the wrong agent or session.
- The current shared stream and transcript model may drop or flatten server results, citations, caller data, and container metadata unless its typed contract is extended.
- A generic list of raw versioned server-tool IDs would expose unstable provider details and could conflict with model compatibility or future Anthropic tool versions.
