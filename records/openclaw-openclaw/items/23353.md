## Summary

Keep open. Current main still does not implement the broad Anthropic native server-tools request: config/schema only covers managed web search/fetch plus OpenAI/Codex-native search, the Anthropic transport still emits client-tool definitions, server tool/result stream blocks are not preserved, and the bundled Anthropic plugin has no web search/fetch/server-tool contract. The related open #50692 is scoped to #49949's narrower native web_search provider request, while #23469 and #23573 are closed unmerged and do not resolve the remaining web_fetch, code_execution/PTC, allowed_callers, rendering, and security-default work.

## What This Changes

Request support for Anthropic server-side web_search, web_fetch, and code_execution/PTC by adding configuration, Anthropic request-tool injection, client-tool suppression where appropriate, and response rendering/preservation for server tool blocks.

## Best Possible Solution

Keep this as the broad Anthropic native server-tools tracker. Let #49949/#50692 cover the narrower web_search provider if it passes review, then design the remaining Anthropic plugin/transport work for web_fetch, code_execution/PTC, allowed_callers, server result block preservation/rendering, docs, tests, and security defaults.
