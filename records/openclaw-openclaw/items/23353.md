## Summary

Keep open. Current main still lacks the broad Anthropic native server-tools support requested here: config/schema cover managed web search/fetch plus OpenAI/Codex-native search, Anthropic Messages transport still emits client-side custom tool definitions, and the related work either closed unmerged or covers only a narrower web_search slice.

## What This Changes

Request Anthropic-native server tool opt-in for web_search, web_fetch, and code_execution/PTC, including config, API payload injection, response block handling, and allowed_callers support.

## Best Possible Solution

Keep this as the broad Anthropic server-tools tracker, let #49949/#50692 handle the narrower web_search slice, and design Anthropic plugin/transport-owned support for web_fetch, code_execution/PTC, allowed_callers, server content-block preservation/rendering, docs, tests, and security defaults.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request for a provider-native server-tools mode. Source inspection on current main confirms the requested config, payload injection, and server block handling are absent.

## Solution Assessment

Unclear until maintainer design is settled. The maintainable direction appears to be Anthropic plugin/transport-owned capability support with explicit config, schema, docs, tests, and security policy rather than a hidden default or core-only bypass.
