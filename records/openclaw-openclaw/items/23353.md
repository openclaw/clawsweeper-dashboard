## Summary

Keep open. Current main still lacks the broad Anthropic native server-tools request: config/schema has no Anthropic server-tool or PTC fields, the Anthropic Messages transport still emits ordinary client tool definitions, streamed server result blocks are not preserved, and the active related work only covers native web_search.

## What This Changes

Request support for Anthropic native server-side web_search, web_fetch, and code_execution/PTC tools with opt-in config, payload injection, and server result rendering.

## Best Possible Solution

Keep this as the broad Anthropic server-tools tracker, let https://github.com/openclaw/openclaw/issues/49949 and https://github.com/openclaw/openclaw/pull/50692 handle the narrower web_search slice, and design the remaining web_fetch/code_execution/PTC work in the Anthropic plugin/transport boundary.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a provider feature request. Current-main source inspection gives a high-confidence absence check for the requested config, payload injection, server content-block preservation, and Anthropic plugin contract.

## Solution Assessment

Unclear until maintainers settle product and security design. The maintainable direction appears to be Anthropic plugin/transport-owned support split across config/schema, payload injection, response rendering/preservation, docs, tests, and least-privilege execution policy.
