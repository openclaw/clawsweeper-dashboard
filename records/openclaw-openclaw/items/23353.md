## Summary

Keep open. Current main still lacks the broad Anthropic native server-tools request: there is no Anthropic server-tool config/schema, no direct server-tool payload injection, no preservation of server-tool result blocks, and no Anthropic web_fetch/code_execution/PTC contract; the linked open work only targets a narrower native web_search slice.

## What This Changes

The issue requests opt-in support for Anthropic native server-side web_search, web_fetch, code_execution, and programmatic tool-calling semantics.

## Best Possible Solution

Keep this as the broad Anthropic server-tools tracker, let the linked native web_search issue/PR handle only that narrower provider slice, and design web_fetch, code_execution/PTC, allowed_callers, server-block preservation/rendering, docs, tests, and least-privilege defaults in the Anthropic plugin/transport boundary.

Not applicable as a bug reproduction: this is a provider feature request. Source inspection gives a high-confidence absence check for the requested config/schema, Anthropic server-tool payload injection, server content-block preservation, and plugin contract.

## Solution Assessment

Unclear until maintainer product and security design is settled. The maintainable direction is Anthropic plugin/transport-owned support, with the narrow web_search provider handled separately from web_fetch and code_execution/PTC policy.

## Security Review

Status: needs_attention

Summary: No patch is under review, but the requested Anthropic web_fetch and code_execution/PTC behavior is security-sensitive.

Concerns:

- **[medium] Define least-privilege server code execution policy:**
  - body: Anthropic’s current code_execution tool can run Bash commands and edit files in a provider-hosted sandbox, and support would add new response/result block handling; OpenClaw should require explicit opt-in, allowlists, transcript/audit semantics, and regression coverage before enabling PTC-style workflows. ([docs.anthropic.com](https://docs.anthropic.com/en/docs/agents-and-tools/tool-use/code-execution-tool?utm_source=openai))
  - confidence: 0.9
- **[medium] Preserve web-fetch exfiltration controls:**
  - body: Anthropic’s web_fetch docs warn about residual data-exfiltration risk with untrusted input and sensitive data and recommend disabling the tool or using max_uses/domain controls; OpenClaw needs equivalent activation and domain policy before exposing provider-hosted fetch. ([docs.anthropic.com](https://docs.anthropic.com/de/docs/agents-and-tools/tool-use/web-fetch-tool?utm_source=openai))
  - confidence: 0.88
