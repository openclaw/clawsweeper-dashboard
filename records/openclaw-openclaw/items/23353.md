## Summary

Keep open. Current main still lacks the broad Anthropic native server-tools surface, and the open related PR only covers the narrower native web_search provider path.

## What This Changes

Request Anthropic-native server-side web_search, web_fetch, code_execution/PTC, allowed_callers, config, payload injection, and response-block handling.

## Best Possible Solution

Keep this as the broad Anthropic server-tools tracker; land the web_search-only provider separately if valid, then design the remaining web_fetch, code_execution/PTC, allowed_callers, server-block preservation, docs, tests, and security defaults together.

## Solution Assessment

Unclear as a one-shot implementation. The web_search provider subset can proceed through #49949/#50692, but web_fetch, code_execution/PTC, allowed_callers, and server result preservation need a provider and security design at the Anthropic plugin/transport boundary.
