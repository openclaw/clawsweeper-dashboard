## Summary

Keep open. Current main still lacks the broad Anthropic server-tools request: there is no Anthropic server-tool config/schema, no direct server-tool payload injection, no preservation of server-tool result blocks, and no Anthropic web-fetch/code-execution/PTC contract; the open related work only covers a narrower native web-search slice.

## What This Changes

The issue requests opt-in Anthropic native server-side tools for web_search, web_fetch, and code_execution/PTC as alternatives to OpenClaw client-side tools.

## Best Possible Solution

Keep this as the broad Anthropic server-tools tracker, let https://github.com/openclaw/openclaw/issues/49949 and https://github.com/openclaw/openclaw/pull/50692 handle only native web_search, and design the remaining web_fetch, code_execution/PTC, allowed_callers, server block rendering, docs, tests, and least-privilege defaults in the Anthropic plugin/transport boundary.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a provider feature request. Source inspection gives a high-confidence absence check for the requested config, payload injection, server content-block preservation, and Anthropic plugin contract.

## Solution Assessment

Unclear until maintainer product and security design is settled. The maintainable direction is Anthropic plugin/transport-owned support, with the narrow web_search provider handled separately from web_fetch and code_execution/PTC policy.

## Security Review

Status: needs_attention

Summary: No patch is under review, but the requested Anthropic web_fetch and code_execution/PTC behavior is security-sensitive.

Concerns:

- **[medium] Define least-privilege server code execution policy:**
  - body: Anthropic code execution creates another execution environment beside OpenClaw client-side tools, and PTC would let server-side code invoke selected OpenClaw tools through `allowed_callers`; a future implementation needs explicit opt-in, allowlists, audit/rendering semantics, and regression coverage. ([docs.anthropic.com](https://docs.anthropic.com/en/docs/agents-and-tools/tool-use/code-execution-tool))
  - confidence: 0.9
- **[medium] Preserve web-fetch exfiltration controls:**
  - body: Anthropic documents residual exfiltration risk for web_fetch when untrusted input is processed with sensitive data, so OpenClaw should require clear activation boundaries plus max-use/domain controls before exposing it. ([platform.claude.com](https://platform.claude.com/docs/en/agents-and-tools/tool-use/web-fetch-tool?2f226f2c_page=2&46f68bc1_page=2&d7430fcd_page=2))
