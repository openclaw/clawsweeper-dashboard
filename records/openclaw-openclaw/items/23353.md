## Security Review

Status: needs_attention

Summary: The requested code_execution/PTC and allowed_callers surface expands tool execution and data-flow authority, so it needs explicit security design before implementation.

Concerns:

- **[medium] Define PTC tool-call authority:**
  - body: allowed_callers would let Anthropic server-side code invoke OpenClaw tools; the design needs default-deny or explicit allowlist behavior so code_execution cannot gain broader local tool access than intended.
  - confidence: 0.86

  - body: Server result blocks can carry encrypted citation/result state and continuation data; dropping or flattening them could break multi-turn citations or code-execution continuation.
