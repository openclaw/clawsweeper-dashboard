## Security Review

Status: needs_attention

Summary: The requested feature expands provider-hosted execution, network fetch, data-retention, and tool-authority surfaces, so it needs security design before implementation.

Concerns:

- **[medium] Define least-privilege server fetch policy:**
  - body: Anthropic web_fetch docs warn about data-exfiltration risk with untrusted input plus sensitive data and recommend controls such as disabling the tool, max_uses, and allowed_domains; OpenClaw should map those into explicit policy before enabling provider-hosted fetch. ([docs.anthropic.com](https://docs.anthropic.com/en/docs/agents-and-tools/tool-use/web-fetch-tool))
  - confidence: 0.9
- **[medium] Constrain programmatic tool authority:**
  - body: Anthropic code execution emits server_tool_use/result blocks and PTC uses allowed_callers so code can call selected tools; OpenClaw needs explicit opt-in, allowlists, transcript behavior, and audit coverage before exposing that authority. ([docs.anthropic.com](https://docs.anthropic.com/en/docs/agents-and-tools/tool-use/code-execution-tool))
  - confidence: 0.86
- **[low] Account for server-side container retention:**
  - body: Anthropic documents server-side code execution containers, workspace scope, and retained container data/artifacts, so OpenClaw should document and test what enters transcripts, files, and user-visible state before enabling this path. ([docs.anthropic.com](https://docs.anthropic.com/en/docs/agents-and-tools/tool-use/code-execution-tool))
  - confidence: 0.82
