## Summary

Keep open. Current main still does not implement the broad Anthropic native server-tools request: config/schema lacks Anthropic server-tool and PTC fields, the Anthropic Messages transport still emits ordinary client tool definitions, server result blocks are not preserved, and the active related work only covers native web_search.

## What This Changes

The issue requests opt-in Anthropic native server-side web_search, web_fetch, code_execution, and PTC support as alternatives to client-side OpenClaw tools.

## Best Possible Solution

Keep this as the broad Anthropic server-tools tracker, implement the remaining work at the Anthropic plugin/transport boundary, and let https://github.com/openclaw/openclaw/issues/49949 plus https://github.com/openclaw/openclaw/pull/50692 handle only the narrower web_search slice.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a provider feature request. Current-main source inspection gives a high-confidence absence check for the requested config, payload injection, server content-block preservation, and Anthropic plugin contract.

## Solution Assessment

Unclear until maintainer product/security design is settled. The maintainable direction appears to be Anthropic plugin/transport-owned support split across config/schema, payload injection, response rendering/preservation, docs, tests, and least-privilege execution policy.
