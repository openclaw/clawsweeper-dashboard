## Summary

Keep open. Current main still lacks the broad Anthropic native server-tools support requested here: no accepted config/schema for Anthropic server tools, no Anthropic server-tool injection in the Messages transport, no preserved server-tool result blocks, and the related implementation work is either closed unmerged or scoped only to the narrower native web_search slice.

## What This Changes

Request support for Anthropic native server-side web_search, web_fetch, code_execution/PTC, allowed_callers, and response block handling.

## Best Possible Solution

Keep this as the broad Anthropic server-tools tracker, with any implementation owned by the Anthropic plugin/transport boundary and split so web_search, web_fetch, code_execution/PTC, allowed_callers, block preservation/rendering, docs, tests, and security defaults are all covered deliberately.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a provider feature request. Source inspection on current main gives a high-confidence absence check for the requested config, payload injection, and server content-block handling.

## Solution Assessment

Unclear until maintainer design is settled. The maintainable direction appears to be Anthropic plugin/transport-owned support with explicit schema, response semantics, docs, tests, and security defaults, while #49949/#50692 remain a narrower web_search slice.
