## Security Review

Status: needs_attention

Summary: No patch is under review, but the requested web_fetch and code_execution/PTC support expands network, execution, and data-exfiltration policy surface.

Concerns:

- **[medium] Define least-privilege code execution policy:**
  - body: Anthropic code execution can run commands, create or edit files, and process uploaded files inside the API conversation; PTC lets code_execution call custom tools through allowed_callers, so OpenClaw needs explicit opt-in, allowlists, transcript/audit behavior, and regression coverage before exposing it. ([docs.claude.com](https://docs.claude.com/en/docs/agents-and-tools/tool-use/code-execution-tool?cmid=b38d5b4f-e65c-4994-b7ff-02472f785995&utm_source=openai))
  - confidence: 0.9
- **[medium] Preserve web_fetch exfiltration controls:**
  - body: Anthropic's web fetch docs call out data-exfiltration risk when untrusted input is processed alongside sensitive data and recommend controls such as max_uses and allowed_domains, so OpenClaw should map those controls into existing web/tool policy before enabling provider-hosted fetch. ([docs.claude.com](https://docs.claude.com/en/docs/agents-and-tools/tool-use/web-fetch-tool?utm_source=openai))
  - confidence: 0.88
