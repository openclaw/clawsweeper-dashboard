## Security Review

Status: needs_attention

Summary: The requested feature expands provider-hosted execution, network fetch, data-retention, and tool-authority surfaces, so it needs security design before implementation.

Concerns:

- **[medium] Define least-privilege server fetch policy:**
  - body: Anthropic web_fetch docs warn about data-exfiltration risk with untrusted input plus sensitive data and recommend controls such as disabling the tool, max_uses, and allowed_domains; OpenClaw should map those into explicit policy before enabling provider-hosted fetch. ([docs.anthropic.com](https://docs.anthropic.com/de/docs/agents-and-tools/tool-use/web-fetch-tool?utm_source=openai))
  - confidence: 0.9
- **[medium] Constrain programmatic tool authority:**
  - body: Anthropic PTC uses code_execution plus allowed_callers so server-side code can call selected custom tools; OpenClaw needs explicit opt-in, allowlists, transcript behavior, and audit coverage before exposing that authority. ([platform.claude.com](https://platform.claude.com/docs/en/agents-and-tools/tool-use/programmatic-tool-calling?2f226f2c_page=4&utm_source=openai))
  - confidence: 0.88
- **[medium] Preserve server blocks without losing session state:** `src/agents/anthropic-transport-stream.ts:1087`
  - body: Anthropic server tools emit server_tool_use/result blocks, while current Anthropic stream parsing only preserves text, thinking, redacted_thinking, and client tool_use blocks; losing or misclassifying those blocks would affect transcript/session state.
  - confidence: 0.84
