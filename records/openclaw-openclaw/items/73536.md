## Summary

Keep open. The TypeScript direction is narrow and addresses the bundled-MCP timeout bug, but this PR is not merge-ready because the new request-timeout inheritance is undocumented and the required user-facing changelog entry is still absent.

## What This Changes

The PR adds a resolved bundled-MCP request timeout, propagates it through MCP transport/session state, passes it to `Client.callTool()` as request options, and adds transport-config regression tests.

## Best Possible Solution

Land this PR or an equivalent patch after documenting the explicit `connectionTimeoutMs` request-timeout inheritance with the 60s unset fallback and adding an Unreleased Fixes changelog entry with contributor credit.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: configure a bundled MCP server with an explicit timeout above 60,000 ms and call a tool that runs longer than 60s; current main passes no `RequestOptions.timeout`, so the SDK default still applies.

## Solution Assessment

No, not as merge-ready. The code fix is the narrow maintainable path and preserves the unconfigured 60s fallback, but the user-facing docs and required changelog need to land with it.
