## What This Changes

The PR adds a bundled-MCP per-call request timeout derived from explicit `connectionTimeoutMs` with a 60s unset fallback, propagates it through MCP transport/session state, passes it to `Client.callTool`, and adds transport-config regression tests.

## Best Possible Solution

Land this PR or an equivalent patch after documenting explicit `connectionTimeoutMs` request-timeout inheritance with the 60s unset fallback and adding an Unreleased Fixes changelog entry with contributor credit.
