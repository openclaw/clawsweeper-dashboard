## Summary

Keep open: current main carries session lifecycle fields through the Gateway and Control UI state, but the Sessions table still does not render an active/completed/timed-out status indicator or include lifecycle status in search/sort.

## What This Changes

Add a visible lifecycle status indicator for each Control UI session row, covering active, completed, timed out, and aborted/failed states.

## Best Possible Solution

Add a compact localized Status column or badge in the Sessions table using existing GatewaySessionRow lifecycle fields, with focused render coverage and a user-facing changelog entry.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high-confidence evidence that lifecycle data exists, while the current Sessions table renders kind, updated time, tokens, and overrides without lifecycle status.

## Solution Assessment

Yes. The narrow maintainable path is UI-only: derive labels from existing GatewaySessionRow lifecycle fields, add focused rendering coverage, and avoid changing Gateway lifecycle semantics unless implementation proves a missing row value.
