## Summary

Keep open: current main carries coarse session lifecycle data through the Gateway and Control UI state, but the normal Sessions table still lacks an always-visible active/completed/timed-out status badge or column. The closed duplicate at https://github.com/openclaw/openclaw/issues/46792 reinforces this as the canonical narrow UI tracker, while https://github.com/openclaw/openclaw/issues/57933 is a broader heartbeat/liveness feature and does not supersede this issue.

## What This Changes

The issue requests a visible Control UI session lifecycle indicator for active, completed, and timed-out sessions.

## Best Possible Solution

Add a compact localized Status badge or column on every Sessions row using existing GatewaySessionRow lifecycle fields, with focused render, i18n, responsive-table coverage, and no new Gateway heartbeat protocol.

## Reproduction Assessment

Not applicable as a feature request. Source inspection of current main gives high-confidence evidence that lifecycle row data exists while the normal Sessions table omits an always-visible lifecycle status indicator.

## Solution Assessment

Yes for a narrow first pass. The maintainable solution is UI-only: derive a readable badge from existing row fields, handle missing values gracefully, and leave richer live/heartbeat semantics to the broader health issue.
