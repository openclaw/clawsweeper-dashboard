## Summary

Keep open: current main still carries session lifecycle data through the Gateway and Control UI state, but the normal Sessions table does not show a visible active/completed/timed-out badge. The closed overlapping https://github.com/openclaw/openclaw/issues/46792 was consolidated into this tracker, while https://github.com/openclaw/openclaw/issues/57933 is a broader heartbeat/stall design and does not replace this narrow UI request.

## What This Changes

Add a visible Control UI Sessions-list status indicator for active, completed, timed-out, or aborted sessions, with last activity easier to scan.

## Best Possible Solution

Add a compact localized Status badge or column in the Sessions table from existing GatewaySessionRow lifecycle fields, with focused rendering coverage and no new heartbeat protocol.
