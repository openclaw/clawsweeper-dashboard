## Summary

Keep open: current main has session lifecycle data and even shows `status` in the expanded details panel, but the main Control UI Sessions table still has no visible active/completed/timed-out status badge or column. The closed duplicate at https://github.com/openclaw/openclaw/issues/46792 reinforces this as the narrow UI tracker; https://github.com/openclaw/openclaw/issues/39127 and https://github.com/openclaw/openclaw/issues/57933 are broader Gateway activity/heartbeat features and do not supersede the simple table indicator.

## What This Changes

Add a visible active/completed/timed-out lifecycle indicator and clearer last-activity context to Control UI session rows.

## Best Possible Solution

Add a compact localized Status badge or column in the Sessions table from existing row lifecycle fields, while leaving richer heartbeat/activity API work to the broader open Gateway liveness trackers.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main gives high-confidence evidence that lifecycle row data exists, but the normal Sessions table omits an always-visible lifecycle status indicator.

## Solution Assessment

Yes for the narrow request. The maintainable first pass is UI-only: derive a readable badge from existing `GatewaySessionRow` fields, handle missing values gracefully, and avoid adding a new heartbeat or activity protocol in this issue.
