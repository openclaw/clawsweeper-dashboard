## Summary

Keep open: current main still carries coarse session lifecycle data, but the normal Control UI Sessions table does not render a visible active/completed/timed-out status indicator. The closed duplicate https://github.com/openclaw/openclaw/issues/46792 points back here, while https://github.com/openclaw/openclaw/issues/57933 is a broader heartbeat/stall design rather than a replacement for this narrow UI gap.

## What This Changes

Add a visible Control UI Sessions-list indicator for active, completed, timed-out, or aborted sessions, with last-activity context easier to see.

## Best Possible Solution

Render a compact localized Status or Live badge/column in the Sessions table from existing row lifecycle fields, with focused UI coverage and no new heartbeat protocol.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main gives high-confidence proof that lifecycle row data exists but the normal Sessions table still lacks an always-visible lifecycle status indicator.

## Solution Assessment
