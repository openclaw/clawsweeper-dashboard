## Summary

Keep open: current main carries the needed lifecycle fields through the Gateway and Control UI state, but the normal Sessions table still does not render an always-visible active/completed/timed-out indicator. The closed duplicate at https://github.com/openclaw/openclaw/issues/46792 should stay consolidated here, while the broader heartbeat/liveness work at https://github.com/openclaw/openclaw/issues/57933 does not supersede this narrower UI gap.

## What This Changes

Request to add visible lifecycle status indicators and prominent last-activity context to Control UI session rows.

## Best Possible Solution

Add a compact localized Status badge or column on every Sessions row using existing GatewaySessionRow lifecycle fields, with focused render, i18n, and responsive-table coverage and no new Gateway activity protocol.

## Reproduction Assessment

## Solution Assessment

Yes for a narrow first pass. The maintainable solution is UI-only: derive a readable badge from existing row fields, handle missing values gracefully, and leave richer heartbeat or stall semantics to https://github.com/openclaw/openclaw/issues/57933.
