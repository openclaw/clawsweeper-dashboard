Keep open: current main carries coarse session lifecycle data and now exposes status only in expanded session details, but the Sessions table still lacks a visible per-row active/completed/timed-out badge or status column.

## What This Changes

Add a Control UI Sessions-list status indicator so users can distinguish active, completed, timed-out, aborted, and recently active sessions at a glance.

## Best Possible Solution

Add a compact localized Status badge or column visible on every Sessions row using existing GatewaySessionRow lifecycle fields, with focused render coverage and a user-facing changelog entry.
