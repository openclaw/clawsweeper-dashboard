## Summary

Keep this open. Current main already carries session lifecycle fields from the Gateway into the Control UI session state, but the Sessions table still renders only kind, updated time, tokens, compaction, and per-session overrides, with no visible active/completed/timed-out status badge or status sort/filter path.

## What This Changes

Add a visible lifecycle status indicator for Control UI session rows so active, completed, timed-out, and aborted/failed sessions are distinguishable at a glance.

## Best Possible Solution

Implement a compact Status column or badge in the Control UI Sessions table using the existing GatewaySessionRow lifecycle fields, with focused rendering coverage for running, completed, failed/killed, and timeout rows. If the UI adds sortable/searchable status, extend the small app state union and tests accordingly without changing Gateway lifecycle semantics.

## Reproduction Assessment

Yes. A source-level reproduction is enough here: renderSessions receives rows that can contain status/runtime fields, but current headers, row rendering, sort, and filter code do not consume those fields for lifecycle status.

## Solution Assessment

Yes. The best fix is a narrow Control UI presentation/state change because the Gateway and websocket update path already provide the lifecycle data needed for the requested indicator.
