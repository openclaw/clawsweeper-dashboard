## Summary

Keep open: current main carries session lifecycle fields through the Gateway and Control UI controller, but the Sessions table still does not show an always-visible active/completed/timed-out indicator. The closed overlapping report points back to this issue as the canonical tracker for the remaining UI gap.

## What This Changes

Add visible lifecycle status indicators to Control UI session rows so active, completed, timed-out, or aborted sessions can be distinguished at a glance.

## Best Possible Solution

Add a compact localized Status badge or column on every Sessions row using existing GatewaySessionRow lifecycle fields, with focused render coverage and a user-facing changelog entry.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main gives high-confidence proof that lifecycle data exists while the Sessions table omits an always-visible lifecycle status indicator.

## Solution Assessment

Yes. The narrow maintainable path is UI-only: derive the row label from existing GatewaySessionRow lifecycle fields, handle missing values gracefully, and avoid changing Gateway lifecycle semantics unless implementation proves a missing row value.
