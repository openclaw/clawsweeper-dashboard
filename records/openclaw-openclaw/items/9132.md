## Summary

Keep open: current main still carries coarse lifecycle fields through the Gateway and Control UI state, but the normal Sessions table does not render an at-a-glance active/completed/timed-out indicator. The closed duplicate https://github.com/openclaw/openclaw/issues/46792 points back here, while https://github.com/openclaw/openclaw/issues/57933 is a broader heartbeat/stall design rather than a replacement for this narrow UI gap.

## What This Changes

Add a visible lifecycle status indicator to each Control UI Sessions list row so active, completed, timed-out, aborted, and recently active sessions are easier to distinguish.

## Best Possible Solution

Render a compact localized Status or Live badge/column in the Sessions table from existing row lifecycle fields, with focused UI coverage and no new heartbeat protocol.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main gives high-confidence proof that lifecycle row data exists but the normal Sessions table lacks an always-visible lifecycle status indicator.

## Solution Assessment

Yes for a narrow first pass. The maintainable direction is a UI-only projection of existing status, active-run, and runtime fields, while leaving heartbeat and stall detection to the broader linked issue.
