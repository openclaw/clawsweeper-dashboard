## Summary

Keep open: current main exposes coarse session lifecycle data through the Gateway and Control UI state, but the normal Sessions table still does not render an always-visible active/completed/timed-out status badge or column.

## What This Changes

The issue requests an always-visible lifecycle status indicator and prominent last-activity context in the Control UI Sessions list.

## Best Possible Solution

Add a compact localized Status badge or column on every Sessions row using existing GatewaySessionRow lifecycle fields, with focused render, i18n, and responsive-table coverage and no new Gateway activity protocol.

## Reproduction Assessment

Not applicable as a feature request. Source inspection of current main gives high-confidence evidence that lifecycle row data exists while the normal Sessions table omits an always-visible lifecycle status indicator.

## Solution Assessment

Yes for a narrow first pass. The maintainable solution is UI-only: derive a readable badge from existing row fields, handle missing values gracefully, and leave richer live/heartbeat semantics to the broader linked activity issues.
