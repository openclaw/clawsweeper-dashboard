## Summary

Keep open: current main already carries lifecycle data into Control UI session rows, but the Sessions table still does not render a lifecycle badge or use status in local search/sort. The closed overlapping #46792 does not supersede this narrower UI gap, while #39127 and #57933 track broader activity/heartbeat APIs.

## What This Changes

The issue requests a visible lifecycle status indicator for Control UI session rows so users can distinguish active, completed, and timed-out sessions.

## Best Possible Solution

Ship a compact Status column or badge in the Sessions table using the existing GatewaySessionRow lifecycle fields, with focused rendering/search or sort coverage and no new gateway activity API in this issue.

## Reproduction Assessment

Yes. Source inspection on current main is enough: lifecycle fields flow from Gateway session rows into Control UI state, but renderSessions/renderRows does not render or search/sort a lifecycle status indicator.

## Solution Assessment

Yes. The narrow maintainable path is a Control UI presentation change over existing row data; broader per-session heartbeat/activity APIs should stay with #39127/#57933.
