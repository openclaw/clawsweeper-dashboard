## Summary

Keep open: current main carries session lifecycle fields through the Gateway and UI state, and raw status can appear only inside checkpoint-expanded details, but the normal Sessions table still lacks an always-visible active/completed/timed-out lifecycle badge or status sort/search/filtering. The closed related dashboard issue at https://github.com/openclaw/openclaw/issues/46792 points back here, while the broader heartbeat issue at https://github.com/openclaw/openclaw/issues/57933 does not supersede this narrower UI projection.

## What This Changes

Add a visible active/completed/timed-out lifecycle indicator and prominent last-activity context to the Control UI Sessions list.

## Best Possible Solution

Add a compact localized Status column or badge in ui/src/ui/views/sessions.ts from existing GatewaySessionRow lifecycle fields, with focused responsive styling and render coverage; leave heartbeat/stall semantics to the broader health-tracking issue.

## Reproduction Assessment

Not applicable as a feature request. Source inspection against current main gives high-confidence proof that lifecycle row data exists but the always-visible Sessions table still renders no active/completed/timed-out status indicator.

## Solution Assessment

Yes. The narrow maintainable fix is a Control UI projection of existing row.status, subagentRunState, hasActiveSubagentRun, startedAt, endedAt, and runtimeMs, without adding a new Gateway heartbeat protocol here.
