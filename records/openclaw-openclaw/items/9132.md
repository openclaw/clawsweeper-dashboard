## Summary

Keep open: current main still carries session lifecycle data into Control UI rows, but the Sessions table and its local search/sort controls do not render or consume lifecycle status. No open PR appears to be closing this issue, and the closed overlapping #46792 does not supersede this focused UI gap.

## What This Changes

Add a visible lifecycle indicator to the Control UI Sessions list for active, completed, timed-out, and related terminal session states.

## Best Possible Solution

Ship a compact Status column or badge in the Sessions table using existing GatewaySessionRow lifecycle fields, with focused i18n and rendering tests; leave broader activity APIs to #39127/#57933.

## Reproduction Assessment

Yes. Source inspection on current main is enough: lifecycle fields flow from the Gateway into Control UI row data, but renderSessions does not render or locally search/sort a lifecycle status indicator.

## Solution Assessment

Yes. The narrowest maintainable solution is a Control UI presentation change using the existing session-row contract, not a new gateway heartbeat or activity API.
