## Summary

Keep open: the regression is real and the PR has useful live proof, but the current patch can double-substitute already-resolved plugin config, breaking escaped placeholders and exposing env values to plugins.

## What This Changes

## Best Possible Solution

Resolve raw plugin-config snapshots before plugin handoff without reprocessing values that already passed read-time env substitution, and lock that with escaped-placeholder regression coverage.

No as currently patched. The validation boundary is a plausible repair point, but the implementation must preserve one-pass `$${VAR}` escape semantics instead of unconditionally re-running env substitution on already-resolved config.
