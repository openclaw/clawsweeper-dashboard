- **[P1] Give the fixture a schema that allows apiKey:** `src/plugins/loader.test.ts:1028`
  - body: This fixture uses `writePlugin`, which writes the default empty schema with `additionalProperties: false` and no properties.   - confidence: 0.94

  - body: `validatePluginConfig` substitutes against `process.env`, but plugin loading already resolves a per-load `env` from `options.env ?? process.env`. Callers that supply an explicit env can still hand unresolved placeholders to plugins when the variable is absent from the process-global environment, so pass the loader env into this helper from both registry load paths.
  - confidence: 0.9
