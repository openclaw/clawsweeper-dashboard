## Summary

Close as ClawHub/plugin scope. The request is an optional supervision policy rather than a missing core primitive: current main exposes subagent lifecycle hooks and plugin runtime subagent controls, and a closely related same-author two-strike request was already closed by a maintainer as ClawHub/community plugin work.

## What This Changes

Add configurable N-strike failure tracking for agents/subagents, threshold actions, and additional subagent completion context.

## Best Possible Solution

Leave core as-is and build the watchdog as a ClawHub/plugin package that listens to subagent lifecycle hooks, stores its own strike counters, and uses plugin runtime/task orchestration with explicit operator permissions for any destructive action such as deleting or disabling sessions.

## Reproduction Assessment

Not applicable as a runtime bug reproduction. There is a high-confidence static verification path: current main lacks the proposed core counters/config keys while exposing the plugin hooks and runtime primitives needed for an external policy.

## Solution Assessment

No, adding this as core agent-defaults enforcement is not the best default path. The narrower maintainable solution is a plugin/ClawHub watchdog, with a core API discussion only if a concrete missing plugin primitive appears during implementation.
