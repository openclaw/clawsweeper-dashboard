## Summary

Close as a duplicate of the older open Private Mode tracker. This issue adds useful hard-gate persistence details, but the same per-session/per-channel no-history/no-memory privacy contract is already owned by the canonical open tracker, and current source inspection shows the feature remains a product/security design request rather than an implemented fix.

## What This Changes

The issue requests a per-session secure/ephemeral mode that prevents transcripts, compaction summaries, LCM/session indexing, memory flushes, and durable session metadata from being persisted.

## Best Possible Solution

Consolidate implementation under the open Private Mode tracker and design one additive, enforceable private/ephemeral session contract that covers protocol, config, session store, transcripts, memory flush, plugin/context-engine hooks, session-memory indexing, channel defaults, and UI status.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high confidence current main cannot express or enforce the requested mode because the protocol/config/session-entry surfaces lack a privacy flag and normal session paths still write transcript/store state.

## Solution Assessment

Yes, closing this issue as a duplicate is the best cleanup path. The older open tracker already owns the broader Private Mode contract, while this issue's valuable hard-gate details should be linked there rather than splitting the same privacy design across two open issues.
