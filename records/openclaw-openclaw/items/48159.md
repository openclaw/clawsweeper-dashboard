## Summary

Close as duplicate/superseded: the remaining secure/ephemeral no-persistence work is already tracked by the open canonical Private/Secret/Incognito tracker at https://github.com/openclaw/openclaw/issues/7403, while current main still lacks the requested hard privacy contract.

## What This Changes

Request a per-session secure/ephemeral mode that prevents transcripts, compaction/LCM indexing, memory flushes, session-store metadata, search, and channel defaults from persisting sensitive conversations.

## Best Possible Solution

Keep the remaining work consolidated under https://github.com/openclaw/openclaw/issues/7403 and design one additive private/ephemeral session contract with hard no-persistence gates and regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high confidence current main cannot express or enforce the requested mode because closed schemas lack an ephemeral/private flag and normal session paths still write durable transcript/store state.

## Solution Assessment

Yes for cleanup routing. Consolidating this issue into the open canonical tracker is the narrowest maintainable path; the actual implementation still needs product/security design before code work.
