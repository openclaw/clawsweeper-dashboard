## Summary

Keep open. Current main still lacks a hard per-session ephemeral/no-persistence contract across config, session store, Gateway RPCs, transcript writes, compaction, memory, and plugin-visible context, and the related HTTP passthrough issue is narrower rather than a replacement.

## What This Changes

Request a per-session secure/ephemeral mode that prevents transcript, compaction, memory, plugin, search, and session-store persistence for sensitive conversations.

## Best Possible Solution

Design an additive core ephemeral session contract with hard no-write gates in session config, Gateway protocol, session store, transcript writes, compaction/context-engine hooks, memory flush, plugin SDK visibility, channel defaults, status/UI verification, and regression tests proving no durable transcript, index, memory file, compaction summary, or surviving sensitive metadata is created.

## Reproduction Assessment

Not applicable as a feature/security design request. Current-main source and docs verification is high-confidence: the requested flag cannot be configured or passed through the session RPC surface, and normal creation/append/compaction paths still use durable session files.

## Solution Assessment

Unclear for implementation details. The requested direction is the right product boundary, but the best exact fix needs maintainer security/product design because it spans core session contracts, plugin APIs, memory/compaction behavior, channel defaults, and user-visible verification.
