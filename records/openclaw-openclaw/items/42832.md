## Summary

Keep open: the Nostr bug still exists on current main, but this PR is not merge-ready because it lacks real relay proof, conflicts with current main after Nostr gateway refactors, and introduces a raw mediaUrl disclosure risk.

## What This Changes

The PR changes the Nostr DM subscription filter, keeps the gateway account task alive until abort, adds a sendMedia attachment-link fallback, and adds focused tests.

## Best Possible Solution

Land a current-main Nostr fix that passes one Filter object, keeps startNostrGatewayAccount pending until abort with cleanup, avoids raw media URL forwarding, and includes focused regression tests plus redacted relay proof.

## Reproduction Assessment

Yes. Source inspection against current main and the pinned nostr-tools 2.23.3 contract gives a high-confidence path for the malformed subscription filter and immediate clean-exit restart behavior, though no live relay was run in this read-only review.

## Solution Assessment

No as-is. The single-filter and long-lived account task directions are the right narrow fix, but the branch needs current-main adaptation, real relay proof, and removal or strict public-URL validation of the media fallback before merge.
