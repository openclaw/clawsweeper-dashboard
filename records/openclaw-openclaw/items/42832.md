## Summary

Keep open. Current main still has the Nostr DM subscription and gateway lifetime bugs this PR targets, but the branch is stale against current Nostr refactors and its media fallback needs a guard before merge.

## What This Changes

The PR changes the Nostr plugin to pass a single DM subscription filter to `subscribeMany()`, keep `startAccount()` alive until abort, and add a `sendMedia` text-link fallback with regression tests.

## Best Possible Solution

Land a rebased or replacement fix on current main that uses a single Nostr `Filter`, keeps `startNostrGatewayAccount()` abort-bound, and only sends public attachment links or otherwise relies on the existing text fallback.

## Reproduction Assessment

Yes, by source inspection. Current main still nests the Nostr DM filter before relay subscribe and returns from Nostr `startAccount()` immediately, while the gateway manager auto-restarts resolved channel tasks.

## Solution Assessment

No, not as-is. The subscription and lifecycle fixes are the right direction, but the patch must be rebased onto current `gateway.ts` and the media fallback should reject or omit non-public media URLs.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Reject local media URLs before DM fallback:** `extensions/nostr/src/channel.ts:167-169`
  - body: This fallback sends `mediaUrl` verbatim in a Nostr DM. Existing send paths can pass `file://` URLs or absolute local paths, so this would leak host filesystem details and produce unusable recipient links; restrict the fallback to public `http(s)` URLs or keep using the text-only fallback for local media.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: The patch introduces a medium-risk data exposure path by forwarding arbitrary media URLs into encrypted DM text.

Concerns:

- **[medium] Local media paths can leak in Nostr DMs:** `extensions/nostr/src/channel.ts:168`
  - body: `sendMedia` formats any `mediaUrl` into the outgoing message, but existing media send paths can pass `file://` URLs or absolute local paths. That exposes local filesystem details to the recipient and should be guarded before merge.
  - confidence: 0.86
