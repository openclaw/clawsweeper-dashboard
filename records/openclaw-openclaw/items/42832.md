## Summary

Keep open: this PR targets valid current and shipped Nostr DM bugs, but it is not mergeable as-is because it lacks real Nostr proof, is dirty against main, and adds a raw media URL fallback that can expose local or internal references.

## What This Changes

The PR changes the Nostr plugin to pass a single DM filter to subscribeMany, keep the gateway account task alive until abort, add a DM media-link fallback, and add focused tests.

## Best Possible Solution

Land a current-main Nostr fix that passes a plain Filter object, keeps startNostrGatewayAccount pending until abort with cleanup, avoids raw media URL DM fallback except for validated public HTTP(S) links, and includes focused regression tests plus redacted relay proof.

## Reproduction Assessment

Yes. The current-main and v2026.5.12 source paths still pass a Filter[] where nostr-tools 2.23.3 expects one Filter object, and the Nostr start task still resolves into the gateway auto-restart path.

## Solution Assessment

No as-is. The filter and long-lived lifecycle direction is right, but the patch needs a current-main adaptation and should remove the raw media fallback or restrict it to validated public URLs.
