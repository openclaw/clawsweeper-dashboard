## Summary

Keep open. Current main still has the Nostr DM subscription-shape and account-lifetime failures this PR is trying to repair, but the PR cannot merge as-is because it is dirty against main and adds an unsafe media fallback that current outbound delivery no longer needs.

## What This Changes

The PR changes the Nostr channel to pass a single DM filter to `subscribeMany`, keep gateway account startup pending until abort, add a `sendMedia` adapter that formats attachment links into DM text, and add focused regression tests.

## Best Possible Solution

Land a refreshed Nostr-owned fix on current main: keep `startNostrGatewayAccount` alive until abort with idempotent cleanup, pass the single `Filter` object required by `nostr-tools@2.23.3` in the DM subscription path, and cover both regressions with focused tests. Do not add Nostr `sendMedia` unless maintainers intentionally change the channel media contract and document/test that broader design.

## Reproduction Assessment

Yes. Static inspection of current main gives a high-confidence reproduction path: the Nostr account task resolves immediately into the supervisor restart path, and the DM subscription still passes the array-wrapped filter that the pinned `nostr-tools@2.23.3` `subscribeMany` API does not expect.

## Solution Assessment

No, not as-is. The subscription and lifecycle parts are the right narrow direction, but the patch must be refreshed for current `gateway.ts` and the `sendMedia` URL fallback should be removed because current core delivery already drops media URLs and sends caption text for no-media adapters.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Drop the Nostr sendMedia URL fallback:** `extensions/nostr/src/channel.ts:164-172`
