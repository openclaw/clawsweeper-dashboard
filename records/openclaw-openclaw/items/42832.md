## Summary

Keep open: current main still carries the Nostr subscription filter and gateway lifecycle failures, so this PR is not obsolete, but the branch is not mergeable as-is because it lacks after-fix real Nostr proof and adds a raw media-URL DM fallback.

## What This Changes

The branch changes the Nostr DM subscription filter, startAccount lifecycle, outbound media handling, and regression tests for NIP-04 DM delivery.

## Best Possible Solution

Land a current-main Nostr fix that passes a plain Filter to subscribeMany, keeps startNostrGatewayAccount pending until abort with cleanup, relies on the existing text fallback or validates only safe public HTTP(S) media URLs, and adds focused regression coverage plus redacted relay proof.

## Reproduction Assessment

Yes, source-reproducible. Configure the Nostr plugin and start the gateway: current main still passes a Filter[] where nostr-tools 2.23.3 expects one Filter object, and the Nostr start task still resolves into the gateway restart path.

## Solution Assessment

No as-is. The filter and long-lived lifecycle changes are the right direction, but the patch should be adapted to current main and should remove the raw media fallback or restrict it to validated public URLs.
