## Summary

Keep open: the Nostr DM delivery bug is still present on current main and the PR contains useful fix direction, but the branch is not merge-ready because it lacks real relay proof and introduces a media URL disclosure regression.

## What This Changes

The PR changes the Nostr plugin to pass a single DM subscription filter, keep gateway account startup pending until abort, add a sendMedia attachment-link fallback, and add focused tests.

## Best Possible Solution

Land a current-main Nostr fix that passes one Filter object, keeps `startNostrGatewayAccount` pending until abort with cleanup, relies on the generic text fallback or validates only public media URLs, and includes tests plus redacted relay proof.

## Reproduction Assessment

Yes. Source inspection on current main plus the `nostr-tools` 2.23.3 type/runtime contract gives a high-confidence path for the malformed subscription filter and immediate clean-exit restart behavior, though I did not run a live relay in this read-only review.

## Solution Assessment

No as-is. The single-filter and long-lived account-task directions are the right narrow fix, but the media fallback should be removed or constrained and the branch still needs redacted real relay proof.
