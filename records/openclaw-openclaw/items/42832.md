## Summary

Keep open: current main still has the Nostr DM subscription and account-lifecycle bugs, while this PR is stale against the current Nostr gateway split and adds an unsafe media fallback.

## What This Changes

The PR changes the Nostr channel to pass a single DM filter to `subscribeMany()`, keep account startup pending until abort, add a `sendMedia` attachment-link fallback, and cover those paths with tests.

## Best Possible Solution

Land a current-main replacement that fixes the `subscribeMany()` filter shape and Nostr account lifecycle, relies on the core text fallback for unsupported media unless public URL handling is explicitly safe, and adds focused regression coverage plus a changelog entry.

## Reproduction Assessment

Yes from source inspection: current main still returns from `startNostrGatewayAccount()` immediately, and the gateway monitor restarts tasks that resolve without abort. The `subscribeMany()` issue is also source-reproducible against the published `nostr-tools@2.23.3` contract.

## Solution Assessment

No as-is. The single-filter and long-lived lifecycle fixes are the right direction, but the PR should be rebased onto current `gateway.ts` and should not ship the raw `mediaUrl` DM fallback without validation.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Reject local media URLs in the Nostr fallback:** `extensions/nostr/src/channel.ts:164-166`
  - body: This new `sendMedia` path copies `mediaUrl` directly into DM text. The outbound runtime can pass `file://` or host-local media references with media access, so Nostr recipients would receive leaked local paths and unusable attachment links; either omit this adapter and use the core text fallback or only include explicitly public HTTP(S) URLs.
  - confidence: 0.88

## Security Review

Status: needs_attention

Summary: The PR introduces a concrete data-exposure risk by forwarding raw media URLs into Nostr DMs.

Concerns:

- **[medium] Raw media URLs can leak local paths:** `extensions/nostr/src/channel.ts:164`
  - body: The added Nostr `sendMedia` fallback formats `Attachment: ${mediaUrl}` without checking whether the URL is public. Existing outbound paths can carry `file://` or local host media references, so this could expose filesystem paths or internal-only references to DM recipients.
  - confidence: 0.86
