## Summary

Keep open: this PR targets valid Nostr DM delivery and restart-loop failures that are still present on current main, but it is not merge-ready because real relay proof is absent and the new media fallback can disclose raw private media references.

## What This Changes

The branch changes the Nostr plugin to pass one DM subscription filter to `subscribeMany()`, keep `startAccount()` alive until abort, add a `sendMedia` attachment-link fallback, and add focused tests.

## Best Possible Solution

Land a current-main Nostr fix that passes one `Filter` object, keeps `startNostrGatewayAccount` pending until abort with cleanup, uses the generic text fallback or validates only public media URLs, and includes regression tests plus redacted relay proof.

## Reproduction Assessment

Yes. Source inspection plus reporter relay logs show current main passes a `Filter[]` where `nostr-tools` 2.23.3 expects one `Filter`, and `startNostrGatewayAccount` resolves into the gateway auto-restart path; no live relay was run in this read-only review.

## Solution Assessment

No as-is. The single-filter and long-lived account-task directions are the right narrow fix, but the media fallback should be removed or constrained and the branch still needs redacted real relay proof.
