## Summary

Keep open: this PR targets real current-main Nostr DM failures, but it cannot merge as-is because it lacks after-fix real behavior proof, is stale against the current Nostr gateway/outbound split, and adds an unsafe raw media URL fallback.

## What This Changes

The PR changes the Nostr DM subscription filter shape, gateway start lifecycle, outbound media fallback, and adds focused regression tests.

## Best Possible Solution

Land a current-main Nostr fix that passes a plain Filter object to subscribeMany, keeps startNostrGatewayAccount pending until abort with cleanup, relies on the existing text fallback or validates only safe public HTTP(S) media URLs, and includes focused regression coverage plus redacted relay proof.

## Reproduction Assessment

Yes. The current-main source path still passes a Filter[] where nostr-tools 2.23.3 expects one Filter object, and the Nostr start task still resolves into the gateway auto-restart path.

## Solution Assessment

No as-is. The filter and long-lived lifecycle direction is right, but the patch needs a current-main rebase/adaptation and should remove the raw media fallback or restrict it to validated public URLs.
