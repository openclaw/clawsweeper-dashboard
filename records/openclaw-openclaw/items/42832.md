## Summary

Keep open: the PR targets real Nostr DM subscription and gateway lifecycle failures that are still present on current main and in the latest checked release, but it cannot merge as-is because it lacks after-fix real relay proof, is now conflicting with current main, and introduces a media URL data-exposure risk.

## What This Changes

The PR changes the Nostr DM subscription filter, keeps Nostr account startup pending until abort, adds a sendMedia attachment-link fallback, and adds focused tests.

## Best Possible Solution

Land a current-main Nostr fix that passes one Filter object, keeps startNostrGatewayAccount alive until abort with cleanup, avoids raw media URL forwarding, and includes focused regression tests plus redacted relay proof.

## Reproduction Assessment

Yes. Source inspection against current main and the pinned nostr-tools 2.23.3 contract gives a high-confidence reproduction path for the malformed subscription filter and immediate clean-exit restart behavior, though no live relay was run in this read-only review.

## Solution Assessment

No as-is. The single-filter and lifecycle directions are the narrow right fix, but the PR needs current-main adaptation, real relay proof, and removal or strict public-URL validation of the media fallback before merge.
