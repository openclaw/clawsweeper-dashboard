## Summary

Keep open: current main still has the Nostr subscription and gateway lifecycle failures, but this PR is not merge-ready because it conflicts with current main, lacks real relay proof, and adds a raw media URL fallback that regresses the current text-only media handling.

## What This Changes

The PR changes Nostr DM subscription filtering to a single filter, keeps Nostr gateway startup pending until abort, adds a sendMedia attachment-link text fallback, and adds focused Nostr tests.

## Best Possible Solution

Rework or replace the PR on current main: keep the single-filter `subscribeMany` call and long-lived `startAccount`, rely on text-only fallback unless safe public media links are explicitly supported, and include redacted real Nostr relay proof.

## Solution Assessment

No, not as-is. The single-filter and long-lived account-task directions are the right fix shape, but the PR needs current-main adaptation, real relay proof, and removal of the raw media URL fallback.
