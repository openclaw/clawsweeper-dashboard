## Summary

Keep open: the branch targets real Nostr DM failures that are still present on current main and the latest checked release, but it should not merge as-is because it lacks real relay proof and the added media fallback can disclose raw media references.

## What This Changes

The PR changes the Nostr plugin to pass a single DM subscription filter, keep account startup pending until abort, add a media-to-text DM fallback, and add focused tests.

## Best Possible Solution

Land a current-main Nostr fix that passes one Filter object, keeps startNostrGatewayAccount pending until abort with cleanup, avoids raw media URL forwarding, and includes focused regression tests plus redacted relay proof.

## Reproduction Assessment

Yes. Source inspection against current main, nostr-tools 2.23.3, NIP-01, and the related relay logs gives a high-confidence path for the malformed subscription filter and clean-exit restart loop, though no live relay was run here.

## Solution Assessment

No as-is. The single-filter and long-lived account-task directions are the right narrow fix, but the media fallback should be removed or constrained to validated public URLs and the branch still needs real relay proof/current-main adaptation.
