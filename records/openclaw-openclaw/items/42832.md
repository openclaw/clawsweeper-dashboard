## Summary

Keep open: the Nostr DM subscription and gateway lifecycle fixes are still needed on current main, but this PR is conflicting, lacks real Nostr relay proof, and adds a media fallback that can leak raw media references.

## What This Changes

The branch changes Nostr DM subscription filtering, makes the Nostr gateway account task wait for abort, adds a sendMedia text fallback, and adds focused tests.

## Best Possible Solution

Land a current-main Nostr fix that passes one Filter object, keeps startNostrGatewayAccount pending until abort with cleanup, avoids raw media URL forwarding, and includes focused tests plus redacted relay proof.

## Reproduction Assessment

Yes, at source level. Configure the Nostr plugin and start the gateway: current main still passes a Filter[] into nostr-tools 2.23.3's single-Filter API and the Nostr start task resolves into the generic restart path; no live relay was run in this read-only review.

## Solution Assessment

No as-is. The single-filter and long-lived account-task directions are right, but the branch needs current-main adaptation, real relay proof, and removal or strict validation of the raw media URL fallback.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Drop the Nostr media fallback:** `extensions/nostr/src/channel.ts:32`
  - body: The new sendMedia path advertises Nostr as media-capable and formats mediaUrl directly into DM text.
