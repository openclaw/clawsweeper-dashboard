## Summary

Keep open, but the patch is not merge-ready. Current main and the latest checked release still carry the Nostr subscription/lifecycle failures, while this PR needs real relay proof and should not add the raw media URL DM fallback as written.

## What This Changes

The PR changes Nostr to pass a single DM filter to `subscribeMany()`, keep gateway account startup pending until abort, add a `sendMedia` attachment-link DM fallback, and add focused tests.

## Best Possible Solution

Land a current-main Nostr fix that passes one Filter object, keeps `startNostrGatewayAccount` pending until abort with cleanup, relies on the generic text fallback or validates public media URLs only, and includes tests plus redacted relay proof.

## Reproduction Assessment

Yes. Source inspection on current main plus the `nostr-tools` 2.23.3 contract and the related relay logs give a high-confidence reproduction path for the malformed subscription filter and immediate clean-exit restart behavior, though I did not run a live relay here.

## Solution Assessment

No as-is. The single-filter and long-lived account-task directions are the right narrow fix, but the media fallback should be removed or constrained and the branch still needs redacted real relay proof.
