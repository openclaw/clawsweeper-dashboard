## Summary

Keep open: current main and the latest checked release still carry the Nostr DM subscription and startup-lifetime failures, but this PR is not ready as-is because it lacks real relay proof and introduces a raw media URL forwarding path.

## What This Changes

The PR changes the Nostr plugin to pass a single DM subscription filter, keep account startup pending until abort, add a sendMedia attachment-link fallback, and add focused tests.

## Best Possible Solution

Land a current-main Nostr fix that passes one Filter object, keeps `startNostrGatewayAccount` pending until abort with cleanup, relies on the generic text fallback or validates only public media URLs, and includes tests plus redacted relay proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure the Nostr plugin and start the gateway, and current main still passes a `Filter[]` where `nostr-tools` 2.23.3 expects one `Filter` while the start task resolves into the auto-restart supervisor path; no live relay was run in this read-only review.

## Solution Assessment

No as-is. The single-filter and long-lived account-task directions are the right narrow fix, but the media fallback should be removed or constrained and the branch still needs redacted real relay proof.
