## Summary

Keep open: this is a valid Nostr DM delivery fix attempt, but current main still has the filter and lifecycle failures, the branch is conflicted against current main, lacks real relay proof, and adds unsafe raw media URL forwarding.

## What This Changes

The branch changes the Nostr DM subscription filter, account lifetime handling, outbound media fallback, and regression tests for NIP-04 DM delivery.

## Best Possible Solution

Land a current-main Nostr fix that passes a plain Filter to subscribeMany, keeps startNostrGatewayAccount pending until abort with cleanup, relies on the existing text fallback or only allows validated public HTTP(S) media URLs, and adds focused regression coverage plus redacted relay proof.

## Reproduction Assessment

Yes, source-reproducible. Configure Nostr and start the gateway: current main still passes a Filter[] where nostr-tools 2.23.3 expects one Filter object, and the Nostr start task still resolves into the gateway restart path.

## Solution Assessment

No as-is. The filter and long-lived lifecycle changes are the right direction, but the patch needs current-main adaptation and should remove the raw media fallback or validate only safe public URLs.
