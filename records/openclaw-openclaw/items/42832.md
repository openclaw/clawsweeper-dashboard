## Summary

Keep open: current main still contains the Nostr DM subscription and lifecycle failures this PR addresses, so it is not obsolete, but the branch should not merge as-is because it lacks real relay proof and adds an unsafe raw media URL fallback.

## What This Changes

The branch changes Nostr DM subscription filtering, gateway startAccount lifetime, outbound sendMedia handling, and regression tests for NIP-04 DM delivery.

## Best Possible Solution

Land a current-main Nostr fix in gateway.ts and nostr-bus.ts that passes a plain Filter, keeps the gateway task pending until abort with cleanup, relies on core text fallback or validates public HTTPS media URLs, and includes focused tests plus redacted relay proof.

## Reproduction Assessment

Yes. Source inspection on current main shows the Nostr plugin still passes Filter[] where nostr-tools 2.23.3 expects one Filter object, and the gateway supervisor still restarts when startAccount resolves immediately.

## Solution Assessment

No as-is. The filter and lifecycle changes are the right direction, but the patch should be rebased to the current gateway.ts surface and should remove the raw media fallback or restrict it to validated public HTTPS URLs.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Reject non-public media URLs in the Nostr fallback:** `extensions/nostr/src/channel.ts:164-169`
  - body: The new sendMedia adapter copies mediaUrl directly into DM text. Outbound media contexts can contain local paths, file:// URLs, data URLs, or internal-only references, so this can leak host details and send unusable private references to Nostr recipients; rely on the core text fallback or allow only validated public HTTPS URLs.
