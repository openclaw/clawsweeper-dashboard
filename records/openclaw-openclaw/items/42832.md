The PR changes the Nostr plugin to pass a single subscription filter, keep account startup pending until abort, add a `sendMedia` URL-to-text fallback, and cover those paths with unit tests.

## Best Possible Solution

Land a current-main Nostr fix in `extensions/nostr/src/gateway.ts` and `extensions/nostr/src/nostr-bus.ts` that uses one `Filter`, keeps the account task pending until abort with cleanup, leaves Nostr text-only unless media policy is designed, and includes focused tests plus redacted relay proof.
