## Summary

Keep open: current main still has the Nostr DM subscription-shape and gateway-lifetime bug paths, so the PR remains useful. It is not mergeable as-is because contributor real relay proof is missing and the added media fallback can forward raw private media references into DMs.

## What This Changes

The PR changes the Nostr channel’s DM subscription filter, keeps gateway account startup pending until abort, adds a media-to-text DM fallback, and adds focused tests.

## Best Possible Solution

Land a current-main Nostr fix that passes one Filter object, keeps startNostrGatewayAccount pending until abort with cleanup, avoids raw media URL forwarding, and includes focused regression tests plus redacted relay proof.

## Reproduction Assessment

Yes. Source inspection against current main, nostr-tools 2.23.3, NIP-01, and the related relay logs gives a high-confidence reproduction path for the malformed subscription filter and immediate clean-exit restart behavior, though no live relay was run here.

## Solution Assessment

No as-is. The single-filter and long-lived account-task directions are the right narrow fix, but the media fallback should be removed or limited to validated public URLs and the branch still needs real relay proof.
