Full review comments:

- **[P1] Export the runtime setter the entry loads:** `extensions/channel-broker/index.ts:29-30`
  - body: `index.ts` tells the bundled loader to import `setChannelBrokerRuntime` from `./runtime-api.js`, but `runtime-api.ts` never exports that symbol and the new guardrail locks the missing export in. Runtime injection for this channel will fail instead of wiring the broker runtime; export the setter from `./src/runtime.js` and update the guardrail.
  - confidence: 0.95
- **[P1] Use pre-auth limits before broker signature verification:** `extensions/channel-broker/src/http-routes.ts:83`
  - body: The inbound handler reads the body with `profile: "post-auth"` before verifying `x-openclaw-broker-signature`. The shared guard defaults make post-auth reads much larger and longer-lived than pre-auth reads, so unauthenticated requests can consume the larger budget before HMAC rejection; use pre-auth or explicit small limits until the signature is accepted.
  - confidence: 0.92
- **[P2] Validate provider receipts before casting them:** `extensions/channel-broker/src/runtime.ts:70`
  - body: `parseBrokerReceipt` only checks that the JSON response is an object and then casts it to `BrokerReceiptV1`. The send path immediately assumes fields such as `status` and `messageIds` are present and well-typed, so malformed provider responses can throw TypeErrors or produce invalid receipt metadata; validate the receipt shape before returning it.
  - confidence: 0.88
