Overall correctness: patch is incorrect

Full review comments:

- **[P1] Export the runtime setter the entry loads:** `extensions/channel-broker/index.ts:30`
  - body: `index.ts` advertises `setChannelBrokerRuntime` from `./runtime-api.js`, but that file never exports the setter. The bundled entry loader throws on missing named exports, so the new channel cannot receive runtime injection until the setter is exported or the entry points at a narrow setter module.
  - confidence: 0.97
- **[P1] Use pre-auth guards before broker signature verification:** `extensions/channel-broker/src/http-routes.ts:83`
  - body: This plugin-managed webhook reads the unsigned request body with the post-auth 1 MiB/30s profile before checking the HMAC signature. Use the pre-auth body profile plus the shared method/content-type/rate/in-flight guards before parsing or signature verification, as other signed webhook routes do.
  - confidence: 0.9
- **[P2] Validate provider receipts before casting them:** `extensions/channel-broker/src/runtime.ts:70`
  - body: `parseBrokerReceipt` only checks that the provider response is an object and then casts it to `BrokerReceiptV1`. A malformed 200 response can still reach durable receipt creation with missing `messageIds`, wrong version, or invalid status, so validate the protocol fields before returning the receipt.
  - confidence: 0.89
