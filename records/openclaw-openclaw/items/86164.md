Full review comments:

- **[P2] Normalize built-in aliases on inbound events:** `extensions/channel-broker/src/http-routes.ts:79-80`
  - body: The Phase 3 docs and outbound path treat aliases like `teams`, `googlechat`, and `qq` as canonical platform aliases, but the inbound webhook path only applies custom `platformAliases` before checking `account.platforms`. A signed Teams event with `platform: "teams"` will be rejected for an account configured with `platforms: ["microsoft-teams"]`, so the official alias matrix is one-sided for inbound delivery.
  - confidence: 0.87
- **[P2] Put inbound webhooks behind the shared guard pipeline:** `extensions/channel-broker/src/http-routes.ts:112-117`
  - body: This new public inbound route reads and parses every POST body before any rate-limit, content-type, or in-flight guard runs. Existing webhook helpers provide `beginWebhookRequestPipelineOrReject`/`applyBasicWebhookRequestGuards` for that pre-body-read protection, so invalid-signature or malformed request floods can spend CPU and request slots on this new route instead of being bounded consistently.
  - confidence: 0.82
