Full review comments:

- **[P2] Normalize built-in aliases on inbound events:** `extensions/channel-broker/src/http-routes.ts:79-81`
  - body: `accounts.ts` canonicalizes configured platform lists/capabilities through the built-in alias map, but inbound events only check user-supplied `platformAliases` and then compare the raw event platform. A provider configured with `platforms: ["teams"]` is stored as `microsoft-teams`, so a signed inbound event with `platform: "teams"` is rejected as `unsupported_platform` instead of reaching the broker runtime with the same canonical platform that outbound uses.
  - confidence: 0.91
- **[P2] Start the shared webhook guard before reading inbound bodies:** `extensions/channel-broker/src/http-routes.ts:112-117`
  - body: The new public broker inbound route reads up to the pre-auth body limit, parses JSON, resolves the provider, and computes HMAC without first entering the shared webhook request pipeline or an in-flight/rate limiter. Existing webhook routes use those guards so invalid traffic is capped before expensive work; this route should do the same and release the guard in a `finally` block.
  - confidence: 0.86
