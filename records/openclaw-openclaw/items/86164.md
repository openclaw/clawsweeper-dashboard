## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Export the runtime setter the entry loads:** `extensions/channel-broker/runtime-api.ts:15-18`
  - body: `index.ts` declares `runtime.exportName: "setChannelBrokerRuntime"`, but this barrel never exports that symbol. The bundled loader injects `api.runtime` through the configured runtime entry, so real Channel Broker registration will not call `setChannelBrokerRuntime` and inbound/custom runtime paths stay unconfigured.
  - confidence: 0.95
- **[P1] Use pre-auth limits before signature verification:** `extensions/channel-broker/src/http-routes.ts:83`
  - body: This route reads the raw request body with the larger post-auth limits before validating the broker signature. Since `auth: "plugin"` makes the plugin own this auth boundary, unsigned requests should use the pre-auth body profile before any JSON parsing or signature verification work.
  - confidence: 0.89
- **[P2] Validate provider receipts before casting them:** `extensions/channel-broker/src/runtime.ts:70`
  - body: `parseBrokerReceipt` accepts any object as `BrokerReceiptV1`, so an external provider can return a malformed `sent` receipt without `messageIds` or with mismatched fields and fail later with unclear behavior. Validate the receipt shape at the HTTP boundary before outbound send handling consumes it.
  - confidence: 0.84
