## Summary

Keep this open. Current main still has narrower session metadata, grouped session selection, single-session chat sends, and checkpoint-only branching; it does not implement the requested Control UI project tree, composer routing/fanout, selected-message split/move triage, or seed-elsewhere workflow, and the recent issue comments explicitly kept the broader feature open.

## What This Changes

Request adding Control UI project-based session trees, threaded composer routing or fanout, selected-message split/move triage, and seed/copy branching across sessions.

## Best Possible Solution

Design the broad Control UI feature explicitly: additive project/session-tree metadata, a tree navigator, composer route targets or fanout, selected-message split/move semantics, seed/copy branch behavior, and matching Gateway protocol, UI, docs, and tests while keeping #55349 separate.

## Solution Assessment
