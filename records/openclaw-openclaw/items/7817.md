## Summary

Keep this open. Current main still has only narrower session metadata, grouped session selection, single-session chat sends, display-only reply pills, slash-command steering, and checkpoint-scoped branching; it does not implement the requested project tree, composer fanout/routing, selected-message split/move triage, or seed-elsewhere workflow.

## What This Changes

The issue requests project-based Control UI session trees, threaded composer routing/fanout, message split/move triage, and seed-elsewhere branching.

## Best Possible Solution

Design and implement the broad Control UI workflow with additive project/session-tree metadata, a tree navigator, composer route targets or fanout, selected-message split/move semantics, seed/copy branch behavior, and matching Gateway protocol, UI, docs, and tests.

## Reproduction Assessment

Not applicable as a feature request. Source and docs inspection on current main gives high confidence that the requested project tree, fanout routing, selected-message split/move, and seed-elsewhere flows are absent.

## Solution Assessment

Unclear until maintainers choose the product and protocol semantics for split, move, copy, and fanout. Reusing parentSessionKey, checkpoint branching, slash-command steering, and reply-pill metadata is sensible, but those primitives do not solve the whole request.
