## Summary

Keep this open. Current main still has only narrower session primitives, while the requested Control UI project tree, composer routing/fanout, selected-message split/move triage, and seed-elsewhere workflow are not implemented; the existing maintainer/Codex comment also explicitly kept the broader product request open and distinguished #55349 as narrower quote-reply UX.

## What This Changes

Add project-based session trees, threaded composer entry points, selected-message split/move triage, and seed/copy branching workflows to the Control UI.

## Best Possible Solution

Keep this as the broad Control UI product feature. A complete design should add additive project/session-tree metadata, a tree navigator, composer route targets or fanout, selected-message split/move transcript semantics, seed/copy branch behavior, and focused Gateway protocol, server, UI, tests, and docs coverage while keeping #55349 separate for quote-reply UX.
