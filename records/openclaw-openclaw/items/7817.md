## Summary

Keep open. Current main still lacks the requested Control UI project tree, composer routing/fanout, selected-message split/move triage, and seed-elsewhere workflow; existing session select, parent/checkpoint branching, and reply primitives are useful but narrower than this product feature.

## Best Possible Solution

Define additive session/project/tree metadata and transcript branch/split semantics first, then land Gateway protocol/storage and Control UI tree/routing/split/seed slices while coordinating narrower reply metadata work separately.

## Solution Assessment

Unclear as one immediate patch. The maintainable path is staged product/API design that reuses parentSessionKey, childSessions, checkpoint branching, and reply primitives without overloading the current flat chat.send/session contracts.
