## Summary

Current main still provides session selection, single-session chat sends, parent/child session metadata, and checkpoint branching, but it does not implement project trees, composer fanout, selected-message split/move, or seed-to-another-branch behavior. The related open reports overlap without fully replacing this issue: https://github.com/openclaw/openclaw/issues/13676 focuses on project/workspace and runtime constraints, https://github.com/openclaw/openclaw/issues/27526 adds a much broader multi-AI collaboration program, and https://github.com/openclaw/openclaw/issues/38364 concerns reply metadata and completion notices. This remains useful as a distinct product tracker, but it needs maintainer decisions on transcript semantics and decomposition before implementation.

## What This Changes

Add project-organized session trees, routed or fanned-out chat entry points, message split/move triage, and branch seeding in the Control UI.

## Best Possible Solution

Keep this as the focused tracker for project-tree navigation and transcript branching, define additive canonical project and branch metadata plus explicit copy/move semantics, and deliver separate reviewed slices for tree navigation, composer targets, and selected-message branching while linking the broader project and collaboration trackers.

## Reproduction Assessment

Not applicable; this issue requests new project, routing, and transcript-management capabilities rather than reporting a failure of an existing documented contract.

## Solution Assessment

No, not as one bundled implementation. The product direction is useful, but project organization, composer routing, and transcript split/copy semantics need explicit contracts and focused delivery slices to avoid competing state models.
