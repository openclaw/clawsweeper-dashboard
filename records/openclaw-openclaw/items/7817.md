## Summary

Keep this open. Current main still has only narrower session metadata, grouped session selection, single-session chat sends, and checkpoint-scoped branching; it does not implement the requested Control UI project tree, composer routing/fanout, selected-message split/move triage, or seed-elsewhere workflow, and the recent comments already distinguish closed #55349 as a narrower quote-reply request.

## What This Changes

Request project-based Control UI session trees, threaded composer routing, selected-message split/move triage, and seed/copy-to-branch workflows.

## Best Possible Solution

Design and implement the broad Control UI workflow with additive project/session-tree metadata, a tree navigator, composer route targets or fanout, selected-message split/move semantics, seed/copy branch behavior, and matching Gateway protocol, UI, docs, and tests.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a feature request; source and docs inspection on current main verifies that project trees, fanout routing, selected-message split/move, and seed-elsewhere flows are absent.

## Solution Assessment

Is this the best way to solve the issue? Unclear until maintainers choose the split/move/copy/fanout semantics; reusing parentSessionKey and checkpoint branching is useful but not sufficient for the requested workflow.
