## Summary

Current main still lacks the central project-tree, routed/fanout composer, selected-message split/move, and seed/copy workflows. Related open issues overlap but do not safely supersede this scope, while the only focused implementation PR was closed unmerged and is not a canonical landing path. The item should remain open for a maintainer product decision because it combines several distinct session, protocol, persistence, and Control UI contracts.

## What This Changes

Requests project-based session trees, routed or fanned-out composer targets, message split/move triage, and context seeding between Control UI threads.

## Best Possible Solution

Split the product design into a lean session-tree and branch/copy UX contract, a separately decided transcript split/move model, and the broader project/workspace isolation model tracked at https://github.com/openclaw/openclaw/issues/13676; implement only after maintainers choose the canonical boundaries and additive Gateway protocol shapes.
