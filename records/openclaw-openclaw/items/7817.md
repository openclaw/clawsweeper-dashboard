## Summary

Current main still provides only narrower session primitives: agent-grouped session selection, one-session chat sends, optional parent linkage, and single-session send/subscribe APIs. It does not implement project trees, selected-message split or move, seed/copy branching, or composer fanout, and the related closed PR is not a viable replacement; this remains a broad product decision rather than an autonomous fix.

## What This Changes

Add project-organized session trees, routed or fanout chat entry points, message split/move triage, and branch seeding in the Control UI.

## Best Possible Solution

Define an additive canonical project/session metadata model, implement non-destructive branch-copy semantics before any move workflow, and add explicit per-target routing results before building the project-tree and triage UI in bounded phases.

## Reproduction Assessment

Not applicable; this issue requests new Control UI, protocol, and transcript-management capabilities rather than reporting broken defined behavior.

## Solution Assessment

No, not as “primarily UI work”; the maintainable solution must first define Gateway-level project, branch-copy or move, fanout, failure, and idempotency semantics, then layer the UI on those contracts.
