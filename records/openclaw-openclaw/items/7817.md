## Summary

Keep open: current main still provides grouped session selection, single-target chat sends, checkpoint branching, and display-only reply pills, but it does not implement project trees, composer routing/fanout, selected-message split/move triage, or seed-elsewhere workflows.

## What This Changes

Request to organize Control UI sessions into project trees and add threaded routing, message split/move triage, and seed-elsewhere branching workflows.

## Best Possible Solution

Keep this as a broad Control UI feature: define additive session/project metadata and transcript split/branch semantics, then implement tree navigation, composer route targets, selected-message triage, seed/copy workflows, and focused protocol/UI/docs coverage.
