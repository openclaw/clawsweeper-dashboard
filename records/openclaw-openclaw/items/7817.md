## Summary

Current main still provides only narrower building blocks—grouped session selection, parent/child linkage, checkpoint branching, and session shortcuts—while project hierarchy, routed or fanout composition, arbitrary message split/move, and cross-project seeding remain undefined. The open pinned-chat PR improves quick access and parallel-session creation but does not own the broader product model, so this issue remains necessary and needs maintainer product design rather than an autonomous fix.

## What This Changes

Add project-organized session trees, thread-targeted or fanout message entry, selected-message split/move triage, and cross-project context seeding to the Control UI.

## Best Possible Solution

Define one canonical additive project/workstream model and immutable branch-or-copy semantics at the Gateway/session boundary, then build the Control UI tree and targeted composer on that contract while keeping pinned shortcuts and quote replies as narrower complementary features.

## Reproduction Assessment

Not applicable: this requests new product capability rather than reporting a broken established contract. Current-main source and docs provide a high-confidence baseline showing only narrower session and checkpoint primitives.

## Solution Assessment

No, the issue’s suggestion that this is primarily UI work is not the best implementation boundary. The maintainable path starts with explicit additive Gateway and transcript semantics, then adds the tree, routing, and triage UI.
