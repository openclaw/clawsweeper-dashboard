## Summary

Keep open. Current main still has only narrower primitives: grouped session selection, single-session chat sends, parent/child session metadata, checkpoint-scoped branching, and display-only reply pills. It does not implement the requested Control UI project tree, composer routing/fanout, selected-message split/move triage, or seed-elsewhere workflow, and the existing maintainer/Codex discussion also kept this broad feature open.

## What This Changes

Request project-based hierarchical session organization, threaded composer routing, selected-message split/move triage, and branch/seed workflows in the Control UI.

## Best Possible Solution

Design an additive Control UI/Gateway feature covering project/session-tree metadata, a tree navigator, composer route targets or fanout, selected-message split/move semantics, seed/copy branch behavior, docs, and focused UI/protocol/server tests.

## Solution Assessment
