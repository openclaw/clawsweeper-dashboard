## Summary

Keep open. Current main still has the reported macOS fallback gap: local-mode saves fall back from the Gateway RPC to `OpenClawConfigFile.saveDict`, and that writer currently stamps and persists the incoming dictionary without restoring `__OPENCLAW_REDACTED__`. The PR targets the right boundary and appears functionally aligned with the existing Gateway restore contract, but it needs the required changelog entry before merge.

## What This Changes

The PR updates the macOS config form and local config-file save path to preserve redacted config sentinels after a Gateway save fallback, adds Swift regression tests, and documents the write-side redaction contract.

## Best Possible Solution

Land a version of this PR that keeps the restore logic in the macOS local config writer, preserves the fallback regression coverage, documents the Gateway/macOS contract, and adds a single-line `Unreleased` changelog fix entry with appropriate contributor credit.

## Reproduction Assessment

Yes. On current main, a redacted config snapshot can be passed to `ConfigStore.save`; if the Gateway save fails in local mode, the fallback calls `OpenClawConfigFile.saveDict` and persists the incoming dictionary without restoration.

## Solution Assessment

Yes, with the changelog added. Fixing the macOS local file writer is the narrow maintainable boundary because it protects both direct macOS saves and the degraded Gateway fallback while leaving the existing Gateway restore path intact.
