## Summary

Keep this PR open. The main macOS fallback fix targets a real current-main secret-clobber path, but the proposed Swift restore is still looser than the Gateway contract for structured SecretRefs, and the prior ClawSweeper changelog finding remains valid.

## What This Changes

The PR updates the macOS config form and local config-file fallback to handle redacted config sentinels, adds Swift regression coverage, and documents the Gateway/macOS write-side redaction restore contract.

## Best Possible Solution

Land a version that keeps the macOS local writer protection, but mirrors the Gateway SecretRef restore semantics before writing: restore redacted ids only when the original SecretRef has the same source/provider, reject mixed-source/provider submissions, keep the fallback regression tests, and add the required single-line `Unreleased` changelog fix entry.

## Reproduction Assessment

Yes. On current main, a redacted config snapshot can reach `ConfigStore.save`; when the Gateway save fails in local mode, the fallback writes that dictionary through `OpenClawConfigFile.saveDict` without redaction restoration.

## Solution Assessment

No, not yet. The fix belongs in the macOS local config writer, but the Swift restore should match the Gateway SecretRef guard before this is the safest implementation.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Reject SecretRef source/provider changes:** `apps/macos/Sources/OpenClaw/OpenClawConfigFile.swift:647-648`
  - body: The restore helper handles every sentinel string independently, so an incoming SecretRef can change `source` or `provider` while leaving `id` as `__OPENCLAW_REDACTED__`; this line restores the old id and keeps the changed source/provider. Gateway restore rejects that exact mixed state, so the macOS fallback can save a SecretRef that points a preserved id at a different provider/source. Detect SecretRef dictionaries and fail unless source/provider match before restoring the id.
