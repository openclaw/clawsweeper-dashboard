## Summary

Keep open, but the branch is not merge-ready: current main still has the reported tailnet/custom bind mismatch at source level, while this PR's client-side host fix overlaps the canonical listener-side PR and its auth hint gives misleading advice for non-token modes.

## What This Changes

The PR changes Gateway CLI connection setup to derive local WebSocket host from `gateway.bind` and adds an auth hint for reasonless gateway close errors when credentials are missing.

## Best Possible Solution

Resolve the bind failure through one canonical Gateway locality design, preferably the listener-side loopback approach already under review, and keep any CLI close-error hint mode-specific with focused tests and redacted real tailnet/custom plus auth-failure proof.

## Reproduction Assessment

## Solution Assessment

No. The branch's client-side host change conflicts with current tests that preserve loopback callers and overlaps the canonical listener-side fix, while the auth hint needs mode-specific wording before it is a maintainable diagnostic.
