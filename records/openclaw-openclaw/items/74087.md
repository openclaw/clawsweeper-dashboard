## Summary

Keep open. Current main and the latest v2026.5.7 release cover runtime compile-cache use and the public `OPENCLAW_NO_RESPAWN` escape hatch, but they still do not provide the requested first-class cache prewarm command, shipped cache artifact, or postinstall warm path; that remaining work needs maintainer product and package-policy direction.

## What This Changes

Request to add a first-class way to prewarm OpenClaw's Node compile cache for cold container startup and a public opt-in to skip CLI self-respawn in controlled environments.

## Best Possible Solution

If accepted, add an explicit operator-run `openclaw warm-cache` flow with Node/version/arch-scoped output, container-build docs, and focused tests; defer automatic postinstall warming or bundled cache artifacts until maintainers approve that policy.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high-confidence proof of the remaining gap: runtime cache and no-respawn support exist, but no first-class prewarm command, shipped artifact, or postinstall warm path exists on current main.

## Solution Assessment

Unclear. `OPENCLAW_NO_RESPAWN` is a narrow maintainable answer for the respawn half, but the compile-cache prewarm half still needs a maintainer choice between an explicit command, postinstall warming, or shipped artifacts.
