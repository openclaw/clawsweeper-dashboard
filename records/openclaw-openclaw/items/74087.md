## Summary

Keep open. Current main covers runtime compile-cache enablement, cache scoping, docs/doctor hints, and the public `OPENCLAW_NO_RESPAWN` escape hatch, but it still does not provide a first-class prewarm command, shipped cache artifact, or install-time warm path; that remaining part is a product/package policy choice.

## What This Changes

The issue requests a prewarmed Node compile-cache flow plus a public opt-in to skip CLI self-respawn for faster stateless-container cold starts.

## Best Possible Solution

If accepted, add an explicit operator-run `openclaw warm-cache` flow with Node/version/arch-scoped output, container-build docs, and focused tests; defer automatic postinstall warming or bundled cache artifacts until maintainers approve that packaging policy.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high-confidence proof of the remaining gap: current main has runtime compile-cache and no-respawn support, but no first-class prewarm command, shipped cache artifact, or postinstall warm path.

## Solution Assessment
