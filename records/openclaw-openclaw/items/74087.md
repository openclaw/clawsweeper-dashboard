## Summary

Keep open. Current main already has runtime compile-cache enablement and the public `OPENCLAW_NO_RESPAWN` escape hatch, but it still lacks the requested first-class prewarm/shipped-cache surface, which needs maintainer product and supply-chain direction.

## What This Changes

The issue requests a first-class way to prewarm and ship OpenClaw's Node compile cache for cold starts plus an opt-in escape hatch that skips CLI self-respawn.

## Best Possible Solution

If accepted, add an explicit operator-run `openclaw warm-cache` flow with cache scoping/invalidation tests and container/server docs, while deferring postinstall or bundled cache artifacts until maintainers approve that policy.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/optimization request. Source inspection gives high-confidence proof of the remaining gap: runtime compile-cache and no-respawn support exist, but no first-class prewarm command, shipped cache artifact, or postinstall warm path exists on current main.

## Solution Assessment

Unclear. `OPENCLAW_NO_RESPAWN` is a narrow maintainable answer for the respawn part, but maintainers still need to choose whether compile-cache prewarming belongs in an explicit command, postinstall, or shipped artifact workflow.
