## Summary

Keep open. Current main already covers runtime Node compile-cache enablement and the public `OPENCLAW_NO_RESPAWN` escape hatch, but it still does not provide the requested pre-warmed cache command, shipped cache artifact, or postinstall warm path; that remaining package/runtime policy needs maintainer direction.

## What This Changes

Request to improve container cold starts by letting operators prewarm or ship Node compile cache artifacts and opt out of CLI self-respawn in controlled environments.

## Best Possible Solution

If accepted, add an explicit operator-run `openclaw warm-cache` flow with Node/version/arch/install-scoped output and container-build docs; defer automatic postinstall warming or bundled cache artifacts until maintainers approve that package policy.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/performance request. Source inspection on current main gives high-confidence proof of the remaining gap: runtime cache and no-respawn support exist, but no prewarm command, shipped artifact, or postinstall warm path is registered.

## Solution Assessment

Unclear. `OPENCLAW_NO_RESPAWN` is already a narrow maintainable answer for the respawn half, but the compile-cache prewarm half needs a maintainer choice between an explicit command, postinstall warming, or shipped artifacts.
