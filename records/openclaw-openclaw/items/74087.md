## Summary

Keep open. Current main already has automatic packaged runtime compile-cache support and a public `OPENCLAW_NO_RESPAWN` escape hatch, but it still lacks the requested first-class `openclaw warm-cache` command, shipped prewarmed cache artifact, or postinstall cache-warming path; that remaining surface needs maintainer product and package-policy review.

## What This Changes

Request to add a pre-warmed Node compile-cache flow and a public self-respawn bypass for faster cold starts in stateless container deployments.

## Best Possible Solution

If accepted, add an explicit operator-run cache prewarm flow with version/arch-scoped output, container-build docs, and regression coverage; defer automatic postinstall warming or bundled artifacts until maintainers approve that policy.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high-confidence proof of the remaining gap: current main has runtime compile-cache and no-respawn support, but no first-class prewarm command, shipped cache artifact, or postinstall warm path.
