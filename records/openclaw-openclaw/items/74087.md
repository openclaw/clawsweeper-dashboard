## Summary

Keep open. Current main already covers automatic runtime compile-cache enablement and the public no-respawn escape hatch, but it still lacks the requested first-class prewarm command, shipped cache artifact, or postinstall warming path; that remaining surface needs maintainer product and packaging direction.

## What This Changes

Request to add first-class cold-start controls: a prewarmed Node compile-cache flow or artifact and a public opt-in to bypass CLI self-respawn.

## Best Possible Solution

If accepted, add an explicit operator-run `openclaw warm-cache` flow with version/install-scoped output, container-build docs, and focused tests; leave automatic postinstall warming or shipped artifacts behind maintainer approval.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives a high-confidence gap: runtime compile-cache and no-respawn support exist, but no first-class prewarm command, shipped artifact, or postinstall warm path is present.

## Solution Assessment

Unclear. `OPENCLAW_NO_RESPAWN` is a narrow maintainable answer for the respawn half, but the compile-cache prewarm half still needs a maintainer choice between an explicit command, postinstall warming, or package artifacts.
