## What This Changes

## Best Possible Solution

Keep the existing env-based compile-cache and OPENCLAW_NO_RESPAWN behavior, then decide whether to add an explicit operator command such as openclaw warm-cache with focused tests and Linux/container docs. Avoid automatic postinstall priming unless maintainers explicitly accept the install-time execution and cache invalidation tradeoffs.
