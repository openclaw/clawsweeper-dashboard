## Summary

Keep open. Current main implements the env-based compile-cache and no-respawn controls, but it still lacks the requested first-class pre-warm command, build artifact, or postinstall priming path. The remaining design has product and supply-chain tradeoffs around install-time execution and optional plugin imports, so it should not be auto-closed or queued as a narrow repair yet.

## What This Changes

The issue requests a first-class way to pre-warm OpenClaw's Node compile cache and an opt-in path to skip the startup self-respawn in controlled container environments.

## Best Possible Solution

Keep the shipped env-based cache and respawn controls, then decide whether to add an explicit operator-run `openclaw warm-cache` command with focused tests and Linux/container docs. Prefer an explicit command over automatic postinstall priming unless maintainers intentionally accept install-time execution and cache-invalidation behavior.
