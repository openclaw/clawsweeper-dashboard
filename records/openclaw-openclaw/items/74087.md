## Summary

Keep open. Current main already has runtime compile-cache support and a public `OPENCLAW_NO_RESPAWN` escape hatch, but the requested first-class prewarming or shipped-cache surface is still missing and needs maintainer policy direction.

## What This Changes

Request to add cold-start prewarming for Node compile cache and a public way to bypass CLI self-respawn in trusted container environments.

## Best Possible Solution

If accepted, add an explicit operator-run `openclaw warm-cache` command with cache scoping, invalidation tests, and container/server docs; defer postinstall or shipped-cache behavior until supply-chain policy is settled.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high-confidence proof of the remaining gap: runtime cache and no-respawn support exist, but no first-class prewarm command, shipped cache artifact, or postinstall warm path exists on current main.

## Solution Assessment

Unclear. The existing `OPENCLAW_NO_RESPAWN` path is a narrow maintainable solution for respawn overhead, but maintainers still need to choose the compile-cache prewarming surface and supply-chain posture.
