## Summary

Keep open. Current main covers runtime Node compile-cache enablement and the public no-respawn path, but it still does not provide the requested first-class cache prewarming command, shipped cache artifact, or automatic postinstall warm path.

## What This Changes

Request to reduce packaged gateway cold starts by prewarming Node's compile cache and exposing an operator-controlled way to skip CLI self-respawn.

## Best Possible Solution

If maintainers accept the feature, prefer an explicit operator-run `openclaw warm-cache` command with cache scoping/invalidation tests and container/server docs; defer postinstall or shipped-cache behavior until supply-chain policy is settled.

## Reproduction Assessment

Yes for the remaining repository gap: current source, docs, tests, and command catalogs show compile-cache and no-respawn support but no first-class compile-cache prewarming command, postinstall warm path, or shipped cache artifact. The exact performance gain still needs a cold-start benchmark on a current packaged build.

## Solution Assessment

Unclear. The existing env knobs are narrow and maintainable, but choosing between an explicit command, postinstall prewarming, or shipped artifacts is a product and supply-chain decision rather than a single obvious repair.
