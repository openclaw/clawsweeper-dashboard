## Summary

Keep open. Current main covers automatic runtime compile-cache enablement and the public `OPENCLAW_NO_RESPAWN` escape hatch, but it still lacks the requested first-class prewarm command, shipped cache artifact, or postinstall warm path; that remaining surface needs maintainer product and package-supply-chain direction.

## What This Changes

Add a way to prewarm or ship OpenClaw's Node compile cache for cold starts and expose a public opt-in to skip CLI self-respawn.

## Best Possible Solution

If accepted, add an explicit operator-run `openclaw warm-cache` flow with Node/version/arch-scoped output, docs for container image builds, and tests for cache directory selection; defer automatic postinstall or bundled cache artifacts until maintainers approve that policy.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high-confidence proof of the remaining gap: current main has runtime compile-cache and no-respawn support, but no first-class prewarm command, shipped cache artifact, or postinstall warm path.

## Solution Assessment

Unclear. `OPENCLAW_NO_RESPAWN` is a narrow maintainable answer for the respawn half, but the compile-cache prewarm half still needs a maintainer choice between an explicit command, postinstall warming, or shipped artifacts.
