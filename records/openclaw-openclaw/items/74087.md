## Summary

Keep open. Current main has runtime compile-cache support and a public no-respawn env knob, but the first-class prewarming/shipped-cache part of the request is still missing and needs maintainer product and supply-chain direction.

## What This Changes

Requests a first-class way to prewarm or ship Node compile caches for cold starts, plus a public opt-in to skip CLI self-respawn in controlled container environments.

## Best Possible Solution

If accepted, prefer an explicit operator-run openclaw warm-cache command with cache scoping, invalidation tests, and container/server docs; defer postinstall or shipped-cache behavior until supply-chain policy is settled.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection does give high-confidence proof of the remaining gap: runtime compile-cache and no-respawn support exist, but no first-class prewarm command, shipped cache artifact, or postinstall warm path exists on current main.

## Solution Assessment

Unclear. The existing OPENCLAW_NO_RESPAWN path is a narrow maintainable solution for respawn overhead, but maintainers still need to choose between an explicit command, postinstall prewarming, and shipped artifacts for compile-cache prewarming.
