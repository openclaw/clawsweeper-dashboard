## Summary

Keep open. Current main partially addresses the report with runtime Node compile-cache enablement and a public no-respawn env var, but the requested first-class cache prewarming command, shipped cache artifact, or automatic postinstall warm path is still missing.

## What This Changes

The issue requests cold-start improvements by adding a prewarmed Node compile-cache path and a public opt-in to skip CLI self-respawn for controlled container deployments.

## Best Possible Solution

If maintainers accept the feature, prefer an explicit operator-run openclaw warm-cache command with cache scoping/invalidation tests and VPS/container docs; defer postinstall or shipped-cache behavior until supply-chain policy is settled.
