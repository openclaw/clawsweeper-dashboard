## Summary

Keep open. Current main already covers runtime compile-cache enablement and the public no-respawn escape hatch, but the first-class prewarm/shipped-cache part of the request is still missing and needs maintainer product and supply-chain direction.

## What This Changes

Request a first-class cold-start path that can prewarm or ship Node compile caches and an operator opt-in to bypass CLI self-respawn.

## Best Possible Solution

If accepted, add an explicit operator-run `openclaw warm-cache` implementation with scoped cache directory/invalidation tests and container/server docs; defer postinstall or bundled cache artifacts until maintainers approve that policy.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high-confidence proof of the remaining gap: runtime compile-cache and no-respawn support exist, but no first-class prewarm command, shipped cache artifact, or postinstall warm path exists on current main.

## Solution Assessment

Unclear. The existing `OPENCLAW_NO_RESPAWN` path is a narrow maintainable solution for respawn overhead, but maintainers still need to choose between an explicit command, postinstall prewarming, and shipped artifacts for compile-cache prewarming.
