## Summary

Keep open. Current main already has runtime Node compile-cache support and a public no-respawn env knob, but it still does not provide the requested first-class prewarm command, shipped cache artifact, or postinstall warming path.

## What This Changes

The issue requests first-class cold-start optimization through pre-warmed Node compile-cache support and a supported opt-in to skip CLI self-respawn.

## Best Possible Solution

If accepted, prefer an explicit operator-run `openclaw warm-cache` command with cache scoping, invalidation coverage, and container/server docs; defer automatic postinstall or shipped-cache behavior until maintainers settle the supply-chain policy.
