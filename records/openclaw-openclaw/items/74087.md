## Summary

Keep open. Current main already has runtime Node compile-cache enablement and the public `OPENCLAW_NO_RESPAWN` escape hatch, but it still lacks the requested first-class prewarm command, shipped cache artifact, or postinstall warm path; that remaining decision belongs with maintainers because it changes CLI and package-install behavior.

## What This Changes

Request to add a first-class compile-cache prewarm/shipping path and a public opt-in to skip CLI self-respawn for container cold starts.

## Best Possible Solution

If accepted, add an explicit operator-run `openclaw warm-cache` flow with Node/version/arch-scoped output, container-build docs, and cache-directory tests; defer automatic postinstall warming or shipped artifacts until maintainers approve that policy.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/performance request. Source inspection gives high-confidence proof that current main lacks a first-class prewarm or shipped-cache path while already covering runtime cache enablement and `OPENCLAW_NO_RESPAWN`.

## Solution Assessment

Unclear. `OPENCLAW_NO_RESPAWN` is already the narrow maintainable answer for the respawn half, but the compile-cache half still needs a maintainer choice between an explicit command, postinstall warming, or shipped artifacts.
