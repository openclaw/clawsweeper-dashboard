## Summary

Keep open. Current main covers the reusable Node compile-cache path and the public `OPENCLAW_NO_RESPAWN` opt-in, but it still lacks the requested first-class cache prewarming surface, shipped cache artifact, or automatic postinstall warm path.

## What This Changes

Add cold-start prewarming for OpenClaw's Node compile cache and provide a supported no-respawn opt-in for controlled container deployments.

## Best Possible Solution

If maintainers accept the feature, prefer an explicit operator-run `openclaw warm-cache` command with cache scoping/invalidation tests and server/container docs; defer postinstall or shipped-cache behavior until supply-chain policy is settled.

## Reproduction Assessment

Yes for the remaining repository gap: current source, docs, tests, and command catalogs show compile-cache and no-respawn support but no first-class compile-cache prewarming command, postinstall warm path, or shipped cache artifact. The exact performance gain still needs a cold-start benchmark on a current packaged build.

## Solution Assessment

Unclear. The existing env knobs are narrow and maintainable, but choosing between an explicit command, postinstall prewarming, or shipped artifacts is a product and supply-chain decision rather than a single obvious repair.
