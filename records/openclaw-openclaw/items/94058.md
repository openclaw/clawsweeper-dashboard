## Summary

Keep open: current main still lets a delivery-dispatch error become the outer isolated cron run status after non-fatal isolated agent execution, and the linked draft fix PR is open but unmerged.

## What This Changes

The issue reports scheduled isolated cron `agentTurn` runs being persisted as failed even when the isolated session ended successfully and produced a final assistant message.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/94116 or a maintainer-approved replacement that preserves successful isolated execution while recording delivery/finalization failure separately; keep the broader warning-status contract under https://github.com/openclaw/openclaw/issues/49190.

## Reproduction Assessment

Yes, source-reproducible: current main can return a dispatcher `status: "error"` after non-fatal isolated payload resolution, and the finalizer still propagates that status. I did not run the original scheduled WSL2/Feishu setup live.

## Solution Assessment

Unclear as the final product shape: the focused linked PR is a plausible narrow fix, but maintainers still need to choose whether interim `ok` plus `not-delivered` semantics are acceptable or whether this should wait for a broader warning-status contract.
