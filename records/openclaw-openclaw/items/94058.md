## Summary

Keep open: current main still lets a delivery-dispatch error become the outer isolated cron run result, and the open draft PR that uses closing syntax for this issue has not merged.

## What This Changes

The issue reports that a scheduled isolated cron agentTurn can finish its isolated session successfully with final output while the parent cron run is recorded as error and not delivered.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/94116, or a maintainer-approved replacement, so isolated agent execution success and post-session delivery/finalization failure remain distinct while the broader warning-status contract stays tracked separately.

## Reproduction Assessment

Yes, source-reproducible: current main can return a dispatcher status=error after non-fatal isolated payload resolution, and the finalizer currently propagates that result. I did not run the original scheduled WSL2/Feishu setup live.

## Solution Assessment

Unclear as a product decision: the linked focused PR is a plausible narrow fix, but maintainers still need to choose whether ok plus not-delivered is acceptable or whether this should wait for the broader warning-status contract.
