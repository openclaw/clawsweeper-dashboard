## What This Changes

The issue reports that scheduled isolated cron agentTurn runs can persist a parent run as error/not-delivered even when the isolated session ended successfully and produced a final assistant message.

## Best Possible Solution

Land or replace the linked cron finalization fix so isolated-agent execution success and delivery/finalization failure are represented separately on main.

## Reproduction Assessment

Yes, source-reproducible: current main can propagate a delivery-dispatch error result into the primary isolated cron result, and the scheduler persists that status as the cron run state. I did not live-reproduce the original WSL2/systemd/Feishu scheduled run.
