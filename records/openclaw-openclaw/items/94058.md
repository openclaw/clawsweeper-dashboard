## Summary

Keep open: current main still lets a delivery-dispatch error become the outer isolated cron run status, and the linked draft PR that explicitly fixes this issue is open but unmerged.

## What This Changes

The issue reports scheduled isolated cron `agentTurn` runs persisting parent `status=error` and `deliveryStatus=not-delivered` even after the isolated session ended successfully with a final assistant message.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/94116 or a maintainer-approved replacement that preserves successful isolated agent execution while recording delivery/finalization failure separately, with the broader warning-status contract kept under https://github.com/openclaw/openclaw/issues/49190.

## Reproduction Assessment

Yes, source-reproducible: current main can return a dispatcher `status: "error"` after non-fatal isolated payload resolution, and the finalizer currently propagates that status. I did not run the original scheduled WSL2/Feishu setup live.

## Solution Assessment

Unclear as the final product shape: the linked focused PR is a plausible narrow fix, but maintainers still need to choose whether interim `ok` plus `not-delivered` semantics are acceptable or whether this should wait for the broader warning-status contract.
