Keep open: current main and v2026.6.9 still contain the source path where a post-run delivery error can become the parent cron run status after a successful isolated agent result, and the direct closing PRs are still open rather than merged.

## What This Changes

The issue requests that scheduled isolated cron agentTurn runs preserve a successful isolated session outcome separately from post-session finalization or delivery failure.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/95419 or a maintainer-approved replacement that separates isolated execution success from delivery/finalization failure while preserving deliberate delivery-target guard errors.
