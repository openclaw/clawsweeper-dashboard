## Summary

Keep open: current main and v2026.6.10 still have the cron finalization path where a delivery-dispatch error can become the parent isolated cron run status after a nonfatal agent result, and the direct fix candidates are still open rather than merged.

## What This Changes

The issue reports that a scheduled isolated `agentTurn` can finish the isolated session successfully with a final assistant message but persist the parent cron run as `error` and `not-delivered`.

## Best Possible Solution

Review and land one maintainer-approved fix, likely https://github.com/openclaw/openclaw/pull/95419 or a replacement, that preserves successful isolated execution separately from post-run delivery failure while keeping delivery diagnostics explicit.

## Reproduction Assessment

Yes, source-reproducible: current main can return a delivery-dispatch error after successful isolated payload resolution, and scheduler persistence consumes that returned status. I did not live-run the original WSL2/systemd/Feishu scheduled scenario.

## Solution Assessment

Unclear as final product policy. The finalizer is the right narrow boundary, but maintainers need to accept `ok` plus `not-delivered` semantics or choose the broader warning-status contract.
