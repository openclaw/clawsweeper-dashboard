## Summary

Keep open: current main still has the stale exec-approvals writeback path, and the related open PR #78226 is the active fix candidate with closing syntax for this issue.

## What This Changes

Request to prevent node-host exec allowlist metadata writeback from saving a stale approvals snapshot that can restore admin-revoked allowlist entries.

## Best Possible Solution

Land a CAS/freshness-guarded exec approvals metadata write path, or skip metadata writeback when the approvals file changed, with regression coverage for the stale snapshot race.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence interleaving: resolveExecApprovals loads a file object, an await can pause, admin set can save a newer file with baseHash validation, and recordAllowlistUse later saves the stale object without a freshness check.

## Solution Assessment

Yes. A reload-and-merge or compare-and-swap guard around execution-side metadata writes is the narrow maintainable fix; the open related PR appears to target that boundary, but it still needs normal review before this issue can close.
