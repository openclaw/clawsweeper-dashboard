## Summary

Keep this issue open: current main and v2026.6.8 still save allowlist usage metadata through the previously resolved approvals object, and the linked fix PR is still open, conflicting, and has unresolved review feedback.

## What This Changes

The issue reports a race where node execution-side allowlist usage writeback can save a stale exec-approvals snapshot and restore admin-revoked system.run approvals.

## Best Possible Solution

Land a reviewed security fix that guards every exec-approvals write path involved in execution metadata and initialization, or separates runtime usage metadata from policy state, with regression coverage and real race proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: resolve a node system.run approvals snapshot, let an admin update the approval file through a base-hash set path, then let execution-side usage writeback save the older snapshot.

## Solution Assessment

Yes, the issue's remediation direction is the right boundary: execution-side metadata writes must be freshness-checked or moved out of the policy file. The current linked PR still needs repair before it is the best complete fix.
