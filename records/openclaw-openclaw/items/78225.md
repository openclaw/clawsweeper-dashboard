## Summary

Keep this issue open: current main and the latest release still save execution-side allowlist usage metadata through the previously resolved approvals object, while the linked fix candidate remains open and unmerged.

## What This Changes

The issue reports a race where node-host execution-side allowlist usage writeback can save a stale exec-approvals snapshot and restore admin-revoked system.run approvals.

## Best Possible Solution

Land a reviewed security fix that guards execution-side exec-approvals metadata and allow-always writes with freshness/locking, or separates runtime metadata from policy state, then close this issue from the merged fix.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: resolve a node system.run approvals snapshot, let an admin update the approval file through a base-hash set path, then let execution-side usage writeback save the older snapshot.

## Solution Assessment

Yes. The requested remediation is at the right boundary: execution-side metadata writes must be freshness-checked, locked, or moved out of the policy file before saving.
