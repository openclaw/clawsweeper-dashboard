## Summary

Keep open: current main still has the stale exec-approvals writeback path, and the linked closing PR is open/unmerged with review blockers. The issue should track that fix until a guarded metadata write path lands.

## What This Changes

The issue reports that node execution-side allowlist metadata writeback can save a stale exec approvals snapshot and restore admin-revoked command approvals.

## Best Possible Solution

Land a CAS/freshness-guarded exec approvals metadata write path, or skip metadata writeback when the approvals file changed after policy evaluation, with regression coverage for the stale snapshot race.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence interleaving: approvals are loaded before an async skill-bin refresh, an admin set can save a newer file with baseHash validation, and `recordAllowlistUse` later saves the stale object without a freshness check.

## Solution Assessment

Yes. A reload-and-merge or compare-and-swap guard around execution-side metadata writes is the narrow maintainable fix; the open related PR targets that boundary but still needs review fixes and real behavior proof before this issue can close.
