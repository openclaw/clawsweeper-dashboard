## Summary

Current main still has the reported Telegram account-selection hole: the runtime account list is built from explicit account keys and binding account IDs, so the implicit default account is not started or probed once a named account exists.

## What This Changes

Investigate a reported Telegram account selection bug where adding a named Telegram account stops the implicit default account from being polled.

## Best Possible Solution

Keep the issue open and repair Telegram account selection so default credentials stay in the polling/status account list when named accounts are added, with regression coverage for the mixed top-level-plus-accounts shape.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: a config with top-level Telegram credentials plus any named account causes `listTelegramAccountIds` to return only the named/bound IDs, so Gateway startup never starts `default`.

## Solution Assessment

Yes. The narrow maintainable fix is to include the implicit Telegram default account when default credentials exist, or promote that mixed shape before runtime; a docs-only warning is not enough because the polling account list is already wrong.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/82780. Current main drops the implicit Telegram `default` account from `listTelegramAccountIds` when `channels.telegram.accounts` or account-scoped bindings are non-empty, even if `channels.telegram.botToken`, `channels.telegram.tokenFile`, or `TELEGRAM_BOT_TOKEN` configures the default bot. Implement a narrow Telegram account-selection fix so the default account is included in the polling/status account list whenever default Telegram credentials are present alongside named accounts. Do not change binding matching semantics, do not create a phantom default account when no default credentials exist, and preserve existing missing-default warnings for true multi-account configs. Add focused regression tests for top-level botToken plus a named disabled account and binding, plus env/tokenFile default credential cases if supported by existing helpers. Include a changelog entry because this is user-visible Telegram polling behavior.
