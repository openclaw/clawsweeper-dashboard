## Summary

Keep open: the report is a fresh, concrete Docker/package Feishu runtime regression, and current main still has a plausible path to the same failure because the Feishu bundled build can pull the Lark SDK ESM bundle containing bare `__dirname` into a plugin package marked `type: module`.

## What This Changes

The issue reports that the Feishu plugin exits in release/2026.5.2 Docker/WSL with `__dirname is not defined in ES module scope`.

## Best Possible Solution

Keep the issue open and fix the Feishu packaged build so Lark SDK code with bare `__dirname` is not emitted into an ESM plugin runtime, with a regression check for the Docker/package build path.

## Reproduction Assessment

Yes. The reporter supplied a concrete Docker gateway error, and the current build path plus upstream SDK source explains how `@larksuiteoapi/node-sdk` can introduce bare `__dirname` under `/app/dist/extensions/feishu/package.json` with `type: module`.

## Solution Assessment

Yes, the narrowest maintainable direction is a build/package fix, likely externalizing Feishu's plugin-local Lark SDK dependency or deriving never-bundle behavior from plugin-local runtime dependency metadata, plus a focused build-output/import regression test.
