## Summary

Keep open: this is a fresh, concrete Feishu Docker/package runtime regression, and current main still has the same plausible failure path from bundling the Lark SDK ESM entry containing bare __dirname into an ESM Feishu plugin package.

## What This Changes

Feishu should start successfully in the 2026.5.2 Docker/release path instead of exiting with `__dirname is not defined in ES module scope`.

## Best Possible Solution

Fix the Feishu package/build path so Lark SDK code with bare `__dirname` is not emitted inside an ESM plugin runtime, and add a regression check for the Docker/package build output.

Yes. The reporter supplied the concrete Docker gateway error, and current source plus the exact upstream SDK source explain the same ESM `__dirname` failure path; the focused reproduction is a Feishu-enabled Docker/package build followed by gateway startup.

## Solution Assessment

Yes. The narrow maintainable fix is packaging/build externalization or equivalent plugin-local dependency handling, not a broad Feishu runtime rewrite or a core startup migration.
