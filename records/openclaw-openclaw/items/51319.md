## Summary

Keep open: current main still has Linux-only zombie filtering in shared PID liveness, so the PR targets a real stale-lock recovery gap, but the added macOS regression test mocks the wrong binding and the user-facing fix lacks the required changelog entry.

## What This Changes

This PR extends shared PID liveness checks to treat macOS `ps` zombie states as dead and adds a Vitest regression for stale lock reclamation.

## Best Possible Solution

Merge the macOS zombie handling after fixing the test seam to mock the actual `spawnSync` dependency, adding the required changelog entry, and validating the shared PID-lock tests plus changed gate.

## Reproduction Assessment

Yes at source level. Current main’s `isPidAlive()` filters zombies only on Linux while session and gateway locks rely on it, but I did not create a live macOS zombie process in this read-only review.

## Solution Assessment

No as proposed. The production direction is narrow and appropriate, but the regression test spies on `childProcess.spawnSync` after `pid-alive.ts` has already imported the named `spawnSync`, so it will call real `ps` instead of the mock; a changelog entry is also required before merge.
