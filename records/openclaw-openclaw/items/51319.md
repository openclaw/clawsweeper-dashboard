## Summary

Keep this PR open. Current main still lacks Darwin zombie detection in the shared PID helper, and the PR is on the right implementation seam, but the submitted test still fails on the PR head and the user-facing fix needs a changelog entry before merge.

## What This Changes

The PR adds macOS zombie detection to `isPidAlive()` via `ps -o state= -p <pid>`, adds a mocked macOS regression test, and removes earlier committed runtime log artifacts from the final tree.

## Best Possible Solution

Keep the implementation generic in `src/shared/pid-alive.ts` so session locks, gateway locks, and SDK file locks inherit the same Darwin zombie semantics.
