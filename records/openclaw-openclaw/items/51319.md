## Summary

Keep this PR open because current main still only detects Linux zombies in the shared PID liveness helper, so the macOS stale-lock fix is useful, but the branch is not mergeable as-is: its new macOS regression test fails in CI and the external contributor has not provided after-fix real behavior proof.

## What This Changes

The PR adds macOS zombie-process detection to `src/shared/pid-alive.ts` using `ps -o state= -p <pid>` and adds a mocked macOS zombie regression test.

## Best Possible Solution

Keep the shared PID helper macOS zombie detection, but make the subprocess seam testable without relying on an already-bound named import and require real macOS after-fix proof before merge.

## Reproduction Assessment

Yes at source level, but not with a live macOS zombie run in this review. Current main treats all non-Linux zombie checks as false, so a Darwin zombie that passes `kill(pid, 0)` will be reported alive; the PR's CI also reproduces a concrete test failure in the proposed fix.

## Solution Assessment

No, not as currently implemented. Adding Darwin zombie detection in `src/shared/pid-alive.ts` is the right narrow location, but the test must mock the same callable the production code uses, and the contributor still needs real after-fix macOS behavior proof.
