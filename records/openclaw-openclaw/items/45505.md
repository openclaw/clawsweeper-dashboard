## What This Changes

The issue asks OpenClaw to generalize post-timeout compaction completion reconciliation across session counters, token snapshots, status/UI readers, and related session metadata.

## Best Possible Solution

Define one authoritative compaction-completion reconciliation boundary, then deliberately move only monotonic compaction-derived session fields into it with focused regression coverage.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection shows the current counter-only late-success helper and separate finalizer/read paths for the broader state requested here.

## Solution Assessment

Unclear until maintainers choose the boundary. The current narrow counter fix is not the full solution; the maintainable path is to make compaction-derived convergence explicit and keep unrelated UI/delivery timing state out unless intentionally included.
