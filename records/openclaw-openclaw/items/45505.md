## Summary

Keep open. Current main still has a narrow late-success compaction count repair, but the broader authoritative reconciliation model for token freshness, transcript rotation, status-visible context, memory flush, and auth-profile state is not implemented.

## What This Changes

Request to generalize the late-success compaction repair into an idempotent reconciliation path for compaction-derived session state.

## Best Possible Solution

Define one idempotent compaction-success reconciliation boundary that accepts count, tokensAfter, rotated session identity, and completion metadata, then routes durable status, memory, and auth consumers through monotonic state updates.

## Reproduction Assessment

Yes, at source level. A successful compaction_end after finalization can currently run only the count/updatedAt late helper, and that helper has no path to converge the richer state requested here.

## Solution Assessment

No. The current count-only repair is the right narrow predecessor, but the requested solution should be a single idempotent reconciliation boundary rather than more scattered finalizer writes.
