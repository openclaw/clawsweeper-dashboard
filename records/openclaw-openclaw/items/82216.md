## Summary

Keep open: current main still enforces `openai-curated` only for `codexPlugins`, but the docs and tests describe that as the intentional V1 support boundary, so expanding to bundled or primary-runtime marketplaces needs maintainer product/ownership review rather than cleanup closure.

## What This Changes

The issue requests accepting first-party Codex plugin marketplaces such as `openai-bundled` and `openai-primary-runtime` in `plugins.entries.codex.config.codexPlugins`.

## Best Possible Solution

Decide the supported first-party Codex marketplace set, then update the Codex plugin schema, policy resolver, inventory/activation, migration, docs, and regression tests around a named allowlist while keeping unsupported marketplaces manual-review only.

## Reproduction Assessment

Yes for the current rejection path from source inspection: the manifest schema permits only `openai-curated`, and the runtime resolver drops any other marketplace. I did not run a live gateway restart in this read-only sweep.

## Solution Assessment

Unclear: the requested allowlist is plausible, but the safest solution is a reviewed supported-marketplace contract across schema, migration, activation, and docs rather than only replacing one string constant.
