## Summary

Keep open: the PR targets a real gateway `models.list` latency path, but the current patch should not merge because the new `readOnly` argument is not reflected in the injected loader type and the shared gateway fast path would shrink the existing full `view: "all"` catalog.

## What This Changes

The PR adds a read-only persisted `models.json` fast path to model catalog loading and changes gateway catalog refresh to request `readOnly: true`.

## Best Possible Solution

Keep the fast read-only catalog path scoped to configured/default UI picker flows while preserving the full catalog for `models.list` `view: "all"`, then add focused regression tests and the required changelog entry.

## Reproduction Assessment

Yes for the patch blockers by source inspection: the patched call passes `readOnly` to a loader type that only accepts `config`, and the all-view handler uses the same gateway catalog wrapper. I did not live-reproduce the reported production timing numbers.

## Solution Assessment

No. The direction is useful, but the implementation should be view-aware instead of changing the shared gateway catalog used by both picker-sized and diagnostic full-catalog requests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Allow the readOnly loader argument:** `src/gateway/server-model-catalog.ts:46`
  - body: The changed call passes `{ config, readOnly: true }`, but `LoadModelCatalog` is still typed as accepting only `{ config }` at the top of this file. Fresh object literals are excess-property checked, so this will fail TypeScript checking until the injected loader type is widened.
  - confidence: 0.93
- **[P2] Preserve the full catalog for all view:** `src/gateway/server-model-catalog.ts:46`
  - body: Forcing every `loadGatewayModelCatalog()` refresh to `readOnly` means `models.list` with `view: "all"` also uses the persisted fast path and skips the full registry/plugin catalog. That view is documented and tested as the full diagnostic catalog, so this would silently return the smaller picker-sized list.
  - confidence: 0.91
