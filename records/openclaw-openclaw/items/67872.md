## Summary

Keep open. The PR targets a real Slack mention-skip observability gap, but the branch reads an invalid Slack config surface for the new ingest policy, lacks the needed Slack public config/schema/docs contract, and still has no real Slack behavior proof.

## What This Changes

The PR emits an internal `message:received` hook from Slack's no-mention skip path when ingest is enabled and adds silent-ingest regression tests.

## Best Possible Solution

Revise the PR around a Slack-owned `ingest` opt-in under `channels.slack.channels`, wire it through Slack types/schema/metadata/docs and `resolveSlackChannelConfig`, then emit the hook only from that resolved policy.

## Reproduction Assessment

Yes, as a source-level path. A Slack room message with `requireMention` enabled and no activation records pending history, returns `null`, and the handler exits before shared message hooks run.

## Solution Assessment

No. The PR edits the right skip branch, but the policy gate should come from Slack's valid public channel config contract instead of the generic `groups` helper used by other channels.
