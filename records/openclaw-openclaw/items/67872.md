## Summary

Keep open: the PR targets a real Slack skip-before-hook gap, but the implementation reads an unsupported Slack config surface and lacks real Slack proof for a privacy-sensitive ingest path.

## What This Changes

The PR adds a Slack no-mention skip-path internal `message:received` hook gated by `channels.slack.groups.*.ingest` plus silent-ingest tests.

## Best Possible Solution

Choose the skipped-message observability contract first, then either wire Slack through a documented `channels.slack.channels.*.ingest` opt-in or implement Slack on top of the approved `inbound_observed` hook without changing reply/session behavior.

## Reproduction Assessment

Yes, source-level: current main records Slack no-mention history and returns `null`, and prepared-null messages return before downstream `message_received` or internal `message:received` hooks. I did not run live Slack transport proof in this read-only review.

## Solution Assessment

No. The PR touches the right skip branch, but it uses the generic `groups` policy instead of Slack's valid `channels.slack.channels` config contract; a Slack-owned opt-in or the canonical observed-hook design is safer.
