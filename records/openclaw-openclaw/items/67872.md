## Summary

Keep open. Current main still skips Slack no-mention channel messages before message-hook dispatch, so the PR targets a real gap, but the patch reads a generic `groups` config surface that Slack does not expose in its typed/strict channel config.

## What This Changes

The PR adds a Slack no-mention skip-path internal `message:received` hook gated by group ingest policy and adds silent-ingest regression tests.

## Best Possible Solution

Add a Slack-owned `ingest` boolean to `channels.slack.channels` with wildcard and channel-specific override semantics, wire it through types/schema/docs/metadata and `resolveSlackChannelConfig`, then emit the hook only for that resolved opt-in.

## Reproduction Assessment

Yes. A Slack room message with `requireMention` true and no effective mention enters the no-mention branch, records pending history, returns `null`, and the caller exits before shared hook dispatch.

## Solution Assessment

No. The PR changes the right behavioral path, but the config gate should be resolved from Slack's public `channels.slack.channels` contract rather than the generic group-policy helper.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Read ingest policy from Slack channel config:** `extensions/slack/src/monitor/message-handler/prepare.ts:549-556`
  - body: This lookup uses `resolveChannelGroupPolicy`, which reads `channels.<id>.groups`, but Slack's public strict config exposes room settings under `channels.slack.channels`. Valid Slack configs therefore cannot set the `ingest` value this branch checks, so the new mention-skip hook path remains disabled outside the test's casted invalid config.
  - confidence: 0.91
