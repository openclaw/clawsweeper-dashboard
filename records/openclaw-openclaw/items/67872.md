## Summary

Keep this PR open, but it is not mergeable as submitted: it targets a real Slack mention-skip hook gap, yet the new gate reads an invalid Slack config surface and the privacy-sensitive opt-in is not typed or documented.

## What This Changes

The branch adds a Slack no-mention skip-path internal `message:received` hook gated by an ingest setting and adds regression tests.

## Best Possible Solution

Add a Slack-owned `ingest` boolean under `channels.slack.channels` with wildcard and channel-specific override semantics, wire it through types/schema/metadata/docs and `resolveSlackChannelConfig`, then emit the internal hook only when the resolved opt-in is true.

## Reproduction Assessment

Yes. Source inspection shows a Slack room message with `requireMention` true and no effective mention records pending history, returns `null`, and never reaches shared hook dispatch.

## Solution Assessment

No. The PR edits the right no-mention path, but the policy gate should come from Slack's public channel config contract instead of the generic `groups` helper used by Telegram and Signal.
