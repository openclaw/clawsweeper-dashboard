## Summary

Keep this PR open: it targets a real Slack hook gap, but the submitted patch reads a generic `groups` policy that valid Slack configs do not expose and still needs explicit maintainer review for the privacy-sensitive Slack ingest contract.

## What This Changes

The PR adds a Slack no-mention skip-path internal `message:received` hook gated by group `ingest` policy and adds silent-ingest regression tests.

## Best Possible Solution

Add a Slack-owned `ingest` boolean under `channels.slack.channels` with wildcard and channel-specific override semantics, wire it through types/schema/metadata/docs and `resolveSlackChannelConfig`, then emit `message:received` only when that resolved opt-in is true.

## Reproduction Assessment

Yes. Source inspection shows a Slack room message with `requireMention` true and no effective mention records pending history, returns `null`, and the caller exits before shared hook dispatch.

## Solution Assessment

No. The PR changes the right no-mention path, but the policy gate should come from Slack's public `channels.slack.channels` contract rather than the generic group-policy helper.
