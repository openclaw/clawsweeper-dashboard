## Summary

Keep open, but the current patch is not merge-ready: current main still drops Slack no-mention messages before message hooks, while this PR reads an invalid Slack config surface and lacks real Slack behavior proof for a privacy-sensitive ingest path.

## What This Changes

The PR adds a Slack no-mention skip-path internal `message:received` hook gated by generic group `ingest` policy plus silent-ingest regression tests.

## Best Possible Solution

Define an explicit Slack opt-in, or align with the canonical early observed-hook contract, with schema/types/docs/privacy wording and emit only from that resolved policy after live Slack proof.

## Reproduction Assessment

Yes, source-level: current main records Slack no-mention dropped history and returns `null`, and prepared-null messages return before dispatch emits public or internal message hooks. I did not run a live Slack reproduction in this read-only review.

## Solution Assessment

No. The PR touches the right skip branch, but it uses the generic `groups` policy instead of Slack's valid channel config contract; a Slack-owned opt-in or the canonical early observed-hook design is the safer path.
