## Summary

Keep open. Current main still skips Slack room messages before message hooks when mention gating drops them, so the behavior gap is real, but this PR reads a non-public Slack `groups` config surface and does not add a valid Slack `ingest` config contract. Because the change can emit unmentioned Slack channel content to internal hooks, it needs an explicit typed, validated, and documented Slack opt-in before merge.

## What This Changes

The PR adds a Slack no-mention skip-path internal `message:received` hook gated by group `ingest` policy and adds silent-ingest regression coverage.

## Best Possible Solution

Add a Slack-owned `ingest` boolean to `channels.slack.channels` entries with wildcard and channel-specific override semantics, resolve it through `resolveSlackChannelConfig`, update Slack types/schema/metadata/docs, then emit internal `message:received` from the no-mention skip path only when that resolved opt-in policy is true.

## Reproduction Assessment

Yes. A Slack room message with `requireMention` true and no bot mention reaches the no-mention branch, records pending history, returns `null`, and the caller exits before dispatch hooks run.

## Solution Assessment

No. The intended behavior is valid, but this patch gates it through the wrong config surface; the narrow maintainable fix is a Slack channel config contract and resolver change before adding the hook emission.
