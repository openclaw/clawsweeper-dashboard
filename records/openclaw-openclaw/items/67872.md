## Summary

Keep this PR open, but it is not mergeable as submitted: current main still has the Slack no-mention skip before shared message hooks, while the PR gates the new hook on a Slack config surface that valid Slack configs do not expose and still lacks real Slack proof.

## What This Changes

The PR adds a Slack no-mention skip-path internal `message:received` hook gated by group `ingest` policy plus silent-ingest regression tests.

## Best Possible Solution

Revise the PR around a Slack-owned `ingest` boolean under `channels.slack.channels`, resolve it through Slack types/schema/metadata/docs and `resolveSlackChannelConfig`, then emit the hook only from that explicit opt-in.

## Reproduction Assessment

Yes, as a source-level path: current main records Slack no-mention dropped history and returns `null`, and prepared-null messages never reach shared dispatch hook emission. I did not run a live Slack reproduction in this read-only review.

## Solution Assessment

No. The PR touches the right skip branch, but the policy must come from Slack's valid public channel config contract with docs and privacy review, not the generic group-policy helper used by other channels.
