The PR adds a default-off Feishu `streamingSearchFallback` config option, sends completed streamed card text as a plain-text search fallback, and updates docs, metadata, tests, and changelog.

## Best Possible Solution

## Reproduction Assessment

## Solution Assessment

Yes as a direction, but not merge-ready. A Feishu-specific default-off fallback is narrow and preserves default behavior, but the branch needs rebase, real Feishu proof, and the reply-fallback parity fix below.
