## Summary

Keep open. Current main still has CardKit-only Feishu streaming replies, while this PR appears technically sound and exact-head CI is green; the remaining question is the visible duplicate-message tradeoff.

## What This Changes

This PR adds a Feishu `streamingSearchFallback` boolean, sends chunked plain-text fallback after streaming cards close when enabled, and updates docs, manifest/config metadata, changelog, and dispatcher tests.

## Best Possible Solution

Feishu should keep streaming default unchanged and offer a documented opt-in searchable fallback for users who prefer searchability over duplicate visible content.

## Reproduction Assessment

Yes, at source level. The linked issue gives concrete Feishu UI steps, and current main closes streaming replies as interactive CardKit messages without sending a searchable text fallback; I did not run a live Feishu search session.

## Solution Assessment

Yes, with maintainer approval. A Feishu-specific, default-off fallback is narrow and avoids core changes, but the visible duplicate-message behavior is a product/UX tradeoff.
