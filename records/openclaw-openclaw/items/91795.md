## Summary

Keep open: current main and the latest release still return only shallow liveness data from `/healthz`, while version metadata exists only through authenticated/probe/UI paths, so the requested unauthenticated operator endpoint remains missing and needs a maintainer endpoint/field decision.

## What This Changes

Request to expose the running gateway version through `/healthz` or a dedicated `/version` endpoint for multi-tenant operational observability.

## Best Possible Solution

Add a small public version metadata surface using the existing runtime version resolver, preserving shallow liveness semantics and documenting exactly which unauthenticated fields are intentionally exposed.

## Reproduction Assessment

Yes. Source and tests show current `/healthz` returns only `ok` and `status`, and the latest release tag has the same behavior.

## Solution Assessment

Unclear. Reusing `resolveRuntimeServiceVersion` is the right implementation primitive, but maintainers should choose `/healthz` versus `/version` and decide whether unauthenticated `commit` or `built_at` fields are acceptable.
