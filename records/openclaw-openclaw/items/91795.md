## Summary

Keep open: current main and the latest release still expose only shallow liveness JSON on `/health` and `/healthz`, so the requested public running-version surface remains missing. The older fleet-ops issue overlaps but bundles a separate high-risk update endpoint, while the linked implementation PR is closed unmerged, so this narrower version-only request is still useful for maintainer decision-making.

## What This Changes

Request a stable unauthenticated gateway endpoint or health-field addition that reports the running OpenClaw version for multi-tenant operators.

## Best Possible Solution

Add a narrow public version metadata surface using `resolveRuntimeServiceVersion`, document the approved unauthenticated field set, and avoid exposing commit/build metadata unless maintainers explicitly accept that fingerprinting tradeoff.

## Reproduction Assessment

Yes at source level: current `/health` and `/healthz` serialize only `ok` and `status`, and tests pin that exact body before config loading. No failing runtime test was run because this is a read-only review.

## Solution Assessment

Unclear: exposing `version` is a small maintainable core change, but maintainers should choose the endpoint and unauthenticated metadata fields before implementation. Reusing the existing runtime version resolver is the safest implementation primitive if approved.
