## Summary

Keep open: current main and the latest release still expose only shallow unauthenticated `/health` and `/healthz` JSON, while the existing runtime version is only available through other surfaces such as the Gateway WebSocket hello payload or CLI/runtime helpers. The closed linked PR was not merged, and the remaining decision is the public unauthenticated metadata contract.

## What This Changes

Request to expose the running OpenClaw Gateway version through an unauthenticated HTTP health or version endpoint for fleet operators.

## Best Possible Solution

Add a documented additive version-only public surface after maintainers approve the unauthenticated field set, reusing `resolveRuntimeServiceVersion` and keeping `/healthz` pre-config cheap.

## Reproduction Assessment

Yes. Current main maps `/health` and `/healthz` to a live probe that serializes only `{ ok: true, status }`, and the latest release has the same behavior.

## Solution Assessment

Unclear until maintainers choose the public contract. A narrow version-only surface using `resolveRuntimeServiceVersion` looks maintainable, but `/healthz` versus `/version` and optional `commit` or `built_at` fields need product and security review first.
