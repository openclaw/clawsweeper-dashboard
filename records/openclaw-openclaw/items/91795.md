## Summary

Keep open: current main and the latest release still return only shallow liveness data from `/health` and `/healthz`; running version exists through other authenticated/WS/UI paths, but not through the requested unauthenticated operator surface.

## What This Changes

Requests a stable unauthenticated gateway endpoint or health payload field exposing running OpenClaw version metadata for multi-tenant operators.

## Best Possible Solution

Add a small documented public version metadata surface after maintainers choose the endpoint and unauthenticated field set, reusing `resolveRuntimeServiceVersion` and keeping liveness cheap.

## Reproduction Assessment

Yes at source level. Current source and probe tests show `/health` and `/healthz` return only `ok` and `status`, while the latest release tag has the same behavior.

## Solution Assessment

Unclear until maintainers choose the public contract. Reusing `resolveRuntimeServiceVersion` is the right implementation primitive, but `/healthz` versus `/version` and the unauthenticated field set need a product/security decision.
