## Summary

This remains a valid open feature request: current main and the latest release expose shallow unauthenticated liveness/readiness but not a shell-free running-version value, and the only linked implementation closed unmerged for contributor queue policy rather than technical supersession. The change is small, but maintainers should choose the public contract and limit unauthenticated metadata before implementation.

## What This Changes

Request a stable unauthenticated gateway endpoint that exposes the running OpenClaw version for fleet operators.

## Best Possible Solution

Expose the existing stable runtime `VERSION` through one approved public contract—an additive liveness-probe field or a dedicated `/version` endpoint—with exact tests and operator documentation; omit commit/build metadata unless separately justified and approved.

## Reproduction Assessment

Yes. Current-main source and exact-response tests show public health probes return liveness/readiness data without the running version; this is source-reproducible without executing the server.

## Solution Assessment

Unclear pending maintainer choice. Reusing the existing `VERSION` constant at the probe boundary is the narrowest implementation, but `/healthz` versus `/version` and the permitted unauthenticated fields are product/security contract decisions.
