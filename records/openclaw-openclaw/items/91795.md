## Summary

Keep open: current main and v2026.6.8 still expose only shallow unauthenticated `/health` and `/healthz` liveness JSON, while running version is available only through authenticated or non-health surfaces.

## What This Changes

Request to expose the running OpenClaw gateway version through `/healthz`, `/health`, or a dedicated `/version` endpoint for multi-tenant operators without shell access.

## Best Possible Solution

Add a small documented public version metadata surface after maintainers choose the endpoint and unauthenticated field set, likely reusing `resolveRuntimeServiceVersion` and preserving shallow probe behavior.

## Reproduction Assessment

Yes, at source level. Current main and v2026.6.8 both show `/health` and `/healthz` returning only shallow liveness data, while the requested unauthenticated HTTP version surface is absent.

## Solution Assessment

Unclear until maintainers choose the public contract. Reusing `resolveRuntimeServiceVersion` is the right primitive, but `/healthz` versus `/version` and optional `commit` or `built_at` fields need product and security review.
