## Summary

Keep open: current main and the latest release still preserve Telegram Bot API tokens in timeout URL path segments, and https://github.com/openclaw/openclaw/pull/97003 is the active closing PR for this security fix.

## What This Changes

Redact Telegram bot tokens embedded in fetch-timeout diagnostic URLs before they are logged or surfaced.

## Best Possible Solution

Review and land exactly one canonical timeout-sanitizer fix, likely https://github.com/openclaw/openclaw/pull/97003, then close this issue after the fix reaches main.

## Reproduction Assessment

Yes. Source inspection shows a Telegram `/bot${token}` URL can flow into `fetchWithTimeout`, and the current sanitizer removes userinfo/query/hash but not the path token.

## Solution Assessment

Yes. Redacting inside `sanitizeTimeoutLogUrl` is the right narrow boundary because downstream log redaction can be configured off and timeout diagnostics store their own sanitized URL field.
