## What This Changes

The issue requests redacting Telegram Bot API tokens embedded in fetch-timeout URL path segments before timeout URLs are logged or surfaced.

## Best Possible Solution

Land one focused security-aware fix that redacts real Telegram `/bot<TOKEN>` path segments inside the fetch-timeout sanitizer and covers both parser and fallback branches with regression tests.

## Reproduction Assessment

Yes. Source inspection shows that calling the timeout helper with a Telegram Bot API URL containing `/bot<TOKEN>/sendMessage` keeps that path after stripping only userinfo, query, and hash; I did not execute tests because this is a read-only review.

## Solution Assessment

Yes. The narrowest maintainable fix is to redact Telegram token-shaped `/bot...` path segments at the timeout URL sanitizer itself, before downstream logging redaction can be disabled or hint-mask secrets.
