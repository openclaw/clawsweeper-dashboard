## Summary

Keep open: the PR has a useful freshness direction, but the current diff still drops sibling descendant outputs in fan-out cron runs and applies an over-broad interim-message heuristic to arbitrary child results.

## What This Changes

The PR changes cron isolated-agent descendant fallback to pick the freshest descendant reply, updates subagent follow-up tests, and adds a type assertion in the global singleton helper.

## Best Possible Solution

Keep the latest-run-per-child dedupe, aggregate one latest usable reply per distinct child, narrow any acknowledgement filtering to true child acknowledgements, and add the required changelog entry.

## Reproduction Assessment

Yes. Current main has a focused unit test for multi-child descendant aggregation, and the PR diff changes that case to expect only the freshest child while returning after the first usable reply.

## Solution Assessment

No. Selecting the newest run per child is useful, but the maintainable fix should preserve distinct-child aggregation and avoid applying the broad cron interim heuristic to child output.
