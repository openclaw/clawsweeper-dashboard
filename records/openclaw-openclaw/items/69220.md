## Summary

Keep open. Current main and the latest tagged release still combine Google/Gemini tagged reasoning-output instructions with native Gemini thinking payloads, and the issue discussion has multiple matching user reproductions. This is a narrow provider/runtime contract bug suitable for a focused repair PR.

## What This Changes

The issue requests changing Google/Gemini reasoning output from forced tagged `<think>/<final>` text mode to native mode because native Gemini `thinkingConfig` is already enabled and the combination can produce empty post-tool turns.

## Best Possible Solution

Update the Google/Gemini provider reasoning contract so native Gemini transports that send `thinkingConfig` resolve to native output mode, while keeping Gemini replay sanitation and thought-signature handling intact. Add focused tests for the provider hook, fallback behavior, and direct transport payloads, plus a user-facing changelog entry.

## Reproduction Assessment

Yes. The current-main path deterministically sets `reasoningTagHint` for Google while the direct Gemini transport sends native `thinkingConfig.includeThoughts`; the issue discussion adds matching real-session reproductions, though this review did not run a paid live Gemini loop.

## Solution Assessment

Yes. Fixing the Google/Gemini reasoning-output contract is narrower and more maintainable than only handling the downstream empty-payload symptom, provided the existing Gemini replay and transport sanitation behavior stays covered.
