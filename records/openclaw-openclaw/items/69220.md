## Summary

Keep open. Current main and the latest release still combine the Google/Gemini tagged reasoning-output prompt with native Gemini thinking payloads, and the issue has multiple user reports plus a closed duplicate that points back here as the canonical tracker.

## What This Changes

The issue reports that Google/Gemini native thinking is combined with tagged `<think>/<final>` reasoning prompts, causing unclosed thinking tags, empty post-tool turns, and `payloads=0` errors.

## Best Possible Solution

Update the Google/Gemini provider reasoning-output contract so Gemini transports that send native `thinkingConfig` resolve to native output, then update the tagged-mode tests/docs while preserving thought-signature and tool-result replay coverage.

## Reproduction Assessment

Yes, at source level: current main deterministically resolves Google/Gemini to tagged reasoning mode while the direct Google transport sends native `thinkingConfig.includeThoughts`. I did not run a paid live Gemini loop in this read-only review.

## Solution Assessment

Yes. Fixing the provider-owned Google/Gemini reasoning-output mode is the narrow maintainable fix; only handling downstream empty payloads would leave the conflicting prompt/API contract intact.
