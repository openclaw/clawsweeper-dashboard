## Summary

Keep open. Current main and latest release still combine Google/Gemini tagged `<think>/<final>` prompting with native Gemini `thinkingConfig.includeThoughts`, and this issue has multiple affected-user confirmations without a merged fix for that direct provider conflict.

## What This Changes

The issue asks OpenClaw to stop combining Gemini text-tag reasoning prompts with native Gemini thinking responses because that can produce unclosed `<think>` output, empty post-tool assistant turns, and wasted paid model calls.

## Best Possible Solution

Move the direct Google/Gemini reasoning output contract to native whenever native Gemini thinking is sent, add regression coverage for the provider hooks and Google thinking requests, and keep broader reasoning-field extraction on its separate canonical issue.

## Reproduction Assessment

Yes, at source level: current main deterministically resolves direct Google/Gemini to tagged reasoning mode while the Google transport can send native `thinkingConfig.includeThoughts`. I did not run a paid live Gemini loop in this read-only review.

## Solution Assessment

Yes. Fixing the provider-owned Google/Gemini reasoning-output contract is the narrow maintainable path; generic empty-payload retries or broad OpenRouter reasoning-field extraction would leave the conflicting prompt/API contract intact.
