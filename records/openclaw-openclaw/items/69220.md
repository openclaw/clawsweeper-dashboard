## Summary

Keep open. Current main and the latest release still combine Google/Gemini tagged reasoning-output instructions with native Gemini thinking payloads, and a related duplicate was already closed in favor of this canonical tracker.

## What This Changes

The issue asks OpenClaw to stop forcing Google/Gemini into tagged reasoning when the Gemini transport also sends native thinking, avoiding unclosed tags and empty post-tool turns.

## Best Possible Solution

Update the Google/Gemini provider reasoning-output contract so Gemini transports that send native thinking resolve to native output, while preserving replay sanitation, thought-signature safeguards, and tool-result pairing coverage.

## Reproduction Assessment

Yes, at source level: current main deterministically resolves Google/Gemini to tagged reasoning mode while the direct Google transport sends native `thinkingConfig.includeThoughts`. I did not run a paid live Gemini loop in this read-only review.

## Solution Assessment

Yes. Fixing the provider-owned Google/Gemini reasoning-output mode is the narrow maintainable fix; downstream empty-payload handling alone would leave the conflicting prompt/API contract intact.
