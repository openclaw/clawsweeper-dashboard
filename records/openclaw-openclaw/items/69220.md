## Summary

Keep open. Current main and the latest release still combine Google/Gemini tagged reasoning-output instructions with native Gemini thinking payloads, and the discussion has multiple matching user reproductions of empty post-tool turns.

## What This Changes

Stop routing Gemini native-thinking providers through text-tag reasoning so native `thinkingConfig` does not conflict with `<think>/<final>` prompt enforcement and produce empty turns.

## Best Possible Solution

Update the Google/Gemini reasoning-output contract so transports that send native Gemini thinking resolve to `native`, while keeping existing Gemini replay sanitation and thought-signature safeguards covered by focused tests.

## Reproduction Assessment

Yes. Current main deterministically resolves Google/Gemini to tagged mode while direct Gemini transport sends native `thinkingConfig.includeThoughts`; the issue discussion adds matching real-session reproductions.

## Solution Assessment

Yes. Fixing the provider-owned Google/Gemini reasoning-output mode is narrower and more maintainable than only handling the downstream empty-payload symptom.

## Repair Work Prompt

Fix the Gemini tagged-vs-native reasoning conflict tracked by https://github.com/openclaw/openclaw/issues/69220. Current main still resolves Google/Gemini reasoning output to `tagged` while the direct Google transport sends native Gemini `thinkingConfig.includeThoughts`, which users report can produce unclosed `<think>` text, a tool pivot, and `payloads=0` empty turns. Update the Google/Gemini provider reasoning-output contract so native Gemini transports that send `thinkingConfig` resolve to `native`; preserve existing Gemini replay sanitation, thought-signature cleanup, tool-result pairing, and Google CLI/other provider behavior unless tests prove a targeted distinction is required. Add/adjust focused tests in provider-utils, shared provider model hooks, Google provider registration, and Google transport payload coverage. Add a user-facing `CHANGELOG.md` fix entry. Do not broaden into the separate textSignature bloat work in #48709 unless the same minimal change requires it.
