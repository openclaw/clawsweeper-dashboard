## Summary

Keep open: current main, the latest release tag, and the deployed docs still describe per-call `host=node` from `auto` without the sandbox-active caveat, while the runtime and tests reject both `node` and `gateway` overrides from `auto` when a sandbox is active. The linked docs-only repair PRs are closed unmerged, and the later Codex node-exec fix explicitly belongs to a different report.

## What This Changes

The issue asks OpenClaw to resolve a docs/runtime mismatch for exec `host=auto` and per-call `host=node` override behavior.

## Best Possible Solution

Land or adopt a narrow docs-only correction across the exec, exec approvals, and nodes pages so `host=node` and `host=gateway` overrides from `auto` are documented as allowed only when no sandbox runtime is active, then ensure the public docs redeploy.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: current docs and deployed docs state the uncaveated `host=node` allowance, while the current runtime guard and tests reject it when a sandbox is active. I did not run tests because this review was required to keep the checkout read-only.

## Solution Assessment

Yes, the best fix is docs-only: the runtime behavior is already fail-closed and covered by tests, so changing runtime would weaken the sandbox boundary instead of resolving the mismatch.
