## Summary

Keep open: current main, the deployed docs, and the latest release still describe `host=node` as allowed from `auto` without the active-sandbox caveat, while runtime and tests reject both `node` and `gateway` overrides from `auto` when a sandbox is active. The direct docs-fix PRs are closed unmerged, and the merged Codex node-exec PR is adjacent work for a different tool-surface problem.

## What This Changes

The issue reports that exec docs say per-call `host=node` is allowed from `tools.exec.host=auto`, while runtime rejects that override when a sandbox is active.

## Best Possible Solution

Land a narrow docs-only correction across the exec, exec approvals, and nodes pages so `host=node` and `host=gateway` overrides from `auto` carry the same no-active-sandbox caveat.

## Reproduction Assessment

Yes. Source inspection shows the docs claim a broader `host=node` override than the runtime and tests allow when a sandbox is active.

## Solution Assessment

Yes. The runtime is already fail-closed and test-covered, so the maintainable fix is to correct docs rather than weaken the sandbox boundary.
