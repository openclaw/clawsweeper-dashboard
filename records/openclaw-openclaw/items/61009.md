## Summary

Keep open. Current main and the latest release still document per-call `host=node` from `tools.exec.host=auto` as allowed, while the runtime rejects that override when a sandbox runtime is active; the linked open docs PR is the active fix path and uses closing syntax for this issue.

## What This Changes

The issue reports a docs/runtime mismatch for per-call `host=node` overrides when `tools.exec.host=auto` is used with an active sandbox runtime.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/73757 or an equivalent docs-only correction across the three affected docs pages while preserving the current fail-closed runtime behavior.

## Reproduction Assessment

Yes. Source inspection on current main shows docs allowing `host=node` from `auto`, while `resolveExecTarget` rejects that request when `sandboxAvailable` is true and the unit test locks the rejection.

## Solution Assessment

Yes. A docs-only correction is the narrowest maintainable fix because the runtime behavior is intentional, security-relevant, and already covered by tests.
