## Summary

Keep open. Current main, the latest release, and the public docs still say per-call `host=node` is allowed from `auto` without the sandbox-active caveat, while the runtime and tests reject that override when a sandbox runtime is active; the related repair PRs are closed unmerged, and https://github.com/openclaw/openclaw/issues/92141 is a separate Codex/node tool-surface report.

## What This Changes

The issue reports that exec docs allow `host=node` per-call overrides from `tools.exec.host=auto` more broadly than the runtime permits.

## Best Possible Solution

Adopt or reopen an equivalent docs-only fix that adds the sandbox-active caveat to `docs/tools/exec.md`, `docs/tools/exec-approvals.md`, and `docs/nodes/index.md`, then let the docs publish pipeline redeploy it.

## Reproduction Assessment

Yes. Source inspection reproduces the mismatch: current docs allow per-call `host=node` from `auto` without the sandbox-active caveat, while `resolveExecTarget` and its tests reject that override when a sandbox runtime is available.

## Solution Assessment

Yes. A narrow docs-only correction is the best fix because the runtime behavior is explicit and test-locked; changing runtime to match the stale wording would weaken the sandbox-active fail-closed rule.
