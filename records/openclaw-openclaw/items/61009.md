## Summary

Keep open. Current main and the latest release still have docs that allow per-call `host=node` from `auto`, while the runtime and tests reject that path when a sandbox is active; the open linked PR is the right fix path.

## What This Changes

The issue asks OpenClaw to resolve the exec docs/runtime mismatch around per-call `host=node` from `tools.exec.host=auto` in sandboxed sessions.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/73757 or an equivalent docs-only correction across the three affected pages while preserving the current fail-closed runtime behavior.

## Reproduction Assessment

Yes. Source inspection on current main shows the docs allow `host=node` from `auto`, while `resolveExecTarget` rejects `requestedTarget: "node"` when `sandboxAvailable: true` and the unit test pins that behavior.

## Solution Assessment

Yes. A docs-only correction is the narrowest maintainable solution because the runtime behavior is intentional, security-relevant, and already covered by tests and changelog history.
