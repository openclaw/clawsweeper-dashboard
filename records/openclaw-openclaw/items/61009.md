## Summary

Keep open. Current main and the latest release still describe per-call `host=node` from `tools.exec.host=auto` as allowed, while the runtime rejects that override when a sandbox is active; the linked open PR is the active fix path and uses closing syntax for this issue.

## What This Changes

Correct the exec docs so they no longer say per-call `host=node` from `tools.exec.host=auto` is allowed while a sandbox runtime is active.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/73757 or an equivalent docs-only correction across the three affected pages while preserving the current fail-closed runtime behavior.

## Reproduction Assessment

Yes. Source inspection on current main and `v2026.5.12` shows docs allowing `host=node` from `auto`, while `resolveExecTarget` rejects `requestedTarget: "node"` when `sandboxAvailable: true` and the unit test pins that throw path.

## Solution Assessment

Yes. A docs-only correction is the narrowest maintainable solution because the runtime behavior is intentional, security-relevant, and already covered by tests.
