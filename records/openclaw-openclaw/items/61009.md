## Summary

Keep open. Current main still has the reported docs/runtime mismatch, and open closing PR #73757 already targets the exact docs-only correction, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue asks OpenClaw to resolve a docs/runtime mismatch where docs say per-call `host=node` from `tools.exec.host=auto` is allowed, while runtime rejects it when a sandbox runtime is active.

## Best Possible Solution

Land PR #73757 or an equivalent docs-only correction across `docs/tools/exec.md`, `docs/tools/exec-approvals.md`, and `docs/nodes/index.md`, then close this issue once the public docs describe the sandbox-active `host=node` rejection rule.

## Reproduction Assessment

Yes. The mismatch reproduces by comparing the current docs wording with `resolveExecTarget({ configuredTarget: "auto", requestedTarget: "node", sandboxAvailable: true })`, which the existing unit test asserts must throw.

## Solution Assessment

Yes. A docs-only correction is the narrowest maintainable solution because runtime, tests, and changelog already agree that sandbox-backed `auto` rejects per-call host overrides.
