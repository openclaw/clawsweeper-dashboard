## Summary

Keep open. Current main still only closes the reset target ACP runtime; child ACP sessions that record the parent in spawnedBy/parentSessionKey are not enumerated, and the related open PRs cover adjacent completion, pressure, and PID surfaces rather than this exact reset gap.

## What This Changes

The issue requests reliable cleanup of ACP mode run/oneshot child sessions, especially closing child ACP session keys when a parent session is reset.

## Best Possible Solution

Add reset/delete cleanup that enumerates ACP entries owned by the reset parent and closes each through the existing bounded ACP cancel/close helper, with focused regression coverage.

## Reproduction Assessment

Yes, source-level. Seed a parent session plus a child ACP entry whose spawnedBy or parentSessionKey points to that parent, invoke sessions.reset for the parent, and current main closes only the target ACP key rather than the child row.

## Solution Assessment

Yes. The narrow reset-path child ACP enumeration is the best immediate fix for this issue's specific gap; completed-run cleanup, PID exposure, and startup process reaping should remain in their related work streams.
