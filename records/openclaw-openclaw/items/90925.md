## Summary

Keep open: current main and the latest local release tag still contain the Codex/OAuth compaction and subagent announce delivery paths that can turn a completed child result into failed requester-visible delivery.

## What This Changes

Issue #90925 reports completed native subagent results being lost when high-context Codex/OpenAI OAuth requester-side announce compaction falls into an OpenAI Responses API-key route or Codex-owned compaction skip failure.

## Best Possible Solution

Keep the issue open until a maintainer-selected fix makes high-context Codex/OAuth subagent announce use the effective Codex compaction/auth path or preserve requester delivery when Codex-owned compaction is skipped, with focused regression tests and live Codex/OAuth proof.

## Reproduction Assessment

No live current-main repro was run, but the source path is clear and matches two live release reports: requester announce compaction can fall through the Codex-owned skip or OpenAI Responses auth path and then fail before visible completion delivery.

## Solution Assessment

The desired behavior is correct, but the best fix shape needs maintainer judgment on whether Codex-owned automatic compaction should be a no-op success, a different native request, or a delivery-preserving fallback.
