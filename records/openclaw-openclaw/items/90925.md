## Summary

## What This Changes

The issue requests fixing subagent completion announce/requester compaction so Codex/OAuth sessions do not fall into OpenAI Responses API-key auth or drop completed subagent results.

## Best Possible Solution

Carry the requester session's effective Codex runtime/auth facts into announce compaction, or make requester wake delivery tolerate Codex-owned/post-success compaction failures, then prove high-context Discord Codex/OAuth subagent completion is visibly delivered.
