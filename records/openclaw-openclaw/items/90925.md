## What This Changes

The issue reports that high-context Codex/OAuth subagent completion announce can fail requester-side compaction through an OpenAI Responses API-key-only route, causing completed child results to miss visible parent delivery.

## Best Possible Solution

Fix the requester completion-announce boundary so Codex/OAuth sessions never fall into an API-key-only OpenAI Responses compaction path, and ensure Codex-owned auto-compaction cannot turn a successful child completion into dropped parent delivery.

## Reproduction Assessment

Yes at source level with reporter logs: current main still routes completed subagent announces through a requester agent handoff, and handoff failure becomes failed or blocked delivery after retry exhaustion. This review did not run a live Discord Codex/OAuth high-context reproduction.

## Solution Assessment

Unclear as a final code direction without maintainer choice: the narrow fix could preserve effective Codex runtime/auth facts through the announce handoff, reinterpret Codex-owned auto-compaction as non-fatal there, or combine both.
