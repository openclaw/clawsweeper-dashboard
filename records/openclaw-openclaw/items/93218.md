Full review comments:

- **[P2] Thread session mode into Discord progress suppression:** `extensions/discord/src/monitor/message-handler.draft-preview.ts:45`
  - body: `discordStreamMode` now honors `sessionStreamingMode`, but the progress-tool and default-progress suppression helpers below still read only `params.discordConfig`. With `/stream progress` overriding an off/default Discord config, the progress draft can run without suppressing the normal tool-progress lane, so the same progress can be delivered twice. Derive those helpers from the same effective session-aware mode or entry.
  - confidence: 0.88
- **[P2] Thread session mode into Slack progress suppression:** `extensions/slack/src/monitor/message-handler/dispatch.ts:691`
  - body: This passes the session override into `resolveSlackStreamingConfig`, but later `previewToolProgressEnabled` and `suppressDefaultToolProgressMessages` still use only `account.config`. A session-level `/stream progress` can therefore enable Slack progress streaming while leaving default tool-progress suppression calculated from the old config, producing noisy duplicate progress delivery. Use the same effective mode for those decisions too.
  - confidence: 0.87
- **[P1] Keep the streaming parser off the public SDK surface:** `src/channels/streaming.ts:72`
  - body: Exporting `parsePreviewStreamingMode` from `src/channels/streaming.ts` makes it visible through `openclaw/plugin-sdk/channel-outbound`'s export-star, even though the PR only needs it for core command parsing. That creates an undocumented plugin SDK addition without the usual contract/docs/baseline discussion. Keep the helper internal through a non-public path, or make the SDK addition explicit.
  - confidence: 0.82
