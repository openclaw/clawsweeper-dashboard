Full review comments:

- **[P2] Thread session mode through Discord progress suppression:** `extensions/discord/src/monitor/message-handler.draft-preview.ts:43-46`
  - body: `discordStreamMode` now includes the session override, but `previewToolProgressEnabled` and `resolveChannelStreamingSuppressDefaultToolProgressMessages` still read only `params.discordConfig`. With config `streaming.mode: "off"` plus `/stream progress`, Discord can start the progress preview while leaving default tool-progress messages unsuppressed, producing duplicate or noisy progress delivery.
  - confidence: 0.87
- **[P2] Thread session mode through Slack progress suppression:** `extensions/slack/src/monitor/message-handler/dispatch.ts:688-692`
  - body: Slack computes `slackStreaming.mode` with `sessionStreamingMode`, but the later tool-progress and suppression helpers still resolve from `account.config` only. A session override from config `off` to `progress` can therefore enable streaming while default tool-progress messages can leak alongside the preview/native stream.
  - confidence: 0.86
- **[P3] Do not treat block delivery as preview streaming config:** `src/auto-reply/reply/commands-session.ts:120-123`
  - body: `blockStreaming` is the legacy block-delivery toggle, not preview-mode configuration. Including it in `hasConfiguredPreviewStreaming` makes `/stream status` report a preview-mode config source for block-delivery-only config, which can mislead users before they set a session override.
  - confidence: 0.78
