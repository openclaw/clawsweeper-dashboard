Full review comments:

- **[P2] Read stream overrides from the command target session:** `extensions/slack/src/monitor/message-handler/dispatch.ts:678-686`
  - body: Native slash commands can arrive with a transport `SessionKey` while `CommandTargetSessionKey` names the chat session that command handlers mutate. This new lookup ignores `CommandTargetSessionKey`, and the Discord path does the same, so a `/stream` override can be stored on the target chat session while subsequent native-command dispatch reads the slash session and ignores the override. Prefer `CommandTargetSessionKey ?? SessionKey ?? route.sessionKey` and cover native slash routing.
  - confidence: 0.88
- **[P3] Do not treat blockStreaming as preview-mode config:** `src/auto-reply/reply/commands-session.ts:121-124`
  - body: `blockStreaming` is the legacy block-reply toggle, not preview-streaming mode. With only Discord `blockStreaming` configured, this status path takes the channel-config branch and reports `off` instead of the effective Discord default `progress`, which can mislead operators checking `/stream status`.
  - confidence: 0.78
