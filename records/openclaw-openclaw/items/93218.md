  - body: Slack computes `slackStreaming.mode` with `sessionStreamingMode`, but the later tool-progress and suppression helpers still resolve from `account.config` only. A session override from config `off` to `progress` can therefore enable streaming while default tool-progress messages can leak alongside the preview/native stream.
  - confidence: 0.86
- **[P3] Do not treat block delivery as preview streaming config:** `src/auto-reply/reply/commands-session.ts:121-123`
  - body: `blockStreaming` is the legacy block-delivery toggle, not preview-mode configuration. Including it in `hasConfiguredPreviewStreaming` makes `/stream status` report a preview-mode source for block-delivery-only config, which can mislead users before they set a session override.
  - confidence: 0.78
