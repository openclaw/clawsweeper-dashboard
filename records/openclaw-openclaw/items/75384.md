## Summary

Keep open. The PR targets a real current-main memory-flush bug, but the updated NBSP placeholder does not survive OpenClaw's default Anthropic messages transport trimming, so the patch can still fail to send a non-empty provider-facing user turn on the reported Anthropic path.

## What This Changes

The PR adds a changelog entry, changes runtime-only embedded-runner blank prompt submission to pass a U+00A0 placeholder, and updates the context-engine regression test to expect that placeholder.

## Best Possible Solution

A correct fix should preserve `transcriptPrompt: ""`, keep the flush instructions in runtime/system context, and submit provider-facing runtime-only content that survives the downstream Anthropic payload conversion as non-empty, with coverage at both the runner and Anthropic transport boundaries.

## Reproduction Assessment

Yes. Static tracing reproduces the issue path on current main, and the PR blocker is reproducible with the Anthropic transport's `trim()` guard plus a local Node check showing U+00A0 trims to empty.

## Solution Assessment

No. Fixing at the runner boundary is a good direction, but U+00A0 is not the best placeholder because the default Anthropic transport treats it as empty; use a trim-surviving placeholder or a runtime-only transport special case instead.

## Review Findings

Overall correctness: patch is incorrect
