## Summary

Keep open: the PR addresses a real swallowed-error path and adds useful logging, but the proposed recovery path does not retry the already-ended completion failure case and the external PR still lacks after-fix real behavior proof.

## What This Changes

The PR changes `waitForSubagentCompletion` to warn on completion-handler errors and schedule orphan recovery instead of silently ignoring the catch block.

## Best Possible Solution

Retry or reschedule the already-ended completion cleanup/announce path directly, keep the warning log, add focused regression coverage, and require redacted real behavior proof before merge.

## Reproduction Assessment

Yes, source-level: make `waitForAgentRun` return `ok` and make `completeSubagentRun` reject after `endedAt` is persisted; current main's empty catch swallows that failure. I did not execute tests because this review is read-only.

## Solution Assessment

No. Logging is useful, but orphan recovery is not the right retry mechanism for ordinary already-ended records because that scanner skips them; the narrower fix is to retry completion cleanup or announce delivery directly.
