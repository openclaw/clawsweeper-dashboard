## Summary

Keep this PR open: the reported swallowed-error bug is real from source inspection, but the patch does not actually recover the main failure case because orphan recovery skips ordinary ended runs. The external real-behavior proof gate is also still unsatisfied.

## What This Changes

The PR replaces the empty `waitForSubagentCompletion` catch block with a warning log and a scheduled orphan-recovery pass when completion handling throws.

## Best Possible Solution

Keep the PR open, but change the fix to log the failure and directly retry or resume the ended-run cleanup/announce path for records with `endedAt` and no `cleanupCompletedAt`, with focused regression coverage and redacted real runtime proof.

## Reproduction Assessment

Yes, source-level: make `waitForAgentRun` return `ok` and make `completeSubagentRun` reject after `endedAt` is persisted; current main's empty catch discards that error. I did not execute tests because this review is read-only.

## Solution Assessment

No. Logging the failure is useful, but orphan recovery is for unended or restart-aborted sessions; a direct retry of the completion cleanup path is the narrower maintainable fix.
