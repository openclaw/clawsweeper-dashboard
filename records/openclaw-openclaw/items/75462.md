## Summary

Keep open: the PR targets a real swallowed-error path, but the current recovery still misses the main failure window because it schedules orphan recovery after the run has already been marked ended, and normal ended runs are skipped by that recovery scan. The PR also still lacks after-fix real behavior proof.

## What This Changes

The PR changes `waitForSubagentCompletion` to log completion-handler failures and schedule orphan recovery instead of silently ignoring caught errors.

## Best Possible Solution

Retry the already-ended completion cleanup/announce path directly when `completeSubagentRun` fails, keep the warning log, add focused regression coverage, and require redacted real behavior proof before merge.

## Reproduction Assessment

Yes, source-level: make `waitForAgentRun` return `ok` and make `completeSubagentRun` reject after `endedAt` is persisted; current main's empty catch swallows that failure. I did not execute tests because this review is read-only.

## Solution Assessment

No. Logging is useful, but orphan recovery is not the right retry mechanism for normal already-ended records because that scanner skips them; the narrower fix is to retry or reschedule completion cleanup directly.
