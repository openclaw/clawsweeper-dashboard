Keep this PR open: it targets a real swallowed-error path, but the proposed recovery mechanism does not retry the already-ended completion failure it describes, and the external PR still lacks after-fix real behavior proof.

## What This Changes

The PR replaces an empty `waitForSubagentCompletion` catch with warning logging and a delayed orphan-recovery schedule in `src/agents/subagent-registry-run-manager.ts`.

## Best Possible Solution

Keep the warning, but retry or reschedule the completion cleanup/announce path directly for already-ended runs, add focused regression coverage, and require redacted real behavior proof before merge.

## Reproduction Assessment

Yes, source-level: make `waitForAgentRun` return `ok` and make `completeSubagentRun` reject after `endedAt` is persisted; current main's empty catch swallows that failure. I did not execute tests because this review is read-only.

## Solution Assessment

No. Logging is useful, but orphan recovery is not the right retry mechanism for normal already-ended records because that scanner skips them; the narrower maintainable fix is to retry completion cleanup/announce directly.
