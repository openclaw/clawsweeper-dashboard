## Summary

Keep open: the patch improves diagnostics but does not recover the main failure it describes, because the scheduled orphan recovery skips normal runs that already have `endedAt`. It also needs real behavior proof before merge.

## What This Changes

The PR replaces the empty `waitForSubagentCompletion` catch with a warning log and a scheduled orphan-recovery pass when completion handling throws.

## Best Possible Solution

Retry the ended-run completion cleanup path directly when `completeSubagentRun` fails after terminal state is persisted, then add focused regression coverage and a changelog entry.

## Reproduction Assessment

Yes, source-level: make `waitForAgentRun` return `ok` and make `completeSubagentRun` reject after the run manager has persisted `endedAt`; current main's empty catch swallows that error. I did not execute tests because this review was read-only.

## Solution Assessment

No. Logging the failure is useful, but orphan recovery is not the right recovery mechanism for normal already-ended records; a direct retry of completion cleanup is the narrower maintainable fix.
