## What This Changes

The PR changes `src/agents/subagent-registry-run-manager.ts` to warn when `waitForSubagentCompletion` catches an error and then schedule orphan recovery.

## Best Possible Solution

Keep the warning, retry or reschedule the already-ended completion cleanup/announce path directly, and add focused regression coverage for that failure mode.

## Reproduction Assessment

Yes, source-level: make `waitForAgentRun` return `ok` and make `completeSubagentRun` reject after `entry.endedAt` is persisted; current main's empty catch swallows that failure. I did not run tests because this review is read-only.

## Solution Assessment

No. Logging is useful, but orphan recovery skips normal already-ended records, so the narrow maintainable fix is to retry the completion cleanup/announce path directly instead of relying on orphan recovery.
