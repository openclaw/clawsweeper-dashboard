## Summary

Keep open: the PR improves observability, but routing every caught completion failure to orphan recovery still misses failures that happen after terminal state is persisted, and the user-facing fix also needs a changelog entry.

## What This Changes

The branch replaces the empty `waitForSubagentCompletion` catch block with warning logging and a scheduled orphan-recovery pass.

## Best Possible Solution

Keep the warning log, but recover post-terminal failures by rearming or retrying the ended-run cleanup/announce path, with focused regression coverage and a single-line active changelog fix entry.

## Reproduction Assessment

Yes. A focused unit test can make `waitForAgentRun` return `ok`, then make cleanup or completion setup throw after `endedAt` and outcome are persisted; the proposed orphan recovery then skips the already-ended run.

## Solution Assessment

No. Logging is appropriate, but orphan recovery is for restart-aborted unended sessions, while this failure needs an ended-run cleanup/announce retry path such as the existing `resumeSubagentRun` flow.
