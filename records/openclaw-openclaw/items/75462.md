## Summary

Keep open: the PR targets a real swallowed-error path, but the proposed recovery does not retry the main already-ended completion failure mode and the external PR still lacks after-fix real behavior proof.

## What This Changes

The PR changes `waitForSubagentCompletion` to log caught completion failures and schedule orphan recovery instead of silently ignoring them.

## Best Possible Solution

Keep the warning log, retry or directly reschedule the already-ended completion cleanup/announce path when `completeSubagentRun` fails, add focused regression coverage, and require redacted real behavior proof before merge.

## Reproduction Assessment

Yes, source-level: make `waitForAgentRun` return `ok` and make `completeSubagentRun` reject after `endedAt` is persisted; current main's broad catch discards that failure. I did not run tests because this review is read-only.

## Solution Assessment

No. Logging is useful, but orphan recovery is not the right retry mechanism for normal already-ended records because that scanner skips them; the narrower maintainable fix is to retry or reschedule completion cleanup directly.
