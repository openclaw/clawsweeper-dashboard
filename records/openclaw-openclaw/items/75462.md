## Summary

Keep open: the PR addresses a real swallowed-error path, but the proposed recovery can still miss failures after terminal state is persisted because orphan recovery skips normal ended runs; it also lacks the required changelog entry.

## What This Changes

The PR replaces the empty `waitForSubagentCompletion` catch with warning logging and a scheduled orphan-recovery pass.

## Best Possible Solution

Keep the warning log, but recover terminal completion failures through an ended-run cleanup/announce retry path with focused regression coverage and a single-line active changelog fix entry.

## Reproduction Assessment

Yes. A focused unit reproduction can make `waitForAgentRun` return `ok`, then make cleanup/runtime retirement reject after `endedAt` and outcome are persisted; the proposed orphan recovery then skips that ended run.

## Solution Assessment

No. Logging is the right direction, but orphan recovery is not the right mechanism for normal ended-run cleanup or announce setup failures; reusing or hardening the ended-run retry path is narrower.
