## Summary

Keep open: the PR improves diagnostics, but its recovery path still cannot recover a normal completion failure after the run has already been marked ended, and it is missing the required changelog entry.

## What This Changes

The branch replaces an empty `waitForSubagentCompletion` catch block with warning logging and a call to `scheduleOrphanRecovery`.

## Best Possible Solution

Keep the warning log, but route failures after terminal state persistence through an ended-run cleanup/announce retry or rearm path, with focused regression coverage and a single-line active changelog fix entry.

## Reproduction Assessment

Yes. A focused unit reproduction can make `waitForAgentRun` return `ok`, then make `completeSubagentRun` reject after `endedAt` and outcome are persisted; the proposed orphan recovery then skips the already-ended run.

## Solution Assessment

No. Logging is the right direction, but orphan recovery is not the right mechanism for normal ended-run cleanup or announce failures; a lifecycle retry/rearm path is narrower and matches the existing cleanup retry design.
