## Summary

Keep open. Current main still has the reported Feishu string-path upload cleanup gap, and the PR is a narrow, plausible fix, but the branch is unmergeable/stale against current main and needs focused regression coverage before it should land.

## What This Changes

The PR wraps Feishu image and file upload SDK calls in try/finally blocks so ReadStreams created from string path inputs are destroyed after upload success or failure.

## Best Possible Solution

Land a current-main version of the Feishu-owned fix that keeps Buffer upload semantics unchanged, destroys only OpenClaw-created ReadStreams on all upload exits, and adds narrow rejection-path tests for both image and file uploads.

## Solution Assessment
