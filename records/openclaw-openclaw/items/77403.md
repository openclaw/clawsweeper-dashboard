## What This Changes

The PR adds a bundled `payment` plugin with Stripe Link virtual cards, HTTP 402 machine payments, a payment CLI, docs, tests, hook allowlist entries, labeler metadata, and a changelog entry.

## Best Possible Solution

Publish and harden the payment capability as an external ClawHub/npm code plugin first, then bring back only narrow generic SDK gaps or an official-promotion proposal with explicit maintainer and security ownership.

Not applicable as a product feature PR rather than a bug report. The review findings are source-reproducible from the PR branch files and lockfile diff.

## Solution Assessment

No. Bundling this optional commercial payment integration in core is not the narrowest maintainable path because current main already supports external plugin tools, CLI commands, hooks, browser fill interception, approvals, and ClawHub distribution.
