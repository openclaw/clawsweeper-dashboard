## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Raise virtual-card approval to critical:** `extensions/payment/src/approvals.ts:135`
  - body: The PR says every money-moving action uses critical approval, but issue_virtual_card returns severity "warning". Issuing a Stripe Link spend request is a live money-moving step, so this weakens the approval signal users see before card issuance.
  - confidence: 0.86
- **[P2] Add payment runtime deps to the lock importer:** `pnpm-lock.yaml:1155-1161`
  - body: extensions/payment/package.json declares runtime dependencies on typebox and zod, but the new pnpm-lock importer only records the plugin-sdk devDependency. Frozen installs can reject this as an out-of-sync lockfile or omit the package runtime dependency metadata.
  - confidence: 0.88
- **[P2] Align provider config schema with runtime parsing:** `extensions/payment/openclaw.plugin.json:17-20`
  - body: The manifest accepts an enabled payment config without a provider, but parsePaymentConfig requires provider and index.ts falls back to the disabled default after logging. That lets control-plane validation pass while startup silently disables the configured plugin.
  - confidence: 0.8
