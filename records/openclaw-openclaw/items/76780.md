## Summary

Keep open: current main still has the source-level partial-registry reuse path, and the linked closing PR is open and unmerged, so the issue should wait for that PR or an equivalent fix to land.

## What This Changes

Make scoped plugin tool registry reuse require coverage for every selected plugin id so allowlisted plugin tools are not skipped.

## Best Possible Solution

Review and land #76781, or an equivalent narrow fix with regression coverage, then close this issue as implemented.

## Reproduction Assessment

Yes. Source inspection shows a high-confidence path where a concrete multi-plugin `runtimePluginIds` request can reuse a registry once any requested plugin has tools, preventing cold-load of the missing selected plugin tools.

## Solution Assessment

Yes. Requiring `registryHasScopedPluginTools` to cover every concrete selected plugin id is the narrow maintainable fix; the open linked PR appears to target that exact boundary and should be reviewed before closing.
