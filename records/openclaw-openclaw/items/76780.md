## Summary

Keep open: current main still reuses a scoped plugin-tool registry when any requested plugin id has tools, and #76781 is already the open linked implementation path for this bug.

## What This Changes

The issue asks OpenClaw to require existing plugin-tool registries to cover every requested plugin id before reuse so missing selected plugin tools are cold-loaded.

## Best Possible Solution

Review and land #76781 or an equivalent narrow fix that requires full requested plugin-id coverage before registry reuse, with regression coverage for wildcard-expanded multi-plugin tool scopes.

## Reproduction Assessment

Yes, at source level: current main returns a scoped registry when any requested plugin id has tools, so a multi-plugin request can reuse a registry that lacks another selected plugin's tools. I did not execute a live Feishu/Lark run in this read-only pass.

## Solution Assessment

Yes: requiring full requested plugin-id coverage before registry reuse is the narrow generic fix and preserves the existing cold-load path for missing selected tools.
