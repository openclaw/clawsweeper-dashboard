## Summary

Keep open: current main still accepts a scoped plugin-tool registry when any requested plugin id has tools, and open draft PR #76781 is already linked as the implementation path.

## What This Changes

Require plugin tool registry reuse to cover every requested selected plugin id so missing plugin tools are cold-loaded instead of skipped.

## Best Possible Solution

Review and land #76781 or an equivalent narrow fix with regression coverage for wildcard-expanded multi-plugin tool scopes, then let the linked issue close after merge.

## Reproduction Assessment

Yes, at source level: current main returns a scoped registry when any requested plugin id has tools, so a multi-plugin request can reuse a registry that lacks another requested plugin's tools. No live Feishu/Lark run was executed in this read-only pass.

## Solution Assessment

Yes: requiring full requested plugin-id coverage before registry reuse is the narrow generic fix and preserves the existing cold-load path for missing selected tools.
