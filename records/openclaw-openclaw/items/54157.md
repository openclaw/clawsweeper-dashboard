## Summary

Keep open. Current main still ships zero cache-read and cache-write pricing in the bundled Volcengine/Doubao and BytePlus model catalogs, and open PR #55913 is already the closing implementation path.

## What This Changes

The issue asks OpenClaw to replace zero cache-read and cache-write prices in the bundled Doubao/BytePlus catalogs with vendor-backed context caching prices.

## Best Possible Solution

Keep this issue open and land, update, or replace PR #55913 with vendor-backed USD-per-million-token cache prices in the provider-owned manifests, focused regression coverage, and a user-facing changelog entry.

## Reproduction Assessment

Yes. Source inspection shows current main stores zero cacheRead/cacheWrite prices for the affected catalogs and estimateUsageCost multiplies cached-token counts by those exact fields, so any normalized cached usage is undercounted without needing a live provider call.

## Solution Assessment

Yes. The narrow maintainable path is to keep this issue open while PR #55913 or a replacement updates the provider-owned catalog data rather than changing the generic cost estimator.
