## Summary

Keep open. Current main and v2026.5.20 still carry zero cache pricing in the active Volcengine/Doubao and BytePlus manifests, the runtime cost path uses those fields directly, and https://github.com/openclaw/openclaw/pull/55913 is already the open closing-PR implementation candidate.

## What This Changes

Fill Doubao/BytePlus cacheRead and cacheWrite model-catalog pricing so cached-token usage is not estimated as free.

## Best Possible Solution

Keep this issue open and review, retarget, or replace https://github.com/openclaw/openclaw/pull/55913 so the active Volcengine and BytePlus plugin manifests get vendor-backed cache pricing with focused regression coverage.

## Reproduction Assessment

Yes, source-level. Current main shows active Volcengine and BytePlus manifest rows with zero cache prices, the manifest loader preserves those fields, and estimateUsageCost multiplies cached usage by them.

## Solution Assessment

Yes for the issue direction, but the maintainable fix belongs in the active plugin manifests rather than the deleted legacy src/agents files. The pricing values should be vendor-backed per model or tier where the vendor uses tiered billing.
