## Summary

Keep open: current main and the latest release still carry zero cache pricing in active Volcengine/Doubao and BytePlus plugin catalogs, and the runtime cost estimator uses those fields directly. The earlier same-repo fix PR is closed unmerged, while the remaining open fork PR is conflicting and targets deleted legacy files.

## What This Changes

Update the Doubao/Volcengine and BytePlus model catalogs so cacheRead/cacheWrite pricing reflects context-cache costs instead of zeros.

## Best Possible Solution

Land a narrow provider-catalog fix against the current Volcengine and BytePlus plugin manifests with vendor-backed USD-per-million-token cache prices and focused regression coverage.

## Reproduction Assessment

Yes at source level: current manifests still carry zero cache prices, the shared catalog loader preserves those values, and the estimator multiplies cached-token usage by them. I did not run a live vendor session in this read-only review.

## Solution Assessment

Yes, the best fix is a narrow current-manifest pricing update plus focused regression tests. The closed same-repo PR and open fork PR are not good landing paths because they target deleted core files and conflict with current main.
