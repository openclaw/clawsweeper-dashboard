## Summary

Keep open. Current main still has zero cache-read and cache-write prices in the bundled Volcengine/Doubao and BytePlus catalogs, and the runtime cost estimator uses those fields directly. The issue is already paired with open unmerged PR #55913, so closing or queuing a separate repair before that PR is resolved would be premature.

## What This Changes

Doubao/Volcengine and BytePlus model catalogs should use vendor-backed non-zero cache-read and cache-write pricing for context caching cost estimates.

## Best Possible Solution

Review, update, or replace PR #55913 as the implementation path. The merged fix should update the bundled Volcengine and BytePlus plugin manifests plus compatibility constants with current USD-per-million-token cache prices, and add focused regression coverage preventing those catalog values from returning to zero.
