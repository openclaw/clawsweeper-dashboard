## Summary

Keep open: current main and the latest release still preserve zero cached-token prices in the active BytePlus and Volcengine/Doubao plugin manifests, and the only open closing-reference PR is a conflicting fork PR that edits deleted core catalog files rather than the current manifest path.

## What This Changes

The issue asks OpenClaw to replace zero Doubao/BytePlus cacheRead/cacheWrite catalog prices with vendor-backed context-caching costs so cached-token usage is not counted as free.

## Best Possible Solution

Land a narrow provider-plugin manifest update for both extensions/byteplus and extensions/volcengine with vendor-backed USD-per-million cache pricing, focused regression tests, and a decision on cache storage/write mapping.

## Reproduction Assessment

Yes, at source level. Current main still contains zero cache prices in active BytePlus and Volcengine/Doubao manifests, and the runtime cost estimator directly multiplies cached-token usage by those fields.

## Solution Assessment

Yes on the layer, but unclear on exact values. The maintainable fix belongs in the current plugin manifests and tests, while the old src/agents catalog path should not be revived and vendor pricing/mapping needs maintainer-confirmed proof.
