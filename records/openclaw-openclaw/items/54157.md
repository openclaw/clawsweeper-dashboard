## Summary

Keep open. Current main still has zero cache-read and cache-write pricing for the bundled Volcengine/Doubao and BytePlus catalog rows, and the runtime cost estimator uses those exact fields. Open PR #55913 already references this issue as its implementation target, so closing this issue or queueing a separate repair job would be premature.

## What This Changes

Requests non-zero context-caching cost values for Doubao/Volcengine and BytePlus model catalogs so cached-token usage is not estimated as free.

## Best Possible Solution

Review, update, or replace PR #55913 as the implementation path. The merged fix should update the provider-owned Volcengine and BytePlus plugin manifest catalog costs, any remaining exported fallback constants that feed live/default configs, and focused regression coverage, with vendor-backed USD-per-million-token prices and a changelog entry for the user-facing cost-tracking fix.

## Reproduction Assessment

Yes. Inspecting current main shows zero cache prices in the bundled provider manifests, and the runtime path preserves those values and multiplies cached-token counts by them; no live provider call is needed to reproduce the estimated-cost understatement once normalized usage includes cacheRead or cacheWrite.

## Solution Assessment

Yes. Keeping this issue open while the closing PR is reviewed is the narrowest maintainable path; the fix belongs in the provider-owned plugin catalog data rather than in the generic cost estimator.
