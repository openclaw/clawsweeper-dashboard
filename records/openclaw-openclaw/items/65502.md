## Summary

Keep open. Current main still lacks the requested configurable per-candidate retry policy and context-aware safe mode, and the remaining work needs maintainer design because it would add public fallback config plus new replay and compaction semantics.

## What This Changes

Request to add configurable model fallback retries plus a safe context mode that compacts oversized session history before trying smaller fallback models.

## Best Possible Solution

Design an explicit fallback policy that adds bounded replay-safe retries, schema/docs coverage, structured retry logs, and guarded candidate-aware compaction using the target fallback model's known context budget.

## Reproduction Assessment

Yes, source-level. Current main shows the fallback runner calls each candidate once, the model schema rejects the proposed policy fields, and context-overflow errors are rethrown before fallback selection can handle them.

## Solution Assessment

Unclear. The requested direction is plausible, but the exact retry defaults, retryable error classes, and safe-mode compaction persistence need maintainer design before implementation.
