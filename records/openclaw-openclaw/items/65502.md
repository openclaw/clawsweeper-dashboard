## Summary

Keep open. Current main still lacks configurable per-candidate model fallback retry and fallback context safe/strict policy: the fallback loop attempts each candidate once, the public agent model schema has no requested policy fields, and escaped context-overflow errors still stop the model fallback chain.

## What This Changes

The issue requests configurable per-candidate model fallback retries plus a context-aware safe mode that can compact oversized history before trying smaller fallback models.

## Best Possible Solution

Design this in the core agent fallback path with a documented public config shape, bounded retries for classified transient failures, replay-safety rules, candidate context-window preflight, and explicit safe/strict behavior that reuses the existing compaction engine only when the target fallback budget is known.

## Reproduction Assessment

Yes. A focused unit check can configure a primary plus fallback, make the primary throw a transient timeout/rate-limit/server error, and assert the primary is called once before fallback; a second check can throw a context-overflow-shaped error and assert the fallback loop rethrows it while the strict schema rejects the proposed policy fields.

## Solution Assessment

Unclear. The requested direction is plausible, but the exact retry and safe-mode semantics need maintainer design before a narrow implementation can be judged as the best solution.
