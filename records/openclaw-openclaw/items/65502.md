## Summary

Keep open. Current main and v2026.5.18 still lack configurable per-candidate retry and context-aware safe fallback, and the related open items cover only parts of the request rather than replacing the combined policy question.

## What This Changes

The issue asks for configurable retries on transient model failures and a safe fallback mode that compacts context before trying smaller fallback models.

## Best Possible Solution

Design one coordinated fallback policy that composes bounded retry budgets with guarded context-window-aware fallback or compaction, then update schema, docs, logs, and focused regression coverage.

## Reproduction Assessment

Yes, source-level: current main attempts each fallback candidate once, rethrows context-overflow errors out of the fallback chain, and the strict agent model schema has no fields for the proposed retry or safe/strict policy.

## Solution Assessment

Unclear. The direction is plausible, but retry defaults, retryable error classes, public config shape, and safe compaction/replay behavior need maintainer product design before implementation.
