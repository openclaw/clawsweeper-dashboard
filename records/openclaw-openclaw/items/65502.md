## Summary

Keep open. Current main and the latest release still lack configurable per-candidate retry and context-aware safe fallback, and the remaining design affects public config, latency/cost policy, replay safety, and compaction semantics.

## What This Changes

Request configurable model fallback retries plus a safe context mode that can compact or degrade when falling back to smaller-context models.

## Best Possible Solution

Design one coordinated fallback policy that composes bounded retry budgets with guarded context-window-aware fallback or compaction, then update schema, docs, logs, and focused regression coverage alongside the narrower open retry and overflow issues.

## Reproduction Assessment

Yes, source-level. Current main attempts each fallback candidate once, rejects the proposed fallback policy fields, and keeps overflow inside active-model compaction/retry rather than candidate-aware fallback.

## Solution Assessment

Unclear. The direction is plausible, but the exact retry defaults, config layer, retryable error classes, and safe compaction/replay behavior need maintainer product design before implementation.
