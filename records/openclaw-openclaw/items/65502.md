## Summary

Keep open. Current main still does not implement configurable per-candidate model retry or a fallback context safe/strict mode: the fallback runner attempts each candidate once, the agent model schema only accepts primary/fallbacks/timeoutMs, and context-overflow errors still stay in the active runner’s compaction path instead of advancing to a smaller fallback with candidate-aware compaction.

## What This Changes

Request configurable agent model fallback retries plus a context-aware safe mode that compacts oversized session history before trying smaller fallback models.

## Best Possible Solution

Design this as an explicit agent model fallback policy: bounded retries only for classified replay-safe transient failures, observable retry events, schema/docs coverage for retry and safe/strict context behavior, and candidate-aware compaction that uses the target fallback model's known context budget without bypassing existing replay-safety rules.

## Reproduction Assessment

Yes. A focused unit reproduction can mock runWithModelFallback with a transient primary failure and fallback success, then assert the primary is called once before moving on; source inspection also shows context overflow is rethrown and the schema rejects the proposed policy fields.

## Solution Assessment

Unclear. The requested direction is plausible, but the exact safe-mode and retry semantics need maintainer design around replay safety, candidate context-window resolution, and whether emergency compaction should persist transcript changes.
