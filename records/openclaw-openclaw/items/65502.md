## Summary

Keep open. Current main and v2026.5.12 still lack configurable per-candidate retry and context-aware safe fallback; the remaining work changes public config, retry policy, context-window semantics, and replay safety, so it needs maintainer design rather than cleanup closure.

## What This Changes

Request resilient model fallback by adding configurable retry budgets and a safe/strict context mode that can compact or degrade before falling back to smaller-context models.

## Best Possible Solution

Design one coordinated fallback policy that composes bounded retry budgets with guarded context-window-aware fallback or compaction, then update schema, docs, logs, and focused regression coverage alongside the narrower context-overflow fallback issue.

## Reproduction Assessment

Yes, source-level. Current main attempts each fallback candidate once, rejects the proposed policy fields, and keeps overflow inside active-model compaction/retry instead of candidate-aware fallback.

## Solution Assessment

Unclear. The direction is plausible, but retry defaults, retryable error classes, config shape, and safe compaction/replay behavior need maintainer product design before implementation.
