## Summary

Keep open. Current main still lacks configurable per-candidate retry and candidate-aware context safe mode, and the remaining work needs maintainer design for public config shape, retry safety, and compaction persistence semantics.

## What This Changes

Add configurable retry and context-aware safe/strict fallback behavior for agent model fallback.

## Best Possible Solution

Design an explicit fallback policy that adds bounded replay-safe retries, schema/docs coverage, structured retry logs, and candidate-aware compaction using the target fallback model's known context budget without bypassing existing compaction safeguards.

## Reproduction Assessment

Yes. Source inspection on current main shows the fallback runner calls each candidate once, the schema rejects the proposed policy fields, and context-overflow errors are rethrown before another candidate can be tried.

## Solution Assessment

Unclear. The requested direction is plausible, but the exact retry defaults, retryable error classes, and safe-mode compaction persistence need maintainer design before implementation.
