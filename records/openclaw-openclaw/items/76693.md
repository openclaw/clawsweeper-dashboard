## Summary

Keep open: the Node-side masking direction is narrow and well-scoped, but the PR still needs maintainer review because it knowingly leaves the bundled macOS CLI wizard echo path to #76698.

## What This Changes

The PR adds a `sensitive` flag to wizard text prompts, routes sensitive Node Clack prompts through `password()`, marks onboarding credential prompts as sensitive, preserves existing gateway secrets via masked-preview confirms, and adds tests plus a changelog entry.

## Best Possible Solution

Keep the Node masking work, but either include the macOS CLI no-echo handling before merge or record explicit maintainer approval to split the residual bundled-client fix to #76698.

## Reproduction Assessment

Yes. Source inspection shows current main routes onboarding credentials through visible text prompts, and the PR body includes Windows PowerShell reproduction/screenshots; I did not run a live TTY transcript in this read-only pass.

## Solution Assessment

No as submitted. The Node Clack implementation is a maintainable fix, but the best repository-level solution must also make every bundled wizard client that receives sensitive text steps honor the flag, or have an explicit maintainer-approved split.
