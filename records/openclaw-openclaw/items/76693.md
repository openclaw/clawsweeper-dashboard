## Summary

Keep open: current main still sends onboarding credential entry through cleartext wizard text prompts, and this PR is the active implementation candidate for the Node CLI path. I found no blocking correctness or supply-chain regression in the diff; the acknowledged macOS wizard-client masking gap remains a separate follow-up.

## What This Changes

The PR adds a `sensitive` wizard text flag, dispatches sensitive Node CLI prompts through `@clack/prompts.password()`, marks onboarding credential prompts sensitive, preserves existing gateway secrets with masked-preview confirms, and adds tests plus a changelog entry.

## Best Possible Solution

Land the Node CLI masking patch if CI and maintainer security review agree, then track the macOS wizard-client masking fix separately so every first-party wizard renderer honors sensitive text steps.

## Reproduction Assessment

Yes. Source inspection on current main shows credential prompts call the cleartext wizard text path, and the clack adapter has no sensitive/password branch; I did not run an interactive TTY reproduction in this read-only review.

## Solution Assessment

Yes for the scoped Node CLI fix. Threading an optional `sensitive` flag through the existing wizard contract and using the already-pinned clack password prompt is the narrowest maintainable path; the macOS renderer should be handled as a separate targeted patch.
