## Summary

Keep open: the PR is a useful Node-side credential-masking fix, but it is not clearly ready to merge without maintainer approval because it leaves the bundled macOS CLI wizard echo path acknowledged in #76698.

## What This Changes

The PR adds a sensitive wizard text flag, routes sensitive Node Clack prompts through password input, marks onboarding credential prompts as sensitive, preserves existing tokens via masked-preview confirms, and adds tests plus a changelog entry.

## Best Possible Solution

Keep the Node masking work, but either include macOS CLI no-echo handling before merge or get explicit maintainer approval to split that residual bundled-client fix to #76698.

## Reproduction Assessment

Yes. Source inspection shows current main routes onboarding credentials through visible text prompts, and the PR body includes Windows PowerShell reproduction/screenshots; I did not run a live TTY transcript in this read-only pass.

## Solution Assessment

No as submitted. The Node Clack implementation is narrow and maintainable, but the best repository fix needs every bundled client that receives sensitive wizard steps, especially openclaw-mac wizard, to honor the flag or have an explicit maintainer-approved split.
