## Summary

Keep open: the Node CLI masking direction is useful and current `main` source shows the reported leak path, but this PR is not merge-ready because it propagates sensitive wizard steps while deliberately leaving the bundled `openclaw-mac wizard` text client echoing those steps.

## What This Changes

The PR adds a `sensitive` wizard text flag, routes sensitive Node CLI prompts through `@clack/prompts.password()`, marks onboarding credential prompts as sensitive, preserves existing gateway credentials via masked-preview confirms, and adds tests plus a changelog entry.

## Best Possible Solution

Keep the Node-side masking, but either carry the small macOS CLI no-echo handling from #76615 before merge or make an explicit maintainer-approved split with a canonical follow-up for that client.

## Reproduction Assessment

Yes from source: current main credential prompts use `prompter.text()` and the Clack adapter renders text input visibly, while the macOS CLI session client also reads text steps with echoing `readLine()`. I did not run a live terminal transcript in this read-only pass.

## Solution Assessment

No as submitted. The Node Clack implementation is a narrow maintainable fix, but the best repository fix needs every bundled client that receives `sensitive` wizard steps, especially `openclaw-mac wizard`, to honor the flag before claiming CLI credential prompts are masked.
