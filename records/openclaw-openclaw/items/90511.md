The PR adds a Discord read-target fallback that reads fetched one-to-one DM metadata and allows reads when the single recipient matches `allowFrom`, plus mocked tests for allowlist forms and DM-channel-id rejection.

Full review comments:

- **[P1] Use the effective DM allowlist for reads:** `extensions/discord/src/actions/runtime.messaging.shared.ts:221-225`
  - body: `mergeDiscordAccountConfig` leaves inherited root `allowFrom` on named accounts even when that account has legacy `dm.allowFrom`, while existing Discord accessors preserve account/legacy precedence.
