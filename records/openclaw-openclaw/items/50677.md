## Summary

Keep open. Current main and v2026.5.22 still use deterministic alphabetical prompt truncation with only a count-based model-facing warning, while the requested omitted-skill recovery, priority/pinning, and user-visible warning semantics have not landed; the open omitted-key PR and separate priority issue cover only parts of the request.

## What This Changes

The issue asks OpenClaw to make skills prompt truncation visible and controllable so user-created or critical skills are not silently omitted from the agent context.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/42637 for exact omitted-skill recovery, document truncation behavior, and make a maintainer product/API decision on priority, pinning, and user-visible warnings, with https://github.com/openclaw/openclaw/issues/50199 as the canonical broader priority discussion.

## Reproduction Assessment

Yes, at source level. Current main sorts prompt-facing skills alphabetically, applies count and character budgets, and emits only a count-based model-facing note when entries are omitted; I did not run tests because this review is read-only.

## Solution Assessment

Unclear as a complete solution. Omitted lookup keys are the narrow recovery path, while priority, pinning, required skills, and user-visible warnings need maintainer product/API direction before implementation.
