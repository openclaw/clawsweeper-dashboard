Keep open. ## What This Changes

The issue asks OpenClaw to make skills prompt truncation visible and controllable so user-created skills are not silently omitted from the agent-visible skills list.

## Best Possible Solution

Ship or replace the focused omitted-key recovery path, document the budget and truncation outcome in public docs, and make a deliberate maintainer decision on priority/pinning within the existing skills config contract.

## Reproduction Assessment

Yes, at source level. Current main sorts prompt-facing skills alphabetically, applies count and character budgets, and emits only a count-based model-facing warning when skills are omitted.

## Solution Assessment

Unclear as a complete solution because the issue bundles discoverability, documentation, priority/pinning, and warning-surface choices. The open omitted-key PR is the focused path for lookup recovery, while priority and user-visible warning behavior need maintainer product/API direction.
