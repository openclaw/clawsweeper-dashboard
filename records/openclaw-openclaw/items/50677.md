## Summary

Keep open. Current main and v2026.6.8 still drop prompt-visible skills to a fitting alphabetical prefix under `maxSkillsPromptChars`, the truncation warning remains count-only, and the priority/pinning part of the request is still an unresolved config/product decision.

## What This Changes

Request to stop silently dropping eligible user skills from the model-facing skills prompt and add warning, recovery, priority, or pinning behavior so critical skills survive truncation.

## Best Possible Solution

Land or replace the omitted-skill recovery path, document the skills prompt budget outcome, and make an explicit maintainer decision on whether priority or pinning belongs in the canonical skills config contract.

## Reproduction Assessment

Yes. Source inspection shows current main still drops skills from the prompt when compact output exceeds `maxSkillsPromptChars`, and tests assert the included/total truncation path.
