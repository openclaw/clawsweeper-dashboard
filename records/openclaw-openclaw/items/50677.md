## Summary

Keep open. Current main still truncates prompt-facing skills to a fitting prefix with a model-facing count-only warning and no omitted-skill lookup keys, priority/pinning contract, or user-visible warning path; the narrow omitted-name PR and the broader priority issue are both still open.

## What This Changes

The issue asks OpenClaw to stop silently losing prompt-facing skills, expose what was omitted, and add priority or pinning controls for critical user skills.

## Best Possible Solution

Keep this open until omitted-skill recovery lands or is rejected, the docs explain the truncation outcome, and maintainers decide whether priority/pinning belongs in the skills config contract.

## Reproduction Assessment

Yes, at source level. Current main sorts prompt skills alphabetically, slices to count and character budgets, and emits only a count-based truncation note, so the reported invisible-skill behavior is source-verifiable without mutating the checkout.

## Solution Assessment

Unclear as a complete solution because the issue bundles a narrow recovery fix with broader API choices. The open omitted-key PR looks like the right narrow path for discovery, while priority/pinning and user-visible warnings need maintainer product design before implementation.
