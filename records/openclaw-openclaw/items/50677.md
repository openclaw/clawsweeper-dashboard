## Summary

Keep open. Current main and the latest release still truncate prompt-facing skills to an alphabetic fitting prefix with only a count-based model-facing note, while the open omitted-key PR and the separate priority issue cover only parts of the requested visibility and prioritization behavior.

## What This Changes

Request to make skills prompt truncation visible and controllable so eligible user-created skills are not silently absent from the agent context.

## Best Possible Solution

Review and land or replace the omitted-key recovery PR, document the truncation budget and outcome, and make an explicit maintainer decision on priority/pinning and user-visible warning semantics before adding new config or channel-specific behavior.

## Reproduction Assessment

Yes, at source level. Current main sorts prompt-facing skills alphabetically, applies count and character budgets, and emits only a count-based model-facing note when skills are omitted; I did not run tests because this review is read-only.

## Solution Assessment

Unclear as a complete solution. Omitted lookup keys are the narrow recovery path, while priority, pinning, required skills, and user-visible warnings need maintainer product/API direction.
