## Summary

Keep this open. Current main still truncates the prompt-facing skills list by count and character budget, returns only the included count and compact-format flag in the model prompt, and has no omitted-skill recovery, operator-visible warning, or priority/pinned/required-skill config contract. The related omitted-name PR #42637 and broader priority issue #50199 remain the right follow-up paths rather than a close reason.

## What This Changes

Add visibility and prioritization so eligible user-created skills are not silently omitted from the runtime skills prompt without an operator-visible warning or a way to keep critical skills loaded.

## Best Possible Solution

Keep this open until OpenClaw ships an omitted-skill discovery or warning path, documents the skills prompt budget and truncation outcome on the public docs, and explicitly decides whether priority, pinning, or context-required skills belong in the skills config contract or remain tracked by narrower follow-ups.

## Reproduction Assessment

Yes. A high-confidence reproduction path is to build a workspace skills prompt with many prompt-visible skills and a low skills.limits.maxSkillsPromptChars or maxSkillsInPrompt; existing tests already assert count and character truncation, including an included-0-of-3 case.

## Solution Assessment

No. Current main is not the best resolution because it preserves only a count-only model-facing notice; #42637 is a plausible narrow omitted-name improvement, while operator-visible warning and priority/pinning semantics need maintainer API/product review.
