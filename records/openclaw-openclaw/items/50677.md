Keep open. Current main and latest release still truncate the model-facing skills catalog to a fitting alphabetical prefix with only a count-based prompt note, while omitted-skill recovery and priority/pinning direction remain unresolved.

## What This Changes

The issue requests that skills omitted by prompt-budget truncation become visible or recoverable and that users can prioritize or pin critical skills.

## Best Possible Solution

Land or replace the omitted-skill recovery path, document the prompt-budget behavior, and make an explicit maintainer decision on whether priority or pinning belongs in the canonical skills config contract.

## Reproduction Assessment

Yes. Current main still slices the prompt-visible skills list under `maxSkillsPromptChars` and emits only a count-based warning; I verified this through the source path and existing truncation tests rather than a live model run.
