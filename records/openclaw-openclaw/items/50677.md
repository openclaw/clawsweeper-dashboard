## Summary

Keep open. Current main and the latest release still truncate prompt-facing skills to a fitting alphabetical prefix with only a count-based model-facing note; the linked omitted-key PR and separate priority issue cover useful pieces, but the requested user-visible warning and priority/pinning behavior have not landed.

## What This Changes

The issue requests visible truncation diagnostics and prioritization or pinning so critical user-created skills are not silently omitted from the agent-visible skills catalog.

## Best Possible Solution

Keep this issue open until OpenClaw lands or replaces omitted-skill recovery, documents the truncation outcome, and makes an explicit maintainer decision on priority/pinning and user-visible warning semantics.

## Reproduction Assessment

Yes, at source level. Current main sorts prompt-facing skills alphabetically, applies count and character budgets, and emits only a count-based model-facing note when skills are omitted; I did not run tests because this review is read-only.

## Solution Assessment

Unclear as a complete solution. Omitted lookup keys are the narrow recovery path, while priority, pinning, required skills, and user-visible warnings need maintainer product/API direction.
