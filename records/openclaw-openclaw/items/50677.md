## Summary

Keep open. Current main still drops prompt-facing skills to a fitting prefix and emits only a generic count warning, while the narrow omitted-key fix and the broader priority/config decision remain open and unmerged.

## What This Changes

Request that skills prompt truncation stop silently hiding user-created skills by adding omitted-skill visibility or recovery, optional priority/pinning/context-aware loading, and documentation.

## Best Possible Solution

Land or replace the omitted-skill-key recovery, document truncation behavior publicly, and make a maintainer decision on whether priority/pinning becomes a supported skills config contract.

## Reproduction Assessment

Yes for the core source path. `buildWorkspaceSkillsPrompt()` with many prompt-visible skills and a low `maxSkillsPromptChars` still sorts by name, slices to the largest fitting prefix, and emits only a generic count warning; I did not run a live model turn.

## Solution Assessment

Unclear as a single all-in-one change. Omitted-key recovery is a narrow and likely best near-term fix, but priority, pinning, per-topic forced skills, and user-visible warning behavior need maintainer product/API review.
