## Summary

Keep open. Current main and the latest release still truncate prompt-facing skills to a fitting alphabetical prefix and emit only a model-facing included/total warning; the focused omitted-key PR is still open at https://github.com/openclaw/openclaw/pull/42637, and the broader priority contract remains open at https://github.com/openclaw/openclaw/issues/50199.

## What This Changes

The issue asks OpenClaw to stop silently dropping eligible skills from the model-facing prompt when the skills catalog exceeds prompt limits, and to add visibility or prioritization for omitted skills.

## Best Possible Solution

Keep this open until omitted-skill lookup recovery lands or is rejected, docs explain truncation outcomes, and maintainers decide whether priority/pinning belongs in the skills config contract.

## Reproduction Assessment

Yes, at source level. Current main sorts prompt skills alphabetically, applies count and character budgets, and emits only a count-based model-facing truncation note, so the invisible omitted-skill behavior is source-verifiable without mutating the checkout.

## Solution Assessment

Unclear as a complete solution because the issue bundles a narrow discovery fix with broader API/UX choices. The open omitted-key PR looks like the right narrow path for discoverability, while priority/pinning and user-visible warnings need maintainer product design.
