## Summary

Keep open. Current main and the latest release still truncate the prompt-facing skills catalog to a fitting prefix, emit only a count-based model-facing warning, and have no supported priority or pinning contract; related work is useful but remains open and partial.

## What This Changes

The issue asks OpenClaw to stop silently dropping eligible user skills from agent context when the skills prompt exceeds its budget, with visibility, prioritization, pinning, or docs as possible remedies.

## Best Possible Solution

Keep this issue open until OpenClaw lands an omitted-skill recovery path, documents the prompt budget outcome, and explicitly decides whether priority or pinning belongs in the canonical skills config contract.

## Reproduction Assessment

Yes. Source inspection shows current main still drops skills from the prompt-facing list when compact output exceeds `maxSkillsPromptChars`, and tests cover the resulting included/total truncation warning.

## Solution Assessment

Unclear. Listing omitted skill keys is a plausible narrow fix, but priority, pinning, per-topic required skills, and user-visible warning behavior are product/API choices that need maintainer direction.
