## Summary

Keep open: current main and v2026.5.18 still use an alphabetically sorted prompt prefix with a count-only model-facing truncation note, while the omitted-key fix at https://github.com/openclaw/openclaw/pull/42637 and priority contract at https://github.com/openclaw/openclaw/issues/50199 remain open.

## What This Changes

The issue asks OpenClaw to make skills prompt truncation visible and actionable, and to add a way to prioritize or force-load critical user skills.

## Best Possible Solution

Land or replace the omitted-key recovery path, document truncation outcomes on the public skills docs, and decide a first-class priority/pinning/required-skill contract in the existing skills configuration model.

## Reproduction Assessment

Yes, at source level. Current main sorts prompt skills alphabetically, applies count and character budgets, and emits only a count-based model-facing warning, so omitted user skills can remain undiscoverable without mutating the checkout.

## Solution Assessment

Unclear as a complete solution. The focused omitted-key PR is the right narrow direction for discoverability, but priority/pinning and user-visible warnings need maintainer product/API decisions before a durable implementation.
