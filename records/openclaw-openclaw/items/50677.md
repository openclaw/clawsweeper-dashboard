## Summary

Keep this open. Current main still truncates the model-facing skills catalog by count and character budget, only reports a count in the injected Skills prompt, and has no priority, pinned, context-required, or operator-visible warning contract; the related omitted-name PR and broader priority issue remain open.

## What This Changes

The issue requests visible and configurable handling for skills prompt truncation so user-created skills are not silently omitted from agent-visible skills.

## Best Possible Solution

Keep the issue open until OpenClaw ships an omitted-skill discovery or operator-visible warning path, documents the skills prompt budget and truncation outcome, and decides whether priority, pinning, or context-required skills belong in the skills config contract.

## Reproduction Assessment

Yes. Source inspection and existing tests show a high-confidence path: build a workspace skills prompt with many eligible skills or a small `skills.limits.maxSkillsPromptChars`/`maxSkillsInPrompt`, and current main drops prompt-facing skills while reporting only `included N of M` inside the model-facing prompt.

## Solution Assessment

No. Current main is not a complete solution because it keeps recovery model-facing and count-only; #42637 is a plausible narrow omitted-name improvement, while priority/pinning and user-visible warning semantics need maintainer API/product review.
