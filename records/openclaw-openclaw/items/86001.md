Current `main` and the latest release still contain the reported Discord bare-numeric channel-target failure path, so this should stay open as a valid channel message-delivery bug rather than be swept closed.

## What This Changes

The issue reports that Discord outbound message-tool sends reject bare numeric channel IDs as ambiguous instead of treating them as channel targets when replying to Discord channels.

## Best Possible Solution

Restore Discord channel-default parsing for outbound message-tool sends in channel context while keeping DM targets explicit, then replace the current ambiguity test with regression coverage for text, component, voice, and session-derived channel sends.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: the current test already asserts that `sendMessageDiscord` with a bare numeric target throws `Ambiguous Discord recipient`. I did not run tests because this review is read-only and must not create artifacts.

## Solution Assessment

Yes, the best fix is a narrow Discord outbound target-default repair, not a new feature or config knob. The repair should be scoped so explicit DM formats still win and bare numerics are only treated as channels where the outbound path has a channel default.
