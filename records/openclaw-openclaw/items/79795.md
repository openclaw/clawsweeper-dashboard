## Summary

Keep open: current main still has a source-level path where a failed iMessage `imsg rpc` probe can be stripped from non-sensitive health output, allowing status formatting to fall back to configured/OK while the transport is unavailable.

## What This Changes

The issue asks for iMessage health/status to report degraded/error when `imsg rpc` cannot access the Messages database because Full Disk Access is missing.

## Best Possible Solution

Propagate a redacted iMessage probe/readiness failure into non-sensitive health and status output, with regression coverage for the configured-but-unavailable path.

## Reproduction Assessment

Yes, at source level: make `probeIMessage` return `ok:false` from the `chats.list` Full Disk Access failure, then request a non-sensitive health/status snapshot. The current code can drop the failed probe payload before formatting, so the formatter can fall back to `configured`.

## Solution Assessment

Yes, the narrow maintainable fix is to carry a redacted probe/readiness failure into channel health/status instead of adding a config option or exposing raw probe payloads.
