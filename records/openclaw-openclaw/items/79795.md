## What This Changes

Report asks status/health to show degraded/error when iMessage `imsg rpc` cannot access the Messages database due to Full Disk Access instead of OK/configured.

## Best Possible Solution

Carry a redacted iMessage probe/readiness failure into both Health and Channels status rows, with regression coverage for the non-sensitive snapshot path.

## Solution Assessment

Yes. The narrow maintainable fix is to propagate a redacted failed probe/readiness state into health/status output, without exposing raw probe payloads or adding a config option.
