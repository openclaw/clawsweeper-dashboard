## Summary

Keep open: current main still has a clear source-level path where the iMessage live probe fails but non-sensitive health/status output can discard the failed probe and fall back to configured/OK.

## What This Changes

The issue asks for iMessage status and health to report a degraded/error state when `imsg rpc` fails from missing Full Disk Access.

## Best Possible Solution

Carry a redacted iMessage probe/readiness failure into both `status --deep` Health and Channels output, with regression coverage for non-sensitive snapshots.

## Reproduction Assessment

Yes, at source level: configure iMessage so `imsg rpc`/`chats.list` fails with the reported Full Disk Access error, then run `openclaw status --deep`; the current health path can run the probe but remove the failed probe payload before formatting. I did not live-run this on macOS.

## Solution Assessment

Yes. The narrow maintainable fix is to propagate a redacted failed probe/readiness state instead of treating configured state as OK; adding a config option or exposing raw probe payloads would be the wrong boundary.
