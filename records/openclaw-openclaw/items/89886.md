## What This Changes

The PR adds an `abortSignal` passthrough from `delegateCompactionToRuntime` to runtime compaction and adds focused delegate tests for forwarded and omitted signals.

## Best Possible Solution

Land this narrow bridge fix after maintainer review and fresh merge gates, while keeping broader stop-button timeout, lock-retention, and manual `/compact` cancellation work in their separate items.

## Reproduction Assessment

Yes. Current main can be source-reproduced by passing an abort signal into context-engine compaction: callers and runtime support the signal, but `delegateCompactionToRuntime` does not forward it.

## Solution Assessment

Yes. Forwarding the already-defined top-level `abortSignal` at the delegate bridge is the narrow maintainable fix because the public context-engine contract and downstream runtime already support cancellation.
