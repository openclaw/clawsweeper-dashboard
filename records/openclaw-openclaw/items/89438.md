- **[P2] Accept existing Slack ID key forms in the warning:** `extensions/slack/src/doctor.ts:56`
  - body: `resolveSlackChannelConfig` currently routes lowercase Slack IDs and `channel:<id>` keys, with adjacent tests covering both forms.   - confidence: 0.94

  - body: Slack account configs inherit top-level settings through `mergeSlackAccountConfig`, and monitor startup uses that merged `slackCfg.groupPolicy`. Here an account scope with no local `groupPolicy` is treated as `allowlist`, so `channels.slack.groupPolicy: "open"` plus `accounts.work.channels` can get a false warning. Compute the effective account policy before emitting the diagnostic.
  - confidence: 0.9
