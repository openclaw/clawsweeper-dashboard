## What This Changes

The PR snapshots plugin command metadata and realtime/provider tool schemas before validation or provider payload construction, with focused regression tests for command registration, realtime voice, OpenAI realtime, Mistral, and Anthropic.

## Best Possible Solution

Store the same validated command snapshot in both the global command table and `PluginRegistry.commands`, keep the provider snapshot policy explicit, and land with gateway plus provider regressions that cover the upgrade-sensitive paths.

## Reproduction Assessment

Yes from source inspection: an active plugin registry still stores the original command object, and `commands.list` reads that registry before the snapshotted global command table. A command whose metadata mutates or becomes unreadable after registration can still affect the gateway listing path.

## Solution Assessment

No: this is a plausible partial fix, but the best fix must share the snapshot with the active gateway registry storage path. The provider snapshotting direction is reasonable once maintainers accept the compatibility policy and the gateway path has matching coverage.
