## Summary

## What This Changes

The issue reports that MiniMax coding-plan quota fields are interpreted with inverted or incomplete semantics, causing status output to show misleading `% left` values.

## Best Possible Solution

Keep this as the canonical tracker and land a parser-level MiniMax fix that recognizes the reported remaining-percent fields, preserves the shared `% left` display contract, and covers general/video/weekly `model_remains` rows with focused tests and live contract proof.

## Solution Assessment
