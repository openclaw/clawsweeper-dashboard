## Summary

Keep this PR open, but do not merge it as-is. The benchmark proof is useful, yet the WeakMap cache changes a package-exported precheck helper so repeated calls over a mutated message identity can return stale token-pressure estimates and alter compaction decisions.

## What This Changes

The branch adds a WeakMap-backed per-message token-pressure cache to preemptive compaction, cache-focused tests, and a synthetic replay benchmark script.

## Best Possible Solution

Preserve fresh-estimate semantics on the package-exported helper, and confine memoization to an explicitly append-only internal path or a maintainer-approved immutable-message SDK contract with tests/docs.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a performance optimization PR. The performance claim has a reproducible terminal benchmark path against the production helper, but there is no correctness failure on current main to reproduce.

## Solution Assessment

No. The optimization should not change the exported precheck helper's freshness semantics unless maintainers explicitly approve an immutable-message SDK contract; a safer solution scopes caching to append-only internal callers.
