## Summary

Keep open: the PR now includes terminal benchmark proof, but the patch changes a shipped plugin SDK precheck helper from fresh per-call estimation to identity-cached estimates. That compatibility and session-state risk needs a maintainer API decision or a narrower internal cache before merge.

## What This Changes

The PR adds a WeakMap-backed per-message token-pressure cache to preemptive compaction, six identity-cache tests, and a synthetic precheck replay benchmark script.

## Best Possible Solution

Preserve fresh-estimate behavior for the shipped plugin SDK helper by keeping the identity cache internal to lifecycle-owned immutable call paths, or land a maintainer-approved public immutable-message contract with compatibility tests.

## Reproduction Assessment

Yes, at source level: current main scans every message on each estimate call, so the repeated precheck work is clear from the reducer. I did not rerun the benchmark because this review is read-only and tests may create artifacts.

## Solution Assessment

No, not as submitted: the WeakMap cache is small, but putting it inside the exported precheck helper changes shipped SDK behavior for mutable message objects. The safer path is an internal lifecycle-owned cache or an explicit maintainer-approved SDK immutability contract.
