## Summary

Keep this PR open. Current main still has the model-usage TODO the PR targets, and the proposed docs change is useful, but the wording should be revised before merge because it currently blurs portable manual script usage with the macOS-only, codexbar-gated OpenClaw skill path.

## What This Changes

The PR updates `skills/model-usage/SKILL.md` to replace the Linux support TODO with a macOS live-CodexBar note and adds input-mode bullets for macOS and Linux/other platforms.

## Best Possible Solution

Keep the focused docs improvement, but make the boundary explicit: macOS live reads use CodexBar installed via Homebrew and available as `codexbar` on PATH, while Linux/other platforms should run the bundled Python summarizer manually with `--input` and exported CodexBar JSON unless maintainers intentionally broaden the skill metadata and install support.

## Reproduction Assessment

Not applicable as a runtime bug; this is a docs clarity PR. The docs gap is high-confidence from current main because the TODO remains, while the script already exposes the portable `--input` path.

## Solution Assessment

No, not as written. The narrow docs direction is right, but the proposed wording should separate manual script usage from normal OpenClaw skill eligibility and name the macOS install/PATH expectation directly.
