## Summary

## What This Changes

The PR changes the Google Chat add-on payload parser to unwrap message, added-to-space, and removed-from-space payloads and map them to MESSAGE, ADDED_TO_SPACE, and REMOVED_FROM_SPACE events.

## Best Possible Solution

Rebase the lifecycle fix onto the current Google Chat parser, preserve card-click handling, add focused lifecycle parser tests, and attach redacted real Google Chat proof before merge.

## Reproduction Assessment

Yes from source inspection: a current-main Workspace Add-on request with `commonEventObject.hostApp: "CHAT"` and `chat.addedToSpacePayload` or `chat.removedFromSpacePayload` is not normalized, then lacks a string event type and returns HTTP 400 `invalid payload`. I did not run live Google Chat proof in this read-only review.

## Solution Assessment

No as submitted: plugin-local parser normalization is the right boundary, but this branch must be rebased onto current main, preserve `buttonClickedPayload`, add focused lifecycle tests, and provide real behavior proof before it is the best fix.
