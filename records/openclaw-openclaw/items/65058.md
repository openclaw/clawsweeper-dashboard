## Summary

Keep open. Current main and v2026.4.29 still reject lifecycle-only Google Workspace Add-on payloads, while this PR is a narrow parser fix that needs mechanical follow-up for changelog and regression coverage before merge.

## What This Changes

The PR extends the Google Chat webhook add-on wrapper parser to recognize `addedToSpacePayload` and `removedFromSpacePayload` and map them to lifecycle event types.

## Best Possible Solution

Land a narrow Google Chat parser fix that explicitly normalizes each supported add-on payload variant, preserves add-on token verification, adds focused lifecycle-envelope tests, and leaves the broader Google Chat follow-ups tracked separately.

## Reproduction Assessment

Yes. A current-main add-on envelope with `commonEventObject.hostApp: "CHAT"`, a body `authorizationEventObject.systemIdToken`, and only `chat.addedToSpacePayload` or `chat.removedFromSpacePayload` skips the wrapper transform and reaches the `invalid payload` path by source inspection.

## Solution Assessment

Yes. Fixing this in the Google Chat plugin parser is the narrow owner-module solution; the safer version keeps payload selection and event labeling in explicit branches, then adds regression tests and the required changelog entry.
