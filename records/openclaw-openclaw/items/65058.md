Keep open. Current main and the latest release still only unwrap Google Chat Workspace Add-on `messagePayload`, while this PR is a focused parser fix for valid lifecycle payloads; the blocker is missing external real-behavior proof, not an obsolete or superseded branch.

The PR changes the Google Chat webhook parser to normalize Workspace Add-on `messagePayload`, `addedToSpacePayload`, and `removedFromSpacePayload` envelopes into flat event types.

## Best Possible Solution

Land a plugin-local lifecycle parser fix with focused regression tests and redacted real Google Chat proof, while leaving the broader wildcard allowlist and reply-thread work to https://github.com/openclaw/openclaw/issues/65007.
