## Summary

Keep open. Current main and v2026.4.29 still unwrap only add-on `chat.messagePayload`, so lifecycle-only add-on space events remain a valid unresolved Google Chat bug; the PR is a narrow plausible fix but needs regression coverage and the required changelog entry before merge.

## What This Changes

The PR updates the Google Chat webhook parser to normalize `messagePayload`, `addedToSpacePayload`, and `removedFromSpacePayload` add-on envelopes into flat Google Chat event types.

## Best Possible Solution

Land the narrow Google Chat parser fix with explicit payload-variant branches, focused webhook tests for add-to-space and remove-from-space add-on envelopes, and a single-line Unreleased changelog entry while keeping the broader #65007 and #65326 work separate.

## Reproduction Assessment

Yes. On current main, a Google Workspace Add-on envelope with `commonEventObject.hostApp: "CHAT"`, body `authorizationEventObject.systemIdToken`, and only `chat.addedToSpacePayload` or `chat.removedFromSpacePayload` is not normalized and reaches the `invalid payload` path by source inspection.

## Solution Assessment

Yes. Fixing this in the Google Chat plugin parser is the narrowest maintainable solution; the safest finish is to keep payload selection and event labeling in explicit branches, then add regression coverage and changelog credit.
