## Summary

Keep open. Current main still rejects lifecycle-only Google Chat Workspace Add-on payloads, and this PR is a narrow useful fix, but it still lacks inspectable after-fix real behavior proof and focused regression tests before merge.

## What This Changes

The branch changes `extensions/googlechat/src/monitor-webhook.ts` to unwrap `chat.addedToSpacePayload` and `chat.removedFromSpacePayload` add-on envelopes and map them to `ADDED_TO_SPACE` and `REMOVED_FROM_SPACE` events.

## Best Possible Solution

Land a focused Google Chat plugin parser fix that accepts lifecycle add-on envelopes, acknowledges non-message events without agent dispatch, adds regression tests, and stays coordinated with https://github.com/openclaw/openclaw/issues/65007.

## Reproduction Assessment

Yes. ## Solution Assessment

Yes for the parser subset. The narrow fix belongs in the Google Chat plugin parser, but this PR still needs focused regression tests and redacted real behavior proof before merge.
