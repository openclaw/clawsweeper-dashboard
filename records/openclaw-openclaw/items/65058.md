## Summary

Keep open. Current main still rejects lifecycle-only Google Chat Workspace Add-on payloads, and this PR is a plausible narrow parser fix, but it needs real behavior proof and likely consolidation with broader open parser work before merge.

## What This Changes

The PR changes the Google Chat webhook parser to unwrap Workspace Add-on message, added-to-space, and removed-from-space payloads into Chat event objects.

## Best Possible Solution

Consolidate the Google Chat add-on parser fix with the active parser issue/PR cluster, add focused lifecycle-envelope regression tests, and require redacted after-fix Google Chat or terminal proof before merge.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: submit a Workspace Add-on request with `commonEventObject.hostApp: "CHAT"`, `authorizationEventObject.systemIdToken`, and only `chat.addedToSpacePayload` or `chat.removedFromSpacePayload`; current main does not unwrap it and returns `invalid payload`.

## Solution Assessment

Yes for the core direction: fixing the Google Chat plugin parser is the narrow maintainable path. The merge-ready version should include regression tests and real behavior proof, and maintainers should choose whether this PR or https://github.com/openclaw/openclaw/pull/80833 becomes the consolidated branch.
