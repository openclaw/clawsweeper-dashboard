## Summary

Keep open: current main still rejects lifecycle-only Google Chat Workspace Add-on payloads, and this external PR remains a useful focused fix path. It should not merge yet because inspectable after-fix real behavior proof is still missing and the parser subset should stay coordinated with the broader open Google Chat issue cluster.

## What This Changes

The PR updates Google Chat webhook parsing to unwrap `chat.addedToSpacePayload` and `chat.removedFromSpacePayload` Workspace Add-on envelopes and map them to lifecycle event types.

## Best Possible Solution

Land a plugin-local parser fix with lifecycle regression tests and redacted real Google Chat proof, then continue tracking group allowlist and reply-thread issues in the canonical reports.

## Reproduction Assessment

Yes. Source inspection against current main shows a Workspace Add-on request with `commonEventObject.hostApp: "CHAT"` and only `chat.addedToSpacePayload` or `chat.removedFromSpacePayload` is not normalized, then fails the root event-type guard with `invalid payload`.

## Solution Assessment

Yes for the requested parser subset: accepting lifecycle payload variants in the Google Chat plugin is narrow and matches Google’s documented union contract. Before merge it still needs inspectable real behavior proof, and it should not be used to close the broader https://github.com/openclaw/openclaw/issues/65007 by itself.
