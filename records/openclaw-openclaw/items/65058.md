## Summary

## What This Changes

The PR changes the Google Chat webhook parser to normalize message, added-to-space, and removed-from-space add-on payload variants into Chat events.

## Best Possible Solution

Land a current-main Google Chat parser update that handles lifecycle variants without regressing card-click callbacks, with focused tests and redacted real Google Chat proof.

## Reproduction Assessment

Yes. Source inspection shows current main only unwraps chat.messagePayload and buttonClickedPayload, so an add-on envelope containing only chat.addedToSpacePayload or chat.removedFromSpacePayload reaches the event-type guard without a root string type and returns invalid payload.

## Solution Assessment

No as submitted. The parser boundary is right, but the branch must be rebased into the current card-click dispatcher and backed by tests plus real Google Chat proof before it is the best fix.
