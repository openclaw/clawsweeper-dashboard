## Summary

Keep open. Current `main` still has an advertised ZAI `glm-4.6v` image-understanding path through the image tool/media-understanding runtime, and the latest discussion only proves the narrower direct multimodal model path works, not the original agent image-tool path.

## What This Changes

The issue reports that `agents.defaults.imageModel = "zai/glm-4.6v"` fails in the built-in image tool with `Request was aborted` even though direct Z.AI API calls with the same image and key succeed.

## Best Possible Solution

Keep this open until maintainers live-check the agent image-tool route and either land a ZAI-owned media-understanding fix or explicitly consolidate the remaining work into the open ZAI image-tool tracker at https://github.com/openclaw/openclaw/issues/70886.
