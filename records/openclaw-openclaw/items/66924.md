## Summary

## What This Changes

The PR adds a 131,072-token context-window patch for all Google Gemma forward-compat clones and a regression test for `gemma-4-26b-a4b-it`.

## Best Possible Solution

Keep the fix inside the Google provider resolver, derive fallback context from exact or family Gemma metadata when available, and preserve configured/discovered metadata precedence.

## Reproduction Assessment

Yes by source inspection: current main's Gemma fallback still clones the Gemini flash template, and `cloneFirstTemplateModel` preserves template `contextWindow` unless a patch or configured metadata overrides it. I did not run tests in this read-only review because local test commands can write artifacts.

## Solution Assessment

No. The PR touches the right provider-owned path, but a fixed 128k value is not the narrowest correct fix for Gemma 4; the safer repair should use exact or family metadata and keep configured overrides authoritative.
