## Summary

Keep open: current main still has the Gemma template context-window inheritance gap, but this PR under-reports 256K-class Gemma 4 models and is missing the required changelog entry.

## What This Changes

The PR adds a 131,072-token Gemma fallback contextWindow in the Google forward-compat resolver and a regression test for the Gemma 4 template-clone path.

## Best Possible Solution

Keep the fix provider-owned, but use Gemma-family or exact-id fallback metadata for known Gemma 4 tiers, preserve configured/discovered metadata precedence, add focused regression coverage, and add the Unreleased changelog entry.

## Reproduction Assessment

Yes. A resolver-level reproduction is clear: resolve gemma-4-26b-a4b-it with only a gemini-3-flash-preview template whose contextWindow is 1,048,576; current main patches only reasoning and the clone helper preserves the template contextWindow.

## Solution Assessment

No. The PR chooses the right provider boundary and test shape, but a one-size 131,072 fallback is not the narrowest correct metadata fix for 256K-class Gemma 4 variants.
