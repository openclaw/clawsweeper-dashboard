## Summary

Keep open: the fallback inheritance bug is real, but this PR replaces the inherited 1M Gemini context with a fixed 128k value that conflicts with current Gemma 4 metadata, and the external proof gate is still missing.

## What This Changes

The PR adds a 131,072-token context-window patch for Gemma forward-compat clones in the Google provider and a regression test for `gemma-4-26b-a4b-it`.

## Best Possible Solution

Keep the fix in the Google provider boundary, but derive Gemma fallback context from authoritative exact or family metadata when available while preserving configured metadata precedence.

## Reproduction Assessment

Yes for the resolver-level fallback path by source inspection: current main maps missing `gemma-*` ids through Gemini flash templates, and the clone helper preserves template context unless patched. The exact linked configured `models list` case is already fixed by configured metadata precedence and current catalog rows.

## Solution Assessment

No. The PR touches the right provider-owned path, but hardcoding 131,072 for every Gemma clone is not the narrowest correct fix because current Gemma 4 metadata is 256000.
