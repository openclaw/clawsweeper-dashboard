## Summary

Keep open: current main and shipped v2026.5.7 still hard-code the docs MCP selector as `SearchOpenClaw`, while the live docs MCP endpoint exposes `search_open_claw` and returns the reported missing-tool error for the old name.

## What This Changes

The issue reports that `openclaw docs <query>` fails on OpenClaw 2026.5.7 because the CLI calls a hosted docs MCP tool name that the endpoint no longer exposes.

## Best Possible Solution

Update the docs CLI to call the canonical hosted MCP tool `search_open_claw`, add focused regression coverage for the selector and command arguments, and include a user-facing changelog fix entry.

## Reproduction Assessment

Yes. Current main calls `https://docs.openclaw.ai/mcp.SearchOpenClaw`, and the live docs MCP endpoint returns the exact missing-tool error for `SearchOpenClaw` while `search_open_claw` returns docs results.

## Solution Assessment

Yes. The narrow maintainable fix is to change the hard-coded selector in `src/commands/docs.ts` to the canonical live MCP tool name and cover that invocation, without adding config or depending on name correction.
