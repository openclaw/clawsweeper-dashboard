## Summary

Keep open: current `main` and shipped `v2026.5.7` still call the hosted docs MCP tool as `SearchOpenClaw`, while the live endpoint exposes `search_open_claw` and returns the exact missing-tool error for the old name. The earlier related report at https://github.com/openclaw/openclaw/issues/39587 was broader; this issue has a narrower current selector mismatch with a clear repair path.

## What This Changes

The issue reports that `openclaw docs <query>` in release `2026.5.7` exits non-zero because the CLI calls a missing hosted docs MCP tool named `SearchOpenClaw`.

## Best Possible Solution

Update the docs CLI to call the canonical hosted MCP tool `search_open_claw`, add focused regression coverage for the selector/argv, and include a user-facing changelog fix entry.

## Reproduction Assessment
