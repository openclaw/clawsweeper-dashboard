## Summary

Keep open. The PR fixes the main Telegram presentation URL-button drop, but it also advertises raw URL buttons without wiring raw `buttons` through the Telegram action runtime, and the user-facing fix still needs a changelog entry.

## What This Changes

The PR updates Telegram inline-button rendering/types, message-tool button schema/prompt guidance, and focused tests so URL inline buttons can be preserved with URL taking precedence over callback data.

## Best Possible Solution

Land a small revision that preserves URL buttons across presentation, interactive, and raw message-tool send paths, with targeted tests and an Unreleased changelog entry.

## Reproduction Assessment

Yes. A focused source-level reproduction is available: on current main, a presentation button with only `url` is normalized but then dropped by `buildTelegramInteractiveButtons`/`buildInlineKeyboard`; for the PR-specific gap, `handleTelegramAction` with raw `buttons` still cannot forward those buttons because it never reads `params.buttons`.

## Solution Assessment

No. The patch is close for presentation and interactive payloads, but it is not the best complete fix while the newly documented raw `buttons` URL path remains disconnected from Telegram action handling.
