## Summary

Keep open. The PR mostly addresses Telegram URL inline-button rendering and exact-head checks are green, but it still advertises and schema-expands raw `buttons` URL support without wiring that path through Telegram send handling, and the Telegram docs remain callback-only.

## What This Changes

The PR adds Telegram URL button support in interactive button mapping and inline-keyboard serialization, updates prompt/schema text, focused tests, and the changelog.

## Best Possible Solution

Land a narrowed Telegram fix that carries URL buttons through the supported presentation path and either wires or removes the raw `buttons` path, with Telegram docs, tests, and the existing changelog aligned.

## Reproduction Assessment

Yes. Current main preserves URL buttons in the shared payload layer, then drops them in Telegram because `buildTelegramInteractiveButtons` requires `value` and `buildInlineKeyboard` requires `callback_data`; the paired issue also includes live DM verification from May 2, 2026.

## Solution Assessment

No as-is. The runtime presentation mapping is the right narrow direction, but the raw button prompt/schema change is incomplete without `params.buttons` wiring, and the public Telegram docs should be updated before merge.
