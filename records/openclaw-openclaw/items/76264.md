## Summary

Keep open. The PR fixes the shared presentation URL-button path, but it now advertises raw `buttons` URL support without wiring `params.buttons` through Telegram message actions, and the public Telegram docs still only describe callback buttons.

## What This Changes

This PR updates Telegram inline-button normalization/rendering, prompt/schema copy, tests, and changelog so URL buttons can be emitted and preferred over callback data.

## Best Possible Solution

Finish the PR by wiring raw Telegram `buttons` through send/edit actions or removing the raw-buttons claim, update the Telegram docs for URL buttons, then rerun focused Telegram/system-prompt tests and the changed gate.

## Reproduction Assessment

Yes. Current main drops URL-only inline buttons in `buildInlineKeyboard`, skips shared URL-only buttons in Telegram button mapping, and ignores raw `params.buttons`; the paired report also includes live DM verification from May 2, 2026.

## Solution Assessment

No as-is. Preferring URL over callback data matches Telegram's one-action-field contract, but the PR should either implement raw message-tool button delivery or stop advertising it, and the public Telegram docs need to match the new schema.
