## Summary

Keep open: the PR addresses a valid doctor/auth diagnostic bug, but the patch has a blocking auth-store routing defect and the protected `maintainer` label requires explicit maintainer handling.

## What This Changes

The PR updates `openclaw doctor` auth health reporting to enumerate per-agent auth stores and label auth notes with agent IDs when multiple stores are present.

## Best Possible Solution

Pass the selected target agent directory through every refresh/read-after-refresh path, or keep non-default labeled passes diagnostics-only while preserving refresh for the true default store, then add regression coverage and redacted real doctor output.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: with a non-default or configured-default expired OAuth profile, the PR loads that target store but calls `resolveApiKeyForProfile` without the target `agentDir`.

## Solution Assessment

No. Agent-labeled diagnostics are the right fix direction, but the refresh path must use the same target identity as the diagnostic pass or avoid refreshing labeled non-default passes.
