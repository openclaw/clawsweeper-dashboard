## Summary

Keep open: the patch is not safe to merge because it leaves the shared-secret backend spoofing path from the linked gateway report in place and widens the pairing bypass to bootstrap-token handshakes.

## What This Changes

The PR adds `bootstrap-token` as a backend self-pairing skip auth method and adds helper-level test/comment coverage for that bypass.

## Best Possible Solution

Bind backend self-pairing bypass to server-verifiable backend identity or a dedicated backend credential, and keep bootstrap-token handshakes on the pairing path unless maintainers approve a broader exception with matching tests.

## Reproduction Assessment

Yes. The helper-level reproduction is clear: a loopback/no-Origin `gateway-client` backend handshake with valid token/password auth makes `shouldSkipLocalBackendSelfPairing` true, and the message handler then preserves unbound scopes by skipping the normal pairing path.

## Solution Assessment

No. The PR adds bootstrap-token to the bypass instead of removing or tightening the shared-secret backend condition, so it is not the narrowest maintainable security fix.
