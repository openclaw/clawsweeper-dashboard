## What This Changes

## Best Possible Solution

Land the narrow gateway-env default after redacted real Windows Scheduled Task proof confirms `gateway.vbs` is the task action and starts without a visible console, with Windows docs updated for the new launcher artifact.

## Solution Assessment

Yes, this is the best code layer: `buildServiceEnvironment()` is the gateway install env source and schtasks already owns the VBS launcher. The remaining gaps are real-host proof, compatibility acceptance, and docs, not a different implementation path.
