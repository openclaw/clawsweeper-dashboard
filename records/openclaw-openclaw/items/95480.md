The PR changes Windows gateway Scheduled Task installs to select the generated `gateway.vbs` hidden launcher by default, preserves caller-selected task identity, adds daemon regression coverage, and updates Windows cleanup docs.

## Best Possible Solution

## Solution Assessment

Yes, with a compatibility caveat. Reusing the existing VBS hidden launcher and resolving from merged generated service env is the narrowest fix for this bug; maintainer acceptance of the VBS default remains the key decision.
