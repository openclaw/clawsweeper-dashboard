## Label Justifications

- `P2`: This is a focused Windows managed-startup bug fix with limited platform-specific blast radius.
- `merge-risk: 🚨 compatibility`: Changing the default Scheduled Task action to `.vbs` can affect Windows hosts where Windows Script Host or `.vbs` execution is blocked.

## Summary

Keep open: this is the current focused fix candidate for the linked Windows gateway Scheduled Task visible-console bug, and current main does not already implement it. Source review found the patch shape sound, but merge still needs maintainer acceptance of the VBS/WScript default compatibility tradeoff and resolution of the currently failing required checks.

## What This Changes

The PR makes generated Windows gateway Scheduled Task installs use the existing `gateway.vbs` hidden launcher by default, preserves caller-selected task identity, and updates focused tests and Windows docs.

## Best Possible Solution

Land this PR after required checks pass and a maintainer accepts the VBS/WScript default, or require a tested fallback/opt-out first if locked-down Windows hosts must remain on direct `.cmd` launch.

## Reproduction Assessment

Yes. Current main shows generated gateway env lacks the hidden-launcher marker while scheduler launch resolution only switches to `.vbs` when that marker is truthy; I did not run a live Windows desktop repro in this read-only review.

## Solution Assessment

Yes, with maintainer compatibility review. Setting the generated gateway service env and resolving the task action from that generated env is the narrow owner-boundary fix; the safer alternative for locked-down hosts is a tested fallback or opt-out before changing the default.
