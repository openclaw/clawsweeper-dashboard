## Summary

Keep open: this PR fixes the central Windows Scheduled Task action path, but it is not merge-ready because the new gateway hidden-launcher default can create `gateway.vbs` while service-only uninstall/reinstall still resolves cleanup from the caller env and can leave that generated launcher behind. Maintainers also still need to accept the Windows Script Host default compatibility tradeoff.

## What This Changes

The branch defaults generated Windows gateway Scheduled Task installs to the existing VBS hidden launcher, resolves task action from generated service env while preserving task identity, adds focused tests, and updates Windows docs.

## Best Possible Solution

Land the Windows gateway hidden-launcher fix only after uninstall/reinstall cleanup removes the generated VBS launcher and maintainers accept the Windows Script Host default compatibility tradeoff.

## Reproduction Assessment

Yes. Source inspection shows current main and `v2026.6.9` omit the gateway hidden-launcher default while the scheduler switches to `.vbs` only when that env value is truthy; the PR body adds real Windows Scheduled Task proof for the fixed install path.

## Solution Assessment

No, not as submitted. Reusing the existing VBS hidden launcher at the generated service-env boundary is the right narrow fix, but the PR should also clean up the generated launcher on uninstall/reinstall before merge.
