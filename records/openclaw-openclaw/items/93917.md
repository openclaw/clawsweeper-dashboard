The issue reports that repeated `exec` failures with slightly varying output never reach the `genericRepeat` critical threshold or global circuit breaker.

## Best Possible Solution

Keep this issue open as the canonical failed-exec volatile-output bug, revise or replace the linked PR with a narrow failed-status hash fix that preserves completed exec output as progress, and track broader similarity/registry/global-breaker work separately.
