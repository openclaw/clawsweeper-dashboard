## Summary

Keep open: current main and v2026.6.6 still lack a first-class runtime no-start/progress-SLA watchdog for accepted channel tasks and delegated subagents. Existing heartbeat, progress drafts, diagnostics, cron watchdogs, and subagent timeout/delivery records are adjacent but do not enforce the requested visible SLA.

## What This Changes

Request a runtime-enforced task watchdog that alerts on accepted work with no start/progress and on long-running agent or subagent gaps.

## Best Possible Solution

Define one lean runtime progress-SLA contract that reuses diagnostics, run-state, progress drafts, heartbeat wakeups, and subagent/task state without creating a parallel scheduler or broad config stack.

## Reproduction Assessment

No live current-main Discord reproduction was established. Source inspection does confirm the requested `taskWatchdog`/no-start/progress-SLA surface is absent and current mechanisms cover narrower cases only.

## Solution Assessment

Unclear as written. Runtime-side enforcement is the right layer, but the proposed config shape needs maintainer design to avoid duplicating existing heartbeat, progress-draft, diagnostics, task-ledger, and subagent delivery contracts.
