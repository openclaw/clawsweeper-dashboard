## Summary

Keep open: current main has heartbeat scheduling, task records, subagent run timeouts, and lost-task reconciliation, but not a runtime watchdog for accepted channel-agent work that has not started or for periodic progress heartbeats during long-running delegated work.

## What This Changes

Request to add a first-class runtime task watchdog for channel-agent assignments and delegated subagents, including no-start deadlines, periodic heartbeats, progress expectations, and configurable violation delivery.

## Best Possible Solution

Design a generic task-lifecycle watchdog around agent events and the task registry, document the config contract, and cover Discord channel acceptance plus sessions_spawn progress cases before implementation.

## Solution Assessment
