## Summary

Keep open: current main has adjacent heartbeat, diagnostics, task-ledger, and subagent timeout machinery, but it does not provide the requested runtime-side SLA watchdog for accepted channel work that has not started or for delegated subagent progress heartbeats.

## What This Changes

The issue requests a runtime-enforced task watchdog for accepted Discord/channel-agent work and delegated subagent progress heartbeats.

## Best Possible Solution

Design a generic session/task watchdog contract around session activity, task records, and channel delivery, then document the config and cover no-start, long-running, and delegated-subagent progress cases with focused tests.

## Solution Assessment
