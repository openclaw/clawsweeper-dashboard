## Summary

Keep open: current main and the latest v2026.6.6 release still lack a first-class runtime no-start/progress-SLA watchdog for accepted channel tasks and delegated subagents. Existing heartbeat, progress-draft, diagnostic recovery, cron watchdog, and subagent timeout paths are adjacent but narrower, and a related closed issue explicitly routes the progress-watchdog portion back to this issue.

## What This Changes

Add a first-class runtime watchdog for accepted channel work and delegated subagents, including visible status or blocker output and possible per-agent/per-session configuration.

## Best Possible Solution

Define a lean runtime progress-SLA contract that reuses diagnostics, run-state, progress drafts, heartbeat wakeups, and the subagent registry instead of adding a parallel scheduler or broad config stack.

## Reproduction Assessment

No live current-main Discord reproduction was established. Source inspection does show the requested `taskWatchdog`/no-start/progress-SLA surface is absent and that existing mechanisms cover narrower cases only.

## Solution Assessment

Unclear as written. Runtime-side enforcement is the right layer, but the proposed config shape needs maintainer design to avoid duplicating existing heartbeat, progress-draft, diagnostics, task-ledger, and subagent delivery contracts.
