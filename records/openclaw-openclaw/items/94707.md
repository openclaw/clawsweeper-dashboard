  - body: Current main intentionally calls `auth.test()` without arguments because `@slack/web-api` sends `options.token` as an Authorization header while also serializing the options object into the POST body. This line reintroduces bot-token-in-body exposure, so keep the token on the client only.
  - confidence: 0.94
- **[P1] Require secure relay transport before sending tokens:** `extensions/slack/src/monitor/relay-source.ts:95-99`
  - body: `openRelayWebSocket` sends `Authorization: Bearer <relay.authToken>` while `buildRelayWebSocketUrl` accepts plaintext `ws:` for any host. Require `wss:` outside local loopback, or add an explicit dangerous opt-in, before sending the bearer token.
  - confidence: 0.88
- **[P2] Keep missing local tools fail-closed:** `scripts/pre-commit/run-node-tool.sh:19-21`
  - body: This path exits successfully when `node_modules/.bin/$tool` is absent, so contributors with incomplete installs can commit without running the expected formatter/linter hook. Current main has a regression test requiring this case to fail before pnpm can hydrate dependencies.
  - confidence: 0.92
