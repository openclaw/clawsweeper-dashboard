Overall correctness: patch is incorrect

Full review comments:

- **[P1] Defer relay acks until debounced dispatch completes:** `extensions/slack/src/monitor/relay-source.ts:252`
  - body: `handleSlackMessage()` can return after enqueueing a debounced message instead of after `dispatchPreparedSlackMessage()` runs. With inbound debounce enabled, this sends a relay ack before dispatch/reply succeeds, so the router can treat a message as delivered even if the delayed flush later fails or the process exits.
  - confidence: 0.9
- **[P2] Cap relay websocket payloads:** `extensions/slack/src/monitor/relay-source.ts:107-112`
  - body: The new WebSocket client only sets headers and `handshakeTimeout`, so it inherits `ws` defaults of a 100 MiB max payload and permessage-deflate enabled. Relay frames should be small Slack envelopes; set an explicit small payload cap and disable compression unless the relay contract requires it.
  - confidence: 0.84
