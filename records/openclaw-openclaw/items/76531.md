## Summary

Keep open. Current main still strips hidden tagged block text before delivery, then only sends the accumulated final TTS payload when synthesis returns a `mediaUrl`, so a synthesis miss can leave no outbound fallback.

## What This Changes

Hidden `[[tts:text]]...[[/tts:text]]` block replies under `messages.tts.auto = "tagged"` can be stripped from visible output and dropped when final TTS synthesis returns no media URL.

## Best Possible Solution

Keep the audio-only success behavior, and add a text fallback from the stripped hidden TTS body only when accumulated final-mode synthesis misses and no visible or final reply was delivered.

## Reproduction Assessment

Yes, source-reproducible: a streamed block containing only `[[tts:text]]hello[[/tts:text]]` is stripped before block delivery, then a no-media final TTS result falls through without a queued fallback. I did not run a live provider/channel test in this read-only pass.

## Solution Assessment

Yes. The narrow maintainable fix is in accumulated-block finalization, preserving media delivery when synthesis succeeds and sending stripped fallback text only when the alternative is a silent drop.
