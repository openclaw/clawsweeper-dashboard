## Summary

Keep open. Current main still strips hidden-only tagged block text before visible delivery, and both generic and ACP final TTS branches only deliver the accumulated synthetic reply when synthesis returns media.

## What This Changes

Prevent hidden `[[tts:text]]...[[/tts:text]]` accumulated block replies from disappearing when final-mode TTS synthesis returns no media URL.

## Best Possible Solution

Add a narrow dispatcher fallback that sends cleaned hidden TTS text only when accumulated final-mode synthesis returns no media, while keeping successful synthesis audio-only.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: stream a hidden-only `[[tts:text]]hello[[/tts:text]]` block with final-mode tagged TTS and make `maybeApplyTtsToPayload` return without `mediaUrl`; current main strips visible block text and queues no fallback final reply.

## Solution Assessment

Yes. The narrow maintainable fix belongs in accumulated-block dispatch finalization, preserving audio-only delivery on synthesis success and routing cleaned hidden text only on synthesis miss.
