## What This Changes

The issue reports that hidden tagged TTS block replies can be silently dropped when final-mode synthetic TTS does not produce media.

## Best Possible Solution

Ensure final-mode hidden-tag TTS always produces a visible outcome: synthesized media when available, otherwise cleaned text fallback, with focused non-ACP dispatcher regression coverage.

## Reproduction Assessment

Yes. Current main still strips hidden `[[tts:text]]` content from visible block delivery, accumulates it for final TTS, and only sends the synthetic final payload when `mediaUrl` exists.

## Solution Assessment

No, the issue is not solved on current main. The active related PR is a plausible fix path, but the narrowest repair for this report is a non-ACP dispatcher fallback that sends cleaned TTS text when media synthesis misses.
