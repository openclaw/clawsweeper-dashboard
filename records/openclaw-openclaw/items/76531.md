## Summary

Keep open. Current `main` still has the accumulated hidden-tag final TTS path that only sends a reply when synthesis returns a `mediaUrl`; when synthesis misses, the hidden text has already been stripped from visible block delivery and no text fallback is queued.

## What This Changes

The issue requests a fallback for hidden `[[tts:text]]...[[/tts:text]]` auto-TTS replies so a synthesis miss does not silently drop the reply.

## Best Possible Solution

Add a narrow fallback in the accumulated-block final TTS branch that sends the stripped hidden TTS body as text only when synthetic TTS produces no media and no visible or final reply was delivered.

## Reproduction Assessment

Yes, source-reproducible: a final-mode streamed block containing only `[[tts:text]]hello[[/tts:text]]`, combined with a synthesis result lacking `mediaUrl`, reaches the current no-delivery branch after visible text was stripped.

## Solution Assessment

Yes, the best fix is a narrow dispatcher fallback for the accumulated hidden-tag finalization path rather than changing provider failure semantics or directive parsing globally.
