The PR adds optional `ctx.activeModel` provider/model/modelRef metadata to plugin tool factory context, threads model IDs through plugin tool construction, varies descriptor cache keys by that metadata, and documents/tests the SDK surface.

## Best Possible Solution

Land the additive SDK context after maintainer review and real behavior proof, keeping the docs clear that plugin-side checks should fail closed and are not a host security boundary; model-scoped registration can remain a separate follow-up if needed.

## Solution Assessment
