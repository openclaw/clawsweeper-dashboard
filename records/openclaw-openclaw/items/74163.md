- **[P1] Preserve resolved tracker state before rewriting:** `scripts/generate-microsoft-tracker.mjs:302-305`
  - body: The generator only scans currently open items and then writes every category summary with `Closed` set to `0`.   - confidence: 0.94
- **[P2] Remove macOS items from the Windows tracker:** `MICROSOFT_TRACKER.md:107`
  - body: This generated row places a macOS WindowServer report in the Windows Platform section, and the file also reintroduces the older Intel Retina iMac WindowServer false positive. The tracker should not count those as Microsoft Windows work; otherwise the generated output misroutes maintainer triage.
  - confidence: 0.9
- **[P2] Emit valid Markdown table separator cells:** `scripts/generate-microsoft-tracker.mjs:362`
  - body: The generator emits a single `-` separator cell for the `#` column in generated issue tables. GitHub-flavored Markdown tables need real separator cells, so rerunning the generator or updating the PR body can produce tables that render inconsistently even if the committed file was later formatted.
  - confidence: 0.86
- **[P2] Remove the release-owned changelog entry:** `CHANGELOG.md:9`
  - body: Root policy makes `CHANGELOG.md` release-owned, but this admin tracker PR adds an Unreleased entry directly. Move any release-note context to the PR body or final commit message and leave the changelog to release generation.
  - confidence: 0.93
- **[P3] Point tracker instructions at the durable source of truth:** `MICROSOFT_TRACKER.md:7`
  - body: The committed tracker still tells maintainers to mark items by editing the PR body. If this file is merged, readers need instructions for the actual maintained source of truth, otherwise updates will happen in the wrong place.
  - confidence: 0.84
