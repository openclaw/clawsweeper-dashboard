## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Neutralize closing keywords before writing tracker rows:** `scripts/generate-microsoft-tracker.mjs:240`
  - body: `prRow()` emits copied PR titles directly into generated Markdown, and `--update-pr-body` copies that Markdown into a PR body. Titles containing closing-keyword references can become live GitHub closing references, so neutralize those phrases before emitting rows.
  - confidence: 0.93
- **[P2] Use boundary-aware category matching:** `scripts/generate-microsoft-tracker.mjs:140`
  - body: `title.includes(term)` lets the Windows term match unrelated words such as WindowServer, and the generated tracker currently places a macOS WindowServer item under Windows Platform. Match on word/domain boundaries or explicit tokens before regenerating the tracker.
  - confidence: 0.92
- **[P2] Emit valid Markdown table separators:** `scripts/generate-microsoft-tracker.mjs:362`
  - body: The generator emits a single `-` separator for the number column in issue, PR, and appendix tables. GitHub-flavored Markdown requires at least `---` per separator cell, so rerunning the generator can produce tables that do not render consistently.
  - confidence: 0.86
- **[P2] Point tracker instructions at the committed file:** `scripts/generate-microsoft-tracker.mjs:280`
  - body: The generated header tells maintainers to edit the PR body, but this PR adds `MICROSOFT_TRACKER.md` as a repository file. That sends readers to the wrong update surface when they are using the committed tracker.
  - confidence: 0.9
