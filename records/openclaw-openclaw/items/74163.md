Full review comments:

- **[P1] Preserve closed tracker state before body rewrites:** `scripts/generate-microsoft-tracker.mjs:302-305`
  - body: The generator fetches only open issues/PRs and then emits every Closed count as `0`; because `--update-pr-body` replaces the PR body from that output, a refresh can erase the closed/resolved history that the discussion shows maintainers have been preserving. Either preserve/import the existing closed rows and counts or make the open-only reset an explicit maintainer-approved mode before merging this automation.
  - confidence: 0.91
- **[P2] Emit valid Markdown table separators:** `scripts/generate-microsoft-tracker.mjs:361-362`
  - body: These separator rows use a single `-` for the `#` column, so rerunning the generator will produce tables that GitHub does not reliably parse as Markdown tables. The checked-in tracker currently has widened separators, but the source generator will regress it on the next refresh unless each separator cell uses at least `---`.
  - confidence: 0.89
- **[P2] Remove the release-owned changelog edit:** `CHANGELOG.md:9`
  - body: Normal PRs should not edit `CHANGELOG.md`; this release note should live in the PR body or squash/merge message until the release-generation flow owns the changelog entry. Keeping this line also conflicts with current main's already-released changelog structure.
  - confidence: 0.86
