  - body: The script writes `MICROSOFT_TRACKER.md`, but the branch adds no package-level generate/check command that regenerates and compares the committed artifact. Since the source is live GitHub state, the file can become stale immediately and CI has no way to detect it.

  - body: The generated tracker tells readers to edit `this PR body`, but the same content is committed as `MICROSOFT_TRACKER.md`. Use wording that works in both places, or generate separate wording for file output and PR-body output, so maintainers do not update the wrong surface.
  - confidence: 0.96
