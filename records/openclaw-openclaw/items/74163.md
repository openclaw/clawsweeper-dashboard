Full review comments:

- **[P1] Preserve resolved tracker state before rewriting the body:** `scripts/generate-microsoft-tracker.mjs:302-305`
  - body: `buildMarkdown()` always emits closed counts as `0`, and the later `--update-pr-body` path replaces the PR body from that generated file. A refresh would silently lose the closed/resolved rows that this tracker has been preserving in comments, so the generator needs to carry forward existing resolved state or avoid destructive body rewrites.
  - confidence: 0.94
- **[P2] Match Windows as a bounded platform term:** `scripts/generate-microsoft-tracker.mjs:140-145`
  - body: The classifier uses substring matching on the whole title, so non-platform phrases like context windows are routed into the Windows platform queue; the generated tracker already lists https://github.com/openclaw/openclaw/issues/87136 there. Please tokenize or otherwise bound the platform terms before this becomes maintainer routing data.
  - confidence: 0.88
- **[P2] Emit valid Markdown table separators from the generator:** `scripts/generate-microsoft-tracker.mjs:361-363`
  - body: The generator writes `-` as a table separator cell in issue, PR, and appendix tables. GitHub-flavored Markdown requires at least `---`, so the next generated refresh can render these blocks inconsistently even though the committed tracker file was manually formatted.
  - confidence: 0.9
- **[P3] Point the tracker instructions at the committed file:** `MICROSOFT_TRACKER.md:7`
  - body: The tracker file tells maintainers to edit this PR body, but after merge the durable artifact would be `MICROSOFT_TRACKER.md`. That instruction sends maintainers to the wrong source of truth for marking resolved items.
  - confidence: 0.92

  - body: Repository policy makes `CHANGELOG.md` release-owned and says normal PRs should carry release-note context in the PR body or commit message instead. Please drop this changelog entry from the branch.
