Full review comments:

- **[P1] Neutralize copied closing references before body updates:** `scripts/generate-microsoft-tracker.mjs:426`
  - body: `--update-pr-body` writes generated Markdown directly into this PR body, but rows copy issue and PR titles unchanged. GitHub already reports this tracker PR as closing https://github.com/openclaw/openclaw/issues/84600, https://github.com/openclaw/openclaw/issues/92054, and https://github.com/openclaw/openclaw/issues/93288, so the generator needs to escape or rewrite closing-keyword title text before any PR-body update.
  - confidence: 0.93
- **[P2] Honor explicit priority labels before keyword escalation:** `scripts/generate-microsoft-tracker.mjs:151-153`
  - body: `priorityFor` derives priority from free text across both title and labels, so a P3-labeled item with a review/status label containing words like security can become P0 in the generated tracker. Prefer the live P-label when present, then use heuristics only as a fallback.
  - confidence: 0.88
- **[P2] Require token-boundary matches for Windows terms:** `scripts/generate-microsoft-tracker.mjs:140`
  - body: The category matcher uses raw substring checks, which still classifies a macOS WindowServer issue as Windows because `WindowServer` contains the `windows` substring. Use token or word-boundary matching for terms like Windows so non-Microsoft rows do not re-enter the tracker.
  - confidence: 0.9
- **[P3] Fix the generated edit instruction for file output:** `scripts/generate-microsoft-tracker.mjs:280`
  - body: The generator always tells maintainers to edit this PR body, but it also writes `MICROSOFT_TRACKER.md` as a committed file. When generating the file, the instruction should point to the file or be parameterized by output mode so maintainers are not sent to the wrong surface.
  - confidence: 0.86
- **[P3] Generate valid Markdown table separator rows:** `scripts/generate-microsoft-tracker.mjs:362`
  - body: The generator emits a single `-` separator cell for the issue number column, which can break GitHub-flavored Markdown table rendering when the file or PR body is regenerated. Emit at least `---` for every separator cell in all generated tables.
  - confidence: 0.78
- **[P3] Remove release-owned changelog edits from this PR:** `CHANGELOG.md:9`
  - body: OpenClaw release generation owns `CHANGELOG.md`, while normal PRs should keep release-note context in the PR body or commit message. This branch should drop the changelog line unless a release maintainer explicitly moves it through the release process.
  - confidence: 0.82
