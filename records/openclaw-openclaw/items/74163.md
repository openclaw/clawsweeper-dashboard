  - body: `priorityFor()` derives P0/P1 from title and label substrings instead of first preserving the live `P0`-`P3` label, so rows like P3/P2-labeled Windows requests are escalated to P0 merely because they carry security-review text. That makes this tracker unsafe as a maintainer queue source.
  - confidence: 0.9

  - body: Title matching uses `title.includes(term)`, so `WindowServer` contains the `windows` term and the generated tracker currently files a macOS WindowServer issue under Windows Platform. Use word/token boundaries or explicit platform labels before categorizing broad platform rows.
  - confidence: 0.92
- **[P3] Emit valid Markdown table separators:** `scripts/generate-microsoft-tracker.mjs:362`
  - body: The generator emits separator cells as `-` for the `#` column, which is not a valid GFM table delimiter and can break table rendering when the script refreshes the PR body. Generate at least `---` for every separator cell.
  - confidence: 0.86
- **[P3] Fix the generated edit instruction:** `scripts/generate-microsoft-tracker.mjs:280`
  - body: The committed tracker tells readers to edit `this PR body`, but the PR also adds `MICROSOFT_TRACKER.md` as a repository file. If the file is kept, the generated instruction needs to point to the durable source of truth rather than sending maintainers to the wrong surface.
  - confidence: 0.89
