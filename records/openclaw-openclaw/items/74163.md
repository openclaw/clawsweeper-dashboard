- **[P1] Neutralize closing refs before PR-body updates:** `scripts/generate-microsoft-tracker.mjs:426`
  - body: This writes generated Markdown directly into a PR body while tracker rows copy issue and PR titles verbatim. The live PR already has GitHub closing references to unrelated issues because titles contain closing-keyword text; sanitize those keywords before body updates or remove this mode.
  - confidence: 0.95

  - body: The heuristic promotes any label text containing `security` to P0, so items carrying review labels such as `clawsweeper:needs-security-review` or `impact:security` become emergency rows even when the existing triage label is P2/P3. That makes the tracker priority queue misleading for maintainers.
  - confidence: 0.88

  - body: With broad MS Teams label matching enabled by default, any PR carrying `channel: msteams` can land in the Microsoft queue even when the title and scope are not Microsoft-specific. The current output includes broad rows like external reranker and a document reader skill, which will misdirect maintainer cleanup.
  - confidence: 0.9

  - body: The generator emits a single `-` for the `#` separator cell in generated tables. GitHub-flavored Markdown expects at least three hyphens per separator cell, so rerunning the generator can produce tables that render inconsistently even though the checked-in file is currently formatted.
  - confidence: 0.78
