Full review comments:

- **[P1] Honor explicit priority labels first:** `scripts/generate-microsoft-tracker.mjs:152-153`
  - body: `priorityFor` builds its classifier input from both title and all label names, then returns P0/P1 from broad keyword hits before considering explicit priority labels.   - confidence: 0.92
- **[P2] Add a generated-output check mode:** `scripts/generate-microsoft-tracker.mjs:422`
  - body: The script writes the committed tracker, but this PR adds no package script or check mode that reruns the generator and fails when `MICROSOFT_TRACKER.md` or the PR body is stale. Without that verifier, the generated artifact can drift immediately after merge and CI cannot catch it.
  - confidence: 0.88
- **[P2] Use location-neutral update instructions:** `MICROSOFT_TRACKER.md:7`
  - body: The committed tracker tells maintainers to edit the PR body even though the PR adds a repository file and a generator whose default output is that file. After merge, that instruction points maintainers at the wrong source of truth.
  - confidence: 0.9
- **[P2] Emit valid Markdown table delimiters:** `scripts/generate-microsoft-tracker.mjs:362`
  - body: The generator emits single-hyphen separator cells for the `#` column, so raw output written with `--update-pr-body` may not render consistently as GitHub tables. Use normal `---` separator cells in every generated table.
  - confidence: 0.82
- **[P2] Remove the release-owned changelog entry:** `CHANGELOG.md:7`
  - body: Root policy says CHANGELOG.md is release-owned for normal PRs. This tracker refresh should keep release-note context in the PR body or commit message instead of adding an active changelog entry that release generation later has to reconcile.
  - confidence: 0.95
