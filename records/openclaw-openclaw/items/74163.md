## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Neutralize closing refs before PR-body updates:** `scripts/generate-microsoft-tracker.mjs:426`
  - body: This path writes generated Markdown directly into a PR body while tracker rows copy issue and PR titles verbatim. The live PR already has GitHub closing references to unrelated issues from those copied titles, so sanitize closing keywords before body updates or remove this mode.
  - confidence: 0.95
- **[P2] Respect existing priority labels before keyword escalation:** `scripts/generate-microsoft-tracker.mjs:153`
  - body: The heuristic promotes any label text containing `security` to P0, so review labels such as `clawsweeper:needs-security-review` or `impact:security` become emergency rows even when the existing triage label is P2/P3. That makes the tracker priority queue misleading for maintainers.
  - confidence: 0.88
- **[P2] Make broad Teams label matching opt-in:** `scripts/generate-microsoft-tracker.mjs:83`
  - body: With broad MS Teams label matching enabled by default, any PR carrying `channel: msteams` can land in the Microsoft queue even when the title and scope are not Microsoft-specific. The generated output includes broad rows like external reranker and a document reader skill, which will misdirect cleanup.
  - confidence: 0.9
- **[P3] Generate valid Markdown table separators:** `scripts/generate-microsoft-tracker.mjs:362`
  - body: The generator emits a single `-` for the `#` separator cell in generated tables. GitHub-flavored Markdown expects normal separator cells, so rerunning the generator can produce tables that render inconsistently even though the checked-in file was later formatted.
  - confidence: 0.78
- **[P3] Avoid PR-body-only instructions in tracker files:** `scripts/generate-microsoft-tracker.mjs:280`
  - body: The generated tracker file tells maintainers to mark items resolved by editing the PR body. If this lands as a repository file, that instruction points maintainers at the wrong update surface; use wording that works for the file output or split file and PR-body templates.
  - confidence: 0.82
