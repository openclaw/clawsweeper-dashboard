## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Neutralize copied closing references before PR-body updates:** `scripts/generate-microsoft-tracker.mjs:240`
  - body: `prRow` preserves raw tracked PR titles, and the later `--update-pr-body` path copies that markdown into this PR body. Several tracked titles contain closing-keyword text such as `fixes ...`; if this tracker PR is merged, those copied titles can create misleading associations or accidental issue closure semantics. Escape or rewrite closing keywords before writing PR-body markdown.
  - confidence: 0.89
- **[P1] Use live priority labels before keyword escalation:** `scripts/generate-microsoft-tracker.mjs:151-153`
  - body: `priorityFor` checks broad keywords across all labels before honoring the actual P labels, so labels like `clawsweeper:needs-security-review` can turn otherwise P2/P3 items into P0 rows. That makes the tracker’s triage ordering disagree with live GitHub labels; extract live `P0`-`P3` labels first and only fall back to keyword heuristics when no priority label exists.
  - confidence: 0.9
- **[P2] Do not route Azure Speech labels into the Teams queue:** `scripts/generate-microsoft-tracker.mjs:68-72`
  - body: The Teams category treats `plugin: azure-speech` as a Teams label match, so non-Teams TTS/Azure Speech PRs are listed under MS Teams even when their titles and extension labels are unrelated to Teams. Remove that label from the Teams category and keep Azure Speech under the Azure/provider bucket unless a separate Teams signal exists.
  - confidence: 0.86
- **[P2] Make generated tracker output valid and checked:** `scripts/generate-microsoft-tracker.mjs:361-362`
  - body: The generator emits table separator rows with a single `-` for the `#` column, while the committed markdown has formatted valid separators, and no package-level check verifies that the committed tracker matches generator output. Rerunning the script or using `--update-pr-body` can therefore produce broken or drifting Markdown; add an aligned gen/check command and make the generator output valid GFM tables directly.
  - confidence: 0.84
- **[P3] Remove the normal-PR changelog entry:** `CHANGELOG.md:9`
  - body: `CHANGELOG.md` is release-owned in this repo, and normal PRs should carry release-note context in the PR body or commit message instead. Drop this unreleased changelog line and keep the Microsoft tracker context in the PR description.
  - confidence: 0.92
