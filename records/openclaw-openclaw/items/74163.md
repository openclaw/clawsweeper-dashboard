## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.96

Full review comments:

- **[P1] Honor explicit priority labels before keyword inference:** `scripts/generate-microsoft-tracker.mjs:151-153`
  - body: Deriving priority from every label string turns lower-priority items into P0/P1 entries when labels contain words such as `security`, `auth`, or `blocked`; the generated tracker already lists a P3 installer proposal as P0 because it carries `needs-security-review`. Read an explicit P0-P3 label first and use keyword inference only when no authoritative priority exists, otherwise this tracker misroutes maintainer attention.
  - confidence: 0.99
- **[P2] Emit valid Markdown table delimiter cells:** `scripts/generate-microsoft-tracker.mjs:360-363`
  - body: The generated issue, PR, and appendix tables use `-` for the number-column delimiter, but GitHub-flavored Markdown requires a delimiter cell with at least three hyphens. Generate `---` consistently so the committed tracker and replaced PR body render as tables.
  - confidence: 0.98
- **[P2] Add a stale-output check for the committed tracker:** `scripts/generate-microsoft-tracker.mjs:421-423`
  - body: This script only overwrites the generated Markdown, while the branch adds no package-level check that regenerates and compares it. Because the source is live GitHub state, the committed file can become stale immediately and CI cannot detect it; add aligned generate/check commands and focused classifier coverage as required by the scripts policy.
  - confidence: 0.95
- **[P2] Remove the release-owned changelog entry:** `CHANGELOG.md:6`
  - body: Normal PRs must not edit `CHANGELOG.md`; release generation owns that file. Keep any release-note context in the PR body or eventual squash message instead.
  - confidence: 0.99
