  - body: `priorityFor` scans every label string for words like `security`, `auth`, and `blocked` before honoring authoritative P0-P3 labels, so lower-priority items can be promoted into the P0/P1 queue.   - confidence: 0.99

  - body: The script writes `MICROSOFT_TRACKER.md`, but the branch adds no package-level generate/check command that regenerates and compares the committed artifact. Since the source is live GitHub state, the file can become stale immediately and CI has no way to detect it.
  - confidence: 0.95
- **[P2] Emit valid Markdown table delimiters:** `scripts/generate-microsoft-tracker.mjs:360-363`
  - body: The generated issue, PR, and appendix tables use `-` as the number-column delimiter cell. GitHub-flavored Markdown table delimiters need at least three hyphens, so the generator should emit `---` consistently instead of relying on a later formatter or manual cleanup.
  - confidence: 0.98
- **[P2] Generate a location-neutral update instruction:** `scripts/generate-microsoft-tracker.mjs:280`
  - body: The generated tracker tells readers to edit `this PR body`, but `MICROSOFT_TRACKER.md` is also committed as a repository file. Use wording that works in both places, or generate different wording for file output versus PR-body output, so maintainers do not update the wrong surface.
  - confidence: 0.96
- **[P2] Remove the release-owned changelog entry:** `CHANGELOG.md:9`
  - body: Normal PRs should not edit `CHANGELOG.md`; release generation owns that file. Keep the release-note context in the PR body or eventual squash message instead of adding this line to the changelog.
