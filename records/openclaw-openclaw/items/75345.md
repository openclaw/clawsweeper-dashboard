## What This Changes

The branch adds a Codex ACP `gpt-5.5` compatibility guard for explicit pre-0.12.0 adapter commands, plus ACPX runtime tests, docs updates, and a changelog entry.

## Best Possible Solution

Keep the compatibility check at the ACPX wrapper boundary, but make it evaluate explicit npm version/range specs conservatively: any explicit `@zed-industries/codex-acp` spec that can resolve below `0.12.0` should fail early for `gpt-5.5`, while unpinned commands, local wrappers, and `0.12.0+` specs continue to pass.

## Reproduction Assessment

Yes for the review finding. With the PR code, an explicit command such as `npx @zed-industries/codex-acp@0.11` produces an unparsable version for `parseSemver`, `supportsCodexAcpGpt55` returns true, and `openai-codex/gpt-5.5` can still be forwarded to the old adapter.

## Solution Assessment

No as written. The wrapper boundary and error shape are maintainable, but the version check should parse npm semver/range syntax or conservatively reject explicit specs that can resolve below `0.12.0` instead of treating unparsable explicit specs as compatible.
