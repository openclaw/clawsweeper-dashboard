## Summary

Keep open for maintainer review: the patch appears correct and now has live Messages.app/imsg proof, but it changes opt-in iMessage delivery latency and intentionally preserves a compatibility path while the teardown decision remains open.

## What This Changes

The PR preserves `channels.imessage.coalesceSameSenderDms`, changes its implicit iMessage debounce from 2500 ms to 7000 ms, narrows URL-balloon coalescing to adjacent pairs, and updates iMessage docs/tests.

## Best Possible Solution

Land this only if maintainers accept the longer opt-in compatibility window and keep the linked teardown issue as the canonical future removal decision.

## Reproduction Assessment

Yes, source/log-level: current main uses a 2500 ms opt-in window, while the PR proof reports a real Messages.app URL-preview row arriving about 6 seconds after the command row. I did not run a live local Messages.app reproduction in this read-only review.

## Solution Assessment

Mostly yes: adjacent URL-balloon pairing with a metadata-less fallback is the narrowest compatibility-preserving owner-boundary fix I found. The remaining question is whether maintainers accept the longer opt-in default while keeping the teardown issue open.
