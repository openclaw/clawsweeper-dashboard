## Summary

Keep open for maintainer judgment: the patch looks like a correct interim iMessage fix with strong live proof, but it intentionally changes the opt-in default debounce latency and message merge/split behavior while the teardown issue remains open.

## What This Changes

The PR preserves `channels.imessage.coalesceSameSenderDms`, widens its implicit iMessage debounce window to 7000 ms, narrows URL-balloon coalescing to adjacent split-send pairs, and updates iMessage docs/tests.

## Best Possible Solution

Land this only if maintainers explicitly accept the wider opt-in iMessage latency/message-delivery tradeoff, and keep the teardown issue open until the supported imsg floor and doctor migration decision are settled.

## Reproduction Assessment

Yes, source and contributor live proof give a high-confidence path: current main's 2500 ms opt-in window cannot hold a Messages.app URL-preview row that arrives about 6 seconds after the text row, so the command and URL dispatch separately.

## Solution Assessment

Yes as an interim fix, not as the final teardown. Preserving the opt-in compatibility path while using `balloon_bundle_id` to narrow merges is the maintainable path until maintainers require the newer imsg behavior and migrate the config away.
