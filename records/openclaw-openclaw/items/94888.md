- `impact:data-loss`: Files remain on disk but can be silently absent from memory search, which is practical loss of indexed user memory.

## Summary

Keep open. Current main still has source-supported paths where an uninvoked, unconfigured non-default memory agent has no long-lived manager or watcher, and status dirty output is still derived from manager flags rather than a disk-versus-index comparison.

## What This Changes

The issue requests memory indexing freshness for files created under an uninvoked agent's memory directory.

## Best Possible Solution

Implement a bounded memory-core or gateway reconciliation policy that detects missing or changed memory files for enabled agents without unbounded watcher startup, and make status surface disk/index drift honestly.

## Reproduction Assessment

Yes, at source level. Current main can still defer unconfigured non-default QMD agents at gateway startup, and status-mode managers do not arm a persistent watcher before first use.

## Solution Assessment

Unclear as proposed. A catch-up or drift scan is the right direction, but maintainers should choose the reconciliation boundary because startup watchers, periodic scans, and status-only checks have different resource costs.
