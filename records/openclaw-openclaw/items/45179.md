## Summary

Keep open. Current main has temp-root hardening and a partial gateway service temp-program guard, but it still lacks the requested generic temp-artifact classification, provenance registry, durable promotion contract, and cleanup semantics.

## What This Changes

The issue asks OpenClaw to classify agent-created system-temp artifacts, record provenance and deletion metadata, route durable outputs away from raw temp paths, and flag runtime-critical temp-backed paths.

## Best Possible Solution

Define a small core contract for OpenClaw-owned temp artifact lifecycle metadata and runtime-critical path validation, while leaving workflow-specific janitor policy in plugins or ClawHub unless a concrete core API gap is identified.
