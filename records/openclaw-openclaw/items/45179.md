## Summary

Keep open. Current main has adjacent temp-path hardening and a service-start guard for temp-backed gateway commands, but it still lacks the requested built-in temp artifact provenance registry, scratch/reproducible/durable classification, durable routing, and janitor/cleanup semantics.

## What This Changes

Requested built-in temp artifact provenance, durable-path guardrails, and cleanup awareness for agent-created files outside the workspace.

## Best Possible Solution

Define a narrow shared artifact-provenance contract, expose it through the appropriate SDK/Gateway helper surfaces, and have cleanup/doctor/runtime checks consume that contract without turning plugin-local temp stores into a one-off janitor implementation.

## Reproduction Assessment

Not applicable as a conventional bug reproduction. The gap is source-verifiable: current main has temp path helpers, transcript-derived artifact RPCs, and a partial service temp-path guard, but no durable temp artifact provenance/classification contract.

## Solution Assessment

Unclear until maintainers choose the product and API boundary. The requested direction is plausible, but the maintainable solution needs a small shared contract coordinated with artifact RPCs, plugin SDK helpers, cleanup, doctor, and targeted runtime guards.
