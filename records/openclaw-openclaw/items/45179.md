## Summary

Keep open: current main and the latest release have useful temp-root hardening, workspace-temp guidance, transcript artifact RPCs, and service PATH filtering, but they still lack the requested creation-time temp artifact classes, persistent provenance/durable-routing contract, and janitor cleanup semantics.

## Best Possible Solution

Keep this as a maintainer-scoped design item that separates generic runtime-critical temp-path guardrails and any narrow SQLite-backed provenance API from workflow-specific audit/janitor behavior that should live in plugins or ClawHub unless a missing core API is proven.
