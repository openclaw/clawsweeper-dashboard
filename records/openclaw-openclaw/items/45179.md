## Summary

Keep open. Current main has adjacent managed-temp-root, transcript-artifact, and service-command temp-path guardrails, but it still lacks the requested temp artifact classification/provenance registry, durable-routing contract, and cleanup/janitor semantics.

## What This Changes

Request a built-in temp artifact provenance model with scratch/reproducible/durable classes, durable-path routing, cleanup awareness, and broader runtime-critical temp-path guardrails.

## Best Possible Solution

Define one generic core temp-artifact provenance and durable-routing contract, then wire it through temp helpers, agent/tool artifact creation, doctor or cleanup reporting, and runtime-critical path validation.

## Solution Assessment
