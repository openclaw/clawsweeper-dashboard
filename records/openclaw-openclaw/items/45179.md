## Summary

Keep open: current main has useful temp-root hardening, temp path helpers, static temp-path checks, and a gateway-service temporary-program guard, but it still lacks the requested artifact classification, provenance registry or sidecar, durable promotion path, and cleanup/janitor semantics. The remaining work is a feature/default-safety design across several ownership boundaries, not an already-implemented bug fix.

## What This Changes

Add default temp artifact classification, provenance metadata, durable-path routing, cleanup awareness, and runtime temp-path guardrails for agent-created files outside the workspace.

## Best Possible Solution

Define a small core contract for temp artifact classification/provenance and durable-path validation where OpenClaw owns file creation or runtime-critical paths, while leaving workflow-specific janitor classifiers to plugins or ClawHub unless a missing core API is identified.

## Reproduction Assessment

Not applicable. This is a feature/default-safety request; source review confirms partial temp hardening exists, but there is no already-defined current behavior whose failure can be reproduced as a bug.

## Solution Assessment

Unclear as one change. The direction is plausible, but maintainers should first split core temp-artifact contracts and runtime durable-path guardrails from optional janitor workflow policy.
