## Summary

Keep open: current main has secure temp-root helpers and source-hygiene checks, but it does not implement temp artifact provenance classes, durable routing, cleanup/janitor semantics, or runtime-critical `/tmp` path detection. The known related issue was narrower review-staging work already routed to ClawHub, while this issue still needs a maintainer product/ownership decision for the broader core safety gap.

## What This Changes

The issue requests OpenClaw-owned provenance and guardrails for agent-created temp artifacts and runtime-critical paths that drift into `/tmp` or `/private/tmp`.

## Best Possible Solution

Approve a narrow design split: keep workflow-specific temp provenance in plugin/ClawHub territory unless a generic core state model is accepted, and consider a core doctor/runtime check only for critical paths that must not resolve into system temp.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature/product-scope request. Source inspection confirms current main lacks the requested built-in provenance and runtime-critical temp-path detector, but no live reboot/cleanup failure was reproduced.

## Solution Assessment

Unclear as proposed. The all-in-one provenance registry is likely too broad for an automatic core fix; the maintainable path is a maintainer-approved split between plugin/workflow provenance and narrow core guardrails for critical temp-backed runtime paths.
