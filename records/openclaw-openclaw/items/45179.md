## Summary

Keep open. Current main has useful partial guardrails for OpenClaw temp roots, service command paths, and transcript-derived artifacts, but it does not implement the requested temp artifact classification/provenance registry, durable-routing contract, or cleanup/janitor semantics; that is a broad safe-default feature needing maintainer design.

## What This Changes

The issue requests built-in provenance, classification, cleanup awareness, and runtime guardrails for agent-created artifacts and runtime-critical paths under system temp directories.

## Best Possible Solution

Define a generic core temp-artifact provenance model and durable-routing policy first, then wire it through temp helpers, first-class tool/agent artifact creation, doctor or cleanup reporting, and runtime-critical path checks without making plugins depend on private internals.

## Reproduction Assessment

Not applicable as a feature request. Source inspection confirms current main lacks the requested temp artifact provenance/classification registry, while the production anecdote is not a concrete current-main bug reproduction path.

## Solution Assessment

Unclear until maintainers choose the product contract. A generic temp provenance and durable-routing design is the maintainable direction; scattered per-tool warnings would leave the cleanup semantics ambiguous.
