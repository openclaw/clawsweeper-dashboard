## Summary

Keep open: current main has partial temp-path hardening and session-artifact cleanup, but it still lacks the requested generic temp artifact lifecycle contract, provenance/classification registry, durable promotion semantics, and janitor policy. The remaining work spans SDK/protocol shape, cleanup behavior, and runtime guardrail ownership, so it needs maintainer product scoping rather than conservative cleanup closure.

## What This Changes

The issue asks OpenClaw to add built-in temp artifact classification/provenance, durable-path promotion, cleanup semantics, and runtime-critical temp-path detection.

## Best Possible Solution

Define a small versioned core contract for OpenClaw-owned temp artifact lifecycle metadata and runtime-critical path validation, while keeping workflow-specific janitor policy in plugins or ClawHub unless a concrete core API gap is identified.
