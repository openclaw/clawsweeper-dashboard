## Summary

Keep open. Current main and the latest release have secure temp-root helpers, transcript artifact APIs, and a narrow gateway service temp-program guard, but they still lack the requested temp artifact classification, provenance registry, durable promotion contract, and cleanup semantics.

## What This Changes

Request a first-class OpenClaw temp artifact provenance model with scratch/reproducible/durable classification, cleanup-aware metadata, and guardrails against runtime-critical paths living in system temp.

## Best Possible Solution

Define a small generic core contract for OpenClaw-owned temp artifact lifecycle metadata and runtime-critical path validation, while keeping workflow-specific janitor policy in plugins or ClawHub unless a concrete core API gap is identified.
