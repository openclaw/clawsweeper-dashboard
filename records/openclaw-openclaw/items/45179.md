## Summary

Keep open. Current main has useful managed-temp-root helpers and a narrow service-command temp-path guardrail, but it still lacks the requested artifact classification/provenance registry, durable-routing contract, and cleanup/janitor semantics; the related closed review-staging issue only covered a narrower workflow-specific slice.

## What This Changes

Add built-in classification, provenance metadata, durable routing, cleanup semantics, and runtime-critical temp path guardrails for OpenClaw-created temp artifacts.

## Best Possible Solution

Define a generic temp-artifact provenance and durable-routing contract, then wire it through temp helpers, SDK/tool artifact creation, doctor or cleanup reporting, and broader runtime-critical path checks.
