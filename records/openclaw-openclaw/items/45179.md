## Summary

Keep open. Current main has adjacent temp-path hardening and SDK artifact RPCs, but it still lacks the requested temp artifact provenance contract, scratch/reproducible/durable classification, durable-path routing, and cleanup/janitor semantics.

## What This Changes

Request built-in classification, metadata, cleanup awareness, and runtime guardrails for agent-created temp artifacts and temp-backed runtime paths.

## Best Possible Solution

Define a narrow shared artifact-provenance contract in core, expose it through the appropriate plugin/App SDK helper surfaces, then make cleanup/doctor/runtime checks consume that contract without turning plugin-local stores into a one-off janitor implementation.

## Reproduction Assessment

Not applicable as a conventional bug reproduction. The gap is source-verifiable: current main has temp-path helpers and transcript artifact RPCs, but no scratch/reproducible/durable provenance contract or durable-path routing behavior.

## Solution Assessment

Unclear until maintainers choose the product and API boundary. The requested direction is plausible, but the maintainable path is a small shared contract coordinated with artifact RPCs and targeted service/runtime guards.
