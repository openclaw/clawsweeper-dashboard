## Summary

Keep open. Current main has useful managed-temp-root and narrow runtime temp-path guardrails, but it still lacks the requested temp artifact classification/provenance registry, durable-routing contract, and cleanup/janitor semantics.

## What This Changes

Request a built-in temp artifact provenance model with scratch/reproducible/durable classes, durable-path routing, cleanup awareness, and broader runtime-critical `/tmp` guardrails.

## Best Possible Solution

Define a generic core temp-artifact provenance and durable-routing contract, then wire it through temp helpers, agent/tool artifact creation, doctor or cleanup reporting, and runtime-critical path checks.
