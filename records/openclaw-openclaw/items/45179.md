Keep open: current main has useful temp-path hardening, service-start temporary-program detection, transcript-derived artifact RPCs, and session-artifact cleanup, but it still does not provide the requested generic temp artifact provenance/classification registry, durable promotion contract, or janitor semantics. ## What This Changes

The issue requests built-in provenance, classification, durable-path routing, janitor semantics, and runtime guardrails for OpenClaw-created temporary artifacts.

Define a small core contract for OpenClaw-owned temp artifact lifecycle metadata and runtime-critical path validation, while keeping workflow-specific janitor policy in plugins or ClawHub unless a generic API gap is identified.
