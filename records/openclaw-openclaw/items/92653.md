Keep open: current `main` supports cron named sessions and cross-channel DM continuity, but it does not provide the requested per-agent inbound routing into an arbitrary persistent `session:<id>` lane.

## What This Changes

Request to add per-agent configuration that routes inbound channel messages into a named persistent session shared with cron `sessionTarget` runs.

## Best Possible Solution

Add a narrow core-owned routing contract only after maintainers decide the privacy, reset, and delivery semantics for converging inbound channel traffic into named persistent sessions.
