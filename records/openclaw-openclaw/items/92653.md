## Summary

Keep open: current main and the latest v2026.6.6 release support cron `session:<id>` targets plus DM-scoped routing, but they do not provide a per-agent inbound route-to-named-session contract for DMs or groups. The request is a valid session-routing feature that needs maintainer product and security design before implementation.

## What This Changes

The issue requests an `agents.list[].session` routing option that sends inbound DMs or group chats for an agent to a configured canonical persistent session, optionally by chat type, so cron and human turns share transcript context.

## Best Possible Solution

Design an additive, opt-in canonical session-routing contract with isolated defaults, validated named-session targets, origin-preserving delivery metadata, documented reset/security semantics, and coordinated scope with #19929, #7524, and #79607.

## Reproduction Assessment

Yes at source level for the feature gap: current main deterministically derives inbound session keys from DM scope or channel/peer identity and cannot express a named canonical inbound target.

## Solution Assessment

Unclear as proposed: `agents.list[].session.routeInboundTo` is plausible, but maintainers should decide whether the permanent abstraction belongs on agent config, route bindings, identity links, channel groups, or a typed session-alias model.
