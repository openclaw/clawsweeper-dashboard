## Summary

Keep open: current main and v2026.6.8 still support cron named sessions and DM-scoped routing, but there is no per-agent inbound route-to-named-session contract for DMs or groups. The issue remains a valid feature/design request, not a cleanup close.

## What This Changes

The issue requests a per-agent config surface that routes inbound channel messages into a named persistent session so cron jobs and human conversations share transcript context.

## Best Possible Solution

Design an additive, opt-in canonical session-routing contract with validated named targets, origin-preserving delivery metadata, explicit security/reset semantics, schema/docs updates, and coordination with the related shared-session trackers.

## Reproduction Assessment

Yes, at source level. Current main deterministically derives inbound session keys from DM scope or channel/peer identity, and strict schemas cannot express a named canonical inbound target.

## Solution Assessment

Unclear as proposed. agents.list[].session.routeInboundTo is plausible, but maintainers should decide whether the permanent abstraction belongs on agent config, route bindings, identity links, channel groups, or a typed session-alias model.
