## What This Changes

The issue reports a compound Telegram operations failure involving boolean `channels.telegram.retry.jitter` validation, misleading Telegram SecretRef doctor output, and service restart storms that made the host appear hung.

## Best Possible Solution

Add a doctor-owned compatibility repair for boolean Telegram `retry.jitter` at top-level and account scopes while keeping the canonical runtime schema numeric; split any still-reproducible SecretRef or browser/CDP leftovers into narrower follow-ups.

## Reproduction Assessment

Yes, at source level. `channels.telegram.retry.jitter: true` still reaches the numeric-only shared retry schema, and the inspected doctor compatibility paths do not normalize that boolean first.

## Solution Assessment

Yes for a narrow doctor compatibility repair, but maintainer approval is still needed for the schema-versus-doctor contract choice. Runtime schema coercion would change the canonical numeric config contract, while `openclaw doctor --fix` matches the repository’s legacy config repair policy.
