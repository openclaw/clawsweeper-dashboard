## Best Possible Solution

Add a doctor-owned compatibility migration for boolean Telegram `retry.jitter` at top-level and account scopes, keep the canonical runtime schema numeric, and split any still-reproducible SecretRef or browser/CDP leftovers into narrower follow-ups.

## Solution Assessment

Yes for the remaining narrow bug if implemented as a doctor compatibility repair. Runtime schema coercion would weaken the canonical numeric config contract; `openclaw doctor --fix` is the safer upgrade path.
