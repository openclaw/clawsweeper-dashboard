## Summary

Close as ClawHub/plugin scope. The request is for an optional new panel_ask skill/tool that can be built as an external native plugin with a bundled skill, and current main already exposes the needed plugin tool registration, plugin-shipped skill loading, and ClawHub distribution surfaces without a missing core API.

## What This Changes

Request to contribute a new @openclaw/skill-panel-ask capability that registers panel_ask, fans a prompt out to multiple external AI voices, writes per-voice artifacts, and optionally synthesizes with a cross-family rule.

## Best Possible Solution

Publish this as an external native OpenClaw plugin on ClawHub, with a bundled SKILL.md if the panel operating guidance is too long for the tool description. Core should only reopen the topic if implementation reveals a missing generic plugin API or maintainers decide the capability needs official bundled ownership for product or security reasons.

## Reproduction Assessment

Not applicable as a feature RFC. The high-confidence verification path is static review of current main showing the existing plugin tool, plugin-shipped skill, and ClawHub publication surfaces needed for an external implementation.

## Solution Assessment

Yes. Routing this to a ClawHub plugin is the narrowest maintainable solution because the behavior is optional, integration-heavy, and enforceable inside plugin code without adding OpenClaw core policy.
