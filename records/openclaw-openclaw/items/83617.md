## What This Changes

The issue asks OpenClaw to accept cron `payload.model` values like `openai/gpt-5.4` and `openai/gpt-5.4-mini` when matching `agents.defaults.models` entries select `agentRuntime.id: "codex"`.

## Best Possible Solution

Keep the canonical OpenAI/Codex migration path, but make cron allowlist resolution accept those canonical entries and cover both direct config and doctor-repaired upgrade config in focused regression tests.

## Reproduction Assessment

Yes. A live v2026.5.20 upgrade repro is now reported, and current main has no relevant cron/model-selection source changes after that release; I did not run a local cron job in this read-only review.

## Solution Assessment

Yes. The maintainable fix is to repair the existing cron allowlist/runtime-resolution contract for canonical `openai/gpt-*` entries with Codex runtime policy, not to require users to revert to legacy `openai-codex/*` refs.
