## What This Changes

The issue requests that isolated cron accept `openai/gpt-5.4` and `openai/gpt-5.4-mini` payload models when matching `agents.defaults.models` entries use `agentRuntime.id: "codex"`.

## Best Possible Solution

Repair the existing cron/model allowlist resolution so canonical `openai/gpt-*` entries remain valid when their provider/model runtime policy selects Codex, with regression coverage for `gpt-5.4`, `gpt-5.4-mini`, and the upgrade shape reported on v2026.5.20.

## Solution Assessment

Yes. The narrow maintainable fix is to repair the existing allowlist/runtime-resolution contract, not to require users to switch cron payloads and allowlists back to legacy `openai-codex/*` refs.
