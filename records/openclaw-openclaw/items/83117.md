## Summary

This PR addresses a real current-main session-state bug, but it should stay open because the proposed whole-string tail slice can drop an intentionally prepended compaction summary in long compacted reseeds. The contributor supplied usable real-environment proof for the primary failure, and the remaining blocker is a narrow repair in the same truncation path.

## What This Changes

The PR changes CLI reseed history rendering to keep the newest transcript tail, moves the truncation marker to the front, and adds a regression test for preserving the latest messages.

## Best Possible Solution

Keep the recent-tail behavior, but make truncation structure-aware so a present compaction summary is pinned while only the post-summary transcript body is tail-capped.

## Reproduction Assessment

Yes. Source inspection on current main shows oldest-first rendering followed by a prefix slice, so an over-cap fixture with final marker messages will drop the latest markers; the PR's added test is the right pure-function reproduction path.

## Solution Assessment

No. Tail slicing is the right direction for the primary bug, but the current patch slices the entire rendered transcript and can discard the deliberately pinned compaction summary; the maintainable fix should split or classify that prefix before applying the cap.

## Review Findings
