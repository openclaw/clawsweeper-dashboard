The PR adds embedded Pi `turn_start`/`turn_end` handling, forwards those boundaries as `stream: "turn"` agent events, and adds callback/global-bus regression tests.

## Best Possible Solution

Land a repaired, rebased patch that forwards Pi turn boundaries through the existing scheduled embedded event chain, uses the current Pi package, preserves both listener paths, and includes focused regression coverage plus real embedded-session proof.

Yes. A high-confidence source reproduction is to pass `turn_start` or `turn_end` through `createEmbeddedPiSessionEventHandler` on current main: the documented dependency events are not handled and fall through the default case.

## Solution Assessment

No. Forwarding turn boundaries as `stream: "turn"` is the right narrow direction, but this branch must remove the parse error, update the Pi import, preserve scheduled event ordering, rebase, and provide real behavior proof before merge.
