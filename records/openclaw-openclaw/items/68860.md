## Summary

Keep this PR open because current main still drops the documented embedded Pi turn boundary events, but the PR head is not mergeable as-is: it has a parse error, a stale Pi package import, bypasses the existing event scheduler, and lacks external real behavior proof.

## What This Changes

The PR adds embedded Pi `turn_start` and `turn_end` handling and regression tests for forwarding those boundaries as `stream: "turn"` agent events.

## Best Possible Solution

Land a repaired and rebased patch that forwards Pi turn boundaries through the existing scheduled embedded event chain, uses the current Pi package, preserves both listener paths, and includes focused regression coverage plus real embedded-session proof.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to send `turn_start` or `turn_end` through `createEmbeddedPiSessionEventHandler` on current `origin/main`: the documented dependency events have no switch cases and fall through `default`.

## Solution Assessment

No. Forwarding turn boundaries as `stream: "turn"` is the right narrow direction, but this branch must remove the parse error, update the Pi import, preserve scheduled event ordering, and provide real behavior proof before merge.
