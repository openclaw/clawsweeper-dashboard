## Summary

Keep this open: current main still drops embedded Pi turn boundary events, but the PR head is not mergeable because it has a syntax error, uses the old Pi package import on current main, bypasses the existing scheduled event ordering, and lacks real behavior proof.

## What This Changes

The PR forwards embedded Pi turn_start and turn_end events as OpenClaw agent stream: "turn" events and adds subscription regression tests.

## Best Possible Solution

Land a repaired patch that handles turn_start and turn_end through the scheduled embedded event chain, imports the current Pi package, emits a stable turn payload to both listener paths, and includes focused regression plus redacted real behavior proof.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to pass turn_start or turn_end through createEmbeddedPiSessionEventHandler on current main: the switch has no matching cases and returns through default while @earendil-works/pi-agent-core 0.74.0 emits those events.

## Solution Assessment

No. Forwarding turn boundaries as a turn stream is the right narrow direction, but this PR must remove the parse error, use the current dependency import, and preserve scheduleEvent ordering before it is mergeable.
