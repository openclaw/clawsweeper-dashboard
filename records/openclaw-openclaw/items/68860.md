## Summary

Keep open: current main still drops embedded Pi turn_start/turn_end events, but the latest PR head is not mergeable because it has a parse error, uses the old Pi package import, bypasses the event scheduling chain, and lacks real embedded-session proof.

## What This Changes

The PR adds turn_start and turn_end handling for embedded Pi sessions, forwards them as stream: "turn" events, and adds focused regression tests for callback and global event bus delivery.

## Best Possible Solution

Land a repaired patch that forwards turn boundaries through the scheduled embedded event chain using the current Pi dependency, keeps the payload stable for both listener paths, and includes focused regression plus real embedded-session proof.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to pass turn_start or turn_end through createEmbeddedPiSessionEventHandler on current main: Pi core emits those events and the docs list them, but the switch has no matching cases and falls through default.

## Solution Assessment

No. Forwarding the boundaries as stream: "turn" is the right narrow direction, but this branch must remove the parse error, import the current Pi package, preserve scheduleEvent ordering, and add real behavior proof before merge.
