## Summary

Keep this PR open: it addresses a real embedded Pi event gap on current main, but the branch is not mergeable because the new handler file is unparsable, imports the old Pi package, bypasses the embedded event scheduler, and still lacks real embedded-session proof.

## What This Changes

The PR adds embedded Pi `turn_start` and `turn_end` handling, emits them as `stream: "turn"` events, and adds regression coverage for callback and global bus delivery.

## Best Possible Solution

Land a repaired patch that forwards Pi turn boundaries through the existing scheduled embedded event chain, uses the current Pi package, preserves both listener paths, and includes focused regression tests plus real embedded-session proof.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to pass `turn_start` or `turn_end` through `createEmbeddedPiSessionEventHandler` on current main: the documented dependency events are not handled and fall through the default case.

## Solution Assessment

No. Forwarding turn boundaries as `stream: "turn"` is the right narrow direction, but this branch must remove the parse error, update the Pi import, preserve scheduled event ordering, and provide real behavior proof before merge.

  - confidence: 0.88
