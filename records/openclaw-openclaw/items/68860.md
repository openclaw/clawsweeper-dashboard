## Best Possible Solution

Land a repaired and rebased patch that forwards Pi turn boundaries through the existing scheduled embedded event chain, uses the current Pi package, preserves both listener paths, and includes focused regression coverage plus real embedded-session proof.

Yes, as a source-level reproduction. Sending `turn_start` or `turn_end` through `createEmbeddedPiSessionEventHandler` on current main follows a switch with no matching cases, despite docs and the Pi core contract defining those events.

## Solution Assessment

No. Forwarding turn boundaries as `stream: "turn"` is the right narrow direction, but this branch must remove the parse error, update the Pi import, preserve scheduled event ordering, and provide real behavior proof before merge.
