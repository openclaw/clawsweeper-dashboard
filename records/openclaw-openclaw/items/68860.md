## Summary

Keep open. Current main still drops embedded Pi turn boundaries, so the PR targets a real agents observability gap, but the current head is not mergeable because it leaves invalid TypeScript and bypasses the existing ordered event scheduling path.

## What This Changes

The PR adds embedded Pi turn_start/turn_end handling, forwards them as stream: "turn" events, and adds focused subscription tests for per-run and global delivery.

## Best Possible Solution

Land a repaired patch on current main that schedules turn_start/turn_end through the existing embedded event chain, emits a stable turn payload to both delivery paths, covers ordering and delivery with focused tests, and records the event-surface change in the changelog.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to send turn_start or turn_end through createEmbeddedPiSessionEventHandler on current main: the switch has no matching case and returns through default while pi-agent-core 0.71.1 emits those events.

## Solution Assessment

No. The direction is the right narrow fix, but this patch needs a syntax repair, scheduler integration, and the required changelog entry before it is the best mergeable solution.
