## Summary

Keep open: current main still drops dependency-documented turn boundary events, but this branch has merge-blocking defects and still lacks real embedded-session proof.

## What This Changes

The PR adds embedded Pi `turn_start` and `turn_end` handling, forwards them as `stream: "turn"` events, and adds subscription/event-bus regression tests.

## Best Possible Solution

Land a repaired patch that forwards turn boundaries through the scheduled embedded event chain using the current Pi dependency, preserves payloads for both listener paths, and includes focused regression plus real embedded-session proof.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to pass `turn_start` or `turn_end` through `createEmbeddedPiSessionEventHandler` on current main: Pi core emits those events and the docs list them, but the switch has no matching cases and falls through `default`.

## Solution Assessment

No. Forwarding the boundaries as `stream: "turn"` is the right narrow direction, but this branch must remove the parse error, import the current Pi package, preserve `scheduleEvent` ordering, and add real behavior proof before merge.
