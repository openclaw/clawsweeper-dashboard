## Summary

Keep open: current main still drops embedded Pi turn boundary events even though the docs and upstream session contract include them, but this PR head is not mergeable as-is because it has a syntax error, stale dependency import, unscheduled event handling, and no real embedded-session proof.

## What This Changes

The PR adds embedded Pi `turn_start` and `turn_end` handling, forwards them as `stream: "turn"` events to both listener paths, and adds regression tests.

## Best Possible Solution

Land a repaired patch that forwards turn boundaries through the scheduled embedded event chain using the current Pi dependency, preserves both listener paths, adds focused regression coverage, and includes real embedded-session proof.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to pass `turn_start` or `turn_end` through `createEmbeddedPiSessionEventHandler` on current main: the upstream session event type includes those events and the docs list them, but the switch drops them via `default`.

## Solution Assessment

No. Forwarding the boundaries as `stream: "turn"` is the right narrow direction, but this branch must remove the parse error, import the current Pi package, preserve `scheduleEvent` ordering, and add real behavior proof before merge.
