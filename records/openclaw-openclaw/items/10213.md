## Summary

Keep open. Current main and the latest release still lack an Anthropic-native server-side compaction path: config/schema support is absent, Anthropic request construction does not send `context_management.edits`, and the stream parser does not preserve compaction blocks or handle the `compaction` stop reason.

## What This Changes

The issue requests opt-in Anthropic server-side compaction support for OpenClaw agents, including provider request shaping, response handling, logging, and docs/config coverage.

## Best Possible Solution

Keep this as the focused Anthropic server-side compaction tracker and implement a maintainer-approved opt-in path that sends Anthropic context-management edits, round-trips compaction blocks, handles `compaction` stop reasons and usage metadata, documents the config, and leaves existing local compaction unchanged by default.

## Reproduction Assessment

Yes, at source level: current main lacks the requested Anthropic compaction config/schema, request payload injection, compaction block parsing, and `compaction` stop-reason handling. This is a missing feature rather than a broken existing behavior contract.

## Solution Assessment

Unclear until maintainers choose the product contract: Anthropic-native compaction is a plausible provider-owned direction, but the proposed nested config is only one option and overlaps broader context-management passthrough design.
