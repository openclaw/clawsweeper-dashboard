## Summary

Keep open. Current main and the latest release still lack the requested Anthropic-native server-side compaction path: the config schema has no opt-in surface, Anthropic request construction does not send `context_management`, and the stream parser has no compaction block or `compaction` stop-reason handling.

## What This Changes

The issue asks OpenClaw to add opt-in Anthropic server-side compaction with config, Anthropic beta/payload support, stream response handling, logging, and Anthropic-only scope.

## Best Possible Solution

Keep this as the focused Anthropic server-side compaction tracker, then implement a maintainer-approved opt-in path in Anthropic provider/transport/config/docs with compaction block round-trip, stop-reason handling, targeted tests, and live beta proof.
