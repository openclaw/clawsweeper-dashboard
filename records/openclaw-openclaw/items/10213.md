## Summary

This focused request is still necessary: current main can add configured Anthropic beta headers, but it does not send native compaction parameters, recognize the compaction stop reason, or preserve provider compaction blocks for later turns. The linked implementation PR closed without merging, while the broader open context-management issue adds separate memory-tool and context-editing scope and does not yet safely supersede this tracker.

## What This Changes

Add opt-in Anthropic-native server-side context compaction, including request parameters, response round-tripping, configuration, logging, and documentation.

## Best Possible Solution

Keep one focused Anthropic-native compaction tracker and implement a provider-owned, opt-in model parameter that follows the current official contract, round-trips compaction blocks and metadata, and leaves OpenClaw's local compaction behavior unchanged unless explicitly selected.

## Reproduction Assessment

Yes at source level: current main directly shows that the Anthropic request and stream paths lack the complete native-compaction lifecycle. This is a missing capability rather than a failing existing behavior contract.

## Solution Assessment

No, the exact global nested configuration proposed in the issue is not yet the best-proven design. A narrow provider/model parameter integrated with the native Anthropic transport would avoid competing compaction-policy sources and keep provider-specific behavior at its owner boundary.
