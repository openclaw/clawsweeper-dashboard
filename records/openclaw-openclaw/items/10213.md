## Summary

## What This Changes

Add opt-in Anthropic-native server-side compaction with provider-specific request shaping, compaction-block history round-trip, stop/usage handling, logging, tests, and documentation.

## Best Possible Solution

Use one opt-in Anthropic model/transport contract that injects the current native compaction request, preserves typed compaction blocks in canonical history, maps stop and iteration usage correctly, and clearly documents interaction with local compaction without adding a parallel policy hierarchy.

## Reproduction Assessment

Yes for the capability gap: Anthropic’s documented native-compaction contract can be directly compared with current-main request construction, content-block types, stop mapping, and tests, which show the path is absent. This is a feature gap rather than a failing existing OpenClaw contract.

## Solution Assessment

The requested capability is the right provider improvement, but the issue’s exact nested global config is not yet the best proven design. The narrowest maintainable solution is a typed Anthropic model/transport option coordinated with the existing local compaction and provider-parameter surfaces.
