## Summary

Current main still lacks Anthropic-native server-side compaction, and neither closed implementation PR provides a merged replacement. This focused tracker remains useful because the broader open requests add separate context-editing, memory-tool, or cache-TTL decisions.

## What This Changes

Add opt-in Anthropic-native server-side compaction, including provider-scoped request configuration, compaction response handling, logging, and unchanged defaults for other providers.

## Best Possible Solution

Implement a direct-Anthropic-only opt-in contract using the current compaction API, round-trip compaction blocks through canonical transcript state, handle streaming and terminal compaction events, aggregate iteration usage, and preserve existing local compaction as the default.

## Reproduction Assessment

Yes at the source level. Current main has no Anthropic-native compaction request, compaction-block history, stop-reason, streaming-delta, or iteration-usage path, while Anthropic's current contract requires those surfaces.

## Solution Assessment

No, not exactly as proposed. The capability is appropriate, but implementation should use Anthropic's current compact-2026-01-12 contract and a maintainer-approved provider/model parameter shape rather than copying the older header and nested global configuration verbatim.
