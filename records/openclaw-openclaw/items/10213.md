Current main still lacks Anthropic-native server-side compaction support, and a member already closed the duplicate Anthropic compaction report in favor of this issue, so this remains cleanup-resistant until maintainers either implement it or explicitly consolidate it into a broader tracker.

## What This Changes

The issue requests opt-in Anthropic server-side compaction support, including public config, Anthropic `context_management.edits` request wiring, compaction stream/history handling, logging, and docs.

## Best Possible Solution

Keep this focused tracker open unless maintainers explicitly consolidate it, then implement a documented, opt-in Anthropic provider path that validates config, sends the required beta/header and `context_management.edits`, preserves compaction blocks through stream/history replay, handles pause/resume and usage metadata, and leaves existing local compaction behavior unchanged by default.
