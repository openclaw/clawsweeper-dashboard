## Summary

Keep open. Current main still lacks an Anthropic-native server-side compaction implementation: there is no accepted config/schema surface, no Anthropic `context_management.edits` request wiring, no compaction block stream/history round-trip, and the open related issues overlap without a clear maintainer-selected supersession.

## What This Changes

The issue requests opt-in Anthropic server-side compaction support with provider-specific config, beta/header handling, response-event handling, logging, and Anthropic-only request integration.

## Best Possible Solution

Keep this as the focused Anthropic server-side compaction tracker unless maintainers explicitly consolidate it, then add a documented opt-in provider-native path that validates config, gates beta headers, injects `context_management.edits`, preserves compaction blocks through streaming/history, handles `compaction` stop reasons and usage, and covers the behavior with targeted tests.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request for a new Anthropic-native compaction mode. Source inspection gives high confidence that current main does not expose the requested config, request payload, stream/history round-trip, stop-reason handling, or docs behavior.

## Solution Assessment

Unclear for the exact proposed `agents.defaults.compaction.serverSide` shape. The provider-native direction is reasonable, but the maintainable fix should be coordinated with the Anthropic provider/config owners and the related context-management issues before choosing names and scope.
