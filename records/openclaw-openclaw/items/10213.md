## Summary

Keep open. Current main still lacks Anthropic-native server-side compaction: the config/schema surface has no Anthropic compaction option, request construction does not inject Anthropic `context_management.edits`, stream parsing does not preserve compaction blocks or `compaction_delta`, and `compaction` remains an unhandled stop reason. The linked implementation PR is closed unmerged, while #54041 and #65287 are broader related trackers rather than a clean replacement for this focused request.

## What This Changes

The issue requests opt-in Anthropic server-side compaction support with provider-specific config, beta/header handling, `context_management.edits` request injection, streaming response handling, logging, and Anthropic-only scoping.

## Best Possible Solution

Keep this as the focused Anthropic server-side compaction tracker, then implement a maintainer-approved opt-in path across Anthropic provider params/config docs, request payload construction, compaction block history round-trip, streaming delta handling, `compaction` stop-reason handling, usage/logging support, and targeted regression tests.
