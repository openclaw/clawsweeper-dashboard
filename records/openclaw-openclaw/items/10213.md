## What This Changes

The issue requests an opt-in Anthropic server-side compaction mode for OpenClaw's Anthropic provider/config surface.

## Best Possible Solution

Keep this as the focused Anthropic native compaction tracker and implement a maintainer-approved opt-in path through Anthropic provider/model params, including beta/header handling, `context_management.edits`, compaction-block history round-trip, `compaction` stop-reason handling, usage metadata, docs, and targeted tests.
