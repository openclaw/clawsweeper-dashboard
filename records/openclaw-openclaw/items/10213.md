## Summary

Keep open. Current main still lacks Anthropic-native server-side compaction across config/schema, Anthropic request payloads, stream/history handling, usage accounting, and provider docs; the related closed duplicate points back to this issue, while the broader open Anthropic context-management issues overlap but do not clearly supersede this focused tracker.

## What This Changes

Request support for an opt-in Anthropic-only server-side compaction mode using Anthropic beta headers, `context_management.edits`, compaction response handling, and docs/config updates.

## Best Possible Solution

Keep this as the focused Anthropic server-side compaction tracker, or explicitly consolidate it into a canonical Anthropic context-management design item, then add a documented opt-in provider-native path covering config/schema, beta/header gating, `context_management.edits`, compaction block replay, `compaction` stop reason handling, usage aggregation, and targeted tests.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request for a new Anthropic-native compaction mode. Source inspection gives high confidence that current main does not expose the requested config, request payload, stream/history round-trip, stop-reason handling, usage handling, or docs behavior.

## Solution Assessment

Unclear for the exact proposed `agents.defaults.compaction.serverSide` shape. The technical direction is valid, but the maintainable fix should be designed with the Anthropic provider/config owners and coordinated with https://github.com/openclaw/openclaw/issues/54041 and https://github.com/openclaw/openclaw/issues/65287 before implementation chooses names and scope.
