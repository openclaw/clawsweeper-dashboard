## Summary

Keep open. Current main still lacks Anthropic-native server-side compaction: there is no config/schema option, Anthropic request construction does not inject `context_management.edits`, stream parsing does not preserve compaction blocks or `compaction_delta`, and `compaction` remains an unhandled stop reason. The prior implementation PR is closed unmerged, while #54041 and #65287 are broader related trackers rather than a clean replacement for this focused request.

## What This Changes

The issue requests opt-in Anthropic server-side compaction support across config, Anthropic beta/header handling, request payload construction, streaming response handling, logging, docs, and tests.

## Best Possible Solution

Keep this as the focused Anthropic server-side compaction tracker, then implement a maintainer-approved opt-in path across Anthropic provider params/config docs, request payload construction, compaction block history round-trip, streaming delta handling, `compaction` stop-reason handling, usage/logging support, and targeted regression tests coordinated with #54041 and #65287.

Not applicable as a feature request rather than a bug report. The current-main verification path is high-confidence: source and docs inspection show the requested config, request payload, stream parsing, stop-reason handling, and docs support are absent.
