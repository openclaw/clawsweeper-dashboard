## Summary

Keep open. Current main still lacks the requested Anthropic-native server-side compaction path: there is no public config/schema support, Anthropic request construction does not inject `context_management.edits`, streaming does not preserve compaction blocks or the `compaction` stop reason, and the related broader trackers remain open.

## What This Changes

Request opt-in Anthropic server-side compaction support through config, Anthropic request shaping, stream response handling, logging, and provider docs/tests.

## Best Possible Solution

Implement a maintainer-approved, opt-in Anthropic provider/native transport path that sends the current beta/header and `context_management.edits`, preserves compaction blocks through history and streaming, accounts for `compaction` stop/usage metadata, and documents the Anthropic-specific behavior.

## Solution Assessment
