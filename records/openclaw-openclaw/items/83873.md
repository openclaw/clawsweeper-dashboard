## Summary

Keep open. This PR targets a valid current-main Telegram inbound formatting bug and moves the fix to the Telegram helper/body layer, but it still needs real Telegram proof and a small preformatted-code correctness fix before merge.

## What This Changes

The PR renders Telegram inbound text entities as Markdown in the agent-facing body, exports the helper, adds Telegram regression tests, and documents the fix in the changelog.

## Best Possible Solution

Keep the Telegram-owned entity renderer, make preformatted fences longer than the content's longest backtick run, add a regression for that case, and provide redacted real Telegram proof of formatted inbound text reaching the agent.

## Reproduction Assessment

Yes, source-reproducible: a Telegram message with text plus a non-link formatting entity reaches the current `expandTextLinks` call, which ignores every non-`text_link` entity before `bodyText` is used for the agent. I did not run a live Telegram repro in this read-only review.

## Solution Assessment

Not yet. The helper/body-layer direction is the right narrow fix, but the PR should produce valid Markdown for `pre` content containing backtick fences and include real Telegram proof before merge.
