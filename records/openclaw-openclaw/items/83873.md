## What This Changes

The PR renders Telegram inbound text entities into Markdown before building the agent-facing prompt body, adds helper and body regression tests, re-exports the helper, and adds a changelog entry.

## Best Possible Solution

Land a Telegram-owned entity renderer that uses Telegram UTF-16 offsets and emits unambiguous Markdown for supported inbound entities before `rawBody` and `bodyText` are finalized.

## Reproduction Assessment

Yes. Current main still routes Telegram text through link-only `expandTextLinks`, and the helper ignores bold/italic/code-style entities before `bodyText` becomes model-facing context; the PR finding is also source-reproducible from the fixed three-backtick `pre` fence.

## Solution Assessment

No, not yet. The boundary is right, but the patch should make `pre` fences content-safe and include real Telegram behavior proof before it is the best mergeable solution.
