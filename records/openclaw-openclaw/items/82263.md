## Summary

Keep open; this PR targets a real source-reproducible OpenAI-completions parser bug, but the current head still fails type checking in the changed finalization loop.

## What This Changes

The PR changes the OpenAI-completions stream parser to accumulate streamed tool-call deltas by index/id and adds a regression test for split parallel tool-call arguments.

## Best Possible Solution

Keep the indexed accumulation and regression coverage, but finalize typed ToolCallBlock values or narrow partialArgs before parsing so the parser builds cleanly.

## Reproduction Assessment

Yes for the source path. Current main's single-block OpenAI-completions parser can misroute interleaved indexed tool-call deltas, and the PR's mocked stream captures that failing shape without live provider access.

## Solution Assessment

Mostly yes, but not as submitted. Keying streamed tool calls by index/id is the narrow maintainable fix, while the finalization loop must avoid reading unknown partialArgs from the broad output.content record type.
