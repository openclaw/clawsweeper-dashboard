## What This Changes

The branch changes the OpenAI-completions streaming parser to accumulate tool-call deltas by index/id with per-block argument byte tracking and adds a regression test for split parallel tool-call indices.

## Best Possible Solution

Keep the indexed accumulation fix, replace the test's `__testing` call with the imported `testing` helper, then land after the focused parser test and `check-test-types` pass.

## Reproduction Assessment

Yes at source level: current main uses a single active tool-call block while the OpenAI stream contract provides indexed tool-call deltas, and the added mocked stream exercises split parallel indices. I did not run local tests in this read-only review.

## Solution Assessment

Mostly yes: keying streamed tool-call deltas by `toolCall.index` with an id fallback is the narrow maintainable parser fix. The current branch is not yet mergeable because the new regression test still has a helper-name type error.
