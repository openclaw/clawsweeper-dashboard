## Summary

Keep open: this is a plausible focused bug fix with sufficient live-output proof, but the current PR head still fails TypeScript because `finishAllToolCallBlocks` reads `partialArgs` from the broad `output.content` element type.

## What This Changes

The PR changes the OpenAI-completions stream parser to accumulate streamed tool-call deltas by `index`/`id` and adds a regression test for split parallel tool-call arguments.

## Best Possible Solution

Keep the indexed parser repair, fix typed finalization without leaking scratch fields into emitted content, then land once type checks and the focused parser regression pass.

## Reproduction Assessment

Yes for source reproduction: current main uses a single active tool-call block while the PR fixture models two streamed tool-call indices with split argument deltas. I did not run tests because this was a read-only review.

## Solution Assessment

No, not as submitted. Index/id keyed accumulation is the right direction, but `finishAllToolCallBlocks` needs to iterate typed `ToolCallBlock` values or narrow `partialArgs` before parsing so the patch builds.
