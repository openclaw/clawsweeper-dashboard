## Summary

Keep open: the PR addresses a real current-main parser bug and the latest head removes the prior scratch-field leak, but the new map typing is a merge blocker because it does not typecheck cleanly under strict TypeScript.

## What This Changes

The branch changes the OpenAI-completions stream parser to accumulate streamed tool-call deltas by index/id with per-block byte tracking and adds a regression test for split parallel tool-call arguments.

## Best Possible Solution

Keep the index/id accumulation approach, but introduce an explicit ToolCallBlock type for the parser maps and byte WeakMap so the implementation compiles and preserves the public output shape.

## Reproduction Assessment

Yes. A mocked OpenAI-completions stream with two tool-call indices introduced together and later split argument deltas exercises the current-main single-currentBlock bug; I did not run the repo test because this review is read-only.

## Solution Assessment

No. The design is the right fix direction, but the implementation should type the accumulated blocks with an explicit tool-call block alias instead of NonNullable<typeof currentBlock>.
