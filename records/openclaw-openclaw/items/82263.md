## What This Changes

The PR changes `src/agents/openai-transport-stream.ts` to accumulate streamed OpenAI-completions tool-call deltas by index/id and adds a regression test for split parallel tool-call arguments.

## Best Possible Solution

Land the indexed OpenAI-completions tool-call accumulation with the focused regression test once the relevant CI and proof gates are reconciled.

## Reproduction Assessment

Yes for source-level reproduction: current main can switch streamed tool-call blocks without considering `toolCall.index`, and the PR adds a mocked stream with two split indices that exercises the failure mode. I did not run local tests or a live provider in this read-only review.

## Solution Assessment

Yes. Keying streamed tool-call deltas by numeric index with an id fallback is the narrow maintainable fix and matches both the OpenAI SDK stream contract and the bundled pi-ai parser pattern.
