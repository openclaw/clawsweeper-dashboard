## Summary

Keep open: the PR addresses a real OpenAI-completions streaming bug, but the latest head still fails type checking in the touched parser because `finishAllToolCallBlocks()` reads `partialArgs` from generic output content without narrowing it to a typed tool-call block.

## What This Changes

The PR changes the OpenAI-completions stream parser to accumulate tool-call deltas by stream index/id and adds a regression test for split parallel tool-call arguments.

## Best Possible Solution

Keep the indexed accumulation design, repair the narrow type error by finalizing typed tool-call blocks only, then rerun the focused parser test and type checks.

## Reproduction Assessment

Yes. Current main source shows a single active tool-call block for all streamed deltas, and the PR adds a focused mocked stream with two `tool_calls` indices that exercises the misrouting path; I did not run it locally in this read-only review.

## Solution Assessment

Mostly yes. Keying streamed tool-call accumulation by `toolCall.index` with id fallback matches the OpenAI stream contract and the pi-ai parser pattern, but this PR head is not mergeable until the new finalization helper is properly typed.
