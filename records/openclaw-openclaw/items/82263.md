## What This Changes

The PR changes the OpenAI-completions stream parser to accumulate tool-call deltas by `tool_call.index`/id with per-block byte tracking and adds a regression test for parallel indexed tool calls.

## Best Possible Solution

Land the index/id accumulation with the regression test after parser-area maintainer review of the exact head, preserving the id fallback for OpenAI-compatible providers.

## Reproduction Assessment

Yes, at source level. A stream with two `delta.tool_calls` indices and later argument deltas exercises current main's single-`currentBlock` path; I did not run tests in this read-only checkout.

## Solution Assessment

Yes. Accumulating by OpenAI's streamed `toolCall.index` with an id fallback matches the SDK contract and the bundled pi-ai parser, and is narrower than provider-specific retries or tool-call dedupe.
