## Summary

Keep open: current main still uses single-block OpenAI-completions tool-call parsing, while this PR implements the indexed accumulation that the dependency contract supports and adds focused regression coverage. I found no blocking code findings; remaining merge work is maintainer review plus CI/proof-gate reconciliation.

## What This Changes

The PR changes `src/agents/openai-transport-stream.ts` to accumulate streamed OpenAI-completions tool-call deltas by index/id with per-block byte accounting, and adds a regression test for split parallel tool-call arguments.

## Best Possible Solution

Land the indexed OpenAI-completions tool-call accumulation with the regression test after relevant CI/proof gates are reconciled; keep https://github.com/openclaw/openclaw/issues/71273 separate unless this parser fix is proven to cover that anthropic-messages loop.

## Reproduction Assessment

Yes for source-level reproduction: current main can switch tool-call blocks without considering `toolCall.index`, and the PR's mocked stream exercises two split indices. I did not run live provider proof in this read-only review.

## Solution Assessment

Yes: keying streamed tool-call deltas by `toolCall.index` with an id fallback and per-block byte limits is the narrow maintainable fix, and it aligns with both openai-node's delta contract and the bundled pi-ai parser pattern.
