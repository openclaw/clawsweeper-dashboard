## Summary

Keep open. The PR addresses a real OpenAI-completions streaming parser bug and includes sufficient copied live before/after output, but the current head fails TypeScript because the new finalization loop reads `partialArgs` from the broad emitted content type.

## What This Changes

The PR changes the OpenAI-completions stream parser to accumulate tool-call deltas by stream index/id and adds a regression test for split parallel tool-call arguments.

## Best Possible Solution

Repair the typed finalization loop, keep the indexed accumulation in the OpenAI-completions transport, and land it with the focused parser regression after typecheck and CI pass.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for the source path: current main's single-block parser and the PR's mocked two-index stream demonstrate the bug without live provider access; CI also reproduces the PR's current type error.

## Solution Assessment

Is this the best way to solve the issue? Mostly yes, but not as submitted: keying by OpenAI's streamed `toolCall.index` is the right boundary, while `finishAllToolCallBlocks` needs a small typed repair before merge.

## Review Findings
