## Summary

Keep open: the reported parser bug is credible and the fix direction is focused, but the PR head still fails type checking in the new finalization loop.

## What This Changes

The PR changes the OpenAI-completions stream parser to accumulate streamed tool-call deltas by index/id and adds a regression test for split parallel tool-call arguments.

## Best Possible Solution

Keep the indexed tool-call accumulation, but finalize typed ToolCallBlock objects or narrow partialArgs before parsing so the parser builds and the regression test can land.

## Reproduction Assessment

Yes for the source path: current main’s single-block OpenAI-completions parser can misroute interleaved tool-call deltas, and the PR’s added mocked stream demonstrates the intended failing shape without live provider access.

## Solution Assessment

Mostly yes, but not as submitted: keying by streamed toolCall.index/id is the narrow maintainable fix, while the finalization loop must avoid reading partialArgs from the broad output.content record type.
