## Summary

Current main and the shipped v2026.6.5+ Feishu plugin already implement the central fix: streaming card updates now send cumulative merged text rather than suffix-only token fragments, so this report matches a previously fixed Feishu CardKit truncation bug.

## What This Changes

The issue requests that Feishu streaming cards preserve cumulative reply content instead of replacing the visible card body with each token, leaving only the last character visible.

## Best Possible Solution

Keep the shipped full-text Feishu CardKit streaming behavior and handle any fresh v2026.6.6 reproduction as a new, evidence-backed report rather than reopening the already-fixed suffix-payload bug.

## Reproduction Assessment

No current-main failure path remains from source inspection: current main and v2026.6.6 send cumulative Feishu CardKit content and tests assert full update bodies. The pre-fix regression is well documented by the closed related issue and merged PR.

## Solution Assessment

Yes. The best solution is the shipped owner-local Feishu plugin fix that removed suffix-only CardKit updates; adding an append-mode or disabling streaming would reintroduce the risky behavior or leave users on a workaround.
