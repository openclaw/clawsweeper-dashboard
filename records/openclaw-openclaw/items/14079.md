## Repair Work Prompt

Fix Linux desktop notification support for the core node-host `system.notify` surface. Current main allowlists `system.notify` for Linux but the headless node host does not advertise or handle it, and docs/CLI still describe notify as macOS-only. Add a focused Linux node-host implementation with desktop-session/backend detection, clear unavailable behavior for headless or missing backend cases, and safe argument handling for title/body values; do not shell-interpolate notification payloads. Keep macOS, Android, iOS, Windows, `system.run`, exec approval, and plugin node-host behavior unchanged. Update CLI/help, docs, tests, and changelog for the new Linux support. Keep fallback scope narrow unless maintainers explicitly choose broader D-Bus or `dunstify` support.
