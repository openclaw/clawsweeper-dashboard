## Summary

Keep open. Current main and the latest shipped release still do not deliver Linux desktop notifications for the reserved `system.notify` node command: Linux policy can allow it, but the headless node host neither advertises nor handles it, and docs/CLI still scope notification delivery to macOS/node-app behavior.

## What This Changes

Add Linux desktop notification delivery for OpenClaw node hosts, with desktop-session detection and graceful degradation when no backend is available.

## Best Possible Solution

Add a focused Linux node-host `system.notify` implementation with desktop-session detection, safe argv-only backend invocation, structured unavailable handling, docs/CLI/changelog updates, and focused regression coverage while preserving existing macOS, iOS, Android, Windows, exec, and plugin behavior.

## Reproduction Assessment

Yes. Source inspection on current main shows a Linux node host identifies as Linux while omitting `system.notify` from declared commands, and the dispatcher has no notify handler even if invocation reached it.

## Solution Assessment

Yes. A narrow core node-host change is the best direction because `system.notify` is a reserved core command; broader D-Bus or `dunstify` support can remain follow-up scope if maintainers want it.
