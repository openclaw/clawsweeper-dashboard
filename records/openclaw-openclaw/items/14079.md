## Summary

Keep open. Current main and v2026.6.5 still lack Linux delivery for the existing core `system.notify` node surface: Linux is allowlisted, but the headless node host does not advertise or dispatch the command, and the docs/CLI still scope notification delivery to macOS node behavior.

## What This Changes

The issue requests Linux desktop notification support for OpenClaw node notifications, with graceful degradation when no desktop notification backend is available.

## Best Possible Solution

Land a focused core Linux headless node-host `system.notify` implementation with desktop-session detection, safe backend invocation, structured unavailable errors, and docs/CLI/test updates while preserving existing macOS, iOS, Android, Windows, exec, and plugin behavior.

## Reproduction Assessment

Yes, source-level rather than live. Current main shows Linux node hosts resolve as Linux but do not declare or handle `system.notify`, and the Gateway rejects undeclared commands before forwarding.

## Solution Assessment

Yes. A narrow core node-host implementation is the maintainable path because `system.notify` is a reserved core node command and an external plugin cannot currently register the exact missing surface.
