## Summary

Keep open. Current main and the latest shipped release still lack Linux desktop delivery for the reserved `system.notify` node command: the headless node host neither advertises nor handles it, while docs and CLI still scope local notification delivery to macOS.

## What This Changes

Request Linux desktop notification delivery for the existing `system.notify` / `openclaw nodes notify` surface, preferably via `notify-send` with graceful headless degradation.

## Best Possible Solution

Add a focused Linux headless node-host `system.notify` implementation with desktop-session/backend detection, structured unavailable handling, docs/CLI updates, and regression coverage while preserving existing macOS, Android, iOS, Windows, exec, and plugin behavior.

## Reproduction Assessment

Yes. Source inspection on current main shows a Linux node host resolves as Linux but does not declare `system.notify`, and the node-host dispatcher has no notify handler even if invoked.

## Solution Assessment

Yes. A narrow core node-host change is the best direction because `system.notify` is a reserved core command; an external plugin cannot fill the exact command surface.
