## Summary

Keep open. Current main and v2026.6.8 still lack Linux desktop notification delivery for the reserved `system.notify` node surface, and the remaining backend/dependency scope needs maintainer direction before implementation.

## What This Changes

Add Linux desktop notification support for OpenClaw node notifications, likely through `notify-send`/libnotify or D-Bus with graceful headless behavior.

## Best Possible Solution

Add a focused core Linux headless node-host `system.notify` implementation with maintainer-chosen backend scope, desktop-session detection, safe invocation, clear unavailable results, docs/CLI updates, and targeted regression coverage.

## Reproduction Assessment

Yes, by source inspection: a Linux headless node host does not advertise or handle `system.notify`, and Gateway dispatch requires the node to declare a command before invocation. I did not run a live Linux desktop notification scenario.

## Solution Assessment

Unclear until maintainers choose backend scope, but the owner boundary should be core node-host code because `system.notify` is a reserved core command and plugins cannot register the exact missing surface.
