## Summary

Keep open. Current main still lacks Linux desktop notification delivery for the reserved `system.notify` node surface: Linux is allowlisted only when the node declares the command, but the headless node host neither advertises nor dispatches it.

## What This Changes

Request Linux desktop notification support for OpenClaw node hosts, primarily via `notify-send`/libnotify with graceful headless degradation.

## Best Possible Solution

Add focused core Linux headless node-host `system.notify` support with desktop-session/backend detection, safe helper invocation, structured unavailable results, docs/CLI/changelog updates, and regression tests while preserving existing macOS, iOS, Android, and Windows behavior.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a Linux headless node host never declares or dispatches `system.notify`, and the Gateway rejects commands missing from the node's declared command list.

## Solution Assessment

Yes. A narrow core node-host implementation is the best direction because `system.notify` is core-reserved and cannot be provided by an external plugin under the current registry contract.
