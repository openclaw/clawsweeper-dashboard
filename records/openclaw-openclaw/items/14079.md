## Summary

Keep open. Current main and the latest shipped release still do not deliver Linux desktop notifications for the reserved core `system.notify` node command; the headless node host does not advertise or handle it, and the related diagnostic issue only improved the rejection error.

## What This Changes

Linux desktop users request native desktop notification support from OpenClaw node hosts, preferably via `notify-send` or another Linux notification backend with graceful headless degradation.

## Best Possible Solution

Add a focused Linux headless node-host `system.notify` implementation with desktop-session/backend detection, structured unavailable handling, docs/CLI updates, and regression coverage while preserving macOS, Android, iOS, Windows, exec, and plugin behavior.

## Reproduction Assessment

Yes. Source inspection on current main shows a Linux node host resolves as Linux but does not declare `system.notify`, and the gateway requires declared commands before forwarding; the dispatcher also has no notify handler if invoked directly.

## Solution Assessment

Yes. A narrow core node-host change is the best direction because `system.notify` is a reserved core command and a plugin cannot register the exact missing surface.
