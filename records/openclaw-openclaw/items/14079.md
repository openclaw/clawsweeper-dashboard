## Label Justifications

## Summary

Current main and v2026.6.5 still lack Linux delivery for the existing core `system.notify` surface: Linux policy permits it, but the headless node host neither advertises nor dispatches it, and no open or merged PR owns this focused gap.

## What This Changes

Add Linux desktop notification delivery for the existing `system.notify` node command.

## Best Possible Solution

Implement a Linux-only core headless node-host handler using a narrow `notify-send` backend, advertise it only when the platform/session can support it, return structured unavailable errors otherwise, and preserve all existing app-node behavior.

## Reproduction Assessment

Yes, from source at high confidence: Linux policy includes `system.notify`, while the headless node host command declaration and dispatcher omit it. This read-only review did not run a live Linux desktop notification.

## Solution Assessment

Yes in direction: extending the existing core-reserved command at the headless node-host boundary is narrower than adding a parallel plugin command. A `notify-send`-first implementation is preferable to multiple fallback mechanisms in the initial patch.
