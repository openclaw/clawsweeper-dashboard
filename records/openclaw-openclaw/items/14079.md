## Summary

Keep open. Current main still lacks Linux desktop delivery for the reserved `system.notify` node surface: Linux policy allowlists the command, but the headless node-host neither advertises nor handles it, and the CLI/docs still describe notify as macOS/node-app-only.

## What This Changes

The issue requests Linux desktop notification support for OpenClaw nodes, primarily via `notify-send` or another Linux desktop notification backend.

## Best Possible Solution

Add a focused Linux node-host `system.notify` implementation with desktop-session/backend detection, structured unavailable handling, CLI/docs/changelog updates, and regression coverage while leaving macOS, iOS, Android, Windows, and exec behavior unchanged.

## Reproduction Assessment

Yes. Source inspection on current main shows a Linux node-host can identify as Linux while omitting `system.notify` from its declared commands, and the dispatcher has no notify handler even if invocation reached it.

## Solution Assessment

Yes. A narrow core node-host change is the best direction because `system.notify` is a reserved core node command; broader D-Bus or `dunstify` support can be a follow-up if maintainers want more backend coverage.
