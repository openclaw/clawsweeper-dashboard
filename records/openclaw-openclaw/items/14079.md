## What This Changes

Add Linux desktop notification support for the existing core `system.notify` node command.

## Best Possible Solution

Add focused Linux headless node-host `system.notify` support with desktop-session/backend detection, structured unavailable handling, docs/CLI/changelog updates, and regression coverage while preserving existing macOS, Android, iOS, Windows, exec, and plugin behavior.

## Reproduction Assessment

Yes, source-level rather than live. Current main shows Linux node hosts resolve as Linux but do not declare or handle `system.notify`, and the gateway rejects undeclared commands before forwarding.

## Solution Assessment

Yes. A narrow core node-host implementation is the maintainable direction because `system.notify` is reserved by core and a plugin cannot fill the exact missing command surface.

Implement Linux desktop notification support for the existing core `system.notify` node command. Keep the change in core node-host paths because `system.notify` is reserved and plugins cannot register that exact command. Add a focused Linux headless node-host notify path with desktop-session/backend detection, argv-only spawning for title/body fields, structured unavailable handling when no desktop session or backend is available, and no changes to macOS/iOS/Android/Windows notify behavior.
