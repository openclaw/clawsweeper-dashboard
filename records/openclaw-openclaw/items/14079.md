## Summary

Keep open. Current main and the latest release still do not implement Linux desktop notification delivery for the existing node `system.notify` surface; the source gap is clear, but backend scope and command-advertisement semantics still need maintainer direction.

## What This Changes

Request to add Linux desktop notification support for `openclaw nodes notify` / `system.notify`, likely via `notify-send` or another Linux notification backend.

## Best Possible Solution

Implement one explicitly supported Linux notification backend in the core headless node host with desktop/backend detection, structured unavailable results, focused tests, and updated CLI/docs after maintainer backend-scope approval.

## Reproduction Assessment

Yes. Source inspection shows the Linux-capable headless node host neither advertises nor dispatches `system.notify`, so a Linux `nodes notify` invocation cannot deliver a desktop notification on current main.

## Solution Assessment

No for the full suggested fallback stack as written. The narrowest maintainable solution is one supported Linux backend with explicit capability detection and failure semantics unless maintainers choose broader fallback behavior.
