## Summary

Keep open. Current main and the latest shipped release still do not provide Linux desktop delivery for `system.notify`: the headless node host omits the command, the dispatcher has no notify path, and docs/CLI still scope notification delivery to macOS while the related closed report only improved the error message.

## What This Changes

Request Linux desktop notification support for `openclaw nodes notify` / `system.notify`, with desktop-session detection and graceful headless degradation.

## Best Possible Solution

Add a focused Linux node-host `system.notify` implementation with desktop-session/backend detection, structured unavailable handling, docs/CLI updates, and regression coverage while preserving existing macOS, Android, iOS, Windows, exec, and plugin behavior.

## Reproduction Assessment

Yes. Source inspection on current main shows a Linux node host identifies as Linux, but its advertised command list omits `system.notify`; even if invoked, the node-host dispatcher has no notify handler.

## Solution Assessment

Yes. A narrow core node-host change is the best direction because `system.notify` is a reserved core command; optional broader backend support can remain follow-up scope.
