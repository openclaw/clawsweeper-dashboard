## Summary

Keep open. Current main still leaves Linux node hosts unable to deliver desktop notifications: the gateway policy includes `system.notify`, but the headless node host neither advertises nor handles it, and the exact command is core-reserved.

## What This Changes

Add Linux desktop notification support for the existing `openclaw nodes notify` / `system.notify` node surface.

## Best Possible Solution

Implement a focused core Linux headless node-host `system.notify` backend with desktop-session detection, safe argument passing, updated CLI/docs/changelog, and regression coverage for command advertisement plus invocation behavior.

## Reproduction Assessment

Yes. Source inspection on current `origin/main` shows Linux node hosts do not advertise or dispatch `system.notify`, while gateway `node.invoke` requires the node to declare the command before forwarding.

## Solution Assessment

Yes. Because `system.notify` is core-reserved and already participates in the gateway node command policy, the narrow maintainable fix belongs in the core node-host path rather than an external plugin claiming the command.
