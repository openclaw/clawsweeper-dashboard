- `impact:other`: Linux desktop users cannot receive local OpenClaw notifications, a concrete user-visible impact not covered by the more specific impact labels.

## Summary

Current main and v2026.6.5 still lack Linux delivery for the established `system.notify` surface: Linux gateway policy permits the command, but the headless node host neither advertises nor dispatches it, and no canonical implementation PR or replacement issue was found.

## What This Changes

Add Linux desktop notification delivery to the existing node `system.notify` capability.

## Best Possible Solution

Approve one minimal Linux backend in the core headless node host, preserve the existing notification schema, advertise the capability under a clear availability rule, and return structured unavailable errors without adding a broad fallback stack.

## Reproduction Assessment

Yes at source level: both current main and v2026.6.5 permit `system.notify` in Linux policy but omit it from the headless host's declared and handled commands, preventing Linux delivery.

## Solution Assessment

A core node-host implementation is the best ownership boundary because `system.notify` is already a reserved cross-platform command; the narrowest maintainable approach is one approved Linux backend rather than several speculative fallbacks.
