## Summary

Keep open. Current main still lacks Linux desktop notification delivery for the reserved `system.notify` node command: Linux can be allowlisted at the gateway layer, but the headless node host neither advertises nor handles the command, and the docs/CLI still scope notify to macOS/node-app behavior.

## What This Changes

Request Linux desktop notification support for OpenClaw nodes, primarily via `notify-send` with graceful degradation on headless Linux sessions.

## Best Possible Solution

Add a focused Linux node-host `system.notify` implementation with desktop-session detection, safe argv-only notification backend invocation, structured unavailable handling, docs/CLI/changelog updates, and focused regression coverage while preserving existing macOS, iOS, Android, Windows, exec, and plugin behavior.

## Reproduction Assessment

Yes. Source inspection on current main shows a Linux node host can identify as Linux while omitting `system.notify` from declared commands, and the dispatcher has no notify handler even if invocation reached it.

## Solution Assessment

Yes. A narrow core node-host change is the best direction because `system.notify` is a reserved core command; broader D-Bus or `dunstify` support can remain follow-up scope if maintainers want it.
