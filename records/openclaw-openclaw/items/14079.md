## Summary

Keep open. Current main and the latest release still lack Linux desktop notification delivery for the core `system.notify` node surface: Linux policy allowlists it, but the headless node host neither advertises nor dispatches it, and the related closed report only improved the rejection/error experience.

## What This Changes

Linux desktop users request `system.notify` support through `notify-send`, libnotify, D-Bus, or another graceful Linux notification backend.

## Best Possible Solution

Add a focused Linux headless node-host `system.notify` backend with safe argv-based delivery, desktop-session/backend detection, structured unavailable behavior, docs/CLI/changelog updates, and regression coverage while leaving macOS, iOS, Android, and Windows behavior unchanged.

## Reproduction Assessment

Yes. Source inspection on current main is high-confidence: Linux allowlists `system.notify`, but the headless node host does not declare or dispatch it, so `node.invoke` rejects the command before any Linux notification backend can run.

## Solution Assessment

Yes. Because `system.notify` is a reserved core node-host command and already has a CLI surface, the maintainable solution belongs in the core node-host path rather than an external plugin claiming the same command.

## Repair Work Prompt

Implement Linux desktop notification support for the existing core `system.notify` node command on the headless node host. Keep the scope focused: advertise `system.notify` only when the Linux backend can be attempted safely, dispatch it in the node-host path, use safe argv-based invocation for `notify-send` or the chosen backend, gracefully return structured unavailable behavior on headless/no-backend systems, and do not change macOS, iOS, Android, or Windows notification behavior. Read the upstream `notify-send`/libnotify contract before relying on flags, exit codes, or environment requirements. Update CLI copy, docs, changelog, and regression tests. Do not make this a plugin workaround because `system.notify` is a reserved core node-host command.
