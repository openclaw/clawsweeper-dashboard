## Summary

Keep open. Current main and the latest shipped release still lack Linux desktop notification delivery for the core `system.notify` node surface: Linux policy allowlists the command, but the headless node host neither advertises nor dispatches it, docs/CLI still scope notification delivery to macOS, and plugins cannot claim the reserved command.

## What This Changes

Request Linux desktop notification support for OpenClaw nodes, preferably via `notify-send`/libnotify or equivalent with graceful headless degradation.

## Best Possible Solution

Add a focused Linux headless node-host `system.notify` backend with safe argv-based delivery, desktop-session/backend detection, structured unavailable failures, docs/CLI/changelog updates, and regression coverage while preserving macOS, iOS, and Android behavior.

## Reproduction Assessment

Yes, as a source-level reproduction. A Linux node host declares only run/which/exec-approval/plugin commands, and Gateway requires `system.notify` to be declared before forwarding, so the request is rejected before delivery can occur.

## Solution Assessment

Yes. Because `system.notify` is a reserved core node-host command and already has a CLI surface, the maintainable fix belongs in the core node-host path rather than an external plugin claiming the same command.
