## Summary

Keep open. Current main and the latest release still do not implement Linux desktop notification delivery for the reserved `system.notify` node surface; the related closed item only improved the rejection/error experience.

## What This Changes

The issue requests Linux desktop notification support for OpenClaw node notifications, likely through `notify-send` or a Linux notification backend.

## Best Possible Solution

Add a focused core Linux headless node-host `system.notify` implementation with desktop-session detection, safe process invocation, clear unavailable results, docs/CLI updates, and regression coverage while leaving macOS/iOS/Android behavior unchanged.

## Reproduction Assessment

Yes, by source inspection: a Linux headless node host does not advertise or handle `system.notify`, and Gateway dispatch requires the node to declare a command before invocation. I did not run a live Linux desktop notification scenario, so backend-specific behavior remains unproven.

## Solution Assessment

Unclear until maintainers choose the Linux backend scope, but the implementation boundary should be core node-host code because `system.notify` is a reserved core node command and plugins cannot register the exact missing command.
