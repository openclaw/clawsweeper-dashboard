## Summary

Current main still has no Linux desktop-notification implementation: the gateway permits `system.notify`, but the cross-platform headless node neither advertises nor handles it, and the reserved command cannot be supplied by a plugin. The request remains useful, but backend scope and dependency expectations need explicit maintainer product judgment.

## What This Changes

Requests native Linux desktop notification delivery for the core `system.notify` node command.

## Best Possible Solution

Implement one explicitly supported, capability-detected Linux backend in the core headless node host, return structured unavailable results when no desktop session or backend exists, and update focused tests, CLI wording, and node documentation without changing other platforms.

## Reproduction Assessment

Yes, from source with high confidence: on current main the Linux-capable headless node neither advertises nor dispatches `system.notify`, so invocation cannot deliver a desktop notification.

## Solution Assessment

No, the suggested cascading `notify-send`/D-Bus/`dunstify` approach is broader than necessary; the narrowest maintainable first solution is one supported backend with explicit capability detection and failure semantics.
