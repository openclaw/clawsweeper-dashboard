## Summary

Keep open. Current main and the latest release still do not implement Linux desktop notification delivery for the existing node `system.notify` surface: Linux policy allowlists the command, but the headless node host neither advertises nor dispatches it, and the plugin registry reserves the command so a plugin cannot fill the exact gap.

## What This Changes

Request Linux desktop notification support for OpenClaw nodes, likely using `notify-send` or a Linux notification backend with graceful headless degradation.

## Best Possible Solution

Implement one explicitly supported Linux notification backend in the core headless node host with desktop/backend detection, structured unavailable results, focused tests, and updated CLI/docs after maintainer backend-scope approval.

## Reproduction Assessment

Yes, from source with high confidence: on current main the Linux-capable headless node neither advertises nor dispatches `system.notify`, so invocation cannot deliver a desktop notification.

## Solution Assessment

No for the full suggested fallback stack as written; the narrowest maintainable first solution is one supported Linux backend with explicit capability detection and failure semantics unless maintainers choose broader fallback behavior.
