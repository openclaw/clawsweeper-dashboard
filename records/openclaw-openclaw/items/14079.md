## Summary

Keep open. Current main still lacks Linux desktop delivery for the reserved core `system.notify` path: Linux is allowlisted in gateway policy, but the headless node host does not advertise or handle the command, and docs/CLI still describe notify as macOS-only.

## What This Changes

The issue requests Linux desktop notification support for `openclaw nodes notify` and the core `system.notify` node command, with graceful degradation when no desktop session or backend is available.

## Best Possible Solution

Add a focused Linux headless node-host `system.notify` implementation with desktop-session/backend detection, structured unavailable handling, CLI/docs/changelog updates, and regression coverage while preserving existing macOS, Android, iOS, Windows, exec, and plugin behavior.

## Reproduction Assessment

Yes. Source inspection on current main shows Linux node hosts resolve as Linux but do not declare `system.notify`, and the gateway requires declared commands before forwarding; direct dispatch also has no notify handler.

## Solution Assessment

Yes. A narrow core node-host change is the right direction because `system.notify` is a reserved core command and a plugin cannot register the exact missing surface.
