## Summary

Keep open. Current main still lacks Linux desktop delivery for the reserved core `system.notify` node surface: Linux is allowlisted, but the headless node host neither advertises nor handles the command, docs and CLI remain macOS/headless-only, and the related closed item covered diagnostics rather than delivery support.

## What This Changes

The issue requests Linux desktop notification support for OpenClaw node notifications, primarily via notify-send/libnotify with graceful headless degradation.

## Best Possible Solution

Add a focused Linux headless node-host `system.notify` implementation with desktop-session/backend detection, structured unavailable handling, CLI/docs/changelog updates, and regression coverage while preserving existing macOS, Android, iOS, Windows, exec, and plugin behavior.

## Reproduction Assessment

Yes, source-level rather than live. Current main shows Linux node hosts resolve as Linux but do not declare `system.notify`, the gateway requires declared commands before forwarding, and direct headless dispatch has no notify handler.

## Solution Assessment

Yes. A narrow core node-host change is the right direction because `system.notify` is reserved as a core node-host command and a plugin cannot register the exact missing surface.
