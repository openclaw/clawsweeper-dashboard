## Summary

Keep open. Current main still does not deliver Linux desktop notifications through the existing `system.notify` node surface: Linux policy allows the command, but the headless node host neither advertises nor handles it, and docs/CLI still describe notify as macOS-oriented.

## What This Changes

The issue requests Linux desktop notification delivery, such as `notify-send`, for OpenClaw node notifications.

## Best Possible Solution

Add a focused core Linux headless node-host `system.notify` implementation with desktop-session/backend detection, safe argv-based execution, docs/CLI updates, changelog, and regression coverage while leaving macOS, iOS, and Android behavior unchanged.

## Reproduction Assessment

Yes, from source inspection. A Linux headless node host does not advertise or handle `system.notify`, and the Gateway requires the live node to declare the command before forwarding it.

## Solution Assessment

Yes, a narrow core implementation is the best path because `system.notify` is a reserved core command and the current plugin surface cannot fill the exact missing command. Keep fallback scope small unless maintainers explicitly choose broader D-Bus or `dunstify` support.

## Repair Work Prompt

Implement Linux desktop notification support for the existing core `system.notify` node surface from https://github.com/openclaw/openclaw/issues/14079. Keep the change focused: advertise `system.notify` from Linux headless node hosts only when appropriate, add a node-host handler that detects a desktop session (`DISPLAY` or `WAYLAND_DISPLAY`) and invokes `notify-send` safely with argv arguments, return structured unavailable/permission-style failures when notification delivery is not possible, and do not add broad D-Bus or `dunstify` fallback unless maintainers explicitly ask. Update CLI/docs so `openclaw nodes notify` is no longer mac-only where Linux is supported, add targeted tests for advertisement, invocation, sanitization, and gateway policy behavior, and add a CHANGELOG entry because this is user-visible. Do not change macOS, iOS, Android, Windows, exec approval, or plugin command ownership behavior.
