## Summary

Keep open. Current main and v2026.6.10 still lack Linux desktop delivery for the reserved `system.notify` node command: Gateway policy can allow it, but the headless node host neither advertises nor handles it, and backend/dependency scope needs maintainer direction.

## What This Changes

Linux desktop users request native OpenClaw desktop notification delivery through the existing node `system.notify` surface.

## Best Possible Solution

Make an explicit maintainer decision on the Linux backend contract, then implement a focused core node-host `system.notify` path with safe invocation, clear unavailable results, docs/CLI updates, and targeted regression tests.

## Reproduction Assessment

Yes. Source inspection shows Linux can pass the Gateway allowlist, but the headless node host does not advertise or handle `system.notify`; I did not run a live Linux desktop notification scenario.

## Solution Assessment

Unclear until maintainers choose backend scope. The owner boundary should be core node-host code because `system.notify` is a reserved core command and plugins cannot register the exact missing surface.
