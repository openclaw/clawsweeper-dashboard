## Summary

Keep open. Current main still does not provide Linux desktop notification delivery for the core `system.notify` node surface: the Linux allowlist includes the command, but the headless node host does not advertise or dispatch it, and docs/CLI still scope notify behavior to macOS.

## What This Changes

Request Linux node-host desktop notifications for `openclaw nodes notify`/`system.notify`, preferably through `notify-send` with graceful desktop-session detection.

## Best Possible Solution

Add a focused Linux headless node-host `system.notify` backend with safe argv-based delivery, desktop-session/backend detection, docs/CLI/changelog updates, and regression coverage for advertisement plus invocation behavior.

## Reproduction Assessment

Yes. Source inspection on current main shows Linux policy allowlists `system.notify`, but the headless node host does not declare or dispatch it, so `node.invoke` rejects the command before any Linux notification backend can run.

## Solution Assessment

Yes. Because `system.notify` is a reserved core node-host command, the narrow maintainable fix belongs in core node-host handling rather than an external plugin claiming the command.
