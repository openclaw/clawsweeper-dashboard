Add Linux desktop notification support for the existing `openclaw nodes notify` / `system.notify` node command.

## Best Possible Solution

Add a focused Linux headless node-host `system.notify` implementation with desktop-session/backend detection, structured unavailable handling, docs/CLI updates, and regression coverage while preserving macOS, Android, iOS, Windows, exec, and plugin behavior.

Yes. Source inspection on current main shows a Linux node host resolves as Linux but does not declare `system.notify`, and the node-host dispatcher has no notify handler even if invoked.

## Solution Assessment

Yes. A narrow core node-host change is the best direction because `system.notify` is a reserved core command and a plugin cannot register the exact missing surface.
