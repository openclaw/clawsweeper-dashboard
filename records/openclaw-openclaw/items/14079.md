## Summary

Keep open. Current main still has the Linux notification gap: Linux is allowlisted for `system.notify`, but the headless node host neither advertises nor handles the command, and the current CLI/docs still describe notification delivery as macOS node behavior.

## What This Changes

The issue requests Linux desktop notification support for OpenClaw's `system.notify` node command surface.

## Best Possible Solution

Add focused Linux headless node-host `system.notify` support with desktop-session/backend detection, structured unavailable handling, docs/CLI/changelog updates, and regression coverage while preserving existing macOS, Android, iOS, Windows, exec, and plugin behavior.

## Reproduction Assessment

Yes, source-level rather than live. Current main shows Linux node hosts resolve as Linux but do not declare or handle `system.notify`, and the Gateway rejects undeclared commands before forwarding.

## Solution Assessment

Yes. A narrow core node-host implementation is the maintainable direction because `system.notify` is reserved by core and a plugin cannot register the exact missing command surface.

## Repair Work Prompt

Implement Linux desktop notification support for the existing core `system.notify` node command. Add a focused headless node-host notify handler, advertise the command only where supported, detect desktop-session availability (`DISPLAY` or `WAYLAND_DISPLAY`), prefer a small safe backend such as `notify-send`, return structured unavailable errors when the session/backend is missing, and pass title/body/sound/priority/delivery values as argv fields only with no shell interpolation. Do not add broad D-Bus or `dunstify` fallback stacks, package installation behavior, or plugin API changes unless the implementation proves a narrow need. Update CLI copy, docs, and CHANGELOG for the user-visible Linux support. Preserve existing macOS app, iOS, Android, Windows, `system.run`, `system.which`, exec approval, and plugin node-host behavior. Related context: https://github.com/openclaw/openclaw/issues/14079 and the already-closed diagnostics-only issue https://github.com/openclaw/openclaw/issues/24616.
