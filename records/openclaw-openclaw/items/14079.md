## Summary

Keep open. Current main still lacks Linux desktop notification delivery: there is no `notify-send`/libnotify backend, the headless node host neither advertises nor handles `system.notify`, docs and CLI still scope notify to macOS, and the related closed issue only addressed the error experience.

## What This Changes

Add Linux desktop notification support for the existing node `system.notify` surface, likely via `notify-send`/libnotify with graceful headless degradation.

## Best Possible Solution

Add a focused Linux headless node-host `system.notify` backend with safe argv-based delivery, desktop-session/backend detection, docs/CLI/changelog updates, and regression coverage for advertisement and invocation behavior.

## Reproduction Assessment

Yes. Source inspection on current main shows Linux platform policy allowlists `system.notify`, but the headless node host does not advertise or dispatch it, and `node.invoke` rejects commands not declared by the node.

## Solution Assessment

Yes. Because `system.notify` is a reserved core node-host command, the narrow maintainable solution belongs in the core node-host path rather than an external plugin claiming the command.
