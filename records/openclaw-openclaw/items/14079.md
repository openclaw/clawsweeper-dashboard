## Summary

Keep open. Current main still lacks Linux desktop notification delivery for the reserved `system.notify` node surface: Linux is allowlisted only if the node declares the command, but the headless node host neither advertises nor dispatches it, and docs/CLI still describe notify as macOS node behavior.

## What This Changes

Add Linux desktop notification delivery for OpenClaw node `system.notify`, likely through a safe `notify-send`/desktop-session path with graceful headless fallback.

## Best Possible Solution

Add focused core Linux headless node-host `system.notify` support with desktop-session/backend detection, safe helper invocation, structured unavailable results, docs/CLI/changelog updates, and regression tests while preserving existing macOS, iOS, Android, and Windows behavior.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a Linux/headless node host never declares or dispatches `system.notify`, and the Gateway rejects commands that the node does not declare.

## Solution Assessment

Yes. A narrow core node-host implementation is the best direction because `system.notify` is core-reserved and cannot be filled by an external plugin under the current command registry.

## Repair Work Prompt

Fix Linux desktop notifications for the core node `system.notify` surface tracked by https://github.com/openclaw/openclaw/issues/14079. Current main allowlists `system.notify` for Linux but the headless node host does not advertise or dispatch it, and plugins cannot register the reserved command. Add a narrow Linux headless node-host implementation that detects a desktop session, invokes `notify-send` safely with title/body as argv fields only, returns a structured unavailable result when headless or the helper is absent, and leaves macOS, iOS, Android, and Windows behavior unchanged. Do not add dependency patches or broaden into D-Bus/`dunstify` fallback unless existing repo patterns make that necessary. Update `openclaw nodes notify` copy, nodes/CLI docs, changelog, and focused regression coverage. Preserve the improved rejection-message behavior from related #24616.
