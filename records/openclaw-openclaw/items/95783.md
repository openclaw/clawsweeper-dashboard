## Summary

Keep this PR open: it is a plausible narrow fix for a real older-browser UI/a11y gap, but the focus-ring cascade does not preserve the intended modern-browser enhancement and the external contributor proof is only a text claim.

## What This Changes

The PR adds CSS fallbacks for Control UI focus rings, the mobile nav drawer shadow/backdrop, and the collapsed sidebar active accent when `color-mix()` is unsupported.

## Best Possible Solution

Land a narrow CSS-only fallback fix that keeps each older-browser fallback immediately paired with a winning modern `color-mix()` enhancement and preserves the existing per-theme blend percentages.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: current main puts `color-mix()` inside the focus-ring and mobile drawer declarations that must render in older browsers. I did not run Firefox 112 locally in this read-only review.

## Solution Assessment

No, not yet: the fallback direction is right, but the `@supports` focus-ring enhancement needs to win after the theme fallbacks and proof needs an actual after-fix artifact from a no-`color-mix` browser.
