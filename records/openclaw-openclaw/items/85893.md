Full review comments:

- **[P2] Clear handlers when cancelling pending Twitch clients:** `extensions/twitch/src/twitch-client.ts:319-323`
  - body: When `disconnect()` hits a client that is still pending, this branch only quits the pending client and clears connection state. Because `onMessage()` registrations are keyed separately and the connected-client path clears them on disconnect, this leaves an old callback alive and a later reconnect can deliver messages to a stale handler.
  - confidence: 0.9
- **[P2] Scope fish value-option skipping to the active command:** `src/cli/completion-cli.ts:88-90`
  - body: `value_options` is collected from the whole command tree, so a value-taking flag on one command makes the helper skip the following token for another command where that flag is boolean. Current commands already reuse `--verbose` this way, so paths like `openclaw sessions --verbose cleanup --<TAB>` can lose the `cleanup` token and miss nested completions.
  - confidence: 0.88
