## Summary

Keep open: current main already covers the logging problem, but the local embedding VRAM fallback is still missing from the active package implementation and this PR has blocking context-size and cleanup defects plus only test-level proof.

## What This Changes

The PR changes logging config/file resolution and local memory embedding initialization/tests to add config-path reads, date expansion, GPU retry, and CPU fallback.

## Best Possible Solution

Port the embedding resilience to packages/memory-host-sdk, preserve local.contextSize in both GPU-fit and createEmbeddingContext calls, serialize VRAM recovery, and dispose temporary CPU resources on every failure path while keeping current main's logging implementation.

## Reproduction Assessment

Yes for source-level review: current main rethrows local llama initialization failures, and the PR tests model the available-VRAM retry/fallback path. I did not run a live GPU reproduction in this read-only review.

## Solution Assessment

No, not as submitted. The maintainable fix is a current-main package patch that preserves the context-size contract and closes the CPU fallback lifecycle gaps.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve the configured embedding context size:** `src/memory-host-sdk/host/embeddings.ts:192`
  - body: The PR creates the GPU embedding context without passing the current local.contextSize value. Current main documents and tests that this value defaults to 4096 and is passed to node-llama-cpp, so dropping it can make constrained local setups use the runtime/model default and reintroduce memory pressure.
  - confidence: 0.88
- **[P2] Dispose CPU fallback resources if context creation fails:** `src/memory-host-sdk/host/embeddings.ts:212`
  - body: If cpuModel.createEmbeddingContext() rejects, this helper throws before returning the disposer, so the temporary CPU model and llama runtime are left alive. Wrap context creation so cpuModel and cpuLlama are disposed on that failure path too.
  - confidence: 0.86
