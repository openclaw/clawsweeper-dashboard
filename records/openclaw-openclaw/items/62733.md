## Summary

Keep open: current main already covers the logging side, but the active local embedding implementation still rethrows VRAM-fit initialization failures. The PR is useful source work, but it is stale against the current package layout and the submitted embedding patch has correctness issues before it can land.

## What This Changes

This PR changes logging config/file resolution and local memory embedding initialization by adding direct logging reads, `YYYY-MM-DD` log path handling, GPU-fit retry, temporary CPU fallback, and targeted tests.

## Best Possible Solution

Keep the logging behavior already on main and port only the missing embedding resilience into `packages/memory-host-sdk`, preserving `local.contextSize`, serializing VRAM recovery, and closing all temporary resource cleanup paths.

## Reproduction Assessment

Yes for source-level review: current main rethrows local llama initialization errors from `ensureContext()`, and the PR's mocked `available VRAM` tests demonstrate the intended retry/fallback path. A live GPU reproduction was not performed in this read-only sweep.

## Solution Assessment

No, not as submitted. The maintainable fix is a current-main rework against the package source that keeps the existing context-size contract and disposes GPU/CPU resources on every success and failure path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve local context size for GPU embeddings:** `src/memory-host-sdk/host/embeddings.ts:192`
  - body: The PR stops passing `local.contextSize` into `createEmbeddingContext()`. Current main documents and tests the default of 4096 plus user overrides, so dropping this can make constrained local setups use the runtime/model default and reintroduce memory pressure.
  - confidence: 0.88
- **[P2] Dispose CPU fallback resources on init failure:** `src/memory-host-sdk/host/embeddings.ts:212`
  - body: If `cpuModel.createEmbeddingContext()` rejects, this helper throws before returning the cleanup callback, leaving the temporary CPU model/runtime undisposed. Wrap CPU context creation so the model and llama instance are disposed on that failure path too.
  - confidence: 0.86
