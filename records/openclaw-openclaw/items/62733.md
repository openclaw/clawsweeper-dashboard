## Summary

Keep open: the logging part is already covered on current main, but the embedding resilience still has useful work and the submitted branch needs to be ported to the active package path, fix lifecycle/context-size regressions, and add real runtime proof before merge.

## What This Changes

The PR updates logging config/file resolution and the old local memory embedding implementation/tests to add date-expanded log paths, GPU retry, CPU fallback, and retryable model-path resolution.

## Best Possible Solution

Port the embedding resilience to packages/memory-host-sdk while preserving local.contextSize, serialized VRAM recovery, full CPU cleanup, and focused tests; keep current main's logging implementation.

## Reproduction Assessment

Yes for source-level review: current main's active local embedding provider rethrows initialization errors, and the PR tests model the VRAM failure path. I did not establish a live GPU/node-llama-cpp reproduction in this read-only pass.

## Solution Assessment

No, not as submitted. The maintainable fix is a current-main package patch that preserves the context-size contract and closes CPU fallback lifecycle gaps instead of landing the stale branch wholesale.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve the embedding context size:** `src/memory-host-sdk/host/embeddings.ts:192`
  - body: Current main's local memory contract passes local.contextSize, defaulting to 4096, into createEmbeddingContext. This patch creates the GPU embedding context with no options after hardcoding a 24-token fitContext on model load, so configured/default sizing is dropped and can recreate the VRAM pressure this change is meant to avoid.
  - confidence: 0.9
- **[P2] Clean up CPU resources when context creation fails:** `src/memory-host-sdk/host/embeddings.ts:212`
  - body: If cpuModel.createEmbeddingContext() rejects, this helper throws before returning the disposer, leaving the temporary CPU model and llama runtime alive. Wrap context creation so both resources are released on that failure path.
  - confidence: 0.87
