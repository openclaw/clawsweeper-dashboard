## Summary

Keep open. Current main still writes prior-session transcript-shaped `user:` and `assistant:` turns into durable session-memory files on both `/new` and `/reset`; startup-context framing only protects the first-turn injection path, while the persisted artifact and exact memory reads remain raw.

## What This Changes

The issue asks OpenClaw to prevent bundled session-memory from replaying prior-session transcript content as trusted context after `/reset` or `/new`.

## Best Possible Solution

Define the bundled session-memory trust boundary end to end: `/reset` should break continuity by default, and any `/new` continuity should be stored and retrieved only as clearly untrusted historical reference or a neutral summary.

## Reproduction Assessment

Yes, at source level. Create a session JSONL with ordinary user/assistant messages and trigger the hook with `action: "reset"`; current code and tests show raw transcript-shaped lines written into `memory/YYYY-MM-DD-*.md`, while live autonomous re-execution depends on model and tool environment.

## Solution Assessment

Unclear for the exact final policy. The direction is right, but a token-only sanitizer is not sufficient; the maintainable fix should combine `/reset` semantics with durable untrusted framing or neutral summaries across persistence and retrieval paths.
