## Summary

Keep open. Current main and the latest release still persist transcript-shaped prior turns on `/reset`; the startup-context untrusted wrapper is only a read-path mitigation, and the related sanitizer PR remains open and does not fully address ordinary prior-command replay.

## What This Changes

The issue asks OpenClaw to stop persisted session-memory transcripts from replaying prior user instructions after `/reset` by treating them as untrusted historical context, summarizing them, or not persisting them on reset.

## Best Possible Solution

Define the session-memory trust boundary end to end: `/reset` should break continuity by default, and any `/new` continuity should be persisted and later injected only as untrusted historical reference or a neutral summary, with regression coverage for the write and startup/read paths.

## Reproduction Assessment

Yes, at source level. A JSONL session with ordinary user and assistant messages is formatted as role-prefixed text and written under `## Conversation Summary` when the hook handles `action: "reset"`; live autonomous re-execution still depends on model and tool environment.

## Solution Assessment

Unclear for the final policy. Wrapping or summarizing historical content is directionally right, but the open sanitizer PR is not sufficient by itself because it does not address ordinary prior commands or `/reset` continuity semantics.
