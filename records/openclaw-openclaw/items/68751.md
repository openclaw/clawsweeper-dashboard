## Summary

Keep open. Current main still persists prior-session transcript-shaped `user:`/`assistant:` text directly into session-memory files on both `/new` and `/reset`; startup-context framing protects only one load path, and the durable Markdown artifact remains raw. The related sanitizer PR at https://github.com/openclaw/openclaw/pull/69961 addresses the malformed token/tool-call subcase for https://github.com/openclaw/openclaw/issues/69943, but it does not fully solve ordinary prior-command replay or the `/reset` continuity policy.

## What This Changes

The issue asks OpenClaw to stop raw historical session turns from being persisted and replayed as current context after `/reset` or `/new`.

## Best Possible Solution

Define the bundled session-memory trust boundary end to end: `/reset` should break continuity by default, and `/new` continuity should be stored and retrieved only as clearly untrusted historical reference or a neutral summary, with regression coverage for persistence, startup context, and memory-tool retrieval.

## Reproduction Assessment

Yes, at source level. A session JSONL with ordinary user/assistant messages will be formatted as role-prefixed text and written raw under `## Conversation Summary` when the hook handles `action: "reset"`; live autonomous re-execution still depends on model and tool environment.

## Solution Assessment

Unclear for the exact final policy. Wrapping or summarizing historical content is the right direction, but a token-only sanitizer is not enough; the maintainable fix should combine `/reset` semantics with durable untrusted framing or neutral summaries across persistence and retrieval paths.
