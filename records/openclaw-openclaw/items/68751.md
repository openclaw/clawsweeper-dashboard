## Summary

Keep open. Current main and the latest release still write transcript-shaped prior turns on `/reset`; the later startup-context wrapper is only a read-path mitigation and the open sanitizer PR covers malformed-token poisoning, not ordinary prior-command replay.

## What This Changes

The issue asks OpenClaw to prevent session-memory from replaying raw prior-session turns as executable current context after `/reset` or `/new`.

## Best Possible Solution

Define the session-memory trust boundary end to end: `/reset` should break continuity by default, and any `/new` continuity should be stored and retrieved only as untrusted historical reference or a neutral summary.

## Reproduction Assessment

Yes, at source level. A JSONL session with ordinary user and assistant messages is formatted as role-prefixed text and written raw under `## Conversation Summary` when the hook handles `action: "reset"`; live autonomous re-execution still depends on model and tool environment.

## Solution Assessment

Unclear for the exact final policy. Wrapping or summarizing historical content is the right direction, but the open sanitizer PR is not sufficient by itself because it does not address ordinary prior commands or `/reset` continuity semantics.
