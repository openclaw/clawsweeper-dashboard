## Summary

Keep open. Current main still writes prior-session transcript-shaped `user:` and `assistant:` lines into durable session-memory files on both `/new` and `/reset`; the newer startup-context wrapper protects only one preload path, and the broader raw-instruction replay problem is not solved by the open token-sanitizer PR.

## What This Changes

The issue asks OpenClaw to prevent bundled `session-memory` from replaying raw prior-session turns as current instructions after `/reset` or `/new`.

## Best Possible Solution

Define one end-to-end session-memory trust boundary: `/reset` should break continuity by default, while `/new` continuity should be persisted and retrieved only as explicitly untrusted historical reference or a neutral summary.

## Reproduction Assessment

Yes, at source level. Create a session JSONL with ordinary user and assistant messages, trigger the bundled hook with `action: "reset"`, and current source plus tests show raw role-shaped turns written into `memory/YYYY-MM-DD-*.md`; live autonomous re-execution remains model and tool-environment dependent.

## Solution Assessment

Unclear for the final policy. Sanitizing malformed artifacts is useful but not sufficient; the safer fix needs a maintainer decision on `/reset` semantics plus durable untrusted framing or neutral summaries across persistence and memory retrieval paths.
