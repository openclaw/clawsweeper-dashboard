## Summary

Keep open. Current main and the latest release still persist ordinary prior-session `user:` and `assistant:` transcript lines under `## Conversation Summary` on both `/new` and `/reset`; startup context now frames one load path as untrusted, but the durable memory artifact and memory tool/index paths still need an explicit trust-boundary decision.

## What This Changes

The issue requests that bundled `session-memory` stop replaying raw prior-session turns as executable current-context material, especially after `/reset`.

## Best Possible Solution

Define the session-memory trust boundary end to end: `/reset` should break continuity by default, while `/new` continuity should be stored and retrieved only as explicitly untrusted historical reference or a neutral summary with regression coverage across persistence, startup context, and memory tools.

## Reproduction Assessment

Yes, at source level. A JSONL session with ordinary user and assistant turns is formatted as role-prefixed text and written raw under `## Conversation Summary` when the hook handles `action: "reset"`; live autonomous re-execution still depends on model and tool environment.

## Solution Assessment

Unclear for the final policy. Wrapping or neutral summaries are plausible, and skipping persistence on `/reset` matches the report, but maintainers should choose the durable behavior before a security-sensitive change lands.
