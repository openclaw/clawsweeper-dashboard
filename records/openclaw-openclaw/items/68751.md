## Summary

Keep open. Current main still persists raw role-shaped prior-session turns on `/reset` and `/new`; startup-context framing protects only one injection path, while the persisted Markdown artifact and `memory_get` exact-read path remain raw.

## What This Changes

The issue asks bundled session-memory to stop replaying raw prior-session user/assistant turns after `/reset` by adding durable untrusted framing, neutral summaries, or reset-specific non-persistence.

## Best Possible Solution

Define the bundled session-memory trust boundary end to end: `/reset` should break continuity by default, and `/new` continuity should be stored and retrieved only as clearly untrusted historical reference or a neutral summary, with regression coverage for persistence, startup context, and memory-tool retrieval.

## Reproduction Assessment

Yes, at source level. Create a session JSONL with ordinary user/assistant messages and trigger the hook with `action: "reset"`; current code and tests show raw transcript-shaped lines written into `memory/YYYY-MM-DD-*.md`, while live autonomous re-execution depends on model and tool environment.

## Solution Assessment

Unclear for the exact final policy. The direction is right, but a token-only sanitizer is not sufficient; the maintainable fix should combine `/reset` semantics with durable untrusted framing or neutral summaries across persistence and retrieval paths.
