## Summary

Keep open. Current main still writes transcript-shaped prior user/assistant turns into bundled session-memory files on both `/new` and `/reset`; startup-context framing only protects one injection path, while the persisted Markdown artifact and `memory_get` retrieval path remain raw.

## What This Changes

The issue asks OpenClaw to stop prior session turns saved by the bundled session-memory hook from replaying as current instructions after `/reset`.

## Best Possible Solution

Define the bundled session-memory trust boundary end to end: `/reset` should break continuity by default, and any `/new` continuity should be stored and retrieved only as clearly untrusted historical reference or as a neutral summary, with regression coverage for persistence, startup context, and memory-tool retrieval.

## Reproduction Assessment

Yes. A high-confidence source/test reproduction exists: create a session JSONL with ordinary user/assistant messages, trigger the hook with `action: "reset"`, and current code/tests show raw transcript-shaped lines written into `memory/YYYY-MM-DD-*.md`; live autonomous re-execution depends on model and tool environment.

## Solution Assessment

Yes for the direction, but the final shape needs maintainer policy. The narrow maintainable fix should combine `/reset` semantics with durable untrusted framing or neutral summaries; token-only sanitizers are useful but do not close this broader issue.
