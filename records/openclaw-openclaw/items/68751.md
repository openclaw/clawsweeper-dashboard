## Summary

Keep open. Current main and v2026.5.12 still persist transcript-shaped prior user and assistant turns on `/reset`; the startup-context untrusted wrapper is only a later read-path mitigation, and the open sanitizer PR handles malformed-token poisoning rather than ordinary prior-command replay.

## What This Changes

The issue asks for session-memory to stop replaying raw prior-session turns after `/reset`, either by untrusted framing, neutral summarization, or not persisting reset material.

## Best Possible Solution

Define the session-memory trust boundary end to end: `/reset` should break continuity by default, and any `/new` continuity should be stored and retrieved only as untrusted historical reference or a neutral summary.

## Reproduction Assessment

Yes, at source level. A JSONL session with ordinary user and assistant messages is formatted as role-prefixed text and written raw under `## Conversation Summary` when the hook handles `action: "reset"`; live autonomous re-execution still depends on model and tool environment.

## Solution Assessment

Unclear for the exact final policy. Wrapping or summarizing historical content is the right direction, but https://github.com/openclaw/openclaw/pull/69961 is not sufficient by itself because it does not address ordinary prior commands or `/reset` continuity semantics.
