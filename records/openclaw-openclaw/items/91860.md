## Summary

Keep open: current `main` and latest release still have the CLI send path that forwards only `parseMode`, while Discord already honors `formatting.maxLinesPerMessage` when it is provided. The linked fix path is https://github.com/openclaw/openclaw/pull/91885, which is still open and not merged, so this issue should stay open until that PR lands or is replaced.

## What This Changes

The issue reports that Discord CLI message sends ignore `channels.discord.maxLinesPerMessage`, split around the Discord default line cap, and return an incomplete send receipt for chunked sends.

## Best Possible Solution

Keep this issue open until https://github.com/openclaw/openclaw/pull/91885 lands with an owner-safe `maxLinesPerMessage` plumbing fix and real Discord readback proof, or is replaced by a narrower equivalent fix.
