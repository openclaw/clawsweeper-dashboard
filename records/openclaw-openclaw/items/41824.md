## Summary

Keep open: current main still has a source-reproducible ACP path where Telegram/chat-visible final text can be delivered after terminal completion without a completed-action evidence contract. The related broader truthfulness and fail-closed items are useful context, but they do not supersede this narrower ACP final-delivery gap.

## What This Changes

The issue asks OpenClaw to prevent Telegram-visible ACP agents from claiming completed side effects before the underlying action is verified.

## Best Possible Solution

Define a generic ACP runtime/final-delivery contract that carries completed-action evidence and can hold, revise, or clearly mark unsupported success claims before chat delivery, then prove it with focused ACP tests and real Telegram validation.

## Reproduction Assessment

Yes, source-reproducible: a mock ACP runtime can emit a completed-action-looking text_delta followed by done, and current main will deliver that text as a final chat reply without completed-action evidence. I did not run a live Telegram/file-system replay of the reporter's private session.

## Solution Assessment

Unclear until maintainers choose the ACP contract. The maintainable direction is a generic evidence/finalization gate, not Telegram-specific phrase filtering or an after-the-fact hook that only logs mismatches.
