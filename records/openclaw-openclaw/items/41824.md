## Summary

Keep this open: current main still accepts and delivers ACP final text on the Telegram/ACP path after a terminal `done` event without structured evidence that claimed file or message actions actually completed.

## What This Changes

Request a Telegram/ACP chat-surface guard that prevents or detects final replies claiming completed file/message actions when no tool/action completion evidence exists.

## Best Possible Solution

Define a generic ACP completed-action evidence contract and final-answer gate that can warn, revise, or suppress unsupported completion claims across chat channels while preserving legitimate text-only answers.

## Reproduction Assessment

Yes. From source, an ACP runtime can emit visible text such as “I wrote it” followed by `done` with no completed-action evidence, and dispatch will flush and finalize that text. I did not run a live Telegram session because this review is read-only.

## Solution Assessment

No. Current main’s final-only buffering and terminal-done guard only control delivery timing/completion; they do not verify claims. The maintainable direction is a generic ACP evidence ledger plus final-answer gate, not Telegram-specific phrase filtering.
