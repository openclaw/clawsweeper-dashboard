## Summary

Keep open: current main still permits ACP text to become the final Telegram/chat-visible reply after a terminal `done` without any completed-action evidence, and the live issue is still labeled for maintainer/product follow-up rather than cleanup.

## What This Changes

The issue asks OpenClaw to stop Telegram/ACP agent turns from reporting completed file-write or messaging actions unless those actions actually executed.

## Best Possible Solution

Define a generic ACP runtime and final-delivery contract that can carry completed-action evidence and hold or revise unsupported completion claims before chat delivery, without adding Telegram-specific phrase filtering.

## Reproduction Assessment

Yes, source-reproducible: a mock ACP runtime can emit visible text such as a completed-action claim followed by `done`, and current main will flush that text as a final chat reply without completed-action evidence. I did not run a live Telegram/file-system replay of the reporter's private session.

## Solution Assessment

Unclear until maintainers choose the ACP contract, but the likely best direction is a generic evidence/finalization gate rather than a Telegram-only workaround. The current source path is too permissive for action-completion claims.
