## Summary

Keep open. Current main has useful mitigations for ACP and message-tool-only delivery, but it still accepts ACP visible text as final output without a completed-action evidence gate, and related Telegram/channel-visible leakage work remains open.

## What This Changes

The issue reports Telegram ACP turns claiming file-write or message-send actions completed before the underlying action actually happened.

## Best Possible Solution

Define a generic ACP/channel finalization policy that prevents visible completed-action claims unless the turn has corresponding tool/action evidence or uses an explicit message-tool send path, with focused Telegram and ACP regression coverage.

## Reproduction Assessment

Yes, from source rather than a live Telegram run. An ACP runtime can emit visible final text such as a completed-action claim followed by `done`, and current projector/dispatch code will finalize that text without checking completed-action evidence.

## Solution Assessment

No, current main is not a complete solution. Message-tool-only guidance and commentary suppression help, but the remaining invariant needs a maintainer decision across ACP final text, direct Telegram delivery, and related explicit-send/final-only policy work.
