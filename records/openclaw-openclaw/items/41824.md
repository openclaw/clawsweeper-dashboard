## Summary

Keep open: current main still gates ACP chat delivery on terminal events, not on completed-action evidence, so the reported Telegram/ACP false-completion path remains possible and needs maintainer design work.

## What This Changes

The issue reports ACP/Telegram agents claiming file-write or message-sending actions completed before the side effects actually happened.

## Best Possible Solution

Define a generic ACP completed-action evidence ledger plus final-answer revision gate that chat channels can use while preserving valid text-only replies.

## Reproduction Assessment

Yes. From source, an ACP runtime can emit visible text such as "I wrote it" followed by done, and dispatch/projector code will flush and finalize that text without completed-action evidence; I did not run a live Telegram session in this read-only review.

## Solution Assessment

No. Current final-only buffering and the terminal done guard address delivery timing, but not claim verification; a generic ACP evidence/finalization contract is safer than Telegram-specific phrase filtering.
