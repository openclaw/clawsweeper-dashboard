## Summary

Keep this open: current main has adjacent ACP streaming and terminal-done hardening, but it still delivers ACP final text on Telegram without checking claims of completed file/message actions against observed completed mutating events.

## What This Changes

The issue asks OpenClaw to prevent or flag Telegram-bound ACP sessions that claim file-write or message-sending actions completed when no matching action actually executed.

## Best Possible Solution

Add a generic ACP final-answer evidence path that records observable mutating tool/native events and warns, revises, or blocks final claims of completed actions when no matching evidence exists.

## Reproduction Assessment

Yes. A focused source-level reproduction is clear: have ACP runTurn emit a visible text_delta such as “I wrote it” followed by done, with no completed tool/action evidence, and dispatch will flush and deliver that text.

## Solution Assessment

No. Current main’s final-only projection and terminal done guard are useful adjacent fixes, but they do not solve the central safety property; the maintainable fix is an explicit completed-action evidence contract plus a final-answer gate.
