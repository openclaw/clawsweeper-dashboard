## Summary

Keep open: current main still permits ACP final text to be delivered after a terminal `done` event without a typed completed-action ledger or final-answer gate, so the reported false-completion failure remains possible on the Telegram/ACP path.

## What This Changes

The issue asks OpenClaw to stop Telegram/ACP agents from reporting completed file or message actions before the underlying tool side effects are evidenced.

## Best Possible Solution

Define a generic ACP completed-action evidence ledger and final-answer or revision gate across chat channels while preserving legitimate text-only replies.

## Reproduction Assessment

Yes, from source rather than a live Telegram run: an ACP runtime can emit visible text such as “I wrote it” followed by `done`, and current dispatch/projector code will flush and finalize that text without completed-action evidence.

## Solution Assessment

No. Current main’s final-only buffering and terminal-done guard solve delivery timing/completion, but not claim verification; the maintainable fix is a generic ACP evidence contract plus final-answer gate, not Telegram-specific string heuristics.
