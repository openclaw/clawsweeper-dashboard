## Summary

Keep open. Current main still preserves same-session serialization and has better generic Gateway-timeout fallback plus clearer diagnostics, but the lane wait facts remain diagnostic-only and are not returned to the CLI as an actionable lane-busy or wait-timeout result.

## What This Changes

This issue asks for `openclaw agent --message` to print a clear lane-busy or wait-timeout error when a same-session request is blocked behind active work instead of exiting without actionable output.

## Best Possible Solution

Add an additive queue/Gateway/CLI lane-wait timeout or busy result that preserves same-session serialization and surfaces lane, queueAhead or activeAhead, waitedMs, and clear retry guidance.

## Reproduction Assessment

Yes, at source level. Same-session agent runs are serialized through the session lane, wait facts are only logged by the queue, and the CLI has only a generic Gateway-timeout fallback path.

## Solution Assessment

Yes. The narrow maintainable fix is to surface existing blocked-lane facts through an additive queue/Gateway/CLI error or result contract, not to relax session serialization or add a new queue policy by default.
