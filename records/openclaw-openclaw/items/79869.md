## Summary

Keep open. The PR is aimed at a real current-main bug, but the latest head still has a blocking fallback scheduling defect and lacks after-fix real behavior proof for the external contributor gate.

## What This Changes

The PR changes `src/infra/heartbeat-runner.ts` and focused tests so targeted `exec-event` heartbeat wakes can run when `heartbeat.every` is disabled with `0m`.

## Best Possible Solution

Preserve the narrow zero-interval `exec-event` bypass, but requeue or schedule deferred fallback wakes for their due time, add close-together completion coverage, and provide redacted real-run proof.

## Reproduction Assessment

Yes, by source inspection: configure `agents.defaults.heartbeat.every` to `0m`, let a background exec finish, and current main requests an `exec-event` wake that both direct and scheduler paths treat as disabled. I did not run a live gateway repro in this read-only review.

## Solution Assessment

No. The PR direction is narrow and appropriate, but the current fallback can return a cooldown deferral without any retry or timer path, so a later close-together completion can remain queued indefinitely.
