## Summary

Keep open. The PR targets a real current-main bug in disabled-heartbeat exec completion delivery, but the current branch still drops deferred zero-interval exec-event wakes and lacks after-fix real behavior proof.

## What This Changes

The PR changes the heartbeat runner and focused tests so targeted exec-event wakes can bypass disabled periodic heartbeat intervals.

## Best Possible Solution

Keep the fix scoped to reliable system-event or targeted exec completion delivery, ensure deferred zero-interval wakes are requeued or scheduled, and land only after real behavior proof with focused regression coverage.

## Reproduction Assessment

Yes, by source inspection: configure `agents.defaults.heartbeat.every` to `0m`, let a background exec finish, and current main requests an `exec-event` wake that the direct and scheduler paths treat as disabled. I did not run a live gateway reproduction in this read-only review.

## Solution Assessment

No. The narrow bypass is plausible, but the PR fallback returns cooldown deferrals without requeueing or timer scheduling, so close-together completions can still remain queued until another wake arrives.
