## Summary

Keep open: the PR targets a real source-reproducible disabled-heartbeat exec completion bug, but PR head still drops deferred zero-interval exec-event wakes and lacks after-fix real behavior proof.

## What This Changes

The PR lets targeted exec-event heartbeat wakes run when `heartbeat.every` is `0m`, adds fallback cooldown state, and adds focused regression tests.

## Best Possible Solution

Keep the narrow exec-event disabled-interval bypass, but requeue or schedule deferred fallback wakes and add real setup proof for background exec completion with `every: "0m"`.

## Reproduction Assessment

Yes, source-reproducible: on current main, `heartbeat.every: "0m"` resolves to no interval, exec completion requests a targeted exec-event wake, and direct/scheduler heartbeat paths skip as disabled. I did not run a live gateway repro in this read-only review.

## Solution Assessment

No. The PR direction is narrow and appropriate, but PR head returns deferred fallback exec-event wakes without a retry or timer path, so close-together completions can still remain queued until an unrelated wake happens.
