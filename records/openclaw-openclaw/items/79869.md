## What This Changes

The PR changes `src/infra/heartbeat-runner.ts` and focused heartbeat tests so targeted `exec-event` wakes can run when `heartbeat.every` is `0m`.

## Best Possible Solution

Keep the narrow targeted `exec-event` bypass, but requeue or schedule deferred zero-interval fallback wakes, cover close-together completions, and prove real background exec completion delivery before merge.

## Reproduction Assessment

Yes, by source inspection: `every: "0m"` resolves disabled, background exec completion requests a targeted `exec-event` wake, and current main's direct and scheduler paths treat it as disabled. I did not run a live gateway reproduction in this read-only review.

## Solution Assessment

No. The PR direction is narrow and maintainable, but PR head consumes deferred zero-interval fallback wakes without a retry or timer path, so it is not merge-ready.
