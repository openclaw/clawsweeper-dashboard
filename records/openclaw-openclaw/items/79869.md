## Summary

Keep open: the PR still has a blocking deferred-wake bug and lacks external real behavior proof, while current main still source-reproduces the narrowed disabled-heartbeat exec completion bug.

## What This Changes

The PR changes heartbeat runner dispatch so targeted exec-event wakes can run when `heartbeat.every` is disabled and adds focused regression coverage.

## Best Possible Solution

Keep the narrow exec-event bypass, but schedule or requeue zero-interval fallback deferrals and prove a real background exec completion with `heartbeat.every: "0m"` before merge.

## Reproduction Assessment

Yes for the narrowed current-main bug by source inspection: configure `heartbeat.every` to `0m`, let a background exec finish, and the current exec-event wake is treated as disabled by both the direct and scheduler paths. I did not live-run the setup in this read-only review.

## Solution Assessment

No. The direction is narrow, but the proposed fallback returns deferred zero-interval exec-event wakes as ordinary skipped outcomes with no retry or schedule path, so it should be revised before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Requeue deferred zero-interval exec events:** `src/infra/heartbeat-runner.ts:2243-2245`
  - body: When `heartbeat.every` is `0m`, the fallback agent lives outside `state.agents`; a second targeted `exec-event` inside cooldown returns `skipped/not-due` or `min-spacing`, but the wake layer only retries busy skips and `scheduleNext()` still exits with no interval agents.
