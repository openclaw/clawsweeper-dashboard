## Summary

Keep this PR open, but do not merge it as-is. The central fix is still useful on current main, but the branch is conflicting, lacks after-fix real behavior proof, and the zero-interval fallback can drop a deferred exec-event wake instead of retrying it after cooldown.

## What This Changes

The PR changes the heartbeat runner so targeted exec-event wakes can bypass a disabled heartbeat interval and adds scheduler/runHeartbeatOnce regression tests for that path.

## Best Possible Solution

Repair the fallback so deferred zero-interval exec-event wakes are rescheduled and eventually delivered, rebase onto current main, add focused regression coverage for the delayed second completion, and provide redacted real gateway/agent proof before merge.

## Reproduction Assessment

Yes, at source level: heartbeat.every "0m" resolves to no interval-backed agent, background exec completion requests an exec-event wake, and current main skips that delivery path. I did not run a live gateway/Codex scenario in this read-only review.

## Solution Assessment

No. The PR is aimed at the right owner boundary, but the current fallback returns a non-retryable cooldown skip for repeated zero-interval exec events, so it is not the best mergeable shape yet.
