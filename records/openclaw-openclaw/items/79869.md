## Summary

Keep open. The narrowed bug is source-reproducible and this PR is directionally relevant, but the current fallback can still lose deferred zero-interval exec-event wakes and the external PR still lacks after-fix real behavior proof.

## What This Changes

The PR changes the heartbeat runner and focused infra tests so targeted exec-event wakes can bypass disabled periodic heartbeat intervals.

## Best Possible Solution

Preserve targeted exec completion delivery when periodic heartbeats are disabled, but make zero-interval deferrals requeue or move completion delivery to a dedicated system-event path, with focused regression coverage and real setup proof.

## Reproduction Assessment

Yes, by source inspection: with `agents.defaults.heartbeat.every` set to `0m`, background exec completion enqueues a targeted `exec-event` wake while current main skips disabled intervals in both direct and scheduler paths. I did not run a live gateway reproduction in this read-only review.

## Solution Assessment

No. The PR direction is plausible, but the fallback state is outside scheduler re-arm and deferred second completions are not requeued; the prior closed attempt also supports considering a dedicated system-event delivery path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Requeue deferred zero-interval exec events:** `src/infra/heartbeat-runner.ts:2245`
  - body: When `heartbeat.every` is `0m`, the fallback agent lives in `state.execEventAgents`, but `scheduleNext()` only scans `state.agents` and the wake layer only requeues retryable busy skips.
