## Summary

Keep open. The PR addresses a source-reproducible routing gap, but the patch currently treats any preloaded non-empty `BOOTSTRAP.md` as hook-provided content and can re-enable full bootstrap for workspaces that current main marks complete.

## What This Changes

The PR preloads hook-adjusted bootstrap files before embedded-runner routing, reuses them for Project Context generation, adds routing coverage, and adds a changelog entry.

## Best Possible Solution

Keep the preload-and-reuse direction, but make the pending override provenance-aware so only hook-added or hook-overridden non-empty `BOOTSTRAP.md` content can bypass filesystem pending state.

## Reproduction Assessment

Yes, via source inspection: current main routes before hook-adjusted bootstrap files are resolved, then filters `BOOTSTRAP.md` out when routing selected `none`. I did not run tests because this review was read-only.

## Solution Assessment

No. The proposed shape is close, but the predicate must distinguish hook-provided content from ordinary preloaded filesystem content so it preserves the existing completed-workspace contract.
