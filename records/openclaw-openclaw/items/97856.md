The PR adds an optional `SubsystemLogger` to `registerChatAbortController`, passes `context.logGateway` from chat and agent handlers, and tests logging when cleanup sees a rejected terminal-persistence promise.

## Best Possible Solution

Log rejected terminal persistence at the gateway owner or tracking boundary so every cleanup ordering is covered once, preserve existing cleanup/recovery behavior, and add both regression coverage and redacted real log proof.

## Reproduction Assessment

Yes from source inspection. A rejected terminal persistence promise can still pass through `trackTrackedRunTerminalPersistence` after cleanup was requested and be consumed by `catch(() => undefined)` without logging.

## Solution Assessment
