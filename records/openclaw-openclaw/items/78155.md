Full review comments:

- **[P1] Match concrete allowlist names against contract globs:** `src/plugins/tools.ts:373-374`
  - body: With `contracts.tools: ["xxx-*"]` and a concrete tool allowlist like `xxx-test`, this code tests the manifest entry `xxx-*` against a glob compiled from `xxx-test`, so `selectedToolNames` is empty and the plugin is never cold-loaded. The matching needs to prove that requested/allowlisted concrete names are covered by the contract pattern, or carry the pattern owner through runtime discovery.
  - confidence: 0.9
- **[P1] Remove the unrelated trajectory config change:** `src/config/types.trajectory.ts:1-10`
  - body: This PR is for `contracts.tools` glob support, but it also adds a new `trajectory.flushTimeoutMs` config/env feature that closes a different issue and has no schema/docs coverage in this branch. Please split or drop this change so the manifest-contract feature can be reviewed on its own.
  - confidence: 0.96
- **[P2] Do not reschedule after the final trajectory flush:** `src/trajectory/runtime.ts:420`
  - body: If the trajectory change is kept elsewhere, `flush()` should not unconditionally schedule another referenced timeout after deleting the writer from the cache. Final cleanup callers use `flush()` to drain and release trajectory state, so this can keep completed runs alive until the timeout and then flush a stale writer.
  - confidence: 0.87
