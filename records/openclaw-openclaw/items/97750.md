## Summary

Keep open: current main and the latest release still do not recognize `antml:` namespaced invoke/parameter tool-call XML in either the visible-text sanitizer or the plain-text tool-call repair parser. The open invoke-style repair PR is the right candidate path, but it is not merged and currently has sanitizer/security-boundary findings, so this issue should remain open until that PR or a corrected replacement lands.

## What This Changes

Issue reports that Anthropic `antml:` namespaced tool-call XML leaks into visible assistant content, and sometimes fails the turn, when an Anthropic-Messages-compatible proxy degrades Claude `tool_use` blocks into text.

## Best Possible Solution

Land a corrected invoke-style repair that extends the existing tool-call repair parser and visible-text sanitizer to handle namespaced Anthropic XML without weakening the boundary that strips internal tool-call text from user-visible output.

## Reproduction Assessment

Yes for the source-level bug: current main does not parse `antml:` tool tags or attribute-style invoke parameters in the cited sanitizer/repair paths. I did not run a live proxy repro.

## Solution Assessment

Yes, the right shape is to extend existing tool-call sanitizer and repair paths; the open candidate PR uses the right general layer but still needs its sanitizer regression fixed.
