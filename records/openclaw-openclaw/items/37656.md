## Summary

Keep open: current main still lacks exec-time per-workspace `.env` loading, and this branch remains useful but not mergeable because it bypasses part of the established workspace dotenv denylist and lacks real behavior proof.

## What This Changes

This PR adds gateway-host exec-time loading of `<workdir>/.env` into the exec environment, filters some host override keys, updates exec/MS Teams tests, and adds a changelog entry.

## Best Possible Solution

If maintainers want this feature, keep it gateway-host only, route parsing through the existing workspace dotenv filtering contract, add regression coverage for blocked keys, and require redacted live exec proof before merge.

## Reproduction Assessment

Not applicable as a user-bug reproduction path for this feature PR. The patch defect is source-reproducible from PR head: parsed `.env` keys are returned after base-env checks while current main blocks additional workspace dotenv key families.

## Solution Assessment

No, not yet. Gateway-only request-time loading is a plausible direction, but the implementation should share the existing workspace dotenv contract instead of duplicating a narrower parser/filter.
