Keep open: the feature is not implemented on current main and the PR is still potentially useful, but it is not merge-ready because it lacks real behavior proof and still has a source-reproducible workspace `.env` trust-boundary bug.

## What This Changes

The PR adds request-time gateway-host exec loading of workspace `.env` variables, filters some host override keys, updates focused exec/MS Teams tests, and adds a changelog entry.

## Best Possible Solution

Reuse or export the current workspace dotenv filtering contract for exec-time gateway loads, add blocked-key regression coverage, and provide redacted live gateway/agent exec proof before maintainer review.

## Reproduction Assessment

Not applicable to the feature request itself. The blocking correctness/security concern is source-reproducible from PR head because parsed workspace `.env` keys are injected before the established workspace dotenv denylist is applied.

## Solution Assessment

No, not yet. Gateway-only request-time loading is a plausible direction, but it should share the current untrusted workspace `.env` filter before workspace-derived values become host exec env.
