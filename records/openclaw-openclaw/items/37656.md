## Summary

Keep open: current main still lacks per-agent exec-time workspace `.env` loading, and this PR targets the right exec boundary, but the current diff has a security-policy bypass plus test/changelog gaps before merge.

## What This Changes

The PR loads resolved gateway workspace `.env` files into exec child environments at request time, skips node/sandbox hosts, filters host override-blocked keys, adds host-selection coverage, and adjusts an MS Teams async assertion.

## Best Possible Solution

Land a revised implementation that reads the resolved gateway workspace `.env` at exec request time, reuses the existing workspace dotenv blocklist through a non-mutating helper, preserves non-override semantics including Windows casing, skips node and sandbox hosts, isolates tests in temp workspaces, and includes the required changelog entry.

## Reproduction Assessment

Yes. Source inspection shows current main passes the agent workspace as exec `cwd` but builds child env only from inherited process env plus request overrides, and the PR defects are reproducible from the diff without running tests.

## Solution Assessment

No, not in the current form. The exec-time boundary is the right place for this feature, but the implementation needs the existing workspace dotenv policy, isolated test fixtures, and release notes before it is maintainable.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Apply the workspace dotenv blocklist:** `src/agents/bash-tools.exec.workspace-env.ts:35-49`
  - body: Current startup workspace `.env` loading blocks credential and runtime-control keys, but this new loader parses every key and only skips already-defined vars. Host override diagnostics block command-pivot keys such as `NPM_CONFIG_*`, but not all workspace-blocked keys like `OPENAI_API_KEY` or `OPENCLAW_GATEWAY_TOKEN`, so repo-controlled `.env` values can enter gateway exec child environments. Reuse or export the workspace dotenv policy before returning injected vars.
  - confidence: 0.86
- **[P2] Use a temp workspace for the dotenv injection test:** `src/agents/bash-tools.test.ts:595-597`
  - body: This test writes `.env` into `process.cwd()`, which is the repo root under Vitest. Other dotenv/config-loading tests also read the CWD `.env`, so parallel workers can observe `GH_CONFIG_DIR` and fail nondeterministically; create an isolated temp workspace and pass that as `cwd` instead.
  - confidence: 0.93
- **[P3] Add the required changelog entry:** `src/agents/bash-tools.exec.ts:365-372`
  - body: This PR adds a user-facing `feat` that changes how exec child environments are populated, but it does not update `CHANGELOG.md`. Add a single-line entry under the active version so the environment-loading behavior is documented for users before merge.
  - confidence: 0.82
