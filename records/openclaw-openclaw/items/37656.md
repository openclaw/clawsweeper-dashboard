## Summary

Keep open: the feature is not implemented on current main, but this PR should not merge yet because it bypasses the current workspace dotenv denylist and still lacks after-fix real behavior proof.

## What This Changes

The PR adds gateway-host exec-time loading of `<workdir>/.env` into exec environments, plus focused exec tests and a changelog entry.

## Best Possible Solution

Keep the feature gateway-host only, route exec-time `.env` parsing through the current workspace dotenv filtering contract, add blocked-key regression coverage, and require redacted real exec proof before merge.

## Reproduction Assessment

Not applicable as a user-bug reproduction path for this feature PR. The patch defect is source-reproducible: PR head returns parsed workspace `.env` keys without the current workspace dotenv denylist.

## Solution Assessment

No. Gateway-only request-time loading is plausible, but the implementation should reuse or factor the current workspace dotenv filtering contract instead of adding a second parser with weaker filtering.
