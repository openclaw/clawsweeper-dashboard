## Summary

## What This Changes

The PR adds gateway-host exec-time workspace `.env` loading from the resolved workdir, filters some workspace-derived keys through host override diagnostics, adds exec coverage, updates an MS Teams async setup test, and adds a changelog entry.

## Best Possible Solution

Keep the gateway-only request-time loading shape only if maintainers want this feature, but reuse the existing workspace dotenv filtering contract before merge and require redacted live exec proof.

## Reproduction Assessment

Not applicable as a user-bug reproduction path for this feature PR. The patch defect is source-reproducible: PR head injects parsed `.env` keys after only base-env and host-override filtering, while current main blocks additional workspace dotenv key families.

## Solution Assessment

No, not yet. Gateway-only request-time loading is a plausible direction, but the implementation should share the existing workspace dotenv contract instead of duplicating a narrower parser/filter.
