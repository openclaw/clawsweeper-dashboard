## Summary

Keep open: current main has not implemented per-agent exec-time workspace `.env` loading, and this PR still needs a security fix plus real behavior proof before it can be considered for merge.

## What This Changes

The PR adds gateway-host exec-time loading of `<workdir>/.env` into the exec environment, filters some host override keys, adds focused exec tests, updates one MS Teams async setup test, and adds a changelog entry.

## Best Possible Solution

If maintainers want this feature, keep it gateway-only but route parsing through the existing workspace dotenv filtering contract and require redacted live exec proof before merge.

## Reproduction Assessment

Not applicable as a user-bug reproduction path for this feature PR. The patch defect is source-reproducible from the PR head: parsed `.env` keys are returned after base-env checks while current main blocks additional workspace dotenv key families.

## Solution Assessment

No, not yet. Gateway-only request-time loading is a plausible direction, but the implementation should share the existing workspace dotenv contract instead of duplicating a narrower parser/filter.
