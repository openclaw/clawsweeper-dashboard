## Summary

Keep open: current main does not implement per-agent exec-time workspace `.env` loading, and the PR branch still has concrete merge blockers in the exec env security boundary, test isolation, and changelog coverage.

## What This Changes

The PR adds gateway-host exec-time loading of `<workdir>/.env` into the host exec environment, adds focused exec host-scope tests, and updates an MS Teams async status assertion.

## Best Possible Solution

Keep the feature scoped to gateway-host exec, reuse the existing workspace-dotenv safety policy for all workspace-loaded keys, isolate regression tests, document the new precedence, and then run the changed gate before maintainer review.

## Reproduction Assessment

Not applicable as a feature PR. Source inspection shows current main only uses startup CWD/global dotenv loading plus `process.env` and request `env` for exec, so there is no broken documented behavior to reproduce as a bug.

## Solution Assessment

No, not yet. The gateway-only direction is plausible, but the PR should first preserve current workspace-dotenv blocklists, avoid writing tests into the repo root, and add the required changelog entry.

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Apply the workspace dotenv blocklist before injecting exec env:** `src/agents/bash-tools.exec.workspace-env.ts:43-49`
  - body: The loader accepts every parsed key that is absent from `baseEnv`, then `createExecTool` filters only host-override blocked keys. Current workspace `.env` policy also blocks `OPENCLAW_*`, credential, endpoint, and runtime-control keys; without reusing that policy, a workspace `.env` can inject values like `OPENCLAW_CONFIG_PATH` into gateway host exec child processes.
  - confidence: 0.87
- **[P2] Isolate the .env test from the repo root:** `src/agents/bash-tools.test.ts:598-620`
  - body: The new test writes `process.cwd()/.env`, which is the repository root during normal Vitest runs. Since OpenClaw's dotenv loader reads the CWD `.env`, parallel tests can observe the temporary `GH_CONFIG_DIR`, and an interrupted run can leave the checkout dirty; use an isolated temp directory as the exec cwd instead.
  - confidence: 0.92
- **[P2] Add the required changelog entry:** `src/agents/bash-tools.exec.ts:365-373`
  - body: This is a user-facing `feat` that changes exec environment behavior, but the PR does not touch `CHANGELOG.md`. Repo policy requires a changelog entry so the new workspace `.env` precedence and gateway-host scope are visible in release notes.
  - confidence: 0.9
