## Summary

Keep open. Current main still does not load each agent workspace `.env` into exec at request time, so this PR is not obsolete, but the current patch should not merge because it bypasses the stricter workspace dotenv blocklist and adds a repo-root `.env` test race.

## What This Changes

The PR adds gateway-only workspace `.env` loading for exec after workdir resolution, filters host override-blocked variables, skips node/sandbox hosts, adds gateway/node coverage, and updates one MS Teams async assertion.

## Best Possible Solution

Land a revised implementation on this PR or a maintainer replacement that reads the resolved gateway agent workspace `.env` at exec request time, reuses the workspace dotenv blocklist through a non-mutating helper, preserves non-override semantics including Windows casing, skips node and sandbox hosts, isolates tests in temp workspaces, and updates docs/changelog when shipped.

## Reproduction Assessment

Yes. By source inspection, current main passes the agent workspace as exec cwd but builds child env only from `process.env` and request overrides; a focused temp-workspace check with `GH_CONFIG_DIR` in `.env` would show the missing behavior, while the PR security issue is source-reproducible with workspace-blocked keys in `.env`.

## Solution Assessment

No, not in the current form. The PR targets the right exec boundary, but the maintainable version needs to reuse the existing workspace dotenv policy before merging values into host exec env and should avoid repo-root filesystem mutation in tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Apply the workspace dotenv blocklist:** `src/agents/bash-tools.exec.workspace-env.ts:35-42`
  - body: Current startup workspace `.env` loading blocks credential and OpenClaw runtime-control keys, but this new loader parses every key and only skips already-defined vars. Host override diagnostics later block command-hijack keys such as `NPM_CONFIG_*`, but not workspace-blocked keys like `OPENAI_API_KEY` or `OPENCLAW_GATEWAY_TOKEN`, so repo-controlled `.env` values can enter gateway exec child environments. Reuse or export the workspace dotenv policy before returning injected vars.
  - confidence: 0.86
- **[P2] Use a temp workspace for the dotenv injection test:** `src/agents/bash-tools.test.ts:841-842`
  - body: This test writes `.env` into `process.cwd()`, which is the repo root under Vitest. Other dotenv/config-loading tests also read the CWD `.env`, so parallel workers can observe `GH_CONFIG_DIR` and fail nondeterministically; create an isolated temp workspace and pass that as `cwd` instead.
  - confidence: 0.93
