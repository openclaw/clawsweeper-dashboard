Keep this PR open. Current main still treats a null registry target as downgrade risk, so the PR addresses a real update CLI bug; the code/test direction looks narrow and correct, with the remaining issue being the required changelog entry plus normal rebase and validation.

The PR changes `openclaw update` downgrade detection so unresolved `latest` dist-tag lookups skip the false downgrade prompt while unresolved non-`latest` tags remain guarded, and adds regression tests for both paths.

## Best Possible Solution
