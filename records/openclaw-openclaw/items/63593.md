## Summary

Keep open. Current main still treats an unresolved registry-resolvable `latest` target as downgrade risk, while this PR keeps the conservative unresolved non-`latest` guard and adds focused regression coverage for the `latest` false positive.

## What This Changes

The PR changes `openclaw update` downgrade detection so unresolved `latest` dist-tag lookups no longer require downgrade confirmation, and adds regression tests for unresolved `latest` and unresolved non-`latest` tags.

## Best Possible Solution

Land a rebased narrow update CLI fix that exempts unresolved `latest` lookups from downgrade confirmation while preserving downgrade protection for comparable older targets and unresolved non-`latest` dist-tags, with the focused regression tests kept in `src/cli/update-cli.test.ts`.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a package install with a known current version, explicit `--tag latest`, and `fetchNpmTagVersion` returning `version: null`; current main then sets `downgradeRisk` and exits in non-interactive mode with the downgrade confirmation error.

## Solution Assessment

Yes, pending normal maintainer review and rebase. The PR is the narrowest maintainable direction because it changes only the null-target downgrade gate for `latest` and adds tests proving both the exempted path and the preserved non-`latest` guard.
