Keep open: current main still has an English-only access-denied matcher for the Windows `schtasks` Startup-folder fallback, while docs and tests establish that denied task creation should fall back instead of failing.

Make Windows `node install` treat Spanish `schtasks` `Acceso denegado` failures as access denied and use the existing Startup-folder fallback.

## Best Possible Solution

Add narrow access-denied detection for localized Windows `schtasks` denial output in the existing fallback predicate, cover it with regression tests, and add a user-facing changelog fix entry.
