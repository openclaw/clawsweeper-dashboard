## Summary

Keep open: current main and the latest release cover pure Codex sessions with `tools.exec.host=node`, but they still do not provide the reported mixed-session surface that keeps Codex native local tools while exposing an explicit OpenClaw-owned node exec tool.

## What This Changes

The issue requests a usable WebChat/OpenAI Codex remote-node exec path when a node is connected, especially for mixed sessions that keep local Codex execution and also need explicit node-side shell execution.

## Best Possible Solution

Keep the issue open for a maintainer-approved design that either adds an explicit, policy-preserving mixed-session node exec dynamic tool or documents why users must choose a fully node-bound Codex session instead.

## Reproduction Assessment

Yes, source-level reproduction is high-confidence: current main has no mixed-session `node_run` surface, `auto` resolves away from node, and node shell tools are only restored when the effective exec host is already node.

## Solution Assessment

Unclear. The reported `node_run` shape is plausible because Codex supports dynamic tools and OpenClaw already has a policy-backed node exec path, but maintainers need to choose between a generic remote shell tool, node-hosted capability tools, or an explicit docs-only stance.
