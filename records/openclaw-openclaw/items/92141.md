## What This Changes

The issue requests a usable Codex/WebChat mixed-session path for executing commands on a paired node while keeping native local Codex shell/files available.

## Best Possible Solution

Define an explicit mixed-session node exec surface or diagnostic for Codex/WebChat that reuses existing `host=node` exec approvals while keeping native Codex local tools separate.

## Reproduction Assessment

Yes from source inspection, not from a live paired Windows node: current Codex tool building only restores OpenClaw exec/process when effective exec host is `node`, while the reported `host=auto` mixed session leaves native local Codex execution enabled and adds no separate node shell tool.

## Solution Assessment

Unclear until maintainers choose the policy: a separate OpenClaw-owned node exec dynamic tool matches Codex's dynamic-tool contract, but it needs explicit security and UX rules instead of silently making `auto` a node route.
