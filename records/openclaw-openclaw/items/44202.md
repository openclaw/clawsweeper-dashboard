## Summary

Keep this open. Current main still lacks a supported backend selector or official Apple Silicon Metal/CPU guidance for built-in local embeddings, and the local GGUF batch path still fans out concurrent `getEmbeddingFor` calls; the QMD and keepmyclaw workaround comments are useful recovery context but do not resolve the OpenClaw reliability request.

## What This Changes

The issue requests stable, documented Apple Silicon local-memory embeddings, including Metal/GPU guidance, CPU-only or backend control, safer batching, and gateway-safe failure handling for native embedding crashes.

## Best Possible Solution

Keep this as the canonical Apple Silicon local-embedding stability tracker until OpenClaw has a documented macOS support matrix, a validated backend selector or CPU-only mode, bounded local embedding concurrency, and a gateway-safe fallback or isolation plan for native crashes.

## Reproduction Assessment

Yes, with source-backed limits rather than a live local repro. The issue and comments give a high-confidence Apple Silicon GGUF/Metal reproduction shape, and current main still lacks the backend controls and safe batching requested for that path.

## Solution Assessment

No complete solution is present on current main. #60347 is a useful partial backend-selector proposal, but the maintainable fix also needs docs/support policy, local batching semantics, and a crash-boundary plan.
