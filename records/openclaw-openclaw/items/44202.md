## Summary

Keep this open. Current main still lacks a supported local embedding CPU/GPU/Metal backend selector, still runs built-in GGUF batch embeddings concurrently through one node-llama-cpp context, and does not document an Apple Silicon Metal-safe path; the keepmyclaw and CPU/QMD workaround comments are useful recovery context but do not resolve the OpenClaw reliability request.

## What This Changes

The issue requests official handling for Apple Silicon local GGUF/Metal embedding crashes, including CPU/GPU guidance, safer gateway failure boundaries, and serialized local embedding behavior.

## Best Possible Solution

Keep this as the canonical Apple Silicon local-embedding stability tracker until OpenClaw has a documented macOS support matrix, validated backend selector or CPU-only mode, bounded or serialized built-in GGUF embedding calls, and a gateway-safe fallback or isolation plan for native crashes.

## Reproduction Assessment

Yes. The issue and follow-up comments give a high-confidence reproduction shape using Apple Silicon, local GGUF embeddings, node-llama-cpp/ggml-metal, restart/shutdown or QMD cleanup, and SIGABRT/non-zero exit behavior; current source inspection confirms the requested controls are still missing.

## Solution Assessment

No complete solution is present on current main. #60347 is a useful partial path for backend selection, but the maintainable fix also needs docs/support policy, local batching semantics, and crash-boundary design.
