## Summary

Keep this open. Current main has partial QMD mitigations, but the built-in local GGUF embedding path still lacks a documented CPU/GPU/Metal backend selector, still runs batch embeddings concurrently through one node-llama-cpp context, and does not yet provide official Apple Silicon Metal-safe guidance or a crash-isolation boundary. The keepmyclaw comments are useful backup/recovery context, but they do not resolve the OpenClaw reliability/configuration request.

## What This Changes

The issue requests official handling for Apple Silicon local GGUF/Metal embedding crashes, including CPU/GPU guidance, safer gateway failure boundaries, and serialized local embedding behavior.

## Best Possible Solution

Keep this as the canonical Apple Silicon local-embedding stability tracker until OpenClaw has a documented macOS support matrix, a validated local embedding backend selector or CPU-only mode, bounded or serialized built-in GGUF embedding calls, and a gateway-safe fallback or isolation plan for native embedding crashes.

## Reproduction Assessment

Yes. The issue and follow-up comments provide a high-confidence reproduction shape using Apple Silicon, local GGUF embeddings, node-llama-cpp/ggml-metal, restart/shutdown or QMD semantic cleanup, and SIGABRT/non-zero exit behavior; current source inspection confirms the requested controls are still missing.

## Solution Assessment

No current complete solution is present on main. #60347 may be a useful partial path for backend selection, but the maintainable fix needs coordinated config/schema, docs, node-llama-cpp backend validation, local batching behavior, and crash-boundary design.
