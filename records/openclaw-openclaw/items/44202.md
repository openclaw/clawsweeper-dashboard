## Summary

Keep open. Current main still lacks a supported Apple Silicon Metal/CPU backend selector, still runs built-in local embedding batches concurrently, and only partially mitigates QMD embedding failures with timers/locks/backoff rather than resolving the gateway crash-boundary and guidance requested here.

## What This Changes

The issue asks for official handling of Apple Silicon local-memory embedding crashes in node-llama-cpp/ggml-metal, including safer backend selection, batching, documentation, and gateway isolation guidance.

## Best Possible Solution

Keep this as the canonical Apple Silicon local-embedding stability tracker until OpenClaw has documented macOS guidance, a supported backend selector or CPU-only path, safe local batching semantics, and real Apple Silicon proof for index/search plus gateway restart/shutdown.

Yes, as source-reproducible rather than locally executed: configure local GGUF embeddings on Apple Silicon/Metal, then index/search or restart/shutdown. The issue and comments provide concrete crash logs, and current main still lacks backend controls, cleanup, or isolation that would make those paths obsolete.

## Solution Assessment

No complete solution is present on current main. The best path is maintainer-approved backend/config/docs work plus a crash-boundary decision, backed by real Apple Silicon proof rather than another narrow workaround.
