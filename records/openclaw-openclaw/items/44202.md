## Summary

Keep open. Current main still leaves the Apple Silicon local-embedding stability and guidance problem unresolved: local node-llama-cpp runs in-process without backend selection or native cleanup, batch embedding remains concurrent, docs lack Metal/CPU guidance, and the related fix/config PRs are still open or closed unmerged.

## What This Changes

The issue asks for a safer supported path for Apple Silicon local memory embeddings after ggml-metal/node-llama-cpp crashes during restart, shutdown, or QMD embedding/search cleanup.

## Best Possible Solution

Keep this as the canonical Apple Silicon local-embedding stability tracker until OpenClaw has documented macOS guidance, a validated backend selector or CPU-only mode, safe local batching semantics, and a native-runtime cleanup/isolation plan with macOS proof.

Yes, as source-reproducible rather than locally executed: configure local GGUF embeddings on Apple Silicon/Metal, then index/search or restart/shutdown. The issue and comments provide concrete crash logs, and current main still lacks backend controls, cleanup, or isolation that would make those paths obsolete.

## Solution Assessment

No complete solution is present on current main. The best path is a maintainer-approved split or refresh of the settings/guidance work and the crash-boundary work, backed by real Apple Silicon proof.
