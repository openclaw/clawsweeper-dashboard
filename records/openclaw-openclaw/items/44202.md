## Summary

Keep this open. Current main still leaves the built-in local GGUF embedding path without a CPU/GPU backend selector, still runs local batch embeddings concurrently, and only has partial QMD-side mitigations; the linked keepmyclaw backup workaround and QMD comments are useful recovery context but do not resolve the OpenClaw local embedding stability request.

## What This Changes

The issue asks OpenClaw to address Apple Silicon local embedding crashes in the node-llama-cpp/ggml-metal path, clarify supported CPU/GPU guidance, and avoid gateway-level instability.

## Best Possible Solution

Use this as the canonical Apple Silicon local-embedding stability tracker until OpenClaw has a documented macOS support matrix, a validated backend selector or CPU-only mode, bounded local embedding concurrency, and a gateway-safe fallback or isolation plan for native aborts.

Yes, with source-backed limits rather than a live local repro. The issue and comments give a high-confidence Apple Silicon GGUF/Metal reproduction shape, and current main still lacks the backend controls and safe batching requested for that path.

## Solution Assessment
