## Summary

Keep this open. Current main still lacks a supported Apple Silicon Metal/CPU backend selector for built-in local embeddings, still runs local GGUF batch embeddings concurrently through one node-llama-cpp context, and the docs do not yet give official Metal-safe guidance; the QMD and keepmyclaw workaround comments are useful recovery context but do not resolve the OpenClaw reliability request.

## What This Changes

The issue requests official handling for Apple Silicon local GGUF/Metal embedding crashes, including CPU/GPU guidance, safer gateway failure boundaries, and serialized local embedding behavior.

## Best Possible Solution

Keep this as the canonical Apple Silicon local-embedding stability tracker until OpenClaw has a documented macOS support matrix, a validated backend selector or CPU-only mode, bounded or serialized built-in GGUF embedding calls, and a gateway-safe fallback or isolation plan for native crashes.

## Reproduction Assessment

Yes. The issue and follow-up comments give a high-confidence reproduction shape using Apple Silicon, local GGUF embeddings, node-llama-cpp/ggml-metal, restart/shutdown or QMD cleanup, and SIGABRT/non-zero exit behavior; current source inspection confirms the requested controls are still missing.

## Solution Assessment

No complete solution is present on current main. #60347 is a useful partial path for backend selection, but the maintainable fix also needs docs/support policy, local batching semantics, and crash-boundary design.
