## Summary

Keep open. Current main still leaves the reported Apple Silicon local-embedding crash/guidance problem unresolved: the built-in local provider has no CPU/GPU backend selector, local batch embedding remains concurrent, docs do not give Metal/CPU guidance, and the broader settings PR is still open with review blockers.

## What This Changes

Reports that local GGUF memory embeddings on macOS Apple Silicon can crash the gateway through node-llama-cpp/ggml-metal and asks for supported CPU/GPU guidance, safer failure behavior, and batching/config controls.

## Best Possible Solution

Keep this as the canonical Apple Silicon local-embedding stability tracker until OpenClaw has documented macOS support guidance, a validated backend selector or CPU-only mode, safe local batching semantics, and a native-runtime isolation or fallback plan with macOS proof.

## Reproduction Assessment

Yes, with source-backed limits rather than a live Apple Silicon run. The issue and comments give concrete local GGUF/Metal restart or teardown crash paths, and current main still lacks controls or isolation that would make those paths obsolete.

## Solution Assessment
