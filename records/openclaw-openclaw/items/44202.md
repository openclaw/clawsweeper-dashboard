## Summary

Keep open. Current main and the latest release still run the builtin local GGUF embedding provider in the gateway process without an Apple Silicon CPU/Metal policy or worker isolation, and an open draft PR already references this issue as the active fix path.

## What This Changes

Request crash containment and official Apple Silicon CPU/Metal guidance for local memory embeddings and QMD when node-llama-cpp/ggml-metal teardown crashes.

## Best Possible Solution

Land or iterate https://github.com/openclaw/openclaw/pull/85348 or a successor that isolates local embeddings from the gateway, defines CPU/Metal policy, preserves safe fallback/search behavior, documents Apple Silicon guidance, and proves it on real Apple Silicon.
