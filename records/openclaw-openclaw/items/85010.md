## Summary

Current `main` already fixes the update/doctor path that could let stale Matrix plugin metadata write flat DM allowlist keys, and Matrix is now consistently declared as nested-only in source and metadata.

## What This Changes

The issue reports that `openclaw update` on release `2026.5.19` can rewrite Matrix DM config into flat `dmPolicy`/`allowFrom` keys that the installed Matrix plugin schema rejects.

## Best Possible Solution

Keep the current main fix and include it in the next release so update/doctor repair cannot use stale Matrix metadata or write flat Matrix DM fields again.

## Reproduction Assessment

No current-main failing repro was run in this read-only review. The reported release failure is source-traceable through update's doctor/convergence repair path, and current main now prunes stale plugin records before those repairs can use old Matrix metadata.

## Solution Assessment

Yes. The current fix addresses the safer boundary by removing stale bundled-plugin shadows from update and doctor registry state while preserving Matrix's nested-only schema and repair contract.
