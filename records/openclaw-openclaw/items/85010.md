## Summary

Current main already addresses the update-path failure: Matrix remains nested-only for DM config, and post-core update/doctor repair now prunes stale compiled bundled-plugin path records so current plugin schemas win after upgrade. The fix is current-main-only and is not included in v2026.5.19.

## What This Changes

The issue reports that `openclaw update` on v2026.5.19 can rewrite Matrix DM allowlist settings into flat `channels.matrix.dmPolicy` and `channels.matrix.allowFrom` keys that the Matrix plugin rejects.

## Best Possible Solution

Keep the current-main update/doctor pruning fix and ship it in the next release so Matrix keeps its nested DM config after upgrades.

## Reproduction Assessment

Yes, at source level rather than through a live packaged rerun. The release code lacks stale-shadow pruning, the issue gives concrete before/after config, and the merged fix PR proved stale compiled plugin output could shadow current bundled schemas until pruned.

## Solution Assessment

Yes. Current main fixes the maintainable boundary by preserving Matrix's nested config contract and removing stale compiled bundled-plugin shadows during update/doctor repair instead of adding another Matrix-specific flat-key fallback.
