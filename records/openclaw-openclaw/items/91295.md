## Summary

Keep open: current main still has only string-based output replacements and no backend-owned post-parse assistant-text transform hook, while the request is a plausible public plugin SDK extension that needs maintainer API direction before implementation.

## What This Changes

The issue requests an optional `CliBackendPlugin.transformAssistantText` hook so CLI backends can rewrite parsed assistant text before static output replacements run.

## Best Possible Solution

Decide the public SDK shape first, then implement the accepted narrow hook or replacement-callback design with runner wiring, documentation, and focused tests that preserve existing `textTransforms.output` ordering.

## Reproduction Assessment

Yes for the source-level gap: current main lacks the requested hook and only supports string output replacements. A live end-to-end repro is not available on current main because the consuming Antigravity backend is still an open pull request.

## Solution Assessment

Unclear until maintainers choose the SDK contract. The proposed backend-owned hook is plausible and consistent with existing provider-owned hooks, but expanding replacement callbacks is a viable API alternative that should be decided intentionally.
