## Summary

Keep open: current main and the latest release still expose only static string output replacements, while the requested backend-owned post-parse assistant-text hook is a plausible public plugin SDK extension that needs maintainer API direction before implementation.

## What This Changes

Request an optional `CliBackendPlugin` hook to rewrite parsed assistant text before static output replacements so text-only CLI backends can do structural presentation rewrites.

## Best Possible Solution

Approve the plugin SDK shape first, then implement the chosen narrow hook or replacement-callback design with runner wiring, docs, API-baseline coverage, and tests that lock ordering relative to `textTransforms.output`.

## Reproduction Assessment

Yes for the source-level gap: current main has no post-parse assistant-text hook and only supports static string output replacements. A live end-to-end repro is not available on current main because the motivating Antigravity backend is still open PR work.

## Solution Assessment

Unclear until maintainers choose the public SDK contract. The proposed backend-owned hook is plausible, but a richer replacement callback or a narrower plugin-local workaround remains a product/API decision.
