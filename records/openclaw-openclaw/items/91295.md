## Summary

Keep open: current main still lacks a backend-owned post-parse assistant-text transform hook, and the existing output transform path is limited to static replacements. The request fits OpenClaw's plugin API direction, but it needs maintainer SDK-shape approval before implementation.

## What This Changes

Add an optional `CliBackendPlugin.transformAssistantText` hook for full-text post-parse assistant text rewrites before static output replacements.

## Best Possible Solution

Approve the SDK contract or choose the replacement-callback alternative, then implement the accepted design through the CLI backend type, resolved backend propagation, runner wiring, docs, API baseline, and focused tests.

## Reproduction Assessment

Yes for source-level reproduction: current main has only string-based output replacements and no post-parse assistant-text hook. No live current-main Antigravity repro is available because the consuming backend is still an open pull request.

## Solution Assessment

Unclear until maintainers choose the public SDK contract. The backend-owned hook is plausible and aligned with existing CLI backend hooks, but callback replacements are a viable alternative that should be accepted or rejected intentionally.
