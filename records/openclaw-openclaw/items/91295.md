## Summary

Keep open: current main and v2026.6.8 still only expose string-based CLI backend output replacements, while this asks for a new public plugin SDK hook that needs maintainer API direction before implementation.

## What This Changes

Requested addition of an optional `CliBackendPlugin.transformAssistantText` hook for structural post-parse assistant text rewrites in CLI backends.

## Best Possible Solution

Choose the SDK contract first, then implement the accepted narrow hook or callback-replacement design with runner wiring, public docs, API baseline updates, and focused tests that preserve existing static replacement behavior.

## Reproduction Assessment

Yes, at source level: current main lacks `transformAssistantText`, and the only CLI backend output rewrite contract is string-based `textTransforms.output`. No live current-main repro exists because the motivating Antigravity backend remains in open PRs.

## Solution Assessment

Unclear until maintainers choose the public SDK contract. The dedicated backend-owned hook is plausible, but callback-capable replacements or an Antigravity-owned workaround should be compared before codifying a new plugin API.
