## Summary

Keep open: current main and the latest release still expose only string-based CLI output replacements, with no backend-owned post-parse assistant-text transform hook. The request is a plausible plugin SDK extension, but it should wait for maintainer approval of the public API shape and ordering.

## What This Changes

The issue requests an optional `CliBackendPlugin.transformAssistantText` hook that lets CLI backends rewrite parsed assistant text before existing output replacements run.

## Best Possible Solution

Approve the public SDK shape first, then implement the accepted narrow hook or replacement-callback design with runner wiring, docs, API/baseline coverage, and focused tests that preserve existing `textTransforms.output` ordering.

## Reproduction Assessment

Yes for the source-level gap: current main still has only string output replacements and no backend-owned post-parse assistant text transform hook. A live end-to-end repro is not available on current main because the consuming Antigravity backend remains an open PR.

## Solution Assessment

Unclear until maintainers choose the SDK contract. The proposed backend-owned hook is plausible and consistent with existing backend hooks, but a callback form for `PluginTextReplacement.to` is still a viable alternate public API.
