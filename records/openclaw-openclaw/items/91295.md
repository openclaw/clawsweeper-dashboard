## Summary

Current main and the latest release still only support static output replacements, so the requested plugin SDK hook remains a real missing extension point. Keep this open for maintainer API/product direction rather than auto-queueing a fix PR.

## What This Changes

The issue requests an optional `CliBackendPlugin.transformAssistantText` hook so CLI backend plugins can rewrite parsed assistant text with full-text logic before static output replacements run.

## Best Possible Solution

Keep this as the canonical API-design thread for post-parse assistant-text transformation and have maintainers decide whether to add this hook, extend replacements, or defer to a broader parser/event contract.

## Reproduction Assessment

Yes at source level: current main only supports string output replacements after CLI parsing, so the proposed full-text rewrite hook is not available. I did not run a live Antigravity session because the missing SDK surface is directly visible in source and docs.

## Solution Assessment

Unclear as an implementation choice: the proposed hook is plausible and aligned with plugin ownership, but maintainers should compare it with extending replacement callbacks and the broader JSONL parser hook before adding a public SDK contract.
