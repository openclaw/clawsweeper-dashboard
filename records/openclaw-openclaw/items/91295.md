## Summary

Keep open: current main and the latest release still expose only string-based CLI output replacements and no backend-owned post-parse assistant-text transform hook; the request fits OpenClaw's plugin API direction but needs maintainer API approval before implementation.

## What This Changes

Add an optional `CliBackendPlugin` hook that rewrites parsed assistant text before static output replacements for text-output CLI backends.

## Best Possible Solution

Approve the SDK contract first, then implement the accepted additive hook with resolved-backend propagation, docs/API baseline updates, and focused tests proving ordering before `textTransforms.output`.

## Reproduction Assessment

Yes for the source-level gap: current main and `v2026.6.6` lack the requested hook and only support string output replacements. No live runtime reproduction exists on current main because the consuming Antigravity backend is still open work.

## Solution Assessment

Unclear until maintainers choose the public SDK contract. The proposed backend-owned hook is plausible and aligned with existing backend hooks, but replacement-callback semantics and streaming/commentary ordering should be decided intentionally.
