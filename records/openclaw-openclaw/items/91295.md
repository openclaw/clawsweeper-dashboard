## Summary

Keep open: current main and v2026.6.9 still expose only string-based CLI backend output replacements, while this issue requests a new public plugin SDK hook that needs maintainer API direction before implementation.

## What This Changes

Add an optional `CliBackendPlugin.transformAssistantText` hook so CLI backends can rewrite parsed assistant text with full-text logic before static output replacements run.

## Best Possible Solution

Choose the stable SDK contract first, then implement the accepted narrow assistant-text hook or callback-replacement design with runner wiring, docs, API baseline updates, and focused tests that preserve existing static replacement behavior.

## Reproduction Assessment

Yes, at source level: current main exposes only string-based `textTransforms.output`, and the CLI runner applies static replacements after parsing. No live current-main Antigravity reproduction is available because the motivating backend remains in open PRs.

## Solution Assessment

Unclear until maintainers choose the public SDK contract. The dedicated backend-owned hook is plausible, but callback-capable replacements and the broader JSONL event-hook discussion should be compared before adding a new public API.
