## Summary

Keep open: current main and the latest release still expose only string-based CLI output replacements, with no backend-owned `transformAssistantText` hook. The request is aligned with plugin API expansion, but it is a public SDK/API decision for maintainers before implementation; the related items at https://github.com/openclaw/openclaw/issues/91282 and https://github.com/openclaw/openclaw/pull/90975 remain open and do not supersede this issue.

## What This Changes

Requests an optional `CliBackendPlugin.transformAssistantText` hook that can rewrite parsed assistant text before static CLI output replacements run.

## Best Possible Solution

Approve a narrow public SDK design for structural parsed-text rewrites, wire it through the resolved CLI backend and runner with focused tests/docs, then let the Antigravity backend consume that generic hook without core provider-specific logic.

## Reproduction Assessment

Yes, source inspection is enough to reproduce the capability gap: current main has only static output replacements and no backend-owned full assistant-text transform hook. I did not run a live Antigravity scenario because the missing SDK hook is provable from the type and runner path.

## Solution Assessment

Unclear until maintainer API review. The proposed hook is plausible and fits the existing provider-owned hook pattern, but it changes the public plugin SDK and should be approved before an implementation PR is queued.
