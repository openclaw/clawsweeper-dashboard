## Summary

Keep open: current main still has only string-based CLI backend output replacements and no backend-owned post-parse assistant-text hook, while the request is a plausible public plugin SDK extension that needs maintainer API direction before implementation.

## What This Changes

The issue requests an optional `CliBackendPlugin.transformAssistantText` hook so CLI backends can rewrite parsed assistant text with full-text logic before static output replacements run.

## Best Possible Solution

Choose the SDK contract first, then implement the accepted narrow hook or replacement-callback design with runner wiring, public docs, API-surface checks, and focused tests that preserve existing `textTransforms.output` behavior.

## Reproduction Assessment

Yes for the source-level gap: current main lacks the requested hook and only supports string output replacements. No live end-to-end reproduction exists on current main because the consuming Antigravity backend is still an open pull request.

## Solution Assessment

Unclear until maintainers choose the public SDK contract. The dedicated backend-owned hook is plausible, but a callback-capable replacement API or a narrower plugin-owned workaround should be compared intentionally before codifying it.
