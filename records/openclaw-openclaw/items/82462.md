## Summary

Keep open. Current main still has tight OpenAI-compatible stream-processing loops without an in-loop abort check or cooperative macrotask yield, and the merged split-frame sanitizer fix only covers one related stalled-stream failure mode, not the broader gateway event-loop starvation reported here.

## What This Changes

The issue reports OpenAI-compatible model-fetch streaming blocking the gateway event loop during large contexts, causing SocketError, delayed `/stop`, channel timeouts, and liveness warnings.

## Best Possible Solution

Profile and reproduce the current-main large-stream path, then add bounded cooperative yielding and in-loop abort checks at the OpenAI-compatible stream boundary with regressions proving timers and aborts are serviced during bursty streams.

## Reproduction Assessment

No high-confidence live reproduction was established in this read-only review. The reporter supplied concrete logs and current source shows plausible tight stream loops, but I did not reproduce the opencode-go large-context SocketError against current main.

## Solution Assessment

Unclear. Per-chunk or batched cooperative yielding plus abort checks looks like the narrowest first fix, but maintainers should profile current main before choosing between stream-loop changes, SDK/fetch wrapping, or worker isolation.
