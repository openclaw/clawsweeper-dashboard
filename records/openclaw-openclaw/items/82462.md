## Summary

Keep open: the report is credible and not fully covered by the merged split-frame sanitizer work or the broader gateway-stall umbrella; current main still has OpenAI-compatible stream-processing loops without cooperative yielding or in-loop abort checks.

## What This Changes

The issue reports large OpenAI-compatible model streams blocking the gateway event loop for 25-30 seconds during model-fetch and stream processing.

## Best Possible Solution

Keep this as a narrow model-fetch stream-fairness bug and add budgeted cooperative yielding plus abort checks in the OpenAI-compatible stream-processing paths, with synthetic high-volume stream coverage and real gateway proof before landing.

## Reproduction Assessment

No high-confidence live current-main reproduction was established in this review. Source inspection does show the reported path is plausible: both OpenAI-compatible stream processors can process event bursts without yielding, and completions abort is checked only after stream processing finishes.

## Solution Assessment

Unclear whether the suggested per-chunk yield is the best exact fix. A budgeted cooperative yield and in-loop abort check inside the existing stream processors looks narrower and safer than worker-thread isolation or a new config knob.
