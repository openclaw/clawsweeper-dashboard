## Summary

Keep open. The report is fresh, current main still processes OpenAI-compatible streaming data on the gateway event loop without an explicit cooperative yield/abort check in the completions processing loop, and the related SSE split-frame bug is not enough to prove this v2026.5.12 opencode-go/model-fetch starvation report is fixed or fully duplicated.

## What This Changes

The issue reports event-loop starvation while processing large OpenAI-compatible streaming model responses, causing SocketError failures and delayed gateway work during inference.

## Best Possible Solution

Profile or minimally reproduce the v2026.5.12 opencode-go streaming path, then add the narrowest cooperative scheduling and abort handling needed in the OpenAI-compatible transport loops without changing provider policy or adding a new config knob.

## Reproduction Assessment

No high-confidence live current-main reproduction was established in this review. Source inspection confirms the relevant OpenAI-compatible stream-processing path and missing in-loop cooperative checks, while the issue provides concrete v2026.5.12 logs that still need a live/profiled repro.

## Solution Assessment

Unclear. Adding yield and abort checks between stream events is a plausible narrow fix, but maintainers should first distinguish pre-response fetch wait, SDK SSE parsing, sanitizer behavior, and OpenClaw event processing so the patch targets the actual hot path.
