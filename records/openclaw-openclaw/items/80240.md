Overall confidence: 0.94

Full review comments:

- **[P1] Restrict public Container App CORS:** `infra/main.bicep:720-727`
  - body: The deployed Container App ingress allows any browser origin while exposing admin and Harvester mutation surfaces. Use explicit expected origins and avoid production insecure ingress unless a documented local-only path requires it.
  - confidence: 0.94
- **[P1] Keep PostgreSQL certificate verification enabled:** `harvester-server.mjs:400`
  - body: Every non-disable SSL mode becomes `{ rejectUnauthorized: false }`, so the default cloud database path skips server certificate validation. Keep verification on by default and reserve insecure TLS for an explicit local/debug setting.
  - confidence: 0.95
- **[P1] Sanitize the deploy helper environment before spawning:** `scripts/azd-deploy-openclaw-from-package.mjs:147-149`
  - body: The helper forwards raw `process.env` plus overrides into `az`, `docker`, and `azd`. This child-process boundary should use the existing host exec sanitizer with explicit safe overrides to avoid inherited environment injection.
  - confidence: 0.9
- **[P2] Use reproducible base images for AlpaCore builds:** `infra/images/alpa-engine/Dockerfile:1`
  - body: These Dockerfiles depend on local unqualified `latest` images, so clean CI or ACR builds will fail unless those tags happen to exist on the builder. Pin registry images, build from source, or make the base image an explicit input.
  - confidence: 0.91
- **[P2] Point Cursor MCP at the serving AlpaCore port:** `.cursor/mcp.json:9`
  - body: Cursor is configured for port 5143, but the VS Code config and bridge default use port 8080. Cursor users will connect to the wrong endpoint unless the port is aligned or the difference is documented and backed by the service config.
  - confidence: 0.9

Status: needs_attention

Summary: The diff introduces concrete security and supply-chain concerns in public ingress, database TLS, child-process env handling, and image provenance.

- **[high] Wildcard CORS on public mutation surfaces:** `infra/main.bicep:831`
