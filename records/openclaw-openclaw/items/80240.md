## Summary

Keep open. This PR is not merge-ready because it mixes a narrow gateway/devices fix with a new AlpaCore/Harvester/Azure surface and introduces blocking security, build, and runtime defects; the supplied live proof only covers one Harvester endpoint slice.

## What This Changes

The PR adds a local AlpaCore MCP bridge/editor configs, Control UI origin persistence/merge behavior, devices fallback test coverage, Azure Container Apps deployment assets, and a new Harvester backend/control UI with scripts and Postman assets.

## Best Possible Solution

Split the narrow gateway/devices fixes into focused, tested PRs and handle the AlpaCore/Harvester/Azure work through an explicit product/plugin architecture decision with auth, CORS, image provenance, and deployment hardening.

## Reproduction Assessment

Yes for the blocking PR defects: source inspection shows the missing-scope fallback expectation has no implementation, public wildcard CORS protects unauthenticated mutation endpoints, pg TLS verification is disabled by default, and azd images/env handling are not reproducible or hardened. I did not run tests because this review is required to keep the checkout read-only.

## Solution Assessment

No. The requested direction may contain useful pieces, but the current PR is too broad and leaves several security/build/runtime blockers; the maintainable path is to split the core fixes and separately design the optional AlpaCore/Harvester deployment surface.

  - body: `sslMode` defaults to `require`, but every non-`disable` value becomes `{ rejectUnauthorized: false }`.   - confidence: 0.94

  - body: The new helper forwards raw `process.env` into `az`, `docker`, and `azd`. ## Security Review

Status: needs_attention

Summary: The diff introduces concrete security and supply-chain concerns in public ingress, database TLS, child-process env handling, and image provenance.

Concerns:

- **[high] Wildcard CORS on public admin and mutation surfaces:** `infra/main.bicep:743`
  - body: The new ACA ingress allows any origin for the OpenClaw gateway and Harvester, while the docs identify Control UI as an admin surface and Harvester adds unauthenticated mutation endpoints.
  - confidence: 0.93
- **[high] PostgreSQL TLS does not verify certificates:** `harvester-server.mjs:294`
  - body: The Harvester database config defaults to SSL but sets `rejectUnauthorized: false`, allowing a MITM with any certificate instead of validating the server certificate.
  - confidence: 0.94

  - body: The helper passes raw environment variables into spawned CLIs, bypassing the repository sanitizer used to strip execution-affecting env keys; GitHub Advanced Security flagged the same path.

  - body: The new infra uses local `:latest` base images and a floating `open-webui:main` reference, which makes builds non-reproducible and weakens supply-chain review.
