Full review comments:

- **[P1] Restrict Harvester CORS before exposing mutation routes:** `infra/main.bicep:831-833`
  - body: The Harvester Container App allows `allowedOrigins: ['*']` while the new server exposes unauthenticated POST routes for target cleanup, target creation, and worker scaling. Any browser origin can drive those mutations against a deployed app; restrict CORS to expected Control UI origins and add auth for non-local deployments.
  - confidence: 0.94
- **[P1] Keep PostgreSQL certificate verification enabled:** `harvester-server.mjs:400`
  - body: `sslMode` defaults to `require`, but every non-`disable` value becomes `{ rejectUnauthorized: false }`. That encrypts the database connection while skipping server certificate validation by default; verification should stay on unless an explicit insecure local option is selected.
  - confidence: 0.95
- **[P1] Sanitize the deploy helper environment before spawning:** `scripts/azd-deploy-openclaw-from-package.mjs:147-150`
  - body: The helper forwards raw `process.env` plus overrides into `az`, `docker`, and `azd`. OpenClaw already has `sanitizeHostExecEnv` for this child-process boundary, so build the env through that sanitizer with explicit safe overrides.
  - confidence: 0.91
- **[P2] Use reproducible base images for AlpaCore services:** `infra/images/alpa-engine/Dockerfile:1`
  - body: The AlpaCore service Dockerfiles use local unqualified `latest` base images, so clean CI or ACR builds will fail unless those images happen to exist on the builder. Use pinned registry images, build from source in this repo, or make the base image an explicit build arg.
  - confidence: 0.91
- **[P2] Point Cursor MCP at the serving AlpaCore port:** `.cursor/mcp.json:9`
  - body: Cursor points `ALPACORE_URL` at port 5143, but the bridge default and VS Code config use port 8080, and the compose service configures the Harvester process on 8080. Cursor users will connect to the wrong listener unless these are aligned or the different port is documented and served.
  - confidence: 0.9
- **[P2] Do not auto-start MCP from repo settings:** `.vscode/settings.json:21`
  - body: Committing `chat.mcp.autoStart: true` makes VS Code users spawn the MCP server when opening the workspace. Keep MCP startup as opt-in user setup instead of a repository-wide setting that launches local code unexpectedly.
  - confidence: 0.86
- **[P3] Remove the user-specific checkout path:** `start-harvester.ps1:9`
  - body: The helper hard-codes `c:\Users\finnu\Documents\GitHub\openclaw`, so it fails for other contributors and CI. Resolve the repo root from `$PSScriptRoot` or use repo-relative instructions.
  - confidence: 0.97

## Security Review

Status: needs_attention

Summary: The diff introduces concrete security and supply-chain concerns in public ingress, database TLS, child-process env handling, image provenance, and editor auto-start behavior.

Concerns:

- **[high] Wildcard CORS on public mutation surfaces:** `infra/main.bicep:831`
  - body: The Harvester Container App permits any origin while the server exposes unauthenticated state-mutating endpoints, creating cross-origin abuse risk for deployed instances.
  - confidence: 0.94
- **[high] PostgreSQL TLS skips certificate verification:** `harvester-server.mjs:400`
  - body: The Harvester database connection treats SSL as required but sets `rejectUnauthorized: false`, so the default cloud database path does not verify the server certificate.
  - confidence: 0.95
- **[high] Deploy helper forwards unsanitized environment:** `scripts/azd-deploy-openclaw-from-package.mjs:147`
  - body: The new helper passes raw process environment values to external tools instead of using the existing host exec sanitizer, leaving workspace or inherited env injection risk at a command-execution boundary.
  - confidence: 0.91
- **[medium] Unqualified latest images weaken build provenance:** `infra/images/alpa-engine/Dockerfile:1`
  - body: The added AlpaCore Dockerfiles depend on local `latest` image tags rather than pinned or source-built inputs, making the deploy path non-reproducible and vulnerable to accidental or untrusted local image substitution.
  - confidence: 0.9
- **[medium] Workspace MCP auto-start launches local code unexpectedly:** `.vscode/settings.json:21`
  - body: The repo-level VS Code setting enables MCP auto-start for every contributor opening the workspace, which can launch the new Node bridge without an explicit user opt-in.
  - confidence: 0.86
