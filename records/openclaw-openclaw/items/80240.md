## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Restrict Harvester ingress before exposing mutations:** `infra/main.bicep:831-832`
  - body: The Harvester Container App is external and allows any browser origin, while the server exposes unauthenticated POST routes for cleanup, target creation, and worker scaling. Restrict CORS to expected Control UI origins and require auth for deployed mutation routes.
  - confidence: 0.94
- **[P1] Keep PostgreSQL certificate verification enabled:** `harvester-server.mjs:400`
  - body: Every non-disable SSL mode becomes `{ rejectUnauthorized: false }`, so the default cloud database path skips server certificate validation. Keep verification on by default and reserve insecure TLS for an explicit local/debug setting.
  - confidence: 0.95
- **[P1] Sanitize the deploy helper environment before spawning:** `scripts/azd-deploy-openclaw-from-package.mjs:147-149`
  - body: The helper forwards raw `process.env` plus overrides into `az`, `docker`, and `azd`. Use the existing host exec sanitizer with explicit safe overrides to avoid inherited environment injection at this child-process boundary.
  - confidence: 0.9
- **[P2] Use reproducible base images for AlpaCore builds:** `infra/images/alpa-engine/Dockerfile:1`
  - body: The AlpaCore Dockerfiles depend on local unqualified `latest` images, so clean CI or ACR builds will fail unless those tags happen to exist on the builder. Pin registry images, build from source, or make the base image an explicit input.
  - confidence: 0.91
- **[P2] Point Cursor MCP at the serving AlpaCore port:** `.cursor/mcp.json:9`
  - body: Cursor is configured for port 5143, but the VS Code config and bridge default use port 8080. Cursor users will connect to the wrong endpoint unless the port is aligned or the difference is documented and backed by the service config.
  - confidence: 0.9

## Security Review

Status: needs_attention

Summary: The diff introduces concrete security and supply-chain blockers in public ingress, database TLS, child-process env handling, and image provenance.

- **[high] Wildcard CORS on public mutation surfaces:** `infra/main.bicep:831`

  - body: The added AlpaCore Dockerfiles depend on local `latest` tags rather than pinned or source-built inputs, making the deploy path non-reproducible and vulnerable to accidental or untrusted local image substitution.
