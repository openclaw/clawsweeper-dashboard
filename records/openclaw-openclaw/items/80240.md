## Summary

Keep open. The force-pushed head fixed some earlier package/Compose regressions, but it still contains a fatal merge-conflict marker, failed required checks, multiple security-sensitive deployment defects, and incomplete real behavior proof for most changed surfaces.

## What This Changes

Adds editor MCP bridge/configuration, devices CLI fallback coverage, Control UI origin persistence/env merging, Azure Container Apps deployment assets, a harvester backend/UI, helper scripts, docs, and Postman assets.

## Best Possible Solution

Split the work: land only narrow, tested core fixes for devices/origin behavior, and move or rework the AlpaCore/harvester/ACA integration as a separately reviewed surface with auth, explicit CORS, reproducible images, and complete real behavior proof.

## Reproduction Assessment

Yes for the review blockers: source inspection and CI logs show unresolved conflict markers, and the security defects are tied to concrete added lines. This is PR-head reproduction rather than a current-main product bug.

## Solution Assessment

No. Bundling editor MCP, gateway fallback, persisted config, Azure infra, and a new harvester app into one branch is not the narrowest maintainable fix, especially with fatal syntax and security blockers present.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.97

Full review comments:

- **[P1] Remove the unresolved conflict markers:** `src/cli/devices-cli.ts:161-163`
  - body: The PR head includes literal `=======` and `>>>>>>>` markers inside `resolveLocalPairingFallback`, and check-prod-types reports TS1185 at these lines. The CLI package cannot compile until this conflict is resolved.
  - confidence: 0.99
- **[P1] Sanitize the deploy helper process environment:** `scripts/azd-deploy-openclaw-from-package.mjs:147-150`
  - body: `runStep` passes a merged `process.env` directly to spawned `az`, `docker`, and `azd` commands. This new command-execution surface is already flagged by Opengrep for the repo's unsanitized env-merge rule; use the repo host-exec sanitizer or an equivalent allowlist before spawning.
  - confidence: 0.91
- **[P1] Validate PostgreSQL certificates by default:** `harvester-server.mjs:294`
  - body: Every non-`disable` database SSL mode becomes `{ rejectUnauthorized: false }`, so production TLS connections do not verify the PostgreSQL server certificate. Keep verification on by default and reserve insecure TLS for an explicit local/debug override.
  - confidence: 0.93
- **[P1] Restrict public Container App CORS:** `infra/main.bicep:632-638`
  - body: The OpenClaw Container App is external, allows insecure HTTP, and uses wildcard CORS. That exposes the gateway admin surface to arbitrary browser origins and undermines the Control UI origin allowlist this PR is trying to preserve.
  - confidence: 0.92
- **[P1] Protect public harvester mutation endpoints:** `harvester-server.mjs:2197-2252`
  - body: The new harvester service exposes unauthenticated POST endpoints for queue cleanup, adding arbitrary targets, and scaling workers. Since the Bicep makes the harvester app externally reachable, anyone who can reach the endpoint can mutate runtime state.
  - confidence: 0.9
- **[P2] Use reproducible base images:** `infra/images/alpa-engine/Dockerfile:1`
  - body: The ACA image Dockerfiles depend on local, unqualified `latest` tags, so CI/ACR builds will fail unless those private local images happen to exist. Pin real registry images or build these services from source in the repo.
  - confidence: 0.91
- **[P2] Point the Cursor bridge at a listening port:** `.cursor/mcp.json:9`
  - body: Cursor config sends the bridge to `127.0.0.1:5143`, but the added harvester server listens on `CONFIG.port` and Compose sets that to 8080. The Cursor MCP bridge will fail unless something else starts on 5143.
  - confidence: 0.88
- **[P2] Avoid overwriting the root package in docs:** `HARVESTER-README.md:144-145`
  - body: The harvester README tells users to copy `harvester-package.json` over the repo-root `package.json`. Following those instructions destroys the OpenClaw workspace manifest and breaks the same package contract this PR otherwise restores.
  - confidence: 0.9
- **[P3] Resolve the repo root from the script location:** `start-harvester.ps1:9`
  - body: This helper hard-codes a contributor-specific checkout path, so it fails for every other developer and CI runner. Resolve the root from `$PSScriptRoot` before invoking Compose.
  - confidence: 0.94
