## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.95

Full review comments:

- **[P1] Restrict Harvester ingress before exposing mutations:** `infra/main.bicep:831-832`
  - body: The Harvester Container App is external and allows any browser origin, while the server exposes state-mutating POST routes for cleanup, target creation, and worker scaling without route auth. Restrict CORS to expected Control UI origins and require auth before deploying this surface.
  - confidence: 0.95
- **[P1] Keep PostgreSQL certificate verification enabled:** `harvester-server.mjs:400`
  - body: Every non-disable SSL mode becomes `{ rejectUnauthorized: false }`, so the default cloud database path encrypts traffic but does not verify the server certificate. Keep verification enabled by default and reserve insecure TLS for an explicit local/debug option.
  - confidence: 0.96
- **[P1] Sanitize the deploy helper environment before spawning:** `scripts/azd-deploy-openclaw-from-package.mjs:147-149`
  - body: The helper forwards raw `process.env` plus overrides into `az`, `docker`, and `azd`. This child-process boundary should use the existing host exec sanitizer with explicit safe overrides so inherited env injection does not cross into deployment tools.
  - confidence: 0.92
- **[P2] Do not reintroduce startup config writes:** `src/gateway/server.impl.ts:615-619`
  - body: Current main intentionally keeps startup-derived Control UI origins runtime-only after the no-automatic-config-writes policy change. Calling `replaceConfigFile` from gateway startup reintroduces implicit config mutation and conflicts with that contract.
  - confidence: 0.91
- **[P2] Use reproducible base images for AlpaCore builds:** `infra/images/alpa-engine/Dockerfile:1`
  - body: The AlpaCore Dockerfiles depend on local unqualified `latest` images, so clean CI or ACR builds will fail unless those tags happen to exist on the builder. Pin registry images, build from source, or make the base image an explicit input.
  - confidence: 0.91
- **[P2] Point Cursor MCP at the serving AlpaCore port:** `.cursor/mcp.json:9`
  - body: The PR body, VS Code config, bridge default, and compose service all center on port 8080, but Cursor is configured for 5143. Cursor users will connect to the wrong endpoint unless the port is aligned or the second port is documented and actually served.
  - confidence: 0.9
- **[P1] Do not deploy the OpenClaw gateway with wildcard insecure ingress:** `infra/main.bicep:720-727`
  - body: The ACA template exposes the OpenClaw gateway publicly with `allowInsecure: true` and wildcard CORS. That bypasses the tightened Control UI origin posture this repo already enforces; use HTTPS-only ingress and an explicit origin allowlist.
  - confidence: 0.9
