The branch adds an AlpaCore MCP bridge, local devices pairing fallback, Control UI allowed-origin startup persistence, Azure Container Apps deployment assets, and a new Harvester backend/control UI stack.

## Security Review

Status: needs_attention

Summary: The diff introduces concrete public ingress, database TLS, child-process environment, and image provenance risks that should block merge until fixed.

Concerns:

- **[high] Wildcard CORS on public mutation surfaces:** `infra/main.bicep:743`
  - body: Harvester ingress allows arbitrary origins while the service exposes mutation routes, creating cross-origin abuse risk for deployed instances.
  - confidence: 0.94
- **[high] PostgreSQL TLS skips certificate verification:** `harvester-server.mjs:294`
  - body: The Harvester database pool disables certificate verification for SSL connections, so the default cloud database path is vulnerable to MITM.
  - confidence: 0.95
- **[high] Deploy helper forwards unsanitized environment:** `scripts/azd-deploy-openclaw-from-package.mjs:147`
  - body: The new helper passes inherited environment values to external commands instead of using the existing sanitizer, leaving a code-execution boundary exposed to env injection.
  - confidence: 0.9
- **[medium] Unqualified latest images weaken build provenance:** `infra/images/alpa-engine/Dockerfile:1`
  - body: The added AlpaCore Dockerfiles depend on local latest image tags rather than pinned or source-built inputs, making the deploy path non-reproducible and vulnerable to accidental local image substitution.
  - confidence: 0.9
