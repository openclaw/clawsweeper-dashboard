## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.96

Full review comments:

- **[P1] Restore the OpenClaw root package manifest:** `package.json:2`
  - body: The PR replaces the root OpenClaw package.json with a standalone harvester manifest, removing the openclaw binary, exports, workspace scripts, published files, and dependency surface. That breaks repo commands and packaging before the new helper can be used.
  - confidence: 0.99
- **[P1] Preserve the existing Docker Compose services:** `docker-compose.yml:5`
  - body: The compose file drops openclaw-gateway and openclaw-cli entirely. Current Docker docs and ClawDock helpers invoke those service names, so Docker users would no longer be able to start or manage OpenClaw with documented commands.
  - confidence: 0.98
- **[P2] Point the UI proxy at the service hostname:** `docker-compose.yml:55`
  - body: Inside the control-ui container, VITE_API_URL=http://localhost:8080 points back at the UI container, not the alpacore service. The Vite /api proxy will fail or loop instead of reaching the backend.
  - confidence: 0.95
- **[P1] Restrict public ingress CORS:** `infra/main.bicep:632-638`
  - body: The OpenClaw Container App ingress is external, allows insecure HTTP, and allows origins '*'. That weakens the Control UI origin guard and exposes the admin gateway surface to arbitrary browser origins.
  - confidence: 0.93
- **[P1] Validate PostgreSQL TLS certificates:** `harvester-server.mjs:294`
  - body: For any sslmode other than disable, the harvester connects to PostgreSQL with rejectUnauthorized:false. That disables server certificate validation in the production Azure path.
  - confidence: 0.92
- **[P2] Keep workspace MCP startup opt-in:** `.vscode/settings.json:21`
  - body: Enabling chat.mcp.autoStart in committed workspace settings means the new MCP bridge can start automatically for contributors opening the repo. Keep this as user-local setup or explicit opt-in documentation.
  - confidence: 0.9
- **[P2] Use reproducible base images:** `infra/images/alpa-engine/Dockerfile:1`
  - body: The AlpaCore Dockerfiles depend on local, unqualified latest tags. The azd/ACR build will fail unless those private local images happen to exist, and the deployment is not reproducible.
  - confidence: 0.91
- **[P3] Remove machine-specific checkout paths:** `start-harvester.ps1:9`
  - body: This helper changes directory to a contributor-specific Windows path, so it fails outside that machine. Resolve the repository root from $PSScriptRoot before running compose commands.
  - confidence: 0.94

## Security Review

Status: needs_attention

Summary: The diff introduces concrete security and supply-chain concerns around public ingress, TLS validation, workspace auto-started code execution, and mutable/unqualified image sources.

Concerns:

- **[high] Wildcard CORS on public admin ingress:** `infra/main.bicep:632`
  - body: infra/main.bicep exposes the OpenClaw gateway with external ingress, allowInsecure:true, and allowedOrigins:['*'], weakening the Control UI origin guard for an admin surface.
  - confidence: 0.93
- **[high] Wildcard CORS on harvester mutation endpoints:** `infra/main.bicep:742`
  - body: The harvester Container App also uses wildcard CORS while the server exposes state-changing target cleanup, target enqueue, and worker scaling endpoints, increasing cross-origin abuse risk in browser-accessible deployments.
  - confidence: 0.88
- **[high] PostgreSQL certificate validation disabled:** `harvester-server.mjs:294`
  - body: harvester-server.mjs sets rejectUnauthorized:false for SSL database connections, so the production database path does not validate the server certificate by default.
  - confidence: 0.92
- **[medium] Workspace MCP bridge auto-starts:** `.vscode/settings.json:21`
  - body: .vscode/settings.json enables chat.mcp.autoStart while the PR adds workspace MCP configs that execute a local Node bridge and make network calls, which should remain explicit opt-in.
  - confidence: 0.9
- **[medium] Mutable or unqualified image sources:** `infra/images/open-webui/Dockerfile:1`
  - body: Open WebUI uses a mutable main tag and the AlpaCore service Dockerfiles use local latest tags, making deployment provenance and rebuilds unsuitable for supply-chain review.
  - confidence: 0.85
- **[low] Secret-looking Postman value:** `postman/Alphabet-Harvester.local.postman_environment.json:13`
  - body: The Postman environment commits a concrete shopify_api_secret value; even as a dummy value, it should be blank or an obvious placeholder to avoid scanner noise and unsafe copy/paste.
