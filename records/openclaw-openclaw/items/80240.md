Full review comments:

- **[P1] Keep startup origin seeding runtime-only:** `src/gateway/server.impl.ts:615-619`
  - body: This reintroduces a startup `replaceConfigFile` write for Control UI origins, which regresses current main’s explicit no-automatic-config-writes behavior from the Nix/runtime-only config work.   - confidence: 0.94
- **[P1] Restrict Harvester ingress before exposing mutations:** `infra/main.bicep:831-833`
  - body: The Harvester Container App is public and allows `*` origins while the server exposes unauthenticated POST routes for cleanup, target creation, and worker scaling. Restrict CORS to expected Control UI origins and require auth for deployed mutation routes.
  - confidence: 0.95
- **[P1] Keep PostgreSQL certificate verification enabled:** `harvester-server.mjs:400`
  - body: Every non-disable SSL mode becomes `{ rejectUnauthorized: false }`, so the default cloud database path skips server certificate validation. Verify certificates by default and reserve insecure TLS for an explicit local/debug setting.
  - confidence: 0.96
- **[P1] Sanitize the deploy helper environment before spawning:** `scripts/azd-deploy-openclaw-from-package.mjs:147-150`
  - body: The helper forwards raw `process.env` plus overrides into `az`, `docker`, and `azd`. Use the existing host exec env sanitizer with explicit safe overrides at this child-process boundary.
  - confidence: 0.93
- **[P2] Pin or build the AlpaCore base images:** `infra/images/alpa-engine/Dockerfile:1`
  - body: These Dockerfiles depend on local unqualified `latest` image tags, so clean CI or ACR builds can fail or pick up untrusted local images. Use pinned registry references, build from source in this repo, or make the base image an explicit reviewed input.
  - confidence: 0.91
- **[P2] Use locked installs for Harvester images:** `infra/images/harvester/Dockerfile:6`
  - body: The Harvester image installs semver-ranged npm dependencies without a committed lockfile, so deploys can resolve different package versions over time. Use the repo package manager/lockfile contract or commit a dedicated lock for this deploy surface.
  - confidence: 0.88
- **[P2] Point Cursor MCP at the serving AlpaCore port:** `.cursor/mcp.json:9`
  - body: Cursor uses `127.0.0.1:5143` while the bridge default and VS Code config use `127.0.0.1:8080`, so one editor integration will connect to the wrong endpoint unless the difference is intentional, documented, and backed by service config.
  - confidence: 0.9
