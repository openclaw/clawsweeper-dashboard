## Summary

Keep open: current main and v2026.6.10 still do not provide all-session Control UI transcript search with result snippets, timestamps, and click-through navigation. Existing surfaces cover loaded chat-message filtering, session metadata search, per-session usage-log search, or opt-in memory recall, so this remains a product/security-scoped feature request.

## What This Changes

Add full-text search across Control UI session history with session metadata, timestamps, and click-through navigation to matching messages.

## Best Possible Solution

Define a bounded gateway-backed transcript search contract with authorization scope, snippets, timestamps, performance limits, and deterministic navigation before implementation.

## Reproduction Assessment

Not applicable: this is a feature request, not a bug report. Source inspection gives high confidence that current main lacks the requested cross-session transcript-search result surface.

## Solution Assessment

Unclear until maintainers choose the product and API boundary. The requested UX is plausible, but the maintainable path should be a bounded gateway-backed transcript-search contract rather than expanding the current client-side active-thread filter.
