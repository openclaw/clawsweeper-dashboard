## Summary

Keep open: current main has active-chat message filtering, metadata-only session list search, and agent-facing memory_search session recall, but it does not provide the requested Control UI cross-session transcript search/results/deep-link flow.

## What This Changes

Add keyword search across all Control UI session message history, showing matching messages with session name, timestamp, and click-through navigation.

## Best Possible Solution

Design a gateway-backed transcript search API plus a Control UI results surface that respects session visibility, scales across transcript stores, and opens the target session at the matched message.

Not applicable: this is a feature request, not a broken existing behavior. Source inspection gives a high-confidence current-main check: the UI only filters loaded active-session messages and session metadata.
