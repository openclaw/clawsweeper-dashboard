## Summary

Keep open: current main only has current-thread message filtering and session-list metadata filtering, while the requested all-session message search with result navigation is a new Control UI/Gateway feature. The related broader issue was intentionally split into focused search and export issues, so this is not stale duplicate noise.

## What This Changes

The issue requests full-text search across all Control UI session messages, with result snippets showing session context and navigation to the matching message.

## Best Possible Solution

Define a bounded Gateway-backed transcript-search API and Control UI result surface that reuses existing session metadata/history projection where safe, returns session/message/timestamp snippets, and navigates to a matched message without overloading the current metadata filters.
