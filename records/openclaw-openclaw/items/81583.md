## Summary

Keep open: current main still has active-session message filtering, metadata-only session picker search, per-session Usage log filtering, and opt-in agent memory_search session indexing, but not the requested Control UI cross-session transcript full-text search with snippets, timestamps, and click-through navigation.

## What This Changes

The issue requests a Control UI chat search bar that searches message history across all sessions and navigates from results to matching transcript messages.

## Best Possible Solution

Define a bounded Gateway and Control UI transcript-search contract with authorization scope, snippets, timestamps, performance limits, and deterministic navigation before implementation.

## Reproduction Assessment

Not applicable: this is a feature request, not broken existing behavior. Source inspection gives high confidence that current main lacks the requested cross-session transcript-search result surface.

## Solution Assessment

Unclear until maintainers choose the product and API boundary. The requested UX is plausible, but the maintainable path should be a bounded gateway-backed transcript-search contract rather than expanding the current client-side active-thread filter.
