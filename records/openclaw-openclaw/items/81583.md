## Summary

Keep open: current main still lacks the requested Control UI full-text search across all session transcripts with result snippets, timestamps, and click-through navigation; existing search surfaces cover only loaded chat messages, session metadata, per-session usage logs, or opt-in memory_search recall.

## What This Changes

Add keyword search across all Control UI session message history, returning matching messages with session name, timestamp, and direct navigation to the matching thread location.

## Best Possible Solution

Define a bounded Gateway and Control UI transcript-search contract with authorization scope, snippets, timestamps, performance limits, and deterministic navigation before implementation.
