## Summary

Keep open: current main still has active-session message filtering, session-picker metadata search, and per-session Usage log filtering, but not the requested cross-session transcript full-text search with snippets, timestamps, and click-through navigation.

## What This Changes

The issue requests a Control UI search bar that searches message bodies across all session history and navigates from result rows to matching transcript messages.

## Best Possible Solution

Define a bounded Gateway and Control UI transcript-search contract with access scope, snippets, timestamps, performance limits, and deterministic navigation before implementing the UI.

## Reproduction Assessment

Not applicable: this is a feature request, not broken existing behavior. Source inspection gives high confidence that current main lacks the requested cross-session transcript-search result surface.

## Solution Assessment

Unclear until maintainers choose the product and API boundary. The requested UX is plausible, but the maintainable path should be a bounded gateway-backed transcript search contract rather than expanding the current client-side active-thread filter.
