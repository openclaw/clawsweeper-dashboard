## Summary

Keep open: current main still lacks the requested cross-session transcript keyword search with snippets, timestamps, and direct message navigation; the existing Control UI search paths cover only the active chat buffer, session metadata, or one selected Usage transcript.

## What This Changes

Requested Control UI keyword search across all session message history with session/timestamp result rows and direct navigation to matching messages.

## Best Possible Solution

Define a bounded transcript-search contract that returns message snippets with session labels and timestamps, then wire the Control UI to navigate or page to the selected message without replacing the existing active-session search or Usage transcript browser.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection confirms current main has only active-session message filtering, metadata session search, and selected-session Usage log search.

## Solution Assessment

Unclear until maintainers choose the product contract. The requested UI is reasonable, but the best implementation needs an agreed gateway/search API, result bounds, privacy/performance behavior, and navigation model.
