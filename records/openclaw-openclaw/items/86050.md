## Summary

Current main still ignores Claude CLI `assistant` records as text deltas, and the clean linked PR explicitly owns the missing parser fallback. Repository policy requires this issue to remain open until that PR is merged, replaced, or closed without a fix.

## What This Changes

Restore incremental Claude CLI assistant updates through Gateway so WebChat and TUI do not wait for the final assembled response.

## Best Possible Solution

Normalize positively identified Claude assistant partials in the existing JSONL parser, reuse the established agent-event/Gateway bridge, deduplicate cumulative text, and verify incremental WebChat and TUI output without duplicating the final response.

## Reproduction Assessment

Yes at source level: current main only emits deltas for raw Claude `stream_event` text records and ignores assistant-record text, matching the reported failure when raw events are absent. The reporter supplied repeated real logs, although another environment did not reproduce that emission pattern.

## Solution Assessment

Yes at the ownership boundary: the existing JSONL parser is the narrowest maintainable place to normalize Claude CLI record variants before the established event and Gateway bridges. The exact positive partial discriminator still needs dependency-backed live proof before the linked PR lands.
