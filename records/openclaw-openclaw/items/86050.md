- `P1`: The regression repeatedly removes progress visibility from long-running agent turns on affected installations, making active work appear hung.

## Summary

Current main still has a conditional parser gap: it relays Claude raw `stream_event` text deltas, but not cumulative `assistant` partial records, so environments emitting only the latter can remain buffered. The issue should stay open because https://github.com/openclaw/openclaw/pull/86649 explicitly owns the fix and is still open; the conflicting live results make end-to-end validation the remaining gate, not issue closure.

## What This Changes

Restore incremental Claude CLI assistant output through the Gateway so WebChat and TUI receive progress before the final assembled response.

## Best Possible Solution

Keep the existing raw `stream_event` path, add narrowly gated support for positively identified cumulative Claude `assistant` partials, and verify one real failing-style turn through Gateway to WebChat or TUI with incremental output and no duplicate final text.

## Reproduction Assessment

No, not universally: a contributor could not reproduce the failure on current main or the reported release shape, but source inspection gives a clear conditional reproduction—when Claude emits cumulative `assistant` partials without raw text-delta events, current main ignores the partial text.

## Solution Assessment

Unclear pending live proof: the linked PR targets the narrowest parser boundary and uses a positive partial marker, but the exact failing upstream sequence and real Gateway/surface behavior still need verification.
