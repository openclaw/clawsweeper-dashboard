## Summary

Keep open: the PR is the right core surface for this Gateway behavior, but the latest branch still has two merge-blocking issues in the startup recovery path and its focused test harness.

## What This Changes

The PR adds a `gateway.reload.recovery` config option, schema/help/docs/changelog metadata, hot-reload handling, and startup recovery gating for last-known-good config restores.

## Best Possible Solution

Keep the new Gateway opt-out, but centralize startup recovery policy so both `recovery: "off"` and `mode: "off"` are honored from parsed source config or the last-known-good policy source, with focused regression tests.

## Reproduction Assessment

Yes. A static reproduction is an invalid startup snapshot whose source config or LKG file has `gateway.reload.mode: "off"` and no explicit `recovery`; the PR leaves `effectiveRecovery` undefined and still permits last-known-good recovery.

## Solution Assessment

No. The hot-reload implementation is narrow, but startup should share the same effective reload-settings semantics instead of checking only the raw `recovery` field.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Mock the new config IO export in startup tests:** `src/gateway/server-startup-config.ts:9`
  - body: The recovery test file uses a factory mock for `../config/io.js` that does not provide `resolveLastKnownGoodConfigPath`. With this new named import, Vitest will fail module loading before the startup recovery tests run; add the mocked export or inject the path resolver.
  - confidence: 0.9
- **[P2] Honor mode off during startup recovery:** `src/gateway/server-startup-config.ts:259-262`
  - body: The PR says `gateway.reload.mode: "off"` makes recovery default to off, and hot reload implements that, but startup only checks `gateway.reload.recovery`. An invalid startup config or parse-failure LKG with `mode: "off"` and no explicit recovery still reaches `recoverConfigFromLastKnownGood`, so the file can be overwritten despite reload being disabled.
  - confidence: 0.9
