## Summary

Keep open: the PR is useful but still not merge-ready. The second commit adds docs/changelog and an LKG fallback for explicit `recovery: "off"`, but startup still ignores the documented `reload.mode: "off"` default and the startup recovery test mock still lacks the new config IO export.

## What This Changes

The PR adds a `gateway.reload.recovery` config setting, schema/help/docs/changelog entries, hot-reload recovery skip behavior, and startup last-known-good recovery gating.

## Best Possible Solution

Keep the new Gateway opt-out, but share one effective reload-recovery resolver for hot reload and startup so both explicit `recovery: "off"` and implicit `mode: "off"` are honored from parsed source config and last-known-good fallback data, with focused regression tests.

## Reproduction Assessment

Yes. A static reproduction is an invalid startup snapshot whose source config or `.last-good` config has `gateway.reload.mode: "off"` and no explicit `recovery`; the PR leaves `effectiveRecovery` undefined and still permits last-known-good recovery.

## Solution Assessment

No. The hot-reload half is on the right path, but startup needs the same effective setting semantics and the test mock repair before this is a maintainable fix.
