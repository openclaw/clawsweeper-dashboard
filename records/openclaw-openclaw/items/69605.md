## Summary

Keep this issue open: current main and the latest release still contain the Control UI source path that can copy the wrong payload, and open in-repo PRs with closing syntax already target the issue.

## What This Changes

The issue asks for the Control UI WebChat code-block copy button to copy only the clicked code block’s content instead of the whole assistant message.

## Best Possible Solution

Land one canonical Control UI patch that copies rendered code-block text only, prevents message-action overlap with `has-copy` or an equivalent layout guard, and adds focused regression coverage.

## Reproduction Assessment

Yes. The issue gives deterministic Control UI steps, and current-main source still shows the code-block handler writing `dataset.code` while message-level actions can overlap because the `has-copy` layout guard is not applied. I did not run a live browser repro, so this is source-reproducible rather than locally reproduced.

## Solution Assessment

Yes. The maintainable fix belongs in the Control UI WebChat markdown/click-handling and grouped chat layout paths; because open PRs already target this issue with closing syntax, maintainers should review, consolidate, land, or close those PRs before closing this issue.
