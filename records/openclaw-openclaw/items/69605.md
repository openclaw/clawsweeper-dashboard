## Summary

Close as implemented on current main: the Control UI now applies the existing `has-copy` layout guard whenever assistant message actions render, preventing the message-level markdown copy control from overlapping the code-block Copy button. The fix is present on current main with regression coverage, but I could not prove it has shipped in a release tag yet.

## What This Changes

The issue requests that Control UI code-block Copy buttons copy only the selected code block instead of the whole assistant message.

## Best Possible Solution

Keep the current `has-copy` layout fix on main, close this issue as fixed there, and handle the now-overlapping open PRs as superseded or follow-up cleanup separately.

## Reproduction Assessment

Yes. The issue gives deterministic Control UI steps, and the source path shows the original full-message clipboard payload came from the floating message-level markdown copy action winning the click target; current main no longer has that overlap path by source inspection.

## Solution Assessment

Yes. Activating the existing `has-copy` CSS contract under the same predicate that renders `.chat-bubble-actions` is the narrowest maintainable fix for the reported full-message copy behavior; the separate rendered-`textContent` PR is an edge-case improvement, not required for this symptom.
