## Summary

Current main and the latest release already fix the reported whole-message copy symptom by reserving action space whenever assistant message copy/expand controls render, so the code-block Copy target is no longer covered by the message-level Copy as markdown action.

## What This Changes

The issue asks for Control UI WebChat code-block Copy buttons to copy only the clicked code block instead of the full assistant message.

## Best Possible Solution

Keep the shipped v2026.6.10 layout fix and treat any remaining rendered-text payload edge case as separate from the resolved whole-message overlap symptom.

## Reproduction Assessment

Yes for the historical bug: v2026.4.14 rendered assistant message actions without the has-copy bubble class, matching the overlap root cause in the linked PRs. A current-main failing repro is no longer expected because the action-space class is present and covered.

## Solution Assessment

Yes. Applying the existing action-space class under the same condition that renders message-level actions is the narrow maintainable fix for the whole-message copy overlap path.
