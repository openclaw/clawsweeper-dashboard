Keep this issue open: current main and the latest release still show the Control UI paths that can copy the wrong payload, and two open in-repo PRs already target the issue with closing references.

## What This Changes

The issue reports that the Control UI WebChat code-block copy control copies the surrounding assistant message instead of only the fenced code block.

## Best Possible Solution

Land one canonical Control UI patch that copies rendered code-block text only, prevents message-action overlap with has-copy or an equivalent layout guard, and adds focused regression coverage.

## Reproduction Assessment

Yes. The issue gives deterministic Control UI steps, and current-main source still shows the code-block handler writing dataset.code while message-level actions can overlap because the has-copy layout guard is not applied; I did not run a live browser repro.

## Solution Assessment

Yes, the best direction is a narrow Control UI WebChat fix in the markdown/click-handling and grouped chat layout paths. Because open PRs already target this issue with closing syntax, maintainers should review, consolidate, land, or close those PRs before closing this issue.
