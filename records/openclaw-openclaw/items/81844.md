## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Use the repository pi-agent-core package:** `src/agents/pi-embedded-runner/run/consecutive-error-guard.ts:17`
  - body: `package.json` declares `@earendil-works/pi-agent-core`, and current source imports that package throughout the repo. Importing `AgentMessage` from `@mariozechner/pi-agent-core` in this new file will fail module resolution before the guard can build.
  - confidence: 0.99
- **[P1] Declare or remove the sessionId option:** `src/agents/pi-embedded-runner/run/attempt.ts:2068`
  - body: `installConsecutiveErrorGuard` is called with `sessionId`, but `ConsecutiveErrorGuardParams` does not define that property. Because this is an object literal passed to a typed parameter, TypeScript rejects the call.
  - confidence: 0.98
- **[P2] Respect the loop-detection configuration boundary:** `src/agents/pi-embedded-runner/run/attempt.ts:2062-2070`
  - body: This installs a hard-aborting same-error/flood detector on every embedded attempt with hardcoded thresholds. Current generic loop detection is configured through `tools.loopDetection`, and unknown-tool retry handling already has a dedicated thresholded guard, so this changes default runtime behavior without the existing config/product boundary.
  - confidence: 0.9
- **[P3] Unwind transform guards in reverse order:** `src/agents/pi-embedded-runner/run/attempt.subscription-cleanup.ts:80`
  - body: The new guard wraps `transformContext` after the existing tool-result/history-image transforms, but cleanup removes the older wrapper first and then restores the newer guard's captured wrapper. Dispose the newest guard first or compose it into the existing remover so cleanup cannot leave a stale wrapper installed.
  - confidence: 0.82
