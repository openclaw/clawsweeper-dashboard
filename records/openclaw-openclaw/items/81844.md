  - confidence: 0.98
- **[P2] Route this through loop detection:** `src/agents/pi-embedded-runner/run/attempt.ts:2059-2067`
  - body: Installing a hard-coded transformContext abort on every embedded attempt bypasses the existing `tools.loopDetection` config/docs/test contract. Any broader consecutive-error breaker should live in the run-scoped detector path with a documented threshold and tests.
  - confidence: 0.91
- **[P3] Dispose transform wrappers in reverse order:** `src/agents/pi-embedded-runner/run/attempt.subscription-cleanup.ts:79`
  - body: The consecutive guard is installed after the existing context transforms, but cleanup removes the older transform chain first and then restores the newer wrapper. Dispose the newest wrapper first or compose it into the existing remover so `agent.transformContext` cannot be left pointing at a stale disposed wrapper during cleanup.
  - confidence: 0.82
