## Summary

Keep open: current main still lacks `engineCandidates` in the memory corpus supplement contract, and the same reporter has an open implementation PR with closing syntax at https://github.com/openclaw/openclaw/pull/82985.

## What This Changes

Request an optional `engineCandidates` parameter for `MemoryCorpusSupplement.search` so reranker supplements can reuse the engine's memory-search results instead of calling `manager.search()` again.

## Best Possible Solution

Review and resolve https://github.com/openclaw/openclaw/pull/82985 against the SDK boundary, then close this issue only after the chosen contract lands or maintainers decide not to add it.

## Reproduction Assessment

Yes for the source-level API limitation: current main has `rawResults` in scope before the supplement call, while the supplement contract and helper do not include `engineCandidates`. This is an API feature gap rather than a broken documented behavior repro.

## Solution Assessment

Unclear. Adding an optional candidate field is a plausible narrow direction, but the candidate shape and preservation of existing supplement params should be settled in PR review.
