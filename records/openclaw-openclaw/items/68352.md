## Summary

Keep open: current main still lacks a populated-but-mismatched `plugins.allow` diagnostic, and https://github.com/openclaw/openclaw/pull/68389 is the active open implementation candidate that explicitly closes this issue.

## What This Changes

The issue asks for clearer plugin allowlist diagnostics when users configure channel-style IDs such as `feishu` instead of discovered plugin IDs such as `openclaw-lark`, plus related plugin ID discoverability and auto-load policy improvements.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/68389 with a focused warning-only fix and regression coverage, leaving channel aliasing and extension auto-load policy as separate follow-up decisions if needed.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure `plugins.allow` with only a channel-style ID while a non-bundled plugin with a different real ID is discoverable, and the current helper returns before comparing the allowlist to discovered plugin IDs.

## Solution Assessment

Yes for the narrow diagnostic fix. A warning-only change with regression coverage is the smallest maintainable solution; automatic aliases and extension auto-load policy changes should remain separate product work.
