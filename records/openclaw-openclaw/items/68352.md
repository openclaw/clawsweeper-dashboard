## Summary

Keep open: current main still skips populated `plugins.allow` lists before checking whether entries match discovered plugin IDs, and the linked fix PR is open and unmerged.

## What This Changes

The issue asks for clearer plugin allowlist diagnostics when users put channel IDs such as `feishu` in `plugins.allow` instead of the discovered plugin ID.

## Best Possible Solution

Keep this issue open until https://github.com/openclaw/openclaw/pull/68389 or an equivalent focused diagnostic lands with regression coverage; track aliasing and auto-load policy separately.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure `plugins.allow` with only a channel-style ID while a workspace/global plugin is discoverable, and the helper returns before any mismatch warning can run.

## Solution Assessment

Yes for the narrow diagnostic. A warning-only change in the plugin-loader provenance helper is the smallest maintainable fix; automatic channel aliases and extension auto-load policy need separate maintainer decisions.
