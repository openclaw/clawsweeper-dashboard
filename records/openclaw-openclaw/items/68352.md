## What This Changes

The issue asks OpenClaw to make plugin allowlist warnings clearer when configured entries such as channel IDs do not match discovered plugin IDs.

## Best Possible Solution

Keep this issue open until https://github.com/openclaw/openclaw/pull/68389 or a maintainer-approved replacement lands a focused diagnostic that identifies unmatched allowlist entries and points users to the discovered plugin ID.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: configure a populated `plugins.allow` with only non-plugin IDs and a non-bundled discovered plugin, and the current helper returns before mismatch comparison. I did not run the gateway because this review is read-only.

## Solution Assessment

Yes for the narrow diagnostic fix. A warning-only change is the maintainable first step; automatic channel aliases or changing extension auto-load policy should stay separate unless maintainers explicitly choose that product direction.
