## Summary

Keep open: current main still lacks the mismatch-specific allowlist diagnostic, and an open unmerged PR already carries the focused fix for this issue.

## What This Changes

Improve plugin allowlist warnings when users configure channel IDs such as `feishu` instead of discovered plugin IDs such as `openclaw-lark`.

## Best Possible Solution

Land a focused diagnostic that warns when every `plugins.allow` entry misses discovered plugin IDs and suggests using plugin IDs, while leaving aliasing and auto-load policy for separate work.

## Reproduction Assessment

Yes. Source inspection on current main shows `loadOpenClawPlugins` passes the allowlist and discovered plugin IDs to `warnWhenAllowlistIsOpen`, but the helper returns for any non-empty allowlist, so `plugins.allow = ["feishu"]` with discovered `openclaw-lark` cannot get a mismatch warning.

## Solution Assessment

Yes for the narrow fix. A warning-only change in `src/plugins/loader-provenance.ts` is the smallest maintainable solution; automatic aliases and extension auto-load policy should remain separate decisions.
