## Summary

## What This Changes

The issue reports that long Dreams Diary entries can scroll the day-chip navigation out of view, preventing date switching in the Web UI.

## Best Possible Solution

Land the linked Dreams UI fix, or an equivalent narrow replacement, after it keeps day chips reachable during long diary scrolls and has focused browser proof.

## Reproduction Assessment

Yes, at source level: current main renders the day-chip rail before long diary article content inside a scrollable `.dreams-diary` container, while the rail is only `position: relative`. I did not run a live browser session, so this is not marked as fully reproduced.

## Solution Assessment

Yes, the narrow maintainable fix is to keep the existing day-chip navigation reachable while long diary content scrolls, either by making that rail sticky or by moving it into sticky chrome. The linked PR or a refreshed replacement should be reviewed and landed before closing this issue.
