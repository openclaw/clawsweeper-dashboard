## Summary

Keep open: current main still leaves the Dreams Diary day-chip strip as relative content in the scrollable diary pane, and open PR #76646 explicitly attempts to fix this issue, so the issue should remain open until that PR lands or is replaced.

## What This Changes

The issue asks for Dreams Diary date chips to remain reachable when a long selected diary entry scrolls the diary panel.

## Best Possible Solution

Land a focused Control UI fix that keeps the day-chip navigation in sticky chrome or gives the diary entry its own scroll region without overlapping the existing header.

## Reproduction Assessment

Yes by source inspection: multiple diary entries render day chips above the article in an overflow-auto `.dreams-diary`, and the chip strip is `position: relative` rather than sticky. I did not run a live browser session.

## Solution Assessment

Yes, the requested sticky-navigation direction is the narrow maintainable fix, but it should account for the existing sticky `.dreams-diary__chrome` so the chips do not overlap the header.
