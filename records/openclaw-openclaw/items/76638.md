## Summary

Keep open: current main still leaves the Dreams Diary day-chip strip as normal scrollable content beneath a sticky header, so a long diary entry can scroll the date controls out of reach. The report is recent, concrete, and narrow enough for a focused Control UI fix.

## What This Changes

The issue asks for Dreams Diary date chips to remain usable when a long diary entry fills the panel.

## Best Possible Solution

Keep the diary date navigation in the sticky control area or give the entry/prose its own scroll region, preserving horizontal chip scrolling and avoiding overlap with the existing sticky header.

## Reproduction Assessment

Yes by source inspection: render a Dreams Diary with multiple entries and a long selected entry, then scroll the `.dreams-diary` pane; the chips are outside the sticky chrome and styled as relative content, so they scroll away. I did not run a live browser session in this read-only pass.

## Solution Assessment

Yes, the requested direction is appropriate, but the repair should account for the existing sticky `.dreams-diary__chrome` instead of blindly adding `top: 0` to the chip strip where it may overlap the header.
