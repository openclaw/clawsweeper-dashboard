## Summary

Keep open: current main still renders the Dreams Diary day chips as normal relative content below the sticky diary chrome, and the linked closing PR #76646 is open rather than merged.

## What This Changes

The issue reports that long Dreams Diary entries can scroll the day-chip date navigation out of reach, preventing users from switching diary dates.

## Best Possible Solution

Review and land PR #76646, or an equivalent focused Control UI patch that keeps the diary date navigation reachable without overlapping the existing sticky diary header.

## Reproduction Assessment

Yes by source inspection: render Dreams Diary with multiple entries and a long selected entry, then scroll `.dreams-diary`; the chip strip is outside the sticky chrome and is styled as relative scroll content. I did not run a live browser session in this read-only review.

## Solution Assessment

Yes, a narrow Control UI layout fix is the right direction. The patch should account for the existing sticky `.dreams-diary__chrome`, either by keeping chips in the sticky area or giving long entry content its own scroll region.
