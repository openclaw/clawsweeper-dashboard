## Summary

Keep open. Current main still requests an exec-event heartbeat after background exec completion for any non-empty session key, while the heartbeat runner explicitly maps forced subagent keys back to the main session; the same-author implementation PR #66749 is still open and the latest reporter comment adds a cron-session exposure that is not yet resolved.

## What This Changes

The issue asks OpenClaw to stop subagent, nested-subagent, and cron-owned background exec completion events from forcing spurious main-session heartbeat wakes.

## Best Possible Solution

Keep the issue open while reviewers update or replace the fix so internal detached exec completions do not wake the main heartbeat for subagent, nested-subagent, and cron run/session keys, while preserving normal user-facing exec completion delivery and adding focused regression coverage for both exec completion paths.

## Reproduction Assessment

Yes. The source-level reproduction path is high confidence: background exec completion reaches maybeNotifyOnExit with a subagent session key, requests an exec-event heartbeat, and existing heartbeat tests show forced subagent keys resolve to the main session.

## Solution Assessment

Unclear. The narrow subagent guard proposed by #66749 fixes the first reported path, but the 2026-04-30 follow-up identifies cron-owned exec completions as a second path that likely needs either an expanded guard or the broader #69492 audience-classification design.
