## Summary

Keep open. Current main still unconditionally requests exec-event heartbeat wakes after background exec completion, and the open paired PR #66749 has not merged; the 2026-04-30 cron/nested-subagent follow-up also means the subagent-only fix may be incomplete.

## What This Changes

Background exec completions from subagent sessions should not force an exec-event heartbeat that wakes the main session.

## Best Possible Solution

Update or supersede #66749 so internal detached exec completions from subagent, nested-subagent, and cron-owned sessions do not wake the parent/main heartbeat path, with focused regression coverage for both exec completion call sites.

## Reproduction Assessment

Yes. The source-level path is high confidence: background exec completion calls requestHeartbeatNow with the child session key, and an existing heartbeat test confirms a forced subagent key runs the model against the main session.

## Solution Assessment

Unclear. A subagent guard is the right direction for the title, but the latest cron reproduction suggests the maintainable fix should also cover cron-owned detached exec completions or be folded into the #69492 event-audience primitive.
