## Summary

Keep open. Current main still has the source-level exec-event heartbeat wake path, and a paired open PR with closing syntax should be reviewed, updated, landed, or superseded before this issue closes.

## What This Changes

Prevent background exec completions from subagent sessions, and possibly cron-run sessions, from waking the main heartbeat session and producing unnecessary heartbeat turns.

## Best Possible Solution

Review and land or supersede https://github.com/openclaw/openclaw/pull/66749 so subagent exec completions skip redundant heartbeat wakes while any cron-run handling stays aligned with the current delivery policy.

## Reproduction Assessment

Yes, at source level. Current main requests exec-event heartbeats for subagent session keys, and heartbeat resolution routes those keys back to the main session; I did not run a live WebChat/subagent setup.

## Solution Assessment

Yes for the narrow subagent producer-side guard. The cron-run extension is plausible but should be reviewed against current main's delivery-mode and cron wake routing policy before merging.
