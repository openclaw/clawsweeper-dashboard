## Summary

Keep open. Current main still has the source-level exec-event heartbeat wake path, and the provided context shows an open closing PR that should be reviewed, updated, landed, or superseded before this can close.

## What This Changes

Background exec completions from subagent and related internal sessions should not force an exec-event heartbeat that wakes the main session.

## Best Possible Solution

Review and land or supersede #66749 so exec completion producers skip heartbeat wakes for internal subagent and cron-owned sessions while preserving the owning session's system event.

## Reproduction Assessment

Yes. Source inspection shows both exec completion call sites request an `exec-event` heartbeat with the child session key, and an existing heartbeat test confirms forced subagent keys resolve to the main session; I did not run a live WebChat reproduction in this read-only review.

## Solution Assessment

Yes for the narrow bug, assuming the paired PR covers the cron-owned session shape that reaches these call sites. A producer-side guard using existing session-key helpers is smaller than changing heartbeat session resolution or waiting for the broader event-audience primitive.
