## Summary

## What This Changes

The issue reports that Control UI WebChat can silently reset a visible session after reconnect or sleep, archive the transcript as `.jsonl.reset.*`, and remove it from the sessions tab.

## Best Possible Solution

Review and either land or reject https://github.com/openclaw/openclaw/pull/89017; if rejected, keep one canonical issue for the maintainer-approved reconnect/session-policy fix.

## Reproduction Assessment

Yes at source level: current main forwards a visible `sessionId`, but the session lifecycle still applies daily/idle freshness and archives stale entries as reset transcripts. I did not run the physical Mac Docker sleep/reconnect scenario.

No current-main fix is complete. The open PR is the focused implementation candidate, but its public protocol field and reset-policy exception need maintainer acceptance before this issue can close.
