## Summary

## What This Changes

The issue asks for Control UI WebChat to resume the visible session after sleep or network reconnect instead of rotating the transcript into `.jsonl.reset.*` and hiding it from the sessions tab.

## Best Possible Solution

Keep this issue open and resolve it by landing or explicitly rejecting the linked reconnect/session-policy PR, with any broader reset-archive history browsing work left to the related open reports.

Unclear until maintainers decide the API/session-policy shape. The linked PR is a plausible narrow fix, but it adds a public reconnect hint and rollover exception that need maintainer acceptance before this issue should close.
