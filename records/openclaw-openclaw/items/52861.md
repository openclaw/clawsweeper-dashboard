## Summary

Keep open. Current main and the latest release have partial mitigations for skill catalog size, explicit tool allowlists, and lightweight bootstrap modes, but they do not implement the central conditional/lazy prompt-surface behavior for tools, skills, workspace files, or compaction; the active related issues and PRs show this still needs maintainer sequencing and product/API decisions.

## What This Changes

The issue requests reducing repeated per-turn prompt overhead by conditionally injecting tools, lazily loading skills, streaming workspace files, and compacting older conversation context.

## Best Possible Solution

Keep this as a maintainer-reviewed umbrella and land incremental, measured changes that preserve compatibility: default-safe tool/schema slimming, opt-in or proven skill progressive disclosure, and a coherent workspace-context policy with docs and before/after token accounting.

## Reproduction Assessment

Yes for a source-level current-main and release gap check: the default paths still build full tool surfaces, inject retained eligible skills, and use coarse workspace bootstrap controls. I did not run a live Feishu/Lark token measurement in this read-only review.

## Solution Assessment

No single automatic patch is the best solution. The maintainable path needs maintainer-approved sequencing across tool-routing semantics, plugin compatibility, skill discovery, workspace bootstrap policy, compaction behavior, docs, and measured before/after proof.
