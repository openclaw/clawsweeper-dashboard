## Summary

Keep open: current main has useful mitigations such as opt-in Tool Search, compact skill fallback, context-injection controls, and compaction recovery, but it still does not implement the requested default conditional/lazy prompt injection across tools, skills, workspace files, and compaction policy.

## What This Changes

The issue requests conditional or lazy injection of tool schemas, skills, workspace files, and compaction context to reduce repeated per-turn static prompt tokens.

## Best Possible Solution

Keep this as the umbrella design thread and land narrower maintainer-reviewed changes for default tool-selection policy, skill resolver or metadata semantics, workspace context injection policy, and compaction behavior.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/performance design request. Source inspection gives a high-confidence current-main gap check: the existing mitigations are opt-in, budget-based, or recovery-oriented rather than the requested conditional injection system.

## Solution Assessment

No, not as one automatic fix. The maintainable path is incremental and policy-driven because tools, skills, workspace bootstrap files, and compaction each have different compatibility and default-behavior tradeoffs.
