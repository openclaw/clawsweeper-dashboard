## Summary

Keep open: current main has useful partial mitigations, but it does not fully solve the broad conditional tool, skill, workspace, and compaction overhead request; related open threads also treat this as the umbrella that needs maintainer design.

## What This Changes

The issue requests reducing per-turn token overhead by conditionally injecting tools, skills, workspace files, and compaction content instead of sending all static prompt material every turn.

## Best Possible Solution

Keep this as the umbrella design thread while pursuing incremental pieces through https://github.com/openclaw/openclaw/issues/14785, https://github.com/openclaw/openclaw/issues/64760, https://github.com/openclaw/openclaw/issues/57745, and https://github.com/openclaw/openclaw/issues/74100.

## Reproduction Assessment

Not applicable as a feature/performance design request. Source inspection confirms the structural state on current main: direct tool schemas still flow without Tool Search, skills are still a flat budgeted catalog, and workspace injection has config mitigations but no relevance streaming.

## Solution Assessment

No as a single automatic fix. Tool Search, Codex deferred tools, skill budgets, and contextInjection are good partial directions, but the best full solution needs maintainer choices about default versus opt-in behavior, skill resolver metadata, workspace-file policy, and compaction semantics.
