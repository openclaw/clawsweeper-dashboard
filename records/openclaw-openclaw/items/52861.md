## Summary

Keep open: current main has useful token-overhead mitigations, but it still does not implement the requested default conditional/lazy injection policy across tool schemas, skills, workspace bootstrap files, and long-session compaction.

## What This Changes

The issue requests reducing repeated per-turn prompt overhead by conditionally injecting tools, lazily exposing skills, streaming workspace files on demand, and compacting older conversation context.

## Best Possible Solution

Keep this as the umbrella design thread while landing narrower, measured changes that preserve default compatibility for tool selection, skill visibility, workspace bootstrap injection, and compaction behavior.

## Reproduction Assessment

Yes for a source-level gap check, but not as a strict bug reproduction: current main still uses opt-in Tool Search, budget-based skill prompt compaction, bootstrap context budgets, and overflow-driven compaction rather than a default conditional injection policy.

## Solution Assessment

No as one automatic fix; the maintainable solution needs maintainer policy choices because tools, skills, workspace bootstrap files, and compaction have different compatibility, prompt-cache, and default-behavior tradeoffs.
