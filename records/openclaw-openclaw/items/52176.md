## Summary

Keep open: current main does not implement this uv Docker build arg, and the PR is not mergeable while it adds an unpinned root-run installer plus incomplete PATH, setup, docs, and changelog surfaces.

## What This Changes

The PR adds an opt-in `OPENCLAW_INSTALL_UV` Docker build arg that downloads Astral's uv installer during image builds and moves `uv`/`uvx` into `/usr/local/bin`.

## Best Possible Solution

If core accepts this Docker knob, install uv/uvx from a pinned or digest-verified artifact, configure a node-writable uv tool bin directory on PATH, wire setup/docs/changelog/tests, and validate the image as the non-root `node` user.

## Reproduction Assessment

Not applicable for the feature request itself. The blocking review issues are source-reproducible from the PR-added Dockerfile block, current Docker setup wiring, and OpenClaw's uv skill installer/PATH checks.

## Solution Assessment

No. The direction is plausible, but this is not the best mergeable shape while it trusts an unpinned installer as root and leaves setup, PATH, docs, tests, and changelog surfaces incomplete.

  - body: This Docker build path executes whatever `https://astral.sh/uv/install.sh` serves at build time as root, so rebuilding the same OpenClaw commit can trust different code. Use a pinned uv release image/artifact with digest or checksum verification before adding this production image knob.

  - confidence: 0.88

  - confidence: 0.84
