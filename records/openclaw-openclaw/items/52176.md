## Summary

Keep open. The feature is useful and current main does not already provide `OPENCLAW_INSTALL_UV`, but this PR should not merge as-is because it adds root-level Docker build execution of a moving remote installer and does not make uv-installed skill tool shims reliably available to the non-root runtime user; the discussion also confirms `OPENCLAW_DOCKER_APT_PACKAGES` is an available workaround meanwhile.

## What This Changes

The PR adds an `OPENCLAW_INSTALL_UV` Docker build arg that downloads Astral's uv installer and moves `uv` and `uvx` into `/usr/local/bin`.

## Best Possible Solution

Land a revised Docker feature that pins and verifies the uv artifact, configures a node-owned uv tool executable directory on PATH, and updates the Docker setup/docs/changelog surface with targeted coverage.

## Reproduction Assessment

Yes for the PR blockers by source inspection: the diff executes the latest uv installer directly, and current OpenClaw skill installation plus uv storage docs show tool shims need explicit PATH or `UV_TOOL_BIN_DIR` handling. I did not run a Docker build because this review is read-only.

## Solution Assessment

No. The direction is reasonable, but the current implementation is not the narrowest maintainable solution until the installer source is pinned/verified and the non-root runtime PATH contract is handled.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Pin and verify the uv installer before executing it:** `Dockerfile:201`
  - body: The new opt-in branch runs `curl -LsSf https://astral.sh/uv/install.sh | sh` as root during Docker build. That endpoint tracks the current uv release, so rebuilding the same OpenClaw commit can execute different network code; the surrounding Dockerfile otherwise pins base images and verifies Docker's apt key fingerprint. Please pin a uv version and verify the downloaded installer or binary checksum before executing it.
  - confidence: 0.9
- **[P2] Expose uv tool shims on the node user's PATH:** `Dockerfile:196-201`
  - body: `installSkill` invokes `uv tool install`, and uv puts tool executable shims in a user executable directory such as `$HOME/.local/bin` unless `UV_TOOL_BIN_DIR` is set. The final image switches to `USER node`, but this block only installs `uv` and `uvx` in `/usr/local/bin`, so uv-backed skill binaries can still fail `requires.bins` eligibility. Add a node-owned tool bin dir to PATH or configure `UV_TOOL_BIN_DIR` accordingly.
  - confidence: 0.86
- **[P3] Add the required changelog entry:** `Dockerfile:197`
  - body: This adds a user-facing Docker build option, but the PR does not update `CHANGELOG.md`. OpenClaw policy requires user-facing `feat` changes to include a changelog entry.
  - confidence: 0.82
