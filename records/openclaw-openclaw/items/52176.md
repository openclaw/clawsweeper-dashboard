## What This Changes

The PR adds an `OPENCLAW_INSTALL_UV` Docker build argument that conditionally runs the uv standalone installer and moves `uv` and `uvx` into `/usr/local/bin`.

## Best Possible Solution

If OpenClaw accepts first-class uv Docker support, land it as a pinned and verified uv artifact or versioned installer with setup/docs/tests, node-visible tool-bin configuration, and real Docker runtime proof; otherwise document the custom-image workaround instead.

  - body: This build path executes whatever `https://astral.sh/uv/install.sh` serves at build time as root, so rebuilding the same OpenClaw commit can trust different code.   - confidence: 0.9

Operators setting `OPENCLAW_INSTALL_UV=1` around setup would still build an image without uv unless setup, docs, and tests are updated or the feature is explicitly raw `docker build` only.
  - confidence: 0.87
