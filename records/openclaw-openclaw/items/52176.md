## Summary

Keep open. Current main does not implement this uv Docker knob, and the latest PR diff still has concrete security, runtime PATH, setup-wiring, and proof blockers for a new Docker build feature.

## What This Changes

The PR adds an optional `OPENCLAW_INSTALL_UV` Docker build argument that runs the Astral uv installer and moves `uv` and `uvx` into `/usr/local/bin`.

## Best Possible Solution

If OpenClaw accepts this Docker feature, install uv and uvx from a pinned and verified source, configure node-visible uv tool paths, wire setup/docs/tests, and require real Docker runtime proof.

## Reproduction Assessment

Not applicable for the requested feature itself. The blocking PR defects are source-reproducible from the Dockerfile diff, current Docker setup arg forwarding, and skill installer/PATH checks.

## Solution Assessment

No. The direction may be useful, but this patch is not the best mergeable form while it trusts a mutable remote installer as root and leaves node-visible uv tool paths plus supported setup wiring incomplete.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Pin and verify the uv artifact:** `Dockerfile:205`
  - body: This build path executes whatever `https://astral.sh/uv/install.sh` serves at build time as root, so rebuilding the same OpenClaw commit can trust different code. Use a pinned uv release artifact with digest or signature verification before adding this production image knob.
  - confidence: 0.94
- **[P2] Add a node-visible uv tool bin path:** `Dockerfile:199-207`
  - body: Installing `uv` and `uvx` does not make later `uv tool install` executables visible to the non-root runtime. Add a node-writable uv tool bin directory and include it on `PATH`, otherwise uv-backed skills can still fail the PATH-based `requires.bins` check after installation.
  - confidence: 0.9
- **[P2] Forward the uv arg through Docker setup:** `Dockerfile:199`
  - body: The supported `scripts/docker/setup.sh` local-build path forwards a fixed build-arg list, and this PR only adds the Dockerfile arg. Operators setting `OPENCLAW_INSTALL_UV=1` around setup would still build an image without uv unless setup/docs are updated or the feature is explicitly raw `docker build` only.
  - confidence: 0.87

## Security Review

Status: needs_attention
