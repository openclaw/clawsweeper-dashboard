## Best Possible Solution

If OpenClaw accepts first-class uv Docker support, implement it with a pinned and verified uv artifact or versioned installer, node-visible tool-bin configuration, setup/docs/tests wiring, and redacted Docker runtime proof; otherwise document the existing custom-image workaround.

  - body: This build path executes whatever `https://astral.sh/uv/install.sh` serves at build time as root, so rebuilding the same OpenClaw commit can trust different code.   - confidence: 0.9

  - confidence: 0.87
