## Summary

Keep open. The PR addresses a real Docker uv pain point and already moved `uvx` after review, but it still adds a root Docker build path that trusts an unpinned remote installer and does not make uv-installed tool executables discoverable on `PATH`.

## What This Changes

The PR adds an optional `OPENCLAW_INSTALL_UV` Docker build arg that installs `uv` and `uvx` into `/usr/local/bin`.

## Best Possible Solution

If core supports this, install uv/uvx from a pinned or digest-verified source and define a node-writable `UV_TOOL_BIN_DIR` on `PATH`, with Docker docs, changelog, and focused validation updated; otherwise document the custom-image approach.

## Reproduction Assessment

Not applicable for the feature request itself. The blocking review concerns are source-reproducible from the PR diff, current Dockerfile, OpenClaw's `uv tool install` path, PATH-based skill eligibility, and official uv tool-bin documentation.

## Solution Assessment

No. The PR is a useful direction, but it is not the best mergeable shape while it executes an unpinned installer as root and leaves uv-installed tool executables outside a deterministic runtime `PATH`.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Pin the uv install source:** `Dockerfile:200`
  - body: This root Docker build path executes whatever `https://astral.sh/uv/install.sh` serves at build time, so rebuilding the same OpenClaw commit can trust different code or fail if the installer layout changes. Official uv Docker docs recommend pinning a uv version and note SHA256 pinning as best practice for reproducible environments.
  - confidence: 0.91
- **[P2] Put uv tool executables on the runtime PATH:** `Dockerfile:199-203`
  - body: Moving only `uv` and `uvx` to `/usr/local/bin` does not make `uv tool install` outputs usable by skills. OpenClaw installs uv-backed skills with `uv tool install` and then checks `requires.bins` on `PATH`, while uv links installed tool executables into the tool bin directory derived from `UV_TOOL_BIN_DIR`, XDG variables, or `$HOME/.local/bin`; set a deterministic node-writable tool bin directory that is on `PATH`.
  - confidence: 0.88
