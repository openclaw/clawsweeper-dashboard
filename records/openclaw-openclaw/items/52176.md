## Summary

Keep open: current main still lacks a first-class uv Docker build arg, but this PR is not mergeable as-is because it adds an unpinned root installer path and leaves runtime PATH, setup, docs, and changelog surfaces incomplete.

## What This Changes

The PR adds an opt-in `OPENCLAW_INSTALL_UV` Docker build argument that runs the uv standalone installer during image build and moves `uv`/`uvx` into `/usr/local/bin`.

## Best Possible Solution

If maintainers want this feature, implement it with a pinned or digest-verified uv/uvx source, a node-writable uv tool bin directory on PATH, setup/docs/changelog updates, and Docker build proof; otherwise keep using the existing custom image or apt-package override path.

## Reproduction Assessment

Not applicable for the requested Docker feature itself. The blocking PR defects are source-reproducible from the added Dockerfile block plus current `scripts/docker/setup.sh`, `src/agents/skills-install.ts`, and `src/shared/config-eval.ts` behavior.

## Solution Assessment

No. The direction is plausible, but this patch is not the best mergeable shape while it trusts a mutable remote installer as root, omits uv tool executable PATH handling, and leaves setup/docs/changelog surfaces incomplete.

Full review comments:

- **[P1] Pin and verify the uv install artifact:** `Dockerfile:200`
  - body: This runs whatever `https://astral.sh/uv/install.sh` returns at build time as root, so rebuilding the same OpenClaw commit can trust different code. Use a pinned uv release/artifact with checksum or signature verification before adding this production image knob.
  - confidence: 0.94
- **[P2] Put uv tool executables on the runtime PATH:** `Dockerfile:198-204`
  - body: The patch exposes `uv` and `uvx`, but `uv tool install` places skill CLIs in uv's executable directory. Because the image never sets a node-writable `UV_TOOL_BIN_DIR` or adds it to PATH, PATH-based skill eligibility can still fail for uv-backed skills.
  - confidence: 0.9
- **[P2] Wire the uv build arg through supported Docker setup:** `Dockerfile:197`
  - body: `scripts/docker/setup.sh` forwards an explicit build-arg list when building `openclaw:local`. Operators setting `OPENCLAW_INSTALL_UV=1` for the supported setup path will not get uv unless setup/docs are updated or the feature is explicitly raw `docker build` only.
  - confidence: 0.86
- **[P3] Add the required changelog and Docker docs:** `Dockerfile:197`
  - body: This is a user-facing Docker feature, but the PR changes only `Dockerfile`. Add a single-line `CHANGELOG.md` entry and document the new build knob in the Docker install docs without inventing contributor attribution.
  - confidence: 0.84
