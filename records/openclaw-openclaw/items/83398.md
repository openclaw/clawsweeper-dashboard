The issue reports Feishu/Lark messages on v2026.5.16-beta.6 where the model generates final text but the channel dispatcher logs `queuedFinal=false, replies=0` and sends no visible reply.

## Best Possible Solution

Confirm the LongCat Feishu effective config and live logs, then either fix a narrow Feishu/shared dispatch regression or make an explicit maintainer policy decision for visible final replies in Feishu direct chats with regression coverage and redacted live proof.

## Reproduction Assessment

The exact v2026.5.16-beta.6 LongCat Feishu setup has not been live-reproduced or tied to that path from effective config/logs.

## Solution Assessment

Unclear as an automated fix: restoring automatic Feishu visible replies broadly could override documented tool-only behavior, while the safer repair depends on proving whether LongCat is entering source-reply suppression or exposing a distinct Feishu dispatch bug.
