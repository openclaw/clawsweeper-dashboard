## Summary

Keep open: current main still cannot express a Feishu document insert at a root-child index, and this PR is a plausible narrow feature, but it remains blocked by missing inspectable live Feishu proof plus two concrete correctness issues in the proposed patch.

## What This Changes

The PR makes Feishu `feishu_doc(action: "insert")` accept an optional `after_block_id`, adds an optional root-child `index`, and forwards that index into Docx descendant insertion.

## Best Possible Solution

Land a rebased Feishu plugin change that exposes root-child index insertion through the Descendant API, validates the position as a non-negative integer, preserves or explicitly rejects empty anchors, adds focused regression coverage, and includes redacted live Feishu proof.

## Reproduction Assessment

Yes by source inspection for the current limitation: current main requires `after_block_id` in the insert schema and only passes that anchor to `insertDoc`, so a root-index insert cannot be expressed. I did not run live Feishu calls in this read-only review.

## Solution Assessment

No as-is. Exposing the Descendant API index is the right narrow direction, but the patch should constrain `index`, preserve or reject empty-anchor input explicitly, add regression coverage, and attach inspectable after-fix proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Constrain insert index validation:** `extensions/feishu/src/doc-schema.ts:43-48`
  - body: The new root-child `index` is documented as a 0-based position, but `Type.Number()` lets negative and fractional values pass OpenClaw validation and fail later in Feishu. Use `Type.Integer({ minimum: 0, ... })`, matching the existing positional image index schema.
  - confidence: 0.92
- **[P2] Preserve provided empty anchors:** `extensions/feishu/src/docx.ts:887`
  - body: With `if (afterBlockId)`, a caller that provides `after_block_id: ""` no longer follows the anchored path and will insert at root index 0. Check for `undefined` explicitly or reject empty strings in the schema so provided anchors keep the old failure behavior instead of silently changing target.
  - confidence: 0.82
