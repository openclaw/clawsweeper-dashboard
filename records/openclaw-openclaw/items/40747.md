## Summary

Keep open. Current main still cannot insert Feishu document content without an anchor block, and this PR targets the right Feishu plugin boundary, but the patch is not merge-ready because it under-validates the new index and changes explicit empty-anchor behavior.

## What This Changes

The PR makes Feishu `feishu_doc(action: "insert")` accept an optional `after_block_id`, adds an optional root-level `index`, and forwards that index into Descendant insertion when no anchor block is supplied.

## Best Possible Solution

Land the Feishu plugin feature after tightening the parameter contract, preserving invalid-anchor failure behavior, adding focused runtime/schema coverage, documenting the new insert mode, and adding the required user-facing changelog entry.

## Reproduction Assessment

Yes. Static reproduction is high-confidence: current main requires `after_block_id` in schema and runtime, while the PR diff shows the invalid `Type.Number()` contract and falsey anchor branch directly.

## Solution Assessment

No as proposed. Exposing Feishu Descendant root insertion through the Feishu plugin is the right maintainable boundary, but the implementation should validate `index` as a non-negative integer and distinguish omitted anchors from invalid provided anchors before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Constrain insert index to non-negative integers:** `extensions/feishu/src/doc-schema.ts:43-48`
  - body: The new `index` is a 0-based root-child position, but `Type.Number()` lets values such as `-1` or `1.5` pass tool validation. The existing `upload_image` positional schema uses `Type.Integer({ minimum: 0 })`; this insert parameter should fail invalid positions before sending them to Feishu.
  - confidence: 0.95
- **[P2] Preserve explicit empty anchor failures:** `extensions/feishu/src/docx.ts:887`
  - body: `after_block_id` is still a plain string schema, so an empty string can be provided. Before this patch that value went through the anchor lookup and failed; the new `if (afterBlockId)` branch treats it as omitted and inserts at the root `index ?? 0`, which can silently place content at the beginning instead of rejecting the invalid anchor.
  - confidence: 0.84
- **[P3] Add the required changelog entry:** `extensions/feishu/src/doc-schema.ts:43-48`
  - body: This PR exposes a user-facing Feishu tool feature, but the branch only changes schema/runtime files. OpenClaw policy requires user-facing `feat` changes to add a single-line `CHANGELOG.md` entry, so the new insert mode would otherwise ship without release-note coverage.
  - confidence: 0.9
