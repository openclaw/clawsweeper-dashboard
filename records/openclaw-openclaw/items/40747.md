## Summary

Keep open. Current main still requires an anchor block for Feishu document inserts, and the PR targets the right plugin boundary, but the patch is not merge-ready because it under-validates the new index parameter and changes explicit empty-anchor behavior.

## What This Changes

The PR makes Feishu `feishu_doc(action: "insert")` accept optional `after_block_id`, adds an optional root-level `index`, and forwards that index into Docx Descendant insertion when no anchor block is supplied.

## Best Possible Solution

Land a corrected Feishu plugin implementation that validates `index` as a non-negative integer, preserves anchored insert failures, adds focused coverage, updates the Feishu doc skill, and adds the required changelog entry.

## Reproduction Assessment

Yes. Static source reproduction is high-confidence: current main requires `after_block_id` in schema and runtime, while the PR diff shows both the proposed no-anchor path and the two introduced defects.

## Solution Assessment

No as proposed. Exposing root-position insert through the Feishu plugin is the right boundary, but the patch should tighten the schema and distinguish omitted anchors from invalid provided anchors before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Constrain insert index to non-negative integers:** `extensions/feishu/src/doc-schema.ts:43-48`
  - body: The new `index` is a 0-based root-child position, but `Type.Number()` lets values such as `-1` or `1.5` pass tool validation. The existing `upload_image` positional schema uses `Type.Integer({ minimum: 0 })`; this insert parameter should fail invalid positions before sending them to Feishu.
  - confidence: 0.95
- **[P2] Preserve explicit empty anchor failures:** `extensions/feishu/src/docx.ts:887`
  - body: `after_block_id` is still a plain string schema, so callers can provide an empty string. Before this patch that value went through the anchor lookup and failed; the new truthy check treats it as omitted and can silently insert at the root instead.
  - confidence: 0.84

  - body: This exposes a user-facing Feishu tool feature, but the branch only changes schema/runtime files. OpenClaw policy requires user-facing `feat` changes to add a single-line `CHANGELOG.md` entry before shipping.
  - confidence: 0.9
