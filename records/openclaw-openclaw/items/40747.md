## Summary

Keep open: the Feishu insert-at-root capability is still missing on current main, and this PR is the right plugin-local direction, but it needs a small validation fix and release-note follow-through before merge.

## What This Changes

The PR makes Feishu `feishu_doc` insert accept an optional root-child `index` when `after_block_id` is omitted.

## Best Possible Solution

Land a rebased or replacement Feishu plugin change that exposes root-position inserts through the Descendant index, validates a non-negative integer index, covers anchored and no-anchor modes, and updates the Feishu skill docs plus changelog.

## Reproduction Assessment

Yes, from source inspection: current main requires `after_block_id` in the insert schema and `insertDoc` always resolves that anchor before inserting. I did not run live Feishu calls in this read-only review.

## Solution Assessment

No as-is. The Descendant API index path is the narrow maintainable solution, but the PR should use non-negative integer validation and add release-note/docs/test follow-through before merge.
