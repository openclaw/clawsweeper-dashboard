## Summary

Current main and the latest release still have the Feishu anchored-insert limitation, so the PR remains useful, but it is not merge-ready because it needs real behavior proof, a rebase over current Docx ordering code, and two small compatibility fixes.

## What This Changes

The PR makes Feishu `feishu_doc(action: "insert")` accept an optional root-child `index` when `after_block_id` is omitted and passes that value into Docx insertion.

## Best Possible Solution

Reapply the index-based insert over current Feishu Docx code with non-negative integer validation, explicit omitted-anchor handling, focused schema/runtime tests, and redacted live Feishu proof for both prepend and anchored insert paths.

## Reproduction Assessment

Yes, by source inspection: current main and `v2026.5.18` require `after_block_id` in the Feishu insert schema and only dispatch anchored Docx insert. I did not run live Feishu calls in this read-only review.

## Solution Assessment

No, not as currently written: exposing the Feishu API's index support is the right direction, but the branch should validate `index` as a non-negative integer, preserve explicit invalid-anchor behavior, and be rebased onto current block-tree normalization.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Constrain insert index validation:** `extensions/feishu/src/doc-schema.ts:44`
  - body: `index` is described as a 0-based document position, but `Type.Number` allows negative and fractional values through OpenClaw validation. Use `Type.Integer({ minimum: 0 })`, matching the existing positional image index schema, so invalid values fail before reaching Feishu.
  - confidence: 0.93
- **[P2] Preserve explicit anchor failures:** `extensions/feishu/src/docx.ts:886`
  - body: This truthiness check treats an explicitly provided empty `after_block_id` the same as omitting the anchor, so malformed anchored inserts can silently target root index 0. Check for `undefined` explicitly or reject empty strings so only truly omitted anchors use `index`.
  - confidence: 0.84
