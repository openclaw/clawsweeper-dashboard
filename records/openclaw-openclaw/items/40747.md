## What This Changes

The PR makes Feishu `feishu_doc(action: "insert")` accept omitted `after_block_id`, adds a root-child `index` parameter, and passes it into Docx descendant insertion.

## Best Possible Solution

Land a rebased Feishu plugin change that exposes root-child index insertion through the schema/runtime, validates `index` as a non-negative integer, preserves anchored insert behavior, adds focused regression tests, and includes redacted live Feishu proof.

## Reproduction Assessment

Yes by source inspection for the reported limitation: current main requires `after_block_id` in the insert schema and the runtime only dispatches anchored insert. I did not run live Feishu calls in this read-only review.

## Solution Assessment

No as-is. Exposing the Feishu Descendant API index is the right narrow direction, but this branch needs tighter validation, explicit empty-anchor handling, a rebase over current ordering logic, regression coverage, and inspectable live proof.
