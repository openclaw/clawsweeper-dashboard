## What This Changes

The PR changes Feishu `feishu_doc(action: "insert")` to make `after_block_id` optional, add an optional root-child `index` parameter, and pass that value into Docx insertion.

## Best Possible Solution

Land a rebased Feishu plugin change that exposes root-child index insertion, validates the new index contract, preserves anchored-insert semantics and current block-tree normalization, adds focused regression coverage, and includes redacted live Feishu proof.

## Reproduction Assessment

Yes by source inspection for the reported limitation: current main requires `after_block_id` in the insert schema and the runtime only dispatches anchored insert. I did not run live Feishu calls in this read-only review.

## Solution Assessment

No as-is. Exposing the Feishu Descendant API index is the right narrow direction, but this branch needs tighter validation, explicit empty-anchor handling, a rebase over current ordering logic, regression coverage, and inspectable live proof.
