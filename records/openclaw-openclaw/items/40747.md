## Summary

Keep open. The Feishu insert-position feature is still absent from current main and the approach is plausible, but the PR has blocking validation/anchored-mode issues and still lacks attached real behavior proof.

## What This Changes

The PR makes Feishu `feishu_doc` insert allow omitted `after_block_id`, adds an optional root-child `index`, and passes that index into Docx descendant insertion.

## Best Possible Solution

Land a rebased Feishu plugin change that exposes no-anchor root-position inserts through the Descendant index, validates `index` and anchor inputs, preserves anchored behavior, and includes focused tests plus redacted live Feishu proof.

## Reproduction Assessment

Yes for the current limitation: source inspection shows current main requires `after_block_id` in the schema and computes insert position only from that anchor. I did not run live Feishu calls in this read-only review.

## Solution Assessment

No as-is. The Descendant API index path is the narrow maintainable direction, but this patch should constrain `index`, preserve provided-anchor semantics, and attach real after-fix proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Constrain insert index validation:** `extensions/feishu/src/doc-schema.ts:43-48`
  - body: The new root-child `index` is documented as a 0-based position, but `Type.Number()` lets negative and fractional values pass OpenClaw validation and fail later in Feishu. Use `Type.Integer({ minimum: 0, ... })`, matching the existing `upload_image` positional index schema.
  - confidence: 0.92
- **[P2] Preserve provided-anchor handling:** `extensions/feishu/src/docx.ts:886`
  - body: `after_block_id` remains a plain optional string, so callers can still provide an empty string. With `if (afterBlockId)`, that provided anchor is treated as omitted and inserts at root index 0 instead of following the old anchored path/error; check for `undefined` explicitly or reject empty anchors in the schema.
  - confidence: 0.82
