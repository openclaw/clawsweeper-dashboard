## Summary

Keep open: current main still lacks root-index Feishu doc insertion, and the PR is a useful narrow feature, but it needs inspectable live proof plus two small correctness fixes before merge.

## What This Changes

The PR makes Feishu `feishu_doc(action: "insert")` accept optional `after_block_id`, adds optional root-child `index`, and forwards that position into Docx descendant insertion.

## Best Possible Solution

Land a rebased Feishu plugin change that exposes root-child index insertion, validates `index` as a non-negative integer, handles empty anchors explicitly, adds focused regression coverage, and includes redacted live Feishu proof.

## Reproduction Assessment

Yes by source inspection: current main requires `after_block_id` in the insert schema and only dispatches anchored insert, so root-index insertion cannot be expressed. I did not run live Feishu calls in this read-only review.

## Solution Assessment

No as-is. Exposing the Descendant API `index` is the right narrow direction, but the patch should constrain the parameter, preserve or explicitly reject empty anchors, add regression coverage, and provide inspectable live proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Constrain insert index validation:** `extensions/feishu/src/doc-schema.ts:43-48`
  - body: The new root-child `index` is documented as a 0-based position, but `Type.Number()` lets negative and fractional values pass OpenClaw validation and fail later in Feishu. Use `Type.Integer({ minimum: 0, ... })`, matching the existing positional image index schema.
  - confidence: 0.92
- **[P2] Preserve provided empty anchors:** `extensions/feishu/src/docx.ts:887`
  - body: With `if (afterBlockId)`, a caller that provides `after_block_id: ""` no longer follows the anchored path and will insert at root index 0. Check for `undefined` explicitly or reject empty strings in the schema so provided anchors keep the old failure behavior instead of silently changing target.
  - confidence: 0.82
