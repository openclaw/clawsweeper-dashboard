## What This Changes

The PR makes Feishu `feishu_doc(action: "insert")` accept optional `after_block_id`, adds an optional root-child `index`, and passes that value into Docx insertion.

## Best Possible Solution

Land a rebased Feishu plugin change that exposes root-child index insertion, validates `index` as a non-negative integer, treats omitted versus empty anchors explicitly, preserves current block-tree normalization, adds focused regression coverage, and includes redacted live Feishu proof.

## Reproduction Assessment

Yes by source inspection for the reported limitation: current main requires `after_block_id` in the insert schema and the runtime only dispatches anchored insert. I did not run live Feishu calls in this read-only review.

## Solution Assessment

No as-is. Exposing Feishu Descendant API index insertion is the right narrow direction, but the branch needs tighter validation, explicit empty-anchor handling, a rebase over current ordering logic, regression coverage, and inspectable live proof.
