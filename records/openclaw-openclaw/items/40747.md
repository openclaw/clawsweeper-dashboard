## Summary

Keep open: current main still cannot insert Feishu document content at a root-child index, and the PR is a plausible plugin-local fix, but it should not merge as-is because index validation is too broad and contributor-supplied real behavior proof is missing.

## What This Changes

The branch makes Feishu `feishu_doc` insert accept optional `after_block_id`, adds an optional root-child `index`, and passes that index into Docx descendant insertion.

## Best Possible Solution

Land a rebased Feishu plugin change that exposes no-anchor root-position inserts through the Descendant index, validates `index` as a non-negative integer, preserves anchored behavior, and includes redacted live Feishu proof.

## Reproduction Assessment

Yes for the current limitation: source inspection shows current main requires `after_block_id` in the schema and computes insert position only from that anchor. I did not run live Feishu calls in this read-only review.

## Solution Assessment

No as-is. The Descendant API index path is the narrow maintainable direction, but the PR should constrain `index` to a non-negative integer and attach real after-fix proof before merge.
