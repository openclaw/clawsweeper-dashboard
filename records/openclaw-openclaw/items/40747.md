## Summary

Keep open: current main still lacks index-only Feishu document insert, and the PR is useful, but the patch is not merge-ready because it changes provided empty-anchor behavior and exposes an unconstrained positional index.

## What This Changes

The PR makes Feishu document insert accept an optional root-child `index` when `after_block_id` is omitted, while keeping the existing anchor-block path.

## Best Possible Solution

Expose root-position insertion in the Feishu plugin with a validated non-negative integer index, preserve provided-anchor semantics, and cover both paths with focused tests and a changelog entry.

## Reproduction Assessment

Yes from source inspection: current main requires `after_block_id` in the insert schema and passes that value to `insertDoc`, so callers cannot request a document-root index-only insert through the tool schema today.

## Solution Assessment

No, not as-is. The proposed runtime direction matches the existing Descendant API insertion path, but the patch should branch on `undefined` for the optional anchor and validate the new index as a non-negative integer before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve provided empty-anchor behavior:** `extensions/feishu/src/docx.ts:887`
  - body: The PR promises existing `after_block_id` behavior is unchanged when that field is provided, but `if (afterBlockId)` treats an empty string as omitted and inserts at `index ?? 0`. Since the schema still accepts an empty string, a malformed anchor that currently errors can now prepend content; branch on `afterBlockId !== undefined` or validate the string before falling back to index mode.
  - confidence: 0.9
- **[P2] Constrain insert index to valid positions:** `extensions/feishu/src/doc-schema.ts:43-48`
  - body: `index` is a 0-based insertion position, but `Type.Number()` accepts values like `-1` and `1.5`. The same tool already constrains `upload_image.index` with `Type.Integer({ minimum: 0 })`; use that shape here so invalid positions fail schema validation instead of surfacing as opaque Feishu API errors.
  - confidence: 0.92
- **[P3] Add the required changelog entry:** `extensions/feishu/src/doc-schema.ts:35-48`
  - body: This is a user-facing `feat` for the `feishu_doc` tool, but the PR only changes the implementation files. Repo policy requires user-facing feature changes to add a single-line `CHANGELOG.md` entry before merge.
  - confidence: 0.86
