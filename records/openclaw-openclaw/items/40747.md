## Summary

Keep open: current main still lacks no-anchor Feishu Docx inserts, but this PR is not merge-ready because it under-validates the new index path, changes empty-anchor behavior, and lacks attached real behavior proof.

## What This Changes

The PR makes Feishu `feishu_doc(action: "insert")` accept optional `after_block_id` plus optional root-child `index` and passes that index into Docx descendant insertion.

## Best Possible Solution

Land a rebased Feishu plugin change that exposes root-position inserts through the Descendant index, validates positional inputs, preserves anchored behavior, adds focused regression coverage, and includes redacted live Feishu proof.

## Reproduction Assessment

Yes for the current limitation: source inspection shows current main requires `after_block_id` in the tool schema and computes insertion only from that anchor. I did not run live Feishu calls in this read-only review.

## Solution Assessment

No as-is. The Descendant API index path is the narrow maintainable direction, but this patch should constrain `index`, preserve provided-anchor semantics, and attach real after-fix proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Constrain insert index validation:** `extensions/feishu/src/doc-schema.ts:43-48`
  - body: The new root-child `index` is documented as a 0-based position, but `Type.Number()` lets negative and fractional values pass OpenClaw validation and fail later in Feishu. Use `Type.Integer({ minimum: 0, ... })`, matching the existing positional image index schema.
  - confidence: 0.92
- **[P2] Treat only undefined as no-anchor mode:** `extensions/feishu/src/docx.ts:886`
  - body: `after_block_id` remains a plain optional string, so callers can provide an empty string. With `if (afterBlockId)`, that provided anchor is treated as omitted and inserts at root index 0 instead of following the old anchored error path; check for `undefined` explicitly or reject empty anchors in the schema.
  - confidence: 0.82
