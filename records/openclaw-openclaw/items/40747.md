## Summary

Keep open: current main still only supports anchor-based Feishu document inserts, and the PR uses the right Descendant API direction, but the patch needs tighter index validation plus user-facing follow-through before merge.

## What This Changes

The PR makes Feishu `feishu_doc` insert accept an optional root-child `index` by making `after_block_id` optional and passing the index into the Docx insert path.

## Best Possible Solution

Land a rebased or replacement Feishu plugin change that exposes root-position inserts through the existing Descendant index, validates non-negative integer positions, covers anchor and no-anchor modes, and updates skill docs plus changelog.

## Reproduction Assessment

Yes, from source inspection: current main requires `after_block_id` in the tool schema and `insertDoc` always resolves that anchor before inserting. I did not run live Feishu calls in this read-only review.

## Solution Assessment

No as-is. The Descendant API index path is the right narrow solution, but the PR should use non-negative integer validation and add regression/docs/changelog follow-through before merge.
