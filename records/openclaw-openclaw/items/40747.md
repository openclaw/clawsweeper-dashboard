## Summary

Keep open: current main still lacks root-position Feishu document inserts and the pinned SDK supports the requested index path, but this PR still needs two correctness fixes and inspectable live Feishu proof before merge.

## What This Changes

The PR makes Feishu `feishu_doc(action: "insert")` accept an optional `after_block_id`, adds an optional root-child `index`, and forwards that index into Docx descendant insertion.

## Best Possible Solution

Land a rebased Feishu plugin change that exposes root-child index insertion through the Descendant API, validates positional input, preserves anchored behavior, adds focused regression coverage, updates Feishu skill guidance if needed, and includes redacted live proof.

## Reproduction Assessment

Yes for the current limitation: current main requires `after_block_id` in the insert schema and passes only `p.after_block_id` to `insertDoc`, so a root-index insert cannot be expressed. I did not run live Feishu calls.

## Solution Assessment

No as-is. Exposing the Descendant API index is the right narrow direction, but the patch should constrain `index`, preserve provided-anchor semantics, add regression coverage, and attach real after-fix proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Constrain insert index validation:** `extensions/feishu/src/doc-schema.ts:43-48`
  - body: The new root-child `index` is documented as a 0-based position, but `Type.Number()` lets negative and fractional values pass OpenClaw validation and fail later in Feishu. Use `Type.Integer({ minimum: 0, ... })`, matching the existing positional image index schema.
  - confidence: 0.92
- **[P2] Preserve provided empty anchors:** `extensions/feishu/src/docx.ts:887`
  - body: With `if (afterBlockId)`, a caller that provides `after_block_id: ""` no longer follows the anchored path and will insert at root index 0. Check for `undefined` explicitly or reject empty strings in the schema so provided anchors keep the old failure behavior instead of silently changing target.
  - confidence: 0.82
