The PR adds optional `sessionKey` and `agentId` to `before_agent_start` and `agent_end` hook event payloads, forwards them through embedded runner paths, and adds hook regression tests.

## Best Possible Solution

Land a rebased additive hook contract in the current split hook files, mirror `sessionKey` and `agentId` into the two event payloads only when present, keep absent-field compatibility, add focused tests on current runner/helper surfaces, and include redacted real plugin-run proof.

## Solution Assessment

No, not as-is: optional event-field mirroring is a reasonable direction, but the branch must be rebased onto the split hook contracts, the test typing issue must be fixed, and real after-fix plugin-run proof must be provided before merge.
