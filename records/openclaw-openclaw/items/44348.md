## What This Changes

The PR adds optional `sessionKey` and `agentId` to `before_agent_start` and `agent_end` hook event payloads, forwards them from embedded runner paths, and adds hook regression tests.

## Best Possible Solution

Either keep `ctx.sessionKey` and `ctx.agentId` as the single lifecycle hook identity surface, or land a rebased additive event-mirroring change with focused tests, docs, and real memory-plugin proof.

## Reproduction Assessment

Yes, at source level: current main does not expose `sessionKey` or `agentId` on the `before_agent_start` or `agent_end` event objects, while the same identity is available on the hook context. I did not run a live memory plugin in this read-only review.

## Solution Assessment

Unclear. Additive event mirroring may be acceptable, but the existing supported `ctx` identity surface means maintainers should first decide whether lifecycle events should duplicate that contract.
