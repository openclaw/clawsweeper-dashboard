## Summary

Keep open. The API addition is still useful because current main still omits `sessionKey` and `agentId` from these legacy event payloads, but the branch targets the old hook contract layout and lacks required real behavior proof.

## What This Changes

This PR adds optional `sessionKey` and `agentId` fields to legacy `before_agent_start` and `agent_end` event payloads, threads them through embedded runner calls, and adds compatibility tests.

## Best Possible Solution

Rebase the additive API change onto the current split hook contract files and runner helper paths, keep optional fields omitted when unavailable, document the public event contract, and let maintainers handle any changelog entry during landing.

## Reproduction Assessment

Yes. Source inspection on current main shows the two legacy event types and emitted event objects omit `sessionKey` and `agentId`, while `PluginHookAgentContext` already carries both values.

## Solution Assessment

No, not as submitted. Mirroring the identity fields onto these legacy event payloads is a narrow backward-compatible direction if maintainers accept it, but the branch must target current files and provide real behavior proof before merge.
