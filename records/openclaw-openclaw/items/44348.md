## Summary

Keep open: current main still omits `sessionKey` and `agentId` from the legacy `before_agent_start` and `agent_end` event payloads, so the central improvement remains useful, but this PR is not mergeable as-is because it lacks real behavior proof, conflicts with the current split hook contracts, and has a type-checking bug in added coverage.

## What This Changes

The PR adds optional `sessionKey` and `agentId` to legacy `before_agent_start` and `agent_end` hook event payloads, threads them through embedded runner call sites, and adds hook regression tests.

## Best Possible Solution

Land a rebased additive hook contract that mirrors available `sessionKey` and `agentId` into the current legacy event payload types and runner helpers, omits fields when unavailable, and verifies it with targeted tests plus redacted real plugin-run proof.

## Reproduction Assessment

Yes, at source level: current main passes `sessionKey` and `agentId` in `PluginHookAgentContext`, but the `before_agent_start` and `agent_end` event objects still omit those fields. I did not run a live memory plugin in this read-only review.

## Solution Assessment

No, not as-is: adding optional event fields is a narrow compatible direction, but the branch must be rebased onto the split hook contracts, the test typing issue must be fixed, and real after-fix plugin-run proof must be provided before merge.
