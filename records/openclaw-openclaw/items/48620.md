## Summary

Keep open: current main and v2026.6.6 still do not provide an admin CLI inventory for approved WhatsApp contacts with merged DM/group permissions and last-active state. The narrower approved-pairing issue overlaps the list/revoke slice, but it does not cover this broader contact, policy, and activity view.

## What This Changes

The issue requests an admin-facing way to list and manage approved WhatsApp contacts, their permissions, and last-active state.

## Best Possible Solution

Define one account-scoped admin inventory path that coordinates approved-pairing management with existing `/allowlist`, channel status, and directory surfaces instead of adding a competing source of truth.

## Reproduction Assessment

Yes from source for the missing surface: current `openclaw pairing list` lists pending requests, `directory peers list` is ID/name lookup, and approved pairing storage has no policy or activity metadata. This is not a bug reproduction; it is a verified missing feature path.

## Solution Assessment

Unclear until maintainers settle the product and security semantics. A conservative account-aware CLI inventory is likely the right direction, but it should be coordinated with https://github.com/openclaw/openclaw/issues/56621 and avoid duplicating `/allowlist`.
