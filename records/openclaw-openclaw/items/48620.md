## Summary

Keep open: current main still lacks an admin-facing contacts inventory for approved WhatsApp senders, permissions, and per-contact activity. The related approved-pairing management request is useful and open, but it is narrower than the broader permissions/activity view requested here.

## What This Changes

Request an admin CLI to list/manage approved WhatsApp contacts with DM/group policies and last-active information instead of manually editing configuration files.

## Best Possible Solution

Add a read-only, account-scoped admin inventory that merges configured allowFrom and pairing-store approvals with explicit DM policy, group policy, and activity semantics, while coordinating approved-list/revoke scope with https://github.com/openclaw/openclaw/issues/56621.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source and docs inspection on current main shows no contacts/admin inventory command, and `pairing list` remains pending-only.

## Solution Assessment

Unclear because no patch is under review. A conservative read-only CLI surface is likely the narrowest maintainable direction, but maintainers should decide naming, privacy, activity semantics, and overlap with approved-pairing work first.
