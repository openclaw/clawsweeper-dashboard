## Summary

Keep open: current main and the latest release still lack an admin-facing contacts inventory for approved WhatsApp senders, permissions, and activity. The related approved-pairing CLI request is still open and narrower, while other related WhatsApp directory/policy issues cover different parts rather than superseding this broader admin view.

## What This Changes

The issue requests a CLI command to list and manage paired WhatsApp contacts with DM policy, group policy, and last-active information.

## Best Possible Solution

Add a read-only, account-scoped admin inventory that merges configured allowFrom and pairing-store approvals with clearly defined DM/group policy and activity fields, while coordinating approved-list/revoke work with https://github.com/openclaw/openclaw/issues/56621.

## Reproduction Assessment

Not applicable as a feature request. Source and docs inspection on current main, plus the v2026.5.12 release tag, show no contacts/admin inventory command and `pairing list` remains pending-only.

## Solution Assessment

Unclear because no patch is under review. A conservative read-only CLI surface is likely the narrowest maintainable direction, but maintainers should decide naming, privacy, activity semantics, and overlap with related approved-pairing work first.
