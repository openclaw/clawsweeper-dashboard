## Summary

Keep open: current main and the latest release still lack the requested admin CLI inventory for approved WhatsApp contacts, merged permission policy, and last-active state. Existing pairing, directory, channel-status, and chat allowlist surfaces cover adjacent pieces but do not solve the central request.

## What This Changes

The issue requests a shell admin command to list/manage paired WhatsApp contacts with sender ID, DM/group permission policy, and last-active information.

## Best Possible Solution

Add an account-scoped admin inventory, coordinated with approved-pairing list/revoke work, that merges config allowlists and pairing-store approvals while defining redaction, authorization, and last-active semantics.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source, docs, live issue context, related issue search, and release-tag inspection show the requested contacts inventory surface is not present on current main or v2026.5.22.

## Solution Assessment

Unclear until maintainers settle product and security semantics. A conservative account-aware CLI inventory is likely the right direction, but it should be coordinated with https://github.com/openclaw/openclaw/issues/56621 and avoid duplicating the existing chat `/allowlist` source of truth.
