## Summary

Keep open: current main and the latest release still lack a terminal admin inventory for approved WhatsApp contacts with merged permission state and last-active data. Existing pairing, directory, channel-status, and in-chat allowlist surfaces cover pieces of the workflow, but this remains a security-sensitive feature needing maintainer product direction.

## What This Changes

The issue requests an admin CLI view for approved WhatsApp contacts, their DM/group permission state, and last-active information.

## Best Possible Solution

Define one account-scoped admin inventory path that coordinates approved-pairing management with existing `/allowlist`, channel status, and directory surfaces instead of adding a competing source of truth.

## Reproduction Assessment

Yes, at source level for the missing surface: current `openclaw pairing list` lists pending requests, `directory peers list` is ID/name lookup, and approved pairing storage has no policy or activity metadata. This is a verified missing feature path rather than a broken existing CLI contract.

## Solution Assessment

Unclear until maintainers settle the product and security semantics. A conservative account-aware CLI inventory is plausible, but it should be coordinated with https://github.com/openclaw/openclaw/issues/56621 and avoid duplicating `/allowlist`.
