## Summary

Keep open: current main and the latest release still do not provide an admin-facing contacts inventory for approved WhatsApp senders, permissions, and activity. The narrower approved-pairing management request remains open and related, but it does not cover the broader per-contact permissions/activity view requested here.

## What This Changes

The issue requests a CLI/admin surface to list paired or connected WhatsApp senders with DM policy, group policy, and last-active details.

## Best Possible Solution

Add a read-only, account-scoped admin inventory that merges configured allowFrom and pairing-store approvals with clearly defined DM policy, group policy, and activity semantics, while coordinating approved-list/revoke work with https://github.com/openclaw/openclaw/issues/56621.

## Reproduction Assessment

Not applicable as a feature request. Source/docs inspection on current main and `v2026.5.12` shows no contacts/admin inventory command, and `pairing list` remains pending-only.

## Solution Assessment

Unclear because no patch is under review. A conservative read-only CLI surface is likely the narrowest maintainable direction, but maintainers should decide naming, privacy, activity semantics, and overlap with related approved-pairing work first.
