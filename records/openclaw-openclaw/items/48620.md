## Summary

Keep open: current main and v2026.6.6 still lack the requested admin CLI inventory for approved WhatsApp contacts, merged permission policy, and last-active state. The open approved-pairing issue overlaps the list/revoke part, but it does not fully cover the broader contact/permission/activity inventory requested here.

## What This Changes

The issue requests an admin CLI surface to list and manage paired WhatsApp contacts, their DM/group permissions, and last-active state.

## Best Possible Solution

Define one account-scoped admin inventory path that coordinates approved-pairing management with existing `/allowlist`, channel status, and directory surfaces instead of adding a competing source of truth.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection verifies the gap: current main has pending pairing, in-chat allowlist management, directory lookup, and channel status, but no CLI command that lists approved WhatsApp contacts with merged permissions and last-active state.

## Solution Assessment

Unclear until maintainers settle the product and security semantics. A conservative account-aware CLI inventory is likely the right direction, but it should be coordinated with https://github.com/openclaw/openclaw/issues/56621 and avoid duplicating the existing `/allowlist` source of truth.
