## Summary

Keep open: current main and latest release still do not provide an admin-facing contacts inventory for approved WhatsApp senders, permission policy, and activity. The adjacent directory and `/allowlist` surfaces are useful context, but they do not satisfy the requested shell CLI workflow, and the related approved-pairing management issue is narrower and still open.

## What This Changes

Request a CLI/admin surface to list paired WhatsApp contacts with sender ID, DM policy, group policy, and last-active information.

## Best Possible Solution

Add a deliberately scoped, account-aware admin inventory that merges config allowlists and pairing-store approvals with defined DM/group policy and activity semantics, while coordinating approved-list/revoke mechanics with https://github.com/openclaw/openclaw/issues/56621.

Not applicable as a bug reproduction; this is a feature request. Source, docs, release-tag inspection, and live issue context show the requested contacts inventory command is not present on current main or v2026.5.20.

## Solution Assessment

Unclear until maintainers settle command naming, privacy, and last-active semantics. A read-only CLI inventory is likely the narrowest maintainable direction, but the approved-pairing commands tracked at https://github.com/openclaw/openclaw/issues/56621 should be coordinated first.
