## Summary

Current main still lacks the requested admin-facing view for approved WhatsApp contacts, permissions, and activity; the adjacent approved-pairing request is narrower and still open, so this should remain open for maintainer scoping.

## What This Changes

Request a CLI/admin surface such as `openclaw contacts list` that shows paired WhatsApp senders, DM policy, group policy, and last active state.

## Best Possible Solution

Add a read-only, account-scoped admin inventory that merges configured `allowFrom`, pairing allow-store entries, WhatsApp DM/group policy state, and a documented activity signal; keep revoke/edit actions separately scoped unless maintainers intentionally combine them.

## Reproduction Assessment

Not applicable as a feature request. The missing surface is source-verifiable on current main: there is no `contacts` command, `pairing list` is pending-only, and directory/status outputs do not combine approved senders with policies and activity.

## Solution Assessment

Unclear because no patch is under review. A conservative read-only list command is the narrowest likely direction, but maintainers should first settle naming, account scoping, privacy posture, and overlap with #56621.
