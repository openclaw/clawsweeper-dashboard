## Summary

Keep open: current main still lacks a single admin-facing view for approved WhatsApp contacts, their DM/group access state, and activity. The available surfaces remain split between pending pairing requests, allow-store/config state, channel status, and directory lookup helpers; related #56621 is narrower and still open.

## What This Changes

Request an admin CLI surface to list or manage paired WhatsApp contacts with sender ID, DM policy, group policy, and last-active information.

## Best Possible Solution

Add a read-only, account-scoped admin view that merges configured `allowFrom`, pairing allow-store entries, effective `dmPolicy`, effective WhatsApp group sender/group membership policy, and any reliable per-sender activity signal. Keep revoke/edit behavior scoped to the narrower pairing-management work unless maintainers intentionally combine the surfaces.

## Reproduction Assessment

Not applicable as a runtime bug, but the missing feature is high-confidence by source inspection: current main has no `contacts` CLI command, `pairing list` reads pending requests only, and directory/status surfaces do not combine approved senders with DM/group permissions and activity.

## Solution Assessment

Unclear until maintainers settle product/API boundaries. A conservative read-only list command is the narrowest maintainable starting point, but naming, privacy posture, account scoping, and coordination with #56621 need human scoping first.
