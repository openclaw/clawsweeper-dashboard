## Summary

Keep open: current main and latest release still lack an admin inventory for approved WhatsApp contacts with merged permission state and last-active data. Existing pairing, directory, channel status, and `/allowlist` surfaces cover pieces of the workflow, but this remains a security-sensitive feature that needs maintainer product direction.

## What This Changes

The issue requests an admin-facing way to list and manage approved WhatsApp contacts, their permissions, and last-active metadata without manually editing configuration files.

## Best Possible Solution

Define one account-scoped admin inventory and permission-management surface that coordinates approved-pairing state, configured allowlists, group policy, and directory lookup without duplicating `/allowlist`.

## Reproduction Assessment

Yes, at source level: current `pairing list`, `directory peers list`, and `channels status` do not expose approved WhatsApp contacts with merged permission state or last-active data.

## Solution Assessment

Unclear until maintainers choose the product and security semantics. A CLI inventory is plausible, but it should be account-scoped and coordinated with https://github.com/openclaw/openclaw/issues/56621 and `/allowlist`.
