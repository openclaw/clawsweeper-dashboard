## Summary

The report is source-reproducible on current main: cached Feishu clients are reused after the test runtime helper swaps the SDK, so this should remain open for a narrow fix and regression test.

## What This Changes

`setFeishuClientRuntimeForTest` should invalidate cached Feishu clients when it swaps the Feishu SDK used by tests.

## Best Possible Solution

Clear the Feishu client cache inside `setFeishuClientRuntimeForTest` and add a focused regression test that proves an SDK swap recreates a cached client.

## Reproduction Assessment

Yes from source inspection, though not from an executed test in this read-only review. Create a Feishu client for an account, swap the SDK with `setFeishuClientRuntimeForTest`, then call `createFeishuClient` again with the same cache key; current code returns the cached client.

## Solution Assessment

Yes. Cache invalidation inside the test runtime helper is the narrowest maintainable fix because cached clients are constructed with the previous SDK and the helper is already the test-only reset point.
