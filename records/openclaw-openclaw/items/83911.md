## Summary

Keep open: current main and the latest release still leave the Feishu test runtime setter swapping the SDK without evicting cached clients. The linked fix PR at https://github.com/openclaw/openclaw/pull/84166 uses closing syntax for this issue, so the issue should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue requests clearing the Feishu client cache when `setFeishuClientRuntimeForTest` swaps the SDK so tests do not reuse clients built with a previous SDK.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/84166, or an equivalent narrow change that clears the cache in the test runtime setter and adds regression coverage.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: create a cached Feishu client, call `setFeishuClientRuntimeForTest` with a replacement SDK, then call `createFeishuClient` again with the same cache key and credentials; current main returns the cached client.

## Solution Assessment

Yes. Calling the existing `clearClientCache()` helper from the test-only runtime setter is the narrowest maintainable fix, and the linked PR adds the expected regression coverage.
