## What This Changes

The issue reports that `setFeishuClientRuntimeForTest` swaps the Feishu SDK test runtime without evicting cached clients created with the previous SDK.

## Best Possible Solution

Review and land or close https://github.com/openclaw/openclaw/pull/84166; the desired end state is cache eviction inside the Feishu test runtime setter plus regression coverage for SDK swaps.

## Reproduction Assessment

Yes. From source inspection, create a Feishu client, call `setFeishuClientRuntimeForTest` with a replacement SDK, then call `createFeishuClient` again with the same account and credentials; current main returns the cached client built with the previous SDK.

## Solution Assessment

Yes. Calling the existing `clearClientCache()` helper inside the test-only runtime setter is the narrowest maintainable solution, and the linked PR implements that shape with regression coverage.
