The issue asks for `setFeishuClientRuntimeForTest` to clear Feishu cached clients when swapping the SDK so tests do not reuse clients built with the prior SDK.

Review and land https://github.com/openclaw/openclaw/pull/84166 if it stays correct, clearing the cache in the test runtime setter while leaving production Feishu client caching unchanged.

## Reproduction Assessment

Yes, this is source-reproducible: create a cached Feishu client, swap the SDK with `setFeishuClientRuntimeForTest`, then call `createFeishuClient` again with the same account and credentials; current code returns the cached client before using the new SDK.
