## Summary

This should stay open: current main still leaves the Feishu client cache intact when the test runtime SDK is swapped, and an open linked fix PR is already the right implementation path to review before closing the issue.

## What This Changes

The issue reports that Feishu's test SDK reset helper leaves `clientCache` intact, so later `createFeishuClient` calls with the same account and credentials can return a stale client.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/84166 if its diff remains correct; otherwise apply the same cache eviction and regression-test shape directly in the Feishu client test surface.

## Reproduction Assessment

Yes. From source inspection, create a Feishu client, swap the SDK through `setFeishuClientRuntimeForTest`, then create the same account again; current main returns the cached client before the new SDK can be used.

## Solution Assessment

Yes. Clearing the Feishu client cache inside the test runtime helper is the narrowest maintainable fix because cached clients are constructed with the previous SDK and the helper is already the test reset point.
