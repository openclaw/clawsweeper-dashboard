## What This Changes

The PR changes macOS Nix-mode defaults resolution to skip creating a stable `UserDefaults(suiteName:)` when the stable suite equals the running bundle identifier, and adds Swift tests for same-bundle and different-bundle behavior.

## Best Possible Solution

Land this narrow defaults-domain fix after redacted macOS runtime launch proof confirms the warning is gone while the alternate-bundle stable-suite path still works.

## Reproduction Assessment

Yes, there is a high-confidence source-level reproduction path: current main and v2026.6.8 pass `ai.openclaw.mac` to `UserDefaults(suiteName:)` while the app bundle ID is also `ai.openclaw.mac`, and the linked issue includes macOS launch logs. I did not live-run the macOS app in this read-only Linux review.

## Solution Assessment

Yes, the code shape appears to be the best narrow fix: avoid only same-name suite construction and preserve stable-suite lookup when the bundle identifier differs. The remaining blocker is real macOS runtime proof, not a better implementation layer.
