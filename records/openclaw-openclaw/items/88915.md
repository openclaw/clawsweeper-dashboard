## What This Changes

The PR changes macOS Nix-mode defaults lookup to skip opening a UserDefaults suite when the stable suite name matches the running bundle identifier, and adds focused Swift tests for same-bundle and different-bundle behavior.

## Best Possible Solution

Land this narrow defaults-domain fix after redacted macOS runtime launch proof, or equivalent maintainer/Mantis proof, shows the warning is gone while the distinct-suite Nix fallback remains covered.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: current main and v2026.6.6 use ai.openclaw.mac as both the bundle identifier and stable defaults suite, then construct UserDefaults(suiteName: launchdLabel) in the launch-time Nix-mode path. I did not live-run the macOS app in this read-only review.

## Solution Assessment

Yes, this is the best narrow fix shape: skip only the same-name suite and preserve the stable-suite fallback when the bundle identifier differs. The remaining blocker is real macOS launch proof, not a better code path.
