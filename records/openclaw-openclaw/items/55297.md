## Summary

Keep open. Current main has explicit skill and plugin update paths plus a package auto-updater, but it does not provide background auto-updates for installed third-party skills/plugins; adding that remains a feature with config, product, and supply-chain policy implications.

## What This Changes

Request adding opt-in background auto-updates for ClawHub-installed skills and installed plugins on gateway start or a configurable interval.

## Best Possible Solution

Ship a maintainer-approved opt-in update policy that reuses existing skill/plugin update helpers with explicit consent, cadence controls, pinning or allowlist semantics, rollback guidance, failure reporting, and user-facing security documentation.

## Reproduction Assessment

Not applicable as a feature request. Current-main source and docs show manual skill/plugin update paths and a package-scoped auto-updater, but no startup or interval updater for installed skills/plugins.

## Solution Assessment

Unclear until maintainers decide the policy. Reusing `updateSkillsFromClawHub`, `updateNpmInstalledPlugins`, and the existing update scheduler is plausible, but silently applying third-party code updates without threat-model controls would not be the best solution.
