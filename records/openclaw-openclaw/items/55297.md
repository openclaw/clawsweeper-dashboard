## Summary

Keep open. Current main has manual skill/plugin update commands and a core package auto-updater, but it still does not provide the requested opt-in background updater for installed ClawHub skills and plugins; the remaining blocker is product and supply-chain policy.

## What This Changes

Request to add an opt-in background auto-update mechanism for ClawHub-installed skills and plugins, likely configured in openclaw.json and run on gateway start or an interval.

## Best Possible Solution

Define an explicit opt-in design that reuses current install/update policy, provenance checks, pinning or rollback semantics, and separate skill/plugin scopes before adding any gateway background updater.
