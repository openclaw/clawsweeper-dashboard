## Summary

Keep open. Current main has manual skill and plugin update paths plus a core package auto-updater, but it still does not provide the requested opt-in background updater for installed third-party ClawHub skills/plugins; the remaining work needs product and supply-chain policy decisions.

## What This Changes

Request an opt-in background auto-updater for ClawHub skills and OpenClaw plugins so installed third-party packages can refresh on startup or on an interval without manual commands.

## Best Possible Solution

Ship a maintainer-approved opt-in update policy that reuses the existing manual update helpers while defining consent, cadence, pinning or allowlists, integrity/signing, rollback, audit metadata, and failure reporting.

## Reproduction Assessment

Not applicable as a feature request. Current-main source and docs show manual update helpers plus a separate OpenClaw package update scheduler, but no background updater for installed third-party skills/plugins.

## Solution Assessment

Unclear until maintainers decide the policy. Reusing `updateSkillsFromClawHub`, `updateNpmInstalledPlugins`, and the package auto-update scheduler is plausible, but unattended third-party code updates need explicit threat-model controls first.
