## Summary

Keep open: current main and latest release provide manual skill/plugin update commands and a core package auto-updater, but not the requested opt-in background updater for installed ClawHub skills and plugins.

## What This Changes

The issue requests opt-in automatic background updates for installed ClawHub skills and OpenClaw plugins, likely on gateway start or an interval.

## Best Possible Solution

Keep this open for a maintainer-owned design that, if accepted, adds opt-in unattended updates with ClawHub provenance/trust gates, installPolicy semantics, rollback/disable controls, and separate handling for core package, plugin, and skill update lifecycles.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing behavior report. Source and docs confirm current main has explicit manual update commands but no background skill/plugin updater.

## Solution Assessment

Unclear: native auto-update may be useful, but the proposed background updater is not ready as a code task until maintainers choose the trust, policy, rollback, and opt-in semantics. The current supported workaround remains explicit `openclaw skills update --all` and `openclaw plugins update --all` automation owned by the operator.
