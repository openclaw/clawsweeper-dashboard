## Summary

Keep open: current main and v2026.6.8 still provide manual skill/plugin update commands plus a core package auto-updater, but not the requested opt-in unattended updater for installed ClawHub skills and plugins. The remaining work is a maintainer product and supply-chain policy decision, not a conservative cleanup close.

## What This Changes

Request to add opt-in background auto-updating for installed ClawHub skills and plugins, configured in `openclaw.json` and run on gateway startup or an interval.

## Best Possible Solution

Keep this open for a maintainer-owned design that separates core, plugin, and skill lifecycles and only adds opt-in auto-updates with explicit trust, policy, pinning, rollback, audit, and failure semantics.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing behavior report. Source and docs confirm current main has explicit manual update commands and no skill/plugin auto-update config surface.

## Solution Assessment

Unclear: native auto-update may be useful, but the proposed background updater is not ready as a code task until maintainers choose the trust, policy, rollback, and opt-in semantics. The supported workaround remains operator-owned automation around `openclaw skills update --all` and `openclaw plugins update --all`.
