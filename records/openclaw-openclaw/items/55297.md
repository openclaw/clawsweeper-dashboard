## Summary

Keep open. Current main has manual skill and plugin update paths plus a package-scoped auto-updater, but it still does not provide the requested opt-in background updater for installed third-party skills/plugins; the remaining work needs maintainer product and supply-chain policy decisions.

## What This Changes

The issue requests an opt-in auto-update mechanism for installed ClawHub skills and OpenClaw plugins.

## Best Possible Solution

Ship a maintainer-approved opt-in update policy that reuses the existing manual update helpers while defining consent, cadence, pinning or allowlists, rollback, failure reporting, and security documentation.

## Reproduction Assessment

Not applicable as a feature request. Current-main source and docs show manual update helpers plus a separate OpenClaw package update scheduler, but no background updater for installed third-party skills/plugins.

## Solution Assessment

Unclear until maintainers decide the policy. Reusing `updateSkillsFromClawHub`, `updateNpmInstalledPlugins`, and the existing package update scheduler is plausible, but unattended third-party code updates need explicit threat-model controls first.
