## Summary

Keep open: current main and the latest release still provide manual skill/plugin update commands and a core package auto-updater, but not the requested opt-in unattended updater for installed ClawHub skills and plugins.

## What This Changes

The issue requests opt-in unattended auto-updates for installed ClawHub skills and OpenClaw plugins, likely via config and gateway-start or interval checks.

## Best Possible Solution

Keep this open for a maintainer-owned design that adds opt-in skill/plugin auto-updates only with explicit ClawHub trust gates, installPolicy semantics, pinning/rollback controls, and separate lifecycle handling for core, plugins, and skills.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing behavior report. Source and docs confirm current main has explicit manual update commands and no `plugins.autoUpdate` or `skills.autoUpdate` config surface.

## Solution Assessment

Unclear: native auto-update may be useful, but the proposed background updater is not ready as a code task until maintainers choose the trust, policy, rollback, and opt-in semantics. The supported workaround remains explicit `openclaw skills update --all` and `openclaw plugins update --all` automation owned by the operator.
