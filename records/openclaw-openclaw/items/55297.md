## Summary

Keep open. Current main has manual skill and plugin update paths plus a package-scoped auto-updater, but it still does not provide opt-in background auto-updates for installed third-party skills/plugins; the remaining work needs product and supply-chain policy decisions.

## What This Changes

The issue requests an opt-in configuration that automatically updates installed ClawHub skills and OpenClaw plugins on startup or a schedule.

## Best Possible Solution

Ship a maintainer-approved opt-in update policy that reuses existing manual update helpers while defining consent, cadence, pinning or allowlists, rollback, failure reporting, and security documentation.

## Reproduction Assessment

Not applicable as a feature request. Current-main source and docs show manual skill/plugin update paths and package-scoped self-update checks, but no background updater for installed skills/plugins.

## Solution Assessment

Unclear until maintainers decide the policy. Reusing `updateSkillsFromClawHub`, `updateNpmInstalledPlugins`, and the existing package update scheduler is plausible, but unattended third-party code updates need explicit threat-model controls first.
