## Summary

Keep open. Current main still has explicit skill/plugin update commands and a package-install auto-updater, but it does not implement the requested opt-in background updater for installed third-party ClawHub skills/plugins; the remaining blocker is product and supply-chain policy.

## What This Changes

Requested opt-in automatic background updates for installed ClawHub skills and OpenClaw plugins.

## Best Possible Solution

Ship a maintainer-approved opt-in update policy that reuses the existing manual update helpers while defining cadence, trust controls, rollback, audit metadata, and failure reporting.

## Reproduction Assessment

Not applicable as a feature request. Current-main source, docs, and tests show explicit skill/plugin update helpers plus a separate package auto-updater, but no background updater for installed third-party ClawHub skills/plugins.

## Solution Assessment

Unclear until maintainers decide the policy. Reusing `updateSkillsFromClawHub`, `updateNpmInstalledPlugins`, and the existing update scheduler is plausible, but unattended third-party code updates need explicit trust controls first.
