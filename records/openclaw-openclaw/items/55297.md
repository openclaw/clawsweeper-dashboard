## Summary

Keep open. Current main has explicit skill/plugin update commands and a separate core package auto-updater, but it still lacks the requested opt-in background updater for installed third-party ClawHub skills/plugins; the remaining decision is product and supply-chain policy.

## What This Changes

Requests an opt-in background auto-update mechanism for installed ClawHub skills and OpenClaw plugins.

## Best Possible Solution

Ship a maintainer-approved opt-in update policy that reuses existing manual update helpers while defining cadence, trust controls, rollback, audit metadata, and failure reporting.

## Reproduction Assessment

Not applicable as a feature request. Current-main source and docs show manual skill/plugin update helpers plus a separate core package auto-updater, but no background updater for installed third-party ClawHub skills/plugins.

## Solution Assessment

Unclear until maintainers decide the policy. Reusing `updateSkillsFromClawHub`, `updateNpmInstalledPlugins`, and the existing update scheduler is plausible, but unattended third-party code updates need explicit trust controls first.
