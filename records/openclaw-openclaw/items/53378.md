## Summary

Keep open: current main still has server-side session channel metadata and channel catalog data, but the Control UI Sessions table does not carry the channel fields through its browser row type/live-event merge path or render a channel label/icon.

## What This Changes

Request to show a recognizable channel name/icon in the Control UI sessions list, with optional custom labels for sessions.

## Best Possible Solution

Carry existing Gateway channel metadata into the browser session row contract and live-event merge path, pass channel catalog metadata into Sessions, and render a compact channel badge/label while preserving existing labels, search, sorting, pagination, selection, checkpoint expansion, and chat navigation.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a telegram-style row without agent identity is expected by the current test to render as the raw session key, and renderRows has no typed channel field or channel catalog input to resolve a label/icon.

## Solution Assessment

Yes. The narrow maintainable path is to reuse the existing Gateway session metadata and channels.status catalog; hardcoding channel names or icons in the Sessions view would duplicate plugin metadata.
