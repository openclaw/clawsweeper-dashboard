## Summary

Keep open. Current main still exposes channel metadata from the Gateway and channel catalog, but the Control UI Sessions row contract and renderer still do not carry or render a resolved channel label/icon for the session list.

## What This Changes

Request to show a channel label/icon, and optionally custom labels, in the Control UI Sessions list so channel-backed sessions are easier to distinguish.

## Best Possible Solution

Carry existing Gateway channel metadata through the Control UI row contract and live event merge, pass the existing channels.status catalog into the Sessions view, and render a stable channel label/badge/icon while preserving editable labels, search, sorting, pagination, selection, checkpoint controls, and chat navigation.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: a row such as `agent:main:telegram:direct:7633` reaches `renderRows` without typed `row.channel` or channel catalog props, so the table renders the raw key or an agent label with raw `(telegram)` instead of a resolved Telegram/Feishu label or icon.

## Solution Assessment

Yes. The best fix is the narrow UI/data-contract path that reuses existing Gateway session metadata and channels.status catalog data; hardcoding Telegram, Feishu, or bundled channel IDs in the renderer would be less maintainable.
