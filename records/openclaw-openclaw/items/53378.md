## Summary

Keep open. Current `main` still has the backend session channel metadata and channel catalog needed for this, but the Control UI Sessions table does not carry those fields through its browser row type/live-event merge path or render a channel label/icon in the session row.

## What This Changes

Request to show a channel name/icon in the Control UI session list, with optional custom session labels, so Telegram/Feishu-style sessions are not identified only by opaque raw keys.

## Best Possible Solution

Carry the existing Gateway channel metadata through the browser GatewaySessionRow contract and live-update allowlist, pass the existing channel catalog metadata into the Sessions view, and render a stable channel label/icon badge alongside the session key/display name while preserving custom labels, search, sorting, pagination, selection, and chat navigation.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: a row like `agent:main:telegram:direct:7633` reaches `renderRows`, and if no agent identity is available the table renders `row.key` because Sessions has no typed `row.channel` or channel catalog props to resolve Telegram/Feishu labels.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse the Gateway session metadata and `channels.status` catalog already present on current `main`; hardcoding channel names/icons in the Sessions table would be less durable.
