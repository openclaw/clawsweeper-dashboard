## What This Changes

The PR preserves existing non-empty session `label` and `displayName` fields when `upsertSessionEntry` receives replacement metadata with omitted, null, or empty title fields.

## Best Possible Solution

Land the store-boundary fix after maintainers accept the exported helper's new title-preservation contract or the PR documents the preserve/replace/clear semantics for plugin callers.

## Reproduction Assessment

Yes. Source inspection shows current main persists a cloned replacement entry in `upsertSessionEntry`, so an existing titled row loses `label` and `displayName` when the replacement omits or empties them.

## Solution Assessment

Yes, with a compatibility caveat. The store boundary is the narrowest durable fix for replacement writers, but maintainers should accept or document the exported SDK behavior change before merge.
