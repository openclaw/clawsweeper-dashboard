## Summary

Keep open. Current main has bounded, flat skills catalog behavior with on-demand `SKILL.md` reads, but it does not implement dependency edges, depth-based expansion, `skills.maxDepth` config, or `openclaw skills graph --dot`.

## What This Changes

The issue requests a Skill Graph that discovers skill dependencies, loads dependency chains on demand, exposes graph visualization, and adds configurable depth to reduce skill prompt token usage.

## Best Possible Solution

Define an additive skill dependency metadata or graph contract, expose CLI graph output only if maintainers accept the surface, then integrate it with the existing skills snapshot, prompt budget, config schema, and docs.

## Reproduction Assessment

Not applicable. This is a feature request, and source inspection confirms current main has only flat, budgeted skills catalog behavior with on-demand `SKILL.md` reads.

## Solution Assessment

Unclear. The direction is plausible, but parsing implicit references and adding `maxDepth` may not be the narrowest maintainable design until maintainers choose the dependency contract and how it aligns with broader prompt-overhead work.
