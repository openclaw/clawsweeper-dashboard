## Summary

Keep open. Current main still has a flat, budgeted skills catalog and one-selected-`SKILL.md` read path, but no public skill dependency metadata, dependency-depth expansion, DOT graph command, or `skills.maxDepth` config contract; #52861 overlaps the token-overhead motivation but does not replace this graph-specific design request.

## What This Changes

The issue requests dependency-aware skill graph loading, configurable expansion depth, and `openclaw skills graph --dot` output to reduce skill prompt token use.

## Best Possible Solution

Define a backward-compatible skill dependency metadata contract first, then implement graph construction, prompt expansion, DOT CLI output, config/schema/docs coverage, and focused tests in coordination with #52861.

Not applicable: this is a feature request rather than a bug report. The high-confidence check is current-main source and docs inspection, which shows the requested graph metadata, dependency-depth loading, DOT command, and depth config are absent.
