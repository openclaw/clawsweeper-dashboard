## Summary

Keep open. Current main has a bounded flat skills catalog and on-demand `SKILL.md` reads, but it does not implement a skill dependency graph, dependency-depth expansion, `skills.maxDepth`-style config, or `openclaw skills graph --dot`.

## What This Changes

Request a skill dependency graph that lazily expands skill dependencies, exposes DOT output, and adds max-depth configuration to reduce prompt token use.

## Best Possible Solution

Define an additive, documented skill dependency metadata or graph contract first, then integrate it with the skills snapshot, prompt budget, config schema, CLI docs, and related prompt-overhead work.

## Reproduction Assessment

Not applicable. This is a feature request, and source inspection confirms current main only has flat, budgeted skill catalog behavior with on-demand `SKILL.md` reads.

## Solution Assessment

Unclear. A graph may be useful, but implicit `sessions_spawn` or sub-skill parsing plus a depth knob is a product/API design choice; an explicit dependency metadata contract aligned with #52861 would be safer.
