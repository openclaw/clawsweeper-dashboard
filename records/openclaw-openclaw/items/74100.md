Keep open. Current main and v2026.6.10 still use a prompt-visible flat skills catalog with budget truncation, while this request would add new dependency graph metadata, runtime expansion semantics, `openclaw skills graph --dot`, and `skills.maxDepth`-style config that need maintainer product/API review.

Add a Skill Graph that parses skill dependencies, expands needed skill chains at runtime, exposes DOT graph output, and adds configurable skill-loading depth.

## Best Possible Solution
