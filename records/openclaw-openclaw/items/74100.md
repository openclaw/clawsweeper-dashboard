## Summary

Keep open. Current main has bounded progressive disclosure for a flat skills catalog, but it still lacks dependency-edge parsing, depth-based dependency expansion, `openclaw skills graph --dot`, and a `skills.maxDepth` or equivalent config contract; the related token-overhead issues overlap but do not fully replace this graph-specific proposal.

## What This Changes

The issue requests a Skill Graph feature that parses skill dependencies, loads dependency layers on demand with configurable depth, and exposes DOT graph output from the skills CLI.

## Best Possible Solution

Define a backward-compatible skill dependency metadata and graph-building contract first, then implement prompt expansion that respects existing eligibility, allowlists, prompt budgets, exposure rules, and docs, with DOT CLI output and config/schema coverage after maintainer approval.

Not applicable: this is a feature request, not a bug report. The high-confidence check is current-main source and docs inspection, which shows the requested dependency graph, DOT CLI, and depth config surfaces are absent.
