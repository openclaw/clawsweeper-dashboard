## Summary

Keep open. Current main still has bounded progressive disclosure for a flat skills catalog, but it does not implement dependency-edge parsing, depth-based dependency expansion, DOT graph output, or a `skills.maxDepth` config contract; the linked token-overhead issues overlap but do not fully replace this graph-specific proposal.

## What This Changes

The issue requests dependency-aware skill graph loading with automatic edge discovery, configurable expansion depth, and `openclaw skills graph --dot` output to reduce prompt token overhead.

## Best Possible Solution

Define a backward-compatible skill dependency metadata contract first, then implement graph construction and prompt expansion that respect existing eligibility, allowlists, prompt budgets, exposure rules, and docs, with DOT output and config/schema coverage after maintainer approval.
