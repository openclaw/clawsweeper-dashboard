## Summary

Keep open. Current main has on-demand `SKILL.md` reads plus compact/truncated skills-catalog fallback, but it does not implement dependency graph parsing, depth-limited skill expansion, or a `skills graph` CLI surface; the requested design overlaps open prompt-overhead work and needs maintainer product/API direction.

## What This Changes

Request to add a skill dependency graph that loads referenced skill chains on demand, exposes a max-depth setting, and provides `openclaw skills graph --dot` visualization.

## Best Possible Solution

Resolve the design through the existing skill-catalog and conditional-injection threads, then add a documented core discovery/narrowing contract or graph command only if maintainers choose dependency-based loading as the product direction.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that the requested graph, max-depth, and DOT-output surfaces are absent from current main.

## Solution Assessment

Unclear because no patch is proposed. A standalone dependency parser is not obviously the best fix until maintainers choose between core skill metadata, hook-based narrowing, and the existing skill-catalog overhead direction.
