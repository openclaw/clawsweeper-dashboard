## Summary

Keep open. Current main still implements a flat, budgeted `<available_skills>` catalog with one selected `SKILL.md` read up front; it does not implement dependency-edge parsing, layered dependency expansion, DOT graph output, or a `skills.maxDepth` config contract. The related token-overhead reports overlap with the motivation, but they do not replace this graph-specific API/product request.

## What This Changes

Request to add a Skill Graph feature that lazily expands skill dependency chains, exposes a DOT graph command, and adds a configurable dependency-depth limit.

## Best Possible Solution
