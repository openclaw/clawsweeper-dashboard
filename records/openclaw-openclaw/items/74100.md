## Summary

Keep open. Current main and v2026.6.6 still use a flat `available_skills` catalog with allowlists and compact/truncation budget fallbacks, but they do not implement dependency graph traversal, depth-limited expansion, `skills.maxDepth`, or `openclaw skills graph --dot`; the closed visualizer issue belongs to ClawHub/external tooling, while https://github.com/openclaw/openclaw/issues/89361 and https://github.com/openclaw/openclaw/pull/75871 overlap with smart indexing/progressive disclosure without safely replacing this graph-specific product decision.

## What This Changes

Request dependency-aware skill loading that expands only needed dependency layers, adds graph visualization output, and exposes a configurable maximum depth to reduce prompt token overhead.

## Best Possible Solution

Have maintainers choose one progressive skill-disclosure and dependency metadata contract, then split core runtime/API work from external visualization so graph traversal lands only if current plugin/ClawHub surfaces cannot express it.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection provides a high-confidence gap check that current main lacks dependency graph loading, graph CLI output, and depth config.

## Solution Assessment

Unclear. A dependency graph may be useful, but the metadata contract, depth semantics, prompt-cache behavior, rollout default, and overlap with smart indexing need maintainer design selection first.
