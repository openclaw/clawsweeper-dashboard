## Summary

Keep open. Current main has a flat, budgeted skills catalog with on-demand `SKILL.md` reads, but it does not implement a skill dependency graph, dependency-depth expansion, `skills.maxDepth` config, or `openclaw skills graph --dot`.

## What This Changes

Request a skill dependency graph that lazily expands dependent skills, adds configurable depth, and exposes DOT graph output to reduce prompt-token overhead.

## Best Possible Solution

Define an additive skill dependency metadata/graph contract, decide explicit versus inferred edges, then wire prompt selection, config/schema/docs, CLI DOT output, and focused regression tests around that contract.

## Reproduction Assessment

Not applicable. This is a feature request, and source inspection confirms current main has only a flat, budgeted `available_skills` catalog with on-demand reads rather than dependency graph expansion.

## Solution Assessment

Unclear. The direction is plausible, but parsing implicit references plus adding depth-based injection and graph output needs a maintainer-owned API/product design before implementation.
