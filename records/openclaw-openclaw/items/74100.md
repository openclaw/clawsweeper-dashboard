## Summary

Keep open. Current main still has a flat, budgeted available-skills catalog with on-demand `SKILL.md` reads, but it does not implement a skill dependency graph, dependency-depth expansion, `skills.maxDepth`, or `openclaw skills graph --dot`; related open items cover broader/lazier prompt narrowing but do not fully subsume this graph-specific request.

## What This Changes

The issue requests a skill dependency graph with on-demand dependency-chain expansion, configurable depth, and DOT graph output to reduce skill prompt token use.

## Best Possible Solution

Define an additive skill dependency metadata or resolver contract, then decide whether graph output and depth-limited expansion belong in core alongside the existing skills snapshot, prompt budget, config schema, prompt-cache policy, and docs.

## Reproduction Assessment

Not applicable. This is a feature/design request, and source inspection confirms current main has only flat catalog, budgeted compact fallback, and allowlist behavior rather than dependency graph loading.

## Solution Assessment

Unclear as a direct implementation plan. The requested direction is plausible, but parsing implicit skill references and adding `skills.maxDepth` are not clearly the safest contract until maintainers choose the dependency metadata/API and how it aligns with the existing prompt-overhead work.
