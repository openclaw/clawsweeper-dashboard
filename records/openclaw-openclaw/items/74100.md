## Summary

Keep open. Current main already uses a bounded available-skills catalog and on-demand `SKILL.md` reads, but it does not implement the requested skill dependency graph, dependency-depth expansion, `skills.maxDepth`-style config, or `openclaw skills graph --dot` surface.

## What This Changes

The issue requests a Skill Graph that discovers skill dependencies, loads dependency chains on demand, exposes graph visualization, and adds configurable depth to reduce skill prompt token usage.

## Best Possible Solution

Define an additive skill dependency metadata or graph contract, expose CLI graph output if maintainers accept it, then integrate it with the existing skills snapshot, prompt budget, config schema, and docs.

## Reproduction Assessment

Not applicable. This is a feature request, and source inspection instead confirms current main has only flat, budgeted skill catalog behavior with on-demand `SKILL.md` reads.

## Solution Assessment

Unclear. The requested direction is plausible, but parsing implicit references and adding `maxDepth` may not be the narrowest maintainable design until maintainers choose the dependency contract and how it should align with broader prompt-overhead work.
