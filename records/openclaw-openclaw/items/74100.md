## Summary

Keep open. Current main still implements a flat, budgeted skills catalog with one selected `SKILL.md` read, but it does not provide skill dependency metadata, dependency-depth expansion, DOT graph output, or a `skills.maxDepth` config contract; related prompt-overhead threads overlap the motivation but do not replace this graph-specific design request.

## What This Changes

The issue requests a skill dependency graph with on-demand dependency expansion, DOT visualization, and a configurable maximum depth to reduce prompt token cost.

## Best Possible Solution

Define a backward-compatible skill dependency metadata contract first, then implement graph construction, prompt expansion, DOT CLI output, config/schema/docs coverage, and focused tests in coordination with the broader prompt-overhead work.
