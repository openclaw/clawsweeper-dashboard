## Summary

Keep open. Current main still uses a prompt-visible skill catalog with compact/truncation limits, and this request adds new dependency-graph, depth, CLI, and config semantics that are not already implemented or safely replaceable by the related smart-indexing issue.

## What This Changes

The issue requests a skill dependency graph that lazy-loads dependency chains, can output a graph, and exposes configurable expansion depth to reduce prompt token use.

## Best Possible Solution

Decide one maintainer-approved progressive skill-disclosure design that covers dependency metadata, graph visualization, runtime loading semantics, prompt budget behavior, and how this relates to the open smart-indexing tracker.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request for new skill graph and loading behavior. Source inspection gives a high-confidence gap check that current main lacks dependency graph expansion, skills.maxDepth, and skills graph output.

## Solution Assessment

Unclear. The direction is plausible, but the requested graph/depth model should be reconciled with the open smart-indexing work before implementation so OpenClaw does not grow competing skill discovery mechanisms.
