## Summary

Keep open. Current main has bounded flat skill-catalog behavior with on-demand `SKILL.md` reads, but it does not implement the requested skill dependency graph, dependency-depth expansion, `skills.maxDepth`-style config, or `openclaw skills graph --dot`; the broader #52861 tracker is related but does not replace this specific graph request.

## What This Changes

Request adds a skill dependency graph that lazily expands dependent skills, exposes DOT graph output, and adds a configurable dependency-depth limit to reduce prompt tokens.

## Best Possible Solution

Define an additive skill dependency metadata contract and graph traversal semantics, then wire it through skill snapshots, prompt budgeting, config schema/help, CLI graph output, docs, and prompt-cache/security rules.

## Reproduction Assessment

Not applicable. This is a feature request; source inspection confirms current main has flat, budgeted skill catalog behavior rather than a failing bug path.

## Solution Assessment

Unclear. A dependency graph may be viable, but implicit parsing of `sessions_spawn` or sub-skill references plus a depth knob needs maintainer design before it is the narrowest maintainable solution.
