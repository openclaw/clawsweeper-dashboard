## Summary

Keep open. Current main has bounded progressive disclosure for a flat eligible-skills catalog, but it still lacks dependency-edge parsing, depth-based expansion, DOT graph output, and a `skills.maxDepth` config contract; the related token-overhead issues overlap but do not replace this specific graph/API proposal.

## What This Changes

Request to add dependency-aware skill graph loading, configurable expansion depth, and DOT visualization to reduce skill prompt token cost.

## Best Possible Solution

Define a backward-compatible skill dependency metadata and discovery contract first, then implement graph construction and prompt expansion that respect existing gating, allowlists, prompt budgets, and exposure rules, with CLI DOT output plus config/schema/docs coverage.
