## Summary

Keep open. Current main and the latest release still use a flat prompt-visible skills catalog with allowlists and compact/truncation budget fallbacks, but they do not implement dependency graph traversal, depth-limited expansion, `skills.maxDepth`, or `openclaw skills graph --dot`; https://github.com/openclaw/openclaw/issues/89361 overlaps on smart indexing but does not supersede this graph-specific product decision.

## What This Changes

Request adding a Skill Graph that parses skill dependencies, dynamically expands only needed dependency depths, exposes `openclaw skills graph --dot`, and adds a `skills.maxDepth` config option.

## Best Possible Solution

Maintainers should choose one canonical progressive skill-disclosure and dependency metadata contract, split core runtime/API work from ClawHub-only visualization, and preserve current prompt compatibility by default.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives a high-confidence gap check that current main lacks dependency graph loading, graph CLI output, and depth config.

## Solution Assessment

Unclear. A dependency graph may be useful, but the metadata contract, depth semantics, prompt-cache behavior, rollout default, and overlap with smart indexing need maintainer design selection first.
