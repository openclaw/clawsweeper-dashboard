## Summary

Keep open. Current main and the latest release still use a flat prompt-visible skills catalog with allowlists and compact/truncation budget fallbacks, but they do not implement dependency graph traversal, depth-limited expansion, `skills.maxDepth`, or `openclaw skills graph --dot`; related smart-indexing and visualizer threads overlap but do not safely replace this graph-specific product decision.

## What This Changes

The issue requests a skill dependency graph that can parse skill-to-skill relationships, lazily expand dependency depth at runtime, expose graph output from the skills CLI, and configure max loading depth.

## Best Possible Solution

Maintainers should choose one canonical progressive skill-disclosure and dependency metadata contract, then split core runtime/API work from ClawHub/external visualization before implementation starts.
