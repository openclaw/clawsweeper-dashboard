## Summary

Keep open. Current main still implements a flat, budgeted skills catalog with compact/truncation fallback and a one-selected-SKILL.md prompt contract, but it does not define skill dependency edges, depth-based dependency expansion, a DOT graph CLI command, or a `skills.maxDepth` config contract. The related token-overhead work in #52861 overlaps the motivation, but this issue adds graph-specific public API, prompt semantics, CLI, config, and docs design that still needs maintainer judgment.

## What This Changes

Request to add a skill dependency graph that lazily expands dependent skills at runtime, exposes DOT visualization, and adds a configurable max dependency depth to reduce skill prompt token cost.

## Best Possible Solution

Define a backward-compatible skill dependency metadata contract first, then implement graph construction and prompt expansion in the skills subsystem with DOT CLI output, config/schema/docs coverage, and explicit integration with the broader conditional tool/skill injection work tracked in #52861.

## Reproduction Assessment

Not applicable: this is a feature request rather than a bug report. The high-confidence check is current-main source, docs, tests, and related-item inspection, which shows the requested graph, dependency metadata, DOT command, and depth config are absent.

## Solution Assessment

Unclear because there is no patch attached. The requested direction is plausible, but the maintainable path is a maintainer-approved design before adding graph traversal, prompt expansion, config, and CLI behavior.
