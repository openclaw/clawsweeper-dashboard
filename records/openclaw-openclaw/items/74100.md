## Summary

Keep open. Current main has flat, budgeted skills prompt disclosure and on-demand reading of one selected `SKILL.md`, but it still lacks skill dependency metadata, dependency-depth traversal, `openclaw skills graph --dot`, and a `skills.maxDepth` config contract; #52861 overlaps the token-cost motivation but does not cover this graph-specific request.

## What This Changes

Add a skill dependency graph that lazily loads dependency layers, emits DOT output, and exposes a max-depth config to reduce prompt token use.

## Best Possible Solution

Define a backward-compatible skill dependency metadata contract first, then implement graph construction, prompt expansion, DOT CLI output, config/schema/docs coverage, and focused tests in coordination with #52861.
