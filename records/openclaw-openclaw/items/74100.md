## Summary

Keep open. Current main still has a flat, budgeted skills catalog with one selected `SKILL.md` read path, but no public skill dependency metadata, dependency-depth expansion, DOT graph command, or `skills.maxDepth` config contract; the broader token-overhead work in #52861 overlaps the motivation but does not replace this graph-specific design request.

## What This Changes

The issue requests dependency-aware skill loading, `openclaw skills graph --dot`, and a configurable traversal depth to reduce skill prompt token overhead.

## Best Possible Solution

Define a backward-compatible skill dependency metadata contract first, then implement graph construction, prompt expansion, DOT CLI output, config/schema/docs coverage, and focused tests in coordination with the token-overhead work in #52861.
