## Summary

Keep open. Current main has a bounded flat skills catalog with compact/truncated prompt fallback, but it still lacks dependency metadata, dependency-depth expansion, a DOT graph command, and a `skills.maxDepth` config contract; #52861 overlaps the token-overhead motivation but does not replace this graph-specific design request.

## What This Changes

The issue requests a skill dependency graph that loads dependency chains on demand, supports depth configuration, and exposes `openclaw skills graph --dot`.

## Best Possible Solution

Define a backward-compatible skill dependency metadata contract first, then implement deterministic graph construction, prompt expansion, DOT CLI output, config/schema/docs coverage, and focused tests in coordination with #52861.

Not applicable: this is a feature request rather than a bug report. The high-confidence check is current-main source and docs inspection, which shows the requested graph, dependency metadata, DOT command, and depth config surfaces are absent.

## Solution Assessment

Unclear: there is no patch attached, and the requested direction needs maintainer-approved API/product design before traversal and prompt expansion behavior are added. A maintainable implementation should start in the skills subsystem and avoid changing unrelated tool injection behavior.
