## Summary

Keep open. Current main still has a flat, budgeted skills catalog and one selected SKILL.md read path, but no dependency-edge metadata, depth-based expansion, DOT graph command, or skills.maxDepth config contract. The related token-overhead issue #52861 overlaps the motivation, but it does not cover this graph-specific API and CLI design.

## What This Changes

Request a Skill Graph feature that parses skill dependency references, lazily expands dependency layers by configurable depth, and exposes a DOT graph command to reduce prompt token use.

## Best Possible Solution

Define a backward-compatible dependency metadata contract first, then implement graph construction and prompt expansion in the skills subsystem with DOT CLI output, config/schema/docs coverage, and integration with the broader token-overhead work tracked in #52861.
