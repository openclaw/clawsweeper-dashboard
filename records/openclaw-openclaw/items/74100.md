## Summary

Keep open. Current main still has a flat, budget-limited skills catalog and on-demand `SKILL.md` reads, but it does not implement a skill dependency graph, dependency-depth expansion, `skills.maxDepth`, or `openclaw skills graph --dot`; related open items cover broader prompt overhead, runtime skill discovery, or lazy catalog loading without fully owning this graph-specific design.

## What This Changes

The issue requests a skill dependency graph with on-demand dependency-chain loading, graph visualization, and a `skills.maxDepth` configuration option.

## Best Possible Solution

Define an additive skill dependency metadata or resolver contract, then decide whether graph output and depth-limited expansion belong in core alongside the existing skills snapshot, prompt budget, config schema, prompt-cache policy, and docs.

## Reproduction Assessment

Not applicable. This is a feature/design request, and source inspection gives a high-confidence current-main absence check for dependency graph loading, depth config, and graph CLI support.

## Solution Assessment

Unclear as a direct implementation plan. The direction is plausible, but parsing implicit skill references and adding `skills.maxDepth` are public API/product choices that should follow a maintainer-approved dependency contract.
