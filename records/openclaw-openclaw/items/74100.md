## Summary

Keep open. Current main still has flat, budgeted skill discovery plus a one-selected-`SKILL.md` read path, but it does not provide skill dependency metadata, dependency-depth traversal, a `skills graph --dot` command, or a `skills.maxDepth` config contract; #52861 overlaps the token-overhead motivation but does not replace this graph-specific request.

## What This Changes

Request to add a skill dependency graph so OpenClaw can lazily expand dependent skills by depth, emit DOT graph output, and configure dependency depth.

## Best Possible Solution

Define a backward-compatible skill dependency metadata contract first, then implement graph construction, prompt expansion, DOT CLI output, config/schema/docs coverage, and focused tests in coordination with #52861.

## Reproduction Assessment

Not applicable: this is a feature request rather than a bug report. The high-confidence check is current-main source and docs inspection, which shows the requested graph, dependency metadata, DOT command, and depth config surfaces are absent.

## Solution Assessment

Unclear: there is no patch attached, and the proposed direction spans public metadata, prompt behavior, CLI, config, and docs. A maintainable solution should start in the skills subsystem with an approved contract and avoid unrelated tool-injection changes.
