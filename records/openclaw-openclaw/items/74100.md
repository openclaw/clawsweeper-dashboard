## Summary

Keep open. Current main has a flat, budgeted skills catalog with on-demand reading of one selected SKILL.md, but it still has no skill dependency-edge metadata, depth-based dependency expansion, DOT graph CLI output, or `skills.maxDepth` config contract. The broader token-overhead item #52861 overlaps the motivation, and #73248 covered compact catalog pressure, but neither supersedes this graph-specific feature request.

## What This Changes

The issue requests dependency-aware skill loading that builds a skill graph, expands only needed dependency layers at runtime, exposes DOT graph output, and adds a configurable max-depth setting to reduce prompt token cost.

## Best Possible Solution

Define a backward-compatible skill dependency metadata contract first, then implement graph construction and prompt expansion in the skills subsystem with DOT CLI output, config/schema/docs coverage, and alignment with the broader token-overhead work tracked in #52861.

Not applicable: this is a feature request rather than a bug report. The high-confidence check is current-main source, docs, and CLI/config inspection, which shows the requested graph, DOT command, dependency metadata, and depth config surfaces are absent.
