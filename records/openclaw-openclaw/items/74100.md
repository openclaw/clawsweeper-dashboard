## Summary

Keep open. Current main still has flat skill prompt discovery with budget-based compact/truncation fallback, but it does not implement dependency graph traversal, depth-limited runtime expansion, `skills.maxDepth`, or `openclaw skills graph --dot`; the closed progressive-disclosure PR and the adjacent open indexing/visualizer issues do not fully replace this graph-specific product decision.

## What This Changes

Request to add dependency-graph-driven skill loading, a configurable dependency depth, and a graph export command to reduce skill prompt token overhead.

## Best Possible Solution

Have maintainers choose one progressive skill-disclosure contract, then split core runtime changes from ecosystem-only visualization so dependency traversal lands only if the accepted design needs a core API.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives a high-confidence absence check for dependency graph loading, depth config, and graph CLI support on current main.

## Solution Assessment

Unclear as written. A dependency graph may be useful, but the metadata contract, depth semantics, prompt-cache behavior, rollout default, and overlap with smart indexing need maintainer design selection first.
