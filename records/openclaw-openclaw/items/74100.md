## Summary

Keep open. Current main has bounded progressive disclosure for a flat skills catalog and on-demand `SKILL.md` reads, but it does not implement dependency-edge metadata, dependency-depth expansion, DOT graph output, or a `skills.maxDepth` config contract; related token-overhead issues overlap but do not supersede this graph-specific feature request.

## What This Changes

The issue requests dependency-graph based skill loading with depth-limited expansion, DOT graph output, and a `skills.maxDepth` config option.

## Best Possible Solution

Define a backward-compatible skill dependency metadata contract, then implement graph construction and prompt expansion that preserve existing eligibility, allowlists, prompt budgets, exposure rules, and docs, followed by CLI DOT output and config/schema coverage if maintainers approve the product direction.

## Reproduction Assessment

Not applicable: this is a feature request, not a bug report. The high-confidence check is current-main source and docs inspection, which shows the requested graph, DOT CLI, dependency metadata, and depth config surfaces are absent.

## Solution Assessment

Unclear because there is no proposed patch. The direction is plausible, but the maintainable route should prefer an explicit dependency contract and documented traversal semantics over ad hoc parsing before implementation.
