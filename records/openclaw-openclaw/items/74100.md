## Summary

Keep open. Current main still has budgeted flat skill discovery and one selected `SKILL.md` read path, but it does not expose skill dependency metadata, dependency-depth expansion, a `skills graph --dot` command, or a `skills.maxDepth` config contract; the broader token-overhead issue overlaps the motivation but does not replace this graph-specific design request.

## What This Changes

Requests a skill dependency graph with on-demand dependency expansion, DOT output, and a `skills.maxDepth` config knob to reduce prompt token cost.
