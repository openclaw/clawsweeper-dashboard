## Summary

Keep open: current main still lacks proactive session archive cleanup, and this PR is in the right area but is not mergeable because the sweeper resolves default maintenance instead of the loaded config and its tests mock a different helper than production imports.

## What This Changes

The branch adds a gateway session-archive sweeper wired to startup/hourly maintenance, artifact helper tests, a changelog entry, and broad formatting-only churn across UI/plugin/test files.

## Best Possible Solution

Keep the proactive cleanup design, but make it honor cfg.session.maintenance, repair the tests, and narrow the branch to the session cleanup surface before maintainer review.

## Reproduction Assessment

Yes, by source inspection: current main only invokes archived transcript cleanup from the session-store write maintenance path and has no startup/hourly archive sweeper, so idle gateways can leave stale artifacts behind. I did not run a live filesystem reproduction in this read-only review.

## Solution Assessment

No, not as written. The cleanup seam is maintainable, but the PR must pass cfg.session.maintenance into the retention resolver and fix the mocked import before merge.
