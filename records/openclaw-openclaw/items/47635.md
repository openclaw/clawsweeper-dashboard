## Summary

Keep this PR open. The linked disk-bloat problem is still real and current main does not yet have an equivalent proactive startup/hourly archive sweep, but this PR is not merge-ready: GitHub reports it as dirty against current main, and one prior review concern remains valid because the sweep only covers default state-dir agent sessions and misses configured custom session-store roots.

## What This Changes

The PR adds a session archive cleanup helper, a UUID-suffixed `.tmp` artifact detector, gateway startup and hourly timer wiring, shutdown cleanup for the timer, and regression tests for stale `.deleted`, `.reset`, `.tmp`, and `.bak` files.

## Best Possible Solution

Land this as a rebased, focused gateway/session maintenance patch that uses current main's config-aware session-store target discovery, wires the sweep into the current startup sidecar/timer lifecycle, preserves active transcripts and young atomic-write temp files, and includes regression coverage for default and custom `session.store` roots.

## Reproduction Assessment

Yes. The linked report gives a high-confidence path by accumulating stale `.deleted`, `.reset`, `.tmp`, and `.bak` files under agent session directories, and current-main source confirms there is no proactive startup/hourly sweep for those artifacts.

## Solution Assessment

No. The general direction is maintainable, but the proposed implementation is not the best current fix until it is rebased onto the current gateway startup structure and made config-aware for custom session-store roots.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Scan configured session stores:** `src/gateway/session-archive-cleanup.ts:23`
  - body: The sweep only calls `resolveAgentSessionDirs(params.stateDir)`, so it cleans `<stateDir>/agents/*/sessions` but skips configured or templated `session.store` roots. Current session discovery includes those roots, so relocated stores will continue accumulating `.deleted`, `.reset`, `.bak`, and orphaned `.tmp` files while this new startup/hourly path reports success. Build the directory list from the config-aware session-store target resolver and add a custom-store regression test.
