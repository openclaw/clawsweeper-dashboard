## Summary

Keep open. ## What This Changes

The branch adds a gateway startup/hourly session archive sweeper for stale `.deleted.*`, `.reset.*`, orphaned `.tmp`, and excess `.bak.*` files, plus tests, timer handle wiring, a changelog entry, and broad formatting-only churn across UI/plugin files.

## Best Possible Solution

Land or replace this with a focused proactive cleanup implementation that preserves the documented warn-mode contract or updates that contract explicitly, covers configured session-store roots, and includes redacted live startup/timer cleanup proof.

## Reproduction Assessment

Yes, at source level. Current main has write-path archive cleanup and CLI cleanup, but no gateway startup/hourly archive sweep, and current tests show old `.bak.*` artifacts remain after normal store pruning.

## Solution Assessment

No, not as currently written. Reusing retention helpers and configured store-root discovery is directionally good, but destructive cleanup should either respect `mode: "enforce"` or land with explicit maintainer approval and docs for changing warn-mode semantics.
