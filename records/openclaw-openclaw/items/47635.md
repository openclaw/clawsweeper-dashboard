## Summary

Keep this PR open: current main still lacks proactive startup/hourly cleanup for session archive artifacts, but the PR branch has blocking correctness issues around configured retention and its new test mock.

## What This Changes

The PR adds a gateway session-archive sweeper, startup/hourly timer wiring, close-handle cleanup, orphan tmp detection, regression tests, and a changelog entry, plus substantial formatting-only churn in unrelated UI/plugin/test files.

## Best Possible Solution

Keep the proactive cleanup design, but make the sweeper honor the loaded gateway config and repair the tests before normal maintainer review or merge.

## Reproduction Assessment

Yes, by source inspection: current main only cleans archived transcripts from the reactive session-store write path and has no startup/hourly archive sweeper, so idle warn-mode gateways can leave stale artifacts behind. I did not run a live filesystem reproduction in this read-only sweep.

## Solution Assessment

No, not as currently written. The approach is narrow and maintainable, but the PR must pass cfg.session.maintenance into the retention resolver and fix the mismatched test mock first.
