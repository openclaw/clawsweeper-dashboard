## Summary

Keep open. The PR is aimed at a real session-archive disk-bloat problem and has addressed several earlier review comments, but the latest head still ignores configured session maintenance retention in the new sweeper, so it is not ready to merge.

## What This Changes

The PR branch adds a gateway startup/hourly session-archive sweeper for stale `.deleted.*`, `.reset.*`, orphaned UUID `.tmp`, and excess `.bak.*` files, wires timer shutdown, adds helper/tests, updates the changelog, and carries broad formatting-only churn in unrelated UI/plugin files.

## Best Possible Solution

Land a narrow proactive cleanup that reuses the existing session maintenance config, covers default and configured session-store roots, has focused regression tests, and trims unrelated churn before maintainer review.

## Reproduction Assessment

Yes. By source inspection, current main lacks the proposed startup/hourly sweeper, and the PR regression is visible because the new sweeper receives config but resolves maintenance with no input.

## Solution Assessment

No, not as currently written. The startup/hourly sweeper is a maintainable direction, but it must pass `params.cfg?.session?.maintenance` into the resolver before it can safely honor user retention settings.
