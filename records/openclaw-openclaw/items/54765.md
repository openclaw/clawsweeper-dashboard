## Summary

Keep open. Current main still swallows durable subagent restore failures, and this PR’s direction is useful, but the proposed rollback is incomplete because it restores `subagentRuns` and the restore latch without restoring `resumedRuns`, so a later retry can still skip runs touched before the failed pass.

## What This Changes

The PR snapshots subagent registry state during durable restore, resets the restore latch on failures, adds restore-retry coverage, and hardens related registry test mocks/cleanup.

## Best Possible Solution

Expand the restore failure rollback to restore all in-memory restore/resume bookkeeping touched during the failed pass, then cover a failure after at least one restored run has been marked resumed.

## Reproduction Assessment

Yes. A focused unit test can restore two active runs, make a later resume step throw after the first run is marked in `resumedRuns`, call `initSubagentRegistry()` again, and assert that first run is retried instead of skipped by the guard.

## Solution Assessment

No. The PR targets the right function and failure path, but the catch block should roll back `resumedRuns` as well as `subagentRuns` and `restoreAttempted`; it also needs the required changelog entry for this user-facing recovery fix.
