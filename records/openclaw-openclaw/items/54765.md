## Summary

Keep open. Current main still has the durable subagent restore failure mode, and this PR is aimed at the right recovery path, but the proposed rollback is incomplete because a failed restore retry can leave `resumedRuns` poisoned and suppress later recovery for already-touched runs.

## What This Changes

The PR snapshots subagent runs during durable restore, restores the run map and resets the restore latch on exceptions, adds restore-retry coverage, and hardens related registry mocks and cleanup.

## Best Possible Solution

Revise the rollback so a failed durable restore attempt restores all in-memory restore/resume bookkeeping consistently, especially `resumedRuns` for ids touched during the failed pass, then add a regression where failure occurs after at least one restored run has been marked resumed and a later init retries it normally.

## Reproduction Assessment

Yes. A focused unit test can restore two active runs, force a transient exception after the first run is marked in `resumedRuns`, call `initSubagentRegistry()` again, and assert the first run is retried instead of skipped by the `resumedRuns.has(runId)` guard.

## Solution Assessment

No. The PR’s direction is narrow and maintainable, but the proposed catch block only restores `subagentRuns`; it should also roll back the restore/resume latch state that can affect subsequent retries.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Roll back resumed-run state on restore failure:** `src/agents/subagent-registry.ts:815-819`
  - body: The catch block restores `subagentRuns` and resets `restoreAttempted`, but the restore loop can already have called `resumeSubagentRun()`, which records ids in `resumedRuns`. If a later restore step throws, the next `initSubagentRegistry()` retry restores the map but skips those ids via `resumedRuns.has(runId)`, so cleanup or wait recovery is not re-run. Snapshot or clear the restored run ids in `resumedRuns` as part of rollback and cover a failure after one resume attempt.
  - confidence: 0.86
