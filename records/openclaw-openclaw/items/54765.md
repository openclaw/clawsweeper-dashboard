## What This Changes

The PR snapshots subagent registry restore state, retries after transient restore failure, adds a focused restore-retry test, and hardens related registry test mocks and cleanup.

## Best Possible Solution

Rollback both `subagentRuns` and restore-time `resumedRuns` state on restore failure, add a regression case for failure after a run is marked resumed, rebase, and include redacted real restore-retry proof before merge.

## Reproduction Assessment

Yes, at source level. A restore failure after any run has been marked in `resumedRuns` leaves that id gated off, so the next `initSubagentRegistry()` retry skips `resumeSubagentRun()` for it.

## Solution Assessment

No. Snapshotting `subagentRuns` and resetting the restore latch is the right area, but the patch also needs to snapshot or clear restore-time `resumedRuns` ids and cover that failure mode.
