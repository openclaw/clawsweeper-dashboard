## Best Possible Solution

Refresh the branch on current main, roll back both registry contents and resume guards after failed restore, and add a regression test that fails when a restored run is marked resumed before the transient exception.

## Reproduction Assessment

Yes. Source inspection shows restoreSubagentRunsOnce() calls resumeSubagentRun() for restored IDs, and resumeSubagentRun() records IDs in resumedRuns; the PR catch path restores only subagentRuns and restoreAttempted, so a retry can skip those IDs.

## Solution Assessment

No. Snapshotting the registry and resetting the restore latch is the right layer, but the fix must also restore or clear resume-side state created during the failed attempt.
