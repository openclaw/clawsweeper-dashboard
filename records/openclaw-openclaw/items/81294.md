## Summary

Keep this PR open: it is a plausible narrow fix for pnpm source-update installs, but the proposed source-spec detector is incomplete for a Git source form that current OpenClaw already accepts, and the PR still lacks after-fix real behavior proof.

## What This Changes

The PR adds `--allow-build=openclaw` to pnpm global update installs for selected OpenClaw Git/GitHub source specs and adds unit assertions for those argv shapes.

## Best Possible Solution

Keep the narrow pnpm argv fix, but make the source-spec predicate match the pnpm Git source forms already accepted by OpenClaw and then prove a real source update succeeds.

## Reproduction Assessment

No high-confidence runtime reproduction was established in this review. Source inspection plus pnpm's documented build-approval behavior show why current main can fail for pnpm source installs, but the actual `openclaw update --tag github:...` flow was not run here.

## Solution Assessment

No: adding `--allow-build=openclaw` for pnpm source specs is likely the narrow fix, but this implementation should cover all accepted pnpm Git source forms and include real after-fix behavior proof before merge.
