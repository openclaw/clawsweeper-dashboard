## Summary

Keep open: the feature request is plausible and has real-use evidence, but this PR still has merge-blocking contract, schema, session-key cleanup, and raw-media egress policy issues that need maintainer direction rather than cleanup closure.

## What This Changes

The PR adds opt-in `tools.media.audio/video.nativeForwarding` handling that captures raw audio/video attachment bytes and injects them into embedded prompt media through a session-keyed store.

## Best Possible Solution

Define a typed native audio/video media contract with provider capability and fallback gates, schema/help/docs, one resolved session key, payload limits, and raw-media egress policy, then converge this PR with https://github.com/openclaw/openclaw/issues/51667 and https://github.com/openclaw/openclaw/pull/78797.

## Reproduction Assessment

Not applicable as a current-main bug reproduction because this PR adds a feature. The merge blockers are source-reproducible from the PR diff, current strict schema, redirected session-key flow, and the pinned pi-ai media contract.

## Solution Assessment

No. The direction is plausible, but this patch is not the best mergeable solution while it routes audio/video through an image-only prompt boundary and leaves config validation plus raw-media egress policy unsettled.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Use the resolved runner key for native media blocks:** `src/media-understanding/apply.ts:599-601`
  - body: In native-command redirects, the run can target `CommandTargetSessionKey`, but this stores raw media under `ctx.SessionKey` while the runner consumes and discards under `params.sessionKey`. Redirected media will not be forwarded, and up-to-25 MiB base64 blocks can remain in the process map.
  - confidence: 0.92
- **[P2] Register nativeForwarding in the strict config schema:** `src/config/types.tools.ts:105-114`
  - body: This adds a public config key only to the TypeScript type. `ToolsMediaUnderstandingSchema` is strict and still omits that key, so user config containing the new option would be rejected and the option remains absent from schema/help surfaces.
  - confidence: 0.91
- **[P2] Add a typed audio/video prompt media contract:** `src/agents/pi-embedded-runner/run/attempt.ts:2572-2574`
  - body: The pinned prompt boundary accepts only text and image content, while these new blocks use `type: "audio" | "video"`. Casting them through `images` can serialize them as unsupported image payloads or lose media until the dependency contract and provider adapters land together.
  - confidence: 0.9
