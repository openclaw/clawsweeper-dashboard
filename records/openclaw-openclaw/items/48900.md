## Summary

Keep open, but the patch is not mergeable as-is: the requested native audio/video direction is plausible, while this implementation still has schema, session-key, prompt-contract, and raw-media egress blockers.

## What This Changes

The PR adds opt-in `tools.media.audio/video.nativeForwarding` handling that captures raw audio/video bytes and injects them into embedded runner prompt media.

## Best Possible Solution

Converge the open native audio/video work on a typed prompt-media or provider-owned transport contract with schema/help/docs, provider capability checks, fallback behavior, and raw-media egress policy.

## Reproduction Assessment

Yes for the PR blockers by source inspection: the strict schema rejection path, retargeted session-key mismatch, and image-only dependency boundary are traceable from the PR diff and current main. The requested native forwarding behavior itself is a feature, not a current-main bug reproduction.

## Solution Assessment

No. The feature direction is plausible, but this patch is not the best implementation while native audio/video blocks are cast through an image-only prompt boundary and raw-media egress/provider support policy remains unsettled.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Use the resolved runner key for native media blocks:** `src/media-understanding/apply.ts:599-601`
  - body: In native-command redirects, `getReplyFromConfig` runs the model under `CommandTargetSessionKey`, but this stores raw media under `ctx.SessionKey` while the runner consumes and discards under `params.sessionKey`. Redirected media will not be forwarded, and up-to-25 MiB base64 blocks can remain in the process map.
  - confidence: 0.92
- **[P2] Register nativeForwarding in the strict schema:** `src/config/types.tools.ts:105-114`
  - body: This adds a public config field only to the TypeScript type. The strict `ToolsMediaUnderstandingSchema` still omits it, so real config containing the advertised option is rejected before the feature can run.
  - confidence: 0.91
- **[P2] Add a typed audio/video prompt-media contract:** `src/agents/pi-embedded-runner/run/attempt.ts:2572-2574`
  - body: The pinned prompt boundary accepts `ImageContent[]`, while these new blocks use `type: "audio" | "video"`. Casting them through `images` can drop or mis-serialize native media unless the dependency contract and provider adapters land together.
  - confidence: 0.9
