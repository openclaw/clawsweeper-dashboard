## Summary

Keep open, but this PR is not mergeable as-is: current main still lacks native audio/video forwarding, and the branch has source-proven blockers in strict config validation, redirected session cleanup, the pinned prompt-media contract, and raw-media egress policy. The related canonical issue and overlapping implementation remain open, so maintainers need one product/provider-contract direction before this can move forward.

## What This Changes

The PR adds opt-in `tools.media.audio/video.nativeForwarding` handling that captures raw attachment bytes during media understanding and injects them into embedded runner prompt media alongside images.

## Best Possible Solution

Define typed native audio/video prompt-media contracts with schema/help/docs, provider capability and fallback gates, resolved session-key handling, payload limits, and raw-media egress policy, then converge the open implementations against the canonical issue.

## Reproduction Assessment

Not applicable as a current-main bug reproduction because this PR adds a feature. The merge blockers are source-reproducible from the PR diff, current strict schema, redirected session-key handling, and the pinned pi-ai media contract.

## Solution Assessment

No. The direction is useful, but this patch is not the best merge shape while it routes audio/video through an image-only prompt boundary and leaves config validation, provider gating, and raw-media egress policy unsettled.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Use the resolved runner key for native media blocks:** `src/media-understanding/apply.ts:599-601`
  - body: In native-command redirects, the run can target `CommandTargetSessionKey`, but this stores raw media under `ctx.SessionKey` while the runner consumes and discards under `params.sessionKey`. Redirected media will not be forwarded, and up-to-25 MiB base64 blocks can remain in the process map.
  - confidence: 0.92
- **[P2] Register nativeForwarding in the strict schema:** `src/config/types.tools.ts:105-114`
  - body: This adds a public config key only to the TypeScript type. `ToolsMediaUnderstandingSchema` is strict and still omits that key, so user config containing the new option would be rejected before the behavior can run.
  - confidence: 0.91
- **[P2] Add a typed audio/video prompt media contract:** `src/agents/pi-embedded-runner/run/attempt.ts:2572-2574`
  - body: The pinned prompt boundary accepts only text and image content, while these new blocks use `type: "audio" | "video"`. Casting them through `images` can serialize them as unsupported image payloads or lose media until the dependency contract and provider adapters land together.
  - confidence: 0.9
