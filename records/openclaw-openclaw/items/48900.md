## Summary

Keep this PR open, but do not merge as-is. The feature direction is plausible and the PR body includes real-run logs, but the patch adds a schema-rejected config key, can miss cleanup/forwarding on retargeted native commands, and sends raw audio/video through an image-shaped prompt contract while raw-media egress policy is unsettled.

## What This Changes

The PR adds opt-in `tools.media.audio/video.nativeForwarding`, a session-keyed native media store, raw attachment capture, and embedded-runner prompt injection.

## Best Possible Solution

Revise or replace this with a typed native audio/video prompt contract, provider capability and fallback semantics, schema/help/docs updates, raw-media egress/audit policy, and one resolved key for capture, consume, and cleanup.

## Reproduction Assessment

Yes for the PR blockers by source inspection: the strict schema rejection path, retargeted session-key mismatch, and dependency prompt type mismatch are all traceable. The user-facing request itself is a feature, not a bug reproduction.

## Solution Assessment

No. The patch should be revised around a typed audio/video prompt contract and explicit capability, fallback, and raw-media egress policy instead of casting through the image path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Use the resolved runner key for native media blocks:** `src/media-understanding/apply.ts:599-601`
  - body: In native-command redirects, the run can target `CommandTargetSessionKey`, but this stores raw media under `ctx.SessionKey` while the runner consumes and discards under `params.sessionKey`. Redirected media will not be forwarded and up-to-25 MiB base64 blocks can remain in the process map.
  - confidence: 0.92
- **[P2] Register nativeForwarding in the strict schema:** `src/config/types.tools.ts:105-114`
  - body: This adds a public config key only to the TypeScript type. `ToolsMediaUnderstandingSchema` is strict and the schema/help metadata omit `nativeForwarding`, so user config containing the new key will be rejected or undocumented.
  - confidence: 0.91
- **[P2] Add a typed audio/video prompt-media contract:** `src/agents/pi-embedded-runner/run/attempt.ts:2572-2574`
  - body: The prompt boundary and pinned pi-ai/pi-coding-agent types still accept image content for media, while these new blocks use `type: "audio" | "video"`. Casting them through `images` can pass unsupported shapes or drop media until the dependency contract and adapters land together.
  - confidence: 0.89
