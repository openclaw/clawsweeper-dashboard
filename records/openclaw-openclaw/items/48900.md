## Summary

Keep open for maintainer review: the feature direction is plausible and the PR body includes real Telegram/OpenRouter log proof, but the patch is not mergeable while the config schema, runner session key, typed media contract, and raw-media egress policy remain unresolved.

## What This Changes

The PR adds opt-in `tools.media.audio/video.nativeForwarding` handling that captures raw audio/video bytes during media understanding and casts them into the embedded runner prompt media array.

## Best Possible Solution

Define a typed native audio/video media contract with schema/help/docs, provider capability and fallback gates, one resolved session key, payload limits, and explicit raw-media egress policy, then converge this PR with the canonical issue and alternate implementation.

## Reproduction Assessment

Not applicable as a current-main bug reproduction because this PR adds a feature. The merge blockers are source-reproducible from the PR diff, current strict schema, redirected session-key handling, and the pinned pi-ai media contract.

## Solution Assessment

No. The direction may be worth pursuing, but this patch is not the best mergeable solution while it routes audio/video through an image-only prompt boundary and leaves config validation plus raw-media egress policy unsettled.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Use the resolved runner key for native media blocks:** `src/media-understanding/apply.ts:599-601`
  - body: In native-command redirects, the run can target `CommandTargetSessionKey`, but this stores raw media under `ctx.SessionKey` while the runner consumes and discards under `params.sessionKey`. Redirected media will not be forwarded, and up-to-25 MiB base64 blocks can remain in the process map.
  - confidence: 0.92
- **[P2] Register nativeForwarding in the strict config schema:** `src/config/types.tools.ts:105-114`
  - body: This adds a public config key only to the TypeScript type. `ToolsMediaUnderstandingSchema` is strict and still omits that key, so user config containing the new option would be rejected and the option remains absent from schema/help surfaces.
  - confidence: 0.91
- **[P2] Add a typed audio/video prompt media contract:** `src/agents/pi-embedded-runner/run/attempt.ts:2572-2574`
  - body: The pinned prompt boundary accepts only text and image content, while these new blocks use `type: "audio" | "video"`. Casting them through `images` can serialize them as unsupported image payloads or lose media until the dependency contract and provider adapters land together.
  - confidence: 0.9
