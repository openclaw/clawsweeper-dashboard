## Summary

Keep open. Current main still lacks native audio/video forwarding, and this PR is the active implementation candidate, but the patch has source-reproducible blockers around config validation, session-key cleanup, typed prompt media, and raw-media egress policy.

## What This Changes

The PR adds opt-in `tools.media.audio/video.nativeForwarding` support that captures raw attachment bytes during media understanding and injects them into embedded-runner prompt media.

## Best Possible Solution

Land a revised feature with typed audio/video prompt-media support, provider capability and fallback semantics, schema/help/docs updates, raw-media egress policy, and one resolved session key for capture, consume, and cleanup.

## Reproduction Assessment

Yes for the PR blockers by source inspection. The strict schema omits the new key, native-command session retargeting can diverge from the PR's storage key, and the pinned prompt-media types only accept images.

## Solution Assessment

No. The feature direction is plausible, but this patch should be revised around a typed audio/video prompt-media contract and explicit raw-media egress/capability policy instead of casting through the image path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Use the resolved runner key for native media blocks:** `src/media-understanding/apply.ts:599-601`
  - body: In native command redirects, reply setup can retarget the run to `CommandTargetSessionKey`, but this stores raw media under the source context key while the runner consumes and discards under `params.sessionKey`. Redirected voice/video blocks are then not forwarded and can remain in the process store.
  - confidence: 0.92
- **[P2] Register nativeForwarding in the strict config schema:** `src/config/types.tools.ts:100`
  - body: Adding `nativeForwarding` to the TypeScript config type is not enough because `ToolsMediaUnderstandingSchema` is strict. User config containing the new audio/video key will be rejected until the schema and generated config/help metadata are updated.
  - confidence: 0.91
- **[P2] Add a typed prompt-media contract before casting:** `src/agents/pi-embedded-runner/run/attempt.ts:2572-2574`
  - body: The prompt boundary and pinned pi-agent/pi-ai types are still image-shaped, while these new blocks use `type: "audio" | "video"`. Until the upstream prompt contract and adapters accept typed audio/video content, this cast can pass unsupported shapes to providers or drop the media.
  - confidence: 0.86
