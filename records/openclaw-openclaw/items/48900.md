## Summary

Keep this PR open, but it should not merge as-is: the feature direction is plausible, yet the branch still needs maintainer product/security/provider-contract decisions and has concrete source-level blockers in schema handling, session-key cleanup, and the prompt media contract.

## What This Changes

The PR adds opt-in `tools.media.audio/video.nativeForwarding` that captures raw audio/video bytes during media understanding and injects them into the embedded-runner prompt media array.

## Best Possible Solution

Define a typed native audio/video media contract with provider capability and fallback gates, strict config/schema/help/docs coverage, raw-media egress policy, and one resolved session key for capture, consume, and cleanup.

## Reproduction Assessment

Yes for the PR blockers by source inspection: the strict schema rejection path, native-command session-key mismatch, and image-only prompt-media boundary are traceable from current main plus the PR diff. The requested behavior itself is a new feature rather than a bug reproduction.

## Solution Assessment

No. The feature direction is plausible, but this patch is not the best maintainable solution until the media prompt contract, provider capability/fallback behavior, strict schema surface, and raw-media egress policy are settled together.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Use the resolved runner key for native media blocks:** `src/media-understanding/apply.ts:599-601`
  - body: In native-command redirects, the run can target `CommandTargetSessionKey`, but this stores raw media under `ctx.SessionKey` while the runner consumes and discards under `params.sessionKey`. Redirected media will not be forwarded and up-to-25 MiB base64 blocks can remain in the process map.
  - confidence: 0.92
- **[P2] Register nativeForwarding in the strict schema:** `src/config/types.tools.ts:105-114`
  - body: This adds a public config key only to the TypeScript type. `ToolsMediaUnderstandingSchema` is strict and still omits `nativeForwarding`, so user config containing the new key would be rejected and undocumented.
  - confidence: 0.91
- **[P2] Add a typed audio/video prompt-media contract:** `src/agents/pi-embedded-runner/run/attempt.ts:2572-2574`
  - body: The current prompt boundary is still image-typed, while these new blocks use `type: "audio" | "video"`. Casting them through `images` can pass unsupported shapes or lose media until the dependency contract and provider adapters land together.
  - confidence: 0.9
