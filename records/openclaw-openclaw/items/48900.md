## Summary

Keep open, but this PR is not merge-ready. Current main still lacks native audio/video forwarding, and the PR has source-backed blockers around session-key cleanup, strict config validation, the typed prompt-media contract, and raw-media egress policy.

## What This Changes

The PR adds opt-in `tools.media.audio/video.nativeForwarding` handling that captures raw audio/video attachment bytes and injects them into embedded-runner prompt media alongside images.

## Best Possible Solution

Land or replace this with a typed native audio/video media contract that preserves provider capability gates, fallback behavior, strict schema/help/docs coverage, one resolved session key, and explicit raw-media egress policy.

## Reproduction Assessment

Yes for the PR blockers by source inspection: the strict schema rejection path, retargeted session-key mismatch, and image-only prompt-media boundary are traceable from current main plus the PR diff. The requested user behavior itself is a feature, not a bug reproduction.

## Solution Assessment

No. The feature direction is plausible, but this patch is not the best solution until audio/video prompt media are typed and gated instead of being cast through an image-only boundary.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Use the resolved runner key for native media blocks:** `src/media-understanding/apply.ts:599-601`
  - body: In native-command redirects, the run can target `CommandTargetSessionKey`, but this stores raw media under `ctx.SessionKey` while the runner consumes and discards under `params.sessionKey`. Redirected media will not be forwarded and up-to-25 MiB base64 blocks can remain in the process map.
  - confidence: 0.92
- **[P2] Register nativeForwarding in the strict schema:** `src/config/types.tools.ts:105-114`
  - body: This adds a public config key only to the TypeScript type. `ToolsMediaUnderstandingSchema` is strict and still omits `nativeForwarding`, so user config containing the new key would be rejected and undocumented.
  - confidence: 0.91
- **[P2] Add a typed audio/video prompt-media contract:** `src/agents/pi-embedded-runner/run/attempt.ts:2572-2574`
  - body: The pinned prompt boundary still accepts `images?: ImageContent[]`, while these new blocks use `type: "audio" | "video"`. Casting them through `images` can pass unsupported shapes or lose media until the dependency contract and provider adapters land together.
  - confidence: 0.9

## Security Review
