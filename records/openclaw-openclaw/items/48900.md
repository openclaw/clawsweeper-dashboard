## Summary

Keep open: the feature direction is useful and overlaps the canonical native-audio work, but this PR is not mergeable while config validation, redirected session-key cleanup, typed audio/video prompt media, provider gating, and raw-media egress policy remain unsettled.

## What This Changes

The branch adds opt-in `tools.media.audio/video.nativeForwarding`, captures raw audio/video attachments during media understanding, stores them in a session-keyed map, and merges them into the embedded runner prompt media array.

## Best Possible Solution

Define a typed native audio/video media contract with schema/help/docs, provider capability and fallback gates, one resolved session key, payload limits, and explicit raw-media egress policy, then converge the open PRs against https://github.com/openclaw/openclaw/issues/51667.

## Reproduction Assessment

Not applicable as a current-main bug reproduction because this PR adds a feature. The merge blockers are source-reproducible from the PR diff, current strict schema, redirected session-key handling, and image-shaped prompt media types.

## Solution Assessment

No. The direction may be useful, but this patch is not the best mergeable solution while it routes audio/video through an image-only prompt boundary and leaves config validation, provider gating, and raw-media egress policy unsettled.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Use the resolved runner key for native media blocks:** `src/media-understanding/apply.ts:599-601`
  - body: In native-command redirects, the run can target `CommandTargetSessionKey`, but this stores raw media under `ctx.SessionKey` while the runner consumes and discards under `params.sessionKey`. Redirected media will not be forwarded, and up-to-25 MiB base64 blocks can remain in the process map.
  - confidence: 0.92
- **[P2] Register nativeForwarding in the strict schema:** `src/config/types.tools.ts:103`
  - body: This adds a public config key only to the TypeScript type. `ToolsMediaUnderstandingSchema` is strict and still omits that key, so user config containing the new option would be rejected and the option remains absent from schema/help surfaces.
  - confidence: 0.91
- **[P2] Add a typed audio/video prompt media contract:** `src/agents/pi-embedded-runner/run/attempt.ts:2572-2574`
  - body: The embedded runner prompt boundary is still typed as `ImageContent[]`, while these new blocks use `type: "audio" | "video"`. Casting them through `images` depends on an unlanded companion contract and can mis-serialize or drop native media.
  - confidence: 0.9
