## Summary

Close as duplicate/superseded: this is an older, blocked implementation of native audio/video forwarding, while the same remaining product and provider-contract work is now carried by the open canonical issue and newer overlapping implementation PR.

## What This Changes

The PR adds opt-in `tools.media.audio/video.nativeForwarding` handling that captures raw audio/video attachment bytes and injects them into the embedded runner prompt media path.

## Best Possible Solution

Keep https://github.com/openclaw/openclaw/issues/51667 as the canonical tracker and converge any implementation against a typed native media contract, schema/help/docs, provider capability and fallback gates, and raw-media egress policy before landing.

Not applicable as a current-main bug reproduction because this PR adds a feature. The merge blockers are source-reproducible from the PR diff, strict config schema, redirected session-key handling, and the pinned pi-ai text/image media contract.

## Solution Assessment

No. The direction is useful, but this branch is not the best merge shape while it routes audio/video through an image-only prompt boundary and leaves schema, provider gating, and raw-media egress policy to be settled in the canonical tracker.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Use the resolved runner key for native media blocks:** `src/media-understanding/apply.ts:599-601`
  - body: In native-command redirects, the run can target `CommandTargetSessionKey`, but this stores raw media under `ctx.SessionKey` while the runner consumes and discards under `params.sessionKey`. Redirected media will not be forwarded, and up-to-25 MiB base64 blocks can remain in the process map.
  - confidence: 0.92
- **[P2] Register nativeForwarding in the strict schema:** `src/config/types.tools.ts:105-114`
  - body: This adds a public config key only to the TypeScript type. `ToolsMediaUnderstandingSchema` is strict and still omits that key, so user config containing the new option would be rejected before the behavior can run.
  - confidence: 0.91
- **[P2] Add a typed audio/video prompt media contract:** `src/agents/pi-embedded-runner/run/attempt.ts:2572-2574`
  - body: The pinned prompt boundary accepts only text and image content, while these new blocks use `type: "audio" | "video"`. Casting them through `images` can serialize them as unsupported image payloads or lose media until the dependency contract and provider adapters land together.
  - confidence: 0.9

## Security Review

Status: needs_attention

Summary: Raw media forwarding is security-sensitive because it sends original audio/video bytes to model providers and needs explicit policy, capability gating, fallback behavior, payload limits, and docs.

Concerns:

- **[medium] Document and gate raw media egress:** `src/media-understanding/apply.ts:568`
  - body: Forwarding original audio/video bytes changes data handling from transcript or description text to provider-bound raw media payloads. Operators need schema-visible opt-in, provider support checks, fallback behavior, payload limits, and documentation explaining which providers receive raw media.
  - confidence: 0.84
