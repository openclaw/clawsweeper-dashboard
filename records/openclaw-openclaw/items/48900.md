## Summary

Keep open. The feature is not implemented on current main, and this PR still has source-proven correctness blockers plus a raw-media egress and prompt-contract review gap before it is safe to merge.

## What This Changes

The PR adds an opt-in `tools.media.audio/video.nativeForwarding` path that captures raw attachment bytes during media understanding and merges them into embedded-runner prompt media.

## Best Possible Solution

Land a revised native media design with typed audio/video prompt content, provider capability and fallback semantics, schema/help/docs updates, raw-media egress policy, and one resolved key for capture, consume, and cleanup.

## Reproduction Assessment

Yes for the PR blockers by source inspection: the strict schema omits `nativeForwarding`, native-command retargeting can store and consume under different keys, and the pinned pi-ai prompt contract only accepts text/image content.

## Solution Assessment

No. The feature direction is plausible, but this patch should be revised around a typed audio/video media contract and explicit raw-media egress/capability policy instead of casting through the image path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Use the resolved runner key for native media blocks:** `src/media-understanding/apply.ts:599-601`
  - body: In native command redirects, reply setup can retarget the run to `CommandTargetSessionKey`, but this stores raw media under `ctx.SessionKey` while the runner consumes and discards under `params.sessionKey`. Redirected voice/video blocks are then not forwarded and can remain in the process store.
  - confidence: 0.92
- **[P2] Register nativeForwarding in the strict schema:** `src/config/types.tools.ts:100`
  - body: Adding `nativeForwarding` to the TypeScript config type is not enough because `ToolsMediaUnderstandingSchema` is strict. User config containing the new audio/video key will be rejected until the schema and generated config/help metadata are updated.
  - confidence: 0.91
- **[P2] Add a typed audio/video prompt-media contract:** `src/agents/pi-embedded-runner/run/attempt.ts:2572-2574`
  - body: The prompt boundary and pinned pi-ai types are still image-shaped, while these new blocks use `type: "audio" | "video"`. Until the upstream prompt contract and adapters accept typed audio/video content, this cast can pass unsupported shapes to providers or drop the media.
  - confidence: 0.88
