## Summary

Keep open: the feature direction is useful and the PR has real log proof, but the patch is not mergeable while config validation, redirected session-key cleanup, the pinned prompt-media contract, and raw-media egress policy remain unresolved.

## What This Changes

The PR adds opt-in `tools.media.audio/video.nativeForwarding` to capture raw audio/video bytes during media understanding and inject them into embedded-runner prompt media alongside images.

## Best Possible Solution

Define a first-class typed native audio/video media contract with schema/help/docs, provider capability and fallback gates, resolved session-key handling, payload limits, and explicit raw-media egress policy; then converge this PR with the canonical native-audio tracker and overlapping implementation.

## Reproduction Assessment

Not applicable as a current-main bug reproduction because this PR adds a feature. The merge blockers are high-confidence from source inspection: strict schema omission, redirected session-key handling, and the pinned pi-ai media contract.

## Solution Assessment

No. Opt-in forwarding is plausible, but this patch is not the best mergeable solution while it casts audio/video through an image-only prompt boundary and leaves schema validation, provider gating, docs, and raw-media egress policy unsettled.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Use the resolved runner key for native media blocks:** `src/media-understanding/apply.ts:599-601`
  - body: In native-command redirects, the model run can target `CommandTargetSessionKey`, but this stores raw media under `ctx.SessionKey` while the runner consumes and discards under `params.sessionKey`. Redirected media will not be forwarded, and up-to-25 MiB base64 blocks can remain in the process map.
  - confidence: 0.92
- **[P2] Register nativeForwarding in the strict schema:** `src/config/types.tools.ts:105-114`
  - body: This adds a public config key only to the TypeScript type. `ToolsMediaUnderstandingSchema` is strict and still omits `nativeForwarding`, so user config containing the new option would be rejected before the behavior can run.
  - confidence: 0.91
- **[P2] Add a typed audio/video prompt media contract:** `src/agents/pi-embedded-runner/run/attempt.ts:2572-2574`
  - body: The pinned prompt boundary accepts only text and image content, while these new blocks use `type: "audio" | "video"`. Casting them through `images` can serialize them as unsupported image payloads or lose media until the dependency contract and provider adapters land together.
  - confidence: 0.9
