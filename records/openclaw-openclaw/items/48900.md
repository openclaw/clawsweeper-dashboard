## Summary

Keep this PR open, but it is not merge-ready: the feature direction is useful and the PR includes real Telegram/log proof, yet the patch still has concrete correctness blockers plus a raw-media egress contract that needs maintainer judgment.

## What This Changes

The PR adds opt-in `tools.media.audio/video.nativeForwarding` support that captures raw audio/video attachments during media understanding and injects them into embedded-runner prompt media.

## Best Possible Solution

Land a revised feature with typed prompt-media support, provider capability and fallback rules, documented raw-media egress, schema/help/docs updates, one resolved session key for capture and cleanup, and focused redirect/config coverage.

## Reproduction Assessment

Yes for the PR blockers. Source inspection shows the strict config schema rejects the proposed key, and native command redirects can retarget the session key while the PR stores media under the source context key.

## Solution Assessment

No. The feature direction is plausible, but this patch should be revised around an explicit raw-media egress policy and a typed audio/video prompt-media contract instead of casting through the image path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Use the resolved runner key for native media blocks:** `src/media-understanding/apply.ts:599-601`
  - body: In native command redirects, session initialization can retarget the run to `CommandTargetSessionKey`, but this stores raw media under the source `ctx.SessionKey`. The runner then consumes and discards under its resolved `params.sessionKey`, so redirected voice/video blocks are not forwarded and can remain in the process store.
  - confidence: 0.92
- **[P2] Register nativeForwarding in the strict config schema:** `src/config/types.tools.ts:100`
  - body: Adding `nativeForwarding` to the TypeScript type is not enough because `ToolsMediaUnderstandingSchema` is strict. User config containing `tools.media.audio.nativeForwarding` or `tools.media.video.nativeForwarding` will be rejected unless the schema and generated config/help metadata are updated with the new key.
  - confidence: 0.91
- **[P2] Add a typed prompt-media contract before casting:** `src/agents/pi-embedded-runner/run/attempt.ts:2572-2574`
  - body: The prompt boundary is still image-shaped, while these new blocks use `type: "audio" | "video"` with base64 data. Until the upstream prompt contract and adapters accept typed audio/video content, this cast can send unsupported prompt shapes to providers or silently drop the media.
  - confidence: 0.84
