## Summary

Keep open. The feature is not implemented on current main, and this PR is a useful implementation candidate, but the patch still has blocking config-schema, session-key cleanup, provider-contract, and raw-media egress issues before it can merge.

## What This Changes

The PR adds opt-in `tools.media.audio/video.nativeForwarding`, captures raw audio/video attachment bytes during media understanding, and injects them into the embedded-runner prompt media path.

## Best Possible Solution

Revise this into a typed native audio/video prompt-media contract with provider capability and fallback semantics, schema/help/docs coverage, raw-media egress policy, and one resolved key for capture, consume, and cleanup.

## Reproduction Assessment

Yes for the PR blockers by source inspection: the strict schema rejection path, retargeted session-key mismatch, and dependency prompt type mismatch are all traceable from current main plus the PR diff. The user-facing request itself is a feature, not a bug reproduction.

## Solution Assessment

No. The feature direction is plausible, but this patch is not the best fix because it casts audio/video through an image-only prompt contract and lacks capability, fallback, schema, docs, and egress policy coverage.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Use the resolved runner key for native media blocks:** `src/media-understanding/apply.ts:599-601`
  - body: In native-command redirects, the run can target `CommandTargetSessionKey`, but this stores raw media under `ctx.SessionKey` while the runner consumes and discards under `params.sessionKey`. Redirected media will not be forwarded and up-to-25 MiB base64 blocks can remain in the process map.
  - confidence: 0.92
- **[P2] Register nativeForwarding in the strict schema:** `src/config/types.tools.ts:105-114`
  - body: This adds a public config key only to the TypeScript type. `ToolsMediaUnderstandingSchema` is strict and still omits `nativeForwarding`, so user config containing the new key would be rejected and undocumented.
  - confidence: 0.91
- **[P2] Add a typed audio/video prompt-media contract:** `src/agents/pi-embedded-runner/run/attempt.ts:2572-2574`
  - body: The pinned prompt boundary still accepts image-shaped media, while these new blocks use `type: "audio" | "video"`. Casting them through `images` can pass unsupported shapes or lose media until the dependency contract and provider adapters land together.
  - confidence: 0.89

## Security Review
