## Summary

Keep open, but not merge-ready. Current main still lacks native audio/video forwarding, and source plus dependency inspection show this patch still has schema, session-key, prompt-media contract, and raw-media egress blockers that need maintainer direction.

## What This Changes

The branch adds opt-in `tools.media.audio/video.nativeForwarding` handling that captures raw audio/video bytes during media understanding and injects them into the embedded runner prompt media array.

## Best Possible Solution

Rework or replace this with a typed native audio/video media contract that includes strict schema/help/docs, provider capability gates, fallback behavior, one resolved session key, payload limits, and explicit raw-media egress policy, converging with https://github.com/openclaw/openclaw/issues/51667 and https://github.com/openclaw/openclaw/pull/78797.

## Reproduction Assessment

Yes for the PR blockers by source inspection: the strict schema rejection path, retargeted session-key mismatch, and image-only dependency boundary are traceable from the PR diff and current main. The requested native forwarding behavior itself is a feature, not a current-main bug reproduction.

## Solution Assessment

No. The feature direction is plausible, but this patch is not the best implementation while audio/video blocks are cast through an image-only prompt boundary and raw-media egress/provider support policy remains unsettled.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Use the resolved runner key for native media blocks:** `src/media-understanding/apply.ts:599-601`
  - body: In native-command redirects, the run can target `CommandTargetSessionKey`, but this stores raw media under `ctx.SessionKey` while the runner consumes and discards under `params.sessionKey`. Redirected media will not be forwarded and up-to-25 MiB base64 blocks can remain in the process map.
  - confidence: 0.92
- **[P2] Register nativeForwarding in the strict schema:** `src/config/types.tools.ts:105-114`
  - body: This adds a public config key only to the TypeScript type. `ToolsMediaUnderstandingSchema` is strict and still omits that key, so user config containing the new option would be rejected and the option remains undocumented.
  - confidence: 0.91
- **[P2] Add a typed audio/video prompt-media contract:** `src/agents/pi-embedded-runner/run/attempt.ts:2572-2574`
  - body: The pinned prompt boundary still accepts image content, while these new blocks use `type: "audio" | "video"`. Casting them through `images` can serialize them as unsupported image payloads or lose media until the dependency contract and provider adapters land together.
  - confidence: 0.9
