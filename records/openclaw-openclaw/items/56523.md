## Summary

Keep open. Current main still accepts partial model cost config and preserves omitted `cacheRead`/`cacheWrite` prices into `estimateUsageCost`; the flat calculation returns `undefined`, and the OTEL metric gate then skips cost emission. The issue is paired with an open external fix proposal, so the maintainer path is to review, land, or replace that proposal.

## What This Changes

The issue asks OpenClaw to treat missing `cost.cacheRead` and `cost.cacheWrite` prices as zero so cost metrics are still reported.

## Best Possible Solution

Normalize omitted cache price multipliers to zero in the shared pricing boundary or estimator, preserve tiered pricing behavior, and add regression coverage for direct cost estimation plus diagnostics cost emission.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a cost config accepted without `cacheRead` or `cacheWrite` flows through `buildProviderCostIndex` into `estimateUsageCost`, whose flat path multiplies by undefined and returns `undefined`.

## Solution Assessment

Yes. Defaulting omitted cache price fields to zero at the shared normalization or estimator boundary is the narrow maintainable fix; patching only the OTEL exporter would leave other shared cost surfaces with the same undefined estimate.
