## Label Justifications

- `P2`: The bug can silently suppress usage-cost telemetry for partial model pricing configs, but it has limited blast radius and does not break the core runtime.
- `impact:data-loss`: The issue is about silently dropped usage-cost telemetry data when partial pricing makes `costUsd` undefined.

## Summary

Keep open. Current `main` and `v2026.5.22` still allow omitted flat cache price fields to make shared cost estimation undefined, and https://github.com/factnest365-ops/openclaw/pull/1 is still an open linked implementation candidate rather than a reason to close this issue early.

## What This Changes

The issue requests zero defaults for omitted `cost.cacheRead` and `cost.cacheWrite` so usage-cost telemetry is still emitted for partial flat pricing configs.

## Best Possible Solution

Normalize omitted flat cache price multipliers to zero in the shared pricing normalization or estimator path, preserve tiered pricing behavior, and add focused regression coverage for cost estimation plus diagnostics cost emission.

## Reproduction Assessment

Yes. Source inspection shows config can omit flat cache price fields, current cost indexing preserves those omissions, and flat cost estimation can return `undefined` before diagnostics export sees a cost value.

## Solution Assessment

Yes. Defaulting omitted flat cache prices to zero at the shared cost boundary is the narrow maintainable fix; changing only diagnostics-otel would leave other cost consumers with the same undefined estimate.
