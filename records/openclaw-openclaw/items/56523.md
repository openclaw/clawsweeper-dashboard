## Label Justifications

- `P2`: The bug affects cost telemetry for partial model pricing configs but has limited blast radius and no core runtime outage.
- `impact:data-loss`: The reported behavior can silently drop usage-cost telemetry data when partial pricing suppresses `costUsd`.

## Summary

Keep open. Current `main` and the latest release still allow partial flat pricing objects to leave cache price multipliers undefined, which can make shared cost estimation return `undefined` and suppress diagnostics cost export; there is also an open external PR using closing syntax for this issue that should be reviewed, updated, landed, or replaced.

## What This Changes

The issue requests that omitted `cost.cacheRead` and `cost.cacheWrite` pricing fields default to zero so cost telemetry is still emitted for partial pricing configs.

## Best Possible Solution

Normalize omitted flat cache price multipliers to zero at the shared pricing normalization or estimator boundary, preserve tiered pricing behavior, and add focused regression coverage for cost estimation plus diagnostics cost emission.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: accepted partial pricing objects can flow into `buildProviderCostIndex`, omitted cache prices remain undefined, and `estimateUsageCost` returns `undefined` after a non-finite flat total.

## Solution Assessment

Yes. Defaulting omitted cache price fields to zero at the shared cost boundary is the narrow maintainable fix; changing only the OTEL exporter would leave other shared cost surfaces with the same undefined estimate.
