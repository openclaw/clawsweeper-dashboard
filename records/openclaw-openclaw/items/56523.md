## Summary

Keep open. Current `main` and the latest release still have a source-reproducible path where accepted partial model cost configs can produce `undefined` cost estimates, which prevents OTEL cost export; the closed session-telemetry issue is a different layer and does not supersede this. The issue is also paired with an open external fix proposal that maintainers should review, land, or replace.

## What This Changes

Default missing `cost.cacheRead` and `cost.cacheWrite` price fields to `0` so cost metrics still report when those optional fields are omitted.

## Best Possible Solution

Normalize omitted cache price multipliers to zero at the shared pricing normalization or estimator boundary, preserve tiered pricing behavior, and add focused regression coverage for cost estimation plus diagnostics cost emission.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: accepted partial cost objects flow through `buildProviderCostIndex` into `estimateUsageCost`, where the flat branch multiplies by omitted cache prices and returns `undefined`.

## Solution Assessment

Yes. Defaulting omitted cache price fields to zero at the shared cost boundary is the narrow maintainable fix; changing only the OTEL exporter would leave other shared cost surfaces with the same undefined estimate.
