## Summary

Keep open. Current main and the latest release still accept partial model pricing objects, carry omitted cache pricing through the shared cost estimator, and can therefore omit `costUsd` from diagnostics/OTEL cost metrics; this issue is also paired with an open external fix proposal that maintainers should review, land, or replace.

## What This Changes

The issue requests defaulting omitted `cacheRead` and `cacheWrite` pricing fields to zero so cost estimation and OTEL cost metrics still report when cache prices are not configured.

## Best Possible Solution

Normalize omitted cache price multipliers to zero at the shared pricing normalization or estimator boundary, preserve tiered pricing behavior, and add focused regression coverage for cost estimation plus diagnostics cost emission.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: accepted partial cost objects flow through `buildProviderCostIndex` into `estimateUsageCost`, where omitted cache prices make the flat total non-finite and return `undefined`.

## Solution Assessment

Yes on direction. The maintainable fix is to default omitted cache price fields at the shared pricing normalization or estimator boundary, not only in the OTEL exporter, while preserving tiered pricing behavior.
