## Summary

Keep open: current main still has a source-reproducible path where missing cacheRead/cacheWrite price fields make cost estimation non-finite, which leaves costUsd absent before the diagnostics-otel cost metric can be recorded. A linked fork PR proposes the right narrow idea, but it is not merged upstream and includes unrelated docs edits.

## What This Changes

The issue asks for missing cacheRead and cacheWrite cost rates to default to zero so estimated cost telemetry is still emitted when input/output pricing is present.

## Best Possible Solution

Add a focused current-main regression for missing cacheRead/cacheWrite cost rates and default those two price fields to 0 in the cost-estimation path without changing provider-reported cache token semantics.

## Reproduction Assessment

Yes. Source inspection shows an incomplete cost object can make estimateUsageCost return undefined, which then leaves model.usage without costUsd and prevents diagnostics-otel from recording the cost metric.

## Solution Assessment

Yes, the narrow maintainable fix is to normalize missing cache cost rates to 0 at cost-estimation or direct cost-resolution boundaries, with a regression test; the linked fork PR has the right idea but needs current-main cleanup and test coverage.
