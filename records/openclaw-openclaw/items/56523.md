## Summary

Keep open: current main and v2026.6.10 still have a source-reproducible cost-pricing path where omitted cache price fields can make estimated cost undefined, so diagnostics-otel can miss the cost counter. The linked fork PR is useful context, but it is still open outside the target repository and is not an upstream fix.

## What This Changes

The issue asks for missing `cost.cacheRead` and `cost.cacheWrite` pricing fields to default to zero so diagnostics cost metrics are still emitted.

## Best Possible Solution

Normalize missing cache price rates to zero at the cost-resolution or cost-calculation boundary, then add regression coverage for an input/output-priced model whose cache rates are omitted.

## Reproduction Assessment

Yes. Source inspection on current main shows partial cost configs can reach `estimateUsageCost`, produce a non-finite total, and leave diagnostics-otel without `costUsd` to record.

## Solution Assessment

Yes for the fix shape: defaulting missing cache price rates at the pricing boundary is narrower than changing provider cache-token reporting. The existing fork PR is not the best landing vehicle unless it is narrowed or ported upstream with tests.
