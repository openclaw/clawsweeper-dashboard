## Label Justifications

- `P2`: The bug can silently suppress usage-cost telemetry for partial model pricing configs, but it has limited blast radius and does not break the core runtime.
- `impact:data-loss`: The issue is about silently dropped usage-cost telemetry data when partial pricing makes `costUsd` undefined.

## Summary

Keep open. Current main and the latest release still allow omitted flat cache price fields to make shared usage-cost estimation return no cost, and the linked external implementation candidate at https://github.com/factnest365-ops/openclaw/pull/1 is still open rather than merged.

## What This Changes

Default omitted `cost.cacheRead` and `cost.cacheWrite` pricing values to zero so OpenTelemetry cost metrics are not silently skipped.

## Best Possible Solution

Normalize omitted flat cache price multipliers to zero in the shared pricing normalization or estimator path, preserve tiered pricing behavior, and add focused regression coverage for cost estimation plus diagnostics cost emission.

## Reproduction Assessment

Yes. Source inspection shows raw `models.json` pricing can preserve omitted cache rates, and flat cost estimation can then become non-finite and suppress diagnostics cost export.

## Solution Assessment

Yes. Defaulting omitted flat cache prices to zero at the shared cost boundary is the narrow maintainable fix; changing only diagnostics-otel would leave other cost consumers with the same undefined estimate.
