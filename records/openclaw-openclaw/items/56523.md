## Summary

Keep open: current main and v2026.6.10 still allow partial cost pricing to make estimated cost undefined, which leaves diagnostics-otel without a cost counter value. The linked fork PR is useful evidence, but it is unmerged outside openclaw/openclaw, so this issue still needs an upstream landing decision.

## What This Changes

The issue requests defaulting missing `cost.cacheRead` and `cost.cacheWrite` rates to zero so OTEL cost metrics are still emitted when cache pricing is omitted.

## Best Possible Solution

Normalize missing cache price rates to zero at the shared pricing boundary and add regression coverage for an input/output-priced model whose cache rates are omitted.

## Reproduction Assessment

Yes. Source inspection on current main shows partial cost configs can reach `estimateUsageCost`, produce a non-finite total, and leave diagnostics-otel without `costUsd` to record.

## Solution Assessment

Yes for the fix shape. Defaulting missing cache price rates at the pricing boundary is narrower than changing provider cache-token reporting or adding OTEL-specific fallback logic.
