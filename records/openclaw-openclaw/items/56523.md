- `impact:data-loss`: The reported failure can silently omit estimated cost telemetry even when model pricing and usage data are otherwise available.

## Summary

Keep open: current main and the latest release still have a source-reproducible path where omitted cache price fields make estimated cost non-finite, so model usage events omit costUsd and diagnostics-otel cannot emit the cost counter. The linked fork PR is useful context, but it is open outside the target repo and mixes unrelated docs edits, so it is not a canonical upstream fix.

## What This Changes

The issue requests default zero cacheRead/cacheWrite cost rates when model pricing omits them so OTEL cost metrics are still reported.

## Best Possible Solution

Normalize missing cache price fields to zero at the cost-resolution or cost-calculation boundary, then add regression coverage for an input/output-priced model whose cache rates are omitted.

## Reproduction Assessment

Yes. Source inspection on current main shows optional cache price fields can flow into estimateUsageCost and produce an undefined cost estimate, which prevents diagnostics-otel from receiving costUsd.

## Solution Assessment

Yes. The best fix is narrow defaulting at the pricing calculation or normalization boundary with regression coverage, not a change to provider cache-token reporting semantics.
