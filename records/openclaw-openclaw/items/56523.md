- `impact:data-loss`: The reported failure can silently drop estimated cost telemetry from diagnostics export when local pricing is otherwise available.

## Summary

Keep open: current main still has a source-reproducible path where missing cacheRead/cacheWrite price fields make cost estimation non-finite, which can leave costUsd absent before diagnostics-otel records the cost metric. The linked fork PR is useful evidence but is open, unmerged, and not a clean upstream landing path as-is.

## What This Changes

The issue requests defaulting missing cacheRead and cacheWrite token pricing fields to zero so cost telemetry still reports metrics when cache token prices are omitted.

## Best Possible Solution

Add a focused current-main regression for missing cacheRead/cacheWrite cost rates and default those two price fields to 0 in the cost-estimation or raw cost-normalization path without changing provider-reported cache token semantics.

## Reproduction Assessment

Yes. Source inspection shows an incomplete cost object can make estimateUsageCost return undefined, which then leaves model.usage without costUsd and prevents diagnostics-otel from recording the cost metric.

## Solution Assessment

Yes. The narrow maintainable fix is to normalize missing cache cost rates to 0 at cost-estimation or direct cost-resolution boundaries, with a regression test; the linked fork PR has the right idea but needs current-main cleanup and test coverage.
