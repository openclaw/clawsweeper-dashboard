## Summary

Keep open: current main and the latest release still have a source-reproducible path where optional cache price fields can make estimated cost non-finite, so diagnostics-otel may miss cost telemetry even when input/output pricing exists. The linked fork PR is useful context, but it is unmerged and includes unrelated docs changes, so it should not close this issue before a clean upstream fix lands.

## What This Changes

The issue asks OpenClaw to treat missing model cost cacheRead/cacheWrite price fields as zero so diagnostics cost metrics still report when cache price rates are omitted.

## Best Possible Solution

Normalize missing cache price fields to zero at the cost-resolution or cost-calculation boundary, add focused regression coverage for an input/output-priced model whose cache rates are omitted, and leave provider-reported cacheWrite token semantics unchanged.

## Reproduction Assessment

Yes. Source inspection on current main shows optional cache price fields can flow into estimateUsageCost and produce an undefined cost estimate, which prevents diagnostics-otel from receiving costUsd; I did not run a live OTel collector in this read-only review.

## Solution Assessment

Yes. The best fix is the narrow defaulting behavior at the pricing calculation or resolution boundary with regression coverage, not a change to provider cache-token reporting semantics.
