## Summary

Keep open. Current main still lets omitted cache pricing fields reach `estimateUsageCost`, where multiplying zero cache usage by undefined cache price fields makes the total non-finite and suppresses the OpenTelemetry cost metric. The issue is also paired with an open external fix proposal, so the best cleanup path is review, land, or replace that proposal rather than close the issue now.

## What This Changes

The issue requests defaulting omitted `cost.cacheRead` and `cost.cacheWrite` model price fields to zero so cost metrics are still emitted when input/output pricing exists.

## Best Possible Solution

Normalize omitted cache price multipliers to zero at the shared pricing boundary or defensively inside `estimateUsageCost`, preserve existing tiered-pricing behavior, and add focused regression coverage for direct model pricing lookup plus diagnostics cost emission.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to pass a cost object with finite `input` and `output` but omitted `cacheRead` or `cacheWrite` through `estimateUsageCost`; current main multiplies by undefined and returns `undefined`.

## Solution Assessment

Yes. Defaulting omitted cache price fields to zero is the narrow maintainable direction; the safest implementation is shared cost normalization or estimator-side coercion with tests covering both flat and tiered pricing paths.
