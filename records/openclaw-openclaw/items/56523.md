## Summary

Keep open: current main still has a source-reproducible cost-estimation path where incomplete model cost objects can make the total non-finite, leaving diagnostics-otel without the cost metric. The linked fork PR is useful context, but it is unmerged and not a clean upstream landing branch.

## What This Changes

The issue requests defaulting missing model cost `cacheRead` and `cacheWrite` price fields to `0` so cost estimation and diagnostics-otel cost metrics are still emitted.

## Best Possible Solution

Normalize missing cache price fields to zero at the cost-estimation or cost-resolution boundary, then add focused regression coverage for an input/output-priced model whose cache rates are omitted.

## Reproduction Assessment

Yes. Source inspection shows an incomplete cost object accepted by the schema can make `estimateUsageCost` return `undefined`, which prevents the diagnostics cost metric path from receiving `costUsd`; I did not run a live OTel collector in this read-only review.

## Solution Assessment

Yes. The narrow maintainable fix is to default missing cache price rates to zero at the cost-estimation or resolution boundary, with regression coverage, rather than changing provider cache-token reporting semantics.
