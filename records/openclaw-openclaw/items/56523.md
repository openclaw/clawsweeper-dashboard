## Summary

Keep open. Current main still has a source-reproducible path where omitted cache price fields can make cost estimation return `undefined`, and this issue is paired with an open external fix proposal that maintainers should review, land, or replace.

## What This Changes

The issue asks for omitted `cost.cacheRead` and `cost.cacheWrite` fields to behave as zeroes so diagnostics OTEL cost metrics are still emitted.

## Best Possible Solution

Normalize omitted cache price multipliers to zero at the shared pricing normalization or estimator boundary, preserve tiered pricing behavior, and add focused regression coverage for cost estimation plus diagnostics cost emission.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a partial cost object can flow through `buildProviderCostIndex` into `estimateUsageCost`, whose flat branch multiplies by omitted cache prices and returns `undefined`.

## Solution Assessment

Yes. Defaulting omitted cache price fields to zero at the shared normalization or estimator boundary is the narrow maintainable fix; changing only the OTEL exporter would leave other shared cost surfaces with the same undefined estimate.
