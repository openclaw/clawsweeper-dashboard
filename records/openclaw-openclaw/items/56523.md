## Label Justifications

- `P2`: The bug can silently suppress usage-cost telemetry for partial model pricing configs, but it has limited blast radius and does not break the core runtime.
- `impact:data-loss`: The issue is about silently dropped usage-cost telemetry data when partial pricing makes `costUsd` undefined.

## Summary

Keep open. Current `main` and the latest release still allow omitted flat cache price fields to flow as `undefined` into shared usage-cost estimation, and https://github.com/factnest365-ops/openclaw/pull/1 is an open linked implementation candidate rather than a reason to close this issue early.

## What This Changes

Default omitted `cost.cacheRead` and `cost.cacheWrite` pricing fields to zero so usage-cost telemetry still emits for partial model pricing configs.

## Best Possible Solution

Normalize omitted flat cache price multipliers to zero in the shared pricing normalization or estimator path, preserve tiered pricing behavior, and add focused regression coverage for cost estimation plus diagnostics cost emission.

## Reproduction Assessment

Yes. Source inspection shows partial model pricing is schema-accepted, copied with undefined cache rates, and then multiplied in flat cost estimation so `estimateUsageCost` can return `undefined` and suppress diagnostics cost export.

## Solution Assessment

Yes. Defaulting omitted flat cache prices to zero at the shared cost boundary is the narrow maintainable fix; changing only diagnostics-otel would leave other cost consumers with the same undefined estimate.
