## Label Justifications

- `P2`: This is a normal-priority telemetry/cost reporting bug with a narrow surface and limited blast radius.
- `impact:data-loss`: The failure silently drops estimated cost telemetry from diagnostics export when local pricing is otherwise available.

## Summary

Keep open: current main and v2026.6.6 still allow an incomplete raw cost config to make cache pricing non-finite, which leaves costUsd undefined before diagnostics-otel can record the cost metric. The linked fork PR has the right narrow idea, but it is unmerged and includes unrelated docs/provider-list edits.

## What This Changes

Default missing cacheRead/cacheWrite pricing fields to zero so cost telemetry can still compute and report costs.

## Best Possible Solution

Add current-main regression coverage and default missing cacheRead/cacheWrite price fields to zero at the cost-estimation or direct raw-cost resolution boundary, without changing provider-reported cache token counts.

## Reproduction Assessment

Yes. Source inspection shows an incomplete raw cost object can make estimateUsageCost return undefined, which then leaves model.usage without costUsd and prevents diagnostics-otel from recording the cost metric.

## Solution Assessment

Yes. The narrow maintainable fix is to normalize missing cache cost rates to zero at the cost-estimation or direct cost-resolution boundary with a focused regression test; the linked fork PR has the right idea but needs current-main cleanup.
