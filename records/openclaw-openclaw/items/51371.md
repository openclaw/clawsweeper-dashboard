## Summary

Keep open: current main and the latest release still have the unguarded ping continuation, and related duplicate reports now point at this PR as the canonical focused fix; the remaining merge blocker is missing after-fix real behavior proof from the external contributor.

## What This Changes

The PR adds an `NSLock`-guarded one-shot wrapper around `WebSocketTaskBox.sendPing()` so duplicate pong callbacks cannot resume the checked continuation twice.

## Best Possible Solution

Land this focused one-shot guard, or a maintainer-selected equivalent, with after-fix real setup proof, current-main Swift validation, and focused duplicate-callback coverage if feasible.

## Reproduction Assessment

Yes, at source level: current main routes every websocket ping callback to the same checked continuation, and a `WebSocketTasking` double can invoke that callback twice. The PR body and related reports also include crash signatures matching `CheckedContinuation.resume` inside `WebSocketTaskBox.sendPing()`.

## Solution Assessment

Yes for the code shape: a local one-shot guard in `WebSocketTaskBox.sendPing()` is the narrowest maintainable fix for duplicate ping callbacks. The merge path still needs after-fix real behavior proof, rebase/current-main validation, and preferably focused regression coverage.
