## What This Changes

The PR adds a private `NSLock`-guarded one-shot continuation wrapper in `GatewayChannel.swift` and routes `WebSocketTaskBox.sendPing()` callbacks through it.

## Best Possible Solution

Land this focused guard, or a maintainer-selected equivalent, after redacted after-fix real behavior proof or an explicit maintainer proof override, with duplicate-callback regression coverage added during landing.

Yes at source level. A `WebSocketTasking` double that invokes `pongReceiveHandler` twice would drive current main into two resumes of the same checked continuation; I did not run a live iOS/macOS device repro in this read-only review.

## Solution Assessment

Yes, the proposed at-most-once wrapper is the narrow maintainable fix for this bridge. The safer merge path is to pair it with after-fix real behavior proof and a focused duplicate-callback regression test.
