## Summary

Keep open. Current main still resumes the WebSocket ping checked continuation directly, and the live keepalive loop still reaches that path; this PR is the selected canonical duplicate for the crash fix, but it still lacks after-fix real behavior proof before merge.

## What This Changes

The PR adds a private NSLock-guarded one-shot continuation wrapper around `WebSocketTaskBox.sendPing()` so duplicate URLSession pong callbacks can only resume the checked continuation once.

## Best Possible Solution

Land this one-shot guard or an equivalent `GatewayChannel.swift` fix after after-fix real device/gateway proof or an explicit maintainer proof override, with focused duplicate-callback regression coverage if feasible.

## Reproduction Assessment

Yes at source level. A `WebSocketTasking` double that invokes `pongReceiveHandler` twice would drive current main into two resumes of the same checked continuation; I did not run a live iOS/macOS device repro in this read-only review.

## Solution Assessment

Yes. The proposed one-shot wrapper is the narrow maintainable fix for this callback-to-async bridge, but merge should wait for after-fix real behavior proof or a maintainer proof override plus focused regression coverage if practical.
