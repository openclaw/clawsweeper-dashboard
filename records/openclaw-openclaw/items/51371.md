## Summary

Keep open: current main and the latest release still resume the websocket ping checked continuation directly, while this PR adds a focused one-shot guard; the code shape looks sound, but external after-fix real behavior proof is still missing before merge.

## What This Changes

The PR adds a private NSLock-guarded OneShotContinuation to WebSocketTaskBox.sendPing so duplicate ping callbacks can only resume the checked continuation once.

## Best Possible Solution

Land this focused one-shot guard, or a maintainer-selected equivalent, with redacted after-fix real setup proof and focused duplicate-callback coverage if feasible.

## Reproduction Assessment

Yes, at source level: current main routes every websocket ping callback to the same CheckedContinuation, and the helper directly resumes it. I did not run a live iOS/macOS gateway repro in this read-only review.

## Solution Assessment

Yes for the code shape: a local at-most-once guard around WebSocketTaskBox.sendPing is the narrowest maintainable fix for duplicate pong callbacks. The merge path still needs after-fix real behavior proof and preferably a focused regression test.
