## Summary

Keep open: current main still has the unguarded ping continuation path, and this PR remains the focused canonical fix, but the external-contributor real behavior proof gate is still missing.

## What This Changes

The branch adds an `NSLock`-guarded `OneShotContinuation` around `WebSocketTaskBox.sendPing()` so duplicate pong callbacks can only resume the checked continuation once.

## Best Possible Solution

Keep this PR as the canonical implementation path, add after-fix proof and focused duplicate-callback coverage if feasible, then rebase and validate the one-shot guard before merge.

## Reproduction Assessment

Yes, at source level: a `WebSocketTasking` test double can invoke the ping callback twice, and current main routes both invocations to the same checked continuation through `ThrowingContinuationSupport.resumeVoid`. I did not run Swift tests because this review was read-only.

## Solution Assessment

Yes for the code shape: a local one-shot guard in `WebSocketTaskBox.sendPing()` is the narrowest maintainable fix for duplicate ping callbacks. The merge path still needs after-fix real behavior proof, current-main validation, and preferably focused regression coverage.
