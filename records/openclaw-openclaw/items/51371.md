## Summary

Keep open: current main and the latest release still route `WebSocketTaskBox.sendPing()` through an unguarded checked continuation, while this PR is a focused source-level fix for the same crash path. The code change looks correct, but the external PR still needs after-fix real behavior proof before merge.

## What This Changes

This PR adds a private `NSLock`-guarded one-shot continuation wrapper around `WebSocketTaskBox.sendPing()` so duplicate pong callbacks only resume the checked continuation once.

## Best Possible Solution

Land this focused one-shot guard, or a maintainer-selected equivalent, after contributor after-fix proof or an explicit maintainer proof override, with duplicate-callback regression coverage added during landing.

## Reproduction Assessment

Yes at source level. A `WebSocketTasking` double that invokes `pongReceiveHandler` twice would drive current main into two resumes of the same checked continuation; I did not run a live iOS/macOS device repro in this read-only review.

## Solution Assessment

Yes, the proposed at-most-once wrapper is the narrow maintainable fix for this bridge. The safer merge path is to pair it with after-fix real behavior proof and a focused duplicate-callback regression test.
