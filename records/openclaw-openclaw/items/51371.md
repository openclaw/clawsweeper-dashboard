## Summary

Keep open: current main still has the unguarded Swift websocket ping continuation path, and the two overlapping PRs were closed in favor of this PR as the canonical focused fix. The patch looks functionally correct, but this external PR still needs after-fix real behavior proof before merge.

## What This Changes

The PR adds an NSLock-guarded one-shot continuation wrapper around `WebSocketTaskBox.sendPing()` so duplicate ping callbacks can only resume the checked continuation once.

## Best Possible Solution

Keep this PR as the canonical implementation path, require contributor after-fix proof, then land the at-most-once guard with focused duplicate-callback coverage if feasible.

## Reproduction Assessment

Yes, at source level. A `WebSocketTasking` test double can invoke `pongReceiveHandler` twice, and current main would route both invocations to the same checked continuation through `ThrowingContinuationSupport.resumeVoid`.

## Solution Assessment

Yes for the code shape: a local one-shot guard in `WebSocketTaskBox.sendPing()` is the narrowest maintainable fix for a duplicate ping callback. The merge path still needs after-fix real behavior proof, current-main validation, and preferably a focused regression test.
