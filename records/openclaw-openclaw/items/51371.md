## Summary

Keep this PR open. Current main and the latest release still have the unguarded `sendPing()` checked-continuation bridge, and the related duplicate PRs were closed in favor of this canonical path; the patch looks focused, but still needs after-fix real behavior proof before merge.

## What This Changes

This PR adds a private NSLock-backed one-shot continuation wrapper around `WebSocketTaskBox.sendPing()` in `GatewayChannel.swift` so duplicate pong callbacks only resume once.

## Best Possible Solution

Land this canonical `GatewayChannel.swift` guard or an equivalent fix after after-fix real gateway proof or a maintainer proof override, with focused duplicate-callback regression coverage if feasible.

## Reproduction Assessment

Yes, at source level. A `WebSocketTasking` double that invokes `pongReceiveHandler` twice would drive current main into two resumes of the same checked continuation; I did not run a live iOS/macOS device repro in this read-only review.

## Solution Assessment

Yes. The one-shot wrapper is a narrow maintainable fix for this callback-to-async bridge, but merge should wait for after-fix real behavior proof or a maintainer proof override plus focused regression coverage if practical.
