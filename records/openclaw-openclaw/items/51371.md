## Summary

Keep open. Current main still resumes the ping continuation directly, and this PR's one-shot guard matches the reported crash path; the remaining blocker is missing after-fix real behavior proof rather than a blocking code-review finding.

## What This Changes

This PR adds a private NSLock-backed one-shot continuation around `WebSocketTaskBox.sendPing()` so duplicate pong callbacks only resume the checked continuation once.

## Best Possible Solution

Land this one-shot guard or an equivalent `GatewayChannel.swift` fix after after-fix real device/gateway proof or an explicit maintainer proof override, with focused duplicate-callback regression coverage if feasible.

## Reproduction Assessment

Yes at source level. A `WebSocketTasking` double that invokes `pongReceiveHandler` twice would drive current main into two resumes of the same checked continuation; I did not run a live iOS/macOS device repro in this read-only review.

## Solution Assessment

Yes. The one-shot wrapper is a narrow maintainable fix for this callback-to-async bridge, but merge should wait for after-fix real behavior proof or a maintainer proof override plus focused regression coverage if practical.
