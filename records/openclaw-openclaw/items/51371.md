## Summary

Keep open: current main and the latest release still use the unguarded WebSocket ping continuation, and the PR’s code shape is a narrow fix, but the external contributor still needs after-fix real behavior proof before merge.

## What This Changes

The PR adds an NSLock-guarded one-shot continuation wrapper around `WebSocketTaskBox.sendPing()` so duplicate pong callbacks only resume the checked continuation once.

## Best Possible Solution

Land this focused one-shot guard, or a maintainer-selected equivalent, after redacted after-fix real setup proof is posted; add duplicate-callback regression coverage if feasible.

## Reproduction Assessment

Yes, at source level: current main gives every `sendPing` callback the same checked continuation, and Swift checked continuations trap on a second resume. I did not run a live iOS or macOS gateway cancellation repro in this read-only review.

## Solution Assessment

Yes for the code shape: a local at-most-once guard around `WebSocketTaskBox.sendPing()` is the narrowest maintainable fix for duplicate pong callbacks. The merge path still needs after-fix real behavior proof and preferably focused regression coverage.
