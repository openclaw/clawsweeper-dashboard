## Summary

Keep open: current main and the latest release still use the unguarded checked-continuation ping bridge, and this PR is a narrow plausible crash fix. The remaining blocker is external contributor after-fix real behavior proof, not a code-review finding.

## What This Changes

The PR adds an NSLock-guarded one-shot continuation wrapper around `WebSocketTaskBox.sendPing()` so duplicate ping callbacks cannot resume the same checked continuation twice.

## Best Possible Solution

Land this focused one-shot guard, or a maintainer-selected equivalent, after redacted after-fix real setup proof is added; add duplicate-callback coverage if feasible.

## Reproduction Assessment

Yes, at source level: current main routes every websocket ping callback to the same checked continuation, and a second callback would resume it twice. I did not run a live iOS/macOS gateway repro in this read-only review.

## Solution Assessment

Yes for the code shape: a local at-most-once guard around `WebSocketTaskBox.sendPing()` is the narrowest maintainable fix for duplicate pong callbacks. The merge path still needs after-fix real behavior proof and preferably focused regression coverage.
