## Summary

Keep open: the reported display problem is plausible, but this PR changes a shared fresh-only helper and would let stale preserved totals bypass runtime transcript estimation in fork, preflight compaction, and memory-flush paths. The updated body includes useful live context, but it still does not show patched OpenClaw behavior running after the change.

## What This Changes

The PR removes the stale-total guard from `resolveFreshSessionTotalTokens`, changes `isSessionTotalTokensFresh`, and rewrites the active changelog section.

## Best Possible Solution

Keep a fresh-only helper for runtime decisions, and add or use a separate preserved-total display path for `/context`, session listing, status, and Control UI callers with regression coverage for both semantics.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: current main preserves stale `totalTokens`, while display callers still use the fresh-only helper and runtime tests show stale totals must remain ignored in fork/preflight decisions. I did not run a live patched UI flow.

## Solution Assessment

No. The requested display behavior is valid, but changing `resolveFreshSessionTotalTokens` globally is not the safest fix because fresh-only runtime callers still depend on its current semantics.
