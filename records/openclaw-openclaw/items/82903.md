## Summary

## What This Changes

This PR changes `resolveFreshSessionTotalTokens` to return preserved `totalTokens` even when `totalTokensFresh` is false and updates session/status tests plus the changelog.

## Best Possible Solution

Keep a fresh-only token helper for runtime decisions, and add a separate preserved-total display path for `/context`, session listing, status, and Control UI callers with regression coverage for both semantics.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main can preserve stale `totalTokens`, while display callers still use the fresh-only helper and runtime tests show stale totals must remain ignored for fork/preflight decisions.

## Solution Assessment

No. The requested display behavior is worth fixing, but globally changing `resolveFreshSessionTotalTokens` is not the narrowest maintainable solution because runtime callers still depend on fresh-only semantics.
