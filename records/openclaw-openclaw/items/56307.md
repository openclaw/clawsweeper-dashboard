## Summary

Keep open: current main still requires explicit `thread: true`/`mode: "session"` inputs and has no channel-owned spawn-default policy for subagent parameter injection, while the related open issues cover adjacent session-mode decoupling rather than this plugin SDK contract.

## What This Changes

Request adds a channel/plugin SDK threading policy so `sessions_spawn` can automatically default subagent `thread` and `mode` per channel instead of relying on model-specified technical parameters.

## Best Possible Solution

Define a backward-compatible plugin SDK spawn-default contract resolved before tool schema and execution, with opt-in metadata, docs, and tests for explicit versus defaulted `thread`/`mode` behavior.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a plugin SDK/runtime feature request. Source inspection gives high confidence that current main still lacks channel-owned automatic `thread`/`mode` defaulting.

## Solution Assessment

Unclear until maintainer design review. Moving ownership from hardcoded or prompt-driven behavior into channel metadata is directionally sound, but the exact API, opt-in policy, and delivery semantics need approval.
