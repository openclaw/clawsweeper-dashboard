## Summary

Keep open: current main and v2026.5.22 still require explicit `thread: true`/`mode: "session"` for thread-bound subagent sessions, and there is no channel-owned spawn-default policy in the plugin SDK. Adjacent issues cover headless session mode, binding presets, or Feishu-specific group agents, but none supersede this plugin metadata contract.

## What This Changes

The issue requests a channel plugin threading policy that can automatically default subagent spawns to thread-bound session mode without hardcoded channel names or prompt instructions.

## Best Possible Solution

Define a backward-compatible plugin SDK spawn-default contract, preferably reusing existing conversation/thread binding metadata where possible, then update tool schema, execution, docs, and focused tests for explicit versus defaulted `thread` and `mode` behavior.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/API request. Source and release inspection give high confidence that current main still lacks channel-owned automatic `thread`/`mode` defaulting.

## Solution Assessment

Unclear pending maintainer design review. Moving ownership from prompts or channel-name conditionals into plugin metadata is directionally sound, but the exact API and interaction with existing thread-binding policy need approval.
