## Summary

Keep open: current main still requires explicit `sessions_spawn` `thread` and `mode` inputs, while Feishu's current-conversation hook only runs after a thread-bound spawn was already requested. The related Feishu binding work solved delivery and binding, and related open issues track headless session semantics, but neither provides the requested channel-owned spawn-default contract.

## What This Changes

Add a channel/plugin SDK threading policy so `sessions_spawn` can automatically default subagent spawns to thread-bound session mode for channels such as Feishu.

## Best Possible Solution

Define a backwards-compatible plugin SDK spawn-default contract resolved before `sessions_spawn` validation, opt in only channels with safe current-conversation binding support, and document/test the resulting defaults.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature gap. Source inspection is high-confidence that current main still derives thread binding from explicit tool input and rejects `mode: "session"` without `thread: true`.

## Solution Assessment

Unclear. The requested ownership boundary is sound, but the exact SDK field and defaulting semantics need maintainer design so they do not conflict with existing headless-session and channel-binding contracts.
