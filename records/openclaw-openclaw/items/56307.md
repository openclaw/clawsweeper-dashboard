- `impact:session-state`: The requested policy controls whether spawned subagents become persistent session-bound work or one-shot runs.

## Summary

Keep open: current main and v2026.5.18 still lack a channel-owned spawn-default contract for automatically setting `thread` or `mode` in `sessions_spawn`; related open issues cover adjacent session/threading work but do not supersede this plugin SDK design question.

## What This Changes

Add a channel/plugin-owned threading policy so `sessions_spawn` can auto-default thread and mode for subagent spawns without channel-name hardcoding.

## Best Possible Solution

Define a backward-compatible plugin SDK spawn-default contract resolved before `sessions_spawn` schema exposure and validation, opt in only channels with safe binding support, and document/test both default and explicit-parameter behavior.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/API gap. Source inspection is high-confidence that current main still requires explicit `thread`/`mode` input and lacks channel-owned spawn-default metadata.

## Solution Assessment

Unclear until maintainer design review. The ownership boundary is sound, but the exact SDK field, schema exposure timing, and defaults need product/API judgment so they do not conflict with existing headless-session and channel-binding contracts.
