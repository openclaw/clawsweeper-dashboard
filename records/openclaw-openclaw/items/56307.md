## Summary

Keep open: current `main` still lacks a channel-owned spawn-default policy for automatically defaulting subagent `thread` and `mode`, and the related open threading issues cover adjacent session-binding behavior rather than this plugin SDK contract.

## What This Changes

The issue requests a channel plugin threading policy that lets channel metadata automatically inject `thread: true` and `mode: "session"` for subagent spawning instead of hardcoded channel names or prompt instructions.

## Best Possible Solution

Define a backward-compatible channel/plugin SDK spawn-default contract resolved before tool schema and execution, with opt-in metadata, docs, and focused tests for explicit versus defaulted `thread` and `mode` behavior.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a plugin SDK/runtime feature request. Source inspection confirms current main still lacks channel-owned automatic `thread`/`mode` defaulting.

## Solution Assessment

Unclear until maintainer design review. Moving ownership from hardcoded channel behavior into channel metadata is directionally sound, but the exact API, opt-in policy, and defaulting semantics need product/API approval.
