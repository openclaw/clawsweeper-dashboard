## Summary

Keep open: current main still has thread-binding support and Feishu binding hooks, but `sessions_spawn` still depends on explicit `thread`/`mode` inputs and there is no channel-owned spawn-default contract for automatic subagent parameter injection.

## What This Changes

The issue requests a channel plugin threading policy so channels such as Feishu can auto-apply subagent spawn `thread: true` and `mode: "session"` without hardcoded channel logic or prompt instructions.

## Best Possible Solution

Add a backwards-compatible plugin SDK/channel spawn-default contract resolved before `sessions_spawn` validation, then opt Feishu in only for supported current-conversation bindings with docs and focused regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature gap. Source inspection is high-confidence that current main still requires explicit `thread: true`/`mode: "session"` inputs and lacks a channel-owned default-injection contract.

## Solution Assessment

Unclear. The requested ownership boundary is sensible, but the maintainable solution should be a generic spawn-default or conversation-binding contract rather than overloading the existing reply-threading adapter semantics.
