## Summary

Keep open: current main has thread-binding policy helpers and spawn gating, but it still does not provide the requested channel-owned default that turns a bare subagent spawn into `thread: true` plus persistent session mode. The remaining work is an SDK/runtime behavior decision, not stale cleanup.

## What This Changes

Request a channel plugin threading policy that lets channels declare automatic `sessions_spawn` thread/session defaults for subagent spawns.

## Best Possible Solution

Define a backwards-compatible plugin SDK spawn-default contract that is resolved before `sessions_spawn` validation, with opt-in channel metadata, tests for default and explicit parameters, and docs for plugin authors.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature gap. Source inspection is high-confidence that current main still requires explicit `thread`/`mode` input and lacks channel-owned spawn-default metadata.

## Solution Assessment

Unclear pending maintainer design review. The ownership boundary is sound, but the exact SDK field and defaulting semantics need to avoid conflicting with existing headless-session, cleanup, and channel-binding contracts.
