- `impact:session-state`: The request changes whether spawned subagents become persistent thread-bound sessions and how follow-up state is associated.

## Summary

Keep open. Current main and v2026.6.8 still require explicit `thread: true` for session-mode native subagent spawning, the channel plugin threading adapter has no requested auto-injection policy, and this remains a plugin API/product decision rather than cleanup-closeable.

## What This Changes

The issue requests a channel plugin threading policy so channels can declare automatic `thread: true` and `mode: "session"` injection for subagent spawning.

## Best Possible Solution

Define one explicit channel-owned session/thread contract, then update the SDK type, `sessions_spawn` schema/execution, channel metadata, docs, and focused tests together while coordinating with the persistent-session decoupling discussion.

## Reproduction Assessment

Yes, at source level. Current main still derives `requestThreadBinding` only from `params.thread === true`, rejects `mode="session"` without that explicit flag, and has no channel threading policy fields for auto-injection.

## Solution Assessment

Unclear. A plugin-owned policy seam fits OpenClaw’s channel architecture, but the exact fields and defaults should be chosen with the broader non-thread persistent-session contract rather than added as a one-off boolean pair.
