## Summary

Keep open: current main still resets token-only WebChat URLs without an explicit session to `main`, and the only linked fix PR was closed unmerged after review found its token-equality reconnect signal unsafe for ordinary dashboard launches.

## What This Changes

Gateway crash/restart should restore the selected WebChat session and conversation history instead of opening a fresh session.

## Best Possible Solution

Define a reconnect/session-known URL contract that preserves the selected WebChat session after restart without weakening token-only dashboard launches, then update Control UI URL hydration and regression tests.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection shows the current URL hydration path resets token-only no-session chat URLs to `main` before Gateway connect, but the full Windows 11 crash/restart flow still needs a real repro or targeted regression.

## Solution Assessment

No. The closed token-equality PR is not the best fix because it conflicts with the existing token-only dashboard launch contract; a safer fix needs explicit session intent or a distinct reconnect marker.
