## Summary

Keep open: the report has concrete v2026.5.3 Docker-image evidence, and current main still shows an unresolved Docker distribution contract for Feishu: the release image only opts `diagnostics-otel` into extension dependency installation while Feishu is documented in some places like a bundled channel and cataloged elsewhere as an official external package.

## What This Changes

The issue asks for the Feishu plugin JavaScript runtime files to be bundled in the official Docker image so `channels.feishu.enabled` works without manual plugin installation.

## Best Possible Solution

Decide the official Docker policy for externalized channel plugins, then either ship Feishu completely in the Docker image with runtime dependencies and a regression check, or keep it external and fix docs/setup/diagnostics so users are not left with partial bundled metadata.

## Reproduction Assessment

No high-confidence live reproduction was run against the Docker image. The reporter provided concrete v2026.5.3 artifact output, and source inspection shows the current Docker release path still excludes Feishu from the extension dependency opt-in set.

## Solution Assessment

Unclear until maintainers choose the distribution contract. Adding `feishu` to the Docker release build args is narrow if Docker should bundle it, but the safer alternative may be to keep Feishu external and make install-on-demand/docs avoid partial runtime discovery.
