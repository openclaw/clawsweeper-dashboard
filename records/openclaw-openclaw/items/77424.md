## Summary

Keep open: the report gives concrete Docker-image evidence for v2026.5.3, and current main still has a distribution-policy mismatch around Feishu being an official external npm/ClawHub plugin while the Docker release image only opts in `diagnostics-otel` during the extension dependency stage.

## What This Changes

The issue requests that the official v2026.5.3 Docker image include Feishu plugin JavaScript/runtime files so `channels.feishu` is discovered without manual npm installation.

## Best Possible Solution

Decide the official Docker policy for externalized channel plugins, then either ship Feishu completely in the Docker image with dependencies and a regression check, or keep it external and make install-on-demand/diagnostics avoid partial bundled metadata.

## Reproduction Assessment

No high-confidence live reproduction was run here. The issue provides concrete v2026.5.3 Docker listings, and source inspection shows the release image still only opts `diagnostics-otel` into the extension dependency stage while Feishu owns plugin-local runtime dependencies.

## Solution Assessment

Unclear until maintainers choose the distribution contract. Adding `feishu` to the official Docker extension build args is narrow if Docker should include it out of the box, but the safer policy alternative is to keep Feishu external and fix install-on-demand or diagnostics around that model.
