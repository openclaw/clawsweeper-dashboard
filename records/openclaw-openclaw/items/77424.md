## Summary

Keep open: source inspection supports the reported Docker packaging mismatch, and current main still needs a maintainer decision on whether Feishu should be fully shipped in the official Docker image or treated as an install-on-demand official external plugin.

## What This Changes

The issue reports that the v2026.5.3 official Docker image includes Feishu plugin metadata but omits the compiled JavaScript runtime files, so Feishu is not discovered until `@openclaw/feishu` is manually installed.

## Best Possible Solution

Decide the official Docker policy for externalized channel plugins, then either ship Feishu completely in the image with dependencies and regression coverage, or keep it external and make install-on-demand diagnostics/docs explicit while avoiding partial bundled metadata.

## Reproduction Assessment

No live Docker reproduction was run under the read-only constraint. Source inspection gives a clear current-main reproduction path for the packaging mismatch: Feishu metadata is emitted, but the Docker release only opts `diagnostics-otel` into extension runtime dependencies.

## Solution Assessment

Unclear until maintainers choose the distribution contract. Adding Feishu to the Docker extension build args is narrow if Docker should include it out of the box; otherwise the safer fix is to keep it external and remove/mask partial bundled metadata while improving doctor/docs guidance.
