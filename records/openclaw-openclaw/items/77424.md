## Summary

Keep open: the report provides concrete v2026.5.3 Docker artifact evidence and current source still shows an unresolved distribution-policy mismatch for Feishu between external install metadata and the official Docker image composition.

## What This Changes

The issue asks OpenClaw to ship the Feishu channel plugin's compiled runtime files in the official Docker image so `channels.feishu` loads without manual plugin installation.

## Best Possible Solution

Choose and codify one Docker policy: either include Feishu as a complete bundled Docker plugin with runtime dependencies and regression coverage, or keep it external and make setup, doctor, and docs install or diagnose it without leaving partial bundled metadata.

## Reproduction Assessment

No high-confidence live reproduction was run here. The issue provides concrete v2026.5.3 Docker evidence, and source inspection shows Feishu is still not opted into the official Docker extension dependency set.

## Solution Assessment

Unclear until maintainers choose the distribution contract. Adding Feishu to the Docker extension build args is narrow if Docker should bundle it, but the safer alternative may be an explicit external install-on-demand and doctor path.
