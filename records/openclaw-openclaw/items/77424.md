## Summary

Keep open: the v2026.5.3 report has concrete Docker artifact evidence, and current main still leaves Feishu in an unresolved distribution state: official external install metadata exists, but the release Docker workflow only opts `diagnostics-otel` into the extension dependency stage.

## What This Changes

The issue requests that the official Docker image include Feishu plugin JavaScript/runtime files so `channels.feishu.enabled` works without a manual npm plugin install.

## Best Possible Solution

Decide the Docker distribution contract, then either bundle Feishu completely with dependencies and a Docker regression check or make the external install-on-demand/docs/diagnostics path authoritative so partial bundled metadata does not confuse users.

## Reproduction Assessment

No high-confidence live reproduction was established in this review. The reporter provided concrete v2026.5.3 Docker listings, and source inspection shows current release packaging still does not opt Feishu into the Docker extension dependency stage.

## Solution Assessment

Unclear until maintainers choose the distribution policy. Adding Feishu to the Docker release extension set is the narrow fix if Docker should ship it out of the box; otherwise the safer solution is to keep Feishu external and fix install-on-demand/docs/diagnostics around that model.
