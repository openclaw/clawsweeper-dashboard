## What This Changes

The PR serializes selected MCP channel tool result arrays into primary text content via `summarizeResult`, preserves `structuredContent`, and adds unit coverage for `summarizeResult`.

## Best Possible Solution

Land a narrow MCP formatting fix that serializes the complete result objects text-only clients need, including `events_poll.next_cursor`, while preserving `structuredContent`, adding focused tests, and updating the changelog.

## Reproduction Assessment

Yes, at source level. Current main's helper returns only `<label>: <count>` in primary content, while the affected tools put usable data and the polling cursor only in `structuredContent`.

## Solution Assessment

No, not yet. The PR's direction matches the MCP structured-content compatibility guidance, but `events_poll` needs to serialize `{ events, next_cursor }` in primary text and the user-facing fix needs a changelog entry.
