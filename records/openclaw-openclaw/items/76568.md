## Summary

Keep this open: current main still has a source-level persistence gap matching the report, where routed block-streamed replies can be delivered while transcript mirroring is disabled and the final payload is later suppressed.

## What This Changes

The issue asks OpenClaw to persist assistant replies to the session JSONL transcript after tool-call turns even when the reply is visible in the Feishu channel.

## Best Possible Solution

Add channel-agnostic regression coverage for a tool-call/block-streaming turn where a routed block reply suppresses the final payload, then persist the delivered assistant text to the active session transcript exactly once.

## Reproduction Assessment

No high-confidence live reproduction was run. The source path is reproducible by inspection: routed block replies disable mirroring, and successful block streaming can suppress the final payload that would otherwise mirror the reply.

## Solution Assessment

Yes, the best fix is a narrow transcript-persistence repair in the generic reply routing/block-streaming path, not a Feishu-specific workaround. The repair should mirror only the delivered visible block reply that would otherwise lose its final payload, with idempotency to prevent duplicates.

## Repair Work Prompt

Fix the transcript persistence gap reported in https://github.com/openclaw/openclaw/issues/76568. In current main, routed block replies in src/auto-reply/reply/dispatch-from-config.ts call sendPayloadAsync(..., false), while src/auto-reply/reply/agent-runner-payloads.ts drops final payloads after successful block streaming. Add a channel-agnostic fix so visible assistant text delivered via routed block streaming is appended to the active session JSONL exactly once when the final payload is suppressed. Prefer reusing appendAssistantMessageToSessionTranscript / delivery-mirror semantics with an idempotency key based on session/run/block identity. Do not add Feishu-specific behavior unless a generic seam cannot cover it. Add regression coverage for a routed channel with block streaming didStream=true, a tool-call/block flush, final payload suppression, and one transcript append.
