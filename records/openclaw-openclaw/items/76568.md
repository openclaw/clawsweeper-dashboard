## Summary

Current main already covers the reported loss mode: routed channel replies are mirrored into the session transcript, and session-file repair now preserves delivered trailing assistant/tool-call turns instead of dropping them on restart.

## What This Changes

The issue asks for Feishu-delivered assistant replies after tool-call turns to be persisted in the session `.jsonl` so they survive gateway restarts.

## Best Possible Solution

Keep the current main behavior and close this release-regression report; reopen with a current-main transcript if Feishu still loses delivered replies after the next build.

## Reproduction Assessment

No high-confidence current-main failure reproduction remains from source inspection: current main mirrors delivered channel replies into the session transcript and has repair tests preserving trailing assistant/tool-call replies. I did not run a live Feishu/DashScope turn because this sweep was read-only.

## Solution Assessment

Yes. The current solution is the narrow maintainable fix: make delivered replies durable through the existing transcript append path and prevent restart repair from rewriting them away, without adding a new queue mode or channel-specific policy.
