## Summary

Keep this PR open: the provider wildcard visibility change is useful, but the latest head still regresses Gateway `models.list` responsiveness by sending wildcard-configured default/configured views through full provider discovery instead of the existing bounded read-only path.

## What This Changes

The PR adds literal `provider/*` entries to `agents.defaults.models` across model selection, Gateway model listing, `/models` commands, Discord pickers, docs, changelog, and focused tests.

## Best Possible Solution

Keep the `provider/*` visibility contract, but implement it on top of the existing bounded read-only `models.list` path for non-`all` views.

## Reproduction Assessment

Yes, for the PR-head regression by source inspection: configure `agents.defaults.models` with `provider/*` and call default/configured `models.list` while catalog discovery is slow or stalled. The PR branch skips the current `Promise.race` timeout and awaits `readOnly: false`.

## Solution Assessment

No as submitted. `provider/*` is a reasonable compact contract, but Gateway model listing should preserve the current read-only 750 ms fallback and apply wildcard filtering after the bounded catalog selection.
