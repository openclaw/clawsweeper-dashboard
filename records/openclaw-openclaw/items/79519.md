## Summary

Keep open: the prior schema-help drift is fixed and the PR includes live Gateway proof, but the latest head still has a source-proven wildcard-only empty-catalog leak through consumers that treat an empty `allowedCatalog` as unrestricted.

## What This Changes

The PR adds literal `provider/*` support in `agents.defaults.models` across shared model selection, Gateway `models.list`, `/model` and `/models`, Discord picker behavior, docs, config help, changelog, and regression tests.

## Best Possible Solution

Ship provider wildcard visibility only after the shared allowlist state and all consumers can represent provider-restricted empty catalog results without broadening, with docs/schema/tests kept aligned.

## Reproduction Assessment

Yes. The original gap is source-reproducible on current main because `agents.defaults.models` is exact-model only, and the PR regression is source-reproducible on head with wildcard-only configs whose provider has no catalog rows.

## Solution Assessment

No as submitted. The literal `provider/*` direction is plausible, but the current patch still lets an empty wildcard-restricted catalog look unrestricted to existing consumers; fix the shared contract or consumers before merge.
