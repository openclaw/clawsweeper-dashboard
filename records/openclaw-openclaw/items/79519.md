## Summary

Keep open: current main does not implement provider-scoped dynamic model visibility, and the PR has sufficient live Gateway proof, but the latest head still drops same-provider exact allowlist entries from visible picker/list catalogs when mixed with a provider wildcard.

## What This Changes

The PR adds literal `provider/*` support in `agents.defaults.models` across shared model selection, Gateway model listing, `/models`, Discord picker behavior, docs, changelog, config help, and focused tests.

## Best Possible Solution

Land the `provider/*` contract only after the shared visible catalog preserves wildcard-discovered rows plus every exact allowlist row, including same-provider synthetic entries, with regression coverage on the shared resolver and picker/list surfaces.

## Reproduction Assessment

Yes, by source inspection: on PR head, a config mixing `openai-codex/*` with exact `openai-codex/custom-preview` can synthesize the exact row, then `resolveVisibleModelCatalog` filters same-provider `allowedCatalog` rows out before returning picker/list visibility.

## Solution Assessment

No as submitted: `provider/*` is a reasonable compact contract, but the visible-catalog merge must keep exact same-provider entries instead of treating wildcard-provider discovery as a replacement for the exact allowlist rows.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve same-provider exact allowlist rows:** `src/agents/model-catalog-visibility.ts:83-85`
  - body: When a config mixes `openai-codex/*` with an exact `openai-codex/custom-preview` entry that is not returned by discovery or `models.providers.*.models`, `buildAllowedModelSet()` creates a synthetic exact row, but this filter removes every `allowedCatalog` row whose provider also has a wildcard. That makes `models.list`, `/models`, and picker data hide an explicitly configured model even though the PR advertises that exact entries can be mixed with wildcard providers. Keep the exact/synthetic row in the visible union, deduping only when the same provider/model is already present.
  - confidence: 0.89
