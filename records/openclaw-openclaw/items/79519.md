## Summary

Keep open: current main does not implement provider wildcard model visibility, and the PR head still has two source-reproducible visibility bugs that can either broaden a restricted catalog or hide an explicitly configured model. The supplied Gateway live-output proof is useful, and the security pass found no supply-chain or secret-handling concern.

## What This Changes

The PR adds `provider/*` model visibility support across shared model selection, Gateway `models.list`, `/model` and `/models`, Discord picker behavior, docs, changelog, and focused tests.

## Best Possible Solution

Land the `provider/*` contract only after the shared policy and every consumer enforce restricted-empty wildcard results, preserve exact same-provider entries, and keep arbitrary glob support out of scope.

## Reproduction Assessment

Yes for the PR defects by source inspection: PR head can return `allowAny: false` with `allowedCatalog: []` for wildcard-only configs while TUI broadens on empty catalogs, and it filters same-provider exact rows from visible catalog output. I did not run a live failing command in this read-only review.

## Solution Assessment

No as submitted. The `provider/*` direction is a reasonable compact contract, but the implementation needs to enforce empty wildcard results everywhere and preserve exact rows before it is the maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Keep empty wildcard catalogs restricted:** `src/agents/model-selection-shared.ts:742-745`
  - body: With only `agents.defaults.models = {"vllm/*": {}}` and no visible vLLM rows, this path now returns `allowAny: false` with `allowedCatalog: []` because the wildcard key makes `allowedKeys` non-empty. Existing TUI code still treats an empty `allowedCatalog` as no restriction and falls back to the full catalog, so a restricted provider policy can leak every provider. Make the policy or consumer carry a restricted-empty result instead of broadening.
  - confidence: 0.91
- **[P2] Keep exact entries on wildcard providers visible:** `src/agents/model-selection-shared.ts:1079-1081`
  - body: Mixed configs such as `openai-codex/*` plus `openai-codex/custom-preview` synthesize the exact row in `allowedCatalog` when discovery lacks it, but this filter drops all allowed rows whose provider is wildcarded. That hides explicitly configured same-provider models from `models.list`, `/models`, and picker data; dedupe by model key instead of filtering by provider.
  - confidence: 0.9
