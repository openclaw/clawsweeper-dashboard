## Summary

Keep open: current main still lacks the provider-scoped dynamic visibility requested by https://github.com/openclaw/openclaw/issues/79485, and this PR is the active implementation, but the patch still leaves existing model-list consumers able to broaden a provider wildcard allowlist and the refreshed PR body lacks after-fix real Gateway output.

## What This Changes

The PR adds literal `provider/*` support inside `agents.defaults.models` and threads it through model selection, Gateway `models.list`, `/models`, Discord picker docs, changelog, config baseline, and targeted tests.

## Best Possible Solution

Land a single accepted config contract that keeps provider-scoped discovery closed across Gateway, slash commands, Discord, TUI, and session override surfaces, with live Gateway proof and targeted coverage.

## Reproduction Assessment

Yes, by source inspection: current main reads every `agents.defaults.models` key as an exact model ref and visible catalog only returns exact `allowedCatalog` rows when present. There is no provider-only dynamic visibility mode on current main.

## Solution Assessment

No as submitted: the `provider/*` shape is a plausible shared contract, but the patch should either populate `allowedCatalog` for wildcard rows or migrate every `allowedCatalog` consumer before merge. It also needs real after-fix Gateway output from the contributor's setup.
