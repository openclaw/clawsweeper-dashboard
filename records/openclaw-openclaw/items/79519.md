## Summary

Keep open: the provider wildcard contract is useful and has live Gateway proof, but the latest PR head still regresses current main's bounded default/configured `models.list` path by awaiting full provider discovery whenever a `provider/*` entry is configured.

## What This Changes

Adds literal `provider/*` entries to `agents.defaults.models` across model-selection policy, Gateway `models.list`, `/models`, Discord picker behavior, docs/changelog, generated baselines, and focused tests.

## Best Possible Solution

Keep the `provider/*` visibility contract, but preserve the current read-only timeout-bounded `models.list` path for every non-`all` view and apply wildcard filtering after the bounded catalog selection.

## Reproduction Assessment

Yes, for the PR-head regression by source inspection: configure `agents.defaults.models` with a `provider/*` entry and call default/configured `models.list` while provider discovery is slow or stalled. The latest branch awaits `readOnly: false` before the existing timeout branch can run.

## Solution Assessment

No as submitted. `provider/*` is a reasonable compact contract, but Gateway model listing should keep the existing read-only 750 ms fallback and filter wildcard providers after catalog selection.

## Review Findings
