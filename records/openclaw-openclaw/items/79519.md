## Summary

Keep open: current main does not implement provider-scoped dynamic model visibility, and this PR is the active implementation for https://github.com/openclaw/openclaw/issues/79485. The latest PR head still has two source-reproducible visibility bugs around wildcard-only empty catalogs and exact same-provider allowlist rows, so it should not merge yet.

## What This Changes

The PR adds literal `provider/*` entries to `agents.defaults.models` and wires that visibility policy through shared model selection, Gateway `models.list`, `/models`, Discord picker behavior, docs, changelog, config help, and focused tests.

## Best Possible Solution

Land the `provider/*` contract only after the shared policy and all consumers enforce empty wildcard results without broadening, preserve exact same-provider entries, and cover the shared resolver, embedded TUI, Gateway, `/models`, and Discord picker seams.

## Reproduction Assessment

Yes. Source inspection on PR head shows both failing paths: wildcard-only configs can produce an empty `allowedCatalog` that an existing TUI consumer treats as unrestricted, and mixed wildcard-plus-exact configs drop exact same-provider synthetic rows from visible catalog output.

## Solution Assessment

No as submitted. The `provider/*` contract is a reasonable compact direction, but the implementation must preserve exact same-provider entries and make wildcard-only empty results fail closed for every consumer before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Keep wildcard-only empty catalogs restricted:** `src/agents/model-selection-shared.ts:827-831`
  - body: When `agents.defaults.models` contains only `provider/*` entries and no rows for those providers are present, this path falls through with `allowAny: false` but `allowedCatalog: []`. Existing consumers such as embedded TUI still use `allowedCatalog.length > 0 ? allowedCatalog : catalog`, so they broaden back to all providers instead of failing closed. Update the policy or those consumers so an empty wildcard result remains restricted.
  - confidence: 0.91
- **[P2] Preserve exact rows on wildcard providers:** `src/agents/model-selection-shared.ts:902-904`
  - body: For mixed configs such as `openai-codex/*` plus exact `openai-codex/custom-preview`, `buildAllowedModelSetWithFallbacks` synthesizes the exact row when discovery lacks it, but this filter drops `allowedCatalog` rows whose provider is also wildcarded. That can hide an explicitly configured model in `models.list`, `/models`, and picker data; dedupe by provider/model instead of filtering by provider.
  - confidence: 0.9
