## Summary

Keep this PR open: it addresses a real provider-scoped model visibility gap and supplies live Gateway proof, but the latest head still has a blocking wildcard allowlist edge where an empty provider catalog can fall back to the full model catalog in existing consumers.

## What This Changes

The PR adds literal `provider/*` entries to `agents.defaults.models` across model selection, Gateway `models.list`, `/models`, Discord picker behavior, docs, changelog, config help, and focused tests.

## Best Possible Solution

Keep the `provider/*` direction only if the shared allowlist contract and all picker/list consumers fail closed for empty or partial wildcard-provider catalogs.

## Reproduction Assessment

Yes, by source inspection: on PR head, `agents.defaults.models = {"vllm/*": {}}` with no current vLLM catalog rows yields an empty `allowedCatalog` while the embedded TUI falls back to the full catalog. I did not run tests because this review was read-only.

## Solution Assessment

No as submitted: `provider/*` is a reasonable compact contract, but the allowlist result must preserve wildcard restrictions even when a selected provider has no current catalog rows. The safer fix is to make the shared result or every consumer distinguish restricted-empty from unrestricted-empty.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve wildcard restrictions when catalogs are empty:** `src/agents/model-selection-shared.ts:824`
  - body: With `agents.defaults.models = {"vllm/*": {}}` and a catalog that currently has no vLLM rows, this returns `allowAny: false` with `allowedKeys` containing `vllm/*` but an empty `allowedCatalog`. Existing consumers such as `src/tui/embedded-backend.ts` still do `allowedCatalog.length > 0 ? allowedCatalog : catalog`, so they fall back to the full catalog and expose providers outside the wildcard policy. Materialize a restricted-empty/provider-scoped result or update those consumers to enforce `allowedKeys`/wildcards instead of treating an empty allowed catalog as unrestricted.
  - confidence: 0.9
