## Best Possible Solution

Land one canonical scanner fix that suppresses only phantom parent warnings, preserves real default-account warnings across account-based channels, propagates the decision through channel doctor hooks, and includes redacted real `openclaw doctor` proof.

## Reproduction Assessment

Yes, source-reproducible. Current main scans the top-level channel record before account records, and the linked issue plus PR tests show how that can produce a false parent warning when real accounts are covered; I did not run `openclaw doctor` because this review is read-only.

## Solution Assessment

No. The direction is useful, but the patch is not the best fix because it bypasses only the generic warning builder, not Telegram's channel-owned warning hook, and it encodes Telegram-only default-account logic in a shared scanner.
