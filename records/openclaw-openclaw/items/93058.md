## Summary

Keep open: the PR targets a real, source-reproducible doctor false positive, but the current patch skips the entire top-level diagnostic pass whenever an accounts map exists, which can hide existing DM and plugin warning paths and is consistent with the focused doctor-shared CI failure.

## What This Changes

This PR guards the top-level doctor empty-allowlist `checkAccount` call when accounts exist and adds a regression test for populated per-account `groupAllowFrom` lists.

## Best Possible Solution

Preserve the top-level diagnostic pass and suppress only the false parent `groupAllowFrom` advisory when enabled accounts have effective group allowlists, then land one canonical PR with real doctor or direct-scanner proof.

## Reproduction Assessment

Yes, source-reproducible. Current main checks the top-level channel record before account records, and the linked issue provides the config shape that makes the parent warning false; I did not run tests because this review is read-only.

## Solution Assessment

No. The scanner is the right layer, but this PR skips the whole parent account pass; the safer fix is to suppress only the false group warning while retaining DM and plugin diagnostics.
