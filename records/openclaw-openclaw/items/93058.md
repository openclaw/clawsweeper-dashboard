## What This Changes

The PR changes the shared doctor empty-allowlist scanner to suppress the parent `groupAllowFrom` warning when enabled accounts appear to provide effective per-account group allowlists, and adds regression tests for Telegram-shaped configs.

## Best Possible Solution

Make the parent-warning suppression use shared channel account semantics or a narrow doctor hook, so root/default account warnings are preserved unless the root record is known to be only a fallback parent.

## Reproduction Assessment

Yes. Current main calls `checkAccount(channelConfig, channels.<channel>)` before iterating account records, so the parent fallback record can emit the reported warning.

## Solution Assessment

No. The fix direction is right, but the latest implementation still hard-codes Telegram implicit-default detection inside a shared scanner that serves other account-capable channels.
