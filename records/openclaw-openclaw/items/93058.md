Full review comments:

- **[P1] Account for implicit default before skipping the warning:** `src/commands/doctor/shared/empty-allowlist-scan.ts:101-115`
  - body: This check treats the named `accounts` map as the whole active account set. Telegram also creates an implicit `default` account from top-level `botToken`, `tokenFile`, or `TELEGRAM_BOT_TOKEN`; when that default scope has empty `groupAllowFrom`/`allowFrom` and the named accounts are covered, this passes `skipGroupAllowFromWarning` and hides the real top-level/default-account warning. Include that default scope, or only skip when the root is truly fallback-only.
  - confidence: 0.9
- **[P2] Use the current Telegram credential key in the fixture:** `src/commands/doctor/shared/empty-allowlist-scan.test.ts:140`
  - body: `channels.telegram.token` is not part of `TelegramConfig`, so the latest `check-test-types` job fails with TS2353 on this fixture. Use `botToken` or `tokenFile`, and keep the expected warning behavior aligned with the implicit default account that those keys create.
  - confidence: 0.95
