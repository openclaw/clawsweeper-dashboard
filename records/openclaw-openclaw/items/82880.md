Full review comments:

- **[P1] Keep non-HTTP scrape targets blocked:** `extensions/firecrawl/src/firecrawl-client.ts:547`
  - body: Current main rejects non-HTTP scrape URLs before any Firecrawl request is built. This branch only runs the new DNS branch for http/https and otherwise continues, so values such as file URLs can reach the Firecrawl POST path instead of being blocked by OpenClaw.
  - confidence: 0.95
- **[P1] Preserve the Firecrawl scrape hostname:** `extensions/firecrawl/src/firecrawl-client.ts:555-556`
  - body: Replacing the requested hostname with the first resolved IP changes what Firecrawl is asked to scrape and can break normal HTTPS virtual-host behavior. The branch needs a design that preserves current URL semantics unless maintainers explicitly accept a breaking mode with proof.
  - confidence: 0.92
- **[P1] Wire ACPX signatures through the launch path:** `extensions/acpx/src/runtime-internals/mcp-proxy.mjs:11-13`
  - body: The proxy only enforces HMAC when OPENCLAW_MCP_PROXY_SECRET is set, but the PR does not add the launcher-side secret generation and payload signing that real ACPX launches would need. Defaults remain unsigned, and setting the env can break actual proxy launches.
  - confidence: 0.89
- **[P2] Bind the allowed-command policy to exact argv:** `extensions/acpx/src/runtime-internals/mcp-proxy.mjs:118`
  - body: startsWith on only target.command is too weak for a command-execution boundary: a broad prefix can match sibling paths, and an allowed executable can still receive attacker-controlled args. Bind the exact executable plus expected argv, or avoid claiming this as a hard security boundary.
  - confidence: 0.82
- **[P2] Remove the unused Firecrawl hostname local:** `extensions/firecrawl/src/firecrawl-client.ts:554`
  - body: The PR introduces originalHostname in production TypeScript and never uses it. With extension noUnusedLocals enabled, the PR's check-prod-types job reports TS6133 for this line.
  - confidence: 0.95
- **[P2] Return a complete pinned-hostname mock:** `extensions/firecrawl/src/firecrawl-dns-pinning.test.ts:17-20`
  - body: resolvePinnedHostnameWithPolicy returns hostname, addresses, and lookup, but the new test mock omits lookup. The PR's check-test-types job reports TS2741 here, so the test suite is not type-correct.
  - confidence: 0.94
- **[P2] Guard the recorded fetch call before reading it:** `extensions/firecrawl/src/firecrawl-dns-pinning.test.ts:54-55`
  - body: fetchSpy.mock.calls[0] can be undefined and the tuple has no guaranteed element at index 1, which the PR's check-test-types job reports. Assert the call exists and narrow the init before reading body.
  - confidence: 0.9
