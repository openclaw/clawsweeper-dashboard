## Summary

Keep this PR open, but it is not merge-ready: the Firecrawl change weakens an existing protocol block and changes normal scrape target semantics, while the ACPX signing path is verifier-only and the posted proof does not show the full real behavior claimed by the hardening.

## What This Changes

The PR adds env-gated HMAC and allowed-prefix checks to the ACPX MCP proxy, rewrites Firecrawl scrape URLs to a resolved IP, and adds targeted ACPX and Firecrawl tests.

## Best Possible Solution

Preserve current Firecrawl protocol and hostname behavior while adding a proven SSRF mitigation, and either wire ACPX signatures through the actual payload creation path with rollout tests or remove the verifier-only env gate.
