## Summary

Keep open. The latest PR fixes the earlier ESM import issue, but the remaining security design is not merge-safe: ACPX signing is optional and not wired into the production producer, Firecrawl rewrites normal HTTPS scrape targets to IP URLs, and the PR body only proves mocked/unit-test behavior.

## What This Changes

Adds optional HMAC signature verification and command-prefix checks to the ACPX MCP proxy, rewrites Firecrawl scrape URLs to DNS-pinned IP addresses, and adds ACPX/Firecrawl security tests.

## Best Possible Solution

Rework ACPX payload authentication at the production producer/consumer boundary, and handle Firecrawl SSRF with a host-preserving enforceable strategy or an explicit maintainer-approved fail-closed mode with upgrade proof.

## Reproduction Assessment

Yes. Source inspection of the PR head gives a high-confidence path: ACPX signature verification returns when the secret is absent and no production producer signs payloads, while Firecrawl replaces the requested hostname with the first resolved IP before posting the scrape body.

## Solution Assessment

No. The proposed fix mixes incomplete opt-in ACPX authentication with a Firecrawl URL rewrite that changes the user's target; the safer direction is a complete auth contract and host-preserving SSRF boundary or explicit fail-closed product decision.
