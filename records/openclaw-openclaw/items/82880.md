Overall confidence: 0.9

Full review comments:

- **[P1] Keep non-HTTP scrape targets blocked:** `extensions/firecrawl/src/firecrawl-client.ts:550`
  - body: The old `runFirecrawlScrape` called `assertFirecrawlScrapeTargetAllowed` before building any Firecrawl request. This branch only runs the new DNS branch for `http:`/`https:` and otherwise continues with the original `urlToScrape`, so `file:///...` and other non-HTTP targets are no longer blocked by OpenClaw before reaching Firecrawl.
  - confidence: 0.95
- **[P1] Preserve the Firecrawl scrape hostname:** `extensions/firecrawl/src/firecrawl-client.ts:558-559`
  - body: Replacing `parsed.hostname` with the first resolved IP changes the URL sent to Firecrawl, which can break normal HTTPS virtual hosts/certificates and scrape attribution; the added comment even notes Firecrawl may reject IP-based requests. Keep the original target URL unless there is a maintainer-approved Firecrawl-compatible pinning design.
  - confidence: 0.9
- **[P1] Wire ACPX signatures through the real launcher:** `extensions/acpx/src/runtime-internals/mcp-proxy.mjs:11-13`
  - body: The proxy only enforces HMAC when `OPENCLAW_MCP_PROXY_SECRET` is set, but this PR does not add any real launcher code that sets the secret and signs the `--payload`. Defaults remain unsigned, while setting the env manually would make actual ACPX proxy launches reject unsigned payloads.
  - confidence: 0.88
- **[P2] Bind command policy to exact argv:** `extensions/acpx/src/runtime-internals/mcp-proxy.mjs:114-115`
  - body: `OPENCLAW_MCP_PROXY_ALLOWED_COMMAND_PREFIX` checks only `target.command.startsWith(...)`, so allowing a generic executable such as `node` still permits dangerous attacker-controlled args, and similar path prefixes can match. Bind policy to the exact executable plus expected arguments, or remove this as a claimed security boundary.
  - confidence: 0.82
