Full review comments:

- **[P1] Keep non-HTTP scrape targets blocked:** `extensions/firecrawl/src/firecrawl-client.ts:547`
  - body: Current main rejects non-HTTP scrape URLs before any Firecrawl request is built. This branch only runs the new DNS branch for http/https and otherwise continues, so values such as file URLs can reach the Firecrawl POST path instead of being blocked by OpenClaw.
  - confidence: 0.95
- **[P1] Preserve the Firecrawl scrape hostname:** `extensions/firecrawl/src/firecrawl-client.ts:555-556`
  - body: Replacing the requested hostname with the first resolved IP changes what Firecrawl is asked to scrape and can break normal HTTPS virtual-host behavior. Keep the original URL semantics unless maintainers explicitly approve and prove a Firecrawl-supported pinned-target mode.
  - confidence: 0.92
- **[P1] Wire ACPX signatures through the launcher:** `extensions/acpx/src/runtime-internals/mcp-proxy.mjs:11-13`
  - body: The proxy only enforces HMAC when OPENCLAW_MCP_PROXY_SECRET is set, but the PR does not add launcher-side secret generation and payload signing for real ACPX launches. Defaults remain unsigned, and setting the env can break actual proxy launches.
  - confidence: 0.89
- **[P2] Bind the allowed-command policy to argv:** `extensions/acpx/src/runtime-internals/mcp-proxy.mjs:118`
  - body: startsWith on only target.command is too weak for a command-execution boundary: a broad prefix can match sibling paths, and an allowed executable can still receive attacker-controlled args. Bind the exact executable plus expected argv, or avoid presenting this as a hard security boundary.
  - confidence: 0.84
- **[P2] Remove the unused Firecrawl hostname local:** `extensions/firecrawl/src/firecrawl-client.ts:554`
  - body: The PR introduces originalHostname in production TypeScript and never uses it. The extension tsconfig enables noUnusedLocals, and CI reports TS6133 for this line.
  - confidence: 0.88
