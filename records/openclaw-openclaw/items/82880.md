Overall confidence: 0.92

Full review comments:

- **[P1] Keep non-HTTP scrape targets blocked:** `extensions/firecrawl/src/firecrawl-client.ts:544`
  - body: `runFirecrawlScrape` no longer calls the existing target validator. For a `file:` URL, the new http/https-only branch is skipped and `urlToScrape` remains the original value that is later posted to Firecrawl, regressing the documented block for non-HTTP targets.
  - confidence: 0.94
- **[P1] Preserve the Firecrawl scrape hostname:** `extensions/firecrawl/src/firecrawl-client.ts:555-556`
  - body: Replacing the requested hostname with an IP changes the URL sent to Firecrawl, so valid HTTPS targets behind CDNs, SNI, or virtual hosts can fail or fetch different content. This needs a host-preserving design or an explicit strict mode instead of changing the default scrape semantics.
  - confidence: 0.9
- **[P1] Wire ACPX signatures through the real launcher:** `extensions/acpx/src/runtime-internals/mcp-proxy.mjs:11-14`
  - body: The verifier only checks `OPENCLAW_MCP_PROXY_SECRET` inside the proxy, but the PR does not change the production payload producer to sign normal launches. Enabling the secret would reject current real payloads, while leaving it unset keeps the default path unsigned.
  - confidence: 0.88
- **[P2] Bind command policy to the arguments:** `extensions/acpx/src/runtime-internals/mcp-proxy.mjs:118`
  - body: The prefix guard validates only the executable, so an allowed interpreter can still run arbitrary flags or scripts through `target.args` if a payload is manipulated. The policy needs to bind the canonical command plus arguments, or rely on a signed full payload from the real producer.
  - confidence: 0.86
- **[P2] Return a complete pinned-host mock:** `extensions/firecrawl/src/firecrawl-dns-pinning.test.ts:18-21`
  - body: `resolvePinnedHostnameWithPolicy` returns a `PinnedHostname` with `hostname`, `addresses`, and `lookup`; the new mocked return value omits `lookup`, which strict test type checks should reject unless the mock uses a full pinned-host object or a typed helper.
  - confidence: 0.82
