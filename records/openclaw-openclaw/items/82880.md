## What This Changes

The PR adds optional HMAC and command-prefix checks to the ACPX MCP proxy, rewrites Firecrawl scrape targets to pinned IP URLs, and adds focused ACPX and Firecrawl tests.

## Best Possible Solution

Rework the security design before merge: ACPX needs full launch-payload binding at the producer or upstream ACPX boundary, and Firecrawl SSRF mitigation must preserve requested URL semantics or stay within the documented pre-forwarding checks.

- **[P1] Preserve the target hostname for Firecrawl scrapes:** `extensions/firecrawl/src/firecrawl-client.ts:555-556`
  - body: This rewrites a request like `https://example.com/page` to `https://<resolved-ip>/page` before sending it to hosted Firecrawl. That no longer asks Firecrawl for the user's original URL and can break ordinary HTTPS virtual-host/SNI routing; current main forwards `params.url` after prechecking the target, so this is a behavior regression rather than a safe SSRF fix.
  - confidence: 0.92

  - body: The new checks only work when env vars are manually present, and the OpenClaw ACPX launch path does not set the secret or sign the proxy payload. With default installs the claimed command binding is off, while setting the secret would make existing unsigned payloads fail before ACPX starts; the fix needs to bind the full producer-created command/args contract instead of adding proxy-local knobs.
  - confidence: 0.88
- **[P2] Remove the unused Firecrawl hostname variable:** `extensions/firecrawl/src/firecrawl-client.ts:554`
  - body: `originalHostname` is declared but never read, and `tsconfig.extensions.json` enables `noUnusedLocals`; the check-prod-types log reports TS6133 for this exact line, so the PR cannot pass prod type checks as submitted.

## Security Review

Status: needs_attention

Summary: The diff changes command-execution and SSRF defenses, but both proposed protections are incomplete or incompatible as currently wired.

Concerns:

- **[high] ACPX command binding is not enforced by OpenClaw:** `extensions/acpx/src/runtime-internals/mcp-proxy.mjs:51`
  - body: The proxy only verifies HMAC payloads when OPENCLAW_MCP_PROXY_SECRET is present, and the inspected OpenClaw launch path does not set that secret or sign payloads, so default users do not receive the claimed command binding.
  - confidence: 0.88
- **[high] Firecrawl pinning alters the remote target identity:** `extensions/firecrawl/src/firecrawl-client.ts:555`
  - body: Replacing the URL hostname with an IP before sending the request to hosted Firecrawl changes the target identity and can break or misdirect HTTPS virtual-hosted scrapes while giving a false sense of complete remote SSRF protection.
  - confidence: 0.9
- **[medium] Prefix-only ACPX policy does not bind arguments:** `extensions/acpx/src/runtime-internals/mcp-proxy.mjs:118`
  - body: When configured without the HMAC secret, OPENCLAW_MCP_PROXY_ALLOWED_COMMAND_PREFIX validates only target.command and leaves target.args mutable, so it is not a complete command-execution boundary by itself.
  - confidence: 0.83
