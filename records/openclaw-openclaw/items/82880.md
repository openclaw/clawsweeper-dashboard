  - body: Current main rejects non-HTTP scrape URLs before any Firecrawl request is built. This branch only runs the new DNS branch for `http:`/`https:` and otherwise continues, so values such as `file:///etc/passwd` can reach the Firecrawl POST body instead of being blocked by OpenClaw.
  - confidence: 0.95

  - body: Replacing the requested hostname with the first resolved IP changes what Firecrawl is asked to scrape and can break normal HTTPS virtual-host and certificate behavior. Keep the original URL unless maintainers approve and prove a Firecrawl-compatible pinning design.
  - confidence: 0.9
- **[P1] Wire ACPX signatures through the launch path:** `extensions/acpx/src/runtime-internals/mcp-proxy.mjs:11-13`
  - body: The proxy only enforces HMAC when `OPENCLAW_MCP_PROXY_SECRET` is set, but the PR does not add the launcher-side secret generation and payload signing that real ACPX launches would need. Defaults remain unsigned, and setting the env can break actual proxy launches.
  - confidence: 0.88
- **[P2] Bind the allowed-command policy to exact argv:** `extensions/acpx/src/runtime-internals/mcp-proxy.mjs:116-118`
  - body: `startsWith` on only `target.command` is too weak for a command-execution boundary: a broad prefix can match sibling paths, and an allowed executable such as `node` can still receive attacker-controlled args. Bind the exact executable plus expected argv, or avoid claiming this as a hard security boundary.
  - confidence: 0.82
- **[P2] Remove the unused Firecrawl hostname variable:** `extensions/firecrawl/src/firecrawl-client.ts:554`
  - body: The PR introduces `originalHostname` in production TypeScript and never uses it, which leaves the branch with a straightforward type/lint blocker on top of the behavioral issues.
  - confidence: 0.78
