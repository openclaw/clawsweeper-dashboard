## Summary

Keep open, but this PR is not merge-ready: the Firecrawl hardening changes every scrape target into a raw IP URL, and the ACPX signature gate is not wired into the OpenClaw/acpx launch path that would need to produce signed payloads.

## What This Changes

## Best Possible Solution

Use a supported security boundary before merge: keep Firecrawl target URLs hostname-preserving unless a Firecrawl-supported pinning/Host strategy is proven, and wire ACPX command binding end to end at the launcher/upstream acpx seam with upgrade-safe tests.
