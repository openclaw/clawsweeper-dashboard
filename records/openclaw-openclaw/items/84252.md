## Summary

This should stay open: current main has a legacy Codex OAuth sidecar doctor repair, but source inspection still supports the reported partial-repair shape where unresolved sidecar profiles can remain operator-misleading, gateway auth state is cached outside the doctor process, and source-checkout service repair errors can mask the auth repair outcome.

## What This Changes

Make doctor, models status, and gateway auth handling fail closed and operator-visible for legacy `openai-codex` OAuth sidecar profiles after v2026.5.18 upgrades.

## Best Possible Solution

Make unresolved legacy Codex sidecar OAuth state a first-class auth-health failure in doctor/status/runtime surfaces, and make doctor either invalidate the running gateway auth state or print an explicit restart requirement after auth profile mutation.

## Reproduction Assessment

Yes from source inspection and reporter evidence, but not from a live macOS LaunchAgent run; the code supports partial sidecar repair, unresolved credential eligibility failure, and stale gateway auth status after CLI-side file mutation.

## Solution Assessment

No current fix is complete enough; the right path is a focused repair that aligns doctor lint/fix, models status, gateway auth refresh/restart guidance, and source-checkout service error scoping around the same auth-health contract.
