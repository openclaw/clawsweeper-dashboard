## What This Changes

The issue reports that Windows doctor/status treats the active OpenClaw Gateway Scheduled Task as a duplicate gateway-like service and suggests deleting it.

## Best Possible Solution

Normalize Windows Scheduled Task names before extra-service classification, add a regression for `\OpenClaw Gateway`, and keep stale or legacy task detection intact.

## Solution Assessment

Yes, a narrow normalization fix in the Windows extra-service classifier is the best first step. Parent-process correlation can remain a follow-up only if stale-path detection still needs stronger proof after the name normalization bug is fixed.
