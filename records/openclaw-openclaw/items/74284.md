## Summary

Keep open. Current main still reuses stored session model overrides during heartbeat runs when no explicit `heartbeat.model` is configured, and open PR #74932 is the active closing implementation path.

## What This Changes

The issue asks heartbeat runs without explicit `heartbeat.model` to stop reusing a persisted auto-fallback model override after `agents.defaults.model.primary` changes.

## Best Possible Solution

Review and either land or supersede PR #74932 with focused coverage for clearing stale direct heartbeat auto-fallback overrides while preserving explicit heartbeat models, user overrides, legacy overrides, parent overrides, and unchanged-default fallback persistence.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is an `agent:main:main` session entry with `modelOverrideSource: "auto"` and older fallback notice metadata, followed by a heartbeat tick after the configured default model changes and without `heartbeat.model`.

## Solution Assessment

Yes in direction. The narrow maintainable fix belongs in the model-selection/session-override seam for direct heartbeat auto fallback state, not in broad startup migration or blanket override clearing.
