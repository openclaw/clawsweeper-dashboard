## Summary

Keep open. Current main still lets heartbeat runs without an explicit `heartbeat.model` apply stored session model overrides, and the current tests/docs preserve auto-fallback overrides until reset. The provided context shows #74932 is an open, unmerged implementation candidate that explicitly targets this issue, so the issue should stay open until that PR lands or is superseded.

## What This Changes

The issue requests that heartbeat sessions re-resolve the updated default model after config changes instead of continuing to use a stale persisted auto-fallback model from the existing `agent:main:main` session.

## Best Possible Solution

Land or supersede #74932 with a narrow heartbeat/default-model-change rule that clears only stale direct `modelOverrideSource: "auto"` session overrides while preserving explicit heartbeat models, user selections, legacy overrides, parent inheritance, channel model selection, and unchanged-default fallback persistence.

## Reproduction Assessment

Yes. Seed `agent:main:main` with a direct auto-sourced stored model override, change `agents.defaults.model.primary`, then run heartbeat without `heartbeat.model`; current main applies the stored override because `hasResolvedHeartbeatModelOverride` remains false.

## Solution Assessment

Yes for the linked implementation direction, but not for closing now. A targeted stale direct heartbeat auto-fallback check is narrower than resetting the whole session or clearing all stored model overrides.
