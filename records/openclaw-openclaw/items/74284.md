## Summary

Keep open. Current main still lets heartbeat runs without an explicit heartbeat model reuse stored session model overrides, including auto-fallback overrides created under an older default, and PR #74932 is still the open implementation candidate linked with closing syntax.

## What This Changes

Heartbeat runs should follow updated `agents.defaults.model.primary` after config changes instead of continuing to use a persisted auto-fallback model from the existing `agent:main:main` session.

## Best Possible Solution

Land or supersede PR #74932 with a targeted heartbeat/default-model-change rule that clears only stale direct auto-fallback session overrides, preserves explicit `heartbeat.model` and user/legacy/parent override behavior, and updates the model-failover documentation and changelog as needed.

## Reproduction Assessment

Yes. Seed `agent:main:main` with `providerOverride`, `modelOverride`, and `modelOverrideSource: "auto"` from an older default, change `agents.defaults.model.primary`, then run heartbeat without `heartbeat.model`; current main applies the stored override because `hasResolvedHeartbeatModelOverride` remains false.

## Solution Assessment

No for current main. The maintainable solution is a narrow stale-heartbeat auto-fallback detection path rather than a broad session reset, and the open linked PR is the right implementation candidate to review or supersede.
