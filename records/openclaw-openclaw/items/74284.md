## Summary

Keep open. Current main still applies stored session model overrides during heartbeat runs when no explicit heartbeat model is configured, and open PR #74932 is already the active closing implementation path.

## What This Changes

The issue reports that heartbeat sessions keep using a stale persisted auto-fallback model after `agents.defaults.model.primary` changes across gateway restarts.

## Best Possible Solution

Review and either land or supersede PR #74932 with focused coverage for stale direct heartbeat auto-fallback overrides while preserving explicit heartbeat and user-selected model behavior.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a direct `agent:main:main` session override with `modelOverrideSource: "auto"` and old `fallbackNoticeSelectedModel`, then a heartbeat tick after the configured default changes and with no explicit `heartbeat.model`.

## Solution Assessment

Yes in direction. The narrow maintainable fix belongs in the heartbeat/model-selection stored-override seam, not in broad startup migration or blanket override clearing.
