## Summary

Keep open. Current main still applies stored session model overrides on heartbeat runs when no explicit `heartbeat.model` is configured, so the stale auto-fallback path remains. The linked fix PR is open and unmerged, so this should not be closed before that PR lands or is superseded.

## What This Changes

The issue reports that heartbeat sessions keep using a persisted auto-fallback model after `agents.defaults.model.primary` changes across gateway restarts.

## Best Possible Solution

Land or supersede #74932 with a narrow heartbeat/default-model-change rule that clears only stale direct auto-fallback session overrides while preserving explicit `heartbeat.model`, user selections, legacy overrides, parent overrides, channel model selection, and unchanged-default auto fallback persistence.

## Reproduction Assessment

Yes. Seed `agent:main:main` with `providerOverride`, `modelOverride`, and `modelOverrideSource: "auto"`, change `agents.defaults.model.primary`, then run heartbeat without `heartbeat.model`; current main will apply the stored override because no heartbeat override was resolved.

## Solution Assessment

Yes for the linked implementation direction. A targeted stale direct heartbeat auto-fallback check is narrower than resetting the whole session or clearing all stored model overrides.
