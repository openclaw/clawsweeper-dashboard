## Summary

Keep open. Current main still applies persisted session model overrides during heartbeat runs when no explicit `heartbeat.model` is configured, and the supplied context shows open PR #74932 uses closing syntax for this issue, so the issue should stay open until that PR lands or is superseded.

## What This Changes

The issue reports that autonomous heartbeat sessions keep using a stale persisted auto-fallback model after `agents.defaults.model.primary` changes.

## Best Possible Solution

Review and either land or supersede PR #74932 with focused coverage for stale direct heartbeat auto-fallback overrides while preserving explicit `heartbeat.model`, user-selected overrides, legacy overrides, parent overrides, and unchanged-default auto fallback persistence.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a heartbeat run with no configured `heartbeat.model` and a session entry containing a direct `modelOverrideSource: "auto"` fallback override from an older default; current main will still apply that stored override.

## Solution Assessment

Yes in direction. The narrow maintainable fix belongs in the model-selection/session-override seam for heartbeat auto fallback state, not in broad startup migration or blanket override clearing.
