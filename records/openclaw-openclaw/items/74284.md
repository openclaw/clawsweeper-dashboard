## Summary

Keep open. Current main still has the reported heartbeat model-pinning path when no explicit `heartbeat.model` is configured, and open PR #74932 is the active closing implementation path rather than a reason to close this issue early.

## What This Changes

The issue reports that heartbeat sessions continue using a persisted auto-fallback model selected at session creation after `agents.defaults.model.primary` changes in `openclaw.json`.

## Best Possible Solution

Review and land or supersede #74932 with a narrow model-selection fix that clears only stale direct heartbeat auto-fallback overrides while preserving explicit heartbeat, user, legacy, parent, and unchanged-default selections.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a heartbeat run without `heartbeat.model` plus a session entry containing an older direct `modelOverrideSource: "auto"`; current main applies that stored override because stored override skipping is only tied to an explicit heartbeat model override.

## Solution Assessment

Yes in direction. The narrow maintainable solution belongs in stored model-selection state, not a startup migration or blanket reset of all session overrides.
