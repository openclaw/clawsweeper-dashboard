## Summary

Keep open. Current main still lets stored session model overrides win on heartbeat runs when no explicit `heartbeat.model` is configured, and open PR #74932 is the active closing implementation path.

## What This Changes

Heartbeat sessions should stop pinning a stale auto-fallback model after `agents.defaults.model.primary` changes and the gateway restarts.

## Best Possible Solution

Land or supersede a focused model-selection fix that clears only stale direct heartbeat auto-fallback overrides after the configured default changes while preserving explicit heartbeat, user, legacy, parent, and unchanged-default overrides.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a heartbeat run without `heartbeat.model` and a session entry containing an older direct `modelOverrideSource: "auto"`; current main applies that stored override because the explicit heartbeat override flag is false.

## Solution Assessment

Yes in direction. The narrow maintainable fix belongs in the stored model-selection seam rather than a startup migration or blanket reset of all session overrides.
