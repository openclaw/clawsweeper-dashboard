## Summary

Keep open. Current main still applies stored session model overrides to heartbeat runs unless an explicit `heartbeat.model` is resolved, so a direct auto-fallback override from an older default can still win. The provided context also shows #74932 is an open linked implementation candidate with closing syntax, so this issue should remain open until that PR is reviewed, landed, or superseded.

## What This Changes

The issue requests that heartbeat sessions stop reusing stale persisted auto-fallback model overrides after `agents.defaults.model.primary` changes.

## Best Possible Solution

Land or supersede #74932 with a narrow heartbeat/default-model-change rule that clears only stale direct auto-fallback session overrides, while preserving explicit `heartbeat.model`, user selections, legacy overrides, parent overrides, channel model selection, and unchanged-default auto fallback persistence.

## Reproduction Assessment

Yes. Seed `agent:main:main` with a direct `providerOverride`, `modelOverride`, and `modelOverrideSource: "auto"`, change `agents.defaults.model.primary`, and run heartbeat without `heartbeat.model`; current main still resolves the stored override because no explicit heartbeat model was resolved.

## Solution Assessment

Yes for the linked implementation direction. A targeted stale direct heartbeat auto-fallback check is the narrowest maintainable fix compared with resetting the whole session or clearing all stored model overrides.
