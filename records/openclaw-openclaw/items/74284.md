## Summary

Keep open. Current main still routes heartbeat runs without an explicit `heartbeat.model` through stored session model overrides, and #74932 is an open linked implementation candidate with closing syntax, so this should remain open until that PR lands or is superseded.

## What This Changes

The issue requests that heartbeat session `agent:main:main` stop reusing stale persisted auto-fallback model overrides after `agents.defaults.model.primary` changes in `openclaw.json` and the gateway restarts.

## Best Possible Solution

Land or supersede #74932 with a narrow heartbeat/default-model-change rule that clears only stale direct auto-fallback session overrides while preserving explicit `heartbeat.model`, user selections, legacy overrides, parent overrides, channel model selection, and unchanged-default auto fallback persistence.

## Reproduction Assessment

Yes. Seed `agent:main:main` with a direct `providerOverride`, `modelOverride`, and `modelOverrideSource: "auto"` from an older default, change `agents.defaults.model.primary`, then run heartbeat without `heartbeat.model`; current main will apply the stored override because no explicit heartbeat override was resolved.

## Solution Assessment

Yes for the linked implementation direction. A targeted stale direct heartbeat auto-fallback check is narrower and safer than resetting the whole session or clearing all stored model overrides.
