## Summary

Keep open. Current main still applies stored heartbeat session model overrides when no explicit `heartbeat.model` is configured, and open PR #74932 is already the paired implementation candidate for this issue.

## What This Changes

Heartbeat runs should stop reusing an auto-pinned fallback model from an older `agents.defaults.model.primary` after the config changes.

## Best Possible Solution

Review and either land or supersede PR #74932 with focused coverage for stale direct heartbeat auto-fallback overrides while preserving explicit `heartbeat.model`, user and legacy overrides, parent overrides, and unchanged-default auto fallback persistence.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a heartbeat run with no configured `heartbeat.model` and a session entry containing a direct `modelOverrideSource: "auto"` fallback override from an older default; current main will still apply that stored override.

## Solution Assessment

Yes in direction. The narrow maintainable fix belongs in the model-selection/session-override seam for heartbeat auto fallback state, not in startup migration or blanket override clearing.
