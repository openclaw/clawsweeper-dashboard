## Summary

Keep open. Current main still lets heartbeat runs without an explicit `heartbeat.model` use stored session model overrides, and the linked open PR #74932 contains closing syntax for this issue, so the issue should remain open until that PR lands or is superseded.

## What This Changes

The issue asks heartbeat runs for `agent:main:main` to pick up changed `agents.defaults.model.primary` config instead of continuing to use a model override persisted from the session's earlier default or auto fallback.

## Best Possible Solution

Review, land, or supersede #74932 with a narrow heartbeat/default-model-change rule that clears or skips only stale direct auto-fallback session overrides while preserving explicit `heartbeat.model`, user selections, legacy overrides, parent overrides, channel model selection, and unchanged-default auto fallback persistence.

## Reproduction Assessment

Yes. Seed `agent:main:main` with a direct `providerOverride`, `modelOverride`, and `modelOverrideSource: "auto"`, change `agents.defaults.model.primary`, then run heartbeat without `heartbeat.model`; current main will apply the stored override because no explicit heartbeat override was resolved.

## Solution Assessment

Yes for the linked implementation direction. A targeted stale direct heartbeat auto-fallback check is narrower than resetting the whole session or dropping all stored model overrides on restart.
