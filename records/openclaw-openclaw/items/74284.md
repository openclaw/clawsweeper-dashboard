## Summary

Keep open. Current main still allows a stored heartbeat session model override to win when no explicit `heartbeat.model` is configured, and open PR #74932 is the active closing implementation path.

## What This Changes

The issue reports that heartbeat runs for `agent:main:main` keep using a session-persisted model after `agents.defaults.model.primary` changes and the gateway restarts.

## Best Possible Solution

Review and land or supersede PR #74932 with a narrow model-selection fix that clears only stale direct heartbeat auto-fallback overrides while preserving explicit heartbeat, user, legacy, parent, and unchanged-default selections.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a heartbeat run without `heartbeat.model` plus a session entry containing an older direct stored model override; current main applies that stored override because the explicit heartbeat override flag is false.

## Solution Assessment

Yes in direction, but it is not implemented on current main yet. The maintainable fix belongs in the stored model-selection seam rather than a startup migration or blanket session reset.
