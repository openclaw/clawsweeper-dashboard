## Summary

Keep open. Current main still lets persisted session model overrides win on heartbeat runs when no explicit `heartbeat.model` is configured, and #74932 is the unmerged closing implementation path.

## What This Changes

The issue requests that heartbeat `agent:main:main` sessions stop reusing stale persisted auto-fallback model overrides after default model config changes.

## Best Possible Solution

Land or supersede #74932 with a focused model-selection fix that clears only stale direct heartbeat auto-fallback overrides after the configured default changes while preserving explicit heartbeat, user, legacy, parent, and unchanged-default overrides.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a heartbeat run without explicit `heartbeat.model` plus a session entry containing an older direct `modelOverrideSource: "auto"`; current main applies the stored override because the explicit heartbeat override flag is false.

## Solution Assessment

Yes in direction. The narrow maintainable fix belongs in the stored model-selection seam rather than a startup migration or blanket reset of all session overrides.
