## Summary

Keep open. Current main still routes heartbeat runs without an explicit `heartbeat.model` through persisted session model overrides, and open PR #74932 is already the active closing path for the heartbeat-specific stale auto fallback bug.

## What This Changes

Heartbeat sessions should stop using a persisted auto-fallback model override after `agents.defaults.model.primary` changes and should follow the current default on later runs.

## Best Possible Solution

Review and either land or supersede PR #74932 with focused coverage for stale direct heartbeat auto fallback overrides while preserving explicit `heartbeat.model`, user-selected overrides, legacy overrides, parent overrides, and unchanged-default auto fallback persistence.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is an `agent:main:main` session entry with a direct `modelOverrideSource: "auto"` fallback override and older `fallbackNoticeSelectedModel`, no configured `heartbeat.model`, then a heartbeat tick after the default model changes.

## Solution Assessment

Yes in direction. The narrow maintainable solution is to handle stale direct heartbeat auto fallback state in the model-selection/session-override seam, not with a broad startup migration or blanket override clearing.
