## Summary

Keep open. Current main still lets heartbeat runs without an explicit `heartbeat.model` reuse stored session model overrides, including auto-fallback overrides selected under an older default, and #74932 is an open linked implementation candidate with closing syntax.

## What This Changes

The issue asks for autonomous heartbeat runs to stop pinning `agent:main:main` to an auto-fallback model persisted before `agents.defaults.model.primary` was changed.

## Best Possible Solution

Land or supersede #74932 with a narrow heartbeat/default-model-change rule that clears only stale direct auto-fallback session overrides while preserving explicit `heartbeat.model`, user selections, legacy overrides, parent overrides, channel model selection, and unchanged-default auto fallback persistence.

## Reproduction Assessment

Yes. Seed `agent:main:main` with `providerOverride`, `modelOverride`, and `modelOverrideSource: "auto"` from an older default, change `agents.defaults.model.primary`, then run heartbeat without `heartbeat.model`; current main applies the stored override because no explicit heartbeat override was resolved.

## Solution Assessment

Yes for the linked implementation direction. A targeted stale direct heartbeat auto-fallback check is narrower and safer than resetting the whole session or clearing all stored model overrides.
