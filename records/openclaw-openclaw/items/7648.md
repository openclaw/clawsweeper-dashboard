## Summary

Keep open. Current main still has bottom-oriented auto-follow behavior and manual-scroll preservation, but no disable/top/off preference, persisted setting, gateway config, or Scroll to Top action for Control UI/WebChat.

## What This Changes

Request an option or action to stop bottom auto-scroll in Control UI/WebChat so long replies can be read from the top.

## Best Possible Solution

Implement a maintainer-chosen per-user scroll preference or visible Scroll to Top affordance across browser Control UI and native WebChat, preserving manual-scroll and New messages behavior; add gateway-level policy only if maintainers choose that scope.

## Solution Assessment
