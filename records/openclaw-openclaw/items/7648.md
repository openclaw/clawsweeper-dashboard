## Summary

Keep open. Current main still has bottom-follow/manual-scroll protection, but it does not expose a Control UI/WebChat auto-scroll preference, top/off mode, or Scroll to Top action; the related TUI issue and open scroll-bug PRs cover different behavior rather than this product request.

## What This Changes

Request to add a Control UI/WebChat auto-scroll opt-out/top mode or a Scroll to Top affordance for long agent responses.

## Best Possible Solution

Implement one coherent per-user scroll preference or action across browser Control UI and native WebChat, preserving existing manual-scroll and new-message behavior; add gateway-level policy only if maintainers choose that scope.

## Reproduction Assessment

Not applicable as a feature request. Source inspection of current main gives high confidence that the requested Control UI/WebChat setting, top/off mode, or Scroll to Top action is still absent.

## Solution Assessment

Unclear until maintainers choose the UX shape. A persisted per-user UI preference or visible action is likely safer than a gateway-only config because this is mainly reading-position behavior, but the final scope needs product direction.
