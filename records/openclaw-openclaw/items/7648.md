## Summary

Keep open. Current main still has bottom-oriented Control UI/WebChat scroll behavior and manual-scroll preservation, but no persisted disable-auto-scroll preference, no top/off scroll mode, and no Scroll to Top action.

## What This Changes

Add a Control UI/WebChat way to disable bottom auto-scroll or jump to the top of long agent responses.

## Best Possible Solution

Add an explicit chat scroll preference or Scroll to Top action for browser Control UI and shared native WebChat, persist any per-user preference in UI settings, and leave TUI-specific work to #44130.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main gives high confidence that the requested Control UI/WebChat setting, top/off mode, or Scroll to Top action is not implemented.

## Solution Assessment

Yes, keeping this open is the best current outcome. The maintainable fix should preserve existing manual-scroll behavior while adding an explicit preference/action, with gateway config only if maintainers choose operator policy.
