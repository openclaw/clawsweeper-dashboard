Keep open. Current main still lacks a Control UI/WebChat disable-auto-scroll preference, top/off mode, or Scroll to Top action; the related TUI report #44130 does not supersede this browser/native WebChat request.

Implement an explicit chat scroll preference or Scroll to Top action across browser Control UI and native WebChat while preserving existing near-bottom/manual-scroll behavior; add gateway-level config only if maintainers want operator policy.
