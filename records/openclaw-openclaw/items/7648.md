## Summary

Keep open. Current main still has bottom-oriented Control UI/WebChat scroll behavior with manual-scroll preservation, but no persisted scroll mode, disable-auto-scroll toggle, or Scroll to Top action; the related TUI report tracks a separate surface.

## What This Changes

The issue requests a Control UI/WebChat option or action to avoid forced bottom auto-scroll on long agent responses.

## Best Possible Solution

Add an explicit chat scroll preference or Scroll to Top action for browser Control UI and shared native WebChat, persist any per-user preference in UI settings, and keep TUI-specific scroll work tracked separately in #44130.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main gives high confidence that the requested Control UI/WebChat setting or action is not implemented.

## Solution Assessment

Yes, keeping this open is the best current outcome. The maintainable fix should preserve existing manual-scroll behavior while adding an explicit preference/action, with gateway config only if maintainers want operator policy.
