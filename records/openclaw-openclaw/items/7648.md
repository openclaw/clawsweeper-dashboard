## Summary

Keep open. Current main still lacks a Control UI/WebChat auto-scroll preference or Scroll to Top action, and the related TUI report is a separate terminal-surface issue rather than a superseding implementation.

## What This Changes

Request an option in Control UI/WebChat to disable bottom auto-scroll or jump to the top of long responses.

## Best Possible Solution

Add a persisted per-user chat scroll preference and/or explicit Scroll to Top action across browser Control UI and native WebChat, with gateway-level config only if maintainers want operator policy.

## Reproduction Assessment

Yes. Source inspection on current main shows bottom-follow behavior in browser and native chat paths, with no persisted UI setting, gateway config field, or Scroll to Top action.

## Solution Assessment

Yes for keeping this open. The narrowest maintainable direction is a UI-level preference/action first, while treating gateway config as a separate maintainer product decision.
