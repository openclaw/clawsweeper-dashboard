## Best Possible Solution

Land the per-agent bootstrap profile contract after maintainer approval with the `/context` truncation guidance updated to mention both default-level and per-agent limit keys.

## Reproduction Assessment

Not applicable: this PR implements a new config capability rather than fixing a directly reproducible bug. Source inspection confirms current main only uses `agents.defaults`, while PR head adds the per-agent schema, resolver plumbing, and focused tests.

## Solution Assessment

Yes, with one small follow-up: adding optional `agents.list[]` fields and threading the resolved session agent id through existing bootstrap resolvers is the narrow backward-compatible solution. The remaining diagnostic copy should be aligned so operators know a per-agent override may be the effective limit.
