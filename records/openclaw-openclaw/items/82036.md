## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Pass the agent id into context estimates:** `src/auto-reply/reply/commands-system-prompt.ts:40-45`
  - body: `resolveCommandsSystemPromptBundle()` resolves `sessionAgentId` for the active command, but this estimate path still calls `resolveBootstrapContextForRun()` without it. After this PR, bootstrap context resolution uses `agentId` to apply `agents.list[]` limits, so `/context` for a non-default agent without a run report can estimate/truncate with `agents.defaults` and display the wrong limits. Pass `agentId: sessionAgentId` here and use the same id for context-report fallback limit resolution.
