## Summary

Keep open. Current main and the latest shipped tag still lack a TUI `/watchdog` command or public/persisted watchdog setting, and the same-run repeated notice path is still source-reproducible; related open work covers parts of the problem but does not make this cleanup-closeable.

## What This Changes

Request a TUI `/watchdog` command to view, disable, or tune the streaming watchdog and suppress repeated same-run watchdog warnings.

## Best Possible Solution

Choose one canonical watchdog UX, then land a focused TUI change that preserves reconnect recovery while adding the selected off/duration control, run-scoped duplicate-warning handling, docs, and fake-timer coverage coordinated with https://github.com/openclaw/openclaw/issues/68596, https://github.com/openclaw/openclaw/pull/77375, and https://github.com/openclaw/openclaw/pull/79997.

Yes for source-level reproduction. Current main has no `/watchdog` or persisted setting, and the handler can clear `activeChatRunId`, re-adopt the same unfinished run on a later delta, and arm another timeout without a warned-run guard.

## Solution Assessment

Unclear. Per-run warning suppression is narrow, but the command/config/persistence surface overlaps active related work and needs a maintainer UX decision before implementation is queued.
