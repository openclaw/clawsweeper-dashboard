Keep open. Current main has softened and partially improved the TUI watchdog behavior, but it still lacks the requested `/watchdog` command, persisted user setting, and per-run duplicate-warning guard; related open issues and PRs need maintainer UX/scope choice before cleanup closure.

## What This Changes

The issue requests a TUI `/watchdog` command with persisted off/duration settings and per-run suppression of duplicate streaming watchdog warnings.

## Best Possible Solution

Choose the canonical TUI watchdog UX, then land a focused change that preserves current reconnect/proof-of-life behavior while adding a per-run warning guard, user-facing off/duration control, persistence or config wiring, docs, and focused fake-timer coverage.
