- `impact:session-state`: The watchdog path can clear the TUI active-run state and show idle while the same backend run may later continue, which is session-state drift.

## Summary

Keep open. Current main and the latest shipped tag still lack a TUI `/watchdog` command, a public/persisted watchdog setting, and run-scoped duplicate-warning suppression; related open work covers pieces of the request but is not merged and does not make this issue cleanup-closeable.

## What This Changes

The issue requests per-run suppression of duplicate TUI streaming watchdog warnings plus a `/watchdog` command to view, disable, tune, and persist the timeout.

## Best Possible Solution

Choose one canonical watchdog UX, then land coordinated TUI changes that preserve reconnect recovery while adding the selected off/duration control, run-scoped duplicate-warning behavior, persistence or documented config, and focused fake-timer coverage.

## Reproduction Assessment

Yes, at source level. Current main has no `/watchdog` command or persisted setting, and the handler can clear `activeChatRunId`, re-adopt the same unfinished run on a later delta, and arm another timeout without a warned-run guard.

## Solution Assessment

Unclear. Per-run warning suppression is narrow, but the command/config/persistence surface overlaps active related work and needs a maintainer UX decision before implementation is queued.
