## Summary

Keep open. Current main still has only an internal 30-second TUI streaming watchdog override, still emits the normal warning without a per-run warned set, and still lacks the requested `/watchdog` command or persisted preference; related #68596 and #69026 overlap but do not implement this combined request.

## What This Changes

The issue requests a TUI `/watchdog` command, persisted runtime watchdog control, and per-run suppression of duplicate streaming watchdog warnings.

## Best Possible Solution

Add a TUI-owned watchdog control with status/off/duration parsing, persistence under the existing OpenClaw state-dir pattern, completion/help/docs coverage, and a per-run warning guard that resets when the run finalizes, terminates, or the session changes, while preserving the merged reconnect/proof-of-life behavior.

## Reproduction Assessment

Yes. A focused fake-timer path can send a delta, advance past the watchdog window, send another delta for the same run after `activeChatRunId` was cleared, and advance again to observe another warning; `/watchdog` absence is directly visible in the command catalog and handler switch.

## Solution Assessment

Yes for the TUI-local portion, with one caveat: a per-run warned set plus local `/watchdog` preference is the narrowest implementation for this issue, while broader OpenClaw config should stay coordinated with #68596.
