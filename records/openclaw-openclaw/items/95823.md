## Summary

Keep open: current main and v2026.6.10 still scope doctor service handling to gateway-like `.service` units and do not enumerate OpenClaw-owned enabled-but-inactive `.path` or `.timer` user units, so the reported silent inbound delivery failure remains unimplemented.

## What This Changes

Request `openclaw doctor` and related post-upgrade/deep/lint flows to detect and optionally restart OpenClaw-owned enabled-but-inactive systemd user units beyond the gateway.

## Best Possible Solution

Add a scoped Linux doctor health check that reports owned enabled non-template user units that are inactive, emits lint JSON, and starts only approved safe unit types under explicit repair consent.

## Reproduction Assessment

Yes, as a source-level reproduction: current main scans gateway-like `.service` files and filters inactive non-legacy Linux units, with no inspected `list-unit-files`/`FragmentPath` enumeration for `.path` or `.timer` units. I did not live-run a multi-agent systemd host in this read-only review.

## Solution Assessment

Unclear until maintainers choose the lifecycle policy. Doctor is a plausible surface, but the best fix must define owned-unit detection, safe unit types, explicit repair consent, and lint/preview behavior before starting non-gateway units.
