## Summary

Keep open: current main and v2026.6.10 still only manage gateway service lifecycle and extra gateway-like `.service` detection, so the enabled-but-inactive `.path`/`.timer` user-unit request remains. The implementation needs maintainer lifecycle policy before doctor starts non-gateway user units.

## What This Changes

Request to extend `openclaw doctor` so it detects and optionally starts OpenClaw-owned enabled-but-inactive systemd user units beyond the gateway.

## Best Possible Solution

Add a scoped Linux doctor health/lint check that reports owned enabled non-template user units that are inactive, and starts only approved safe unit types under explicit repair consent.

## Reproduction Assessment

Yes for the source-level gap: current main handles gateway service lifecycle and extra gateway-like `.service` detection, but no inspected path enumerates enabled `.path` or `.timer` user units through `list-unit-files` and `FragmentPath`. I did not run a live multi-agent systemd reproduction.

## Solution Assessment

Unclear until maintainers choose the lifecycle policy. Doctor is likely the right surface, but the best fix must define ownership, safe unit types, explicit repair consent, and preview/lint behavior before starting non-gateway units.
