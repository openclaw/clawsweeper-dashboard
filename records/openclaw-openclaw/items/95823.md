## Summary

Keep open: current main and the latest release still audit and repair gateway-scoped supervisor state, but they do not enumerate OpenClaw-owned enabled systemd user `.path`/`.timer` units or restore enabled-but-inactive per-agent units. The request is useful, but starting non-gateway units needs maintainer lifecycle policy before implementation.

## What This Changes

Extend `openclaw doctor` to detect enabled-but-inactive OpenClaw-owned systemd user units and optionally start safe unit types under repair consent.

## Best Possible Solution

Add a scoped Linux doctor health check that reports owned enabled non-template user units that are inactive, emits lint JSON, and starts only approved safe unit types under explicit repair consent.

## Reproduction Assessment

Yes for the source-level feature gap. Current main scans gateway-like `.service` files and filters inactive non-legacy services, while no inspected doctor path enumerates enabled `.path` or `.timer` user units through `systemctl --user list-unit-files` and `FragmentPath`.

## Solution Assessment

Unclear until maintainers choose the lifecycle policy. The proposed doctor surface is likely right, but the repair must be scoped by owned-unit detection, safe unit type, explicit consent, and preview behavior.
