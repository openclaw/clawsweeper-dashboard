## Summary

Keep open: current main and v2026.6.10 still only scan gateway-like `.service` units and explicitly suppress inactive non-legacy Linux units, so the requested enabled-but-inactive `.path`/`.timer` user-unit reconciliation remains missing and needs maintainer lifecycle policy before implementation.

## What This Changes

The issue requests a doctor check and repair path for OpenClaw-owned enabled-but-inactive Linux `systemd --user` units beyond the gateway service.

## Best Possible Solution

Add a scoped Linux doctor health check that reports OpenClaw-owned enabled non-template user units that are inactive, exposes lint JSON findings, and starts only approved safe unit types under explicit repair consent.

## Reproduction Assessment

Yes for the source-level gap: current main scans `.service` files and filters inactive non-legacy systemd units, while no inspected path enumerates enabled `.path` or `.timer` user units. I did not live-run a multi-agent Linux host reproduction in this read-only review.

## Solution Assessment

Unclear until maintainers choose the lifecycle policy. Doctor is the right surface, but the best fix must define owned-unit detection, safe unit types, explicit repair consent, and lint/preview behavior before starting non-gateway units.
