## Summary

Keep open: current main only checks gateway-like service units and explicitly ignores inactive non-legacy Linux gateway-like units, while this issue asks for a broader doctor capability to find and optionally restart OpenClaw-owned enabled-but-inactive systemd user units such as per-agent path watchers.

## What This Changes

Add an OpenClaw doctor check and repair flow for enabled-but-inactive OpenClaw-owned systemd user units beyond the gateway service.

## Best Possible Solution

Add a narrow Linux/systemd doctor health check for OpenClaw-owned enabled user units, read-only by default, with cautious repair semantics and structured lint output coordinated with the existing doctor roadmap.

## Reproduction Assessment

Yes from source inspection: current doctor does not enumerate enabled systemd user units or FragmentPath ownership, and current docs/tests explicitly ignore inactive non-legacy Linux gateway-like units. I did not run a live systemd reproduction in this read-only review.

## Solution Assessment

Unclear until maintainer direction is set: the requested doctor surface is plausible, but the best implementation must define safe ownership detection, repair consent, and interaction with doctor dry-run/extensibility before starting user units.
