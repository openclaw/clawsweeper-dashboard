## Summary

Keep open: current main and v2026.6.10 still scope doctor service handling to gateway-like `.service` units and deliberately ignore inactive non-legacy Linux units, so the enabled-but-inactive `.path`/`.timer` user-unit failure remains unimplemented. The request is useful and source-reproducible, but repair needs maintainer lifecycle policy because doctor would start non-gateway user units.

## What This Changes

The issue requests extending `openclaw doctor` to detect and optionally restart OpenClaw-owned enabled-but-inactive systemd user units beyond the gateway.

## Best Possible Solution

Add a scoped Linux doctor health check that reports owned enabled non-template user units that are inactive, emits lint JSON, and starts only approved safe unit types under explicit repair consent.

## Reproduction Assessment

Yes for the source-level gap, not a runtime bug reproduction. Current main scans gateway-like `.service` files and filters inactive non-legacy Linux services, while no checked path enumerates enabled `.path` or `.timer` user units through `list-unit-files` and `FragmentPath`.

## Solution Assessment

Unclear until maintainers choose the lifecycle policy. Doctor is likely the right surface, but the best fix must define owned-unit detection, safe unit types, explicit repair consent, and preview/lint behavior before starting non-gateway units.
