## Summary

Keep open: current main and the latest release still scope doctor service handling to gateway-like `.service` units and deliberately suppress inactive non-legacy units, so the enabled-but-inactive `.path`/`.timer` user-unit problem remains. The proposed repair path is plausible, but it needs maintainer lifecycle policy before implementation because doctor would start non-gateway systemd units.

## What This Changes

The issue requests an `openclaw doctor` check and repair path for OpenClaw-owned enabled-but-inactive Linux user units such as per-agent `.path` watchers and timers.

## Best Possible Solution

Add a scoped Linux doctor health check that reports owned enabled non-template user units that are inactive, emits lint JSON, and starts only approved safe unit types under explicit repair consent.

## Reproduction Assessment

Yes for the source-level gap. Current main scans gateway-like `.service` files and filters inactive non-legacy services, while no inspected doctor path enumerates enabled `.path` or `.timer` user units through `systemctl --user list-unit-files` and `FragmentPath`.

## Solution Assessment

Unclear until maintainers choose the lifecycle policy. Doctor is likely the right surface, but the best fix must define owned-unit detection, safe unit types, explicit repair consent, and preview/lint behavior before starting non-gateway units.
