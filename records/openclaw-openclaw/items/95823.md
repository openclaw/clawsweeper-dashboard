## Summary

Keep open: current main still scopes doctor service handling to gateway-like `.service` units and explicitly filters inactive non-legacy Linux units, so the requested enabled-but-inactive `.path`/`.timer` reconciliation remains missing and needs maintainer lifecycle policy.

## What This Changes

Request to extend `openclaw doctor` to detect and repair OpenClaw-owned enabled but inactive systemd user units beyond the gateway service.

## Best Possible Solution

Add a scoped Linux doctor check that reports owned enabled non-template user units that are inactive, emits lint JSON, and starts only approved safe unit types under explicit repair consent.

## Reproduction Assessment

Yes for the source-level gap. Current main scans gateway-like `.service` files and filters inactive non-legacy units, with no inspected path that enumerates enabled `.path` or `.timer` user units through `list-unit-files` and `FragmentPath`.

## Solution Assessment

Unclear until maintainers choose the lifecycle policy. Doctor is likely the right surface, but the best fix must define owned-unit detection, safe unit types, explicit repair consent, and preview/lint behavior before starting non-gateway units.
