## Summary

Keep open: current main and v2026.6.10 still scope doctor service handling to gateway-like `.service` units and deliberately ignore inactive non-legacy Linux units, so the requested `.path`/`.timer` user-unit reconciliation remains unimplemented and needs maintainer lifecycle policy.

## What This Changes

Request to extend `openclaw doctor` so it detects OpenClaw-owned enabled-but-inactive systemd user units and optionally starts safe units under repair mode.

## Best Possible Solution

Add a scoped Linux doctor health check that reports owned enabled non-template user units that are inactive, emits lint JSON, and starts only approved safe unit types under explicit repair consent.

## Reproduction Assessment

Yes for the source-level gap, though I did not recreate the live multi-agent host incident. Current main and v2026.6.10 show `.service`-only gateway scanning, inactive non-legacy suppression, and no post-upgrade systemd unit reconciliation path.

## Solution Assessment

Unclear until maintainers choose the lifecycle policy. Doctor is the likely surface, but the best implementation must define owned-unit detection, safe unit types, explicit repair consent, and preview/lint behavior before starting non-gateway units.
