## Summary

Keep open: current main and v2026.6.10 still scope doctor service handling to gateway-like `.service` units and filter inactive non-legacy Linux units, so the requested enabled-but-inactive `.path`/`.timer` reconciliation remains. The issue is actionable as product work, but it needs maintainer lifecycle policy before doctor starts non-gateway user units.

## What This Changes

Request to extend `openclaw doctor` to detect and optionally restart OpenClaw-owned enabled-but-inactive Linux systemd user units such as per-agent `.path` and `.timer` watchers.

## Best Possible Solution

Add a scoped Linux doctor health check that reports owned enabled non-template inactive user units, emits lint JSON, and starts only maintainer-approved safe unit types under explicit repair consent.

## Reproduction Assessment

Yes for the source-level gap. Current main scans gateway-like `.service` files and filters inactive non-legacy Linux units, while no inspected doctor path enumerates enabled `.path` or `.timer` user units through `systemctl --user list-unit-files` and `FragmentPath`.

## Solution Assessment

Unclear until maintainers choose the lifecycle policy. Doctor is likely the right surface, but the best fix must define owned-unit detection, safe unit types, explicit repair consent, and preview/lint behavior before starting non-gateway units.
