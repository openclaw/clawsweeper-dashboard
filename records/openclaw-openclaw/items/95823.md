## Summary

Keep open: current main and v2026.6.10 still scope doctor service handling to gateway-like `.service` units and explicitly ignore inactive non-legacy Linux units, while no canonical PR now owns enabled `.path`/`.timer` reconciliation.

## What This Changes

The issue asks for `openclaw doctor` to detect and optionally start OpenClaw-owned enabled-but-inactive Linux user systemd `.path`, `.timer`, and service units beyond the gateway.

## Best Possible Solution

Add a scoped Linux doctor health check that reports owned enabled non-template inactive user units, emits lint JSON, and starts only approved safe unit types under explicit repair consent.

## Reproduction Assessment

Yes for a source-level reproduction: current main only scans gateway-like `.service` files and filters inactive non-legacy systemd units, with no checked path for enabled `.path`/`.timer` units via `list-unit-files` and `FragmentPath`. I did not run a live systemd host reproduction.

## Solution Assessment

Unclear until maintainers choose the lifecycle policy. Doctor is likely the right surface, but the maintainable fix must define owned-unit detection, safe unit types, explicit consent, and preview/lint behavior before starting non-gateway units.
