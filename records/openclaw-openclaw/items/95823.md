## Summary

Keep open: current main and v2026.6.10 still scope doctor service handling to gateway-like `.service` units and intentionally ignore inactive non-legacy Linux units, so the enabled-but-inactive `.path` and `.timer` user-unit gap remains. The request is useful, but starting non-gateway OpenClaw-owned units needs maintainer lifecycle and consent policy first.

## What This Changes

Request to extend `openclaw doctor` so it detects and optionally restarts OpenClaw-owned enabled but inactive systemd user units beyond the gateway.

## Best Possible Solution

Add a scoped Linux doctor check that reports owned enabled inactive non-template user units in lint/JSON and starts only approved safe unit types under explicit repair consent or preview.

## Reproduction Assessment

Yes, source-level reproduction is high confidence. Current main scans gateway-like `.service` files and suppresses inactive non-legacy services, while no inspected path enumerates enabled `.path` or `.timer` user units through `FragmentPath`.

## Solution Assessment

Unclear until maintainers choose the lifecycle policy. Doctor is likely the right surface, but the best fix must define owned-unit detection, safe unit types, explicit repair consent, and preview/lint behavior before starting non-gateway units.
