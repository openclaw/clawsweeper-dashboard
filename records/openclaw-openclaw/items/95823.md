## What This Changes

The issue asks `openclaw doctor` and post-upgrade checks to detect enabled but inactive OpenClaw-owned systemd user units, report them in lint JSON, and start safe unit types under explicit repair.

## Best Possible Solution

Add a scoped Linux doctor health check that detects owned enabled non-template inactive user units, emits lint JSON, and repairs only maintainer-approved safe unit types under explicit preview and consent.

Yes for the source-level feature gap: current main scans gateway-like `.service` files and filters inactive non-legacy Linux units, while no inspected path enumerates enabled `.path` or `.timer` user units. I did not run a live systemd user-unit reproduction in this read-only review.

## Solution Assessment

Unclear until maintainers choose the lifecycle policy. Doctor is a plausible home, but the best fix must define owned-unit detection, safe unit types, explicit consent, and preview/lint behavior before starting non-gateway units.
