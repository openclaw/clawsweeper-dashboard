## Summary

Keep open: current main and v2026.6.9 still audit gateway-like `.service` units and gateway service config only; they do not enumerate enabled OpenClaw-owned `.path`/`.timer` or per-agent user units and restore enabled-but-inactive units. The requested check is useful but needs maintainer lifecycle policy before implementation because repair would start non-gateway user units.

## What This Changes

The issue requests extending `openclaw doctor` to detect and optionally start enabled but inactive OpenClaw-owned systemd user units beyond the gateway service.

## Best Possible Solution

Add a scoped Linux doctor health check that reports owned enabled non-template user units that are inactive, emits lint JSON, and starts only approved safe unit types under explicit repair consent.

## Reproduction Assessment

Yes for the source-level feature gap. Current main scans gateway-like `.service` files and filters inactive non-legacy services, while no inspected doctor path enumerates enabled `.path` or `.timer` user units through `systemctl --user list-unit-files` and `FragmentPath`.

## Solution Assessment

Unclear until maintainers choose the lifecycle policy. The proposal is directionally strong, but the best fix should be scoped by owned-unit detection, safe unit type, explicit repair consent, and the adjacent doctor dry-run/operator-intent work.
