## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Prevent silent SOUL writes on channel turns:** `src/agents/agent-command.ts:909`
  - body: This hook runs for every non-raw turn and the only forced notice after an append is `runtime.log(...)`. For Discord, Telegram, Slack, HTTP, or other ingress, that log is process output rather than a message in the user's channel, so enabling `autoUpdate` can persist a new SOUL.md rule without the user-visible notice the security model depends on. Gate this to local/TUI surfaces until a portable channel notice is delivered, or send the notice through the same channel before the write is considered complete.
  - confidence: 0.88
- **[P2] Keep SOUL read errors from aborting the main turn:** `src/agents/soul-reflection-runner.ts:64`
  - body: `readLastSoulRule()` can throw on ordinary filesystem errors, but the before/after reads sit outside the `try` that catches reflection failures. With `autoUpdate` enabled, an unreadable or racing SOUL.md can fail the user's main turn even though the runner comment says reflection errors are swallowed. Wrap both reads in the same containment path or make `readLastSoulRule` return null on read errors.
  - confidence: 0.82
- **[P2] Type the mocked ingress call before dereferencing it:** `src/agents/soul-reflection-runner.test.ts:96`
  - body: The current head fails `check-test-types` because `ingressSpy.mock.calls[0][0]` is inferred as `unknown`, and the test dereferences `call.message` and `call.inputProvenance`. Give the call a concrete ingress option type or narrow it before those expectations so the test suite typechecks.
  - confidence: 0.99
