## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Run reflection only after delivery policy allows the turn:** `src/agents/agent-command.ts:899-900`
  - body: This hook now fires before the existing `opts.deliver` send-policy check, so a channel or ingress turn that should throw `send blocked by session policy` can still run the reflection model call and append to SOUL.md first. Move reflection behind the same early policy/validation gates, or make it share the same deny decision before any sub-turn or write is possible.
  - confidence: 0.9
- **[P1] Deliver SOUL notices through the originating channel:** `src/agents/agent-command.ts:909`
  - body: The forced notice after an append is only `runtime.log(...)`, which writes to process output rather than Discord, Telegram, Slack, or HTTP callers. With `autoUpdate` enabled, channel users can trigger a persistent SOUL.md rule without seeing the notice the feature relies on; gate this to local/TUI surfaces or send a portable channel-visible notice before allowing channel-triggered writes.
  - confidence: 0.88
- **[P2] Keep SOUL read errors from aborting the main turn:** `src/agents/soul-reflection-runner.ts:64`
  - body: Both `readLastSoulRule()` calls sit outside the try/catch that is supposed to swallow reflection failures, and `readLastSoulRule` itself can throw on ordinary read errors. With `autoUpdate` enabled, an unreadable or racing SOUL.md can still fail the user's main turn before the reflection error path runs.
  - confidence: 0.82
