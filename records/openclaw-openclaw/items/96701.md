## What This Changes

The PR adds an internal SSE byte guard, applies it to Anthropic Messages streaming 200 success-body reads with a 16 MiB cap, and adds a regression test for oversized delimiter-free streams.

## Best Possible Solution

Land the narrow Anthropic streaming guard if maintainers accept the 16 MiB fail-closed boundary; otherwise tune the cap before merge.

## Reproduction Assessment

## Solution Assessment

Yes: guarding the reader at the Anthropic Messages parser boundary preserves streaming semantics and mirrors the existing bounded JSON success-body cap. Broader helper promotion and the other SSE parsers are better handled in separate, focused PRs.
