## Summary

Keep open: the PR targets a real status/context-window bug, but the current branch introduces read-only discovery side effects, mis-resolves raw slash-containing model IDs, and has a compile/lint failure.

## What This Changes

The branch changes context-window cache lookup, status message model/provider splitting, and CLI status summary resolution so status output can use the shared context resolver for slash-containing model IDs.

## Best Possible Solution

Keep the shared CLI resolver direction, but preserve read-only lookup semantics, preserve raw slash-model IDs unless provider ownership is proven, remove the stale import, and add real terminal or runtime proof for the corrected status output.

## Reproduction Assessment

Yes for the patch regressions: existing source tests and CI logs show the read-only cache load and raw slash-model status behavior failing under this branch. The original CLI 200K bug is also source-reproducible from the duplicated resolver described in #92760.

## Solution Assessment

No, not in its current form. Delegating CLI status to the shared resolver is the right direction, but the branch must not add background discovery to read-only calls or infer providers from raw slash model IDs without stronger ownership evidence.
