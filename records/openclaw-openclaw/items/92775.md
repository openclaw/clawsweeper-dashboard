Full review comments:

- **[P1] Keep read-only context lookups side-effect free:** `src/agents/context.ts:398`
  - body: This starts `ensureContextWindowCacheLoaded()` even when callers pass `cfg` and `allowAsyncLoad: false`. Status/session diagnostics rely on that mode to avoid runtime config/model discovery and possible models.json writes, and CI now fails the existing read-only test because config is loaded where zero calls are expected.
  - confidence: 0.93
- **[P1] Preserve raw providerless slash-model lookups:** `src/status/status-message.ts:628-629`
  - body: For a providerless runtime model like `google/gemini-2.5-pro`, this now treats `google` as the provider and scans Google config instead of preserving the raw model ID. Current status code keeps those raw so OpenRouter-style proxy IDs do not pick the first segment's provider context window.
  - confidence: 0.9
- **[P2] Do not make first-status correctness depend on a race:** `src/gateway/server-startup-early.ts:126`
  - body: The startup warmup is fire-and-forget, while status rendering remains synchronous/read-only. A first `/status` can still arrive before discovery fills the cache and fall back to the default window, so this does not deterministically fix the behavior named in the title.
  - confidence: 0.82
- **[P2] Remove the unused provider normalizer import:** `src/commands/status.summary.runtime.ts:4`
  - body: After deleting the local status resolver, `normalizeProviderId` is no longer used in this module. The current PR's prod type and lint checks both fail on this import, so the branch cannot pass required validation until it is removed.
  - confidence: 0.99
