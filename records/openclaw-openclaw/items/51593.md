## Best Possible Solution

Enable a Moonshot-safe duplicate-ID rewrite for replayed history and live outbound request copies while preserving native Kimi `functions.<name>:<index>` IDs, then validate with focused unit coverage and a real Moonshot/Kimi tool-turn smoke.

Yes. The narrow maintainable fix is to adapt or enable the existing occurrence-aware sanitizer for Moonshot while preserving native Kimi IDs, rather than starting with a WhatsApp session ownership rewrite.
