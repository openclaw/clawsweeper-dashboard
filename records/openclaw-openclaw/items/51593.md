Keep open. Current main still skips the duplicate raw tool-call-ID rewrite for bundled Moonshot because its replay policy omits sanitizer fields; the merged Kimi native-ID fixes protect adjacent paths but do not cover this WhatsApp/Moonshot `exec:26` duplicate symptom.

## What This Changes

The issue reports Moonshot/Kimi requests failing in WhatsApp group chats with duplicated raw OpenClaw tool-call IDs such as `exec:26`.

## Best Possible Solution

Add a Moonshot-safe duplicate-ID rewrite path for replayed history and live outbound requests that disambiguates repeated raw OpenClaw IDs while preserving native Kimi `functions.<name>:<index>` IDs.

## Solution Assessment
