## Summary

Keep open. Current main still opts bundled Moonshot out of tool-call ID sanitization, so repeated raw OpenClaw IDs can bypass the replay and live outbound sanitizer paths even though the generic sanitizer can already disambiguate duplicates.

## What This Changes

The issue asks OpenClaw to prevent duplicate Moonshot/Kimi tool-call IDs in WhatsApp group-chat requests so Kimi no longer returns HTTP 400.

## Best Possible Solution

Add a Moonshot-safe duplicate-ID rewrite path for replayed history and live outbound requests while preserving native Kimi IDs, with focused regression coverage.

## Reproduction Assessment

Yes for a source-level reproduction: bundled Moonshot resolves without sanitizer flags, and both replay and live outbound duplicate-ID rewrites are gated behind those flags. A live WhatsApp/Kimi run was not established in this read-only review.

## Solution Assessment

Yes, the narrow maintainable fix is to adapt or activate the existing occurrence-aware sanitizer for Moonshot without mangling native Kimi IDs. A broader WhatsApp session-scoping rewrite should wait for evidence that outbound replay/stream sanitization cannot repair the duplicate IDs.
