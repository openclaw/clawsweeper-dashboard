## Summary

Keep this open. Current main still short-circuits the original short explicit tagged-TTS reproduction before synthesis, and open PR #73911 explicitly targets this issue with a narrow fix, so the issue should remain open until that PR or an equivalent change lands.

## What This Changes

Make tagged TTS directives from claude-cli/Telegram replies synthesize and deliver voice notes instead of leaving empty voice-only payloads that are skipped.

## Best Possible Solution

Land #73911 or an equivalent narrow change that lets explicit non-empty directives.ttsText bypass the short-text suppression while preserving the untagged short auto-TTS guard.

## Reproduction Assessment

Yes. A final payload like [[audio_as_voice]] [[tts:text]]hello[[/tts:text]] follows a source-visible path where the hidden text is captured, visible text is stripped, the under-10-character guard returns before synthesis, and Telegram skips the empty voice-only reply.

## Solution Assessment

Yes for the original bug. The narrowest maintainable fix is to exempt explicit non-empty tagged TTS text from the short-text guard, not to change Telegram's empty-payload skip behavior or broaden provider credential handling in the same patch.
