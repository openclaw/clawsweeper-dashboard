## Summary

Keep open: current main has partial fallback visibility, but it is still verbose-gated and does not provide the requested default-on, per-session/per-topic notification policy or `models.fallbackNotify` config surface.

## What This Changes

Request default per-session/per-topic chat notifications when model fallback activates and when the primary model recovers, with a config switch to disable them.

## Best Possible Solution

Define a product-approved fallback notification contract that reuses the existing session fallback state, emits activation and recovery notices to the affected conversation target, and documents the config/schema default alongside the recovery design.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high confidence that current main only has verbose-gated chat notices and lifecycle events, not default-on fallback/recovery notifications.

## Solution Assessment

Unclear until maintainer product design is done. The existing fallback transition state is a good implementation base, but the requested default chat behavior and config contract need an explicit policy decision before a narrow fix is safe.
