## Summary

Keep open. Current main has fallback transition state, lifecycle events, verbose-mode reply notices, /status visibility, and Web UI indicators, but it still does not provide default-on activation and recovery notifications in the affected chat/session/topic when verbose mode is off.

## What This Changes

The issue requests default-on model fallback and recovery notifications delivered to the affected session/topic, with a config switch to disable them.

## Best Possible Solution

Define a product-owned fallback notification policy that sends one-time activation and recovery notices to the affected conversation by default, redacts sensitive details, documents throttling/recovery semantics, and exposes a stable disable/tuning config.

## Reproduction Assessment

Yes. Source and existing e2e coverage provide a high-confidence path: simulate fallback and recovery with resolvedVerboseLevel set to off, then observe lifecycle events while the reply payload omits both activation and cleared notice text.

## Solution Assessment

No. The existing verbose-only notices and lifecycle events are useful groundwork, but they do not solve the requested default-on affected-conversation behavior; maintainer-owned policy should come before a code change.
