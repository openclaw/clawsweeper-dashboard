## Summary

Keep open: the PR is a plausible narrow fix for the WebChat agent-run TTS final broadcast gap, but the latest patch can still let the live final audio differ from the persisted `chat.history` audio after a media block precedes the final TTS payload, and the proof remains a mocked Vitest/gateway harness rather than real WebChat/runtime proof.

The PR changes WebChat Gateway chat delivery so agent-run TTS/media final payloads can be broadcast as final `chat` events and adds focused handler plus running-gateway regression tests.

## Best Possible Solution
