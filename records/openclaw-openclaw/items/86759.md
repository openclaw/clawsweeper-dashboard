## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Use the final media payload after rewrite skips:** `src/gateway/server-methods/chat.ts:3620-3626`
  - body: When an earlier media block has already been appended, any later transcript entry after that block makes `rewriteTranscriptEntriesInSessionFile` return `changed: false` because `allowedRewriteSuffixEntryIds` only allows the media entry. This branch then sets `appendedMessage` to the existing block message, and the final broadcast prefers `appendedMessage.content`, so the live final can still send the old block media instead of the final TTS/media payload. Build the live final message from the current `persistedContentForAppend` even when the history rewrite is skipped.
  - confidence: 0.89
- **[P2] Pass the agent id to media final broadcasts:** `src/gateway/server-methods/chat.ts:4291-4296`
  - body: `broadcastChatFinal` only includes `agentId` and routes node-session delivery to `agent:<id>:global` when the caller passes `agentId`. This new media-final path omits it, so a non-default global-agent TTS/media final is broadcast without its scoped agent identity and node subscribers are routed as the default global session. Pass `agentId` here like the other final/error branches do.
  - confidence: 0.86
