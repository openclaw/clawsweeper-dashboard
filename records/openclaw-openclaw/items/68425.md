## Summary

Keep open. Current main still masks agent-visible session/tool text without the requested stable marker or redacted-field metadata, and the linked implementation PR is open with prior review blockers.

## What This Changes

Request to make existing redaction explicit to agents by adding a stable machine-readable marker or metadata when tool output has been redacted.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/68483 or an equivalent narrow patch that emits an additive stable marker only when redaction affects agent-visible output.

## Reproduction Assessment

Yes, at source level. A sessions_history response containing an sk-style token is masked and sets contentRedacted, while current main returns no stable marker, warning, or redacted-field metadata.

## Solution Assessment

Yes. An additive marker or notice at the agent-visible output boundary is the narrow maintainable fix because it preserves existing masking while making redaction detectable; current main is not sufficient.
