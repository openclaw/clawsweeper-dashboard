## Summary

Keep open. Current main still redacts agent-visible session/tool text without the requested stable machine-readable marker or redacted-field metadata, and open PR #68483 explicitly targets this issue with closing syntax, so the issue should remain open until that PR or an equivalent patch lands.

## What This Changes

Request a stable machine-readable signal when OpenClaw redacts tool output so agents can avoid writing placeholders back into config files.

## Best Possible Solution

Review and land PR #68483 or an equivalent narrow patch that adds a stable additive marker/notice only when redaction actually fires, with focused coverage for the shared redaction helper and `sessions_history` behavior.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to have `sessions_history` return a message containing an `sk-...` token: current main masks the token and sets `contentRedacted: true`, but returns no stable marker or warning field.

## Solution Assessment

Yes. An additive marker or notice at the agent-visible output boundary is the narrow maintainable fix because it preserves existing masking while making redaction detectable; current main is not sufficient.
