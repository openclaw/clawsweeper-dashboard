## Summary

Keep open. Current main still masks agent-visible session-history text without a stable redaction marker or warning, and the linked closing implementation is an open, conflicting PR that still has review/proof follow-up rather than a merged fix.

## What This Changes

The issue requests a stable machine-readable marker or metadata on redacted agent-visible tool output so agents can avoid writing redacted placeholders back to config.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/68483 or a narrow replacement that emits a stable marker only when final returned agent-visible output contains redaction, with docs and regression coverage.

## Reproduction Assessment

Yes, at source level. A `sessions_history` response containing an API-key-shaped token is masked and sets `contentRedacted`, while current main returns no stable marker, warning, or redacted-field metadata.

## Solution Assessment

Yes, directionally. An additive marker or notice at the agent-visible redaction boundary is the narrow maintainable fix, but the open PR or a corrected replacement still needs to land before this issue can close.
