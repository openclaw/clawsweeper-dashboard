## Summary

Keep open. Current main and the latest release have useful partial coverage via `sessions_history.contentRedacted`, but the shared redaction helper still returns only text and there is no common machine-readable redaction marker for ordinary live tool outputs.

## What This Changes

The issue requests a stable machine-readable signal on redacted agent-visible tool output so agents do not write placeholder secrets back into config.

## Best Possible Solution

Keep this open until maintainers choose a common redaction-state contract and implement it at the shared tool-output/redaction boundary, while retaining `sessions_history.contentRedacted` as partial coverage.

## Reproduction Assessment

Yes, at source level: the shared redaction helper returns only a string, and the only explicit current-main signal I found is the local `sessions_history.contentRedacted` field. I did not run a live tool/writeback reproduction because this is a read-only review.

## Solution Assessment

No. A `sessions_history`-only signal is not the best full solution for the reported terminal/file-read writeback path; the durable fix should be a maintainer-approved common redaction-result contract.
