## Summary

Keep open. Current main and v2026.6.8 still redact generic agent-visible tool output without a generic machine-readable redaction marker; `sessions_history` only exposes `contentRedacted`, the marker PR is closed unmerged, and the remaining protocol shape is security/product-sensitive.

## What This Changes

Request a stable machine-readable marker on redacted agent-visible tool output so agents can avoid writing redacted placeholders back into config or secret files.

## Best Possible Solution

Add a stable agent-visible redaction signal at the generic tool-result boundary, with focused coverage for exec/read/session-history paths and maintainer/security approval of the marker contract.
