## Summary

Keep open. Current main still redacts agent-visible session history text without a stable machine-readable marker or warning, and the linked closing implementation remains open, conflicting, and under review rather than merged.

## What This Changes

The issue asks OpenClaw to add a stable machine-readable marker for redacted agent-visible tool output so agents do not write masked placeholders back to config.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/68483 or a narrow replacement that emits a stable marker only when final returned agent-visible output contains redaction, with docs and regression coverage.

## Reproduction Assessment

Yes, at source level. A `sessions_history` response containing an API-key-shaped token is masked and sets `contentRedacted`, while current main returns no stable marker, warning, or redacted-field metadata.

## Solution Assessment

Yes, directionally. An additive marker or notice at the final agent-visible redaction boundary is the narrow maintainable fix, but the open PR or a corrected replacement still needs to land before this issue can close.
