## Summary

Keep open. Current main still redacts agent-visible session/tool output without the requested stable marker or redacted-field metadata, and the linked closing PR remains open and unmerged.

## What This Changes

Request to add a machine-readable redaction marker or metadata to agent-visible redacted tool output so agents can avoid writing masked placeholders back to config files.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/68483 or an equivalent narrow patch that emits a stable marker only when redaction affects returned agent-visible output, with docs and regression coverage.

## Reproduction Assessment

Yes, at source level. A `sessions_history` response containing an API-key-shaped token is masked and sets `contentRedacted`, while current main returns no stable marker, warning, or redacted-field metadata.

## Solution Assessment

Yes, directionally. An additive marker or notice at the agent-visible redaction boundary is the narrow maintainable fix, but current main does not implement it and the open PR still needs review follow-up.
