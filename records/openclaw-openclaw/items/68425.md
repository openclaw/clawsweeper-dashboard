## Summary

Keep open. Current main still redacts agent-visible session/tool text without the requested stable machine-readable marker or redacted-field metadata, and open PR #68483 explicitly targets this issue with closing syntax, so the issue should remain open until that PR or an equivalent patch lands.

## What This Changes

The issue asks OpenClaw to mark redacted agent-visible tool output with a stable machine-readable signal so agents do not write placeholders back to config.

## Best Possible Solution

Review and land PR #68483 or an equivalent narrow patch that adds a stable additive marker or notice only when agent-visible redaction actually fires, with coverage for the shared helper and `sessions_history` response.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a `sessions_history` response containing an `sk-...` token: current main masks the token and sets `contentRedacted: true`, but returns no stable marker or warning field.

## Solution Assessment

Yes. An additive machine-readable marker or notice at the agent-visible output boundary is the narrow maintainable fix because it preserves existing masking while making redaction detectable; current main is not sufficient.
