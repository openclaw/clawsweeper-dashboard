## Summary

Keep open. Current main still masks sensitive values in agent-visible session/tool surfaces, but the relevant contracts expose only redacted text plus flags such as `contentRedacted`; they do not add the stable machine-readable marker, notice field, or redacted-field metadata requested here. PR #68483 is open and explicitly targets this issue, so the issue should remain open until that implementation candidate is reviewed, updated, landed, or closed.

## What This Changes

Request to add a stable machine-readable redaction marker or notice to agent-visible redacted tool output so agents can detect masked placeholders before writing content back to config.

## Best Possible Solution

Review and land PR #68483 or an equivalent narrow patch that adds a stable additive marker/notice only when redaction actually fires, with focused coverage for the shared redaction helper and `sessions_history` behavior. Keep the backup-pollution recovery work in #68423 separate.

## Reproduction Assessment

Yes. A high-confidence reproduction path is to have `sessions_history` return a message containing a token-like string: current main masks the token and sets `contentRedacted: true`, but returns no stable marker or warning field.

## Solution Assessment

Yes. The requested additive marker or notice is the narrow maintainable fix for this issue because it preserves existing masking while making redaction detectable at the agent-visible output boundary; current main is not sufficient.
