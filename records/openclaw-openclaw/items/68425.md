## Summary

Keep open. Current main still redacts agent-visible session history text and exposes `contentRedacted`, but it does not provide a stable marker, notice field, or redacted-fields metadata such as `[OPENCLAW-REDACTED]`. The linked PR #68483 is open and explicitly targets this issue, so the issue should remain open until that candidate is reviewed, landed, or closed.

## What This Changes

Request stable machine-readable redaction metadata or a marker on agent-visible redacted tool output so agents can detect placeholders and avoid writing them back to configuration files.

## Best Possible Solution

Keep this issue open and review PR #68483, or an equivalent narrow patch, for landing. The desired end state is an additive, stable redaction marker or notice on agent-visible redacted tool output, with regression coverage for the shared helper and `sessions_history` behavior while preserving existing masking semantics.

## Reproduction Assessment

Yes. A high-confidence reproduction path is to have `sessions_history` return text containing a secret-looking token: current main redacts the token and sets `contentRedacted: true`, but source, tests, and docs show no stable marker or notice is returned.

## Solution Assessment

Yes, the requested additive marker or notice is the narrow maintainable direction for this issue. Current main is not sufficient, and the open PR should be reviewed rather than closing the issue ahead of merge.
