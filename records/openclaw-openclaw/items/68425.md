## Summary

Keep open. Current main and v2026.6.10 still redact generic agent-visible tool output without a generic machine-readable redaction marker; `sessions_history` has only a partial `contentRedacted` flag, and the open related PRs cover adjacent redaction boundaries rather than the stable marker contract requested here.

## What This Changes

The issue requests an explicit, machine-readable signal when redacted tool output is shown to agents so they do not write redacted placeholders back to config or secrets.

## Best Possible Solution

Add a maintainer-approved redaction-state contract at the generic tool-result boundary, then align `sessions_history` and relevant exec/read paths with focused tests and security review.

## Reproduction Assessment

Yes, at source level: current main redacts generic tool results and `sessions_history` content but does not emit a generic machine-readable marker. I did not rerun the live config writeback incident path in this read-only review.

## Solution Assessment

Unclear as an implementation shape: an explicit marker is the right safety direction, but maintainers/security should choose the stable structured or text contract before changing model-visible tool-result protocol.
