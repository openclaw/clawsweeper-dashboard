## Summary

Keep open. Current main still redacts agent-visible tool output without a generic machine-readable redaction marker; the only current partial signal is `sessions_history.contentRedacted`, and the related marker PR is closed unmerged.

## What This Changes

The issue requests a stable machine-readable redaction marker in agent-visible tool output so agents can avoid writing redacted placeholders back into config or secret files.

## Best Possible Solution

Add a generic, stable, agent-visible redaction signal at the model-visible tool-result boundary, with focused tests for exec/read/session-history paths and maintainer approval for the exact structured/text marker contract.

## Reproduction Assessment

Yes by source inspection at medium confidence: the generic tool-result sanitizer redacts secret strings but returns no machine-readable marker, and the sessions-history path only exposes `contentRedacted`. A live incident-style config writeback was not reproduced in this read-only review.

## Solution Assessment

Unclear as an implementation detail: the requested marker is the right end state, but maintainers should choose the stable structured/text contract before changing the model-facing tool-result protocol.
