## Summary

Keep open. Current main still redacts agent-visible session/tool text without the requested stable machine-readable marker or redacted-field metadata, and the linked implementation PR is still open.

## What This Changes

The issue requests a stable machine-readable marker or metadata whenever agent-visible tool output has been redacted.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/68483 or an equivalent narrow patch that emits an additive stable marker only when redaction actually affects agent-visible output.

## Reproduction Assessment

Yes, at source level. A sessions_history response containing an sk-style token is masked and sets contentRedacted, but current main returns no stable marker, warning, or redacted-field metadata.

## Solution Assessment

Yes. An additive marker or notice at the agent-visible output boundary is the narrow maintainable fix because it preserves existing masking while making redaction detectable; current main is not sufficient.
