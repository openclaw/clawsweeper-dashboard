## Summary

Keep open. Current main still redacts agent-visible session/tool text without the requested stable marker or redacted-field metadata, and the linked implementation PR remains open with prior review blockers.

## What This Changes

The issue requests a stable machine-readable marker or metadata whenever agent-visible tool output has been redacted so agents do not write redacted placeholders back into config files.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/68483 or an equivalent narrow patch that emits an additive stable marker only when redaction affects agent-visible output, with docs, regression coverage, and real behavior proof.

## Reproduction Assessment

Yes, at source level. A `sessions_history` response containing an API-key-shaped token is masked and sets `contentRedacted`, while current main returns no stable marker, warning, or redacted-field metadata.

## Solution Assessment

Yes, directionally. An additive marker or notice at the agent-visible redaction boundary is the narrow maintainable fix, but current main does not implement it and the open PR still needs normal review follow-up.
