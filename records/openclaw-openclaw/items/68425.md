## Summary

Keep open. Current main still redacts agent-visible session/tool text but exposes only masked text plus flags such as `contentRedacted`; it does not add the stable marker, notice field, or redacted-field metadata requested here, and open PR #68483 is the active implementation candidate.

## What This Changes

The issue requests a stable machine-readable marker or notice on agent-visible redacted tool output so agents can detect masked placeholders before writing content back to config.

## Best Possible Solution

Review and land PR #68483 or an equivalent narrow patch that adds a stable additive marker/notice only when redaction actually affects the agent-visible output, while keeping backup-pollution recovery in #68423 separate.

## Reproduction Assessment

Yes. A high-confidence reproduction path is to have `sessions_history` return a message containing a token-like string: current main masks the token and sets `contentRedacted: true`, but returns no stable marker or warning field.

## Solution Assessment

Yes. The additive marker/notice direction is the narrow maintainable fix because it preserves existing masking while making redaction detectable at the agent-visible output boundary; current main is not sufficient.
