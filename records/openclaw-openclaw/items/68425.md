## Summary

Keep open. Current main still redacts agent-visible session/tool text without the requested stable machine-readable marker or redacted-field metadata, and the linked implementation PR remains open and unmerged.

## What This Changes

Request an explicit machine-readable marker or structured metadata whenever redaction changes agent-visible tool output so agents do not write redacted placeholders back to config files.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/68483 or an equivalent narrow patch that emits a stable additive marker only when agent-visible output was actually redacted, with regression coverage for both the helper and `sessions_history` response.

## Reproduction Assessment

Yes, at source level. A `sessions_history` response containing secret-like text sets `contentRedacted` after masking, while current main returns no stable marker, notice, warning, or redacted-field metadata.

## Solution Assessment

Yes, directionally. An additive marker or notice at the agent-visible redaction boundary is the narrow maintainable fix, but the current linked implementation still needs PR review or replacement before this issue can close.
