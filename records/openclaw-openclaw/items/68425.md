## Summary

Keep open. Current main and v2026.6.10 still have generic redaction paths that return sanitized agent-visible tool payloads without a stable redaction-state marker; `sessions_history` has only a partial `contentRedacted` flag, and related PRs are adjacent or closed unmerged rather than a landed generic contract.

## What This Changes

The issue requests a stable machine-readable marker on redacted tool output so agents can avoid writing redacted placeholders back into config or secrets.

## Best Possible Solution

Add a maintainer-approved redaction-state contract at the generic tool-result boundary, then align partial surfaces like `sessions_history` and exec/read outputs with focused tests and security review.

## Reproduction Assessment

Yes, at source level: current main and v2026.6.10 redact generic tool results and `sessions_history` content, but generic redaction returns no stable marker for the agent-visible payload. I did not rerun the live config writeback incident path in this read-only review.

## Solution Assessment

Unclear as an implementation shape: an explicit marker is the right safety direction, but maintainers/security should choose the stable structured or text contract before changing model-visible tool-result protocol.
