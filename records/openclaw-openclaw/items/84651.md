## Summary

Keep open. Current main still bounds WebChat history and the existing side-panel actions reuse the already-loaded assistant/tool text, while https://github.com/openclaw/openclaw/pull/84670 is an open implementation candidate that specifically links to this issue and has not merged.

## What This Changes

Request a WebChat/Control UI side-panel reader that can fetch and show full assistant/tool content when `chat.history` returns truncated previews.

## Best Possible Solution

Review and either land or deliberately reject https://github.com/openclaw/openclaw/pull/84670, keeping the issue open until a bounded full-message reader or accepted equivalent is on main.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: load a long history row through bounded `chat.history`, then open the existing assistant or tool side panel and it reuses the truncated text already in client state.

## Solution Assessment

Yes as a direction, but not complete on main. An on-demand bounded reader is more maintainable than raising default history payloads, and the open linked PR is the current concrete solution to review.
