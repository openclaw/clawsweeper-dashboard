## Summary

Keep open. ## What This Changes

Request adds a WebChat/Control UI full-message reader that can fetch truncated assistant and tool content on demand instead of repeating the capped preview.

## Best Possible Solution

Land or revise the linked bounded full-message reader after maintainer API review, keeping default history payloads bounded while sharing one assistant/tool sidebar retrieval contract.

## Reproduction Assessment

Yes. Source inspection shows a long assistant/tool row can be capped by `chat.history`, marked with `...(truncated)...`, and then reopened in the side reader as the same already-loaded string.

## Solution Assessment

Yes as a direction: an on-demand full-message sidebar is the maintainable shape because it preserves bounded history payloads instead of raising caps or expanding huge messages inline. The exact Gateway method and unavailable-state contract still need maintainer API/UX approval.
