## Summary

Keep open: current main and v2026.6.10 still compose exec/bash failure warnings by wrapping a command-like framework summary in markdown and appending `failed`, so the reported cron/chat ambiguity remains a focused presentation bug.

## What This Changes

The issue asks OpenClaw to make exec tool failure text separate the framework/action label from the literal command, ideally including exit code when available.

## Best Possible Solution

Update the shared tool-failure presentation path so exec/bash warnings name the tool failure separately while rendering only the literal command as command text, with focused tests for payload warnings, cron diagnostics, and any affected chat display.

Yes, at source level: current main still builds warning text as `⚠️ ${toolSummary} failed`, and exec/bash summaries can contain markdown-wrapped `run python3 ...` text. I did not execute a live cron/chat run in this read-only review.
