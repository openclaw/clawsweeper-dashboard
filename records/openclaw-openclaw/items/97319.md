## Summary

Keep open: current main and v2026.6.10 still compose exec/bash failure warnings as a compact command-like summary plus `failed`, and the exec display helper can still produce `run python3 <script>` as that summary, so the reported ambiguity remains a narrow display bug.

## What This Changes

Clarify exec tool-failure display so framework-generated action labels are visually separate from the literal command shown in chat and cron run history.

## Best Possible Solution

Keep this as a narrow presentation bug: update shared tool-failure wording so the tool label/status is outside the literal command, with regression coverage for embedded-runner payload warnings and cron diagnostics.

## Reproduction Assessment

Yes at source level: current main still builds warning text as `⚠️ ${toolSummary} failed`, exec display can produce `run python3 <script>`, and cron persists that error text as `lastError`. No live cron or chat run was executed in this read-only review.

## Solution Assessment

Yes: a shared presentation-layer wording change is the narrowest maintainable fix. The linked exec nonzero-exit semantics report should own exit-code behavior, and this issue should not add config or change runtime semantics.
