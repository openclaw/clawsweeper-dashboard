## Summary

Keep open: current main still has only provider-global realtime voice config for voice-call bridge sessions, and the requested per-agent voice selection would add a new public plugin config/API surface.

## What This Changes

Request per-agent OpenAI Realtime voice selection in the voice-call plugin, such as a `realtime.agentVoices` map that overrides the global provider voice by `agentId`.

## Best Possible Solution

Expose a documented, provider-aware per-agent realtime provider override in the voice-call plugin while preserving the existing global provider default.

## Reproduction Assessment

Not applicable: this is a feature/config request, not broken documented behavior. Source inspection confirms current main creates realtime bridge sessions with the same stored provider config for every call.

## Solution Assessment

Unclear: the proposed `agentVoices` map solves the immediate OpenAI voice case, but maintainers should choose whether the public API is voice-only, provider-generic, or composed with per-call agent routing semantics.
