## Summary

Keep open: current main still has only provider-global realtime voice config for voice-call bridge sessions, and per-agent OpenAI Realtime voice selection would add a new public plugin config/API surface. The related per-call agent routing work in https://github.com/openclaw/openclaw/pull/77763 is useful context but does not implement this per-agent realtime voice override.

## What This Changes

The issue requests per-agent OpenAI Realtime voice selection in the voice-call plugin instead of one global `realtime.providers.openai.voice` value.

## Best Possible Solution

Expose a documented, provider-aware per-agent realtime provider override in the voice-call plugin while preserving the existing global provider default and composing with per-call agent routing.

## Reproduction Assessment

Not applicable: this is a feature/config request, not broken documented behavior. Source inspection confirms current main creates realtime bridge sessions with the same stored provider config for every call.

## Solution Assessment

Unclear: the proposed `agentVoices` map solves the immediate OpenAI voice case, but maintainers should choose whether the public API is voice-only, provider-generic, or tied to a broader per-agent realtime override model.
