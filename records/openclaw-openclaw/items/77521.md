## Summary

Keep open: this PR targets a source-visible prompt-cache regression, and the diff follows the documented hook/runtime-message contracts. It should go through normal maintainer review and exact-head merge gating rather than cleanup closure.

## What This Changes

The branch stops appending non-runtime-only per-turn runtime context to the agent system prompt, preserves only current-turn runtime custom messages at the LLM boundary, adds regression coverage, and updates the changelog.

## Best Possible Solution

Land this PR, or an equivalent narrow fix, after maintainer review confirms current-turn runtime context remains model-visible while historical runtime context stays out of provider replay.

## Reproduction Assessment

Yes. Source inspection of current main shows the transcriptPrompt/effectivePrompt split returns runtimeContext and runEmbeddedAttempt appends that context to systemPromptText before activeSession.prompt; I did not run live cache telemetry.

## Solution Assessment

Yes. The PR is a narrow fix: it removes the cache-busting system-prompt append for ordinary turns and relies on the existing Pi nextTurn custom-message path to keep current-turn context visible to the model.
