- `P3`: This is a useful security and automation enhancement, while supported manual migration and scripted SecretRef assignment workflows remain functional.
- `impact:security`: The issue concerns reducing plaintext credential persistence and safely moving credentials behind SecretRefs.

## Summary

Current main supports non-interactive single and batch SecretRef assignment and validated application of a prebuilt migration plan, but plan generation remains interactive, so the central automation gap is still real. The request fits OpenClaw’s security and CLI-ergonomics direction, but automatic naming, collision handling, destination writes, and plaintext-output rules require a maintainer product/security decision before implementation.

## What This Changes

Add a non-interactive CLI workflow that discovers plaintext credentials, generates SecretRefs using a selected strategy, validates them, and applies the migration without a TTY.

## Best Possible Solution

Add a policy-driven non-interactive planner that converts supported audit findings into the existing versioned apply plan, emits a redacted dry-run, and requires explicit consent before the existing apply engine performs validated writes.

## Reproduction Assessment

Not applicable. Source and docs establish a missing requested capability—automatic non-interactive plan generation—but this is not a failure of an existing documented command.

## Solution Assessment

Partly. Automatic migration is a sound goal, but the maintainable solution should extend plan generation and reuse the existing apply contract rather than introduce an independent migration and write path.
