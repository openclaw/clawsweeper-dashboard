## Label Justifications

- `P3`: This is a useful but non-urgent automation capability rather than a regression in an existing non-interactive contract.
- `impact:security`: The requested workflow migrates plaintext credentials and selects SecretRef storage destinations.

## Summary

Current main and v2026.6.6 still require interactive plan creation, so the requested automation remains missing. The feature fits OpenClaw’s security and CLI goals, but maintainers must define safe destination, naming, scope, and environment-write policy before implementation.

## What This Changes

Requests a non-interactive command that discovers plaintext credentials, generates a SecretRef migration plan, validates it, and applies it for unattended automation.

## Best Possible Solution

Add a deterministic non-interactive plan generator over the existing target registry and apply engine, with explicit maintainer-approved destination and naming rules, machine-readable dry-run output, and no implicit operating-system environment mutation.

## Reproduction Assessment

Not applicable as a bug reproduction; source, documentation, current main, and v2026.6.6 all confirm the product limitation that plan creation requires a TTY and unattended apply requires a prebuilt plan.

## Solution Assessment

No, not exactly as proposed: automatic plan generation is the narrowest maintainable extension, but implicit `.env` or `setx` writes and automatically derived identifiers require an explicit security and UX policy.
