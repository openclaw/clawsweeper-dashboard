Keep this open: current main and v2026.6.8 still lack a non-interactive audit-to-plan `secrets migrate --auto` workflow, while the related merged work only covers explicit `config set` SecretRef/provider assignment. ## What This Changes

Request a new non-interactive `openclaw secrets migrate --strategy env --auto --yes` workflow that scans plaintext credentials, generates SecretRefs, validates them, and applies the migration for automation scripts.

## Best Possible Solution

Design a narrow first-party CLI migration command that reuses audit discovery and `secrets apply` preflight/write semantics, with explicit dry-run/confirmation, strategy rules, safe env-name derivation, and docs/tests for upgrade automation.
