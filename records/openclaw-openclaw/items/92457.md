## Summary

Keep this open: current main and v2026.6.8 still provide audit, interactive configure, and saved-plan apply, but not the requested non-interactive audit-to-plan `secrets migrate --auto` workflow. The related merged work is useful context, but it only covers explicit `config set` SecretRef/provider assignment, so the remaining request needs maintainer product and security direction.

## What This Changes

Request adds a non-interactive `openclaw secrets migrate --strategy env --auto` workflow for automation scripts to scan plaintext credentials, generate SecretRefs, validate them, and apply the migration without a TTY.

## Best Possible Solution

Define a narrow first-party CLI migration contract that reuses audit discovery and `secrets apply` preflight/write semantics, with explicit dry-run/confirmation, safe env-name derivation, exec-provider opt-in, and docs/tests for upgrade automation.
