## Summary

Keep this open: current main and the latest release still do not provide the requested non-interactive batch SecretRef migration command, and the linked closed work only added supported non-interactive config assignment paths rather than auto-generating and applying a migration from audit findings.

## What This Changes

Request to add a non-interactive `openclaw secrets migrate --strategy env --auto` flow that scans plaintext credential findings, generates SecretRefs, validates them, and applies the migration for automation scripts.

## Best Possible Solution

Design a narrow first-party secrets migration workflow that reuses audit target discovery and apply preflight/atomic writes, keeps dry-run and explicit confirmation as first-class behavior, and documents the exact supported target and env-mapping contract.

## Solution Assessment
