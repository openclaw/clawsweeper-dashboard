## Best Possible Solution

If maintainers want this, add a narrow first-party migration workflow that turns audit findings into the existing `SecretsApplyPlan` and `runSecretsApply` path with explicit dry-run, strategy/category semantics, resolution checks, and docs for one-way scrubbing.
