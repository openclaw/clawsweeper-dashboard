## Summary

Keep open. ## What This Changes

The PR adds a Control UI Usage agent selector, sends selected `agentId` in `sessions.usage`, adjusts usage rendering and i18n, and adds controller, view, and mocked E2E coverage.

## Best Possible Solution

Keep the server-scoped selector approach, but separate or rewire the legacy Agent query filter so any agent choice sends `agentId`, then add redacted live multi-agent gateway proof or a maintainer override before merge.

## Reproduction Assessment

Yes from source inspection. Current main defaults omitted `sessions.usage` `agentId` to the default agent while the Control UI sends no `agentId`; this review did not run a live multi-agent gateway.

## Solution Assessment

No, not yet. Sending `agentId` from the Control UI is the right fix boundary, but the branch still leaves the old query-only Agent filter advertising configured agents without changing server scope.
