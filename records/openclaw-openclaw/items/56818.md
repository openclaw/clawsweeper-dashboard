## Summary

Keep open. Current main still lets a top-level Feishu card-action callback `open_message_id` win before the real `context.open_message_id`, so the reported streaming reply failure remains source-reproducible.

## What This Changes

The issue requests that Feishu card-action callbacks stop using temporary `card-action-c-*` callback IDs as streaming reply targets.

## Best Possible Solution

Fix Feishu card-action normalization so a real `context.open_message_id` is preferred over temporary callback IDs, while preserving SDK-flat real top-level IDs and no-real-ID plain-send fallback.

## Reproduction Assessment

Yes. Source inspection gives a focused path: dispatch a Feishu `card.action.trigger` payload with top-level `open_message_id: "card-action-c-..."`, `context.open_message_id: "om_original"`, and a reply-producing action, then assert the dispatcher receives `replyToMessageId: "om_original"`.

## Solution Assessment

Yes. The narrow maintainable fix is at the Feishu card-action parser/synthetic-event boundary, not in core reply dispatch, because the bad ID is created by Feishu-specific callback normalization.
