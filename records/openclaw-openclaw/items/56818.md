## What This Changes

The issue reports Feishu card-action callbacks using a temporary `open_message_id` as the reply target, causing streaming replies to fail with `Invalid open_message_id`.

## Best Possible Solution

Fix Feishu card-action target selection in the Feishu plugin: preserve real `context.open_message_id` targets, suppress only selected temporary `card-action-c-*` targets, fall back to a new message when no valid reply target exists, and cover mixed/no-target cases with regression tests.

## Reproduction Assessment

Yes at source level, but not through a live tenant in this read-only pass. A payload with temporary `open_message_id: card-action-c-*` is parsed into reply metadata and then passed to the Feishu streaming reply path.

## Solution Assessment

Yes, the best fix is a narrow Feishu plugin target-selection repair rather than disabling streaming globally. The repair should preserve valid context reply ids and suppress only temporary selected targets.
