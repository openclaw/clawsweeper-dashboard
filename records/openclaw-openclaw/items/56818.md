## Summary

Keep open. Current main still lets a top-level Feishu card-action callback id win over the real `context.open_message_id`, so the reported `card-action-c-*` streaming reply failure remains source-reproducible.

## What This Changes

The issue requests that Feishu card action callbacks stop using temporary `card-action-c-*` ids as reply targets for streaming replies.

## Best Possible Solution

Fix the Feishu card-action normalization so a real `context.open_message_id` is preferred over temporary `card-action-c-*` callback ids, while preserving SDK-flat real top-level ids and no-real-id plain-send fallback.

## Reproduction Assessment

Yes. Source inspection gives a focused path: dispatch `card.action.trigger` with top-level `open_message_id: "card-action-c-..."`, `context.open_message_id: "om_original"`, and a reply-producing action, then assert the dispatcher receives `replyToMessageId: "om_original"`.

## Solution Assessment

Yes. The narrowest maintainable fix is at the Feishu card-action parser/synthetic-event boundary, not in core reply dispatch, because the bug is Feishu-specific callback id normalization.
