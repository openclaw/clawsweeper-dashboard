## Summary

Keep open: current main still lets Feishu card-action callback IDs flow into the synthetic reply target path, and the open linked fix PR https://github.com/openclaw/openclaw/pull/90967 has not merged. This issue should stay open until that PR lands or maintainers choose a different Feishu-side fix.

## What This Changes

Feishu interactive card action callbacks can use a temporary `open_message_id` that OpenClaw treats as a reply target, causing streaming replies to fail instead of delivering a visible response.

## Best Possible Solution

Review and land, revise, or close https://github.com/openclaw/openclaw/pull/90967 so Feishu temporary card-action IDs are never used as streaming reply targets while valid original message IDs still work.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: a Feishu card-action callback ID can become `reply_target_message_id`, then the streaming path uses it for `im.message.reply`. I did not run a live Feishu tenant test in this read-only review.

## Solution Assessment

Yes, the narrow fix is to stop treating Feishu temporary card-action callback IDs as valid reply targets in the card-action synthetic-message path. The linked PR targets the right surface, but it should be reviewed with focused regression coverage or stronger live proof before merge.
