## Summary

Keep open. Current main and the latest release still select a top-level Feishu card-action `open_message_id` before `context.open_message_id`, so the reported temporary callback ID can still become the streaming reply target.

## What This Changes

The issue asks Feishu card-action replies to use the real original message ID instead of a temporary `card-action-c-*` callback ID.

## Best Possible Solution

Fix Feishu card-action normalization so a real `context.open_message_id` wins over temporary callback IDs, while preserving SDK-flat real top-level IDs and no-real-ID plain-send fallback.

## Reproduction Assessment

Yes. Source inspection gives a focused path: dispatch a Feishu `card.action.trigger` payload with top-level `open_message_id: "card-action-c-..."`, `context.open_message_id: "om_original"`, and a reply-producing action, then assert the dispatcher uses `om_original`.

## Solution Assessment

Yes. The narrow maintainable fix belongs at the Feishu card-action parser/synthetic-event boundary, because that is where the temporary callback ID is currently normalized into the reply target.

Current main parses `open_message_id` with top-level precedence, so a temporary `card-action-c-*` value can override the real `context.open_message_id` and then reach streaming replies as `replyToMessageId`.
