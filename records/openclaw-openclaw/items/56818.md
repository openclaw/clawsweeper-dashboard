## Summary

Keep open: current main still lets Feishu card-action callback IDs become reply targets for Feishu replies/streaming cards, and the only linked fix PR was closed unmerged after proof and correctness concerns.

## What This Changes

Feishu card-action callbacks with temporary `card-action-c-*` IDs should avoid using those IDs as streaming reply targets and instead preserve a real original message ID or send a new message.

## Best Possible Solution

Filter Feishu card-action reply targets in the Feishu plugin so temporary `card-action-c-*` IDs are suppressed, valid `context.open_message_id` targets are preserved, and focused regression coverage proves both cases.

## Reproduction Assessment

Yes at source level, but not through a live tenant in this read-only review. A Feishu card-action payload with `open_message_id: card-action-c-*` is parsed into the synthetic event, copied into `reply_target_message_id`, and then used by the streaming card reply path.

## Solution Assessment

Yes, the best fix is narrow and belongs in Feishu card-action target selection. Preserve real `context.open_message_id` values, suppress only temporary selected targets, and add regression coverage instead of disabling Feishu streaming globally.
