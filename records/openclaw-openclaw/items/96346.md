## Summary

Keep this open: current main still uses tail-only bounded process capture for cron command output, and the user-facing wait path reads the resulting run-log summary. The issue now has open fixing PRs, with https://github.com/openclaw/openclaw/pull/96393 looking like the strongest canonical candidate, so the issue should stay open until a fix merges.

## What This Changes

The issue reports that bounded async/cron command output can truncate early auth/setup instructions such as device-code prompts, making awaited command results unactionable.

## Best Possible Solution

Land one canonical bounded fix, likely through https://github.com/openclaw/openclaw/pull/96393 if maintainers accept it, that preserves action-critical lines for operator-facing cron results while redacting or omitting those values from external delivery.

## Reproduction Assessment

Yes, by source inspection: cron command jobs pass outputMaxBytes into tail-only process capture, then wait-mode reads the run-log entry built from that truncated output. I did not run a live cron job because this review is read-only.

## Solution Assessment

Yes, the best shape is a bounded preservation path at the command-output/cron-summary boundary paired with separate external-delivery redaction. Current main lacks that, and the open canonical candidate appears to cover both sides.
