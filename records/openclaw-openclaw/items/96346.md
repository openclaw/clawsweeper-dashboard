## Label Justifications

- `P1`: The reported bug can make awaited cron command auth/setup results unactionable for real users when truncation drops the device code or next-action line.
- `impact:message-loss`: The issue is specifically about user-facing command result lines being omitted during output truncation.
- `impact:auth-provider`: The lost lines include device-code auth prompts and setup instructions needed to complete provider authentication flows.

## Summary

Keep this open: current main and the latest release still use tail-only process output capture, so early auth/setup prompt lines can still be dropped before cron run summaries are stored. Several open PRs are candidate fixes, with https://github.com/openclaw/openclaw/pull/96393 currently the broadest candidate, but none has merged yet.

## What This Changes

The issue requests preserving action-critical auth/setup prompt lines in interactive async or cron command results when stdout/stderr truncation would otherwise keep only the tail.

## Best Possible Solution

Land one canonical cron-scoped fix that keeps process capture bounded, preserves action-critical lines for operator-facing run history, and redacts the same value class at non-interactive delivery boundaries.

## Reproduction Assessment

Yes, by source inspection: current main tail-truncates process stdout/stderr before cron summaries and run-log output are built. I did not run a live cron job because this review is read-only.

## Solution Assessment

Yes in shape: bounded preservation plus external redaction is the maintainable fix, not unbounded output retention. The remaining path is to review and land one canonical linked PR rather than queue another duplicate fix.
