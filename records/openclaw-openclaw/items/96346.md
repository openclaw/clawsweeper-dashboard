## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/96346. Current-main cron command jobs pass payload.outputMaxBytes to runCommandWithTimeout, which keeps only a bounded tail; runCronCommandJob then delivers that already-truncated stdout/stderr through cron run summaries and run-log entries. Add a narrow shared action-critical output classifier/presentation helper so interactive cron command summaries preserve device-login URLs, device/setup code lines, localhost/callback URLs, and explicit next-action instructions even when stdout/stderr exceeds outputMaxBytes. Keep capture bounded; do not add unbounded stdout/stderr retention, config flags, changelog edits, or broadcast redaction changes beyond using a shared classifier if it reduces duplication. Preserve or add a visible truncation/recovery note pointing to `openclaw cron runs --id <job> --run-id <run>` when a run id is available. Add regression coverage for Microsoft-style fake device code prompts, ordinary truncation without auth/setup content, timeout/SIGTERM/no-output summaries where feasible, and the related broadcast redaction invariant from https://github.com/openclaw/openclaw/pull/95809 if that helper is shared.
