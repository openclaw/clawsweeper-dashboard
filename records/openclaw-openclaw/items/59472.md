## Summary

Current main and the v2026.5.7 release already solve the PR's user-facing bug by removing provider-usage loading from `channels list` entirely, so `channels list --json` now emits channel-only JSON and cannot fail because `loadProviderUsageSummary()` throws.

## What This Changes

The PR adds a JSON-mode fallback around `loadProviderUsageSummary()` in `channels list` and a focused mocked regression test.

## Best Possible Solution

Keep the current channel-only `channels list` contract and leave provider usage/quota reporting in `openclaw status` and `openclaw models list`.

## Reproduction Assessment

No. On current main, `channels list` no longer imports or calls provider-usage loading, so a `loadProviderUsageSummary()` failure cannot prevent `channels list --json` from writing JSON.

## Solution Assessment

Yes. Removing provider-usage fetching from a channel-management command is the narrower maintainable solution than catching one usage-loader failure mode in the old JSON path.
