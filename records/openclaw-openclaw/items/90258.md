## Summary

Keep open. ## What This Changes

This PR replaces full-size terminal QR rendering with OpenClaw’s matrix renderer using truecolor white/black rows, updates QR renderer tests, and removes a comment from the plugin-sdk testing facade.

## Best Possible Solution

Merge only after CI and maintainer acceptance that `renderQrTerminal` should standardize on truecolor matrix output for full-size terminal QR codes; add an explicit fallback or option only if legacy terminal support remains required.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Source inspection shows current main delegates full-size QR output to qrcode’s ANSI-palette terminal renderer, and the PR screenshots show the dark before state; I did not run a live terminal-theme repro locally.

## Solution Assessment

Is this the best way to solve the issue? Yes, it is a clean shared fix for Feishu and other full-size terminal QR callers, with the caveat that maintainers must accept the public SDK output compatibility change.
