## Summary

No blocking code findings from the read-only review. Keep this PR open for normal maintainer merge review because it addresses the linked gateway startup regression, while the main merge decision is accepting the compatibility change from unbounded non-live provider discovery to a bounded 30 second skip-on-timeout default.

## What This Changes

The PR changes implicit provider catalog discovery to always use a default timeout when callers do not pass one, and adds focused tests for non-live, live, valid override, and invalid override timeout resolution.

## Best Possible Solution

Land the shared provider-catalog timeout if maintainers accept 30 seconds as the upgrade behavior for uncredentialed or unresponsive providers, with the existing 5 second gateway startup override preserved.

## Reproduction Assessment

Yes from source inspection, not from a live repro: current main passes `null` for non-live discovery and then awaits catalog discovery without a timer, matching the linked hang report. I did not run the Kubernetes or live gateway scenario.

## Solution Assessment

Yes, with one upgrade-safety caveat: the patch fixes the shared catalog wrapper and preserves the existing 5 second gateway startup override instead of adding provider-specific handling. Maintainers still need to accept the new 30 second default for slow providers.
