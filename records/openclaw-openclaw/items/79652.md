## Summary

Keep open: the branch adds the intended Gateway protocol surface, but the implementation only skips the Gateway-side transcript event while real agent turns still go through the Pi session prompt path that persists a user message. The supplied live proof covers a slash-command-shaped send and attachment guard, not the normal agent/model path that can leak the hidden prompt into chat history.

## What This Changes

The PR adds an admin-scoped `chat.send.hideUserMessage` protocol field, docs, Swift model support, changelog text, and gateway tests for hidden chat sends.

## Best Possible Solution

Carry hidden-turn intent into the real agent runner/session persistence path, keep the admin and attachment/deliver guards, and prove a normal non-slash agent send leaves no triggering user row in `chat.history`.

## Reproduction Assessment

Yes for the review finding from source inspection: normal `chat.send` agent turns still reach `activeSession.prompt(...)`, and the Pi dependency persists a `role: "user"` message. The provided live proof does not cover that path because it sends `/context list`.

## Solution Assessment

No. Skipping only `emitUserTranscriptUpdate` is not the narrow complete solution because the durable Pi transcript write still happens downstream; the fix should suppress or redact that persisted user turn for hidden sends and test the real agent path.
