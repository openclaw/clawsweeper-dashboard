## Summary

Keep open: this is a focused Gateway feature PR, but the current patch suppresses only the Gateway transcript helper while normal agent sends can still persist the hidden prompt in durable session history, and the supplied proof covers the slash-command path instead of that persistence path.

## What This Changes

The PR adds an admin-scoped `chat.send.hideUserMessage` Gateway protocol field, Swift model/docs/changelog updates, validation guards, and gateway tests for hidden bootstrap sends.

## Best Possible Solution

Carry hidden-turn intent into the durable agent session path, keep the admin/deliver/attachment guards, update the protocol surface consistently, and require redacted live proof from a normal agent send.

## Reproduction Assessment

Yes for the review finding from source inspection: a normal non-slash `chat.send` reaches `dispatchInboundMessage` and the Pi runner's `activeSession.prompt(...)`, while `chat.history` reads durable session messages. The supplied live proof exercises `/context list`, so it does not cover that path.

## Solution Assessment

No. Skipping only the Gateway transcript helper is not sufficient; the hidden intent needs to reach the agent/session persistence path or use a runner option that omits or redacts the persisted user turn.
