## Summary

Keep open: this is a focused and useful feature PR, but it is not merge-ready because the hidden-send flag only skips the Gateway transcript helper while normal agent sends can still persist the prompt through the Pi session path. The supplied terminal proof covers a slash-command/control path, not a normal model turn through durable history.

## What This Changes

The branch adds an admin-scoped `chat.send.hideUserMessage` Gateway protocol field, Swift model/docs/changelog updates, server-side guards, and focused gateway tests.

## Best Possible Solution

Carry hidden-turn intent into the real agent runner/session persistence path, keep the admin and attachment/deliver guards, and require fresh redacted live proof from a normal hidden agent send.

## Reproduction Assessment

Yes for the review finding from source inspection: normal non-slash `chat.send` reaches `dispatchInboundMessage` and `activeSession.prompt(...)`, while `chat.history` reads persisted transcript messages. The supplied proof uses a slash command, so it does not cover that path.

## Solution Assessment

No. Skipping only the Gateway transcript-update helper is incomplete because the durable agent session path can still record the prompt; the narrower maintainable fix should suppress, redact, or filter that persisted user turn for hidden sends.
