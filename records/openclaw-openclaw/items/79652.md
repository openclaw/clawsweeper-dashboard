## Summary

Keep this PR open: the protocol direction is plausible, but the current patch only skips the Gateway-side transcript helper and can still leak a normal hidden prompt through durable Pi session history; the supplied proof also does not exercise that normal agent path.

## What This Changes

The PR adds an optional admin-scoped `chat.send.hideUserMessage` Gateway protocol field with Swift/docs/changelog updates, validation guards, and focused Gateway tests.

## Best Possible Solution

Carry hidden-turn intent through the normal agent runner and Pi session persistence path, keep the admin/deliver/attachment guards, and require redacted live proof from a non-slash hidden agent send.

## Reproduction Assessment

Yes, for the review finding: source inspection shows a non-slash `chat.send` reaches `activeSession.prompt(...)`, while `chat.history` reads the durable transcript. I did not run a live repro because this review is read-only and the supplied proof exercises `/context list` instead of the normal agent path.

## Solution Assessment

No. The additive protocol shape is reasonable, but skipping only the Gateway transcript helper is not sufficient; the hidden flag needs to affect the runner/session persistence path too.
