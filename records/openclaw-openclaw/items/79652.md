## Summary

Keep open: the protocol direction is useful, but this PR is not merge-ready because it only skips the Gateway transcript helper while normal hidden model sends still flow into Pi session persistence and can later appear in `chat.history`. The supplied live proof covers a slash-command path and attachment rejection, not the normal agent/model path that carries the privacy-sensitive behavior.

## What This Changes

The PR adds an admin-scoped `chat.send.hideUserMessage` Gateway protocol field with Swift model, docs, changelog, validation guards, and gateway tests.

## Best Possible Solution

Carry hidden-turn intent through the normal agent runner/session persistence path, keep the admin and attachment/deliver guards, and require redacted live proof from a non-slash hidden agent send.

## Reproduction Assessment

Yes, for the review finding from source inspection: a non-slash `chat.send` reaches `runEmbeddedPiAgent(...)`, then `activeSession.prompt(...)`, while `chat.history` reads the durable session transcript. I did not run a live repro in this read-only review, and the supplied proof uses `/context list` instead of that normal model path.

## Solution Assessment

No. The additive admin-scoped protocol field is plausible, but skipping only the Gateway transcript helper is not sufficient; the hidden flag needs to affect the downstream runner/session persistence path too.
