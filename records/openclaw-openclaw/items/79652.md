## Summary

Keep open: this is a useful additive Gateway protocol feature, but the patch only suppresses the Gateway-local transcript helper and does not carry hidden-turn intent into normal Pi agent session persistence. The supplied proof exercises a `/context list` slash-command path, not a normal model turn.

## What This Changes

The PR adds admin-scoped `chat.send.hideUserMessage` support, Gateway validation guards, Swift protocol model/docs/changelog updates, and Gateway tests for hidden chat sends.

## Best Possible Solution

Finish hidden sends by ensuring normal admin-scoped hidden chat sends omit or redact the triggering user turn across durable Pi transcripts and `chat.history`, then add fresh live proof for a normal model turn.

## Reproduction Assessment

Yes for the PR finding by source inspection: a normal non-slash `chat.send` reaches `activeSession.prompt(...)`, Pi persists message-end user messages, and `chat.history` reads the durable transcript. I did not run a live failing Gateway session because this review is read-only.

## Solution Assessment

No. Skipping only the Gateway transcript helper is not complete; the maintainable fix needs to carry hidden-turn intent into the real agent/session persistence path or otherwise suppress the persisted user turn before history reads it.
