## Summary

Keep open: the branch is useful but not mergeable as-is because hidden non-slash sends can still persist the prompt through the normal Pi session path, and the supplied live proof only exercises a slash-command path.

## What This Changes

Adds an admin-scoped `chat.send.hideUserMessage` Gateway protocol field with validation guards, docs/changelog, Swift model support, and gateway tests.

## Best Possible Solution

Carry hidden-turn intent through the existing Pi user-message suppression path or an equivalent transcript policy while preserving the admin, `deliver`, and attachment guards, then prove a non-slash hidden model turn against a real Gateway.

## Reproduction Assessment

Yes. Source inspection gives high confidence: a normal non-slash send still reaches `runEmbeddedPiAgent(... transcriptPrompt ...)` and `activeSession.prompt(...)`, while `chat.history` reads the durable transcript; the supplied proof uses `/context list`, which does not exercise that path.

## Solution Assessment

No. The protocol direction is plausible, but skipping only the Gateway transcript-update helper is not enough; the hidden flag needs to reach the agent/session persistence layer.
