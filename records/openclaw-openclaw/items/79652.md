## Summary

Keep this PR open, but do not merge it as-is: the hidden-send flag only skips the Gateway transcript helper, while normal non-slash sends still go through the Pi agent session path that writes the model-visible user prompt into durable history. The supplied real proof is a live terminal run, but it exercises a slash command and attachment rejection rather than the normal agent/model path that the new API promises to hide.

## What This Changes

The PR adds an admin-scoped `chat.send.hideUserMessage` Gateway protocol field, Swift model support, docs, changelog text, validation guards, and gateway tests for hidden bootstrap sends.

## Best Possible Solution

Carry hidden-turn intent through the durable agent session path or add a narrow runtime-only bootstrap-turn contract, keep the admin and attachment/deliver guards, and prove it with a real non-slash hidden agent send.

## Reproduction Assessment

Yes for source-level reproduction: a non-slash `chat.send` reaches `activeSession.prompt(...)`, while `chat.history` reads the durable transcript. I did not run a live gateway reproduction in this read-only review, and the PR proof covers a slash-command path instead.

## Solution Assessment

No. Skipping only the Gateway transcript-update helper is not complete because the normal agent session path can still persist the prompt; the safer narrow fix is to suppress or redact that persisted user turn for hidden sends.
