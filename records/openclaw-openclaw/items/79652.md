## Summary

Keep open: the PR adds a useful hidden-send protocol surface, but the implementation only skips the Gateway-side transcript helper while normal non-slash sends still flow into durable Pi session persistence.

## What This Changes

Adds an admin-scoped `chat.send.hideUserMessage` Gateway protocol field with docs, Swift model support, validation guards, changelog text, and gateway tests.

## Best Possible Solution

Carry hidden-turn intent through the normal agent runner/session persistence path, keep the admin and attachment/deliver guards, and require redacted live proof from a non-slash hidden agent send.

## Reproduction Assessment

Yes, for the review finding from source inspection: a non-slash `chat.send` continues through the agent runner to `activeSession.prompt(...)`, and `chat.history` reads the durable transcript. The supplied proof does not exercise that path because it sends `/context list`.

## Solution Assessment

No. The protocol direction is plausible, but skipping only the Gateway transcript-update helper is not enough; the hidden-send flag needs to affect the runner/session persistence path too.
