## Summary

Keep open: the PR adds a useful hidden-send protocol foundation, but source inspection shows normal agent sends can still persist the supposedly hidden prompt, and the supplied live proof only exercises a slash-command path.

## What This Changes

The PR adds admin-scoped `chat.send.hideUserMessage` schema, docs, Swift model support, gateway validation guards, changelog text, and gateway tests.

## Best Possible Solution

Carry hidden-turn intent into the real agent runner/session persistence path, keep the admin and attachment/deliver guards, and require live proof from a normal hidden agent send.

## Reproduction Assessment

Yes for the review finding from source inspection: non-slash `chat.send` routes to the agent runner, which calls `activeSession.prompt(...)`, while `chat.history` reads persisted transcript messages. The supplied proof uses `/context list`, so it does not cover that path.

## Solution Assessment

No. Skipping only the Gateway transcript-update helper is not complete because the durable agent session path can still record the prompt; the narrow fix should suppress or redact that persisted user turn for hidden sends.
