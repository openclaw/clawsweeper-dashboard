## Summary

Keep open: current main does not implement this protocol field, and the PR still has a blocking persistence gap. The supplied proof covers a slash-command path, not a normal hidden model turn that exercises durable session history.

## What This Changes

Adds an optional admin-scoped Gateway `chat.send.hideUserMessage` flag, Swift/docs/changelog updates, validation guards, and Gateway tests for hidden sends.

## Best Possible Solution

Carry hidden-turn intent through the normal agent runner/session persistence boundary, keep the admin/deliver/attachment guards, and prove a non-slash hidden model turn against a real Gateway.

## Reproduction Assessment

Yes, for the review finding: source inspection shows a non-slash `chat.send` still flows into `runEmbeddedPiAgent` and `activeSession.prompt(...)`, while `chat.history` reads durable session messages. The supplied proof does not exercise that path because it sends `/context list`.

## Solution Assessment

No. The additive protocol direction is plausible, but suppressing only the Gateway transcript helper is not enough; the hidden flag needs to affect the runner/session persistence path as well.
