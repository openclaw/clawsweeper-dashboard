## Summary

Keep open: the Gateway protocol feature is plausible and split from the UI follow-up, but the patch only skips the Gateway-side transcript helper. Normal non-slash sends still go through the Pi agent/session persistence path, and the supplied live proof exercises `/context list` instead of that path.

## What This Changes

The PR adds admin-scoped `chat.send.hideUserMessage` support to Gateway schema/docs/Swift models, validation guards, transcript handling, changelog, and gateway tests.

## Best Possible Solution

Carry hidden-turn intent into the normal agent/session persistence path, keep the admin and attachment/deliver guards, and require fresh redacted live proof from a normal hidden model send.

## Reproduction Assessment

Yes, from source inspection: normal non-slash `chat.send` routes through `dispatchInboundMessage` into `activeSession.prompt(...)`, while `chat.history` reads durable session transcript messages. The supplied proof uses the slash-command/control path instead.

## Solution Assessment

No. Skipping only the Gateway transcript helper is not a complete fix; the narrow maintainable solution should suppress or replace the persisted user turn in the Pi session path as well.
