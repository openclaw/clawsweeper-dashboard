## Summary

Keep open: the PR targets a valid read-tool image media loss, but the new async persistence still runs under detached `tool_execution_end`, so final reply delivery can consume pending media before the saved image is queued; current proof is also only tests/CI, not real behavior proof.

## What This Changes

## Best Possible Solution
