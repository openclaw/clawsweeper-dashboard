## Summary

Keep open. Current main has coarse active-run affordances and tool/activity streams, but it does not implement the requested compact phase-based run progress with latest meaningful action, last activity, elapsed time, and reconnect context; no canonical merged or open replacement was found for that exact Control UI UX work.

## What This Changes

The issue requests a compact Control UI active-run progress indicator for long and multi-agent tasks.

## Best Possible Solution

Define a small gateway/UI run-progress contract for active chat runs and render a compact status strip near the composer or thread, reusing existing chat/session/tool events where possible and avoiding raw-log noise.

## Reproduction Assessment

No high-confidence live reproduction was run, because this is a feature request rather than a broken existing contract. Source inspection does show current main lacks the requested compact run-progress surface and only exposes coarse status/tool activity pieces.

## Solution Assessment

Unclear until maintainers choose the UX/API shape. A compact status strip is a plausible direction, but the best implementation should first settle the phase model and backend event source instead of hardcoding UI-only guesses.
