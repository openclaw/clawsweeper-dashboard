## Summary

Keep open. Current main exposes streaming text/tool cards, queued follow-ups, Stop/abort, compaction/fallback notices, and reconnect recovery, but not the requested compact active-run phase/progress model; PR #31192 is unrelated delivery retry work.

## What This Changes

Request a compact Control UI active-run progress/status strip for long or multi-agent tasks, including phase, latest action, last activity, elapsed time, and reconnect context.

## Best Possible Solution

Define a small additive Gateway/Control UI active-run progress contract and render a low-noise status strip near the thread or composer, deriving from chat, tool, lifecycle, subagent, delivery, and reconnect signals with focused UI/protocol tests and docs.

## Reproduction Assessment

No bug reproduction is applicable because this is a feature request. The feature gap is source-verifiable: current main has stream/tool/queue/fallback/compaction state but no active-run phase/latest-action/last-activity/reconnect progress model.

## Solution Assessment

Yes, the requested direction is the right class of solution, but the maintainable path is a shared active-run status contract plus compact UI rather than raw event-log rendering or unrelated delivery retry changes.
