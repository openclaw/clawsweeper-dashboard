## Summary

Keep open. Current main has a useful but coarse composer run-status chip, while the requested compact progress model still lacks latest action, elapsed/last-activity timing, subagent/tool/finalizing phases, delivery-delay state, and reconnect-during-run history.

## What This Changes

Request a compact Control UI run-progress display for long or multi-agent tasks with phase, latest action, activity timing, elapsed time, and reconnect awareness.

## Best Possible Solution

Define an additive active-run activity contract and render a compact Control UI status strip that consumes explicit stream, tool, subagent, delivery, and reconnect signals without becoming a raw event log.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature request. Source inspection gives high confidence that current main only has a coarse three-state chip and not the requested active-run progress payload or renderer.

## Solution Assessment

Unclear until maintainers settle the UX/API boundary. The requested direction is sound, but the maintainable path is a shared activity contract plus compact UI instead of brittle client-side inference.
