## Summary

Keep open. Current main and the latest release have adjacent Control UI pieces, but not the requested compact phase-based active-run progress surface with latest meaningful action, last activity, elapsed time, and reconnect context.

## What This Changes

The issue requests a compact Control UI run-progress display for long or multi-agent tasks, including run phase, latest meaningful action, last activity, elapsed time, and reconnect context.

## Best Possible Solution

Define a small canonical run-progress contract for active chat runs, then render a compact Control UI status strip near the composer/thread that reuses existing tool, session, and lifecycle events where possible without exposing raw logs or private reasoning.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection gives a high-confidence current-main gap because the active chat UI exposes only coarse run status and tool/activity pieces, not the requested compact phase/progress model.

## Solution Assessment

Unclear until maintainers choose the UX/API shape. A compact status strip is a good target, but the maintainable implementation should extend canonical session/run lifecycle signals instead of hardcoding UI-only guesses.
