## Summary

Keep open: current main and v2026.5.22 still provide only coarse Control UI run/session status, while the requested compact active-run progress model remains a product/API design task.

## What This Changes

Request a compact Control UI active-run progress strip with phase, latest meaningful action, last activity, elapsed time, and reconnect context for long or multi-agent runs.

## Best Possible Solution

Define an additive active-run activity contract in the Gateway and render a compact Control UI status strip from explicit stream, tool, subagent, delivery, and reconnect signals.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature request. Source inspection confirms current main has only coarse active/terminal status surfaces, so the requested gap is source-verifiable rather than a failing existing contract.

## Solution Assessment

Unclear until maintainers settle the UX/API boundary. The requested direction is sound, but the maintainable path is an explicit shared activity contract rather than brittle client-side inference from existing stream and session fields.
