## What This Changes

The issue requests a compact Control UI run-progress strip for long or multi-agent tasks, including phase, latest action, timing, and reconnect context.

## Best Possible Solution

Define or reuse one canonical gateway session activity/attention contract, then render a compact Control UI status strip near the composer or thread without raw logs or message-text inference.

## Solution Assessment

No current implementation is sufficient; the maintainable path is a gateway-backed activity/attention contract plus a small Control UI presentation layer, not local UI inference from message text, abortability, or tool streams alone.
