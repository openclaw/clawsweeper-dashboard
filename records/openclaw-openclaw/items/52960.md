## Summary

Keep open: current main and v2026.5.22 still expose only coarse Control UI run/session state, while the requested compact latest-action, wait-state, elapsed, last-activity, and reconnect-aware progress model remains unimplemented and product/API-shaped. Related work at https://github.com/openclaw/openclaw/issues/39127, https://github.com/openclaw/openclaw/issues/80039, https://github.com/openclaw/openclaw/pull/72771, and https://github.com/openclaw/openclaw/pull/80192 is adjacent but does not currently supersede this Control UI feature request.

## What This Changes

The issue requests a compact Control UI active-run progress surface with phase, latest activity, elapsed/last-activity timing, and reconnect awareness for long or multi-agent runs.

## Best Possible Solution

Define an additive Gateway-owned active-run activity contract, then render a compact Control UI status strip from explicit stream, tool, subagent, delivery, and reconnect signals without exposing sensitive reasoning or tool payloads.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature request. Source inspection gives high-confidence verification that current main only has coarse active/terminal status surfaces, not the requested compact progress model.

## Solution Assessment

Unclear until maintainers settle the UX/API boundary. The direction is sound, but the maintainable path is an explicit shared activity contract rather than client-side inference from existing stream and session fields.
