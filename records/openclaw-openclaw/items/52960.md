## Summary

Keep open. Current main still exposes chat streaming, live tool cards, queue controls, fallback/compaction indicators, abort handling, and reconnect cleanup, but not the requested compact active-run progress model for phase, latest action, last activity, elapsed time, wait target, or reconnect history.

## What This Changes

The issue requests a compact active-run progress indicator in the Control UI for long or multi-agent tasks.

## Best Possible Solution

Define a small Gateway/Control UI active-run progress contract and render a low-noise status strip near the thread or composer, backed by existing chat, tool, lifecycle, subagent, delivery, and reconnect signals.

## Reproduction Assessment

Yes, at source level. The current Control UI chat path carries stream/tool/queue/fallback/compaction state, but no protocol payload, controller state, render props, or tests for the requested compact run-progress model.

## Solution Assessment

Yes, the requested direction is the right class of solution, but the maintainable path is a narrow shared active-run status contract plus compact UI rather than rendering raw lifecycle logs or relying on the unrelated delivery retry fix.
