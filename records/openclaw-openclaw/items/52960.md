## Summary

Keep open. Current main still does not expose the requested compact active-run progress model in Control UI, and the closest open follow-ups split the work into broader session-activity API design and a narrower composer status chip rather than fully replacing this request.

## What This Changes

The issue requests a compact Control UI run-progress display for long or multi-agent chat runs, including phase, latest activity, elapsed time, last activity, and reconnect context.

## Best Possible Solution

Define a small additive active-run activity contract and render a low-noise Control UI status strip that can consume existing stream, tool, subagent, delivery, and reconnect signals without turning the chat into a raw event log.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature request. Static source inspection gives a high-confidence gap: current main has stream/tool/queue/fallback/compaction state, but no compact active-run progress payload or renderer.

## Solution Assessment

Unclear until maintainers settle the product/API boundary. The requested direction is reasonable, but the maintainable path is likely a narrow shared activity contract plus compact UI, coordinated with the existing session-activity and composer-status issues.
