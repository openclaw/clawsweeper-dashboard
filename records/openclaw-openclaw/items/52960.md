## Summary

Keep open: current main and v2026.6.8 still expose only coarse Control UI run status, session-row lifecycle fields, and tool-scoped activity, while the requested compact active-run progress strip remains missing. The open gateway activity issue is the canonical backend/API design dependency, but it does not fully replace this issue's Control UI presentation scope.

## What This Changes

Request a compact Control UI run-progress strip for long or multi-agent tasks showing phase, latest action, last activity, elapsed time, and reconnect/session context.

## Best Possible Solution

Define a small canonical session/run progress contract coordinated with the activity-state issue, then render a compact Control UI strip near the composer or thread without turning it into a raw event log.

## Reproduction Assessment

Yes at source level: current main has only coarse active/terminal composer state, tool-scoped activity entries, and partial session-row lifecycle fields. This is a feature gap rather than a failing documented command or test reproduction.

## Solution Assessment

Unclear until maintainers choose the UX/API shape. A compact status strip is a good target, but the maintainable solution should extend canonical gateway/session progress or attention signals rather than infer lifecycle state locally in the UI.
