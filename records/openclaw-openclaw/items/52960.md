## Summary

Keep open: current main and v2026.5.22 still expose coarse Control UI run status, while the requested compact active-run progress model remains a UX and Gateway activity-contract design task.

## What This Changes

The issue requests a compact Control UI run-progress display for long or multi-agent tasks, including current phase, latest meaningful action, activity timing, elapsed time, and reconnect context.

## Best Possible Solution

Define an additive Gateway-owned active-run activity contract, then render a compact Control UI status strip from explicit stream, tool, subagent, delivery, and reconnect signals while coordinating with the related open activity issues and PR.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature request. Source inspection gives a high-confidence current-main gap: the UI and Gateway expose only coarse run/session status rather than the requested progress model.

## Solution Assessment

Unclear until maintainers settle the UX and Gateway contract boundary. The direction is sound, but the maintainable solution is an explicit shared activity contract rather than deriving phases in the client from existing stream and session fields.
