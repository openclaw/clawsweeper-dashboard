## Summary

Keep open: current main and v2026.6.10 still expose only coarse Control UI run status plus session/tool events, not the requested compact active-run progress strip with phase, latest action, last activity, elapsed time, and reconnect context. The backend activity contract remains tracked at https://github.com/openclaw/openclaw/issues/39127, and https://github.com/openclaw/openclaw/pull/96672 is an open implementation candidate that still needs maintainer API/UX review and real behavior proof before this issue can close.

## What This Changes

The issue requests a compact Control UI run-progress surface for long and multi-agent tasks, including phase, latest action, last activity, elapsed time, and reconnect context.

## Best Possible Solution

Define or reuse one canonical gateway session activity/attention contract, then render a compact Control UI status strip near the composer or thread without raw logs or message-text inference.

## Reproduction Assessment

Yes, at source level: current main renders only a coarse in-progress/done/interrupted composer chip and exposes no sessions.activity RPC or activity statechange event. This is a feature gap rather than a failing documented command reproduction.

## Solution Assessment

No current implementation is sufficient. The maintainable path is a gateway-backed activity/attention contract plus a small Control UI presentation layer, not local UI inference from message text, abortability, or tool streams alone.
