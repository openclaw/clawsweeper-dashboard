## Summary

Keep open. Current main still has chat streaming, live tool cards, queued follow-ups, Stop/abort, compaction/fallback indicators, session active-run flags, and reconnect recovery, but it does not expose the requested compact active-run progress model; the linked delivery-queue PR is correctly treated as unrelated.

## What This Changes

The issue requests a compact Control UI active-run progress indicator for long or multi-agent tasks, including phase, latest action, last activity, elapsed time, wait target, and reconnect visibility for https://github.com/openclaw/openclaw/issues/52960.

## Best Possible Solution

Define a small Gateway/Control UI active-run progress contract and render a low-noise status strip near the thread or composer, using existing chat, tool, agent lifecycle, subagent, delivery, and reconnect signals with focused tests and Control UI docs.

## Reproduction Assessment

Yes, at source level. The current Control UI chat path only carries stream/tool/queue/fallback/compaction state and has no fields, protocol payload, controller state, or render props for the requested compact run-progress model.

## Solution Assessment

Yes, the requested direction is the right class of solution, but the maintainable path is a narrow shared active-run status contract plus compact UI, not raw event-log rendering or unrelated delivery retry logic.
