## Summary

Keep open. Current main and v2026.6.6 still provide only coarse composer status, tool-scoped activity, and session-row lifecycle fields, not the requested compact active-run progress strip with phase, latest meaningful action, last activity, elapsed time, and reconnect context.

## What This Changes

Request to add a compact Control UI active-run progress surface for long or multi-agent tasks with phase, latest action, timing, and reconnect context.

## Best Possible Solution

Define a small canonical session/run progress contract, likely coordinated with the open activity-state work, then render a compact Control UI strip near the composer or thread without turning it into a raw event log.

## Reproduction Assessment

Yes at source level: current main has coarse active/terminal composer status plus tool and session-row visibility, but no compact run-progress model with phase, latest action, last activity, elapsed time, or reconnect context. This is a feature gap rather than a failing documented behavior with a command-line repro.

## Solution Assessment

Unclear until maintainers choose the UX/API shape. A compact status strip is a good target, but the maintainable implementation should extend canonical gateway/session progress or attention signals rather than infer lifecycle state from UI-local queue and tool events.
