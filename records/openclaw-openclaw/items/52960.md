## Summary

Keep open. Current main and the latest release still have only adjacent pieces: coarse Control UI run status, tool-scoped activity entries, and session-row lifecycle fields, not the requested compact active-run progress strip with phase, latest meaningful action, last activity, elapsed time, and reconnect context.

## What This Changes

The issue requests a compact Control UI active-run progress/status surface for long and multi-agent tasks.

## Best Possible Solution

Define a small canonical session/run progress contract, likely coordinated with the open activity-state work, then render a compact Control UI strip near the composer/thread that shows phase, latest safe progress text, last activity, elapsed time, and reconnect context without becoming a raw log.

## Reproduction Assessment

Yes, at source level: current main has coarse active/terminal composer status plus tool and session-row visibility, but no compact run-progress model with phase, latest action, last activity, elapsed time, or reconnect context. This is a feature gap rather than a failing documented behavior with a command-line repro.

## Solution Assessment

Unclear until maintainers choose the UX/API shape. A compact status strip is a good target, but the maintainable implementation should extend canonical gateway/session progress or attention signals rather than infer lifecycle state from UI-local queue and tool events.
