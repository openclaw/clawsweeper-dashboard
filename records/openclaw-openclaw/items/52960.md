## Summary

Keep open. Current main has a coarse composer run-status chip and separate tool/compaction indicators, but it still does not expose the requested active-run progress model with latest action, last-activity/elapsed timing, subagent/tool/finalizing states, or reconnect-during-run history.

## What This Changes

The issue requests a compact Control UI run-progress/status strip for long or multi-agent tasks, including phase, latest action, timing, and reconnect context.

## Best Possible Solution

Define an additive session activity contract and render a compact Control UI status strip that consumes explicit stream, tool, subagent, delivery, and reconnect signals without becoming a raw event log.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature request. Source inspection gives a high-confidence current-main gap: the UI has a coarse three-state chip and internal event machinery, but not the requested active-run progress payload or renderer.

## Solution Assessment

Unclear until maintainers settle the UX/API boundary. The requested direction is reasonable, but the maintainable path should align with https://github.com/openclaw/openclaw/issues/39127 rather than inferring progress from local stream or message text alone.
