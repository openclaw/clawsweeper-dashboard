## What This Changes

Request asks for a compact Control UI active-run progress/status strip for long or multi-agent tasks, including phase, latest activity, timing, and reconnect context.

## Best Possible Solution

Define an additive active-run activity contract, then render a compact Control UI status strip from explicit stream, tool, subagent, delivery, and reconnect signals without turning it into a raw event log.

## Solution Assessment

Unclear until maintainers settle the UX/API boundary. The requested direction is sound, but the maintainable path is a shared activity contract plus compact UI rather than brittle client-side inference.
