- `impact:security`: The issue concerns whether OpenClaw state directories that can contain credentials-adjacent metadata, sessions, and logs remain private as documented.

## Summary

Keep open: current main still has agent/session state paths that create state-backed parent directories with recursive mkdir and no private mode, while the documented security contract requires the state directory to stay 700. Related work such as https://github.com/openclaw/openclaw/pull/89635 addresses a JSON-writer permission regression for https://github.com/openclaw/openclaw/issues/89589, but it does not cover the raw session transcript mkdir paths called out here.

## What This Changes

The issue asks OpenClaw agent startup to preserve the default state directory as private instead of creating or relaxing it through mode-less recursive directory creation.

## Best Possible Solution

Centralize default state-root and agent session parent creation behind a private 0700 state-directory helper, add focused POSIX regression coverage, and keep doctor/security audit as repair paths for already-open directories.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not for the exact existing-700 directory reset claim, but current source gives a clear missing-state/session-parent path where agent session startup can create state-backed directories without a private mode.

## Solution Assessment

Is this the best way to solve the issue? The suggested mode argument is directionally right but incomplete; the maintainable fix should consolidate state-root/session-parent creation and existing-directory tightening policy rather than patching one mkdir call.
