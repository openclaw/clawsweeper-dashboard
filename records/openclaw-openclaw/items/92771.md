## Summary

Keep open: the exact reset mechanism is not reproduced because Node recursive mkdir preserves an existing 700 directory, but current main and the latest release still have agent/session paths that can create a missing state tree with default 755 despite docs requiring 700. This is a security-sensitive state-directory hardening bug that needs maintainer review rather than cleanup closure.

## What This Changes

The issue asks OpenClaw to keep the state directory at 700 during agent startup and security-audit-adjacent paths instead of creating or leaving it group/world-readable via default-mode recursive mkdir.

## Best Possible Solution

Centralize private state-directory creation so default state roots and agent session parents are created and retained at 700, while keeping security audit and doctor as repair paths for already-open directories.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not for the exact existing-700 reset path: the focused Node check preserves 700 for an existing directory. Source and filesystem checks do show a high-confidence missing-state creation path where recursive mkdir can create the state root as 755.

## Solution Assessment

Is this the best way to solve the issue? The suggested mode argument is directionally right but incomplete by itself because it does not tighten existing open directories; the maintainable fix should put private state-dir creation/chmod in the shared state/session directory boundary and add regression coverage.
