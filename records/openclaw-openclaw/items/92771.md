## Summary

Keep open: current main and the latest release still have mode-less recursive mkdir calls on agent session paths under the OpenClaw state directory, while docs require the state directory to stay private. The exact existing-700 reset path is not live-proven, so this should stay open for maintainer/security review rather than cleanup closure.

## What This Changes

The issue asks OpenClaw to stop agent startup and related state-path creation from leaving `~/.openclaw` with 755 permissions instead of 700.

## Best Possible Solution

Centralize private state/session directory creation so missing state roots and agent session parents are created with 700, while keeping doctor and security audit as repair paths for already-open directories.

## Reproduction Assessment

No high-confidence live reproduction for the exact existing-700 reset path. Source inspection does show a high-confidence missing-state creation path where agent session writes can create state-backed parents without a private mode.

## Solution Assessment

The suggested `mode: 0o700` direction is plausible but incomplete by itself because existing open directories still need repair. The best fix is a shared private-directory creation boundary with focused regression coverage plus audit/doctor repair verification.
