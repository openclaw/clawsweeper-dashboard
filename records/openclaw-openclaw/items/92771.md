## Summary

Keep open: current main still has state-backed agent/session directory creation paths that call recursive mkdir without a private mode, while docs and security audit/fix contracts require the OpenClaw state tree to stay private. The exact existing-700 reset claim is not reproduced by the focused Node permission check, but the missing-state-root creation problem remains present on current main and in the latest release.

## What This Changes

The issue requests hardening `openclaw agent` startup/session paths so `~/.openclaw` and agent state directories are not created or left at default 755 permissions.

## Best Possible Solution

Centralize private OpenClaw state/session directory creation at `0o700`, keep doctor/security audit as the repair path for already-open directories, and add POSIX regression coverage for both missing and existing state roots.

## Reproduction Assessment

No for the exact existing-directory reset claim: the focused Node check keeps an existing 700 directory at 700. Yes at source level for the remaining security problem: current session mkdir paths can create missing state parents as 755 under umask 022.

## Solution Assessment

No as written: adding a mode argument to one mkdir is directionally right but too narrow. The better fix is a shared private state/session directory creation path plus audit/doctor repair coverage for already-open directories.
