## Summary

Keep open: current main still has mode-less agent session/transcript/store directory creation under the state directory, and the latest release has the same behavior, while docs and security audit expect the state directory to remain private. The exact existing-700 reset path is still unproven, but the missing-state creation path is source-reproducible; the related open work at https://github.com/openclaw/openclaw/issues/89589 and https://github.com/openclaw/openclaw/pull/89635 covers update-check JSON writes, not these session mkdir paths.

## What This Changes

The issue asks OpenClaw to stop agent/session startup paths from creating or relaxing the default state directory with umask-derived permissions instead of preserving `0700`.

## Best Possible Solution

Centralize private state-directory creation/hardening for default state, agent session store, and transcript paths, with focused tests for missing and pre-existing state dirs and coordination with the adjacent update-check permission fix.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Partially: not for the exact existing-`0700` reset path, because a focused Node check preserves an existing `0700` parent; source and Node proof do show a high-confidence missing-state creation path where recursive mkdir creates the state tree as `0755`.

## Solution Assessment

Is this the best way to solve the issue? The requested direction is right but incomplete if applied to one mkdir; the best fix is a shared private state/session directory boundary plus regression coverage, without changing generic helper defaults unless maintainers explicitly approve that compatibility impact.
