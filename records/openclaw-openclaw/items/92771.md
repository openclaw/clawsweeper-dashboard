## Summary

Keep open: current main and the latest release still have mode-less recursive mkdir paths for managed session/state directories, while the documented security contract requires the OpenClaw state directory to remain private. The linked session-permissions PR is an open candidate fix, but it is dirty against main and still has proof/security review blockers, so this issue is not implemented or safely superseded yet.

## What This Changes

The issue asks OpenClaw agent startup and session-state paths to preserve private `~/.openclaw` permissions instead of creating state directories with default public modes.

## Best Possible Solution

Centralize private managed state/session directory creation, preserve caller-supplied custom path behavior, and land the linked PR or a narrow replacement after security review and fresh/upgrade permission proof.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Partially: the exact existing-`700` reset was not reproduced, but current source plus a scratch Node filesystem check show missing managed state/session parents can be created as `755` under a normal `022` umask.

## Solution Assessment

Is this the best way to solve the issue? The suggested explicit mode is directionally right but incomplete alone; the maintainable fix should live in shared managed state/session directory creation and include coverage for fresh, upgrade, custom-path, and repair behavior.
