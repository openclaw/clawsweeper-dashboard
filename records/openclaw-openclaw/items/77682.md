## Summary

Keep open: current main still has the reported channel-lifecycle failure path, and this PR is a focused fix with no blocking review findings from this read-only pass.

## What This Changes

This PR adds an internal non-manual channel stop mode for health-monitor restarts, detaches timed-out recovery tasks, guards stale task status/completion writes, and adds gateway regression coverage plus a changelog entry.

## Best Possible Solution

Land the focused gateway lifecycle fix after normal CI and maintainer review, preserving explicit manual-stop semantics while allowing health recovery to replace a timed-out stale task.

## Reproduction Assessment

Yes. On current main, a health-monitor restart calls `stopChannel()` without a recovery mode, and `stopChannel()` always adds the account to `manuallyStopped` before the 5s timeout branch returns with the stale task still tracked.

## Solution Assessment

Yes. The PR keeps the new mode internal to the channel manager and health monitor, preserves manual stop duplicate-task behavior, and adds active-task identity guards so detached stale tasks cannot clobber replacement runtime state.
