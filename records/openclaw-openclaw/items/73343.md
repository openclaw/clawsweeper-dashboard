## Summary

Keep open. Current main still lacks the requested sender/channel identity seam for `before_tool_call`, shell exec environment construction, and bundled MCP tool calls; related PRs #72796 and #72797 are open implementation candidates but not merged.

## What This Changes

The issue requests exposing channel requester identity to skills, shell tool execution, and MCP tool calls so downstream systems can perform per-user attribution.

## Best Possible Solution

Add an additive, opt-in SDK/runtime contract that carries normalized requester identity into trusted hook context, validates any plugin-contributed exec env through the existing env safety boundary, threads approved metadata into MCP where appropriate, and documents the privacy/trust model.

## Reproduction Assessment

Yes. A high-confidence code-inspection reproduction exists by tracing `SenderId` into run state and plugin-owned tool context, then observing that `before_tool_call`, shell exec env construction, and bundled MCP calls do not receive it.

## Solution Assessment

Unclear pending maintainer design. A plugin-driven opt-in seam is safer than automatic global env injection, but the final fix should cover hook context, exec env, MCP metadata, docs, and privacy controls together.

## Security Review

Status: needs_attention

Summary: The request is security-sensitive because it would expose requester identity to executable and third-party tool surfaces.

Concerns:

- **[medium] Design identity exposure as opt-in:**
  - body: Sender and channel identifiers may be personal or tenant-scoped data. They should not be exposed to arbitrary tools, env vars, or MCP servers by default without explicit plugin/admin opt-in and documented redaction expectations.
  - confidence: 0.94
- **[medium] Validate hook-provided exec env keys:** `src/agents/bash-tools.exec.ts:1615`
  - body: Any exec-env hook must reuse or extend the current host-env validation boundary so plugins cannot inject blocked loader, path, interpreter-startup, or runtime-control variables into gateway, node, sandbox, or MCP subprocess paths.
  - confidence: 0.88
