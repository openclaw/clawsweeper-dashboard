## Summary

Keep open. Current main only partially carries requester identity: plugin-owned tool factories can receive `requesterSenderId`, but `before_tool_call` hooks, shell exec environment construction, and bundled MCP `callTool` still do not receive sender/channel identity; the linked implementation PRs are open candidates, not merged fixes.

## What This Changes

Expose trusted channel requester identity to skill/tool execution, especially hook context, exec env construction, and MCP tool calls.

## Best Possible Solution

Define an additive, opt-in SDK/runtime contract that threads normalized requester identity into approved hook context, guarded exec env injection, and MCP metadata, with sanitizer reuse, docs, API baseline updates, and regression coverage.

## Reproduction Assessment

Yes, by source inspection. Current main normalizes `SenderId` into run state, but the hook context, exec env builder, and bundled MCP call path do not forward requester identity.

## Solution Assessment

Unclear pending maintainer design. A plugin-driven opt-in hook/context contract is safer than automatic global env injection, but the final design must preserve env sanitizer behavior and define MCP forwarding boundaries.

## Security Review

Status: needs_attention

Summary: The request is security-sensitive because it would expose requester identity to executable tool and third-party MCP surfaces.

Concerns:

- **[medium] Define identity exposure as opt-in:** `src/plugins/hook-types.ts:404`
  - body: Sender and channel identifiers can be personal or tenant-scoped data, so they should not be exposed to arbitrary tools, env vars, or MCP servers by default without explicit plugin/admin opt-in and documentation.
  - confidence: 0.94
- **[medium] Preserve exec env reserved-key filtering:** `src/agents/bash-tools.exec.ts:1456`
  - body: Any hook-provided exec env values must pass through the same host/node/sandbox validation boundary as request env so plugins cannot reintroduce blocked PATH, loader, proxy, TLS, or runtime-control pivots.
  - confidence: 0.91
- **[medium] Define MCP requester metadata boundaries:** `src/agents/pi-bundle-mcp-runtime.ts:358`
  - body: Forwarding requester identity to MCP servers would extend the trust boundary beyond local shell execution, so the contract should specify what identity is sent, when it is sent, and how plugin/server authors should treat it.
  - confidence: 0.86
