## Summary

Keep open. Current main still does not expose requester identity to `before_tool_call`, shell exec environment construction, or bundled MCP `callTool`; the two related implementation PRs remain open and are active review candidates rather than merged fixes.

## What This Changes

The issue requests an opt-in Plugin SDK/runtime path for channel requester identity to reach shell/skill and MCP tool execution.

## Best Possible Solution

Define a small additive SDK/runtime contract that threads normalized requester identity into approved hook context, guarded exec env injection, and MCP metadata with docs, API baseline updates, and regression coverage.

## Reproduction Assessment

Yes, by source inspection. Current main normalizes `SenderId` into run state, but `before_tool_call`, exec env construction, and bundled MCP calls do not receive or forward requester identity.

## Solution Assessment

Unclear pending maintainer design. A plugin-driven opt-in hook/context contract is safer than automatic global env injection, but the final path must preserve env sanitizer behavior and define MCP forwarding boundaries.

## Security Review

Status: needs_attention

Summary: The request is security-sensitive because it would expose requester identity to executable tools and potentially third-party MCP servers.

Concerns:

- **[medium] Define identity exposure as opt-in:** `src/plugins/hook-types.ts:428`
  - body: Sender and channel identifiers can be personal or tenant-scoped data, so they should not be exposed to arbitrary tools, env vars, or MCP servers by default without explicit plugin/admin opt-in and documentation.
  - confidence: 0.94
- **[medium] Preserve exec env reserved-key filtering:** `src/agents/bash-tools.exec.ts:1452`
  - body: Any hook-provided exec env values must pass through the same host/node/sandbox validation boundary as request env so plugins cannot reintroduce blocked PATH, loader, proxy, TLS, or runtime-control pivots.
  - confidence: 0.91
- **[medium] Define MCP requester metadata boundaries:** `src/agents/pi-bundle-mcp-runtime.ts:351`
  - body: Forwarding requester identity to MCP servers would extend the trust boundary beyond local shell execution, so the contract should specify what identity is sent, when it is sent, and how plugin/server authors should treat it.
  - confidence: 0.86
