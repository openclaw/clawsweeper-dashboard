## Summary

Keep open. Current main still lacks the requested opt-in sender/channel identity seam for `before_tool_call`, shell exec environment construction, and bundled MCP tool calls; #72796 and #72797 are open implementation candidates but are not merged, and the remaining design is security-sensitive.

## What This Changes

Expose channel sender/channel identity to skills, tool hooks, shell exec, and MCP tool calls so downstream integrations can attribute user-triggered work.

## Best Possible Solution

Add an additive, opt-in SDK/runtime contract that carries normalized requester identity into trusted hook context, guarded exec env injection, and MCP metadata where appropriate, with sanitizer reuse, manifest/docs transparency, and privacy controls.

## Reproduction Assessment

Yes. A code-inspection reproduction traces `sessionCtx.SenderId` into run state, then verifies `before_tool_call`, exec environment construction, and bundled MCP calls omit sender/channel identity.

## Solution Assessment

Unclear pending maintainer design. A plugin-driven opt-in seam is safer than automatic global env injection, but it should cover hook context, exec env, MCP metadata, docs, and privacy controls together.

## Security Review

Status: needs_attention

Summary: The requested change is security-sensitive because it would expose requester identity to executable and third-party tool surfaces.

Concerns:

- **[medium] Design identity exposure as opt-in:**
  - body: Sender and channel identifiers can be personal or tenant-scoped data, so they should not be exposed to arbitrary tools, env vars, or MCP servers by default without explicit plugin/admin opt-in and documented redaction expectations.
  - confidence: 0.94
- **[medium] Validate hook-provided exec env keys:** `src/agents/bash-tools.exec.ts:1617`
  - body: Any exec-env hook should reuse or extend the current host-env validation boundary so plugins cannot inject blocked loader, path, interpreter-startup, or runtime-control variables into gateway, node, sandbox, or MCP subprocess paths.
  - confidence: 0.9
- **[medium] Gate MCP requester metadata:** `src/agents/pi-bundle-mcp-runtime.ts:358`
  - body: Threading sender identity into MCP `_meta` or equivalent request metadata should require explicit enablement per trusted server or plugin, otherwise external MCP servers could receive user identifiers outside the operator's intended boundary.
  - confidence: 0.86
