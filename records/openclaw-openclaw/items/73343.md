## Summary

Keep open. Current main carries sender identity into run state and plugin-owned tool factories, but the public `before_tool_call` contract, exec environment construction, and bundled MCP call path still omit requester/channel identity and env metadata; #72796 and #72797 are open candidates rather than merged fixes.

## What This Changes

Expose channel sender/channel identity to skill/tool shell execution and MCP tool calls through an opt-in plugin/runtime seam.

## Best Possible Solution

Add an additive, opt-in SDK/runtime contract that carries normalized requester identity into trusted hook context, guarded exec env injection, and MCP metadata where appropriate, with sanitizer reuse and privacy docs.

## Reproduction Assessment

Yes. A code-inspection reproduction traces `sessionCtx.SenderId` into run state, then verifies `before_tool_call`, exec env construction, and bundled MCP calls omit sender/channel identity.

## Solution Assessment

Unclear pending maintainer design. A plugin-driven opt-in seam is safer than automatic global env injection, but it should cover hook context, exec env, MCP metadata, docs, and privacy controls together.
