## What This Changes

This PR adds guarded Tool Search schema fingerprinting, stores JSON-safe catalog parameters, and covers an unreadable parameters proxy without dropping a healthy sibling tool.

## Best Possible Solution

Land one canonical Tool Search schema-hardening PR that centralizes descriptor/schema sanitization at the catalog projection boundary and covers plugin plus client tools; this PR is acceptable if maintainers prefer safe empty parameters over skipping malformed entries.

## Reproduction Assessment

Yes. Source inspection shows current main reads and fingerprints catalog `parameters` without guarding `Object.keys` or property access, so a throwing getter/proxy can abort Tool Search catalog setup; I did not run tests because this review was read-only.

## Solution Assessment

Mostly yes. The fix is at the shared catalog projection/fingerprint boundary used by Tool Search, Code Mode, and client appends; the main open choice is whether maintainers want this safe-empty-parameters behavior or one of the broader skip-entry drafts.
