## Summary

## What This Changes

The branch catches unreadable Tool Search schema metadata, serializes catalog parameters into JSON-safe metadata, and adds a regression where the malformed tool remains describable while a healthy sibling remains searchable.

## Best Possible Solution

Land one rebased catalog-boundary fix that skips or quarantines unreadable and non-JSON-projectable Tool Search entries while preserving healthy siblings and covering normal compaction plus client append paths.

## Reproduction Assessment

## Solution Assessment

No. The guard belongs at the shared Tool Search catalog boundary, but unreadable or non-JSON-projectable entries should be skipped or quarantined instead of registered as callable `{}`-schema tools.
