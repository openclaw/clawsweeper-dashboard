## Summary

Keep open: the Tool Search crash path is real on current main and the latest release, but this PR still catalogs malformed schema descriptors as callable `{}` tools and is a protected maintainer draft that needs an explicit skip/quarantine policy choice before merge.

## What This Changes

This PR makes Tool Search tolerate unreadable catalog parameter metadata by JSON-safing parameters and adds a regression where a healthy sibling stays searchable while the malformed tool describes as `{}`.

## Best Possible Solution

Land one rebased canonical Tool Search descriptor-quarantine change that skips or quarantines unreadable and non-JSON-projectable catalog entries while preserving healthy sibling tools and client append behavior.

## Reproduction Assessment

Yes, source-reproducible. Current main and v2026.6.10 directly read and fingerprint Tool Search catalog parameters during compaction and client append, so a throwing getter or proxy can abort catalog setup before registration.

## Solution Assessment

No. Guarding the shared catalog boundary is the right layer, but replacing unreadable parameters with `{}` keeps malformed tools callable; skip/quarantine behavior matches the existing runtime schema projection boundary better.
