## Summary

Keep open because current main still has the source-level backup/tempdir overlap, and https://github.com/openclaw/openclaw/pull/79790 is already the open closing fix candidate.

## What This Changes

Scheduled `openclaw backup create --verify` should pass verification when the daemon sets `TMPDIR` inside the OpenClaw state directory.

## Best Possible Solution

Review and land a focused backup-create fix that stages the temporary manifest outside backed-up assets while preserving the daemon's macOS state-local `TMPDIR` behavior.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to run `openclaw backup create --verify` with `TMPDIR` inside the state directory: current main stages the manifest under `os.tmpdir()`, archives that manifest plus the state asset, and then the verifier requires exactly one root manifest.

## Solution Assessment

Yes. The narrow maintainable fix belongs in backup-create's temp manifest placement, not in changing the daemon's macOS `TMPDIR` policy that current service-env tests assert as intentional.
