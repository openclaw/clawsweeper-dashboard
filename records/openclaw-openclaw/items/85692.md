## What This Changes

The issue reports intermittent Feishu message dispatches that complete with zero final replies after a per-chat task timeout, allegedly because the timed-out agent turn continues in the background.

## Best Possible Solution

Keep the issue open and decide a bounded fix that either aborts or fences timed-out Feishu same-session turns, while adding diagnostics or regression coverage that proves zero-final replies cannot silently persist after timeout recovery.

## Reproduction Assessment

No, we do not have a high-confidence live reproduction of the full issue. Source and tests do show the timeout can let same-key Feishu work proceed before the original task resolves, but the persistent `replies=0` state still needs sanitized diagnostics or a live Feishu repro.

## Solution Assessment

Unclear. Aborting the timed-out task is a plausible fix, but maintainers should first choose the intended timeout semantics versus session-level inflight fencing and then prove the chosen path with focused Feishu/core coverage.
