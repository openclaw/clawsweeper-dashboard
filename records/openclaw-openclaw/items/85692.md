## Summary

Keep open: current main still leaves a timed-out Feishu queue task running after eviction, and the newer generic reply-lane serialization only waits for active ownership before it can still return a zero-final busy dispatch. The full live Feishu path to persistent `replies=0` is not proven, so this needs maintainer review rather than cleanup closure.

## What This Changes

The issue reports a Feishu per-chat timeout race where queue eviction lets later same-session turns run or skip while the original agent task remains active, producing silent `replies=0` completions.

## Best Possible Solution

Keep this issue open until maintainers choose a bounded Feishu/core policy and prove that timed-out same-session work cannot silently produce zero-final replies after queue eviction.

## Reproduction Assessment

No high-confidence live reproduction for the full Feishu account failure. Source and tests do show the key race ingredients: timeout eviction does not abort the original task, later same-key work can proceed, and active reply ownership can still lead to a zero-final busy dispatch.

## Solution Assessment

Unclear. Aborting timed-out tasks is plausible, but maintainers should first choose the intended cancellation or session-fencing semantics and prove it across Feishu queueing plus shared reply-run behavior.
