## Summary

Keep open. Current main still lacks the requested persistent usage-footer default and model/thinking footer metadata, but this PR is not mergeable as-is because it is stale, has correctness bugs in usage-mode precedence and thinking reporting, and overlaps with broader open #69618.

## What This Changes

This PR adds model/thinking arguments to the response usage formatter, adds an `agents.defaults.responseUsageDefault` fallback in the reply runner, updates config type/schema files, and includes a root runtime log artifact.

## Best Possible Solution

Port the useful feature to the current usage-line module, preserve explicit session-level `responseUsage: "off"` versus inherited default semantics, use the resolved run thinking level, align docs/generated config surfaces/tests/changelog, remove the log artifact, and coordinate with #69618 before landing either path.

## Reproduction Assessment

Not applicable as a feature PR, but the review defects have high-confidence source-level reproduction paths: `/usage off` deletes the session field before the PR fallback re-enables the default, `/usage tokens` receives model/thinking metadata, and default-driven thinking is not read from the resolved run state.

## Solution Assessment

No. The product direction is reasonable, but this patch is not the best mergeable solution until it is rebased/ported and the off-inheritance, resolved-thinking, mode-contract, docs/tests/changelog, and artifact issues are fixed.
