Current main still lacks the requested usage-footer metadata and persistent default, but this PR is not mergeable as drafted because it is stale against the current formatter split and has correctness issues around explicit `/usage off`, resolved thinking level, mode semantics, and an unrelated log artifact.

## What This Changes

This PR adds model/thinking metadata to the per-response usage footer, adds an `agents.defaults.responseUsageDefault` config option in type/schema files, and includes a root runtime log file.

## Best Possible Solution

Port the useful feature to the current usage-line module, preserve explicit session-level `responseUsage: "off"` versus clear-to-inherit semantics, use the resolved run thinking level, align config type/schema/help/generated docs and token-use docs, add focused tests, remove the log artifact, and coordinate with #69618 before landing either path.

## Reproduction Assessment

Not applicable as a feature PR, but the current-main verification path is high-confidence: source, docs, and tests show no `responseUsageDefault` and no model/thinking metadata in the response usage formatter.

## Solution Assessment

No. The direction is reasonable, but this patch is not the best mergeable solution until it is rebased/ported and the session-off, resolved-thinking, mode-contract, docs, tests, and artifact issues are fixed.

Full review comments:

- **[P1] Preserve explicit session-level usage off:** `src/auto-reply/reply/agent-runner.ts:579-580`
  - body: Adding the config default as a `??` fallback makes `/usage off` ineffective when a non-off default is configured, because the current command path deletes `responseUsage` from the session entry. The next reply falls through to `responseUsageDefault` and re-enables the footer instead of preserving the session override.
  - confidence: 0.94
- **[P2] Use the resolved thinking level in the footer:** `src/auto-reply/reply/agent-runner.ts:598`
  - body: `activeSessionEntry?.thinkingLevel` only reflects persisted overrides. Sessions using `thinkingDefault` or model/provider defaults resolve the actual value onto the run, so this footer can omit or misreport the thinking mode for default-driven runs.
  - confidence: 0.89
- **[P2] Gate model and thinking metadata by usage mode:** `src/auto-reply/reply/agent-runner.ts:597-598`
  - body: The new config/type docs say `tokens` shows token counts while `full` adds cost, model, and thinking. Passing model and thinking unconditionally means `/usage tokens` also shows the extra metadata, which breaks the documented mode contract.
  - confidence: 0.86
- **[P3] Remove the committed runtime log:** `openclaw-2026-03-17.log:1`
  - body: The PR adds a root `openclaw-2026-03-17.log` file containing generated runtime warnings and local install/state path metadata. This is unrelated to the usage feature and should not be merged or shipped with the package.
