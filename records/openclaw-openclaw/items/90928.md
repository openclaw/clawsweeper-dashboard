## Summary

Keep this open: current main and the latest release still initialize the Google Play services barcode scanner during first-run Android onboarding, and no merged or viable open related item owns the no-GMS fallback yet.

## What This Changes

The issue requests HarmonyOS/Huawei no-GMS support for the Android companion app, including graceful operation without GMS and possible Huawei distribution or push support.

## Best Possible Solution

Make the QR scanner optional or lazily unavailable on no-GMS devices while keeping setup-code/manual pairing usable, then track Huawei Push or AppGallery distribution separately if maintainers choose that scope.

## Reproduction Assessment

No high-confidence live reproduction is available. Source inspection shows the first-run onboarding path eagerly creates a Google Play services scanner, but a HarmonyOS logcat trace is still needed to confirm the exact runtime crash.

## Solution Assessment

The broad requested direction is not the narrowest single fix. The best first step is graceful no-GMS scanner degradation and preserved manual/setup-code pairing, with Huawei Push and AppGallery handled as separate maintainer-scoped work.
