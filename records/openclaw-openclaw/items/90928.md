## Summary

Keep this open: current main and the latest release still construct the Google Play services code scanner in first-run Android onboarding, so the reported no-GMS launch/setup risk is not solved. The broad AppGallery/Huawei Push asks need maintainer scoping, but the setup-code/manual pairing fallback is a bounded repair path.

## What This Changes

Request support for Huawei/HarmonyOS no-GMS devices by preventing the Android app from crashing on launch and preserving Gateway pairing without Google services.

## Best Possible Solution

Add a bounded no-GMS onboarding fallback that defers or disables Google Code Scanner and keeps setup-code/manual Gateway pairing usable; track AppGallery and Huawei Push separately if maintainers want them.

## Reproduction Assessment

No live high-confidence device reproduction is available. Source inspection does show first-run onboarding constructs a Google Play services scanner on current main and the latest release, matching the reported no-GMS failure mode.

## Solution Assessment

Yes for the narrow repair: graceful scanner unavailability plus manual/setup-code pairing is the most maintainable first step. Full GMS-free distribution, AppGallery, or Huawei Push support is broader product scope.
