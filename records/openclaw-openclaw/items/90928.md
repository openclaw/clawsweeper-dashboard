## Summary

Keep open: current main and the latest shipped release still initialize the Google Play services code scanner in first-run Android onboarding, so no-GMS HarmonyOS devices plausibly remain blocked before manual/setup-code pairing can help.

## What This Changes

The issue requests HarmonyOS/Huawei no-GMS support for the Android companion app, with the narrow reported failure being an immediate launch crash from Google Play services scanner dependencies.

## Best Possible Solution

Make QR scanning gracefully unavailable or lazy on devices without Google Play services while preserving setup-code/manual pairing; track AppGallery and Huawei Push separately if maintainers want that scope.

## Reproduction Assessment

No live high-confidence reproduction is available. The issue is source-reproducible because current main still constructs the Google Play services scanner client in first-run onboarding, but a redacted HarmonyOS logcat is still needed to prove the exact crash frame.

## Solution Assessment

Yes for the narrow repair: lazy or guarded QR scanner setup with a clear manual-pairing fallback is the smallest maintainable fix. A dedicated build, AppGallery publishing, or Huawei Push support is separate product scope.
