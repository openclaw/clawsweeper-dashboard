## Label Justifications

- `P2`: The report is a platform-specific Android companion-app launch/setup failure with a clear source path but no live device log yet.
- `impact:crash-loop`: The issue reports the Android app crashing immediately before reaching the main interface on HarmonyOS no-GMS devices.

## Summary

Keep open: current main and v2026.6.6 still eagerly initialize the Google Play services code scanner in first-run Android onboarding, so no-GMS HarmonyOS devices plausibly cannot reach setup-code or manual pairing.

## What This Changes

Support HarmonyOS/Huawei no-GMS devices by letting the Android companion app launch and pair without requiring Google Mobile Services.

## Best Possible Solution

Make QR scanning gracefully unavailable or lazy on no-GMS devices while preserving setup-code/manual pairing; track AppGallery or Huawei Push separately if maintainers want that scope.

## Reproduction Assessment

No live high-confidence device repro is available. The issue is source-reproducible: first-run onboarding creates a Google Play services scanner client, but the report still needs a redacted HarmonyOS logcat to prove the exact crash frame.

## Solution Assessment

Yes for the narrow repair: graceful no-GMS degradation for QR scanning is the smallest maintainable fix that preserves existing manual/setup-code pairing. A dedicated build, AppGallery publishing, or Huawei Push support is separate product scope.
