## Summary

Keep this open: current main and the latest release still route first-run Android users through onboarding that constructs the Google Play services code scanner, while both Android flavors still include the Play Services scanner dependency. The narrow actionable part is no-GMS graceful degradation for QR scanning/manual pairing; AppGallery or Huawei Push support remains broader product scope.

## What This Changes

Requests HarmonyOS/Huawei no-GMS support for the Android companion app so it can launch and pair without Google Mobile Services.

## Best Possible Solution

Make QR scanning lazy or gracefully unavailable on no-GMS devices while preserving setup-code/manual pairing, and track AppGallery or Huawei Push separately if maintainers choose to support that distribution path.

## Reproduction Assessment

No live high-confidence reproduction is available, but the source path is clear: first-run onboarding constructs the Google Play services scanner while the reporter's device lacks GMS. A redacted HarmonyOS logcat would make the crash signature high confidence.

## Solution Assessment

The full request should not be solved as one broad HarmonyOS distribution feature. The best near-term fix is a narrow no-GMS scanner fallback that leaves existing manual/setup-code pairing usable.
