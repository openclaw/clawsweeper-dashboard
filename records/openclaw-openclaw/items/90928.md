## Summary

Keep open: current main and the latest release still make Google Play services code-scanner part of first-run Android onboarding, so the no-GMS HarmonyOS launch/setup problem is not implemented or superseded.

## What This Changes

The issue asks for the Android companion app to work on HarmonyOS/Huawei devices without GMS, with GMS-dependent QR/push behavior optional and possible GMS-free distribution.

## Best Possible Solution

Make QR scanning lazy or gracefully unavailable when Google Play services is absent while preserving setup-code/manual pairing in both Android flavors; track AppGallery or Huawei Push as separate product work if maintainers want it.

## Reproduction Assessment

No live high-confidence reproduction is available. The issue is source-reproducible because current main still constructs the Google Play services scanner client in first-run onboarding, but a redacted HarmonyOS logcat is needed to prove the exact crash frame.

## Solution Assessment

Yes for the narrow repair: lazy or guarded QR scanner setup with a manual/setup-code fallback is the smallest maintainable solution. A GMS-free distribution channel, AppGallery publishing, or Huawei Push support is separate product scope.
