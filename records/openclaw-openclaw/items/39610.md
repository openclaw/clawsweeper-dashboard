## Summary

Keep open: current main still has only global `agents.defaults.userTimezone`, and this PR is the active implementation path, but the patch still has timezone propagation bugs and lacks required real behavior proof.

## What This Changes

The PR adds `agents.list[].userTimezone`, centralizes agent-aware timezone resolution, threads it through prompt/status/envelope/heartbeat/memory paths, validates IANA timezone config, and updates docs/tests.

## Best Possible Solution

Keep the per-agent timezone config direction, complete propagation through every `user` timezone path with config-aware session resolution, add focused regression coverage, and require redacted real runtime proof before merge.

## Reproduction Assessment

Not applicable as a feature request, but the remaining PR defects are source-reproducible: legacy/global session keys still resolve memory flush to `main`, and heartbeat prompt time-line injection omits the resolved agent id. I did not run tests because the review is read-only.

## Solution Assessment

No, not as currently patched. The centralized resolver is the right shape, but the implementation must finish the memory flush and heartbeat propagation before it is the narrow maintainable solution.
