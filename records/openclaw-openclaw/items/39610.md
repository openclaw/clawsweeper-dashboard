## Summary

Keep open: current main still lacks the requested per-agent timezone contract, and this PR remains useful but is not merge-ready because source review found incomplete agent-id propagation in heartbeat, memory flush, and envelope paths.

## What This Changes

The PR adds optional `agents.list[].userTimezone`, shared agent-aware timezone resolution, config validation, docs, tests, and runtime/channel/SDK plumbing for per-agent user-local time behavior.

## Best Possible Solution

Land the feature only after maintainer acceptance of the public config contract and a rebase that consistently resolves the active config-aware agent for every `"user"` timezone path.

## Reproduction Assessment

Yes, from source inspection. Current main has a strict `agents.list[]` schema without `userTimezone`, and the current-time, heartbeat, system-event, and envelope helpers still resolve only `agents.defaults.userTimezone`.

## Solution Assessment

No, not yet. The branch is the right maintainable direction, but it still leaves heartbeat current-time injection, memory flush default-agent resolution, and envelope callers able to fall back to the global timezone.
