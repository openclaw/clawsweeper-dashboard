## Summary

Keep open: current main still only supports global `agents.defaults.userTimezone`, and this PR is the only open implementation found, but the branch is not merge-ready because heartbeat prompt injection still drops `agentId` and external real behavior proof is missing.

## What This Changes

Adds `agents.list[].userTimezone`, shared agent-aware timezone resolution, schema/docs/tests, and agent-id threading across prompt, heartbeat, envelope, gateway, memory, and system-event time paths.

## Best Possible Solution

Land a rebased implementation that makes user timezone resolution a single agent-aware helper used consistently by config/schema, prompts, heartbeat scheduling and prompt body, envelopes/system events, memory, and docs.

## Reproduction Assessment

Yes for source-level behavior: current main lacks `agents.list[].userTimezone` in `AgentConfig` and `AgentEntrySchema`, and `resolveCronStyleNow` reads only `agents.defaults.userTimezone`. The PR defect is also source-reproducible because active-hours receives `agentId` while the heartbeat prompt body does not.

## Solution Assessment

No, not yet. The centralized resolver direction is maintainable, but the heartbeat body needs the same agent context as active-hours, the branch needs current-main reconciliation, and the external PR needs real behavior proof before merge.
