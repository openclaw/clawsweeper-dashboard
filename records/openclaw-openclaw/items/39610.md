## Summary

Keep this PR open, but it is not merge-ready: current main still lacks per-agent `userTimezone`, while the PR head has two source-reproducible propagation defects and no inspectable real behavior proof from the contributor.

## What This Changes

The PR adds optional `agents.list[].userTimezone`, validates IANA timezone config, documents the setting, and threads agent-aware timezone resolution through prompt, status, envelope, heartbeat, cron, memory, gateway, plugin SDK, and bundled channel paths.

## Best Possible Solution

Keep the per-agent timezone direction, fix the remaining propagation holes, rebase against current main, add focused regressions for legacy/default-agent sessions and heartbeat prompt time, and require redacted real runtime proof before merge.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable for the feature request itself, but the remaining PR defects are source-reproducible from the live head: memory flush uses the raw session-key resolver, and heartbeat prompt time-line injection omits the resolved agent id; I did not run tests because this review is read-only.

## Solution Assessment

Is this the best way to solve the issue? No, not as currently patched. The centralized resolver is the right maintainable shape, but the implementation needs the memory-flush and heartbeat propagation fixes before it is safe to merge.

## Review Findings

Overall correctness: patch is incorrect

Full review comments:

- **[P2] Resolve flush timezone with the config-aware agent:** `src/auto-reply/reply/agent-runner-memory.ts:470`
  - body: `resolveAgentIdFromSessionKey` maps legacy/global keys to `main`, so memory flushes for a non-`main` default agent can still render `YYYY-MM-DD` and `Current time` in the wrong timezone. Use `resolveSessionAgentId({ sessionKey, config: params.cfg })` here and cover the legacy default-agent session case.
  - confidence: 0.89

  - body: This run now checks active hours with `agentId`, but it still builds the heartbeat `Body` with `appendCronStyleCurrentTimeLine(prompt, cfg, startedAt)` and no agent id. Per-agent heartbeat windows can be evaluated in one timezone while the prompt receives another; pass the same agent id into the helper and add coverage.
  - confidence: 0.87
