Full review comments:

- **[P2] Resolve flush timezone with the config-aware agent:** `src/auto-reply/reply/agent-runner-memory.ts:469-470`
  - body: `resolveAgentIdFromSessionKey` maps legacy/global keys to `main`, so memory flushes for a non-`main` default agent can still render `YYYY-MM-DD` and `Current time` in the wrong timezone.   - confidence: 0.9

  - body: This run checks active hours with `agentId`, but still builds the heartbeat body without passing that agent into `appendCronStyleCurrentTimeLine`. Multi-agent configs can evaluate quiet hours in one timezone while injecting a different `Current time` line into the prompt.
  - confidence: 0.88
- **[P2] Pass the active agent into inbound envelope options:** `src/auto-reply/reply/get-reply-run.ts:318`
  - body: `runPreparedReply` has `agentId` in scope but resolves envelope options from only `cfg`, so `Conversation info` and chat-window timestamps can still use the global default when `envelopeTimezone: "user"`. Pass the active agent id and add a regression around inbound context timestamps.
  - confidence: 0.82
