## Summary

Keep this PR open, but it still needs changes before merge: current main lacks non-retryable metadata for these validation failures, and the PR only covers one required-parameter helper while leaving shared input errors and the linked web_fetch invalid-URL case unchanged.

## What This Changes

The PR adds a retryable flag to Pi required-parameter validation errors and propagates it through the tool-definition adapter’s serialized error result.

## Best Possible Solution

Land a small agents fix that uses a typed non-retryable validation/input-error contract across required read/write/edit params, shared ToolInputError helpers, and web_fetch URL validation, with targeted tests and a changelog entry.

## Reproduction Assessment

Yes. On current main, call a wrapped read/write/edit tool with missing required params or call web_fetch with an invalid URL, then observe that the adapter serializes an error result without `retryable: false`.

## Solution Assessment

No. The direction is right, but this patch is too narrow; a shared typed input-error or validation helper is the safer maintainable solution than tagging only one helper with an ad-hoc Error property.
