## Summary

Keep open, but block merge: the linked Web UI regression is source-plausible, yet this branch fixes it by making omitted `agentId` mean all-agent gateway discovery, which reverses the scoped session lookup behavior added for cross-agent isolation.

## What This Changes

The branch changes `sessions.usage` so list requests without `agentId` return sessions from all configured agents and updates the gateway usage unit test to expect that behavior.

## Best Possible Solution

Preserve the scoped backend default and fix the Usage page by sending an explicit `agentId`, or add a separately approved aggregate-all contract with permission, compatibility, tests, and real proof.

## Reproduction Assessment

Yes from source: the Web UI `sessions.usage` request omits `agentId`, while current main scopes omission to `resolveDefaultAgentId(config)`. I did not run a live multi-agent Control UI repro in this read-only review.

## Solution Assessment

No. The PR solves the visibility symptom by broadening the backend omission path; the narrower maintainable fix is to keep gateway scoping and have the UI send an explicit selected agent scope, unless maintainers approve an explicit aggregate-all mode.
