## Summary

Keep open: current main still has the Control UI `/status` routing gap, and this PR is a focused replacement that fixes it by reusing the canonical `session_status` path. I found no blocking correctness or security findings in the patch.

## What This Changes

The PR makes Control UI `/status` execute locally through `tools.invoke`/`session_status`, bypasses active-run queueing, and adds focused regression coverage.

## Best Possible Solution

Land this focused implementation after normal maintainer approval/checks, then close the linked bug and keep the older browser-formatter PR superseded.

## Reproduction Assessment

Yes, source-level. Current main omits `status` from `LOCAL_COMMANDS`, and the slash-menu Enter handler only submits no-argument commands when `executeLocal` is true.

## Solution Assessment

Yes. Delegating local `/status` to `session_status` is the narrow maintainable fix because it preserves canonical `buildStatusText` output while fixing one-Enter submission and active-run queueing.
