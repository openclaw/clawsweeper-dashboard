## What This Changes

The issue reports that trajectory sidecar files from prior sessions are being loaded into fresh chat sessions, causing runaway input-token use and unexpected Gemini billing.

## Best Possible Solution

Capture redacted Windows/Gemini request-payload or provider-token evidence for repeated fresh sessions, then fix sidecar prompt inclusion if proven or separately tune trajectory capture limits/defaults if billing comes from ordinary runtime context.

## Reproduction Assessment

## Solution Assessment

Unclear. If sidecar replay is proven, excluding trajectory artifacts from prompt assembly is the narrow fix; if not, maintainers need to decide whether trajectory capture limits/defaults or token-accounting diagnostics should change.
