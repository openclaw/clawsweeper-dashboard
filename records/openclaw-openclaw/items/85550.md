## Best Possible Solution

Capture redacted provider request/token evidence for repeated fresh Windows/Gemini sessions, then fix the proven path: exclude sidecars from prompt assembly if replay exists, or separately tune trajectory capture limits/defaults and token diagnostics if billing comes from normal runtime context.
