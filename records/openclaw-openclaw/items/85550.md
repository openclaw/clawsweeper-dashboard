## What This Changes

The issue reports that `.trajectory.jsonl` session artifacts are being loaded into new chat sessions as model context, causing runaway input-token usage and billing on Windows/Gemini.

## Best Possible Solution

Capture Windows/Gemini request-payload or provider-token evidence for repeated fresh sessions, then either block trajectory sidecars from prompt assembly if replay is proven or separately tune trajectory capture limits/docs if the billing comes from ordinary compiled runtime/tool context.

## Reproduction Assessment

No. Source inspection explains why `.trajectory.jsonl` files can be large, but I did not find a current-main path where those sidecars are read back into provider context; Windows/Gemini payload or token evidence is still missing.

## Solution Assessment

Unclear. If sidecar replay is proven, the narrow fix is to exclude trajectory artifacts from prompt assembly; if not, the safer path is to investigate token accounting and trajectory capture defaults before changing runtime behavior.
