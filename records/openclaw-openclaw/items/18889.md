## Summary

## What This Changes

## Best Possible Solution

Replace the stale diff with a current-main patch that adds only the missing thinking and response boundaries at canonical stage transitions, serializes each start/end pair, uses explicit run correlation instead of fabricating session identity, and reuses existing exactly-once tool lifecycle handling.

Yes for the review defect: source inspection gives a high-confidence ordering path because start emissions are deliberately unawaited while end emissions can run independently. The requested lifecycle capability itself is additive, not a current-main bug reproduction.

## Solution Assessment

No. The desired observability boundaries are worthwhile, but the best fix is a current-main implementation using canonical runner stages, serialized event pairs, explicit correlation, and existing tool-hook ownership.
