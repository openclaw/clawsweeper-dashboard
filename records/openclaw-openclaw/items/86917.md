## Summary

Keep this PR open: it is a focused session-store bug fix with sufficient terminal real-behavior proof and no blocking code or security findings; the remaining concern is the session-state blast radius of changing the shared merge helper.

## What This Changes

The PR updates the shared session-entry merge helper to rewrite recognized generated transcript `sessionFile` paths when `sessionId` changes, and adds regression coverage for generated, topic-scoped, and custom paths.

## Best Possible Solution

Land this PR if required CI passes and maintainers accept the shared session-store invariant; keep the heartbeat-specific archival work in https://github.com/openclaw/openclaw/pull/73809 separate.

## Reproduction Assessment

Yes at source level: current main's merge helper preserves `sessionFile` while changing `sessionId`, so the PR's generated-path regression cases describe a clear failing path before the patch. I did not run tests because this was a read-only review.

## Solution Assessment

Yes: applying the existing generated-path rewrite helper at `mergeSessionEntryWithPolicy` centralizes the invariant and preserves custom paths when the helper cannot derive a replacement. The heartbeat archive behavior remains a separate concern in https://github.com/openclaw/openclaw/pull/73809.
