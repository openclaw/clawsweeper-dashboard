## Label Justifications

- `P1`: The first outbound reply can split one live WeChat DM into two sessions and make session reset operate on the phantom entry.
- `impact:session-state`: The defect creates a second session key for the same peer and separates outbound mirror state from the live inbound conversation.

## Summary

Current main and the latest release still preserve the faulty group classification for direct-only plugins. The focused linked repair remains open, so this issue should stay open until that PR is landed or rejected.

## What This Changes

Fix WeChat DM outbound mirroring so a bare peer ID reuses the canonical direct session instead of creating a phantom group session.

## Best Possible Solution

Keep explicit plugin routing authoritative, then use a plugin's sole declared chat type as the generic fallback when target resolution is ambiguous, with coverage distinguishing direct-only, group-capable, and channel-only plugins.

## Reproduction Assessment

Yes. Source inspection provides a high-confidence path from the plugin-normalized `group` result through `inferPeerKind` to a separately persisted group session key, although this review did not perform a live WeChat run.

## Solution Assessment

Yes in direction. A capability-aware fallback is narrower and more maintainable than scanning persisted sessions or embedding WeChat-specific ID rules in core, while explicit plugin routing remains authoritative.
