## Summary

Keep open: current main and v2026.6.6 still carry the source-reproducible fallback that can route a direct-only WeChat peer into a phantom group session, and the linked closing PR is closed unmerged rather than a viable owner of the fix.

## What This Changes

The issue reports duplicate WeChat DM sessions when outbound mirror routing classifies a bare direct-only WeChat peer ID as a group target.

## Best Possible Solution

Add a generic capability-constrained fallback that routes a resolved group guess to direct only when the channel plugin declares direct support and no group/channel support, with regression tests for both bare and pre-resolved targets.

## Reproduction Assessment

Yes at source level: current main defaults unresolved bare targets to `group`, then preserves that resolved group kind while building the outbound session key for a direct-only plugin. I did not perform a live WeChat reproduction.

## Solution Assessment

Yes, the narrow core fix is to honor declared direct-only channel capabilities in the generic fallback rather than hardcoding WeChat or adding broad session-store lookup to the outbound hot path.
