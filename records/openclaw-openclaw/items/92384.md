## Label Justifications

- `P1`: The defect deterministically duplicates every affected WeChat DM conversation and can make session reset operate on the phantom entry.
- `impact:session-state`: Outbound mirror state is persisted under a conflicting group session key for the same direct peer.

## Summary

Current `main` and release `v2026.6.6` still route a resolver-produced `group` target to a group session unless the channel is channel-only; direct-only capabilities are ignored. The clean open closing PR is therefore still necessary, so the issue should remain open until that implementation lands or is replaced.

## What This Changes

Route outbound bare peer targets for direct-only channel plugins to the existing direct session instead of creating a duplicate group-keyed delivery-mirror session.

## Best Possible Solution

Land the capability-aware core fallback from the linked PR, preserving explicit plugin routing and mixed-capability behavior while making the impossible direct-only/group combination resolve to the canonical direct session.

## Reproduction Assessment

Yes, at source level with high confidence: current main deterministically carries the erroneous group kind into session-key construction and persistence for the reported direct-only plugin shape. This review did not run a live WeChat account.

## Solution Assessment

Yes. A narrow capability-aware correction in the shared outbound fallback is more maintainable than WeChat-specific ID rules or a broad persistent-session lookup, and the linked PR covers both target-resolution entry paths.
