## Summary

This issue should stay open: current main and the latest release still carry the source-reproducible direct-to-group outbound session path, and the active repair is the open closing PR at https://github.com/openclaw/openclaw/pull/92995 rather than a merged fix.

## What This Changes

The issue asks OpenClaw to stop creating duplicate WeChat DM sessions when outbound delivery-mirror routing classifies a bare WeChat peer ID as a group target instead of the existing direct session.

## Best Possible Solution

Land or revise the open closing PR so direct-only channel plugins resolve bare DM peer IDs to the canonical direct session without adding a second runtime session-key source of truth.

## Reproduction Assessment

Yes at source level: current main still has the group-default target classifier and group-preserving outbound session fallback, while the published WeChat plugin is direct-only and marks @im.wechat IDs as id-like. I did not run a live WeChat account flow in this read-only review.

## Solution Assessment

Yes, the best path is to review, land, or revise the open focused fix PR rather than queue a duplicate repair. The fix should stay generic to channel capability contracts instead of hardcoding WeChat-specific behavior in core.
