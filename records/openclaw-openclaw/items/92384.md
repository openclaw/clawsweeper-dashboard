## Summary

Keep this open: current main still has the source-reproducible outbound fallback that can turn a WeChat direct-only peer into a phantom `group` session, and the only linked fix PR was closed unmerged.

## What This Changes

The issue requests a fix for WeChat direct-message outbound mirroring so bare `@im.wechat` peer IDs reuse the canonical direct session instead of minting a duplicate group session.

## Best Possible Solution

Land a generic capability-constrained core fallback that routes impossible `group` guesses to `direct` only for direct-only channel plugins, with focused outbound-session regression coverage and no WeChat-specific core literal.

## Reproduction Assessment

Yes at source level: current main still defaults an unresolved normalized target to `group`, then preserves that group kind when building outbound session metadata even for a direct-only plugin. I did not run a live WeChat account reproduction in this read-only review.

## Solution Assessment

Yes, the best fix is the narrow core hardening from the closed linked PR: use the plugin's declared `chatTypes` to reject an impossible group route for direct-only plugins. A plugin-side hook is still useful, but core should not mint a group session for a channel that declares no group or channel support.
