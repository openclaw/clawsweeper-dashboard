## Label Justifications

- `P1`: The report describes an always-reproducible broken WeChat DM workflow that creates conflicting session state on the first outbound reply.
- `impact:session-state`: One WeChat peer is represented by separate direct and group session identities, causing duplicate listings and resets against the wrong session.
- `impact:message-loss`: The same routing family can target the wrong WeChat session type and prevent proactive messages from reaching the intended peer.

## Summary

Current main still has the reported unknown-target-to-group fallback, and the resulting outbound route can mint a second group session for a direct-only WeChat peer. The older open WeChat issue covers the same routing family but a different full-session-key input, so this report remains useful evidence and should join one canonical repair cluster rather than close as a duplicate.

## What This Changes

Fix WeChat outbound routing so delivery mirrors for a DM peer reuse the canonical direct session instead of creating a phantom group session.

## Best Possible Solution

Implement WeChat's direct-only target inference and canonical outbound-session route through the existing plugin hooks, then add only narrowly proven core hardening that uses declared capabilities or an unambiguous existing session instead of guessing across all channels.

## Reproduction Assessment

Yes at source level: current main contains the described bare-target `group` defaults and outbound session-entry path, and the report gives a concrete observable key pair. This read-only review did not exercise a live WeChat account.

## Solution Assessment

Partly: plugin-owned `inferTargetChatType` and `resolveOutboundSessionRoute` are the narrowest primary repair, while unconditional lookup-before-mint or globally removing the group fallback is not yet proven safe for sibling channels.
