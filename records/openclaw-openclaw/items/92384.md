## Summary

Current main and the latest release still contain the source-reproducible outbound classification bug, while the clean linked closing PR remains open; keep this issue open until that PR lands or maintainers choose a different fix.

## What This Changes

Correct outbound session routing so a direct-only WeChat peer reuses its canonical direct session instead of creating a phantom group session.

## Best Possible Solution

Land a focused capability-aware fallback that maps a group-default classification to direct only when the channel declares direct support and no group or channel support, while preserving explicit plugin resolvers and mixed-chat routing.

## Reproduction Assessment

Yes, at high confidence from source: the published plugin’s direct-only capability and missing inference hooks feed current main’s group default, which the outbound fallback uses to construct a second session key. This review did not run a live WeChat account reproduction.

## Solution Assessment

Yes, the linked capability-aware core fallback is the narrowest maintainable repair because it corrects generic fallback behavior while leaving plugin-owned explicit routing authoritative; regression coverage should include both direct-only and mixed-chat siblings.
