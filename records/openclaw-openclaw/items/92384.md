## Summary

Current main still permits a resolved bare target to become a group session for a direct-only plugin, matching the reported duplicate-session path. The issue should remain open because the clean linked PR is an active closing implementation and has not merged.

## What This Changes

Prevent outbound replies through direct-only channel plugins from creating a phantom group session for the same bare peer ID.

## Best Possible Solution

Land a focused capability-aware fallback that maps an otherwise guessed group target to direct only for plugins advertising direct and neither group nor channel, with regression coverage preserving explicit and mixed-capability routing.

## Reproduction Assessment

Yes, from source with high confidence. Current main carries the resolved group kind into the fallback session key for a direct-only plugin, although this review did not perform a live WeChat run.

## Solution Assessment

Yes, conditionally: the linked capability-aware fallback is narrower and more reusable than a WeChat-specific core rule, provided it preserves explicit group routing and mixed-capability channels.
