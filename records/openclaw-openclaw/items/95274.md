## What This Changes

The PR adds Windows-aware normalization for `memory.qmd.command` in the memory host backend and regression tests for drive-letter, wrapper, and UNC QMD command paths.

## Best Possible Solution

Land one focused QMD command-normalization fix that preserves Windows drive-letter and UNC executable paths before POSIX parsing, backed by focused regression tests and redacted Windows QMD spawn/index or search proof.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: current main sends `memory.qmd.command` through `splitShellArgs`, which consumes unquoted Windows backslashes. I did not run a live Windows QMD repro in this read-only review.

## Solution Assessment

Yes, this is the right fix boundary: preserving Windows absolute commands before the POSIX shell parser is narrower than changing `splitShellArgs` globally, and fixing Windows spawn later is too late after separators are lost. The remaining gap is live Windows QMD workflow proof, not a different code location.
