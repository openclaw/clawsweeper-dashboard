## What This Changes

The branch changes the QMD memory backend command resolver to preserve Windows drive-letter and UNC `memory.qmd.command` executable paths before falling back to shell-style parsing, with regression tests for those forms.

## Best Possible Solution

Land exactly one canonical backend-config command-tokenization fix, with this PR as the strongest current candidate, then close the linked issue and same-root sibling PRs after merge.

## Reproduction Assessment

Yes, source-level: current main still routes unquoted `memory.qmd.command` through `splitShellArgs`, whose unquoted-backslash handling strips Windows separators before QMD spawn resolution. I did not run a fresh live Windows repro in this read-only review.

## Solution Assessment

Yes, this is the best current fix shape: preserving Windows drive-letter and UNC executable tokens in backend config fixes the point where data is lost. Changing the shared shell parser globally or trying to recover later in the spawn layer would be broader and riskier.
