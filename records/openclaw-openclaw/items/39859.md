## Summary

Keep open: the reported bug is still source-reproducible on current main, and this stale PR is a useful starting point but does not cover the session-store and status-summary paths named in the report.

## What This Changes

The PR changes the shared context-token resolver to cap a positive override by a known model window and adds resolver-focused regression coverage.

## Best Possible Solution

Rebase or replace this with a current-main fix that caps known effective context limits consistently in the shared resolver, session persistence, and status summary while preserving unknown-model override behavior and current provider lookup semantics.

## Reproduction Assessment

Yes. Source inspection on current main shows a high-confidence path: pass a positive contextTokensOverride above a lower configured provider/model window through the shared resolver, session-store persistence, or status-summary path, and the override can still win.

## Solution Assessment

No. Capping in the shared resolver is the right direction, but this patch needs to reuse the current configured-context lookup and cover the session-store and status-summary bypasses named in the report.
