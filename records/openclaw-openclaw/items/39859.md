## Summary

Keep open: this is a useful bug-fix PR, but the current head is dirty and still misses the reported session metadata and `/status` paths plus current provider lookup semantics; contributor real behavior proof is also absent.

## What This Changes

The PR caps `resolveContextTokensForModel()` overrides by a configured/discovered model window, adds resolver/model-config tests, and raises one legacy plugin-loader test timeout.

## Best Possible Solution

Rebase or replace this with a current-main patch that caps known effective context limits consistently in the shared resolver, session persistence, and status summary while preserving unknown-model override behavior and current provider lookup semantics.

## Reproduction Assessment

Yes. Source inspection on current main shows that passing a positive `contextTokensOverride` above a lower configured provider/model window through the shared resolver, session-store persistence, or status-summary path can still return or persist the override first.

## Solution Assessment

No, not yet. Capping in the shared resolver is the right direction, but this PR needs to rebase onto the current lookup helper and also patch the session-store and status-summary bypasses.
