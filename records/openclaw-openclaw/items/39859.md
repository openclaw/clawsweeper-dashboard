## Summary

Keep open. The bug remains source-reproducible, but the PR head is not merge-ready because it predates current provider-aware context resolution, would drop resolver safeguards, leaves raw session/status override paths, and lacks after-fix real behavior proof.

## What This Changes

The PR changes `resolveContextTokensForModel` to cap a positive context override with a known per-model context window, adds alias/capping tests, adjusts a model-config fixture, and increases one plugin-loader test timeout.

## Best Possible Solution

Rework the fix so configured overrides are applied as ceilings after the current provider-aware model-window resolver, then cover session persistence and status summary/display paths with focused regressions.

## Reproduction Assessment

Yes. Source inspection on current main shows a high `agents.defaults.contextTokens` or persisted context override can be returned before a known smaller provider/model window in shared resolver, session persistence, and status summary paths.

## Solution Assessment

No. The `min(override, knownWindow)` direction is right, but this branch should layer that cap over current main's configured-provider resolver and update raw persistence/status callers instead of replacing the resolver with narrower lookup logic.
