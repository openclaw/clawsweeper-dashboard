## Summary

Keep open: current main still has the reported source-reproducible override-first behavior, and this PR remains a relevant implementation candidate, but the patch is incomplete and lacks required real behavior proof before merge.

## What This Changes

The branch caps `resolveContextTokensForModel()` overrides by configured model context windows, adds resolver/model-config tests, and increases one legacy plugin-loader test timeout.

## Best Possible Solution

Rebase or replace this with a current-main patch that caps known effective context limits consistently in the shared resolver, session persistence, and status summary while preserving unknown-model override behavior and current provider lookup semantics.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: pass a positive `contextTokensOverride` larger than a known configured provider/model window into the shared resolver, session-store persistence, or status-summary path, and current main returns or persists the override first.

## Solution Assessment

No, not yet. Capping in the shared resolver is the right direction, but this PR misses the session metadata and `/status` bypasses and should reuse the fuller current-main provider context-token lookup.
