## Summary

Keep open. The PR targets a valid source-reproducible bug, but the current branch only caps the shared resolver, lacks after-fix real behavior proof, and does not yet cover the session-store and status-summary paths named in the report.

## What This Changes

The branch changes the shared context-token resolver to cap overrides by a known model window, adds resolver regression tests, adjusts one model-config fixture, and raises one plugin-loader test timeout.

## Best Possible Solution

Rebase or replace this with a current-main fix that reuses the existing configured-context lookup and caps known effective context limits consistently in the shared resolver, session persistence, and status summary while preserving unknown-model override behavior.

## Reproduction Assessment

Yes. Source inspection against current main shows a positive contextTokensOverride can still be returned or persisted before known provider/model context limits in the shared resolver, session-store path, and status-summary path.

## Solution Assessment

No. Capping in the shared resolver is the right direction, but this patch does not cover the named session/status surfaces and duplicates a narrower configured-context lookup instead of preserving the current contract.
