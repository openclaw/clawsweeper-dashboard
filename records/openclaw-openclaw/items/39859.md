## What This Changes

The PR caps positive context-token overrides in `resolveContextTokensForModel`, adds resolver regression tests, adjusts one model-config fixture, and raises one plugin-loader test timeout.

## Best Possible Solution

Rebase or replace the branch with a current-main fix that applies the min cap consistently in the shared resolver, session-store persistence, and status-summary paths while preserving current provider lookup semantics.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists by passing a positive `contextTokensOverride` larger than a known configured model window into current main's resolver, session-store, or status-summary paths; those paths still return or persist the override first.

## Solution Assessment

No, not yet. Capping in the shared resolver is the right direction, but this PR does not cover the persistence/status bypasses and should reuse current main's fuller provider context-token lookup semantics.
