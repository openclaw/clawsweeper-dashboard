## What This Changes

This PR caps `resolveContextTokensForModel()` overrides by a known configured model window, adds resolver regression tests, adjusts a model-config fixture, and increases one plugin-loader test timeout.

## Best Possible Solution

Land a rebased fix that caps configured or persisted context tokens by the known provider/model window across resolver, session persistence, reply model selection, and status while preserving unknown-model override behavior.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: configure `agents.defaults.contextTokens` above a configured or discovered model window, then follow the resolver/session/status paths where current main returns or stores the larger override first.

## Solution Assessment

No. The `min(override, knownWindow)` direction is right, but this branch is stale and should reuse current main's provider-aware resolver semantics while also fixing callers that bypass the resolver.

## Review Findings

Overall confidence: 0.87

Full review comments:

- **[P2] Reuse existing configured context caps:** `src/agents/context.ts:275-281`
  - body: The branch only reads the matched model's `contextWindow`. Current main also honors model `contextTokens`, provider-level caps, and exact-before-normalized provider matching, so an override above those lower caps can still over-report or regress existing lookups.   - confidence: 0.88
- **[P2] Preserve provider-aware lookup safeguards:** `src/agents/context.ts:283-286`
  - body: The new fallback uses direct qualified and bare `lookupContextTokens` calls. Current main has explicit-provider guards, provider-qualified ordering, and `allowAsyncLoad` forwarding to avoid cross-provider slash-ID collisions and read-only side effects; keep those paths and cap their resolved value instead of replacing them.
  - confidence: 0.84
- **[P2] Cap callers that bypass the resolver:** `src/agents/context.ts:291-292`
  - body: This `Math.min` only helps callers that reach the resolver. Session persistence still accepts `contextTokensOverride` before invoking it, and reply model selection returns `agentCfg.contextTokens` first, so stored and reply effective windows can remain inflated. Route those call sites through the same capping logic and cover them.
  - confidence: 0.82
