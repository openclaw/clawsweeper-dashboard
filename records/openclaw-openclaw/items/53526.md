## What This Changes

The PR changes implicit group and thread session reset defaults to a 7-day idle window while keeping direct sessions on daily reset, and updates reset-policy tests.

## Best Possible Solution

If maintainers accept the retention change, rebase it into src/config/sessions/reset-policy.ts, preserve explicit base/type/channel overrides, add exact 10080-minute coverage for no-config and unrelated-override group/thread paths, update docs/changelog, and prove the behavior in a real shared-channel setup.

## Reproduction Assessment

Yes. Source inspection on current main shows resolveSessionResetPolicy falls through to DEFAULT_RESET_MODE (daily) for group/thread when no explicit policy applies, and initSessionState already passes the derived resetType into that resolver.

## Solution Assessment

No, not as submitted. The direction may be acceptable after product/privacy approval, but the patch scopes the new default to absence of any resetByType object, edits a stale implementation target, leaves docs stale, and lacks real behavior proof.
