## Summary

Keep open: this is a plausible fix for a real shared-channel continuity problem, but the branch is not merge-ready because the submitted guard suppresses the new default in a common config shape, the behavior changes default shared-channel retention, and the external PR still lacks real behavior proof.

## What This Changes

The PR changes implicit group/thread session reset defaults to idle with a 10080-minute window while leaving direct/DM defaults daily and updating reset-policy tests.

## Best Possible Solution

If maintainers accept the retention tradeoff, implement the type-aware default in `src/config/sessions/reset-policy.ts`, preserve explicit base/type/channel overrides, add exact group/thread coverage, and update docs plus changelog.

## Reproduction Assessment

Yes. Source inspection shows current main still defaults to daily, and the PR diff's broad `hasExplicitReset` guard can be traced to a daily fallback when only an unrelated direct/dm `resetByType` entry exists.

## Solution Assessment

No. The direction may be acceptable, but this branch is not the best fix as submitted because it has a type-scoping bug, targets a stale helper location, and needs an explicit retention/privacy decision.
