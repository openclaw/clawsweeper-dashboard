## Summary

Keep this PR open, but do not merge it as submitted: current main still uses the daily implicit reset, the branch has a blocking guard bug, and the requested default-retention change needs explicit product/privacy approval plus contributor real behavior proof.

## What This Changes

The PR changes implicit group and thread session reset defaults to a seven-day idle reset while leaving direct/DM sessions on the daily default.

## Best Possible Solution

If maintainers accept the retention tradeoff, implement the type-aware implicit default in `src/config/sessions/reset-policy.ts`, preserve explicit overrides, add exact no-config and unrelated-override coverage, and update the session docs during landing.

## Reproduction Assessment

Yes, as source-level behavior: current main still routes group/thread reset types into a policy resolver whose implicit fallback is `daily`, and the tests still encode the old unrelated-DM override behavior. I did not run tests because this was a read-only review and `pnpm` is not available in the checkout environment.

## Solution Assessment

No. The direction may be acceptable, but this branch is not the best fix as submitted because the guard is scoped too broadly, the touched implementation moved on main, and the default retention change needs maintainer approval.
