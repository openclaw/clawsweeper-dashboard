## Summary

Keep this PR open, but not merge-ready. Current main still uses the documented daily implicit reset, while this branch changes a user-visible shared-channel retention default and has a blocking guard bug, stale implementation target, and missing real behavior proof.

## What This Changes

The PR changes implicit session reset defaults so group and thread sessions use a 7-day idle reset while direct/DM sessions keep daily reset, and updates reset-policy tests.

## Best Possible Solution

If maintainers accept the retention change, implement the type-aware default in `src/config/sessions/reset-policy.ts`, keep explicit overrides authoritative, add exact no-config and unrelated-override coverage, and update user-facing session/config docs plus changelog during landing.

## Reproduction Assessment

Yes, as source-level behavior: current main resolves unconfigured group/thread reset policy through the same daily fallback and current tests still encode daily behavior for the unrelated-DM override case. I did not run tests because this read-only review must not create repo artifacts.

## Solution Assessment

No. The direction may be acceptable, but this branch is not the best fix as submitted because the default guard is scoped too broadly and the implementation target moved to `src/config/sessions/reset-policy.ts`.
