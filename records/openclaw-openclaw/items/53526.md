## Summary

Keep open: current main has not implemented the requested type-aware default, but this PR is not merge-ready because the submitted guard suppresses the new group/thread idle default whenever any unrelated `resetByType` entry exists, the branch targets a stale helper location, and the default retention change needs maintainer product/privacy acceptance.

## What This Changes

The PR changes implicit group/thread session reset defaults to 7-day idle while keeping direct/DM sessions daily, and updates reset-policy tests.

## Best Possible Solution

If maintainers accept the changed default, implement it in `src/config/sessions/reset-policy.ts` with type-specific explicit-override semantics, exact no-config and unrelated-override tests, docs, and changelog; otherwise keep the documented `resetByType` workaround as the supported path.

## Reproduction Assessment

Yes, by source inspection. Current main falls back to `DEFAULT_RESET_MODE = "daily"` for group/thread policies, and the PR diff can be traced to a daily fallback when an unrelated direct/dm `resetByType` entry makes `hasExplicitReset` true.

## Solution Assessment

No. The requested direction may be acceptable, but this branch is not the best fix as submitted because it has a type-scoping bug, targets a stale helper location, lacks docs, and needs explicit shared-channel retention review.
