## Summary

Keep open. Current main and the latest release still use the existing daily implicit reset, while this PR is not merge-ready because the submitted diff has a blocking type-override bug and changes shared-channel retention without the needed docs/changelog and privacy/product acceptance.

## What This Changes

The PR changes implicit session reset defaults so group and thread sessions use idle mode with a 10080-minute window while direct/DM sessions keep daily reset, and updates reset-policy tests.

## Best Possible Solution

If maintainers accept the retention change, implement it in `src/config/sessions/reset-policy.ts`, scope implied defaults to the current reset type, keep explicit base/type/channel overrides authoritative, and update exact tests, docs, and changelog; otherwise keep the current daily default and document the explicit `resetByType` workaround.

## Reproduction Assessment

Yes. A source-level trace of the PR diff shows `resetByType: { dm: ... }` makes `hasExplicitReset` true for a group policy, suppressing the proposed idle default; current main also has tests that encode the existing daily behavior.

## Solution Assessment

No. The direction may be valid, but this submitted patch is not the best solution because it has a blocking default-selection bug, targets the pre-split helper location, and omits the required user-facing docs/changelog for a privacy-sensitive default change.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Scope the default check to this reset type:** `src/config/sessions/reset.ts:114-115`
  - body: The new `impliedDefaultMode` uses `!hasExplicitReset`, but `hasExplicitReset` becomes true when any `resetByType` entry exists. A config with only `dm` or `direct` configured therefore still resolves group/thread sessions to `daily`, so the primary fix does not apply for a common override shape.
  - confidence: 0.93
- **[P2] Update docs and changelog for the new default:** `src/config/sessions/reset.ts:114`
  - body: This line changes the implicit retention behavior for group/thread sessions, but the session docs still call daily reset the default and this user-facing fix has no changelog entry. Merging as-is would leave published guidance inconsistent with runtime behavior.
  - confidence: 0.84
- **[P3] Pin the group idle timeout assertion:** `src/config/sessions/sessions.test.ts:151`
  - body: The updated test only checks that group `idleMinutes` is not `45`, which would also pass for `undefined`, `0`, or another wrong value. Pin the intended `10080`-minute default so the new behavior is actually covered.
  - confidence: 0.82
