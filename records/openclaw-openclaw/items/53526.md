## Summary

Keep open: current main and the latest release still keep implicit session resets daily, while this branch has a blocking default-scope bug, lacks real behavior proof, and changes shared-channel retention without clear maintainer product/privacy acceptance.

## What This Changes

The PR changes implicit group/thread session reset defaults to idle with a 10080-minute window while leaving direct/DM defaults daily, and updates reset-policy tests.

## Best Possible Solution

If maintainers accept the retention tradeoff, implement the type-aware implicit default in `src/config/sessions/reset-policy.ts`, keep explicit base/type/channel overrides authoritative, add exact group/thread coverage, and update docs plus changelog.

## Reproduction Assessment

Yes. A source trace of the PR diff shows the blocking defect: with only a direct/dm `resetByType` entry, `hasExplicitReset` is true and group/thread policy falls back to `daily` instead of the proposed idle default.

## Solution Assessment

No. The requested direction may be acceptable, but this branch is not the best fix as submitted because it scopes the implied default too broadly, targets a stale helper location, and changes shared-channel retention without visible maintainer acceptance.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Scope the idle default to the current reset type:** `src/config/sessions/reset.ts:114-115`
  - body: The new implied default checks `!hasExplicitReset`, but `hasExplicitReset` is true when any `resetByType` entry exists. A config with only direct/dm configured still resolves group/thread sessions to `daily`, so the main behavior change does not apply in a common override shape.
  - confidence: 0.93
- **[P3] Pin the group idle timeout expectation:** `src/config/sessions/sessions.test.ts:151`
  - body: The test only asserts that `idleMinutes` is not `45`, which would also pass for `undefined`, `0`, or another wrong value. Assert the expected `10080` minute group/thread default so the new behavior is actually covered.
  - confidence: 0.82
