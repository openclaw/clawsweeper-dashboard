## Summary

This PR should stay open because the requested default change is still absent from current main, but the submitted patch is not merge-ready: it has a blocking guard bug, lacks real behavior proof, and changes shared-channel context retention in a way that needs explicit maintainer policy review.

## What This Changes

The PR changes implicit group and thread session reset defaults from daily reset to seven-day idle expiry while keeping direct sessions on daily reset, and updates reset-policy tests.

## Best Possible Solution

If maintainers accept the retention tradeoff, refresh the change against `src/config/sessions/reset-policy.ts`, preserve explicit base/type/channel overrides, add exact group/thread idle coverage, and update the public session docs.

## Reproduction Assessment

Yes, as source-level behavior: current main still routes group/thread reset types into resolveSessionResetPolicy, whose implicit fallback remains daily. The PR's unrelated-override bug is also directly traceable from the diff without running tests.

## Solution Assessment

No. The requested direction may be acceptable, but this branch is not the best mergeable solution because the default guard is scoped too broadly, the implementation target moved, and the retention policy needs maintainer approval.
