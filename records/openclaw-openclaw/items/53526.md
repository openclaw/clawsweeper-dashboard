## Summary

Keep open: current main and the latest release still use the daily implicit reset default, so the central request is not implemented. The PR is useful but not merge-ready because the submitted patch has a type-scoping bug, targets a helper that has since moved, lacks contributor real behavior proof, and changes shared-channel retention policy.

## What This Changes

The PR changes implicit group and thread session reset defaults from daily to 7-day idle while leaving direct/DM sessions daily, with a focused reset-policy test update.

## Best Possible Solution

If maintainers accept the retention tradeoff, refresh the change in `src/config/sessions/reset-policy.ts`, preserve base/type/channel overrides, add exact group/thread coverage for no-config and unrelated-override cases, and update session docs plus changelog during landing.

## Reproduction Assessment

Yes. Source inspection reproduces the current behavior: group/thread sessions without explicit reset config fall through to `DEFAULT_RESET_MODE`, which is still `daily`, while runtime already passes the reset type into the policy resolver.

## Solution Assessment

No. The direction may be acceptable, but this branch is not the best fix as submitted because the implied-default guard is scoped too broadly, the implementation target moved, and the shared-channel retention default needs maintainer acceptance.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Scope the idle default to the current reset type:** `src/config/sessions/reset.ts:114-115`
  - body: The new `impliedDefaultMode` uses `!hasExplicitReset`, but `hasExplicitReset` becomes true when any `resetByType` entry exists. A config with only `dm` or `direct` configured therefore still resolves group/thread sessions to `daily`, so the primary behavior change does not apply for a common override shape.
  - confidence: 0.93
- **[P3] Pin the group idle timeout expectation:** `src/config/sessions/sessions.test.ts:151`
  - body: The test only asserts that `idleMinutes` is not `45`; `undefined`, `0`, or any wrong value would also pass. Assert the expected `10080` minute default so the new behavior is actually covered.
  - confidence: 0.82
