## Summary

Keep open: current main still uses and documents the daily implicit default, but this PR is not merge-ready because the submitted guard suppresses the new group/thread default when any unrelated resetByType entry exists, and the change needs explicit shared-channel retention review.

## What This Changes

The PR changes implicit group and thread session reset defaults to seven-day idle expiry while leaving direct/DM sessions on daily reset, and updates one reset-policy test.

## Best Possible Solution

If maintainers accept the retention tradeoff, implement the type-aware implicit default in src/config/sessions/reset-policy.ts, preserve explicit base/type/channel overrides, add exact no-config and unrelated-override coverage, and update the user-facing session docs.

## Reproduction Assessment

Yes, as source-level behavior: current main routes group/thread reset types into resolveSessionResetPolicy, whose implicit fallback is still daily, and the current test suite expects an unrelated dm override to leave group sessions daily. I did not run tests because this was a read-only review.

## Solution Assessment

No. The direction may be acceptable, but this branch is not the best fix as submitted because the default guard is scoped too broadly, the implementation target moved on main, and the default-retention policy needs maintainer approval.
