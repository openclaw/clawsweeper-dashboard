## Summary

Keep this PR open but do not merge it as submitted: current main and the latest release still use the daily implicit default, while the PR has a blocking default-scoping bug and needs maintainer acceptance for the longer shared-channel retention behavior.

## What This Changes

The PR changes implicit group/thread session reset defaults from daily to idle with a 10080-minute window and updates the reset-policy compatibility test.

## Best Possible Solution

If maintainers accept the behavior change, implement the type-aware default in `src/config/sessions/reset-policy.ts`, keep explicit base/type/channel overrides authoritative, add exact no-config and unrelated-override coverage for group/thread defaults, and update docs plus changelog.

## Reproduction Assessment

Yes. A source trace of the PR diff reproduces the blocking bug: with `resetByType` containing only a direct/DM entry, `hasExplicitReset` is true and group/thread policy falls back to `daily` instead of the proposed idle default.

## Solution Assessment

No. The direction may be acceptable, but this branch is not the best fix as submitted because it scopes the implied default too broadly, targets a stale helper location, and changes shared-channel retention without visible maintainer acceptance.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Scope the idle default to the current reset type:** `src/config/sessions/reset.ts:114-115`
  - body: The new `impliedDefaultMode` uses `!hasExplicitReset`, but `hasExplicitReset` becomes true when any `resetByType` entry exists. A config with only a direct/DM override therefore keeps group/thread sessions on `daily`, so the main behavior change does not apply in a common override shape.
  - confidence: 0.93
- **[P3] Pin the group idle timeout assertion:** `src/config/sessions/sessions.test.ts:151`
  - body: The updated assertion only checks that `idleMinutes` is not `45`, which would also pass for `undefined`, `0`, or another wrong value. Pin the expected `10080`-minute default so the behavior this PR adds is actually covered.
  - confidence: 0.82
