## Summary

Keep open. Current main still lacks a WhatsApp group-create tool, and exposing agent-driven group creation plus participant invites from a linked WhatsApp account is a new product/security capability rather than a narrow bug fix.

## What This Changes

Request to add a `whatsapp_create_group` agent tool that creates WhatsApp groups with a name and phone-number participants through Baileys.

## Best Possible Solution

Maintainers should decide the WhatsApp product/security policy first; if approved, implement it inside the WhatsApp plugin as an owner-only, explicitly enabled capability with participant normalization, redacted audit logs, docs, and focused tests, otherwise document it as unsupported and leave Desktop automation or custom variants to ClawHub.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main gives high confidence that OpenClaw does not currently expose a WhatsApp group-create tool or listener/runtime path.

## Solution Assessment

Unclear. A direct ungated `groupCreate` agent tool is not the best maintainable solution until maintainers approve the product/security posture and required operator controls.
