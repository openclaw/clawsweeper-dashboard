## Summary

Keep open. Current main still exposes WhatsApp login plus message send/poll/reaction paths, but no group-creation tool or runtime method, and the discussion raises account-safety concerns that need maintainer product/security approval.

## What This Changes

The request is to add a WhatsApp `whatsapp_create_group` agent tool that creates groups from a name and participant phone numbers.

## Best Possible Solution

If approved, implement this inside the WhatsApp plugin as an explicitly gated owner-controlled capability with participant normalization, identifier redaction, outbound activity logging, abuse/account-safety docs, focused tests, and a changelog entry; otherwise document that agent-created WhatsApp groups are unsupported.

## Reproduction Assessment

Not applicable as a feature request. Current-main verification is high-confidence because source, config schema, tests, and docs show no `whatsapp_create_group` tool or group-creation runtime path.

## Solution Assessment

Unclear. A Baileys-backed implementation may be technically narrow, but the best maintainable solution needs maintainer-approved gating and account-safety policy rather than an automated plugin patch.

## Security Review

Status: needs_attention

Summary: The requested capability is security-sensitive because it would let an agent create WhatsApp groups and invite phone numbers from the linked operator identity.

Concerns:

- **[medium] Gate agent-created WhatsApp group invites:**
  - body: Any implementation should require explicit operator-controlled enablement, validate and normalize participant numbers, redact group and phone identifiers in logs, and record outbound activity so one agent turn cannot silently invite people through the linked WhatsApp account.
  - confidence: 0.9
- **[medium] Review WhatsApp account enforcement risk:**
  - body: The discussion reports that repeated Baileys group creation can look abusive to WhatsApp, especially for fixed repeated listing groups, so maintainers should approve the production posture before exposing this as an agent tool.
  - confidence: 0.82
