## Summary

Keep open. Current main still has no WhatsApp group-creation agent tool or runtime method, and the requested operation would create real WhatsApp groups and invite participants through the operator's linked account, so this needs maintainer product/security approval before any implementation work.

## What This Changes

Request to add a WhatsApp agent tool that creates a group from a group name and participant phone numbers using the WhatsApp/Baileys runtime.

## Best Possible Solution

If maintainers approve the tradeoff, implement this inside the WhatsApp plugin as a gated WhatsApp-owned capability: normalize participant numbers to WhatsApp JIDs, call group creation through an additive live-runtime method, redact identifiers in logs, record outbound activity, document account-abuse limits, and add focused tests. If they reject agent-created group invites, document that limitation in the WhatsApp docs.

## Reproduction Assessment

Not applicable as a feature request. Current-main verification is high-confidence because source and docs inspection shows login, send, poll, reaction, typing, media, and group-reply handling only, with no group-create path.

## Solution Assessment

Unclear until maintainer product/security review. A WhatsApp-owned gated tool or additive runtime seam would be the narrow maintainable path if approved; a core-generic helper or standalone ClawHub workaround would not safely access the existing linked WhatsApp session.
