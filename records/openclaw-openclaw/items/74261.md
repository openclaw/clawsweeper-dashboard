## Summary

Keep open. Current main still lacks a WhatsApp group-creation agent tool or runtime path, and the latest discussion raises credible account-safety concerns that need maintainer product/security review before implementation.

## What This Changes

Request adding a WhatsApp agent tool named `whatsapp_create_group` that creates a group from a group name and participant phone numbers via Baileys.

## Best Possible Solution

If approved, implement this inside the WhatsApp plugin as an explicit gated capability with participant normalization, identifier redaction, outbound activity logging, abuse/account-safety docs, focused tests, and a changelog entry; otherwise document that agent-created WhatsApp groups are unsupported.

## Reproduction Assessment

Not applicable as a feature request. Current-main verification is high-confidence because source, docs, runtime exports, and dependency inspection show Baileys has `groupCreate` but OpenClaw exposes no `whatsapp_create_group` tool or group-creation runtime path.

## Solution Assessment

Unclear. Calling Baileys `groupCreate` is technically narrow, but the safest maintainable solution needs maintainer-approved gating and account-safety policy before exposing group invites to agents.

## Security Review

Status: needs_attention

Summary: This issue is security-sensitive because it would let an agent create WhatsApp groups and invite phone numbers through the operator's linked identity.

Concerns:

- **[medium] Gate agent-created WhatsApp group invites:**
  - body: Any implementation should require explicit operator enablement, validate and normalize participant numbers, redact group and phone identifiers in logs, and record outbound activity so one agent turn cannot silently invite people through the linked WhatsApp account.
  - confidence: 0.9
- **[medium] Resolve WhatsApp account-enforcement posture:**
  - body: The discussion notes that repeated Baileys group creation can look like abuse to WhatsApp, so maintainers should decide whether this belongs in the plugin, needs an official-app automation path, or should remain unsupported.
  - confidence: 0.84
