## Summary

Keep open. Current main still has no WhatsApp group-creation tool or runtime path, and the requested capability would let agents create real WhatsApp groups and invite phone-number participants through the linked operator account, so it needs maintainer product/security judgment rather than an automated close or repair.

## What This Changes

Add a WhatsApp agent tool named `whatsapp_create_group` that accepts a group name and participant phone numbers and creates a WhatsApp group through Baileys.

## Best Possible Solution

If approved, the end state should live inside the WhatsApp plugin as a gated, operator-controlled capability: normalize participant numbers to WhatsApp JIDs, call group creation through an additive live-runtime method, redact phone numbers/group identifiers in logs, record outbound activity, document abuse/account-safety limits, and add focused tests/docs/changelog. If maintainers reject the risk, the WhatsApp docs should explicitly say agent-created groups are unsupported.

## Reproduction Assessment

Not applicable as a feature request. Current-main verification is high-confidence because source, runtime contracts, capability metadata, and docs inspection show no WhatsApp group-create tool, no `groupCreate` wrapper, and no documented group-creation workflow.

## Solution Assessment

Unclear. The requested Baileys `groupCreate` path is technically narrow, but the discussion raises credible account-safety concerns; a WhatsApp-owned gated runtime/tool or an explicit unsupported-docs decision would be safer than a generic core helper or automated repair.
