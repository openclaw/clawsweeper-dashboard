## Summary

Keep open. Current main still has no WhatsApp group-creation agent tool, message action, active-listener method, send/runtime API, or advertised group-management capability, and this is not a clean ClawHub-only close because an external tool cannot create groups through the already-linked WhatsApp session without a WhatsApp-owned runtime surface. The newest discussion adds credible account-abuse risk, so maintainer product/security approval is needed before implementation.

## What This Changes

Request to add a `whatsapp_create_group` tool to the WhatsApp plugin so agents can create WhatsApp groups with a name and participant phone numbers.

## Best Possible Solution

If maintainers approve the product and abuse-risk tradeoff, implement this inside the WhatsApp plugin as a gated owner/trusted capability: normalize participant numbers to WhatsApp JIDs, call group creation through the live WhatsApp runtime, redact identifiers in logs, record outbound activity, add focused WhatsApp tests, and document the limitation and risk. If they reject agent-created invites, document that explicitly in the WhatsApp channel docs.

## Reproduction Assessment

Not applicable as a feature request. Current-main verification is high-confidence because source and docs inspection shows WhatsApp login, send, poll, reaction, typing, and group-reply handling only, with no group-create path.

## Solution Assessment

Unclear until maintainer product/security review. The narrow maintainable path would be a WhatsApp-owned gated tool or additive runtime seam; a broad core helper or ClawHub-only workaround would not safely expose the existing linked WhatsApp session.
