## Summary

Keep open. Current main still has no WhatsApp group-creation tool or Baileys `groupCreate` runtime path, and the discussion raises credible account-safety concerns that need maintainer product/security review before implementation.

## What This Changes

The issue requests a WhatsApp plugin agent tool named `whatsapp_create_group` that creates a WhatsApp group from a name and participant phone numbers.

## Best Possible Solution

If approved, implement this inside the WhatsApp plugin as an explicitly enabled, owner-controlled capability with participant normalization, redacted logs, outbound activity/audit records, docs, tests, and a changelog entry; otherwise document agent-created WhatsApp groups as unsupported.

## Reproduction Assessment

Not applicable as a feature request. Current-main verification is high-confidence because exact symbol search, agent-tool registration, runtime exports, tests, and docs show no `whatsapp_create_group` or group-creation path.

## Solution Assessment

Unclear. Calling Baileys `groupCreate` is the narrowest code path, but the safer maintainable solution needs maintainer-approved gating or an official-client automation posture before exposing account-level group invites to agents.

## Security Review

Status: needs_attention

Summary: The requested feature is security-sensitive because it would let an agent create groups and invite phone numbers through the operator's WhatsApp identity.

Concerns:

- **[medium] Gate agent-created WhatsApp group invites:**
  - body: Any implementation should require explicit operator enablement, owner-only execution, participant normalization/validation, identifier redaction, and outbound activity logging so a model turn cannot silently invite people through a linked WhatsApp account.
  - confidence: 0.91
- **[medium] Review WhatsApp account-enforcement risk:**
  - body: The discussion notes that repeated Baileys group creation can resemble abuse to WhatsApp heuristics, especially for fixed per-listing invite patterns, so maintainers should decide the supported production posture before adding the tool.
  - confidence: 0.84
