## Summary

Keep open. Current main still lacks a WhatsApp group-creation agent tool or runtime wrapper, while the requested upstream Baileys method does exist; the remaining question is whether OpenClaw should expose an agent-controlled group/invite capability at all, which needs maintainer product and security judgment rather than cleanup closure or an automated repair.

## What This Changes

Add a WhatsApp agent tool that creates a named WhatsApp group and invites phone-number participants through the WhatsApp/Baileys integration.

## Best Possible Solution

If maintainers approve the capability, implement it inside the WhatsApp plugin as an explicitly gated, owner-controlled operation: normalize participant numbers to WhatsApp JIDs, require clear config/tool policy for who may invoke it, redact group and participant identifiers in logs, record outbound activity, document account-safety limits, and add focused tests/docs/changelog. If maintainers reject the risk, document that agent-created WhatsApp groups are unsupported.

## Reproduction Assessment

Not applicable as a feature request. Current-main source inspection gives high-confidence verification that OpenClaw has WhatsApp group messaging support but no group-creation tool, runtime wrapper, tests, or docs workflow.

## Solution Assessment

Unclear. Baileys groupCreate is the narrow technical call, but the best maintainable OpenClaw solution needs a WhatsApp-owned policy and safety design rather than a generic core helper or an automated one-off tool addition.
