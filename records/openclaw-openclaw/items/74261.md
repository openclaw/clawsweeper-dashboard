## Summary

Keep open. Current main still lacks a WhatsApp group-creation tool or runtime method, and the discussion raises credible account-safety concerns for repeated agent-created groups, so this needs maintainer product/security review rather than cleanup closure or an automated patch.

## What This Changes

Request to add a WhatsApp agent tool named `whatsapp_create_group` that creates a group from a name and participant phone numbers.

## Best Possible Solution

If approved, implement this inside the WhatsApp plugin as an explicit opt-in capability with participant normalization, allowlist/rate controls, redacted outbound activity logging, focused tests, docs, and a changelog entry; otherwise document that agent-created WhatsApp groups are unsupported.

## Reproduction Assessment

Not applicable as a feature request. Current-main verification is high-confidence because source, docs, and action/tool discovery show no `whatsapp_create_group` or group-creation runtime path.

## Solution Assessment

Unclear. Calling Baileys `groupCreate` may be technically narrow, but the best maintainable path needs maintainer-approved account-safety policy and gating; the thread also identifies official WhatsApp Desktop automation as a safer production alternative for some users.

## Security Review

Status: needs_attention

Summary: The requested feature would let an agent create WhatsApp groups and invite phone numbers from the linked account, so it needs explicit security and account-safety review before implementation.

Concerns:

- **[medium] Gate WhatsApp group creation:**
  - body: Any implementation should require explicit operator enablement, validate and normalize participants, constrain who/what can be invited, redact identifiers in logs, and record outbound activity so one agent turn cannot silently create groups through the operator identity.

  - body: The issue discussion notes that repeated Baileys group creation can look like abuse to WhatsApp, especially for fixed recurring listing groups, so maintainers need to approve the production posture before exposing this as an agent tool.
