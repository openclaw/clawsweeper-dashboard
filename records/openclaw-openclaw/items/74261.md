## Summary

Keep open. Current main still has no WhatsApp group-creation tool or runtime seam, and exposing group creation plus participant invites from a linked WhatsApp identity needs explicit maintainer product/security review before implementation.

## What This Changes

The issue requests a `whatsapp_create_group` agent tool for the WhatsApp plugin, taking a group name and participant phone numbers and using Baileys group creation.

## Best Possible Solution

If maintainers approve the capability, add it inside the WhatsApp plugin behind explicit operator enablement with owner/trusted-context gating, participant normalization, redacted audit logging, account-safety docs, focused tests, and a changelog entry; otherwise document agent-created WhatsApp groups as unsupported or ClawHub/plugin-owned automation.

## Reproduction Assessment

Not applicable as a feature request. Current-main verification is high-confidence because source, docs, and dependency checks show Baileys has `groupCreate`, but OpenClaw exposes only WhatsApp login plus message/poll/reaction/composing paths.

## Solution Assessment

Unclear. A direct `whatsapp_create_group` wrapper is technically plausible, but it is not the best maintainable solution until maintainers decide the security posture, enablement gate, participant handling, and account-enforcement tradeoff.
