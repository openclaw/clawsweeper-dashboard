## Summary

Keep open. Current main still has no `whatsapp_create_group` tool or WhatsApp group-creation runtime path; the request is a new high-risk WhatsApp identity operation, and the follow-up comment about account-enforcement risk makes maintainer product/security review the right next step.

## What This Changes

Add an agent-callable WhatsApp tool that creates a WhatsApp group from a group name and participant phone numbers.

## Best Possible Solution

If approved, add this inside the WhatsApp plugin as an explicitly enabled owner-controlled capability with participant validation, redacted audit logging, abuse/account-safety docs, focused tests, and a changelog entry; otherwise document agent-created WhatsApp groups as unsupported.

## Reproduction Assessment

Not applicable as a feature request. Current-main verification is high-confidence because source and docs searches show no `whatsapp_create_group`, no `groupCreate` wrapper, and WhatsApp agent-visible actions currently stop at login, polls, and reactions.

## Solution Assessment

Unclear. Baileys makes the primitive technically available, but a direct ungated tool is not the narrowest maintainable solution until maintainers approve the product/security posture and required operator controls.
