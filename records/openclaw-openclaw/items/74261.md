## Summary

Keep open. Current main still lacks a WhatsApp group-create tool, and exposing agent-driven group creation/invites from a linked WhatsApp account is a new product and security decision rather than conservative cleanup.

## What This Changes

The issue requests a new `whatsapp_create_group` agent tool that creates a WhatsApp group with a name and phone-number participants.

## Best Possible Solution

Maintainers should first decide the WhatsApp account-safety policy; if approved, implement the capability in the WhatsApp plugin with explicit operator enablement, owner-only access, participant normalization, redacted logging, docs, and focused tests.

## Reproduction Assessment

Not applicable as a feature request. Source and docs inspection on current main gives high confidence that OpenClaw exposes WhatsApp send/reply/reaction/poll paths but no group-create tool or runtime method.

## Solution Assessment

Unclear. A direct ungated `whatsapp_create_group` wrapper around Baileys is not the best maintainable solution until maintainers approve the product/security policy and required operator controls.
