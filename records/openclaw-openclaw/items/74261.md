## Summary

Keep open. Current main still has no WhatsApp group-creation tool or runtime path, and the newer discussion raises credible WhatsApp account-safety concerns that need maintainer product/security judgment before exposing agent-created group invites.

## What This Changes

Request to add a `whatsapp_create_group` agent tool that creates WhatsApp groups from a group name and participant phone numbers via Baileys `groupCreate`.

## Best Possible Solution

If approved, implement this inside the WhatsApp plugin as an explicitly gated owner-controlled capability with participant normalization, identifier redaction, outbound activity logging, focused tests, docs, and a changelog entry; otherwise document that agent-created WhatsApp groups are unsupported.

## Reproduction Assessment

Not applicable as a feature request. Source, config, runtime, and docs inspection provide a high-confidence current-main verification path showing no `whatsapp_create_group` tool or group-creation runtime method.

## Solution Assessment

Unclear. The likely implementation boundary is the WhatsApp plugin rather than core, but direct Baileys `groupCreate` is not clearly the best production posture until maintainers decide how to handle account-safety and consent gating.

## Security Review

Status: needs_attention

Summary: This request is security-sensitive because it would let an agent create WhatsApp groups and invite phone numbers from the operator’s linked identity.

Concerns:

- **[medium] Gate group creation and invitations:**
  - body: Any implementation should require explicit operator enablement, owner-only or similarly trusted access, participant normalization/validation, redacted group and phone identifiers in logs, and durable outbound activity records before an agent can invite people.
  - confidence: 0.9
- **[medium] Review WhatsApp account-enforcement risk:**
  - body: The discussion flags that repeated Baileys `groupCreate` calls with fixed participant patterns can look abusive to WhatsApp, so maintainers should approve the production posture before exposing this as an agent tool.
  - confidence: 0.84
