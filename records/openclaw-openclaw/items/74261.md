## Summary

Keep open. Current main still lacks a `whatsapp_create_group` tool or WhatsApp group-creation runtime path; Baileys exposes the primitive, but agent-controlled group creation and participant invites need maintainer product/security approval before implementation.

## What This Changes

Add a `whatsapp_create_group` agent tool that creates a WhatsApp group from a group name and participant phone numbers using Baileys `groupCreate`.

## Best Possible Solution

If approved, implement this inside the WhatsApp plugin as a disabled-by-default, owner-controlled capability with participant normalization, identifier redaction, outbound activity logging, abuse/account-safety docs, focused tests, and a changelog entry; otherwise document that agent-created WhatsApp groups are unsupported.

## Reproduction Assessment

Not applicable as a feature request. Current-main verification is high-confidence because source search plus the WhatsApp channel, login tool, active listener, send wrapper, config schema, docs, and action discovery all show no group-creation surface.

## Solution Assessment

Unclear. Baileys makes the primitive technically available, but a direct `groupCreate` agent tool is not the best maintainable solution until maintainers approve gating, logging, and account-safety policy.
