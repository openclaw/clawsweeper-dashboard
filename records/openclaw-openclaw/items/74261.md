Keep open. Current main still has no WhatsApp group-creation tool or runtime path, and the discussion adds credible account-safety concerns, so this needs maintainer product/security review rather than cleanup closure or an automated repair.

## What This Changes

Add a WhatsApp plugin agent tool named `whatsapp_create_group` that creates a WhatsApp group from a supplied group name and phone-number participants.

## Best Possible Solution

If approved, implement this inside the WhatsApp plugin as an explicit gated capability with participant normalization, identifier redaction, outbound activity logging, abuse/account-safety docs, focused tests, and a changelog entry; otherwise document that agent-created WhatsApp groups are unsupported.

## Security Review

Status: needs_attention

Summary: This issue is security-sensitive because it proposes agent-created WhatsApp groups and phone-number invites from the operator's identity.

Concerns:

- **[medium] Gate agent-created WhatsApp group invites:**
  - body: Any implementation should require explicit operator-controlled enablement, normalize and validate participant numbers, redact group and phone identifiers in logs, and record outbound activity so one agent turn cannot silently invite people through the linked WhatsApp account.
  - confidence: 0.9
- **[medium] Account enforcement risk needs product review:**
  - body: A maintainer comment notes repeated Baileys group creation can look like abuse to WhatsApp, so the project needs an approved production posture before exposing this as an agent tool.
  - confidence: 0.84
