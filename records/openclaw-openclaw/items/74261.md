## Summary

Keep open. Current main still has no WhatsApp group-creation tool or listener/send API method, while the installed Baileys dependency does expose `groupCreate`; the remaining decision is product/security policy for agent-created groups and participant invites from the linked WhatsApp identity.

## What This Changes

Request adding a WhatsApp-owned `whatsapp_create_group` agent tool that accepts a group name and phone-number participants and creates a WhatsApp group through Baileys.

## Best Possible Solution

If maintainers approve the product and account-risk tradeoff, implement this inside the WhatsApp plugin as a gated owner-only or trusted-requester capability: normalize participants to WhatsApp JIDs, expose a narrow active-listener/send API method that calls Baileys `groupCreate`, redact identifiers in logs, record outbound activity, add focused WhatsApp tests, and document the safety limits. If maintainers do not approve it, document group creation as unsupported and point users to safer manual or official-app automation paths.

## Reproduction Assessment

Not applicable as a feature request. Current-main verification is high-confidence: source, docs, runtime API, dependency source, and tool/action surfaces show login/send/poll/reaction/typing only, with no group-create path.

## Solution Assessment

Unclear until maintainer product/security review. The proposed Baileys call is technically possible, but the maintainable solution is either a WhatsApp-owned gated runtime/tool surface or an explicit docs limitation; a broad core helper or ClawHub-only workaround is not sufficient today.
