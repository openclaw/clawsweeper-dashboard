## Summary

Keep open. Current main still does not expose a WhatsApp group-create tool, and turning Baileys group creation into an agent-callable capability is a new product/security decision rather than conservative cleanup.

## What This Changes

Request an agent-callable `whatsapp_create_group` tool for the WhatsApp plugin that accepts a group name and participant phone numbers and calls Baileys `groupCreate`.

## Best Possible Solution

Maintainers should decide the WhatsApp product/security policy first; if approved, implement it in the WhatsApp plugin as an owner-only, explicitly enabled capability with participant normalization, redacted audit logs, docs, and focused tests.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main gives high confidence that OpenClaw currently exposes WhatsApp send/reply/reaction/poll paths but no group-create tool or runtime method.

## Solution Assessment

Unclear. A direct ungated `groupCreate` agent tool is not the best maintainable solution until maintainers approve the account-safety policy and required operator controls.
