## Summary

Keep open: current main still has the reported managed npm-root conflict path, and no related fix is present after v2026.6.5. Source inspection confirms OpenClaw imports shipped workspace overrides into managed plugin roots while preserving stale OpenClaw-managed peer dependency pins, which can produce npm's documented `EOVERRIDE` root dependency conflict.

## What This Changes

The issue reports that `openclaw plugins install` can fail with npm `EOVERRIDE` when stale managed peer dependency pins conflict with pnpm workspace overrides exported into a managed npm project.

## Best Possible Solution

Fix managed npm-root reconciliation so OpenClaw-managed peer pins cannot remain out of sync with OpenClaw-managed overrides before npm runs, with a regression test for a stale peer pin plus a conflicting host override.

## Reproduction Assessment

Yes, from source inspection: seed a managed npm project with `dependencies.hono = 4.12.23` and `openclaw.managedPeerDependencies = ["hono"]`, then run the current install path that imports `overrides.hono = 4.12.18` before npm install. I did not execute the live install because this review is read-only.

## Solution Assessment

Yes, the best fix belongs in the managed npm-root reconciliation layer, not in a plugin-specific workaround. The narrow repair should advance or reconcile only OpenClaw-managed peer pins and managed override conflicts before npm sees the root manifest.
