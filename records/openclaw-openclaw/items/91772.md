## Summary

Keep open: current main and the latest release still contain the managed npm-root conflict path, and https://github.com/openclaw/openclaw/pull/91786 is an open closing PR rather than a merged fix. The reporter’s follow-up also says the behavior still reproduces on v2026.6.6, so this issue should track review and landing of that PR or an equivalent fix.

## What This Changes

Issue reports that managed plugin npm roots can preserve stale OpenClaw-managed peer pins while importing newer workspace overrides, producing npm EOVERRIDE during plugin install or update.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/91786, or replace it with an equivalent managed npm-root fix that guarantees OpenClaw-managed peer pins and managed overrides cannot conflict before npm runs.

## Reproduction Assessment

Yes. Seed a managed npm root with a stale direct dependency such as `hono` listed in `openclaw.managedPeerDependencies`; current main preserves that dependency spec and writes the current managed override before invoking npm. I did not run the mutating install path in this read-only review.

## Solution Assessment

Yes at the issue level: the fix belongs in the managed npm-root manifest reconciliation layer before npm runs, not in a plugin-specific workaround. The open linked PR is the active implementation path and should be reviewed for its upgrade behavior before merge.
