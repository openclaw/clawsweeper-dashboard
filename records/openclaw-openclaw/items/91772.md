## What This Changes

The issue reports `openclaw plugins install` failing with npm `EOVERRIDE` when stale OpenClaw-managed peer pins conflict with newer managed overrides imported into plugin npm roots.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/91786 if maintainers accept its managed-root upgrade behavior, then close this issue as implemented while tracking the misleading post-failure hook-pack message separately.

## Reproduction Assessment

Yes. Source inspection shows a managed root with stale dependencies.hono and openclaw.managedPeerDependencies can receive a newer managed overrides.hono before npm runs, which npm's Arborist rejects as EOVERRIDE.

## Solution Assessment

Yes. The fix belongs in shared managed npm-root reconciliation across install/update/rollback writers; the open PR is the best current path, pending maintainer acceptance of the upgrade rewrite behavior.
