## What This Changes

The issue reports that plugin install/update can fail with npm EOVERRIDE when stale OpenClaw-managed peer dependency pins conflict with newer workspace overrides imported into the managed npm root.

## Best Possible Solution

Land the shared managed npm-root reconciliation from the linked PR, then track installer error wording or plugin-disablement behavior separately if maintainers want that changed.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to seed a managed npm root with `dependencies.hono` at an old managed peer pin and `openclaw.managedPeerDependencies` containing `hono`, then run the current plugin install path that imports the newer `overrides.hono` before npm install.

## Solution Assessment

Yes. The best fix belongs in the shared managed npm-root manifest reconciliation layer so every install, update, rollback, and peer-sync writer preserves npm's root override invariant before npm sees the manifest.
