## Summary

Keep open: current main and v2026.6.6 still preserve stale managed peer pins while importing managed overrides, and https://github.com/openclaw/openclaw/pull/91786 is the active open fix path.

## What This Changes

The issue reports plugin installs/updates failing with npm EOVERRIDE when OpenClaw-managed npm roots contain stale peer pins that conflict with shipped pnpm workspace overrides.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/91786 or an equivalent managed-root reconciliation that aligns OpenClaw-managed peer pins with OpenClaw-managed overrides before npm runs.

## Reproduction Assessment

Yes. Source inspection plus the reporter's v2026.6.6 trace show that a managed npm root with `dependencies.hono` at an older managed peer pin and `openclaw.managedPeerDependencies` for the same package can receive a newer `overrides.hono` before npm runs, which npm Arborist rejects as EOVERRIDE.

## Solution Assessment

Yes. The best fix belongs in shared managed npm-root manifest reconciliation rather than a plugin-specific workaround; the open fix PR appears to target that owner boundary, but this issue should stay open until it merges.
