## Summary

Keep open. The issue is maintainer-authored, so it should not be auto-closed, and the requested SDK-facing Gateway task RPCs are still missing on current main: the SDK task namespace intentionally throws unsupported, and the Gateway advertised method list, handler map, protocol schemas, and scope table do not expose `tasks.list`, `tasks.get`, or `tasks.cancel`.

## What This Changes

Expose background task ledger operations through stable Gateway RPCs and wire `@openclaw/sdk` task list/get/cancel methods to them.

## Best Possible Solution

Add additive, schema-backed Gateway task RPCs that wrap the existing task registry through an explicit access contract, classify read/write scopes, document the public payloads, and update `@openclaw/sdk` to stop throwing unsupported for the basic list/get/cancel operations.
