## Summary

Keep this PR open, but it is not merge-ready: it fixes the reported invalid-config exit-code symptom, while changing a compatibility-sensitive plain-doctor contract and leaking `process.exitCode` through shared doctor health into embedded callers.

## What This Changes

The PR makes final doctor config validation set `process.exitCode = 1` for invalid final config, passes that through the non-lint doctor command, and adds focused regression tests.

## Best Possible Solution

Land a version that reports invalid final config through an explicit doctor status boundary, with the plain-doctor exit contract either preserved or deliberately documented and covered for embedded callers.

## Reproduction Assessment

## Solution Assessment

No. The patch targets the right symptom, but mutating `process.exitCode` inside a shared health contribution is the wrong boundary because embedded doctor callers can inherit the status.
