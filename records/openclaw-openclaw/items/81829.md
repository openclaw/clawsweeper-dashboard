## Summary

Keep this PR open, but not merge-ready: it fixes a real current-main gap, while the chosen implementation changes the plain `openclaw doctor` exit contract and mutates `process.exitCode` inside shared doctor health code that is also called by update finalization.

## What This Changes

The PR makes final doctor config validation set `process.exitCode = 1` for an invalid final config snapshot, passes that code through the top-level doctor command, and adds focused regression tests.

## Best Possible Solution

Decide the intended plain-doctor exit contract, then keep exit status local to that command path or return an explicit doctor status object, with update-finalization coverage and docs updated if plain doctor intentionally fails closed.

## Reproduction Assessment

Yes from source inspection: current main prints final `Invalid config:` issues but the top-level non-lint doctor command still exits `0`. I did not run a live current-main CLI repro because this review is read-only.

## Solution Assessment

No. The fix targets the right symptom, but mutating `process.exitCode` inside a shared health contribution is not the best boundary because it can affect embedded callers outside the top-level doctor CLI.
