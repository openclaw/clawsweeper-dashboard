## Summary

Keep open: https://github.com/openclaw/openclaw/pull/73362 fixed the original default-image fallback and shipped in v2026.5.7, but current main still lets custom/slim sandbox images fail write/edit with the raw pinned-helper missing-Python error. The recent reporter update matches current source, and https://github.com/openclaw/openclaw/pull/56785 tracks the remaining diagnostic work but is still open and stale against the current helper launcher.

## What This Changes

The issue asks OpenClaw to ensure Docker sandbox write/edit dependencies are available in the default sandbox image or to surface a clear failure when Python is missing.

## Best Possible Solution

Keep the shipped default-image fix, then land a current-main version of the missing-Python diagnostic that maps both local and remote mutation-helper failures to actionable image guidance with focused tests and real Docker proof.

## Reproduction Assessment

Yes, source-reproducible: a sandbox image without python3/python reaches the pinned mutation helper, which exits with the current missing-Python message, and the local fs bridge does not translate it. I did not run a live Docker container in this read-only review.

## Solution Assessment

No, current main is only a partial solution: the default-image fallback is fixed, but the custom/slim-image helper guidance remains unresolved. The narrow maintainable path is a rebased https://github.com/openclaw/openclaw/pull/56785-style diagnostic fix rather than closing the issue now.
