## Summary

Keep open: current main and the shipped default-image fix address the old plain-Debian fallback, but the current mutation helper still requires Python inside the active sandbox image and still surfaces the generic missing-Python failure for custom/slim images. The fresh 2026-05-09 report and the open diagnostic PR mean this is still actionable as the canonical remaining issue rather than safe to close.

## What This Changes

The issue asks OpenClaw to make Docker sandbox write/edit work with the default/slim sandbox image dependency, or fail with clear guidance when Python is missing.

## Best Possible Solution

Keep this issue open as the canonical thread until a current-main patch maps missing-Python sandbox mutation failures to actionable image guidance for custom/slim images, likely by repairing or replacing https://github.com/openclaw/openclaw/pull/56785.
