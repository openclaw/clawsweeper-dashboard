Keep open. The PR refreshes the shared typing controller TTL after keepalive-triggered typing sends and adds a regression test that advances past the original TTL while keepalive ticks continue.

## Best Possible Solution

Land the TTL-renewal fix after replaying it onto the current triggerInFlight/error-handling shape, keep the regression test, add an Unreleased/Fixes changelog entry crediting the contributor, and leave #65919 open for the broader active-run presence lease work.

## Solution Assessment

Yes. Renewing the run-side TTL only after the existing guarded keepalive typing send succeeds is the narrowest maintainable fix for this Discord liveness bug and avoids changing default progress-message semantics or broad presence architecture.
