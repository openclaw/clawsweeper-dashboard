## What This Changes

Request native audio-file handling in the read tool so supported models receive audio attachments instead of raw binary text.

## Best Possible Solution

Define a shared typed audio attachment contract across read, prompt refs, gateway/media understanding, and provider runtime with MIME sniffing, workspace/sandbox guards, capability gates, raw-media egress policy, docs/tests, and live provider proof.

I did not run a live provider in this read-only review because the missing contract is visible in source and docs.

## Solution Assessment

No, not as a simple extension-list patch to read. The maintainable solution needs a typed audio media contract, provider gates, raw-media egress policy, payload limits, docs, and real provider proof.
