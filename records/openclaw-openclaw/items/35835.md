## Risks / Open Questions

- Raw audio forwarding changes data egress from derived transcript text to original local or user audio bytes, so maintainers need explicit opt-in, retention/audit, payload-size, and provider-boundary policy.

- Unsupported-provider fallback, model capability detection, and auth/profile routing need real provider proof before users rely on this in sessions.
