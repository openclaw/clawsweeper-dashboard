## Summary

Keep open: current main and the latest stable release still have the reported undocumented cron edit behavior, and the conservative fix is small help/docs/test cleanup rather than removing compatibility.

## What This Changes

The issue asks to document or change the cron edit behavior where toggling best-effort delivery without an explicit delivery mode also patches delivery mode to announce.

## Best Possible Solution

Preserve the shipped compatibility behavior, update the CLI help and public cron CLI docs to state that best-effort delivery edits imply announce mode when no delivery mode flag is supplied, and add a focused regression test for the standalone flag case.

## Reproduction Assessment

Yes, from source inspection: `openclaw cron edit <id> --best-effort-deliver` makes `opts.bestEffortDeliver` boolean, which enters the `hasBestEffort` branch and patches `delivery.mode` to `announce`. I did not execute the CLI because this review was read-only.

## Solution Assessment

Yes, documenting and testing the compatibility behavior is the best narrow fix. Removing the branch or requiring `--announce` would be a compatibility decision rather than a conservative cleanup.
