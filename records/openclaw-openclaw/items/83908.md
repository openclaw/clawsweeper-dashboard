## Summary

Keep open: current main and v2026.5.18 still have the reported cron edit behavior and the help/docs still do not explain the implied announce-mode patch. The conservative repair is to preserve compatibility, clarify the CLI/docs wording, and add a focused regression test.

## What This Changes

The issue asks to document or remove the cron edit compatibility behavior where `--best-effort-deliver` by itself patches `delivery.mode` to `announce`.

## Best Possible Solution

Preserve the shipped compatibility behavior, update CLI help and public cron CLI docs to state that standalone best-effort edits imply announce mode, and add a focused regression test for the standalone flag case.

## Reproduction Assessment

Yes. Source inspection shows `openclaw cron edit <id> --best-effort-deliver` makes `opts.bestEffortDeliver` boolean, enters the `hasBestEffort` branch, and patches `delivery.mode` to `announce`; I did not execute the CLI because this review is read-only.

## Solution Assessment

Yes. Documenting and testing the compatibility behavior is the narrowest maintainable fix; removing the branch or requiring `--announce` would be a separate compatibility decision.
