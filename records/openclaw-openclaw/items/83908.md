## What This Changes

Clarify the standalone `openclaw cron edit --best-effort-deliver` behavior because it currently implies `delivery.mode = "announce"` without CLI help or docs explaining that effect.

## Best Possible Solution

Preserve the shipped compatibility behavior, document the implication in CLI help and public cron docs, and add a focused regression test for the standalone best-effort edit case.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: `openclaw cron edit <id> --best-effort-deliver` makes `opts.bestEffortDeliver` boolean and enters the branch that patches `delivery.mode` to `announce`. I did not execute the CLI because this was a read-only review.

## Solution Assessment

Yes, documenting and testing the compatibility behavior is the narrowest maintainable fix. Removing the branch or requiring `--announce` would be a separate compatibility decision.
