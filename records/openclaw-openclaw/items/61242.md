Overall confidence: 0.9

Full review comments:

- **[P1] Reopen flows before linking retry tasks:** `src/tasks/task-executor.ts:569-570`
  - body: This retry branch allows `failed` and `lost` flows, but it passes `parentFlowId` into `spawnSubagentDirect` before the flow is moved out of its terminal state. The task registry rejects linking a new task to a terminal parent flow, so failed/lost subagent retries can fail before the relaunch is tracked.
  - confidence: 0.92
- **[P1] Let cancellation win before completion sync:** `src/tasks/task-registry.ts:1121-1130`
  - body: `syncManagedFlowCompletionFromTask` runs before cancellation sync. If a user has requested cancellation and the child then exits, completion can mark the flow `succeeded`/`failed`/`lost` first; the cancellation path then returns because the flow is terminal, so the requested cancel is lost.
  - confidence: 0.9
- **[P2] Require wait identifiers to match terminal tasks:** `src/tasks/task-registry.ts:815-825`
  - body: When `waitJson` names a `runId` or `childSessionKey`, a terminal task that omits that field still matches because the checks only reject mismatches when both sides provide the identifier. A later linked task without the identifier can therefore close the flow with the wrong child outcome; require the task to contain and match each identifier present in the wait state.
  - confidence: 0.87
- **[P2] Base retry guidance on retry eligibility:** `src/tasks/task-flow-guidance.ts:41-47`
  - body: This labels every failed/lost managed flow as `retry_available`, but the PR also records non-retryable launches such as attachment-backed spawns. That produces contradictory UI/CLI output where guidance says retry is available while the retry object says it is not; derive this guidance from the same eligibility check.
  - confidence: 0.86
- **[P3] Remove localized docs edits from this repo:** `docs/zh-CN/tools/subagents.md:65-76`
  - body: The scoped docs guide says foreign-language docs under `docs/<locale>/**` are not maintained here. Please drop this `docs/zh-CN` edit and let the publish/i18n pipeline handle localized output from the English source docs.
  - confidence: 0.95
