  - confidence: 0.88

  - body: `updateTask` finalizes managed completion before cancellation, and the completion path does not skip flows with `cancelRequestedAt`. If a user cancels a managed flow and a child exits afterward, the child outcome can win and prevent cancellation from being reflected.
  - confidence: 0.9
- **[P2] Require wait identifiers when matching child completions:** `src/tasks/task-registry.ts:815-825`
  - body: `matchesManagedChildWaitState` only rejects mismatches when both sides provide an identifier. A flow waiting on a specific `runId` or `childSessionKey` can still be closed by a different linked terminal task that omits that field.
  - confidence: 0.88
- **[P2] Base retry guidance on actual eligibility:** `src/tasks/task-flow-guidance.ts:41-47`
  - body: `getTaskFlowGuidance` reports `retry_available` for every failed or lost managed flow, but the retry resolver can reject missing launch metadata, active children, cancellation, or attachment-backed launches. Use the same eligibility logic for guidance.
  - confidence: 0.9
- **[P2] Discard stale flow loads after session switches:** `ui/src/ui/controllers/task-flows.ts:23-33`
  - body: `loadLatestTaskFlow` returns while a load is in flight and writes the awaited response using mutable state. If the user switches sessions during that request, the old flow can populate the new chat and action buttons can target stale state.
  - confidence: 0.87
