  - body: This file now imports from `@sinclair/typebox`, but the repo depends on `typebox` and current main imports `Type` from `typebox`. With pnpm's isolated dependency layout, this changed source will fail module resolution before the TaskFlow path can run.

  - body: The ACP branch now calls `spawnAcpDirect` without the current-main checks for unsupported inherited tool deny/allow policy. That lets an ACP child spawn proceed even when the parent session's inherited tool restrictions should forbid it, so the taskFlow work needs to keep those guards.
  - confidence: 0.91
- **[P2] Keep current sessions_spawn controls in the schema:** `src/agents/tools/sessions-spawn-tool.ts:68-83`
  - body: The static schema on this branch drops current-main fields and gating such as `taskName`, `context`, `lightContext`, dynamic ACP availability, and thread-aware mode restrictions. Merging this would silently remove accepted `sessions_spawn` controls while adding `taskFlow`; add the new parameter on top of the current schema instead.
  - confidence: 0.9
- **[P1] Reopen terminal flows before retry spawns:** `src/tasks/task-executor.ts:569-574`
  - body: The retry resolver allows failed/lost flows, but this branch passes `parentFlowId` into a new child spawn while the parent flow is still terminal. The task registry rejects linked tasks for terminal parent flows, so retries can start or attempt child work that cannot be attached back to the managed flow.
  - confidence: 0.92
- **[P1] Pass requester origin into subagent retries:** `src/tasks/task-executor.ts:572-574`
  - body: The subagent retry path reconstructs context with only `agentSessionKey`, while `spawnSubagentDirect` uses requester-origin fields for thread and channel binding. Stored launches from a threaded/channel-bound flow can lose their target on retry even though the flow is otherwise eligible.
  - confidence: 0.88
- **[P1] Give cancellation precedence over completion sync:** `src/tasks/task-registry.ts:1121-1130`
  - body: `updateTask` finalizes managed completion before cancellation, and the completion path does not skip flows with `cancelRequestedAt`. If a user cancels a managed flow and a child exits afterward, the child outcome can win and prevent the flow from ever showing cancellation.
  - confidence: 0.9
- **[P2] Require wait identifiers when matching child completions:** `src/tasks/task-registry.ts:815-825`
  - body: `matchesManagedChildWaitState` only rejects mismatches when both sides provide the identifier. A flow waiting on a specific `runId` or `childSessionKey` can still be closed by a different linked terminal task that omits that field; require identifiers present in `waitJson` to be present and equal on the task.
  - confidence: 0.87
- **[P2] Base retry guidance on actual retry eligibility:** `src/tasks/task-flow-guidance.ts:41-47`
  - body: `getTaskFlowGuidance` reports `retry_available` for every failed or lost managed flow, but the retry resolver can reject missing launch metadata, active children, cancellation, or attachment-backed launches. Use the same eligibility logic for the guidance to avoid contradictory UX.
  - confidence: 0.9
- **[P2] Discard stale flow loads after session switches:** `ui/src/ui/controllers/task-flows.ts:23-33`
  - body: `loadLatestTaskFlow` returns while a load is in flight and writes the awaited response using mutable state. If the user switches sessions during that request, the old flow can populate the new chat and the action buttons can target stale state.
  - confidence: 0.87
- **[P3] Remove localized docs edits from this repo:** `docs/zh-CN/tools/subagents.md:65-76`
  - body: The scoped docs guide says localized docs under `docs/<locale>/**` are not maintained here. This branch edits `docs/zh-CN/tools/subagents.md`, which will drift from the publish-repo i18n pipeline; keep the English source change and glossary/update path instead.
  - confidence: 0.82
