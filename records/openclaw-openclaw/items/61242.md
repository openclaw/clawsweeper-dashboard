  - confidence: 0.93
- **[P2] Keep the current sessions_spawn contract:** `src/agents/tools/sessions-spawn-tool.ts:68-83`
  - body: The static schema drops current-main fields and gating such as `taskName`, `context`, `lightContext`, dynamic ACP availability, and thread-aware mode restrictions. Add `taskFlow` on top of the current schema instead of replacing the contract.
  - confidence: 0.92
- **[P1] Reopen terminal flows before retrying children:** `src/tasks/task-executor.ts:552-574`
  - body: The retry path permits failed/lost flows, but it passes `parentFlowId` into the new child spawn while the parent flow is still terminal. Current task linking rejects terminal parent flows, so retries can fail or detach before the flow is reopened.
  - confidence: 0.92
- **[P1] Forward requester origin on subagent retries:** `src/tasks/task-executor.ts:572-574`
  - body: The subagent retry path reconstructs context with only `agentSessionKey`, while `spawnSubagentDirect` uses requester-origin fields for thread and channel binding. Threaded or channel-bound retries can lose their target.
  - confidence: 0.88
- **[P1] Apply cancellation before completion closeout:** `src/tasks/task-registry.ts:1120-1130`
  - body: `updateTask` finalizes managed completion before cancellation, and the completion path does not skip flows with `cancelRequestedAt`. If a user cancels a managed flow and the child exits afterward, the child outcome can win.
  - confidence: 0.9
- **[P2] Require wait identifiers to match:** `src/tasks/task-registry.ts:815-825`
  - body: `matchesManagedChildWaitState` only rejects mismatches when both sides provide an identifier. A flow waiting on a specific `runId` or `childSessionKey` can still be closed by a linked terminal task that omits that field.
  - confidence: 0.88
- **[P2] Base guidance on retry eligibility:** `src/tasks/task-flow-guidance.ts:41-47`
  - body: `getTaskFlowGuidance` reports `retry_available` for every failed or lost managed flow, but retry can reject missing launch metadata, active children, cancellation, or attachment-backed launches. Use the same eligibility logic for guidance.
  - confidence: 0.9
- **[P2] Discard stale TaskFlow loads:** `ui/src/ui/controllers/task-flows.ts:23-33`
  - body: `loadLatestTaskFlow` returns while a load is in flight and writes the awaited response using mutable state. If the user switches sessions during that request, the old flow can populate the new chat and action buttons can target stale state.
  - confidence: 0.87
- **[P3] Remove generated localized docs edits:** `docs/zh-CN/tools/subagents.md:62-76`

  - confidence: 0.82
