## Summary

Keep open. The underlying Copilot multi-profile cache bug is still source-reproducible on current main, but this stale PR patches copied helper bodies from an old base instead of the current canonical provider-auth resolver and still lacks real behavior proof.

## What This Changes

This PR adds a GitHub-token fingerprint to Copilot token cache entries and expands token cache tests for same-profile reuse, cross-profile cache misses, and legacy cache entries.

## Best Possible Solution

Port the fingerprint field, cache-hit guard, and cache write to src/plugin-sdk/provider-auth.ts, keep wrapper files as re-exports, preserve integrationId/Copilot-Integration-Id handling, update focused regression tests, and attach redacted real rotation proof.

## Reproduction Assessment

Yes for source-level reproduction: a valid shared cache entry in src/plugin-sdk/provider-auth.ts returns before the rotated params.githubToken is used, while the provider passes the selected ctx.apiKey into that helper. No live quota run is attached.

## Solution Assessment

No, not as currently patched. The fingerprint check is the right narrow direction, but it needs to patch the current canonical provider-auth resolver and preserve the existing Copilot integration identity behavior.
