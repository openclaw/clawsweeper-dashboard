## Summary

Keep open: the reported Copilot profile-rotation bug is valid, but this PR should not merge as-is because current main centralizes the token cache helper in `src/plugin-sdk/provider-auth.ts` while the PR patches stale wrapper-local copies and still lacks real behavior proof.

## What This Changes

The PR adds a GitHub-token fingerprint to Copilot token cache entries in Copilot helper files and expands token cache tests for same-profile reuse, cross-profile misses, and legacy cache entries.

## Best Possible Solution

Move the fingerprint field, cache-hit guard, and cache write into `src/plugin-sdk/provider-auth.ts`, keep wrapper files as re-exports, update focused regressions, and attach redacted real multi-profile Copilot rotation proof.

## Reproduction Assessment

Yes for source-level reproduction, but not a live quota test. Current main returns a usable shared cache entry before comparing the selected GitHub PAT, while the Copilot provider passes the rotated `ctx.apiKey` into that helper.

## Solution Assessment

No, not as currently patched. The fingerprint check is the right narrow fix, but it needs to patch the canonical provider-auth resolver instead of stale wrapper-local copies.
