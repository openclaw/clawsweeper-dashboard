## Summary

Keep open. The underlying Copilot profile-rotation bug is still source-reproducible on current main, but this PR is not merge-ready because it patches stale wrapper-local token helpers instead of the active shared resolver and still lacks real multi-profile behavior proof.

## What This Changes

The PR adds a GitHub PAT fingerprint to cached Copilot API tokens in Copilot token helper files and updates token tests for same-profile reuse, cross-profile misses, and legacy unowned cache entries.

## Best Possible Solution

Port the fingerprint owner check into src/plugin-sdk/provider-auth.ts, keep the wrapper files as re-exports, preserve integrationId behavior, add focused regressions, and include redacted real multi-profile proof.

## Reproduction Assessment

Yes. Source inspection of current main shows resolveCopilotApiToken returns a usable shared cache entry before using params.githubToken, while Copilot auth-profile rotation passes the new ctx.apiKey into that helper; I did not run a live quota test.

## Solution Assessment

No, not as currently patched. The fingerprint check is the right narrow fix, but it needs to live in the active provider-auth resolver while the wrapper files remain re-exports.
