## Summary

Keep open: the underlying Copilot profile-rotation bug is still present on current main, but this PR is not merge-ready because it is dirty against main and applies the cache guard to wrapper-local implementations instead of the shared provider-auth helper current callers use.

## What This Changes

The PR adds a GitHub-token fingerprint to cached Copilot token entries in GitHub Copilot token helpers and adds regression coverage for same-profile hits, cross-profile misses, and legacy cache misses.

## Best Possible Solution

Port the cache ownership check into src/plugin-sdk/provider-auth.ts, keep the wrapper files as re-exports, add regression coverage for same-profile hits, different-profile misses, and legacy unowned-cache misses, and add the required changelog entry.

## Reproduction Assessment

Yes, by source inspection. A usable cached Copilot token returned from loadJsonFileImpl is accepted before resolveCopilotApiToken uses a different githubToken for exchange, so a rotated profile can receive the previous profile's token.

## Solution Assessment

No as-is. The fingerprint guard is a plausible narrow fix, but this branch patches obsolete wrapper-local implementations and is merge-conflicted; the current implementation point is the shared provider-auth helper.
