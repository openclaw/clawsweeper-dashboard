## Summary

Keep open. The bug is still source-reproducible on current main, but this PR is not merge-ready because it patches stale Copilot token resolver copies instead of the current shared provider-auth helper and still lacks real multi-profile proof.

## What This Changes

The PR adds a SHA-256 GitHub PAT fingerprint to cached GitHub Copilot API tokens in plugin/agent helper copies and extends token cache tests for same-profile, cross-profile, and legacy-cache behavior.

## Best Possible Solution

Move the owner-aware cache check into src/plugin-sdk/provider-auth.ts, keep the wrapper exports, preserve Copilot-Integration-Id/integrationId behavior, add focused regressions, and require redacted live multi-profile proof before merge.

## Reproduction Assessment

Yes. Source inspection on current main shows resolveCopilotApiToken returns a valid shared cached token before using a different params.githubToken from a rotated profile; I did not run a live multi-profile Copilot setup in this read-only review.

## Solution Assessment

No, not as currently patched. The fingerprint approach is the right narrow fix, but it needs to be applied to the shared provider-auth helper while preserving the current vscode-chat integration identity contract.
