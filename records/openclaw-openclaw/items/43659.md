## Summary

Keep open. The reported Copilot profile-rotation bug is still real on current main, and this PR is useful source work, but the current diff is not merge-ready because it patches stale wrapper-local token helpers while current main routes the active and public implementation through `src/plugin-sdk/provider-auth.ts`; the PR head is also dirty against main and needs explicit review of the credential-fingerprint cache design.

## What This Changes

The PR adds a SHA-256-derived GitHub PAT fingerprint to cached GitHub Copilot token entries in the plugin and legacy helper, requires matching fingerprints for cache hits, and adds regression tests for cross-profile and legacy-cache misses.

## Best Possible Solution

Apply the cache-ownership check in `src/plugin-sdk/provider-auth.ts`, keep `extensions/github-copilot/token.ts` and `src/agents/github-copilot-token.ts` as thin re-exports, choose an accepted cache owner key design, add central regression coverage for same-profile hits, different-profile misses, and legacy unowned-cache misses, and include the required changelog entry before closing the linked issue.

## Reproduction Assessment

Yes. On current main, a test or focused helper call can inject a valid `CachedCopilotToken` through `loadJsonFileImpl` and then call `resolveCopilotApiToken` with a different `githubToken`; the helper returns the cached Copilot token before using the new GitHub token for a fetch.

## Solution Assessment

No as-is. The fingerprint idea can solve the observable rotation bug, but this PR applies it to stale local implementations instead of the shared public helper current main exports and uses; the safer path is a central provider-auth fix with wrapper re-exports preserved.
