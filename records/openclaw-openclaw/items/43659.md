## Summary

Keep open: the linked Copilot profile-rotation bug still reproduces in current main and latest release, but this PR is not merge-ready because it patches older token helper paths while the active runtime now uses `src/plugin-sdk/provider-auth.ts`; it also lacks real behavior proof.

## What This Changes

The PR adds a GitHub-token fingerprint to the Copilot API token cache and tests cache misses for profile switches and legacy entries, but applies the changes to older token helper files.

## Best Possible Solution

Refresh the branch so the cache owner check and regression tests live in `src/plugin-sdk/provider-auth.ts` and its tests, keep the extension as a re-export, then require redacted real two-profile proof before merge.

## Reproduction Assessment

Yes at source level: current main returns a usable shared Copilot token cache entry before using the newly selected `params.githubToken`, so two profiles can receive the same cached Copilot API token. I did not run a live two-account Copilot repro in this read-only review.

## Solution Assessment

No. The credential-fingerprint approach is likely the right narrow fix, but this branch must be ported to the current provider-auth helper before it is a mergeable solution.
