## Summary

Keep open: the reported auth-rotation bug is valid, but this PR is stale against current main because it patches wrapper-local Copilot token implementations while current main uses the shared `provider-auth` helper.

## What This Changes

The PR adds a GitHub-token fingerprint to Copilot token cache entries in the Copilot token wrappers and extends GitHub Copilot token tests for same-profile, cross-profile, and legacy cache cases.

## Best Possible Solution

Port the cache ownership check into `src/plugin-sdk/provider-auth.ts`, keep the wrapper files as re-exports, add focused cache-ownership regression coverage, and add the required `CHANGELOG.md` fix entry.

## Reproduction Assessment

Yes, by source inspection. Current main returns a usable cached Copilot API token based only on expiry while the runtime passes the selected auth profile credential into that helper.

## Solution Assessment

No as-is. The fingerprint guard is a narrow fix direction, but it needs to live in the current shared `provider-auth` implementation and the persistent fingerprint design should be kept explicit.
