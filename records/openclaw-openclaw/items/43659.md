## Summary

Keep open. The bug remains source-reproducible on current main, and the PR's fingerprint approach is the right narrow direction, but the branch is conflicting and applies the change to stale helper copies instead of the current shared provider-auth implementation.

## What This Changes

The PR adds a GitHub PAT fingerprint to cached GitHub Copilot tokens in plugin/agent token helpers and extends token-cache tests for same-profile, cross-profile, and legacy-cache behavior.

## Best Possible Solution

Apply the fingerprint ownership check in src/plugin-sdk/provider-auth.ts, preserve integrationId/Copilot-Integration-Id behavior and wrapper entrypoints, add regressions, then land with redacted live multi-profile proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a valid provider-auth cache entry is returned before params.githubToken is used, so a rotated profile can keep receiving the previous account's Copilot API token.

## Solution Assessment

No, not as currently patched. The fingerprint approach is appropriate, but it needs to live in the shared provider-auth helper instead of reintroducing divergent token resolver copies.
