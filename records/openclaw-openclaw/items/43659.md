## Summary

Keep open. The underlying Copilot profile-rotation bug is still present on current main, but this PR is not ready because its diff targets stale wrapper-local token helpers instead of the active shared provider-auth helper, and the external contributor has not supplied real multi-profile proof.

## What This Changes

The PR adds SHA-256 GitHub-token fingerprints to Copilot token cache entries in the plugin wrapper and legacy agent helper, plus regression tests for same-profile reuse, profile mismatch misses, and legacy misses.

## Best Possible Solution

Move the fingerprint owner field, cache-hit check, and cache write into src/plugin-sdk/provider-auth.ts, keep both wrappers as re-exports, preserve integrationId behavior, and land with focused regression tests plus redacted live multi-profile proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a valid shared Copilot cache entry is returned before resolveCopilotApiToken consults the rotated params.githubToken, while the runtime passes the selected ctx.apiKey into that helper.

## Solution Assessment

No, not as currently patched. The fingerprint approach is the right narrow fix, but it needs to be applied in src/plugin-sdk/provider-auth.ts where current main owns the active helper, not in stale wrapper-local resolver copies.
