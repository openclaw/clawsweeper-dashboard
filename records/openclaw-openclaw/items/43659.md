## Summary

Keep open. The linked bug is still source-reproducible on current main, but this PR is not mergeable as-is because it patches reintroduced local token resolver copies instead of the active shared provider-auth helper and still lacks real multi-profile proof.

## What This Changes

The PR adds a GitHub-token fingerprint to cached Copilot tokens in plugin and legacy agent token helpers and extends token-cache tests for cross-profile and legacy-cache behavior.

## Best Possible Solution

Move the fingerprint ownership check into src/plugin-sdk/provider-auth.ts, keep the wrapper entrypoints and integrationId headers, add focused regressions, and land with redacted live multi-profile proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a valid cached Copilot API token is returned before a rotated profile's githubToken is used for exchange.

## Solution Assessment

No, not as currently patched. The fingerprint approach is appropriate, but it needs to be applied in the shared provider-auth helper rather than by reintroducing local resolver copies under wrapper files.
