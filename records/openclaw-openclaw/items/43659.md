## Summary

Keep open. The linked Copilot auth-rotation bug is still source-reproducible on current main, but this PR is not merge-ready because current main moved the active token exchange helper into the shared provider-auth SDK surface while the branch still patches older wrapper-local implementations.

## What This Changes

The PR adds a GitHub PAT fingerprint to cached Copilot token entries, requires fingerprint matches for cache hits, and adds regression coverage for cross-profile and legacy-cache misses.

## Best Possible Solution

Apply the cache ownership check in `src/plugin-sdk/provider-auth.ts`, keep the wrapper files as thin re-exports, add central regression tests for same-profile hits, different-profile misses, and legacy unowned-cache misses, and include the required changelog entry.

## Reproduction Assessment

Yes, by source inspection. A focused helper call can return a valid `CachedCopilotToken` from `loadJsonFileImpl`, then call `resolveCopilotApiToken` with a different `githubToken`; current main returns the cached Copilot token before the new GitHub token affects a fetch.

## Solution Assessment

No as-is. The fingerprint approach can address the observable bug, but this PR no longer patches the active shared helper on current main and the credential-derived owner key needs an accepted cache design.
