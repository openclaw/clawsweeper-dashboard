## Summary

Keep open: the underlying Copilot profile-rotation bug is still present on current main, and this PR targets the right problem, but the branch is stale against the active implementation and lacks required real behavior proof.

## What This Changes

The PR adds a GitHub-token fingerprint to Copilot token cache entries and extends GitHub Copilot token tests for same-profile cache hits, cross-profile misses, and legacy unowned cache entries.

## Best Possible Solution

Implement the ownership check in `src/plugin-sdk/provider-auth.ts`, preserve the existing `integrationId` guard, keep wrappers as re-exports, and add regression coverage for same-profile, different-profile, and legacy cache entries.

## Reproduction Assessment

Yes, by source inspection. A valid cached Copilot token is returned before the selected profile's `githubToken` is used, while the GitHub Copilot runtime passes the selected `ctx.apiKey` into that helper.

## Solution Assessment

No as-is. A fingerprint or profile-safe cache key is a narrow maintainable fix, but this PR needs to move the guard into the current shared helper and preserve the current `vscode-chat` cache identity behavior.
