## Summary

Keep open. The PR targets a real GitHub Copilot auth-profile rotation bug, but the branch is stale against current main: main now routes both Copilot token wrappers through `src/plugin-sdk/provider-auth.ts`, while this PR patches wrapper-local implementations and has no changelog entry.

## What This Changes

The PR adds a SHA-256 GitHub-token fingerprint to Copilot token cache entries, requires matching fingerprints for cache hits, and adds token-cache regression tests.

## Best Possible Solution

Implement cache ownership in `src/plugin-sdk/provider-auth.ts`, keep the wrapper files as re-exports, cover same-profile, different-profile, and legacy-cache cases, and add the required changelog entry.

## Reproduction Assessment

Yes, by source inspection on current main. A usable cached Copilot token without ownership metadata is returned before the selected `githubToken` is used, and the runtime passes the selected auth profile credential into that helper.

## Solution Assessment

No as-is. The fingerprint guard is a narrow plausible fix, but it needs to be ported to the active shared helper on current main and paired with changelog/regression coverage.
