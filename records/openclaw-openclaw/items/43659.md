## Summary

Keep open. The reported Copilot profile-rotation bug is still present on current main, but this PR is stale against the current implementation because the active token helper has moved into the shared provider-auth module, and the external contributor has not provided real behavior proof.

## What This Changes

The PR adds a GitHub-token fingerprint to cached Copilot API tokens so profile changes reject stale cached tokens and fetch a token for the selected profile.

## Best Possible Solution

Port the cache-owner check into `src/plugin-sdk/provider-auth.ts`, keep the wrapper files as re-exports, add regression coverage for same-profile hits, cross-profile misses, and legacy unbound cache misses, then require redacted real multi-profile runtime proof before merge.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: with a valid shared cached Copilot token, current `resolveCopilotApiToken` returns it before using a different `githubToken` from a rotated profile.

## Solution Assessment

No. The fingerprint idea is a narrow maintainable fix, but this branch is no longer the best patch because the active implementation now lives in `src/plugin-sdk/provider-auth.ts`, not the files changed by the PR.
