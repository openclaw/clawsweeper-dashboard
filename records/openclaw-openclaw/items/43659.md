## Summary

Keep open: the underlying Copilot profile-rotation bug is still present on current main, but this PR is stale against the current shared provider-auth implementation and needs to be ported before it can land.

## What This Changes

The PR adds a GitHub PAT fingerprint to the Copilot token cache in the GitHub Copilot plugin and legacy agent helpers, plus regression tests for same-profile hits, profile-mismatch misses, saved fingerprints, and legacy cache misses.

## Best Possible Solution

Land a rebased fix in `src/plugin-sdk/provider-auth.ts` that binds cached Copilot API tokens to the GitHub token that produced them, preserves the wrapper modules, adds focused regression coverage, and records the user-facing fix in the changelog.

## Reproduction Assessment

Yes, from source inspection: current main’s `resolveCopilotApiToken` returns a still-usable cached token without considering the current GitHub auth profile. I did not run a live repro or tests because this review is read-only and `pnpm` is unavailable.

## Solution Assessment

No, not as-is. The fingerprint approach is the narrow maintainable fix, but it needs to be applied to the current shared provider-auth resolver rather than the stale implementation paths shown by this PR.
