## What This Changes

The branch adds a GitHub PAT fingerprint to Copilot token cache records in the Copilot runtime and agent helpers, with regression tests for profile-switch and legacy-cache misses.

## Best Possible Solution

Port the owner-aware cache check into src/plugin-sdk/provider-auth.ts, re-export any needed helper through the wrapper modules, and keep focused regression coverage for same-profile hits, cross-profile misses, and legacy cache entries.

## Reproduction Assessment

Yes. Source inspection on current main shows a valid shared Copilot API token cache entry is returned before the selected GitHub PAT is used, so rotating profiles can keep using the previous account's Copilot token.

## Solution Assessment

No, not as currently patched. The fingerprint approach is the narrow maintainable fix, but it needs to be applied to the current shared provider-auth helper rather than stale per-path implementations.
