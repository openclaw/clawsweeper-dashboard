## Summary

Keep open: the reported cache collision is still present on current main, and this PR contains the right general fix direction, but it now needs to be ported to the current shared Copilot token helper before it can merge.

## What This Changes

The PR binds cached GitHub Copilot API tokens to a fingerprint of the GitHub PAT and adds regression tests for profile switches and legacy cache entries.

## Best Possible Solution

Put the fingerprint-bound cache ownership in `src/plugin-sdk/provider-auth.ts`, keep both wrapper modules as re-exports, add focused regression coverage, and include the required changelog entry.

## Reproduction Assessment

Yes, from source inspection: a valid `github-copilot.token.json` entry is accepted solely by expiry, so a later call with a different GitHub PAT still receives the previous Copilot API token. I did not run a failing test in this read-only review.

## Solution Assessment

No. The fingerprint approach is the right narrow fix, but the branch currently targets wrapper/pre-refactor locations instead of the shared `src/plugin-sdk/provider-auth.ts` implementation on current main.
