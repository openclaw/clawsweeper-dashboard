## Summary

Keep open for human review: the underlying GitHub Copilot auth-profile rotation bug is still real on current main and in the latest release, but this PR is not merge-ready because it is conflicting, reintroduces wrapper-local token resolvers instead of patching the canonical provider-auth helper, and still has only test/CI proof.

## What This Changes

The PR adds a GitHub-token fingerprint to GitHub Copilot token cache entries and regression tests so profile rotation bypasses a shared cached Copilot API token.

## Best Possible Solution

Port the fingerprint field, cache-hit guard, and cache write into src/plugin-sdk/provider-auth.ts, keep both wrappers as re-exports, preserve Copilot integration identity behavior, add focused regression coverage, and attach redacted real rotation proof.

## Reproduction Assessment

Yes, source-level: current main can return a usable shared cached Copilot token before using the rotated params.githubToken, while docs say auto-pinned profiles may rotate on rate limits/timeouts. I did not run a live multi-account quota scenario.

## Solution Assessment

No, not as currently patched. Fingerprinting the selected GitHub token is the right narrow fix, but the implementation should live in the canonical provider-auth resolver and preserve the current Copilot integration identity contract.
