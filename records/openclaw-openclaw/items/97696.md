## What This Changes

The PR replaces Anthropic and Gemini native PDF success-response `res.json()` parsing with the shared bounded provider JSON reader and adds local HTTP-server coverage for under-cap and oversized responses.

## Best Possible Solution

Land the shared bounded-reader fix after redacted real behavior proof or maintainer proof override, with an explicit maintainer decision to keep or tune the 16 MiB native PDF success-response cap.

## Reproduction Assessment

Yes. Current main has two successful native PDF provider paths that call unbounded `res.json()`, and the PR's local HTTP-server tests show an oversized-body path that would cross the new cap.

## Solution Assessment

Yes, with merge conditions. Reusing `readProviderJsonResponse` is the narrowest maintainable fix because it uses the existing shared cap/cancel helper, but the cap policy and external-PR proof gate still need maintainer-visible closure.
