## Best Possible Solution

Define one maintainer-approved dotenv expansion policy for trusted global, untrusted workspace, legacy gateway, config, and service-env paths, then implement it with coverage for precedence, blocked keys, provider credentials, unresolved refs, and upgrade behavior.
