## Summary

Keep open. Current main still assigns parsed dotenv values literally, and the companion fix PR was closed without merging; the remaining question is the product/security contract for adding expansion to credential-loading paths.

## What This Changes

Request that OpenClaw expand `${VAR}` and `$VAR` references when loading project and global `.env` files while preserving existing environment precedence.

## Best Possible Solution

Define a bounded dotenv-expansion contract in `src/infra/dotenv.ts` that runs after key filtering, preserves process-env precedence, and covers trusted global, untrusted workspace, same-file, chained, missing-var, and blocked-key cases.

## Reproduction Assessment

Yes, source-reproducible: current main parses `.env` files and assigns parsed values without an expansion step, so `${OPENAI_API_KEY}` remains literal. I did not run a temp-file repro because this review was read-only and dependencies are not installed.

## Solution Assessment

Unclear. The requested expansion is plausible, but maintainers should first choose the trusted-global versus untrusted-workspace expansion rules; a narrow loader contract is safer than provider-specific alias handling.
