## Summary

Keep open. Current main still parses `.env` files with `dotenv.parse` and assigns accepted values directly to `process.env`, so `$VAR` / `${VAR}` aliases remain literal. The direct implementation PR #38254 is closed unmerged and useful as prior art, but no current-main or shipped fix covers the requested behavior.

## What This Changes

Expand `$VAR` and `${VAR}` references inside OpenClaw-loaded project and global `.env` files while preserving existing environment precedence.

## Best Possible Solution

Land a narrowly scoped dotenv expansion implementation that runs only after key normalization and workspace/global filtering, expands from process env plus earlier accepted dotenv entries without overriding existing keys, preserves current CWD/global/gateway fallback precedence and warnings, documents the behavior, and adds regression tests for aliases, chaining, missing/cyclic references, escaped dollars, and blocked-key interactions.
