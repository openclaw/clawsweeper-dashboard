## What This Changes

The PR changes stuck diagnostic session recovery to release orphaned per-session lanes unless non-session lane backpressure can explain the active session task, with focused runtime and integration tests.

## Best Possible Solution

Scope shared-lane backpressure to the specific lane that can block this session, add a regression for unrelated non-session lane work, and attach redacted real gateway proof before merge.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence repro: with a stale session lane active and any unrelated non-session lane snapshot active or queued, the PR helper returns shared_lane_backpressure and skips the stale lane release.

## Solution Assessment

No as written. The right fix should distinguish the global/shared lane that can actually block this session from unrelated non-session lanes, then prove both stale release and true shared-lane backpressure.
