## Summary

Keep open: the core-to-plugin direction has useful support from the original oc-path author and the PR includes terminal behavior proof, but the current head still has merge-blocking default CLI availability and documentation drift.

## What This Changes

The PR removes the core oc-path implementation and reintroduces `openclaw path` as a bundled `oc-path` plugin with local parser dependencies, tests, docs, labeler, and lockfile updates.

## Best Possible Solution

Land the plugin-owned oc-path direction only after maintainers confirm it, then preserve default `openclaw path` availability and align docs, changelog, generated plugin inventory, and validation evidence with the final md/jsonc/jsonl grammar.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: the PR removes the core `path` registrar, the new bundled manifest lacks `enabledByDefault`, and current plugin loading disables and skips such bundled plugins before CLI registration.

## Solution Assessment

No. Moving oc-path into a bundled plugin is plausible, but this patch is not the best mergeable shape until it preserves fresh default command availability and removes docs, changelog, and generated inventory drift.
