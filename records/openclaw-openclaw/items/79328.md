## Summary

Keep open: this active replacement PR has useful terminal proof and the original oc-path author supports the plugin direction, but current head still appears to drop default `openclaw path` availability and leaves docs/inventory out of sync with the final plugin contract.

## What This Changes

The PR removes the core oc-path implementation and core `openclaw path` wiring, then reintroduces the command as a bundled `extensions/oc-path` plugin with plugin-local parser dependencies, reduced md/jsonc/jsonl support, docs, labeler, and lockfile updates.

## Best Possible Solution

Keep the plugin-owned direction only if a fresh default install still gets `openclaw path`, the docs match the reduced md/jsonc/jsonl grammar, and generated plugin inventory/reference output includes `oc-path`.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: run `openclaw path` on the PR branch from a fresh default config after core path registration is removed, while the bundled plugin lacks default enablement and the loader skips disabled plugins; the docs mismatch is directly visible from PR head.

## Solution Assessment

No. The core-to-plugin move may be acceptable, especially with the original oc-path author agreeing, but this patch should first preserve default command availability, align docs with the actual grammar, refresh generated inventory, and resolve the branch conflict.
