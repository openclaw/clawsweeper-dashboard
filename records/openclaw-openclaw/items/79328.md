## Summary

Keep open: this maintainer-labeled PR still has blocking source-proven issues around fresh default `openclaw path` availability and stale path grammar docs, though the submitted terminal proof is sufficient for the manually enabled plugin path.

## What This Changes

The PR removes the core oc-path substrate and core `openclaw path` wiring, then reintroduces the CLI as a bundled `oc-path` plugin with plugin-local parser dependencies, migrated tests, docs, and labeler ownership.

## Best Possible Solution

Land the plugin-owned oc-path direction only after maintainers confirm the default-vs-opt-in CLI policy, preserve the intended command availability, and align docs, changelog, generated plugin inventory, and validation evidence with the final md/jsonc/jsonl grammar.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: the PR removes the core `path` registrar, the bundled manifest lacks `enabledByDefault`, and current plugin CLI loading skips disabled bundled plugins before registration.

## Solution Assessment

No. Moving oc-path into a bundled plugin is plausible and was supported in the related discussion, but this patch is not the best mergeable shape until default availability policy and docs/changelog/inventory drift are resolved.
