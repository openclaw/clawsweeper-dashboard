## Summary

Keep open: the bundled-plugin direction is supported by the current plugin CLI contract and sufficient real behavior proof, but current head still has two source-reproducible correctness/contract issues before merge.

## What This Changes

This PR adds a bundled `oc-path` plugin that registers `openclaw path`, implements md/jsonc/jsonl `oc://` parsing and editing, and updates docs, labels, package metadata, lockfile entries, and tests.

## Best Possible Solution

Land the plugin-owned CLI after fixing the two contract mismatches while keeping parser dependencies plugin-local and avoiding any return to core `src/oc-path`.

## Reproduction Assessment

Yes, source-reproducible. At head ca6d65abd5461c47ac45ce83a1460e9731bac128, positional JSONC insertion parents and validate parse-error exit behavior can be traced directly through the changed source and docs.

## Solution Assessment

No. The bundled-plugin direction is maintainable and agreed in the related discussion, but this is not the best mergeable form until JSONC insertion parent normalization and validate exit-code behavior are aligned.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Normalize JSONC insertion parent segments:** `extensions/oc-path/src/oc-path/universal.ts:607-609`
  - body: `resolveJsoncInsertion` accepts insertion parents reached through positional segments such as `items.$last`, but `mutateJsoncContainer` replays the raw parent path and array descent uses `Number(seg)`. That means a path can report a valid insertion point and then fail as unresolved during `set`; normalize positional segments during mutation or pass the concrete resolved parent path through.
  - confidence: 0.91
- **[P3] Return exit 2 for validate parse errors:** `extensions/oc-path/src/cli.ts:363`
  - body: The docs reserve exit code 2 for argument or parse errors, and the other commands use that code via `tryParse`, but `validate` catches `OcPathError` and exits 1. Invalid syntax is therefore indistinguishable from a no-match result for scripts; make `validate` exit 2 or document and test the exception consistently.
  - confidence: 0.86
