## Summary

Keep open: the plugin-owned direction is supported by the discussion and proof, but the latest head still has merge-blocking doc/release-note gaps and a JSONL append line-ending regression.

## What This Changes

The PR adds a bundled `oc-path` plugin that registers `openclaw path`, moves the oc:// parser/editor surface under `extensions/oc-path`, adds docs/tests/fixtures, and keeps parser dependencies plugin-local.

## Best Possible Solution

Land the bundled-plugin direction after preserving JSONL append line endings and aligning the docs, exit-code contract, and changelog with the opt-in oc-path surface.

## Reproduction Assessment

Yes for the review findings: source inspection shows `appendJsonlOcPath` drops `ast.lineEnding`, the manifest is opt-in while the docs omit the enable step, and the changelog lacks an oc-path entry. I did not execute the suite because this review is read-only.

## Solution Assessment

No. The plugin boundary is the right maintainable direction, but the PR is not the best mergeable form until the JSONL append path preserves CRLF files and the public docs/release notes match the chosen opt-in activation contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Preserve JSONL line endings on append:** `extensions/oc-path/src/oc-path/jsonl/edit.ts:191`
  - body: `appendJsonlOcPath` rebuilds the next AST without copying `ast.lineEnding`, so appending to a CRLF JSONL file renders through `emitJsonl` with the LF default. That rewrites Windows-authored logs even though the parser/emitter already track line-ending convention for edits.
  - confidence: 0.91
- **[P2] Document the oc-path enable step:** `docs/cli/path.md:12-16`
  - body: The manifest leaves this bundled plugin disabled by default, but the path page goes straight into `openclaw path` usage without telling fresh installs to run `openclaw plugins enable oc-path`. Users following the page will hit the disabled-plugin diagnostic instead of the documented command.
  - confidence: 0.86
- **[P3] Add the required changelog entry:** `CHANGELOG.md:7`
  - body: This branch introduces a user-facing plugin-provided `openclaw path` CLI and changes the supported grammar/file-kind surface, but `CHANGELOG.md` has no Unreleased entry for it. Release notes would omit the new public surface.
  - confidence: 0.8
- **[P3] Align the validate exit-code contract:** `docs/cli/path.md:90-96`
  - body: The exit-code table says parse errors exit 2, but `pathValidateCommand` catches `OcPathError` and exits 1 for invalid paths, with a test pinning that behavior. Either the docs or the command should be aligned before users script against this table.
  - confidence: 0.78
