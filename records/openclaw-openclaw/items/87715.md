## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Make handoff copy names collision-proof:** `extensions/imessage/src/send.ts:195-198`
  - body: The copied attachment path is only `Date.now()` + `process.pid` + sanitized basename. Two same-process sends of files with the same basename in the same millisecond can target the same file, so one copy can overwrite the other and either cleanup can remove the path while the other `send-attachment` is still reading it. Use a per-send unique path such as `randomUUID`, `mkdtemp`, or exclusive creation, and cover concurrent same-basename sends.
  - confidence: 0.86
- **[P2] Document the outbound handoff directory env:** `extensions/imessage/src/send.ts:179-184`
  - body: This adds `OPENCLAW_IMESSAGE_OUTBOUND_HANDOFF_DIR` and rejects relative paths, but there is no user-facing documentation for when remote/bot-readable iMessage setups need it, what absolute path should point to, what permissions are used, or when temporary copies are cleaned up. That makes the new delivery path hard to operate safely outside the contributor's local setup.
  - confidence: 0.78
