## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Make handoff copy names unique per send:** `extensions/imessage/src/send.ts:195-198`
  - body: `Date.now() + process.pid + basename` is not collision-proof. Two concurrent sends of files with the same basename in the same process and millisecond will target the same handoff path, so one `copyFile` can overwrite the other and either `finally` cleanup can delete the file while the other `send-attachment` call is still reading it. Use a truly unique per-send path and add a concurrent same-basename regression.
  - confidence: 0.88
- **[P2] Document the outbound handoff env:** `extensions/imessage/src/send.ts:102`
  - body: This introduces `OPENCLAW_IMESSAGE_OUTBOUND_HANDOFF_DIR` as an operator-controlled media handoff path, including an absolute-path requirement and cleanup behavior, but the iMessage docs do not mention it. Remote/bot-readable bridge setups need public guidance for where to point it, permissions, cleanup expectations, and troubleshooting.
  - confidence: 0.86
