## Summary

Keep open: the patch addresses the source-reproducible iMessage media routing gap, but it needs redacted live iMessage proof and a fix or maintainer decision for copied handoff media retention before merge.

## What This Changes

The PR changes the iMessage sender to resolve direct media targets to chat GUIDs, send media via `send-attachment`, split captions into a follow-up text send, add an outbound handoff directory env path, and extend send tests.

## Best Possible Solution

Land a version that keeps direct/caption media on `send-attachment`, makes handoff copies ephemeral or explicitly documented and owned, and includes redacted live iMessage proof.

## Reproduction Assessment

Yes, from source inspection: current main only attempts `send-attachment` for media-only explicit chat sends, while direct handles and captioned media fall through to RPC `send` with `file`. I did not run a live iMessage bridge repro.

## Solution Assessment

No, not as-is. Routing through `send-attachment` is the right direction, but the handoff-copy lifecycle and missing live transport proof need to be resolved before this is the best mergeable fix.
