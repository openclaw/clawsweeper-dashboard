## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Use a collision-proof handoff copy name:** `extensions/imessage/src/send.ts:195-198`
  - body: The handoff copy name is only `Date.now()`, `process.pid`, and the sanitized basename. Two concurrent sends of the same basename in the same process can hit the same millisecond, hand the same path to `send-attachment`, and then one `finally` cleanup can remove the other send's in-flight media. Use a per-send unique component such as `randomUUID()` or a private temp directory, and cover concurrent same-basename sends.
  - confidence: 0.9
- **[P2] Document the outbound handoff env var:** `extensions/imessage/src/send.ts:102`
  - body: This adds `OPENCLAW_IMESSAGE_OUTBOUND_HANDOFF_DIR` as a new operator setup surface, including absolute-path validation, directory creation, `0640` copies, and cleanup behavior, but the public iMessage docs and config reference do not mention it. Remote/bot-readable media setup will be hard to configure and troubleshoot without docs for path choice, permissions, cleanup, and failure modes.
  - confidence: 0.88
