## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Make handoff copy names collision-proof:** `extensions/imessage/src/send.ts:197`
  - body: The handoff copy path is only `${Date.now()}-${process.pid}-${basename}`. Two concurrent media sends of the same basename in the same process/millisecond can write the same path, and whichever send finishes first can remove the copy while the other `send-attachment` still needs it. Use a per-send random/temp path and cover concurrent same-basename sends.
  - confidence: 0.83
- **[P2] Document the outbound handoff directory env:** `extensions/imessage/src/send.ts:102`
  - body: This introduces `OPENCLAW_IMESSAGE_OUTBOUND_HANDOFF_DIR` as an operator-facing absolute path that changes media send behavior and can fail sends when invalid, but the iMessage docs still only describe inbound attachment roots and remote fetches. Add docs for when to set it, absolute/shared-directory requirements, cleanup/permissions, and remote-Mac expectations.
  - confidence: 0.88
