Full review comments:

- **[P2] Use a collision-proof handoff copy name:** `extensions/imessage/src/send.ts:195-198`
  - body: The generated handoff filename combines only `Date.now()`, `process.pid`, and the sanitized basename. Two concurrent sends of the same basename in the same process and millisecond can copy to the same path, and the first `finally` cleanup can remove the file while the other `send-attachment` still needs it. Use a per-send random or exclusive temp path and add a concurrent same-basename regression test.
  - confidence: 0.88
- **[P2] Document the outbound handoff environment variable:** `extensions/imessage/src/send.ts:179-184`
  - body: This adds `OPENCLAW_IMESSAGE_OUTBOUND_HANDOFF_DIR` as an operator-facing path with absolute-path validation, copy permissions, and cleanup behavior, but the public iMessage docs only describe inbound/remote attachments. Operators who need a bot-readable outbound directory will not know how to configure or troubleshoot it before direct media sends still fail in remote setups.
  - confidence: 0.86
