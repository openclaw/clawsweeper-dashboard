## Summary

Keep open. Current main still lacks `failureAlert.silent` in the strict cron schemas, runtime alert resolution, Gateway notification plumbing, Control UI/CLI surfaces, and docs, while shared outbound delivery and the Telegram plugin already support quiet sends.

## What This Changes

Add a `silent` boolean to cron job `failureAlert` so Telegram failure alerts can be delivered without push or sound notifications.

## Best Possible Solution

Add a backward-compatible `silent?: boolean` failure-alert option and thread it through cron contracts, runtime resolution, Gateway announce delivery, UI/CLI editing, docs, and tests while leaving omitted or false alerts audible.

## Reproduction Assessment

Yes. Source inspection on current main shows `failureAlert.silent` is rejected by the strict Gateway schema and there is no runtime or UI path that can preserve or deliver it.

## Solution Assessment

Yes. The requested additive field is the narrowest maintainable path because shared outbound delivery and the Telegram plugin already support `silent`; cron should pass the existing generic option through.
