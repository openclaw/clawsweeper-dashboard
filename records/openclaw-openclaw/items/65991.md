## Summary

Keep open. Current main still lacks `failureAlert.silent` in the cron failure-alert contract and plumbing, even though the shared outbound delivery and Telegram send paths already support silent delivery. This is a narrow core cron/delivery feature, not a duplicate or ClawHub-only request.

## What This Changes

The issue requests a backward-compatible `silent` boolean on cron `failureAlert` so Telegram cron failure notifications can be delivered without sound or push notifications.

## Best Possible Solution

Add an optional `failureAlert.silent?: boolean` contract that is preserved through cron config/job update paths and passed only into announce/channel failure-alert delivery, with omitted or false remaining audible. Keep the cron core generic and rely on existing outbound/Telegram silent support rather than adding Telegram-specific branching.

## Reproduction Assessment

Yes. A `cron.add` or `cron.update` payload containing `failureAlert.silent` can be checked against `CronFailureAlertSchema`, which omits the field and rejects additional properties; source inspection also shows no runtime or UI path that preserves it.

## Solution Assessment

Yes. The requested additive boolean is the narrowest maintainable solution because the downstream outbound and Telegram layers already support silent delivery; the safer boundary is to plumb a generic delivery option through cron failure alerts without changing webhook behavior or default audible delivery.
