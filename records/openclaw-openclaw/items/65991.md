## Summary

Keep open. The prior ClawSweeper review still matches current main: cron failure-alert contracts, validation, runtime plumbing, and Control UI state do not carry a `silent` option, while the outbound and Telegram delivery layers already know how to send silently.

## What This Changes

Add a `silent` boolean to cron job `failureAlert` configuration so Telegram failure alerts can be delivered without a notification sound.

## Best Possible Solution

Add a backward-compatible `silent?: boolean` cron failure-alert contract, resolve it from alert config, pass it only through announce/channel delivery into `deliverOutboundPayloads`, update Control UI/docs/tests/schema outputs, and keep omitted or false alerts audible.

## Reproduction Assessment

Yes. A `cron.add` or `cron.update` payload containing `failureAlert.silent` would fail the strict `CronFailureAlertSchema`, and source inspection shows no runtime, gateway, or UI path that could preserve the field.

## Solution Assessment

Yes. The requested additive field is the narrowest maintainable solution because the shared outbound layer and Telegram plugin already support quiet sends; the fix should avoid Telegram-specific branching in cron core.
