## Summary

Keep open. Current main still lacks `failureAlert.silent` in the cron wire contract, runtime alert resolution, Gateway notification plumbing, Control UI state, and docs, while the shared outbound and Telegram plugin layers already support quiet delivery.

## What This Changes

Add an additive `silent` option to cron `failureAlert` so Telegram failure alerts can be delivered without sound or push notification.

## Best Possible Solution

Add a backward-compatible `silent?: boolean` failure-alert contract and thread it through cron resolution, Gateway announce delivery, Control UI, docs, and tests while leaving omitted or false alerts audible.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: a `cron.add` or `cron.update` payload containing `failureAlert.silent` is outside the strict Gateway schema, and no current runtime/UI path can preserve or deliver it.

## Solution Assessment

Yes. The requested additive field is the narrowest maintainable solution because shared outbound delivery and the Telegram plugin already support `silent`; cron should pass a generic delivery option rather than add Telegram-specific branching.
