## Summary

## What This Changes

The branch guards agent tool metadata reads, substitutes an invalid schema sentinel when `parameters` is unreadable, and adds a regression test for `toToolDefinitions`.

## Best Possible Solution

Keep the unreadable-`parameters` sentinel behavior, but preserve quarantine for unreadable tool identity fields before the tool can reach model-visible runtime surfaces.

Yes. Current main directly reads `tool.parameters` during `toToolDefinitions`, and the PR’s focused regression demonstrates the intended after-fix path; I did not execute tests because this was a read-only review.

## Solution Assessment

No. The invalid schema sentinel is the right shape for unreadable parameter schemas, but masking an unreadable tool name as `tool` is not the best fix because it bypasses the existing quarantine contract for unreadable tool identity.
