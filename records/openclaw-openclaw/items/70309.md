## Summary

Keep open. Current main still has only configured MCP counts plus successful tool catalog/effective-tool views; it does not expose a Gateway MCP health/reconnect API or a Control UI panel that preserves per-server failed state and safe error details.

## What This Changes

The issue requests a Control UI MCP connectivity panel showing per-server status, tool listings, reconnect/refresh, and safe error details for configured MCP servers.

## Best Possible Solution

Define an additive, redacted Gateway MCP diagnostics and refresh contract shared with the runtime reliability work, then render per-server health, tool counts/names, and safe errors in Control UI.
