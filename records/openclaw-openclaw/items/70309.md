Keep open. Current main and v2026.6.8 have a static MCP settings page, CLI status/probe/doctor commands, and warm-session MCP tool inventory notices, but they still do not provide the requested live per-server Control UI connectivity panel with refresh/reconnect, tool names, and surfaced connection errors.

The issue requests a Control UI panel showing per-MCP-server connectivity, available tools, refresh/reconnect controls, and error details.

## Best Possible Solution

Add an explicit user-initiated MCP diagnostics panel backed by existing probe/reload/catalog primitives, with redacted error details and non-connecting default page loads.
