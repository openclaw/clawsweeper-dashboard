## Summary

Keep open. Current main still lacks a Gateway MCP diagnostics method and a Control UI connectivity panel; existing surfaces cover saved config, server counts, and session-scoped tool inventory only, while related #64411 is broader runtime-degradation work rather than a replacement.

## What This Changes

Request to add a Control UI MCP Connectivity section that reports per-server connection/error state, tool names/counts, and manual refresh/reconnect actions for configured `mcp.servers`.

## Best Possible Solution

Design an additive operator-only Gateway MCP diagnostics surface with redacted per-server status, safe last-error details, tool names/counts after handshake, and scoped refresh controls rendered under Control UI Infrastructure > MCP, coordinated with #64411 for shared health semantics.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken behavior report. Source inspection on current main shows the missing surface: Gateway has no MCP diagnostics RPC and Control UI only shows configuration/count surfaces.

## Solution Assessment

Yes in direction, but not ready as a blind automation fix. The maintainable path is an additive Gateway diagnostics API plus Control UI panel with redacted errors and explicit runtime lifecycle semantics.
