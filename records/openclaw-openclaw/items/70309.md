## Summary

Keep this open. Current main and v2026.6.8 provide an MCP settings page, CLI status/probe/doctor commands, and warm-session MCP tool notices, but they still do not provide the requested Control UI live per-server connectivity panel with reconnect/refresh, tool names, and surfaced connection errors.

## What This Changes

Add a Control UI MCP connectivity panel showing live per-server status, available tools, refresh/reconnect controls, and connection error details.

## Best Possible Solution

Add an explicit user-initiated MCP diagnostics panel backed by existing probe/reload/catalog primitives, with redacted error details and non-connecting default page loads.

Not applicable: this is a feature request for a new Control UI diagnostics panel, not a broken existing documented behavior. Source and docs inspection show current main and v2026.6.8 still route live MCP proof through CLI probe commands or already-warm session state.
