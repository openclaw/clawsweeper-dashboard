## Summary

Keep open. Current main still has only MCP config/count and session-scoped tool inventory surfaces, not the requested per-server connectivity, last-error, tool-list, or reconnect/refresh workflow in the Gateway or Control UI.

## What This Changes

Request a Control UI MCP connectivity panel that shows configured server health, exposed tools, reconnect/refresh controls, and visible connection errors.

## Best Possible Solution

Add an operator-scoped Gateway MCP diagnostics API backed by redacted config/runtime state, then render it under Control UI Infrastructure > MCP with per-server status, safe error details, tool names/counts after handshake, and a scoped refresh action.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high-confidence current-main verification that MCP is exposed as config/count plus session tool inventory, not as per-configured-server connectivity diagnostics.

## Solution Assessment

Yes in direction: an additive, operator-scoped Gateway diagnostics surface plus Control UI panel is the maintainable path. Reusing `tools.catalog` or `tools.effective` alone is insufficient because failed MCP servers are logged and removed before a session tool inventory exists.
