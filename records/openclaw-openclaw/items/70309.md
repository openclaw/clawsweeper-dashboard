## Summary

Keep this open. Current main and v2026.6.8 have an MCP settings page, CLI probe/status commands, and warm-session MCP tool notices, but they still do not provide the requested Control UI per-server connectivity panel with live status, reconnect/refresh, tool listing, and surfaced connection errors.

## What This Changes

The issue requests a Control UI panel that shows per-MCP-server connectivity, tool inventory, refresh/reconnect, and error details.

## Best Possible Solution

Add an explicit user-initiated MCP diagnostics panel that reuses existing probe/reload/catalog primitives, redacts sensitive details, and keeps routine dashboard reads non-connecting.

## Reproduction Assessment

Not applicable: this is a feature request for a new Control UI diagnostics panel, not a broken existing documented behavior. Source and docs inspection confirm current main and v2026.6.8 still route live MCP proof through CLI probe commands or already-warm session state.

## Solution Assessment
