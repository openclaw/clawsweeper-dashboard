## Summary

Keep this open. Current main has MCP config, CLI status/probe support, and partial Control UI visibility for already-warmed MCP tools, but it does not provide the requested Control UI connectivity panel with per-server live state, reconnect/refresh, and surfaced connection errors.

## What This Changes

The issue requests a Control UI panel that shows MCP server connectivity, discovered tools, reconnect controls, and connection error details.

## Best Possible Solution

Add an explicit, user-initiated Control UI MCP diagnostics panel backed by the existing MCP probe/runtime catalog primitives, with redacted errors, per-server tool summaries, and no implicit probing from ordinary dashboard reads.

## Reproduction Assessment

Not applicable: this is a feature request for a new Control UI diagnostics panel, not a broken existing documented behavior. Source review does confirm the requested panel is not present on current main.

## Solution Assessment

Unclear until maintainers choose the API and probing semantics. The maintainable direction is to reuse the existing MCP probe/catalog code while keeping routine Control UI reads non-connecting.
