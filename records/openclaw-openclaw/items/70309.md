## Summary

Keep open. Current main and the latest release provide static MCP configuration inventory, CLI live probe/status commands, and warm-session tool visibility, but not the requested live Control UI panel with per-server connected/error state, fresh tool names, surfaced errors, or one-click reconnect/refresh.

Requests a Control UI MCP Connectivity panel for per-server live status, tool listing, refresh/reconnect, and error details.

## Best Possible Solution

Define a maintainer-approved, user-triggered MCP diagnostics API and Control UI surface that reuses existing status/probe/reload semantics without automatic page-load probing, with redacted errors and clear auth/session scope.
