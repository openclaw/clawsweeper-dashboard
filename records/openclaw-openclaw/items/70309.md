## Summary

Keep open. Current main still has only configuration counts and successful tool catalog data for MCP; it does not expose a Gateway MCP diagnostics/reconnect contract or a Control UI panel that preserves per-server failed state, so the requested feature remains meaningful and needs maintainer product/security design.

## What This Changes

Request adds a Control UI MCP Connectivity section showing per-server status, tool lists, refresh/reconnect controls, and safe error details for configured `mcp.servers`.

## Best Possible Solution

Define an additive, redacted Gateway MCP diagnostics and refresh contract, then render per-server health, safe errors, tool counts/names, and reconnect actions in Control UI.
