## Summary

Keep open: current main still has a source-proven Slack identity propagation gap, and the linked fix PR is open but not merged. Cron announce now resolves identity before durable delivery, but the CLI lazy sender still drops that identity before the Slack adapter, and heartbeat target-channel sends do not pass identity at all.

## What This Changes

Per-agent `agents.list[].identity` should be applied to Slack cron announce and heartbeat target-channel outbound messages, matching the already-fixed reply path.

## Best Possible Solution

Review and land or revise https://github.com/openclaw/openclaw/pull/84335, or replace it with a narrow fix that threads `OutboundIdentity` through the lazy runtime sender and heartbeat delivery calls with focused regression tests and clear Slack emoji semantics.

## Reproduction Assessment

Yes for a source-level reproduction: current main has an identity-capable Slack adapter and docs for active agent identity, but the lazy runtime sender drops `opts.identity` and heartbeat sends omit identity entirely. I did not run a live Slack cron or heartbeat proof in this read-only review.

## Solution Assessment

Yes: the narrow maintainable solution is to carry the existing `OutboundIdentity`/`ChannelOutboundContext.identity` contract through the missing runtime sender and heartbeat paths, rather than adding Slack-specific username or icon fields in core.
