## Summary

Keep open: current main still has a source-proven identity propagation gap. Slack outbound honors `ChannelOutboundContext.identity`, cron already resolves an agent identity before durable delivery, but the lazy runtime sender drops that identity and heartbeat target-channel sends never pass one.

## What This Changes

The issue requests that cron `--announce` and heartbeat Slack target-channel messages use the configured per-agent identity overlay, matching the already-fixed Slack reply path.

## Best Possible Solution

Thread the existing `OutboundIdentity` contract through `createChannelOutboundRuntimeSend` and heartbeat target-channel delivery, then add focused regression coverage for runtime sender, cron announce, and heartbeat Slack identity propagation.

## Reproduction Assessment

Yes for a source-level reproduction: current main has an identity-capable channel contract and Slack adapter, while the lazy runtime sender drops identity and heartbeat sends omit it. I did not establish a live Slack cron or heartbeat run in this read-only review.

## Solution Assessment

Yes. The narrow maintainable fix is to use the existing `OutboundIdentity`/`ChannelOutboundContext.identity` contract rather than adding Slack-specific username or icon fields in core.
