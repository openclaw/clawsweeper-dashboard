Keep open: current main still has the source-proven identity propagation gap, and the linked closing PR is open rather than merged. Cron announce now resolves identity upstream, but the lazy runtime sender still drops it and heartbeat target-channel sends still omit it.

## What This Changes

The issue reports that per-agent Slack identity overlays are not applied to cron announce and heartbeat target-channel Slack sends, even though Slack replies use the configured persona.

## Best Possible Solution

Review and either land or supersede the linked focused fix PR, preserving the current cron identity resolution while threading OutboundIdentity through heartbeat sends and the lazy runtime sender with targeted regression coverage.

## Reproduction Assessment

Yes for a source-level reproduction: current main still drops identity in createChannelOutboundRuntimeSend and omits identity from heartbeat target-channel sendDurableMessageBatch calls. I did not run a live Slack cron or heartbeat proof in this read-only review.

## Solution Assessment

Yes: the narrow maintainable fix is to carry the existing OutboundIdentity/ChannelOutboundContext.identity contract through the missing runtimeSend and heartbeat paths, not to add Slack-specific persona fields in core.
