## Summary

Keep open: current main still has a source-proven identity propagation gap, and an open closing PR already owns the fix path. Cron announce now resolves and passes identity, but heartbeat target-channel sends and the legacy runtime sender still omit it before Slack can apply the per-agent persona.

## What This Changes

The issue reports that configured per-agent Slack identity overlays are dropped on scheduled/background Slack sends, especially heartbeat target-channel pushes and the cron announce/runtimeSend path.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/84335 or an equivalent focused patch that threads the existing `OutboundIdentity` contract through heartbeat and runtimeSend without regressing the already-wired cron announce path.

## Reproduction Assessment

Yes for a source-level reproduction: current main still omits identity from the legacy runtimeSend context and heartbeat `sendDurableMessageBatch` calls while Slack only applies persona metadata when `ctx.identity` reaches the adapter. I did not run live Slack cron or heartbeat proof in this read-only review.

## Solution Assessment

Yes, the maintainable fix is to use the existing `OutboundIdentity` and `ChannelOutboundContext.identity` contract rather than adding Slack-specific username/icon fields in core. The linked open PR is the current implementation candidate and should be reviewed or replaced before this issue closes.
