## Summary

Keep open: current main still has a source-proven identity propagation gap. Slack outbound honors `context.identity`, and cron delivery now resolves an agent identity, but the lazy runtime sender drops `opts.identity`, and heartbeat target-channel sends do not resolve or pass identity at all.

## What This Changes

The issue asks OpenClaw to apply `agents.list[].identity` to Slack cron `--announce` and heartbeat target-channel outbound messages, matching Slack reply behavior.

## Best Possible Solution

Thread the existing `OutboundIdentity` contract through the lazy channel runtime sender and heartbeat delivery calls, then pin cron and heartbeat Slack identity behavior with focused regression tests.

## Reproduction Assessment

Yes for a source-level reproduction: current main has an identity-capable Slack adapter, but the lazy runtime sender drops identity and heartbeat sends omit it. I did not run a live Slack cron/heartbeat proof in this read-only review.

## Solution Assessment

Yes, the maintainable fix is to use the existing `OutboundIdentity`/`ChannelOutboundContext.identity` contract rather than adding Slack-specific username/icon fields in core. The repair should preserve existing cron identity resolution and add heartbeat identity propagation.
