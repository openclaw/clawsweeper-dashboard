## Summary

Keep open: current main still depends on the node host live `connect.commands` list for command support, file-transfer declares the four expected node commands, and there is no source, test, release, or related-item evidence that Windows node hosts now advertise them reliably.

## What This Changes

The issue reports that the bundled file-transfer plugin's node-side commands are missing from the Windows node host live handshake even when the plugin and gateway allowlist are enabled.

## Best Possible Solution

Make enabled plugin `nodeHostCommands` advertise in the live node-host connect payload on Windows, with package-level or integration coverage for file-transfer under node-host startup.

## Reproduction Assessment

No high-confidence live reproduction was performed in this review. The reporter provides a concrete Windows 11 npm setup, and source inspection confirms the observed gateway error follows when `nodeSession.commands` lacks `dir.list`.

## Solution Assessment

Unclear until the live Windows node-host registry state is captured. The maintainable direction is to fix plugin registry activation or packaged bundled-plugin resolution for node-host startup, not to bypass the gateway declared-command gate.
