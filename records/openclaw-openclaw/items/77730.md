## What This Changes

The issue reports that Windows node hosts do not advertise the file-transfer plugin's `file.fetch`, `dir.list`, `dir.fetch`, and `file.write` commands during the live gateway handshake.

## Best Possible Solution

Fix the existing node-host plugin activation or handshake path so enabled plugin `nodeHostCommands` are advertised consistently on live reconnects, with regression coverage for file-transfer commands and declared-command rejection behavior.

## Reproduction Assessment

No high-confidence current-main live reproduction was run. The report gives detailed live Windows steps, and current source confirms the exact gateway rejection path when the node's live `commands` payload lacks `dir.list`, but this review did not prove where current main drops those declarations.

## Solution Assessment

Unclear until the node-host activation boundary is proven. The best path is a narrow repair in plugin registry loading, node-host config activation, bundled plugin resolution, or pairing reconciliation rather than a new config knob or a bypass around declared-command policy.
