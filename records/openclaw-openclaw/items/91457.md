## Summary

Keep open: current main and v2026.6.8 still route the Agents page Set Default action to a draft-only config mutation, while this PR is a focused, well-proven fix with no blocking review findings.

## What This Changes

This PR changes Control UI Set Default to call a new controller helper that stages `agents.list[].default`, saves clean drafts through `config.set`, leaves pre-existing dirty drafts staged, and adds unit plus browser regression coverage.

## Best Possible Solution

Land this focused controller-level persistence fix after normal maintainer gates, while leaving the broader visible-list UX and optional dedicated RPC direction to their separate items.

## Reproduction Assessment

Yes. Current main wires Set Default to a draft-only helper without `config.set`, and the PR body supplies before/after real Gateway readback showing the default flag is dropped before and persisted after.

## Solution Assessment

Yes. Composing the existing staging helper with `saveAgentsConfig` at the controller boundary is the narrowest current UI fix; the future `agents.setDefault` RPC is adjacent and not a merge-ready replacement for this bug fix.
