Keep open: the PR addresses a real current-main Control UI persistence bug and the updated PR body now supplies credible real Gateway proof, but the patch should not merge yet because the new one-click save path can also persist unrelated dirty config-form edits.

## What This Changes

The PR adds a Control UI controller helper that stages `agents.list[].default`, calls the existing agent config save path from the Agents page Set Default handler, and adds unit plus mocked browser regression coverage.

## Best Possible Solution

Scope Set Default persistence to the default-agent change, or preserve pre-existing dirty drafts so unrelated config edits remain unsaved until the user presses Save.

## Reproduction Assessment

Yes. Source inspection shows current main wires Set Default only to `stageDefaultAgentConfigEntry`, which mutates the config draft without a `config.set`; the PR body also includes before/after Gateway readback proof.

## Solution Assessment

No. The controller boundary is the right place, but calling `saveAgentsConfig` unconditionally is too broad because it saves the entire dirty form rather than only the Set Default change.
