## Summary

Keep open: current main still matches the reported iOS group-chat live update gap, and targeted GitHub searches did not find a duplicate issue or linked fix PR. The existing Gateway event contract is present, but the iOS transport and shared chat view model still do not consume assistant transcript events for the active session.

## What This Changes

The issue reports that iOS group chat assistant messages do not appear in real time until the chat view is reopened.

## Best Possible Solution

Wire iOS chat to the existing `sessions.messages.subscribe` and `session.message` contract, then update the shared view model to merge active-session assistant transcript events without adding polling, new protocol events, or group-chat-specific policy.

## Reproduction Assessment

Yes from source inspection, but not from a live iOS run: current main does not subscribe to active-session transcript events, drops `session.message` frames in the iOS transport, and filters assistant transcript messages in the shared view model.

## Solution Assessment

Yes: the narrow maintainable fix is to use the existing Gateway session-message subscription and payload types, mirroring the macOS path where appropriate, rather than adding polling or a new group-chat channel.
