## Summary

Keep open: current main still has the iOS/shared event-plumbing gap described by this issue. The Gateway already publishes per-session transcript events, and macOS consumes that contract, but the iOS transport neither subscribes to nor decodes `session.message`, while the shared view model currently ignores assistant session-message payloads.

## What This Changes

The issue reports that iOS group-chat agent replies do not appear in real time until leaving and reopening the chat.

## Best Possible Solution

Wire iOS to the existing Gateway per-session transcript subscription and update the shared chat model to merge active-session assistant transcript events with focused Swift regression coverage.

## Reproduction Assessment

Yes from source inspection, not from a live iOS run: current main still does not subscribe to `sessions.messages.subscribe`, does not decode `session.message` in the iOS transport, and filters assistant session-message payloads in the shared view model.

## Solution Assessment

Yes: the narrowest maintainable fix is to use the existing Gateway `sessions.messages.subscribe` and `session.message` contract, matching macOS, rather than adding polling, a new protocol event, or a group-chat-specific workaround.
