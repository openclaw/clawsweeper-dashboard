## Summary

Keep open: current main still has the native iOS group-chat update gap. The Gateway already emits and documents `session.message`, but Aight's iOS transport does not subscribe to or decode that event, and the shared view model currently ignores assistant `session.message` payloads.

## What This Changes

Report that Aight iOS group chats do not show new agent replies in real time until the user leaves and re-enters the chat.

## Best Possible Solution

Wire native iOS chat to the existing Gateway session-message subscription/decoding path, then let the shared view model merge active-session assistant transcript events with focused Swift regression coverage.

## Reproduction Assessment

Yes from source inspection, but not from a live iOS device run: current main does not subscribe to `sessions.messages.subscribe`, drops `session.message` frames in the iOS transport, and filters assistant session-message payloads in the shared view model.

## Solution Assessment

Yes: the narrowest maintainable fix is to consume the existing Gateway `sessions.messages.subscribe` and `session.message` contract rather than adding polling, a new protocol event, or a group-chat-specific path.
