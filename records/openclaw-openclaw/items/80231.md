## Summary

Keep open: current main still matches the reported iOS group-chat real-time update gap, and live searches found no linked or duplicate fix PR. The existing Gateway session-message contract is present, but the iOS transport and shared chat model still do not consume active-session assistant transcript events end to end.

## What This Changes

iOS group-chat assistant messages should appear in the active conversation without requiring the user to leave and reopen the chat.

## Best Possible Solution

Wire iOS to the existing session-message subscription and decoding path, then let the shared chat model merge active-session assistant transcript events with focused Swift regression coverage.

## Reproduction Assessment

Yes from source inspection, though not from a live iOS run. Current main leaves `setActiveSessionKey` as a no-op, drops `session.message` frames in iOS, and filters assistant session-message payloads in the shared view model.

## Solution Assessment

Yes. The narrowest maintainable fix is to use the existing `sessions.messages.subscribe` and `session.message` Gateway contract rather than adding polling, a new protocol event, or a group-chat-specific server path.
