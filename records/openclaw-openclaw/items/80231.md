## Summary

Keep open: current main still has the reported iOS group-chat live update gap. The Gateway already exposes and emits active-session transcript events, but the iOS transport does not subscribe to or decode them, and the shared chat view model currently ignores non-user session-message payloads.

## What This Changes

Fix iOS group-chat views so new multi-agent replies appear in real time without leaving and reopening the conversation.

## Best Possible Solution

Wire iOS chat to the existing Gateway session-message subscription/decoding path and update the shared view model to merge active-session assistant transcript events with focused Swift regression coverage.

## Reproduction Assessment

Yes, from source inspection: current main does not subscribe iOS chat to `sessions.messages.subscribe`, drops `session.message` frames, and filters assistant session-message payloads in the shared view model. I did not run a live iOS device or simulator flow.

## Solution Assessment

Yes: the narrow maintainable fix is to reuse the existing Gateway `sessions.messages.subscribe` and `session.message` contract, not add polling or a group-chat-specific protocol path.
