## Summary

Keep open: current main still has the reported iOS group-chat live-update gap, with no linked or discovered iOS fix PR. The Gateway already exposes the session-message event path, but the iOS transport and shared view model do not consume assistant transcript updates for the active chat.

## What This Changes

The issue reports that iOS/Aight group chat assistant replies do not appear in real time until the user leaves and re-enters the chat.

## Best Possible Solution

Wire iOS chat to the existing `sessions.messages.subscribe` and `session.message` path, then update the shared view model to merge active-session assistant transcript events with focused Swift coverage.

## Reproduction Assessment

Yes from source inspection, but not from a live iOS run: current main does not subscribe to session-message events in iOS, drops `session.message` frames in the iOS transport, and filters assistant session messages in the shared view model.

## Solution Assessment

Yes: the narrowest maintainable fix is to use the existing Gateway session-message subscription and event contract rather than adding polling, a new protocol event, or group-chat-specific plumbing.
