## Summary

Keep open: current main still matches the reported iOS group-chat real-time update gap, and I found no duplicate or open fix PR. The existing Gateway session-message event path is present, but the iOS chat transport and shared view model do not consume assistant transcript events for the active session.

## What This Changes

The issue asks for iOS group-chat agent replies to appear in real time without requiring the user to leave and reopen the chat.

## Best Possible Solution

Wire iOS chat to the existing Gateway session-message subscription/decoding path and update the shared view model to merge active-session assistant transcript events with focused Swift regression coverage.

## Reproduction Assessment

Yes from source inspection, but not from a live iOS run: current main does not subscribe to session-message events, drops `session.message` frames in the iOS transport, and filters non-user session messages in the shared view model.

## Solution Assessment

Yes: the narrowest maintainable fix is to use the existing Gateway `sessions.messages.subscribe` and `session.message` contract rather than adding polling, a new protocol event, or a group-chat-specific path.
