## What This Changes

The issue asks for iOS group chats to show new agent messages in real time without leaving and re-entering the chat.

## Best Possible Solution

Use the existing Gateway session-message subscription contract in the iOS chat transport, decode `session.message` into the shared event model, and merge active-session assistant transcript updates with focused Swift regression coverage.

## Reproduction Assessment

Yes from source inspection, but not from a live iOS run: current main does not subscribe to `sessions.messages.subscribe`, drops `session.message` frames in the iOS transport, and ignores assistant session-message payloads in the shared view model.

## Solution Assessment

Yes: the narrowest maintainable fix is to use the existing `sessions.messages.subscribe` and `session.message` contract rather than adding polling, a new protocol event, or a group-chat-specific workaround.
