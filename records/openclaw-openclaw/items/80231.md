## Summary

Keep open: current main still has the reported iOS group-chat live-update gap. The Gateway already exposes transcript subscriptions, but the iOS chat transport does not subscribe to or decode `session.message`, and the shared chat view model currently drops assistant session-message events.

## What This Changes

The issue reports that iOS group-chat agent replies do not appear live until the user leaves and reopens the chat.

## Best Possible Solution

Wire the iOS transport to the existing Gateway session-message subscription/decoding path and update the shared view model to merge active-session assistant transcript events with focused Swift regression coverage.

## Reproduction Assessment

Yes from source inspection, though not from a live iOS run: current main does not subscribe to active-session transcript events, does not decode `session.message` on iOS, and filters assistant session messages in the shared view model.

## Solution Assessment

Yes: the narrow maintainable fix is to consume the existing `sessions.messages.subscribe` and `session.message` contract, not to add polling, a new protocol event, or a group-chat-specific backend path.
