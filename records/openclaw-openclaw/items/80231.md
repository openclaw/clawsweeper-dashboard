## Summary

Keep open: current main still has the iOS/shared live-update gap described by this issue. The Gateway already emits per-session transcript events and macOS consumes them, but the iOS transport does not subscribe to or decode `session.message`, and the shared view model currently ignores assistant session-message payloads.

## What This Changes

The issue reports that iOS group-chat agent replies do not appear live until the user leaves and re-enters the conversation.

## Best Possible Solution

Wire iOS to the existing per-session transcript subscription and update the shared chat model to merge active-session assistant transcript events live, with focused dedupe coverage.

## Reproduction Assessment

Yes from source inspection, not from a live iOS run: current main still does not subscribe to `sessions.messages.subscribe`, does not decode `session.message` in the iOS transport, and filters assistant session-message payloads in the shared view model.

## Solution Assessment

Yes: the narrowest maintainable fix is to use the existing Gateway `sessions.messages.subscribe` and `session.message` contract, matching macOS, rather than adding polling, a new protocol event, or a group-chat-specific workaround.
