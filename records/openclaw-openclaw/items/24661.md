## Summary

Current main does not implement first-class Cohere onboarding/auth-choice support. The codebase has since moved provider auth wiring into bundled plugin manifests/runtime, so the PR’s goal is still valid but its core-file patch shape is stale against current architecture.

## Evidence

- **No Cohere onboarding/auth implementation on main:** A repo-wide search for `cohere-api-key`, `COHERE_API_KEY`, `command-a-03-2025`, and `api.cohere.ai/compatibility/v1` returned no hits in `src`, `docs`, `extensions`, or `CHANGELOG.md`. That means current main still lacks the requested Cohere auth choice, env var handling, default model, and provider docs.
  - command: `rg -n "cohere-api-key|COHERE_API_KEY|command-a-03-2025|api\.cohere\.ai/compatibility/v1" src docs extensions CHANGELOG.md`
  - sha: [d3f6783b16ec](https://github.com/openclaw/openclaw/commit/d3f6783b16ecc10c67b101bf019ca4294453206a)
- **Onboarding flags now come from provider plugin manifests:** `register.onboard.ts` builds provider auth flags from `resolveManifestProviderOnboardAuthFlags()`, and `provider-auth-choices.ts` documents that these flags are sourced from plugin manifest metadata rather than hardcoded core enums.
  - file: [src/cli/program/register.onboard.ts:50](https://github.com/openclaw/openclaw/blob/d3f6783b16ecc10c67b101bf019ca4294453206a/src/cli/program/register.onboard.ts#L50)
  - sha: [d3f6783b16ec](https://github.com/openclaw/openclaw/commit/d3f6783b16ecc10c67b101bf019ca4294453206a)
- **Current architecture expects provider auth choices in plugin metadata:** The plugin architecture docs say provider env vars and onboarding/auth-choice wiring should be declared via `providerAuthEnvVars` and `providerAuthChoices`, and the manifest reference defines the CLI flag fields there. This makes the PR’s older core-centric implementation approach stale, but not obsolete.
  - file: [docs/plugins/architecture.md:655](https://github.com/openclaw/openclaw/blob/d3f6783b16ecc10c67b101bf019ca4294453206a/docs/plugins/architecture.md#L655)
  - sha: [d3f6783b16ec](https://github.com/openclaw/openclaw/commit/d3f6783b16ecc10c67b101bf019ca4294453206a)
- **Bundled providers expose auth choice metadata via their own plugin packages; Cohere is absent:** `extensions/mistral/openclaw.plugin.json` shows the current pattern for a bundled provider (`providerAuthEnvVars` plus `mistral-api-key` choice metadata). There is no `extensions/cohere/**` package, and `docs/providers/index.md` lists Mistral but not Cohere.
  - file: [extensions/mistral/openclaw.plugin.json:1](https://github.com/openclaw/openclaw/blob/d3f6783b16ecc10c67b101bf019ca4294453206a/extensions/mistral/openclaw.plugin.json#L1)
  - sha: [d3f6783b16ec](https://github.com/openclaw/openclaw/commit/d3f6783b16ecc10c67b101bf019ca4294453206a)

## Risks / Open Questions

- The PR appears stale relative to the plugin-manifest/provider-runtime architecture on current main, so landing it would likely require a rework into a bundled Cohere provider plugin instead of the original core-file edits.
- I did not run onboarding or tests because this review was explicitly read-only and the checkout had to remain byte-for-byte clean.
