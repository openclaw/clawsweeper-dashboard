## Summary

Keep open. Current main shipped the default Docker image fallback/guidance fix from #73362, but it does not fully implement this PR's intended local-and-remote mutation error translation. The branch is also stale against the current local mutation helper, so merging it as-is would miss the current local Docker failure message while the remote-shell path still lacks friendly missing-python guidance.

## What This Changes

The PR adds a sandbox mutation error helper, wraps local and remote sandbox filesystem mutation execution to translate missing-`python3` failures, and adds a local bridge unit test.

## Best Possible Solution

Land a fresh current-main patch that preserves the shipped default-image behavior while adding scoped missing-interpreter guidance for the current local mutation helper and the remote-shell mutation helper, preserving the original error as `cause` and covering both paths with focused tests.

## Reproduction Assessment

Yes. The remaining behavior can be reproduced with mocked mutation execution: current main has no local wrapper around `runCheckedCommand()`, and `createRemoteShellSandboxFsBridge().writeFile()` still reaches `runRemoteScript()` directly when `python3 /dev/fd/3` fails.

## Solution Assessment

No. The PR direction is useful, but the proposed local detector is stale; the safer solution is to map both the current `python_cmd` helper failure and the remote `python3: not found` failure in scoped mutation paths.
