## What This Changes

The PR changes LINE auto-reply delivery to batch rich/media and text into one Reply API call up to five messages, push overflow, and add regression tests for batching, overflow, and Reply API fallback.

## Best Possible Solution

Land a corrected version after fixing the test type error and adding real LINE proof or maintainer override, while preserving the batching, overflow, and quick-reply behavior covered by tests.

## Reproduction Assessment

Yes, at source level: current main extracts tables into Flex messages, replies with text first, and then forces rich/media through Push API. I did not run a live exhausted-quota LINE account.

## Solution Assessment

Yes with corrections: batching eligible rich/media/text before the token is consumed is the narrow maintainable fix. This PR needs the test type error fixed and real behavior proof or maintainer override before merge.
