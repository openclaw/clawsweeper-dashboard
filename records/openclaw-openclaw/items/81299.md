## What This Changes

The PR changes LINE auto-reply delivery so rich/media and text are batched into one Reply API call up to five messages, pushes overflow, and updates focused regression tests.

## Best Possible Solution

Land a refreshed version after live LINE proof or maintainer override confirms Reply API batching and maintainers accept the five-slot ordering; otherwise adjust the overflow ordering before merge.

Yes for the code boundary: batching inside deliverLineAutoReply is the narrow owner-side fix before Push overflow. The remaining open decision is proof and maintainer acceptance of rich-first overflow ordering, not a different implementation layer.
