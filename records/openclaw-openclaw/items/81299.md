## What This Changes

The PR changes LINE auto-reply delivery to pack rich/media/text messages into one Reply API batch up to LINE's 5-message limit, push only overflow, and add focused regression tests.

## Best Possible Solution

Land the focused batching fix only after maintainers accept the overflow ordering and either receive redacted live LINE proof or explicitly override the real-proof gate.

## Reproduction Assessment

## Solution Assessment

Yes for the narrow no-quick-replies bug: batching eligible rich/media/text into the Reply API before consuming the token is the right LINE-plugin-local fix. The remaining question is maintainer acceptance of rich/media-first overflow ordering plus live-proof sufficiency.
