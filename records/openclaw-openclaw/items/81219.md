## What This Changes

The PR adds `suppressPlanLog` to migration options, passes it from post-install migration, skips formatted plan logging when set, and moves Codex selection prompt cursors to the first selectable item.

## Best Possible Solution

Scope plan-log suppression to Codex handoffs that still show an equivalent selection UI, or add a provider-independent preview before any suppressed apply confirmation.

## Reproduction Assessment

Yes. Source inspection shows a non-Codex accepted candidate would receive `suppressPlanLog`, skip Codex-only selection, and then reach `Apply this migration now?` without the formatted plan preview.

## Solution Assessment

No. Suppressing the duplicate Codex wizard plan is reasonable, but doing it unconditionally in the generic helper is broader than the stated UX fix and weakens the migration preview contract.
