## Summary

Keep open. This is a protected maintainer-labeled feature PR with useful proof and a maintainer comment supporting the selection-picker direction, but the latest diff still broadens plan-log suppression to every post-install migration provider and leaves the public migrate docs behind the new picker behavior.

## What This Changes

The PR adds an internal `suppressPlanLog` migration option for onboarding and adds an `Accept recommended` default row to Codex migration skill/plugin pickers with related tests.

## Best Possible Solution

Keep the `Accept recommended` affordance if maintainers agree, but preserve preview-first safety for non-Codex migrations and update the public migrate docs for the new selector layout.

## Reproduction Assessment

Yes, by source inspection for the review blocker. The PR passes `suppressPlanLog: true` from a provider-generic onboarding helper, while non-Codex providers skip the Codex selectors and can proceed to the final apply confirmation without the itemized plan log.

## Solution Assessment

No. The `Accept recommended` picker direction is plausible, but suppressing the plan through the generic onboarding handoff is broader than the Codex UX fix; a Codex-only suppression or equivalent non-Codex preview is safer.
