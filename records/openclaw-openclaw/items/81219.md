## Summary

Keep open: the PR has a protected `maintainer` label and the patch should not merge as-is because the new suppression is applied from a provider-generic onboarding helper, which can remove the only plan preview for non-Codex migration providers. The PR body describes a local run, but it does not include inspectable real behavior proof.

## What This Changes

The PR adds a `suppressPlanLog` migrate option and passes it from the post-install migration prompt so the Codex onboarding handoff skips the upfront formatted plan dump.

## Best Possible Solution

Keep plan suppression scoped to Codex paths that still show an equivalent item-selection UI, or add a provider-independent reviewed preview before any apply confirmation when the full plan dump is suppressed.

## Reproduction Assessment

Yes. Source inspection shows the accepted helper path is provider-generic, while only Codex gets selection prompts; a non-Codex accepted migration would skip the formatted plan and then ask for apply confirmation.

## Solution Assessment

No. The requested suppression is reasonable for the Codex wizard handoff, but applying it unconditionally in the generic helper is broader than the stated problem and weakens the migrate preview contract.
