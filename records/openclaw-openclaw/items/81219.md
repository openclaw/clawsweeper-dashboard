## Summary

Keep open: this is a protected-label feature PR with useful maintainer discussion and sufficient behavior proof, but the latest patch has blocking source-visible issues before it can merge.

## What This Changes

The PR adds an internal migrate plan-log suppression option for onboarding and changes Codex migration multiselects to start on an `Accept recommended` row.

## Best Possible Solution

Keep the `Accept recommended` UX if maintainers agree, restore or deliberately replace the documented skip semantics, scope plan-log suppression so non-Codex providers still get an itemized preview, and update the migrate docs.

## Reproduction Assessment

Yes. Source inspection of the PR head is enough to reproduce the main blocker: the branch removes the skip sentinel export while changed tests still import and use it, and the generic suppression path is visible in the onboarding helper.

## Solution Assessment

No. The feature direction has maintainer support, but this implementation currently breaks imports, drops documented skip behavior, and suppresses itemized previews too broadly.
