## Summary

Keep open: the PR has the protected `maintainer` label and the latest diff still has a merge-blocking behavior issue where a provider-generic onboarding path can suppress the itemized migration preview for non-Codex providers.

## What This Changes

The PR adds an internal migrate plan-log suppression option for onboarding and an `Accept recommended` row/default cursor behavior for Codex migration selectors, with focused tests.

## Best Possible Solution

Keep the `Accept recommended` UX if maintainers agree, but scope plan-log suppression to the Codex onboarding duplicate-log case or provide an equivalent itemized preview for every provider, then update the public migrate docs.

## Reproduction Assessment

Yes. Source inspection is enough for the review blocker: the PR passes `suppressPlanLog` from a provider-generic onboarding helper, while non-Codex providers skip the Codex selectors and can proceed to the final apply confirmation without the itemized plan log.

## Solution Assessment

No. The recommended-selection affordance is plausible and has maintainer product support, but generic plan suppression is broader than the Codex onboarding duplicate-preview problem; a Codex-scoped suppression or non-Codex preview is safer.
