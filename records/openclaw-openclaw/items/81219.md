## Summary

Keep open: the earlier Codex review still applies. The PR needs inspectable real behavior proof and has a blocking preview regression because `suppressPlanLog` is passed from a generic post-install migration helper while non-Codex providers do not show an alternate selection preview.

## What This Changes

The branch adds an internal `suppressPlanLog` migration option, uses it from the post-install migration prompt, removes custom Enter reconciliation from the Codex migration multiselect, and adjusts related tests.

## Best Possible Solution

Scope plan-log suppression to Codex handoffs that still show equivalent selection UI, or add a provider-independent preview before apply; update the migration docs for the new Enter behavior and attach real runtime proof.

## Reproduction Assessment

Yes, by source inspection. A freshly installed non-Codex migration provider can be accepted in `offerPostInstallMigrations`, receive `suppressPlanLog: true`, bypass the Codex selectors, and reach `Apply this migration now?` without `formatMigrationPlan` output.

## Solution Assessment

No. Suppressing the duplicate Codex plan is reasonable, but doing it unconditionally in the generic helper is broader than the UX fix and weakens preview-first behavior for Claude and Hermes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Keep non-Codex migration previews visible:** `src/wizard/setup.post-install-migration.ts:153`
  - body: The post-install helper can offer any provider owned by a freshly installed plugin, but this option suppresses the formatted plan for all of them. For providers such as Claude or Hermes, the Codex selector returns immediately, so after accepting the onboarding prompt the user reaches `Apply this migration now?` without the itemized preview promised by the migration safety docs. Gate suppression to Codex or show an equivalent preview before apply.
  - confidence: 0.9
- **[P3] Update docs for the new Enter contract:** `src/commands/migrate/skill-selection-prompt.ts:172-181`
  - body: This changes Enter from activating the highlighted row to submitting the current visual selection, but `docs/cli/migrate.md` still tells users to press Enter to activate the highlighted row and continue. Update that section to describe Space-then-Enter for sentinel rows and Enter as submit.
  - confidence: 0.8
