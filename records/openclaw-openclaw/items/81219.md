## Summary

Keep open: this is an active protected-label PR, and the latest head fixed the earlier missing-export problem, but one concrete safety issue remains around suppressing itemized migration previews from the generic onboarding helper.

## What This Changes

The PR adds an internal migrate plan-log suppression option, changes Codex migration pickers to start on an Accept recommended row, removes the Skip for now sentinel, and updates related tests.

## Best Possible Solution

Keep the Accept recommended UX if maintainers want it, but preserve an itemized preview or tightly scope suppression so embedded onboarding does not hide migration plans for providers without equivalent selection detail.

## Reproduction Assessment

Yes. Source inspection is enough: the PR passes suppressPlanLog from the generic post-install migration helper, and migratePlanCommand then skips the formatted plan output before the final apply prompt.

## Solution Assessment

No. The selector change appears aligned with the maintainer discussion, but the generic plan suppression is broader than the current preview-first migration contract and should be narrowed or paired with an embedded preview.
