## Summary

Keep open: this is an active maintainer-labeled PR with useful migration UX work, but the branch still has a preview-safety regression at the provider-generic post-install migration helper and leaves the public Enter-key docs stale.

## What This Changes

The PR adds an internal migration option to suppress the formatted plan log from onboarding, removes custom Enter reconciliation from the Codex migration multiselect, and updates related tests.

## Best Possible Solution

Keep the Enter-as-submit behavior if maintainers agree, but preserve preview-first migration safety for non-Codex providers and update the public migrate docs.

## Reproduction Assessment

Yes, by source inspection for the review blocker. A non-Codex candidate accepted by `offerPostInstallMigrations` receives `suppressPlanLog: true`, skips Codex selectors, and can reach `Apply this migration now?` without the itemized preview.

## Solution Assessment

No. Suppressing the duplicate Codex plan is reasonable, but applying it through the generic helper is broader than the UX fix; the safer solution is Codex-only suppression or an equivalent preview for all providers, plus a docs update.
