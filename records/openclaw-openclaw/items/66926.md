## Summary

Keep open. Current main still routes direct Google/Gemini reasoning output to the tagged `<think>/<final>` contract, still injects the reported “everything else is discarded” prompt language, and still uses separate native Google function-calling transport, so the reported conflict remains actionable rather than implemented or superseded.

## What This Changes

The issue requests removing or revising Google tagged reasoning so Gemini/Gemma models invoke native tool calls instead of writing tool calls as text.

## Best Possible Solution

Fix the provider-owned reasoning-output path so direct Google Gemini/Gemma transports with native thinking and function calling do not receive a tagged-only text contract, while preserving any Google CLI or replay behavior that still genuinely needs tags. Add regression tests around Google reasoning mode resolution, prompt composition with tools, and existing Google `functionDeclarations`/`functionCall` transport behavior.

## Reproduction Assessment

Yes for the code-level failure path: current source and tests still show Google direct/replay-family paths resolving to tagged, the embedded runner injecting the tagged prompt, and the Google transport separately sending/parsing native function calls. A live model run would still be useful for the Gemma 4-specific follow-up.

## Solution Assessment

Yes, a narrow provider-owned fix is the best direction. Prefer changing the direct Google API reasoning-output mode or explicitly exempting native function calls from the tagged text contract, rather than adding core owner-specific migrations or parsing textual tool-call prose into real calls.
