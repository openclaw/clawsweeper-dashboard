## Summary

Keep open. Current main still maps Google direct/Gemini provider reasoning output to the tagged `<think>/<final>` contract, still injects the reported “everything else is discarded” wording, and still sends Google tools through native `functionDeclarations`, so the tool-calling conflict remains actionable.

## What This Changes

Stop Google Gemini/Gemma reasoning tag prompts from making tool-capable models write tool calls as text instead of using native function calls.

## Best Possible Solution

Fix the provider-owned reasoning-output path so direct Google Gemini/Gemma transports that use native thinking and native function calling no longer receive a tagged-only text contract, while preserving tagged behavior only where it is genuinely required.

## Reproduction Assessment

Yes for the code-level path: current source still resolves Google direct/Gemini providers to tagged reasoning, injects the reported prompt wording, and sends tools separately through native Google function declarations. A live model run would be useful for the Gemma 4 follow-up, but the source conflict is clear.

## Solution Assessment

Yes, a narrow provider-owned fix is the best direction. Prefer changing Google direct/Gemini reasoning output mode or explicitly carving native function calls out of the tagged text contract, not parsing textual tool-call prose into real calls.

Keep the fix provider-owned: adjust `google-generative-ai` / bundled Google Gemini reasoning-output resolution or the tagged prompt contract so native function calls are not treated as discarded text.
