## Summary

Keep open. Current main still resolves direct Google/Gemini provider paths to tagged reasoning, still injects the reported final-tag prompt text, and still relies on native Google function-calling plumbing for tools; the related Gemini thought-signature work fixes a different Google tool-calling failure and does not supersede this report.

## What This Changes

The issue requests removing or revising Google tagged reasoning so Gemini/Gemma models invoke native tool calls instead of writing tool calls as text.

## Best Possible Solution

Fix the provider-owned reasoning-output path so direct Google Gemini/Gemma transports with native thinking and function calling do not receive a tagged-only text contract, while preserving any Google CLI or replay behavior that still genuinely needs tags. Add regression tests around Google reasoning mode resolution and prompt composition with tools, and keep existing Google functionDeclaration/functionCall transport tests green.

## Reproduction Assessment

Yes for the code-level failure path: current tests and source show google-generative-ai resolves to tagged, the embedded runner injects the <think>/<final> prompt, and the direct Google transport separately sends native function declarations. A live model run would still be useful for the Gemma 4 model-specific follow-up.

## Solution Assessment

Yes, a narrow provider-owned fix is the maintainable direction. Prefer changing Google direct API reasoning-output mode or explicitly exempting native function calls from the tagged text contract, rather than adding core owner-specific migrations or post-hoc parsing textual tool-call prose into real calls.
