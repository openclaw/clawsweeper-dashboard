## Summary

Keep open. Current main still applies the tagged `<think>/<final>` reasoning contract to direct Google Gemini/Gemma paths while also sending native Google function declarations, so the reported prompt/tool-calling conflict remains actionable.

## What This Changes

Prevent Google/Gemini reasoning tag prompts from causing native Google tool calls to be emitted as literal text instead of function calls.

## Best Possible Solution

Fix the provider-owned reasoning-output path so direct Google Gemini/Gemma transports with native thinking and function calling do not receive a tagged-only text contract, while preserving tagged behavior for any Google replay or CLI surface that still needs it.

## Reproduction Assessment

Yes for the code-level failure path: current source and tests still show Google direct/provider paths resolving to tagged, the runner injecting the reported prompt, and the Google transport separately sending/parsing native function calls. A live model run is optional follow-up for the Gemma-specific behavior noted by the reporter.

## Solution Assessment

Yes, a narrow provider-owned fix is the best direction. Prefer changing Google reasoning-output mode selection or explicitly exempting native function calls from the tagged text contract, rather than parsing textual tool-call prose into real calls.
