## Summary

Keep open. Current main still routes direct Google/Gemini reasoning output to the tagged `<think>/<final>` contract while the Google transport separately sends and parses native function calls, so the reported prompt/tool-call conflict remains actionable.

## What This Changes

The issue requests fixing Google/Gemini reasoning prompt handling so tagged `<think>/<final>` instructions do not cause models to write tool calls as text instead of invoking native function calls.

## Best Possible Solution

Fix the provider-owned reasoning-output path so direct Google/Gemini transports with native thinking and function calling do not receive a tagged-only text contract, while preserving separate Gemini CLI or replay sanitation behavior that still needs tags.

## Reproduction Assessment

Yes. The issue provides concrete live steps, and current source gives a high-confidence code-level reproduction path: Google resolves to tagged, the embedded runner injects the final-tag prompt, and the Google transport expects native `functionCall` parts for tool execution.

## Solution Assessment

Yes. A narrow provider/prompt fix is the best direction; avoid core owner-specific migrations or parsing textual tool-call prose into real calls.
