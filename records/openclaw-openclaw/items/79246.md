## Summary

Keep open: current main still has a source-level token scope widening path, and the linked open fix PR https://github.com/openclaw/openclaw/pull/79206 uses closing syntax for this issue. The issue should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue requests preserving deliberately narrowed device role-token scopes when a pairing scope upgrade is approved.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/79206 or an equivalent narrow fix that keeps the approved baseline as the full union while minting role tokens from the existing token scopes plus only newly requested deltas, with regression tests and the existing changelog entry.

## Reproduction Assessment

Yes, at source level: current main widens the approved baseline from existing plus pending scopes, while token minting returns the pending role-scoped list directly when scopes are present. I did not execute a live pairing flow in this read-only review.

## Solution Assessment

Yes, the linked PR direction is the narrow maintainable fix: preserve the separate approved baseline contract but compute newly minted token scopes from the current token plus the upgrade delta. The issue should remain open while that PR is reviewed and landed.
