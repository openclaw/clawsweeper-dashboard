## Summary

Keep open: current main still has the reported token-scope upgrade path, and the linked closing PR is open and unmerged, so this issue should not be closed before that implementation lands.

## What This Changes

Prevent device-pairing scope-upgrade approval from regranting scopes that were deliberately omitted from the current role token.

## Best Possible Solution

Land the linked PR or an equivalent narrow change in the generic device-pairing approval helper so minted role tokens preserve current narrowed scopes plus newly approved deltas while the approved baseline records the full approval.

## Reproduction Assessment

Yes, from source inspection. Current main’s resolver returns non-empty pending role scopes directly and approval writes those resolved scopes into the replacement role token, while existing tests document that narrowed token scopes and the approved baseline are separate.

## Solution Assessment

Yes, the linked PR’s direction is the narrow maintainable fix because it changes the shared device-pairing token-scope resolver and adds focused regression tests. The issue should stay open until that PR or an equivalent fix merges.
