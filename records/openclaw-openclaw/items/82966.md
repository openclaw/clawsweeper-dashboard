## Summary

Keep open: the reported display/session-state bug is valid, but this branch is not merge-ready because the global helper semantic change can break stale-session preflight compaction and the PR still lacks patched-run real behavior proof.

## What This Changes

The PR makes `resolveFreshSessionTotalTokens` return preserved stale `totalTokens`, adds freshness checks in memory/fork/session-row consumers, updates session/status/gateway tests, and adds a changelog entry.

## Best Possible Solution

Expose preserved totals to display/session-list consumers while keeping fresh-only semantics for compaction, memory flush, fork, and compact paths, with regression coverage for stale low cached totals plus larger transcript estimates.

## Reproduction Assessment

Yes, at source level: current main hides finite preserved totals from `/context` and session-row display when `totalTokensFresh` is false. I did not run a live Gateway/UI reproduction because this review was read-only.

## Solution Assessment

No: the display direction is right, but the PR changes the fresh-only helper globally and misses the agent-runner preflight consumer. The safer fix is to split helper semantics or guard every freshness-sensitive caller before using preserved stale totals.
