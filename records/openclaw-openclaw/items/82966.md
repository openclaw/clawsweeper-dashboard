## Summary

Keep this PR open, but it is not ready to merge: the display-side fix is plausible, yet the patch changes a helper that current main still uses as a fresh-only gate in preflight compaction, and the PR body does not show a patched real environment run.

## What This Changes

The PR changes stale preserved `totalTokens` handling for session status, `/context`, gateway session rows, fork token counts, memory flush gating, tests, and the changelog.

## Best Possible Solution

Keep the preserved-token display fix, but separate display-safe token reads from fresh-only gating or update every freshness-sensitive consumer, including preflight compaction, with regression coverage for both UI/status visibility and compaction refresh behavior.

## Reproduction Assessment

Yes, at source level: current main hides stale preserved totals because `/context` and gateway rows call the fresh-only resolver. I did not establish a live current-main or patched-binary reproduction in this read-only review.

## Solution Assessment

No: the direction is maintainable, but changing `resolveFreshSessionTotalTokens` globally is too broad unless all fresh-only consumers are updated. A safer fix would add/use a preserved-total resolver for display paths or explicitly guard every freshness-gated runtime path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Keep preflight compaction from trusting stale totals:** `src/config/sessions/types.ts:553-556`
  - body: Changing `resolveFreshSessionTotalTokens` to return stale preserved values also changes `agent-runner-memory.ts`, where preflight compaction uses this helper to decide whether to re-read transcript usage. With a stale entry, `freshPersistedTokens` becomes a number, `transcriptUsageTokens` is skipped, and compaction can run from an old lower token count instead of refreshing before the threshold check. Please either keep this helper fresh-only and use a separate display resolver, or update the preflight consumer and tests at the same time.
  - confidence: 0.87
