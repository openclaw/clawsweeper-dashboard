## Summary

Keep this PR open: the reported display bug is valid at source level, but the latest head still changes a fresh-only helper globally and leaves preflight compaction able to trust stale preserved totals instead of re-reading the transcript. The PR also still needs after-fix real behavior proof from a patched run before merge.

## What This Changes

The branch returns preserved session totalTokens for stale entries, adds freshness checks in several consumers, updates session/status/gateway tests, and adds a changelog entry.

## Best Possible Solution

Use a preserved-total path for display/status surfaces while keeping preflight compaction, memory-flush, and fork decisions on fresh totals or explicit transcript estimates, with regression coverage for a stale low cached total plus a larger transcript estimate.

## Reproduction Assessment

Yes at source level: current main hides stale preserved totals from /context and status because those display paths call the fresh-only resolver. I did not run a live current-main or patched-binary reproduction in this read-only review.

## Solution Assessment

No: the display direction is right, but changing resolveFreshSessionTotalTokens globally is too broad while agent-runner preflight still treats that helper as fresh-only. The safer fix is either a separate preserved-total resolver for display or explicit freshness guards on every freshness-sensitive consumer, including preflight compaction.
