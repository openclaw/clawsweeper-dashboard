## What This Changes

The PR makes Tool Search config resolution agent/session-aware so local-model lean agents use smaller implicit search result limits, with tests and docs updates.

## Best Possible Solution

Keep the lean implicit defaults, but preserve an explicit `searchDefaultLimit` unless the user explicitly sets a lower `maxSearchLimit`, and add a regression test for that single-key config.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction: on PR head, a lean config with `searchDefaultLimit: 10` and no `maxSearchLimit` resolves through the new lean max of 8. Current main allowed the same explicit default through the normal max of 20.

## Solution Assessment

No, not yet. The resolver is the right layer for lean defaults, but the current implementation should preserve explicit single-key overrides before this is the best fix.
