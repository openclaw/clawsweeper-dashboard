Full review comments:

- **[P2] Guard cache deletion against newer managers:** `extensions/memory-core/src/memory/manager.ts:985`
  - body: `closeMemoryIndexManagersForAgent` removes the cache key before awaiting the old manager close, so a caller can create and cache a replacement while provider close or pending sync is still blocked. When the old close reaches this `finally`, it deletes `this.cacheKey` unconditionally and evicts the replacement, which can recreate duplicate managers/providers after a timeout. Delete only when the cached value still points at this manager.
  - confidence: 0.91
