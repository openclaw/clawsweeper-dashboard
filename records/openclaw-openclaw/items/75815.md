## Summary

Keep open: current main still lacks the stale/auth Gateway fallback guard and gateway.mode rejection, but this PR is not merge-ready because it can create an extra secret-bearing rejected config artifact without explicit private permissions and it lacks changelog coverage for the remaining user-visible guard behavior.

## What This Changes

The PR changes macOS config persistence to reject stale/auth-like Gateway fallback writes, block suspicious direct writes that remove gateway.mode or shrink sharply, surface rejected saves, and add Swift regression tests.

## Best Possible Solution

Secure or redact rejected config artifacts, add a one-line changelog entry for the stale fallback and gateway.mode guard, then re-review the remaining #64973/#74890 scope against the merged #75953 auth-preservation base.

## Reproduction Assessment

Yes. The static reproduction path is current main’s local ConfigStore.save catch path falling back to a direct protected file write after stale/baseHash/auth-like config.set failures, with #64973 and #74890 providing matching runtime symptoms.

## Solution Assessment

No as currently proposed. The guard direction is maintainable, but rejected snapshots must not create extra readable copies of gateway auth material and the branch should account for the auth-preservation work already merged in #75953.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Set private permissions on rejected snapshots:** `apps/macos/Sources/OpenClaw/OpenClawConfigFile.swift:647`
  - body: saveDict can preserve existing gateway.auth into the candidate output before rejecting a suspicious write, then persistRejectedConfigWrite saves that JSON with default Data.write permissions. On hosts with a loose umask or custom state dir, this can create an extra token-bearing config copy that is group/world-readable; write the artifact with 0600-style permissions or redact auth before persisting it.
  - confidence: 0.87
- **[P3] Add changelog coverage for the new guard:** `CHANGELOG.md:213`

  - confidence: 0.9
