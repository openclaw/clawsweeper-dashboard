## Summary

Keep this PR open. The underlying macOS/Gateway config clobber problem is real and the proposed guard is on the right path, but the branch introduces a secret-bearing rejected-config artifact without explicit private permissions and is missing the required user-facing changelog entry.

## What This Changes

The PR updates macOS config save paths to block stale/auth-like Gateway-save fallback writes, preserve gateway auth/mode during direct saves, surface rejected saves in app call sites, and add focused Swift regression tests.

## Best Possible Solution

Land the macOS guard after rejected config artifacts are either written with private 0600-style permissions or stripped of preserved auth, then add an Unreleased Fixes changelog entry. The longer-term product direction can still move macOS UI writes toward conflict-aware config.patch flows.

## Reproduction Assessment

Yes. The static reproduction path is high-confidence: current main catches a stale/invalid Gateway config.set failure in local mode and falls back to a direct full-file save, while the PR’s new test models that stale error and expects the file to remain unchanged.

## Solution Assessment

No, not yet. The fallback guard and auth/mode preservation are the right narrow direction, but the rejected-payload behavior should match the private-permission core writer precedent before this is the best fix.
