## Summary

Keep this PR open: it targets a real remaining macOS config-guard gap, but the current diff is not merge-ready because rejected config snapshots can persist secret-bearing Gateway auth without explicit private permissions, and the extra stale/mode guard behavior still needs changelog coverage.

## What This Changes

The PR updates macOS config persistence to reject stale/auth-like Gateway fallback writes, block suspicious direct writes that remove `gateway.mode` or sharply shrink the config, surface rejected saves, persist rejected payloads, and add focused Swift regression tests.

## Best Possible Solution

Rebase the remaining stale-fallback and `gateway.mode` guard work onto the #75953 implementation, make rejected artifacts private or redacted, and add a single-line changelog entry for the additional macOS config guard behavior.

## Reproduction Assessment

Yes. The static reproduction path is current main’s local ConfigStore.save catch path falling back to a direct protected save after a stale/auth-like `config.set` failure, with #64973 and #74890 providing matching runtime symptoms.

## Solution Assessment

No as currently proposed. The guard direction is narrow and useful, but rejected snapshots must be secured or redacted before merge and the branch should avoid duplicating the auth-preservation work already on main.
