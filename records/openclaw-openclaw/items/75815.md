## Summary

Keep this PR open. It addresses a real macOS config-write failure mode on current main, but the proposed rejected-payload path can create an extra secret-bearing config artifact with default file permissions, and the user-facing fix is missing the required changelog entry.

## What This Changes

The PR adds macOS ConfigStore fallback guards, hardens direct config saves to preserve gateway auth and reject gateway-mode/size-drop clobbers, surfaces rejected saves in the app UI, and adds focused Swift regression tests.

## Best Possible Solution

Land the macOS guard after rejected config artifacts use private 0600-style permissions or omit preserved auth from the debug artifact, then add the required Unreleased Fixes changelog entry. The longer-term direction remains migrating macOS UI writes toward conflict-aware config.patch flows.

## Reproduction Assessment

Yes. The static reproduction path is high-confidence: current main catches a stale/invalid config.set failure in local mode and falls back to a direct full-file save, while the PR’s new ConfigStore test models that stale error and asserts the config file remains unchanged.

## Solution Assessment

No. The fallback guard and auth/mode preservation are the right direction, but the proposed implementation should not introduce a loose-permission rejected snapshot for secret-bearing config data; matching the core writer’s 0600 artifact behavior is the safer fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Set private permissions on rejected config snapshots:** `apps/macos/Sources/OpenClaw/OpenClawConfigFile.swift:610`
  - body: This rejected payload can include `gateway.auth` because `saveDict` preserves existing auth before calling `persistRejectedConfigWrite`. `Data.write(to:options: [])` creates the new `.rejected.*` file with the process umask, unlike the core config writer which uses mode `0600`, so a rejected macOS save can leave an extra token-bearing config copy group/world-readable on hosts with a loose umask. Create it with private permissions or chmod it before returning.
  - confidence: 0.87
