## Summary

Keep this PR open, but do not merge it as-is. Current main already covers the main auth-preservation path through the merged #75953 fix, while this PR still adds potentially useful stale-fallback and gateway.mode rejection behavior with a secret-handling blocker.

## What This Changes

The PR changes macOS config persistence to block stale/auth-like Gateway fallback saves, reject suspicious direct config writes, surface rejected saves, and add Swift regression tests.

## Best Possible Solution

Rebase the remaining stale-fallback and gateway.mode guard work onto the #75953 implementation, make rejected artifacts private or redacted, and add changelog coverage for the added behavior.

## Reproduction Assessment

Yes. The static reproduction path is current main’s local ConfigStore.save catch path falling back to a protected direct save after a stale/auth-like config.set failure, with #64973 and #74890 providing matching runtime symptoms.

## Solution Assessment

No as currently proposed. The guard direction is reasonable, but the rejected snapshot path must be secured or redacted and the branch should avoid duplicating the auth-preservation work already merged in #75953.
