## Summary

Keep this PR open: it targets a real remaining macOS stale/mode config-write guard, but the current diff adds a rejected-config snapshot path that can persist Gateway auth without explicit private permissions and still needs changelog coverage.

## What This Changes

The PR changes macOS config persistence to block stale/auth-like Gateway fallback writes, reject suspicious direct config writes, surface rejected saves, and add Swift regression tests.

## Best Possible Solution

Rebase on the merged #75953 auth-preservation base, keep the stale-fallback and gateway.mode rejection guard, write rejected artifacts with owner-only permissions or redact auth, and add a user-facing changelog entry.

## Reproduction Assessment

Yes. The static reproduction path is current main's local ConfigStore.save catch path falling back to a protected direct save after a stale/auth-like config.set failure, with #64973 and #74890 providing matching runtime symptoms.

## Solution Assessment

No as currently proposed. The guard direction is narrow and maintainable, but rejected snapshots must be secured or redacted before merge, and the user-visible guard needs changelog coverage.
