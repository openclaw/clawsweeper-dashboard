## Summary

Keep this PR open. It addresses a real macOS/Gateway config clobber path that still exists on current main, but the latest head still has a security-sensitive rejected-config artifact permission bug and lacks the required user-facing changelog entry.

## What This Changes

The PR changes macOS config persistence to avoid direct fallback after stale/auth Gateway config writes fail, preserve `gateway.auth` and `gateway.mode` in protected direct saves, surface rejected saves, and add Swift regression tests.

## Best Possible Solution

Land the macOS guard after rejected config artifacts are private or redacted and an Unreleased Fixes changelog entry is added; keep the longer-term `config.patch` migration as follow-up.

## Reproduction Assessment

Yes. The high-confidence static path is current main’s local `ConfigStore.save` catch block falling back to `OpenClawConfigFile.saveDict(root)` after a stale/auth-like `config.set` failure, and the related reports provide matching runtime logs.

## Solution Assessment

No. The fallback guard and preservation checks are the right direction, but the proposed implementation should first secure or redact rejected config artifacts and add the required changelog entry.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Set private permissions on rejected snapshots:** `apps/macos/Sources/OpenClaw/OpenClawConfigFile.swift:610`
  - body: `saveDict` can preserve existing `gateway.auth` into `output` before rejecting a suspicious write, and `persistRejectedConfigWrite` then saves that JSON with default `Data.write` permissions. On hosts with a loose umask, this can create an extra token-bearing config copy that is group/world-readable. Create the artifact with private `0600`-style permissions or redact auth before persisting it.
  - confidence: 0.87

  - body: This is a user-facing macOS/Gateway config fix, but the PR file list does not include `CHANGELOG.md`. Add a single-line Unreleased `### Fixes` entry without inventing bot or maintainer attribution.
  - confidence: 0.94
