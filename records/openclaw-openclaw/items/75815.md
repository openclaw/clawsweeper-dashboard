## Summary

Keep this PR open for maintainer handling: current main still has the macOS fallback/direct-write risk and this branch targets the right surface, but it should not merge while rejected-config artifacts can expose auth material and the required changelog entry is missing. Related maintainer PR #75953 is an active safer alternative for the auth-preservation part.

## What This Changes

The PR changes macOS config persistence to block stale/auth-like Gateway save fallback, preserve `gateway.auth` during protected direct saves, reject destructive `gateway.mode` or large-drop writes with audit artifacts, surface rejections, and add Swift regression tests.

## Best Possible Solution

Land a macOS config hardening patch that preserves auth and unrelated keys without creating insecure rejected artifacts, adds the Unreleased Fixes changelog entry, and clearly chooses whether #75953 replaces or absorbs the stale-fallback and `gateway.mode` guard behavior.

## Reproduction Assessment

Yes. The high-confidence static path is current `main`'s local `ConfigStore.save` catch block falling back to `OpenClawConfigFile.saveDict(root)` after a Gateway `config.set` error, and the related reports provide matching stale-write, token-mismatch, auth-drop, and mode-drop logs.

## Solution Assessment

No. The branch points at the right macOS persistence boundary, but the implementation should first secure or redact rejected artifacts and add the required changelog entry; #75953 is a lower-risk auth-preservation alternative if maintainers do not want rejected snapshots.
