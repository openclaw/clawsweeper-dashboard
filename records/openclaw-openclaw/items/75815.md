## Summary

Keep this PR open. It addresses a real macOS Gateway config clobber path, but the current patch still adds a secret-bearing rejected-config artifact written with default file permissions, and it is missing the required user-facing changelog entry.

## What This Changes

The branch updates macOS ConfigStore/OpenClawConfigFile saves to block stale/auth-like Gateway fallback writes, preserve gateway auth/mode during direct saves, surface rejected writes in AppState/DebugSettings, and add focused Swift regression tests.

## Best Possible Solution

Land the macOS guard after rejected config artifacts either use private 0600-style creation or omit preserved auth from the debug artifact, then add an Unreleased Fixes changelog entry. The follow-up product direction should still move macOS UI writes toward conflict-aware config.patch flows.

## Reproduction Assessment

Yes. Current main has a static, high-confidence reproduction path: a stale/invalid config.set failure in local mode falls through to a direct full-file save, and the PR’s new ConfigStore test models that stale error and expects the file to remain unchanged.

## Solution Assessment

No. The fallback guard and auth/mode preservation are the right direction, but the patch should not add a default-permission rejected snapshot that can contain Gateway auth; private artifact creation or auth scrubbing is the safer implementation.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Set private permissions on rejected config snapshots:** `apps/macos/Sources/OpenClaw/OpenClawConfigFile.swift:610`
  - body: This rejected payload can include gateway.auth because saveDict preserves existing auth before calling persistRejectedConfigWrite. Data.write(to:options: []) creates the new .rejected file with the process umask, unlike the core config writer's 0600 rejected-artifact behavior, so a rejected macOS save can leave an extra token-bearing config copy group/world-readable on hosts with a loose umask. Create it with private permissions or chmod it before returning.
  - confidence: 0.87
