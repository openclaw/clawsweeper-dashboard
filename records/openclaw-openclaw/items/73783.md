## Summary

Keep open. The PR targets a real subagent announce race, and the compaction marker detection is pointed at a current transcript shape, but the branch still lets a pending compaction surface as a terminal successful completion with `(no output)` because the announce flow treats an undefined reader result as missing content rather than a defer signal.

## What This Changes

The PR adds compaction-marker handling and output-selection tests so subagent announce output suppresses pre-compaction raw tool text while preserving timeout partial-progress output.

## Best Possible Solution

Keep the compaction marker detector, but extend the reader/retry/announce boundary with a typed continuation state so a child whose latest visible history is an auto-compaction marker remains pending until a post-compaction assistant result is available, while true timeouts can still emit partial progress.

## Reproduction Assessment

Yes. A focused code-path reproduction uses an assistant tool call, a tool result, a synthetic compaction system marker, and no later assistant turn; the branch suppresses the raw text to undefined, but the caller still converts missing text into a terminal `(no output)` completion.

## Solution Assessment

No. The detector is useful, but returning undefined is not the best fix because undefined already means no selected output; the safer solution is to carry an explicit defer/continuation result through `readLatestSubagentOutputWithRetry` into `runSubagentAnnounceFlow`.
