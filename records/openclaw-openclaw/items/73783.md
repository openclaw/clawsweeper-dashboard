## What This Changes

The PR adds compaction-marker detection in subagent output selection and tests to suppress raw tool output between auto-compaction and a later assistant answer.

## Best Possible Solution

Keep the canonical fix path in https://github.com/openclaw/openclaw/pull/80110 and close this narrower overlapping branch.

## Reproduction Assessment

Yes, at source level. Current main can select latestRawText from tool/toolResult history before latest-assistant fallback, and compaction markers do not clear that raw state.

## Solution Assessment

No. The compaction detector is in the right area, but this branch still collapses pending compaction to undefined at a caller boundary that can finalize a successful no-output announcement; the broader canonical replacement removes the raw-output fallback with stronger coverage and proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Propagate pending compaction instead of returning no output:** `src/agents/subagent-announce-output.ts:322-324`
  - body: When the newest child history is a compaction marker, this branch makes `readSubagentOutput` return `undefined`. The caller retries through the same `string | undefined` API and can then build a successful `(no output)` completion before the post-compaction assistant answer arrives, so the parent can still miss the real result. Carry a distinct pending/defer state or use the canonical raw-output fallback removal instead of merging this branch.
  - confidence: 0.9
