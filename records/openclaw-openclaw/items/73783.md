## Summary

Keep open. Current main still has the raw-tool fallback that this PR is trying to address, but the PR only suppresses selection inside `readSubagentOutput`; the announce flow still treats an undefined reply as a successful terminal completion with `(no output)`, so the production failure can turn into an empty premature completion rather than being deferred.

## What This Changes

The PR adds compaction-marker handling and unit tests in the subagent announce output reader so pre-compaction raw tool output is not selected as a completed subagent result.

## Best Possible Solution

Carry compaction continuation through the output reader, retry helper, and announce flow as an explicit typed defer state, then keep the child run pending until a post-compaction assistant turn is available or a real timeout should emit partial progress. Preserve the useful marker detection, add flow-level coverage, and coordinate the broader raw-tool fallback policy with #74340.

## Reproduction Assessment

Yes. A focused code-path reproduction is available with an assistant tool call, a toolResult, a synthetic compaction system marker, and no later assistant turn: current main can select raw tool text, while this PR returns undefined and the caller still completes with `(no output)`.

## Solution Assessment

No. The compaction detector is the right lower-level signal, but returning undefined from the reader is not enough because the caller does not distinguish â€śstill waiting for continuationâ€ť from â€śno output available.â€ť
