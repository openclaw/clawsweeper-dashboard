## Summary

Keep this PR open. It targets a real subagent announce race and the compaction-marker detection is pointed at the right transcript shape, but the patch only suppresses selected text; the caller can still emit a terminal successful announce with `(no output)` before the post-compaction assistant answer arrives.

## What This Changes

The PR adds compaction-marker handling to subagent output selection and unit tests so pre-compaction tool output is not returned as the child result after auto-compaction.

## Best Possible Solution

Carry a distinct continuation/defer state from compaction/yield output analysis into `runSubagentAnnounceFlow`, so the announce remains pending or retries until a post-compaction assistant turn or a true timeout is available, while preserving timeout partial-progress behavior and aligning with the raw-tool fallback policy in #74340.

## Reproduction Assessment

Yes. A focused code-path reproduction is available with assistant tool use, a tool result, a synthetic compaction system marker, and no later assistant turn; current main selects raw tool output, while this PR returns no selected output but still lets the caller build a terminal `(no output)` completion.

## Solution Assessment

No. Detecting the compaction marker is the right lower-level signal, but returning `undefined` from the selector is not enough because the caller treats that as an empty final result instead of a deferred run.

- **[P1] Defer instead of announcing no output while compacting:** `src/agents/subagent-announce-output.ts:324-326`
