## Summary

Keep open. The PR targets a real subagent announce race and adds useful compaction-marker detection, but it still turns the compaction limbo window into an undefined reply while the current announce flow can publish a successful `(no output)` completion instead of deferring for the real post-compaction answer.

## What This Changes

The PR adds compaction-marker detection to subagent announce output selection and unit tests so pre-compaction raw tool results are not selected before a post-compaction assistant reply.

## Best Possible Solution

Preserve the compaction marker detection, but carry a typed continuation/defer state through output reading and retry into `runSubagentAnnounceFlow` so a still-compacting child stays pending until a post-compaction assistant reply is available or a real timeout emits partial progress.

## Reproduction Assessment

Yes. A focused code-path reproduction uses an assistant tool call, a tool result, a synthetic compaction system marker, and no later assistant turn; the PR reader returns undefined while the current caller can still finalize that as `(no output)`.

## Solution Assessment

No. The detector is the right lower-level signal, but returning undefined is not enough because the caller treats missing text as empty terminal content rather than a reason to defer and retry later.
