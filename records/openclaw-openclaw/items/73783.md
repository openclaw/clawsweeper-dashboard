## Summary

Keep open. The PR addresses a real subagent announce race and adds useful compaction-marker detection, but it still converts the pending-compaction window into the existing no-content path, so the caller can emit a successful `(no output)` completion before the real post-compaction answer arrives.

## What This Changes

Adds compaction-system-message handling to subagent output selection and unit tests to suppress pre-compaction raw tool output while preserving timeout partial-progress text.

## Best Possible Solution

Carry a typed pending/defer state through the output capture boundary and defer terminal announce until post-compaction assistant text or true timeout partial progress exists, with a changelog entry for the user-facing fix.

## Reproduction Assessment

Yes. A focused code-path reproduction is an assistant tool call, a toolResult, a synthetic compaction system marker, and no later assistant turn; with this PR, the reader returns `undefined` and the unchanged caller can still announce `(no output)`.

## Solution Assessment

No. The compaction detector is the right low-level signal, but returning `undefined` is not enough because the announce flow already treats that as ordinary missing content rather than a pending continuation.
