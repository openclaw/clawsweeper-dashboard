## Summary

Keep open. The PR targets a real current-main announce bug, and the compaction marker shape it detects exists in the session reader, but the branch still converts the compaction limbo window into a terminal successful announce with empty output because the caller has no distinct defer state. Related #74073, #74340, and #75196 show overlapping remaining work around stale/raw/empty subagent completion payloads.

## What This Changes

The PR adds compaction-marker detection to `src/agents/subagent-announce-output.ts` and unit tests so `readSubagentOutput` suppresses pre-compaction raw tool output while preserving timeout partial-progress output.

## Best Possible Solution

Keep the compaction marker detection, but make the output reader or retry layer return a typed continuation/defer result and have `runSubagentAnnounceFlow` leave the run pending until a post-compaction assistant reply is available or a true timeout should emit partial progress.

## Reproduction Assessment

Yes. A focused code-path reproduction is available with assistant tool use, a tool result, a synthetic compaction system marker, and no later assistant turn: the PR makes the reader return undefined, while the caller still emits a successful `(no output)` completion after retry.

## Solution Assessment

No. The low-level detector is useful, but returning undefined is not the best fix because the caller treats undefined as missing terminal content rather than as an instruction to defer delivery.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Defer completion while compaction is still pending:** `src/agents/subagent-announce-output.ts:324-326`
  - body: When the newest child history is the compaction marker, the new logic suppresses raw tool text by returning `undefined`, but the caller still turns that into a successful completion with `(no output)`. In the reported race, the real assistant answer arrives later, so the parent would still receive a terminal completion before the actual result and no later announce would fire. Propagate a distinct continuation/defer state to the announce flow or keep the run pending until a post-compaction assistant turn or true timeout is available.
  - confidence: 0.87
