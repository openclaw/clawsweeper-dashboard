## Summary

Keep open. The PR targets a real abort-cleanup bug, but the proposed fix is incomplete because final delivery still consumes collected `assistantTexts`, and the PR has no after-fix real behavior proof.

## What This Changes

The PR adds abort-path cleanup in `src/agents/pi-embedded-runner/run/attempt.ts` to remove trailing assistant messages with `stopReason: "aborted"` from live session state and SessionManager entries.

## Best Possible Solution

Implement abort cleanup at the runner/subscription boundary so persisted aborted messages and already-collected assistant payloads cannot be returned, then add focused regression coverage and real runtime proof.

## Reproduction Assessment

No high-confidence live reproduction from this review. The source path is clear enough to reproduce conceptually: a runner abort can leave collected assistant text, `buildEmbeddedRunPayloads` still consumes it, and aborted attempts bypass the incomplete-turn replacement path.

## Solution Assessment

No. The patch is directionally right, but stripping only persisted session entries is not the best fix because delivery also depends on subscription-collected `assistantTexts`.
