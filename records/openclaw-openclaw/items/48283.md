## Summary

Keep open: the linked bug still has a current-main source reproduction, but this branch is not merge-ready because it misses the gateway abort-buffer path and lacks real behavior proof.

## What This Changes

The PR adds abort-error cleanup to strip trailing `stopReason: "aborted"` assistant messages from the embedded runner session and session manager.

## Best Possible Solution

Revise or replace this PR with a gateway-level abort buffer gate that prevents stale or invalid aborted content from being broadcast or persisted while preserving or explicitly changing the documented partial-retention contract.

## Reproduction Assessment

Yes for source-level reproduction: current main tests seed `chatRunBuffers` with nonblank text and assert that abort broadcasts or persists assistant content. The exact Telegram/sub2api runtime race was not live-reproduced in this read-only review.

## Solution Assessment

No. Cleaning the embedded runner session tail after `activeSession.prompt()` aborts is too late and too narrow for the user-visible gateway buffer path that broadcasts and persists aborted text.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:
