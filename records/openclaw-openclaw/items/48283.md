## Summary

Keep this PR open, but it is not merge-ready: the patch only cleans the embedded-runner session tail after an abort, while the gateway still broadcasts and persists buffered abort text before that cleanup can address the reported user-visible stale content.

## What This Changes

The PR adds embedded-runner abort cleanup that strips trailing assistant messages with `stopReason: "aborted"` from live session state and session-manager file entries.

## Best Possible Solution

Revise the fix to gate or clear untrusted stale abort buffers before broadcast and transcript persistence, while preserving the documented legitimate partial-retention behavior or explicitly changing that contract with tests and docs.

## Reproduction Assessment

Yes for source-level reproduction: current main code and tests show nonblank `chatRunBuffers` are broadcast and persisted on abort. I did not live-reproduce the exact Telegram/sub2api race in this read-only review.

## Solution Assessment

No. Cleaning the embedded runner session tail after `activeSession.prompt()` aborts is too late and too narrow for the gateway path that delivers and persists buffered abort text.

## Review Findings
