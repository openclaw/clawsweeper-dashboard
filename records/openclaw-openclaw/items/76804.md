## Summary

Keep open: current main still lacks the transcript repair/fallback needed for the reported agent-run final-text loss, and open closing PRs already track the fix path.

## What This Changes

The issue reports that WebChat/agent-run assistant final text can stream to the UI without becoming durable in the session JSONL transcript, causing later `chat.history` reloads to hide prior answers.

## Best Possible Solution

Land #77033 or an equivalent transcript-aware fix that preserves visible assistant final text only when the canonical Pi transcript lacks it, then close this issue through the merged PR.

## Reproduction Assessment

Yes. Source inspection shows the current WebChat agent-run branch does not append final assistant text itself, the UI reloads `chat.history` after tool finals, and the current fixture covers an agent-run final payload with no assistant transcript update.

## Solution Assessment

Yes. The durable fix belongs in the transcript persistence path, not in a UI-only workaround; the safer current direction is the transcript-aware #77033 approach because it can avoid duplicating normal Pi-managed assistant turns.
