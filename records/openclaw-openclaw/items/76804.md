## Summary

Keep open: current main still has the WebChat agent-run transcript persistence gap, and #76819 is an open closing PR for the same bug.

## What This Changes

WebChat agent-run final assistant text is streamed to the UI but not persisted to session JSONL, so history reload can remove earlier assistant replies.

## Best Possible Solution

Land #76819 or an equivalent gateway fix that appends agent-run final assistant text with idempotency and media guards, then close this issue through the merge.

## Reproduction Assessment

Yes. Source inspection shows a `chat.send` run that triggers `onAgentRunStart` reaches the current agent-run completion branch, and the existing fixture demonstrates a final text payload with no assistant transcript update.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the gateway WebChat agent-run persistence path, not in a UI-only workaround; #76819 appears to target that boundary with idempotency and media guards.
