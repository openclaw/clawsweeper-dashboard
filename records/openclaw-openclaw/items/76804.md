## Summary

## What This Changes

Persist durable assistant final text for WebChat agent-run/tool-using turns so `chat.history` reloads and downstream transcript consumers retain the answer.

## Best Possible Solution

Ensure one canonical parent-linked assistant final message is stored per agent-run turn, with regression coverage for tool-use history reload plus media/TTS dedupe.

## Reproduction Assessment

Yes. A source reproduction is available: a `chat.send` path that sets `agentRunStarted` reaches the completion branch that emits only the user transcript update, and the UI then reloads from persisted `chat.history` after tool events.

## Solution Assessment

Yes, the fix belongs in the Gateway/Pi transcript persistence boundary rather than a UI-only workaround. Maintainers should choose the canonical implementation between the open closing PRs, with explicit duplicate-turn protection.
