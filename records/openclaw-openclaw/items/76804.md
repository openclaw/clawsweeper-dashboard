## Summary

Keep open: current main still has the WebChat agent-run assistant-text persistence gap, and #76819 is an open closing PR for the same bug.

## What This Changes

WebChat assistant final text from agent-run replies should be persisted to the session JSONL so history reloads and page refreshes keep the streamed answer.

## Best Possible Solution

Review and land #76819, or an equivalent gateway fix that appends final agent-run assistant text with idempotency and media guards.

## Reproduction Assessment

Yes. A `chat.send` run that triggers `onAgentRunStart` reaches the current agent-run completion branch that emits only the user transcript update, while the fixture at `chat.directive-tags.test.ts` shows final text with no assistant transcript update.

## Solution Assessment

Yes. The narrowest maintainable fix is in the gateway WebChat agent-run transcript persistence path; a UI-only history workaround would not repair the missing JSONL data for history, memory, or compaction consumers.
