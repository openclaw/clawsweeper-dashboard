## Summary

Keep open: current main still skips assistant transcript persistence on the WebChat agent-run completion path, and an open closing PR is already paired with this issue for review.

## What This Changes

Persist WebChat assistant final text from agent-run responses into session JSONL so `chat.history` reload keeps prior assistant replies.

## Best Possible Solution

Land a focused gateway fix that appends agent-run final text exactly once with regression coverage for text-only and stale-media cases, then let the linked PR close this issue.

## Reproduction Assessment

Yes, source-reproducible: a `chat.send` run that triggers `onAgentRunStart` and then sends a final text payload reaches the current agent-run completion branch, which only emits the user transcript update. The existing stale-media fixture also asserts that no assistant update is emitted for an agent-run final text payload.

## Solution Assessment

Yes in direction: the narrow maintainable fix is to persist final delivered assistant text in the agent-run branch using the existing transcript append helper and idempotency. The implementation should avoid a second UI broadcast and preserve the existing media/TTS duplicate guards.
