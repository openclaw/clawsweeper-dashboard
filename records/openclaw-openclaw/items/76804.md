## Summary

Keep open: this is still an active transcript-persistence report with an open closing PR, and current main has since documented/tested that normal agent-run assistant turns should be persisted by Pi rather than mirrored by WebChat live delivery.

## What This Changes

The report asks OpenClaw to persist WebChat/agent-run assistant final text in session JSONL so `chat.history` reloads do not remove streamed answers.

## Best Possible Solution

Resolve the linked PR against the current Pi-owned transcript model, then close this issue only after durable agent-run assistant text is proven on main or the remaining loss is moved to a narrower canonical issue such as #77012.

## Reproduction Assessment

Unclear. Source inspection confirms WebChat does not append normal agent-run final text itself and history reload can expose missing transcript data, but current main defines that omission as intentional because Pi should persist assistant `message_end`; I did not establish a current-main live transcript failure.

## Solution Assessment

Unclear. A direct WebChat mirror could address the symptom, but current docs/tests say Pi's SessionManager is the canonical owner for normal assistant turns, so the maintainable fix should either repair that owner path or add only a narrowly guarded WebChat supplement.
