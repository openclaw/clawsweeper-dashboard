## Summary

Keep open: current main still skips plain assistant transcript persistence on the WebChat agent-run completion path, while chat history reloads use the persisted transcript as the source of truth.

## What This Changes

Persist assistant final text from WebChat agent-run responses into session transcripts so `chat.history` reloads do not remove prior assistant messages.

## Best Possible Solution

Share the WebChat assistant transcript append logic with the agent-run completion path, guarded by idempotency and media/TTS dedupe, with focused regression coverage for plain final text and stale-media final text.

## Reproduction Assessment

Yes, source-reproducible. A `chat.send` run that fires `onAgentRunStart` and then delivers final text reaches the agent-run completion branch, which currently emits only the user transcript update.

## Solution Assessment

Yes on the persistence boundary, but the proposed fix should append transcript state without rebroadcasting duplicate final UI text. The narrow maintainable repair is server-side persistence plus regression tests rather than a UI-only history merge workaround.
