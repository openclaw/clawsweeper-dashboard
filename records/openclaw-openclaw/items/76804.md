Keep open: current main still has the WebChat agent-run completion gap, and #76819 is an open closing PR for the same transcript persistence bug.

## What This Changes

Persist WebChat agent-run final assistant text to the session transcript so `chat.history` reloads do not drop streamed answers.

## Best Possible Solution

Land #76819 or an equivalent gateway fix that appends agent-run final assistant text exactly once, with regression coverage for text-only and stale-media final payloads.

## Reproduction Assessment

Yes, source-reproducible. A `chat.send` path that triggers `onAgentRunStart` reaches the current completion branch that emits only the user transcript update, and the existing fixture delivers final text while expecting no assistant transcript update.

## Solution Assessment

Yes, the narrow maintainable fix is server-side WebChat transcript persistence with idempotency and media guards. A UI-only history merge would not repair the missing session JSONL data for downstream transcript consumers.
