## Summary

Keep open: current main still has a WebChat agent-run completion branch that only emits the user transcript update, while the non-agent path builds final reply payloads and appends an assistant transcript message.

## What This Changes

WebChat agent-run replies should persist assistant final text/content into the session transcript so chat.history reloads do not drop prior responses.

## Best Possible Solution

Share the assistant transcript append logic with the agent-run completion path, guarded by idempotency, media, and reasoning dedupe, and cover it with a WebChat agent-run regression test.

## Reproduction Assessment

Yes, source-reproducible. A chat.send run that calls onAgentRunStart and then delivers final text reaches the agentRunStarted completion branch, which currently emits only the user transcript update; the existing test fixture also asserts no assistant update for an agent-run final text payload with stale media.

## Solution Assessment

Partly yes. Persisting final deliveredReplies in the agent-run branch is the right boundary, but the implementation should be persistence-focused and avoid re-broadcasting or duplicating the already streamed UI text.
