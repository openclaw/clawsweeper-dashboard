## Summary

Keep open: current main still has the WebChat agent-run completion gap, and an open closing PR (#76819) already targets the same transcript persistence bug.

## What This Changes

Assistant final text from WebChat agent-run responses should be persisted to the session JSONL transcript so history reloads retain the streamed answer.

## Best Possible Solution

Persist agent-run final assistant text in the gateway transcript path with idempotency and media guards, plus regression coverage for text-only and stale-media agent finals.

## Reproduction Assessment

Yes, source-reproducible. A `chat.send` path that triggers `onAgentRunStart` reaches the current branch that emits only the user transcript update, while the existing fixture delivers final text and expects no assistant transcript update.

## Solution Assessment

Yes. The narrow maintainable fix is in the gateway agent-run persistence path, not a UI-only workaround; #76819 appears to target that boundary and should be reviewed or replaced before this issue closes.
