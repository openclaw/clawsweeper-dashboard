## Summary

Keep open: current main still shows the WebChat agent-run completion gap, and #76819 is an open closing PR for the same transcript persistence bug.

## What This Changes

Persist assistant final text from WebChat agent-run turns into the session transcript so `chat.history` reloads do not drop streamed responses.

## Best Possible Solution

Review and land #76819 or an equivalent gateway fix that appends agent-run final text with idempotency and media guards, then close this issue through the merge.

## Reproduction Assessment

Yes, source-reproducible. A `chat.send` run that triggers `onAgentRunStart` reaches the agent-run completion branch that only emits the user transcript update, and the current fixture still asserts no assistant transcript update for an agent-run final text payload.

## Solution Assessment

Yes, the best fix boundary is the gateway WebChat agent-run persistence path. A UI-only history merge would hide the symptom but leave JSONL transcripts incomplete for reload, memory, and downstream consumers.
