## Summary

Close as implemented on current main: the Codex app-server transcript mirror now writes assistant messages from `messagesSnapshot` into the session JSONL, so WebChat agent-run final text no longer depends on the `chat.send` agent branch appending it separately.

## What This Changes

The issue requests that WebChat persist assistant final text for agent-run responses into the session JSONL transcript so history reloads do not make prior replies disappear.

## Best Possible Solution

Keep the current Codex app-server transcript mirror as the persistence boundary and let the next release carry it, avoiding a second WebChat-side append that could duplicate already streamed UI output.

## Reproduction Assessment

No for current main. The reported v2026.5.2 failure path is explained by missing assistant persistence, but current main appends assistant messages from the Codex `messagesSnapshot` into the transcript before WebChat history reloads.

## Solution Assessment

Yes. Persisting the agent runtime’s mirrored assistant message is the narrowest maintainable solution because it writes the source transcript once and avoids rebroadcasting or duplicating streamed WebChat text.
