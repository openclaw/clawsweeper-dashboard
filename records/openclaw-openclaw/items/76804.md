## Summary

Keep open: current main still has the agent-run completion branch that only emits the user transcript update, and #76819 is an open closing PR rather than a merged fix.

## What This Changes

WebChat should persist final assistant text from agent-run responses into the session JSONL so history reloads keep streamed answers.

## Best Possible Solution

Land #76819 or an equivalent gateway fix that appends final agent-run assistant text with idempotency and media guards, then close this issue through the merge.

## Reproduction Assessment

Yes. A `chat.send` turn that triggers `onAgentRunStart` reaches the current `agentRunStarted` completion branch, which only emits the user transcript update while the existing fixture demonstrates final text with no assistant transcript update.

## Solution Assessment

Yes. The narrowest maintainable fix is gateway agent-run transcript persistence; a UI-only history workaround would not repair the missing JSONL data for history, memory, or other transcript consumers.
