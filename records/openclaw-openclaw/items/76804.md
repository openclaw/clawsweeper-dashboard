## Summary

Keep open: current main still has the agent-run transcript persistence gap, and #76819 is already the open closing PR for the same bug.

## What This Changes

Persist WebChat agent-run assistant final text into the durable session transcript so `chat.history` reloads do not drop prior replies.

## Best Possible Solution

Land #76819 or an equivalent gateway fix that appends final agent-run assistant text with idempotency and media/TTS guards, then close this issue through the merge.

Yes, source-reproducible. A `chat.send` run that triggers `onAgentRunStart` reaches the current `agentRunStarted` completion branch, which emits only the user transcript update while the existing fixture demonstrates final text with no assistant transcript update.

## Solution Assessment

Yes, the best fix boundary is the gateway WebChat agent-run persistence path rather than a UI-only workaround. #76819 appears to target that boundary with idempotency and media guards and should be reviewed or replaced before closing this issue.
