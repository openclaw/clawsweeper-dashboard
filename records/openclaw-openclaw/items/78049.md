## Summary

Keep open: current main still supports a launchd-managed Gateway path, and the code/docs do not provide the requested macOS TCC-safe lifecycle alternative for general local CLI/file workflows. The existing `system.run` app-broker path and prior iMessage-focused fixes help adjacent cases, but this issue remains broader and has protected product/security-review labels that require maintainer handling.

## What This Changes

The issue asks OpenClaw to provide a reliable macOS Gateway startup or broker path for launchd-managed local CLI/file workflows that need access to TCC-protected folders such as Documents, Downloads, and Desktop.

## Best Possible Solution

Keep one canonical macOS TCC lifecycle decision open: either make the official startup path own TCC permissions through a signed app/helper, route local CLI/file actions through an authorized app broker by default, or document and support a Terminal-backed mode with live macOS proof.
