## Summary

Keep open: current main and v2026.6.10 still support the macOS local Gateway through a launchd-managed external Node/CLI process, while the remaining issue is the unresolved TCC owner for protected-folder local file and CLI work.

## What This Changes

The issue requests a supported macOS Gateway startup or broker model that can access user-authorized TCC-protected folders without relying on a bare launchd-managed Node process.

## Best Possible Solution

Choose one supported macOS TCC owner for protected-folder local file and CLI work, then align launchd, native app/helper or Terminal-backed mode, doctor/onboarding diagnostics, docs, upgrade behavior, and live protected-folder proof around that model.

## Reproduction Assessment

Yes at source level and with fresh external confirmation: current main still exposes the LaunchAgent-managed external Node Gateway path, and the issue gives concrete macOS TCC steps plus a June 26, 2026 local confirmation. I did not run an independent live macOS protected-folder reproduction in this read-only review.

## Solution Assessment

Unclear on the exact implementation: the current launchd Node path is not the full fix, and the best shape needs maintainer choice between a signed app/helper, app-brokered execution expansion, or an official Terminal-backed startup mode. A narrow bundled-Node preference is only a partial mitigation.
