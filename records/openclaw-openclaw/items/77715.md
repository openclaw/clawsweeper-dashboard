## Summary

This is a real-looking Honcho plugin bug, but the failing session ID helper is in the external @honcho-ai/openclaw-honcho package, not in OpenClaw core, and current OpenClaw already exposes the hook context needed for that plugin to normalize its own Honcho session IDs.

## What This Changes

The issue asks for Honcho session IDs derived from long subagent and cron session keys to stay within Honcho's 100-character limit.

## Best Possible Solution

Track and fix the length-preserving session ID mapping in the external Honcho plugin while keeping OpenClaw core's generic hook contract unchanged.

## Reproduction Assessment

Yes. Source inspection shows OpenClaw can pass long cron/subagent session keys through plugin hook context, while the external Honcho plugin's buildSessionKey can exceed the Honcho SDK's 100-character SessionIdSchema limit.

## Solution Assessment

Yes, but the narrow maintainable fix belongs in @honcho-ai/openclaw-honcho, where OpenClaw session keys are mapped onto Honcho session IDs; OpenClaw already supplies the needed sessionKey and messageProvider context.
