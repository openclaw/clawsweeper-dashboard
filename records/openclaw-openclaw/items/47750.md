## Summary

Keep open: current main still has no non-interactive OpenAI Codex OAuth setup path. The supported automation path is API-key based, while Codex OAuth/device-code remain interactive flows, and adding an env/callback import would be a new security-sensitive provider auth surface that needs maintainer design.

## What This Changes

The issue requests a non-interactive OpenAI Codex OAuth setup path, proposed as an environment-supplied callback URL for ChatGPT subscription auth.

## Best Possible Solution

Design an OpenAI-plugin-owned non-interactive Codex auth contract that preserves state/PKCE validation, avoids leaking callback material, documents the chosen CLI/env behavior, and keeps broader prompt-lifecycle work separate.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection gives a high-confidence current-behavior check that non-interactive provider setup requires runNonInteractive and the OpenAI Codex OAuth/device-code methods do not implement it.

## Solution Assessment

Unclear: accepting a full callback URL from env is one possible interface, but maintainers should first choose a leak-resistant provider-owned OAuth code/state handling contract.
