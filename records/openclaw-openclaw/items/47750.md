## Summary

Keep open: current main still has no non-interactive OpenAI Codex OAuth setup path. The existing implementation supports interactive browser OAuth, interactive device-code auth, and non-interactive API-key setup, while the requested callback/env flow would add a new security-sensitive auth and config surface that needs maintainer product/security design.

## What This Changes

Request a non-interactive OpenAI Codex OAuth setup path, proposed as an environment-supplied callback URL for ChatGPT/Codex subscription auth.

## Best Possible Solution

Design an OpenAI-plugin-owned non-interactive Codex auth contract that validates state/PKCE, avoids persisting callback material in env/logs, documents the chosen CLI/env behavior, and includes focused auth/setup coverage.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection gives a high-confidence current-behavior check that non-interactive setup requires runNonInteractive while OpenAI Codex OAuth/device-code expose interactive run handlers only.

## Solution Assessment

Unclear: accepting a full callback URL from env is one possible interface, but it is not clearly the best solution until maintainers choose a provider-owned OAuth code/state handling contract.
