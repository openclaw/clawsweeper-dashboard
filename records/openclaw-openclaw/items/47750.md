## Summary

Keep open: current main still has no non-interactive OpenAI Codex OAuth setup path, and the requested env/callback import would add a new security-sensitive auth/config surface rather than fix a broken existing contract.

## What This Changes

Request a non-interactive OpenAI Codex OAuth setup path, such as supplying a redirect/callback URL through environment, for ChatGPT subscription auth.

## Best Possible Solution

Design an OpenAI-plugin-owned non-interactive Codex auth contract that validates state/PKCE, avoids persisting callback material in env/logs, documents CLI/env behavior, and includes focused auth/setup tests.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection gives a high-confidence current-behavior check that non-interactive setup requires runNonInteractive while OpenAI Codex OAuth/device-code only expose interactive run handlers.

## Solution Assessment

Unclear: accepting a full callback URL from env is one possible interface, but it is not clearly the best solution until maintainers choose a provider-owned OAuth code/state handling contract.
