## Summary

Keep open: current main still supports non-interactive OpenAI setup through API-key auth, but ChatGPT/Codex browser OAuth and device-code auth remain interactive provider methods. The requested environment/callback import would be a new security-sensitive auth-provider/config contract, and the adjacent callback-settled issue and open OAuth cleanup PR do not implement this non-interactive setup path.

## What This Changes

Request a non-interactive way to set up OpenAI Codex OAuth for the model provider, proposed as an environment-supplied OAuth callback URL.

## Best Possible Solution

Design an OpenAI-plugin-owned non-interactive Codex auth contract that preserves PKCE/state validation, avoids leaking OAuth material, documents the CLI/env behavior, and covers credential storage for fresh setup and upgrades.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection gives a high-confidence current-behavior check that non-interactive provider setup requires runNonInteractive, and OpenAI Codex OAuth/device-code methods do not implement it.

## Solution Assessment

Unclear: accepting a full callback URL from env is one possible interface, but maintainers should first choose a leak-resistant provider-owned OAuth code/state handling contract.
