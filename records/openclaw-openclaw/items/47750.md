## Summary

Keep open: current main and the latest release still have non-interactive OpenAI setup only for API-key style auth, while OpenAI Codex browser OAuth and device-code auth remain interactive provider methods. The requested env/callback import would add a new auth-provider and secret-handling contract, so this needs maintainer product and security design rather than cleanup closure.

## What This Changes

Request a non-interactive way to configure OpenAI Codex OAuth for model-provider setup, such as supplying an OAuth redirect callback URL through an environment variable.

## Best Possible Solution

Design an OpenAI-plugin-owned non-interactive Codex auth/import contract that preserves PKCE/state validation, avoids leaking callback material, documents CLI/env behavior, and covers credential storage for fresh setup and upgrades.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection gives a high-confidence current-behavior check that non-interactive provider setup requires runNonInteractive, and OpenAI Codex OAuth/device-code methods do not implement it.

## Solution Assessment

Unclear: accepting a full callback URL from env is one possible interface, but maintainers should first choose a leak-resistant provider-owned OAuth code/state handling contract.
