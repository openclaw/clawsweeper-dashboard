## What This Changes

The issue requests a non-interactive environment-driven way to configure OpenAI/Codex OAuth for the OpenAI model provider.

## Best Possible Solution

Design an OpenAI-plugin-owned non-interactive Codex auth contract that preserves PKCE/state validation, avoids leaking OAuth material, documents the CLI/env behavior, and covers credential storage for fresh setup and upgrades.

Not applicable as a bug reproduction: this is a feature/config request. Source inspection gives a high-confidence current-behavior check that non-interactive setup requires runNonInteractive and OpenAI Codex OAuth/device-code methods do not implement it.

## Solution Assessment

Unclear: accepting a full callback URL from env is one possible interface, but maintainers should first choose a leak-resistant provider-owned OAuth code/state, token, or auth-session handling contract.
