Keep open: current main still only exposes interactive Codex OAuth/device-code setup, and the requested non-interactive callback/env flow would add a new auth/config surface that needs maintainer design review.

## What This Changes

The issue requests a non-interactive way to configure OpenAI Codex OAuth for model providers, such as supplying an OAuth callback URL through environment configuration.

## Best Possible Solution

Design a provider-owned non-interactive Codex auth setup contract with explicit secret-handling rules, config/docs coverage, and regression tests before adding an env or CLI surface.

Not applicable as a bug reproduction: this is a feature request. Source inspection shows current main has interactive OAuth/device-code flows and rejects non-TTY models auth login rather than providing the requested non-interactive callback/env path.

## Solution Assessment
