## Best Possible Solution

Design an OpenAI-plugin-owned non-interactive Codex auth contract that preserves PKCE/state validation, avoids persisting or logging callback material, documents the CLI/env behavior, and covers credential storage for fresh setup and upgrades.

## Solution Assessment

Unclear: accepting a full callback URL from env is one possible interface, but maintainers should first choose a leak-resistant provider-owned OAuth code/state handling contract.
