## Summary

Keep open: current main and v2026.6.10 still lack the requested WhatsApp disappearing-message expiration config and do not forward Baileys `ephemeralExpiration`; the exact prior implementation PRs are closed unmerged, so this still needs maintainer/security product review.

## What This Changes

The issue requests WhatsApp channel/account config that applies a disappearing-message expiration to outbound OpenClaw replies.

## Best Possible Solution

Add a plugin-local, opt-in WhatsApp expiration setting that resolves channel/account defaults and forwards Baileys `ephemeralExpiration` consistently for supported outbound text, media, quoted reply, payload, and poll paths, with docs and regression coverage.

## Reproduction Assessment

Yes by source inspection: current main lacks the requested config key and the WhatsApp outbound path never passes `ephemeralExpiration` to Baileys. I did not run a live WhatsApp disappearing-message send.

## Solution Assessment

Likely yes for the implementation layer: plugin-local config plus Baileys `ephemeralExpiration` is the narrow path. The credential-default fallback and public privacy wording still need maintainer/security approval.
