## Summary

Keep open: current main still lacks the WhatsApp config/schema field, account-level resolution, and Baileys send-option plumbing needed to apply disappearing-message expiration to outbound replies. The two related implementation PRs, #73080 and #73083, are closed unmerged, so this remains actionable repo work rather than cleanup-close material.

## What This Changes

Request adds WhatsApp channel-level and per-account outbound disappearing-message expiration config, forwarding positive values to Baileys sends across text, quoted replies, media, polls, and tool/channel send paths.

## Best Possible Solution

Implement this inside the bundled WhatsApp plugin with typed/schema-validated channel and account config, account override semantics, positive-integer normalization, Baileys option merging for all outbound send paths, focused tests, docs, generated config metadata, and a user-facing changelog entry.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: the proposed config key is absent from strict schema/types, resolved account state cannot carry it, and current WhatsApp send paths never pass Baileys ephemeralExpiration even though the pinned dependency exposes that option.

## Solution Assessment

Yes. Plugin-owned config plus Baileys send-options plumbing is the narrowest maintainable fix; no new core channel abstraction is needed unless multiple channels later need a shared disappearing-message contract.

## Repair Work Prompt

Fix this issue by adding opt-in WhatsApp outbound disappearing-message expiration support inside the bundled WhatsApp plugin. Add `channels.whatsapp.disappearingMessagesSeconds?: number` and `channels.whatsapp.accounts.<accountId>.disappearingMessagesSeconds?: number` to the typed config and strict zod schema, with account-level override semantics matching existing shared account fields. Resolve a positive effective expiration on `ResolvedWhatsAppAccount` or a small helper; treat omitted, null, zero, negative, and non-finite values as no OpenClaw override. Forward the normalized value as Baileys `MiscMessageGenerationOptions.ephemeralExpiration`, merging it with quoted-reply options, across normal text, media, voice-note companion text, poll sends, channel outbound sends, and tool/sendPayload paths. Review closed PRs #73080 and #73083 as source context only, and credit hclsys in the replacement PR body if reusing their approach. Update docs and generated config metadata as needed, add a user-facing `CHANGELOG.md` entry with appropriate non-bot attribution, and do not add a generic core channel abstraction unless implementation proves multiple channel owners need it. Do not change unrelated WhatsApp auth, reconnect, media compression, or access-control behavior.
