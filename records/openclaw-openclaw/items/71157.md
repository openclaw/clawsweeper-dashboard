## Summary

Keep open: current main still lacks the WhatsApp config/schema field, account-level effective value, and Baileys send-option plumbing needed to apply disappearing-message expiration to outbound replies. The two related implementation PRs are useful source context, but both are closed unmerged and no open PR currently tracks the fix.

## What This Changes

Add WhatsApp channel/account configuration for outbound disappearing-message expiration and pass the effective value to Baileys `ephemeralExpiration` across outbound reply paths.

## Best Possible Solution

Implement this inside the bundled WhatsApp plugin: add typed and schema-validated channel/account config, resolve a positive effective expiration with account override semantics, merge it into Baileys message-generation options for text, media, quoted replies, voice-note companion text, polls, and channel/tool send paths, and document the opt-in behavior without changing unset, null, or zero behavior.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: the proposed config key is absent from strict schema/types, resolved account state cannot carry it, and current WhatsApp send paths never pass Baileys `ephemeralExpiration` despite the pinned dependency exposing that option.

## Solution Assessment

Yes. The requested WhatsApp-plugin config plus Baileys options plumbing is the narrowest maintainable solution; it should reuse existing account config merge patterns and does not need a new core channel abstraction unless multiple channels later need a shared disappearing-message contract.
