Add `channels.whatsapp.disappearingMessagesSeconds` and `channels.whatsapp.accounts.<accountId>.disappearingMessagesSeconds` as optional config, with account-level override, positive whole-second values forwarded to Baileys as `MiscMessageGenerationOptions.ephemeralExpiration`, and `0`/`null`/omitted/non-positive values preserving current behavior. Thread the resolved value through resolved account state and outbound send options so normal text, quoted replies, media, voice-note trailing text, and poll sends are covered where Baileys accepts generation options.
