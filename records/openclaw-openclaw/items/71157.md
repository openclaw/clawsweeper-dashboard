## Summary

Keep open: current main still lacks the WhatsApp config/schema field, account-level resolution, and Baileys send-option plumbing needed to set disappearing-message expiration on outbound replies. The two related implementation PRs are closed unmerged, so this remains a valid narrow WhatsApp plugin repair rather than cleanup-close material.

## What This Changes

Add WhatsApp channel/account `disappearingMessagesSeconds` config and apply the resolved positive value as Baileys `ephemeralExpiration` across outbound WhatsApp reply paths.

## Best Possible Solution

Implement this inside the bundled WhatsApp plugin: add typed and schema-validated channel/account config, resolve a positive effective expiration with account override semantics, merge it into Baileys message-generation options for text, media, quoted replies, voice-note companion text, polls, and channel/tool send paths, and document the opt-in behavior without changing unset, null, or zero behavior.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: the proposed config key is absent from strict schema/types, resolved account state cannot carry it, and current WhatsApp send paths never pass Baileys `ephemeralExpiration` despite the pinned dependency exposing that option.

## Solution Assessment

Yes. The requested plugin-owned config plus send-options plumbing is the narrowest maintainable solution; no new core channel abstraction is needed unless multiple channels later need a shared disappearing-message contract.
