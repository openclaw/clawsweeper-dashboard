Keep open: current main and v2026.6.8 still lack WhatsApp disappearing-message expiration config and do not forward Baileys `ephemeralExpiration`; the prior implementation PRs are closed unmerged, so they are useful context but not a safe canonical fix.

## What This Changes

Request support for a WhatsApp outbound disappearing-message expiration setting that applies to channel/account config and outbound reply send paths.

## Best Possible Solution

Add a maintainer-approved, plugin-local, opt-in WhatsApp expiration config that resolves channel/account defaults and forwards Baileys `ephemeralExpiration` consistently for text, media, quoted replies, polls if supported, and payload sends, with docs and regression tests.

## Reproduction Assessment

Yes. Source inspection shows the requested config key is absent and current outbound WhatsApp sends do not pass `ephemeralExpiration`; I did not run a live WhatsApp expiration check.

## Solution Assessment

Likely yes for the implementation layer: plugin-local config plus Baileys `ephemeralExpiration` is the narrow path. The exact config semantics, docs wording, and optional credential-default fallback still need maintainer/security review.
