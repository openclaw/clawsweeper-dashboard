## Summary

Keep open: current main still lacks the WhatsApp config/schema field, account-level effective value, and Baileys send-option plumbing needed to apply disappearing-message expiration to outbound replies. The closed unmerged implementation PRs #73080 and #73083 are useful source context, but they did not land and no current-main implementation is present.

## What This Changes

Add WhatsApp channel/account config that forwards a positive disappearing-message expiration to outbound WhatsApp replies.

## Best Possible Solution

Implement this inside the bundled WhatsApp plugin with typed/schema-validated channel and account config, account override semantics, Baileys option merging for all outbound send paths, focused tests, docs, and a changelog entry.

## Reproduction Assessment

Yes for the missing capability: source inspection verifies the proposed config key is absent, strict schema rejects it, resolved account state cannot carry it, and outbound WhatsApp sends never pass `ephemeralExpiration`. No live WhatsApp account is needed to establish that current main lacks the requested plumbing.

## Solution Assessment

Yes. The narrow maintainable fix is WhatsApp-plugin local config plus Baileys send-option plumbing, reusing existing account merge and outbound send patterns; a new core channel abstraction is not needed unless multiple channels later need the same contract.
