## Summary

Keep open: current main and v2026.6.8 still lack a WhatsApp disappearing-message expiration config and do not pass Baileys expiration metadata on outbound sends. The request is valid, but it adds a privacy-sensitive config surface, so it should get maintainer confirmation before an automated fix lane implements it.

## What This Changes

Add WhatsApp channel/account configuration to apply disappearing-message expiration metadata to outbound OpenClaw replies.

## Best Possible Solution

Add a plugin-local, opt-in WhatsApp expiration config that resolves channel/account defaults and passes Baileys `ephemeralExpiration` consistently across outbound text, media, quoted replies, polls, and payload sends, with docs and regression tests.

## Reproduction Assessment

Yes, at source level: current main lacks a WhatsApp expiration config key and the outbound send paths do not pass `ephemeralExpiration`. I did not run a live WhatsApp disappearance check.

## Solution Assessment

Yes for the implementation layer: plugin-local config plus the existing Baileys send option is narrower than changing core session privacy. The optional credential-default fallback and user-facing privacy wording still need maintainer confirmation.
