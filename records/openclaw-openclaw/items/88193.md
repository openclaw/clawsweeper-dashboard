Keep open: the behavior fix is useful, but this PR is not merge-ready because it widens an exported plugin SDK result contract, crosses the Telegram plugin boundary with a core internal import, and still lacks actual redacted Telegram/live output proof.

## What This Changes

## Best Possible Solution

No as submitted. Preserving durable suppression reasons and narrowing retry eligibility is the right direction, but the SDK contract, plugin boundary, and inspectable live-proof gaps should be resolved before merge.
