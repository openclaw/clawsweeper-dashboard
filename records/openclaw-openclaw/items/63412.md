## Summary

Keep open. Current main still lacks Discord blockquote carryover, and this PR is a focused bug-fix direction, but the latest head still has a blocking maxChars regression and no after-fix real behavior proof.

## What This Changes

The branch adds Discord blockquote-prefix carryover to the text chunker, adds blockquote regression tests, and records a changelog entry.

## Best Possible Solution

Land a rebased Discord chunker fix that carries blockquote prefixes only for mid-line continuations, reserves prefix space under `maxChars`, keeps Unicode/fence/italic behavior intact, adds focused regression coverage, and includes redacted real behavior proof.

## Reproduction Assessment

Yes, by source inspection rather than a live run. Current main splits long Discord lines and only reseeds open code fences, so a long single-line `> ...` blockquote can be split mid-line and the continuation segment has no `> ` prefix.

## Solution Assessment

No, not yet. The approach is narrow, but the latest head still appends a full continuation segment after seeding the quote prefix, so it must reserve prefix space or recheck size before append.
