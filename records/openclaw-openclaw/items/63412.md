## Summary

Keep open. The underlying Discord blockquote split bug is still source-reproducible on current main, and the PR is useful, but this branch is stale against the current Discord chunker and would regress existing Unicode/CJK chunk boundaries while also using a deprecated SDK test barrel and missing the required changelog entry.

## What This Changes

The PR adds Discord blockquote-prefix tracking in `chunkDiscordText()` plus tests for quoted continuation chunks across character and line-limit splits.

## Best Possible Solution

Transplant the blockquote continuation behavior onto the current Discord chunker while preserving existing code-point/CJK split helpers, focused SDK test imports, and a user-facing changelog entry.

Yes. Source inspection gives a high-confidence path: a long single-line Discord blockquote split by `maxChars` creates continuation segments, and current main's `flush()` only re-seeds open fences, so the continuation chunk is not re-prefixed with `> `.

## Solution Assessment

No, not as-is. The blockquote approach is narrow and maintainable, but the branch must be rebased or replaced so it preserves current Unicode/CJK split contracts, focused SDK test imports, and changelog policy.
