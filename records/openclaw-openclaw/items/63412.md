## Summary

Keep open: this is a focused Discord bug-fix PR and current main still lacks blockquote carryover, but the branch can still exceed the Discord message cap on restored quote continuations and it lacks required after-fix real behavior proof.

## What This Changes

The PR adds Discord chunking state to reapply blockquote prefixes on mid-line continuation chunks, adds focused regression tests, and adds a changelog entry.

## Best Possible Solution

Land a rebased, narrow Discord chunker fix that preserves blockquote prefixes only where needed, reserves prefix space within the Discord cap, keeps the focused tests, and includes redacted real behavior proof.

## Reproduction Assessment

Yes, by source inspection. A long Discord blockquote split by `maxChars` goes through `chunkDiscordText`, and current main only reopens code fences, so continuation chunks lose quote context.

## Solution Assessment

No, not yet. The patch is on the right surface, but it must reserve room for the restored quote prefix and include after-fix real behavior proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Reserve room for restored quote prefixes:** `extensions/discord/src/chunk.ts:217-219`
  - body: When a long quoted line is split by `maxChars`, a continuation segment can already be sized to the full `charLimit`. After `flush({ preserveBlockquote: true })` seeds `current` with the quote prefix, that full segment is appended without another size check, so the emitted Discord message can exceed the configured cap; reserve prefix space for continuation segments or re-split after re-seeding.
  - confidence: 0.88
