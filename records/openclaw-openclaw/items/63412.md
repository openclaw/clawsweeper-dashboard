## What This Changes

The PR adds Discord chunker blockquote-prefix tracking and tests so character-split quoted continuations keep the `> ` prefix.

## Best Possible Solution

Yes. Source inspection shows current main splits long quoted lines with `splitLongLine()` and `flush()` only re-seeds open fences, so a single long `> ...` line split by `maxChars` produces continuation chunks without a leading quote prefix.

## Solution Assessment

No, not as-is. The branch's blockquote-state approach is the right narrow direction, but it must be transplanted onto current main's Discord chunker while preserving Unicode/CJK helpers, focused test imports, and changelog policy.
