## Summary

Keep this PR open: current main still lacks Discord blockquote carryover, and the branch appears to address the prior empty-quote-line regression, but the external real-behavior proof gate is still missing before merge.

## What This Changes

The PR adds Discord-specific blockquote prefix preservation to `chunkDiscordText` for mid-line character splits, adds focused chunker tests, and adds a changelog entry.

## Best Possible Solution

Land the narrow Discord chunker fix after targeted validation and redacted real Discord proof confirm long quoted continuations render as quoted text without line-limit artifacts.

## Reproduction Assessment

Yes, at source level. Current main routes Discord text through `buildDiscordTextChunks` and `chunkDiscordTextWithMode`, while `chunkDiscordText` only reopens fenced code blocks after flush, so a long `> ...` line split by `maxChars` continues without the quote marker.

## Solution Assessment

Yes for the code direction. Keeping the change local to the Discord chunker and preserving blockquote prefixes only for mid-line continuations is the narrow maintainable fix; merge readiness still depends on real Discord proof and normal validation.
