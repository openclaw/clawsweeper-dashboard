## Summary

Keep open: current main still has the Discord blockquote chunking gap, and this PR is pointed at the right code path, but the latest head is not merge-ready because it reintroduces a core-internal production import, makes the new test file redeclare imported helpers, and still lacks contributor-supplied real Discord proof.

## What This Changes

The PR adds Discord blockquote-prefix carryover for long quoted-line chunk continuations, focused regression tests, and a changelog entry.

## Best Possible Solution

Keep the blockquote carryover local to the Discord chunker, restore the current SDK import/test-helper contracts, and land only after focused tests plus redacted real Discord proof show quoted long messages stay quoted across chunk boundaries.

## Reproduction Assessment

Yes, source-reproducible. Current main sends Discord output through `buildDiscordTextChunks()` and the chunker only re-seeds open fences, so a long quoted line split by `maxChars` can produce a continuation chunk without the `>` prefix.

## Solution Assessment

Yes for the intended direction, but no for the current head as-is. The narrow maintainable fix is Discord chunker blockquote carryover, rebased onto the current SDK import/test-fixture surface and backed by real Discord proof.
