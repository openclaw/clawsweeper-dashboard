## Summary

Keep open, but this PR is not mergeable as-is. Current main still lacks configurable session-lane concurrency, while the PR head still carries blocking gateway reload, config schema, SDK export, and type-surface regressions plus missing real behavior proof; the broader umbrella remains tracked at https://github.com/openclaw/openclaw/issues/53023.

## What This Changes

The PR adds an `agents.defaults.sessionLaneMaxConcurrent` option and wires it into session lane creation, gateway startup/reload, config types, and the plugin SDK config-runtime barrel.

## Best Possible Solution

Rework this as a narrow additive queue/config change that preserves managed reload secrets/shared auth, SDK exports, cron-nested behavior, and existing config types, then add strict schema/docs/generated metadata, focused tests, and real channel or gateway proof.

## Reproduction Assessment

Yes. Source inspection shows current main still creates session lanes at concurrency 1 and excludes session lanes from gateway concurrency setup; the PR-head blockers are also static through the missing export, schema omission, removed SDK export, and deleted reload wrapper.

## Solution Assessment

No. Configurable session-lane concurrency is a reasonable direction, but this branch is not the best implementation because it mixes the additive feature with unrelated gateway reload/security, SDK, and public type-surface regressions.
