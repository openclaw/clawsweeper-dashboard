## Summary

Keep open. Current main still retains per-workspace skills snapshot versions after the watcher-disable teardown path, and the linked closing PR #77998 is open rather than merged.

## What This Changes

The issue requests clearing the skills snapshot `workspaceVersions` entry when a skills watcher is disabled.

## Best Possible Solution

Review and land or reject #77998; the desired end state is for `workspaceVersions` to follow watcher teardown without being cleared during same-workspace watcher replacement.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: fire a skills watcher event for a workspace, then call `ensureSkillsWatcher` with `skills.load.watch=false`; current main deletes the watcher but leaves the per-workspace version entry readable.

## Solution Assessment

Yes. The linked PR’s direction is the narrow maintainable fix: add a production clear helper and call it only from watcher-disable teardown, leaving watcher replacement semantics intact.
