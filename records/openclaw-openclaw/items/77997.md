## Summary

Keep open. Current main still retains per-workspace skills snapshot versions after watcher teardown, and the linked closing PR #77998 is open rather than merged.

## What This Changes

The issue asks for skills refresh watcher lifecycle cleanup so `workspaceVersions` and watcher entries do not accumulate after workspace teardown or idle workspaces.

## Best Possible Solution

Review and land or reject #77998; the end state should clear per-workspace versions on watcher teardown and bound abandoned watcher lifetime without breaking same-workspace watcher replacement.

## Reproduction Assessment

Yes. Source inspection gives a deterministic path: a skills watcher event bumps a workspace version, then `ensureSkillsWatcher` with `skills.load.watch=false` closes the watcher while leaving the per-workspace version readable.

## Solution Assessment

Yes. The linked PR direction is the narrow maintainable fix: add a production clear helper for watcher teardown and cover the lifecycle behavior with focused regression tests.
