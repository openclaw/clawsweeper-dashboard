## Summary

Keep open. ## What This Changes

The issue reports a 2026.6.1 regression where Gemini-backed memory status differs between normal and `--deep` checks, memory file changes stop auto-indexing, and manual indexing is required.

## Best Possible Solution

Keep this open and coordinate it with the linked provider-upgrade reports while fixing the status identity path to resolve adapter defaults consistently and deciding whether builtin memory watchers should be restored at gateway startup or documented as lazy manager behavior.

## Reproduction Assessment

Partially: current source gives a high-confidence explanation for the bare status versus `--deep` Dirty mismatch with Gemini default-model resolution. The watcher failure is plausible from the builtin startup/lifecycle path but was not live-reproduced here.

## Solution Assessment

Unclear: the status inconsistency likely belongs in the memory identity/status path, but the watcher part needs a maintainer decision on whether to restore eager builtin manager startup or adjust the documented watcher lifecycle.
