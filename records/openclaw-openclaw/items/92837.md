## What This Changes

The PR updates the Android chat composer to read live session token usage from gateway session metadata/events, display stale or missing usage as `Context --`, and add focused context-meter unit tests.

## Best Possible Solution

Land the Android-client fix after maintainers accept the bounded session-state UI risk on this head; keep the solution on existing gateway metadata contracts rather than changing token accounting.

## Reproduction Assessment

## Solution Assessment

Yes. This is the best layer for the fix: gateway already publishes `sessionInfo` and session snapshots with `totalTokens`, `totalTokensFresh`, and `contextTokens`, so Android should consume that contract instead of adding token-accounting changes or a second source of truth.
