## Summary

Keep open: current main still tags provider variants mainly for telemetry, while the mock dispatcher and Anthropic adapter share one prompt-driven plan path; I do not see a live OpenAI-vs-Anthropic parity lane that satisfies the alternative.

## What This Changes

Request to make QA Lab parity proof non-tautological by adding provider-aware mock dispatch or a live-frontier parity lane.

## Best Possible Solution

Make the deterministic mock intentionally provider-aware for a small, documented set of parity scenarios, and keep live-frontier comparison as an explicit separately approved lane.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists by inspecting the shared buildResponsesPayload() path, the Anthropic adapter, the source-discovery branch, and the test that pins the Anthropic plan to the same QA_SCENARIO_PLAN.md read; I did not run tests because this review is read-only.

## Solution Assessment

Yes for scoped Option A: threading the normalized provider variant into selected deterministic branches with tests is the narrow maintainable fix. The live-frontier lane is useful but needs separate owner review because it uses secrets and provider budget.
