## Summary

Keep this issue open. Current main still records a normalized provider variant only around request/debug metadata, while the mock scenario dispatcher and the Anthropic adapter still share the same deterministic plan path; QA parity workflows also still compare candidate and baseline through `--provider-mode mock-openai`, not real OpenAI-vs-Anthropic behavior.

## What This Changes

Request provider-aware qa-lab mock dispatch for GPT/Claude parity, or a separate live-frontier parity lane against real providers.

## Best Possible Solution

Implement deterministic Option A first: pass a normalized provider variant into the mock dispatcher, add at least three intentional OpenAI-vs-Anthropic scenario plan differences, and leave live-frontier parity CI as a separate owner-approved lane.
