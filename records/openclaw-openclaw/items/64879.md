## Summary

Keep this issue open. Current main records a provider variant for qa-lab mock requests, but the scenario dispatcher still does not use that variant for plan selection and the parity workflows still compare OpenAI and Anthropic through the same mock provider lane.

## What This Changes

The issue requests provider-aware qa-lab mock dispatch or a live-frontier parity lane so GPT and Claude parity runs can produce legitimately different plans.

## Best Possible Solution

Land deterministic Option A first: make selected agentic parity mock scenarios branch on a normalized provider variant and pin at least three legitimate OpenAI-vs-Anthropic plan differences in qa-lab tests.
