## Summary

Keep open. Current main records a provider variant in the qa-lab mock, but the shared scenario dispatcher still does not branch on that signal, and the parity workflows still compare OpenAI and Anthropic through the deterministic mock path rather than a real provider parity-report lane.

## What This Changes

Request provider-aware qa-lab mock dispatch, or alternatively a live-frontier CI parity lane, so OpenAI and Anthropic parity runs are not identical by construction.

## Best Possible Solution

Land the deterministic provider-aware mock-dispatch path first: compute the provider variant inside the dispatcher, make several agentic parity scenarios return intentionally different but valid OpenAI and Anthropic plans, and pin those differences in qa-lab mock server tests. Treat a real-provider parity workflow as a separate maintainer-approved follow-up.
