## Summary

Keep open. Current main still renders the Agents page selector as a native select with only a change handler, and neither current tests/docs nor the related open UX tracker prove that simulated-click browser automation is fixed or intentionally unsupported.

## What This Changes

The issue reports that the Control UI Agents page selector cannot be opened or changed through simulated click/browser automation in v2026.3.23-2.

## Best Possible Solution

Keep this open until maintainers decide the Control UI automation contract. If pointer-based browser automation should operate the Agents selector, replace the native selector with an accessible listbox/sidebar/list view and add focused browser/UI coverage; if synthetic click-to-open on native selects is unsupported, document the supported automation path and coordinate with #57067.
