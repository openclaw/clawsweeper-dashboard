## Summary

Keep open. Current main still carries the double-apply path for indexed array unsets, and the open linked PR #76294 uses closing syntax, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue reports that `openclaw config unset 'agents.list[1]'` silently removes more than the single indexed array element during persistence.

## Best Possible Solution

Land #76294 or an equivalent narrow fix so indexed array unsets are applied once, with regression coverage that preserves existing object-key unset behavior.

## Reproduction Assessment

Yes. Source inspection on current main shows the CLI splices the array once, then passes the same indexed path to the write layer where it can be spliced again during persistence.

## Solution Assessment

Yes. The narrow maintainable fix is to avoid replaying leaf array-index unsets after the CLI has already carried the spliced array into `nextConfig`; the open PR #76294 matches that boundary and should be reviewed or landed.
