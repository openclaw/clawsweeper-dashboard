## Summary

Keep open. Current main still has the indexed array unset double-splice path, and the open PR #76294 references this issue with closing syntax, so the issue should remain open until that fix lands or maintainers choose a replacement.

## What This Changes

The issue asks for `openclaw config unset 'array[index]'` to remove only the selected array element instead of truncating later entries.

## Best Possible Solution

Land a narrow fix like #76294 that applies indexed array unsets exactly once while preserving the existing `unsetPaths` behavior for object-key deletions, with regression coverage for `agents.list[1]` persisting `[agent-a, agent-c]`.

## Reproduction Assessment

Yes. Source inspection on current main shows `runConfigUnset` splices the indexed array path, then passes the same path to write preparation where the shifted array can be spliced again during persistence.

## Solution Assessment

Yes. The narrow maintainable solution is to avoid replaying indexed leaf array deletions through `writeOptions.unsetPaths` while retaining object-key unset handling; the open PR appears aligned and should be reviewed or landed before this issue closes.
