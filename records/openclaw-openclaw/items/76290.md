## Summary

Keep open. Current main still matches the reported data-loss path: `config unset` removes the selected array slot in the CLI clone, then the config writer receives the same indexed unset path and can remove the shifted successor before persisting.

## What This Changes

Fix `openclaw config unset` so bracket-index array paths remove only the selected element instead of deleting a shifted successor during write.

## Best Possible Solution

Persist array-index unsets exactly once while still carrying enough unset metadata for object-key default suppression and authored-config preservation.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists in current main: the CLI splice turns `[agent-a, agent-b, agent-c]` into `[agent-a, agent-c]`, and the writer can apply the same index unset again to persist `[agent-a]`.

## Solution Assessment

Yes. The narrow maintainable fix is to adjust the unset/write contract so supported bracket-index paths behave like a single-element deletion, rather than rejecting documented path syntax.
