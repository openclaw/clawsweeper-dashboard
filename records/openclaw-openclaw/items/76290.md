## Summary

Keep open. Current main still replays array-index `unsetPaths` after `runConfigUnset` has already spliced the array, and the open PR #76294 references this issue with closing syntax, so this should remain open until that PR or an equivalent fix lands.

## What This Changes

`openclaw config unset 'array[index]'` should remove only the indexed array element without deleting later entries.

## Best Possible Solution

Land the narrow CLI fix that skips replaying `unsetPaths` for array-index leaf unsets while preserving object-key unset persistence, with a regression for `agents.list[1]` persisting `[agent-a, agent-c]`.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: `runConfigUnset` splices `agents.list[1]` in the cloned config, then passes the same path into the write layer where array leaf unsets also splice by index.

## Solution Assessment

Yes, the narrow maintainable solution is to skip `unsetPaths` only for numeric leaf array unsets and keep it for object-key removal; #76294 implements that shape and should be reviewed or landed before this issue closes.
