## Summary

Keep open. Current main still has the reported double-splice path, and open PR #76294 already references this issue with closing syntax, so the issue should stay open until that fix or a replacement merges.

## What This Changes

The issue requests that `openclaw config unset 'agents.list[1]'` remove only that array element without deleting later entries.

## Best Possible Solution

Review and land PR #76294, or a narrow replacement, so indexed array unsets are applied once and a regression confirms `[agent-a, agent-c]` is persisted.

## Reproduction Assessment

Yes. The current-main source path is high-confidence: the CLI splices `agents.list[1]`, then the write layer receives the same numeric unset path and can splice the shifted element at index 1 again.

## Solution Assessment

Yes. A narrow config CLI/write-path fix is the maintainable direction, preserving object-key `unsetPaths` semantics while making indexed array unsets idempotent across the two layers.
