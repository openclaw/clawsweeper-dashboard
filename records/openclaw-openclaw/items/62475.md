## Summary

Keep open. Current main has cache retention, cache-ttl pruning, heartbeat keep-warm, Anthropic cache-boundary payload shaping, Google cachedContent refresh, and prompt-cache runtime context, but it still lacks the requested opt-in invisible Anthropic non-turn keep-warm ping with adaptive timing, dirty cancellation, and a hard cost cap.

## What This Changes

Add an opt-in prompt-cache keep-warm mechanism that sends invisible non-turn provider pings between user turns to prevent Anthropic cache TTL expiry.

## Best Possible Solution

Make an explicit product decision; if accepted, add a documented opt-in Anthropic maintenance mode that reuses existing payload policy and cache telemetry, avoids transcript/tool/compaction activity, cancels on user turns or cache-dirty events, and enforces a hard cost cap.

## Reproduction Assessment

Not applicable as a feature request. The absence of the requested behavior is high-confidence because current docs, typed config, gateway timers, heartbeat execution, Anthropic payload shaping, and prompt-cache runtime plumbing were inspected with no non-turn keep-warm ping surface found.

## Solution Assessment

Unclear. The proposed design is plausible, but the best fix depends on maintainer approval for hidden billable provider calls versus relying on heartbeat/cache-ttl pruning or aligning with the open session-maintenance design.
