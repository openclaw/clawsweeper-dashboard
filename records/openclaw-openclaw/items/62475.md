## Summary

Keep open. Current main and the latest release still have cache retention, cache-ttl pruning, heartbeat keep-warm, Anthropic payload shaping, Google cachedContent refresh, and prompt-cache runtime context, but not the requested opt-in invisible Anthropic non-turn keep-warm ping with adaptive timing, dirty cancellation, and a hard cost cap. The open related session-maintenance PR at https://github.com/openclaw/openclaw/pull/56575 is adjacent, not a merged implementation of this ping mode.

## What This Changes

The issue requests an opt-in, invisible Anthropic prompt-cache keep-warm ping that prevents cache TTL expiry between user turns without creating a normal agent turn.

## Best Possible Solution

Make an explicit product decision; if accepted, add a documented opt-in Anthropic maintenance mode that reuses existing payload policy and cache telemetry, performs no transcript/tool/compaction activity, cancels on user turns or cache-dirty events, and enforces a hard cost cap.

## Reproduction Assessment

Not applicable as a feature request. The absence of the requested behavior is high-confidence from current source, docs, typed config, gateway timers, heartbeat execution, payload shaping, and prompt-cache telemetry inspection.

## Solution Assessment

Unclear. The proposed design is plausible, but hidden billable non-turn provider calls need maintainer approval and should be reconciled with heartbeat/cache-ttl pruning and the open session-maintenance direction.
