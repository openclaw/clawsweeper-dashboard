## Summary

Keep open: current main and v2026.5.7 have cache retention, cache-ttl pruning, heartbeat keep-warm, Anthropic payload shaping, and prompt-cache runtime telemetry, but not the requested opt-in invisible Anthropic non-turn keep-warm ping with adaptive timing, dirty cancellation, cache-hit verification, and cost cap.

## What This Changes

Request an opt-in prompt-cache keep-warm mechanism that sends invisible minimal Anthropic pings between user turns to prevent cache TTL expiry.

## Best Possible Solution

If maintainers accept the tradeoff, add a documented opt-in Anthropic maintenance mode that reuses normal payload policy and cache telemetry, performs no transcript/tool/compaction activity, cancels on dirty/user-turn events, and enforces a hard cost cap.

## Solution Assessment
