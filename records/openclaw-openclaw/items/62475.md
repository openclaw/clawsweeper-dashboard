## Summary

Keep open. Current main has prompt-cache retention, cache-TTL pruning, heartbeat-based warming, Anthropic payload cache shaping, and prompt-cache runtime context, but it still does not implement the requested invisible non-turn Anthropic keep-warm ping with an opt-in config, adaptive interval, dirty cancellation, and hourly cost cap. The remaining work is a product/cost decision around hidden billable provider calls, not conservative cleanup closure.

## What This Changes

Requests an opt-in Anthropic prompt-cache keep-warm mechanism that sends invisible non-turn pings between user messages with adaptive TTL timing, dirty-cache cancellation, and cost caps.

## Best Possible Solution

Make an explicit product decision first. If accepted, add a documented opt-in Anthropic maintenance mode that reuses the existing payload policy and prompt-cache telemetry, never creates transcript/tool/compaction activity, cancels on user turns and cache-dirty events, and enforces a hard cost cap; otherwise document heartbeat plus cache-TTL pruning as the supported workaround.

## Reproduction Assessment

Not applicable as a feature request. The absence of the requested behavior is high-confidence because current-main docs, typed config, gateway maintenance timers, heartbeat execution, and prompt-cache plumbing were inspected and no keep-warm ping surface exists.

## Solution Assessment

Unclear. The proposed non-turn ping design is plausible, but it is not clearly the best fix until maintainers decide whether OpenClaw should perform hidden billable provider calls rather than relying on heartbeat/cache-TTL pruning or the open session-maintenance design.
