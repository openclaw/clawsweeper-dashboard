## Summary

Keep open. Current main and the latest release still have prompt-cache retention, cache-ttl pruning, heartbeat keep-warm guidance, Anthropic payload shaping, prompt-cache telemetry, and Claude CLI session warming, but not the requested opt-in invisible Anthropic non-turn keep-warm ping with adaptive TTL timing, dirty cancellation, hit verification, and a hard cost cap.

## What This Changes

Request an opt-in prompt-cache keep-warm mechanism that sends invisible Anthropic non-turn pings between user interactions with adaptive TTL timing, dirty cancellation, hit verification, and a cost cap.

## Best Possible Solution

Keep this open until maintainers choose a canonical cache-warmer design; if accepted, implement a documented opt-in Anthropic maintenance mode that reuses exact request shaping, avoids transcript/tool/channel side effects, cancels on dirty/user activity, and enforces a cost cap.
