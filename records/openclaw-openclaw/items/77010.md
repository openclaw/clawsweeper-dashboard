## What This Changes

## Best Possible Solution

Keep the production CONNECT guard, docs, and changelog direction, then fold in the temp-directory fixture fix from #77061 or an equivalent change before automerge re-review.

## Reproduction Assessment

Yes. Source inspection on current main shows that with OPENCLAW_PROXY_ACTIVE=1, a debug proxy CONNECT request reaches net.connect() without a managed-proxy guard, while the docs define managed proxy routing as the intended egress guardrail for supported runtime traffic.

## Solution Assessment

No, not quite as submitted. The production guard is the narrow maintainable fix before direct raw-socket CONNECT egress, but the regression fixture should use an OS temp root so generated CA files never land under the checkout.
