## Summary

Keep open for maintainer review/merge: current main still has a source-reproducible builtin-backend cold-start race for session archive indexing, and this scoped PR addresses that path with no blocking findings found.

## What This Changes

The PR adds a builtin-backend gateway startup sidecar that preloads sessions-enabled memory managers, plus focused regression tests and a changelog entry.

## Best Possible Solution

Land the scoped builtin-backend preload after maintainer approval, and keep qmd listener arming separate so the fallback manager design is reviewed on its own.

## Reproduction Assessment

Yes, at source level. Current main emits archive updates into an unqueued listener set, while the sessions listener subscribes only after a non-transient sessions-enabled memory manager is constructed.

## Solution Assessment

Yes for the scoped builtin fix. Preloading the cached non-transient manager before archive events is the narrowest maintainable repair, and the explicit qmd guard avoids claiming coverage for the lazy fallback-manager path.
