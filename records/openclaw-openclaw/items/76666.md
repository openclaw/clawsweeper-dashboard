## Summary

Keep open. The PR targets a real source-visible race on current main, and the patch direction is narrow for the builtin memory backend; it is still a draft PR and needs normal maintainer/CI validation before landing.

## What This Changes

The PR adds a gateway post-attach sidecar that preloads default-purpose memory managers for agents with session-memory sources, plus unit coverage and a changelog entry.

## Best Possible Solution

Land a ready version of this startup preload with focused regression coverage and keep qmd boot-sync behavior unchanged unless maintainers accept a separate qmd transcript-event requirement.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: current main emits archive updates through an unbuffered listener set, while builtin memory only subscribes after the first non-transient manager is created. I did not run live or unit tests during this read-only review.

## Solution Assessment

Yes for the builtin backend: preloading default-purpose managers is the narrowest way to arm the existing listener without changing config or memory-search semantics. If maintainers want the “regardless of backend” claim to be literal for qmd, that should be reviewed as qmd-specific event-sync follow-up rather than blocking the builtin fix.
