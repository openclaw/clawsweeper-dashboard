## Summary

Keep open. The underlying memory/session archive race is source-visible and worth fixing, but this draft does not yet satisfy its backend-neutral claim because the new preload path goes through the active memory backend and does not instantiate the builtin listener when QMD succeeds.

## What This Changes

The PR adds a gateway sidecar that preloads memory search managers for agents with the sessions memory source, plus focused tests and a changelog entry.

## Best Possible Solution

Add a backend-aware startup hook that explicitly arms the session transcript indexing path for the active backend, with regression coverage for builtin and QMD-success behavior.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for source-level review: current main emits archive updates through `transcript-events`, while the builtin listener is only registered after a non-transient `MemoryIndexManager` is constructed; the PR body also describes repeated cold-start builtin deployment reproductions.

## Solution Assessment

Is this the best way to solve the issue? No. The direction is narrow for builtin memory, but routing through the active manager means QMD success does not instantiate `MemoryIndexManager` or its listener, so the backend-neutral fix should use an explicit runtime hook or QMD equivalent.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Preload the listener when QMD is active:** `src/gateway/server-startup-memory.ts:152-155`
  - body: This call goes through the active memory backend. With `memory.backend = "qmd"`, `getMemorySearchManager` returns the cached QMD wrapper and only constructs `MemoryIndexManager` from the fallback factory after QMD fails, so `ensureSessionListener()` is never registered on successful QMD deployments. The new sidecar can log that it armed the preload while archive emits still have no builtin session listener, leaving the advertised backend-neutral fix incomplete.
  - confidence: 0.84
