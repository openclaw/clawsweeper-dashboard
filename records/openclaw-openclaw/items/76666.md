## Summary

Keep open: current main still lacks this builtin-backend startup preload, and the PR's scoped implementation matches the proven listener-race root cause without a blocking review finding.

## What This Changes

This PR adds a gateway startup sidecar that preloads non-transient sessions-enabled memory managers for the builtin backend, adds focused tests, and documents the builtin-only fix in the changelog.

## Best Possible Solution

Land the scoped builtin-backend preload after maintainer approval, and keep qmd listener arming as a separate follow-up tied to the qmd manager design.

## Reproduction Assessment

Yes, at source level. Current main emits archive updates immediately into a process-local listener set, while the sessions listener only subscribes after a non-transient sessions-enabled `MemoryIndexManager` is constructed.

## Solution Assessment

Yes for the scoped builtin-backend fix. Preloading the cached non-transient manager before archive events is the narrowest maintainable fix, and the qmd guard avoids claiming coverage for the lazy `FallbackMemoryManager` path.
