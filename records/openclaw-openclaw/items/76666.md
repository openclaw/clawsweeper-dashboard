## Summary

Keep open. Source review confirms the current builtin memory path can drop session archive events before the lazy manager is first loaded, and the revised PR addresses the prior test-fixture gap; it still needs conflict resolution/rebase and CI before it can land.

## What This Changes

The PR adds a builtin-backend gateway startup sidecar that preloads session-capable memory managers, skips qmd, adds focused gateway tests, and updates the changelog.

## Best Possible Solution

Land the builtin-only preload with its regression coverage once the branch is mergeable and CI is green, then handle qmd listener arming in a separate backend-aware follow-up.

## Reproduction Assessment

Yes, at source level: current main only attaches the session listener from non-transient MemoryIndexManager instances, while archiveFileOnDisk emits immediately. A cold gateway with no prior memory_search can therefore drop the archive event before any listener exists.

## Solution Assessment

Yes for the builtin backend scope: preloading the non-transient builtin manager is the narrowest fix because it reuses the existing listener and cache semantics. Skipping qmd is appropriate here because that path returns a lazy FallbackMemoryManager wrapper and needs a separate design.
