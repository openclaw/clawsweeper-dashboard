## Summary

Keep open: current main still lacks the builtin session-listener preload, while the PR addresses a source-reproducible cold-start race and the scoped builtin-only diff looks correct after the qmd path was explicitly deferred.

## What This Changes

The PR adds a gateway startup sidecar that preloads builtin session-memory managers for sessions-enabled agents, covers the qmd skip/no-close behavior in tests, and documents the scoped fix in the changelog.

## Best Possible Solution

Land the builtin-backend preload if exact-head CI remains green, and track qmd listener arming separately in the qmd/FallbackMemoryManager path.

## Reproduction Assessment

Yes, at source level. Current main emits reset/deleted archive updates into a process-local listener set, while a non-transient sessions-enabled memory manager is only constructed after the first memory search or explicit indexing path.

## Solution Assessment

Yes for the scoped builtin-backend fix. Preloading the cached non-transient manager before archive events is the narrowest maintainable repair, and the explicit qmd skip avoids claiming coverage for the lazy `FallbackMemoryManager` path.
