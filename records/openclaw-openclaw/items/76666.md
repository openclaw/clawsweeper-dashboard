## What This Changes

The PR adds a gateway startup sidecar that preloads builtin sessions-enabled memory managers without closing them, adds focused gateway tests, and documents the builtin-only fix in the changelog.

## Best Possible Solution

## Reproduction Assessment

Yes, at source level. Current main emits archive updates immediately into a process-local listener set, while the memory index only subscribes after a non-transient sessions-enabled manager is constructed.

## Solution Assessment

Yes for the scoped builtin-backend fix. Preloading the cached non-transient manager before archive events is narrow and maintainable, and the explicit qmd skip avoids claiming coverage for the lazy `FallbackMemoryManager` path.
