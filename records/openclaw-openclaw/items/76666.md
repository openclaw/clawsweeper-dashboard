## Summary

Keep this PR open: current main still lacks a builtin-backend startup path that arms session transcript listeners before `/reset` or `/new`, while the PR is a focused implementation with tests, real deployment proof, and no blocking review findings found.

## What This Changes

The PR adds a gateway startup sidecar that preloads builtin sessions-enabled memory managers, adds focused tests including the qmd skip guard, and documents the builtin-only behavior in `CHANGELOG.md`.

## Best Possible Solution

Land the scoped builtin-backend preload after maintainer review, and track qmd listener arming as a separate backend-design follow-up.

## Reproduction Assessment

Yes, at source level. Current main emits archive updates into a process-local listener set, while the sessions listener subscribes only after a non-transient sessions-enabled `MemoryIndexManager` exists.

## Solution Assessment

Yes for the scoped builtin fix. Preloading the cached non-transient manager before archive events is narrow, and the explicit qmd skip keeps the PR from over-claiming coverage of the lazy fallback wrapper path.
