## Summary

Keep open for normal PR review. The revised branch now honestly narrows the startup preload to the builtin backend and no longer claims qmd coverage, but one test fixture needs tightening before this is a strong regression guard.

## What This Changes

The PR adds a gateway startup sidecar that preloads builtin session-memory managers, skips qmd, adds focused gateway tests, and adds an Unreleased changelog entry.

## Best Possible Solution

Land the builtin-only preload after tightening the regression test, then track qmd listener arming as a separate backend-aware follow-up.

## Reproduction Assessment

Yes, at source level. Current main only subscribes from a non-transient MemoryIndexManager, while archiveFileOnDisk emits immediately during reset/delete, so a cold gateway with no prior memory_search can drop the event.

## Solution Assessment

Yes for the builtin backend scope, with a small test correction. Skipping qmd is the maintainable narrow choice for this PR because the qmd wrapper does not construct the listener-owning builtin manager until fallback is needed.
