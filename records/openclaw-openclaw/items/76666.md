## Summary

Keep open for normal PR review: current main has a source-reproducible builtin memory cold-start race, and the PR's builtin-only preload matches the existing manager/listener contract with focused regression coverage. I did not find a blocking diff bug, but the latest head still needs green CI before merge.

## What This Changes

The PR adds a builtin-backend gateway startup sidecar that preloads non-transient session-memory managers, adds gateway tests for sessions/qmd/error cases, and documents the fix in the changelog.

## Best Possible Solution

Land the builtin-only preload after green CI, and handle qmd listener arming in a separate qmd/FallbackMemoryManager follow-up.

## Reproduction Assessment

Yes, at source level. Current main only subscribes session transcript listeners when a non-transient sessions-enabled MemoryIndexManager exists, while archive emits are delivered immediately to the current listener set.

## Solution Assessment

Yes for the builtin-backend scope. Preloading the cached non-transient manager is the narrow maintainable fix, and the explicit qmd skip avoids claiming coverage for the lazy FallbackMemoryManager path.
