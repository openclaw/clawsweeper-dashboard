## Summary

Keep open: current main still has the cold-start listener race, and the PR’s builtin-only preload matches the source-level bug without a blocking correctness finding. It should proceed through normal maintainer review and exact-head CI before merge.

## What This Changes

The PR adds a gateway startup sidecar that preloads builtin sessions-enabled memory managers, keeps them cached, adds focused gateway tests, and documents the builtin-only fix.

## Best Possible Solution

Land the builtin-backend preload after exact-head validation, and track qmd listener arming separately in the qmd/FallbackMemoryManager path.

## Reproduction Assessment

Yes, at source level. Current main emits archive updates immediately into a process-local listener set, while the builtin memory index subscribes only after a non-transient sessions-enabled manager is constructed.

## Solution Assessment

Yes for the scoped builtin-backend fix. Preloading the cached non-transient manager before archive events is narrow, and the explicit qmd skip avoids overclaiming coverage for the lazy fallback wrapper path.
