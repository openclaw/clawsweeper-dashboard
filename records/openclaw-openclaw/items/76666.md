## Summary

Keep this PR open: current main still lacks the builtin startup listener preload, so the branch contains useful unique work, but current-main memory-core now drops reset/deleted archive events after the listener is armed, so this PR no longer proves the advertised end-to-end reset-archive indexing fix by itself.

## What This Changes

The PR adds a Gateway startup task that preloads builtin memory managers for sessions-enabled agents so session transcript listeners are attached before the first `memory_search` call.

## Best Possible Solution

Rebase against current main, land the builtin startup preload only with the archive-event scheduling fix from https://github.com/openclaw/openclaw/pull/96132 or equivalent behavior, and keep qmd listener arming as an explicit follow-up if maintainers accept builtin-only scope.

## Solution Assessment

No, not as-is against current main. The builtin preload is the right timing fix, but it needs the current archive-listener guard fixed or merged first so the preloaded listener actually schedules reset/deleted archives.
