The PR adds a gateway startup sidecar that eagerly preloads non-transient builtin memory managers for sessions-enabled agents so reset/new transcript archive events are indexed before the first memory_search call.

## Best Possible Solution

Rebase on current main, keep the builtin preload if maintainers accept the qmd deferral and startup lifetime tradeoff, remove the release-owned changelog edit, fix the sidecar comment, and rerun focused gateway memory coverage plus required checks.

## Solution Assessment

Yes for the builtin-backend scope. Current main's catch-up path only starts after manager construction, while qmd parity needs a separate owner-boundary decision around FallbackMemoryManager or qmd-native transcript handling.
