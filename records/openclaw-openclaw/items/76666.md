## Summary

Keep open: this is a focused, still-relevant bug-fix PR, and current main still lacks the builtin listener preload for the cold-start reset/new archive race.

## What This Changes

The PR adds a gateway startup sidecar that preloads builtin sessions-enabled memory managers, tests builtin and qmd-skip behavior, and documents the builtin-only scope in the changelog.

## Best Possible Solution

Land the scoped builtin-backend preload after maintainer review and current-head validation, while keeping qmd listener arming as a separate backend follow-up.

## Reproduction Assessment

Yes, at source level. Current main emits reset/new archive updates into a non-queued listener set, while the sessions listener is installed only after a non-transient sessions-enabled `MemoryIndexManager` exists.

## Solution Assessment

Yes for the scoped builtin fix. Preloading the cached non-transient manager before archive events is narrow, and the qmd early return avoids claiming coverage of the lazy fallback wrapper path.
