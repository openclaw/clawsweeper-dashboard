## Summary

Keep open: current main still lacks a builtin backend session-listener preload, while the PR addresses the source-level cold-start race with a scoped gateway startup sidecar and no blocking review findings found.

## What This Changes

The PR adds a gateway startup sidecar that preloads builtin sessions-enabled `MemoryIndexManager` instances, regression tests for skip/success/failure paths, and a changelog entry documenting the builtin-only scope.

## Best Possible Solution

Land the scoped builtin-backend preload after maintainer review and exact-head CI, then handle qmd listener ownership in a separate qmd/FallbackMemoryManager follow-up.

## Reproduction Assessment

Yes, at source level. Current main emits archive transcript updates immediately into a process-local listener set, while the memory index subscribes only after a non-transient sessions-enabled manager is constructed.

## Solution Assessment

Yes for the scoped builtin-backend fix. Preloading the cached non-transient manager before archive events is narrow and maintainable, and the explicit qmd skip avoids claiming coverage for the lazy `FallbackMemoryManager` path.
