## What This Changes

The PR changes /v1/responses client-tool streaming so client-tool handoffs stop the agent loop and streamed function-call arguments are sent through dedicated SSE argument events.

## Best Possible Solution

Rebase onto current main's multi-call client-tool loop, emit documented argument delta/done events for every pending call while preserving completed snapshots, and require redacted live HTTP/SSE proof before merge.

## Reproduction Assessment

Yes, by source inspection: current main's gateway emits client-tool arguments only on output_item events, and the OpenAI Responses stream parser handles delta events but not done-only argument finalization. I did not run a live gateway reproduction in this read-only review.

## Solution Assessment

Partly: dedicated argument events and done-event parsing are the right layer, but the PR must be rebased onto the current multi-call implementation and its new event shape should match the documented Responses contract more closely.
