## Summary

Keep open. Current main still has the source-visible false-alarm path, and an open closing PR (#76797) is already paired with this issue, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue reports that `openclaw doctor` emits a no-active-memory-plugin warning even when memory search is enabled and the running gateway reports embeddings ready.

## Best Possible Solution

Land or close PR #76797 after review; doctor should suppress or reword the no-runtime warning when the gateway memory probe proves embeddings are ready.

## Reproduction Assessment

Yes from source inspection. If `resolveActiveMemoryBackendConfig` returns null while the gateway probe is ready, current main emits the no-active-plugin note and returns before the ready probe can suppress it.

## Solution Assessment

Yes, the narrow repair is to use the existing gateway memory probe before emitting the no-runtime warning. A full CLI plugin bootstrap or new RPC design would be broader than needed for this false alarm.
