## What This Changes

The PR adds an opt-in `sessionMode` hook option across config/schema/docs, direct and mapped hook dispatch, replay scoping, cron session reuse, and focused tests.

## Best Possible Solution

Merge the explicit opt-in persistent hook session feature after adding an active-version changelog entry and letting exact-head required checks gate the PR.

## Reproduction Assessment

Yes. On current main, repeated direct or mapped hook-agent requests with the same allowed `sessionKey` still route through an isolated cron job and reach `resolveCronSession` with `forceNew: true`, so code inspection gives a high-confidence reproduction path for fresh session IDs each time.

## Solution Assessment

Yes. An explicit `sessionMode: "persistent"` opt-in is the narrowest maintainable fix because it preserves isolated mode as the default while keeping existing hook token and session-key prefix policy checks in place.
