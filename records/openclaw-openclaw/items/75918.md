## Summary

## What This Changes

The PR adds `sessionMode: "isolated" | "persistent"` for hook-agent payloads, hook mappings, transform results, config schema/help/docs, cron isolated-session dispatch, plus related agent harness/runtime and Codex auth changes.

## Best Possible Solution

Land a focused current-main implementation of explicit persistent hook sessions with isolated as the default, and either remove the legacy Codex auth/runtime changes from this PR or get explicit maintainer/security approval plus migration proof for them.

## Reproduction Assessment

Yes, at source level: current main still dispatches hook-agent requests through isolated cron session creation, and `resolveCronSession` only reuses existing sessions when `forceNew` is false. I did not run a live webhook in this read-only review.

## Solution Assessment

Partly. Explicit `sessionMode` is the best product direction for the hook-session problem because it preserves default isolation, but this branch is broader than the hook fix and should not land with the legacy auth-provider runtime route unresolved.
