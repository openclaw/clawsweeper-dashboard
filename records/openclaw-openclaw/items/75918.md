## What This Changes

The PR adds a `sessionMode: "isolated" | "persistent"` hook option across hook config, request handling, mapping transforms, docs, schema metadata, tests, and isolated cron session reuse.

## Best Possible Solution

## Solution Assessment

Yes for the product direction, but no for the merge-ready patch. Explicit persistent mode is the safer narrow design, while this branch still needs rebase, dependency cleanup, runtime proof, and maintainer choice against the automatic-reuse alternative.
