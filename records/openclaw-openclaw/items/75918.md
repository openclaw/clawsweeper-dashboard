## Summary

## What This Changes

The PR adds a hook `sessionMode: "isolated" | "persistent"` option across config/schema/docs, direct and mapped hook dispatch, replay keys, cron session reuse, and targeted tests.

## Best Possible Solution

Land a narrow PR that keeps isolated mode as the default, adds the active changelog entry, documents the trust boundary, preserves session-key policy checks, and includes real proof of repeated stable-key hook calls reusing one fresh session.

## Reproduction Assessment

Yes. Source inspection on current main shows direct and mapped hook-agent runs still lack a persistence selector and reach `resolveCronSession` with `forceNew: true`, while the reuse branch only runs when `forceNew` is false.

## Solution Assessment

Yes for the proposed design: an explicit `sessionMode` opt-in is the narrowest maintainable way to enable trusted multi-turn webhook threads without changing default isolation. The current PR is not merge-ready until real behavior proof and the changelog entry are added.
