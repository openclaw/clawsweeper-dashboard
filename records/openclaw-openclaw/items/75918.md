## Summary

Keep open: this PR addresses a real current-main hook-session gap with the safer explicit opt-in shape, but it is not merge-ready because the supplied proof only covers config acceptance and the branch still carries unrelated dependency-resolution churn.

## What This Changes

The branch adds hook `sessionMode` support for direct `/hooks/agent`, hook mappings, transforms, docs, schema metadata, tests, and cron session reuse, plus an unrelated `basic-ftp` override and lockfile change.

## Best Possible Solution

Land a scoped explicit `sessionMode: "isolated" | "persistent"` implementation after removing unrelated package churn, preserving default isolation and session-key gates, and adding redacted runtime proof of stable-key hook reuse.

## Reproduction Assessment

Yes, by source inspection. Approved stable-key `/hooks/agent` requests become isolated cron jobs, `run.ts` passes `forceNew: true`, and `resolveCronSession` cannot reuse the stored session id.

## Solution Assessment

Yes for the direction, but not for the merge-ready patch. Explicit persistent mode is the narrow safer design, while this branch still needs dependency cleanup, contributor runtime proof, and maintainer choice against the automatic-reuse alternative.
