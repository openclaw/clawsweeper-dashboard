Full review comments:

- **[P1] Gate request-controlled persistent hook sessions:** `src/gateway/hooks.ts:446-449`
  - body: This accepts `sessionMode` from every direct `/hooks/agent` body before the hook policy has resolved whether the caller supplied an allowed scoped `sessionKey`. With a configured `hooks.defaultSessionKey`, a third-party hook sender that is not allowed to choose `sessionKey` can still set `sessionMode: "persistent"` and share that default transcript across invocations, despite the docs warning persistent mode is only for trusted scoped keys. Please gate persistent mode behind the same explicit/scoped-key policy or make it server-owned mapping/config only.
  - confidence: 0.86
- **[P2] Do not revive legacy openai-codex implicit auth routing:** `extensions/codex/src/app-server/auth-bridge.ts:37-41`
  - body: The current docs and provider contract say `openai-codex` profile ids/order are legacy migration input and should be repaired by doctor to the canonical `openai` route. Adding `openai-codex` to the implicit app-server profile search makes stale profiles a live automatic auth route again when no `openai` profile is present, which can mask required migration and conflict with canonical auth ordering. Please keep implicit selection on canonical `openai` and put legacy repair in doctor/migration code.
  - confidence: 0.82
