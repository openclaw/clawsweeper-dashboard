## Summary

Keep open. Current main still lacks the requested `agent.wait` usage metadata, and the PR addresses a real gateway contract gap, but the patch is not merge-ready because lifecycle-first waits can still lose the new metadata and optional cost calculation can fail an otherwise successful run.

## What This Changes

The PR adds optional usage, cost, provider, model, and session metadata to gateway wait snapshots and forwards it through gateway dedupe and `agent.wait` terminal responses.

## Best Possible Solution

Refresh the branch against current main, add the optional wait metadata as a documented gateway response contract, keep lifecycle-first waits from aborting the metadata source before fallback, make cost calculation unable to fail completed runs, preserve current terminal snapshot fields, and add focused tests plus a changelog entry while leaving the OpenRouter billed-cost issue separate.

## Reproduction Assessment

Yes for source-level reproduction. Current main visibly omits the requested metadata, and the PR race is visible by following the lifecycle-first branch: it aborts the dedupe signal, whose wait helper resolves `null`, before trying to use dedupe metadata as fallback.

## Solution Assessment

No. The additive metadata direction is reasonable, but this implementation is not the best mergeable solution until the lifecycle/dedupe race, non-best-effort cost lookup, stale-main refresh, docs/tests, and changelog are handled.
