## Summary

Keep open. Current main still lacks usage/cost metadata in `agent.wait`, and this PR addresses a real gateway protocol gap, but the patch is not merge-ready because the lifecycle-first wait path can still drop metadata and optional cost estimation can turn a completed run into an error.

## What This Changes

The branch adds optional usage, cost, provider, model, and session metadata to gateway `agent.wait` terminal snapshots, derives it from completed embedded-agent run metadata, stores it in dedupe payloads, and forwards it through wait responses.

## Best Possible Solution

Refresh the PR against current main, add a typed optional wait metadata shape without dropping existing terminal fields, make cost lookup best-effort, make lifecycle-first waits deterministically obtain metadata, and document/changelog the new wire response.

## Reproduction Assessment

Not applicable as a normal user-facing reproduction, but there is a high-confidence source inspection path. Current main omits metadata despite available `EmbeddedPiAgentMeta` data, and the PR's lifecycle-first race is visible in the `agent.wait` race/dedupe control flow.

## Solution Assessment

No. The additive response field is the right direction, but this patch is not the best implementation until the dedupe race, non-fatal cost calculation, and release documentation are fixed.

  - body: When lifecycle wins, this path has already aborted the dedupe wait before the new fallback awaits `dedupePromise` for `meta`.
