## What This Changes

The PR adds optional usage, cost, provider/model, and sessionId metadata to gateway `agent.wait` terminal snapshots and agent dedupe responses.

## Best Possible Solution

Land a rebased gateway protocol change that adds optional metadata without changing run success/failure semantics, covers cached and lifecycle-first wait paths, and includes focused tests plus real behavior proof.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a new gateway response capability. Source inspection does confirm current main does not expose usage/cost metadata from `agent.wait`.

## Solution Assessment

No. The direction is reasonable, but this implementation should keep the dedupe wait alive for metadata fallback, make cost lookup best-effort, and rebase onto current `agent.wait` fields before merge.
