## Summary

Keep this PR open: it addresses a real gateway protocol gap, but the current patch still has merge-blocking correctness issues in wait metadata delivery and optional cost calculation, plus a missing changelog entry.

## What This Changes

The PR adds optional usage, cost, provider, model, and session metadata to gateway `agent.wait` terminal snapshots by deriving it from completed embedded-agent run metadata and forwarding it through dedupe/wait responses.

## Best Possible Solution

Refresh the PR against current main, keep all existing terminal snapshot fields, make wait metadata deterministic and best-effort, and add focused coverage plus the required changelog entry.

## Reproduction Assessment

Not applicable as a normal bug reproduction, but there is a high-confidence source inspection path. Current main omits metadata despite available `EmbeddedPiAgentMeta` data, and the PR's lifecycle-first metadata race is visible in the wait/dedupe control flow.

## Solution Assessment

No. The additive `meta` response field is the right direction, but this patch is not the best implementation until the dedupe race, non-fatal cost calculation, current-main refresh, and changelog coverage are fixed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Keep dedupe alive before reading metadata fallback:** `src/gateway/server-methods/agent.ts:879`
  - body: When lifecycle wins, this path has already aborted the dedupe wait before the new fallback awaits `dedupePromise` for `meta`. The abort handler can resolve `null`, so in-flight `agent.wait` calls still return without usage/cost metadata even though the terminal dedupe payload is written moments later.
  - confidence: 0.89
- **[P2] Make cost metadata calculation best-effort:** `src/gateway/server-methods/agent.ts:166-170`
  - body: `buildUsageMetaFromRunResult` calls `loadConfig()` while constructing optional metadata in the success path. If config loading or validation throws, the surrounding catch reports the completed run as an error, so cost lookup should use runtime config or be wrapped so metadata cannot fail the run.
  - confidence: 0.86
- **[P3] Add the required changelog entry:** `src/gateway/server-methods/agent-wait-dedupe.ts:4`
  - body: This is a user-facing gateway protocol feature, but the PR changes only gateway source files. Repo policy requires a `CHANGELOG.md` entry for `feat` changes so the new response metadata is visible in release notes.
  - confidence: 0.82
