## Summary

Keep open. Current main still lacks the requested `agent.wait` usage/cost metadata and the linked issue was closed in favor of this PR, but this external PR is not merge-ready because it lacks real behavior proof and has source-inspected Gateway correctness blockers on the current wait path.

## What This Changes

The PR adds an optional `meta` envelope with session id, usage, last-call usage, provider/model, and estimated cost to Gateway `agent.wait` terminal snapshots.

## Best Possible Solution

Port the metadata addition onto current main as an additive Gateway contract, preserve existing wait/dedupe/status fields, keep cost calculation best-effort, add focused coverage/docs, and require redacted real Gateway output before merge.

## Reproduction Assessment

Yes for the blocking PR defects: PR-head source inspection shows the abort-before-fallback race, fatal optional cost path, and missing current terminal fields. I did not run a live Gateway flow in this read-only review.

## Solution Assessment

No. The API direction is plausible, but this branch is not yet the best maintainable implementation until it is rebased onto current Gateway code, preserves current fields, keeps metadata best-effort, adds coverage, and supplies real behavior proof.

Full review comments:

- **[P2] Keep the dedupe wait alive for metadata fallback:** `src/gateway/server-methods/agent.ts:856`
  - body: When lifecycle wins, this aborts `dedupePromise` before the later metadata fallback awaits it. `waitForTerminalGatewayDedupe` resolves `null` on abort, so the common lifecycle-first `agent.wait` path can still return without the usage/cost `meta` that becomes available in the dedupe payload moments later.
  - confidence: 0.91

  - body: `buildUsageMetaFromRunResult` runs after `agentCommandFromIngress` succeeds, but it calls `loadConfig()` while building optional metadata. If config reading or validation throws, the surrounding promise catch converts a completed run into an error response instead of omitting `costUsd` and preserving the successful result.
  - confidence: 0.84
- **[P2] Preserve current wait terminal fields:** `src/gateway/server-methods/agent.ts:891`
  - body: Current main returns `stopReason`, `livenessState`, and `yielded` from `agent.wait`, and the SDK uses those fields to classify cancelled and timed-out runs. This stale response shape only adds `meta` and would drop those fields if carried forward during rebase.
  - confidence: 0.86
