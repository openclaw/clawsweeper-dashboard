## Summary

Keep this PR open. Current main still lacks the requested `agent.wait` usage/cost metadata, and the PR is still relevant, but the proposed patch is not ready as-is because lifecycle-first waits can still drop the new metadata and optional cost estimation can turn a successful run into an error.

## What This Changes

The PR adds an optional `AgentWaitUsageMeta` to gateway wait snapshots, extracts usage/session/provider/model/cost data from completed agent runs, stores it in the gateway dedupe payload, and forwards it through `agent.wait` responses.

## Best Possible Solution

Refresh the PR against current main, preserve the existing wait terminal fields, add a typed optional usage metadata shape, compute cost as non-fatal best-effort metadata, make lifecycle-first waits deterministically wait for or reread the dedupe metadata, and add focused gateway tests plus protocol documentation for the chosen response surfaces.

## Reproduction Assessment

Not applicable as a user-facing runtime bug reproduction, but there is a high-confidence source inspection path: current main’s wait snapshot and both `agent.wait` response paths lack the requested metadata, while `EmbeddedPiAgentMeta` already has the source data.

## Solution Assessment

No. The requested direction is reasonable and additive, but this patch is not the best implementation until the lifecycle/dedupe race and non-fatal cost estimation issues are fixed and the new wire shape is documented/tested.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Keep dedupe alive before reading metadata fallback:** `src/gateway/server-methods/agent.ts:868`
  - body: When lifecycle wins the wait race, this patch aborts the dedupe wait before awaiting it for `meta`. That makes the new metadata nondeterministic for the common in-flight `agent.wait` flow because the aborted dedupe promise can resolve `null` before the terminal payload with usage data is written.
  - confidence: 0.89
- **[P2] Make cost metadata calculation best-effort:** `src/gateway/server-methods/agent.ts:160-164`
  - body: `buildUsageMetaFromRunResult` calls `loadConfig()` while constructing optional response metadata in the successful completion path. If config loading or validation throws, the outer `.catch` reports the already-completed run as an error; cost lookup should be wrapped or use already-loaded config so metadata cannot fail the run.
  - confidence: 0.86
