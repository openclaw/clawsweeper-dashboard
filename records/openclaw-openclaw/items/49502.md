## Summary

Keep open: current main still lacks the requested Gateway wait metadata, but this PR remains blocked by source-visible correctness issues and missing contributor real behavior proof.

## What This Changes

The PR adds an optional `meta` envelope with session id, token usage, last-call usage, provider/model, and estimated USD cost to Gateway `agent.wait` terminal snapshots and dedupe parsing.

## Best Possible Solution

Add the metadata as an additive Gateway contract while preserving current terminal fields and cancellation semantics, treating cost as best-effort estimated metadata, adding focused wait-path coverage, documenting the response shape, and requiring redacted real Gateway output before merge.

## Reproduction Assessment

Yes for the blocking PR defects: source inspection of the PR head shows the aborted dedupe fallback, fatal optional cost path, and stale terminal response shape. I did not run a live Gateway flow in this read-only review.

## Solution Assessment

No. The API direction is plausible, but this branch is not yet the best maintainable implementation until the race, best-effort metadata handling, current-main field preservation, rebase, tests, and real behavior proof are addressed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Keep the dedupe wait alive for metadata fallback:** `src/gateway/server-methods/agent.ts:879`
  - body: When the lifecycle promise wins, this path has already aborted `dedupePromise`, and `waitForTerminalGatewayDedupe` resolves `null` on abort. Awaiting that same promise here cannot reliably recover the terminal dedupe payload carrying `meta`, so the common lifecycle-first `agent.wait` path can still return without usage/cost metadata.
  - confidence: 0.91
- **[P2] Keep cost metadata best-effort:** `src/gateway/server-methods/agent.ts:166-170`
  - body: `buildUsageMetaFromRunResult` runs after `agentCommandFromIngress` succeeds, but it calls `loadConfig()` while building optional metadata. If config reading or validation throws, the surrounding catch turns a completed run into an error response instead of omitting `costUsd` and preserving the successful result.
  - confidence: 0.84
- **[P2] Preserve existing wait terminal fields:** `src/gateway/server-methods/agent.ts:891`
  - body: Current main already returns `stopReason`, `livenessState`, and `yielded` from `agent.wait`, and the SDK uses those fields to classify cancelled and timed-out runs. This stale response shape only adds `meta` and would drop those fields if taken during rebase.
  - confidence: 0.86
