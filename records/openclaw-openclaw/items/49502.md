## Summary

Keep open. Current main still lacks the requested `agent.wait` usage/cost metadata, but this PR is not merge-ready because real behavior proof is missing and PR-head source inspection still shows stale wait-contract and optional-metadata failure paths.

## What This Changes

The PR adds an optional `meta` envelope with session id, usage, last-call usage, provider/model, and estimated cost to Gateway `agent.wait` terminal snapshots.

## Best Possible Solution

Rebase onto current main, add the optional metadata alongside the existing wait fields, keep metadata/cost lookup best-effort, and prove a real Gateway `agent.wait` response after the patch.

## Reproduction Assessment

Yes for the blocking PR defects by source inspection: the PR aborts the dedupe wait before awaiting it for metadata fallback, and it calls `loadConfig()` inside optional metadata construction on the success path. I did not run a live Gateway flow in this read-only review.

## Solution Assessment

No. The API direction is useful, but the current branch needs a rebase and targeted fixes so metadata is additive, deterministic, and best-effort rather than a new success-path failure mode.
