## Summary

Keep open: current main still lacks this Gateway metadata surface, but this PR is not merge-ready because the lifecycle-first wait path can still lose metadata and optional cost estimation can make a successful run fail; external real-behavior proof is also missing.

## What This Changes

The PR adds optional usage, cost, provider, model, and session metadata to Gateway `agent.wait` terminal snapshots and forwards it from completed agent runs.

## Best Possible Solution

Add a documented, optional `agent.wait` metadata envelope with best-effort cost estimation, preserve current terminal fields, and cover cached, lifecycle-first, no-meta, cache-only, and config-failure paths.

## Reproduction Assessment

Yes for the blocking review findings: source inspection shows the PR aborts the dedupe wait before its metadata fallback awaits it, and calls `loadConfig()` in the success path. I did not run a live Gateway `agent.wait` flow.

## Solution Assessment

No. The API direction is plausible, but the current patch needs a rebase plus fixes for lifecycle-first metadata loss and best-effort cost calculation before it is the narrow maintainable solution.
