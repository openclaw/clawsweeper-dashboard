## What This Changes

The PR adds an optional Gateway `agent.wait` metadata envelope with session, usage, last-call usage, estimated cost, provider, and model fields by threading agent run metadata through dedupe and wait responses.

## Best Possible Solution

Rebase onto current main, add the optional metadata envelope without dropping existing terminal fields, make cost estimation best-effort from a runtime config snapshot, document the wire contract, add focused wait-path regressions, and post redacted real Gateway output.

## Reproduction Assessment

Yes for the blocking PR defects by source inspection: PR head aborts the dedupe wait before the metadata fallback, calls `loadConfig()` in optional success-path metadata construction, and omits current main wait terminal fields. I did not run a live Gateway flow in this read-only review.

## Solution Assessment

No, not in the current branch shape. The API direction is useful, but the maintainable fix should be additive to the current wait contract, use best-effort metadata construction, and prove the real Gateway response after rebase.
