## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Preserve current wait terminal fields:** `src/gateway/server-methods/agent.ts:891`
  - body: Current main returns `stopReason`, `livenessState`, and `yielded` from `agent.wait`, and the SDK uses those fields to distinguish accepted waits from terminal cancellation/timeout/yield states. This response shape drops them while adding `meta`, so a rebase that keeps this shape would hide existing terminal state from callers; add `meta` without removing the current fields in both wait response paths.
  - confidence: 0.91
- **[P2] Keep dedupe wait alive for metadata fallback:** `src/gateway/server-methods/agent.ts:856`
  - body: When the lifecycle promise wins, this aborts the dedupe wait before the later metadata fallback awaits `dedupePromise`. The dedupe waiter resolves `null` on abort, so the common lifecycle-first path can still miss the terminal dedupe payload that contains the new metadata.
  - confidence: 0.88
- **[P2] Make cost metadata best-effort:** `src/gateway/server-methods/agent.ts:169`
  - body: `loadConfig()` can throw on config parse or validation errors, and this call runs inside the success `.then` before the terminal response is written. Since the metadata is optional, a config/pricing failure should omit `costUsd` or log best-effort failure, not convert an already completed agent run into an error response.
  - confidence: 0.86
