## Summary

Keep open. Current main still lacks the requested `agent.wait` usage/cost metadata, but this PR is not merge-ready because source inspection still shows the lifecycle-first metadata race and optional cost calculation can turn a completed run into an error response; the external real-behavior proof gate is also still missing.

## What This Changes

The branch adds an optional Gateway `agent.wait` metadata envelope with session id, token usage, last-call usage, provider/model, and estimated cost fields.

## Best Possible Solution

Rebase onto current main, preserve the current terminal snapshot fields, make metadata and cost calculation best-effort and race-safe, document the estimated metadata contract, add focused Gateway wait coverage, and provide redacted real `agent.wait` output.

## Reproduction Assessment

Yes for the blocking PR findings: source inspection of the PR head shows the dedupe wait is aborted before the metadata fallback awaits it, and optional cost metadata calls `loadConfig()` inside the success path. I did not run a live Gateway flow in this read-only review.

## Solution Assessment

No. The API direction is plausible, but the current branch needs race-safe wait logic, best-effort cost handling, current-main rebase preservation, tests, and real behavior proof before it is the narrow maintainable solution.
