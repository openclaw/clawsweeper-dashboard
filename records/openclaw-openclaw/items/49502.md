## Summary

Keep open. Current main still does not expose usage/cost metadata on `agent.wait`, and the closed companion issue points to this PR as the canonical implementation path, but this branch is stale/dirty and still has source-level correctness blockers plus missing real behavior proof.

## What This Changes

The PR adds an optional `meta` object with session id, token usage, last-call usage, provider/model, and estimated cost to Gateway `agent.wait` terminal snapshots and dedupe parsing.

## Best Possible Solution

Add the metadata as an additive Gateway contract while preserving current terminal fields and cancellation semantics, keeping cost calculation best-effort, adding focused wait-path coverage/docs, and requiring redacted real Gateway output before merge.

## Reproduction Assessment

Yes for the blocking PR defects: source inspection of PR head shows the aborted dedupe fallback, fatal optional cost path, and stale terminal response shape. I did not run a live Gateway flow in this read-only review.

## Solution Assessment

No. The API direction is plausible, but this branch is not yet the best maintainable implementation until the race, best-effort metadata handling, current-main field preservation, rebase, tests, and real behavior proof are addressed.

  - confidence: 0.86
