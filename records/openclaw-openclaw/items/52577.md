## What This Changes

Requests that Discord inbound messages sent during the post-restart WebSocket not-ready window be queued or backfilled from channel history instead of silently dropped.

## Best Possible Solution

Land or replace the Discord-plugin-owned reconnect backfill in https://github.com/openclaw/openclaw/pull/81226, using REST history after READY/RESUMED with existing preflight filters, run queue behavior, and replay dedupe, then close this issue after merged proof.

## Solution Assessment

Yes. The narrow maintainable direction is a Discord plugin fix that reuses REST channel history, existing preflight filters, replay dedupe, and run-queue behavior rather than adding global core policy.
