## Summary

Keep open: current main and v2026.5.18 still make post-core convergence fatal for any repair warning, while the repair/update producer and docs show some warnings are nonfatal fallback observations.

## What This Changes

The issue asks OpenClaw to distinguish hard post-core plugin convergence failures from non-blocking repair warnings so soft warnings do not block gateway restart.

## Best Possible Solution

Keep mandatory convergence fatal for failed repair installs and payload smoke failures, but let successful repair/update fallback warnings surface as plugin warning status without blocking restart.

## Reproduction Assessment

Yes. From source inspection, a repair result with warnings but no failedPluginIds and no smoke failures still makes runPostCorePluginConvergence return errored: true on current main.

## Solution Assessment

No. The current implementation is not the best solution because it uses warning count as the fatality signal; the narrower fix is to key fatality on repair.failedPluginIds and smoke failures while preserving warning output.
