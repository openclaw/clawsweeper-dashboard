## Summary

Keep open: current main and the latest release still make warning-only post-core convergence fatal, and an open linked PR with closing syntax is the right implementation path rather than closing this issue before merge.

## What This Changes

The issue asks OpenClaw to distinguish soft post-core plugin repair warnings from hard repair or smoke-check failures so warning-only convergence does not block the update restart path.

## Best Possible Solution

Review and land or supersede https://github.com/openclaw/openclaw/pull/84431 so convergence preserves warning visibility while only failed repair ids and payload smoke failures block restart.

## Reproduction Assessment

Yes. From source, make `repairMissingConfiguredPluginInstalls` return a warning with no `failedPluginIds` while the smoke check returns no failures; current main still returns `errored=true`.

## Solution Assessment

Yes. The maintainable fix is to use the existing structured `failedPluginIds` and `smoke.failures` signals instead of parsing warning text or adding new configuration.
