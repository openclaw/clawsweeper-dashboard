## Summary

Keep open: current main still makes post-core convergence fatal for any repair warning, and the linked fix PR is open rather than merged. Source, tests, and docs show soft plugin fallback warnings are intended to remain visible without failing the core update path.

## What This Changes

The issue asks OpenClaw to distinguish fatal post-core plugin convergence failures from non-blocking repair warnings.

## Best Possible Solution

Land a resolved fix that keys fatal convergence on failed repair ids and smoke-check failures while preserving soft repair warnings as warning status, then close the issue after merge.

## Reproduction Assessment

Yes. From source inspection, make repairMissingConfiguredPluginInstalls return only a soft warning while the smoke check returns no failures; current main still returns errored: true because it keys fatality on warnings.length.

## Solution Assessment

Yes. The maintainable direction is to keep warnings visible but mark convergence fatal only for failedPluginIds or smokeFailures; the open linked PR is aimed at that boundary.
