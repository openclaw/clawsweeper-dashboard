## Summary

Keep open: current main still makes post-core convergence fatal for any repair warning, while the repair/update contracts already distinguish nonfatal warning text from failed plugin IDs and smoke-check failures.

## What This Changes

The issue asks OpenClaw update convergence to treat soft plugin repair warnings as nonblocking while preserving fatal restart blocking for real repair or payload-validation failures.

## Best Possible Solution

Base convergence fatality on explicit failed repair IDs and smoke-check failures, while preserving soft repair/update warnings as operator-visible warning status with focused regression coverage.

## Reproduction Assessment

Yes. From source inspection, a warning-only `repairMissingConfiguredPluginInstalls` result with no smoke failures still makes `runPostCorePluginConvergence` return `errored: true`, which the update path maps to a pre-restart exit.

## Solution Assessment

Yes. The narrow maintainable fix is to keep mandatory convergence fail-closed for `repair.failedPluginIds` and smoke failures, but stop using total warning count as the fatality signal.
