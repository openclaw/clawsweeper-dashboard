## Summary

Keep open: current main still makes any post-core repair warning fatal, even though the repair/update path has separate failure signals and documented nonfatal fallback warnings.

## What This Changes

The issue asks OpenClaw to distinguish hard post-core convergence failures from non-blocking repair warnings so successful plugin update warnings do not block restart.

## Best Possible Solution

Use the existing hard-failure signals, such as `repair.failedPluginIds` and payload smoke failures, for convergence fatality while preserving soft repair/update warnings as operator-visible `warning` status.

## Reproduction Assessment

Yes, source-level: make `repairMissingConfiguredPluginInstalls` return a warning with no `failedPluginIds` while the smoke check returns no failures, and current main still returns `errored: true`. I did not execute a live update or test in this read-only sweep.

## Solution Assessment

Yes, the issue's direction is the right narrow fix: convergence should key fatality on explicit repair failures and smoke failures, not on total warning count.
