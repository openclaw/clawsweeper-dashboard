  - body: `loadMergedBundleMcpConfig` filters `enabled: false` MCP servers before launch, but this collector still resolves refs from every saved server. A disabled server with an unresolved SecretRef can therefore block agent startup even though that server would not run.
  - confidence: 0.9

  - body: This only checks `isDangerousHostEnvVarName`, while current main's MCP launch path also drops inherited host config pivots except explicit credential keys.   - confidence: 0.88

  - body: MCP env/header strings are valid literals today, but passing every value through `collectSecretInputAssignment` lets shared coercion parse `$NAME` and `${NAME}` as env SecretRefs. Existing literal configs can start resolving host env or failing as unresolved refs unless this surface is structured-ref-only or explicitly approved with upgrade proof.
