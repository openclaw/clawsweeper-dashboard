Full review comments:

- **[P1] Preserve literal MCP env/header strings:** `src/secrets/runtime-config-collectors-core.ts:230-240`
  - body: This new path passes every MCP env/header value through `collectSecretInputAssignment`, which uses `coerceSecretRef` and therefore treats exact `$FOO` and `${FOO}` strings as env SecretRefs. Current main accepts those values as literal MCP strings, so existing configs can change launch values or fail when the env var is absent; keep shorthand disabled for this surface unless maintainers approve an explicit migration/compat plan.
  - confidence: 0.91
- **[P1] Cover MCP SecretRefs in policy evidence:** `src/secrets/target-registry-data.ts:212-231`
  - body: The PR makes `mcp.servers.*.env.*` and `.headers.*` supported SecretRef targets, but the policy scanner still skips generic env maps and has no core MCP header pattern. That lets `secrets.requireManagedProviders` and `secrets.denySources` miss these new refs, so add policy evidence and update the old raw-MCP-env test to preserve only non-SecretRef literals.
  - confidence: 0.88
- **[P2] Audit non-string MCP credential literals:** `src/secrets/mcp-target-sensitivity.ts:106-107`
  - body: MCP env/header config still accepts number and boolean literals, and the launch path stringifies them, but this helper returns before checking sensitive names for non-string values. A credential like `API_KEY: 123456` can be launched but skipped by `secrets audit` and configure candidate detection; either audit these accepted values or stop accepting them on credential-bearing MCP fields.
  - confidence: 0.86
- **[P2] Mirror stdio env filtering for resolved MCP refs:** `src/secrets/runtime-config-collectors-core.ts:225-229`
  - body: The collector only treats `isDangerousHostEnvVarName` keys as inactive, while the stdio launch path drops a broader MCP-specific set via `toMcpEnvRecord`. Refs under keys such as dangerous inherited override variables can therefore be resolved or reported even though launch will drop them; use the same MCP stdio env predicate for active-surface resolution.
  - confidence: 0.82
