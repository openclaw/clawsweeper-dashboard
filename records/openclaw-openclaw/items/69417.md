Full review comments:

- **[P1] Preserve literal MCP env/header strings:** `src/secrets/runtime-config-collectors-core.ts:230-236`
  - body: `collectSecretInputAssignment` uses generic SecretInput coercion here, which treats exact `$NAME` and `${NAME}` strings as env SecretRefs. Current MCP launch behavior passes strings literally, so existing configs that intentionally send those literal values can start resolving them or fail when the env var is absent; keep MCP strings literal unless maintainers approve and test a migration.
  - confidence: 0.9
- **[P2] Reuse the MCP stdio env drop predicate:** `src/secrets/runtime-config-collectors-core.ts:225-228`
  - body: This check only uses `isDangerousHostEnvVarName`, but the launch path also drops dangerous inherited config keys such as `ANSIBLE_CONFIG` and `TF_CLI_CONFIG_FILE`. A SecretRef on those keys can still be resolved or required even though the spawned MCP process will not receive it, so the collector should share the launch predicate.
  - confidence: 0.9
