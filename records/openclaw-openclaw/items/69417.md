Full review comments:

- **[P1] Detect template refs before skipping MCP resolution:** `src/agents/agent-runtime-config.ts:80-82`
  - body: The new MCP gate only calls `hasNestedSecretRef`, which recognizes object refs via `isSecretRef` but not supported string templates like `${MC_API_KEY}`. With only template refs under `mcp.servers.*.env.*` or `headers.*`, `resolveCommandConfigWithSecrets` is skipped and the raw template reaches MCP launch instead of the resolved secret.
  - confidence: 0.9
- **[P2] Skip blocked stdio env refs before resolving them:** `src/secrets/runtime-config-collectors-core.ts:217-224`
  - body: The collector marks the entire stdio env surface active and resolves every env SecretRef, but the launcher drops keys such as `NODE_OPTIONS` and `LD_PRELOAD` through the host env safety policy. Mirror that blocked-key filter here so OpenClaw does not resolve or require secrets that the spawned process will never receive.
  - confidence: 0.9
- **[P2] Check benign MCP literals before sensitive names:** `src/secrets/mcp-target-sensitivity.ts:107-108`
  - body: `shouldAuditPlaintextMcpValue` returns true as soon as the key name looks sensitive, before checking benign values. This makes values such as `AUTH_ENABLED: "enabled"` or `X-Token-Refresh: "disabled"` look like plaintext secrets even though they are ordinary state literals.
  - confidence: 0.88
- **[P2] Audit credential-bearing MCP URLs:** `src/secrets/mcp-target-sensitivity.ts:123-125`
  - body: Returning false for every URL-shaped literal skips neutral MCP keys that contain real credentials in userinfo or query params, such as `https://user:pass@host` or `?api_key=...`. Check for embedded credentials before treating URL-like values as benign.
  - confidence: 0.86

  - body: The configure candidate filter only checks the leaf key name, so supported MCP env/header paths with neutral names but existing SecretRefs, bearer strings, or credential-bearing URLs are dropped. Use value-aware sensitivity logic so `secrets configure` can migrate real MCP credential paths without adding false positives.
  - confidence: 0.86
