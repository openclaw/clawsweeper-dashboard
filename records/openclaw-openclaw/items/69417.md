  - body: `loadMergedBundleMcpConfig` excludes `enabled: false` servers before runtime/probe use, but this collector still resolves refs from every saved server.   - confidence: 0.9

  - body: Current main's stdio launch filter blocks inherited host config pivots in addition to direct dangerous env names, but this collector only checks `isDangerousHostEnvVarName`. That can resolve refs for env keys the launch path will drop, so the SecretRef collector and launch code can disagree about which credentials are active.
  - confidence: 0.86

  - body: MCP env/header strings are valid literals today, but this collection path uses shared SecretInput coercion that parses `$NAME` and `${NAME}` as env SecretRefs. Existing literal configs can start resolving host env values or failing as unresolved refs without an explicit maintainer-approved upgrade policy and tests.
