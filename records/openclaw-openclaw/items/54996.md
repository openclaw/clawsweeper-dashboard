## Summary

Keep this open. Current main has a partial transcript-token estimate for preflight compaction, but the shared session/status/UI contract still does not expose an estimated context snapshot when provider usage is missing or marked stale, so the user-facing problem remains valid.

## What This Changes

Request local transcript-derived estimated context-token accounting when provider usage is missing or all-zero, including estimated status display and compaction threshold use.

## Best Possible Solution

Add one context-usage snapshot contract that can represent authoritative provider prompt totals or conservative transcript-derived estimates with an explicit source/estimated flag, then render and use that snapshot consistently while preserving fresh provider data as authoritative.
