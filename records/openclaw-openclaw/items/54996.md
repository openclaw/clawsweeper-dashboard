## Summary

Keep open. Current main has added transcript-based estimates for preflight compaction, but the status, session row, /usage, and Control UI paths still do not expose a shared estimated context snapshot when provider usage is missing or stale.

## What This Changes

Estimate context-token usage locally from session transcripts when provider usage is missing or all-zero, then display and use that estimate for status and compaction decisions.

## Best Possible Solution

Add an additive, source-aware context-usage snapshot that distinguishes provider-fresh totals from transcript-estimated totals and have status, session rows, /usage, Control UI notices, and compaction consume it consistently.

## Reproduction Assessment

Yes. A source-level reproduction is to use a session with normal transcript messages and totalTokensFresh false: current status tests expect Context: ?/limit, Control UI returns no context notice, and gateway transcript fallback only reads explicit usage snapshots.

## Solution Assessment

Yes, with a refinement. Local transcript estimation is the right direction, but the maintainable fix is one shared estimated snapshot contract rather than separate ad hoc estimates in each display and compaction path.

Add a shared context-usage snapshot path that can estimate current context tokens from session transcript messages when provider usage is missing/all-zero or totalTokensFresh is false, mark that value as estimated rather than provider-fresh, and consume it consistently in /status, session rows, /usage, Control UI context notices, and preflight compaction. Keep the contract additive and provider-agnostic; do not add MiniMax-specific branches or a config option.
