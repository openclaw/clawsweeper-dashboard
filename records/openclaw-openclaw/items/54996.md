## Summary

Keep open. Current main only implements transcript-based estimation for preflight compaction; status, session rows, /usage, and the Control UI still require fresh usage snapshots or render unknown/hidden context when provider usage is missing.

## What This Changes

The issue requests local estimated context-token tracking when provider responses do not include usable token usage data.

## Best Possible Solution

Add a shared source-aware context-usage snapshot that distinguishes provider-fresh totals from transcript-estimated totals and have status, session rows, /usage, Control UI notices, and compaction consume it consistently.

## Reproduction Assessment

Yes. Source inspection shows a session with normal transcript messages and totalTokensFresh false still renders Context: ?/limit in status, hides the Control UI notice, and omits /usage context percent instead of estimating from transcript text.

## Solution Assessment

Yes, with refinement. Local transcript estimation is the right direction, but the maintainable fix is one shared estimated snapshot contract rather than separate ad hoc estimates in each display and compaction path.
