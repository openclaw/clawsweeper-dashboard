## Summary

Keep open. Current main covers the auto-compaction part with transcript estimation, but status, `/usage`, and the Control UI still suppress or hide context usage when `totalTokensFresh` is false, so the reporter's visibility request remains valid.

## What This Changes

The issue requests local transcript-based context token estimation when provider usage data is missing, including estimated status display and compaction behavior.

## Best Possible Solution

Add one shared estimated-context snapshot contract that labels estimated values and feeds status, sessions.list/detail rows, `/usage`, Control UI notices, and compaction consistently while preserving unknown for genuinely unavailable totals.

## Reproduction Assessment

Yes for the remaining behavior: source inspection shows totalTokensFresh false still renders unknown or hides context in status, `/usage`, and Control UI despite transcript data being available. I did not run a live MiniMax session.

## Solution Assessment

Yes, with refinement. Local transcript estimation is the right direction, but the maintainable fix is a shared source-aware usage snapshot rather than provider-specific MiniMax handling or separate per-surface heuristics.
