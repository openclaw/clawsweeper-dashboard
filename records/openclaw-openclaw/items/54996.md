## Summary

Keep open. Current main has several adjacent fixes for prompt-token persistence, transcript-usage fallback, and preflight compaction, but the reported no-usage/all-zero provider path still lacks a shared estimated context snapshot and still hides stale context in status/UI surfaces.

## What This Changes

The issue requests local transcript-derived context token estimates when provider usage data is missing or unusable.

## Best Possible Solution

Add an additive, shared context-usage snapshot contract that distinguishes provider-fresh totals from transcript-estimated totals, then render estimated context consistently in status, session rows, /usage, and Control UI while using the estimate conservatively for compaction thresholds.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is to seed a session with missing/all-zero provider usage and normal transcript messages: current tests and implementation keep totalTokensFresh false, return no transcript usage snapshot, and render context as unknown or hidden instead of estimated.

## Solution Assessment

Yes, with a refinement: local transcript estimation is the right direction, but it should be implemented as a single source-aware snapshot contract rather than separate ad hoc estimates in each display or compaction path.
