## What This Changes

The issue asks OpenClaw to estimate context token usage locally when provider usage telemetry is missing or stale, display that estimate, and use it for compaction decisions.

## Best Possible Solution

Review and either land or close the active reporting-only estimated context budget PR, then make a separate maintainer decision on whether estimated snapshots should drive automatic compaction thresholds.

Yes, source inspection gives a high-confidence path: stale `totalTokensFresh === false` still suppresses transcript-derived context in `/status`, `/usage`, and the Control UI notice on current main. I did not run a live MiniMax session.

## Solution Assessment

Unclear as a complete solution: adding one shared estimated-vs-reported context snapshot is the maintainable first step, but using estimates for automatic compaction needs explicit product policy so display semantics and runtime behavior do not drift.
