## Summary

This PR should stay open, but the patch is not merge-ready because it revives whole-agent runtime pins that current main explicitly treats as legacy ignored config. The linked user problem in https://github.com/openclaw/openclaw/issues/81395 may be real, but changing this contract needs maintainer product approval and coordinated docs/doctor/test updates.

## What This Changes

The PR changes agent runtime policy resolution and metadata/tests so `agents.list[].agentRuntime` and `agents.defaults.agentRuntime` can affect harness/subagent runtime selection after provider/model policy.

## Best Possible Solution

Either keep the current provider/model-scoped runtime contract and guide the linked issue to that config path, or make a maintainer-approved contract change that updates runtime selection, doctor migrations, docs, and regression tests together.

## Reproduction Assessment

Yes for the review finding: current main's source, docs, doctor migration rules, and tests consistently ignore whole-agent runtime keys, while the PR diff makes those keys effective. I did not run tests because this was a read-only review.

## Solution Assessment

No. As written, the PR revives legacy whole-agent runtime pins while the current public contract and doctor guidance still tell users those keys are ignored; the safer path is provider/model-scoped config or an explicit maintainer-approved contract update.
