## Summary

Keep open: this is a plausible fix path for the linked runtime/billing report and it includes real resolver proof, but the branch is not merge-ready because it makes a legacy whole-agent runtime key active while current main still documents, labels, tests, and doctor-migrates that key as ignored.

## What This Changes

The PR makes `resolveModelRuntimePolicy` fall back to `agents.defaults.agentRuntime`, propagates a `defaults` runtime source, and adds or updates runtime-policy tests.

## Best Possible Solution

Decide the runtime config contract first, then either keep whole-agent runtime keys ignored and guide users to provider/model policy, or deliberately revive inheritance with matching doctor, schema help, docs, tests, precedence, and per-agent semantics.

## Reproduction Assessment

Yes, source-level reproduction is clear: current main falls through after model/provider runtime policy and explicitly tests/docs `agents.defaults.agentRuntime` as ignored. I did not run tests because this review is read-only, but the PR body also supplies terminal proof of the patched resolver returning `runtimeSource: "defaults"`.

## Solution Assessment

No, not as-is. The patch activates a legacy config key without updating the doctor migration, schema help, docs, and existing ignore-behavior test, and the related reviews still need a maintainer decision on precedence and per-agent semantics.
