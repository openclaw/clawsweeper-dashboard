## Summary

Keep this PR open, but do not merge it as-is: it re-enables a legacy whole-agent runtime key that current docs, tests, and recent merged runtime-policy work explicitly say is ignored. The PR has useful harness metadata propagation and real resolver proof, but the core behavior needs maintainer policy direction across the linked PR cluster.

## What This Changes

The PR makes `agents.defaults.agentRuntime` a fallback runtime policy source after model/provider policy and updates harness metadata plus related tests for the new `defaults` source.

## Best Possible Solution

Preserve the provider/model-scoped runtime contract unless maintainers explicitly change it; address the linked report through documented model/provider runtime policy, doctor guidance, or a new maintainer-approved subagent-specific contract.

## Reproduction Assessment

Yes for the observed behavior: current main can be traced in source to ignore `agents.defaults.agentRuntime` during runtime policy resolution. That behavior is currently documented and covered by tests, so reproducing it does not by itself prove a bug against main's contract.

## Solution Assessment

No as written. The patch reactivates a legacy whole-agent runtime pin globally, while the narrower maintainable path is to keep runtime policy on provider/model scopes unless maintainers explicitly decide to revise that contract.
