## Summary

Keep this PR open: the main auth migration approach is targeted and reuses doctor, but the current hook can still prompt on existing `--yes` no-confirmation command paths because it runs before Commander parses command options.

## What This Changes

The PR adds a macOS interactive CLI startup hook to migrate legacy Codex OAuth sidecar credentials through doctor, plus a headless warning, docs/changelog wording, and regression tests.

## Best Possible Solution

Keep doctor as the migration engine, but make the startup trigger skip every established no-prompt or machine-readable invocation before landing the auth migration policy.

## Reproduction Assessment

Yes, at source level: with a migratable sidecar on darwin + TTY, `openclaw reset --scope config --yes` reaches the new hook before command parsing, and the skip rule only checks JSON and `--non-interactive`, not `--yes`.

## Solution Assessment

No, not as currently written; reusing doctor is the right boundary, but the hook needs to preserve existing no-prompt command contracts before it is the best fix.
