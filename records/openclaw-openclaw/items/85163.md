## Summary

This PR should stay open: it is maintainer-labeled and the proposed CLI-triggered migration is plausible, but the new preflight can repeatedly prompt macOS users who only have unreferenced legacy sidecars because doctor intentionally leaves those files in place.

## What This Changes

The branch adds a macOS/TTY CLI startup hook that invokes the existing legacy Codex OAuth sidecar doctor migration, adds a headless warning when Keychain prompts are blocked, and updates tests, docs, changelog, and migration wording.

## Best Possible Solution

Keep doctor as the canonical migration path, but make the startup trigger require an active migratable `oauthRef` profile or persist a durable no-op/cool-off state for unreferenced-only detections, with focused regression coverage.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: on macOS with a TTY and only an unreferenced valid legacy sidecar JSON, preflight passes, doctor reports the sidecar as detected, accepting performs zero changes, and the next command sees the same sidecar again.

## Solution Assessment

No, not yet; the owner boundary is good, but the auto-trigger must distinguish active migratable profiles from retained unreferenced sidecars or cool off no-op accepts before merge.
