## Summary

Keep open: the latest head fixes the JSON-output blocker, but the startup hook can still prompt before command-specific `--non-interactive` flags are honored, which can interrupt existing non-interactive CLI workflows. Because this is a protected maintainer PR touching auth-provider upgrade behavior, it should land only after that narrow compatibility repair and maintainer acceptance of the first-run migration UX.

## What This Changes

The branch adds a macOS TTY-only CLI startup hook that offers to run the existing doctor migration for migratable legacy Codex OAuth sidecars, emits a one-shot headless warning, updates docs/changelog wording, and adds regression tests.

## Best Possible Solution

Keep doctor as the canonical migration path, make the startup hook skip both machine-readable and argv-level non-interactive invocations before prompting, then land only after maintainers accept the macOS first-run auth migration policy.
