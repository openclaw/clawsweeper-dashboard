## Summary

This PR should stay open: the core migration direction is well scoped, but the new startup prompt can still appear before commands that explicitly asked to suppress prompts, so it is not merge-ready yet.

## What This Changes

The PR adds a macOS interactive CLI startup hook that offers to run the existing legacy Codex OAuth sidecar doctor migration, adds a headless warning, updates doctor wording/docs/changelog, and adds focused regression tests.

## Best Possible Solution

Keep the startup-triggered doctor migration, but centralize or expand prompt-suppression detection so every CLI no-prompt contract is honored before the auth migration prompt can run.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: current main blocks Keychain prompts in embedded auth loading and documents `openclaw doctor --fix` as the required workaround. I did not run a live macOS Keychain repro in this read-only review.

## Solution Assessment

No, not as-is. Reusing the doctor migration from an interactive startup hook is maintainable, but the implementation must honor all existing no-prompt CLI contracts before it is the safest fix.
