## Best Possible Solution

Keep doctor as the canonical migration path, but make the startup trigger require an active migratable oauthRef profile or persist a durable no-op cooldown for unreferenced-only detections, with focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: on macOS with a TTY and only an unreferenced valid legacy sidecar JSON, preflight passes, doctor reports it as detected, accepting performs zero changes, and the next command sees the same sidecar again.

## Solution Assessment

No, not yet. Reusing doctor is the right boundary, but the startup hook must distinguish active migratable profiles from retained unreferenced sidecars or cool off accepted no-op detections before merge.
