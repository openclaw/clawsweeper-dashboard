## Summary

Keep open: the auth migration direction is useful, but the latest PR head still runs the interactive migration prompt before routed commands apply JSON stdout protections, so machine-readable CLI output can be corrupted or blocked.

## What This Changes

The PR adds a macOS interactive CLI startup hook to offer doctor-based migration for legacy Codex OAuth sidecar profiles, adds a headless Keychain warning, updates wording/docs/changelog, and adds regression tests.

## Best Possible Solution

Keep doctor as the canonical migration path, but make the startup hook skip machine-readable `--json` invocations before prompting and land only after maintainers accept the macOS first-run auth migration policy.

## Reproduction Assessment

Yes, at source level. On the PR head, a darwin TTY invocation with a migratable `oauthRef` sidecar and `--json` falls through `shouldSkip` and can prompt before `tryRouteCli` applies JSON stdout suppression; I did not run a live macOS Keychain prompt.

## Solution Assessment

No, not as-is. Reusing doctor from an interactive CLI hook is the right owner boundary, but the hook must honor machine-readable invocations before any prompt or doctor note can reach stdout.
