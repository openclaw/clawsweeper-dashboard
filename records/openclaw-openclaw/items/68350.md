## Summary

This PR should stay open, but it is not mergeable as-is: the Telegram idempotency cache can collapse distinct chunks/media sends, and the external contribution still lacks real behavior proof for the changed routing path.

## What This Changes

The PR threads reply-routing idempotency keys through outbound delivery and Telegram sends, adds Telegram send deduplication, normalizes Discord send/poll targets, and adds focused tests.

## Best Possible Solution

Repair the dedupe so retry attempts coalesce without collapsing distinct text chunks or media sends, keep the Discord target normalization, and require redacted real Telegram/Discord behavior proof before merge.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: route a Telegram reply over 4000 characters, or a payload with multiple media sends, with one idempotency key and later send units reuse the same Telegram cache key.

## Solution Assessment

No. Threading idempotency through routing is the right direction, but the dedupe must include a send-unit discriminator or sit above per-provider send calls so chunks and media remain distinct.
