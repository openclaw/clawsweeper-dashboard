## Summary

Keep open: the PR targets plausible channel reply-routing gaps, but it cannot merge as-is because the new Telegram idempotency cache can collapse distinct chunks or media sends, and the external PR still lacks real Telegram/Discord behavior proof.

## What This Changes

This PR threads inbound idempotency keys through reply routing and Telegram sends, adds a Telegram send dedupe cache, normalizes Discord send/poll targets, and adds focused tests.

## Best Possible Solution

Scope dedupe per outbound send unit or move it above provider sends, preserve the Discord target normalization, and require real Telegram/Discord proof before merge.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a routed Telegram reply that splits into multiple text chunks or media sends would reuse the same PR-added Telegram cache key for distinct provider sends.

## Solution Assessment

No. Threading idempotency through routing is directionally right, but the proposed per-send Telegram cache needs a chunk/media discriminator or should dedupe at the outer delivery attempt instead.
