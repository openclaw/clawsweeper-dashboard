## Summary

Keep open: the Discord target normalization is useful, but the Telegram idempotency design still has a blocking send-unit collision and the external PR still lacks real Telegram/Discord behavior proof.

## What This Changes

The PR threads reply-routing idempotency keys into outbound delivery and Telegram sends, adds Telegram send dedupe, normalizes Discord send/poll targets, and adds focused tests.

## Best Possible Solution

Scope dedupe per outbound send unit or move it above provider sends, preserve the Discord target normalization, and require real Telegram/Discord proof before merge.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: with this PR applied, a routed Telegram reply that splits into multiple text chunks or media sends would reuse the same PR-added Telegram cache key for distinct provider sends.

## Solution Assessment

No. Threading idempotency through routing is directionally useful, but the proposed per-send Telegram cache needs a chunk/media discriminator or should dedupe at the outer delivery attempt instead.
