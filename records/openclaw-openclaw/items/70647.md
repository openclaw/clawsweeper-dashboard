## Summary

Keep open. The PR is useful test-only coverage for reported Nemotron empty-turn hangs, but it is not merge-ready because it pins one-attempt/no-retry behavior while current main treats NVIDIA/Nemotron as an OpenAI-compatible provider and routes empty terminal turns through the retry path.

## What This Changes

The PR adds two regression tests in `src/agents/pi-embedded-runner/run.incomplete-turn.test.ts` for NVIDIA/Nemotron empty `stopReason="stop"` turns with unsigned thinking-only content and fully empty content.

## Best Possible Solution

Keep the coverage test-only, but make it faithful to the current provider contract: mock NVIDIA/Nemotron as `api: "openai-completions"` and assert the retry/exhaustion behavior, unless maintainers intentionally add a narrow production exception for NVIDIA/Nemotron and cover that exception explicitly.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: the PR tests rely on the harness default `api: "messages"`, while a current-main NVIDIA/Nemotron mock with `api: "openai-completions"` will enter the empty-response retry path.

## Solution Assessment

No. Test-only coverage is the right boundary, but the current implementation pins behavior that does not match current bundled NVIDIA metadata or the generic OpenAI-compatible retry policy.
