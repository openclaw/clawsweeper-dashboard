## Summary

Keep open: this is a recent external test-only PR with useful coverage intent, but the proposed tests still pin a no-retry path that does not match current NVIDIA/Nemotron provider metadata or the current OpenAI-compatible empty-turn retry contract.

## What This Changes

The PR adds two embedded-runner regression tests for NVIDIA/Nemotron empty terminal stop turns, asserting a visible terminal error and a single attempt.

## Best Possible Solution

Keep the regression coverage test-only, but model NVIDIA/Nemotron with api: "openai-completions" and assert the current retry, continuation, or exhaustion behavior unless maintainers deliberately add a production exception.

## Reproduction Assessment

Yes. Source-level reproduction is clear: the PR leaves mockedResolveModelAsync on the harness default api: "messages", while current NVIDIA/Nemotron catalog metadata is api: "openai-completions" and that API is retry-eligible.

## Solution Assessment

No. Test-only coverage is the right boundary, but this implementation pins no-retry behavior through an inaccurate model mock instead of matching the current provider contract.
