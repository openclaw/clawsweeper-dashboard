## Summary

Keep this PR open, but it is not merge-ready. The requested test-only Nemotron coverage is relevant and current main does not already contain exact Nemotron/NemoClaw coverage, but the proposed tests pin a no-retry path by leaving the harness on its default `api: "messages"` model while the bundled NVIDIA provider is `openai-completions` and current main retries non-visible OpenAI-compatible turns.

## What This Changes

The PR adds two embedded-runner regression tests for NVIDIA/Nemotron empty terminal stop turns, covering unsigned thinking-only content and fully empty content while asserting a visible error payload and a single attempt.

## Best Possible Solution

Keep the coverage test-only, but make it faithful to the current provider contract: mock NVIDIA/Nemotron as `api: "openai-completions"` and assert the retry/exhaustion or successful continuation behavior, unless maintainers intentionally add a narrow production exception for NVIDIA/Nemotron and cover that exception explicitly.

Yes. A source-level reproduction is clear: the PR's tests leave `mockedResolveModelAsync` on the harness default `api: "messages"`, while changing that mock to NVIDIA's current `api: "openai-completions"` sends the empty stop turn through the retry path instead of the asserted single attempt.

## Solution Assessment

No. Test-only regression coverage is the right boundary, but this implementation pins a no-retry behavior that does not match the bundled NVIDIA provider metadata or the current OpenAI-compatible empty-turn retry contract.
