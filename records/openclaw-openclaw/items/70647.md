## Summary

This PR is useful in intent but should not merge as written: it pins a no-retry NVIDIA/Nemotron path while the current provider catalog resolves that model through the OpenAI-compatible API, which enters the empty-response retry guard. It also still lacks contributor-supplied real behavior proof for the reported NVIDIA/Nemotron runtime path.

## What This Changes

Adds two embedded-runner regression tests for NVIDIA/Nemotron empty terminal turns that expect one attempt and a terminal error payload.

## Best Possible Solution

Keep the regression coverage test-only, but mock NVIDIA's resolved model as `api: "openai-completions"` and assert the current retry, continuation, or exhaustion behavior unless maintainers deliberately add a production exception.
