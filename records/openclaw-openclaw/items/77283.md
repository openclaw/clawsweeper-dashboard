## Summary

Keep open: current main still leaves the normal Discord model picker component path without an early component acknowledgement while it does route, model, and preference work before the first response.

## What This Changes

The issue asks for Discord model picker select/button interactions to acknowledge promptly after upgrading to 2026.5.3 so model selection does not hit Discord's interaction timeout.

## Best Possible Solution

Add an early deferred component acknowledgement for model picker fallback interactions and route subsequent picker refreshes through edit/follow-up semantics, with regression coverage for delayed model-data and preference reads.

## Reproduction Assessment

Yes, source-reproducible: the fallback components inherit defer=false and the normal handler awaits route/model/preference work before its first update. I did not establish a live Discord timeout run, so this is not marked reproduced.

## Solution Assessment

Yes, the requested direction is the right fix boundary, but the exact one-line deferUpdate suggestion needs adaptation to OpenClaw's interaction wrapper. The maintainable fix is to acknowledge early and then edit the original component response or use existing follow-up/edit semantics.
