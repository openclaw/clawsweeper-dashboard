## Summary

Keep open: current main still leaves the Discord model picker component path unacknowledged while it awaits route, model-catalog, and recent-preference work, so the reported interaction timeout remains a valid narrow Discord plugin bug.

## What This Changes

The issue reports that selecting a Discord model from the picker fails to acknowledge the component interaction after upgrading to OpenClaw 2026.5.3.

## Best Possible Solution

Discord model picker component interactions should acknowledge immediately and route all later picker refresh/apply responses through existing edit/follow-up semantics with regression coverage.

## Reproduction Assessment

Yes, source-reproducible: current main's picker button/select path does not defer for the owner interaction and awaits route/model/preference work before the first update. I did not live-run Discord, so this is not marked as fully reproduced.

## Solution Assessment

No, the issue's exact one-line deferUpdate direction is incomplete for this wrapper because interaction.update() sends an initial callback. The maintainable fix is early component acknowledgement plus edit/follow-up response handling using the existing interaction response-state helpers.
