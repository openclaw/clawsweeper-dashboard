## Summary

Keep open: current main still leaves owner Discord model picker component interactions unacknowledged while doing async route, model-data, and recent-preference work before the first component response, which matches the reported timeout path.

## What This Changes

The issue reports that selecting a Discord model picker component in `@openclaw/discord` 2026.5.3 times out before the model switch is applied.

## Best Possible Solution

Acknowledge valid model picker component interactions before route/model/preference I/O, then route later picker UI refreshes through edit-original semantics with focused regression coverage.

## Reproduction Assessment

Yes, source-reproducible: the fallback button/select owner path reaches route resolution, `loadDiscordModelPickerData`, and `readDiscordModelPickerRecentModels` before any acknowledgement or update. I did not mark it live reproduced because no Discord run was performed.

## Solution Assessment

Yes, the narrow direction is to acknowledge early in the Discord model picker component path, but a literal one-line `deferUpdate` equivalent is not sufficient unless subsequent updates use edit-original semantics instead of another callback.
