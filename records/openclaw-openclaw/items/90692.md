## What This Changes

The issue reports WebChat history reloads exposing assistant reasoning tags/content and adding a truncation marker after long streamed assistant messages are projected from transcript history.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/90924 or an equivalent gateway projection fix that strips assistant-visible reasoning/final scaffolding before truncation while preserving exact tool payloads.

## Reproduction Assessment

Yes, source-reproducible: current main routes `chat.history` through `projectRecentChatDisplayMessages`, where assistant text is directive-stripped and truncated without the shared reasoning sanitizer. I did not rerun the original Windows WebChat plus Gemini heartbeat session.

## Solution Assessment

Yes, the best fix direction is server-side history projection sanitization before truncation. Increasing or configuring the history cap alone would still allow malformed private scaffolding to reach history consumers.
