## Summary

Keep open. Current main still lacks a first-class path to import loose `.jsonl` session backups into the registered session store, so copied transcript files cannot appear in the native session menu unless a matching `sessions.json` row already exists.

## What This Changes

Request to add a way to import `.jsonl` session backup files so they become registered sessions visible in the session menu.

## Best Possible Solution

Add an explicit import flow that validates a transcript, copies or adopts it into the selected agent session directory, creates the matching `sessions.json` entry, and then lets the existing `sessions.list`, `sessions.preview`, `chat.history`, CLI, and Control UI paths render it normally.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to place a valid transcript `.jsonl` under an agent sessions directory without adding a `sessions.json` row; `sessions.list` only iterates store entries, so that transcript will not appear in the menu.

## Solution Assessment

Yes, an explicit `openclaw sessions import <path>` plus a shared Gateway/session helper is the narrowest maintainable direction. Startup auto-discovery would add broader path-safety and performance behavior, while an archive-only viewer would not satisfy the native session menu request.
