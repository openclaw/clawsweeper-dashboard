## Summary

Keep open. Current main still builds session menu rows from the registered session-store index, not from loose `.jsonl` transcript files, so the requested import/archive browsing workflow remains a new user-facing feature.

## What This Changes

Add a way for backed-up `.jsonl` session transcript files to be imported or browsed so they appear in the native session menu/history UI.

## Best Possible Solution

Define one supported archive workflow, then implement it through a validated import or read-only archive viewer that preserves the session-store contract instead of implicitly trusting loose files.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives high-confidence current behavior: `sessions.list` consumes the registered session store and has no import/discovery path for loose `.jsonl` files.

## Solution Assessment

Unclear. The issue lists several possible designs, and the maintainable path should be chosen before implementation, with explicit validation if transcript files can be imported.
