## Summary

Keep open: current main still lists sessions from the Gateway session store, and there is no supported CLI/RPC/UI path that imports or discovers loose `.jsonl` transcript backups into that store.

## What This Changes

The issue requests importing or discovering backed-up `.jsonl` session files so they appear in OpenClaw’s session menu.

## Best Possible Solution

Define a supported import path that validates an OpenClaw transcript, creates a durable session-store entry, exposes it through existing Gateway/UI session listing, and documents backup/archive semantics.

## Reproduction Assessment

Yes, from source inspection: `sessions.list` reads `sessions.json` store entries, so a copied transcript without a store entry is not selected. I did not run a live fixture because this review was constrained to read-only inspection.

## Solution Assessment

Unclear. An explicit import command/API looks like the safest starting point, but maintainer product judgment is needed before choosing import, discovery, or archive-viewer behavior.
