## Summary

Keep open: current main still leaves the broader Control UI ultrawide layout request unresolved for non-chat forms and tables, while the related chat-width issue and PR are narrower and do not cover the shared layout policy.

## What This Changes

Request ultrawide-aware Control UI layouts that cap or restructure stretched forms and tables on very wide viewports.

## Best Possible Solution

Define shared Control UI responsive layout primitives and apply them per surface so forms keep actions visually attached and wide tables use deliberate density, filters, preview/detail panels, or caps instead of stretching by default.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction path is to open Control UI Config or Sessions at a viewport wider than 2560px; current CSS still has an uncapped content shell, a full-width config action bar, and `width: 100%` sessions tables.

## Solution Assessment

Unclear. The requested direction is reasonable, but the maintainable fix should be a shared layout policy plus targeted Config/Sessions/table decisions rather than a blanket max-width across the whole app.
