## Summary

Keep open. Current main still leaves the broader Control UI ultrawide problem unresolved: Config and Sessions use full-width content/table paths, while the related chat-width issue and PR are narrower and do not cover forms, data tables, or a shared responsive layout policy.

## What This Changes

Request to adopt ultrawide-aware Control UI layouts by capping readable content, using side-panel or multi-column patterns, and avoiding stretched forms and tables on very wide viewports.

## Best Possible Solution

Define shared Control UI responsive layout primitives, then apply them per surface: cap readable form/detail content, keep section actions visually attached, and give wide tables explicit density, filter, or detail-panel behavior instead of stretching every view by default.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction path is to open Control UI Config or Sessions at a viewport wider than 2560px; current main still has an uncapped content shell, a full-width single-column config layout, and a `width: 100%` sessions data table.

## Solution Assessment

Unclear. The requested direction is reasonable, but the maintainable fix should be a shared layout policy plus targeted Config/Sessions/table decisions, not a one-size-fits-all max-width across the entire app.
