## Summary

Keep open. Current main still lacks a shared ultrawide-aware layout policy for the broader Control UI form and table surfaces; the related chat-width issue and PR are narrower and do not supersede this request.

## What This Changes

Request to make the Control UI ultrawide-aware by capping primary content, using multi-column or side-panel patterns for forms, and improving wide table ergonomics.

## Best Possible Solution

Define a shared Control UI responsive layout policy, then apply it per surface: cap readable form/detail content, keep actions visually attached to their sections, and handle wide tables with constrained containers, filters/detail panels, or density-specific layouts rather than stretching every view by default.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction path is to open the Control UI Config or Sessions view at a viewport wider than 2560px; current main still has an uncapped `.content`, a single-column `.config-layout`, and full-width `.data-table` styling.

## Solution Assessment

Unclear. The requested direction is reasonable, but the maintainable solution needs shared layout primitives plus targeted Config/Sessions/table decisions, not a one-size-fits-all max-width on the whole app.
