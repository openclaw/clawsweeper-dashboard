## Summary

Keep open. Current main only addresses the narrower configurable chat-message width path; the broader Control UI ultrawide request still has full-width shared shell, config/action, and table surfaces that need a deliberate layout policy.

## What This Changes

Request ultrawide-aware Control UI layouts with capped content, multi-column or side-panel patterns, and better form/table scanning on very wide displays.

## Best Possible Solution

Define shared Control UI ultrawide layout primitives and apply them deliberately to Config, Sessions/data tables, Usage, and other affected views while preserving surfaces that intentionally use wide layouts.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction path is Control UI Config or Sessions at a viewport wider than 2560px: `.content`, `.config-layout`/`.config-actions`, and `.data-table` still use full-width patterns on current main.

## Solution Assessment

Unclear. The requested direction is reasonable, but the maintainable fix should be a shared layout policy plus targeted per-view decisions rather than a single global max-width rule.
