## Summary

Keep open. Current main still has full-width Control UI form and table surfaces, and the related chat-width issue/PR only cover chat readability, not the broader ultrawide form/table layout request.

## What This Changes

Request ultrawide-aware Control UI layout patterns that cap or reorganize primary forms and tables instead of stretching every surface across very wide viewports.

## Best Possible Solution

Define a shared Control UI responsive layout policy, then apply it per surface: cap readable form/detail content, keep action controls visually attached to their sections, and handle wide tables with constrained containers, side filters/detail panels, or density-specific layouts instead of stretching all views by default.

Yes. A high-confidence reproduction path is to open the Control UI Config or Sessions views at a viewport wider than 2560px; current source shows the shared shell, Config actions, and shared data table still consume the available width.

## Solution Assessment

Unclear. The requested direction is sound, but the narrowest maintainable fix is likely shared layout primitives plus targeted Config/Sessions/table changes, not a blanket max-width across every Control UI view.
