## Summary

Keep open. Current main addresses the narrower grouped chat-message width request from #67935, but source inspection still shows full-width non-chat shell, Config actions, and Sessions table paths, so the broader ultrawide Control UI feature remains unresolved and needs maintainer layout policy.

## What This Changes

The issue requests ultrawide-aware Control UI layouts that cap or restructure forms and tables instead of stretching them across very wide viewports.

## Best Possible Solution

Define shared Control UI ultrawide layout primitives and per-surface rules, then apply them deliberately to Config, Sessions/data tables, and other affected views while preserving intentionally full-width surfaces.
