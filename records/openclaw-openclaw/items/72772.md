## Summary

Keep open. Current main has implemented the narrower configurable chat-message width path, but the broader Control UI ultrawide request still has source-backed full-width shell, Config action-bar, and shared table behavior that needs a deliberate layout policy.

## What This Changes

Request ultra-widescreen-aware Control UI layouts for forms, tables, and primary content instead of stretching those surfaces across the full viewport.

## Best Possible Solution

Define shared Control UI ultrawide layout primitives, then apply them intentionally to Config/forms, Sessions/data tables, Usage/Quick Settings, and other affected views while preserving chat-width configuration and surfaces that legitimately need full width.

## Reproduction Assessment

Yes. Source inspection on current main shows `.content` is uncapped, Config actions still split status and buttons across flex space, and shared data tables still render at `width: 100%`, which is a high-confidence source reproduction of the ultrawide stretch pattern.

## Solution Assessment

Unclear. The requested direction is reasonable, but the best maintainable solution is a shared Control UI layout policy plus targeted per-view changes, not a blanket max-width or immediate preference toggle.
