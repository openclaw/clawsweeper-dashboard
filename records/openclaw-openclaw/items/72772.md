## Summary

Keep open. Current main has a narrow chat-message width setting, but the broader Control UI ultrawide request still applies to shared shell, Config form/action, and Sessions table surfaces and needs a deliberate layout policy.

## What This Changes

Adopt ultrawide-aware Control UI layouts for forms, tables, and primary content instead of stretching them to the full viewport width.

## Best Possible Solution

Define shared Control UI ultrawide layout primitives and apply them selectively to Config, Sessions, and shared form/table surfaces while preserving intentionally full-width views.

## Reproduction Assessment

Yes. Source inspection on current main shows the non-chat shell, Config action bar, and Sessions data table still use full-width patterns that reproduce the reported ultrawide behavior.

## Solution Assessment

Unclear. The requested direction is reasonable, but the maintainable fix should be a shared layout policy plus targeted view changes, not a blanket global max-width or immediate preference toggle.
