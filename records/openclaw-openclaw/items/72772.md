## Summary

Keep open. Current main only addresses the narrower configurable chat-message width path, while the broader ultrawide form/table layout request still reproduces in Control UI shared content, Config, and Sessions surfaces.

## What This Changes

Request to make Control UI layouts ultrawide-aware by capping or restructuring forms, tables, and wide content instead of stretching primary content across very wide viewports.

## Best Possible Solution

Define shared Control UI ultrawide layout primitives and apply them deliberately to Config, Sessions/data tables, Usage, and other affected views while preserving surfaces that intentionally use wide space.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction path is Control UI Config or Sessions at a viewport wider than 2560px: `.content`, `.config-layout`/`.config-actions`, and `.data-table` still use full-width patterns.

## Solution Assessment

Unclear. The requested direction is reasonable, but the maintainable fix should be a shared layout policy plus targeted per-view decisions rather than a one-size-fits-all max-width across the app.
