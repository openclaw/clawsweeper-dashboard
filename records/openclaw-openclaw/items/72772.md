## Summary

Keep open. Current main still leaves broad Control UI form, shell, and table surfaces using full-width patterns, so the core ultrawide readability request remains unresolved outside the narrower chat-message width work.

## What This Changes

The issue asks for ultrawide-aware Control UI layouts that cap or restructure wide forms and tables instead of stretching primary content across very large viewports.

## Best Possible Solution

Define shared Control UI ultrawide layout primitives and apply them deliberately to Config, Sessions/data tables, Usage, and related views while preserving surfaces that intentionally use full-width space.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction path is Config or Sessions at a viewport wider than 2560px: `.content`, `.config-layout`/`.config-actions`, and `.data-table` still use full-width patterns on current main.

## Solution Assessment

Unclear. The requested direction is reasonable, but the maintainable fix should be a shared layout policy plus targeted per-view decisions rather than a blanket max-width applied across the app.
