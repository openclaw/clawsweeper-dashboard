## Summary

Keep open: current main resolves only the narrower configurable chat-message width case, while the broader Control UI ultrawide layout request for shared shell, forms, and tables remains unresolved and needs a layout policy decision.

## What This Changes

The issue requests ultrawide-aware Control UI layouts that cap or repurpose very wide form and table surfaces instead of stretching them across the viewport.

## Best Possible Solution

Define shared Control UI ultrawide layout primitives, then apply them deliberately to Config/forms, Sessions/data tables, Usage/Quick Settings, and similar views while preserving the existing chat-width setting and views that legitimately need full width.

## Reproduction Assessment

Yes. Source inspection of current main shows uncapped shared content, Config actions that spread across available width, and Sessions rendered through a full-width shared data table; I did not run a live ultrawide browser screenshot.

## Solution Assessment

Unclear. The requested direction is reasonable, but the maintainable fix should be a shared Control UI layout policy plus targeted per-view treatment, not a blanket max-width or immediate preference toggle.
