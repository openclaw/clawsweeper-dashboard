## Summary

Keep open. Current main still leaves the broad Control UI ultrawide request unresolved: shared non-chat content, Config, and Sessions tables continue to use full-width patterns, while the related chat-width issue and PR are narrower and do not cover forms, tables, or shared layout policy.

## What This Changes

Request ultrawide-aware Control UI layout patterns that cap or repurpose wide form and table space instead of stretching all primary content across the viewport.

## Best Possible Solution

Define shared Control UI responsive layout primitives, then apply them deliberately to Config, Sessions/data tables, and other affected views while preserving existing per-view wide layouts that already use caps or dense side-by-side panels.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction path is to open Control UI Config or Sessions at a viewport wider than 2560px; current `.content`, `.config-layout`/`.config-actions`, and `.data-table` styles still let those surfaces occupy the full available width.

## Solution Assessment

Unclear. The requested direction is reasonable, but the maintainable fix should be a shared layout policy plus targeted Config/Sessions/table decisions rather than a one-size-fits-all max-width across the app.
