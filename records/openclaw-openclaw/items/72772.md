## Summary

Keep open. Current main has chat-message max-width support, but the broader ultrawide Control UI request still applies to shared content, Config actions, and data-table surfaces and needs a shared layout policy before implementation.

## What This Changes

Adopt ultrawide-aware Control UI layout patterns for forms, tables, and primary content instead of stretching them across very wide viewports.

## Best Possible Solution

Define shared Control UI ultrawide layout primitives, then apply them deliberately to Config actions/forms, Sessions/data tables, and other affected views while preserving intentionally full-width surfaces.

## Reproduction Assessment

Yes for the reported layout pattern. Source inspection on current main shows uncapped `.content`, right-stretched `.config-actions`, and `width: 100%` data tables that match the ultrawide behavior described.

## Solution Assessment

Unclear. The requested direction is reasonable, but the best maintainable solution is a shared Control UI layout policy plus targeted per-view changes, not a blanket max-width or immediate preference toggle.
