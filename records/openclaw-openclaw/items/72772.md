Keep open: current main has handled the narrower configurable chat-message width path, but the broader Control UI ultrawide request for shared shell, form actions, and table layout policy is still unresolved.

## What This Changes

The issue requests ultrawide-aware Control UI layouts that cap, split, or repurpose wide viewport space for forms and tables instead of stretching every surface full-width.

## Best Possible Solution

Define shared Control UI ultrawide layout primitives, then apply them deliberately to Config/forms, Sessions/data tables, Usage/Quick Settings, and similar views while preserving the existing chat-width setting and views that legitimately need full width.
