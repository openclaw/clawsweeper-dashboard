## Summary

Keep open: current main solves the narrower configurable chat-message width problem, but the broader Control UI ultrawide policy for shared shell content, Config action bars/forms, and Sessions/data tables remains unresolved and needs maintainer layout direction.

## What This Changes

Request to make the Control UI more usable on ultrawide monitors by capping overly stretched forms/tables or using multi-column/side-panel layout patterns.

## Best Possible Solution

Define shared Control UI ultrawide layout primitives, then apply them deliberately to Config/forms, Sessions/data tables, Usage/Quick Settings, and similar views while preserving the existing chat-specific width setting.

## Reproduction Assessment

Yes for a source-level layout concern: current main still has uncapped shared content, widely distributed Config actions, and a full-width Sessions table. This is not a strict bug repro because the request is for a new layout policy rather than a broken documented contract.

## Solution Assessment

Unclear. The requested direction is reasonable, but the maintainable fix should be a shared layout policy plus targeted per-view treatment, not a blanket max-width or immediate preference toggle.
