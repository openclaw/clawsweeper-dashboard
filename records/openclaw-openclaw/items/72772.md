## Summary

Keep open. Current main only covers the narrower chat-message width case; the broader Control UI request still has source-backed full-width shell, Config action-bar, and table surfaces that need a shared ultrawide layout policy.

## What This Changes

Request ultrawide-aware Control UI layouts that cap or restructure forms and tables instead of stretching them across the full viewport.

## Best Possible Solution

Define shared Control UI ultrawide layout primitives, then apply them deliberately to Config forms/action bars, Sessions/data tables, and other affected non-chat views while preserving views that intentionally need full width.

## Reproduction Assessment

Yes for the reported current layout pattern. Source inspection on current main shows representative non-chat shell, Config actions, and Sessions table paths still using full-width behavior without needing a live UI run.

## Solution Assessment

Unclear. The requested direction is reasonable, but the maintainable fix is a shared Control UI layout policy plus targeted per-view changes, not a blanket max-width or immediate preference toggle.
