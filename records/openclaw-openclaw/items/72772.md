## Summary

Keep open: current main and v2026.6.8 cover the narrower chat message-width setting, but the broader Control UI ultrawide policy for shared shell, config action bars, and data tables remains unresolved and needs maintainer design direction.

## What This Changes

The issue requests ultrawide-aware Control UI layouts that cap or split wide forms and tables instead of stretching primary content across very large viewports.

## Best Possible Solution

Define shared Control UI ultrawide layout primitives, then apply them deliberately to Config/forms, Sessions/data tables, Usage/Quick Settings, and similar views while preserving the existing chat-specific width setting.

## Reproduction Assessment

Yes for the source-level layout concern: current main still has uncapped shared content, widely separated Config action groups, and full-width Sessions table rendering. This is not a strict bug repro because the request is for a new layout policy rather than a broken documented contract.

## Solution Assessment

Unclear. The requested direction is reasonable, but the maintainable fix should be shared layout policy plus targeted per-view treatment, not a blanket max-width or immediate preference toggle.
