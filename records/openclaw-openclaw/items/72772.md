## Summary

Keep open. Current main and v2026.5.2 address the narrower grouped chat-message width request, but the broader Control UI ultrawide layout request still has source-backed full-width form, shell, and table surfaces that need a shared design policy.

## What This Changes

Adopt ultrawide-aware Control UI layouts so forms, actions, and data tables do not stretch across 21:9/32:9 viewports.

## Best Possible Solution

Define shared Control UI ultrawide layout primitives, then apply them deliberately to Config actions/forms, Sessions/data tables, and other affected views while preserving surfaces that intentionally use full width.

## Reproduction Assessment

Yes. Source inspection on current main shows `.content`, `.config-layout`/`.config-actions`, and `.data-table` still use full-width patterns that reproduce the reported ultrawide behavior without needing a failing test.

## Solution Assessment

Unclear. The requested direction is reasonable, but the maintainable fix should be a shared layout policy plus targeted per-view changes rather than a blanket max-width or an unreviewed preference toggle.
