## Summary

Keep open. Current main still leaves the broad Control UI ultrawide request unresolved: Config and Sessions retain full-width action/table patterns, and the related chat-width issue/PR only cover a narrower chat surface.

## What This Changes

Adopt ultrawide-aware Control UI layouts that cap or reorganize forms and tables instead of stretching them across very wide viewports.

## Best Possible Solution

Define shared Control UI responsive layout primitives, then apply them per surface so forms keep actions visually attached and wide tables use deliberate density, filters, detail panels, or caps.

## Reproduction Assessment

Yes. A high-confidence source-level path is to open Config or Sessions at a viewport wider than 2560px; current CSS still has an uncapped content shell, right-justified config actions, and a full-width sessions data table.

## Solution Assessment

Unclear. The requested direction is reasonable, but the maintainable fix should be shared layout guidance plus targeted Config/Sessions/table decisions, not a one-size-fits-all max-width across the app.
