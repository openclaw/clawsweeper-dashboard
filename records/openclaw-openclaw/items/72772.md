## Summary

Keep open. Current main still leaves the broader Control UI ultrawide request unresolved: Config and Sessions retain full-width form/action and table patterns, while the related chat-width issue and PR are narrower and do not cover shared layout policy, forms, or data tables.

## What This Changes

Request ultrawide-aware Control UI layouts, including capped readable content, two-column form/action patterns, and better use of wide table space instead of stretching every surface full-width.

## Best Possible Solution

Define shared Control UI responsive layout primitives and apply them per surface: cap readable form/detail content, keep section actions visually attached, and give wide tables deliberate density, filters, or detail panels instead of stretching every view by default.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction path is to open Control UI Config or Sessions at a viewport wider than 2560px; current CSS still has an uncapped content shell, a full-width single-column config layout/action bar, and a `width: 100%` sessions data table.

## Solution Assessment

Unclear. The requested direction is reasonable, but the maintainable fix should be a shared layout policy plus targeted Config/Sessions/table decisions rather than a one-size-fits-all max-width across the whole app.
