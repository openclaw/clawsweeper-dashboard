## Summary

Keep open. Current main addresses the narrower chat-message width path, but the broader Control UI ultrawide request still has source-backed full-width shell, form/action, and table surfaces and needs a shared layout policy before implementation.

## What This Changes

Adopt ultrawide-aware Control UI layouts so forms, tables, and action areas do not stretch across very wide viewports.

## Best Possible Solution

Define shared Control UI ultrawide layout primitives, then apply them deliberately to Config actions/forms, Sessions/data tables, and other affected views while preserving intentionally full-width surfaces.

## Reproduction Assessment

Yes. Source inspection on current main shows the non-chat shell, Config action/form layout, and Sessions table still use full-width patterns that reproduce the reported ultrawide behavior.

## Solution Assessment

Unclear. The requested direction is reasonable, but the maintainable path is a shared Control UI layout policy with targeted per-view changes rather than a one-size-fits-all max-width or immediate preference toggle.
