## Summary

Keep open. Current main covers the narrower chat-message width configuration path, but the broader Control UI ultrawide request still has unresolved full-width shell, Config form/action, and Sessions table surfaces.

## What This Changes

Request ultrawide-aware Control UI layouts that cap or restructure forms and tables instead of stretching primary content to the full viewport width.

## Best Possible Solution

Define shared Control UI ultrawide layout primitives, then apply them deliberately to Config, Sessions/data tables, Usage, and other affected views while preserving purposeful wide layouts.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction path is Control UI Config or Sessions at a viewport wider than 2560px: current main still shows uncapped `.content`, full-width Config actions, and shared tables set to `width: 100%`.

## Solution Assessment

Unclear. The requested direction is reasonable, but the maintainable fix should be a shared layout policy plus targeted per-view decisions rather than a one-size-fits-all cap across the app.
