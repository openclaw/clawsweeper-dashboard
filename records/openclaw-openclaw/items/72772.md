## Summary

Keep open. Current main still leaves the broader Control UI ultrawide layout request unresolved: Config, Sessions, and shared table/content surfaces still use full-width patterns, while the related chat-width items only cover the chat message column.

## What This Changes

Request ultrawide-aware Control UI layouts that cap or repurpose wide space for forms, tables, side panels, and related actions instead of stretching primary content across the full viewport.

## Best Possible Solution

Define shared Control UI responsive layout primitives, then apply them per surface: cap readable form/detail content, keep section actions visually attached, and give wide tables explicit density, filter, or detail-panel behavior instead of stretching every view by default.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction path is to open Control UI Config or Sessions on a viewport wider than 2560px; current main still has an uncapped content shell, a full-width single-column config layout, and a `width: 100%` sessions data table.

## Solution Assessment

Unclear. The requested direction is reasonable, but the maintainable fix should be a shared layout policy plus targeted Config/Sessions/table decisions, not a one-size-fits-all max-width across the whole app.
