## Summary

Keep open. Current main has narrow chat message max-width support, but the broader Control UI ultrawide request still has source-backed full-width shell, Config, and table surfaces and needs a shared layout policy.

## What This Changes

The issue requests ultrawide-aware Control UI layout patterns that cap or repurpose wide horizontal space for forms and tables.

## Best Possible Solution

Define shared Control UI ultrawide layout primitives, then apply them deliberately to Config actions/forms, Sessions/data tables, and other affected views while preserving surfaces that intentionally need full width.

## Reproduction Assessment

Yes. Source inspection on current main shows `.content`, `.config-actions`, and `.data-table` still use full-width patterns that reproduce the reported ultrawide behavior without needing a failing test.

## Solution Assessment

Unclear. The requested direction is reasonable, but the narrow maintainable path is a shared layout policy plus targeted per-view changes, not a blanket max-width or immediate preference toggle.
