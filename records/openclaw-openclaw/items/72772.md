## Summary

Keep open. Current main addresses the narrower grouped chat-message width problem, but the broader Control UI ultrawide form and table layout remains source-reproducible and needs a shared layout policy before implementation.

## What This Changes

Request asks for ultrawide-aware Control UI layouts that cap or repurpose wide content space for forms and tables.

## Best Possible Solution

Define shared Control UI wide-screen layout primitives, then apply per-view caps, columns, or side panels to config forms and data tables while preserving intentionally full-width admin views.

## Reproduction Assessment

Yes. Source inspection of current main shows uncapped non-chat content, config actions separated across a full-width bar, and shared data tables using `width: 100%`; opening those views above 2560px exercises the reported behavior.

## Solution Assessment

Unclear. The requested direction is reasonable, but the best maintainable fix is a shared layout policy plus targeted per-view changes rather than a blanket max-width or immediate preference toggle.
