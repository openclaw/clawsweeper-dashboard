## Summary

Keep open. Current main still lacks a shared ultrawide-aware Control UI layout policy for the reported form and table surfaces; the related chat-width issue and PR cover a narrower chat readability problem and do not supersede this broader request.

## What This Changes

Request ultrawide-aware Control UI layouts that cap or restructure wide forms and tables instead of stretching primary content across very wide viewports.

## Best Possible Solution

Define shared Control UI responsive layout primitives for wide screens, then apply them per surface: cap readable form/detail content, keep action groups visually attached to their section, and handle wide tables with constrained containers or purposeful adjacent panels rather than a blanket full-app width cap.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction path is to open the Control UI Config or Sessions view at a viewport wider than 2560px; current main still has an uncapped `.content`, a single-column `.config-layout`, right-justified Config actions, and full-width shared `.data-table` styling.

## Solution Assessment

Unclear. The requested direction is reasonable, but the best maintainable fix is likely a shared layout policy plus targeted Config/Sessions/table changes, not a one-size-fits-all CSS max-width across the whole UI.
