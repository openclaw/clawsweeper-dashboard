Keep open: current main covers the narrower configurable chat-message width problem, but the broader Control UI ultrawide layout request remains unresolved for shared shell, Config actions, and shared table surfaces.

Request ultrawide-aware Control UI layouts that avoid stretching forms and tables across very wide viewports.

## Best Possible Solution

Define shared Control UI ultrawide layout primitives, then apply them deliberately to Config/forms, Sessions/data tables, Usage/Quick Settings, and similar views while preserving chat-width configuration and views that legitimately need full width.

## Solution Assessment

Unclear. The requested direction is reasonable, but the maintainable fix should be a shared Control UI layout policy plus targeted per-view treatment, not a blanket max-width or immediate preference toggle.
