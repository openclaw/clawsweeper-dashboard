## Summary

Keep open. Current main still lacks a shared ultrawide-aware layout policy for the Control UI forms and data-table surfaces; the linked chat-width issue and PR are related but only cover chat readability and do not supersede the broader forms/tables request.

## What This Changes

The issue requests ultrawide-aware Control UI layouts that cap or repurpose very wide form and table surfaces instead of stretching them across 21:9 and 32:9 viewports.

## Best Possible Solution

Define a shared Control UI responsive layout policy, then apply it per surface: cap readable form/detail content, keep section actions visually attached to their context, and handle wide tables with constrained containers, side filters/detail panels, or density-specific layouts instead of stretching every view by default.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction path is to open the Control UI Config or Sessions view at a viewport wider than 2560px; current main still has an uncapped `.content`, a single-column `.config-layout`, separated Config action groups, and a full-width `.data-table`.

## Solution Assessment

Unclear. The requested direction is reasonable, but the maintainable fix is likely shared layout primitives plus targeted Config/Sessions/table changes, not a one-size-fits-all max-width on the whole app.
