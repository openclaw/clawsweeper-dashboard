## Summary

Keep open. Current main handles the narrower grouped chat-message width request, but the broader Control UI ultrawide request is still visible in full-width config/action and data-table surfaces and needs a shared layout policy before implementation.

## What This Changes

Request ultrawide-aware Control UI layouts that cap or restructure forms and tables instead of stretching across 21:9 and 32:9 viewports.

## Best Possible Solution

Define shared Control UI ultrawide layout primitives, then apply them deliberately to Config, Sessions, Usage, Quick Settings, and other affected views while preserving surfaces that should remain full width.

## Reproduction Assessment

Yes. Source inspection on current main shows full-width `.content`, `.config-layout`/`.config-actions`, and `.data-table` paths that reproduce the reported ultrawide behavior without needing a failing runtime test.

## Solution Assessment

Unclear. The requested direction is reasonable, but the narrowest maintainable fix is a shared layout/design policy plus targeted per-view changes, not a one-size-fits-all max-width or immediate preference toggle.
