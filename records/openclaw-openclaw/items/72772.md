## Summary

Keep open. Current main fixes the narrower configurable chat-message width path, but the broader Control UI ultrawide request still has unresolved full-width form/table surfaces and needs a shared layout policy.

## What This Changes

Request ultrawide-aware Control UI layout patterns for forms, tables, and primary content instead of stretching them across 21:9/32:9 viewports.

## Best Possible Solution

Define shared Control UI ultrawide layout primitives and apply them deliberately to Config, Sessions/data tables, and other affected views while preserving surfaces that already use purposeful wide layouts.

Yes. A high-confidence source-level reproduction path is Control UI Config or Sessions at a viewport wider than 2560px: `.content`, `.config-layout`/`.config-actions`, and `.data-table` still use full-width patterns.

## Solution Assessment

Unclear. The requested direction is reasonable, but the maintainable fix should be a shared layout policy plus targeted per-view decisions rather than a one-size-fits-all max-width across the app.
