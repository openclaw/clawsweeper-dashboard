## Summary

Keep open. Current main ships the Korean Control UI locale, docs, and structural tests, but the normal agent prompt still has no general reply-language persistence policy and web search only accepts explicit language parameters instead of deriving Korean-aware hints; the paired implementation PR is closed unmerged.

## What This Changes

The issue requests Korean Control UI localization plus persistent Korean agent responses and Korean-aware web-search language handling.

## Best Possible Solution

Keep the shipped Korean Control UI locale, then define a generic user-language persistence policy and provider-aware search-language hint contract that applies across locales without Korean-specific prompt hardcoding.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence current-state check: Korean UI support is present, while normal reply-language persistence and automatic Korean search-language selection are absent.

## Solution Assessment

Unclear for the remaining work. The UI locale implementation is in the right surface, but a Korean-specific prompt block or Brave-only default would be a weaker fix than a generic language policy and search hint contract.
