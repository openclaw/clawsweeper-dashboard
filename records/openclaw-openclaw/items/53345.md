## Summary

Keep open. Current main already implements the Korean Control UI locale, and that portion is present in the v2026.4.27 release, but the requested normal agent response-language persistence and automatic Korean-aware search behavior are still not implemented; the paired implementation PR is closed unmerged.

## What This Changes

The request asks to add Korean Control UI localization plus persistent Korean agent replies and Korean-aware web search language parameters.

## Best Possible Solution

Keep the shipped Korean Control UI locale, then design and implement a generic user-language persistence policy and provider-aware search-language hint contract that works across locales without Korean-specific hardcoding.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence current-state check: Korean UI localization is present, while normal agent reply-language persistence and automatic search-language selection are absent.

## Solution Assessment

Unclear for the remaining work. The UI portion is already in the right owner surface, but Korean-specific prompt text or Brave-only defaults would be narrower than the product need; a generic language policy and provider-aware search contract would be safer.
