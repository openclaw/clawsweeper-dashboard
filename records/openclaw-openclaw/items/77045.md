## Summary

Keep open. The PR targets the right Slack cascade bug, but its metadata classifier misses the current `Model login failed on the gateway` OAuth-refresh failure text, so one gateway-auth failure path can still echo through shared Slack rooms with `allowBots: true`; the PR also carries the protected `maintainer` label and needs explicit maintainer handling.

## What This Changes

The PR adds Slack metadata tagging for OpenClaw gateway failure messages, drops tagged bot-authored room echoes on inbound handling, updates Slack message typing, adds regression tests, and adds a changelog entry for #51832.

## Best Possible Solution

Keep the metadata-origin suppression design, but cover every gateway auth/failure reply string that can be posted to Slack and preserve existing bot-to-bot interop for untagged messages.

## Reproduction Assessment

Yes at source level: current main emits `⚠️ Model login failed on the gateway...` for some OAuth refresh failures, while the PR only tags the expired-login variant. I did not run a live Slack flood reproduction in this read-only review.

## Solution Assessment

No, not yet. Tagging gateway failure messages and dropping only tagged echoes is the narrow maintainable approach, but the prefix set must include `Model login failed on the gateway` before this fully solves the reported cascade.
