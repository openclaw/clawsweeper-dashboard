## Best Possible Solution

Merge the narrow retry/fallback with its regression coverage once the separate dependency-audit gate is resolved, preserving npm alias overrides by default and falling back only for the older-npm comparator error.

## Solution Assessment

Yes for the install regression. The PR keeps normal managed overrides intact and retries only after the exact npm comparator failure, which is narrower than removing the alias override globally or adding a config knob.
