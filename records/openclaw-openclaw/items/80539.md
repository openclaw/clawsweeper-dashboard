## Summary

This is a plausible focused fix for a real managed npm plugin install edge case, but the PR should stay open because current main does not contain the retry behavior and the external-PR real behavior proof is still too thin for merge.

## What This Changes

The PR retries managed npm plugin installs after filtering npm alias override leaves when npm reports `Invalid comparator: npm:`.

## Best Possible Solution

Land this narrow retry approach after the contributor adds redacted real behavior proof from an OpenClaw managed npm plugin install on an affected npm version.

## Reproduction Assessment

No, not as live-verified in this review. The current source path and PR body make the failure credible for npm 9.2.0, but the review did not establish a high-confidence current-main runtime reproduction.

## Solution Assessment

Yes, the proposed direction is the narrow maintainable fix: preserve aliases for npm versions that accept them, and only retry the managed root with alias override leaves removed after the specific npm comparator failure.
