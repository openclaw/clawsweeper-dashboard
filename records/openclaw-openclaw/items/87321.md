## What This Changes

The issue reports that WebChat replay loses user-visible `sessions_yield` wait/status text when it is persisted only as a tool-result payload.

## Best Possible Solution

Land one canonical gateway display-projection fix that mirrors successful `sessions_yield` wait text as assistant-visible replay while preserving raw tool results for debug/show-tool/API consumers.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence repro: build replay history with a `sessions_yield` `toolResult` containing `{status:"yielded", message:"..."}` and default WebChat will hide that raw tool result because current projection has no `sessions_yield` assistant mirror.

## Solution Assessment

Yes. Gateway display projection is the right layer because `chat.history` is already display-normalized; the remaining maintainer choice is which linked PR should become the canonical implementation.
