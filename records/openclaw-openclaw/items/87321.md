- `impact:message-loss`: A status message visible during the live session can disappear after WebChat refresh or replay.

## Summary

Keep open: current main and the latest release still only mirror `message` tool sends for WebChat replay, while `sessions_yield` wait text remains stored as a hidden tool result; the two linked open PRs are candidate fixes and should be resolved before this issue closes.

## What This Changes

Make WebChat replay keep user-visible `sessions_yield` status text durable instead of losing it when the text is stored only as a tool result.

## Best Possible Solution

Land one canonical Gateway display-projection fix that mirrors successful `sessions_yield` wait text as assistant-visible replay while preserving raw tool results for debug/show-tool/API consumers.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence repro: replay history with a `sessions_yield` `toolResult` containing `{status:"yielded", message:"..."}` has no assistant mirror, and default WebChat hides the raw tool result.

## Solution Assessment

Yes. Gateway display projection is the right layer because `chat.history` is already display-normalized; the remaining maintainer choice is which linked PR should become the canonical implementation.
