## Label Justifications

- `impact:message-loss`: A status message visible during the live session can disappear after WebChat refresh or replay.
- `impact:session-state`: The failure is tied to persisted session transcript replay and subagent wait-state visibility.

## Summary

Keep open: current main and the latest release still only mirror `message` tool sends during WebChat replay, while `sessions_yield` keeps the visible wait text as a hidden tool-result/context path. Two linked fix PRs already target this issue but both remain open and conflicting, so the issue should stay open until one is landed or replaced.

## What This Changes

The issue reports that WebChat replay loses user-visible `sessions_yield` status text because it is persisted only as a hidden tool-result payload.

## Best Possible Solution

Land or replace the open WebChat replay projection fix so successful `sessions_yield` tool results produce durable assistant-visible replay text while raw tool results remain hidden by default.

## Reproduction Assessment

Yes, source-reproducible: construct history with a `sessions_yield` `toolResult` containing `{status:"yielded", message:"..."}` and default WebChat replay will not render it because the projection lacks a `sessions_yield` mirror and the UI hides tool results.

## Solution Assessment

Yes: the narrow maintainable fix is to extend the existing display projection/mirror pattern to successful `sessions_yield` results rather than making generic tool results visible.
