## Summary

Keep open: current main and the latest release still lack a `sessions_yield` replay mirror, and two linked closing PRs remain open instead of merged. The source path matches the report, so the issue should stay open until maintainers choose and land one canonical fix.

## What This Changes

WebChat should keep user-visible `sessions_yield` wait/status text durable across transcript refresh or replay instead of losing it when it is stored only as a tool result.

## Best Possible Solution

Land one canonical gateway display-projection fix that mirrors successful `sessions_yield` wait text as assistant-visible replay while preserving raw tool results for debug and API consumers.

## Reproduction Assessment

Yes. A transcript containing a `sessions_yield` `toolResult` with `{ status: "yielded", message }` will pass through current replay projection without an assistant mirror, and default WebChat rendering hides raw tool results.

## Solution Assessment

Yes, extending the gateway display projection is the right boundary; making generic tool results visible would expose too much raw tool payload. The remaining maintainer decision is which linked PR should be refreshed and landed.
