## Summary

Keep open: current main still lacks a `sessions_yield` replay mirror, default WebChat replay hides raw `toolResult` rows, and two open closing PRs already target this exact bug but have not merged.

## What This Changes

WebChat replay should preserve user-visible `sessions_yield` status text stored as a tool result instead of dropping it after refresh.

## Best Possible Solution

Land one canonical gateway display-projection fix that mirrors successful `sessions_yield` wait text as assistant-visible replay while preserving raw tool results for debug/show-tool/API consumers.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence repro: build replay history with a `sessions_yield` `toolResult` containing `{status:"yielded", message:"..."}` and default WebChat will hide that raw tool result because current projection has no `sessions_yield` assistant mirror.

## Solution Assessment

Yes, the right layer is the gateway display projection rather than making generic tool results visible. The remaining question is which of the two open fix PRs should become the canonical implementation.
