## What This Changes

The issue reports that WebChat replay can lose user-visible `sessions_yield` status text because the durable transcript stores it only as a tool result.

## Best Possible Solution

Land one canonical gateway display-projection fix that preserves raw `sessions_yield` tool results for debug/API use while replaying exactly one assistant-visible wait message in default WebChat history.

## Reproduction Assessment

Yes. Source inspection shows a high-confidence current-main path: `sessions_yield` persists the wait text as a `toolResult`, the display projection only mirrors `message` tool sends, and default WebChat hides raw tool results.

## Solution Assessment

Yes. The narrow maintainable solution is a gateway display-projection/session-history fix, not rendering generic raw tool-result JSON or adding a second persisted assistant transcript path.
