## Summary

Keep open: current main still appears to lose this replay-only status path because `sessions_yield` stores the visible message as a tool result, WebChat projection only promotes `message` tool sends, and the UI hides tool-result history when tool calls are not shown.

## What This Changes

The issue requests durable WebChat replay for user-visible `sessions_yield` status text that currently survives only as a persisted tool-result payload.

## Best Possible Solution

Add a durable replay projection for successful `sessions_yield` messages, preferably in the WebChat display projection with focused gateway/UI regression coverage.

## Reproduction Assessment

Yes, source-reproducible: construct replay history where `sessions_yield` exists only as a `toolResult` with `{status:"yielded", message:"..."}` and WebChat will not promote it to assistant-visible text. I did not run a live WebChat refresh because this review is read-only.

## Solution Assessment

Yes: the narrowest maintainable fix is to mirror or persist successful `sessions_yield` messages as assistant-visible replay text, following the existing `message` tool mirror pattern rather than exposing generic tool results.
