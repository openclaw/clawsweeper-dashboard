## Summary

Keep open. Current main and v2026.6.6 still keep ordinary Gateway startup on validation/runtime-only changes plus manual `openclaw doctor --fix` guidance, while the issue discussion shows credible security and channel-availability risks from running broad doctor repair unattended at startup.

## What This Changes

Request to make Gateway startup automatically apply safe doctor repairs, or expose a `doctor.autoFix` option, while logging any applied changes.

## Best Possible Solution

Define a narrow classified startup-repair contract: only audited safe migrations run unattended, every mutation is logged with backup/recovery details, and security, sandbox, channel/plugin disabling, package-manager, and broad config cleanup stay manual or explicit opt-in.

## Reproduction Assessment

Yes, at source level. Current startup code validates and hints at doctor repair without running a safe auto-repair classifier or broad `doctor --fix` sequence on ordinary Gateway startup.

## Solution Assessment

No for the broad suggested shape. Running full `doctor --fix` on Gateway start is too broad; the safer solution is a maintainer-approved safe-repair contract with explicit exclusions and logging.
