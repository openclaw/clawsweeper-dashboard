## What This Changes

The PR adds optional `AssembleResult.promptAuthority`, makes assembled context the default overflow-precheck authority in the embedded runner, preserves pre-assembly overflow gating behind an explicit opt-in, and adds regression tests for both paths.

## Best Possible Solution

Land this PR after completing the public-contract follow-through: regenerate the plugin SDK API baseline, document the new context-engine prompt-authority semantics, add the required changelog entry, rebase if needed, then rerun focused runner and plugin SDK validation so the linked bug can close on merge.

Yes. ## Solution Assessment

Yes, directionally. Making assembled messages authoritative by default, with an explicit opt-in for engines that want pre-assembly overflow gating, is the narrow maintainable fix; the proposed patch is incomplete only in generated API, docs, changelog, and final validation follow-through.
