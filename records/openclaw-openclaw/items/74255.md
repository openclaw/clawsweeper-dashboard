## Summary

Keep open. Current main still has the reported prompt-admission bug, and this PR is the right narrow direction, but it is not merge-ready because the submitted diff no longer applies cleanly to current main and the public SDK contract follow-through is still missing.

## What This Changes

The PR adds optional `AssembleResult.promptAuthority`, makes assembled context-engine messages the default overflow-precheck authority, preserves pre-assembly overflow gating behind an explicit opt-in, and adds runner/precheck regressions.

## Best Possible Solution

Land a rebased or replacement version of this narrow host-side fix that keeps assembled messages authoritative by default, documents the `preassembly_may_overflow` opt-in, updates the plugin SDK API baseline and changelog, and preserves explicit raw-history overflow gating only for engines that request it.

## Reproduction Assessment

Yes. Current main gives a high-confidence source-level reproduction: context-engine assembly can install a small prompt-ready `activeSession.messages`, but the later precheck still receives and prefers the larger pre-assembly `unwindowedMessages` estimate.

## Solution Assessment

Yes directionally, but not as submitted. The assembled-by-default plus explicit preassembly opt-in model is the narrowest maintainable fix, but the branch needs a rebase/replacement and public SDK/docs/changelog follow-through before merge.
