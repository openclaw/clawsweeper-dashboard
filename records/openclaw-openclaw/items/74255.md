## Summary

Keep open. The runtime direction is correct and still needed on current main, but the latest PR diff adds a public `AssembleResult` field without the required Plugin SDK API baseline, public docs, or changelog follow-through. The branch addressed earlier type/perf review comments and includes focused regressions for both default assembled authority and explicit pre-assembly authority.

## What This Changes

The PR adds optional `AssembleResult.promptAuthority`, defaults embedded-runner overflow prechecks to assembled messages, preserves pre-assembly gating for opt-in engines, and adds runner/precheck regressions.

## Best Possible Solution

Land this PR after adding the SDK API baseline, documenting the prompt-authority contract, adding the required changelog entry, and waiting for exact-head checks; then the linked host-side bug can close while the LCM cache/provenance follow-ups remain in their own repository.

## Reproduction Assessment

Yes. Current main at `8a399ec5b46ba25b5ab1ef48c5ee426d3811528d` has a source-level reproduction path: a context engine can replace the active messages with a small assembled prompt, but the later precheck still receives the larger pre-assembly snapshot and can choose that larger estimate.

## Solution Assessment

Yes, directionally, but the proposed branch is not merge-ready. Making assembled messages authoritative by default with an explicit `preassembly_may_overflow` opt-in is the narrow maintainable fix; the safer final solution includes API baseline, docs, changelog, and exact-head validation follow-through.
