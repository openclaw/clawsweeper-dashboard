## Summary

Keep open: current main has global bootstrap injection controls and a lightweight run seam, but Telegram/native skill commands still enter the normal model-request path without a skill-specific or per-command bootstrap policy. The related open context-bloat issue is broader and does not cover this fine-grained skill-command first-turn problem.

## What This Changes

Request to make Telegram skill-dispatched slash commands avoid full workspace bootstrap context on each invocation, either automatically or through per-command context policy.

## Best Possible Solution

Add a deliberate skill-dispatch context policy, preferably reusing the existing `bootstrapContextMode` seam for native skill runs after deciding whether the default should be automatic lightweight mode or an explicit per-command override.

## Reproduction Assessment

No high-confidence live reproduction is available here because this checkout does not exercise the reported Telegram plus Anthropic plus Pi setup. Source inspection does show that current main routes skill commands through the normal model path without a skill-specific lightweight bootstrap override.

## Solution Assessment

Unclear. The proposed directions are plausible, but maintainers need to choose between automatic lightweight skill turns and an explicit per-command config because current docs define skill commands as normal model requests by default.
