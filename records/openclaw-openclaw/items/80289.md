## Summary

Keep open: current main still has global and per-agent bootstrap controls, but non-tool Telegram skill commands such as `/weather` still enter the normal model path without a skill-specific lightweight bootstrap policy or per-command override. The source path is clear, but the fix requires a maintainer policy choice about context loss versus latency.

## What This Changes

The issue requests reducing Telegram skill-command latency by applying lightweight or configurable bootstrap context for skill-dispatched turns instead of full workspace injection.

## Best Possible Solution

Add a deliberate skill-command bootstrap policy by threading `bootstrapContextMode` or a documented command context override through native/text skill dispatch, with regression tests for Telegram and inline skill commands.

## Reproduction Assessment

No, not for the full timing claim: this review did not run Telegram, Anthropic Haiku 4.5, and Raspberry Pi hardware. Source inspection does reproduce the core routing condition: `/weather` is a non-tool skill command, Telegram does not set `bootstrapContextMode`, and omitted `contextInjection` still resolves to `always`.

## Solution Assessment

Unclear. The proposed directions are plausible, but maintainers need to choose whether the best product behavior is automatic lightweight context for skill-dispatched turns or an explicit per-command/per-channel override.
