## Summary

Keep open: current main still has global bootstrap controls and per-run lightweight context plumbing, but Telegram native skill dispatch does not request lightweight context and `/weather` remains a model-forwarded skill command, so the fine-grained skill-command performance request is unresolved.

## What This Changes

Request a per-skill or per-command way for Telegram skill invocations such as `/weather` to avoid full workspace bootstrap context injection.

## Best Possible Solution

Add a first-class skill-command context policy that safely routes skill-dispatched turns with lightweight bootstrap or exposes an explicit per-command override, with docs, regression coverage, and redacted before/after latency proof.

## Reproduction Assessment

No high-confidence live reproduction of the latency claim was established because this review did not exercise Telegram, Anthropic Haiku, and Raspberry Pi hardware. Source inspection does reproduce the routing condition: `/weather` is model-forwarded, Telegram does not pass `bootstrapContextMode`, and default context injection remains `always`.

## Solution Assessment

Unclear. The suggested directions are plausible, but maintainers need to choose whether the maintainable fix is automatic lightweight context for skill-dispatched turns or an explicit per-command/channel override.
