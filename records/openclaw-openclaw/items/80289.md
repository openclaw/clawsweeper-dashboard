## Summary

Keep open: current main and v2026.6.10 still route Telegram native skill commands into normal model-backed reply dispatch, and bootstrap injection remains controlled only by global/per-agent modes or explicit run-level lightweight mode, not by skill or command invocation.

## What This Changes

The issue requests a minimal-context or per-command context mode so Telegram skill commands such as `/weather` avoid full workspace bootstrap injection latency.

## Best Possible Solution

Define one canonical skill-command bootstrap policy at the auto-reply/agent boundary, apply it consistently to Telegram native and text skill paths, and document whether it is default behavior or an explicit override.

## Reproduction Assessment

Yes for the source-level policy gap: current main shows Telegram native skill commands flow through normal reply dispatch while agent attempts only honor global/per-agent contextInjection or explicit run-level lightweight mode. No live Telegram/Pi/Haiku benchmark was run, so the exact latency claim was not independently reproduced.

## Solution Assessment

Unclear: default minimal context for model-backed skill commands is plausible and avoids more config, but it changes prompt semantics for existing skills. A per-command override is safer for compatibility but adds config surface, so maintainers should choose the policy first.
