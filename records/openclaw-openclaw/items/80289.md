## Summary

Keep open: current main still has only global bootstrap controls, and non-tool Telegram skill commands such as `/weather` still route through the normal model path without a skill-specific lightweight bootstrap policy or per-command context override.

## What This Changes

Request a lightweight or per-command context policy for Telegram skill slash commands so `/weather` does not pay full workspace bootstrap cost on each skill-dispatched turn.

## Best Possible Solution

Define an explicit skill-command bootstrap policy, likely by reusing `bootstrapContextMode` or adding a documented per-command override, then cover Telegram dispatch, auto-reply skill routing, config/docs, and latency proof.

## Reproduction Assessment

No high-confidence live reproduction of the latency number was established here. Source inspection does reproduce the routing condition: Telegram `/weather` is model-forwarded, Telegram does not set `bootstrapContextMode`, and default context injection remains `always`.

## Solution Assessment

Unclear. The proposed directions are plausible, but maintainers need to choose whether the best product behavior is automatic lightweight context for skill-dispatched turns or an explicit per-command/channel override.
