## Summary

Keep open: current main still routes non-tool Telegram skill commands through the normal model path with full default bootstrap context, and there is no skill-specific lightweight bootstrap policy or per-command context override. The remaining work is a context-policy/product decision, not a conservative cleanup close.

## What This Changes

The issue requests lower-latency Telegram skill command handling by avoiding full workspace bootstrap context for invocations such as `/weather`.

## Best Possible Solution

Add a maintainer-approved skill-command bootstrap policy, likely by threading bootstrapContextMode through native skill dispatch or defining a documented per-command override, with docs and before/after Telegram latency proof.

## Reproduction Assessment

No high-confidence live reproduction of the 25-35 second latency claim was performed. Source inspection does confirm the reported routing condition: `/weather` is model-forwarded, Telegram does not set a lightweight bootstrap mode, and default context injection remains `always`.

## Solution Assessment

Unclear. Automatic lightweight skill turns are plausible, but maintainers need to choose that policy against a safer explicit per-command or per-agent override because skills may depend on injected persona/context files.
