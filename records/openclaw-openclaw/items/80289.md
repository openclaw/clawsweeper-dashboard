## Summary

Keep open: current main still routes Telegram/native skill commands through the normal model reply path with only global bootstrap-injection controls, so the fine-grained skill-command latency problem is not implemented or superseded.

## What This Changes

Request lower-latency Telegram skill slash-command turns by applying lightweight or disabled bootstrap context per skill/command, or by documenting the global workaround.

## Best Possible Solution

Add a deliberate skill-dispatch bootstrap policy using the existing `bootstrapContextMode` seam, with maintainers choosing automatic lightweight skill turns versus an explicit per-command override, then cover Telegram and text skill commands in docs and tests.

## Reproduction Assessment

No high-confidence live reproduction was established here because the review did not exercise Telegram, Anthropic Haiku, and Raspberry Pi hardware. Source inspection does show current main routes skill commands without a skill-specific lightweight bootstrap override.

## Solution Assessment

Unclear. The proposed directions are plausible, but maintainers need to choose whether skill-dispatched turns should automatically use lightweight context or whether this should be an explicit command-level policy.
