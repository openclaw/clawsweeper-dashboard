## Summary

Keep open: current main and the latest release still support only global/per-agent bootstrap injection controls, while non-tool Telegram skill commands such as `/weather` are documented and implemented as normal model-forwarded requests with no skill-specific lightweight bootstrap policy.

## What This Changes

Request a lower-latency context-injection mode for Telegram skill slash commands, either automatically for skill-dispatched turns or through per-command configuration.

## Best Possible Solution

Add an explicit skill-command bootstrap policy after maintainer direction, preferably by threading existing `bootstrapContextMode` for skill-dispatched turns or by adding a documented scoped command override.

## Reproduction Assessment

No high-confidence live reproduction of the latency claim was performed because this review did not exercise Telegram, Anthropic Haiku, and Raspberry Pi hardware. Source inspection does reproduce the relevant condition: `/weather` is a normal model-forwarded skill command, Telegram does not set `bootstrapContextMode`, and default context injection remains full unless configured otherwise.

## Solution Assessment

Unclear. The requested directions are plausible, but maintainers need to choose whether the durable policy is automatic lightweight context for skill-dispatched turns or an explicit per-command/channel config override.
