## Summary

Keep open: current main still has only global bootstrap-injection controls, and Telegram/native skill commands still dispatch through the normal agent turn without a skill-specific lightweight bootstrap policy.

## What This Changes

Request minimal or per-command bootstrap-context handling for Telegram skill slash commands so simple skills like `/weather` avoid full workspace prompt injection.

## Best Possible Solution

Add a deliberate skill-command context policy using the existing `bootstrapContextMode` seam after maintainers choose automatic lightweight skill turns versus an explicit per-command override, then cover Telegram/native and text skill paths with docs and regression tests.

## Reproduction Assessment

No high-confidence live reproduction of the reported latency is available here because this review did not run Telegram, Anthropic, and Raspberry Pi hardware. Source inspection does reproduce the current behavior: skill commands use normal model turns unless they opt into direct tool dispatch, and Telegram native skill dispatch does not pass a lightweight bootstrap option.

## Solution Assessment

Unclear. The suggested directions are plausible, but maintainers need to decide whether the narrowest maintainable fix is an automatic skill-dispatch policy or an explicit per-command config surface because the current docs define normal model dispatch as the default.
