## Summary

Keep open: current main has global and per-agent bootstrap controls plus a run-level lightweight bootstrap seam, but Telegram native skill commands still dispatch `/weather` through the normal model path without a skill- or command-scoped bootstrap policy. This needs a maintainer product decision before implementation because lightweight skill turns could drop context that some skills intentionally use.

## What This Changes

The issue requests a skill- or command-scoped lightweight bootstrap policy for Telegram slash-command skill invocations such as `/weather` to avoid full workspace context injection on each run.

## Best Possible Solution

Have maintainers choose an explicit skill-dispatch bootstrap policy, then implement it by reusing the existing run-level `bootstrapContextMode` seam or by adding a documented command override with focused Telegram, auto-reply, bootstrap, and latency proof.

## Reproduction Assessment

No high-confidence live reproduction of the 25-35 second latency claim was run here. Source inspection does confirm the relevant current-main path: `/weather` is a model-forwarded skill command, Telegram does not set `bootstrapContextMode`, and default context injection remains full unless configured at agent/global scope.

## Solution Assessment

Unclear. The proposed directions are plausible, but maintainers need to choose whether automatic lightweight bootstrap for skill-dispatched turns or an explicit command/channel override is the maintainable behavior.
