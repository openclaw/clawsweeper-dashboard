## Summary

Keep open: current main has global bootstrap controls and a lightweight run seam, but Telegram/native skill commands still enter the normal reply path without a skill-specific or per-command bootstrap policy. The related context-bloat issues are broader and do not replace this fine-grained skill-command latency request.

## What This Changes

Requests a skill-command-specific bootstrap context policy so Telegram `/weather`-style skill invocations can avoid full workspace context without disabling context globally.

## Best Possible Solution

Add a deliberate skill-dispatch context policy, likely by threading the existing bootstrapContextMode seam from native and text skill dispatch after maintainers choose automatic lightweight behavior versus an explicit per-command config surface.

## Reproduction Assessment

No high-confidence live reproduction of the 25-35 second timing claim was run against Telegram, Anthropic, and Raspberry Pi hardware. Source inspection does verify that current-main Telegram skill commands lack a skill-specific lightweight bootstrap override and therefore follow the normal model request path.

## Solution Assessment

Unclear because no patch is proposed and the right product shape is still open. Reusing bootstrapContextMode is a plausible narrow implementation seam, but maintainers need to decide whether skill turns should be automatically lightweight or controlled by explicit per-command config.
