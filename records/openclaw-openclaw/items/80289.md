## Summary

Keep open: current main still has only global/per-agent bootstrap controls, and Telegram native skill commands such as `/weather` still enter the normal model-forwarded skill path without a skill-specific lightweight bootstrap policy.

## What This Changes

Request a skill-command-specific lightweight bootstrap/context-injection policy for Telegram slash skill invocations such as `/weather` to avoid paying full workspace bootstrap latency.

## Best Possible Solution

Have maintainers choose the skill-command bootstrap policy, then wire that policy through Telegram native skill dispatch into the existing `bootstrapContextMode` path with focused tests and docs.

## Reproduction Assessment

No high-confidence live reproduction of the latency claim was established here because this review did not exercise Telegram, Anthropic Haiku, and Raspberry Pi hardware. Source inspection does reproduce the routing condition: `/weather` is a normal model-forwarded skill command, Telegram does not pass `bootstrapContextMode`, and default context injection remains `always`.

## Solution Assessment

Unclear. Reusing lightweight bootstrap for skill-dispatched turns is plausible, but maintainers need to decide whether that should be automatic for all skill commands or exposed as an explicit per-command/channel policy.
