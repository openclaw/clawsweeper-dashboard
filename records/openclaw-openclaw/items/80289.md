## Summary

Keep open: current main still exposes only global context-injection controls; Telegram `/weather` remains a model-forwarded skill command with no lightweight bootstrap mode or per-command override, so the requested performance policy is unresolved.

## What This Changes

Request a skill-command-specific lightweight context path or per-command contextInjection override so Telegram slash-command skills like `/weather` avoid full workspace bootstrap cost.

## Best Possible Solution

Add an explicit skill-dispatch context policy after maintainer choice, likely by reusing `bootstrapContextMode` for skill-command turns or by adding a scoped per-command override with docs, tests, and live Telegram latency proof.

## Reproduction Assessment

No high-confidence live reproduction of the latency claim was established here because this review did not exercise Telegram, Anthropic Haiku, and Raspberry Pi hardware. Source inspection does reproduce the routing condition: `/weather` is a normal model-forwarded skill command, Telegram does not set `bootstrapContextMode`, and default context injection remains `always`.

## Solution Assessment

Unclear. The proposed directions are plausible, but maintainers need to choose whether the maintainable default is automatic lightweight context for skill-dispatched turns or an explicit per-command/channel config override.
