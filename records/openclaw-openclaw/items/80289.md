## Summary

Keep open: current main still lacks a skill-specific lightweight bootstrap policy or per-command context override for Telegram skill slash commands. The source path remains consistent with the report, although the exact Raspberry Pi/Haiku latency was not live-reproduced in this read-only review.

## What This Changes

Requests lightweight or per-command context injection for Telegram/native skill slash commands so invocations like `/weather` avoid full workspace bootstrap on each turn.

## Best Possible Solution

Define and test an explicit skill-command bootstrap policy, either automatic lightweight context for model-forwarded skill invocations or an opt-in per-command/channel override, with docs for the latency tradeoff.

## Reproduction Assessment

No high-confidence live reproduction of the issue was run here. Source inspection does reproduce the routing condition: Telegram skill slash commands pass no bootstrapContextMode, `/weather` is not tool-dispatched, and non-tool skills continue into the normal model path.

## Solution Assessment

Unclear. The proposed directions are plausible, but maintainers need to choose whether the maintainable default is automatic lightweight skill turns or an explicit per-command/channel config override.
