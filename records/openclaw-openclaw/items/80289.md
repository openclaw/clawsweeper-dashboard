## Summary

Keep open: current main still routes non-tool Telegram skill commands through the normal model reply path without a skill-specific lightweight bootstrap policy or command context override, so this remains a valid product/performance follow-up rather than a cleanup close.

## What This Changes

Request lightweight or per-command context injection for Telegram skill slash commands such as `/weather` to avoid full bootstrap prompt cost.

## Best Possible Solution

Add a maintainer-approved skill-dispatch context policy and wire it through Telegram native/text skill dispatch to `GetReplyOptions`, with docs and focused regression coverage.

## Reproduction Assessment

No high-confidence live reproduction of the full latency claim was established because this review did not run Telegram, Anthropic Haiku, and Raspberry Pi hardware. Source inspection does reproduce the routing condition: `/weather` is model-forwarded, Telegram does not pass lightweight bootstrap context, and the default context injection mode remains `always`.

## Solution Assessment

Unclear. Automatic lightweight context for skill-dispatched turns and an explicit per-command override are both plausible, but maintainers need to choose the product contract before implementation.
