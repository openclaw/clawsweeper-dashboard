## Summary

Keep open: current main still has only global bootstrap-injection controls, while Telegram native skill commands dispatch through the normal model-request path without a skill-specific or per-command bootstrap policy. The related context-bloat issue remains broader and the open draft PR only covers continuation-skip detection, not this first-turn skill-command policy gap.

## What This Changes

The issue requests a lower-bootstrap context path for Telegram slash-command skill invocations such as `/weather`.

## Best Possible Solution

Add an intentional skill-command context policy that reuses the existing bootstrapContextMode plumbing only after maintainers decide the default behavior and any per-command override shape.

## Reproduction Assessment

No high-confidence live reproduction was run for the Telegram plus Anthropic plus Pi setup. Source inspection does show that current main routes Telegram native skill commands through the normal model path without a skill-specific lightweight bootstrap override.

## Solution Assessment

Unclear. Reusing bootstrapContextMode is the likely implementation seam, but maintainers still need to choose automatic lightweight skill turns versus explicit per-command configuration and define what context remains available.
