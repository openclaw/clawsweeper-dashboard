## Summary

Keep open: current main still has the reported source-level condition. Telegram native skill commands pass through as normal model-backed skill invocations without a skill-specific lightweight bootstrap policy or per-command context-injection override.

## What This Changes

Request lightweight or configurable bootstrap context for Telegram/native skill command turns such as `/weather` to avoid paying full workspace prompt cost.

## Best Possible Solution

Define an explicit skill-command bootstrap policy, then thread it through Telegram native skill dispatch and the agent runner with focused config/docs/tests if maintainers choose an opt-in override.

## Reproduction Assessment

No high-confidence live reproduction of the 25-35 second latency claim is available here. Source inspection does confirm the current-main routing condition: `/weather` is a model-forwarded non-tool skill command and Telegram does not set a lightweight bootstrap mode.

## Solution Assessment

Unclear until maintainers choose the product boundary. Automatic lightweight context for all skill turns is plausible, but an explicit per-command/channel override may be safer for skills that depend on workspace identity files.
