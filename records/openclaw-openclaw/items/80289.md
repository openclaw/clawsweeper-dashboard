## Summary

Keep open: current main and v2026.6.6 still have only global/per-agent contextInjection controls, while Telegram skill commands like /weather remain model-backed unless the skill declares direct tool dispatch. The remaining work is a maintainer context-policy decision with a live latency proof gap, not stale cleanup.

## What This Changes

The issue requests lower-latency context handling for Telegram skill slash commands, either by using minimal bootstrap context for skill-driven turns or by adding a per-skill/per-command context override.

## Best Possible Solution

Maintainers should choose one canonical skill-command bootstrap policy at the core command/agent boundary, cover both text and Telegram native skill paths, and document whether minimal context is the default or an explicit override.

## Reproduction Assessment

Yes for the source-level policy gap: current main still rewrites non-tool skill slash commands into normal agent turns and only has global/per-agent contextInjection. No live Telegram/Pi/Haiku run was performed, so the exact wall-clock latency is not independently reproduced here.

## Solution Assessment

Unclear: default minimal context for model-backed skill commands may be the cleanest product behavior, but it changes prompt semantics for existing skills. A maintainer-selected policy should come before implementation.
