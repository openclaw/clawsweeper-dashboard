## Summary

Keep open: current main still has global bootstrap controls and Telegram native skill dispatch, but `/weather` remains a model-forwarded skill command with no skill-specific lightweight bootstrap policy or per-command context override.

## What This Changes

The issue asks for Telegram skill invocations such as `/weather` to avoid full default workspace bootstrap injection by using lightweight skill context or a per-command override.

## Best Possible Solution

Maintainers should choose the policy boundary first, then implement either automatic lightweight context for skill-dispatched model turns or a documented per-command override with focused Telegram, skill-command, bootstrap, docs, and latency/trace proof.

## Reproduction Assessment

No high-confidence live reproduction of the 30s latency claim was established here. Source inspection does reproduce the routing condition: `/weather` is a non-tool skill command, Telegram does not set bootstrapContextMode, and default context injection remains the normal global policy.

## Solution Assessment

Unclear. The suggested directions are plausible, but the best maintainable fix depends on a maintainer choice between automatic lightweight skill-dispatch context and an explicit per-command/channel config override.
