## Summary

Keep open: current main still forwards non-tool Telegram skill slash commands through the normal model path, does not pass a lightweight bootstrap mode from Telegram native skill dispatch, and has no per-skill or per-command context-injection policy. The report is plausible and well scoped, but the fix changes product policy for skill-command context, so it needs maintainer direction before automation.

## What This Changes

The issue requests a faster context policy for Telegram skill slash commands so simple commands like `/weather` do not pay full workspace bootstrap cost on every model turn.

## Best Possible Solution

Add an explicit skill-dispatch context policy after maintainers choose automatic lightweight skill turns versus an opt-in command/channel override, then update Telegram routing, config/docs, and focused regression coverage together.

## Reproduction Assessment

No high-confidence live reproduction of the wall-clock latency was performed. Source inspection does reproduce the current-main routing condition: /weather is a non-tool skill command, Telegram does not pass bootstrapContextMode, and omitted contextInjection still defaults to always.

## Solution Assessment

Unclear until maintainers choose the product boundary. Reusing bootstrapContextMode for skill-dispatched turns is plausible, but automatic lightweight context and an explicit per-command override have different compatibility tradeoffs.
