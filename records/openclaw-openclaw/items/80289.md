## Summary

Keep open: current main still has only global/per-agent bootstrap context controls plus the direct tool-dispatch escape hatch, while model-backed skill slash commands such as `/weather` are still rewritten into a normal agent request. This needs a maintainer context-policy decision rather than stale cleanup.

## What This Changes

The issue requests a low-latency context policy for Telegram skill slash commands such as `/weather` without disabling workspace bootstrap context globally.

## Best Possible Solution

Pick one canonical skill-command bootstrap policy, implement it once across text and native command paths, and document whether users get default minimal context or an explicit per-skill/per-command override.

## Reproduction Assessment

Yes from source for the policy gap: normal skill slash commands are rewritten into an agent request, while contextInjection is only agent/global and first turns still resolve bootstrap context. I did not reproduce the reported wall-clock latency on Telegram, Pi 5, and Haiku 4.5.

## Solution Assessment

Unclear: minimal context for every model-backed skill command may be the right end state, but it changes context semantics and needs maintainer product judgment before implementation.
