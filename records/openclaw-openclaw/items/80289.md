## Summary

Keep open: current main still has only global bootstrap injection controls, and Telegram/native skill commands dispatch through the normal reply path without a skill-specific `bootstrapContextMode` override. The closed https://github.com/openclaw/openclaw/issues/9157 and open https://github.com/openclaw/openclaw/issues/67419 cover broader/global context injection behavior, not this fine-grained first-turn skill-command policy.

## What This Changes

Request a Telegram/native skill-command context policy so `/weather`-style skill invocations can avoid full workspace bootstrap injection without disabling context for all sessions.

## Best Possible Solution

Add a deliberate skill-dispatch bootstrap policy through the existing `bootstrapContextMode` seam after choosing automatic lightweight skill turns versus an explicit per-command override.

## Reproduction Assessment

No high-confidence live reproduction was established for the reported Telegram, Anthropic, and Raspberry Pi latency. Source inspection does show the missing skill-specific context path: Telegram/native skill commands do not set `bootstrapContextMode`, while default bootstrap injection remains global.

## Solution Assessment

Unclear. Reusing `bootstrapContextMode` is the narrow technical seam, but maintainers need to decide whether skill commands should default to lightweight context or expose an explicit per-command override.
