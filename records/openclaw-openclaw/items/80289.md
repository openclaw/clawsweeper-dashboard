## Summary

Keep open: current main still has only global bootstrap context controls, and Telegram native skill commands such as `/weather` still route through the normal model-backed skill path without a skill-specific lightweight bootstrap mode or per-command override.

## What This Changes

Request lightweight context injection for Telegram skill-dispatched slash commands, or an explicit per-command context override, so trivial skills avoid full workspace bootstrap latency.

## Best Possible Solution

Add an explicit skill-dispatch bootstrap policy, reusing `bootstrapContextMode` where possible after maintainers choose automatic lightweight skill turns versus an opt-in per-command/channel config surface.

## Reproduction Assessment

No high-confidence live reproduction of the latency number is available here because this review did not exercise Telegram, Anthropic Haiku, and Raspberry Pi hardware. Source inspection does reproduce the relevant current-main path: `/weather` is not tool-dispatched, Telegram does not set `bootstrapContextMode`, and default context injection remains `always`.

## Solution Assessment

Unclear. The proposed directions are plausible, but maintainers need to choose whether the narrowest maintainable behavior is automatic lightweight context for skill-dispatched turns or an explicit per-command/channel override.
