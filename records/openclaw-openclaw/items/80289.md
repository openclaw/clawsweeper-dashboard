## Summary

Keep open: current main and v2026.6.8 still route non-tool Telegram skill commands through normal model-backed agent runs with only global/per-agent contextInjection, so the requested skill-command minimal-context policy remains unresolved.

## What This Changes

Request a fine-grained context-injection policy for Telegram/native skill commands so /weather-style turns avoid full bootstrap prompt cost without disabling context globally.

## Best Possible Solution

Define one canonical skill-command bootstrap policy at the auto-reply/agent boundary, apply it consistently to Telegram native and text skill paths, and document whether it is default behavior or an explicit override.

## Reproduction Assessment

Yes for the source-level policy gap: current main rewrites non-tool skill slash commands into normal agent turns and only supports global/per-agent contextInjection. No live Telegram/Pi/Haiku benchmark was run, so the exact wall-clock latency remains reporter-provided.

## Solution Assessment

Unclear: default minimal context for model-backed skill commands is plausible, but it changes prompt semantics for existing skills; a per-command override avoids default breakage but adds config surface. Maintainers should choose the policy before implementation.
