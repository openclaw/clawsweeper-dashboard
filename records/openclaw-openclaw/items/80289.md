## Summary

Keep open: current main still only exposes global bootstrap controls, and Telegram native skill commands such as `/weather` still enter the normal model-forwarded skill path without a skill-specific lightweight bootstrap mode. This remains a valid feature/performance policy request rather than something implemented or superseded.

## What This Changes

Request a Telegram skill-command context policy so `/weather`-style skill invocations can avoid full workspace bootstrap injection without disabling context globally.

## Best Possible Solution

Add an explicit skill-dispatch context policy after maintainers choose automatic lightweight skill turns versus an opt-in per-command override, then update config/schema/docs/tests and collect before/after Telegram latency or trace proof.

## Reproduction Assessment

No high-confidence live reproduction of the latency number was established here. Source inspection does reproduce the routing condition: `/weather` is a non-tool skill command, Telegram does not set `bootstrapContextMode`, and omitted `contextInjection` still resolves to `always`.

## Solution Assessment

Unclear. The requested directions are plausible, but maintainers need to choose whether the narrowest maintainable fix is automatic lightweight context for skill-dispatched turns or an explicit per-command/channel config surface.
