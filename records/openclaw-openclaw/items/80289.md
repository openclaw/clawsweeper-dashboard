## Summary

Keep open: current main still has only global/per-agent bootstrap controls, and Telegram native skill commands such as `/weather` still enter the model-forwarded skill path without a lightweight bootstrap override. The related open context issues are adjacent, but this issue carries a distinct Telegram skill-command latency case and needs maintainer policy direction rather than cleanup closure.

## What This Changes

Request skill-dispatched Telegram slash commands, especially `/weather`, to avoid full workspace bootstrap cost through lightweight context or a per-command override.

## Best Possible Solution

Define an owner-approved skill-command bootstrap policy that reuses `bootstrapContextMode` where possible, preserves normal chat context, and updates config/docs/tests if a per-command override is chosen.

## Reproduction Assessment

Yes for the source-level routing condition: `/weather` is a model-forwarded skill command, Telegram does not set `bootstrapContextMode`, and default context injection remains `always`. No live Pi/Haiku latency reproduction was performed in this read-only review.

## Solution Assessment

Unclear pending product direction. The proposed directions are plausible, but maintainers should first choose automatic lightweight skill turns versus an explicit per-command/channel context override.
