## Summary

Keep open: current main still routes non-tool skill slash commands into the normal model path, and there is no per-skill or per-command bootstrap-context policy to avoid full workspace injection for commands such as `/weather`. The existing global `contextInjection` settings and direct `command-dispatch: tool` fast path help adjacent cases, but they do not cover this requested skill-turn policy.

## What This Changes

Requests a lightweight or configurable bootstrap-context mode for Telegram skill slash-command turns so commands such as `/weather` avoid full workspace injection.

## Best Possible Solution

Choose and document a context policy for skill-dispatched turns, then wire it through the shared skill-command/Telegram native dispatch path so non-tool skills can avoid full bootstrap when that is intended.

## Reproduction Assessment

Yes for the source path, but not for the exact latency measurement: current code shows `/weather` is a non-tool skill command that rewrites into a normal agent run, and the Telegram native path does not set `bootstrapContextMode`. The Pi 5/Haiku 30s round trip was not live-reproduced in this read-only review.

## Solution Assessment

Unclear until maintainers choose the policy: direct tool dispatch already solves opt-in deterministic skills, while global `contextInjection` is too broad for the reporter's use case. The maintainable fix is either a default lightweight mode for skill turns or an explicit per-command/per-skill override with docs and tests.
