## Summary

Keep open: current main has global bootstrap controls and a native slash-command fast path, but non-tool Telegram skill commands such as `/weather` still fall through to the normal model run without a skill-specific lightweight bootstrap policy or per-command config surface.

## What This Changes

The issue requests a faster context/bootstrap policy for Telegram skill-command turns so commands like `/weather` do not pay full workspace injection cost.

## Best Possible Solution

Add an explicit skill-dispatch context policy, preferably by reusing `bootstrapContextMode` after maintainers choose automatic lightweight skill turns versus an opt-in per-command override.

## Reproduction Assessment

No high-confidence live reproduction of the latency claim is available here because this review did not exercise Telegram, Anthropic Haiku, and Raspberry Pi hardware. Source inspection does reproduce the routing condition: `/weather` is a normal model-forwarded skill command, Telegram does not set `bootstrapContextMode`, and default context injection remains `always`.

## Solution Assessment

Unclear. The proposed directions are plausible, but maintainers need to choose whether the maintainable default is automatic lightweight context for skill-dispatched turns or an explicit per-command/channel config override.
