## Summary

Keep open. Current main partially fixes the misleading soft wait-window timeout by returning `accepted` for non-terminal waits, but it still lacks caller-visible inter-session progress streaming, `sessions_subscribe`/`streamProgress`, or an agent-callable buffered run-event API.

## What This Changes

Add caller-visible progress streaming or subscription support for long-running `sessions_send` runs, especially ACP/Codex-backed sessions.

## Best Possible Solution

Add an additive, permission-checked run-event observation surface with bounded buffering, then expose it through `sessions_subscribe` or an explicit `sessions_send` progress option while preserving accepted/final-announce behavior.

## Reproduction Assessment

Yes, at source level rather than with a live ACP run. Current main has no `sessions_subscribe`, `streamProgress`, `agent.events`, or buffered caller-visible run-event path, while `sessions_send` and A2A announce still wait for completion/final delivery.

## Solution Assessment

Unclear pending maintainer design. The requested direction is valid, but the final surface needs authorization, redaction, buffering, retention, and channel delivery policy before implementation.
