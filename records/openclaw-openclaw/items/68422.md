The PR modifies `ui/src/styles/chat/layout.css` to add fixed 920px max-width centering for `.chat-thread-inner` and `.agent-chat__input`.

## Best Possible Solution

Constrain the welcome/thread/composer readability surface while preserving `gateway.controlUi.chatMessageMaxWidth`, then attach redacted wide-viewport browser proof.

## Reproduction Assessment

Yes, from source inspection: current main renders `.chat-thread-inner` and `.agent-chat__input`, while `layout.css` has no width/max-width centering for either selector. I did not run a live browser session.

## Solution Assessment

No as written. The PR is in the right CSS owner area, but a fixed 920px `.chat-thread-inner` ancestor would prevent documented wider `gateway.controlUi.chatMessageMaxWidth` values from taking effect.
