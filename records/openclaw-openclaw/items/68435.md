## Summary

Keep open. Current main still lacks chat-completions audio/file handling, so the feature remains useful, but the PR head still has blocking audio validation and prompt-placement bugs and lacks inspectable real behavior proof.

## What This Changes

The branch adds gateway/config/schema/test support for `/v1/chat/completions` `input_audio` and `file` blocks, reuses existing audio transcription and file extraction, and rejects `video_url`.

## Best Possible Solution

Revise the PR so chat-completions audio fails closed when unusable, transcripts are folded into the active user turn before prompt formatting, and redacted real gateway proof is provided.

## Reproduction Assessment

Yes. Source inspection of PR head 44f27d9b4416dda4db935b1232ba5b40625544d3 shows blank `input_audio.data` is dropped before validation and `transcribeFirstAudio` returning undefined lets the request continue without transcript text.

## Solution Assessment

No. Reusing existing STT and file-extraction pipelines is the right general direction, but the PR should reject malformed or untranscribable audio and merge successful transcripts into the active user turn before prompt assembly.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Reject audio when transcription produces no text:** `src/gateway/openai-http.ts:647-649`
  - body: `transcribeFirstAudio` returns `undefined` for disabled, unconfigured, failed, or empty transcription, but this branch only skips adding text and lets the request continue. For an audio-only turn, the gateway can return 200 while the user's speech is absent from the prompt; return an `invalid_request_error` when supplied audio cannot produce transcript text.
  - confidence: 0.91
- **[P2] Attach audio transcript to the active turn:** `src/gateway/openai-http.ts:1071`
  - body: `buildAgentPrompt` has already formatted the current-message boundary before this prefix is added. In multi-turn audio-only requests, the active turn remains a generic placeholder while the spoken text sits outside the current user turn; merge the transcript before prompt assembly or make it the active turn content.
  - confidence: 0.86
- **[P2] Reject empty input_audio blocks instead of dropping them:** `src/gateway/openai-http.ts:359-360`
  - body: When an `input_audio` part is present but `input_audio.data` is missing or blank, this path returns `undefined` and silently skips it. If that malformed block is the newest user turn after prior history, the request can ignore the active turn and answer older context instead of returning a 400 validation error.
  - confidence: 0.88
