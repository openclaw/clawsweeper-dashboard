## What This Changes

The PR adds gateway, config, schema, and test support for `input_audio`, `file`, and explicit `video_url` handling in `/v1/chat/completions`.

## Best Possible Solution

Rebase on current main, preserve recent OpenAI chat-completions behavior, fix the audio validation and prompt-assembly bugs, and attach redacted live gateway proof before merge.

## Reproduction Assessment

Yes. Source inspection of PR head 44f27d9b4416dda4db935b1232ba5b40625544d3 shows blank `input_audio.data` is dropped, empty transcription results are not rejected, and transcripts are added only after prompt assembly.

## Solution Assessment

No. Reusing the existing STT and file-extraction pipelines is a reasonable direction, but this branch should fail closed for unusable audio and merge transcript text into the active user turn before prompt formatting.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Reject audio when transcription produces no text:** `src/gateway/openai-http.ts:647-649`
  - body: `transcribeFirstAudio` returns `undefined` for disabled, unconfigured, failed, or empty transcription, but this path only skips adding text and lets the request continue. For an audio-only turn, the gateway can return 200 while the user's speech is absent from the prompt; return an `invalid_request_error` when supplied audio cannot produce transcript text.
  - confidence: 0.91
- **[P2] Attach audio transcripts to the active turn:** `src/gateway/openai-http.ts:1071`
  - body: `buildAgentPrompt` has already formatted the current-message boundary before this prefix is added. In multi-turn audio-only requests, the active turn remains a generic placeholder while the spoken text sits outside the current user turn; merge the transcript before prompt assembly or make it the active turn content.
  - confidence: 0.86
- **[P2] Reject empty input_audio blocks:** `src/gateway/openai-http.ts:359-360`
  - body: When an `input_audio` part is present but `input_audio.data` is missing or blank, this path returns `undefined` and silently skips it. If that malformed block is the newest user turn after prior history, the request can ignore the active turn and answer older context instead of returning a 400 validation error.
  - confidence: 0.88
