## Summary

This contributor PR is still useful because current main does not support audio/file content blocks on `/v1/chat/completions`, but the current head has blocking audio-handling correctness issues and needs revision before merge.

## What This Changes

The PR adds `/v1/chat/completions` handling for `input_audio`, `file`, and `video_url` content blocks, new chatCompletions audio/files config schema entries, and focused multimodal gateway/config tests.

## Best Possible Solution

Keep the multimodal gateway direction, but revise it so audio inputs fail closed when malformed or untranscribable and successful transcripts are part of the active user turn before prompt formatting.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: make transcribeFirstAudio return undefined for a request containing input_audio, or send a latest user turn with blank input_audio.data after prior history; the PR code can proceed without the supplied speech or fall back to stale context.

## Solution Assessment

No. Reusing the existing STT and file-extraction pipelines is a maintainable direction, but the patch should reject unusable audio and merge transcript text into the active user message before buildAgentPrompt selects the current turn.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Reject audio when transcription produces no text:** `src/gateway/openai-http.ts:647-649`
  - body: `transcribeFirstAudio` returns `undefined` for disabled/unconfigured providers, transcription errors, and missing transcripts, but this branch only skips adding transcript text and lets the request continue. For an audio-only turn, the gateway can return 200 while the user's supplied speech is absent from the prompt; return an `invalid_request_error` when supplied audio cannot produce transcript text.
  - confidence: 0.91
- **[P2] Merge transcripts into the active user turn:** `src/gateway/openai-http.ts:1071`
  - body: `buildAgentPrompt` has already selected and formatted the current user message before this prefix is added. In a multi-turn request, the speech transcript sits outside the current-message block while the active turn remains a media placeholder, so the model can underweight or miss the spoken request; add the transcript before prompt assembly or replace the active turn content with it.
  - confidence: 0.86
- **[P2] Reject empty input_audio blocks:** `src/gateway/openai-http.ts:357-360`
  - body: When an `input_audio` part is present but `data` is missing or blank, the parser returns `undefined` and silently drops the block. If that malformed block is the newest user turn after prior history, the request can ignore the active turn and answer older context instead of returning a 400 validation error.
  - confidence: 0.88
