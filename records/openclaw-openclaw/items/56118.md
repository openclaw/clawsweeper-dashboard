## Summary

Keep this open: current main still leaves the reporter's direct `openclaw nodes invoke --command browser.proxy` path with shorter outer timeouts than the Chrome MCP existing-session attach/consent flow, while the newer 45s timeout only covers higher-level browser tool calls.

## What This Changes

The issue requests making Chrome MCP existing-session `user` profile access work through the node browser proxy instead of failing before Chrome shows the consent dialog.

## Best Possible Solution

Align the bundled browser plugin/node-proxy timeout contract so direct `browser.proxy` manage routes for existing-session profiles get a consent-aware budget and propagate timeout/abort options into Chrome MCP reachability and tab listing.

## Reproduction Assessment

Yes. The issue gives concrete remote gateway/node `openclaw nodes invoke --command browser.proxy` steps, and current source shows that direct path can still expire at 15s/20s around a 30s Chrome MCP attach flow.

## Solution Assessment

Yes. The narrowest maintainable solution is to adjust browser plugin/node-proxy timeout policy and Chrome MCP list/reachability plumbing; reusing a local gateway MCP session would be a larger behavior change and is not needed for this failure mode.

## Repair Work Prompt

Fix openclaw/openclaw#56118. The observable bug is that direct `openclaw nodes invoke --command browser.proxy --params '{"path":"/tabs","query":{"profile":"user"},"method":"GET"}'` can fail before Chrome MCP `--autoConnect` has enough time to show and complete the consent dialog. Keep the fix inside the bundled browser plugin/node-proxy and generic node invoke timeout plumbing only as needed. Align direct `browser.proxy` manage routes for Chrome MCP existing-session profiles with the consent-aware timeout budget already used by the browser tool, propagate timeout/abort options into Chrome MCP reachability/list-tabs paths, and keep diagnostics actionable. Do not bypass or weaken Chrome consent, profile allowlists, persistent profile mutation guards, SSRF checks, or external dependency pinning. Add focused regression coverage for direct `browser.proxy` `/tabs` or manage calls, not only the higher-level browser tool path. Add a user-facing `CHANGELOG.md` fix entry unless the final diff is tests-only/internal.
