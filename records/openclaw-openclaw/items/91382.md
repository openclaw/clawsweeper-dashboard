Adds optional compactable transcript count fields to `SessionStatus`, computes them from session transcript messages with the compaction real-conversation predicate, and appends the real-message count to the compact token display.

## Best Possible Solution

Keep the narrow status diagnostic direction, but carry transcript resolution context into every rendered recent-session row and require redacted after-fix `/status` output before merge.

No, not yet; reusing the compaction predicate is the right layer, but the implementation should preserve store-path context for the rendered recent rows and include actual `/status` proof.
