Overall confidence: 0.86

Full review comments:

- **[P1] Declare the canonical model provider for this backend:** `extensions/anthropic/cli-backend-interactive.ts:91-92`
  - body: `listCliRuntimeModelBackendBindings()` only creates runtime aliases for CLI backends that declare `modelProvider`, and the existing `claude-cli` backend sets `modelProvider: "anthropic"`. This new backend omits it, so `anthropic` model-runtime policy and auth-profile routing can fail to select `claude-cli-interactive` even though direct refs may work.
  - confidence: 0.9
- **[P2] Document and contract-test backend inheritance:** `src/plugins/cli-backend.types.ts:201-204`
  - body: This adds `inheritUserConfigFrom` to `CliBackendPlugin`, which is exported through `openclaw/plugin-sdk/cli-backend`, but the public CLI backend docs and SDK contract coverage do not define the new inherited-config semantics. Plugin authors need a documented contract and a focused compatibility test before this becomes a public API surface.
  - confidence: 0.86
- **[P1] Constrain or classify CONNECT proxy egress:** `extensions/anthropic/interactive-proxy/mitm-server.ts:284-288`
  - body: For any CONNECT host other than `api.anthropic.com`, the loopback proxy forwards to the requested destination. Because the proxy is unauthenticated while alive and the port is logged, this needs a narrow allowlist, explicit unsupported classification, or documented maintainer-approved egress policy before merging a new raw socket forwarding surface.
  - confidence: 0.84
