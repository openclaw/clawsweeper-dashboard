## Summary

Keep open: this is an active draft feature PR, but the patch still has blocking wrapper state-machine bugs, insufficient real behavior proof for the visible reasoning path, and a security-sensitive TLS MITM trust boundary that needs maintainer/security approval.

## What This Changes

Adds an opt-in Anthropic `claude-cli-interactive` backend that runs Claude through a Bun loopback TLS proxy to reconstruct Claude stream JSONL, wires thinking deltas into CLI/auto-reply reasoning, adds backend inheritance/static assets/tests, and also changes doctor message-tool warnings.

## Best Possible Solution

Keep the backend experimental and opt-in only after the wrapper preserves Claude tool-loop continuations, finalizes terminal stop reasons, fails closed on proxy key hardening, and has real Telegram/JSONL proof plus security review.

## Reproduction Assessment

Yes for the review blockers: source inspection plus the Anthropic stream/stop-reason contract shows the wrapper drops post-tool streams and skips finalization for terminal non-`end_turn` stops. I did not run a live Claude subscription or Telegram turn.

## Solution Assessment

No in its current form: an opt-in backend may be a maintainable direction, but the wrapper state machine and MITM key handling need fixes before the design can be judged on product grounds.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve post-tool message streams:** `extensions/anthropic/interactive-proxy/wrapper.ts:301-319`
  - body: Claude tool use can end one `/v1/messages` stream with `stop_reason: "tool_use"` and then continue after the CLI runs the tool. This branch treats every `message_start` after the first as auxiliary, so tool-using turns can lose the continuation and never emit the final result.
  - confidence: 0.91
- **[P2] Finish terminal non-end_turn stops:** `extensions/anthropic/interactive-proxy/wrapper.ts:347-363`
  - body: The Messages API has terminal success stop reasons such as `max_tokens`, `stop_sequence`, and `refusal`. When one arrives, this branch forwards the stream event but skips result emission and never kills the child, turning a valid final response into a watchdog timeout.
  - confidence: 0.87
- **[P2] Fail closed when proxy key hardening fails:** `extensions/anthropic/interactive-proxy/cert-manager.ts:119-132`
  - body: `ensureCerts()` returns cached or generated MITM signing keys even when `icacls` fails or no Windows user is available. For reusable proxy CA keys, failed ACL repair should stop use or regenerate safely instead of continuing with possibly broad access.
  - confidence: 0.89

## Security Review

Status: needs_attention

Summary: Needs attention: the PR introduces a child-scoped local TLS MITM proxy and still treats proxy private-key ACL hardening as best-effort.

Concerns:

- **[medium] Fail closed on proxy key hardening failures:** `extensions/anthropic/interactive-proxy/cert-manager.ts:130`
  - body: The certificate manager ignores `icacls`/`chmod` failures and continues with cached or newly generated CA/leaf private keys. A reusable local MITM signing key should not be used after ACL repair fails.
  - confidence: 0.89
- **[medium] Review the TLS interception trust boundary:** `extensions/anthropic/interactive-proxy/wrapper.ts:191`
  - body: The wrapper forces Claude through a loopback HTTPS proxy with child-scoped `HTTPS_PROXY` and `NODE_EXTRA_CA_CERTS`, decrypting Anthropic API traffic locally to recover thinking events. Even opt-in, this needs explicit maintainer/security acceptance and user-facing documentation.
  - confidence: 0.88
