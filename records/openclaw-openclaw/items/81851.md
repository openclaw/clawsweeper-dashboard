## What This Changes

The PR adds an opt-in Anthropic `claude-cli-interactive` backend that runs Claude Code through a Bun wrapper and loopback TLS proxy to reconstruct interactive-mode SSE reasoning output, plus supporting parser/runtime/auth changes, tests, and unrelated doctor warning edits.

## Best Possible Solution

Land this only after the opt-in backend preserves the existing Claude CLI contracts, documents its runtime/security requirements, and has current-head live proof plus maintainer approval for the TLS trust model.

## Reproduction Assessment

Not applicable as a current-main bug: this PR proposes a new backend. Source inspection confirms current main only registers `claude-cli`, while the supplied proof is contributor-side evidence for the proposed branch rather than a failing main reproduction.

## Solution Assessment

No as merge-ready. The Anthropic plugin boundary is plausible, but the branch still skips an existing Claude CLI skills integration and needs security approval plus current-head live proof before this is the best maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Wire interactive backend into Claude skills plugin:** `extensions/anthropic/cli-backend-interactive.ts:67-69`
  - body: The new backend registers a different id, but `prepareClaudeCliSkillsPlugin` only materializes the Claude plugin when `backendId` is exactly `claude-cli`. That means `claude-cli-interactive` runs will skip the native Claude skills plugin even though current Claude CLI behavior exposes the OpenClaw skills snapshot through that plugin. Add the interactive id to that gate and cover it so users do not lose skill resolver support when opting into this backend.
  - confidence: 0.83

## Security Review

Status: needs_attention

Summary: The patch adds an opt-in local TLS MITM path around Anthropic traffic, so maintainer security/product approval is still required.

Concerns:

- **[medium] Approve the local TLS interception boundary:** `extensions/anthropic/interactive-proxy/wrapper.ts:431`
  - body: The wrapper sets `HTTPS_PROXY` and `NODE_EXTRA_CA_CERTS` for the spawned Claude process so OpenClaw can decrypt Anthropic SSE traffic locally. That may be acceptable as an explicit backend, but it is a new trust boundary and should not land without owner approval and user-facing documentation.
  - confidence: 0.94
- **[medium] Require current-head proof for key hardening:** `extensions/anthropic/interactive-proxy/cert-manager.ts:147`
  - body: The current PR code attempts to re-harden cached and regenerated proxy key permissions, but the linked proof branch still shows older broad key modes and predates later hardening commits. Before merge, proof should show the current head repairing or creating the CA and leaf private keys with the intended restrictions on a supported platform.
  - confidence: 0.88
