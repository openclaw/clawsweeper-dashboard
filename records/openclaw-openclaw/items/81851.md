## Summary

Keep this PR open for review, not cleanup-close: current main does not have the proposed backend, but the latest branch still has a blocking session-state defect and needs maintainer security/product approval plus current-head live proof.

## What This Changes

The PR adds an opt-in Anthropic `claude-cli-interactive` backend with a Bun loopback TLS proxy, JSONL reasoning parsing, setup registration, static assets, tests, and a Doctor warning change.

## Best Possible Solution

Land only if the backend remains opt-in and plugin-owned, preserves same-session Claude serialization, has an accepted TLS-proxy trust model, and has current-head real proof showing reasoning and final replies through Telegram.

## Reproduction Assessment

Not applicable as a current-main bug; this PR adds a new opt-in backend. The blocking review finding is source-reproducible because `serialize: false` bypasses Claude session queueing in `resolveCliRunQueueKey`.

## Solution Assessment

No, not yet. The plugin-owned backend boundary is plausible, but this patch should not merge until session serialization is fixed, the TLS proxy trust model is accepted, and latest-head Telegram proof is supplied.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Keep Claude sessions serialized:** `extensions/anthropic/cli-backend-interactive.ts:127`
  - body: The new backend sets `serialize: false`, which makes `resolveCliRunQueueKey` return a per-run key before the Claude session/workspace queueing branch. Two runs resuming the same Claude session can then spawn concurrently and race on the same session transcript; the existing `claude-cli` backend keeps `serialize: true`, which still allows different sessions to run independently while protecting same-session state.
  - confidence: 0.88

## Security Review

Status: needs_attention

Summary: The diff introduces a local TLS interception path and reusable local signing keys, so the trust model needs explicit maintainer approval before merge.

Concerns:

- **[medium] Approve local TLS interception:** `extensions/anthropic/interactive-proxy/wrapper.ts:408`
  - body: The wrapper points the spawned Claude process at a local proxy and trusts a generated CA so OpenClaw can decrypt Anthropic API traffic and inspect plaintext SSE events. That may be acceptable for an opt-in backend, but it is a security and product trust-boundary decision.
  - confidence: 0.94

  - confidence: 0.86
