## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Pin every proxy env alias to the loopback proxy:** `extensions/anthropic/interactive-proxy/wrapper.ts:590-595`
  - body: `claudeEnv` preserves the parent environment and then sets only uppercase `HTTPS_PROXY`. If the gateway process already has `https_proxy`, `HTTP_PROXY`, `ALL_PROXY`, `NO_PROXY`, or lowercase `no_proxy` set, the Claude child can bypass or override the intended loopback proxy, so the wrapper may never see the SSE events used for reasoning/tool progress and Telegram delivery. Clear the proxy/no-proxy aliases and set the supported proxy variables consistently for this child.
  - confidence: 0.91
- **[P1] Normalize and fail closed before opening CONNECT tunnels:** `extensions/anthropic/interactive-proxy/mitm-server.ts:422-426`
  - body: The CONNECT router compares the raw host to `api.anthropic.com` and tunnels every other target with `net.connect`. Case/format variants of the Anthropic API host can bypass capture, and non-Anthropic CONNECT requests get raw egress through OpenClaw's MITM process. Normalize host/port first, route only the exact approved Anthropic target to the local TLS server, and reject or explicitly allowlist everything else.
  - confidence: 0.94
- **[P1] Declare the Bun type package used by the proxy tsconfig:** `extensions/anthropic/interactive-proxy/tsconfig.json:4`
  - body: This tsconfig asks TypeScript for `bun-types`, and the PR adds this project to `tsgo:prod`, but the Anthropic plugin package at the same head does not declare `bun-types`. A fresh production typecheck should not depend on an undeclared ambient package, so add the owning devDependency or remove the type reference.
  - confidence: 0.9
- **[P1] Keep the inheritance hook private or document the SDK contract:** `src/plugins/cli-backend.types.ts:201-204`
  - body: `CliBackendPlugin` is exported through `openclaw/plugin-sdk/cli-backend`, so adding `inheritUserConfigFrom` creates a public plugin API. Current CLI backend plugin docs list the advanced hooks and do not define this one. Either keep the inheritance mechanism internal to bundled backends, or update the public SDK docs/API contract and add focused compatibility coverage.
  - confidence: 0.86
- **[P1] Carry current Claude CLI capability flags onto the variant:** `extensions/anthropic/cli-backend-interactive.ts:116`
  - body: Current main's `claude-cli` backend declares `sideQuestionToolMode: "disabled"` and `ownsNativeCompaction: true`; this new backend declares only `nativeToolMode`. Because it spawns the same Claude CLI and shares the same resumable transcript behavior, the rebased variant needs the same current capability semantics or side-question/tool-disable and compaction paths will treat it differently from `claude-cli`.
  - confidence: 0.84
