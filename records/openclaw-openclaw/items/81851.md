## Summary

Keep this PR open, but it is not merge-ready: the latest patch still has source-level integration bugs, the direct wrapper proof does not exercise the registered OpenClaw/Telegram path, and the TLS-interception design needs explicit maintainer security/product approval.

## What This Changes

The PR adds an experimental Anthropic `claude-cli-interactive` CLI backend using a Bun loopback TLS proxy to capture Claude CLI SSE thinking/text deltas and route them into OpenClaw reasoning streams.

## Best Possible Solution

Accept this only after maintainer-approved TLS interception consent, backend-specific live-session and error contracts, safe config inheritance, focused regression tests, and redacted integrated OpenClaw plus Telegram proof.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not as a current-main bug because this is a new backend feature, but the blocking PR defects are source-reproducible from the PR diff and current live-session/parser contracts.

## Solution Assessment

Is this the best way to solve the issue? No; the wrapper experiment is plausible, but the submitted integration still breaks established CLI config inheritance, reuses print-mode live argv, and reports proxy startup failures in a parser-incompatible shape.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Keep inherited Claude config from replacing the wrapper command:** `src/agents/cli-backends.ts:230`
  - body: When `claude-cli-interactive` has no direct override, this spreads the whole `claude-cli` override into the variant. Existing supported configs can set `agents.defaults.cliBackends.claude-cli.command` to an absolute Claude binary path, and `mergeBackendConfig` then replaces the interactive backend's `command: "bun"` while args still start with `wrapper.ts`, so OpenClaw spawns `claude wrapper.ts ...` instead of `bun wrapper.ts ...`. Inherit only safe fields or handle the Claude binary path separately.
  - confidence: 0.91
- **[P2] Keep interactive sessions off print-mode argv:** `src/agents/cli-runner/claude-live-session.ts:119`
  - body: Adding `claude-cli-interactive` to the generic Claude live-session gate sends it through `buildClaudeLiveArgs`, which appends `--input-format stream-json`, `--output-format stream-json`, `--permission-prompt-tool stdio`, and `--replay-user-messages`. Those are the stream-json/print-mode arguments this backend is trying to avoid, so the registered backend no longer matches the no-print-mode subscription path shown in the PR proof. Add a backend-specific live argv path or keep this backend out of the generic live-session builder.
  - confidence: 0.9
- **[P2] Mark proxy fallback results as parser errors:** `extensions/anthropic/interactive-proxy/wrapper.ts:165-168`
  - body: `runClaudeDirect` emits a `result` record with `subtype: "error"`, an empty `result`, and an `error` field, but the Claude live-session parser only treats result records as failures when `is_error === true`. A cert or proxy startup failure can therefore resolve the turn as an empty successful reply before the child exit is observed. Emit `is_error: true` with the message in `result`, or update the parser contract for this shape.
  - confidence: 0.88

## Security Review

Status: needs_attention

Summary: The diff intentionally adds local TLS interception and a persisted CA, so it needs explicit maintainer security/product approval before merge.

Concerns:

- **[medium] Approve the TLS interception opt-in boundary:** `extensions/anthropic/interactive-proxy/wrapper.ts:135`
  - body: The wrapper sets `HTTPS_PROXY` and `NODE_EXTRA_CA_CERTS` for the spawned Claude CLI so OpenClaw can decrypt Anthropic API traffic. Even though the proxy is loopback and child-scoped, this should not ship without an explicit maintainer-approved consent and opt-in model.
  - confidence: 0.96
- **[medium] Approve the persisted local CA lifecycle:** `extensions/anthropic/interactive-proxy/cert-manager.ts:8`
  - body: The cert manager stores a reusable CA under the user's OpenClaw directory for 365 days. Rotation, cleanup, permissions, and disclosure need product/security approval before exposing this backend to users.
  - confidence: 0.9
