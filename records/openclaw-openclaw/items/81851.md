## Summary

Keep open. This draft experimental backend still has integrated OpenClaw contract defects around Claude live-session routing, fallback error handling, and session persistence, and the intentional TLS interception needs maintainer security/product review before merge.

## What This Changes

The PR adds an experimental Anthropic `claude-cli-interactive` backend with a Bun TLS MITM proxy, JSONL thinking/text parsing, reasoning-lane bridging, CLI backend inheritance support, package assets/deps, and unrelated doctor/changelog edits.

## Best Possible Solution

Decide the TLS-interception consent and key-storage policy first; if accepted, give the backend an execution path whose stdin, session, and error contracts match interactive Claude and require integrated channel proof.

## Reproduction Assessment

Yes for the blocking patch defects by source inspection: PR head routes `claude-cli-interactive` into live-session mode, which injects stream-json flags and writes JSON stdin, and the wrapper records do not satisfy existing error/session contracts. I did not run a live Claude subscription session.

## Solution Assessment

No. The wrapper idea may be worth exploring after maintainer approval, but the current patch is not the best mergeable shape because the integrated runtime path contradicts the intended interactive contract and lacks end-to-end proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Keep interactive backend out of Claude live-session mode:** `src/agents/cli-runner/claude-live-session.ts:119`
  - body: Adding `claude-cli-interactive` here makes OpenClaw start the wrapper through `buildClaudeLiveArgs`, which appends `--input-format stream-json`, `--output-format stream-json`, `--permission-prompt-tool stdio`, and `--replay-user-messages`, then sends JSON user records to stdin. That is not the no-`-p` interactive path demonstrated by the wrapper proof, so selecting this backend from OpenClaw can exercise a different contract than the proposed backend relies on.
  - confidence: 0.93
- **[P2] Mark proxy fallback results as live-session errors:** `extensions/anthropic/interactive-proxy/wrapper.ts:215-220`
  - body: When cert generation or proxy startup fails, `runClaudeDirect` emits a `result` with `subtype: "error"` but no `is_error: true`. The live-session handler only rejects result records when `parsed.is_error === true`, so this can resolve the turn as an empty success before the child exit is handled.
  - confidence: 0.88
- **[P2] Return a session id for fresh interactive turns:** `extensions/anthropic/interactive-proxy/wrapper.ts:166`
  - body: Fresh live-session starts strip `--session-id` before launching the process, and the wrapper ignores Claude stdout, so `extractSessionId` is empty and successful first-turn results omit `session_id`. With `sessionMode: "always"`, OpenClaw cannot persist the session it needs to resume after the warm process exits.
  - confidence: 0.78

## Security Review

Status: needs_attention

Summary: The diff intentionally intercepts Anthropic TLS traffic and persists a local CA, so maintainer security and consent review is required.

Concerns:

- **[medium] Review TLS interception and persisted CA:** `extensions/anthropic/interactive-proxy/wrapper.ts:177`
