## Summary

Keep open: the draft PR is not merge-ready because the new backend is not wired through the current Claude JSONL/parser/reasoning contracts, the supplied proof only runs the wrapper directly, and the branch also adds security-sensitive TLS interception plus unrelated write-capable CI automation.

## What This Changes

The PR adds an experimental Anthropic `claude-cli-interactive` CLI backend using a local TLS proxy to surface streamed thinking deltas, plus a model-catalog refresh workflow and script.

## Best Possible Solution

Split the catalog-refresh workflow into its own reviewed PR; if maintainers accept the TLS-proxy backend direction, wire it through the existing CLI dialect/parser/manifest contracts with focused tests, integrated real behavior proof, and explicit opt-in/security documentation.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence failure path: selecting `claude-cli-interactive` uses a provider id and synthetic output shape that current main's parser and reasoning bridge do not consume.

## Solution Assessment

No. The proposed capability may be useful, but this implementation bypasses existing CLI backend contracts and introduces a TLS MITM trust boundary plus unrelated CI automation before the product/security decision is settled.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Declare the Claude stream JSON dialect:** `extensions/anthropic/cli-backend-interactive.ts:76-77`
  - body: The new backend id is `claude-cli-interactive`, but current main only parses Claude stream-json when `jsonlDialect` is `claude-stream-json` or the provider id is exactly `claude-cli`. Without declaring the dialect and updating the reasoning bridge policy, normal Claude `result` and `text_delta` output will not flow through the assistant/reasoning path for this backend.
  - confidence: 0.91
- **[P1] Emit a parser-supported thinking delta:** `extensions/anthropic/interactive-proxy/wrapper.ts:93-113`
  - body: The wrapper writes synthetic partial `assistant` messages containing a `thinking` content block, but `createCliJsonlStreamingParser` only emits streaming assistant deltas from Claude `stream_event` / `content_block_delta` / `text_delta` records. These injected thinking records will be ignored by the current OpenClaw parser, so the tapped content will not reach the reasoning lane.
  - confidence: 0.88
- **[P2] Check wrapper existence before using the compiled path:** `extensions/anthropic/cli-backend-interactive.ts:23-27`
  - body: `fileURLToPath(new URL("./interactive-proxy/wrapper.js", selfUrl))` only converts the URL; it does not check whether `wrapper.js` exists. In source/Bun dev mode this selects a missing compiled file instead of falling back to `wrapper.ts`, even though the proof command runs the TypeScript wrapper directly.
  - confidence: 0.86
- **[P2] Declare the backend in plugin metadata:** `extensions/anthropic/register.runtime.ts:603`
  - body: The runtime registers `claude-cli-interactive`, but the Anthropic manifest still declares only `claude-cli`. Metadata-driven setup and ownership discovery use declared CLI backend ids before runtime registration, so this backend will be missing from those flows.
  - confidence: 0.84

## Security Review

Status: needs_attention

Summary: The PR introduces local TLS interception and an unrelated write-capable scheduled workflow, both of which need explicit security/supply-chain review before merge.

Concerns:

- **[medium] Review the local TLS MITM trust boundary:** `extensions/anthropic/interactive-proxy/cert-manager.ts:6`
  - body: The backend generates a local CA and leaf certificate, injects `NODE_EXTRA_CA_CERTS`, forces Claude traffic through `HTTPS_PROXY`, terminates TLS locally, and reads Anthropic API SSE traffic. That needs an explicit opt-in threat model, key lifetime/storage policy, and documented failure behavior before merge.

  - body: The PR adds a scheduled workflow with `contents: write`, `pull-requests: write`, and a new Anthropic API secret even though the PR title is about an interactive backend. This widens the trusted CI surface and should be reviewed separately or removed from this feature branch.
