## Summary

Keep open: this is an active draft feature PR adding a new Claude interactive backend with local TLS interception; current main does not implement it, and source review found blocking correctness, security, and proof gaps.

## What This Changes

The PR adds an opt-in `claude-cli-interactive` Anthropic backend, Bun loopback TLS proxy assets, thinking-stream routing, CLI backend config inheritance, doctor warning changes, tests, and a Bun type dependency.

## Best Possible Solution

Keep this as a draft until maintainers accept the trust model, then narrow the implementation so all real turns stream JSONL reliably, config overrides preserve existing CLI backend semantics, lightweight policy artifacts stay aligned, and proof includes redacted live JSONL plus Telegram output.

## Reproduction Assessment

Yes for the review findings: source inspection of PR head shows tool-free SSE streams are not tapped, direct command overrides are misclassified, and the policy artifact misses the new alias. I did not run a live Claude subscription or Telegram flow.

## Solution Assessment

No in its current shape: an opt-in backend may be the right direction, but this implementation needs correctness fixes, stronger proof, and a maintainer/security decision on the TLS interception trust model before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Tap tool-free conversation streams:** `extensions/anthropic/interactive-proxy/mitm-server.ts:87-88`
  - body: `startMitmProxy` returns the upstream SSE uninspected whenever the `/v1/messages` request has no non-empty `tools` array. The wrapper builds JSONL output only from `proxy.onEvent`, so tool-free real turns, tools-disabled runs, or no-MCP configurations can emit `init` and then never emit `stream_event` or `result`, ending in a timeout instead of a reply.
  - confidence: 0.91
- **[P2] Preserve direct wrapper command overrides:** `extensions/anthropic/cli-backend-interactive.ts:145-146`
  - body: `normalizeConfig` treats any merged command other than `bun` as the Claude binary and then forces `command` back to `bun`. That works for inherited `claude-cli.command`, but a direct `cliBackends.claude-cli-interactive.command` override, such as an absolute Bun path, is ignored and passed to the wrapper as `OPENCLAW_INTERACTIVE_CLAUDE_BINARY`, so the wrapper tries to spawn that runner as Claude.
  - confidence: 0.88
- **[P2] Keep thinking policy artifacts aligned:** `extensions/anthropic/provider-contract-api.ts:10`
  - body: This adds `claude-cli-interactive` to the lightweight provider contract, but the sibling `provider-policy-api.ts` still returns Claude thinking profiles only for `anthropic` and `claude-cli`. Cold `/think` and model-policy paths that resolve bundled provider policy before runtime activation will miss the Claude thinking ladder for the new backend.
  - confidence: 0.85

## Security Review

Status: needs_attention

Summary: The diff introduces a local TLS interception path and supply-chain/config changes that need explicit security review before merge.

Concerns:

- **[medium] Review TLS interception trust boundary:** `extensions/anthropic/interactive-proxy/wrapper.ts:143`
  - body: The wrapper forces the spawned Claude child through a loopback HTTPS proxy using `HTTPS_PROXY` and trusts a generated CA via `NODE_EXTRA_CA_CERTS`. Even scoped to the child process, this changes the Anthropic traffic trust boundary and should require explicit maintainer/security acceptance and user-facing consent semantics.
  - confidence: 0.93
- **[medium] Lock down persisted proxy key material:** `extensions/anthropic/interactive-proxy/cert-manager.ts:186`
  - body: The cert manager persists CA and leaf private keys under the user OpenClaw directory, but skips permission tightening on Windows and the supplied cert-listing artifact shows key files as `-rw-r--r--`. For a CA trusted by the spawned child, the key files should be generated and verified as private on every supported platform.
  - confidence: 0.82
- **[low] Pin the Bun type dependency:** `package.json:1816`
  - body: The PR adds root `bun-types` with a caret range while nearby root dev dependencies are exact-pinned. The lockfile fixes the version today, but the manifest range broadens future resolution for this new build/type surface.
  - confidence: 0.76
