## What This Changes

The PR adds an experimental `claude-cli-interactive` Anthropic backend that runs Claude Code through a Bun wrapper and loopback TLS proxy to reconstruct stream-json reasoning output from interactive-mode SSE traffic.

## Best Possible Solution

Land this only as an opt-in Anthropic plugin backend after maintainer approval of the TLS trust model, current-head live Telegram proof, cross-platform/build validation, and confirmation that existing `claude-cli` behavior remains unchanged.

## Reproduction Assessment

Not applicable as a current-main bug: this is a new backend feature PR. Source inspection confirms current main only registers `claude-cli`, while the PR proof shows an older-head real setup was exercised but not current-head visible Telegram behavior.

## Solution Assessment

Unclear as a merge-ready solution. The Anthropic plugin boundary is plausible, but the TLS interception trust model, Claude Code internal coupling, and stale proof make maintainer review and live validation the safer path before landing.
