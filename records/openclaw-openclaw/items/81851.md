Keep open: this is an active draft feature PR for a new Anthropic CLI backend, not work that is already on main. The branch still has blocking runtime defects, lacks required real behavior proof, and needs maintainer security/product approval for the TLS interception design.

## What This Changes

The PR adds an experimental `claude-cli-interactive` Anthropic backend with a Bun local TLS proxy, CLI-backend config inheritance, reasoning stream plumbing, packaging assets, tests, and doctor preview-warning changes.

## Best Possible Solution

Decide the TLS-interception policy first; if accepted, repair the default binary and packaged asset paths, split unrelated doctor work, add focused coverage, and require integrated backend plus Telegram proof.

## Reproduction Assessment

Yes for the blocking review findings: PR-head source inspection shows the default config can set the Claude binary env to `bun`, and the package asset list omits a preload required by the wrapper. I did not run a live Claude subscription or Telegram session.

## Solution Assessment

No. The backend idea may be useful after maintainer approval, but this is not the best merge shape until the security policy is accepted, the runtime packaging/default-path bugs are fixed, and integrated behavior is proven.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Keep the default backend pointed at Claude:** `extensions/anthropic/cli-backend-interactive.ts:139-147`
  - body: `normalizeConfig` always copies `merged.command` into `OPENCLAW_INTERACTIVE_CLAUDE_BINARY`. With no inherited `claude-cli` override, `merged.command` is this backend's own `bun`, and `wrapper.ts` then spawns that env value as the Claude binary, so selecting the default backend cannot start Claude.
  - confidence: 0.94
- **[P1] Ship the TTY preload with the proxy assets:** `extensions/anthropic/package.json:18-30`
  - body: `wrapper.ts` preloads `interactive-proxy/tty-spoof.cjs`, but the package metadata declares only the wrapper, MITM server, and cert manager as static assets. Packaged installs will miss the preload file and the Claude child will fail before exercising the proxy.
  - confidence: 0.91

## Security Review

Status: needs_attention

Summary: The diff intentionally adds local TLS interception for Anthropic traffic and a persisted local CA, which needs explicit maintainer security and consent review before merge.

Concerns:

- **[medium] Review TLS interception and local CA consent:** `extensions/anthropic/interactive-proxy/wrapper.ts:130`
  - body: The wrapper sets `HTTPS_PROXY` and `NODE_EXTRA_CA_CERTS` for the Claude child, while `cert-manager` stores a CA and leaf key under the user's OpenClaw directory. Even though the proxy is loopback-scoped, this changes the trust boundary for Anthropic API traffic and should not land without a clear product/security decision and user-consent model.
  - confidence: 0.94
