## What This Changes

The branch adds an experimental Anthropic `claude-cli-interactive` backend with a Bun local TLS proxy, TTY preload, JSONL thinking-stream parsing, config inheritance, and related CLI reasoning bridge wiring.

## Best Possible Solution

Keep the backend explicitly experimental and opt-in only if maintainers accept the TLS-proxy trust model, then fix inherited print-mode arg filtering, provide inspectable live proof, rebase, and run focused Anthropic/CLI plus Telegram validation.

## Reproduction Assessment

Yes for the blocking review finding: source inspection shows inherited args strip `-p` but not `--print`, and the Claude CLI contract treats both as non-interactive print mode. I did not reproduce the full live Claude subscription and Telegram proxy flow locally.

## Solution Assessment

No in its current shape: the opt-in backend may be viable, but it is not merge-ready until the trust-boundary decision, inherited flag handling, and inspectable real behavior proof are resolved.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Strip the long print-mode flag:** `extensions/anthropic/cli-backend-interactive.ts:24-25`
  - body: The inherited `claude-cli` override path removes `-p` but leaves `--print`, which Claude treats as the same non-interactive print mode. Any user override written with the documented long flag will keep this backend out of interactive mode, bypassing the proxy behavior this PR is meant to enable and losing the subscription `thinking_delta` stream. Treat `--print` like `-p` before forwarding inherited args.
  - confidence: 0.88

## Security Review

Status: needs_attention

Summary: The patch intentionally intercepts Anthropic TLS traffic for a spawned Claude child and adds a new root dev dependency, so maintainer security and supply-chain review are still needed.

Concerns:

- **[medium] Review TLS interception and local CA consent:** `extensions/anthropic/interactive-proxy/wrapper.ts:130`
  - body: The wrapper routes Claude through a loopback HTTPS proxy using `NODE_EXTRA_CA_CERTS`, and the cert manager persists CA and leaf private keys under the user's OpenClaw directory. Even scoped to the spawned child, this changes the trust boundary for Anthropic traffic and needs explicit consent/security review before merge.
  - confidence: 0.95
- **[low] Pin the new Bun type dependency exactly:** `package.json:1816`
  - body: The PR adds `bun-types` to the root devDependencies with a caret range. The lockfile pins the current version, but the manifest range broadens future dependency resolution for a repository that otherwise keeps this dependency block exact.
  - confidence: 0.78
