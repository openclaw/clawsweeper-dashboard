- **[P1] Pin every proxy env alias to the MITM proxy:** `extensions/anthropic/interactive-proxy/wrapper.ts:488-495`
  - body: The child env spreads inherited `process.env` and only overrides uppercase `HTTPS_PROXY`. Current proxy/env resolution treats lowercase proxy vars and `NO_PROXY` as meaningful, so a gateway started with `https_proxy` or a bypass list can make Claude skip the local MITM, losing SSE capture or operator proxy policy. Build a fixed child env that sets or clears HTTP/HTTPS/ALL proxy aliases and `NO_PROXY` consistently, with focused coverage.
  - confidence: 0.86

Current network-proxy docs require protected runtime egress through the operator-managed proxy and call raw sockets out as unsupported unless explicitly wired, so managed-proxy deployments can lose enforced egress/audit. Either fail closed for non-API tunnels or route them through the managed proxy path with proof.
  - confidence: 0.82
