## What This Changes

The PR adds an experimental Anthropic `claude-cli-interactive` backend that runs Claude through Bun, a loopback TLS proxy, stream-json parsing, config inheritance, and reasoning-stream wiring.

## Best Possible Solution

Keep the PR as draft until the source and lockfile blockers are fixed, then require redacted live JSONL plus Telegram proof and an explicit maintainer/security decision on the proxy trust model.

## Reproduction Assessment

Yes for the review findings: source inspection of PR head shows the exact `{"title":"..."}` suppression path, and the manifest/lockfile specifiers disagree. I did not run a live Claude subscription or Telegram flow in this read-only review.

## Solution Assessment

No in its current shape: an opt-in backend may be a viable direction, but this patch still needs correctness fixes, synchronized dependency metadata, stronger real behavior proof, and maintainer acceptance of the TLS proxy design.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Stop suppressing valid title JSON replies:** `extensions/anthropic/interactive-proxy/wrapper.ts:246-251`
  - body: The title-generation guard still classifies any exact short JSON object with only a `title` string as auxiliary.   - confidence: 0.9

  - body: `package.json` pins `bun-types` to `1.3.14`, but the lockfile importer still records `^1.3.14`. That leaves dependency metadata out of sync and can fail frozen-lockfile install checks; update the lockfile specifier to match the manifest.
  - confidence: 0.94

## Security Review

Status: needs_attention

Summary: The diff introduces local TLS interception plus persisted proxy CA/private keys, so the trust and storage model needs explicit security review before merge.

Concerns:

- **[medium] Review TLS interception trust boundary:** `extensions/anthropic/interactive-proxy/wrapper.ts:155`
  - body: The wrapper sets `HTTPS_PROXY` and `NODE_EXTRA_CA_CERTS` for the Claude child so OpenClaw can terminate and inspect Anthropic API traffic locally. Even scoped to the child, this changes the provider traffic trust boundary and needs explicit maintainer/security acceptance and user-consent semantics.
  - confidence: 0.93

  - body: The cert manager persists CA and leaf private keys and treats Windows ACL tightening failures as non-fatal. The linked proof listing shows readable key files, so platform-specific key permissions should be verified before shipping.
  - confidence: 0.78
