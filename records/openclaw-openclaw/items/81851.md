## Summary

Keep this PR open for maintainer review: it is an active draft adding a new security-sensitive Anthropic backend, and the latest branch fixed several earlier wiring issues but still has correctness and proof blockers.

## What This Changes

The PR adds an experimental Anthropic `claude-cli-interactive` backend with a Bun local TLS proxy plus parser, runner, reasoning-lane, manifest, and static-asset wiring for streamed `thinking_delta` events.

## Best Possible Solution

Land this only if maintainers approve the TLS-proxy direction, with explicit opt-in policy/docs, constrained CA lifecycle, fallback-only reasoning behavior, serialized stream delivery, tests, and integrated redacted live proof.

## Reproduction Assessment

Yes for the review findings: source inspection shows the PR sends thinking events to `stream: "thinking"` while also mirroring assistant text to `onReasoningStream` for the same backend, and the new thinking bridge does not drain async deliveries. I did not run a live Claude subscription session.

## Solution Assessment

No. The narrow maintainable path is to make assistant-text mirroring a fallback only when no thinking stream is present, serialize/drain thinking deliveries like the existing bridge, and require maintainer approval for the TLS interception boundary.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Make assistant reasoning a fallback only:** `src/auto-reply/reply/agent-runner-execution.ts:1558-1562`
  - body: Adding `claude-cli-interactive` to the assistant-text reasoning bridge while also forwarding `stream: "thinking"` means thinking-capable runs can send both captured thinking and normal assistant text to `onReasoningStream`. The assistant-text bridge should be disabled after real thinking arrives, or otherwise made conditional as the fallback described in the PR body.
  - confidence: 0.88
- **[P2] Drain thinking stream deliveries before cleanup:** `src/auto-reply/reply/agent-runner-execution.ts:1577-1579`
  - body: `onReasoningStream` may be async, and the existing assistant/reasoning bridges serialize and drain those callbacks before the run completes. This new bridge fires the callback with `void` and has no `drain()`, so channel updates can race run cleanup and rejections are no longer contained.
  - confidence: 0.84

## Security Review

Status: needs_attention

Summary: The PR intentionally decrypts Claude CLI Anthropic API traffic through a generated local CA/TLS proxy, so it needs explicit maintainer security review before merge.

Concerns:

- **[medium] Review the TLS interception trust boundary:** `extensions/anthropic/interactive-proxy/wrapper.ts:85`
  - body: The wrapper injects `HTTPS_PROXY` and `NODE_EXTRA_CA_CERTS` into the Claude CLI child so OpenClaw can decrypt Anthropic API traffic. That is security-sensitive by design and should ship only with explicit opt-in policy, docs, failure behavior, and maintainer approval.
  - confidence: 0.93
- **[medium] Constrain the generated CA lifecycle:** `extensions/anthropic/interactive-proxy/cert-manager.ts:54`
  - body: The proxy generates and reuses a local CA under the OS temp directory with a long CA lifetime. Before merge, the implementation should document and constrain file permissions, rotation, cleanup, and what can trust that CA.
  - confidence: 0.84
