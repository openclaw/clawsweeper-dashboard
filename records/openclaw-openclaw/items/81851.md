## Summary

Keep this PR open, but it is not merge-ready: the integrated backend path still has two source-provable correctness blockers, the real behavior proof only exercises the wrapper directly, and the intentional TLS interception design still needs maintainer approval.

## What This Changes

The PR adds an experimental Anthropic `claude-cli-interactive` backend that launches Claude through a Bun wrapper and local TLS proxy to stream captured `thinking_delta` and `text_delta` events into OpenClaw's reasoning path.

## Best Possible Solution

If maintainers accept the TLS-proxy direction, keep the backend isolated behind an explicit opt-in path, fix the default binary and live-session argv handling, add focused regression coverage, and require redacted integrated OpenClaw proof before merge.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: select `claude-cli-interactive` with default config and the wrapper receives `OPENCLAW_INTERACTIVE_CLAUDE_BINARY=bun`; integrated live-session runs also pass the generic stream-json print-mode flags.

## Solution Assessment

No. The direction may be useful, but this patch is not yet the best maintainable implementation because the integrated execution path is still wrong and the TLS interception trust model needs explicit maintainer approval.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Default the wrapper back to the Claude binary:** `extensions/anthropic/cli-backend-interactive.ts:147`
  - body: With no inherited `claude-cli` override, `merged.command` is this backend's own `bun`, so `normalizeConfig` stores `OPENCLAW_INTERACTIVE_CLAUDE_BINARY=bun`. `wrapper.ts` then spawns Bun with Claude flags instead of launching `claude`, which breaks the default `claude-cli-interactive` path. Only set this env var when a real inherited/custom Claude binary exists, or have the wrapper fall back to `claude`.
  - confidence: 0.94
- **[P2] Keep interactive runs out of the print-mode arg builder:** `src/agents/cli-runner/claude-live-session.ts:119`
  - body: Adding `claude-cli-interactive` to `shouldUseClaudeLiveSession` sends it through `buildClaudeLiveArgs`, which still appends `--input-format stream-json` and `--output-format stream-json` before the wrapper launches Claude. That contradicts the PR's no-print-mode subscription path and means integrated OpenClaw runs do not match the wrapper proof. Split this backend out of the generic Claude live arg builder or make the builder backend-aware.
  - confidence: 0.9

## Security Review

Status: needs_attention

Summary: The diff intentionally adds local TLS interception for Anthropic API traffic, so the consent and trust boundary need maintainer approval before merge.

Concerns:

- **[medium] Review the local TLS interception trust boundary:** `extensions/anthropic/interactive-proxy/wrapper.ts:135`
  - body: The wrapper starts a loopback MITM proxy, injects `HTTPS_PROXY` and `NODE_EXTRA_CA_CERTS` into the Claude child, and relies on a persisted local CA key under the user's OpenClaw directory. The scope is intentionally constrained, but shipping this changes decrypted-network trust and should be an explicit product/security decision.
