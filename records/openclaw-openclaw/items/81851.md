## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Preserve resumed system prompt files:** `src/agents/cli-runner/execute.ts:289-290`
  - body: Current main deliberately keeps `systemPromptWhen: "always"` active for resumed Claude CLI sessions. Gating system prompt file creation on `!useResume` makes existing `claude-cli` runs drop OpenClaw's system prompt again on resume.
  - confidence: 0.96
- **[P1] Keep loadSessionEntry clone support:** `src/gateway/session-utils.ts:890-900`
  - body: Current gateway callers still pass `{ clone: false }` to borrow the cached session store. Removing that option from `loadSessionEntry` and always calling `loadSessionStore(storePath)` will break those callers/typecheck and change the hot-path store semantics.
  - confidence: 0.95
- **[P1] Keep external CLI auth overlays in harness selection:** `src/agents/command/attempt-execution.ts:131-133`
  - body: `ensureAuthProfileStore` needs the external CLI profile/provider overlays here so runtime Claude/Codex CLI profiles are visible during auth resolution. Dropping them can hide the selected CLI auth profile and route the run with the wrong auth context.
  - confidence: 0.9
- **[P2] Preserve user-turn transcript hook writes:** `src/agents/command/attempt-execution.ts:232-243`
  - body: The current path writes user turns through `appendUserTurnTranscriptMessage` with `runAgentHarnessBeforeMessageWriteHook` and the structured user message when present. Replacing it with a plain transcript append drops that hook and user-turn normalization.
  - confidence: 0.87

## Security Review

Status: needs_attention

Summary: The patch introduces a security-sensitive local TLS MITM proxy for Anthropic traffic and needs explicit maintainer trust-boundary review before merge.

Concerns:

- **[high] Approve the local MITM trust boundary:** `extensions/anthropic/interactive-proxy/mitm-server.ts:64`
  - body: The backend terminates Anthropic TLS locally, observes credential-bearing SSE traffic, and caches reusable CA/leaf private keys under the OpenClaw state dir. Loopback binding and key hardening help, but this is still a new security boundary.
  - confidence: 0.88
- **[medium] Constrain the pass-through CONNECT proxy:** `extensions/anthropic/interactive-proxy/mitm-server.ts:284`
  - body: The CONNECT server forwards non-Anthropic tunnels to the requested host while it is alive. A local process that discovers the ephemeral port could use that unauthenticated loopback proxy for arbitrary TCP egress during the turn.
  - confidence: 0.72
