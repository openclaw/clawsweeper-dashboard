## Summary

Keep open. Current main does not implement this backend, but the draft patch still has blocking correctness issues, lacks inspectable real behavior proof for the live Telegram/proxy path, and needs explicit maintainer security judgment for the TLS interception design.

## What This Changes

The branch adds an opt-in Anthropic `claude-cli-interactive` backend, Bun loopback TLS proxy assets, thinking-stream routing, CLI backend config inheritance, and related doctor warning updates.

## Best Possible Solution

Keep the backend draft and experimental until maintainers accept the trust model, then land a narrow version that streams thinking immediately, keeps lightweight provider surfaces aligned, preserves fallback context, and includes redacted live Telegram/proxy proof.

## Reproduction Assessment

Yes for the review findings: source inspection of the latest PR head shows delayed thinking-event emission, stale lightweight provider artifacts, and a hard-coded fallback session binding. I did not reproduce the live Claude subscription plus Telegram flow.

## Solution Assessment

No in its current shape: a separate opt-in backend may be maintainable, but this implementation needs correctness fixes, real behavior proof, and a maintainer trust-boundary decision before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Stream thinking events before title filtering:** `extensions/anthropic/interactive-proxy/wrapper.ts:191-192`
  - body: The wrapper starts buffering at `message_start` and only flushes once a later `text_delta` proves the response is not title JSON. Extended-thinking turns usually emit `thinking_delta` before assistant text, so the new reasoning path still withholds the live thinking stream until text arrives or the turn stops.
  - confidence: 0.91
- **[P2] Keep provider artifacts in sync with the new alias:** `extensions/anthropic/register.runtime.ts:489`
  - body: The runtime provider adds `claude-cli-interactive` as an Anthropic hook alias, but the lightweight provider contract, policy, and discovery artifacts still only expose `claude-cli`. Control-plane paths that avoid loading full runtime code can miss the new backend's Anthropic ownership, synthetic auth, or thinking policy unless those artifacts and tests are updated too.
  - confidence: 0.86
- **[P2] Read the matching CLI session when falling back:** `src/agents/command/attempt-execution.ts:393-396`
  - body: After the predicate starts treating `claude-cli-interactive` as a Claude CLI provider, fallback retries can enter this branch for the new backend, but the prelude still reads the hard-coded `claude-cli` session binding. That drops the interactive session context when failing over to another provider; use the original provider's binding instead.
  - confidence: 0.83

## Security Review

Status: needs_attention

Summary: The diff introduces a local TLS interception path and a root dependency range change that need explicit security and supply-chain review.
