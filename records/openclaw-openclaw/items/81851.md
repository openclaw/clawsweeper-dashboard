  - body: The manifest now advertises `claude-cli-interactive`, but setup lookup narrows to the manifest owner and returns only backends registered by `setup-api`.   - confidence: 0.87
- **[P2] Bind proxy ports without the probe-close race:** `extensions/anthropic/interactive-proxy/mitm-server.ts:35-37`
  - body: `startMitmProxy()` probes a free port, closes that probe socket, and only then binds the actual Bun TLS/CONNECT servers. Because the new backend sets `serialize: false`, concurrent turns can pick the same preferred ports and one wrapper will fail to start the proxy; bind port 0 directly or retry around the actual bind.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The diff introduces child-scoped TLS interception and persisted proxy CA material, so it needs explicit maintainer/security review before merge.

Concerns:

- **[medium] Review the TLS interception trust boundary:** `extensions/anthropic/interactive-proxy/wrapper.ts:122`
  - body: The wrapper routes the Claude child through a loopback HTTPS proxy and trusts a generated CA via `NODE_EXTRA_CA_CERTS`, decrypting `api.anthropic.com` traffic locally. Even though this is child-scoped, it changes the provider traffic trust boundary and should require explicit user-facing opt-in and maintainer/security approval.
  - confidence: 0.93
- **[medium] Verify proxy CA key permissions and lifetime:** `extensions/anthropic/interactive-proxy/cert-manager.ts:145`
  - body: The cert manager persists reusable CA and leaf private keys under the user's OpenClaw config directory for 365 days, and Windows ACL hardening failures are swallowed. Key lifetime, rotation, and platform permissions need explicit verification before this ships.
  - confidence: 0.82
