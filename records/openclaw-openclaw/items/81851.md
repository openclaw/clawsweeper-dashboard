## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Pin proxy environment aliases to the loopback proxy:** `extensions/anthropic/interactive-proxy/wrapper.ts:590-596`
  - body: `claudeEnv` preserves the parent environment and sets only uppercase `HTTPS_PROXY`. If the gateway process has lowercase proxy variables, `HTTP_PROXY`, `ALL_PROXY`, or `NO_PROXY` set, the Claude child can bypass or override the intended loopback proxy, so the wrapper may never see the SSE events used for reasoning/tool progress. Clear the proxy/no-proxy aliases and set the supported proxy variables consistently for this child.
  - confidence: 0.92
- **[P1] Reject or normalize CONNECT targets before tunneling:** `extensions/anthropic/interactive-proxy/mitm-server.ts:422-426`
  - body: The CONNECT router compares the raw host to `api.anthropic.com` and tunnels every other target with `net.connect`. Case/format variants of the Anthropic API host can miss capture, and non-Anthropic CONNECT requests get raw egress through OpenClaw's MITM process. Normalize host/port first and reject or explicitly allowlist everything else.
  - confidence: 0.94
- **[P1] Wire the interactive backend into Claude CLI skills:** `extensions/anthropic/cli-backend-interactive.ts:103`
  - body: Registering a new `claude-cli-interactive` backend is not enough for skills: at this head `prepareClaudeCliSkillsPlugin` still returns empty args unless the backend id is exactly `claude-cli`. That means selected OpenClaw skills are silently omitted for interactive runs. Route the helper through the Claude CLI family matcher and add the regression test before registering the variant.
  - confidence: 0.91
- **[P1] Carry Claude CLI side-question and compaction flags forward:** `extensions/anthropic/cli-backend-interactive.ts:114-116`
  - body: The new backend spawns the same Claude Code session backend but only declares `nativeToolMode`; current `claude-cli` also declares `sideQuestionToolMode: "disabled"` and `ownsNativeCompaction: true`. Without those flags, `/btw` no-tools handling and over-budget compaction behavior diverge from existing Claude CLI sessions.
  - confidence: 0.88
- **[P2] Declare the Bun types used by the proxy typecheck:** `extensions/anthropic/interactive-proxy/tsconfig.json:4`
  - body: This tsconfig asks TypeScript for `bun-types`, and the PR adds this project to `tsgo:prod`, but the Anthropic plugin package at this head does not declare `bun-types`. A fresh production typecheck should not depend on undeclared ambient types, so add the owning devDependency or remove the type reference.
  - confidence: 0.9
- **[P2] Document the new CLI backend inheritance hook:** `src/plugins/cli-backend.types.ts:201-204`
  - body: `CliBackendPlugin` is exported through `openclaw/plugin-sdk/cli-backend`, so adding `inheritUserConfigFrom` creates a public plugin API. Current CLI backend plugin docs list the advanced hooks and do not define this one. Either keep the inheritance mechanism internal to bundled backends, or update the public SDK docs/API contract and focused compatibility coverage.
  - confidence: 0.86
