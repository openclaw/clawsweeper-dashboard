## Summary

Keep this PR open for normal review: current main does not have the new Anthropic interactive backend, but the branch is still draft and has a blocking source-level issue, insufficient current-head real behavior proof, and a security/product trust-boundary review requirement.

## What This Changes

The PR adds an opt-in `claude-cli-interactive` Anthropic backend that runs Claude Code interactive mode through a Bun wrapper and local TLS proxy, then wires captured SSE thinking events into OpenClaw CLI reasoning streams.

## Best Possible Solution

Land this only as an opt-in Anthropic plugin backend after fixing request classification, documenting and approving the TLS trust model, supplying current-head live proof, and keeping existing `claude-cli` behavior unchanged.

## Reproduction Assessment

Not applicable as a current-main bug: this is a new backend feature and current main only registers `claude-cli`. The blocking PR defect is source-reproducible from the request classifier and CLI preparation paths, but I did not run a live Claude turn.

## Solution Assessment

Unclear as a merge-ready solution. The plugin-owned opt-in backend shape is plausible, but the tool-less stream drop, local TLS interception model, and stale proof need resolution before this is the narrow maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Do not drop tool-less user streams:** `extensions/anthropic/interactive-proxy/mitm-server.ts:158-160`
  - body: The classifier marks every `/v1/messages` request with no `tools` array as `auxiliary`, and the wrapper drops auxiliary streams without emitting a result. The CLI runner only adds the OpenClaw MCP config when the loopback runtime exists; if that startup path warns and continues without MCP tools, a normal user turn can arrive with no tools and will hang until the watchdog instead of replying. Classify user turns without depending on `tools` being present, or fail before launch when the backend requires MCP tools.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: Needs maintainer attention because the PR adds opt-in TLS interception and reusable local CA material around Anthropic traffic.

Concerns:

- **[medium] Approve the local TLS interception boundary:** `extensions/anthropic/interactive-proxy/wrapper.ts:431`
  - body: The wrapper sets `HTTPS_PROXY` and `NODE_EXTRA_CA_CERTS` for the Claude child so OpenClaw can decrypt Anthropic SSE traffic. That may be acceptable for an explicit backend, but it changes the provider trust boundary and should be owner-approved and documented before merge.
  - confidence: 0.94
- **[medium] Require current-head proof for CA key hardening:** `extensions/anthropic/interactive-proxy/cert-manager.ts:119`
  - body: The code now attempts to harden cached and regenerated proxy keys, but the supplied proof artifact still shows older broad key modes and predates the latest commits. Current-head proof should show the expected permissions or equivalent live setup output.
  - confidence: 0.86
