## Summary

Keep this draft PR open, but it is not merge-ready: source review still finds blocking correctness issues in the new reasoning/proxy path, the TLS interception design needs explicit security/product review, and the external real-behavior proof is descriptive rather than inspectable.

## What This Changes

The PR adds an opt-in Anthropic `claude-cli-interactive` backend, Bun loopback TLS proxy assets, thinking-stream routing, CLI backend config inheritance, doctor warning changes, tests, and a root Bun type dependency.

## Best Possible Solution

Keep this as an experimental draft until the proxy trust model is accepted, then land a narrow version that streams thinking immediately, keeps lightweight provider metadata aligned, preserves fallback context, and includes redacted live proxy/Telegram proof.

## Reproduction Assessment

Yes for the review findings: source inspection of PR head shows delayed thinking emission, stale lightweight provider artifacts, and a hard-coded fallback session binding. I did not reproduce the live Claude subscription plus Telegram flow.

## Solution Assessment

No in its current shape: an opt-in backend may be the right product direction, but this implementation needs correctness fixes, inspectable proof, and a maintainer/security decision on the local TLS interception model before merge.

## Review Findings

Overall correctness: patch is incorrect

Full review comments:

- **[P2] Stream thinking events before title filtering:** `extensions/anthropic/interactive-proxy/wrapper.ts:191-192`
  - body: The wrapper starts buffering at `message_start` and routes every event through `emitOrBuffer`, so `thinking_delta` events that arrive before the first non-title `text_delta` stay pending until later text or `message_stop`. That defeats the live reasoning stream for extended-thinking turns whose thinking arrives before assistant text.
  - confidence: 0.91

Metadata/control-plane paths that rely on those artifacts can miss synthetic auth and thinking policy for the new backend.
  - confidence: 0.86
- **[P2] Read the matching CLI session on fallback:** `src/agents/command/attempt-execution.ts:278`
  - body: This predicate now includes `claude-cli-interactive`, but the fallback prelude still reads the hard-coded `claude-cli` session binding. Fallbacks from the interactive backend will drop the prior CLI session context instead of carrying the matching session into the retry.
  - confidence: 0.83

## Security Review

Status: needs_attention

Summary: The diff introduces a local TLS interception path and a root dependency range change that need explicit security and supply-chain review.
