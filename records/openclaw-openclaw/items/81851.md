## What This Changes

This PR adds an experimental Anthropic `claude-cli-interactive` backend that runs Claude through a Bun loopback TLS proxy, reconstructs Claude stream-json events, extends Claude CLI routing/auth gates, adds proxy static assets, tests, and a `bun-types` dev dependency.

## Best Possible Solution

Keep the backend opt-in, repair state-dir and overflow handling, then require explicit maintainer acceptance of the local TLS proxy design with packaged cross-platform proof before merge.

## Reproduction Assessment

Yes for the PR-introduced defects: source inspection shows hard-coded proxy key storage outside `OPENCLAW_STATE_DIR`, and the long-prompt spill path asks Claude to use `Read` without auto-allowing it. The feature path itself is supported by supplied Windows live logs rather than a current-main bug reproduction.

## Solution Assessment

No. The backend direction is plausible, but this is not the best merge shape until persistent key storage follows OpenClaw's state-dir contract, overflow recovery cannot hang on permissions, and maintainers accept the TLS proxy boundary.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Store proxy keys under the configured state directory:** `extensions/anthropic/interactive-proxy/cert-manager.ts:9`
  - body: The reusable MITM CA and leaf keys are hard-coded under `homedir()/.openclaw`, but OpenClaw's mutable state root can be moved with `OPENCLAW_STATE_DIR`. Operators using profiles or an explicit state directory would still get signing keys in the default home tree, which breaks isolation and cleanup expectations. Resolve the cache from the OpenClaw state dir contract or pass the resolved state dir into the wrapper, and add override coverage.
  - confidence: 0.91
- **[P2] Make overflow prompt recovery non-interactive:** `extensions/anthropic/interactive-proxy/wrapper.ts:334`
  - body: When the prompt exceeds `maxPromptArgChars`, OpenClaw sends it on stdin; the wrapper then writes it to a temp file and asks Claude to read it, but the backend only auto-allows `mcp__openclaw__*`. In a non-bypass setup this can stop on a `Read` permission prompt while stdio is being spoofed/piped, so long prompts can hang instead of completing. Use a non-interactive delivery path or explicitly allow the narrow temp-file read with tests.
  - confidence: 0.86
