## Summary

Keep this PR open for maintainer review: it adds a plausible Anthropic plugin-owned backend, but it is still draft, not implemented on current main, security-sensitive, and the supplied live proof is stale relative to the latest head.

## What This Changes

The PR adds an opt-in `claude-cli-interactive` Anthropic backend that runs Claude through a Bun wrapper and loopback TLS proxy, registers the backend in plugin/setup paths, streams captured thinking into agent events, and adjusts CLI config inheritance/static assets.

## Best Possible Solution

Land this only as an opt-in Anthropic plugin backend after security owners approve the local TLS proxy model, current-head Telegram/log proof is attached, and the branch is rebased with focused package/build validation for the static proxy assets.

## Reproduction Assessment

Not applicable as a current-main bug: this is a feature PR for a new backend. Current main only registers `claude-cli`; the supplied proof exercises branch behavior rather than a failing current-main reproduction.

## Solution Assessment

Unclear as a merge-ready solution. The Anthropic plugin boundary is plausible, but the TLS interception design, current-head proof gap, cross-platform gaps, and draft/conflicting state need maintainer review before this can be accepted as the best path.
