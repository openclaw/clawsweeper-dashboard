Full review comments:

- **[P1] Pin every proxy alias to the local MITM:** `extensions/anthropic/interactive-proxy/wrapper.ts:492`
  - body: The wrapper spreads the managed-proxy/host environment and then overwrites only `HTTPS_PROXY`. Current managed proxy mode sets both upper- and lower-case proxy aliases and clears bypass vars; if Claude or its runtime honors `https_proxy`, `HTTP_PROXY`, `ALL_PROXY`, or `NO_PROXY` instead, the request can skip the MITM and silently lose the SSE stream this backend depends on. Set or clear the full alias/bypass set for the spawned Claude process.
  - confidence: 0.88
- **[P1] Keep MITM upstream egress on the managed proxy path:** `extensions/anthropic/interactive-proxy/mitm-server.ts:208-213`
  - body: The MITM child forwards decrypted API requests with Bun `fetch`, and the same server later opens pass-through tunnels with raw `net.connect`. Because this Bun child does not install OpenClaw's inherited Proxyline runtime or use a managed CONNECT helper, `proxy.enabled=true` deployments can send Anthropic traffic directly instead of through the operator proxy. Route these upstream calls through the managed proxy contract or fail closed while it is active.
  - confidence: 0.86
- **[P1] Remove or authorize the dependency graph change:** `extensions/anthropic/package.json:9`
  - body: This external PR adds `bun-types` and lockfile entries, and the dependency graph guard says external dependency graph changes are blocked unless secops approves the exact head SHA. Either remove/internalize the dependency delta or get the required approval before merge.
