## Review Findings

Full review comments:

- **[P2] Register the interactive backend in setup-api:** `extensions/anthropic/openclaw.plugin.json:46`
  - body: The manifest now advertises `claude-cli-interactive`, and setup lookup first narrows to the manifest owner before returning only backends registered by `setup-api`. Since the Anthropic setup entry still registers only `buildAnthropicCliBackend()`, cold setup/live-test/fallback paths cannot resolve this backend until the full runtime registry is active. Register `buildAnthropicInteractiveCliBackend()` from the setup entry as well.
  - confidence: 0.9
- **[P2] Update the static asset inventory test:** `extensions/anthropic/package.json:18-34`
  - body: These new `staticAssets` entries change `listStaticExtensionAssetOutputs()`, but `test/scripts/runtime-postbuild.test.ts` still expects only the existing acpx/diffs assets. The runtime postbuild inventory test will fail until the Anthropic outputs are added or the inventory expectation is otherwise updated.
  - confidence: 0.87
- **[P2] Bind proxy ports without the probe-close race:** `extensions/anthropic/interactive-proxy/mitm-server.ts:35-37`
  - body: `startMitmProxy()` probes a free port, closes that probe socket, and only then binds the actual Bun TLS/CONNECT servers. Because the new backend sets `serialize: false`, concurrent turns can pick the same preferred ports and one wrapper will fail to start the proxy. Bind on port 0 directly or retry around the actual server creation/listen calls.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The diff introduces child-scoped TLS interception and persisted proxy CA material, so it needs explicit maintainer/security review before merge.

Concerns:

- **[medium] Review the TLS interception trust boundary:** `extensions/anthropic/interactive-proxy/wrapper.ts:122`
  - body: The wrapper routes the Claude child through a loopback HTTPS proxy and trusts a generated CA via `NODE_EXTRA_CA_CERTS`, decrypting `api.anthropic.com` traffic locally. Even though this is child-scoped, it changes the provider traffic trust boundary and should require explicit user-facing opt-in and maintainer/security approval.
  - confidence: 0.93
- **[medium] Verify proxy CA key permissions and lifetime:** `extensions/anthropic/interactive-proxy/cert-manager.ts:145`
  - body: The cert manager persists reusable CA and leaf private keys under `~/.openclaw/proxy-certs` for 365 days, and Windows ACL hardening failures are swallowed. The key lifetime, rotation, and platform permissions need explicit verification before this ships.
  - confidence: 0.82
