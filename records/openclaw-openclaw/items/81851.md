## Summary

Keep open: current main lacks the backend, and the PR head still has two source-reproducible correctness blockers plus security trust-boundary review needs before merge.

## What This Changes

Adds an experimental Anthropic `claude-cli-interactive` backend that runs Claude through a Bun wrapper and loopback TLS proxy to reconstruct Claude stream-json events and route captured thinking into the reasoning preview.

## Best Possible Solution

Keep this as a draft until maintainers accept the proxy trust model, then fix the two correctness blockers and require redacted live JSONL plus Telegram proof before ready-for-review.

## Reproduction Assessment

Yes for the review findings: source inspection of PR head shows title-shaped real replies can be dropped and direct interactive backend command overrides are overwritten. I did not run a live Claude subscription or Telegram flow.

## Solution Assessment

No in its current shape: an opt-in backend may be the right direction, but this implementation needs correctness fixes, stronger proof, and maintainer/security acceptance of the proxy trust model before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Do not suppress real JSON title replies:** `extensions/anthropic/interactive-proxy/wrapper.ts:234-239`
  - body: The title-generation guard treats any buffered response whose complete text parses as an object with a string `title` field as auxiliary and returns before emitting the final result. A real user turn that asks Claude to output exactly `{"title":"..."}` and does not produce `thinking_delta` will therefore drop the reply and wait for the watchdog instead.
  - confidence: 0.87
- **[P2] Preserve direct Bun command overrides:** `extensions/anthropic/cli-backend-interactive.ts:170`
  - body: A direct `cliBackends.claude-cli-interactive.command` override is still discarded because `normalizeConfig` always returns `command: "bun"`. Operators who configure an absolute Bun path for this backend will silently fall back to PATH lookup, so installs where Bun is not on PATH or must use a specific binary cannot launch the wrapper.
  - confidence: 0.84

## Security Review

Status: needs_attention

Summary: The diff introduces a local TLS interception path and proxy key material that need explicit maintainer/security review before merge.

Concerns:

- **[medium] Review TLS interception trust boundary:** `extensions/anthropic/interactive-proxy/wrapper.ts:143`
  - body: The wrapper forces the spawned Claude child through a loopback HTTPS proxy and trusts a generated CA through `NODE_EXTRA_CA_CERTS`; even scoped to the child, this changes the Anthropic traffic trust boundary and needs explicit security/product acceptance and user-consent semantics.
  - confidence: 0.93
- **[medium] Lock down persisted proxy private keys:** `extensions/anthropic/interactive-proxy/cert-manager.ts:183`
  - body: The cert manager persists CA and leaf private keys under the user's OpenClaw directory and skips chmod on Windows; the linked proof listing shows key files with read bits, so platform-specific ACL handling should be verified before merge.
  - confidence: 0.78
- **[low] Pin the new Bun type dependency:** `package.json:1816`
  - body: The PR adds root `bun-types` with a caret range while adjacent root dev dependencies are exact-pinned, broadening future package resolution for the new build/type surface.
  - confidence: 0.72
