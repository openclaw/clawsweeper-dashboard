## Summary

Keep this PR open but do not merge it yet: the current patch still misses the OpenClaw parser/reasoning path, does not package the proxy wrapper for dist, and needs explicit maintainer security/product review for TLS interception.

## What This Changes

The PR adds an experimental Anthropic `claude-cli-interactive` CLI backend that runs Claude CLI through a Bun local TLS proxy to surface streamed `thinking_delta` events.

## Best Possible Solution

If maintainers approve the TLS proxy direction, land it only after the wrapper is shipped in dist, the parser/runner has an explicit reasoning-delta contract with tests, the Claude CLI helper predicates cover the new backend, and integrated redacted live proof shows the OpenClaw reasoning lane working.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence failure path: the PR emits `thinking_delta` JSONL that current main's CLI streaming parser ignores, and the new backend id is not accepted by the current CLI reasoning bridge.

## Solution Assessment

No. The approach may be viable if maintainers approve TLS interception, but this patch needs explicit parser/reasoning support, packaged runtime scripts, and security documentation before it is a maintainable OpenClaw core solution.

## Review Findings

Overall correctness: patch is incorrect

Full review comments:

- **[P1] Parse thinking deltas before claiming reasoning streaming:** `extensions/anthropic/interactive-proxy/wrapper.ts:71-72`
  - body: The wrapper now emits `stream_event` records whose delta type is `thinking_delta`, but current `createCliJsonlStreamingParser` only forwards `text_delta` events. These records will be ignored, so the captured thinking never reaches the reasoning lane.
  - confidence: 0.93
- **[P1] Ship the proxy wrapper in packaged builds:** `extensions/anthropic/cli-backend-interactive.ts:25`
  - body: `resolveWrapperPath()` falls back to `interactive-proxy/wrapper.ts` when `wrapper.js` is absent, but packaged OpenClaw only ships built dist entries/static assets. Because these nested scripts are not imported build entries or declared static assets, the published backend can point at a missing wrapper file.
  - confidence: 0.9
- **[P1] Route the new backend through the Claude reasoning bridge:** `extensions/anthropic/cli-backend-interactive.ts:32`
  - body: Registering `claude-cli-interactive` creates a new CLI provider id, but the current reasoning bridge only treats `claude-cli` as bridgeable. Even parser-supported assistant deltas from this backend would skip `onReasoningStream` unless the shared Claude CLI predicate is used there too.
  - confidence: 0.88

## Security Review

Status: needs_attention

Summary: The diff intentionally decrypts Claude CLI Anthropic API traffic through a local CA/TLS proxy, so it needs explicit maintainer security review before merge.

Concerns:

- **[medium] Review the TLS interception trust boundary:** `extensions/anthropic/interactive-proxy/wrapper.ts:85`
  - body: The wrapper injects `HTTPS_PROXY` and `NODE_EXTRA_CA_CERTS` into the Claude CLI child so OpenClaw can decrypt Anthropic API traffic. That is security-sensitive by design and needs explicit opt-in policy, docs, failure behavior, and approval before shipping.
  - confidence: 0.92
- **[medium] Constrain the generated CA lifecycle:** `extensions/anthropic/interactive-proxy/cert-manager.ts:60`
  - body: The proxy generates and reuses a local CA under the OS temp directory with a long certificate lifetime. Before merge, the implementation should document and constrain storage permissions, rotation, cleanup, and what can trust that CA.
  - confidence: 0.82
