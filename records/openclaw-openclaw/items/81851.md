## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Constrain the local CONNECT proxy:** `extensions/anthropic/interactive-proxy/mitm-server.ts:284-288`
  - body: The CONNECT listener only treats `api.anthropic.com` specially and otherwise forwards to whatever host/port the client asks for. Because it is an unauthenticated localhost proxy while each wrapper run is active, any local process that discovers the port can use OpenClaw as an arbitrary TCP tunnel; please reject non-Anthropic CONNECTs or restrict them to a documented allowlist/security boundary.
  - confidence: 0.89
- **[P2] Document the backend inheritance hook:** `src/plugins/cli-backend.types.ts:199-202`
  - body: This adds `inheritUserConfigFrom` to `CliBackendPlugin`, which is exported through `openclaw/plugin-sdk/cli-backend`, but the CLI-backend plugin docs still omit the hook from the public advanced-hook table. Plugin SDK changes are compatibility-sensitive, so third-party backend authors need the inheritance semantics and filtering contract documented before this ships.
  - confidence: 0.84
- **[P2] Keep the proxy runtime in checked lanes:** `tsconfig.extensions.json:19`
  - body: The proxy `.ts` files are shipped as static runtime assets, but this exclusion removes the whole `interactive-proxy` subtree from the normal extensions typecheck. The added local tsconfig is not wired into package scripts or the Anthropics package tsconfig, so CI can package broken Bun/TypeScript runtime code without noticing.
  - confidence: 0.86
