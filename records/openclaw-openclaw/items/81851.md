## Summary

The PR should stay open as draft feature work, but current head still has blocking correctness issues and needs maintainer/security acceptance for the local TLS interception model plus stronger real behavior proof.

## What This Changes

The PR adds an experimental `claude-cli-interactive` Anthropic backend that runs Claude through a Bun wrapper and loopback TLS proxy to reconstruct stream-json records from interactive-mode SSE traffic.

## Best Possible Solution

Keep this as a draft until maintainers accept the trust model, then narrow the implementation so real user turns cannot be mistaken for auxiliary title generation, backend config overrides preserve established CLI semantics, and proof includes redacted live JSONL plus Telegram output.

## Reproduction Assessment

Yes for the review findings: source inspection of PR head shows valid JSON replies can be suppressed by the title filter, and direct `command` overrides are overwritten. I did not run a live Claude subscription or Telegram flow.

## Solution Assessment

No in its current shape: an opt-in backend may be the right direction, but the implementation needs correctness fixes, stronger proof, and maintainer/security acceptance of the proxy trust model before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Do not suppress real JSON replies:** `extensions/anthropic/interactive-proxy/wrapper.ts:234-239`
  - body: The title-generation guard treats any buffered response whose complete text parses as an object with a string `title` field as auxiliary and returns before emitting a `result` or killing the child. A real user turn that asks Claude to output `{"title":"..."}` will therefore produce no final reply on this backend and wait for the watchdog instead.
  - confidence: 0.88
- **[P2] Preserve direct Bun command overrides:** `extensions/anthropic/cli-backend-interactive.ts:168-170`
  - body: A direct `cliBackends.claude-cli-interactive.command` override is still discarded because `normalizeConfig` always returns `command: "bun"`. Operators who configure an absolute Bun path for this backend will silently fall back to PATH lookup, so installs where Bun is not on PATH or must use a specific binary cannot launch the wrapper.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The diff introduces a local TLS interception path and supply-chain/config changes that need explicit security review before merge.

Concerns:

- **[medium] Review TLS interception trust boundary:** `extensions/anthropic/interactive-proxy/wrapper.ts:143`
  - body: The wrapper forces the spawned Claude child through a loopback HTTPS proxy and trusts a generated CA via `NODE_EXTRA_CA_CERTS`; even scoped to the child, this changes the Anthropic traffic trust boundary and needs explicit maintainer/security acceptance and user consent semantics.
  - confidence: 0.93
- **[medium] Lock down persisted proxy key material:** `extensions/anthropic/interactive-proxy/cert-manager.ts:183`
  - body: The cert manager persists CA and leaf private keys under the user's OpenClaw directory and only chmods on non-Windows platforms; the linked Windows proof lists key files as readable in the directory listing, so private-key ACL handling needs a platform-specific check.
  - confidence: 0.8
- **[low] Pin the Bun type dependency:** `package.json:1816`
  - body: The PR adds root `bun-types` with a caret range while adjacent root dev dependencies are exact-pinned, broadening future resolution for the new build/type surface.
