## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Store proxy keys under the configured state dir:** `extensions/anthropic/interactive-proxy/cert-manager.ts:9`
  - body: `CERT_DIR` is hard-coded to `homedir()/.openclaw/proxy-certs`, but OpenClaw supports `OPENCLAW_STATE_DIR` for isolated profiles and multiple gateways. This makes the reusable MITM CA/leaf keys shared outside the selected state root, so deleting or switching a profile will not control the proxy trust material; resolve the state dir from the environment or pass it in and add coverage for the override.
  - confidence: 0.9
- **[P2] Preapprove the overflow Read path before prompting Claude:** `extensions/anthropic/interactive-proxy/wrapper.ts:324-334`
  - body: For prompts over `maxPromptArgChars`, the wrapper writes the prompt to a temp file and tells Claude to read it, but the backend only passes `--allowedTools mcp__openclaw__*`. Claude Code documents `--allowedTools` as permission preapproval, not a restriction, so `Read` can still raise an interactive permission prompt under non-bypass modes and hang the turn; add an exact `Read` allowance for the per-run file or fail fast instead.
  - confidence: 0.86
