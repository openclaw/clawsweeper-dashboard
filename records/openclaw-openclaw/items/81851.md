## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Pin every proxy env alias to the loopback proxy:** `extensions/anthropic/interactive-proxy/wrapper.ts:590-594`
  - body: `claudeEnv` preserves the parent environment and then sets only uppercase `HTTPS_PROXY`. If the gateway process has lowercase proxy variables, `HTTP_PROXY`, `ALL_PROXY`, or `NO_PROXY` set, the Claude child can bypass or override the intended loopback proxy, so the wrapper may never see the SSE events used for reasoning/tool progress and Telegram delivery. Clear the proxy/no-proxy aliases and set the supported proxy variables consistently for this child.
  - confidence: 0.91
- **[P1] Normalize and fail closed before opening CONNECT tunnels:** `extensions/anthropic/interactive-proxy/mitm-server.ts:422-426`
  - body: The CONNECT router compares the raw host to `api.anthropic.com` and tunnels every other target with `net.connect`. Case/format variants of the Anthropic API host can bypass capture, and non-Anthropic CONNECT requests get raw egress through OpenClaw's MITM process. Normalize host/port first, route only the approved Anthropic target to the local TLS server, and reject or explicitly allowlist everything else.
  - confidence: 0.94
- **[P1] Include the interactive backend in Claude skills setup:** `extensions/anthropic/cli-backend-interactive.ts:103-116`
  - body: The new backend id is passed through the normal CLI runner, but `prepareClaudeCliSkillsPlugin` at this head still returns empty args unless `backendId` is exactly `claude-cli`. That means selected OpenClaw skills are silently omitted for `claude-cli-interactive` runs. Extend the Claude CLI family helper or the skills helper and add a regression test before registering the variant.
  - confidence: 0.9
- **[P1] Mirror current Claude CLI capability flags:** `extensions/anthropic/cli-backend-interactive.ts:114-116`
  - body: Current `claude-cli` declares `sideQuestionToolMode: "disabled"` and `ownsNativeCompaction: true`, while this variant declares only `nativeToolMode`. Since it spawns the same Claude Code session backend, `/btw` no-tools handling and compaction lifecycle will diverge from `claude-cli`. Carry the same capability flags with focused tests unless maintainers intentionally want different behavior.
  - confidence: 0.86
- **[P1] Declare the Bun type package used by the proxy tsconfig:** `extensions/anthropic/interactive-proxy/tsconfig.json:4`
  - body: This tsconfig asks TypeScript for `bun-types`, and the PR adds this project to `tsgo:prod`, but the Anthropic plugin package at the same head does not declare `bun-types`. A fresh production typecheck should not depend on an undeclared ambient package, so add the owning devDependency or remove the type reference.
  - confidence: 0.9
- **[P1] Document the public inheritance hook:** `src/plugins/cli-backend.types.ts:201-204`
  - body: `CliBackendPlugin` is exported through `openclaw/plugin-sdk/cli-backend`, so adding `inheritUserConfigFrom` creates a public plugin API. Current CLI backend plugin docs list the advanced hooks and do not define this one. Either keep the inheritance mechanism internal to bundled backends, or update the public SDK docs/API contract and add focused compatibility coverage.
  - confidence: 0.86
