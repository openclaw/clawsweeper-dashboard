## Summary

Keep open: this is a recent draft feature PR, current main does not implement the new interactive backend, and the branch still has blocking correctness, proof, and security-review issues before it can be considered merge-ready.

## What This Changes

The PR adds an experimental Anthropic `claude-cli-interactive` backend that runs Claude Code through a Bun loopback TLS proxy and emits interactive SSE events as CLI JSONL reasoning output.

## Best Possible Solution

Keep the backend opt-in and experimental only if maintainers accept the TLS-proxy model, then fix the streaming and inherited-flag defects and require inspectable live proof before merge.

## Reproduction Assessment

Yes for the blocking review findings: source inspection of the PR diff shows thinking events are buffered until text or end-of-turn, and inherited `--print` is not stripped. I did not reproduce the full live Claude subscription plus Telegram flow.

## Solution Assessment

No in its current shape: a separate opt-in backend may be maintainable, but this implementation needs correctness fixes, real behavior proof, and a maintainer trust-boundary decision before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Stream thinking events before title filtering:** `extensions/anthropic/interactive-proxy/wrapper.ts:153-164`
  - body: The wrapper starts buffering at `message_start` and only flushes when a later `text_delta` proves the response is not a title JSON object. Any preceding `thinking_delta` stream events stay in `pendingEvents` until assistant text arrives or `message_stop`, so extended-thinking turns do not stream live reasoning through the new path the PR is meant to add.
  - confidence: 0.9
- **[P2] Strip the long print-mode flag:** `extensions/anthropic/cli-backend-interactive.ts:23-24`
  - body: Inherited `claude-cli` overrides can use `--print`, which Claude documents as the long form of noninteractive print mode. The filter removes `-p` but leaves `--print`, so those users would still spawn Claude outside interactive mode and bypass the backend's subscription/proxy behavior.
  - confidence: 0.88

## Security Review

Status: needs_attention

Summary: The diff introduces an opt-in TLS interception path and a dependency manifest change that need maintainer security and supply-chain review.

Concerns:
