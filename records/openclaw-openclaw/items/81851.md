## Summary

Keep open: this is an active draft feature PR, but it is not merge-ready because the integrated OpenClaw path still routes the new backend through Claude JSON live-session mode, real behavior proof is wrapper-only, and the TLS interception design needs explicit maintainer security/product approval.

## What This Changes

The branch adds an experimental Anthropic `claude-cli-interactive` backend with Bun TLS proxy assets, CLI JSONL thinking parsing/reasoning bridging, a CLI backend inheritance seam, tests/package metadata, and unrelated doctor/changelog edits.

## Best Possible Solution

Settle the TLS-interception opt-in/security policy first; if accepted, give this backend an execution path that does not reuse Claude JSON live-session flags, then fix error/session records and provide integrated reasoning-lane proof before merge.

## Reproduction Assessment

Yes for the blocking PR defects: source inspection shows `claude-cli-interactive` is routed into the existing Claude JSON live-session helper, and the wrapper fallback error record lacks `is_error`. I did not run a live Claude subscription session.

## Solution Assessment

No. The direct wrapper concept may be useful after maintainer security approval, but the current patch is not the best mergeable shape because the integrated path contradicts the intended interactive contract and lacks end-to-end proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Keep interactive backend out of Claude JSON live mode:** `src/agents/cli-runner/claude-live-session.ts:119`
  - body: Adding `claude-cli-interactive` here routes it through `buildClaudeLiveArgs`, which re-adds `--input-format stream-json`, `--output-format stream-json`, `--permission-prompt-tool stdio`, and `--replay-user-messages`, then sends JSON user records to stdin. That is not the no-`-p` interactive subscription path demonstrated by the wrapper proof, so the integrated OpenClaw backend path will not exercise the proxy as intended.
  - confidence: 0.93
- **[P2] Mark proxy fallback JSON as an error:** `extensions/anthropic/interactive-proxy/wrapper.ts:216-220`
  - body: `runClaudeDirect` emits a `result` record with `subtype: "error"`, but the Claude live-session handler only fails turns when `parsed.is_error === true`. If cert generation or proxy startup fails, OpenClaw can finish the turn as an empty success before the child exit is observed.
  - confidence: 0.88
- **[P2] Preserve the fresh proxy session id:** `extensions/anthropic/interactive-proxy/wrapper.ts:166`
  - body: Fresh live-session starts strip `--session-id` before spawning the long-lived process, and the wrapper ignores Claude stdout, so this conditional leaves successful first-turn results without a `session_id`. With `sessionMode: "always"`, OpenClaw then has no CLI session handle to resume after the warm wrapper closes.
  - confidence: 0.78
