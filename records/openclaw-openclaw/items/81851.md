## Summary

This PR should stay open for maintainer product and security review. It is a draft new backend that intentionally intercepts Anthropic TLS traffic, and the latest head still has functional issues in the integrated OpenClaw launch path.

## What This Changes

The PR adds an experimental Anthropic `claude-cli-interactive` backend, Bun-based local TLS proxy scripts, CLI backend inheritance plumbing, thinking-delta parsing, and reasoning-lane routing for captured Claude CLI subscription-path SSE events.

## Best Possible Solution

If maintainers approve this product and security direction, the backend should launch behind a clear opt-in path, preserve existing Claude CLI session/error semantics, avoid print-mode-only flags, add focused regression coverage, and include redacted integrated proof through OpenClaw's normal backend selection and reasoning-lane delivery.

## Reproduction Assessment

Yes, by source inspection of the PR diff and current main: the integrated backend enters the shared Claude live-session argv builder and the proxy fallback result lacks the current live-session error sentinel. I did not run a live Claude subscription session.

## Solution Assessment

No. The proposed direction may be useful, but this is not the best mergeable solution until the launch-path bugs, session/error semantics, security approval, and integrated proof gap are handled.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Keep interactive runs out of print-mode argv:** `src/agents/cli-runner/claude-live-session.ts:119`
  - body: Adding `claude-cli-interactive` to this gate sends integrated runs through `buildClaudeLiveArgs`, which appends `--input-format stream-json`, `--output-format stream-json`, `--permission-prompt-tool stdio`, and `--replay-user-messages`. Those are documented as print/non-interactive mode flags, so the OpenClaw path no longer preserves the interactive subscription-mode launch this backend is meant to provide. ([docs.claude.com](https://docs.claude.com/en/docs/claude-code/cli-reference?utm_source=openai))
  - confidence: 0.92
- **[P2] Set is_error on proxy startup failures:** `extensions/anthropic/interactive-proxy/wrapper.ts:176-181`
  - body: `runClaudeDirect()` emits `type: "result"` with `subtype: "error"`, but the live-session handler only treats `is_error === true` as a failed turn. If cert generation or proxy startup fails, OpenClaw can resolve the turn as an empty success before the child exit is processed.
  - confidence: 0.86
- **[P2] Return a session id for fresh interactive turns:** `extensions/anthropic/interactive-proxy/wrapper.ts:127-128`
  - body: Fresh live-session launches strip the generated `--session-id`, and the wrapper only echoes a session id already present in argv while ignoring Claude's stdout. The first successful turn therefore cannot persist a Claude session id for later resume after the warm wrapper process exits.
  - confidence: 0.78
