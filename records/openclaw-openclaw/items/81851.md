## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve system prompts on resumed CLI runs:** `src/agents/cli-runner/execute.ts:289`
  - body: This drops the `systemPromptWhen === "always"` escape hatch when creating the prompt file. Current main sets bundled `claude-cli` to `systemPromptWhen: "always"` and has regression coverage expecting resumed turns to keep `--append-system-prompt-file`; with this gate, resumed Claude CLI and interactive runs silently lose the OpenClaw system prompt.
  - confidence: 0.94
- **[P1] Keep loadSessionEntry clone support:** `src/gateway/session-utils.ts:887`
  - body: The PR removes the `clone` option from `loadSessionEntry`, but existing gateway callers and tests still pass `{ clone: false }`. That will either fail typecheck or force those hot paths back to cloned stores, undoing the borrowed-store semantics current gateway code depends on.
  - confidence: 0.9
- **[P1] Keep external CLI auth overlays in harness selection:** `src/agents/command/attempt-execution.ts:130-131`
  - body: Removing `externalCliProfileIds` and `externalCliProviderIds` means `ensureAuthProfileStore` no longer overlays runtime CLI profiles for the selected session or harness provider. External Claude/Codex-style auth profiles can disappear from `resolveAuthProfileOrder`, changing auth routing for existing users.
  - confidence: 0.86
- **[P2] Preserve user-turn transcript hook writes:** `src/agents/command/attempt-execution.ts:229-235`
  - body: Replacing `appendUserTurnTranscriptMessage` with the generic session append path drops the structured `userMessage`, `sessionKey`, and `runAgentHarnessBeforeMessageWriteHook` flow. Harnesses that rely on that pre-write hook or structured user-turn metadata will lose transcript behavior on CLI turns.
  - confidence: 0.82
