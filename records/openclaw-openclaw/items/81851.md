## Summary

Keep open: this draft PR is not ready to merge because the integrated OpenClaw launch path still has blocking bugs, the proof only exercises the wrapper directly, and the local TLS interception design needs explicit maintainer security/product approval.

## What This Changes

The PR adds an experimental Anthropic `claude-cli-interactive` backend that uses a local TLS proxy to capture Claude CLI subscription-path SSE thinking/text events and route them into OpenClaw streams.

## Best Possible Solution

If maintainers accept the TLS-proxy direction, give this backend a launch path that never re-adds print-mode flags, preserve real Claude binary and session/error semantics, add focused regression coverage, and require redacted integrated proof.

## Reproduction Assessment

Yes, by source inspection: default config sets the wrapper's Claude child binary to `bun`, and integrated runs enter the shared live-session argv builder that adds print-mode flags. I did not run a live Claude subscription session.

## Solution Assessment

No. The direction may be useful, but this patch is not the maintainable solution until the launch-path bugs, error/session semantics, integrated proof gap, and security approval are handled.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve the default Claude binary fallback:** `extensions/anthropic/cli-backend-interactive.ts:147`
  - body: When there is no inherited `claude-cli` override, `merged.command` is this backend's own `bun`. Storing that in `OPENCLAW_INTERACTIVE_CLAUDE_BINARY` makes `wrapper.ts` spawn `bun` with Claude args, so the default path never runs Claude or reaches the proxy.
  - confidence: 0.94
- **[P2] Keep interactive runs out of print-mode argv:** `src/agents/cli-runner/claude-live-session.ts:119`
  - body: Adding `claude-cli-interactive` to this gate sends it through `buildClaudeLiveArgs`, which appends print-mode stream-json, permission-prompt, and replay flags. Those flags are absent from the wrapper proof and conflict with the no-print-mode subscription path this backend is meant to preserve.
  - confidence: 0.9
- **[P2] Mark proxy startup failures as live-session errors:** `extensions/anthropic/interactive-proxy/wrapper.ts:179-181`
  - body: `runClaudeDirect()` emits a `type: "result"` record with `subtype: "error"`, but the live-session handler only treats `is_error: true` as failure. In integrated live mode, cert or proxy startup failure can resolve the turn as an empty success before the child exit is processed.
  - confidence: 0.82
- **[P2] Return a session id for fresh interactive sessions:** `extensions/anthropic/interactive-proxy/wrapper.ts:127`
  - body: Fresh live-session launches strip `--session-id`, and the wrapper ignores Claude's TUI stdout where the real session id would normally surface. Because the result only echoes an id already present in argv, first-run interactive sessions cannot persist a Claude session id for later resume after the warm process exits.
  - confidence: 0.8
