## Summary

Keep open. This is a recent draft feature PR and current main does not already provide the interactive Claude backend, but the latest diff still has blocking behavior issues plus security/proof gaps before it can be merged.

## What This Changes

The PR adds an opt-in Anthropic `claude-cli-interactive` backend that runs Claude Code through a Bun loopback TLS proxy, reconstructs stream-json events, wires thinking events into auto-reply reasoning, and adds related tests/static assets.

## Best Possible Solution

Land this only as an opt-in Anthropic plugin backend after fixing request classification and setup fallback inheritance, documenting/approving the TLS trust model, and supplying current-head live Telegram/proxy proof.

## Reproduction Assessment

No as a current-main bug; this is a new backend feature. Yes for the blocking findings: source inspection shows Haiku-model requests are classified as auxiliary and setup fallback resolution ignores inherited backend config.

## Solution Assessment

No, not yet. The plugin-owned backend boundary is plausible, but the latest diff needs the two blocking fixes plus current-head proof and a maintainer decision on the local TLS proxy trust boundary.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Do not classify Haiku user turns as auxiliary:** `extensions/anthropic/interactive-proxy/mitm-server.ts:119-120`
  - body: The proxy marks every request whose model contains `haiku` as `auxiliary` and drops that whole stream. The PR's own config proof advertises `claude-cli-interactive/claude-haiku-4-5`, so a real Haiku turn would never emit a `result` and would stall until the watchdog; classify side calls from request purpose/body instead of model family, or do not expose Haiku for this backend.
  - confidence: 0.91
- **[P2] Apply inherited CLI config on setup fallback paths:** `src/agents/cli-backends.ts:221-236`
  - body: `inheritUserConfigFrom` is only consulted through `registered`, but the PR also registers this backend through `setup-api` for cold setup/live-test/fallback paths. In those paths a user with only `cliBackends.claude-cli.command` configured loses the inherited Claude binary and the wrapper falls back to PATH `claude`; carry the inheritance metadata through the fallback policy or share the same override resolution there.
  - confidence: 0.78

## Security Review

Status: needs_attention

Summary: Needs maintainer attention because the patch introduces opt-in local TLS interception and reusable CA material around Anthropic traffic.

Concerns:

- **[medium] Approve the local TLS interception trust boundary:** `extensions/anthropic/interactive-proxy/wrapper.ts:431`
  - body: The wrapper routes Claude through `HTTPS_PROXY` and trusts a generated CA via `NODE_EXTRA_CA_CERTS` so OpenClaw can decrypt Anthropic SSE traffic. That may be acceptable for an explicit backend, but it is a security/product trust-boundary decision and should be documented and approved before merge.
  - confidence: 0.94
