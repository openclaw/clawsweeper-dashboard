## Summary

Keep open for review rather than cleanup-close: the PR is recent and proposes a real new Anthropic CLI backend, but the current branch is not mergeable because the new backend is not wired into OpenClaw's existing Claude CLI JSONL/parser/reasoning contracts and it adds security-sensitive local TLS interception plus unrelated write-capable CI automation.

## What This Changes

The PR adds a `claude-cli-interactive` Anthropic CLI backend that runs Claude through a Bun TLS MITM wrapper to surface `thinking_delta` data, and also adds CI/script automation to refresh Claude CLI model allowlists.

## Best Possible Solution

Split the catalog-refresh workflow into its own reviewed PR; if maintainers accept the local TLS proxy direction, wire the backend through the existing CLI dialect/parser/manifest contracts with focused tests, integrated real behavior proof, and explicit security documentation or opt-in gating.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence failure path: selecting `claude-cli-interactive` uses a provider id and synthetic output shape that the current parser and reasoning bridge do not consume; I did not run the live backend because this was a read-only review.

## Solution Assessment

No. The proposed direction may be useful, but a TLS MITM runtime backend is a product/security decision and the implementation should first use OpenClaw's existing CLI backend/parser/manifest contracts or add explicit, tested new contracts.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Wire the backend into the Claude stream contracts:** `extensions/anthropic/cli-backend-interactive.ts:76-77`
  - body: The new backend uses provider id `claude-cli-interactive`, but current main only treats Claude stream JSON as such when `jsonlDialect` is `claude-stream-json` or the provider id is exactly `claude-cli`. It also means the CLI assistant-to-reasoning bridge will not run for this backend. As added, selecting this backend will not parse the normal Claude `result`/`text_delta` stream into OpenClaw's assistant/reasoning path.
  - confidence: 0.91
- **[P1] Emit a parser-supported thinking delta:** `extensions/anthropic/interactive-proxy/wrapper.ts:93-113`
  - body: The wrapper writes synthetic partial `assistant` messages whose content block is `thinking`, but `createCliJsonlStreamingParser` only turns Claude `stream_event` / `content_block_delta` / `text_delta` records into assistant deltas. Those injected records will be ignored by the current OpenClaw parser, so the tapped `thinking_delta` content will not reach the reasoning lane even if the proxy captures it.
  - confidence: 0.88
- **[P2] Check the wrapper file before choosing the compiled path:** `extensions/anthropic/cli-backend-interactive.ts:23-27`
  - body: `fileURLToPath(new URL("./interactive-proxy/wrapper.js", selfUrl))` only converts a URL; it does not check whether `wrapper.js` exists. In source/Bun dev mode this always returns the missing compiled path instead of falling back to `wrapper.ts`, so the registered backend can fail even though the proof command runs `wrapper.ts` directly.
  - confidence: 0.86
- **[P2] Declare the new CLI backend in plugin metadata:** `extensions/anthropic/register.runtime.ts:603`
  - body: The runtime registers `claude-cli-interactive`, but the Anthropic plugin manifest still declares only `claude-cli`. Setup and ownership lookup use manifest-declared CLI backend ids before executing setup code, so this backend will be missing from metadata-driven discovery and setup flows.
  - confidence: 0.84
