## Summary

Keep open. This is a new experimental Anthropic backend, not cleanup that current main already implements; it remains draft and needs maintainer product/security approval, integrated real behavior proof, and fixes for concrete reliability issues before merge.

## What This Changes

The PR adds a `claude-cli-interactive` Anthropic CLI backend that runs Claude CLI through a Bun local TLS proxy, captures SSE thinking/text deltas, and routes them into OpenClaw CLI parsing and chat reasoning streams.

## Best Possible Solution

Accept this only if maintainers approve explicit opt-in TLS interception and the backend gains robust CA generation, structured fallback behavior, focused coverage, and redacted integrated OpenClaw plus Telegram proof.

## Reproduction Assessment

Not applicable: this is a new backend feature, not a current-main bug. The useful verification path is an integrated `claude-cli-interactive` OpenClaw run with Telegram reasoning-stream proof, not a failing baseline reproduction.

## Solution Assessment

Unclear. The implementation may be a workable experiment, but TLS interception, reusable CA storage, fallback semantics, and consent/opt-in policy need maintainer approval and hardening before this is the best maintained solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Serialize cert generation before sharing proxy files:** `extensions/anthropic/interactive-proxy/cert-manager.ts:82-83`
  - body: `ensureCerts` uses one shared `~/.openclaw/proxy-certs` path and deletes/regenerates the cert/key files without an inter-process lock or temp-file rename. Two first-run or expired sessions can both enter this block and leave the CA/key/leaf set mismatched or missing, which breaks the advertised concurrent-session safety.
  - confidence: 0.89
- **[P2] Preserve JSONL output on proxy fallback:** `extensions/anthropic/interactive-proxy/wrapper.ts:160`
  - body: When cert generation or proxy startup fails, the wrapper falls back to `claude` with the interactive backend's args. Those args no longer include the normal `-p` stream-json path, and stdout is inherited, so OpenClaw's JSONL parser can receive TUI output or no `result` record instead of a structured fallback.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: Needs attention: the PR deliberately decrypts Anthropic traffic with a generated local CA and should not ship without explicit opt-in/security approval and hardened cert lifecycle.

Concerns:

- **[medium] Approve the TLS interception opt-in boundary:** `extensions/anthropic/interactive-proxy/wrapper.ts:135`
  - body: The wrapper sets `HTTPS_PROXY` and `NODE_EXTRA_CA_CERTS` for the spawned Claude CLI so OpenClaw can decrypt Anthropic API traffic. That capability needs an explicit maintainer-approved consent and opt-in model before release.
  - confidence: 0.95
- **[medium] Harden the generated CA lifecycle:** `extensions/anthropic/interactive-proxy/cert-manager.ts:60`
  - body: The cert manager persists a reusable CA and regenerates shared cert files without inter-process locking. Rotation, cleanup, permissions, and concurrent first-run behavior should be approved and hardened before exposing this backend.
  - confidence: 0.9
