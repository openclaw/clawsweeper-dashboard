## Summary

Keep this PR open: it is a recent draft feature PR, current main does not already provide the `claude-cli-interactive` backend, and the latest head still has blocking test, proof, and security issues that need contributor and maintainer follow-up.

## What This Changes

The PR adds an opt-in Anthropic `claude-cli-interactive` backend with a Bun wrapper and loopback TLS proxy to run Claude CLI in interactive mode and stream captured thinking events into OpenClaw.

## Best Possible Solution

Land an opt-in Anthropic plugin backend only after the test contract matches the implementation, overflow prompt storage is private, current-head live proof shows the reasoning stream, and maintainers accept the local TLS proxy trust model.

## Reproduction Assessment

Not applicable for a current-main bug: this is a new opt-in backend. The review findings are source-reproducible from the latest-head test call without inheritance context and the wrapper's temp-file write path for overflow prompts.

## Solution Assessment

No, not as-is. The plugin-owned backend boundary is plausible, but the patch needs the test fix, private overflow-file handling, current-head real behavior proof, and maintainer approval of the TLS interception design.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Protect overflow prompt files before writing them:** `extensions/anthropic/interactive-proxy/wrapper.ts:286-293`
  - body: When a prompt exceeds the argv cap, the wrapper writes the full prompt into `tmpdir()/openclaw-interactive-proxy` using default directory and file modes. On Unix-style temp directories this can leave sensitive prompt text in a predictable, non-private location, and the comment already acknowledges files may survive crashes; create a private 0700 directory and 0600/randomized file, or avoid writing prompt content to shared temp storage.
  - confidence: 0.91
- **[P2] Pass normalizeConfig the inheritance context in tests:** `extensions/anthropic/cli-backend-interactive.test.ts:11`
  - body: Latest head classifies an inherited Claude binary only when `context.config.agents.defaults.cliBackends["claude-cli"].command` matches the merged command, but these tests still call `backend.normalizeConfig!(merged)` without that context. The inherited-command tests will keep treating the fake Claude path as a direct Bun override instead of stashing it in `OPENCLAW_INTERACTIVE_CLAUDE_BINARY`.
  - confidence: 0.96

## Security Review

Status: needs_attention

Summary: The patch has a concrete prompt-file exposure bug and also introduces a TLS interception trust model that needs maintainer approval.

Concerns:

- **[high] Overflow prompts can be exposed through shared temp storage:** `extensions/anthropic/interactive-proxy/wrapper.ts:286`
  - body: Long prompts are written to `tmpdir()/openclaw-interactive-proxy` without explicit private directory or file modes, so sensitive prompt text can be exposed to other local users and may remain after abnormal termination.
  - confidence: 0.91
- **[medium] Local TLS interception needs explicit approval:** `extensions/anthropic/interactive-proxy/wrapper.ts:421`
  - body: The backend sets `HTTPS_PROXY` and `NODE_EXTRA_CA_CERTS` for the spawned Claude process so OpenClaw can terminate Anthropic API TLS locally and inspect plaintext SSE events. Even though the scope is child-local, this is a security/product decision rather than ordinary backend plumbing.
  - confidence: 0.93
- **[medium] Reusable local CA keys need accepted storage policy:** `extensions/anthropic/interactive-proxy/cert-manager.ts:147`
  - body: The cert manager generates and reuses local CA and leaf private keys under the user's OpenClaw directory for up to 365 days. The hardening path helps, but maintainers still need to accept the persisted local signing-key model for this backend.
  - confidence: 0.86
