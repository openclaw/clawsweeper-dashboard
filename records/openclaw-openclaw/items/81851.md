## Summary

Keep open: this draft PR is still not mergeable because latest head has two source-proven runtime blockers, lacks integrated real behavior proof, and introduces TLS interception that needs explicit maintainer security/product approval.

## What This Changes

The PR adds an experimental Anthropic `claude-cli-interactive` backend using a per-turn Bun wrapper with local TLS interception, reasoning JSONL reconstruction, config inheritance, static assets, tests, and related reasoning-stream wiring.

## Best Possible Solution

If maintainers accept the TLS-interception policy, keep the backend explicitly experimental/opt-in, fix the default binary and packaged asset paths, remove unrelated edits, add focused regression coverage, and require integrated OpenClaw plus Telegram proof.

## Reproduction Assessment

Yes for the blocking PR defects: source inspection of latest head shows the default no-override path stores `bun` as the Claude binary and the package metadata omits the required TTY preload. I did not establish a live Claude subscription or Telegram run.

## Solution Assessment

No. The proposed backend may be viable after maintainer approval, but this patch is not the best merge shape until the trust-boundary decision is made, runtime packaging/defaults are fixed, and integrated behavior is proven.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Keep the default backend pointed at Claude:** `extensions/anthropic/cli-backend-interactive.ts:139-147`
  - body: When users select `claude-cli-interactive` without a `claude-cli` override, `merged.command` is the backend's own `bun`. `normalizeConfig` stores that value in `OPENCLAW_INTERACTIVE_CLAUDE_BINARY`, and `wrapper.ts` prefers the env var over its `claude` fallback, so the wrapper tries to spawn Bun as the Claude binary and cannot run the backend.
  - confidence: 0.95
- **[P1] Ship the TTY preload with the proxy assets:** `extensions/anthropic/package.json:18-30`
  - body: `wrapper.ts` injects `interactive-proxy/tty-spoof.cjs` through `NODE_OPTIONS=--require`, but the package static asset list only ships the wrapper, MITM server, and cert manager. Packaged installs will miss the preload and the spawned Claude process will fail before reaching interactive mode.
  - confidence: 0.93

## Security Review

Status: needs_attention

Summary: The patch intentionally intercepts Anthropic TLS traffic for a spawned Claude child and persists a local CA/key pair, so it needs explicit maintainer security and consent review.

Concerns:

- **[medium] Review TLS interception and local CA consent:** `extensions/anthropic/interactive-proxy/wrapper.ts:130`
  - body: The wrapper routes the Claude child through a loopback HTTPS proxy with `NODE_EXTRA_CA_CERTS`, while `cert-manager.ts` generates and caches CA and leaf private keys under the user's OpenClaw directory. Even if scoped to the child process, this changes the trust boundary for Anthropic traffic and should not merge without a product/security decision and clear user consent model.
  - confidence: 0.95
