## What This Changes

The PR adds an opt-in Anthropic `claude-cli-interactive` backend with a Bun wrapper, local TLS proxy, JSONL thinking-stream reconstruction, provider/config wiring, tests, static assets, and separate doctor preview-warning changes.

## Best Possible Solution

Keep the backend plugin-owned and opt-in only if maintainers accept the TLS proxy approach, then fix error propagation, separate unrelated changes, and require current-head live Telegram/log proof plus focused cross-platform validation before merge.

## Reproduction Assessment

Not applicable for the requested feature on current main, but the blocking review finding is source-reproducible: the wrapper emits `is_error` JSONL and exits 0 while the non-live CLI runner treats exit-0 parsed output as success.

## Solution Assessment

No, not yet: a plugin-owned opt-in backend is the right boundary if maintainers accept the approach, but the current patch must preserve error semantics and clear the security/proof gates first.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Propagate proxy API errors as failed CLI runs:** `extensions/anthropic/interactive-proxy/wrapper.ts:369-370`
  - body: When `/v1/messages` returns 4xx/5xx, this branch emits an `is_error: true` result and then marks the kill as intentional, so the wrapper exits 0. The non-live CLI path only fails on nonzero exit/timeout and its JSONL parser drops `is_error`, so auth, billing, or rate-limit failures become normal assistant replies instead of failover/visible error handling. Exit nonzero for this synthetic error path or teach the non-live JSONL runner to fail on `is_error`.
  - confidence: 0.88

## Security Review

Status: needs_attention

Summary: The diff introduces a local TLS interception proxy with reusable local CA material, so it needs explicit security review before merge.

Concerns:

- **[medium] Review the local TLS interception trust boundary:** `extensions/anthropic/interactive-proxy/wrapper.ts:264`
  - body: The wrapper routes Claude traffic through a loopback proxy and trusts a generated CA via child-scoped `NODE_EXTRA_CA_CERTS`, decrypting provider traffic locally to reconstruct stream events. That is a security-model decision and should be explicitly accepted by maintainers before shipping.
  - confidence: 0.92
- **[medium] Require current-head CA key-permission proof:** `extensions/anthropic/interactive-proxy/cert-manager.ts:147`
  - body: The PR caches CA and leaf private keys under the user's OpenClaw config directory for a 365-day certificate lifetime. Current code attempts fail-closed hardening, but the posted artifact is from an older broad-permission run, so current-head proof across supported OSes is still needed.
  - confidence: 0.86
