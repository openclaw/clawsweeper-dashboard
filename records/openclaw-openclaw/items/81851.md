## Summary

Keep this draft PR open for maintainer review: it is a substantial new Anthropic backend with an intentional TLS-interception trust boundary, and the current patch still has source-level blockers in the integrated OpenClaw execution path.

## What This Changes

The PR adds an experimental Anthropic `claude-cli-interactive` CLI backend that runs Claude through a local TLS proxy to capture `thinking_delta` and `text_delta` SSE events.

## Best Possible Solution

If maintainers accept the product direction, revise the backend so integrated OpenClaw runs preserve the interactive Claude CLI dialect, keep wrapper-owned command fields insulated from inherited config, add focused regression coverage, and provide redacted real proof from the actual backend path.

## Reproduction Assessment

Yes, by source inspection: selecting `claude-cli-interactive` routes through the generic Claude live-session path that appends print-mode stream-json flags, and inherited `claude-cli` config can replace the wrapper command. I did not run a live Claude session in this read-only review.

## Solution Assessment

No. The overall direction may be useful, but this implementation needs maintainer security/product approval and a backend-specific execution path before it is the maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Keep interactive runs out of the print-mode arg builder:** `src/agents/cli-runner/claude-live-session.ts:119`
  - body: Adding `claude-cli-interactive` here sends the backend through `buildClaudeLiveArgs`, which still appends the stream-json input/output flags and other live-session flags before the wrapper launches `claude`. That contradicts this backend's stated subscription-mode design and the direct proof command, so integrated OpenClaw runs will not exercise the same interactive path the PR is trying to add. Split the interactive backend out of this generic arg builder or make the builder backend-aware before enabling this ID.
  - confidence: 0.9
- **[P2] Do not inherit the canonical backend command:** `src/agents/cli-backends.ts:228-234`
  - body: When no direct `claude-cli-interactive` override exists, this copies the entire inherited `claude-cli` config into the merge. Since `mergeBackendConfig` lets the override replace `command`, a normal custom `claude-cli.command` would replace the interactive backend's `bun` command while `filterArgs` still prepends the Bun wrapper path, launching the wrong executable instead of the proxy wrapper. Strip wrapper-owned fields from inherited config or restore the interactive command after filtering.
  - confidence: 0.87

## Security Review

Status: needs_attention

Summary: The diff intentionally adds local TLS interception for Anthropic traffic, so the trust boundary and consent surface need maintainer approval before merge.

Concerns:
