## Summary

Keep this draft PR open: current main does not implement the new Anthropic interactive backend, and the PR still has a concrete security blocker around reusable proxy key hardening plus insufficient visible proof for the Telegram reasoning behavior.

## What This Changes

The PR adds an opt-in `claude-cli-interactive` Anthropic backend with Bun loopback TLS proxy assets, thinking-delta parsing, reasoning-stream wiring, inherited CLI backend config support, tests, static asset packaging, and a Doctor warning change.

## Best Possible Solution

Keep the backend experimental and opt-in, fail closed on proxy key hardening, add visible Telegram or stdout JSONL proof for current head, and then route the TLS interception design through maintainer/security review.

## Reproduction Assessment

Yes for the review blocker: the PR-head source shows permission hardening failures are swallowed before `ensureCerts()` returns reusable proxy keys, and the supplied artifact shows broad key permissions. I did not reproduce a live Claude subscription or Telegram turn locally.

## Solution Assessment

No in its current form: an opt-in backend may be a viable direction, but the reusable MITM signing keys need fail-closed hardening and the larger TLS interception approach needs maintainer/security approval.

## Security Review
