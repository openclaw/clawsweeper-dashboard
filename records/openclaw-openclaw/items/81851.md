## Summary

Keep open: this is an active draft feature PR with meaningful unique work, but it is not merge-ready because the default launch path points the wrapper back at Bun, the integrated live-session path still re-adds print-mode JSON flags, contributor proof only covers the wrapper, and the TLS interception design needs explicit product/security approval.

## What This Changes

The PR adds an experimental Anthropic `claude-cli-interactive` backend that captures Claude CLI subscription-path SSE thinking/text events through a local TLS proxy and routes them into OpenClaw CLI/reasoning streams.

## Best Possible Solution

If maintainers accept the TLS-proxy direction, the PR should isolate the interactive backend from the print-mode live argv builder, fix the default Claude binary fallback, add regression coverage for default and inherited config paths, and include redacted integrated OpenClaw proof.

## Reproduction Assessment

Yes, by source inspection: the default config path sets the wrapper child binary to `bun`, and selecting `claude-cli-interactive` routes integrated runs through the generic live-session arg builder that re-adds print-mode JSON flags. I did not run a live Claude subscription session in this read-only review.

## Solution Assessment

No. The direction may be useful, but this implementation needs the launch-path bugs fixed, integrated real proof added, and explicit maintainer product/security approval before it is the maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve the Claude binary fallback:** `extensions/anthropic/cli-backend-interactive.ts:147`
  - body: When this backend has no inherited `claude-cli` override, `merged.command` is the backend's own `bun`. This line still writes that value to `OPENCLAW_INTERACTIVE_CLAUDE_BINARY`, so `wrapper.ts` spawns `bun` with Claude args instead of spawning `claude` and the default backend path never exercises the proxy. Only set the env var when a real inherited Claude binary is present, or let the wrapper use its `claude` fallback.
  - confidence: 0.93
- **[P2] Keep interactive runs out of print-mode argv:** `src/agents/cli-runner/claude-live-session.ts:119`
  - body: Adding the new ID to this gate reuses `buildClaudeLiveArgs`, which unconditionally appends `--input-format stream-json`, `--output-format stream-json`, `--permission-prompt-tool stdio`, and `--replay-user-messages`. Those are the print/stream-json stdin dialect flags the PR is trying to avoid for the subscription path, and they are absent from the wrapper proof command; give this backend its own live-session argv path before enabling it here.
  - confidence: 0.88
