## Summary

Keep open: this draft PR proposes a new Anthropic CLI backend and write-capable CI automation, but source review shows blocking build/integration issues, insufficient integrated real behavior proof, and security/product decisions around local TLS interception.

## What This Changes

The PR adds an experimental `claude-cli-interactive` Anthropic backend using a Bun local TLS proxy, plus a scheduled script/workflow to refresh Claude CLI model catalogs.

## Best Possible Solution

Split the catalog-refresh workflow into its own reviewed change; if maintainers approve the TLS proxy direction, wire the backend through manifest, auth, parser, and reasoning contracts with focused tests and integrated real behavior proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence failure path: the PR adds Bun-only files under the extension typecheck glob, registers a new provider id without the current Claude stream/reasoning contracts, and emits a JSONL shape the current parser ignores.

## Solution Assessment

No. The proposed direction may be useful, but a TLS MITM runtime backend is a security/product decision and the patch should first remove unrelated CI work, compile cleanly, and use or extend existing OpenClaw contracts explicitly.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Exclude or type the Bun-only proxy files:** `extensions/anthropic/interactive-proxy/mitm-server.ts:52`
  - body: `tsconfig.extensions.json` currently includes `extensions/**/*`, and this PR does not add the claimed exclusion or Bun type setup. The new proxy file references the global `Bun` object, so the extension typecheck path will fail before this backend can ship.
  - confidence: 0.9
- **[P1] Set the Claude stream dialect for the backend:** `extensions/anthropic/cli-backend-interactive.ts:70`
  - body: The new backend sets `output: "jsonl"` but not `jsonlDialect: "claude-stream-json"`, and its provider id is not exactly `claude-cli`. Current main gates Claude stream parsing on one of those two conditions, so normal Claude `result` and `text_delta` JSONL will not be parsed for this backend.
  - confidence: 0.91
- **[P1] Emit a parser-supported thinking event:** `extensions/anthropic/interactive-proxy/wrapper.ts:93-113`
  - body: The wrapper writes synthetic partial `assistant` records with a `thinking` content block, but the current CLI JSONL parser only emits deltas for `stream_event` / `content_block_delta` / `text_delta`. The tapped thinking content will be ignored by OpenClaw instead of reaching the reasoning lane.
  - confidence: 0.88
- **[P2] Check for wrapper.js before choosing it:** `extensions/anthropic/cli-backend-interactive.ts:23-27`
  - body: `fileURLToPath(new URL(...))` only converts the URL; it does not verify that `wrapper.js` exists. In source/Bun development mode this will choose a missing compiled wrapper instead of falling back to `wrapper.ts`.
  - confidence: 0.88
- **[P2] Publish the backend in plugin metadata:** `extensions/anthropic/register.runtime.ts:604`
  - body: The runtime registers `claude-cli-interactive`, but the Anthropic manifest still declares only `claude-cli` for CLI backend and synthetic auth metadata, and the provider still aliases/synthesizes auth only for `claude-cli`. Metadata-driven setup and auth discovery will miss this backend.
  - confidence: 0.84

## Security Review

Status: needs_attention

Summary: The diff adds local TLS interception and an unrelated write-capable scheduled workflow, both requiring explicit security/supply-chain review before merge.
