## Summary

Keep open: the feature is useful and has strong Windows live proof, but the current branch is not merge-ready because it drops current-main auth-profile CLI runtime routing, broadens Claude's native Read permission on overflow prompts, and leaves a static-asset test expectation inconsistent with the new package metadata. The opt-in local TLS interception design also needs explicit maintainer security/provider approval before merge.

## What This Changes

Adds an experimental opt-in Anthropic `claude-cli-interactive` backend using a Bun loopback TLS proxy, wrapper static assets, reasoning stream parsing, runtime/provider gate updates, focused tests, and a `bun-types` dev dependency.

## Best Possible Solution

Rebase on current main, preserve auth-profile CLI runtime routing, replace the overflow `Read` grant with a permission-preserving prompt delivery path or an explicitly approved narrow permission model, sync the static-asset tests, and get maintainer security/provider approval for the opt-in TLS interception design with cross-platform proof.

## Solution Assessment
