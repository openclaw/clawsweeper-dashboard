## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Spawn Claude instead of Bun by default:** `extensions/anthropic/cli-backend-interactive.ts:147`
  - body: With no inherited `claude-cli` override, `merged.command` is this backend's own `bun`, so this stores `OPENCLAW_INTERACTIVE_CLAUDE_BINARY=bun`. The wrapper then spawns Bun as the child process, and the default backend path never launches the Claude CLI traffic the proxy is meant to intercept.
  - confidence: 0.93
- **[P2] Mark proxy fallback results as errors:** `extensions/anthropic/interactive-proxy/wrapper.ts:226-228`
  - body: `runClaudeDirect` emits `subtype: "error"`, but the live-session handler only fails turns when `parsed.is_error === true`. If cert generation or proxy startup fails, this can be parsed as an empty successful result before the child exit is observed.
  - confidence: 0.88
- **[P2] Return a session id from proxy results:** `extensions/anthropic/interactive-proxy/wrapper.ts:166`
  - body: Fresh live-session starts strip `--session-id`, and the wrapper ignores Claude stdout, so `sessionId` is empty on new sessions and successful results omit `session_id`. When the warm wrapper closes, OpenClaw has no CLI session id to resume despite `sessionMode: "always"`.
  - confidence: 0.78
