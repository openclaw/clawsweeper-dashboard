- **[P1] Keep loadSessionEntry clone support:** `src/gateway/session-utils.ts:890-900`
  - body: The PR removes `clone` from `loadSessionEntry` and always calls `loadSessionStore(storePath)`, but the merge result still has gateway/session callers passing `{ clone: false }`. That will fail typechecking and also removes the borrowed-store semantics those hot paths requested, so restore the option and forward it to both target resolution and store loading.
  - confidence: 0.95
- **[P1] Keep scoped external CLI auth overlays:** `src/agents/command/attempt-execution.ts:177-179`
  - body: These store reads no longer pass `externalCliProfileIds` / `externalCliProviderIds`, but runtime-only CLI credentials are only overlaid when those scoped options reach the auth-profile store. An explicit Claude CLI auth profile or harness-selected external CLI provider can therefore disappear during selection, so keep the scoped overlay options on both paths.
  - confidence: 0.9
- **[P2] Preserve user-turn transcript hook writes:** `src/agents/command/attempt-execution.ts:232-243`
  - body: The PR bypasses `appendUserTurnTranscriptMessage` and writes the user prompt directly with `appendSessionTranscriptMessage`, while `persistCliTurnTranscript` still accepts `userMessage`. That drops the harness `beforeMessageWrite` hook and structured user-turn metadata for CLI transcripts, so restore the user-turn helper path instead of raw appends.
  - confidence: 0.88
- **[P2] Document the new CLI backend inheritance contract:** `src/plugins/cli-backend.types.ts:201-204`
  - body: `inheritUserConfigFrom` is added to `CliBackendPlugin`, which is exported through `openclaw/plugin-sdk/cli-backend`, but the CLI backend plugin docs and contract coverage do not describe the new hook. Plugin authors would get an undocumented public compatibility surface, so add the SDK/docs/tests that lock the intended behavior or keep the helper internal.
  - confidence: 0.82
