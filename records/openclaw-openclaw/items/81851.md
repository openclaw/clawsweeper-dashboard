- **[P1] Pin every child proxy alias to the local MITM:** `extensions/anthropic/interactive-proxy/wrapper.ts:488-495`
  - body: `claudeEnv` inherits the parent environment and only overwrites uppercase `HTTPS_PROXY`. Managed-proxy setups set lowercase aliases too, and some clients prefer lowercase or honor no-proxy bypasses, so Claude can connect through the operator proxy or bypass the local MITM instead of streaming through it. Set `http_proxy`, `https_proxy`, `HTTP_PROXY`, and `HTTPS_PROXY` to the loopback proxy and clear `no_proxy`/`NO_PROXY` for the spawned Claude process, with a regression test.
  - confidence: 0.92

In proxy-managed deployments this can bypass the operator's filtering/audit boundary. Route upstream traffic through the managed proxy/CONNECT helper or explicitly reject this backend under managed-proxy mode, and cover both paths in tests.
  - confidence: 0.9
