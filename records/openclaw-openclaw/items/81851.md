  - confidence: 0.98

  - body: The auth-store reads here used to pass `externalCliProfileIds`/`externalCliProviderIds`, which is how runtime-only external CLI profiles are hydrated for harness selection.   - confidence: 0.96

  - body: `persistTextTurnTranscript` now writes a raw user message with `appendSessionTranscriptMessage` instead of `appendUserTurnTranscriptMessage`, and `persistCliTurnTranscript` no longer forwards its `userMessage`. That bypasses the prepared user-turn message path and the `beforeMessageWrite` harness hook, so provenance and hook-mutated transcript fields can be lost.
  - confidence: 0.93
- **[P2] Document the new CLI backend inheritance contract:** `src/plugins/cli-backend.types.ts:201-204`

## Security Review

Status: needs_attention

Summary: The diff introduces a security-sensitive local TLS interception path and reusable CA key cache that needs explicit maintainer/security approval before merge.

- **[high] Local TLS interception trust model needs approval:** `extensions/anthropic/interactive-proxy/wrapper.ts:492`
  - body: The wrapper sets `HTTPS_PROXY` and `NODE_EXTRA_CA_CERTS` for the Claude child process, the proxy reads plaintext Anthropic SSE events, and `cert-manager` caches reusable CA key material.
