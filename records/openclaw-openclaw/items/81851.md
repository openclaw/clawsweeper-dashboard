## Summary

Keep open: this draft PR is not mergeable as written. The proposed backend is a new, security-sensitive Anthropic CLI transport, but the current parser and plugin metadata paths do not recognize the new backend id, the wrapper emits a stream shape OpenClaw ignores, source checkout path resolution is broken, and the branch also carries unrelated repo-write catalog automation.

## What This Changes

Adds an experimental `claude-cli-interactive` Anthropic CLI backend with a Bun-based local TLS proxy that taps Anthropic SSE thinking deltas, plus an unrelated scheduled model-catalog refresh workflow and script.

## Best Possible Solution

Integrate the backend through the existing Claude CLI parser, manifest, synthetic-auth, catalog, and reasoning-preview contracts with focused tests and security-reviewed proxy handling, while splitting catalog-refresh automation into its own owner-reviewed PR for https://github.com/openclaw/openclaw/issues/81216.

## Reproduction Assessment

Yes for the review blockers from source inspection: current main only treats `claude-cli` or `jsonlDialect: "claude-stream-json"` as Claude stream JSON, while this PR adds `claude-cli-interactive` without that dialect and emits ignored assistant-thinking partials. No live repro was run because this was a read-only PR review.

## Solution Assessment

No. The idea needs a cleaner, security-reviewed integration with the existing CLI parser/metadata/auth surfaces, and the unrelated catalog workflow should be split out before this backend can be evaluated on its own.

Full review comments:

- **[P1] Set the Claude JSONL dialect for this backend:** `extensions/anthropic/cli-backend-interactive.ts:72`
  - body: `claude-cli-interactive` is a new provider id, but the current parser only enables Claude stream-json parsing for `claude-cli` or for configs with `jsonlDialect: "claude-stream-json"`. Without that, the wrapper's Claude JSONL result and `content_block_delta` lines fall through as raw JSON text and no streaming assistant deltas are delivered.
  - confidence: 0.93
- **[P1] Emit parser-supported thinking deltas:** `extensions/anthropic/interactive-proxy/wrapper.ts:89-100`
  - body: The wrapper writes synthetic `type: "assistant"` partials containing `type: "thinking"` blocks, but the CLI streaming parser only calls `onAssistantDelta` for Claude `stream_event` `content_block_delta` `text_delta` records. These injected partials are ignored by the live parser, so the tapped thinking will not reach `onReasoningStream` even after the dialect is fixed.
  - confidence: 0.9
- **[P2] Declare the backend in plugin metadata:** `extensions/anthropic/cli-constants.ts:2`
  - body: Adding `CLAUDE_CLI_INTERACTIVE_BACKEND_ID` creates a selectable backend id, but the Anthropic plugin manifest still declares only `claude-cli`. Plugin owner resolution and setup fallback paths use manifest `cliBackends`, so a configured `claude-cli-interactive/...` run can fail to activate or discover the Anthropic backend before `resolveCliBackendConfig` looks for it.
  - confidence: 0.88
- **[P2] Remove unrelated catalog automation:** `.github/workflows/refresh-cli-runtime-catalogs.yml:12-14`
  - body: This PR is scoped to an experimental interactive Anthropic backend, but it also adds a scheduled workflow with repo write permissions and an Anthropic refresh secret. That broadens the CI/security surface and should be split into the existing catalog-refresh work tracked at https://github.com/openclaw/openclaw/issues/81216.
  - confidence: 0.94
- **[P2] Check wrapper file existence before choosing .js:** `extensions/anthropic/cli-backend-interactive.ts:23-25`
  - body: `fileURLToPath(new URL("./interactive-proxy/wrapper.js", selfUrl))` does not check whether the file exists, so the `try` branch always wins. In a source checkout where this PR adds only `wrapper.ts`, OpenClaw will invoke `bun .../wrapper.js` and fail before the fallback can run.
  - confidence: 0.91
