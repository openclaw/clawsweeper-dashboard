## Security Review

Status: needs_attention

Summary: The patch introduces a local TLS interception backend and one concrete prompt-log leak that need attention before merge.

Concerns:

- **[medium] Redact debug prompt logging:** `extensions/anthropic/interactive-proxy/wrapper.ts:121`
  - body: Debug mode logs the full Claude argv, including the prompt passed as a positional arg, to stderr/gateway logs. That can expose private user content or secrets in logs when the debug flag is enabled.
  - confidence: 0.92
- **[medium] Review TLS interception trust boundary:** `extensions/anthropic/interactive-proxy/wrapper.ts:202`
  - body: The wrapper forces Claude through a loopback HTTPS proxy and trusts a generated CA via `NODE_EXTRA_CA_CERTS`, decrypting Anthropic API traffic locally to recover thinking events. Even when opt-in and child-scoped, this needs explicit maintainer/security acceptance and user-facing documentation.
  - confidence: 0.9
- **[medium] Re-prove proxy key hardening on current head:** `extensions/anthropic/interactive-proxy/cert-manager.ts:164`
  - body: The posted certificate artifact shows broad private-key file permissions, while PR head later added hardening and fail-closed behavior. Current-head live proof should show repaired permissions or the fail-closed path before merge.
  - confidence: 0.86
