## Summary

Keep open: this is a recent draft feature PR with useful direction, but current main does not implement the new backend and the PR still has blocking proof, security, and source-review issues before it can merge.

## What This Changes

The branch adds an opt-in Anthropic `claude-cli-interactive` backend with a Bun loopback TLS proxy for Claude SSE thinking capture, core CLI thinking-stream plumbing, backend config inheritance, tests/static assets, and unrelated Doctor/Codex preview-warning changes.

## Best Possible Solution

Land the opt-in Anthropic backend only after prompt spill storage is private, tests match the inheritance contract, current-head live proof shows the reasoning stream, and maintainers accept the local TLS proxy trust model.

## Reproduction Assessment

Not applicable for a current-main bug: this is a new opt-in backend. The blocking findings are source-reproducible from the wrapper's temp-file write path and the normalizeConfig tests lacking inheritance context.

## Solution Assessment

No, not as-is. The plugin-owned backend boundary is plausible, but the patch needs the privacy fix, context-correct tests, current-head live proof, and maintainer approval for the TLS interception design.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Protect overflow prompt files before writing them:** `extensions/anthropic/interactive-proxy/wrapper.ts:286-293`
  - body: When a prompt spills past the argv cap, the wrapper creates a predictable directory under `tmpdir()` and writes the full prompt with default modes. On Unix shared temp locations this can leave sensitive prompt text readable by other local users, and the code already acknowledges crash paths can leave the file behind. Create a private 0700 directory and 0600 randomized file, or avoid writing prompt content to shared temp storage.
  - confidence: 0.92
- **[P2] Pass normalizeConfig the inheritance context in tests:** `extensions/anthropic/cli-backend-interactive.test.ts:11`
  - body: The implementation only classifies a non-`bun` command as inherited when `context.config.agents.defaults.cliBackends["claude-cli"].command` matches it, but these tests still call `normalizeConfig(merged)` without that context. The inherited-command assertions now exercise the direct Bun override path instead of the runtime inheritance contract.
  - confidence: 0.96

## Security Review

Status: needs_attention

Summary: The patch introduces a local TLS interception backend and still has a concrete prompt-file exposure bug.

Concerns:

- **[high] Overflow prompts can be exposed through shared temp storage:** `extensions/anthropic/interactive-proxy/wrapper.ts:286`
  - body: Long prompts are written to a predictable file under `tmpdir()/openclaw-interactive-proxy` without explicit private directory or file modes, so sensitive prompt text can be exposed to other local users and may remain after abnormal termination.
  - confidence: 0.92
- **[medium] Local TLS interception needs explicit approval:** `extensions/anthropic/interactive-proxy/wrapper.ts:421`
  - body: The backend sets `HTTPS_PROXY` and `NODE_EXTRA_CA_CERTS` for the spawned Claude process so OpenClaw can terminate Anthropic API TLS locally and inspect plaintext SSE events. Even though the scope is child-local, this is a security and product trust-model decision rather than ordinary backend plumbing.
  - confidence: 0.94
- **[medium] Persistent local CA keys need accepted storage policy:** `extensions/anthropic/interactive-proxy/cert-manager.ts:147`
  - body: The cert manager generates and reuses local CA and leaf private keys under the user's OpenClaw directory for up to 365 days. The hardening path helps, but maintainers still need to accept the persisted local signing-key model for this backend.
  - confidence: 0.86
