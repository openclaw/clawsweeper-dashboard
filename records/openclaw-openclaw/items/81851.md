  - body: The branch narrows `loadSessionEntry` to only accept `{ agentId?: string }`, but current branch callers still pass `{ clone: false }` in gateway session hot paths. This will fail typechecking and also drops the no-clone cached-store behavior those paths deliberately requested.

Without `externalCliProfileIds` and `externalCliProviderIds`, runtime-only Claude/Codex CLI auth profiles are only overlaid when they already exist in the store, so selected harness auth can fall back to the wrong provider or appear missing.
  - confidence: 0.93

  - body: The new backend does not set `modelProvider: "anthropic"`, but the runtime-alias registry only binds CLI runtimes that declare a canonical model provider. As a result, the documented `anthropic/...` model with `agentRuntime.id: "claude-cli-interactive"` path will not resolve to this backend.
  - confidence: 0.9
- **[P2] Preserve user-turn transcript hook writes:** `src/agents/command/attempt-execution.ts:232-243`
