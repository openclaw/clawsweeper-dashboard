## Best Possible Solution

Land this only if maintainers approve an explicit opt-in TLS interception design and the branch preserves Claude CLI launch, error, and session semantics with focused regression coverage and integrated proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Avoid passing bun as the Claude child binary:** `extensions/anthropic/cli-backend-interactive.ts:147`
  - body: With no inherited `claude-cli` override, `merged.command` is the backend's own `bun`, so `OPENCLAW_INTERACTIVE_CLAUDE_BINARY` makes the wrapper spawn Bun instead of Claude. The default backend path then never starts the Claude process this proxy is meant to intercept.
  - confidence: 0.93
- **[P2] Keep interactive runs out of stream-json live args:** `src/agents/cli-runner/claude-live-session.ts:119`
  - body: Adding `claude-cli-interactive` to `shouldUseClaudeLiveSession` sends it through `buildClaudeLiveArgs`, which appends stream-json/stdio flags and writes JSON user-message records to stdin. That contradicts the no-`-p` interactive launch and can feed protocol JSON to the TUI path.
  - confidence: 0.91
- **[P2] Emit live-session errors with is_error:** `extensions/anthropic/interactive-proxy/wrapper.ts:192-197`
  - body: `runClaudeDirect` emits `subtype: "error"`, but the live-session result handler fails turns only when `parsed.is_error === true`. If cert generation or proxy startup fails, this can resolve the turn as an empty successful result before the child exit is processed.
  - confidence: 0.87
- **[P2] Preserve a session id in proxy results:** `extensions/anthropic/interactive-proxy/wrapper.ts:134`
  - body: Fresh live-session launches strip `--session-id`, and the wrapper ignores Claude stdout, so `sessionId` is empty and successful first turns return no `session_id`. That breaks resume after the warm wrapper process closes despite `sessionMode: "always"`.
  - confidence: 0.8
