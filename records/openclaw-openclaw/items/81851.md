  - confidence: 0.93

  - confidence: 0.88
- **[P1] Do not publish backend inheritance accidentally:** `src/plugins/cli-backend.types.ts:201-204`
  - body: `CliBackendPlugin` is plugin SDK surface, so adding `inheritUserConfigFrom` makes sibling-backend config inheritance a public third-party contract. ## Security Review

Status: needs_attention

Summary: The diff introduces a security-sensitive local MITM proxy and child-process preload path, and it also changes auth-provider selection behavior.

Concerns:

- **[high] Local MITM proxy changes provider trust boundary:** `extensions/anthropic/interactive-proxy/wrapper.ts:492`
  - body: The wrapper launches Claude with a loopback `HTTPS_PROXY`, generated CA via `NODE_EXTRA_CA_CERTS`, and `NODE_OPTIONS` preload, so token-bearing Anthropic traffic is intentionally intercepted by OpenClaw code before reaching the upstream API.
  - confidence: 0.92
- **[medium] Persisted signing keys need maintainer approval:** `extensions/anthropic/interactive-proxy/cert-manager.ts:28`
  - body: The cert manager stores reusable CA and leaf private keys under the OpenClaw state directory; even with permission hardening, this is a new local credential/trust-store surface that should be accepted explicitly.
  - confidence: 0.88

  - body: GitHub Advanced Security flagged the new `net.connect` tunnel callsite for classification; the PR should document or gate whether this raw egress is managed/proxied, local-only, diagnostic guarded, or unsupported.
  - confidence: 0.82
