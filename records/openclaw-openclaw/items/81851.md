Full review comments:

- **[P1] Pin every proxy alias to the local MITM:** `extensions/anthropic/interactive-proxy/wrapper.ts:492-494`
  - body: `claudeEnv` inherits the parent environment and only overwrites uppercase `HTTPS_PROXY`. OpenClaw's own env-proxy contract mirrors common client behavior where lower-case `https_proxy` can win and `NO_PROXY` can bypass, so the spawned Claude process can skip this local proxy and the backend silently loses SSE/reasoning capture. Set every proxy alias that Claude may honor to the local proxy and clear both bypass aliases for the child process.
  - confidence: 0.9
- **[P1] Keep pass-through CONNECTs on the managed proxy path:** `extensions/anthropic/interactive-proxy/mitm-server.ts:343-347`
  - body: For allowed non-API Anthropic hosts, the new CONNECT handler opens `net.connect(destPort, destHost)` directly. In managed-proxy deployments that bypasses the operator filtering/audit boundary that OpenClaw documents for raw sockets, so these tunnels need to go through the managed proxy path or fail closed until maintainers explicitly approve a direct bypass.
  - confidence: 0.91
- **[P1] Remove or authorize the dependency graph change:** `extensions/anthropic/package.json:7-9`
  - body: This external PR adds `bun-types` to the Anthropic plugin manifest and updates the lockfile, and the dependency-graph guard reports that dependency graph changes are blocked unless secops authorizes the current head.
