## Summary

Keep this draft PR open: the feature is not on current main, but the current head still has merge-blocking correctness issues, insufficient real behavior proof for the claimed streaming behavior, and a security-sensitive TLS interception design that needs maintainer review.

## What This Changes

Adds an experimental Anthropic `claude-cli-interactive` backend that runs Claude through a Bun wrapper and loopback TLS proxy, emits Claude stream-json events, and bridges captured thinking into the reasoning preview.

## Best Possible Solution

Keep the PR as draft until the title-response suppression and lockfile mismatch are fixed, then require redacted live JSONL plus Telegram proof and an explicit maintainer decision on the proxy trust model.

## Reproduction Assessment

Yes for the review findings: source inspection of current PR head shows exact short `{"title":"..."}` replies still match the suppression path, and `package.json` disagrees with `pnpm-lock.yaml` for `bun-types`. I did not run a live Claude subscription or Telegram flow.

## Solution Assessment

No in its current shape: an opt-in backend may be the right direction, but this patch needs correctness fixes, synchronized dependency metadata, stronger real behavior proof, and maintainer acceptance of the local TLS proxy design before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Do not drop legitimate title JSON replies:** `extensions/anthropic/interactive-proxy/wrapper.ts:246-247`
  - body: The tightened auxiliary-response guard still suppresses a real user reply that is exactly a short JSON object like `{"title":"Status"}`. In that case `isAuxiliaryResponse()` returns true, the message-stop handler clears the pending events and returns before emitting `result`, so the user gets no reply and the turn waits for the watchdog instead.
  - confidence: 0.9
- **[P2] Regenerate the lockfile after pinning bun-types:** `pnpm-lock.yaml:235`
  - body: `package.json` now pins `bun-types` to `1.3.14`, but the lockfile importer still records the requested specifier as `^1.3.14`. That leaves the dependency metadata out of sync and will fail frozen-lockfile install checks; regenerate or edit the lockfile so the importer specifier matches the manifest.
  - confidence: 0.94

## Security Review

Status: needs_attention

Summary: The diff introduces a local TLS interception path and persisted proxy key material, so the security model needs explicit review before merge.

Concerns:

- **[medium] Review TLS interception trust boundary:** `extensions/anthropic/interactive-proxy/wrapper.ts:155`
  - body: The wrapper sets `HTTPS_PROXY` and `NODE_EXTRA_CA_CERTS` for the Claude child so OpenClaw can terminate and inspect Anthropic traffic locally. Even scoped to the child, that changes the provider traffic trust boundary and should have explicit maintainer/security acceptance and user-consent semantics.
  - confidence: 0.93
- **[medium] Verify proxy private-key ACLs on Windows:** `extensions/anthropic/interactive-proxy/cert-manager.ts:191`
  - body: The cert manager persists CA and leaf private keys and applies Windows ACL tightening on a best-effort basis; the linked proof listing still shows readable key files, so the final design should verify platform-specific ACL behavior before shipping.
  - confidence: 0.78
- **[low] Keep dependency metadata exact:** `pnpm-lock.yaml:235`
  - body: The lockfile still records `bun-types` with a caret specifier even though the manifest was pinned, leaving package resolution metadata inconsistent for the new Bun type surface.
  - confidence: 0.86
