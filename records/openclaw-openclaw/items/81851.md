## Summary

Keep open: current main does not implement this backend, and the current PR head still needs correctness fixes, inspectable live proof, and maintainer acceptance of the TLS-interception trust boundary before merge.

## What This Changes

The PR adds an experimental Anthropic `claude-cli-interactive` backend that runs Claude Code through a Bun loopback TLS proxy to reconstruct stream-json events and surface interactive-mode reasoning deltas.

## Best Possible Solution

Keep the backend opt-in and experimental only if maintainers accept the TLS-proxy model, then fix the streaming and inherited-flag defects, tighten dependency pinning or justify the range, and require inspectable live Telegram or CLI proof before merge.

## Reproduction Assessment

Yes for the blocking review findings: source inspection of the current PR head shows thinking events are buffered until text or end-of-turn, and inherited `--print` is not stripped. I did not reproduce the full live Claude subscription plus Telegram flow.

## Solution Assessment

No in its current shape: a separate opt-in backend may be maintainable, but this implementation needs correctness fixes, contributor-supplied real behavior proof, and a maintainer trust-boundary decision before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Stream thinking events before title filtering:** `extensions/anthropic/interactive-proxy/wrapper.ts:179-180`
  - body: The wrapper starts buffering on `message_start` and sends every event through `emitOrBuffer`, but buffering is only cleared once a later `text_delta` proves the response is not title JSON. Any preceding `thinking_delta` events stay in `pendingEvents` until assistant text or `message_stop`, so the new backend does not stream live reasoning for the path it is meant to add.
  - confidence: 0.9
- **[P2] Strip the long print-mode flag:** `extensions/anthropic/cli-backend-interactive.ts:24-25`
  - body: Inherited `claude-cli` overrides can use `--print`, which Claude documents as the long form of noninteractive mode. The filter removes `-p` but leaves `--print`, so those users would still spawn Claude outside interactive mode and bypass the backend's subscription/proxy behavior. ([docs.claude.com](https://docs.claude.com/en/docs/claude-code/sdk/sdk-headless?utm_source=openai))
  - confidence: 0.88

## Security Review

Status: needs_attention
