Full review comments:

- **[P1] Pin proxy aliases to the local MITM:** `extensions/anthropic/interactive-proxy/wrapper.ts:493-495`
  - body: `claudeEnv` spreads the inherited environment and then sets only uppercase `HTTPS_PROXY`. Common proxy env parsers give lowercase variables precedence and honor `NO_PROXY`, so an existing `https_proxy`, `ALL_PROXY`, or `NO_PROXY=api.anthropic.com` can make Claude skip this local proxy and silently lose SSE capture. Set/clear the full proxy alias set for the child and add a regression test with conflicting lowercase/no-proxy values.
  - confidence: 0.89
- **[P1] Route MITM upstream through managed proxy policy:** `extensions/anthropic/interactive-proxy/mitm-server.ts:210-217`
  - body: The Bun child forwards the decrypted Anthropic request with a direct `fetch`/socket path, outside the Node Proxyline hooks documented for `proxy.enabled=true`. In managed-proxy deployments this can send API traffic directly to `api.anthropic.com` instead of the operator egress proxy; use the managed proxy/CONNECT contract or fail closed when managed proxy mode is active, with focused proof.
  - confidence: 0.86
