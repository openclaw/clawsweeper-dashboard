## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Keep large prompts out of argv:** `extensions/anthropic/cli-backend-interactive.ts:175`
  - body: `resolvePromptInput` sends the full prompt as a command-line argument whenever `input: "arg"` is active and `maxPromptArgChars` is unset. This backend explicitly clears `maxPromptArgChars`, so long OpenClaw history prompts are always appended to `bun ... wrapper.ts` argv; on Windows and other hosts this can exceed command-line limits before Claude starts. Keep a bounded argv path and add a stdin/file handoff that the wrapper can safely pass to interactive Claude.
  - confidence: 0.86
- **[P2] Preserve inherited Claude command overrides:** `extensions/anthropic/cli-backend-interactive.ts:155-162`
  - body: The inherited `claude-cli.command` is treated as the Claude binary only when its basename is exactly `claude`, `claude.exe`, or `claude.cmd`. Documented CLI backend command overrides can point at a wrapper or symlink with a different name, and those will now be misclassified as the Bun command and invoked with `wrapper.ts` as an argument. Track whether the value came from inherited `claude-cli` config instead of guessing from the basename.
  - confidence: 0.79

## Security Review

Status: needs_attention

Summary: The PR introduces local TLS interception and reusable local CA material, so it needs explicit maintainer security review before merge.

Concerns:

- **[medium] Review the local TLS interception trust boundary:** `extensions/anthropic/interactive-proxy/wrapper.ts:271`
  - body: The wrapper launches Claude with child-scoped `HTTPS_PROXY` and `NODE_EXTRA_CA_CERTS`, and the Bun proxy terminates TLS for Anthropic API traffic to read plaintext SSE events. That is a security-model decision, not a routine backend detail.
  - confidence: 0.93
- **[medium] Require current-head proof for reusable proxy keys:** `extensions/anthropic/interactive-proxy/cert-manager.ts:147`
  - body: The PR stores a 365-day local CA and leaf private key under `~/.openclaw/proxy-certs`. The latest code attempts fail-closed key hardening, but the posted cert artifact predates those fixes and shows broadly readable key files, so current-head proof on supported OSes is still needed.
  - confidence: 0.86
