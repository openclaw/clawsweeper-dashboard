## Security Review

Status: needs_attention

Summary: The diff intentionally decrypts Anthropic traffic through a generated local CA, so it needs explicit maintainer security/product approval before merge.

Concerns:

- **[medium] Approve the TLS interception opt-in boundary:** `extensions/anthropic/interactive-proxy/wrapper.ts:135`
  - body: The wrapper sets `HTTPS_PROXY` and `NODE_EXTRA_CA_CERTS` for the spawned Claude CLI so OpenClaw can decrypt Anthropic API traffic. Even though it is loopback and child-scoped, this capability needs an explicit maintainer-approved consent and opt-in model before release.
  - confidence: 0.95
- **[medium] Approve the persisted local CA lifecycle:** `extensions/anthropic/interactive-proxy/cert-manager.ts:8`
  - body: The cert manager stores a reusable CA under `~/.openclaw/proxy-certs/` for 365 days. The rotation, cleanup, permission, and user-consent lifecycle should be approved as a product/security boundary before exposing this backend.
  - confidence: 0.88
