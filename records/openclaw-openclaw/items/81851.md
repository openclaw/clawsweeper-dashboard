Full review comments:

- **[P1] Preserve resumed system prompts for always backends:** `src/agents/cli-runner/execute.ts:290`
  - body: The PR now writes the system prompt file only when `!useResume`, and `buildCliArgs` also skips file/config system-prompt args on resumed runs. Current `claude-cli` intentionally uses `systemPromptWhen: "always"`, and the regression test asserts resumed turns keep `--append-system-prompt-file`, so this would re-break existing Claude CLI sessions.
  - confidence: 0.94
- **[P1] Keep loadSessionEntry clone support:** `src/gateway/session-utils.ts:890-900`
  - body: This narrows `loadSessionEntry` to `{ agentId?: string }` and always calls `loadSessionStore(storePath)`, but PR-head gateway callers still pass `{ clone: false }`. That will fail typechecking and also removes the borrowed-store semantics those hot paths requested, so keep the option and forward it to target resolution and store loading.
  - confidence: 0.95
- **[P1] Keep scoped external CLI auth overlays:** `src/agents/command/attempt-execution.ts:131-133`
  - body: These auth store reads no longer pass `externalCliProfileIds` or `externalCliProviderIds`, but runtime-only CLI credentials are only overlaid when those scoped options reach the auth-profile store. Explicit Claude CLI auth profiles or harness-selected external CLI providers can therefore disappear during selection.
  - confidence: 0.9
- **[P2] Preserve user-turn transcript hook writes:** `src/agents/command/attempt-execution.ts:233-243`
  - body: The PR bypasses `appendUserTurnTranscriptMessage` and writes the user prompt directly with `appendSessionTranscriptMessage`, while current main uses the user-turn helper with `runAgentHarnessBeforeMessageWriteHook`. That drops structured user-turn metadata and harness hook handling for CLI transcripts.
  - confidence: 0.88
- **[P2] Document the new CLI backend inheritance contract:** `src/plugins/cli-backend.types.ts:201-204`
  - body: `inheritUserConfigFrom` is added to `CliBackendPlugin`, which is exported through `openclaw/plugin-sdk/cli-backend`, but the CLI backend plugin docs and contract coverage do not describe the new hook. Add the SDK/docs/tests that lock the intended behavior or keep this inheritance mechanism internal.
  - confidence: 0.82

## Security Review

Status: needs_attention

- **[high] Approve the local TLS interception trust model:** `extensions/anthropic/interactive-proxy/mitm-server.ts:6`
