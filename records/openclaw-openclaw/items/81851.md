## Summary

Keep this PR open for maintainer review: current main does not implement the new backend, but the PR is draft/conflicting and still has blocking correctness, proof, and security-trust issues.

## What This Changes

Adds an opt-in Anthropic `claude-cli-interactive` backend that runs Claude Code through a Bun loopback TLS proxy and TTY spoof, taps SSE thinking/text deltas, registers the backend/auth alias, and bridges thinking events into CLI reasoning output.

## Best Possible Solution

Keep the backend opt-in only if maintainers accept the TLS-proxy model, then fix streaming/filtering defects, rebase the branch, and require inspectable Telegram/proxy proof before merge.

## Reproduction Assessment

Yes for the blocking review findings: source inspection shows the wrapper buffers pre-text stream events and the inherited-arg filter omits `--print`. I did not reproduce the full live Claude subscription plus Telegram flow.

## Solution Assessment

No in its current shape: a separate opt-in backend may be a workable direction, but this implementation needs correctness fixes, real behavior proof, conflict resolution, and a maintainer trust-boundary decision first.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Stream thinking events before title filtering:** `extensions/anthropic/interactive-proxy/wrapper.ts:172-180`
  - body: The wrapper starts buffering at `message_start` and only flushes once a later text delta proves the response is not title JSON, so any preceding `thinking_delta` events remain queued until assistant text or `message_stop`. That defeats the live reasoning stream this backend is meant to provide for extended-thinking turns.
  - confidence: 0.9
- **[P2] Strip the long print-mode flag:** `extensions/anthropic/cli-backend-interactive.ts:24-25`
  - body: Inherited `claude-cli` overrides can use `--print`, which Claude documents as the long form of noninteractive print mode, but the filter only removes `-p`. Users with that override would still launch Claude outside interactive mode and bypass the backend's subscription/proxy behavior.
  - confidence: 0.88

## Security Review

Status: needs_attention

Summary: The PR introduces an opt-in TLS interception path and a non-exact root devDependency range that need maintainer security and supply-chain review.

Concerns:

- **[medium] Review TLS interception and local CA consent:** `extensions/anthropic/interactive-proxy/wrapper.ts:130`
  - body: The wrapper routes the spawned Claude child through `HTTPS_PROXY` and `NODE_EXTRA_CA_CERTS`, while `cert-manager.ts` persists CA and leaf private keys under the user's OpenClaw directory. Even scoped to one child process, this changes the trust boundary for Anthropic traffic and needs explicit maintainer approval and user-facing consent semantics.
  - confidence: 0.94
- **[low] Pin the new Bun type dependency exactly:** `package.json:1816`
  - body: The PR adds `bun-types` to root `devDependencies` with a caret range while adjacent root development dependencies are exact-pinned, broadening future dependency resolution for a root package surface.
