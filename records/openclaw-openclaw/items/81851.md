Full review comments:

- **[P1] Route pass-through CONNECTs through managed egress:** `extensions/anthropic/interactive-proxy/mitm-server.ts:347`
  - body: For allowed non-API Anthropic hosts, this branch opens a raw `net.connect` directly from the proxy. In managed-proxy deployments that bypasses the operator egress/audit path the network proxy docs require raw socket callsites to preserve or explicitly classify, so route pass-through tunnels through the managed proxy helper or fail closed when that path is required.
  - confidence: 0.86
- **[P1] Override every proxy env variant for Claude:** `extensions/anthropic/interactive-proxy/wrapper.ts:488-492`
  - body: The child env inherits all existing proxy variables and only overwrites uppercase `HTTPS_PROXY`. A lowercase proxy, `ALL_PROXY`, or `NO_PROXY` inherited by the spawned Claude process can send traffic around the local proxy or bypass it entirely, which breaks the SSE capture this backend depends on; clear/set the full proxy and bypass family consistently for the child.
  - confidence: 0.8
- **[P1] Preserve answer-lane previews while bridging reasoning:** `src/auto-reply/reply/agent-runner-execution.ts:2075-2079`
  - body: This return suppresses every assistant stream chunk for Claude-CLI-compatible backends until a thinking event arrives. Redacted-thinking or no-thinking turns still need the assistant stream as the persistent answer preview, so Telegram can lose live answer updates and only see the final result at the end; dedupe reasoning without dropping the answer lane.
  - confidence: 0.82
- **[P1] Remove or authorize the dependency graph change:** `extensions/anthropic/package.json:9`
  - body: This external PR adds `bun-types` and updates the lockfile, and the repository dependency guard explicitly blocks dependency graph changes until secops authorizes the exact head SHA. Either avoid the new graph delta or get the security authorization before this can merge.
