## Summary

Keep open: this is an active draft feature PR with a security-sensitive local TLS proxy, and current head still has a concrete proxy-key hardening issue plus insufficient current-head real behavior proof.

## What This Changes

Adds an opt-in Anthropic `claude-cli-interactive` backend that runs Claude through a Bun loopback TLS proxy, maps captured `thinking_delta` JSONL into reasoning streams, registers the backend, and adds tests/static assets plus a Doctor warning change.

## Best Possible Solution

Keep the backend experimental and opt-in, but require fail-closed proxy-key handling, current-head live JSONL/Telegram proof, and explicit maintainer security approval before merge.

## Reproduction Assessment

Yes for the review blocker: PR-head source catches key-hardening failures and still returns proxy keys, and the supplied artifact shows a real world-readable key state. I did not reproduce a live Claude subscription or Telegram turn locally.

## Solution Assessment

No in its current form: the opt-in backend may be a maintainable direction, but proxy-key hardening must be verified or fail closed before cached signing keys are trusted.

## Review Findings

Overall correctness: patch is incorrect

- **[P2] Reject keys when permission hardening fails:** `extensions/anthropic/interactive-proxy/cert-manager.ts:130-132`

## Security Review

Status: needs_attention

Summary: The diff introduces a child-scoped local TLS interception backend and still accepts proxy keys if permission hardening fails.

Concerns:

- **[medium] Fail closed on proxy key hardening failures:** `extensions/anthropic/interactive-proxy/cert-manager.ts:130`
  - body: The certificate manager ignores failures from the permission-hardening step and still returns reusable CA/leaf private keys. For a local MITM signing key, a failed repair should prevent reuse rather than preserving broader ACLs.
  - confidence: 0.86
- **[medium] Review the TLS interception trust boundary:** `extensions/anthropic/interactive-proxy/wrapper.ts:122`
  - body: The wrapper forces the Claude child through a loopback HTTPS proxy and adds a generated CA via `NODE_EXTRA_CA_CERTS`, decrypting Anthropic API traffic locally to extract thinking events. Even child-scoped and opt-in, this needs explicit maintainer/security acceptance and documentation.
  - confidence: 0.9
