## Summary

Keep open: this is a useful, time-sensitive Anthropic backend with supplied Windows live proof, but the current diff still has source-backed merge blockers around MITM key isolation and long-prompt handling, plus an explicit security/provider-boundary decision before merge.

## What This Changes

Adds an opt-in Anthropic `claude-cli-interactive` backend that runs Claude Code through a Bun loopback TLS proxy, reconstructs stream-json events, extends Claude CLI routing/auth gates, packages proxy assets, and adds focused tests.

## Best Possible Solution

Keep the backend opt-in, but store proxy key material under the configured OpenClaw state root, make long-prompt delivery deterministic without unattended permission prompts, and require maintainer acceptance of the local TLS interception boundary before merge.

## Reproduction Assessment

Yes for the review blockers by source inspection. A custom `OPENCLAW_STATE_DIR` still leaves proxy keys at `homedir()/.openclaw`, and over-cap prompts spill to stdin before the wrapper asks Claude to use a `Read` tool that is not default-allowed by this backend.

## Solution Assessment

No. The overall direction may be valuable, but this implementation needs state-root-correct key storage, deterministic long-prompt behavior, and an explicit maintainer security decision before it is the best merge path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Store proxy keys under the configured state dir:** `extensions/anthropic/interactive-proxy/cert-manager.ts:9`
  - body: `CERT_DIR` is hard-coded to `homedir()/.openclaw/proxy-certs`, but OpenClaw supports `OPENCLAW_STATE_DIR` for isolated gateway state and treats that tree as secret-bearing. This writes reusable MITM CA/private keys outside the configured state root, so multi-instance/profile installs can share keys and cleanup/rotation will not follow the selected state dir. Please derive this path from OpenClaw's state-dir contract or pass the resolved state root into the wrapper and cover the override.
  - confidence: 0.93
- **[P2] Avoid recovering prompts through an unapproved Read call:** `extensions/anthropic/interactive-proxy/wrapper.ts:324-334`
  - body: When a prompt exceeds `maxPromptArgChars`, current main sends it on stdin; this wrapper writes it to a temp file and tells Claude to read it, but the backend only auto-allows `mcp__openclaw__*`. Without an inherited `bypassPermissions` override, an unattended interactive run can stop on a `Read` permission prompt instead of processing the turn. Please either fail fast before spawning, preapprove only the generated file path if Claude supports that safely, or use a prompt channel that does not require a tool permission.
  - confidence: 0.88
