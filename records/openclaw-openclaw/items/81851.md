## Summary

Keep this PR open for review: it is an active draft feature PR with a useful Anthropic plugin direction, but the branch is not merge-ready because the new backend is not fully wired through the model/runtime selection contracts, the TLS interception boundary still needs human security approval, and the supplied live proof predates the latest head.

## What This Changes

The PR adds an opt-in Anthropic `claude-cli-interactive` backend that runs Claude Code through a Bun wrapper and loopback TLS proxy to reconstruct Claude stream-json output from intercepted SSE events.

## Best Possible Solution

Wire `claude-cli-interactive` through the canonical runtime/model selection contracts with regression tests, split unrelated doctor/changelog work, attach current-head live Telegram/log proof, and require explicit security approval for the opt-in TLS proxy before merge.

## Reproduction Assessment

Not applicable as a current-main bug; this is a new backend feature. The blocking wiring issue is source-reproducible from PR head plus current main: runtime aliasing only recognizes `claude-cli`, and forward-compatible model cloning only preserves that provider.

## Solution Assessment

No as currently implemented. The Anthropic plugin is the right ownership boundary, but the branch needs complete runtime/model wiring, current-head proof, and explicit approval for the TLS interception design before it is the maintainable solution.
