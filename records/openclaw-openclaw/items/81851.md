## What This Changes

The PR adds an experimental Anthropic `claude-cli-interactive` backend that runs Claude CLI through a Bun local TLS proxy, captures `thinking_delta` SSE chunks, and routes them into OpenClaw reasoning streams.

## Best Possible Solution

If maintainers accept the TLS-interception direction, land it as an explicitly opt-in experimental backend with documented CA lifecycle, focused parser/bridge coverage, and redacted integrated OpenClaw plus Telegram proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Serialize cert generation before sharing proxy files:** `extensions/anthropic/interactive-proxy/cert-manager.ts:82-83`
  - body: `ensureCerts` uses one shared `~/.openclaw/proxy-certs` path and deletes/regenerates the cert/key files without an inter-process lock or temp-file rename. Two first-run or expired sessions can both enter this block and leave the CA/key/leaf set mismatched or missing, which breaks the advertised concurrent-session safety.
  - confidence: 0.89
- **[P2] Preserve JSONL output on proxy fallback:** `extensions/anthropic/interactive-proxy/wrapper.ts:160`
  - body: When cert generation or proxy startup fails, the wrapper falls back to `claude` with the interactive backend's args. Those args no longer include the normal `-p` stream-json path, and stdout is inherited, so OpenClaw's JSONL parser can receive TUI output or no `result` record instead of a structured fallback.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The diff intentionally decrypts Claude CLI Anthropic traffic through a local proxy and persists a CA, so it needs maintainer security/product approval.

Concerns:

- **[medium] Approve the TLS interception consent boundary:** `extensions/anthropic/interactive-proxy/wrapper.ts:105`
  - body: The wrapper sets `HTTPS_PROXY` and `NODE_EXTRA_CA_CERTS` for the spawned Claude CLI so OpenClaw can decrypt Anthropic API traffic. Maintainers should approve the opt-in and consent model before this ships.
  - confidence: 0.94

  - body: The cert manager persists a reusable CA under the user's `.openclaw/proxy-certs` directory for 365 days. Rotation, cleanup, platform permissions, and user expectations should be explicitly approved for this backend.
  - confidence: 0.88
