Keep this PR open: the opt-in backend has useful Windows live proof, but the current patch still has two source-backed merge blockers around proxy key placement and overflow prompt handling, plus an explicit security/provider decision for the local TLS MITM boundary.

## What This Changes

The PR adds an opt-in Anthropic `claude-cli-interactive` backend that runs Claude through a Bun loopback TLS proxy, reconstructs Claude stream-json events, extends Claude CLI routing/auth gates, adds static proxy assets, tests, and a `bun-types` dev dependency.

## Best Possible Solution

Keep the backend opt-in, repair the state-dir and overflow prompt issues, then require explicit maintainer security/provider approval plus packaged cross-platform proof before merging the TLS proxy design.

## Reproduction Assessment

Yes for the PR-introduced defects: source inspection shows cert keys hard-code `homedir()/.openclaw`, while current OpenClaw supports `OPENCLAW_STATE_DIR`, and long-prompt recovery asks Claude to read a temp file while only `mcp__openclaw__*` is auto-allowed. The feature itself has supplied Windows live logs, not a current-main bug reproduction.

## Solution Assessment

No. The backend direction is plausible, but this is not merge-ready until key storage follows the state-dir contract, overflow prompt recovery cannot stall on permissions, and maintainers explicitly accept the local TLS MITM/key lifecycle.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Store proxy keys under the configured state directory:** `extensions/anthropic/interactive-proxy/cert-manager.ts:9`
  - body: `CERT_DIR` hard-codes `homedir()/.openclaw/proxy-certs`, so an operator running OpenClaw with `OPENCLAW_STATE_DIR` or separate profiles will still get reusable MITM signing keys in the default home directory. That breaks the repo's state-dir isolation contract and can leave security-sensitive key material outside the selected state/profile root; resolve the cache path from `OPENCLAW_STATE_DIR` with the same home expansion semantics before generating keys.
  - confidence: 0.93
- **[P2] Make overflow prompt recovery non-interactive:** `extensions/anthropic/interactive-proxy/wrapper.ts:334`
  - body: When prompts exceed `maxPromptArgChars`, core sends the prompt on stdin and this wrapper rewrites it into a temp file plus a request for Claude to read that file. The backend only auto-allows `mcp__openclaw__*`, so non-bypass permission configurations can block on an interactive `Read` approval instead of processing the turn. Either avoid the tool call or explicitly handle the required permission path without broadening unrelated tool access.
  - confidence: 0.84
