## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Keep large prompts out of argv:** `extensions/anthropic/cli-backend-interactive.ts:175`
  - body: `resolvePromptInput` sends the full prompt as an argv value when `input: "arg"` is active and `maxPromptArgChars` is unset. This backend explicitly clears that limit, so long OpenClaw history prompts can exceed Windows or host command-line limits before the wrapper starts; keep a bounded argv path and hand long prompts to the wrapper via stdin or a file.
  - confidence: 0.86
- **[P2] Preserve inherited Claude command overrides:** `extensions/anthropic/cli-backend-interactive.ts:155-162`
  - body: The inherited `claude-cli.command` is treated as the Claude binary only when its basename is exactly `claude`, `claude.exe`, or `claude.cmd`. A configured Claude wrapper or symlink with another name will be misclassified as the Bun command and launched with `wrapper.ts` as an argument; carry inheritance provenance instead of guessing from the basename.
  - confidence: 0.79

## Security Review

Status: needs_attention

Summary: The diff introduces local TLS interception and reusable proxy CA material, so it needs explicit maintainer security review before merge.

Concerns:

- **[medium] Review the local TLS interception trust boundary:** `extensions/anthropic/interactive-proxy/wrapper.ts:271`
  - body: The wrapper launches Claude with child-scoped `HTTPS_PROXY` and `NODE_EXTRA_CA_CERTS`, and the Bun proxy terminates TLS for Anthropic API traffic to read plaintext SSE events. That is a security-model decision, not a routine backend detail.
  - confidence: 0.93
- **[medium] Require current-head proof for reusable proxy keys:** `extensions/anthropic/interactive-proxy/cert-manager.ts:147`
  - body: The PR stores a 365-day local CA and leaf private key under `~/.openclaw/proxy-certs`. Latest code attempts fail-closed key hardening, but the posted cert artifact predates those fixes and shows broadly readable key files, so current-head proof on supported OSes is still needed.
  - confidence: 0.86
