## Summary

Keep this draft PR open, but it is not ready to merge: latest head fixes several earlier review blockers, yet the wrapper still drops real post-tool continuation streams, the proof is stale/insufficient for current head, and the TLS interception design needs explicit maintainer/security acceptance.

## What This Changes

Adds an opt-in Anthropic `claude-cli-interactive` backend that runs Claude through a Bun loopback TLS proxy, emits tapped SSE thinking as CLI stream JSON, wires that stream into reasoning, and updates Anthropic setup/build metadata plus related tests.

## Best Possible Solution

Keep the backend draft until it preserves multi-request tool loops, repairs cached proxy key permissions, has current-head Telegram/log proof, and maintainers accept or redesign the TLS proxy trust model.

## Reproduction Assessment

Yes for the blocking review finding: source inspection shows all message streams after the first are suppressed, while the backend enables MCP tools and Anthropic documents client tool use as a stop-and-continue workflow. I did not run a live Claude subscription or Telegram turn.

## Solution Assessment

No in its current shape: an opt-in backend may be viable, but dropping post-tool continuation streams and relying on an unapproved TLS interception trust model are not the narrowest maintainable path yet.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve tool-loop streams after tool use:** `extensions/anthropic/interactive-proxy/wrapper.ts:163-180`
  - body: The wrapper treats every `message_start` after the first as title-generation noise and drops all following events. A normal Claude client-tool workflow can stop with `tool_use`, execute the tool, then continue in a later `/v1/messages` stream, so tool-using turns will lose the continuation and never emit the final `end_turn` result.
  - confidence: 0.87

## Security Review

Status: needs_attention

Summary: The diff introduces a local TLS interception path and persisted CA/leaf keys, with one concrete cached-key permission hardening gap.

Concerns:

- **[medium] Repair cached proxy key permissions before accepting them:** `extensions/anthropic/interactive-proxy/cert-manager.ts:123`
  - body: `ensureCerts()` returns immediately when existing cert/key files are valid, before running the new chmod/icacls hardening. Any keys generated by an older build or copied with broad permissions remain reusable until expiry; the proof artifact shows that exact world-readable state for `ca.key` and `leaf.key`.
  - confidence: 0.86
- **[medium] Review the TLS interception trust boundary:** `extensions/anthropic/interactive-proxy/wrapper.ts:122`
  - body: The wrapper forces the Claude child through a loopback HTTPS proxy and adds a generated CA through `NODE_EXTRA_CA_CERTS`, decrypting provider traffic locally to extract thinking events. Even when child-scoped, this should be an explicit documented opt-in security/product decision.
  - confidence: 0.9
