## Summary

Keep open: current main does not contain the proposed backend, and the PR is active draft feature work with a remaining prompt-delivery correctness issue plus unresolved real-behavior-proof and security-review gates.

## What This Changes

The PR adds an opt-in Anthropic `claude-cli-interactive` backend using a Bun local TLS proxy to reconstruct Claude stream-json reasoning, with parser/reasoning bridge updates, config inheritance, static assets, tests, package metadata, and unrelated doctor/changelog edits.

## Best Possible Solution

Keep the backend plugin-owned and opt-in only if maintainers accept the TLS proxy model; fix prompt-presence provenance, split unrelated edits, and require current-head Telegram/log proof plus cross-platform validation before merge.

## Reproduction Assessment

Not applicable as a feature PR: current main lacks `claude-cli-interactive`, and the remaining checks are proposed-backend validation and proof, not reproduction of an existing bug.

## Solution Assessment

No, not ready: an opt-in Anthropic plugin backend is a plausible ownership boundary if maintainers accept the TLS proxy design, but the current patch should carry prompt presence explicitly instead of inferring it from unknown argv entries.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Carry prompt presence explicitly:** `extensions/anthropic/interactive-proxy/wrapper.ts:197`
  - body: `claudeArgsContainPositional` treats any argument outside its hard-coded flag sets as the prompt. If a user inherits or configures a valid Claude flag this wrapper does not know about and the real prompt exceeds `maxPromptArgChars`, OpenClaw spills the prompt to stdin but this check still passes, launching interactive Claude with no delivered prompt until the watchdog fires. Carry an explicit prompt-present signal from the runner or otherwise distinguish configured flags from the prompt.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The PR introduces a local TLS interception path and reusable proxy CA material, so explicit maintainer security review is still needed.

Concerns:

- **[medium] Review the local TLS interception trust boundary:** `extensions/anthropic/interactive-proxy/wrapper.ts:315`
  - body: The wrapper launches Claude with child-scoped `HTTPS_PROXY` and `NODE_EXTRA_CA_CERTS`, while the Bun proxy terminates Anthropic API TLS to read plaintext SSE events. That is a security-model decision, not routine backend plumbing.
  - confidence: 0.93
- **[medium] Review reusable proxy CA key handling:** `extensions/anthropic/interactive-proxy/cert-manager.ts:147`
  - body: The cert manager creates and reuses a 365-day local CA and leaf key under the user's OpenClaw directory. Latest code attempts fail-closed hardening, but this design still needs maintainer acceptance and current-head proof on supported OSes.
  - confidence: 0.86
