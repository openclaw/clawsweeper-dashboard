## Summary

Keep this draft PR open: it is a meaningful feature, but the current head still has a setup/fallback inheritance bug, the attached proof does not show the claimed current-head thinking stream or visible Telegram reasoning, and the local TLS proxy trust model needs explicit maintainer/security review.

## What This Changes

Adds an opt-in Anthropic `claude-cli-interactive` backend that runs Claude through a Bun loopback TLS proxy, converts tapped SSE events into Claude stream-json output, wires thinking events into CLI reasoning streams, and updates setup/config/tests/static assets plus an unrelated Doctor warning change.

## Best Possible Solution

Keep the experimental backend as a draft until setup/fallback inheritance is fixed, current-head live reasoning proof is attached, and maintainers explicitly accept or revise the TLS proxy trust model.

## Reproduction Assessment

Yes for the review blocker: source inspection of PR head shows inheritance only runs for runtime-registered backends, while the setup fallback path skips it. I did not reproduce a live Claude subscription or Telegram turn.

## Solution Assessment

No in its current shape: an opt-in backend may be viable, but this implementation needs the setup/fallback fix, stronger current-head proof, and a maintainer decision on the TLS interception design before merge.

## Security Review

Status: needs_attention

Summary: The diff introduces local TLS interception and persisted signing material, so it needs explicit maintainer/security review before merge.

Concerns:

- **[medium] Review the TLS interception trust boundary:** `extensions/anthropic/interactive-proxy/wrapper.ts:122`
  - body: The wrapper forces the Claude child through a loopback HTTPS proxy and adds a generated CA via `NODE_EXTRA_CA_CERTS`, decrypting provider traffic locally to extract thinking events. Even when child-scoped, this changes the provider traffic trust boundary and should be an explicit, documented opt-in decision.
  - confidence: 0.92
- **[medium] Fail closed or repair insecure proxy key state:** `extensions/anthropic/interactive-proxy/cert-manager.ts:123`
  - body: The cert manager accepts any existing unexpired CA/leaf key files on the fast path, while the proof artifact shows `ca.key` and `leaf.key` as world-readable. The implementation should verify/repair permissions for existing files and decide whether ACL hardening failures are acceptable before shipping reusable 365-day signing keys.
  - confidence: 0.82
