## Summary

Keep open: current main does not have this opt-in backend, and the latest PR head fixes the earlier same-session serialization blocker, but merge should wait on the P1 overflow-file isolation fix, current-head real behavior proof, and explicit maintainer approval of the TLS interception trust boundary.

## What This Changes

The PR adds an opt-in Anthropic `claude-cli-interactive` backend that runs Claude Code through a Bun loopback TLS proxy, reconstructs stream-json reasoning events, registers the new backend/auth aliases, and adds related tests/static assets.

## Best Possible Solution

Land only as an opt-in, plugin-owned Anthropic backend after isolating overflow prompt files per run, documenting and approving the TLS proxy trust model, and supplying current-head live proof that reasoning and final replies work through Telegram.

## Reproduction Assessment

No as a current-main bug; this is a new backend feature. Yes for the blocking review finding: source inspection shows spill recovery uses one shared temp directory and adds that whole directory to Claude's accessible workspace.

## Solution Assessment

No, not yet. The plugin-owned backend boundary is plausible, but the shared overflow directory should be narrowed, the TLS trust model needs approval, and the latest branch needs real Telegram/proxy proof.

## Review Findings

Overall correctness: patch is incorrect

Full review comments:
