## What This Changes

The PR adds an experimental Anthropic `claude-cli-interactive` backend that runs Claude Code through a Bun wrapper and loopback TLS proxy to reconstruct Claude stream-json reasoning from interactive-mode SSE traffic.

## Best Possible Solution

Land this only as an opt-in Anthropic plugin backend after the proxy key lifecycle is hardened from creation time, unrelated changes are split out, current-head live Telegram proof is attached, and maintainers approve the TLS interception model.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P1] Create proxy keys inside a private directory:** `extensions/anthropic/interactive-proxy/cert-manager.ts:187-193`
  - body: `CERT_DIR` is created with default permissions, then `openssl genrsa` writes `ca.key` and `leaf.key` before `hardenKeyPermissions()` runs. On Unix-like hosts with a normal umask, another local user can read the reusable MITM CA key during that window; create/chmod the directory to `0700` before generation, or generate the keys with restrictive permissions from the start.
  - confidence: 0.88

## Security Review

Status: needs_attention

Summary: The diff introduces a local TLS interception backend and still has a concrete private-key generation hardening issue.

Concerns:

- **[high] Proxy private keys are exposed during generation:** `extensions/anthropic/interactive-proxy/cert-manager.ts:187`
  - body: The cert directory is not forced private before `openssl` writes reusable CA and leaf private keys, and hardening happens only after generation. A same-host user could read the MITM CA key during that window on Unix-like systems.
  - confidence: 0.88
- **[medium] Approve the TLS interception trust boundary:** `extensions/anthropic/interactive-proxy/wrapper.ts:431`
  - body: The backend routes Claude through a loopback proxy and injects a generated CA with `NODE_EXTRA_CA_CERTS`. That may be acceptable for an explicit experimental backend, but it needs owner approval and documentation because it decrypts Anthropic API traffic inside the child process path.
  - confidence: 0.92
