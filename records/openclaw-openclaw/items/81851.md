## Summary

Keep open. This active draft feature PR is still product- and security-sensitive, and source inspection shows the integrated backend path conflicts with the stated interactive subscription-mode design; the direct wrapper proof is useful, but integrated OpenClaw/Telegram reasoning-lane proof is still missing.

## What This Changes

The PR adds an experimental Anthropic `claude-cli-interactive` backend, Bun TLS proxy assets, thinking/text JSONL parsing, reasoning-lane bridging, backend inheritance support, package metadata, and unrelated doctor/changelog edits.

## Best Possible Solution

Decide the TLS-interception consent and key-storage policy first; if accepted, give the backend an execution path that does not reuse Claude stream-json live-session mode, preserve error/session contracts, and require integrated reasoning-lane proof.

## Reproduction Assessment

Yes for the blocking PR defects: source inspection shows `claude-cli-interactive` is routed through the existing Claude stream-json live-session helper, and the fallback result lacks the `is_error` field that helper uses. I did not run a live Claude subscription session.

## Solution Assessment

No. The wrapper idea may be useful after maintainer approval, but the current patch is not the best mergeable shape because the integrated path contradicts the intended interactive contract and lacks end-to-end proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Keep interactive backend out of Claude JSON live mode:** `src/agents/cli-runner/claude-live-session.ts:119`
  - body: Adding `claude-cli-interactive` here makes OpenClaw run it through `buildClaudeLiveArgs`, which unconditionally appends `--input-format stream-json`, `--output-format stream-json`, `--permission-prompt-tool stdio`, and `--replay-user-messages`, then writes JSON user records to stdin. That is not the interactive no-`-p` path shown in the wrapper proof, so selecting this backend from OpenClaw will not exercise the intended subscription-mode wrapper.
  - confidence: 0.93
- **[P2] Mark proxy fallback JSON as an error:** `extensions/anthropic/interactive-proxy/wrapper.ts:215-220`
  - body: `runClaudeDirect` emits a `result` record with `subtype: "error"`, but the Claude live-session handler only fails turns when `parsed.is_error === true`. If cert generation or proxy startup fails, OpenClaw can consume this as an empty successful result before the child exit is handled.
  - confidence: 0.88
- **[P2] Return a usable session id from fresh turns:** `extensions/anthropic/interactive-proxy/wrapper.ts:166`
  - body: The wrapper only copies `--session-id` or `--resume` from argv, but fresh live-session args strip `--session-id` and the wrapper ignores Claude stdout. Successful first-turn results therefore have no `session_id`, so OpenClaw cannot persist the provider session needed when the warm process closes and `sessionMode: "always"` tries to resume.
  - confidence: 0.76

## Security Review

Status: needs_attention

Summary: The diff intentionally adds local TLS interception of Anthropic traffic and a persisted local CA, so it needs explicit maintainer security and consent review before merge.

Concerns:

- **[medium] Review TLS interception and persisted CA:** `extensions/anthropic/interactive-proxy/wrapper.ts:177`
  - body: The wrapper sets `HTTPS_PROXY` and `NODE_EXTRA_CA_CERTS` for the Claude child, while cert generation stores a CA and leaf private key under the user's OpenClaw directory for up to 365 days.
