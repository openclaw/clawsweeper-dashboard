## What This Changes

Adds an opt-in Anthropic `claude-cli-interactive` backend that runs Claude through a Bun wrapper and loopback TLS proxy to reconstruct Claude stream-json events, with config inheritance, routing, packaging metadata, and focused tests.

## Best Possible Solution

Land only after the proxy runtime is covered by a wired check, maintainers explicitly accept the local TLS interception and key lifecycle, and the supported-platform scope is either proven or documented as experimental.

## Reproduction Assessment

Not applicable as a current-main bug reproduction: current main has no `claude-cli-interactive` backend. The PR supplies live Windows logs for the proposed path, but this is a new opt-in capability rather than broken existing behavior.

## Solution Assessment

Unclear until maintainer approval: the opt-in backend is a plausible way to preserve interactive-subscription streaming, but a core-owned local MITM proxy and cached key lifecycle are product/security decisions, and the shipped proxy runtime needs wired validation first.
