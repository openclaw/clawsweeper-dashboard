## Summary

Keep open: this is an active draft feature PR, not something already implemented on main, and the current head still has a source-visible prompt redaction bug plus unresolved real-proof and TLS trust-boundary blockers.

## What This Changes

The branch adds an opt-in Anthropic `claude-cli-interactive` backend that runs Claude interactive mode through a Bun loopback TLS proxy, parses `thinking_delta` into CLI reasoning events, wires provider/backend config, adds static assets/tests, and adjusts doctor warning logic.

## Best Possible Solution

Keep the backend experimental and opt-in only after prompt redaction is complete, current-head live and Telegram proof is supplied, and maintainers explicitly accept and document the TLS proxy trust boundary.

## Reproduction Assessment

Not applicable: this is a feature PR rather than a bug report. The blocking review finding is source-reproducible from the wrapper redactor and the CLI runner's final positional prompt contract.

## Solution Assessment

No in its current form: a plugin-owned opt-in backend is a plausible boundary, but the redaction bug, stale proof, and TLS interception decision must be resolved first.

## Security Review

Status: needs_attention

Summary: The diff introduces a local TLS interception backend and still has a concrete debug-log prompt leak.

Concerns:

- **[medium] Dash-prefixed prompts can leak to debug logs:** `extensions/anthropic/interactive-proxy/wrapper.ts:148`
  - body: The debug redactor preserves every dash-prefixed argv, but the prompt is a positional arg and can itself start with `-`; debug mode can still write private prompt text to stderr and gateway logs.
  - confidence: 0.9
- **[medium] Accept local TLS interception trust boundary:** `extensions/anthropic/interactive-proxy/wrapper.ts:242`
  - body: The wrapper forces Claude through a loopback HTTPS proxy and trusts a generated CA via `NODE_EXTRA_CA_CERTS`, decrypting Anthropic API traffic locally to recover thinking events; this needs explicit maintainer/security acceptance and documentation even if opt-in and child-scoped.
  - confidence: 0.9
- **[low] Refresh current-head key-permission proof:** `extensions/anthropic/interactive-proxy/cert-manager.ts:119`
  - body: The posted certificate artifact shows broad private-key permissions from an older revision; PR head attempts fail-closed hardening, but this security-sensitive behavior needs current-head proof showing repaired permissions or fail-closed behavior.
  - confidence: 0.82
