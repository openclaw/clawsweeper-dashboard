## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Normalize the CONNECT host before routing:** `extensions/anthropic/interactive-proxy/mitm-server.ts:334-336`
  - body: `isAllowedConnectHost()` trims and lowercases the host, but the routing check below still compares the raw `host` to `api.anthropic.com`. A valid case-varied CONNECT such as `API.Anthropic.Com:443` is allowed, then tunneled instead of sent to the local TLS server, so the wrapper never sees the SSE events it needs for reasoning/tool streaming. Use one normalized host value for both the allowlist decision and the MITM/pass-through branch.
  - confidence: 0.88
- **[P2] Preserve the interactive provider on all forward-compatible models:** `extensions/anthropic/register.runtime.ts:267-272`
  - body: This new provider patch preserves `claude-cli-interactive` for the cloned 4.6/4.7/4.8 template path, but `buildAnthropicForwardCompatModel()` below still maps every non-`claude-cli` modern model back to `anthropic`. The next modern model selected through `claude-cli-interactive/...` would therefore route to the Anthropic API runtime instead of the interactive backend. Extract this provider-preservation logic into a shared helper and cover both forward-compatible paths.
  - confidence: 0.79
