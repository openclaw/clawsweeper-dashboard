## Summary

Keep this PR open: current main and the latest release still only ship the existing non-interactive `claude-cli` backend, while this recent draft branch adds a large opt-in backend that still needs a correctness fix, current-head real behavior proof, and maintainer security/product approval for the local TLS proxy model.

## What This Changes

Adds an opt-in Anthropic `claude-cli-interactive` backend that runs Claude Code through a Bun loopback TLS proxy, captures Claude SSE thinking events into JSONL/reasoning streams, and wires backend config, static assets, tests, and related CLI plumbing.

## Best Possible Solution

Land this only after compaction detection uses a deterministic request/content-level signal, current-head proof demonstrates the reasoning stream in a real setup, the branch scope is cleaned up, and maintainers explicitly accept the child-scoped TLS proxy and CA storage model.

## Reproduction Assessment

Not applicable as a current-main bug because this PR adds a new opt-in backend. The blocking correctness issue is source-reproducible from the wrapper’s usage-based compaction suppression and Anthropic’s documented cache usage fields.

## Solution Assessment

No, not as-is. The plugin-owned backend boundary is plausible, but token-count-based compaction detection is unsafe for cached user turns and the TLS proxy design needs maintainer approval plus current-head real proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Don't suppress cached user turns by usage tokens:** `extensions/anthropic/interactive-proxy/wrapper.ts:569-570`
  - body: The wrapper classifies any `message_start` with small `input_tokens` and large `cache_read_input_tokens` as an internal compaction stream, then drops every event until `message_stop`. Anthropic documents that normal cached requests can have tiny uncached `input_tokens` with very large `cache_read_input_tokens`, so a short resumed user turn in a long cached session can match this heuristic and never emit a reply. Use a request/content-level compaction signal instead of usage totals. ([docs.anthropic.com](https://docs.anthropic.com/en/docs/build-with-claude/prompt-caching))
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: No new dependency-execution issue was found, but the patch intentionally decrypts Anthropic API traffic locally and persists a local CA, so the trust model needs maintainer approval.

Concerns:

- **[medium] Approve the local TLS interception trust model:** `extensions/anthropic/interactive-proxy/wrapper.ts:431`
  - body: The backend sets child-scoped `HTTPS_PROXY` and `NODE_EXTRA_CA_CERTS` so OpenClaw can terminate Anthropic API TLS locally and inspect SSE events; this is a security/product boundary decision, not ordinary backend plumbing.
  - confidence: 0.94
- **[medium] Accept persistent local CA key storage:** `extensions/anthropic/interactive-proxy/cert-manager.ts:147`
  - body: The cert manager creates and reuses a local CA and leaf private key under the user's OpenClaw directory for up to 365 days; the latest code hardens permissions, but maintainers still need to accept the stored signing-key model for this backend.
  - confidence: 0.86
