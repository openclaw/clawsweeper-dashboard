## Best Possible Solution

Land only after the network-error exit path is fixed, the backend remains opt-in, focused regression coverage is added, and maintainers explicitly approve the proxy security/support boundary.

## Solution Assessment

No, not yet. The opt-in backend direction is plausible, but the proxy must fail closed on upstream network errors and the TLS interception/key-cache design needs explicit maintainer approval before this is the best maintainable core solution.
