## Summary

Keep open: this draft feature PR still needs maintainer security/product approval and has a blocking source-level integration bug because it opts the interactive backend into the existing `claude-stdio` live-session path even though that path writes the `claude -p` JSON stdin protocol.

## What This Changes

The PR adds an experimental `claude-cli-interactive` Anthropic CLI backend that runs Claude through a loopback TLS proxy, captures SSE thinking/text deltas, and routes captured thinking into OpenClaw reasoning streams.

## Best Possible Solution

Redesign this as an explicitly opted-in Anthropic backend with a separate stdin/live-session contract and maintainer-approved TLS interception policy, while keeping the existing `claude-cli` reasoning fallback as the shipped path until then.

## Reproduction Assessment

Yes for the introduced PR bug from source inspection: current main's `claude-stdio` path writes JSON user envelopes, while this PR routes `claude-cli-interactive` through that path and forwards stdin unchanged to interactive `claude`. I did not run the live Claude/TLS path in this read-only review.

## Solution Assessment

No. The current branch needs either a wrapper-level stdin protocol translator or a separate live-session mode before the backend can be considered functionally correct, and the TLS interception policy still needs maintainer approval.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Do not reuse claude-stdio for interactive stdin:** `src/agents/cli-runner/claude-live-session.ts:118`
  - body: This gate now sends `claude-cli-interactive` through the existing Claude live-session writer, but that writer always emits JSON user envelopes for the `claude -p` protocol. The new wrapper removed `-p` and forwards stdin unchanged to interactive `claude`, so real OpenClaw turns will send JSON text instead of the raw prompt used in the wrapper proof. Add a wrapper translator or a separate live-session mode before opting this backend into `claude-stdio`.
  - confidence: 0.91
