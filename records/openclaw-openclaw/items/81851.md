## Summary

Keep open: this is a recent draft feature PR, not obsolete maintainer cleanup. The branch adds a meaningful opt-in Anthropic backend, but it still needs a session-serialization fix, maintainer approval of the TLS proxy trust model, and current-head real Telegram proof before merge.

## What This Changes

The PR adds an opt-in Anthropic `claude-cli-interactive` backend that runs Claude CLI through a Bun loopback TLS proxy to reconstruct stream-json reasoning events, plus parser/runner plumbing, setup registration, static assets, tests, and a Doctor warning change.

## Best Possible Solution

Land only if the backend remains opt-in and plugin-owned, preserves Claude session serialization, has an accepted TLS-proxy trust model, and has current-head live proof showing reasoning and final replies through Telegram.

## Reproduction Assessment

Not applicable as a current-main bug; this PR adds a new opt-in backend. Source inspection shows current main only registers `claude-cli`, while the branch adds `claude-cli-interactive`.

## Solution Assessment

No, not yet. The plugin-owned backend boundary is plausible, but this exact patch should not merge until session serialization is fixed, the TLS proxy trust model is accepted, and latest-head Telegram proof is supplied.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Keep Claude sessions serialized:** `extensions/anthropic/cli-backend-interactive.ts:127`
  - body: The new backend sets `serialize: false`, which makes `resolveCliRunQueueKey` return a per-run key before the Claude session/workspace queueing branch. Two runs resuming the same Claude session can then spawn concurrently and race on the same session transcript; the existing `claude-cli` backend keeps `serialize: true`, which still allows different sessions to run independently while protecting same-session state.
  - confidence: 0.88

## Security Review

Status: needs_attention

Summary: The diff introduces a local TLS interception path and reusable local signing keys, so the trust model needs explicit maintainer approval before merge.

Concerns:

- **[medium] Approve local TLS interception:** `extensions/anthropic/interactive-proxy/wrapper.ts:408`
  - body: The wrapper points the spawned Claude process at a local proxy and trusts a generated CA so OpenClaw can decrypt Anthropic API traffic and inspect plaintext SSE events. That may be acceptable for an opt-in backend, but it is a security and product trust-boundary decision.
  - confidence: 0.94
- **[medium] Accept reusable local CA storage:** `extensions/anthropic/interactive-proxy/cert-manager.ts:147`
  - body: The cert manager generates and reuses local CA and leaf private keys under OpenClaw config storage. Latest head hardens permissions and fails closed, but maintainers still need to accept the reusable signing-key model.
  - confidence: 0.86
