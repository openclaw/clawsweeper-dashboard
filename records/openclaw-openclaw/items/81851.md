## Summary

Keep open: current main does not contain this backend, so the PR is meaningful feature work, but current head still has a debug prompt-redaction bug, the linked real-behavior proof is stale relative to later security fixes, and the local TLS interception boundary needs explicit maintainer/security acceptance.

## What This Changes

Adds an opt-in Anthropic `claude-cli-interactive` backend with a Bun loopback TLS proxy to run Claude in interactive mode and reconstruct Claude stream-json events, plus parser/reasoning bridge updates, config inheritance, tests, static assets, and doctor-warning changes.

## Best Possible Solution

Keep the backend draft/experimental until prompt redaction is fixed, current-head Telegram/log proof is supplied, and owners accept and document the TLS proxy boundary.

## Reproduction Assessment

Not applicable for the main item because this is a feature PR, not a bug report. The review finding is source-reproducible from the PR head redactor: a final positional prompt beginning with `-` is preserved as a flag in debug output.

## Solution Assessment

No in its current form: a plugin-owned opt-in backend is a plausible direction, but the prompt-redaction bug, stale current-head proof, and TLS interception acceptance gap must be resolved before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Redact dash-prefixed prompt arguments:** `extensions/anthropic/interactive-proxy/wrapper.ts:148-151`
  - body: The debug redactor still treats every argument starting with `-` as a flag. Because this backend passes the user prompt as the final positional arg, a prompt beginning with `-` or `--` is logged verbatim when `OPENCLAW_INTERACTIVE_PROXY_DEBUG=1`; redact the prompt by position or known argv structure rather than by leading character.
  - confidence: 0.9

## Security Review

Status: needs_attention

Summary: The PR still has a concrete debug prompt leak and introduces a local TLS interception trust boundary that needs owner/security acceptance.

Concerns:

- **[medium] Dash-prefixed prompts can leak to debug logs:** `extensions/anthropic/interactive-proxy/wrapper.ts:148`
  - body: `redactArgsForLog` preserves every dash-prefixed argument, but the prompt is a positional argument and can itself begin with `-`; debug mode can still write private prompt text to stderr and gateway logs.
  - confidence: 0.9
- **[medium] Review the local TLS interception boundary:** `extensions/anthropic/interactive-proxy/wrapper.ts:242`
  - body: The wrapper forces Claude traffic through a loopback HTTPS proxy and trusts a generated CA via `NODE_EXTRA_CA_CERTS`, decrypting Anthropic API traffic locally to recover stream events. Even if opt-in and child-scoped, this needs explicit security/product acceptance and user-facing documentation.
  - confidence: 0.86
