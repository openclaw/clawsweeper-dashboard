## Summary

Keep this PR open: it is an active draft for a new Anthropic backend with useful wrapper proof, but the current diff still has blocking runtime defects and needs explicit TLS-interception product/security approval plus integrated OpenClaw and Telegram proof before merge.

## What This Changes

The PR adds an experimental `claude-cli-interactive` backend that runs Claude CLI through a Bun local TLS proxy to capture Anthropic SSE `thinking_delta`/`text_delta` events and route thinking into OpenClaw reasoning streams.

## Best Possible Solution

Accept this only after a maintainer-approved TLS-interception opt-in model, backend-specific live-session argv handling, parser-compatible error records, focused tests, and redacted integrated OpenClaw plus Telegram proof.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not as a current-main bug because this is a new backend feature, but the blocking PR defects are source-reproducible from the live-session gate, `buildClaudeLiveArgs`, wrapper error output, and current live-session parser contract.

## Solution Assessment

Is this the best way to solve the issue? No; the wrapper experiment is promising, but the current OpenClaw integration reintroduces noninteractive print-mode argv and reports proxy startup failures in a shape the live parser treats as success.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Keep interactive runs off print-mode argv:** `src/agents/cli-runner/claude-live-session.ts:119`
  - body: Adding `claude-cli-interactive` to the Claude live-session gate sends it through `buildClaudeLiveArgs`, which still appends `--input-format stream-json`, `--output-format stream-json`, `--permission-prompt-tool stdio`, and `--replay-user-messages` before spawning `bun wrapper.ts`. The wrapper forwards those args to `claude`, so the registered backend no longer matches the no-`-p` interactive/subscription path proved in the PR. Make the live-session argv builder backend-aware or add a separate interactive transport path.
  - confidence: 0.91
- **[P2] Emit proxy startup failures as CLI error results:** `extensions/anthropic/interactive-proxy/wrapper.ts:164-169`
  - body: `runClaudeDirect` writes a `type: "result"` record with `subtype: "error"`, an empty `result`, and an `error` field, but the Claude live-session parser only treats result records as failures when `is_error === true`. A cert or proxy startup failure can therefore resolve the turn as an empty successful result before the child exit is observed. Emit `is_error: true` with the message in `result`, or teach the parser this error shape.
  - confidence: 0.88

## Security Review

Status: needs_attention

Summary: The PR intentionally decrypts Anthropic traffic via a local CA and loopback proxy, so it needs explicit maintainer security/product approval before merge.

Concerns:

- **[medium] Approve the TLS interception opt-in boundary:** `extensions/anthropic/interactive-proxy/wrapper.ts:135`
  - body: The wrapper sets `HTTPS_PROXY` and `NODE_EXTRA_CA_CERTS` for the spawned Claude CLI so OpenClaw can decrypt Anthropic API traffic. Even though it is loopback and child-scoped, this capability needs an explicit maintainer-approved consent and opt-in model before release.
  - confidence: 0.96
- **[medium] Approve the persisted local CA lifecycle:** `extensions/anthropic/interactive-proxy/cert-manager.ts:9`
  - body: The cert manager stores a reusable CA under the user's OpenClaw directory for 365 days. The rotation, cleanup, permission, and consent lifecycle should be approved as a product/security boundary before exposing this backend.
  - confidence: 0.9
