## Summary

Keep open: this draft PR still has blocking runtime defects, needs maintainer security/product approval for TLS interception, and only proves the direct wrapper path rather than the integrated OpenClaw/Telegram behavior.

## What This Changes

The PR adds an experimental Anthropic `claude-cli-interactive` backend that uses a local TLS proxy to capture Claude CLI subscription-mode reasoning/text SSE deltas and route them into OpenClaw CLI and reasoning streams.

## Best Possible Solution

Decide the TLS-interception policy first; if accepted, split or repair the backend so it has a non-recursive default command, a correct interactive execution path, preserved error/session contracts, focused tests, and integrated redacted Telegram proof.

## Reproduction Assessment

Yes for the blocking PR defects: source inspection of the PR head shows the default wrapper child can be Bun, the backend is routed into Claude live JSON mode, and the fallback error record misses `is_error: true`. I did not run a live Claude subscription session.

## Solution Assessment

No. The proposed capability may be useful after maintainer approval, but this patch is not the best mergeable shape because it combines a sensitive TLS proxy with broken default/runtime contracts and incomplete integrated proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Do not point the default wrapper child at Bun:** `extensions/anthropic/cli-backend-interactive.ts:140-147`
  - body: With no user `claude-cli` override, `merged.command` is the interactive backend's own `bun` command, so this stores `OPENCLAW_INTERACTIVE_CLAUDE_BINARY=bun`. The wrapper then spawns Bun instead of Claude and can recursively launch the wrapper rather than exercising the subscription CLI path.
  - confidence: 0.94
- **[P1] Keep interactive backend out of Claude JSON live mode:** `src/agents/cli-runner/claude-live-session.ts:119`
  - body: Adding `claude-cli-interactive` here sends it through `buildClaudeLiveArgs`, which injects stream-json live-session flags and writes JSON user records to stdin. That is not the no-`-p` interactive path the wrapper proof demonstrates, so selecting the backend from OpenClaw will not run the intended mode.
  - confidence: 0.93
- **[P2] Mark proxy fallback results as live-session errors:** `extensions/anthropic/interactive-proxy/wrapper.ts:215-220`
  - body: `runClaudeDirect` emits a JSONL `result` with `subtype: "error"`, but the live-session handler only fails a turn when `is_error === true`. If cert or proxy startup fails after this PR, OpenClaw can parse the fallback as a successful empty result before the child exit is handled.
  - confidence: 0.88
- **[P2] Return a real session id for fresh interactive turns:** `extensions/anthropic/interactive-proxy/wrapper.ts:166`
  - body: Fresh interactive results only include `session_id` when one was already present in the args. Since the wrapper ignores Claude stdout and no fresh id is extracted from the proxy stream, successful first turns cannot persist the CLI session needed for later resume after the warm process closes.
  - confidence: 0.76

## Security Review

Status: needs_attention

Summary: The diff intentionally introduces local TLS interception of Anthropic traffic plus a persisted local CA, which needs explicit maintainer security and consent review.
