Full review comments:

- **[P2] Normalize the CONNECT host before choosing the tunnel target:** `extensions/anthropic/interactive-proxy/mitm-server.ts:334-336`
  - body: `isAllowedConnectHost` trims and lowercases the host, but the routing check below still compares the original `host` to `api.anthropic.com`. A valid case-insensitive CONNECT target such as `API.Anthropic.Com:443` is allowed but sent through as an undecrypted pass-through tunnel, so the wrapper will not see SSE events and can lose streamed reasoning/tool progress. Normalize once and use that value for both allowlisting and `isApiHost`, with a regression test for the mixed-case API host path.
  - confidence: 0.89

Add the interactive runtime/session ownership and focused doctor coverage with the manifest change.
