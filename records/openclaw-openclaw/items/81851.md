Full review comments:

- **[P1] Constrain the local CONNECT proxy:** `extensions/anthropic/interactive-proxy/mitm-server.ts:284-286`
  - body: This CONNECT handler tunnels every non-`api.anthropic.com` target requested by any localhost client while the wrapper is running. Because the proxy port is logged and there is no per-run auth or host allowlist, the new backend creates a temporary unauthenticated local TCP proxy unrelated to Anthropic SSE capture; reject unexpected targets or add a per-run proxy-auth/allowlist boundary before shipping it.
  - confidence: 0.78
- **[P2] Document the new CLI backend inheritance hook:** `src/plugins/cli-backend.types.ts:199-202`
  - body: `CliBackendPlugin` is exported through `openclaw/plugin-sdk/cli-backend`, so adding `inheritUserConfigFrom` creates a public plugin API surface. The current CLI backend docs and contract surface do not describe how sibling config inheritance, arg filtering, or upgrade behavior should work; either document/test it as SDK contract or keep the behavior internal to the bundled Anthropic backend.
  - confidence: 0.86
- **[P2] Keep the shipped proxy runtime in checked lanes:** `tsconfig.extensions.json:19`
  - body: The PR ships the proxy `.ts` files as runtime static assets, but this exclusion removes the whole `interactive-proxy` directory from the normal extension typecheck lane. The added nested tsconfig is useful only if CI or a package script actually runs it; otherwise wrapper/mitm/cert-manager drift can ship untyped in packaged installs.
  - confidence: 0.82
