## Summary

Keep open. Current main still exposes session storage as filesystem `sessions.json` paths with direct JSON file IO; the exact adapter PR is closed unmerged, and the related directory/SQLite PRs are narrower local-store optimizations rather than the requested external backend contract.

## What This Changes

Request an adapter/config surface for file, SQLite, Redis, Convex, and custom session-store backends so multiple gateways can share session state.

## Best Possible Solution

Define an additive, versioned session persistence contract with filesystem storage as the default, one supported external/plugin-backed path as proof, and aligned config/schema/docs, doctor/migration behavior, and session/transcript regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source and docs inspection gives high confidence that current main still lacks the requested driver/config/API seam.

## Solution Assessment

Unclear; a `SessionStoreAdapter` direction is plausible, but maintainers first need to choose the core, SDK/plugin, credential, migration, durability, and transcript-consistency boundaries.
