## Summary

Keep open. Current main removed the old `sessions.json.lock` contention path for in-process writes, but session storage is still configured, resolved, read, written, documented, and exposed to plugins as a file path rather than a pluggable backend contract.

## What This Changes

Request to add a configurable session-store adapter interface for file, SQLite, Redis, Convex, custom, or other shared backends.

## Best Possible Solution

Define an additive, versioned session-store backend contract with the existing file store as the compatibility default, or explicitly document that OpenClaw will support only local file/directory session storage for now.

## Reproduction Assessment

Not applicable; this is a feature request rather than a broken existing behavior report. Source inspection gives high confidence that current main still lacks the requested adapter/config path because session storage is resolved and operated as a file path.

## Solution Assessment

Unclear; a `SessionStoreAdapter`-style seam is plausible, but it is not automatically the best fix until maintainers choose the core, SDK, plugin, migration, and durability boundaries. A safer near-term path is to keep local writer/directory-store improvements separate from the external backend contract design.
