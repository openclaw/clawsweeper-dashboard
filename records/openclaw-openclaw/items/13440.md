## Summary

Keep open. Current main still resolves, documents, reads, writes, and exposes session storage as local `sessions.json` paths; the requested pluggable backend contract is not implemented and remains a storage/API design decision.

## What This Changes

The issue requests a configurable session-store adapter contract so OpenClaw can use backends such as file, SQLite, Redis, Convex, or custom modules instead of only a local JSON session file.

## Best Possible Solution

Define an additive, versioned session persistence/backend contract that keeps filesystem storage as the compatibility default, proves one supported backend or plugin seam, and updates config/schema/docs, migrations, doctor repair, and session/transcript tests.

## Reproduction Assessment

Not applicable; this is a feature request rather than a broken existing behavior report. Source and docs inspection gives high confidence that current main still lacks the requested driver/config/API seam.

## Solution Assessment

Unclear; a `SessionStoreAdapter` direction is plausible, but it is not clearly the best fix until maintainers choose the core, SDK/plugin, credential, migration, durability, and transcript-consistency boundaries.
