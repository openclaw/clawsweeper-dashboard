## Summary

Keep open. Current main still exposes session storage as a path-backed filesystem store with local JSON reads/writes and no driver/config/API seam for Redis, SQLite, Convex, or custom backends; adjacent storage PRs are useful but do not yet settle this external-backend contract.

## What This Changes

Request to add a pluggable session-store backend contract so OpenClaw can use file, SQLite, Redis, Convex, or custom session persistence.

## Best Possible Solution

Define an additive, versioned session persistence contract that keeps filesystem storage as the default, proves at least one supported backend, and updates config, schema, docs, migration/doctor behavior, SDK/runtime helpers, and regression coverage together.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source, docs, and related-thread inspection give high confidence that current main still lacks a driver/config/API seam for session-store backends.

## Solution Assessment

Unclear for the exact proposed SessionStoreAdapter shape. The pluggable-backend direction is plausible, but maintainers first need to choose the core vs plugin boundary, credential model, migration strategy, durability guarantees, and multi-instance consistency semantics.
