## Summary

Keep open. Current main still resolves, documents, reads, writes, and exposes session storage as a local `sessions.json` path, so the requested pluggable backend contract is not implemented and remains a maintainer design item.

## What This Changes

The issue requests a pluggable session-store backend contract for file, database, cache, cloud, or custom session persistence.

## Best Possible Solution

Define an additive, versioned session-store backend contract with the existing file store as the compatibility default, or explicitly document that OpenClaw supports only local file-backed session storage for now.

## Reproduction Assessment

Not applicable; this is a feature request rather than a broken existing behavior report. Source inspection gives high confidence that current main still lacks the requested driver/config/API seam.

## Solution Assessment

Unclear; a `SessionStoreAdapter` direction is plausible, but it is not clearly the best fix until maintainers choose the core, SDK/plugin, migration, credential, and durability boundaries.
