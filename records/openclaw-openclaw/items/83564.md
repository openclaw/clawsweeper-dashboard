## Summary

Keep open: current main and v2026.6.8 still render a single `Think` value, and source inspection shows a configured `adaptive` default can win before model-aware default resolution; the remaining fix needs a maintained status contract for configured versus resolved/provider thinking.

## What This Changes

Request `/status` to show both the configured `adaptive` thinking value and the effective resolved thinking level for the current turn.

## Best Possible Solution

Define one effective-thinking status contract, then update the status renderer and callers to show configured plus resolved values only where that layer is unambiguous and tested.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure `thinkingDefault: "adaptive"`, then `/status` can render the configured value before the model-aware default resolver is consulted.

## Solution Assessment

Unclear. Showing a configured/resolved pair is plausible, but maintainers should first choose whether `/status` reports configured OpenClaw level, canonical supported level, provider payload effort, or multiple fields.
