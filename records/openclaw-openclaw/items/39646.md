## Summary

Keep open: the status bug is source-reproducible on current main for provider-qualified raw defaults, but this branch is incomplete, stale, and bundles unrelated changes that current main has largely superseded.

## What This Changes

The branch validates raw default model provider display in model status, expands rate-limit matchers, treats empty legacy message targets as absent, moves Podman env sourcing earlier, and adds WebChat slash-command autocomplete.

## Best Possible Solution

Create or request a focused status-model resolver change that feeds one provider-valid effective default into JSON output, rich output, auth probes, and the status summary, with regression coverage.

## Reproduction Assessment

Yes. Source inspection of current main shows provider-qualified raw defaults are still used by models status JSON, auth probe candidates, rich terminal output, and the status summary before configured-provider fallback.

## Solution Assessment

No. The branch validates only defaultLabel while resolvedLabel is already computed from the stale raw value, and it leaves other status consumers plus unrelated UI/script changes in the same PR.
