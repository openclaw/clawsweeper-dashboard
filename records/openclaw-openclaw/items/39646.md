## Summary

Keep open. Current main still has source-reproducible stale default-model status paths, but this PR is not merge-ready because it validates only one status label and bundles unrelated UI, script, retry, and channel-target changes.

## What This Changes

The PR validates raw default-model provider strings for status output and also changes Podman env loading, rate-limit matching, channel-target normalization, embedding/cron retry matching, and WebChat slash-command autocomplete.

## Best Possible Solution

Ship a focused status/model-resolution repair that feeds one provider-valid effective default into JSON output, rich output, probe candidates, and the `openclaw status` summary, with regression tests and a changelog entry.

## Reproduction Assessment

Yes, from source inspection. With `agents.defaults.model.primary` set to a removed provider ref and `models.providers` containing another provider, current main still lets the raw model drive JSON status, probes, rich output, and the separate status summary path.

## Solution Assessment

No. The PR validates only one `defaultLabel` assignment and bundles unrelated changes; a narrow shared effective-default resolver scoped to status is the more maintainable fix.
