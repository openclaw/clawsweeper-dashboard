## Summary

Keep this PR open, but it should not merge as-is: current main still has a source-reproducible stale raw-default status path, while the branch only partially fixes that path and bundles unrelated changes that are now partly superseded.

## What This Changes

The PR validates raw default-model provider display in model status, expands rate-limit matchers, treats blank legacy outbound targets as absent, moves Podman env sourcing before bind resolution, and adds WebChat slash-command autocomplete.

## Best Possible Solution

Land a focused status-model resolver that computes one provider-valid effective default and reuses it for JSON output, rich output, auth probes, and status summary, with regression coverage.

## Reproduction Assessment

Yes. Source inspection is high-confidence: current main still uses raw provider-qualified defaults in model status JSON, probe candidates, rich output, and status summary before configured-provider fallback; I did not run a live CLI repro in this read-only review.

## Solution Assessment

No. The branch validates only `defaultLabel`, leaves other status consumers able to use stale provider refs, and bundles unrelated UI/script/rate-limit/outbound changes that should be split or rebased away.
