## Summary

Keep this PR open as useful source work, but do not merge it as-is. The stale raw default-model status bug is still valid on current main, but this branch only partially fixes the status path, leaves `openclaw status` untouched, and bundles unrelated changes that current main has already handled differently in several areas.

## What This Changes

The PR validates the raw default model provider before using it in `models status`, and also changes Podman env loading, rate-limit matching, channel-target blank handling, embedding retry matching, and WebChat slash-command autocomplete.

## Best Possible Solution

Land a focused status/model-resolution fix that shares one effective default-model resolver across `models status --json`, rich `models status`, auth-probe candidates, and `openclaw status`, while preserving valid CLI-backed and plugin/synthetic providers. Leave the unrelated Podman, rate-limit, channel-target, and WebChat work to the separate implementations already on main or to separate PRs.

## Reproduction Assessment

Yes. Use a config where `agents.defaults.model.primary` is a removed provider ref such as `anthropic/claude-opus-4-6` while `models.providers` only contains another provider; current-main code inspection shows both `models status` and `openclaw status` can still prefer that raw value before fallback.

## Solution Assessment

No. The proposed branch is not the best fix because it validates only one `defaultLabel` assignment, leaves other status consumers on raw values, does not update the separate status summary path, and bundles unrelated changes.
