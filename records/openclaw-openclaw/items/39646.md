  - confidence: 0.86
- **[P2] Return focus to the composer after command selection:** `ui/src/ui/views/chat.ts:497-501`
  - body: Clicking an autocomplete item only updates the draft and leaves focus on the clicked button. The next keystrokes will not continue in the textarea, which breaks commands that need arguments.
  - confidence: 0.85
- **[P3] Hide autocomplete when there are no matches:** `ui/src/ui/views/chat.ts:489-491`
  - body: The dropdown renders for any connected draft starting with `/`, even when there are no matching commands. Unknown commands therefore leave an empty styled box above the composer.
  - confidence: 0.82

  - body: The keydown handler special-cases Arrow and Tab but no dropdown keyboard path is wired, so navigation and selection do not work. Either implement selection state or let the textarea handle those keys.
  - confidence: 0.8
