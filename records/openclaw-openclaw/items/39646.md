## Summary

Keep this PR open. Current main still has the stale raw default-model behavior on the status surfaces this branch targets, but the branch is not merge-ready because the status fix is incomplete and the PR bundles unrelated UI, Podman, retry-pattern, channel-target, and embedding changes.

## What This Changes

The PR validates the raw default model provider before using it in `models status --json`, and also changes Podman env loading, rate-limit matching, channel-target blank handling, embedding retry matching, and WebChat slash-command autocomplete.

## Best Possible Solution

Land a focused status/model-resolution fix that shares one effective default-model resolver across `models status --json`, rich `models status`, auth-probe candidates, and `openclaw status`, while preserving valid CLI-backed and plugin/synthetic providers and adding regression tests for the removed-provider case.

## Reproduction Assessment

Yes. A high-confidence reproduction path is a config with `agents.defaults.model.primary` set to a removed provider such as `anthropic/claude-opus-4-6` while `models.providers` lacks that provider; current-main code inspection and the existing fixture show the raw value is still emitted by status paths.

## Solution Assessment

No. The proposed branch is not the best fix because it validates only one `defaultLabel` assignment, leaves other status paths raw, and bundles unrelated changes; a narrower shared resolver plus targeted tests is safer.
