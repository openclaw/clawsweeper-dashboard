## Summary

Keep this PR open but do not merge it as-is. The underlying status bug is still valid on current main, while the branch only partially addresses one status path and bundles unrelated Podman, retry-pattern, channel-target, and WebChat changes.

## What This Changes

The PR validates a raw default model provider in `models status`, broadens several rate-limit matchers, treats empty legacy channel targets as absent, moves Podman env-file sourcing earlier, and adds a WebChat slash-command autocomplete UI.

## Best Possible Solution

Create a focused status-model resolver repair that feeds one provider-valid effective default into `models status` JSON, rich output, auth probes, and the `openclaw status` summary, with regression tests and a changelog entry; leave the unrelated changes to separate PRs or existing mainline work.

## Reproduction Assessment

Yes. By source inspection, a provider-qualified `agents.defaults.model.primary` pointing at a removed provider is parsed before configured-provider fallback, then reused by `models status` JSON, auth probes, rich output, and the status summary path.

## Solution Assessment

No. The PR falls back to `resolvedLabel`, but that label can already be stale, and it leaves other status consumers on raw values while bundling unrelated UI/script/runtime changes.
