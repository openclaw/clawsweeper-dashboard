## Summary

Keep open: current main still has the raw removed-provider default flowing through status output and probe candidates, but this PR is stale, conflicting, incomplete, lacks real behavior proof, and bundles unrelated changes that are partly superseded on main.

## What This Changes

The PR validates raw default model provider display in model status, broadens rate-limit matching, adjusts legacy target handling and Podman env sourcing, and adds WebChat slash-command autocomplete.

## Best Possible Solution

Refresh this as a focused status-model resolver change that feeds one provider-valid effective default into JSON output, rich output, auth probes, and the status summary, leaving unrelated UI/script/rate-limit work to separate already-shipped or separately reviewed changes.

## Reproduction Assessment

Yes, source-reproducible. A provider-qualified `agents.defaults.model.primary` whose provider has been removed from `models.providers` still flows through `defaultLabel`, `rawCandidates`, `displayDefault`, and the status summary resolver on current main.

## Solution Assessment

No. The PR validates only one display label while other status consumers keep using raw or stale resolved values, and the branch also includes unrelated UI/script changes that have since diverged from main.
