## What This Changes

The PR changes models status default-label validation, rate-limit error matching, outbound target empty-string handling, Podman env sourcing, and WebChat slash-command autocomplete.

## Best Possible Solution

Create a focused status-model resolver repair that feeds one provider-valid effective default into `models status` JSON, rich output, auth probes, and the `openclaw status` summary, with regression tests and a changelog entry.

## Reproduction Assessment

Yes. By source inspection, a provider-qualified `agents.defaults.model.primary` pointing at a removed provider is parsed before configured-provider fallback, then reused by `models status` JSON, auth probes, rich output, and the status summary path.

## Solution Assessment

No. The PR validates only `defaultLabel`, falls back to a `resolvedLabel` that can already be stale, leaves other status consumers on raw values, and bundles unrelated changes.
