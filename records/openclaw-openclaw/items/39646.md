## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve the sticky composer while positioning the menu:** `ui/src/styles/chat/layout.css:140`
  - body: Changing `.chat-compose` from `sticky` to `relative` disables the existing bottom-sticky composer behavior in scrollable chats. Keep the sticky container and add a positioned inner wrapper for the autocomplete instead of making the composer scroll away.
  - confidence: 0.89
- **[P2] Use the validated default for probe candidates:** `src/commands/models/list.status-command.ts:230-236`
  - body: The PR validates `defaultLabel`, but `rawCandidates` still starts with `rawModel || resolvedLabel`. When the raw default points at a removed provider, auth probing can still include the stale provider and produce misleading probe results, so this should reuse the same validated value.
  - confidence: 0.86
- **[P2] Keep stale raw defaults out of rich status output:** `src/commands/models/list.status-command.ts:401-402`
  - body: The rich `openclaw status` display still formats `resolvedLabel (from rawModel)`, so the removed provider can remain visible even after the JSON label path falls back. Reuse the validated display value here too.
  - confidence: 0.86
- **[P2] Return focus to the composer after selecting a command:** `ui/src/ui/views/chat.ts:497-501`
  - body: Clicking an autocomplete button only calls `onDraftChange`, leaving focus on the clicked button. The next keystrokes will not continue in the textarea, which breaks flows like selecting `/model` and immediately typing its argument.
  - confidence: 0.85
- **[P3] Hide the autocomplete shell when there are no matches:** `ui/src/ui/views/chat.ts:489-491`
  - body: The dropdown is rendered for any connected draft starting with `/`, even when `getMatchingCommands()` returns an empty list. Typing an unknown command like `/xyz` leaves a styled empty box above the composer.
  - confidence: 0.82
- **[P3] Implement the keyboard path before intercepting navigation keys:** `ui/src/ui/views/chat.ts:461-463`
  - body: The keydown handler claims Arrow/Tab are handled by the dropdown but only returns without preventing default or selecting an item. Either wire a selected index and Tab/Enter behavior, or do not special-case those keys.
  - confidence: 0.8
