## Summary

Keep open: the underlying status bug is still valid on current main, but this branch is not merge-ready because the status fix only covers one label and the PR bundles unrelated script, retry, channel-target, embedding, cron, and WebChat changes.

## What This Changes

The PR attempts to validate raw default model providers for status output, while also changing Podman env loading, rate-limit matching, channel target normalization, embedding/cron retry matching, and WebChat slash-command autocomplete.

## Best Possible Solution

Implement one focused status/model-resolution repair that feeds an effective provider-valid default into JSON status, rich status, auth-probe candidates, and the `openclaw status` summary while preserving valid aliases, CLI providers, and plugin/synthetic providers.

## Reproduction Assessment

Yes. With `agents.defaults.model.primary` set to a removed provider ref and `models.providers` containing another provider, current source still lets `rawModel` drive `models status` JSON, probe candidates, rich output, and the separate status summary path.

## Solution Assessment

No. The proposed branch validates only one `defaultLabel` assignment, leaves other status consumers on raw values, misses the separate `openclaw status` resolver, and bundles unrelated changes; a narrow shared resolver is safer.
