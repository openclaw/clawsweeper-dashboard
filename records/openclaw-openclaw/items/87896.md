## Review Findings

Overall correctness: patch is incorrect

Full review comments:

- **[P1] Only mark streaming closes visible after content is sent:** `extensions/feishu/src/reply-dispatcher.ts:398`
  - body: In `renderMode: "card"`, `onReplyStart` starts a streaming card before any assistant content. If the turn then completes with `counts.final === 0`, `onIdle` closes that empty card and this line marks `visibleReplySent`, so `ensureNoVisibleReplyFallback` returns false and the user still gets a blank or silent completion. Gate the visible marker on non-empty sent content and add a zero-final card-mode test.
  - confidence: 0.95
- **[P2] Wire the fallback into active broadcast completion:** `extensions/feishu/src/bot.ts:1491`
  - body: Passing `sessionKey` here only helps silent-final detection; this active-broadcast branch still discards the `core.channel.inbound.run` result, unlike the single-agent branch that checks `counts.final === 0`. Active broadcast Feishu turns can therefore still finish with no visible reply and no fallback, so capture the dispatched result and call the fallback for zero-final active broadcast turns.
  - confidence: 0.93
