Full review comments:

- **[P1] Only mark streaming visible after content is sent:** `extensions/feishu/src/reply-dispatcher.ts:398`
  - body: When `renderMode: "card"` starts a streaming card before any text and the turn later completes with `counts.final === 0`, `closeStreaming()` can call `streaming.close("")`. `FeishuStreamingSession.close()` does not send a final content update for empty text, but this line still sets `visibleReplySent`, so `ensureNoVisibleReplyFallback()` returns false and the no-visible-output turn remains silent. Gate this on non-empty displayed content and add a card-mode zero-final regression.
  - confidence: 0.88
- **[P2] Apply the fallback to active broadcast dispatches:** `extensions/feishu/src/bot.ts:1491`
  - body: The active broadcast path now passes `sessionKey` into the same Feishu dispatcher but still discards the `core.channel.inbound.run()` result and never calls the zero-final fallback guard. In multi-agent group/broadcast mode, the active agent can still accept a turn and finish with no visible reply; capture that turn result and run the same guard used by the single-agent path.
  - confidence: 0.76
