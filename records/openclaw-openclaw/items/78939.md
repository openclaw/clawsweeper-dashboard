## Summary

Keep open: current main and the latest shipped release still lack stale usage-cost temp-file reclamation, and the open linked fix PR should be reviewed or landed before this issue closes.

## What This Changes

Temporary usage-cost cache files can remain in agent session directories after failed or interrupted cache writes and accumulate on disk.

## Best Possible Solution

Land the focused cleanup from the linked PR, keeping deletion limited to stale usage-cost temp names after the existing refresh lock is acquired and preserving recent temp files.

## Reproduction Assessment

Yes, source-reproducible. Seed stale `.usage-cost-cache*.tmp` files in an agent session directory and call `refreshCostUsageCache`; current main has no stale-temp cleanup path while the linked PR demonstrates the expected cleanup behavior.

## Solution Assessment

Yes. The linked PR's refresh-time cleanup after acquiring the existing usage-cost cache lock is narrower than a startup migration and matches the affected cache ownership boundary.
