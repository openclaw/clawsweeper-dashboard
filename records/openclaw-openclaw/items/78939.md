## Summary

Keep open. Current main already replaced the release write path with `replaceFileAtomic`, which should stop ordinary failed-write temp leaks, but there is still no source or test coverage that removes stale `.usage-cost-cache.json.*.tmp` files already left in session stores or left by hard interruptions.

## What This Changes

The issue asks OpenClaw to stop leaving orphaned usage-cost cache temp files in agent session stores and clean up stale leftovers safely.

## Best Possible Solution

Keep the fs-safe atomic write path and add a small stale-temp cleanup for old usage-cost cache temp files, with regression coverage that proves active/recent temp files are preserved.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path for the remaining issue: seed a stale `.usage-cost-cache.json.*.tmp` sibling in an agent session store and refresh the usage cache; current main has no matching cleanup path. I did not run that mutating repro because this review is read-only.

## Solution Assessment

No, the current main change is only a partial solution. `replaceFileAtomic` is the right write primitive for future failed writes, but the requested stale-orphan cleanup belongs in the usage-cache maintenance path with guarded deletion and tests.
