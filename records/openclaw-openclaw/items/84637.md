## Summary

Keep open: current main has several Codex/OpenAI runtime/auth split guardrails, but it does not provide the requested machine-checkable audit that says “no Codex runtime” while preserving Codex-named OpenAI model fallbacks routed through PI.

## What This Changes

Requested doctor/status/docs guardrails that distinguish Codex runtime or harness routing from model IDs containing `codex`, especially for PI-routed fallback policy.

## Best Possible Solution

Add a read-only runtime-policy diagnostic that reports effective runtime separately from model ID and fallback policy, plus docs that explicitly say Codex-named model IDs are allowed when their provider/model runtime policy is PI.

## Reproduction Assessment

No high-confidence reproduction of OpenClaw itself removing a fallback was established. Source inspection does confirm current `models status` and doctor lack the requested effective-runtime policy audit for PI-routed Codex-named fallbacks.

## Solution Assessment

Unclear as a final implementation because the requested `--runtime-policy normal=pi` shape is new product surface. The maintainable direction is a read-only diagnostic plus docs, not a startup migration or broad string-based cleanup.
