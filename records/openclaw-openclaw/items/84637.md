## Summary

Add guardrails so Codex runtime or harness routing is clearly separated from Codex-named model fallbacks such as `openai/gpt-5.3-codex`.

## Best Possible Solution

Provide a read-only route-policy audit that derives runtime from provider/model/session metadata, reports stale labels separately, and explicitly marks Codex-named OpenAI fallbacks routed through PI as allowed.

## Solution Assessment
