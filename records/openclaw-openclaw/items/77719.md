## Summary

Keep this open: current main still omits exec node context from the transparency formatter, and two open implementation PRs already reference this issue with closing syntax, so the issue should remain open until a canonical PR lands or is rejected.

## What This Changes

The issue requests that exec tool transparency messages include the selected `node` when a command is routed to a paired node.

## Best Possible Solution

Land one canonical narrow PR that appends node context in `resolveExecDetail`, preserves existing command/cwd formatting, adds focused regression tests, and treats default resolved-node display as separate follow-up if needed.

## Reproduction Assessment

Yes. Source inspection on current main shows an exec call with `args.node` reaches `resolveExecDetail`, but that function formats only command and cwd/workdir context.

## Solution Assessment

Yes. The narrow maintainable solution is to update the exec-specific formatter and tests; changing only `detailKeys` is insufficient because exec display uses `resolveExecDetail` first.
