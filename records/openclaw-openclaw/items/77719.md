## Summary

This should stay open: current main still formats exec transparency from the command-only exec detail path, while the exec schema and docs already carry a `node` argument that can be surfaced.

## What This Changes

Show the selected remote node name in exec tool-call transparency messages when exec is routed to a paired node.

## Best Possible Solution

Extend the exec-specific display formatter to include a sanitized node label when known, keep the existing command summary, update the shared tool-display snapshot, and add regression coverage for channel/UI-facing formatting.

## Reproduction Assessment

Yes from source inspection: an exec call with `args.node` reaches `resolveExecDetail`, which currently formats only command and cwd context. I did not run a live paired-node scenario because this review is read-only.

## Solution Assessment

Yes, but the narrow fix should update `resolveExecDetail`, not only `detailKeys`, because the exec-specific formatter short-circuits generic key formatting for normal exec calls.
