## Summary

Keep this open: current main still omits node context from exec transparency details, and an open implementation PR already references this issue with closing syntax, so the issue should remain open until the canonical fix lands or is rejected.

## What This Changes

The issue requests that exec tool transparency messages include the target node name when commands run on a paired node.

## Best Possible Solution

Land one canonical narrow PR that appends target node context only for node-target exec calls in `resolveExecDetail`, preserves cwd/raw/explain formatting, and adds regression tests for node and non-node hosts.

## Reproduction Assessment

Yes. Source inspection on current main shows exec transparency goes through `resolveExecDetail`, which formats command and cwd/workdir context but does not read `node`, while the exec schema and runtime already carry node information for `host=node`.

## Solution Assessment

Yes for the intended end state, but not by closing this issue now. The narrow maintainable path is to update the exec-specific formatter and tests through the open implementation PR or a replacement; changing only `detailKeys` is insufficient because exec uses the special formatter first.
