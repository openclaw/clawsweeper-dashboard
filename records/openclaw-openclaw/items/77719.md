## Summary

Keep this open: current main still omits node context from exec transparency details, and two open implementation PRs already close this issue, so it should stay open until a canonical PR lands or is rejected.

## What This Changes

Request to show the exec target node name in the user-facing tool-call transparency message when `exec` runs with `host=node`.

## Best Possible Solution

Review and land one canonical narrow PR, preferably the better-proven https://github.com/openclaw/openclaw/pull/78288 after rebase and checks, then let the issue close through that merged fix.

## Reproduction Assessment

Yes. Source inspection on current main shows exec transparency routes through `resolveExecDetail`, and that formatter ignores `args.node` even though `node` is already an exec argument for `host=node`.

## Solution Assessment

Yes. The narrow maintainable solution is to update `resolveExecDetail` and focused tests; changing only `detailKeys` is insufficient because exec display uses the special formatter first.
