## Summary

Keep open: the patch fixes the intended `agents.defaults.imageModel` direction, but it is too broad because it also bypasses explicit `tools.media.image.models` whenever the active model supports vision.

## What This Changes

The PR removes the explicit-image-understanding guard from `runCapability`'s native-vision skip and updates vision-skip tests to use text-only active models for configured image-model fallback coverage.

## Best Possible Solution

Keep native-vision skip independent of `agents.defaults.imageModel`, preserve explicit `tools.media.image.models` as an override, and add regression coverage for the reported imageModel-plus-vision-primary case.

## Reproduction Assessment

Yes, source-level. The linked issue and current `runCapability` path show how `agents.defaults.imageModel` bypasses native-vision skip, and this PR's new regression is visible from the early return before explicit model resolution.

## Solution Assessment

No. The best fix is narrower: remove `agents.defaults.imageModel` from the skip blocker without also removing the explicit `tools.media.image.models` override path.
