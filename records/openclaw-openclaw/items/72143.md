- `impact:session-state`: The bug is in resuming the original agent session for exec approval follow-up delivery.

## Summary

Keep open. Current main and the latest release still make only one exec approval session-followup attempt before direct fallback, and the internal resume-failure prefix can still be prepended for non-success fallback messages; current coverage only proves the successful `code 0` and denial cases are quieter.

## What This Changes

The issue asks OpenClaw to retry exec approval session-resume follow-ups before direct fallback, suppress the internal resume-failure prefix in user-facing fallback delivery, and log resume failures internally.

## Best Possible Solution

Finish the narrow exec approval follow-up fix in the owning agent helper: retry transient session-followup/resume failures with bounded backoff, keep all direct fallback user messages free of internal resume wording, and preserve focused coverage for success, denial, and error completions.

## Solution Assessment

Yes. The narrow maintainable solution is to add bounded retry around the existing session-followup call, keep fallback user copy silent about internal resume failure, and log the resume failure internally with routing metadata.
