## Summary

Keep open. Current main still exposes the internal resume-failure prefix for denied and non-success direct fallbacks, and the session follow-up path still makes only one agent resume attempt before direct fallback.

## What This Changes

The issue asks for exec approval follow-up fallback to retry session resume, hide the internal resume-failure prefix from user-facing messages, log resume failures internally, and preserve the command result content.

## Best Possible Solution

Add bounded short retry/backoff around the session resume call, remove the resume-failure prefix from every direct fallback message, and log sanitized resume-failure metadata internally after retries are exhausted.

## Reproduction Assessment

Yes, at source level. Mocking `callGatewayTool("agent")` rejection with a deliverable Telegram or Discord route exercises the direct fallback path, and current tests still expect the internal prefix for denied resume-failure output.

## Solution Assessment

Yes. The narrow maintainable fix is the requested bounded retry plus silent direct fallback; the open overlapping PRs cover adjacent or partial slices but do not make current main correct.
