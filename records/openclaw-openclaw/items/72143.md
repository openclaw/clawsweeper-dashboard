## Summary

Keep open. Current main still has the reported exec follow-up fallback behavior for denied and non-success results, and the session follow-up path still makes a single resume attempt before direct fallback.

## What This Changes

The issue requests retry/backoff for exec approval follow-up session resume, silent direct fallback without the internal prefix, and internal-only logging of resume failures.

## Best Possible Solution

Implement bounded retry/backoff in the exec approval follow-up session path, remove the internal prefix from all direct fallback messages, preserve the command status text, and log sanitized resume-failure metadata internally.

## Reproduction Assessment

Yes, from source inspection. Mocking callGatewayTool rejection in the existing exec approval follow-up test exercises the fallback path, and current main still expects the internal prefix for denied fallback while making only one session-follow-up attempt.

## Solution Assessment

Yes. The narrow maintainable fix is to add short bounded retries around the existing session follow-up call and make direct fallback user-facing content prefix-free; broader lifecycle bugs in #76162 and #76233 should stay separate unless a shared helper naturally falls out of the implementation.
