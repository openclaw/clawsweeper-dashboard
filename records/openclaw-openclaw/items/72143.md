## Summary

Keep open. Current main still exposes the internal resume-failure prefix on denied and non-success direct fallback paths, and the session follow-up path still makes a single `agent` gateway call before falling back. No merged fix on main covers the requested retry/backoff plus silent direct fallback; the open overlapping PR #68129 and closed unmerged PR #72148 should be checked before repair work proceeds.

## What This Changes

The issue requests bounded retry/backoff for exec approval follow-up session resume, silent direct fallback without the internal resume-failure prefix, and internal-only diagnostics when resume fails.

## Best Possible Solution

Land a narrow agents fix that retries the session follow-up call with bounded backoff, removes the resume-failure banner from every direct fallback delivery path, preserves sanitized result/status text, and emits sanitized internal diagnostics after retries are exhausted.

## Reproduction Assessment

Yes. A high-confidence unit reproduction exists by mocking `callGatewayTool` to reject in `src/agents/bash-tools.exec-approval-followup.test.ts`; the denied fallback test currently expects the internal prefix in the outbound message, and the source shows no retry wrapper around the session follow-up call.

## Solution Assessment

Yes. The requested direction is the narrow maintainable fix: retry the existing `agent` follow-up briefly, then silently direct-deliver sanitized result content and keep resume diagnostics in logs. PR #72148’s prefix-only approach would be partial, and PR #68129 is not merged into current main.
