## Summary

Keep open. Current main still exposes the internal session-resume fallback prefix for denied and non-success direct fallback messages, and the session follow-up path still makes a single resume attempt before falling back; the shipped changelog entry only covers successful async completion messages.

## What This Changes

Request to retry exec approval session follow-up before direct fallback, hide the internal resume-failure prefix from user-facing messages, log resume failures internally, and preserve command status content.

## Best Possible Solution

Add bounded short retry/backoff around session follow-up, remove the internal prefix from all direct fallback messages, and log sanitized resume-failure metadata only after retries are exhausted.

## Reproduction Assessment

Yes. Mocking callGatewayTool rejection in sendExecApprovalFollowup reproduces the direct fallback path, and the current denied resume-failure unit test still asserts the internal prefix.

## Solution Assessment

Yes. The narrow retry plus silent direct fallback requested here addresses the observed delivery bug without requiring a broader delivery architecture rewrite; #68129 and closed #72148 should be checked for overlap before implementation.
