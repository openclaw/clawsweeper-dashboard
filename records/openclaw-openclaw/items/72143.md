## Summary

Keep open. Current main still exposes the internal resume-failure prefix for denied and non-success direct fallback messages, still makes only one session follow-up attempt before falling back, and does not log the swallowed resume failure when direct fallback succeeds.

## What This Changes

The issue requests retry/backoff and prefix-free direct fallback delivery for exec approval follow-up resume failures.

## Best Possible Solution

Add bounded short retry/backoff around the existing session follow-up call, remove the resume-failure prefix from all direct fallback messages, and log sanitized resume-failure metadata internally after retries are exhausted.

## Reproduction Assessment

Yes from source inspection. Mocking `callGatewayTool` rejection exercises the fallback path, and current main still has a test expecting the internal prefix while the source shows only one resume attempt.

## Solution Assessment

Yes. The narrow maintainable fix is retry plus silent direct fallback with internal logging; broader lifecycle issues in #76162 and #76233 should stay separate unless the retry implementation exposes a shared helper need.
