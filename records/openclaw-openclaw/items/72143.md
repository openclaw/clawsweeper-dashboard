## Summary

Keep open. Current main still exposes the internal resume-failure prefix for denied or non-success direct fallback messages, and the session follow-up path still makes one resume attempt before falling back.

## What This Changes

The issue asks for exec approval follow-ups to retry session resume briefly, fall back without an internal user-facing prefix, log resume failures internally, and preserve the command result text.

## Best Possible Solution

Add bounded short retry/backoff around the existing agent session follow-up call, remove the resume-failure prefix from all direct fallback messages, and log sanitized resume-failure metadata internally after retries are exhausted.

## Reproduction Assessment

Yes. From source inspection, mocking callGatewayTool rejection exercises the direct fallback path, and the current denied test still expects the internal prefix while the implementation has only one session-resume attempt.

## Solution Assessment

Yes. The narrow maintainable fix is the requested retry plus silent direct fallback; the broader lifecycle race in https://github.com/openclaw/openclaw/issues/76233 should remain separate unless this work exposes a shared helper need.
