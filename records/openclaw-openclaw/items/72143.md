## Summary

Keep open. Current main still exposes the internal resume-failure banner for denied and non-success direct fallback paths, and the session follow-up path still makes a single `agent` call before falling back. The prior partial fix only covers successful `code 0` completions.

## What This Changes

This issue asks exec approval completion follow-ups to retry session resume with short backoff, fall back to direct delivery without an internal prefix, log resume failures internally, and preserve the command result/status.

## Best Possible Solution

Land a narrow agents fix that retries the existing session follow-up call with short bounded backoff, removes the resume-failure banner from every user-facing direct fallback path, preserves sanitized command result/status content, and logs sanitized resume-failure diagnostics only after retry exhaustion.

## Reproduction Assessment

Yes. The focused reproduction is to mock `callGatewayTool` rejection in `src/agents/bash-tools.exec-approval-followup.test.ts`; the denied resume-failure test currently expects the internal prefix, and the source shows only one session follow-up attempt.

## Solution Assessment

Yes. The requested direction is the narrow maintainable fix: add bounded retry around the existing follow-up call, then silently direct-deliver sanitized content if retries fail. A broader delivery rewrite is not required, but #68129 and #72148 should be checked before implementation.
