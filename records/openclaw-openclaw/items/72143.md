## Summary

Keep open. Current main still exposes the internal session-resume fallback prefix for denied and non-success direct fallback messages, and the session follow-up path still makes a single resume attempt before falling back.

## What This Changes

The issue requests bounded retry/backoff for exec approval follow-up session resume, prefix-free direct fallback delivery, internal-only failure logging, and preserved command result content.

## Best Possible Solution

Add bounded retries around the session resume call, remove the resume-failure prefix from all user-facing direct fallback messages, and log sanitized resume-failure metadata internally after retry exhaustion.

## Reproduction Assessment

Yes, at source level. Mocking callGatewayTool("agent") rejection with a deliverable Telegram or Discord route exercises the direct fallback path, and current tests still expect the internal prefix for denied resume-failure output.

## Solution Assessment

Yes. The requested bounded retry plus silent direct fallback is the narrow maintainable fix for the documented follow-up contract, with related raw-output leakage handled by separate linked issues.
