## Summary

Keep open. Current main still exposes the internal resume-failure banner for denied and non-success direct fallback paths, and the session follow-up path still makes a single resume attempt before falling back.

## What This Changes

The issue asks for async exec approval follow-up fallback to retry session resume, hide internal resume-failure text from users, log diagnostics internally, and preserve command result content.

## Best Possible Solution

Land a narrow agents fix that retries the existing session follow-up call with bounded short backoff, removes the resume-failure banner from every user-facing direct fallback path, preserves sanitized result content, and logs sanitized resume-failure diagnostics only after retries are exhausted.

## Reproduction Assessment

Yes. The focused reproduction is to mock `callGatewayTool` rejection in `src/agents/bash-tools.exec-approval-followup.test.ts`; the denied resume-failure test currently expects the internal prefix, and the source shows only one session follow-up attempt.

## Solution Assessment

Yes. The requested retry plus silent direct fallback is the narrow maintainable fix; a broader delivery rewrite is not required, but #68129 and closed #72148 should be checked for overlap before implementation.
