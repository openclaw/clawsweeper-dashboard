## Summary

Keep open. Current main only partially addresses the report: successful completion fallback no longer shows the banner, but denied and non-success direct fallback paths still expose the internal resume-failure prefix, and the session follow-up path still makes a single agent call before falling back.

## What This Changes

Request retry/backoff for async exec approval follow-up session resume, remove the internal resume-failure prefix from user-facing direct fallback messages, and keep resume diagnostics internal.

## Best Possible Solution

Land a narrow agents fix that retries the existing session follow-up call with short bounded backoff, removes the resume-failure banner from every user-facing direct fallback path, preserves sanitized result/status content, and emits sanitized internal diagnostics only after retry exhaustion.

## Reproduction Assessment

Yes. The focused unit reproduction is to mock `callGatewayTool` rejection in `src/agents/bash-tools.exec-approval-followup.test.ts`; the denied fallback test currently expects the internal prefix, and the source shows the one-shot session call.

## Solution Assessment

Yes. The requested direction is the narrow maintainable solution: add bounded retry around the existing follow-up call, then silently direct-deliver sanitized content if retries fail. A broader delivery-policy rewrite is not required, but overlap with #68129 should be reconciled first.
