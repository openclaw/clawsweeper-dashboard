## Summary

Keep open. Current main still exposes the internal resume-failure banner on denied and non-success direct fallback paths, and the session follow-up path still makes a single `agent` gateway call before falling back.

## What This Changes

The issue asks for exec approval follow-up delivery to retry session resume briefly, remove the internal fallback prefix from user-facing messages, log sanitized resume failures internally, and preserve the command result content.

## Best Possible Solution

Land a narrow agents fix that retries the session follow-up call with bounded backoff, removes the resume-failure banner from every direct fallback delivery path, preserves sanitized result/status text, and emits internal-only sanitized diagnostics after retries are exhausted.

## Reproduction Assessment

Yes. A high-confidence unit reproduction exists by mocking `callGatewayTool` to reject in `src/agents/bash-tools.exec-approval-followup.test.ts`; the denied fallback test currently expects the internal prefix in the outbound message.

## Solution Assessment

Yes. The requested direction is the narrow maintainable fix: retry the existing `agent` follow-up call briefly, then silently direct-deliver sanitized result content and keep resume diagnostics in logs. A broader delivery-policy rewrite is not needed for this report.
