## Summary

Keep open. Current main still exposes the internal resume-failure prefix for denied and non-success direct fallback messages, and the session follow-up path still makes a single resume attempt before direct fallback.

## What This Changes

The issue requests bounded retry/backoff for exec approval follow-up session resume, silent direct fallback without the internal prefix, and internal-only resume-failure logging.

## Best Possible Solution

Patch the exec approval follow-up path to retry session resume briefly, silently deliver sanitized command status only after retries fail, and keep resume-failure diagnostics in internal logs.

## Reproduction Assessment

Yes from source inspection. Mocking callGatewayTool rejection exercises the fallback path, the current denied test still expects the internal prefix, and the implementation has only one session resume attempt.

## Solution Assessment

Yes. The requested bounded retry plus prefix-free direct fallback is the narrow maintainable fix; broader lifecycle issues in #76162 and #76233 should remain separate unless a shared helper naturally falls out.
