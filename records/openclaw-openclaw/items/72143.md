## Summary

Keep open. Current main still exposes the internal resume-failure prefix on denied and non-success direct fallback messages, and the session follow-up path still makes only one resume attempt before falling back.

## What This Changes

Request to retry async exec approval session resume before direct fallback and keep internal resume-failure diagnostics out of user-facing chat.

## Best Possible Solution

Add bounded short retry/backoff around the existing session follow-up call, remove the resume-failure prefix from all direct fallback messages, and log sanitized resume-failure metadata internally after retries are exhausted.

## Reproduction Assessment

Yes. The focused reproduction is to mock callGatewayTool rejection in src/agents/bash-tools.exec-approval-followup.test.ts; the denied resume-failure test currently expects the internal prefix, and the source shows a single resume attempt.

## Solution Assessment

Yes. The requested retry plus silent direct fallback is the narrow maintainable fix; a broader delivery rewrite is not required, but #68129 and closed #72148 should be checked before implementation.
