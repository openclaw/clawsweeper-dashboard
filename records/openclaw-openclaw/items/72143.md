## Summary

Keep open. Current main still prepends the internal resume-failure prefix for denied and non-success direct fallbacks, and the session follow-up path still makes one agent resume attempt before direct fallback; only the successful completion prefix case was fixed.

## What This Changes

The issue requests retry/backoff for exec approval follow-up session resume failures, silent direct fallback without the internal prefix, and internal logging of resume-failure metadata while preserving command result text.

## Best Possible Solution

Add bounded short retry/backoff around the existing agent session follow-up call, remove the resume-failure prefix from all direct fallback messages, and log sanitized resume-failure metadata internally after retries are exhausted.

## Reproduction Assessment

Yes, at source level. Mocking callGatewayTool rejection exercises the direct fallback path, and the current denied test still expects the internal prefix while the implementation makes only one session-resume attempt.

## Solution Assessment

Yes. The narrow maintainable fix is the requested retry plus silent direct fallback; the broader lifecycle race in https://github.com/openclaw/openclaw/issues/76233 should remain separate unless a shared helper naturally falls out of the fix.
