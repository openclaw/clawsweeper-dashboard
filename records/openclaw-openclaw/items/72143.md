## Summary

Keep open. Current main still exposes the internal resume-failure prefix on denied/non-success direct fallback messages, and the session follow-up path still makes only one resume attempt before falling back.

## What This Changes

The issue asks for exec approval follow-up delivery to retry session resume before direct fallback, suppress the internal resume-failure prefix from user-facing messages, and log resume failures internally.

## Best Possible Solution

Add bounded short retry/backoff around the existing session follow-up call, remove the resume-failure prefix from all direct fallback messages, and log sanitized resume-failure metadata internally after retries are exhausted.

## Reproduction Assessment

Yes from source inspection. Mocking `callGatewayTool` rejection in `src/agents/bash-tools.exec-approval-followup.test.ts` exercises the fallback path, and the current denied test still expects the internal prefix while the source shows only one session-resume attempt.

## Solution Assessment

Yes. The requested retry plus silent direct fallback is the narrow maintainable fix; broader delivery lifecycle issues in #76162 and #76233 should remain separate unless the retry work exposes a shared helper need.
