## Summary

Keep open. Current main still has the reported exec approval follow-up gaps: session resume is attempted once before direct fallback, and non-success direct fallback can still prepend the internal resume-failure prefix to user-facing delivery.

## What This Changes

The issue asks OpenClaw to retry exec approval follow-up session resume with short backoff, silently fall back to direct delivery without internal wording, and log resume failures internally.

## Best Possible Solution

Land a narrow owner-approved fix that retries only the session-resume handoff before any direct send, removes internal resume-failure wording from all user-facing fallback delivery, logs correlation metadata, and adds focused regression tests.

## Reproduction Assessment

Yes by source inspection: current main still makes one agent resume call and then direct-fallbacks, and the direct fallback can still prepend the internal prefix for non-success payloads after a resume error. I did not run a live channel repro in this read-only review.

## Solution Assessment

Yes on the requested end state, but the exact retry schedule and duplicate-delivery behavior need maintainer approval before a repair PR is safe. The current code and the broad open PR are not yet the best landed solution.
