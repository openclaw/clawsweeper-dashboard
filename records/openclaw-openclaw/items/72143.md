## Summary

Keep open. Current main still has the user-facing resume-failure prefix for denied and other non-success direct fallback paths, and the session follow-up path still performs a single `agent` call before direct delivery.

## What This Changes

The issue requests bounded retry for async exec approval session follow-up and removal of the internal resume-failure prefix from user-facing fallback delivery.

## Best Possible Solution

Land a narrow agents fix that retries the existing session follow-up call with short bounded backoff, removes the resume-failure banner from all user-facing fallback paths, preserves sanitized result content, and logs sanitized diagnostics internally after retry exhaustion.

## Reproduction Assessment

Yes. A focused reproduction is to mock `callGatewayTool` rejection in `src/agents/bash-tools.exec-approval-followup.test.ts`; current source then sends direct fallback, and the denied-path test still expects the internal prefix.

## Solution Assessment

Yes. The requested retry/backoff plus silent direct fallback is the narrow maintainable fix; a broader delivery rewrite is unnecessary, but #68129 and #72148 should be inspected before implementation.
