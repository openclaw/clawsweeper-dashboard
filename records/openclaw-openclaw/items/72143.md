## Summary

Keep open. Current main still exposes the internal resume-failure prefix for denied and non-success direct fallback messages, and the session-resume path still makes a single `callGatewayTool("agent")` attempt before falling back; the successful-completion prefix case was fixed separately, but the full request remains unresolved.

## What This Changes

The issue asks for exec approval follow-up fallback delivery to retry session resume, remove the internal resume-failure prefix from user-facing fallback messages, preserve the result content, and log resume-failure metadata internally.

## Best Possible Solution

Add bounded short retry/backoff around the existing agent session follow-up call, remove the resume-failure prefix from all direct fallback messages, and log sanitized resume-failure metadata internally after retries are exhausted.

## Reproduction Assessment

Yes, at source level. Mocking `callGatewayTool("agent")` rejection exercises the direct fallback path, and the current denied test still expects the internal prefix while the implementation makes only one session-resume attempt.

## Solution Assessment

Yes. The narrow maintainable fix is the requested retry plus silent direct fallback; broader raw-output guardrails and bundle-MCP lifecycle races are separately tracked and should not be folded into this issue unless the implementation naturally shares a helper.
