## Summary

Keep open. Current main still exposes the internal resume-failure prefix for denied and non-success direct fallbacks, and the session follow-up path still makes only one agent resume attempt before falling back.

## What This Changes

The issue requests retry/backoff for exec approval follow-up session resume, silent direct fallback without the internal resume-failure prefix, and internal logging of resume failures.

## Best Possible Solution

Add bounded short retry/backoff around the existing agent session follow-up call, remove the resume-failure prefix from every direct fallback message, and log sanitized resume-failure metadata internally after retries are exhausted.

## Reproduction Assessment

Yes, at source level. Mocking `callGatewayTool("agent")` rejection with a deliverable Telegram or Discord route exercises the direct fallback path, and current tests still expect the internal prefix for denied resume-failure output.

## Solution Assessment

Yes. The narrow maintainable fix is the requested bounded retry plus silent direct fallback; the broader lifecycle race in https://github.com/openclaw/openclaw/issues/76233 and source-like fallback leak in https://github.com/openclaw/openclaw/issues/74283 should remain separate unless a shared helper naturally falls out of the fix.
