## Summary

Keep open. Current main still leaves the requested behavior partially unresolved: successful resume-failure direct fallbacks avoid the internal prefix, but denied and other non-success direct fallbacks can still expose it, and the session resume path still performs a single `agent` gateway call with no retry/backoff before falling back.

## What This Changes

Retry exec approval follow-up session resume with short backoff, silently direct-deliver preserved result/status content if resume still fails, and keep resume-failure diagnostics in sanitized internal logs.

## Best Possible Solution

Land a narrow agents fix that wraps the follow-up `agent` gateway call in bounded retry/backoff, removes the resume-failure prefix from every direct fallback delivery path, and logs sanitized resume failure metadata only after retries are exhausted.

## Reproduction Assessment

Yes. Mock `callGatewayTool` to reject in the denied approval follow-up path with a deliverable Telegram target; current test coverage already exercises that path and expects the internal prefix in the outbound message.

## Solution Assessment

Yes. The requested direction is the narrow maintainable solution: retry session resume briefly, then direct-deliver only sanitized result content and keep resume diagnostics internal; a broad delivery-policy rewrite is not required for this report.
