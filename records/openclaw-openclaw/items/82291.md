## Summary

Keep open. The report matches a real workspace bootstrap failure boundary: normal inbound replies ensure the workspace before the agent turn, and current main still has stale-bootstrap repair depend on successfully removing `BOOTSTRAP.md` before recording completion.

## What This Changes

The issue reports that upgrading a legacy workspace to 2026.5.12 leaves a regular immutable `BOOTSTRAP.md` that message handling treats like a directory, causing inbound Telegram and Slack messages to fail with `ENOTDIR`.

## Best Possible Solution

Decouple setup completion from physical deletion of legacy `BOOTSTRAP.md`: persist or honor completed state first, exclude stale completed bootstrap files from active routing/context, and leave undeletable cleanup as a warning or `openclaw doctor --fix` repair.

## Reproduction Assessment

Yes for a focused source reproduction, not for the exact Docker immutable-bit environment in this read-only review. Current main still routes inbound replies through `ensureAgentWorkspace`, and the stale-bootstrap repair can throw before persisting completion when `BOOTSTRAP.md` removal fails.

## Solution Assessment

Yes, the narrow maintainable fix is in workspace bootstrap reconciliation and selection, not in Telegram or Slack. The safer approach is to make completed setup authoritative and treat stale-file deletion as best-effort cleanup.
