## Summary

Keep open. Current main and v2026.6.10 still expose Skill Workshop lifecycle actions without a delete/remove path, but the requested applied-proposal cleanup is destructive and needs maintainer product/security direction before implementation.

## What This Changes

Add a `skill_workshop` delete action that removes an applied proposal record and its live skill directory.

## Best Possible Solution

Define a safe destructive lifecycle contract in the shared Skill Workshop service, then update the agent tool, CLI, Gateway, docs, approval policy, and focused tests together.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection confirms current main lacks `delete` across the Skill Workshop tool, service, CLI/Gateway surface, docs, and latest release.

## Solution Assessment

Unclear. A delete action is plausible for applied create proposals, but the proposed delete-record-plus-skill-directory behavior is not yet the best shape for applied update proposals because it could remove an existing skill.
