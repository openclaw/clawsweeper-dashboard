## Summary

Keep open. The PR targets a real Feishu reaction/chat-type gap, but the current branch still has source-reproducible regressions in reaction dedupe and topic-session routing, lacks contributor-supplied real Feishu proof, and is stale against current main's reply-target and topic-group handling.

## What This Changes

The PR changes Feishu reaction synthetic event IDs and reply anchoring, increases reaction verification timeout, infers missing chatType from chat_id prefixes in fetched messages, and updates reaction tests.

## Best Possible Solution

Rework the branch to infer missing chatType while preserving topic_group, keep synthetic reaction IDs event-unique, carry the original Feishu message through reply/thread metadata before session routing, and add focused tests plus real Feishu proof.

## Reproduction Assessment

Yes for the blocking review findings from source inspection: the PR's deterministic synthetic ID is the same for repeated same-message/same-action/same-emoji/same-sender reaction events, and current main dedupes that message key for 24 hours. I did not run a live Feishu reproduction in this read-only review.

## Solution Assessment

No. The maintainable fix should use the existing reply_target_message_id/root/thread metadata before routing and keep the synthetic processing ID unique, rather than parsing a deterministic synthetic ID late in bot.ts.

Full review comments:

  - body: The new synthetic ID is deterministic for messageId, action, emoji, and sender. In group_topic or group_topic_sender mode, history and bindings can be written under the synthetic reaction key while the visible reply targets the original message; put the real anchor into reply/thread metadata before routing.
  - confidence: 0.82
- **[P2] Preserve topic_group when inferring chat types:** `extensions/feishu/src/send.ts:266-271`
  - body: Current main accepts topic_group from fetched Feishu messages, but this branch's parser only treats group/private/p2p as explicit chat types before falling back to chat_id prefixes. A fetched topic_group item with an oc_ chat_id is downgraded to group, breaking topic-specific routing; keep topic_group as an explicit valid value before prefix inference.
