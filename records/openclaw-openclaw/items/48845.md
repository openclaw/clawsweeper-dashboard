## Summary

Keep open. Current main still has the missing-Feishu-chat-type behavior this PR tries to fix, but the PR should not merge as-is because it can still drop repeated reaction lifecycle events by dedupe key, loses topic-session metadata for reactions, and regresses explicit `topic_group` parsing.

## What This Changes

The PR changes the Feishu plugin to infer missing chat types from Feishu IDs, adjust synthetic reaction IDs and reply targeting, raise reaction verification timeout, and update reaction tests.

## Best Possible Solution

Refresh the Feishu plugin fix so chat-type inference preserves current `topic_group` and DM behavior, reaction synthetic events keep unique processing IDs while carrying the original message/thread anchor, and topic metadata is routed before reply selection.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: with the PR's deterministic `message_id`, repeated same-action reactions for the same source message, sender, and emoji reuse the Feishu dedupe key; current main also has a test-backed path where missing `chat_type` causes reaction lookup drops.

## Solution Assessment

No. The fix belongs in the Feishu plugin, but this PR conflates event identity with reply anchoring and misses existing topic metadata/chat-type cases; a narrow refreshed patch is safer.
