## Summary

Keep open: the Feishu bug-fix direction is plausible, but this patch is not merge-ready because deterministic reaction IDs still collide under Feishu dedupe, fetched-message parsing drops `topic_group`, and the PR lacks real Feishu behavior proof.

## What This Changes

The PR changes Feishu reaction synthetic event IDs and reply targeting, infers missing fetched-message chat types from chat ID prefixes, raises the reaction verification timeout, and updates reaction tests.

## Best Possible Solution

Land one reconciled Feishu plugin fix that preserves unique internal synthetic reaction IDs, normalizes to base Open Message IDs only at Feishu API boundaries, preserves `topic_group`, and includes real Feishu runtime proof.

## Reproduction Assessment

Yes, source-reproducible. Current main dedupes Feishu text events by `message_id`, and PR head changes reaction IDs to a deterministic tuple that can repeat; current main also preserves `topic_group` while PR head drops it.

## Solution Assessment

No. The safer fix is Feishu-local API-boundary normalization plus a verified chat-type fallback; this branch changes internal reaction identity and loses current topic-group parsing.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Keep reaction synthetic IDs unique per event:** `extensions/feishu/src/monitor.account.ts:129`
  - body: The new synthetic ID is deterministic for `messageId + action + emoji + sender`. `handleFeishuMessage` dedupes by the event `message_id`, so a valid repeat like created -> deleted -> created for the same user and emoji within the dedupe window reuses the first `created` ID and the later event is skipped. Keep a per-event token internally and normalize to the base Feishu message ID only at API boundaries.
  - confidence: 0.9
- **[P2] Preserve topic_group when parsing fetched messages:** `extensions/feishu/src/send.ts:266-268`
  - body: This parser branch no longer accepts `topic_group`, which current main preserves. When Feishu returns a topic-group message, the new prefix fallback degrades it to plain `group`, so reaction routing loses topic-specific session and reply context.
  - confidence: 0.86
