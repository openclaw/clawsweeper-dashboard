## Summary

Keep open: current main still lacks the Feishu missing-chat-type fallback, but this PR is not mergeable as-is because the branch regresses existing `topic_group` parsing and still uses dedupe-colliding reaction IDs.

## What This Changes

The PR updates the Feishu plugin to infer chat type from `chat_id`, change synthetic reaction event IDs and reply targeting, raise reaction verification timeout, and adjust reaction tests.

## Best Possible Solution

Refresh the Feishu plugin fix so chat-type inference preserves explicit `topic_group` and DM behavior, reaction synthetic events keep unique processing IDs while carrying original reply/root/thread anchors separately, and the user-facing fix has focused tests plus a changelog entry.

## Solution Assessment
