## Summary

Keep open: this targets a real Feishu reaction gap, but the branch is not merge-ready because it introduces deterministic reaction IDs that collide under existing dedupe, regresses current `topic_group` preservation, and lacks real Feishu proof.

## What This Changes

The PR changes Feishu reaction synthetic-event IDs and reply targeting, infers missing fetched-message chat types from chat ID prefixes, increases reaction verification timeout, and updates Feishu reaction tests.

## Best Possible Solution

Keep synthetic reaction IDs unique inside OpenClaw, normalize them only at Feishu API boundaries, preserve `topic_group`, and add verified chat-type fallback coverage plus real Feishu runtime proof.

## Reproduction Assessment

Yes, source-reproducible. Current main can be traced from missing fetched chat type to skipped reaction handling, and from synthetic `:reaction:` IDs through dedupe/API-boundary paths, although I did not run a live Feishu account in this read-only review.

## Solution Assessment

No. The safer fix is Feishu-local API-boundary normalization plus a verified chat-type fallback; this branch instead makes internal reaction IDs deterministic and loses current `topic_group` parsing.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Keep reaction synthetic IDs unique per event:** `extensions/feishu/src/monitor.account.ts:129`
  - body: The new ID is deterministic for `messageId + action + emoji + sender`. `handleFeishuMessage` dedupes by the event `message_id`, so a valid repeat like created -> deleted -> created for the same user and emoji within the dedupe window reuses the first `created` ID and the later event is skipped. Keep a per-event token for the internal synthetic ID and normalize to the base Feishu message ID only when calling Feishu APIs.
  - confidence: 0.9
- **[P2] Preserve topic_group when parsing fetched messages:** `extensions/feishu/src/send.ts:266-268`
  - body: This parser branch no longer accepts `topic_group`, which current main preserves. When Feishu returns a topic-group message, the new prefix fallback degrades it to plain `group`, so reaction routing loses topic-specific session and reply context.
  - confidence: 0.86
