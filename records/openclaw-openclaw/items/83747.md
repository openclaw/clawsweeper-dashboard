## Label Justifications

- `P2`: This is a real upgrade/recovery bug with a limited affected set: Linux root-owned system-global installs recovering manually after updater EACCES.
- `impact:crash-loop`: The report is about gateway crash/restart/outage behavior during an upgrade package replacement window.

## Summary

Keep open: current main still restarts the managed gateway after an EACCES package-update failure, and its recovery guidance points operators toward sudo/manual npm without a stop-gateway step. The reporter's logs and workaround match the current source path, while current docs only say to restart after manual package-manager updates.

## What This Changes

Prevent root-owned system-global npm recovery from running the gateway against an OpenClaw/plugin package tree while npm is replacing files.

## Best Possible Solution

Make the EACCES/system-global recovery path fail closed in user guidance at minimum: tell operators to stop the managed gateway before sudo/manual package recovery, then reinstall, restart, and verify; only change auto-restart lifecycle if maintainers explicitly accept that outage tradeoff.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes at the source-path level, but not from a live rerun in this review: root-owned staged npm setup can fail with EACCES, the updater then restarts the managed gateway, and current hints do not tell the operator to stop it before sudo/manual recovery.

## Solution Assessment

Is this the best way to solve the issue? The best narrow fix is to make the EACCES recovery instructions and docs explicitly stop the gateway before manual system npm recovery; a broader lifecycle change to leave the service stopped after EACCES should be a maintainer choice.

Fix https://github.com/openclaw/openclaw/issues/83747 narrowly. When an npm/global package update fails at `global install stage` or another EACCES global-install step after OpenClaw has stopped a managed gateway, the recovery guidance must not lead operators into running sudo/manual npm while the gateway is active.
