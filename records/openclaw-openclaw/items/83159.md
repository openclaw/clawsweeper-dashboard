## Summary

Keep open: this is a fresh P1 Telegram delivery regression, and current main still has source paths where transcript-only OpenClaw assistant entries can participate in Telegram-visible finalization without a provider/model filter.

## What This Changes

Prevent Telegram from delivering or using `provider=openclaw, model=delivery-mirror` transcript-only assistant entries to edit or replace visible assistant replies.

## Best Possible Solution

Keep Telegram delivery mirrors for transcript hygiene, but filter transcript-only OpenClaw assistant messages out of every Telegram-visible finalization, redelivery, and latest-transcript candidate path before any edit or send decision.

## Reproduction Assessment

No high-confidence runnable reproduction is available yet. The issue has detailed production timing evidence, and current source inspection shows the likely unfiltered delivery-mirror path.

## Solution Assessment

Yes, the structural filter direction is the best fix shape. Same-text mirror dedupe is only a safety net and cannot protect against the reported earlier-turn mirror entries.
