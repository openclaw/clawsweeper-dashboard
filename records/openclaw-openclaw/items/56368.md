## What This Changes

The issue requests a ClawHub login hint when unauthenticated skill search/install requests receive a 429 rate-limit response.

## Best Possible Solution

Keep the shared ClawHub error-builder implementation that covers search, detail, package downloads, and skill downloads consistently.

Yes. The current shared `buildClawHubError()` path is narrower and more maintainable than CLI-only formatting, and it solves the central request without adding config surface.
