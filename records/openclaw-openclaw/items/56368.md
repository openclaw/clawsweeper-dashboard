## Summary

Close as implemented: current main and release v2026.5.18 now add an unauthenticated ClawHub 429 sign-in hint in the shared ClawHub error path, with regression coverage for unauthenticated, authenticated, and headerless 429 responses.

## What This Changes

The issue requests a clearer ClawHub login hint when unauthenticated skill search/install requests receive a 429 rate-limit response.

## Best Possible Solution

Keep the centralized ClawHub request-error implementation that applies consistent unauthenticated 429 guidance across skill search, skill detail/list calls, and archive downloads.

## Reproduction Assessment

Yes. A focused source-level reproduction is covered by the current tests: clear ClawHub auth, make `searchClawHubSkills()` receive a mocked 429 response, and assert the thrown ClawHub error includes the sign-in hint.

## Solution Assessment

Yes. The current solution is the maintainable one because it formats 429 guidance once in the shared ClawHub request-error helper instead of duplicating CLI-specific error text across search and install paths.
