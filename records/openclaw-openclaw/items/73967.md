## Summary

Keep open. Current main already forwards Discord send components, but it still does not advertise a Discord `components` schema and the PR's added edit path targets the pre-split messaging runtime, so the branch needs adaptation before merge.

## What This Changes

The PR adds a Discord message-tool `components` schema contribution, forwards top-level components through Discord send/edit actions, and adds focused Discord tests.

## Best Possible Solution

Rebase or replace the branch against current main so schema discovery matches supported Discord message actions and component edits are implemented in `runtime.messaging.messages.ts` plus `runtime.messaging.runtime.ts`, or leave edits out of this change.

## Reproduction Assessment

Yes. Static inspection of current main gives a high-confidence path: send components are now forwarded, but discovery still has no schema contribution and component edits still require text-only handling.

## Solution Assessment

No. The object-shaped schema direction matches the OpenClaw Discord component spec, but the edit support is aimed at the old monolithic runtime and needs to move into the current split runtime layout.
