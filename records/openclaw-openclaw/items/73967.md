## Summary

Keep open: the Discord component schema/send fix is still useful, but the latest edit support targets the old monolithic Discord runtime and lacks real behavior proof, so the patch should not merge as-is.

## What This Changes

The PR adds a Discord message-tool `components` schema, forwards top-level components through send/edit actions, and adds tests around Discord component sends and edits.

## Best Possible Solution

Rebase or replace the branch on current main, keep the schema/send forwarding that matches the docs, and implement component edits in the split messaging runtime with real Discord proof.

## Reproduction Assessment

Yes. Source inspection on current main shows docs advertise Discord `components` payloads while configured discovery still returns no schema; the edit path requires content and does not route components through `editDiscordComponentMessage`.

## Solution Assessment

No. The object-shaped schema and send forwarding fit the documented OpenClaw component spec, but the edit support must move into `runtime.messaging.messages.ts` plus `runtime.messaging.runtime.ts`, or be left out until it is proven.
