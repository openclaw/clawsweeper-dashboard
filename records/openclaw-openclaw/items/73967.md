## What This Changes

The branch adds a Discord message-tool `components` schema, forwards top-level components through send/edit actions, and adds focused Discord tests.

## Best Possible Solution

Rebase or replace this branch against current main, keep the Discord component schema/send forwarding that matches docs, and implement component edits in the split messaging runtime only with real Discord proof.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: Discord component sends are forwarded, but configured Discord discovery still contributes no `components` schema and component edits still use the content-only edit handler.

## Solution Assessment

No. The object-shaped schema matches the documented OpenClaw Discord component spec, but the edit support is aimed at the old monolithic runtime and must move into `runtime.messaging.messages.ts` plus `runtime.messaging.runtime.ts`, or be left out.
