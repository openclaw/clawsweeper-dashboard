## Summary

Keep open: current main still has the abandoned guarded streamed-response cleanup gap, and the macOS plus Linux reports match the source path. I found no merged OpenClaw fix or canonical duplicate that owns this leak.

## What This Changes

The issue requests reliable cleanup of guarded Anthropic streamed responses when a consumer abandons the stream before EOF or explicit cancel.

## Best Possible Solution

Add shared exact-once abandonment cleanup for partially consumed guarded streamed responses, with focused regression coverage for the no-EOF/no-cancel path.

## Reproduction Assessment

Yes, at source level: return a streamed response through buildGuardedModelFetch, read one chunk from response.body, then drop it without calling body.cancel(); current main has no path from that state to finalize(). I did not run the long-duration socket-count reproduction.

## Solution Assessment

Yes. A generic fix in buildManagedResponse is the narrowest maintainable direction because that wrapper owns result.release and covers Anthropic plus other guarded streamed provider consumers.
