## Summary

Keep open. Current main and the latest stable release have reduced the reported Anthropic-specific risk by replacing the Anthropic SDK stream iterator with OpenClaw's own parser that drains SSE bodies to EOF, but the shared managed-response wrapper still has no cleanup path for an abandoned body that reaches neither EOF nor explicit cancel.

## What This Changes

The issue reports streamed Anthropic model calls leaking guarded-fetch undici sockets when a managed response body is abandoned without EOF or cancel.

## Best Possible Solution

Add an idempotent managed-response cleanup safety net for abandoned streaming bodies while preserving the current Anthropic parser's EOF-draining behavior.

## Reproduction Assessment

Yes, by source inspection: a focused test can mock a streaming guarded response, read one chunk from buildGuardedModelFetch's returned body, then abandon it without EOF or cancel and observe that result.release is never called. I did not run that test because this pass is read-only.

## Solution Assessment

Unclear on the exact mechanism, but the narrow direction is right: fix cleanup at the managed response boundary, not with a cron restart workaround. A deterministic cleanup path is preferable where possible; a FinalizationRegistry fallback must be idempotent and must not retain its own target.
