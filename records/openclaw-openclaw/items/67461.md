## Summary

Keep open: current main still lacks an exact-once cleanup path when a guarded streamed response is partially read and then abandoned, and the macOS report plus Linux confirmation match that source gap. The linked Nessie FinalizationRegistry work is useful prior art, but it is external and no OpenClaw PR or canonical issue appears to own the remaining cleanup.

## What This Changes

The issue reports leaked undici sockets from abandoned streamed Anthropic API responses because managed response cleanup only runs on EOF, read error, or explicit cancel.

## Best Possible Solution

Add shared exact-once cleanup for abandoned partially consumed guarded streamed responses, with regression coverage for the no-EOF/no-cancel path and no new config knob.

## Reproduction Assessment

Yes, at source level: return a streamed response through buildGuardedModelFetch, read part of response.body, then abandon it without EOF or body.cancel; current main has no path from that state to finalize. I did not run the long-duration socket-count reproduction.

## Solution Assessment

Yes. A generic fix in buildManagedResponse is the narrowest maintainable direction because that wrapper owns result.release and covers Anthropic plus other guarded streamed provider consumers.
