## Summary

Keep open: current main still lacks abandoned-stream cleanup in the shared guarded model response wrapper, and the issue has credible macOS plus Linux reports of gateway FD/RSS growth leading to failures.

## What This Changes

The issue reports leaked guarded streaming HTTP response resources when an Anthropic streamed response is abandoned before EOF or explicit cancellation.

## Best Possible Solution

Add exact-once abandoned-stream cleanup in `buildManagedResponse` with focused regression coverage, without changing provider identity, auth, proxy policy, or adding a config knob.

## Reproduction Assessment

Yes, at source level: return a streamed response through `buildGuardedModelFetch`, read part of `response.body`, then abandon it without EOF or `body.cancel()`. Current main has no path from that state to `finalize()`, though I did not run the long-duration socket-count repro.

## Solution Assessment

Yes. The narrow maintainable fix is a generic exact-once cleanup safety net in `buildManagedResponse`, because that wrapper owns `result.release`; fixing only the Anthropic consumer would leave the shared guarded-fetch contract incomplete.
