## What This Changes

The issue reports leaked undici sockets when streamed Anthropic responses are abandoned before EOF or explicit cancellation.

## Best Possible Solution

Add shared exact-once cleanup for abandoned partially consumed guarded streamed responses, with regression coverage for the no-EOF/no-cancel path and without adding a config knob.

## Reproduction Assessment

Yes, at source level: return a streamed response through buildGuardedModelFetch, read part of response.body, then abandon it without EOF or body.cancel; current main has no path from that state to finalize. I did not run the long-duration socket-count reproduction.

## Solution Assessment

Yes. A generic fix in `buildManagedResponse` is the narrowest maintainable direction because that wrapper owns `result.release()` and covers Anthropic plus other guarded streamed provider consumers.
