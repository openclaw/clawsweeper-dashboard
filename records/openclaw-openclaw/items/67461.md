## Summary

Keep open: current main still lacks a shared abandoned-stream cleanup path for managed provider responses, and the issue has concrete macOS/Linux live evidence plus source-level reproduction evidence.

## What This Changes

Issue requests a reliability fix for leaked gateway sockets when streamed Anthropic responses are abandoned before the managed response wrapper finalizes the guarded fetch release hook.

## Best Possible Solution

Land a focused shared provider-transport cleanup so `result.release()` runs exactly once when a streamed managed response is abandoned before EOF, with Anthropic consumer cleanup audited as a secondary hardening step.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to create a managed streamed response through `buildGuardedModelFetch`, read part of `response.body`, then drop it without `body.cancel()`; current main has no path that reaches `finalize()` in that case.

## Solution Assessment

Yes. A generic safety net in `buildManagedResponse`, backed by focused tests, is the narrowest maintainable fix because single-provider parser changes would leave other SDK/provider consumers exposed.
