## Summary

Keep open: current main and the latest shipped release still lack an abandoned-stream cleanup path in `buildManagedResponse`, and the macOS report plus Linux confirmation make this a credible gateway availability bug.

## What This Changes

The issue asks OpenClaw to release guarded provider stream resources when a streamed Anthropic/LLM response is abandoned before EOF or explicit cancel.

## Best Possible Solution

Add exact-once abandoned-stream cleanup in `buildManagedResponse` with focused regression coverage, without changing provider identity, auth, proxy policy, or adding a config knob.

## Reproduction Assessment

Yes, at source level: return a streamed response through `buildGuardedModelFetch`, read part of `response.body`, then abandon it without EOF or `body.cancel()`. Current main has no path from that state to `finalize()`, though I did not run the long-duration socket-count repro.

## Solution Assessment

Yes. The narrow maintainable fix belongs in `buildManagedResponse`, because that wrapper owns the guarded fetch `release` callback and consumer-only Anthropic parsing cannot protect every guarded stream consumer.
