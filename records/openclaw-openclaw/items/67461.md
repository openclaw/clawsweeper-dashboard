## Summary

Keep open: current main and the latest release still rely on EOF, read error, or explicit cancel to call the guarded fetch release hook, so the abandoned partially consumed stream path remains valid.

## What This Changes

Ensure guarded streamed model responses release their undici dispatcher/socket when a consumer abandons the stream before EOF or cancel.

## Best Possible Solution

Add shared exact-once cleanup for abandoned partially consumed guarded streamed responses, with focused regression coverage for the no-EOF/no-cancel path and no new config knob.

## Reproduction Assessment

Yes, at source level: return a streamed response through buildGuardedModelFetch, read part of response.body, then abandon it without EOF or body.cancel. Current main has no path from that state to finalize.

## Solution Assessment

Yes. A generic fix in buildManagedResponse is the narrowest maintainable direction because that wrapper owns result.release and covers guarded streamed provider consumers.
