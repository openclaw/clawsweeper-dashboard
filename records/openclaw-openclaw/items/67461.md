## Summary

Keep open: current main mitigates the original Anthropic SDK path by using OpenClaw's own Anthropic SSE parser, but `buildManagedResponse` still has no cleanup path for a partially read guarded stream that is abandoned without EOF or `body.cancel()`. The macOS report, Linux confirmation, Undici contract, and current source all point to a real reliability bug that still needs a narrow core fix.

## What This Changes

Fix guarded streamed response cleanup so abandoned Anthropic/undici streams release sockets instead of leaking file descriptors.

## Best Possible Solution

Add exact-once abandoned-stream cleanup in `buildManagedResponse`, with deterministic regression coverage for partial read followed by no EOF and no cancel, without adding a config knob or changing provider/run identity behavior.

## Reproduction Assessment

Yes, at source level: return a streamed response through `buildGuardedModelFetch`, read part of `response.body`, then abandon it without EOF or `body.cancel()`; current `buildManagedResponse` has no path from that state to `finalize()`. I did not run the long-duration socket-count reproduction.

## Solution Assessment

Yes. A generic exact-once cleanup in `buildManagedResponse` is the narrow maintainable fix because that wrapper owns `result.release`; the native Anthropic parser and the adjacent abort-teardown PR only cover specific consumer or abort paths.
