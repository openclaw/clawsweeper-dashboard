## Summary

Keep open: current main still releases managed provider streams only on EOF, read error, or explicit cancel, so the abandoned streamed-response leak described by the macOS and Linux reports remains source-reproducible and narrow enough for a focused fix.

## What This Changes

Ensure streamed Anthropic/provider HTTP responses release guarded fetch resources when consumers abandon the stream before EOF or explicit cancel.

## Best Possible Solution

Land a focused shared provider-transport cleanup so `result.release()` runs exactly once when a managed streamed response is abandoned before EOF, with regression coverage for the partial-read/no-cancel path.

## Reproduction Assessment

Yes, from source inspection. A high-confidence path is to create a managed streamed response through `buildGuardedModelFetch`, read one chunk from `response.body`, then drop it without `body.cancel()`; current `buildManagedResponse` has no path that reaches `finalize()` in that case.

## Solution Assessment

Yes. A generic safety net in `buildManagedResponse`, backed by focused provider-transport tests, is the narrowest maintainable fix because provider-specific parser cleanup would leave other SDK consumers exposed.
