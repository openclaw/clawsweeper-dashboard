## Summary

Keep open: current main still lacks a release path for a partially consumed managed provider stream that is dropped without `cancel()` or EOF, and the issue discussion includes credible macOS and Linux long-running gateway evidence.

## What This Changes

The issue asks OpenClaw to stop leaking guarded undici dispatcher sockets when streamed Anthropic responses are abandoned before normal stream finalization.

## Best Possible Solution

Add a shared provider-transport abandonment cleanup so guarded fetch resources and local-service leases are released exactly once for partially consumed streamed responses, with regression coverage for partial-read/no-cancel behavior.

## Reproduction Assessment

Yes, source-level: return a streamed response through `buildGuardedModelFetch`, read one chunk from `response.body`, then drop it without calling `body.cancel()`; current `buildManagedResponse` has no path that reaches `finalize()` in that case. I did not run the long-duration live gateway socket-count reproduction.

## Solution Assessment

Yes. A generic fix in `buildManagedResponse` is the narrowest maintainable solution because it owns `result.release()` and covers Anthropic plus any other guarded streamed provider consumer.
