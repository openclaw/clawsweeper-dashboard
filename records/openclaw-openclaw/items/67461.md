## Summary

Keep open: current main still lacks an abandoned-stream cleanup path in the managed provider response wrapper, and there is no canonical OpenClaw PR or issue superseding this leak report. The reporter and a second user provided credible v2026.4.14/15 FD/RSS evidence, while current source still releases guarded-fetch resources only on EOF, error, or explicit cancel.

## What This Changes

The issue asks OpenClaw to stop leaking guarded-fetch/undici resources when streamed Anthropic responses are abandoned before EOF or explicit cancel.

## Best Possible Solution

Add a robust cleanup path in the managed streamed response wrapper, with focused coverage proving guarded dispatcher release and local service lease release happen exactly once when a consumer abandons the stream.

## Reproduction Assessment

No high-confidence live reproduction on current main. The report has credible shipped-version FD evidence, and the current source still exposes the missing abandoned-stream release path, but I did not run the long-lived gateway plus Anthropic stream workload needed to observe FD growth directly.

## Solution Assessment

Unclear as stated. A FinalizationRegistry backstop is plausible, but the maintainable fix should prove deterministic cleanup where possible and treat GC finalization or an idle guard as a last-resort safety net, with regression coverage for abandonment and double-release behavior.
