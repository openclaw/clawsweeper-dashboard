## Summary

Keep open: current main still has the managed provider-stream cleanup gap described in the report, and the macOS/Linux discussion provides credible live evidence that the missing abandoned-stream release path matters in long-running gateways.

## What This Changes

Fix gateway provider streamed-response cleanup so abandoned Anthropic/Vertex streams release guarded undici dispatcher/socket resources instead of accumulating FDs and RSS.

## Best Possible Solution

Add a generic shared provider-transport abandonment cleanup so `result.release()` runs exactly once for partially consumed streamed responses, with focused regression coverage for partial-read/no-cancel behavior.

## Reproduction Assessment

Yes. A high-confidence source path is to return a streamed response through `buildGuardedModelFetch`, read one chunk from `response.body`, then drop it without `body.cancel()`; current `buildManagedResponse` has no path that reaches `finalize()` in that case.

## Solution Assessment

Yes. A generic safety net in `buildManagedResponse`, backed by provider-transport tests, is the narrowest maintainable fix because Anthropic-only parser cleanup would leave other guarded SDK stream consumers exposed.

Current main's `buildManagedResponse` in `src/agents/provider-transport-fetch.ts` only calls `result.release()` on EOF, read error, or explicit `body.cancel()`; add a generic cleanup path so release runs exactly once when a streamed `Response` is abandoned after partial reads, without releasing active streams prematurely.
