## Summary

Keep this open. Current main at 231e5c618f8ae700d51d080e4182a57588da888b still has the reported abandoned-stream gap: the managed provider response releases the guarded dispatcher only on no body, EOF, read error, or explicit cancel, while Anthropic Messages still releases only its reader lock on generator cleanup and Anthropic Vertex delegates streaming into dependency code without a local safety net.

## What This Changes

The issue requests a gateway/provider transport fix so streamed Anthropic responses release undici dispatcher sockets even when the stream consumer abandons the response before EOF or explicit cancellation.

## Best Possible Solution

Land a focused reliability fix in the shared provider response wrapper so result.release runs exactly once when a streamed response is abandoned before EOF, and add explicit cancellation/draining in Anthropic stream consumers where feasible. Keep the fix generic to provider transport, preserve SSRF/dispatcher policy behavior, and add regression coverage for abandoned streamed bodies.

## Reproduction Assessment

Yes. A source-level reproduction is clear: buildGuardedModelFetch can return a managed streamed Response, read part of body, then drop it without body.cancel(); current buildManagedResponse never reaches finalize in that path. The issue also includes live macOS and Linux FD/RSS evidence, though this review did not run a multi-hour live provider test.

## Solution Assessment

Yes. The best repair direction is a shared buildManagedResponse safety net plus focused Anthropic consumer cleanup; fixing only one Anthropic parser would leave other SDK/provider consumers exposed, while changing fetch-guard globally would risk broader network semantics.
