## What This Changes

The issue requests fixing gateway socket and FD leaks from abandoned streamed Anthropic responses by ensuring managed response cleanup always releases guarded fetch resources.

## Best Possible Solution

Add exact-once abandonment cleanup to `buildManagedResponse` for guarded streamed provider responses, with focused regression coverage for partial reads that are dropped without EOF or explicit cancellation.

## Reproduction Assessment

Yes, at source level: return a streamed response through `buildGuardedModelFetch`, read one chunk from `response.body`, then drop it without `body.cancel()` or EOF; current main has no path from that state to `finalize()`. I did not run the long-duration gateway socket-count reproduction.

## Solution Assessment

Yes. A generic fix in `buildManagedResponse` is the narrowest maintainable solution because that wrapper owns `result.release()` and local service lease cleanup across guarded streamed provider consumers.

## Repair Work Prompt

Fix the guarded streaming response leak reported in https://github.com/openclaw/openclaw/issues/67461. In `src/agents/provider-transport-fetch.ts`, ensure `buildManagedResponse` releases `result.release()` and the local service lease exactly once when a consumer partially reads the wrapped `ReadableStream` and then abandons it without EOF or `body.cancel()`. Keep the fix generic to guarded provider transport; do not add a config option or an Anthropic-only workaround unless the generic wrapper still needs a parser-side cancellation call. If using `FinalizationRegistry`, avoid holdings that retain the target, unregister on normal finalize, and preserve existing EOF/error/cancel behavior. Add focused regression coverage in `src/agents/provider-transport-fetch.test.ts`; add Anthropic stream coverage in `src/agents/anthropic-transport-stream.test.ts` only if needed to prove generator abandonment. Add a changelog fix entry if the repo policy expects one for this user-visible reliability fix.
