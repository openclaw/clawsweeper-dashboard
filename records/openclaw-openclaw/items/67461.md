## Summary

Keep open: current main still lacks a cleanup path for a partially consumed guarded provider stream that is dropped without EOF or explicit cancellation. The issue has credible macOS/Linux runtime evidence, and the linked Nessie work is useful corroboration but not an OpenClaw fix.

## What This Changes

The issue reports leaked gateway sockets when streamed Anthropic responses are partially consumed without EOF or cancellation.

## Best Possible Solution

Add shared provider-transport abandonment cleanup for partially consumed streamed responses, with regression coverage proving guarded fetch resources and local-service leases are released exactly once.

## Reproduction Assessment

Yes, at source level: return a streamed response through `buildGuardedModelFetch`, read one chunk from `response.body`, then drop it without calling `body.cancel()`; current `buildManagedResponse` has no path that reaches `finalize()`. I did not run the long-duration live gateway socket-count reproduction.

## Solution Assessment

Yes. A generic fix in `buildManagedResponse` is the narrowest maintainable solution because it owns the guarded fetch `release()` handle and covers Anthropic plus any other guarded streamed provider consumer.
