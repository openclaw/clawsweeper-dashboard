## Summary

Keep open: current main still lacks a cleanup path for a partially read managed provider stream that is abandoned without EOF or `body.cancel()`. The reporter's macOS FD leak, the Linux confirmation, the Undici contract, and current source all support a narrow core reliability fix.

## What This Changes

The issue asks for `buildManagedResponse` to release guarded Undici transport resources when a streamed provider response is abandoned before EOF or explicit cancel.

## Best Possible Solution

Add exact-once abandoned-stream cleanup in `buildManagedResponse`, with deterministic regression coverage for partial read followed by no EOF and no cancel.

## Reproduction Assessment

Yes, at source level: return a streamed response through `buildGuardedModelFetch`, read part of `response.body`, then abandon it without EOF or `body.cancel()`. I did not run the long-duration socket-count reproduction.

## Solution Assessment

Yes. A generic exact-once cleanup in `buildManagedResponse` is the narrow maintainable fix because that wrapper owns `result.release` across provider transports.
