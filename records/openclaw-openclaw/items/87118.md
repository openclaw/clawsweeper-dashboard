## What This Changes

This PR changes Claude CLI skill handling to omit the OpenClaw skills prompt when a Claude Code plugin can carry skills, adds regression tests for plugin and fallback cases, and updates the CLI backend docs.

## Best Possible Solution

Land a plugin-first fix that omits `<available_skills>` only when the Claude skills plugin is known to be prepared, while preserving the prompt catalog fallback for materialization failures.

## Reproduction Assessment

Yes at source level: current main builds the OpenClaw skills prompt into the CLI system prompt while the Claude CLI execution path also prepares the native skills plugin. I did not run the live Claude CLI flow in this read-only review.

## Solution Assessment

No, not yet: the plugin-first direction is right, but a file-access precheck is not the best boundary for suppressing the fallback. The safer fix is to key suppression to the actual plugin preparation result or an equivalent shared materialization decision.
