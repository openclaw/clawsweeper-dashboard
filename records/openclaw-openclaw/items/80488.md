## Summary

Keep open: the PR is addressing a real existing bug, but the latest patch still does not pass the resolved model API through the normal `getApiKeyForModel` auth entry point, so the reported runtime path can remain broken.

## What This Changes

The PR adds per-model `api`/`baseUrl` handling to provider hook resolution and model transport selection, plus auth and model-resolution regression tests.

## Best Possible Solution

Carry the effective resolved model `api` into every provider synthetic-auth hook discovery path from `getApiKeyForModel`, and cover that production entry point with a focused regression test.

## Reproduction Assessment

Yes, source-reproducible: current main accepts and documents per-model `api`/`baseUrl`, but auth hook refs and `getApiKeyForModel` still use only the provider-level API. I did not run a live Docker/router reproduction in this read-only pass.

## Solution Assessment

No. Adding `modelApi` to the lower helper is the right direction, but this PR must also thread `params.model.api` from `getApiKeyForModel` and make the new regression test exercise that production path.
