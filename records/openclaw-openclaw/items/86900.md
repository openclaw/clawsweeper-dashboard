## Summary

Keep open: this is a useful compaction fix for the linked token-burn bug, but the current patch still treats partial summaries as generic fallbacks and can open the breaker after useful progress; the contributor proof is also mock/test-script only.

## What This Changes

The PR changes compaction fallback results to carry metadata, adds a staged circuit breaker in `summarizeInStages`, updates related tests, and adds a local proof script.

## Best Possible Solution

Preserve the circuit-breaker idea, but make only true generic no-progress fallbacks count toward the breaker and add real redacted runtime proof before merge.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: current `summarizeInStages` processes every split and each failed split can run `retryAsync` for 3 attempts. I did not run a live current-main reproduction in this read-only review.

## Solution Assessment

No, not yet: the circuit breaker is a reasonable narrow fix, but this implementation counts partial-summary progress as generic fallback. The safer fix is to count only no-progress generic fallbacks and prove the runtime path with a real summarizer failure.

- **[P2] Keep partial summaries out of the generic fallback count:** `src/agents/compaction.ts:513`
  - body: `partialSummaryFallback` is set when `summarizeChunks` already produced useful summary text before a later chunk failed, but this branch marks it as `isGenericFallback: true`. Mark partial summaries as non-generic and add a regression that they reset the breaker counter.
