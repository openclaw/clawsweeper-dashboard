## Summary

Keep open for maintainer review: current main still summarizes every staged split without a breaker, while this PR adds typed fallback metadata, a two-generic-fallback stop condition, and focused proof. I found no blocking code defect, but maintainers should explicitly accept the session-state tradeoff before merge.

## What This Changes

The PR changes built-in staged compaction so `summarizeWithFallback` returns typed fallback metadata, `summarizeInStages` stops after two generic fallback splits, and focused compaction tests cover the behavior.

## Best Possible Solution

Land the typed breaker if maintainers accept the two-fallback session-state policy; otherwise adjust the threshold or returned summary shape in this PR before merge.

## Reproduction Assessment

Yes. Current main shows the staged split loop calls `summarizeWithFallback` for every chunk and then merges, while the PR tests demonstrate repeated generic fallback calls and the new early-stop behavior.

## Solution Assessment

Yes, as an implementation shape: typed fallback metadata is safer than string-prefix control flow and keeps the fix inside built-in compaction. The remaining question is whether the fixed two-fallback policy is the right product default.
