## Summary

Keep this PR open for maintainer review: the patch is focused, source-backed, and has sufficient terminal proof, but it intentionally changes outage-time session summary shape by returning collected split fallback text instead of forcing a final merge.

## What This Changes

The PR changes staged compaction to return a typed fallback result, opens a circuit breaker after two consecutive generic fallback splits, and updates regression tests for the new result shape.

## Best Possible Solution

Land the typed circuit breaker only if maintainers accept the bounded-token-burn tradeoff; otherwise adjust the open-breaker path to preserve a more structured fallback before merge.

## Reproduction Assessment

Yes. A source-level repro is clear: make split-plan compaction call `generateSummary` with persistent non-timeout failures, and current main retries per chunk across each split before falling back.

## Solution Assessment

Yes, as an acceptable mitigation if maintainers accept the tradeoff. The typed fallback boolean is narrower and safer than string-prefix control flow; the only better alternative would be a different product choice for the open-breaker fallback shape.
