## Summary

Keep open: the breaker is now in the right split loop, but the diff still falls through to a merge summarization call after the breaker opens, so an unavailable summarizer can burn more retries and discard collected partial summaries. The supplied proof is still a simulation rather than a real OpenClaw compaction run.

## What This Changes

The PR adds a split-loop circuit breaker to `summarizeInStages` and a colocated Vitest file for unavailable summarizer cases.

## Best Possible Solution

Track when the breaker opens and return the collected partial or fallback summaries without another model call, then add call-count regression coverage and real compaction proof.

## Reproduction Assessment

Yes from source: current `summarizeInStages` retries each split through `summarizeWithFallback`, and non-terminal summarizer failures return generic fallbacks per split. I did not run a live provider repro in this read-only review.

## Solution Assessment

No. The breaker is in the right loop, but the implementation should return collected summaries when it opens instead of immediately calling the unavailable summarizer again for the merge step.
