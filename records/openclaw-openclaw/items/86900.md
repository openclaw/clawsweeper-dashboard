## Summary

Keep this PR open. The patch addresses the linked compaction token-burn bug with typed fallback state and focused coverage, but it intentionally changes session compaction behavior during sustained summarizer outages, so maintainers still need to accept or tune that session-state tradeoff before merge.

## What This Changes

The PR changes compaction fallback results from a plain string to typed metadata, adds a two-generic-fallback circuit breaker in `summarizeInStages`, and adds focused compaction regression tests.

## Best Possible Solution

Land a typed compaction circuit breaker only after maintainers explicitly accept the two-generic-fallback cutoff or adjust the threshold/fallback behavior to match the desired session-state policy.

## Reproduction Assessment

Yes, from source inspection. Current `summarizeInStages()` still calls `summarizeWithFallback()` for every planned split, and each split can still reach `summarizeChunks()` with a three-attempt retry budget before returning the generic fallback string.

## Solution Assessment

Yes as an implementation shape, but not yet as a final merge decision. Typed fallback metadata is the narrow maintainable way to avoid string-prefix control flow; the remaining question is whether the two-fallback cutoff is the desired session-state policy.
