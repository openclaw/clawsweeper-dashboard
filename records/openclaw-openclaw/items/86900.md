## Summary

Keep open: the circuit-breaker direction matches the linked bug, but the current patch still treats useful partial summaries as generic fallbacks and the supplied proof is mocked Vitest output rather than real compaction behavior.

## What This Changes

The PR changes compaction summarization to return text plus fallback metadata, stops staged summarization after two generic fallback splits, updates compaction tests, and adds a proof script.

## Best Possible Solution

Land the circuit breaker only after partial-summary results are distinguished from generic fallback results and real compaction proof shows token burn is bounded without dropping recoverable session context.

## Reproduction Assessment

Yes at source level: current main iterates every split and retryAsync can attempt generateSummary three times per split when non-timeout failures persist. I did not run a live reproduction in this read-only review.

## Solution Assessment

No, not yet: putting the breaker in summarizeInStages is the right area, but the implementation must not count recovered partial summaries as generic fallback failures and still needs real behavior proof.
