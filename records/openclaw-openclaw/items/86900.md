## Summary

Keep this PR open: the linked bug is valid and the circuit-breaker direction is useful, but the current patch regresses current-main compaction planning and changes an exported fallback helper contract while still relying on mock-only proof for a session-state tradeoff.

## What This Changes

The PR adds typed fallback metadata, stops staged compaction after two generic fallback split results, skips the merge pass when the breaker opens, copies compaction-planning helpers into `compaction.ts`, and adds a focused circuit-breaker test file.

## Best Possible Solution

Preserve the current planning worker and public string-return helper, add internal fallback metadata only where the staged breaker needs it, update the existing fallback tests, and land after maintainers accept the cutoff with real proof or an explicit override.

## Reproduction Assessment

Yes, source-reproducible: current main's staged compaction loops over every planned split and each split delegates to retry-backed summarization, so persistent non-terminal failures multiply attempts per split. I did not run a live unavailable-summarizer scenario.

## Solution Assessment

No as written; the breaker should be added around the current worker-backed staged plan while preserving the existing string-return fallback helper. Copying planning back into `compaction.ts` and changing the exported return shape is not the narrowest maintainable fix.
