## What This Changes

The PR changes compaction fallback results to carry text plus fallback metadata, adds a two-generic-fallback breaker in `summarizeInStages`, updates compaction tests, and adds an executable proof helper script.

## Best Possible Solution

Keep the breaker scoped to true no-progress generic fallbacks, model partial-summary progress separately, add focused regression coverage for that case, and require real OpenClaw runtime proof before merge.

## Reproduction Assessment

Yes, at source level: current main retries `generateSummary` inside `summarizeChunks` and `summarizeInStages` calls `summarizeWithFallback` for each split, so persistent non-timeout failures repeat the retry budget. I did not run a live repro in this read-only review.

## Solution Assessment

No, not yet. The breaker direction is appropriate, but partial-summary fallback must not count as no-progress generic fallback, and the external PR gate still needs real runtime proof.
