## Summary

Keep open. The PR addresses a valid source-reproducible compaction retry-burn bug, but it still needs real OpenClaw behavior proof and has a session-state regression where useful partial summaries are marked as generic fallback and can trip the breaker early.

## What This Changes

The PR changes `summarizeWithFallback` to return text plus fallback metadata, adds a two-fallback circuit breaker in `summarizeInStages`, and adds/updates compaction tests.

## Best Possible Solution

Keep the breaker scoped to truly generic no-progress fallbacks, preserve partial-summary progress as non-generic or as a separate status, add a regression test for that path, and require real OpenClaw runtime proof before merge.

## Reproduction Assessment

Yes, at source level: current main retries `generateSummary` inside `summarizeChunks` and `summarizeInStages` calls `summarizeWithFallback` for each split, so persistent non-timeout failures repeat split-level retry budgets. I did not run a live repro because this review was read-only.

## Solution Assessment

No, not yet. The circuit-breaker direction is appropriate, but the patch should not classify useful partial summaries as generic fallback and should include real runtime proof before it is the best merge path.
