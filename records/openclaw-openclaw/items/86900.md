## What This Changes

The PR adds a two-consecutive-generic-fallback cutoff in `summarizeInStages` plus a new regression test for the unavailable-summarizer split path.

## Best Possible Solution

Land a typed fallback-state circuit breaker with regression coverage for successful prefix-like summaries and explicit live proof or maintainer override for the outage tradeoff.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes from source: current split-mode compaction calls `summarizeWithFallback` for each chunk, while `summarizeChunks` wraps each chunk summary in retries; I did not run a live unavailable-summarizer scenario.

## Solution Assessment

Is this the best way to solve the issue? No as written. The circuit-breaker direction is plausible, but the implementation should use typed fallback state instead of inspecting summary text and needs proof or maintainer acceptance for the context-retention tradeoff.
