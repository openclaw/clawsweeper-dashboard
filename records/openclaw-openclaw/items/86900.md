## Summary

Keep open: current main and the latest release still lack this breaker, and the patch looks correct, but the breaker deliberately changes persisted compaction state during summarizer outages and needs maintainer acceptance before merge.

## What This Changes

Adds typed fallback results to compaction summarization, opens a staged compaction circuit breaker after two consecutive generic fallbacks, and updates focused compaction tests.

## Best Possible Solution

Land the PR if maintainers accept the breaker-open summary behavior; otherwise adjust the breaker return shape or threshold before merge while keeping the typed fallback signal and regression tests.

## Reproduction Assessment

Yes by source inspection: current `summarizeInStages` on main loops over every planned split and then calls merge summarization, so persistent `generateSummary` failures repeatedly enter `summarizeWithFallback`. I did not run tests because this review is read-only.

## Solution Assessment

Yes, as an acceptable mitigation: typed fallback state is the right local signal and avoids string-prefix control flow. The open question is whether maintainers accept returning joined partial or generic summaries after two failures rather than attempting a final merge.
