## Summary

Keep open: the PR targets a real, source-reproducible compaction token-burn bug, but the current patch still marks useful partial summaries as generic fallbacks, so the breaker can prematurely stop later split summaries and preserve less session context. The contributor proof is also mock-only because it shows Vitest output with a mocked summarizer rather than a real OpenClaw compaction run.

## What This Changes

The PR changes compaction summarization to return fallback metadata, adds a staged-summarization circuit breaker after two generic fallback splits, updates compaction tests, and adds a proof script.

## Best Possible Solution

Use a run-scoped staged compaction breaker that trips only on true generic no-summary fallbacks, preserves partial summaries as successful context, skips the merge only after true generic failures, and is backed by focused tests plus real compaction proof.

## Reproduction Assessment

Yes, source-level: mock generateSummary to reject with a non-timeout provider error and call summarizeInStages with enough messages to split; current main spends retry attempts across split summaries and the merge. I did not run the repro because this review is read-only.

## Solution Assessment

No, not yet: the breaker location is plausible, but partial summaries need to be classified as useful summarized context rather than generic fallback. The narrow fix is to trip only on true generic fallback results and add regression coverage for partial-summary splits.
