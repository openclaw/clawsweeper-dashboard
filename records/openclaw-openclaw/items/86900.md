## Summary

Keep open: current main and the latest release still retry every staged compaction split, while this PR adds the missing typed fallback signal and breaker coverage. The patch looks correct, but merging still needs maintainer acceptance of the hard-coded two-generic-fallback session-state tradeoff.

## What This Changes

The PR changes compaction fallback results to carry typed generic-fallback metadata, stops staged summarization after two consecutive generic fallback splits, and adds focused regression tests.

## Best Possible Solution

Land the breaker once maintainers accept the two-generic-fallback outage policy, or tune the threshold/conditions with matching tests before merge.

## Reproduction Assessment

Yes from source, though I did not run tests in this read-only review. Current main's staged compaction calls summarizeWithFallback for each split, and summarizeChunks wraps model summarization in three retry attempts, so persistent failures repeat across splits.

## Solution Assessment

Unclear as a final product policy, acceptable as code shape. The typed fallback result avoids brittle string matching and the tests cover the intended behavior, but the fixed threshold is a session-state tradeoff maintainers should explicitly accept.
