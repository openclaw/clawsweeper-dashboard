## Summary

Keep open: the linked WSL2 gateway restart regression has useful logs and the PR targets the right shutdown area, but this patch is not mergeable because `ChildProcess` does not expose `detached` at runtime and the branch still lacks after-fix real behavior proof.

## What This Changes

The PR changes qa-lab gateway child shutdown to guard Unix negative-PID group signaling with `child.detached` and otherwise fall back to direct `child.kill`.

## Best Possible Solution

Carry the spawn-time detached state into qa-lab shutdown or reuse the explicit detached-option process-tree pattern, preserving group-kill for actually detached children and direct-kill only for non-detached children, then prove both with tests and WSL2 logs.

## Reproduction Assessment

Yes for the patch defect: Node v24.16.0 shows `child.detached` is absent even for a detached spawn, and current main only knows detachment at the spawn sites. I did not run the full WSL2 systemd gateway smoke for the original report.

## Solution Assessment

No; the patch is in the right owner area, but reading `child.detached` is not the best fix because the detached state must be carried from spawn time or passed through an explicit process-tree option.
