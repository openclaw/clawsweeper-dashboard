## Summary

Keep this PR open, but it is not merge-ready: the proposed guard reads `child.detached`, which is absent on a Node 24.16.0 `ChildProcess`, so the patch would skip group termination for the detached Unix qa-lab gateway children it is trying to protect.

## What This Changes

The PR changes qa-lab gateway child shutdown so process-group signaling is guarded by a detached check and otherwise falls back to direct child signaling.

## Best Possible Solution

Carry the spawn-time detached mode into qa-lab shutdown, or reuse the existing explicit detached-option process-tree pattern, then prove detached and non-detached teardown plus the WSL2/Node 24 scenario.

## Reproduction Assessment

Yes for the patch defect: current main spawns qa-lab gateway children as detached on Unix, and a Node 24.16.0 runtime check shows the proposed `child.detached` property is absent. I did not run the full WSL2 systemd reproduction.

## Solution Assessment

No; the patch is in the right shutdown area, but reading `child.detached` is not the best fix because detached state must be carried from spawn time or passed through an explicit kill-tree option.
