## Summary

Keep open. Current main still rejects cross-agent `sessions_send` under non-`all` visibility before A2A policy is evaluated, while this PR edits the stale pre-SDK implementation path and lacks real behavior proof.

## What This Changes

The branch changes session access checks and tests so cross-agent `sessions_send` under `tree` visibility can bypass visibility when an A2A allow policy permits the target.

## Best Possible Solution

Rebase or replace the branch with a current-main change in `src/plugin-sdk/session-visibility.ts` that applies a send-only A2A exception, preserves list/history/status isolation, and handles sandboxed resolution safely if maintainers approve the policy split.

## Reproduction Assessment

Yes, by source inspection. `sessions_send` calls the active SDK guard, and that guard rejects a cross-agent `tree` target before evaluating an allow-matching A2A policy.

## Solution Assessment

No, not as currently targeted. The branch edits the stale `src/agents/tools/sessions-access.ts` implementation instead of the active SDK helper, and it does not prove the real sandboxed send flow.
