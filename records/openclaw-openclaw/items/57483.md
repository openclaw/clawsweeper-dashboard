## Summary

Keep open. Current main still denies cross-agent `send` when `tools.sessions.visibility` is not `all` before the a2a allow policy can authorize the target, and this PR’s source edit targets the pre-move helper rather than the active SDK implementation.

## What This Changes

The PR changes the session access guard so cross-agent `send` skips the visibility check and is authorized by a2a policy, removes the send-specific visibility denial branch, and adds regression tests for allowed send versus blocked history.

## Best Possible Solution

Rebase the PR and put the send-only visibility exception in `src/plugin-sdk/session-visibility.ts`, keep `list`, `history`, and `status` visibility-gated, add focused a2a-allowed and a2a-denied regression tests, and handle sandboxed label resolution as an explicit scoped decision.

## Reproduction Assessment

Yes. A source-level reproduction is a `createSessionVisibilityGuard` call with `action: "send"`, requester `agent:main:main`, target `agent:ops:main`, `visibility: "tree"`, and an a2a policy allowing both agents; current main returns the visibility-forbidden branch before evaluating a2a allow.

## Solution Assessment

No, not as currently targeted against main. The policy split is plausible, but the patch must update the SDK helper that `sessions_send` now calls before it fixes the reported behavior.
