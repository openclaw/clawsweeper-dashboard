## Summary

## What This Changes

The PR adds a mapped `qa run --profile smoke-ci|release` dispatcher over the QA scorecard taxonomy mapping, tests CLI/runtime routing, and adds a scoped QA `AGENTS.md` plus `CLAUDE.md` symlink.

## Best Possible Solution

Preserve the root `--profile` state selector and expose mapped QA profile selection through a non-conflicting flag or an explicit maintainer-approved compatibility plan, then land this after the taxonomy mapping base is accepted.

## Reproduction Assessment

Yes for the patch defect: source inspection shows current main strips interleaved `qa run --profile work` as a root profile, while the PR forwards it to the QA command and rejects values outside `smoke-ci` or `release`. I did not execute the command because this review is read-only.

## Solution Assessment

No. The mapped QA runner is a plausible feature, but overloading root `--profile` is not the best merge shape unless maintainers intentionally accept and document the compatibility break.
