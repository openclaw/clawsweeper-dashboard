## Summary

Keep open. Current main still renders legacy migration `changeLines` under `Doctor changes` during a read-only doctor run, while the same path records a `--fix` hint and does not write config unless repair is applied.

## What This Changes

The issue asks for read-only `openclaw doctor` output to stop claiming legacy config keys were removed when no `--fix` repair was applied.

## Best Possible Solution

Make doctor config repair notes preview-safe: `Doctor changes` should describe only applied repairs, while read-only runs should use pending/future-tense wording or suppress mutation notes when existing legacy warnings and final `--fix` hints are enough.

## Reproduction Assessment

Yes. A config containing `agents.defaults.agentRuntime` plus `openclaw doctor` without `--fix` reaches the ungated `Doctor changes` note path on current main; I did not execute the CLI because this review kept the checkout read-only.

## Solution Assessment

Yes. The narrow maintainable fix is to separate preview wording from applied repair wording in the existing doctor config flow, preserving actual `--fix` behavior and adding regression coverage for both modes.
