## Summary

Keep open: current main still emits legacy migration change lines through the applied `Doctor changes` panel before any read-only doctor run has written the config, so the report is a valid behavior bug.

## What This Changes

Read-only `openclaw doctor` should not print past-tense applied migration text for pending config repairs that require `--fix` or explicit confirmation.

## Best Possible Solution

Keep the migration semantics, but render non-applied doctor mutations as pending preview text or suppress that panel until `--fix` or confirmation actually applies them.

## Reproduction Assessment

Yes, source-level reproduction is high-confidence: current main still combines non-repair legacy `changeLines`, the `Doctor changes` title, and a no-write finalization path. I did not run the CLI because this review is constrained to keep the checkout read-only.

## Solution Assessment

Yes, the best fix is a narrow doctor-output change that distinguishes preview from applied repairs without changing legacy migration or `--fix` write behavior.
