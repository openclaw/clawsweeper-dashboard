## Summary

Keep open: current main still routes the WhatsApp terminal QR path around the injected runtime, while this PR is a narrow WhatsApp-owned fix with no blocking review findings from source inspection.

## What This Changes

The PR routes WhatsApp `loginWeb` QR heading and terminal QR art through a shared runtime callback for initial and 515-restart sockets, adds restart coverage, and updates the changelog.

## Best Possible Solution

Land this PR after normal CI and mergeability checks; it is the narrow WhatsApp-owned fix and should close #76213 once merged.

## Reproduction Assessment

Yes. Source inspection shows current main calls `createWaSocket(true, ...)` from `loginWeb`, and `session.ts` prints QR output through `console.log` and `process.stdout.write` when `printQr` is true.

## Solution Assessment

Yes. The PR uses the existing `onQr` socket seam and `renderQrTerminal`, hoists the callback once, and forwards it into the 515 restart path without adding core API, config, or product-policy surface.
