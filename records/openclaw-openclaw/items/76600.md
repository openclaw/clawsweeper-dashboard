## Best Possible Solution

Keep the OpenClaw outer run timeout, but do not replay `timeout` through ACP `session/set_config_option` for Claude unless the backend explicitly supports it; preserve supported controls and surface nested JSON-RPC error details.

## Repair Work Prompt

Fix this issue: `sessions_spawn({ runtime: "acp", agentId: "claude", runTimeoutSeconds })` persists `timeoutSeconds` and pre-turn ACP runtime controls send `session/set_config_option` key `timeout`, which `@agentclientprotocol/claude-agent-acp@0.31.4` rejects. Keep the OpenClaw outer dispatch timeout intact, but do not replay `timeout` to ACP backends/adapters unless support is explicitly advertised or handled safely. Preserve model/thinking/permission behavior and avoid changing dependency versions. Also improve ACP error conversion/formatting so nested JSON-RPC `data.details` such as `Unknown config option: timeout` is visible instead of only `Internal error`. Add regression coverage for Claude/non-Codex timeout suppression and nested error details, and add a user-facing `CHANGELOG.md` fix entry.
