## Summary

Keep open: the report targets a documented ACP spawn path and has strong release-environment evidence, but current main plus the pinned acpx source do not confirm that OpenClaw sends a raw string as the final JSON-RPC prompt payload.

## What This Changes

The issue asks OpenClaw to make Claude ACP `sessions_spawn` runs work with `claude-agent-acp@0.31.4` by sending array-form ACP prompt content and surfacing nested ACP invalid-params details.

## Best Possible Solution

Capture the final OpenClaw/acpx `session/prompt` JSON-RPC payload for the failing Claude run, then patch the confirmed boundary and add regression coverage for text-only Claude ACP turns plus nested ACP error details.

## Reproduction Assessment

No: the issue gives a detailed release-environment repro, but this review did not establish a high-confidence current-main reproduction path. Static source inspection shows acpx converts text-only string prompts to ACP text blocks before `connection.prompt`, so a live wire trace is still needed.

## Solution Assessment

Unclear: normalizing the acpx `toPromptInput` intermediate is only the best fix if a real OpenClaw trace still shows a raw string reaching `session/prompt`. The safer solution is to trace the final payload and then fix either upstream acpx prompt conversion, OpenClaw's integration boundary, or error propagation at the proven layer.
