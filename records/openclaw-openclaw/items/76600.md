## Summary

Keep open: the reporter’s corrected trace matches current main’s ACP runtime-option path, where a Claude ACP spawn with a run timeout can replay `timeout` through `session/set_config_option` before the prompt turn and fail before `session/prompt`.

## What This Changes

Handle ACP timeout runtime-option replay so Claude sessions spawned with `runTimeoutSeconds` do not fail and nested ACP adapter errors are visible.

## Best Possible Solution

Land a narrow ACP/acpx fix that preserves the Gateway run timeout while suppressing or capability-gating ACP `timeout` config replay for Claude/unsupported adapters, with clearer nested ACP error details.

## Reproduction Assessment

Yes, source-reproducible with high confidence: `sessions_spawn({ runtime: "acp", agentId: "claude", runTimeoutSeconds: N })` stores `timeoutSeconds`, emits config key `timeout`, and forwards it to Claude ACP 0.31.4, whose published code rejects unknown config options. I did not run a live Claude ACP session in this read-only review.

## Solution Assessment

Yes: the narrow maintainable fix is to stop sending unsupported ACP `timeout` config controls to Claude/unsupported acpx adapters while keeping OpenClaw’s own turn timeout enforcement. Prompt array normalization is not the right fix for the corrected failure path.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/76600. Current main maps `sessions_spawn` ACP `runTimeoutSeconds` into ACP runtimeOptions, emits config key `timeout`, and the acpx extension forwards that key for Claude/non-Codex agents; `@agentclientprotocol/claude-agent-acp@0.31.4` rejects it as an unknown config option before the prompt turn. Keep `runTimeoutSeconds` as the Gateway/OpenClaw turn timeout, but do not send `session/set_config_option timeout` to Claude ACP unless the backend clearly advertises support for that key. Preserve existing Codex ACP timeout suppression, avoid changing prompt content-block handling, and improve error surfacing so nested ACP details like `Unknown config option: timeout` are visible in the OpenClaw failure. Add focused regression coverage for Claude/non-Codex timeout replay and keep a generic supported-adapter path if needed. Add a user-facing CHANGELOG entry.
