## Summary

Keep open: the report describes a real documented ACP spawn surface, but current source and the published acpx dependency do not yet confirm that OpenClaw sends a raw string all the way to the Claude ACP wrapper.

## What This Changes

The issue asks OpenClaw to make Claude ACP `sessions_spawn` send array-form prompt content and expose the underlying ACP invalid-params error details.

## Best Possible Solution

Capture the final `session/prompt` payload from a failing Claude ACP spawn, then patch the confirmed boundary and add regression coverage for text-only Claude ACP turns plus clearer nested ACP error details.

## Reproduction Assessment

No high-confidence current-main reproduction is established. The reporter provides a detailed 2026.5.2 repro, but current source plus published acpx source shows strings should be converted to text blocks before JSON-RPC, and no live trace was captured here.

## Solution Assessment

Unclear: wrapping `toPromptInput` would be the right fix only if a trace proves a raw string reaches the wrapper. The safer path is to trace the emitted payload and then patch acpx or OpenClaw error propagation at the confirmed layer.
