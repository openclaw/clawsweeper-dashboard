## Summary

Keep open: the report is detailed and affects the documented ACP Claude spawn path, but current source and dependency inspection do not yet confirm that OpenClaw/acpx sends a raw string as the final `session/prompt` payload.

## What This Changes

Request that ACP text-only prompts be sent as ACP content-block arrays for `claude-agent-acp@0.31.4`, with clearer propagation of underlying invalid-params errors.

## Best Possible Solution

Capture the final OpenClaw/acpx `session/prompt` payload for the failing Claude turn, then fix the confirmed boundary and add regression coverage for text-only Claude ACP turns plus nested ACP error details.

## Reproduction Assessment

No high-confidence current-main reproduction is established. The reporter provides strong release-environment evidence, but this review did not run a live Claude ACP turn and source inspection shows acpx should wrap string prompts before JSON-RPC transport.

## Solution Assessment

Unclear: normalizing at the OpenClaw/acpx prompt boundary is the right fix only if tracing confirms a raw string reaches the wrapper. A safer path is to capture the emitted payload and preserve the underlying JSON-RPC `-32602` details at the layer that actually loses them.
