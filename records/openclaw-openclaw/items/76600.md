## Summary

Keep open: the reporter's corrected trace matches current main source. The suspected raw prompt issue is superseded, but `runTimeoutSeconds` is still persisted as an ACP runtime timeout option and replayed as `session/set_config_option` key `timeout`, while `claude-agent-acp@0.31.4` rejects unknown config options.

## What This Changes

The issue asks OpenClaw to stop failing Claude ACP `sessions_spawn` runs with timeouts and to surface the underlying ACP adapter error details.

## Best Possible Solution

Keep the issue open and fix the ACP timeout-control boundary so OpenClaw keeps its own turn watchdog but does not replay `timeout` to Claude ACP unless the backend explicitly supports that config option; preserve or improve nested ACP error detail reporting.

## Reproduction Assessment

Yes, source-reproducible with the reporter's live trace: call `sessions_spawn` with `runtime="acp"`, `agentId="claude"`, and a positive `runTimeoutSeconds`; current main stores that timeout and replays it as ACP config key `timeout`. I did not run the live Claude ACP session locally during this read-only review.

## Solution Assessment

Yes, the narrow maintainable fix is to preserve OpenClaw's timeout as an OpenClaw turn/gateway guard while suppressing or capability-gating the backend `timeout` config replay for Claude ACP. If the behavior belongs in shared acpx capability detection, move that part upstream instead of adding broad core special cases.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/76600. Current main stores ACP spawn `runTimeoutSeconds` as `runtimeOptions.timeoutSeconds`, maps that to backend config key `timeout`, and replays it through `session/set_config_option`; `@agentclientprotocol/claude-agent-acp@0.31.4` rejects that key. Preserve `runTimeoutSeconds` as OpenClaw's own turn/gateway timeout, but do not send `timeout`/`timeout_seconds` to Claude ACP unless the backend explicitly advertises support. Prefer a narrow acpx-wrapper or capability-gating fix; if the right layer is shared ACPX runtime behavior, follow `extensions/acpx/AGENTS.md` and stop before changing dependency pins without maintainer approval. Update the existing acpx runtime tests that currently expect Claude timeout forwarding, add a regression for Claude timeout suppression, and keep Codex timeout suppression intact. If feasible in the same narrow patch, improve propagation of nested ACP error details so `Unknown config option: timeout` is not collapsed to only `Internal error`. Add a `CHANGELOG.md` fix entry because this is user-facing.
