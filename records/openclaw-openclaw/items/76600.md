## Summary

Keep open: current main still matches the reported ACP prompt schema bug. OpenClaw pins `acpx@0.6.1`, whose published runtime sends a bare string for text-only turns, while the pinned ACP SDK/Claude adapter path expects `session/prompt.params.prompt` to be an array of content blocks.

## What This Changes

The issue asks OpenClaw to make ACP text-only prompts schema-compatible with current `claude-agent-acp` and to surface the underlying invalid-params error details.

## Best Possible Solution

Fix prompt construction in acpx so text-only `session/prompt` requests always use content-block arrays, then bump the OpenClaw acpx plugin dependency and add regression coverage plus clearer ACP invalid-params error surfacing.

## Reproduction Assessment

Yes, at source level: current main pins `acpx@0.6.1`, and that package returns a bare string for text-only prompts while ACP SDK 0.21 validates `prompt` as an array. I did not perform a live Claude ACP run in this read-only review.

## Solution Assessment

Yes: normalizing at the acpx prompt-construction boundary is the narrowest maintainable fix. OpenClaw should consume a fixed acpx release or approved patch rather than add a config option or special-case only Claude in core.
