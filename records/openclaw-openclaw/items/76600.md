## Summary

Keep open: this is a plausible existing ACP Claude runtime bug, and current main does not prove it fixed; however source and dependency inspection show the suspected raw-string prompt layer is not yet confirmed because acpx converts string prompts before `session/prompt`.

## What This Changes

Normalize ACP text-only prompts for Claude/acpx so `sessions_spawn(runtime: "acp", agentId: "claude")` succeeds and surfaces underlying invalid-params errors.

## Best Possible Solution

Capture the final `session/prompt` payload and nested ACP error from a failing Claude ACP run, then patch the confirmed boundary and add regression coverage for text-only Claude turns and clearer invalid-params diagnostics.

## Reproduction Assessment

No. The reporter provides a strong OpenClaw 2026.5.2 container repro and a raw-wrapper control test, but this review did not run a live Claude ACP session and current acpx source appears to wrap string prompts before JSON-RPC.

## Solution Assessment

Unclear. Wrapping `toPromptInput` alone may patch the wrong layer because `AcpClient.prompt` already converts string prompts to content blocks; the narrow maintainable fix depends on a live trace of the actual failing payload or thrown error.
