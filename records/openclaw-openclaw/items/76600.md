## Summary

Keep open: the reporter’s corrected trace is consistent with current main, which still turns `runTimeoutSeconds` into an ACP `timeout` config replay for Claude/acpx even though `claude-agent-acp@0.31.4` rejects unknown config options before `session/prompt`.

## What This Changes

The issue requests that ACP Claude runs stop failing when timeout options are supplied and that nested ACP adapter errors be surfaced clearly.

## Best Possible Solution

Pass `runTimeoutSeconds` through the ACP turn timeout path and only replay `timeout` as `session/set_config_option` when the adapter advertises or is known to support that option, while preserving the underlying ACP error details in user-visible failures.

## Reproduction Assessment

Yes, by source inspection plus the reporter’s trace: `sessions_spawn(runtime="acp", agentId="claude", runTimeoutSeconds=45)` persists `timeoutSeconds`, current control replay sends `session/set_config_option` for `timeout`, and Claude ACP rejects unknown config ids before the prompt runs. I did not execute the live smoke locally.

## Solution Assessment

No: the original prompt-array normalization theory is not the best fix after the corrected trace. The maintainable fix is to stop sending unsupported timeout config controls and keep the timeout as OpenClaw/acpx turn supervision unless the adapter actually supports that ACP config option.

## Repair Work Prompt

Fix openclaw/openclaw#76600. Current main maps ACP spawn `runTimeoutSeconds` into persisted runtime option `timeoutSeconds`, then `buildRuntimeConfigOptionPairs` replays it as `session/set_config_option` key `timeout`; `@agentclientprotocol/claude-agent-acp@0.31.4` rejects that with `Unknown config option: timeout` before `session/prompt`. Preserve `runTimeoutSeconds` as an OpenClaw/Gateway/acpx turn watchdog, but do not send `timeout` via ACP `session/set_config_option` to Claude or any adapter that has not advertised/supports that config option. Prefer a generic capability/advertised-key path if feasible; otherwise keep the compatibility filter in the `acpx` plugin wrapper and do not hardcode bundled ids in core. Also improve ACP failure formatting so nested adapter error details such as `Unknown config option: timeout` are visible instead of only `AcpRuntimeError: Internal error`. Add/update regression coverage for Claude/non-Codex timeout spawns and error detail propagation. Add a user-facing changelog fix entry; do not change dependency pins or patch/vendor upstream packages without maintainer approval.
