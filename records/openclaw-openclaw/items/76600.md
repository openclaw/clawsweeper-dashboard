## Summary

Keep open: the report is current, detailed, and affects the documented Claude ACP `sessions_spawn` path, but current-main source does not confirm the suspected raw-string JSON-RPC payload. OpenClaw delegates the turn to `acpx@0.6.1`, and that client source wraps string prompts into ACP text content blocks before `session/prompt`, so a live payload trace is needed before choosing the fix layer.

## What This Changes

The issue requests schema-compatible content-block prompts for text-only Claude ACP turns and clearer nested ACP invalid-params errors.

## Best Possible Solution

Trace the actual OpenClaw-to-wrapper `session/prompt` payload for a failing Claude ACP run, then patch the confirmed layer and add regression coverage for text-only Claude turns plus nested ACP invalid-params error detail.

## Reproduction Assessment

No. The issue body gives strong release-environment reproduction and a direct JSON-RPC probe, but this review did not run a live Claude ACP session and current source shows acpx converting string prompts before `session/prompt`.

## Solution Assessment

Unclear. Normalizing at the OpenClaw boundary is the right fix only if a real runtime trace still sends raw strings; otherwise the safer fix belongs in the acpx wrapper/runtime or in nested ACP error surfacing.
