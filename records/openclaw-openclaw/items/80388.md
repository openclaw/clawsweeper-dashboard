Overall correctness: patch is incorrect

Full review comments:

- **[P1] Mint plugin UI tokens with entry-scoped permissions:** `src/gateway/server-methods/plugin-host-hooks.ts:147`
  - body: `plugins.uiEntryPointLaunch` filters visibility with `requiredScopes`, but the token it issues stores the full WebSocket client scopes. Since the Gateway client defaults to `operator.admin`, a normal admin Control UI opening an entry declared as `operator.read` gives the plugin route admin/write authority through the launch token and session cookie. Please derive the granted token scopes from the entry point's required scopes, with a read default, and add a regression test for an admin caller launching a read-only entry.
  - confidence: 0.88
- **[P3] Document the plugin UI entry-point contract:** `src/plugins/types.ts:2548-2549`
  - body: This adds `registerControlUiEntryPoint` plus two Gateway RPC methods, but no public docs explain how plugin authors should use the API, which paths are allowed, what the open modes mean, or how `requiredScopes` and launch tokens work. The plugin SDK and protocol guides are listed as source-of-truth surfaces for these contracts, so please update them before merge.
  - confidence: 0.93
