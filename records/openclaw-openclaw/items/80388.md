## Summary

Keep this PR open: the feature direction is plausible, but the patch introduces a security-sensitive scope escalation path for plugin UI launch tokens and the supplied proof does not directly show the browser Control UI entry appearing and opening.

## What This Changes

The PR adds generic plugin-owned Control UI entry point registration, gateway listing and launch-token plumbing, Control UI navigation/rendering, SDK docs, tests, and refreshed Swift gateway models.

## Best Possible Solution

Keep the generic entry-point seam, but make launch-token and session-cookie auth preserve the exact authorized scope set for every plugin route surface, add a regression test for `trusted-operator` routes, and provide direct browser proof of the Control UI launch path.

## Reproduction Assessment

Yes for the blocking review finding: source inspection shows a read-scoped launch token can be converted into trusted-operator default scopes because no `x-openclaw-scopes` header is supplied. A focused regression would register a `trusted-operator` gateway route under the plugin path, launch it with only `operator.read`, and assert admin/write helpers are not available.

## Solution Assessment

No. The proposed plugin entry-point API is a reasonable direction, but the auth plumbing is not the safest implementation until trusted-operator routes consume the launch/session scope set instead of falling back to default operator scopes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve launch scopes on trusted-operator routes:** `src/gateway/server-http.ts:451`
  - body: `plugins.uiEntryPointLaunch` can issue a token with only `operator.read`, but this block authenticates the resulting plugin HTTP request as `trustDeclaredOperatorScopes: true` without adding an `x-openclaw-scopes` header. Existing `trusted-operator` plugin routes ignore `gatewayRequestOperatorScopes` and resolve missing headers to `CLI_DEFAULT_OPERATOR_SCOPES`, including admin/write, so a read-scoped entry can reach a trusted route under the same plugin path with full helper scopes. Pass the launch/session scopes through that path or deny the trusted-operator default for UI-entry token auth.
  - confidence: 0.93

## Security Review
