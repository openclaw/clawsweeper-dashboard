## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Mint plugin UI tokens with entry-scoped permissions:** `src/gateway/server-methods/plugin-host-hooks.ts:147`
  - body: The normal Control UI connects with broad operator scopes, but this handler stores that entire `client.connect.scopes` set in the launch token. `server-http` then trusts those scopes for iframe/session-cookie plugin requests, so a read-scoped app-nav entry opened from the standard Control UI can make plugin-root requests with admin/write scopes via the bridge. Mint the token from the entry's required scopes, default to `operator.read`, intersect with the caller, and add an admin-caller/read-entry regression.
  - confidence: 0.95
- **[P3] Document the plugin UI entry-point contract:** `src/plugins/types.ts:2549`
  - body: This adds a public Plugin SDK method plus Gateway entry-point/list/launch behavior, but the plugin docs still only describe `registerControlUiDescriptor` and no docs files are changed. Plugin authors need the supported path rules, `openMode`, `requiredScopes`, launch-token behavior, and preferred grouped API documented before this becomes a public contract.
  - confidence: 0.88
