## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Make the sandboxed iframe bridge deliverable:** `ui/src/ui/app.ts:1054`
  - body: The plugin iframe is sandboxed without `allow-same-origin`, so it has a special origin; `targetWindow.postMessage(..., window.location.origin, [port])` requires an exact recipient origin match and will not deliver the initial bridge port to that frame. Keep the sandbox boundary, but use a bridge design that can actually deliver the MessagePort and add a test that exercises real `postMessage` delivery instead of mocking `postPluginUiBridgeConnect`. ([developer.mozilla.org](https://developer.mozilla.org/en-US/docs/Web/HTML/Reference/Elements/iframe))
  - confidence: 0.93
- **[P2] Export the plugin UI response contracts:** `packages/gateway-protocol/src/index.ts:216-219`
  - body: The schema layer defines `PluginsUiEntryPointsResultSchema` and `PluginsUiEntryPointLaunchResultSchema`, but the public package entrypoint imports, exports, and validates only the new params. Downstream gateway-protocol consumers cannot import or validate these RPC responses, and the new handlers cannot mirror the existing `plugins.uiDescriptors` result-validation pattern until the result contracts are exported.
  - confidence: 0.88
