## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Gate the plugin bridge before transferring the port:** `ui/src/ui/app.ts:898`
  - body: The load handler sends a fresh MessagePort to whatever document is currently loaded in the plugin iframe using `postMessage(..., "*", [port])`. If the plugin page navigates its own frame to an external page, that page can receive the port and use the parent proxy to fetch the original plugin's same-origin routes with credentials, then receive the response body. Please prove the frame is still the intended plugin-owned document before transferring the port, or tear down/disable the bridge when the frame navigates away.
  - confidence: 0.86
