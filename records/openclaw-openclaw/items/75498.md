Keep open. Current main still has a confirmed Telegram partial-preview path where ambiguous final edit failures are retained and marked delivered, while the polling recovery, DM silent-reply, and auto model-override symptoms are only partially fixed, separately tracked, or current policy.

## What This Changes

The issue requests reliable Telegram final delivery during partial streaming/polling stalls, no Web UI delivery conflation, no DM silent-reply chatter, and no unintended persistent session model overrides.

## Best Possible Solution

## Solution Assessment
