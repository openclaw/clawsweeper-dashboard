## Summary

Keep open. Current main still has a Telegram partial-preview path where ambiguous final edit failures are retained and marked delivered, while the silent-DM and model-failover symptoms are still separately open or documented policy rather than fully resolved.

## What This Changes

The issue reports unreliable Telegram final delivery during partial streaming and polling stalls, DM silent-reply chatter, and unintended persistent session model overrides.

## Best Possible Solution

Define Telegram delivery state so only confirmed final preview edits count as delivered, land or supersede the focused silent-DM fix, and keep model failover persistence explicit in policy and UI.

## Reproduction Assessment

Yes for the central delivery-state class: source inspection and tests define a final edit timeout after an existing preview as retained and delivered, which suppresses fallback. The full macOS polling-stall incident was not live-reproduced in this read-only review.

## Solution Assessment

No. Current main improves polling transport and reset behavior, but marking uncertain final edits as delivered is not the narrowest durable delivery solution; the remaining pieces need scoped follow-ups.
