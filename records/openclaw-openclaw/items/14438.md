Keep open: current main and the latest release still do not provide a supported plugin source/Jiti cache reload or clear-cache workflow, and the existing discussion explicitly routes this to Gateway/plugin-loader lifecycle design rather than cleanup closure.

## What This Changes

The issue requests a supported development workflow to reload TypeScript plugin code or clear Jiti cache without restarting the OpenClaw process/container.

## Best Possible Solution

Define a supported Gateway/API-backed plugin development reload contract that invalidates OpenClaw loader/module state and safely swaps plugin registrations, then expose it through plugin CLI UX or document restart-only behavior with precise cache guidance if live source reload remains unsupported.

## Reproduction Assessment

Not applicable as a strict bug reproduction; this is a feature request. Source, docs, tests, latest-release inspection, Jiti source, and live issue context give high-confidence evidence that plugin source/code changes still lack a reload or clear-cache workflow.

## Solution Assessment

Unclear as proposed. A bare cache-delete command is not enough unless paired with safe Gateway runtime replacement and resource cleanup; the maintainable path is a designed reload contract or explicit restart-only documentation.
