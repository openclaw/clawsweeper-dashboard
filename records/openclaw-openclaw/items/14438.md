## Summary

Keep open: current main has partial plugin management hot reload, but source/Jiti cache reload and a public reload or cache-clear workflow are still missing, and the live discussion already left this lifecycle design open.

## What This Changes

Request a supported way to reload changed TypeScript plugin code or clear its Jiti/module cache without restarting the OpenClaw process or container.

## Best Possible Solution

Design and document a supported plugin development reload lifecycle that safely invalidates loader/Jiti/module state, swaps active registrations, disposes stale resources, and exposes an additive CLI/RPC workflow such as `openclaw plugins reload <id>` or `openclaw plugins clear-cache`.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature request. Source, docs, tests, and issue discussion give a high-confidence inspection path showing source/load/install changes remain restart-backed and no public reload/cache-clear command exists.

## Solution Assessment

Unclear until maintainers choose lifecycle semantics. A cache-delete command alone is not the best fix because it would not safely replace active registrations or dispose plugin-owned resources.
