## What This Changes

The issue requests a supported plugin development reload or cache-clear workflow for TypeScript plugin edits without restarting the OpenClaw process or container.

## Best Possible Solution

Define a supported Gateway/API-backed plugin development reload contract that invalidates loader/module state and safely swaps plugin registrations, then expose it through plugin CLI UX or document restart-only behavior with precise cache guidance if live source reload remains unsupported.

## Reproduction Assessment

Not applicable as a strict bug reproduction; this is a feature request. Source, docs, tests, and live issue context give high-confidence evidence that plugin source/code changes still require restart and no reload or clear-cache command exists.

## Solution Assessment

Unclear as proposed. A bare cache-delete command is not enough unless paired with safe Gateway runtime replacement and resource cleanup; the maintainable path is a designed reload contract or explicit restart-only documentation.
