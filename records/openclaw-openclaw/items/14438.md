## Summary

Keep open: current main has plugin-config hot reload now, but it still does not provide the requested developer workflow for reloading TypeScript plugin source/Jiti cache state without a process/container restart, and the latest substantive discussion explicitly left this for maintainer design.

## What This Changes

Request support for reloading TypeScript plugin source and Jiti cache state, via a plugin reload or clear-cache command or watcher, without restarting the OpenClaw process or container.

## Best Possible Solution

Design a supported developer reload workflow that distinguishes config hot reload from source changes, then expose a Gateway-backed `openclaw plugins reload <id>` or `openclaw plugins clear-cache` path, or a scoped watcher, with safe loader/Jiti invalidation and runtime replacement docs.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature request. Source and docs inspection give high confidence that current main lacks a supported plugin source/Jiti reload command or watcher, while only plugin config changes hot-reload today.

## Solution Assessment

Unclear until maintainers choose the product boundary. A targeted CLI/Gateway reload command looks safer as a first supported surface than automatic source watching, because it can make cache invalidation and runtime disposal explicit.
