## Summary

Keep open: current main still lacks a supported workflow to reload TypeScript plugin source or clear Jiti/module loader state without a process/container restart, and prior discussion explicitly leaves this for plugin-loader and Gateway lifecycle design.

## What This Changes

Request a plugin development reload or cache-clear workflow, such as `openclaw plugins reload <id>`, a source watcher, or a cache-clear command for Jiti-backed TypeScript plugins.

## Best Possible Solution

Design a supported developer reload workflow that distinguishes config hot reload from plugin source changes, then expose a Gateway-backed reload or cache-clear command, or document restart-only behavior if live source reload remains unsupported.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature request. Source and docs inspection give high confidence that current main lacks a supported plugin source/Jiti reload command or watcher while source/install changes remain restart-backed.

## Solution Assessment

Unclear until maintainers choose the lifecycle boundary. A targeted Gateway-backed `openclaw plugins reload <id>` or `clear-cache` command looks safer as a first surface than automatic source watching because it can make cache invalidation and runtime disposal explicit.
