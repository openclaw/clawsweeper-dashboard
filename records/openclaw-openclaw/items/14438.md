## Summary

Keep open: current main and v2026.6.5 now have some plugin config hot-reload and install-triggered restart handling, but they still do not provide the requested plugin source/Jiti cache reload, file watcher, or `openclaw plugins clear-cache` workflow. The remaining work is a core Gateway/plugin-loader lifecycle design item, not conservative cleanup.

## What This Changes

The issue requests a supported plugin development reload or cache-clear workflow so TypeScript plugin edits can take effect without a full process or container restart.

## Best Possible Solution

Define a Gateway/API-backed plugin development reload contract that invalidates OpenClaw loader/module/Jiti state and safely swaps plugin registrations, then expose that through plugin CLI UX or document restart-only behavior with precise cache guidance if live source reload remains unsupported.

## Reproduction Assessment

Not applicable as a strict bug reproduction; this is a feature request. Source, docs, tests, latest-release inspection, dependency source, and live issue context give high-confidence evidence that plugin source/code changes still lack a supported reload or clear-cache workflow.

## Solution Assessment

Unclear as proposed. A bare cache-delete command is not the best standalone fix unless it is paired with safe Gateway runtime replacement and resource cleanup; the maintainable path is a designed reload contract or explicit restart-only documentation.
