## Best Possible Solution

Define a supported Gateway/API-backed plugin development reload contract that invalidates OpenClaw loader/module state and safely swaps plugin registrations, then expose it through plugin CLI UX or document restart-only behavior with precise cache guidance if live source reload remains unsupported.

Unclear as proposed. A bare cache-delete command is not enough unless paired with safe Gateway runtime replacement and resource cleanup; the maintainable path is a designed reload contract or explicit restart-only documentation.
