## Summary

Keep open: current main still has only config/policy-driven plugin reload, while plugin source/Jiti cache reload and a public reload or cache-clear workflow remain missing. The recent issue discussion also explicitly leaves this for Gateway/plugin-loader lifecycle design.

## What This Changes

The issue requests a supported plugin development workflow to reload TypeScript plugin code or clear the Jiti cache without restarting the OpenClaw container.

## Best Possible Solution

Design and document a supported plugin development reload lifecycle that safely invalidates loader/Jiti/module state, swaps active registrations, disposes stale resources, and exposes an additive CLI/RPC workflow such as `openclaw plugins reload <id>` or `openclaw plugins clear-cache`.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature request. Source, docs, tests, and issue discussion give a high-confidence inspection path showing plugin source/load/install changes remain restart-backed and no public reload or cache-clear command exists.

## Solution Assessment

Unclear until maintainers choose lifecycle semantics. A raw cache-clear command is not the best standalone fix because it would not safely replace active registrations or dispose plugin-owned resources.
