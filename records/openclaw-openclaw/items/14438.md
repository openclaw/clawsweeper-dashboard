## Summary

Keep open: current main now hot-reloads some plugin config and enablement changes, but plugin code/source changes still require a Gateway restart, the plugin CLI has no reload or cache-clear command, and the existing maintainer comment correctly treats this as Gateway/plugin-loader lifecycle design work.

## What This Changes

Request to add a supported plugin development reload or cache-clear workflow so TypeScript plugin code changes can take effect without restarting the OpenClaw process or container.

## Best Possible Solution

Design a supported Gateway/API-backed plugin development reload path that invalidates loader/Jiti/module state and safely swaps plugin registrations, then expose it through `openclaw plugins reload <id>` or document restart-only behavior with precise cache guidance if live source reload remains unsupported.

## Reproduction Assessment

Not applicable as a strict bug reproduction; this is a feature request. Source, docs, and tests give high-confidence evidence that plugin source/code changes still require restart and no reload or cache-clear command exists.

## Solution Assessment

Unclear as proposed. A bare cache delete command would not be enough unless it is paired with safe Gateway runtime replacement and resource cleanup; the maintainable path is a designed reload contract or explicit restart-only documentation.
