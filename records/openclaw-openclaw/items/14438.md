## Summary

Keep open: current main still lacks a supported plugin source hot-reload or cache-clear command, and the latest related work only covers adjacent config reload, Jiti cache-directory, or plugin-management slices. The remaining request crosses plugin loader/Jiti cache invalidation, active registry replacement, runtime cleanup, Gateway lifecycle, and CLI/RPC UX, so it needs maintainer design rather than cleanup closure.

## What This Changes

Request a supported plugin development workflow to reload TypeScript plugin code or clear Jiti cache without restarting the OpenClaw process/container.

## Best Possible Solution

Design a supported plugin development reload lifecycle that invalidates loader/Jiti/module state, replaces active registrations safely, disposes stale resources, and exposes an additive CLI/RPC workflow such as `openclaw plugins reload <id>` or `openclaw plugins clear-cache` with docs.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature request. Source, docs, release-tag inspection, related PR/issue context, and live issue discussion give a high-confidence inspection path showing plugin source/load/install changes remain restart-backed and no public reload or cache-clear command exists.

## Solution Assessment

Unclear until maintainers choose lifecycle semantics. A standalone cache-clear command is not the best complete fix because it would not safely replace active registrations or dispose plugin-owned resources.
