## Summary

Keep open: current main and the latest release still do not provide a supported plugin source/Jiti hot-reload or cache-clear command, and the live discussion adds more evidence that plugin install/source activation remains restart-backed.

## What This Changes

The issue requests a supported plugin hot-reload or cache-clear workflow for TypeScript plugin development without restarting OpenClaw.

## Best Possible Solution

Design a Gateway-owned plugin development reload lifecycle that safely disposes old runtime state, invalidates loader/Jiti/module caches, rebuilds and swaps the active registry, and then exposes a documented CLI/API command; if maintainers reject live reload, document restart-only behavior and supported cache diagnostics precisely.

## Reproduction Assessment

Yes, at source level. Current main has no public plugin reload/cache-clear command, restart-backs plugin source/install paths, and documents Gateway restart after plugin code or source changes; I did not run a live container edit cycle in this read-only review.

## Solution Assessment

No, manually deleting dependency cache files is not the best permanent solution. The maintainable fix is a lifecycle-owned Gateway reload operation with explicit teardown, cache invalidation, registry replacement, and CLI/API semantics.
