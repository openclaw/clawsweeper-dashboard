## What This Changes

The issue asks for top-level `browser.cdpUrl` to be honored when `browser.defaultProfile` is `"user"`, instead of falling back to Chrome MCP auto-connect and `DevToolsActivePort`.

## Best Possible Solution

Land or close the linked resolver PR, or an equivalent fix, so selected existing-session defaults inherit normalized top-level `cdpUrl` only when no profile-local endpoint exists.

## Reproduction Assessment

Yes. Source inspection of current main shows `defaultProfile: "user"` selects the built-in existing-session profile with an empty profile-local `cdpUrl`, so Chrome MCP falls back to `--autoConnect` instead of the configured endpoint.

## Solution Assessment

Yes. The narrow maintainable fix belongs in browser profile resolution, and the linked PR implements that shape while preserving explicit per-profile `cdpUrl` precedence.
