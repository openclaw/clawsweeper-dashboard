## Summary

Keep open: current main still lets `defaultProfile: "user"` resolve to the built-in existing-session profile without inheriting top-level `browser.cdpUrl`, and the linked fix PR is still open.

## What This Changes

Honor top-level `browser.cdpUrl` when `browser.defaultProfile` selects an existing-session profile so Chrome MCP uses the configured endpoint instead of DevToolsActivePort auto-connect.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/80143 after endpoint redaction is addressed, then close this issue as implemented.

## Reproduction Assessment

Yes. Source inspection on current main shows the reported config resolves `user` as an existing-session profile with empty `cdpUrl`, which sends Chrome MCP down the `--autoConnect` path.

## Solution Assessment

Yes, with one merge blocker. Resolver-side inheritance of the top-level endpoint is the narrow fix boundary, but Chrome MCP failure text should redact sensitive endpoints before the linked PR lands.
