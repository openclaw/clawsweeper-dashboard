## What This Changes

The issue asks for top-level `browser.cdpUrl` to be honored when `browser.defaultProfile: "user"` selects the built-in existing-session browser profile.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/80143 or an equivalent resolver fix after preserving endpoint redaction and adding focused regression coverage.

## Reproduction Assessment

Yes. Source inspection shows `defaultProfile: "user"` selects an existing-session profile whose own `cdpUrl` is empty, so Chrome MCP receives `--autoConnect` instead of the configured endpoint.

## Solution Assessment

Yes. The maintainable fix belongs in browser profile resolution so a selected existing-session default profile inherits the normalized top-level endpoint while explicit profile-local `cdpUrl` still wins; the linked PR should also preserve endpoint redaction before merge.
