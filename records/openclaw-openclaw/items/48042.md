## Summary

Keep open. Current main still has the reported source path, and https://github.com/openclaw/openclaw/pull/80143 is the active closing PR, so this issue should wait for that PR to be repaired and merged or replaced.

## What This Changes

The issue reports that top-level `browser.cdpUrl` is ignored when `browser.defaultProfile` selects `user`, causing Chrome MCP to fall back to `DevToolsActivePort`.

## Best Possible Solution

Repair and land https://github.com/openclaw/openclaw/pull/80143, or a replacement, so selected existing-session default profiles inherit top-level `browser.cdpUrl` safely while preserving CDP endpoint redaction.

## Reproduction Assessment

Yes. A source-level reproduction is `resolveBrowserConfig({ defaultProfile: "user", cdpUrl: "http://127.0.0.1:9222" })` followed by `resolveProfile`, which selects the built-in existing-session `user` profile with an empty `cdpUrl` and therefore reaches Chrome MCP `--autoConnect`.

## Solution Assessment

Yes, the resolver is the narrow owner-local place to normalize this legacy top-level config, but the active PR should not merge until the endpoint-redaction concern is resolved.
