## Summary

Keep open because current main and the latest release still leave the reported `browser.cdpUrl` + `defaultProfile: "user"` path source-reproducible, and the focused closing fix remains open at https://github.com/openclaw/openclaw/pull/80143.

## What This Changes

The issue reports that top-level `browser.cdpUrl` is ignored when `browser.defaultProfile` selects `user`, causing Chrome MCP to use its DevToolsActivePort auto-connect path.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/80143 or an equivalent browser config resolver fix that honors top-level `browser.cdpUrl` for the selected existing-session default profile while preserving profile-local precedence and endpoint redaction.

## Reproduction Assessment

Yes. Source inspection shows `defaultProfile: "user"` selects an existing-session profile whose own `cdpUrl` is empty, so Chrome MCP receives `--autoConnect` instead of the configured endpoint.

## Solution Assessment

Yes, with one caveat. The maintainable fix belongs in browser profile resolution, but the linked PR still needs its endpoint-redaction concern handled before merge.
