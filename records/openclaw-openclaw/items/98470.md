## Summary

Keep open: current main and the latest release only wire cloud-synced state-dir doctor warnings for macOS, and the direct fixing candidate at https://github.com/openclaw/openclaw/pull/98471 is still open and draft.

## What This Changes

The issue asks for `openclaw doctor` to warn when Windows `OPENCLAW_STATE_DIR` is under common cloud-synced folders such as OneDrive, Dropbox, Google Drive, or iCloud Drive.

## Best Possible Solution

Review and either land the linked Windows doctor fix PR or close it and keep this issue open for a narrow replacement that adds equivalent doctor coverage.

Yes, extending doctor state-integrity with Windows cloud-synced path detection is the right surface because `openclaw doctor` already owns state-dir safety warnings.
