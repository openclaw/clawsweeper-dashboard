## Summary

Keep open. Current main still advertises `github-copilot/claude-opus-4.6`, still classifies the reported unrecognized 400 response as a format failure, and still makes non-rate-limit cooldowns profile-wide; the open closing PR #75391 has not merged and only addresses the catalog half of the report.

## What This Changes

The issue requests removing or dynamically validating the stale GitHub Copilot `claude-opus-4.6` default and scoping `model_not_supported` failures to the rejected model instead of cooling down the whole provider profile.

## Best Possible Solution

Fix this in two layers: keep the bundled Copilot catalog conservative or runtime-backed, and classify Copilot unsupported-model responses as model-scoped unavailability so other Copilot models can continue using the same auth profile. #75391 should be expanded to cover the cooldown behavior or superseded by a complete fix.

## Reproduction Assessment

Yes. The reporter provides a concrete v2026.4.29 command and gateway log, and current-main source still deterministically contains both the stale advertised model and the profile-wide cooldown path for an unclassified 400 `model_not_supported` response.

## Solution Assessment

Yes for the issue's two-layer direction; no for a catalog-only change as the complete fix. Removing `claude-opus-4.6` is a useful mitigation, but unsupported-model classification must avoid poisoning the whole Copilot auth profile.
