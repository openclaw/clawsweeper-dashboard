## Summary

Keep open. Current main still advertises `github-copilot/claude-opus-4.6`, still classifies an unrecognized 400 payload such as `model_not_supported` as `format`, and still treats non-rate-limit auth-profile failures as profile-wide; the open closing PR #75391 is unmerged and only addresses the catalog half.

## What This Changes

The issue asks to stop advertising unsupported `github-copilot/claude-opus-4.6` and prevent a Copilot `model_not_supported` 400 from cooling down the entire provider profile.

## Best Possible Solution

Ship a complete Copilot provider fix that keeps the bundled catalog conservative or runtime-backed and scopes unsupported-model rejections to the failing model so other Copilot models can still use the same auth profile.

## Reproduction Assessment

Yes. The reporter provided a concrete v2026.4.29 command and gateway log, and current main still deterministically contains both the stale advertised model and the profile-wide non-rate-limit cooldown path; I did not live-call Copilot because no token was present.

## Solution Assessment

Yes for the issue's two-part direction; no for a catalog-only fix by itself. Removing `claude-opus-4.6` mitigates this stale default, but the maintainable fix also needs unsupported-model classification that does not poison the whole profile.
