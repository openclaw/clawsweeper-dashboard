## Summary

Keep open. Current main still advertises the stale GitHub Copilot `claude-opus-4.6` model and still treats unrecognized 400/422 provider bodies as `format`, which the auth-profile cooldown path stores as a profile-wide cooldown. The open closing PR #75391 is a relevant implementation candidate, but it only removes one catalog entry and has not merged, so this issue should remain open for that PR to be reviewed, expanded, or superseded.

## What This Changes

The issue asks OpenClaw to remove or runtime-gate stale GitHub Copilot Claude defaults and prevent a single `model_not_supported` response from cooling down the entire Copilot auth profile.

## Best Possible Solution

Keep this issue open until the bundled Copilot catalog is made conservative or runtime-backed, and unsupported-model Copilot responses are classified as model-scoped unavailability so other Copilot models can continue using the same auth profile. PR #75391 can be the starting point if it is expanded beyond the single catalog deletion.

## Reproduction Assessment

Yes. The issue gives a concrete v2026.4.29 command and gateway log, and current main still contains the same stale model entry plus the static classifier/cooldown path that makes the rejection profile-wide.

## Solution Assessment

Yes, the requested two-layer direction is the narrow maintainable fix. Removing `claude-opus-4.6` alone is a useful mitigation but not the best complete fix because future Copilot catalog churn would still poison the whole provider profile.
