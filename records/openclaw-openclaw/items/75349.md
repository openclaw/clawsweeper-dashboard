Keep open. Current main still advertises `github-copilot/claude-opus-4.6`, still tests that default, and still clears model scope for non-rate-limit auth-profile cooldowns; open PR #75391 is linked with closing syntax but only covers the catalog half of the report.

## What This Changes

The issue requests removing stale `github-copilot/claude-opus-4.6` defaults and scoping Copilot `model_not_supported` failures to the rejected model instead of the whole auth profile.

## Best Possible Solution

Ship a complete Copilot provider fix that keeps the bundled catalog conservative or runtime-backed and treats unsupported-model rejections as model-scoped so healthy Copilot sibling models can still use the same auth profile.
