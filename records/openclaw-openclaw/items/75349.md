## Summary

Keep open. Current main still advertises `github-copilot/claude-opus-4.6`, still has tests that expect it in the Copilot defaults, and still records non-rate-limit failures as profile-wide cooldowns. The linked open PR #75391 is an implementation candidate for only the static catalog part, so this issue should remain open until that PR is expanded, superseded, or followed by a cooldown-scoping fix.

## What This Changes

The issue requests removing or runtime-filtering stale GitHub Copilot model defaults and handling `model_not_supported` as model-scoped unavailability instead of cooling down the whole Copilot auth profile.

## Best Possible Solution

Implement the two-part fix: make the bundled GitHub Copilot catalog conservative or runtime-backed, and classify Copilot `model_not_supported` responses as model-scoped unsupported/unavailable so sibling Copilot models can continue using the same auth profile. PR #75391 should be expanded to cover the cooldown behavior or superseded by a complete fix.

## Solution Assessment
