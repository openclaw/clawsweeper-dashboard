## Summary

Keep open. Current main still advertises `github-copilot/claude-opus-4.6`, still locks that default in tests and manifest metadata, and still clears model scope for non-rate-limit auth-profile cooldowns; open PR #75391 is linked with closing syntax but only covers the catalog half of the report.

## What This Changes

The issue asks to remove stale Copilot Claude defaults and prevent unsupported-model errors from cooling the whole GitHub Copilot auth profile.

## Best Possible Solution

Ship a complete Copilot provider fix that keeps the bundled catalog conservative or runtime-backed and treats unsupported-model rejections as model-scoped so healthy Copilot sibling models can still use the same auth profile.

## Reproduction Assessment

Yes. The reporter provided a concrete v2026.4.29 command and gateway log, and current main still contains the stale advertised model plus the profile-wide non-rate-limit cooldown path; no Copilot token was visible in env/profile for a fresh live API call.

## Solution Assessment

Yes for the issue's two-layer direction. Removing the stale default mitigates the immediate bad catalog entry, but the maintainable fix also needs `model_not_supported` or equivalent model-unavailable classification that does not poison the whole profile.
