## Summary

Keep open. Current main still advertises `github-copilot/claude-opus-4.6`, and the reported `400 model_not_supported` shape still falls through to a non-rate-limit `format` failure path that records a provider-profile cooldown. The open closing PR is an active implementation candidate, but it has not merged and appears catalog-only, so the issue should remain open for review or expansion.

## What This Changes

The issue asks to remove stale GitHub Copilot model defaults such as `claude-opus-4.6` and prevent unsupported-model errors from cooling down the entire Copilot auth profile.

## Best Possible Solution

Keep this tracked until the Copilot plugin has a conservative or runtime-backed model catalog and unsupported-model responses are classified as model-scoped unavailability so other Copilot models can keep using the same profile. The existing PR should either be expanded to cover both layers or superseded by a complete provider-owned fix.

## Reproduction Assessment

Yes. The reporter provided a concrete v2026.4.29 command and logs, and current main still has the stale model plus deterministic source paths that map an unrecognized Copilot `400 model_not_supported` response to a non-rate-limit profile-wide cooldown.

## Solution Assessment

Yes for the issue's two-layer direction; no for a catalog-only fix as the complete solution. Removing `claude-opus-4.6` is a useful mitigation, but the maintainable fix also needs unsupported-model classification that does not poison the entire Copilot auth profile.
