## Summary

Keep open. Current main still advertises `github-copilot/claude-opus-4.6`, still has tests that lock that default in, and still treats non-rate-limit Copilot 400 failures as profile-wide cooldowns; linked PR #75391 is open and only addresses the catalog half.

## What This Changes

The issue asks to remove unsupported `github-copilot/claude-opus-4.6` defaults and stop a `model_not_supported` Copilot 400 from cooling down the whole provider profile.

## Best Possible Solution

Ship or expand the linked Copilot fix so the bundled catalog is conservative or runtime-backed and unsupported-model rejections are handled as model-scoped fallback rather than profile-wide auth health.

## Reproduction Assessment

Yes. The reporter provided a concrete v2026.4.29 command and gateway log, and static inspection of current main still shows the stale advertised model plus the profile-wide non-rate-limit cooldown path.

## Solution Assessment

Yes for the issue's two-layer direction. Removing `claude-opus-4.6` is the immediate mitigation, but the maintainable fix also needs `model_not_supported` or equivalent model-unavailable classification that does not poison the whole Copilot profile.
