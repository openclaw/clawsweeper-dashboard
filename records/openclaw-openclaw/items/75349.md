## Summary

Keep open. Current main still advertises `github-copilot/claude-opus-4.6`, and unsupported Copilot 400 responses still fall into a non-rate-limit failure path that clears model scoping and can cooldown the provider profile. Implement the fix in two layers: make the bundled Copilot catalog conservative or runtime-backed, and classify Copilot unsupported-model responses as model-scoped unavailability so other Copilot models can keep using the same auth profile. PR #75391 should be expanded to cover the cooldown behavior or replaced by a complete fix.
