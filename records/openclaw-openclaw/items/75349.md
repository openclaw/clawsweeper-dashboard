## Summary

Keep open. Current main still advertises `github-copilot/claude-opus-4.6`, and the current unsupported-model path still records non-rate-limit failures as profile-wide auth-profile cooldowns. Open PR #75391 is an unmerged catalog-only candidate, so this issue should stay open until a complete fix lands or that PR is expanded/superseded.

## What This Changes

The issue reports that `github-copilot/claude-opus-4.6` is still advertised after Copilot dropped support and that the resulting `model_not_supported` 400 marks the whole Copilot auth profile in cooldown.

## Best Possible Solution

Ship a complete provider-owned fix: make the bundled Copilot catalog conservative or runtime-backed, and treat Copilot unsupported-model responses as model-scoped unavailability so other Copilot models can continue using the same auth profile.

Yes. The issue provides a concrete v2026.4.29 command and gateway log, and current main still contains the advertised model plus the source path that classifies an unrecognized 400 as `format` and records non-rate-limit failures as profile-wide cooldowns.

## Solution Assessment

Yes, the requested two-part direction is the maintainable fix. Removing only `claude-opus-4.6` is a useful mitigation but not sufficient because unsupported-model failures would still poison the provider profile when another Copilot model rotates out.
