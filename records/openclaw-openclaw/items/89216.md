## Label Justifications

- `impact:session-state`: The reported failure creates or uses sessions under `agent:main` instead of the configured room agent.

## Summary

Keep open. Current main and latest release still preserve the default-only behavior for omitted binding account IDs, Matrix passes its active account ID into shared routing, and the open closing PR remains the implementation candidate rather than a merged fix.

## What This Changes

Matrix per-room agent bindings without an explicit `accountId` are reported to be ignored for named Matrix accounts, causing room messages to route to the default `main` agent.

## Best Possible Solution

Fix the Matrix named-account room-binding path while preserving the documented shared contract that omitted `match.accountId` means the default account and `accountId: "*"` means any account.

## Reproduction Assessment

Yes. Current source and tests show that omitted `match.accountId` is scoped to `default`; Matrix passes a named account such as `ops` into the shared resolver, so that binding shape misses the named-account route.

## Solution Assessment

No for the broad wildcard approach in the linked PR. The best fix should be Matrix-scoped or account-selection scoped while preserving explicit `accountId: "*"` as the shared any-account contract.
