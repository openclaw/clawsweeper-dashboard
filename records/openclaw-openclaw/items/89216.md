## Summary

Keep open. Current main still leaves a plausible Matrix default-account routing gap: omitted `match.accountId` bindings are indexed under the literal `default` account, while Matrix can route a single named account as the implicit default; the earlier related report was closed for inactivity, not by a fix.

## What This Changes

The issue reports Matrix room messages ignoring configured per-room agent bindings and always creating sessions under the default agent.

## Best Possible Solution

Make omitted `match.accountId` honor the effective channel default account for Matrix named-default room routing, with regression coverage that preserves explicit and wildcard account behavior.

## Reproduction Assessment

Yes by source inspection: a Matrix single named account can become the implicit default account, while a binding with omitted `match.accountId` is still indexed under literal `default` and is not considered for that named account. I did not run a live Matrix transport reproduction.

## Solution Assessment

Yes, the maintainable fix is a focused route-level regression plus a narrow default-account binding repair or Matrix normalization fix. It should not add a new config option or broaden omitted account bindings to every account.
