## Summary

Keep open. Current main covers explicit Matrix account bindings, but the reported binding shape omits `match.accountId`; source inspection still shows that omitted-account bindings are indexed under literal `default` and can be missed when Matrix selects a single named account as the effective account.

## What This Changes

Matrix per-room bindings should route room messages to the configured agent instead of creating sessions under the default agent.

## Best Possible Solution

Keep this as the canonical issue and add a narrow Matrix-side routing fix that treats omitted `match.accountId` as the effective Matrix default or only account without broadening the shared core contract to every account.

## Reproduction Assessment

Yes, by source inspection. Current main can run Matrix as a single named account while omitted `match.accountId` bindings are only considered for the literal `default` account, which matches the reported missed binding shape.

## Solution Assessment

Yes. The best fix is a focused Matrix/plugin routing repair plus regression coverage, not a new config option or a broad core change that makes omitted account bindings match every account.
