## Summary

Keep open: this is a focused bug fix, but it is not merge-ready because the new fixed 250 ms empty-final fallback can still beat the Gateway's async source-reply final and leave the original Talk voice/UI divergence possible.

## What This Changes

The PR keeps the Control UI realtime Talk consult listener open after an empty chat final and adds a regression test so a later delivery-mirror final can supply the realtime tool result.

## Best Possible Solution

Make Talk consult resolution wait for the canonical source-reply or delivery-mirror final deterministically, while preserving a true no-text fallback only after the Gateway can no longer emit text for that run.

## Reproduction Assessment

Yes. Source inspection shows current main resolves the Talk consult on the first empty final, while the Gateway can later broadcast the source-reply final from async chat-send post-dispatch work.

## Solution Assessment

No. A fixed 250 ms grace window improves the immediate case but is not the safest fix because it can still resolve before the canonical source-reply final arrives.
