## Summary

Keep open: the PR targets a valid Telegram media-ingest bug, but this patch is not merge-ready because it omits wildcard/default ingest precedence, can turn passive ingest media failures into visible group error replies, and lacks live Telegram hook proof.

## What This Changes

The PR adds an early Telegram media pre-download bypass when `topicConfig?.ingest ?? groupConfig?.ingest` is true for unaddressed mention-gated group media.

## Best Possible Solution

Align the media skip gate with the text path's topic, exact group, and wildcard/default ingest precedence; preserve no-visible-reply behavior for passive ingest failures; add focused regressions and redacted live Telegram proof.

## Reproduction Assessment

Yes at source level: current main can return from the media pre-download skip before any ingest check, while the sibling text path honors topic, exact group, and wildcard/default ingest. I did not run a live Telegram session in this read-only review.

## Solution Assessment

No. The fix touches the right helper, but the best fix should mirror the full text-path ingest hierarchy and preserve silent media failure handling for ingest-bypassed unaddressed media.
