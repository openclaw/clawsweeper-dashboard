## Summary

Keep open. The alias work is useful and has real terminal proof, but the current PR head still needs a focused refresh/repair because it collides with current main's Docker browser build-arg contract and introduces two public-docs terminology regressions.

This PR adds `OPENCLAW_IMAGE_APT_PACKAGES` as the preferred Docker/Podman apt-package image-build variable while retaining `OPENCLAW_DOCKER_APT_PACKAGES` as a legacy fallback and updating related docs/tests.

## Best Possible Solution

Refresh the branch onto current main, keep the backward-compatible apt-package alias, preserve the Docker browser build-arg path, and use plugin terminology in public install docs.

## Reproduction Assessment

Not applicable as a bug reproduction target; this is a feature/config alias PR. The review blockers are source-reproducible from current main and the PR diff: current main guards `OPENCLAW_INSTALL_BROWSER`, while the stale branch edits the same build-arg path without preserving it.

## Solution Assessment

Yes, with repairs: a preferred runtime-agnostic alias with a legacy fallback is a narrow maintainable solution. The safer merge path is to restore the browser build arg during refresh and replace internal extension wording in public docs.
