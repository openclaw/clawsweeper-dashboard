The PR should stay open because the alias implementation is useful but the current branch still has two merge-blocking review issues: it drops existing Docker browser build-arg wiring and uses internal extension wording in public install docs.

## Best Possible Solution

Land a refreshed branch that keeps the backward-compatible apt alias, preserves the existing Docker browser build-arg contract, updates focused coverage, and uses plugin terminology in public docs.

## Solution Assessment
