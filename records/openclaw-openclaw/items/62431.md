## Summary

This PR should stay open: the alias-and-fallback design is useful and the PR body has real terminal proof, but the branch still drops current-main Docker browser build-arg wiring and uses internal extension wording in public install docs.

## What This Changes

The branch adds `OPENCLAW_IMAGE_APT_PACKAGES` as the preferred Docker/Podman apt-package build argument, keeps `OPENCLAW_DOCKER_APT_PACKAGES` as a fallback, and updates the Dockerfile, setup scripts, docs, and Docker setup e2e coverage.

## Best Possible Solution

Land a refreshed branch that keeps the alias-with-fallback design, preserves Docker browser setup wiring, uses plugin wording in public install docs, and passes focused Docker/docs validation.

## Reproduction Assessment

Yes for the review blockers: source inspection shows PR head omits `OPENCLAW_INSTALL_BROWSER` from Docker setup export/upsert/build-arg sections while current main includes it, and the docs terminology mismatch is visible in the PR head. The requested alias itself is a feature, so no bug reproduction applies to the main change.

## Solution Assessment

Yes, with repairs: the alias-with-fallback approach is a narrow backward-compatible way to introduce a runtime-neutral name. The branch should first preserve current-main browser setup wiring and replace public `extension` wording with plugin terminology.
