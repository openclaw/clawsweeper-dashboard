## Summary

Keep open: the alias change is useful and has terminal proof, but the branch still drops existing Docker browser build-arg wiring and uses internal extension wording in public install docs.

## What This Changes

The branch adds `OPENCLAW_IMAGE_APT_PACKAGES` as the preferred Docker/Podman apt build variable, keeps `OPENCLAW_DOCKER_APT_PACKAGES` as a fallback, and updates setup scripts, docs, and Docker setup coverage.

## Best Possible Solution

Refresh the branch onto current main, keep the backward-compatible apt-package alias, restore Docker browser build-arg wiring, and use plugin terminology in public docs.

## Reproduction Assessment

Yes for the review blockers: current main source and tests show Docker setup must carry `OPENCLAW_INSTALL_BROWSER`, while PR head omits it, and the docs wording mismatch is visible in changed install pages. The requested apt alias itself is a feature/config addition rather than a bug reproduction target.

## Solution Assessment

Yes, with repairs: a preferred alias with legacy fallback is a narrow maintainable solution. The branch should first restore the existing browser setup contract and replace public `extension` wording with plugin wording.
