## Summary

This PR adds `OPENCLAW_IMAGE_APT_PACKAGES` as the preferred Docker/Podman apt-package build variable, keeps `OPENCLAW_DOCKER_APT_PACKAGES` as a fallback, and updates Dockerfile, setup scripts, docs, and Docker setup tests.

## Best Possible Solution

Refresh the branch onto current main, keep the apt-package alias and fallback behavior, restore the existing Docker browser build-arg contract, and use plugin terminology in public install docs.

## Reproduction Assessment

Yes for the review blockers: current-main source and tests show `OPENCLAW_INSTALL_BROWSER` is still part of Docker setup, while the PR head's edited build-arg path omits it. The requested apt alias is a feature/config addition, not a bug reproduction target.

## Solution Assessment

Yes, with repairs: a preferred alias with legacy fallback is a narrow backward-compatible approach. Restoring the browser build arg and replacing internal extension wording are the safer merge path.
