## Best Possible Solution

Land a refreshed branch that keeps the alias-with-fallback design, preserves current-main browser setup wiring, uses plugin terminology in user docs, and has focused Docker/docs checks green.

Not applicable for the requested feature itself; it adds a preferred env-var alias rather than fixing broken existing behavior. The review blockers are source-reproducible by comparing current main with the PR head.
