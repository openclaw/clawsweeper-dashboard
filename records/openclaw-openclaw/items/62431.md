The PR adds `OPENCLAW_IMAGE_APT_PACKAGES` as the preferred Docker/Podman apt-package build variable across the Dockerfile, setup scripts, install docs, and Docker setup e2e tests while retaining the legacy Docker-named fallback.

## Best Possible Solution

Land a refreshed branch that keeps the alias-with-fallback design, preserves current Docker browser behavior, uses plugin wording in public docs, and passes focused Docker setup validation.

## Reproduction Assessment

Yes for the review blockers: source comparison between PR head and current main shows the missing Docker browser wiring/cache ENV and the public docs terminology mismatch. The requested alias itself is a feature, so there is no bug reproduction path for the main change.

## Solution Assessment

Yes with repairs: a backward-compatible preferred alias is a narrow maintainable solution for runtime-neutral Docker/Podman naming. The branch should first be rebased or updated to preserve current-main Docker browser contracts and replace `extension` wording with plugin terminology.
