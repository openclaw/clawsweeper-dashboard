## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/74907. The observable bug is that Claude CLI/OAuth live sessions can replay a prior assistant turn with multiple tool calls without immediately adjacent matching tool results, producing Anthropic orphan `tool_use` 400s on the next user message. Scope the fix to the Claude CLI live-session/replay boundary: `src/agents/cli-runner/claude-live-session.ts`, `src/agents/cli-runner/execute.ts`, and related session-history helpers/tests. Do not churn the direct Anthropic HTTP converter unless a failing test proves that path is involved; it already groups consecutive `toolResult` records. Add focused regression coverage for a retained assistant message with 2+ tool calls plus split `toolResult` records followed by another user turn, and assert the live/replay path cannot emit orphan tool-use history. Preserve existing warm live-session behavior where possible, document any intentional replay flag change, add a single-line user-facing `CHANGELOG.md` fix entry, and run the targeted tests plus `pnpm check:changed` in Testbox before handoff.
