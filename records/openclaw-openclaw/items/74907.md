## Summary

Keep open. Current main still routes the affected Claude CLI OAuth path through live stdio replay that writes only the current user prompt while relying on Claude's replay machinery, and the existing tests do not cover a retained assistant turn with multiple tool calls followed by split tool-result records.

## What This Changes

Investigate and fix a 2026.4.x replay regression where Claude/Anthropic sessions containing multi-tool assistant turns can send orphan `tool_use` blocks on a later user message.

## Best Possible Solution

Add focused regression coverage for Claude CLI live stdio replay after a parallel tool-call turn, then fix replay so retained assistant tool calls have all matching results adjacent/grouped or are pruned symmetrically when results are unavailable.

## Reproduction Assessment

Yes. The issue discussion gives a high-confidence fresh-session reproduction: use `anthropic:claude-cli` OAuth, trigger one assistant turn with multiple parallel tool calls, then send any follow-up user message and observe Anthropic reject orphan `tool_use` IDs.

## Solution Assessment

Unclear because there is no proposed patch. The narrowest maintainable fix appears to be at the Claude CLI live-session/replay boundary, while keeping the already-correct direct Anthropic HTTP converter unchanged except for shared helpers if needed.
