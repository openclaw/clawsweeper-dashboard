## Summary

Keep open. Current main still routes Anthropic Claude CLI OAuth sessions through warm `claude-stdio` with `--replay-user-messages`; the direct Anthropic converter already groups split tool results, but the live stdin route bypasses that converter and only writes the current user prompt, with no regression coverage for retained parallel tool-call turns.

## What This Changes

The issue asks OpenClaw to stop replaying Anthropic/Claude sessions with assistant `tool_use` blocks whose matching split `toolResult` records are not adjacent or grouped after multi-tool turns.

## Best Possible Solution

Add focused regression coverage for Claude CLI live stdio replay after a parallel tool-call turn, then make that boundary preserve all matching tool results adjacent/grouped or avoid replaying retained assistant tool calls when results are missing.

## Reproduction Assessment

Yes. The reporter gives a high-confidence fresh-session path, and current main still contains the same Claude CLI live stdio replay route needed to exercise it.

## Solution Assessment

Unclear because there is no proposed patch. The narrow maintainable direction is to fix the Claude CLI live-session replay boundary while leaving the already-correct direct Anthropic HTTP converter behavior intact.
