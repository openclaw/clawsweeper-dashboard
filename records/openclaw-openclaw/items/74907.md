## Summary

Keep open. The report is current and still matches the current Claude CLI live-session path: the direct Anthropic Messages converter already groups consecutive tool results, but the OAuth Claude CLI route still runs through `claude-stdio` with `--replay-user-messages` and writes only the current user message into stdin on follow-up turns.

## What This Changes

The issue asks OpenClaw to preserve Anthropic tool-use/tool-result pairing when replaying sessions that contain assistant turns with multiple parallel tool calls, especially on the Claude CLI OAuth live-session route.

## Best Possible Solution

Add a focused regression around Claude CLI live stdio replay after a parallel tool-call turn, then fix that boundary so every retained assistant tool call is replayed with all matching results adjacent/grouped or the assistant tool calls are pruned together with missing results. Keep the direct Anthropic HTTP converter's existing grouping behavior intact and check whether GitHub Copilot Claude needs the same invariant.

## Reproduction Assessment

Yes. The reporter gives a high-confidence fresh-session path: start a new Claude CLI OAuth session, trigger one assistant turn with multiple parallel tool calls, then send any follow-up user message and observe Anthropic rejecting orphan `tool_use` IDs.

## Solution Assessment

Unclear because there is no proposed patch, but the narrowest maintainable fix appears to be in the Claude CLI live-session/stdin replay boundary, with shared replay helpers reused only if they can be applied without changing the direct Anthropic converter behavior.
