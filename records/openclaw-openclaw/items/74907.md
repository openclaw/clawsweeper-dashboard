## Summary

Keep open. Current main still routes Claude CLI OAuth through the live stdio path with `--replay-user-messages`, writes only the current user prompt to stdin, and lacks regression coverage for replaying a retained assistant turn with multiple tool calls plus split tool-result records.

## What This Changes

Fix a 2026.4.x Claude/Anthropic replay regression where sessions containing multi-tool assistant turns can later send orphan `tool_use` blocks.

## Best Possible Solution

Add focused regression coverage for Claude CLI live stdio replay after a parallel tool-call turn, then repair replay so retained assistant tool calls have all matching results adjacent/grouped or are pruned symmetrically when results are unavailable.

## Reproduction Assessment

Yes. The issue discussion gives a high-confidence fresh-session path: use `anthropic:claude-cli` OAuth, trigger one assistant turn with multiple parallel tool calls, then send any follow-up user message and observe Anthropic reject orphan `tool_use` IDs.

## Solution Assessment

Unclear because there is no proposed patch. The narrowest maintainable fix appears to be at the Claude CLI live-session/replay boundary, while keeping the already-correct direct Anthropic HTTP converter unchanged except for shared helpers if needed.
