## Summary

Keep open. Current main still routes Anthropic OAuth/Claude CLI sessions through the Claude live stdio path with `--replay-user-messages` while writing only the current user prompt, and the already-correct direct Anthropic HTTP converter does not cover that path.

## What This Changes

Fix replay for sessions whose retained assistant turns contain multiple parallel tool calls followed by split tool-result records, so follow-up Anthropic/Claude turns do not produce orphan `tool_use` blocks.

## Best Possible Solution

Land a focused Claude CLI live-session replay fix that preserves or reconstructs strict tool-use/tool-result adjacency for retained multi-tool turns, leaving the already-correct direct Anthropic HTTP converter mostly unchanged.

## Reproduction Assessment

Yes. The issue provides a high-confidence fresh-session reproduction, and source inspection shows current main still uses the implicated Claude live stdio route with single-prompt writes on follow-up turns.

## Solution Assessment

No patch is proposed yet. The direct HTTP converter suggestion is not the best remaining target because current main already groups direct Anthropic `toolResult` records; the narrow fix belongs in the Claude CLI live stdio replay/resume boundary.
