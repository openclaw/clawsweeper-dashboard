## Summary

Keep open. Current main still matches the reported failing route: OAuth Claude CLI uses the long-lived stdio live-session path with replay enabled, while the only confirmed grouping repair is in the direct Anthropic Messages HTTP converter and shared embedded-runner hygiene. The issue is recent, detailed, corroborated by another Claude provider report, and has a narrow likely repair boundary.

## What This Changes

Fix Anthropic/Claude replay after parallel tool-call turns so retained assistant tool uses are followed by all matching tool results, avoiding provider 400s on later user messages.

## Best Possible Solution

Add a focused regression for Claude CLI live stdio replay after a parallel tool-call turn, then fix that boundary so replayed history either keeps every assistant tool call adjacent to all matching results or prunes both sides symmetrically. Keep the existing direct Anthropic HTTP converter grouping behavior intact, and check whether the GitHub Copilot Claude path shares the same strict replay invariant.
