Keep open. ## What This Changes

The issue reports that v2026.4.x Claude/Anthropic session replay can resend parallel assistant tool uses without the matching grouped tool results, causing Anthropic 400 errors after a follow-up user turn.

## Best Possible Solution

Add a focused regression for Claude CLI live stdio replay after a parallel tool-call turn, then fix that boundary so every retained assistant tool call is replayed with all matching results adjacent/grouped, or the assistant tool calls are pruned together with missing results. Keep the direct Anthropic HTTP converter’s existing grouping behavior intact and separately check whether the GitHub Copilot Claude comment hits the same invariant.
