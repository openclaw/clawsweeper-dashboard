## Summary

Keep open. Current main still routes Claude CLI OAuth through the live `claude-stdio` path with `--replay-user-messages` and writes only the current prompt to stdin; the direct Anthropic HTTP converter and newer replay sanitizers do not prove the affected live CLI replay path is fixed.

## What This Changes

The issue reports an OpenClaw 2026.4.x regression where Anthropic/Claude CLI replay of retained multi-tool turns can send assistant `tool_use` blocks without all matching `tool_result` blocks immediately after them.

## Best Possible Solution

Land a focused Claude CLI live-session replay fix that preserves or reconstructs strict tool-use/tool-result adjacency for retained multi-tool turns, with regression coverage for the live stdio path.

## Reproduction Assessment

Yes. The reporter provides a fresh-session repro, Anthropic documents the strict adjacency requirement, and current main still uses the implicated Claude live stdio route with single-prompt writes; I did not run the live OAuth setup in this pass.
