## Summary

Keep open. Current main still leaves the reported Claude CLI OAuth replay path actionable: the direct Anthropic HTTP converter and generic transcript repair have pairing logic, but the live `claude-stdio` path still hands each follow-up prompt to Claude's own replay machinery and lacks focused coverage for retained multi-tool assistant turns followed by split tool results.

## What This Changes

The issue asks OpenClaw to stop replaying multi-tool Claude/Anthropic history in a shape that leaves assistant `tool_use` blocks without immediately matching `tool_result` blocks on later turns.

## Best Possible Solution

Add regression coverage at the Claude CLI live stdio replay boundary and make that path preserve or synthesize all matching tool results adjacent to retained assistant tool calls, or prune the assistant tool calls symmetrically when results are unavailable.

## Reproduction Assessment

Yes. The issue discussion gives a concrete fresh-session path: use Claude CLI OAuth, trigger one assistant turn with multiple parallel tool calls, then send any follow-up user message and observe Anthropic reject orphan `tool_use` IDs.

## Solution Assessment

Yes, the narrowest maintainable direction is to fix the Claude CLI live-session replay boundary; the direct Anthropic converter and shared sanitizer should only change if a common helper is needed for that path.
