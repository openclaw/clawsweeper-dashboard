## Summary

Keep this issue open. Current main still has a Control UI local slash-command dispatch gap: normal `/think` executor failures are handled, but the UI dispatch path can still clear the composer and then produce no visible message when the browser client is unavailable or an unexpected dispatch error escapes.

## What This Changes

The issue reports that the Control UI swallows a bare `/think` command with no visible response after the input is cleared.

## Best Possible Solution

Land a focused Control UI dispatch fix that makes every locally executed slash command render inline feedback when it cannot run or unexpectedly fails, while preserving the existing `/think` executor behavior and adding regression coverage around `handleSendChat`.

## Reproduction Assessment

Yes for the remaining dispatch-layer failure class: current source shows a local slash command can clear the composer before dispatch, then return silently when `host.client` is absent or let an unexpected `executeSlashCommand` rejection escape. The exact original browser stack is still unavailable.

## Solution Assessment

Yes, the narrowest maintainable fix is in `ui/src/ui/app-chat.ts`: inject a visible not-connected message and wrap unexpected local-command dispatch failures before result injection. Changing only the `/think` executor would miss the UI dispatch gap that the discussion and current source identify.
