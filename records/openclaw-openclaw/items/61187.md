## Summary

Keep open: this PR addresses a real Kimi setup-picker gap, but current main has moved under it. Kimi now has a stable `kimi-for-coding` id, Moonshot rows are already manifest-owned, and the added Kimi test imports a removed helper, so the branch needs narrow repair before merge.

## What This Changes

The PR adds Kimi and Moonshot `augmentModelCatalog` hooks plus tests/export changes so setup model picker rows appear without API-key-backed discovery.

## Best Possible Solution

Keep Moonshot on the manifest/static catalog path, and expose only the current Kimi no-key picker row for `kimi-for-coding` with SDK-surface regression coverage.

## Reproduction Assessment

Yes by source inspection. Current Kimi returns no catalog without an API key and has no manifest model rows, while the preferred-provider picker falls back to the broader catalog when no static Kimi rows exist; Moonshot no longer reproduces because current main has static manifest rows.

## Solution Assessment

No. The Kimi direction is the right narrow fix, but this branch should align with the stable Kimi id, remove the obsolete Moonshot augment/export/test path, and use the SDK plugin test runtime.
