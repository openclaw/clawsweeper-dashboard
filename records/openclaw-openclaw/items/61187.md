## Summary

Keep this PR open, but it still needs changes before merge: the Kimi bug is real, while the proposed runtime-hook fix uses a deprecated path, exposes legacy model aliases, and adds obsolete Moonshot changes that current main already covers through manifest catalog rows.

## What This Changes

The PR adds Kimi and Moonshot `augmentModelCatalog` hooks, exports Moonshot catalog data for the hook, and adds hook-level tests.

## Best Possible Solution

Add Kimi manifest `modelCatalog` metadata for the canonical `kimi-for-coding` row, keep legacy Kimi IDs as normalization-only compatibility, and rely on the existing Moonshot manifest catalog.

## Reproduction Assessment

Yes. Source inspection shows Kimi has no no-key manifest catalog rows while its runtime catalog returns `null` without an API key; the Moonshot half no longer reproduces because current main already has static manifest rows.

## Solution Assessment

No. The PR's runtime-hook approach is not the best fix because the hook is deprecated and the current catalog path should be manifest-backed, with Moonshot left unchanged.
