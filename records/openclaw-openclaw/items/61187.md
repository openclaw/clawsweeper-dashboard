## Summary

Keep open: the PR addresses a real Kimi setup picker gap, but current main shows Moonshot already has a manifest/static picker path and the branch still promotes Kimi compatibility aliases instead of the stable model row.

## What This Changes

The branch adds Kimi and Moonshot `augmentModelCatalog` hooks, exports Moonshot catalog data, and adds provider catalog hook tests.

## Best Possible Solution

Keep Moonshot on the manifest/static catalog path, expose only the stable Kimi no-key picker row, and cover that behavior with current SDK-runtime tests.

## Reproduction Assessment

Yes. Source inspection shows Kimi has no no-key catalog source on current main, and the PR screenshots show the setup picker falling back to unrelated provider rows; the Moonshot half no longer reproduces because current main has manifest-backed static rows.

## Solution Assessment

No. The PR points at the right Kimi symptom, but the maintainable fix is Kimi-only stable catalog exposure while leaving Moonshot on its existing manifest/static path and using current SDK test helpers.

  - confidence: 0.92
