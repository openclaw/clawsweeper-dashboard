## Summary

Keep this PR open because it is a plausible bug fix for Kimi's no-key model picker gap, but the current patch should not merge as written. The useful Kimi part needs to move to manifest/control-plane catalog metadata, while the Moonshot changes are obsolete on current main and the added Kimi test uses a removed helper path.

## What This Changes

Adds runtime `augmentModelCatalog` hooks for Kimi and Moonshot provider plugins, exports the Moonshot catalog, and adds hook-level tests for those catalog rows.

## Best Possible Solution

Add Kimi manifest/control-plane catalog metadata for only the canonical `kimi-for-coding` picker row, keep legacy ids as normalization compatibility, leave Moonshot on its manifest catalog, and cover the picker path with focused tests.

## Reproduction Assessment

Yes, from source inspection rather than a live run: Kimi has no manifest `modelCatalog` and its runtime catalog returns `null` without an API key, so the selected-provider picker has no static Kimi row before auth. The Moonshot half does not reproduce on current main because Moonshot already has static manifest catalog rows and the picker prefers them.

## Solution Assessment

No. The PR uses deprecated `augmentModelCatalog`, duplicates Moonshot's manifest-backed catalog path, and exposes Kimi legacy aliases as primary picker choices; the narrower maintainable fix is manifest/control-plane Kimi metadata for the canonical model.
