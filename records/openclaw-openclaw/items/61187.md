## Summary

Keep open: the reported picker problem is real for Kimi on current main, but this PR should not merge as written because it uses a deprecated runtime catalog hook, re-adds Moonshot rows that current main already serves from manifest metadata, and adds a stale test helper import.

## What This Changes

The branch adds Kimi and Moonshot `augmentModelCatalog` runtime hooks, exports Moonshot catalog data for that hook, and adds hook-level tests.

## Best Possible Solution

Add Kimi manifest `modelCatalog` metadata for the canonical `kimi-for-coding` row, keep legacy Kimi IDs as normalization compatibility, and leave Moonshot on its existing manifest-backed catalog.

## Reproduction Assessment

Yes. Source inspection shows Kimi has no no-key manifest catalog rows while its runtime catalog returns `null` without an API key; Moonshot no longer reproduces because current main already has static manifest rows.

## Solution Assessment

No. The PR's runtime-hook approach is not the best fix because the hook is deprecated and the correct current-main path is manifest-backed catalog metadata, with Moonshot left unchanged.
