## Best Possible Solution

Keep Moonshot's manifest/static catalog as the source of truth, and add only the current Kimi offline picker row for kimi-for-coding with SDK-surface regression coverage.

## Solution Assessment

No, this PR is not the best solution as-is. The Kimi intent is useful, but the branch should be rebased to current stable model IDs and SDK test imports, and the stale Moonshot augment/export/test changes should be removed.
