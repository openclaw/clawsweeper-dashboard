## Summary

Keep open: the reported Kimi setup picker problem is real, but this branch is stale against current main and would add obsolete Kimi rows, duplicate Moonshot catalog ownership, and a failing test import.

## What This Changes

The PR adds Kimi and Moonshot provider model-catalog augmentation hooks plus tests so the setup model picker can show provider-specific models before credentials are configured.

## Best Possible Solution

Keep Moonshot catalog ownership in the manifest/static catalog path, add a Kimi no-key picker source only for stable kimi-for-coding, and cover the flow with SDK-surface provider tests plus model-picker regression coverage.

## Reproduction Assessment

Yes, by source inspection and the PR screenshots: Kimi has no no-key catalog source on current main, so the provider-scoped picker can fall back to unrelated rows. Moonshot now has manifest/static catalog ownership, so the branch's Moonshot runtime augment path should be dropped or replaced by the current catalog path.

## Solution Assessment

No. The Kimi direction is right, but the patch should expose only the stable Kimi row, avoid a duplicate stale Moonshot catalog source, and use the current SDK plugin test runtime.
