## Summary

Keep open: the Kimi picker symptom is valid, but this PR should not merge as-is because it exposes legacy Kimi aliases, adds a deprecated Moonshot runtime catalog path that current main already supersedes with manifest rows, and uses a stale test helper import.

## What This Changes

The PR adds Kimi and Moonshot `augmentModelCatalog` hooks, exports Moonshot catalog data, and adds provider catalog hook tests.

## Best Possible Solution

Expose the Kimi stable no-key picker row through the current manifest/control-plane catalog surface, leave Moonshot on its static manifest catalog, and keep legacy Kimi ids as normalization-only compatibility.

## Reproduction Assessment

Yes for the Kimi half by source inspection: current main returns no Kimi catalog without an API key and the Kimi manifest has no `modelCatalog` rows, while the PR screenshots exercise the setup picker. The Moonshot half no longer reproduces on current main because Moonshot static manifest rows already feed the picker.

## Solution Assessment

No. The PR uses deprecated runtime augment hooks and publishes legacy Kimi aliases; the narrower maintainable fix is Kimi-only stable catalog exposure through the current manifest/control-plane path, with Moonshot left on existing manifest rows.

  - body: This hook maps every model from `buildKimiCodingProvider()` into setup picker rows, including legacy aliases `kimi-code` and `k2p5`.   - body: Moonshot already declares static manifest rows, including the current `kimi-k2.6` default, and the picker reads those rows before runtime fallback. Adding a runtime `augmentModelCatalog` hook from an older catalog source can duplicate or drift from the manifest-backed picker rows.
  - confidence: 0.92

  - body: The added Kimi test imports `registerSingleProviderPlugin` from `../../test/helpers/plugins/plugin-registration.js`, but that path is not present on current main. Bundled provider tests import this helper from `openclaw/plugin-sdk/plugin-test-runtime`, so the test will fail after rebase unless it uses the public SDK helper.
  - confidence: 0.93
