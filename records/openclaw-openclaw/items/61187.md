## Summary

Keep open: current main still has the Kimi no-key picker gap, but this branch is not correct against current main because Kimi now has a stable model id and Moonshot already has manifest-owned catalog rows.

## What This Changes

Adds Kimi and Moonshot `augmentModelCatalog` hooks, exports Moonshot catalog data, and adds provider hook tests so setup model picker rows appear without API-key-backed discovery.

## Best Possible Solution

Expose a single current Kimi `kimi-for-coding` no-key picker row, keep legacy ids as normalization-only compatibility, and leave Moonshot on its manifest-owned static catalog.

## Reproduction Assessment

Yes, by source inspection. Kimi returns no runtime catalog without an API key and has no manifest catalog rows, so the preferred-provider picker can fall back to the broader catalog; Moonshot no longer reproduces because current main has manifest rows.

## Solution Assessment

No. The Kimi direction is narrow, but this branch should expose the stable Kimi row only, remove the obsolete Moonshot augment/export/test path, and use the public SDK plugin test runtime.
