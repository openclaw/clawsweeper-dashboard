## Summary

Keep open: the Kimi picker bug is still valid, but this branch is not the right fix because it uses a deprecated runtime augmentation path, exposes legacy Kimi aliases as picker choices, and changes Moonshot even though current main already has manifest-backed Moonshot rows.

## What This Changes

Adds Kimi and Moonshot `augmentModelCatalog` hooks with hook-level tests and exports Moonshot catalog data for reuse.

## Best Possible Solution

Add Kimi manifest `modelCatalog` metadata for the canonical `kimi-for-coding` row, keep legacy Kimi ids as normalization-only compatibility, and leave Moonshot on its existing manifest catalog rows.

## Reproduction Assessment

Yes, source-reproducible. Current main's Kimi manifest has no static catalog rows and the Kimi runtime catalog returns `null` without an API key, while Moonshot no longer reproduces because it already has manifest rows.

## Solution Assessment

No. The branch uses the deprecated runtime augmentation hook and touches Moonshot unnecessarily; the narrower maintainable fix is manifest-backed Kimi catalog metadata only.
