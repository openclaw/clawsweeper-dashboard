## Summary

Keep this open. Current main and the latest shipped release still model each exec approval completion as its own follow-up agent turn, while the remaining fix needs maintainer policy on cancellation, batching, user-input priority, and audit preservation.

## What This Changes

The issue requests stopping async exec approval completion cascades so stale completion turns cannot block new user commands after a task is done.

## Best Possible Solution

Keep this as the canonical approved-completion cascade report and design a core scheduling rule that can coalesce or cancel stale exec approval follow-ups while letting stop/new user input preempt safely.

## Reproduction Assessment

Yes, at source level. Multiple same-session gateway or node exec approvals still create one approval-id-scoped follow-up agent turn each, and the issue discussion includes a recent deterministic production reproduction.

## Solution Assessment

Unclear. The current per-approval follow-up design is not enough, but the best fix needs maintainer policy on whether to drop, batch, reorder, direct-deliver, or audit stale completions.
