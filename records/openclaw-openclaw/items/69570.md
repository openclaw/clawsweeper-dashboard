## Summary

Keep this open. Current main still schedules a separate agent follow-up for each resolved exec approval, keyed by approval id, and the existing stop/queue cleanup paths do not cancel already-created approval completion producers before they dispatch later follow-up turns.

## What This Changes

The issue requests cancellation, coalescing, and priority handling for async exec approval completions so multiple approved commands do not trigger a delayed cascade of agent turns.

## Best Possible Solution

Keep this as the canonical completion-cascade bug. The durable end state should add core scheduling semantics that coalesce safe same-session completion follow-ups, let /stop and newer user input preempt stale pending completion turns, preserve per-approval audit/result traceability, and leave direct diagnostics delivery intact.

## Reproduction Assessment

Yes. The source-level reproduction is multiple same-session gateway or node exec calls that each require approval and later complete: current main creates one follow-up agent turn per approval id. The issue discussion also includes a recent deterministic production reproduction on OpenClaw 2026.4.21.

## Solution Assessment

No. The current per-approval follow-up path is not the best solution for the reported UX; a safer fix needs explicit scheduler/circuit-breaker semantics rather than relying on existing queue clearing or loop-detection knobs.
