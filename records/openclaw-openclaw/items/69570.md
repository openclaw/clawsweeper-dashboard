## Summary

Keep open. Current main still has the reported per-approval async follow-up shape, and the remaining fix needs a scheduler policy for coalescing, cancellation, user-input priority, and audit preservation rather than a narrow cleanup close.

## What This Changes

The issue asks OpenClaw to prevent approved exec completion follow-ups from cascading into many delayed agent turns that block later user input.

## Best Possible Solution

Keep this as the canonical approved-completion cascade report and define a core exec approval scheduling rule that can coalesce or cancel stale follow-ups while preserving result traceability and allowing stop or newer user input to preempt safely.

## Reproduction Assessment

Yes, at source level. Multiple same-session gateway or node exec approvals still create one approval-id-scoped follow-up agent turn each, and the issue discussion includes a deterministic production reproduction.

## Solution Assessment

Unclear. The current per-approval follow-up design is not enough, but the best fix requires maintainer policy on whether to pause, batch, drop, direct-deliver, or audit stale completion messages.
