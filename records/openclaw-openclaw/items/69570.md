## Summary

Keep open. Current main still has the reported per-approval async completion shape, and the related open reports do not supersede this issue’s approved-completion cascade; fixing it needs an explicit maintainer policy for exec approval scheduling, cancellation, prioritization, and auditability.

## What This Changes

The issue requests preventing cascades where multiple approved exec commands each enqueue a delayed follow-up agent turn that blocks newer user input.

## Best Possible Solution

Define one core exec approval completion scheduler that can coalesce or cancel stale follow-ups, prioritize explicit stop or newer user input, and still preserve a per-approval audit trail.

## Reproduction Assessment

Yes, at source level. Configure exec approvals and approve multiple foreground commands in one session; current main returns approval-pending immediately and schedules one detached follow-up agent turn per approval id.

## Solution Assessment

Unclear. The requested direction is valid, but maintainers need to choose the safest contract among pausing, batching, dropping stale completions, direct delivery, and cancellation while preserving exec auditability.
