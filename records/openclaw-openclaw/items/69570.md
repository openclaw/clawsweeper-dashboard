Keep this open. Current main still creates approval-id-scoped async exec follow-up agent turns, so the reported cascade is still source-reproducible and the remaining fix needs an explicit scheduler policy for coalescing, cancellation, user-input priority, and audit preservation.

## What This Changes

## Best Possible Solution

Keep this as the canonical approved-completion cascade report and design a core exec-approval scheduling rule that can coalesce or cancel stale follow-ups while preserving result/audit traceability and allowing stop or newer user input to preempt safely.
