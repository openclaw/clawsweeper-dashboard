## Summary

## What This Changes

The PR makes `resolveEffectiveModelFallbacks()` return configured fallbacks for `modelOverrideSource: "user"` and updates one command test to expect retry through a fallback model.

## Best Possible Solution

Preserve strict user-selected model pins by default, then either repair the stall as a clean visible failure or add an explicit maintainer-approved opt-in fallback policy with matching docs, tests, and redacted runtime proof.

Yes for the source-level current behavior: current main returns `[]` for user model overrides and tests assert a single failed attempt. I did not reproduce the live provider-outage deadlock or after-fix recovery path in this read-only review.

## Solution Assessment

No. The PR fixes availability by changing strict pin semantics globally; a safer path is maintainer-approved opt-in fallback routing or a narrower stall/visible-failure repair that keeps existing pins exact by default.
