## Summary

Keep open: the PR addresses a real source-level fallback absence, but it changes shipped strict user-selected model semantics across provider/auth boundaries and is currently conflicting with main, so it needs maintainer policy approval plus stronger after-fix runtime proof before merge.

## What This Changes

The PR makes user-sourced model overrides return configured fallback models from `resolveEffectiveModelFallbacks()` and updates one agent command test to expect fallback recovery after the selected model fails.

## Best Possible Solution

Preserve strict user-selected model pins by default, then either make selected-model failures fail visibly without stalls or add an explicit maintainer-approved fallback policy with matching docs, tests, and real runtime proof.

## Reproduction Assessment

Yes for the source-level behavior: current main resolves user-sourced model overrides to an empty fallback list, and tests assert one failed attempt. I did not reproduce the full live provider-outage deadlock or after-fix recovery path in this read-only review.
