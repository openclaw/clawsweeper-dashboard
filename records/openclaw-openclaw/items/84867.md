## Summary

Keep this PR open, but the current patch should not merge as-is because it changes a documented exact-model routing boundary for user-selected models and still lacks after-fix real behavior proof of the live fallback path.

## What This Changes

This PR makes user session model overrides use the agent-configured fallback list on failure and updates an agent command test to expect a retry on the fallback model.

## Best Possible Solution

Keep user-selected model pins exact by default and fix the outage stall into a visible or cleanly recoverable failure path, or add an explicit opt-in user-pin fallback policy with docs, tests, and redacted live proof.

## Reproduction Assessment

Partial/source: current main clearly returns an empty fallback list for modelOverrideSource "user", and the linked report includes incident logs with fallbackConfigured=false. I do not have a high-confidence reproduced live stall or after-fix failover path from this review.

## Solution Assessment

No: returning agent fallbacks for every user-selected model is not the narrowest maintainable fix because it conflicts with documented exact-pin semantics. The safer path is strict-by-default recovery or an explicit opt-in fallback policy.
