## Summary

Keep open: the outage report is useful, but current main documents and tests user-selected models as exact selections, so this PR changes provider/model routing policy and needs maintainer approval plus after-fix fallback proof before merge.

## What This Changes

This PR changes `src/agents/agent-scope.ts` so `modelOverrideSource: "user"` can use the agent fallback chain instead of disabling configured fallbacks.

## Best Possible Solution

Decide the model-selection policy first: either keep user selections strict and fix the stall so failures surface visibly, or add an explicit opt-in fallback policy with docs, tests, and redacted live proof that fallback works without persisting the selected fallback.

## Reproduction Assessment

Partial: source inspection gives a high-confidence path for the disabled fallback chain because current main returns `[]` for `modelOverrideSource: "user"`. I did not reproduce the full live provider-outage stall in this read-only review.

## Solution Assessment

No: the proposed implementation is not the safest fix because it unconditionally changes a documented exact-selection contract. A safer path is a maintainer-approved policy change with opt-in configuration, or a direct fix that makes strict user selections fail visibly instead of stalling.
