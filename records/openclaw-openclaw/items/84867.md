## Summary

Keep this PR open, but it is not merge-ready as written: it changes the documented exact model-pin contract into fallback-eligible routing, which needs maintainer policy approval, aligned docs/tests, and after-fix failover proof.

## What This Changes

The PR makes user-selected session models use the agent fallback chain on failure and updates one agent command test to expect a retry on the fallback model.

## Best Possible Solution

Keep user-selected model pins exact by default and fix the stall into a visible/recoverable failure path, or add an explicit opt-in fallback policy with docs, tests, and redacted live failover proof.

## Reproduction Assessment

Partial/source: current main clearly disables configured fallbacks for `modelOverrideSource: "user"`, and the PR body includes incident logs showing `fallbackConfigured=false`. I did not reproduce the full live provider-outage stall or the claimed after-fix failover path in a real environment.

## Solution Assessment

No: the patch solves availability by changing a documented exact-selection boundary. The safer solution is a maintainer-approved policy choice, preferably strict-by-default failure recovery or an explicit opt-in fallback mode with aligned docs/tests/proof.
