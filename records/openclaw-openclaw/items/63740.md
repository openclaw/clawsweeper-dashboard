## Summary

Keep open. Current `main` and the latest tagged release still validate packaged `dist` by path membership only and import `run-main` without a byte-integrity preflight, so the reported installed-code corruption is not fixed or safely closeable.

## What This Changes

Investigate and fix a reported installed `dist/run-main-*.js` corruption that makes the OpenClaw CLI fail during ESM compilation.

## Best Possible Solution

Add digest-backed package dist inventory verification before loading installed CLI chunks, then audit update, postinstall, logging, diagnostic, and plugin flows for any write path that could target installed package files.

## Reproduction Assessment

Unclear for the original writer: the report has a concrete corrupted installed-file snippet, but I did not establish a deterministic current-main path where OpenClaw writes shell output into `dist`. Source inspection does show that a byte-modified expected `run-main` chunk is not caught before import.

## Solution Assessment

No. The current path-inventory hardening is useful for missing and stale files, but this report needs byte integrity verification plus a root-cause write audit and a recovery policy.
