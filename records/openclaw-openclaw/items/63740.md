## Summary

Keep open. Current main has useful path-level package-dist inventory and stale-file pruning hardening, but it still does not content-verify expected dist chunks or prove/disprove the reported installed-code mutation, so this remains a security-sensitive integrity investigation.

## What This Changes

The issue reports a globally installed OpenClaw 2026.4.5 CLI whose packaged run-main chunk was modified with a shell `tail` command, causing Node to throw `SyntaxError: Unexpected number` before CLI startup.

## Best Possible Solution

Audit runtime, update, postinstall, logging, and diagnostic paths for writes under the installed package root, then add digest-backed packaged dist verification with a clear startup/update/doctor repair path for modified expected files.

## Reproduction Assessment

Unclear. Source inspection shows a corrupted expected run-main chunk still reaches dynamic import and would fail with the reported startup error, but there is no deterministic current-main path proving OpenClaw itself writes the `tail` command into `dist`.

## Solution Assessment

No. The existing path-inventory hardening is useful but not the complete fix for byte-modified expected chunks; digest verification plus root-cause audit is the safer maintainable direction.
