## Summary

Keep open. Current main has useful package-install hardening for missing or stale dist paths, but it still cannot detect a byte-modified expected dist chunk, and the reported installed-code mutation remains unproven and security-sensitive.

## What This Changes

The issue reports installed CLI JavaScript corruption in a packaged dist chunk causing startup to fail with a SyntaxError.

## Best Possible Solution

Treat this as a package/runtime integrity investigation: audit update, postinstall, restart logging, diagnostics, and bundled runtime-dependency paths for writes under the installed package root, then add content-digest verification for packaged dist chunks with a clear startup or doctor repair path for corrupted expected files.

## Reproduction Assessment

Unclear. The report has a high-signal corrupted installed-file snippet, and manually corrupting the packaged run-main chunk would reproduce the startup import failure, but there is not yet a deterministic reproduction showing OpenClaw itself writing the tail command into dist.

## Solution Assessment

No. The current path-only inventory is useful for missing and stale chunks, but it is not the complete fix for installed-code mutation; hash-based verification plus root-cause audit is the safer maintainable direction.
