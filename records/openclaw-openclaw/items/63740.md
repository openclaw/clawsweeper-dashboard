## Summary

Keep open. Current main and the latest release still use path-only packaged dist verification and import `run-main` without a byte-integrity preflight, so the installed-code corruption report is not fixed or safely closeable.

## What This Changes

The issue reports installed OpenClaw dist source corruption where a `tail -n 200` fragment in `dist/run-main-*` makes the CLI crash during startup.

## Best Possible Solution

Add digest-backed package dist inventory verification before loading installed CLI chunks, then audit update, postinstall, logging, diagnostic, and plugin flows for write paths that could target installed package files.

## Reproduction Assessment

No high-confidence reproduction exists for the original writer that put shell text into `dist`. Source inspection does show that a byte-modified expected `run-main` chunk is not caught by current path-only verification before import.

## Solution Assessment

No. The current path-inventory hardening helps missing and stale files, but this issue needs byte-integrity verification plus a root-cause write audit and recovery policy.
