## Summary

Keep open. Current main and v2026.5.19 still use path-only packaged dist verification and load the CLI run chunk before any byte-integrity preflight, so the reported installed-code corruption is not fixed or safely closeable.

## What This Changes

Investigate and fix an installed OpenClaw CLI dist chunk corruption that made `openclaw doctor --fix` fail at ESM compilation with `SyntaxError: Unexpected number`.

## Best Possible Solution

Add digest-backed package dist inventory verification before loading installed CLI chunks, then audit update, postinstall, logging, diagnostic, and plugin flows for write paths that could target installed package files.

## Reproduction Assessment

No high-confidence reproduction exists for the original writer that put shell text into `dist`. Source inspection does show that a byte-modified expected `run-main` chunk would not be caught by current path-only inventory verification before import.

## Solution Assessment

No. The current path-inventory hardening helps with missing and stale files, but this issue needs byte-integrity verification plus a root-cause write audit and a maintainer-owned recovery policy.
