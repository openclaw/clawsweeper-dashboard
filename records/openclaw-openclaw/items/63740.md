## Summary

Keep open. Current main and the latest release still use a path-only packaged `dist` inventory and import the CLI chunk without a byte-integrity preflight, so the reported installed-code mutation is not resolved.

## What This Changes

The issue reports OpenClaw 2026.4.5 failing at startup because an installed `dist/run-main-*` JavaScript chunk appeared to contain a shell `tail` command fragment.

## Best Possible Solution

Add digest-backed verification for expected packaged dist files before installed CLI chunks are imported, then audit runtime, update, logging, diagnostics, and plugin flows for package-root write paths and define the recovery policy.

## Reproduction Assessment

Unclear for the original writer: the report has a concrete corrupted installed-file snippet, but there is no deterministic current-main command proving OpenClaw writes shell output into `dist`. Source inspection does show current main would not detect changed bytes inside an expected chunk before import.

## Solution Assessment

No. The current path-inventory hardening is useful for missing and stale files, but this issue needs byte integrity verification plus a focused package-root write audit and recovery policy.
