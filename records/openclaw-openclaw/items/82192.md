## Summary

Keep open: the PR targets a real, recently reported completion startup regression and supplies real benchmark proof, but the diff currently removes plugin-provided commands from explicit completion cache writes, so it should be revised before merge.

## What This Changes

The branch makes completion cache writes skip plugin CLI registration, adds the same skip environment to doctor-triggered cache generation, and adds focused tests plus a benchmark case.

## Best Possible Solution

Keep automated doctor/update completion refreshes on the env-gated core-only path, and preserve or redesign explicit cache generation so installed completion still includes plugin-owned command groups without loading unnecessary runtime work.

## Reproduction Assessment

Yes for the PR regression by source inspection: with the new guard, any `completion --write-state` call sets `writeState` and skips plugin command registration before `writeCompletionCache` serializes the command tree. I did not run commands because this review was read-only.

## Solution Assessment

No. The doctor/update fast path is appropriate, but making all write-state invocations core-only is not the best fix because explicit cache rebuilds are the documented path for full plugin-command completions.
