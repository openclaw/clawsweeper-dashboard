## Summary

Keep open: the patch is a focused, source-proven fix for the doctor completion-cache path, and current main still lacks the skip-env propagation. I found no blocking code-review findings, but merge needs maintainer coordination with the overlapping draft PR and normal CI resolution.

## What This Changes

The PR makes doctor-owned completion cache generation pass `OPENCLAW_COMPLETION_SKIP_PLUGIN_COMMANDS=1` to its spawned `completion --write-state` process, preserves explicit completion plugin registration, and adds focused tests plus a startup benchmark case.

## Best Possible Solution

Land one narrow env-guard implementation for doctor/setup-owned cache writes, keep explicit `completion --write-state` as the full plugin-command rebuild, and leave https://github.com/openclaw/openclaw/issues/82070 open for broader cold-start work.

## Reproduction Assessment

Yes by source inspection: current main doctor cache generation spawns `completion --write-state` with the ambient environment, while completion only skips plugin CLI registration under the existing skip env. I did not run the slow path in this read-only review.

## Solution Assessment

Yes for this narrow path: reusing the existing skip-env completion contract in the doctor child process is smaller and safer than changing explicit completion semantics. The remaining coordination is choosing this PR or the overlapping draft branch.
