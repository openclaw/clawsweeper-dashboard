## Summary

Keep open: the current head appears to have addressed the earlier plugin-completion regression, but the external PR merge gate still needs current-head real behavior proof because the copied benchmark output no longer exercises the env-gated doctor/update cache path.

## What This Changes

This PR sets doctor-triggered completion-cache generation to pass the plugin-command skip environment, preserves explicit completion plugin registration, and adds focused tests plus a startup benchmark case.

## Best Possible Solution

Land the narrow doctor completion-cache env fix after the contributor updates the PR with redacted current-head real behavior proof, and keep the broader cold-start investigation tracked under https://github.com/openclaw/openclaw/issues/82070.

## Reproduction Assessment

Yes for the narrow doctor completion-cache bug by source inspection: current main spawns `completion --write-state` without the skip env, and completion eagerly registers plugin commands unless that env is set. I did not run a live packaged reproduction in this read-only review.

## Solution Assessment

Yes for the narrow doctor/update completion-cache slice: reusing the existing skip env in the doctor child process is the smallest maintainable fix, while explicit completion writes still preserve plugin commands. It is not a complete solution for the broader cold-start issue linked from the PR.
