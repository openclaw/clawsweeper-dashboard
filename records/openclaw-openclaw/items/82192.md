## What This Changes

The PR adds a completion write-state benchmark, sets doctor-spawned completion cache writes to pass the existing plugin-skip environment flag, adds completion/doctor tests, and also includes unrelated channel-add and cron persistence edits.

## Best Possible Solution

Land a focused completion-only patch that mirrors the update-owned skip-env path for doctor cache writes, preserves explicit plugin completion rebuilds, and leaves https://github.com/openclaw/openclaw/issues/82070 open for the broader cold-start investigation.

Yes, by source inspection: current main's doctor cache generation uses the ambient environment while completion and update already define the skip-env contract. ## Solution Assessment
