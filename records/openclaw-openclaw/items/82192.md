## Summary

Keep open: the completion-cache fix itself is narrow and well supported, but the current branch also relaxes unrelated cron persisted-job validation in a way that can resurrect malformed blank-payload jobs.

## What This Changes

The PR makes doctor-triggered completion cache generation pass the existing plugin-command skip environment variable, adds completion/doctor regression tests, adds a startup benchmark case, and includes unrelated cron/pairing stabilization edits.

## Best Possible Solution

Land a completion-only change that applies the existing skip-env contract to doctor-owned cache writes, keeps explicit completion rebuilds full-fidelity, and leaves broader CLI cold-start work in https://github.com/openclaw/openclaw/issues/82070.

## Reproduction Assessment

Yes by source inspection and contributor proof: current main doctor cache generation uses ambient environment, while completion already has an env-gated plugin-skip path. I did not run the slow benchmark locally in this read-only review.

## Solution Assessment

Yes for the narrow completion fix, but no for the current branch as a whole because the unrelated cron persisted-boundary relaxation should be removed before merge.
