## What This Changes

The PR makes doctor-owned completion cache generation spawn `completion --write-state` with `OPENCLAW_COMPLETION_SKIP_PLUGIN_COMMANDS=1`, preserves explicit completion behavior, and adds focused completion/doctor tests plus a benchmark case.

## Best Possible Solution

Land one narrow implementation that applies the existing skip-env completion contract to doctor/setup-owned cache writes while preserving explicit full plugin-command rebuilds and keeping https://github.com/openclaw/openclaw/issues/82070 open for broader cold-start work.

## Reproduction Assessment

Yes by source inspection: current main doctor cache generation spawns `completion --write-state` with the ambient environment, while completion only skips plugin command registration under the existing skip env. I did not execute the slow path in this read-only review.

## Solution Assessment

Yes for the narrow doctor/update completion-cache boundary: reusing the existing env-gated completion fast path is smaller and safer than changing explicit completion semantics. The only coordination issue is choosing this PR or the overlapping draft branch.
