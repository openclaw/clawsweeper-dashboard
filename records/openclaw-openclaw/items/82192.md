## Summary

Keep open: the PR targets a real startup pain point, but the current patch would remove plugin-owned command completions from explicit cache rebuilds; current main and docs keep only update/doctor refreshes on the skip-env path.

## What This Changes

The branch skips plugin CLI registration for completion cache writes, adds the same skip env to doctor-triggered cache generation, updates focused tests, and adds a startup benchmark case.

## Best Possible Solution

Keep automated doctor/update completion refreshes on the env-gated core-only path while preserving explicit completion --write-state as the full plugin-command rebuild, with regression coverage for both paths.

## Reproduction Assessment

Yes for the PR regression by source inspection: with the new guard, any completion --write-state call skips plugin registration before writing the cache. I did not run commands because this review was read-only.

## Solution Assessment

No. The doctor/update fast path is appropriate, but making all write-state invocations core-only is not the narrowest maintainable fix because explicit cache rebuilds are documented as full plugin-command rebuilds.
