## Summary

## What This Changes

This PR updates `resolveModelAuthLabel` and agent auth tests so literal per-provider `apiKey` labels can appear as `api-key (models.json)` for custom providers such as `zai-light`.

## Best Possible Solution

Keep `resolveModelAuthLabel` on the same precedence as `resolveApiKeyForProvider`: profile refs before env, env before late literal config unless runtime also treats the explicit config key as terminal.

## Reproduction Assessment

Yes for the PR-introduced mismatch by source inspection: the patched helper returns `models.json` for literal per-entry keys before env lookup, while runtime can still reach env before the late custom-key fallback.

## Solution Assessment

No, not yet; the proposed fix is close for the `zai-light` happy path, but the literal shortcut is broader than the runtime contract and should be narrowed or removed.
