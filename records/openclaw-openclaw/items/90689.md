The PR updates `resolveModelAuthLabel` to label literal per-provider `apiKey` entries as `api-key (models.json)` before env fallback and adds focused auth-label regression tests.

Keep one credential-source precedence invariant across `/status`, `/models`, and `/model status`, with regression coverage for `zai-light` plus `ZAI_API_KEY` and env-before-late-custom behavior when no literal per-entry binding exists.
