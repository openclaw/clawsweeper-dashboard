## Summary

Keep open. Current main still preserves and reuses stale subagent lineage for ordinary sessions, and the linked implementation PR is open and unmerged, so this should stay tracked until that PR or an equivalent fix lands.

## What This Changes

Normal top-level sessions should not preserve stale subagent lineage that makes `sessions_spawn` enforce depth limits as if the session were nested.

## Best Possible Solution

Keep this issue open until https://github.com/openclaw/openclaw/pull/67946 or an equivalent narrow fix lands with regression coverage for stale top-level lineage.

## Reproduction Assessment

Yes, at source level. Seed a normal top-level session entry with stale `spawnedBy` or `spawnDepth`, run another ordinary top-level turn, then `sessions_spawn` can compute depth from the stored lineage under a restrictive `maxSpawnDepth`.

## Solution Assessment

Yes. Clearing lineage only at ordinary top-level session boundaries while preserving real spawned/ACP metadata is the narrow maintainable fix, and the linked PR is already the active implementation path.
