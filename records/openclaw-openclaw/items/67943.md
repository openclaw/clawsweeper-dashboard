## Summary

Keep open. Current main still has the stale-lineage source path described by the report, and https://github.com/openclaw/openclaw/pull/67946 is still the open, unmerged implementation path with closing syntax for this issue.

## What This Changes

The issue asks OpenClaw to stop ordinary top-level sessions from inheriting persisted subagent lineage that can make `sessions_spawn` hit depth limits incorrectly.

## Best Possible Solution

Keep this issue open until https://github.com/openclaw/openclaw/pull/67946 or an equivalent fix lands with regression coverage for stale top-level lineage.

## Reproduction Assessment

Yes, at source level. Seed a normal top-level session entry with stale `spawnedBy` or `spawnDepth`, run another ordinary top-level turn, then `sessions_spawn` can compute depth from the stored lineage under a restrictive `maxSpawnDepth`.

## Solution Assessment

Yes. Clearing lineage only at ordinary top-level session boundaries while preserving real subagent and ACP session lineage is the narrow maintainable fix, and the linked PR is already the active implementation path.
