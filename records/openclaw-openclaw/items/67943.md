## Summary

Keep open. Current main still preserves and reuses persisted subagent lineage in the normal agent-run path, and the linked same-author implementation PR is open and unmerged, so this issue should remain the tracking item until that PR or an equivalent fix lands.

## What This Changes

Prevent ordinary top-level sessions from inheriting stale subagent lineage metadata so `sessions_spawn` depth checks treat normal chats as depth 0.

## Best Possible Solution

Keep this issue open until #67946 or an equivalent maintainer-approved patch lands, with code that clears stale lineage on ordinary top-level turns, preserves lineage for real spawned/ACP sessions, avoids stale `sessionEntry.spawnedBy` as normal-run context, and adds focused regression coverage.

## Reproduction Assessment

Yes. The current-main static path is high confidence: a normal session entry with stale `spawnDepth` or `spawnedBy` can be merged forward, reused by `agent-command`, and then interpreted by `sessions_spawn` as nested depth. I did not run tests because this review was required to keep the checkout read-only.

## Solution Assessment

Yes. The narrow maintainable fix is to make lineage explicit: keep it only for actual spawned sessions, clear it from ordinary top-level turns, and cover the stale-store regression. The existing linked PR is the right path to review first.
