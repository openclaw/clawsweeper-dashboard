## Summary

Keep open. Current main still has a credible stale-lineage path: ordinary agent runs can fall back to persisted `spawnedBy`, session merging preserves omitted lineage fields, and depth calculation trusts stored `spawnDepth`/`spawnedBy`. The linked same-author implementation PR remains open and unmerged, so this issue should stay open until that fix or an equivalent maintainer-approved change lands.

## What This Changes

Clear stale subagent lineage from normal top-level session turns so `sessions_spawn` depth checks only use live spawned-session context.

## Best Possible Solution

Land #67946 or an equivalent narrow fix that clears subagent lineage on ordinary top-level turns, preserves it for real spawned/ACP sessions, stops normal runs from using stale stored lineage as spawn context, and adds a focused regression test.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to seed a normal top-level session entry with stale `spawnedBy` or `spawnDepth`, run another ordinary top-level turn, then call `sessions_spawn` with a restrictive `maxSpawnDepth`; current main has the code path needed for that stale metadata to affect depth checks.

## Solution Assessment

Yes. The proposed fix direction is the narrow maintainable path: sanitize lineage at the normal top-level session boundary while leaving actual spawned session metadata intact.
