## Summary

Keep open. Current main still has the stale-lineage source path described in the report, and the same-author implementation PR #67946 remains the active open fix path in the provided GitHub context, so this issue should not be closed before that PR is reviewed, merged, or replaced.

## What This Changes

Normal top-level sessions should clear stale subagent lineage metadata so later ordinary runs do not incorrectly hit `sessions_spawn` depth limits.

## Best Possible Solution

Keep this issue open while #67946 remains open. The desired end state is a landed fix that clears persisted subagent lineage on ordinary top-level turns, preserves lineage for real spawned and ACP sessions, stops normal runs from using stale stored `spawnedBy`, and adds regression coverage for the stale-lineage depth-limit case.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction path exists: persist stale `spawnedBy` or `spawnDepth` on a normal session entry, run an ordinary top-level turn, then invoke `sessions_spawn` and observe the depth gate deriving depth from that stored lineage.

## Solution Assessment

Yes. The narrow maintainable direction is to clear lineage on ordinary top-level session turns and preserve it only for real spawned/ACP sessions; maintainers should review #67946 or land an equivalent fix rather than close the issue early.
