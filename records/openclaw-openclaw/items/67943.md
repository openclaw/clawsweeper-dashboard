## Summary

Keep open. Current main still preserves and reuses persisted subagent lineage on ordinary session runs, and the linked implementation PR at https://github.com/openclaw/openclaw/pull/67946 remains open with closing syntax for this issue.

## What This Changes

Clear stale subagent lineage fields on ordinary top-level session turns so normal sessions do not inherit spawned-session depth metadata.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/67946 or an equivalent focused fix with regression coverage for stale top-level lineage.

## Reproduction Assessment

Yes, at source level. Seed an ordinary top-level session entry with stale `spawnedBy` or `spawnDepth`, then a later normal run can feed stored lineage into `sessions_spawn` depth resolution under a restrictive `maxSpawnDepth`.

## Solution Assessment

Yes. Gating persisted lineage to real subagent/ACP session keys and refusing stored `sessionEntry.spawnedBy` for ordinary top-level runs is the narrow maintainable fix direction.
