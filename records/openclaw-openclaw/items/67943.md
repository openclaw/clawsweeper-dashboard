## Summary

Keep open. Current main still has the stale-lineage source path, and the linked same-author implementation PR remains open and unmerged with closing syntax for this issue.

## What This Changes

The issue requests clearing stale subagent lineage from ordinary top-level sessions so `sessions_spawn` depth checks only see real spawned-session context.

## Best Possible Solution

Keep this issue open until https://github.com/openclaw/openclaw/pull/67946 or an equivalent fix lands with regression coverage for stale top-level lineage.

## Reproduction Assessment

Yes, at source level. Seed a normal top-level session entry with stale `spawnedBy` or `spawnDepth`, run another ordinary top-level turn, then `sessions_spawn` can compute depth from stored lineage under a restrictive `maxSpawnDepth`.

## Solution Assessment

Yes. Gating persisted lineage to real subagent/ACP session keys and refusing stored `spawnedBy` for ordinary top-level runs is the narrow maintainable fix, and the linked PR is already the active implementation path.
