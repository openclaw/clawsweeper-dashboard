## Summary

Keep open. Current main still has the stale-lineage source path, and the same-author implementation PR that explicitly closes this issue remains open and unmerged.

## What This Changes

The issue reports that ordinary top-level sessions can retain stale subagent lineage metadata and then fail `sessions_spawn` depth checks as if they were nested.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/67946 or an equivalent focused fix, then close this issue once main clears stale top-level lineage with regression coverage.

## Reproduction Assessment

Yes, at source level. Seed a normal top-level session entry with stale `spawnedBy` or `spawnDepth`, then current main can feed that stored lineage into `getSubagentDepthFromSessionStore` before `sessions_spawn` enforces `maxSpawnDepth`.

## Solution Assessment

Yes. Gating persisted lineage to real spawned/ACP sessions and not using stale `sessionEntry.spawnedBy` for ordinary top-level runs is the narrow maintainable fix direction.
