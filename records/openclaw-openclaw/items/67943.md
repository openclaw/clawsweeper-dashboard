## Summary

Keep open. Current main still preserves and reuses stale subagent lineage for ordinary session turns, and #67946 is already the open closing PR for this bug.

## What This Changes

Normal top-level session turns should clear stale subagent lineage so later `sessions_spawn` depth checks treat them as top-level sessions.

## Best Possible Solution

Land #67946 or an equivalent narrow fix that clears lineage on ordinary top-level turns, preserves real spawned/ACP lineage, and adds stale-lineage regression coverage.

## Reproduction Assessment

Yes, at source level. Seed a normal session entry with stale `spawnedBy` or `spawnDepth`, run an ordinary top-level turn, and then call `sessions_spawn` with a restrictive `maxSpawnDepth`; current main still contains that path.

## Solution Assessment

Yes. Sanitizing lineage at the ordinary top-level session boundary while preserving actual spawned/ACP metadata is the narrow maintainable fix.
