## Summary

Keep open. Current main still has the stale-lineage source path, and the linked same-author implementation PR is open and unmerged, so the issue should remain tracked until that PR or an equivalent fix lands.

## What This Changes

Normal top-level sessions should stop preserving or reusing stale subagent lineage metadata that can make later `sessions_spawn` calls fail depth checks incorrectly.

## Best Possible Solution

Land #67946 or an equivalent narrow fix that clears lineage only for ordinary top-level runs, preserves real spawned/ACP lineage, and adds stale-lineage regression coverage.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to seed a normal top-level session entry with stale `spawnedBy` or `spawnDepth`, run another ordinary top-level turn, and then call `sessions_spawn` under a restrictive `maxSpawnDepth`; current main still contains that path.

## Solution Assessment

Yes. Sanitizing lineage at the top-level session boundary while preserving actual spawned-session metadata is the narrow maintainable fix, and the linked PR is already the active implementation path.
