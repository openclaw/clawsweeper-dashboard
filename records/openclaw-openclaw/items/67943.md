## Summary

Keep open. Current main still contains the stale-lineage source path, and the linked same-author implementation PR #67946 is open and unmerged with closing syntax, so this issue should remain tracked until that PR or an equivalent fix lands.

## What This Changes

Clear stale subagent lineage metadata from ordinary top-level session turns so `sessions_spawn` depth checks treat them as depth 0.

## Best Possible Solution

Land PR #67946 or an equivalent narrow fix that clears lineage only for ordinary top-level runs, preserves real spawned/ACP lineage, and adds stale-lineage regression coverage.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to seed a normal top-level session entry with stale `spawnedBy` or `spawnDepth`, run another ordinary top-level turn that preserves it, then call `sessions_spawn` under a restrictive `maxSpawnDepth`.

## Solution Assessment

Yes. Sanitizing lineage at the top-level session boundary while preserving actual spawned-session metadata is the narrow maintainable fix, and the linked PR is already the active implementation path.
