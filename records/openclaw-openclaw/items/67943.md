- `impact:session-state`: The issue concerns stale persisted session lineage metadata causing ordinary sessions to be treated as nested subagent sessions.

## Summary

Keep open. Current main still preserves and reuses persisted subagent lineage for ordinary session runs, and the linked implementation PR remains open and unmerged with closing syntax for this issue.

## What This Changes

Clear stale subagent lineage from ordinary top-level sessions so later normal runs do not hit nested `sessions_spawn` depth limits.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/67946 or an equivalent focused fix that gates persisted lineage to real subagent/ACP sessions and adds regression coverage.

## Reproduction Assessment

Yes, at source level. Seed an ordinary top-level session entry with stale `spawnedBy` or `spawnDepth`; current main can preserve/reuse that lineage, and `sessions_spawn` depth enforcement then reads the stale stored depth under restrictive `maxSpawnDepth`.

## Solution Assessment

Yes. The narrow maintainable fix is to clear subagent lineage on ordinary top-level turns and only reuse stored lineage for real subagent/ACP session keys, which is the shape of the linked PR.
