## Summary

Keep open: current main still has the documented SecretRef docs gap, and an open linked PR already directly addresses it with closing syntax. The issue should stay open until that PR merges or maintainers choose an equivalent docs fix.

## What This Changes

Document that `allowInsecurePath` also bypasses POSIX uid ownership checks for SecretRef provider paths, including trusted root-owned system-package binaries.

## Best Possible Solution

Review and land `https://github.com/openclaw/openclaw/pull/79273`, or an equivalent docs-only update, then close this issue as implemented after merge.

## Reproduction Assessment

Yes. Source inspection on current main shows `allowInsecurePath` bypasses the POSIX uid check, and the current docs only describe the Windows ACL fallback; no live `/usr/bin/op` setup is needed to confirm the docs gap.

## Solution Assessment

Yes. A focused docs update is the narrowest maintainable solution for this issue; splitting `allowInsecurePath` into new config knobs would be a separate product/API decision.
