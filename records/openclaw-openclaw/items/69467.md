## Summary

Keep open. Current main and the latest release still compose the reported destructive path: cron can persist a live DM session target, one-shot jobs can default cleanup on, runtime reuses that target, and direct delivery cleanup deletes the supplied session key; the related fixes are still open rather than merged.

## What This Changes

The issue reports that one-shot agent-turn cron reminders created from a live non-default DM can normalize to that DM session and delete or rebind it through deleteAfterRun cleanup, causing context loss.

## Best Possible Solution

Land or replace the open cron fixes with one narrow implementation that keeps live direct-message session keys out of reusable cleanup targets, deletes only cron-owned sessions, and adds regression coverage across normalization, runtime reuse, and direct delivery cleanup.

## Reproduction Assessment

Yes, as a source-reproducible path. The issue provides a detailed sanitized incident timeline, and current main still persists current/session targets, defaults one-shot cleanup on, reuses explicit session targets, and deletes the supplied key during direct cleanup.

## Solution Assessment

Yes. Keeping this open for a coordinated cron-session ownership fix is the narrow maintainable path; current main has not implemented the needed guard or direct-session fallback yet.
