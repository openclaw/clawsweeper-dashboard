## What This Changes

The PR changes Ollama model compatibility metadata, native chat request parameters, local-provider idle timeout resolution, and agent-default model config schema support.

## Best Possible Solution

Keep the useful Ollama compatibility fixes, but preserve documented timeout and thinking override semantics and either use the existing provider-model `compat` path or wire the new agent-default compat surface end to end.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve configured local model idle timeouts:** `src/agents/pi-embedded-runner/run/llm-idle-timeout.ts:102-111`
  - body: Returning `0` before checking `modelRequestTimeoutMs` makes `models.providers.<id>.timeoutSeconds` stop wrapping local Ollama streams with the configured idle watchdog, even though current docs and tests say that knob applies to slow local/self-hosted providers. Keep the local default exemption below explicit model/agent timeouts or update the contract deliberately.
  - confidence: 0.9
- **[P2] Honor explicit Ollama thinking params:** `extensions/ollama/src/stream.ts:399-404`
  - body: This branch forces `think: false` whenever `model.reasoning` is not exactly true, so explicit `params.think` or `params.thinking` on a configured native Ollama model can no longer force thinking as documented, and non-off runtime levels on models without discovered metadata are silently downgraded. Preserve explicit model params or reject them during config/model validation.
  - confidence: 0.84
- **[P2] Wire agent model compat overrides into resolution:** `src/config/zod-schema.agent-defaults.ts:74-76`
  - body: Adding `compat` to `agents.defaults.models` validation makes that config pass, but runtime resolution currently only reads `.params` from `agents.defaults.models` and gets compat from `models.providers.*.models[]`. As written, the new override is silently ignored; either propagate it into the resolved model compat or keep users on the existing provider-model compat path.
  - confidence: 0.87
