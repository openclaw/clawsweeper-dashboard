## Summary

Keep open: this is a plausible Ollama bug-fix PR, but the patch still introduces contract regressions, validates a config field that runtime ignores, changes native Ollama request defaults globally, and lacks inspectable after-fix real behavior proof.

## What This Changes

The PR changes Ollama model compatibility metadata, native chat request shaping, local-provider idle-timeout resolution/tests, and agent-default model config typing/schema.

## Best Possible Solution

Keep the useful Ollama compatibility intent, but preserve documented timeout and per-model thinking/keep_alive semantics, wire or remove the new compat config surface, and coordinate the thinking fix with https://github.com/openclaw/openclaw/pull/82445.

## Reproduction Assessment

Partially: source inspection confirms the touched code paths and shows existing timeout/request-shape contracts that this patch would break. The reported live Ollama qwen2.5/tool and idle-timeout failures still lack inspectable real-environment proof.

## Solution Assessment

No; the patch mixes useful fixes with contract changes and a no-op config surface. The safer solution is a narrower Ollama fix that preserves explicit user config and existing provider-model compat paths.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve configured local idle timeouts:** `src/agents/pi-embedded-runner/run/llm-idle-timeout.ts:102-111`
  - body: Moving the local-provider return ahead of configured timeout handling makes models.providers.<id>.timeoutSeconds and agents.defaults.timeoutSeconds stop controlling local Ollama streams, contrary to current docs and tests that say explicit local/self-hosted timeouts still apply.
  - confidence: 0.91
- **[P2] Honor explicit Ollama thinking params:** `extensions/ollama/src/stream.ts:399-404`
  - body: This branch forces think:false whenever model.reasoning is not exactly true, so a configured native Ollama model can no longer use params.think or params.thinking to force thinking as documented. Preserve explicit model params or reject them during validation instead of silently overriding them.
  - confidence: 0.86
- **[P2] Wire compat overrides into model resolution:** `src/config/zod-schema.agent-defaults.ts:74-76`
  - body: Adding compat to agents.defaults.models validation makes that config parse, but runtime model resolution only reads entry.params from that map and still takes compat from the provider/discovered model. As written, users can configure compat there and OpenClaw will ignore it.
  - confidence: 0.88
- **[P2] Keep Ollama keep_alive opt-in:** `extensions/ollama/src/stream.ts:444`
  - body: Adding keep_alive:"1h" to every native chat request changes the current request contract and breaks the existing exact request-shape test that omits keep_alive when no model param is configured. Keep this per-model via params.keep_alive unless maintainers explicitly change the documented default.
  - confidence: 0.83
