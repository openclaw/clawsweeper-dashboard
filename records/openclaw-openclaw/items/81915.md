## Best Possible Solution

Preserve the current timeout and explicit request-param contracts, take a narrow Ollama compatibility fix for tool/thinking behavior, keep keep_alive opt-in via model params unless maintainers explicitly change the default, and coordinate the non-reasoning thinking work with https://github.com/openclaw/openclaw/pull/82445.

Partially: source inspection gives a high-confidence path for the regressions introduced by this patch against current docs and tests. The reported live Ollama qwen2.5/tool and idle-timeout failures still lack inspectable real-environment proof.

## Solution Assessment

  - confidence: 0.91

  - confidence: 0.86

  - body: Adding compat to agents.defaults.models validation makes that config parse, but runtime resolution still reads only entry.params from that map and takes compat from provider/discovered model metadata. As written, users can configure compat there and OpenClaw will ignore it.
  - confidence: 0.88
- **[P2] Keep Ollama keep_alive opt-in:** `extensions/ollama/src/stream.ts:444`
  - body: Adding keep_alive:"1h" to every native chat request changes the current request contract and breaks the existing exact request-shape expectation that omits keep_alive when no model param is configured. Keep this per-model via params.keep_alive unless maintainers explicitly change the default.
  - confidence: 0.83
