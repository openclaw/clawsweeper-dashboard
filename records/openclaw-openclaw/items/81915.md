  - confidence: 0.91

  - body: This branch forces `think: false` whenever `model.reasoning` is not exactly true, so a configured native Ollama model with explicit `params.think` or `params.thinking` can no longer force thinking when the active run is otherwise implicit `off`. Preserve explicit model params or reject them during validation instead of silently overriding them.
  - confidence: 0.86

  - body: Adding `compat` to `agents.defaults.models` validation makes that config pass, but runtime model resolution only reads `.params` from those entries and takes compat from provider model metadata. As written, the new override is silently ignored, so either propagate it into the resolved model or keep users on the existing provider-model compat path.
  - confidence: 0.88
