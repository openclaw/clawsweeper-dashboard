## Summary

Keep open: the PR is a plausible Ollama bug-fix bundle, but the current diff still regresses documented timeout/thinking semantics, adds a config schema path that runtime ignores, forces a new keep_alive default, and lacks inspectable real Ollama proof.

## What This Changes

The branch changes Ollama model compatibility metadata, native `/api/chat` request parameters, local-provider idle-timeout resolution, agent-default model config schema/types, and related idle-timeout tests.

## Best Possible Solution

Keep the useful Ollama capability fixes, but preserve explicit timeout, thinking, and keep_alive semantics, and either wire the new compat override end to end or keep users on the existing provider-model compat path.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Partially: source inspection confirms the touched code paths and current contracts, but the reported live Ollama failures are not reproducible from the provided PR evidence alone.

## Solution Assessment

Is this the best way to solve the issue? No; the patch should be narrowed so explicit local timeouts, explicit Ollama thinking params, and Ollama keep_alive defaults keep their documented behavior, and any new compat config surface is either propagated or removed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Preserve configured local model idle timeouts:** `src/agents/pi-embedded-runner/run/llm-idle-timeout.ts:102-111`
  - body: Returning `0` before checking `modelRequestTimeoutMs` makes configured provider and agent timeouts stop wrapping local Ollama streams, even though current docs/tests say those knobs apply to slow local/self-hosted providers. Keep the local default exemption below explicit timeout handling or change the contract deliberately.
  - confidence: 0.9
- **[P2] Honor explicit Ollama thinking params:** `extensions/ollama/src/stream.ts:399-404`
  - body: This branch forces `think: false` whenever `model.reasoning` is not exactly true, so explicit `params.think` or `params.thinking` on a configured native Ollama model can no longer force thinking as documented. Preserve explicit model params or reject them during config/model validation.
  - confidence: 0.84
- **[P2] Wire agent model compat overrides into resolution:** `src/config/zod-schema.agent-defaults.ts:74-76`
  - body: Adding `compat` to `agents.defaults.models` validation makes that config pass, but runtime resolution only reads `.params` from that surface and gets compat from provider model metadata. Either propagate this override into the resolved model or keep users on the existing provider-model compat path.
  - confidence: 0.87
- **[P2] Preserve Ollama keep_alive defaults unless configured:** `extensions/ollama/src/stream.ts:444`
  - body: Adding a default `keep_alive: "1h"` sends a per-request retention override on every native Ollama chat call, bypassing Ollama's server default and OpenClaw's current opt-in `params.keep_alive` behavior. Leave it unset by default unless maintainers intentionally change that resource policy.
  - confidence: 0.78
