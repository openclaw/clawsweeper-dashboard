Full review comments:

- **[P2] Preserve configured local idle timeouts:** `src/agents/pi-embedded-runner/run/llm-idle-timeout.ts:102-111`
  - body: Moving the local-provider return ahead of configured timeout handling makes `models.providers.<id>.timeoutSeconds` and `agents.defaults.timeoutSeconds` stop controlling local Ollama streams, contrary to current docs and tests that say explicit local/self-hosted timeouts still apply.
  - confidence: 0.91
- **[P2] Do not treat unknown thinking support as false:** `extensions/ollama/src/stream.ts:399-404`
  - body: This branch forces `think:false` unless `model.reasoning` is exactly true, so configured native Ollama models without a resolved reasoning flag can no longer use explicit `params.think` or `params.thinking`. Current main fixes the 400 path more narrowly by suppressing truthy `think` only when the model is known as `reasoning:false`.
  - confidence: 0.88
- **[P2] Wire compat overrides into model resolution:** `src/config/zod-schema.agent-defaults.ts:74-76`
  - body: Adding `compat` to `agents.defaults.models` validation makes that config parse, but runtime model resolution still reads only `entry.params` from that map and takes `compat` from provider/discovered model metadata. Users could configure this field and have OpenClaw ignore it.
  - confidence: 0.88
- **[P2] Keep Ollama keep_alive opt-in:** `extensions/ollama/src/stream.ts:444`
  - body: Adding `keep_alive:"1h"` to every native chat request changes the current request contract and bypasses Ollama's default unless users override it. Keep this per model via `params.keep_alive` unless maintainers intentionally change the documented default.
  - confidence: 0.84
