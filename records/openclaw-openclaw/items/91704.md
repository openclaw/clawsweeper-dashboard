## Summary

Keep this open: current main does load workspace bootstrap files into the run prompt, but the reported slow-backend first-turn failure is not solved because local-model lean can still expose Tool Search controls and compact discovery before the model uses `TOOLS.md` guidance. The related open PRs address adjacent lean-mode fanout/config work, not this issue’s ordering behavior, and at least one still needs maintainer/product review.

## What This Changes

The issue asks OpenClaw to prioritize first-turn workspace guidance from `TOOLS.md` and `SOUL.md` before expensive Tool Search discovery on slow local inference backends.

## Best Possible Solution

Keep bootstrap injection as the canonical context path, then make a maintainer-approved Tool Search/local-model policy change that lets first-turn endpoint-style prompts use workspace guidance before expensive discovery without treating `TOOLS.md` as a source of tool availability.

## Reproduction Assessment

No high-confidence live repro yet. Source inspection gives a medium-confidence path: current main injects workspace bootstrap files, but local-model lean can still expose Tool Search controls and compact discovery on the first turn; the NemoClaw/vLLM timeout was not reproduced here.

## Solution Assessment

Unclear. Prioritizing workspace guidance is plausible, but the best fix needs maintainer choice between prompt ordering, Tool Search defaults, directory-mode behavior, or local-model policy instead of an unconditional runtime shim.
