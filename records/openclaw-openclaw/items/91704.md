## Summary

Current main disproves the report’s literal ordering theory because workspace bootstrap files are loaded into the system prompt before inference, but it still intentionally exposes Tool Search controls during that turn and lean local-model mode defaults to them. The timeout therefore remains plausible as a model/tool-planning problem and needs a same-stack current-main reproduction before maintainers change local-model policy.

## What This Changes

Investigate first-turn natural-language timeouts where a slow local model invokes Tool Search instead of using endpoint guidance supplied through `TOOLS.md`.

## Best Possible Solution

Reproduce on current main with the same provider stack and redacted model-payload and first-event timing, then make the narrowest policy change at the Tool Search/local-model boundary while keeping bootstrap files injected once.

## Reproduction Assessment

No, we do not yet have a high-confidence current-main reproduction. Source inspection confirms that bootstrap guidance and Tool Search controls coexist in the first model turn, but only the reporter’s external stack can establish the claimed 600-second planning timeout.

## Solution Assessment

No, the suggested separate `TOOLS.md` and `SOUL.md` read phase is not the best fix because those files are already injected before inference. A better fix must be based on live evidence and preserve the reason lean mode compacts large tool catalogs.
