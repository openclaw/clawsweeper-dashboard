## Summary

Current main already loads workspace bootstrap files into agent context, but the reported first-turn timeout remains a plausible core runtime issue because Tool Search/local-model lean can still expose discovery controls before the model uses `TOOLS.md` guidance on slow backends. I would keep this open for maintainer review rather than close it as implemented.

## What This Changes

The issue requests that workspace guidance files such as `TOOLS.md` and `SOUL.md` be consulted before dynamic tool discovery so natural-language endpoint queries do not time out on slow local inference backends.

## Best Possible Solution

Keep bootstrap injection as the canonical context path, then add a narrow tested Tool Search/local-model policy so first-turn natural-language endpoint queries can use workspace guidance before expensive catalog discovery without making `TOOLS.md` a source of tool availability.

## Reproduction Assessment

Yes, source inspection gives a medium-confidence path: workspace files are loaded, but Tool Search/local-model lean can still present discovery controls on first turn before the model uses `TOOLS.md`. I did not live-reproduce the NemoClaw/vLLM timeout environment.

## Solution Assessment

Unclear: prioritizing workspace guidance is reasonable, but the best fix needs maintainer choice between prompt ordering, Tool Search defaults, or an explicit local-model policy rather than an unconditional runtime shim.
