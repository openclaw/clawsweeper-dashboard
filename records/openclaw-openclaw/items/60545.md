## Summary

Keep open: current main only covers part of the intended runtime fixes, and the proposed branch is not mergeable as-is because it introduces typecheck blockers, incomplete session-continuity cleanup, and an unsafe sandbox browser-bridge teardown path.

## What This Changes

The PR changes agent fallback retry prompting, explicit session-id rollover persistence, `openclaw agent --json` output routing, sandbox browser-bridge disposal, and related tests.

## Best Possible Solution

Split the useful remaining work into current-main-native fixes: keep the shipped retry and JSON implementations, add a focused explicit-session rollover repair that clears all stale session-scoped continuity fields through existing helpers, and only add sandbox browser-bridge teardown with clear one-shot/private-scope ownership or refcounting.

## Reproduction Assessment

Yes for the PR blockers: static source review is enough to reproduce the wrong module path and required-`scopeKey` type failures, and the current `resolveSession` branch shows how explicit session ids can still reuse an old session entry. Runtime tests were not run because this was a read-only review.

## Solution Assessment

No. The branch is not the best way to solve the issue now because current main has newer implementations for two of the four behaviors, while the remaining session and sandbox changes need narrower, current-code fixes.

  - body: The new type annotation imports `./sandbox/context.js` from inside `src/agents/sandbox`, which resolves to the non-existent `src/agents/sandbox/sandbox/context.js`. The runtime import below uses `./context.js`, so this test file will fail TypeScript module resolution.

  - confidence: 0.9
- **[P2] Preserve all CLI continuity fields on rollover:** `src/agents/command/attempt-execution.ts:101-106`
  - body: The rollover clear-list deletes `cliSessionIds` but leaves `cliSessionBindings` and legacy `claudeCliSessionId`. Since CLI session lookup reads those fields too, an explicit fresh OpenClaw session can still resume the old Claude/Codex CLI session.
  - confidence: 0.87
- **[P1] Avoid disposing shared browser bridges per attempt:** `src/agents/pi-embedded-runner/run/attempt.ts:1976`
  - body: This finally block disposes the browser bridge for the whole sandbox scope after every embedded attempt. Agent/shared scopes can be used by overlapping runs, so one completed run can remove the bridge another run is still using; teardown needs private-scope ownership or refcounting.
  - confidence: 0.86
