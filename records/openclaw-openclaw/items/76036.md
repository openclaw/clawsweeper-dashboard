## What This Changes

The branch makes message_received hook content skip blank command bodies before falling back to raw or generic inbound body, with Slack regression tests, docs, and changelog coverage.

## Best Possible Solution

Land this PR after normal exact-head CI and maintainer merge gating; linked issue #76035 should close when it merges.

Yes. A FinalizedMsgContext with blank or whitespace BodyForCommands and nonblank RawBody reproduces the empty content path on current main, and the Slack prepare path can create that shape from bot alert attachments.

## Solution Assessment

Yes. The proposed nonblank precedence rule is the narrow maintainable fix because it preserves command-body priority while allowing raw inbound content only when command content is blank.
