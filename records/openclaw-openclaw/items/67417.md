## Summary

Keep open. Current main and the latest checked release cover known volatile paths and EOF-class tar races, but `openclaw backup create` still aborts on deleted-file ENOENT from `node-tar` before OpenClaw's filter can skip the entry, and recent issue comments report the same failure on newer releases.

## What This Changes

The issue asks for `openclaw backup create` to tolerate live session or state files that disappear during archive creation instead of aborting the whole backup.

## Best Possible Solution

Implement one coherent backup archiver fix that treats expected per-entry ENOENT under volatile live state as skipped, reports the skipped class/count, cleans failed temp archives, and adds a regression test for deletion between directory walk and lstat.

## Reproduction Assessment

Yes. Source inspection shows `node-tar` can emit ENOENT before OpenClaw's filter runs, current retry logic rejects non-EOF errors, and recent comments provide matching failures on 2026.6.x builds.

## Solution Assessment

No. The current EOF retry and volatile extension filter are only partial; the best fix is per-entry deleted-file tolerance or an equivalent archive walk that cannot fail the whole backup on expected live-state ENOENT.
