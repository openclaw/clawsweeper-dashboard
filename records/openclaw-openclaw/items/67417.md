## Label Justifications

- `P2`: The issue affects scheduled backup reliability for live installs but has a bounded CLI/session-state blast radius.
- `impact:session-state`: The reported failure is triggered by live session and agent state files disappearing while backup archives the state tree.

## Summary

Keep open. Current main and the latest release still retry only EOF-class tar races, while this issue now has recent release confirmations and an open candidate PR for the ENOENT deletion race.

## What This Changes

The issue requests that `openclaw backup create` tolerate live state/session files disappearing during archive creation instead of aborting the whole backup with ENOENT.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/96911, or an equivalent archiver fix, with regression coverage and real proof that backup creation survives a planned state file disappearing mid-archive.

## Reproduction Assessment

Yes for source-level reproduction: current main calls `tar.c` over live state roots and only retries EOF-class tar errors, while tar emits lstat ENOENT as a rejected Pack error. I did not run a live current-main race in this read-only review.

## Solution Assessment

Yes, the best fix is in the backup archiver rather than another list of special-case filters. The open candidate PR is a plausible narrow path, but it needs real disappearing-file proof before the issue should close.
