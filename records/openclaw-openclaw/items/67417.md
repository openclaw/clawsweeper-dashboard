## Summary

Keep open. Current main still has the deleted-before-lstat ENOENT path by source and dependency-contract inspection, and the open closing PR at https://github.com/openclaw/openclaw/pull/67472 remains unmerged and conflicting.

## What This Changes

The issue reports that `openclaw backup create` can abort with ENOENT when a session transcript is removed between backup planning and tar archive creation.

## Best Possible Solution

Land or replace the open archiver fix so expected per-entry ENOENT/ENOTDIR during archive creation is non-fatal, skipped paths are reported clearly, and regression plus real backup proof cover a transcript removed during the archive walk.

## Reproduction Assessment

Yes. By source inspection, current main feeds planned state roots to `tar.c`, tar@7.5.15 emits lstat errors before the filter can skip a volatile file, and session maintenance can rename transcript files while backup is walking the tree.

## Solution Assessment

No. Current main's volatile filter helps only when stat succeeds; the deleted-before-lstat path still needs the open ENOENT PR or a narrow replacement that avoids hanging and keeps backup metadata coherent.
