## Summary

Keep open. Current main and the latest release still only filter known volatile paths and retry EOF-class tar races; deleted-file ENOENT can still abort `openclaw backup create`, and recent comments confirm the same live-backup failure on 2026.6.x builds.

## What This Changes

The issue asks for `openclaw backup create` to tolerate session or live-state files disappearing during archive creation instead of failing the whole backup.

## Best Possible Solution

Implement one coherent backup archiver fix that treats expected per-entry live-state ENOENT as skipped, reports the skipped class/count, cleans failed temp archives, and preserves hard failures for unexpected paths.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `tar@7.5.16` can emit ENOENT before OpenClaw's filter runs, and current retry logic rejects non-EOF errors; this read-only review did not run a live deletion race.

## Solution Assessment

No. The current EOF retry and known-path volatile filter are only partial; the narrow maintainable fix is per-entry deleted-file tolerance or an equivalent archive walk that cannot fail the whole backup on expected live-state ENOENT.
