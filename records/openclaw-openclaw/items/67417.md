## Summary

Keep open. Current main mitigates some live-backup races with volatile filtering and EOF retries, but it still does not implement the requested generic per-entry ENOENT skip for files that disappear during archive creation.

## What This Changes

`openclaw backup create` should tolerate live session or runtime files disappearing between archive enumeration and entry write instead of failing the whole backup.

## Best Possible Solution

Implement one backup archiver path that preserves the current volatile filter and EOF retry behavior while demoting expected per-entry ENOENT to a reported skipped entry.

## Reproduction Assessment

Yes for source-level reproduction: node-tar can emit ENOENT from `lstat`, and current OpenClaw only retries EOF-class tar errors. A live current-main race proof is still needed after a fix.

## Solution Assessment

Yes, the per-entry fail-open direction is the best fix shape; adding more known volatile filters helps but cannot cover every expected disappearing file under a live state tree.
