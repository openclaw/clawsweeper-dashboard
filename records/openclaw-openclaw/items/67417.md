## Summary

Keep open. Current main still has the deleted-file race: `openclaw backup create` uses one `tar.c(...)` walk, and tar can emit `lstat` ENOENT before OpenClaw's volatile-file filter runs. The open closing PR at https://github.com/openclaw/openclaw/pull/67472 remains the right review, rebase, or replacement target.

## What This Changes

The issue asks for `openclaw backup create` to skip session transcript files that disappear during archive creation instead of failing the whole backup.

## Best Possible Solution

Land one coherent backup archiver fix that treats expected per-entry deleted-file ENOENT or ENOTDIR as non-fatal, reports skipped files clearly, and adds regression coverage for a planned session transcript removed during archive creation.

## Reproduction Assessment

Yes, by source inspection. Session maintenance can rename a transcript while backup is walking the state tree, and tar 7.5.15 emits lstat ENOENT before OpenClaw's current filter can skip the entry.

## Solution Assessment

No, current main is not enough yet. The merged volatile filter and EOF retry reduce adjacent live-backup failures, but this deleted-before-lstat path still needs the open closing PR or a narrow replacement fix.
