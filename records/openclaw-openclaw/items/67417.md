## Summary

Keep open. Current main skips known volatile files and retries EOF-class tar races, but it still uses `tar.c` and the tar dependency still `lstat`s filesystem entries before OpenClaw's filter can skip them, so a session transcript deleted during the archive walk can still fail with the reported ENOENT. The open closing PR at https://github.com/openclaw/openclaw/pull/67472 remains the right review or replacement target.

## What This Changes

The issue asks for `openclaw backup create` to treat session transcript files deleted during archive creation as per-entry skips instead of aborting with ENOENT.

## Best Possible Solution

Land one coherent backup archiver fix that treats expected deleted-file ENOENT as a non-fatal per-entry skip, reports skipped files clearly, and adds regression coverage for a planned session transcript removed during archive creation.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: session cleanup can rename a transcript while `backup create` is walking state, and node-tar emits lstat ENOENT before OpenClaw's current volatile filter runs.

## Solution Assessment

No. Current main's volatile filter and EOF retry are useful adjacent hardening, but the deleted-before-lstat ENOENT path needs an ENOENT-aware archiver fix or a repaired replacement for https://github.com/openclaw/openclaw/pull/67472.
