## Summary

Keep open. Current main still routes `openclaw backup create` through a single `tar.c` archive operation where tar can emit lstat ENOENT before OpenClaw's volatile-file filter runs; the open closing PR at https://github.com/openclaw/openclaw/pull/67472 remains the right review or replacement target.

## What This Changes

The issue requests that `openclaw backup create` skip session transcript files that disappear during live backup instead of aborting with ENOENT.

## Best Possible Solution

Land one coherent backup archiver fix that treats expected per-entry ENOENT/ENOTDIR as non-fatal, reports skipped files clearly, and adds regression coverage for a planned session transcript removed during archive creation.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: session cleanup can rename a transcript while `backup create` is walking state, and tar 7.5.15 emits lstat ENOENT before OpenClaw's current filter can run.

## Solution Assessment

No. Current main's volatile filter and EOF retry are useful adjacent hardening, but the deleted-before-lstat ENOENT path still needs a per-entry skip archiver fix or a repaired replacement for https://github.com/openclaw/openclaw/pull/67472.
