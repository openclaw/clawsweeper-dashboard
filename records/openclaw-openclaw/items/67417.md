## Summary

Keep open. Current main and the latest release still handle EOF-class live-write races and known volatile filters, but a deleted-file ENOENT from node-tar lstat can still abort `openclaw backup create`; recent issue comments confirm the same failure family on 2026.6.x builds.

## What This Changes

`openclaw backup create` is reported to fail when live session or agent-state files disappear after backup planning but before tar archiving.

## Best Possible Solution

Keep this as the canonical backup ENOENT race and land one coherent archiver fix that treats expected live-state disappearance as a reported skip while preserving hard failures for unexpected paths.

## Reproduction Assessment

Yes. Current OpenClaw source plus node-tar 7.5.16 source show that deleted files can surface as non-EOF lstat ENOENT errors, and recent comments provide live 2026.6.x confirmations for disappearing usage-cost and Codex shell snapshot paths.

## Solution Assessment

No. The existing EOF retry and known volatile filter are useful partial mitigations, but the best remaining solution is explicit per-entry deleted-file tolerance or an equivalent archive walk policy for expected live-state paths.
