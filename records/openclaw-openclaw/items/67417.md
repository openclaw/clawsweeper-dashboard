## Summary

Keep open. Current main and the latest release still use `tar.c` with an EOF-only retry wrapper, so a source file that disappears before node-tar's `lstat` remains fatal; the existing volatile-file filter helps after stat succeeds but is not a pre-lstat guard. The issue is also paired with an open closing PR at https://github.com/openclaw/openclaw/pull/67472, so it should stay open until that PR or a replacement lands.

## What This Changes

Make `openclaw backup create` tolerate session transcript files being deleted while the backup archive is being built.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/67472 with a narrow archiver fix that treats expected per-entry ENOENT/ENOTDIR as non-fatal and adds regression proof for a transcript removed during archive creation.

## Reproduction Assessment

Yes by source inspection. Current main feeds source roots to node-tar, node-tar emits lstat errors before the volatile filter can run, and session maintenance can rename transcript files while backup walks the tree.

## Solution Assessment

No, current main is not enough. The merged volatile-filter and EOF retry work handles live append races, but the deleted-before-lstat path still needs the linked PR or a safer replacement implementation.
