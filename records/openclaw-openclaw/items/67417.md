## Summary

Keep open. Current main has adjacent live-backup hardening, but the deleted-before-lstat ENOENT path is still source-reproducible, and the open closing PR at https://github.com/openclaw/openclaw/pull/67472 remains the review, rebase, or replacement target.

## What This Changes

The issue asks for `openclaw backup create` to tolerate session transcript files disappearing during archive creation instead of failing the entire backup.

## Best Possible Solution

Land one coherent backup archiver fix that treats expected per-entry deleted-file ENOENT/ENOTDIR as non-fatal, reports skipped files clearly, and adds regression coverage for a planned session transcript removed during archive creation.

## Reproduction Assessment

Yes, by source inspection rather than a live run: current main feeds state roots to `tar.c`, node-tar emits lstat errors before filter, and session maintenance can rename transcript files.

## Solution Assessment

No, current main is not enough yet. The merged volatile filter handles existing volatile files after successful stat, but the deleted-before-lstat ENOENT path still needs the open closing PR or a narrow replacement fix.
