## Summary

Keep open. Current main reduces related live-backup failures by skipping known volatile files when `node-tar` reaches them, but the dependency source shows `node-tar` still `lstat`s filesystem paths before applying the filter, so a session transcript deleted between directory enumeration and stat can still surface the reported ENOENT; the open closing PR remains the right review target.

## What This Changes

The issue asks for `openclaw backup create` to tolerate session transcript files disappearing during archive creation instead of aborting the whole backup with ENOENT.

## Best Possible Solution

Land one current-main backup archiver fix that treats expected ENOENT from disappearing volatile session transcripts as a per-entry skip, reports the skip count clearly, and adds regression coverage for a file removed between directory enumeration and archive stat.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a session transcript under `agents/<agent>/sessions/` can be renamed by cleanup while `node-tar` walks the state tree, and `node-tar` emits stat ENOENT before OpenClaw's current filter can skip it.

## Solution Assessment

No. The current merged volatile-file filter is a good related EOF fix, but it is not sufficient for this ENOENT race because the filter runs after stat; the narrow maintainable solution is an ENOENT-aware archiver path or a rebased replacement for the existing closing PR.
