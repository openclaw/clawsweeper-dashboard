## Summary

Keep open. Current main still sends `openclaw backup create` through a single `node-tar` archive operation without an ENOENT-specific skip path, while session maintenance can rename transcript files during the archive walk. The issue is already paired with open closing PR #67472 and overlaps with broader open PR #72251, so it should wait for maintainer PR selection or a merged replacement fix.

## What This Changes

The issue requests that `openclaw backup create` skip session transcript files that disappear during backup creation and report them instead of aborting the whole backup.

## Best Possible Solution

Review and land one coherent backup archiver fix in `src/infra/backup-create.ts` that treats expected deleted-file ENOENT during archive creation as non-fatal, reports skipped paths clearly, and adds a regression test proving backup creation succeeds when a planned session transcript is removed or renamed during the archive walk.

## Reproduction Assessment

Yes. The issue gives a concrete race path, and current source confirms backup planning can hand a state tree to node-tar while session maintenance can rename transcript files during the later archive walk.

## Solution Assessment

No, current main is not the best solution because it still lets node-tar abort the whole archive on missing entries. The reporter's per-entry ENOENT skip is a narrow maintainable direction, while PR #72251 should only supersede it if it explicitly covers deleted-file ENOENT as well as EOF races.
