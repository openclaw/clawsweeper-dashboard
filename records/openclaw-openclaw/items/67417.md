## Label Justifications

- `P2`: The issue affects scheduled backup reliability for live installs, but the blast radius is bounded to the backup CLI/state archive path.
- `impact:session-state`: The reported failure is triggered by live session and agent-state files disappearing while the backup archives the state tree.

## Summary

Keep open. Current main still does not treat node-tar ENOENT as retryable or skippable, and the open candidate PR that targets this race has not merged yet.

## What This Changes

The issue asks for `openclaw backup create` to tolerate live session or temp files disappearing during archive creation instead of aborting the whole backup.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/96911, or an equivalent backup-archiver fix, with regression coverage and real proof that backup creation survives a planned state file disappearing mid-archive.

## Reproduction Assessment

Yes for source-level reproduction: current main calls `tar.c` over live state roots, tar 7.5.16 emits `lstat` errors before filtering when a file disappears, and reporters confirmed the failure on released builds. I did not run a live timing race in this read-only review.

## Solution Assessment

Yes, keeping this issue open is the best current outcome. The fix belongs in backup archiver deletion-race handling, and the open candidate PR or an equivalent patch should prove the real disappearing-file path before closure.
