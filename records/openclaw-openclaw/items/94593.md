Full review comments:

- **[P1] Preserve sibling pointers when the env changes:** `src/gateway/session-transcript-files.fs.ts:391-392`
  - body: This check only accepts the sanitized override-directory basename. If a session was created before `OPENCLAW_TRAJECTORY_DIR` was enabled, its pointer names the default `<sid>.trajectory.jsonl` sibling; after the env var is set, `defaultRuntimePath` changes, this validation rejects the pointer-owned sibling, and reset can still leave the forensic trajectory to be removed later. Accept the env-free sibling path the way `resolveTrajectoryRuntimeFile` does before applying the basename gate.
  - confidence: 0.88
- **[P1] Constrain override cleanup to owned archives:** `src/gateway/session-transcript-files.fs.ts:640-642`
  - body: Adding the whole override directory to generic archive cleanup means every file whose name matches `.reset.<ts>` or `.deleted.<ts>` can be removed, even if OpenClaw did not create it. Since `OPENCLAW_TRAJECTORY_DIR` is operator supplied and can point outside session storage, filter cleanup to proven trajectory archives or track the archive paths explicitly.
  - confidence: 0.84
