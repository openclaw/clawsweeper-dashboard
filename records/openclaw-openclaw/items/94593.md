## Summary

The PR is a real candidate for the linked reset trajectory preservation bug, but it should not merge yet: the current diff can still miss pointer-owned sibling trajectory files after an env change and can delete archive-looking files from an operator-configured override directory without proving ownership.

## What This Changes

The PR archives reset-time session trajectory runtime and pointer artifacts alongside transcript reset archives, excludes those archives from transcript classifiers, extends override-dir retention, and adds filesystem regression tests.

## Best Possible Solution

Keep the shared reset archive helper, but align pointer ownership with the existing trajectory runtime resolver and constrain override-directory retention to OpenClaw-owned trajectory archives before merge.

Yes. Current main archives only transcript candidates on reset, and the PR body includes copied before/after output showing default trajectory loss on main and preservation on the branch; the remaining blockers are source-reproducible ownership-boundary cases.

## Solution Assessment

No. The shared reset archive helper is the right layer for the main bug, but this branch still mishandles pointer-owned sibling runtimes when env/default locations differ and expands override cleanup too broadly.
