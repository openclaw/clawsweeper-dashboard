Full review comments:

- **[P1] Preserve sibling pointers when the env changes:** `src/gateway/session-transcript-files.fs.ts:391-392`
  - body: This check only trusts the sanitized override-directory basename. Existing sessions can have a pointer to the default `<sid>.trajectory.jsonl` sibling; if `OPENCLAW_TRAJECTORY_DIR` is later enabled, `defaultRuntimePath` changes, the sibling pointer is rejected, and reset can still leave the trajectory to later cleanup. Accept the env-free sibling path as owned before the basename/header gate.
  - confidence: 0.88
- **[P1] Constrain override cleanup to owned archives:** `src/gateway/session-transcript-files.fs.ts:640-642`
  - body: Adding the whole override directory to generic archive cleanup means every file matching `.reset.<ts>` or `.deleted.<ts>` can be removed, even if OpenClaw did not create it. Since `OPENCLAW_TRAJECTORY_DIR` is operator supplied and may be shared, filter cleanup to proven trajectory archives or track archived paths explicitly.
  - confidence: 0.84
