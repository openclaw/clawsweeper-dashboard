## Summary

Keep open. Current main still has Discord native reply threading and outbound mention-alias rewriting, but it does not expose the requested default-off config or prepend a visible original-author mention to guild replies; adding that notification behavior needs maintainer product approval.

## What This Changes

The issue requests a default-off Discord setting that prepends a visible original-author mention to guild reply messages when the model omits it.

## Best Possible Solution

Add a Discord-plugin-owned, default-off reply mention formatter with schema/type/docs coverage, applying only to guild reply first chunks that lack the resolved original-author mention.

## Reproduction Assessment

Yes for the current source behavior. `sendMessageDiscord` with `replyTo` passes unchanged content into `buildDiscordMessageRequest`, which only adds `message_reference`, and the requested config key is absent.

## Solution Assessment

Yes, if maintainers accept the feature. A default-off formatter in the Discord plugin delivery path is the narrowest maintainable direction; core prompt guidance or global reply behavior would be weaker boundaries.
