## Summary

Keep open. Current main and the latest release still have Discord native reply references and outbound mention-alias rewriting, but no default-off config or formatter that prepends the original guild author mention to reply text; this remains a product/config decision rather than conservative cleanup.

## What This Changes

The issue requests a default-off Discord config option that prepends the original author’s visible mention to the first outbound guild reply chunk when a reply reference exists.

## Best Possible Solution

Add a Discord-plugin-owned, default-off reply mention formatter with schema, type, docs, and tests, limited to guild reply first chunks that lack the resolved original-author mention.

## Reproduction Assessment

Yes, at source level. Current main and `v2026.5.19` lack the requested config key and visible reply-author mention injection while retaining native `message_reference` reply behavior.

## Solution Assessment

Unclear until maintainers approve the product behavior. If accepted, a default-off formatter in the Discord plugin send path is the narrowest maintainable direction; prompt rules or global core behavior would be weaker boundaries.
