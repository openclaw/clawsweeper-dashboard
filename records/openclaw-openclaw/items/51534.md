## Summary

Keep open. Current main still has native Discord reply references and outbound mention-alias rewriting, but it does not provide the requested default-off auto-mention config or any formatter that prepends the original guild author mention to reply text; adjacent routing work does not replace this outbound notification feature.

## What This Changes

The issue requests a default-off `channels.discord.autoMentionReply` option that prepends the original author's visible Discord mention to guild reply messages when needed.

## Best Possible Solution

Add a Discord-plugin-owned, default-off reply mention formatter with schema, type, docs, and tests, limited to guild reply first chunks that lack the resolved original-author mention.

## Reproduction Assessment

Yes, at source level. Current main has reply references and mention-alias rewriting, but the requested config key and visible reply-author mention injection are absent; I did not run a live Discord guild scenario.

## Solution Assessment

Unclear until maintainers approve the product behavior. If accepted, a default-off formatter in the Discord plugin send path is the narrowest maintainable direction; prompt rules or global core behavior would be weaker boundaries.
