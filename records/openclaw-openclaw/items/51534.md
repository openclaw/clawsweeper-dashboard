## Summary

Keep open. Current main still lacks the requested automatic original-author mention injection; the related `mentionAliases` work helps deterministic known-name rewrites but does not prepend the author of a replied-to guild message. Because this adds a new Discord config option and changes notification behavior, it needs maintainer product/owner review rather than a conservative cleanup close.

## What This Changes

The issue requests an optional Discord setting that prepends the original author mention to guild reply messages when the model omits it.

## Best Possible Solution

If maintainers want this behavior, implement it inside the Discord plugin as an explicit opt-in formatting step that uses trusted inbound reply author facts, only applies to guild replies, skips existing mentions, documents notification semantics, and covers text, media, webhook/thread-bound, and chunked sends.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature/config request. Source inspection does show current main supports native reply references and mention alias rewriting, but not the requested automatic original-author mention prefix.

## Solution Assessment

Unclear until maintainers decide the policy. A plugin-local opt-in formatting layer is plausible, but adding a notification-affecting config option and defining behavior without reply context should be an owner decision first.
