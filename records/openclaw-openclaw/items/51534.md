## Summary

Keep open. Current main supports Discord native reply references and deterministic mention-alias rewriting, but it still has no config option or delivery formatter that prepends a visible original-author mention; the related merged threading work only fixed reply/thread plumbing.

## What This Changes

The issue requests a default-off Discord `channels.discord.autoMentionReply` option that prepends `<@originalAuthorId>` to eligible guild reply messages when the first outbound chunk lacks it.

## Best Possible Solution

Add a Discord plugin-owned, default-off formatter that injects a visible original-author mention only for eligible guild reply text, with schema/types/docs/changelog and regression coverage across the Discord delivery paths.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: sending Discord text with `replyTo: "orig-123"` produces `message_reference`, while the schema/types lack `autoMentionReply` and the mention rewriter leaves text without `@` unchanged.

## Solution Assessment

Yes. A default-off Discord delivery formatter is the narrowest maintainable direction; native replies and `mentionAliases` are insufficient because they do not add a visible original-author mention when the model omits it.
