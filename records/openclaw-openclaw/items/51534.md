## Summary

Keep open: current main supports Discord reply threading and configured mention alias rewrites, but it does not implement an opt-in auto-mention pass for the original author. This is a new Discord plugin feature/config decision rather than an implemented cleanup candidate.

## What This Changes

The issue requests a Discord option that prepends an explicit original-author mention to guild reply messages when the model omits it.

## Best Possible Solution

If maintainers accept the behavior, implement it inside the Discord plugin outbound formatting path with an opt-in config surface, focused schema/types/docs updates, and regression tests for guild-only first-chunk mention injection.

## Reproduction Assessment

Not applicable. This is a feature/config request; source inspection confirms current main only attaches native Discord reply references and rewrites mentions already present in outbound text.

## Solution Assessment

Unclear. The requested direction is plausible, but the maintainable shape should be a Discord-plugin-owned formatting/delivery policy rather than generic gateway behavior, and the no-`replyTo` session cases need explicit scope decisions.
