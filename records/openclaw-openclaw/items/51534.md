## Summary

Keep open. Current main has Discord native reply threading and explicit mention-alias rewriting, but it still has no default-off auto-mention config or formatter that prepends the original author's visible mention to guild replies; this is a feature/config decision, not an implemented cleanup item.

## What This Changes

Request to add a default-off Discord config that prepends a visible original-author mention to guild reply messages when the model did not include one.

## Best Possible Solution

Add a Discord-plugin-owned, default-off reply mention formatter with schema/type/docs coverage, limited to guild reply first chunks that lack the resolved original-author mention.

## Reproduction Assessment

Yes for source-level behavior. `sendMessageDiscord` passes reply content through existing mention-alias rewriting and `buildDiscordMessageRequest` only adds `message_reference`, while the requested config key is absent from schema, types, and docs.

## Solution Assessment

Unclear until maintainers accept the product behavior. If accepted, a default-off formatter in the Discord plugin delivery path is the narrowest maintainable approach; prompt instructions or global core behavior would be weaker boundaries.
