## Summary

Keep open. Current main still has Discord native reply references and deterministic outbound mention rewriting, but no config surface or delivery formatter that prepends a visible original-author mention for guild replies.

## What This Changes

Add a default-off Discord config option that prepends a visible original-author mention to eligible guild reply messages when the model omits it.

## Best Possible Solution

Add the formatter inside the Discord plugin delivery path, default it off, document the public config key, and cover text/media/webhook/component reply paths without changing core reply semantics.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a Discord send with text "hello" and replyTo "orig-123" keeps content as "hello" while adding only message_reference, and the schema/types do not expose autoMentionReply.

## Solution Assessment

Yes. A default-off Discord plugin-owned delivery formatter is the narrowest maintainable direction; native replies and mentionAliases do not add a visible original-author mention when the model omits it.
