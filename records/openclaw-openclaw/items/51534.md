## Summary

Keep this open: current main and v2026.6.6 handle Discord native replies and mention-bearing final text, but they still do not synthesize an explicit mention of the original reply author. The remaining request is a new Discord-plugin behavior/config decision, not an implemented cleanup item.

## What This Changes

The issue requests an opt-in Discord setting that prepends the original author mention to guild reply messages when the agent did not include it.

## Best Possible Solution

Make an explicit Discord-plugin product decision: either keep the current mentionAliases/prompt-guidance model, or add a narrowly scoped opt-in author-mention path with schema/docs/tests and live Discord proof.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection does show that current Discord delivery only rewrites existing mentions and carries message_reference, so the requested auto-injected original-author mention is absent.

## Solution Assessment

Unclear. The proposed opt-in config is plausible, but the best fix needs maintainer judgment because OpenClaw has a high bar for new config surfaces and current mentionAliases/final-reply behavior already solves part of the notification problem.
