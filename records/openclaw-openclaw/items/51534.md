## Summary

Keep this open: current main and the latest release still support Discord native reply references and deterministic mention aliases, but they do not implement automatic original-author mention injection for guild replies.

## What This Changes

The issue requests an opt-in Discord setting that prepends the original author’s `<@userId>` mention to guild reply messages when the agent did not include it.

## Best Possible Solution

If maintainers approve the behavior, implement it inside the Discord plugin as a narrow opt-in post-processing path with schema/docs/tests and live Discord proof; otherwise keep `mentionAliases` and explicit model guidance as the supported path.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection shows current Discord delivery carries `message_reference` and rewrites existing mention text, but does not auto-inject a mention for the original reply author.

## Solution Assessment

Unclear. The proposed opt-in config is plausible, but the best fix needs maintainer judgment because OpenClaw has a high bar for new config surfaces and existing `replyToMode`/`mentionAliases` solve adjacent parts of the notification problem.
