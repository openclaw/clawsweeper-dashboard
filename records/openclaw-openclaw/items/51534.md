## Summary

Keep this open: current main still has Discord reply references and deterministic mention-alias rewriting, but it does not implement automatic original-author mention injection for guild replies. The request is concrete and aligned with Discord channel reliability work, but it adds a new Discord behavior/config surface and still needs maintainer product direction plus live Discord proof.

## What This Changes

Request to add an opt-in Discord setting that prepends the original author's `<@userId>` mention to guild reply messages when the model did not include it.

## Best Possible Solution

If maintainers approve the product direction, implement this inside the Discord plugin as a narrow opt-in post-processing path with schema/docs/tests and live Discord proof; otherwise keep `replyToMode`, `mentionAliases`, and explicit prompt guidance as the supported path.

## Reproduction Assessment

Yes for source-level behavior, but not for live transport. Current main serializes Discord reply references and rewrites existing mention aliases only; no live Discord guild proof was run for the notification behavior.

## Solution Assessment

Unclear. The proposed opt-in Discord config is plausible, and the Discord plugin post-processing layer is the right owner if approved, but the best permanent behavior needs maintainer product judgment because this expands channel config and notification policy.
