## Summary

Keep this open: current main and the latest release still thread Discord replies by message id but do not synthesize an explicit mention of the original reply author. A related merged fix preserves notifications when the assistant text already contains a targeted mention, but it does not implement the requested opt-in auto-mention behavior.

## What This Changes

The issue requests a Discord configuration option that automatically prepends the original author mention to guild reply messages when reply context exists and the agent output omitted it.

## Best Possible Solution

Keep the behavior Discord-plugin-owned: if maintainers approve the product knob, carry reply-author identity through the reply delivery context and inject the mention only for opted-in guild replies without duplicating existing mentions or escalating broadcasts.

## Reproduction Assessment

Yes at source level for the missing capability: current main can thread a Discord reply by message id but does not carry the original author id into outbound message construction for mention injection. I did not run a live Discord guild proof, so notification behavior itself remains unverified here.

## Solution Assessment

Unclear until maintainers decide the config/product contract. A Discord-plugin-owned opt-in that reuses reply context would be the maintainable shape, while unconditional injection or core-level Discord policy would be the wrong layer.
