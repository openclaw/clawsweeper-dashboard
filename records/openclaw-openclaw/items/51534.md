## Summary

Keep this open: current main and v2026.6.6 support Discord reply references, configured mention aliases, and fresh-send handling for final text that already contains a targeted mention, but they still do not synthesize an explicit original-author mention from reply context.

## What This Changes

The issue requests an opt-in Discord behavior/config option that prepends `<@originalAuthorID>` to guild reply messages when the outbound text does not already mention that author.

## Best Possible Solution

Make an explicit Discord-plugin product decision: either keep the supported `mentionAliases` and model-guidance path, or add a narrowly scoped opt-in author-mention path with schema/docs/tests and live Discord proof.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection does show that current Discord delivery carries `message_reference` and rewrites existing mention text, but does not auto-inject a mention for the original reply author.

## Solution Assessment

Unclear. The proposed opt-in config is plausible, but the best fix needs maintainer judgment because OpenClaw has a high bar for new config surfaces and existing `mentionAliases`/fresh-send behavior already solves part of the notification problem.
