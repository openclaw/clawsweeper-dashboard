## Summary

Keep open. Current main and the latest release still only preserve Discord reply references and rewrite existing outbound mentions; they do not provide the requested default-off formatter that prepends the original guild author mention, and the remaining decision is a public Discord config/API choice.

## What This Changes

Add a default-off Discord config option that prepends the original guild author mention to outbound guild replies when the reply text does not already mention them.

## Best Possible Solution

If maintainers accept the behavior, implement it inside the Discord plugin outbound delivery path with a default-off config, schema/docs/tests, author-resolution guards, and first-chunk-only duplicate protection.

## Reproduction Assessment

Yes, at source level. Current main accepts replyTo and emits message_reference, while targeted source/schema/docs searches show no autoMentionReply option or first-chunk reply-author mention formatter.

## Solution Assessment

Unclear until maintainers approve the product behavior. If accepted, a Discord-plugin-owned default-off formatter is the narrow maintainable direction; prompt rules or core-wide behavior would be weaker boundaries.
