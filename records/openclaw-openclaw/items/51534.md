## Summary

Keep this open: current main and the latest release still support native Discord reply references and deterministic outbound mention aliases, but they do not synthesize an explicit original-author mention from reply context.

## What This Changes

Request to add an opt-in Discord `autoMentionReply` setting that prepends the original guild reply author mention to outbound reply chunks when the agent omitted it.

## Best Possible Solution

If maintainers want this behavior, implement it inside the Discord plugin as a narrowly scoped opt-in post-processing path with schema/docs/tests/live Discord proof; otherwise keep `mentionAliases` and model guidance as the supported path.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection shows current Discord delivery carries `message_reference` and rewrites existing mention text, but does not auto-inject a mention for the original reply author.

## Solution Assessment

Unclear. The proposed opt-in config is plausible, but the best fix needs maintainer judgment because OpenClaw has a high bar for new config surfaces and existing `mentionAliases`/fresh-send behavior already solves part of the notification problem.
