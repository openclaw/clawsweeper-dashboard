## Summary

Keep open. Current main and v2026.5.20 still support native Discord reply references and outbound mention-alias rewriting, but they do not implement a default-off formatter that prepends the original guild author mention; the adjacent implicit-reply PR covers inbound wake-up policy, not this outbound visible mention request.

## What This Changes

Request a default-off Discord config option that prepends a visible mention of the original guild message author to outbound reply text.

## Best Possible Solution

Add a Discord-plugin-owned, default-off reply-author mention formatter with schema, docs, and tests, limited to first guild reply chunks where the original author is resolved and not already mentioned.

## Reproduction Assessment

Yes, at source level. Current main and v2026.5.20 expose replyToMode/message_reference and mentionAliases, but no autoMentionReply config or first-chunk reply-author mention formatter in the send path.

## Solution Assessment

Unclear until maintainers approve the product behavior. If accepted, a default-off formatter in the Discord plugin send path is the narrowest maintainable direction; prompt rules or global core behavior would be weaker boundaries.
