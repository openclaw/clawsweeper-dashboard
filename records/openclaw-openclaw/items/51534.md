## Summary

Keep this open: current main supports Discord reply references and deterministic mention rewriting when the mention is already present, but it does not auto-inject the original author's `<@id>` into guild replies. The request is aligned with Discord reliability work, but it adds a new notification/config behavior and still needs maintainer product direction plus live Discord proof.

## What This Changes

Request an opt-in Discord setting that prepends the original author's mention to guild reply messages when the outbound text lacks that mention.

## Best Possible Solution

Decide the Discord notification policy first; if accepted, implement it in the Discord plugin reply delivery path with narrow opt-in config, schema/docs/tests, and live guild proof that only the intended reply author is pinged.

## Reproduction Assessment

No high-confidence live reproduction is available. Source inspection does show the requested `autoMentionReply` behavior is absent: replies add `message_reference`, and mention rewriting only handles mentions already present in outbound text.

## Solution Assessment

Unclear pending product direction. Discord-plugin post-processing is the plausible owner boundary, but the config name, default, inheritance scope, and notification safety rules need maintainer approval before this is the best fix.
