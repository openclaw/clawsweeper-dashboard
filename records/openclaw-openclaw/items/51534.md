## Summary

Keep open. Current main still supports Discord native reply references and deterministic outbound mention rewriting, but it does not implement a default-off formatter/config path that prepends a visible original-author mention for guild replies.

## What This Changes

Requests a default-off Discord config option that prepends a visible original-author mention to eligible guild reply messages.

## Best Possible Solution

Add a Discord plugin-owned, default-off formatter that injects the visible original-author mention only for eligible guild reply text, with schema/types/docs/changelog and regression coverage across Discord delivery paths.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: a Discord text send with replyTo creates message_reference, while the config schema lacks autoMentionReply and the content path has no formatter that can add a missing <@userId> prefix.

## Solution Assessment

Yes. A default-off Discord delivery formatter is the narrowest maintainable direction; native reply references and mentionAliases are already present but do not create a visible original-author mention when the model omits it.

Add a default-off `channels.discord.autoMentionReply` option, and if the account/channel override pattern requires it, support the matching per-account config path.
