## Summary

Keep open. Current main and v2026.5.22 still support Discord native reply references and outbound mention alias rewriting, but they do not implement a default-off formatter that prepends the original guild author mention.

## What This Changes

Request adds a default-off Discord config option to prepend original guild author mentions to outbound reply chunks when the model omitted them.

## Best Possible Solution

Add a Discord-plugin-owned, default-off reply-author mention formatter with schema, docs, and tests, limited to first guild reply chunks where the original author is resolved and not already mentioned.

## Reproduction Assessment

Yes, at source level. Current main accepts replyTo and emits message_reference, but targeted searches and config/schema/docs inspection show no autoMentionReply option or first-chunk reply-author mention formatter.

## Solution Assessment

Unclear until maintainers approve the product behavior. If accepted, a default-off formatter in the Discord plugin send path is the narrowest maintainable direction; prompt rules or global core behavior would be weaker boundaries.
