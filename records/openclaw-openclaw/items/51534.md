## Summary

Keep open. Current main has native Discord reply threading and mention-alias rewriting, but it still lacks the requested default-off config and delivery formatter that would prepend a visible original-author mention for guild replies; the related merged reply-threading PR only fixed `replyToId`/`message_reference` plumbing.

## What This Changes

Request adding a default-off Discord `autoMentionReply` option that prepends a visible original-author mention to guild reply messages when the model omits it.

## Best Possible Solution

Add a Discord plugin-owned, default-off reply mention formatter with schema/type/docs coverage and focused tests across text, media, webhook, component, and threaded reply delivery paths.

## Reproduction Assessment

Yes. Source inspection on current main shows a Discord send with `replyTo` keeps content unchanged while adding only `message_reference`, and the schema/types do not expose `autoMentionReply`.

## Solution Assessment

Yes, if maintainers accept the feature. A default-off formatter in the Discord plugin delivery path is the narrowest maintainable direction; core reply semantics should stay unchanged.
