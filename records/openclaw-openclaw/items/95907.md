Keep open: current main and the latest release still default `deleteAfterRun` only for one-shot jobs while isolated-session cleanup is gated on that flag. ## What This Changes

The issue reports recurring isolated cron jobs leaving their agent session alive after a successful run, causing continued model-token consumption.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/95950 or a replacement that handles both fresh and existing recurring isolated cron jobs while preserving explicit `deleteAfterRun: false`.

## Reproduction Assessment

No high-confidence live-provider reproduction was run here. Source inspection does reproduce the gating problem: recurring isolated jobs omit `deleteAfterRun`, and the cleanup helper exits unless that flag is truthy.

## Solution Assessment

Not yet. The linked PR is the right candidate direction for fresh jobs, but the best fix should also settle upgrade behavior for existing omitted persisted jobs before this issue closes.
