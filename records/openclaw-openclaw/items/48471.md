## Summary

Keep open: current main and the latest release still provide separate installer, onboarding, dashboard, channel-add, Telegram allowlist, and resolver pieces, but not one supported bootstrap surface that composes daemon setup, dashboard auth, Telegram bot setup, and owner preauthorization end to end. The external sentinel-stack-public guide is useful evidence for the owner `allowFrom` preseed pattern, but it is not a shipped OpenClaw CLI/onboarding path.

## What This Changes

The issue requests a supported one-line local bootstrap path that installs or refreshes OpenClaw, starts the daemon gateway, exposes dashboard auth, configures Telegram, and preauthorizes the intended Telegram owner.

## Best Possible Solution

Define an official bootstrap/onboarding path that composes daemon setup, dashboard token delivery, Telegram bot setup, and numeric owner allowlist or explicit preapproval, with docs and tests preserving the DM pairing trust model.
