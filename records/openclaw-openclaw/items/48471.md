## Summary

Keep open: current main and v2026.5.22 have the separate installer/onboard, daemon, dashboard URL, Telegram setup, and resolver pieces, but still no supported one-line bootstrap surface that safely composes daemon setup, dashboard auth, Telegram bot setup, and owner preauthorization. The external sentinel-stack-public guide shared in the discussion is useful reference material, not a shipped OpenClaw core flow.

## What This Changes

The issue requests a supported one-line local bootstrap path that installs or refreshes OpenClaw, starts the gateway, provides dashboard auth, configures Telegram, and pre-authorizes the Telegram owner safely.

## Best Possible Solution

Define an official bootstrap/onboarding path that composes daemon setup, dashboard token delivery, Telegram bot setup, and numeric owner allowlist or token-verified preapproval, with docs and tests preserving the DM pairing trust model.
