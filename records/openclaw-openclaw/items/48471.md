## Summary

Keep open. Current main has several supporting pieces, and the external sentinel-stack-public workflow is useful reference context, but OpenClaw still lacks an official one-line localhost bootstrap that installs/starts the daemon, delivers dashboard auth, configures Telegram, and preauthorizes an owner end to end.

## What This Changes

Request a supported one-line local setup path for daemonized gateway, dashboard auth, Telegram bot configuration, and Telegram owner preauthorization.

## Best Possible Solution

Define an official bootstrap/onboarding path that composes daemon setup, dashboard token delivery, Telegram token setup, and numeric owner allowlist or explicit preapproval, with docs and tests preserving the DM pairing trust model.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source and docs give high-confidence current-state proof that the requested end-to-end bootstrap still requires separate setup surfaces.

## Solution Assessment

Unclear as an implementation path. The goal is valid, but maintainers should choose the official surface and security model before implementation rather than letting automation pick flags or a wrapper shape ad hoc.
