## Summary

Keep open. Current main has useful pieces, including installer handoff, non-interactive gateway setup, dashboard token URLs, Telegram numeric allowlists, and a Telegram resolver, but it still lacks one supported localhost bootstrap path that composes daemon setup, dashboard auth, Telegram bot setup, and owner preauthorization end to end; the external sentinel-stack-public guide is useful reference work, not an OpenClaw core surface.

## What This Changes

The issue asks for a supported one-line local bootstrap that installs or refreshes OpenClaw, starts the daemon, provides dashboard auth, configures Telegram, and preauthorizes the intended owner.

## Best Possible Solution

Define an official bootstrap/onboarding path that composes daemon setup, dashboard token delivery, Telegram bot setup, and numeric owner allowlist or explicit preapproval, with docs and tests preserving the DM pairing trust model.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source and docs give high-confidence current-state proof that the requested one-line bootstrap still requires separate setup surfaces.

## Solution Assessment

Unclear as an implementation path. The goal is valid, but maintainers should choose the official bootstrap surface and Telegram preauthorization model before implementation rather than letting automation pick flags or a wrapper shape ad hoc.
