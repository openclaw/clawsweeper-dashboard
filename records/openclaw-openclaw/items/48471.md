## Summary

Keep open: current main has the installer, non-interactive gateway setup, dashboard token URLs, Telegram setup, and a Telegram resolver, but not one supported bootstrap surface that composes daemon setup, dashboard auth, Telegram bot setup, and owner preauthorization end to end.

## What This Changes

Request a one-line local bootstrap path that installs or refreshes the daemon, provides dashboard auth, configures a Telegram bot, and preauthorizes the intended Telegram owner.

## Best Possible Solution

Define an official bootstrap/onboarding path that composes daemon setup, dashboard token delivery, Telegram bot setup, and numeric owner allowlist or explicit preapproval, with docs and tests preserving the DM pairing trust model.
