## Summary

Keep open. Current main has useful building blocks, including installer handoff, daemon refresh, dashboard token URLs, Telegram numeric allowlist setup, and a Telegram target resolver, but it still does not ship an official one-line localhost bootstrap that installs/starts the daemon, delivers dashboard auth, configures Telegram, and preauthorizes an owner end to end.

## What This Changes

Request a supported one-line local bootstrap path that installs/starts the gateway, surfaces dashboard auth, configures Telegram, and preauthorizes the intended Telegram owner.

## Best Possible Solution

Define an official bootstrap/onboarding path that composes daemon setup, dashboard token delivery, Telegram bot setup, and numeric owner allowlist or explicit preapproval, with docs and tests preserving the DM pairing trust model.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source and docs give high-confidence current-state proof that the requested one-line bootstrap still requires separate setup surfaces.

## Solution Assessment

Unclear as an implementation path. The goal is valid, but maintainers should choose the official bootstrap surface and Telegram preauthorization model before implementation rather than letting automation pick flags or a wrapper shape ad hoc.
