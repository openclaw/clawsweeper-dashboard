## Summary

Keep open: current main and v2026.6.6 still provide the installer/onboarding, daemon, dashboard URL, Telegram token, resolver, allowlist, and pairing pieces as separate surfaces, but no supported one-line flow configures the daemon, dashboard auth, and Telegram owner access end to end. ## What This Changes

Request a supported one-line local bootstrap path that installs or refreshes OpenClaw, starts the local gateway daemon, opens or prints dashboard auth, configures Telegram, and pre-authorizes the intended Telegram owner securely.

## Best Possible Solution

Keep this open for a maintainer-sponsored design that either adds a secure high-level bootstrap/non-interactive onboarding path or explicitly documents the supported allowlist/preapproval path without weakening pairing.
