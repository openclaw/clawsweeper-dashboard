## Summary

Keep open. Current main has several building blocks, including non-interactive daemon setup, dashboard token delivery, Telegram numeric allowlist setup, and a Telegram username resolver, but it still does not provide a supported one-line local bootstrap that configures Telegram owner access end to end.

## What This Changes

Request a supported one-line local bootstrap that installs or refreshes the gateway, delivers localhost dashboard auth, configures Telegram, and securely preauthorizes owner access.

## Best Possible Solution

Define an official bootstrap/onboarding path that composes daemon setup, dashboard token delivery, Telegram token setup, and numeric owner allowlist or explicit preapproval, with docs and regression coverage that keep DM pairing explicit unless preauthorized.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source and docs give high-confidence current-state proof that the requested end-to-end bootstrap still requires separate setup surfaces.

## Solution Assessment

Unclear as an implementation path. The goal is valid, but maintainers should choose the official surface and security model before implementation rather than letting automation pick flags or a wrapper shape ad hoc.
