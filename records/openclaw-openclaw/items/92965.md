## Summary

Keep this PR open for changes: it targets the right update-respawn bug, but the implementation rewrites only `args[0]`, which misses the script path whenever Node exec arguments precede `process.argv[1]`, and the supplied proof is unit/build output rather than a real respawn run.

## What This Changes

The PR rewrites pnpm versioned gateway respawn entry paths to `node_modules/openclaw/openclaw.mjs` and adds focused tests around that rewrite.

## Best Possible Solution

Rewrite the actual script argument after any Node exec arguments, keep exec flags preserved, add a pnpm-plus-execArgv regression test, and provide redacted real respawn output before merge.

## Reproduction Assessment

Yes at source level: current main builds update respawn arguments from `process.execArgv` plus `process.argv.slice(1)`, and the PR changes that same path. I did not run a live package-manager self-update in this read-only review.

## Solution Assessment

No. Rewriting to a stable wrapper is the right owner boundary, but this implementation rewrites `args[0]`, which is not the script path when Node exec arguments are present.
