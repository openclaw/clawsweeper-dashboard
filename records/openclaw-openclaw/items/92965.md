## Summary

Keep this PR open, but do not merge it yet: the fix targets a real Gateway update-respawn bug, but the patch misses respawns with Node runtime flags, misses Windows pnpm paths, and only supplies unit/build proof rather than a real update respawn run.

## What This Changes

The PR adds a pnpm path-rewrite helper in `src/infra/process-respawn.ts` and tests so update respawn children can use `node_modules/openclaw/openclaw.mjs` instead of version-pinned `.pnpm` entry paths.

## Best Possible Solution

Keep the stable wrapper approach, but rewrite the actual entrypoint argument across Node flags and platform path separators, then add real update-respawn proof before merge.

## Reproduction Assessment

Yes at source level: current main builds update respawn args from `process.execArgv` followed by `process.argv.slice(1)`, and the PR mutates only the first combined arg. I did not run a live package-manager self-update cycle.

## Solution Assessment

No. Stable entrypoint rewriting in the detached update respawn path is the right boundary, but this patch must handle Node runtime flags and Windows path separators before it is the best fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Rewrite the actual entry argument after execArgv:** `src/infra/process-respawn.ts:86`
  - body: `spawnDetachedGatewayProcess()` builds child args as `process.execArgv` followed by `process.argv.slice(1)`, so when the Gateway runs with Node flags like `--import`, `--require`, or `--inspect`, `args[0]` is a flag and the pnpm entry path remains versioned. The update child can still respawn from a removed `.pnpm` path; compute the entry index from `process.execArgv.length` or rewrite the argv slice before concatenating.
  - confidence: 0.95
- **[P2] Normalize Windows pnpm paths:** `src/infra/process-respawn.ts:44-45`
  - body: The matcher looks for the POSIX marker `node_modules/.pnpm/` and a slash-only regex. On Windows, `process.argv` paths use backslashes, and update respawn explicitly allows unmanaged Windows package swaps, so pnpm-installed Windows Gateways will keep the stale versioned entry path. Normalize separators or use platform-aware parsing, and add a Windows backslash regression test.
  - confidence: 0.91
