## Summary

Keep open. Current main has several opt-in mitigations for prompt cost, but the default short-reply path still builds the normal OpenClaw system prompt, injects workspace bootstrap context by default, and creates the tool inventory/schemas for tool-capable runs rather than lazy-loading a lightweight baseline automatically.

## What This Changes

The issue requests reducing baseline context load for minimal replies by lazy-loading context only when needed or adding a lightweight-vs-full mode.

## Best Possible Solution

Define a maintainer-approved context-cost policy for normal agent turns: keep full context where safety/bootstrap/tool use requires it, reduce repeated workspace and tool-heavy context for safe short continuations, preserve explicit full mode, and document verification through `/context detail`, `/usage`, and provider usage reporting.

## Reproduction Assessment

Yes for the structural issue: current source and docs show normal tool-capable turns still assemble the full prompt/tool/bootstrap path unless opt-in knobs are set. The precise token numbers should be reproduced with `/context detail` or `/usage` on a representative configured workspace.

## Solution Assessment

Unclear. The existing `continuation-skip`, `never`, automation lightweight context, context-engine, and `localModelLean` knobs are useful partial mitigations, but they are not the requested default lazy lightweight behavior for ordinary short replies.
