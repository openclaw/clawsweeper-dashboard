## Summary

Keep open: current main and v2026.5.28 still skip Teams Graph media lookup unless the inbound payload contains an HTML `<attachment>` marker, while the docs say Graph-enabled channel/group attachments should be downloadable.

## What This Changes

The issue asks OpenClaw to retrieve inbound Microsoft Teams channel attachments through Graph and surface them to the agent, including threaded replies that need parent/reply message addressing.

## Best Possible Solution

Define the Teams channel/group Graph lookup policy for markerless uploads, then update the msteams inbound media path so documented Graph-enabled attachments are surfaced without regressing mention-only messages.

## Reproduction Assessment

No live high-confidence reproduction was run. Source inspection shows the conditional failure path: when a Teams channel message has no direct downloadable media and no HTML `<attachment>` ID, current main skips Graph media lookup and returns no media.

## Solution Assessment

Unclear as an implementation direction. The requested behavior matches the docs, but an unconditional Graph fallback could regress the mention-only path that the current gate was added to protect, so maintainers should choose the trigger policy first.
