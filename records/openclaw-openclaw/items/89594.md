## Summary

Keep open: the report describes a credible Teams channel attachment gap, and current main still gates Graph media lookup on an inbound HTML `<attachment>` marker even though the report says some Teams channel uploads do not provide a usable marker.

## What This Changes

The issue asks OpenClaw to recognize and fetch inbound Microsoft Teams channel attachments, including thread replies, via Microsoft Graph when Bot Framework webhook payloads do not include file bytes.

## Best Possible Solution

Decide the Graph lookup policy for Teams channel/group messages, then update the Teams plugin so documented Graph-enabled attachments are surfaced without regressing the mention-only/no-file path.

## Reproduction Assessment

No live high-confidence reproduction was run. Source inspection does show a reproducible conditional path: if a Teams channel activity has no downloadable direct attachment and no HTML `<attachment>` tag, current code skips Graph media lookup and returns no media.

## Solution Assessment

Unclear as an implementation direction: the requested behavior is correct under the docs, but the safest fix needs maintainer choice on when to query Graph without an attachment hint so ordinary mention-only messages do not create noisy or expensive Graph traffic.
