## Summary

Keep open. Current main and v2026.5.7 still lack non-interactive trusted-proxy onboarding; runtime auth, interactive onboarding, and docs already support trusted-proxy, and https://github.com/openclaw/openclaw/pull/49107 is the active same-author draft implementation path.

## What This Changes

Add non-interactive onboard CLI support for selecting and configuring trusted-proxy gateway auth behind reverse proxies.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/49107 with a reviewed core onboarding change that writes the same trusted-proxy config shape as interactive onboarding and updates focused tests, docs, and changelog as needed.

## Reproduction Assessment

Yes. A high-confidence source-level path is `openclaw onboard --non-interactive --gateway-auth trusted-proxy --accept-risk`, which reaches applyNonInteractiveGatewayConfig and is rejected by the token/password-only validation.

## Solution Assessment

Yes on direction, but not by closing this issue now. Core onboarding parity is the maintainable fix because trusted-proxy already exists as a core runtime and interactive mode, while the open draft PR still needs review and proof.
