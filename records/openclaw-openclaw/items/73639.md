## Summary

Keep open. Current main and the latest release still accept only token/password in non-interactive onboarding, while trusted-proxy is already a core runtime and interactive gateway auth mode; draft PR #49107 is the active same-author implementation path and needs maintainer/security review.

## What This Changes

Add non-interactive onboarding support for configuring `gateway.auth.mode = "trusted-proxy"` and its proxy/user-header fields.

## Best Possible Solution

Review or land a narrow core onboarding change with typed trusted-proxy non-interactive flags, validation, docs, tests, and a changelog entry while preserving the existing gateway trust model.

## Reproduction Assessment

Yes. A source-level reproduction is `openclaw onboard --non-interactive --gateway-auth trusted-proxy --accept-risk`; current main reaches `applyNonInteractiveGatewayConfig` and rejects the mode before it can write trusted-proxy config.

## Solution Assessment

Yes. Core onboarding parity is the maintainable direction because trusted-proxy is already a core runtime and interactive gateway auth mode; manual config edits or a plugin would bypass the typed onboarding contract.
