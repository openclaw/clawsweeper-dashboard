## Summary

Keep open. Current main still lets an omitted message-tool `accountId` resolve from inherited/default account context before the calling agent binding is consulted, and the focused implementation PR was closed unmerged.

## What This Changes

The issue requests that omitted Slack `message` tool `accountId` values default to the calling agent's configured channel account instead of an inherited/default account.

## Best Possible Solution

Implement the shared precedence as explicit `accountId` first, calling-agent channel binding second, inherited/default account last, with SecretRef scoping using the same selected account.

## Reproduction Assessment

Yes, at source level. With omitted `params.accountId`, `defaultAccountId: "parent-account"`, `agentId: "agent-b"`, and an `agent-b` binding to `account-b`, current main chooses the default before the binding lookup can run.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the shared message-tool/outbound account-selection path, not in Slack-specific delivery code, and should reuse the existing binding helper.
