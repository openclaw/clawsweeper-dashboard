## Summary

Keep open. Current main still resolves omitted message-tool accounts through the inherited/default account before the calling agent’s binding, and #74052 is the active open closing PR for this behavior.

## What This Changes

Request that omitted `message` tool `accountId` values prefer the calling agent’s configured channel account while preserving explicit overrides.

## Best Possible Solution

Review and land #74052, or an equivalent narrow outbound-runner change, so explicit accounts win, omitted accounts prefer the calling agent binding, and inherited/default context is last fallback.

## Reproduction Assessment

Yes. A focused `runMessageAction` regression can configure `defaultAccountId: "ops"` plus an `agent-b` binding to `account-b`, omit `params.accountId`, and observe current main choose the default before checking the binding.

## Solution Assessment

Yes. The linked PR direction is the narrow maintainable fix because the precedence bug sits in the shared message-tool wrapper/runner path rather than Slack delivery code.
