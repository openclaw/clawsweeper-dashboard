## Summary

Keep open. Current main and the latest release fix the config-write preflight half, but the default Windows file SecretRef path still uses `@openclaw/fs-safe@0.3.0`, whose Windows ACL inspector invokes `icacls.exe <path> /sid` and fails closed when ACL inspection is unavailable.

## What This Changes

Fix Windows file SecretRef ACL verification so supported `icacls` behavior works by default and `allowInsecurePath` is honored consistently during config writes.

## Best Possible Solution

Repair fs-safe Windows ACL inspection with supported `icacls` syntax or a narrow exit-87 fallback, bump the OpenClaw dependency, keep the bypass explicit, and prove both `secrets audit` and `config.patch` on Windows.

## Reproduction Assessment

Yes for a high-confidence source-level reproduction path, but not for a live Windows run in this review. Default file SecretRefs call fs-safe ACL inspection, the pinned package invokes `icacls /sid`, and unavailable ACL verification fails closed unless `allowInsecurePath` bypasses it.

## Solution Assessment

No, closing is not the best way to solve this issue. Current main covers the config-write preflight side, but the default Windows ACL path still needs a security-reviewed fs-safe repair and Windows proof.
