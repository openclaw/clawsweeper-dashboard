## Summary

Keep open for normal maintainer review: this is a fresh, focused bug-fix PR, current main still has the source-level fallback gap, and I did not find a blocking correctness or security issue in the patch.

## What This Changes

The PR strips standalone trailing `notify=false` markers from heartbeat fallback replies, forwards them as durable silent delivery, and adds heartbeat/exec regression coverage plus test-helper forwarding.

## Best Possible Solution

Merge the scoped fallback normalizer after normal maintainer and CI gates, keeping structured `heartbeat_respond` metadata canonical and using the existing durable `silent` delivery contract.

## Reproduction Assessment

Yes, at the source level: current main strips heartbeat tokens but not a trailing standalone `notify=false`, then sends the fallback batch without `silent`. That matches the linked report at https://github.com/openclaw/openclaw/issues/80569, though I did not run a live current-main Telegram repro.

## Solution Assessment

Yes. Parsing only a standalone trailing marker in the heartbeat fallback path and forwarding the existing durable `silent` option is narrower than global Telegram text parsing or a new delivery-control surface.
