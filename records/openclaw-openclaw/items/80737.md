## Summary

Keep open: this is the canonical PR for the linked Telegram heartbeat leak, but the proposed parser still misses a common terminal-newline form of the marker, so the fix can still leak visible `notify=false` text.

## What This Changes

The PR strips standalone trailing `notify=false` markers from heartbeat fallback replies, forwards silent delivery metadata, and adds heartbeat/exec regression coverage.

## Best Possible Solution

Tighten the heartbeat fallback marker parser to accept trailing line endings after a standalone final `notify=false`, add regression coverage for that form, then land this canonical fix for the linked Telegram bug.

## Reproduction Assessment

Yes. Current-main source shows fallback replies are sent without consuming `notify=false`, and a focused regex check shows the PR still misses `notify=false` followed by a final newline; I did not run a live current-main Telegram reproduction.

## Solution Assessment

Not yet. The owner-boundary approach is right, but the parser should tolerate terminal line endings so common model output cannot leak the marker before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Allow trailing line endings after notify=false:** `src/infra/heartbeat-runner.ts:738`
