The PR strips standalone trailing `notify=false` markers from heartbeat fallback replies, converts them to `silent: true`, forwards that through durable delivery, and adds focused regression tests.

## Best Possible Solution

Merge the scoped heartbeat fallback normalizer after normal maintainer and CI gates, keeping structured `heartbeat_respond` metadata as canonical and using the existing durable delivery silent contract.

## Solution Assessment

Yes. Parsing only a standalone trailing marker in the heartbeat fallback path and forwarding existing `silent` metadata is narrower than changing global Telegram text handling or adding a new delivery-control surface.
