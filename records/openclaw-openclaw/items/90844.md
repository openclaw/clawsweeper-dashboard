## Summary

Keep open: current main and the latest release still export only the OpenClaw transcript without a backend-delegated warning, and the linked open PR is the active fix path.

## What This Changes

The issue reports that `/export-session` exports only user messages for CLI/ACP-backed sessions, leaving assistant replies, tool calls, usage, and cost out of the HTML export.

## Best Possible Solution

Land a narrow warning in the command reply and exported HTML after real CLI/ACP proof, then close this issue; treat full backend transcript merging as separate product work if desired.

## Reproduction Assessment

Yes, source-reproducible: a CLI/ACP session whose OpenClaw transcript has only user entries will export only those entries because the command reads only the OpenClaw transcript file. I did not run a live CLI/ACP export in this review.

## Solution Assessment

Yes for the minimum-fix path: a clear warning in the command reply and HTML export is narrower and safer than backend-specific transcript merging. Full merging can be considered separately after maintainers define the cross-backend contract.
