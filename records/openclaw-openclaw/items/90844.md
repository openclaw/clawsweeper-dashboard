## Summary

Keep this open: current `main` and the latest release still do not warn when `/export-session` is built from a user-only backend-delegated transcript, and the linked open PR is the active fix path rather than a reason to close this issue early.

## What This Changes

Request that `/export-session` include CLI/ACP backend assistant/tool/usage data or at least warn when the exported OpenClaw transcript only contains user messages.

## Best Possible Solution

Review and land the linked warning mitigation, then track full backend-transcript merging separately if maintainers want richer tool/usage export.

## Reproduction Assessment

Yes, source-reproducible: a user-only transcript loaded by `buildExportSessionReply` is exported without a backend-delegated warning on current main. I did not run a live CLI/ACP session in this read-only review.

## Solution Assessment

Yes for the minimum fix: warning in the command reply and exported HTML is the narrowest maintainable mitigation. Full backend transcript merging is a larger enhancement that can be tracked separately.
