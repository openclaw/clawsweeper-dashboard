## Summary

Current main still exports only the resolved OpenClaw transcript and lacks a backend-delegated warning, while the linked closing PR remains open and unmerged; this issue should stay open as the canonical tracker until that PR lands or maintainers choose a different fix.

## What This Changes

The issue requests that `/export-session` stop producing misleading user-only exports for CLI/ACP backend sessions, either by merging backend transcript data or warning that assistant/tool/usage data are not included.

## Best Possible Solution

Review and either land or explicitly reject https://github.com/openclaw/openclaw/pull/90867; close this issue only after the accepted warning or transcript-merge fix reaches main.

## Reproduction Assessment

Yes from source inspection. A transcript whose message entries are all `role: user` will be exported as-is by `buildExportSessionReply`, and the HTML summary computes assistant/tool/model/usage stats only from those exported entries.

## Solution Assessment

Yes for the minimum fix path. A visible command and HTML warning is the narrowest maintainable mitigation while full backend transcript merging remains a larger follow-up.
