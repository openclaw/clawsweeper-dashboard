## Summary

Keep open. Current main still exports exactly the OpenClaw transcript entries and the HTML summary derives assistant/model/tool/usage data only from those entries, so a user-only CLI/ACP transcript remains a misleading user-only export. The linked fix PR is still open with closing syntax, so this issue should wait for that PR to land or be replaced.

## What This Changes

The issue reports that `/export-session` on CLI/ACP-backed sessions can produce an HTML export containing only user messages, leaving assistant replies, tool calls, model, token, and cost summaries empty.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/90867 for the warning mitigation, then track full backend transcript merging separately if maintainers want richer exports.

## Reproduction Assessment

Yes, source-reproducible: feed the current exporter an OpenClaw transcript whose message entries are all `role: user`, and the command will export those entries without a backend-delegated warning while the template computes zero assistant/tool/usage stats. I did not run a live CLI/ACP backend reproduction in this read-only review.

## Solution Assessment

No current-main fix is present for the reported misleading export. The linked warning PR is the narrowest practical mitigation, while full backend transcript merging is a broader product/design follow-up.
