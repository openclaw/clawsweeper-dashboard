## Summary

Current main still has the reported gap: `/export-session` serializes only the OpenClaw transcript and has no backend-delegated warning, while the linked candidate fix is still open and unmerged. This issue should remain the canonical tracker until that PR lands or maintainers choose the larger backend-transcript merge path.

## What This Changes

The issue asks for CLI/ACP backend-delegated session exports to include assistant/tool/usage data from the backend transcript, or at least warn clearly when only OpenClaw user messages are exported.

## Best Possible Solution

Review and land or explicitly reject https://github.com/openclaw/openclaw/pull/90867; close this issue only after an accepted warning or backend-transcript merge fix reaches main.

Yes, from source inspection. A transcript whose message entries are all `role: user` will be exported as-is by `buildExportSessionReply`, and the HTML summary computes assistant/tool/model/usage stats only from those exported entries.

Yes for the minimum-fix path. A visible command and HTML warning is the narrowest maintainable mitigation while full backend transcript merging remains a larger follow-up.
