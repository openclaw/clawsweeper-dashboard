## Summary

Keep open: the remaining `/export-session` backend-transcript limitation is not fully implemented on current `main`, and https://github.com/openclaw/openclaw/pull/90867 is an open closing PR, so this issue should stay open until that PR lands or is replaced.

## What This Changes

The issue reports that CLI/ACP-backed `/export-session` exports can contain only OpenClaw-local user entries, leaving assistant replies, tool calls, model, usage, and cost absent or unwarned.

## Best Possible Solution

Review and repair https://github.com/openclaw/openclaw/pull/90867 if the warning mitigation is acceptable; otherwise replace it with a backend-transcript merge that keeps export ownership generic.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path for the remaining limitation: the exporter serializes only OpenClaw-local transcript entries, while CLI backend history can live in an external Claude transcript. I did not run a live CLI/ACP backend session in this read-only review.

## Solution Assessment

Yes for the minimum fix path: a clear warning in the command reply and exported HTML is the narrowest maintainable mitigation. A full backend transcript merge would be more complete but needs tighter ownership and validation across backend types.
