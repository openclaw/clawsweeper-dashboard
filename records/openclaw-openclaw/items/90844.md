## Summary

Current main still has no backend-delegated warning or backend transcript merge for `/export-session`; it only exports the resolved OpenClaw transcript. The linked open PR is the active fix path, so this issue should stay open until that PR lands or maintainers choose a fuller transcript-import design.

## What This Changes

The issue asks for `/export-session` on CLI/ACP-backed sessions to include backend assistant/tool/usage data or warn clearly when the export only contains OpenClaw’s partial transcript.

## Best Possible Solution

Land a reviewed warning mitigation now, then track full backend-transcript import separately only if maintainers want richer CLI/ACP export fidelity.

## Reproduction Assessment

Yes, source inspection gives a medium-confidence reproduction path: use a CLI/ACP-backed session whose OpenClaw transcript lacks backend-native assistant/tool/usage rows, then run `/export-session`; the exporter only serializes that OpenClaw transcript. I did not run a live Claude CLI/ACP session in this read-only review.

## Solution Assessment

Unclear as a product choice. The linked warning PR is the narrowest mitigation for misleading exports, while a full backend transcript merge would be more complete but requires backend-specific ownership and mapping decisions.
