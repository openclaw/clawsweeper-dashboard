## Summary

Keep open: current main has useful Codex-specific and model-suppression guardrails, but it still does not validate active agent model refs, stale `agents.*.models` runtime-policy keys, enabled plugin/provider availability, and config save/reload acceptance as one consistency check.

## What This Changes

The issue requests config-save, reload, and/or doctor validation for contradictory agent model refs, runtime-policy declarations, and disabled plugin/provider paths.

## Best Possible Solution

Add a shared model-route consistency analyzer surfaced through doctor and config validation warnings, covering active refs, disabled provider/runtime plugins, stale runtime-policy keys, and missing explicit runtime policy without changing runtime fallback semantics by surprise.

## Reproduction Assessment

No, not as a high-confidence live repro: this read-only review did not run the gateway or config writer. Source inspection shows a medium-confidence path where current validation only checks suppressed model refs while broader plugin/runtime consistency is deferred until later routing.

## Solution Assessment

Yes for the warning/lint direction: a shared analyzer reused by config validation and doctor is the narrow maintainable fix. Refusing saves outright should be treated as a follow-up maintainer policy choice if warnings are not enough.
