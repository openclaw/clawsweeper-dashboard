## Summary

Keep open: current main has partial doctor coverage for disabled Codex routes, but the config save/read/reload validation paths still appear to rely on schema/plugin validation and do not run the cross-section model-runtime-plugin consistency checks described here.

## What This Changes

The issue asks OpenClaw to catch contradictory config edits that leave active agent model references, runtime policy declarations, and enabled plugin availability out of sync.

## Best Possible Solution

Add a shared model-route consistency check that can be reused by config write/read warnings, hot reload diagnostics, and doctor lint, with hard failures reserved for clearly invalid active refs and compatibility-safe warnings for upgrade-sensitive cases.

## Reproduction Assessment

No high-confidence executed reproduction was established in this read-only pass. Source inspection shows the reported shape remains plausible because save/read paths use schema/plugin validation while the existing Codex route consistency warning is only wired into doctor preview warnings.

## Solution Assessment
