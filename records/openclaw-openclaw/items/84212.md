## Summary

Keep open: current main has partial Codex-specific doctor warnings and repair, but still lacks a general validation path that ties active agent model refs, runtime-policy keys, enabled provider/plugins, and save/reload acceptance together.

## What This Changes

The issue asks OpenClaw to warn or reject contradictory config edits where agent model refs, runtime declarations, and plugin/provider availability no longer resolve together.

## Best Possible Solution

Add a shared model-route consistency analyzer used by doctor and config validation warnings, covering active refs, disabled provider/runtime plugins, stale runtime-policy keys, and missing explicit runtime policy without changing runtime fallback behavior by surprise.

## Reproduction Assessment

No, not as a high-confidence live repro: this read-only review did not run the gateway or config writer. Source inspection does show the current validator and doctor lint paths still miss the requested cross-section consistency check.

## Solution Assessment

Yes for the warning/lint direction: a shared analyzer reused by doctor and config validation is the narrow maintainable fix. Hard-failing saves or reloads should wait for maintainer policy on catalog authority and compatibility.
