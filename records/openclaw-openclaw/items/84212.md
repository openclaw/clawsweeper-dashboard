## Summary

Keep open: current main has schema validation and warning-level doctor coverage for one disabled-Codex-plugin route mismatch, but it still does not block or lint the full active model-reference/runtime-policy/plugin-availability inconsistency at config persistence or reload time.

## What This Changes

The issue asks OpenClaw to validate plugin availability, runtime declarations, and agent model references together so config edits cannot leave agents with unresolvable model routing.

## Best Possible Solution

Add a maintainer-approved cross-section validator that rejects or clearly blocks unresolvable active model refs at config write/reload time while keeping doctor repair guidance for recoverable existing configs.
