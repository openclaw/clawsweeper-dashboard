## Summary

Keep open: current main and v2026.6.8 still provide only narrow Codex-disabled-route warning/repair coverage, while config write and reload paths do not run the requested active model-ref, runtime-policy, provider, and plugin consistency gate.

## What This Changes

The issue asks OpenClaw to reject or lint persisted config edits whose agent model references, runtime declarations, and enabled plugins/providers are mutually inconsistent.

## Best Possible Solution

Add one shared model-route consistency analyzer used by doctor, config persistence, and reload, with maintainer-approved rules for which mismatches hard-fail, warn, or repair.

## Reproduction Assessment

Yes, source-reproducible: current validation, write, and reload paths accept schema-valid configs after plugin validation and SecretRef checks, while doctor only warns for a narrow disabled-Codex subset.

## Solution Assessment

Keeping this open is the best outcome: the merged doctor warning subset is useful but does not stop contradictory persisted config from becoming active, and the save/reload hard-fail severity needs maintainer policy.
