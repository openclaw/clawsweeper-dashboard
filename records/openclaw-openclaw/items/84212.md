## Summary

Keep open: current main has Codex-specific doctor repair/warnings and static model-suppression validation, but not the broader consistency check that ties active model refs, stale runtime-policy keys, and enabled plugin/provider availability together at config save or reload time.

## What This Changes

The issue asks OpenClaw to warn on or reject contradictory config edits where agent model refs, runtime policy declarations, and enabled plugin/provider availability no longer resolve together.

## Best Possible Solution

Add a shared model-route consistency analyzer surfaced through doctor and config validation warnings, covering active refs, disabled provider/runtime plugins, stale runtime-policy keys, and missing explicit runtime policy without changing runtime fallback semantics by surprise.
