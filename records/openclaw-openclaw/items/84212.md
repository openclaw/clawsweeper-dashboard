## What This Changes

The issue asks OpenClaw to warn or reject persisted config edits that make active agent model refs, runtime policy declarations, and enabled plugin/provider availability inconsistent.

## Best Possible Solution

Add a shared model-route consistency analyzer surfaced through doctor and config validation warnings, covering active refs, disabled provider/runtime plugins, stale runtime-policy keys, and missing explicit runtime policy without changing runtime fallback semantics by surprise.

## Solution Assessment

Yes for the warning/lint direction: a shared analyzer reused by config validation and doctor is the narrow maintainable fix. Refusing saves outright should be treated as a follow-up maintainer policy choice if warnings are not enough.
