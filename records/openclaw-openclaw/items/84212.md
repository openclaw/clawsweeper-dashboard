## Summary

Keep open: current main has partial Codex-specific guardrails and suppressed-model validation, but it still lacks a general consistency check that ties active agent model refs, runtime-policy keys, enabled plugins/providers, and save/reload acceptance together. The related open deep-config PR is not merged and does not close this issue.

## What This Changes

The issue asks OpenClaw to validate active agent model refs, runtime-policy declarations, and plugin/provider availability together during config save/reload or doctor lint so contradictory edits do not fail later at routing time.

## Best Possible Solution

Add a shared model-route consistency analyzer surfaced through doctor/config validation warnings, covering active refs, disabled provider/runtime plugins, stale runtime-policy keys, and missing explicit runtime policy without changing runtime fallback semantics by surprise.

## Reproduction Assessment

No, not as a high-confidence live repro: this read-only review did not run the gateway or config writer. Source inspection does show a medium-confidence path where current validation only checks suppressed model refs while broader plugin/runtime consistency is deferred until later routing.

## Solution Assessment

Yes for the warning/lint direction: a shared analyzer reused by config validation and doctor is the narrow maintainable fix. Refusing saves outright should remain a maintainer policy choice if warnings are not enough.
