## Summary

Keep open: current main has Codex-specific doctor warnings and suppression-only config validation, but it still lacks a shared consistency check tying active agent model refs, stale `agents.*.models` runtime policy keys, and enabled plugin/runtime availability together.

## What This Changes

The issue asks OpenClaw to warn or validate when config edits leave agent model refs, runtime policy declarations, and plugin/provider availability mutually inconsistent.

## Best Possible Solution

Add a shared model-route consistency analyzer surfaced through doctor and config warnings, covering active refs, disabled provider/runtime plugins, stale runtime-policy keys, and missing explicit runtime policy without changing runtime fallback behavior by surprise.
