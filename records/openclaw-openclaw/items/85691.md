## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve XML escaping when repairing prompt paths:** `src/commands/doctor-session-snapshots.ts:330-344`
  - body: `extractSkillLocations()` decodes XML entities before producing `finding.cachedPath`, but the persisted prompt stores XML-escaped `<location>` values. If a stale runtime path contains an escaped character such as `&`, this replacement never matches the raw `sessions.json`, so `doctor --fix` leaves that stale prompt path behind. Re-escape prompt-location paths, or repair the parsed prompt and serialize it, and add coverage for an escaped path.
  - confidence: 0.86
- **[P2] Handle Windows baseDir suffixes:** `src/commands/doctor-session-snapshots.ts:350-352`
  - body: For `resolvedSkills.baseDir`, the scanner synthesizes a `...SKILL.md` path with the platform separator. On Windows that finding ends in `\SKILL.md`, so this `/SKILL.md` guard is false and the stored `baseDir` directory is never rewritten, leaving a stale directory even when the matching `filePath` is repaired. Use a platform-neutral suffix check or path parsing for the baseDir repair.
  - confidence: 0.91
- **[P2] Write repaired session stores atomically:** `src/commands/doctor-session-snapshots.ts:371-372`
  - body: This direct `writeFileSync` bypasses the existing session-store atomic write path and writes the only `sessions.json` copy in place. A failed or interrupted doctor repair can leave persisted session metadata truncated or partially written; use the repository's atomic writer/session-store save path or an equivalent validated atomic raw-file write before replacing the store.
  - confidence: 0.78
