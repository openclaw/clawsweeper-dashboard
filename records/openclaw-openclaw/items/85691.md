## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve XML-escaped prompt paths:** `src/commands/doctor-session-snapshots.ts:331-345`
  - body: The repair searches for `JSON.stringify(finding.cachedPath)`, but `finding.cachedPath` has already been XML-decoded by the scanner. Prompt `<location>` values are written with XML escaping, so a stale path containing `&`, `<`, quotes, or apostrophes is reported but not replaced in the raw JSON. Match the XML-escaped prompt form or update the parsed store and reserialize it through the store writer.
  - confidence: 0.9
- **[P2] Handle Windows baseDir separators:** `src/commands/doctor-session-snapshots.ts:350-364`
  - body: This `baseDir` branch only runs when the collected path ends with `/SKILL.md`. On Windows the collected path is `...\SKILL.md`, so the directory replacement is skipped and a legacy `resolvedSkills[].baseDir` can remain pointed at the stale runtime root after `--fix`. Use separator-aware handling for both POSIX and Windows paths.
  - confidence: 0.88
- **[P2] Use the session-store writer for repairs:** `src/commands/doctor-session-snapshots.ts:371-372`
  - body: The new repair writes `sessions.json` directly with `fs.writeFileSync`, bypassing the existing session-store writer's lock, atomic write path, Windows retry behavior, cache updates, and normalization. Since this is a `doctor --fix` mutation of persisted session state, the repair should go through `updateSessionStore`/`saveSessionStore` or a narrow helper with the same persistence guarantees.
  - confidence: 0.86
