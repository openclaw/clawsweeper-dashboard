## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Avoid rewriting the hydrated scanner store:** `src/commands/doctor-session-snapshots.ts:462-490`
  - body: `loadSessionStoreForSnapshotScan()` mutates every entry through promptRef hydration, and that helper deletes `skillsSnapshot` when a promptRef blob is missing or invalid. If any other session in the same store is repairable, this block serializes the whole hydrated store, so `doctor --fix` can persist unrelated snapshot loss. Use a targeted/raw-preserving write path and add a regression with an unrelated invalid promptRef session.
  - confidence: 0.9
- **[P1] Use the session-store write queue for repairs:** `src/commands/doctor-session-snapshots.ts:489-490`
  - body: This repair reads and writes `sessions.json` directly instead of going through the existing queued session-store writer. Atomic replace prevents partial files, but it does not prevent a gateway/session update written after this read from being overwritten by the later doctor write.
  - confidence: 0.82
