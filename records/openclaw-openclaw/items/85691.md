- **[P1] Route repairs through the session-store writer:** `src/commands/doctor-session-snapshots.ts:397`
  - body: This writes `sessions.json` directly with `writeTextAtomic`, bypassing the session-store writer path that serializes store writes, updates session caches, and uses the session-store temp-prefix contract. Because this repair can run under `doctor --fix` after update, a same-process cached writer or later save can overwrite the repair or leave non-session temp artifacts; route the rewrite through a session-store-safe helper/writer path before merging.
  - confidence: 0.86

  - body: After any replacement changes the raw file, the function reports success and returns without reloading the store to confirm that all stale findings disappeared. If one escaping form or field is not actually rewritten, `doctor --fix` will hide the remaining stale session snapshot warning; rescan the affected stores and report any leftovers before returning success.
  - confidence: 0.82
