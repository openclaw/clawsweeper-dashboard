## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Limit repairs to snapshot metadata fields:** `src/commands/doctor-session-snapshots.ts:362-364`
  - body: This replacement runs over the entire raw `sessions.json` string for every stale finding. `SessionEntry` also persists plugin-owned and delivery state, so a matching stale runtime path outside `skillsSnapshot` or `systemPromptReport` would be silently rewritten even though it was not part of the validated snapshot metadata. Build the repaired JSON by mutating only the matched metadata fields, or otherwise scope replacements to those fields, before writing.
  - confidence: 0.86
- **[P3] Use an XML-sensitive fixture in the XML proof:** `repro-doctor-session-snapshot-repair.mjs:277-282`
  - body: Scenario 4 says it covers `&amp;`, but `xmlStaleRoot` contains no `&`, `<`, `>`, quote, or apostrophe, so the entity-escaping branch is not actually exercised. Use a path with a real XML-sensitive character and assert the raw stored prompt contains the entity text before repair.
  - confidence: 0.93
