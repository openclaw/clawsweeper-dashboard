Full review comments:

- **[P1] Route snapshot repair through the session writer:** `src/commands/doctor-session-snapshots.ts:501`
  - body: This direct `writeTextAtomic` update bypasses the session-store writer and its cache refresh path. In a doctor/update process that already warmed the session store, or has a queued store write, a later canonical save can serialize older cached state and hide or overwrite the repaired snapshot metadata. Please route the repair through the session-store writer/cache contract, or a narrow helper owned there, and cover a warmed-cache or queued-writer case.
  - confidence: 0.88

  - body: This root proof script copies the scanner and repair algorithm, so it can drift from production while still appearing to prove the fix. The proof should stay in the PR body/artifacts, and the durable coverage should be the colocated Vitest cases that exercise the real implementation.
  - confidence: 0.9
