Full review comments:

- **[P1] Route snapshot repair through the session writer:** `src/commands/doctor-session-snapshots.ts:497-501`
  - body: This repair writes `sessions.json` directly, but normal session persistence is serialized through `runExclusiveSessionStoreWrite` and refreshes writer-owned caches after the atomic replace. A warmed same-process cache or pending queued write can make doctor report a repair that is later hidden or overwritten, so route this through the session-store writer/cache path and add a regression for that case.
  - confidence: 0.89
- **[P2] Remove the one-off root proof script:** `repro-doctor-session-snapshot-repair.mjs:1-10`
  - body: The root script reimplements the scanner and repair logic instead of exercising production doctor code, and its header already describes an older proof path. Keeping it in the repo can drift from the real implementation and provide a passing proof command after production behavior regresses; keep this coverage in colocated tests or PR proof instead.
  - confidence: 0.84
