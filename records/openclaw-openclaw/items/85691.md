## Review Findings

Overall confidence: 0.9

Full review comments:

- **[P2] Handle XML-escaped prompt locations:** `src/commands/doctor-session-snapshots.ts:331-345`
  - body: `extractSkillLocations` decodes XML entities before producing `finding.cachedPath`, but the raw `sessions.json` prompt still contains escaped values such as `&amp;`. Searching only the decoded JSON/raw path forms means `doctor --fix` leaves those stale prompt paths behind when the runtime path contains XML-escaped characters.
  - confidence: 0.89
- **[P2] Handle Windows baseDir snapshot paths:** `src/commands/doctor-session-snapshots.ts:350`
  - body: For `resolvedSkills.baseDir`, the scanner synthesizes a `SKILL.md` path with platform `path.join`. On Windows that synthetic path ends in `\SKILL.md`, so this `/SKILL.md` guard is false and the raw `baseDir` field is never repaired even though the PR claims Windows path support.
  - confidence: 0.9
- **[P2] Use the session-store write path for repairs:** `src/commands/doctor-session-snapshots.ts:371-372`
  - body: This writes `sessions.json` directly after a raw string replacement, bypassing the normal atomic write and cache-update/invalidation path used by `saveSessionStore` and `updateSessionStore`. If the doctor process has session-store caches or a write fails mid-file, a later store write can undo the repair or leave corrupted session state.
  - confidence: 0.78
