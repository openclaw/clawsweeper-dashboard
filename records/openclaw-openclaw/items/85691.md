Full review comments:

- **[P1] Route the rewrite through the session-store writer:** `src/commands/doctor-session-snapshots.ts:397`
  - body: This writes `sessions.json` with `writeTextAtomic` after a raw read, bypassing `runExclusiveSessionStoreWrite`/`updateSessionStore` and the session-store caches. Existing session-store repair paths hold the writer and refresh caches; this direct rewrite can race with Gateway or another doctor repair and overwrite or be overwritten by queued session mutations.
  - confidence: 0.88

  - body: The function returns after any store changed. If one encoded form is missed, the remaining stale findings are hidden from the normal warning and the user sees only a success note, so reload and rescan after writes before suppressing the warning.
  - confidence: 0.84

Move these scenarios into `src/commands/doctor-session-snapshots.test.ts` so CI covers the real doctor path, then remove the root proof artifact.
