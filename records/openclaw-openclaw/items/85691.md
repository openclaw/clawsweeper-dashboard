## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Serialize repairs through the session-store writer:** `src/commands/doctor-session-snapshots.ts:372`
  - body: This writes `sessions.json` from a raw snapshot outside `runExclusiveSessionStoreWrite`/`saveSessionStore`. A queued session update can overwrite the repair or be overwritten by it, and this also bypasses the atomic write/cache update path used by normal session-store mutations. Put the repair under the existing writer and save path.
  - confidence: 0.9
- **[P2] Preserve XML-escaped prompt locations:** `src/commands/doctor-session-snapshots.ts:331-344`
  - body: The scanner decodes `<location>` text before producing `finding.cachedPath`, but skill prompt formatters store paths with XML entities. If a stale path contains characters like `&`, the raw JSON contains `&amp;`, so neither the JSON-escaped nor raw decoded replacement matches and `doctor --fix` leaves that prompt stale. Replace the XML-escaped forms too and add a regression test.
  - confidence: 0.88
