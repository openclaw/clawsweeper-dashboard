## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Replace XML-escaped prompt paths:** `src/commands/doctor-session-snapshots.ts:331-332`
  - body: `extractSkillLocations` decodes XML entities before producing `finding.cachedPath`, but persisted skill prompts are written with `escapeXml(skill.filePath)`. For paths containing `&`, `<`, `>`, quotes, or apostrophes, the raw JSON contains the XML-escaped form, so these replacements never touch the stale prompt path and `doctor --fix` can leave the session broken.
  - confidence: 0.91
- **[P2] Write session repairs through the store writer:** `src/commands/doctor-session-snapshots.ts:372`
  - body: This direct overwrite bypasses the session-store writer and atomic `writeTextAtomic` path used by normal `sessions.json` saves. A doctor repair running around active session writes can expose a partial file or stale in-process session-store state, so route the repair through an equivalent exclusive atomic writer and cache update/invalidation path.
  - confidence: 0.84
