## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve XML-escaped prompt paths:** `src/commands/doctor-session-snapshots.ts:331-344`
  - body: `extractSkillLocations` decodes XML entities before producing `finding.cachedPath`, but this repair only searches the raw JSON for the decoded path. A prompt location like `.../a&amp;b/SKILL.md` is reported as stale, then `--fix` cannot find and rewrite it, leaving the session broken.
  - confidence: 0.87
- **[P2] Handle Windows baseDir separators:** `src/commands/doctor-session-snapshots.ts:350-352`
  - body: For Windows `resolvedSkills[].baseDir`, `collectResolvedSkillPaths` appends `\SKILL.md`, but this block only recognizes `"/SKILL.md"`. That means the file path can be repaired while the stored `baseDir` remains pointed at the old runtime root on Windows.
  - confidence: 0.82
- **[P2] Use the session-store writer for repairs:** `src/commands/doctor-session-snapshots.ts:371-372`
  - body: This writes the live `sessions.json` with plain `fs.writeFileSync`, bypassing the existing session-store writer/atomic write path and cache update. A doctor repair can expose readers to a partial file or be overwritten by later session-store saves that did not see the repair.
  - confidence: 0.8
