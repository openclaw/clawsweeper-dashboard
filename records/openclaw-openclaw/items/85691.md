## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve XML entities when replacing prompt paths:** `src/commands/doctor-session-snapshots.ts:331-332`
  - body: `skillsSnapshot.prompt` stores `<location>` values through XML escaping, but `extractSkillLocations` decodes entities before producing `finding.cachedPath`. For a stale path containing `&`, `<`, `>`, quotes, or apostrophes, the raw JSON contains `&amp;` or similar text, so this replacement misses it and `doctor --fix` leaves the stale prompt path behind.
  - confidence: 0.91
- **[P2] Handle Windows separators when repairing baseDir:** `src/commands/doctor-session-snapshots.ts:350`
  - body: On Windows, the `resolvedSkills` baseDir finding can be synthesized with `\SKILL.md`, but this branch only recognizes `/SKILL.md`. That leaves `resolvedSkills[].baseDir` pointed at the old runtime root even after the file path is repaired.
  - confidence: 0.89
- **[P2] Use an atomic session-store write path:** `src/commands/doctor-session-snapshots.ts:371-372`
  - body: This reads and overwrites the active `sessions.json` with `fs.writeFileSync`, bypassing the session-store writer queue and atomic write path used by normal saves. An interrupted doctor run or concurrent writer can truncate or clobber persistent session metadata.
  - confidence: 0.86
