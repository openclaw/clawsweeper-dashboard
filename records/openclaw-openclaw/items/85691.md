## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve XML-escaped prompt paths:** `src/commands/doctor-session-snapshots.ts:331-344`
  - body: `extractSkillLocations` decodes XML entities before producing `finding.cachedPath`, but this repair only searches the raw JSON for the decoded path and its JSON-escaped form. A prompt location stored as XML text with `&amp;`, `&lt;`, quotes, or apostrophes can still be detected as stale but not rewritten by `doctor --fix`.
  - confidence: 0.84
- **[P2] Handle Windows baseDir suffixes:** `src/commands/doctor-session-snapshots.ts:350`
  - body: On Windows, `collectResolvedSkillPaths` can synthesize the baseDir probe with `\SKILL.md`, but this branch only strips the directory prefix when the finding ends with `/SKILL.md`. That means Windows `resolvedSkills.baseDir` values can remain pointed at the stale runtime root even though the filePath was repaired.
  - confidence: 0.82
- **[P2] Use the session-store writer for repairs:** `src/commands/doctor-session-snapshots.ts:371-372`
  - body: Normal session-store updates go through `runExclusiveSessionStoreWrite`, atomic text writes, and cache updates, but this repair writes `sessions.json` directly. In a doctor/update process that has already loaded the store, or overlaps another session-store write, the raw rewrite can leave stale in-memory cache or be overwritten by the canonical writer.
  - confidence: 0.8
