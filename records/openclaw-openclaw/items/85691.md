## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Route snapshot repairs through the session writer:** `src/commands/doctor-session-snapshots.ts:397`
  - body: This writes the repaired `sessions.json` text directly with `writeTextAtomic` after an earlier raw read, outside the `updateSessionStore`/`runExclusiveSessionStoreWrite` path used for session metadata. If another writer-owned update lands between the read and this write, this repair can publish stale JSON over newer session state; move the repair under a writer-owned helper that reloads before saving and updates or invalidates caches.
  - confidence: 0.88
- **[P3] Exercise a real XML-escaped path:** `repro-doctor-session-snapshot-repair.mjs:277-282`
  - body: Scenario 4 says it tests a path with `&`, but `xmlStaleRoot` contains no XML-escaped character, so the test would pass even if the XML replacement branch were removed. Use a stale path containing `&`, `<`, or another escaped character, and preferably cover `noteSessionSnapshotHealth({ shouldRepair: true })` directly.
  - confidence: 0.9
