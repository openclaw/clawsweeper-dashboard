## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Serialize repairs through the session-store writer:** `src/commands/doctor-session-snapshots.ts:371-372`
  - body: `doctor --fix` now reads a snapshot of `sessions.json` and writes the repaired text back with `fs.writeFileSync`. Normal session updates go through `runExclusiveSessionStoreWrite` plus the atomic writer/cache update path, so this can clobber a concurrent save or leave the process with stale session-store caches. Route the repair through the session-store writer or an equivalent locked atomic helper.
  - confidence: 0.9
- **[P2] Repair XML-escaped prompt locations:** `src/commands/doctor-session-snapshots.ts:331-344`
  - body: `extractSkillLocations` decodes `&amp;`, `&lt;`, and related XML entities before creating `finding.cachedPath`, but this repair only searches the raw JSON text for the decoded path or its JSON-escaped form. A stale `<location>` containing XML-escaped characters will be reported but not fixed by `--fix`. Add coverage and repair the serialized XML form or update the parsed prompt value before writing.
  - confidence: 0.84
- **[P2] Handle Windows baseDir suffixes:** `src/commands/doctor-session-snapshots.ts:350-352`
  - body: On Windows, `collectResolvedSkillPaths` appends `SKILL.md` with `path.join`, so the synthetic `baseDir` finding can end in `\SKILL.md`. This condition only checks `"/SKILL.md"`, so the directory-prefix replacement is skipped and stale `resolvedSkills.baseDir` values remain. Use platform-agnostic suffix handling for both separators.
  - confidence: 0.82
