## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve XML escaping when repairing prompt paths:** `src/commands/doctor-session-snapshots.ts:331-342`
  - body: `finding.cachedPath` has already been XML-decoded by the scanner, but real skill prompts write `<location>` with XML entities. If a stale path contains `&`, `<`, quotes, or apostrophes, neither this JSON-escaped decoded string nor the raw decoded fallback matches the raw `sessions.json` prompt, so `doctor --fix` leaves that stale prompt path behind.
  - confidence: 0.88
- **[P2] Handle Windows baseDir suffixes:** `src/commands/doctor-session-snapshots.ts:350`
  - body: This only strips `baseDir` when the synthetic cached path ends in `/SKILL.md`. On Windows, `path.join(baseDir, "SKILL.md")` produces a `\SKILL.md` suffix, so the PR repairs `filePath` but leaves the raw `baseDir` value stale and the next doctor pass still reports the session.
  - confidence: 0.9
- **[P2] Write repaired session stores atomically:** `src/commands/doctor-session-snapshots.ts:372`
  - body: The existing session store writer uses atomic replacement and Windows retry handling, but this repair truncates and rewrites `sessions.json` directly. If `doctor --fix` crashes or overlaps with gateway/session activity, readers can see a partial file or a concurrent session update can be lost despite the backup.
  - confidence: 0.84
