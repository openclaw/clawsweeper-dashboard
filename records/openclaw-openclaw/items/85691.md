## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Handle XML-escaped prompt paths:** `src/commands/doctor-session-snapshots.ts:329-330`
  - body: `cachedPath` has already been XML-decoded by the scanner, but persisted skill prompts are formatted with XML escaping. For a stale path containing `&`, `<`, quotes, or apostrophes, these `JSON.stringify` search strings will not match the actual `<location>` text in `sessions.json`, so `doctor --fix` can report a repair candidate and then leave the stale prompt unchanged. Re-escape location text or mutate the parsed prompt field with the same XML contract.
  - confidence: 0.9
- **[P2] Handle Windows baseDir separators:** `src/commands/doctor-session-snapshots.ts:348`
  - body: This special-case only runs when the synthesized `cachedPath` ends with `/SKILL.md`. On Windows, `path.join(skill.baseDir, "SKILL.md")` uses backslashes, so a legacy `resolvedSkills[].baseDir` entry is detected but the stored `baseDir` string is not repaired. Use platform-aware suffix handling or mutate the parsed `baseDir` field directly.
  - confidence: 0.86
- **[P2] Write repairs through the session-store writer:** `src/commands/doctor-session-snapshots.ts:373`
  - body: The patch writes the raw JSON text directly after global replacements, bypassing the session-store writer, atomic cache updates, and field-level normalization. If the same path string appears in another session metadata field or a queued writer has newer data, this can rewrite unrelated state or lose session updates; parse the store, mutate only the diagnosed snapshot fields, and save through the existing writer path.
  - confidence: 0.88
