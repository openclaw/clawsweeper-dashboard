## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Preserve XML entities when replacing prompt paths:** `src/commands/doctor-session-snapshots.ts:331-346`
  - body: The scan decodes `<location>` XML entities before producing `finding.cachedPath`, but the repair only looks for the decoded JSON string. A prompt containing an XML-escaped path segment such as `R&amp;D/.../SKILL.md` will still be reported but will not be replaced in raw `sessions.json`; add an XML-escaped replacement form or repair the parsed store before serializing.
  - confidence: 0.87
- **[P2] Handle Windows separators when repairing baseDir:** `src/commands/doctor-session-snapshots.ts:350`
  - body: This branch only repairs `baseDir` when the synthesized cached path ends with `"/SKILL.md"`. On Windows, `path.join(baseDir, "SKILL.md")` produces a `\SKILL.md` suffix, so stale `resolvedSkills[].baseDir` entries remain unchanged even if `filePath` was fixed; use path APIs or handle both separators and add a win32 `baseDir` test.
  - confidence: 0.9
- **[P2] Use the session-store writer for snapshot repair:** `src/commands/doctor-session-snapshots.ts:372`
  - body: The repair reads `sessions.json` and writes it back directly. Other session mutations go through `updateSessionStore` / `runExclusiveSessionStoreWrite` and `writeTextAtomic`, so this can race an active gateway or CLI session and overwrite unrelated updates or leave a non-atomic store write. Wrap the repair in the existing writer/atomic path or an equivalent raw-text helper.
  - confidence: 0.86
