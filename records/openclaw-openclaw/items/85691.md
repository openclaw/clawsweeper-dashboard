## Review Findings

Overall correctness: patch is incorrect

Full review comments:

- **[P1] Preserve unrelated promptRef snapshots during repair:** `src/commands/doctor-session-snapshots.ts:462`
  - body: `loadSessionStoreForSnapshotScan()` hydrates every entry, and current main deletes `skillsSnapshot` when an unrelated `promptRef` blob is unreadable. Because this block later serializes the whole hydrated store whenever any session is repaired, `doctor --fix` can permanently drop snapshot data for sessions it did not repair. Keep raw/unmodified entries intact or only persist the repaired entries.
  - confidence: 0.93
- **[P2] Normalize repaired stores before writing:** `src/commands/doctor-session-snapshots.ts:480-490`
  - body: The canonical session-store path strips `skillsSnapshot.resolvedSkills` because it is a runtime cache, but this write path only projects prompt blobs before serializing `persisted`. A legacy store with repairable `resolvedSkills` paths will be written back with those bulky runtime caches still present, so the doctor repair should normalize/drop them after using them for findings.
  - confidence: 0.84
