## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Handle XML-escaped prompt paths:** `src/commands/doctor-session-snapshots.ts:331-344`
  - body: `extractSkillLocations` decodes XML entities before producing `finding.cachedPath`, and skill prompt formatters XML-escape `<location>` values. A stored prompt path like `/old/a&amp;b/...` yields a finding for `/old/a&b/...`, but this repair only searches the JSON-escaped/raw decoded string, leaving the prompt stale after `--fix`.
  - confidence: 0.88
- **[P2] Handle Windows baseDir suffixes:** `src/commands/doctor-session-snapshots.ts:350-352`
  - body: On Windows, `collectResolvedSkillPaths` appends `SKILL.md` with the platform path module, so the synthetic `cachedPath` for `resolvedSkills[].baseDir` ends with `\SKILL.md`. This POSIX-only suffix check skips the directory rewrite there, leaving `baseDir` pointed at the old runtime root.
  - confidence: 0.9
- **[P2] Use the session-store writer for repairs:** `src/commands/doctor-session-snapshots.ts:371-372`
  - body: The session store has a writer/update path that serializes writes and refreshes store caches after saving. Writing `sessions.json` directly bypasses those semantics, so a later queued store save in the same process can overwrite the repair or leave cache state stale.
  - confidence: 0.78
