## Summary

This is a useful fix direction for a real doctor/session-state gap, but the PR is not merge-ready: the repair can report success without proving the session store is clean afterward, the after-fix proof is a standalone mirror script rather than patched `openclaw doctor --fix`, and the new script currently fails whitespace checks.

## What This Changes

The PR adds a `shouldRepair` path to session snapshot doctor health so `doctor --fix` can rewrite stale bundled skill paths in `sessions.json`, wires that flag from the doctor flow, and adds a standalone reproduction script.

## Best Possible Solution

Keep the `doctor --fix` repair path, but land it only after the production function is covered by Vitest, the repair path re-scans and reports any remaining findings, and a redacted patched `openclaw doctor --fix` run proves the store is clean after repair.

## Solution Assessment
