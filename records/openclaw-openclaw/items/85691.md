Full review comments:

- **[P1] Route repairs through the session writer:** `src/commands/doctor-session-snapshots.ts:397`
  - body: This writes the repaired `sessions.json` directly with `writeTextAtomic`, so it bypasses `runExclusiveSessionStoreWrite` and never updates or drops the session-store caches. Existing session-store mutations use `updateSessionStore` to serialize writes and refresh caches; a later queued store write in the same process can write a cached pre-repair store back over this repair. Route this mutation through the session-store writer instead of raw file replacement.
