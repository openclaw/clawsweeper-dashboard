## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Replace XML-escaped prompt locations too:** `src/commands/doctor-session-snapshots.ts:331-344`
  - body: The scanner decodes <location> text before building finding.cachedPath, but the skill prompt formatters XML-escape skill.filePath. For paths containing &, <, >, quotes, or apostrophes, neither the JSON-escaped decoded string nor the raw decoded path exists in sessions.json, so doctor --fix leaves a reported stale prompt path unchanged. Mutate the parsed prompt or replace the XML-escaped form as well.
  - confidence: 0.88
- **[P2] Handle Windows separators when repairing baseDir:** `src/commands/doctor-session-snapshots.ts:350`
  - body: On a native Windows doctor run, collectResolvedSkillPaths appends SKILL.md with the platform path module, so a legacy baseDir finding ends in \SKILL.md. This condition only accepts /SKILL.md, skipping the directory-prefix replacement and leaving resolvedSkills[].baseDir stale even though the PR claims to repair that field.
  - confidence: 0.86
- **[P2] Route repairs through the session-store writer:** `src/commands/doctor-session-snapshots.ts:372`
  - body: Writing sessions.json directly here bypasses the session-store writer, atomic replacement helper, and cache refresh used by updateSessionStore. If doctor --fix runs while another OpenClaw process writes sessions, this can overwrite newer session updates or leave process-local caches stale after the file changes.
  - confidence: 0.84
