  - body: `extractSkillLocations` decodes XML entities before creating `finding.cachedPath`, but the persisted prompt was formatted with escaped `<location>` text.   - confidence: 0.9

  - body: On Windows, the synthetic baseDir finding ends with `\SKILL.md`, so this `/SKILL.md` check skips the directory-prefix replacement. The `filePath` may be repaired, but the raw `baseDir` field remains pointed at the stale runtime root.
  - confidence: 0.88

  - body: This direct write bypasses `updateSessionStore`'s exclusive writer queue, atomic write helper, normalization, and cache refresh. A doctor repair can race with normal session writes or leave in-process session-store caches stale, which is risky for persisted session state.
  - confidence: 0.87
