## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Repair XML-escaped prompt paths:** `src/commands/doctor-session-snapshots.ts:331-332`
  - body: The scanner decodes `<location>` entities before producing `finding.cachedPath`, but persisted `skillsSnapshot.prompt` locations are XML-escaped. For paths containing `&`, `<`, `>`, quotes, or apostrophes, neither the decoded raw path nor its JSON escape appears in the file, so the stale prompt path can remain after `--fix`; add the XML-escaped replacement form and regression coverage.
  - confidence: 0.9
- **[P2] Use the session-store writer for repairs:** `src/commands/doctor-session-snapshots.ts:372`
  - body: This writes the live `sessions.json` with `fs.writeFileSync`, bypassing the session-store writer and atomic write path used for normal store mutations. A crash during the write can corrupt the store, and the writer owns serialization, cache updates, and normalization; route the repair through `updateSessionStore` or an equivalent atomic raw transform.
  - confidence: 0.86
- **[P2] Handle Windows baseDir separators:** `src/commands/doctor-session-snapshots.ts:350`
  - body: The baseDir rewrite only runs when the synthetic cached path ends with `/SKILL.md`. On Windows, a `baseDir` path is synthesized with `\SKILL.md`, so the sibling `filePath` can be repaired while `baseDir` remains stale; check both separators or derive the directory with the matching path API.
  - confidence: 0.88
