## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Repair XML-escaped prompt locations:** `src/commands/doctor-session-snapshots.ts:331-345`
  - body: `extractSkillLocations` decodes XML entities before emitting `finding.cachedPath`, but prompt locations are stored XML-escaped. If a stale path contains `&`, `<`, `>`, quotes, or apostrophes, this replacement searches for the decoded value and leaves the stale prompt path behind; update parsed prompt data or replace the XML-escaped form too.
  - confidence: 0.91
- **[P2] Handle Windows baseDir suffixes:** `src/commands/doctor-session-snapshots.ts:350`
  - body: On Windows, a `resolvedSkills[].baseDir` finding is built with a path ending in `\SKILL.md`, but this guard only matches `/SKILL.md`. That lets `filePath` repair while the sibling `baseDir` remains stale, so use platform-aware suffix handling or derive the directory from path segments.
  - confidence: 0.89
- **[P2] Route repairs through the session-store writer:** `src/commands/doctor-session-snapshots.ts:372`
  - body: This directly truncates and rewrites `sessions.json`, bypassing the existing per-store writer queue, cache update, and atomic replace path used by `saveSessionStore`/`updateSessionStore`. A crash or concurrent session write can leave a partial file or lose newer metadata; use the store mutation path or an equivalent locked atomic raw-text repair.
  - confidence: 0.86
