## Summary

Keep this PR open, but it is not merge-ready: the repair path is useful and addresses a real report-only `doctor --fix` gap, but the current diff can miss stale XML-escaped prompt paths while reporting a successful repair, and the supplied proof exercises a standalone mirror script rather than the patched `openclaw doctor --fix` command.

## What This Changes

The PR adds `shouldRepair` handling to session snapshot health, passes the doctor repair flag into it, rewrites stale bundled skill paths in `sessions.json`, and adds a standalone reproduction script.

## Best Possible Solution

Keep the repair behind `doctor --fix`, but make it cover every encoded form the detector can report, add focused Vitest coverage for the write/backup path, and replace the mirror-script proof with redacted output from the actual doctor command.
