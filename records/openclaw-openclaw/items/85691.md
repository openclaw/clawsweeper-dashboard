## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Serialize repairs through the session-store writer:** `src/commands/doctor-session-snapshots.ts:371-372`
  - body: This writes `sessions.json` from a raw snapshot outside `runExclusiveSessionStoreWrite` and the normal save path. A queued session update can overwrite the repair or be overwritten by it, and the cache/atomic-save behavior used by normal session-store mutations is bypassed.
  - confidence: 0.9
- **[P2] Preserve XML-escaped prompt locations:** `src/commands/doctor-session-snapshots.ts:331-344`
  - body: The scanner decodes `<location>` text before producing `finding.cachedPath`, but persisted skill prompts escape file paths. If a stale path contains `&` or another XML entity, neither the decoded JSON-escaped form nor the decoded raw form matches the stored prompt, so the repair can leave stale prompt paths behind.
  - confidence: 0.88
- **[P2] Repair Windows baseDir paths too:** `src/commands/doctor-session-snapshots.ts:350-352`
  - body: The new baseDir repair branch only handles findings ending in `/SKILL.md`. On Windows, `collectResolvedSkillPaths` builds `baseDir + \SKILL.md`, so the branch never strips the directory prefix and `resolvedSkills.baseDir` can stay stale after `--fix`.
  - confidence: 0.83
