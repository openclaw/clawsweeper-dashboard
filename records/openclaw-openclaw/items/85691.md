## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Repair XML-escaped prompt paths:** `src/commands/doctor-session-snapshots.ts:331-332`
  - body: The scanner decodes `<location>` entities before producing `finding.cachedPath`, but persisted `skillsSnapshot.prompt` locations are XML-escaped. For paths containing `&`, `<`, `>`, quotes, or apostrophes, neither the decoded raw path nor its JSON escape appears in the file, so the stale prompt path can remain after `--fix`; add the XML-escaped replacement form and regression coverage.
  - confidence: 0.9
- **[P2] Write repairs through a session-store-safe path:** `src/commands/doctor-session-snapshots.ts:372`
  - body: This writes the live `sessions.json` with `fs.writeFileSync`, bypassing the session-store writer lock, atomic write helper, cache updates, and normalization path used for normal mutations. Route the repair through `updateSessionStore` where possible or an equivalent writer-locked atomic raw transform.
  - confidence: 0.86
- **[P2] Handle Windows baseDir separators:** `src/commands/doctor-session-snapshots.ts:350`
  - body: The baseDir rewrite only runs when the synthetic cached path ends with `/SKILL.md`. On Windows, `path.join(baseDir, "SKILL.md")` produces `\SKILL.md`, so a `resolvedSkills.baseDir` entry can remain stale even when its sibling `filePath` is repaired; derive the directory with the matching path API or check both separators.
  - confidence: 0.88
