## Summary

## What This Changes

The PR adds `shouldRepair` plumbing to session snapshot doctor health, raw JSON replacement/backups for stale bundled-skill paths, and a standalone reproduction script.

## Best Possible Solution

Repair the stale persisted prompt/report paths through a session-store-owned path or parsed migration that also drops runtime-only `resolvedSkills`, then prove the real `doctor --fix` flow on an affected store.

## Reproduction Assessment

Yes, from source inspection: current main detects stale snapshot paths but `runSessionSnapshotsHealth` does not pass `shouldRepair`, so `doctor --fix` cannot repair this check. I did not run the full CLI path in this read-only review.

## Solution Assessment

No. The repair should use the session-store contract or remove runtime-only `resolvedSkills`; raw replacement is too easy to diverge from the persisted session semantics.
