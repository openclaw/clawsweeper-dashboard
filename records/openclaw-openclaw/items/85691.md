Full review comments:

- **[P1] Route repairs through the session-store writer:** `src/commands/doctor-session-snapshots.ts:501`
  - body: This direct `sessions.json` write happens outside `saveSessionStore`/`updateSessionStore`, so it neither serializes with the per-store writer queue nor refreshes the object, snapshot, serialized, and promptRef caches that normal writes update.   - confidence: 0.88

  - body: The added root script copies scanner and repair logic instead of invoking production doctor code, so it can keep passing while `src/commands/doctor-session-snapshots.ts` drifts. The colocated Vitest coverage is the durable proof surface; delete this standalone script rather than shipping a second implementation at repo root.
