## Summary

Keep this PR open. The linked bug is real and the direction is useful, but the patch still has session-state correctness blockers: it rewrites `sessions.json` outside the existing writer/cache/atomic path, misses XML-escaped prompt paths, and still has a Windows `baseDir` suffix gap.

## What This Changes

The PR adds `shouldRepair` support to doctor session snapshot health, wires `doctor --fix` into that check, rewrites stale skill snapshot paths in `sessions.json`, and adds a standalone repro script.

## Best Possible Solution

Keep the auto-repair behavior, but implement it through the existing session-store write path with focused coverage for XML-escaped prompt locations, Windows `baseDir` paths, cache behavior, and real `openclaw doctor --fix` proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve XML escaping for prompt repairs:** `src/commands/doctor-session-snapshots.ts:331-332`
  - body: `skillsSnapshot.prompt` locations are XML-escaped when generated, but this repair searches for the decoded `finding.cachedPath` and writes the decoded `expectedPath`. A stale root containing `&`, `<`, quotes, or similar characters will either not be replaced in the raw JSON prompt or will be written back without the XML escaping the prompt formatter expects.
  - confidence: 0.88
- **[P2] Handle Windows SKILL.md suffixes for baseDir repairs:** `src/commands/doctor-session-snapshots.ts:350`
  - body: This branch only enters the `baseDir` repair when the synthetic cached path ends with `/SKILL.md`. On Windows, `path.join(baseDir, "SKILL.md")` produces a `\SKILL.md` suffix, so stale `resolvedSkills.baseDir` values can be left behind even though the PR claims to repair Windows paths.
  - confidence: 0.9
- **[P2] Use the session store writer for repairs:** `src/commands/doctor-session-snapshots.ts:372`
  - body: Writing `sessions.json` with `fs.writeFileSync` bypasses the existing session-store writer queue, atomic replacement helper, normalization, and cache updates. If a session update is in flight or the process cache is warm, `doctor --fix` can overwrite newer session state or leave later reads using stale cached data.
  - confidence: 0.86
