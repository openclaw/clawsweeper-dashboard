## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve XML-escaped prompt paths:** `src/commands/doctor-session-snapshots.ts:331-344`
  - body: `extractSkillLocations` decodes XML entities before producing `finding.cachedPath`, but this repair only searches for the JSON-escaped decoded string and the raw decoded string. If the prompt stored `<location>...&amp;...</location>`, the stale path in `sessions.json` remains XML-escaped and neither replacement matches, so `doctor --fix` still leaves that stale prompt path behind. Please encode the prompt form or update the parsed field directly before writing.
  - confidence: 0.9
- **[P2] Handle Windows baseDir suffixes:** `src/commands/doctor-session-snapshots.ts:350-352`
  - body: This baseDir repair only runs when the synthesized cached path ends with `/SKILL.md`. On Windows, `path.join(baseDir, "SKILL.md")` produces a `\SKILL.md` suffix, so resolvedSkills.filePath may be repaired while the sibling baseDir remains pointed at the stale runtime root. Use path-aware suffix handling for both separators before deriving the directory replacement.
  - confidence: 0.88
- **[P2] Write repairs through the session-store writer:** `src/commands/doctor-session-snapshots.ts:371-372`
  - body: The repair reads raw `sessions.json`, rewrites matching text, and writes it directly with `fs.writeFileSync`, bypassing the per-store writer used by `updateSessionStore`. If a Gateway/runtime session update is active, this offline write can race and lose or stale cached store state; please route the mutation through the session-store writer or an owner-approved offline repair helper that explicitly handles that boundary.
  - confidence: 0.82
