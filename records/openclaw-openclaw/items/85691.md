Full review comments:

- **[P1] Keep repair writes inside the session-store writer:** `src/commands/doctor-session-snapshots.ts:501`
  - body: This `--fix` path parses a snapshot of `sessions.json` and then replaces the whole file with `writeTextAtomic`, bypassing the session-store writer and cache update path used by `saveSessionStore`/`updateSessionStore`. If another doctor/session path has the store cached or writes while this repair runs, the repair can resurrect stale session data or overwrite fresh updates; run the targeted repair under the session-store write boundary, or add an equivalent helper that serializes the write and updates/invalidates caches before replacing the store.
  - confidence: 0.84
- **[P3] Remove the root proof script from the product tree:** `repro-doctor-session-snapshot-repair.mjs:1`
  - body: `repro-doctor-session-snapshot-repair.mjs` is a one-off proof harness that duplicates the repair logic outside the maintained test suite. The PR already has colocated Vitest coverage and PR-body proof; keeping this root script would add an unowned second implementation that can drift from production doctor behavior.
  - confidence: 0.82
