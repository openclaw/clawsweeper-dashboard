## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve unrelated promptRef snapshots before writing:** `src/commands/doctor-session-snapshots.ts:462-490`
  - body: `loadSessionStoreForSnapshotScan()` hydrates every session, and current promptRef hydration deletes `skillsSnapshot` when a blob is unreadable. Because this block then serializes the whole store, running `doctor --fix` for one repairable session can permanently drop unrelated snapshot data from another session. Please preserve raw untouched sessions or write only targeted repaired entries through the session-store writer.
  - confidence: 0.88
- **[P2] Route the rewrite through session-store persistence:** `src/commands/doctor-session-snapshots.ts:489-490`
  - body: This writes `sessions.json` directly with `writeTextAtomic`, bypassing the existing session-store writer that coordinates prompt blob staging and write caches. In a process that owns session-store cache state, a later save can work from stale cached data or miss the doctor repair. Please use the canonical writer path or explicitly update the same cache/prompt-blob invariants.
  - confidence: 0.75
- **[P2] Add regression coverage for the real repair path:** `repro-doctor-session-snapshot-repair.mjs:1`
  - body: The added root repro script duplicates repair logic but is not wired into the Vitest suite, and the existing doctor snapshot tests still only cover scanning/reporting. For this persisted session-state mutation, please add focused tests for `shouldRepair`, promptRef-backed repairs, unreadable unrelated promptRefs, and scoped replacements.
  - confidence: 0.8
