## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Repair XML-escaped prompt locations:** `src/commands/doctor-session-snapshots.ts:331-345`
  - body: `extractSkillLocations` decodes XML entities before emitting `finding.cachedPath`, but real skill prompt locations are XML-escaped when formatted. If a stale path contains `&`, `<`, `>`, quotes, or apostrophes, the raw `sessions.json` text contains the escaped form while this replacement searches for the decoded value, so `doctor --fix` reports the finding and leaves the stale prompt path behind. Please update the stored prompt via parsed data or replace the XML-escaped form too.
  - confidence: 0.91
- **[P2] Handle Windows baseDir suffixes:** `src/commands/doctor-session-snapshots.ts:350`
  - body: On Windows, a `resolvedSkills[].baseDir` finding is built with a path ending in `\SKILL.md`, but this guard only matches `/SKILL.md`. That means the `filePath` can be repaired while the sibling `baseDir` remains pointed at the stale runtime root, so later consumers may still read stale metadata. Use platform-aware suffix handling or derive the directory from path segments instead of a POSIX-only suffix.
  - confidence: 0.89
- **[P2] Route repairs through the session-store writer:** `src/commands/doctor-session-snapshots.ts:372`
  - body: This directly truncates and rewrites `sessions.json`, bypassing the existing session-store writer queue, cache update, and atomic replace path used by `saveSessionStore`/`updateSessionStore`. A crash or concurrent gateway/session write can leave a partial file or lose newer session metadata. Please perform the repair through the existing store mutation path, or add an equivalent writer-locked atomic raw-text repair with cache invalidation.
  - confidence: 0.86
