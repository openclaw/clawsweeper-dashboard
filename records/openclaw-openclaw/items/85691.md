## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Handle XML-escaped prompt locations:** `src/commands/doctor-session-snapshots.ts:331-344`
  - body: `extractSkillLocations` decodes `<location>` text before producing `finding.cachedPath`, but prompt locations are written with XML escaping. For a path containing `&`, `<`, quotes, or similar characters, raw `sessions.json` contains the escaped form such as `&amp;`, so replacing only `JSON.stringify(finding.cachedPath)` or the decoded raw string will leave the stale prompt path unrepaired under `--fix`.
  - confidence: 0.9
- **[P2] Handle Windows baseDir suffixes:** `src/commands/doctor-session-snapshots.ts:350`
  - body: On Windows, the synthetic `baseDir` finding produced by `path.join(skill.baseDir, "SKILL.md")` ends with `\SKILL.md`, but this branch only recognizes `/SKILL.md`. That means `resolvedSkills[].baseDir` can stay pointed at the old runtime even when the filePath replacement succeeds.
  - confidence: 0.86
- **[P2] Write sessions.json through the session-store-safe path:** `src/commands/doctor-session-snapshots.ts:372`
  - body: Normal session-store saves are serialized and written atomically through `writeTextAtomic`, but this repair overwrites `sessions.json` directly after creating a backup. If doctor runs while an agent or gateway is active, or the process is interrupted, this can race with normal session writes or leave a partial/stale store, so the repair should use the existing session-store writer/atomic semantics.
  - confidence: 0.84
