## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Preserve XML entities when replacing prompt paths:** `src/commands/doctor-session-snapshots.ts:328-341`
  - body: The scan decodes `<location>` XML entities before producing `finding.cachedPath`, but the repair only searches for the decoded JSON string. A prompt path containing an XML-escaped segment such as `R&amp;D/.../SKILL.md` will still be reported but not replaced in raw `sessions.json`; add an XML-escaped replacement form or repair through parsed data and focused coverage.
  - confidence: 0.87
- **[P2] Handle Windows separators when repairing baseDir:** `src/commands/doctor-session-snapshots.ts:347`
  - body: This branch only repairs `baseDir` when the synthesized cached path ends with `"/SKILL.md"`. On Windows, `path.join(baseDir, "SKILL.md")` produces a `\SKILL.md` suffix, so stale `resolvedSkills[].baseDir` values can remain unchanged even if `filePath` is fixed; handle both separators and add a win32-style test.
  - confidence: 0.9
- **[P2] Use the session-store writer for snapshot repair:** `src/commands/doctor-session-snapshots.ts:369`
  - body: The repair reads `sessions.json` and writes it back directly. Other session mutations use `runExclusiveSessionStoreWrite` plus atomic replacement, so this can race an active gateway or CLI session and overwrite unrelated updates or leave a non-atomic store write. Wrap the raw repair in the existing writer/atomic path or an equivalent helper.
  - confidence: 0.86
