## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Serialize repairs through the session-store writer:** `src/commands/doctor-session-snapshots.ts:371-372`
  - body: This branch reads raw `sessions.json` and writes it back directly. Current session-store writes are serialized through `updateSessionStore` / `runExclusiveSessionStoreWrite` and update the process caches; bypassing that path during `doctor --fix` can overwrite concurrent Gateway session updates or leave stale cache state. Please route this repair through the session-store writer or make the command explicitly offline-safe.
  - confidence: 0.86
- **[P2] Preserve XML-escaped prompt locations:** `src/commands/doctor-session-snapshots.ts:331-338`
  - body: Skill prompt locations are XML-escaped when rendered, and the scanner decodes them before returning `finding.cachedPath`. Searching the raw JSON for the decoded path will miss valid paths containing `&`, `<`, `>`, quotes, or apostrophes, leaving those stale prompt locations unrepaired. Repair the parsed prompt value semantically or re-escape the path for XML before replacement, with a regression test.
  - confidence: 0.88
- **[P2] Handle Windows baseDir suffixes:** `src/commands/doctor-session-snapshots.ts:350-352`
  - body: For legacy `resolvedSkills.baseDir` entries on Windows, `collectResolvedSkillPaths` appends `SKILL.md` with the platform separator, so the synthesized cached path can end in `\SKILL.md`. This guard only accepts `/SKILL.md`, so the baseDir field is not rewritten on Windows even though the PR says it is. Accept both separators or derive the directory with a platform-aware helper.
  - confidence: 0.82
