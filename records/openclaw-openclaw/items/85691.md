## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Handle XML-escaped prompt paths:** `src/commands/doctor-session-snapshots.ts:331-332`
  - body: `cachedPath` has already been XML-decoded by the scanner, but persisted skill prompts escape XML characters inside `<location>`. For a stale path containing `&`, `<`, quotes, or apostrophes, these JSON string searches will not match the stored prompt text, so `doctor --fix` can report a repair candidate and leave the stale prompt unchanged. Re-escape location text or mutate the parsed prompt field with the same XML contract.
  - confidence: 0.9
- **[P2] Handle Windows baseDir separators:** `src/commands/doctor-session-snapshots.ts:350`
  - body: This special case only runs when the synthesized path ends with `/SKILL.md`. On Windows, `path.join(skill.baseDir, "SKILL.md")` uses backslashes, so a legacy `resolvedSkills[].baseDir` entry is detected but the stored `baseDir` string is not repaired. Use platform-aware suffix handling or mutate the parsed `baseDir` field directly.
  - confidence: 0.86
- **[P2] Write repairs through the session-store writer:** `src/commands/doctor-session-snapshots.ts:372`
  - body: The patch writes the raw JSON text directly after global replacements, bypassing the session-store writer, atomic cache updates, and field-level normalization. If the same path string appears in another metadata field or a queued writer has newer data, this can rewrite unrelated state or lose session updates; parse the store, mutate only the diagnosed snapshot fields, and save through the existing writer path.
  - confidence: 0.88
