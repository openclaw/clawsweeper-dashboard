## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Preserve XML escaping when replacing prompt paths:** `src/commands/doctor-session-snapshots.ts:331-332`
  - body: `extractSkillLocations` decodes `&amp;`, `&lt;`, and other XML entities before creating each finding, but this repair only searches for the JSON-escaped decoded path. If the stale runtime path contains an XML-sensitive character, the raw `skillsSnapshot.prompt` still contains the escaped form, so `doctor --fix` reports the stale path but leaves it unrepaired. Re-encode prompt-location text for `skillsSnapshot.prompt` replacements, or repair the parsed session entry and serialize it safely.
  - confidence: 0.88
- **[P2] Handle Windows baseDir suffixes:** `src/commands/doctor-session-snapshots.ts:350`
  - body: On Windows, `collectResolvedSkillPaths` appends `SKILL.md` with a backslash, so `finding.cachedPath.endsWith("/SKILL.md")` is false for baseDir-derived findings. That means a stale `resolvedSkills[].baseDir` can survive `doctor --fix` even though the scan found it. Make the suffix handling path-aware or accept both `/SKILL.md` and `\SKILL.md`.
  - confidence: 0.9
- **[P2] Use the session store writer for repairs:** `src/commands/doctor-session-snapshots.ts:372`
  - body: This writes `sessions.json` directly from a raw snapshot instead of going through the per-store session writer used by current session mutations. If the Gateway or another doctor/session repair writes the same store while this runs, this raw write can clobber fresh session state and bypass the store caches. Route the repair through `updateSessionStore` or an equivalent serialized atomic writer.
  - confidence: 0.86
