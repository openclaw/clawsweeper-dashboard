## Summary

Keep open: the linked doctor bug is source-supported and the branch targets the right repair path, but the current implementation is not merge-ready because it rewrites raw JSON bytes and can miss encoded session paths while reporting success; the provided proof also does not show the patched `openclaw doctor --fix` behavior running.

## What This Changes

This PR adds `shouldRepair` handling to session snapshot health so `openclaw doctor --fix` rewrites stale bundled skill paths in `sessions.json` and wires the doctor health flow to pass the repair flag.

## Best Possible Solution

Keep the repair direction, but mutate parsed session-store values, write a validated backup/store, re-scan or report any leftovers, and add focused coverage for POSIX plus JSON-escaped Windows or XML-escaped path cases.

## Reproduction Assessment

Yes, source-level: current main detects stale snapshot paths but `runSessionSnapshotsHealth` never passes repair intent and `noteSessionSnapshotHealth` only reports findings. I did not run a live stale-store repro because this review is read-only.

## Solution Assessment

No. Wiring `shouldRepair` is the right shape, but the repair should operate on parsed session data with validation and post-repair reporting instead of raw byte replacement.
