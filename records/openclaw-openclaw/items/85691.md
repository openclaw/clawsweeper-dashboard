Full review comments:

- **[P1] Route session repairs through the store writer:** `src/commands/doctor-session-snapshots.ts:501`
  - body: This writes `sessions.json` directly after reading and mutating a raw parsed copy. Existing session-store mutations go through `saveSessionStore`/`updateSessionStore`, which serialize writes and update the in-process caches after the atomic write; bypassing that path can clobber a queued session update or leave later reads in the same process using stale cached state after `doctor --fix` reports success.
  - confidence: 0.86
- **[P2] Remove the root proof script:** `repro-doctor-session-snapshot-repair.mjs:1`
  - body: The PR already has colocated Vitest coverage for the repair path. Keeping a root-level proof script that duplicates scanner and repair logic creates a second implementation that can drift and look like a supported entrypoint; keep proof in the PR body/artifacts and the regression tests instead.
  - confidence: 0.82
