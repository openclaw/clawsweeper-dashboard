## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Repair XML-escaped prompt locations:** `src/commands/doctor-session-snapshots.ts:331-344`
  - body: Skill prompts escape `<location>` values before persistence, and the scanner decodes them before producing `finding.cachedPath`. Replacing only `JSON.stringify(finding.cachedPath)` or the raw decoded string will not match stores containing `&amp;`, `&lt;`, etc., so `doctor --fix` can still leave those sessions stale.
  - confidence: 0.9
- **[P2] Handle Windows baseDir suffixes:** `src/commands/doctor-session-snapshots.ts:350-352`
  - body: The baseDir branch only runs when the synthetic path ends with `/SKILL.md`. On Windows that path can end with `\SKILL.md`, so the stored `baseDir` remains pointed at the stale runtime root even after the file path is repaired.
  - confidence: 0.83
- **[P2] Write repairs through the session-store writer:** `src/commands/doctor-session-snapshots.ts:371-372`
  - body: This writes the whole store from a raw snapshot outside `updateSessionStore` and the writer/atomic/cache path. If another session save lands during doctor, this can overwrite it, and same-process session-store caches are not updated after the repair.
  - confidence: 0.88
