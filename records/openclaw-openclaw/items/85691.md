## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Preserve unrelated promptRef snapshots before writing:** `src/commands/doctor-session-snapshots.ts:462`
  - body: This repair reloads the whole store through `loadSessionStoreForSnapshotScan()` and later serializes that hydrated object. That loader calls the promptRef hydrator, which deletes `skillsSnapshot` when a prompt blob is unreadable; if any other session in the same store has a repairable stale path, this PR will persist that unrelated deletion. Preserve unhydrated entries or mutate through a writer path that does not serialize scan-only cleanup.
  - confidence: 0.88
- **[P2] Route the repair through the session-store writer:** `src/commands/doctor-session-snapshots.ts:490`
  - body: Writing `sessions.json` directly bypasses the session-store queue, cache update path, session temp-file naming, and Windows retry behavior used by `saveSessionStore` and `updateSessionStore`. A doctor repair should either use those APIs or factor the same invariants into this path so later session mutations cannot overwrite or miss the repaired state.
  - confidence: 0.74
