Full review comments:

- **[P1] Route repairs through the session-store writer:** `src/commands/doctor-session-snapshots.ts:501`
  - body: This writes `sessions.json` directly with `writeTextAtomic`, bypassing the session-store writer queue and cache update path that other session doctor repairs use. If the process has a warmed store or a queued writer, a later session-store save can persist the stale in-memory snapshot back over the repaired paths while doctor has already reported success. Please perform the repair through `updateSessionStore` or an equivalent writer-boundary helper and cover the warmed or in-flight writer case.
  - confidence: 0.86

  - confidence: 0.9
