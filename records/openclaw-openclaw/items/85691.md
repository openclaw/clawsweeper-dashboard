## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Repair XML-escaped prompt locations:** `src/commands/doctor-session-snapshots.ts:331-344`
  - body: `extractSkillLocations` decodes XML entities before producing `finding.cachedPath`, but persisted skill prompts escape `<location>` values. A stale path containing XML-reserved characters is detected from the decoded value, then neither the JSON-escaped decoded string nor the raw decoded string matches the stored `&amp;`/`&lt;` form, so `doctor --fix` can still report the stale snapshot afterward.
  - confidence: 0.88
- **[P2] Handle Windows resolvedSkills baseDir paths:** `src/commands/doctor-session-snapshots.ts:350`
  - body: This baseDir repair only runs when the synthetic cached path ends with `/SKILL.md`. On Windows, `collectResolvedSkillPaths` uses the platform `path.join`, so the baseDir-derived finding ends in `\SKILL.md` while the raw JSON stores only the directory path; the file-path replacement misses it and this branch never repairs the stored `baseDir`.
  - confidence: 0.84
- **[P2] Write repairs through the session-store writer:** `src/commands/doctor-session-snapshots.ts:371-372`
  - body: The repair writes `sessions.json` directly with `fs.writeFileSync`, bypassing the session-store writer's normalization, atomic write, Windows retry, and cache update path. Because this mutates persisted session state, use the existing writer or an equivalent helper so the repaired store is serialized and cached consistently.
  - confidence: 0.8
