## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Repair XML-escaped prompt paths:** `src/commands/doctor-session-snapshots.ts:331-332`
  - body: The scanner decodes `<location>` entities before producing `finding.cachedPath`, but persisted `skillsSnapshot.prompt` is generated with XML escaping. For paths containing `&`, `<`, `>`, quotes, or apostrophes, neither the decoded raw path nor its JSON escape appears in the file, so `doctor --fix` reports success only if another field changed and leaves the stale prompt path behind. Add the XML-escaped replacement form and cover it with a regression test.
  - confidence: 0.9
- **[P2] Use the session-store writer for repairs:** `src/commands/doctor-session-snapshots.ts:372`
  - body: This writes the live `sessions.json` with `fs.writeFileSync`, bypassing the session-store writer and `writeTextAtomic` path used for normal store mutations. A crash during the write can corrupt the store, and the writer also owns cache updates and normalization such as stripping persisted `resolvedSkills`; route the repair through `updateSessionStore` or an equivalent atomic writer that preserves the current store safely.
  - confidence: 0.86
- **[P2] Handle Windows baseDir separators:** `src/commands/doctor-session-snapshots.ts:350`
  - body: The baseDir rewrite only runs when the synthetic cached path ends with `/SKILL.md`. On Windows, `collectResolvedSkillPaths()` uses `path.join`, so a Windows `baseDir` path is synthesized with `\SKILL.md`; the sibling `filePath` can be repaired while `baseDir` remains stale. Check both separators or derive the directory with the matching path API.
  - confidence: 0.88
