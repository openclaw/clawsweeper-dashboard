## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Handle XML-escaped prompt paths:** `src/commands/doctor-session-snapshots.ts:331-332`
  - body: `extractSkillLocations` decodes XML entities before producing `finding.cachedPath`, while prompt generation writes escaped `<location>` values. For paths containing `&` or another XML-escaped character, this searches for the decoded JSON string and leaves the serialized stale prompt path unchanged.
  - confidence: 0.88
- **[P2] Handle Windows baseDir separators:** `src/commands/doctor-session-snapshots.ts:350`
  - body: `collectResolvedSkillPaths` builds `baseDir` findings with the platform path separator; on Windows those findings end in `\SKILL.md`. This forward-slash-only suffix check skips the directory replacement, leaving stale `resolvedSkills.baseDir` values behind on Windows stores.
  - confidence: 0.86
- **[P2] Write repairs through the session-store path:** `src/commands/doctor-session-snapshots.ts:372`
  - body: This overwrites `sessions.json` with `fs.writeFileSync` outside `runExclusiveSessionStoreWrite`, `writeTextAtomic`, and the store cache updates. A `doctor --fix` run interrupted or racing with normal session persistence can corrupt or lose session metadata; route the final repair through the existing session-store write contract or an equivalent atomic/cache-safe helper.
  - confidence: 0.84
