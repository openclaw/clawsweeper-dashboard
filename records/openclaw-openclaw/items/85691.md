## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve XML-escaped prompt locations:** `src/commands/doctor-session-snapshots.ts:331-332`
  - body: `finding.cachedPath` has already been decoded from the `<location>` text, but persisted skill prompts are written with XML escaping. For a stale path containing `&`, `<`, quotes, or similar characters, neither the JSON-escaped decoded path nor the raw decoded path exists in `sessions.json`, so `doctor --fix` will leave that prompt stale and fall back to the warning. Repair prompt locations using the same XML escaping contract and cover it with a regression test.
  - confidence: 0.89
- **[P2] Serialize repairs through the session-store writer:** `src/commands/doctor-session-snapshots.ts:372`
  - body: This writes `sessions.json` directly after a separate raw read, bypassing the store writer queue, atomic write helper, and cache updates that normal session-store mutations use. A same-process store update can be overwritten or a later cached write can re-save stale content; wrap the raw repair in the session-store write lane or otherwise use the established atomic/cache invalidation path.
  - confidence: 0.8
