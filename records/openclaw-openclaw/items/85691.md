- **[P1] Preserve unrelated promptRef sessions during repair:** `src/commands/doctor-session-snapshots.ts:462-483`
  - body: `loadSessionStoreForSnapshotScan()` hydrates the whole store before this write path, and the hydrator deletes `skillsSnapshot` when a promptRef blob cannot be read.   - confidence: 0.88

  - body: This direct `writeTextAtomic(storePath, fixed)` skips the session-store writer queue, cache updates, session-temp naming, and prompt-blob-before-rename ordering used by current session mutations. A doctor repair in a process with cached session state, or racing another session update, can leave stale caches or lose queued changes; use the existing `updateSessionStore`/writer path for the repair write.
