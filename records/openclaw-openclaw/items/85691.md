## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Serialize repairs through the session-store writer:** `src/commands/doctor-session-snapshots.ts:369`
  - body: This repair reads a raw `sessions.json` snapshot and then writes it back with `fs.writeFileSync`, bypassing `updateSessionStore`/`saveSessionStore`'s queued atomic writer and cache update path. If `doctor --fix` runs while another OpenClaw process writes the same store, the stale raw snapshot can overwrite newer session entries on the session-state surface this PR is trying to repair.
  - confidence: 0.87
- **[P2] Preserve XML-escaped prompt locations:** `src/commands/doctor-session-snapshots.ts:331`
  - body: `finding.cachedPath` has already been XML-decoded by the scanner, but persisted skill prompts are written with XML escaping. A path containing `&`, `<`, quotes, or similar characters is stored as entities, so this replacement will not match it and may still return after repairing other findings; encode the XML location form too and cover it in `doctor-session-snapshots` tests.
  - confidence: 0.86
