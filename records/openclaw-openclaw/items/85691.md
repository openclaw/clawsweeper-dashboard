## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve XML escaping when replacing prompt paths:** `src/commands/doctor-session-snapshots.ts:331-332`
  - body: skillsSnapshot.prompt stores locations produced by escapeXml(...), while the scanner decodes them before setting finding.cachedPath. Searching raw JSON for JSON.stringify(finding.cachedPath) will not match stored paths containing XML entities such as &amp;, so doctor --fix can still leave those stale prompt locations unrepaired.
  - confidence: 0.91
- **[P2] Handle Windows separators when repairing baseDir:** `src/commands/doctor-session-snapshots.ts:350`
  - body: On Windows, collectResolvedSkillPaths synthesizes the baseDir finding with a \SKILL.md suffix, but this branch only recognizes /SKILL.md. That leaves resolvedSkills[].baseDir pointed at the old runtime root even after filePath is repaired.
  - confidence: 0.89
- **[P2] Use the session-store atomic writer for repairs:** `src/commands/doctor-session-snapshots.ts:371-372`
  - body: This writes the active sessions.json directly after reading a raw snapshot. If doctor is interrupted or a gateway/session writer has newer data, this can truncate or clobber persistent session metadata; use the session-store update/atomic write path or an equivalent atomic replacement.
  - confidence: 0.86
