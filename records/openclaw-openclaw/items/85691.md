## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Serialize repairs through the session-store writer:** `src/commands/doctor-session-snapshots.ts:371-372`
  - body: This new repair branch writes sessions.json directly. Normal store mutations run through updateSessionStore/runExclusiveSessionStoreWrite, which serializes writes and refreshes caches; bypassing it during doctor --fix can overwrite concurrent Gateway updates or leave in-process state stale. Route the repair through the writer or make it explicitly offline-only.
  - confidence: 0.86
- **[P2] Preserve XML-escaped prompt locations:** `src/commands/doctor-session-snapshots.ts:331-338`
  - body: The scanner decodes <location> text before returning finding.cachedPath, but prompt locations are XML-escaped when stored. If a path contains &, <, >, quotes, or apostrophes, this replacement searches for a decoded string that is not in the raw JSON, leaving the stale prompt path unrepaired. Repair the parsed prompt or XML-escape before raw replacement.
  - confidence: 0.88
- **[P2] Handle Windows baseDir suffixes:** `src/commands/doctor-session-snapshots.ts:350`
  - body: On Windows, collectResolvedSkillPaths appends SKILL.md with path.join, so a baseDir finding can end in \SKILL.md. This guard only matches /SKILL.md, so the raw baseDir value is not repaired even though the PR says it is. Derive the directory with a separator-aware helper.
  - confidence: 0.82
