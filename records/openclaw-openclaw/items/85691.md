## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Route the rewrite through the session-store writer:** `src/commands/doctor-session-snapshots.ts:397`
  - body: This writes `sessions.json` with `writeTextAtomic` after a raw read, bypassing `runExclusiveSessionStoreWrite`/`updateSessionStore` and the session-store caches. Existing doctor repairs that mutate sessions use `updateSessionStore`; this path can race with Gateway or another doctor repair and overwrite or be overwritten by queued session mutations. Add a session-store-owned raw rewrite helper or mutate through the writer so caches are refreshed or dropped.
  - confidence: 0.88
- **[P2] Rescan before returning a repair success:** `src/commands/doctor-session-snapshots.ts:412`
  - body: The function returns after any store changed. If one encoded form is missed, the remaining stale findings are hidden from the normal warning and the user sees only a success note. Reload and rescan after writes, then report any leftovers instead of returning unconditionally.
  - confidence: 0.84
- **[P2] Replace the mirror script with production tests:** `repro-doctor-session-snapshot-repair.mjs:1`
  - body: The added root script is not part of the Vitest suite and duplicates the repair algorithm instead of exercising `noteSessionSnapshotHealth({ shouldRepair: true })`. Move these scenarios into `src/commands/doctor-session-snapshots.test.ts` so CI covers the real doctor path, then remove the root proof artifact.
  - confidence: 0.86
