Full review comments:

- **[P1] Normalize the session store before writing:** `src/commands/doctor-session-snapshots.ts:397`
  - body: This writes the raw string-replaced `fixed` text back to `sessions.json`. When the stale path comes from a legacy `skillsSnapshot.resolvedSkills` payload, that preserves the runtime-only resolved skill cache that current session-store normalization deliberately strips, leaving `doctor --fix` to rewrite a bloated legacy store instead of bringing it back to the supported persisted shape. Parse the repaired JSON and save through the existing session-store normalization/write path before writing.
  - confidence: 0.89
- **[P2] Use the real resolvedSkills nesting in proof:** `repro-doctor-session-snapshot-repair.mjs:54`
  - body: The proof script checks `session.resolvedSkills` and builds the fixture with root-level `resolvedSkills`, but the production scanner reads `entry.skillsSnapshot.resolvedSkills`. This means the script still does not prove the real baseDir repair path; make the fixture match the current `SessionEntry` shape or replace it with a focused Vitest regression.
  - confidence: 0.92
- **[P2] Make the XML fixture actually exercise entity escaping:** `repro-doctor-session-snapshot-repair.mjs:277`
  - body: The XML scenario claims to test `&amp;`, but `xmlStaleRoot` contains no `&`, `<`, `>`, quotes, or apostrophes, so the escaped value is the same as the raw path and the scenario would pass without the XML-specific branch. Add a character that is actually entity-escaped in the stored prompt.
  - confidence: 0.88
