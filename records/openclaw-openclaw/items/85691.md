- **[P1] Normalize the session store before writing:** `src/commands/doctor-session-snapshots.ts:397`
  - body: This writes the raw string-replaced JSON back to `sessions.json`, so a legacy store that contains `skillsSnapshot.resolvedSkills` will still persist that runtime-only cache after doctor claims it repaired the store.   - confidence: 0.9

That means the baseDir scenario can pass without exercising the real persisted session shape this PR repairs; move the fixture under `skillsSnapshot.resolvedSkills` or replace the script with a focused Vitest case against `noteSessionSnapshotHealth`.
  - confidence: 0.91

  - body: This scenario says it proves an `&amp;` path, but `xmlStaleRoot` contains no XML-special character, so the replacement calls are no-ops and the test is effectively another raw path replacement. Include a stale path segment with `&` or another escaped character so the XML branch is really covered.
