## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Handle XML-escaped prompt paths:** `src/commands/doctor-session-snapshots.ts:331-332`
  - body: `extractSkillLocations()` decodes XML entities before creating findings, but persisted skill prompts are written with XML-escaped `<location>` values. If a stale path contains `&`, `<`, quotes, or apostrophes, this repair searches for the decoded JSON string instead of the escaped text in `sessions.json`, so `doctor --fix` can leave the stale prompt path in place.
  - confidence: 0.9
- **[P2] Handle Windows baseDir suffixes:** `src/commands/doctor-session-snapshots.ts:350`
  - body: This branch only repairs `resolvedSkills.baseDir` when the synthesized cached path ends with `/SKILL.md`. On Windows, `path.join(baseDir, "SKILL.md")` uses `\SKILL.md`, so the `baseDir` rewrite is skipped and the next scan can still report the stale directory even if `filePath` was repaired.
  - confidence: 0.88
- **[P2] Use atomic session-store writes:** `src/commands/doctor-session-snapshots.ts:372`
  - body: The repair writes `sessions.json` directly with `fs.writeFileSync`, bypassing the session-store writer and atomic persistence path used by current session mutations. A process interruption can leave the live store partially written, and same-process session-store cache state will not be updated after the raw text rewrite.
  - confidence: 0.84
