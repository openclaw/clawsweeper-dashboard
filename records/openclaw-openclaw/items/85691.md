## Summary

## What This Changes

This PR adds `shouldRepair` handling to session snapshot health checks so `openclaw doctor --fix` rewrites stale cached snapshot paths in session stores and passes the repair flag from the doctor health flow.

## Best Possible Solution

Keep the `shouldRepair` wiring, but repair via the parsed session store or JSON-escaped values with focused POSIX and Windows regression coverage plus redacted after-fix `doctor --fix` proof.

## Reproduction Assessment

Yes from source inspection, though I did not run a full current-main live repro. Current main never passes `shouldRepair` into `noteSessionSnapshotHealth`, so stale session snapshot findings can only be reported, not repaired.

## Solution Assessment

No, not yet. Wiring repair mode is the right direction, but replacing parsed paths directly in raw JSON is not the narrow maintainable repair because it misses JSON-escaped values that the detector already supports.
