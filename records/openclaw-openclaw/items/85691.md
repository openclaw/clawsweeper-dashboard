## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Avoid persisting scan-only promptRef cleanup:** `src/commands/doctor-session-snapshots.ts:480-483`
  - body: The repair projects and later writes the whole hydrated store. `loadSessionStoreForSnapshotScan()` calls `hydrateSessionStoreSkillPromptRefs()`, which deletes `skillsSnapshot` when a promptRef blob is missing or invalid; if any other session in the same store has that state, repairing one stale path will persist the unrelated deletion. Use the canonical session-store writer/normalization or keep persistence scoped to the sessions actually changed by the repair.
  - confidence: 0.86
- **[P2] Normalize repaired stores before serializing:** `src/commands/doctor-session-snapshots.ts:489-490`
  - body: This serializes `persisted` directly, but the normal save path runs `normalizeSessionStore()`, which strips `skillsSnapshot.resolvedSkills` because it is an in-turn cache. A `doctor --fix` run that repairs a resolvedSkills path will now keep that bulky runtime cache on disk instead of removing it. Route the repair through the canonical save path or normalize before writing.
  - confidence: 0.81
