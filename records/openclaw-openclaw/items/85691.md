- **[P1] Keep the snapshot health params compatible:** `src/flows/doctor-health-contributions.ts:414`
  - body: `noteSessionSnapshotHealth` is retyped to require `config`, but the doctor flow still calls it with `cfg`. That means the PR head either fails type-checking or reaches this function with `params.config` undefined, so `doctor --fix` cannot reliably run the new repair path.
  - confidence: 0.96
- **[P1] Do not append skills to the bundled skills root:** `src/commands/doctor-session-snapshots.ts:269`
  - body: `resolveBundledSkillsDir()` already returns the live `.../skills` directory, but this replacement adds another `/skills`. A stale `.../node_modules/openclaw/skills/doctor/SKILL.md` would be repaired to `.../skills/skills/doctor/SKILL.md`, leaving sessions with a bad snapshot path.
  - confidence: 0.94
- **[P2] Keep scanning resolvedSkills filePath and baseDir:** `src/commands/doctor-session-snapshots.ts:53`
  - body: The current scanner detects stale `resolvedSkills.filePath` and `resolvedSkills.baseDir` values, and the linked bug explicitly needs baseDir repair. Mapping only `entry.path` skips the persisted resolvedSkills fields used by existing sessions, so `doctor --fix` would miss part of the stale snapshot metadata.
  - confidence: 0.92
- **[P2] Hydrate prompt refs before scanning stores:** `src/commands/doctor-session-snapshots.ts:254`
  - body: The current implementation hydrates blobbed skill prompts before scanning, and current tests cover stores where the raw JSON has `promptRef` instead of inline `prompt`.
