## Summary

Keep open: the reported doctor bug is valid and the repair direction is useful, but this PR is not merge-ready because it can report repair success without proving no stale session paths remain, and its proof/test coverage is a standalone mirror script rather than the actual `doctor --fix` path.

## What This Changes

The PR adds a `shouldRepair` mode to session snapshot health, wires `doctor --fix` into it, rewrites stale cached skill paths in `sessions.json` with backup and atomic writes, and adds a standalone root reproduction script.

## Best Possible Solution

Land a production-backed doctor repair that atomically rewrites stale session snapshot paths, creates a private backup, revalidates and reports any remaining stale paths, and is covered by production Vitest plus redacted after-fix `doctor --fix` proof.

## Reproduction Assessment

Yes, from source and the linked live output: current main detects stale session snapshot paths but `runSessionSnapshotsHealth` never passes repair mode, so `doctor --fix` cannot repair them. I did not run a local repro because this was a read-only review.

## Solution Assessment

No, not yet. The repair direction is narrow, but the mergeable solution should exercise the production doctor path, re-scan/report leftovers, and avoid carrying a root mirror script as the main proof artifact.
