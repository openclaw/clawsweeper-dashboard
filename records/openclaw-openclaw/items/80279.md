## What This Changes

The issue reports that auth-profile usage updates rewrite `auth-profiles.json`, changing the model-cache fingerprint and forcing repeated `ensureOpenClawModelsJson` cache misses under plugin-backed provider traffic.

## Best Possible Solution

Converge review and validation on https://github.com/openclaw/openclaw/pull/73260, carrying this issue's live 2026.5.7 timing evidence into that PR if maintainers want it preserved.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: warm `ensureOpenClawModelsJson`, complete a plugin-backed successful run that calls `markAuthProfileUsed`, and the next resolution sees a changed `auth-profiles.json` mtime because the save path rewrites that file.

## Solution Assessment

Yes. The content-hash/drift approach in the canonical PR is the best current fix path because it removes volatile mtime invalidation while preserving meaningful auth-profile and `models.json` change detection.
