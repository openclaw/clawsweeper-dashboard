## Summary

## What This Changes

The PR rewrites `openclaw channels list` to emit channel catalog/status rows instead of auth-provider and usage output, with updated command tests and one help example.

## Best Possible Solution

Exclude shared channel meta keys such as `modelByChannel`, add regression coverage for that config shape, update stale help/docs/changelog text, and provide a real `openclaw channels list` output transcript before merge.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: a config containing `channels.modelByChannel` will pass the PR's configured-channel filter because only `defaults` is excluded.

## Solution Assessment

No, not yet; the channel-only list approach is maintainable, but the patch should reuse the existing shared-key treatment and finish the user-facing help/docs/changelog cleanup.

Because this filter only skips `defaults`, users with model overrides will see a bogus `modelByChannel: configured, enabled, not installed` row in both text and JSON output. Add `modelByChannel` to the ignored set or reuse the existing shared-key helper, and cover it in the list tests.
  - confidence: 0.92

Update or remove those strings so `openclaw channels list --help` matches the new behavior.
  - confidence: 0.85
