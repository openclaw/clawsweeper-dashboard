Full review comments:

- **[P1] Keep unconfigured rows behind --all:** `src/commands/channels/list.ts:171-181`
  - body: This loop inserts every bundled channel into `channels list` output for every invocation, so the default text and JSON output no longer remain configured-only. Current docs and current `main` preserve that compatibility by showing catalog/installable rows only when `--all` is passed.
  - confidence: 0.9
- **[P2] Use the catalog install probe:** `src/commands/channels/list.ts:199-209`
  - body: Catalog entries mark `installed` from `Boolean(plugin)`, but read-only plugin loading can omit a catalog plugin that is installed on disk but not configured. Use the catalog install probe, or WeCom-class channels will still render as not installed or get the wrong JSON origin.
  - confidence: 0.92
- **[P2] Remove or honor the usage option:** `src/commands/channels/list.ts:148`
  - body: The command now discards `opts.usage`, while `--no-usage` still exists in the branch's CLI/fast-route/docs surfaces. That leaves scripts with an accepted flag that no longer controls anything, so the flag must be removed everywhere or the behavior must stay wired.
  - confidence: 0.9
- **[P2] Ignore shared channel config keys:** `src/commands/channels/list.ts:104`
  - body: `NON_CHANNEL_CONFIG_KEYS` only excludes `defaults`, so a config with `channels.modelByChannel` can produce a fake channel row. Reuse the shared channel-config key treatment so model override config is not listed as a channel.
  - confidence: 0.86
