## Summary

Keep open. Current main still does not ship the requested official all-platform desktop app: OpenClaw has a macOS companion app, iOS/Android node apps, a browser/PWA Control UI, and Gateway/CLI support on Windows/Linux, while native Windows and Linux companion apps are explicitly documented as planned work.

## What This Changes

The issue requests an official cross-platform OpenClaw desktop app similar to Codex.

## Best Possible Solution

Keep this as a platform roadmap item, define the desktop product model and security boundary, then split it into narrower platform-specific implementation issues once maintainers choose native apps, a packaged web shell, or another companion-app direction.

## Reproduction Assessment

Not applicable as a feature request. The verification path is current-main source and docs inspection: platform docs and the apps tree show macOS/mobile/PWA surfaces but no official all-platform desktop app, with Windows and Linux companion apps still planned.

## Solution Assessment

Unclear. The request identifies a real missing product surface, but the best maintainable solution needs product design first because native cross-platform apps, a packaged PWA, and separate platform companion apps carry different security, packaging, and maintenance costs.
