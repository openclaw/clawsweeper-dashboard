## Summary

Keep open. Current main still does not provide persistent ACPX stderr artifacts: OpenClaw delegates subprocess stderr handling to the published acpx runtime, exposes no stderr artifact/config surface, and the related artifact PR remains open with product/security proof blockers.

## What This Changes

Request persistent per-subprocess ACPX stderr logs for post-mortem debugging after successful initialization.

## Best Possible Solution

Define a bounded stderr/run-artifact contract in `openclaw/acpx`, then expose only the needed OpenClaw plugin config/docs surface with redaction, permissions, retention, and real ACPX smoke proof.

## Reproduction Assessment

Yes. Source inspection is enough: current main passes no stderr artifact option into `AcpxRuntime`, and the pinned `acpx@0.7.0` runtime only preserves bounded startup stderr plus verbose mirroring.

## Solution Assessment

No. The user-visible need is valid, but the always-on raw tee sketched in the issue is not the safest merge shape; the maintainable solution needs an upstream artifact contract with redaction, permissions, retention, and clear plugin config semantics.
