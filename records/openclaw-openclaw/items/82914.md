## Summary

Keep open: the thread timestamp fallback is useful, but this PR does not yet make the reported Slack assistant status appear because interactive follow-up turns run through heartbeat typing and the Slack channel plugin still lacks a heartbeat status adapter. The PR also has only unit-test proof, not after-fix real Slack behavior proof.

## What This Changes

The PR adds a Slack block-action fallback from `container.thread_ts` to `message.thread_ts` and a unit test that forwards that thread timestamp into the queued interaction event.

## Best Possible Solution

Finish the Slack plugin fix by preserving `message.thread_ts` and adding heartbeat typing/status support that calls and clears Slack `assistant.threads.setStatus` for thread-targeted interactive follow-up turns.

## Reproduction Assessment

Yes, source-level: current main drops a block-action thread when Slack provides it only as `message.thread_ts`, and the PR's new unit case exercises that input shape. A live Slack workspace reproduction was not provided or run.

## Solution Assessment

No: the fallback is necessary but not sufficient for the claimed visible assistant status fix. The maintainable solution should also wire Slack heartbeat typing/status because heartbeat callbacks currently require a channel `heartbeat.sendTyping` adapter that Slack does not define.
