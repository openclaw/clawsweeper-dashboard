## Summary

## What This Changes

The PR dispatches internal `command:daily`/`command:idle` and `before_reset` hooks during lazy daily/idle session rollovers, widens bundled `session-memory` to consume those actions, and updates docs, changelog, and tests.

## Best Possible Solution

Land one canonical implementation that persists bundled session-memory exactly once for automatic daily and idle rollovers, derives the reset reason from the same freshness facts that caused rollover, documents the chosen hook contract, and includes focused tests plus real rollover proof.

## Reproduction Assessment

Yes, by source inspection. Current main can roll over a stale existing session while bundled `session-memory` only accepts `command:new` and `command:reset`, so automatic daily/idle rollovers do not trigger that hook.

## Solution Assessment

No, not yet. The direction addresses the right gap, but the PR must align stale reason derivation with freshness evaluation and needs maintainer selection of the public hook contract before it is the best merge path.
