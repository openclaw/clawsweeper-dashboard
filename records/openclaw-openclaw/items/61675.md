## What This Changes

The branch fires internal reset hooks and before_reset for automatic daily/idle session rollovers, widens bundled session-memory to accept those rollover actions, and updates docs, changelog, and tests.

## Best Possible Solution

Land one maintainer-approved implementation that persists session-memory exactly once for automatic daily and idle rollovers, derives the reset reason from the same freshness facts that caused rollover, preserves current session-memory safety behavior, and includes focused tests plus real proof.

## Reproduction Assessment

Yes, by source inspection. Current main can roll a stale session and archive the previous transcript while only running session_start/session_end, and bundled session-memory only accepts command:new/command:reset.

## Solution Assessment

No, not yet. The direction is useful, but the branch should first resolve the hook-contract choice, align reason derivation with freshness evaluation, rebase cleanly, and provide real rollover proof.
