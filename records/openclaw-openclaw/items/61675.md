## Summary

Keep open: this PR targets a real source-reproducible gap in automatic session rollover memory capture, but it still lacks real behavior proof, has a stale-reason mismatch that can skip the new hooks for valid daily rollovers, and overlaps an open competing hook-contract implementation at https://github.com/openclaw/openclaw/pull/71792.

## What This Changes

The PR adds lazy daily/idle session rollover dispatch for internal command hooks and before_reset, widens bundled session-memory metadata and filtering, and adds docs, changelog, and regression tests.

## Best Possible Solution

Choose one auto-reset hook contract, derive the reset reason from the same freshness inputs that trigger rollover, preserve before_reset only if that contract is approved, and require real daily/idle memory proof before merge.

## Reproduction Assessment

Yes, by source inspection. Current main rolls stale sessions through initSessionState without triggerInternalHook/runBeforeReset, while bundled session-memory only persists command:new and command:reset events.

## Solution Assessment

No. The PR touches the right path, but the reset reason must be derived from the same freshness inputs that trigger rollover and maintainers still need to choose the public hook contract before this is the best fix.
