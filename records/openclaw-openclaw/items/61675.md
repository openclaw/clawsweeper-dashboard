## Summary

Keep open. The PR targets a real session-memory persistence gap on automatic daily/idle rollover, but the patch still has a correctness bug in daily reset reason derivation, lacks artifact-backed real behavior proof, and overlaps with another open implementation path that maintainers need to choose between.

## What This Changes

The PR wires daily/idle automatic session rollovers into internal command hooks and before_reset, widens bundled session-memory to accept daily/idle actions, and updates docs, changelog, and regression tests.

## Best Possible Solution

Land one maintainer-approved implementation that persists session-memory exactly once for automatic daily and idle rollovers, aligns reset reason calculation with freshness evaluation, preserves manual reset behavior, and documents the chosen hook contract.

## Reproduction Assessment

Yes, by source inspection. Current main captures the prior session on stale daily/idle rollover and then only runs session_start/session_end, while bundled session-memory only persists command:new and command:reset.

## Solution Assessment

No, not as-is. The direction is plausible, but it must align lazy reset reason derivation with the freshness evaluator and maintainers need to choose it over the overlapping implementation in https://github.com/openclaw/openclaw/pull/71792.
