The PR adds lazy daily/idle reset internal hook dispatch, fires plugin `before_reset` for automatic rollovers, widens bundled `session-memory` to `command:daily` and `command:idle`, and adds docs, changelog, and regression tests.

## Best Possible Solution

Land one maintainer-approved implementation that persists session-memory exactly once for automatic daily and idle rollovers, preserves manual reset and lifecycle semantics, and documents the chosen hook contract.

## Reproduction Assessment
