## Best Possible Solution

Land a rebased branch only if maintainers accept command:daily/command:idle plus lazy before_reset as the hook contract and the PR includes real rollover proof, with release notes handled outside CHANGELOG.md.

## Reproduction Assessment

Unclear as a product decision: the patch is a narrow practical repair for session-memory, but it deliberately makes daily/idle rollovers visible as command hook actions. A typed lifecycle hook design would be cleaner if maintainers do not want non-command reset causes on the command surface.
