## What This Changes

The issue reports slow embedded-agent startup and prep before streaming on a fast local OpenAI-compatible backend, plus a possible Telegram `getMe` health/probe path that does not consistently honor `channels.telegram.proxy`.

## Best Possible Solution

Keep this open and split it into two maintainable tracks: measure and optimize the shared embedded-runner warm path with explicit invalidation boundaries, and separately reproduce the Telegram SOCKS `getMe` behavior on a host where direct Telegram egress fails but the configured proxy succeeds.
