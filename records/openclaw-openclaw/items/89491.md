## Best Possible Solution

Land the OpenAI plugin OAuth hardening once malformed-token failures use generic user-facing messages and the callback socket cleanup tests and proof stay in place.

## Reproduction Assessment

## Solution Assessment

Is this the best way to solve the issue? Not as written: the callback socket cleanup is in the right plugin runtime layer, but malformed token handling should not append parser detail because it can echo response bytes.
