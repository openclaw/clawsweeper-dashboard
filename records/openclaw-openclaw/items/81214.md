## Summary

Keep open: current main still routes isolated lightweight native subagent spawns through context-engine preparation, and the open linked fix PR already owns the implementation path for this issue.

## What This Changes

The issue reports that `sessions_spawn` with `runtime="subagent"`, `context="isolated"`, and `lightContext=true` regressed from a lightweight local Ollama subagent run into excessive context/payload expansion after OpenClaw 2026.5.7.

## Best Possible Solution

Review and land a focused fix through the linked PR that prevents inherited payload bloat for isolated lightweight native subagent spawns while preserving or deliberately revising the context-engine contract, with regression coverage and redacted real-runtime proof.

## Reproduction Assessment

No high-confidence end-to-end reproduction was established for the full Ollama CPU/timeout symptom because this review did not run a distributed local Ollama setup. Source inspection does reproduce the reported current-main path: `sessions_spawn` forwards `context` and `lightContext`, then isolated native subagent runs still invoke context-engine preparation before the lightweight child call.

## Solution Assessment

Unclear. The linked early-return proposal targets the suspect path, but current tests and exported types model isolated `prepareSubagentSpawn` as a supported lifecycle hook, so maintainers should confirm whether isolated mode should skip the hook entirely or use a no-inherit preparation path.
