## Summary

Keep open: current main and the latest release still run context-engine subagent preparation for isolated native subagent spawns, and the linked closing PR remains open rather than merged.

## What This Changes

The issue reports that isolated lightweight `sessions_spawn` subagent runs still invoke context-engine preparation, causing large payloads and local Ollama stalls in OpenClaw 2026.5.7.

## Best Possible Solution

Land the linked fix PR or a replacement that keeps isolated subagent spawns lightweight while preserving or deliberately updating the context-engine lifecycle contract, backed by redacted Ollama proof.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? No for the full live Ollama CPU/timeout symptom; source inspection does show the current-main path where `sessions_spawn` forwards `context`/`lightContext` and isolated spawns still enter context-engine preparation.

## Solution Assessment

Is this the best way to solve the issue? Unclear. The reported bypass targets the suspect path, but the maintainable fix should remove inherited payload bloat without silently breaking the documented `prepareSubagentSpawn` lifecycle contract.
