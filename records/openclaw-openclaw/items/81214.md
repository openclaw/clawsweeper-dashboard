## Summary

Keep open: current main still has the source path implicated by the report, and the issue is already paired with an open fix PR that uses closing syntax. It should not close until that PR, or an equivalent reviewed fix, lands.

## What This Changes

The issue asks OpenClaw to keep isolated lightweight native subagent runs from invoking heavyweight context-engine preparation or inheriting large context payloads with local Ollama runtimes.

## Best Possible Solution

Review and land, or replace, https://github.com/openclaw/openclaw/pull/81242 with a focused fix that prevents heavyweight inherited context for isolated `lightContext` native spawns while preserving or deliberately updating the context-engine lifecycle contract.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? No for the full distributed Ollama CPU/timeout symptom; the current-main source path is clear because `lightContext` is passed through while context-engine preparation still runs for isolated native spawns.

## Solution Assessment

Is this the best way to solve the issue? Unclear for the exact bypass patch: skipping all isolated preparation may be too broad, so the safer solution should avoid heavyweight inherited payload for isolated `lightContext` runs while preserving or explicitly revising the hook contract.
