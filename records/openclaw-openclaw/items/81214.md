## What This Changes

The issue reports that isolated lightweight native subagent runs still invoke context-engine spawn preparation, causing inherited context expansion and local Ollama runtime instability.

## Best Possible Solution

Keep this issue open while maintainers review or replace https://github.com/openclaw/openclaw/pull/81242 with a fix that prevents inherited payload bloat for isolated lightweight spawns, preserves the context-engine lifecycle contract where needed, and adds regression plus redacted runtime proof.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? No for the full multi-host Ollama CPU/timeout symptom, but yes at source level for the implicated current-main path: isolated context is selected and context-engine preparation still runs afterward.

## Solution Assessment

Is this the best way to solve the issue? Not as a blanket bypass of all isolated preparation; the safer fix is to stop the inherited payload behavior for isolated lightweight native spawns while preserving or deliberately updating the documented `prepareSubagentSpawn` contract.
