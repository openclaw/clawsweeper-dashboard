## Summary

Keep open: current main still reaches context-engine subagent preparation for isolated lightweight native subagent spawns, and the linked open fix PR already owns the implementation path but has not merged.

## What This Changes

The issue reports a regression where `sessions_spawn` with `runtime="subagent"`, `context="isolated"`, and `lightContext=true` can still invoke context-engine preparation and overload local Ollama subagents.

## Best Possible Solution

Review and land a focused fix through https://github.com/openclaw/openclaw/pull/81242, preserving or intentionally revising the context-engine lifecycle contract and requiring redacted real-runtime proof.

## Reproduction Assessment

No for the full live Ollama CPU/timeout behavior because this review did not exercise a distributed local Ollama setup. Source inspection does show the reported current-main path: `sessions_spawn` forwards `context`/`lightContext`, and isolated subagent runs still enter context-engine preparation.

## Solution Assessment

Unclear. The linked early-return proposal targets the suspect call path, but current tests, docs, and exported types make isolated `prepareSubagentSpawn` a supported hook; the safer fix should avoid inherited payload bloat while preserving or deliberately updating that contract.
