## Summary

Keep open: current main and v2026.5.12 still route isolated native subagent spawns through context-engine preparation, and the open closing PR is the active fix path. The source path matches the reporter's mitigation, but the full distributed Ollama CPU/timeout behavior was not live-reproduced in this read-only review.

## What This Changes

The issue asks OpenClaw to keep isolated/lightContext native subagent spawns lightweight by avoiding inherited context-engine payload bloat with local Ollama runtimes.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/81242 with a fix that keeps isolated lightweight spawns from inheriting heavy context while preserving or intentionally revising the context-engine lifecycle contract, backed by redacted live Ollama proof.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? No for the full live Ollama CPU/timeout behavior, because this read-only review did not exercise a distributed local Ollama setup. Source inspection does show the reported current-main path: `sessions_spawn` forwards `context` and `lightContext`, and isolated subagent runs still enter context-engine preparation.

## Solution Assessment

Is this the best way to solve the issue? Unclear. The linked early-return proposal targets the suspect path, but current tests, docs, and exported types make isolated `prepareSubagentSpawn` a supported hook, so the safer fix should avoid inherited payload bloat while preserving or deliberately revising that contract.
