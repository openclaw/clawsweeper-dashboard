## Summary

Keep open: current main and the latest release still invoke context-engine preparation for isolated lightweight native subagent spawns, and https://github.com/openclaw/openclaw/pull/81242 already owns a proposed fix path but remains open pending real-runtime proof and contract review.

## What This Changes

The issue reports a 2026.5.7 regression where `sessions_spawn` with `runtime="subagent"`, `context="isolated"`, and `lightContext=true` becomes heavyweight for local Ollama subagents because context-engine preparation still runs.

## Best Possible Solution

Land a focused fix that prevents inherited payload bloat for isolated lightweight native subagent spawns while preserving or intentionally revising the context-engine lifecycle contract, with updated tests and redacted real-runtime proof.

## Reproduction Assessment

No for the full Ollama CPU/timeout behavior because this review did not exercise a distributed local Ollama setup. Source inspection does show the reported current-main path: `sessions_spawn` forwards `context`/`lightContext`, and isolated subagent runs still enter context-engine preparation.

## Solution Assessment

Unclear. The linked early-return proposal targets the suspect call path, but current tests, docs, and exported types make isolated `prepareSubagentSpawn` a supported hook; the safer fix should either preserve a no-inherit preparation path or deliberately update that contract with maintainer review.
