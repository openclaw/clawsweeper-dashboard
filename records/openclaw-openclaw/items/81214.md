## Summary

Keep open: current main and the v2026.5.7 release still run context-engine subagent preparation for isolated native subagent spawns, and the linked PR is an open feature branch rather than a fix for this regression.

## What This Changes

Issue requests preserving lightweight isolated `sessions_spawn` subagent execution when `runtime="subagent"`, `context="isolated"`, and `lightContext=true` are used with local Ollama subagents.

## Best Possible Solution

Define lightweight isolated subagent semantics for context-engine preparation, then make the spawn path and context-engine contract enforce that boundary with regression coverage.

## Reproduction Assessment

No high-confidence live reproduction of the Ollama/lossless-claw CPU spike is available here. Source inspection does confirm the disputed current-main path: isolated lightweight native subagent spawns still run context-engine preparation before child startup.

## Solution Assessment

Unclear. The reporter's bypass is directionally plausible, but the safest fix should preserve the documented context-engine lifecycle by scoping the skip or passing lightweight intent explicitly rather than silently disabling all isolated preparation.
