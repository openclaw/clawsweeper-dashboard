## Summary

Keep open: current main and the latest release still invoke context-engine subagent preparation for isolated native subagent spawns, and the open closing PR is the active fix path but has not merged.

## What This Changes

The issue reports a 2026.5.7 regression where native `sessions_spawn` with `context="isolated"` and `lightContext=true` still enters context-engine preparation, causing local Ollama payload expansion, CPU spikes, fallback logs, and stalls.

## Best Possible Solution

Fix isolated subagent payload bloat through the linked PR or a replacement that keeps isolated spawns lightweight while preserving or deliberately revising the context-engine lifecycle contract, backed by redacted live Ollama proof.

## Reproduction Assessment

No for the full live Ollama CPU/timeout behavior because this review did not exercise a distributed local Ollama setup. Source inspection does show the reported current-main path: `sessions_spawn` forwards `context` and `lightContext`, and isolated subagent runs still enter context-engine preparation.

## Solution Assessment

Unclear. The linked early-return proposal targets the suspect path, but current tests, docs, and exported types make isolated `prepareSubagentSpawn` a supported hook, so the safer fix should avoid inherited payload bloat while preserving or intentionally updating that contract.
