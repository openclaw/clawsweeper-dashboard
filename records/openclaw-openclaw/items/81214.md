## Summary

Keep open: current main and the latest release still run context-engine spawn preparation for isolated native subagent spawns, and the linked open fix PR explicitly closes this issue but has not merged.

## What This Changes

The issue reports a regression where `sessions_spawn` with `runtime="subagent"`, `context="isolated"`, and `lightContext=true` still invokes context-engine preparation and becomes heavy for local Ollama subagents.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81242 or an equivalent focused fix that keeps isolated/lightweight subagent runs lightweight while preserving or deliberately revising the context-engine lifecycle contract.

## Reproduction Assessment

No for a high-confidence full reproduction of the live Ollama CPU/timeout symptom, because this review did not run the distributed local-Ollama setup. Source inspection does confirm the reported current-main path: `sessions_spawn` forwards `context` and `lightContext`, and isolated native subagent runs still enter context-engine preparation.

## Solution Assessment

Unclear. The linked early-return PR matches the reporter's mitigation, but it bypasses a context-engine hook that current tests, docs, and exported types treat as valid for isolated mode; a safer fix should remove inherited payload bloat while preserving or intentionally updating that contract.
