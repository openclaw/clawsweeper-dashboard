## Summary

Keep open: current main and the latest release still run context-engine preparation for isolated native subagent spawns, and the linked fix PR remains open rather than merged.

## What This Changes

The issue reports a 2026.5.7 regression where isolated `sessions_spawn` subagents with `lightContext=true` still run context-engine preparation and overload local Ollama execution.

## Best Possible Solution

Keep the issue open until a focused fix lands that prevents inherited payload bloat for isolated lightweight native subagent spawns while preserving or explicitly revising the context-engine lifecycle contract.

## Reproduction Assessment

No for the full Ollama CPU/timeout behavior in a real deployment. Source inspection on current main does show `sessions_spawn` forwarding `lightContext` while isolated native spawns still run `prepareContextEngineSubagentSpawn`, matching the reporter's mitigation path.

## Solution Assessment

Unclear. The bypass proposed in the linked PR targets the right code path, but a broad skip changes the context-engine hook contract; the safer fix should preserve the hook or deliberately update the contract with tests and real runtime proof.
