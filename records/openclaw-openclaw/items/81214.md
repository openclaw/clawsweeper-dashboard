## Summary

Keep open: current main and the v2026.5.7 release still run context-engine subagent preparation for isolated native subagent spawns, matching the reported root-cause path. An open fix PR already references this issue, but it is not merged and has review/proof blockers, so this issue should remain open until a fix lands.

## What This Changes

Report a regression where isolated lightweight native subagent spawns with local Ollama/lossless-claw still invoke context-engine preparation, inflating context and causing CPU spikes or timeouts.

## Best Possible Solution

Land a focused fix that keeps isolated lightweight native spawns lightweight while preserving or deliberately updating the context-engine hook contract, with targeted tests and redacted real Ollama proof.

## Reproduction Assessment

No, not for the full Ollama CPU/timeout behavior in a live distributed setup. Source inspection of current main and v2026.5.7 does verify the implicated path: isolated native spawns still reach context-engine preparation while `lightContext` only affects bootstrap context.

## Solution Assessment

Yes, a narrow native subagent spawn fix is the right direction. The safer implementation should avoid heavyweight inherited payloads for isolated lightweight runs without bypassing all isolated context-engine lifecycle hooks unless maintainers intentionally change that contract.
