## Summary

Keep open: current main still routes isolated native subagent spawns through context-engine preparation, and an open linked fix PR is the active implementation path rather than a reason to close this issue.

## What This Changes

The issue requests fixing a 2026.5.7 regression where `sessions_spawn` with `runtime="subagent"`, `context="isolated"`, and `lightContext=true` still runs context-engine preparation and bloats local Ollama subagent runs.

## Best Possible Solution

Review or replace the linked fix PR so isolated `lightContext` subagent runs stay lightweight while preserving or deliberately updating the context-engine lifecycle contract.

## Reproduction Assessment

No, not for the full Ollama CPU/timeout symptom in a real deployment. Source inspection does show a current-main path where isolated native subagent spawns still invoke context-engine preparation after `lightContext` is passed through.

## Solution Assessment

No, the broad isolated-mode skip proposed in the linked PR is not clearly the best fix because current tests and docs treat isolated `prepareSubagentSpawn` as a valid hook. A safer solution should prevent inherited-payload bloat for lightweight isolated runs while preserving the hook or updating the contract intentionally.
