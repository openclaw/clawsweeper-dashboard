## Summary

Keep open: current main still sends isolated native subagent spawns through context-engine preparation, and the active closing fix path is the open PR at https://github.com/openclaw/openclaw/pull/81242 rather than an already-merged change.

## What This Changes

The issue reports a regression where isolated lightweight `sessions_spawn` runs with local Ollama subagents inherit context-engine payloads and stop behaving lightweight.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81242, or a replacement, with redacted real Ollama proof and a deliberate update or preservation of the context-engine lifecycle contract.

## Reproduction Assessment

No for the full live Ollama CPU and timeout behavior because this review did not exercise a distributed local Ollama setup. Source inspection does show the current-main path: `sessions_spawn` forwards `context` and `lightContext`, and isolated subagent runs still enter context-engine preparation.

## Solution Assessment

Unclear. The linked early-return proposal matches the reporter's mitigation, but current tests, docs, and exported types treat isolated `prepareSubagentSpawn` as a supported hook, so the safest fix should remove payload bloat without accidentally breaking that contract.
