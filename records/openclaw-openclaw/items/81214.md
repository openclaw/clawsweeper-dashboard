## What This Changes

The issue reports a 2026.5.7 regression where isolated lightweight native `sessions_spawn` subagents with local Ollama runtimes still run context-engine preparation and inherit heavy context.

## Best Possible Solution

Keep this issue open while maintainers review, revise, land, or close https://github.com/openclaw/openclaw/pull/81242; the eventual fix should prevent isolated/lightweight spawns from inheriting heavyweight context without accidentally breaking the context-engine hook contract.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? No, not for the full live Ollama CPU/timeout symptom; source inspection does give a clear current-main path where isolated lightweight native spawns still reach context-engine preparation.

## Solution Assessment

Is this the best way to solve the issue? Unclear for the currently linked patch: skipping all isolated preparation is directionally related to the report, but a safer fix should preserve or deliberately update the `prepareSubagentSpawn` contract.
