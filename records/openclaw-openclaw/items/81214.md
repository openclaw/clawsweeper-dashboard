## Summary

Keep open: current main still runs context-engine subagent preparation for isolated native spawns after `lightContext` is passed through, and an open linked fix PR still closes this issue, so cleanup should wait for that PR or a replacement to land.

## What This Changes

The issue reports a regression where `sessions_spawn` with `runtime="subagent"`, `context="isolated"`, and `lightContext=true` still prepares context-engine state, causing oversized payloads and local Ollama instability.

## Best Possible Solution

Keep the issue open until a focused fix lands that prevents inherited payload bloat for isolated lightweight native subagent spawns while preserving or deliberately revising the context-engine lifecycle contract.

## Reproduction Assessment

No, not for the full Ollama CPU/timeout behavior in a real deployment. Source inspection does show current main still reaches context-engine preparation for isolated `lightContext` native subagent spawns, matching the reporter's mitigation path.

## Solution Assessment

Yes, a narrow bug fix is the right direction, but an unconditional isolated-mode hook skip is not clearly the best solution because current tests/docs expose isolated preparation as valid lifecycle behavior. The safer fix should preserve the hook contract or update it deliberately with focused tests and real runtime proof.
