## What This Changes

The issue reports a 2026.5.7 regression where isolated lightweight native subagent spawns with local Ollama runtimes still run context-engine preparation and inherit heavy context.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81242 or a replacement that keeps isolated lightContext spawns lightweight while preserving or explicitly updating the context-engine lifecycle contract.

## Reproduction Assessment

No, not for the full live Ollama CPU/timeout behavior in this review. Source inspection does provide a clear current-main reproduction path for the implicated behavior: isolated native subagent spawns still call context-engine preparation even when `lightContext` is passed through.

## Solution Assessment

Unclear for the proposed linked fix as written. A narrow fix should avoid heavyweight inherited context for isolated lightweight spawns while preserving the existing `prepareSubagentSpawn` contract or changing that contract deliberately with tests and docs.
