Keep open: current main and the latest release still take the reported isolated/lightContext subagent path into context-engine preparation, and the open closing PR that owns the fix has not merged. ## What This Changes

The issue reports a 2026.5.7 regression where isolated lightweight native subagent spawns with local Ollama still invoke context-engine preparation and expand inherited context.

## Best Possible Solution

Review and land a focused fix through https://github.com/openclaw/openclaw/pull/81242 or a replacement that prevents inherited payload bloat for isolated/lightweight native subagents while preserving or deliberately updating the context-engine lifecycle contract and proving the local Ollama path.

## Solution Assessment

Unclear. The open early-return PR targets the suspect call path, but current tests, docs, and exported types make isolated prepareSubagentSpawn a supported hook; the safer fix should avoid inherited payload bloat while preserving or intentionally revising that contract.
