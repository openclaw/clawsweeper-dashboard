## What This Changes

Ensure isolated native `sessions_spawn` runs with `lightContext=true` stay lightweight for local Ollama subagents instead of inheriting heavyweight context-engine state.

## Best Possible Solution

Implement a narrow native spawn fix for `context="isolated"` plus `lightContext=true` that keeps the child lightweight without breaking normal isolated and forked context-engine hooks.

No, not for the full Ollama CPU/timeout symptom in a real deployment. Source inspection does show a high-confidence current-main path where `lightContext` is forwarded but context-engine subagent preparation still runs for isolated native spawns.

## Solution Assessment

Yes, a focused fix is the best direction if it is scoped to isolated lightweight native spawns. Broadly skipping all isolated `prepareSubagentSpawn` calls would be risky because the documented context-engine hook supports isolated and forked contexts.
