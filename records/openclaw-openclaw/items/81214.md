## Summary

Keep open: current main still prepares the active context engine for isolated native subagent runs even when `lightContext` is true, matching the reported root-cause path rather than an already-fixed item.

## What This Changes

The issue asks for isolated lightweight `sessions_spawn` native subagent runs with local Ollama/lossless-claw to avoid inherited context-engine preparation and embedded fallback work.

## Best Possible Solution

Teach native subagent spawning to honor `lightContext` when the resolved context is isolated, with regression coverage and any necessary context-engine contract documentation.

## Reproduction Assessment

No, not for the full Ollama CPU/timeout symptom in a real deployment. Source inspection does give a clear current-main path to the implicated behavior: `lightContext` is passed through, but context-engine subagent preparation still runs for isolated native spawns.

## Solution Assessment

Yes, a focused fix is appropriate if it preserves normal isolated and forked context-engine hooks. The safest shape is to skip or constrain context-engine preparation only for isolated lightweight native subagent runs, then add targeted tests around that boundary.
