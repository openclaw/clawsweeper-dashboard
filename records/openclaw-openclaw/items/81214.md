Keep open: current main still follows the reported source path, and the issue is now paired with an open unmerged fix PR that uses closing syntax. The right cleanup outcome is to keep this issue open until that PR lands or maintainers replace it with a safer fix.

## What This Changes

The report asks OpenClaw to keep isolated lightweight native subagent runs from inheriting large context-engine payloads when `sessions_spawn` uses local Ollama subagents.

## Best Possible Solution

Land or replace the open fix path with a scoped change that keeps isolated/lightweight native subagent spawns lightweight while preserving the context-engine hook contract or updating it deliberately.

## Reproduction Assessment

No, not for the full Ollama CPU/timeout symptom in a real distributed setup. Source inspection does give a clear current-main reproduction path for the implicated behavior: isolated native subagent spawns still run context-engine preparation even when `lightContext` is true.

## Solution Assessment

Yes, keeping this issue open is the right handling while the linked fix PR is open. The best implementation should be narrower than a broad isolated-hook bypass unless maintainers decide to change the plugin-facing context-engine contract.
