## What This Changes

The issue reports a regression where isolated `lightContext` native subagent spawns with local Ollama runtimes invoke context-engine preparation and inflate inherited payloads.

## Best Possible Solution

Land a focused fix through the linked PR or a replacement that prevents inherited payload bloat for isolated lightweight native subagent spawns while preserving or intentionally revising the context-engine lifecycle contract.

## Reproduction Assessment

No for the full Ollama CPU/timeout behavior in a real deployment. Source inspection does show current main forwarding `lightContext` while isolated native spawns still enter `prepareContextEngineSubagentSpawn`, matching the reporter's mitigation path.

## Solution Assessment

Unclear. The linked PR targets the implicated path, but the broad bypass conflicts with current tests and the exported context-engine hook contract; the safer fix should preserve the hook or explicitly update that contract with maintainer review and runtime proof.
