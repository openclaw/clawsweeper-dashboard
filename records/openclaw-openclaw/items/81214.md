## What This Changes

The issue reports that isolated lightweight native subagent spawns in OpenClaw 2026.5.7 still run context-engine preparation, causing heavyweight inherited context behavior with local Ollama subagents.

## Best Possible Solution

Keep this issue open while https://github.com/openclaw/openclaw/pull/81242 is reviewed; the final fix should avoid heavyweight inherited payloads for isolated lightweight spawns while preserving or deliberately revising the context-engine hook contract.

## Reproduction Assessment

No, not for the full Ollama CPU/timeout symptom in a real deployment. Source inspection does show current main still sends isolated native subagent spawns through context-engine preparation after `lightContext` is passed through.

## Solution Assessment

Unclear until maintainer review settles the context-engine hook contract. The linked fix direction is focused, but the current contract and tests expect isolated spawns to call `prepareSubagentSpawn`, so the implementation needs review rather than an unconditional bypass.
