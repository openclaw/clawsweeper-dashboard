## Summary

No blocking patch findings. Current main and v2026.6.8 still have the queued follow-up typing gap and Telegram room-event suppression gap, and this PR fixes them through the existing typing signaler and configured typingMode precedence with focused tests; remaining work is maintainer merge review, especially explicit acceptance of the additive SDK type export.

## What This Changes

The PR starts run-start typing for queued follow-up turns, lets explicit Telegram typingMode config override quiet room-event suppression, adds regression tests, and re-exports TypingMode from plugin-sdk/config-contracts.

## Best Possible Solution

Land the runtime/channel fix with explicit maintainer acceptance of the additive type-only SDK export, or remove that export and derive the Telegram helper type locally if maintainers want no SDK expansion.

Yes for the runtime/channel behavior: reusing the existing run-start typing signal and configured typingMode precedence is the narrowest maintainable fix. The only maintainer choice is whether the additive TypingMode SDK export should become public API or stay local to Telegram through a derived type.
