## Summary

Keep open. Current main has broad operational diagnostics, OTEL/Prometheus export, tool-execution events, and external-content hardening, but it does not have the requested first-class AI safety and quality event taxonomy; the issue is fresh and now has a concrete schema proposal that needs maintainer design rather than cleanup closure.

Request structured AI safety and quality diagnostics events plus exporter and operator surfaces for prompt-injection signals, external content, tool policy decisions, memory/source quality, and user feedback metadata.

## Best Possible Solution

Define a minimal additive event schema and exporter mapping first, then add emitters at existing owner boundaries with bounded metadata defaults and low-cardinality metric labels.
