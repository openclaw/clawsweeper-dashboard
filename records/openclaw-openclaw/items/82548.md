## Summary

Keep open. Current main has broad operational diagnostics, OTEL/Prometheus export, tool execution telemetry, and prompt-injection hardening, but it does not have the requested first-class AI safety and quality event taxonomy; the reporter's follow-up schema proposal makes this an active feature-design issue, not conservative cleanup material.

## What This Changes

Add structured AI safety and quality diagnostics events for prompt-injection signals, external content, tool-policy decisions, source quality, memory/context selection, and user feedback across existing logging and exporter surfaces.

## Best Possible Solution

Define a small additive diagnostic event schema and exporter mapping first, then add bounded-metadata emitters across external-content handling, tool policy, context/memory, source-quality, and feedback surfaces.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing behavior report. Source inspection gives a high-confidence current-state check that the requested AI safety/quality event taxonomy is not present on current main.

## Solution Assessment

Yes, the schema-first direction is a reasonable starting point, but it needs maintainer design for event names, privacy/cardinality policy, exporter mapping, and which emitters belong in core versus plugins.
