## Summary

Keep open. Current main has operational diagnostics, OTEL/Prometheus exporters, plugin hook gates, external-content wrapping/sanitization, and a Teams feedback transcript path, but it does not implement the requested first-class AI safety/quality telemetry taxonomy; the related open issues cover narrower pieces rather than superseding this broader core observability request.

## What This Changes

The issue requests structured AI safety and quality observability events across diagnostics, OpenTelemetry, logs, operator inspection, and user feedback surfaces.

## Best Possible Solution

Define a versioned, privacy-preserving core diagnostics taxonomy for AI safety and quality metadata, map it into existing OTEL/Prometheus/log surfaces with bounded labels, and coordinate scanner/blocking policy plus unified run-trace schema with the existing narrower trackers.

## Reproduction Assessment

Not applicable: this is a feature request for new telemetry and UX surfaces, not a broken existing behavior report. Source inspection gives a high-confidence gap check that the requested safety/quality event families are not present on current main.

## Solution Assessment

Unclear for the exact implementation. The diagnostics-backed direction is plausible, but taxonomy, privacy defaults, cardinality limits, feedback semantics, and exporter mappings need maintainer design before a maintainable implementation can be queued.
