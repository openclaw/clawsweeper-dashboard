## Summary

Keep open. Current main has broad operational diagnostics, OTEL/Prometheus exporters, external-content hardening, and tool-execution telemetry, but it still lacks the requested first-class AI safety and quality event taxonomy; the reporter also added a concrete schema proposal after the prior ClawSweeper review, so this remains an active feature-design thread rather than cleanup material.

## What This Changes

Add structured AI safety and quality observability events for prompt-injection exposure, external content, tool-policy decisions, memory/context selection, source quality, and user feedback.

## Best Possible Solution

Define a small additive AI diagnostics schema and exporter mapping first, then add bounded-metadata emitters at external-content, tool-policy, context/memory, source-quality, and feedback boundaries.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing behavior report. Source inspection gives a high-confidence current-state check that the requested AI safety/quality event taxonomy is not present on main.

## Solution Assessment

Yes for the schema-first direction, but the exact implementation needs maintainer decisions on names, trust semantics, privacy/cardinality policy, exporter mapping, and core-versus-plugin boundaries.

## Security Review

Status: needs_attention

Summary: Security-sensitive design review is needed because these telemetry events would describe prompt-injection exposure, untrusted content, tool decisions, and feedback without leaking raw private content.

Concerns:

- **[medium] Bound safety telemetry before export:** `docs/gateway/opentelemetry.md:126`
  - body: The docs explicitly keep raw prompts, responses, tool inputs, tool outputs, and identifiers out of default OTEL/Prometheus output; any AI safety event schema should preserve that boundary with bounded labels, hashes, and counts only.
  - confidence: 0.88
- **[medium] Keep Prometheus labels low-cardinality:** `docs/gateway/prometheus.md:123`
  - body: The Prometheus exporter documents a bounded label policy and series cap, so proposed fields such as source names, content hashes, run ids, message ids, and feedback text must not become metric labels.
  - confidence: 0.86
