## Summary

Keep open: the PR is a plausible narrow fix for a source-proven Telegram polling race, but it still lacks after-fix real Telegram proof and currently includes malformed changelog artifacts that would ship literal debris.

## What This Changes

The PR makes Telegram polling transport acquisition async, awaits stale transport close on dirty rebuild, updates the polling-session caller/tests, and adds a changelog entry.

## Best Possible Solution

Land a cleaned-up Telegram plugin-scoped fix that serializes stale transport teardown before the next polling cycle starts, with focused tests and redacted live Telegram proof, then close https://github.com/openclaw/openclaw/issues/75852 on merge.

## Reproduction Assessment

Yes, as a source-level reproduction: current main marks the transport dirty on 409 or recoverable polling errors, then returns a replacement before the old transport close promise resolves. The linked reports provide Bot API stop/start and local-patch proof, though I did not rerun a live token.

## Solution Assessment

Yes on the code direction: awaiting stale `TelegramTransport.close()` inside the Telegram polling transport state is the narrow owner-module fix. This PR is not merge-ready until the malformed changelog lines are removed and real Telegram proof is added.
