## Summary

Keep this PR open: current main still lacks the canonical transcript fallback, and the PR is a useful narrow fix direction, but the patch needs repair-only mutation accounting before merge. It also needs the required changelog entry for a user-facing cleanup bug fix.

## What This Changes

The PR changes `sessions cleanup --fix-missing` to preserve and repair sessions whose stored `sessionFile` path is stale when the canonical `<sessionId>.jsonl` transcript exists.

## Best Possible Solution

Land a focused cleanup fix that preserves valid sessions when the canonical transcript exists, reports repaired `sessionFile` paths as mutations in both dry-run and applied JSON summaries, keeps true missing-transcript pruning unchanged, and records the user-facing fix in the active changelog.

## Reproduction Assessment

Yes. The issue is reproducible by creating a store entry whose `sessionFile` points to a missing file while the canonical `<sessionId>.jsonl` exists, then running `sessions cleanup --fix-missing`; current main only checks the stored-path result. The PR finding is also reproducible from the diff because the repair branch mutates `entry.sessionFile` but returns no repair count.

## Solution Assessment

No, not as-is. The canonical fallback is the right narrow direction and matches existing transcript-read ordering, but the implementation should return or otherwise track repaired paths so summary mutation fields stay accurate; a small result object such as `{ removed, repaired }` would keep the fix maintainable.
