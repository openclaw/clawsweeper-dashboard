## Summary

Keep open. The cleanup bug is still source-reproducible on current main, but this PR is not merge-ready because it is dirty against current main, targets the old command-local cleanup path, misses repair-mutation summary accounting, and lacks external real behavior proof.

## What This Changes

The branch modifies session cleanup and tests to keep and repair a session when a stale stored `sessionFile` is missing but the canonical `<sessionId>.jsonl` transcript exists.

## Best Possible Solution

Port the fallback-and-repair behavior into `src/config/sessions/cleanup-service.ts`, track repaired-path mutations in preview and applied summaries, keep true-missing pruning unchanged, and add focused service-level regression coverage.

## Reproduction Assessment

Yes, at source level. A store entry with a valid `sessionId`, missing stale `sessionFile`, and existing canonical transcript still reaches the single resolved-path cleanup check on current main and would be pruned.

## Solution Assessment

No, not as-is. The fallback-and-repair direction is the narrow maintainable fix, but this branch needs current-main porting plus repaired-path mutation accounting before it is safe to merge.
