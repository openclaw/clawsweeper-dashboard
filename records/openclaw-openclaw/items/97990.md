## Summary

Keep open: the PR now uses the shared bundle detector for ClawHub bundle records, but it still trusts bundle-family metadata before confirming the installed payload is truly bundle-only, which can bypass native package validation for dual-format installs.

## What This Changes

The branch changes update-time plugin payload checks to validate ClawHub bundle-plugin records with the shared bundle manifest detector/loader and adds regression tests for Claude, Codex, and Cursor bundle layouts.

## Best Possible Solution

Keep the bundle fix in the update payload checks, but classify payloads with the existing native-first contract: native package payloads keep package.json/entry/peer validation, and true bundle-only payloads use the shared bundle detector/loader.

## Solution Assessment

No. The PR is in the right updater paths and uses the right bundle loader for true bundles, but the best fix must preserve native package validation before treating a bundle-family install record as bundle-only.
