## Summary

Keep open: current main has partial post-tool retry, replay-safety, warning, liveness, and transcript persistence paths, but the central recovery problem remains unresolved because a successful tool action followed by failed assistant continuation still ends as an error/abandoned run rather than a deterministic session resume.

## What This Changes

## Best Possible Solution

Define the post-tool continuation recovery contract around durable session survival/resume first, then add lifecycle receipts as diagnostics and validate with a live Ollama/Gemma persistent-agent reproduction.

## Solution Assessment
