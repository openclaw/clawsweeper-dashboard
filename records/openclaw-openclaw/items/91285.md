## Summary

Keep open: current main has useful safety guards for incomplete post-tool turns and missing tool results, but it still does not prove the reported Ollama/Gemma persistent-agent continuation can resume from intact session state after a successful tool action fails to produce a response.

## What This Changes

The issue reports successful filesystem tool execution followed by failed assistant continuation in persistent local Ollama/Gemma agents and asks for deterministic recovery state after post-tool failure.

## Best Possible Solution

Define a core recovery contract that preserves or resumes the same session after post-tool continuation failure, with redacted tool-completion diagnostics as supporting evidence, then prove it against a live Ollama/Gemma persistent-agent scenario and focused embedded-run/gateway tests.
