The newest reporter update gives a narrower macOS/Ollama/Gemma failure boundary, and the related open PRs/issues cover adjacent subsets rather than a proven canonical fix.

## What This Changes

The issue reports successful tool execution followed by failed agent continuation or response delivery, leaving users without a durable recovery receipt for completed tool actions.

## Best Possible Solution

Design a core, privacy-aware durable recovery receipt stored in the canonical session/state layer, then expose sanitized last-tool and continuation-failure state through CLI/UI diagnostics while preserving the existing replay-safety warning.
