## Summary

Current main still lacks the requested first-run setup heads-up before the first chat handoff. The prior ClawSweeper comment’s keep-open conclusion still holds after rechecking current main at 542821cd: the classic onboarding finalizer, Crestodian startup message, and onboarding docs still do not warn that first-run provider probes or setup work may temporarily hit rate limits.

## What This Changes

Request a first-run setup notification warning users that background initialization may temporarily cause provider API rate-limit errors before normal chat begins.

## Best Possible Solution

Add a concise setup-time provider-limit heads-up in the classic onboarding finish flow, with matching focused tests and onboarding docs. Mirror it in Crestodian only where that path can directly hand a configured user into normal agent chat, while leaving provider runtime, auth, failover, cooldown, quota, and retry behavior unchanged.
