## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Route remote_trigger through guarded fetch:** `src/agents/tools/claw-compat-tools.ts:635-640`
  - body: `remote_trigger` performs a direct `fetch` on a model-supplied URL with caller-controlled method, headers, and body. This bypasses the existing SSRF-protected web fetch path and can reach localhost, metadata endpoints, or private-network services from any session with the coding-profile tool.
  - confidence: 0.96
- **[P1] Redact or allowlist config_compat reads:** `src/agents/tools/claw-compat-tools.ts:577-583`
  - body: When no path is supplied, `config_compat` returns the entire loaded runtime config, and dotted paths can select sensitive fields. That can expose provider keys, auth tokens, webhook secrets, or auth settings to model-visible tool results.
  - confidence: 0.95
- **[P1] Gate config_compat writes behind owner authorization:** `src/agents/tools/claw-compat-tools.ts:551-555`
  - body: Passing `setting` plus `value` persists caller-supplied config through `writeConfigFile`, but this compat tool is not owner-only. Any session that can call it can mutate runtime configuration, including provider or permission-related settings.
  - confidence: 0.94
- **[P1] Gate plan-mode config writes behind owner authorization:** `src/agents/tools/claw-compat-tools.ts:238-239`
  - body: `enter_plan_mode` and `exit_plan_mode` persist `permissions.defaultMode` through broadly registered compatibility tools without owner-only metadata. Non-owner sessions could change control-plane permission state.
  - confidence: 0.92
- **[P2] Return unsupported until send_user_message dispatches:** `src/agents/tools/claw-compat-tools.ts:192`
  - body: `send_user_message` returns `delivered: true` but only echoes JSON and never calls the message/channel delivery path. Models using it will believe a user-facing message was sent when it was silently dropped.
  - confidence: 0.91
- **[P2] Preserve path-to-URI conversion for LSP aliases:** `src/agents/pi-embedded-runner/run/attempt.tool-call-normalization.ts:413-417`
  - body: The alias shim copies `path` into `uri`, but `createLspTool` treats supplied `uri` values as canonical and only converts `path` when `uri` is absent. A relative path like `src/main.ts` is sent as a non-URI document id.
  - confidence: 0.9
- **[P2] Resolve safe MCP server names before dispatch:** `src/agents/tools/mcp-tool.ts:151-153`
  - body: Qualified MCP names parse the prefix before `__` and pass it directly to `runtime.callTool`. Catalog entries expose sanitized safe server names while sessions are keyed by original server names, so catalog-driven calls can fail for sanitized servers.
  - confidence: 0.86
- **[P2] Use flat enum schemas for bridge actions:** `src/agents/tools/mcp-tool.ts:11`
  - body: The new bridge schemas use `Type.Union` over literal action values, which emits `anyOf`. Agent tool schemas should use the repo's flat string enum helpers so provider schema ingestion remains compatible.
  - confidence: 0.88
- **[P2] Preserve empty team arrays on update:** `src/agents/team-registry.ts:104-106`
  - body: `updateTeam` only applies `members`, `labels`, and `taskIds` when the patch arrays are truthy, and the tool normalizers turn empty arrays into `undefined`. Callers cannot clear existing team members, labels, or task links.
  - confidence: 0.84
- **[P1] Preserve LSP process cleanup on startup failures:** `src/agents/pi-bundle-lsp-runtime.ts:477-480`
  - body: Current main disposes a spawned LSP session when initialization fails, but this branch's catch block only logs and continues. Rebase/restack must keep that cleanup or repeated misconfigured LSP starts can leave child processes behind.
  - confidence: 0.82
