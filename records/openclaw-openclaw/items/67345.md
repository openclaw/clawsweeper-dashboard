## Summary

Keep open: this is a broad feature PR with high-confidence blocking correctness and security findings, no real behavior proof, and overlapping sandbox/parity work still open in https://github.com/openclaw/openclaw/pull/67333.

## What This Changes

The branch adds claw-code compatibility aliases, MCP/LSP/Hermes bridge tools, claw-compat utilities, task/team tools, sandbox/parity diagnostics, registry/catalog wiring, and tests.

## Best Possible Solution

Split the work into maintainer-approved slices, land any safe parity diagnostics separately, and gate compatibility tools behind existing network, config, owner-only, and provider-schema guardrails before merge.

## Reproduction Assessment

Yes for the blocking PR findings: PR-head source shows direct fetch, raw config read/write, ungated plan-mode config writes, false send_user_message delivery, LSP path rewriting, and unsafe MCP name dispatch. I did not run runtime tests because this review is read-only.

## Solution Assessment

No: the compatibility goal may be useful, but this PR is too broad and introduces security-sensitive tool surfaces before applying existing guardrails. A split, guardrail-first implementation is the safer maintainable path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Route remote_trigger through the guarded fetch path:** `src/agents/tools/claw-compat-tools.ts:635-640`
  - body: remote_trigger performs a direct fetch on a model-supplied URL with caller-controlled method, headers, and body. This bypasses the existing SSRF-protected web_fetch path, allowing agent calls to target localhost, metadata endpoints, or private-network services.
  - confidence: 0.96
- **[P1] Redact or allowlist config_compat reads:** `src/agents/tools/claw-compat-tools.ts:577-583`
  - body: When no path is supplied, config_compat returns the entire loadConfig object, and dotted paths can select sensitive fields. That can put provider keys, auth tokens, webhook secrets, or auth settings into model-visible tool results.
  - confidence: 0.95
- **[P1] Gate config_compat writes behind owner authorization:** `src/agents/tools/claw-compat-tools.ts:551-555`
  - body: Passing setting plus value writes persisted config with writeConfigFile, but this compat tool is not marked owner-only. Any session that can call it can mutate runtime configuration, including provider or permission-related settings.
  - confidence: 0.94
- **[P1] Gate plan-mode config writes behind owner authorization:** `src/agents/tools/claw-compat-tools.ts:238-239`
  - body: enter_plan_mode and exit_plan_mode persist permissions.defaultMode without an owner-only marker. In non-owner sessions this changes control-plane permission state through a broadly registered compatibility tool.
  - confidence: 0.92
- **[P2] Do not claim message delivery without dispatch:** `src/agents/tools/claw-compat-tools.ts:191-192`
  - body: send_user_message returns delivered: true but only echoes JSON and never calls the message/channel delivery path. Models using this tool will believe a user-facing message was sent when it was silently dropped.
  - confidence: 0.91
- **[P2] Preserve LSP paths for URI conversion:** `src/agents/pi-embedded-runner/run/attempt.tool-call-normalization.ts:413-417`
  - body: The alias shim copies path into uri, but createLspTool treats any supplied uri as already canonical and only converts path values when uri is absent. A relative path such as src/main.ts is sent to the language server as a non-URI document id.
  - confidence: 0.9
- **[P2] Resolve safe MCP server names before dispatch:** `src/agents/tools/mcp-tool.ts:62-68`
  - body: Qualified MCP names parse the prefix before __ and pass it directly to runtime.callTool. The catalog exposes sanitized safeServerName values while sessions are stored under original server keys, so catalog-driven calls can fail for sanitized server names.
  - confidence: 0.86
- **[P2] Preserve empty team update arrays:** `src/agents/tools/team-tool.ts:130-135`
  - body: team update converts empty members, labels, and task arrays to undefined, and updateTeam skips undefined fields. Callers cannot clear existing members, labels, or task links with an explicit empty list.
  - confidence: 0.84
- **[P2] Use flat enum schemas for bridge actions:** `src/agents/tools/mcp-tool.ts:9-12`
  - body: The new bridge schemas use Type.Union over literal action lists, which emits anyOf. Agent tool schemas should use the repo's flat string enum helpers so provider tool-schema ingestion remains compatible.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The PR introduces broadly available network, config-read, and config-write compatibility tools without the existing safety gates.

Concerns:

- **[high] Unrestricted outbound fetch in remote_trigger:** `src/agents/tools/claw-compat-tools.ts:635`
  - body: remote_trigger accepts model-supplied URLs and performs direct fetches without the existing SSRF guard, allowing requests to localhost, metadata endpoints, or private networks from sessions with the tool.
  - confidence: 0.96
- **[high] Raw runtime config exposure:** `src/agents/tools/claw-compat-tools.ts:577`
  - body: config_compat can return the full runtime config or arbitrary dotted paths into model context, risking exposure of inline provider keys, tokens, webhook secrets, or auth settings.
  - confidence: 0.95
- **[high] Unauthorized persisted config writes:** `src/agents/tools/claw-compat-tools.ts:551`
  - body: config_compat persists caller-supplied settings through writeConfigFile without owner-only authorization, turning normal tool access into control-plane mutation capability.
  - confidence: 0.94
- **[high] Unauthorized plan-mode permission writes:** `src/agents/tools/claw-compat-tools.ts:238`
  - body: The plan-mode compatibility tools persist permissions.defaultMode without owner-only gating, allowing non-owner sessions to modify permission state.
  - confidence: 0.92
