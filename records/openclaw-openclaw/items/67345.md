The PR adds claw-code compatibility aliases plus new MCP, LSP, Hermes, task/team, sandbox/parity, and claw-compat tools wired into core agent tool registration and tests.

## Best Possible Solution

Split and rebase this into smaller maintainer-reviewed changes: safe alias normalization, MCP/LSP bridge work, sandbox parity diagnostics coordinated with https://github.com/openclaw/openclaw/pull/67333, and compat tools only after guardrails, flat schemas, tests, and real behavior proof are in place.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Import Type from the declared typebox package:** `src/agents/tools/claw-compat-tools.ts:5`
  - body: The new tool files import `@sinclair/typebox`, but current `package.json` declares `typebox` and existing agent tools import from that package. As written, these modules can fail to resolve before the new tools load; update the new imports consistently.
  - confidence: 0.96
- **[P1] Route remote_trigger through the guarded fetch path:** `src/agents/tools/claw-compat-tools.ts:635-640`
  - body: `remote_trigger` performs a direct `fetch` on a model-supplied URL with caller-controlled method, headers, and body. This bypasses the existing SSRF-protected `web_fetch` path, allowing tool calls to target localhost, metadata endpoints, or private-network services.
  - confidence: 0.96
- **[P1] Redact or allowlist config_compat reads:** `src/agents/tools/claw-compat-tools.ts:577-583`
  - body: When no path is supplied, `config_compat` returns the entire `loadConfig()` object, and dotted paths can select sensitive fields. That can put provider keys, auth tokens, webhook secrets, or auth settings into model-visible tool results.
  - confidence: 0.95
- **[P1] Gate config_compat writes behind owner authorization:** `src/agents/tools/claw-compat-tools.ts:551-555`
  - body: Passing `setting` plus `value` writes persisted config with `writeConfigFile`, but this compat tool is not marked owner-only. Any session that can call it can mutate runtime configuration, including provider or permission-related settings.
  - confidence: 0.94
- **[P1] Gate plan-mode config writes behind owner authorization:** `src/agents/tools/claw-compat-tools.ts:238-239`
  - body: `enter_plan_mode` and `exit_plan_mode` persist `permissions.defaultMode` without owner-only metadata. In non-owner sessions this changes control-plane permission state through broadly registered compatibility tools.
  - confidence: 0.92
- **[P2] Do not claim message delivery without dispatch:** `src/agents/tools/claw-compat-tools.ts:192`
  - body: `send_user_message` returns `delivered: true` but only echoes JSON and never calls the message/channel delivery path. Models using this tool will believe a user-facing message was sent when it was silently dropped.
  - confidence: 0.91
- **[P2] Preserve LSP paths for URI conversion:** `src/agents/pi-embedded-runner/run/attempt.tool-call-normalization.ts:413-417`
  - body: The alias shim copies `path` into `uri`, but `createLspTool` treats any supplied `uri` as already canonical and only converts `path` values when `uri` is absent. A relative path such as `src/main.ts` is sent to the language server as a non-URI document id.
  - confidence: 0.9
- **[P2] Resolve safe MCP server names before dispatch:** `src/agents/tools/mcp-tool.ts:151-153`
  - body: Qualified MCP names parse the prefix before `__` and pass it directly to `runtime.callTool`. Catalog entries expose sanitized safe server names while sessions are stored under original server keys, so catalog-driven calls can fail for sanitized server names.
  - confidence: 0.86
- **[P2] Use flat enum schemas for bridge actions:** `src/agents/tools/mcp-tool.ts:11`
  - body: The bridge schemas use `Type.Union` over literals for action enums, which emits `anyOf`. Agent tool schemas should use the repo's flat string enum helpers so provider tool-schema ingestion remains compatible.
  - confidence: 0.88
- **[P1] Parse LSP payloads by byte offsets:** `src/agents/pi-bundle-lsp-runtime.ts:79-80`
  - body: `Content-Length` is measured in bytes, but this parser slices a UTF-16 JavaScript string using that byte count. Non-ASCII hover, diagnostic, or symbol text can truncate or offset the JSON chunk and desynchronize later frames.
  - confidence: 0.9
- **[P1] Dispose failed LSP child processes:** `src/agents/pi-bundle-lsp-runtime.ts:477-480`
  - body: If LSP initialization throws, the catch block only logs and continues; the spawned child is not added to `sessions` and is not disposed. Repeated failed startup attempts can accumulate orphan LSP processes.
  - confidence: 0.87
