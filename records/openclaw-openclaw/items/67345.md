## Summary

Keep open: current main does not implement the proposed parity bridges, and the PR branch introduces concrete security and correctness blockers in broadly registered agent tools.

## What This Changes

The PR adds claw-code alias normalization, MCP/LSP/Hermes bridge tools, claw-compat utility tools, task/team tools, sandbox parity checks, catalog wiring, and related tests.

## Best Possible Solution

Restack this as smaller PRs: safe alias normalization, MCP/LSP runtime bridge additions, sandbox/parity diagnostics alongside #67333, and separate maintainer-approved compatibility tools behind SSRF protection, config allowlists or redaction, owner-only mutation gates, and provider-safe schemas.

Yes: the blocking issues are reproducible from the PR-head diff lines and the current-main contracts for guarded fetch, owner-only tools, MCP safe names, and LSP URI handling. Current-main absence of the proposed tools is verified by exact-name source search.

## Solution Assessment

No: this is not the best way to solve the request because it bundles several product surfaces and exposes broadly available security-sensitive tools before existing guardrails are applied. A split, guardrail-first implementation is safer and easier to review.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Route remote_trigger through the SSRF guard:** `src/agents/tools/claw-compat-tools.ts:635-640`
  - body: `remote_trigger` performs a direct `fetch()` on a model-supplied URL with caller-controlled method, headers, and body. Because this bypasses the existing web-fetch SSRF guard, agent calls can target localhost, cloud metadata, or private-network services; route this through the guarded fetch path or remove the network tool from the broad tool set.
  - confidence: 0.96
- **[P1] Redact or allowlist config_compat reads:** `src/agents/tools/claw-compat-tools.ts:577-583`
  - body: When no path is supplied, `config_compat` returns the entire `loadConfig()` object, and dotted paths can select sensitive fields. This can put provider keys, auth tokens, or webhook secrets into model-visible tool results; return only a documented safe allowlist or redact sensitive fields before serialization.
  - confidence: 0.95
- **[P1] Gate config_compat writes behind owner authorization:** `src/agents/tools/claw-compat-tools.ts:551-555`
  - body: Passing `setting` plus `value` writes persisted config with `writeConfigFile`, but the tool is not marked owner-only. Non-owner sessions that can call this tool can mutate runtime configuration, including provider or permission-related settings, so the write path needs owner authorization or should be removed.
  - confidence: 0.94
- **[P1] Gate plan-mode config writes behind owner authorization:** `src/agents/tools/claw-compat-tools.ts:238-239`
  - body: `enter_plan_mode` and `exit_plan_mode` persist `permissions.defaultMode`, but these compatibility tools are broadly registered and not owner-only. That lets non-owner sessions change control-plane permission state; mark these tools owner-only or move the mutation behind an owner-checked gateway path.
  - confidence: 0.92
- **[P2] Do not claim message delivery without dispatch:** `src/agents/tools/claw-compat-tools.ts:189-193`
  - body: `send_user_message` returns `delivered: true` but only echoes JSON and never calls the existing message/channel delivery path. Models using this tool will believe a user-facing message was sent when it was silently dropped; wire it to real delivery or return an unsupported/pending status.
  - confidence: 0.91
- **[P2] Preserve LSP paths for URI conversion:** `src/agents/pi-embedded-runner/run/attempt.tool-call-normalization.ts:413-417`
  - body: The alias shim copies `path` into `uri`, but `createLspTool` treats any provided `uri` as already canonical and only converts `path` values to file URLs when `uri` is absent. A common call like `LSP { path: "src/main.ts" }` is sent to the language server as a non-URI document id; leave `path` intact or convert it to a file URL in the shim.
  - confidence: 0.9
- **[P2] Resolve safe MCP server names before dispatch:** `src/agents/tools/mcp-tool.ts:62-68`
  - body: Qualified MCP names parse the prefix before `__` and pass it directly to `runtime.callTool`, but the runtime catalog exposes sanitized `safeServerName` values while sessions are stored under original config keys. Calls copied from the catalog for servers with sanitized names will parse successfully and then fail as disconnected; map safe names back to real server names before dispatch.
  - confidence: 0.86
- **[P2] Preserve empty team update arrays:** `src/agents/tools/team-tool.ts:130-136`
  - body: `team update` converts empty `members`, `labels`, and task arrays to `undefined`, and `updateTeam` skips undefined fields. Callers cannot clear existing members, labels, or task links with an explicit empty list, so update should distinguish omitted fields from empty arrays.
  - confidence: 0.84
- **[P2] Use flat enum schemas for bridge actions:** `src/agents/tools/mcp-tool.ts:9-12`
  - body: The new bridge tools use `Type.Union([...Type.Literal])` for action enums, which emits `anyOf` schemas. Agent tool schemas should use the repo's flat `stringEnum` or `optionalStringEnum` helpers so provider tool-schema ingestion remains compatible.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: Security review needs attention: the PR introduces broadly available network, config-read, and config-write compatibility tools without the existing guardrails.

Concerns:

- **[high] Unrestricted outbound fetch in remote_trigger:** `src/agents/tools/claw-compat-tools.ts:635`
  - body: The new `remote_trigger` tool accepts model-supplied URLs and performs direct fetches without the existing SSRF guard, allowing requests to localhost, metadata endpoints, or private networks from any session with the tool.
  - confidence: 0.96
- **[high] Raw runtime config exposure:** `src/agents/tools/claw-compat-tools.ts:577`
  - body: The new `config_compat` read path can return the full runtime config or arbitrary dotted paths into model context, which risks exposing inline provider keys, tokens, webhook secrets, or auth settings.
  - confidence: 0.95
- **[high] Unauthorized persisted config writes:** `src/agents/tools/claw-compat-tools.ts:551`
  - body: The new `config_compat` write path persists settings through `writeConfigFile` without owner-only authorization, turning normal tool access into control-plane mutation capability.
  - confidence: 0.94
- **[high] Unauthorized plan-mode permission writes:** `src/agents/tools/claw-compat-tools.ts:238`
  - body: The plan-mode compatibility tools persist `permissions.defaultMode` without owner-only gating, allowing non-owner sessions to modify permission state.
  - confidence: 0.92
