## Summary

Keep this PR open. Current main still does not implement the proposed claw-code parity bridge, MCP/LSP/Hermes compatibility wrappers, or claw-compat utility tools, and the PR branch has concrete merge-blocking security and correctness findings in broadly registered agent tools.

## What This Changes

The PR adds claw-code tool-call aliases and argument shims, MCP/LSP/Hermes bridge tools, claw-compat utility tools, task/team tools, sandbox/parity harness checks, catalog wiring, and related tests.

## Best Possible Solution

Restack this as smaller PRs: one for safe runner alias normalization, one for MCP resource/auth runtime APIs, one for LSP dispatch extensions, one for sandbox/parity diagnostics with #67333, and separate maintainer-approved compatibility tools only after SSRF protection, config redaction or strict allowlists, owner-only persisted mutations, and provider-safe schemas are in place.

## Reproduction Assessment

Yes: the current-main comparison is reproducible by inspecting the tool registry, catalog, MCP runtime types, LSP runtime builder, and exact-name searches for the proposed tools. The PR branch findings are reproducible from the head-sha source lines without needing a live runtime.

## Solution Assessment

No: this PR is not the best way to solve the issue because it bundles multiple product surfaces and adds broadly available security-sensitive tools without existing guardrails. The safer solution is to split the work and land only the approved core pieces behind explicit security and ownership boundaries.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Route remote_trigger through the SSRF guard:** `src/agents/tools/claw-compat-tools.ts:635-640`
  - body: `remote_trigger` performs a direct `fetch()` on a model-supplied URL with caller-controlled method, headers, and body. Because this bypasses the existing web-fetch SSRF guard, agent calls can target localhost, cloud metadata, or private-network services; route this through the guarded fetch path or remove the network tool from the broad tool set.
  - confidence: 0.96
- **[P1] Redact or allowlist config_compat reads:** `src/agents/tools/claw-compat-tools.ts:577-583`
  - body: When no path is supplied, `config_compat` returns the entire `loadConfig()` object, and dotted paths can select sensitive fields. This can put provider keys, auth tokens, or webhook secrets into model-visible tool results; return only a documented safe allowlist or redact sensitive fields before serialization.
  - confidence: 0.95
- **[P1] Gate config_compat writes behind owner authorization:** `src/agents/tools/claw-compat-tools.ts:551-555`
  - body: Passing `setting` plus `value` writes persisted config with `writeConfigFile`, but the tool is not marked owner-only. Non-owner sessions that can call this tool can mutate runtime configuration, including provider or permission-related settings, so the write path needs owner authorization or should be removed.
  - confidence: 0.94
- **[P1] Gate plan-mode config writes behind owner authorization:** `src/agents/tools/claw-compat-tools.ts:238-239`
  - body: `enter_plan_mode` and `exit_plan_mode` persist `permissions.defaultMode` via `writeConfigFile`, but these compatibility tools are broadly registered and not owner-only. That lets non-owner sessions change control-plane permission state; mark these tools owner-only or move the mutation behind an owner-checked gateway path.
  - confidence: 0.92
- **[P2] Do not claim message delivery without dispatch:** `src/agents/tools/claw-compat-tools.ts:189-193`
  - body: `send_user_message` returns `delivered: true` but only echoes JSON and never calls the existing message/channel delivery path. Models using this tool will believe a user-facing message was sent when it was silently dropped; wire it to real delivery or return an unsupported/pending status.
  - confidence: 0.91
- **[P2] Preserve LSP paths for URI conversion:** `src/agents/pi-embedded-runner/run/attempt.tool-call-normalization.ts:413-417`
  - body: The alias shim copies `path` into `uri`, but `createLspTool` treats any provided `uri` as already canonical and only converts `path` values to file URLs when `uri` is absent. A common call like `LSP { path: "src/main.ts" }` is rewritten to a non-URI document id and sent to the language server as-is; leave `path` intact or convert it to a file URL in the shim.
  - confidence: 0.9
- **[P2] Resolve safe MCP server names before dispatch:** `src/agents/tools/mcp-tool.ts:62-68`
  - body: Qualified MCP names parse the prefix before `__` and pass it directly to `runtime.callTool`, but the runtime catalog exposes sanitized `safeServerName` values while sessions are stored under original config keys. Calls copied from the catalog for servers with sanitized names will parse successfully and then fail as disconnected; map safe names back to real server names before dispatch.
  - confidence: 0.86
- **[P2] Preserve empty team update arrays:** `src/agents/tools/team-tool.ts:130-136`
  - body: `team update` passes `members`, `labels`, and `taskIds` through helpers that convert empty arrays to `undefined`, and `updateTeam` skips undefined fields. Callers cannot clear existing members, labels, or task links with an explicit empty list, so update should distinguish omitted fields from empty arrays.
  - confidence: 0.84
- **[P2] Use flat enum schemas for bridge actions:** `src/agents/tools/mcp-tool.ts:9-12`
  - body: The new bridge tools use `Type.Union([...Type.Literal])` for action enums, which emits `anyOf` schemas. Repo policy and provider compatibility prefer the local flat string enum helpers because some providers reject `anyOf`; use the existing enum helpers for MCP, LSP, Hermes, task, and team action fields.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The PR introduces broadly available network, config, and permission-control tools with concrete security and supply-chain-adjacent risks before existing guardrails are applied.

- **[high] Unrestricted outbound fetch in remote_trigger:** `src/agents/tools/claw-compat-tools.ts:635`
  - body: `remote_trigger` directly fetches arbitrary model-supplied URLs without the existing SSRF guard, allowing access to loopback, private networks, or metadata endpoints from any session with the tool enabled.
