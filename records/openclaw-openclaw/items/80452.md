Keep open: current main still has a source-level OpenAI Responses replay gap where route-switch cleanup can remove `rs_*` reasoning while a replayable assistant `msg_*` id can remain, and no merged related item proves the Gemini-to-OpenAI direction is fixed.

## What This Changes

The issue asks OpenClaw to make Gemini-to-OpenAI failover replay avoid sending OpenAI Responses assistant `message` items without their required paired `reasoning` items.

## Best Possible Solution

Add a focused OpenAI Responses replay sanitizer that removes or neutralizes assistant `message` item ids whenever their paired replayable reasoning item has been removed, while preserving same-route valid reasoning/message replay.
