## Summary

Keep open, but this PR is not merge-ready: the native audio/video idea is useful and still tracked by related open issues, but the patch sends media through an unsupported text/image-only model context, bypasses provider transport/auth controls, leaves the new output kinds unwired, and is currently conflicting with main.

## What This Changes

The PR adds native audio/video media-understanding provider hooks, result kinds, auto-registration for audio/video-capable models, runner selection logic, and tests.

## Best Possible Solution

Define the native audio/video contract in the current LLM/media provider layer first, then rebase and implement it with typed content blocks, provider-specific serialization, explicit capability and egress policy, updated media output consumers, and live transport proof.

## Solution Assessment

No. This is a plausible feature direction, but the current implementation is the wrong layer until the typed native media/provider contract and raw-media egress policy are settled and wired through current main.
