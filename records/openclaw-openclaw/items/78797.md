## Summary

Keep open: this is a new media-understanding feature, but the current PR is not mergeable because the native path uses an unsupported media contract, can bypass existing transport controls, and returns new result kinds that current runtime helpers do not consume. The author has also asked for maintainer guidance on the contract direction.

## What This Changes

The PR adds native `understandAudio` and `understandVideo` media-understanding hooks, auto-registers configured audio/video-capable models, adds new output kinds, and tries native paths before existing transcription/description fallbacks.

## Best Possible Solution

Define a typed native audio/video contract or provider-owned native transport that preserves existing media request controls, update runtime consumers/docs/changelog/tests, then validate native and fallback paths with redacted runtime proof.

## Reproduction Assessment

Not applicable as a bug reproduction because this PR adds a new feature rather than fixing broken current behavior. Source inspection gives a high-confidence way to verify the patch defects through the runtime output contract and pi-ai media serialization path.

## Solution Assessment

No. The current approach casts unsupported audio/video content through `complete()` and does not preserve OpenClaw's media transport controls; a safer direction is a formal media content contract or provider-specific native transport with matching runtime consumers.

  - body: This builds a pi-ai user content block with `type: "audio"` via `any`, but pi-ai 0.73.0 only types user blocks as text or image, and the OpenAI-completions converter serializes every non-text block as `image_url`. Please add a real audio/video media contract or a provider-owned native transport before routing provider media through `complete()`; the video helper has the same unsupported shape.
  - confidence: 0.91

Native media requests can bypass configured media auth overrides, proxy/TLS controls, private-network policy, and API-key rotation.
  - confidence: 0.88
