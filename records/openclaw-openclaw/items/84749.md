## Summary

This PR is still needed because current main and the latest release still choose `--before` from npm's computed `before` value in hosted installers, but the submitted fix has one remaining compatibility gap: the new raw-key probe misses npm's resolved default `userconfig` path when `HOME` is unset.

## What This Changes

The PR adds raw user/global npmrc `min-release-age` detection to hosted shell and PowerShell installers and adds installer regression coverage for npm freshness bypass selection.

## Best Possible Solution

Land the targeted installer fix after adding effective `userconfig --global` discovery to the raw-key probes and keeping the existing project-.npmrc exclusion for global installs.

## Reproduction Assessment

Yes. Source inspection shows current main and v2026.5.22 can still choose `--before` from npm's computed `before` value, and the PR body supplies terminal proof against npm 11.12.1 for the raw `min-release-age` conflict.

## Solution Assessment

No, not quite yet. Raw npmrc detection is the right maintainable shape, but it should also probe npm's effective `userconfig --global` path before merge so HOME-unset service contexts do not keep the original failure mode.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Probe userconfig when HOME is unset:** `scripts/install.sh:769-774`
  - body: `npm_config_has_raw_key` only checks an explicit `NPM_CONFIG_USERCONFIG` or `${HOME}/.npmrc` before globalconfig. npm still resolves a default userconfig path when `HOME` is unset (`npm config get userconfig --global` does so here), so a raw `min-release-age` in that file is missed and the installer can fall back to `--before`, recreating the npm conflict this PR fixes. Please query the effective userconfig path the same way this helper already probes globalconfig, and mirror it in `install-cli.sh`.
  - confidence: 0.87
- **[P2] Probe PowerShell userconfig when HOME is unset:** `scripts/install.ps1:962-968`
  - body: The PowerShell helper has the same gap: it only adds env userconfig or `$HOME\.npmrc`, then probes globalconfig. On Windows or service-style environments where npm resolves userconfig independently, raw `min-release-age` can still be missed and the installer can emit `--before`; add an `npm config get userconfig --global` probe before scanning files.
  - confidence: 0.8
