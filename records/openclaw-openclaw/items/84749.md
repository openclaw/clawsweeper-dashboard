## What This Changes

The PR adds raw npmrc min-release-age detection to hosted shell and PowerShell installers and expands installer freshness tests.

## Best Possible Solution

Keep the raw-npmrc fix, but scope hosted global-installer probes to only the npmrc files that npm install -g actually reads for each invocation and add mixed-scope regression coverage before merge.

## Reproduction Assessment

Yes. The linked issue plus current main show a source-level path where npm reports min-release-age as null, exposes a computed before value, and the hosted installer emits --before for npm install -g.

## Solution Assessment

No, not as submitted. Scanning raw npmrc files is the right direction, but the hosted global installer path must not use project .npmrc as an effective config source for npm install -g.
