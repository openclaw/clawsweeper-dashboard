## What This Changes

The PR updates hosted shell and PowerShell installers to scan raw npmrc files for `min-release-age` before choosing npm freshness bypass flags, and adds focused shell installer fixture tests.

## Best Possible Solution

Land this after contributor or maintainer proof shows the patched hosted installer path with real npm under `min-release-age`; keep the shell/PowerShell raw-npmrc detection aligned with the existing TypeScript helper.

## Reproduction Assessment

Yes. Current `main` has a clear source-level reproduction: if npm returns `min-release-age` as null but returns a computed `before`, the hosted installers emit `--before=<now>`, which npm documents as mutually exclusive with raw `min-release-age`.

## Solution Assessment

Yes for the shell hosted-installer path: scanning raw npmrc files before using computed `before` is consistent with the existing TypeScript helper. The remaining merge gate is real behavior proof, not a different code shape.
