- `P2`: This fixes a focused opt-in plugin behavior bug that can leave noisy shell output uncompacted, with limited blast radius.

## Summary

Current head remains useful and is not cleanup-closeable: current main still forwards raw `details` into tokenjuice, while tokenjuice@0.8.0 skips exec-like results without completed/failed details. The updated PR addresses the earlier metadata-only gap, has structured live-output proof, and I found no blocking code or security findings.

## What This Changes

The PR synthesizes exec-style details for tokenjuice `exec`/`bash` middleware results when details are absent or lack status, preserves metadata-only details, and adds regression tests for no-details, metadata-only, and existing-status cases.

## Best Possible Solution

Land the tokenjuice-local normalization with its regression coverage after normal maintainer and CI gates, keeping the generic OpenClaw/Codex tool-result contracts unchanged.

## Reproduction Assessment

Yes, from source inspection. Current main forwards undefined or metadata-only details, tokenjuice@0.8.0 requires completed/failed details, and current bash can emit the affected metadata-only shape.

## Solution Assessment

Yes. The plugin-local normalization is the narrowest maintainable fix because the dependency requires exec-style details, while OpenClaw and Codex middleware contracts do not require every tool emitter to provide tokenjuice-ready details.
