## Summary

Keep open: the plugin-local fix is in the right area, but the new guard still skips exec-like bash results that already carry metadata without a completed/failed status, so noisy truncated outputs can remain un-compacted.

## What This Changes

The PR adds Tokenjuice middleware normalization for bash/exec tool results whose content and command args are present but whose details are absent, plus a regression test for that path.

## Best Possible Solution

Normalize bash/exec details when the existing details are absent or lack completed/failed status, preserving metadata such as truncation/fullOutputPath, and cover both cases in regression tests.

## Reproduction Assessment

Yes. Source inspection shows tokenjuice@0.8.0 requires completed/failed details, while current bash results can carry truncation metadata without that status, and the PR returns those metadata objects unchanged.

## Solution Assessment

No. The plugin-local normalization layer is appropriate, but the guard should key on the exec-detail status contract rather than treating any details object as already usable.
