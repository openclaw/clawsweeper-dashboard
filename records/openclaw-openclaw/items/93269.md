## Summary

Keep open for normal maintainer review; I found no blocking correctness issue. Current main can skip Tokenjuice compaction for bash/exec output without completed/failed details, and this PR fixes that inside the Tokenjuice adapter while preserving existing metadata and status-bearing details.

## What This Changes

This PR updates the Tokenjuice tool-result middleware to synthesize exec-style details for bash/exec outputs without usable details and adds focused regression tests.

## Best Possible Solution

Land the plugin-local normalization after exact-head merge gating stays clean, keeping Tokenjuice optional and preserving existing status-bearing details unchanged.

## Reproduction Assessment

Yes, source-reproducible. Current main can produce bash details containing only truncation/fullOutputPath, while tokenjuice@0.8.0 skips compaction unless details has completed/failed status.

## Solution Assessment

Yes. The Tokenjuice adapter is the narrowest maintainable layer because it owns the reducer contract and avoids changing core bash result semantics for users without the plugin.
