## Summary

Keep this PR open: the packageManager-based pnpm source is useful, but the patch changes pnpm store cache saving from best-effort to job-failing behavior through pnpm/action-setup's post step, which is too risky for shared CI and release workflows.

## What This Changes

This PR bumps the root pnpm packageManager pin to pnpm 11.2.2, removes duplicated PNPM_VERSION workflow plumbing, switches shared setup to pnpm/action-setup, and adds exact minimumReleaseAge exclusions for locked packages.

## Best Possible Solution

Keep the centralized packageManager pnpm source, but preserve OpenClaw's current best-effort cache-save behavior and keep the release-age exclusions exact and auditable.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path for the PR defect: enable pnpm/action-setup cache and make its post-action cache save throw, and the upstream code calls setFailed. I did not induce a live cache conflict, but the current-main versus PR contract difference is explicit in source.

## Solution Assessment

No, not as written. Centralizing pnpm version selection through packageManager is a good direction, but cache save should remain best-effort instead of being delegated to a post action that fails the job on save errors.
