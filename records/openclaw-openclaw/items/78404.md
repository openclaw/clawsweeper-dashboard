Keep open: current main still has the WhatsApp no-op reload path that leaves an unloaded WhatsApp plugin undiscovered, and this issue is explicitly paired with an open fixing PR that uses closing syntax. The issue should remain open until that PR is landed or closed.

## What This Changes

The issue reports that externally hot-enabling `channels.whatsapp.enabled` can be ignored and later persisted back to `false` when the WhatsApp plugin was not loaded at gateway startup.

## Best Possible Solution

Keep this issue open while maintainers review https://github.com/openclaw/openclaw/pull/78414, then close it only after the chosen restart/warning/hot-load contract lands or the PR is replaced by another fix.
