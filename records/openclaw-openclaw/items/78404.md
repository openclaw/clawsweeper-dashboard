## Summary

Keep open: current main still treats `channels.whatsapp` edits as a no-op reload path when WhatsApp is not already loaded, and the linked same-author fix PR is still open rather than merged.

## What This Changes

The issue requests a clear supported behavior for hot-enabling `channels.whatsapp.enabled=true` when the running gateway has not loaded the WhatsApp plugin.

## Best Possible Solution

Refine and land the linked gateway reload fix so enabling an unloaded channel plugin either triggers the documented restart path with a clear warning or gets an explicit maintainer-approved hot-load design.

## Reproduction Assessment

No, not for the full silent demotion. The production report is detailed and current source proves the WhatsApp no-op/reload gap, but I did not establish a live current-main reproduction of the later true-to-false writer.

## Solution Assessment

Unclear. Restart-on-activation in https://github.com/openclaw/openclaw/pull/78414 is a plausible narrow repair, but maintainers still need to decide how it should interact with `gateway.reload.mode="hot"`.
