## Summary

## What This Changes

The issue reports that an external hot write of `channels.whatsapp.enabled=true` can be silently ignored or reverted when the running gateway did not load the WhatsApp plugin at cold boot.

Land or explicitly reject the maintainer replacement PR, then close this issue only after the chosen fix is merged and release/current-main provenance is available.
