- **[P1] Persist the media-bearing turn, not only an update:** `src/gateway/server-node-events.ts:409-418`
  - body: `emitSessionTranscriptUpdate()` notifies transcript listeners; it does not append or rewrite the JSONL transcript. Because `agent.request` still dispatches without a durable user-turn recorder/append path, persisted history can remain the plain text turn without `MediaPath`/`MediaPaths` after reload, which is the bug this PR is meant to fix.
  - confidence: 0.92
- **[P2] Preserve parsed image order when building MediaPaths:** `src/gateway/server-node-events.ts:358-369`
  - body: `parseMessageWithAttachments()` returns `imageOrder` so inline and offloaded images keep the user's original order, but this helper saves all inline images first and then appends every offloaded ref. A request whose first image is offloaded and second is inline will record reversed `MediaPaths`/`MediaTypes`; interleave saved entries using `imageOrder` like the `chat.send` path.
  - confidence: 0.9
