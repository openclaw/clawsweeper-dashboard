- **[P1] Persist the media-bearing turn, not only an update:** `src/gateway/server-node-events.ts:397`
  - body: `emitSessionTranscriptUpdate()` only notifies listeners; it does not append or rewrite the JSONL transcript.   - confidence: 0.92
- **[P2] Preserve parsed image order when building MediaPaths:** `src/gateway/server-node-events.ts:356-358`
  - body: `parseMessageWithAttachments()` returns `imageOrder` so inline and offloaded images keep the user's original order, but this helper saves all inline images first and then appends every offloaded ref. A request whose first image is offloaded and second is inline will record reversed `MediaPaths`/`MediaTypes`; interleave saved entries using `imageOrder` like the `chat.send` path.
  - confidence: 0.88
