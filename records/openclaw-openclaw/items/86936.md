## What This Changes

This PR adds media persistence and transcript-update helpers to the gateway `agent.request` node-event path plus regression tests for media transcript fields.

## Best Possible Solution

Persist `MediaPath`/`MediaPaths` on the actual `agent.request` user transcript entry, preserving `imageOrder` like `chat.send`, then prove the iOS/node share flow against a real transcript file.

## Reproduction Assessment

Yes. Source inspection shows current main parses `offloadedRefs` in `agent.request` but does not attach `MediaPath` fields to the persisted user turn.

## Solution Assessment

No. The proposed fix uses `emitSessionTranscriptUpdate`, which is a listener fan-out, so it should be replaced with a persistence-path change that updates the actual transcript entry and preserves media ordering.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Persist media on the actual transcript entry:** `src/gateway/server-node-events.ts:690`
  - body: `emitSessionTranscriptUpdate` only fan-outs an event to listeners; the JSONL write happens earlier when the SessionManager append path persists the real user message. Emitting this synthetic user message after dispatch can show a duplicate live update with `MediaPath` fields while the actual transcript file still contains the unmodified user turn, so the reported history bug remains. Please carry these media fields into the SessionManager-written user message or rewrite that entry after it is appended.
  - confidence: 0.9
- **[P2] Preserve attachment order for mixed media:** `src/gateway/server-node-events.ts:354`
  - body: `parseMessageWithAttachments` returns `imageOrder` because inline and offloaded images need to stay in the same order the agent receives them. This helper saves all inline images first and then appends all offloaded refs, so a request with an offloaded image before an inline image records the wrong `MediaPath`/`MediaType` order. Reuse the chat.send interleaving logic here.
  - confidence: 0.86
