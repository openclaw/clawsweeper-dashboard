## Summary

Keep open. Current main still handles Discord reaction add/remove events as routed system events only; neither the plugin hook catalog/runner nor the internal hook docs/types expose a reaction hook surface. Related #52658 and #60805 point back to this same hook/API gap, while #17840 tracks the separate question of reactions waking an agent turn.

## What This Changes

Expose Discord reaction add/remove events through OpenClaw hooks so plugins or hook automation can respond to user emoji reactions.

## Best Possible Solution

Add an additive, documented reaction event contract for hooks, fire it after existing Discord reaction filtering and route resolution, carry structured reaction metadata, and cover the plugin/internal hook types, docs, and tests while leaving reaction-triggered agent turns to #17840.
