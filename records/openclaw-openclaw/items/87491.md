## Summary

Keep open. Current main and the latest release still have the doctor workspace-status detector wired to a function that always returns no legacy/extra workspace directories, while related memory-write hardening only covers part of the reported wrong-workspace symptom.

## What This Changes

## Best Possible Solution

Keep this issue open for a maintainer-sponsored doctor/upgrade design that detects legacy/rebrand sibling workspace directories, reports a preservation-safe recovery path, and keeps memory writes anchored to the resolved active workspace.

## Reproduction Assessment

Yes for the doctor detector gap: current source always returns an empty legacy-directory list, so the documented workspace-status warning cannot fire. No for the exact upgrade race: the macOS phantom directory creation still needs live reproduction or maintainer-provided upgrade evidence.

## Solution Assessment

Unclear as an implementation plan. The needed direction is not just adding a warning; maintainers should choose a data-preserving repair policy for legacy/rebrand sibling directories before any automated fix writes, symlinks, or removes user state.
