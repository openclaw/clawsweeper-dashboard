## Summary

Keep open: this is a protected-label feature PR, and latest PR head still has a concrete P2 recovery bug where partial multi-app plugin bindings can be treated as complete.

## What This Changes

The PR adds explicit Codex native plugin config, curated plugin inventory/activation, per-thread app config, plugin elicitation policy, migration support, docs, tests, and changelog coverage.

## Best Possible Solution

Fix the partial-binding recovery bug, then let the Codex owners review and land the explicit curated-only native plugin path with exact-head checks.

## Reproduction Assessment

Yes for the review finding: source inspection shows a saved partial `pluginAppPolicyContext.pluginAppIds` is reused as the expected app set, so the recovery check can miss newly available apps from the same plugin. No runtime reproduction was run because this was a read-only PR review.

## Solution Assessment

No for the current PR head; the general explicit curated-only design is plausible, but the recovery check should compare the saved binding against a freshly rebuilt expected app set or otherwise force a rebuild when inventory readiness expands.
