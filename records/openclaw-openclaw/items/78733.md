## Summary

Keep open: this is a protected maintainer-labeled feature PR, and this review found a concrete auth-boundary issue in the source plugin inventory path before owner review or merge.

## What This Changes

The PR adds native Codex plugin app support for migrated `openai-curated` Codex plugins, including `codexPlugins` config, app inventory and activation, thread app policy, migration CLI support, docs, changelog entries, and tests.

## Best Possible Solution

Keep the explicit fail-closed native plugin model, but make migration source inventory read only from the source Codex home's existing account before owner review and exact-head gates.

## Reproduction Assessment

Not applicable as a current-main bug: current main intentionally lacks `codexPlugins` and keeps native Codex plugins as manual-review migration artifacts. The PR body supplies after-fix live Showboat and gateway/TUI proof for the new feature path.

## Solution Assessment

No for the current diff: the explicit config and fail-closed app policy are the right shape, but source migration inventory should not run through the target OpenClaw auth bridge. The safer fix is an auth-disabled source app-server inventory path, while keeping target-side install/auth behavior for apply.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Keep source discovery out of target auth:** `extensions/codex/src/migration/source.ts:145-160`
  - body: The new source inventory call goes through `requestCodexAppServerJson`, whose shared-client path applies the current OpenClaw Codex auth profile or inherited API-key fallback before issuing `plugin/list`. During `openclaw migrate codex --dry-run --from <other Codex home>`, that can log the target OpenClaw account into the source `CODEX_HOME` and inventory the wrong account instead of the source-installed plugins. Use an auth-disabled/isolated app-server request for source discovery, or otherwise guarantee it only reads the source home's existing account.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: Needs attention: source plugin discovery appears to reuse the target Codex auth bridge during migration inventory, which can cross account boundaries during dry-run.

Concerns:

- **[medium] Source inventory can use target Codex credentials:** `extensions/codex/src/migration/source.ts:145`
  - body: The PR starts a source `CODEX_HOME` app-server through `requestCodexAppServerJson`, but that helper applies OpenClaw's normal Codex auth profile/env fallback. A migration dry-run can therefore authenticate a source home with target credentials or mutate source account state while only trying to inventory source-installed plugins.
