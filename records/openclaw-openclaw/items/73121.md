## Summary

Keep open. Current main has scheduled task maintenance and partial CLI reconciliation, but stale CLI task rows with no live run context can still remain active when the row has no child session key or is backed by the durable main session, and the human task list still prints shortened IDs that exact lookup cannot resolve.

## What This Changes

The issue asks OpenClaw to auto-reconcile dead CLI runtime tasks instead of leaving them indefinitely in running state, and to make task cancellation lookup reliable.

## Best Possible Solution

Task-registry maintenance should treat stale CLI rows as backed only when a live run context or runtime recovery hook proves they are active, while the task CLI should expose usable identifiers or resolve unique prefixes with explicit ambiguity errors.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is a stale runtime cli TaskRecord older than the five-minute grace window, with no getAgentRunContext result and either no childSessionKey or childSessionKey set to agent:main:main with a persisted session row; current main leaves it running/stale instead of terminal.

## Solution Assessment

Yes. The narrowest maintainable fix is in task-registry maintenance and task lookup/list semantics with focused regression tests, not a broad OS PID monitor or startup migration.
