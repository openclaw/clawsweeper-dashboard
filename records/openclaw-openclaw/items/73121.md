## Summary

Keep open. Current main has partial CLI stale-task reconciliation, but the same remaining gaps from the prior review are still present: stale CLI rows without a child session are treated as backed forever, durable main-session rows can keep dead CLI tasks active, and the human task table still shows truncated IDs while lookup remains exact-only.

## What This Changes

The issue requests automatic reconciliation of dead CLI runtime task records and clearer task cancellation lookup behavior.

## Best Possible Solution

Update task-registry maintenance so stale CLI tasks without a live run context are unbacked when they have no child session or only the durable main session, unless a runtime-specific recovery hook proves they are still active. Align task list/show/cancel lookup so displayed identifiers are usable or produce explicit ambiguity errors.

## Reproduction Assessment

Yes. A code-level reproduction is an active runtime: cli TaskRecord older than the 5-minute grace window with no getAgentRunContext result and either no childSessionKey or childSessionKey agent:main:main; current maintenance keeps it running.

## Solution Assessment

Yes. The narrowest maintainable fix is in task-registry liveness semantics plus task CLI lookup/list behavior, rather than adding a broad OS process monitor or rewriting session retention.
