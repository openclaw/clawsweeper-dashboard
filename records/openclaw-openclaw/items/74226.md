## Summary

Keep open: the bug is valid and the PR is in the right Telegram edit path, but the submitted guard counts rendered HTML instead of Telegram's post-entity text length, leaves an unused helper that fails lint, and still lacks real Telegram after-fix proof.

## What This Changes

The PR adds a Telegram edit-message truncation helper, applies it to HTML/plain `editMessageText` calls, and adds unit coverage for long edit payloads.

## Best Possible Solution

Implement the edit guard against Telegram-counted text after entity parsing, avoid truncating valid entity-expanded HTML, remove the unused helper or wire it into real handling, add regression coverage, and verify the marker in a real Telegram edit.

## Reproduction Assessment

Yes, source-reproducible rather than live-reproduced. Current main sends rendered HTML and plain fallback text directly to `editMessageText`, while Telegram caps edited text at 4096 characters after entity parsing.

## Solution Assessment

No, not as submitted. The Telegram edit path is the right boundary, but the guard should count the text Telegram counts after entity parsing and the patch must pass lint and include real Telegram proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Count Telegram edit length after entity parsing:** `extensions/telegram/src/send.ts:215-218`
  - body: Telegram's `editMessageText` limit is after entities parsing, but this guard trips on `htmlText.length` and slices rendered markup. Inputs like 1000 ampersands render as 5000 characters of `&amp;` while Telegram-counted text is 1000 characters, so this would truncate valid edits and may add no marker because `omittedChars` uses `rawText.length`. Base the guard and omitted count on parsed text and cover entity-expanded input.
  - confidence: 0.93
- **[P2] Remove or wire the unused message-too-long helper:** `extensions/telegram/src/send.ts:192-194`
  - body: `isMessageTooLongTelegramError` is private and never called; CI `check-lint` already fails with `Function 'isMessageTooLongTelegramError' is declared but never used.` Delete it or use it in the edit retry path before this can merge.
  - confidence: 0.99
