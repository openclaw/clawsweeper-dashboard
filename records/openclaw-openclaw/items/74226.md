## Summary

Keep open: this PR targets a real Telegram edit-length bug, but the submitted patch still counts the wrong length, leaves a lint-blocking unused helper, and has no real Telegram after-fix proof.

## What This Changes

The branch adds a Telegram edit-message truncation helper, applies it to HTML/plain `editMessageText` calls, and adds unit coverage in `extensions/telegram/src/send.test.ts`.

## Best Possible Solution

Keep the fix in the Telegram plugin, but implement the edit guard against Telegram-counted parsed text, add entity-expanded regression coverage, remove or use the unused helper, and verify the truncation marker in a real Telegram edit.

## Reproduction Assessment

Yes, source-reproducible rather than live-reproduced. Current main sends rendered HTML/plain text directly to `editMessageText`, while Telegram documents the edit text limit as 1-4096 characters after entities parsing. ([core.telegram.org](https://core.telegram.org/bots/api))

## Solution Assessment

No, not as submitted. The Telegram edit path is the right owner boundary, but the implementation should count the text Telegram counts after entity parsing and should not leave a lint-failing unused helper.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Count Telegram edit length after entity parsing:** `extensions/telegram/src/send.ts:215-218`
  - body: Telegram's `editMessageText` limit is 1-4096 characters after entities parsing, but this helper gates on rendered HTML length and derives `omittedChars` from `rawText`. Inputs such as repeated ampersands render as `&amp;` above 4096 while parsed text stays under the limit, so this branch can truncate valid content and append no marker. Base the guard and omitted count on the Telegram-counted text and cover entity-expanded input. ([core.telegram.org](https://core.telegram.org/bots/api))
  - confidence: 0.94
- **[P2] Remove the unused Telegram length-error helper:** `extensions/telegram/src/send.ts:192-194`
  - body: CI lint fails because `isMessageTooLongTelegramError` is declared but never used. Remove it or wire it into the edit-message error path before merge so the lint lanes can pass.
  - confidence: 0.98
