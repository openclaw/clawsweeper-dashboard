## What This Changes

The PR imports `normalizeVerboseLevel` and changes `dispatchReplyFromConfig` so `/verbose on` or `/verbose full` allows text tool summaries in group chats while keeping native command-source suppression.

## Best Possible Solution

Land a current-main fix that layers explicit verbose `on` or `full` into the existing shared progress policy only where maintainers accept it, preserving Slack non-direct suppression, forum-topic delivery, native direct delivery, media passthrough, and exec-approval exceptions.

## Solution Assessment

No for this PR as written. The safer solution is to update current main's shared progress-delivery policy instead of applying the older group/native condition from the branch.
