The PR makes stored `/verbose on` or `/verbose full` override group-chat tool-summary suppression in `dispatchReplyFromConfig` while keeping native command-source sessions suppressed.

Extend current main's shared verbose/tool-summary gate so explicit verbose `on` or `full` enables ordinary non-Slack group tool progress, while preserving Slack non-direct suppression, forum-topic allowance, native direct delivery, preview suppression, media/exec-approval fallbacks, and adding focused tests plus changelog coverage.
