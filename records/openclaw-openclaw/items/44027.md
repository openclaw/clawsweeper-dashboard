## Summary

Keep open: the reported bug is still plausible on current main, but this PR is stale against the newer shared delivery policy and would regress Slack, forum-topic, and native direct behavior if merged as-is.

## What This Changes

The PR changes `dispatchReplyFromConfig` to let stored `/verbose on` or `/verbose full` override group-chat tool-summary suppression while keeping native command-source suppression.

## Best Possible Solution

Layer explicit session verbose `on` or `full` into current main's shared progress-delivery policy, preserving Slack non-direct suppression, forum-topic and native behavior, media/exec-approval fallbacks, and privacy documentation.

## Reproduction Assessment

Yes. A focused unit test can seed an ordinary non-Slack group session with `verboseLevel: "on"`, call `opts.onToolResult({ text: "tool progress" })`, and assert `dispatcher.sendToolResult` receives it; current main has adjacent negative and exception-path coverage but not that positive case.

## Solution Assessment

No for this PR as written. The safer fix is to update current main's shared delivery policy and tests instead of reintroducing the older group/native condition from the stale branch.
