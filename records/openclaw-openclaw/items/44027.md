## Summary

Keep open for maintainer handling. Current main still suppresses ordinary group verbose/tool summaries, but this PR is not safe to merge as-is because it changes privacy-sensitive group delivery without a trusted-room config policy, is conflicting against current main, regresses current native command-source tool-summary behavior, and lacks after-fix real behavior proof.

## What This Changes

The PR changes reply dispatch to read the stored verbose level and allow `/verbose on` or `/verbose full` to deliver text tool summaries in group chats while preserving default group suppression otherwise.

## Best Possible Solution

Define a fail-closed, typed trusted-room/group verbose-progress policy with global and per-room semantics, preserve ordinary group defaults, document the privacy warning, and add regression coverage that reconciles this PR with https://github.com/openclaw/openclaw/issues/76135.

## Reproduction Assessment

Yes, at source level. Current main sets shouldSendVerboseProgressMessages false for ordinary groups, drops text-only tool payloads on that path, and has a regression test asserting Telegram group text summaries stay suppressed.

## Solution Assessment

No. The PR addresses the symptom with a session-level verbose override, but the maintainable fix needs an explicit trusted-room config policy and the branch also conflicts with current native command-source delivery behavior.
