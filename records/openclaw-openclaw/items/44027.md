## Summary

Keep open for maintainer review: current main still suppresses ordinary group verbose/tool output, but this PR is not a merge-ready fix because it conflicts with current native command-source behavior, lacks real behavior proof, and needs a trusted-room policy decision before expanding group-visible diagnostics.

## What This Changes

The PR changes reply dispatch so stored `/verbose on` or `/verbose full` can let ordinary group chats receive tool summaries while preserving default group suppression otherwise.

## Best Possible Solution

Add a typed, fail-closed trusted-room/group verbose-progress policy with global and per-room semantics, preserve ordinary group defaults, document the privacy warning, and cover direct, ordinary group, forum, Slack non-direct, and native command-source cases with regression tests.

## Reproduction Assessment

Yes, at source level. Current main reads the stored verbose level, but the separate group gate still disables verbose progress/tool summaries for ordinary groups and tests assert text tool summaries stay suppressed there.

## Solution Assessment

No. The PR addresses the symptom with a session-level verbose override, but the maintainable fix needs explicit trusted-room config semantics and must not regress current native command-source tool-summary delivery.
