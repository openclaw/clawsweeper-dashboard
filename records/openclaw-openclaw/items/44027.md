## Summary

Keep open: current main still suppresses ordinary group verbose/tool output, and this PR needs maintainer judgment on trusted-room disclosure policy plus contributor real-behavior proof before it can be merged.

## What This Changes

The PR changes reply dispatch so stored `/verbose on` or `/verbose full` can override ordinary group-chat tool-summary suppression while keeping native command-source suppression.

## Best Possible Solution

Define a fail-closed, typed trusted-room/group verbose-progress policy with global and per-room semantics, preserve ordinary group defaults, document the privacy warning, and add regression coverage that reconciles this PR with https://github.com/openclaw/openclaw/issues/76135.

## Reproduction Assessment

Yes, at source level. Current main gates verbose progress and tool summaries off for ordinary groups, drops text-only tool payloads on that path, and has regression coverage asserting Telegram group text summaries stay suppressed.

## Solution Assessment

No. The PR addresses the symptom with a session-level verbose override, but the maintainable fix needs an explicit trusted-room config policy and the branch conflicts with current native command-source delivery behavior.
