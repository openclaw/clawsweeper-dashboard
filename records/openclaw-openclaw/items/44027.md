## Summary

Keep open: the underlying group-chat verbose/tool-output gap still exists on current main, but this PR needs a rebase/logic update, real behavior proof, and maintainer policy review before it can merge.

## What This Changes

The branch imports `normalizeVerboseLevel` and changes reply dispatch to deliver tool summaries in group chats when the persisted session verbose level is `on` or `full`, while suppressing native command-source summaries.

## Best Possible Solution

Add a provider-agnostic, fail-closed group verbose-progress policy with typed config, docs, and tests, then allow session `/verbose` to emit group tool output only when that trusted-room policy permits it.

## Reproduction Assessment

Yes, at source level. Current main blocks ordinary group verbose/tool summaries through `shouldSendVerboseProgressMessages`, drops text-only tool payloads on that path, and has a regression test asserting the suppression baseline.

## Solution Assessment

No. A `/verbose` override addresses the symptom, but this branch is stale against current main, would regress native command-source tool-summary delivery if rebased literally, and needs an explicit trusted-room policy decision for group output.
