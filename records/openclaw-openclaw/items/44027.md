## Summary

Keep open: the source-level problem is real, but this branch should not merge as-is because it is based on the older group/native gate, misses current main's shared progress policy, and still lacks after-fix real behavior proof.

## What This Changes

The PR changes `dispatchReplyFromConfig` to read the session `verboseLevel` and allow group-chat tool summaries when `/verbose on` or `/verbose full` is set.

## Best Possible Solution

Update current main's shared verbose/tool progress policy through an explicit trusted-room/config decision, preserving Slack non-direct suppression, forum-topic allowance, native direct delivery, media and exec-approval exceptions, with focused tests and real chat proof.

## Reproduction Assessment

Yes, at source level. Seed an ordinary non-forum group session with `verboseLevel: "on"`, emit `onToolResult({ text: "tool progress" })`, and current main still drops the text because `shouldSendVerboseProgressMessages` remains false for ordinary groups.

## Solution Assessment

No, not as written. The safer solution is to layer the verbose opt-in into current main's shared progress-delivery policy instead of reintroducing the older group/native-only condition.
