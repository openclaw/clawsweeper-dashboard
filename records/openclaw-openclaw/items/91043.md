## Label Justifications

- `P2`: This is a normal-priority Telegram/core reply-context bug with bounded prompt-context impact rather than a crash or outage.
- `impact:message-loss`: The selected quote path can omit the rest of the replied-to message from the model-visible prompt.
- `impact:session-state`: Dropping the full reply target can make the next agent turn lose the conversation item the user is pointing at.

## Summary

Keep open: current main and v2026.6.8 still render only the Telegram selected `ReplyToQuoteText` in the inline current-message block and suppress the full `ReplyToBody`/`ReplyChain` fallback. The source path is clear, but the prompt labels and shared budget when quote and body differ still need maintainer product direction.

## What This Changes

The issue requests preserving selected Telegram quote text as the primary signal while also exposing a bounded full replied-to body excerpt when `ReplyToQuoteText` differs from `ReplyToBody`.

## Best Possible Solution

Keep selected quote text primary, then emit a bounded full-body excerpt only when the selected quote and replied-to body differ, with regression tests for equal, substring, divergent, and short-snippet cases.

## Reproduction Assessment

Yes, at source level: a Telegram context containing both `ReplyToQuoteText` and `ReplyToBody` renders only the selected quote and suppresses the fallback full-body blocks. I did not run live Telegram Desktop proof in this read-only review.

## Solution Assessment

Mostly yes: the requested direction is the narrow maintainable fix, but maintainers still need to decide the exact labels and shared prompt-budget policy before implementation.
