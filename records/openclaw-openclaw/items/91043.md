## Label Justifications

- `P2`: This is a normal-priority core reply-context bug with bounded Telegram/session impact rather than a crash or outage.
- `impact:message-loss`: A selected quote can cause the rest of the replied-to message to be omitted from the model-visible prompt.
- `impact:session-state`: Dropping the full reply target can make the next agent turn lose the conversation item the user is pointing at.

## Summary

Keep open: current main and v2026.6.8 still let Telegram inline ReplyToQuoteText replace ReplyToBody in the model-visible current-message block, and the separate ReplyToBody/ReplyChain block is suppressed once that inline block exists. Related truncation fixes are landed, but the selected-quote versus full-body prompt contract remains unresolved and needs maintainer direction.

## What This Changes

The issue asks OpenClaw to preserve the full replied-to message context when Telegram provides a selected quote span instead of letting ReplyToQuoteText replace ReplyToBody.

## Best Possible Solution

Keep selected quote text as the primary signal, but emit a bounded full-body excerpt when the selected quote is only a span or materially diverges from the replied-to body.

## Reproduction Assessment

Yes from source inspection: a Telegram context with ReplyToQuoteText and ReplyToBody uses only ReplyToQuoteText in the inline current-message block and suppresses the fallback body block. I did not run a live Telegram selected-quote reproduction in this read-only review.

## Solution Assessment

Mostly yes: the proposed direction is the narrow maintainable fix, but maintainers still need to choose the prompt labels and shared budget for quote/body divergence before implementation.
