- `impact:message-loss`: The selected quote path can omit the rest of the replied-to message from the model-visible prompt.

## Summary

Keep open: current main and v2026.6.9 still render only the Telegram selected quote in the inline current-message block, then suppress the separate full-body reply chain/fallback blocks. The source path is clear, but the exact prompt labels and shared quote/body budget remain a maintainer product-contract decision.

## What This Changes

The issue asks OpenClaw to keep Telegram selected quote text primary while also preserving a bounded excerpt of the full replied-to message when the selected quote and body differ.

## Best Possible Solution

Keep selected quote text primary, then emit a bounded full-body excerpt only when the selected quote and replied-to body differ, with regression tests for equal, substring, divergent, and short-snippet cases.

## Reproduction Assessment

Yes, at source level: a Telegram context containing both `ReplyToQuoteText` and `ReplyToBody` renders only the selected quote and suppresses the fallback full-body blocks. I did not run live Telegram proof in this read-only review.

## Solution Assessment

Mostly yes: the requested direction is the narrow maintainable fix, but maintainers still need to decide the exact labels and shared prompt-budget policy before implementation.
