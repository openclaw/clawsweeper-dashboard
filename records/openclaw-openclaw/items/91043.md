- `impact:message-loss`: The model-facing prompt can lose the rest of the replied-to message when Telegram supplies a selected quote span.

## Summary

Keep open: current main and latest release v2026.6.6 still prefer `ReplyToQuoteText` over `ReplyToBody`, so a Telegram selected quote can hide the rest of the replied-to message. The issue is source-reproducible, but the exact quote/body labels and shared budget need maintainer product-contract direction before a fix PR is safe.

## What This Changes

The issue asks OpenClaw to preserve full replied-to context when Telegram supplies a selected reply quote span separately from the full replied-to body.

## Best Possible Solution

Maintainers should choose the selected-quote prompt contract, then update `resolveInlineReplyQuote` and regression coverage so selected quote primacy is preserved without dropping the full replied-to body excerpt when it carries needed context.

## Reproduction Assessment

Yes, at source level: provide Telegram context with both `ReplyToQuoteText` and a different `ReplyToBody`, and current main formats only the selected quote. I did not run a live Telegram reproduction in this read-only review.

## Solution Assessment

Unclear until maintainers choose the prompt contract. The proposed direction is narrow and plausible, but label wording and budget allocation should be decided before implementation.
