## Summary

Keep open. Current main still gates short-term promotion on the aggregate signal count, the memory-core tests still encode daily-only promotion under the default gates, and open PR #74114 is already the active implementation path for this issue.

## What This Changes

The issue requests that short-term memory promotion distinguish real user recall counts from daily/session/grounded signal counts so automated signals do not satisfy or distort recall-based promotion thresholds.

## Best Possible Solution

Codify the threshold contract, then land or revise #74114 so ranking and apply use the intended recall-only gate or an explicitly documented alternative, with tests, docs, and changelog updated to preserve or redefine daily/session and grounded scoring semantics.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for the mixed-signal gate: current source and tests show daily-only signals can clear default promotion gates through aggregate `signalCount`. The exact reported score from the private recall-store fixture is not independently reproducible from repo data.

## Solution Assessment

Is this the best way to solve the issue? Unclear until #74114 is reviewed against the current daily/session and grounded promotion contract. A narrow recall-only gate is plausible, but the final fix should update the encoded tests and docs rather than silently contradicting them.
