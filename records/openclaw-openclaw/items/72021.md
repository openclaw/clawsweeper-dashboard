## Summary

Keep open: current main and the latest release still gate deep promotion on aggregate signalCount, while current tests deliberately allow daily-only promotion, so the report remains valid and needs a maintainer signal-contract decision.

## What This Changes

The issue asks memory-core short-term promotion to stop treating daily/session/grounded aggregate signals as real recalls for minRecallCount eligibility and scoring behavior.

## Best Possible Solution

Maintainers should define whether deep promotion gates require organic recalls or aggregate short-term evidence, then update rank/apply gates, scoring, audit output, docs, and regression tests around that contract.

## Reproduction Assessment

Yes, at source level. Current main still lets aggregate daily/grounded signals satisfy minRecallCount, and an existing test explicitly ranks a recallCount=0, dailyCount=3 candidate under default gates; no live workspace command was run in this read-only review.

## Solution Assessment

Unclear. A recallCount-only gate is a plausible repair for the false-positive path, but current tests and docs preserve aggregate signal promotion, so maintainers should settle the signal policy before code changes.
