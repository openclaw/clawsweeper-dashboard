## Summary

Keep this open. Current main still gates short-term promotion on aggregate signalCount, the test suite still encodes daily-only promotion under default gates, and open PR #74114 is already the active closing-syntax implementation path that should land, be revised, or be closed before this issue is resolved.

## What This Changes

The issue requests that short-term promotion distinguish real recallCount from automated daily/session signal counts when applying the recall-count promotion gate.

## Best Possible Solution

Codify the threshold contract, then land or revise #74114 so rank/apply use the intended recall-only gate while tests, docs, and changelog clearly preserve or redefine daily/session and grounded scoring semantics.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for the mixed-signal gate: current source and tests show daily-only signals can clear default promotion gates through aggregate signalCount. The exact reported numeric score depends on the reporter's private recall-store fixture.

## Solution Assessment

Is this the best way to solve the issue? Unclear until #74114 is reviewed against the existing daily/session and grounded promotion contract. A narrow recall-only gate is plausible, but the final fix should update the encoded tests/docs rather than silently contradicting them.
