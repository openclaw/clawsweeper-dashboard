## Summary

Keep open. Current main still gates short-term promotion on aggregate signalCount rather than real recallCount, and an open closing-syntax PR, #74114, is already the active implementation path for this issue.

## What This Changes

The issue asks memory-core short-term promotion to stop treating daily/session/grounded signals as real recalls for the minRecallCount gate and related scoring behavior.

## Best Possible Solution

Codify the threshold contract, then land or revise #74114 so promotion gates use the intended recall-only behavior while tests, docs, and changelog explicitly preserve or redefine daily/session and grounded scoring semantics.

## Reproduction Assessment

Yes for the mixed-signal gate: current source and tests show daily-only signals can clear default promotion gates through aggregate signalCount. The exact reported score distribution needs the reporter's private recall-store fixture.

## Solution Assessment

Unclear until #74114 is reviewed against the existing daily/session and grounded promotion contract. A narrow recall-only gate is plausible, but the final fix should update encoded tests and docs rather than silently contradicting them.
