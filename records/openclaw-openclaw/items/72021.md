## Summary

Keep open. Current main still gates promotion on aggregate signalCount, and the open closing PR at https://github.com/openclaw/openclaw/pull/74114 is the right place to resolve the memory-core signal-contract decision before this issue closes.

## What This Changes

The issue reports that memory-core short-term promotion uses combined recall, daily, and grounded signals for the minRecallCount gate instead of real recallCount.

## Best Possible Solution

Settle one memory-core promotion signal contract, then update gate logic, diagnostics, tests, docs, and release notes so organic recalls, daily/session evidence, and grounded evidence are handled deliberately.

## Reproduction Assessment

Yes. Source and test inspection show current main aggregates recallCount, dailyCount, and groundedCount for the minRecallCount gate, and the daily-only fixture demonstrates recallCount 0 with dailyCount 3 can clear default gates; I did not run tests in this read-only review.

## Solution Assessment

Unclear as a recallCount-only gate. That direction addresses the reported false eligibility, but the maintainable fix must reconcile or deliberately retire the current daily and grounded promotion contracts.
