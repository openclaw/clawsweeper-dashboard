## Summary

Keep open. Current main still gates short-term promotion on aggregate signalCount rather than real recallCount, and open PR #74114 is already the closing-syntax implementation path for this issue.

## What This Changes

The issue asks memory-core promotion to stop letting daily/session and grounded signals satisfy the real-recall promotion gate.

## Best Possible Solution

Review and land or revise #74114 so the minRecallCount contract is explicit, with tests and docs clarifying how recall, daily/session, and grounded signals affect promotion.

## Reproduction Assessment

Yes for the mixed-signal gate: current source and tests show daily-only signals can clear default promotion gates through aggregate signalCount. The exact reported score distribution needs the reporter's private recall-store fixture.

## Solution Assessment

Unclear until #74114 is reviewed against the current daily/session and grounded promotion semantics. A narrow recall-only gate is plausible, but it should update encoded tests and docs rather than silently contradicting them.
