## Summary

Keep open. Current main still applies the deep promotion recall threshold to aggregate signalCount, and the open closing PR at https://github.com/openclaw/openclaw/pull/74114 has not merged and still needs a maintainer decision because current tests/docs also support daily and grounded promotion evidence.

## What This Changes

The issue asks memory-core short-term promotion to stop treating daily/session/grounded aggregate signals as equivalent to real recallCount for recall gates and scoring.

## Best Possible Solution

Define the promotion contract for organic recalls versus daily/session/grounded evidence, then revise or replace the open PR with aligned implementation, regression tests, docs, and changelog.

## Reproduction Assessment

Yes, from source inspection: current main gates minRecallCount on aggregate signalCount, and the daily-only fixture shows recallCount 0 plus dailyCount 3 can rank under default gates. The exact score-distribution example still needs the reporter's private recall store.

## Solution Assessment

Unclear as a simple recall-only gate. That direction matches the issue's report, but current tests and docs also treat daily and grounded signals as valid promotion evidence, so the maintainable fix needs a deliberate signal-contract update rather than a four-line gate change alone.
