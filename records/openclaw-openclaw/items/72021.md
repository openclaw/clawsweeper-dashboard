## Summary

Keep open. Current main still gates short-term promotion on aggregate signalCount, and the open closing PR has not merged; resolving this needs maintainer policy judgment because current tests and docs also treat daily and grounded evidence as promotion inputs.

## What This Changes

The issue requests separating real recallCount from daily/session and grounded signals in short-term promotion gates and scoring.

## Best Possible Solution

Define the memory-core promotion signal contract for organic recalls, daily/session evidence, and grounded evidence, then land or revise the linked PR with matching implementation, tests, and docs.

## Reproduction Assessment

Yes, at source level: current main aggregates recallCount, dailyCount, and groundedCount for the minRecallCount gate in both ranking and apply paths. The reporter's private score-distribution example is not directly replayable from repository data.

## Solution Assessment

Unclear as a simple recallCount-only gate. That direction addresses the reported false eligibility, but the maintainable fix must preserve or explicitly retire daily/session and grounded promotion semantics instead of changing them silently.
