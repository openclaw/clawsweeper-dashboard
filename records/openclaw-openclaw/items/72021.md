## Summary

Keep open. Current main still has the reported mixed-signal promotion gate, and open PR #74114 is the explicit implementation path for this issue rather than a reason to close it before merge.

## What This Changes

The issue asks for short-term promotion gates to distinguish real recallCount from daily/session/grounded signals so automated signals do not satisfy minRecallCount or suppress real recalled entries.

## Best Possible Solution

Review and land or revise PR #74114 so the minRecallCount contract is explicit, with tests and docs clarifying how recall, daily/session, and grounded signals affect promotion.

## Reproduction Assessment

Yes for the mixed-signal gate: current source and tests show daily-only signals can clear default promotion gates through aggregate signalCount. The exact reported score distribution needs the reporter's private recall-store fixture.

## Solution Assessment

Unclear until PR #74114 is reviewed against current daily/session and grounded promotion semantics. A narrow recall-only gate is plausible, but it should update encoded tests and docs rather than silently contradicting them.
