## Summary

Keep this open. Current main still applies the promotion recall threshold to aggregate signalCount, the tests still encode daily-only promotion under default gates, and open PR #74114 is the active closing-syntax implementation path that should be reviewed, revised, or closed before this issue is resolved.

## What This Changes

The issue requests separating real short-term memory recalls from daily/session/grounded signals for the promotion minRecallCount gate.

## Best Possible Solution

Codify the threshold contract, then land or revise #74114 so rank/apply use the intended recall-only gate while tests, docs, and changelog preserve or explicitly redefine daily/session and grounded scoring semantics.

## Reproduction Assessment

Yes for the mixed-signal gate: current source and tests show daily-only signals can clear default promotion gates through aggregate signalCount. The exact reported numeric score requires the reporter's private recall-store fixture.

## Solution Assessment

Unclear until #74114 is reviewed against the existing daily/session and grounded promotion contract. A narrow recall-only gate is plausible, but the final fix should update the encoded tests and docs rather than silently contradicting them.
