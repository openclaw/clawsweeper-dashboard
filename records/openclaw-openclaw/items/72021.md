## Summary

Keep open. Current main still gates short-term promotion on aggregate signalCount, and the open closing PR has not merged; resolving this needs maintainer policy judgment because existing tests and docs also treat daily and grounded evidence as valid promotion inputs.

## What This Changes

Separate real recall signals from daily/session/grounded aggregate signal counts when short-term memory promotion gates and scores candidates.

## Best Possible Solution

Define the memory-core promotion signal contract for organic recalls, daily/session evidence, and grounded evidence, then update implementation, regression tests, docs, and the linked PR accordingly.

## Reproduction Assessment

Yes, at source level: current main aggregates recallCount, dailyCount, and groundedCount for the minRecallCount gate, and the daily-only fixture demonstrates recallCount 0 plus dailyCount 3 can clear default gates. The reporter's private score-distribution example is not directly reproducible from repository data.

## Solution Assessment

Unclear as a simple recallCount-only gate. That direction addresses the reported false eligibility, but the maintainable fix must first reconcile existing daily and grounded promotion contracts rather than silently removing them.
