## Summary

Keep open. Current main still gates `minRecallCount` on aggregate signal count, and open PR #74114 explicitly closes this issue as the active implementation path.

## What This Changes

Short-term promotion should avoid treating daily/session-derived signals as real recalls for the `minRecallCount` promotion gate.

## Best Possible Solution

Review and land or revise PR #74114 so recall-gate semantics are explicit, with tests aligned for recall, daily/session, and grounded signals.

## Reproduction Assessment

Yes for the mixed-signal gate: current source and the existing test fixture show three daily-only signals can clear default promotion gates with `recallCount` 0. The exact score-distribution example still needs the reporter's private recall-store data.

## Solution Assessment

Unclear until PR #74114 is reviewed. A narrow `recallCount`-only gate is plausible, but the final change should deliberately preserve or revise scoring semantics and update encoded expectations.
