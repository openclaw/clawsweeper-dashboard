## Summary

Keep open. Current main still applies the promotion gate to aggregate signal counts, and the linked open implementation PR at https://github.com/openclaw/openclaw/pull/74114 is the right place to resolve the remaining signal-contract decision and proof gap.

## What This Changes

The issue requests separating organic recall signals from daily/session and grounded signal counts in short-term memory promotion gates and scoring.

## Best Possible Solution

Define one memory-core promotion signal contract for organic recalls, daily/session evidence, and grounded evidence, then update implementation, regression tests, docs, and changelog together.

## Reproduction Assessment

Yes. Source and test inspection show current main aggregates recallCount, dailyCount, and groundedCount for the minRecallCount gate, and the daily-only fixture demonstrates recallCount 0 with dailyCount 3 can clear default gates.

## Solution Assessment

Unclear as a simple recallCount-only gate. That direction addresses the reported false eligibility, but the maintainable fix must reconcile or deliberately retire the current daily and grounded promotion contracts.
