- `P2`: The issue describes a concrete memory promotion correctness problem with limited blast radius and no crash, security, or core-runtime outage evidence.
- `impact:session-state`: The affected behavior is persisted recall, dreaming, and durable memory promotion state drifting from real user recall signals.

## Summary

Keep open. Current main still gates short-term promotion on aggregate signal count, and the open closing PR at https://github.com/openclaw/openclaw/pull/74114 is the right implementation thread to review rather than closing this issue early.

## What This Changes

The issue asks memory-core to stop treating daily/session/grounded signal counts as equivalent to real recallCount for short-term promotion gates and scoring.

## Best Possible Solution

Settle one memory-core promotion signal contract for organic recalls, daily/session evidence, and grounded evidence, then land or replace the linked PR with aligned implementation, regression tests, docs, and release notes.

## Reproduction Assessment

Yes, at source level. Current main still aggregates recallCount, dailyCount, and groundedCount for the minRecallCount gate, and current fixtures show daily-only and grounded-only candidates can clear default promotion paths.

## Solution Assessment

Unclear as a simple recallCount-only gate. That direction addresses the reported false eligibility, but the maintainable fix must preserve or explicitly retire the existing daily and grounded promotion contracts.
