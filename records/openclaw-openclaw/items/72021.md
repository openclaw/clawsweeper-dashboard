## Summary

Keep open. Current main still applies the promotion gate to aggregate signalCount, and an open linked PR is already the implementation candidate, so this should wait for maintainer policy review rather than be closed or duplicated.

## What This Changes

The issue requests separating organic recall counts from daily/session/grounded aggregate signal counts in short-term promotion gates and scoring.

## Best Possible Solution

Define and document one memory-core promotion signal contract for organic recalls, daily/session evidence, and grounded evidence, then update implementation, tests, diagnostics, and docs together.

## Reproduction Assessment

Yes, source-level reproduction is high-confidence: current main aggregates recall, daily, and grounded counts for the minRecallCount gate, and the daily-only fixture shows recallCount 0 with dailyCount 3 can clear default gates. The reporter's private ranking distribution was not replayed.

## Solution Assessment

Unclear as a simple recallCount-only gate. That direction addresses the reported false eligibility, but the maintainable fix must reconcile or deliberately retire the current daily and grounded promotion contracts.
