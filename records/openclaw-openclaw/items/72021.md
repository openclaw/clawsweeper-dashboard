## Summary

Keep open. Current main still applies the recall threshold to aggregate signalCount, and #74114 is the open closing-syntax implementation path rather than a merged fix.

## What This Changes

Short-term memory promotion should distinguish real recallCount from daily/session and other aggregate signals when applying promotion thresholds.

## Best Possible Solution

Review and land or revise #74114, or a successor PR, so the minRecallCount contract is explicit and the tests, docs, and changelog match the chosen daily/session/grounded signal semantics.

## Reproduction Assessment

Yes for the mixed-signal gate: current source and tests show daily-only signals with recallCount 0 and dailyCount 3 can clear default promotion gates. The exact private recall-store scoring example cannot be replayed from the repo alone.

## Solution Assessment

Unclear. A recall-only gate is a plausible narrow fix for the reported false candidates, but it should reconcile or deliberately retire the currently tested daily and grounded promotion behavior.
