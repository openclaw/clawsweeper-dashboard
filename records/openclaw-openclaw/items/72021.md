## Summary

Keep open. Current main still applies the promotion recall threshold to aggregate signalCount, and PR #74114 is already the open closing implementation path for changing that behavior.

## What This Changes

The issue asks for short-term promotion to distinguish real recallCount from daily/session/grounded signal totals when evaluating promotion eligibility and scoring.

## Best Possible Solution

Review and land or revise PR #74114 so minRecallCount semantics are explicit, with tests and docs aligned for recall, daily/session, and grounded signals.

## Reproduction Assessment

Yes for the mixed-signal gate: current source and the existing fixture show three daily-only signals can clear default promotion gates with recallCount 0. The precise score-distribution example still needs the reporter's private recall-store data.

## Solution Assessment

Unclear until PR #74114 is reviewed. A narrow recallCount-only gate is plausible, but the final change should deliberately preserve or revise scoring semantics and update encoded expectations.
