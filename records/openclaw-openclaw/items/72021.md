## Summary

Keep open. Current main and the latest release still gate short-term promotion on aggregate signal counts, and the open closing PR at https://github.com/openclaw/openclaw/pull/74114 remains the active implementation path for the unresolved signal-contract decision.

## What This Changes

The issue asks memory-core short-term promotion to stop treating daily/session promotion signals as equivalent to real user recalls for `minRecallCount` eligibility and scoring.

## Best Possible Solution

Settle one memory-core promotion signal contract, then land the linked PR or a replacement with implementation, regression tests, docs, and changelog aligned.

## Reproduction Assessment

Yes, source-level reproduction is high-confidence. Current main still aggregates recall, daily, and grounded counts for the `minRecallCount` gate, and current fixtures demonstrate daily-only candidates with `recallCount: 0` clearing default gates; I did not run tests in this read-only review.

## Solution Assessment

Unclear as a recallCount-only gate. That direction addresses the reported false eligibility, but the maintainable fix must deliberately preserve or retire current daily and grounded promotion contracts.
