## Summary

Keep open. Current main still applies the promotion recall threshold to aggregate signalCount, and open PR #74114 explicitly closes this issue but has not merged.

## What This Changes

Short-term memory promotion should treat minRecallCount as real recallCount instead of combined recall, daily, and grounded signal counts.

## Best Possible Solution

Review and land or revise #74114 so promotion gate semantics are explicit, with aligned tests for recall, daily/session, and grounded signals.

## Reproduction Assessment

Yes for the mixed-signal gate from source: current code gates on aggregate signalCount, and the existing fixture shows three daily-only signals can rank with recallCount 0. The exact score-distribution example still needs the reporter's private recall-store data.

## Solution Assessment

Unclear until #74114 is reviewed. A narrow recallCount-only gate matches the docs and CLI inspection semantics, but maintainers should confirm whether aggregate signalCount should still drive avgScore and frequency.
