## Summary

Keep this open. Current main still applies the promotion recall threshold to aggregate signalCount, and the open closing-syntax PR #74114 is the active implementation path that should be reviewed, revised, or closed before resolving this issue.

## What This Changes

The issue asks for short-term memory promotion gates to distinguish real recallCount from daily/session and grounded aggregate signals when enforcing minRecallCount.

## Best Possible Solution

Codify the threshold contract, then land or revise #74114 or an equivalent patch so rank and apply use the intended gate while tests, docs, and changelog preserve or explicitly redefine daily/session and grounded scoring semantics.

## Reproduction Assessment

Yes for the mixed-signal gate: current source and tests show daily-only signals can clear default promotion gates through aggregate signalCount. The exact reported numeric score requires the reporter's private recall-store fixture.

## Solution Assessment

Unclear until #74114 is reviewed against the existing daily/session and grounded promotion contract. A narrow recall-only gate is plausible, but the final fix should update encoded tests and docs rather than silently contradicting them.
