## Summary

Keep open. Current `main` still gates memory-core promotion on aggregate signal count, and the open closing PR at https://github.com/openclaw/openclaw/pull/74114 is the focused implementation path for the unresolved promotion-signal contract.

## What This Changes

The issue asks memory-core short-term promotion to stop treating daily/session/grounded signals as equivalent to real `recallCount` for the promotion recall gate and related scoring behavior.

## Best Possible Solution

Settle one memory-core promotion signal contract, then land the gate logic, regression tests, diagnostics/docs, and release notes together through the linked implementation path.

## Reproduction Assessment

Yes. Source inspection on current `main` shows the rank and apply paths still gate on aggregated `recallCount + dailyCount + groundedCount`, and existing fixtures cover daily-only eligibility; I did not run tests because this is a read-only review.

## Solution Assessment

Unclear as a pure recallCount-only change. That direction addresses the reported false eligibility, but the best fix must explicitly preserve, replace, or retire the current daily and grounded promotion contracts.
