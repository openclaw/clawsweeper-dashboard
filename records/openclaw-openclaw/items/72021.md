## Summary

Keep open. Current main and the latest release still apply the promotion `minRecallCount` gate to aggregate `signalCount`, and an open linked PR is already the implementation candidate, but the fix needs maintainer policy review because current tests and docs also treat daily and grounded signals as valid promotion evidence.

## What This Changes

Requests that short-term memory promotion stop letting daily/session signals satisfy recall-based promotion gates or otherwise recalibrate signal scoring so real recalls are not hidden.

## Best Possible Solution

Define and document one memory-core promotion signal contract for organic recalls, daily/session evidence, and grounded evidence, then update the linked PR, tests, docs, and diagnostics to match that contract.

## Reproduction Assessment

Yes, source-level reproduction is high-confidence: current main aggregates recall, daily, and grounded counts for the promotion gate, and the daily-only fixture shows `recallCount: 0` plus `dailyCount: 3` can clear default gates. The reporter's private ranking distribution is not directly reproducible from repository data.

## Solution Assessment

Unclear as a simple recallCount-only gate. That direction addresses the reported false eligibility, but the maintainable fix must reconcile existing daily and grounded promotion contracts instead of silently removing them.
