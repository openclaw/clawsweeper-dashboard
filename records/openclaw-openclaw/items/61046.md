## Summary

## What This Changes

The PR changes configured `models list` resolution to infer a provider for bare model keys and fallback entries, and adds unit coverage for those inferred-provider cases.

## Best Possible Solution

Land a rebased narrow fix that applies providerless resolution consistently to configured rows, fallback rows, and alias indexing, with alias regression coverage and current display-normalization guards preserved.

## Reproduction Assessment

Yes. Source inspection shows a config with `models.providers.google` containing a bare Gemini model still flows through `DEFAULT_PROVIDER` in the configured-list path on current main.

## Solution Assessment

No, not as submitted. Inferring the provider is the right direction, but the alias index and alias-based fallback resolution need the same provider inference before the patch is maintainable.
