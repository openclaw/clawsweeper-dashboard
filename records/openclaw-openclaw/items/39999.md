## Summary

Keep open. The PR targets a real update-safety gap that current main has not implemented, but the branch is outdated and incorrect because rollback is added in the old CLI-local path while current package updates run through the shared helper, and no real behavior proof was provided.

## What This Changes

The PR adds a CLI update rollback branch that reinstalls `packageName@beforeVersion` and reruns doctor when post-update doctor fails after a package-manager update.

## Best Possible Solution

Implement accepted rollback behavior in `runGlobalPackageUpdateSteps` after a successful package replacement plus failing post-verify doctor, shared by CLI and gateway, with focused regression coverage and clear operator-facing result reporting.

## Reproduction Assessment

Yes for the review findings: the PR diff and current-main source show the rollback branch is CLI-local and only gated on doctor failure plus `beforeVersion`. I did not run a live global package update in this read-only review.

## Solution Assessment

No. The targeted rollback idea may be useful, but this patch should be moved to the shared package-update helper, gated after a successful package replacement, and accepted as update policy before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Move rollback into the shared update helper:** `src/cli/update-cli/update-command.ts:312`
  - body: Current main routes both CLI and gateway package-manager updates through `runGlobalPackageUpdateSteps`. Adding rollback only to this old CLI-local body misses the Control UI/gateway path and leaves the maintained update boundary without rollback behavior.
  - confidence: 0.86
- **[P2] Gate rollback after a successful package replacement:** `src/cli/update-cli/update-command.ts:312`
  - body: The rollback branch runs whenever doctor fails and `beforeVersion` exists. If the package-manager update failed before replacing the live install, this can reinstall the already-installed version and obscure the actual install failure; require a successful update/swap before rollback.
  - confidence: 0.9
