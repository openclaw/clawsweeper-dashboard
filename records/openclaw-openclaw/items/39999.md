## Summary

Keep open: the rollback idea is still useful, but this stale PR is not merge-ready because current main moved package updates into a shared helper and the branch only patches the older CLI-local path.

## What This Changes

The PR adds a CLI-local rollback attempt that reinstalls the previous global package version and reruns doctor when post-update doctor fails.

## Best Possible Solution

Implement targeted rollback in runGlobalPackageUpdateSteps after a successful package replacement plus failing post-verify doctor, shared by CLI and gateway, with focused regression coverage; keep broader snapshot/state restore policy under https://github.com/openclaw/openclaw/issues/14526.

## Reproduction Assessment

Yes for the review findings: source inspection shows the PR adds rollback under the older CLI-local package-update body, while current main uses runGlobalPackageUpdateSteps for both CLI and gateway package updates. I did not run a live global package update in this read-only review.

## Solution Assessment

No. The narrow rollback behavior is reasonable, but the patch should move to the shared package-update helper and gate rollback on a successful package replacement before attempting a reinstall.
