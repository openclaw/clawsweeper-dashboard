## Summary

Keep open. The PR addresses a real updater safety gap, but the current branch adds rollback to a stale CLI-local path, lacks after-fix real behavior proof, and overlaps the broader safe-update policy discussion.

## What This Changes

The PR adds a previous-version global package reinstall and second doctor run after post-update doctor failure in the CLI package-update path.

## Best Possible Solution

Implement targeted rollback in runGlobalPackageUpdateSteps only after a successful package replacement followed by a failed postVerifyStep, with CLI and gateway regression coverage and coordination with the safe-update policy item.

## Reproduction Assessment

Yes for source-level review: current main’s runGlobalPackageUpdateSteps appends postVerifyStep and returns doctor-failed without rollback, and update-runner.test.ts locks in that behavior. I did not run a live global package update in this read-only review.

## Solution Assessment

No. The rollback direction is plausible, but this patch should target the shared package-update helper and gate on successful package replacement; automatic rollback scope should be coordinated with https://github.com/openclaw/openclaw/issues/14526.
