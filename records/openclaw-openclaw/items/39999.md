## Summary

Keep open, but the patch is not ready: current main still lacks targeted rollback after a failed package post-verify doctor step, while this branch is conflicting, implements the rollback in a stale CLI-local path, and lacks real behavior proof.

## What This Changes

The PR adds a previous-version global package reinstall and a second doctor run when the post-update doctor step fails.

## Best Possible Solution

Implement targeted rollback in runGlobalPackageUpdateSteps after a successful package replacement followed by a failed postVerifyStep, with CLI and gateway regression coverage and a clear boundary versus the broader safe-update workflow.

## Reproduction Assessment

Yes for source-level review: current main can be traced through runGlobalPackageUpdateSteps, which reports a failed post-verify doctor step as an error without rollback. I did not run a live global package update in this read-only review.

## Solution Assessment

No. The rollback direction is reasonable, but this patch targets a stale CLI-local path and should move to the shared helper with rollback gated to a successful package replacement.
