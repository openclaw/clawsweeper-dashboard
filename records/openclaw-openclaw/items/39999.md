## Summary

Keep open: current main still lacks automatic previous-version rollback after a successful global package update followed by a failing post-update doctor step, but this PR is out of date and needs a shared-helper replacement before merge.

## What This Changes

The PR adds a CLI global package rollback that reinstalls the previous package version when post-update doctor fails, then optionally reruns doctor on the rolled-back entrypoint.

## Best Possible Solution

Land a current-main replacement that adds previous-version rollback at the shared package-update boundary only after a successful package replacement/swap followed by post-verify doctor failure, with rollback doctor context, focused tests, and a changelog entry.

## Reproduction Assessment

Yes for the review finding: the PR diff statically shows rollback is gated only on doctor failure plus `beforeVersion`. For the broader bug, current main source and tests show doctor failure returns an error without automatic previous-version rollback, but no live global update was executed.

## Solution Assessment

No. The rollback safety net is useful, but this PR is not the best mergeable form because current main routes CLI and gateway package updates through the shared helper, and the PR also needs a successful-install guard.
