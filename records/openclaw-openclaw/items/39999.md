## Summary

Keep open. The rollback goal is still useful, but the branch targets an older CLI-local update body while current main uses a shared package-update helper for CLI and gateway, and the external PR still lacks real behavior proof.

## What This Changes

The PR adds a CLI rollback step that reinstalls the previous global package version and reruns doctor when post-update doctor fails.

## Best Possible Solution

Implement targeted rollback in `runGlobalPackageUpdateSteps` after a successful package replacement plus failed post-verify doctor, shared by CLI and gateway, with focused regression coverage; keep broader snapshot/state restore policy under https://github.com/openclaw/openclaw/issues/14526.

## Reproduction Assessment

Yes for review purposes: the PR diff and current source show the rollback is attached to the older CLI-local path while current main routes CLI and gateway package updates through the shared helper. I did not run a live global package update in this read-only review.

## Solution Assessment

No. The narrow rollback behavior is reasonable, but the patch should move to the shared package-update helper and gate rollback on a successful package replacement before attempting a reinstall.
