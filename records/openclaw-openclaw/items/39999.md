## Summary

Keep open for maintainer review, but the current branch should not merge as-is: it adds rollback in an older CLI-local path, while current main centralizes package updates in the shared helper used by both CLI and gateway updates, and the PR still lacks after-fix real behavior proof.

## What This Changes

The PR adds a previous-version global package reinstall and a second doctor run after a failed post-update doctor step in the CLI package-update path.

## Best Possible Solution

If maintainers want automatic doctor-failure rollback, implement it in runGlobalPackageUpdateSteps after a successful package replacement, cover both CLI and gateway callers, and align the rollback policy with the broader safe-update design.

## Reproduction Assessment

Yes for source-level review: current main runs the post-verify doctor through runGlobalPackageUpdateSteps and reports the failed doctor step without rollback. I did not run a live global package update in this read-only review.

## Solution Assessment

No. The rollback direction may be useful, but this patch puts it in the wrong boundary and should also gate rollback on a successful package replacement before the doctor failure.
