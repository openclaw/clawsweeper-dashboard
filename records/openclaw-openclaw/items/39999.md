## Summary

Keep open, but this patch is not mergeable as-is: current main routes package updates through the shared helper for CLI and gateway, while this PR modifies an older CLI-local path and lacks real behavior proof.

## What This Changes

The branch adds rollback to the package-install update path when post-update doctor fails, reinstalling the previous version and optionally rerunning doctor.

## Best Possible Solution

Implement targeted rollback in `runGlobalPackageUpdateSteps` after a successful package replacement followed by failed doctor/post-verify, shared by CLI and gateway, with focused regression coverage while keeping the broader safe-update workflow under https://github.com/openclaw/openclaw/issues/14526.

## Reproduction Assessment

Yes for source-level review: current main can be traced to `runGlobalPackageUpdateSteps`, which reports a failed post-verify doctor step as an error without rollback. I did not run a live global package update in this read-only review.

## Solution Assessment

No. The rollback direction is reasonable, but the current patch targets a stale CLI-local boundary and should instead live in the shared helper with a guard that rollback only follows a successful package replacement.
