## Summary

Keep open for maintainer handling: current main still reports failed post-update doctor without automatic reinstall rollback, but this branch is not mergeable as-is because it patches an old CLI-local path, misses the shared gateway package-update helper, and has no after-fix real behavior proof.

## What This Changes

The PR modifies the CLI package-update path to reinstall the previous global package version and rerun doctor when the post-update doctor step fails.

## Best Possible Solution

If maintainers want automatic doctor-failure rollback, implement it in `runGlobalPackageUpdateSteps` after a confirmed successful package replacement, cover CLI and gateway callers, and align the policy with the broader safe-update design.

## Reproduction Assessment

Yes for source-level review: current main can be traced through `runPackageInstallUpdate` and `runGatewayUpdate` into `runGlobalPackageUpdateSteps`, where failed post-verify doctor becomes `failedStep` without reinstalling the previous version. I did not run a live global package update in this read-only review.

## Solution Assessment

No. The rollback direction may be useful, but this patch puts it in the wrong boundary and should also gate rollback on a successful package replacement before doctor failure.
