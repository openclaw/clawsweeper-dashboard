## Summary

Keep open. The rollback idea remains useful, but this branch is not merge-ready because it patches an old CLI-local path, omits a successful-replacement guard, and lacks required real behavior proof.

## What This Changes

The PR adds a rollback attempt that reinstalls the previous global package version and reruns doctor when the post-update doctor step fails.

## Best Possible Solution

Implement targeted rollback in runGlobalPackageUpdateSteps after successful package replacement plus failed postVerifyStep, shared by CLI and gateway, with focused regression coverage; keep broader snapshot/state restore under https://github.com/openclaw/openclaw/issues/14526.

## Reproduction Assessment

Yes for the review findings: the PR diff and current-main source show the rollback is CLI-local and gated only on doctor failure plus beforeVersion. I did not run a live global package update in this read-only review.

## Solution Assessment

No. The targeted rollback behavior is reasonable, but this patch should move to the shared package-update helper and require a successful install replacement before attempting rollback.
