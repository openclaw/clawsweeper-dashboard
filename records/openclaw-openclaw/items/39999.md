## Summary

Keep open. The rollback request is not implemented on current main, and this PR still represents a plausible safety improvement, but the submitted patch is not mergeable as-is because it targets a stale CLI-local path, misses the shared gateway update helper path, lacks real behavior proof, and needs a clear automatic rollback boundary.

## What This Changes

The PR adds a CLI-local previous-version global package reinstall and second doctor run when post-update doctor fails.

## Best Possible Solution

Implement targeted rollback in runGlobalPackageUpdateSteps after a successful package replacement followed by a failed postVerifyStep, with CLI and gateway regression coverage and explicit coordination with the broader safe-update policy.

## Reproduction Assessment

Yes for source-level review: current main can be traced through runGlobalPackageUpdateSteps, which reports a failed post-verify doctor step as an error without rollback. I did not run a live global package update in this read-only review.

## Solution Assessment

No. The rollback direction is reasonable, but this patch should move to the shared package-update helper and rollback only after a successful package replacement; maintainers should also coordinate it with https://github.com/openclaw/openclaw/issues/14526.
