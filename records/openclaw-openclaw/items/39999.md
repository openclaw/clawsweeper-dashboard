## Summary

Keep open: the rollback idea is still useful, but this branch is not merge-ready against current main because it patches an old CLI-local path, misses the shared gateway/package update boundary, triggers rollback without proving the update replaced the install, and lacks after-fix real behavior proof.

## What This Changes

The branch adds a CLI-local rollback step that reinstalls the previous package version and reruns doctor when post-update doctor fails.

## Best Possible Solution

Implement rollback in `runGlobalPackageUpdateSteps` after a successful package replacement plus failing post-verify doctor, shared by CLI and gateway, with focused regression coverage; keep broader snapshot/state restore under https://github.com/openclaw/openclaw/issues/14526.

## Reproduction Assessment

Yes for the review findings: the PR diff and current-main source show the rollback is CLI-local and gated only on doctor failure plus `beforeVersion`. I did not run a live global package update in this read-only review.

## Solution Assessment

No. The targeted rollback behavior is reasonable, but this patch should move to the shared package-update helper and gate rollback on a successful package replacement before attempting a reinstall.

## Review Findings

Overall correctness: patch is incorrect

- **[P2] Move rollback into the shared update helper:** `src/cli/update-cli/update-command.ts:312`
