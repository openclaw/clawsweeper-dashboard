## Summary

Keep open. The target failure mode still exists on current main, but this PR is not merge-ready because it targets an obsolete CLI-local update boundary, misses the gateway path, lacks contributor real behavior proof, and has a rollback guard that can run after a failed package replacement.

## What This Changes

The PR adds a previous-version reinstall rollback attempt when a global package update's post-update doctor step fails.

## Best Possible Solution

Implement targeted rollback in runGlobalPackageUpdateSteps after a successful package replacement followed by a failed post-verify doctor step, with CLI and gateway regression coverage and coordination with the broader safe-update proposal.

## Reproduction Assessment

Yes for source-level review: current main can be traced through runGlobalPackageUpdateSteps, which reports a failed post-verify doctor step as an error without rollback. I did not run a live global package update in this read-only review.

## Solution Assessment

No. The rollback direction is reasonable, but the current patch targets a stale CLI-local boundary and should instead live in the shared helper with rollback gated to a successful package replacement.
