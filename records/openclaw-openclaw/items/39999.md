## Summary

Keep open: the rollback idea is useful, but this PR targets an obsolete CLI-local path, misses the current shared package-update helper used by gateway updates, and still lacks required real behavior proof.

## What This Changes

The PR adds CLI-local rollback that reinstalls the previous global package version and reruns doctor when post-update doctor fails.

## Best Possible Solution

Implement targeted rollback in runGlobalPackageUpdateSteps after a successful package replacement plus failed post-verify doctor, shared by CLI and gateway, with focused regression coverage while keeping the broader safe-update workflow under https://github.com/openclaw/openclaw/issues/14526.

## Reproduction Assessment

Yes for review purposes: the PR diff and current source show the rollback is attached to an older CLI-local path while current main routes CLI and gateway package updates through runGlobalPackageUpdateSteps. I did not run a live global package update in this read-only review.

## Solution Assessment

No. The rollback direction is reasonable, but the patch should move to the shared package-update helper and only attempt rollback after a successful package replacement followed by failed doctor verification.
