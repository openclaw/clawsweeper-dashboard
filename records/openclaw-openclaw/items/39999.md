## Summary

Keep open: the rollback idea is still useful, but this branch is not mergeable against current main because package updates now flow through a shared helper and the PR adds rollback only to the older CLI-local path.

## What This Changes

The PR adds a CLI-local rollback branch that reinstalls `packageName@beforeVersion` and reruns doctor when package-update doctor fails.

## Best Possible Solution

Implement rollback in `runGlobalPackageUpdateSteps` after a successful package replacement plus failing post-verify doctor, shared by CLI and gateway, with focused regression coverage; keep broader snapshot/state restore under the related safe-update proposal.

## Reproduction Assessment

Yes for the review findings: the PR diff and current-main source show the rollback is CLI-local and gated only on doctor failure plus `beforeVersion`. I did not run a live global package update in this read-only review.

## Solution Assessment

No. The targeted rollback behavior is reasonable, but this patch should be moved to the shared package-update helper and gated on a successful package replacement before rollback is attempted.
