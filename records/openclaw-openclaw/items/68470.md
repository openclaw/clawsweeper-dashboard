## Summary

Keep open. Current main still lacks the reported MiniMax cache-hit alias, the shared usage helper still only de-dupes known OpenAI/Kimi cache fields, and timeout compaction still depends on additive prompt-token math. The linked implementation candidate remains open and should be reviewed, landed, or replaced before this issue closes.

## What This Changes

The issue requests MiniMax prompt cache-hit usage normalization so cached input is not counted twice in context and timeout-compaction accounting.

## Best Possible Solution

Keep this issue open until #68750 or an equivalent narrow fix lands. The fix should normalize the MiniMax cache-hit usage shape at the shared usage boundary, preserve the existing additive internal usage contract, and add focused regression coverage in the usage helper tests.

## Reproduction Assessment

Yes for a focused source-level reproduction: a prompt-total usage shape with separately surfaced cache-read tokens still feeds additive `derivePromptTokens`, and the reported alias is absent from current main. A live MiniMax run would be useful before merging the PR, but it is not needed to decide that the issue should stay open.

## Solution Assessment

Yes, the narrow maintainable direction is to extend `src/agents/usage.ts` normalization and add `src/agents/usage.test.ts` coverage, rather than changing compaction thresholds or disabling memory flush. The open PR should also verify the exact MiniMax payload shape before merge.
