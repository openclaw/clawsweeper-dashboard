## Summary

Keep open. Current main still lacks the MiniMax `prompt_cache_hit_tokens` usage alias, the shared prompt-token helper still uses additive cache math, and #68750 is an open unmerged implementation candidate linked with closing syntax.

## What This Changes

The report asks OpenClaw to normalize MiniMax prompt-cache hit usage so cached prompt tokens are not counted twice in context and compaction math.

## Best Possible Solution

Keep this issue open until #68750 or an equivalent narrow fix lands with focused usage-helper regression coverage.

## Reproduction Assessment

Yes. A source-level reproduction is clear: current `normalizeUsage` does not recognize the reported MiniMax alias, while downstream context math still adds `cacheRead` to `input`.

## Solution Assessment

Yes. The narrow maintainable fix is to normalize the MiniMax cache-hit shape at `src/agents/usage.ts` and test it there, rather than changing compaction thresholds or disabling memory flush.
