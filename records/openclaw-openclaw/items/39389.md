## Summary

Keep open: the PR addresses a real Feishu docs gap, but the current wording can still mislead multi-account users by conflating the inbound default routing bucket with Feishu outbound `defaultAccount`. The existing narrow docs finding remains actionable.

## What This Changes

The PR adds Feishu channel documentation for `match.accountId` semantics and an `accountId: "*"` cross-account routing example.

## Best Possible Solution

Amend the Feishu docs to say omitted `match.accountId` maps to OpenClaw's literal default routing bucket, distinguish that from outbound `channels.feishu.defaultAccount`, and keep `accountId: "*"` as the cross-account binding example.

## Reproduction Assessment

Yes. The docs defect is reproducible from source by comparing the PR wording with `src/routing/resolve-route.ts`, the route tests, and Feishu inbound's `account.accountId` route call.

## Solution Assessment

No. A Feishu-specific clarification is the right surface, but this patch should revise the wording so it does not conflate inbound default-bucket routing with Feishu's outbound `defaultAccount` setting.

## Review Findings
