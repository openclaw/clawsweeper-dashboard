## Summary

Keep open; the PR is useful but should not merge as-is because its “default account” wording is ambiguous on the Feishu page, where `defaultAccount` already means outbound account selection while inbound routing uses a literal default routing bucket.

## What This Changes

The PR adds a Feishu docs bullet for `match.accountId` and a wildcard `accountId: "*"` routing example in `docs/channels/feishu.md`.

## Best Possible Solution

Update the Feishu page to distinguish the literal default routing bucket from `channels.feishu.defaultAccount`, then keep the `accountId: "*"` cross-account example.

## Reproduction Assessment

Yes. The docs defect is reproducible by comparing the PR wording with `src/routing/resolve-route.ts`, its route tests, and Feishu inbound’s concrete `account.accountId` call path.

## Solution Assessment

No. A Feishu-specific clarification is the right direction, but this patch should revise the wording so it does not conflate inbound routing’s literal default bucket with Feishu’s outbound `defaultAccount` setting.
