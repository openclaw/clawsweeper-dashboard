## Summary

This PR should stay open: current main still lacks Feishu-specific `match.accountId` routing guidance, but the proposed wording can mislead multi-account users by conflating inbound routing's literal default bucket with Feishu's outbound `defaultAccount` setting.

## What This Changes

The PR adds Feishu docs for omitted `match.accountId`, `accountId: "*"`, and a cross-account DM routing example.

## Best Possible Solution

Revise the Feishu docs to name OpenClaw's literal default routing bucket, distinguish it from outbound `channels.feishu.defaultAccount`, and keep `accountId: "*"` as the cross-account binding guidance.

## Reproduction Assessment

Yes. The docs defect is reproducible by comparing the PR wording with current Feishu docs, routing source, route tests, and Feishu inbound's concrete `account.accountId` call path.

## Solution Assessment

No. A Feishu-specific clarification is the right direction, but the patch should avoid saying omitted `match.accountId` matches Feishu's configured default account.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9
