## Summary

Keep this PR open, but not mergeable as-is: the Feishu-specific docs clarification is useful, yet the added “default account” wording can be read as `channels.feishu.defaultAccount`, which current routing does not consult for inbound binding matches.

## What This Changes

The PR adds Feishu `match.accountId` documentation and an `accountId: "*"` example for cross-account Feishu routing.

## Best Possible Solution

Amend the Feishu page to distinguish omitted `match.accountId` as OpenClaw's default routing bucket from `channels.feishu.defaultAccount`, and show `accountId: "*"` for cross-account Feishu bindings.

Yes. The docs defect is source-reproducible by comparing the PR wording with `src/routing/resolve-route.ts`, the existing route tests, and Feishu inbound’s concrete `account.accountId` call path.

## Solution Assessment
