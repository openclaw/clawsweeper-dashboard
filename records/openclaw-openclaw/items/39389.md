## Summary

Keep this PR open: the Feishu docs gap still exists on current main, but the proposed wording is still ambiguous against the actual routing bucket behavior and the external PR lacks required real behavior proof.

## What This Changes

The PR adds Feishu documentation for `match.accountId`, including a note that omitted account scope is not wildcard scope and an `accountId: "*"` example.

## Best Possible Solution

Revise the Feishu page to document `match.accountId` accurately as the literal `default` routing bucket versus `"*"`, keep the wildcard example, rebase onto current docs, and validate the rendered page.

## Reproduction Assessment

Yes. The docs defect is source-reproducible by comparing the proposed wording with `src/routing/resolve-route.ts`, `src/routing/resolve-route.test.ts`, and Feishu's concrete inbound `account.accountId` route call.

## Solution Assessment

No. A Feishu-specific clarification is the right direction, but this wording should distinguish the literal `default` routing bucket from `channels.feishu.defaultAccount` before merge.
