## Best Possible Solution

Update the Feishu page to name the literal `default` routing bucket separately from `channels.feishu.defaultAccount`, keep the wildcard example, and validate the rendered docs.

## Reproduction Assessment

Yes. The docs defect is source-reproducible by comparing the PR wording with `src/routing/resolve-route.ts`, `src/routing/resolve-route.test.ts`, and the Feishu inbound `account.accountId` call path.

## Solution Assessment

No. A Feishu-specific clarification is the right direction, but this wording should distinguish the literal `default` routing bucket from Feishu's outbound `defaultAccount` setting before merge.
