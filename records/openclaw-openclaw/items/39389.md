## Summary

Keep open. Current main still has the Feishu-specific documentation gap, but this PR should not merge as-is because the added “default account” wording is ambiguous on a page where `channels.feishu.defaultAccount` is documented as outbound-only while inbound routing treats omitted `match.accountId` as the literal default routing bucket.

## What This Changes

The PR adds Feishu channel documentation for binding `match.accountId` semantics, including omitted-account behavior and an `accountId: "*"` wildcard routing example.

## Best Possible Solution

Revise `docs/channels/feishu.md` to list `match.accountId`, distinguish omitted default-bucket matching from explicit account ids and `accountId: "*"`, keep a Feishu-specific wildcard example, and explicitly separate inbound binding scope from `channels.feishu.defaultAccount` outbound account selection.

## Reproduction Assessment

Yes. The docs gap and wording risk are reproducible by comparing the current Feishu page with `src/routing/resolve-route.ts`, the route tests, and Feishu inbound’s concrete `account.accountId` call path.

## Solution Assessment

No. A Feishu-specific clarification is the right direction, but the current patch should be revised to avoid conflating the default routing bucket with Feishu’s outbound `defaultAccount` setting.
