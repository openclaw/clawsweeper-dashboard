## Summary

Keep open: current main still lacks the Feishu-specific `match.accountId` documentation, but the branch's "default account" wording is ambiguous against Feishu's outbound `defaultAccount`; the PR also lacks contributor real behavior proof. The related broader accountId work at https://github.com/openclaw/openclaw/pull/49769 covers generic guidance and CLI behavior, but it does not replace this Feishu page change.

## What This Changes

The PR adds Feishu docs wording for `match.accountId` and a wildcard `accountId: "*"` routing example in `docs/channels/feishu.md`.

## Best Possible Solution

Update the Feishu page to distinguish the literal `default` binding bucket from `channels.feishu.defaultAccount`, keep the wildcard example, and verify the rendered docs.

## Reproduction Assessment

Yes. Source inspection reproduces the documentation defect: current Feishu docs omit `match.accountId`, routing maps omitted accountId to the literal default bucket, tests show named accounts fall back unless `accountId: "*"` is used, and Feishu inbound passes concrete account ids.

## Solution Assessment

No. A Feishu-specific note and wildcard example are the right direction, but the proposed sentence should name the literal `default` routing bucket instead of Feishu's outbound default account before merge.
