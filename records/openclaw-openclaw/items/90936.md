## What This Changes

The PR adds `isSignalTimeoutReason`, switches the two embedded-agent external abort handlers to use it instead of broad timeout heuristics, and adds helper coverage for AbortError versus TimeoutError reasons.

## Best Possible Solution

Land this narrow abort-signal classification fix after maintainer review, and keep provider-side AbortError fallback behavior on its separate canonical PR.

## Reproduction Assessment

Yes from source inspection and reporter fleet evidence, though not from a live 180-second proxy-disconnect run. Current main routes a no-reason client-disconnect abort into the embedded attempt, and Node reports that reason as `AbortError: This operation was aborted`, which the broad timeout classifier can match.

## Solution Assessment

Yes. Narrowing only abort-signal reason classification is the best fit because it preserves broad provider/failover error matching while distinguishing intentional `TimeoutError` sources from plain client cancellations.
