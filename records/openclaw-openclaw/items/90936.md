## Summary

Keep open. The patch is a narrow, source-backed fix for a real embedded-agent abort classification bug, and this maintainer/member PR should proceed through review/automerge gates rather than cleanup closure.

## What This Changes

The PR adds an abort-signal-specific timeout classifier, switches the two embedded attempt external-abort handlers to it, and adds classifier regression tests.

## Best Possible Solution

Land the narrow abort-signal classifier change after normal maintainer, CI, and automerge gates while keeping the broader provider failover timeout heuristic unchanged.

## Reproduction Assessment

Yes for the core code path: current main aborts client disconnects with no reason, Node reports that as AbortError, and current main feeds that reason into the broad timeout classifier. A full live reverse-proxy idle-eviction reproduction was not run.

## Solution Assessment

Yes, this is the best narrow fix: timeout-vs-cancel classification belongs at the abort-signal decision point, while isTimeoutError should stay broad for provider and transport failover errors. Changing watchClientDisconnect to synthesize a special cancellation reason would be broader and would not address other plain AbortError signal sources as directly.
