## What This Changes

The PR adds an abort-signal-specific timeout classifier, switches the two embedded attempt abort handlers to it, and adds focused failover-error tests.

## Best Possible Solution

Land the narrow signal-reason classifier after normal CI, keeping broad isTimeoutError heuristics for provider and failover error classification.

## Reproduction Assessment

Yes, at source level: current main has gateway disconnects abort without a reason and embedded attempts call the broad timeout classifier on that AbortError reason. I did not run a full reverse-proxy idle-eviction scenario.

## Solution Assessment

Yes. This is the best fix shape because it narrows only abort-signal reason classification while preserving isTimeoutError for general provider and failover error messages.
