## Summary

This PR is still necessary: current main still routes client-disconnect abort reasons through broad timeout text matching, while the branch narrows only the embedded abort-signal timeout decision and leaves provider error heuristics intact. I found no blocking patch findings.

## What This Changes

Adds an abort-signal-specific timeout classifier, uses it in the two embedded attempt abort handlers, and adds focused failover-error coverage.

## Best Possible Solution

Land the narrow signal-reason classifier after normal maintainer review and CI, while keeping the broader provider timeout heuristics for general LLM error classification.

## Reproduction Assessment

Yes from source and runtime-contract inspection: the gateway aborts on socket close without a custom reason, Node produces a plain `AbortError`, and current main feeds that reason into broad timeout text matching. I did not run a full gateway reverse-proxy idle eviction scenario.

## Solution Assessment

Yes. The patch fixes the decision at the abort-signal boundary, which is narrower and safer than weakening `isTimeoutError` for all provider and transport error classification.
