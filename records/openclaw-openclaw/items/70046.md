## Summary

Keep open: this is a plausible scoped parser change for a current-main gap, but the external PR still lacks after-fix real behavior proof and has one small deterministic-test issue to address before merge.

## What This Changes

The branch adds `HH:MM`/`HH:MM:SS` parsing for cron `--at`, applies `--tz` using today's date in that timezone, rejects invalid time-only inputs/timezones, and adds parser tests.

## Best Possible Solution

Land the focused parser support after real behavior proof is added and the date-boundary assertions use a captured or frozen clock; keep broader help-text wording coordinated with https://github.com/openclaw/openclaw/pull/69975.

## Reproduction Assessment

Yes for the current rejection path by source inspection and a focused Node check: current main has no time-only branch, `resolveDirectSchedule` turns null into `Invalid --at`, and `Date.parse('09:00')` is `NaN`. This is source-reproducible rather than a full CLI reproduction in this read-only pass.

## Solution Assessment

Yes, with one small test fix. `parseAt` is the narrow layer that has access to `--tz` and can reuse the existing offset-less zoned datetime helper without changing the lower-level absolute-time parser.

## Review Findings

Overall correctness: patch is correct
