## Summary

Keep open: current main still lacks the time-only `--at` parse path and this clean respin is the active implementation candidate, but it needs contributor real-behavior proof before merge and has one low-priority test determinism issue.

## What This Changes

The PR adds bare `HH:MM`/`HH:MM:SS` parsing for cron one-shot `--at`, applies `--tz` by resolving against today's date in that timezone, and adds parser tests.

## Best Possible Solution

Land the focused parser support after contributor adds redacted real cron behavior proof and the date-boundary tests freeze or capture the clock; keep help-text wording coordinated with https://github.com/openclaw/openclaw/pull/69975.

## Reproduction Assessment

Yes. Current main's parser path is source-reproducible: `parseAt` has no time-only branch, the CLI turns null into `Invalid --at`, and Node returns `NaN` for `Date.parse('09:00')`.

## Solution Assessment

Yes, with one small test hardening. `parseAt` is the narrow layer with access to `--tz`, and reusing the existing zoned datetime helper avoids teaching the lower-level absolute-time parser an ambiguous date.
