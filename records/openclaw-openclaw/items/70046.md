## What This Changes

The PR adds `HH:MM[:SS]` handling to cron `--at` with `--tz` resolution, updates cron add/edit help and cron docs, and adds parser/help tests.

## Best Possible Solution

Accept the additive `HH:MM[:SS]` cron `--at` contract if maintainers want that UX, and keep the implementation at the CLI parser/help/docs boundary while persisted cron runtime continues storing canonical ISO timestamps.
