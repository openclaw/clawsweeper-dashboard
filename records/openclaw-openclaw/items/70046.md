## Summary

Keep this PR open: current main still lacks the time-only `--at` parser path, and the related bug report was closed because this PR is the active canonical fix. The patch is narrowly scoped, but it still needs contributor real behavior proof and a small deterministic-test cleanup before merge.

## What This Changes

The PR adds `HH:MM`/`HH:MM:SS` handling to cron `--at`, applies `--tz` to those time-only values, rejects invalid time parts/timezones, and adds parser tests.

## Best Possible Solution

Land the focused parser fix after the contributor adds real behavior proof and the date-boundary tests are made deterministic; keep broader help-text wording work coordinated with https://github.com/openclaw/openclaw/pull/69975.

## Reproduction Assessment

Yes by source inspection and a focused runtime check. Current main has no time-only branch in `parseAt`, `resolveDirectSchedule` turns a null parse into `Invalid --at`, and `Date.parse('09:00')` is `NaN` in this Node runtime.

## Solution Assessment

Yes, with a small test fix. `parseAt` is the narrow layer that has access to `--tz` and can reuse the existing offset-less zoned datetime helper without changing the lower-level absolute-time parser.
