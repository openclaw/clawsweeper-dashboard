## Summary

Keep this PR open: the focused parser change is still useful and not present on current main, but the external PR still needs after-fix real behavior proof before merge and has one low-priority test determinism issue.

## What This Changes

The PR adds bare HH:MM/HH:MM:SS handling for cron --at, resolves those values against today with --tz or UTC, rejects invalid time/timezone input, and adds focused parser tests.

## Best Possible Solution

Land the focused parser support after the contributor adds redacted real cron behavior proof and the date-boundary tests use a frozen or captured clock; keep help wording coordinated with the separate help-text PR.

## Reproduction Assessment

Yes. Current main source passes --at and --tz to parseAt, parseAt has no time-only branch, the null parse becomes Invalid --at, and this Node runtime returns NaN for Date.parse('09:00').

## Solution Assessment

Yes, with one small test hardening. parseAt is the narrow layer with access to --tz, and reusing the existing offset-less zoned datetime helper avoids making the lower-level absolute-time parser guess a date.
