## Summary

Keep open: current main still lacks bare time-only `--at` support, and this PR is a focused parser update for that gap. It should not merge yet because the external PR still lacks after-fix real behavior proof and has one small test determinism issue.

## What This Changes

The branch adds bare `HH:MM`/`HH:MM:SS` parsing to cron `--at`, resolves those values using `--tz` or UTC for today, rejects invalid time-only values/timezones, and adds parser tests.

## Best Possible Solution

Land focused time-only parser support after the contributor adds real behavior proof and the tests freeze or capture the clock; keep broader help-text wording coordinated with https://github.com/openclaw/openclaw/pull/69975.

## Reproduction Assessment

Yes by source inspection plus a focused Node check. Current main has no time-only `parseAt` branch, `resolveDirectSchedule` turns null into `Invalid --at`, and `Date.parse('09:00')` is not finite in this runtime.

## Solution Assessment

Yes, with one small test fix. `parseAt` is the narrow layer that has access to `--tz` and can reuse the existing offset-less zoned datetime helper without changing the lower-level absolute-time parser.

## Review Findings

Overall correctness: patch is correct

- **[P3] Freeze the date used by time-only assertions:** `src/cli/cron-cli/parse-at.test.ts:32-47`
