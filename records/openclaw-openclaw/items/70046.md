## What This Changes

The branch adds `HH:MM`/`HH:MM:SS` parsing for cron `--at` with timezone resolution, updates cron add/edit help and cron docs, and adds parser/help tests.

## Best Possible Solution

Keep the current ISO/duration cron `--at` contract unless a maintainer explicitly sponsors `HH:MM[:SS]` as an accepted public CLI format and reopens or lands a sponsored branch.

## Reproduction Assessment

Yes from source: current main routes `--at` through `parseAt`, and `parseAt` has no time-only branch, so bare `09:00` falls through to invalid input. I did not run a live CLI repro in this read-only cleanup pass.

## Solution Assessment

Conditionally yes: resolving time-only input in the CLI parser is the right technical layer because it has access to `--tz` and preserves canonical ISO storage. It is not the best merge path without maintainer sponsorship for the new accepted CLI format.
