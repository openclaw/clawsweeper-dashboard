## What This Changes

## Best Possible Solution

Land a refreshed branch only if maintainers explicitly accept `HH:MM[:SS]` as a supported cron `--at` shorthand, keeping parsing at the CLI boundary and persisted schedules as canonical ISO timestamps.

## Reproduction Assessment

## Solution Assessment

Yes for implementation shape, but not yet for product acceptance. The CLI parser has the `--tz` context and can persist canonical ISO timestamps; the unresolved question is whether maintainers want this new public shorthand.
