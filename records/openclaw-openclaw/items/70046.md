## Summary

Keep open. Current main still rejects bare time-only `--at` values, and this PR is the clean active respin, but the external PR still lacks after-fix real behavior proof; the only code issue found is a low-priority deterministic-test concern.

## What This Changes

The PR adds bare `HH:MM`/`HH:MM:SS` parsing for cron `--at`, applies `--tz` by resolving against today's date in that timezone, rejects invalid time-only values/timezones, and adds focused parser tests.

## Best Possible Solution

Land the focused parser support after contributor adds redacted real cron behavior proof and the date-boundary test is frozen or uses a captured clock; keep help-text wording coordinated with https://github.com/openclaw/openclaw/pull/69975.

## Reproduction Assessment

Yes. Current main source shows no time-only parsing branch, the CLI turns a null parse into `Invalid --at`, and `Date.parse('09:00')` is `NaN` in the local Node runtime; this is source-reproducible rather than a full CLI run in this read-only pass.

## Solution Assessment

Yes, with one small test hardening. `parseAt` is the narrow layer with access to `--tz`, and reusing the existing offset-less zoned datetime helper avoids changing the lower-level absolute-time parser.
