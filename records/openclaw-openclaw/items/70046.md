## What This Changes

The PR adds `HH:MM`/`HH:MM:SS` handling to cron CLI `parseAt`, applies `--tz` to those time-only values, updates help/docs, and adds parser/CLI tests.

## Best Possible Solution

Keep the existing ISO/duration cron `--at` contract unless a maintainer explicitly sponsors bare time-only values; if sponsored, this PR's parser-layer shape with tests/docs is the right landing direction.

## Reproduction Assessment

## Solution Assessment

Yes technically if maintainers choose the behavior: the CLI `parseAt` layer already owns `--tz` and duration context and can reuse the zoned offset-less helper. No as an unsponsored merge path, because the remaining blocker is product acceptance for new public syntax.
