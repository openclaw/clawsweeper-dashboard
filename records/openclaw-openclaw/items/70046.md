## Summary

Keep open: current main still lacks the parser branch for bare time-only `--at` values, and this focused PR is the active fix path, but the external PR still needs real behavior proof before merge.

## What This Changes

The PR adds time-only `cron add --at` parsing in the cron CLI, applies `--tz` when resolving those values, validates invalid time/timezone inputs, and adds focused parser tests.

## Best Possible Solution

Land the focused parser support after the contributor adds real behavior proof and the time-only tests freeze or capture the clock; keep broader help-text wording coordinated with https://github.com/openclaw/openclaw/pull/69975.

## Reproduction Assessment

Yes. Current main has no time-only branch in `parseAt`, `resolveDirectSchedule` turns a null parse into `Invalid --at`, and `Date.parse('09:00')` is not finite in this Node runtime.

## Solution Assessment

Yes, with one small test fix. `parseAt` is the narrow layer that has access to `--tz` and can reuse the existing offset-less zoned datetime helper without changing the lower-level absolute-time parser.
