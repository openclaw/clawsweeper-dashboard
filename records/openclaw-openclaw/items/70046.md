## Summary

Keep open: current main still lacks bare time-only `--at` support, and the PR is a focused parser-layer implementation, but this external PR still needs after-fix real behavior proof before merge and has one low-priority test determinism issue.

## What This Changes

The branch adds time-only cron `--at` parsing, applies `--tz` when present, rejects invalid time parts/timezones, and adds parser tests.

## Best Possible Solution

Land the parser-layer support after the contributor adds redacted real CLI behavior proof and the date-boundary assertions use a frozen or captured clock; keep help text coordinated with https://github.com/openclaw/openclaw/pull/69975.

## Reproduction Assessment

Yes, for current behavior. Current main routes `--at` through `parseAt`, which lacks a time-only branch, falls back to `Date.parse('09:00')`, and the Node check returns `NaN`; I did not run a full CLI command in this read-only pass.

## Solution Assessment

Yes, with one small test hardening. `parseAt` is the narrow layer with access to `--tz`, and reusing the existing offset-less zoned datetime helper avoids broadening the lower-level absolute-time parser.
