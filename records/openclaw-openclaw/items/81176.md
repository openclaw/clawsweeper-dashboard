## Summary

Keep open. The PR is a plausible feature direction, but it accepts new public compaction share keys before memory preflight/flush planning and generated config docs/help/baseline consistently understand them, and the contributor proof is source inspection rather than a real OpenClaw run.

## What This Changes

The PR adds optional `reserveTokensShare`, `keepRecentTokensShare`, and `reserveTokensFloorShare` config keys, resolves them in Pi compaction settings, and adds focused unit tests.

## Best Possible Solution

Land one backward-compatible relative-budget config API that covers Pi settings, memory preflight/flush planning, schema help/labels/baseline, focused tests, and public docs while preserving existing absolute-token semantics.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature PR. The review findings are source-reproducible: accepting `reserveTokensFloorShare` while preflight still reads only absolute floor values leaves an inconsistent runtime path.

## Solution Assessment

No. Additive share fields may be a maintainable API, but this patch is not the best merge path until every threshold consumer and config metadata/docs surface is updated consistently.
