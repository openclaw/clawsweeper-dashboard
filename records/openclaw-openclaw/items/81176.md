## What This Changes

The PR adds optional `reserveTokensShare`, `keepRecentTokensShare`, and `reserveTokensFloorShare` fields, resolves them against `contextTokenBudget` in Pi settings, and adds focused unit coverage.

## Best Possible Solution

Land a backward-compatible relative-budget config API only after every compaction and preflight consumer resolves the same shared budget and the public config docs/help baseline describe the fields.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature PR. Source inspection does show the proposed end-to-end behavior is incomplete because memory preflight paths still read only absolute `reserveTokensFloor`.

## Solution Assessment

No. Additive share fields are a plausible solution, but the implementation needs to apply them consistently in all threshold consumers and publish docs/help/baseline updates before merge.
