## What This Changes

The PR adds pre-bootstrap Telegram forum-topic target parsing for isolated cron announce delivery, slash-form Telegram target support, cfg-backed explicit target resolution, delivery-error propagation, docs, and regression tests.

## Best Possible Solution

Land or replace this with a focused current-main fix that keeps Telegram-owned parsing consistent, extracts every documented Telegram forum-topic target form before isolated cron depends on loaded-plugin parsing, preserves the existing `:topic:` behavior, surfaces delivery-specific failures in persisted cron state, documents only covered formats, and adds a required changelog entry.

Yes. ## Solution Assessment

No. The PR direction is close, but duplicating only part of Telegram’s parser in a pre-bootstrap helper leaves a supported target form outside the claimed fix; the safer current-main solution is a complete shared helper or Telegram-owned parser path covering `:topic:`, bare colon shorthand, and slash forms together.
