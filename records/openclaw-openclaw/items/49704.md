## Summary

Keep open: this is a focused Telegram cron bug-fix PR with useful remaining work, but it is not merge-ready because the isolated pre-parser still misses a documented shorthand topic format and the external PR has no real Telegram after-fix proof.

## What This Changes

The PR pre-parses Telegram forum-topic cron targets, adds slash target parsing, threads config into target parsing, propagates delivery errors into cron state, updates cron docs, and adds regression tests.

## Best Possible Solution

Refresh the PR on current main, cover every documented Telegram topic target syntax through a shared or tightly bounded parser, and provide redacted live Telegram forum-topic cron proof before merge.

## Reproduction Assessment

Yes, at source level. In the PR head, an isolated cron announce target like `-1001234567890:123` is documented and parsed by the Telegram parser but is not extracted by the new no-plugin pre-parser before fallback resolution.

## Solution Assessment

No, not as written. The direction is reasonable, but the patch must cover the shorthand topic path and satisfy the real Telegram proof gate before it is the narrow maintainable fix.
