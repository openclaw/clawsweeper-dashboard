## Summary

Keep open: this is a plausible bug fix for Telegram forum-topic cron delivery, but the branch is not merge-ready because one documented target format still loses the thread id in isolated cron and the external PR only has test/CI proof, not real Telegram behavior proof.

## What This Changes

The PR pre-parses Telegram forum-topic cron targets, adds slash target parsing, threads config into target parsing, propagates delivery errors into cron state, updates cron docs, and adds regression tests.

## Best Possible Solution

Cover every documented Telegram topic target syntax in the isolated cron path through a shared or tightly bounded parser, then provide live Telegram forum-topic cron proof before merge.

## Reproduction Assessment

Yes. Source inspection of the PR head shows the docs and Telegram parser support `chatId:topicId`, while the new isolated-cron pre-parser only extracts `chatId:topic:topicId` and `chatId/topicId` before the no-plugin fallback.

## Solution Assessment

No, not as written. The approach is directionally reasonable, but it must cover the shorthand topic path and satisfy the real Telegram proof gate before it is the narrow maintainable fix.
