## Best Possible Solution

Keep the fix inside the Telegram sendChatAction classifier, but cover structured Telegram HTTP errors and recoverable network-error codes before any message fallback, add focused regression coverage, and require real Telegram or redacted runtime proof before merge.

- **[P2] Short-circuit recoverable network errors before fallback:** `extensions/telegram/src/sendchataction-401-backoff.ts:77-82`
  - body: The new guard only returns `false` when the error carries Telegram's HTTP `error_code`. Please route known transient/network classifiers before the bare-message fallback, or remove the numeric fallback for code-less errors.
