## Summary

Keep open: current main still has the empty-scope harness wipe, but this patch restores every prior harness, which can keep plugin-owned executors alive after an explicit empty plugin scope. The PR also still needs after-fix proof from a patched real gateway before merge.

## What This Changes

The PR snapshots and restores registered agent harnesses in the plugin-loader and gateway empty-plugin-scope branches and adds a loader regression test for harness preservation.

## Best Possible Solution

Preserve the always-on provider harnesses needed for Discord/cron recovery without retaining harnesses owned by plugins that are no longer in the active scope, with loader and gateway coverage plus patched real-gateway proof.

## Solution Assessment

No, not as written. Restoring every previous harness fixes the reported missing bundled harness but also preserves plugin-owned harnesses after an empty scope; the safer path is to preserve only the intended always-on provider harnesses and keep plugin-owned runtime state scoped.
