## Summary

Keep open: this targets a real current-main and shipped gateway/plugin lifecycle bug, but the restore-all patch can keep inactive plugin-owned harnesses globally selectable and still lacks after-fix proof from a patched gateway.

## What This Changes

The branch snapshots registered agent harnesses before empty plugin-scope reloads in the plugin loader and gateway startup path, restores them afterward, and adds a loader regression test.

## Best Possible Solution

Preserve only the maintainer-approved always-on/native provider harness contract across empty scopes, clear plugin-owned harnesses when their plugins are inactive, and prove both loader/gateway behavior plus a patched gateway run.

## Reproduction Assessment

Yes at source level: current main and the latest release still have empty-scope branches that clear the global harness registry without restoration. I did not run a live gateway timing repro in this read-only review.

## Solution Assessment

No. Snapshot-and-restore is a plausible mitigation, but restoring every previous harness is too broad because it crosses the active plugin ownership boundary; the safer fix is ownership-aware preservation with explicit tests.
