## Best Possible Solution

Land the shell-first LaunchAgent serialization/readback path after maintainers accept the macOS launchd compatibility change, keeping env values in the owner-only env file and leaving gateway command construction unchanged.

## Solution Assessment

Yes. Starting launchd from `/bin/sh` while preserving owner-only env-file loading is narrower than changing gateway command construction, changing status parsing only, or embedding secrets in the plist.
