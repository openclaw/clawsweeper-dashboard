## Summary

No blocking findings found. The PR is still necessary because current main still writes the generated env wrapper as the executable launchd spawns, and the patch moves the fix to the LaunchAgent writer/readback boundary while preserving legacy wrapper-first plists.

## What This Changes

The PR changes macOS LaunchAgent env-wrapper serialization to launch `/bin/sh` before the generated wrapper and adds regression coverage for generation/readback behavior.

## Best Possible Solution

Land the focused shell-first LaunchAgent serialization with legacy wrapper-first readback/rewrite preserved, while treating live macOS launchd proof as the remaining optional platform-confidence gap.

## Reproduction Assessment

Yes. Current main serializes managed service-env LaunchAgents with the generated wrapper as `ProgramArguments[0]`, and the linked issue reports macOS launchd rejecting that executable on an external home volume.

## Solution Assessment

Yes. The shell-first change is narrower than changing the gateway command or embedding secrets in the plist, and current CLI evidence shows bare `gateway --port` remains a supported runtime path.
