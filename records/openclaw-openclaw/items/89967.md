## Summary

Keep open: this PR is still necessary because current main and v2026.6.9 still serialize service-env LaunchAgents with the generated wrapper as the launchd executable, while this branch implements the shell-first fix for the linked macOS gateway startup failure. The patch looks correct, but merging it is compatibility-sensitive because existing macOS service-env plists will be rewritten to spawn `/bin/sh` before the generated wrapper.

## What This Changes

The PR changes macOS LaunchAgent service-env wrapper generation/readback to serialize shell-first `ProgramArguments` and adds regression coverage for new and legacy layouts.

## Best Possible Solution

Land the shell-first LaunchAgent serialization/readback path after maintainer acceptance of the macOS launchd compatibility change, keeping env values in the owner-only env file and the existing gateway command construction.

## Reproduction Assessment

Yes. Current main and v2026.6.9 still emit wrapper-first `ProgramArguments` when service env exists, and the PR body includes after-fix macOS launchd output; I did not run the exact external-home setup in this read-only review.

## Solution Assessment

Yes. Starting launchd from `/bin/sh` while preserving owner-only env-file loading is narrower than changing gateway command construction, changing status parsing only, or embedding secrets in the plist.
