## Summary

## What This Changes

The PR changes macOS LaunchAgent service-env wrapper generation/readback to use a shell-first ProgramArguments layout and adds regression coverage for new and legacy layouts.

## Best Possible Solution

Land the shell-first LaunchAgent serialization/readback path after maintainers accept the macOS launchd compatibility change, while keeping env values in the owner-only env file and preserving the existing gateway command construction.

## Reproduction Assessment

## Solution Assessment

Yes. Starting launchd from /bin/sh while preserving owner-only env-file loading is narrower than changing gateway command construction, changing status parsing only, or embedding secrets in the plist.
