## What This Changes

The PR changes macOS LaunchAgent service-env wrapper generation/readback to spawn `/bin/sh` before the generated env wrapper and adds tests for new and legacy plist layouts.

## Best Possible Solution

Land the shell-first LaunchAgent serialization/readback path after maintainers accept the macOS launchd compatibility change, while keeping env values in the owner-only env file and preserving the existing gateway command construction.

## Reproduction Assessment

Yes, for the source-level failure surface: current main and v2026.6.8 serialize the generated env wrapper as `ProgramArguments[0]` when service env entries exist. I did not run the exact external-home macOS setup in this read-only review.

## Solution Assessment

Yes. Starting launchd from `/bin/sh` while preserving owner-only env-file loading is narrower than changing gateway command construction, changing status parsing only, or embedding secrets in the plist.
