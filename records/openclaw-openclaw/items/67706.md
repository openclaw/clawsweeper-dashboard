## Summary

Keep open. The PR addresses a real missing event path on current main, but the branch still needs contributor real behavior proof, a rebase, and fixes for two compaction-completion contract regressions before merge.

## What This Changes

The PR imports `emitAgentEvent` and emits compaction start/end events from embedded runner overflow and timeout recovery helpers.

## Best Possible Solution

Rebase the branch, emit recovery lifecycle events with the same result-based completion contract as the subscribe path, update strict consumers/tests where needed, and add redacted runtime proof showing a subscriber or UI receives the new events.

## Reproduction Assessment

Yes. Current main has a source-level reproduction path: timeout and overflow recovery call `contextEngine.compact` through helpers that run hooks but do not emit standard compaction lifecycle events.

## Solution Assessment

No. Emitting the events is the right direction, but this patch should align `completed` with the existing result/non-abort contract and avoid making failed recovery end events look successful to current memory consumers.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Align completed with the canonical result contract:** `src/agents/pi-embedded-runner/run.ts:596`
  - body: The new recovery end event reports completion as `ok && compacted`, while the subscribe handler reports completion only when compaction produced a result and was not aborted. Because `CompactResult.result` is optional, a context engine can return `ok: true` and `compacted: true` without a result and make subscribers see a completed compaction contrary to the existing contract.
  - confidence: 0.88
- **[P2] Gate memory counting on completed compactions:** `src/agents/pi-embedded-runner/run.ts:605`
  - body: This emits `phase: "end"` for failed timeout/overflow recovery attempts, but `agent-runner-memory` still treats every compaction end event as completed before incrementing stored compaction count. Either avoid failed end events in this path or update the consumer/tests to require `completed === true`.
  - confidence: 0.87
