## Summary

Keep open: the PR targets a real current-main gap, but it is not merge-ready because the new end-event semantics diverge from the canonical compaction contract, can make memory flush count failed recovery compactions, and lacks external-PR real behavior proof.

## What This Changes

The PR adds compaction lifecycle agent-event emission from embedded runner overflow and timeout recovery helpers.

## Best Possible Solution

Rebase the patch onto current main, emit recovery start/end lifecycle events with the same result-based completion contract as the subscribe path, harden memory counting to require completed compactions, add focused regression tests, and collect redacted real behavior proof from a real subscriber or UI path.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: current main calls `contextEngine.compact` during timeout and overflow recovery, but the recovery helpers only fire hooks and do not emit standard compaction lifecycle events to subscribers.

## Solution Assessment

No. Emitting recovery lifecycle events is the right direction, but this patch should align `completed` with the canonical result/abort contract and prevent failed recovery end events from being counted as successful memory compactions.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Align completed with the canonical result contract:** `src/agents/pi-embedded-runner/run.ts:593`
  - body: The new recovery end event reports `completed` as `ok && compacted`, while the subscribe handler reports completion only when compaction produced a result and was not aborted. Since `CompactResult.result` is optional, a context engine can return `ok: true` and `compacted: true` without a result and make subscribers see a completed compaction contrary to the existing contract.
  - confidence: 0.88
- **[P2] Require completion before counting memory compactions:** `src/agents/pi-embedded-runner/run.ts:598-604`
  - body: This emits a compaction `phase: "end"` event for failed timeout/overflow recovery attempts, but `agent-runner-memory` currently treats any end event as completed before incrementing stored compaction count. Please either avoid failed end events here or update the consumer/tests to require `completed === true`.
  - confidence: 0.87
