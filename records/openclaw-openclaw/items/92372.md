## What This Changes

The issue reports that secondary agents with explicit workspaces emit `read` calls without the `memory/` prefix for daily memory files, causing workspace-root ENOENT errors.

## Best Possible Solution

Add focused current-main proof for a non-default agent with an explicit workspace on heartbeat and inbound-message startup, then fix the startup-context or workspace-routing path if it emits a bare filename; avoid adding date-basename guessing to the generic read tool.

## Reproduction Assessment

## Solution Assessment

Unclear until the current-main multi-agent route is exercised. If confirmed, the best fix belongs in startup context or workspace routing rather than changing generic read semantics to infer `memory/` from a date-shaped basename.
