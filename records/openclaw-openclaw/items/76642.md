## Summary

Keep open. Current main still has a source-visible stale-turn path: sub-agent completions can be steered into an active requester or sent as a direct fallback, while Telegram’s supersession fence only covers authorized abort commands, not ordinary newer user messages.

## What This Changes

The issue reports stale and duplicate Telegram direct-chat replies when user messages, sub-agent completion announcements, tool work, and heartbeat events overlap.

## Best Possible Solution

Add turn-bound delivery semantics for background completion and heartbeat events so stale work cannot be emitted as a reply to an unrelated newer Telegram user turn, with regression coverage for the reported interleaving.

## Reproduction Assessment

Yes for source-level reproduction: current main can steer or direct-fallback a sub-agent completion into an active Telegram requester without a per-inbound turn guard. I did not establish a live end-to-end Telegram heartbeat/tool reproduction.

## Solution Assessment

Unclear. A turn-binding or generation guard is the right direction, but maintainers need to choose the intended semantics for background completions that finish after newer user input.
