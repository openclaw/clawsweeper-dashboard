## Summary

Keep open: the verifier half is still not implemented on main because current `backup verify` only handles root-relative hardlink targets, while the reported cwd-relative legacy shape remains in an open PR. The create path has shipped mitigations for hardlink emission, EOF retry, and temp cleanup, but the reporter's exit-13 path has not been live-proved fixed on current main or the latest release.

## What This Changes

The issue asks for `openclaw backup create --verify` to either produce a verified usable archive or fail cleanly, and for `backup verify` to continue accepting older valid archives with internal hardlinks.

## Best Possible Solution

Keep this issue open until the legacy hardlink verifier fix lands or is replaced, then prove `backup create --verify` on a faithful Linux state fixture or reporter-like live setup and decide whether stale `.tmp` cleanup is part of this fix or the narrower cleanup PR.
