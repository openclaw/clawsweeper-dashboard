## Summary

Close as ClawHub: Manifest is a reasonable optional OpenAI-compatible router/provider, but current main already has provider-plugin, auth-choice, catalog, install, and ClawHub distribution surfaces for this to live outside the core repository. The PR is also not merge-ready as a bundled plugin because it adds a manifest-only extension directory without the runtime/package entry that bundled build inventory expects.

## What This Changes

Adds a bundled Manifest provider manifest plus Manifest provider documentation and provider directory/table entries for `MANIFEST_API_KEY` and `manifest/auto`.

## Best Possible Solution

Publish Manifest as an external provider plugin through ClawHub/npm using the existing provider-plugin SDK. If maintainers later decide Manifest should be officially bundled, that should come through a separate product/security review with a complete provider plugin package and runtime entry.

## Reproduction Assessment

Not applicable as a feature PR rather than a bug report. The high-confidence verification path is static: current main has no Manifest refs, and the current provider-plugin/ClawHub docs and SDK cover the requested integration outside core.

## Solution Assessment

No. Adding another optional router directly under bundled core is not the narrowest maintainable path when the existing provider-plugin and ClawHub surfaces can support it, and the proposed bundled shape is incomplete without a runtime/package entry.
