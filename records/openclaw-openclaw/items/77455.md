## Summary

Keep open: current main still has the reported response-vs-disk mismatch, and #77462 is the open implementation candidate that references this issue with closing syntax. The issue should remain open until that PR, or an equivalent fix, lands.

## What This Changes

The issue asks for config.set, config.patch, and config.apply to return the canonical persisted config written to disk, not the pre-write in-memory config.

## Best Possible Solution

Review and land #77462, or an equivalent patch, so the shared config write abstraction returns the redacted canonical persisted config with regression coverage for transformed writes.

## Reproduction Assessment

Yes, from source inspection. Current main shows all three write RPC handlers responding with pre-write config values while the disk write path can transform the persisted payload before commit.

## Solution Assessment

Yes. The narrow maintainable fix is to propagate persistedConfig from the write layer through replaceConfigFile and commitGatewayConfigWrite, then have the handlers respond with the redacted persisted config.
