## Summary

Keep open: current main still has the write-response mismatch, and #77462 is already open as the paired fix, so this issue should remain open until that PR lands or is closed.

## What This Changes

Return the canonical persisted disk config from `config.set`, `config.patch`, and `config.apply` responses after writes, instead of the pre-write in-memory config.

## Best Possible Solution

Review and land #77462, or an equivalent narrow fix, so the shared config write path returns a redacted post-write persisted config for all three write RPCs with regression coverage.

## Reproduction Assessment

Yes. Current-main source shows `writeConfigFile` can persist a transformed `persistedConfig`, while `config.set`, `config.patch`, and `config.apply` respond with the pre-write parsed or validated config after the write completes.

## Solution Assessment

Yes. The maintainable fix is to propagate `persistedConfig` through `replaceConfigFile` and `commitGatewayConfigWrite`, then have the write RPC responses redact that canonical post-write config; #77462 appears to target that boundary.
