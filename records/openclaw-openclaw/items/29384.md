## Summary

Keep open: current main already supports the GPU passthrough part through a typed `sandbox.docker.gpus` config, but it does not support `--ipc=host` or generic Docker run parameters, and the remaining request expands sandbox isolation boundaries enough to need maintainer/security product judgment.

## What This Changes

Allow Docker-backed sandbox configuration to pass additional Docker run parameters such as GPU passthrough and host IPC.

## Best Possible Solution

Keep the typed GPU support, and if maintainers accept host IPC, add a narrow documented config key with schema, hashing, recreate behavior, security validation, docs, changelog, and targeted tests instead of a generic raw Docker-args escape hatch.

## Reproduction Assessment

Not applicable as a feature request. Source inspection verifies the GPU path exists on current main and that no Docker IPC or generic custom-flag surface currently exists.

## Solution Assessment

No for the generic proposed direction: arbitrary Docker parameters are not the safest maintainer surface for sandbox isolation. A typed, validated, documented config key would be the narrower solution if host IPC is accepted.
