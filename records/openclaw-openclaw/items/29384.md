## Summary

The GPU portion was deliberately merged in https://github.com/openclaw/openclaw/pull/73491 and shipped, carrying forward the earlier contributor proposal. This issue remains necessary because host IPC is still unsupported, has no canonical follow-up, and changes a more sensitive namespace boundary requiring maintainer product and security judgment.

## What This Changes

Add opt-in Docker sandbox configuration for GPU passthrough and host IPC access.

## Best Possible Solution

Keep the shipped typed GPU option and, only if maintainers approve host IPC, add a narrowly named break-glass IPC setting with schema enforcement, security diagnostics, explicit documentation, and Docker-backed validation rather than arbitrary Docker arguments.

## Reproduction Assessment

Yes at the source level: release and current-main inspection proves GPU support exists, while the sandbox schema and Docker argument builder contain no IPC mode or `--ipc` path. This is a missing capability, not a broken existing contract.

## Solution Assessment

No, arbitrary custom Docker flags are not the best solution because they create an unbounded privilege surface; a dedicated, explicitly dangerous IPC field is the narrowest maintainable option if the security tradeoff is accepted.
