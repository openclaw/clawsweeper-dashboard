## Summary

Keep open. Current main implements typed Docker GPU passthrough, but the host IPC/raw Docker parameter half remains absent and requires maintainer/secops policy judgment because it changes sandbox isolation.

## What This Changes

Request support for custom sandbox Docker run parameters, specifically GPU passthrough and host IPC options.

## Best Possible Solution

Keep the shipped typed GPU support, then decide whether to add a guarded documented `sandbox.docker.ipc` setting or explicitly leave host IPC/raw Docker passthrough out of scope.

## Reproduction Assessment

Not applicable as a bug reproduction. Current-main source inspection is high confidence: `gpus` is implemented and covered, while strict config and create-arg code expose no IPC/raw Docker argument path.

## Solution Assessment

Unclear until maintainers decide the security policy. If approved, a narrow typed IPC field with explicit warnings is more maintainable than arbitrary raw Docker argument passthrough.
