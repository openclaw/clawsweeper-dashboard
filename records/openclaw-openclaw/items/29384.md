## Summary

Keep open: current main and shipped docs cover GPU passthrough, but host IPC and a generic Docker-parameter surface are still absent and would expand a security-sensitive sandbox boundary.

## What This Changes

Allow sandbox Docker containers to receive custom Docker run parameters such as `--gpus all` and `--ipc=host`.

## Best Possible Solution

Keep the shipped typed GPU support; if host IPC is accepted, add a narrow validated config key with warnings, docs, config hashing/recreate coverage, and targeted tests instead of arbitrary raw Docker args.

## Reproduction Assessment

Not applicable; this is a feature/config request, not a report of broken existing behavior. Source inspection gives high confidence that GPU support exists while host IPC and generic Docker flag passthrough do not.

## Solution Assessment

No for the generic raw-parameter proposal: arbitrary Docker args are not the safest sandbox contract. A typed, validated host-IPC option is the narrower maintainable path if maintainers choose to support it.
