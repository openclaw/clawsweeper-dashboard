## Summary

Keep this open. Current main implements the GPU passthrough half, but the remaining host IPC or raw Docker argument request is still absent and sits on the sandbox isolation boundary, so it needs maintainer/secops policy review before code should be written.

## What This Changes

Request to let Docker-backed sandbox containers pass custom Docker run parameters, specifically GPU passthrough and host IPC.

## Best Possible Solution

Have sandbox/secops maintainers decide the policy first: either add a narrow, typed, documented `sandbox.docker.ipc` setting with validation and security tests, or document that host IPC/raw Docker arg passthrough is unsupported. Broad raw Docker args should only be added after explicit maintainer approval of the audit and support tradeoffs.
