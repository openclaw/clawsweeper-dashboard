## Summary

Keep this open: current main and v2026.6.6 already support the GPU portion through a typed `gpus` setting, but the host IPC portion is still missing and is security-sensitive product work rather than a narrow bug fix.

## What This Changes

The issue requests Docker sandbox configuration for GPU passthrough and host IPC, originally framed as support for Docker flags such as `--gpus all` and `--ipc=host`.

## Best Possible Solution

Keep the shipped typed GPU option and, only if maintainers approve host IPC, add a narrowly named break-glass IPC setting with schema enforcement, security diagnostics, explicit docs, and Docker-backed validation instead of arbitrary Docker arguments.

## Reproduction Assessment

Yes at the source level: current main and v2026.6.6 show typed GPU support, while the sandbox schema and Docker create-args path have no IPC field or `--ipc` emission. This is a missing capability, not a broken existing contract.

## Solution Assessment

No, the requested generic custom Docker-flag passthrough is not the best solution because it would create an unbounded privilege surface. If maintainers accept the tradeoff, the maintainable path is a dedicated, explicitly dangerous IPC option with validation and security review.
