## Summary

Keep this open: current main and the latest release cover the GPU half with typed `sandbox.docker.gpus`, but host IPC and generic Docker-argument passthrough remain absent and need maintainer security/product direction.

## What This Changes

The issue requests Docker sandbox configuration for GPU passthrough and host IPC, such as `--gpus all` and `--ipc=host`.

## Best Possible Solution

Keep the shipped GPU support; if maintainers approve host IPC, add a narrow explicitly dangerous IPC setting with schema validation, audit visibility, docs, and focused Docker-args tests rather than arbitrary flag passthrough.

## Reproduction Assessment

Yes. Source inspection shows current main and `v2026.6.10` expose `sandbox.docker.gpus`, but neither has an IPC field or `--ipc` emission for Docker sandbox creation.

## Solution Assessment

No. Generic Docker flag passthrough is not the best shape for this codebase because sandbox config is typed and security-audited; a narrow dangerous IPC option would be safer if maintainers accept the tradeoff.
