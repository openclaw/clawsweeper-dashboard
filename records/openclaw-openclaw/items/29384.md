## Summary

Keep open: current main and v2026.5.22 cover the GPU use case through typed `sandbox.docker.gpus`, but host IPC and generic Docker flag passthrough remain unsupported and security-sensitive.

## What This Changes

The issue requests sandbox Docker configuration that can pass custom Docker parameters such as `--gpus all` and `--ipc=host`.

## Best Possible Solution

Keep the shipped typed GPU option; if host IPC is approved, add a narrow explicit Docker-only config field with validation, audit/docs coverage, config-hash behavior, and focused tests instead of arbitrary Docker args.

## Reproduction Assessment

Not applicable; this is a feature/config request rather than broken existing behavior. Source inspection, release inspection, and Docker CLI help give high confidence that GPU support exists while host IPC and raw Docker argument config remain absent.

## Solution Assessment

No for generic Docker-parameter passthrough because it would bypass the typed sandbox contract. A reviewed explicit host-IPC option is the narrower maintainable path if maintainers choose to support it.
