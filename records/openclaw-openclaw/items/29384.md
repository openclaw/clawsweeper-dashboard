## Summary

Keep open: current main and v2026.6.5 already support the GPU half through typed `sandbox.docker.gpus`, but the `--ipc=host` or general custom Docker-flag request remains missing and security-sensitive.

## What This Changes

Request to add configuration for Docker sandbox runtime parameters such as `--gpus all` and `--ipc=host`.

## Best Possible Solution

Keep the issue open for maintainer/secops direction; if accepted, implement a narrow typed option such as `sandbox.docker.ipc` with validation, audit warnings, docs, and tests rather than arbitrary Docker-args passthrough.

## Reproduction Assessment

Yes for the missing capability, from source inspection: current main accepts and emits `sandbox.docker.gpus`, but strict config/schema and Docker arg construction have no `ipc` or raw custom-args path. This is not a bug reproduction; it is a source-verifiable feature gap.

## Solution Assessment

No for the proposed arbitrary custom flags. The safer maintainable path is a typed, validated, auditable Docker sandbox setting only if maintainers accept the security tradeoff.
