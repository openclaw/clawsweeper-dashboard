## Summary

Keep open: current main and v2026.5.20 cover the GPU half through typed `sandbox.docker.gpus`, but host IPC and raw Docker-flag passthrough remain unsupported and would change the sandbox isolation boundary.

## What This Changes

The issue requests configurable sandbox Docker parameters such as `--gpus all` and `--ipc=host` for GPU-accelerated sandbox workloads.

## Best Possible Solution

Keep the shipped typed GPU option; if host IPC is approved, add a narrow explicit validated sandbox option with docs, audit coverage, config-hash behavior, focused tests, and security warnings instead of arbitrary Docker args.

## Reproduction Assessment

Not applicable; this is a feature/config request rather than broken existing behavior. Source, docs, release-tag inspection, and related PR history give high confidence that GPU support exists while host IPC and raw Docker argument config remain absent.

## Solution Assessment

No for generic Docker-parameter passthrough because it would sidestep the typed sandbox contract. A reviewed explicit host-IPC option is the narrower maintainable path if maintainers choose to support it.
