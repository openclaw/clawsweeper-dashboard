## Summary

Keep open: GPU passthrough is present on current main and shipped, but host IPC and a generic Docker-parameter surface are still absent; the remaining request is a security-sensitive sandbox config expansion that needs maintainer product judgment.

## What This Changes

The issue requests Docker sandbox configuration for runtime parameters such as `--gpus all`, `--ipc=host`, or custom Docker run flags.

## Best Possible Solution

Keep the shipped typed GPU support; if maintainers accept host IPC, add a narrow documented config key with validation, warnings/audit behavior, config hashing/recreate coverage, docs, changelog, and targeted tests instead of arbitrary raw Docker args.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that the GPU portion exists while host IPC and generic Docker flag passthrough do not.

## Solution Assessment

No for the generic proposed direction: arbitrary Docker parameters are not the safest sandbox contract. A typed, validated option is the narrower maintainable path if maintainers choose to support host IPC.
