## Summary

Keep this open: current main and the latest release already cover the GPU passthrough half, but host IPC support is still missing and would be a deliberate sandbox-boundary change rather than a routine bug fix.

## What This Changes

The issue requests Docker sandbox configuration for GPU passthrough and host IPC workflows such as `--gpus all` and `--ipc=host`.

## Best Possible Solution

Keep the shipped typed GPU option; if maintainers approve host IPC, add a narrowly named dangerous IPC setting with schema validation, audit/docs coverage, and Docker argument tests instead of arbitrary Docker args.

## Reproduction Assessment

Yes, for the remaining gap from source inspection: current main accepts and emits Docker GPU passthrough, but the inspected schema/types/create-args path has no IPC setting or `--ipc` emission. This is a missing feature capability, not a broken existing contract.

## Solution Assessment

No, the issue's generic Docker-argument passthrough is not the best way to solve the remaining problem because it would create an unbounded privilege surface. If accepted, host IPC should be a dedicated explicit break-glass config path with tests and security documentation.
