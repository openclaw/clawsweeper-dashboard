## Summary

Keep this open: current main and v2026.6.8 cover the GPU half with typed `sandbox.docker.gpus`, but there is still no host IPC or generic Docker-argument sandbox surface, and the remaining request needs maintainer security/product approval.

## What This Changes

Allow OpenClaw Docker sandbox configuration to pass runtime flags such as `--gpus all` and `--ipc=host`.

## Best Possible Solution

Keep the shipped typed GPU support; if maintainers approve host IPC, add a narrow dangerous IPC setting with schema validation, audit visibility, docs, and Docker create-args tests rather than arbitrary Docker flag passthrough.

## Reproduction Assessment

Yes, at the source level: current main and v2026.6.8 show typed GPU support, while strict sandbox Docker config and create-args handling have no IPC field or `--ipc` emission. This is a missing feature capability rather than a broken existing contract.

## Solution Assessment

No, generic custom Docker flag passthrough is not the best way to solve this because it would bypass OpenClaw's typed schema and sandbox validation. A dedicated, explicitly dangerous IPC option is the maintainable path if maintainers accept the tradeoff.
