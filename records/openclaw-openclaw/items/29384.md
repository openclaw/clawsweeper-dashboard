## Summary

Keep open: current main and v2026.6.5 already cover the GPU half through typed `sandbox.docker.gpus`, but host IPC and generic Docker flag passthrough remain unsupported and security-sensitive.

## What This Changes

Request to let sandbox Docker configuration pass custom Docker create/run flags such as `--gpus all` and `--ipc=host`.

## Best Possible Solution

Keep the typed GPU option; if maintainers approve host IPC, add a narrow validated Docker-only config field with warning/audit/docs/tests rather than arbitrary Docker args.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source, docs, Docker CLI help, and release checks show GPU passthrough exists while host IPC and raw Docker args remain absent.

## Solution Assessment

No for generic Docker-parameter passthrough because it would bypass the typed sandbox contract. A reviewed explicit host-IPC option is the narrower maintainable path if maintainers choose to support it.
