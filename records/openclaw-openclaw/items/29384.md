## Summary

Keep open: current main covers GPU passthrough, but host IPC and generic sandbox Docker-flag passthrough are still absent and would change the sandbox isolation boundary.

## What This Changes

The issue requests sandbox Docker configuration support for custom Docker run parameters such as `--gpus all` and `--ipc=host` for GPU/AI workloads.

## Best Possible Solution

Keep the typed GPU support; if host IPC is accepted, add a narrow validated dangerous sandbox option with warnings, docs, config-hash/recreate coverage, and tests instead of arbitrary raw Docker args.

## Reproduction Assessment

Not applicable; this is a feature/config request rather than broken existing behavior. Source inspection gives high confidence that GPU support exists while host IPC and raw Docker args are absent.

## Solution Assessment

No for generic raw Docker-parameter passthrough because it would bypass the typed sandbox contract. A reviewed, explicit host-IPC option is the narrower maintainable path if maintainers choose to support it.
