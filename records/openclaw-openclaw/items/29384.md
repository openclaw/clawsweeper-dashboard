## Summary

Keep open: GPU passthrough is already implemented and shipped, but host IPC and generic sandbox Docker-parameter passthrough remain absent and would weaken the sandbox isolation boundary, so this needs maintainer/secops product review rather than cleanup closure.

## What This Changes

Request support for sandbox Docker parameters such as `--gpus all` and `--ipc=host` so GPU/IPC-heavy workloads can run inside OpenClaw sandboxes.

## Best Possible Solution

Keep the shipped typed GPU support; if host IPC is approved, add a narrow validated dangerous sandbox option with warnings, docs, config-hash/recreate coverage, and tests instead of arbitrary raw Docker args.

## Reproduction Assessment

Not applicable; this is a feature/config request rather than broken existing behavior. Source inspection against current main and `v2026.5.12` gives high confidence that GPU support exists while host IPC and raw Docker args are absent.

## Solution Assessment

No for generic raw Docker-parameter passthrough because it would bypass the typed sandbox contract. A reviewed, explicit host-IPC dangerous option is the narrower maintainable path if maintainers choose to support it.
