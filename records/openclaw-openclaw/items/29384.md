## Summary

Keep open: current main already covers the GPU part with typed `sandbox.docker.gpus`, but `--ipc=host` and raw Docker argument passthrough are still absent and security-sensitive. The related `capAdd` PR was closed after a member noted sandbox permission expansion needs explicit maintainer/security approval.

## What This Changes

The issue requests configurable Docker sandbox flags for GPU passthrough and host IPC support.

## Best Possible Solution

Keep the typed GPU support already on main; if host IPC is approved, add a narrow explicit dangerous sandbox option with warnings, audit/docs coverage, config-hash/recreate behavior, and focused tests instead of arbitrary Docker args.

## Reproduction Assessment

Not applicable; this is a feature/config request rather than broken existing behavior. Source inspection gives high confidence that GPU support exists while host IPC and raw Docker args remain absent.

## Solution Assessment

No for generic raw Docker-parameter passthrough because it would bypass the typed sandbox contract. A reviewed explicit host-IPC option is the narrower maintainable path if maintainers choose to support it.
