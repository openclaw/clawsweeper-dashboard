## Summary

Keep open: current main implements the GPU half of the request, but host IPC and generic sandbox Docker flag passthrough are still absent and would change the sandbox isolation model.

## What This Changes

The issue requests configurable sandbox Docker parameters, especially `--gpus all` and `--ipc=host`, for GPU and host-IPC workloads.

## Best Possible Solution

Keep typed GPU support; if host IPC is accepted, add a narrow validated dangerous sandbox Docker config option with warnings, docs, config-hash/recreate coverage, and focused tests rather than arbitrary raw Docker args.

## Reproduction Assessment

Not applicable; this is a feature/config request, not a broken existing behavior report. Source inspection gives high confidence that GPU support exists while host IPC and raw sandbox Docker args are absent.

## Solution Assessment

No for generic raw Docker-parameter passthrough: it would bypass the typed sandbox contract. A typed, reviewed host-IPC option is the narrower maintainable path if maintainers choose to support it.
