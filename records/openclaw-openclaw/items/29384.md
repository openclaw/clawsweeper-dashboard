## Summary

Keep this open. Current main has implemented the GPU passthrough half through typed `sandbox.docker.gpus` config and Docker `--gpus` emission, but the requested host IPC/raw Docker parameter support remains absent and sits on the sandbox isolation boundary that needs maintainer/secops judgment.

## What This Changes

Request to let sandbox Docker configuration pass custom Docker parameters, specifically GPU passthrough and host IPC support.

## Best Possible Solution

Keep the typed GPU support as shipped, then have maintainers/secops decide whether the remaining need should be a narrow, documented, guarded `sandbox.docker.ipc` setting or explicitly out of scope. Avoid arbitrary raw Docker argument passthrough unless the project accepts the validation, audit, docs, and support burden.

## Reproduction Assessment

Not applicable as a bug reproduction, but current-main verification is high confidence: the schema/tests/source prove `gpus` works, while strict config and Docker create-arg code show no IPC/raw-argument surface.

## Solution Assessment

Unclear until maintainers decide the security policy. A narrow typed IPC field with explicit warnings would be more maintainable than raw Docker argument passthrough if the capability is approved at all.
