## Summary

Keep open: current main implements typed GPU passthrough, but host IPC and generic Docker-parameter passthrough are still absent and would expand a security-sensitive sandbox boundary.

## What This Changes

The issue requests configurable sandbox Docker parameters such as `--gpus all` and `--ipc=host` for GPU-accelerated sandbox workloads.

## Best Possible Solution

Keep typed GPU support; if host IPC is accepted, add a narrow validated dangerous sandbox Docker config key with warnings, docs, config-hash/recreate coverage, and focused tests rather than arbitrary raw Docker args.

## Reproduction Assessment

Not applicable; this is a feature/config request, not a broken existing behavior report. Source inspection gives high confidence that GPU support exists while host IPC and raw Docker flag passthrough do not.

## Solution Assessment

No for the generic raw-parameter proposal: arbitrary Docker args are not the safest sandbox contract. A typed, validated host-IPC option is the narrower maintainable path if maintainers choose to support it.
