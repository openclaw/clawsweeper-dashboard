## Summary

Keep open: current main already covers the GPU use case through typed `sandbox.docker.gpus`, but host IPC and generic Docker flag passthrough remain unsupported and security-sensitive.

## What This Changes

The issue requests custom Docker sandbox parameters such as `--gpus all` and `--ipc=host` for GPU and host-IPC workloads.

## Best Possible Solution

Keep the shipped typed GPU option; if host IPC is approved, add a narrow explicit Docker-only config field with validation, audit/docs coverage, config-hash behavior, and focused tests instead of arbitrary Docker args.

## Reproduction Assessment

Not applicable as a bug repro; this is a feature/config request. Source inspection gives high confidence that typed GPU support exists and host IPC/raw Docker argument config remains absent.

## Solution Assessment

No for generic Docker-parameter passthrough because it would bypass the typed sandbox contract. A reviewed explicit host-IPC option is the narrower maintainable path if maintainers choose to support it.
