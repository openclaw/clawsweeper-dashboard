## Summary

Keep this open: the GPU half is already implemented and shipped, but the requested host IPC support is still missing and would deliberately weaken the sandbox boundary.

## What This Changes

Request to allow Docker sandbox containers to receive custom Docker runtime flags such as GPU access and host IPC.

## Best Possible Solution

Keep the shipped typed GPU option and, if maintainers approve host IPC, add a narrowly named break-glass IPC setting with schema enforcement, security diagnostics, docs, and targeted Docker argument tests instead of arbitrary Docker args.

## Reproduction Assessment

Yes, at the source level: current main and v2026.6.6 show typed GPU support, while the strict sandbox Docker schema and create-args path have no IPC field or `--ipc` emission. This is a missing feature capability, not a broken existing contract.

## Solution Assessment

No, the generic custom Docker-flag passthrough proposed in the issue is not the best solution because it would bypass the repository's typed config and security guardrails. A dedicated, explicitly dangerous IPC option is the maintainable path if maintainers accept the tradeoff.
