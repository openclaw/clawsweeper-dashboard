## Summary

Keep open: current main and v2026.5.12 cover the GPU part through typed `sandbox.docker.gpus`, but host IPC and generic Docker flag passthrough remain unsupported and would change the sandbox isolation boundary.

## What This Changes

Request support for custom Docker sandbox runtime parameters such as `--gpus all` and `--ipc=host`.

## Best Possible Solution

Keep the shipped typed GPU option; if host IPC is approved, add a narrow explicit sandbox option with validation, docs, config-hash coverage, audit/security warnings, and focused tests instead of arbitrary Docker args.

## Reproduction Assessment

Not applicable; this is a feature/config request rather than broken existing behavior. Source inspection gives high confidence that GPU support exists while host IPC and raw Docker argument config remain absent.

## Solution Assessment

No for generic Docker-parameter passthrough because it would bypass the typed sandbox contract. A reviewed explicit host-IPC option is the narrower maintainable path if maintainers choose to support it.
