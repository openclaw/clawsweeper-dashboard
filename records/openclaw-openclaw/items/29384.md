## Summary

Keep this open: current main and v2026.6.8 already cover the `--gpus` half with typed `sandbox.docker.gpus` support, but there is still no host IPC or generic Docker-argument sandbox surface. The remaining request is security- and product-sensitive, so it should not be swept closed as implemented.

## What This Changes

The issue requests configurable sandbox Docker run parameters such as GPU access and host IPC for AI/ML workloads.

## Best Possible Solution

Keep the shipped typed GPU support; if maintainers approve host IPC, add a narrow dangerous IPC setting with schema validation, audit/guard visibility, docs, and Docker create-args tests rather than arbitrary Docker flag passthrough.

## Reproduction Assessment

Yes, at the source level: current main and v2026.6.8 show typed GPU support, while the strict Docker sandbox schema and create-args path have no IPC field or `--ipc` emission. This is a missing feature capability, not a broken existing contract.

## Solution Assessment

No, generic custom Docker flag passthrough is not the best solution because it would bypass typed schema and security validation. The maintainable path is a dedicated, explicitly dangerous IPC option if maintainers accept the tradeoff.
