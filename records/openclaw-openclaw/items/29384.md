## Summary

Keep this open: current main and `v2026.6.10` cover the GPU half with typed `sandbox.docker.gpus`, but host IPC and generic Docker-argument passthrough remain absent and security-sensitive.

## What This Changes

Request to support additional Docker sandbox parameters such as GPU passthrough and host IPC for sandbox containers.

## Best Possible Solution

Keep the shipped GPU support; if maintainers approve host IPC, add a narrow explicitly dangerous IPC setting with schema validation, audit visibility, docs, and focused Docker-args tests rather than arbitrary flag passthrough.

## Reproduction Assessment

Yes. Source inspection shows current main and `v2026.6.10` expose `sandbox.docker.gpus`, but neither has an IPC field, raw Docker-args field, or `--ipc` emission for Docker sandbox creation.

## Solution Assessment

No for the proposed generic Docker flag passthrough. The safer OpenClaw-shaped solution is a narrow typed host IPC option only if maintainers accept the sandbox security tradeoff.
