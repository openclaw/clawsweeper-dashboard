## Summary

Keep open: current main and v2026.5.12 already cover the GPU half with typed `sandbox.docker.gpus`, but host IPC and generic Docker flag passthrough remain absent. The remaining work changes Docker sandbox isolation and already has maintainer/product/security review labels, so it should stay with maintainers rather than be swept closed or queued for automation.

## What This Changes

The issue requests OpenClaw sandbox Docker configuration for custom Docker parameters such as `--gpus all` and `--ipc=host`.

## Best Possible Solution

Keep the shipped typed GPU support; if host IPC is approved, add a narrow dangerous sandbox option with validation, audit/docs coverage, config-hash behavior, and focused tests instead of arbitrary Docker args.

## Reproduction Assessment

Not applicable; this is a feature/config request rather than broken existing behavior. Source inspection gives high confidence that GPU support exists while host IPC and raw Docker argument config remain absent.

## Solution Assessment

No for generic Docker-parameter passthrough because it would bypass the typed sandbox contract. A reviewed explicit host-IPC option is the narrower maintainable path if maintainers choose to support it.
