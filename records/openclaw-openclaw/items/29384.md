## Summary

Keeping this open: current main and v2026.5.20 already support the GPU part through typed `sandbox.docker.gpus`, but host IPC and arbitrary Docker flag passthrough remain unsupported and would expand the sandbox isolation boundary.

## What This Changes

Request to allow sandbox Docker configuration for custom Docker create flags such as `--gpus all` and `--ipc=host`.

## Best Possible Solution

Keep the shipped typed GPU option; if host IPC is approved, add a narrow explicit validated sandbox option with docs, security warnings, config-hash coverage, and tests instead of arbitrary Docker args.

## Reproduction Assessment

Not applicable; this is a feature/config request rather than broken existing behavior. Source, docs, release-tag inspection, and PR history give high confidence that GPU support exists while host IPC and raw Docker argument config remain absent.

## Solution Assessment

No for generic Docker-parameter passthrough because it would sidestep the typed sandbox contract. A reviewed explicit host-IPC option is the narrower maintainable path if maintainers choose to support it.
