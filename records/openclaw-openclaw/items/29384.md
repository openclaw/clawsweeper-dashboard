## Summary

Keep open: current main and the latest release already support typed Docker GPU passthrough, but `--ipc=host` and generic sandbox Docker argument passthrough are still absent. The remaining request changes the sandbox isolation boundary and needs maintainer/secops product approval before implementation.

## What This Changes

The issue requests sandbox Docker configuration for custom run parameters such as `--gpus all` and `--ipc=host`.

## Best Possible Solution

Keep the shipped typed GPU support; if host IPC is approved, add a narrow dangerous sandbox option with validation, audit/docs coverage, config-hash/recreate behavior, and focused tests instead of arbitrary Docker args.

## Reproduction Assessment

Not applicable; this is a feature/config request rather than broken existing behavior. Source inspection gives high confidence that GPU support exists while host IPC and raw Docker argument config remain absent.

## Solution Assessment

No for generic raw Docker-parameter passthrough because it would bypass the typed sandbox contract. A reviewed explicit host-IPC option is the narrower maintainable path if maintainers choose to support it.
