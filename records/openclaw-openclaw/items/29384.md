## Summary

Keep open: typed GPU passthrough is already present, but `--ipc=host` and arbitrary Docker argument passthrough remain absent and security-sensitive. The related `capAdd` PR was closed after a maintainer noted that sandbox permission expansion needs explicit maintainer/security approval.

## What This Changes

Request to let sandbox Docker configuration pass custom Docker flags such as `--gpus all` and `--ipc=host` into sandbox container creation.

## Best Possible Solution

Keep the shipped typed GPU support; if host IPC is approved, add a narrow dangerous sandbox option with warnings, audit/docs coverage, config-hash/recreate behavior, and focused tests instead of arbitrary Docker args.

## Reproduction Assessment

Not applicable; this is a feature/config request rather than broken existing behavior. Source inspection gives high confidence that GPU support exists while host IPC and raw Docker args remain absent.

## Solution Assessment

No for generic raw Docker-parameter passthrough because it would bypass the typed sandbox contract. A reviewed explicit host-IPC option is the narrower maintainable path if maintainers choose to support it.
