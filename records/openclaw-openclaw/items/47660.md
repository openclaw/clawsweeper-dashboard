## Summary

Keep open. The PR is a broad Dockerfile refactor that removes current Docker supply-chain and selective plugin-build contracts, and the branch is now conflicting with current main; the attached proof shows the default image can build but does not resolve the policy and compatibility blockers.

## What This Changes

The branch collapses the root Dockerfile to two stages and removes digest-pinned image inputs, Bun/Corepack retry or pinned-copy behavior, runtime variant handling, and selective plugin dependency staging.

## Best Possible Solution

Preserve current main's pinned Docker inputs, selective plugin dependency staging, and runtime hardening, and only accept narrower Dockerfile cleanup that keeps docs, tests, and workflows aligned.

## Reproduction Assessment

Yes for the review findings. PR-head source inspection shows unpinned inputs, unconditional plugin copying, and a stage-scoped ARG issue, while current main docs, tests, and workflows encode the opposite contracts.

## Solution Assessment

No. This broad simplification removes established security and compatibility contracts; the safer path is a maintainer-directed cleanup that preserves pinned images, pinned Bun installation, and `OPENCLAW_EXTENSIONS` unless a migration is explicitly approved.

  - confidence: 0.86
