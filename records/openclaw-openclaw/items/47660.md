- **[P1] Keep Node and Bun inputs pinned:** `Dockerfile:29-33`
  - body: Current main pins Node image digests and copies Bun from a pinned Bun image. This branch switches the build/runtime stages back to rolling Node tags and executes `https://bun.sh/install` during the image build, reopening the supply-chain issue current main already fixed.
  - confidence: 0.96
- **[P1] Preserve selective plugin manifest staging:** `Dockerfile:43`
  - body: Copying the entire plugins tree before `pnpm install` bypasses the documented `OPENCLAW_EXTENSIONS` dependency-baking contract and invalidates the expensive install layer on unrelated plugin source changes. Keep the manifest-only staging path and prune behavior from current main.
  - confidence: 0.95
- **[P1] Keep the runtime-assets validation stage:** `Dockerfile:77-78`
  - body: This branch prunes in the build stage and copies runtime files directly from `build`, so it skips `postinstall-bundled-plugins.mjs`, Docker plugin dist pruning, package dist import validation, patch-file copies, and qa assets that current main explicitly ships or checks before the final image.
  - confidence: 0.96
- **[P1] Restore slim-runtime trust and tool packages:** `Dockerfile:106-108`
  - body: The runtime install list omits `ca-certificates`, `python3`, and `tini` from current main. On `node:24-bookworm-slim`, missing CA certificates breaks Node-driven HTTPS egress and optional browser installation, while missing python3/tini regresses container and skill behavior.
  - confidence: 0.96
- **[P1] Keep Docker GPG multi-key rejection:** `Dockerfile:170-177`
  - body: The Docker CLI install path verifies only the first fingerprint returned by `gpg --with-colons`; current main first requires exactly one primary key before dearmoring so a multi-key blob cannot pass while apt trusts extra keys. Preserve that guard on rebase.
  - confidence: 0.93
- **[P1] Create the state directory before USER node:** `Dockerfile:197`
  - body: Current main pre-creates `/home/node/.openclaw` as `node:node 700` before switching users so named volumes inherit the correct owner and secrecy mode. Dropping that block can make first-run state unwritable or persist credentials with weaker permissions.
  - confidence: 0.94
