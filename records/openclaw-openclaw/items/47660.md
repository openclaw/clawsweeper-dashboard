## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Keep Docker build inputs pinned:** `Dockerfile:33`
  - body: This line reintroduces fetching and executing Bun from `bun.sh` during the image build while the PR also moves the Node stages back to rolling tags. Current main replaced this with digest-pinned Node images and a pinned Bun image COPY, so merging this would undo the reviewed supply-chain fix and make release builds consume unreviewed inputs.
  - confidence: 0.96
- **[P1] Preserve selective plugin manifest staging:** `Dockerfile:43`
  - body: Copying the full `extensions` tree before `pnpm install` bypasses the current manifest-only workspace staging controlled by `OPENCLAW_EXTENSIONS`. That breaks the documented selective plugin dependency contract and makes unrelated plugin source edits invalidate the expensive install layer.
  - confidence: 0.93
- **[P1] Keep the runtime-assets validation stage:** `Dockerfile:77-78`
  - body: This replacement prunes directly in the build stage and drops the current runtime-assets stage that runs bundled-plugin postinstall, Docker plugin dist pruning, package export checks, and runtime-only copies such as `patches`, `pnpm-workspace.yaml`, and `qa`. The resulting image can miss runtime hotfixes and ship unvalidated or incomplete package bytes.
  - confidence: 0.94
- **[P1] Install required slim-runtime packages:** `Dockerfile:107`
  - body: The slim runtime apt list omits `ca-certificates`, `python3`, and `tini`. Current main installs those because the slim Node image otherwise lacks HTTPS trust, skills need Python support, and the container entrypoint needs an init process for long-running gateway behavior.
  - confidence: 0.97
- **[P1] Keep the Docker apt key primary-key guard:** `Dockerfile:172`
  - body: The Docker CLI install path compares only the first fingerprint from the downloaded key file. Current main first requires exactly one primary `pub` key before dearmoring, so removing that guard can let a multi-key blob pass fingerprint comparison while apt trusts the extra keys.
  - confidence: 0.94
- **[P1] Pre-create the node-owned state directory:** `Dockerfile:197`
  - body: The PR switches to `USER node` without the current build-time creation and assertion for `/home/node/.openclaw` as `node:node 700`. First-run named volumes can therefore start as root-owned or too permissive, causing state write failures or weaker credential/auth-profile permissions.
  - confidence: 0.94
- **[P2] Do not start Docker with unconfigured gateway mode:** `Dockerfile:213`
  - body: The PR's runtime command adds `--allow-unconfigured`, while current main starts `openclaw gateway` normally. That changes the Docker image's failure mode for missing configuration and should not be reintroduced as part of a Dockerfile simplification.
  - confidence: 0.82
