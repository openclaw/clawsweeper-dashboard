## Summary

This PR should stay open because the proposed Dockerfile simplification still weakens maintained Docker build contracts: it removes digest-pinned base images, switches Bun installation back to unverified `curl | bash`, drops selective bundled plugin dependency staging, and hardcodes the slim runtime without preserving the current CA certificate setup.

## What This Changes

The PR rewrites the root `Dockerfile` from the current multi-stage, digest-pinned build into a two-stage build with mutable Node image tags, `curl | bash` Bun installation, unconditional plugin copying, and a fixed `node:24-bookworm-slim` runtime.

## Best Possible Solution

Rework any Dockerfile simplification on current main while preserving pinned Node and Bun sources, selective `OPENCLAW_EXTENSIONS` dependency staging, slim-runtime CA certificates, and the documented Docker workflows.

## Reproduction Assessment

Yes. Source inspection of the PR head reproduces the regressions against current main: mutable base tags, unverified Bun installer execution, unconditional plugin workspace installation, and missing slim-runtime CA setup.

## Solution Assessment

No. A broad two-stage rewrite that removes maintained Docker build and supply-chain contracts is not the narrowest maintainable way to simplify this file; simplification should preserve those contracts first.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Keep Docker base images digest-pinned:** `Dockerfile:29`
  - body: The branch replaces the digest-pinned Node image ARGs with mutable `node:24-bookworm` and `node:24-bookworm-slim` tags. Future builds would silently consume different base contents outside the repo's reviewed digest update flow.
  - confidence: 0.94
- **[P1] Keep Bun sourced from the pinned image:** `Dockerfile:33`
  - body: Current main copies Bun from the digest-pinned `OPENCLAW_BUN_IMAGE`; this branch runs `curl -fsSL https://bun.sh/install | bash`, reintroducing unverified build-time code execution in the Docker image build path.
  - confidence: 0.94
- **[P1] Install CA certificates in the slim runtime:** `Dockerfile:106-107`
  - body: The runtime is now always `bookworm-slim`, but the package list omits `ca-certificates` and does not run `update-ca-certificates`. Current main keeps that setup because outbound HTTPS from slim images needs a populated trust store.
  - confidence: 0.9
- **[P2] Preserve selective plugin dependency baking:** `Dockerfile:43`
  - body: Copying all of `extensions/` before `pnpm install` makes every bundled plugin workspace participate in the install, so the documented `OPENCLAW_EXTENSIONS` build arg no longer controls which bundled plugin dependencies are baked into the image.
  - confidence: 0.93
- **[P3] Refresh stale Dockerfile comments:** `Dockerfile:3-14`
  - body: The header and install comments still describe removed stages, variant behavior, digest pinning, and retry behavior. That makes the simplified Dockerfile misleading for future maintainers reading it.
  - confidence: 0.87
