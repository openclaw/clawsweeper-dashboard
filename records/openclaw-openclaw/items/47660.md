## Summary

Keep open for maintainer review: the PR is not merge-ready because it removes Docker supply-chain and selective plugin build contracts that current main now protects, even though the contributor's default-build proof is useful.

## What This Changes

This PR collapses the root Dockerfile into two stages and removes digest-pinned Node/Bun inputs, retry loops, runtime variant handling, and selective plugin dependency staging.

## Best Possible Solution

Preserve current main's pinned Docker inputs and selective plugin dependency contracts, and accept only a narrower Dockerfile cleanup that keeps docs, tests, and workflows aligned.

## Reproduction Assessment

Yes for the review findings. PR-head source inspection shows unpinned inputs, unconditional extension copying, and a stage-scoped ARG issue, while current main tests and docs encode the opposite contracts.

## Solution Assessment

No. This broad simplification removes security and compatibility contracts; the safer path is a maintainer-directed cleanup that preserves pinned images, pinned Bun installation, and `OPENCLAW_EXTENSIONS` behavior unless a migration is explicitly approved.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Keep Docker build inputs pinned:** `Dockerfile:29-33`
  - body: The branch switches the build/runtime bases to rolling `node:24-*` tags and runs `curl -fsSL https://bun.sh/install | bash`. Current main pins Node and Bun image inputs and copies Bun from a pinned image, so this regresses reproducibility and supply-chain hardening.
  - confidence: 0.95
- **[P2] Preserve selective plugin dependency staging:** `Dockerfile:43`
  - body: Copying all of `extensions` before `pnpm install` makes every plugin workspace participate regardless of `OPENCLAW_EXTENSIONS`. Current docs, setup scripts, workflows, and tests still use that build arg to control plugin dependency baking.
  - confidence: 0.93
- **[P2] Redeclare runtime build args in the runtime stage:** `Dockerfile:103-104`
  - body: `OPENCLAW_DOCKER_APT_UPGRADE` is declared before the first `FROM`, but the runtime-stage check uses it without redeclaring it. Docker does not carry global ARG values into later stages, so the opt-out expands empty and is ignored on this branch.
  - confidence: 0.86
- **[P3] Update comments that describe removed Docker behavior:** `Dockerfile:3-17`
  - body: The header and Bun install comments still describe removed `OPENCLAW_VARIANT`, `ext-deps`, digest pinning, and retry behavior. If this refactor continues, those comments need to match the actual build path.
  - confidence: 0.82
