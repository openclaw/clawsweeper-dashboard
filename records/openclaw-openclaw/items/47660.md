## Summary

Keep open. The attached log is enough real behavior proof for the default after-change Docker build, but the PR remains unsafe to merge because it rolls back current Docker supply-chain hardening and the documented selective plugin dependency contract.

## What This Changes

The branch collapses the root Dockerfile into a two-stage build with rolling Node tags, a curl-installed Bun, unconditional plugin source copying, and simplified runtime apt/cache handling.

## Best Possible Solution

Keep main's pinned Docker inputs and `OPENCLAW_EXTENSIONS` workspace-dependency contract; any simplification should be a smaller maintainer-directed refactor that preserves those contracts and updates docs/tests together.

## Reproduction Assessment

Yes for the review findings: source inspection of the PR head and current main shows the branch drops pinned inputs, reintroduces a remote Bun installer, and bypasses selective plugin dependency staging. The attached log proves the default after-change build path only.

## Solution Assessment

No. The broad simplification is not the best path because it removes current security and compatibility contracts; a targeted refactor that keeps pinned inputs and selective plugin dependency staging would be safer.
