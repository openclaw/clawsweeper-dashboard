## Summary

Keep open, but the current PR patch is not merge-ready. It simplifies the Dockerfile by removing stages, but it also breaks the documented selective bundled-plugin dependency contract and reintroduces Docker supply-chain regressions that current main has since hardened.

## What This Changes

The PR rewrites the root Dockerfile into a two-stage build, removes retry loops and digest-pinned image references, hardcodes the slim runtime base, and copies all extensions before dependency installation.

## Best Possible Solution

Keep the current pinned/verified Docker input policy and selective bundled-plugin build contract; if maintainers still want simplification, rebase into a narrower refactor that preserves those contracts and updates docs and Docker smoke coverage.

## Reproduction Assessment

Yes, for the source-level regressions. The PR head copies all extensions before install while current docs and setup still use `OPENCLAW_EXTENSIONS`, and it bypasses current main's pinned Node/Bun inputs; I did not run a local Docker build in this read-only review.

## Solution Assessment

No. The simplification removes active Docker contracts and current security hardening; the safer path is a smaller current-main refactor that keeps digest-pinned inputs, selective plugin dependency baking, and existing Docker validation intact.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Keep Docker build inputs pinned and verified:** `Dockerfile:29-33`
  - body: This branch declares digest-pinned image args but then builds from bare `node:24-*` tags and reinstates `curl | bash` for Bun. Current main has since pinned the Bun image and Node bases, so merging this would reintroduce a supply-chain/reproducibility regression fixed by the Docker hardening work.
  - confidence: 0.94

  - body: Copying the entire `extensions/` tree before `pnpm install` makes `OPENCLAW_EXTENSIONS` ineffective. The setup script and Docker docs still use that build arg to include selected bundled plugin helpers, so users who request one plugin would get all extension dependencies baked into the image.
  - confidence: 0.93
- **[P2] Re-declare the apt upgrade arg in the runtime stage:** `Dockerfile:103`
  - body: `OPENCLAW_DOCKER_APT_UPGRADE` is only declared before the first `FROM`, so it is out of scope in this runtime-stage `RUN`. The condition expands empty and always runs the distro upgrade even when callers pass `--build-arg OPENCLAW_DOCKER_APT_UPGRADE=0`, defeating the documented smoke-build fast path in this branch.
  - confidence: 0.86
