## Summary

Keep open. The PR includes real Docker build proof, but as written it regresses Docker supply-chain pinning and the selective plugin dependency contract that current main, docs, setup scripts, and tests preserve.

## What This Changes

The PR simplifies the root Dockerfile by reducing stages and removing digest-pinned image args, Bun/Corepack retry loops, runtime variant selection, and selective extension manifest staging.

## Best Possible Solution

Extract only simplifications that preserve digest-pinned Docker/Bun inputs, `OPENCLAW_EXTENSIONS` selective install/prune behavior, current runtime packages and init handling, and the existing Docker smoke/tests.

## Reproduction Assessment

Yes for the review findings. Static inspection of the PR head versus current main docs and tests proves the pinning and selective-extension regressions without needing to run Docker.

## Solution Assessment

No. The maintainable path is to rebase onto current main and keep the Docker security, plugin-selection, and runtime contracts while simplifying only the parts that do not change those behaviors.
