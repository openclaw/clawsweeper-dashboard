## Summary

Keep open: the PR is not ready because it removes Docker build and supply-chain contracts that current main, docs, and install workflows rely on.

## What This Changes

The PR rewrites the root Dockerfile into a two-stage build, removes selective bundled plugin staging and runtime variant handling, drops digest-pinned image sources, and keeps the optional Docker runtime blocks.

## Best Possible Solution

Rework any Dockerfile simplification on current main while preserving pinned Node and Bun sources, selective `OPENCLAW_EXTENSIONS` dependency staging, runtime CA certificates, and the documented Docker workflows.

## Reproduction Assessment

Yes. Source inspection of the PR head reproduces the regressions: mutable base images, `curl | bash` Bun install, full bundled plugin workspace install, and missing slim-runtime CA certificates.

## Solution Assessment

No. A broad two-stage rewrite that removes maintained Docker build and supply-chain contracts is not the narrowest maintainable way to simplify the Dockerfile.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Keep Bun sourced from the pinned image:** `Dockerfile:33`
  - body: Current main copies Bun from the digest-pinned `OPENCLAW_BUN_IMAGE`. This branch runs `curl -fsSL https://bun.sh/install | bash`, which reintroduces unverified build-time code execution in the release image path.
  - confidence: 0.92
- **[P1] Keep Docker base images digest-pinned:** `Dockerfile:29`
  - body: The branch replaces the digest-pinned Node base image ARGs with mutable `node:24-bookworm` tags. Future builds would silently consume different base contents outside the repo's reviewed digest update flow.
  - confidence: 0.93
- **[P1] Install CA certificates in the slim runtime:** `Dockerfile:106-107`
  - body: The runtime is hardcoded to `bookworm-slim`, but the package list omits `ca-certificates` and never runs `update-ca-certificates`. Current main installs both because HTTPS outbound calls fail in the slim image without a populated trust store.
  - confidence: 0.92
- **[P2] Preserve selective plugin dependency baking:** `Dockerfile:43`
  - body: Copying all of `extensions/` before `pnpm install` makes every bundled plugin workspace participate in the install. Docs, setup, and smoke workflows still use `OPENCLAW_EXTENSIONS` to bake only selected plugin dependencies.
  - confidence: 0.94
- **[P3] Refresh stale Dockerfile comments:** `Dockerfile:3-14`
  - body: The header still describes the removed `ext-deps` and runtime-variant behavior, and the Bun comment still promises retry behavior after the retry loop was removed. That leaves misleading instructions in the file this PR is simplifying.
  - confidence: 0.88
