## Summary

Keep open for maintainer judgment: the PR is a broad Docker infrastructure refactor that currently regresses pinned build inputs and the documented selective plugin dependency contract, and the live PR is conflicting with current main.

## What This Changes

The PR collapses the root Dockerfile into a two-stage build while removing digest-pinned Node/Bun inputs, retry loops, runtime variant handling, and selective plugin dependency staging.

## Best Possible Solution

Preserve current main's pinned Docker inputs and selective plugin dependency contracts, and consider only a smaller Dockerfile cleanup that keeps docs, tests, and workflows aligned.

## Reproduction Assessment

Yes for the review findings. Source inspection of the PR head against current main shows the branch drops pinned inputs, bypasses selective plugin dependency staging, and has a stage-scoped ARG bug; the attached log only proves the default build path.

## Solution Assessment

No. This broad simplification is not the best fix because it removes Docker security and compatibility contracts; a targeted maintainer-directed refactor that preserves those contracts would be safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Keep Docker build inputs pinned:** `Dockerfile:29-33`
  - body: The branch switches the build/runtime bases to rolling `node:24-*` tags and reinstates `curl -fsSL https://bun.sh/install | bash`. Current main pins the Node images and copies Bun from digest-pinned `OPENCLAW_BUN_IMAGE`, so this regresses reproducibility and supply-chain hardening.
  - confidence: 0.95
- **[P2] Preserve selective plugin dependency staging:** `Dockerfile:43`
  - body: Copying all of `extensions` before `pnpm install` makes every plugin workspace participate regardless of `OPENCLAW_EXTENSIONS`. Current docs, setup scripts, workflows, and tests still use that build arg to control plugin dependency baking, so this breaks the documented Docker build contract.
  - confidence: 0.93
- **[P2] Keep the runtime APT upgrade arg in stage scope:** `Dockerfile:103-104`
  - body: This branch declares `OPENCLAW_DOCKER_APT_UPGRADE` only before the first `FROM`, but checks it in the runtime stage. Docker does not carry global ARG values into later stages unless redeclared, so `--build-arg OPENCLAW_DOCKER_APT_UPGRADE=0` is ignored on the PR head.
  - confidence: 0.86
- **[P3] Update stale Dockerfile comments:** `Dockerfile:12-26`
  - body: The comments still describe removed `OPENCLAW_VARIANT`, `ext-deps`, digest pinning, and retry behavior. If this refactor continues, those comments need to match the actual build path so contributors do not follow removed options.
  - confidence: 0.82
