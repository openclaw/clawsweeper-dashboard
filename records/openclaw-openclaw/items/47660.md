## Summary

Keep this PR open for maintainer handling: current main does not implement the proposed simplification, and the branch still removes maintained Docker supply-chain and plugin-selection contracts. The existing ClawSweeper review and the `triage: risky-infra` label align with the source review here.

## What This Changes

The PR rewrites the root Dockerfile into two build/runtime stages, removing digest-pinned base args, selective `OPENCLAW_EXTENSIONS` staging, Bun/Corepack retry logic, and runtime variant handling.

## Best Possible Solution

Rework any Dockerfile simplification on current main while preserving pinned Node and Bun inputs, selective `OPENCLAW_EXTENSIONS` staging, slim-runtime CA setup, and documented Docker workflows unless maintainers explicitly approve those contract changes.

## Reproduction Assessment

Yes. Source inspection of the PR head reproduces mutable base tags, unverified Bun installer execution, all-extension dependency installation, missing slim-runtime CA setup, and the scoped ARG issue; current main, docs, setup, and workflows show these are still maintained contracts.

## Solution Assessment

No. A broad Dockerfile simplification that removes supply-chain pins and supported build args is not the narrowest maintainable path; the safer direction is a smaller refactor that keeps those contracts intact.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Keep Docker base images digest-pinned:** `Dockerfile:29`
  - body: Current main builds through digest-pinned Node image args and records base-image metadata. This line switches the build stage to a mutable tag, and the runtime stage does the same, so future release builds could silently consume unreviewed upstream contents.
  - confidence: 0.94
- **[P1] Keep Bun sourced from a pinned image:** `Dockerfile:33`
  - body: Current main copies Bun from the digest-pinned `OPENCLAW_BUN_IMAGE`. Running the remote installer as root with no version, checksum, or image digest reintroduces unverified build-time code execution in the Docker image path.
  - confidence: 0.94
- **[P1] Install CA certificates in the slim runtime:** `Dockerfile:106-107`
  - body: The runtime is now always `node:24-bookworm-slim`, but this package list omits `ca-certificates` and never runs `update-ca-certificates`. Current main installs both because HTTPS outbound calls fail in the slim image without a populated trust store.
  - confidence: 0.92
- **[P2] Preserve selective plugin dependency staging:** `Dockerfile:43`
  - body: Copying the full `extensions/` tree before `pnpm install` makes every bundled plugin workspace participate in the install. Docs, setup, and CI still pass `OPENCLAW_EXTENSIONS` to bake only selected plugin dependencies, so this regresses that contract.
  - confidence: 0.93
- **[P2] Re-declare the apt upgrade build arg:** `Dockerfile:103`
  - body: `OPENCLAW_DOCKER_APT_UPGRADE` is declared only before the first `FROM`, so it is out of scope in this runtime-stage `RUN`. The condition expands empty and callers cannot disable the upgrade path with the build arg.
  - confidence: 0.88
- **[P3] Refresh comments after removing build modes:** `Dockerfile:3-14`
  - body: The header still documents `OPENCLAW_EXTENSIONS`, `OPENCLAW_VARIANT`, the `ext-deps` stage, digest pinning, and Bun retry behavior that this branch removes. That leaves misleading operator guidance in the file being simplified.
  - confidence: 0.86
