## Summary

Keep open for human Docker/security review: the PR is a real refactor proposal, but it removes maintained Docker build contracts and reintroduces supply-chain risks that current main has since hardened.

## What This Changes

The PR branch simplifies the root Dockerfile to two stages while removing digest-pinned Node/Bun inputs, selective plugin dependency staging, runtime variant handling, and retry/cache paths.

## Best Possible Solution

Rework any Dockerfile simplification on current main while preserving pinned Node and Bun inputs, selective OPENCLAW_EXTENSIONS dependency staging, slim-runtime CA setup, and documented Docker setup workflows unless Docker/security owners explicitly approve contract changes.

## Reproduction Assessment

Yes. Source inspection of the PR head against current main reproduces the mutable base tags, unverified Bun installer execution, all-plugin dependency install, missing slim-runtime CA setup, and scoped ARG issue; a live rebuild was not needed to establish those source-level regressions.

## Solution Assessment

No. A broad Dockerfile simplification that removes supply-chain pins and supported build args is not the narrowest maintainable solution; a safer refactor would keep those contracts intact and simplify only around them.
