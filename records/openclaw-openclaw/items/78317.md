Full review comments:

- **[P1] Clamp reply-cache permissions after appending:** `extensions/imessage/src/monitor-reply-cache.ts:156-159`
  - body: Existing reply-cache files keep their old mode here: `appendFileSync`'s `mode` only applies on creation, and chmod only runs when `fileExisted` is false. Upgrade files at 0644 remain readable after normal append traffic, so chmod the directory and file after every successful append.
  - confidence: 0.96
- **[P1] Protect the persisted echo cache:** `extensions/imessage/src/monitor/persisted-echo-cache.ts:105-110`
  - body: `sent-echoes.jsonl` stores conversation scope, text, and message IDs, but this write path uses default directory/file modes. Under normal umask it reintroduces the state-file disclosure class this PR is trying to fix; create/chmod this cache owner-only too.
  - confidence: 0.95
- **[P2] Use the implemented imsg surface for typing/read:** `extensions/imessage/src/chat.ts:90-95`
  - body: These calls go through `createIMessageRpcClient`, but current `imsg rpc` only handles `chats.list`, `messages.history`, `watch.subscribe`, `watch.unsubscribe`, and `send`; `typing` and `read` are standalone CLI commands. If capability gating ever enables these methods, they fail with method-not-found, so call the CLI bridge or wait for a real RPC contract.
  - confidence: 0.9
- **[P2] Preserve markdown when RPC send cannot format it:** `extensions/imessage/src/send.ts:219-236`
  - body: This strips markdown markers before the normal RPC `send` path, then passes `formatting` metadata that current `imsg rpc` ignores. Plain sends like `**bold**` therefore deliver marker-stripped text with no styling; leave the text unchanged unless routing through the bridge-backed rich sender.
  - confidence: 0.88
- **[P3] Map the advertised effect aliases:** `extensions/imessage/src/actions.ts:314-317`
  - body: The error text advertises `echo`, `happybirthday`, `shootingstar`, `sparkles`, and `spotlight`, but the alias table never maps those values to the canonical effect IDs. Passing those documented aliases falls through to the same unknown-effect error; add the aliases or remove them from the guidance.
  - confidence: 0.9
