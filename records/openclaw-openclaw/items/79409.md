## Summary

Keep open: current main still source-reproduces the Feishu slash-command authorization drop, and the open/closed related PRs have not shipped a fix.

## What This Changes

Feishu should intercept text slash commands such as `/compact`, `/new`, and `/stop` as Gateway commands instead of forwarding them to the agent as plain text.

## Best Possible Solution

Land a narrow Feishu command-authorization fix that treats already-admitted senders as command-authorized when no explicit Feishu command allowlist is configured, while preserving explicit allowlist denial and owner-only command enforcement.

## Reproduction Assessment

Yes, source-level: current main can carry an admitted Feishu slash command into `CommandAuthorized=false`, and the reply path then returns no response for unauthorized whole-message control commands. I did not run a live Feishu bot workspace.

## Solution Assessment

Yes for the narrow direction: fix the Feishu authorization call site or add an explicitly scoped helper mode, not a broad shared helper change that widens unrelated command authorization.
