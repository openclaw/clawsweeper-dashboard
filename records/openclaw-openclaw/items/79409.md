## Summary

Keep open: current main still source-reproduces the Feishu slash-command authorization drop, and the related implementation PRs are open or closed-unmerged rather than shipped.

## What This Changes

The issue asks for Feishu text slash commands such as `/compact`, `/new`, and `/stop` to be intercepted by the Gateway instead of being sent to the agent as plain text.

## Best Possible Solution

Land a narrow authorization fix that treats already-admitted Feishu senders as command-authorized when no explicit command allowlist is configured, while keeping explicit allowlists and owner-only commands fail-closed.

## Reproduction Assessment

Yes, at source level: current main can carry a Feishu slash command with no configured command allowlist into `CommandAuthorized=false`, and the reply path then drops whole-message control commands. I did not run the exact live Feishu DM setup.

## Solution Assessment

Yes for the narrow authorization direction: fixing the Feishu command authorization call path or a carefully scoped shared helper is the maintainable repair. The open `CommandSource`-only PR is not sufficient if `CommandAuthorized` remains false.
