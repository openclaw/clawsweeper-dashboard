## Summary

Keep open: current main still has a source-reproducible Feishu text-command gap for the stateful recovery commands named here, and the open related fast-path PR explicitly excludes `/compact`, `/new`, and `/stop`.

## What This Changes

The issue requests that Feishu text slash commands such as `/compact`, `/new`, and `/stop` be intercepted and executed as commands instead of being sent to the agent as plain text.

## Best Possible Solution

Land or sponsor a focused Feishu text-command fix that marks whole-message slash text as structured command turns, preserves explicit authorization semantics, and covers `/compact`, `/new`, and `/stop` with Feishu regression coverage.

## Reproduction Assessment

Yes, at source level: current Feishu context construction still omits explicit text-slash command facts, and the shared command gate plus reply runner preserve the no-reply path when command authorization fails. I did not run a live Feishu tenant reproduction in this read-only review.

## Solution Assessment

Unclear as a product decision: a simple allow-by-default authorization change would be risky for stateful commands. The best fix should pair Feishu text-slash command-turn metadata with a maintainer-approved authorization rule and focused regression coverage.
