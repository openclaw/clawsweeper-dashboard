## Summary

Keep open: current main still source-reproduces the Feishu slash-command authorization drop, and the related fix attempts are unmerged or still under review rather than shipped.

## What This Changes

The issue asks Feishu websocket messages like `/compact`, `/new`, and `/stop` to execute as text slash commands instead of reaching the agent as plain text.

## Best Possible Solution

Land a narrow Feishu command-authorization fix that treats admitted senders as command-authorized when no explicit command allowlist is configured, while preserving fail-closed behavior for explicit allowlists and owner-only commands.

## Reproduction Assessment

Yes, source-reproducible: current main can carry an admitted Feishu slash command into `CommandAuthorized=false`, and the reply path then returns no response for unauthorized whole-message control commands. I did not run a live Feishu bot workspace.

## Solution Assessment

Yes for the narrow direction: fix the Feishu authorization call site or add an explicit scoped helper mode, not a broad shared helper change that widens unrelated DM command authorization.
