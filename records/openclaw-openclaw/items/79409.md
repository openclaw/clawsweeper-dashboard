## Summary

Keeping this open: current main still has a source path where Feishu text slash commands can be marked unauthorized and swallowed, and an open linked PR references this issue as a fix path.

## What This Changes

The issue asks for Feishu text slash commands such as `/compact`, `/new`, and `/stop` to be handled as commands instead of being delivered to the agent as plain text.

## Best Possible Solution

Align Feishu command authorization with the documented access model: explicit allowlists or wildcard-open access should permit text commands, while the empty-allowFrom open-DM case should either be rejected clearly or deliberately allowed with tests and docs.

## Reproduction Assessment

No high-confidence live reproduction was established because I did not run a real Feishu websocket bot. Source inspection matches the reported swallow path: Feishu can set `CommandAuthorized: false`, and `runPreparedReply` returns `undefined` for unauthorized whole-message control commands.

## Solution Assessment

Unclear. A Feishu-local fix may be narrow, but changing empty-allowFrom command authorization conflicts with existing fail-closed tests and docs that require `allowFrom: ["*"]` for open DMs.
