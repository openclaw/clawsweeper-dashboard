## Summary

Keep open: current main still source-reproduces the Feishu slash-command authorization drop, and the linked fix PRs are open or closed-unmerged rather than shipped.

## What This Changes

Feishu channel should handle standalone text slash commands such as `/compact`, `/new`, and `/stop` instead of forwarding them to the agent as plain text.

## Best Possible Solution

Review and land a narrow Feishu command-authorization fix that treats admitted senders as command-authorized when no explicit command allowlist is configured, while preserving explicit allowlist and owner-command enforcement.

## Reproduction Assessment

Yes. Current main can still carry a Feishu slash command into `CommandAuthorized=false` with `commands.useAccessGroups=true` and no configured command allowlist, and the reply path returns no response for unauthorized whole-message control commands.

## Solution Assessment

Yes for the narrow direction, but not for closing now. The best fix should address Feishu command authorization or add an explicit shared helper mode, not rely only on `CommandSource`, and the open linked PRs need review or revision before this issue is resolved.
