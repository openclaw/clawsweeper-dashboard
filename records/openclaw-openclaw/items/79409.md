## Summary

Keep open: current main still has a source-reproducible Feishu text-command gap for the stateful commands named here, and the visible related PR work does not yet close that behavior.

## What This Changes

Issue reports Feishu text slash commands such as `/compact`, `/new`, and `/stop` being routed as normal agent text instead of OpenClaw control commands.

## Best Possible Solution

Maintainers should choose the canonical Feishu text-command contract, preferably structured `CommandTurn` plus explicit authorization semantics, then cover `/compact`, `/new`, and `/stop` in focused Feishu DM/group tests.

## Reproduction Assessment

Yes. Source inspection shows Feishu currently supplies the command body but no text command turn/source, and the reply runner can return no reply for unauthorized whole-message control commands; I did not run live Feishu in this read-only review.

## Solution Assessment

No current main implementation fully solves this issue. The old same-repo PR is closed unmerged, the open fast-path PR explicitly excludes the named stateful commands, and the best fix needs a maintainer decision on the command-turn and authorization boundary.
