## Summary

Keep open: current main still source-reproduces the Feishu slash-command authorization drop, and the related fix PRs are open, closed-unmerged, or under maintainer review rather than shipped.

## What This Changes

Feishu should intercept standalone text slash commands such as `/compact`, `/new`, and `/stop` instead of dispatching them as agent text.

## Best Possible Solution

Land a narrow Feishu command-authorization fix that authorizes admitted DM/group senders when no explicit command allowlist is configured while preserving explicit allowlist and owner-only command enforcement.

## Reproduction Assessment

Yes, source-reproducible: current main can carry an admitted Feishu slash command into `CommandAuthorized=false`, and the reply path then returns no response for unauthorized whole-message control commands. I did not run a live Feishu bot workspace.

## Solution Assessment

Yes for the narrow direction: a Feishu-scoped authorization fix, or an explicit scoped helper mode, is the maintainable path. A `CommandSource`-only fix does not address the `CommandAuthorized=false` root cause, and a broad shared-helper change needs care because it can widen unrelated command authorization.
