## Summary

Keep open: current main still source-reproduces the Feishu slash-command authorization failure, and the linked fixes are open or closed-unmerged rather than shipped.

## What This Changes

The issue asks for Feishu DM/group text slash commands such as `/compact`, `/new`, and `/stop` to be handled by the Gateway command path instead of being dispatched to the agent as plain text.

## Best Possible Solution

Land one narrow authorization fix that lets already-admitted Feishu senders run text slash commands when no explicit command allowlist is configured, while preserving explicit allowlists and owner-only command checks.

## Reproduction Assessment

Yes. Current main source-reproduces the failure path: Feishu can produce `CommandAuthorized=false` for a slash command with no command allowlist, and the reply runner then drops unauthorized whole-message control commands without a response.

## Solution Assessment

Yes for the narrow direction. The maintainable fix should adjust the Feishu command authorization path or the shared command-gating helper with regression coverage, without widening explicit allowlists or owner-only command behavior.
