## Summary

Keep open: current main and the latest release still preserve the command-authorization path that can make Feishu text control commands unauthorized and then silently drop whole-message `/compact`, `/new`, or `/stop` turns; the linked fix PRs are open/conflicting or partial rather than merged.

## What This Changes

The issue asks OpenClaw to intercept Feishu text slash commands such as `/compact`, `/new`, and `/stop` instead of dispatching them to the agent as plain text.

## Best Possible Solution

Review, refresh, or replace the open Feishu command PR with a narrow fix that covers admitted-sender command authorization and explicit acceptance coverage for `/compact`, `/new`, and `/stop`.

## Reproduction Assessment

Yes from source, but not by live Feishu execution in this review. Current main still denies text commands when access groups are enabled and no authorizer is configured, then the reply path silently drops unauthorized whole-message control/reset commands.

## Solution Assessment

No, this is not solved on main. A CommandSource-only fix is too narrow while `CommandAuthorized` remains false; the best fix should repair the Feishu/core command authorization boundary and cover the reported stateful commands.
