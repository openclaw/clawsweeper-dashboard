## What This Changes

Feishu text slash commands such as `/compact`, `/new`, and `/stop` should be intercepted by Gateway instead of reaching the agent as plain text.

## Best Possible Solution

Land one narrow Feishu command-authorization fix that authorizes admitted Feishu senders when no explicit command allowlist is configured, while preserving explicit allowlist and owner-only command enforcement.

## Reproduction Assessment

Yes. Current main source-reproduces the path: Feishu can carry an admitted slash command into `CommandAuthorized=false`, and the reply path drops unauthorized whole-message control commands.

## Solution Assessment

Unclear across the existing PRs. A Feishu-scoped fix or explicit scoped helper mode is safer than broadly changing shared command-gating semantics without preserving other fail-closed paths.
