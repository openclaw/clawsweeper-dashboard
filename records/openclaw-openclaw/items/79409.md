## Summary

Keep open: current main still source-reproduces the Feishu slash-command authorization drop, and the related fix PRs are open or closed unmerged rather than shipped.

## What This Changes

Feishu text slash commands such as `/compact`, `/new`, and `/stop` should be intercepted by Gateway instead of being sent to the agent as plain text.

## Best Possible Solution

Land a narrow Feishu command-authorization fix that treats admitted Feishu senders as command-authorized when no explicit command allowlist is configured, while preserving explicit allowlists and owner-only command gates.

## Reproduction Assessment

Yes, source-reproducible: current main can carry Feishu `/compact` or `/status` messages into `CommandAuthorized=false` when no command allowlist is configured, and the reply path drops unauthorized whole-message control commands. I did not run a live Feishu workspace in this read-only review.

## Solution Assessment

Yes for the narrow Feishu authorization direction; no for closing now or spawning a third repair branch while open related PRs already claim the fix path. A shared helper change would need explicit scope and SDK/root-alias alignment to avoid widening unrelated command access.
