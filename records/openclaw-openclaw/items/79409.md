Keep open: current main still has the Feishu command-authorization and unauthorized whole-message control-command path that can turn `/compact`, `/new`, and `/stop` into a no-reply outcome, and the open linked fix PR is still unresolved.

## What This Changes

The issue requests that Feishu websocket text slash commands such as `/compact`, `/new`, and `/stop` be intercepted as control commands instead of being handled as ordinary agent text.

## Best Possible Solution

Keep this issue open until maintainers choose one canonical Feishu command path that covers `/compact`, `/new`, and `/stop` with explicit authorization semantics and live Feishu acceptance proof, then close from the merged fix.

No final fix is proven yet. The safer path is a maintainer-approved refresh or replacement of the linked Feishu fix path with explicit authorization semantics and Feishu `/compact`, `/new`, and `/stop` coverage, not closing this issue or queueing a duplicate automated repair.
