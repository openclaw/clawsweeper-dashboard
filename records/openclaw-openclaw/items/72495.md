## Summary

Keep open: current main still keys the guarded workspace bootstrap file cache without `ctimeMs`, so the equal-size/restored-mtime stale-cache gap remains. The PR's production change and updated regression test are narrow and substantively correct, with merge hygiene and validation still needed before landing.

## What This Changes

This PR adds `ctimeMs` to the guarded workspace bootstrap file-cache identity, adds a restored-mtime regression test, and records the user-facing fix in the changelog.

## Best Possible Solution

Land this PR or an equivalent maintainer patch after rebasing to current main, preserving the ctime-backed identity, the restored-mtime regression test, and the changelog entry.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to load a workspace `AGENTS.md`, rewrite it in place with equal-size content, restore the original mtime, and load again; current main can reuse the old cache entry because its identity lacks ctime.

## Solution Assessment

Yes. Adding `ctimeMs` to the existing fstat-backed identity is the narrowest maintainable fix for this cache gap; hashing every read or removing the lower cache would broaden the existing performance/security tradeoff.
