## What This Changes

This PR adds `stat.ctimeMs` to the guarded workspace bootstrap file-cache identity, adds restored-mtime regression coverage, and records the fix in `CHANGELOG.md`.

## Best Possible Solution

Land this PR or an equivalent maintainer patch after rebasing to current main, preserving the ctime-backed identity, restored-mtime regression test, and changelog entry.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to load a workspace `AGENTS.md`, rewrite it in place with equal-size content, restore the original mtime, and load again; current main can reuse the old cache entry because its identity lacks ctime.

## Solution Assessment

Yes. Adding `ctimeMs` to the existing fstat-backed identity is the narrowest maintainable fix for this cache gap; hashing every read or removing the lower cache would broaden the performance/security tradeoff.
