## Best Possible Solution

Land this PR or an equivalent maintainer patch that adds `ctimeMs` to the existing fstat-backed workspace file identity, keeps the focused restored-mtime regression test, and preserves the current lightweight cache design.

Yes. The source-level reproduction is high-confidence: load a workspace `AGENTS.md`, rewrite it in place with equal-size content, restore the original mtime, and current main can reuse the old cached content because the identity lacks ctime.

## Solution Assessment
