## What This Changes

The PR adds `stat.ctimeMs` to the workspace bootstrap guarded file-cache identity, adds a same-inode/same-size/restored-mtime regression test, and records the fix in the changelog.

## Best Possible Solution

Land this PR or an equivalent maintainer patch after rebasing to current main, preserving the ctime-backed identity, the restored-mtime regression test, and the user-facing changelog entry without broadening the bootstrap cache design.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to load a workspace `AGENTS.md`, rewrite it in place with equal-size content, restore the original mtime, and load again; current main can reuse the old entry because its identity lacks ctime.

## Solution Assessment

Yes. Adding `ctimeMs` to the existing fstat-backed identity is the narrowest maintainable fix for this cache gap; hashing every read or removing the lower cache would be broader than the existing performance/security tradeoff requires.
