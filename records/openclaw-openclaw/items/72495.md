## Summary

Keep open: current main still omits `ctimeMs` from the guarded workspace bootstrap file identity, and this PR is a focused fix with matching regression coverage and changelog entry.

## What This Changes

This PR adds `ctimeMs` to the workspace bootstrap file-cache identity, adds restored-mtime cache-invalidation coverage, and records the user-facing fix in `CHANGELOG.md`.

## Best Possible Solution

Land this PR or an equivalent maintainer patch with the ctime-backed identity, restored-mtime regression test, and changelog entry preserved.

## Reproduction Assessment

Yes, at source level. On current main, load a workspace `AGENTS.md`, rewrite it in place with equal-size content, restore the original mtime, and load again; the cache identity lacks ctime, so the stale entry can still match.

## Solution Assessment

Yes. Adding `ctimeMs` to the existing fstat-backed identity is the narrowest maintainable fix for this cache gap; hashing every read or removing the lower cache would broaden the performance tradeoff.
