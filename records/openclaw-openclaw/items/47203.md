## Summary

Keep open. Current main still exposes documented browser navigation while dropping navigate timeout/load-state controls before the host client, node proxy path, `/navigate` route, and Playwright call, so the focused 20s navigate timeout report remains actionable rather than closeable as stale or duplicate.

## What This Changes

The issue asks for `browser navigate` to avoid the 20s timeout race by honoring navigate wait-state/timeout controls and reporting navigation timeouts accurately.

## Best Possible Solution

Propagate explicit navigate timeout/load-state controls through the browser tool, CLI/client, node proxy, `/navigate` route, and Playwright guard while preserving SSRF checks and current defaults unless maintainers separately choose a default wait-state change.

## Reproduction Assessment

Yes, at source level. Passing `timeoutMs` or `loadState` to browser navigate on current main is enough to see those fields stop before the route and Playwright call, though this review did not run the Linux VNC setup live.

## Solution Assessment

Yes for a narrow pass-through and timeout-budget repair. A raw-CDP fallback or changed default wait state may be useful later, but that is broader than needed to fix the demonstrated option-drop and timeout-race path.
