The issue asks for `browser navigate` to stop timing out at the fixed 20s budget on Linux VNC by honoring navigation timeout/load-state controls and improving timeout handling.

## Best Possible Solution

Implement a narrow browser plugin repair that threads normalized navigate timeout and supported wait/load-state controls through the existing tool, client, proxy, route, and Playwright seams while preserving SSRF checks and current defaults unless the caller opts in.
