## Summary

Keep open. Current main still exposes browser navigation but drops `timeoutMs`/`loadState` before the `/navigate` route and Playwright call, while the client keeps a fixed 20s outer timeout; related #72653 is broader, but this remains the focused navigate timeout report.

## What This Changes

The issue requests that `browser navigate` avoid the Linux VNC 20s timeout race by honoring navigation timeout/load-state controls and reporting navigation timeouts accurately.

## Best Possible Solution

Propagate explicit navigate timeout/load-state controls through the tool, CLI/client, proxy route, and Playwright navigation guard while preserving SSRF checks and current defaults unless maintainers choose a separate default wait-state change.

## Reproduction Assessment

Yes, at source level. Passing `timeoutMs` or `loadState` to browser navigate on current main is enough to see the fields stop before the route/Playwright call, though I did not run the Linux VNC environment live.

## Solution Assessment

Yes for a narrow pass-through and timeout-budget repair. A raw-CDP fallback or changed default wait state may be useful later, but it is broader than needed to fix the demonstrated option-drop and timeout-race path.
