## Summary

Keep open. Current main still drops navigate timeout/load-state controls before the browser route and Playwright call, so the reported 20s timeout race remains a valid browser plugin bug.

## What This Changes

The issue asks OpenClaw to make browser navigate honor load-state/timeout controls and avoid the fixed 20s timeout race on Linux VNC.

## Best Possible Solution

Add a focused browser plugin fix that validates and propagates navigate timeout/load-state controls through the tool, CLI/client, proxy route, and Playwright navigation guard while preserving SSRF checks and current default semantics.

## Reproduction Assessment

Yes. The issue provides concrete Linux VNC steps, and current-main source inspection gives a high-confidence unit-level reproduction: pass `timeoutMs` or `loadState` to browser navigate and observe both are dropped before the route or `page.goto`.

## Solution Assessment

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/47203 in the browser plugin. Current main parses/exposes navigate-adjacent `timeoutMs` and `loadState`, but the navigate tool/client/route path forwards only `url` and `targetId`, the CLI/client use a fixed 20000 ms request timeout, and the Playwright `page.goto` call has no explicit wait-state option. Implement a focused fix that validates and propagates supported navigate options through `extensions/browser/src/browser-tool.ts`, `extensions/browser/src/browser/client-actions-core.ts`, `extensions/browser/src/browser/routes/agent.snapshot.ts`, `extensions/browser/src/browser/pw-tools-core.snapshot.ts`, `extensions/browser/src/browser/pw-session.ts`, and the CLI navigation command as needed. Give the outer request/proxy timeout enough slack over the inner navigation timeout so the wrapper does not win the race. Preserve SSRF pre-navigation and post-navigation checks, targetId replacement resolution, and existing defaults unless a maintainer explicitly chooses a default wait-state change. Do not add a config option or raw-CDP fallback in this PR. Add regression coverage for tool forwarding, client body/request timeout, route forwarding, and the Playwright goto options. Add a single-line `CHANGELOG.md` fix entry crediting @ldfandian.
