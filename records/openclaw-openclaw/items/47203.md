## Summary

Keep open. Current main still drops the navigate timeout/load-state controls before the browser client, node-proxy request, route handler, and Playwright `page.goto`, so the Linux VNC timeout report remains actionable and is not fixed or superseded.

## What This Changes

The issue asks for `browser navigate` to stop timing out behind a fixed 20s budget by honoring timeout/load-state controls and reporting navigation timeouts accurately.

## Best Possible Solution

Thread normalized navigate `timeoutMs` and optional `loadState` through the browser tool, direct client, node proxy, `/navigate` route, and Playwright guard, while preserving SSRF checks and current defaults unless the caller opts in.

Yes, at source level. Supplying `timeoutMs` or `loadState` to browser navigate on current main is enough to see those fields accepted by the broad schema but dropped before the client, route, and Playwright call; this review did not run the reporter's Linux VNC environment live.

## Solution Assessment

Yes for a narrow pass-through and timeout-budget repair. A raw-CDP fallback or default wait-state change may be useful later, but that is broader than needed to fix the demonstrated dropped-controls path.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/47203 on current main. The bug is that browser navigate accepts broad `timeoutMs`/`loadState` inputs but drops them before the direct client, node proxy, `/navigate` route, and Playwright `page.goto`, leaving a fixed 20s outer timeout and Playwright's default `waitUntil: "load"`. Implement a narrow repair: normalize optional navigate timeout/load-state controls, pass them through `extensions/browser/src/browser-tool.ts`, `extensions/browser/src/browser/client-actions-core.ts`, `extensions/browser/src/browser/routes/agent.snapshot.ts`, and the Playwright navigation helper/guard; align outer request/proxy timeouts with the inner navigation budget plus slack; preserve SSRF checks and existing defaults unless an explicit load state is provided. Add regression tests for direct and proxied navigate paths and route/helper forwarding. Coordinate with https://github.com/openclaw/openclaw/pull/75702, which is snapshot-only, and avoid broad raw-CDP fallback/default wait-state changes unless maintainers explicitly choose that larger behavior change. Add a user-facing changelog fix entry.
