## Summary

Keep open. Current main still exposes navigate-adjacent `timeoutMs` and `loadState` inputs, but the `navigate` tool path only forwards `url` and `targetId`, the client/CLI still use a fixed 20000 ms request timeout, and the Playwright navigation call still gets no `waitUntil` option.

## What This Changes

The issue requests that browser navigate honor navigation timeout/load-state controls and avoid misleading 20s timeouts on Linux VNC/Playwright page navigation.

## Best Possible Solution

Add a focused browser plugin fix that validates navigate-specific `timeoutMs` and `loadState`, propagates supported options through the agent tool, CLI/client, proxy dispatch, `/navigate`, and Playwright guarded navigation, and gives outer requests slack while preserving SSRF checks and final URL validation.

## Reproduction Assessment

Yes. The issue provides concrete Linux VNC steps, and current-main static inspection gives a high-confidence unit-level reproduction: pass `timeoutMs` or `loadState` through browser navigate and observe both are dropped before `page.goto`.

## Solution Assessment

Yes for a narrow pass-through and timeout-budget repair. A raw-CDP fallback or default load-state change may be useful later, but it is broader than needed to fix the demonstrated option-drop and timeout-race path.
