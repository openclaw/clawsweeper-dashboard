## Summary

Keep open. Current main and the latest release still drop navigate timeout and load-state controls before the browser route/Playwright call, so the reported 20s browser navigate timeout remains a valid browser plugin bug.

## What This Changes

The issue asks for `browser navigate` to stop timing out on Linux VNC by honoring navigation timeout/load-state controls and distinguishing service reachability from navigation timeout failures.

## Best Possible Solution

Add a focused browser plugin fix that propagates explicit navigate timeout/load-state controls through the tool, client, proxy route, and Playwright navigation guard while preserving SSRF checks and current default semantics.

## Reproduction Assessment

Yes, at source level. Pass `timeoutMs` or `loadState` into browser `navigate` on current main and the code drops those values before `/navigate` and `page.goto`, leaving the fixed 20s outer timeout to race navigation.

## Solution Assessment

Yes for a narrow pass-through and timeout-budget repair. A raw-CDP fallback or default wait-state change may be useful later, but it is broader than needed to fix the demonstrated dropped-control path.
