## Summary

Keep open. Current main still lacks Kimi/Kimi Coding/Moonshot provider-usage support, while the existing plugin API already has the right provider-owned usage-auth and snapshot hooks, so this is actionable in the repository rather than stale, duplicate, or ClawHub-only work.

## What This Changes

Add Kimi/Kimi Coding/Moonshot provider usage and balance reporting to the existing OpenClaw provider usage surfaces.

## Best Possible Solution

Implement provider-owned usage auth and snapshot fetching in the Kimi/Moonshot plugins, extend the generic usage registry only enough for existing status and UI surfaces to find those providers, and document supported credentials without adding unrelated quota UI scope.
