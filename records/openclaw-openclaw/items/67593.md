Keep open. ## Best Possible Solution

Add provider-owned Kimi/Moonshot usage auth and snapshot hooks, extend the usage registry aliases only as needed, document supported credentials, and keep any new command or broad fuel-gauge UX as separate work.

## Solution Assessment

Yes for the provider-usage portion. The narrow maintainable path is provider-owned usage auth and snapshot hooks plus minimal registry aliases; reviving the old broad quota PR or adding a new /quota/UI surface should be avoided unless maintainers explicitly choose that scope.
