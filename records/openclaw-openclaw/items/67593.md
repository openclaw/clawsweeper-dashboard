Keep open. Current main still lacks Kimi, Kimi Coding, and Moonshot in the provider-usage registry, and the bundled Kimi/Moonshot provider plugins do not expose usage auth or usage snapshot hooks; the closed prior implementation at https://github.com/openclaw/openclaw/pull/20343 is useful context but was not merged, and the broader Control UI fuel-gauge request at https://github.com/openclaw/openclaw/issues/75905 does not supersede this provider-specific gap.

## What This Changes

Request to add Kimi/Moonshot usage and balance reporting to OpenClaw provider usage surfaces, including app/status output and possibly a quota command.

## Best Possible Solution

Add provider-owned Kimi/Moonshot usage auth and snapshot hooks, extend the existing usage registry/aliases only as needed, and document supported credentials while keeping any new /quota command or broader fuel-gauge UX as separate maintainer decisions.

## Solution Assessment

Yes for the provider-usage portion. The narrow maintainable fix is to add registry aliases plus Kimi/Moonshot plugin-owned usage hooks; reviving the older broad PR or adding a new /quota command should be avoided unless maintainers explicitly want that surface.
