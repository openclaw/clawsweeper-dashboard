## Summary

Keep this open because current main and the latest release still route structured `delta.content` through the visible text path, and the open closing PR has not merged yet.

## What This Changes

Report asks OpenClaw to stop rendering `[object Object]` in replies, session files, and memory when Mistral thinking returns structured streaming content.

## Best Possible Solution

Normalize structured content blocks at the shared `openai-completions` stream boundary, add regression coverage, and close only after PR #75339 or an equivalent replacement merges.

## Reproduction Assessment

Yes. The reporter gives concrete Mistral thinking-mode steps, and current-main source inspection shows structured `choice.delta.content` can still reach visible text handling without a string guard.

## Solution Assessment
