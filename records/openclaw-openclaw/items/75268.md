## Summary

Keep this open. Current main still routes non-string OpenAI-compatible `delta.content` into visible transcript text before reasoning normalization, and PR #75339 already tracks the intended fix but is still open and unmerged.

## What This Changes

The issue asks OpenClaw to prevent Mistral thinking-mode structured content from appearing as `[object Object]` in replies, session files, and memory.

## Best Possible Solution

Land PR #75339 or an equivalent shared `openai-completions` stream normalization fix with regression coverage for structured `delta.content`, then close this issue after merge.

## Reproduction Assessment

Yes. The report gives concrete Mistral thinking-mode steps, and current-main source inspection shows a high-confidence reproduction path by feeding structured object or array `choice.delta.content` through `processOpenAICompletionsStream`.

## Solution Assessment

Yes. The shared OpenAI-compatible stream parser is the narrow maintainable fix boundary; a Mistral-only wrapper or post-hoc memory cleanup would leave the visible transcript corruption path available elsewhere.
