## Summary

Keep this open. Current main still routes truthy OpenAI-compatible `delta.content` directly into visible text before reasoning normalization, and open PR #75339 already uses closing syntax for this issue, so the right path is to review, land, or replace that PR.

## What This Changes

Mistral thinking-mode replies can leak structured content objects into visible assistant text, session files, and memory as repeated `[object Object]`.

## Best Possible Solution

Normalize structured content blocks at the shared `openai-completions` streaming boundary, route reasoning blocks away from visible text, add regression coverage, and close after #75339 or an equivalent replacement merges.
