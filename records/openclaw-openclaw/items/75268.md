## Summary

Keep this open: current main still routes truthy OpenAI-compatible `delta.content` directly into visible text, while the linked open PR is already the active fix path for the Mistral thinking structured-content leak.

## What This Changes

The issue reports `[object Object]` leaking into replies, session files, and memory when Mistral thinking returns structured content blocks.

## Best Possible Solution

Land #75339 or an equivalent shared OpenAI-compatible stream parser fix that splits structured content blocks into hidden thinking and visible text before transcript or memory persistence.

## Reproduction Assessment

Yes. The report gives concrete versioned steps, and source inspection gives a focused reproduction: feed Mistral-style structured `delta.content` through `processOpenAICompletionsStream` and check that `[object Object]` does not enter visible text.

## Solution Assessment

Yes. Normalizing at the shared OpenAI-compatible streaming boundary is the narrowest maintainable fix; a Mistral-only cleanup after display or memory persistence would leave the live text corruption path in place.
