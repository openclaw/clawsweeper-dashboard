## Summary

Keep this open. Current main still has the non-string `delta.content` stream path, and open PR #75339 is already paired with this issue but has not merged, so the right outcome is to review, land, or replace that PR before closing.

## What This Changes

The issue reports `[object Object]` leaking into assistant replies, session files, and memory when Mistral thinking returns structured content chunks.

## Best Possible Solution

Normalize structured content at the shared OpenAI-compatible completions stream boundary: route `thinking` blocks into hidden thinking output, route `text` blocks into visible assistant text, drop unrecognized objects without string coercion, and add regression coverage before closing after PR #75339 or an equivalent fix lands.

## Reproduction Assessment

Yes. The report provides concrete versioned steps, and source inspection gives a focused unit reproduction: feed a Mistral-style `delta.content` array with typed `thinking`/`text` blocks into the completions stream parser and verify no `[object Object]` reaches visible text.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the shared OpenAI-compatible stream parser rather than memory cleanup or a Mistral-only postprocessor, because the corruption starts in live assistant text before persistence and memory ingestion.
