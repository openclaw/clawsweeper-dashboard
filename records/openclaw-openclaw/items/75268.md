Keep this open because current main still has the structured Mistral thinking-content leak path, and open PR #75339 already uses closing syntax for this issue.

## What This Changes

Mistral thinking responses can put structured content blocks into OpenAI-compatible streaming `delta.content`, which currently reaches visible assistant text and memory as `[object Object]`.

## Best Possible Solution

Land or replace PR #75339 with regression coverage that normalizes structured `delta.content` at the shared OpenAI-compatible streaming boundary.

Yes. The issue has concrete versioned steps, and source inspection gives a focused reproduction: stream a Mistral-style structured `delta.content` array through the OpenAI-compatible completions parser and assert `[object Object]` never enters visible text.

## Solution Assessment

Yes. The shared OpenAI-compatible parser is the narrow maintainable fix boundary; a Mistral-only cleanup or memory scrub would leave the live assistant text corruption path in place.
