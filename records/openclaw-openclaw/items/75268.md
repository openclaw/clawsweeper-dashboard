## Summary

Keep this open. Current main still sends truthy `choice.delta.content` directly to the visible text path before reasoning normalization, and open PR #75339 already uses closing syntax for this issue, so closure should wait for that PR or an equivalent fix to merge.

## What This Changes

The issue asks OpenClaw to stop leaking Mistral thinking-mode structured stream blocks as `[object Object]` into assistant replies, transcripts, and memory.

## Best Possible Solution

Normalize structured `delta.content` at the shared OpenAI-compatible streaming boundary, routing thinking blocks to hidden thinking output and text blocks to visible assistant text, with regression coverage before closing after #75339 or an equivalent fix lands.

## Reproduction Assessment

Yes. The issue provides concrete versioned steps, and a focused current-main reproduction is to feed a Mistral-style structured `delta.content` array through `processOpenAICompletionsStream` and verify `[object Object]` does not enter visible text.

## Solution Assessment

Yes. The shared OpenAI-compatible stream parser is the narrow maintainable fix boundary; a Mistral-only cleanup or memory-only repair would leave the live assistant text corruption path in place.
