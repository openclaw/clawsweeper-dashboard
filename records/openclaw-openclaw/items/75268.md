## Summary

Keep this open. Current main still routes truthy OpenAI-compatible `delta.content` directly into the visible text path before reasoning normalization, and open PR #75339 already uses closing syntax for this issue but remains unmerged.

## What This Changes

The issue asks OpenClaw to stop leaking Mistral thinking-mode structured content objects as `[object Object]` in chat replies, session files, and memory.

## Best Possible Solution

Normalize structured content blocks at the shared `openai-completions` streaming boundary, route reasoning blocks away from visible text, add regression coverage, and close only after PR #75339 or an equivalent replacement merges.

## Reproduction Assessment

Yes. The report gives concrete Mistral thinking-mode steps, and current-main source inspection gives a focused reproduction path: feed a structured `choice.delta.content` object or array through `processOpenAICompletionsStream` and verify it does not enter visible text as `[object Object]`.

## Solution Assessment

Yes. The shared OpenAI-compatible stream parser is the narrow maintainable fix boundary; a Mistral-only wrapper or memory cleanup would leave the live chat/session corruption path available for other structured OpenAI-compatible streams.
