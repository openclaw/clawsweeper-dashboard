## Summary

Keep this open. Current main still sends truthy OpenAI-compatible `delta.content` straight into visible text before reasoning normalization, Mistral uses that transport, and open PR #75339 is the active closing fix path.

## What This Changes

Fix Mistral thinking-mode replies so structured reasoning/text content blocks do not render as `[object Object]` or contaminate session files and memory.

## Best Possible Solution

Normalize structured content blocks at the shared OpenAI-completions stream parser, route thinking blocks away from visible text, add regression tests, then close after #75339 or an equivalent fix merges.

Yes. The report gives concrete Mistral thinking-mode steps, and source inspection gives a focused reproduction path: stream a chunk whose `choice.delta.content` is an array/object through `processOpenAICompletionsStream` and assert visible output never contains `[object Object]`.

## Solution Assessment

Yes. The shared OpenAI-compatible stream parser is the narrow maintainable boundary; a Mistral-only postprocessor or memory cleanup would leave the live assistant text corruption path in place.
