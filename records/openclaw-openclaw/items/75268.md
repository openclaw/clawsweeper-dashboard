## Summary

Keep open. Current main still has the reported OpenAI-compatible stream parsing gap, and the issue is explicitly paired with open PR #75339, so the terminal path is to review and land or replace that PR before closing this issue.

## What This Changes

Prevent Mistral thinking/reasoning structured content chunks from surfacing as `[object Object]` in chat replies, session files, and memory.

## Best Possible Solution

Normalize structured content at the shared OpenAI-compatible stream boundary so typed `thinking` chunks become thinking blocks, typed `text` chunks become visible text, and unknown objects are dropped rather than coerced; then close this issue after PR #75339 or an equivalent tested fix lands.

## Reproduction Assessment

Yes. The report gives concrete versioned Mistral thinking-mode steps, and current-main static inspection gives a focused mocked reproduction path: feed a Mistral-style `delta.content` array into `__testing.processOpenAICompletionsStream` and assert no `[object Object]` reaches output text.

## Solution Assessment

Yes. The shared OpenAI-compatible stream parser is the best fix layer because the corruption starts before session persistence and memory ingestion; cleaning memory after the fact would be too late and provider-specific patching would leave the shared parser unsafe.
