Full review comments:

- **[P2] Document the exact provider alias pattern:** `docs/gateway/secrets-plan-contract.md:93-94`
  - body: This rule says aliases may use alphanumeric, underscore, and hyphen characters, but current main enforces `/^[a-z][a-z0-9_-]{0,63}$/`. As written, the docs would allow aliases such as `1password` or uppercase names that `isSecretsApplyPlan` rejects.
  - confidence: 0.93
- **[P2] Align providerDeletes docs with the actual apply order:** `docs/gateway/secrets-plan-contract.md:113-115`
  - body: This says deletes run after targets and after upserts, but current `applyProviderPlanMutations` deletes aliases before applying upserts, and `projectPlanState` runs provider mutations before target mutations. A plan that includes the same alias in both arrays keeps the upserted provider today, so this wording would publish the wrong contract.
  - confidence: 0.95
- **[P2] Do not claim apply prints the configure summary:** `docs/gateway/secrets-plan-contract.md:120-123`
  - body: The `Plan: targets=..., providerUpserts=..., providerDeletes=...` line is emitted by `openclaw secrets configure`, not by `openclaw secrets apply --from`. The apply command currently reports only dry-run/change results, so this section would tell users to expect output they will not see.
  - confidence: 0.9
- **[P3] Split out the unrelated avatar docs change:** `docs/gateway/config-agents.md:1012`
  - body: This PR is scoped to the secrets plan contract, but this line documents the separate avatar-size issue at https://github.com/openclaw/openclaw/issues/65312. Please remove it from this branch or move it to a dedicated PR.
  - confidence: 0.88
