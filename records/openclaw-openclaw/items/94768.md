## Summary

Current main now forwards trusted turn-source channel, target, account, and thread metadata into plugin approval requests, and Telegram native approval handling filters requests by that channel/account binding. The fix is on main after the merged routing-field PR, but it is not in v2026.6.8.

## What This Changes

The issue reports that plugin approval prompts can be broadcast to unrelated connected channels instead of only the requesting agent/channel.

## Best Possible Solution

Keep the current-main route-field implementation and verify any future report against a build containing the merged fix before reopening this delivery-leak path.

## Reproduction Assessment

Yes. The shipped v2026.6.8 source lacks the turn-source fields at the plugin.approval.request call site, while current main forwards those fields and has gateway regression coverage for routed versus no-route plugin approvals.

## Solution Assessment

Yes. Forwarding trusted runtime route metadata through the existing plugin approval contract is narrower and better-aligned with the channel owner boundary than adding a separate agent-id visibility rule.
