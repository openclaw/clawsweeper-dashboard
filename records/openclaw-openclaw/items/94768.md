## Summary

Keep open: current main still has a plausible plugin-approval routing leak for multi-agent Telegram setups, and the reporter’s latest-release evidence is not fully covered by the older shipped target-account fix.

## What This Changes

Issue reports that a plugin approval prompt from one Telegram-bound agent is delivered through another connected Telegram bot in the same chat.

## Best Possible Solution

Land a focused approval-routing fix that carries trusted turn-source/account identity into plugin approvals and proves multi-agent Telegram delivery reaches only the requesting account, then close this issue and its overlapping satellites from the merged implementation.

## Reproduction Assessment

Yes at source level: current main can emit plugin approval requests without turn-source account fields, while trusted channel approval runtimes receive plugin approval events and rely on channel-side filtering. I did not run the reporter's two-bot Telegram setup live.

## Solution Assessment

Unclear as proposed: the reporter’s unfiltered-broadcast fallback is not the full live-server root because empty targeted broadcasts no-op, but the requested account-scoped delivery outcome is correct and needs a focused routing fix.
