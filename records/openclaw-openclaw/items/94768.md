## Summary

Keep open: current main and v2026.6.8 still omit trusted turn-source routing fields from before_tool_call plugin approval requests, which leaves a plausible Telegram multi-agent/native-approval scoping leak. The work overlaps the open canonical route-forwarding issue and PR, so this should stay in maintainer review rather than spawn another fix lane.

## What This Changes

Plugin approval prompts should be delivered only through the requesting agent/channel instead of leaking to unrelated connected Telegram bots or channels.

## Best Possible Solution

Merge or replace the focused turn-source routing fix so trusted channel, target, account, and thread metadata reaches plugin.approval.request, then verify Telegram native delivery targets only the originating bot/account.

## Reproduction Assessment

Yes at source level: current main omits turn-source route fields from before_tool_call plugin approvals while native approval runtimes can receive plugin approval events. A live two-bot Telegram repro was not run here, so confidence is medium rather than high.

## Solution Assessment

The issue’s generic agent-id visibility suggestion is not the best standalone fix. The cleaner path is to carry trusted turn-source channel/account/thread metadata through the existing plugin approval contract, with Telegram multi-agent regression proof.
