## Summary

Keep open: current main and v2026.6.9 still route renderable active-run final assistant payloads through a stream-fallback-only append helper, and the open linked pull request is the current fix candidate rather than a merged implementation.

## What This Changes

The issue requests that WebChat render each assistant message once when a final chat status event races with already-loaded chat history.

## Best Possible Solution

Land a corrected version of https://github.com/openclaw/openclaw/pull/86646, or an equivalent assistant-tail-aware Control UI event/history reconciliation fix, then close this issue from the merged implementation.

Yes, source-level. A current-main state whose chat history already contains the assistant final can still append the same renderable final payload through handleChatEvent; I did not run a live browser reproduction.
