## Summary

Keep open: current main still enforces short-only plugin approval descriptions, and the requested long-form text plus delivery opt-out would change public plugin, gateway, and channel rendering contracts.

## What This Changes

Request adds optional long-form plugin approval text and a prompt-delivery opt-out for richer channel-specific approval context.

## Best Possible Solution

Define an additive, documented plugin approval body contract that preserves the short fallback while updating validation, SDK types, renderers, channel behavior, docs, and focused tests together.

## Reproduction Assessment

Yes for the current behavior by source inspection: the schema rejects descriptions over 256 characters and the node policy helper slices approval descriptions to 256. I did not run a live channel flow because this is a feature/API request rather than a regression.

## Solution Assessment

Unclear. longDescription is plausible, but deliverPrompt:false changes prompt-delivery ownership and should be decided by maintainers before implementation.
