## Summary

Keep open: current main still has a short-only plugin approval contract, and the requested long-form approval body plus prompt-delivery opt-out would change public gateway, SDK, and channel behavior.

## What This Changes

The issue requests optional long-form plugin approval context and a prompt-delivery opt-out for channels that can carry richer approval prompts.

## Best Possible Solution

Define an additive, documented plugin approval body contract after maintainer API review, preserving the short description fallback while updating gateway validation, SDK types, renderers, channel behavior, docs, and focused tests together.

## Reproduction Assessment

Yes for the current limitation: source inspection shows plugin.approval.request rejects descriptions over 256 characters and node policy approvals truncate to 256. I did not run a live channel flow because this is a new API capability request rather than a regression.

## Solution Assessment

Unclear. longDescription is a plausible additive direction, but deliverPrompt:false changes prompt-delivery ownership and should be designed before implementation is treated as the narrowest maintainable fix.
