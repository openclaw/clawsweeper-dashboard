## Summary

Keep open: current main and the latest release still expose only the short plugin approval description, while the requested long-form field and prompt-delivery opt-out would expand public gateway/plugin and channel-rendering contracts.

## What This Changes

The issue requests an additive long-form plugin approval payload plus an optional per-request prompt delivery opt-out for richer channel-specific approval prompts.

## Best Possible Solution

Define an additive, documented approval body contract that preserves a short fallback while updating gateway validation, SDK types, channel renderers, UI parsing, docs, and focused tests together.

## Reproduction Assessment

Yes. Source inspection shows current main rejects plugin.approval.request descriptions over 256 characters and the node policy helper slices descriptions to 256 before creating the record.

## Solution Assessment

Unclear. longDescription is plausible, but deliverPrompt:false changes prompt-delivery ownership and needs maintainer API/product review before implementation is the narrowest maintainable path.
