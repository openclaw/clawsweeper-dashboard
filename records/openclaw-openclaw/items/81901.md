## Summary

Keep open: current main and the latest release still only accept and render the 256-character plugin approval `description`, while https://github.com/openclaw/openclaw/pull/82240 is an open, unmerged implementation candidate with review blockers and the `deliverPrompt: false` part still needing an API/product decision.

## What This Changes

The issue requests optional long-form plugin approval context and a possible prompt-delivery bypass for richer Telegram, Slack, Discord, Matrix, and similar channel approvals.

## Best Possible Solution

Review or rework the linked PR into a complete additive `longDescription` contract across protocol schema, gateway payload creation, SDK/view models, channel renderers, docs, and live-channel proof; split `deliverPrompt: false` into a separate design item if it is not part of the first landing path.
