## Summary

Keep open: current main still only carries the 256-character plugin approval description, and the requested long-form approval field plus prompt-delivery opt-out would change public gateway/plugin and channel rendering contracts.

## What This Changes

Request adds optional long-form plugin approval context and a prompt-delivery opt-out for rich chat channels while preserving the short description fallback.

## Best Possible Solution

Define an additive, documented plugin approval body contract after maintainer API review, preserving the short description fallback while adapting gateway validation, SDK types, renderers, channel limits, UI parsing, docs, and focused tests together.

## Reproduction Assessment

Yes. Source inspection shows current main rejects plugin.approval.request descriptions over 256 characters and the node policy approval helper slices descriptions to 256, though I did not run a live channel flow because this is a feature request rather than a regression.

## Solution Assessment

Unclear. The longDescription direction is plausible, but deliverPrompt:false changes ownership of prompt delivery and needs maintainer design review before implementation is the narrowest maintainable fix.
