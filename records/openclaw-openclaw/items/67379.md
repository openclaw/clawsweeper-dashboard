## Summary

Keep open. Current main still has the reported QMD scope denial path for spawned subagents, and the remaining fix changes memory-access semantics rather than being a safe cleanup close.

## What This Changes

Ensure QMD memory search scopes spawned subagent sessions intentionally instead of resolving their channel/chatType as unknown and denying by default.

## Best Possible Solution

Keep this open and define a fail-closed subagent QMD scope contract that preserves requester/session metadata, with regression tests for default scope and explicit rawKeyPrefix overrides.

## Reproduction Assessment

Yes. By source inspection, DEFAULT_QMD_SCOPE applied to agent:marketing-coo:subagent:<uuid> strips to subagent:<uuid>, returns no parsed chatType, and falls through to default deny.

## Solution Assessment

Unclear until maintainers choose the scope semantics. Requester-scope inheritance or explicit subagent classification is safer than a blanket agent: raw-prefix allow because QMD recall is a memory-access boundary.
