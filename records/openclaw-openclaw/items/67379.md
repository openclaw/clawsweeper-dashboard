## Summary

Keep open. Current main still drops `agent:<id>:subagent:<uuid>` during QMD scope parsing, so default-deny QMD scope rejects subagent memory search; the remaining decision is the safe memory-scope semantics for delegated sessions.

## What This Changes

The issue asks for QMD memory search to work for `sessions_spawn` subagent sessions instead of resolving their scope to unknown and being denied.

## Best Possible Solution

Define intentional fail-closed subagent QMD scope semantics, likely preserving requester or parent classification, then add focused scope and manager regression coverage.

## Reproduction Assessment

Yes, at source level. `sessions_spawn` emits `agent:<id>:subagent:<uuid>`, QMD normalization strips that to `subagent:*` and returns no scope, and the default-deny direct rule then rejects the search.

## Solution Assessment

Unclear until maintainers choose the memory-access semantics. Preserving requester or parent scope, or adding explicit subagent classification, is safer than a broad raw `agent:` allow rule.
