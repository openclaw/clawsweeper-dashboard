## Summary

Keep open. Current main still has the reported QMD scope denial path for spawned subagents, and the related #70297 PR was closed unmerged and covered a different default-scope concern.

## What This Changes

The issue asks for spawned subagent QMD memory searches to resolve an intentional channel/chatType scope instead of being denied as unknown.

## Best Possible Solution

Keep this open and implement a fail-closed subagent QMD scope contract that preserves requester intent, with regression tests and updated QMD scope docs.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: evaluate the default QMD scope for agent:marketing-coo:subagent:<uuid>, and normalization returns no parsed chatType, causing isQmdScopeAllowed() to fall through to default deny.

## Solution Assessment

Unclear until maintainers choose the exact subagent scope semantics. A narrow requester-scope inheritance or explicit subagent classification is safer than a blanket agent: raw-prefix allow because QMD recall is a memory-access boundary.
