## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Validate mission IDs before resolving mission paths:** `src/claw/service.ts:1717-1718`
  - body: The new service resolves RPC-supplied `missionId` values with `path.join(resolveMissionsRoot(...), missionId)` and then reads or writes mission files. Because the Gateway read/write handlers accept arbitrary non-empty strings, `..` or path separators can escape the missions root if a matching `mission-state.json` exists, so mission IDs need format validation or normalized containment before any filesystem access.
  - confidence: 0.88
- **[P1] Resolve mission state ownership before merging:** `src/claw/service.ts:1774`
  - body: This writes canonical mission state to `mission-state.json` and mirrors runtime state into workspace Markdown/JSONL files. Root policy says OpenClaw-owned runtime state should default to SQLite rather than JSON/JSONL sidecars, so the PR needs either a SQLite-backed model or an explicit maintainer decision that these files are product artifacts with migration/upgrade ownership.
  - confidence: 0.9
- **[P1] Preserve HTTP/1-only pinned dispatchers:** `src/infra/net/ssrf.ts:430-438`
  - body: `createPinnedDispatcher` now constructs raw Undici dispatchers, bypassing the current `createHttp1*` helpers that enforce `allowH2: false` for guarded network paths. That can re-enable HTTP/2 on SSRF-guarded requests, so keep the HTTP/1 helper path while adding pinned lookup behavior.
  - confidence: 0.87
- **[P2] Skip malformed audit lines when reading audit logs:** `src/claw/service.ts:3842-3847`
  - body: `getAudit` parses every JSONL row with a bare `JSON.parse`, so one truncated line from an interrupted append makes the entire audit feed fail. Catch parse errors per line and skip or quarantine malformed rows so operators can still inspect the rest of the mission audit.
  - confidence: 0.82
- **[P2] Gate Claw dashboard reloads to the Claw tab:** `ui/src/ui/app-gateway.ts:424-426`
  - body: Every `claw.*` event triggers a full dashboard reload even when the user is on another tab. Mission execution can emit several events, so connected UI clients will fan out background `claw.*` RPCs unrelated to the active view; gate this to the Claw tab or apply incremental cached updates.
  - confidence: 0.78
