## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Move mission state into SQLite:** `src/claw/service.ts:68-70`
  - body: The branch introduces `mission-state.json`, `control-state.json`, and `AUDIT_LOG.jsonl` as durable Claw runtime state. OpenClaw-owned runtime state belongs in SQLite by default, so this would create a second persistent state model without migration or upgrade proof.
  - confidence: 0.9
- **[P1] Reject unsafe mission IDs before resolving paths:** `src/claw/service.ts:1717-1722`
  - body: `missionId` is accepted from RPC callers and joined directly into mission paths. Reject separators/traversal or enforce normalized containment before reading or writing mission files, otherwise a crafted ID can escape the missions root if a matching state file exists.
  - confidence: 0.9
- **[P1] Preserve HTTP/1-only pinned SSRF dispatchers:** `src/infra/net/ssrf.ts:426-431`
  - body: The pinned dispatcher path now constructs raw Undici dispatchers instead of the current `createHttp1*` helpers, dropping the guarded-networking helper path that enforces HTTP/1-only behavior for SSRF-pinned requests.
  - confidence: 0.86
- **[P1] Reload live mission state after agent calls:** `src/claw/service.ts:2873`
  - body: Helper, runner, and verifier cycles await long-running agent work and then mutate the stale in-memory mission object. If an operator pauses or cancels during the call, these writes can undo the operator action, so reload and gate on current mission status before persisting outcomes.
  - confidence: 0.85
- **[P2] Return after missionId validation before reading limit:** `src/gateway/server-methods/claw.ts:372-373`
  - body: `claw.audit.get` still calls `readOptionalPositiveNumber` even after `readRequiredString` may already have responded for a missing missionId. If both `missionId` and `limit` are invalid, the handler can respond twice for one request.
  - confidence: 0.82
- **[P2] Skip malformed audit lines when building the feed:** `src/claw/service.ts:3846-3847`
  - body: `getAudit` parses every JSONL line with bare `JSON.parse`, so one truncated audit record makes the whole audit feed fail. Skip or quarantine malformed lines so operators can still inspect the rest of the mission history.
  - confidence: 0.78
