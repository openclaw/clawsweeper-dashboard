## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve HTTP/1-only dispatchers in the pinned SSRF path:** `src/infra/net/ssrf.ts:426-447`
  - body: This replaces the guarded `createHttp1*` dispatcher helpers with raw Undici dispatchers, bypassing the `allowH2: false` contract used by current main for pinned SSRF-guarded fetches. Please keep this path on `createHttp1Agent`, `createHttp1EnvHttpProxyAgent`, and `createHttp1ProxyAgent` so HTTP/2 cannot re-enter the guarded dispatcher path.
  - confidence: 0.93
- **[P1] Let mission cycles continue when recovery fails:** `src/claw/runtime.ts:75-82`
  - body: `recoverInterruptedMissions()` runs before `runMissionCycles()` inside a single try block. If one bad persisted mission throws during recovery, this iteration exits before healthy queued or active missions run, and the next wake retries the same recovery failure again. Isolate recovery failures or mark recovery attempted so normal cycles can still progress.
  - confidence: 0.9
- **[P1] Reload mission status after long-running agent calls:** `src/claw/service.ts:2954-3035`
  - body: After `runMissionAgent()` returns, the runner/helper/verifier paths immediately mutate the stale in-memory mission record. If an operator pauses or cancels the mission while the agent call is in flight, these writes can overwrite that control action and resume or complete a mission unexpectedly. Reload and validate the live mission status before applying each post-run outcome.
  - confidence: 0.88
- **[P2] Only refresh the Claw dashboard when it is visible:** `ui/src/ui/app-gateway.ts:424-426`
  - body: Every `claw.*` event reloads the full dashboard for every connected UI client, even when the user is on another tab. Runtime cycles can emit several events per mission, so this fans out avoidable gateway reads and UI work in the background. Gate this reload to the Claw tab or update local state from the event payload.
  - confidence: 0.82
- **[P2] Skip malformed audit lines when reading the audit feed:** `src/claw/service.ts:3846-3848`
  - body: `getAudit()` parses every JSONL line with a bare `JSON.parse`, so one truncated audit append makes the entire audit request fail. Since this is an append-only runtime log, skip or mark malformed records instead of hiding all valid audit history for the mission.
  - confidence: 0.84
