## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Move mission persistence into SQLite:** `src/claw/service.ts:67-70`
  - body: This introduces mission/control/audit runtime state as JSON and JSONL sidecars under `missions/`. OpenClaw's current runtime-state contract is SQLite-first, with legacy files limited to migration/debug/export paths, so this would add a second durable state system for mission lifecycle, recovery, audit, and upgrades.
  - confidence: 0.93
- **[P1] Validate mission IDs before resolving paths:** `src/claw/service.ts:1717-1722`
  - body: Gateway methods pass user-provided `missionId` values into `resolveMissionDir`, which joins the ID directly under the missions root. A value containing `..` or path separators can escape that root and target arbitrary `mission-state.json` paths, so mission IDs need a generated-format or normalized-containment check before any load or write.
  - confidence: 0.91
- **[P1] Preserve the HTTP/1 pinned dispatcher helpers:** `src/infra/net/ssrf.ts:430-432`
  - body: The PR now constructs Undici dispatchers directly instead of using the current `createHttp1*` helpers. That drops the explicit guarded-networking wrapper that sets `allowH2: false` and carries proxy TLS/client-factory behavior, so pinned SSRF fetches can lose the policy the current tests assert.
  - confidence: 0.9
- **[P1] Reload mission state after verifier calls return:** `src/claw/service.ts:3092-3094`
  - body: After the verifier agent returns, this code immediately marks the mission done or blocked using the stale in-memory state. If an operator paused or cancelled the mission while the verifier was running, this can overwrite the operator action; reload and revalidate the current mission status before applying verifier outcomes.
  - confidence: 0.86
- **[P2] Reject unsupported decision action pairs:** `src/claw/service.ts:3764-3769`
  - body: Unsupported decision/action combinations fall through and are marked resolved. For example, an invalid action for a recovery decision can clear the pending decision without running the recovery transition, so replies should enforce the allowed actions per decision kind and return an error otherwise.
  - confidence: 0.86
- **[P2] Skip malformed audit log lines:** `src/claw/service.ts:3841-3847`
  - body: `getAudit` parses every JSONL record with a bare `JSON.parse`, so one truncated audit write makes the whole audit feed fail. Audit history should skip or surface the bad record without hiding every valid entry for that mission.
  - confidence: 0.84
- **[P2] Gate Claw dashboard reloads to the Claw view:** `ui/src/ui/app-gateway.ts:424-426`
  - body: Every `claw.*` event reloads the full Claw dashboard for every connected Control UI client, even when the user is on an unrelated tab. Runtime cycles can emit several Claw events, so this fans out avoidable Gateway calls and background UI churn.
  - confidence: 0.82
