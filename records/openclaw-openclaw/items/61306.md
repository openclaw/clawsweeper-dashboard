## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve HTTP/1-only pinned dispatchers:** `src/infra/net/ssrf.ts:430-432`
  - body: `createPinnedDispatcher` now constructs Undici dispatchers directly, bypassing the current `createHttp1*` helpers that enforce `allowH2: false` for guarded SSRF paths. That can re-enable HTTP/2 on pinned guarded requests and weaken the assumptions those SSRF guards rely on.
  - confidence: 0.91
- **[P1] Isolate recovery failures before mission cycles:** `src/claw/runtime.ts:76-82`
  - body: `recoverInterruptedMissions()` runs before `runMissionCycles()` inside one outer try/catch. If one bad mission makes recovery throw, no queued or active mission cycles run and `state.recovered` remains false, so later wakes can keep retrying the same blocker.
  - confidence: 0.88
- **[P1] Re-check live mission state after agent runs:** `src/claw/service.ts:2954-2960`
  - body: Runner, helper, and verifier cycles mutate the in-memory mission after `runMissionAgent()` returns without reloading current status. If an operator pauses or cancels during the long-running agent call, these writes can restore the mission to running/verifying or mark it done over the operator action.
  - confidence: 0.86
- **[P2] Avoid global Claw dashboard reloads off-tab:** `ui/src/ui/app-gateway.ts:424-426`
  - body: Every `claw.*` event reloads the full dashboard even when the user is not on the Claw tab. Runtime mission events can therefore fan out list/get/audit/artifact RPCs from unrelated UI tabs, adding avoidable gateway load during autonomous execution.
  - confidence: 0.82
- **[P2] Skip malformed audit JSONL records:** `src/claw/service.ts:3847`
  - body: `getAudit()` parses each audit line with bare `JSON.parse`, so one truncated or corrupt append makes the whole audit feed fail. Operators should still be able to inspect the valid audit history while the bad record is skipped or surfaced separately.
  - confidence: 0.83
