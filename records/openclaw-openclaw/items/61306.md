## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Validate mission IDs before resolving mission paths:** `src/claw/service.ts:1717-1718`
  - body: Gateway RPCs accept missionId strings that reach resolveMissionDir, which joins the value directly under the missions root. A value containing path separators can escape the intended mission tree and make subsequent state, audit, or artifact reads target files outside the mission directory if those paths exist.
  - confidence: 0.91
- **[P1] Preserve HTTP/1-only pinned dispatchers:** `src/infra/net/ssrf.ts:430-447`
  - body: Current main routes pinned SSRF dispatchers through createHttp1* helpers that disable HTTP/2. This diff constructs Undici dispatchers directly, so guarded pinned lookups can negotiate H2 again and weaken the SSRF guard contract added for Undici 8 compatibility.
  - confidence: 0.92
- **[P1] Move mission runtime state to approved storage:** `src/claw/service.ts:1774`
  - body: The PR persists mission and control state as JSON files plus JSONL audit sidecars. Repository policy and the current TaskFlow sibling use SQLite for OpenClaw-owned runtime state, so this creates a second state store without migration, doctor, or product-artifact proof.
  - confidence: 0.9
- **[P1] Reload mission state after long agent runs:** `src/claw/service.ts:2922-2956`
  - body: Runner and verifier cycles await runMissionAgent and then mutate the stale in-memory state. If an operator pauses or cancels while the agent call is in flight, the returned cycle can overwrite that control action and resume or complete the mission unexpectedly.
  - confidence: 0.87
- **[P2] Gate preflight reruns by allowed mission status:** `src/claw/service.ts:3780-3798`
  - body: rerunPreflight updates any loaded mission and moves it back to awaiting_approval or awaiting_setup. Without checking that the mission is actually in a preflight-blocked state, a rerun request can rewrite active or terminal mission state.
  - confidence: 0.84
- **[P2] Skip malformed audit records:** `src/claw/service.ts:3841-3847`
  - body: getAudit parses every JSONL line with a bare JSON.parse, so one truncated or corrupt audit line makes the whole audit RPC fail instead of returning the remaining valid mission audit entries.
  - confidence: 0.82
- **[P2] Return after the first invalid audit parameter:** `src/gateway/server-methods/claw.ts:372-374`
  - body: claw.audit.get reads both missionId and limit before returning on invalid input, and both helpers can call respond on failure. A request with both an invalid missionId and invalid limit can therefore produce two responses for one RPC.
  - confidence: 0.8
