## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Preserve the HTTP/1-only pinned dispatcher wrappers:** `src/infra/net/ssrf.ts:430-447`
  - body: `createPinnedDispatcher` now instantiates Undici agents directly, bypassing the current `createHttp1*` wrappers that add `allowH2: false`, proxy TLS, IP-safe proxy client factories, and timeout/connect policy. For SSRF-guarded pinned dispatchers this can re-enable behavior the guard tests expect to stay disabled, so use the shared wrappers instead of raw constructors.
  - confidence: 0.9
- **[P1] Use effective browser plugin activation for preflight:** `src/claw/service.ts:600-602`
  - body: The preflight check only treats the browser plugin as enabled when config explicitly sets `plugins.entries.browser.enabled === true`. The bundled browser plugin is enabled by default, so normal configs with no explicit entry will be blocked as "browser plugin is not enabled" for browser-required missions. Check effective plugin activation or treat unset as enabled unless explicitly disabled.
  - confidence: 0.91
- **[P2] Return after missionId validation fails before reading limit:** `src/gateway/server-methods/claw.ts:372-374`
  - body: In `claw.audit.get`, `readRequiredString` can already call `respond(false)`, but the handler still calls `readOptionalPositiveNumber` before the guard. A request with missing `missionId` and invalid `limit` therefore sends two responses on one RPC; match sibling handlers by returning immediately after `missionId` validation.
  - confidence: 0.93
- **[P2] Gate Claw dashboard reloads to the Claw tab:** `ui/src/ui/app-gateway.ts:424-426`
  - body: This branch reloads the full Claw dashboard for every `claw.*` event even when the user is on another tab. Mission cycles can emit many events, so every connected UI client can fan out background list/get/audit/artifact requests; gate the reload to the Claw tab or update a small cached signal instead.
  - confidence: 0.86
- **[P2] Reject unsupported decision action-kind pairs:** `src/claw/service.ts:3764-3769`
  - body: After handling the supported approval, recovery-continue, cancel/reject, and pause paths, the default branch marks any remaining action as resolved. For example, `approve` on a `recovery_uncertain` decision resolves the gate without resuming recovery, leaving mission state inconsistent with the operator workflow. Return an error for invalid action/kind pairs.
  - confidence: 0.87
