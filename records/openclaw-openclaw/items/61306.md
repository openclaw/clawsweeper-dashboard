## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Validate mission IDs before resolving paths:** `src/claw/service.ts:1717-1718`
  - body: RPC-provided mission IDs are joined directly into mission paths. A value with `..` or path separators can escape the missions root and make later load/write paths target files outside the intended mission directory, so mission IDs need a generated-format validator or normalized containment check before path resolution.
  - confidence: 0.91
- **[P1] Move mission state out of workspace sidecars:** `src/claw/service.ts:1774`
  - body: The runtime writes mission/control/audit state to JSON/JSONL sidecars under the workspace. OpenClaw-owned runtime state is now expected to live in SQLite-backed stores with migration/doctor ownership, so shipping this would create a second persistence contract and upgrade debt.
  - confidence: 0.89
- **[P1] Preserve HTTP/1-only pinned dispatchers:** `src/infra/net/ssrf.ts:430`
  - body: This replaces current main’s `createHttp1*` dispatcher helpers with direct Undici constructors. That drops the `allowH2: false` enforcement used by guarded pinned dispatchers, reopening the HTTP/2 compatibility/security issue already fixed on main.
  - confidence: 0.9
- **[P1] Reload mission state after long agent calls:** `src/claw/service.ts:2954-2956`
  - body: The runner path applies parsed output to the original in-memory state after `runMissionAgent` returns. If an operator pauses or cancels the mission while the agent call is in flight, this can persist over the operator action; the helper and verifier paths need the same live-state guard.
  - confidence: 0.86
- **[P2] Route browser readiness through a plugin seam:** `src/claw/service.ts:5-10`
  - body: The new core Claw service imports the browser plugin runtime directly from `extensions/browser`. Core should depend on SDK/capability or manifest-backed seams here; otherwise the feature hard-couples core runtime to a bundled plugin implementation and breaks the extension boundary.
  - confidence: 0.82
