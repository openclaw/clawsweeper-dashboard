Full review comments:

- **[P1] Move mission state into approved storage:** `src/claw/service.ts:67-70`
  - body: The branch introduces `mission-state.json`, `control-state.json`, and `AUDIT_LOG.jsonl` as durable Claw runtime state. OpenClaw-owned runtime state belongs in SQLite unless maintainers approve a named product artifact and migration plan, so this would create a competing persistent state model.
  - confidence: 0.93
- **[P1] Reject unsafe mission IDs before resolving paths:** `src/claw/service.ts:1717-1722`
  - body: Gateway callers provide `missionId` as an arbitrary non-empty string, and the service joins it directly under the missions root before reads and writes. Values containing `..` or path separators need generated-ID validation or normalized containment before any filesystem access.
  - confidence: 0.9
- **[P1] Preserve HTTP/1-only pinned SSRF dispatchers:** `src/infra/net/ssrf.ts:426-432`
  - body: This path now constructs Undici dispatchers directly instead of using the `createHttp1*` helpers that apply OpenClaw's HTTP/1-only guarded-networking policy. That can re-enable unsupported guarded-networking behavior and weaken pinned-lookup assumptions.
  - confidence: 0.88
- **[P1] Reload live mission state after agent calls:** `src/claw/service.ts:3029-3035`
  - body: After `runMissionAgent(...)` returns, the mission-cycle paths persist transitions using stale in-memory state. If an operator pauses or cancels during the agent call, this can overwrite that control action; the runner, verifier, and helper paths need a fresh status guard before writing.
  - confidence: 0.87
- **[P2] Return before parsing optional audit limits:** `src/gateway/server-methods/claw.ts:372-374`
  - body: `claw.audit.get` still reads `limit` after `readRequiredString` may already have responded for a missing `missionId`. If both inputs are invalid, the handler can call `respond` twice for one request; guard after the required field first like the sibling handlers.
  - confidence: 0.86
- **[P2] Reject invalid decision action-kind pairs:** `src/claw/service.ts:3764-3769`
  - body: Unsupported action/kind combinations fall through and mark the decision resolved without performing the matching state transition. Enforce allowed actions per decision kind so a wrong action cannot clear a recovery or approval gate into an inconsistent mission state.
  - confidence: 0.84
- **[P2] Keep reconcile no-op signatures symmetric:** `src/claw/service.ts:2183-2195`
  - body: `beforeSignature` omits `resumeTargetStatus` and `recoveryTargetStatus`, while `afterSignature` includes them. That makes unchanged non-terminal missions look dirty and rewrites mission state/docs on dashboard refreshes, adding avoidable disk churn.
  - confidence: 0.82
- **[P2] Skip malformed audit lines when reading audit feeds:** `src/claw/service.ts:3846-3847`
  - body: `getAudit` parses every JSONL line with a bare `JSON.parse`, so one truncated audit append makes the entire audit request fail. Skip or quarantine malformed lines so operators do not lose the whole audit feed after a partial write.
  - confidence: 0.78
