## Review Findings

Overall confidence: 0.87

Full review comments:

- **[P1] Validate mission IDs before resolving paths:** `src/claw/service.ts:1717-1722`
  - body: Gateway handlers accept `missionId` as any non-empty string, then this helper joins it directly under the missions root. A value containing `../` or path separators can escape the mission directory and make mission state, control, or audit reads/writes target unintended files, so mission IDs need a strict format and containment check before path resolution.
  - confidence: 0.91
- **[P1] Move mission runtime state into SQLite:** `docs/claw/02-mission-engine-spec.md:32-53`
  - body: The new mission engine defines OpenClaw-owned mission state, control state, and audit logs as workspace JSON/JSONL sidecars. That creates a new durable runtime state model outside the shared SQLite store required for OpenClaw state, making upgrades and doctor migrations harder before this becomes a user contract.
  - confidence: 0.88
- **[P1] Reload live mission state after agent calls:** `src/claw/service.ts:2922-2955`
  - body: Runner cycles await a long `runMissionAgent(...)` call and then write transitions using the stale in-memory mission state. If an operator pauses or cancels the mission while the agent is running, this path can overwrite that control action; the helper and verifier cycles need the same post-await live-status guard before persisting outcomes.
  - confidence: 0.86
- **[P1] Preserve HTTP/1-only pinned dispatchers:** `src/infra/net/ssrf.ts:426-453`
  - body: This branch constructs Undici dispatchers directly in the SSRF-pinned path instead of using the current HTTP/1-only helper constructors from main. That can re-enable HTTP/2 for guarded requests and weaken the pinned lookup assumptions, so the PR needs to rebase or keep the `createHttp1*` dispatcher path.
  - confidence: 0.84
- **[P2] Reject invalid decision action pairs:** `src/claw/service.ts:3764-3776`
  - body: Unsupported action/kind combinations fall through to the default branch and are marked resolved. For example, approving a recovery decision can clear the gate without running the recovery continuation transition, leaving mission state inconsistent with the operator decision.
  - confidence: 0.82
