## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve HTTP/1-only pinned dispatchers:** `src/infra/net/ssrf.ts:430-431`
  - body: `createPinnedDispatcher` now constructs raw Undici dispatchers instead of the `createHttp1*` helpers that force `allowH2: false` for guarded networking. This can re-enable HTTP/2 on pinned SSRF fetch paths where the guarded lookup overrides are not the supported path.
  - confidence: 0.88
- **[P1] Isolate recovery failures before mission cycles:** `src/claw/runtime.ts:76-80`
  - body: `drainClawRuntimeQueue` awaits recovery before normal mission cycles and only marks recovery complete after it succeeds. If one mission throws during recovery, the catch skips queued or active mission cycles and the next wake retries the same failing recovery path.
  - confidence: 0.86
- **[P1] Re-check live mission status after runner returns:** `src/claw/service.ts:2956-2960`
  - body: The runner mutates and persists the pre-run `state` after a long agent call. If an operator pauses or cancels while that call is in flight, these writes can overwrite the operator action; reload and validate current status before persisting outcomes.
  - confidence: 0.83
- **[P2] Reload the Claw dashboard only when active:** `ui/src/ui/app-gateway.ts:424-425`
  - body: This handler reloads the full Claw dashboard for every `claw.*` event on every connected UI, even when the user is on another tab. Runtime cycles can emit several events, causing avoidable background mission, audit, and artifact RPC fanout.
  - confidence: 0.78
- **[P2] Skip malformed audit log lines:** `src/claw/service.ts:3847`
  - body: `getAudit` parses every JSONL line with a bare `JSON.parse`, so one truncated audit append makes the whole audit feed fail. Skip or quarantine malformed records so operators can inspect the remaining mission audit history.
  - confidence: 0.76
