## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Move mission persistence into SQLite:** `src/claw/service.ts:67-70`
  - body: This introduces `mission-state.json`, `control-state.json`, and `AUDIT_LOG.jsonl` as durable runtime state. OpenClaw-owned runtime/session state is supposed to live in SQLite or an explicitly designed database owner, so landing these sidecars would create migration debt and a second state model for missions.
  - confidence: 0.9
- **[P1] Validate mission IDs before resolving paths:** `src/claw/service.ts:1717-1722`
  - body: `missionId` comes from Gateway RPC as any non-empty string, then this helper appends it under the missions root with `path.join`. Reject path separators and `..` or enforce normalized containment before reading/writing mission state so a crafted ID cannot escape the mission directory.
  - confidence: 0.89
- **[P1] Preserve the HTTP/1 pinned dispatcher helpers:** `src/infra/net/ssrf.ts:430-432`
  - body: The branch bypasses the current `createHttp1*` helper path for pinned SSRF dispatchers. Those helpers force `allowH2: false`; constructing Undici dispatchers directly can re-enable HTTP/2 on guarded requests and weaken the pinned-lookup assumptions.
  - confidence: 0.86
- **[P2] Use effective browser plugin activation:** `src/claw/service.ts:600-602`
  - body: The browser plugin manifest is enabled by default, but this check only accepts `plugins.entries.browser.enabled === true`. Default installs that do not spell out the redundant config entry will be treated as browser-disabled and can block browser-required missions incorrectly.
  - confidence: 0.84
- **[P2] Skip malformed audit log lines:** `src/claw/service.ts:3846-3848`
  - body: `getAudit` parses every JSONL line with a bare `JSON.parse`, so one truncated or corrupt audit append makes the whole audit request fail. For an operator audit feed, skip or surface only the bad record instead of losing every valid entry.
  - confidence: 0.8
