Full review comments:

- **[P1] Move mission persistence into SQLite:** `src/claw/service.ts:67-70`
  - body: This introduces durable mission, control, and audit state as workspace JSON/JSONL sidecars. OpenClaw runtime state is expected to live in the shared/per-agent SQLite stores, so merging this as-is creates migration and session-state debt before the feature has shipped.
  - confidence: 0.91
- **[P1] Validate mission IDs before resolving paths:** `src/claw/service.ts:1717-1722`
  - body: RPC-provided mission IDs are joined directly into the missions root, so values containing path separators or `..` can escape the intended mission directory and target arbitrary mission-state paths. Validate against the generated mission-id format or enforce normalized containment before any path join.
  - confidence: 0.93
- **[P1] Preserve the HTTP/1 pinned dispatcher helpers:** `src/infra/net/ssrf.ts:430-432`
  - body: The PR replaces the current `createHttp1*` helpers with direct Undici constructors on the pinned SSRF path, dropping the helper-enforced `allowH2: false` policy. Keep the existing helper path so guarded networking retains the HTTP/1-only assumptions.
  - confidence: 0.9
- **[P2] Skip malformed audit log lines:** `src/claw/service.ts:3847`
  - body: `getAudit` parses every JSONL line with a bare `JSON.parse`, so a single truncated audit record makes the entire audit feed unavailable. Skip or quarantine malformed records so operators can still inspect the valid audit history.
  - confidence: 0.82
