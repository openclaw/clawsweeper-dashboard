Full review comments:

- **[P1] Preserve HTTP/1-only dispatchers in the pinned path:** `src/infra/net/ssrf.ts:430-447`
  - body: `createPinnedDispatcher` now constructs raw Undici dispatchers, bypassing the current `createHttp1*` helpers that enforce `allowH2: false` for guarded networking. Rebase this path through the helpers so SSRF-pinned requests do not regain the HTTP/2 behavior current main deliberately disabled.
  - confidence: 0.92
- **[P1] Isolate recovery failures before running normal cycles:** `src/claw/runtime.ts:75-82`
  - body: `recoverInterruptedMissions()` runs before `runMissionCycles()` and any exception leaves `state.recovered` false, so one bad recovery record can repeatedly skip all queued or active mission progress. Catch recovery failures per mission or otherwise let normal cycles run after logging the bad recovery item.
  - confidence: 0.88
- **[P1] Re-check runner status before writing results:** `src/claw/service.ts:2956-2960`
  - body: After `runMissionAgent(...)` returns, the runner path writes transitions using the stale in-memory mission object. If an operator paused or cancelled the mission during the agent call, this can push the mission back to running/verifying/blocked and undo the operator action; reload and validate current status before applying the result.
  - confidence: 0.9
- **[P1] Re-check verifier status before completing missions:** `src/claw/service.ts:3083-3093`
  - body: The verifier path can mark a mission done, blocked, or running immediately after a long verifier call without checking whether the operator paused or cancelled it while the call was in flight. Add the same fresh status/revision guard before applying verifier outcomes.
  - confidence: 0.89
- **[P1] Re-check helper status before reopening execution:** `src/claw/service.ts:2871-2874`
  - body: The helper cycle also mutates state after a long agent call and can move a mission back to running or blocked even if an operator control changed the mission during that call. Reload the mission and bail on paused/cancelled/terminal states before writing helper results.
  - confidence: 0.86
- **[P2] Keep verifier failures in verifier recovery:** `src/claw/service.ts:3179`
  - body: `executeMissionCycle` catches errors from runner and verifier paths but always moves non-terminal failures back to `running`, so a transient verifier error can resume runner mutations instead of retrying or preserving verification state. Preserve the failing phase when deciding the recovery transition.
  - confidence: 0.82
- **[P2] Avoid global dashboard reloads for every Claw event:** `ui/src/ui/app-gateway.ts:424-426`
  - body: Every `claw.*` event reloads the full Claw dashboard even when the user is on another tab, so active mission loops can fan out background RPC/audit/artifact refreshes across all connected UI clients. Gate the reload to the Claw tab or apply the event payload incrementally.
  - confidence: 0.78
