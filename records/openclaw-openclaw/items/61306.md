Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Move mission persistence into SQLite:** `src/claw/service.ts:67-70`
  - body: This adds durable mission, control, and audit runtime state as workspace JSON/JSONL sidecars. OpenClaw runtime state is expected to live in the shared or per-agent SQLite stores, so merging this as-is creates migration and session-state debt before the feature has shipped.
  - confidence: 0.91
- **[P1] Validate mission IDs before resolving paths:** `src/claw/service.ts:1717-1722`
  - body: RPC-provided mission IDs are joined directly into the missions root, so values containing path separators or `..` can escape the intended mission directory and target arbitrary mission-state paths. Validate against the generated mission-id format or enforce normalized containment before any filesystem access.
  - confidence: 0.93
- **[P1] Preserve the HTTP/1 pinned dispatcher helpers:** `src/infra/net/ssrf.ts:430-432`
  - body: The PR replaces the current `createHttp1*` helpers with direct Undici constructors on the pinned SSRF path, dropping the established helper layer that carries guarded transport options. Keep the existing helper path so guarded networking retains the HTTP/1-only assumptions.
  - confidence: 0.9
- **[P2] Use effective browser plugin activation:** `src/claw/service.ts:600-602`
  - body: The Claw preflight check treats the browser plugin as available only when `plugins.entries.browser.enabled === true`, but the bundled browser plugin is enabled by default. In normal configs with no explicit browser entry, browser-required missions can be blocked as if the plugin were disabled.
  - confidence: 0.86
- **[P2] Skip malformed audit log lines:** `src/claw/service.ts:3847`
  - body: `getAudit` parses every JSONL line with a bare `JSON.parse`, so a single truncated audit record makes the whole audit feed unavailable. Skip or quarantine malformed records so operators can still inspect valid audit history.
  - confidence: 0.82
