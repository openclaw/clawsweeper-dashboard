Full review comments:

- **[P1] Preserve HTTP/1-only pinned dispatchers:** `src/infra/net/ssrf.ts:430-447`
  - body: This SSRF-guarded path now constructs raw Undici dispatchers instead of the createHttp1* helpers, dropping the shared HTTP/1-only, proxy TLS, timeout, and IP-safe proxy policy. Use the existing helpers so pinned guarded requests keep the same network boundary as current main.
  - confidence: 0.9
- **[P1] Use effective browser plugin activation:** `src/claw/service.ts:600-602`
  - body: This only treats an explicit plugins.entries.browser.enabled === true as enabled, but the bundled browser plugin is enabledByDefault and current helpers treat unset config as enabled. Default configs can therefore block browser-required Claw missions until users add redundant config.
  - confidence: 0.9
- **[P1] Revalidate mission state after runner returns:** `src/claw/service.ts:2956-2960`
  - body: After runMissionAgent awaits, this mutates stale in-memory state without checking whether an operator paused or canceled the mission during the run. Reload and guard live status before applying runner output, and apply the same invariant to verifier/helper result paths.
  - confidence: 0.84
- **[P2] Return after invalid audit mission id:** `src/gateway/server-methods/claw.ts:372-374`
  - body: readRequiredString and readOptionalPositiveNumber both call respond on invalid input, so a missing missionId plus invalid limit sends two responses for one claw.audit.get request. Return after the required-field failure before parsing optional fields.
  - confidence: 0.86
