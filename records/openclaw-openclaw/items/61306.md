Full review comments:

- **[P1] Preserve HTTP/1-only pinned dispatchers:** `src/infra/net/ssrf.ts:426-447`
  - body: This path now constructs Undici dispatchers directly instead of using the `createHttp1*` helpers. Current main centralizes `allowH2: false`, proxy TLS, IP-safe proxy pools, and timeout propagation in those helpers, and the pinned-dispatcher tests assert that contract; bypassing them can re-enable HTTP/2 or drop guarded proxy behavior on SSRF-pinned requests.
  - confidence: 0.9
- **[P1] Resolve mission state ownership before merging:** `docs/claw/02-mission-engine-spec.md:32-52`
  - body: The Claw engine is specified and implemented as workspace mission folders with Markdown, JSON, and JSONL runtime state. Repository policy requires OpenClaw-owned runtime state to use SQLite unless this is a named product artifact, so this data model needs maintainer approval or a SQLite-backed redesign before merge.
  - confidence: 0.86
- **[P2] Skip malformed audit lines when reading audit logs:** `src/claw/service.ts:3841-3848`
  - body: `getAudit` parses every JSONL line with a bare `JSON.parse`, so one truncated or corrupt append makes the whole audit feed fail. Mission audit history is operator-facing recovery evidence, so a partial write should drop or flag the bad record instead of hiding every valid entry.
  - confidence: 0.82
- **[P2] Gate Claw dashboard reloads to the Claw tab:** `ui/src/ui/app-gateway.ts:424-426`
  - body: Every `claw.*` event reloads the full Claw dashboard for every connected UI client, even outside the Claw tab. During active missions that can fan out repeated list/get/audit/artifact requests in the background and add avoidable Gateway/UI load.
  - confidence: 0.78
