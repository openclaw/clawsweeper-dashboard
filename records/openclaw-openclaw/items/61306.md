Overall correctness: patch is incorrect

Full review comments:

- **[P1] Preserve HTTP/1-only pinned dispatchers:** `src/infra/net/ssrf.ts:430-447`
  - body: This replaces the current `createHttp1*` dispatcher helpers with raw Undici constructors in the pinned SSRF path. Those helpers enforce the HTTP/1-only/proxy-safe policy used by current guarded networking, so this can weaken the security boundary for SSRF-pinned requests.
  - confidence: 0.92
- **[P1] Use effective browser plugin activation:** `src/claw/service.ts:600-602`
  - body: `isBrowserPluginEnabled` only accepts `plugins.entries.browser.enabled === true`, but the bundled browser plugin is enabled by default when not explicitly disabled. Browser-required missions in the default setup can be blocked as if the plugin were off until users add redundant config.
  - confidence: 0.87
- **[P1] Revalidate live mission state after agent calls return:** `src/claw/service.ts:2956-2960`
  - body: Runner, verifier, and helper cycles mutate and persist the in-memory mission after awaiting a long-running agent call. If an operator pauses or cancels while that call is in flight, these writes can overwrite the control action and resume or complete a mission unexpectedly.
  - confidence: 0.86
