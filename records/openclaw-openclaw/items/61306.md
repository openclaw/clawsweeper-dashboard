Full review comments:

- **[P1] Move mission persistence into SQLite:** `src/claw/service.ts:67-70`
  - body: This introduces durable OpenClaw-owned runtime state as `mission-state.json`, `control-state.json`, and `AUDIT_LOG.jsonl`. The repo policy requires runtime state to use the SQLite state stores unless there is an approved artifact/migration boundary, so this would create a second persistence model with no doctor migration path.
  - confidence: 0.92
- **[P1] Validate mission IDs before path joins:** `src/claw/service.ts:1717-1722`
  - body: `missionId` comes from Gateway RPC as a non-empty string, then `resolveMissionDir` appends it with `path.join`. A value containing `..` or path separators can escape the missions root and target arbitrary `mission-state.json` locations, so the service needs generated-ID validation or normalized containment before resolving paths.
  - confidence: 0.9
- **[P1] Keep the pinned dispatcher on HTTP/1 helpers:** `src/infra/net/ssrf.ts:430-432`
  - body: This constructs Undici dispatchers directly in the pinned SSRF path instead of using the current `createHttp1*` helpers. Those helpers enforce OpenClaw's HTTP/1-only dispatcher options and proxy TLS policy, so bypassing them weakens a guarded networking boundary.
  - confidence: 0.88
- **[P2] Use effective browser plugin activation:** `src/claw/service.ts:600-602`
  - body: The readiness check only accepts `plugins.entries.browser.enabled === true`, but the bundled browser plugin is enabled by default. Default configs that omit this field can be falsely preflight-blocked as if the browser plugin were disabled.
  - confidence: 0.86
- **[P2] Skip malformed audit log lines:** `src/claw/service.ts:3847`
  - body: `getAudit` parses each JSONL line with a bare `JSON.parse`, so one truncated audit write makes the entire audit feed fail. Operator audit views should skip or quarantine the bad row and preserve the rest of the mission history.
  - confidence: 0.82
- **[P2] Run Claw smoke for runner pipeline changes:** `.github/workflows/claw-smoke.yml:17-18`
  - body: The new Claw smoke workflow watches `src/agents/system-prompt*` but not the embedded runner pipeline that Claw execution calls. Future runner changes could skip this workflow even though they affect mission execution.
  - confidence: 0.78
