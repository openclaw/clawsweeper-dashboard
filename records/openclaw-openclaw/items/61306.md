## Summary

Keep open: current main does not contain the Claw subsystem, and this PR remains a large architecture branch with blocking runtime/UI defects plus a security-sensitive SSRF dispatcher regression.

## What This Changes

This PR adds a new Claw mission-control subsystem across `src/claw`, gateway RPC/events/scopes, shared/config types, Control UI, Claw docs, browser/runtime hardening, workflows, and related extension/test stabilization.

## Best Possible Solution

Keep the mission-control work in active maintainer review, decompose unrelated browser/memory/Zalo/CI changes where practical, preserve guarded-networking guarantees, align the mission lifecycle with Task Flow, add the required changelog entry, and require green org CI before merge.

## Solution Assessment

No; the current PR is not yet the best way to solve the issue because it bundles a broad product architecture with unrelated stabilization and introduces a guarded-networking regression. A safer path is maintainer-led decomposition or targeted fixes plus validation against Task Flow and org CI.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve HTTP/1-only pinned dispatchers:** `src/infra/net/ssrf.ts:430-431`
  - body: `createPinnedDispatcher` constructs raw Undici dispatchers instead of the `createHttp1*` helpers that force `allowH2: false` for guarded networking. This can re-enable HTTP/2 on SSRF-pinned fetch paths, where dispatcher overrides are not the contract current main relies on.
  - confidence: 0.88
- **[P1] Isolate recovery failures before cycles:** `src/claw/runtime.ts:76`
  - body: `drainClawRuntimeQueue` awaits recovery before running normal mission cycles and only marks recovery complete after it succeeds. If one bad mission throws during recovery, the shared catch skips all queued/active mission cycles and the next wake retries the same failing recovery path.
  - confidence: 0.86
- **[P1] Re-check live mission status after runner returns:** `src/claw/service.ts:2956-2960`
  - body: The runner applies state transitions to the pre-run `state` object after the long `runMissionAgent` call returns. If an operator pauses or cancels the mission while that call is in flight, these writes can overwrite the operator action; reload and validate current mission status before persisting runner, helper, and verifier outcomes.
  - confidence: 0.83
- **[P2] Reload the Claw dashboard only when needed:** `ui/src/ui/app-gateway.ts:424-425`
  - body: This handler reloads the full Claw dashboard for every `claw.*` event on every connected UI, even when the user is on another tab. Runtime cycles can emit several events, so unrelated tabs fan out repeated mission, audit, and artifact RPCs in the background.
  - confidence: 0.78
- **[P2] Skip malformed audit log lines:** `src/claw/service.ts:3847`
  - body: `getAudit` parses every JSONL line with a bare `JSON.parse`, so one truncated audit append makes the whole audit feed fail. Skip or quarantine malformed records so operators can still inspect the remaining mission audit history.
  - confidence: 0.76
