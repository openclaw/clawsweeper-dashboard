  - body: This introduces durable mission/control/audit runtime state as JSON/JSONL sidecars. Root policy requires OpenClaw-owned runtime state and caches to live in SQLite-backed state stores unless maintainers approve a named storage exception, so this would create immediate migration and upgrade debt.
  - confidence: 0.92

  - body: Gateway methods accept `missionId` from request params and the service joins it directly under the missions root. Any path separators or `..` segments can escape that root for reads/writes if a matching path exists, so mission IDs need a generated-ID regex plus normalized containment before filesystem access.
  - confidence: 0.88
- **[P1] Preserve HTTP/1-only pinned dispatchers:** `src/infra/net/ssrf.ts:426-432`
  - body: Current main routes guarded pinned dispatchers through `createHttp1*` helpers so `allowH2:false` is enforced and covered by tests. This change constructs Undici dispatchers directly in the SSRF-guarded path, dropping that helper contract and risking HTTP/2 behavior where the guarded overrides are not relied on.
  - confidence: 0.9
- **[P2] Skip malformed audit lines when building the feed:** `src/claw/service.ts:3841-3848`
  - body: `getAudit` parses every JSONL line with bare `JSON.parse`, so one truncated or corrupt audit entry makes the entire `claw.audit.get` request fail. Runtime audit feeds should skip or surface the bad record without hiding all earlier valid mission audit entries.
  - confidence: 0.78
