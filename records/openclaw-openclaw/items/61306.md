## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Move mission state into SQLite:** `src/claw/service.ts:67-70`
  - body: This PR declares and writes `mission-state.json`, `control-state.json`, and `AUDIT_LOG.jsonl` as durable mission/control/audit runtime state. OpenClaw runtime state is supposed to use SQLite, so merging this sidecar storage model would create migration and upgrade debt for any operator who enables Claw.
  - confidence: 0.96
- **[P1] Validate mission IDs before resolving paths:** `src/claw/service.ts:1717-1722`
  - body: Gateway handlers accept any non-empty `missionId`, and this helper joins it directly under the missions root. A value containing `..` or path separators can escape the intended mission directory and reach another `mission-state.json`, so the service needs a canonical mission-id validator or normalized containment check before path resolution.
  - confidence: 0.94
- **[P1] Preserve HTTP/1-only pinned dispatchers:** `src/infra/net/ssrf.ts:426-432`
  - body: The patch switches `createPinnedDispatcher` to construct Undici dispatchers directly, bypassing the current `createHttp1*` helpers that apply `allowH2: false` and proxy safety handling for guarded networking. That can re-enable unsupported HTTP/2 behavior on SSRF-guarded requests.
  - confidence: 0.92
- **[P2] Treat default-enabled browser plugin as enabled:** `src/claw/service.ts:600-602`
  - body: The browser plugin manifest is enabled by default, but this check only accepts `plugins.entries.browser.enabled === true`. Normal configs that omit the entry while using the default-enabled browser plugin will be preflight-blocked as if browser support were disabled.
  - confidence: 0.9
- **[P2] Return after the first invalid audit parameter:** `src/gateway/server-methods/claw.ts:372-374`
  - body: `claw.audit.get` reads both `missionId` and `limit` before returning on invalid input, and both helpers call `respond` on failure. If both fields are invalid, the handler can send two responses for one Gateway request, violating the RPC response contract.
  - confidence: 0.88
- **[P2] Skip malformed audit lines when building the feed:** `src/claw/service.ts:3841-3848`
  - body: `getAudit` parses every JSONL line with a bare `JSON.parse`, so one truncated or corrupt audit record makes the whole audit request fail. Operators should lose only the bad entry, not the entire mission audit feed after an interrupted append.
  - confidence: 0.84
