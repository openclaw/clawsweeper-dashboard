Full review comments:

- **[P1] Move mission runtime state into SQLite:** `src/claw/service.ts:68-70`
  - body: The new Claw mission engine persists core runtime state through `mission-state.json`, `control-state.json`, and `AUDIT_LOG.jsonl` under a workspace `missions/` tree. OpenClaw-owned runtime/session state is supposed to live in SQLite, so merging this would create a second durable state model with no migration or upgrade story.
  - confidence: 0.93
- **[P1] Reject unsafe mission IDs before resolving paths:** `src/claw/service.ts:1717-1722`
  - body: Mission IDs supplied to Gateway methods are joined directly under the missions root. A crafted ID containing `..` or path separators can escape the mission tree and make later reads/writes target arbitrary `mission-state.json` paths, so validate the generated ID format or enforce normalized containment before resolving paths.
  - confidence: 0.88
- **[P1] Preserve HTTP/1-only pinned dispatchers:** `src/infra/net/ssrf.ts:426-432`
  - body: `createPinnedDispatcher` now constructs Undici `Agent`/proxy dispatchers directly instead of using the `createHttp1*` helpers that set `allowH2: false`. That can re-enable HTTP/2 on SSRF-guarded requests and weaken the pinned-lookup assumptions those guards rely on.
  - confidence: 0.86
