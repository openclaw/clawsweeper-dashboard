This PR adds an initial Claw mission-control subsystem across service/runtime state, gateway RPC/events/scopes, shared/config types, Control UI, docs, workflows, and adjacent browser/memory/Zalo validation changes.

## Best Possible Solution

Keep the mission-control work in maintainer review, fix the guarded-networking and lifecycle blockers, decompose unrelated stabilization where practical, add the required user-facing changelog entry, and require green org CI before merge.

## Solution Assessment

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve HTTP/1-only pinned dispatchers:** `src/infra/net/ssrf.ts:430-431`
  - body: `createPinnedDispatcher` constructs raw Undici dispatchers instead of the HTTP/1-only helper path that enforces `allowH2: false`, which can weaken the guarded SSRF networking contract for pinned fetches.
  - confidence: 0.88
- **[P1] Isolate recovery failures before cycles:** `src/claw/runtime.ts:76`
  - body: `drainClawRuntimeQueue` awaits recovery before normal mission cycles and only marks recovery complete after success, so one bad recovery can repeatedly block all queued or active missions from progressing.
  - confidence: 0.86
- **[P1] Re-check live mission status after runner returns:** `src/claw/service.ts:2956-2960`
  - body: The runner writes transitions using stale in-memory mission state after a long agent call, so an operator pause or cancel during the call can be overwritten when the runner finishes.
  - confidence: 0.83
- **[P2] Reload the Claw dashboard only when needed:** `ui/src/ui/app-gateway.ts:424-425`
  - body: The UI reloads the full Claw dashboard for every `claw.*` event on every connected client, even outside the Claw tab, causing avoidable background gateway fanout during mission cycles.
  - confidence: 0.78
- **[P2] Skip malformed audit log lines:** `src/claw/service.ts:3847`
  - body: `getAudit` parses every JSONL line with a bare `JSON.parse`, so one truncated audit append makes the entire mission audit feed fail instead of skipping only the bad record.
  - confidence: 0.76
