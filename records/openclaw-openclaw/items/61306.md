Full review comments:

- **[P1] Move Claw mission state into the state database:** `src/claw/service.ts:67-70`
  - body: The PR makes `missions/mission-state.json`, `control-state.json`, and `AUDIT_LOG.jsonl` canonical runtime state. OpenClaw runtime state is database-first; keep user-facing Markdown as artifacts if needed, but store mission/control/audit state in the shared/per-agent SQLite path or Task Flow-owned tables before merge.
  - confidence: 0.93
- **[P1] Preserve the HTTP/1-only pinned dispatcher wrappers:** `src/infra/net/ssrf.ts:426-430`
  - body: `createPinnedDispatcher` now creates raw Undici dispatchers, bypassing the current `createHttp1*` wrappers that enforce `allowH2: false` and proxy-safe behavior for guarded networking. Restore the shared wrappers so SSRF-pinned requests keep the existing transport contract.
  - confidence: 0.9
- **[P1] Use effective browser plugin activation for preflight:** `src/claw/service.ts:600-602`
  - body: This check only accepts `plugins.entries.browser.enabled === true`, but the bundled browser plugin is enabled by default unless plugin policy disables it. Default installs can therefore block browser-required missions as not enabled even when the plugin is effectively active.
  - confidence: 0.9
- **[P2] Return before validating audit limit after missing missionId:** `src/gateway/server-methods/claw.ts:372-374`
  - body: `readRequiredString` can already send an invalid-request response for `missionId`, but the handler still validates `limit` before the guard. A request with both a missing missionId and invalid limit can produce two responses on one RPC.
  - confidence: 0.93
- **[P2] Gate Claw dashboard reloads to the Claw tab:** `ui/src/ui/app-gateway.ts:424-426`
  - body: Every `claw.*` event reloads the full Claw dashboard for every connected UI client, even off the Claw tab. Mission cycles can emit many events, causing avoidable background list/get/audit/artifact traffic and UI churn.
  - confidence: 0.86
