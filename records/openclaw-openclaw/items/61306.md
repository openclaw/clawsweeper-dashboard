Overall confidence: 0.88

Full review comments:

- **[P1] Move mission runtime state into SQLite:** `src/claw/service.ts:68-70`
  - body: The new Claw mission engine persists OpenClaw-owned runtime state through JSON/JSONL sidecar files under the workspace. That conflicts with the repository storage model and would create a second durable state path without migration or upgrade ownership.
  - confidence: 0.93
- **[P1] Reject unsafe mission IDs before resolving paths:** `src/claw/service.ts:1717-1722`
  - body: Mission IDs supplied through Gateway methods are joined directly under the missions root. A value containing path separators or `..` can escape the intended mission tree and make later reads or writes target paths outside it.
  - confidence: 0.89
- **[P1] Preserve HTTP/1-only pinned dispatchers:** `src/infra/net/ssrf.ts:426-432`
  - body: `createPinnedDispatcher` now constructs Undici dispatchers directly instead of using the current `createHttp1*` helpers that enforce `allowH2: false`. That can re-enable HTTP/2 on SSRF-guarded requests and weaken the pinned-lookup assumptions those guards rely on.
  - confidence: 0.87
- **[P1] Use effective browser plugin activation in preflight:** `src/claw/service.ts:600-602`
  - body: The preflight check only treats the browser plugin as ready when config explicitly sets `plugins.entries.browser.enabled === true`, but the bundled browser plugin is enabled by default. Default installs can therefore block browser-required missions with a false setup error.
  - confidence: 0.84
