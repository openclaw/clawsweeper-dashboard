Full review comments:

- **[P1] Move mission state into approved storage:** `src/claw/service.ts:67-70`
  - body: This introduces OpenClaw-owned runtime mission, control, and audit state as workspace JSON/JSONL sidecar files. Root policy requires runtime state like queues, checkpoints, and plugin scratch data to live in the shared or agent SQLite stores with migrations/doctor owning legacy transitions, so this would add a second persisted state model before the feature even ships.
  - confidence: 0.9
- **[P1] Preserve HTTP/1-only pinned dispatchers:** `src/infra/net/ssrf.ts:430-431`
  - body: `createPinnedDispatcher` now creates raw Undici dispatchers directly, bypassing the current `createHttp1*` helper seam that forces `allowH2: false` for guarded fetches. That can re-enable HTTP/2 on SSRF-pinned requests and weaken the lookup/dispatch assumptions the guard relies on.
  - confidence: 0.86
- **[P2] Return after the first invalid audit parameter:** `src/gateway/server-methods/claw.ts:372-373`
  - body: `claw.audit.get` reads `missionId` and `limit` before checking whether the first helper already responded with an invalid-request error. When both parameters are invalid, both helper reads can call `respond(false, ...)` for one RPC request, unlike the neighboring handlers that return immediately after each failed read.
  - confidence: 0.82
- **[P1] Reload mission state after long agent calls:** `src/claw/service.ts:2922`
  - body: Runner cycles await a long `runMissionAgent` call and then write lifecycle state using the stale pre-call mission object. If an operator pauses or cancels the mission while the agent is in flight, this path can overwrite that control action when the call returns, so the cycle should reload and verify the live mission status before persisting transitions.
  - confidence: 0.78
