## What This Changes

The issue reports that Linux systemd user-service auto-update spawns `openclaw update` as a Gateway descendant, causing the CLI ancestry guard to abort instead of using a detached managed-service handoff.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/91833 or an equivalent Linux systemd auto-update handoff fix, then close this issue as implemented after merge.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: on Linux with only the generic OpenClaw gateway service markers, current main does not detect systemd supervision and falls through to a direct updater spawn that can hit the Gateway ancestry guard.

## Solution Assessment

Yes at the issue level. The linked PR’s direction is the narrow maintainable fix because it reuses the existing managed-service handoff instead of adding a second updater path; this issue should remain open until that PR merges.
