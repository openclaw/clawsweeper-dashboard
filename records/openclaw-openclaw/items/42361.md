## What This Changes

The PR wraps web_fetch network failures with URL/cause context and also changes Feishu media-key validation, Telegram allowlist parsing, package overrides, model input handling, config schema, and tool-result text validation.

## Best Possible Solution

Land a narrow web_fetch network-error wrapper with focused regression coverage, and split or drop the Feishu, Telegram, dependency, config/model, and tool-result changes into separate owner-reviewed PRs.

## Reproduction Assessment

Yes, by source inspection: current main catches direct fetch failures, tries provider fallback, and rethrows the original error when no fallback payload exists. A focused mocked fetch failure can exercise that no-fallback path without needing a live network outage.

## Solution Assessment

No. The web_fetch wrapper is the right narrow direction, but this branch is not the best delivery vehicle because it bundles unrelated package, Feishu, Telegram, config/model, and tool-result changes with proof and correctness blockers.
