## Summary

Keep open. The underlying web_fetch diagnostic bug is still present on current main, and the PR contains the intended wrapper, but the branch is not safe to merge because it bundles unrelated package, Feishu, and Telegram changes with concrete regressions. A narrow replacement PR is the safest cleanup path.

## What This Changes

The PR wraps web_fetch direct network failures with URL and cause context while also changing Feishu external-key validation, package overrides, model/input handling, tool-result text checks, agent model schema, and Telegram allowlist parsing.

## Best Possible Solution

Replace this branch with a web_fetch-only patch that wraps direct network failures after provider fallback is unavailable, extracts a safe reason from error.cause/message, preserves the original error as cause, and adds a focused regression test. Leave Feishu, Telegram, package overrides, config schema, and model/tool-result changes to separate reviewed PRs.

## Reproduction Assessment

Yes. A focused unit test can mock the guarded fetch to reject with a Node-style TypeError("fetch failed") plus cause, disable provider fallback, and verify current main rethrows the bare error from runWebFetch.

## Solution Assessment

No. The web_fetch wrapper is the right direction, but this PR is not the best way to solve the issue because unrelated bundled changes introduce install, Feishu, and Telegram regressions; a narrow replacement is safer.
