Full review comments:

- **[P1] Acquire the busy lease before sync initialization:** `extensions/memory-core/src/memory/manager.ts:747`
  - body: `sync()` does not acquire the idle-eviction busy ref until after `ensureProviderInitialized()`. Direct watch/interval/session syncs are not wrapped by `search()`, so an idle sweep can close the manager after provider init starts but before `this.syncing` is set; `close()` then captures a null pendingSync and may close the provider/database while that sync continues. Acquire/release around the whole direct sync startup path and cover this race with a focused test.
  - confidence: 0.9

That breaks the config contract for the very knobs operators need if the new default gateway sidecar causes problems.
  - confidence: 0.82
