## Summary

Keep open: current main and v2026.6.9 still drop the `skipDeferral` intent before close-stage shutdown, while the linked fixing PR is open and unmerged.

## What This Changes

The issue asks for `openclaw gateway restart --safe --skip-deferral` to bypass or clearly document the close-stage pending-reply drain for its own tool/final replies.

## Best Possible Solution

Refresh and land or replace the linked PR with a narrow intent-threading fix that preserves ordinary safe-restart reply draining while making explicit skip-deferral restarts bypass the close-stage reply drain.

## Reproduction Assessment

Yes, at source level. Current main records `skipDeferral` for scheduling but does not carry it in `GatewayRestartIntent`, and close-stage reply draining still computes the normal restart drain timeout.

## Solution Assessment

Yes. Carrying the explicit skip-deferral intent into the SIGUSR1 close-drain decision is the narrow maintainable fix; a docs-only clarification would leave the operator escape hatch behavior broken.
