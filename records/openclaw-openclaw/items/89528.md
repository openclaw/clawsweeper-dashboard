## Summary

Keep open: current main and v2026.6.8 still lack `skipDeferral` state in the restart intent consumed by close-stage shutdown, and the linked fixing PR https://github.com/openclaw/openclaw/pull/89604 is open and unmerged.

## What This Changes

The issue reports that `openclaw gateway restart --safe --skip-deferral` can still wait on the close-stage pending reply drain for the restart-requesting turn's own tool/final replies.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/89604 with a narrow fix that carries `skipDeferral` into SIGUSR1 close handling while preserving normal safe-restart reply draining.

## Reproduction Assessment

Yes, at source level. Current main forwards `skipDeferral` only into scheduling/preflight, then computes close-stage reply drain timeout without skip-deferral state, matching the reporter's pending-reply shutdown log path.

## Solution Assessment

Yes. Carrying the explicit skip-deferral intent into the restart close-drain decision is the narrow maintainable fix; a docs-only clarification would leave the operator escape hatch behavior broken.
