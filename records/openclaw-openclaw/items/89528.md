## What This Changes

The issue reports that `openclaw gateway restart --safe --skip-deferral` can still wait on close-stage pending replies from the restart-requesting turn instead of acting as an operator escape hatch.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/89604 with a narrow fix that carries `skipDeferral` into SIGUSR1 close handling while preserving normal safe-restart reply draining.

## Reproduction Assessment

Yes. Source inspection on current main shows `skipDeferral` bypasses pre-SIGUSR1 deferral but is not preserved for the close-stage reply drain, matching the reporter's pending-reply shutdown logs.

## Solution Assessment

Yes. Carrying the explicit skip-deferral intent into the restart close-drain decision is the narrow maintainable fix; a docs-only clarification would leave the operator escape hatch behavior broken.
