## Summary

The report is valid in source: Feishu’s registered capability authorizes approvers but does not provide native approval delivery, so pending workshop requests can have no route. Current main and v2026.6.5 already bypass lifecycle approval for `approvalPolicy: "auto"`, so that part of the 2026.6.1 report is fixed, but interactive Feishu approval remains a distinct open gap with an existing Feishu approval-card PR that needs maintainer evaluation.

## What This Changes

Add a usable Feishu approval route for agent-driven skill workshop apply actions, while ensuring auto approval policy bypasses prompting.

## Best Possible Solution

Extend Feishu’s existing approval capability with the shared native runtime for both exec and plugin events, reusing its authorization adapter and card-action handler, while retaining the shipped auto-policy bypass and proving direct-chat apply/deny resolution end to end.

## Reproduction Assessment

Yes at source level: a pending workshop action requests plugin approval, while Feishu registers authorization without native delivery, producing the documented no-route branch. No live Feishu bot reproduction was performed in this review.

## Solution Assessment

Partly. The auto-policy fast path is already the correct shipped solution for pre-approved operation; interactive mode should use the shared Feishu native approval runtime rather than special-casing workshop or reinterpreting auth registration as delivery.
