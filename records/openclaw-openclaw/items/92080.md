## Summary

Keep open: current `main` and v2026.6.6 still have the reported Skill Workshop approval/no-route behavior, and the open closing PR for this issue has not merged. Feishu does expose an `approvalCapability`, but current source shows that capability is auth-only, while the failing path depends on a routable plugin approval request or an honored `skills.workshop.approvalPolicy: "auto"` config.

## What This Changes

The issue reports that Feishu-originated `skill_workshop action=apply` returns `Plugin approval unavailable (no approval route)` despite Feishu approval capability metadata and `skills.workshop.approvalPolicy: "auto"`.

## Best Possible Solution

Honor the real configured `skills.workshop.approvalPolicy: "auto"` for Skill Workshop lifecycle calls while preserving fail-closed pending approval, either by carrying canonical config/turn-source context into the hook or by proving the linked narrow policy change is security-safe.

## Reproduction Assessment

Yes at source level, but not as a live Feishu run. Current `main` can default missing hook config to pending approval, the gateway returns `decision: null` when no approval route exists, and the before-tool-call path maps that to the reported error.

## Solution Assessment

Unclear. The linked one-file fix is plausible for the reporter's auto-policy symptom, but maintainers should decide whether skipping approval on missing config is safe or whether the better fix is to pass canonical config and turn-source routing into the approval request.
