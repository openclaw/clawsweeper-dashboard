## Summary

Keep open: current main and v2026.6.6 still have the no-route failure path for plugin approvals, and this Feishu report also points at Skill Workshop config propagation that is not safely fixed by the linked workaround PR. The older canonical routing issue and proof-positive PR cover part of the root cause, but this issue should stay open until that lands and the Feishu/Skill Workshop apply flow is verified.

## What This Changes

The issue reports that Feishu `skill_workshop action=apply` returns `Plugin approval unavailable (no approval route)` and ignores `skills.workshop.approvalPolicy: "auto"`, leaving proposals pending.

## Best Possible Solution

Land or reject the canonical turn-source routing fix, repair Skill Workshop config propagation without fail-open approval bypasses, and verify the Feishu `skill_workshop apply` lifecycle path with redacted live proof.

## Reproduction Assessment

Yes. Current-main source shows Skill Workshop lifecycle calls can still enter plugin approval, the request omits turn-source fields, and the gateway returns `decision: null` when no route accepts delivery; I did not run a live Feishu bot.

## Solution Assessment

No. The linked workaround PR is not the best fix because missing config must not become authorization; the maintainable path is trusted config and turn-source propagation plus Feishu/live proof.
