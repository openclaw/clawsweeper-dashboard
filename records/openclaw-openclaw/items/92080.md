## What This Changes

The issue reports that Feishu `skill_workshop apply` returns `Plugin approval unavailable` and leaves proposals pending even though Feishu registers an approval capability.

## Best Possible Solution

Land the existing turn-source forwarding fix in https://github.com/openclaw/openclaw/pull/90918, then close this Feishu-specific report as covered or track any remaining `approvalPolicy: "auto"` config propagation as a narrow follow-up.

## Reproduction Assessment

Yes, at source level. Current main omits turn-source routing fields from plugin approval requests, while the gateway no-route branch returns `decision: null`, matching the reported `Plugin approval unavailable` failure; no live Feishu run was executed in this read-only review.

## Solution Assessment

Yes for the route-forwarding direction, no for the missing-config bypass. Forwarding trusted HookContext turn-source data through the existing gateway contract is the narrow fix; skipping approval when config is absent would weaken authorization for apply, reject, and quarantine.
