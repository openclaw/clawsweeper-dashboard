## Best Possible Solution

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Read durationMs from a typed field:** `extensions/codex/src/app-server/event-projector.ts:816`
  - body: This call does not compile: `readNumber` takes a `JsonObject`, but `CodexThreadItem` has an `unknown` index signature, and both prod and test typecheck jobs fail here with TS2345. Read `params.item.durationMs` directly with a finite-number guard or add a helper that accepts `unknown` records before this can merge.
  - confidence: 0.99
- **[P2] Preserve ordering for blocked tool events:** `extensions/codex/src/app-server/event-projector.ts:819-824`
  - body: `tool.execution.started` is dispatched asynchronously, but `tool.execution.blocked` is not in the async lifecycle set. For a declined native item, this new blocked emission can run before the queued start event, so `recordToolEnded` finds no active tool and the later start leaves a stale active tool that keeps the watchdog in `blocked_tool_call` state; queue blocked with the other tool lifecycle events or otherwise preserve cleanup order.
  - confidence: 0.89
