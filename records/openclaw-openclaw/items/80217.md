## Summary

Keep this PR open: it addresses a real Codex diagnostics gap and the updated terminal proof is sufficient, but the patch introduces a blocked-tool ordering bug that can leave a denied native tool permanently marked active.

## What This Changes

The PR emits trusted diagnostic tool lifecycle events for Codex-native tool items and adds regression coverage for stuck-session classification while a native tool is active.

## Best Possible Solution

Keep the Codex projector as the owner boundary for native tool diagnostics, but make terminal event dispatch ordering guarantee that completed, failed, and blocked native tools always clear active diagnostic state, with regression coverage for the blocked path.

## Reproduction Assessment

No live current-main repro was run in this read-only review, but the issue is source-reproducible: current main emits Codex-native tool progress without feeding the diagnostic active-tool tracker, while the watchdog only recognizes active tools from `tool.execution.*` diagnostic events.

## Solution Assessment

No, not as written. Emitting native diagnostic events from the Codex projector is the right boundary, but the blocked terminal event must be ordered consistently with started events before this is safe to merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Keep blocked tool events ordered with starts:** `extensions/codex/src/app-server/event-projector.ts:820-825`
  - body: The new blocked path emits `tool.execution.blocked`, but `tool.execution.started` is queued asynchronously while blocked events are not in the async diagnostic set. If a native item starts and then completes as `declined` before the queue drains, `recordToolEnded` sees no active tool, then the queued start marks it active forever; later watchdog passes classify it as `blocked_tool_call` and skip embedded-run recovery. Make blocked terminal events use the same dispatch ordering as started/completed/error, and add a regression for a start-plus-declined completion in the same tick.
  - confidence: 0.91
