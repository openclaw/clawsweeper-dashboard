## Summary

Keep open. The PR addresses a real ACP identity mismatch, but the current patch can make ensure-derived IDs authoritative over already-resolved status/event IDs and persist stale ACP identifiers before status reconciliation repairs them.

## What This Changes

The PR marks ACP ensure-derived identities as resolved when stable session IDs are present, adds focused identity-construction tests, and updates an ACP spawn fixture expectation.

## Best Possible Solution

Keep resolved-on-ensure for new or still-pending identities, but make merge precedence source-aware so later ensure results cannot replace already-resolved status/event identifiers.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: merge an already-resolved status/event identity with a later ensure identity carrying stale stable IDs, then let status refresh return early with `failOnStatusError: false`.

## Solution Assessment

No. Marking ensure identities as resolved is directionally useful, but resolvedness alone is too broad as merge authority; explicit precedence for ensure versus status/event identities is safer.
