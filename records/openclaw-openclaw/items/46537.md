## Summary

Keep open. Current main still keeps ensure-derived ACP identities pending, so the intended fix has not landed, but this PR still has a blocking merge-precedence regression: resolving ensure identities makes stale ensure IDs authoritative over fresher resolved status/event IDs.

## What This Changes

The PR marks ACP ensure-derived identities as resolved when stable session IDs are present and adds focused identity tests plus an ACP spawn fixture update.

## Best Possible Solution

Keep the resolved-on-ensure behavior for new or still-pending identities, but make merge precedence source-aware so ensure results cannot overwrite already-resolved status/event identifiers.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: merge a resolved status/event identity with a later ensure identity carrying stale stable IDs, then take the `failOnStatusError: false` early-return path before status can repair it.

## Solution Assessment

No. The PR direction is useful, but resolvedness alone is too broad as merge authority; source-aware precedence is the narrower maintainable fix.
