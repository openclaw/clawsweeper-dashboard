## Summary

Keep open: current main still has the ensured ACP identity bug, but this PR is not merge-ready because making ensure identities resolved also lets stale ensure IDs overwrite fresher resolved status/event IDs before status reconciliation can repair them.

## What This Changes

The PR marks ensure-derived ACP identities with stable session IDs as resolved and updates focused identity and ACP spawn tests.

## Best Possible Solution

Keep the resolved-on-ensure behavior for new or still-pending identities, but add source-aware merge precedence so ensure results cannot replace already-resolved status/event identifiers.

## Reproduction Assessment

Yes, via source inspection. Current main keeps stable ensure identities pending, and the PR-introduced stale-overwrite path follows from mergeSessionIdentity plus ensureRuntimeHandle and the failOnStatusError:false reconcile path.

## Solution Assessment

No. The direction is useful, but resolvedness alone is too broad as merge authority; source-aware precedence is the narrower maintainable fix.
