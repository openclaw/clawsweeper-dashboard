## Summary

Keep open: the PR addresses a real ACP identity bug that current main still has, but the patch can let stale ensure-sourced IDs replace already-resolved status/event IDs.

## What This Changes

The PR marks ensure-derived ACP identities as resolved when stable session IDs are present, adds focused identity tests, and updates an ACP spawn fixture expectation.

## Best Possible Solution

Keep resolved-on-ensure for new or still-pending identities, but make merge precedence source-aware so ensure results cannot overwrite already-resolved status/event identifiers.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: start with a resolved status/event identity, merge a later ensure identity carrying stale stable IDs, then let a `failOnStatusError: false` status refresh return early.

## Solution Assessment

No. The PR’s direction is narrow and useful, but resolvedness alone is too broad as merge authority; explicit source-aware precedence is safer.
