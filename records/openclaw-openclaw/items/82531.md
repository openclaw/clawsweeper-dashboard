## Summary

Current main and the shipped v2026.5.12 release already provide the requested per-channel DM isolation via `session.dmScope`; the reported shared-main behavior is the documented default, and the Control UI-to-external-channel delivery concern is separately tracked.

## What This Changes

Request per-channel session isolation so Control UI/webchat and WeChat/openclaw-weixin conversations do not share one agent session.

## Best Possible Solution

Keep the shipped `session.dmScope` contract for isolation and direct any separate Control UI operator-reply behavior to https://github.com/openclaw/openclaw/issues/45028.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/configuration question. The default shared-DM behavior and the isolation configuration path are both source-backed and documented.

## Solution Assessment

Yes. The existing `session.dmScope` setting is the narrow maintainable solution for channel/session isolation; changing whether Control UI messages deliver to WeChat would be a separate product decision tracked by https://github.com/openclaw/openclaw/issues/45028.
