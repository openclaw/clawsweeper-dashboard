## Summary

This remains a real bug-fix candidate, but the proposed patch should not merge as-is because it starts the persistent skills watcher from the one-shot CLI path and can keep local agent commands alive after the reply.

## What This Changes

The branch adds a CLI-only skills snapshot refresh flag, starts the skills watcher, manually bumps the skills snapshot version before reuse checks, and extends the agent command test to assert those calls.

## Best Possible Solution

Land the CLI-only manual snapshot-version bump before the cached snapshot reuse check, keep watcher startup owned by the long-lived session update path, and cover CLI bump plus ingress no-bump behavior in focused tests.

## Reproduction Assessment

Yes. Source inspection on current main shows the CLI path can reuse a persisted `skillsSnapshot` based only on the stored snapshot version and filter, and the PR body provides a fixed-session before/after local CLI verification for an extra-dir skill.

## Solution Assessment

No. The manual snapshot-version bump is the right narrow fix, but starting `ensureSkillsWatcher` from `agent-command.ts` is not the best solution because that watcher is persistent and already belongs to long-lived session startup.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Avoid starting the persistent watcher in one-shot CLI runs:** `src/agents/agent-command.ts:503`

Since `openclaw agent --local` is documented as a one-shot command and chokidar watchers are persistent by default, this can keep the CLI process alive after the reply; keep the manual bump but leave watcher startup in the long-lived session path.
  - confidence: 0.91
