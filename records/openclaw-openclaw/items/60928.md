## Summary

Keep open: the reported stale CLI skills snapshot path is still plausible on current main and the PR contains useful proof, but the patch is not merge-ready because it starts a persistent skills watcher from a documented one-shot CLI flow.

## What This Changes

The branch makes CLI agent runs force a skills snapshot refresh and extends the agent command test assertions for that path.

## Best Possible Solution

Land a current-main CLI-only manual skills snapshot version bump before the version read, while leaving watcher startup in long-lived gateway/session code.

## Reproduction Assessment

Yes. Current main shows the reused-session snapshot path is keyed by snapshot version and skill filter, and the PR body gives a fixed-session extra-dir before/after CLI verification path.

## Solution Assessment

No. The manual version bump is the right direction, but starting `ensureSkillsWatcher` from the one-shot CLI path is not the narrowest maintainable fix; use the refresh-state seam and keep watcher ownership unchanged.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Avoid starting the skills watcher from CLI agent runs:** `src/agents/agent-command.ts:503`
