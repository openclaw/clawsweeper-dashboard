## Summary

Keep open: this PR targets a real stale skills snapshot path, but the current patch starts the persistent skills watcher from the one-shot CLI agent flow. The useful part is the CLI-only manual version bump; the watcher lifecycle should stay with the long-lived gateway/session path.

## What This Changes

The branch adds a forced skills snapshot refresh to CLI `openclaw agent` runs and extends `src/commands/agent.test.ts` to assert the refresh calls.

## Best Possible Solution

Land a CLI-only manual skills snapshot version bump before the snapshot version read, without starting the persistent watcher from the one-shot CLI path.

## Reproduction Assessment

Yes. Current main source shows reused `skillsSnapshot` state when the version/filter match and no CLI manual bump before the version read, and the PR body provides before/after fixed-session CLI verification.

## Solution Assessment

No. The manual version bump is the right direction, but starting `ensureSkillsWatcher` in the CLI path is not the best fix because watcher ownership already lives in the long-lived session path and chokidar watchers are persistent by default.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Avoid starting the persistent skills watcher from CLI agent runs:** `src/agents/agent-command.ts:503`

Since `--local`/embedded runs are documented as one-shot and chokidar's `persistent` default is true, this can keep `openclaw agent --local` alive after the reply; keep the manual version bump but avoid starting the watcher here.
