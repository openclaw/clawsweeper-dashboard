## Summary

Keep open: the PR targets a valid stale CLI skills-snapshot bug, but the current patch should not merge because it starts the persistent skills watcher from the one-shot CLI agent path. The useful part is the manual snapshot-version bump, which should be ported to current main through the `refresh-state` seam without taking watcher ownership.

## What This Changes

The PR makes CLI `openclaw agent` runs force a skills snapshot refresh and adds focused assertions in the agent command test.

## Best Possible Solution

Land a CLI-only manual skills snapshot version bump before the snapshot-version read, while leaving watcher startup in long-lived gateway/session paths.

## Reproduction Assessment

Yes. Current main source shows reused session snapshots are keyed only by snapshot version and skill filter, and the PR body provides a fixed-session extra-dir before/after CLI verification; I did not rerun the live flow in this read-only review.

## Solution Assessment

No. The manual version bump is the right direction, but starting `ensureSkillsWatcher` from the one-shot CLI path is not the narrowest maintainable fix; use `refresh-state` only and keep ingress watcher ownership unchanged.
