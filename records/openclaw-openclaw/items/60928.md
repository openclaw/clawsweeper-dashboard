## Summary

Keep open: the PR targets a real stale skills snapshot bug, but the proposed patch starts a persistent skills watcher in the one-shot CLI path without teardown.

## What This Changes

The PR adds a forced skills-snapshot refresh path for `openclaw agent` and extends the agent command test to assert watcher and manual snapshot-version bump calls.

## Best Possible Solution

Keep the CLI manual snapshot-version bump, adapt it to current main's lazy refresh-state loader, and leave watcher lifecycle owned by long-lived gateway/session update code.

## Reproduction Assessment

Yes, source inspection plus the PR body's extraDirs/reused-session steps give a high-confidence reproduction path. Current main can reuse a persisted `skillsSnapshot` solely from the stored version unless another lifecycle bumps it.

## Solution Assessment

No. The manual version bump before snapshot comparison is the narrow fix, but starting `ensureSkillsWatcher` from `openclaw agent` changes lifecycle behavior and needs removal or explicit teardown.
