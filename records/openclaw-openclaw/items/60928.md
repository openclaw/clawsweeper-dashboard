## Summary

Keep open: the stale CLI skills snapshot problem is still source-plausible on current main, but this PR introduces a blocking lifecycle regression by starting a persistent skills watcher from the one-shot `openclaw agent` path.

## What This Changes

The PR adds a CLI skills snapshot refresh flag that starts the skills watcher and manually bumps the snapshot version before `openclaw agent` compares cached session snapshots, with a focused command test.

## Best Possible Solution

Land a rebased CLI-only manual snapshot-version bump before snapshot comparison, without watcher startup in one-shot agent runs, plus regression coverage for CLI and ingress behavior.

## Reproduction Assessment

Yes. I did not run the live CLI path in this read-only review, but current main’s source can reuse a cached snapshot when the version still matches, and the PR body provides concrete `skills.load.extraDirs` plus fixed-session reproduction steps.

## Solution Assessment

No. The manual version bump is the right narrow direction, but starting `ensureSkillsWatcher` from `openclaw agent` is not the best fix because watcher lifecycle belongs to long-lived gateway/session update code.

## Review Findings
