## Summary

Keep open: the PR targets a real stale skills-snapshot bug, but the patch should not merge as written because it starts a persistent skills watcher from the one-shot agent command path and is now based on older pre-lazy-loader agent-command code.

## What This Changes

The PR adds a forced skills snapshot refresh for local `openclaw agent` runs and extends `src/commands/agent.test.ts` to assert watcher and manual version-bump calls.

## Best Possible Solution

Keep the trusted local CLI manual snapshot-version bump, load it through current main's refresh-state boundary, and leave watcher lifecycle owned by long-lived gateway/session code.

## Reproduction Assessment

Yes. Source inspection plus the PR body's extraDirs/reused-session steps give a high-confidence path: a reused CLI session can keep a version-0 snapshot while live skill discovery sees a newly added extra-dir skill.

## Solution Assessment

No. The manual bump before snapshot comparison is the narrow fix, but starting `ensureSkillsWatcher` from `openclaw agent` changes process lifecycle and should be removed or replaced with explicit teardown.
