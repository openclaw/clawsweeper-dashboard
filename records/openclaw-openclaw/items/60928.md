## Summary

This should stay open as a focused bug-fix PR: current main still has the CLI persisted-snapshot reuse path, and the branch’s manual snapshot-version bump is aimed at the right source path. The patch still needs changes because it starts a persistent skills watcher from a one-shot CLI run, which conflicts with the documented command lifecycle.

## What This Changes

The PR adds a CLI-only skills snapshot refresh flag for `openclaw agent` and extends `src/commands/agent.test.ts` to assert watcher and manual version-bump calls.

## Best Possible Solution

Land a rebased CLI-only manual snapshot-version bump before the snapshot comparison, with focused regression coverage and no watcher startup in one-shot agent runs.

## Reproduction Assessment

Yes. Source inspection shows current main can reuse a persisted skills snapshot when the stored version still matches, and the PR body provides concrete `skills.load.extraDirs` plus fixed-session CLI steps with before/after output; I did not run the live CLI path in this read-only review.

## Solution Assessment

No. The manual version bump is the right narrow fix direction, but starting `ensureSkillsWatcher` from `openclaw agent` changes the one-shot process lifecycle and should be removed or paired with explicit teardown.
