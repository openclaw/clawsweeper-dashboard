## Summary

Keep open: the stale CLI skills-snapshot bug is credible and the manual version-bump direction is useful, but the current patch starts a persistent skills watcher from the one-shot CLI path and should not merge as-is.

## What This Changes

The PR makes CLI `openclaw agent` runs force a skills snapshot refresh and adds assertions in `src/commands/agent.test.ts`.

## Best Possible Solution

Add a CLI-only manual `bumpSkillsSnapshotVersion` through the lazy `refresh-state.js` seam before the snapshot version read, while keeping `ensureSkillsWatcher` owned by long-lived session paths.

## Reproduction Assessment

Yes. Current main source shows the cached snapshot reuse path, and the PR body gives fixed-session extra-dir CLI steps with before/after output; I did not run the live flow in this read-only review.

## Solution Assessment

No. The manual version bump is the right narrow fix direction, but starting `ensureSkillsWatcher` from the one-shot CLI agent path is not the best implementation.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Avoid starting the skills watcher from CLI runs:** `src/agents/agent-command.ts:503`
