## Summary

Keep open: the stale CLI skills snapshot bug is valid and the manual version bump is useful, but this patch also starts a persistent watcher from a one-shot CLI path, which can keep local agent runs alive after the reply.

## What This Changes

The PR adds a forced skills snapshot refresh to CLI agent runs and extends agent command tests to assert the refresh calls.

## Best Possible Solution

Land a rebased CLI-only manual skills snapshot version bump before cached snapshot comparison, without starting a persistent watcher in one-shot agent runs.

## Reproduction Assessment

Yes. Source inspection on current main shows cached skillsSnapshot reuse when the version is unchanged, and the PR body provides a fixed-session extraDirs reproduction with after-fix CLI output.

## Solution Assessment

No, not as written. The manual snapshot version bump is the narrow fix, but starting ensureSkillsWatcher from the CLI path conflicts with the one-shot local-run contract; use the existing refresh-state seam instead.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Skip the persistent watcher in one-shot agent runs:** `src/agents/agent-command.ts:503`
  - body: The added `ensureSkillsWatcher` call creates a chokidar watcher before the version read, but `openclaw agent --local` is documented as one-shot and this path does not close the watcher. With chokidar's default persistent watcher, the command can remain alive after printing the reply; keep the manual version bump but do not start the watcher from this CLI path.
  - confidence: 0.91
