## Summary

Keep open: the stale skills snapshot problem is valid, but this patch starts a persistent file watcher from a one-shot CLI path with no teardown, so it needs a narrow repair before merge.

## What This Changes

This PR adds a forced skills snapshot refresh to CLI `openclaw agent` runs and extends `src/commands/agent.test.ts` to assert the refresh calls.

## Best Possible Solution

Land a CLI-only manual skills snapshot version bump on current main while keeping watcher lifecycle ownership in the long-lived gateway/session path.

## Reproduction Assessment

Yes. Source inspection on current main shows cached `skillsSnapshot` reuse when the version is unchanged, and the PR body provides a fixed-session `skills.load.extraDirs` before/after CLI verification with after-fix live output.

## Solution Assessment

No. The manual snapshot version bump is the right direction, but the CLI path should use the existing refresh-state seam and avoid starting a persistent watcher in a one-shot run.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Avoid starting the persistent watcher in one-shot agent runs:** `src/agents/agent-command.ts:503`
  - body: The new `ensureSkillsWatcher` call creates a chokidar watcher before the version read, but the CLI `--local` path is documented as one-shot and this flow has no teardown for that watcher. Since chokidar defaults to `persistent: true`, `openclaw agent --local` can stay alive after printing the reply; keep the manual version bump, but do not start the watcher from this path.
  - confidence: 0.91
