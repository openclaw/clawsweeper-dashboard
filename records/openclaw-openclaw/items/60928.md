## Summary

Keep open: the stale CLI skills snapshot behavior is a valid, source-reproducible bug, but this patch starts a persistent skills watcher from the one-shot CLI path without teardown.

## What This Changes

The PR adds a CLI-only forced skills snapshot refresh path, starts the skills watcher, bumps the snapshot version for `openclaw agent`, and extends agent command tests.

## Best Possible Solution

Add a current-main CLI-only manual snapshot version bump before reading the version, leave watcher startup in long-lived session handling, and cover CLI plus ingress behavior in focused tests.

## Reproduction Assessment

Yes. Current main reuses persisted CLI session skill snapshots by version/filter, and the PR body gives a fixed-session `skills.load.extraDirs` before/after CLI verification path for the stale prompt behavior.

## Solution Assessment

No. The manual version bump is the right direction, but starting `ensureSkillsWatcher` from one-shot CLI agent runs is not the narrowest maintainable fix because watcher lifecycle ownership belongs to long-lived session code.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91
