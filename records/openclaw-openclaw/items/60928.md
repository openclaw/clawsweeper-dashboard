## Summary

This PR targets a real CLI stale-skills bug that current main still appears to have, but the proposed patch should not merge as-is because it starts a persistent watcher in the one-shot agent command path.

## What This Changes

The PR adds a CLI agent path flag that starts the skills watcher, bumps the skills snapshot version before snapshot reuse, and extends the agent command test assertions.

## Best Possible Solution

Land a rebased CLI-only manual skills snapshot version bump before cached snapshot comparison, without starting a persistent watcher in one-shot agent runs, and keep watcher lifecycle in long-lived session handling.

## Reproduction Assessment

Yes, by source inspection and the PR body's fixed-session extraDirs steps: current main can reuse a cached skillsSnapshot when the version has not changed, while openclaw skills list uses live discovery. I did not run the live CLI path in this read-only review.

## Solution Assessment

No. The manual version bump is the narrow maintainable direction, but starting ensureSkillsWatcher in the one-shot CLI path is not the best fix because that watcher has persistent lifecycle and no command teardown.
