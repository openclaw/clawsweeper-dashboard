## Summary

Keep open: the stale skills-snapshot bug is valid and the manual version bump is useful, but this branch also starts a persistent skills watcher in the one-shot CLI path and is now conflicting with current main's lazy refresh-state code.

## What This Changes

The PR adds a CLI force-refresh path for skills snapshots during `openclaw agent` runs and extends `src/commands/agent.test.ts` to assert the watcher and manual snapshot-version bump calls.

## Best Possible Solution

Land a rebased CLI-only manual snapshot-version bump before snapshot comparison, with focused regression coverage and no watcher startup in one-shot agent runs.

## Reproduction Assessment

Yes. Source inspection shows current main can reuse a persisted skills snapshot when the stored version still matches, and the PR body provides concrete `skills.load.extraDirs` plus fixed-session CLI steps; I did not run the live CLI path in this read-only review.

## Solution Assessment

No. The manual version bump is the right narrow direction, but starting `ensureSkillsWatcher` from `openclaw agent` changes one-shot process lifecycle and should be removed or replaced with explicit teardown.
