## Summary

Keep open: the stale CLI skills snapshot bug is plausible and the manual version bump is the right direction, but the patch also starts a persistent skills watcher in the one-shot CLI path and is stale against current main's lazy refresh-state seam.

## What This Changes

The PR adds a CLI-only forced skills snapshot refresh before cached snapshot reuse and extends agent command tests to assert the refresh calls.

## Best Possible Solution

Land a rebased CLI-only manual skills snapshot version bump before cached snapshot comparison, without starting a persistent watcher in one-shot agent runs.

## Reproduction Assessment

Yes. Source inspection shows current main can reuse a cached skillsSnapshot when the version stays unchanged, and the PR body provides a fixed-session extraDirs repro with before/after CLI output.

## Solution Assessment

No, not as written. The manual version bump is the narrow maintainable fix, but watcher startup belongs to long-lived session handling and current main's lazy refresh-state import boundary should be preserved.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Skip the persistent watcher in one-shot agent runs:** `src/agents/agent-command.ts:503`
  - body: The added call starts ensureSkillsWatcher before the snapshot version read. That helper stores a chokidar watcher in module state and only closes it on disabled-watch replacement, replacement, or test reset; with chokidar's default persistent watcher, `openclaw agent --local` can stay alive after printing the reply. Keep the manual version bump, but do not start an unclosed watcher from this one-shot path.
  - confidence: 0.91
