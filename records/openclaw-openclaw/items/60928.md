## Summary

Keep open, but the PR needs changes before merge. The stale CLI skills snapshot bug is credible and the PR body has sufficient real CLI proof, but the patch starts a persistent skills watcher from the one-shot CLI agent path; current main should get only the manual snapshot-version bump through the refresh-state seam.

## What This Changes

The PR adds a CLI-only skills snapshot refresh flag, starts the skills watcher, manually bumps the snapshot version for `openclaw agent`, and extends `src/commands/agent.test.ts` assertions.

## Best Possible Solution

Land a current-main CLI-only manual snapshot version bump before the version read, leave watcher startup in long-lived session code, and cover CLI plus ingress behavior in focused tests.

## Reproduction Assessment

Yes, source-reproducible. Current main reuses session skill snapshots by version/filter, and the PR body gives a fixed-session extra-dir before/after CLI verification path for the stale prompt behavior.

## Solution Assessment

No. The manual version bump is the right direction, but starting `ensureSkillsWatcher` from one-shot CLI agent runs is not the narrowest maintainable fix because that watcher has persistent module lifecycle ownership elsewhere.
