## Summary

Keep open: current main and the latest release already cover part of this through `messages.inbound` and Telegram forwarded-burst coalescing, but the requested DM/session-specific aggregation contract with explicit source-message metadata is still missing and needs maintainer direction.

## What This Changes

Request an optional Telegram DM debounce window that groups closely spaced same-sender messages into one ordered agent input bundle with original message metadata.

## Best Possible Solution

Define one canonical session-ingress aggregation contract, preferably extending the existing `messages.inbound` path, then add Telegram-specific behavior only where Telegram needs transport-owned handling.

## Reproduction Assessment

Not applicable: this is a feature request, not a current-main bug report. Source inspection shows partial existing batching behavior, but no failing reproduction path is required to understand the remaining request.

## Solution Assessment

No: the proposed Telegram-only `dmAggregationWindowMs` shape is only one plausible option and would overlap existing `messages.inbound` configuration. The safer direction is a maintainer-approved session-ingress bundle contract with explicit metadata semantics.
