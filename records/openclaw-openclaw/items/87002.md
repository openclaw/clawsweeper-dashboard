## Summary

Keep open: current main and v2026.6.8 already support fixed same-sender Telegram batching through generic `messages.inbound`, but the requested explicit bundle metadata, original message IDs/timestamps, and product choice around Telegram/session config remain missing.

## What This Changes

Requested an optional Telegram DM/session aggregation window that batches closely spaced same-sender messages into one ordered bundle with metadata.

## Best Possible Solution

Keep the existing fixed inbound debounce for basic batching; if maintainers want debuggable bundles, add one generic session-ingress bundle receipt/metadata contract used by Telegram rather than a parallel Telegram-only timing key.

## Reproduction Assessment

Not applicable: this is a feature request rather than a current-behavior bug. Source and docs confirm current main has fixed same-sender debounce, while explicit bundle metadata and original message ID/timestamp preservation are missing.

## Solution Assessment

No: the proposed Telegram-only config key overlaps the existing `messages.inbound.byChannel.telegram` timing knob. The safer solution is a maintainer-approved generic session-ingress bundle metadata contract, or docs clarifying that existing generic debounce is the supported timing shape.
