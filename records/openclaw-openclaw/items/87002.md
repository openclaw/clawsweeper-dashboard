## Summary

Keep open: current main and v2026.6.10 already provide fixed same-sender Telegram batching through generic inbound debounce, but the requested explicit bundle metadata, original message IDs/timestamps, and config/product shape are still missing.

## What This Changes

Request to add optional Telegram DM debounce/message aggregation for closely spaced same-sender messages, including bundle metadata and original message identifiers.

## Best Possible Solution

Keep the existing fixed inbound debounce for basic batching; if maintainers want debuggable bundles, add one generic session-ingress bundle metadata contract used by Telegram rather than a parallel Telegram-only timing key.

## Reproduction Assessment

Not applicable: this is a feature request rather than a current-behavior bug. Source and docs confirm current main has fixed same-sender debounce, while explicit bundle metadata and original message ID/timestamp preservation are missing.

## Solution Assessment

No: the proposed Telegram-only config key overlaps the existing messages.inbound.byChannel.telegram timing knob. The safer solution is a maintainer-approved generic session-ingress bundle metadata contract, or docs clarifying that existing generic debounce is the supported timing shape.
