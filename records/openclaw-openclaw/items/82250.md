## Summary

Keep open: current main and v2026.5.20 still generate boolean macOS `KeepAlive=true`, and the launchd duplicate-gateway path still returns successfully after detecting a healthy existing gateway. The latest reporter comment adds current-stable live evidence, while the related closed PRs did not land and the open launchd PR covers an overlapping but not identical lifecycle policy.

## What This Changes

The issue asks OpenClaw to prevent launchd from repeatedly respawning a gateway process that cleanly exits after detecting an already-running healthy gateway.

## Best Possible Solution

Implement a launchd lifecycle fix that distinguishes healthy duplicate-gateway bow-out from intentional restart/update handoffs, then update plist generation, service audit/doctor behavior, regression tests, docs, and live macOS LaunchAgent proof together.

## Reproduction Assessment

Yes, source-backed and supported by live reporter logs: current main writes boolean `KeepAlive=true`, launchd duplicate-gateway detection returns successfully, and v2026.5.20 reportedly respawns roughly every throttle interval. I did not run a live macOS LaunchAgent repro in this review.

## Solution Assessment

Unclear as currently proposed. `SuccessfulExit=false` fixes the observed clean-exit loop, but the maintainable solution must also preserve or replace the existing clean-exit launchd restart/update handoff semantics.
