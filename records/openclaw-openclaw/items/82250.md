## Summary

Keep open: current main and v2026.5.12 still generate unconditional macOS `KeepAlive=true`, and the launchd duplicate-gateway path returns cleanly after detecting a healthy existing gateway, so the reported respawn loop is source-backed. Related launchd PRs are open and point to an unresolved lifecycle-policy tradeoff rather than a shipped fix.

## What This Changes

Request that macOS LaunchAgent startup stop respawning after a clean exit caused by detecting an already-running healthy gateway on the configured port.

## Best Possible Solution

Add a focused launchd lifecycle fix that prevents healthy-duplicate gateway exits from respawning forever while preserving intentional update/restart handoffs and bounded startup-failure behavior.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes, source-backed: start a foreground gateway, load the macOS LaunchAgent on the same port, and current code exits cleanly after the healthy duplicate check while launchd's boolean `KeepAlive=true` respawns it.

## Solution Assessment

Is this the best way to solve the issue? Unclear as proposed: `SuccessfulExit=false` stops this loop, but it also changes clean-exit relaunch behavior used by update/restart handoffs; a safer fix may need a canonical KeepAlive policy plus distinct duplicate-gateway exit semantics.
