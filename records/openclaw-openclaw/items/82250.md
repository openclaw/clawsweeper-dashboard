## Summary

Keep open: current main and v2026.5.12 still generate unconditional macOS `KeepAlive=true`, and the launchd duplicate-gateway path returns successfully after detecting a healthy existing gateway. The reported loop is source-backed, while the linked `SuccessfulExit=false` direction still needs lifecycle-policy review because current update/restart handoffs also rely on clean-exit launchd relaunches.

## What This Changes

The issue requests changing macOS LaunchAgent lifecycle behavior so a clean duplicate-gateway exit is not repeatedly respawned by launchd.

## Best Possible Solution

Add a focused launchd lifecycle fix that prevents healthy-duplicate gateway exits from respawning forever while preserving intentional update/restart handoffs, explicit stop behavior, crash recovery, and bounded startup-failure handling.

## Reproduction Assessment

Yes, source-backed: start a foreground gateway, load the macOS LaunchAgent on the same port, and current code returns cleanly after the healthy duplicate check while the generated boolean `KeepAlive=true` policy tells launchd to respawn it. I did not run live macOS launchd in this read-only review.

## Solution Assessment

Unclear as proposed: `SuccessfulExit=false` stops this clean-exit loop, but current launchd update/restart handoffs also exit 0 and rely on KeepAlive relaunches. A safer fix needs explicit lifecycle-policy review or distinct duplicate-gateway exit semantics.
