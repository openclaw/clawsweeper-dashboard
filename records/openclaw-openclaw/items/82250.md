## Summary

Keep open: current main and v2026.5.12 still generate unconditional macOS `KeepAlive=true`, and the launchd duplicate-gateway path returns successfully after a healthy existing gateway is found, so the reported respawn loop remains source-backed. The newer related launchd PR targets nonzero startup-failure loops with `SuccessfulExit=true`, which does not resolve this clean-exit duplicate case.

## What This Changes

The issue requests that macOS LaunchAgent startup avoid repeated launchd respawns when a clean duplicate-gateway exit leaves an already-running healthy gateway in control.

## Best Possible Solution

Add a focused launchd lifecycle fix that stops healthy duplicate-gateway clean exits from respawning forever while preserving intentional update/restart handoffs and teaching service audit/doctor the canonical policy.

## Reproduction Assessment

Yes. Source inspection shows current main emits boolean `KeepAlive=true`, exits cleanly for a healthy launchd duplicate, and launchd keeps boolean-keepalive jobs alive; the reporter also provided live 2026.5.12 output for the workaround, though I did not run a live macOS smoke here.

## Solution Assessment

Unclear as proposed. `SuccessfulExit=false` addresses this clean-exit churn in the reporter's live workaround, but a maintainable upstream fix must also preserve clean-exit restart/update handoffs or make duplicate-gateway exits distinguishable from intentional supervisor handoffs.
