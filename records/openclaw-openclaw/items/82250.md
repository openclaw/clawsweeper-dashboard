## Summary

Keep open: current `main` and latest release still generate unconditional macOS `KeepAlive=true`, while the launchd duplicate-gateway path returns cleanly after detecting a healthy existing gateway. The reporter’s follow-up strengthens the workaround evidence, but the exact `SuccessfulExit=false` fix needs lifecycle-policy review because that policy previously broke intentional clean restart handoffs.

## What This Changes

The issue asks OpenClaw to prevent macOS launchd from repeatedly respawning a cleanly exiting gateway when another healthy gateway already owns the configured port.

## Best Possible Solution

Define one canonical launchd lifecycle policy that stops healthy duplicate-gateway exits from respawning while preserving intentional clean update/restart handoffs, crash recovery expectations, and service audit/doctor repair guidance.

## Reproduction Assessment

Yes, source-backed: current `main` writes boolean `KeepAlive=true`, the launchd duplicate-healthy path returns cleanly, and launchd unconditional keepalive restarts exited jobs. I did not run a live macOS LaunchAgent smoke in this read-only review.

## Solution Assessment

Unclear as proposed: `SuccessfulExit=false` is validated by the reporter for this loop, but prior OpenClaw history shows it breaks intentional clean restart handoffs. The safer solution is to coordinate plist policy, duplicate-gateway exit semantics, audit, and live macOS proof together.
