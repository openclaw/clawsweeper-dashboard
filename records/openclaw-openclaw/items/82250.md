## Summary

Keep open: current main and v2026.6.8 still generate unconditional macOS LaunchAgent KeepAlive, the healthy launchd duplicate-start path still returns cleanly after “leaving it in control,” and the open related audit PR does not fix the generated plist or runtime loop.

## What This Changes

The issue reports that macOS LaunchAgent KeepAlive=true respawns a gateway process after it cleanly exits because another healthy gateway already owns the configured port.

## Best Possible Solution

Choose and land a maintainer-approved launchd lifecycle contract that stops healthy duplicate respawn loops, aligns generated plist, runtime exit or handoff behavior, and service audit, with focused tests and live macOS proof.

## Reproduction Assessment

Yes at source level, but not live-reproduced in this review. Current main generates KeepAlive=true, the launchd duplicate path returns after “leaving it in control,” and reporter comments show launchctl runs climbing until SuccessfulExit=false stabilizes the service.

## Solution Assessment

Unclear as a final implementation choice. SuccessfulExit=false has independent live workaround evidence, but maintainers need to choose whether plist policy, runtime exit handling, or a handoff/idle contract is the safest long-term behavior.
