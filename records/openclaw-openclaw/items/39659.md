## Summary

Keep open. Current main and v2026.6.6 provide manual Docker environment and Compose primitives plus one-project ClawDock helpers, but there is no first-class `openclaw instance` lifecycle; the narrower sandbox-name collision remains related but not a full supersession at https://github.com/openclaw/openclaw/issues/51363.

## What This Changes

Add first-class Docker multi-instance lifecycle commands for creating, listing, updating, and deleting isolated OpenClaw instances on one host.

## Best Possible Solution

Design a minimal core Docker instance lifecycle that reuses existing env/Compose primitives, records safe per-instance state, and explicitly handles cleanup, update, port allocation, and sandbox isolation before implementation.

## Reproduction Assessment

Not applicable as a feature request. Source and docs confirm current main exposes manual Docker env/Compose primitives and no high-level instance lifecycle to reproduce as broken existing behavior.

## Solution Assessment

Unclear until maintainer product design. Manual env/Compose overrides and ClawDock are useful primitives, but the best final solution needs an approved CLI/state/delete/update design rather than a one-off wrapper.
