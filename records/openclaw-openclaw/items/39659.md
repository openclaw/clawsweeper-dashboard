## Summary

Keep open. Current main still supports manual profile/port isolation, ClawDock helper commands, and `--container` exec targeting, but it does not provide the requested named Docker instance lifecycle with managed metadata, safe port/path allocation, Compose project isolation, or create/list/delete/update commands.

## What This Changes

Request first-class Docker multi-instance lifecycle commands for named OpenClaw instances, including isolated persistent directories, ports, Compose/container names, updates, and cleanup.

## Best Possible Solution

Add or explicitly decline a supported Docker instance lifecycle after maintainer design review. A maintainable implementation would record per-instance metadata, reserve or validate unique host ports and paths, isolate Compose project/container names, provide safe create/list/update/delete behavior with confirmation or dry-run for destructive actions, and update Docker/ClawDock docs plus targeted tests.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is source and docs inspection on current main showing only manual profile/port/container-targeting surfaces and no named Docker instance lifecycle.

## Solution Assessment

Unclear. The proposed `openclaw instance` direction is plausible, but the best solution should be designed around existing profile, Docker setup, ClawDock, Podman, and gateway service primitives before implementation is queued.
