## Summary

Keep open. Current main and the latest release have manual profile/env/Compose/ClawDock primitives for same-host Docker and gateway isolation, but no first-class `openclaw instance` create/list/delete/update lifecycle; the related sandbox collision at https://github.com/openclaw/openclaw/issues/51363 is narrower and does not supersede the broader product request.

## What This Changes

The issue requests first-class Docker multi-instance lifecycle commands with isolated persistent directories, compose/container names, host ports, and safe create/list/delete/update workflows.

## Best Possible Solution

Design a minimal Docker instance lifecycle on top of existing profile/env/Compose primitives, with explicit instance identity, port assignment, update/delete safety, and sandbox-isolation semantics approved before implementation.

## Reproduction Assessment

Not applicable as a feature request. Source and docs show current main exposes manual profile/env/Compose primitives and ClawDock helpers, but no first-class Docker instance lifecycle to reproduce as broken existing behavior.

## Solution Assessment

Unclear until maintainer product design. The requested direction is plausible, but the best final shape needs approved CLI/state/delete/update semantics and coordination with sandbox instance identity rather than a one-off wrapper.
