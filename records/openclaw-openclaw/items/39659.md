## Summary

Keep open. Current main and the latest release provide manual profile/state-dir/Compose isolation plus ClawDock helpers, but they still do not provide the requested first-class Docker instance lifecycle.

## What This Changes

The issue requests first-class Docker multi-instance lifecycle commands for creating, listing, deleting, and updating isolated OpenClaw instances on one host.

## Best Possible Solution

Design a maintainer-approved Docker instance lifecycle on top of existing profiles, state dirs, and Compose, with explicit identity, port allocation, safe deletion/update semantics, and sandbox-isolation coordination.

## Reproduction Assessment

Not applicable as a feature request. Source and docs checks confirm manual isolation primitives exist, but the requested first-class Docker lifecycle command surface is absent.

## Solution Assessment

Unclear until maintainer product design. The direction is plausible, but it should not land as an ad hoc Compose wrapper without owning identity, persistence, cleanup, update, and sandbox-boundary semantics.
