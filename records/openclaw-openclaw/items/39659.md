No open or merged canonical replacement was found, and defining registry, update, port-allocation, and cleanup behavior requires a maintainer product decision.

## What This Changes

Requests first-class Docker commands to create, list, update, and delete isolated OpenClaw instances on one host.

## Best Possible Solution

Design an explicit Docker-instance lifecycle around existing named-profile and path-isolation primitives, with operator-visible port selection, durable registry ownership, collision detection, clearly separated image/config updates, and guarded deletion that names every persistent resource affected.

## Reproduction Assessment

Not applicable: current main documents a working manual profile-based isolation path, while this issue requests a new managed Docker lifecycle rather than reporting broken defined behavior.

## Solution Assessment

No, the command list alone is not yet the best complete solution; it is a useful UX outline, but maintainers must first define registry ownership, derived-port policy, update boundaries, and safe persistent-state deletion while reusing existing profiles.
