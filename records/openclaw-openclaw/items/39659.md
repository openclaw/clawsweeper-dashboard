## Summary

Keep open. Current main and v2026.6.10 support manual multi-gateway isolation through profiles, state dirs, env/Compose variables, and ClawDock helpers, but they still do not provide the requested first-class Docker instance create/list/delete/update lifecycle; related items cover narrower sandbox, config-schema, cron, or profile-targeting work.

## What This Changes

The issue requests first-class Docker install commands for creating, listing, deleting, and updating multiple OpenClaw instances on one host.

## Best Possible Solution

Design a maintainer-approved Docker instance lifecycle on top of existing profiles, state dirs, and Compose, with explicit instance identity, port allocation, safe delete/update semantics, and coordination with sandbox isolation fixes.

## Reproduction Assessment

Not applicable as a feature request. Current source and docs show manual isolation primitives, but no first-class Docker instance lifecycle command surface.

## Solution Assessment

Unclear until maintainer product design. The direction is plausible, but it should not land as an ad hoc Compose wrapper without owning identity, persistence, cleanup, update, and sandbox-boundary semantics.
