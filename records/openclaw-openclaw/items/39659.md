## Summary

Keep open: current main has Docker setup knobs, ClawDock single-project helpers, and manual multi-gateway/profile isolation docs, but it still lacks first-class Docker instance create/list/delete/update lifecycle management. The related open reports at https://github.com/openclaw/openclaw/issues/51363 and https://github.com/openclaw/openclaw/issues/71216 cover narrower sandbox-name collision and broader config-schema design, not this complete Docker lifecycle surface.

## What This Changes

The issue requests first-class Docker multi-instance lifecycle commands for creating, listing, updating, and deleting isolated OpenClaw instances on one host.

## Best Possible Solution

Define the official Docker instance model and safety rules first, then implement the selected lifecycle surface across CLI registration, Docker setup, Compose handling, docs, and focused fresh-install plus upgrade coverage.

## Reproduction Assessment

Not applicable; this is a feature request for a new Docker instance lifecycle, not broken existing documented behavior. Source and docs review shows manual Docker and multi-gateway isolation primitives but no named lifecycle to reproduce against.

## Solution Assessment

Unclear; the proposed command family is plausible, but maintainers should choose the registry, port allocation, Compose naming, update/recreate, and deletion semantics before implementation.
