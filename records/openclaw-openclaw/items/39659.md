## Summary

Keep open: current main has manual multi-gateway/profile primitives and Docker env overrides, but it does not provide the requested first-class Docker instance lifecycle. This is a real feature/product-surface decision rather than stale cleanup or an already-implemented bug fix.

## What This Changes

The issue requests first-class Docker multi-instance lifecycle commands for creating, listing, deleting, and updating isolated OpenClaw instances.

## Best Possible Solution

Define a supported Docker instance model first, then implement it in the chosen install surface with explicit per-instance directories, compose project names, port allocation, update/recreate behavior, and safe deletion semantics.
