## Summary

Keep open. Current main and v2026.6.8 still provide manual profile/env/Compose/ClawDock primitives for same-host isolation, but not the requested first-class `openclaw instance create/list/delete/update` lifecycle; the related open issues only partially overlap.

## What This Changes

Requests first-class Docker multi-instance lifecycle commands for create/list/delete/update with isolated directories, compose names, host ports, update flow, and cleanup.

## Best Possible Solution

Design a maintainer-approved Docker instance lifecycle on top of profiles, state dirs, Compose env, and ClawDock, with explicit identity, port allocation, safe delete/update behavior, and clear sandbox-isolation boundaries.

## Reproduction Assessment

## Solution Assessment
