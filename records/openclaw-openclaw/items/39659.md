## Summary

Keep open. Current main and v2026.6.6 provide manual profile/env/Compose/ClawDock primitives for same-host Docker or gateway isolation, but they still do not provide the requested first-class `openclaw instance create/list/delete/update` lifecycle.

## What This Changes

The issue requests first-class Docker multi-instance lifecycle commands with per-instance persistence, Compose/container naming, port assignment, update, and cleanup behavior.

## Best Possible Solution

Design a minimal Docker instance lifecycle on top of profiles, state dirs, and Compose env, with explicit instance identity, port allocation, safe delete/update behavior, and sandbox-isolation coordination.

## Reproduction Assessment

Not applicable as a feature request. Source and docs show current main has manual profile/env/Compose/ClawDock primitives, but no first-class Docker instance lifecycle to reproduce as broken existing behavior.

## Solution Assessment

Unclear until maintainer product design. The requested direction is plausible, but the best final shape needs approved CLI/state/delete/update semantics and coordination with existing profile/env primitives rather than a one-off wrapper.
