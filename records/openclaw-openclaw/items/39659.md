## Summary

Keep this open. Current main still supports manual profile, port, Docker Compose, and ClawDock helper workflows, but it does not provide the requested named Docker instance lifecycle; this needs maintainer product design rather than conservative cleanup.

## What This Changes

Request first-class Docker multi-instance lifecycle commands such as create, list, delete, and update with isolated directories, names, ports, and cleanup.

## Best Possible Solution

Design or explicitly decline a supported Docker instance lifecycle around existing profile, port, Compose, and ClawDock primitives, with metadata, collision checks, safe destructive flows, docs, and targeted tests.

## Reproduction Assessment

Not applicable as a feature request. Source and docs inspection on current main shows only manual profile/port isolation plus Docker/ClawDock helper flows, not a named Docker instance lifecycle.

## Solution Assessment

Unclear as an implementation path. The proposed openclaw instance command shape is plausible, but maintainers should first decide the lifecycle model, metadata location, allocation policy, and cleanup safety rules.
