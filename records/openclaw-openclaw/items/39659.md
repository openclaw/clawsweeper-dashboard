## Summary

Keep open. Current main and v2026.6.6 provide manual multi-gateway/profile/env isolation plus Docker and ClawDock helper primitives, but there is still no first-class `openclaw instance` create/list/delete/update lifecycle; the related sandbox collision at https://github.com/openclaw/openclaw/issues/51363 is open and narrower, not a full supersession.

## What This Changes

Add first-class Docker multi-instance lifecycle commands with isolated persistent directories, Compose/container naming, port assignment, updates, and cleanup.

## Best Possible Solution

Define an official Docker instance lifecycle that reuses the existing profile/env/Compose primitives, persists clear instance identity, and coordinates sandbox naming before implementation.
