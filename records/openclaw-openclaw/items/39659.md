## Summary

Keep open. Current main and v2026.6.10 still provide manual profile/state-dir/Compose isolation plus ClawDock helpers, but they do not provide the requested first-class Docker instance create/list/delete/update lifecycle; the related open items cover narrower sandbox or config-schema pieces.

## What This Changes

Request first-class Docker multi-instance lifecycle commands that isolate config, state, workspaces, Compose/container names, ports, updates, and deletion for co-hosted OpenClaw installs.

## Best Possible Solution

Design a maintainer-approved Docker instance lifecycle on top of existing profiles, state dirs, and Compose, with explicit instance identity, port allocation, safe delete/update semantics, and coordination with sandbox isolation fixes.

## Reproduction Assessment

Not applicable as a feature request. Source and docs checks confirm manual isolation primitives exist, but the requested first-class Docker lifecycle command surface is absent.

## Solution Assessment

Unclear until maintainer product design. The direction is plausible, but it should not land as an ad hoc Compose wrapper without owning identity, persistence, cleanup, update, and sandbox-boundary semantics.
