## Summary

Keep open. Current main and v2026.6.10 still provide manual profile/env/Compose/ClawDock primitives for same-host isolation, but not the requested first-class Docker `openclaw instance create/list/delete/update` lifecycle; the related open issues only partially overlap.

## What This Changes

The issue requests first-class Docker multi-instance lifecycle commands for create/list/delete/update with isolated directories, Compose/container names, ports, and safe update/delete cleanup.

## Best Possible Solution

Design a maintainer-approved Docker instance lifecycle on top of profiles, state dirs, Compose env/project naming, and ClawDock, with explicit instance identity, port allocation, safe update/delete behavior, and documented sandbox-isolation boundaries.

## Reproduction Assessment

Not applicable as a feature request. Source and docs confirm current main lacks a first-class Docker instance lifecycle, so there is no broken existing behavior to reproduce.

## Solution Assessment

Unclear until maintainer product design. The requested direction is plausible, but the best final shape needs approved CLI/state/delete/update semantics and coordination with existing profile/env/Compose primitives rather than a one-off wrapper.
