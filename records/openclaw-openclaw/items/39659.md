## Summary

Keep open. Current main and v2026.6.10 still provide profiles, env vars, Compose, and ClawDock-style helpers for manual isolation, but they do not provide the requested first-class Docker instance lifecycle; the related open issues only cover narrower sandbox naming, profile targeting, or config-schema pieces.

## What This Changes

Request first-class Docker install commands for creating, listing, updating, and deleting isolated OpenClaw instances on one host.

## Best Possible Solution

Design a maintainer-approved lifecycle layer on top of profiles, state dirs, and Compose with explicit instance identity, safe cleanup/update behavior, port allocation, and coordination with sandbox-isolation fixes.

## Reproduction Assessment

Not applicable as a feature request. Source and docs checks confirm manual isolation primitives exist, but the requested first-class Docker lifecycle command surface is absent.

## Solution Assessment

Unclear until maintainer product design. The direction is plausible, but it should not land as a thin Compose wrapper without owning identity, persistence, cleanup, update, and sandbox-boundary semantics.
