## Summary

Keep open. Current main and v2026.6.6 still lack a first-class `openclaw instance` Docker lifecycle; existing Docker env/Compose primitives and ClawDock help with one project but do not provide named create/list/delete/update management, and the related sandbox collision issue remains narrower rather than a full supersession.

## What This Changes

Requests official Docker multi-instance lifecycle commands for named instances, including isolated directories, compose/container naming, port assignment, update, and cleanup.

## Best Possible Solution

Design a minimal Docker instance lifecycle that reuses the existing env/Compose primitives, persists clear instance identity, and defines safe update, delete, port, and sandbox-isolation semantics before implementation.

## Reproduction Assessment

Not applicable as a feature request. Source and docs confirm current main exposes manual Docker env/Compose primitives and no high-level instance lifecycle to reproduce as broken existing behavior.

## Solution Assessment

Unclear until maintainer product design. Manual env/Compose overrides and ClawDock are useful primitives, but the best final solution needs an approved CLI/state/delete/update design rather than a one-off wrapper.
