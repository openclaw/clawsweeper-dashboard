## Summary

Keep open: current main and v2026.6.5 support manual multi-gateway isolation through profiles, separate state paths, and ports, but they do not provide the requested Docker instance create/list/update/delete lifecycle. The remaining work is a product decision around Docker orchestration, registry ownership, derived-port allocation, updates, and destructive cleanup.

## What This Changes

Request first-class Docker install commands to create, list, update, and delete isolated OpenClaw instances with per-instance directories, Compose/container naming, and host ports.

## Best Possible Solution

Design a bounded Docker-instance lifecycle that reuses existing profile/path isolation, stores any registry in an explicit SQLite-owned state surface, validates derived-port collisions, separates image updates from config migrations, and makes deletion name every persistent resource it will affect.

## Reproduction Assessment

Not applicable: this is a feature request, not a report of broken defined behavior. Current main documents manual profile/state/port isolation, while the requested managed Docker lifecycle remains absent.

## Solution Assessment

No, the command list is a useful UX outline but not yet a complete best solution. Maintainers need to decide ownership, registry storage, derived-port policy, update boundaries, and safe deletion semantics before implementation.
