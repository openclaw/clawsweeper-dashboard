Full review comments:

- **[P1] Gate scheduled live evals behind an explicit opt-in:** `.github/workflows/agent-role-evals.yml:63`
  - body: The schedule branch currently runs whenever `vars.OPENCLAW_AGENT_ROLE_EVAL_LIVE != '0'`. On a repository without that variable set, the missing context property evaluates to an empty string, so the daily schedule still enters the live job, hydrates live auth secrets, starts/pulls Ollama, runs evals, and uploads logs. That contradicts the PR's stated default of deterministic catalog checks unless live settings opt in; make the schedule require an explicit enabled value such as `== '1'` while keeping `workflow_dispatch` controlled by `run_live`. ([docs.github.com](https://docs.github.com/en/actions/reference/contexts-reference))
  - confidence: 0.89
- **[P2] Use the canonical OpenClaw config path:** `scripts/lib/agent-role-evals.mjs:1115`
  - body: The new default `agents:eval` path is `~/.openclaw/openclaw.director.json`, but current OpenClaw uses `~/.openclaw/openclaw.json` unless `OPENCLAW_CONFIG_PATH` overrides it. Running the new package script without `--config` on a normal install will fail with a missing file instead of evaluating the user's active config; derive from the current config helper or at least default to `~/.openclaw/openclaw.json`.
  - confidence: 0.86
