## Best Possible Solution

Rebase the empty-MIME image fallback onto current main's payload-store attachment flow, keep focused regression coverage, and add redacted real Control UI proof that an empty-MIME image attaches and sends.

## Reproduction Assessment

Yes, at source level: current main stores empty-type dropped/selected files as application/octet-stream and send serialization depends on the data URL MIME, so an empty-MIME image filename is not preserved as image/png. I did not run a live browser drag/drop session in this read-only review.

## Solution Assessment

Yes in direction, but no as currently rebased: an extension fallback only for missing MIME types is the narrow maintainable fix, but it must preserve current main's payload-store attachment flow.
