## What This Changes

The PR adds a Control UI chat attachment MIME resolver with image-extension fallback for missing browser MIME types, wires it into selected/dropped attachment handling and send serialization, adds focused UI/controller tests, and adds a changelog entry.

## Best Possible Solution

Land the missing-MIME image fallback on top of current main while keeping attachment bytes in attachment-payload-store, preserving preview/send serialization, focused regression tests, the changelog entry, and contributor credit.

Yes. Current main gives a code-level reproduction: an empty-type File named photo.png passes file filtering, is stored as application/octet-stream, and chat.send serialization does not recover image/png from the filename or attachment metadata.

## Solution Assessment
