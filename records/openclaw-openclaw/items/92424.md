## Summary

This PR targets a real model-switch regression, but the helper resolves the registry using the stale snapshot’s provider and model ID, so switching to a different model still returns the previous model. The tests duplicate the helper outside AgentSession and cannot detect that failure.

## What This Changes

The PR changes eight AgentSession post-turn reads to consult the model registry using the snapshot model key and adds isolated tests for a copied lookup helper.

## Best Possible Solution

Update canonical AgentSession model identity from the selected session override at the model-switch boundary, then prove the actual post-turn consumers against the new provider and model.

## Reproduction Assessment

Yes. Apply a session override to a different provider or model while `agent.state.model` retains the old snapshot; the proposed helper deterministically queries the old registry key.

## Solution Assessment

No. Refreshing registry metadata for the stale identity cannot adopt the newly selected model; the canonical model-switch boundary must update or carry the active identity.
