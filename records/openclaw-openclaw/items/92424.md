## Summary

This PR does not fix the linked regression because its registry lookup is still keyed by AgentSession’s unchanged previous model. The implementation needs correction and real post-fix behavior proof before merge.

## What This Changes

The PR makes eight AgentSession post-turn model reads consult a new registry-resolution helper and adds isolated tests for that helper.

## Best Possible Solution

Synchronize AgentSession with the authoritative active provider/model selection at the model-switch owner boundary, then test a lifecycle that changes both keys and verifies every affected post-turn decision uses the new model.

## Reproduction Assessment

Yes, from source inspection: the override path can change the persisted provider/model key without updating AgentSession’s snapshot, while the patch still queries the registry with that old key. A live reproduction was not independently run.

## Solution Assessment

No. Refreshing registry metadata for the stale provider/id cannot select the newly switched model when either key changes; synchronization must start from the authoritative active selection.
