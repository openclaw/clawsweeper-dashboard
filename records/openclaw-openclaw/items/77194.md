## Summary

Keep open: the PR targets a real bundled-provider allowlist gap, but the latest head still has a source-reproducible master-toggle regression and is draft with the protected `maintainer` label.

## What This Changes

The PR adds `plugins.bundledDiscovery`, gates bundled provider discovery through `plugins.allow` by default, migrates legacy restrictive allowlists to `compat`, and updates provider, web-search, doctor, schema, docs, and changelog coverage.

## Best Possible Solution

Update the PR so allowlist mode preserves `plugins.enabled: false`, add disabled-master-toggle regression coverage, then let maintainers decide whether this new config policy should land.

## Reproduction Assessment

Yes. Source inspection of PR head shows `withBundledPluginEnablementCompat` can still return `plugins.enabled: true` from a config that set `plugins.enabled: false` when allowlist discovery is enabled.

## Solution Assessment

No as written. The `allowlist`/`compat` policy is plausible, but the helper should gate the global-enable compatibility override to explicit compat mode and add coverage for the disabled master toggle.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88
