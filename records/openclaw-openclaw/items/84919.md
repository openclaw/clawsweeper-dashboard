## What This Changes

Reported that Gemini cached-content requests with `cacheRetention` and tools can send `cachedContent` together with `systemInstruction`, `tools`, and `toolConfig`, causing Google 400 errors.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/84958 or an equivalent current-main fix, then close this issue as implemented.

## Reproduction Assessment

Yes at source level: current main can build a Gemini GenerateContent payload that combines `cachedContent` with request-level `systemInstruction`, `tools`, and `toolConfig`, and the managed wrapper can patch `cachedContent` after leaving tools in the downstream context. I did not run a live failing Google request during this read-only review.

## Solution Assessment

Yes. The current open PR is the narrow maintainable path because it covers both explicit `params.cachedContent` and managed `cacheRetention` entry points, while keeping uncached Gemini requests unchanged.
