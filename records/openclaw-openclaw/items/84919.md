## What This Changes

The issue requests fixing Gemini cached-content requests so OpenClaw does not send request-level system instructions or tool settings alongside `cachedContent`.

## Best Possible Solution

Apply the equivalent of the closed related PR on current main: omit request-level prompt and tool settings whenever Gemini `cachedContent` is present, clean the managed prompt-cache payload patch, and cover both direct and managed-cache paths.

## Reproduction Assessment

Yes for the payload-construction path: current main visibly emits the conflicting `cachedContent` plus request-level prompt/tool fields, and existing tests still expect that combination. A live Google API call was not run during this read-only review.

## Solution Assessment

Yes, the narrow maintainable fix is to keep `cachedContent` and suppress the incompatible request-level Google fields in both direct and managed-cache paths. This matches the closed unmerged PR and does not require a product decision.
