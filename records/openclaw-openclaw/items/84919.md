## Summary

Keep open: current main still constructs Gemini requests that can combine `cachedContent` with request-level `systemInstruction`, `tools`, and `toolConfig`, and the latest release has the same behavior. The closed related PR contains the right narrow fix shape, but that commit is not on main or in a shipped tag.

## What This Changes

The issue requests that direct Gemini cached-content requests omit `systemInstruction`, `tools`, and `toolConfig` when `cacheRetention` or explicit `cachedContent` is in use.

## Best Possible Solution

Apply the equivalent of https://github.com/openclaw/openclaw/pull/71441 on current main: guard Google transport prompt/tool fields when cached content is present, strip conflicting fields from managed prompt-cache payload patches, and add regression coverage for both paths.

## Solution Assessment
