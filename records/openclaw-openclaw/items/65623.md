## Summary

Keep open. Current main still renders the raw parsed config for both full and path-scoped `/config show` replies, while the gateway config API already uses schema-aware redaction; the open closing PR #65637 has not merged, so this should remain open for maintainer/security review.

## What This Changes

Redact secret values from `/config show` chat replies, including both full-config and path-scoped output.

## Best Possible Solution

Ship a maintainer-approved patch that redacts the cloned parsed config before any `/config show` rendering, covers both full and path-scoped replies with regression tests, and then close this after the fix is merged and release provenance is known.

## Reproduction Assessment

Yes. A high-confidence source reproduction is available: mock `readConfigFileSnapshot()` with a parsed sensitive value such as `gateway.auth.token`, call `handleConfigCommand` for `/config show` or `/config show gateway.auth.token`, and observe the current code stringifies the raw value into the reply.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse the existing schema-aware config redaction path for chat-command output before path lookup and full JSON rendering, matching `config.get` rather than adding a separate ad hoc masking implementation.
