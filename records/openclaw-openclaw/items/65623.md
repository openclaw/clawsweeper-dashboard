## What This Changes

The issue asks for `/config show` chat replies to redact sensitive config values before rendering full or path-scoped JSON.

## Best Possible Solution

Land the linked PR or an equivalent maintainer-approved patch that applies schema-aware config redaction before both full and path-scoped `/config show` rendering, then add focused regression coverage.

Yes. Reusing the existing schema-aware config redaction helper before lookup and rendering is the narrowest maintainable fix and matches the gateway `config.get` behavior.
