## Summary

Keep open. Current main and the latest release still clone the raw parsed config and stringify it for full and path-scoped `/config show` replies, while the same config surface is already redacted in the gateway API; the same-author closing PR at https://github.com/openclaw/openclaw/pull/65637 is still open.

## What This Changes

The issue asks for `/config show` chat replies to redact sensitive config values for both full-config and path-scoped output.

## Best Possible Solution

Ship a maintainer-approved patch that applies schema-aware redaction before both full and path-scoped `/config show` rendering, with focused regression tests, then close after merge and release/main provenance is known.

## Reproduction Assessment

Yes. Mocking `readConfigFileSnapshot()` with a parsed secret and invoking `handleConfigCommand` for `/config show` or `/config show gateway.auth.token` will stringify the raw value on current main.

## Solution Assessment

Yes. Reusing the existing config redaction helper with schema UI hints before lookup/rendering is the narrowest maintainable fix and matches the established `config.get` contract.
