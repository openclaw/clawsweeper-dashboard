## Summary

Keep open. Current main and the latest release still serialize the raw parsed config for full and path-scoped `/config show` chat replies, while the same config surface is already redacted in the gateway API; the same-author closing PR at https://github.com/openclaw/openclaw/pull/65637 is still open.

## What This Changes

The issue reports that owner-triggered `/config show` chat replies serialize raw config data and can expose stored secrets in channel history.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/65637 or an equivalent maintainer-approved patch that applies schema-aware redaction before both full and path-scoped `/config show` rendering, with focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: mock `readConfigFileSnapshot()` with a parsed secret and invoke `handleConfigCommand` for `/config show` or `/config show gateway.auth.token`; current main stringifies the raw value.

## Solution Assessment

Yes. Reusing the existing schema-aware config redaction helper before lookup/rendering is the narrowest maintainable fix and matches the established `config.get` contract.
