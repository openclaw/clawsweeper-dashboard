## Summary

Keep open. Current main and the latest release seed hardcoded Slack assistant suggested prompts when an assistant thread starts, but the requested configurable `openclaw.json` surface and dynamic message-tool action are still missing, and the related merged Slack lifecycle work explicitly left this issue as the narrower follow-up.

## What This Changes

The issue requests Slack support for assistant thread suggested prompts, including configurable default prompts and an optional dynamic agent-controlled prompt action.

## Best Possible Solution

Treat the current hardcoded defaults as partial support, then decide whether the remaining behavior should be a Slack-plugin config option, a Slack-only message action, or a portable channel action before implementing the missing surface with schema, docs, tests, and live Slack proof.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: current main has only fixed Slack assistant prompts on `assistant_thread_started` and no `assistantPrompts` config key or `set-suggested-prompts` message action. I did not run a live Slack workspace scenario.

## Solution Assessment

No, not as an automated fix yet. The Slack plugin is the right owner, but maintainers still need to choose whether the remaining dynamic surface is Slack-only or a portable channel action, and whether a new config key is worth the compatibility cost.
