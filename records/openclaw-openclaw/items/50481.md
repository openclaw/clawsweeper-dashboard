Current main still has Slack Assistant thread status support but no suggested-prompt config, helper, post-reply dispatch, dynamic message action, docs, or tests; the pinned Slack Bolt package exposes the upstream helper, so this remains a concrete bundled Slack plugin feature rather than cleanup to close.

## What This Changes

Implement the static suggested-prompt feature inside the Slack plugin first: add a small validated account/root config with up to four prompts, call Slack's suggested-prompts API only after successful assistant-thread replies with a thread timestamp, document the behavior, and add the dynamic action only if it stays a narrow additive contract change.
