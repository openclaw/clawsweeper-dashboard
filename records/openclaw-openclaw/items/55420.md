## Summary

Keep open. Current main and the latest release still expose only the existing string-valued session visibility modes, and the adjacent open items cover broader per-capability, per-agent, sandbox, or A2A-send policy axes rather than replacing this session-key pattern allowlist request.

## What This Changes

Request adds a new `tools.sessions.visibility="allowlist"` mode plus a `tools.sessions.allowlist` array of session-key patterns for trusted cross-channel context sharing.

## Best Possible Solution

Define one maintainer-approved session authorization model, then implement the chosen config shape across schema, shared visibility checks, session tool callers, docs, generated help, and isolation tests without weakening current `self`/`tree` defaults.

## Reproduction Assessment

Yes for source verification, not as a bug reproduction. Current main can be checked in the schema, docs, and shared guard, and all still lack `allowlist` visibility or `tools.sessions.allowlist`.

## Solution Assessment

Unclear. A session-key allowlist is plausible for trusted group sharing, but maintainers should first choose whether the durable model is session-pattern based, per-capability, per-agent, A2A-specific, or a consolidated combination.
