Keep open. Current main and v2026.6.8 still expose session-tool visibility as the four existing modes with no session-key allowlist, and the request needs maintainer product/security review before adding a new public session authorization contract.

Add an `allowlist` mode to `tools.sessions.visibility` plus a `tools.sessions.allowlist` array of session-key patterns.

## Best Possible Solution

Define one maintainer-approved session authorization model that covers target allowlists, per-capability behavior, sandbox clamps, memory search, and A2A interactions, then update schema, docs, and focused isolation tests together.
