## Summary

Keep open. Current main still lacks a session-key allowlist mode in runtime visibility, strict config schema, typed config, docs, and the shared session-tool guard; related #48785 is adjacent permission-model work but does not supersede this narrower pattern allowlist request.

## What This Changes

Add `tools.sessions.visibility: "allowlist"` plus a `tools.sessions.allowlist` array of session-key glob patterns for selected cross-session access.

## Best Possible Solution

Implement a maintainer-approved design in the shared session visibility layer, extending config types/schema/docs and adding regression coverage for list/history/send/status, canonical key matching, A2A, and sandbox behavior.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that current main has no `allowlist` mode or `tools.sessions.allowlist` config surface.

## Solution Assessment

No current implementation exists on main. The requested direction is plausible, but the best fix needs maintainer policy decisions for A2A, sandboxed sessions, and exact pattern-matching semantics before code changes.
