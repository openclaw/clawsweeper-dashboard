## Summary

Keep open. Current main and v2026.6.8 still expose session-tool visibility as the four existing modes with no session-key allowlist, and the request remains a security-sensitive product/config decision rather than conservative cleanup.

## What This Changes

Add a `tools.sessions.visibility="allowlist"` mode plus `tools.sessions.allowlist` session-key patterns so trusted sessions can share context without granting same-agent or all-session access.

## Best Possible Solution

Define one maintainer-approved session authorization model that covers target allowlists, per-capability behavior, sandbox clamps, memory search, and A2A interactions, then update schema, docs, and focused isolation tests together.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source inspection gives high confidence that current main and v2026.6.8 lack the requested allowlist mode and companion session pattern array.

## Solution Assessment

Unclear. A session-key allowlist is plausible, but maintainers should first choose whether the durable model is target-pattern based, per-capability, per-agent, A2A-specific, or consolidated across those axes.
