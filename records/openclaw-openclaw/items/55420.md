## Summary

Keep open. Current main and the latest release still expose only the four existing session visibility scopes, with no `allowlist` visibility mode or `tools.sessions.allowlist` config surface; related granular-visibility and A2A-send issues are adjacent, not a complete substitute for this session-key allowlist request.

## What This Changes

Request to add a `tools.sessions.visibility="allowlist"` mode plus `tools.sessions.allowlist` session-key patterns for selective cross-channel session sharing.

## Best Possible Solution

Define a maintainer-approved, fail-closed session-key allowlist contract that is aligned with granular visibility/A2A work, then implement it centrally across schema, types, docs, the shared visibility guard, session tools, memory search, and focused access-control tests.

## Reproduction Assessment

Not applicable as a feature/config request. Source, schema, docs, tests, and the latest release tag give high confidence that current OpenClaw does not have `visibility="allowlist"` or `tools.sessions.allowlist`.

## Solution Assessment

Unclear until maintainers choose the access-control semantics. The allowlist direction is plausible, but it should be designed together with per-capability visibility, canonical key matching, sandbox clamps, and A2A send behavior.
