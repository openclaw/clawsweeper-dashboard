## Summary

Keep open. Current main and v2026.5.18 still expose only `self`, `tree`, `agent`, and `all` session visibility, with no `allowlist` mode or `tools.sessions.allowlist`; the related granular visibility and A2A-send items are adjacent rather than a complete substitute.

## What This Changes

Add `tools.sessions.visibility="allowlist"` plus a `tools.sessions.allowlist` session-key pattern array for trusted cross-channel session sharing.

## Best Possible Solution

Define a maintainer-approved, fail-closed session-key allowlist contract aligned with granular visibility and A2A send, then implement it across schema, types, docs, the shared visibility guard, session tools, memory search, and access-control tests.

## Reproduction Assessment

Not applicable as a feature/config request. Source, schema, docs, tests, and the latest release tag give high confidence that current OpenClaw does not have `visibility="allowlist"` or `tools.sessions.allowlist`.

## Solution Assessment

Unclear until maintainers choose the access-control semantics. The allowlist direction is plausible, but it should be designed together with per-capability visibility, canonical key matching, sandbox clamps, and A2A send behavior.
