## Summary

Keep open. Current main and v2026.6.6 still expose session-tool visibility as the four-value `self | tree | agent | all` scope with no `allowlist` mode or `tools.sessions.allowlist`, and the request needs maintainer product/security review before adding a new session authorization contract.

## What This Changes

Request to add `tools.sessions.visibility="allowlist"` plus a `tools.sessions.allowlist` array of session-key patterns for trusted cross-session context sharing without full same-agent visibility.

## Best Possible Solution

Define one maintainer-approved session authorization model that covers target allowlists, per-capability behavior, sandbox clamps, and A2A interactions, then update schema, docs, and focused isolation tests together.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source inspection gives high confidence that current main lacks the requested allowlist mode and companion session pattern array.

## Solution Assessment

Unclear. The allowlist is plausible, but maintainers should first choose whether session authorization should be target-pattern based, per-capability, per-agent, A2A-specific, or a consolidated model across those axes.
