## Summary

Keep open. Current main and the latest release still only support the four string `tools.sessions.visibility` scopes and no `tools.sessions.allowlist`; the related granular visibility, A2A-send, and per-agent scoping items are adjacent but do not replace this session-key pattern request.

## What This Changes

Request adds `tools.sessions.visibility="allowlist"` plus `tools.sessions.allowlist` glob or prefix patterns so only matching trusted session keys are readable or targetable.

## Best Possible Solution

Define one backward-compatible, fail-closed session-tool authorization model in the shared visibility helper, covering allowlist scope alongside related per-capability/per-agent/A2A semantics, with schema, docs, and access-control tests for list/history/send/status and memory search.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source, docs, tests, and the v2026.5.20 tag give high confidence that allowlist visibility is not present today.

## Solution Assessment

Unclear until maintainers choose the public permission model. The allowlist direction is plausible, but it should be designed with the adjacent per-capability, per-agent, A2A-send, sandbox, status, and memory-search semantics.
