## Summary

Keep open. Current main still has only the four existing session visibility modes and no `tools.sessions.allowlist`; #48785 is adjacent permission-model work but does not cover this narrower pattern-scoped allowlist request.

## What This Changes

Add a new `tools.sessions.visibility="allowlist"` mode plus `tools.sessions.allowlist` session-key patterns for trusted cross-session access.

## Best Possible Solution

Approve and implement a core session-access design that adds pattern-scoped visibility in the shared guard, config schema/types/docs, and regression coverage across list/history/send/status, A2A, canonical keys, and sandboxed sessions.

## Solution Assessment

No current implementation exists on main. The requested direction is plausible, but the best fix needs maintainer policy decisions for access-control semantics before code changes.
