## Summary

Keeping this open: current main documents and implements backup creation and verification, but not a supported restore path, so the reported user problem remains valid and needs maintainer direction.

## What This Changes

Request a documented restore step or `openclaw backup restore` command for archives created by `openclaw backup`.

## Best Possible Solution

Define a supported restore workflow for backup archives, then update `https://docs.openclaw.ai/cli/backup`; if it becomes a CLI command, add registration and archive-restore tests.

## Reproduction Assessment

Yes. The current docs and CLI registration reproduce the gap directly: `/cli/backup` lists create/verify only, and the backup namespace registers no restore subcommand.

## Solution Assessment

Unclear. A restore command may be the right end state, but the safe restore semantics need maintainer product and safety decisions before an automated implementation.
