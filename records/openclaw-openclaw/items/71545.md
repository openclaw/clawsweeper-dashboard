## What This Changes

The PR adds streamable-http MCP lost-session detection with one reconnect/retry around bundled MCP tool calls, plus regression tests, a streamable HTTP test harness, and a changelog entry.

## Best Possible Solution

Land a narrow streamable-http-only recovery path that recreates lost bundled MCP sessions in place, keeps the custom schema validator and SDK tool metadata intact after reconnect, and leaves SSE recovery to its own tracked follow-up.

## Reproduction Assessment

Yes. ## Solution Assessment

Yes. The PR's hardened shape is the narrow maintainable fix: it is scoped to streamable-http sessions, reuses existing transport resolution, preserves the custom MCP client validator, shares concurrent reconnects, retries once, and leaves stdio/SSE behavior unchanged.
