## Summary

Keep this PR open for maintainer review. The patch is a focused Codex app-server bug fix with adequate terminal proof and no blocking code-review findings, but it intentionally changes message-delivery semantics by treating concurrent identical same-turn dynamic-tool calls with different callIds as one in-flight execution.

## What This Changes

The branch adds a Codex dynamic-tool in-flight coalescer keyed by thread, turn, namespace, tool, and canonicalized arguments, then wires it into the main app-server run path and /btw side-question path with regression coverage.

## Best Possible Solution

Land the narrow in-flight coalescer only after maintainers accept the callId-independent delivery semantics, while keeping https://github.com/openclaw/openclaw/issues/80040 open for the broader duplicate classes.

Yes, at source/probe level: current main handles each valid Codex item/tool/call by calling the tool execution path independently, and the PR body describes a local runtime probe with two same-thread/same-turn duplicate calls. I did not execute the probe during this read-only review.

## Solution Assessment

Yes for the narrow duplicate class: an in-flight, per-run coalescer avoids persistent dedupe and preserves later sequential repeats. It is not the complete fix for the broader duplicate-tool cascade tracked in https://github.com/openclaw/openclaw/issues/80040.
