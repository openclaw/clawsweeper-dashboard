## Summary

Keep open: current main and the latest shipped release still hydrate diff cards in one unguarded loop, so a per-card `FileDiff.hydrate()` failure can abort later cards.

## What This Changes

The issue requests that one failed diff card hydration not prevent remaining diff cards on the page from hydrating.

## Best Possible Solution

Keep this open and fix `hydrateViewer` to isolate per-card hydration failures, add focused viewer-client DOM coverage, and regenerate the bundled viewer runtime asset.

## Reproduction Assessment

Yes, source-level: mock or trigger `FileDiff.hydrate()` to throw for the first of two `.oc-diff-card` elements, and current `hydrateViewer` will reject before reaching the second card. I did not run the jsdom/browser repro in this read-only checkout.

## Solution Assessment

Yes: a per-card guard around the hydration body, plus warning or card-local error state, is the narrowest maintainable fix while leaving the outer `main()` catch for fatal viewer setup failures.
