## Summary

Keep open: current main and the latest shipped release still use an unguarded per-card hydration loop, and the linked closing PR is open rather than merged.

## What This Changes

The issue asks OpenClaw to isolate diff viewer hydration failures so one bad `.oc-diff-card` does not prevent later cards from hydrating.

## Best Possible Solution

Review and land a corrected `https://github.com/openclaw/openclaw/pull/84263` or an equivalent focused fix that guards each card, updates the served viewer runtime asset, and adds DOM-level regression coverage.

## Reproduction Assessment

Yes, at source level: force `FileDiff.hydrate()` to throw for the first of two parsed `.oc-diff-card` payloads and the current loop rejects before it can hydrate the second card. I did not run a live DOM/browser repro in this read-only checkout.

## Solution Assessment

Yes, the narrow maintainable solution is a per-card guard in `hydrateViewer`, keeping the outer `main()` catch for fatal setup failures, plus updating the generated runtime asset and adding regression coverage.
