## Summary

Keep open: current main still lacks external workspace agent-config auto-discovery. The existing `agents.create` RPC and hot `agents.list` reload path reduce the zero-restart pain TinyTb described, but they do not provide the requested discovery source, config schema, precedence, or trusted-directory contract.

## What This Changes

Request to let OpenClaw discover and load agent configurations from configured external workspace directories instead of requiring every agent in central `agents.list`.

## Best Possible Solution

Define a supported discovery design or plugin/API path covering trusted scan roots, validation, deterministic ordering, explicit-agent precedence, and hot add/remove behavior before writing code.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection confirms current main has explicit `agents.list` registration plus limited state-dir awareness, but no external workspace config discovery path.

## Solution Assessment

Unclear: `agents.autoDiscover` is a plausible interface, but maintainers still need to decide whether the narrowest maintainable solution is core config, a plugin/API path, or an external scanner using existing agent RPCs.
