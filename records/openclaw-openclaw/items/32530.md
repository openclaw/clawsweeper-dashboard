## Summary

Keep open: current main and latest release v2026.5.22 still lack an `agents.autoDiscover` config contract or external workspace scan for agent config files. The adjacent `agents.create`/`agents.update` RPCs and hot `agents.list` reload handling are useful manual-management pieces, but they do not satisfy the requested trusted discovery feature.

## What This Changes

Request adds an `agents.autoDiscover` configuration surface that scans trusted workspace directories for agent config files and automatically loads discovered agents.

## Best Possible Solution

Define a core-owned trusted discovery and live-agent registration contract first, then add schema, docs, tests, deterministic precedence, explicit reload lifecycle, and security boundaries before scanning workspace-local agent files.
