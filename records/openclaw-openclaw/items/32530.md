## Summary

Keep open: current main still lacks external workspace-owned agent config discovery. It now has explicit `agents.create`/`agents.update`/`agents.delete` and hot application for `agents.list`, but the requested `agents.autoDiscover`-style config authority remains a new security-sensitive product decision.

## What This Changes

Requested feature: automatically discover agent configurations from external workspace directories instead of requiring every agent to be manually listed in central OpenClaw config.

## Best Possible Solution

Define a maintainer-approved, operator-controlled discovery or import design with trusted roots, deterministic conflict handling, validation failure behavior, and session lifecycle semantics before adding a core config source.

## Reproduction Assessment

Not applicable: this is a new discovery/configuration capability request, not a report of broken existing behavior. Source inspection confirms current main still uses explicit `agents.list` and explicit `agents.create`/`agents.update`/`agents.delete` as the canonical agent definition path.

## Solution Assessment

No: the proposed glob-based merge is not yet the best solution because trust, conflict, validation, persistence, and session lifecycle rules are undefined. The safer current path is explicit agent registration with hot `agents.list` reload while maintainers decide whether workspace discovery belongs in core.
