## Summary

Keep open: current main still lacks an external workspace agent auto-discovery contract, even though recent agent-create and hot `agents.list` reload paths reduce some restart pain. This remains a feature/design item with config and security choices, not a conservative cleanup close.

## What This Changes

Request automatic discovery and loading of agent configs from configured external workspace directories instead of manually maintaining central `agents.list` entries.

## Best Possible Solution

Define a supported discovery design covering trusted roots, schema validation, deterministic ordering, precedence against explicit agents, removal/watch behavior, hot-load behavior, and whether this belongs in core config or a plugin.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The absence is high-confidence from current schema, enumeration code, gateway listing behavior, and docs, all of which are centered on explicit `agents.list` entries.

## Solution Assessment

Unclear: `agents.autoDiscover` is a plausible shape, but it is not yet established as the narrowest maintainable solution. Maintainers should first decide whether this belongs in core `agents.*` config or in a plugin/API contribution path.
