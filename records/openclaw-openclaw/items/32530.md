## Summary

Keep open: current main still lacks external workspace agent-configuration discovery. The existing manual `agents.list` / `agents.create` paths and hot reload handling cover adjacent agent management, but not trusted glob scanning, config-file discovery, merge precedence, or hot add/remove semantics.

## What This Changes

Request automatic discovery and loading of agent configs from configured external workspace directories instead of manually registering every agent in `agents.list`.

## Best Possible Solution

Define a supported discovery contract for trusted roots, validation, deterministic ordering, explicit-agent precedence, and hot add/remove behavior, then implement it with focused schema, gateway, and agent-enumeration coverage.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection on current main shows no `agents.autoDiscover` schema/type path and no external workspace config scan in the agent enumeration paths.

## Solution Assessment

Unclear: `agents.autoDiscover` is a plausible direction, but maintainers still need to decide the discovery contract, trust boundary, precedence rules, and how it should interact with zero-restart agent management.
