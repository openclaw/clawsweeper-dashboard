## Summary

Current main does not auto-discover agent configurations from external workspaces. Explicit `agents.list` registration remains canonical, while `$include` and hot reload address only parts of the operational burden; the remaining capability needs a maintainer product and security decision.

## What This Changes

Add automatic discovery and loading of agent configurations from configured external workspace directories.

## Best Possible Solution

Define an explicit, validated live agent registration API with add/remove and hot-reload semantics first; only then consider optional trusted-directory discovery as a thin layer with clear collision, symlink, removal, and session rules.

## Reproduction Assessment

Not applicable: this requests a new agent discovery and lifecycle capability rather than reporting broken existing behavior. Current source and docs clearly establish the explicit registration workflow it would augment.

## Solution Assessment

No: ambient glob discovery combines registration, trust, precedence, removal, and session policy in one mechanism. A validated live registration contract is the narrower maintainable foundation.
