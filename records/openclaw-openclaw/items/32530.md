## Summary

Keep open: current main still lacks external workspace agent-config auto-discovery. The existing `agents.create` RPC and hot `agents.list` reload path reduce some registration/restart pain, and TinyTb's #27706 link is relevant, but the requested discovery/config/security contract remains unimplemented.

## What This Changes

Request adding an `agents.autoDiscover` configuration option that scans trusted workspace directories for agent configuration files and merges discovered agents with explicit `agents.list` entries.

## Best Possible Solution

Define a supported discovery design or plugin/API path covering trusted roots, validation, deterministic ordering, explicit-agent precedence, and hot add/remove behavior before implementation.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection confirms current main only supports explicit `agents.list` entries plus limited disk session-store awareness, not external workspace config discovery.

## Solution Assessment

Unclear: `agents.autoDiscover` is a plausible shape, but maintainers still need to decide whether the narrowest maintainable solution is core config, a plugin/API path, or an external scanner using existing agent RPCs.
