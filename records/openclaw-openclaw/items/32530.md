## Summary

Keep open. Current main still lacks an agent auto-discovery config contract or pre-enumeration extension point, and the implemented disk-agent scanning is limited to session-store visibility rather than trusted external workspace config loading.

## What This Changes

The issue requests automatic discovery and loading of agent configuration files from configured external workspace directories instead of requiring each agent to be listed manually in the central OpenClaw config.

## Best Possible Solution

Define either a first-class `agents.autoDiscover` config contract or a documented plugin-backed agent-definition contribution API that runs before normal agent enumeration, with trusted roots, accepted filenames, schema validation, deterministic scan order, explicit precedence versus `agents.list`, reload/watch behavior, docs, and regression tests.
