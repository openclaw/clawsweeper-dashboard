## Summary

Keep open: current main still lacks an external workspace agent auto-discovery contract. Existing code and docs are centered on explicit `agents.list` entries, while related hot-reload and disk-session scanning work does not provide trusted workspace scanning or discovered agent definition loading.

## What This Changes

Request adding an `agents.autoDiscover` configuration path that scans specified workspace directories for agent config files and loads them as agents.

## Best Possible Solution

Define a supported agent-discovery design that covers trusted roots, deterministic ordering, schema validation, precedence against explicit agents, removal behavior, hot-load behavior, docs, and regression tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The absence is high-confidence from source inspection: current schema and enumeration paths support explicit `agents.list` plus default-agent fallback, not external workspace scanning.

## Solution Assessment

Unclear: `agents.autoDiscover` is a plausible shape, but it is not yet established as the narrowest maintainable solution. A maintainer design should first decide whether this belongs in core config or in a plugin/API contribution path.
