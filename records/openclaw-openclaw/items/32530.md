## Summary

Keep open: current main still lacks external workspace agent-configuration discovery. Adjacent zero-restart support now exists for hot-reloading `agents.list`, but there is still no `agents.autoDiscover` schema, typed contract, workspace config-file scan, or decided trust/precedence model.

## What This Changes

Add an `agents.autoDiscover` configuration surface that scans trusted workspace directories for agent config files and merges discovered agents with explicit `agents.list` entries.

## Best Possible Solution

Define a core-owned discovery contract for trusted roots, validation, deterministic ordering, explicit-agent precedence, and hot add/remove behavior, then implement it in the agreed config or live agent-registry surface.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection on current main confirms there is no supported `agents.autoDiscover` path or external workspace config scan.

## Solution Assessment

Unclear: `agents.autoDiscover` is plausible, but maintainers need to decide whether discovery belongs in config, a live agent registry API, or a broader zero-restart agent-management design.
