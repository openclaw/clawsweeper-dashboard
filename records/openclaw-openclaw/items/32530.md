## Label Justifications

- `P2`: This is a useful multi-workspace ergonomics feature with limited urgency, but it spans config, gateway lifecycle, and security boundaries.
- `impact:security`: External workspace discovery would let filesystem content influence agent policy, tools, models, and routing unless trust roots are tightly defined.
- `impact:session-state`: Automatic addition, mutation, or removal of agents can affect routing and live or persisted session association with agent identities.

## Summary

Keep open for maintainer product and security design. Current main and v2026.6.5 support explicit agent creation plus hot application of `agents.list` changes, but they still do not auto-discover agent configuration files from external workspaces.

## What This Changes

Request to add an `agents.autoDiscover`-style configuration mode that scans external workspace directories for agent config files and loads them into OpenClaw.

## Best Possible Solution

Keep `agents.list` as the canonical runtime registry and, if maintainers approve discovery, add an operator-controlled reconciliation layer with explicit trusted roots, deterministic precedence, validation, and active-session semantics.

## Reproduction Assessment

Not applicable: this requests a new discovery mode rather than reporting broken existing behavior. Source and docs confirm explicit registration remains the supported current workflow.

## Solution Assessment

No, not as proposed. Directly merging arbitrary glob-discovered files into runtime config would create competing sources of truth; a validated reconciler feeding the canonical agent registry is safer.
