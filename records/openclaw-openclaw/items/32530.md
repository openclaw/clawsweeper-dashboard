## Summary

Keep open: current main still has explicit agent registration and partial zero-restart support, but it does not implement external workspace agent-config discovery, a discovery schema, precedence rules, or trusted-directory handling.

## What This Changes

The issue requests an `agents.autoDiscover` configuration path that scans external workspace directories for agent config files and loads them without manual `agents.list` registration.

## Best Possible Solution

Define a supported discovery design or plugin/API path covering trusted scan roots, validation, deterministic ordering, explicit-agent precedence, and hot add/remove behavior before implementation.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection confirms current main has explicit `agents.list` registration and `agents.create`, but no external workspace config discovery path.

## Solution Assessment

Unclear: `agents.autoDiscover` is plausible, but maintainers still need to decide whether the narrowest maintainable solution is core config, a plugin/API path, or an external scanner using existing agent RPCs.
