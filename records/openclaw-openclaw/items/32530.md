## Summary

Keep open: current main still lacks external workspace agent-configuration discovery. The adjacent manual `agents.create`, configured `agents.list`, disk session discovery, and hot-reload planning do not satisfy the requested `agents.autoDiscover` workspace scan, and the remaining work is a new config/product/security surface.

## What This Changes

The issue requests automatic discovery and loading of agent configs from trusted external workspace directories instead of requiring every agent to be registered in central config.

## Best Possible Solution

Define a supported discovery contract for trusted roots, validation, ordering, precedence, and hot add/remove behavior, then implement it in the core config path or a documented plugin/API surface after maintainer approval.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection confirms current main has no `agents.autoDiscover` schema/type path and no external workspace config scan in the agent enumeration paths.

## Solution Assessment

Unclear: `agents.autoDiscover` is plausible, but maintainers still need to decide whether the narrowest maintainable solution is core config, a plugin/API surface, or an external scanner using existing agent RPC and hot reload paths.
