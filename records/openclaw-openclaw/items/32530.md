## Summary

Keep open: current main still supports explicit `agents.list` registration and manual/Gateway RPC agent creation, but it does not implement external workspace agent-config discovery, trusted scan roots, precedence rules, or discovered-agent hot add/remove semantics.

## What This Changes

Request to add `agents.autoDiscover` so OpenClaw can scan configured workspace directories for agent config files and load discovered agents without manual central `agents.list` edits.

## Best Possible Solution

Define a supported discovery contract or plugin/API path covering trusted scan roots, validation, deterministic ordering, explicit-agent precedence, and hot add/remove behavior before implementation.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection confirms current main has explicit `agents.list` and `agents.create` paths but no external workspace config discovery path.

## Solution Assessment

Unclear: `agents.autoDiscover` is plausible, but maintainers still need to decide whether the narrowest maintainable solution is core config, a plugin/API surface, or an external scanner using existing agent RPCs.
