## Summary

Keep open: current main and v2026.6.10 still lack `agents.autoDiscover` or equivalent workspace-owned agent config loading. Current code does support explicit agent registration and hot `agents.list` reload, but the requested external discovery path remains a product and security design decision.

## What This Changes

The issue requests automatic discovery and loading of agent configurations from configured external workspace directories instead of requiring every agent in central `openclaw.json`.

## Best Possible Solution

Define a maintainer-approved, operator-controlled discovery or import design with trusted roots, deterministic conflict handling, validation failure behavior, and session lifecycle semantics before adding a core config source.

No: the proposed glob-based merge is not yet the best solution because trust, conflict, validation, persistence, and session lifecycle rules are undefined.
