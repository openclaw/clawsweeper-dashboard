## Summary

Keep open: current main still has no external workspace agent-configuration discovery, and the existing manual agent creation plus hot config reload paths do not settle the new discovery contract, trust boundary, precedence, or hot add/remove semantics.

## What This Changes

Request to add an `agents.autoDiscover` style config that scans trusted workspace directories for agent config files and merges discovered agents automatically.

## Best Possible Solution

Define a supported discovery contract for trusted roots, validation, deterministic ordering, explicit-agent precedence, and hot add/remove behavior, then implement it in the agreed core config or agent-registry surface with focused coverage.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection on current main shows no `agents.autoDiscover` schema/type path and no external workspace config scan in agent enumeration.

## Solution Assessment

Unclear: `agents.autoDiscover` is plausible, but maintainers still need to decide the discovery contract, trust boundaries, precedence rules, and whether zero-restart hot-load or a registry/API surface is the better long-term path.
