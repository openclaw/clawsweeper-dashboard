## Summary

Keep open: current main still has no external workspace agent-config discovery path, and this is a feature/security design request rather than a stale bug cleanup. The related human comment usefully reframes the pain as zero-restart agent management, but the linked binding-reload thread is closed stale and does not supersede this workspace discovery request.

## What This Changes

Request to add `agents.autoDiscover` so OpenClaw can scan trusted workspace directories for agent config files instead of requiring every agent in central `agents.list` config.

## Best Possible Solution

Define a core-owned discovery and live-registry contract for trusted workspace roots, validation, deterministic precedence, and hot add/remove semantics before implementing config-file scanning.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection on current main confirms there is no supported `agents.autoDiscover` key or external workspace config-file scan.

## Solution Assessment

Unclear: `agents.autoDiscover` is plausible, but maintainers need to decide whether the durable solution belongs in config discovery, live agent registry APIs, or broader zero-restart agent management.
