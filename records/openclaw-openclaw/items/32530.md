## Summary

Keep open: current main still lacks external workspace agent-config discovery. The code validates and enumerates explicit `agents.list` entries, while adjacent hot reload and `agents.create` support explicit management but do not define trusted scanning, precedence, or workspace config-file loading.

## What This Changes

Add an `agents.autoDiscover` config surface that scans specified workspace directories for agent config files and merges them into the agent list.

## Best Possible Solution

Define a core-owned discovery contract for trusted workspace roots, validation, deterministic ordering, explicit-agent precedence, and hot add/remove semantics before implementing config-file scanning.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection confirms current main has no supported `agents.autoDiscover` path or external workspace config scan.

## Solution Assessment

Unclear: `agents.autoDiscover` is plausible, but maintainers need to decide whether the durable solution belongs in config discovery, live agent registry APIs, or a broader zero-restart agent-management design.
