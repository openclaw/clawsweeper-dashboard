## Summary

Keep open: current main still computes TaskFlow audit age from the TaskFlow timestamps only, while this small code/test PR addresses a source-reproducible false positive; merge should wait on contributor real behavior proof and fresh current-base validation.

## What This Changes

The branch changes TaskFlow audit freshness to use the freshest linked task activity timestamp and adds a regression test for active managed flows.

## Best Possible Solution

Land the narrow TaskFlow audit freshness fix with regression coverage after contributor real behavior proof and fresh current-base validation are present.

## Reproduction Assessment

Yes, source-reproducible: current main ages a running managed TaskFlow from `flow.updatedAt ?? flow.createdAt`, while managed flows do not mirror linked child task progress. A linked child with a recent `lastEventAt` can therefore leave the parent flow looking stale to the audit code.

## Solution Assessment

Yes for the code direction: reusing already linked tasks and the existing task activity timestamp precedence is narrower than changing managed-flow sync semantics. Merge should still wait for real behavior proof and fresh validation.
