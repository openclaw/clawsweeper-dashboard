The PR changes TaskFlow audit freshness to use the newest linked task activity timestamp and adds a regression for active managed flows avoiding `stale_running`.

## Best Possible Solution

Land a current-main version of this narrow audit freshness fix with its regression test and a single-line changelog entry, while leaving managed-flow lifecycle synchronization semantics unchanged.

## Solution Assessment

Yes, with the changelog repair. Using linked task activity only for TaskFlow audit freshness is the narrowest maintainable fix because managed flows intentionally do not mirror child task activity into the flow record.
