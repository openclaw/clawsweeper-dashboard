## What This Changes

The PR changes TaskFlow audit staleness to compute freshness from the max of the parent flow timestamp and linked task activity, plus a regression test for an active linked child suppressing `stale_running`.

## Best Possible Solution

Land a refreshed version of this focused audit fix with real task-audit proof and an explicit linked-task scope decision, while leaving broader terminal reconciliation to the separate stale-parent TaskFlow issue.

Yes for the active-child false positive: computing audit age after loading linked tasks is the narrow owner-boundary fix. The remaining solution-fit question is whether the freshness source should include all linked tasks or only active linked tasks.
