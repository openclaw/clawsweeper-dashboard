## What This Changes

The branch updates TaskFlow audit freshness to use the most recent linked task activity timestamp and adds regression coverage for active managed flows.

## Best Possible Solution

Land the narrow TaskFlow audit freshness fix with its regression coverage after contributor real behavior proof and fresh current-base validation are present.

## Reproduction Assessment

Yes, source-reproducible: on current main, a running managed TaskFlow is aged from `flow.updatedAt ?? flow.createdAt` even though managed flows do not mirror linked child task activity. A linked child with recent `lastEventAt` can therefore still leave the parent flow looking stale to the audit code.

## Solution Assessment

Yes for the code direction: reusing already linked tasks and the existing task activity timestamp precedence is narrower than changing managed-flow sync semantics. Merge should still wait for real behavior proof and fresh validation.
