## What This Changes

The PR changes TaskFlow audit freshness to consider linked task activity timestamps and adds a regression test for an active linked child suppressing a stale-running finding.

## Best Possible Solution

Land a refreshed version after real audit proof and maintainer confirmation that linked task activity, including terminal child activity, should drive TaskFlow audit freshness.

## Reproduction Assessment

Yes, at source level: current main computes TaskFlow stale age from flow.updatedAt/createdAt before linked tasks are loaded, while managed child task progress can remain on TaskRecord.lastEventAt. I did not run tests because this review is read-only.

## Solution Assessment

Mostly yes: TaskFlow audit is the right owner boundary because managed flows intentionally do not mirror child task progress. The remaining best-fix question is whether freshness should include all linked tasks or only active linked tasks.
