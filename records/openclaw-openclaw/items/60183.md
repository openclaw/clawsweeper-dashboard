## What This Changes

The PR changes TaskFlow audit freshness to use the most recent linked task activity timestamp when evaluating stale running TaskFlows and adds regression coverage for active child-task progress.

## Best Possible Solution

Land the narrow linked-task freshness fix after contributor proof and current-base validation, with maintainer acceptance or adjustment of whether terminal linked tasks should extend TaskFlow freshness.

## Reproduction Assessment

Yes, source-reproducible: current main computes stale-flow age from only the parent flow timestamp, while managed flows do not mirror linked child progress. A stale parent flow with a recently active linked child can therefore still produce `stale_running`.

## Solution Assessment

Yes for the code direction: using linked tasks and the existing task activity timestamp precedence is a narrow fix. Maintainers should still decide whether recently terminal child tasks should also extend the freshness window.
