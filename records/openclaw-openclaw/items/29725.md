## Summary

Keep open: current main and v2026.6.10 still provide static Discord presence, auto-presence, and the gated `set-presence` message action, but not the requested plugin-runtime `api.runtime.channel.discord.updatePresence` API. This remains a Discord/plugin API and security-boundary product decision, not conservative cleanup.

## What This Changes

Request to expose a Discord plugin runtime API for dynamic bot presence and activity updates without restarting OpenClaw.

## Best Possible Solution

Approve a Discord-owned, default-off, account-scoped plugin runtime presence contract that reuses the existing gateway update path without exposing provider internals, then add focused docs and tests for authorization and precedence.
