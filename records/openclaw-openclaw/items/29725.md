## Summary

Keep open: current main still lacks the requested plugin-facing `api.runtime.channel.discord.updatePresence` helper. Existing Discord presence support covers static config, the gated `setPresence` message action, the gateway writer, and auto-presence, but exposing this as a stable plugin runtime API still needs maintainer product/API review.

## What This Changes

The issue requests a plugin-facing Discord runtime helper so plugins can update bot presence dynamically without restarting.

## Best Possible Solution

Define a Discord-owned runtime helper only after maintainer API review, reusing the existing gateway registry, validation, and presence policy where appropriate, then document routing and auto-presence precedence.
