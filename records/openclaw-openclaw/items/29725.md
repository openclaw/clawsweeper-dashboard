## Summary

Keep open: current main and v2026.6.5 can update Discord presence through configuration, auto-presence, and a gated agent-facing `setPresence` action, but they still do not expose the requested code-plugin runtime contract. The related self-profile issue is not a safe duplicate because it adds nickname/avatar scope and does not clearly preserve this plugin-runtime use case.

## What This Changes

Expose a supported Discord plugin runtime method for changing bot activity and status dynamically without restarting.

## Best Possible Solution

Define one Discord-owned typed presence operation with explicit account selection and configured/auto-presence precedence, then reuse it from separately authorized agent-action and approved code-plugin runtime adapters.

## Reproduction Assessment

Not applicable: this issue requests a new plugin capability, and source plus release inspection confirms that current supported surfaces are configuration, auto-presence, and a gated agent action rather than the requested method.

## Solution Assessment

No, the guessed standalone method is not yet the best solution; maintainers should first define a shared Discord-owned operation and the distinct authorization contracts for agent actions and trusted code plugins.
