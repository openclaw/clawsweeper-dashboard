## Summary

Keep open: current main and v2026.5.20 still do not expose the requested plugin-facing `api.runtime.channel.discord.updatePresence` helper. Existing support covers static config, auto-presence, the gated `setPresence` message action, and an internal gateway writer, but making this a public plugin runtime API still needs maintainer API/product review; the related self-profile request at https://github.com/openclaw/openclaw/issues/34004 is adjacent, not a complete duplicate.

## What This Changes

Request to add a Discord plugin runtime helper that lets plugins update bot presence dynamically without restarting.

## Best Possible Solution

If accepted, add a Discord-owned runtime helper in the Discord plugin that reuses the gateway registry, presence gate, account routing, validation, docs, tests, and live proof while defining interaction with `setPresence` and auto-presence.
