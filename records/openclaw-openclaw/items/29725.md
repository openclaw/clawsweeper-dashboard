## Summary

Keep open: current main and v2026.5.20 still lack the requested plugin-facing `api.runtime.channel.discord.updatePresence` helper. Existing support covers static config, opt-in auto-presence, the gated `setPresence` message action, and an internal gateway writer, but exposing this as public plugin runtime API still needs maintainer API/product review.

## What This Changes

The issue requests adding `api.runtime.channel.discord.updatePresence(...)` so plugins can dynamically change the Discord bot's presence without accessing Discord provider internals.

## Best Possible Solution

If accepted, implement the helper inside the Discord plugin runtime, reusing the gateway registry, presence gate, account routing, validation, docs, focused tests, and live Discord proof while defining interaction with auto-presence.

## Reproduction Assessment

Not applicable: this is a feature request, not broken existing behavior. Source and docs inspection on current main plus v2026.5.20 give a high-confidence absence check for the requested plugin-facing helper.

## Solution Assessment
