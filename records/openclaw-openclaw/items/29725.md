## Summary

Keep this open: current main has static Discord presence, auto-presence health signaling, and a gated `setPresence` message action, but it still does not expose the requested code-plugin helper on `api.runtime.channel.discord`; adding that public runtime surface needs maintainer API and policy judgment.

## What This Changes

Request to add `api.runtime.channel.discord.updatePresence()` so plugins can dynamically update Discord bot activity and status at runtime.

## Best Possible Solution

Add a Discord-owned, documented runtime helper only after settling permission gating, account selection, payload shape, and auto-presence precedence; otherwise document the existing gated `setPresence` message action as the supported path.

## Reproduction Assessment

Yes. Source inspection on current main shows `DiscordChannelRuntime` lacks `updatePresence`, while the adjacent gated `setPresence` action and auto-presence paths exist.

## Solution Assessment

Unclear. The requested shape is plausible, but the maintainable solution should reuse the existing gateway/action validation path and first answer gating, account selection, and auto-presence precedence.
