## Summary

Keep this open: current main still lacks the requested `api.runtime.channel.discord.updatePresence` helper, while adjacent Discord presence paths exist only as a gated message action, static startup config, and auto-presence controller that need maintainer API and policy decisions before exposing plugin writes.

## What This Changes

Request to add a Discord runtime helper so plugins can dynamically update bot activity and status without restarting OpenClaw.

## Best Possible Solution

Add a Discord-owned, documented runtime helper that reuses existing gateway registration, validation, and presence gating, or explicitly document the gated message action as the supported dynamic presence path for plugins.

## Reproduction Assessment

Yes for the requested missing capability by source inspection: current main does not expose `updatePresence` on the Discord plugin runtime surface, while adjacent `setPresence` and `autoPresence` writers exist.

## Solution Assessment

Unclear. The proposed helper is plausible, but the maintainable design should first settle gate reuse, account selection, and how manual writes interact with auto-presence.
