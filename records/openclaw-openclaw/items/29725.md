## Summary

Keep open: current main still lacks the requested plugin-facing `api.runtime.channel.discord.updatePresence` helper. The related work proves Discord presence can be changed through static config, the gated `setPresence` message action, and auto-presence health signaling, but none of those exposes a supported plugin runtime API for dynamic presence updates.

## What This Changes

Request adding `api.runtime.channel.discord.updatePresence` so plugins can update Discord bot activity/status at runtime.

## Best Possible Solution

Add a documented Discord-owned runtime helper after maintainer API review, reusing existing validation/gating and defining account routing plus auto-presence precedence.

## Reproduction Assessment

Not applicable: this is a feature request, not broken existing behavior. Source inspection gives a high-confidence absence check for the requested `api.runtime.channel.discord.updatePresence` helper on current main.

## Solution Assessment

Unclear: the requested helper is plausible, but the maintainable solution should first settle gate reuse, account routing, payload shape, and interaction with auto-presence rather than exposing a raw gateway writer.

- A plugin-facing helper could bypass the existing default-off `channels.discord.actions.presence` policy unless the design explicitly reuses or replaces that gate.
