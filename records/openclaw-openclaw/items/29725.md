## Summary

Keep this open: current main supports static Discord presence, auto-presence, and a gated `setPresence` message action, but it still does not expose the requested code-plugin runtime helper; adding that public API needs maintainer policy/design judgment.

## What This Changes

Expose a Discord `api.runtime.channel.discord.updatePresence()` helper so plugins can update bot activity and status at runtime.

## Best Possible Solution

Add a Discord-owned, documented runtime helper that reuses the existing gateway registry and validation/gating path, or document the gated message action as the supported dynamic path if code-plugin presence writes are intentionally unsupported.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for the requested capability by source inspection: `DiscordChannelRuntime` lacks `updatePresence`, while the existing `setPresence` action and auto-presence code paths are present on current main.

## Solution Assessment

Is this the best way to solve the issue? Unclear. The helper is plausible, but the maintainable design should reuse the current presence action validation/gateway path and first settle gating, account selection, and auto-presence precedence.
