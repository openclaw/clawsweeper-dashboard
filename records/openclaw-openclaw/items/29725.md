## Summary

Keep open: current main still lacks the requested plugin-facing `api.runtime.channel.discord.updatePresence` helper. Existing Discord presence support covers static config, auto-presence, the gated `setPresence` message action, and the internal gateway writer, but making this a stable plugin runtime API still needs maintainer API/product review; the broader open self-profile request at https://github.com/openclaw/openclaw/issues/34004 is related but does not fully own this runtime-helper request.

## What This Changes

Add a Discord runtime channel helper so plugins can update the bot's Discord status/activity dynamically without restarting.

## Best Possible Solution

If accepted, implement a Discord-owned runtime helper that reuses the existing gateway registry, validation, account routing, and presence policy, then document how it interacts with `setPresence` and auto-presence.
