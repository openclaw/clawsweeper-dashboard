## Summary

Keep open: current main and the latest release support static Discord presence config, auto-presence, and a gated `set-presence` message action, but they still do not expose the requested plugin-runtime `api.runtime.channel.discord.updatePresence` API. The related self-profile issue overlaps on Discord status/activity but does not clearly own this plugin SDK contract.

## What This Changes

Request to add a Discord plugin runtime API for dynamically updating bot presence/status without restarting OpenClaw.

## Best Possible Solution

Keep this open until maintainers choose a secure Discord-owned API/action contract for dynamic plugin presence updates, ideally reusing the existing default-off presence gate and gateway registry without broadening access to provider internals.

## Reproduction Assessment

Not applicable: this is a feature request, not broken existing behavior. Source and release inspection give a high-confidence gap for the requested plugin-runtime Discord presence API.

## Solution Assessment

Unclear as proposed: dynamic presence control is plausible, but a direct `api.runtime.channel.discord` slot does not match the current generic runtime boundary. Maintainers should choose whether this belongs in a Discord plugin facade/action, a generic SDK seam, or the broader self-profile design.
