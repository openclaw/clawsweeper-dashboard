## Summary

Keep open: current main and the latest release still lack the requested plugin-facing `api.runtime.channel.discord.updatePresence` helper. Existing Discord presence support is static config, auto-presence, the gated `setPresence` message action, and the internal gateway writer, so promoting this to stable plugin runtime API still needs maintainer API/product review.

## What This Changes

The issue requests a Discord plugin runtime helper so plugins can dynamically update the bot's Discord status and activity without restarting.

## Best Possible Solution

If accepted, implement a Discord-owned runtime helper that reuses the existing gateway registry, action policy, account routing, validation, and docs, without exposing Discord client internals to plugins.
