Keep open: current main and the latest release still lack the requested plugin-facing `api.runtime.channel.discord.updatePresence` helper. Existing code supports static config, opt-in auto-presence, the gated `setPresence` message action, and an internal gateway writer, but exposing presence writes as public plugin runtime API still needs maintainer API/product review.

## What This Changes

The issue requests a plugin-facing Discord runtime helper so plugins can update bot presence dynamically without restarting OpenClaw.

## Best Possible Solution

If accepted, implement the helper inside the Discord plugin runtime with the existing gateway registry, presence gate, account routing, validation, docs, focused tests, live Discord proof, and explicit auto-presence precedence.
