Keep open: current main and the latest release still do not expose the requested plugin-facing `api.runtime.channel.discord.updatePresence` helper. The existing support covers static Discord presence config, auto-presence, a gated `setPresence` message action, and an internal gateway writer, but making that a public plugin runtime API still needs maintainer product/API review.

## What This Changes

Request adding a plugin-facing `api.runtime.channel.discord.updatePresence` helper so plugins can change Discord bot activity/status at runtime without restarting.

## Best Possible Solution

If accepted, implement a Discord-owned runtime helper that reuses the existing gateway registry, validation, account routing, and presence policy, then document its interaction with `setPresence` and auto-presence.
