## Summary

Keep open: current main and v2026.6.5 still lack a plugin-facing `api.runtime.channel.discord.updatePresence` helper. Discord has static/auto presence config and a gated `setPresence` message action, but adding a public plugin runtime writer remains a product and security boundary decision.

## What This Changes

Request to add a Discord runtime helper so plugins can update the bot's status/activity dynamically without restarting OpenClaw.

## Best Possible Solution

Define one Discord-plugin-owned presence control contract that reuses the existing gateway path and gate semantics, documents auto-presence precedence, and avoids competing plugin/API/message-action sources of truth.

## Reproduction Assessment

Not applicable: this is a feature request, not broken existing behavior. Source, docs, release, and related-item checks give a high-confidence absence check for the requested plugin-facing helper.

## Solution Assessment

Unclear pending maintainer API review. A Discord-owned helper is plausible, but maintainers should first decide whether it should reuse the existing `presence` gate, how it composes with auto presence, and how it relates to https://github.com/openclaw/openclaw/issues/34004.
