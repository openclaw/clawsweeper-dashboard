## Summary

Keep open. Current main still does not expose the requested `api.runtime.channel.discord.updatePresence()` helper for code plugins; it only has static/auto presence configuration and the gated `setPresence` message action. Adding this would be a public plugin-runtime API decision with gate, account-selection, and auto-presence precedence semantics, so it needs Discord/plugin-runtime maintainer review rather than conservative cleanup closure.

## What This Changes

The issue requests a Discord plugin runtime helper that lets plugins dynamically update bot activity and status without restarting OpenClaw.

## Best Possible Solution

If maintainers want this capability, add a Discord-owned, typed runtime helper that reuses the existing gateway registry, account selection, action gate, validation rules, and tests, with documented precedence against auto-presence. If they do not want a new runtime API, document the existing gated `setPresence` message action as the supported path for plugin-authored automation.
