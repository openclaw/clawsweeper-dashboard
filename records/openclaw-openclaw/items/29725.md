## Summary

Keep open: current main still lacks the requested plugin-facing `api.runtime.channel.discord.updatePresence` helper. Existing Discord presence support covers static config, a gated `setPresence` message action, and auto-presence health signaling, but none of those is the supported runtime API requested here.

## What This Changes

The issue requests a plugin-facing Discord runtime helper that lets plugins update the bot's Discord activity and status dynamically.

## Best Possible Solution

Add a documented Discord-owned runtime helper only after maintainer API review, reusing existing validation/gating and defining account routing plus auto-presence precedence.
