## Summary

Keep open: current main and the latest release still lack the requested plugin-runtime Discord presence API. The related self-profile issue overlaps on Discord status/activity, but it does not clearly own this plugin SDK contract, and the remaining work needs product and security design.

## What This Changes

Request to add `api.runtime.channel.discord.updatePresence` so plugins can dynamically update Discord bot status/activity without restarting.

## Best Possible Solution

Design a narrow Discord-owned presence contract that defines authorization, account selection, and precedence with static, action, and auto-presence paths, then add focused tests and docs before implementation.

## Reproduction Assessment

Not applicable: this is a missing feature/API request, not broken existing behavior. Source inspection confirms `api.runtime.channel` has only generic helpers while Discord presence updates exist through config, message-action, and auto-presence paths.

## Solution Assessment

No, not exactly as proposed: a raw `api.runtime.channel.discord.updatePresence` slot would add a plugin-id-specific runtime seam and leave authorization, account targeting, and auto-presence precedence unresolved. The safer path is a maintainer-approved Discord-owned or narrow generic contract with explicit policy.
