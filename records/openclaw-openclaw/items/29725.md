## Summary

Keep this open: current main still lacks the requested `api.runtime.channel.discord.updatePresence` helper. OpenClaw has adjacent static presence, auto-presence, and a gated `setPresence` message action, but exposing code-plugin presence writes needs maintainer API and policy judgment.

## What This Changes

Requests a Discord plugin runtime helper so code plugins can update the bot activity and status dynamically without a restart.

## Best Possible Solution

Add a Discord-owned, documented runtime helper that reuses the existing gateway registry, validation, and presence gate, or explicitly document the gated message action as the supported dynamic presence path.

## Reproduction Assessment

Yes for the requested missing capability by source inspection: the typed Discord runtime surface lacks `updatePresence`, while current main contains adjacent `setPresence` and `autoPresence` writers.

## Solution Assessment

Unclear. The helper is plausible, but the maintainable design should first settle gate reuse, multi-account selection, and how manual presence writes interact with auto-presence.
