## Summary

Keep open: current main and the latest release have static Discord presence config, auto-presence, and a gated Discord `set-presence` message action, but they still do not expose the requested plugin-runtime presence API. The related self-profile issue overlaps on Discord status/activity, but it is broader and does not clearly own this plugin SDK contract.

## What This Changes

The issue requests a supported Discord runtime API so plugins can update bot presence dynamically without restarting static Discord channel configuration.

## Best Possible Solution

Add a narrow Discord-owned presence contract only after maintainers choose the trust boundary, account-selection behavior, and precedence rules with existing static, message-action, and auto-presence paths.

## Reproduction Assessment

Not applicable: this is a missing feature/API request, not a broken current behavior with reproduction steps. Source inspection does confirm the current API gap on main and in `v2026.6.6`.

## Solution Assessment

No, not exactly as proposed: a raw `gateway.updatePresence` wrapper would leave authorization, multi-account targeting, and auto-presence precedence undefined. The safer solution is a Discord-owned, documented SDK or action contract that reuses explicit gating and stable product-level types.
