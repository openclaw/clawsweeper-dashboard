- **[P1] Preserve inline cooldown stats when saving:** `src/agents/auth-profiles/usage.ts:857`
  - body: This new inline marker writes `usageStats["inline-api-key:<provider>"]`, but `saveAuthProfileStore` still builds a local store through `pruneAuthProfileStoreReferences`, which keeps usage stats only when the key is a real profile id. The cooldown can therefore be present in memory but dropped before `auth-state.json` is written, so a restart/reload immediately reuses the exhausted inline key again.
  - confidence: 0.92
- **[P2] Use the cfg default agent dir in availability checks:** `src/agents/model-auth.ts:1081-1083`
  - body: `resolveApiKeyForProvider` now resolves the cfg default agent dir before checking inline cooldowns, but `hasAvailableAuthForProvider` still resolves its store from only `params.agentDir`. Callers that pass `cfg` without `agentDir` can read the main auth store, miss the cooled inline key under the configured default agent, and report the provider as available.
  - confidence: 0.84
