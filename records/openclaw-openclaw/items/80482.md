Full review comments:

- **[P2] Check profile fallbacks before returning unavailable:** `src/agents/model-auth.ts:1075-1079`
  - body: When an env-backed inline key is cooled down, this branch returns `false` before examining auth-profile order. A provider with a cooled inline env key and a healthy stored profile will be reported unavailable even though normal profile-first resolution can still use that profile, so let this path fall through to the profile checks before declaring the provider unavailable.
  - confidence: 0.87
- **[P2] Use the resolved agent directory for cooldown checks:** `src/agents/model-auth.ts:777`
  - body: These new early checks load the auth store with `params.agentDir`, but this function already resolves `agentDir` from `cfg` when the caller omits it. Calls for a configured default agent can therefore check the global default store instead of the agent store where the inline cooldown was recorded, letting the exhausted key be reused.
  - confidence: 0.8
