## Summary

Keep open. The cleaned-up branch is now focused on the auth fix, but it still leaves documented env-backed inline API keys and warmed provider availability able to bypass the new cooldown state, so the linked billing-loop bug is not fully fixed yet.

## What This Changes

The PR adds provider-level usage state for inline provider API-key failures, checks inline key cooldown during parts of auth resolution, and wires no-profile runner failures to mark that state.

## Best Possible Solution

Finish one provider-level inline-key cooldown path that preserves provenance for literal keys, structured env SecretRefs, legacy env markers, runtime availability, and gateway auth-state invalidation, with focused regression coverage for each path.

## Reproduction Assessment

## Solution Assessment

No. The provider-level cooldown design is the right direction, but the current implementation needs to carry inline credential provenance through env-backed config and provider availability before it is the narrow maintainable fix.
