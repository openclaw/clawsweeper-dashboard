## Summary

Keep open: the PR is directionally right and the author has addressed several earlier auth-path gaps, but the latest head still appears to drop the new inline cooldown entry when the auth store is saved, so the central billing-cooldown behavior is not merge-safe yet.

## What This Changes

The PR adds provider-level cooldown tracking for inline `models.providers.<id>.apiKey` credentials, gates cooled inline keys during auth resolution and availability checks, marks no-profile runner failures, and adds regression coverage.

## Best Possible Solution
