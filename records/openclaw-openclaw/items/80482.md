## Summary

Keep open. This PR takes the right provider-level cooldown direction, but the actual embedded-runner no-profile billing/auth failure paths still skip the new inline marker, and the supplied proof only exercises the helper directly.

## What This Changes

The PR adds provider-level cooldown tracking for inline `models.providers.<id>.apiKey` credentials and checks that cooldown during auth resolution and availability checks.

## Best Possible Solution

Keep the provider-level, non-secret cooldown identity, but route prompt, assistant, and incomplete-turn billing/auth failures through it whenever the active credential is inline API-key auth with no profile id.

## Reproduction Assessment

Yes, source-level. Inline `models.providers.<id>.apiKey` credentials resolve without `profileId`, and the remaining runner call sites still guard failure marking on a profile id before the PR's no-profile branch can run.

## Solution Assessment

No. The provider-level cooldown approach is maintainable, but this patch must connect the no-profile inline branch to the actual embedded-runner failure paths and add/provide proof for that path.
