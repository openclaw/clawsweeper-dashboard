- **[P2] Preserve the restart reason during deferral:** `src/infra/restart.ts:483`
  - body: After the scheduled timer fires, the runtime clears `pendingRestartReason` before starting `deferGatewayRestartUntilIdle`, but `gateway.restart.pending` only reads `pendingRestartReason ?? emittedRestartReason`. A restart that is preparing and actively deferred will therefore report `pending: true` with `reason: null`, even though this PR advertises reason visibility for restart/update guards.
  - confidence: 0.95

  - body: Adding the core Gateway method advertises it to WebSocket callers, but the bundled admin HTTP RPC allowlist still only permits `gateway.restart.request`. Host/update guards using that documented operator surface will still get a method-not-allowed response instead of the new read-only pending state, so add the method to the admin HTTP allowlist/docs/tests or make the WebSocket-only boundary explicit.
  - confidence: 0.86
