## Summary

Keep open: the PR fixes the reported local Whisper/API prompt mismatch for installable and env-only skills, but it still drops the credential prompt path for documented `primaryEnv` skills that have missing binaries and no installer metadata.

## What This Changes

The PR changes `setupSkills` to include env-only credential skills in the dependency multiselect, gate API-key prompts on selected skill names, add onboarding regression tests, and add a changelog entry.

## Best Possible Solution

Keep opt-in gating for skills shown in the setup multiselect, while preserving the existing credential prompt for documented no-installer `primaryEnv` skills and covering that shape with a regression test.

## Reproduction Assessment

Yes. Source inspection is enough: a missing skill with `requires.bins`, `requires.env`, `primaryEnv`, and no `install` metadata is promptable on current main but is absent from the PR's `configurable` list and then rejected by the universal `installSelected` guard.

## Solution Assessment

No. The opt-in gate is the right direction for installable and env-only choices, but the guard should apply only to skills that were actually presented for selection or otherwise leave non-presented `primaryEnv` prompts reachable.
