## Summary

Keep open: the PR fixes the reported Whisper API-key prompt direction, but the latest guard would remove the existing credential prompt for documented primaryEnv skills that have missing binaries and no install metadata.

## What This Changes

The branch changes skill onboarding to put env-only credential setup in the dependency multiselect, gate API-key prompts on explicit selection, and add regression tests plus a changelog entry for #74382.

## Best Possible Solution

Track which skills were actually presented in the multiselect, and skip the post-install key prompt only for presented-but-unselected skills while preserving the existing prompt for primaryEnv skills omitted because they have no installer metadata.

## Reproduction Assessment

Yes. Source inspection is enough: current main prompts for missing primaryEnv skills regardless of install selection, and the PR patch skips any primaryEnv skill not present in installSelected, including documented no-installer missing-bin shapes.

## Solution Assessment

No. The opt-in gate is the right fix direction for the Whisper report, but the guard needs to apply only to skills that were actually offered in the setup multiselect.
