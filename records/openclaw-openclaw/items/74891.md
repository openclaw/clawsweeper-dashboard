## Summary

Keep open: the PR addresses the reported `openai-whisper` / `openai-whisper-api` prompt confusion, but current head still blocks the existing credential prompt path for documented primaryEnv skills that have missing binaries and no installer metadata.

## What This Changes

This PR changes `openclaw onboard` skill setup to gate installable and env-only credential prompts on explicit multiselect selection, adds onboarding regression tests, and records a changelog fix for #74382.

## Best Possible Solution

Gate credential prompts only for skills that were actually presented as optional setup choices, while preserving the current prompt path for non-selectable primaryEnv skills and covering that shape with a regression test.

## Reproduction Assessment

Yes. Source inspection reproduces the remaining PR regression with a missing skill that has `requires.bins`, `requires.env`, `primaryEnv`, and no `install` metadata: current main prompts for its key, while the PR excludes it from `configurable` and then skips it because it is not in `installSelected`.

## Solution Assessment

No. The PR direction is right for the reported selected/unselected Whisper cases, but the broad `installSelected` guard should be scoped to skills shown in the multiselect or otherwise preserve no-installer credential prompts.
