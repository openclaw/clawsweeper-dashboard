## Summary

Keep open. The PR is aimed at the right onboarding bug, but the proposed guard still allows the reported API-key prompt when the API skill has no missing binary dependency, and the user-facing fix is missing the required changelog entry.

## What This Changes

The PR tracks skills selected in the onboarding dependency multiselect, skips primary environment prompts for unselected skills with missing binary dependencies, and adds regression tests for skipped and selected install paths.

## Best Possible Solution

Credential prompts should be tied to an explicit skill configuration choice for skipped skills, including primaryEnv-only cases, with regression coverage and a CHANGELOG.md entry for the onboarding fix.

## Reproduction Assessment

Yes. Model openai-whisper-api with curl already satisfied and OPENAI_API_KEY missing, then select only local openai-whisper; the PR's missing-bins-only guard still lets the API-key confirm prompt run.

## Solution Assessment

No. Tracking selected installs is useful, but the guard should suppress prompts for unselected primaryEnv skills regardless of whether their binary requirements are already satisfied.
