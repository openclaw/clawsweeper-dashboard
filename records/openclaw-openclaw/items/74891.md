## Summary

Keep open. The proposed patch is directionally useful, but it does not fully fix the linked onboarding bug: skipped API-backed skills can still prompt for credentials when their binary requirement is already satisfied, which is the common `openai-whisper-api` case on systems with `curl` installed.

## What This Changes

The PR records which skills were selected in the onboarding dependency multiselect and skips primary environment-variable prompts for unselected skills that still have missing binary dependencies, with regression tests for skipped and selected install paths.

## Best Possible Solution

Revise the onboarding prompt contract so credential prompts are tied to an explicit skill configuration choice rather than only to missing binary installs. The regression coverage should include `openai-whisper-api` with `curl` already satisfied and only `OPENAI_API_KEY` missing, plus a changelog entry for the user-facing fix.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to model `openai-whisper-api` with no missing bins and missing `OPENAI_API_KEY`, select only local `openai-whisper`, and observe that the proposed loop still reaches the API-key confirm prompt.

## Solution Assessment

No. Tracking selected installs is useful, but the current guard is too narrow because it treats env-only primary-env skills as always promptable even when the user skipped or never selected that skill.
