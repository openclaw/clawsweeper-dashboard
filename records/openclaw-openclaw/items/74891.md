## Summary

Keep this PR open because the follow-up fix handles the reported Whisper/API split for installable and env-only cases, but the latest diff still blocks credential prompts for a documented skill shape that has missing bins plus `primaryEnv` and no installer metadata.

## What This Changes

The PR changes `openclaw onboard` to gate skill API-key prompts on explicit setup multiselect choices, adds onboarding regression tests, and records a changelog fix for #74382.

## Best Possible Solution

Keep the opt-in gate for skills shown in the setup multiselect, while preserving or explicitly presenting credential configuration for primaryEnv skills that cannot be installed by onboarding.

## Reproduction Assessment

Yes. Source inspection reproduces the current-main bug path for the linked report and also the PR regression path: a missing skill with bins, env, primaryEnv, and no install metadata reaches the prompt on main but is never selectable under the PR guard.

## Solution Assessment

No. The PR is the right direction for the reported Whisper/API-key prompt, but the guard is too broad unless it is scoped to skills actually presented for setup or a configure-only path is added for no-installer primaryEnv skills.
