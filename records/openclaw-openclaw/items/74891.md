## Summary

Keep open: the PR fixes the reported OpenAI Whisper prompt confusion in the common installable/env-only paths, but the final guard still drops credential prompts for documented skills that require bins and env while having no installer metadata.

## What This Changes

The PR changes onboard skill setup so API-key prompts are gated by the dependency multiselect selection, adds onboarding regression tests, and records a changelog fix for the linked Whisper issue.

## Best Possible Solution

Preserve explicit opt-in for skills shown in the setup multiselect, while keeping credential prompts reachable for supported primaryEnv skills that cannot be selected because they have no installer metadata.

## Reproduction Assessment

Yes. Source inspection reproduces the original issue on current main via the unconditional `primaryEnv` loop, and reproduces the remaining PR regression with a missing-bin plus missing-env primaryEnv skill that has no install metadata.

## Solution Assessment

No. The PR direction is right for `openai-whisper` and `openai-whisper-api`, but the current guard is broader than the documented skill contract and should only suppress prompts for skills that were actually presented and skipped.

## Review Findings
