## Summary

Keep open: the PR fixes the reported opt-in confusion, but the current head still removes the credential prompt path for documented `primaryEnv` skills that have missing binaries and no installer metadata.

## What This Changes

The PR gates onboard skill API-key prompts on explicit dependency-multiselect selection, adds onboarding regression tests, and records a changelog fix for #74382.

## Best Possible Solution

Keep the explicit opt-in behavior for presented skills, but preserve a credential prompt path for documented non-installable `primaryEnv` skills and cover that shape with a regression test.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a missing skill with `install: []`, missing binaries, `primaryEnv`, and missing env is excluded from PR head `configurable` and then skipped by `!installSelected.has(skill.name)`.

## Solution Assessment

No. The opt-in direction is right for the reported `openai-whisper` case, but the PR applies the guard too broadly; tracking which skills were actually presented, or exempting non-presented no-installer skills, is narrower and preserves current behavior.
