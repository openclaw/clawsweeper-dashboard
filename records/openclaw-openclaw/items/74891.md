## Summary

Keep open. The branch fixes the reported Whisper env-only path and adds the required changelog entry, but the latest guard overcorrects and suppresses API-key setup for documented primaryEnv skills that also have missing binaries but no installer option.

## What This Changes

The PR changes skill onboarding to gate post-install API-key prompts on explicit multiselect choices, adds env-only skill selection coverage, updates regression tests, and adds a changelog entry for #74382.

## Best Possible Solution

Keep the explicit opt-in gate for skills shown in the setup multiselect, but preserve a reachable primaryEnv prompt for missing primaryEnv skills that were never selectable because they have no installer metadata, with a regression test for that documented shape.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction: model a missing skill with requires.bins plus requires.env plus primaryEnv and no install metadata; current main reaches the API-key prompt, while this PR never adds it to configurable and then skips it at the installSelected guard.

## Solution Assessment

No. Tracking explicit choices is the right direction for selected/skipped installer and env-only cases, but the guard needs to apply only to skills that were actually presented for opt-in so documented non-installable primaryEnv skills remain configurable.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Keep non-installable key prompts reachable:** `src/commands/onboard-skills.ts:219-220`
  - body: The new `installSelected` guard skips every primaryEnv prompt unless the skill was selected from `configurable`, but `configurable` excludes skills with missing bins and no install metadata. Scope the skip to skills that were actually presented in the multiselect, and add a regression for a non-installable primaryEnv skill with missing bins and env.
