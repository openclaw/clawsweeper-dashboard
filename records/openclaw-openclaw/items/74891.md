## Summary

Keep open. The branch addresses the reported keyed/keyless onboarding prompt path, but its latest guard still drops credential setup for a documented skill shape: missing binaries plus missing primaryEnv and no installer metadata.

## What This Changes

The PR gates onboard skill credential prompts on explicit dependency-multiselect selection, adds regression tests for binary-bearing and env-only skills, and adds a changelog entry for #74382.

## Best Possible Solution

Keep the explicit opt-in gate for skills shown in the multiselect, while preserving a reachable primaryEnv prompt for documented missing-bin skills that have no installer metadata, with a focused regression test.

## Reproduction Assessment

Yes. Source inspection is enough: model a missing skill with requires.bins, requires.env, primaryEnv, and no install metadata; current main reaches the credential prompt, while this PR never adds it to configurable and then skips it at installSelected.

## Solution Assessment

No. Tracking explicit choices is the right direction for installable and env-only choices, but the skip should apply only to skills that were actually presented for opt-in, or the PR should provide an equivalent reachable configuration path.
