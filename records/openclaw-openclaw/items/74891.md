## Summary

Keep open: the direct Whisper/API prompt bug is real, but the latest PR still suppresses an existing credential prompt path for documented no-installer skills, so it should not merge as-is.

## What This Changes

The PR gates onboarding skill API-key prompts on explicit setup multiselect choices, adds onboarding regression tests, and records a changelog fix for #74382.

## Best Possible Solution

Keep the opt-in behavior for skills shown in the setup multiselect, but preserve the existing credential prompt for no-installer `primaryEnv` skills and cover that shape with a regression test.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main prompts every missing `primaryEnv`, while the PR skips a documented no-installer missing-bin `primaryEnv` skill because it can never enter `installSelected`.

## Solution Assessment

No. The direction is right for the reported Whisper/API prompt confusion, but the guard should require selection only for skills actually offered in the setup multiselect, not for every credentialed missing skill.
