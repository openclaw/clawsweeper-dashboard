## Summary

Keep open: the latest patch fixes the reported OpenAI Whisper API opt-in paths, but it still blocks API-key prompts for documented `primaryEnv` skills with missing binaries and no install metadata because those skills can never enter `installSelected`.

## What This Changes

The PR changes CLI skill onboarding to add env-only `primaryEnv` skills to the dependency multiselect, gates post-install API-key prompts on selected skill names, adds regression tests, and updates the changelog for #74382.

## Best Possible Solution

Preserve explicit opt-in for skills shown in the setup multiselect while keeping the existing key prompt reachable for `primaryEnv` skills that cannot be presented because they have no installer metadata.

## Reproduction Assessment

Yes. Source inspection is enough: model a missing skill with `missing.bins`, `missing.env`, `primaryEnv`, and `install: []`; current main reaches the key confirm, while this PR excludes it from `configurable` and then skips it at the universal `installSelected` guard.

## Solution Assessment

No. The opt-in gate is the right direction for the reported OpenAI Whisper and env-only cases, but the guard should apply only to skills that were actually presented for selection.
