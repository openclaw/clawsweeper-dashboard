## Summary

Keep open. The PR fixes part of the onboarding prompt bug, but the current guard still leaves the reported `openai-whisper-api` prompt reachable when `curl` is already installed, and it introduces a prompt regression for documented non-installable `primaryEnv` skills; it also lacks the required changelog entry.

## What This Changes

The PR tracks selected skills in onboarding's dependency multiselect, uses that set to gate primary environment prompts for skills with missing binaries, and adds onboarding regression tests.

## Best Possible Solution

Credential prompts should follow explicit skill configuration choices while preserving documented non-installable skill prompts, with focused regression coverage and a `CHANGELOG.md` entry.

## Reproduction Assessment

Yes. Model `openai-whisper-api` with `curl` already satisfied and `OPENAI_API_KEY` missing, then select only local `openai-whisper`; the PR guard does not skip because `missing.bins.length` is empty, so the API-key confirm prompt still appears.

## Solution Assessment

No. Tracking selected installs is useful, but the guard should distinguish skipped installable skills from env-only and non-installable `primaryEnv` cases instead of keying only on missing binary count.
