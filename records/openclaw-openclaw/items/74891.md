## Summary

Keep open. The PR targets a real onboarding bug from #74382, but the patch is still incomplete because the new guard only skips unselected skills when they still have missing binaries; an API skill with `curl` already present and only `OPENAI_API_KEY` missing still reaches the credential prompt. The production change also lacks the required changelog entry.

## What This Changes

The PR records selected onboarding skill dependency installs, skips primary env prompts for unselected skills with missing binary dependencies, and adds two Vitest cases for skipped and selected install paths.

## Best Possible Solution

Tie credential prompts to explicit skill configuration intent rather than only to missing binary installation state, add a regression case for the satisfied-binary/env-missing API skill path, and include the required user-facing changelog entry.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to model `openai-whisper-api` with `curl` satisfied and `OPENAI_API_KEY` missing, select only local `openai-whisper`, and observe that the proposed guard does not skip the API-key confirm prompt because `missing.bins` is empty.

## Solution Assessment

No. Tracking selected installs is useful, but this patch is too narrow because env-only or satisfied-binary skills can still prompt even when the user did not choose that skill; a safer fix should gate credential prompting on explicit configuration intent and cover that case in tests.
