## Summary

Keep open: the latest patch fixes the reported Whisper selection path, including the curl-present env-only case, but it still removes the existing API-key setup path for documented `primaryEnv` skills that have missing binaries and no install metadata because those skills are never selectable and fail the new `installSelected` guard.

## What This Changes

The PR gates onboard skill API-key prompts on explicit skill setup multiselect choices, adds onboarding regression tests, and records a changelog fix for #74382.

## Best Possible Solution

Keep the explicit opt-in behavior for skills shown in the setup multiselect, but preserve the existing credential prompt for `primaryEnv` skills that are not selectable because they have no installer metadata, with regression coverage for that shape.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction: model a missing skill with `requires.bins`, `requires.env`, `primaryEnv`, and no `install` metadata; current main reaches the key prompt, while this PR never puts it in `configurable` and then skips it at `installSelected`.

## Solution Assessment

No. The direction is right for the reported `openai-whisper` and `openai-whisper-api` cases, but the unconditional `installSelected` guard is too broad for documented skills that cannot be presented in the multiselect.

## Review Findings

Overall correctness: patch is incorrect

- **[P2] Keep non-installable key prompts reachable:** `src/commands/onboard-skills.ts:219-220`
