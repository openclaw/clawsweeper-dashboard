## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve the output cap across completions:** `src/auto-reply/reply/acp-projector.ts:505`
  - body: This call preserves `finalOnlyOutputText` across intermediate `done` events, but `resetTurnState` still resets `emittedOutputChars` and `truncationNoticeEmitted`. A final_only turn with multiple completions can now accept up to `maxOutputChars` per completion and emit duplicate truncation notices, so the final message can exceed the configured ACP output limit. Keep those counters with the accumulated text until the explicit final flush, and add a regression with a small `maxOutputChars`.
  - confidence: 0.9
- **[P2] Carry hidden-boundary state across preserved completions:** `src/auto-reply/reply/acp-projector.ts:505`
  - body: Default final_only hides tool-call tags and uses a paragraph hidden-boundary separator. When text before a hidden tool call is preserved across `done`, this reset clears `pendingHiddenBoundary` and `lastVisibleOutputTail`, so the next visible text is appended without the configured separator. Preserve the boundary/tail state with the accumulated output, and cover a hidden tool_call -> done -> text case.
  - confidence: 0.84
