## Summary

Keep open: current main still normalizes Feishu chat targets for delivery but `resolveFeishuOutboundSessionRoute` defaults normalized `oc_` targets to `direct`, so outbound and delivery-mirror session entries can still split from group sessions. The related inbound PR https://github.com/openclaw/openclaw/pull/52342 is open and separately reviewed; it does not close this outbound/session-key path.

## What This Changes

Request to stop Feishu OpenChat `oc_` chats from getting separate outbound/direct and inbound/group session keys.

## Best Possible Solution

Fix the Feishu plugin-owned outbound route normalization so `chat:oc_...`, delivery-mirror replies, and bare OpenChat chat IDs produce the same group session key; handle legacy duplicate cleanup separately if maintainers want it.

## Reproduction Assessment

Yes by source inspection. Current main can route a normalized Feishu `oc_...` target through the outbound session helper as `direct`, and the May 8 comment confirms the same session shape on OpenClaw 2026.5.7.

## Solution Assessment

Yes for the narrow Feishu route fix. Generic session-key deduplication and startup reconciliation are broader cleanup mechanisms and should not be the first-line fix for this plugin-owned routing bug.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/58537. Current main records Feishu `oc_...` OpenChat/chat targets as `direct` in outbound/delivery-mirror session metadata even though Feishu docs and delivery target handling treat them as chat/group IDs. Make the Feishu outbound session route helper preserve group/channel classification for normalized Feishu chat targets, including targets produced from `chat:oc_...` by `resolveDeliveryTarget` and bare `oc_...` OpenChat sends, so the route/session key/chatType/from fields are group-consistent. Prefer using the existing `ChannelOutboundSessionRouteParams.resolvedTarget` and Feishu target semantics rather than adding a core migration. Do not change the inbound `p2p` OpenChat classifier from https://github.com/openclaw/openclaw/issues/52238 or https://github.com/openclaw/openclaw/pull/52342, do not add generic startup session merging, and do not change unrelated channel behavior. Add focused regression coverage for `resolveFeishuOutboundSessionRoute` and the normal reply/delivery-mirror path where `chat:oc_...` is normalized before session routing. Add a changelog entry because this is user-visible Feishu behavior.
