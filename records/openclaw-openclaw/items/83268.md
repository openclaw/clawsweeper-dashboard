## Summary

Keep open: current main still has no bare `claude` auth alias or orphan-profile repair, so a stored `claude:*` credential remains incompatible with `anthropic/*` auth resolution even though the canonical Anthropic API-key path is documented and registered under `anthropic`.

## What This Changes

The issue reports that Anthropic API keys can be stored as `claude:manual`, leaving them unused for `anthropic/*` models unless both auth-profile JSON registries are manually synchronized.

## Best Possible Solution

Normalize or repair Anthropic API-key profiles to canonical `anthropic`, and add focused status or doctor coverage for existing `claude:*` orphan profiles without creating a broad ambiguous `claude` alias unless maintainers explicitly want that compatibility behavior.

## Reproduction Assessment

Yes for the resolver state: source inspection shows `claude:*` profiles are filtered out for `anthropic/*` model auth unless an alias or repair exists. The full interactive wizard path that originally creates `claude:manual` was not live-reproduced in this read-only pass.

## Solution Assessment

Unclear. The manual two-file edit is a workaround; the maintainable fix should be a bounded provider-owned normalization, repair, or diagnostic path chosen by maintainers.
