## Summary

Keep open: current main still treats a stored `claude` auth profile as incompatible with `anthropic/*` model auth, and I found no merged fix or canonical duplicate for this specific Anthropic API-key orphaning path.

## What This Changes

The issue reports that Anthropic API keys can be stored as `claude:manual`/`provider: claude`, leaving them unused by `anthropic/*` model routing.

## Best Possible Solution

Normalize Anthropic API-key profiles to canonical `anthropic` at the provider-owned setup boundary, and add a bounded doctor/auth-status warning for existing `claude:*` orphan profiles.

## Reproduction Assessment

Partly. Source inspection gives a high-confidence reproduction of the orphaned-profile routing state for `claude:manual` versus `anthropic/*`, but I did not run the full interactive wizard path that creates it.

## Solution Assessment

Unclear. The manual two-file edit is only a workaround; the maintainable fix should be provider-owned normalization or a narrow repair/diagnostic, not an unreviewed broad `claude` auth alias.
