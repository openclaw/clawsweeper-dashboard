## Summary

Keep open: current main still treats a stored `claude:*` auth profile as incompatible with `anthropic/*` model auth, and I found no merged fix or canonical duplicate for the existing orphan-profile repair path. The current Anthropic API-key setup path is canonical, but existing `claude:manual` profiles still need an auth-provider decision on repair, aliasing, or diagnostics.

## What This Changes

Anthropic API keys entered through the auth wizard should be stored and resolved under canonical `anthropic` profiles instead of becoming orphaned `claude` profiles.

## Best Possible Solution

Normalize Anthropic API-key creation to canonical `anthropic` and add a bounded doctor/status repair or warning for existing `claude:*` orphan profiles without a broad alias unless maintainers approve it.
