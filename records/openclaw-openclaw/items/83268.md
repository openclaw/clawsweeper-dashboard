## Summary

Keep open: current main still makes a stored `claude:*` auth profile invisible to `anthropic/*` auth resolution, and I found no merged fix or exact canonical duplicate for this API-key orphaning path. The canonical Anthropic API-key setup path appears to use `anthropic`, so the remaining decision is how to repair or diagnose existing misbucketed profiles without adding an unsafe broad alias.

## What This Changes

The issue reports that an Anthropic API key can be stored as `claude:manual`, leaving it orphaned from `anthropic/*` model routing and causing fallback to Claude OAuth.

## Best Possible Solution

Add a bounded Anthropic-owned repair or doctor diagnostic that normalizes known Anthropic API-key profiles from `claude:*` to `anthropic:*` in both config and auth store, while keeping new setup on the canonical `anthropic` provider and avoiding a broad `claude` alias unless maintainers explicitly choose it.
