## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve buffered boundaries when merging media:** `src/auto-reply/reply/block-reply-pipeline.ts:253-258`
  - body: `drainBufferedText()` only returns text, but the existing coalescer buffer also carries reply target, voice, reasoning, and status state. When a buffered block with one of those boundaries is followed by media, this branch now sends a single spread media payload instead of preserving the old split, which can mis-target or expose the wrong message content. Drain a structured buffered payload/state and merge only when the buffered and media payload boundaries are compatible.
  - confidence: 0.9
- **[P2] Use the coalescer joiner for media captions:** `src/auto-reply/reply/block-reply-pipeline.ts:257`
  - body: When the media payload already has text, this concatenates `bufferedText` directly with `payload.text`, ignoring the configured coalescer joiner and the normalized `reply.text`. A streamed `Hello` followed by a media caption `world` would send `Helloworld` and can miss final-payload dedupe; build the merged text with the same joiner semantics as the coalescer.
  - confidence: 0.86
