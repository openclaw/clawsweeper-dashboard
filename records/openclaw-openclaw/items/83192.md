## Summary

Keep open: current main still has the orphan `liveModelSwitchPending` reset behavior at source level, and the linked closing PR is still open rather than merged.

## What This Changes

The issue reports that `sessions.patch` with `model: null` clears model override fields but leaves `liveModelSwitchPending` persisted, which can later surface stale model/provider labels.

## Best Possible Solution

Review and land or close https://github.com/openclaw/openclaw/pull/83213; once a fix merges, close this issue as implemented with the merge commit and release provenance.

## Reproduction Assessment

Yes, at source level: current main’s `sessions.patch` reset path passes `markLiveSwitchPending: true`, and the focused gateway test expects `liveModelSwitchPending` to remain true after `model: null`. I did not run the gateway repro because this review was read-only.

## Solution Assessment

Yes, the narrow maintainable fix is to clear or avoid setting `liveModelSwitchPending` on explicit model reset while preserving pending live-switch behavior for concrete model changes. The linked PR implements that shape and should be reviewed rather than opening another fix lane.
