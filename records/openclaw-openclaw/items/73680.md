## Summary

Keep this issue open. Current main still rejects the reported per-agent keys because `agents.list[]` is validated through a strict `AgentEntrySchema` that omits `verboseDefault` and `elevatedDefault`, while defaults, generated docs, and runtime consumers still expose the values; open PR #74643 is the active unmerged fix path.

## What This Changes

Allow per-agent `agents.list[].verboseDefault` and `agents.list[].elevatedDefault` to match existing runtime resolution and config type expectations.

## Best Possible Solution

Land #74643 once refreshed, or an equivalent narrow fix that adds both per-agent schema fields, aligns types/help/docs/generated config metadata, and covers valid plus invalid enum cases.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: `agents.list[]` is validated by strict `AgentEntrySchema`, and that schema omits both reported keys while global defaults and runtime consumers still accept or read them.

## Solution Assessment

Yes. Adding the missing per-agent schema/type/help/docs/test coverage is the narrowest maintainable fix because runtime code already consumes these defaults; removing the runtime surface would be broader and more disruptive.
