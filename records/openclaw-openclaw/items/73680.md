## Summary

Keep this issue open. Current main still rejects the reported per-agent keys: `agents.list[]` goes through strict `AgentEntrySchema`, which omits `verboseDefault` and `elevatedDefault`, while defaults and runtime paths still expose those settings; open PR #74643 is the active unmerged implementation for this issue.

## What This Changes

The issue asks OpenClaw to accept `agents.list[].verboseDefault` and `agents.list[].elevatedDefault`, or remove the runtime/type surfaces that imply per-agent support.

## Best Possible Solution

Land #74643 once refreshed, or land an equivalent narrow fix that adds both per-agent schema fields, updates public types/help/generated docs, and covers valid plus invalid enum cases.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: `agents.list[]` is validated through strict `AgentEntrySchema`, and that schema omits both reported keys while global defaults accept them.

## Solution Assessment

Yes. Adding the missing per-agent schema/type/help/docs/test coverage is the narrowest maintainable fix because runtime code already consumes these defaults; removing the runtime surface would be broader and more disruptive.
