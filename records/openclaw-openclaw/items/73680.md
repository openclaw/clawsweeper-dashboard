## Summary

Keep this issue open. Current main still has the reported config-contract mismatch: `agents.list[]` is validated through strict `AgentEntrySchema`, that schema omits `verboseDefault` and `elevatedDefault`, and the active closing implementation is still open PR #74643 rather than merged code.

## What This Changes

The issue asks OpenClaw to align per-agent `verboseDefault` and `elevatedDefault` config handling so the strict schema, types, docs, and runtime resolver agree.

## Best Possible Solution

Review and land PR #74643, or an equivalent narrow fix, so per-agent schema validation, public config types, generated schema/help metadata, docs, and regression tests all support `verboseDefault` and `elevatedDefault` consistently.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: `agents.list` validates through strict `AgentEntrySchema`, that schema omits the two keys, and Zod strict objects reject unknown keys; I did not run the CLI because this review is read-only.

## Solution Assessment

Yes. Adding the two fields to the per-agent schema and documenting their precedence is the narrowest maintainable solution because current runtime/status paths already consume per-agent values, while removing those paths would be broader and more disruptive.
