## Summary

Keep this issue open. Current main still has the reported schema/runtime mismatch: `agents.list[]` is validated by a strict schema that omits `verboseDefault` and `elevatedDefault`, while defaults and runtime/status paths still consume those values; #74643 is the active open fix PR with closing syntax.

## What This Changes

Allow per-agent `verboseDefault` and `elevatedDefault` config overrides, or remove/document the per-agent runtime surface if they should remain global-only.

## Best Possible Solution

Land #74643 or an equivalent narrow fix that adds both per-agent schema fields, aligns generated metadata/help/docs/types, and covers valid plus invalid enum cases.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: `agents.list[]` flows through strict `AgentEntrySchema`, and that schema still omits both reported keys while runtime/default consumers read them.

## Solution Assessment

Yes. Adding the missing schema/type/help/docs/test coverage is the narrowest maintainable fix because runtime code already consumes these defaults; removing the runtime surface would be broader and more disruptive.
