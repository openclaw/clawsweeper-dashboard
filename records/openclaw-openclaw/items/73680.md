## Summary

Keep this issue open. Current main still has the reported config-schema mismatch: strict `agents.list[]` validation omits `verboseDefault` and `elevatedDefault`, while defaults and runtime/status paths still expose related values; open PR #74643 is the active unmerged fix path.

## What This Changes

Allow `verboseDefault` and `elevatedDefault` under per-agent `agents.list[]` entries, or otherwise remove/document the unsupported per-agent surface.

## Best Possible Solution

Review and land #74643, or an equivalent narrow fix that aligns per-agent validation, public types, generated metadata, docs/help text, and regression tests with the existing default-resolution surface.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists on current main: `agents.list[]` is validated through a strict `AgentEntrySchema` that omits both reported keys, so those keys are rejected before runtime can use them.

## Solution Assessment

Yes. Adding the missing per-agent schema/type/help/docs/generated metadata and parser regression coverage is the narrowest maintainable fix because current runtime paths already consume related defaults; removing the runtime surface would be broader.
