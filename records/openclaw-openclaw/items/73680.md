## Summary

Keep this issue open. Current main still has the reported schema/runtime mismatch: `agents.list[]` is validated by a strict per-agent schema that omits `verboseDefault` and `elevatedDefault`, while defaults and runtime/status paths still expose related values; open PR #74643 is the active unmerged fix path.

## What This Changes

Allow or explicitly de-scope per-agent `verboseDefault` and `elevatedDefault` so the config schema matches the resolver, SDK/type surface, and docs.

## Best Possible Solution

Land #74643 once refreshed, or an equivalent narrow fix that adds both per-agent schema fields, aligns types/help/docs/generated metadata, and covers valid plus invalid enum cases.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists on current main: `agents.list[]` flows through strict `AgentEntrySchema`, and that schema omits both reported keys while defaults and runtime/status consumers still expose related values.

## Solution Assessment

Yes. Adding the missing per-agent schema/type/help/docs/test coverage is the narrowest maintainable fix because runtime code already consumes these defaults; removing the runtime surface would be broader and more disruptive.
