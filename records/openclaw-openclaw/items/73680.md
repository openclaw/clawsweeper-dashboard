## Summary

Keep this issue open because current main still rejects the reported per-agent config keys while related runtime/type/doc surfaces remain inconsistent, and the active closing implementation is still an open unmerged PR.

## What This Changes

## Best Possible Solution

Align the config contract by accepting both keys on `agents.list[]`, exposing both through the resolved agent config used by directive/status paths, updating labels/help/generated schema/docs, and adding parser plus resolver precedence tests for valid and invalid values.
