## Summary

Keep this issue open. Current `main` still has the reported config-contract mismatch: `agents.list[]` is validated through strict `AgentEntrySchema`, that schema still lacks `verboseDefault` and `elevatedDefault`, and the active closing implementation is open PR #74643 rather than merged code.

## What This Changes

The issue requests that per-agent `agents.list[]` entries accept `verboseDefault` and `elevatedDefault`, or that runtime/types/docs stop implying those per-agent overrides exist.

## Best Possible Solution

Review and land PR #74643 or an equivalent narrow fix that aligns the per-agent schema, config types, generated schema/help metadata, docs, and regression tests before closing this issue.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: `agents.list` validates through strict `AgentEntrySchema`, and that schema omits the two reported keys, so config validation rejects them as unrecognized; I did not run the CLI because this review is read-only.

## Solution Assessment

Yes. Accepting the two fields in the per-agent schema and documenting their precedence is the narrowest maintainable fix because current runtime/status/test surfaces already treat these values as per-agent-capable, while removing those paths would be broader and more disruptive.
