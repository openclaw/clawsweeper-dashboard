## Summary

Keep this issue open. Current main still omits `verboseDefault` and `elevatedDefault` from the strict per-agent schema while defaults and runtime paths still accept/read those settings, and open PR #74643 is the active unmerged closing implementation.

## What This Changes

Allow durable per-agent `verboseDefault` and `elevatedDefault` config entries, or otherwise remove and document the unsupported per-agent surface.

## Best Possible Solution

Land #74643 after refresh, or an equivalent narrow fix that aligns schema, public type/help/docs/generated schema, and regression tests with the existing resolver behavior.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: `validateConfigObjectRaw` reaches `OpenClawSchema`, `agents.list` uses strict `AgentEntrySchema`, and that schema omits both reported keys while defaults accept them.

## Solution Assessment

Yes. Adding the two missing per-agent schema fields plus synchronized type/help/docs/generated schema and enum regression tests is the narrowest maintainable fix; removing the runtime surface would be broader and disruptive.
