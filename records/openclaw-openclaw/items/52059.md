## Summary

Keep open: current main still lacks `gateway.auth.scopes`, and the live PR head still has merge-blocking gateway auth issues around trusted-proxy scope regranting and stale generated config schema artifacts.

## What This Changes

The PR adds a `gateway.auth.scopes` config field and re-applies configured scopes to device-less shared-auth WebSocket connections after unbound scopes are cleared.

## Best Possible Solution

Agree the gateway auth boundary, then land an implementation that gates configured scopes to verified token/password auth, regenerates schema artifacts, adds changelog/docs/regression coverage, and covers trusted-proxy and Tailscale paths.

## Reproduction Assessment

Yes. Source and existing tests show a current-main path where a device-less operator connects with a valid shared token, has scopes cleared, and then fails a scoped call with `missing scope`.

## Solution Assessment

No. The feature direction may be useful, but the current PR head is not the best mergeable fix because it uses a trusted-proxy-inclusive auth gate and leaves generated config artifacts and release notes stale.
