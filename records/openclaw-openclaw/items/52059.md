## Summary

Keep open: the PR addresses a real gateway auth gap, but the current head still has a security-sensitive overbroad auth gate, config-baseline drift, no changelog, and no after-fix real behavior proof.

## What This Changes

The PR adds a `gateway.auth.scopes` config option and re-applies those configured scopes to device-less shared-auth WebSocket connections.

## Best Possible Solution

Agree the gateway auth boundary first, then land a current-main implementation with an explicit token/password auth-method gate, regenerated config baseline, changelog/docs, and regression coverage for token, password, trusted-proxy, and Tailscale paths.

## Reproduction Assessment

Yes. Source and existing tests show current main clears requested scopes for device-less shared-token/password operator connects, and the requested config field is absent from current main.

## Solution Assessment

No. The feature direction may be valid, but this PR is not the best mergeable fix because it gates scope regranting on a trusted-proxy-inclusive predicate and leaves required release/config artifacts missing.
