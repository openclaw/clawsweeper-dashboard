## Summary

Keep this PR open: current main still has the device-less shared-auth scope gap, but the supplied PR head is not merge-ready because it broadens scope grants through `sharedAuthOk`, leaves generated config schema artifacts and changelog coverage missing, and needs to be ported to the current auth resolver split.

## What This Changes

The PR adds a `gateway.auth.scopes` config option intended to grant configured scopes to device-less token/password gateway connections for auto-pairing flows.

## Best Possible Solution

Agree the gateway auth boundary first, then implement a token/password-only device-less scope grant on current main with generated config artifacts, docs/changelog, and regression coverage for token, password, trusted-proxy, and identity-bearing paths.

## Reproduction Assessment

Yes, from source and existing tests: current main accepts a device-less operator with a valid shared token but clears its scopes, so privileged calls fail with `missing scope`. I did not run tests because this was a read-only review and `pnpm` is unavailable.

## Solution Assessment

No. The direction may be useful, but this PR is not the best mergeable fix until scope regranting is gated on explicit token/password auth, the resolver changes are ported to current main, generated config artifacts and changelog are updated, and focused tests are added.
