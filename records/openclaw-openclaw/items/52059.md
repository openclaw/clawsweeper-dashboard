## Summary

Keep open. The feature targets a real gateway auth gap, but the current PR still grants configured scopes through a trusted-proxy-inclusive predicate, lacks required config-baseline regeneration, and has no after-fix real gateway proof.

## What This Changes

The PR adds a `gateway.auth.scopes` config field, config schema/help labels, auth-resolution plumbing, and WebSocket scope regranting for device-less shared-auth connections.

## Best Possible Solution

Agree the gateway auth boundary first, then land a current-main implementation with an explicit token/password auth-method gate, config baseline/docs updates, and regression coverage plus redacted real gateway proof.

## Reproduction Assessment

Yes. Source and checked-in tests show current main accepts device-less shared-token/password operator connects with scopes cleared, and the proposed `gateway.auth.scopes` field is absent from current main; I did not run a live gateway in this read-only review.

## Solution Assessment

No. The feature direction may be useful, but this PR is not the best mergeable fix because `sharedAuthOk` also covers trusted-proxy auth and the config baseline/proof obligations are still unmet.
