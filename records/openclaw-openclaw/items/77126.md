## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Classify createConnection and Socket.connect callsites:** `scripts/check-raw-socket-callsite-classification.mjs:108-118`
  - body: The matcher only recognizes `.connect` when the receiver is a raw module alias. Current code also uses `net.createConnection(...)` and `new net.Socket().connect(...)`, so new or moved raw sockets in those forms can bypass the classification guard.
  - confidence: 0.92
- **[P2] Include every managed proxy env key:** `scripts/check-managed-proxy-runtime-mutation.mjs:47-58`
  - body: The forbidden key set omits `GLOBAL_AGENT_FORCE_GLOBAL_AGENT` and `OPENCLAW_PROXY_LOOPBACK_MODE`, even though the proxy lifecycle writes both. Those mutations will not be emitted, leaving part of the lifecycle unclassified.
  - confidence: 0.9
- **[P2] Resolve spread-built proxy key arrays:** `scripts/check-managed-proxy-runtime-mutation.mjs:90-100`
  - body: The dynamic-key collector only marks arrays with direct string elements. `ALL_PROXY_ENV_KEYS` is composed from spread constants, so a spread-built proxy-key loop can avoid classification unless another same-named loop variable happened to be marked earlier in the file.
  - confidence: 0.82
- **[P2] Track GLOBAL_AGENT object aliases:** `scripts/check-managed-proxy-runtime-mutation.mjs:170-180`
  - body: The matcher only recognizes direct `global.GLOBAL_AGENT` expressions. The proxy lifecycle writes through local `agent` aliases, so those runtime mutations stay outside the guard.
  - confidence: 0.89
