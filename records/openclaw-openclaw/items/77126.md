## Summary

Keep open. The latest head fixes the earlier static named-import gap, but the new raw-socket guard still misses Socket instance aliases and destructured require/dynamic-import aliases, so this draft maintainer-labeled security guardrail should not merge yet.

## What This Changes

The PR adds raw socket and managed-proxy runtime mutation lint guards, wires them into package and boundary-check scripts, adds guard tests, and extends managed proxy documentation.

## Best Possible Solution

Keep the guardrail direction, but land it only after the raw-socket detector tracks stored Socket instances and destructured require/dynamic-import aliases with focused regression tests.

## Reproduction Assessment

Yes. Source inspection of PR head shows the matcher only catches inline Socket construction, while current main uses a stored Socket instance before connect, and destructured require/dynamic-import aliases are skipped by the identifier-only alias collector.

## Solution Assessment

No. The security lint direction is maintainable, but the detector needs these normal raw-socket call shapes covered before it becomes a boundary pipeline guard.

  - body: The guard allowlists `src/infra/jsonl-socket.ts:35`, but that call is `client.connect(...)` after `const client = new net.Socket()`. Direct calls from those bindings remain unclassified raw socket calls.
