## Label Justifications

- `impact:session-state`: The report concerns stale memory index and sync state causing recall/search state drift rather than missing source memory files.

## Summary

Keep open. Current main and v2026.5.19 still let builtin memory_search query a populated dirty or reindex-needed index while search-triggered sync runs in the background; the related reindex rollback PR is adjacent but explicitly does not change query behavior.

## What This Changes

The issue reports builtin memory_search returning empty results from a stale or transitional builtin memory index until a forced reindex refreshes it.

## Best Possible Solution

Add a builtin memory search preflight that detects dirty, session-dirty, or full-reindex-needed state before querying, then either waits for bounded sync/reindex or returns an explicit transient unavailable result while leaving QMD/provider behavior unchanged.

## Reproduction Assessment

Yes by source inspection: a populated builtin index skips the synchronous bootstrap, then dirty or session-dirty search sync starts in the background while the current query proceeds. I did not run a live BigModel endpoint repro in this read-only review.

## Solution Assessment

Yes, the narrow maintainable fix is a builtin memory-core query preflight/guard with regression coverage; the manual reindex workaround and background search sync are not sufficient.
