## Summary

This PR is still necessary and appears correct: current main still compares GIF content type with a lowercase-only exact match, while the patch reuses the existing MIME normalizer and preserves filename fallback behavior.

## What This Changes

The PR changes `isGifMedia` in media-core to normalize `contentType` before comparing with `image/gif` and adds focused regression coverage for parameterized, padded, non-GIF, and filename-fallback cases.

## Best Possible Solution

Merge the focused shared-helper fix after ordinary maintainer review; it keeps GIF classification in media-core instead of duplicating MIME normalization in Telegram callers.

## Reproduction Assessment

Yes, from source inspection. Current main compares `opts.contentType?.toLowerCase()` directly to `image/gif`, so `image/gif; charset=binary` and padded values do not match without a `.gif` filename fallback.

## Solution Assessment

Yes. Reusing `normalizeMimeType` in media-core is the narrowest maintainable fix; changing only Telegram call sites would duplicate shared MIME policy and leave other callers behind.
