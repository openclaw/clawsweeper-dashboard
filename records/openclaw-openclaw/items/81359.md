## What This Changes

Request is to suppress raw child transcript, tool-result, and latestRawText fallback output when a subagent ends with lost active execution context.

## Best Possible Solution

Review and land the linked fix PR if its proof and gates satisfy maintainers; otherwise keep this issue open until the unsafe lost-context output path is fixed or the PR is rejected with a replacement plan.

## Reproduction Assessment

Yes. A source-level reproduction is clear: pass a lost-context error outcome with child history containing only tool/raw text, and current readSubagentOutput can select latestRawText or fall back to latestAssistantReply.

## Solution Assessment

Yes at the issue level. The narrow maintainable direction is to treat lost active execution context as a conservative failure case, suppress raw/tool fallback output, and preserve normal assistant-authored successful output.
