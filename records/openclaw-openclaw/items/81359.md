## What This Changes

The report asks OpenClaw to suppress raw/tool-result fallback text from subagent completion announcements when a child run ends with `subagent run lost active execution context`.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81360 or an equivalent narrow fix that suppresses lost-context raw/tool fallback while preserving successful assistant final output.

## Reproduction Assessment

Yes, at source level. Current main can emit the lost-context outcome and still lets `readSubagentOutput()` select raw transcript text or latest assistant fallback without checking that outcome.

## Solution Assessment

Yes, the linked PR direction is the narrowest maintainable fix: gate only the lost-active-execution-context output path and add focused regression coverage. It still needs normal PR review and after-fix real behavior proof before merge.
