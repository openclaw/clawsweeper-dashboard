## Summary

Close as implemented: current main no longer records or returns non-assistant/toolResult raw text from subagent history, regression tests cover the old tail-of-tool-output path, and the fixing commit is contained in a shipped release.

## What This Changes

The issue asks OpenClaw to stop surfacing raw startup, tool-result, or transcript context as a subagent completion result after a lost active execution context failure.

## Best Possible Solution

Keep the shipped assistant-only output selection behavior for this leak, and leave the broader lost-context status/output lifecycle mismatch to the separate canonical follow-up.

## Reproduction Assessment

No current-main failing repro remains. The original bug had production and source evidence against older code, but current main ignores toolResult/raw transcript content and tests cover the old fallback shape.

## Solution Assessment

Yes. The shipped fix is the narrow maintainable solution for this issue because it removes the unsafe raw fallback at the output-selection boundary while preserving normal assistant-authored completions.
