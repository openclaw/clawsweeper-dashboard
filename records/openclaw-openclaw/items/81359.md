## Summary

Keep open: current main still allows the lost-context subagent announce path to select raw child output, and the exact closing fix candidate is still open and unmerged.

## What This Changes

The issue asks OpenClaw to prevent lost-context subagent completions from surfacing raw transcript, tool-result, startup, or runtime context as the child result.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/81360 or an equivalent narrow fix that suppresses raw/toolResult/latestRawText/latest-assistant fallback for lost-context outcomes while preserving normal assistant final output.
