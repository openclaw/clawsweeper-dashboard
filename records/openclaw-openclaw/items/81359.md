## Summary

Keep open: current main and the latest release still allow the lost-context subagent error path to capture and return raw child transcript/tool-result text, and the linked closing PR is open and unmerged.

## What This Changes

The issue reports that lost-context subagent completions can expose raw startup or tool-result transcript content instead of returning a bounded failure result.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/81360 or an equivalent focused fix that suppresses raw/toolResult/latestRawText and unsafe latest-assistant fallback for lost-context outcomes while preserving normal assistant-authored completions.

Yes, at source level: current main emits the lost-context error, permits failed-output capture, and can return raw `latestRawText` or latest assistant text without a lost-context guard. I did not run a live subagent failure because this review is read-only.

## Solution Assessment

Yes. The linked PR direction is the narrow maintainable fix because it changes the output selection boundary for this exact error path while preserving normal successful assistant final output.
