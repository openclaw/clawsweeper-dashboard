Keep open: current main still has the source-level raw-output fallback described here, and the issue is already paired with an open closing PR that should be reviewed or replaced before the issue is closed.

## What This Changes

The issue reports that lost-context subagent completion handling can leak raw child transcript or tool-result text into the parent-visible completion result.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/81360 or an equivalent narrow fix that suppresses raw/tool-result fallback only for lost-context failures while preserving normal successful assistant final output.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: a lost-context outcome with only `toolResult` or other raw transcript text still reaches `snapshot.latestRawText`, and current main can return that as the child result. I did not run a live subagent failure because this review is read-only.

## Solution Assessment

Yes, the linked PR direction is the narrow maintainable fix: special-case the lost-active-execution-context outcome in the output selector rather than changing normal successful subagent result handling.
