## Summary

Keep open: current main and the latest release still allow the lost-context subagent error path to capture raw child output, and the open closing PR is the active fix candidate.

## What This Changes

The issue reports that lost-context subagent completions can surface raw child transcript or tool-result text instead of a bounded failure or assistant-authored final result.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/81360 or an equivalent narrow fix that suppresses raw/toolResult/latestRawText/latest-assistant fallback for lost-context outcomes and adds regression coverage.

## Reproduction Assessment

Yes, at source level. Current main emits the lost-context error, permits failed-output capture, and can return `latestRawText` or latest assistant text without a lost-context guard.

## Solution Assessment

Yes. The linked PR direction is the narrow maintainable fix because it changes the output selection boundary for this exact error path while preserving normal successful assistant final output.
