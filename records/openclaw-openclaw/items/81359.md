Keep open: current main still has the unsafe lost-context completion path, and the linked closing fix PR is open and unmerged. Because that PR uses closing syntax for this issue, the issue should remain open until the PR lands or is replaced.

## What This Changes

The issue asks OpenClaw to prevent lost-context subagent completions from surfacing raw tool-result/transcript text or internal startup/runtime context.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/81360 or an equivalent narrow fix that suppresses raw/toolResult/latestRawText and latest-assistant fallback for lost-context outcomes while preserving normal successful final output.

## Reproduction Assessment

Yes, source-level reproduction is high confidence. Current main emits the lost-context error, permits failed-output capture, and can return raw/tool-result `latestRawText` or latest assistant text without a lost-context guard.

## Solution Assessment

Yes, the linked PR direction is the narrow maintainable solution. It targets the output-selection boundary for the exact error outcome instead of adding new policy or configuration.
