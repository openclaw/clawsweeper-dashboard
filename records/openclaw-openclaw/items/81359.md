## Summary

Keep open: current main still has the unsafe lost-context output path, and https://github.com/openclaw/openclaw/pull/81360 is open with closing syntax as the active fix candidate.

## What This Changes

The issue asks OpenClaw to suppress raw startup, runtime, tool-result, and latest-assistant fallback text when a subagent fails with lost active execution context.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/81360 or an equivalent narrow fix that suppresses raw/latest assistant fallback for lost-context outcomes and adds regression coverage plus real behavior proof.

## Reproduction Assessment

Yes, at source level. Current main emits the lost-context error, permits failed-output capture, and can select raw/tool-result `latestRawText` or latest assistant text without a lost-context guard.

## Solution Assessment

Yes. The linked PR direction is the narrow maintainable fix because it changes the output-selection boundary for this exact error path while preserving normal successful assistant final output.
