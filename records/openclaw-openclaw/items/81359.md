## Summary

Close as implemented: current main and shipped releases no longer promote raw tool/toolResult transcript text or latest-assistant fallback into subagent completion results, so the reported lost-context leak path is covered without needing the still-open stale PR.

## What This Changes

The issue reports that lost-context subagent completions can announce raw child transcript or tool output instead of a safe assistant-authored result.

## Best Possible Solution

Keep the shipped selector contract that only uses child assistant-authored completion text and leaves broader lost-context lifecycle variants to their separate canonical issues.

## Reproduction Assessment

No for current main: the old raw/toolResult and latest-assistant fallback paths are gone, and regression coverage now asserts raw tool output is not announced. The issue comments and older release checks still provide high-confidence pre-fix evidence.

## Solution Assessment

Yes. Removing raw fallback at the shared subagent output selector is the narrow maintainable fix for this leak path, and current main also documents and tests that contract.
