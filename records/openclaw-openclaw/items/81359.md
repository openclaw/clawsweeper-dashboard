## What This Changes

The issue reports that lost-context subagent completion errors can expose raw child transcript, tool-result, or startup context instead of a bounded failure result.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/81360 or an equivalent narrow fix that suppresses raw/toolResult/latestRawText/latest-assistant fallback for lost-context outcomes and adds regression coverage.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: current main emits the lost-context error, permits failed-output capture, and can return raw/tool-result `latestRawText` or latest assistant text without a lost-context guard.

## Solution Assessment

Yes. The linked PR direction is the narrow maintainable fix because it changes the output selection boundary for this exact error path while preserving normal successful assistant final output.
