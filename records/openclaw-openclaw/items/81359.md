## Summary

Keep open: current main still has the unsafe lost-context subagent output path, and the open closing PR is the active fix candidate rather than a reason to close this issue before merge.

## What This Changes

The issue reports that lost-context subagent completion handling can surface raw child transcript, startup, or tool-result text to the parent completion flow.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81360 or an equivalent narrow fix that fails closed for lost-context output selection while preserving normal successful assistant final output.

## Reproduction Assessment

Yes. Source inspection on current main shows the lost-context error can reach failed-output capture and then select snapshot.latestRawText or latest assistant text without a lost-context guard.

## Solution Assessment

Yes. The linked PR direction is the narrow maintainable fix because it changes the output-selection boundary for this exact error path and adds regression coverage without adding new product policy.
