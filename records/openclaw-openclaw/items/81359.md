## Summary

Close: the unsafe lost-context/raw-output selector path is fixed in current main and in the stable `v2026.6.6` release. The still-open linked PR is now superseded by released selector/test behavior rather than being the remaining fix path.

## What This Changes

The issue reports that lost-context subagent completion handling can announce raw transcript or tool-result content instead of a safe assistant-authored result.

## Best Possible Solution

Keep the shipped selector behavior that only surfaces assistant-authored or bounded status output; broader direct-text parent delivery bugs should remain tracked separately, such as https://github.com/openclaw/openclaw/issues/90840 and https://github.com/openclaw/openclaw/issues/90944.

## Reproduction Assessment

Yes. The old source-level path at `90ae151154bf5eefbeaeb0b05a13c8fe41bbb6ad` stored non-assistant/tool text as `latestRawText` and could return it; the fixed release source removes that path.

## Solution Assessment

Yes. The shipped fix is the right boundary: `readSubagentOutput()` now selects only assistant/silent text or a bounded tool-count status and returns undefined instead of raw transcript/tool output.
