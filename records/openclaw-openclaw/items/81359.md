## Label Justifications

- `P1`: The original report describes a production-observed agent workflow leak where raw child runtime/tool context could reach a parent-visible completion.
- `impact:security`: The issue concerns exposure of startup, runtime, user, system, or tool context across a completion boundary.
- `impact:message-loss`: The expected structured subagent result could be replaced by unrelated raw transcript/tool-result content in the parent flow.

## Summary

Close as implemented: current main and shipped releases no longer promote raw child tool/toolResult text into subagent completion results, which resolves the privacy/message-delivery failure reported here; the broader lost-context lifecycle-status mismatch remains separately tracked.

## What This Changes

The issue asks OpenClaw to prevent lost-context subagent completions from surfacing raw startup, runtime, user, or tool context as the child result.

## Best Possible Solution

Keep the shipped selector/docs/tests behavior as canonical, and handle any remaining lost-context lifecycle reconciliation under https://github.com/openclaw/openclaw/issues/90299.

## Reproduction Assessment

Yes for the original bug: the fix commit shows the pre-fix latestRawText/toolResult fallback and current tests lock in that tool output is no longer returned. Current main no longer reproduces the raw-output leak.

## Solution Assessment

Yes. The shipped selector boundary is the narrow maintainable solution for this issue because it only accepts visible assistant output and leaves broader lost-context lifecycle recovery to the separate canonical lifecycle thread.
