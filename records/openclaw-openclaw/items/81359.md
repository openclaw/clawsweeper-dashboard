The issue requests suppressing raw/tool-result/latestRawText output for subagent runs that fail with `subagent run lost active execution context`.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/81360 or an equivalent narrow fix that suppresses raw/tool-result/latest-assistant fallback for lost-context outcomes and adds regression coverage.

## Reproduction Assessment

Yes. Source inspection shows current main can emit the lost-context error, allow failed-output capture, and return raw `latestRawText` or latest assistant fallback without a lost-context guard.

## Solution Assessment

Yes. The linked PR direction is the narrow maintainable solution because it changes the output-selection boundary for this exact error path while preserving normal successful assistant output.
