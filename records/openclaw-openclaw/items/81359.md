## Label Justifications

- `P1`: The issue is a production-reproduced broken subagent completion workflow that can expose raw child context in a parent-visible result.
- `impact:security`: The report concerns privacy-sensitive startup, runtime, user, or tool context being surfaced outside the child transcript boundary.
- `impact:message-loss`: The completion announcement can deliver the wrong child result instead of the expected structured completion text.
- `impact:session-state`: The failure comes from lost subagent execution context and transcript/output selection drift across parent and child sessions.

## Summary

Keep open: current main and the latest shipped release still allow the lost-context error path to capture child transcript output, and the linked closing fix is open rather than merged. The recent production repro on the paired PR strengthens the source-level finding by showing a child `grep -n` tool result announced instead of the intended parent completion text.

## What This Changes

The issue asks OpenClaw to suppress raw child transcript, tool-result, and startup-context fallbacks when a subagent completion ends with `subagent run lost active execution context`.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/81360 or fold its lost-context-specific guard and regression coverage into the broader raw-output selector fix, preserving normal successful assistant final output while suppressing raw/tool/internal fallback on lost-context failures.

## Reproduction Assessment

Yes. Source inspection shows current main still emits the lost-context error, permits failed-output capture, and can select `latestRawText`; the latest issue discussion also includes a deterministic production reproduction on OpenClaw 2026.5.3-1.

## Solution Assessment

Yes. Suppressing raw/tool/latest-assistant fallback specifically for lost-context failures is the narrow maintainable fix; if the broader raw-output PR lands first, it should include this lost-context case and regression coverage before this issue closes.
