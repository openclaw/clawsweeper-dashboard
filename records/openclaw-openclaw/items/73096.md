## Summary

Keep open. Current main no longer treats CLI aliases such as `claude-cli` as embedded harness ids, but explicit plugin runtime pins still bypass the config-backed fallback resolver and force `fallback: "none"`, so the central issue remains and needs maintainer policy on billing-sensitive PI fallback behavior.

## What This Changes

The issue asks OpenClaw to reconcile pinned agent harness fallback behavior so explicit plugin runtimes either honor configured PI fallback or fail closed with clearer docs and warnings.

## Best Possible Solution

Define the explicit-runtime fallback contract, then align selector behavior, diagnostics, billing/cost warnings, docs/schema copy, and a composed attempt-execution regression test around that contract.

## Reproduction Assessment

Yes. Configure `agents.defaults.agentRuntime: { id: "codex", fallback: "pi" }` and run through `runAgentAttempt`; the configured runtime is converted to `agentHarnessId: "codex"`, then the pinned selector still forces `fallback: "none"`.

## Solution Assessment

Unclear. Passing configured fallback into the pinned resolver is the narrow code change, but the best maintainable solution depends on whether maintainers want explicit plugin pins to opt into PI fallback and how prominently billing behavior must be surfaced.
