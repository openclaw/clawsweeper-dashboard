## Summary

Close as implemented on current main. OpenClaw has moved to a strict runtime contract: the fallback knob is no longer part of agent runtime policy, CLI aliases no longer pass through pinned harness selection, and docs/tests now cover explicit runtimes failing closed.

## What This Changes

The issue asks OpenClaw to resolve the mismatch where pinned agent harness selection ignored configured PI fallback and created confusing fail-closed versus billing-sensitive fallback behavior.

## Best Possible Solution

Keep the current strict runtime contract: explicit plugin runtimes fail closed, CLI aliases route through CLI backends, and legacy fallback keys migrate away through doctor.

## Reproduction Assessment

No on current main. The reported reproduction depended on a supported `fallback` runtime field and on CLI aliases reaching pinned harness selection; current source removes that fallback field and skips CLI aliases before embedded harness pinning.

## Solution Assessment

Yes. Current main chose the safer maintainable path: explicit runtimes fail closed, auto mode owns PI fallback, unsupported fallback config is removed, and CLI aliases use the CLI runner instead of embedded harness lookup.
