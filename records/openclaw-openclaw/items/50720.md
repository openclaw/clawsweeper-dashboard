## Summary

Keep open, but not mergeable as submitted. The private-delivery feature is not implemented on current main and may be useful, but this stale branch drops several current sessions_send safeguards and still lacks required real behavior proof.

## What This Changes

The branch adds deliveryMode/announce parameters to sessions_send, suppresses final external announce delivery for private mode, updates A2A prompt guidance plus docs/tests, and includes a secrets runtime integration test tweak.

## Best Possible Solution

Choose a backward-compatible delivery/dispatch contract, then rebase the feature onto current sessions_send while preserving provenance, lane, thread, delayed-reply, docs, and test behavior with real after-fix proof.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection shows current main has no deliveryMode/announce parameter and still uses the normal A2A reply-back plus announce lifecycle, so this is a new public feature/API request.

## Solution Assessment

No, not as submitted. Private delivery may be the right direction, but this patch needs maintainer API approval and a rebase that preserves current sessions_send safeguards before it is a maintainable solution.
