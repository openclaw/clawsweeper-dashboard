## Summary

Keep open. Current main still lacks private/announce delivery controls, but this PR is not ready to land because it is stale against current sessions_send safeguards, still needs maintainer API approval, and lacks real behavior proof.

## What This Changes

The PR adds deliveryMode/announce parameters to sessions_send, suppresses final external announce delivery for private mode, updates A2A prompt guidance, docs, and tests, and includes an unrelated secrets runtime test change.

## Best Possible Solution

Maintainers should choose the backward-compatible sessions_send delivery/dispatch API, then rebase or replace this patch so it preserves current A2A safeguards, removes unrelated test churn, updates docs/tests, and includes real after-fix proof.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives high confidence that this is a feature/API gap: current main has no deliveryMode or announce parameter and still runs the normal reply-back plus announce lifecycle.

## Solution Assessment

No, not as submitted. A private-delivery control may be the right direction if maintainers approve the API, but this branch needs a rebase that preserves current provenance, lane, thread, delayed-reply, parser, docs, and test behavior.
