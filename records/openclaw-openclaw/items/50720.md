  - confidence: 0.9

  - body: The branch returns `timeout` immediately when `agent.wait` reports a timeout. Current main treats non-terminal waits as accepted and keeps the A2A flow alive, so delayed replies are not lost or confused with stale history.
  - confidence: 0.9

  - body: The submitted announce send params omit `threadId`. Current main passes `announceTarget.threadId`, and without it threaded Telegram, Slack, or Discord completions can fall back to the parent chat instead of the bound thread or topic.
  - confidence: 0.88

  - body: The branch lacks current main's guard that rejects resolved thread session keys before dispatch. That lets `sessions_send` target human-facing thread sessions even though current docs and code require the parent channel session key for inter-agent coordination.
  - confidence: 0.84

  - body: The branch replaces existing `resolveAnnounceTargetFromKey` tests with prompt-guidance tests. Keep the prompt coverage, but do not drop coverage for plugin-owned target normalization, Slack decimal thread IDs, Matrix colon IDs, and Feishu IDs used by announce delivery.
