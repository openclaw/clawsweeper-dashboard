## Summary

Keep open: current main still has a source-reproducible Codex dynamic-tool result path where a raw object without `content[]` can reach the Codex response converter and crash before the assistant response.

## What This Changes

The issue requests runtime hardening so Codex dynamic tool execution normalizes raw plugin payload objects into an `AgentToolResult` envelope while preserving structured results.

## Best Possible Solution

Normalize raw dynamic-tool returns at the Codex bridge before middleware/extensions and response conversion, reusing the existing `AgentToolResult` envelope convention with focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a dynamic tool returning a plain object without `content[]` can pass through to `convertToolContents(result.content)`, whose first operation is `content.reduce`.

## Solution Assessment

Yes. Bridge-level normalization before middleware/extensions and Codex response conversion is the narrowest maintainable fix; changing Codex upstream or adding plugin-specific handling would leave the OpenClaw envelope mismatch in place.
