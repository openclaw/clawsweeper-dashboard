## Summary

Keep open: current main still lets a raw Codex dynamic-tool return reach the response conversion path without an AgentToolResult envelope, and an open linked fix PR already owns the implementation lane.

## What This Changes

The issue asks Codex dynamic-tool execution to normalize plain plugin tool return values into AgentToolResult envelopes before middleware, extensions, and response conversion.

## Best Possible Solution

Keep this issue open until https://github.com/openclaw/openclaw/pull/97916 or an equivalent canonical fix lands with a regression for raw payload returns and unchanged structured AgentToolResult behavior.

## Reproduction Assessment

Yes. Source inspection on current main shows a raw tool.execute result can pass through middleware and then reach convertToolContents(result.content), which calls reduce on content without a raw-payload envelope guard.

## Solution Assessment

Yes. Normalizing at the Codex dynamic-tool bridge before middleware, extensions, and Codex response conversion is the narrowest maintainable fix; plugin-specific handling or an upstream Codex protocol change would leave OpenClaw's runtime boundary inconsistent.
