## Summary

Keep open: current main still has the source-reproducible raw-result path, and two open pull requests specifically target this issue, so the issue should remain open until one canonical fix lands.

## What This Changes

The issue requests Codex dynamic-tool runtime hardening so raw plugin tool returns are normalized into an `AgentToolResult` envelope before middleware, legacy extensions, and Codex response conversion.

## Best Possible Solution

Land one bridge-boundary normalizer that preserves structured `AgentToolResult` values, safely serializes raw payloads, adds focused regression coverage, and then close this issue after the selected PR merges.

## Reproduction Assessment

Yes. Source inspection shows a raw dynamic-tool return can reach `convertToolContents(result.content)`, whose first operation is `content.reduce`; a plain object without `content[]` matches the reported failure.

## Solution Assessment

Yes. The best fix is to normalize unknown plugin/runtime outputs at the Codex dynamic-tool bridge before middleware/extensions and before replying to upstream Codex, rather than relying on plugin-specific handling or middleware to repair the envelope later.
