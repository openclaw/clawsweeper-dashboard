## Summary

Keep open: current main still passes a raw Codex dynamic-tool return through middleware/extensions and then calls `convertToolContents(result.content)`, so a plain payload object without `content[]` remains a source-reproducible runtime failure.

## What This Changes

The issue asks OpenClaw to normalize raw Codex dynamic-tool plugin payloads into `AgentToolResult` envelopes before middleware and response conversion.

## Best Possible Solution

Add a small Codex dynamic-tool result normalizer at the bridge boundary, reuse existing `AgentToolResult` payload conventions, and cover raw payload plus already-structured results in focused tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a dynamic tool returning a plain object without `content[]` can reach `convertToolContents(result.content)`, which calls `reduce` on undefined.

## Solution Assessment

Yes. Normalizing at the Codex dynamic-tool bridge before middleware/extensions and Codex response conversion is the narrowest maintainable fix; plugin-specific handling or Codex protocol changes would leave the OpenClaw runtime boundary inconsistent.
