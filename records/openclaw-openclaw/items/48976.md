## Summary

Keep open. Current main still has the Feishu quoted/thread mention bug, and this PR targets the linked issue, but the branch has blocking correctness/security findings and lacks after-fix real behavior proof.

## What This Changes

This PR adds Feishu fetched-message mention replacement in `getMessageFeishu()` and also changes command authorization, gateway/browser/cron/health/device behavior, package dependencies, tests, and changelog entries.

## Best Possible Solution

Land a narrow Feishu-plugin-only fix that models fetched mentions, adapts `{key, id, id_type, name}` into the existing mention-normalization contract, covers quoted/root/thread content, and adds focused send/bot regression tests.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to mock Feishu `im.message.get` or `im.message.list` with `body.content` containing `@_user_1` plus item-level `mentions`; current `parseFeishuMessageItem()` ignores that metadata and returns parsed raw content.

## Solution Assessment

No. The Feishu direction is right, but this branch is not the best fix because it uses the live-event mention shape for fetched API data and carries unrelated security/dependency changes; a narrow fetched-shape adapter with tests is safer.
