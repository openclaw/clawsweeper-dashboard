## Summary

Keep open. Current main still has the Feishu quoted/thread mention-normalization bug, and this PR targets the right user problem, but the branch is broad, missing real Feishu proof, and introduces blocking correctness/security issues.

## What This Changes

The PR adds Feishu fetched-message mention replacement in `getMessageFeishu()` while also changing dependencies, command authorization, browser auth, cron cleanup, health status merging, device bootstrap behavior, tests, and changelog entries.

## Best Possible Solution

Land a narrow Feishu-plugin patch that models fetched-message `mentions`, adapts the SDK `{id, id_type}` shape, normalizes quoted/root/thread content, and adds focused `send.test.ts` coverage while dropping or splitting unrelated changes.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to mock Feishu `im.message.get` or `im.message.list` with `body.content` containing `@_user_1` plus item-level `mentions`; current main parses only raw content and ignores that metadata.

## Solution Assessment

No. The Feishu direction is right, but this PR reads the live-event mention shape for fetched API data and carries unrelated security-sensitive changes; a narrow adapter plus regression tests is safer.
