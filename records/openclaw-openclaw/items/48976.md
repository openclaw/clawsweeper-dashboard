- **[P1] Adapt fetched Feishu mentions before normalizing:** `extensions/feishu/src/send.ts:244-247`
  - body: The Feishu SDK returns fetched-message mentions as a flat `{ id, id_type }` shape, but this code reads `mention.id.open_id`. With the real response shape, mentioned users fall back to `@name` instead of `<at>` tags, so the quoted-message bug remains for the API path this PR is fixing.
  - confidence: 0.95
- **[P1] Restore owner-only gating for external /config reads:** `src/auto-reply/reply/commands-config.ts:41-45`
  - body: After this change, `/config show` reaches the config-read path after only sender allowlist authorization, so a non-owner who is allowed to use commands on an external channel can read parsed config. Current main has an owner check here and tests that authorized non-owners are blocked.
  - confidence: 0.96
- **[P1] Keep /debug mutations owner-only:** `src/auto-reply/reply/commands-config.ts:211-215`
  - body: `handleDebugCommand` now only checks command sender authorization before it can reset, unset, or set process-wide runtime overrides. That is a privilege regression from current main's owner-only `/debug` behavior for shared bots.
  - confidence: 0.95
- **[P1] Match cron session keys exactly before deleting:** `src/gateway/server-methods/cron.ts:205-209`
  - body: The added cleanup finds a session entry with `includes(sessionKey)` or `includes(jobId)`, so removing `job-1` can delete a different persisted session such as `job-10`. Session cleanup needs an exact key or deterministic mapping before mutating the store.
  - confidence: 0.92
- **[P1] Remove dependency edits or update the lockfile:** `package.json:353-356`
  - body: The PR adds and upgrades package dependencies in `package.json` without the corresponding lockfile update. Frozen installs will fail, and these dependency changes are unrelated to the Feishu mention fix.
  - confidence: 0.9
- **[P2] Read gateway account status from channelAccounts:** `src/commands/health.ts:511-518`
  - body: `channels.status` returns account snapshots under `payload.channelAccounts[channelId]`, not under `channels[channelId].accounts`, so this merge path never populates the account-level runtime fields it is trying to add.
  - confidence: 0.88
