- **[P1] Restore owner gating for external /config reads:** `src/auto-reply/reply/commands-config.ts:42`
  - body: This removes the owner check after `rejectUnauthorizedCommand`, so any sender merely allowed by command allowlists can run `/config show` on external channels and read the parsed config. Keep the internal read-only gateway exception, but retain owner-only gating for external channels.
  - confidence: 0.95
- **[P1] Reinstate owner checks before /debug mutations:** `src/auto-reply/reply/commands-config.ts:212`
  - body: `/debug` now reaches reset/set/unset override operations after only the generic sender allowlist check. That lets non-owner command users mutate process-wide in-memory overrides on shared bots, which is a privilege regression from current main.
  - confidence: 0.94
- **[P1] Match cron session keys exactly before deleting:** `src/gateway/server-methods/cron.ts:205-207`
  - body: Using `includes(sessionKey)` or `includes(jobId)` can delete the wrong persisted session when identifiers overlap, such as `job-1` matching a `job-10` key. Cleanup should use an exact key or a deterministic job-to-session mapping.
  - confidence: 0.9
- **[P2] Adapt fetched Feishu mentions before replacing placeholders:** `extensions/feishu/src/send.ts:244-247`
  - body: The pinned Feishu SDK exposes fetched message mentions as `{ key, id: string, id_type, name }`, but this code declares `id.open_id` and reads `mention.id.open_id`. With the real shape, quoted mentions fall back to plain `@name` instead of producing Feishu `<at>` tags.
  - confidence: 0.88
- **[P2] Commit the lockfile with dependency changes:** `package.json:353`
  - body: This PR changes root dependency versions in `package.json` but does not include `pnpm-lock.yaml`. CI uses `pnpm install --frozen-lockfile`, so the branch will fail install or leave the package changes unapplied until the lockfile is reconciled.
