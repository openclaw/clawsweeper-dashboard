## Summary

Keep open. The linked Feishu bug is still source-reproducible on current main, but this PR is not mergeable as-is because it bundles unrelated security-sensitive changes, removes existing owner gates for `/config` and `/debug`, has an ineffective dependency security update, and lacks after-fix real behavior proof.

## What This Changes

The PR adds Feishu quoted-message mention normalization and also changes gateway cron cleanup, command authorization, browser control auth, health status merging, device bootstrap tokens, node-host logging, package dependencies, and the changelog.

## Best Possible Solution

Narrow or replace this with a Feishu-plugin-only patch that models fetched-message `mentions`, reuses/adapts the existing mention normalizer for quoted/root/thread content, and adds focused `extensions/feishu/src/send.test.ts` coverage without unrelated security or dependency churn.

## Reproduction Assessment

Yes, source-level. A mocked `getMessageFeishu()` response whose `body.content` contains `@_user_1` plus mention metadata would still return the raw placeholder on current main because the fetched parser ignores mentions and quoted context uses that content directly.

## Solution Assessment

No. The Feishu fix direction is right, but this PR is not the best solution because it open-codes the normalization and bundles unrelated security-sensitive changes that introduce regressions.
