## Summary

Keep open. The Feishu bug remains valid and this PR is a plausible implementation candidate, but the branch is not mergeable as reviewed because it broadens into unrelated command, gateway, health, dependency, browser, cron, and device-bootstrap changes and introduces blocking security/correctness regressions.

## What This Changes

This PR attempts to normalize Feishu fetched quoted-message mentions, while also changing command authorization, config-write policy, browser control auth, cron cleanup, health status merging, device bootstrap behavior, root dependencies, tests, and changelog entries.

## Best Possible Solution

Land a narrow Feishu-plugin patch that models fetched-message mentions, adapts the fetched shape into the existing normalization behavior, normalizes quoted/root/thread content, and adds focused `extensions/feishu/src/send.test.ts` regression coverage; split or drop the unrelated branch changes.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to mock Feishu `im.message.get` or `im.message.list` with `body.content` containing `@_user_1` plus item-level mention metadata; current main parses the raw content and never normalizes those fetched mentions.

## Solution Assessment

No. The Feishu direction is right, but this branch is not the best way to solve it because it uses the wrong fetched mention shape and carries unrelated security-sensitive changes; a narrow fetched-message adapter with regression tests is safer.
