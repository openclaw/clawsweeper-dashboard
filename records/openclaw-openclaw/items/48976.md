## Summary

Keep open. Current main still has the Feishu fetched-message mention bug, but this PR is not mergeable as-is because it needs real Feishu behavior proof, uses the wrong fetched mention shape, and bundles unrelated security-sensitive command/dependency/runtime changes.

## What This Changes

The PR adds Feishu fetched-message mention normalization and also changes command/config authorization, browser control auth, cron cleanup, health status merging, device bootstrap behavior, dependency pins, tests, and changelog entries.

## Best Possible Solution

Land a narrow Feishu-plugin patch that models fetched `mentions`, adapts the `{id, id_type}` shape, normalizes quoted/root/thread content, and adds focused `extensions/feishu/src/send.test.ts` coverage; split the unrelated command, dependency, cron, health, browser, and infra changes into separate reviews.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to mock Feishu `im.message.get` or `im.message.list` with `body.content` containing `@_user_1` plus item-level fetched `mentions`; current main ignores that metadata and returns raw parsed content.

## Solution Assessment

No. The Feishu direction is right, but this PR is not the best way because it uses the wrong fetched mention shape and carries unrelated security-sensitive changes; a narrow parser adapter with regression tests is safer.
