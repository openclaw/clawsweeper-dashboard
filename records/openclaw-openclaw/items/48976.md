## Summary

Keep this PR open. The Feishu quoted/replied mention bug is still present on current main, but this branch is not mergeable as-is because it mixes the useful Feishu fix with unrelated command authorization, dependency, browser auth, cron, health, device, and changelog changes, including security-sensitive regressions.

## What This Changes

The PR adds mention handling to Feishu fetched-message parsing while also changing package dependencies, command/config authorization, browser control auth, cron cleanup, health reporting, origin parsing, device bootstrap behavior, node-host logging, tests, and the changelog.

## Best Possible Solution

Land a narrow Feishu-plugin-only change that models fetched-message mentions, adapts that shape into the existing mention-normalization behavior, applies it after fetched content parsing for quoted/root/thread messages, and adds focused getMessageFeishu tests. Leave the unrelated security, dependency, cron, health, browser, and bootstrap work to separate PRs.

## Reproduction Assessment

Yes. A high-confidence unit reproduction is to mock im.message.get returning text or post content with @_user_N plus a mentions array; current main returns the raw placeholder because getMessageFeishu ignores mentions before the quoted content is injected.

## Solution Assessment

No. The Feishu direction is correct, but this PR is not the best way to solve it because it copies mention-normalization logic and mixes unrelated security-sensitive changes; the maintainable path is a small plugin-owned patch with tests.
