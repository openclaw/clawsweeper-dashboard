## Summary

Keep open. The Feishu bug is valid and still source-reproducible on current main, but this branch is not mergeable as-is because it bundles unrelated command/security/dependency/gateway changes, lacks real Feishu behavior proof, and the Feishu mention adapter uses the wrong fetched-message shape.

## What This Changes

This PR adds fetched-message mention replacement for Feishu quoted content and also changes command authorization, gateway cron/health/browser/device behavior, dependencies, tests, and changelog entries.

## Best Possible Solution

Land a narrow Feishu-plugin-only fix that models fetched mentions, adapts {key, id, id_type, name} into the existing mention-normalization contract, covers quoted/root/thread content, and adds focused send/bot regression tests.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to mock Feishu message.get or message.list with body.content containing @_user_1 plus item-level mentions; current parseFeishuMessageItem() ignores that metadata and returns parsed raw content.

## Solution Assessment

No. The Feishu direction is right, but this branch is not the best fix because it uses the live-event mention shape for fetched API data and carries unrelated security/dependency changes; a narrow fetched-shape adapter with tests is safer.
