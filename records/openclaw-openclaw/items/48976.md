## Summary

Keep open: the linked Feishu bug is still source-reproducible on current main, but this PR is not safe to merge because it lacks real Feishu behavior proof, uses the wrong fetched-message mention shape, and bundles unrelated security-sensitive changes.

## What This Changes

The PR normalizes Feishu fetched quoted-message mention placeholders and also changes command authorization, config-write policy, browser auth, cron cleanup, health status merging, device bootstrap behavior, root dependencies, tests, and changelog entries.

## Best Possible Solution

Land a narrow Feishu-plugin patch that models fetched-message mentions, adapts `{id, id_type}` into the existing normalization helper, normalizes quoted/root/thread content, and adds focused send.test.ts coverage while dropping or splitting unrelated changes.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to mock Feishu `im.message.get` or `im.message.list` with `body.content` containing `@_user_1` plus fetched `mentions`; current main parses only raw content and ignores that metadata.

## Solution Assessment

No. The Feishu direction is right, but this PR is not the best solution because it uses the live-event mention shape for fetched API data and carries unrelated security-sensitive changes; a narrow adapter plus regression tests is safer.
