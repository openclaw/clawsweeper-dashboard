## Summary

Keep this PR open. Current main still lacks Feishu quoted/replied-message mention normalization, but the PR branch is too broad, mixes unrelated security and dependency changes, and has blocking review findings that make it unsafe to merge as-is.

## What This Changes

The PR adds mention handling to Feishu fetched-message parsing while also changing package dependencies, command/config authorization, browser control auth, cron cleanup, health reporting, origin parsing, device bootstrap behavior, node-host logging, tests, and the changelog.

## Best Possible Solution

Use a narrow Feishu-only change: model the im.message.get mentions array, reuse or share the existing mention normalization for fetched quoted/replied content, add focused getMessageFeishu regression tests for text and post placeholders, and leave dependency/security/cron/health/bootstrap work to separate PRs.

## Reproduction Assessment

Yes. A high-confidence unit reproduction is to mock im.message.get returning text or post content with @_user_N plus a mentions array; current main will return the raw placeholder because getMessageFeishu ignores mentions before the quoted content is injected.

## Solution Assessment

No. The requested Feishu direction is correct, but this PR is not the best implementation because it copies mention-normalization logic and mixes unrelated security-sensitive changes; a smaller plugin-only patch is the maintainable path.
