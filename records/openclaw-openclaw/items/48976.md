## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Adapt fetched Feishu mentions before normalizing:** `extensions/feishu/src/send.ts:244-247`
  - body: The SDK returns fetched-message mentions as a flat `{ key, id, id_type, name }` shape, but this code types `id` as an object and reads `mention.id.open_id`. At runtime `mentionId` is undefined for SDK-shaped responses, so quoted mentions degrade to `@name` instead of the requested `<at user_id="...">name</at>` tags.
  - confidence: 0.95
- **[P1] Restore owner-only gating for external /config reads:** `src/auto-reply/reply/commands-config.ts:42`
  - body: Deleting the `rejectNonOwnerCommand` check lets any command-allowlisted external sender run `/config show` after `rejectUnauthorizedCommand`. Shared bots often allow command senders who are not owners, so this exposes parsed config and sensitive subpaths that were previously owner-only.
  - confidence: 0.93
- **[P1] Keep /debug mutations owner-only:** `src/auto-reply/reply/commands-config.ts:212`
  - body: After this deletion, `/debug set`, `/debug unset`, and `/debug reset` only require general command authorization before mutating process-wide runtime overrides. That is a privilege regression for external channels where command allowlists are broader than owners.
  - confidence: 0.93
- **[P1] Match cron session keys exactly before deleting:** `src/gateway/server-methods/cron.ts:209`
  - body: Using `includes(sessionKey) || includes(jobId)` can delete an unrelated persisted session when identifiers overlap, such as `job-1` matching a `job-10` key. Cleanup should use an exact key or stored mapping, and it should resolve the correct agent store instead of assuming `main`.
  - confidence: 0.88
- **[P2] Read account runtime status from channelAccounts:** `src/commands/health.ts:518`
  - body: `channels.status` returns account snapshots under `payload.channelAccounts[channelId]`, not under `channels[channelId].accounts`. This merge path will leave `gwAccount` undefined for real multi-account runtime status, so it does not fix the account-level health fields it claims to merge.
  - confidence: 0.9
- **[P1] Commit the lockfile with dependency changes:** `package.json:356`
  - body: This PR edits `package.json` dependencies and overrides but does not include `pnpm-lock.yaml`. Frozen-lockfile installs in CI will reject the manifest/lockfile mismatch unless the dependency changes are removed or the lockfile is regenerated on the rebased branch.
  - confidence: 0.82
