## Summary

Keep open. Current main still has the Feishu quoted-message mention bug and this PR is the active linked implementation candidate, but the branch should not merge as-is because it lacks real Feishu proof, uses the wrong fetched mention shape, and bundles unrelated security-sensitive changes.

## What This Changes

The PR adds Feishu quoted-message mention normalization and also changes command/config authorization, browser control auth, cron cleanup, health status merging, device bootstrap behavior, dependency versions, tests, and changelog entries.

## Best Possible Solution

Land a narrow Feishu-plugin patch that models fetched mentions, adapts the string id/id_type shape into the existing normalization behavior, normalizes quoted/root/thread content, and adds focused send/bot regression coverage; split or drop the unrelated branch changes.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to mock Feishu im.message.get or im.message.list with body.content containing @_user_1 plus item-level mentions; current main parses the raw content and never normalizes those fetched mentions.

## Solution Assessment

No. The Feishu direction is right, but this branch is not the best solution because it uses the wrong fetched mention shape and carries unrelated security-sensitive changes; a narrow parser adapter with regression tests is safer.
