## Summary

Keep open. The Feishu quoted-mention bug still exists on current main, and this PR targets the right area, but the branch is not merge-ready: it bundles many unrelated changes, has security and CI regressions, and has no real after-fix behavior proof.

## What This Changes

This PR adds Feishu fetched-message mention handling in `getMessageFeishu()` while also changing dependencies, command authorization, browser auth, cron cleanup, health status merging, device bootstrap, origin parsing, sessions, and node-host logging.

## Best Possible Solution

Narrow this to a Feishu-plugin-only fix that models fetched-message mentions, adapts them into the existing normalization path, applies it to quoted/root/thread fetched content, and adds focused `extensions/feishu/src/send.test.ts` coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence mocked path: a fetched Feishu message with `body.content` containing `@_user_1` plus mention metadata still returns raw parsed content on current main because `parseFeishuMessageItem()` ignores mentions.

## Solution Assessment

No. The Feishu parser direction is right, but this PR is not the best submitted form because it bundles unrelated security-sensitive changes and introduces regressions; a focused Feishu-only patch or replacement is safer.
