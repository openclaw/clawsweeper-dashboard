## Summary

Keep open: current main still throws plain runtime errors for both cited resolver failures, and no stable OC-style public diagnostic-code or known-errors contract exists. The related open repair at https://github.com/openclaw/openclaw/pull/84417 addresses a Windows command-override validation bug for https://github.com/openclaw/openclaw/issues/84365, but it does not implement the stable error-code convention requested here.

## What This Changes

The issue requests stable public diagnostic error codes for Codex managed app-server and native hook relay runtime/spawn failures.

## Best Possible Solution

Define a small stable runtime diagnostic-code convention, add a typed helper or constants for the Codex managed-binary and native-hook relay failures, and document or doctor-map those codes while tracking resolver-specific Windows bugs separately.

## Reproduction Assessment

Not applicable as a bug repro: this is a request for a new stable diagnostic-code surface. Source inspection on current main confirms the two cited paths still lack stable public IDs.

## Solution Assessment

Unclear until maintainers choose the convention. Adding ad hoc strings to two messages would be a weak contract; the maintainable solution is a small shared namespace with docs or doctor mapping before public codes ship.
