## Summary

Keep open: this PR targets a valid source-supported WebChat transcript persistence bug, but the current patch can duplicate long Pi-persisted assistant turns, removes unrelated changelog entries, and still lacks after-fix real WebChat/session JSONL proof.

## What This Changes

The PR adds an embedded-runner assistant transcript repair helper, wires it from `agent-command.ts`, exports it from the attempt runtime barrel, and updates `CHANGELOG.md`.

## Best Possible Solution

Use a transcript-aware embedded fallback that checks complete active transcript records under the session write lock, appends only a missing current assistant turn, preserves metadata, restores the unrelated changelog entries, and includes focused duplicate-regression coverage plus redacted live WebChat/session JSONL proof.

## Reproduction Assessment

Yes for the PR regression by source inspection: an already Pi-persisted final assistant JSONL record longer than 4096 bytes makes the new tail parser see an incomplete fragment and append a duplicate assistant turn. The original WebChat loss is source- and incident-supported, but I did not live-run WebChat in this read-only pass.

## Solution Assessment

No. The embedded fallback direction is reasonable, but a fixed-size tail parse is not a reliable duplicate check; the safer approach is to inspect complete transcript records, ideally while holding the session write lock, before appending.
