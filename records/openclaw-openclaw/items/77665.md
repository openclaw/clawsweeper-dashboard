## Summary

Keep open: this PR still broadens the CLI transcript fallback to all embedded runner results, which conflicts with the current WebChat/Pi transcript ownership model and can duplicate normal persisted turns. It also lacks after-fix real WebChat or JSONL behavior proof. https://github.com/openclaw/openclaw/pull/77665

## What This Changes

The PR extends post-run transcript persistence in `agent-command.ts` to embedded runners, keeps CLI compaction limited to CLI, and adds a changelog entry for the related WebChat persistence issue.

## Best Possible Solution

Implement a guarded embedded persistence fallback that appends only when the current assistant turn is missing, keeps CLI compaction CLI-only, and adds regression coverage for both missing-turn repair and normal no-duplicate embedded runs.

## Reproduction Assessment

Yes for the PR regression: source inspection shows normal embedded runs already use Pi `SessionManager` and `activeSession.prompt(...)`, while the patch would append a second user/assistant pair afterward. No live WebChat reproduction of the original related issue was run in this read-only pass.

## Solution Assessment

No. Extending `persistCliTurnTranscript` to every embedded runner result is broader than the durable fix because it blindly mirrors turns that Pi normally owns; the maintainable path is a missing-turn-specific fallback with duplicate detection.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Guard embedded transcript fallback against duplicate Pi turns:** `src/agents/agent-command.ts:1200`
  - body: This broadens the CLI transcript fallback to every `runner === "embedded"` result. Normal embedded runs already persist through Pi's guarded `SessionManager`, and `persistCliTurnTranscript` appends a full user/assistant pair tagged as `api: "cli"`, so successful WebChat turns can be duplicated in JSONL. Please append only when the active transcript is missing the current assistant turn, and cover both the missing-turn and no-duplicate cases.
