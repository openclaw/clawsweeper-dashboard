## Summary

Keep open: the updated PR is narrower than the first version, but its duplicate guard can still append a second assistant turn for normal embedded replies whose final JSONL record is longer than the 4 KiB tail read, and the external real-behavior proof gate is still unmet.

## What This Changes

The PR wires an embedded-runner transcript repair helper into `agent-command.ts`, exports it from the attempt runtime, and adds a changelog entry for the WebChat session JSONL persistence issue.

## Best Possible Solution

Use a transcript-aware embedded fallback that checks complete active transcript records under the session write lock, appends only the missing current assistant turn, preserves metadata, and covers both missing-turn repair and long-message no-duplicate cases.

## Reproduction Assessment

Yes for the PR regression: a normal embedded run whose Pi-written final assistant JSONL record is larger than the 4 KiB tail window will make the new guard parse a partial line and append a duplicate. The original WebChat drop remains source- and incident-supported, but I did not live-run WebChat in this read-only pass.

## Solution Assessment

No. The new helper is the right general direction, but a fixed-size tail parse is not a reliable duplicate check; the safer solution is to inspect complete transcript records or an active-branch/index helper before appending.
