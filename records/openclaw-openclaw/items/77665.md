## Summary

Keep open: the PR targets the right webchat transcript regression, but the implementation reuses a CLI helper that appends both the user and assistant sides of a turn. Embedded runs already persist user messages through Pi's SessionManager, so this would replace missing assistant entries with duplicated user entries in session JSONL.

## What This Changes

The PR extends post-run transcript persistence to embedded runners, keeps CLI compaction CLI-only, and adds a changelog fix entry for the webchat JSONL regression.

## Best Possible Solution

Add an embedded-specific assistant-turn persistence fallback that only writes when Pi did not already persist the assistant turn, with duplicate/idempotency coverage for webchat reload history.

## Reproduction Assessment

No high-confidence live reproduction was run in this review. Source inspection does trace the current main failure path and the PR-introduced duplicate-user path clearly enough for review feedback.

## Solution Assessment

No; broadening the CLI helper is not the best fix because that helper has CLI semantics and appends user turns. The safer solution is an assistant-only embedded fallback with duplicate detection and regression coverage.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Avoid replaying embedded user turns:** `src/agents/agent-command.ts:1200`
