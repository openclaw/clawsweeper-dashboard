## Summary

Keep open: this PR targets a real transcript persistence bug, but the patch is unsafe as written because it calls the CLI transcript helper for every embedded run even though normal embedded Pi runs already own JSONL persistence through `SessionManager`. The related #77518 review explicitly called out that the repair needs duplicate protection, which this diff does not add.

## What This Changes

The PR extends post-run transcript persistence in `agent-command.ts` to embedded runners, keeps CLI compaction gated to CLI, and adds a changelog entry for #77518.

## Best Possible Solution

Implement a guarded embedded persistence fallback that inspects the active session transcript and only repairs a missing assistant turn, with regression coverage proving reload history keeps the assistant response without duplicating normal Pi turns.

## Reproduction Assessment

Yes for the PR regression: source inspection shows a normal embedded run already uses a guarded Pi `SessionManager` before this new post-run block would append another user/assistant pair. No live WebChat reproduction of #77518 was run in this read-only pass.

## Solution Assessment

No. Extending `persistCliTurnTranscript` to all embedded runners is not the narrow maintainable fix because it blindly mirrors turns that Pi normally already persisted; the repair needs duplicate detection or a missing-turn-specific fallback.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Guard embedded transcript repair against duplicate Pi turns:** `src/agents/agent-command.ts:1200`
  - body: This broadens the CLI transcript fallback to every `runner === "embedded"` result. Normal embedded runs already open a guarded Pi `SessionManager`, pass it into `createAgentSession`, and submit `activeSession.prompt(...)`; WebChat also documents that agent-run replies are not mirrored because Pi owns those JSONL turns. Calling `persistCliTurnTranscript` here appends a second user/assistant pair, tagged as `api: "cli"`, after successful embedded turns. Please only append when the active transcript is missing the current assistant turn, and cover both the missing-turn case and the normal no-duplicate case.
  - confidence: 0.9
