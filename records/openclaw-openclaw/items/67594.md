## Summary

Keep open. Current main still only parses top-level dated memory files for temporal decay, while nested dated files are treated as evergreen, and an open linked implementation PR should be reviewed or superseded rather than closing this issue early.

## What This Changes

Apply temporal decay to dated Markdown files nested under `memory/` instead of only top-level `memory/YYYY-MM-DD.md` files.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/68155, or fold the same focused matcher and regression coverage into the broader recursive-memory work while preserving non-dated memory files as evergreen.

## Reproduction Assessment

Yes, source-level. Current main's dated-memory regex matches top-level `memory/YYYY-MM-DD.md` but rejects nested dated examples, and the timestamp extraction path then treats memory-source non-matches as evergreen.

## Solution Assessment

Yes. A narrow memory-core matcher update plus regression tests is the smallest maintainable fix; configurable evergreen path policy would be broader config/product work and is not required for this bug.
