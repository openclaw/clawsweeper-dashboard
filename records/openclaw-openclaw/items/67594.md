## Summary

Keep open. Current main still treats nested dated memory files as evergreen, and #68155 is open, unmerged, and explicitly linked as the implementation candidate.

## What This Changes

Apply temporal decay to dated markdown files under `memory/` subdirectories instead of only top-level `memory/YYYY-MM-DD.md` files.

## Best Possible Solution

Review and land #68155 or an equivalent focused memory-core change that expands dated-path parsing for nested memory files, adds regression coverage including non-ASCII subdirectories, and preserves `MEMORY.md` plus non-dated topic files as evergreen.

## Reproduction Assessment

Yes. The exact current regex matches a flat dated memory file but rejects nested dated examples, and the source path then classifies those memory files as evergreen before decay.

## Solution Assessment

Yes. A narrow matcher update plus regression tests is the smallest maintainable fix; configurable evergreen path patterns would be separate product/config work and are not required for this bug.
