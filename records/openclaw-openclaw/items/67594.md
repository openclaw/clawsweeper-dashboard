## Summary

Keep open. Current main still only recognizes top-level `memory/YYYY-MM-DD.md` files for temporal decay, so nested dated memory files are treated as evergreen; open, unmerged #68155 is already the explicit implementation candidate for this issue.

## What This Changes

The issue requests temporal decay for dated markdown files stored under nested `memory/` directories instead of only top-level dated memory files.

## Best Possible Solution

Review and land #68155, or an equivalent focused memory-core matcher and regression-test change, then close this issue after merge.

## Reproduction Assessment

Yes. A direct current-main regex probe shows top-level dated memory paths match while nested, timestamped, and Unicode-subdirectory dated paths do not, and source inspection shows those memory-source misses return `null` as evergreen.

## Solution Assessment

Yes. A narrow matcher update with regression coverage is the smallest maintainable fix; configurable evergreen path patterns are broader product work and are not required for this bug.
