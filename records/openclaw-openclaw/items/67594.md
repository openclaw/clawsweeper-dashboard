## Summary

Keep open: current main still treats nested dated memory files as evergreen for temporal decay, and an open linked implementation PR already uses GitHub closing syntax for this issue.

## What This Changes

Temporal decay should apply to dated markdown files stored under nested `memory/` subdirectories instead of only top-level dated memory files.

## Best Possible Solution

Review and land the focused matcher and regression coverage in https://github.com/openclaw/openclaw/pull/68155, or fold equivalent changes into https://github.com/openclaw/openclaw/pull/72748 while preserving non-dated memory files as evergreen.

## Reproduction Assessment

Yes, source-level. Current main’s dated-memory regex accepts top-level `memory/YYYY-MM-DD.md` but rejects nested dated examples, and the extraction path then treats those memory-source misses as evergreen.

## Solution Assessment

Yes. A narrow matcher update plus regression coverage is the smallest maintainable fix; configurable evergreen path policy would be broader config/product work and is not required for this bug.
