## Summary

Keep open. Current main still treats nested dated memory files as evergreen, and the open linked implementation PR should be reviewed or replaced rather than closing this issue now.

## What This Changes

Temporal decay should recognize dated markdown files under nested `memory/` subdirectories instead of only top-level `memory/YYYY-MM-DD.md` files.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/68155 or an equivalent focused memory-core change that expands dated-path parsing and adds nested-path regression coverage while preserving non-dated memory files as evergreen.

## Reproduction Assessment

Yes. Current main reproduces with the existing regex: top-level dated memory files match, while nested dated memory files return no parsed date and are treated as evergreen.

## Solution Assessment

Yes. A narrow memory-core matcher update plus regression tests is the smallest maintainable fix; configurable evergreen path policy would be broader product/config work and is not required for this bug.
