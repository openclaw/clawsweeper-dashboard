## Summary

Keep open. Current main still rejects nested dated memory paths, and open PR #68155 is already the linked implementation candidate using closing syntax for this issue.

## What This Changes

Temporal decay should recognize dated Markdown files nested under `memory/` subdirectories instead of only top-level `memory/YYYY-MM-DD.md` files.

## Best Possible Solution

Review and land #68155 or an equivalent narrow memory-core change that expands dated-path parsing and adds nested-path regression coverage while keeping `MEMORY.md` and non-dated topic files evergreen.

## Reproduction Assessment

Yes. Current main's matcher accepts a top-level dated memory file but rejects nested dated paths; source inspection shows those rejected memory paths return `null` as evergreen before decay.

## Solution Assessment

Yes. A focused matcher update plus regression tests is the smallest maintainable fix; configurable evergreen path patterns are broader product work and are not required to fix the reported bug.
