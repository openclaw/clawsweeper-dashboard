## Summary

Keep open. Current main and the latest release still classify nested dated memory files as non-dated memory, and https://github.com/openclaw/openclaw/pull/68155 is already the focused open implementation PR tied to this issue.

## What This Changes

The issue asks for temporal decay to apply to dated Markdown files stored in nested `memory/` subdirectories, not only top-level `memory/YYYY-MM-DD.md` files.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/68155, or fold equivalent nested dated-path and timestamp-suffix regression coverage into https://github.com/openclaw/openclaw/pull/72748 while preserving non-dated memory files as evergreen.

## Reproduction Assessment

Yes, source-level. Current main's regex rejects nested dated memory paths, and the extraction path then treats those memory-source non-matches as evergreen before ranking.

## Solution Assessment

Yes. A narrow matcher update with regression coverage is the maintainable fix for the reported bug; configurable evergreen path policy would be broader product/config work and is not required here.
