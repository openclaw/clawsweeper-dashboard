## Summary

Keep open. Current main still only parses top-level dated memory files, and open unmerged #68155 is already the linked implementation candidate for this issue: https://github.com/openclaw/openclaw/issues/67594

## What This Changes

Apply temporal decay to dated Markdown files under `memory/` subdirectories instead of treating them as evergreen memory.

## Best Possible Solution

Land #68155 or an equivalent focused memory-core change that expands dated-path parsing and adds nested-path regression coverage while preserving `MEMORY.md` and non-dated topic files as evergreen.

## Reproduction Assessment

Yes. On current main, the dated-memory regex matches a top-level daily file but rejects nested dated paths, including the Chinese month subdirectory example from the latest comment.

## Solution Assessment

Yes. A narrow matcher update plus regression tests is the smallest maintainable fix; configurable evergreen path patterns are broader product work and are not required for this bug.
