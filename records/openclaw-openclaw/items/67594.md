## Summary

Keep open. Current main still has the flat-only dated-memory matcher, so nested dated memory files are treated as evergreen; the open linked implementation PR should be reviewed or replaced rather than closing this issue now.

## What This Changes

Temporal decay should treat dated Markdown files beneath `memory/` subdirectories as dated memory instead of evergreen memory.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/68155 or an equivalent focused memory-core change that expands dated-path parsing and adds nested-path regression coverage while preserving non-dated memory files as evergreen.

## Reproduction Assessment

Yes. A current-main regex probe and source read show top-level dated memory files match while nested dated memory files do not, and the missed memory-source paths return `null` before decay.

## Solution Assessment

Yes. A narrow memory-core matcher update plus regression tests is the smallest maintainable fix; configurable `evergreenPaths` would be broader product/config work and is not required for this bug.
