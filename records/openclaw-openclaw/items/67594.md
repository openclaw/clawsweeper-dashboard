## Summary

Keep open. Current main still only parses top-level dated memory files, and open unmerged #68155 is already the linked implementation candidate for this issue.

## What This Changes

Temporal decay should treat dated markdown files under `memory/` subdirectories as dated memory instead of evergreen memory.

## Best Possible Solution

Land #68155 or an equivalent focused memory-core change that expands dated-path parsing and adds nested-path regression coverage while preserving `MEMORY.md` and non-dated topic files as evergreen.

## Reproduction Assessment

Yes. On current main, the regex matches a top-level dated file but rejects nested dated memory paths; the rejected memory-source path then returns `null` as evergreen before decay.

## Solution Assessment

Yes. A narrow memory-core matcher update plus regression tests is the smallest maintainable fix; configurable evergreen path patterns are broader product work and are not required for this bug.
