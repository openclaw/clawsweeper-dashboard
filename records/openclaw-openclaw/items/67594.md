## Summary

Keep open. Current main still only recognizes top-level dated memory files for temporal decay, and open PR #68155 is already the linked implementation candidate for this bug.

## What This Changes

Apply temporal decay to dated Markdown memory files stored under `memory/` subdirectories, not only top-level `memory/YYYY-MM-DD.md` files.

## Best Possible Solution

Review and land #68155 or an equivalent focused memory-core change that expands dated-path parsing and adds nested-path regression coverage while preserving non-dated memory files as evergreen.

## Reproduction Assessment

Yes. Source inspection and a direct regex check on current main show the matcher accepts top-level dated files but rejects nested dated memory paths, which then flow to the evergreen branch before decay is applied.

## Solution Assessment

Yes. A narrow memory-core matcher update plus regression tests is the smallest maintainable fix; configurable evergreen path patterns are broader product work and are not required for this bug.
