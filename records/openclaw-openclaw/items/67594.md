## Summary

Keep open. Current main still treats nested dated memory files as evergreen for temporal decay, and https://github.com/openclaw/openclaw/pull/68155 is already an open implementation PR tied to this issue through GitHub closing syntax.

## What This Changes

The issue requests temporal decay for dated Markdown files stored under nested `memory/` subdirectories.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/68155, or fold equivalent matcher and regression coverage into https://github.com/openclaw/openclaw/pull/72748 while preserving non-dated memory files as evergreen.

## Reproduction Assessment

Yes, source-level. Current main's dated-memory regex matches top-level `memory/YYYY-MM-DD.md` but rejects nested dated examples, and the extraction path then treats those memory-source non-matches as evergreen.

## Solution Assessment

Yes. A narrow matcher update plus regression coverage is the smallest maintainable fix; configurable evergreen path policy would be broader config/product work and is not required for this bug.
