## Summary

Keep open. Current main still has the flat dated-memory matcher, and the linked open PR should be reviewed rather than closing this issue early.

## What This Changes

Temporal decay should apply to dated Markdown files under nested `memory/` subdirectories instead of treating them as evergreen.

## Best Possible Solution

Temporal-decay date parsing should be expanded in memory-core, with regression coverage for nested dated paths and unchanged evergreen handling for `MEMORY.md` and non-dated memory files.

## Reproduction Assessment

Yes. A direct check of current main's exact `DATED_MEMORY_PATH_RE` accepts a top-level dated memory file but rejects nested dated paths, including the localized month-folder example from the latest comment; source inspection then shows those rejected memory-source paths are treated as evergreen.

## Solution Assessment

Yes. A narrow matcher/parser update plus regression tests is the smallest maintainable fix; configurable `evergreenPaths` would be broader product work and is not required to fix the existing behavior.
