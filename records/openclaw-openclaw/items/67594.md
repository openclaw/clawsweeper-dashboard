## Summary

Keep open. Current main still has the flat dated-memory matcher and treats nested dated memory files as evergreen, while #68155 is open and unmerged as the implementation candidate that explicitly closes this issue.

## What This Changes

Request temporal decay to recognize dated Markdown files under `memory/` subdirectories while preserving evergreen behavior for non-dated memory files.

## Best Possible Solution

Keep this issue open and review/land #68155, or an equivalent focused memory-core fix, so the matcher decays dated files under `memory/` subdirectories while leaving `MEMORY.md` and non-dated topic files evergreen.

## Reproduction Assessment

Yes. A direct check of the current matcher reproduces the report, and the source path shows those unmatched nested memory files return `null` timestamps as evergreen memory entries.

## Solution Assessment

Yes, a narrow matcher update plus regression tests in memory-core is the smallest maintainable fix for the reported bug. The configurable evergreen-pattern option is broader product work and is not required to restore the expected default behavior.
