## Summary

Keep open. Current main still treats nested dated memory files as evergreen, and the open unmerged #68155 PR is already the linked implementation candidate for this issue.

## What This Changes

Temporal decay should recognize dated Markdown files under nested `memory/` subdirectories instead of only top-level `memory/YYYY-MM-DD.md` files.

## Best Possible Solution

Land #68155 or an equivalent focused memory-core fix that decays dated files below `memory/` while keeping `MEMORY.md` and non-dated topic files evergreen, with regression coverage for nested dated paths.

## Reproduction Assessment

Yes. The current regex check reproduces the report directly, and the source path shows those unmatched nested memory files return `null` timestamps as evergreen entries.

## Solution Assessment

Yes. A narrow matcher update plus regression tests in memory-core is the smallest maintainable fix; configurable evergreen path patterns are broader product work and are not required for this bug.
