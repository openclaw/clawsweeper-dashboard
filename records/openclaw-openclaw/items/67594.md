## Summary

Keep open. Current main still treats dated memory files in subdirectories as evergreen, and open PR #68155 is already the linked implementation candidate for this bug.

## What This Changes

Apply temporal decay to dated markdown files under `memory/` subdirectories, not only top-level `memory/YYYY-MM-DD.md` files.

## Best Possible Solution

Review and land #68155, or an equivalent focused memory-core matcher and regression-test fix, while preserving `MEMORY.md` and non-dated memory topic files as evergreen.

## Reproduction Assessment

Yes. Source inspection and a direct check of the current regex show top-level dated files match while nested dated memory paths, including the reported Chinese month subdirectory, do not.

## Solution Assessment

Yes. A narrow matcher update plus regression coverage is the smallest maintainable fix; configurable evergreen path patterns are broader product work and are not required for this bug.
