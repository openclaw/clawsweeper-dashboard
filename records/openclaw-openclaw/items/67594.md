## Summary

Keep open. Current main still has the flat dated-memory matcher, nested dated memory files are treated as evergreen, and open PR #68155 is already the linked implementation candidate.

## What This Changes

The issue asks for temporal decay to recognize dated memory markdown files nested under `memory/` subdirectories instead of only top-level `memory/YYYY-MM-DD.md` files.

## Best Possible Solution

Review and land #68155 or an equivalent focused memory-core change that expands dated-path parsing and adds nested-path regression coverage while preserving non-dated evergreen files.

## Reproduction Assessment

Yes. Source inspection plus a direct regex check show top-level dated memory files match while nested dated paths do not, and those memory-source paths then return an evergreen `null` timestamp before decay.

## Solution Assessment

Yes. A narrow matcher update with regression tests is the smallest maintainable fix; configurable evergreen path patterns are broader product work and are not required for this bug.
