## Summary

Keep open. Current main still has the reported nested-memory temporal-decay gap, and open PR #68155 is already the explicit implementation candidate with closing syntax.

## What This Changes

Temporal decay should recognize dated Markdown files under nested `memory/` subdirectories instead of only top-level `memory/YYYY-MM-DD.md` files.

## Best Possible Solution

Review and land #68155 or an equivalent focused memory-core fix with regression coverage, then close this issue as implemented after merge.

## Reproduction Assessment

Yes. A direct regex probe against current main reproduces the report, and the source path shows unmatched nested memory files return `null` timestamps as evergreen entries.

## Solution Assessment

Yes. A narrow matcher update plus regression tests is the smallest maintainable fix; configurable evergreen path patterns are broader product work and are not required for this bug.
