## Summary

Keep open. Current main still matches the reported Control UI OpenAI Realtime Talk bug: the browser posts the WebRTC SDP offer to OpenAI from a page whose CSP does not allow that origin, and the WebRTC data-channel handler still drops OpenAI error and lifecycle events instead of surfacing a useful Talk failure state.

## What This Changes

The issue requests a Control UI Realtime Talk fix for OpenAI WebRTC CSP blocking, missing browser-session audio input/VAD configuration, and hidden realtime audio errors.

## Best Possible Solution

Land a focused bundled OpenAI browser Talk fix in current main: allow the required OpenAI Realtime origin in Control UI CSP, add explicit browser-session audio input/VAD settings, surface OpenAI data-channel errors and lifecycle status, and cover the behavior with targeted tests plus a changelog entry.

## Reproduction Assessment

Yes. Static reproduction is high confidence: current main combines a direct browser POST to OpenAI's WebRTC SDP endpoint with a CSP that omits that origin, and the event switch visibly ignores the diagnostic events the provider emits.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the existing Control UI CSP, OpenAI realtime browser-session provider, and WebRTC transport rather than in a broad configurable CSP feature or the closed Google Live issue.
