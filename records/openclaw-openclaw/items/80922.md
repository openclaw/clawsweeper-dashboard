## Summary

Keep open: this is a protected, active maintainer PR with a useful planner direction, but the current head still has blocking correctness issues in the exec approval trust path.

## What This Changes

The PR adds a command authorization planner and routes POSIX allowlist evaluation, allow-always persistence, approval summaries, control-command rejection, and enforced shell rendering through it.

## Best Possible Solution

Keep the planner direction, preserve literal argv tokens, ensure approval requests carry commandAnalysis when emitted or have a real update event, and add regressions for both boundaries before merge.

## Reproduction Assessment

Yes for the review findings from source inspection. PR-head planArgvCommand trims tokenized argv entries, and PR-head approval handling broadcasts the request before the async commandAnalysis mutation can reach serialized WebSocket clients.

## Solution Assessment

No. The direction is maintainable, but the patch must preserve tokenized argv bytes and avoid sending approval request frames with null risk analysis before this authorization contract is safe to merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve argv tokens before planning:** `src/infra/command-authorization/plan.ts:134`
  - body: This maps every argv element through trim() before analysis. Tokenized argv arguments preserve leading and trailing spaces as literal bytes, so ['printf', ' x '] would be authorized and reported as ['printf', 'x'] while the runner receives a different argument. Filter whitespace-only entries without rewriting nonempty tokens.
  - confidence: 0.91
- **[P2] Include command analysis before broadcasting approvals:** `src/gateway/server-methods/exec-approval.ts:315`
  - body: The request is created with commandAnalysis: null, then handlePendingApprovalRequest broadcasts requestEvent before the promise callback mutates record.request. Real WebSocket clients get the serialized frame from that broadcast, so approval UIs can miss the planner risk summary; await the analysis before emitting or send a real follow-up update.
  - confidence: 0.88
