## What This Changes

The PR changes the macOS VoiceWakeOverlay height measurement path to defer `isOverflowing` updates with `DispatchQueue.main.async`.

## Best Possible Solution

Land a narrow macOS overlay fix that moves the relevant observable/layout mutation out of the SwiftUI update turn, de-duplicates pending overflow writes, and includes real macOS app proof.

## Reproduction Assessment

Yes for source reproduction, but not live reproduction in this review. Current main and `v2026.6.6` still have the attributed `onChange` -> `updateWindowFrame` -> `targetFrame` -> `measuredHeight` -> `isOverflowing` write chain described in the linked issue logs.

## Solution Assessment

No, not yet merge-ready as written. Deferring the overflow write is a plausible mitigation, but the queued write still needs de-duplication and the user-visible macOS overlay path needs live proof before this is the best fix.
