## Summary

Keep open: this is a focused candidate for the linked Weixin setup bug, and current main plus the latest release still only auto-allowlist ClickClack. The patch has no concrete code-correctness finding, but it deliberately broadens a restrictive plugin allowlist trust behavior and needs maintainer acceptance before merge.

## What This Changes

The PR generalizes explicit plugin enablement so any allowlist-blocked explicit selection is appended to `plugins.allow`, with updated helper tests for a generic channel plugin and denylist precedence.

## Best Possible Solution

Merge the helper-level fix only if maintainers confirm explicit selection is the intended allowlist trust gesture across setup, onboarding, and CLI enablement; otherwise narrow the mutation to channel setup or add a confirmation prompt.

## Solution Assessment
