## Summary

## What This Changes

The PR generalizes explicit plugin enablement so any explicitly selected plugin can be appended to a restrictive plugins.allow list before being enabled.

## Best Possible Solution

Merge or revise this only after maintainers confirm where the explicit-selection trust gesture belongs; if helper-level policy is too broad, scope the mutation to channel setup or add an explicit prompt before editing plugins.allow.

## Reproduction Assessment

Yes. Source inspection shows current main rejects non-ClickClack explicit selections under a restrictive allowlist, and channel setup routes selected plugins through that helper.

## Solution Assessment

Likely yes if maintainers accept explicit selection as the allowlist trust gesture, because the helper centralizes setup, onboarding, and CLI enablement. If that policy is too broad, the safer solution is to scope auto-append to channel setup or prompt before mutation.
