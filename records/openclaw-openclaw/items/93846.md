## Best Possible Solution

Carry `topLevel` as placement metadata through `prepareSendPayload`/`channelData` and the existing MS Teams outbound/send-context path for text, media, and cards, then require redacted live Teams proof before merge.

## Reproduction Assessment

Yes, source-level: current main and `v2026.6.9` expose no MS Teams `topLevel` send parameter and derive proactive placement only from static `replyStyle` config. The PR defect is also source-reproducible because the new `topLevel=true` branch forwards only text/topLevel while existing outbound code owns media and presentation delivery.

## Solution Assessment

No as implemented. The per-call override is the right narrow direction, but the patch should carry it through the canonical outbound path named by prior maintainer review instead of adding a partial direct send path.
