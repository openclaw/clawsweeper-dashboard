- **[P1] Preserve the relay-auth browser epoch:** `src/agents/sandbox/constants.ts:42`
  - body: Current main uses the relay-auth epoch and image contract to recreate browser containers that lack the CDP auth contract.   - confidence: 0.93
- **[P1] Propagate the init abort signal into backend startup:** `src/agents/sandbox/backend.types.ts:31`
  - body: `resolveSandboxContext` passes an abort signal into the backend factory, but the Docker backend still calls `ensureSandboxContainer` without forwarding it. In the hung-daemon case, the outer timeout can reject while `docker inspect/create/start/setup` keeps running in the background, so the cooperative-abort guarantee does not hold.
  - confidence: 0.9
- **[P2] Pass abort signals through CDP source-range probes:** `src/agents/sandbox/browser.ts:299-302`
  - body: The restored source-range derivation performs `docker network inspect` through helpers that do not accept or pass the init abort signal. If Docker hangs during browser setup, these child processes are not killed by the new timeout and can remain as the zombie work this PR is trying to prevent.
  - confidence: 0.88
- **[P2] Align the timeout config contract with the implementation:** `src/config/types.agents-shared.ts:50`
  - body: The new public type comment promises a 60s Docker default and 5min SSH/remote default, but the implementation falls back to one 60s timeout regardless of backend. Slow SSH/remote backends can therefore fail earlier than the exposed contract says, so the default semantics and help/docs need to be made consistent.
  - confidence: 0.83
