## Summary

Keep open: current main still rejects root-level `paperclip` on Gateway `agent` requests, and #74974 is the active open implementation candidate rather than a reason to close early.

## What This Changes

Allow Paperclip's `openclaw_gateway` adapter metadata on Gateway agent invocation payloads so agent runs are not rejected before dispatch.

## Best Possible Solution

Review and land #74974 or a superseding additive Gateway protocol compatibility fix that keeps strict rejection for unrelated agent fields and updates schema, tests, generated models, and changelog together.

## Reproduction Assessment

Yes. A minimal `agent` request with `message`, `idempotencyKey`, and a root-level `paperclip` object follows the report and fails current `validateAgentParams` before dispatch.

## Solution Assessment

Unclear. Adding `paperclip: Type.Optional(Type.Unknown())` is the narrow immediate unblocker, while a generic adapter metadata namespace may be more maintainable if maintainers want a public integration surface.
