## Summary

Keep open: current main still exposes only the standard Xiaomi API-key onboarding path and default API-balance URL. The reporter's follow-up and Xiaomi's static docs support the requested Token Plan endpoints, but maintainers should choose the provider/auth-profile shape before implementation because Token Plan credentials and provider naming are distinct from the existing preset.

## What This Changes

Add Xiaomi MiMo Token Plan regional OpenAI-compatible and Anthropic-compatible onboarding choices for China, Singapore, and Europe endpoints.

## Best Possible Solution

Define the Xiaomi Token Plan product shape, then add plugin-local regional presets with matching docs, manifest metadata, onboarding tests, and any needed Token Plan model catalog coverage.

## Reproduction Assessment

Not applicable; this is a feature request rather than a broken existing contract. Source inspection confirms current main lacks the requested Xiaomi Token Plan auth-choice IDs and endpoint presets.

## Solution Assessment

Unclear until maintainers decide the provider/auth shape. The regional preset approach has Z.AI precedent and upstream endpoint support, but Xiaomi's OpenClaw docs currently steer Token Plan users to a separate provider name rather than the existing `xiaomi` provider.
