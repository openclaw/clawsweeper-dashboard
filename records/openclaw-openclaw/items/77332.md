## Repair Work Prompt

Implement Xiaomi MiMo Token Plan regional onboarding choices for this issue. Keep the change provider-local under `extensions/xiaomi`; do not add core special-casing or change the existing `xiaomi-api-key` behavior. Add six auth choices: `xiaomi-token-plan-cn`, `xiaomi-token-plan-sgp`, `xiaomi-token-plan-ams`, `xiaomi-token-plan-anthropic-cn`, `xiaomi-token-plan-anthropic-sgp`, and `xiaomi-token-plan-anthropic-ams`. The OpenAI-compatible choices should configure `api: "openai-completions"` with `https://token-plan-cn.xiaomimimo.com/v1`, `https://token-plan-sgp.xiaomimimo.com/v1`, and `https://token-plan-ams.xiaomimimo.com/v1`; the Anthropic-compatible choices should configure `api: "anthropic-messages"` with the matching `/anthropic` base URLs. Reuse the existing Xiaomi model catalog unless implementation evidence shows a provider-local model metadata adjustment is necessary; stop for maintainer review before adding broad new model catalogs, key-prefix validation, or config policy. Update manifest auth-choice metadata, runtime onboarding, docs at `docs/providers/xiaomi.md`, focused tests, and a user-facing `CHANGELOG.md` entry.
