## Summary

Keep this PR open: the warning path is useful, but the submitted patch still does not warn for the Bedrock hardcoded-default case described in the PR because Bedrock fallback values arrive at runtime as model metadata.

## What This Changes

The PR adds a guard-result flag and runtime warning when context-window resolution falls back to the default source, plus focused guard tests.

## Best Possible Solution

Land one coherent provenance-aware path, likely by folding this warning into the Bedrock provenance branch or rebasing the branches together so known models keep real metadata and unknown literal fallbacks emit the warning.

## Reproduction Assessment

Yes. Source review shows an unknown Bedrock model/profile still gets a positive fallback contextWindow, and current runtime classification treats positive model values as source "model", so this PR's source-default warning does not fire for the stated Bedrock case.

## Solution Assessment

No. The generic default-source warning is useful, but the Bedrock fix needs fallback provenance or non-materialized defaults before the warning can reliably describe the user-visible problem.
