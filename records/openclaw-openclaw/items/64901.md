## Best Possible Solution

Yes, source-level. Current main classifies a positive Bedrock fallback `contextWindow` as `source: "model"` and has no default-source warning, while this PR only warns after the source is already `default`.

## Solution Assessment

No, not as submitted. The warning hook is narrow and maintainable, but it needs fallback provenance handling or coordination with https://github.com/openclaw/openclaw/pull/65030 before it solves the user-visible Bedrock case named in the PR.
