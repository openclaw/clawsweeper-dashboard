## Summary

Keep this PR open, but it is not merge-ready. The patch adds a generic default-source warning, while the Bedrock fallback named in the PR still reaches runtime as model metadata, so the new warning would not fire for that primary scenario; open #65030 is the provenance-aware branch maintainers should reconcile with this one.

## What This Changes

The PR adds a `shouldWarnDefault` guard result flag, logs a runtime warning when the resolved context-window source is `default`, and extends context-window guard tests for that flag.

## Best Possible Solution

Land one coherent provenance-aware path: keep known Bedrock context-window metadata as model-owned, mark only literal unknown fallbacks as default-derived, and emit the runtime warning from that provenance while preserving user-configured overrides.

## Reproduction Assessment

Yes. The source-level reproduction is to discover an unknown Bedrock inference profile, observe `contextWindow: 32000`, then pass that value as `modelContextWindow` through runtime resolution and see `source: "model"` rather than `source: "default"`.

## Solution Assessment

No. The proposed `source === "default"` flag is useful for generic missing metadata, but it is not the complete fix for the stated Bedrock fallback because Bedrock discovery needs to carry fallback provenance or avoid materializing literal defaults as model metadata.
