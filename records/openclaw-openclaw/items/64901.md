## Summary

Keep open: this PR adds a useful diagnostic path, but as submitted it still misses the Bedrock hardcoded-default case named in the PR body, and the external contributor has not provided real Bedrock/runtime proof.

## What This Changes

The PR adds a `shouldWarnDefault` guard result flag, logs a runtime warning when the guard source is `default`, and extends the context-window guard tests.

## Best Possible Solution

Land one provenance-aware path, likely by folding this warning into https://github.com/openclaw/openclaw/pull/65030 or rebasing the branches together so known Bedrock models use table values and unknown literal fallbacks emit a clear warning.

## Reproduction Assessment

Yes, by source inspection rather than a live run: unknown Bedrock fallbacks still become positive `contextWindow` model metadata, and current runtime classification treats that as `source: "model"`, so this PR's source-default warning does not fire for the stated case.

## Solution Assessment

No. A source-default warning is useful, but the Bedrock fix needs fallback provenance or non-materialized defaults before the warning can reliably describe the user-visible problem.
