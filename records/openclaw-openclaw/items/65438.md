## Summary

Keep open. Current main and v2026.6.10 still hardcode bootstrap file order, classify only `HEARTBEAT.md` as dynamic below the cache boundary, and expose no `bootstrapDynamicFiles` or `bootstrapFileOrder` config surface; the remaining blocker is maintainer product/API direction.

## What This Changes

The issue requests configurable bootstrap file ordering and dynamic-file classification so frequently changing workspace files can be placed after the prompt cache boundary.

## Best Possible Solution

Maintainers should choose one documented cache-boundary/config contract, preserve current defaults unless an intentional migration is approved, then wire it through prompt assembly, config schema/types/help, docs, and focused prompt-cache tests.
