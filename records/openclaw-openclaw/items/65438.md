## Summary

Keep open. Current main and v2026.6.6 still hardcode bootstrap file order and classify only `HEARTBEAT.md` as dynamic, while the related open tiered-bootstrap PR tracks a different file-selection surface rather than cache-boundary dynamic classification.

## What This Changes

Request a configurable bootstrap-file dynamic classification and injection-order surface so frequently changing workspace files can be placed after the prompt-cache boundary.

## Best Possible Solution

Choose one documented agent-default or per-agent cache-boundary classification and ordering contract, preserve current defaults unless maintainers intentionally migrate them, then wire it through prompt assembly, schema/types/help/docs, and focused cache-boundary tests.

## Reproduction Assessment

Yes at source level: current main deterministically hardcodes the file order and treats only `heartbeat.md` as dynamic, and the requested config keys are absent. I did not run a live Anthropic billing/cache experiment.
