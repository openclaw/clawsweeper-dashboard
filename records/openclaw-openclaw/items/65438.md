## Summary

Keep open. Current main and v2026.6.10 still hardcode bootstrap context ordering and classify only `HEARTBEAT.md` as dynamic below the prompt-cache boundary; the requested `bootstrapDynamicFiles` / `bootstrapFileOrder` surface remains missing and needs maintainer product/API direction before implementation.

## What This Changes

The issue requests configurable controls for marking workspace bootstrap files as dynamic and optionally ordering them to preserve prompt-cache stability.

## Best Possible Solution

Maintainers should choose one documented prompt-cache/context contract, then implement dynamic-file classification and deterministic ordering through that surface with schema, docs, and focused prompt-shape coverage.

## Reproduction Assessment
