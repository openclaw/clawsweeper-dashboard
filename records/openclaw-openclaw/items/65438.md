## Summary

Keep open. Current main still hardcodes bootstrap-file order and dynamic cache-boundary classification, the requested config keys are absent from source/schema/docs, and related discussion points to this as the canonical tracker for the public config/API decision.

## What This Changes

Request configurable bootstrap-file ordering and dynamic cache-boundary classification so volatile workspace files can be placed after the prompt-cache boundary.

## Best Possible Solution

Design one backward-compatible agent config surface that preserves current defaults, keeps `HEARTBEAT.md` dynamic by default, supports additional dynamic basenames and deterministic ordering, and updates schema, docs, and focused regression tests together.

Unclear pending maintainer API review. The requested direction is plausible, but the maintainable solution should settle config naming, precedence, defaults, hook behavior, and docs before implementation.
