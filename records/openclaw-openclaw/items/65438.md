## Summary

Keep open. Current main and the latest release still hardcode bootstrap file order and classify only `HEARTBEAT.md` as dynamic, with no `bootstrapDynamicFiles` or `bootstrapFileOrder` config surface; related cleanup already treats this as the canonical tracker.

## What This Changes

Add a user-facing way to mark workspace bootstrap files as dynamic and optionally control their prompt injection order so cache-sensitive prompt prefixes stay stable.

## Best Possible Solution

Keep this issue open as the canonical product/API tracker and decide one documented agent-default/per-agent surface for cache-boundary classification and ordering, with default behavior preserved unless maintainers explicitly choose a migration.
