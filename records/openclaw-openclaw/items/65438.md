## Summary

Keep open. Current main and v2026.6.6 still hardcode bootstrap ordering and classify only `HEARTBEAT.md` as dynamic, while the requested `bootstrapDynamicFiles` / `bootstrapFileOrder` config surface remains absent.

## What This Changes

Request configurable bootstrap file order and dynamic-file classification so frequently changing workspace context can be placed after the prompt-cache boundary.

## Best Possible Solution

Keep this issue as the canonical product/API tracker and design one documented agent-default/per-agent surface for cache-boundary classification and ordering, preserving current defaults unless maintainers explicitly choose an upgrade path.
