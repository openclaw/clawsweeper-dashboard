## Summary

Keep open. Current main still hardcodes bootstrap prompt ordering and dynamic cache-boundary classification, lacks the requested config keys in schema/docs, and related issue history continues to point here as the canonical tracker for this public config/API decision.

## What This Changes

Request configurable bootstrap dynamic-file classification and optional injection ordering so volatile workspace files can live after the prompt cache boundary.

## Best Possible Solution

Design one backward-compatible agent-defaults config surface that preserves current defaults, keeps `HEARTBEAT.md` dynamic by default, supports additional dynamic basenames and deterministic ordering, and updates schema, docs, and focused tests together.

## Reproduction Assessment

Yes for the source-level behavior: an extra bootstrap file or `MEMORY.md` is treated as stable because only `heartbeat.md` is dynamic, and there is no supported config override. No live Anthropic billing reproduction was performed.

## Solution Assessment

Unclear pending maintainer API review. The requested direction is plausible, but the maintainable fix should settle config naming, matching rules, precedence, and hook interaction before implementation.
