## Summary

Keep open. Current main and the latest release still hardcode bootstrap-file order and dynamic cache-boundary classification, the requested config keys are absent from source/schema/docs, and related discussion still treats this as the canonical config/API decision for volatile bootstrap files.

## What This Changes

Requests agent configuration to mark bootstrap files as dynamic and control prompt injection order so volatile workspace files can land after the prompt-cache boundary.

## Best Possible Solution

Design one backward-compatible agent config surface that preserves current defaults, keeps HEARTBEAT.md dynamic by default, supports additional dynamic basenames and deterministic ordering, and updates schema, docs, UI/help metadata, and focused regression tests together.

## Reproduction Assessment

Yes for the source-level product gap, not as a strict runtime bug. Current main has no supported path to mark MEMORY.md or hook-added files as dynamic, and no config path to reorder bootstrap context before prompt assembly.

## Solution Assessment

Unclear until maintainers settle the API. The requested direction is plausible, but the maintainable solution needs one documented config contract for naming, precedence, defaults, hook interaction, and cache-boundary behavior.
