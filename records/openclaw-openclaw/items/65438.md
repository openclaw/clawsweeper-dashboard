## Summary

Keep open. Current main and the latest release still hardcode bootstrap file order and classify only `HEARTBEAT.md` as dynamic, while the requested `bootstrapDynamicFiles` / `bootstrapFileOrder` config surface is still absent; related cleanup has already closed https://github.com/openclaw/openclaw/issues/59050 as a duplicate of this issue, and https://github.com/openclaw/openclaw/issues/67419 treats this as the narrower dynamic-file tracker.

## What This Changes

Requested configurable ordering and dynamic classification for workspace bootstrap files so volatile files can move after the prompt-cache boundary while stable prefix ordering remains predictable.

## Best Possible Solution

Keep this open for maintainers to choose one documented agent-default/per-agent API for cache-boundary classification and ordering, preserving current defaults unless they explicitly choose an upgrade path.

## Reproduction Assessment

Yes. Source inspection on current main and `v2026.6.6` shows the dynamic classifier is still hardcoded to `heartbeat.md`, prompt assembly splits stable and dynamic files around the cache boundary, and the proposed config keys are absent.

## Solution Assessment

Unclear. A config-based solution is plausible, but the best shape needs maintainer judgment because this is a durable public config/default surface that changes prompt-cache semantics.
