## Summary

Keep open. Current main still hardcodes bootstrap file ordering and dynamic cache-boundary classification, the requested config keys are absent from schema/docs, and related discussion keeps this as the canonical tracker for the public config/API decision.

## What This Changes

The issue requests agent config for marking bootstrap files dynamic and controlling deterministic injection order so volatile files can be placed below the prompt-cache boundary.

## Best Possible Solution

Keep this as the canonical tracker and design one backward-compatible config surface that preserves current defaults, keeps `HEARTBEAT.md` dynamic by default, supports additional dynamic basenames and optional deterministic ordering, and updates schema, docs, and focused tests together.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a file such as `SESSION_HANDOFF.md` or `MEMORY.md` is not in the hardcoded dynamic set, so it is rendered above `OPENCLAW_CACHE_BOUNDARY` with no supported config override.

## Solution Assessment

Unclear pending maintainer API review. The requested direction is plausible, but the narrowest maintainable solution should settle config naming, precedence, hook interaction, and relationship to broader context-bloat work before implementation.
