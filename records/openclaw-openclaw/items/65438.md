## Summary

Keep open. Current main and the latest release still hardcode bootstrap prompt ordering, classify only `HEARTBEAT.md` as dynamic, and expose no `bootstrapDynamicFiles` or `bootstrapFileOrder` config surface; the related merged cache-boundary PR explicitly left configurable dynamic ordering out of scope.

## What This Changes

The issue requests configurable bootstrap file ordering and dynamic-file classification so volatile workspace files can be placed after `OPENCLAW_CACHE_BOUNDARY` for prompt-cache stability.

## Best Possible Solution

Maintainers should choose one documented agent-default or per-agent cache-boundary contract, preserve current defaults unless an intentional migration is approved, then wire it through prompt assembly, config schema/types/help, docs, and focused cache-boundary tests.

## Reproduction Assessment

Yes for the current behavior by source inspection: current main and `v2026.6.6` still hardcode file order, treat only `heartbeat.md` as dynamic, and lack the requested config keys. I did not live-reproduce Anthropic billing/cache numbers.

## Solution Assessment

No, not as an automatic fix yet. A config surface is plausible, but the best shape needs maintainer judgment because OpenClaw has a high bar for new config and this affects prompt-cache semantics.
