## Summary

Keep open. Current main and the latest release still classify only `HEARTBEAT.md` as dynamic and expose no bootstrap dynamic-file or file-order config; the merged cache-boundary work explicitly scoped that config out, so this remains real product/API work rather than implemented cleanup.

## What This Changes

The issue requests configurable bootstrap dynamic-file classification and optional bootstrap file ordering so volatile workspace files do not invalidate the prompt-cache stable prefix.

## Best Possible Solution

Choose one documented agent-default or per-agent cache-boundary classification and ordering contract, preserve current defaults unless maintainers intentionally migrate them, then wire it through prompt assembly, config schema/types/help, docs, and focused cache-boundary tests.

## Reproduction Assessment

Yes at source level: current main and the latest release still hardcode the file order, treat only `heartbeat.md` as dynamic, and have no `bootstrapDynamicFiles` or `bootstrapFileOrder` config keys. I did not run a live Anthropic billing/cache experiment.

## Solution Assessment

No, not as an automatic fix yet. A config surface is plausible, but the best shape needs maintainer judgment because OpenClaw has a high bar for new config and this affects prompt-cache semantics.
