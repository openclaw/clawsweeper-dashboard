## Summary

Keep open. Current main and the latest release still hardcode the bootstrap file order, classify only `HEARTBEAT.md` as dynamic, and expose no `bootstrapDynamicFiles` or `bootstrapFileOrder` config surface; the related merged work explicitly left arbitrary dynamic ordering/config out of scope.

## What This Changes

The issue requests configurable bootstrap dynamic-file classification and optional file-order control so frequently changing workspace files do not invalidate the prompt-cache-stable prefix.

## Best Possible Solution

Choose one documented agent-default/per-agent bootstrap cache-boundary contract, preserve current defaults unless maintainers intentionally migrate them, and wire it through prompt assembly, config schema/types/help, docs, and focused cache-boundary tests.

## Reproduction Assessment

Yes at source level: current main still hardcodes the file order, treats only `heartbeat.md` as dynamic, and has no `bootstrapDynamicFiles` or `bootstrapFileOrder` config keys. I did not run a live Anthropic billing/cache experiment.

## Solution Assessment

Unclear as an implementation choice. A configurable classification surface is plausible, but the best shape needs maintainer judgment because OpenClaw has a high bar for new config and this affects prompt-cache semantics.
