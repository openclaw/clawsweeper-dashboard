## Summary

Keep open. Current main still has hardcoded bootstrap prompt ordering and hardcoded dynamic-file classification, with only `HEARTBEAT.md` placed below the cache boundary and no `bootstrapDynamicFiles` or `bootstrapFileOrder` config/docs/schema surface; related #59050 is a duplicate already closed in favor of this tracker, while #67419 is broader context-bloat policy work rather than a replacement for this API request.

## What This Changes

The issue requests config-driven bootstrap file ordering and dynamic-file placement below the system-prompt cache boundary to preserve prompt-cache prefix stability for frequently changing workspace files.

## Best Possible Solution

Keep this issue as the canonical tracker and design one backward-compatible config surface that preserves current defaults, keeps `HEARTBEAT.md` dynamic by default, lets operators add dynamic basenames, optionally controls ordering, and documents deterministic fallback behavior.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: pass an additional context file such as `SESSION_HANDOFF.md` into prompt assembly and it is treated as stable, sorted with the fixed fallback order, and rendered above `OPENCLAW_CACHE_BOUNDARY` with no supported config override.

## Solution Assessment

Unclear pending maintainer API review. The requested direction is plausible, but the narrowest maintainable solution should settle config precedence and how it interacts with bootstrap hooks or extra loaded files before code is merged.
