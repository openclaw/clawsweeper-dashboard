## Summary

Keep open. Current main and the latest release still keep bootstrap ordering and dynamic classification hardcoded, and adding user-configurable cache-boundary behavior is real product/config work rather than cleanup that can be closed.

## What This Changes

Request a user-facing config surface to mark workspace bootstrap files as dynamic and optionally control prompt injection order for prompt-cache stability.

## Best Possible Solution

Choose one documented agent-default or per-agent cache-boundary classification and ordering contract, preserve current defaults unless maintainers intentionally migrate them, then wire it through prompt assembly, config schema/types/help, docs, and focused cache-boundary tests.

## Reproduction Assessment

Yes at source level: current main and the latest release still hardcode file order, treat only `heartbeat.md` as dynamic, and expose no `bootstrapDynamicFiles` or `bootstrapFileOrder` config. I did not live-test Anthropic billing or cache-hit telemetry.

## Solution Assessment

No, not as an automatic fix yet. A config surface is plausible, but the best permanent shape needs maintainer judgment because OpenClaw has a high bar for new config and this affects prompt-cache semantics.
