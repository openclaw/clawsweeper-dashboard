## Summary

Keep open. Current main and the latest release still hardcode bootstrap file ordering, classify only `HEARTBEAT.md` as dynamic below the prompt-cache boundary, and expose no `bootstrapDynamicFiles` or `bootstrapFileOrder` config surface; the remaining blocker is maintainer product/API direction, not missing source evidence.

## What This Changes

Request configurable workspace bootstrap file ordering and dynamic cache-boundary classification so volatile files can move below `OPENCLAW_CACHE_BOUNDARY`.

## Best Possible Solution

Maintainers should choose one documented cache-boundary/config contract, preserve current defaults unless an intentional migration is approved, then wire it through prompt assembly, config schema/types/help, docs, and focused prompt-cache tests.

## Reproduction Assessment

Yes, at source level: current main still hardcodes the file order, treats only `heartbeat.md` as dynamic, and lacks the requested config keys. I did not live-reproduce the Anthropic billing numbers.

## Solution Assessment

Unclear as proposed. A config surface is plausible, but the best maintainable shape needs maintainer judgment because OpenClaw has a high bar for new config and this affects prompt-cache semantics.
