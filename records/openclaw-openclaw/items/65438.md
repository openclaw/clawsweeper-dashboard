## Summary

Keep open. Current main and the latest release still lack a user-facing `bootstrapDynamicFiles` or `bootstrapFileOrder` config surface; the ordering-only report is already closed as a duplicate of this issue, while the broader context-bloat report does not supersede this cache-boundary API request.

## What This Changes

The issue requests config-driven bootstrap file ordering and dynamic-file placement below the system-prompt cache boundary to keep volatile workspace files from invalidating the stable prompt prefix.

## Best Possible Solution

Keep this issue as the canonical tracker and design one backward-compatible config surface that preserves current defaults, keeps `HEARTBEAT.md` dynamic by default, lets operators add dynamic basenames, optionally controls ordering, and documents deterministic fallback behavior.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: any additional context file such as `SESSION_HANDOFF.md` that reaches `contextFiles` is not in the fixed dynamic set, so current prompt assembly renders it above `OPENCLAW_CACHE_BOUNDARY` with no supported config override.

## Solution Assessment

Unclear pending maintainer API review. The requested direction is plausible, but the narrowest maintainable fix should settle global versus per-agent precedence and the relationship to the inaccessible #72520 implementation candidate before code is merged.
