## Summary

Keep open. Current main still hardcodes bootstrap prompt ordering and dynamic cache-boundary classification, the requested config keys are absent, and related discussion points to this as the canonical tracker for a public config/API decision.

## What This Changes

Add operator config to choose bootstrap file order and mark additional bootstrap files as dynamic below the prompt-cache boundary.

## Best Possible Solution

Keep this as the canonical tracker and design one backward-compatible config surface that preserves current defaults, keeps `HEARTBEAT.md` dynamic by default, allows additional dynamic basenames, optionally controls deterministic ordering, and updates schema, docs, and focused tests together.
