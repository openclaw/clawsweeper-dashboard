## Summary

Keep open. Current main and the latest release still hardcode bootstrap file ordering and only classify `HEARTBEAT.md` as dynamic, while the existing config schema/docs do not expose `bootstrapDynamicFiles` or `bootstrapFileOrder`; the remaining work is a product-sensitive config API decision, not an implemented fix or duplicate cleanup target.

## What This Changes

The issue requests configurable bootstrap file dynamic classification and injection order so frequently changing workspace files can be placed after the prompt cache boundary.

## Best Possible Solution

Keep this issue open as the canonical tracker, choose a documented bootstrap order/dynamic-file config contract, and then implement schema, type, docs, prompt-ordering, and regression coverage without changing existing defaults for users who do not opt in.

## Reproduction Assessment

Yes for the source-level gap: current main still hardcodes context file order and only treats `HEARTBEAT.md` as dynamic. No live Anthropic billing/cache reproduction was run, so the reported spend numbers remain discussion evidence rather than independently reproduced behavior.

## Solution Assessment

Unclear until maintainers choose the product contract. The proposed knobs are plausible, but repository policy sets a high bar for new config surfaces and a cleaner final design should cover ordering and cache-boundary placement together.
