## Summary

Keep open. Current main still hardcodes bootstrap prompt ordering and dynamic cache-boundary classification, and the requested `bootstrapDynamicFiles` / `bootstrapFileOrder` config surface is not present in source, schema, UI, or docs.

## What This Changes

The issue requests config-driven bootstrap file ordering and dynamic-file placement below the prompt cache boundary to preserve cache-prefix stability for frequently changing workspace files.

## Best Possible Solution

Keep this as the canonical tracker and design one backward-compatible config surface that preserves current defaults, keeps `HEARTBEAT.md` dynamic by default, allows additional dynamic basenames, optionally controls ordering, and documents deterministic fallback behavior.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: pass an additional context file such as `SESSION_HANDOFF.md` into prompt assembly and it is treated as stable, sorted by fixed fallback order, and rendered above `OPENCLAW_CACHE_BOUNDARY` with no supported config override.

## Solution Assessment

Unclear pending maintainer API review. The requested direction is plausible, but the narrowest maintainable solution should settle config precedence and how it interacts with bootstrap hooks or extra loaded files before implementation.
