## Summary

Keep open: this is a focused and useful direction, but it is not merge-ready because the patch only memoizes list normalization while the large-list membership checks still run linearly on every authorization pass, and the external PR does not include real Gateway behavior proof for the reported latency path.

## What This Changes

The PR adds a WeakMap cache around `resolveOwnerAllowFromList` keyed by the raw allowlist array plus provider/account/plugin inputs, with colocated memoization tests.

## Best Possible Solution

Land a command-auth fix that memoizes the normalized owner allowlist together with Set-backed membership for sender checks, preserves the existing `ownerList` output, removes misleading test-only no-op exports, and includes redacted large-list Gateway proof.

## Reproduction Assessment

Yes, by source inspection: current main and the PR branch both route command contexts through `resolveCommandAuthorization`, and the PR branch still does linear owner membership checks after memoization. I did not run a live 9000-entry Gateway benchmark in this read-only review.

## Solution Assessment

No. Memoizing the filtered array is a useful partial mitigation, but the best fix should also make the membership checks O(1) while preserving the current owner authorization contract.
