## Summary

Keep open: the PR is a useful command-auth performance direction, but it is not merge-ready because it caches only normalized arrays, leaves per-message membership checks linear, and lacks after-fix real Gateway proof for the large allowlist scenario.

## What This Changes

This PR adds a WeakMap-backed cache around `resolveOwnerAllowFromList` and a colocated memoization regression test for large `commands.ownerAllowFrom` arrays.

## Best Possible Solution

Land or replace this with a narrow command-auth fix that caches normalized owner allowlist state plus Set-backed membership, removes the no-op test helper, and includes redacted large-list proof.

## Reproduction Assessment

Yes by source inspection: current main and v2026.6.8 still walk raw owner allowlists and use linear membership checks in the command authorization path. I did not run a live 9,000-entry Gateway benchmark in this read-only review.

## Solution Assessment

No. Memoizing normalized arrays is useful, but it is too narrow for the linked performance report because membership checks remain linear and the PR provides no real after-fix large-list proof.
