## What This Changes

The PR adds WeakMap-backed memoization and a colocated Vitest file for resolving `commands.ownerAllowFrom` lists in command authorization.

## Best Possible Solution

Cache both the normalized owner list and O(1) membership data per stable config/provider/account inputs, remove the no-op test hook, and add redacted real Gateway proof for the large-list path.

## Reproduction Assessment

Yes. Source inspection shows current main and `v2026.6.6` still re-walk raw `commands.ownerAllowFrom` arrays and perform linear owner membership checks; I did not run a live 9,000-entry Gateway benchmark in this read-only review.

## Solution Assessment

No. Memoizing normalized arrays is a useful partial fix, but the best fix also caches membership as a `Set` or equivalent O(1) structure and proves the reported large-list Gateway path in a real run.
