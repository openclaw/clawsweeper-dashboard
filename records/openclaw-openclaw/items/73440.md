## Summary

Keep this PR open, but it is not merge-ready: it is a meaningful fix attempt for the canonical large `ownerAllowFrom` slowdown, yet it still leaves per-message owner membership scans linear and lacks after-fix real Gateway or benchmark proof.

## What This Changes

The PR adds a WeakMap cache for normalized `commands.ownerAllowFrom` arrays and a colocated memoization regression test.

## Best Possible Solution

Land or replace this PR with a narrow command-auth fix that caches normalized owner allowlist state plus Set-backed membership checks, preserves the returned ownerList contract, removes the no-op test helper, and includes redacted large-list proof.

## Reproduction Assessment

Yes by source inspection: current main and the latest release still resolve owner allowlists by walking the raw array and then use linear includes membership checks during command authorization. I did not run a live 9,000-entry Gateway benchmark in this read-only review.

## Solution Assessment

No. Memoizing the filtered array is useful, but it is too narrow for the linked report because sender and command-owner membership checks remain O(n) on every message.
