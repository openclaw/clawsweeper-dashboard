## Summary

Keep open: this PR is a plausible fix path for the open ownerAllowFrom performance bug, but the current branch still leaves O(n) owner lookup work after the first cache fill and lacks required real behavior proof from an external contributor.

## What This Changes

The branch adds a WeakMap memoization layer for resolveOwnerAllowFromList plus a colocated memoization test for commands.ownerAllowFrom.

## Best Possible Solution

Cache compiled owner authorization state or membership sets under the relevant raw allowlist, provider, account, and plugin key while preserving provider-prefix filtering, wildcard semantics, and config-reload invalidation.

## Reproduction Assessment

Yes, source-level. Configure a large commands.ownerAllowFrom list and send an inbound message; current main builds command context, resolves owner allowlists, and performs linear owner membership checks on that path.

## Solution Assessment

No. The WeakMap list cache is directionally useful, but it is not the best complete fix because downstream owner-state construction and membership checks remain linear after the first cache fill.
