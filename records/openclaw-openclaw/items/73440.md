## Summary

Keep open: this PR targets a valid current-main performance bug, but the proposed cache is still partial and the external-contributor real behavior proof gate is not satisfied.

## What This Changes

This PR adds a WeakMap cache around `resolveOwnerAllowFromList`, exposes test-only helpers, and adds a colocated memoization test for `commands.ownerAllowFrom`.

## Best Possible Solution

Cache compiled owner authorization state or membership sets keyed by the relevant raw arrays, provider, account, and plugin while preserving provider-prefix filtering, wildcard behavior, and config-reload invalidation.

## Reproduction Assessment

Yes, source-level. Current main builds command context for inbound messages, resolves owner allowlists twice, and performs linear array membership checks in `resolveCommandAuthorization`.

## Solution Assessment

No. The WeakMap direction is reasonable, but caching only the formatted list leaves per-message owner-state construction and `includes` scans linear, so the PR is not the narrowest complete fix for the reported performance bug.
