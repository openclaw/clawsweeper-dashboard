## Summary

Keep open. Current main still lacks the ownerAllowFrom performance fix, and this PR is the active implementation for #50289, but its current head only memoizes normalized arrays while per-message owner-state derivation and linear membership checks remain; it also leaves a no-op test helper and no changelog entry.

## What This Changes

This PR adds a WeakMap-backed cache and tests for `resolveOwnerAllowFromList`, keyed by the raw allowlist array and channel/account/plugin lookup tuple.

## Best Possible Solution

Keep the narrow command-auth fix, but cache compiled owner authorization state for the same raw-array/provider/account/plugin key, including wildcard flags and membership sets, while preserving channel-prefix filtering, config reload semantics, and fail-closed owner authorization.

## Reproduction Assessment

Yes. Configure a large `commands.ownerAllowFrom` list and send an inbound message; current main and the PR head both route through `resolveCommandAuthorization`, and the PR head still performs per-message owner-state derivation plus linear membership checks.

## Solution Assessment

No. The WeakMap boundary is reasonable, but the current patch should cache compiled lookup state instead of only `string[]`, and it should fix or remove the no-op test helper and add the required changelog entry.
