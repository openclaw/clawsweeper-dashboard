## Summary

Keep open. This is an active, useful fix candidate for the linked owner allowlist performance bug and current main has not implemented the memoization, but the PR still needs a required changelog entry and cleanup of a no-op internal test helper before merge.

## What This Changes

The PR adds WeakMap memoization around `resolveOwnerAllowFromList` in command authorization and adds a colocated regression test for repeated large `commands.ownerAllowFrom` normalization.

## Best Possible Solution

Land a narrow command-auth optimization that caches normalized owner allowlists per raw array/provider/account/plugin, preserves fail-closed owner-only authorization semantics, includes focused regression coverage, and records the user-facing fix in the changelog.

## Reproduction Assessment

Yes. Configure a large `commands.ownerAllowFrom` list and send an inbound message; current main's `buildCommandContext` -> `resolveCommandAuthorization` path resolves owner allowlists twice and uses linear membership checks.

## Solution Assessment

Yes directionally. A private per-array cache is a narrow maintainable fix for the hot path, but this PR should add the required changelog entry and remove or make real the no-op reset helper before merge.
