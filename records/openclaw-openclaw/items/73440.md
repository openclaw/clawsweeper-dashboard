## Summary

Keep open: this is a focused and useful performance PR, but it is not merge-ready because it only memoizes normalized arrays while the hot path still does linear owner membership scans, and the external PR still lacks real Gateway proof for the large-list case.

## What This Changes

The PR adds a WeakMap-backed cache and colocated tests for resolving `commands.ownerAllowFrom` lists in command authorization.

## Best Possible Solution

Land a narrow command-auth fix that caches both normalized owner lists and O(1) membership data per stable config/provider/account inputs, preserves `ownerList` semantics, removes the no-op helper, and proves the large-list Gateway path.

## Reproduction Assessment

Yes. Source inspection shows current main still walks `commands.ownerAllowFrom` and then uses linear `includes` checks during command authorization; I did not run a live 9,000-entry Gateway benchmark in this read-only review.

## Solution Assessment

No. Memoizing normalized arrays is a useful layer, but the best fix also caches membership as a `Set` or equivalent O(1) structure and includes real Gateway proof for the reported large-list path.

- **[P2] Cache owner membership as a Set:** `src/auto-reply/command-auth.ts:283-286`

  - confidence: 0.9

  - confidence: 0.82
