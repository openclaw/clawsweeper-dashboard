## Summary

Keep open. Current main has useful optional hardening primitives, but it still does not implement the requested native data-centric Security Profile v1.1, and the discussion now includes both an explicit maintainer keep-open signal and a newer AX/screen-capture threat-model gap.

## What This Changes

Request a native data-centric security profile with data levels, cross-tool filesystem/exec/network policy, approvals, audit logging, session isolation, TTL elevation, and plugin verification.

## Best Possible Solution

Keep this as the native security-profile umbrella until maintainers define the product/security spec or split it into canonical narrower issues, using #60981/#7722 for the filesystem slice and ClawMoat as external prior art.

## Reproduction Assessment

Not applicable. This is a feature/security design proposal rather than a reported current-main failure; source inspection shows partial controls, not a single failing reproduction path.

## Solution Assessment

No. Current main's optional workspace-only FS, sandbox, exec approvals, and operator proxy are useful primitives, but treating them or ClawMoat as complete would leave native data-level, cross-tool, audit, elevation, plugin-verification, and screen-capture gaps.
