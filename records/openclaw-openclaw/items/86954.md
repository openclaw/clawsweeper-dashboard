## Reproduction Assessment

Yes, from source inspection: apply a `sessions.changed` row with `phase: "start"` and `status: "running"` while `chatRunStatus` still holds a terminal status and `chatRunId` is null; the PR's helper can normalize that fresh row to terminal.

## Solution Assessment

No. The stale-row direction is reasonable, but the implementation should distinguish stale terminal echoes from fresh start events or newer run ids before rewriting session state.
