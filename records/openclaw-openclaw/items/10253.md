## Summary

Keep open. Current main still lacks a hook/webhook-specific lifecycle policy for on-complete cleanup, delayed cleanup, transcript retention, or completed-session visibility; the existing generic session maintenance and cron run-session reaper do not cover the requested hook-owned semantics.

## What This Changes

Request a configurable lifecycle policy for webhook/hook sessions, including auto-archive or removal, optional delay, transcript retention, and completed-session visibility.

## Best Possible Solution

Define an explicit hook-owned cleanup policy with config schema/help/docs and runtime finalization for direct hooks, mapped hooks, Gmail presets, generated hook keys, completed-session visibility, and transcript handling.

## Reproduction Assessment

Not applicable as a strict bug reproduction. Source inspection on current main confirms the requested hook cleanup fields and post-completion cleanup path are absent while hook sessions persist as agent-scoped rows.

## Solution Assessment

Unclear until maintainers choose the lifecycle semantics. The maintainable direction is an explicit hook-owned policy rather than silently reusing subagent archival or relying only on generic session maintenance.
