## Summary

Keep open. Current main still lacks hook/webhook-specific cleanup config and post-run archive/delete finalization, and the broader ephemeral-sessions thread remains open as overlapping work rather than a clear replacement for this hook-specific request.

## What This Changes

The issue requests configurable hook/webhook session lifecycle cleanup, including on-complete or delayed cleanup and optional transcript retention.

## Best Possible Solution

Add an explicit hook lifecycle policy with config types, strict schema/help/docs, runtime finalization for direct hooks, mapped hooks, Gmail presets, generated hook keys, visibility semantics, and transcript handling.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature/config request. Source inspection on current main confirms the requested cleanup fields and post-completion hook cleanup path are absent while hook sessions persist as agent-scoped entries.

## Solution Assessment

Unclear until maintainers choose the lifecycle semantics. The maintainable direction is an explicit hook-owned policy coordinated with https://github.com/openclaw/openclaw/issues/60745, rather than silently reusing subagent archival or relying only on generic retention timers.
