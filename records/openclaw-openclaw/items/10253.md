## Summary

Keep open: current main still lacks a hook/webhook-specific session lifecycle policy; generic age/count cleanup exists, but it does not provide the requested on-complete or delayed archive/delete behavior for Gmail-style hook sessions.

## What This Changes

Request configurable cleanup/archive lifecycle policy for hook/webhook-generated sessions, including on-complete or delayed cleanup and transcript retention controls.

## Best Possible Solution

Define an explicit opt-in core hook lifecycle policy with schema/help/docs and runtime finalization for direct hooks, mapped hooks, and Gmail presets, including archive/delete/no-op, delay, transcript retention, and visibility semantics.

## Reproduction Assessment

Yes for source-level verification: current main exposes hook session keys and dispatches them through isolated cron runs, but has no hooks.sessionCleanup config/schema or completion finalizer. This is not a strict bug reproduction because the item requests a new policy surface.

## Solution Assessment

Unclear until maintainers choose the lifecycle contract. The narrowest maintainable direction is an explicit opt-in hook lifecycle policy rather than overloading subagent archive settings or generic retention.
