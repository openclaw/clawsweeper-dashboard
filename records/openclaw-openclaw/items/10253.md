## Summary

Keep open. Current main still lacks hook/webhook-specific lifecycle cleanup, and the broader ephemeral-session issue overlaps but does not fully replace this hook-specific policy request.

## What This Changes

The issue requests configurable auto-archive/delete behavior for completed webhook/hook-triggered sessions, including delayed cleanup and transcript retention controls.

## Best Possible Solution

Define an additive, opt-in hook lifecycle policy covering direct /hooks/agent, mapped hooks, Gmail presets, generated hook:* keys, delayed cleanup, archive/delete behavior, and completed-session visibility.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature/config request. Source inspection gives high confidence current main cannot express or apply the requested hook-session cleanup policy.

## Solution Assessment

Unclear until maintainers choose lifecycle semantics. The maintainable direction is an explicit hook cleanup policy or a coordinated ephemeral-session design with https://github.com/openclaw/openclaw/issues/60745, not overloading generic session.maintenance or subagent archive settings.
