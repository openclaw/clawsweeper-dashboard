## Summary

Keep open. Current main still has retention-based session maintenance and cron cleanup, but no hook/webhook-specific lifecycle policy for on-complete archive/delete, delayed cleanup, transcript retention, or hiding completed hook sessions.

## What This Changes

Request to add configurable auto-cleanup and archival behavior for hook-triggered sessions such as Gmail webhook runs.

## Best Possible Solution

Define an additive opt-in hook lifecycle policy covering direct /hooks/agent, hook mappings, Gmail presets, generated hook:* keys, delayed/on-complete archive or delete behavior, transcript retention, and completed-session visibility.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature/config request. Source inspection gives high confidence that current main cannot express or apply the requested hook-session lifecycle policy.

## Solution Assessment

Unclear until maintainers choose lifecycle semantics. The maintainable direction is an explicit opt-in hook cleanup policy, not overloading generic session maintenance, cron retention, subagent archive settings, or deleteAfterRun.
