## Summary

Keep open. Current main still lacks a hook-owned cleanup lifecycle policy; the existing generic session maintenance and cron reaper do not provide the requested on-complete, delayed cleanup, transcript-retention, or completed-session visibility semantics.

## What This Changes

Request a configurable auto-cleanup/archive policy for hook and webhook sessions, including optional delay and transcript retention.

## Best Possible Solution

Define an explicit hook-owned cleanup policy with config schema/help/docs and runtime finalization for direct hooks, mapped hooks, Gmail presets, generated hook keys, completed-session visibility, and transcript handling.

## Reproduction Assessment

Not applicable as a strict bug reproduction. Source inspection on current main confirms the requested hook cleanup fields and post-completion cleanup path are absent while hook sessions persist as agent-scoped rows.

## Solution Assessment

Unclear until maintainers choose the retention semantics. The maintainable direction is an explicit hook-owned lifecycle policy rather than silently reusing subagent archival or relying only on generic session maintenance.
