## Summary

Keep open. Current main and the latest release still lack a hook/webhook-specific lifecycle policy for on-complete cleanup, delayed cleanup, transcript retention, or completed-session visibility, and the related broader ephemeral-session issue does not fully replace this hook/Gmail-specific request.

## What This Changes

Add configurable cleanup and archival behavior for completed webhook/hook-created sessions.

## Best Possible Solution

Define an additive, opt-in hook lifecycle policy for direct /hooks/agent, mapped hooks, Gmail presets, generated hook:* keys, delayed cleanup, archive/delete behavior, transcript retention, and completed-session visibility, coordinated with the broader ephemeral-session design.

## Solution Assessment

Unclear until maintainers choose lifecycle semantics. The maintainable direction is an explicit opt-in hook cleanup policy, not overloading generic session.maintenance, cron.sessionRetention, subagent archive settings, or cron deleteAfterRun.
