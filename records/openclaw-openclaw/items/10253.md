## Summary

Keep this issue open: current main still persists hook-triggered isolated sessions without completion-aware cleanup, and the broader ephemeral-session issue remains an open product proposal rather than an implemented or clearly designated replacement.

## What This Changes

Add configurable immediate or delayed cleanup, archival, transcript retention, and completed-session visibility for hook-triggered sessions.

## Best Possible Solution

Define one completion-aware lifecycle contract for isolated one-shot sessions, then let hooks and cron select it with explicit archive/delete behavior, transcript retention, failure handling, and safe precedence over existing maintenance policies.

## Reproduction Assessment

Yes at source level: hook:* keys flow through isolated execution, while the cron reaper only accepts canonical cron run-session keys and generic maintenance is age/count/budget based rather than completion-aware.

## Solution Assessment

No, the hook-only configuration sketch is not clearly the best abstraction; a shared ephemeral one-shot lifecycle would avoid competing cleanup rules for hooks and isolated cron jobs.
