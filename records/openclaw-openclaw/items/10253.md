## Summary

Keep open. Current main still lacks a hook/webhook-specific lifecycle policy for completed hook sessions, and the remaining work adds public configuration plus destructive retention semantics that need maintainer product review.

## What This Changes

Request configurable archival or cleanup behavior for completed webhook/hook-triggered sessions, including delay, transcript retention, and completed-session visibility controls.

## Best Possible Solution

Add an explicit hook-owned lifecycle policy with config types, schema/help/docs, and runtime finalization for direct hooks, mapped hooks, Gmail presets, generated hook keys, completed-session visibility, and transcript handling.

## Reproduction Assessment

Not applicable as a strict bug reproduction. Source inspection on current main confirms the requested hook cleanup fields and post-completion cleanup path are absent while hook sessions persist as agent-scoped rows.

## Solution Assessment

Unclear until maintainers choose the lifecycle semantics. The maintainable direction is an explicit hook-owned policy rather than silently reusing subagent archival or relying only on generic session maintenance.
