## Summary

Current main still persists non-cron hook runs as ordinary agent sessions and only provides age/count-based maintenance, so the requested completion lifecycle remains missing. This issue should stay open for a product decision rather than close as a duplicate: https://github.com/openclaw/openclaw/issues/60745 covers the same root problem and should be consolidated with this report, but it does not fully preserve the delayed-cleanup, transcript-retention, and completed-session visibility requirements documented here.

## What This Changes

Add a configurable lifecycle policy for hook-triggered sessions covering completion-based or delayed cleanup, transcript retention, and completed-session visibility.

## Best Possible Solution

Consolidate this report with https://github.com/openclaw/openclaw/issues/60745 into one canonical isolated task-session lifecycle that marks completion, supports explicit immediate or delayed disposal where justified, defines transcript archival/deletion separately, and has clear precedence over existing retention maintenance.

## Reproduction Assessment

Yes at source level: non-cron hook keys follow the persistent agent-session path, and existing cleanup only provides timer/budget-based maintenance. No live hook execution was performed during this read-only review.

## Solution Assessment

No, not as an independent hook-only cleanup subsystem; the narrowest maintainable solution is a shared isolated-session completion lifecycle that incorporates the hook-specific retention and visibility requirements.
