## Summary

Keep open. Current main still lacks hook/webhook-specific session lifecycle controls, and adding cleanup/archive/delete behavior is a public feature/config decision with retention semantics to define.

## What This Changes

Request configurable cleanup, archival, visibility, and transcript-retention policy for completed webhook/hook agent sessions.

## Best Possible Solution

Add an explicit hook-owned lifecycle policy with config types, strict schema/help/docs, runtime finalization for direct hooks, mapped hooks, Gmail presets, generated hook keys, completed-session visibility, and transcript handling.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature/config request. Source inspection on current main confirms the requested hook cleanup fields and post-completion cleanup path are absent while hook sessions persist as agent-scoped rows.

## Solution Assessment

Unclear until maintainers choose the lifecycle semantics. The maintainable direction is an explicit hook-owned policy rather than silently reusing subagent archival or relying only on generic session maintenance.
