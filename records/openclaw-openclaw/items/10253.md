## Summary

Keep open. Current main still lacks a hook-specific cleanup, archival, or transcript-retention policy, and the related open items cover overlapping immediate-disposal or generic retention work rather than the full hook lifecycle policy requested here.

## What This Changes

Request adds configurable cleanup/archive lifecycle controls for webhook/hook-triggered sessions, including on-complete or delayed cleanup and transcript retention behavior.

## Best Possible Solution

Add an explicit hook-owned cleanup policy with config schema/docs and runtime finalization for direct hooks, mapped hooks, Gmail presets, generated hook keys, completed-session visibility, and transcript handling.

## Reproduction Assessment

Not applicable as a traditional bug reproduction. Source inspection on current main confirms the requested hook cleanup fields and post-completion cleanup path are absent while hook sessions persist as agent-scoped rows.

## Solution Assessment

Unclear until maintainers choose retention semantics. The maintainable direction is an explicit hook-owned lifecycle policy rather than silently reusing subagent archival or relying only on generic session maintenance.
