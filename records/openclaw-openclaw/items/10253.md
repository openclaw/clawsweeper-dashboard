## Summary

Keep open. Current main and v2026.5.22 still lack a hook/webhook-specific lifecycle policy for on-complete archive/delete, delayed cleanup, transcript retention, or completed-session visibility; the related ephemeral-session issue and persistent-hook-session PR are adjacent but do not supersede this Gmail/hook cleanup request.

## What This Changes

The issue requests configurable lifecycle cleanup for webhook/hook-triggered sessions, including on-complete or delayed archive/delete behavior and optional transcript retention.

## Best Possible Solution

Define an additive opt-in hook lifecycle policy covering direct /hooks/agent, hook mappings, Gmail presets, generated hook:* keys, delayed or on-complete archive/delete behavior, transcript retention, and completed-session visibility.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature/config request. Source inspection gives high confidence that current main cannot express or apply the requested hook-session lifecycle policy.

## Solution Assessment

Unclear until maintainers choose lifecycle semantics. The maintainable direction is an explicit opt-in hook cleanup policy, not overloading generic session maintenance, cron retention, subagent archive settings, or deleteAfterRun.
