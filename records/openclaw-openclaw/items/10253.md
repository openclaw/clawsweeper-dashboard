## Summary

Keep open. Current main still lacks a hook/webhook-specific cleanup, archival, delay, transcript-retention, or completed-session visibility policy, and the related open items cover overlapping persistent/ephemeral behavior rather than the full hook lifecycle contract requested here.

## What This Changes

Add configurable cleanup and archival behavior for completed webhook/hook-triggered sessions, including timing, transcript retention, and sessions-list visibility.

## Best Possible Solution

Define an explicit hook-owned cleanup policy with config schema/help/docs and runtime finalization for direct hooks, mapped hooks, Gmail presets, generated hook keys, completed-session visibility, and transcript handling.

## Reproduction Assessment

Not applicable as a traditional bug reproduction. Source inspection on current main confirms the requested hook cleanup fields and post-completion cleanup path are absent while hook sessions persist as agent-scoped rows.

## Solution Assessment

Unclear until maintainers choose retention semantics. The maintainable direction is an explicit hook-owned lifecycle policy rather than silently reusing subagent archival or relying only on generic session maintenance.
