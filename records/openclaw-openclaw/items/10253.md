## Summary

Keep open. Current main and the latest release still lack a hook/webhook-specific lifecycle policy for on-complete archive/delete, delayed cleanup, transcript retention, or completed-session visibility; related work is adjacent but does not supersede this hook/Gmail cleanup request.

## What This Changes

Request an opt-in configurable cleanup and archival policy for completed webhook/hook sessions, especially Gmail hook sessions that accumulate in the session store and transcript files.

## Best Possible Solution

Define an additive opt-in hook lifecycle policy covering direct /hooks/agent, hook mappings, Gmail presets, generated hook:* keys, delayed/on-complete archive or delete behavior, transcript retention, and completed-session visibility.

## Solution Assessment

Unclear until maintainers choose lifecycle semantics. The maintainable direction is an explicit opt-in hook cleanup policy, not overloading generic session maintenance, cron retention, subagent archive settings, or deleteAfterRun.
