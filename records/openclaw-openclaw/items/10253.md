## Summary

Keep open. Current main still lacks a hook-specific configurable cleanup, archival, or transcript-retention policy; related ephemeral-session work overlaps immediate disposal but does not fully cover the requested hook lifecycle policy.

## What This Changes

Request a configurable lifecycle policy for webhook and hook-triggered sessions so completed hook sessions can be archived, removed, delayed, or retained intentionally.

## Best Possible Solution

Add an explicit hook-owned cleanup policy with config schema/docs and runtime finalization for direct hooks, mapped hooks, Gmail presets, generated hook keys, and transcript handling.

## Reproduction Assessment

Yes for the missing capability. Source inspection on current main shows hooks.sessionCleanup/sessionRetention is not accepted by strict schemas, and hook agent runs persist as agent:<id>:hook:* rows without post-completion cleanup.

## Solution Assessment

Unclear until maintainers choose retention semantics. The maintainable direction is an explicit hook-owned cleanup policy rather than silently reusing subagent archiving or relying only on generic session maintenance.
