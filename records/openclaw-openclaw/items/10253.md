## Summary

Keep open. Current main still lacks a hook-specific cleanup or retention policy, post-run hook finalization, and transcript-retention semantics; generic session maintenance can eventually prune synthetic hook rows but does not cover the requested on-complete or delayed lifecycle behavior.

## What This Changes

Request a configurable webhook/hook session lifecycle policy for auto-archive or removal, delayed cleanup, transcript retention, and completed-session visibility.

## Best Possible Solution

Add an explicit hook cleanup policy, likely hooks.sessionCleanup or hooks.sessionRetention, with config types, strict schema/help/docs, and runtime finalization for direct /hooks/agent, mapped hooks, Gmail presets, and transcript archival behavior.

## Reproduction Assessment

Yes for the missing capability. Source inspection on current main shows hooks.sessionCleanup/sessionRetention is not accepted by config schemas, while hook agent runs persist as agent:<id>:hook:* rows without a post-completion cleanup path.

## Solution Assessment

Unclear until maintainers choose retention semantics. The requested direction is reasonable, but the maintainable fix should be an explicit hook-owned cleanup policy rather than silently reusing subagent archive settings or relying only on generic session maintenance.
