## Summary

Keep open. Current main still lacks a hook-specific cleanup, archival, or transcript-retention policy, and related open/closed items are either narrower immediate-disposal work or unmerged prior attempts rather than an implementation on main.

## What This Changes

Request configurable lifecycle handling for hook/webhook sessions, including on-complete or delayed archive/removal and optional transcript retention.

## Best Possible Solution

Add an explicit hook-owned cleanup policy with config schema/docs and runtime finalization for direct hooks, mapped hooks, Gmail presets, generated hook keys, and transcript handling.

## Reproduction Assessment

Yes for the missing capability. Source inspection shows the proposed hook cleanup fields are absent from strict config schemas, and hook agent runs persist as agent-scoped rows without post-completion cleanup.

## Solution Assessment

Unclear until maintainers choose retention semantics. The maintainable direction is an explicit hook-owned policy rather than silently reusing subagent archiving or relying only on generic session maintenance.
