## Summary

Keep open. Current main still lacks hook-specific cleanup configuration and post-run archive/delete finalization, and the broader ephemeral-sessions issue is overlapping work rather than a complete replacement for this hook-focused request.

## What This Changes

The issue requests a configurable lifecycle policy to archive or delete completed webhook/hook sessions, optionally delayed and with transcript retention controls.

## Best Possible Solution

Add an explicit hook lifecycle policy with config types, strict schema/help/docs, and runtime finalization for direct /hooks/agent, mapped hooks, Gmail presets, generated hook keys, visibility semantics, and transcript handling.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature/config request. Source inspection gives high confidence that current main lacks the requested cleanup fields and post-completion hook cleanup path while hook sessions persist as agent-scoped entries.

## Solution Assessment

Unclear until maintainers choose the lifecycle semantics. The maintainable direction is an explicit hook-owned policy coordinated with https://github.com/openclaw/openclaw/issues/60745, not silent reuse of subagent archival or generic retention timers.
