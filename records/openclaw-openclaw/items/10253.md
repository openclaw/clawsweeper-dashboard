## Summary

Keep open. Current main still lacks a hook/webhook-specific cleanup policy and completion finalizer, and the open ephemeral-sessions thread is overlapping context rather than a clear replacement for this hook-specific request.

## What This Changes

The issue requests configurable cleanup or archival behavior for completed hook/webhook-triggered sessions.

## Best Possible Solution

Define an explicit opt-in hook lifecycle policy with config schema/docs and runtime finalization for direct hooks, mapped hooks, Gmail presets, generated hook keys, visibility, and transcript handling.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature/config request. Source inspection on current main confirms the requested cleanup fields and post-completion hook cleanup path are absent while hook sessions persist as agent-scoped entries.

## Solution Assessment

Unclear until maintainers choose lifecycle semantics. The maintainable direction is an explicit hook-owned cleanup policy coordinated with https://github.com/openclaw/openclaw/issues/60745, rather than silently reusing subagent archival or relying only on generic retention timers.
