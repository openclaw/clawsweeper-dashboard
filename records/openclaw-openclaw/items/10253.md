## Summary

Keep open. Current main still lacks hook/webhook-specific lifecycle cleanup, and the related ephemeral-session issue overlaps but does not replace this hook-specific config and Gmail/webhook policy request.

## What This Changes

The issue requests configurable cleanup, archival, transcript retention, and completed-session visibility behavior for hook/webhook-triggered sessions.

## Best Possible Solution

Define an additive, opt-in hook lifecycle policy for direct /hooks/agent, mapped hooks, Gmail presets, generated hook:* keys, delayed cleanup, archive/delete behavior, transcript retention, and completed-session visibility, coordinated with https://github.com/openclaw/openclaw/issues/60745.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature/config request. Source inspection gives high confidence current main cannot express or apply the requested hook-session cleanup policy.

## Solution Assessment

Unclear until maintainers choose lifecycle semantics. The maintainable direction is an explicit hook cleanup policy or coordinated ephemeral-session design, not overloading generic session.maintenance, cron.sessionRetention, or subagent archive settings.
