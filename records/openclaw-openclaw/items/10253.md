## Summary

Keep open: current main still lacks a hook/webhook-specific completion cleanup or archival policy, and the related cron cleanup PR is scoped to cron `deleteAfterRun` rather than hook lifecycle semantics.

## What This Changes

Request to add configurable cleanup/archive behavior for webhook and hook-triggered sessions, including completion cleanup, delayed cleanup, and transcript-retention control.

## Best Possible Solution

Define one maintainer-approved hook session lifecycle contract with safe defaults, explicit archive/delete/no-op modes, transcript-retention semantics, direct/mapped/Gmail hook coverage, precedence over existing maintenance, docs, schema, and focused gateway/cron tests.

## Reproduction Assessment

Yes, at source level. Current main dispatches direct and mapped hook agent runs through isolated cron execution, stores non-`cron:` hook keys as ordinary agent session rows, and has only generic or cron-specific retention cleanup.

## Solution Assessment

Unclear until maintainers choose the lifecycle contract. A hook-only config knob is plausible, but the best fix must align destructive cleanup with existing session maintenance, cron retention, and persistent hook-session semantics.
