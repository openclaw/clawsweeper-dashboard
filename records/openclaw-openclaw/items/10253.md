## Summary

Keep open. Current main still lacks a hook/webhook-specific lifecycle policy, and existing cron/session maintenance does not provide on-complete or delayed hook-session archive/delete behavior with transcript-retention control.

## What This Changes

Request adds a configurable lifecycle policy for webhook/hook sessions, including archive/delete timing and transcript retention.

## Best Possible Solution

Add an explicit hooks.sessionCleanup policy with conservative defaults, schema/help/docs coverage, runtime finalization for direct and mapped/Gmail hooks, and focused tests for agent:<id>:hook:gmail:* cleanup semantics.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a mapped Gmail hook can resolve hook:gmail:<messageId>, dispatch as an isolated CronJob, persist as agent:<id>:hook:gmail:<messageId>, and bypass the cron-run-only reaper.

## Solution Assessment

Unclear until maintainers choose retention semantics. The requested direction is reasonable, but the safer shape is an explicit hooks.sessionCleanup policy rather than implicitly reusing subagent archival settings.
