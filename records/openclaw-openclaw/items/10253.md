## Summary

Keep open: current main still lacks a hook-specific cleanup/archive lifecycle policy, and the merged cron delete-after-run cleanup only covers cron session keys, not completed webhook or Gmail hook sessions.

## What This Changes

The issue requests configurable cleanup/archive behavior for completed hook/webhook sessions.

## Best Possible Solution

Define a maintainer-approved hook session lifecycle contract with safe defaults, explicit archive/delete/no-op or delay modes, transcript-retention semantics, direct/mapped/Gmail hook coverage, docs/schema, and precedence with existing maintenance.

## Reproduction Assessment

Yes, at source level. Current main has no hook cleanup config/schema surface, dispatches hook agents through isolated cron-style execution, and only the cron-specific cleanup helper deletes cron session keys.

## Solution Assessment

Unclear until maintainers choose the lifecycle contract. A hook-specific cleanup policy is plausible, but cleanup/archive/delete defaults must be coordinated with generic session maintenance, cron retention, and persistent hook-session semantics.
