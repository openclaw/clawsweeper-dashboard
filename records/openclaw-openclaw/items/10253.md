## Summary

Keep open: current main and v2026.6.5 still lack an explicit hook/webhook session lifecycle policy; generic session and cron retention handle age/count cleanup, not the requested completion-based archive/delete/transcript controls.

## What This Changes

Add configurable lifecycle cleanup for hook/webhook-created sessions, including on-complete or delayed cleanup, archival, transcript retention, and completed-session visibility.

## Best Possible Solution

Define an explicit opt-in hook lifecycle policy in core hooks config with schema/help/docs, runtime finalization for direct hooks, mappings, and Gmail presets, plus tests for archive/delete/no-op, transcript retention, and completed-session visibility.

## Reproduction Assessment

Yes for source-level verification: current main exposes hook session keys and dispatches them through isolated cron runs, but has no hooks.sessionCleanup config/schema or completion finalizer. This is not a strict bug reproduction because the item requests a new policy surface.

## Solution Assessment

Unclear until maintainers choose the lifecycle contract. The narrowest maintainable direction is an explicit opt-in hook lifecycle policy rather than overloading subagent archive settings or generic retention.
