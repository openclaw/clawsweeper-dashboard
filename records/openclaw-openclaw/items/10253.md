## Summary

Keep this issue open: current main and v2026.6.5 still persist hook-triggered isolated sessions without completion-aware cleanup, and the related generic ephemeral-session issue does not fully cover delayed cleanup, completed-session visibility, or audit-retention policy.

## What This Changes

Add configurable completion-aware cleanup, optional delay, transcript retention, and completed-session visibility for hook-triggered sessions.

## Best Possible Solution

Consolidate the overlapping requests into one opt-in isolated-session lifecycle contract supporting immediate or delayed disposal, explicit transcript retention or archival, and consistent completed-session visibility for hooks and cron.

## Reproduction Assessment

Yes. Current main routes hook agent work through isolated session persistence, while available cleanup operates by retention age, entry count, archive age, or disk budget rather than run completion.

## Solution Assessment

The requested outcome is valid, but the hook-only configuration is not the best final boundary; it should share one lifecycle contract with the broader ephemeral isolated-session proposal while retaining this issue's delayed-cleanup and audit requirements.
