## Summary

Keep open. Current main still lacks a hook/webhook lifecycle policy, and the broader ephemeral-session request at https://github.com/openclaw/openclaw/issues/60745 overlaps but does not clearly supersede this hook-specific policy request or its delayed cleanup/transcript-retention semantics.

## What This Changes

The issue requests configurable cleanup or archival for completed hook-triggered sessions, including Gmail webhook sessions and transcript retention controls.

## Best Possible Solution

Define an additive, opt-in hook lifecycle policy that covers direct /hooks/agent, mapped hooks, Gmail presets, generated hook:* keys, delayed cleanup, archive/delete behavior, and completed-session visibility without relying on generic age-based maintenance.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature/config request. Source inspection gives high confidence current main cannot express or apply the requested hook-session cleanup policy.

## Solution Assessment

Unclear until maintainers choose lifecycle semantics. The maintainable direction is an explicit hook cleanup policy or a coordinated ephemeral-session design, not overloading generic session.maintenance or subagent archive settings.
