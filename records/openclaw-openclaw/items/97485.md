## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Honor subagent iteration caps:** `src/agents/embedded-agent-runner/run.ts:1591`
  - body: `subagentMaxIterations` is added to the schema/config surface, but every run constructs `IterationBudget` with `resolvedBudgetConfig.maxIterations`. A user who configures the advertised subagent cap gets the parent budget instead, so the new safety setting is silently ineffective for subagent sessions. Select the subagent limit for `agent:<parent>:subagent:*` sessions or remove the field until it is supported.
  - confidence: 0.88
- **[P1] Count each budgeted tool round only once:** `src/agents/embedded-agent-runner/run.ts:1945`
  - body: The same `IterationBudget` is consumed once by the outer attempt loop and again by `onBeforeToolCallingRound`. That makes the configured limit smaller than advertised, and `maxIterations: 1` can exhaust the budget before the first tool-calling round is allowed. Keep one canonical counter for the budgeted unit or use separate counters with explicit semantics.
  - confidence: 0.86
- **[P2] Document the new config surface:** `src/config/zod-schema.agent-runtime.ts:77-84`
  - body: This strict `iterationBudget` config block is added to runtime schema without the matching config help, labels, or gateway docs entries. Users and UI/help surfaces will not discover or understand the new keys, and config-contract policy expects schema, metadata, and docs to move together for new public config.
  - confidence: 0.82
