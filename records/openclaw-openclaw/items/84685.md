## Summary

## What This Changes

The issue reports that the infra-state CI shard fails because the secret-file symlink test expects the non-throwing helper to return undefined while fs-safe now throws FsSafeError.

## Best Possible Solution

Align src/infra/secret-file.test.ts with fs-safe 0.2.7's fail-closed secret-file contract, preserving undefined only for blank or missing paths unless a deliberate public SDK compatibility wrapper is added with tests.

## Solution Assessment

Yes, the narrowest maintainable fix is to update the stale test expectation for the symlink validation case while keeping blank and missing path behavior unchanged. A wrapper change would be broader and should only happen if maintainers want an explicit OpenClaw public SDK compatibility contract different from fs-safe.
