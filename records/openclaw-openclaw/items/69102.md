## Summary

Keep open. Current main and the latest release still lack a Google/Gemini Flex service-tier config, normalization, documentation, or payload emission path; the duplicate was closed back to this canonical tracker, and the remaining API shape needs maintainer product judgment.

## What This Changes

Request support for configuring Google/Gemini Flex service tier so Gemini requests can include `service_tier`/`serviceTier: "flex"`.

## Best Possible Solution

Add Google-plugin-owned `serviceTier`/`service_tier` normalization and direct Gemini payload forwarding, document Flex timeout expectations, and keep cron/idle-timeout controls aligned with the separate timeout tracker.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for the missing capability: current main and `v2026.5.18` have no Google service-tier option, alias normalization, docs contract, or payload emission path. I did not run a live paid Gemini Flex call.

## Solution Assessment

Is this the best way to solve the issue? Mostly yes: provider-owned params support is the narrowest maintainable direction, while query-string model refs and cron/heartbeat-specific controls should remain separate maintainer decisions.
