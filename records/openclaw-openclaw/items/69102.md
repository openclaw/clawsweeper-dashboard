## Summary

Keep open. Current main still lacks a Google/Gemini Flex service-tier path: Google documents `service_tier: "flex"` and the JS SDK exposes `serviceTier`, but OpenClaw's Google transport, extra-param wrapper, docs, and tests only cover existing fields such as `cachedContent`, thinking, and tool options. ([googleapis.github.io](https://googleapis.github.io/js-genai/release_docs/interfaces/types.GenerateContentConfig.html))

## What This Changes

Add a way to configure Gemini requests so direct Google model calls can send `service_tier: "flex"`.

## Best Possible Solution

Implement opt-in provider-local Flex support in the bundled Google plugin by normalizing `serviceTier`/`service_tier`, forwarding a valid `flex` value to direct Gemini payloads, and documenting the model-param setup.

## Reproduction Assessment

Yes. Configure `agents.defaults.models["google/gemini-3-flash-preview"].params.serviceTier = "flex"` and inspect current main: the extra-param path does not copy it into Google stream options, and the direct Gemini payload builder cannot emit `service_tier`.

## Solution Assessment

Yes. The narrowest maintainable fix is in the Google provider transport and extra-param plumbing; a generic core service-tier API or cron-specific control would be broader than the reported missing Gemini field.
