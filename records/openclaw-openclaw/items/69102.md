## Summary

Keep open. Current main still lacks a Google/Gemini service-tier configuration path: `serviceTier`/`service_tier` is not typed on the bundled Google transport, not copied from model extra params into Google stream options, not emitted as top-level `service_tier` in direct Gemini payloads, and not documented or tested for Google Flex.

## What This Changes

Add a Google/Gemini configuration option that forwards `serviceTier` or `service_tier: "flex"` into Gemini requests so users can select the Flex service tier.

## Best Possible Solution

Add provider-owned support in the bundled Google plugin: normalize `serviceTier`/`service_tier` from model extra params, pass a valid `flex` value into Google transport options, emit top-level `service_tier` for direct Gemini API payloads, and document the model-param setup. Cron and heartbeat should not need a separate core feature if they already resolve to a configured Google model.

## Reproduction Assessment

Yes. A static reproduction is high-confidence: configure `agents.defaults.models["google/<model>"].params.serviceTier = "flex"` and inspect the current Google extra-param and direct Gemini payload path; current main has no field or test that can produce `service_tier`.

## Solution Assessment

Yes. The narrowest maintainable fix is in the Google provider transport plus existing extra-param plumbing, with tests/docs/changelog; a generic core service-tier API or scheduler-specific cron feature would be broader than the reported need.
