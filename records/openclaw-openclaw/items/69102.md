## Summary

Keep open. Current main still lacks Google/Gemini Flex service-tier plumbing, the related report is closed as a duplicate of this canonical issue, and live PR search did not find an implementation candidate.

## What This Changes

The issue requests Google/Gemini provider support for configuring `service_tier: "flex"` so lower-priority Gemini API requests can use the Flex tier.

## Best Possible Solution

Add Google-plugin-owned `serviceTier`/`service_tier` normalization and forwarding to the direct Gemini request payload, document the timeout/queueing caveat, and cover the params path with focused tests.

## Reproduction Assessment

Yes for source-level reproduction: configure a Google model params value such as `serviceTier: "flex"`, and current main never normalizes that value into Google stream options or emits `service_tier` in the direct Gemini payload. I did not run a live Gemini request because this read-only review should not use external credentials.

## Solution Assessment

Yes. The narrow maintainable solution is Google-provider-owned params support; model-ref query-string syntax and cron-only policy can stay out of scope unless maintainers choose those APIs separately.
