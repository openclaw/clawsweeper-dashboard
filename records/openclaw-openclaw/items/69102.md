## Summary

Keep open. Current main still lacks a Google/Gemini Flex service-tier configuration path: the bundled Google transport does not type or forward `serviceTier`/`service_tier`, the direct Gemini payload builder cannot emit top-level `service_tier`, and the existing docs/tests only cover `cachedContent` for this provider.

## What This Changes

Request to support Gemini Flex inference by letting Google model params forward `serviceTier` or `service_tier: "flex"` into direct Gemini requests.

## Best Possible Solution

Add provider-owned support in the bundled Google plugin: normalize `serviceTier`/`service_tier`, pass a valid `flex` value through Google stream options, emit top-level `service_tier` for direct Gemini payloads, and document/test the model-param setup.

## Reproduction Assessment

Yes. A static reproduction is high-confidence: configure `agents.defaults.models["google/<model>"].params.serviceTier = "flex"` and inspect the current Google extra-param and direct Gemini payload path; current main has no field or test that can produce `service_tier`.

## Solution Assessment

Yes. The narrowest maintainable path is a Google-provider transport and extra-param change with docs/tests/changelog; a generic core tier API or separate cron/heartbeat feature would be broader than the reported need.
