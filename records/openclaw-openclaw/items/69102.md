## Summary

Keep open. Native Google/Gemini Flex support is still missing on current main and the latest release; the OpenRouter workaround comment does not satisfy the requested first-party Google provider contract.

## What This Changes

Request adding a Google/Gemini service-tier config path so OpenClaw can send `service_tier: "flex"` or `serviceTier` for Gemini requests.

## Best Possible Solution

Implement Google-plugin-owned `serviceTier`/`service_tier` normalization and direct Gemini payload forwarding, document the supported config surface, and keep broader cron/idle-timeout controls aligned with the linked timeout tracker.

## Reproduction Assessment

Yes for the missing capability: source and release inspection show no native Google `serviceTier`/`service_tier` option, docs contract, or payload emission path. I did not run a live paid Gemini Flex request.

## Solution Assessment

Unclear until maintainers choose the public surface. Provider-owned params forwarding is the narrowest likely implementation, while cron/heartbeat overrides and query-string model refs are broader product decisions.
