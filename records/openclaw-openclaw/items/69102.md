## Summary

Keep open. Native Google/Gemini still lacks a Flex service-tier request path on current main and the latest release; the OpenRouter-style workaround in the discussion may help some users but does not implement the requested Google provider contract.

## What This Changes

Request support for configuring Gemini Flex so OpenClaw can send `service_tier`/`serviceTier` on Google Gemini model requests.

## Best Possible Solution

Add Google-plugin-owned Flex params support through existing `extraParams` and Gemini payload plumbing, document timeout guidance using the existing provider timeout knob, and avoid query-string model refs unless maintainers explicitly choose that contract.

## Reproduction Assessment

Yes for the missing capability: current main and `v2026.6.6` have no native Google Flex service-tier config, normalization, docs, or payload emission path. I did not run a live paid Gemini Flex call in this read-only review.

## Solution Assessment

Unclear until maintainers choose the public API shape. The narrowest maintainable direction appears to be Google-provider-owned params support, while query-string model refs and cron-specific service-tier controls should not be added without an explicit product decision.
