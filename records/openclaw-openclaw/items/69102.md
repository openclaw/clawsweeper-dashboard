## Summary

Keep open. Current main still lacks a Google/Gemini Flex service-tier configuration path, the duplicate report at https://github.com/openclaw/openclaw/issues/78258 was closed in favor of this canonical issue, and no open or merged fix PR was found.

## What This Changes

Request support for configuring Google/Gemini model requests to send the Flex `service_tier` option.

## Best Possible Solution

Add Google-plugin-owned `serviceTier`/`service_tier` handling through the existing model params pipeline, emit direct Gemini `service_tier`, document the timeout/Flex caveat, and leave broader cron or model-ref query syntax for a separate maintainer decision if needed.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to configure `agents.defaults.models["google/<model>"].params.serviceTier = "flex"`; current source never copies that field into Google stream options and the direct Gemini payload builder cannot emit `service_tier`.

## Solution Assessment

Yes. The narrow maintainable solution is provider-owned params support in the Google plugin; parsing `?serviceTier=flex` model refs or adding cron-only policy would be broader than the core request.
