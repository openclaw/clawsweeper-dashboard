## Summary

Keep open. Current main still lacks a Google/Gemini Flex service-tier configuration path, the later duplicate was closed in favor of this canonical issue, and no open or merged fix PR was found.

## What This Changes

Request support for configuring Gemini Flex so Google model requests can send the upstream `service_tier`/`serviceTier` flex option.

## Best Possible Solution

Add Google-plugin-owned `serviceTier`/`service_tier` handling through the existing params pipeline, emit direct Gemini `service_tier`, document the timeout caveat, and leave model-ref query syntax or cron-only policy for separate discussion.

## Reproduction Assessment

Yes, source-level verification is high confidence: configure `agents.defaults.models["google/<model>"].params.serviceTier = "flex"`, and current main never copies that field into Google stream options or the direct Gemini request payload. A live paid Gemini call was not needed to prove the missing plumbing.

## Solution Assessment

Yes. The narrow maintainable path is provider-owned params support in the Google plugin; parsing `?serviceTier=flex` model refs or adding cron-specific policy is broader than the core request.
