## Summary

Keep open. Current main only partially addresses the report: it supports a documented `dreaming.model` path, but request-scoped Dream Diary fallback still writes the first raw snippet, timeout/no-text terminal states can still leave `DREAMS.md` unchanged, and `memory status` does not expose fallback counts.

## What This Changes

The issue reports memory-core Dreaming cron producing raw or missing Dream Diary entries, ingesting routing metadata into the dreaming corpus, and lacking fallback diagnostics in `memory status`.

## Best Possible Solution

Keep this as a current memory-core bug, review the existing diary fallback and raw-dump PRs, then settle one coherent fallback, ingestion-filter, and status-diagnostic contract with tests and docs.

## Reproduction Assessment

Yes, source-level. Current main has clear paths where request-scoped fallback writes a raw snippet, non-ok/no-text narrative runs return before appending, and `memory status` lacks fallback-count reporting.

## Solution Assessment

Unclear.
