## Summary

Keep open. Current main and the latest release still preserve the main reported gaps: request-scoped Dream Diary fallback can write raw snippets, terminal/no-text narrative paths can skip `DREAMS.md`, routing-metadata filtering is only partial, and `memory status` does not expose narrative fallback counts.

## What This Changes

The issue requests reliable Dream Diary narrative generation, denoising of routing metadata from Dreaming session ingestion, and visibility when narrative fallback is used.

## Best Possible Solution

Keep this as a current Dream Diary fallback/diary tracker, review or replace the overlapping fallback and hardening pull requests, and leave the public Dreaming execution config shape coordinated with the related config issue.
