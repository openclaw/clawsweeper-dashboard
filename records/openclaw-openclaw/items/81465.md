## Summary

Keep open. Current main and v2026.5.18 still have source-backed gaps for this report: request-scoped Dream Diary fallback writes the first snippet verbatim, terminal/no-text narrative paths can return without a `DREAMS.md` append, session ingestion still relies on provenance or cron classification rather than filtering the literal routing text reported here, and `memory status` does not expose narrative fallback counts.

## What This Changes

The issue asks memory-core Dreaming to stop raw fallback diary output, filter inter-session routing metadata from the dreaming corpus, and surface fallback/no-diary diagnostics in memory status.

## Best Possible Solution

Keep this as a canonical Dream Diary fallback/ingestion tracker, land or replace the focused fallback and denoise PRs, and coordinate the public config/status contract with the related config issue.

## Reproduction Assessment

Yes, at source level. Current main has deterministic paths where request-scoped fallback writes raw snippets and timeout/no-text narrative runs return without appending `DREAMS.md`, and reporter logs show the same cron-success/no-diary behavior on recent releases.

## Solution Assessment

Unclear. The issue bundles fallback writing, corpus filtering, status visibility, and config shape; the narrow maintainable path is to harden fallback and ingestion while leaving the public execution fallback contract to https://github.com/openclaw/openclaw/issues/70616.
