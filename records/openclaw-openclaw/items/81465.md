## Summary

Keep open. Current main still leaves meaningful source-level gaps for raw fallback diary text, partial transcript hygiene, and fallback observability, while the already-open PRs and related issues only cover pieces of the reported behavior.

## What This Changes

The issue reports memory-core Dreaming cron runs producing phase reports while Dream Diary narrative generation falls back or skips `DREAMS.md`, with raw routing metadata in the corpus and no fallback visibility in `openclaw memory status`.

## Best Possible Solution

Keep the documented `dreaming.model` path, then harden memory-core so fallback diary writes are formatted and non-raw, session ingestion drops OpenClaw routing metadata, and `memory status` exposes a deliberate fallback diagnostic.

## Reproduction Assessment

Unclear for a full live cron reproduction because this read-only review did not run the gateway scheduler. Yes at source level: current main still has raw request-scoped fallback, terminal no-append paths, partial literal metadata filtering, and no fallback count in status output.

## Solution Assessment

Unclear. Adding the rejected direct `dreaming.execution.model` key alone is not the best fix because current main already has `dreaming.model` and `dreaming.execution.defaults.model`; the maintainable fix should coordinate fallback prose, corpus hygiene, and diagnostics with the open memory-core PRs.
