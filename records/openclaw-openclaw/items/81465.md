## Summary

Keep open. Current main and the latest release still have source-backed gaps for this report: request-scoped Dream Diary fallback can write a raw snippet, terminal narrative/no-text paths can return without a diary write, session-corpus filtering does not cover the literal routing text called out here, and `memory status` does not report narrative fallback counts.

## What This Changes

The issue reports that Dreaming cron writes phase reports but does not produce real Dream Diary entries because narrative generation falls back or skips writes and session-corpus ingestion admits routing metadata.

## Best Possible Solution

Keep this as the canonical v2026.5.7 Dream Diary fallback/diary tracker, land or replace the focused fallback and denoise PRs after review, and coordinate the public config/status contract with the related config issue.
