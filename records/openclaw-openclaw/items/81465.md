## Summary

Keep open. Current main and v2026.5.12 still contain source-backed gaps for this report: request-scoped Dream Diary fallback can write a raw snippet, terminal/no-text narrative paths can return without a diary write, literal routing text filtering is incomplete, and memory status still does not surface narrative fallback counts.

## What This Changes

The issue asks for Dream Diary narrative generation to avoid raw fallback entries, denoise inter-session routing metadata from memory ingestion, and expose fallback/status information when DREAMS.md is not receiving real entries.

## Best Possible Solution

Keep this as the canonical Dream Diary fallback and diary-write tracker, land or replace the focused fallback and denoise fixes, and coordinate the public execution-model config contract with the related open config issue.
