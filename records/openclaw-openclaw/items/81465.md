## Summary

Keep open. Current main and v2026.5.12 still have source-backed gaps for the reported Dream Diary behavior: request-scoped fallback writes the first raw snippet, timeout/error/no-text paths can return without a diary append, session ingestion only filters structured inter-session provenance and cron-classified transcripts, and `memory status` has no Dreaming narrative fallback count.

## What This Changes

The issue reports that Dreaming narrative generation falls back or skips diary writes, leaving `DREAMS.md` without generated Dream Diary entries and exposing routing metadata in memory corpus inputs.

## Best Possible Solution

Keep this as the composite Dream Diary fallback/diary tracker, land or replace the focused fallback and denoise PRs after review, and coordinate the public execution/status contract with https://github.com/openclaw/openclaw/issues/70616.
