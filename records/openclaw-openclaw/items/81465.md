## Summary

Keep open. Current main and v2026.5.18 still have source-backed gaps for the reported Dream Diary path: request-scoped fallback writes the first raw snippet, terminal/no-text narrative paths return without a diary append, session ingestion only partially filters the routing wrappers named in the report, and status output still has no narrative fallback count.

## What This Changes

The issue reports that memory-core Dreaming cron runs write phase reports but fall back to raw snippets or skip `DREAMS.md` writes, while routing metadata pollutes the Dreaming corpus and status output hides fallback use.

## Best Possible Solution

Keep this as the canonical v2026.5.7 Dream Diary fallback/no-entry tracker, land or replace focused fallback and denoise fixes, and route the public execution fallback contract through the related config issue.

## Reproduction Assessment

Yes for the core failure classes by source inspection: current main can write raw fallback snippets, skip diary writes on terminal/no-text narrative runs, and ingest unstructured routing wrapper text. No live current-main cron reproduction was executed in this read-only review.

## Solution Assessment

Unclear as a single change. The maintainable path is to split the bug fixes for fallback/no-write and corpus filtering from the public config/status contract, with https://github.com/openclaw/openclaw/issues/70616 owning the broader execution fallback API decision.
