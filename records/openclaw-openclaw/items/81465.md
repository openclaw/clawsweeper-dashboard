## Summary

Keep open. Current main still has source-backed gaps for raw request-scoped fallback diary writes, incomplete routing-metadata filtering in the session corpus path, and no `memory status` fallback counter; related open PRs cover pieces but do not make this issue obsolete.

## What This Changes

Report that memory-core Dream Diary cron/fallback can produce no real `DREAMS.md` entries, ingest routing metadata, and hide fallback usage from status output.

## Best Possible Solution

Keep one coordinated memory-core fix path: make fallback diary entries safe and deterministic, filter routing metadata before corpus scoring, and add an owner-approved fallback diagnostic while relying on the existing `dreaming.model` contract unless the separate config issue changes it.

## Reproduction Assessment

No for a high-confidence full live cron reproduction in this review. Source inspection does reproduce the narrower raw fallback and missing status-count paths, and the issue provides concrete v2026.5.7 cron/log evidence for the broader symptom.

## Solution Assessment
