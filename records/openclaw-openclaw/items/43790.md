## Summary

Keep open. Current main already covers the central explicit message-tool routing through session-agent plumbing, but this PR is not merge-ready because it still has a source-level fallback-routing finding, no real behavior proof, and a conflicting branch against newer main.

## What This Changes

The branch threads an explicit agent id into message-tool routing, switches tool-policy session fallback to `resolveSessionAgentId`, adds focused routing/default-agent tests, and adds a changelog line.

## Best Possible Solution

Refresh against current main, preserve only authoritative explicit or config-aware agent ids in message routing, keep the configured-default policy coverage if still needed, and add redacted real-run proof of correct send attribution.

## Reproduction Assessment

Yes for source-level paths: construct message-tool routing with conflicting explicit and session-derived agent ids, and construct `createOpenClawCodingTools` without config using an opaque session key plus a persisted non-main default. I did not run tests in this read-only review.

## Solution Assessment

No as-is: preferring a true explicit/effective agent id is the right direction, but passing a default-derived `main` as explicit when config is unavailable bypasses the lazy config-aware fallback. The safer solution is to refresh on current main and preserve config-aware fallback unless the caller supplied a real explicit agent id.
