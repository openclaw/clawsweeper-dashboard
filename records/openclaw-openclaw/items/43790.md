## Summary

Keep open: current main still routes message-tool sends from the session key instead of the already-resolved active agent, and this PR is useful, but the latest PR head is merge-conflicted and still has a configless opaque-session routing edge that should be fixed before merge.

## What This Changes

The PR adds explicit agent routing through OpenClaw/message tool construction, switches policy fallback toward session-agent resolution, and adds focused routing/default-agent tests plus a changelog entry.

## Best Possible Solution

Keep the explicit-agent precedence, but adapt it so only a truly explicit or config-backed requester agent overrides the message toolâ€™s session fallback; preserve regression coverage and the changelog entry.

## Reproduction Assessment

Yes. Source inspection on current main shows `createOpenClawCodingTools` resolves an active agent id, `createOpenClawTools` can compute `sessionAgentId`, but `createMessageTool` still derives the routed `agentId` only from `agentSessionKey`.

## Solution Assessment

Yes, directionally. Passing the resolved active agent into the message tool and preferring it over the session fallback is the narrow maintainable fix, but the PR needs a rebase and one fallback guard before merge.
