## Summary

Keep this PR open: current main already overlaps the explicit message-tool routing part, but the branch still contains a meaningful configured-default policy change and also still has a blocking configless fallback edge plus no real behavior proof.

## What This Changes

The branch threads an explicit/effective agent id into message-tool routing, changes policy fallback for non-agent session keys, and adds focused routing/default-agent tests plus a changelog entry.

## Best Possible Solution

Refresh against main, only pass authoritative explicit or config-aware agent ids into message routing, preserve configured-default regression coverage, and add redacted real-run or terminal proof of send attribution.

## Reproduction Assessment

Yes for a focused source-level path: construct message-tool routing with conflicting explicit and session-derived agent ids, and construct createOpenClawCodingTools without config using an opaque session key plus a non-main persisted default. I did not run tests in this read-only review.

## Solution Assessment

No as-is: preferring an authoritative explicit agent id is the right direction, but passing a derived fallback main as explicit can still bypass the configured default. The safer fix is to preserve lazy/config-aware session fallback when the caller did not supply a true explicit agent id.
