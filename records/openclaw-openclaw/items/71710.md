## Summary

Keep open. Current main still lacks a CLI-owned SIGTERM/SIGINT cancellation path for one-shot `openclaw agent` runs, and the public CLI/cron docs still do not document `timeout -k` for external cron wrappers. The report has concrete Linux/coreutils reproduction data and remains a narrow availability repair candidate.

## What This Changes

The issue requests that `openclaw agent` either exit gracefully on external SIGTERM/SIGINT from cron/coreutils `timeout` or clearly document SIGKILL escalation such as `timeout -k`.

## Best Possible Solution

Implement a focused one-shot shutdown contract: SIGTERM/SIGINT should abort the pending Gateway request, stop any accepted Gateway agent run through the existing abort registry or owner-disconnect path, pass the same signal into local and embedded fallback runs, preserve session-lock cleanup, and document `timeout -k` as the external-supervisor safety belt.

## Reproduction Assessment

Yes. The issue gives concrete Linux cron/coreutils steps with observed process chains and memory pressure, and current source supports a focused unit reproduction by holding a Gateway agent request or local embedded run pending, emitting SIGTERM/SIGINT, and observing that no CLI-owned abort path is wired.

## Solution Assessment

Yes. Bridging process signals into the existing Gateway agent abort path and `AgentCommandOpts.abortSignal` is the narrowest maintainable root fix; docs-only `timeout -k` guidance is useful mitigation but would leave the underlying hung-process behavior for other supervisors.
