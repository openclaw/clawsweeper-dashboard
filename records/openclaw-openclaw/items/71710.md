## Summary

Keep open. Current main still has server-side abort plumbing for Gateway agent runs, but the one-shot `openclaw agent` CLI path does not bridge SIGTERM/SIGINT into the pending Gateway request, the accepted run id, or embedded fallback, and the cron docs still lack the external `timeout -k` mitigation.

## What This Changes

The issue requests bounded SIGTERM handling for externally supervised `openclaw agent` runs, or at least documentation requiring `timeout -k` for cron wrappers.

## Best Possible Solution

Add a bounded shutdown path for one-shot `openclaw agent` runs that aborts the pending Gateway request, aborts the accepted Gateway run when known, passes the same signal to embedded/local execution, and documents `timeout -k` as a supervisor backstop.

## Reproduction Assessment

Yes. The issue gives concrete Linux cron/coreutils steps, and current source supports a focused unit reproduction by holding a Gateway `agent` request pending, sending SIGTERM/SIGINT to the CLI path, and observing that no CLI-owned abort path is wired.

## Solution Assessment

Yes. A CLI signal bridge into the existing Gateway and embedded abort seams is the narrow maintainable fix; documentation-only `timeout -k` guidance is useful as a backstop but should not be the whole fix.
