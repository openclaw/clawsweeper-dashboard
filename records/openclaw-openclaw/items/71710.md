## Summary

Keep open. Current main has server-side abort plumbing for Gateway agent runs, but the `openclaw agent` CLI still does not bridge SIGTERM/SIGINT into the pending Gateway request, accepted run id, or embedded fallback path, and the docs still lack the reporter's external `timeout -k` cron mitigation.

## What This Changes

The issue requests that cron-spawned `openclaw agent` runs exit promptly on SIGTERM or, at minimum, document supervisor wrappers such as `timeout -k`.

## Best Possible Solution

Add a bounded shutdown path for one-shot `openclaw agent` runs that forwards SIGTERM/SIGINT into both Gateway and embedded execution, aborts the accepted Gateway run when known, and documents `timeout -k` as a supervisor backstop.

## Reproduction Assessment

Yes. The issue gives concrete Linux cron/coreutils steps, and current source supports a focused unit reproduction by holding a Gateway `agent` request pending, delivering SIGTERM/SIGINT to the CLI path, and observing that no CLI-owned abort path is wired.

## Solution Assessment

Yes. The primary fix direction is the narrow maintainable path because Gateway and embedded agent execution already expose abort seams; the documentation mitigation should be included as a fallback, not treated as the complete fix.
