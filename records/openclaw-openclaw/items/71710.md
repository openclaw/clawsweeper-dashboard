## Summary

Keep open. Current main still leaves the one-shot `openclaw agent` path without a SIGTERM/SIGINT bridge into the pending Gateway request or embedded fallback run, and the CLI/cron docs still do not document an external `timeout -k` backstop.

## What This Changes

The issue asks OpenClaw to make cron-spawned `openclaw agent` invocations exit on SIGTERM, or at least document supervisor escalation for cron wrappers.

## Best Possible Solution

Add a bounded one-shot `openclaw agent` shutdown path that forwards SIGTERM/SIGINT into Gateway and embedded execution, aborts an accepted run when known, and documents `timeout -k` as a supervisor backstop.

## Reproduction Assessment

Yes for source-level reproduction: hold a Gateway `agent` request pending, deliver SIGTERM/SIGINT to the CLI path, and current main has no CLI-owned abort signal, accepted-run callback, or embedded fallback signal wiring to observe. I did not run the multi-day cron scenario live.

## Solution Assessment

Yes. The narrow maintainable fix is to bridge process signals into the existing Gateway and embedded abort seams, with docs as a supervisor backstop rather than the only fix.
