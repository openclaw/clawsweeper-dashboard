## Summary

Keep open. Current main still lacks a one-shot `openclaw agent` SIGTERM/SIGINT cancellation path for Gateway-backed and embedded/local runs, and the public agent/cron docs still do not mention the `timeout -k` mitigation.

## What This Changes

The issue asks for `openclaw agent` to cancel or exit promptly on SIGTERM under cron, or at minimum document `timeout -k` for cron-spawned one-shot agent runs.

## Best Possible Solution

Add a focused shutdown path for `openclaw agent`: SIGTERM/SIGINT should trigger a shared AbortController, abort the pending Gateway request, stop any accepted Gateway agent run through the existing abort registry or equivalent scoped path, pass the same signal to embedded/local runs, and document `timeout -k` as an external supervisor safety belt.

## Reproduction Assessment

Yes. The issue includes concrete Linux cron/coreutils steps and observed process chains, and current source supports a focused unit reproduction by holding a Gateway `agent` request or local embedded run pending, delivering SIGTERM/SIGINT, and observing that no CLI-owned abort path is wired.

## Solution Assessment

Yes. The primary fix is the narrowest maintainable direction because the agent runner and Gateway already have AbortSignal seams; a docs-only `timeout -k` warning is useful mitigation but would leave other supervisors vulnerable.
