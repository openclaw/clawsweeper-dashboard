## Summary

Keep open. Current main still lacks a SIGTERM/SIGINT bridge from the one-shot `openclaw agent` CLI into the pending Gateway request or embedded fallback run, and the docs still do not document `timeout -k` as a supervisor backstop.

## What This Changes

The issue asks OpenClaw to make cron-spawned `openclaw agent` exit promptly on SIGTERM or document a required external SIGKILL escalation wrapper.

## Best Possible Solution

Add a bounded shutdown path for one-shot `openclaw agent` that forwards SIGTERM/SIGINT into Gateway and embedded execution, aborts the accepted run when known, and documents `timeout -k` as an external supervisor backstop.

## Reproduction Assessment

Yes at source level. Hold a Gateway `agent` request pending, deliver SIGTERM/SIGINT to the CLI process, and current main has no CLI signal handler, caller abort signal, accepted-run callback, or embedded fallback signal wiring to observe.

## Solution Assessment

Yes. The narrow maintainable fix is to bridge process signals into the existing Gateway and embedded abort seams, with docs as a supervisor backstop rather than the only fix.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/71710. On current main, `openclaw agent` waits for `callGateway({ method: "agent", expectFinal: true })` without any SIGTERM/SIGINT bridge; `callGateway` and `GatewayClient.request` also cannot expose the accepted run id or accept caller cancellation, while the Gateway `agent` handler already registers an abort controller usable through `chat.abort`/`sessions.abort`. Implement a bounded one-shot shutdown path for `agentCliCommand`: install temporary SIGTERM/SIGINT handling for the command invocation, cancel the pending Gateway request, learn and abort the accepted Gateway run when available, thread the same signal into embedded fallback/local execution, and exit/settle within a small bounded grace window while preserving existing `expectFinal`, auth scope, timeout, JSON stdout, and embedded fallback behavior. Add focused regression tests for pending Gateway request cancellation after accepted ack, signal cleanup/removal, embedded fallback/local abort signal propagation, and no duplicate run/session corruption. Document `timeout -k <delay> <duration>` or equivalent as the recommended external cron supervisor backstop until every caller has hard-kill escalation. Add a single `CHANGELOG.md` fix entry. Do not solve the broader exec supervisor SIGTERM-to-SIGKILL policy from https://github.com/openclaw/openclaw/issues/66399 in this PR except where a small shared helper is necessary for the agent CLI path.
