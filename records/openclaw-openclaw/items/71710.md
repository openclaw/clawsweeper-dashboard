## Summary

Keep open. Current main still has no one-shot `openclaw agent` SIGTERM/SIGINT bridge into the Gateway request, the accepted Gateway run, or the embedded fallback path, and the public docs still do not document `timeout -k` as the cron-supervisor backstop.

## What This Changes

Make cron-spawned one-shot `openclaw agent` invocations terminate cleanly on SIGTERM or document `timeout -k` as the required supervisor fallback.

## Best Possible Solution

Add a bounded one-shot CLI shutdown path that forwards SIGTERM/SIGINT into Gateway and embedded agent execution, aborts the accepted run when known, and documents `timeout -k` as an external supervisor backstop.

## Reproduction Assessment

Yes, at source level. Hold a Gateway `agent` request pending, deliver SIGTERM/SIGINT to the one-shot CLI process, and current main has no CLI signal handler, caller AbortSignal, accepted-run callback, or embedded fallback signal wiring to observe.

## Solution Assessment

Yes. The narrow maintainable fix is to bridge process signals into the existing Gateway `agent` abort controller and embedded runner abort surfaces, with documentation as a backstop rather than the only protection.
