## Summary

Keep open. Current main has server-side abort support for Gateway agent runs, but the `openclaw agent` CLI still does not wire SIGTERM/SIGINT into the pending Gateway request, accepted run id, or embedded fallback path, and the docs still do not mention the reporter's `timeout -k` cron mitigation.

## What This Changes

The issue requests bounded SIGTERM/SIGINT shutdown for cron-spawned `openclaw agent` runs, plus cron documentation for `timeout -k` as a mitigation.

## Best Possible Solution

Add a bounded shutdown path for one-shot `openclaw agent` runs that forwards SIGTERM/SIGINT into Gateway and embedded execution, aborts the accepted Gateway run when known, and documents `timeout -k` as an external-supervisor backstop.

## Reproduction Assessment

Yes. The issue gives concrete Linux cron/coreutils steps, and current source supports a focused unit reproduction by holding an `agent` Gateway request pending, delivering SIGTERM/SIGINT to the CLI path, and observing that no CLI-owned abort path is wired.

## Solution Assessment

Yes. The reporter's primary direction is the narrow maintainable fix because Gateway and embedded agent execution already expose abort seams; the docs mitigation should be included as a fallback, not used as the only fix.
