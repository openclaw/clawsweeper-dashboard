## Summary

Keep open. Current main still does not connect external SIGTERM/SIGINT for one-shot `openclaw agent` runs to an active Gateway or embedded agent cancellation path, and the CLI/cron-facing docs still lack the `timeout -k` mitigation described by the reporter.

## What This Changes

The issue requests bounded SIGTERM handling for cron-spawned `openclaw agent` runs, or at minimum documentation requiring a SIGKILL escalation wrapper such as `timeout -k`.

## Best Possible Solution

Add a narrow shutdown path for `openclaw agent`: process SIGTERM/SIGINT should trigger a shared AbortController, stop the pending Gateway request, capture the accepted Gateway run id early enough to call the authorized abort path for that exact run, pass the same signal into embedded/local execution, bound the shutdown grace window, and document `timeout -k` as an external supervisor backstop.

## Reproduction Assessment

Yes. The issue gives concrete Linux cron/coreutils steps and current source supports a focused unit reproduction by holding a Gateway `agent` request or embedded `agentCommand` pending, delivering SIGTERM/SIGINT, and observing that no CLI-owned abort path is wired.

## Solution Assessment

Yes. The primary signal-handling fix is the narrowest maintainable solution because the Gateway and embedded runner already expose abort seams; the documentation-only fallback is useful mitigation but not sufficient as the final fix.
