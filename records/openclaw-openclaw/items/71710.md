## Summary

Keep open. Current main still lacks a one-shot `openclaw agent` SIGTERM/SIGINT path that cancels the pending Gateway request, aborts the accepted Gateway run, or passes a caller abort signal into embedded/local fallback; the docs also still do not document the reporter’s `timeout -k` cron mitigation.

## What This Changes

The issue requests graceful `openclaw agent` SIGTERM/SIGINT handling for cron/supervisor runs, with documentation for `timeout -k` as a mitigation.

## Best Possible Solution

Add a bounded shutdown path for `openclaw agent` that bridges SIGTERM/SIGINT to a shared AbortController, aborts the accepted Gateway run through the existing authorized abort seam when a run id is known, passes the same signal into embedded/local execution, and documents `timeout -k` as an external-supervisor backstop.

## Reproduction Assessment

Yes. The issue provides concrete Linux cron/coreutils steps, and current source supports a focused unit reproduction by holding a Gateway `agent` request or embedded `agentCommand` pending, delivering SIGTERM/SIGINT, and observing that no CLI-owned abort path is wired.

## Solution Assessment

Yes. The requested primary fix is the narrowest maintainable direction because Gateway and embedded runners already expose abort seams; the docs-only mitigation is useful but should remain a fallback rather than the final product behavior.
