## Summary

Keep open. Current main still does not wire SIGTERM/SIGINT into the `openclaw agent` one-shot paths: the Gateway request is timeout-only, local and embedded fallback runs do not receive an abort signal, and the existing Gateway agent abort controller is only reachable through Gateway abort APIs rather than the CLI process signal.

## What This Changes

Make `openclaw agent` terminate predictably on SIGTERM/SIGINT under external cron/coreutils timeout wrappers, with a bounded abort/drain path and operator documentation for `timeout -k` as mitigation.

## Best Possible Solution

Implement a focused one-shot CLI shutdown contract: SIGTERM/SIGINT should abort the pending Gateway request, stop the accepted Gateway agent run through the existing abort registry when possible, pass the same signal into local and embedded fallback runs, preserve session-lock cleanup, exit within a bounded grace deadline, and add operator docs for external cron wrappers that should use `timeout -k`.

## Reproduction Assessment

Yes. The issue gives concrete Linux cron/coreutils steps and process-tree evidence, and current main supports a focused unit reproduction by holding a Gateway `agent` request or embedded run pending, emitting SIGTERM/SIGINT, and observing that no CLI-owned abort path is wired.

## Solution Assessment

Yes. Bridging process signals into the existing Gateway and embedded abort seams is the narrowest maintainable root fix; documentation for `timeout -k` is useful mitigation guidance but should not be the only fix.
