## Summary

Keep open. Current main still lacks a one-shot `openclaw agent` SIGTERM/SIGINT bridge into the Gateway request, accepted agent run, or embedded fallback path, and the docs only cover Gateway-managed cron timeouts rather than external `timeout` wrappers.

## What This Changes

The issue requests bounded SIGTERM handling for cron-spawned `openclaw agent` runs, or at minimum documentation requiring `timeout -k` escalation for external supervisors.

## Best Possible Solution

Add a bounded one-shot CLI shutdown path that forwards SIGTERM/SIGINT into Gateway and embedded agent execution, aborts the accepted run when known, and documents `timeout -k` as an external-supervisor backstop.

## Reproduction Assessment

Yes, at source level. Hold a Gateway `agent` request pending, deliver SIGTERM/SIGINT to the one-shot CLI process, and current main has no CLI signal handler, caller AbortSignal, accepted-run callback, or embedded fallback signal wiring to observe.

## Solution Assessment

Yes. The narrow maintainable fix is to bridge process signals into the existing Gateway `agent` abort controller and embedded runner abort surfaces; docs-only `timeout -k` guidance is useful as a backstop but should not be the only fix.
