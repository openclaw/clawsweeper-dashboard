## Summary

Keep open for maintainer review: the patch looks like the right shared fix for the linked managed npm-root EOVERRIDE bug, but it intentionally changes upgrade-time writes to persisted managed plugin root package.json data and the PR carries a protected maintainer label.

## What This Changes

The PR updates managed plugin npm-root manifest writers so OpenClaw-managed peer dependency pins and managed npm overrides are reconciled before npm runs, with regression coverage for stale pins, ownership transfer, override shapes, and rollback restore.

## Best Possible Solution

Land the shared reconciliation after maintainer acceptance of the managed-root upgrade rewrite behavior, while leaving the misleading post-failure hook-pack message and uninstall alias-retry follow-ups separate.

## Reproduction Assessment

Yes. Source inspection plus reporter runtime logs show a stale managed peer pin can be combined with a newer managed override before npm runs, and npm 11.13 Arborist rejects that root as EOVERRIDE; I did not execute the packaged install scenario in this read-only review.

## Solution Assessment

Yes, if maintainers accept the upgrade-time rewrite behavior. The shared managed-root invariant is the best fix because upsert, peer sync, and rollback restore can each write a rejected manifest; a plugin-specific workaround or manual repair would leave sibling writers exposed.
