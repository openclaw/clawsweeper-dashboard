## Summary

Keep open: this maintainer-authored protected PR is the active fix candidate for the managed npm-root EOVERRIDE bug, and current main plus the latest release still have the stale managed peer-pin conflict shape. The patch looks correct from source, tests, caller tracing, CI, and npm Arborist contract inspection, but it still needs maintainer acceptance of the compatibility-sensitive upgrade-time manifest rewrites.

## What This Changes

The PR changes managed plugin npm-root manifest writers to reconcile OpenClaw-managed peer dependency pins with managed npm overrides and adds regression tests for stale pins, owned dependency conflicts, override shapes, explicit ownership transfer, fallback planning, and rollback restore.

## Best Possible Solution

Land the shared managed-root reconciliation after maintainer acceptance of the upgrade-time rewrite behavior, while leaving the misleading post-failure hook-pack message and uninstall alias-retry follow-ups separate.

## Reproduction Assessment

Yes. Source inspection plus reporter runtime evidence show a stale managed peer pin can be combined with a newer managed override before npm runs, and npm 11.13 Arborist rejects that root as EOVERRIDE; I did not execute the packaged install scenario in this read-only review.

## Solution Assessment

Yes. The shared managed-root invariant is the best fix because upsert, peer sync, and rollback restore can each write the rejected manifest; a plugin-specific workaround, manual repair, or sync-only patch would leave sibling writers exposed.
