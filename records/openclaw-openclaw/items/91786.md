## What This Changes

The PR updates managed plugin npm-root package.json writers to reconcile managed peer dependency pins with managed overrides and adds regression tests for stale pins, ownership transfer, override shapes, and rollback restore.

## Best Possible Solution

Land the shared reconciliation after maintainer acceptance of the upgrade-time managed-root rewrite behavior, while leaving the misleading post-failure hook-pack message and uninstall alias-retry follow-ups separate.

Yes. Source inspection plus reporter runtime logs show a stale managed peer pin can be combined with a newer managed override before npm runs, and npm 11.13 Arborist rejects that root as EOVERRIDE; I did not execute the packaged install scenario in this read-only review.

## Solution Assessment

Yes. The shared managed-root invariant is the best fix because upsert, peer sync, and rollback restore can each write a rejected manifest; a plugin-specific workaround or manual repair would leave sibling writers exposed.
