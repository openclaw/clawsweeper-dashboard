## What This Changes

This PR changes managed plugin npm-root package.json writers to reconcile OpenClaw-managed peer dependency pins with managed npm overrides and adds focused regression tests for stale pins, owned dependency conflicts, object/wildcard/alias overrides, and rollback restore.

## Best Possible Solution

Land the shared managed-root reconciliation after maintainer acceptance of the upgrade rewrite behavior, while keeping installer error-message and uninstall alias-retry follow-ups separate.

## Solution Assessment

Yes. The shared managed-root manifest invariant is the best fix because upsert, peer sync, and rollback restore can all write the conflicting package.json state; a plugin-specific workaround or manual-only repair would leave sibling writers exposed.
