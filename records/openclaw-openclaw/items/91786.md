The PR updates managed plugin npm-root package.json writers to reconcile OpenClaw-managed peer dependency pins with managed npm overrides and adds regression tests for stale pins, explicit ownership transfer, override shapes, and rollback restore.

## Solution Assessment

Yes. The shared managed-root invariant is the best fix because upsert, peer sync, and rollback restore can each write the rejected manifest; a plugin-specific workaround or manual repair would leave sibling writers exposed.
