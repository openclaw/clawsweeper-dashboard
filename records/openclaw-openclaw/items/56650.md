## Summary

Keep this PR open: the underlying non-interactive Copilot auth problem is still real on current main, but this patch is not merge-ready because it targets stale token plumbing and bypasses existing token validation/security expectations.

## What This Changes

The PR adds non-interactive `models auth paste-token` token sources and GitHub Copilot PAT handling that bypasses token exchange and routes to the enterprise Copilot API.

## Best Possible Solution

Rebase the work onto current main, implement PAT support in the active shared Copilot resolver/facade, attach required Copilot request headers through the prepared-auth request override seam, preserve paste-token validation for every token source, and add docs/changelog/tests for the supported non-interactive paths.

## Reproduction Assessment

Yes for the source-level problem: current main has no `paste-token --token` option and still exchanges all Copilot GitHub tokens through the token endpoint. The external PAT 403 was not live-retested.

## Solution Assessment

No. The direction is useful, but this patch is not the best current-main implementation because it targets a stale token facade and stores non-interactive tokens without the prompt path's validation.
