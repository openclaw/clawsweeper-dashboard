## Summary

Keep this PR open. The underlying automation problem still has real value, but the branch is not merge-ready against current main because it targets a stale Copilot token path and lets non-interactive token inputs bypass existing provider-specific validation.

## What This Changes

The PR adds non-interactive token sources to `models auth paste-token` and GitHub Copilot PAT handling that bypasses Copilot token exchange and uses the enterprise Copilot API.

## Best Possible Solution

Rebase and rework the branch against current main: add any generic non-interactive `paste-token` surface through the current CLI/auth command with shared validation for every token source, and implement Copilot PAT behavior in the active shared resolver/facade used by the plugin runtime, using current request-header or dynamic-header seams where needed. Add focused tests and docs for the supported automation path, and get security review for any inline secret flag.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path for the missing `paste-token --token` support and for PATs still entering the current Copilot token-exchange call; the live GitHub 403 was not re-run.

## Solution Assessment

No. The PR addresses the right problem, but as proposed it is not the narrowest maintainable fix for current main because it targets a stale resolver path and bypasses existing token validation behavior.
