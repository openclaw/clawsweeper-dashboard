## Summary

Keep open, but this PR should not merge as-is: the non-interactive setup problem is real, the `paste-token` half is now better covered by the narrower open PR at https://github.com/openclaw/openclaw/pull/63050, and the Copilot PAT half still needs a current-main implementation, security review, and real behavior proof.

## What This Changes

The PR adds scripted token input to `models auth paste-token` and adds a GitHub Copilot PAT path that bypasses token exchange for the enterprise Copilot endpoint.

## Best Possible Solution

Let the narrower `paste-token --token` PR own the generic CLI surface, and implement Copilot PAT support separately in the active provider-auth/runtime path with tests, docs, and redacted real PAT proof.

## Reproduction Assessment

Yes for the source-level current-main gaps: `paste-token` has no token flag and Copilot runtime auth still exchanges every GitHub token. A live PAT-backed Copilot failure path was not reproduced in this read-only review.

## Solution Assessment

No. The direction is useful, but this branch is not the best current-main fix because it overlaps a narrower open `paste-token` PR, patches stale Copilot resolver plumbing, and lacks real runtime proof for PAT auth.
