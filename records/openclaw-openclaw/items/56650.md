## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Attach Copilot headers in the PAT runtime path:** `extensions/github-copilot/index.ts:162-167`
  - body: The PAT branch returns only `apiKey`, `baseUrl`, and `expiresAt`, while the editor-header constant introduced by this PR is never passed through the runtime request config. Since direct Copilot API calls need those editor-identification headers, PAT-backed requests can still be sent without the required identity and fail at runtime.
  - confidence: 0.9
- **[P2] Do not route classic PATs as supported Copilot tokens:** `src/agents/github-copilot-token.ts:16-19`
  - body: `isGitHubPAT` treats both fine-grained `github_pat_` and classic `ghp_` tokens as supported direct Copilot credentials. GitHub's current Copilot CLI docs support fine-grained PATs with Copilot Requests permission but explicitly mark classic PATs unsupported, so this branch would advertise and route an unsupported credential type.
  - confidence: 0.84
- **[P2] Warn before accepting inline token arguments:** `src/cli/models-cli.ts:350-352`
  - body: The new help text advertises `--token <value>` without warning that inline secrets can be captured by shell history, command echoing, CI logs, or process listings. Add CLI/docs guidance that prefers stdin, env refs, or other non-argv secret paths before documenting inline token input as an automation path.
  - confidence: 0.78
