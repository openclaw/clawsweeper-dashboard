## Summary

Keep open: the automation problem is valid, but the branch is not mergeable against current main because the PAT handling is wired to stale token surfaces and the generic non-interactive paste path bypasses existing token validation.

## What This Changes

The branch adds `models auth paste-token` token/stdin/env inputs and attempts to let GitHub Copilot PATs bypass token exchange and use the enterprise Copilot API.

## Best Possible Solution

Rebase the work onto current main, implement PAT handling in the shared provider-auth resolver/facades used by the GitHub Copilot plugin runtime, attach any required enterprise headers through the current request-header or transport-turn seam, and route every non-interactive pasted token through the same provider-specific normalization and validation as the interactive prompt. Add focused tests, docs, and a changelog entry for the supported automation path.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main lacks `paste-token --token`, the active Copilot resolver still exchanges every token, and the PR diff targets helpers that are no longer the active runtime facade. The external 403 from GitHub was not live-reproduced in this read-only review.

## Solution Assessment

No. The PR targets the right user problem, but this is not the best current-main fix because it wires PAT support into stale token code and bypasses existing token validation; the safer path is a rebase through the current provider-auth and request-header seams.
