## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Attach the PAT headers to runtime requests:** `extensions/github-copilot/index.ts:157-164`
  - body: The PAT branch returns only `apiKey`, `baseUrl`, and `expiresAt`, while the new editor-header constant is never passed to the transport. Since the PR says those headers are required for PAT auth, PAT-backed requests can still go out without the required editor identity and fail at runtime.
  - confidence: 0.9
- **[P2] Warn before encouraging inline secret arguments:** `src/cli/models-cli.ts:347-351`
  - body: The new help text advertises `--token <value>` without warning that inline secrets can be captured by shell history, command echoing, CI logs, or process listings. Add CLI/docs guidance that prefers stdin or env/ref-backed storage for automation before exposing this as a documented path.
  - confidence: 0.78
- **[P2] Do not treat classic PATs as supported Copilot auth:** `src/agents/github-copilot-token.ts:15-17`
  - body: `isGitHubPAT` routes both fine-grained `github_pat_` and classic `ghp_` tokens directly to Copilot, but current GitHub Copilot CLI docs list fine-grained PATs as supported and classic PATs as unsupported. Either remove `ghp_` from the supported path or add live proof and docs explaining why OpenClaw can safely support it.
  - confidence: 0.74
