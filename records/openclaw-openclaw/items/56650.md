## Summary

Keep this PR open, but it is not merge-ready. The automation problem is real, yet the branch is stale against current auth seams, omits the runtime headers its own PAT design depends on, lacks after-fix real behavior proof, and needs maintainer/security judgment for argv-based secret input.

## What This Changes

The PR adds non-interactive token input to `models auth paste-token` and GitHub Copilot PAT handling that bypasses token exchange for the provider auth flow.

## Best Possible Solution

Rework PAT support on current main in the active SDK/provider-auth resolver with runtime `request.headers`, and let the narrower paste-token PR or an equivalent maintainer patch handle the CLI token flag if that policy is accepted.

## Reproduction Assessment

Yes, source-reproducible. Current main lacks a `paste-token --token` path and still routes GitHub Copilot runtime auth through the token-exchange resolver for all GitHub tokens; I did not run a live PAT-backed Copilot request.

## Solution Assessment

No. The direction is useful, but this PR is not the best current-main solution because it omits required runtime headers, bypasses non-interactive token validation, and patches stale Copilot token plumbing rather than the active provider-auth seam.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

  - body: The PAT branch switches to the enterprise endpoint but returns only `apiKey`, `baseUrl`, and `expiresAt`.   - confidence: 0.88

  - body: This adds PAT detection to the old `src/agents` helper, but current main's GitHub Copilot runtime resolves through `extensions/github-copilot/token.ts` and `openclaw/plugin-sdk/provider-auth`. After resolving conflicts, leaving the active SDK resolver unchanged would keep sending PATs to `/copilot_internal/v2/token`.
  - confidence: 0.84
- **[P2] Validate non-interactive tokens before storing:** `src/commands/models/auth.ts:384-392`
  - body: The flag, stdin, and env-var branches assign `token` directly and skip the interactive validator and Anthropic whitespace normalization. That lets malformed Anthropic setup tokens be persisted from automation even though the prompt path rejects or normalizes them before storage.
  - confidence: 0.82
