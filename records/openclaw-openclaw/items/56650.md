## Summary

Keep open: the automation gap is real and the Copilot PAT work remains unique, but this branch is stale and not merge-ready; the paste-token half is now covered by a narrower open PR, while the PAT half needs current-main provider-auth rework and proof.

## What This Changes

Adds non-interactive `models auth paste-token` token inputs plus GitHub Copilot PAT handling that bypasses token exchange and targets the enterprise Copilot endpoint.

## Best Possible Solution

Rework PAT support on current main in the active SDK/provider-auth path with runtime `request.headers`, and let the narrower paste-token PR or an equivalent maintainer patch handle the CLI token flag if that policy is accepted.

## Reproduction Assessment

Yes, source-reproducible: current main lacks a `paste-token --token` path and still sends every GitHub token through the Copilot token-exchange endpoint. I did not re-run a live PAT-backed Copilot request.

## Solution Assessment

No. The direction is useful, but this PR is not the best current-main solution because it omits required runtime headers, bypasses validation in non-interactive token storage, and targets stale Copilot auth plumbing instead of the active provider-auth seam.

## Review Findings

  - confidence: 0.84
