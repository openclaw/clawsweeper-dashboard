## Summary

Keep open: the auth automation problem is still real on current main, but this PR is not ready to merge because it targets stale Copilot token plumbing, bypasses existing token validation, and needs maintainer security review for inline secret input.

## What This Changes

The PR adds non-interactive token sources for `models auth paste-token` and GitHub Copilot PAT handling that bypasses token exchange and uses the enterprise Copilot API.

## Best Possible Solution

Rebase the PR onto current main, implement PAT handling in the shared active Copilot resolver/facade, attach Copilot editor headers through the existing prepared-auth request override seam, and route all non-interactive paste-token sources through the same provider-specific normalization and validation as the prompt path.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: current main has no `paste-token --token` option, and current Copilot runtime still exchanges every GitHub token through the Copilot token endpoint; the reported live PAT 403 itself was not re-run.

## Solution Assessment

No. The proposed direction is valid, but this patch is not the best current-main implementation because it targets stale resolver plumbing and bypasses existing validation; the safer path is a focused rebase through the current provider-auth and request-header seams.
