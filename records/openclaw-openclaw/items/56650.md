## Summary

Keep open: the underlying non-interactive Copilot automation need remains valid and is not implemented on current main, but this external PR should not merge yet because the patch misses runtime header propagation/validation, touches credential ingestion policy, and lacks after-fix real behavior proof.

## What This Changes

Adds non-interactive token sources for `models auth paste-token` and GitHub Copilot PAT handling that bypasses token exchange and selects the enterprise Copilot API.

## Best Possible Solution

Rework this on current main by preserving provider-specific token validation, putting Copilot PAT handling in the active shared resolver, attaching editor headers through `ProviderPreparedRuntimeAuth.request`, and validating with redacted real Copilot output.

## Reproduction Assessment

Yes. Source inspection on current main shows `paste-token` still lacks a non-interactive token option and the active Copilot resolver still sends every GitHub token to the token-exchange endpoint; the live PAT 403 itself was not re-run.

## Solution Assessment

No. The direction is useful, but this patch is not the best current-main implementation because it omits required runtime headers, bypasses existing token validation, and targets stale auth plumbing instead of the active provider-auth seam.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Attach PAT editor headers through runtime auth:** `extensions/github-copilot/index.ts:162-167`
  - body: The PAT branch switches to the enterprise endpoint but returns only `apiKey`, `baseUrl`, and `expiresAt`. Since the PR says editor-identification headers are required, return them through the current `request.headers` runtime-auth override so PAT-backed requests do not go out headerless.
  - confidence: 0.88
- **[P2] Route PAT support through the active Copilot resolver:** `extensions/github-copilot/index.ts:6-12`
  - body: Current main has moved the Copilot token facade to the SDK provider-auth resolver, while this branch adds PAT logic to the older agents helper. Rebase the change into the active resolver/facade used by `register.runtime.ts`, or the current runtime path will keep exchanging PATs.
  - confidence: 0.84
- **[P2] Validate non-interactive tokens before storing:** `src/commands/models/auth.ts:384-392`
  - body: The flag, stdin, and env branches assign the token directly, bypassing the interactive path's provider-specific validation and normalization. For example, Anthropic setup tokens that the prompt rejects or normalizes can be persisted unchanged from automation.
  - confidence: 0.82
