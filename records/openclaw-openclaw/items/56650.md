## Summary

Keep open for maintainer review: the automation problem is real, but this branch is not mergeable as-is because the paste-token half is now covered by a narrower open PR and the Copilot PAT half needs to be rebased through the active provider-auth resolver with runtime headers and real proof.

## What This Changes

Adds non-interactive token inputs for `models auth paste-token` and GitHub Copilot PAT handling that bypasses token exchange and targets the enterprise Copilot endpoint.

## Best Possible Solution

Rework this on current main: land or reuse the narrower paste-token PR if maintainers accept the argv-token policy, then implement Copilot PAT support in the active SDK provider-auth resolver with `request.headers` and redacted real Copilot proof.

## Reproduction Assessment

Yes. Source inspection on current main shows `paste-token` has no non-interactive token option and the active Copilot resolver still sends every GitHub token to the token-exchange endpoint; the live PAT 403 was not re-run.

## Solution Assessment

No. The direction is useful, but this patch is not the best current-main solution because it omits required runtime headers, bypasses provider-specific token validation, and targets stale auth plumbing instead of the active provider-auth seam.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Attach PAT editor headers through runtime auth:** `extensions/github-copilot/index.ts:162-167`
  - body: The PAT branch switches to the enterprise endpoint but returns only `apiKey`, `baseUrl`, and `expiresAt`. Since the PR says editor-identification headers are required and current main can carry `request.headers`, PAT-backed runtime requests would still go out without those headers.
  - confidence: 0.88
- **[P2] Move PAT support to the active Copilot resolver:** `src/agents/github-copilot-token.ts:108-118`
  - body: This adds PAT detection to the legacy Copilot token helper. Current main's GitHub Copilot facade re-exports the SDK provider-auth resolver, so after rebasing the runtime path would continue sending PATs to `/copilot_internal/v2/token` unless the active resolver is updated.
  - confidence: 0.84
- **[P2] Validate non-interactive tokens before storing:** `src/commands/models/auth.ts:384-392`
  - body: The flag, stdin, and env-var branches assign `token` directly and skip the interactive validator. Anthropic setup tokens that the prompt rejects or normalizes can be persisted unchanged from automation.
  - confidence: 0.82
