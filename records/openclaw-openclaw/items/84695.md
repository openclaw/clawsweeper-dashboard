## Summary

Keep open: current `main` and latest release `v2026.6.11` still run preflight compaction before the memory-flush phase, while the docs describe memory flush as pre-compaction behavior. The open linked fix at https://github.com/openclaw/openclaw/pull/84792 specifically owns this direct ordering regression, so this issue should stay open until that PR lands or is replaced.

## What This Changes

The issue reports that automatic compaction shows the compaction notice but does not append the configured memory-flush summary to `memory/YYYY-MM-DD.md`.

## Best Possible Solution

Review and land or intentionally replace https://github.com/openclaw/openclaw/pull/84792, then close this issue after merge while keeping in-turn, reset, sandbox, and append-validation follow-ups separate.

## Reproduction Assessment

Yes. Current source shows `runReplyAgent` runs preflight compaction before memory flush, and the later memory-flush gate can return without running after compaction has already reduced pressure; no live long-context Telegram/provider reproduction was run.

## Solution Assessment

Yes. The narrow maintainable fix for this direct bug is to run the existing memory-flush phase before direct preflight compaction, while keeping in-turn catch-up, reset flushing, sandbox restrictions, and bounded append validation in their own trackers.
