## Summary

Keep open: current main and v2026.6.9 still fail closed unless an external channel is bundled or has trusted official-install provenance, and the thread narrowed the ask to a provenance-preserving immutable-container deployment path rather than raw load-path trust.

## What This Changes

The issue requests a supported self-hosted/containerized deployment path for externalized official channel plugins that need trusted keyed state without weakening plugin provenance.

## Best Possible Solution

Keep the fail-closed trust gate and design a supported portable official-plugin install/provenance path for immutable images whose plugin package and install record survive a mounted runtime state directory.

## Reproduction Assessment

Yes at source level: a non-bundled external channel without matching trusted official-install metadata will throw when it opens keyed state, and Microsoft Teams opens keyed stores during runtime. I did not run the ECS/Fargate reproduction in this read-only review.

## Solution Assessment

No to raw trust allowlists or implicit load-path trust; the maintainable solution is a provenance-preserving portable install/deployment contract. The current issue is best kept open until maintainers choose that trust and storage shape.
