## Label Justifications

- `P1`: A supported official external channel can load in a common self-hosted container model and then repeatedly terminate when it reaches the trusted keyed-state gate.
- `impact:security`: The requested capability crosses the provenance boundary that controls trusted plugin access to host-managed persistent state and channel ingress queues.
- `impact:crash-loop`: The reported Microsoft Teams channel starts, exits on the keyed-store trust error, and auto-restarts repeatedly.

## Summary

Keep open: current main and latest release still require bundled origin or recorded trusted official-install provenance before external channel plugins can use keyed state, and there is still no supported immutable image-baked plugin root that survives a mounted state directory. The maintainer discussion correctly rejects raw load-path trust, but the reporter’s narrower container deployment gap remains a real product/security-boundary decision.

## What This Changes

The issue requests a supported container/self-hosted deployment path for official external channel plugins whose package and trust provenance can live outside a mounted OpenClaw state directory while still allowing `openKeyedStore`.

## Best Possible Solution

Design a provenance-preserving official plugin install/export contract for immutable images, with package identity and install records available outside the mutable state mount, documented container handoff, focused upgrade tests, and arbitrary load paths still untrusted.

## Reproduction Assessment

Yes. Source inspection shows a deterministic path: a non-bundled channel loaded without matching trusted official-install metadata throws at `openKeyedStore`; the issue also provides a concrete 2026.6.6 container trace, though this review did not run ECS/Fargate live.

## Solution Assessment

No to raw trust allowlists or implicit load-path trust because those weaken package provenance. The best direction is the narrower immutable official-install provenance contract the reporter clarified after maintainer feedback.
