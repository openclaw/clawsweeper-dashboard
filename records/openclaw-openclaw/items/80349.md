## Summary

Keep open: current main still does not pass thinking state into the embedded LLM idle-timeout resolver, while Gemini 2.5 Flash medium thinking still builds the reported 8192-token thinking budget. The nearby Google end-of-stream/SSE fixes address a distinct stall mode, and the direct thinking-idle PR was closed unmerged after review found it needed stronger live proof and a timeout-ordering correction.

## What This Changes

The issue reports Gemini 2.5 Flash timing out after about 125 seconds when non-minimal thinking is enabled, despite Google reporting successful streaming requests.

## Best Possible Solution

Live-reproduce current main with Gemini 2.5 Flash at medium versus minimal/off, then land a narrow thinking-aware idle-timeout or Google transport fix with focused regression coverage while keeping broader per-model timeout configuration under the separate feature request.

## Reproduction Assessment

No high-confidence live reproduction was run against current main. The issue has concrete logs and current source still shows that non-off Gemini thinking can use the default 120s cloud idle watchdog, so the path is source-reproducible but not fully reproduced.

## Solution Assessment

Unclear: closing is not the best outcome because current main lacks a thinking-aware idle-timeout fix. The safest implementation should be chosen after live provider confirmation, because the fix could belong in stream parsing, first-response handling, or idle-watchdog policy.
