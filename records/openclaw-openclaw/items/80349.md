Keep open: current main still has a plausible Gemini 2.5 Flash liveness gap, with `medium` mapping to an 8192 thinking budget, the default cloud idle watchdog at 120s, and the existing Gemini first-response retry intentionally scoped to Gemini 3 only.

## What This Changes

## Best Possible Solution

Capture a redacted live Google SSE/trajectory trace for `gemini-2.5-flash` with `thinkingDefault: medium`, then make a narrow Google transport or thinking-policy fix while leaving generic timeout config to the existing timeout issues.

## Reproduction Assessment

No high-confidence live reproduction was established here. The source path is clear enough to keep investigating: Gemini 2.5 Flash medium sends a large thinking budget under a 120s idle watchdog, but the actual Google SSE activity during the failing turns still needs capture.

## Solution Assessment

Unclear. Extending the Gemini first-response retry to 2.5 or defaulting Flash to lower thinking might help, but maintainers need a live trace and a decision on retry cost/default-thinking behavior before choosing the narrowest fix.
