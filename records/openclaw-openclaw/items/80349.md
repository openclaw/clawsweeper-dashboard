## Summary

Keep this open: current main and the latest release include the adjacent Google SDK/SSE and Gemini 3 first-response fixes, but the specific Gemini 2.5 Flash non-off thinking path still falls through to the default cloud idle watchdog and can match the reported roughly 125 second timeout.

## What This Changes

The issue asks OpenClaw to stop timing out Gemini 2.5 Flash turns when non-minimal thinking is enabled and Google reports successful streaming responses.

## Best Possible Solution

Add a narrow thinking-aware embedded idle-timeout or first-response recovery path for affected Google thinking runs, preserve explicit timeout caps, and keep broader per-model generation-timeout semantics in the separate open timeout-design issue.

## Reproduction Assessment

Yes at the source level, but not with a live Gemini run in this review. Current main still sends the reported Gemini 2.5 Flash medium thinking budget and applies the default cloud idle watchdog without any thinking-aware bypass.

## Solution Assessment

Yes, a narrow runtime fix is the best next direction. The closed linked PR had the right area but needed timeout-ordering corrections and live proof; a broad new per-model timeout config should stay separate unless maintainers choose that design.
