## Summary

Keep open: current main and latest release still route Gemini 2.5 Flash medium-thinking runs through the normal 120s cloud idle watchdog without passing thinking state into the timeout resolver. The related Google SSE/end-of-stream fix is adjacent, and the direct thinking-idle fix PR was closed unmerged after review found it needed stronger live proof and a timeout-ordering correction.

## What This Changes

The issue reports Gemini 2.5 Flash timing out after roughly 125 seconds with non-minimal thinking enabled, despite Google reporting successful streaming requests.

## Best Possible Solution

Live-reproduce current main with Gemini 2.5 Flash at medium versus minimal/off, then land a narrow thinking-aware idle-timeout or Google transport fix with focused regression coverage while keeping broader timeout configuration under the separate open timeout requests.

No high-confidence live reproduction was run against current main. The issue has concrete logs and current source still shows non-off Gemini thinking can use the default 120s cloud idle watchdog, so the path is source-reproducible but not fully reproduced.

Unclear: closing is not the best outcome because current main and v2026.5.12 lack a thinking-aware idle-timeout fix. The safest implementation should be chosen after live provider confirmation, because the fix could belong in stream parsing, first-response handling, or idle-watchdog policy.
