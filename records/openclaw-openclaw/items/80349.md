## Summary

Keep this open: current main and the latest release still leave the reported direct Google Gemini 2.5 Flash non-minimal-thinking path on the default cloud idle watchdog unless an operator opts into a longer provider timeout, and the only linked fix PR closed unmerged.

## What This Changes

The issue reports Gemini 2.5 Flash agent turns timing out after about 125 seconds when non-minimal thinking is enabled, despite Google reporting successful streaming responses.

## Best Possible Solution

Add a narrow thinking-aware idle-watchdog or first-response recovery path for affected Google Gemini 2.5 thinking runs, while keeping broad per-model generation-timeout product design in the separate timeout issue.

## Reproduction Assessment

Yes at the source level: current main still applies the default cloud idle watchdog without seeing the non-off thinking level, while Gemini 2.5 Flash medium thinking still sends a large thinking budget. I did not run a live Google credential repro in this read-only review.

## Solution Assessment

Yes, a narrow runtime/provider timeout recovery path is the best fix for this concrete bug. The existing `models.providers.*.timeoutSeconds` knob is a useful workaround, but it should not be the only answer for the default Gemini 2.5 Flash thinking path.
