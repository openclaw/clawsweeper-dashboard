## Summary

Keep open: current main still does not pass the active thinking level into the embedded LLM idle-timeout resolver, while Google Gemini 2.5 Flash with medium thinking still builds the reported 8192-token budget. The related Google SSE/end-of-stream fix covers a different stall mode, and the direct fix PR was closed unmerged after review found missing live proof and a timeout-ordering bug.

## What This Changes

Ensure Gemini 2.5 Flash non-minimal thinking runs do not hit OpenClaw's default idle timeout and fail over when Google successfully completes the streaming request.

## Best Possible Solution

Live-reproduce current main with Gemini 2.5 Flash at medium versus minimal/off, then land a narrow thinking-aware idle-timeout or Google transport fix with regression coverage while leaving broader per-model timeout configuration to the separate feature thread.

## Reproduction Assessment

No high-confidence live reproduction was run against current main. The source path is reproducible: non-off Gemini thinking can still be routed through the default 120s cloud idle watchdog, matching the reporter's repeated roughly 125s timeout logs.

## Solution Assessment

Unclear: a thinking-aware idle-watchdog fix is plausible, but the prior PR showed an ordering bug and lacked live provider proof. The safest fix should preserve explicit run/request hard caps and be chosen after live Google confirmation.
