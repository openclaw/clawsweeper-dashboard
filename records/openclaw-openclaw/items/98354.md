## Summary

Keep open: the bug is source-reproducible on current main and the patch appears scoped, but this external PR still lacks after-fix real behavior proof beyond focused test output.

## What This Changes

The PR changes SQLite delivery queue inflation to return null for corrupted `entry_json` rows, filters corrupt rows out of batch loads, and adds corrupt-row regression tests.

## Best Possible Solution

Land this shared SQLite inflate-boundary fix after the contributor adds real behavior proof and required checks finish cleanly.

## Reproduction Assessment

Yes from source inspection: current main calls `JSON.parse(row.entry_json)` outside a try/catch in the shared queue inflate helper, and startup recovery reads pending session and outbound rows through that path. I did not run the repro because this review is read-only.

## Solution Assessment

Yes, guarding the shared SQLite inflate chokepoint is the narrowest maintainable fix for preventing one bad row from aborting every valid row in the same queue. A future diagnostic enhancement could mark corrupt rows failed without parsing, but that is not necessary to fix the central crash path.
