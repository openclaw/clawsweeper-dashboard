## Summary

Keep open. Current main still lacks Signal edit-aware handling: edit envelopes are flattened into normal inbound messages, the edit target timestamp is not modeled, and the closed prior PR is not shipped.

## What This Changes

Add Signal plugin handling for `editMessage` envelopes so edited messages preserve the original target timestamp, edit context, and reply behavior.

## Best Possible Solution

Implement this in the bundled Signal plugin with typed edit metadata, target-timestamp identity, edit marking or structured metadata, mention-gated reply suppression, and focused regression tests.

## Reproduction Assessment

Yes, source-level. A Signal receive event with a fresh `envelope.timestamp` and `editMessage.targetSentTimestamp` currently follows the normal data-message path and enqueues `MessageSid` from the fresh envelope timestamp.

## Solution Assessment

Yes. The narrow maintainable path is a fresh Signal plugin repair; PR #9701 is useful prior art but is closed, unmerged, and targeted the old `src/signal` layout.
