## Summary

Keep open; current main still flattens Signal edit envelopes into ordinary inbound messages and the only Signal fix PR found is closed unmerged.

## What This Changes

The issue requests Signal editMessage envelope handling so edited messages retain their target timestamp relationship, avoid duplicate context, and do not trigger duplicate replies.

## Best Possible Solution

Implement edit-aware handling in the bundled Signal plugin with typed target timestamps, edit metadata or marker, target-timestamp dedupe or transcript replacement, and mention-gated reply suppression.

## Reproduction Assessment

Yes, source-level. A Signal receive event with envelope.editMessage.dataMessage, editMessage.targetSentTimestamp, and a fresh envelope.timestamp follows the ordinary data-message path and uses the fresh timestamp as MessageSid.

## Solution Assessment

Yes. The narrow maintainable solution belongs in extensions/signal because core inbound dedupe already keys off MessageSid once the plugin supplies edit-aware identity and admission behavior.
