## Summary

Keep open; current main still treats Signal edit envelopes as ordinary inbound messages, and the only Signal-specific fix PR found is closed unmerged.

## What This Changes

Handle Signal editMessage envelopes by preserving the edit target relationship, avoiding duplicate context, and suppressing unmentioned group edit replies.

## Best Possible Solution

Implement edit-aware handling in the bundled Signal plugin with typed target timestamps, edit metadata or an [edited] marker, target-timestamp identity or dedupe, mention-gated reply suppression, and focused Signal tests.

## Reproduction Assessment

Yes, source-level. A Signal receive event with envelope.editMessage.dataMessage, editMessage.targetSentTimestamp, and a fresh envelope.timestamp follows the ordinary data-message path and uses the fresh timestamp as MessageSid.

## Solution Assessment

Yes. The narrowest maintainable solution belongs in extensions/signal because core inbound dedupe already keys off MessageSid once the plugin supplies edit-aware identity and admission behavior.
