## Summary

Keep open. Current main and the latest release still lack a public `config unset` size-drop override, while the lower config writer already has the narrow primitive; this remains a valid feature request that needs maintainer CLI/safety-policy judgment.

## What This Changes

Requests a public `openclaw config unset` opt-in flag that allows intentional large deletions to bypass only the size-drop guard.

## Best Possible Solution

Add a maintainer-approved `config unset` flag that threads `allowConfigSizeDrop` only for explicit existing-path unsets, while preserving schema validation, rejected-artifact behavior, and gateway-mode protection.

## Reproduction Assessment

Yes, source-level. Current `config unset` can remove an existing path but never passes `allowConfigSizeDrop`, while the shared writer blocks `size-drop:*` unless that option is true.

## Solution Assessment

Unclear pending maintainer product decision. The narrow flag is the best apparent implementation shape because it reuses the existing lower-layer guard option, but maintainers should choose the public flag name and confirm this escape hatch belongs on `config unset`.
