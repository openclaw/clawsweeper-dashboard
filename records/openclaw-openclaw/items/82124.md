## Summary

Keep open. Current main still has the reported CLI product gap: `config unset` has no explicit size-drop override, while the lower config writer already has a narrow `allowConfigSizeDrop` primitive that is not reachable from this command.

## What This Changes

Request adding a narrow `openclaw config unset` opt-in flag that bypasses only the `size-drop:*` write guard for intentional large deletions.

## Best Possible Solution

Add a maintainer-approved `config unset` flag that threads only to `allowConfigSizeDrop`, with focused coverage for default rejection, flagged success, missing paths, schema validation, and hard-guard preservation.

## Reproduction Assessment

Yes. The issue provides a concrete temp-config reproduction, and source inspection on current main shows the same path: `config unset` can delete the large node but cannot pass `allowConfigSizeDrop`, so the writer can still reject the size drop.

## Solution Assessment

Unclear pending maintainer policy. The proposed narrow flag matches the existing writer primitive, but maintainers should choose the public option name and confirm that this CLI escape hatch belongs on `config unset`.
