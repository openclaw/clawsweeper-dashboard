## Summary

Keep open. Current main and the latest release still lack a public `config unset` size-drop override, while the lower config write layer already has the narrow guard option; this remains a valid feature request that needs maintainer CLI/safety-policy judgment before implementation.

## What This Changes

Request to add a narrow explicit `config unset` flag that bypasses only the size-drop guard for intentional large config deletions.

## Best Possible Solution

Add a maintainer-approved CLI flag that threads the existing write-layer `allowConfigSizeDrop` option only for explicit `config unset` deletes while preserving missing-path, schema, rejected-artifact, and gateway-mode protections.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current `config unset` removes an existing path but never passes `allowConfigSizeDrop`, while the write guard rejects `size-drop:*` unless that option is true.

## Solution Assessment

Unclear pending maintainer product decision. The narrow flag is the best apparent implementation shape because it reuses the existing lower-layer guard option, but maintainers should choose the public flag name and confirm this escape hatch belongs on `config unset`.
