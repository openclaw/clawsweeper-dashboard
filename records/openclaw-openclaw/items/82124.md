## Summary

Keep open. Current main and v2026.5.18 still lack a public `config unset` flag that forwards the existing narrow `allowConfigSizeDrop` write option, so this remains a valid focused feature request that needs maintainer CLI-policy approval.

## What This Changes

Request a narrow `openclaw config unset` opt-in flag to allow intentional large config deletions to bypass only the `size-drop:*` guard.

## Best Possible Solution

Add a clearly named `config unset` flag that sets `allowConfigSizeDrop` only for existing-path deletes while preserving schema validation, `unsetPaths`, rejected artifacts, and hard guards such as `gateway-mode-removed`.

## Reproduction Assessment

Yes from source inspection, not an executed current-main repro. `runConfigUnset` can remove a large node but never passes `allowConfigSizeDrop`, while the write guard rejects `size-drop:*` unless that option is true.

## Solution Assessment

Unclear pending maintainer product decision. The proposed path is technically narrow because the lower layer already has `allowConfigSizeDrop`, but maintainers should choose the public flag name and confirm this escape hatch belongs on `config unset`.
