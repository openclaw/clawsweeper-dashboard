## Summary

Keep open. Current main still has the reported product gap: `config unset` exposes only a path argument and does not forward the existing narrow `allowConfigSizeDrop` write option, so this needs maintainer judgment on the public CLI affordance.

## What This Changes

Add an explicit `openclaw config unset` flag that bypasses only the size-drop guard for intentional large config-node deletion.

## Best Possible Solution

Expose a narrow `config unset` opt-in flag that maps to `allowConfigSizeDrop` only for explicit deletes, with focused CLI docs and regression tests for default rejection, allowed size drop, and hard-guard preservation.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence current-main path: a large explicit `config unset` deletion can produce a candidate below the size-drop threshold, and the CLI does not pass the existing bypass option.

## Solution Assessment

Unclear pending maintainer product decision. The proposed narrow flag fits the existing lower-layer contract, but maintainers should choose the CLI name and confirm this escape hatch belongs on `config unset`.
