Fix the QQBot cron auto-TTS false-delivered bug reported in https://github.com/openclaw/openclaw/issues/92816. Current source shows `dispatchCronDelivery` applies `maybeApplyTtsToCronPayloads`, TTS writes generated audio under the temp root, QQBot local media resolution only trusts OpenClaw/QQBot media roots, `sendQQBotMedia` can return an empty `messageId` for an engine error, and the shared media branch in `src/infra/outbound/deliver.ts` records returned media objects as sent without checking delivery identity. Add focused regression coverage for the failed/no-identity media case and the successful media path.
