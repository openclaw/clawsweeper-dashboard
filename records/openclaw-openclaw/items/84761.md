- **[P1] Scan backups beside each agent models.json:** `src/secrets/storage-scan.ts:82-90`
  - body: `listKnownSecretFileBackups()` only reads files directly under the state directory and `stateDir/agents`, but generated models files live under `agents/<id>/agent/models.json`. A real backup such as `agents/main/agent/models.json.20260501.bak` would be skipped, so the PR misses one of its stated security targets.
  - confidence: 0.9
- **[P1] Guard backup reads before parsing them:** `src/secrets/audit.ts:309`
  - body: The new collector calls `readJsonObjectIfExists(backupPath)` without `requireRegularFile` or a byte limit. Because the matcher accepts broad `.bak/.backup/.old` names, audit can follow symlinks or try to read oversized/non-regular files; use `lstat` plus the existing size/regular-file protections before parsing.
  - confidence: 0.87
- **[P2] Use the config secret target registry for config backups:** `src/secrets/audit.ts:324-358`
  - body: The backup parser only looks for provider `apiKey` fields and tests a non-canonical top-level `providers` shape. Real `openclaw.json` secrets are registered under paths like `models.providers.*.apiKey` plus channel tokens, gateway tokens, headers, and plugin secrets, so this creates a partial competing scanner instead of using the existing audit target contract.
  - confidence: 0.82
