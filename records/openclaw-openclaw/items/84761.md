## What This Changes

The PR adds backup-file scanning to `openclaw secrets audit` for plaintext provider `apiKey` values in legacy config backups.

## Best Possible Solution

Implement backup auditing by deriving candidate sidecars from the same resolved config, auth-store, legacy-auth, and per-agent models paths already used by audit, bound reads to regular JSON files, and require redacted real CLI proof before merge.

## Solution Assessment

No: the requested hardening is useful, but this implementation should reuse existing storage-path discovery and file-safety limits instead of doing a shallow scan of broad backup names.

Use the existing `requireRegularFile` and `maxBytes` protections like the active `models.json` scanner before reading backup candidates.
  - confidence: 0.84
