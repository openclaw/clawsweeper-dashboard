## Summary

Keep open. Current main still has the strict shared Anthropic setup-token validator, and this PR is both stale against the current implementation location and incorrect because it lets too-short legacy-prefixed tokens pass validation.

## What This Changes

The PR lowers Anthropic setup-token validation from 80 to 40 characters in `src/commands/auth-token.ts` and accepts tokens without the `sk-ant-oat01-` prefix.

## Best Possible Solution

Confirm the accepted Anthropic credential formats first. Keep token and API-key modes distinct: handle standard `sk-ant-api03-...` values through the existing API-key path or a clear UX pointer, and only relax setup-token validation in `src/plugins/provider-auth-token.ts` with an unconditional length guard plus runtime/test updates if prefixless setup tokens are confirmed valid.

## Reproduction Assessment

Yes. The PR diff is enough to reproduce the regression by inspection: `sk-ant-oat01-x` starts with the legacy prefix, so the new combined guard skips the length check where current main rejects it as too short.

## Solution Assessment

No. The patch changes a stale implementation location, relaxes validation too broadly, and does not update downstream Anthropic runtime behavior or focused tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Reject too-short prefixed setup tokens:** `src/commands/auth-token.ts:33-38`
  - body: The new guard only checks length when the token lacks `sk-ant-oat01-`, so truncated legacy values such as `sk-ant-oat01-x` would be accepted and stored instead of failing at paste/setup time. Keep prefix relaxation separate from an unconditional minimum-length check.
  - confidence: 0.94

## Security Review

Status: needs_attention

Summary: The diff weakens validation on an auth-token input path by allowing malformed legacy-prefixed Anthropic setup tokens through.

Concerns:

  - body: Because length validation only applies to prefixless tokens, a too-short value with the legacy Anthropic prefix would pass upfront validation and be stored as a credential, making auth failures harder to diagnose and weakening the existing guardrail.
