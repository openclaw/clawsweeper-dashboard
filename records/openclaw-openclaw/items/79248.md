## What This Changes

The PR adds locked refresh-time cleanup for stale `.usage-cost-cache*.tmp` files, a regression test that preserves recent temp files, and a changelog note.

## Best Possible Solution

Land this PR or an equivalent cleanup so stale current and legacy usage-cost cache temp files older than the lock grace period are pruned during refresh while recent in-progress temp files are preserved.

## Reproduction Assessment

Yes. Current main has a cache writer that can leave sibling temp files after hard termination and a refresh path with no reclamation, while the linked issue provides production evidence of accumulated orphaned files.

## Solution Assessment

Yes. Refresh-time best-effort cleanup under the existing cache lock is narrower than startup scanning, avoids changing cache contents or pricing behavior, and matches both the legacy and current temp-file shapes.
