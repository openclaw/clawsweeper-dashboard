## Summary

Keep open: current main and v2026.6.9 still use matrix-js-sdk 41.6.0, and the inspected OpenClaw Matrix verification path does not normalize a methodless m.key.verification.accept before the SDK forwards raw event content into rust crypto.

## What This Changes

The issue reports Matrix SAS verification stalling when Element sends m.key.verification.accept without method and requests tolerant handling or a clear recovery path.

## Best Possible Solution

Keep this open and choose a supported Matrix SDK or plugin event boundary for a narrow compatibility repair, with regression coverage for methodless accept events and clear warning text when the compatibility path is used.

## Reproduction Assessment

Yes at source level, but not with a live Element/Synapse run in this review. matrix-js-sdk 41.6.0 forwards raw verification event content into rust crypto, and the inspected OpenClaw paths do not normalize the methodless accept payload before that handoff.

## Solution Assessment

Yes, the requested direction is broadly right, but the local node_modules hotfix is not the right permanent shape. The best repair is a plugin-owned or upstream-supported compatibility boundary plus regression coverage, not a vendored file edit.
