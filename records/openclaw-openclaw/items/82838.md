## Summary

Keep open: current main and the latest release still treat `$include` as parsed JSON5 config data, while `systemPromptOverride` remains a string-only full prompt replacement. This is a new public config/API feature with prompt-file trust-boundary implications, not broken documented behavior.

## What This Changes

The issue requests resolving a sole-key `$include` object as raw string content for string-typed config fields such as `agents.defaults.systemPromptOverride`.

## Best Possible Solution

Keep this open while maintainers decide the public config contract; if accepted, add an explicit raw-text include or prompt-file contract with schema/types, docs, include guard reuse, and focused regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection confirms current main parses included files as JSON5 config data before schema validation, while `systemPromptOverride` accepts only strings after resolution.

## Solution Assessment

Unclear. The requested syntax is ergonomic, but reusing `$include` for raw text is not obviously the safest contract because the current resolver is schema-agnostic JSON config loading; a separate text include or prompt-file field may be cleaner if maintainers approve the capability.
