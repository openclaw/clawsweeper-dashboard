## Summary

Keep open: current main and the latest release treat `$include` as parsed JSON5 config data, not raw file text, and `systemPromptOverride` remains a string-only full prompt override. This is a new public config/API feature with prompt-file trust implications, not a conservative cleanup close.

## What This Changes

The issue requests raw-text `$include` support for string-valued config fields such as `systemPromptOverride`.

## Best Possible Solution

Keep this open for a maintainer-owned config/API decision; if accepted, add an explicit raw-text prompt include contract with schema/types, docs, include guard reuse, and regression tests.

## Reproduction Assessment

Not applicable as a bug reproduction. Current-main source shows `$include` parses target files as JSON5 before validation, and the requested raw `.md` text form is a new config contract rather than a failing documented path.

## Solution Assessment

Unclear. Reusing `$include` for raw text is ergonomic, but it would overload a schema-agnostic JSON5 include directive; maintainers should choose between explicit raw-text include syntax and a prompt-file field before implementation.
