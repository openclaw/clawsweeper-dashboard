## Summary

Keep open: current main and v2026.5.12 still treat `$include` targets as parsed JSON5 config data, while `systemPromptOverride` remains a string-only full-prompt replacement. This is a new public config/API feature with prompt-file trust-boundary decisions, not a conservative cleanup close.

## What This Changes

The issue requests treating a sole-key `$include` object assigned to a string config field as raw file contents, especially for `systemPromptOverride`.

## Best Possible Solution

Keep this open for a maintainer-approved raw-text prompt include contract; if accepted, implement schema/types/docs, include-guard reuse, clear validation errors, and prompt regression tests without weakening current JSON5 include safety.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection confirms current main resolves `$include` by parsing target files as JSON5 before schema validation, so a raw `.md` prompt file is a new config contract rather than broken documented behavior.

## Solution Assessment

Unclear. Reusing `$include` for raw text is ergonomic, but maintainers should decide whether to overload the schema-agnostic JSON5 include directive or add an explicit raw-text/prompt-file contract.
