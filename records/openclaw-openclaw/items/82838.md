## Summary

Keep open: current main and v2026.6.6 still treat `$include` targets as parsed JSON5 config, not raw text, and the related `$includeText` implementation remains open with maintainer API/security questions.

## What This Changes

Request support for file-backed raw string config values so long prompt fields can live in external Markdown/prose files instead of inline JSON strings.

## Best Possible Solution

Have maintainers choose and land one guarded raw-text config include contract, likely an explicit `$includeText`-style directive, with recovery/write-back coverage, docs, and tests before closing this request.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a new config feature request. Source and release inspection give high-confidence proof that current `$include` parses included files as JSON5 rather than resolving raw text.

## Solution Assessment

Unclear until maintainers accept the public config API shape. The related `$includeText` direction is safer than overloading `$include`, but it still needs maintainer approval for compatibility and prompt-file security semantics.
