## Best Possible Solution

Have maintainers settle and land one guarded raw-text config include contract, likely an explicit `$includeText` directive, with include guards, recovery/write-back coverage, docs, and tests before closing this request.

## Solution Assessment

Unclear until maintainers accept the public config API and security boundary. The related `$includeText` direction looks safer than overloading `$include`, but it still needs maintainer approval and repair before this issue can close.
