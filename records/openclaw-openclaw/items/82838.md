## Summary

Keep open: current main treats `$include` as a JSON5 config include resolved before schema validation, while `systemPromptOverride` is a string-only full prompt replacement. The requested raw text include behavior is a new public config/API feature, not broken documented behavior, and it needs maintainer product/security judgment before implementation.

## What This Changes

The issue requests resolving `{ "$include": "./file.md" }` as raw file contents for string-valued config fields such as `agents.defaults.systemPromptOverride`.

## Best Possible Solution

Keep the issue open while maintainers decide the public config contract; if accepted, implement an explicit raw-text include or prompt-file string contract with schema/types, docs, include security guard reuse, and focused regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection confirms current main's documented `$include` path parses included files as JSON5 config data and `systemPromptOverride` only accepts a string after resolution.

## Solution Assessment

Unclear. The requested syntax is ergonomic, but reusing `$include` for raw text is not obviously the safest contract because the current resolver is schema-agnostic JSON5 config loading; a separate explicit text include or prompt-file field may be cleaner if maintainers approve the capability.

- The feature overlaps existing prompt customization discussions, so field shape and precedence should be coordinated rather than patched only for one config key.
