## Summary

Keep open. Current main still treats `$include` as a JSON5 config-fragment resolver rather than raw text inclusion, and I did not find a canonical issue or PR that already owns the raw string include request. Because this changes the public config include contract, it needs maintainer product/API judgment before implementation.

## What This Changes

Request support for using a `$include` object to populate string-valued config fields such as `agents.defaults.systemPromptOverride` from raw file contents.

## Best Possible Solution

Define a deliberate text-include config contract, preserve the existing include file guards, and add focused docs plus regression coverage for `systemPromptOverride` and any other approved string fields.

## Reproduction Assessment

Yes, from source inspection. A config using `agents.defaults.systemPromptOverride: { "$include": "./system-prompt.md" }` will route the file through `JSON5.parse` before validation, so ordinary Markdown/prose does not become a string; I did not run a live config load.

## Solution Assessment

Unclear. The requested behavior solves a real ergonomics problem, but the best implementation shape needs maintainer agreement because the existing `$include` contract is structured JSON5 config inclusion, not typed raw text inclusion.
