## Summary

Keep open: current main and v2026.6.10 still treat `$include` files as JSON/JSON5 config fragments, not raw string values, and the active implementation candidate remains open with API/security review and merge conflicts outstanding.

## What This Changes

Request file-backed raw string config values, especially prompt-like fields, by resolving an include directive to text instead of JSON/JSON5 objects.

## Best Possible Solution

Land one maintainer-approved guarded raw-text include contract, likely an explicit `$includeText`, integrated with include scanning, recovery/write-back invariants, docs, validation, and prompt-data trust boundaries.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a new config feature request. Source, docs, and release inspection show current includes are JSON/JSON5 config fragments only and no raw-text directive exists.

## Solution Assessment

Unclear until maintainers accept the public config API and security boundary. A separate `$includeText` directive looks safer than context-sensitive overloading of `$include`, but it must cover scanning, recovery, write-back, docs, and trust-boundary behavior.
