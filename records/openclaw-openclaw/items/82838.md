## Summary

Keep open: current main and the latest release still treat `$include` files as JSON5 config data before schema validation, while `systemPromptOverride` remains a string-only replacement field. This is a new public config/API contract with prompt-file trust implications, and no merged implementation or exact canonical issue fully supersedes it.

## What This Changes

Request support for using `$include` to load raw file text into string config fields such as `agents.defaults.systemPromptOverride`.

## Best Possible Solution

Keep this open until maintainers choose the public config shape; if accepted, implement an explicit raw-text include or prompt-file contract with schema/types, docs, include guard reuse, and regression coverage for defaults and per-agent overrides.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection confirms current main parses include targets as JSON5 before validation and validates `systemPromptOverride` as a string, so the requested raw `.md` file form is a new config contract rather than a failing documented path.

## Solution Assessment

Unclear. Reusing `$include` is ergonomic, but maintainers should decide whether raw text belongs in a typed include syntax, a dedicated prompt-file field, or the broader prompt-injection contract before implementation.
