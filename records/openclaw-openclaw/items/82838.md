## Summary

Keep open: current main still treats `$include` as parsed JSON5 config data and `systemPromptOverride` as a string-valued full prompt override, so raw text inclusion for prompt files is a new public config feature that needs maintainer product/security judgment.

## What This Changes

The issue requests allowing `$include` objects assigned to string config fields, especially `agents.defaults.systemPromptOverride`, to resolve to raw file contents.

## Best Possible Solution

Keep the issue open for a maintainer decision on the public config contract; if accepted, implement an explicit raw-text include or prompt-file field with schema/types, docs, include guard reuse, and focused regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection confirms current main resolves `$include` as parsed JSON5 config data before validating `systemPromptOverride` as a string field.

## Solution Assessment

Unclear. The requested syntax is ergonomic, but maintainers should first decide whether `$include` should gain raw-text semantics or whether a separate prompt-file/text-include contract is safer.
