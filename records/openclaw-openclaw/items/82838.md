## Summary

Keep open: current main and the latest release still treat `$include` as parsed JSON5 config data, while `systemPromptOverride` remains a string-only full-prompt replacement. The requested raw text include form is a new public config/API contract with prompt-file trust implications, not a conservative cleanup close.

## What This Changes

Request support for resolving `$include` objects as raw string contents for string-valued config fields such as `agents.defaults.systemPromptOverride`.

## Best Possible Solution

Keep this open for a maintainer-owned config/API decision; if accepted, add an explicit raw-text prompt include or prompt-file contract with schema/types, docs, include guard reuse, and regression tests.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection verifies the requested raw `.md` include behavior is a new config contract rather than a failing documented path.

## Solution Assessment

Unclear. Reusing `$include` is ergonomic, but an explicit raw-text include syntax or prompt-file field would avoid overloading the existing schema-agnostic JSON5 include directive.
