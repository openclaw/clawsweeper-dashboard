## Summary

Keep open. The branch adds a new security-facing plugin trust-tier surface that is not on current main, but the diff still under-reports missing-format native plugins as content-tier and lacks required real behavior proof; maintainers should decide the taxonomy before merge.

## What This Changes

The PR adds plugin trust-tier types and resolution, stores the tier on plugin records, surfaces it in plugin CLI output, and documents content/sandboxed/native tiers.

## Best Possible Solution

Land a rebased, maintainer-approved version that derives tiers from the effective plugin format, documents only the current trust boundaries plus clearly reserved future behavior, and covers native, bundle, missing-format, list, and info/inspect behavior.

## Reproduction Assessment

Yes for the PR-introduced mismatch: source inspection shows current main normalizes missing format to openclaw while the PR resolver returns content for undefined format. The overall trust-tier taxonomy is feature work, so bug reproduction is otherwise not applicable.

## Solution Assessment

No. The direction may be useful, but this branch is not the best merge path until it computes from the normalized effective format, rebases onto the current loader/registry layout, and maintainers accept the security-facing taxonomy.
