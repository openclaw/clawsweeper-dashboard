## Summary

Keep open: current main does not implement trust tiers, and the PR still has a source-reproducible correctness/security issue in how it derives the new trust metadata.

## What This Changes

The PR adds plugin trust-tier types, resolver logic, registry metadata, CLI output, tests, and docs for content, sandboxed, and native plugin classifications.

## Best Possible Solution

If maintainers want this taxonomy, land a rebased implementation that derives tiers from normalized record metadata, defines the public CLI/JSON/status contract, and documents reserved sandbox behavior as non-current.

## Reproduction Assessment

Yes for the PR-introduced issue: source inspection shows current records normalize missing format to openclaw while the PR computes trust from raw manifestRecord.format. That is enough to reproduce the metadata mismatch without running the branch.

## Solution Assessment

No. The direction may be useful, but the branch is not the best merge path until security-facing metadata uses the same effective format the loader stores and maintainers accept the public trust taxonomy.
