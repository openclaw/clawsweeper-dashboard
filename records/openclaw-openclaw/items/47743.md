## Summary

Keep open: current main still has no plugin trust-tier model, and this PR remains a security-facing feature branch with a concrete metadata mismatch and no real CLI behavior proof.

## What This Changes

This PR adds plugin trust-tier types and resolution, stores the tier on plugin records, prints it in plugin CLI output, and documents/tests content, sandboxed, and native tiers.

## Best Possible Solution

Land a rebased version only if maintainers want this public taxonomy, with tiers derived from the effective plugin format and tests/docs covering missing-format, native, bundle, list, and inspect output.

## Reproduction Assessment

Yes for the PR-introduced issue: source inspection shows current records normalize a missing format to `openclaw`, while the PR computes the new trust tier from raw `manifestRecord.format`. That is enough to reproduce the metadata mismatch from source without running the branch.

## Solution Assessment

No. The taxonomy may be useful, but this branch is not the best merge path until security-facing metadata is computed from the same effective format that the loader stores and the CLI displays.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:
