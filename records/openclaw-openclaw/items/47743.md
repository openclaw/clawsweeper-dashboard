## Summary

Keep open. Current main still lacks plugin trust-tier metadata, so the feature is not obsolete, but the PR should not merge as-is because it can under-report native-capable missing-format plugins as content-tier and still needs contributor real behavior proof plus maintainer product/security review.

## What This Changes

This PR adds plugin trust-tier types and resolution, stores the tier on plugin records, prints it in plugin list/info output, and adds docs/tests for content, sandboxed, and native tiers.

## Best Possible Solution

Land a rebased implementation only if maintainers want this public taxonomy, with tiers derived from normalized record metadata and tests/docs covering native, bundle, missing-format fallback, list output, and inspect output.

## Reproduction Assessment

Yes for the PR-introduced issue: source inspection shows current records normalize missing format to openclaw while the PR computes trust from raw manifestRecord.format. That is enough to reproduce the metadata mismatch without running the branch.

## Solution Assessment

No. The direction may be useful, but the branch is not the best merge path until security-facing metadata uses the same effective format the loader stores and maintainers accept the public trust taxonomy.
