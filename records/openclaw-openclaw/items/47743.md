## What This Changes

The PR adds plugin trust-tier types and resolution, stores the tier on plugin records, prints it in plugin CLI output, and documents/tests content, sandboxed, and native tiers.

## Best Possible Solution

If maintainers want this taxonomy, land a rebased implementation that derives tiers from normalized record metadata, documents only current capabilities plus clearly reserved future behavior, and covers native, bundle, missing-format fallback, list output, and info/inspect output in tests.
