## What This Changes

The PR adds plugin trust-tier types and resolution, stores the tier on plugin records, prints it in plugin CLI output, and adds docs plus unit tests.

## Best Possible Solution

If maintainers want this taxonomy, land a rebased implementation that derives tiers from the effective normalized plugin format, defines the supported CLI/JSON/status contract, documents only current behavior plus clearly reserved future behavior, and proves native, bundle, missing-format, list, and inspect/info paths.
