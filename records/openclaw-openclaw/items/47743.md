## Summary

This PR should stay open: current `main` still has no explicit plugin trust-tier model, and the branch is useful but has a concrete correctness bug in how it labels missing-format plugins. The earlier review discussion identified the same mismatch, and current loader behavior confirms it would under-report native-capable plugins as content-tier if merged as-is.

## What This Changes

This PR adds a plugin trust-tier type and resolver, stores the tier on plugin records, exposes it in plugin CLI output, and documents the content/sandboxed/native trust model.

## Best Possible Solution

Land a rebased version only if maintainers want this taxonomy, with trust tiers computed from the normalized effective plugin format and tests covering bundle, native, missing-format fallback, CLI list/info output, and current-versus-reserved behavior in docs.

## Reproduction Assessment

Yes for the PR defect: source inspection shows missing-format plugins are normalized to `openclaw` in `createPluginRecord`, while this branch computes `trustTier` from the raw possibly undefined manifest format. I did not run tests because this review was required to keep the checkout read-only.

## Solution Assessment

No. The requested trust-tier model may be useful, but the branch should first compute from the same effective format stored on the plugin record and get maintainer sign-off on the public security taxonomy.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Resolve trust tiers from the normalized plugin format:** `src/plugins/loader.ts:849-853`
  - body: `createPluginRecord` stores missing formats as `openclaw`, but this new call passes the raw `manifestRecord.format` into `resolveTrustTier`, whose fallback is `content`. A manifest without an explicit format would therefore be displayed as `format: openclaw` with `trustTier: content`, under-reporting a native-capable plugin in CLI/security metadata. Use the same effective format at both loader call sites and align the undefined-format docs/tests with that behavior.
  - confidence: 0.93
