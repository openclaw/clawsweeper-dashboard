## Summary

Keep this PR open, but it should not merge as-is: current main still lacks a trust-tier model, and the branch introduces a security-facing metadata mismatch by resolving trust from the raw manifest format instead of the effective plugin record format. The PR also still lacks after-patch real CLI behavior proof.

## What This Changes

This PR adds plugin trust-tier types, resolver logic, loader/registry wiring, CLI output, docs, and tests for content, sandboxed, and native plugin tiers.

## Best Possible Solution

If maintainers want this taxonomy, land a rebased implementation that derives trust from normalized record metadata, documents only current capabilities plus reserved future behavior, and covers native, bundle, missing-format fallback, list output, and info/inspect output.

## Reproduction Assessment

Yes for the PR-introduced mismatch: source inspection shows the branch computes trust from raw manifestRecord.format while current main normalizes missing format to openclaw. The trust-tier feature itself is not a bug reproduction request.

## Solution Assessment

No. The direction may be useful, but this branch is not the best merge path until it resolves tiers from the effective plugin format, is rebased, and maintainers accept the security-facing public taxonomy.
