## Summary

Keep open. Current main still lacks this trust-tier model, but this PR is not merge-ready because it computes security-facing trust metadata from the raw manifest format while records normalize missing formats to `openclaw`, and it lacks after-fix real CLI proof.

## What This Changes

This PR adds plugin trust-tier types, resolution, docs, tests, record metadata, and plugin CLI output for content/sandboxed/native plugin tiers.

## Best Possible Solution

Land a rebased, maintainer-approved taxonomy that computes tiers from the normalized effective plugin format, updates the current loader/registry/CLI/docs surfaces, and proves native, bundle, and missing-format cases with CLI output.

## Reproduction Assessment

Not applicable: this is a feature PR adding public metadata and docs, not a bug report. Source review shows current main lacks the proposed trust-tier field/output, while the PR diff introduces a metadata mismatch.

## Solution Assessment

No. The direction may be useful, but this patch under-reports missing-format native plugin records as `content`; the safer solution is to rebase and compute from the same normalized format stored on `PluginRecord`.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:
