## Summary

Keep open. The security intent is valid, but current main already hardened the local attachment read path with `openLocalFileSafely`/`readLocalBuffer`; this PR needs an update against that implementation plus real behavior proof before merge.

## What This Changes

The PR replaces the attachment cache local read with fs-safe reading, maps oversized safe-read errors to `MediaUnderstandingSkipError`, and writes generated temp media files with mode `0o600`.

## Best Possible Solution

Keep the current safe-open attachment cache design, and if maintainers still want the remaining hardening, add a focused `0o600` temp-file-mode update with coverage and real behavior proof.

## Reproduction Assessment
