## Summary

Keep this PR open for maintainer/contributor follow-up rather than cleanup-close: current main already implements the attachment read hardening in a newer fd-first shape, but this branch still contains a small unique temp-file permission hardening and is not old enough for the mostly-implemented stale-PR close path.

## What This Changes

The PR routes MediaAttachmentCache local reads through readLocalFileSafely, maps oversized safe-open errors to MediaUnderstandingSkipError, canonicalizes the resolved path, and writes generated temp files with mode 0o600.

## Best Possible Solution

Keep the shipped fd-first attachment read hardening, then either land a tiny current-main patch for explicit temp-file mode with proof or close this PR after maintainers decide the private temp root is sufficient.

## Reproduction Assessment
