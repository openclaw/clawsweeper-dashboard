## Summary

Keep open. Current main still does not implement the full hardening requested here: local attachment reads use the cache-local helper rather than the shared identity-checked safe-read path, generated temp files still omit an explicit mode, and GitHub reports the source branch as dirty against current main; the prior oversized-file review concern has been addressed in the PR diff.

## What This Changes

The PR replaces attachment-cache local reads with `readLocalFileSafely`, maps oversized `SafeOpenError` to `MediaUnderstandingSkipError`, and writes generated media temp files with `0o600` permissions.

## Best Possible Solution

Port the hardening onto current main by using the shared identity-checked safe-read/open path or an equivalent helper, preserving current root checks and fallback semantics, adding `0o600` temp writes, and covering the behavior with focused media tests.
