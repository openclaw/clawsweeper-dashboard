## Summary

Keep this PR open: the hardening direction is still useful, but current main has diverged from the raw-read implementation the PR edits, so the change needs to be ported into the current attachment-cache helper rather than closed or merged as-is.

## What This Changes

The PR changes the media attachment cache to read local attachments through `readLocalFileSafely`, map oversized safe-open errors back to media skip errors, and write generated temp files with mode `0o600`.

## Best Possible Solution

Port the hardening onto current main by reusing the shared identity-checked safe-read/open path or equivalent logic inside the current attachment-cache local-read flow, while preserving canonical-root validation, URL fallback, and `MediaUnderstandingSkipError("maxBytes")` semantics.

## Reproduction Assessment

Yes for the hardening gap by source inspection. Current main still routes local attachment reads through `readLocalBuffer` and writes generated temp files without an explicit mode.

## Solution Assessment
