## Summary

Keep open. Current main already supersedes the safe-read part with handle-based `openLocalFileSafely` reads, but this PR still has a meaningful temp-file `0o600` hardening that is not on main, and it needs rebase plus real behavior proof before merge.

## What This Changes

The PR replaces the attachment cache's raw local file read with fs-safe reading, maps oversized safe-open errors to media skip errors, and writes temp copies with mode `0o600`.

## Best Possible Solution

Rebase or replace the PR against current main by preserving only the remaining temp-file permission hardening while keeping the existing `openLocalFileSafely`/`readLocalBuffer` implementation and tests.

## Reproduction Assessment

## Solution Assessment
