## What This Changes

The issue requests concatenating Anthropic transport `signature_delta` chunks and aligning thinking display handling so persisted signed thinking can replay safely.

## Best Possible Solution

Patch the Anthropic transport stream to accumulate all `signature_delta` chunks, add a multi-delta regression test, and align request-level thinking display behavior with the legacy provider without inventing a persisted content-block field.

Yes from source, but not from a live run in this read-only review. Current main assigns each `signature_delta` over the previous value, while the sibling provider appends chunks and the latest release still contains the overwrite.

## Solution Assessment

Yes for the core fix: appending signature deltas in the transport stream is the narrowest maintainable repair. The display part should be handled as request-level `thinking.display` alignment, not as a new persisted thinking-block property unless maintainers confirm that contract.
