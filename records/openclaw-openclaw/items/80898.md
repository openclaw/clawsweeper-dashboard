## Summary

Keep open. This protected, maintainer-labeled iMessage access-control PR addresses a source-provable sender-allowlist issue, but the latest head still leaves public config/migration docs inconsistent and lacks required real behavior proof.

## What This Changes

The PR changes iMessage and shared chat sender matching so sender allowlists can opt out of conversation-target matches, preserves group/reply-context chat-target uses, updates iMessage setup/docs, and adds regression tests.

## Best Possible Solution

Land a maintainer-approved hardening that keeps sender authorization tied to handles or sender access groups, keeps chat targets only for routing/group admission/reply context, updates every public config and migration reference, and includes redacted runtime proof.

## Reproduction Assessment

Yes, for a source-level path: current main's shared matcher and iMessage target test show `chat_id` in `allowFrom` authorizes a sender when chat context is present. I did not establish a live iMessage reproduction path in this read-only review.

## Solution Assessment

No as submitted. The code direction is narrow, but the remaining public docs and migration guidance are inconsistent with the new contract and the external PR still needs real after-fix behavior proof before merge.
