## Summary

Keep open: this PR addresses a real startup-auth bug that current main still does not solve, but the branch is not mergeable as-is because it targets the deleted startup module, uses a narrower auth check than runtime, has misleading remediation text, and lacks real behavior proof.

## What This Changes

The PR adds a gateway startup guard that throws before channel startup when the resolved default model provider has no configured auth, plus focused unit tests.

## Best Possible Solution

Port a runtime-aligned auth warning or fail-fast guard into the current post-attach startup path, using the same auth contract as model execution and covering skip flags plus valid non-profile auth modes.

## Reproduction Assessment

Yes. Source inspection shows current main starts channels without a default-provider auth gate, and the linked bug has concrete fresh-profile logs and a working model-switch workaround; I did not run a live gateway in this read-only review.

## Solution Assessment

No. The fail-fast direction is plausible, but this patch is not the best fix until it is moved to the current startup path, uses runtime-equivalent provider auth, and avoids circular remediation text.
