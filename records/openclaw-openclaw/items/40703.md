## Summary

Keep open. Current main still lacks the intended default-model auth gate before channel startup, but this PR is not merge-ready because it patches the old monolithic startup surface and uses a narrower auth helper that can reject valid runtime auth configurations.

## What This Changes

The PR adds a gateway startup guard and focused tests that throw before channel startup when the resolved default model provider appears to have no configured auth.

## Best Possible Solution

Implement the guard in the current post-attach sidecar startup path immediately before channel start, honor skip flags, reuse `hasAuthForModelProvider` with cfg/workspace/env/agent-dir context, avoid circular remediation text, and cover the supported auth modes.

## Reproduction Assessment

Yes for the PR defects. Static inspection shows valid runtime auth modes that the PR helper misses, and the hardcoded remediation reproduces whenever the failing default is already the suggested model.

## Solution Assessment

No. The target behavior is plausible, but this patch is not the best solution because it is based on the old startup surface and does not reuse the runtime-aligned provider auth checker exposed on current main.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Use runtime auth availability before blocking startup:** `src/gateway/server-startup.ts:46-49`
  - body: The guard checks `hasAuthForProvider` with only `provider` and `agentDir`. That helper only sees env keys and auth profiles, while runtime accepts config/workspace keys, AWS SDK Bedrock auth, local synthetic auth, and plugin synthetic auth, so valid gateways can be rejected before channels start.
  - confidence: 0.9
- **[P3] Avoid suggesting the failing default model:** `src/gateway/server-startup.ts:58`
  - body: The error text always suggests `openclaw models set "openai-codex/gpt-5.4"`. When that is already the default model whose provider lacks auth, the remediation points users back to the same broken configuration.
  - confidence: 0.82
