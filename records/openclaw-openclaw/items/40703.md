## Summary

Keep open: current main still does not implement the stricter fail-fast startup behavior, but this PR should not merge as-is because its auth check is narrower than runtime auth resolution and the external contributor has not provided after-fix real behavior proof.

## What This Changes

The PR adds a gateway startup guard and tests that throw before channel startup when the resolved default model provider has no configured auth.

## Best Possible Solution

Add the guard in the current post-attach startup path, use runtime-equivalent provider auth resolution, avoid misleading remediation text, and include focused regression coverage plus redacted real startup proof.

## Reproduction Assessment

Yes. Source inspection shows current main starts channels after scheduling non-blocking model prewarm without a default-provider auth gate, and the PR's false-positive path is visible from its narrower helper call.

## Solution Assessment

No. The fail-fast direction is plausible, but the patch should be ported to src/gateway/server-startup-post-attach.ts and use hasAuthForModelProvider or an equivalent runtime-aligned check before blocking startup.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Use runtime-equivalent auth before blocking startup:** `src/gateway/server-startup.ts:46-50`
  - body: This guard calls hasAuthForProvider with only provider and agentDir, but runtime auth also accepts config-backed provider keys, AWS SDK auth, local synthetic auth, plugin synthetic auth, and workspace-aware env lookup. Valid configurations can now fail before channels start, so the startup gate needs the same availability contract as model runtime auth.
  - confidence: 0.9
- **[P3] Avoid suggesting the same failing model:** `src/gateway/server-startup.ts:61`
  - body: The error always suggests openclaw models set "openai-codex/gpt-5.4". If that model is already the default whose provider lacks auth, the remediation points users back to the failing configuration instead of an actionable next step.
  - confidence: 0.84
