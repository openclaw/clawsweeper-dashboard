## Summary

Keep open: the linked startup-auth problem still appears valid on current main, but this PR is not merge-ready because it targets an obsolete startup module, uses narrower auth resolution than runtime, and lacks required real behavior proof.

## What This Changes

The PR adds a gateway startup guard and tests that throw before channel startup when the resolved default model provider lacks configured auth.

## Best Possible Solution

Implement auth visibility in the active post-attach startup path with runtime-equivalent provider auth resolution, provider-specific remediation, skip-flag handling, and coverage for env, config, profile, AWS, local synthetic, and plugin synthetic auth.

## Reproduction Assessment

Yes, from source and the linked bug path. Current main still reaches channel startup without a default-model auth guard or startup warning, though I did not run a live gateway in this read-only review.

## Solution Assessment

No. The direction is plausible, but this patch is not the best current fix because it targets a removed startup module, uses narrower auth resolution than runtime, and hardcodes circular remediation text.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Use runtime-equivalent auth before blocking startup:** `src/gateway/server-startup.ts:49-52`
  - body: This gate calls hasAuthForProvider with only provider and agentDir, but runtime auth also accepts config-backed provider keys, workspace-aware env lookup, AWS/Bedrock auth, and synthetic local/plugin auth. A fail-fast startup guard using this narrower helper can reject configurations that would work at request time.
  - confidence: 0.93
- **[P2] Port the guard to the active startup path:** `src/gateway/server-startup.ts:92-104`
  - body: Current main no longer has this startup module; channel startup now runs through src/gateway/server-startup-post-attach.ts. The guard needs to be rebased and ported before it can affect real gateway startup.
  - confidence: 0.92
- **[P3] Avoid suggesting the same failing model:** `src/gateway/server-startup.ts:61`
  - body: The error always suggests openclaw models set "openai-codex/gpt-5.4". When that is already the failing default model, the remediation points back to the same broken setup instead of giving an actionable alternative.
  - confidence: 0.87
