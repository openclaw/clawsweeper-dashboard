## Summary

Keep open. Current main does not implement this PR's fail-fast default-model auth guard, and the proposed guard is not safe to merge because it uses a narrower auth helper than runtime model execution, which would reject valid config-backed, AWS SDK, or synthetic local provider auth setups.

## What This Changes

This PR adds a gateway startup guard and tests to fail before channel startup when the resolved default model provider lacks configured auth.

## Best Possible Solution

Rebase the guard onto the current sidecar startup code and reuse the runtime-compatible auth availability contract, while keeping the clear actionable error and regression tests.

## Reproduction Assessment

Yes. From source inspection, applying this PR and starting with a default provider whose auth is configured only through `models.providers.<id>.apiKey`, Bedrock `aws-sdk`, or synthetic local provider auth would trip the new guard even though runtime auth resolution accepts those configurations.

## Solution Assessment

No. The PR addresses the right user-visible failure mode, but the implementation should use the same auth availability semantics as runtime model calls and avoid the hardcoded circular model suggestion.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Use runtime-compatible auth resolution before failing startup:** `src/gateway/server-startup.ts:48-51`
  - body: The new guard calls `hasAuthForProvider` with only `provider` and `agentDir`, but that helper only checks env vars and auth profiles. Runtime model calls also accept config-backed API keys, Bedrock/AWS SDK auth, and synthetic local/provider auth, so this would fail startup for valid configurations before channels can start.
  - confidence: 0.94
- **[P3] Avoid suggesting the same failing default model:** `src/gateway/server-startup.ts:61`
  - body: The error message hardcodes `openai-codex/gpt-5.4` as the example replacement. When that is already the failing default, the message tells the user to switch to the same model that just failed, which makes the startup error misleading.
  - confidence: 0.88
