## Summary

Keep open: current main still starts channel sidecars without a default-model auth fail-fast guard, and this PR is useful source work but is not merge-ready because its guard is narrower than runtime auth resolution and targets the old startup file shape.

## What This Changes

The branch adds a gateway startup helper and tests intended to fail before channel startup when the resolved default model provider lacks configured auth.

## Best Possible Solution

Port the guard into the current post-attach sidecar startup path, use the runtime-equivalent provider auth contract, preserve valid config-backed and synthetic auth modes, add focused regression coverage, and include the required changelog entry.

## Reproduction Assessment

Yes. Source inspection shows current main starts channels after only scheduling non-blocking model warmup, and the PR guard can falsely reject providers whose auth is accepted by runtime auth resolution.

## Solution Assessment

No. The fail-fast direction is appropriate, but this patch should be ported to current startup code and use hasAuthForModelProvider or an equivalent runtime-aligned check instead of the narrower tool helper.
