## Summary

Close as ClawHub: this is a useful optional Telegram output-normalization feature, but current main already exposes plugin hooks that can rewrite outbound content before delivery, so it does not need to become a bundled Telegram/core option.

## What This Changes

The issue requests an optional Telegram-safe outbound text sanitizer that converts common LaTeX syntax to Unicode/plain text before Telegram delivery.

## Best Possible Solution

Publish a ClawHub code plugin that scopes the existing message_sending hook to Telegram and rewrites LaTeX-like output before OpenClaw's Telegram renderer sends it.

## Reproduction Assessment

Not applicable: this is a feature request for optional formatting behavior, not a reported regression with a defined failing contract. Current source inspection shows Markdown/HTML Telegram formatting but no LaTeX cleanup contract to reproduce against.

## Solution Assessment

Yes, closing toward ClawHub is the best repo-maintenance outcome: the existing plugin hook API can implement this without a new bundled Telegram setting. Adding a core or bundled telegram_safe_text option would require product choices about opt-in defaults, conversion scope, and false positives.
