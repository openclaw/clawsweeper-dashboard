## Summary

Close as ClawHub-scoped: this is an optional Telegram-specific outbound text transformer, current main and v2026.5.7 already expose the plugin hook needed to rewrite Telegram content before send, and a community plugin has already been published for this exact LaTeX-to-Unicode path.

## What This Changes

Add an optional Telegram-safe sanitizer that converts common LaTeX syntax to Unicode/plain text before Telegram delivery.

## Best Possible Solution

Keep the sanitizer as a ClawHub/OpenClaw plugin using message_sending filtered to channelId "telegram", and only return to core if that path exposes a concrete missing API.

## Reproduction Assessment

Not applicable; this is a feature request for optional output normalization, not a report of broken documented Telegram behavior. Source and docs inspection show the relevant outbound hook already exists for a plugin implementation.

## Solution Assessment

Yes for this cleanup decision: the existing plugin hook is the narrowest maintainable solution. Adding a built-in Telegram LaTeX sanitizer would add channel-specific formatting policy and likely config surface to core.
