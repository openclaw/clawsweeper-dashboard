## Summary

Close as ClawHub-scoped: this is an optional Telegram-specific outbound text transformer, and current main already exposes a plugin message_sending hook that can rewrite Telegram content before delivery without adding a core feature or new SDK seam.

## What This Changes

The issue requests an optional Telegram output transformer that converts common LaTeX/math markup into readable Unicode or plain text before Telegram delivery.

## Best Possible Solution

Publish a ClawHub code plugin that registers message_sending, filters for channelId "telegram", rewrites configured LaTeX patterns to Unicode/plain text, and only return to core if that plugin path exposes a concrete missing API.

## Reproduction Assessment

Not applicable; this is a feature request, not a report of broken documented Telegram behavior. Source inspection confirms current main formats Telegram replies as Markdown-to-HTML with fallback, while plugin hooks can already rewrite outbound content.

## Solution Assessment

Yes for this cleanup decision: a ClawHub plugin using the existing message_sending hook is the narrowest maintainable path. Adding a bundled core Telegram sanitizer would introduce optional formatting policy and likely config surface that the current plugin boundary is meant to avoid.
