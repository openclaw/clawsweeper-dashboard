## Summary

Keep open. Current main and the latest release still expose generic/write-only config mutation feedback, while the related schema lookup and restart-sentinel wording work do not provide the requested post-write apply/reload status contract.

## What This Changes

Request a deterministic post-config-write status contract for CLI and gateway RPC responses, including write result, reload plan/outcome, restart requirement, and runtime verification limits.

## Best Possible Solution

Define one shared config-write acknowledgement contract for CLI and gateway RPC that reports write result, changed paths, reload plan, actual apply outcome, restart requirement, and verification limits.

## Reproduction Assessment

Yes. Source inspection on current main shows the CLI still emits generic restart guidance and gateway config writes return write/sentinel metadata rather than deterministic apply status.

## Solution Assessment

Unclear. The goal is valid, but the exact public response fields and synchronous versus deferred verification semantics need maintainer agreement before a narrow implementation is safe.
