## Summary

Keep open. Current main and the latest release still do not expose the requested deterministic post-write apply/reload contract; the merged `reloadKind` work is only preflight lookup metadata, and the open related PR targets narrower CLI wording rather than gateway RPC apply status.

## What This Changes

The issue asks for `openclaw config patch`, `config set`, and gateway `config.patch`/`config.apply` to return deterministic post-write reload/apply status.

## Best Possible Solution

Define one shared config-write acknowledgement contract for CLI and gateway RPC that reports write result, changed paths, reload plan, apply outcome, restart requirement, and verification limits while keeping `config.schema.lookup` as preflight metadata.

## Reproduction Assessment

Yes. Source inspection on current main shows gateway config write responses lack the requested changed-path/reload/apply/effective-runtime fields, and CLI config writes still emit generic restart guidance.

## Solution Assessment

No proposed patch is under review. The best path is a maintainer-approved gateway/CLI response contract that extends the existing write flow, not closing this as covered by the preflight `reloadKind` lookup or the narrower CLI-hint PR.
