## Summary

Keep open. Current main has the related pre-write `reloadKind` lookup work, but the post-write `config.set`/`config.patch`/CLI acknowledgement still does not expose a deterministic apply result for write status, hot reload completion, reload failure, or remaining restart requirement.

## What This Changes

Request deterministic machine-readable and human-readable post-write config apply status for `openclaw config patch`, `config set`, and gateway config write RPCs.

## Best Possible Solution

Define one shared config-write acknowledgement contract for CLI and gateway RPC that reports write result, changed paths, reload plan, actual apply outcome, restart requirement, and verification limits while keeping `config.schema.lookup` as preflight metadata.

## Reproduction Assessment

Yes, from source inspection: current main’s gateway write handlers and CLI write path still expose generic restart/sentinel feedback rather than deterministic post-apply status. I did not run a live write because this read-only review can confirm the missing contract from the source paths.

## Solution Assessment

No patch is under review. The best path is a maintainer-approved shared API/CLI acknowledgement contract, not closing this as covered by the preflight `reloadKind` lookup from https://github.com/openclaw/openclaw/pull/81612.
