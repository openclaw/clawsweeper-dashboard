## Summary

Keep open. Current main has the linked preflight `reloadKind` lookup work, but the post-write acknowledgement requested here is still not a deterministic apply/reload contract for either gateway RPC writes or direct CLI config writes.

## What This Changes

Request deterministic machine-readable and human-readable post-write feedback for `openclaw config patch`, `config set`, and gateway `config.patch`/`config.apply`.

## Best Possible Solution

Define one shared config-write acknowledgement contract for CLI and gateway RPC that reports write result, changed paths, reload plan, actual apply outcome, restart requirement, and any verification limits while keeping `config.schema.lookup` as preflight metadata.

## Reproduction Assessment

Yes, from source inspection: current main’s CLI write path still prints generic restart guidance, and gateway config writes return only write plus restart/sentinel metadata rather than deterministic apply status. I did not run a live write because this review is read-only and the source path is sufficient to confirm the missing contract.

## Solution Assessment

No proposed patch is under review. The best path is a maintainer-approved API/CLI contract that extends the existing write flow, not closing this as covered by the preflight `reloadKind` lookup from https://github.com/openclaw/openclaw/pull/81612.
