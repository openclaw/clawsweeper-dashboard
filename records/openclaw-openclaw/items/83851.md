## Summary

Keep open. Current main and the latest release still do not expose a deterministic post-write config acknowledgement; existing code returns either generic CLI restart text or gateway restart/sentinel metadata, while the merged reload-kind work remains preflight lookup metadata.

## What This Changes

The issue asks for CLI and gateway config writes to return deterministic post-write write/reload/restart/effective-runtime status instead of generic restart guidance.

## Best Possible Solution

Define one shared post-write acknowledgement contract for gateway `config.set`/`config.patch`/`config.apply` and CLI JSON/human output, grounded in the reload planner and explicit about async verification limits.

## Reproduction Assessment

Yes. Source inspection on current main shows gateway config write responses lack the requested changed-path/reload/apply/effective-runtime fields, and CLI mutation success paths still emit generic restart guidance.

## Solution Assessment

No patch is under review. The best path is a maintainer-approved public response contract that extends the existing write/reload flow, not closing this as covered by preflight `reloadKind` lookup or the narrower CLI-hint PR.
