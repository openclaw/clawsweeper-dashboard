## Summary

Keep open. Current main has the preflight `reloadKind` lookup, but post-write gateway RPC and CLI config writes still return generic restart/sentinel metadata rather than the deterministic apply/reload status this asks for.

## What This Changes

Request deterministic machine-readable and human config write acknowledgements for CLI config set/patch and gateway config writes showing write, reload/apply, restart, and runtime verification status.

## Best Possible Solution

Define one shared config-write acknowledgement contract for CLI and gateway RPC that reports write result, changed paths, reload plan, actual apply outcome, restart requirement, and verification limits while keeping `config.schema.lookup` as preflight metadata.

## Reproduction Assessment

Yes, from source inspection: current main's CLI write path still prints generic restart guidance, and gateway config writes return only write plus restart/sentinel metadata rather than deterministic apply status. A live write was not run because this review is read-only and the source path is sufficient to confirm the missing contract.

## Solution Assessment

No proposed patch is under review. The best path is a maintainer-approved API/CLI contract that extends the existing write flow, not closing this as covered by the preflight `reloadKind` lookup from https://github.com/openclaw/openclaw/pull/81612.
