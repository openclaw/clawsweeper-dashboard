## Summary

Keep open. Current main now has partial restart-required signals and better CLI human hints, but it still lacks the requested deterministic post-write config apply contract for CLI JSON, gateway config.set, and gateway config.patch/apply reload completion or effective-runtime status.

## What This Changes

Request deterministic machine-readable post-write apply/reload status for CLI config writes and gateway config write RPC responses.

## Best Possible Solution

Define a backward-compatible shared config-write acknowledgement contract that reuses the reload planner, reports persisted write result and changed paths, states restart requirement and reasons, and explicitly defines reload completion or verification semantics for gateway RPC and CLI JSON output.

## Reproduction Assessment

Yes. Source inspection on current main shows non-dry-run CLI config patch still has no success JSON path, config.set returns only ok/path/config, and config.patch/apply omit the requested changed-path, reload-plan, hot-reload, restart-reason, and effective-runtime status fields.

## Solution Assessment

No patch is under review. The best path is a maintainer-approved gateway/CLI response contract rather than treating the merged preflight reloadKind, CLI hint, or restart-sentinel wording work as sufficient.
