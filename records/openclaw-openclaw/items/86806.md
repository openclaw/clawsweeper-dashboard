## Summary

Keep open: current main still skips restart recovery when lock cleanup finds transcript locks but removes none, and this PR adds a focused fallback for that gap with source-level tests and terminal proof. ## What This Changes

The branch adds a startup session-lock fallback that marks running main sessions as restart-aborted from remaining pre-start lock files, wires it into gateway sidecar cleanup, and adds regression tests.

## Best Possible Solution

Land the focused lock-cleanup fallback only after required checks pass and maintainers accept the pre-start lock heuristic; request a Windows/WebChat crash-restart replay if that heuristic needs stronger end-to-end proof.
