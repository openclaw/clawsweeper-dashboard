## Summary

Keep open: the fix is on the right path and the supplied proof is useful, but the sidecar only invokes the remaining-lock fallback when no locks were cleaned, so mixed crash residue can still leave some sessions unrecovered.

## What This Changes

The PR adds a startup helper that marks running main sessions from pre-start remaining transcript lock files, wires it into the gateway session-lock sidecar, and adds focused restart-recovery tests.

## Best Possible Solution

Keep the startup-time lock-evidence design, but wire remaining-lock marking independently of the cleaned-lock branch and prove mixed cleanup results before merge.

## Reproduction Assessment

Yes. Source inspection provides a high-confidence path: cleanStaleLockFiles can return both locks and cleaned, the PR fallback is gated behind else-if, and recovery still requires abortedLastRun.

## Solution Assessment

No. The helper itself is a good fit, but the else-if wiring is not the narrowest correct fix because it ignores remaining pre-start locks whenever another lock was cleaned.
