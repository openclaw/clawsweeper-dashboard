## Summary

## What This Changes

The PR adds a startup fallback that marks running main sessions as aborted when non-cleaned session lock files remain, wires it into Gateway startup cleanup, and adds focused tests.

## Best Possible Solution

Keep the startup fallback only after it excludes or proves safe live-owned remaining locks, with regression coverage for both crash residue and live pre-start lock cases.

No. Startup-time marking is the right layer, but the mtime-only remaining-lock heuristic is not the safest solution because it ignores the lock inspection contract that preserves live OpenClaw and unknown owners.
