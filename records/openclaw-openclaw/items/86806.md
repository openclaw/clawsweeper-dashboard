## Summary

Keep open for normal merge review. Current main still only marks restart-aborted main sessions when stale locks are actually removed, and this PR is a focused fix for the linked Gateway crash-recovery path; I did not find a blocking diff defect, but the session-state and message-delivery risk deserves maintainer attention before landing.

## What This Changes

The PR adds a startup fallback that marks running main sessions as aborted when pre-start leftover transcript locks remain after stale-lock cleanup, plus focused restart-recovery tests.

## Best Possible Solution

Land the startup-bound lock-evidence recovery path if CI and maintainer review accept the remaining live-crash proof gap; otherwise request one live Gateway crash/restart artifact rather than changing the recovery design.
