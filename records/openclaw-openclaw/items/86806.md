## Summary

Keep open: the PR targets a real P1 session-recovery gap, but the new fallback can treat live pre-start OpenClaw lock owners as crash residue, so it needs a guard or maintainer acceptance before merge.

## What This Changes

The PR adds a startup main-session recovery marker for remaining non-stale session lock files, wires it into Gateway session-lock cleanup, and adds focused regression tests.

## Best Possible Solution

Keep the startup fallback, but exclude or explicitly prove-safe live OpenClaw-owned remaining locks before setting `abortedLastRun`, then cover both crash residue and live pre-start locks in tests.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: current main skips restart marking whenever startup lock cleanup finds locks but removes none, so recovery later ignores running sessions without `abortedLastRun`. The PR body also supplies terminal proof for the narrowed marker/recovery path.

## Solution Assessment

No, not yet: startup-time marking is the right layer, but this implementation uses mtime-only filtering for all remaining locks and does not rule out live OpenClaw-owned pre-start locks. The safer solution needs an owner/liveness guard or explicit maintainer acceptance of that heuristic.
