## Summary

Keep open. The recovery goal is valid and the supplied proof covers the intended crash-residue path, but the merge result can still treat a live pre-start lock that cleanup intentionally preserved as crash evidence and set `abortedLastRun` on an active session.

## What This Changes

The PR adds a Gateway startup fallback that marks running main sessions as crash-aborted when non-cleaned session lock files remain, plus focused recovery tests.

## Best Possible Solution

Require crash-residue evidence that excludes live and unknown-owner remaining locks before setting `abortedLastRun`, and cover both the intended crash recovery and live pre-start lock cases in regression tests.

## Reproduction Assessment

Yes. Source inspection shows a pre-start remaining lock with a matching running session passes the new mtime filter even when cleanup preserved it because the lock was not safe to remove.

## Solution Assessment

No. The PR is a plausible recovery path for the reported crash residue, but it is not the best fix until the remaining-lock path proves the lock owner is gone or otherwise safe to recover.
