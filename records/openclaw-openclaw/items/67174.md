## Summary

Keep open: current main does not implement the split-tenant Teams Graph token capability, but the latest PR head still has security-sensitive regressions and lacks inspectable real split-tenant proof.

## What This Changes

The PR adds `channels.msteams.graphTenantId` / `MSTEAMS_GRAPH_TENANT_ID`, wires Teams Graph token acquisition through a tenant-aware provider, and updates Teams docs, config metadata, tests, and changelog.

## Best Possible Solution

Land a rebased Teams plugin change that adds the Graph tenant override while preserving current secret/token storage and Graph helper guardrails, with docs/tests and redacted proof from a real split-tenant deployment.

## Reproduction Assessment

No, not for the full live split-tenant failure: the PR has no inspectable 403 reproduction or after-fix artifact. Source inspection does show the current-main gap because Teams thread Graph calls use the normal token provider and there is no `graphTenantId` config surface.

## Solution Assessment

No, not as currently written. A dedicated Graph token provider is the right direction, but the raw-secret cache key, private token-store regression, and missing real behavior proof need to be resolved first.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Avoid caching raw Teams client secrets:** `extensions/msteams/src/graph.ts:49`
  - body: `buildGraphTokenProviderCacheKey` includes `creds.appPassword`, so the resolved Azure client secret is retained in a module-level string for the process lifetime. Use a non-secret discriminator or a one-way fingerprint for the cache key instead.
  - confidence: 0.9
- **[P2] Preserve private delegated-token writes:** `extensions/msteams/src/token.ts:179`
  - body: This branch writes delegated Teams OAuth token JSON with raw filesystem calls, while current main uses `privateFileStoreSync(...).writeJson(...)`. Keep the new `graphTenantId` changes without dropping the private token-store protections for refresh and access tokens.
  - confidence: 0.86
