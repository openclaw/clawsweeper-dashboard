- **[P1] Preserve Teams cloud options in Graph token app:** `extensions/msteams/src/graph.ts:71`
  - body: The new cached Graph token provider constructs its SDK App with `loadMSTeamsSdkWithAuth(creds)` and no `resolveMSTeamsSdkCloudOptions(msteamsCfg)`. Current main passes cloud/serviceUrl options through, and the Teams SDK derives Graph scope/base URL from that cloud, so USGov/DoD/China setups can be routed across the wrong auth boundary.
  - confidence: 0.9
- **[P2] Apply graphTenantId before delegated token lookup:** `extensions/msteams/src/graph.ts:264`
  - body: `graphTenantId` is only applied after the delegated-token branch, so `preferDelegated` calls still refresh or return a Graph token using `creds.tenantId`. In a split-tenant deployment with delegated auth enabled, this can keep Graph calls pointed at the bot tenant and reproduce the 403 this PR is meant to fix.
  - confidence: 0.82
