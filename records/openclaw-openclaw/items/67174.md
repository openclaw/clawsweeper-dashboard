## What This Changes

The branch adds `channels.msteams.graphTenantId` and `MSTEAMS_GRAPH_TENANT_ID`, wires Teams Graph token acquisition through a tenant-specific provider, and updates docs, generated config metadata, tests, and changelog.

## Best Possible Solution

Land a rebased Teams plugin change that adds the Graph tenant override while preserving current secret/token storage and Graph helper guards, with docs/tests and redacted proof from a real split-tenant deployment.

## Reproduction Assessment

No for the full live split-tenant failure: the PR has no inspectable 403 reproduction artifact. The source-level gap is clear because current main has no `graphTenantId`/env surface and uses the normal Teams token provider for thread Graph calls.

## Solution Assessment

No, not as currently written. A dedicated Graph token provider is the right direction, but the raw-secret cache key, stale token-store write path, dirty head, and missing real behavior proof need to be resolved first.
