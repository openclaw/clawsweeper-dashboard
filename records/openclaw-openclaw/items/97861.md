## Summary

Keep open: this is a focused bug-fix PR that still appears necessary because current main undercounts provider-rendered bash and summary transcript roles before prompt submission; I found no blocking review findings.

## What This Changes

The PR updates preemptive compaction token estimation to count `bashExecution`, `branchSummary`, and `compactionSummary` provider-rendered text, and adds focused regression tests for bash and summary overflow routing.

## Best Possible Solution

Land the focused estimator and regression-test fix after normal CI/review gates, keeping provider-rendered text as the token-accounting contract.

## Reproduction Assessment

Yes, source-reproducible: current main estimates non-tool/non-assistant records from `record.content`, while bash and summary records render provider-visible text from `command`/`output` or `summary`. I did not run tests because this review is read-only, but the PR body includes before/after live output from the relevant exported functions.

## Solution Assessment

Yes, this is the best local fix: the estimator should count the same text that `convertToLlm` sends to providers. A broader rewrite that runs the whole precheck after conversion would touch more routing surface without being needed for this bug.
