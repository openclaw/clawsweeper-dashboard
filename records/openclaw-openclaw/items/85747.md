## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Stream chunks instead of buffering worker responses:** `src/agents/model-worker.ts:50`
  - body: This reads the full upstream response with `response.arrayBuffer()` before posting anything back to the main thread, so SSE/model output is delivered only after the request finishes. That breaks the streaming behavior the worker pool is supposed to protect; pass chunks over the worker boundary or keep streaming fetches on the current provider transport.
  - confidence: 0.97
- **[P1] Gate force-drain behind explicit config:** `src/infra/restart.ts:521-523`
  - body: `opts.forceDrain` is optional, but this normalizes `undefined` into default 10/20/30/40s phases and the hard-kill hook calls `process.exit(0)`. A normal active-work restart can now hard-kill after 40 seconds instead of honoring the existing deferral behavior.
  - confidence: 0.95
- **[P1] Cancel the worker request before falling back:** `src/agents/model-worker-pool.ts:300-303`
  - body: On timeout this deletes the pending entry and rejects, then the wrapper falls back to the base fetch while the worker fetch is still running. A late worker response is ignored, the worker can remain busy, and the same model request may be sent twice.
  - confidence: 0.9
- **[P2] Queue requests instead of reusing a busy worker:** `src/agents/model-worker-pool.ts:184-191`
  - body: When every worker is busy, this selects a busy worker and posts another request to it rather than queueing. That means `maxWorkers` does not actually bound concurrent model fetches and the single `busy` flag can be cleared while another request on the same worker is still in flight.
  - confidence: 0.86
