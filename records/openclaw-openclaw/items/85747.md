- **[P1] Preserve guarded fetch semantics across the worker boundary:** `src/agents/model-worker.ts:43-47`
  - body: The worker reconstructs requests with only method, headers, and body, so current redirect handling, per-request dispatcher policy, SSRF guard behavior, and abort signals are not preserved for offloaded model calls.
  - confidence: 0.93
- **[P1] Forward stream chunks instead of buffering full responses:** `src/agents/model-worker.ts:50`
  - body: Calling `response.arrayBuffer()` waits for the upstream response to finish before returning anything to the main thread, so SSE/model output is delivered only after completion instead of streaming incrementally.
  - confidence: 0.97
- **[P2] Cancel worker requests before fallback retry:** `src/agents/model-worker-pool.ts:300-303`
  - body: On timeout the pending entry is deleted and the wrapper retries via the main thread, but the worker fetch can still be running, which can send the same model request twice.
  - confidence: 0.9
- **[P2] Queue busy-worker requests instead of reusing a busy worker:** `src/agents/model-worker-pool.ts:184-191`
  - body: When all workers are busy, this returns an already-busy worker and posts another request to it, so `maxWorkers` does not actually bound concurrent fetches and the single busy flag can be cleared while work remains in flight.
  - confidence: 0.86
- **[P1] Keep force-drain disabled unless explicitly configured:** `src/infra/restart.ts:521-523`
  - body: `opts.forceDrain` is optional, but the code normalizes `undefined` into default phases and wires hard kill to `process.exit(0)`, so normal restart deferral can now hard-exit active gateway work before the existing drain budget.
  - confidence: 0.95
