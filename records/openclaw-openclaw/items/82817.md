## Summary

Keep this PR open for normal maintainer review and landing: current main still has the Qwen chat-template thinking mismatch, the proposed change is narrowly scoped to the Qwen wrapper and regression tests, and I found no blocking correctness or security finding.

## What This Changes

The PR updates the Qwen provider stream wrapper and tests so qwen-chat-template payloads mirror the wrapper's session thinking decision into `chat_template_kwargs.enable_thinking`.

## Best Possible Solution

Land the narrow Qwen wrapper fix after maintainer CI/review, then let the linked regression issue close through the merge.

## Reproduction Assessment

Yes, source-reproducible. Current main lets session `thinkingLevel: "off"` omit request `reasoning`, while the qwen-chat-template transport defaults omitted reasoning to `"high"` and writes enabled thinking into `chat_template_kwargs`.

## Solution Assessment

Yes. The Qwen wrapper already owns provider-specific payload patching and has the session thinking level, so mirroring that existing decision there is narrower than adding new core option plumbing.
