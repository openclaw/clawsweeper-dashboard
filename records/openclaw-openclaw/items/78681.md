Keep open: current main still sends the delegated task only through subagent system/developer instructions while the child’s first user turn is a generic kickoff, so the reported non-main run-mode failure remains plausible and source-backed.

## What This Changes

## Best Possible Solution

Make the delegated task reliably model-actionable for run-mode subagents across target agents/runtimes, then add regression coverage that proves the requested token is observable in the child’s executable prompt or final live output.
