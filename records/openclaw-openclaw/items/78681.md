## Summary

Keep open: current main still carries the delegated task only through the subagent system/developer-instruction path while the child run starts from a generic kickoff user message, so the reported non-main Codex run-mode failure remains source-backed and not implemented away.

## What This Changes

Ensure `sessions_spawn` run-mode subagents with an explicit `agentId` execute the delegated `task` instead of responding with a generic kickoff or handshake.

## Best Possible Solution

Make the delegated task reliably model-actionable for run-mode subagents across native and Codex app-server runtimes without reintroducing full task duplication, then cover the non-main `agentId` path with regression and live proof.

## Reproduction Assessment

No live high-confidence reproduction was established in this read-only review. Source inspection gives a credible current-main path: the delegated task is absent from the first user turn and carried only through system/developer instructions on the Codex app-server path.

## Solution Assessment

Yes, the maintainable direction is a narrow prompt-construction/runtime handoff fix plus regression coverage, not a new setting. The fix should keep the task single-sourced or cheaply referenced so it does not undo the prior duplicate-token fix.
