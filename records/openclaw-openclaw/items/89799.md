## What This Changes

Disables compile cache for Windows packaged startup on Node 24.0–24.14 across both pre-import launcher paths, clears inherited cache state through a one-shot respawn, and adds boundary regression coverage.

## Best Possible Solution

Land the narrow Windows-only Node 24.0–24.14 mitigation with both launcher paths and their regression tests kept aligned, then close the linked issue after the PR merges.

## Reproduction Assessment

Yes. The PR supplies high-confidence native Windows proof using the same packed artifact: Node 24.14 takes the no-cache respawn and completes CLI/gateway operations, while Node 24.15 retains compile cache and also completes them.

## Solution Assessment

Yes. A Windows-only guard for Node 24.0–24.14 is narrower and more maintainable than disabling compile cache for every platform or every Node 24 release, and the PR covers both unavoidable pre-import startup implementations.
