## Summary

Keep this PR open, but it is not merge-ready: the patch guards the TypeScript compile-cache helper while the shipped npm/global launcher still enables compile cache in `openclaw.mjs` before that helper can run, and the required native Windows package proof is still absent.

## What This Changes

The PR adds a Node-version/platform guard to the TypeScript compile-cache helper and unit tests that disable compile cache for Windows packaged installs on Node 24.0-24.14.

## Best Possible Solution

Move the guard into the shipped `openclaw.mjs` launcher path, add launcher-level regression coverage for Windows Node 24.0-24.14 and 24.15+, then verify native Windows npm-global startup commands exit successfully.

## Reproduction Assessment

Unclear for the underlying Node deadlock because no native Windows npm-global run was performed. Source inspection does reproduce the PR defect: the package bin enters `openclaw.mjs`, which still enables compile cache before the patched helper can run.

## Solution Assessment

No. The mitigation direction is plausible, but the submitted fix is at the wrong layer; the maintainable fix needs the same guard in the published launcher with launcher e2e coverage and native Windows proof.
