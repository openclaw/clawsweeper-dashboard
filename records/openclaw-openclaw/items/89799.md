## Summary

Keep this PR open for normal maintainer review: the patch is a focused fix for the reported Windows startup deadlock, applies the same policy at both packaged startup layers, preserves compile caching outside the demonstrated affected range, and has credible native Windows after-fix proof.

## What This Changes

The PR disables compile cache and performs a one-shot no-cache respawn for packaged Windows startup on Node 24.0–24.14, while preserving cache use elsewhere and adding entry and launcher regression coverage.

## Best Possible Solution

Land the narrow Windows packaged-startup guard after ordinary owner review and exact-head checks, retaining compile cache for non-Windows runtimes and Windows Node 24.15 or newer.

## Reproduction Assessment

Yes at high confidence from source and supplied native proof: compile cache is activated before the CLI import path, and the PR demonstrates the affected packaged Windows path under Node 24.14 with successful behavior after disabling it.

## Solution Assessment

Yes, subject to owner acceptance of the cutoff; the fix is narrower than disabling compile cache across all Windows or all early Node 24 installations and applies consistently at both startup boundaries.
