## Summary

Keep open: current main still lacks the Windows early-Node-24 compile-cache mitigation, the linked startup-hang issue is still open, and this PR is a focused candidate with supplied native Windows proof and no blocking line-level findings from this review.

## What This Changes

The branch disables inherited or explicitly enabled Node compile cache during packaged Windows startup on Node 24.0-24.14 across `openclaw.mjs` and `src/entry.compile-cache.ts`, with focused launcher and helper regression tests.

## Best Possible Solution

Land the narrow Windows Node 24.0-24.14 startup mitigation on both launcher surfaces once maintainers accept the cutoff and required checks are green, while preserving compile cache everywhere else.

## Reproduction Assessment

Yes, there is a high-confidence reproduction path from the linked report and PR proof: native Windows npm-global OpenClaw on Node 24.1.0 running `openclaw --version`, `doctor --fix`, or `gateway start`. I did not independently rerun that Windows path in this read-only review, so the structured status is source-reproducible rather than reproduced.

## Solution Assessment

Yes, this is an acceptable and likely best mitigation: compile cache is an optimization, the patch scopes the skip to Windows Node 24.0-24.14, and it covers both packaged launcher and typed entry helper paths. The only remaining solution-fit question is maintainer acceptance of the 24.15 cutoff without a Windows-deadlock-specific upstream fix note.
