## Summary

Keep open: the linked Windows startup hang is not fixed on current main, and this PR is a focused implementation candidate with sufficient after-fix native Windows proof and no blocking review findings.

## What This Changes

This PR disables and strips Node compile cache for Windows packaged startup on Node 24.0-24.14 in both the packaged launcher and TypeScript entry helper, with focused regression coverage.

## Best Possible Solution

Land the scoped Windows Node 24.0-24.14 mitigation if maintainers accept the cutoff, then close the linked issue after merge with the native Windows proof.

## Reproduction Assessment

Yes for the code path, but not by a local run in this read-only review: current main enables packaged compile cache before CLI import, and the linked issue plus PR proof identify the Windows Node 24.1 npm-global hang at that startup boundary.

## Solution Assessment

Yes, this appears to be the best bounded mitigation: it changes the pre-import startup layer that owns the compile-cache trigger, avoids disabling all Windows or all Node 24 users, and keeps the optimization enabled outside the proven affected boundary.
