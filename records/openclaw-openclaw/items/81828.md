## Best Possible Solution

Approve and land a narrow admin-scoped protocol, handler, docs, and test change that forwards `allowConfigSizeDrop` only for intentional skill cleanup while preserving the other destructive config guards.

## Solution Assessment

Mostly yes, pending maintainer policy approval. Forwarding the existing writer option through the admin-scoped schema and mutation helper is the narrow technical fix, but maintainers need to decide the acceptable safety constraints.
