## What This Changes

The issue requests a trusted `skills.update` option that forwards `allowConfigSizeDrop` for intentional large skill env cleanup writes.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/81833 or an equivalent narrow admin-scoped protocol change that forwards the existing writer option only for intentional skill cleanup and keeps the rest of the config guard intact.

## Solution Assessment
