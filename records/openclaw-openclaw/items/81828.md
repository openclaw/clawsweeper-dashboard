## Summary

Keep open. Current main still lacks the requested `skills.update` size-drop opt-in: the protocol schema rejects the field, the mutation path cannot forward writer options, and the prior closing PR was closed unmerged. Because this exposes an admin RPC safety override around credential cleanup, it needs maintainer policy approval before a replacement implementation lands.

## What This Changes

Request adds a `skills.update` `allowConfigSizeDrop` opt-in so intentional large skill credential cleanup writes can bypass only the size-drop guard.

## Best Possible Solution

Land a narrow admin-scoped `skills.update` API flag that forwards only `allowConfigSizeDrop` for intentional cleanup, with docs and regression coverage, after maintainers confirm the safety constraints.

## Reproduction Assessment

Yes, at source level. Current main rejects `allowConfigSizeDrop` as an extra `skills.update` property, while the mutation path can delete large env values and the shared writer blocks large size drops unless that option is true.

## Solution Assessment

Unclear until maintainer policy approval. The previous PR's narrow admin-scoped forwarding path is a maintainable implementation shape, but maintainers should choose the allowed caller and mutation constraints before it lands.
