## Summary

Keep open because current main and the latest release still lack a `skills.update` way to opt into `allowConfigSizeDrop`; the related implementation PR was closed unmerged, so the admin RPC policy remains unresolved.

## What This Changes

Request that config-mode `skills.update` accept and forward `allowConfigSizeDrop` for intentional large skill env cleanup writes.

## Best Possible Solution

Reopen or recreate https://github.com/openclaw/openclaw/pull/81833 as a narrow admin-scoped protocol change with schema, handler, docs, and tests proving other destructive config guards remain active.

## Reproduction Assessment

Yes, as a source-level reproduction: current main rejects `allowConfigSizeDrop` as an additional `skills.update` property and the mutation helper cannot forward write options, while the writer blocks large size drops without that option.

## Solution Assessment

Mostly yes, but maintainer policy approval is still needed. Forwarding the existing writer option through the admin-scoped RPC is narrow, provided the implementation keeps default protection and constrains the override to intentional skill cleanup.
