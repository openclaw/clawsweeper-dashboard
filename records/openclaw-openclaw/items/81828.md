## Summary

Keep open: current main and the latest release still lack the requested `skills.update` size-drop opt-in, and the only linked implementation was closed unmerged. The source path is clear, but exposing this admin RPC override still needs maintainer product/security approval.

## What This Changes

Allow trusted config-mode `skills.update` calls to request the existing config writer `allowConfigSizeDrop` option when intentionally deleting large skill env credentials.

## Best Possible Solution

Land a narrow admin-scoped protocol and handler change that forwards only the existing size-drop writer option for intentional skill credential cleanup, with docs and regression coverage for default rejection and allowed cleanup.

## Reproduction Assessment

Yes. Source inspection shows config-mode `skills.update` rejects `allowConfigSizeDrop` as an extra property and has no path to pass writer options; a cleanup that shrinks `openclaw.json` below the writer threshold still hits the shared size-drop guard.

## Solution Assessment

Mostly yes, pending maintainer policy approval. Forwarding the existing writer option through the admin-scoped schema, handler, docs, and focused tests is the narrowest implementation path, but maintainers should decide the exact safety constraints.
