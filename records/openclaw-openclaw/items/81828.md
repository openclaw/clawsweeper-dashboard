## Summary

Keep open because current main still lacks the requested admin-scoped `skills.update` size-drop opt-in; the issue is source-reproducible, and the linked implementation PR was closed unmerged.

## What This Changes

The issue asks for config-mode `skills.update` to accept `allowConfigSizeDrop` and forward it to the existing config writer for intentional large skill env cleanup.

## Best Possible Solution

Reopen and land https://github.com/openclaw/openclaw/pull/81833 or implement the same narrow admin-only schema, handler, mutation, docs, and test change after maintainer policy approval.

## Reproduction Assessment

Yes, source-level. Current main rejects `allowConfigSizeDrop` as an extra `skills.update` property and cannot forward writer options, while the shared writer blocks large size drops unless that option is true.

## Solution Assessment

Unclear pending maintainer policy approval. Forwarding the existing writer option through the admin-scoped `skills.update` path is the narrow technical solution, but maintainers should approve the caller and mutation-shape constraints.
