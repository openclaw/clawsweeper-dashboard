## Summary

Keep open because current main and the latest release still lack the requested admin `skills.update` size-drop opt-in, and the only linked closing implementation was closed unmerged. Source inspection shows the protocol rejects the field and the mutation path cannot forward writer options, while the shared writer still blocks large size drops unless `allowConfigSizeDrop` is true.

## What This Changes

Add an admin-scoped `skills.update` request option that forwards `allowConfigSizeDrop` to the config writer for intentional large skill env cleanup.

## Best Possible Solution

Restore or replace https://github.com/openclaw/openclaw/pull/81833 with a narrow admin-scoped protocol change that forwards the existing writer option for intentional skill credential cleanup while keeping other destructive config guards active.

## Reproduction Assessment

Yes, source-level. Current main rejects `allowConfigSizeDrop` as an extra `skills.update` property and has no path to pass writer options; if env cleanup shrinks the file below the writer threshold, the shared guard blocks unless that option is true.

## Solution Assessment

Mostly yes, pending maintainer policy approval. Forwarding the existing writer option through an admin-scoped schema, handler, docs, and focused tests is narrow, but the exact safety constraints are a product/API decision.
