## Summary

Keep open: current main and the latest stable release still reject the requested field and cannot forward writer options from config-mode skills.update; the linked implementation PR closed unmerged, so a maintainer policy/security decision remains.

## What This Changes

The issue asks to add an admin-scoped allowConfigSizeDrop parameter to config-mode skills.update so intentional large skill env credential cleanup can pass the existing config writer guard.

## Best Possible Solution

Revive or replace https://github.com/openclaw/openclaw/pull/81833 with a narrow admin-scoped schema, handler, docs, and regression-test change that forwards the existing writer option only for intentional skill credential cleanup while keeping other destructive guards active.

## Reproduction Assessment

Yes, source-level. Current main rejects allowConfigSizeDrop as an extra skills.update property and has no path to pass writeOptions; if env cleanup shrinks the file below the writer threshold, the shared guard blocks unless that option is true.

## Solution Assessment

Mostly yes, pending maintainer policy approval. Forwarding the existing writer option through an admin-scoped protocol field, handler, docs, and focused tests is the narrow maintainable path, but the exact allowed mutation shape is a product/security decision.
