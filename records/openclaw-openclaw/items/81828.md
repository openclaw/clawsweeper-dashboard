## What This Changes

The issue requests allowing trusted `skills.update` calls to opt into the existing config writer size-drop bypass for intentional skill environment cleanup.

## Best Possible Solution

Approve and land a narrow admin-scoped protocol, handler, docs, and test change that forwards `allowConfigSizeDrop` only for intentional skill cleanup while preserving the other destructive config guards.

## Reproduction Assessment

Yes, source-level: current main rejects `allowConfigSizeDrop` as an extra `skills.update` property and the mutation helper has no write-options path, so a large env cleanup still hits the shared size-drop guard without an opt-in.

## Solution Assessment

Mostly yes, pending maintainer policy approval: forwarding the existing writer option through the admin-scoped schema and mutation helper is the narrow technical fix, but maintainers need to decide the acceptable safety constraints.
