## Summary

Close: current main fixes the Matrix global-install startup regression by externalizing Matrix from the root package and keeping its runtime dependencies plugin-local, so the root-dependency hoist is now obsolete and conflicts with the chosen package boundary.

## What This Changes

The PR hoists five Matrix runtime dependencies into the root manifest/lockfile and updates dependency ownership tests and knip config for a bundled-Matrix model.

## Best Possible Solution

Keep the current externalized Matrix plugin path with plugin-local dependencies and guardrails; do not merge a root Matrix dependency hoist.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? The original packaged-root failure is well supported by source evidence and reporter logs on 2026.5.x, but it no longer reproduces on current main because Matrix is excluded from the root package and installed as a plugin.

## Solution Assessment

Is this the best way to solve the issue? No for this PR now: hoisting Matrix dependencies solved the old bundled shape, but current main chose the narrower package-boundary fix of externalizing Matrix and keeping its dependency closure plugin-local.
