## Summary

Keep open: the Azure Responses default change is supported, but the PR also changes Azure image generation/edit defaults without live proof for that runtime path and while OpenClaw still builds deployment-scoped image URLs rather than the documented v1 image route.

## What This Changes

The PR changes Azure OpenAI default API versions from `2024-12-01-preview` to `preview` in the Responses transport, bundled OpenAI image provider, tests, and OpenAI provider docs.

## Best Possible Solution

Land the Responses default update with matching tests/docs, but keep the image provider on its proven default unless the exact Azure image route is updated or proven with redacted live image evidence.

## Reproduction Assessment

No high-confidence local reproduction path is available because the failure requires a real Azure OpenAI resource. Source inspection shows current main passes the old default into Azure Responses, and the PR body supplies redacted live negative-control and after-fix output for that path.

## Solution Assessment

No, not as submitted. The Responses transport change is the narrow fix, but the image-provider default change should be removed or backed by live proof for the exact Azure image generation/edit route OpenClaw builds.
