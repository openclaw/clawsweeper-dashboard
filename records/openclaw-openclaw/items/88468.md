- **[P1] Carry legacy model settings into the canonical primary:** `src/plugins/provider-auth-choice.ts:230-234`
  - body: When `previousPrimary` is `codex/gpt-5.5` or `openai-codex/gpt-5.5`, this path now stops preserving the old primary and selects `openai/gpt-5.5`, but nothing moves the old entry's `params`, `agentRuntime`, or `streaming` settings onto the canonical entry. Please merge/copy the legacy entry metadata onto the canonical primary and cover those fields in the regression test.
  - confidence: 0.92

  - body: This helper hardcodes OpenAI and legacy Codex provider ids inside the generic provider-auth path. Repository policy keeps provider auth/catalog/default behavior in the owning plugin or a generic seam, so this will make future provider-specific migrations accumulate in core unless maintainers explicitly approve the exception. Please move the rule into an approved provider-owned/generic migration mechanism, or get that boundary decision recorded before merge.
  - confidence: 0.86
