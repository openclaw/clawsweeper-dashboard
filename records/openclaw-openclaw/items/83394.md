## Summary

Keep open: this is a plausible schema-level start, but it is not merge-ready because it would accept per-agent `tools.codeMode` while the actual runtime activation path still reads only top-level `tools.codeMode`.

## What This Changes

This PR adds `codeMode: CodeModeSchema` to `AgentToolsSchema` and a schema regression test for `agents.list[].tools.codeMode` forms.

## Best Possible Solution

Resolve code mode from the active agent’s effective tools config, align schema/types/docs/tests, and prove one-agent enablement without changing fleet-wide behavior.

## Reproduction Assessment

Yes. Current main has a clear source reproduction: docs promise agent-level `tools.codeMode`, strict `AgentToolsSchema` rejects it, and the runtime resolver currently reads only the root tools object.

## Solution Assessment

No. Adding schema acceptance alone is not the best fix because runtime activation and the exported config type still do not honor the agent-scoped value.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Wire codeMode into active-agent runtime config:** `src/config/zod-schema.agent-runtime.ts:711`
  - body: Adding `codeMode` here only makes `agents.list[].tools.codeMode` parse. Runtime activation still calls `resolveCodeModeConfig(params.config)`, and that resolver reads only `config.tools.codeMode`, so the accepted per-agent setting stays off and the model can still receive the normal tool surface. Please resolve the active agent's `tools.codeMode` into the activation path and payload guard before accepting this schema key.
  - confidence: 0.91
- **[P2] Keep the agent tools type aligned:** `src/config/zod-schema.agent-runtime.ts:711`
  - body: The schema would accept `codeMode` under `agents.list[].tools`, but `AgentToolsConfig` still has no `codeMode?: CodeModeConfig`. That leaves the public config contract inconsistent for typed callers and internal code using `OpenClawConfig`; add the type field alongside the schema/runtime change.
  - confidence: 0.84
