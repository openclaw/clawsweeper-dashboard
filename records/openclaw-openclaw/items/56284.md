## Summary

Keep open: current main has meaningful restart-drain work and a documented `--safe` path, but the reported default Windows Scheduled Task restart/session-loss behavior is not fully settled or proven fixed.

## What This Changes

The issue asks for Windows Scheduled Task gateway restart to wait for active work, preserve restart/session state, and return only after a healthy restart.

## Best Possible Solution

Settle one Windows restart contract, then either make plain Scheduled Task restart use the same drain-safe behavior by default or keep `--safe` as the supported path while landing the narrower session-preservation fix and documenting the limitation clearly.

Unclear. The existing `--safe` path and restart intent are maintainable partial fixes, but the request to make plain Windows restart drain-safe conflicts with the current documented compatibility behavior.
