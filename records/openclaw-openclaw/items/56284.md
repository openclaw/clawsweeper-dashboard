## Summary

Keep open: current main has substantial restart-drain work and an opt-in `--safe` path, but this Windows report still has unresolved scope around plain Scheduled Task restart semantics and session preservation, with related recovery work still open.

## What This Changes

The issue requests that Windows Scheduled Task gateway restart wait for active work, write restart recovery state, and preserve sessions before reporting success.

## Best Possible Solution

Settle one Windows restart contract: either make plain Scheduled Task restart OpenClaw-aware by default, or explicitly keep `--safe` as the supported drain path while fixing the remaining session-preservation gap and documenting the limitation clearly.

## Reproduction Assessment

No. Source inspection gives a plausible current-main path and enough evidence to keep the issue open, but high-confidence reproduction requires a live Windows Scheduled Task restart with active task/session work to observe whether the service-manager cleanup interrupts the drain.

## Solution Assessment

Unclear. The existing `--safe` path plus PR 71465's restart intent are maintainable partial fixes, but the report asks for plain Windows restart to be drain-safe and that now conflicts with documented compatibility behavior.
