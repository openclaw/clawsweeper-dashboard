## Summary

Keep open: current main has an explicit `--safe` restart path and restart-intent drain work, but plain Windows Scheduled Task restart still follows the documented service-manager path and the mid-turn session reset remains source-reproducible.

## What This Changes

The issue asks for Windows Scheduled Task gateway restarts to drain active work and preserve or restore session state instead of killing tasks and clearing mid-turn sessions.

## Best Possible Solution

Settle the Windows restart contract, land the narrow session-preservation fix, and require native Windows proof that the chosen restart modes preserve active work or clearly document when they do not.

## Reproduction Assessment

## Solution Assessment

Unclear. The opt-in `--safe` path is a maintainable partial solution, but making plain Windows restart drain-safe by default conflicts with the documented compatibility behavior and needs maintainer choice.
