## Summary

Keep open: current main has restart-intent draining and an opt-in `--safe` path, but plain Windows Scheduled Task restart still goes through `/End` plus short listener cleanup, and the user-only transcript reset path from the session-loss report is still present.

## What This Changes

Make Windows Scheduled Task gateway restarts drain active work and preserve interrupted session state instead of killing tasks and losing transcripts.

## Best Possible Solution

Define one Windows restart contract, then make the Scheduled Task implementation and session recovery match it with regression coverage and live Windows proof.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection shows the relevant current-main paths: plain Windows restart still uses Scheduled Task `/End`/listener cleanup, and the user-message/no-assistant transcript reset condition still exists.

## Solution Assessment

Unclear. The current `--safe` path is a maintainable partial solution, but it does not satisfy the plain `gateway restart` expectation and does not cover the current user-only transcript reset behavior.
