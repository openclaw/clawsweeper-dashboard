## Summary

Keep open: current main has an opt-in safe restart path, but plain Windows Scheduled Task restarts still go through the service-manager end/run path and related session-preservation work remains unmerged.

## What This Changes

The issue requests Windows Scheduled Task `openclaw gateway restart` to drain active Gateway work and preserve or recover session state instead of abruptly killing tasks.

## Best Possible Solution

Define and ship one Windows Scheduled Task restart contract that preserves active work and session state, either by making plain restart safe or by explicitly documenting and validating `--safe` as the supported drain path while fixing remaining session-loss gaps.

## Reproduction Assessment

No. Source inspection shows a plausible current-main path where plain Windows Scheduled Task restart can still cut off active work, but high-confidence reproduction requires a live Windows Scheduled Task run with an active Gateway task/session.

## Solution Assessment

Unclear. The current `--safe` path is a maintainable opt-in and matches one acceptable direction, but it does not make plain Windows Scheduled Task restart drain-safe and does not address the still-open mid-turn session reset path.
