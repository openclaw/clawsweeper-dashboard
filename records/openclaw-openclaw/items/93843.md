This PR should stay open for maintainer review: current main still has the silent overwrite path, the patch is focused and source-supported, and the remaining issue is conscious acceptance of the collision-case persisted identity contract plus coordination with the adjacent reserved-index PR.

## What This Changes

The PR adds a shared title-keyed memory-wiki slug resolver, routes synthesis creation and source ingest through it, and adds regression coverage for distinct titles that normalize to the same slug.

## Best Possible Solution

Land this focused resolver once maintainers accept hash-suffixed identities for genuine distinct-title collisions, and keep reserved `index.md` handling composed with the adjacent PR.

## Reproduction Assessment

Yes. Current main source shows both affected writer paths derive page path and ID from the bare slug, so titles like `Q3 Report` and `Q3-Report` target the same page; the PR body also includes before/after real memory-wiki output.

## Solution Assessment

Yes. A shared resolver next to `slugifyWikiSegment` is the narrow maintainable boundary for the two direct page writers; the older source-path-only approach would miss the agent-facing synthesis path, and changing the shared slugifier would churn sibling composed identities.
