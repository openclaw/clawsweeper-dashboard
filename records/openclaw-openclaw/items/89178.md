## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Use no-create opens for SQLite reads:** `src/config/sessions/session-accessor.sqlite.ts:90`
  - body: These read helpers open the normal agent database path, which creates the directory/file, applies schema DDL, and registers the DB. When doctor/startup preview or other read-only scans switch to this accessor, an absent agent DB will be materialized instead of returning empty, repeating the failure class called out from https://github.com/openclaw/openclaw/pull/91322. Add a read-only/no-create opener and use it for read helpers.
  - confidence: 0.93
- **[P1] Export transcript rows before deleting sessions:** `src/config/sessions/session-accessor.sqlite.ts:660`
  - body: Deleting the `sessions` row here triggers the schema's `transcript_events` cascade, but this path only counts the transcript state as archived and does not write an archive/export receipt first. Once SQLite transcripts are live, lifecycle cleanup can silently drop transcript history; enforce export-before-delete structurally before issuing session deletes.
  - confidence: 0.9
- **[P3] Document the transitional entry_json bridge:** `src/config/sessions/session-accessor.sqlite.ts:822`
  - body: The writer persists the whole entry in `entry_json` while typed session columns are also maintained, and the reader still parses `entry_json` as the canonical entry. Add the inline contract and retirement milestone so this does not become a permanent blob-store side path after the typed schema lands.
  - confidence: 0.82
