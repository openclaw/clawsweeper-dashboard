Full review comments:

- **[P1] Use no-create opens for SQLite read helpers:** `src/config/sessions/session-accessor.sqlite.ts:143`
  - body: These read helpers call the normal agent database opener, which creates directories/files, applies schema DDL, and registers the DB. When doctor/startup preview or other read-only scans switch to this accessor, an absent agent DB will be materialized instead of returning empty results.
  - confidence: 0.93
- **[P1] Require export receipts before cascading transcript deletes:** `src/config/sessions/session-accessor.sqlite.ts:1015`
  - body: Deleting the sessions row here triggers the schema's transcript_events cascade, but this path only counts transcript state as archived and does not write or require an archive/export receipt first. Once SQLite transcripts are live, lifecycle cleanup can silently drop transcript history.
  - confidence: 0.9
- **[P3] Document the transitional entry_json bridge:** `src/config/sessions/session-accessor.sqlite.ts:1177`
  - body: The writer persists the whole entry in entry_json while typed session columns are also maintained, and readers still parse entry_json as the entry payload. Add the local transition contract and retirement milestone so this does not become a permanent blob-store side path after the typed schema lands.
  - confidence: 0.82
