## Summary

Keep open. Current main still constructs `activeSession.prompt(...)` before the abort wrapper can observe a pre-aborted signal, and open PR #74979 already uses closing syntax for this issue, so this should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue reports that pre-aborted embedded runs can start `activeSession.prompt()` before `abortable()` rejects, leaving a background Agent loop alive.

## Best Possible Solution

Land PR #74979 or an equivalent narrow fix that checks abort state before prompt construction and adds a cleanup backstop; treat a configurable per-run LLM-call cap as separate hardening.

## Reproduction Assessment

Yes. The issue includes a deterministic pre-aborted `AbortSignal` test, and current main still shows the prompt call being constructed before `abortable()` can reject; I did not execute tests in this read-only review.

## Solution Assessment

Yes, the narrow fix direction is the best path. A pre-prompt abort guard addresses the root race, while cleanup abort/queue clearing is a prudent backstop and the proposed max-call cap should be follow-up defense-in-depth.
