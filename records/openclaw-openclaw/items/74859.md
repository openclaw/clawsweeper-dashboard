## Summary

Keep open. Current main still has the pre-aborted signal race at the embedded prompt boundary, and open PR #74979 is already the active closing implementation, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue reports that pre-aborted embedded run signals can let `activeSession.prompt()` start a floating Agent loop that keeps calling the model after the attempt exits.

## Best Possible Solution

Land PR #74979 or an equivalent narrow fix that checks abort state before every embedded prompt start and adds a cleanup backstop; keep any configurable per-run LLM-call cap as separate hardening.

## Reproduction Assessment

Yes. The supplied pre-aborted `AbortSignal` reproduction maps directly to current main because `activeSession.prompt(...)` is still evaluated before `abortable(...)` can reject, and interrupt mode still aborts active runs before starting the newest message.

## Solution Assessment

Yes. A pre-prompt abort guard plus escaped-agent cleanup backstop is the narrowest maintainable repair for this lifecycle race; a max-call cap would be useful defense-in-depth but should not replace the direct fix.
