## Summary

Keep open. Current main still has the reported source-level prompt-start race, and the supplied GitHub context shows open PR #74979 already paired with this issue, so the issue should stay open until that PR or an equivalent replacement merges.

## What This Changes

Prevent pre-aborted embedded Pi Agent prompts from starting a floating Agent run that can continue making LLM calls after an interrupted attempt exits.

## Best Possible Solution

Land #74979 or an equivalent narrow replacement that guards the run abort state before every embedded Pi prompt-start boundary, adds a cleanup backstop for escaped active runs and queued messages, and includes focused regression coverage for pre-aborted, abort-before-active-run, and normal prompt paths.

## Reproduction Assessment

Yes. The issue includes a deterministic pre-aborted `AbortSignal` test shape, and current main still has `activeSession.prompt(...)` evaluated before the abort wrapper can reject; this read-only review did not run the test.

## Solution Assessment

Yes for the narrow fix direction. A pre-prompt abort guard plus escaped-agent cleanup backstop is the maintainable repair; the configurable max-LLM-call cap is useful hardening but should be a separate decision.
