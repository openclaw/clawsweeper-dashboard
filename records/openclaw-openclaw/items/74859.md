## Summary

Keep open. Current main still has the reported prompt-start race: the external abort can be observed before submission, but the embedded runner still constructs `activeSession.prompt(...)` before the local abort wrapper can reject. The issue is also paired with open PR #74979 using closing syntax, so it should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue requests preventing aborted embedded agent attempts from starting or leaving behind a background Agent loop when the run abort signal is already aborted before prompt submission.

## Best Possible Solution

Land PR #74979 or an equivalent narrow replacement that checks the run abort state before every embedded prompt-start boundary, adds an escaped-agent cleanup backstop, and includes focused regression coverage for pre-aborted, abort-before-active-run, and normal prompt paths. Keep the configurable max-LLM-call cap as separate hardening unless maintainers explicitly want that scope in the same fix.

## Reproduction Assessment

Yes. The issue includes a deterministic pre-aborted `AbortSignal` test shape, and current main still shows the same evaluation order that can start `activeSession.prompt(...)` before the abort wrapper rejects; this review did not run the test.

## Solution Assessment

Yes for the narrow repair direction. A pre-prompt abort guard plus cleanup backstop is the smallest maintainable fix for the lifecycle race, while the proposed LLM-call cap is useful defense-in-depth but should not block the core bug fix.
