## Summary

Keep open. Current main still has the prompt-start race described in the report, and the issue is already paired with an open closing PR, so it should remain open until that PR or an equivalent replacement lands.

## What This Changes

The issue asks OpenClaw to prevent pre-aborted embedded Pi prompt submissions from starting floating Agent loops after the attempt lifecycle exits.

## Best Possible Solution

Land PR #74979 or an equivalent narrow replacement that makes the prompt-start boundary abort-aware, adds cleanup backstops for escaped active Agent work and queued Pi messages, and includes focused regression coverage. Treat a configurable max-LLM-call cap as separate hardening unless maintainers explicitly want that broader scope.

## Reproduction Assessment

Yes. The issue includes a deterministic pre-aborted `AbortSignal` reproduction shape, and current-main source inspection still shows `activeSession.prompt(...)` being evaluated before `abortable(...)` can reject.

## Solution Assessment

Yes for the narrow fix direction. Guarding cancellation before every prompt-start boundary plus adding cleanup backstops is the maintainable repair; the proposed per-run LLM call cap is useful defense in depth but should not block the core bug fix.
