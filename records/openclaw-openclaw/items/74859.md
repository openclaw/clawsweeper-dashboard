## Label Justifications

- `P1`: The issue describes a silent zombie agent loop that can continue making model calls after an interrupted run in real interrupt-mode workflows.
- `impact:session-state`: The escaped prompt loop outlives the intended embedded attempt lifecycle and can drift from the session run state.

## Summary

Keep open: current main and the latest release still create the embedded session prompt promise before the abort wrapper can reject a pre-aborted signal, and the linked fixing PR is still open rather than merged.

## What This Changes

The issue reports that a pre-aborted embedded run signal can create a floating agent prompt loop that continues making LLM calls after the attempt exits.

## Best Possible Solution

Land or replace the linked pre-prompt guard with production-path regression coverage and runtime proof that no LLM calls continue after an interrupted run exits.

## Reproduction Assessment

Yes, source-level: current main still evaluates `activeSession.prompt()` before `abortable()` can reject a pre-aborted signal, and the issue includes a deterministic AbortSignal-based repro. I did not run the full interrupt-mode reproduction in this read-only pass.

## Solution Assessment

Yes in direction: a pre-prompt guard in `promptActiveSession` is the narrowest fix for the creation window. The open linked PR owns that path, but the issue should stay open until the fix lands or is replaced.
