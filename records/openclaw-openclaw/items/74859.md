## Label Justifications

- `P1`: The issue describes a silent zombie agent loop that can continue model calls after an interrupted run in a real interrupt-mode workflow.
- `impact:session-state`: The escaped prompt loop outlives the intended embedded attempt lifecycle and can drift from the active session run state.
- `impact:crash-loop`: The repeated background loop can create unbounded runtime activity and resource exhaustion until the process is restarted or the run is otherwise stopped.

## Summary

Keep open: current main and v2026.6.10 still construct the embedded session prompt before the abort wrapper can reject a pre-aborted signal, and the linked fixing PR remains open rather than merged.

## What This Changes

The issue requests that aborted embedded agent runs avoid starting or leaving a background Agent prompt loop when the run signal is already aborted before `activeSession.prompt()`.

## Best Possible Solution

Land or replace the linked pre-prompt guard with production-path regression coverage and runtime proof that no LLM calls continue after an interrupted run exits.

## Reproduction Assessment

Yes, source-level: current main still evaluates `activeSession.prompt()` before the abort helper can reject a pre-aborted run signal, and the issue includes a deterministic AbortSignal repro. I did not run the full interrupt-mode live reproduction in this read-only pass.

## Solution Assessment

Yes in direction: a guard before `activeSession.prompt()` is the narrowest fix for the creation window. The linked open PR owns that path, so this issue should stay open until that fix lands or is replaced.
