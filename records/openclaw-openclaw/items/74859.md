## Summary

Keep open. Current main still starts `activeSession.prompt(...)` before the abort wrapper can observe a pre-aborted signal, and open PR #74979 already uses closing syntax for this issue, so this should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue requests fixing a pre-aborted embedded-run abort path that can leave an Agent prompt loop running after the attempt has exited.

## Best Possible Solution

Review and land PR #74979 or an equivalent narrow fix that checks the abort signal before every embedded prompt starts and aborts or clears escaped agent work during cleanup; treat a configurable per-run LLM-call cap as separate hardening.

## Reproduction Assessment

Yes. The issue includes a deterministic pre-aborted `AbortSignal` reproduction, and current main still has the exact direct `abortable(activeSession.prompt(...))` source pattern needed to trigger it; I verified the source path rather than executing tests in this read-only review.

## Solution Assessment

Yes for the primary repair direction. A pre-prompt abort guard plus cleanup backstop is the narrow maintainable fix for the lifecycle race, while the proposed max-call cap is useful defense-in-depth but not required to fix the bug.
