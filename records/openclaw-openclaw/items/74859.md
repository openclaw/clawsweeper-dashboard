## Label Justifications

- `P1`: The issue describes a silent embedded-agent zombie loop that can keep making model calls after an interrupted run in real user workflows.

## Summary

Keep open: current main and the latest release still create the embedded session prompt promise before the abort wrapper can reject a pre-aborted signal, and the linked closing PR remains open and conflicting rather than merged.

## What This Changes

The issue reports that a pre-aborted embedded attempt signal can still start `activeSession.prompt()`, leaving an Agent loop running after the interrupted attempt exits.

## Best Possible Solution

Land or replace the linked pre-prompt guard with production-path regression coverage and runtime proof that interrupted runs make no post-abort LLM calls.

## Reproduction Assessment

Yes, source-level: current main still constructs `activeSession.prompt()` before `abortable()` can reject a pre-aborted signal, and the issue includes a deterministic AbortSignal-based repro. I did not run the live interrupt-mode reproduction in this read-only pass.

## Solution Assessment

Yes in direction: a pre-prompt guard in `promptActiveSession` is the narrowest fix for the promise-creation window. This issue should stay open because the linked fix PR has not landed and currently needs refresh/review.
