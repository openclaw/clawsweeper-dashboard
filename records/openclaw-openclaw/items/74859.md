## Summary

Keep open. Current main still creates `activeSession.prompt(...)` promises before the abort wrapper can observe a pre-aborted run signal, and the issue is already paired with open PR #74979 using closing syntax, so the issue should remain open until that PR is revised, merged, or replaced.

## What This Changes

The issue requests preventing pre-aborted embedded Pi Agent prompt submissions from spawning unowned background Agent loops after interrupt or timeout aborts.

## Best Possible Solution

Revise and land #74979, or an equivalent narrow replacement, so the embedded runner checks the run abort signal before every `activeSession.prompt()` start, cancels and clears escaped active Agent work during abort cleanup, and adds regression coverage for pre-aborted, mid-abort, and normal prompt paths. A configurable per-run LLM-call cap can remain a separate defense-in-depth follow-up.

## Reproduction Assessment

Yes. The issue includes a deterministic pre-aborted `AbortSignal` test shape, and current main still has the same source-level call-order hazard; this review did not execute the test because the task is read-only.

## Solution Assessment

Yes for the narrow repair direction. A pre-prompt abort guard plus cleanup backstop is the maintainable fix boundary; the proposed max-LLM-call cap is useful hardening but should not block the focused bug fix.
