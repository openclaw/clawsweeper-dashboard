## Summary

Keep open. Current main still starts `activeSession.prompt(...)` before the abort wrapper can observe an already-aborted signal, and https://github.com/openclaw/openclaw/pull/74979 remains the open closing implementation path.

## What This Changes

The issue requests fixing embedded-runner abort handling so a pre-aborted `activeSession.prompt()` call cannot spawn an unowned Pi Agent loop.

## Best Possible Solution

Fix the prompt-start boundary so already-aborted and mid-preflight aborts are observed before Pi can start `Agent.prompt`, add focused regression coverage, and treat a per-run LLM-call cap as separate hardening.

## Reproduction Assessment

Yes. The deterministic pre-aborted `AbortSignal` path is source-reproducible on current main because the external abort flips the run controller, but `promptActiveSession` still evaluates `activeSession.prompt(...)` before the abort wrapper can reject.

## Solution Assessment

Yes. The narrow maintainable fix is an abort-aware prompt-start boundary plus escaped-agent cleanup coverage; the proposed LLM-call cap is useful defense-in-depth but should not block the primary bug fix.
