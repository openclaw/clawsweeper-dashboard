## Summary

Keep open. Current main still starts `activeSession.prompt(...)` before `abortable(...)` can reject an already-aborted run signal, and the supplied GitHub context shows open PR #74979 is explicitly paired with this issue, so the maintainer path is to review, revise, land, or replace that PR rather than close the issue now.

## What This Changes

The issue asks OpenClaw to prevent pre-aborted embedded Pi Agent prompt submissions from spawning a background Agent loop after an interrupted run exits.

## Best Possible Solution

Land PR #74979 or an equivalent narrow replacement that checks abort state before each embedded Pi Agent prompt start, adds cleanup backstops for escaped active runs and queued messages, and includes focused regression coverage for pre-aborted, abort-before-active-run, and normal prompt paths.

## Reproduction Assessment

Yes. The issue includes a deterministic pre-aborted `AbortSignal` test shape, and current main still has source-level prompt-call ordering that reproduces the hazard; this read-only pass did not run tests.

## Solution Assessment

Yes for the narrow repair direction. A pre-prompt abort guard plus cleanup backstop is the maintainable fix; the configurable max-LLM-call cap is useful hardening but should not block or broaden the immediate bug fix.
