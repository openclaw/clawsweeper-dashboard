## Label Justifications

## Summary

Keep open: current main and latest release still have the prompt-before-abortable shape, and the linked fix PR remains open rather than merged.

## What This Changes

The issue reports that a pre-aborted run signal can reject `abortable(activeSession.prompt(...))` after starting a floating Agent prompt loop that continues LLM calls.

## Best Possible Solution

Review, rebase, and land the linked pre-prompt guard, or replace it with the same narrow production-path fix plus regression coverage and redacted interrupt-mode proof.

## Reproduction Assessment

Yes, source-level: current main still evaluates `activeSession.prompt()` before `abortable()` can reject a pre-aborted signal. I did not run the full interrupt-mode repro in this read-only pass.

## Solution Assessment

Yes in direction: a pre-prompt guard in `promptActiveSession` is the narrowest fix for the creation window. The open linked PR owns that direction, but it should be made merge-ready before this issue closes.
