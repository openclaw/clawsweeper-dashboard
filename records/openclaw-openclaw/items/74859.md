## Summary

Keep open. Current main still has the `abortable(activeSession.prompt(...))` argument-evaluation race, and #74979 is an open unmerged PR with closing syntax, so this issue should stay open until that PR or an equivalent fix lands.

## What This Changes

Prevent aborted embedded agent attempts from starting or leaving behind a floating Agent prompt loop when the run signal is already aborted before `activeSession.prompt()`.

## Best Possible Solution

Review and land PR #74979 or an equivalent narrow fix that checks abort state before starting any prompt and adds a cleanup backstop for escaped agent work.

## Reproduction Assessment

Yes, from source inspection. The issue includes a deterministic pre-aborted `AbortSignal` test, and current main still calls `activeSession.prompt(...)` before `abortable()` can reject an already-aborted signal; I did not execute tests in this read-only review.

## Solution Assessment

Yes. A pre-prompt abort guard plus cleanup backstop is the narrow maintainable fix for the lifecycle race; a configurable max-call cap is useful hardening but not required for the primary bug repair.
