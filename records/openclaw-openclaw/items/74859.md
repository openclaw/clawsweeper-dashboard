## Summary

Keep open. Current main still has the reported prompt-start abort race at the embedded Pi Agent boundary, and open PR #74979 is already paired with this issue, so the right path is to review, revise, merge, or replace that PR rather than close the issue now.

## What This Changes

The issue requests fixing a pre-aborted `AbortSignal` race in embedded Pi Agent prompt submission that can leave a background Agent loop running after the attempt exits.

## Best Possible Solution

Land #74979 or an equivalent narrow replacement that checks the run abort state before every Pi Agent prompt-start boundary, adds cleanup backstops for escaped active runs and queued messages, and includes focused regression coverage for pre-aborted, abort-before-active-run, and normal prompt paths.

## Reproduction Assessment

Yes. The issue includes a deterministic pre-aborted `AbortSignal` test shape, and current main still has the source-level call-order hazard that would let `activeSession.prompt()` start before `abortable()` rejects; this read-only pass did not run the test.

## Solution Assessment

Yes for the narrow fix direction. A pre-prompt abort guard plus escaped-agent cleanup backstop is the maintainable repair; the proposed configurable max-LLM-call cap is useful defense-in-depth but should be a separate hardening decision.
