## Summary

Keep open. Current main still has the reported prompt-start race at the embedded runner prompt boundary, and open PR #74979 already uses closing syntax for this issue, so this should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue reports that a pre-aborted embedded run signal can reject the attempt while leaving an un-aborted Pi Agent prompt loop making repeated background LLM calls.

## Best Possible Solution

Land PR #74979 or an equivalent narrow replacement that guards abort state before every embedded prompt start and adds a cleanup backstop for escaped active Agent work; treat a configurable per-run LLM-call cap as separate hardening.

## Reproduction Assessment

Yes. The issue includes a deterministic pre-aborted `AbortSignal` reproduction, and current main still evaluates `activeSession.prompt(...)` before `abortable(...)` can reject on that signal; this read-only review did not execute tests.

## Solution Assessment

Yes for the narrow fix direction. A pre-prompt abort guard plus escaped-agent cleanup is the smallest maintainable repair, while the proposed hard LLM-call cap should not block the core bug fix.
