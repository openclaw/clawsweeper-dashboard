## Summary

Keep open. Current main still evaluates `activeSession.prompt(...)` before the abort wrapper can reject a pre-aborted signal, and open PR #74979 already uses closing syntax for this issue, so the issue should stay open until that PR or an equivalent fix lands.

## What This Changes

Prevent pre-aborted embedded prompts from starting an Agent loop that keeps making LLM calls after the attempt lifecycle exits.

## Best Possible Solution

Review and land PR #74979 or an equivalent narrow fix that guards before every embedded prompt start and adds a cleanup backstop; treat a configurable per-run LLM-call cap as separate hardening.

## Reproduction Assessment

Yes. The issue includes a deterministic pre-aborted `AbortSignal` reproduction, and current main still has the exact argument-evaluation pattern needed to trigger it; I did not execute tests because this was a read-only review.

## Solution Assessment

Yes. A pre-prompt abort guard plus cleanup backstop is the narrow maintainable fix for the lifecycle race, while the suggested max-call cap is useful defense-in-depth but not required for the primary repair.
