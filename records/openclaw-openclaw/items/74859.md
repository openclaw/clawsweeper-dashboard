## Summary

Keep open. Current main still constructs `activeSession.prompt(...)` before `abortable(...)` can reject an already-aborted run signal, and open PR #74979 is linked with closing syntax, so this issue should remain open until that fix or an equivalent lands.

## What This Changes

Prevent pre-aborted embedded prompt attempts from starting an unowned Agent loop that continues making LLM calls after the run exits.

## Best Possible Solution

Land #74979 or an equivalent narrow fix that checks abort state before every embedded prompt start and adds an Agent cleanup backstop, leaving configurable LLM-call caps to separate hardening.

## Reproduction Assessment

Yes. The issue includes a deterministic pre-aborted `AbortSignal` reproduction, and current main still has the vulnerable `abortable(activeSession.prompt(...))` call pattern; I did not execute tests because this review is read-only.

## Solution Assessment

Yes for the current fix direction. A pre-prompt abort guard prevents creating the floating promise, and an Agent cleanup backstop covers escaped work; max-call limits are useful defense-in-depth but should not block the core repair.
