## Summary

Keep open. Current main still has the reported prompt-start race at the embedded runner prompt boundary, and open PR #74979 already uses closing syntax for this issue, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue requests that aborted embedded runs avoid starting a floating `activeSession.prompt()` chain and clean up escaped Agent work when an abort signal is already aborted before prompt submission.

## Best Possible Solution

Land PR #74979 or an equivalent narrow replacement that checks abort state before every embedded prompt start and adds a cleanup backstop for escaped active Agent work; treat a configurable per-run LLM-call cap as separate hardening.

## Reproduction Assessment

Yes. The issue includes a deterministic pre-aborted `AbortSignal` reproduction, and current main still constructs `activeSession.prompt(...)` before `abortable(...)` can reject on that signal; I did not execute tests because this was a read-only review.

## Solution Assessment

Yes for the proposed narrow fix direction. A pre-prompt abort guard plus cleanup backstop addresses the lifecycle race directly, while a new max-call configuration would be useful defense-in-depth but should not block the core repair.
