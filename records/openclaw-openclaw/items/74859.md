## Summary

Keep open. Current main still evaluates `activeSession.prompt(...)` before `abortable()` can reject an already-aborted run signal, and open PR #74979 already references this issue with closing syntax, so the issue should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue asks OpenClaw to prevent pre-aborted embedded agent runs from starting an escaped Agent loop that continues making LLM calls after the attempt exits.

## Best Possible Solution

Review and land PR #74979 or an equivalent narrow fix that checks abort state before every embedded prompt start and adds an escaped-agent cleanup backstop; treat a configurable LLM-call cap as separate hardening.

## Reproduction Assessment

Yes. The deterministic pre-aborted `AbortSignal` path is source-reproducible on current main: `onAbort()` aborts the run controller, but `activeSession.prompt(...)` is still evaluated before `abortable()` rejects.

## Solution Assessment

Yes. A pre-prompt abort guard plus cleanup backstop is the narrow maintainable fix for the escaping promise; the requested max-call cap is useful defense-in-depth but not required for the primary bug.
