## Summary

Keep open. Current main still starts `activeSession.prompt()` before the abort wrapper can observe a pre-aborted run signal, and open PR #74979 is already the linked closing fix candidate.

## What This Changes

Prevent pre-aborted embedded runner abort signals from starting escaped Pi Agent prompt loops that continue making LLM calls after attempt cleanup.

## Best Possible Solution

Land PR #74979 or an equivalent narrow fix that guards abort state before every embedded prompt start and adds cleanup for escaped active Agent work; treat a per-run LLM-call cap as separate hardening.

## Reproduction Assessment

Yes. The issue includes a deterministic pre-aborted `AbortSignal` reproduction, and current main still has the source-level evaluation order that starts `prompt()` before the abort wrapper rejects.

## Solution Assessment

Yes for the narrow fix direction. A pre-prompt abort guard plus an escaped-agent cleanup backstop is the smallest maintainable repair; the proposed hard LLM-call cap should not block the core bug fix.
