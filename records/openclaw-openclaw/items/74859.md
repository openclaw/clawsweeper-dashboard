## Label Justifications

## Summary

Keep open: current `main` and the latest release still construct the `activeSession.prompt()` promise before `abortable()` can reject a pre-aborted signal, and the linked fix PR remains open/unmerged.

## What This Changes

The issue reports that a pre-aborted embedded-run abort signal can create a floating Agent prompt loop that keeps making LLM calls after the attempt exits.

## Best Possible Solution

Land the linked pre-prompt guard PR, or an equivalent current-main patch, so the embedded runner checks the run abort signal before calling `activeSession.prompt()` and covers that path with focused regression proof.

## Reproduction Assessment

Yes, source-level: current `main` still creates the `activeSession.prompt()` promise before the abortable helper can reject a pre-aborted signal, and the issue provides a deterministic AbortSignal-based test path. I did not run tests because this was a read-only review.

## Solution Assessment

Yes for direction: a guard before `activeSession.prompt()` is the narrowest maintainable fix for the creation window. The issue should stay open until the linked PR or equivalent production-path fix is merged and validated.
