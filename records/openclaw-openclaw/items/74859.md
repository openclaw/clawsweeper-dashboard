## Summary

Keep open. Current main still has the reported prompt-start race, and the issue is already paired with open closing PR #74979, so the safe maintainer path is to review and land that PR or an equivalent replacement before closing this issue.

## What This Changes

Prevent pre-aborted embedded Pi prompt submissions from starting floating Agent loops, and ensure aborted attempts clean up escaped Agent work.

## Best Possible Solution

Guard the run abort state before every embedded prompt-start boundary, add a cleanup backstop that aborts/clears escaped active Agent work, and include focused regression coverage for pre-aborted and normal prompt paths. Treat a configurable max-LLM-call cap as separate hardening unless maintainers explicitly want that scope included.

## Reproduction Assessment

Yes. The issue includes a deterministic pre-aborted `AbortSignal` test shape, and current main still evaluates `activeSession.prompt(...)` before the abort wrapper can reject; this read-only review did not run the test.

## Solution Assessment

Yes for the narrow fix direction. A pre-prompt abort guard plus cleanup backstop is the maintainable repair; a per-run LLM call cap is useful defense-in-depth but should not block the core bug fix.
