## Label Justifications

- `P1`: The issue describes a silent zombie Agent loop that can continue making model calls after an interrupted run in real interrupt-mode workflows.
- `impact:session-state`: The escaped prompt loop outlives the intended embedded attempt lifecycle and can drift from the session run state.

## Summary

Keep open: current main and the latest release still construct the embedded session prompt promise before the abort wrapper can reject a pre-aborted signal, and the linked closing PR remains open and conflicting rather than merged.

## What This Changes

The issue reports that a pre-aborted embedded run signal can let `activeSession.prompt()` start a background Agent loop that keeps making LLM calls after the attempt exits.

## Best Possible Solution

Land or replace the linked pre-prompt guard with production-path regression coverage and runtime proof that no LLM calls continue after an interrupted run exits.

## Reproduction Assessment

Yes, source-level: current main still constructs `activeSession.prompt()` before `abortable()` can reject a pre-aborted signal, and the issue includes a deterministic AbortSignal-based repro. I did not run the live interrupt-mode reproduction in this read-only pass.

## Solution Assessment

Yes in direction: a pre-prompt guard in `promptActiveSession` is the narrowest fix for the creation window. The issue should stay open because the linked fix PR has not landed and currently needs refresh/review.
