## Summary

Keep open. Current main still has the pre-aborted signal race in the embedded runner, and open PR #74979 already uses closing syntax for this issue, so this issue should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue reports that a pre-aborted signal passed to `activeSession.prompt()` can leave a floating Agent loop that continues making LLM calls after the attempt exits.

## Best Possible Solution

Review and land PR #74979 or an equivalent narrow fix that checks abort state before every embedded prompt start and adds a cleanup backstop; treat a configurable per-run LLM-call cap as separate hardening.

## Reproduction Assessment

Yes. The deterministic pre-aborted `AbortSignal` path is source-reproducible on current main: the signal is handled as aborted, but `activeSession.prompt(...)` is still evaluated before `abortable()` immediately rejects.

## Solution Assessment

Yes. The narrow maintainable fix is a pre-prompt abort guard plus an escaped-agent cleanup backstop; the suggested max-call cap is useful defense-in-depth but not required for the primary bug fix.
