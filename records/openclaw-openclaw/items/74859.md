## Summary

Keep open. Current main still has the pre-aborted signal race in the embedded runner, and https://github.com/openclaw/openclaw/pull/74979 remains the open closing PR for this issue.

## What This Changes

Report that a pre-aborted embedded-runner prompt can leave a background Agent loop making unbounded LLM calls after an interrupt or timeout exits.

## Best Possible Solution

Fix the prompt boundary so already-aborted and mid-preflight aborts are observed before Pi can start `Agent.prompt`, add focused regression coverage, and treat a per-run LLM-call cap as separate hardening.

## Reproduction Assessment

Yes. The deterministic pre-aborted `AbortSignal` path is source-reproducible on current main because the external abort is observed, but `activeSession.prompt(...)` is still evaluated before the abort wrapper can reject.

## Solution Assessment

Yes. The narrow maintainable fix is an abort-aware prompt-start boundary plus an escaped-agent cleanup backstop; the suggested max-call cap is useful defense-in-depth but not required for the primary bug fix.
