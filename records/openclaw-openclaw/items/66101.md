## Summary

Keep this PR open: it proposes a useful but security-sensitive main-agent exec exception, and the current branch still has concrete boundary bypasses that would allow mutating OpenClaw commands or skip enforcement on common Linux/path and pipe forms. Current main does not already contain an equivalent implementation, so this is not obsolete, but it should not merge without maintainer/security review and revision.

## What This Changes

The PR adds a main-agent host-level OpenClaw self-inspection exception in `src/agents/bash-tools.exec.ts` and focused path tests in `src/agents/bash-tools.exec.path.test.ts` for selected read-only commands and blocked mixed or mutating forms.

## Best Possible Solution

First confirm that `main` should have this narrow full-trust self-inspection exception at all. If yes, implement it with parser/segment-based OpenClaw detection, cross-platform binary/path recognition, a closed read-only argv policy that rejects all mutating flags and shell composition, and regression tests for pipes, Linux paths, doctor repair/fix forms, and memory status repair/reindex flags.

## Reproduction Assessment

Yes. The review has high-confidence static reproduction paths from the PR diff and current CLI contracts: `echo ok | openclaw update`, `/home/user/.npm-global/bin/openclaw update`, `openclaw doctor --repair`, and `openclaw memory status --fix` or `--index` exercise the affected boundary cases.

## Solution Assessment

No. The product direction may be acceptable after maintainer review, but this implementation is not the safest fix because it gates enforcement with a partial regex before doing structured command analysis and uses an argv allowlist that is not actually read-only.
