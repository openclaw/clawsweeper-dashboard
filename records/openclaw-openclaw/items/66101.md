## Summary

Keep open and do not merge yet: the PR changes a host command-execution boundary and the proposed guard has concrete bypasses for piped commands, Linux absolute paths, and mutating `doctor`/`memory status` flags. Current `main` does not already implement the requested exception, so this remains useful only after maintainer policy review and a safer revision.

## What This Changes

The PR adds a `main`-agent exception for selected host-level OpenClaw self-inspection commands under unsandboxed `tools.exec.security=full`, plus focused path tests for allowed and blocked forms.

## Best Possible Solution

If maintainers want this exception, implement it with parser/segment-based OpenClaw detection, platform-independent binary/path recognition, a closed read-only argv policy, and regression tests for pipes, Linux paths, wrapper invocations, `doctor` repair/token flags, and `memory status` repair/reindex flags.

## Reproduction Assessment

Yes. Static reproduction paths are high confidence from the PR diff and current CLI contracts: `echo ok | openclaw update`, `/home/user/.npm-global/bin/openclaw update`, `openclaw doctor --repair`, `openclaw doctor --fix=true`, and `openclaw memory status --fix` or `--index` exercise the boundary bypasses.

## Solution Assessment

No. The requested product direction may be acceptable after maintainer review, but this implementation is not the safest fix because it gates enforcement with a partial regex and allows argv forms that current CLI code treats as mutating.

  - body: `assertMainHostLevelOpenClawBoundary` only runs when the trigger regex matches, but the regex does not treat a single `|` as a command boundary. A full-trust command such as `echo ok | openclaw update` can therefore skip the new guard and run a mutating OpenClaw command.

  - confidence: 0.9

  - body: The doctor allowlist blocks only the exact `--fix` token. Current CLI registration also exposes `--repair`, `--force`, and `--generate-gateway-token`, and commander accepts value forms like `--fix=true`, so mutating doctor operations can pass this exception.
  - confidence: 0.96
- **[P1] Reject mutating memory status flags:** `src/agents/bash-tools.exec.ts:228-230`
