## What This Changes

Request to expose WhatsApp Baileys `contextInfo.isForwarded` and `contextInfo.forwardingScore` through normalized inbound context and plugin hook metadata.

## Best Possible Solution

Land an additive WhatsApp-owned pass-through that extracts forwarding once, stores typed fields on normalized inbound/canonical context, exposes them in `inbound_claim` and `message_received` metadata, and adds focused mapper plus WhatsApp tests.

## Reproduction Assessment

Yes, source-level. A Baileys message with `extendedTextMessage.contextInfo.forwardingScore` or `isForwarded` can reach `extractContextInfo`, but current main has no normalized carrier or plugin hook metadata field for those values.

## Solution Assessment

Yes as a direction, but it is not implemented yet. The narrow maintainable solution is an additive WhatsApp-owned metadata pass-through with maintainer/API judgment on the public hook shape.
