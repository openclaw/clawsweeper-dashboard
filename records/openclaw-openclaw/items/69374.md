## Summary

Keep open. Current main still drops WhatsApp forwarding metadata before plugin hook events, and the related open PR is the right implementation path to review or adapt rather than duplicating the work.

## What This Changes

Expose WhatsApp Baileys `contextInfo.isForwarded` and `contextInfo.forwardingScore` through normalized inbound handling so plugins can distinguish forwarded content from user-authored messages.

## Best Possible Solution

Land a narrow additive WhatsApp-owned change, preferably by reviewing or adapting https://github.com/openclaw/openclaw/pull/46291, that extracts forwarded status and score, carries them through normalized inbound context, and exposes them to both plugin hook events with focused regression tests.

## Reproduction Assessment

Yes. A high-confidence source-level path is to trace a Baileys inbound message with `extendedTextMessage.contextInfo.forwardingScore` or `isForwarded`; current main extracts `contextInfo` but has no carrier or hook metadata field for those values.

## Solution Assessment

Yes, as a direction. The narrow maintainable solution is an additive pass-through from the WhatsApp plugin into existing hook metadata, with maintainer review deciding the exact public event shape.
