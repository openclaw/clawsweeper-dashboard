## What This Changes

Expose WhatsApp `contextInfo.forwardingScore` and `contextInfo.isForwarded` to plugin inbound hook events.

## Best Possible Solution

Land a narrow additive WhatsApp-owned fix, preferably by reviewing or adapting https://github.com/openclaw/openclaw/pull/46291, that extracts forwarded status and score, carries them through normalized inbound context, and exposes them as untrusted channel metadata on both plugin hook events with focused tests.

## Solution Assessment

Yes. The narrow maintainable solution is an additive pass-through from the WhatsApp plugin into existing hook metadata, without a core migration or channel-wide redesign.
