## Summary

Keep open. Current main still does not expose WhatsApp Baileys `contextInfo.forwardingScore` or `contextInfo.isForwarded` to plugin inbound hooks, and related PR #46291 is still open as the natural implementation path.

## What This Changes

The issue requests carrying WhatsApp forwarded-message metadata from Baileys inbound envelopes through normalization into `inbound_claim` and `message_received` plugin events.

## Best Possible Solution

Land a narrow additive WhatsApp-owned fix, likely by reviewing or adapting #46291, that extracts `contextInfo.isForwarded` and `contextInfo.forwardingScore`, normalizes forwarded status when the flag is true or score is at least 1, carries both through the inbound message and finalized hook context, and exposes them as untrusted channel metadata for `inbound_claim` and `message_received` with focused mapper and WhatsApp inbound tests.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is available by tracing a Baileys inbound message with `extendedTextMessage.contextInfo.forwardingScore` or `isForwarded`; current main can locate `contextInfo` but has no typed or mapped path that reaches plugin hook events.

## Solution Assessment

Yes. The requested additive pass-through on the existing hook metadata surface is the narrow maintainable direction; the extraction should stay in the WhatsApp plugin while the generic hook mapper only carries normalized metadata.
