## Summary

Keep open. Current main and the latest v2026.5.18 release still drop WhatsApp forwarding metadata before plugin hook events, and the related open PR only targets agent prompt context rather than the plugin-facing `inbound_claim` / `message_received` contract.

## What This Changes

Expose WhatsApp Baileys `contextInfo.forwardingScore` and `contextInfo.isForwarded` to plugin `inbound_claim` and `message_received` hook events.

## Best Possible Solution

Add a WhatsApp-owned forwarding extractor and batch-safe hook metadata propagation for `inbound_claim` and `message_received`, coordinated with https://github.com/openclaw/openclaw/pull/46291 and covered by focused tests plus redacted live proof.

## Reproduction Assessment

Yes, source-level. Baileys exposes the forwarding fields and current main can extract raw `contextInfo`, but no current code path reads or carries those fields into WhatsApp normalized messages or plugin hook events.

## Solution Assessment

Yes as a direction, but the final shape needs maintainer judgment. The maintainable path is additive WhatsApp metadata propagation with explicit batch semantics and plugin-hook contract tests.
