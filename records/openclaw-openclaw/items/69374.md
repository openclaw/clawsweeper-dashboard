## What This Changes

## Best Possible Solution

Land a refreshed additive WhatsApp-owned pass-through, preferably by adapting https://github.com/openclaw/openclaw/pull/46291 or a follow-up, that extracts forwarding info once, carries it through normalized inbound context, and exposes it through plugin hook metadata with focused tests.

## Solution Assessment

Yes as a direction, but this should stay open until implemented. The narrow maintainable path is an additive WhatsApp metadata pass-through, with maintainer review deciding the public hook shape and how it relates to the open forwarding PR.
