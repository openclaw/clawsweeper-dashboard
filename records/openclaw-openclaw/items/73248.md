## Summary

Keep open. Current main still uses a flat skills catalog plus count/character budget mitigations; it has not shipped hierarchical discovery, tag grouping, lazy skill retrieval, or a plugin hook that can narrow the skill catalog per turn. The related hook-narrowing PR #68734 is still open, and #74100 tracks a broader skill-graph design rather than a shipped replacement for this request.

## What This Changes

The issue requests hierarchical, tag-based, or lazy skill discovery to reduce system prompt overhead and selection noise when many skills are installed.

## Best Possible Solution

Keep this as a core skills prompt/discovery design tracker. A maintainable end state should add an additive, documented narrowing or discovery contract, likely via hook-provided skill filtering, categories/tags, or a skill graph, while preserving deterministic ordering, explicit allowlists, prompt budgets, bundled plugin skills, and on-demand SKILL.md loading.

## Reproduction Assessment

Yes for the structural behavior: current main source and docs show a flat `<available_skills>` catalog with budgeted compact/truncation fallback. No for the reporter's exact token measurements because no concrete 80-plus-skill fixture, transcript, or provider usage log is attached.

## Solution Assessment

Unclear. The requested direction is valid, but the narrowest maintainable fix needs maintainer API/product judgment across #68734, tag/category config, lazy retrieval, and #74100 rather than an autonomous one-file repair.
