## Summary

Keep open: the PR’s Android APK artifact idea is narrow, but the current patch breaks the existing CI routing contract by running Android on every non-docs workflow run. Current main, docs, and changed-scope tests keep Android limited to Android-relevant changes or explicit manual dispatch, while official release APK distribution remains tracked separately on #9443.

## What This Changes

The PR changes `.github/workflows/ci.yml` to run Android CI for every non-docs run and upload APK artifacts from the `build-play` Android matrix task.

## Best Possible Solution

Preserve the existing Android routing gate and, if maintainers want transient CI artifacts, add the upload step only inside the already-scoped Android `build-play` path. Official downloadable APK release assets should be handled through release tooling tied to #9443, with matching docs/release approval.

Yes. The diff itself reproduces the routing regression by setting `runAndroid = !docsOnly`, and the PR head check-runs show Android jobs queued for a workflow-only change while current tests expect `runAndroid: false` for that path.

## Solution Assessment

No. Uploading an APK artifact from every non-docs CI run is not the narrowest maintainable solution; restoring the current scoped gate and limiting uploads to selected Android builds is the safer path.
