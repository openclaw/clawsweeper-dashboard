## Summary

Keep open: current main still has the bootstrap TLS fingerprint gap, and this recent external PR is a focused fix with sufficient terminal proof, green CI, and no blocking correctness or security findings from the read-only review.

## What This Changes

The PR adds a shared gateway TLS fingerprint resolver, reuses it from `callGateway` and client bootstrap, forwards `tlsFingerprint` into ACP/MCP/operator `GatewayClient` construction, and adds regression coverage plus a changelog entry.

## Best Possible Solution

Land the focused bootstrap propagation after normal maintainer review and merge policy, keeping TLS fingerprint precedence centralized and covered by targeted tests.

## Reproduction Assessment

Yes. Current main is source-reproducible: TLS fingerprint selection exists in `callGateway`, while `resolveGatewayClientBootstrap` and the ACP/MCP/operator clients do not carry `tlsFingerprint`; I did not run a live repro in this read-only review.

## Solution Assessment

Yes. Centralizing the fingerprint resolver and forwarding `bootstrap.tlsFingerprint` into each affected `GatewayClient` is the narrow maintainable fix, and it preserves the existing client pinning contract instead of adding new TLS policy.
