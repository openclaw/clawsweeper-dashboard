## Summary

Keep this PR open because it addresses a real gateway TLS bootstrap bug, but the branch is not mergeable as submitted: the useful TLS resolver/pass-through is bundled with unrelated release, dependency, workflow, generated protocol, and version churn.

## What This Changes

The PR adds a shared gateway TLS fingerprint resolver and forwards the resolved bootstrap fingerprint into ACP, MCP channel bridge, and operator approval GatewayClient construction.

## Best Possible Solution

Land a narrow rebased TLS bootstrap patch that keeps the shared resolver, ACP/MCP/operator pass-through, and focused tests, while dropping unrelated release, dependency, generated protocol, and version churn.

## Reproduction Assessment

Yes. Source inspection on current main shows `GatewayClient` and `callGateway` support TLS fingerprints, while bootstrap helper consumers do not receive or pass one, so the local self-signed WSS failure path is source-reproducible without a live gateway run.

## Solution Assessment

Yes for the narrow shared resolver and pass-through, but no for the branch as submitted. The maintainable solution is to rebase or cherry-pick only the TLS files and regression tests onto current main.
