## Summary

Keep open for normal PR review and merge gates. The current main code path is source-reproducible, and the PR fixes it by centralizing TLS fingerprint resolution, forwarding the prepared fingerprint into ACP/MCP/approval GatewayClient construction, and adding focused regression coverage with supplied terminal proof.

## What This Changes

The PR adds a shared gateway TLS fingerprint resolver, uses it from client bootstrap and callGateway, forwards bootstrap.tlsFingerprint into ACP, MCP, and operator approval clients, and adds regression tests plus a changelog entry.

## Best Possible Solution

Land the focused bootstrap propagation after normal CI and maintainer review, keeping TLS fingerprint precedence centralized and covered by the new targeted tests.

## Reproduction Assessment

Yes: this is source-reproducible on current main because local TLS fingerprint selection exists only in callGateway while ACP/MCP/approval clients use resolveGatewayClientBootstrap and construct GatewayClient without tlsFingerprint. I did not run a live repro during this read-only review.

## Solution Assessment

Yes: centralizing the fingerprint resolver and carrying bootstrap.tlsFingerprint into each GatewayClient is the narrowest maintainable fix. It avoids scattered duplicate precedence rules and keeps the existing GatewayClient pinning contract unchanged.
