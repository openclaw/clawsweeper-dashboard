## What This Changes

The issue requests StepFun image-generation provider registration so `image_generate` can route `stepfun-plan/step-image-edit-2`.

## Best Possible Solution

Add StepFun image-generation support inside the bundled StepFun plugin after confirming the provider id and endpoint mapping, then declare the manifest contract, document the supported model, and add focused request/registration tests.

## Reproduction Assessment

Yes for the root cause from source: `image_generate` can only resolve registered `imageGenerationProviders`, and current StepFun registers none. No credentialed StepFun API run was performed, so the exact silent async failure remains unverified.

## Solution Assessment

Unclear until maintainers confirm the provider id/base URL contract. The right surface is plugin-owned image-provider registration, but the implementation should not hard-code the report's `stepfun-plan` mapping without checking StepFun's documented `/v1/images` endpoints.
