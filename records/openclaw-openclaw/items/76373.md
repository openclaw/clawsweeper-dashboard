Keep open. Current main still sends bare `brave` through bundled-source lookup and then raw npm fallback in packaged installs, while an open closing PR already targets that alias gap; the later `web_search` comments look related but partly overlap the separate #76416/#76418 runtime-provider thread.

## What This Changes

Restore the 2026.5.2 Brave plugin install path so users can install the official external Brave plugin after it was removed from the bundled package.

## Best Possible Solution

Review and land one narrow official-id alias resolution fix, then keep any remaining runtime-provider scoping problem on #76416/#76418 if it still reproduces.

## Reproduction Assessment

Yes. Source inspection shows packaged installs exclude bundled Brave, `plugins install brave` only consults bundled sources before npm fallback, and the official catalog already maps `brave` to `@openclaw/brave-plugin`.

## Solution Assessment

Yes for the related PR direction: resolving exact official external plugin ids before raw npm fallback is the narrowest maintainable fix, as long as explicit npm specs stay npm-only. The later `web_search` provider symptom should stay on its separate runtime issue if it remains after install.
