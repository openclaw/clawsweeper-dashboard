## What This Changes

The PR adds per-row guarding for `plugins.uiDescriptors` so unreadable plugin-owned Control UI descriptor rows are skipped while healthy descriptor rows are still returned.

## Best Possible Solution

Land one focused Gateway descriptor-projection guard after maintainer review and broad changed-gate or CI proof, then close the duplicate sibling PR as superseded by the chosen merged implementation.

## Reproduction Assessment

Yes. Source inspection shows current main dereferences each `controlUiDescriptors` row while building the Gateway response, so a throwing descriptor getter can abort the whole response; I did not run tests because this is a read-only review.

## Solution Assessment

Yes. The Gateway response projection is the narrowest maintainable boundary because plugin registration already validates normal descriptor rows, while this PR isolates unreadable rows at projection time.
