## Summary

Keep open: current main and v2026.6.6 still have the unguarded Gateway descriptor projection, and this PR’s row-level guard is a focused fit for the gateway/plugin boundary. ## What This Changes

The PR adds row-level isolation to `plugins.uiDescriptors` so unreadable Control UI descriptor registrations are skipped while healthy descriptor rows are preserved, with a focused contract regression.

## Best Possible Solution

Land exactly one focused Gateway projection guard with the contract regression after maintainer review and broad validation, while keeping registry validation as the canonical plugin contract boundary.

## Reproduction Assessment

Yes from source inspection: current main and v2026.6.6 dereference every Control UI descriptor row in one unguarded map, so a throwing descriptor getter can abort the whole response. I did not execute tests in this read-only review, but the PR body reports the focused red repro.

## Solution Assessment

Mostly yes: row-level isolation in the Gateway projection is the right narrow boundary because normal registry registration already validates descriptor contracts. Maintainers should still choose between this PR and https://github.com/openclaw/openclaw/pull/89794 rather than landing both overlapping implementations.
