## Summary

Keep open: the current-main bug is real, but this PR still misclassifies a valid gateway send receipt shape as a missing delivery result and it lacks after-fix real behavior proof.

## What This Changes

The branch adds a message-tool guard that returns an error for non-dry-run send results without a nested delivery messageId and adds regression coverage for missing, dry-run, and nested-success paths.

## Best Possible Solution

Keep the missing-delivery error guard, but recognize all valid receipt shapes or scope it to core sends, then add focused regression coverage and redacted real-run proof.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: inject a non-dry-run send result with no toolResult and no delivery result, and current main falls through to jsonResult(result.payload). I did not run tests in this read-only review.

## Solution Assessment

No. The direction is narrow and maintainable, but the current PR must accept the existing top-level gateway payload.messageId shape or avoid applying the guard to gateway plugin sends before it is safe.
