## Summary

Keep open: this PR targets a real current-main message-tool bug, but the patch still turns valid gateway-backed sends into errors and the external PR still lacks after-fix real behavior proof.

## What This Changes

The branch adds a message-tool guard that returns a tool error for non-dry-run sends without a nested delivery messageId and adds regression tests.

## Best Possible Solution

Keep the missing-delivery guard, but make it respect all valid receipt shapes or scope it to core sends that truly lack a receipt, then require redacted after-fix runtime proof.

## Reproduction Assessment

Yes, from source inspection. Current main falls through to jsonResult(result.payload) when no toolResult is present, while MessageSendResult.result is optional and best-effort tests cover result: undefined.

## Solution Assessment

No. The direction is narrow and reasonable, but this patch must either scope the new error guard to core sends or accept top-level gateway payload.messageId before it is safe to merge.
