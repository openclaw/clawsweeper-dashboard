## Summary

Keep open: the reported bug is source-reproducible on current main, but this PR still has a blocking gateway-send regression and lacks after-fix real behavior proof from the reported Feishu or equivalent delivery path.

## What This Changes

The branch adds a message-tool guard and tests so non-dry-run sends without a nested delivery messageId return a status error instead of the raw payload.

## Best Possible Solution

Keep the missing-delivery guard concept, but make it respect existing core and gateway receipt shapes before requiring redacted real behavior proof for the Feishu or equivalent path.

## Reproduction Assessment

Yes. From source inspection, current main returns jsonResult(result.payload) when runMessageAction produces no toolResult, while MessageSendResult.result is optional and best-effort coverage expects result: undefined.

## Solution Assessment

No. The direction is narrow and reasonable, but this implementation must either scope the guard to core sends or accept top-level gateway payload messageIds before it is safe to merge.
