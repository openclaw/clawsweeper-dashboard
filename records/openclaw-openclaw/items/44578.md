## Summary

Keep open: this is a plausible current-main bug fix, but the proposed patch is not merge-ready because it can mark successful gateway-backed sends as failed and the external PR still lacks after-fix real behavior proof.

## What This Changes

The PR adds a message-tool guard and regression tests so non-dry-run sends without a concrete delivery messageId return a status error instead of the raw outbound payload.

## Best Possible Solution

Keep the missing-delivery guard, but make it respect existing delivery receipt shapes and ownership boundaries, then prove the fixed behavior with a redacted real Feishu or equivalent message-tool run.

## Reproduction Assessment

Yes, from source inspection: current main can return jsonResult(result.payload) for a non-dry-run core send whose MessageSendResult has no result.messageId. I did not run a live Feishu reproduction, but the current code and existing best-effort test shape give a high-confidence source reproduction path.

## Solution Assessment

No. The direction is narrow and reasonable for core direct sends, but this implementation must either scope the error guard to core sends or recognize top-level gateway payload messageIds before it is safe to merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Accept gateway send messageIds before failing delivery:** `src/agents/tools/message-tool.ts:683`
  - body: This helper only looks for `sendResult.result.messageId` or `payload.result.messageId`. Gateway-backed plugin sends currently return the gateway payload directly, and existing coverage expects `{ ok: true, messageId: "gw-send-1" }` at the top level with `handledBy: "plugin"`; Telegram routes send actions through that gateway path. With this patch, a successful gateway send is classified as `status: "error"`, so successful Telegram-style sends would look failed. Please either scope the missing-delivery guard to core sends or recognize top-level `payload.messageId`.
  - confidence: 0.9
