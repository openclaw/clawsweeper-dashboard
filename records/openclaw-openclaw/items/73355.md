  - body: This test seeds an empty store, so `armTimer(state)` leaves `state.timer` null and CI fails at the first assertion. Even after seeding a job, directly calling `armTimer` would not exercise the new `.catch` branch; drive a rejecting timer path and assert the re-arm.
  - confidence: 0.96
- **[P2] Remove or use the imported onTimer symbol:** `src/cron/service/timer.watchdog.test.ts:3`
  - body: `onTimer` is imported but never used, which is another `check-lint` failure on this branch. Either remove it or use it as part of a real catch-path test.
  - confidence: 0.94
- **[P2] Isolate the watchdog from pending-timer test flows:** `src/cron/service/ops.ts:201`
  - body: Starting the interval from every `cron.start()` means existing fake-timer tests that call `vi.runOnlyPendingTimersAsync()` can run the watchdog too. CI shows the cron-expression scheduling test now advances `nextRunAtMs` by five minutes instead of one, so the watchdog needs test isolation or the affected timer flow needs to avoid running it as an unrelated pending timer.
  - confidence: 0.84
