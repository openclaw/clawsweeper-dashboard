## Summary

Keep open: current main and v2026.6.8 still detach session-backed media generation and expose no synchronous override; related delivery fixes reduce some pain but do not decide the model-facing execution contract.

## What This Changes

Request to add an opt-in synchronous execution mode to `music_generate`, `image_generate`, and `video_generate` so session-bound callers can wait for generated media inline.

## Best Possible Solution

Keep repaired async media delivery as the default, and only add a distinctly named opt-in synchronous execution mode after maintainers accept the cross-tool schema, timeout, cancellation, and duplicate-guard contract.

## Reproduction Assessment

Partly. Source inspection confirms session-backed media calls still detach and no synchronous override exists, but I did not establish a live subagent orphaned-completion failure on current main.

## Solution Assessment

Unclear. A synchronous media mode is plausible, but maintainers should first decide whether it is better than continuing to repair async completion delivery, and the proposed `background` field name is not viable for `image_generate`.
