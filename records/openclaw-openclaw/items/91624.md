## Summary

Current main and v2026.6.5 intentionally detach session-backed image, music, and video generation into a requester-bound task lifecycle, while no-session or task-registration failure remains inline; the requested per-call synchronous mode is still absent, but the reported orphaning or wrong-session behavior is not proven and the proposed field conflicts with an existing image option, so this needs a maintainer execution-contract decision rather than cleanup closure or automated implementation.

## What This Changes

Add an optional model-facing control that lets media-generation tool calls wait inline instead of detaching in session-backed runs.

## Best Possible Solution

Preserve one canonical requester-bound media task lifecycle, reproduce and repair its subagent completion routing if it is defective, and add a distinctly named inline execution mode only if maintainers explicitly choose to support long-running synchronous tool turns.

## Reproduction Assessment

Partially: source inspection gives a high-confidence path showing that session-backed media calls detach, but there is no high-confidence reproduction of the claimed orphaned, duplicate, or wrong-session completion on current main.

## Solution Assessment

No, not as proposed: the field name conflicts with existing image semantics and a new synchronous path would bypass a deliberate shared lifecycle before that lifecycle has been shown to be the root defect.
