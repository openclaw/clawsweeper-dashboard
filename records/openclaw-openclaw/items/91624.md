Keep the repaired async background lifecycle as the default, and only add a clearly named synchronous media execution mode if maintainers accept its timeout, cancellation, duplicate-guard, schema, docs, and cross-tool semantics.

## Solution Assessment

Unclear. A synchronous mode is plausible, but maintainers should first decide whether adding a new model-facing execution contract is better than continuing to repair async completion delivery.
