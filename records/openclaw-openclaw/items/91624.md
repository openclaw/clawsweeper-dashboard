## Summary

Keep open: current main and the latest release still make session-backed media generation detach automatically and expose no model-facing synchronous execution override; the related webchat completion fix repaired one async delivery failure but did not implement this requested tool contract.

## What This Changes

The issue requests a model-facing synchronous execution option for media generation tools so session-bound callers can block until generation completes instead of always detaching.

## Best Possible Solution

Keep repaired async media delivery as the default and add a distinct, maintainer-approved opt-in synchronous execution mode only after the cross-tool schema, timeout, cancellation, and duplicate-guard contract is settled.

## Reproduction Assessment

Partly. Source inspection confirms session-backed media calls still detach and no synchronous override exists, but this review did not establish a live subagent orphaned-completion failure on current main.

## Solution Assessment

Unclear. A synchronous execution mode is plausible, but the proposed `background` field name is not the best shape for `image_generate`, and maintainers should decide the cross-tool lifecycle contract first.
