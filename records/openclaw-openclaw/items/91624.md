## Summary

Keep open: current main and the latest release still lack a supported synchronous execution override for session-backed media generation. The related merged fixes improve async media completion delivery, but they do not settle the requested model-facing execution contract.

## What This Changes

The issue asks for an optional media-tool parameter that lets session-bound callers force inline media generation instead of detaching to a background task.

## Best Possible Solution

Keep async background media generation as the default, and add a clearly named synchronous execution mode only if maintainers accept its cross-tool schema, timeout, cancellation, duplicate-guard, docs, and test contract.

## Reproduction Assessment

Partly. Source inspection on current main confirms session-backed media calls still detach and no synchronous override exists, but I did not reproduce the full orphaned subagent completion failure in a live run.

## Solution Assessment

Unclear. A synchronous mode is plausible, but background:false is not the best obvious shape because it collides with image_generate background output semantics and needs a maintainer product decision first.
