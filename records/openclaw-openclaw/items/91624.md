## Label Justifications

- `P3`: This is a concrete but speculative feature request for media-generation ergonomics rather than a confirmed regression in a documented workflow.
- `impact:session-state`: The reported pain is detached media completion state becoming orphaned, stale, duplicated, or associated with the wrong session in subagent workflows.

## Summary

The requested synchronous media-generation mode is still absent on current main: session-backed image, music, and video generation still detach solely because an agent session key exists, and the issue asks for a new model-facing tool contract that needs maintainer judgment.

## What This Changes

Add an optional media tool parameter that lets session-bound callers run image, music, or video generation synchronously instead of always detaching to background work.

## Best Possible Solution

Choose a single media-generation execution contract that either fixes subagent completion routing within the existing background lifecycle or adds a clearly named synchronous mode with shared schema, tests, and lifecycle proof across image, music, and video.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Source inspection confirms that any non-empty agent session key forces the media tools onto the detached path, but I did not reproduce the reported orphaned subagent completion behavior in a live run.

## Solution Assessment

Is this the best way to solve the issue? Unclear: an explicit synchronous mode is plausible, but maintainers should first decide whether blocking long media jobs is better than improving the existing background completion and fallback lifecycle.
