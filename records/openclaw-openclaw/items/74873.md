## Summary

Keep open for maintainer review. Current main still lacks any shutdown drain-manifest producer, and the PR head is a narrow implementation with focused tests and green exact-head check runs; the remaining decision is whether this runId-based manifest is the right contract for the broader recovery work tracked in #40712.

## What This Changes

The PR snapshots active chat runs during gateway shutdown, persists/reads/deletes a versioned `draining-sessions.json` manifest, wires the registry into the close handler, adds focused tests, and marks runtime-secrets reload system events untrusted.

## Best Possible Solution

Land the narrow shutdown manifest producer once a maintainer confirms the runId-based manifest contract, then implement the boot-time consumer and recovery policy under #40712/AI-588.

## Reproduction Assessment

Yes. Source inspection on current main reproduces the missing behavior: active chat-run state is cleared during shutdown without any persisted drain manifest or registry snapshot API.

## Solution Assessment

Yes for this slice. The PR uses the existing chat-run registry, snapshots before clear, persists a versioned local manifest, and leaves resume policy to the separate boot-time consumer instead of bundling broader recovery behavior here.
