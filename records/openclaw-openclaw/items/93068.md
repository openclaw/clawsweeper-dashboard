## Summary

Keep open: current main and the latest release still lack a gateway-wide SSRF policy, MS Teams has no equivalent network SSRF config surface, and OpenAI OAuth token refresh still uses the SSRF guard outside the model-provider request policy path. This is real remaining work, but it needs maintainer/security product review because the requested global inheritance would add a high-power config surface across multiple egress owners.

## What This Changes

Request a gateway-level SSRF policy/default that subsystem-specific web, browser, provider, auth, and channel egress paths can inherit while preserving per-subsystem overrides.

## Best Possible Solution

Define a narrow, opt-in global SSRF policy contract with explicit subsystem override semantics, security documentation, and regression coverage for model transport, provider auth refresh, and channel SDK/media paths.

## Reproduction Assessment

Yes at source level: current main exposes only per-subsystem SSRF controls, MS Teams lacks a network SSRF config surface, and OpenAI OAuth token refresh calls fetchWithSsrFGuard without model-provider request policy. No live fake-IP DNS run was performed.

## Solution Assessment

Unclear: a global inherited policy may be the right operator UX, but maintainers need to choose the security boundary, inheritance rules, and plugin/provider/channel ownership before implementation. Per-plugin parity alone would not cover the OAuth refresh gap called out in the discussion.
