## Summary

Keep open: current main and the latest release still lack a gateway-level SSRF policy surface, and the MS Teams SDK plus OpenAI OAuth refresh paths do not consume an operator SSRF override. The request is valid but security/config sensitive, so it needs maintainer product/security review rather than an automatic fix PR.

## What This Changes

Request a default-off gateway SSRF policy that can be inherited by subsystems, with per-subsystem overrides and at least MS Teams parity for trusted fake-IP/private-network environments.

## Best Possible Solution

Define a single explicit, default-off SSRF policy inheritance model with safe per-subsystem overrides, then cover MS Teams SDK calls, provider OAuth refresh, model transport, browser/web fetch, docs, and upgrade behavior with focused tests.

## Reproduction Assessment

No live high-confidence fake-IP reproduction was run here. Source inspection does confirm the requested config gap: there is no gateway SSRF policy, MS Teams SDK app creation has no policy input, and OpenAI OAuth refresh uses the guard without a configurable policy.

## Solution Assessment

Unclear until maintainers choose the policy boundary. A global default could be the right maintainable fix if it is explicit and default-off, while an MS Teams-only knob would not cover the provider OAuth-refresh data point.
