## What This Changes

The branch imports `pathToFileURL` and uses it for `scripts/generate-base-config-schema.ts` direct-entrypoint detection.

## Best Possible Solution

Land the narrow `pathToFileURL` guard fix for this generator, with optional focused guard coverage; any broader sweep of similar script entrypoint guards should be handled separately.

## Reproduction Assessment

Yes, via source inspection and a Node API probe rather than a full Windows run. Current main uses `new URL(path, "file://")`, and the probe shows that shape does not produce the `file:///D:/...` URL that a Windows module URL comparison needs.

## Solution Assessment

Yes. For this scoped bug, `pathToFileURL(process.argv[1] ?? "").href` is the narrowest maintainable fix and matches Node's documented API plus existing repo precedent.
