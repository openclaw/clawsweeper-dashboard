## Summary

Keep this PR open. Current main still has the Windows-sensitive entrypoint guard, and the proposed patch is a narrow, correct fix for the scoped generator script rather than obsolete work.

## What This Changes

The PR updates `scripts/generate-base-config-schema.ts` to import `pathToFileURL` from `node:url` and use it for direct-entrypoint detection.

## Best Possible Solution

Land a narrow `pathToFileURL` entrypoint fix for this generator, preferably with a focused guard test or reuse of the existing generated-output CLI helper; any broader sweep of similar script guards should be separate.

## Reproduction Assessment

Yes. The direct comparison can be reproduced with a Windows-style script path, where `new URL(path, "file://")` does not produce the `file:///D:/...` shape used by module URLs, while `pathToFileURL` does.

## Solution Assessment

Yes. For this PR's scoped bug, using Node's `pathToFileURL` is the narrowest maintainable fix and matches existing repo precedent; adding regression coverage would strengthen it but is not a safer alternative.
