- `impact:session-state`: The core failure mode is session/context state drift between OpenClaw session JSONL and the live Claude CLI transcript.
- `impact:data-loss`: Accepting an empty compaction summary and rotating the session can silently discard the user's prior conversation context.

## Summary

Keep open: current main still has source-level support for the reported Dashboard context-meter and Claude CLI manual-compaction failure mode, and no merged or viable canonical PR was found that resolves the whole issue.

## What This Changes

The issue asks for Dashboard context usage to reflect current live context and for Claude CLI `/compact` to either preserve context or fail visibly instead of rotating after an empty summary.

## Best Possible Solution

Dashboard should display a fresh current-context estimate only, and manual Claude CLI compaction should summarize or import the live Claude CLI transcript, or fail visibly without rotating or checkpointing an empty placeholder summary.

## Reproduction Assessment

No, not with a live 1M-token Claude CLI session in this review. Source inspection does show current Dashboard math and manual compaction source selection matching the reported failure mode closely enough to keep the issue open.

## Solution Assessment

Yes for the direction, but the exact fix should be reviewed carefully. The maintainable path is to use one canonical current-context source for Claude CLI sessions and reject empty placeholder summaries, not to patch only Dashboard copy.
