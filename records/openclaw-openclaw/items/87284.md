## Summary

Keep open: current main has partial safeguards for the old silent-empty-compaction path, but Claude CLI over-cap recovery is still native-owned/no-op from OpenClaw's pre-turn compaction path and needs maintainer product review plus live repro before this can be closed.

## What This Changes

The issue reports that Dashboard context usage and `/compact` can mislead users and lose Claude CLI session context once the native Claude transcript is over the 1M-token cap.

## Best Possible Solution

Keep this issue open until maintainers choose the Claude CLI native-compaction contract and implement a clear over-cap outcome: successful native recovery, explicit refusal with recovery/export guidance, or a tested fallback that never rotates away useful context silently.

## Reproduction Assessment

No high-confidence live current-main reproduction was established. Source inspection does show the remaining boundary: Claude CLI pre-turn compaction is deferred as native-owned/no-op, while manual Dashboard compaction now reports a skipped reason when the OpenClaw transcript has no real conversation messages.

## Solution Assessment

No, current main is only a partial safety improvement, not the complete best fix. The best fix needs an explicit maintainer decision for how OpenClaw should handle already-over-cap Claude CLI native transcripts instead of only avoiding the old silent empty-checkpoint path.
