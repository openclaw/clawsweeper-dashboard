## What This Changes

The issue requests Feishu topic-group replies to stay in the existing topic when events provide `thread_id` without `root_id`.

## Best Possible Solution

Review, land, or explicitly reject the canonical Feishu anchor-routing fix, ensuring thread_id-only topic messages resolve a safe final reply anchor while preserving normal group reply behavior.

## Solution Assessment

No for current main: the existing fallback does not solve the reporter’s thread_id-only topic case. The best fix should stay in the Feishu plugin and preserve separate anchors for routing, typing, and final reply delivery.
