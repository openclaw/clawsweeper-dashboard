## What This Changes

The issue asks Feishu/Lark topic-group bot replies to stay in the existing topic when Feishu events provide `thread_id` but omit `root_id` and `parent_id`.

## Best Possible Solution

Finish, repair, or replace https://github.com/openclaw/openclaw/pull/78964 with a narrow Feishu plugin fix that resolves the topic send anchor for thread_id-only events while preserving current normal-group, P2P, typing-target, message-tool, media, and subagent delivery invariants.
