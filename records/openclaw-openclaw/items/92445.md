  - body: `sendCardFeishu` now runs recursive redaction over every string in the card object.   - confidence: 0.9
- **[P2] Keep streaming merge state canonical before redaction:** `extensions/feishu/src/streaming-card.ts:462-488`
  - body: These calls redact the merged stream text before assigning `pendingText` and `currentText`, but `mergeStreamingText` expects successive full model snapshots. After an email becomes `[email redacted]`, the next unredacted snapshot no longer contains the previous stored state, so the merge can append instead of replacing and show duplicate content. Keep unredacted merge state and redact only the payloads sent to Feishu.
  - confidence: 0.87
