## Summary

Keep open. Current main still lacks the requested Discord parent-channel `thread.requireMention` override, and open PR #76058 already uses closing syntax for this issue, so the issue should stay open until that PR is reviewed and merged or closed.

## What This Changes

Request Discord per-parent-channel `thread.requireMention` config so ordinary thread replies can skip mention gating while parent channels still require mentions.

## Best Possible Solution

Review and land or close PR #76058; the desired end state is a backward-compatible Discord config/schema/runtime/docs/test change that applies the thread override during preflight before typing can start.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: with parent `requireMention: true`, an unmentioned ordinary thread message inherits the parent channel config, while the proposed nested `thread.requireMention` key is absent from the strict schema and never read by the mention gate.

## Solution Assessment

Yes. The narrow maintainable fix is in the Discord plugin/config surface: add the nested channel thread policy, document it, and evaluate it in preflight before the existing no-mention drop path.
