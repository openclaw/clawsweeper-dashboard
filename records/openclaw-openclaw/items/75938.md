## Summary

Keep open. Current main has Discord thread detection, parent-channel config inheritance, and bot-owned auto-thread mention bypasses, but it does not support the requested reusable parent-channel `thread.requireMention` override.

## What This Changes

Add a Discord guild-channel `thread.requireMention` override so threads can use a different mention gate from their parent channel.

## Best Possible Solution

Add a backward-compatible Discord plugin config surface that evaluates `guilds.<guild>.channels.<parent>.thread.requireMention` for thread messages after parent detection and before mention gating, while preserving existing parent inheritance, per-thread-ID overrides, and bot-owned auto-thread behavior.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: with parent `requireMention: true`, an unmentioned thread message inherits the parent config and is dropped; the proposed nested `thread.requireMention: false` shape is not in the strict schema, so it cannot currently change that behavior.

## Solution Assessment

Yes. The narrowest maintainable solution is in the Discord plugin/config contract because preflight already resolves thread parent context before mention filtering and before the typing pipeline is started.
