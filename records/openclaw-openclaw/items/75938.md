## Summary

Keep open. Current main still lacks the requested Discord parent-channel `thread.requireMention` override, and #76058 is open with closing syntax for the active implementation path.

## What This Changes

Add Discord parent-channel thread-specific `requireMention` configuration so ordinary threads can use a different mention gate than their parent channel.

## Best Possible Solution

Review and land or close #76058; the final implementation should align Discord config types, strict schema, generated metadata, docs, runtime preflight mention gating, and regression tests.

## Reproduction Assessment

Yes. Source inspection on current main shows the proposed nested `thread.requireMention` key is not part of the strict Discord guild-channel schema, and ordinary thread mention gating resolves to inherited channel/guild `requireMention`.

## Solution Assessment

Yes. The narrow maintainable direction is a backward-compatible Discord plugin config extension plus aligned schema/docs/tests, and #76058 is already attempting that path.
