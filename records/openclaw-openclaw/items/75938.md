## Summary

Keep open. Current main still has Discord thread detection, parent-channel config inheritance, and early no-mention preflight drops, but it does not expose or apply a reusable parent-channel `thread.requireMention` override for ordinary Discord threads.

## What This Changes

Add thread-specific Discord `requireMention` configuration so parent channels can require mentions while ordinary thread replies can continue without repeated mentions.

## Best Possible Solution

Add a backward-compatible Discord plugin config surface under guild channel entries, evaluate it for real thread messages after parent resolution and before mention filtering, and document the behavior without changing broader channel access semantics.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: with a parent channel `requireMention: true`, an unmentioned thread message inherits that parent setting and is dropped; the proposed nested `thread.requireMention: false` key is not accepted by the strict schema.

## Solution Assessment

Yes. The narrowest maintainable solution is a Discord plugin config/schema/docs/test change plus a preflight mention-gating branch, preserving current parent inheritance, per-thread-ID overrides, and bot-owned auto-thread behavior.
