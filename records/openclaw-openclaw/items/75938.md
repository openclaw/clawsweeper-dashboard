## Summary

Keep open. Current main still lacks a reusable Discord parent-channel thread mention override: thread messages resolve the concrete thread or parent channel config, then use the resolved channel or guild `requireMention` value, so the requested thread default remains valid.

## What This Changes

Add Discord configuration that lets a parent channel require mentions while its threads can allow natural unmentioned follow-up messages.

## Best Possible Solution

Add a backward-compatible Discord plugin config surface for parent-channel thread mention policy and evaluate it during preflight before mention gating and typing behavior.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: configure a parent channel with `requireMention: true`, send an unmentioned thread message, and current preflight resolves the parent config and drops the message.

## Solution Assessment

Yes. The narrowest maintainable fix belongs in the Discord plugin config/schema/preflight path because thread parent context already exists before mention gating and no core API seam appears missing.
