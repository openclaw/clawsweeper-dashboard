## Summary

Keep open. Current main still lacks the requested Discord parent-channel `thread.requireMention` override, and an open contributor PR already closes this issue, so the issue should remain open until that PR is reviewed and merged or closed.

## What This Changes

Add thread-specific Discord `requireMention` configuration so parent channels can require mentions while ordinary thread replies do not.

## Best Possible Solution

Review and land or close PR #76058; once the fix merges, this issue can close as implemented by the linked PR.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: with parent `requireMention: true`, an unmentioned ordinary thread message follows the parent config into `resolveDiscordShouldRequireMention`, while the proposed nested `thread.requireMention` key is not accepted by the strict schema.

## Solution Assessment

Yes. The narrowest maintainable solution is a backward-compatible Discord plugin config/schema/runtime/docs change that evaluates a parent-channel thread override before the existing mention preflight drop.
