## Summary

Keep open. Current main has Discord thread detection, parent-channel config fallback, and early mention filtering, but it does not expose or evaluate the requested reusable parent-channel `thread.requireMention` override.

## What This Changes

Add a Discord parent-channel thread override so thread messages can use a different `requireMention` policy than their parent channel.

## Best Possible Solution

Add typed/schema/docs support and tests for `guilds.<guild>.channels.<parent>.thread.requireMention`, then resolve it in the Discord plugin preflight path while preserving parent inheritance, exact thread-ID entries, and bot-owned auto-thread behavior.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: with parent `requireMention: true`, an unmentioned thread message inherits the parent config and is dropped; the proposed nested `thread.requireMention: false` shape is not accepted by the strict schema and cannot currently alter that behavior.

## Solution Assessment

Yes. The narrowest maintainable fix is Discord-plugin-owned config and preflight mention resolution, because parent thread context is already known before mention filtering and before the Discord typing pipeline starts.
