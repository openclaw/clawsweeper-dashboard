## Summary

Keep open. Current main still lacks a Discord parent-channel `thread.requireMention` override, and the linked open PR is the active implementation candidate rather than a reason to close before merge.

## What This Changes

Add a Discord guild-channel `thread.requireMention` override so ordinary thread messages can use a different mention gate than their parent channel.

## Best Possible Solution

Land a reviewed equivalent of #76058 that adds the backward-compatible Discord config/schema/docs surface and applies the thread override in preflight before mention filtering.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: the proposed nested channel `thread` key is not accepted by the strict schema, and ordinary thread messages currently inherit the parent/channel `requireMention` value before the no-mention drop.

## Solution Assessment

Yes. The narrow maintainable direction is Discord-plugin-owned config, schema, docs, and preflight runtime/test coverage, preserving existing parent inheritance and bot-owned auto-thread behavior.
