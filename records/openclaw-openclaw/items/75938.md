## Summary

Keep open. Current main and the latest release still only support Discord guild/channel-level `requireMention`, while https://github.com/openclaw/openclaw/pull/76058 is an open, currently conflicting implementation candidate for this exact thread override.

## What This Changes

Add a Discord per-channel thread-level `requireMention` override so thread replies can use a different mention requirement from their parent channel.

## Best Possible Solution

Review, land, or replace https://github.com/openclaw/openclaw/pull/76058 with an additive Discord-owned `thread.requireMention` policy that preserves current defaults, updates schema/docs/tests, and explicitly coordinates with the related autoThread follow-up policy.

## Reproduction Assessment

Yes, source-level. Current main has no Discord `thread.requireMention` config field, and the preflight mention gate only uses the matched channel or guild `requireMention` before dropping unmentioned guild messages.

## Solution Assessment

Unclear until maintainers choose the public API. The linked PR is a plausible narrow implementation, but it adds a compatibility-sensitive config surface and currently needs conflict resolution and maintainer review.
