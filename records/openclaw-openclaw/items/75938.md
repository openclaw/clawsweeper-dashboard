## Summary

Keep open. Current main has Discord thread detection, parent-channel config inheritance, and bot-owned auto-thread mention bypasses, but it still has no reusable parent-channel `thread.requireMention` override for ordinary Discord threads.

## What This Changes

Request thread-scoped Discord mention gating so a parent channel can require @mentions while its threads allow natural follow-up messages without repeated mentions.

## Best Possible Solution

Add a backward-compatible Discord plugin channel config surface that evaluates parent-channel `thread.requireMention` for thread messages after parent resolution and before mention gating, while preserving existing parent inheritance, per-thread-ID overrides, wildcard fallback, and bot-owned auto-thread behavior.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: with parent `requireMention: true`, an unmentioned thread message inherits the parent config and is dropped, and the proposed nested `thread.requireMention: false` shape is not accepted by the current strict schema.

## Solution Assessment

Yes. The narrowest maintainable fix belongs in the Discord plugin config/runtime path because preflight already knows the thread parent before mention filtering and before the typing pipeline starts.
