## Summary

Keep open. Current main and v2026.6.10 still only support Discord guild/channel-level `requireMention`; https://github.com/openclaw/openclaw/pull/76058 is the open implementation candidate for this exact thread override and should be reviewed, replaced, or closed by maintainers before this issue is resolved.

## What This Changes

Request thread-specific Discord `requireMention` configuration so parent channels can require mentions while child threads can use a different mention gate.

## Best Possible Solution

Review, land, or replace https://github.com/openclaw/openclaw/pull/76058 with an additive Discord-owned `thread.requireMention` policy that preserves current defaults, updates schema/docs/tests, and coordinates with the related autoThread follow-up policy.

## Reproduction Assessment

Yes, source-level. Current main resolves Discord thread messages through parent/channel config and `resolveDiscordShouldRequireMention`, which has no nested thread-specific `requireMention` branch.

## Solution Assessment

Unclear until maintainers choose the public API. The linked PR is a plausible narrow implementation for this request, but it adds compatibility-sensitive Discord config surface and currently needs conflict resolution and maintainer acceptance.
