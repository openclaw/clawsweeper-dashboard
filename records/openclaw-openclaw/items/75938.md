## Summary

Keep open. Current main still lacks a Discord parent-channel `thread.requireMention` override, and open PR #76058 already uses closing syntax for the requested work.

## What This Changes

Add a parent-channel Discord `thread.requireMention` override so threads can use different mention gating than their parent channels.

## Best Possible Solution

Review and land or close PR #76058; after a merge, this issue can close as implemented by that PR.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: configure a parent Discord channel with `requireMention: true` and proposed `thread.requireMention: false`; current schema rejects the nested key and runtime would otherwise inherit the parent channel mention gate.

## Solution Assessment

Yes. The narrow maintainable solution is a backward-compatible Discord plugin config/schema/runtime/docs change that evaluates the thread override before the existing mention preflight drop.
