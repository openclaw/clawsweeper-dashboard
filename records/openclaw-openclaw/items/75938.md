## Summary

Keep open. Current main still lacks the requested Discord parent-channel `thread.requireMention` override, and open PR #76058 already uses closing syntax for the active implementation path.

## What This Changes

Add a Discord guild-channel `thread.requireMention` override so parent channels can require mentions while their threads can allow natural follow-up conversation.

## Best Possible Solution

Review and land or close #76058; the merged solution should keep Discord config types, strict schema, runtime mention preflight, docs, generated metadata, and regression tests aligned.

## Reproduction Assessment

Yes. The reported absence is source-reproducible on current main: a parent channel with `requireMention: true` still drives ordinary thread mention gating, while the proposed nested `thread.requireMention` key is not accepted by the strict schema.

## Solution Assessment

Yes. The narrowest maintainable path is a backward-compatible Discord plugin config/schema/runtime/docs/test change that applies a parent-channel thread override before the existing mention preflight drop; #76058 is already attempting that direction.
