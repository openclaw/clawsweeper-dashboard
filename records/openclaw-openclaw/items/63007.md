## Summary

Keep open. The PR still targets a real current-main gap, but the branch is draft/conflicting and stale against the current durable-send and hook-context contracts, so it needs rebase and API review before merge.

## What This Changes

The branch adds optional outbound agent/session identity to `message_sending`, reports hook cancellation from outbound delivery, and makes gateway send return an explicit guarded-denial error, with focused tests.

## Best Possible Solution

Rebase or replace this with a current-main patch that threads available agent/session identity into `message_sending`, maps durable `cancelled_by_message_sending_hook` outcomes to a structured gateway denial, preserves correlation fields, and coordinates with https://github.com/openclaw/openclaw/issues/63011 and https://github.com/openclaw/openclaw/issues/18571.

## Reproduction Assessment

Yes. A high-confidence source path exists: a `message_sending` cancellation produces a suppressed/empty durable send, gateway `send` turns non-`sent` status into `No delivery result`, and the hook context still lacks outbound agent/session identity.

## Solution Assessment

No, not as written. The gateway-only direction may be acceptable, but the branch must be rebased onto `sendDurableMessageBatch`, `hook-message.types.ts`, and the current mapper/cancellation contracts.
