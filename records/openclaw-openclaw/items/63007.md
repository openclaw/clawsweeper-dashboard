## Summary

Keep open. The PR targets a real gateway outbound guard gap, but the branch is draft and stale against current durable-send and hook-context contracts, has blocking correctness issues, and lacks inspectable real behavior proof.

## What This Changes

The PR adds outbound agent/session identity to `message_sending`, reports hook cancellation from outbound delivery, and returns a structured gateway send denial with tests.

## Best Possible Solution

Rebase or replace this on the current durable message lifecycle: thread available identity into the live hook context, preserve existing correlation fields, consume suppressed hook-cancellation outcomes in gateway send, use `cancelReason`, and coordinate the public contract with the related open items.

## Reproduction Assessment

Yes, source-reproducible: current main routes gateway send through `sendDurableMessageBatch`, maps suppressed/non-sent sends to an empty result and throws `No delivery result`, while `applyMessageSendingHook` passes only channel/account/conversation context. I did not run tests because this review is read-only.

## Solution Assessment

No, not as written. The gateway-only direction may be acceptable, but the branch must be rebased onto the durable send lifecycle, live hook-message types, existing mapper fields, and the documented `cancelReason` contract.
