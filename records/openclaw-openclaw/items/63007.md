## Summary

Keep open for maintainer review. The PR targets a real current-main gap, but the branch is draft/conflicting, now patches obsolete gateway/outbound/hook type shapes, lacks inspectable real behavior proof, and overlaps the canonical outbound-send contract discussion.

## What This Changes

The branch adds optional outbound agent/session identity to message hook context, forwards it through outbound delivery, reports `message_sending` cancellations to gateway send, and adds tests.

## Best Possible Solution

Rebase or replace this with a current-main patch that consumes durable suppressed hook outcomes in gateway send, threads available identity through the current hook-message context without dropping correlation fields, uses `cancelReason`, and coordinates with the canonical outbound-send contract.

## Reproduction Assessment

Yes by source inspection, not by a live run: current main represents hook cancellation as durable `status: "suppressed"`, gateway send ignores that status and throws `No delivery result`, and outbound `message_sending` receives only channel/account/conversation context.

## Solution Assessment

No. The useful direction should be rebased around current durable suppressed outcomes and current hook-message types, while maintainers decide the canonical gateway/direct send contract.
