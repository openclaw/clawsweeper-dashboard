## Summary

Keep open: the underlying gateway/hook gap is still real on current main, but this branch is draft/conflicting, lacks inspectable real behavior proof, and its diff is stale against the current durable send and hook type surfaces.

## What This Changes

The PR adds outbound session identity to message hook context, reports message_sending cancellation through outbound delivery, and changes gateway send to surface guarded denial instead of a generic missing-result error.

## Best Possible Solution

Rebase or replace this with a current-main patch that consumes durable suppressed sends in gateway send, threads agreed identity through the live hook-message contract, and coordinates scope with the canonical outbound-send issue.

## Reproduction Assessment

Yes, at source level. Current main can produce durable suppressed outcomes for message_sending cancellation, but gateway send filters those into No delivery result while outbound message_sending context still omits available identity fields.

## Solution Assessment

No. The direction is useful, but the patch targets an older delivery path and stale hook type surface; the maintainable fix should use sendDurableMessageBatch suppression, cancelReason, and the current hook-message context contract.
