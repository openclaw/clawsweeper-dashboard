## What This Changes

The PR adds outbound session identity to `message_sending`, reports hook cancellation from outbound delivery, and changes gateway `send` to return an explicit guarded denial instead of `No delivery result`.

## Best Possible Solution

Replace or rebase the branch onto current main so gateway `send` consumes durable suppressed outcomes, the live hook context contract carries agreed identity fields without dropping existing correlation fields, and the result is coordinated with the canonical outbound-send and hook-identity items.

## Reproduction Assessment

Yes, at source level. Current main can produce durable `suppressed` outcomes for hook-cancelled sends, but gateway `send` converts non-sent durable outcomes to `No delivery result` while outbound `message_sending` still receives only channel/account/conversation context.

## Solution Assessment

No, not as written. The direction is useful, but the branch predates current main's durable send path and live `hook-message.types.ts` surface, so the maintainable fix needs a current-main rebase or replacement.
