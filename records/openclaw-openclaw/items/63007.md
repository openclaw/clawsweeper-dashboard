## Summary

Keep open. The underlying gateway/hook problem is source-reproducible on current main, but this branch is draft/conflicting and targets an obsolete delivery shape, drops current hook context fields, edits the old type surface, and lacks inspectable real behavior proof.

## What This Changes

The branch adds optional outbound agent/session identity to message hook context, reports message_sending cancellations through outbound delivery, surfaces guarded gateway send cancellation, and adds targeted tests.

## Best Possible Solution

Rebase or replace this with a current-main patch that surfaces durable suppressed hook outcomes in gateway send, threads canonical session identity through the live hook-message contract without losing correlation fields, uses cancelReason, and coordinates broader send-path policy in https://github.com/openclaw/openclaw/issues/63011.

## Reproduction Assessment

Yes, by source inspection rather than a live run: current main can produce durable status "suppressed" for message_sending cancellation, but gateway send ignores that status and still falls through to "No delivery result"; current outbound message_sending context also omits nearby session identity.

## Solution Assessment

No. The proposed direction needs to be rebased onto sendDurableMessageBatch suppressed outcomes, the live hook-message types, preserved mapper fields, and the documented cancelReason contract before it is a maintainable fix.
