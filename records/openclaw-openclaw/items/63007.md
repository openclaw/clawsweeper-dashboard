## Summary

## What This Changes

The PR adds optional agent/session identity to message hook context, reports outbound `message_sending` cancellation, and changes gateway `send` to surface guarded denial instead of `No delivery result`, with focused tests.

## Best Possible Solution

Rework or replace the branch on current main so gateway `send` consumes durable suppressed outcomes, hook identity is added to the live message-hook contract without dropping existing fields, and the final scope is coordinated with the canonical outbound-send and sessionKey PRs.

## Reproduction Assessment

Yes, at source level. Current main records hook-cancelled delivery as a durable suppressed outcome, but gateway `send` maps non-sent statuses to empty results and throws, while `message_sending` still lacks the requested identity context.

## Solution Assessment

No, not as written. The direction is useful, but the branch must target current durable send outcomes and the live hook context contract, and maintainers still need to settle the public identity/canonical-send scope.
