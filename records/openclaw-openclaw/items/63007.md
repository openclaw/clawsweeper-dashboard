## Summary

Keep open. The PR targets a real outbound guard/audit gap, but it is still draft/conflicting, lacks inspectable real behavior proof, and its patch is stale against current main's durable send API and moved hook-context type surface.

## What This Changes

The PR adds outbound agent/session identity to message hook context, forwards it through delivery, reports `message_sending` cancellations, and tests gateway guarded-denial behavior.

## Best Possible Solution

Rebase or replace this with a current-main patch that consumes durable suppressed hook outcomes in gateway send, threads available outbound identity through the live hook context without dropping correlation fields, and coordinates with https://github.com/openclaw/openclaw/issues/63011 and https://github.com/openclaw/openclaw/issues/18571.

## Reproduction Assessment

Yes. Source inspection on current main shows a hook-cancelled durable send becomes `status: "suppressed"`, gateway `send` maps non-`sent` statuses to `No delivery result`, and outbound `message_sending` still lacks the requested identity context.

## Solution Assessment

No. The gateway-only direction may be acceptable, but this branch must be rebased onto the durable send API, current hook-message type file, current mapper contract, and documented `cancelReason` behavior.
