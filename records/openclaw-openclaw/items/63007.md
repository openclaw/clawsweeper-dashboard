## Summary

Keep this PR open. Current main still lacks the requested guarded gateway-send behavior, but the draft branch is stale against the live hook context/mappers and overlaps active canonical outbound-send work.

## What This Changes

The PR adds outbound agent/session identity to `message_sending`, reports hook cancellation from outbound delivery, and makes gateway send surface explicit guarded denial instead of `No delivery result`.

## Best Possible Solution

Rebase or replace this draft with a current-main patch that updates `hook-message.types.ts`, preserves existing mapper correlation fields, passes outbound session identity into `message_sending`, and defines the guarded gateway-denial response with focused tests and changelog coverage.

## Reproduction Assessment

Yes. A source-level reproduction is clear: make `message_sending` return `{ cancel: true }`, call gateway send with an outbound session, and current main skips delivery then reports `No delivery result`; the author also provided a live Telegram guarded-send sequence.

## Solution Assessment

No, not as written. The narrow gateway-send direction is plausible, but the patch targets obsolete hook type placement and would drop existing mapper fields unless rebased additively.
