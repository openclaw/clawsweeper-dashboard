## Summary

Keep this PR open. Current main still drops outbound agent identity before `message_sending`, still has no cancellation callback or blocked delivery result, and gateway `send` still reports hook-cancelled empty delivery as `No delivery result`. The PR is a useful draft, but it is stale against the current hook-context files and should be reviewed with the related canonical outbound-send work.

## What This Changes

The PR branch adds optional outbound agent/session identity propagation into `message_sending`, reports hook cancellation through outbound delivery, surfaces explicit guarded gateway-send denial, and adds focused tests.

## Best Possible Solution

Rebase or replace this draft with a current-main patch that updates the live hook context type, preserves existing mapper correlation fields while adding outbound overrides, passes `OutboundSessionContext` into `message_sending`, returns an explicit guarded-denial outcome for gateway send, and documents/tests the public plugin contract.

## Reproduction Assessment

Yes. A source-level reproduction is clear: make `message_sending` return `{ cancel: true }`, call gateway `send` with an outbound session, and current main skips delivery, returns an empty result, and reports `No delivery result`; the PR author also provided a live Telegram guarded-send sequence.

## Solution Assessment

Not as written. The narrow gateway-send direction is plausible, but the branch is stale against current hook types/mappers and needs maintainer review for the denial-detail response contract.
