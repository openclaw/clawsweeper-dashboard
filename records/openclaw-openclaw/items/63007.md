## Summary

Keep open: current main already landed the sessionKey subset, but gateway send still maps hook-cancelled suppressed delivery to `No delivery result`; this draft branch is useful but stale, conflicting, and missing inspectable real behavior proof.

## What This Changes

The PR adds outbound session identity to message hook context and makes gateway send surface `message_sending` guarded cancellation instead of a generic no-result error.

## Best Possible Solution

Rebase the useful denial-surfacing behavior onto current main by handling `sendDurableMessageBatch` suppressed results in gateway send, while leaving broader `agentId`/`runId` hook API decisions to the canonical plugin-hook issue.

## Reproduction Assessment

Yes at source level: current gateway send converts non-sent durable results into an empty result list and then throws `No delivery result`. I did not run a live Telegram reproduction in this read-only review.

## Solution Assessment

No as-is: the old branch is plausible on its original base, but current main needs the same denial handling implemented against `sendDurableMessageBatch` rather than a direct `deliverOutboundPayloads` callback.
