## Summary

Keep open. The reported gap still exists on current main, but this PR is stale against the current durable send path, has blocking correctness issues after rebase, is draft/conflicting, and still lacks contributor-owned real behavior proof.

## What This Changes

The PR adds outbound agent/session identity to `message_sending`, reports hook cancellation through outbound delivery, surfaces guarded gateway send denial, and adds focused tests.

## Best Possible Solution

Rebase or replace this with a current-main patch that consumes durable suppressed sends in gateway `send`, threads agreed identity through the live hook-message contract, and coordinates scope with the canonical outbound-send issue.

## Reproduction Assessment

Yes, at source level. Current main can produce durable `suppressed` outcomes for `message_sending` cancellation, but gateway `send` filters those into `No delivery result` while outbound `message_sending` context still omits available identity fields.

## Solution Assessment

No. The direction is useful, but the patch targets an older delivery path and stale hook type surface; the maintainable fix should be rebased onto `sendDurableMessageBatch`, use `cancelReason`, and preserve current mapper correlation fields.
