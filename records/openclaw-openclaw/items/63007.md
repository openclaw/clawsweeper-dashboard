## Summary

Keep open. The problem is still real, but this PR is stale against current main’s durable outbound send path, lacks inspectable real behavior proof, and overlaps open canonical follow-up work rather than being ready to merge as-is.

## What This Changes

The PR adds outbound agent/session identity to message_sending hook context, reports hook cancellation through outbound delivery, surfaces guarded gateway send denial, and adds targeted tests.

## Best Possible Solution

Rebase or replace this with a current-main patch that handles sendDurableMessageBatch suppressed outcomes, threads identity through the live hook-message contract without dropping existing fields, uses cancelReason, and coordinates the broader send-path decision in the canonical issue.

## Reproduction Assessment

Yes, at source level. Current main can produce durable suppressed outcomes for message_sending cancellation, but gateway send filters those out into "No delivery result" while outbound message_sending context still omits available identity fields.

## Solution Assessment

No. The PR’s direction is useful, but the diff targets an older delivery path and type surface; a maintainable fix should be rebased onto the durable send contract and live hook-message types.
