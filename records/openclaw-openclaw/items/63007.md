## Summary

Keep open. The requested gateway guarded-denial and outbound hook-identity behavior is still relevant on current main, but this draft PR is conflicting, targets stale delivery/type surfaces, lacks contributor-owned real behavior proof, and overlaps active canonical outbound-send and hook-identity decisions.

## What This Changes

The PR adds optional outbound agent/session context to `message_sending`, reports hook cancellation through outbound delivery, surfaces guarded gateway send cancellation, and adds focused tests.

## Best Possible Solution

Rebase or replace this with a current-main patch that consumes durable suppressed gateway sends, threads agreed identity through the live hook-message contract, and coordinates scope with the canonical outbound-send issue.

## Reproduction Assessment

Yes, at source level. Current main can produce durable `suppressed` outcomes for `message_sending` cancellation, but gateway `send` filters those into `No delivery result` while outbound `message_sending` context still omits available identity fields.

## Solution Assessment

No. The direction is useful, but the patch targets an older delivery path and stale hook type surface; the maintainable fix should use `sendDurableMessageBatch`, `cancelReason`, and the live `src/plugins/hook-message.types.ts` context.
