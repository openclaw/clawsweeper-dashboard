## What This Changes

The branch adds optional outbound identity fields to typed message hook context, passes session identity into `message_sending`, and makes gateway `send` return an explicit guarded-denial error when the hook cancels delivery.

## Best Possible Solution

Rebase or replace this with a maintainer-approved patch that handles durable `suppressed` gateway sends structurally, adds agreed hook identity fields on the live hook context without dropping correlation fields, and includes redacted real Telegram proof.

## Reproduction Assessment

Yes, at source level. Current main records hook-cancelled delivery as durable `suppressed`, but gateway `send` still maps non-`sent` statuses to an empty result and throws while outbound `message_sending` lacks the requested live identity context.

## Solution Assessment

No, not as written. The useful direction should be rebased onto the durable send result and live hook type/mapper contracts, with the final public identity semantics coordinated through the canonical outbound-send work.
