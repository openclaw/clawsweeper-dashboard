## Summary

Keep open: the reported gap is source-visible on current main, but this patch is not ready because it also enables Ollama's short-approval ack fast path, which current Ollama retry policy explicitly kept disabled, and the PR lacks real Ollama/GLM behavior proof.

## What This Changes

The PR extends incomplete-turn retry logic so Ollama planning-only turns, including `update_plan` with no visible text, return the standard act-now retry instruction and adds helper tests.

## Best Possible Solution

Keep a narrow Ollama plan-only retry for the `update_plan` no-text/prose failure while preserving the existing exclusion from short-approval ack prompts, then validate it with a real Ollama or GLM run log.

## Reproduction Assessment

Yes, from source inspection: current main's planning-only guard excludes `ollama`, and the no-visible-text path returns null before an act-now retry can be produced. I did not see live Ollama/GLM runtime proof in the PR body or comments.

## Solution Assessment

No. The proposed route reuses a shared helper and unintentionally opts Ollama into the ack fast path; the safer fix is to split the planning-only retry allowlist from ack prompt eligibility.

## Review Findings
