## Summary

Keep open: the reported Ollama planning-only stall is plausible from current source, but this PR broadens Ollama into the unrelated short-approval ack fast path that current main explicitly keeps out, and it still lacks required after-fix real behavior proof.

## What This Changes

The PR extends incomplete-turn planning-only retry handling to Ollama and adds tests for update_plan-only turns with no visible assistant text.

## Best Possible Solution

Separate Ollama planning-only retry eligibility from ack-fast-path eligibility, add focused regression coverage for both paths, and require redacted real Ollama/GLM proof showing the stalled update_plan turn recovers.

## Reproduction Assessment

Yes, at source level: current main excludes Ollama from the planning-only guard and returns null for empty assistant text before any update_plan-only retry can be emitted. I did not run a live Ollama model in this read-only review.

## Solution Assessment

No; the proposed implementation is too broad because it routes Ollama through a shared guard that also enables the ack fast path. The narrower fix is to handle the replay-safe plan-only/no-text case without changing short approval prompts unless maintainers explicitly want that behavior.
