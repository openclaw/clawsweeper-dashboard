## What This Changes

The PR adds token-presence tracking to prompt-template argument parsing and a focused regression test so quoted empty arguments remain positional arguments.

## Best Possible Solution

Land one shared parser path that preserves quoted empty tokens; if the apostrophe parser PR also lands, it should carry this invariant and regression coverage forward.

## Reproduction Assessment

## Solution Assessment

Yes. Separating token presence from accumulated text is the narrow shared-parser fix; the safer alternative is only to fold the same invariant into https://github.com/openclaw/openclaw/pull/90468, not to treat that PR as already covering it.
