The PR switches plugin `minHostVersion` parsing/checking from the runtime triple parser to the shared prerelease-aware semver comparator and adds prerelease precedence tests.

## Best Possible Solution

Keep the centralized helper fix, but compare with OpenClaw-aware version ordering that honors explicit prerelease floors without regressing numeric stable correction releases.

## Solution Assessment

No. The helper is the right layer, but the implementation should use or mirror OpenClaw-aware version comparison so prerelease floors are ordered without breaking numeric correction releases.
