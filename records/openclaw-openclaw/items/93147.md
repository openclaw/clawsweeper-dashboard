## Summary

## What This Changes

The branch adds an explicit `cron edit --dry-run` option that reads the existing cron job, emits JSON containing the proposed patch and before/after/diff fields, and skips `cron.update` plus the scheduler warning check.

## Best Possible Solution

Keep the opt-in dry-run shape for compatibility, but make the preview share the real update validation/normalization semantics and only close the linked request if maintainers accept dry-run as the chosen UX instead of an apply confirmation.

## Reproduction Assessment

## Solution Assessment
