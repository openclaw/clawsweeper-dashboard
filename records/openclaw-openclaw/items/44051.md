## Summary

Keep open. Current `main` still applies the realpath containment helper to `openclaw-managed` skill roots, and this issue is already paired with open fix PRs that use closing syntax, so it should remain open until one of those PRs is reviewed and merged or replaced.

## What This Changes

Managed skills installed via ClawHub under `~/.openclaw/skills` should load in WSL or symlink-backed layouts instead of being skipped as resolving outside their configured root.

## Best Possible Solution

Land a narrow skills-loader fix that scopes realpath root containment to untrusted roots, preserves `SKILL.md` confinement and workspace/extra fail-closed behavior, adds managed symlink regression coverage, and closes or supersedes the overlapping open PRs after one fix merges.

## Reproduction Assessment

Yes. A high-confidence reproduction is a managed skill directory under `~/.openclaw/skills` whose realpath points outside that managed root; current `main` routes `openclaw-managed` through `resolveContainedSkillPath()`, which skips outside realpaths.

## Solution Assessment

No, current `main` is not the best solution because it still enforces the containment check on managed roots. The best path is the source-policy approach described in the related PR discussion rather than closing this before a PR lands.
