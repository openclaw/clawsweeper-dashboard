## Summary

Keep open. Current main still applies the same realpath containment check to managed skills as it does to workspace and extra-dir roots, and the issue is already paired with open fix PRs that reference it, so it should not be closed before a canonical fix lands or those PRs are closed.

## What This Changes

The issue reports that officially installed ClawHub skills under the managed skills directory can be skipped on WSL/symlink-backed layouts because the skills loader treats their realpath as outside the configured root.

## Best Possible Solution

Current main should gain a source-scoped containment policy: keep fail-closed realpath checks for workspace, extra-dir, project-agent, and bundled roots as appropriate, but allow trusted managed/personal skill roots to load valid symlink-backed skill directories while still rejecting escaping SKILL.md files where required by the threat model.
