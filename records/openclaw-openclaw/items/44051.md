## Summary

Keep open: current main still applies the same realpath containment path to managed skills, and open linked PRs already target the fix with closing syntax, so this should remain the tracking report until one fix lands or is superseded.

## What This Changes

Fix managed ClawHub-installed skills under `~/.openclaw/skills` being skipped when their realpath resolves outside the managed root on WSL or symlink-backed layouts.

## Best Possible Solution

Land one narrow skills-loader fix that scopes root containment to untrusted roots while keeping per-skill `SKILL.md` confinement and workspace/extra/bundled escape coverage intact.

## Reproduction Assessment

Yes, there is a high-confidence source reproduction path: a managed skill directory whose realpath resolves outside the managed root still flows through `resolveContainedSkillPath()` and is skipped. I did not run a live WSL gateway session in this read-only review.

## Solution Assessment

Yes for the source-scoped containment approach described in the linked PR discussion; current main has not applied that policy yet. The safer fix is to exempt trusted managed/personal roots from root-level containment without weakening workspace and extra-dir fail-closed behavior.
