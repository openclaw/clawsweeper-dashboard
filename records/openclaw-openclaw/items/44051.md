## Summary

Keep open: current main still routes managed skill roots through source-agnostic realpath containment, and open PRs with closing syntax already target the same regression.

## What This Changes

The issue reports managed ClawHub skills under `~/.openclaw/skills` being skipped on WSL because realpath containment rejects paths that resolve outside the managed root.

## Best Possible Solution

Land or replace one narrow skills-loader fix that scopes root containment to untrusted roots while preserving per-`SKILL.md` confinement and workspace/extra/bundled fail-closed coverage.

## Reproduction Assessment

Yes. A managed skill directory whose realpath resolves outside `managedSkillsDir` still flows through `loadSkills(... source: "openclaw-managed")` into `resolveContainedSkillPath()` and is skipped before loading.

## Solution Assessment

Yes. The source-scoped containment approach described in the linked PR discussion is the narrow maintainable fix; current main has not applied that policy yet.
