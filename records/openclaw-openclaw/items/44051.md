## What This Changes

Managed ClawHub-installed skills in symlink-backed WSL layouts should load from the shared managed skills root instead of being rejected as outside the configured root.

## Best Possible Solution

Review and land or supersede one narrow skills-loader fix that scopes root containment to untrusted roots while preserving per-SKILL.md confinement and workspace/extra/bundled fail-closed coverage.
