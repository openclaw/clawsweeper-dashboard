## Summary

Keep open. Current main still routes managed skills through the shared realpath containment helper, and open PRs with closing syntax already target this regression.

## What This Changes

The issue asks OpenClaw to load ClawHub-installed managed skills on WSL when symlink or junction realpaths leave the managed skills root.

## Best Possible Solution

Review and land one narrow skills-loader PR that exempts trusted managed and personal roots from root-level realpath containment while keeping workspace, extra, bundled, and per-skill `SKILL.md` escape checks fail-closed.

## Reproduction Assessment

Yes. A managed skill directory whose realpath resolves outside the managed root still reaches `resolveContainedSkillPath()` with `source: "openclaw-managed"`, which returns `null` and skips loading.

## Solution Assessment

No. Current main is not the best solution because it still applies the fail-closed workspace-style root check to managed skills; the source-policy approach in the related PR discussion is the maintainable fix.
