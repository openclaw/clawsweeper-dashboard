## Summary

Keep this open: both v2026.6.8 and current `origin/main` still build the Telegram `/think` argument menu without passing the runtime model catalog, so live-discovered Ollama reasoning metadata can still be missing from the menu. The issue is already covered by several open candidate fix PRs, with https://github.com/openclaw/openclaw/pull/94067 looking like the strongest canonical review path because it resolves the menu choices against the runtime catalog instead of changing Ollama's undefined-reasoning policy globally.

## What This Changes

The issue requests that Telegram's bare `/think` menu show supported Ollama thinking levels for live-discovered reasoning models, matching accepted `/think <level>` commands.

## Best Possible Solution

Review and land one focused runtime-catalog fix for native `/think` menu choices, preferably the existing https://github.com/openclaw/openclaw/pull/94067 path if it passes maintainer review and proof.

## Reproduction Assessment

Yes, source-reproducible with high confidence: v2026.6.8 and current `origin/main` both build Telegram's `/think` menu from the configured catalog while live-discovered Ollama metadata only exists in the runtime catalog. I did not execute the live Telegram/Ollama repro in this read-only pass.

## Solution Assessment

Yes, the best solution appears to be feeding native `/think` menu choice resolution the same runtime catalog used by the current-level/default path. A provider-wide undefined-reasoning fallback is a plausible workaround but risks advertising levels for unknown non-reasoning Ollama models.
