## Summary

Keep open: current main and v2026.6.8 still resolve the Telegram `/think` menu without the runtime catalog, so live-discovered Ollama reasoning metadata can be lost. The issue already has open candidate fixes, with https://github.com/openclaw/openclaw/pull/94067 looking like the strongest review path because it feeds native menu choices the runtime catalog instead of changing Ollama's unknown-reasoning policy globally.

## What This Changes

The issue asks OpenClaw to show supported Ollama thinking levels in Telegram's bare `/think` menu for live-discovered reasoning models instead of only `default, off`.

## Best Possible Solution

Review and land one focused runtime-catalog fix for native `/think` menu choices, preferably the existing https://github.com/openclaw/openclaw/pull/94067 path if maintainer review and proof stay clean.

## Reproduction Assessment

Yes. The current source path shows Telegram's `/think` menu falls back to configured-only catalog data while Ollama live discovery stores reasoning capability in the runtime catalog; I did not execute the live Telegram/Ollama repro.

## Solution Assessment

Yes. The best fix appears to be passing native `/think` menu choice resolution the same runtime catalog used by the current-level/default thinking path; provider-wide undefined-reasoning fallbacks are broader and risk advertising levels for unknown non-reasoning Ollama models.
