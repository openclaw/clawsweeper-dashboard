## Summary

Keep open: current main and v2026.6.9 still route the Telegram /think menu through configured-only catalog data, so the reported live-discovered Ollama reasoning metadata can still be lost. The issue already has open closing PRs, with https://github.com/openclaw/openclaw/pull/94067 still the strongest canonical fix path because it passes the runtime catalog into native /think menu resolution instead of broadening Ollama's unknown-reasoning policy.

## What This Changes

The issue reports that Telegram bare /think lists only default/off for a live-discovered Ollama reasoning model even though explicit /think levels are accepted.

## Best Possible Solution

Review and land one focused runtime-catalog fix for native /think menu choices, preferably the existing https://github.com/openclaw/openclaw/pull/94067 path if maintainer review and proof stay clean.

## Reproduction Assessment

Yes. The current source path shows Telegram /think menu choices receive configured-only catalog data while Ollama live discovery can put reasoning: true only in the runtime catalog; I did not execute the live Telegram/Ollama repro.

## Solution Assessment

Yes. The best fix appears to be passing native /think menu choice resolution the same runtime catalog used by the current-level/default thinking path; provider-wide undefined-reasoning fallbacks are broader and risk advertising levels for unknown non-reasoning Ollama models.
