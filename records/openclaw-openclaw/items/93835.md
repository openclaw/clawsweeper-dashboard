Keep open: current main still has the reported native `/think` menu/catalog mismatch, and this issue is already paired with multiple open closing PRs that should be reviewed rather than closing the issue early.

## What This Changes

The issue requests that Telegram bare `/think` list full thinking levels for live-discovered Ollama reasoning models instead of only `default, off`.

## Best Possible Solution

Land one reviewed runtime-catalog native `/think` menu fix, preferably the broad channel-side path that reuses existing catalog metadata, then close this issue and supersede duplicate PRs.

## Reproduction Assessment

Yes for source reproduction: the report gives concrete v2026.6.8 Telegram/Ollama steps, and current main shows the menu path omits the runtime catalog while the resolver falls back to configured-only metadata. I did not run a live Telegram/Ollama instance in this read-only review.

## Solution Assessment

Yes: the narrowest maintainable fix is to pass the existing runtime model catalog into native `/think` menu resolution so discovered `reasoning: true` metadata reaches the shared thinking-profile resolver. Provider-policy heuristics are less direct and risk changing behavior for unknown non-thinking Ollama models.
