Keep open: current main and the latest release still show the same menu/catalog mismatch, and several open PRs already reference this issue as their fix target. The issue should close only after one canonical fix merges.

## What This Changes

The issue reports that Telegram’s bare `/think` menu omits live-discovered Ollama thinking levels even though `/think <level>` accepts those levels for the selected model.

## Best Possible Solution

Land one canonical fix that makes native `/think` menu choices resolve against the same runtime catalog capability metadata used by current-level display and command validation.

## Reproduction Assessment

Yes. Source inspection reproduces the mismatch: Telegram builds the `/think` menu without passing a runtime catalog, while the current-level path loads runtime catalog metadata and the Ollama policy returns off-only when reasoning is undefined.

## Solution Assessment

Yes for a runtime-catalog menu fix. That solves the mismatch at the native menu layer without globally broadening provider policy for every unknown Ollama model.
