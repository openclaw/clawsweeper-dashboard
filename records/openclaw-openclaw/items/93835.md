## Summary

Keep this issue open: current main still loses runtime Ollama capability metadata when building native `/think` menu choices, and an open fix PR already targets this issue with a broader channel-side runtime-catalog fix.

## What This Changes

The issue reports that Telegram’s bare `/think` menu omits Ollama reasoning levels for a live-discovered model even though `/think <level>` works.

## Best Possible Solution

Land the channel-side runtime-catalog fix in https://github.com/openclaw/openclaw/pull/94067, then close this issue and supersede narrower sibling PRs that solve only part of the same symptom.

## Reproduction Assessment

Yes from source, though not from a live Telegram/Ollama run here. Current main’s Telegram menu path passes no runtime catalog, and the registry therefore loses live-discovered Ollama `reasoning` metadata before reaching the Ollama thinking profile.

## Solution Assessment

Yes. Passing the existing runtime model catalog into native `/think` menu resolution is the narrow owner-boundary fix; provider-policy fallback PRs are broader and do not directly address Slack/Discord native menu catalog resolution.
