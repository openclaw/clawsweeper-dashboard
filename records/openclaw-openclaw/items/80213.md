## Summary

Keep this open: current main still has no skill-authored install/update setup hook, and an open PR with closing syntax is already the implementation candidate. This is a new, security-sensitive lifecycle capability rather than a stale cleanup target.

## What This Changes

Request to add `metadata.openclaw.setup.script` so skills can run a bundled setup script after install/update with a defined environment contract.

## Best Possible Solution

Review the open implementation PR as the canonical path, and land only a deliberately scoped setup-hook contract that preserves env safety, stable install/update semantics, rollback behavior, public docs, and real install/update proof.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source and docs inspection give a high-confidence current-main check that the requested setup-hook surface is absent.

## Solution Assessment

Unclear until maintainer review. A post-install/update setup hook may be the right direction, but the current decision should happen through https://github.com/openclaw/openclaw/pull/80266 because the feature changes the skill execution and credential trust boundary.
