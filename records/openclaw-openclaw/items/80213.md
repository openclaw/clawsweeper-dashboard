## Summary

This should stay open: current main and the latest shipped release still expose only predefined skill install specs and plugin `before_install` policy hooks, while the requested skill-authored post-install/update setup runner is a new security-sensitive lifecycle surface. The closest related open issue, https://github.com/openclaw/openclaw/issues/23926, covers broader plugin-author install hook events and does not replace this narrower skill-author setup contract.

## What This Changes

Add a `metadata.openclaw.setup.script` SKILL.md frontmatter hook that runs a skill-supplied script after skill install/update with a defined env contract.

## Best Possible Solution

Define a first-class skill setup lifecycle contract with constrained bundle-relative scripts, explicit env allowlisting, install/update ordering, failure/rollback behavior, docs, and tests across both ClawHub and gateway skill install paths.

## Reproduction Assessment

Not applicable; this is a feature request rather than a broken existing behavior. Source and shipped-release inspection confirm the requested setup hook is absent from the current skill metadata parser and install/update paths.

## Solution Assessment

Unclear until maintainer design review; `setup.script` is a plausible narrow surface, but install-time execution of skill-supplied scripts with secret env access needs an explicit product and security contract before implementation.
