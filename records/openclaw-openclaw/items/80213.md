## Summary

This should stay open: current main still lacks a skill-authored install/update setup execution surface, and two open PRs already reference this issue with closing syntax. The request is a new security-sensitive skill lifecycle feature, so maintainers should decide the product/security contract before landing either implementation.

## What This Changes

Add a `metadata.openclaw.setup.script` skill frontmatter hook that runs after skill install/update with a defined environment contract.

## Best Possible Solution

Define a narrow skill setup contract, then review and either land one of the linked implementation PRs or close them in favor of a different maintainer-approved design.

Not applicable as a bug reproduction: this is a feature request. Source and docs inspection do give a high-confidence current-main check that the requested hook surface is absent.

## Solution Assessment

Unclear until maintainer review. A post-install/update setup hook may be the right direction, but executing skill-supplied scripts with selected credential env needs deliberate path confinement, allowlisting, logging, timeout, failure, and ClawHub trust rules first.
