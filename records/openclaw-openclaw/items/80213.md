This should stay open: current main still lacks a skill-authored install/update setup execution surface, and the comments plus open implementation PR show active, concrete follow-up rather than a cleanup-close candidate.

## What This Changes

The request is to add a `metadata.openclaw.setup.script` skill metadata hook that runs a skill-supplied script after successful skill install or update with a defined environment contract.

## Best Possible Solution

Define a narrow skill setup contract only after maintainer review covers execution trust boundaries, environment allowlisting, timeout ownership, rollback semantics, logging, docs, tests, and ClawHub update behavior.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source and docs inspection give a high-confidence current-main check that the requested setup hook surface is absent.

## Solution Assessment

Unclear until maintainer review. A post-install/update setup hook may be the right direction, but executing skill-supplied scripts with selected credential env needs a deliberately designed security and product contract first.
