## Summary

This should stay open: current main still lacks a skill-authored install/update setup execution surface, and the systemd drop-in comment adds useful prior art rather than a cleanup-close reason. The request is a new security-sensitive skill lifecycle and manifest feature that needs maintainer product/security design before implementation.

## What This Changes

Add a `metadata.openclaw.setup.script` skill frontmatter hook that runs a bundled script after skill install/update with a defined environment contract.

## Best Possible Solution

Define a narrow skill setup contract only after maintainer review covers path confinement, env allowlisting, failure semantics, timeout behavior, logging, ClawHub trust posture, update ordering, and docs/tests.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source and docs inspection provide a high-confidence current-main check that the requested hook surface is absent.

## Solution Assessment

Unclear until maintainer review. A post-install/update `setup.script` hook may be the right direction, but executing skill-supplied scripts with selected credential env needs deliberate security and product design first.
