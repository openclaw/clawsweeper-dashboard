## Summary

Keep this open: current main still lacks a skill-authored install/update setup execution surface, and the request is a new security-sensitive skill lifecycle feature that needs maintainer product/security design rather than cleanup closure.

## What This Changes

The issue requests a `metadata.openclaw.setup.script` skill metadata hook that runs a skill-supplied setup script after install/update with defined environment semantics.

## Best Possible Solution

Define a documented skill setup contract only after maintainer review covers install/update ordering, path confinement, env allowlisting, timeout/failure behavior, logging/redaction, and ClawHub trust posture.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source and docs inspection give a high-confidence current-main check that the requested skill-authored setup surface is absent.

## Solution Assessment

Unclear until maintainer review. A post-install/update setup hook may be the right direction, but executing skill-supplied scripts with credential env needs a deliberately designed security and product contract first.
