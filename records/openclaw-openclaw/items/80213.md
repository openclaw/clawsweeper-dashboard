## Summary

Keep open: current main and v2026.6.10 still lack a skill-authored `setup.script` metadata contract, a post-install/update setup execution path, or a `skills.setup` core Gateway method. The published ClawHub plugin is useful prior art, but its README says it is a manual admin RPC and not the automatic install/update lifecycle hook requested here; the remaining boundary is still a maintainer product/security decision.

## What This Changes

The issue requests a skill-authored `metadata.openclaw.setup.script` hook that runs after skill install/update with a defined setup environment contract.

## Best Possible Solution

Pick one security-reviewed setup execution boundary: either a core install/update hook with explicit path/env/timeout/logging semantics, or an external ClawHub plugin path backed by stable SDK helpers and documented operator expectations.

## Reproduction Assessment

Not applicable as a bug reproduction, but yes at source level. Current main lacks a `setup.script` metadata field, frontmatter parsing for it, post-install/update setup execution, and a core `skills.setup` method.

## Solution Assessment

Unclear until maintainer decision. The proposed hook is plausible, but the safer permanent boundary may be a reviewed core lifecycle hook or the published ClawHub/admin-RPC plugin backed by stable Plugin SDK helpers.
