## Summary

Keep this open: current main still has no skill-authored install/update setup hook, and two open closing PRs are competing implementation candidates. The published ClawHub plugin is useful positive context, but it does not settle the upstream install/update contract or the security boundary.

## What This Changes

The issue requests a skill-authored `metadata.openclaw.setup.script` hook that runs after skill install/update with a defined environment contract.

## Best Possible Solution

Keep this issue attached to the implementation reviews until maintainers choose a single vetted setup contract or explicitly decide that the ClawHub/plugin path is the supported outcome.

Not applicable as a bug reproduction; this is a new feature request. Source inspection gives a high-confidence current-main check that the requested setup hook surface is absent.

## Solution Assessment

Unclear. A setup hook or admin plugin RPC may be viable, but the best solution still needs maintainer agreement on execution timing, env sanitization, consent/logging, rollback semantics, and ClawHub trust boundaries.
