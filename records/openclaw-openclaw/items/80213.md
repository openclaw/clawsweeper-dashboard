## Summary

Keep open: current main still lacks a skill-authored setup hook, and the two linked open PRs are implementation candidates rather than a reason to close this issue before one merges. The requested lifecycle surface is security-sensitive because it would run third-party skill code during install/update with selected environment values.

## What This Changes

Add a skill-authored `metadata.openclaw.setup.script` hook that runs after skill install/update with a defined environment contract.

## Best Possible Solution

Keep this issue as the canonical design anchor, settle the setup-hook security/product contract, then land one reviewed implementation and close or supersede competing PRs.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection against current main gives high-confidence evidence that the requested setup hook surface is absent.

## Solution Assessment

Unclear until maintainer review. A post-install/update setup hook may be the right direction, but the best solution needs one documented lifecycle contract rather than ad hoc install-time script execution.
