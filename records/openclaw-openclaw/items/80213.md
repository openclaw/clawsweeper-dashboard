## Summary

This should stay open: current main has skill install metadata and plugin-side `before_install` policy hooks, but not a skill-authored post-install/update setup script surface, and adding one is a new security-sensitive feature rather than cleanup.

## What This Changes

Add a `metadata.openclaw.setup.script` SKILL.md hook that runs after skill install/update with a defined environment contract.

## Best Possible Solution

Define a narrow, explicit skill setup contract only after maintainer security/product review covers path confinement, env allowlisting, failure semantics, timeout behavior, ClawHub trust posture, and install/update ordering.

Not applicable as a bug reproduction: this is a feature request. Source inspection does confirm the requested hook surface is absent from current main.

## Solution Assessment

Unclear until maintainer review. A post-install/update setup hook may be the right direction, but the exact contract needs security and product decisions before it is safe to implement.
