## Summary

Keep this open: current main still lacks a skill-authored setup execution surface, and two open PRs use closing references for this issue. The published ClawHub plugin is useful prior art, but it does not settle the upstream install/update contract or security boundary.

## What This Changes

The issue requests a `metadata.openclaw.setup.script` hook that can run a skill-supplied script after install or update with a defined environment contract.

## Best Possible Solution

Keep the issue attached to the open implementation reviews until maintainers choose either a vetted core install/update setup contract or an explicit plugin/ClawHub-only outcome.

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high-confidence proof that current main lacks the requested setup-hook surface.

Unclear. A setup hook or admin plugin RPC may be viable, but maintainers still need to settle the security, env-sanitizer, timing, documentation, and ClawHub trust contract before this should close.

Status: needs_attention
