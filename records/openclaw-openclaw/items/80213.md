## Summary

Keep open: current main and the latest release still lack a skill-authored install/update setup hook, and there is an open implementation PR with closing syntax for this issue. The request is an active, security-sensitive feature proposal rather than a cleanup-close candidate.

## What This Changes

Add `metadata.openclaw.setup.script` in skill frontmatter to run a skill-supplied post-install/update script with a defined environment contract.

## Best Possible Solution

Use the open implementation PR as the review vehicle, and land only a narrowly specified setup contract after maintainers settle the execution, environment, timeout, rollback, logging, docs, and ClawHub rules.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source and docs inspection give a high-confidence current-main check that the requested setup hook surface is absent.

## Solution Assessment

Unclear until maintainer review. A post-install/update setup hook may be the right direction, but executing skill-supplied scripts with selected credential env needs a deliberately designed security and product contract first.
