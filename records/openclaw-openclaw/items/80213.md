## Summary

This should stay open: current main still lacks a skill-authored install/update setup execution surface, and the later systemd-drop-in workaround comment is useful supporting evidence rather than a cleanup-close reason. The request is a new security-sensitive skill lifecycle feature that needs maintainer product/security design before implementation.

## What This Changes

Add `metadata.openclaw.setup.script` to SKILL.md so skill authors can run a bundle-relative setup script after skill install/update with a defined environment contract.

## Best Possible Solution

Define a narrow, explicit skill setup contract only after maintainer review covers path confinement, env allowlisting, failure semantics, timeout behavior, ClawHub trust posture, logging, and install/update ordering.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source and docs inspection do give a high-confidence current-main check that the requested hook surface is absent.

## Solution Assessment

Unclear until maintainer review. A post-install/update setup hook may be the right direction, but executing skill-supplied scripts with credential env needs a deliberately designed security and product contract first.
