## Summary

Keep open. The PR addresses a valid overwrite-only write-tool gap for #75321, and the append-flag direction is narrow, but the latest diff is not merge-ready because append mode still bypasses the existing non-empty `path`/`content` validation for whitespace-only inputs.

## What This Changes

The PR adds an optional `append` parameter to the model-facing `write` tool, implements host and sandbox append operations, adds focused append tests, and updates the changelog.

## Best Possible Solution

Make append mode a strict extension of the existing `write` contract: preserve the flat provider-safe schema, call the same required-param validation before append writes, keep host/workspace/sandbox boundaries unchanged, and add regression coverage for malformed append calls before landing.

## Reproduction Assessment

Yes. The original issue is source-reproducible on current main because normal `write` still overwrites outside the memory-flush wrapper; the PR regression is source-reproducible by calling `write` with `append: true` and whitespace-only `path` or `content`.

## Solution Assessment

No. The optional append flag is the narrow maintainable direction, but this implementation is not the best fix until append mode uses the same non-empty required-param validation as overwrite mode.
