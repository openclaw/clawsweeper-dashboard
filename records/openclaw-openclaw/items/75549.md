## Summary

Keep open. The append-mode direction is valid and current main still lacks a general append surface, but this PR still has correctness blockers in the new append path: sandbox append suppresses all read failures as if the file were missing, and append execution ignores cancellation semantics that normal writes already honor.

## What This Changes

The PR adds an optional `append` parameter to the agent `write` tool, routes host/workspace/sandbox append operations, adds focused host append tests, and records the user-facing change in the changelog.

## Best Possible Solution

Land the append flag once it matches the existing write tool's safety envelope: flat provider-safe schema, unchanged overwrite default, workspace and sandbox boundaries preserved, same-file mutation serialization, narrow handling for missing sandbox files, cancellation behavior consistent with normal writes, and regression coverage for host, workspace-only, sandbox, concurrent, and aborted append calls.

## Reproduction Assessment

Yes. Current main is source-reproducible as overwrite-only outside the memory-flush wrapper, and the PR regressions are source-reproducible by making sandbox `readFile` fail for a reason other than not-found or by invoking `append: true` with an already-aborted signal.

## Solution Assessment

No. The optional `append` flag is the narrow maintainable solution direction, but this implementation needs the sandbox error handling and abort semantics fixed before it is the best way to solve the issue.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve sandbox read failures before rewriting:** `src/agents/pi-tools.read.ts:741-743`
  - body: The sandbox append path treats every `bridge.readFile` exception as a missing file and then rewrites `existing + content`. `readFile` can fail for safety or I/O reasons other than absence; those failures should surface instead of being converted into an append that may replace unread data. Check existence with `stat` or only suppress a verified not-found case before writing.
  - confidence: 0.86
- **[P2] Honor aborted append calls:** `src/agents/pi-tools.read.ts:859`
  - body: The append branch receives `signal` but never checks it or passes it into the new append operations, while the upstream write tool rejects aborted writes before mutating. An already-canceled cron or agent run can still create or append to a file. Mirror the normal write abort guard and thread the signal through sandbox bridge calls where practical.
  - confidence: 0.8
