## Summary

Keep open for maintainer review: the fix addresses a source-reproducible sandbox CLI regression, but the chosen policy intentionally makes sandbox commands activate all enabled plugins, so maintainers should accept that availability tradeoff before merge.

## What This Changes

The PR adds a sandbox CLI command policy that loads plugin runtime for sandbox commands and adds regression coverage for command policy resolution and plugin-owned sandbox backend status lookup.

## Best Possible Solution

Land after maintainers explicitly accept full plugin activation for sandbox commands, or introduce a narrow sandbox-backend plugin activation scope before merging.

## Reproduction Assessment

Yes at source level: current main defaults sandbox commands to loadPlugins:"never", OpenShell registers its manager only during full plugin registration, and the container listing path reports stopped when that manager is absent.

## Solution Assessment

Mostly yes: the patch is the narrowest fix using the current CLI/plugin registry policy surface, but the all-plugin activation scope is a maintainer policy tradeoff rather than a purely mechanical bug fix.
