## Summary

Keep open: the PR is a focused candidate fix for the linked OpenShell sandbox status regression, and current main plus v2026.6.8 still lack the sandbox CLI plugin-loading override. The remaining blocker is maintainer acceptance of making sandbox commands run full enabled-plugin activation before command execution.

## What This Changes

The PR adds a sandbox CLI command policy that loads plugins for `openclaw sandbox` commands and adds regression tests for sandbox command policy and plugin-owned backend status lookup.

## Best Possible Solution

Land after maintainers explicitly accept full plugin activation for sandbox commands, or replace it with a narrow sandbox-backend activation scope before merge.

## Reproduction Assessment

Yes at source level: current main defaults sandbox commands to no plugin loading, OpenShell registers its manager only during full plugin registration, and missing managers report registry entries as stopped. I did not run a live WSL+OpenShell environment in this read-only review.

## Solution Assessment

Mostly yes: the PR is the narrowest fix using the existing CLI policy surface, but the all-plugin activation scope is a maintainer availability tradeoff rather than a purely mechanical bug fix.
