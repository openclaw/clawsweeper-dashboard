## Summary

Keep open. Current main appears to have fixed the explicit SSE/WebSocket selection part, but the OpenAI Chat Completions fallback still shapes a request that can include both function tools and `reasoning_effort` for `gpt-5.4-mini`, matching the reported provider 400.

## What This Changes

The issue reports that Telegram-routed `openai/gpt-5.4-mini` turns hang or fail because OpenAI transport selection ignores explicit SSE and the Chat Completions fallback sends an unsupported `tools` plus `reasoning_effort` payload.

## Best Possible Solution

Land a focused OpenAI completions payload fix that keeps explicit SSE behavior intact and suppresses or reroutes the unsupported native OpenAI Chat Completions `tools` plus `reasoning_effort` combination for `gpt-5.4-mini`, with regression coverage.

Yes for a code-level reproduction: construct an OpenAI Chat Completions payload for a reasoning `gpt-5.4-mini` model with function tools and current main can emit both `tools` and `reasoning_effort`. The end-to-end Telegram hang still needs live verification after the payload fix.

## Solution Assessment
