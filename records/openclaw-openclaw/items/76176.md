## Summary

Keep open. Current main handles the explicit SSE override, but source inspection still shows the Chat Completions fallback can produce the reported unsupported `tools` plus `reasoning_effort` payload for `gpt-5.4-mini`, and open PR #76326 already targets that remaining bug.

## What This Changes

Telegram-routed agents using `openai/gpt-5.4-mini` should reply reliably and avoid unsupported `reasoning_effort` in Chat Completions fallback payloads.

## Best Possible Solution

Review and land or replace PR #76326 with regression coverage so `gpt-5.4-mini` Chat Completions requests avoid unsupported `reasoning_effort` while preserving Responses and explicit SSE behavior.

## Reproduction Assessment

Yes for a source-level reproduction: current main can build a Chat Completions payload with function `tools` and `reasoning_effort` for a reasoning `gpt-5.4-mini` model. I did not run a live Telegram/OpenAI turn, so the end-to-end hang remains unverified.

## Solution Assessment

Yes, a narrow OpenAI payload/model-capability fix is the best direction because the remaining failure is provider compatibility, not Telegram delivery. The open closing PR is the right implementation candidate to validate before this issue closes.
