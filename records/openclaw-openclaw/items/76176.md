## Summary

Keep open. Current main still has the source-visible OpenAI Chat Completions payload bug: the explicit SSE/WebSocket part is fixed, but a native OpenAI completions fallback can still combine function tools with `reasoning_effort` for a reasoning `gpt-5.4-mini` model.

## What This Changes

The issue reports that Telegram-routed `openai/gpt-5.4-mini` turns hang or fail because OpenAI transport selection ignores explicit SSE and the Chat Completions fallback sends an unsupported `tools` plus `reasoning_effort` payload.

## Best Possible Solution

Land a focused OpenAI completions payload fix that preserves explicit SSE handling and prevents the unsupported native OpenAI Chat Completions function-tools plus `reasoning_effort` combination for `gpt-5.4-mini`, with regression coverage.

## Reproduction Assessment

Yes for a code-level reproduction: construct an OpenAI Chat Completions payload for a reasoning `gpt-5.4-mini` model with function tools, and current main can emit both `tools` and `reasoning_effort`. The full Telegram hang remains a live integration verification step after the payload fix.

## Solution Assessment

Yes. A narrow provider payload/routing fix is the best next step because the remaining observed failure is in OpenAI model/API compatibility, while Telegram is only the delivery surface.
