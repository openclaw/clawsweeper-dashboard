Keep open. Current main still contains the source path for the all-candidates-aborted fallback cascade, while the relevant fixes are still open PRs rather than shipped behavior.

## What This Changes

The issue reports that Control UI webchat can time out before an embedded Codex run starts and then abort every configured model fallback candidate with the same signal.

## Best Possible Solution

Keep this issue open until main has a reviewed fix that stops terminal run-level aborts from consuming every fallback candidate and handles WebChat timeout surfacing without changing the existing timeout contract.

## Reproduction Assessment

Partly: no live Control UI reproduction was run, but the fallback cascade is source-reproducible on current main. The gateway can pass one timeout-backed webchat abort signal into the agent path, and the fallback layer cannot inspect terminal caller abort reasons before trying later candidates.

## Solution Assessment

Unclear for the full issue. The open terminal-abort PR is the right direction for the fallback cascade, but the Control UI pre-start timeout source and WebChat terminal-error behavior still need focused review or proof before this can be considered solved.
