## Summary

Keep this PR open for maintainer review: current main and the latest release still use the hardcoded sandbox skill-sync fallback, while this PR supplies a narrow fix with terminal proof and regression coverage. No blocking code-review finding surfaced, but the fallback behavior is compatibility-sensitive and should be accepted by a maintainer before merge.

## What This Changes

The PR changes sandbox skill synchronization fallback to resolve the source workspace from agent config when `workspaceDir` is omitted or blank, and adds regression plus standalone repro coverage.

## Best Possible Solution

Land the focused sandbox fallback fix after maintainer acceptance of the configured-workspace semantics, while keeping broader Gateway install/update workspace routing tracked separately in https://github.com/openclaw/openclaw/pull/89767.

## Reproduction Assessment

Yes. Current source shows omitted or blank sandbox `workspaceDir` falls back to `DEFAULT_AGENT_WORKSPACE_DIR`, and the PR body supplies after-fix terminal proof plus a pre-fix revert failure for the configured-workspace path.

## Solution Assessment

Yes. Reusing `resolveAgentWorkspaceDir` at the sandbox layout fallback point is the narrowest maintainable fix; changing each caller would leave the exported fallback broken, while the Gateway routing PR covers a distinct mutation API path.
