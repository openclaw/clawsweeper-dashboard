## Summary

Keep this PR open as the focused fix candidate: current main and the latest release still use the hardcoded sandbox workspace fallback, while the PR switches that fallback to the canonical agent workspace resolver and adds targeted regression coverage. ## What This Changes

The PR changes sandbox skill-sync fallback to resolve the source workspace from agent config when `workspaceDir` is omitted or blank, and adds two regression tests for that fallback.

## Best Possible Solution

Yes. Reusing `resolveAgentWorkspaceDir` at the fallback point is the narrow owner-boundary fix; duplicating config parsing or changing every caller would be more brittle.
