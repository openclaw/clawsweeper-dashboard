The PR changes sandbox skill-sync fallback to use `resolveAgentWorkspaceDir` when `workspaceDir` is omitted or blank, and adds two regression tests for omitted and empty-string inputs.

## Best Possible Solution

Land the focused sandbox fallback fix after maintainer acceptance of the configured-workspace semantics, keep Gateway install/update routing separate in https://github.com/openclaw/openclaw/pull/89767, and clean up the stale proof-script wording if maintainers want the proof command rerunnable.

## Solution Assessment

Yes. Reusing `resolveAgentWorkspaceDir` at the sandbox layout fallback is the narrowest maintainable fix; changing only callers would leave the exported fallback behavior wrong, and the Gateway workspace-routing PR covers a distinct mutation API path.
