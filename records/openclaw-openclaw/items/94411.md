## What This Changes

The PR changes sandbox skill-sync fallback to resolve the agent workspace from OpenClaw config when `workspaceDir` is omitted or blank, with focused regression tests.

## Best Possible Solution

Land the narrow fallback change after maintainer acceptance of the config-sensitive behavior, keeping sandbox skill sync aligned with the existing agent workspace resolver and documented config contract.

## Reproduction Assessment

Yes, at source level: current `main` and `v2026.6.9` route omitted sandbox skill-sync workspace input through `DEFAULT_AGENT_WORKSPACE_DIR` even though docs say explicit `agents.defaults.workspace` takes precedence. I did not run a live Windows/npm reproduction in this read-only review.

## Solution Assessment

Yes, the proposed fix uses the existing canonical `resolveAgentWorkspaceDir` resolver at the fallback point instead of introducing a parallel workspace rule. The alternative of relying on every caller to pass a resolved workspace is weaker because the helper still has an omitted/blank-input fallback that must follow the same config contract.
