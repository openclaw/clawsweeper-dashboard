## Summary

Keep this PR open for maintainer review: current main and the latest release still use the env-only/default workspace fallback in the sandbox skill-sync helper, and the branch fixes that with focused tests plus terminal proof. I found no blocking patch defect, but the fallback semantics are compatibility-sensitive because omitted or blank sandbox workspace input can change which workspace supplies skills.

## What This Changes

The PR changes sandbox skill-sync fallback resolution for omitted or blank `workspaceDir`, adds sandbox regression tests, and adds a standalone repro script.

## Best Possible Solution

Land the narrow resolver change once maintainers accept the fallback semantics, keeping `resolveAgentWorkspaceDir` as the canonical source for sandbox skill-sync workspace selection.

## Reproduction Assessment

Yes, at source level. Current main still routes omitted or blank sandbox helper input through `DEFAULT_AGENT_WORKSPACE_DIR`, and the PR body supplies after-fix plus revert-test terminal output for the configured-workspace case.

## Solution Assessment

Yes. Using the existing agent workspace resolver at the sandbox fallback point is the narrowest maintainable fix; passing explicit workspaces everywhere would leave the exported helper inconsistent with the documented config contract.
