- `impact:other`: Malformed Canvas options can cross the CLI boundary or silently discard the requested timeout, producing confusing operator failures.

## Summary

Current main still resolves the node before validating `--invoke-timeout` and accepts any finite snapshot quality, while the clean open closing PR contains the focused fix; keep this issue open until that PR merges.

## What This Changes

Require Canvas CLI invoke timeouts to be positive integers and snapshot quality to stay within the inclusive 0–1 range before node or gateway work begins.

## Best Possible Solution

Land one plugin-local fix that validates invoke timeout before node lookup, enforces inclusive 0–1 snapshot quality, and proves invalid values never reach node resolution or gateway invocation.

## Reproduction Assessment

Yes. Current-main source deterministically shows `--quality 5` passes finite-number parsing and `--invoke-timeout 20ms` becomes an omitted timeout only after node resolution starts.

## Solution Assessment

Yes. Strict plugin-local validation before resolver or gateway work is the narrowest maintainable solution and aligns the CLI with the existing Canvas tool contract.
