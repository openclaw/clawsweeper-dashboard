## Summary

Current main still accepts `--quality 5` and silently drops `--invoke-timeout 20ms` after beginning node resolution, so this issue remains necessary. The defect is narrow, source-reproducible, aligned with repository priorities, and suitable for a focused automated fix PR.

## What This Changes

Validate Canvas snapshot quality as an inclusive 0–1 value and Canvas invoke timeout as a positive integer before node resolution or gateway invocation.

## Best Possible Solution

Keep the fix inside the Canvas plugin: parse and reject invoke timeout before node lookup, enforce an inclusive 0–1 quality bound, and add command-level tests proving neither the resolver nor gateway is called for invalid values.

## Reproduction Assessment

Yes. Current-main source deterministically shows that quality `5` passes finite-number parsing and timeout `20ms` becomes an omitted timeout only after node resolution begins.

## Solution Assessment

Yes. Strict plugin-local validation before resolver or gateway work is the narrowest maintainable solution and reuses established SDK numeric parsing contracts.
