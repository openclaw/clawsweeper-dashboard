- **[P2] Trigger self-heal for sessions-only indexes:** `extensions/memory-core/src/memory/manager-sync-ops.ts:2365-2367`
  - body: The FTS-format self-heal only runs after `search()` calls `sync({ reason: "search" })`, but a clean sessions-only legacy index can have `indexIdentityDirty=true` while `dirty` and `sessionsDirty` stay false. Please include identity dirtiness in the search-sync trigger or mark these indexes dirty when the mismatch is detected.
