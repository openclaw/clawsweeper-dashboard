## Summary

Keep open: the PR addresses a real xAI payload bug with sufficient live proof, but the current diff still introduces a Grok 4.3 default-reasoning regression by advertising `none` as a normal supported effort.

## What This Changes

The PR gates OpenAI-compatible reasoning-effort payloads behind xAI compat metadata and adds focused tests for older xAI reasoning models.

## Best Possible Solution

Keep the provider-owned compat gate, but handle disabled reasoning separately so older xAI models suppress unsupported controls while Grok 4.3 keeps omitted/default low reasoning unless the user explicitly disables it.

## Reproduction Assessment

Yes. Current main's source path can emit unsupported xAI Responses reasoning controls for older reasoning-capable models, and the PR body includes live xAI 400/200 proof for the unsupported and fixed payloads.

## Solution Assessment

No. The provider-owned compat approach is the right fix direction, but the current implementation should not make default Grok 4.3 calls send `reasoning: { effort: "none" }`.

## Review Findings
