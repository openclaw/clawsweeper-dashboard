## Summary

Keep this PR open: it addresses a real xAI Responses compatibility bug and the supplied live proof is sufficient, but the current branch still has a blocking Grok 4.3 default-reasoning regression identified by the prior ClawSweeper review.

## What This Changes

The PR adds xAI compat metadata, payload gating, and focused tests so OpenAI-compatible reasoning controls are emitted only for models marked as supporting configurable reasoning effort.

## Best Possible Solution

Keep the provider-owned compat gate, but represent disabled reasoning separately so older xAI models suppress unsupported controls while Grok 4.3 keeps omitted/default low reasoning unless the user explicitly turns reasoning off.

## Reproduction Assessment

Yes. Current main’s source path can emit unsupported xAI reasoning controls for older reasoning-capable models, and the PR body includes live xAI 400/200 proof for the unsupported and fixed payloads.

## Solution Assessment

No. The provider-owned compat direction is the narrow maintainable fix, but this branch should not make ordinary Grok 4.3 Responses calls send `reasoning: { effort: "none" }`.
