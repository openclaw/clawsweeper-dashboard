## Summary

The PR should stay open: it fixes a real xAI payload bug with sufficient live proof, but the current branch regresses ordinary Grok 4.3 calls by advertising `none` as a supported effort and causing the no-options Responses path to send disabled reasoning.

## What This Changes

The branch gates OpenAI-compatible reasoning controls for older xAI reasoning models and adds focused xAI/core tests around the compat metadata.

## Best Possible Solution

Keep the provider-owned compat gate, but separate explicit reasoning disable from advertised configurable efforts so older xAI models suppress unsupported controls while Grok 4.3 keeps omitted/default low reasoning unless the user explicitly turns reasoning off.

## Reproduction Assessment

Yes. Current main's source path can emit unsupported xAI Responses reasoning controls for older reasoning-capable models, and the PR body includes live xAI proof for the rejected control payload and accepted stripped payload.

## Solution Assessment

No. The provider-owned compat approach is the right direction, but the current diff should not make default Grok 4.3 calls send `reasoning: { effort: "none" }`.
