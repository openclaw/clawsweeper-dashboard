## Summary

Keep this PR open: it targets a real xAI reasoning-effort regression and includes sufficient live proof, but the current diff introduces a Grok 4.3 default-reasoning regression that should be repaired before merge.

## What This Changes

The branch adds xAI compat metadata to suppress unsupported OpenAI-compatible reasoning-effort payloads for older Grok models, strips those controls in the xAI wrapper, and adds targeted tests.

## Best Possible Solution

Keep the provider-owned compat gate, but separate explicit disabled thinking from advertised configurable efforts so older xAI models suppress unsupported controls while Grok 4.3 omitted/default calls keep low reasoning.

## Reproduction Assessment

Yes. Current main's source path can emit unsupported xAI Responses reasoning controls for older reasoning-capable models, and the PR body includes live xAI 400/200 proof for the unsupported and fixed payloads.

## Solution Assessment

No. The compat direction is maintainable, but the current implementation should not advertise `none` in a way that turns ordinary Grok 4.3 calls into explicit reasoning-disabled calls.
