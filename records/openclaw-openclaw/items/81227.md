## Summary

Keep open: the PR fixes a real xAI compatibility bug, but the current diff appears to introduce a Grok 4.3 default-behavior regression by advertising `none` as a supported effort through the generic Responses resolver.

## What This Changes

The PR gates OpenAI-compatible reasoning payloads on compat metadata, limits configurable xAI reasoning effort to Grok 4.3 models, strips disabled xAI reasoning controls, and adds regression tests.

## Best Possible Solution

Keep the provider-owned compat gate, but adjust Grok 4.3 effort metadata/default handling so older xAI reasoning models omit unsupported controls while Grok 4.3 preserves xAI's default low reasoning unless the user explicitly selects off or none.

## Reproduction Assessment

Yes. Current main's source path can emit unsupported Responses reasoning/include controls for older reasoning-capable xAI models, and the PR body includes live xAI 400/200 proof for the old and fixed payloads.

## Solution Assessment

No. The provider-owned compat approach is the right direction, but the current implementation should not make ordinary Grok 4.3 calls send `reasoning: { effort: "none" }` when no explicit thinking override was requested.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88
