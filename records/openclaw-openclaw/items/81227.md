## Summary

Keep open: the PR fixes a real xAI provider rejection for older reasoning-capable Grok models, but the current branch introduces a Grok 4.3 default-regression by advertising `none` as a supported effort and causing no-option calls to disable reasoning.

## What This Changes

The PR adds xAI compat metadata and OpenAI-compatible transport/wrapper checks so unsupported Grok reasoning models omit reasoning-effort payload fields, with focused regression tests.

## Best Possible Solution

Keep the provider-owned compat gate, but separate explicit disabled reasoning from the advertised supported-effort list so older xAI models suppress unsupported controls while Grok 4.3 default calls omit effort and retain xAI's default low reasoning.

## Reproduction Assessment

Yes. Current main's source path can emit unsupported xAI Responses reasoning controls for older reasoning-capable models, and the PR body includes live xAI 400/200 proof for the unsupported and fixed payloads.

## Solution Assessment

No. The fix direction is narrow and maintainable, but this branch should not advertise `none` in a way that makes default Grok 4.3 calls explicitly disable reasoning.
