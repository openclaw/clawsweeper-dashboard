## Summary

Keep open: this PR addresses a real xAI payload bug and has sufficient live proof, but the branch still has a blocking Grok 4.3 default-reasoning regression because it advertises `none` as a supported effort while the no-options Responses path treats that as an explicit disable. ([docs.x.ai](https://docs.x.ai/developers/model-capabilities/text/reasoning))

## What This Changes

The PR adds xAI compat metadata and tests so older Grok reasoning models suppress OpenAI-compatible reasoning-effort fields while Grok 4.3 remains configurable.

## Best Possible Solution

Keep the provider-owned compat gate, but separate explicit disabled reasoning from the advertised supported-effort list so unsupported older Grok routes drop controls while default Grok 4.3 calls preserve xAI's low reasoning default.

## Reproduction Assessment

Yes. Current main's source path can emit unsupported xAI Responses reasoning controls for older reasoning-capable models, and the PR body supplies live xAI 400/200 proof for the unsupported and fixed older-model payloads.

## Solution Assessment

No. The fix direction is narrow and maintainable, but the branch should not make ordinary Grok 4.3 no-options calls send explicit `none` reasoning.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9
