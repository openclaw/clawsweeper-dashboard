## Summary

Keep open: the xAI compat-gate direction is valid and well proved, but the current branch can disable Grok 4.3's documented default reasoning on ordinary no-option Responses calls.

## What This Changes

The PR gates OpenAI-compatible reasoning-effort payloads behind xAI runtime compat metadata and adds tests for older Grok reasoning models.

## Best Possible Solution

Keep the provider-owned compat gate, but separate explicit-off handling from advertised supported efforts so older xAI models suppress unsupported controls while Grok 4.3 preserves its default low reasoning.

## Reproduction Assessment

Yes. Source inspection shows current main can emit unsupported xAI reasoning controls for older reasoning-capable Grok models, and the PR includes live xAI 400/200 proof for the old and fixed payload shapes.

## Solution Assessment

No. The fix direction is narrow and maintainable, but this branch should not advertise `none` in a way that turns ordinary Grok 4.3 calls into explicit disabled-reasoning calls.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9
