## Summary

Keep open: the PR addresses a real xAI provider-validation bug and has sufficient live proof, but the current patch introduces a Grok 4.3 default-reasoning regression that should block merge.

## What This Changes

The PR gates xAI OpenAI-compatible reasoning-effort payload fields behind runtime compat metadata and adds tests for older Grok reasoning models.

## Best Possible Solution

Keep the provider-owned compat gate, but separate disabled reasoning from the advertised supported-effort list so older xAI models suppress unsupported controls while Grok 4.3 preserves default low reasoning unless the user explicitly disables it.

## Reproduction Assessment

Yes. Current main's source path can emit unsupported xAI Responses reasoning controls for older reasoning-capable models, and the PR body includes live xAI 400/200 proof for the unsupported and fixed payloads.

## Solution Assessment

No. The fix direction is narrow and maintainable, but the branch should not advertise `none` in a way that makes default Grok 4.3 calls explicitly disable reasoning.
