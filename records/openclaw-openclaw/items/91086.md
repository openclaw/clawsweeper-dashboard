## Best Possible Solution

Track this as a macOS update-handoff availability bug: confirm the actual launchd label/script source, make one-shot update jobs self-disarming or non-KeepAlive, extend legacy cleanup to the real labels if needed, and prove the fix with a macOS launchd update.run scenario.
