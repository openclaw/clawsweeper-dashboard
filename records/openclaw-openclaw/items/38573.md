## Summary

Keep open: current main and v2026.6.5 still force thread/topic cron completions through direct outbound delivery, and no schema, docs, or linked merged work provides the requested opt-in main-session agent-turn route. This remains a product/config contract decision, not conservative cleanup.

## What This Changes

Requests a configurable route mode so text-only topic/forum isolated cron completions can opt into main-session announce/agent-turn delivery instead of forced direct send.

## Best Possible Solution

Define one per-job text-only completion-routing option, keep direct topic delivery as the default, and specify silence, error, structured-payload, transcript-continuity, and fallback semantics before implementation.

## Reproduction Assessment

Yes, at source level. Current main checks `deliveryPayloadHasStructuredContent || resolvedDelivery.threadId != null`, and schema/docs expose no opt-in main-session agent-turn route for topic cron completions.

## Solution Assessment

Unclear. The desired workflow is coherent, but the API location and fallback/silence contract need maintainer selection before this is the best fix shape.
