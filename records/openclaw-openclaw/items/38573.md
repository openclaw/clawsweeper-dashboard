## Summary

Keep open: current main and v2026.5.12 still force any resolved thread/topic isolated cron completion through direct delivery, and the public cron contract has no opt-in selector for main-session announce/agent-turn routing. The related subagent work solved group/topic completion routing generally, but it does not provide the cron route selector requested here.

## What This Changes

The issue requests an opt-in cron delivery setting so text-only topic/forum isolated cron completions can use main-session announce/agent-turn routing instead of mandatory direct delivery.

## Best Possible Solution

Add an additive, documented selector for text-only isolated cron completions to thread/topic targets, preserve direct delivery as the default, keep structured payloads direct, and define explicit fallback semantics.

## Reproduction Assessment

Yes, for the current limitation: current main and v2026.5.12 compute direct delivery from structured content or any resolved threadId, while the type, gateway schema, CLI, and docs expose no opt-in route selector. I did not run a live Telegram cron delivery flow.

## Solution Assessment

Unclear until maintainers choose the public option shape and fallback policy. The narrow maintainable direction is an additive selector that preserves current direct-send defaults rather than changing topic/forum behavior globally.
