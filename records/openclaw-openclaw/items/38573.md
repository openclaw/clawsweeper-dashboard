## Summary

Keep open: current main still forces text-only threaded/topic isolated cron completions through direct outbound delivery, and the cron delivery contract has no opt-in selector for the requested announce/main-agent-turn route.

## What This Changes

The issue requests an additive config or per-cron option that lets text-only topic/forum cron completions use announce/main-agent-turn delivery instead of forced direct send.

## Best Possible Solution

Add an additive, default-preserving route selector for text-only isolated cron completions to thread/topic targets, keep structured payloads direct, and document/test explicit failure fallback behavior across protocol, CLI/tool, cron runtime, and channel delivery.

## Reproduction Assessment

Yes, for the current limitation: source inspection on current main shows any resolved threadId selects the direct branch, and the delivery schema/tool surfaces expose no opt-in route selector. I did not run a live cron delivery flow in this read-only review.

## Solution Assessment

Unclear until maintainers choose the public option shape and fallback policy. The maintainable direction is an additive selector that preserves current direct-send defaults rather than changing topic/forum behavior globally.
