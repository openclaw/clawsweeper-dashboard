## Label Justifications

- `P2`: The limitation consistently affects an advanced cron/topic workflow, but current direct delivery still sends cron output to the target thread.
- `impact:session-state`: The requested mode changes whether isolated cron completion output re-enters the associated main topic session for agent processing and continued conversation.

## Summary

Keep open. Current main still forces thread-bound isolated cron completions through direct outbound delivery, and the requested opt-in main-session announce/agent-turn route is absent; the change would add public cron routing/config and fallback semantics, so it needs maintainer product/API review before implementation.

## What This Changes

Allow isolated cron text completions for topic/forum targets to opt into main-session announce/agent-turn delivery instead of always direct-sending to the thread.

## Best Possible Solution

Define one job-level text-completion routing enum, keep direct topic delivery as the default, preserve topic identity, and specify delivery status plus fallback behavior for silent, failed, and partially delivered main-agent turns.

## Reproduction Assessment

Yes. Source inspection reproduces the limitation with high confidence because current main deterministically selects direct delivery for any resolved cron target with `threadId`, leaving no opt-in route to the main-session announce flow.

## Solution Assessment

Unclear. The opt-in direction is reasonable, but maintainers still need to choose the durable API shape and the delivery/fallback semantics before this becomes an implementation task.

## Risks / Open Questions

- An announce-triggered main-agent turn can intentionally produce no visible reply, so counting that route as delivery without explicit semantics could silently drop scheduled output.
- Fallback from a failed or partially delivered main-agent turn to direct delivery could duplicate cron output if the agent already sent through a message tool or partial channel send.
- A persisted routing option would affect cron schemas, storage codecs, gateway/CLI inputs, docs, delivery status, and upgrade expectations.
