## Summary

Keep open. Current main and v2026.6.10 still force thread/topic isolated cron completions through direct outbound delivery, while the requested opt-in main-session announce/agent-turn route is absent and needs a maintainer API/fallback decision.

## What This Changes

The issue requests an opt-in cron delivery mode that lets text-only isolated cron completions for thread/topic targets use the main-session announce/agent-turn path instead of always direct-sending.

## Best Possible Solution

Define one explicit job-level text-completion routing enum, keep direct topic delivery as the default, preserve topic identity, and specify delivery status plus fallback behavior for silent, failed, and partially delivered main-agent turns.

## Reproduction Assessment

Yes. Source inspection reproduces the limitation with high confidence because current main deterministically selects direct delivery for any resolved cron target with threadId, and no current cron config or job field opts into a main-session announce-agent-turn route.

## Solution Assessment

Unclear. The opt-in direction is reasonable, but maintainers still need to choose the durable API shape and silent, failed, partial, and fallback semantics before this becomes a safe implementation task.
