## Summary

Keep open. Current main still has the reported thread/topic cron delivery limitation, and the public cron config/protocol/tool/CLI surfaces still do not expose a supported opt-in for text-only topic/forum completions to route through a main-session announce/agent-turn flow.

## What This Changes

Request an opt-in cron delivery mode that lets text-only isolated cron completions for thread/topic targets use the main-session announce/agent-turn path while keeping direct delivery as the default.

## Best Possible Solution

Add a backward-compatible explicit route selector for text-only isolated cron completions targeting thread/topic destinations, keep structured payloads and the default thread/topic behavior on direct delivery, define fail-closed versus direct-fallback behavior, and update cron types, normalization, Gateway schema, agent tool/CLI/docs, and targeted tests together.

## Reproduction Assessment

Yes for the current limitation: static inspection of current main shows a resolved threadId forces direct delivery, and the supported cron delivery contracts have no opt-in route selector. No live Telegram/forum cron run was needed to verify that missing API surface.

## Solution Assessment

Unclear until maintainers choose the API shape. The requested direction is reasonable, but the maintainable fix should be an additive public contract with explicit fallback semantics, not a silent change to the existing direct-delivery default.
