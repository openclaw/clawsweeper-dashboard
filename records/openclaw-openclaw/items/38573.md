## Summary

Keep open. Current main still routes isolated cron deliveries with any resolved thread/topic id through the direct outbound path, and the public cron type, Gateway schema, agent tool schema, CLI/docs surface do not expose an opt-in selector for text-only topic/forum completions to use a main-session announce/agent-turn route. The related subagent announce work covers a neighboring path, but not this cron completion-routing contract.

## What This Changes

Add a backward-compatible cron delivery option so text-only isolated cron completions to thread/topic targets can opt into main-session announce/agent-turn routing instead of the current forced direct-send path.

## Best Possible Solution

Add an additive public cron delivery-route selector for isolated text-only completions, preserve current direct delivery as the default for thread/topic and structured payloads, define the fallback policy explicitly, and update normalization, Gateway protocol validation, agent tool schema/help, CLI/docs, and focused regression coverage together.
