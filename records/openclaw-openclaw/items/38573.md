## Summary

Keep open: current main still forces resolved thread/topic isolated cron deliveries through the direct outbound path, and there is no public cron route selector for opting into announce/main-agent-turn delivery.

## What This Changes

Request an opt-in cron delivery mode/config for text-only topic/forum targets to use main-session announce/agent-turn handling while preserving direct delivery as the default.

## Best Possible Solution

Add a backward-compatible explicit route selector for text-only isolated cron completions to thread/topic targets, keep structured payloads and the default topic path on direct delivery, and document/test the chosen fallback semantics.

## Reproduction Assessment

Yes, for the current limitation: source inspection on current main shows any resolved threadId selects direct delivery, and the cron type/schema/CLI/tool surfaces expose no opt-in route selector. A live Telegram forum run is not needed to verify the missing API surface.

## Solution Assessment

Unclear until maintainers choose the public option shape and fallback policy. The requested direction is reasonable, but the maintainable fix should be additive and keep the current direct-delivery default.
