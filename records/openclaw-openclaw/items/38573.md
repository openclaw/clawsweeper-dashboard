## Summary

Keep open: current main and v2026.6.5 intentionally retain direct delivery for topic/thread cron targets, no equivalent configurable main-session agent-turn route exists, and the remaining work requires a maintainer product decision rather than cleanup.

## What This Changes

Add an opt-in, text-only cron completion route that lets thread/topic targets use the main-session announce and agent-turn flow while retaining direct delivery as the default.

## Best Possible Solution

Define one per-job, text-only completion-routing enum, keep direct topic delivery as the default, and specify deterministic behavior for silence, agent errors, structured payloads, transcript continuity, and optional direct fallback before implementation.

## Reproduction Assessment

Yes. Current source, regression coverage, public documentation, and v2026.6.5 all show that resolved thread/topic targets use direct delivery and expose no opt-in main-session agent-turn route.

## Solution Assessment

Unclear. The requested workflow is coherent, but the proposed API locations and fallback policies are alternatives; the narrowest maintainable approach is a single per-job route selected after owners decide the silence and failure contract.
