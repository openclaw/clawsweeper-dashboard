## Summary

Keep open: current main still hard-codes resolved thread/topic isolated cron fallback delivery through the direct outbound path, and the public cron API surfaces do not expose an opt-in announce/main-agent-turn route selector. This is a valid feature/API gap that needs maintainer design, not a shipped fix or duplicate cleanup.

## What This Changes

Add an explicit cron delivery option for text-only thread/topic targets to opt into announce/main-agent-turn routing while keeping direct delivery as the default.

## Best Possible Solution

Add an additive, default-preserving route selector for text-only isolated cron completions to thread/topic targets, keep structured payloads direct, and document/test explicit fallback semantics across protocol, CLI, agent tool, UI, and docs surfaces.

## Reproduction Assessment

Yes, for the current limitation: source inspection on current main shows any resolved `threadId` selects the direct branch, and tests assert Telegram topic direct delivery. This is a feature/API gap rather than a broken existing contract.

## Solution Assessment

Unclear until maintainers choose the public option shape and fallback policy. The maintainable direction is an additive, default-preserving selector rather than changing the existing topic default globally.
