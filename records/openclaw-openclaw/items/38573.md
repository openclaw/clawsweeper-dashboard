## Summary

Keep open: current main and v2026.5.12 still force thread/topic isolated cron completions through direct delivery, and the public cron contract has no opt-in selector for main-session announce/agent-turn routing. This is a valid feature/config request that needs maintainer API and fallback-policy direction before implementation.

## What This Changes

Request to add a config or per-cron option so text-only topic/forum cron completions can opt into main-session announce/agent-turn routing instead of forced direct delivery.

## Best Possible Solution

Add an additive, documented selector for text-only isolated cron completions to thread/topic targets, preserve current direct delivery as the default, keep structured payloads direct, and define explicit fallback semantics.

## Reproduction Assessment

Yes, for the current limitation: current main and v2026.5.12 compute direct delivery from structured content or any threadId, while the cron type, gateway schema, CLI, and docs expose no opt-in route selector. I did not run a live Telegram cron delivery flow.

## Solution Assessment

Unclear until maintainers choose the public option shape and fallback policy. The maintainable direction is an additive selector that preserves current direct-send defaults rather than changing topic/forum behavior globally.
