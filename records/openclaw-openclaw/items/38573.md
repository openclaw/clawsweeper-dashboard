## Summary

Keep open: current main still routes resolved thread/topic isolated cron fallback delivery through the direct outbound path, and the public cron schema, CLI, tool, docs, and UI surfaces do not expose an opt-in announce/main-agent-turn route selector.

## What This Changes

Add a backward-compatible cron option that lets text-only topic/forum isolated cron completions use main-session announce/agent-turn delivery instead of forced direct delivery.

## Best Possible Solution

Add an explicit additive route selector for text-only isolated cron completions to thread/topic targets, keep current direct delivery as the default, leave structured payloads direct, and document/test the fallback semantics.

## Reproduction Assessment

Yes, for the current limitation: source inspection on current main shows any resolved `threadId` selects direct delivery, tests assert Telegram topic direct delivery, and the public cron surfaces expose no opt-in route selector. This is a feature/API gap rather than a broken existing contract.

## Solution Assessment

Unclear until maintainers choose the public option shape and fallback policy. The maintainable direction is an additive, default-preserving route selector rather than flipping the hardcoded topic behavior globally.
