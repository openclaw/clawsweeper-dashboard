## Summary

Keep open: current main still forces resolved thread/topic isolated cron completions through direct delivery, and the public cron contract does not expose a selector for opting text-only topic deliveries into the main-session announce/agent-turn path. This is a valid feature/config request that needs maintainer API and fallback-policy judgment before implementation.

## What This Changes

Request to add a config or per-cron option so text-only thread/topic cron completions can opt into main-session announce/agent-turn delivery instead of forced direct send.

## Best Possible Solution

Add an additive, documented selector for text-only isolated cron completions to thread/topic targets, keep structured payloads direct, preserve the current default, and define explicit fallback semantics.

## Reproduction Assessment

Yes, for the current limitation: source inspection on current main and v2026.5.12 shows any resolved threadId selects direct delivery while the type, gateway, CLI, tool, and docs surfaces expose no opt-in route selector. I did not run a live Telegram cron delivery flow in this read-only review.

## Solution Assessment

Unclear until maintainers choose the public option shape and fallback policy. The maintainable direction is an additive selector that preserves current direct-send defaults rather than changing topic/forum behavior globally.
