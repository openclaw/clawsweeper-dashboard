## Summary

Keep open: current main and the latest release still route any resolved thread/topic cron completion through direct delivery, and the public cron contract has no selector for opting text-only topic deliveries into the main-session announce/agent-turn path. This is a valid feature/config request that needs maintainer API and fallback-policy judgment before implementation.

## What This Changes

The issue requests a default-preserving cron delivery option so topic/forum text completions can use the main-session announce/agent-turn route instead of forced direct delivery.

## Best Possible Solution

Add an additive, documented selector for text-only isolated cron completions to thread/topic targets, keep structured payloads direct, preserve the current default, and define explicit fallback semantics.

## Reproduction Assessment

Yes, for the current limitation: source inspection on current main and v2026.5.12 shows any resolved threadId selects direct delivery, while the type, gateway, CLI, and tool surfaces expose no opt-in route selector. I did not run a live Telegram cron delivery flow in this read-only review.

## Solution Assessment

Unclear until maintainers choose the public option shape and fallback policy. The maintainable direction is an additive selector that preserves current direct-send defaults rather than changing topic/forum behavior globally.
