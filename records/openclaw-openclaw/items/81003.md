## Summary

Keep open: current main still has the resume-token reuse path and no Gateway or Control UI hard-reset surface; the existing `/reset soft` command is a workaround but does not settle the requested API/UX direction.

## What This Changes

Request a deliberate Gateway/Control UI hard-reset action that closes a Claude live session and forces the next CLI run to abandon the stored Claude CLI resume token while preserving OpenClaw chat history.

## Best Possible Solution

Add an explicit, documented hard-recovery control that clears persisted CLI resume bindings for a deliberate reset and wires Gateway protocol, Control UI/TUI behavior, docs, and regression coverage consistently.

## Reproduction Assessment

No, there is not a high-confidence live reproduction of the corrupted Claude transcript loop. Source inspection does show current main can preserve a Claude CLI binding through Gateway reset and then resume from it on the next CLI run.

## Solution Assessment

Unclear: a hard-reset control is plausible, but maintainers should choose the API and UX shape before implementation. The narrowest maintainable direction is to reuse the existing CLI-binding clear semantics instead of inventing a second resume-state store.
