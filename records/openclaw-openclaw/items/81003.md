## Summary

Keep open: current main still lacks the requested Gateway/Control UI hard-reset surface, while the existing restart/reset paths can continue to reuse persisted Claude CLI resume bindings.

## What This Changes

Request a deliberate hard-recovery control that discards the Claude CLI resume token for the next run while preserving OpenClaw-visible chat history.

## Best Possible Solution

Expose a deliberate documented hard-recovery control that clears CLI resume bindings for recovery, preserves visible chat history, and updates the Gateway protocol plus Control UI consistently.

## Reproduction Assessment

No, there is no high-confidence live reproduction of the corrupted Claude transcript loop on current main. Source inspection does show the relevant path: persisted CLI bindings make `useResume=true`, live-session restart does not discard them, and no hard-reset RPC/UI exists.

## Solution Assessment

Unclear: an explicit hard-reset control is plausible, but maintainers should choose whether this belongs as `sessions.hardReset`, a `sessions.reset` option, or a Control UI wrapper around existing CLI-binding clear semantics.
