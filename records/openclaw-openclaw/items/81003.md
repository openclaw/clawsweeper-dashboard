## Summary

Keep open: current main still lacks a Gateway or Control UI hard-reset surface, and the existing reset path intentionally preserves Claude CLI resume bindings. The source-supported workaround is `/reset soft`, but exposing that recovery semantics to operator clients needs a maintainer API/product decision.

## What This Changes

Add a hard recovery control that abandons Claude CLI resume tokens after a live-session restart while preserving OpenClaw-visible chat history.

## Best Possible Solution

Expose a deliberate documented Gateway/Control UI hard-reset action that clears CLI session bindings once while preserving OpenClaw chat history, with protocol, UI, and regression tests updated together.

## Reproduction Assessment

No, there is not a high-confidence live reproduction of the corrupted Claude transcript loop. Source inspection does show the relevant current-main path: `sessions.reset` preserves CLI bindings and CLI dispatch resumes when a binding is present.

## Solution Assessment

Unclear: a hard-recovery control is plausible, but maintainers should choose whether this is a new `sessions.hardReset` RPC, a `sessions.reset` option, or a Control UI wrapper around `/reset soft` semantics.
