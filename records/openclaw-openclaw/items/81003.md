## Summary

Keep open: current main still lacks the requested Gateway/Control UI hard-recovery surface. The existing `/reset soft` path clears CLI resume bindings, but the Gateway `sessions.reset` and Claude live-session restart paths still preserve or reuse the saved Claude CLI session binding.

## What This Changes

The issue requests a hard recovery control that abandons a stuck Claude CLI resume token while preserving OpenClaw-visible chat history.

## Best Possible Solution

Expose a deliberate, documented hard-recovery control that reuses the existing CLI-binding clear semantics and updates the Gateway protocol, Control UI, tests, and user-facing docs consistently.

## Reproduction Assessment

No, there is not a high-confidence live reproduction of the corrupted Claude transcript loop. Source inspection does show that current main can reuse a persisted Claude CLI binding after a live-session restart, and no hard-reset RPC or UI surface exists.

## Solution Assessment

Unclear: the requested hard-reset control is plausible, but maintainers should decide whether the best surface is a new `sessions.hardReset` method, an option on `sessions.reset`, or a Control UI path around the existing CLI-binding clear behavior.
