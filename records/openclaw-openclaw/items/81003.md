## Summary

Keep open: current main still lacks the requested Gateway/Control UI hard-recovery surface, even though `/reset soft` can already clear CLI resume bindings from a chat command. Source inspection supports the resume-token reuse path, but the exact corrupted Claude transcript loop was not live-reproduced and the public API/UX shape needs maintainer direction.

## What This Changes

Add a deliberate Gateway/Control UI hard-reset path that discards the Claude CLI resume token for a recovery turn while preserving OpenClaw chat history.

## Best Possible Solution

Expose a deliberate documented hard-recovery control that reuses the existing CLI-binding clear semantics, preserves OpenClaw-visible history, and updates the Gateway protocol plus Control UI consistently.

## Reproduction Assessment

No, there is not a high-confidence live reproduction of the corrupted Claude transcript loop. Source inspection does show current main can restart a Claude live process while still reusing the persisted Claude CLI resume binding, and no hard-reset RPC/UI surface exists.

## Solution Assessment

Unclear: an explicit hard-reset control is plausible, but maintainers should choose whether this is a new `sessions.hardReset` method, a `sessions.reset` option, or a Control UI wrapper around existing soft-reset semantics.
