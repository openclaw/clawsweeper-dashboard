## Summary

Keep open: current main still has a slash-command recovery path that clears CLI resume bindings, but it does not expose the requested Gateway/Control UI hard-reset surface, and the existing `sessions.reset` path preserves CLI resume state.

## What This Changes

Add a hard-recovery Gateway/Control UI action that discards Claude CLI resume state for the next run while preserving OpenClaw-visible chat history.

## Best Possible Solution

Expose a deliberate, documented hard-recovery control that reuses existing CLI-binding clear semantics, preserves OpenClaw-visible history, and updates the Gateway protocol plus Control UI consistently.

## Reproduction Assessment

No, there is not a high-confidence live reproduction of the corrupted Claude transcript loop. Source inspection does show the relevant current-main behavior: stored CLI bindings drive resume, live restarts do not discard them, and no hard-reset RPC/UI surface exists.

## Solution Assessment

Unclear: an explicit `sessions.hardReset` is plausible, but maintainers should decide whether this should be a new method, an option on `sessions.reset`, or a Control UI wrapper around `/reset soft` semantics.
