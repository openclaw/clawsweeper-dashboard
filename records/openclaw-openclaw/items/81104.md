Full review comments:

- **[P1] Reject array policy containers at runtime:** `extensions/policy/src/runtime-tool-policy.ts:441-443`
  - body: `validateRuntimePolicyShape()` relies on `isRecord()` to decide whether policy roots and sections are valid, but this helper accepts arrays. With `policy.jsonc` like `[]` or `{ "tools": [] }`, the runtime gate can treat an invalid policy container as valid and then proceed with no tool requirements instead of failing closed. Please reject arrays in this guard and add runtime tests for array-shaped roots/sections.
  - confidence: 0.94
- **[P1] Keep secret-file symlink rejection fail-closed:** `src/infra/secret-file.test.ts:123-125`
  - body: Current main expects `tryReadSecretFileSync(..., { rejectSymlink: true })` to throw for rejected symlinked secret files, and channel credential tests rely on that fail-closed behavior. This PR changes the test to expect `undefined`, which either fails against the current implementation or weakens the documented security contract. Please keep the throwing expectation for rejected files and reserve `undefined` for missing/blank optional paths.
  - confidence: 0.92
