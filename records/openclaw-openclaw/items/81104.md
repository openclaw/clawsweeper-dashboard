## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Keep policy checks read-only:** `extensions/policy/src/cli.ts:142`
  - body: `policy check` and `policy watch` build reports through `readConfigFileSnapshot()` without `{ observe: false }`. Because default snapshot reads observe config state, these audit commands can update `.openclaw/logs/config-health.json` despite the docs saying they are read-only; pass the non-observing option and cover the command path.
  - confidence: 0.91
- **[P2] Keep final lint validation read-only:** `src/flows/doctor-core-checks.ts:170`
  - body: `doctor --lint` starts with a non-observing config snapshot, but the final config validation check calls `readConfigFileSnapshot()` again with default observing behavior. That lets a documented read-only lint run touch config observation state; pass `{ observe: false }` here or reuse the existing lint snapshot.
  - confidence: 0.9
