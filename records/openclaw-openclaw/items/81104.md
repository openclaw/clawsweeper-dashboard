## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Reject array-shaped policy containers:** `extensions/policy/src/runtime-tool-policy.ts:441-442`
  - body: The new runtime policy validator relies on `isRecord`, but this helper returns true for arrays. A `policy.jsonc` like `[]` or `{ "tools": [] }` therefore passes `validateRuntimePolicyShape`, produces an attestation, and lets the runtime gate continue instead of failing closed on malformed policy; current main's policy conformance helper excludes arrays for this reason. Make this a plain-object guard and cover array-shaped top-level and nested policy containers.
  - confidence: 0.88
- **[P1] Keep symlink rejection fail-closed:** `src/infra/secret-file.test.ts:123-125`
  - body: This changes the security regression test to expect `tryReadSecretFileSync(..., { rejectSymlink: true })` to return `undefined` for a symlinked credential file. Current main intentionally expects a throw, and channel credential loaders use this helper with `rejectSymlink` so a rejected secret file is surfaced as a configuration error rather than silently treated as absent. Keep the throwing contract and rebase the branch instead.
  - confidence: 0.84
