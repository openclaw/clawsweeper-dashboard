## Summary

Keep open: the PR addresses a real installer failure and has sufficient after-fix terminal proof, but the latest head still introduces a release-ref regression by letting branch handling preempt immutable tag resolution.

## What This Changes

The branch updates both Bash installers and focused script tests to fetch git main/branch refs without tags, run pnpm installs noninteractively, and use non-frozen lockfiles only for moving git refs.

## Best Possible Solution

Land the installer fix only after preserving no-tag fetches for main/exact branches while keeping tag, semver, and commit installs immutable and ahead of branch fallback in both published installers.

## Reproduction Assessment

Yes for the PR regression: source inspection plus live git ls-remote checks show branch-first handling can preempt valid tag installs. The original main/non-TTY installer failure is source-reproducible from current main, but I did not run a failing current-main install in this read-only review.

## Solution Assessment

No, not in the current form. The direction is right, but the maintainable fix must make remote branch detection exact and keep immutable release refs on the tag/commit path in both installer scripts.
