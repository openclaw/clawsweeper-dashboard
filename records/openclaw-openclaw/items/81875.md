## Summary

Keep open. The patch fixes part of the reported installer path, but the same published git-install flow in `scripts/install-cli.sh` remains unchanged, and the external PR still lacks inspectable after-fix behavior proof.

## What This Changes

The PR changes `scripts/install.sh` so the `main` git install path fetches and pulls without tags and defaults `CI=true` for the git dependency install.

## Best Possible Solution

Update both published Bash installers consistently, then add focused coverage or artifact-backed proof for moved local tags and non-TTY pnpm module purge behavior.

Yes. The current source contains the exact failing commands, a local moved-tag git scenario reproduces the tag clobber rejection, and pnpm 11.1.0 source confirms why `CI=true` bypasses the non-TTY modules purge prompt.

## Solution Assessment

No, not yet. The `scripts/install.sh` edits are the right shape for that script, but the best fix should also cover the matching published `scripts/install-cli.sh` git path and include inspectable real behavior proof.
