## Summary

Keep open: the bug is valid, but the latest PR remains incomplete because the published `install-cli.sh` git path still has the same failure mode, the new `--no-frozen-lockfile` applies to immutable tag installs too, and the after-fix proof is not inspectable.

## What This Changes

The PR modifies `scripts/install.sh` so `main` git checkout fetch/pull skip tags and git-source pnpm install defaults `CI=true` with `--no-frozen-lockfile`.

## Best Possible Solution

Update both published Bash installers consistently, keep immutable tag installs pinned to the checked-in lockfile or narrowly scope non-frozen behavior to moving branch refs, and attach redacted real behavior proof.

## Reproduction Assessment

Yes. Current source shows the failing all-tags fetch and non-CI pnpm install paths, the earlier ClawSweeper comment records a moved-tag reproduction, and pnpm 11.1.0 source confirms why `CI=true` bypasses the non-TTY purge prompt.

## Solution Assessment

No. The no-tag fetch and default `CI=true` direction is reasonable for `main`, but the fix should also cover `install-cli.sh` and avoid disabling frozen lockfile installs for immutable git refs.
