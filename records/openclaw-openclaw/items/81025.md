## Summary

Close: the mismatch is tied to the external `@tencent-weixin/openclaw-weixin` runtime, while current OpenClaw core already carries `accountId` through generic route/session-key construction when a plugin provides it.

## What This Changes

The report asks OpenClaw to prevent WeChat direct-message session context loss caused by pre-restart keys missing the WeChat account id.

## Best Possible Solution

Keep OpenClaw core channel-agnostic and fix this in the external WeChat plugin by resolving a stable account id before recording inbound sessions, with any plugin-created stale-row repair handled by the plugin or a targeted support tool.

## Reproduction Assessment

No. The report gives clear observed keys, but the runtime that appears to create the missing-account key is the external WeChat plugin, whose source is not in this repository; current core source only proves the generic route path handles `accountId` when supplied.

## Solution Assessment

Not applicable as an OpenClaw core patch. The narrow maintainable fix is in `@tencent-weixin/openclaw-weixin`, unless a plugin maintainer identifies a concrete missing OpenClaw SDK seam.
