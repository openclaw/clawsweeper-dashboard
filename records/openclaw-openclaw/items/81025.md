## Summary

Close as not actionable in OpenClaw core: current main provides account-scoped routing helpers and passes accountId into channel startup, while WeChat session generation and account monitoring are owned by the external openclaw-weixin plugin.

## What This Changes

The issue asks OpenClaw to keep WeChat direct-message session keys stable across gateway restarts when `session.dmScope` is `per-account-channel-peer`.

## Best Possible Solution

Fix the external openclaw-weixin runtime to use its stable account id, or the OpenClaw routing helper with accountId, before persisting direct-message sessions.

## Reproduction Assessment

No, not within this repository. The reported live WeChat restart path depends on the external `@tencent-weixin/openclaw-weixin` runtime, while current OpenClaw core source shows the per-account key builder would not omit accountId.

## Solution Assessment

Yes for repository cleanup: closing this core issue and routing the fix to the external plugin is the narrowest maintainable outcome because core already passes accountId and exposes generic routing helpers.
