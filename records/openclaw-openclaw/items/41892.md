## Summary

Keep open. Current main still does not include a cron calendar/timeline view, and this PR is the active implementation for the still-open feature request, but the current head is not merge-ready because the new occurrence helper can still plot cron runs at times or dates that do not match the Gateway scheduler.

## What This Changes

The PR adds a Control UI Cron Jobs timeline strip with high-frequency chips, supporting CSS/i18n/changelog updates, and a `cron.status` timezone field for client-side schedule plotting.

## Best Possible Solution

Land the timeline after occurrence generation is scheduler-compatible: reuse or expose the runtime Croner/stagger semantics, or return a server-computed occurrence summary, then cover staggered jobs and Croner calendar modifiers with focused helper/render tests while coordinating the cron-view split.

## Reproduction Assessment

Yes. A static reproduction is high-confidence: top-of-hour cron jobs have a runtime per-job hash stagger while the PR plots `staggerMs / 2`, and Croner expressions using `L`, `W`, `#`, or `+` are accepted by runtime Croner but treated as wildcard-like constraints by the PR parser.

## Solution Assessment

No. The feature direction is useful, but the current implementation is not the narrowest maintainable fix until the timeline stops approximating scheduler behavior in a separate parser or proves parity with focused tests; a shared runtime-backed helper or server-provided occurrence summary is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Match the scheduler's per-job stagger offset:** `ui/src/ui/helpers/cron-timeline.ts:384-388`
  - body: Runtime cron scheduling hashes the job id to choose a stable offset inside `staggerMs`, including the default top-of-hour window. This helper applies the same average `staggerMs / 2` to every cron job, so markers can be minutes away from actual execution and flip past/pending state or history navigation.
  - confidence: 0.9
- **[P2] Don't wildcard Croner calendar modifiers:** `ui/src/ui/helpers/cron-timeline.ts:431-436`
  - body: Croner supports calendar modifiers such as `L`, `W`, `#`, and `+`, and runtime scheduling delegates the full expression to Croner. This parser skips those tokens and then treats an empty restricted field as wildcard, so last-day, nearest-weekday, nth-weekday, or AND-constrained schedules can show markers on days when they will not run.
  - confidence: 0.86
