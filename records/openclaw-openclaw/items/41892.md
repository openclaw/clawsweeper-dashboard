## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Use the scheduler's stable stagger offset for markers:** `ui/src/ui/helpers/cron-timeline.ts:386-388`
  - body: Current main schedules cron staggering with a stable `sha256(jobId) % staggerMs` offset and tests assert that exact offset. Plotting every staggered cron at `staggerMs / 2` makes the timeline disagree with actual run times for most jobs, which can flip past/pending state and send users to the wrong history window. Reuse the scheduler-compatible offset contract or pass exact runtime next/previous run data instead of using the midpoint.
  - confidence: 0.9
- **[P2] Mark copied locale strings as fallbacks:** `ui/src/i18n/locales/de.ts:1241-1251`
  - body: Several non-English locale bundles add English timeline strings while their metadata reports no fallback keys and all keys translated. That hides untranslated timeline UI from the i18n report/check path; regenerate so these strings are translated or recorded as fallbacks.
  - confidence: 0.86
