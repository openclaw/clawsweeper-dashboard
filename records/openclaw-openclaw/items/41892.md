Full review comments:

- **[P2] Use real cron run status for timeline markers:** `ui/src/ui/helpers/cron-timeline.ts:542`
  - body: The timeline currently sets every past marker to `ok` based only on the clock. A job whose latest run failed or was skipped earlier today will still get a green `ok` marker and popup, even though the existing UI status path resolves `lastRunStatus ?? lastStatus`; please derive marker state from the recorded cron run status instead of past/future time alone.
  - confidence: 0.91
- **[P2] Apply the runtime's deterministic stagger offset:** `ui/src/ui/helpers/cron-timeline.ts:386`
  - body: Runtime cron scheduling uses a deterministic job-id hash inside the configured/default stagger window, but the timeline plots `staggerMs / 2`. For ordinary top-of-hour jobs with the default five-minute stagger, the marker can be minutes away from the actual run and flip past/pending state; pass enough job context to use the same effective offset or avoid pretending the marker is exact.
  - confidence: 0.88
