## Summary

Keep open: this is still a useful Control UI feature candidate that is not on current main, but current PR head still has a scheduler-alignment bug and lacks current-head real behavior proof before merge.

## What This Changes

Adds a Control UI cron 24-hour timeline with helper calculations, view rendering, styles, locale keys, changelog text, and cron status gatewayTimezone plumbing.

## Best Possible Solution

Keep this PR as the active timeline candidate, but make marker computation mirror or share the scheduler's stable per-job/default stagger logic, add focused helper coverage, and require current-head real browser proof before merge.

## Reproduction Assessment

Yes. Source inspection at PR head shows getTimelineMarkers passes only job.schedule into getTodayRunHours, which then adds schedule.staggerMs / 2; current runtime scheduling hashes job.id and applies default top-of-hour stagger windows.

## Solution Assessment

No. The timeline direction is useful, but the stagger implementation should mirror or share scheduler logic instead of approximating with a midpoint that cannot account for job id or default stagger behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88
