Full review comments:

- **[P2] Regenerate locale output before marking it translated:** `ui/src/i18n/locales/de.ts:1241-1251`
  - body: The new German timeline strings are still English, but the generated German metadata has no fallback keys and marks all keys translated. German users would see English copy while the i18n report cannot flag it as fallback, so regenerate the locale bundle or mark these keys as fallback before merge.
  - confidence: 0.91
- **[P2] Match Croner special tokens instead of wildcarding them:** `ui/src/ui/helpers/cron-timeline.ts:431-436`
  - body: Runtime scheduling uses Croner, and Croner supports `L`, `W`, and `#`, but this parser skips those tokens and lets empty restricted fields behave like wildcards. A valid schedule such as a last-day or nearest-weekday expression can therefore appear on every day in the timeline even though the scheduler will only run it on constrained dates.
  - confidence: 0.87
- **[P2] Use the per-job stagger offset for timeline markers:** `ui/src/ui/helpers/cron-timeline.ts:384-388`
  - body: The runtime hashes the job id to compute a deterministic stagger offset, but the timeline uses `staggerMs / 2` for every job. That makes default top-of-hour cron jobs flip past/pending and show marker times that can differ from the actual scheduled slot, so the timeline should reuse the runtime offset logic or server-computed occurrences.
  - confidence: 0.78
