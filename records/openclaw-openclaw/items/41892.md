## Summary

## What This Changes

The PR adds a Control UI cron timeline with markers, clusters, high-frequency chips, zoom controls, styles, locale keys, changelog text, and cron status timezone plumbing.

## Best Possible Solution

Keep this PR as the active timeline candidate, but make marker computation mirror or share the scheduler's stable per-job stagger offset, add focused helper coverage, and require current-head real behavior proof before merge.

## Reproduction Assessment

Yes. The blocking patch issue is source-reproducible: the PR plots staggered cron jobs at staggerMs / 2, while current runtime scheduling hashes the job id and uses a stable offset modulo the stagger window.

## Solution Assessment

No. The timeline direction is useful, but the implementation should mirror or share the scheduler's stable per-job stagger calculation instead of approximating it with a midpoint.
