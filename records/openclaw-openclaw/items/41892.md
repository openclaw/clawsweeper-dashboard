- **[P1] Preserve current cron.status fields:** `src/cron/service/ops.ts:228`
  - body: This branch adds `gatewayTimezone` on top of an older `cron.status` response, while current main also returns `storage` and `sqlitePath`. Because live GitHub reports the PR as conflicting, conflict resolution must keep the SQLite fields and add timezone additively or newer clients/operators lose the current status contract.
  - confidence: 0.91
- **[P2] Use runtime scheduler occurrences:** `ui/src/ui/helpers/cron-timeline.ts:384-388`
  - body: The timeline applies `staggerMs / 2` to every cron marker, but runtime scheduling uses a stable per-job hash offset through `computeStaggeredCronNextRunAtMs`. This can plot jobs at times they will never actually run; compute markers from the same runtime-owned contract or expose exact occurrences from the gateway.
  - confidence: 0.9
- **[P2] Do not mark past markers as successful:** `ui/src/ui/helpers/cron-timeline.ts:542`
  - body: Past markers are labeled `ok` solely because the scheduled hour is before now. Failed, skipped, disabled, auto-disabled, or never-observed runs can therefore display as successful; derive marker state from job/run state or use a neutral scheduled label when outcome is unknown.
  - confidence: 0.92
- **[P3] Remove the release-owned changelog entry:** `CHANGELOG.md:1988`
  - body: Normal PRs should not edit `CHANGELOG.md`; release generation owns that file. Keep this release-note context in the PR body or squash message and remove the changelog line from the branch.
  - confidence: 0.88
- **[P3] Regenerate generated locale bundles:** `ui/src/i18n/locales/de.ts:1241-1251`
  - body: The UI guide treats non-English locale bundles and `.i18n` metadata as generated output, but this branch adds English strings to several non-English locale files.
