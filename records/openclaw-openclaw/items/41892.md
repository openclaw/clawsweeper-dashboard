## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Use the runtime stagger offset for markers:** `ui/src/ui/helpers/cron-timeline.ts:384-388`
  - body: The timeline shifts cron markers by `schedule.staggerMs / 2`, but runtime scheduling derives a stable offset from `job.id` and `staggerMs`. Two jobs with the same schedule can execute at different offsets, so this can mark the wrong dot as past/pending and send users to irrelevant history; pass job identity into the projection or share the scheduler logic instead of using an average.
  - confidence: 0.9
- **[P3] Regenerate non-English timeline translations:** `ui/src/i18n/locales/de.ts:1241-1251`
  - body: The German bundle declares no fallback keys in metadata, but the new timeline strings are still English. That makes the locale look fully translated while users see English controls; regenerate these through the i18n pipeline or mark them as fallback output.
  - confidence: 0.82
