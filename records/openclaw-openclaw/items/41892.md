## Review Findings

Full review comments:

- **[P1] Use the scheduler's stable stagger offset:** `ui/src/ui/helpers/cron-timeline.ts:384-388`
  - body: The timeline adds `staggerMs / 2` to every cron marker, but current runtime scheduling hashes `job.id` into a stable per-job offset and uses that offset for next/previous run math. Any job whose hash offset is not the midpoint will be plotted at the wrong time, which can flip past/pending state and send users to irrelevant history; derive marker times from the same scheduler semantics instead.
  - confidence: 0.9
- **[P2] Mark fallback strings instead of translated:** `ui/src/i18n/locales/de.ts:1241-1251`
  - body: This locale adds English timeline strings while its generated metadata says there are no fallback keys and all keys are translated. That makes the existing i18n reporting treat untranslated UI as complete; either provide real translations or mark these keys as fallbacks consistently across the locale metadata.
  - confidence: 0.86
