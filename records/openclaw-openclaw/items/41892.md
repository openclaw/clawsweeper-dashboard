Full review comments:

- **[P1] Do not wildcard unsupported Croner tokens:** `ui/src/ui/helpers/cron-timeline.ts:433-436`
  - body: Runtime cron scheduling uses Croner, which supports `L`, `W`, `#`, `+`, and `?` syntax. This parser skips unrecognized tokens and callers treat an empty restricted field as a match, so a supported expression like `0 9 L * *` can render as if it runs every day instead of only the last day; use Croner-backed enumeration or fail closed for unsupported expressions.
  - confidence: 0.91
- **[P2] Use the real per-job stagger for marker times:** `ui/src/ui/helpers/cron-timeline.ts:384-388`
  - body: The runtime derives a deterministic stagger offset from the job id and effective stagger window, including implicit top-of-hour defaults. Adding `schedule.staggerMs / 2` plots an average slot that can differ from the actual scheduled run and can flip past/pending near the boundary; thread the job id/effective stagger into the calculation or reuse the runtime scheduler path.
  - confidence: 0.88
- **[P2] Translate the added German timeline strings:** `ui/src/i18n/locales/de.ts:1241-1251`
  - body: The PR claims German timeline translations, but the German bundle adds English copy such as `Today's Schedule`, `Work`, and `Click → view run history` while its metadata says there are no fallback keys. German users would see English strings in this new UI, so regenerate or supply real translations for declared translated locales.
  - confidence: 0.84
- **[P3] Remove the changelog entry from this PR:** `CHANGELOG.md:1988`
  - body: Root policy says `CHANGELOG.md` is release-owned and normal PRs should put release-note context in the PR body or squash message instead. Please remove this entry and leave release generation to collect it later.
  - confidence: 0.78
