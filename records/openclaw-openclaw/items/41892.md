- **[P2] Use the runtime's deterministic cron stagger offset:** `ui/src/ui/helpers/cron-timeline.ts:384-388`
  - body: The timeline adds `staggerMs / 2` to cron markers, but runtime scheduling hashes the job id into a stable offset and also applies a default top-of-hour stagger.   - confidence: 0.91

  - body: Every marker before the current time is emitted as `status: "ok"`, regardless of `job.state.lastRunStatus` or `lastStatus`. A failed or skipped job will appear green in the timeline even though the existing Cron UI already exposes canonical run status.
  - confidence: 0.89
