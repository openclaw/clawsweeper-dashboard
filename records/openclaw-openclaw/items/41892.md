## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Stop widening Croner-only fields to wildcards:** `ui/src/ui/helpers/cron-timeline.ts:431-436`
  - body: Runtime cron evaluation uses Croner, and Croner supports `L`, `W`, `#`, `+`, and an optional year field. This helper skips unrecognized tokens and its callers treat an empty parse as a match, so a valid schedule such as `0 9 L * *` can be plotted every day instead of only the last day; use Croner/runtime-backed occurrence calculation or show an explicit unable-to-plot state instead of false markers.
  - confidence: 0.9
- **[P2] Use the runtime stagger offset for markers:** `ui/src/ui/helpers/cron-timeline.ts:384-388`
  - body: Runtime scheduling hashes the job id to pick a stable offset inside `staggerMs`, but this helper plots every staggered cron job at the midpoint of the window. That can put the marker and `ok`/`pending` split on the wrong side of the actual run time; pass the job id through and share or mirror the deterministic runtime offset instead of using `staggerMs / 2`.
  - confidence: 0.86
- **[P3] Remove the release-owned changelog entry:** `CHANGELOG.md:1988`
  - body: OpenClaw release generation owns `CHANGELOG.md`, so normal feature PRs should keep release-note context in the PR body or squash message rather than adding a changelog line that can conflict with generated release notes.
  - confidence: 0.82
