Overall correctness: patch is incorrect

Full review comments:

- **[P2] Use the scheduler's stable stagger offset:** `ui/src/ui/helpers/cron-timeline.ts:384-388`
  - body: This plots staggered cron markers at `staggerMs / 2`, but runtime scheduling uses a deterministic SHA-256 offset derived from the job id and stagger window. Any job whose stable offset is not the midpoint can be shown minutes early or late, which makes the `ok`/`pending` split and click-to-history behavior misleading; derive the same stable offset or share scheduler timing semantics instead.
  - confidence: 0.92
- **[P3] Regenerate translated timeline locale metadata:** `ui/src/i18n/locales/de.ts:1241-1251`
  - body: The German bundle adds the new timeline strings in English while its metadata still reports no fallback keys and all keys translated. That makes the i18n report lie about fallback state and ships English copy under a non-English locale; regenerate the locale bundles or mark these keys as fallback metadata consistently.
  - confidence: 0.86
