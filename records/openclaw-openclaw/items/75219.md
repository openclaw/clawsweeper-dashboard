## What This Changes

The PR adds opt-in same-key transient retries to provider API-key execution, enables them for media audio/video and Gemini embeddings, and adds retry regression tests.

## Best Possible Solution

Land the opt-in same-key transient retry path for idempotent provider operations while preserving 429 alternate-key rotation and fail-fast permanent/auth/caller-abort behavior.

## Reproduction Assessment

Yes. Against current main, call `executeWithApiKeyRotation` with one key and an `execute` callback that throws an HTTP 500-style error or `AbortError`; the wrapper calls `execute` once and rethrows because the default predicate is rate-limit-only.

## Solution Assessment

Yes, with one process fix. Extending the existing rotation helper with an explicit `transientRetry` option is the narrow maintainable direction; the PR should add the required changelog entry before merge.
