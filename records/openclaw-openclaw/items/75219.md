## What This Changes

The PR adds opt-in same-key transient retries to `executeWithApiKeyRotation`, enables them for media audio/video and Google embedding executions, and adds retry regression tests.

## Best Possible Solution

Land the opt-in same-key retry path for idempotent provider operations while preserving 429 alternate-key rotation and fail-fast behavior for permanent/auth/caller-aborted errors, with a single-line `CHANGELOG.md` fix entry credited to the contributor.

## Reproduction Assessment

Yes. Against current main, call `executeWithApiKeyRotation` with one API key and an `execute` function that throws an HTTP 500-style error or generic `AbortError`; the wrapper calls `execute` once and rethrows because only rate-limit errors trigger retry/rotation by default.

## Solution Assessment

Yes, with one process fix. Extending the existing helper with an explicit opt-in `transientRetry` option is narrower than global provider retries and keeps the existing rotation contract intact; the PR should add the required changelog entry before merge.
