## What This Changes

The PR adds opt-in same-key transient retries for selected provider executions, enables them for media audio/video and Google embeddings, tags timeout aborts, adds regression tests, and updates the changelog.

## Best Possible Solution

Keep the opt-in same-key retry design and require redacted real provider evidence before landing; after merge, it can close https://github.com/openclaw/openclaw/issues/60422.

## Reproduction Assessment

Yes. Source inspection on current main shows executeWithApiKeyRotation only retries/rotates for shouldRetry or 429-style rate-limit errors, so a one-key 500, timeout, or selected network failure throws after the first provider call.

## Solution Assessment

Yes, pending proof. Extending the existing rotation helper with an explicit opt-in same-key transient retry option is narrower than global retries, and the current PR head preserves 429 key rotation while keeping generic AbortError retries timeout-scoped.
