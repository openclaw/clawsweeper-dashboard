## Summary

Keep open as a normal merge candidate. Current main still excludes terminal `lost` cron rows from durable cron recovery, and this PR is a focused fix with constrained recovery, stale-error clearing, supplied real CLI proof, green CI, and no blocking review findings.

## What This Changes

The PR lets task maintenance recover terminal `lost` cron task rows from exact durable cron run evidence when the lost state came from a backing-session-missing false positive, clears stale terminal errors, adds regression tests, and updates the changelog.

## Best Possible Solution

Task maintenance should recover only terminal cron lost rows whose exact cron execution is proven terminal by durable run logs or job state, and successful recovery should remove the stale lost error.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a `runtime: "cron"`, `status: "lost"`, `error: "backing session missing"` row with `runId: "cron:<jobId>:<startedAt>"` and matching finished durable cron history; current main skips durable recovery because the row is not active.

## Solution Assessment

Yes. The patch uses the existing exact durable cron recovery source, widens eligibility only for empty/backing-session-missing lost rows, and adds the needed stale-error clearing instead of broad pruning or job-id-only recovery.
