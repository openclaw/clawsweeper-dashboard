## What This Changes

The branch extends task maintenance to recover terminal `lost` cron rows from durable cron evidence, clears stale terminal errors when recovery has no error, adds regression tests plus a changelog entry, and tightens two task-delivery test assertions.

## Best Possible Solution

Land the scoped durable-history recovery after normal maintainer review and required checks, while leaving broader stale-running cleanup to the linked follow-up issues.

## Reproduction Assessment

Yes. Source inspection shows current main only attempts durable cron recovery for active cron tasks, so a terminal `lost` cron row with `backing session missing` and matching durable finished-run evidence remains lost unless this PR's widened guard is applied.

## Solution Assessment

Yes. Reusing the existing durable cron recovery path and constraining it to backing-session-missing lost rows is narrower than pruning terminal rows or adding a migration, and the added tests cover both recovery and non-recovery cases.
