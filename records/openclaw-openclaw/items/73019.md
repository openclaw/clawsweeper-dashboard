## Summary

Keep open. Current main still exposes `/voiceclaw/realtime` as a Gemini-only realtime brain path, xAI realtime voice is documented as not exposed, and the same author already has #73032 open as the implementation candidate with closing syntax.

## What This Changes

Request xAI Realtime Voice Agent support for `/voiceclaw/realtime`, preferably through a shared OpenAI-Realtime-protocol adapter plus provider dispatch, config, tests, and docs.

## Best Possible Solution

Review #73032 as the canonical implementation path, then land it or a maintainer-approved variant that settles the adapter boundary, provider dispatch, docs, tests, and secret sanitization together.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection gives high-confidence verification that current main does not accept or dispatch `provider: "xai"` through `/voiceclaw/realtime`.

## Solution Assessment

Unclear until maintainer review.
