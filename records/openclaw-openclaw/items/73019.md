## Summary

Keep open. Current main still lacks xAI selection for `/voiceclaw/realtime`, and open draft PR #73032 is the same-author implementation candidate with closing syntax, so this issue should remain open until that PR is reviewed and either landed or closed.

## What This Changes

Add xAI Realtime Voice Agent support to the `/voiceclaw/realtime` gateway using an OpenAI-Realtime-compatible adapter reusable for future OpenAI Realtime work.

## Best Possible Solution

Use PR #73032 as the canonical implementation path, require provider dispatch plus mocked protocol coverage, docs/changelog, and secret sanitization, and close this issue only after that path is resolved.

## Reproduction Assessment

Yes. Static current-main verification is enough: the provider type excludes `xai`, session startup rewrites provider selection to Gemini, and the xAI docs explicitly say realtime voice is not exposed.

## Solution Assessment

Unclear until maintainer review. A shared OpenAI-Realtime-compatible adapter is plausible, but maintainers should settle whether this belongs in the `/voiceclaw/realtime` gateway path or through the existing plugin realtime voice provider surface.
