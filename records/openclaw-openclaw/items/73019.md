## Summary

Keep open. Current main still lacks xAI dispatch for `/voiceclaw/realtime`, docs say xAI realtime voice is not exposed, and #73032 is the same-author open implementation candidate with closing syntax, so this issue should remain open until that PR is reviewed and either landed or closed.

## What This Changes

The issue requests adding xAI Realtime Voice Agent support to `/voiceclaw/realtime`, ideally through a shared OpenAI-Realtime-protocol adapter usable by xAI and later OpenAI Realtime.

## Best Possible Solution

Review and either land #73032 or a maintainer-approved variant that adds provider dispatch, adapter behavior, docs, tests, and credential sanitization while settling the core-versus-plugin boundary.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives high-confidence feature verification: current main only accepts/dispatches Gemini for `/voiceclaw/realtime`, and docs mark xAI realtime voice as not exposed.

## Solution Assessment

Unclear until maintainer review. The shared adapter direction is plausible, but the maintainable solution depends on whether maintainers want this in the core gateway path, the plugin realtime-voice provider surface, or a shared base used by both.
