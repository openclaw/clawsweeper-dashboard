## Summary

Keep open. Current main still contains the reported empty-roots client-side hard deny before the authoritative server metadata check, and open PR #67916 is already paired as the implementation candidate.

## What This Changes

The issue requests fixing Control UI local assistant media attachments that are marked “Outside allowed folders” when `localMediaPreviewRoots` has not loaded yet.

## Best Possible Solution

Keep this issue open until PR #67916 or an equivalent focused replacement lands with the empty-roots fallback and regression coverage while preserving populated-roots client blocking.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: render a local assistant `MEDIA:` attachment with `localMediaPreviewRoots: []`, and current main returns `Outside allowed folders` before any server metadata request can run.

## Solution Assessment

Yes. The narrow maintainable fix is to treat empty preview roots as a bootstrap-not-loaded state and fall through to the authenticated server metadata endpoint, while keeping populated roots as a client-side fast deny for clearly outside paths.
