## Summary

Keep open: current main and the latest release still hard-deny local assistant attachments while `localMediaPreviewRoots` is empty, and https://github.com/openclaw/openclaw/pull/78034 is an open closing PR for the same Control UI bootstrap-window bug. The report is well supported by source inspection and by later user comments confirming the same symptom, but the issue should close only after the linked fix lands or is replaced.

## What This Changes

The issue requests that Control UI local assistant media fall through to the server `assistant-media?meta=1` availability check while bootstrap preview roots have not loaded yet.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/78034, or an equivalent focused fix, so empty bootstrap roots fall through to the authenticated server metadata check while populated-root blocking remains intact.

## Reproduction Assessment

Yes, at source level: render a local `MEDIA:` attachment while `localMediaPreviewRoots` is `[]`, and current main returns `Outside allowed folders` before the server metadata check can run. I did not execute a live Control UI scenario in this read-only review.

## Solution Assessment

Yes, the narrowest maintainable fix is to treat empty roots as a bootstrap-not-loaded state and fall through to the existing authenticated server metadata check. The open PR follows that shape, but the issue should remain open until that candidate or an equivalent replacement lands.
