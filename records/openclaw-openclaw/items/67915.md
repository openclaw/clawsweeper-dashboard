## Summary

Keep open. Current main still contains the reported Control UI empty-roots hard-deny path, and the open linked implementation pull request at https://github.com/openclaw/openclaw/pull/78034 is the active fix path with closing syntax for this issue.

## What This Changes

The issue requests that local assistant media attachments fall through to the authorized server metadata check when Control UI bootstrap preview roots have not loaded yet, instead of rendering as outside allowed folders.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/78034 with a focused fallback that treats empty preview roots as bootstrap-not-ready, preserves populated-root client blocking, and adds regression coverage for local attachment metadata lookup.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction path exists: render a local `MEDIA:` attachment while `localMediaPreviewRoots` is `[]`, and current main returns `Outside allowed folders` before the metadata endpoint can run.

## Solution Assessment

Yes. The best fix direction is the narrow empty-roots fallback to the existing authorized `assistant-media?meta=1` check, while preserving populated-root client denial for genuinely outside paths.
