## Summary

Keep open. Current main still hard-denies local assistant attachments when `localMediaPreviewRoots` is empty, and the open linked pull request at https://github.com/openclaw/openclaw/pull/78034 is the active implementation path, so this issue should remain open until that fix or an equivalent replacement lands.

## What This Changes

Local Control UI assistant media attachments should not be shown as outside allowed folders before bootstrap local media roots have loaded.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/78034 with a focused empty-roots fallback to the existing authorized assistant-media metadata endpoint, preserving populated-root client blocking and adding regression coverage.

## Reproduction Assessment

Yes. A high-confidence source-level path exists: render a local `MEDIA:` attachment while `localMediaPreviewRoots` is `[]`, and current main returns `Outside allowed folders` before the server metadata request can run.

## Solution Assessment

Yes. The narrow maintainable fix is to skip the client-side preview-root fast-deny only while roots are empty and fall through to the existing authorized server metadata endpoint, while keeping populated-root blocking unchanged.
