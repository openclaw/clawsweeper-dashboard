## What This Changes

The issue reports that Control UI local assistant media is shown as outside allowed folders during the bootstrap window even though the server media endpoint authorizes and serves it.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/78034 or an equivalent focused fix so empty bootstrap roots fall through to the authenticated server metadata check while populated-root blocking remains intact.

## Reproduction Assessment

Yes, at source level: render a local `MEDIA:` attachment while `localMediaPreviewRoots` is `[]`, and current main returns `Outside allowed folders` before the `assistant-media?meta=1` metadata check can run. I did not run a live Control UI/OpenShift scenario in this read-only review.

## Solution Assessment

Yes: the best fix is the narrow empty-roots fallback to the existing authenticated server metadata endpoint, not a new config option or a broader allowlist change. The issue should stay open while the linked open PR is reviewed and either landed or replaced.
