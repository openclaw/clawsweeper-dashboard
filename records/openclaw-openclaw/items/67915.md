## Summary

Keep open: current main and the latest release still have the empty-bootstrap hard-deny path, and https://github.com/openclaw/openclaw/pull/78034 is the open closing PR for the same Control UI attachment bug.

## What This Changes

The issue reports that Control UI local assistant MEDIA attachments render as unavailable during bootstrap even though the server-side assistant-media metadata endpoint says they are available.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/78034 or an equivalent focused patch that skips the client hard-deny only while roots are empty, preserves populated-root blocking, and keeps regression coverage.

## Reproduction Assessment

Yes, at source level: render a local MEDIA attachment while localMediaPreviewRoots is [], and current main returns Outside allowed folders before the assistant-media metadata check can run. I did not run a live OpenShift/Control UI scenario in this read-only review.

## Solution Assessment

Yes: falling through to the existing authenticated server metadata endpoint only during the empty-bootstrap window is the narrowest maintainable fix. The issue should remain open until the linked PR or an equivalent patch lands.
