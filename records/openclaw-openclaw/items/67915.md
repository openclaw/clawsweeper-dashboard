## Summary

Keep open: current main and v2026.6.9 still hard-deny local Control UI assistant attachments while `localMediaPreviewRoots` is empty, and https://github.com/openclaw/openclaw/pull/78034 is an open closing PR for the same bootstrap-window bug. Later user comments report the same symptom, so this issue should close only after that PR or an equivalent replacement lands.

## What This Changes

The issue reports that Control UI local assistant media should not render as `Unavailable — Outside allowed folders` when the server bootstrap roots and `assistant-media?meta=1` endpoint already allow the file.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/78034, or an equivalent focused replacement, so empty bootstrap roots fall through to the authenticated server metadata check while populated-root blocking remains intact.

## Reproduction Assessment

Yes. Source inspection shows a local `MEDIA:` attachment rendered while `localMediaPreviewRoots` is `[]` returns `Outside allowed folders` before the `assistant-media?meta=1` server check can run.

## Solution Assessment

Yes for the linked PR shape. Treating empty roots as bootstrap-not-loaded and falling through to the existing authenticated server metadata check is narrower than changing the root predicate or adding a new config path.
