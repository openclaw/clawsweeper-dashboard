## Summary

Keep open. Current main still has the empty-bootstrap `localMediaPreviewRoots` hard-deny path, and https://github.com/openclaw/openclaw/pull/78034 is an open closing PR for the same bug, so this issue should stay open until that PR or an equivalent replacement lands.

## What This Changes

Local Control UI assistant attachments should not be shown as unavailable during the bootstrap window when server metadata already confirms the media is allowed.

## Best Possible Solution

Review, fix, and land https://github.com/openclaw/openclaw/pull/78034 or an equivalent focused replacement that falls through to server metadata only while roots are empty and preserves populated-root blocking.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: render a local `MEDIA:` attachment while `localMediaPreviewRoots` is `[]`, and current main returns `Outside allowed folders` before the server metadata check can run.

## Solution Assessment

Yes. The open PR direction is the narrow maintainable fix: skip only the empty-bootstrap client fast-deny and let the existing authenticated `assistant-media?meta=1` endpoint decide, while keeping populated-root blocking unchanged.
