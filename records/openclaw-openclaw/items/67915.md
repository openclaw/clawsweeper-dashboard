## Summary

Keep open. Current main still hard-denies local assistant attachments when `localMediaPreviewRoots` is empty before reaching the server metadata check, and the paired PR #67916 is closed unmerged, so a replacement fix is needed.

## What This Changes

The issue asks Control UI to treat empty local media preview roots as bootstrap-not-loaded rather than a permanent outside-folder denial for local assistant media.

## Best Possible Solution

Land a narrow UI regression fix that lets empty preview roots fall through to the authenticated server metadata check, preserves populated-root client blocking, and adds regression coverage plus a changelog entry.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to render a local assistant `MEDIA:` attachment with `localMediaPreviewRoots: []`; current main returns `Outside allowed folders` before any metadata fetch can run.

## Solution Assessment

Yes. The narrow maintainable fix is to treat empty roots as a bootstrap-not-loaded state and defer to the existing authenticated server metadata endpoint, while keeping the populated-roots fast denial for paths outside configured roots.
