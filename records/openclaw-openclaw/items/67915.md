## Summary

Keep open. Current main still hard-denies local assistant attachments when `localMediaPreviewRoots` is empty, and an open linked pull request is the active implementation path rather than a reason to close early.

## What This Changes

The issue requests that Control UI local assistant media fall through to the server `assistant-media?meta=1` check during the bootstrap window instead of rendering as unavailable.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/78034 with a focused empty-roots fallback to the existing authorized server metadata endpoint, preserving populated-root blocking and adding regression coverage.

## Reproduction Assessment

Yes. A high-confidence source-level path exists: render a local `MEDIA:` attachment while `localMediaPreviewRoots` is `[]`, and current main returns `Outside allowed folders` before issuing the server metadata request.

## Solution Assessment

Yes. The narrow maintainable fix is to skip the client-side preview-root denial only while roots are empty and fall through to the existing authorized `assistant-media?meta=1` endpoint; the linked PR is the active path for that fix.
