## Summary

Keep open. Current main still has the empty-`localMediaPreviewRoots` hard-deny path, and https://github.com/openclaw/openclaw/pull/78034 is an open closing PR for the same bug, so this issue should remain open until that PR or an equivalent replacement lands.

## What This Changes

The issue reports that Control UI local assistant attachments can render as `Unavailable — Outside allowed folders` before the bootstrap config has populated local media preview roots.

## Best Possible Solution

Review, fix, and land https://github.com/openclaw/openclaw/pull/78034 or an equivalent focused replacement that falls through to server metadata only while roots are empty and preserves populated-root blocking.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: render a local `MEDIA:` attachment while `localMediaPreviewRoots` is `[]`, and current main returns `Outside allowed folders` before it can fetch server metadata.

## Solution Assessment

Yes. The linked PR direction is the narrow maintainable fix: skip only the empty-bootstrap client fast-deny and let the existing authenticated `assistant-media?meta=1` endpoint decide, while keeping populated-root blocking unchanged.
