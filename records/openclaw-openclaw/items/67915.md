## Summary

Keep open. Current main still hard-denies local assistant attachments when `localMediaPreviewRoots` is empty, and https://github.com/openclaw/openclaw/pull/78034 is an open closing PR for the same fix path, so this issue should remain open until that PR or an equivalent replacement lands.

## What This Changes

The issue reports Control UI local assistant media attachments being shown as `Unavailable — Outside allowed folders` when attachment rendering runs before bootstrap preview roots are populated.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/78034 with a focused fix that falls through to the authenticated server metadata endpoint only while preview roots are empty, preserves populated-root client blocking, and adds regression coverage plus accepted runtime proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: render a local `MEDIA:` attachment while `localMediaPreviewRoots` is `[]`, and current main returns `Outside allowed folders` before building or fetching the server metadata URL.

## Solution Assessment

Yes. The narrow maintainable solution is the linked PR's direction: skip only the empty-bootstrap-roots client fast-deny and let the existing authorized `assistant-media?meta=1` endpoint decide, while keeping populated-root blocking unchanged.
