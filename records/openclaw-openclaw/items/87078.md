## Summary

The requested AGT-specific `GOVERNANCE_PROXY` core integration should close because current OpenClaw already exposes generic plugin hooks for prompt and tool governance, and repository policy routes optional third-party governance integrations to plugins/ClawHub when no missing core API is shown.

## What This Changes

The issue requests native `GOVERNANCE_PROXY` support that pre-flights OpenClaw tool execution through an AGT governance sidecar.

## Best Possible Solution

Keep core plugin-agnostic and implement the AGT sidecar adapter as a ClawHub/native plugin; open a narrow plugin API issue only if AGT proves a missing generic hook field.

Not applicable: this is a feature request for a new AGT sidecar integration, not a reported current-main bug with a failing reproduction path.

No: adding an AGT-specific env var inside core execution code is not the best fit when existing plugin hooks can preflight prompts and tool calls without coupling OpenClaw to one governance sidecar.
