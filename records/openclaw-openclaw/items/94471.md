## Summary

## What This Changes

The branch exports `checkPortInUse` from `src/infra/ports-inspect.ts` and changes `isPortBusy` in `src/cli/ports.ts` to use that shared multi-address probe.

## Best Possible Solution

Land one canonical shared-helper fix that detects IPv4-only listeners, preserves indeterminate probe failures as an error path, adds focused force-path coverage, and includes redacted real listener proof.

## Reproduction Assessment

Yes at source level: current main uses a hostless `isPortBusy` probe before the force-free path, and the shared checker exposes `busy`, `free`, and `unknown` states. I did not run a live IPv4-only listener repro in this read-only review.

## Solution Assessment

No as submitted. Reusing the shared multi-family probe is the right direction, but the boolean adapter must distinguish `unknown` from `free` so the CLI preserves its fail-closed, user-visible error path.
