## Summary

This should stay open: current main still exposes Perplexity-specific content limits for the native Search API, but it does not expose or forward Perplexity Sonar's `search_context_size` chat-completions option. The request is a narrow, dependency-backed provider feature rather than a stale or duplicate cleanup target.

## What This Changes

The issue requests a `search_context_size` web_search argument that maps to Perplexity Sonar `web_search_options.search_context_size` values `low`, `medium`, or `high`.

## Best Possible Solution

Add an enum-validated `search_context_size` hint to the shared and Perplexity provider tool schemas, forward it as `web_search_options.search_context_size` only on the Perplexity chat-completions/Sonar path, include it in cache keys, and document/test unsupported transport behavior.

## Reproduction Assessment

Yes for the source-level behavior: current main's shared schema, Perplexity schema, runtime request body, and docs all omit `search_context_size`. This is a feature gap rather than a failing existing contract.

## Solution Assessment

Yes, the requested direction is aligned with the existing provider-specific `web_search` parameter pattern. The maintainable version should be chat-completions-specific, enum-validated, cache-keyed, and covered by focused tests plus docs.
