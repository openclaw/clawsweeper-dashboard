## What This Changes

Active Memory embedded recall should keep `memory_search` and `memory_get` callable after upgrading to 2026.5.2.

## Best Possible Solution

Keep the merged #76380 implementation on main and ship it in the next release so v2026.5.2 installations receive the Active Memory recall-tool fix.

## Reproduction Assessment

Yes, source-backed. The linked #76343 report and PR #76380 describe the same empty memory-tool allowlist failure for Active Memory embedded recall, and the pre-fix source derived memory tool context from a synthetic `:active-memory:` session key instead of the target agent.

## Solution Assessment

Yes. The merged fix is the narrow maintainable path: preserve the trusted requester agent in plugin tool context and Memory Core tool resolution instead of broadening Active Memory's tool surface or adding a config switch.
