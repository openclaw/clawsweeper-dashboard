## Best Possible Solution

## Reproduction Assessment

Yes, at source level: current main's helper returns true for a numeric `0` matching the defaulted compaction count of `0`, and the runtime uses that helper to skip memoryFlush. I did not run a live over-threshold session in this read-only review.

## Solution Assessment

Yes, with proof and cleanup caveats: `memoryFlushAt` is the narrow existing discriminator for real recorded cycle-0 flushes versus legacy or never-flushed 0/0 rows. A broader hash, sentinel, or migration rewrite would be more invasive for this specific bug.
