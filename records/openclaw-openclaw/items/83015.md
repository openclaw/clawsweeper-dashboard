This PR adds a `memoryFlushAt` discriminator to the memory-flush compaction dedupe guard, updates focused helper tests for ambiguous 0/0 rows, and adds a changelog entry.

## Best Possible Solution

## Solution Assessment

Yes, with proof and cleanup caveats: `memoryFlushAt` is the narrow existing discriminator for real recorded cycle-0 flushes versus legacy or never-flushed 0/0 rows. A sentinel or migration rewrite would be more invasive for this specific bug.
