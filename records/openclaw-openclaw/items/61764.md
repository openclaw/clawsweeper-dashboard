## Summary

Keep open. Current main still contains several of the named Channel dashboard performance and UX gaps, including inline full-snapshot JSON rendering and eager per-channel config-form rendering. The report is valid, but broad enough that maintainers should split and prioritize it rather than send it straight to an automated repair lane.

## What This Changes

The issue requests a performance and UX overhaul for the Channel dashboard, including hiding debug-heavy surfaces, reducing repeated schema/JSON/timestamp work, and redesigning status/config flows for non-developer users.

## Best Possible Solution

Split the work into staged maintainer-owned pieces: quick render-path fixes first, lazy/memoized channel config rendering next, and a product-reviewed Channel dashboard UX redesign with i18n/docs/test coverage after the desired user flow is settled.
