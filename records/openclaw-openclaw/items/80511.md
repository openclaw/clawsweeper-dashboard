## Summary

Keep open. The PR targets the right bug, but the patch changes the dry-run send contract in a way that drops existing structured JSON/preflight payload details, and it does not address the poll dry-run fallthrough called out by the linked report.

## What This Changes

The PR changes outbound send dry-runs to return `handledBy: "dry-run"` from `executeSendAction` and updates the send result type and one outbound unit test.

## Best Possible Solution

Fix the formatter/result contract so all send and poll dry-runs render the dry-run text while preserving the existing structured dry-run payload from the core send/poll planning path.

## Reproduction Assessment

Yes. Source inspection shows current main still sends dry-run send/poll results to the formatter with `dryRun: true` while the success branches depend on `handledBy`, and the linked issue includes live CLI output from v2026.5.7.

## Solution Assessment

No. The narrow safer fix is to key the formatter or returned result contract from `dryRun: true` for send and poll without dropping the existing dry-run `sendMessage`/`sendPoll` result payload.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve the core dry-run payload:** `src/infra/outbound/outbound-send-service.ts:253-255`
  - body: This early return skips the existing `sendMessage(... dryRun: true)` path, so `openclaw message send --dry-run --json` no longer gets the structured payload that currently includes `channel`, `to`, `via`, media fields, and `dryRun`. The text bug can be fixed by marking the result as dry-run after the core dry-run send result is produced, instead of replacing the payload with `{ ok: true, dryRun: true }`.
  - confidence: 0.88
- **[P2] Cover poll dry-runs before closing the linked bug:** `src/infra/outbound/message-action-runner.ts:128`
  - body: The formatter dry-run guard is shared, but this type change only allows `send` results to use `handledBy: "dry-run"`. `executePollAction` still returns `handledBy: "core"` with a `pollResult`, so text mode can still render the poll success branch for `--dry-run`; extend the same contract to poll or make the formatter check `result.dryRun` for both send and poll.
  - confidence: 0.83
