## Summary

Keep open. Current main still eagerly materializes image, video, and music provider lists before the complete explicit modelConfig short-circuit, and PR #74097 is the open paired implementation candidate; #73793 mitigates the provider-registry reload side but does not remove the unused lookup reported here.

## What This Changes

The issue reports that image, video, and music generation tool model-config resolution eagerly lists capability providers even when complete explicit model configuration would make provider discovery unnecessary.

## Best Possible Solution

Review and land #74097 or an equivalent lazy-provider thunk change in the shared resolver and three media call sites, while preserving fallback discovery when modelConfig is incomplete.

## Reproduction Assessment

Yes. Current main has a static reproduction path: the image, video, and music helpers evaluate listRuntime*GenerationProviders before resolveCapabilityModelConfigForTool can return a complete explicit modelConfig.

## Solution Assessment

Yes. Deferring provider discovery behind a thunk at the shared resolver boundary is the narrowest maintainable solution because it avoids only the unused explicit-config lookup and keeps candidate fallback behavior intact.
