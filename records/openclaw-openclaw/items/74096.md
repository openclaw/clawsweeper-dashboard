## Summary

Keep open. Current main has mitigated part of the registration/availability path, but the shared media model-config resolver and the three image/video/music callers still eagerly materialize providers; #74097 remains the open closing implementation candidate.

## What This Changes

The issue reports eager image, video, and music generation provider listing during media model-config resolution, adding avoidable plugin reload latency when explicit media model configs should short-circuit.

## Best Possible Solution

Review and land #74097, or an equivalent lazy-provider thunk change in the shared resolver and three media call sites, while preserving fallback discovery when modelConfig is incomplete.

## Reproduction Assessment

Yes for the remaining code-level bug: in current main, any image/video/music model-config helper with a complete explicit modelConfig evaluates the corresponding `listRuntime*GenerationProviders` call before the shared resolver can return the explicit config. The original all-turn production timing is only partially reproducible from source now because the registration availability path has changed.

## Solution Assessment

Yes. Deferring providers behind a thunk in the shared resolver is the narrowest maintainable fix because it avoids unused provider discovery only on the explicit-config path and leaves incomplete-config fallback resolution intact.
