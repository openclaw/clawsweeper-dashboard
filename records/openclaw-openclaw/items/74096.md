## Summary

Keep open. Current main appears to mitigate the reported registration-time provider discovery path for complete explicit media model config, but the paired PR #74097 is still open with closing syntax and the shared execution-time resolver still accepts eagerly computed provider arrays.

## What This Changes

The issue reports that image, video, and music generation tools eagerly list runtime capability providers despite complete explicit model config, adding repeated plugin-registry reload latency on hosted gateways.

## Best Possible Solution

The end state should keep media generation tool registration and execution from doing provider discovery when complete explicit tool model config is already available, while preserving provider lookup for incomplete config and explicit list/status actions.

## Reproduction Assessment

Yes. The registration path can be checked with a providers thunk spy and complete explicit model config, and current tests cover that short-circuit; the remaining eager execution helper is directly visible in source.

## Solution Assessment

Unclear. Current main is a narrow fix for the per-turn registration reload described in the issue, but #74097 remains a plausible narrow cleanup if maintainers want the execution-time resolver to avoid the same unnecessary lookup.
