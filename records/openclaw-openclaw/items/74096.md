## Summary

Keep open. Current main still eagerly lists image/video/music runtime providers before the shared resolver can return the complete explicit modelConfig, and PR #74097 is the open closing implementation candidate rather than a reason to close the issue before merge.

## What This Changes

The issue requests deferring image/video/music capability-provider listing when explicit generation model config is already complete.

## Best Possible Solution

Land PR #74097 or an equivalent lazy-provider thunk change in the shared resolver and all three media model-config call sites, with regression tests that prove provider discovery is skipped for complete explicit config and preserved for fallback resolution.

## Reproduction Assessment

Yes. A code-level reproduction is to call any image/video/music model-config resolver with a complete explicit generation modelConfig; current main still evaluates the corresponding listRuntime*GenerationProviders call before resolveCapabilityModelConfigForTool can short-circuit.

## Solution Assessment

Yes. Deferring providers behind a thunk is the narrowest maintainable fix because it avoids unused discovery only on the explicit-config path while preserving candidate resolution when config is incomplete.
