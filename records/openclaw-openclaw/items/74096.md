## Summary

Keep open. Current main still eagerly materializes image, video, and music provider lists before the explicit modelConfig short-circuit can run, and open PR #74097 is the paired implementation candidate rather than a reason to close this issue early.

## What This Changes

The issue requests deferring image/video/music provider listing during tool model-config resolution so complete explicit modelConfig can avoid unnecessary plugin registry reloads.

## Best Possible Solution

Review and land #74097 or an equivalent change that makes shared media model-config resolution accept lazy provider discovery while preserving fallback lookup when modelConfig is incomplete.

## Reproduction Assessment

Yes. The current source gives a high-confidence static reproduction path: each media model-config helper calls listRuntime*GenerationProviders before resolveCapabilityModelConfigForTool can return a complete explicit modelConfig.

## Solution Assessment

Yes. A provider-list thunk in the shared resolver is the narrowest maintainable fix because it avoids only the wasted explicit-config lookup while keeping existing candidate resolution behavior for incomplete config.
