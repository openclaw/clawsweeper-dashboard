## Summary

Keep open. Current main appears to have addressed the original unused-tool registration hot path through manifest/auth metadata gating, but open PR #74097 still explicitly closes this issue and the narrower shared resolver eager-listing behavior remains present when generation tools execute.

## What This Changes

The issue asks to stop image, video, and music generation tools from eagerly listing runtime capability providers during turns where an explicit model config makes that list unused.

## Best Possible Solution

Keep the manifest/auth registration path and either land an equivalent lazy-provider resolver change or explicitly close #74097 as superseded after maintainer review.

## Reproduction Assessment

Partially. The original unused-tool registration reload path does not reproduce by static source review on current main, but the remaining helper-level eager lookup is still visible because the three execute-time model-config helpers materialize providers before the shared resolver can short-circuit.

## Solution Assessment

Unclear. A provider-list thunk remains the narrowest fix for the helper-level eager lookup, but current main already addresses the reported per-turn registration latency, so maintainers should decide whether #74097 is still needed or superseded.
