## Summary

Keep open. Current main partially mitigates generation tool registration availability, but the shared model-config resolver and the image/video/music model-config call sites still eagerly materialize provider lists before complete explicit modelConfig can short-circuit; open PR #74097 is the paired implementation candidate.

## What This Changes

The issue requests lazy provider discovery for image/video/music generation model-config resolution to avoid repeated plugin registry reloads when explicit modelConfig is already complete.

## Best Possible Solution

Review and land #74097 or an equivalent change that makes shared media model-config resolution accept lazy provider discovery while preserving provider lookup when modelConfig is incomplete.

## Reproduction Assessment

Yes. The current source still has a static reproduction path: the image, video, and music model-config helpers call listRuntime*GenerationProviders before resolveCapabilityModelConfigForTool can return a complete explicit modelConfig.

## Solution Assessment

Yes. A provider-list thunk is the narrowest maintainable fix because it avoids only the unnecessary explicit-config lookup and keeps existing fallback behavior for incomplete config.
