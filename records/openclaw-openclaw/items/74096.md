## Summary

Keep open. Current main still passes eagerly materialized image, video, and music provider arrays into the shared model-config resolver before its explicit modelConfig short-circuit, and the paired fix PR is still open and unmerged.

## What This Changes

The issue requests deferring image/video/music capability provider listing until media tool model-config resolution actually needs fallback provider discovery.

## Best Possible Solution

Land #74097 or an equivalent lazy-provider thunk change in the shared resolver and the three media tool call sites while preserving fallback discovery when modelConfig is incomplete.

## Reproduction Assessment

Yes. A static reproduction path remains: with complete cfg.agents.defaults media modelConfig, each resolve*GenerationModelConfigForTool call evaluates listRuntime*GenerationProviders before resolveCapabilityModelConfigForTool can return at hasToolModelConfig.

## Solution Assessment

Yes. Deferring provider discovery behind a thunk at the shared resolver is the narrowest maintainable fix because it removes only the unused explicit-config lookup and keeps fallback candidate resolution intact.
