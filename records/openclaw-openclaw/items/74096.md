## Summary

Keep open. Current main still eagerly materializes image/video/music provider lists before the explicit modelConfig short-circuit, and open PR #74097 is the paired implementation candidate.

## What This Changes

Avoid eager image/video/music capability provider discovery when a complete explicit generation model config is already set.

## Best Possible Solution

Land #74097 or an equivalent lazy-provider thunk change in the shared resolver and the three media call sites, preserving fallback discovery when modelConfig is incomplete.

## Reproduction Assessment

Yes. Current main has a static reproduction path: the image, video, and music helpers evaluate listRuntime*GenerationProviders before resolveCapabilityModelConfigForTool can return a complete explicit modelConfig.

## Solution Assessment

Yes. Deferring provider discovery behind a thunk at the shared resolver is the narrowest maintainable fix because it removes only the unused explicit-config lookup and keeps candidate resolution behavior intact.
