## Summary

Keep open. Current main appears to fix the reported registration-time provider discovery path for complete explicit media model configs, but #74097 is still an open paired PR with closing syntax and the shared execution-time resolver still exposes the eager provider-array surface that PR targets.

## What This Changes

The issue reports that image_generate, video_generate, and music_generate eagerly enumerate capability providers during tool registration, adding plugin registry reload latency even when the agent already has complete explicit media model config.

## Best Possible Solution

Reconcile #74097 against current main: close or rebase it as superseded if the registration fast path and provider-runtime cache fixes are sufficient, or land an equivalent narrow thunk/test cleanup if maintainers want the residual execution-time eager lookup removed too.

Yes.
