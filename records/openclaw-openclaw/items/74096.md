## Summary

Keep open. Current main still materializes image, video, and music provider lists before the shared resolver can return a complete explicit modelConfig, and #74097 remains the paired closing PR rather than a merged fix.

## What This Changes

The issue requests deferring image/video/music capability-provider listing so complete explicit generation modelConfig values can short-circuit without plugin provider discovery.

## Best Possible Solution

Review and land #74097 or an equivalent lazy-provider resolver change that defers provider discovery only until candidate fallback actually needs it.

## Reproduction Assessment

Yes. Current main provides a static reproduction path: each media model-config helper evaluates listRuntime*GenerationProviders before resolveCapabilityModelConfigForTool can return the complete explicit modelConfig.

## Solution Assessment

Yes. A provider-list thunk is the narrowest maintainable fix because it removes the unnecessary explicit-config lookup while keeping existing fallback behavior for incomplete configs.
