## Summary

Keep open. Current main has mitigated part of the registration/availability path, but the reported shared resolver and the three media model-config call sites still eagerly materialize runtime providers; PR #74097 is the open closing implementation candidate.

## What This Changes

The issue requests lazy provider discovery for image/video/music generation model-config resolution to avoid unnecessary plugin-registry reloads when explicit modelConfig already resolves.

## Best Possible Solution

Review and land PR #74097, or an equivalent lazy-provider thunk change in the shared resolver and three media call sites, while preserving fallback discovery when modelConfig is incomplete.

## Reproduction Assessment

Yes for the remaining code-level bug: call any of the image/video/music model-config resolver helpers with a complete explicit modelConfig and the corresponding listRuntime*GenerationProviders call is evaluated before the shared resolver can short-circuit. The original per-turn registration timing needs fresh hosted measurement because current main now uses a separate availability path.

## Solution Assessment

Yes. Deferring providers behind a thunk in the shared resolver is the narrowest maintainable fix for the remaining eager lookup because it avoids unused discovery only on the explicit-config path and leaves incomplete-config fallback resolution intact.
