- **[P2] Gate stickers after room-event classification:** `extensions/telegram/src/bot-message-context.ts:596-600`
  - body: `shouldAckSticker()` runs before `buildTelegramInboundContextPayload()` determines whether this inbound group message is a `room_event`, and it does not include the `canShowStatusReaction` guard used by the existing ack reaction path. With `ackSticker.scope: "all"` or `"group-all"`, an unmentioned room-event group message can still get a visible sticker even though OpenClaw suppresses other feedback for that path and may not send a visible reply. Build the inbound payload first or apply the same room-event gate before sending the sticker.
  - confidence: 0.9

Either keep the resolver local to the Telegram plugin or promote it deliberately with docs and contract coverage so third-party plugins are not left with an undocumented surface.
