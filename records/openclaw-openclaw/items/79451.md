Keep open: current main still builds the CLI backend MCP config without an agent-scoped effective tool policy, and the open fix PR that explicitly closes this issue has not merged yet.

## What This Changes

The issue requests enforcing per-agent `tools.deny` for MCP tools exposed through the `claude-cli` backend's generated MCP config.

## Best Possible Solution

Review and land `https://github.com/openclaw/openclaw/pull/79480`, or a narrow replacement, so effective tool policy is applied before CLI backend MCP config is written without widening exact tool allowlists.

## Reproduction Assessment

Yes, source-reproducible: the reporter gave a concrete `claude-cli` MCP setup, and current main still merges configured MCP servers into the generated config without agent-scoped policy. A live external CRM MCP reproduction was not run in this read-only review.

## Solution Assessment

Yes: enforcing effective policy before writing the generated CLI MCP config is the narrowest maintainable boundary because `claude-cli` consumes that config directly. The linked open PR is the right path once its matcher/security concerns are resolved.
