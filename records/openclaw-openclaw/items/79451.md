## Summary

Keep open: current main still builds the `claude-cli` MCP config without applying the effective per-agent tool policy, and the linked fix PR at https://github.com/openclaw/openclaw/pull/79480 is still open with closing syntax for this issue.

## What This Changes

The issue asks OpenClaw to enforce per-agent `tools.deny` when generating MCP config for `claude-cli` so denied MCP tools are not exposed to that backend.

## Best Possible Solution

Review and land the linked implementation PR after resolving any remaining policy-matching concerns, or otherwise enforce effective policy before writing CLI backend MCP config with regression coverage.

## Reproduction Assessment

Yes, source-reproducible on current main: docs promise deny-wins and per-agent restrictions, while the CLI MCP config path still merges configured servers without effective policy inputs. I did not live-run the external CRM MCP setup.

## Solution Assessment

Yes: applying effective policy at the generated CLI MCP config boundary is the narrowest maintainable fix because `claude-cli` reads the MCP tool surface directly from that config. Closing the issue before the linked PR merges would be premature.
