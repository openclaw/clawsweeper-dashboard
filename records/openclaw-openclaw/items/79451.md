## Summary

Keep open: current main still does not apply the documented per-agent deny-wins tool policy before generating the CLI backend MCP config, and the directly linked fix PR is still open and unmerged.

## What This Changes

Enforce `agents.list[].tools.deny` for MCP tools exposed through the `claude-cli` backend so denied MCP servers or tools are not callable by the agent.

## Best Possible Solution

Land a corrected version of https://github.com/openclaw/openclaw/pull/79480 that applies effective tool policy before writing CLI MCP config, preserves deny-wins semantics, and keeps server-level versus exact-tool allow behavior fail-closed.

## Reproduction Assessment

Yes. The reporter gives a concrete 2026.5.7 `claude-cli` reproduction, and current main still writes the merged CLI MCP config without passing agent-scoped tool policy into that path.

## Solution Assessment

Yes, enforcing the effective policy at the generated MCP config boundary is the narrow maintainable fix because `claude-cli` reads tools directly from that config. The existing linked PR is the right implementation path, but it still needs review/repair before merge.
