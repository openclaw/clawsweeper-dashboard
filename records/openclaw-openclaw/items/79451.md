The issue reports that `agents.list[].tools.deny` is not applied to MCP servers exposed through the `claude-cli` backend generated MCP config.

## Best Possible Solution

Land a reviewed fix through https://github.com/openclaw/openclaw/pull/79480 or a replacement PR that filters CLI backend MCP configs with the effective agent/provider tool policy before launch.

## Reproduction Assessment

Yes, source-reproducible: the reporter gives concrete 2026.5.7 `claude-cli` steps, and current source still writes merged CLI MCP config without an agent-scoped effective tool-policy filter. I did not live-run the external CRM MCP setup.

## Solution Assessment
