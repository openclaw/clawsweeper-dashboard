## Summary

Keep open: current main still has the source-reproducible provider visibility failure path, and #77074 is already the open closing PR for this exact bug.

## What This Changes

The issue reports that first-class assistant `web_search` can lose configured runtime web-search providers in late-bound agent contexts even when capability and direct runtime resolution succeed.

## Best Possible Solution

Review and land or close #77074; the durable fix should preserve explicit empty plugin scopes while falling through from empty compatible registries and keeping late-bound provider selection intact.

## Reproduction Assessment

Yes. Source inspection on current main shows the late-bound tool can lose captured runtime metadata and the runtime provider resolver can return an empty active-registry result before scoped provider loading.

## Solution Assessment

Yes. The linked PR direction is the narrowest maintainable path because it targets the two reported seams and adds focused regression coverage rather than changing provider policy.
