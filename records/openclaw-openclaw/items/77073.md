## Summary

Keep open: current main still contains the source-visible late-bound runtime/provider resolution gap, and the open closing PR #77074 is the right place to finish the fix before this issue closes.

## What This Changes

The issue reports first-class assistant `web_search` failing to use configured runtime web providers in late-bound agent contexts even though capability and direct runtime paths can resolve Brave.

## Best Possible Solution

Review and land #77074, including the focused regression coverage for late-bound provider selection and empty active-registry fall-through, then close this issue through the merge linkage.

## Reproduction Assessment

Yes. Source inspection on current main shows the late-bound tool can lose captured runtime metadata and the web-provider resolver can treat an empty active runtime registry as authoritative before scoped provider loading.

## Solution Assessment

Yes. The linked PR targets the two narrow runtime seams described by the report; keeping this issue open until that PR merges is the maintainable path.
