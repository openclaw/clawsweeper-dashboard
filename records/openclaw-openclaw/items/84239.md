## What This Changes

The PR replaces `process.exit(0)` with `process.exitCode = 0` in `scripts/print-cli-backend-live-metadata.ts` for the unsupported `codex-cli` metadata branch.

## Best Possible Solution

Keep the natural-exit direction, but restructure the script so the unsupported `codex-cli` branch writes exactly one JSON payload and the normal metadata path runs only in the other branch, with a focused parse regression check.

## Reproduction Assessment

Yes from source inspection: the PR changes the only terminating statement in the `codex-cli` branch to `process.exitCode`, and the current file has another `process.stdout.write` after that branch. I did not run the patched branch in the read-only checkout.

## Solution Assessment

No, not as patched. `process.exitCode` is the right drain-safe primitive, but the script must also preserve the original single-output control flow with an `else` branch or a `main()` return path.
