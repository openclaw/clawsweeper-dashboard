## What This Changes

The issue reports that an external Weixin channel configured only with `channels.openclaw-weixin.enabled=true` is not loaded into the gateway runtime, causing `channels.start` to reject after QR login.

## Best Possible Solution

Land one canonical gateway startup-planner fix that treats enabled-only external channel config as startup intent while preserving plugin trust, disablement, and workspace-origin boundaries.

## Reproduction Assessment

Yes, source-reproducible: current startup planning still excludes enabled-only channel config from configured-channel ids, and `channels.start` still rejects channels missing from the active registry. I did not run a live WSL2 Weixin setup in this read-only review.

## Solution Assessment

Yes for the direction: this should be fixed generically in gateway startup/plugin activation, not by a Weixin-specific special case. The exact PR to land still needs maintainer convergence because multiple open candidates overlap the same compatibility-sensitive path.
