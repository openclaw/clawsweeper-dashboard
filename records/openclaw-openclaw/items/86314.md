## What This Changes

The issue requests that an installed external `openclaw-weixin` channel configured with `channels.openclaw-weixin.enabled=true` be registered and startable by the gateway runtime after QR login.

## Best Possible Solution

Land one generic gateway startup-planner fix that treats enabled-only external channel config as startup intent while preserving external plugin trust, allowlist, denylist, and explicit disable boundaries.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: current startup planning still depends on a helper that ignores `{ enabled: true }` alone, and `channels.start` still rejects channels absent from the active registry. I did not run a live WSL2 Weixin setup in this read-only review.

## Solution Assessment

Yes, the best direction is a generic gateway startup-planner repair rather than a Weixin-specific special case or docs-only workaround. The fix should make enabled-only external channel config reach the runtime load path without weakening plugin trust boundaries.
