## Summary

Keep open. Current main still reuses persisted skill snapshot prompts unless normal version/filter refresh conditions fire, and `openclaw doctor` state integrity still does not inspect cached `skillsSnapshot.prompt` for inactive, missing, or temp-backed bundled runtime roots. The related draft PR at https://github.com/openclaw/openclaw/pull/71497 is adjacent snapshot-version work, but it does not add the requested doctor diagnostic or safe stale-cache remediation.

## What This Changes

Request `openclaw doctor` detection and safe remediation for stale cached session skill snapshot paths after runtime root changes.

## Best Possible Solution

Add a focused doctor state diagnostic that resolves the active bundled skills directory, scans persisted session `skillsSnapshot.prompt` for missing, inactive, or temp-backed bundled runtime roots, reports cached-session metadata drift separately from live runtime health, and in repair mode backs up the session store before deleting only affected `skillsSnapshot` fields.

## Reproduction Assessment

Yes, source-reproducible. A session store entry with `skillsSnapshot.prompt` pointing at `/tmp/openclaw` or an old install root will be reused unless normal version/filter refresh conditions fire, while current doctor state integrity never inspects that cached prompt.

## Solution Assessment

Yes. A narrow doctor state-integrity diagnostic with safe `skillsSnapshot` invalidation is the maintainable fix; adjacent snapshot refresh work does not give operators the requested drift warning or remediation path.
