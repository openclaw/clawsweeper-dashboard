Keep open. Current main still lacks the requested `openclaw doctor` diagnostic or repair path for stale persisted `skillsSnapshot.prompt` paths, and the related open PR is adjacent snapshot-refresh work rather than this doctor/state-integrity check.

## What This Changes

Add an `openclaw doctor` check that detects and safely invalidates stale cached session skill snapshots after runtime-root changes.

## Best Possible Solution

Add a focused doctor state-integrity diagnostic that distinguishes live runtime health from cached-session metadata drift and, in repair mode, backs up the store before deleting only affected `skillsSnapshot` fields.

## Solution Assessment

Yes. A doctor/session-store diagnostic and explicit repair flow is the narrowest maintainable solution; startup or load-time mutation would alter long-lived session state outside an operator-requested repair pass.
