## Summary

Keep open. Current main still lacks the requested `openclaw doctor` diagnostic or repair path for stale persisted `skillsSnapshot` prompt paths, and the issue discussion gives concrete persisted-state evidence plus a narrow safe remediation.

## What This Changes

The issue requests an `openclaw doctor` diagnostic and safe repair guidance for stale persisted `skillsSnapshot` metadata that references old runtime roots.

## Best Possible Solution

Add a focused doctor state diagnostic that distinguishes live runtime health from cached-session metadata drift and, in repair mode, backs up the store before invalidating only stale snapshot fields.

## Reproduction Assessment

Yes. Source inspection shows that a `sessions.json` entry with `skillsSnapshot.prompt` pointing at `/opt/homebrew`, `/tmp`, `/private/tmp`, or another inactive runtime root would be loaded while doctor never checks that field.

## Solution Assessment

Yes. A doctor/session-store diagnostic and explicit repair flow is the narrowest maintainable solution; startup or load-time mutation would change long-lived session state outside an operator-requested repair pass.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/47534 by adding a focused `openclaw doctor` diagnostic for stale cached session skill snapshots after runtime-path changes. Resolve the active bundled skills/runtime root, scan persisted session entries for `skillsSnapshot.prompt` and any snapshot path metadata that references missing, inactive, or temp-backed bundled runtime roots such as `/opt/homebrew/lib/node_modules/openclaw`, `/tmp`, or `/private/tmp`, and report the problem as stale cached session metadata while keeping live runtime health separate. In `--fix`/repair mode, back up the session store and delete only the affected `skillsSnapshot` fields so sessions rebuild snapshots on the next turn; do not wipe sessions, do not add startup/load-time migrations, and do not change unrelated snapshot refresh semantics or PR #71497's restart-version work. Add regression coverage for detection and repair, update doctor docs and CHANGELOG if user-facing output changes.
