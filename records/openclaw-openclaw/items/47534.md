## Summary

Keep open. Current main still has no doctor diagnostic for stale `skillsSnapshot.prompt` paths, while the runtime still reuses persisted skill snapshots unless version/filter refresh conditions fire; the adjacent open draft PR only targets restart-version invalidation and does not add the requested doctor warning or repair path.

## What This Changes

Add an `openclaw doctor` diagnostic and safe remediation for persisted session skill snapshots that reference inactive, missing, or temp-backed runtime roots.

## Best Possible Solution

Add a focused doctor state-integrity diagnostic that resolves the active bundled skills directory, scans persisted `skillsSnapshot.prompt` for inactive/missing/temp-backed bundled roots, reports cached-session metadata drift separately from live runtime health, and in repair mode backs up the store before deleting only affected `skillsSnapshot` fields.

## Reproduction Assessment

Yes. A source-level reproduction is a session store entry whose `skillsSnapshot.prompt` points at `/tmp/openclaw` or an old install root; current main will reuse it unless the normal version/filter refresh predicate fires, and doctor does not inspect that prompt.

## Solution Assessment

Yes. The narrow maintainable fix is a doctor state-integrity check plus safe snapshot invalidation; the adjacent restart-version PR is useful but does not give operators the requested stale-cache warning or remediation path.
