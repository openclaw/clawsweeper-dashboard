## Summary

Keep open. Current main still does not have an `openclaw doctor` diagnostic for persisted `skillsSnapshot.prompt` paths that point at inactive, missing, or temp-backed runtime roots; the adjacent refresh-state work does not cover this doctor/reporting request.

## What This Changes

Add `openclaw doctor` detection and safe remediation guidance for stale cached session skill snapshots after runtime-path changes.

## Best Possible Solution

Add a focused doctor state-integrity check that resolves the active bundled skills directory, scans persisted `skillsSnapshot.prompt` for stale bundled/temp roots, reports cached-session metadata drift separately from live runtime health, and in `--fix` removes only affected snapshot fields after backup.

## Reproduction Assessment

Yes, source-level. Seed a session entry with `skillsSnapshot.prompt` containing an old `/opt/homebrew` or `/tmp/openclaw` bundled `SKILL.md` location and current `openclaw doctor` has no inspected code path that reads or reports that stale prompt field.

## Solution Assessment

Yes, the narrow maintainable path is a doctor state-integrity diagnostic plus conservative `--fix` invalidation of only stale snapshot fields. Runtime restart invalidation remains adjacent but does not replace the requested operator-facing drift check.
