The issue reports OpenClaw write/edit tool calls returning success while the target files are not created on disk.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/67202 or an equivalent adapter-layer verifier that checks post-write existence, file type, and expected UTF-8 byte size before upstream can report write success.
