## Summary

The reported profile-less Codex quota regression is real and the supplied live proof is convincing, but the shared CLI path introduces a blocking auth-routing defect: it selects the synthetic Codex query from configured runtime alone and can therefore report Codex subscription quota for an API-key-backed OpenAI route.

The PR centralizes Codex synthetic-usage helpers, enables profile-less Codex usage in chat status, and augments shared CLI status with a second synthetic OpenAI quota query for configured Codex runtimes.

## Best Possible Solution

Gate the shared CLI synthetic query on the canonical effective `codex-app-server` auth route, rather than configured runtime alone, and add regression coverage proving both API-key-backed and OpenClaw-pinned routes perform only the normal usage query.

## Solution Assessment
