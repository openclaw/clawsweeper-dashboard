## Summary

Keep open. Current main has improved skill discovery, but existing agent sessions can still keep a stale `skillsSnapshot` when the process-local snapshot version is reset and no new watcher bump occurs, which matches the related restart-safety diagnosis in the discussion.

## What This Changes

The issue requests that valid workspace skills consistently appear in `<available_skills>` instead of disappearing from an agent session's context.

## Best Possible Solution

Make skill snapshot freshness depend on a deterministic current skill-inventory fingerprint across process restarts, while keeping the watcher version as a fast path and preserving existing prompt size limits.

## Reproduction Assessment

Yes, source-reproducible for existing sessions: a persisted `skillsSnapshot.version` of 0 and a fresh process `globalVersion` of 0 make the refresh predicate false, so the old prompt can be reused. I did not run a live CLI reproduction because this review was read-only.

## Solution Assessment

No, the current version-counter-only solution is not sufficient because it is not restart-safe and the plain agent path does not install a watcher. A deterministic inventory fingerprint or equivalent persisted freshness marker is the narrower maintainable fix; #74161 addresses path-use prompting but not snapshot freshness.
