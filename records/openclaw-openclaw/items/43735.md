## Summary

Keep open. Current main supports workspace skill discovery, but the stale persisted snapshot path still remains for `<available_skills>`: a restarted process can treat a cached version-0 prompt as fresh, and the newer hydration commits preserve that old prompt while only reloading `resolvedSkills`.

## What This Changes

The issue asks OpenClaw to reliably load and list valid active-workspace skills in the agent's initial `<available_skills>` context.

## Best Possible Solution

Land #71497 or an equivalent shared restart-safe snapshot invalidation fix that refreshes stale persisted prompts after process restart while preserving documented roots, filters, containment checks, and prompt-size limits.

## Reproduction Assessment

Yes. A high-confidence static reproduction is a persisted session with `skillsSnapshot.version: 0`, a skill inventory change while OpenClaw is stopped, and a restart where `getSkillsSnapshotVersion(workspaceDir)` is still 0 so both gateway and CLI checks reuse the old prompt.

## Solution Assessment

No. Current main is not the solved state because hydration reloads runtime `resolvedSkills` but intentionally preserves the persisted `<available_skills>` prompt; the maintainable fix belongs in shared restart-safe snapshot freshness logic.
