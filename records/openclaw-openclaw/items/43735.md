## Summary

Keep open: current main has partial mitigations for prompt-budget truncation and live watcher/config refresh, but the persisted session snapshot path is still restart-sensitive and can keep advertising an old skills list after missed filesystem changes.

## What This Changes

The issue requests that every valid workspace skill under `~/.openclaw/workspace/skills` appear in the agent's `<available_skills>` context instead of only a subset.

## Best Possible Solution

Keep the current compact prompt fallback, then add a durable inventory fingerprint or restart-safe invalidation check so existing sessions rebuild `skillsSnapshot` when workspace/managed skill inventory changes outside a live watcher event.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: an existing session with a persisted version-0 or undefined `skillsSnapshot` can survive a process restart, then reuse stale skills after the filesystem changed while no watcher was running. The key path is `getSkillsSnapshotVersion() === 0` plus `shouldRefreshSnapshotForVersion(..., 0) === false`.

## Solution Assessment

No, current main is not the complete fix because watcher/config bumps only cover live events and config writes. A durable inventory fingerprint or restart-safe snapshot version check is the narrowest maintainable fix; a manual reload command would be useful but should not be the only repair.
