## What This Changes

The issue asks OpenClaw to load all valid `SKILL.md` folders from `~/.openclaw/workspace/skills` into the agent `<available_skills>` context instead of showing a partial stale subset.

## Best Possible Solution

Keep the durable skills snapshot refresh fix concentrated in the canonical stale-invalidation work so all skill roots and both session paths refresh from stable inventory/config inputs.

## Reproduction Assessment

Yes. ## Solution Assessment

Yes. Consolidating this into the canonical restart-stable snapshot invalidation issue and maintainer draft PR is the narrowest maintainable path; a workspace-skills-only patch would leave the same cache bug in other skill roots and session paths.
