## Summary

Keep open. Current main has improved skills discovery and watcher-based refresh, but the restart-safe stale snapshot path reported in the discussion is still source-reproducible because persisted version-0 snapshots can remain fresh-looking after a process restart.

## What This Changes

The issue reports that valid workspace skills under ~/.openclaw/workspace/skills/ sometimes do not appear in the agent <available_skills> context.

## Best Possible Solution

Add a deterministic skills inventory fingerprint to session snapshots and refresh when it changes, while keeping watcher version bumps as a fast path.

## Reproduction Assessment

Yes, source-reproducible. Current main can preserve a stale version-0 skillsSnapshot after restart because the live inventory version also starts at 0 and no inventory fingerprint is compared.

## Solution Assessment

No. The current watcher/version refresh is useful but not sufficient for restart-safe correctness; a persisted inventory fingerprint comparison is the narrow maintainable fix.
