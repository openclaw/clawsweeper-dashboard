## Summary

Keep open. Current main discovers workspace skills and intentionally caps prompt injection, but the stale persisted `skillsSnapshot` restart path still exists: version state starts at `0`, so an old persisted version `0` snapshot can be treated as current after restart in both gateway and CLI paths.

## What This Changes

Ensure valid skills under the agent workspace skills directory are reflected in `<available_skills>` instead of disappearing or showing stale subsets.

## Best Possible Solution

Land #71497 or an equivalent shared refresh-state fix that invalidates persisted `version: 0` skill snapshots after restart while preserving source precedence, skill filters, safety checks, and prompt-size limits.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to persist a session with `skillsSnapshot.version: 0`, restart OpenClaw so process-local skill versions are again `0`, change the skill inventory, and observe the gateway or CLI path reuse the old snapshot because `shouldRefreshSnapshotForVersion(0, 0)` is false.

## Solution Assessment

No, current main is not the solved state. The narrow maintainable fix is restart-safe invalidation in the shared skills refresh-state layer, not a separate reload command or unbounded inclusion of every skill in the prompt.
