## Summary

Keep open: current main and the latest release still flag `models.providers.*.request.headers.*` plaintext strings, and the open draft PR at https://github.com/openclaw/openclaw/pull/95303 is the candidate fix using closing syntax. This should close only after maintainers accept the request-header audit policy and that PR lands, or after they explicitly reject the behavior change.

## What This Changes

The issue requests that `openclaw doctor --fix` and `openclaw secrets audit` stop treating static non-credential provider request headers such as `Wafer-ZDR: required` as plaintext secrets.

## Best Possible Solution

Capture a maintainer/secops-approved request-header audit rule, then review and either land or reject the linked draft PR while keeping doctor and secrets audit aligned.

## Reproduction Assessment

Yes. Current main registers `models.providers.*.request.headers.*` as an audited target and existing doctor/secrets-audit tests expect static request headers to produce plaintext findings; I did not run the CLI because this review is read-only.

## Solution Assessment

Unclear until maintainers accept the security policy. Reusing the shared header-name predicate in both collectors, as the linked PR proposes, looks like the narrow mechanical fix, but the best end state depends on whether request headers should be name-heuristic rather than always audited.
