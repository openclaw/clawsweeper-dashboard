## Summary

Keep open: current main and the latest release still flag `models.providers.*.request.headers.*` as an audited plaintext-secret surface, and the open draft fixing PR has not merged yet.

## What This Changes

The issue requests that static non-credential model-provider request headers, such as `Wafer-ZDR: required`, stop triggering plaintext secret warnings in `openclaw doctor` and `openclaw secrets audit`.

## Best Possible Solution

Review and land or revise the focused linked PR so doctor and secrets audit share one maintainer-approved request-header policy while preserving warnings for credential-bearing headers.

## Reproduction Assessment

Yes. Source inspection on current main and `v2026.6.9` shows request headers remain audited without the non-sensitive header-name skip, and adjacent tests still expect the reported warning path.

## Solution Assessment

Mostly yes, pending maintainer security-policy approval. Reusing the existing shared sensitive-header predicate in both collectors is narrower than a Wafer-specific carveout, but maintainers must accept the false-negative tradeoff for neutral-named credential headers.
