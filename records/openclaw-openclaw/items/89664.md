## Summary

Keep open: current main and the latest release still treat `models.providers.*.request.headers.*` as an audited plaintext SecretRef surface, and tests explicitly expect static request headers to warn. The report is valid from source, but changing it needs a maintainer security policy decision rather than conservative cleanup closure.

## What This Changes

The issue requests that `openclaw doctor --fix` and `openclaw secrets audit` stop warning for static non-credential model-provider request headers such as `Wafer-ZDR: required`.

## Best Possible Solution

Maintainers should define the intended request-header credential policy, then either apply the existing sensitive-name heuristic to request headers across audit and doctor or document the conservative all-request-headers warning contract.

## Reproduction Assessment

Yes from source inspection: current main registers `models.providers.*.request.headers.*` as audited and only applies the non-sensitive header-name skip to top-level provider headers. I did not run the CLI because this review is read-only, but adjacent tests assert the reported warning path.

## Solution Assessment

Unclear until maintainers choose the security policy. Applying the existing header-name predicate to request headers is plausible, but the best fix must also update the current tests and docs or explicitly preserve the conservative warning behavior.
