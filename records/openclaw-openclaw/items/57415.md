## Summary

Keep open: current main still does not implement `openclaw agent --cwd`, and the branch carries useful focused work, but it is not ready because the trusted gateway `workspaceDir` path still accepts raw relative/home-relative values and the external PR has no after-fix real behavior proof.

## What This Changes

This PR adds `openclaw agent --cwd`, forwards a normalized `workspaceDir` through local and gateway agent runs, extends the gateway/Swift protocol models, and adds regression tests.

## Best Possible Solution

Land a rebased revision that keeps CLI-side normalization, normalizes or rejects owner-authorized gateway `workspaceDir` overrides at ingress, preserves spawned-session inheritance, updates the public CLI docs, and includes redacted real behavior proof.

## Reproduction Assessment

Yes, source-level reproduction is high confidence. Current main lacks the CLI option, forwarding, and schema field, while PR head visibly forwards owner-authorized gateway `workspaceDir` values after trimming only.

## Solution Assessment

No, not yet. Reusing existing workspace plumbing and normalizing CLI input is the right shape, but trusted gateway ingress should normalize or reject raw overrides before dispatch and the public docs need the new option.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Normalize trusted workspace overrides before dispatch:** `src/gateway/server-methods/agent.ts:104`
  - body: This returns an owner-authorized `workspaceDir` after trimming only. Owner/admin gateway clients can send values like `.` or `~/repo`, which are then resolved later by the gateway process instead of at request ingress, so the run can target the daemon cwd/home rather than the caller's intended workspace. Normalize with the same path resolver here, or require absolute paths before assigning the override.
  - confidence: 0.9

  - body: This adds a user-facing `openclaw agent --cwd` option, but the CLI reference linked from command help still has no option entry or example for it. Update `docs/cli/agent.md` so users can discover the new behavior outside generated help.
  - confidence: 0.82
