## Summary

Keep open. Current main still lacks `openclaw agent --cwd`, and this PR is the active implementation, but PR head still has a merge-blocking workspace path normalization issue plus docs/changelog drift.

## What This Changes

The PR adds `openclaw agent --cwd <dir>`, forwards it as `workspaceDir` through gateway and local agent runs, extends the agent RPC/Swift protocol models, and adds CLI/gateway regression tests.

## Best Possible Solution

Land a rebased revision that keeps the CLI-side normalization, normalizes or rejects owner-authorized gateway `workspaceDir` overrides at request ingress, and updates the public CLI docs and changelog before running the focused CLI/gateway checks.

## Reproduction Assessment

Yes. Current-main absence is reproducible by inspecting the CLI registration, forwarding path, and strict `AgentParamsSchema`; the PR-specific bug is reproducible from the diff by sending an owner-authorized gateway `agent` request with `workspaceDir: "."` or `"~/repo"` and observing that the server forwards the raw string.

## Solution Assessment

No, not yet. Reusing the existing `workspaceDir` plumbing and normalizing CLI input is the right shape, but accepted gateway overrides should be normalized with the same path resolver or restricted to absolute paths before dispatch, with docs and changelog aligned.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Normalize trusted workspace overrides before dispatch:** `src/gateway/server-methods/agent.ts:102`
  - body: The server accepts `workspaceDir` from owner-authorized clients but returns the trimmed raw value. A request with `workspaceDir: "."` or `"~/repo"` then reaches `agentCommand` and is resolved later by the gateway process, so non-CLI admin clients can run against a different workspace than intended. Normalize at gateway ingress, or require absolute paths, before assigning `trustedWorkspaceOverride`.
  - confidence: 0.9
- **[P3] Update docs and changelog for --cwd:** `src/cli/program/register.agent.ts:39`
  - body: This adds a user-facing `openclaw agent --cwd` option, but the diff only updates inline CLI help. `docs/cli/agent.md` and `CHANGELOG.md` still omit the new behavior, so the public CLI reference and release notes will drift.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: The diff does not show an untrusted privilege bypass after the owner-scope gate, but the new workspace-selection surface still needs path normalization before merge.

  - body: Owner-authorized gateway callers can supply relative or home-relative `workspaceDir` values that are forwarded raw and resolved later by the gateway process. Because this controls the agent tool workspace, normalize or reject those paths at request ingress.
