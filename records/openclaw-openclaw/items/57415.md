## Summary

Keep open: this is a useful CLI feature PR, but the current head still leaves a workspace-selection boundary bug in the gateway path and lacks required real behavior proof before merge.

## What This Changes

The PR adds an `openclaw agent --cwd <dir>` CLI option, forwards a normalized `workspaceDir` through local and gateway agent runs, extends the agent RPC/Swift protocol models, and adds regression tests.

## Best Possible Solution

Land a rebased revision that keeps CLI-side normalization, normalizes or rejects owner-authorized gateway `workspaceDir` overrides at ingress, preserves internal spawned-session inheritance, updates the public CLI docs, and includes redacted real behavior proof.

## Reproduction Assessment

Yes, source-level reproduction is high confidence. Current main lacks the option, forwarding, and schema field, and PR head visibly returns owner-authorized gateway `workspaceDir` values after trimming only.

## Solution Assessment

No, not yet. Reusing the existing workspace plumbing and normalizing CLI input is the right shape, but the public gateway ingress path should normalize or reject overrides before dispatch.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Normalize trusted workspace overrides before dispatch:** `src/gateway/server-methods/agent.ts:104`
  - body: This returns an owner-authorized `workspaceDir` after trimming only. Non-CLI owner/admin clients can send values like `.` or `~/repo`, which are then resolved later by the gateway process instead of at request ingress, so the run can target the gateway daemon cwd/home rather than the caller's intended workspace. Normalize with the same path resolver here, or require absolute paths before assigning the override.
  - confidence: 0.9
- **[P3] Document the new agent cwd option:** `src/cli/program/register.agent.ts:39`
  - body: This adds a user-facing `openclaw agent --cwd` option, but the linked CLI reference at PR head still has no option entry or example for it. Update `docs/cli/agent.md` so users following the command help can discover the new behavior.
  - confidence: 0.82
