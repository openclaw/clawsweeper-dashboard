## Summary

Keep open: current main still lacks the requested `openclaw agent --cwd` feature, but the PR needs a rebased revision because it opens a gateway workspace-selection surface with raw path handling, lacks docs, and has no real behavior proof.

The PR adds an `openclaw agent --cwd <dir>` option, forwards it as `workspaceDir` through local and gateway agent paths, updates gateway protocol models, and adds focused regression tests.

## Best Possible Solution

Rebase onto current main, make the workspace override contract explicit and normalized or absolute-only, preserve spawned-session workspace inheritance, update `docs/cli/agent.md` and generated protocol clients as needed, and add redacted real behavior proof.

## Reproduction Assessment

## Solution Assessment

No, not yet. Reusing existing workspace plumbing and normalizing CLI-originated cwd is the right direction, but gateway ingress needs a deliberate normalized/absolute contract and the docs/proof gaps must be closed before merge.
