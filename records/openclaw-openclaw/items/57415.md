## Summary

Keep open: current main still does not implement `openclaw agent --cwd`, and the related workspace PR was closed as superseded by this PR. The PR is still useful, but it needs contributor real behavior proof plus fixes for the gateway workspace path contract and CLI docs before merge.

## What This Changes

The PR adds an `openclaw agent --cwd <dir>` option, forwards normalized CLI workspaces through local and gateway agent runs, exposes `workspaceDir` in the gateway agent protocol and Swift models, and adds focused tests.

## Best Possible Solution

Rebase and refresh the PR, normalize or reject gateway `workspaceDir` at ingress, preserve spawned-session inheritance, update CLI docs and generated clients, and add redacted real behavior proof.

## Reproduction Assessment

Yes for source-level PR review. Current main lacks the CLI option and public gateway schema field, while the PR diff shows the raw owner-authorized gateway override and missing docs without needing to run the feature.

## Solution Assessment

No, not yet. CLI-side normalization is the right direction, but gateway ingress needs a clear normalized or absolute-only contract, and the docs/proof gaps must be closed before merge.
