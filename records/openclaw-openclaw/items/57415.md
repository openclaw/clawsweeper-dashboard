## Summary

Keep open. Current main still lacks the requested `openclaw agent --cwd` surface, and the PR is a relevant implementation path, but the patch still has a concrete gateway workspace-selection bug plus missing user-facing docs/release-note coverage before merge.

## What This Changes

The PR adds an `openclaw agent --cwd <dir>` option, forwards it through local and gateway agent execution as `workspaceDir`, extends the agent RPC protocol and generated Swift models, and adds CLI/gateway regression tests.

## Best Possible Solution

Land a rebased revision that preserves spawned-session workspace inheritance, normalizes or rejects owner-authorized gateway `workspaceDir` overrides at request ingress, keeps protocol schema/generated clients/docs/changelog aligned, and verifies the focused CLI and gateway regression tests before merge.

## Reproduction Assessment

Yes. Current main absence is reproducible by inspecting the CLI registration, CLI forwarding, and `AgentParamsSchema`; the PR-specific bug is reproducible from the diff by sending an owner-authorized gateway `agent` request with `workspaceDir: "."` or `"~/repo"` and observing that the server forwards the raw string.

## Solution Assessment

No, not yet. Reusing existing `workspaceDir` plumbing and normalizing CLI input is the right shape, but accepted gateway overrides should be normalized with the same path resolver or restricted to absolute paths before dispatch, with docs and changelog updated.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Normalize trusted workspace overrides before dispatch:** `src/gateway/server-methods/agent.ts:102`
  - body: The server accepts `workspaceDir` from owner-authorized clients but returns the trimmed raw value. A request with `workspaceDir: "."` or `"~/repo"` then reaches `agentCommand` and is resolved later by `ensureAgentWorkspace` in the gateway process, so non-CLI admin clients can run against a different workspace than intended. Normalize at gateway ingress, or require absolute paths, before assigning `trustedWorkspaceOverride`.
  - confidence: 0.9
- **[P3] Update docs and changelog for --cwd:** `src/cli/program/register.agent.ts:39`
  - body: This adds a user-facing `openclaw agent --cwd` option, but the diff only updates inline CLI help. `docs/cli/agent.md` and `CHANGELOG.md` still omit the new behavior, so the public CLI reference and release notes will drift.
  - confidence: 0.86
