## Summary

Keep open: the branch fixes a real Control UI gap and includes useful proof, but streamed tool-result images can still render twice because the PR copies top-level image blocks into the last tool card while grouped-render already renders those blocks.

## What This Changes

The PR adds Control UI extraction, stream propagation, typing, styling, rendering, and unit coverage so tool output cards can display image blocks from direct URLs and base64 sources.

## Best Possible Solution

Keep the UI renderer improvement, but choose one owner for streamed result images and add regression coverage proving a streamed image tool result renders exactly once.

## Reproduction Assessment

Yes, by source inspection: streamed result images are emitted as top-level image blocks, grouped-render renders top-level images, and the PR also copies those blocks into the last tool card. I did not run tests because this cleanup review is read-only.

## Solution Assessment

No, not quite yet. The maintainable fix is to keep image rendering in a single owner path for streamed tool results, then cover that path with focused grouped-render and stream regression tests.
