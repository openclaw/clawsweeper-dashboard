## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Reuse the authenticated image renderer for tool-card images:** `ui/src/ui/chat/tool-cards.ts:704`
  - body: This adds a second image renderer that sets `src` directly. Existing chat images special-case `/api/chat/media/outgoing/` by fetching with the Control UI auth token and requester session key before rendering a blob URL; a plain `<img>` request cannot send those headers, so generated/managed image URLs that work in normal chat will fail when shown inside expanded tool cards. Please share the existing image render path or factor it so tool-card images get the same managed-image handling.
  - confidence: 0.89
- **[P2] Preserve source-object images in streamed tool results:** `ui/src/ui/app-tool-stream.ts:164`
  - body: The PR body claims `{ source: { type: "base64", data, media_type } }` is supported, and `extractToolImages` handles it later, but live tool events pass through `extractToolOutputImages` first. This helper only accepts `url` and `data`, then returns null for `source`, so a streamed result using the structured source-object format is still emitted without an image block.
  - confidence: 0.88
