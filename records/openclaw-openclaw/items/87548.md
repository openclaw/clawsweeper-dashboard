## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Reuse the existing image render path for tool images:** `ui/src/ui/chat/tool-cards.ts:704`
  - body: The new tool-card renderer writes raw `card.images` into `<img src=...>`, but the existing message-image path resolves managed outgoing images by fetching them with auth and rendering blob previews. A tool result that returns the same `/api/chat/media/outgoing/...` URL would still be broken here because an `<img>` tag cannot attach the required headers, so this should share the current message-image resolver instead of duplicating a separate path.
  - confidence: 0.87
- **[P2] Preserve structured image sources in streamed tool results:** `ui/src/ui/app-tool-stream.ts:164`
  - body: The PR claims support for `source.data` plus `source.media_type`, and `extractToolImages` can read that shape if it reaches tool-card extraction, but live tool events first pass through `extractToolOutputImages()`. This helper returns `null` for `{ type: "image", source: { type: "base64", data, media_type } }`, so those streamed results still render as JSON/text instead of images.
  - confidence: 0.9
