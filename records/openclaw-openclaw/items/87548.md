## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Do not render streamed image results twice:** `ui/src/ui/chat/tool-cards.ts:346`
  - body: `buildToolStreamMessage` now emits each output image as a top-level `image` block, and `renderGroupedMessage` already renders those top-level images before inline tool cards via `renderMessageImages(extractImages(message))`. Copying those same message images into the last card makes direct/data streamed image results appear twice when the tool message is expanded.
  - confidence: 0.86
- **[P2] Preserve structured image sources during normalization:** `ui/src/ui/chat/message-normalizer.ts:412-415`
  - body: This new normalized `image` item keeps only `url`, `data`, and `mimeType`, so `{ type: "image", source: { type: "base64", data, media_type } }` becomes an image block with no renderable data after normalization. That leaves the source-object format path-dependent even though the PR says it supports it.
  - confidence: 0.82
