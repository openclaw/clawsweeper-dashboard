Full review comments:

- **[P1] Reuse the authenticated image renderer for tool-card images:** `ui/src/ui/chat/tool-cards.ts:704`
  - body: This new rendering path writes the extracted URL directly into `<img src>`, but current chat image rendering special-cases `/api/chat/media/outgoing/...` by fetching with the auth token and requester session key before creating a blob URL. Tool results can carry the same managed outgoing image URLs, so expanded tool cards can fail to load images that regular chat bubbles handle correctly; route these through the shared managed-image path instead of a plain img.
  - confidence: 0.91
- **[P2] Preserve source-object images during normalization:** `ui/src/ui/chat/message-normalizer.ts:409-416`
  - body: The new image branch keeps `url`, `data`, and `mimeType`, but drops `source`, so `{ type: "image", source: { type: "base64", data, media_type } }` normalizes into an image item with no renderable payload. The PR explicitly claims source-object support and the existing raw message image extractor already supports that shape, so normalization should preserve or convert it too.
  - confidence: 0.84
