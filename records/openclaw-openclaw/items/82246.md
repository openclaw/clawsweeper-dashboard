## Summary

Keep open: current main still lacks both requested `gateway.webchat` config keys, the live progress and transcript caps remain hardcoded, and both related implementation PRs were closed unmerged after real WebChat behavior did not match the proposed approach.

## What This Changes

Request to add `gateway.webchat.toolProgressMaxChars` and `gateway.webchat.toolTranscriptMaxChars` so live tool-progress output and persisted tool-result transcript truncation limits are configurable.

## Best Possible Solution

Decide the public WebChat truncation contract, then implement both caps through the actual live progress and persisted transcript/history paths with docs, schema/help/labels, generated baselines, focused tests, and redacted real WebChat proof.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable; this is a new config-surface request, and source inspection on current main confirms the proposed keys are absent while the existing caps remain hardcoded.

## Solution Assessment

Is this the best way to solve the issue? Unclear; the separate-knob direction follows the existing `chatHistoryMaxChars` pattern, but the closed PRs show the runtime and UI behavior need an end-to-end design before implementation.
