## Summary

Keep open: this remains a concrete, recent systemd status bug report, and the linked open pull request still references this issue with closing syntax. It should stay open until that PR lands or maintainers choose a different fix.

## What This Changes

The issue asks `openclaw status` to report an active/running systemd user Gateway service as running instead of `stopped (state failed)` when systemd and gateway health both show it is active.

## Best Possible Solution

Land a focused fix that makes the systemd runtime probe prefer the actual active user-unit state over stale failed metadata, with regression coverage for the reported active-unit case.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? No. The report has strong live evidence and current source shows how a failed `ActiveState` would render as `stopped (state failed)`, but this review did not establish the full Ubuntu user-systemd reproduction on current main.

## Solution Assessment

Is this the best way to solve the issue? Yes. The narrow maintainable solution is to fix the existing systemd service-state reader and add a regression for stale `show` failed-state output, with https://github.com/openclaw/openclaw/pull/79538 as the current implementation candidate.
