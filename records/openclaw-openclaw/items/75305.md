## Summary

Keep open. Current main still routes pre-compaction memory flush through a blank `transcriptPrompt`, converts that into a runtime-only empty prompt submission, and has tests asserting the blank path; the linked fix PR #75384 is closed unmerged.

## What This Changes

The issue reports that automatic pre-compaction memory flush can submit an empty user prompt to Anthropic and reset the session after a provider 400.

## Best Possible Solution

Land a narrow memory-flush handoff fix that sends a non-empty provider-visible flush prompt while preserving silent `NO_REPLY` delivery and hidden runtime context separation.
