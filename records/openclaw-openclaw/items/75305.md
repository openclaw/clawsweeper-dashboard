## Summary

Keep open. Current main still routes the pre-compaction memory flush through an empty `transcriptPrompt`, converts that to a runtime-only `prompt: ""`, and submits that blank prompt; open PR #75384 already tracks the narrow repair with closing syntax, so this issue should remain open until that PR is resolved.

## What This Changes

The issue reports that automatic pre-compaction memory flush submits an empty user prompt to Anthropic, causing a 400 error and session reset.

## Best Possible Solution

Keep the silent memory-flush behavior, but make the provider-facing flush turn non-empty and valid; land or revise PR #75384 or an equivalent narrow patch, then close this issue after the fix reaches main.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction path exists: memory flush passes `transcriptPrompt: ""`, runtime prompt resolution turns that into `prompt: ""` with `runtimeOnly: true`, and the embedded runner submits that empty prompt.

## Solution Assessment

No. Current main is not the best solution because it preserves the blank provider-visible submission; the maintainable fix is a focused non-empty flush prompt path that still suppresses user-visible delivery.
