## Summary

Keep open. Current main still sends the pre-compaction memory flush through the blank `transcriptPrompt` runtime-only path and then submits `prompt: ""`; the linked open PR #75384 is the appropriate implementation candidate, so this issue should stay open until that PR or an equivalent fix is merged.

## What This Changes

Report asks OpenClaw to stop submitting an empty user prompt during the silent pre-compaction memory flush because Anthropic rejects it and resets Telegram sessions.

## Best Possible Solution

Preserve the silent pre-compaction memory-flush behavior while making the provider-facing user turn non-empty, preferably by reviewing and landing PR #75384 or an equivalent narrow patch with updated tests; close this issue only after that fix merges.

## Reproduction Assessment

Yes. The source-level reproduction path is high confidence: memory flush sets `transcriptPrompt: ""`, runtime prompt resolution returns `prompt: ""` for a runtime-only turn, and the attempt path submits that blank prompt.

## Solution Assessment

No. Current main is not the best solution because it still submits the blank prompt; the narrow maintainable fix is to send a non-empty memory-flush prompt to the provider while keeping the turn silent to users.
