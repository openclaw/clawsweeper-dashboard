## Summary

Keep open: the feature gap is valid, but this PR is not merge-ready because the current head introduces a TypeScript parameter mismatch, generated-schema/source drift, and a regression in embedded MCP header placeholder handling.

## What This Changes

The branch adds a default-off `mcp.servers.<name>.injectCallerContext` option that injects OpenClaw caller headers into owner-trusted remote bundle MCP servers across CLI and embedded Pi paths, with schema, changelog, and tests.

## Best Possible Solution

Repair the narrow correctness issues, keep the opt-in owner-declared exact-URL trust boundary, then have maintainers decide whether to ship the session-identity forwarding contract.

## Reproduction Assessment

Yes. The review findings are source-reproducible from the PR head: the changed call site passes an option the exported helper type does not accept, the generated schema output lacks matching source metadata, and the embedded placeholder helper rewrites unknown `${...}` header values.

## Solution Assessment

No, not as-is. The owner-managed exact-URL opt-in is a maintainable direction, but the PR needs the type surface, schema source maps, and embedded placeholder expansion fixed before trust-model review and validation can proceed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Accept callerContext in the public runtime helper:** `src/agents/pi-embedded-runner/run/attempt.ts:1070`
  - body: The new call passes `callerContext` into `getOrCreateSessionMcpRuntime`, but that exported helper's parameter type still only allows `sessionId`, `sessionKey`, `workspaceDir`, and `cfg`. TypeScript will reject this object literal as an excess property in the embedded runner path; add `callerContext` to the helper parameter type or route through the typed manager API.
  - confidence: 0.96
- **[P2] Add schema help sources before regenerating:** `src/config/schema.base.generated.ts:27688-27691`
  - body: This generated label/help entry is not backed by `FIELD_LABELS` or `FIELD_HELP` entries for `mcp.servers.*.injectCallerContext`. `schema.base.generated.test.ts` recomputes the base schema from those source maps and compares it with this generated file, so the generated-schema check will drift or fail until the source metadata is added and outputs are regenerated.
  - confidence: 0.93
- **[P2] Leave unrelated header placeholders unchanged:** `src/agents/pi-bundle-mcp-runtime.ts:213`
  - body: The embedded expansion path claims to replace `OPENCLAW_MCP_*` placeholders, but this regex replaces every `${VAR}` with the caller env map or `""`. An opted-in remote MCP server that already has `Authorization: "Bearer ${MY_MCP_TOKEN}"` would lose its auth header value; preserve unknown placeholders or only rewrite the four OpenClaw caller placeholders.
  - confidence: 0.88
