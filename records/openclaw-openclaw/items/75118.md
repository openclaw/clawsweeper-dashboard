## Summary

Keep open: current main still lacks this feature, and the PR is useful but not merge-ready because its config schema sources/generated output are inconsistent and the embedded Pi path can rewrite existing header placeholders when caller context is enabled.

## What This Changes

The PR adds a default-off `mcp.servers.<name>.injectCallerContext` option that injects OpenClaw caller headers into owner-trusted remote bundle MCP servers for CLI and embedded Pi runtimes, with schema, changelog, and test updates.

## Best Possible Solution

Land a repaired version that keeps the owner-scoped exact-URL opt-in, derives config docs from source metadata, preserves existing operator headers, and passes targeted schema and bundle MCP validation after maintainer trust review.

## Reproduction Assessment

Yes. The feature gap is source-verifiable on current main, and the PR defects are source-verifiable from the generated-schema equality test and the embedded header replacement code.

## Solution Assessment

No, not as-is. The owner-managed exact-URL opt-in is a plausible implementation shape, but the schema drift and embedded placeholder rewriting need repair before maintainers can judge and merge the feature.
