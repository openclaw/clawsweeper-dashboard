## Summary

Keep open: current main has partial fallback hardening, including recent silent-error retry work, but the central report is not fully implemented because exact auth/code classification, raw-vs-formatted assistant error classification, and strict user session override policy still need maintainer review and focused proof.

## What This Changes

The issue requests that configured model fallback continue through later candidates after Codex/OpenAI auth-style failures, zero-output assistant failures, session/model override fallback drops, or OpenAI Responses auth-profile mismatches.

## Best Possible Solution

Split the work by contract: add focused classifier and zero-output fallback regressions for the true bug paths, and keep user session model overrides strict unless maintainers explicitly decide to change that documented behavior.

## Reproduction Assessment

The session/model override piece is source-proven as current behavior, but whether it should change is a policy question.

## Solution Assessment

Unclear as a single fix: adding auth classifier and zero-output fallback regressions is likely the narrow repair for the bug portions, but preserving fallbacks for user model overrides conflicts with current docs and tests. The best solution is to separate the bug fixes from any intentional policy change.
