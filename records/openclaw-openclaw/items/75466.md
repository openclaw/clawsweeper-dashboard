Keep this PR open: current main still uses the raw textarea, so the CodeMirror feature is not already implemented, but the latest PR head is not mergeable because it removes the sensitive raw-config reveal control while the hidden-state UI and browser test still depend on it. The branch also has a changelog credit mismatch and a formatter path that can silently drop JSON5 comments.

## What This Changes

The PR adds a Lit CodeMirror 6 JSON5 editor for the Control UI raw config view, wires it into the config view and browser test, adds CodeMirror-related UI dependencies and lockfile entries, and adds a changelog note.

## Best Possible Solution

Keep the CodeMirror JSON5 editor direction, but merge it only after the branch preserves the current sensitive-value header, pill, reveal/hide button, and redacted-until-reveal DOM guard; fixes the changelog attribution; and makes the Format action safe for JSON5 comments or clearly guarded. Richer editor navigation/readability should stay coordinated with the still-open #40732 work.

No. CodeMirror is a reasonable solution for the raw JSON5 editor, but this patch is not the best mergeable form until it preserves the existing reveal-to-edit safety flow, avoids destructive JSON5 formatting, and corrects contributor credit.
