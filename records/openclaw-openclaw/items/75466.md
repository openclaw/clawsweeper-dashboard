  - confidence: 0.95

  - body: `codemirror-json5` already exports `json5ParseLinter` and a parse cache, but this component reimplements a separate parser and error-offset path. That duplicates the dependency's supported behavior and can drift from the language mode used by the editor.
  - confidence: 0.86
- **[P2] Avoid destructive JSON5 formatting:** `ui/src/ui/views/config-editor.ts:346-349`
  - body: `JSON5.parse()` returns a plain value and `JSON5.stringify()` serializes that value, so clicking Format on a JSON5 config with comments removes comments and source trivia before save. The raw config format allows JSON5 comments, so this needs a syntax-preserving formatter or an explicit normalization flow.
  - confidence: 0.88
