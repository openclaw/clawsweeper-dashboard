## Summary

Keep this PR open as a draft implementation candidate, but do not merge it as-is. Current main still uses the raw textarea flow, while this patch introduces a sensitive raw-config disclosure regression and unrelated dependency/changelog churn that should block merge.

## What This Changes

The PR adds a Lit CodeMirror 6/JSON5 raw config editor, wires it into the Control UI raw config view, updates the raw-config browser test, adds CodeMirror/JSON5 dependencies with lockfile changes, and edits the changelog.

## Best Possible Solution

Land a focused Control UI editor change that preserves the existing reveal-to-edit secret boundary, keeps dependency changes strictly additive for the editor, restores unrelated changelog entries, and adds coverage for hidden, revealed, edit, and format paths.

## Reproduction Assessment

Yes. Render the current raw config browser test scenario with `formMode: "raw"`, `raw` containing `openai.apiKey`, and `rawRevealed=false`; the PR mounts `config-editor` with the full raw string before reveal.

## Solution Assessment

No. CodeMirror 6/JSON5 is a reasonable direction for this surface, but the current patch is not the narrow maintainable fix because it bypasses existing redaction semantics and mixes unrelated package/changelog changes.
