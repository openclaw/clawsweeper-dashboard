## Summary

Keep this PR open: current main still uses the textarea raw editor, so the CodeMirror direction is not already implemented, but this PR is not mergeable as-is because PR head removes the sensitive raw-config reveal control, the Format action can silently drop JSON5 comments, and the changelog credits the wrong GitHub handle. ## What This Changes

This PR adds a Lit CodeMirror 6 JSON5 editor for the Control UI raw config view, wires it into raw-mode rendering and browser tests, adds CodeMirror/JSON5 dependencies and lockfile entries, and adds a changelog note.

## Best Possible Solution

Land a focused CodeMirror raw editor that preserves the existing sensitive-value reveal header and DOM guard, avoids destructive comment-stripping format behavior, fixes the contributor credit, and then coordinates richer editor readability with #40732.

## Reproduction Assessment

Yes. A second focused check is clicking Format on a JSON5 document with comments, which follows a parse/stringify path that cannot retain comments.

## Solution Assessment

No. CodeMirror is a reasonable implementation direction, but this patch is not the best mergeable form until it preserves the reveal-to-edit flow, prevents comment loss from formatting, fixes changelog credit, and rebases cleanly onto current main.
