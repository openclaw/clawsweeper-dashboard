## Summary

Keep open. Current main still matches the gateway-host full/off exec behavior described in the report, with fresh child-shell execution and no checked-in stale-read mitigation or reproduction harness, so this remains a plausible macOS/APFS correctness issue that needs maintainer reproduction and design review.

## What This Changes

The issue reports stale file reads after atomic-rename writes across separate gateway-host YOLO exec calls on macOS/APFS beginning with the 2026.4.20 release.

## Best Possible Solution

Keep this open for focused maintainer investigation on an affected macOS/APFS gateway-host setup, compare v2026.4.19-era behavior, v2026.4.20, v2026.4.29, and current main, then land either a narrow runtime fix or a documented platform workaround for atomic-rename writes followed by reads from separate exec calls.
