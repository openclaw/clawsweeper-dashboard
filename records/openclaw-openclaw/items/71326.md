## Summary

Keep open. Current main still matches the reported gateway-host full/off exec shape, and I did not find a current-main mitigation, shipped fix, duplicate tracker, or linked fix PR that would make this safe to close.

## What This Changes

The issue reports stale reads after atomic-rename writes when separate gateway-host exec calls read the same file on macOS/APFS, suspected as a regression around the 2026.4.20 YOLO exec/preflight path.

## Best Possible Solution

Keep this open for focused maintainer investigation on an affected macOS/APFS gateway-host setup, compare v2026.4.19-era behavior, v2026.4.20, v2026.4.29, and current main, then land either a narrow runtime fix or a documented platform workaround for atomic-rename writes followed by reads from separate exec calls.

## Reproduction Assessment

Unclear. The report gives a concrete write/read sequence and current source confirms the relevant exec path still exists, but this review did not have a macOS/APFS gateway-host environment or checked-in harness that reproduces the stale-read window against current main.

## Solution Assessment

Unclear. The requested investigation is appropriate, but the proposed long-lived interpreter or F_NOCACHE directions are broader than the evidence supports until maintainers reproduce the race and choose the correct exec/runtime boundary.
