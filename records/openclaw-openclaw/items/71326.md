## What This Changes

The issue asks OpenClaw to investigate and fix stale file reads observed across separate gateway-host exec calls on macOS/APFS after atomic-rename writes, reportedly beginning with the 2026.4.20 release.

## Best Possible Solution

Keep this open for focused maintainer investigation on an affected macOS/APFS gateway-host setup, compare pre/post YOLO-preflight behavior against current main, then land either a narrow exec-runtime mitigation or a documented platform workaround for atomic-rename writes followed by reads from separate exec calls.
