Keep open. The report is recent, has a second independent Linux/ext4 confirmation, and current main still matches the implicated gateway-host YOLO path: script preflight is skipped and each non-PTY host exec is spawned through a fresh child process. I found no canonical duplicate or implemented mitigation, but there is still no deterministic checked-in or live-reproduced current-main harness, so this needs maintainer reproduction and design judgment before a safe fix can be scoped.

## What This Changes

The issue asks OpenClaw to investigate and mitigate stale cross-exec file reads in gateway-host YOLO mode after 2026.4.20, especially around atomic-rename writes and Active Memory sub-agent runs.

## Best Possible Solution

Keep this open for maintainer-owned live reproduction on affected gateway-host YOLO setups, then choose the narrowest exec/runtime or plugin write-boundary mitigation and update docs if the mechanics remain user-visible.
