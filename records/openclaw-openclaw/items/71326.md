## Summary

Keep open. Current main still has the gateway-host `security=full` plus `ask=off` YOLO preflight skip and the per-call child-shell exec path that match the reported surface, and I found no source, docs, tests, changelog, or related-item evidence that a stale-read mitigation has landed. ## What This Changes

The issue reports stale file reads across separate gateway-host exec calls on macOS/APFS after atomic rename writes, allegedly regressing in v2026.4.20.

## Best Possible Solution

Keep this open for focused maintainer investigation on an affected macOS/APFS gateway-host setup, compare v2026.4.19, v2026.4.20, v2026.4.29, and current main, then land either a narrow exec-runtime mitigation or a documented platform workaround for atomic-rename writes followed by separate exec-call reads.
