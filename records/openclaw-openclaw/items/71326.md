Keep open. The issue is recent, has a second independent Linux/ext4 confirmation, current main still has the reported gateway-host YOLO per-call child-shell execution path, and I found no merged fix or canonical duplicate that resolves the stale-read behavior.

## What This Changes

The issue reports stale cross-exec file reads after atomic rename writes in gateway-host YOLO exec mode, suspected to come from per-call short-lived shell/process execution.

## Best Possible Solution

Keep this open for maintainer-owned live reproduction on affected gateway-host setups, then choose the narrowest exec/runtime or plugin-write mitigation with focused regression coverage.

Unclear. The requested investigation is appropriate, but selecting long-lived process reuse, cache-control calls, write-side fsync/sleep, retries, or docs needs maintainer reproduction and runtime design judgment first.
