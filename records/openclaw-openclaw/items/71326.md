## Summary

Keep open. Current main still matches the reported gateway-host full/off exec shape: YOLO mode skips script preflight, non-sandbox exec builds a fresh shell child argv per call, and the relevant exec/runtime files have not changed since the latest release tag, so there is no strong implemented-on-main or duplicate/superseded basis for cleanup.

## What This Changes

The issue reports stale reads of atomically renamed files across separate gateway-host full/off exec calls on macOS/APFS after 2026.4.20 and asks maintainers to investigate or mitigate the exec preflight/runtime boundary.

## Best Possible Solution

Keep this open for focused maintainer investigation on an affected macOS/APFS gateway-host setup, compare behavior around the 2026.4.20 change, v2026.4.29, and current main, then land either a narrow exec/runtime fix or a documented platform workaround for atomic-rename writes followed by reads from separate exec calls.
