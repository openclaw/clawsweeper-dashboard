## Summary

Current main already fixes the reported Codex image_generate dynamic-tool timeout: the bridge now derives the watchdog from per-call timeoutMs or agents.defaults.imageGenerationModel.timeoutMs and the lower app-server fallback is 600000 ms. The fix is current-main-only relative to v2026.5.7, so released users may still see the old behavior until the next release.

## What This Changes

The issue requests that Codex dynamic-tool calls honor image_generate per-call and configured timeoutMs budgets instead of failing at the 30000 ms bridge default.

## Best Possible Solution

Keep the current-main fix for image_generate timeout budgets; broader media-tool or configurable watchdog work should continue through the existing Codex watchdog follow-up rather than reopening this specific image_generate bug.

## Reproduction Assessment

Yes for the reported v2026.5.7 behavior by source inspection: that release still routes dynamic-tool calls through 30000 ms bridge constants. Current main no longer reproduces the issue because the relevant timeout resolver and 600000 ms client fallback are present with regression coverage.

## Solution Assessment

Yes. The current-main implementation is the narrow maintainable fix for this issue: it preserves the default bridge watchdog while honoring bounded per-call and configured image_generate timeout budgets, without requiring a new config surface or background-task redesign for this specific bug.
