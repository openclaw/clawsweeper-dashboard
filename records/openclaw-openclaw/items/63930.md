## Summary

Keep open. Current main still lacks Anthropic advisor support and the underlying server-side Anthropic block handling, while open PR #64064 by the same reporter tracks this exact work and remains the appropriate implementation path for maintainer review.

## What This Changes

The issue requests Anthropic advisor beta tool support, generic handling for Anthropic server-side tool/result stream blocks, and safer unknown stop-reason handling.

## Best Possible Solution

Land #64064 or a maintainer-approved replacement that keeps the owner boundary intact: core transport should handle Anthropic server-side stream/display/replay semantics generically, while the bundled Anthropic plugin owns advisor config parsing, beta-header/tool injection, model validation, docs, and focused tests. Deferred encrypted-content round-tripping or advisor-cost accounting should be tracked as explicit follow-ups if not included.

## Reproduction Assessment

Yes. A focused Anthropic stream fixture containing server_tool_use or advisor_tool_result would currently be ignored because the content_block_start handler only maps text, thinking, redacted_thinking, and tool_use; an unknown stop_reason would still hit mapStopReason's throwing default branch.

## Solution Assessment

Unclear for the open implementation until maintainers review #64064's tradeoffs. The issue itself should remain open and attached to that PR rather than being closed or duplicated by a separate automated repair branch.
