## Summary

Keep open. Current main still supports macOS Gateway installs that run through an external/shared Node executable unless an optional wrapper is supplied, so the requested default OpenClaw-owned TCC boundary is not implemented.

## What This Changes

Request to ship a bundled or dedicated OpenClaw-owned Node runtime on macOS so FDA/Accessibility grants are scoped to OpenClaw rather than a shared Node install.

## Best Possible Solution

Keep this as core macOS security packaging work: use a stable signed OpenClaw-owned identity for TCC-sensitive Gateway/node-host/channel work and align installer, LaunchAgent repair/update, doctor, onboarding, and docs.

## Reproduction Assessment

Yes. The source-level reproduction is `openclaw gateway install --runtime node` without `--wrapper`, including the macOS app local-mode path, which can resolve a shared Node executable and write it as the LaunchAgent executable.

## Solution Assessment

No. The current wrapper and app-broker mitigations are useful but incomplete because the default supported Gateway path can still run under shared Node; a complete fix needs a maintainer-approved signed runtime identity and migration plan.
