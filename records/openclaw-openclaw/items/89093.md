## Summary

Keep open: the patch addresses a source-backed supervised auto-update bug and I found no blocking correctness issue, but it changes restart handoff behavior for live gateway services, so maintainer acceptance of the availability risk remains the gate.

## What This Changes

The branch moves the managed-service update handoff into `src/infra` and routes supervised startup auto-updates through it with channel/timeout forwarding, marker stripping, restart scheduling, and focused tests.

## Best Possible Solution

Land the shared managed-service handoff after maintainers accept the Linux package proof plus focused cross-supervisor tests, or request one more packaged launchd/Scheduled Task proof before merge if that availability risk needs direct coverage.

## Reproduction Assessment

Yes. Current main and `v2026.6.1` show startup auto-update spawning the update child with inherited env, while the CLI package-update guard rejects gateway-service marker env.

## Solution Assessment

Yes. Reusing the managed-service handoff is a better fix than only stripping markers because it avoids replacing the package tree from a live gateway child and aligns startup auto-update with `update.run`; the remaining question is platform proof depth.
