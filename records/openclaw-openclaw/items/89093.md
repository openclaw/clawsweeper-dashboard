The PR moves the managed-service update handoff helper into `src/infra`, re-exports it for gateway callers, routes supervised startup auto-updates through that handoff with channel, timeout, and auto-update env, and adds infra/gateway tests.

## Best Possible Solution

Land the shared infra handoff only after maintainers either accept the helper-level proof for this availability-sensitive path or get packaged launchd/systemd proof that the gateway exits, updates, and returns healthy.

## Solution Assessment

Is this the best way to solve the issue?
