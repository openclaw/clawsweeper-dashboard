The PR moves the managed-service update handoff helper into `src/infra`, re-exports it for gateway callers, and routes supervised startup auto-updates through that handoff with channel, timeout, and `OPENCLAW_AUTO_UPDATE=1` handling.

## Best Possible Solution

Merge only after redacted launchd or systemd packaged auto-update proof shows the handoff starts, the package update completes, and the gateway returns healthy; otherwise a maintainer should explicitly accept the availability risk.

## Reproduction Assessment

No live reproduction was run, but the bug is source-reproducible: current main spawns the auto-update child with only `OPENCLAW_AUTO_UPDATE=1`, while the CLI guard blocks inherited gateway service markers.

## Solution Assessment

Yes, the direction looks like the best maintainable fix because it reuses the existing managed-service handoff instead of adding a one-off marker-stripping child update path; the missing piece is real supervisor proof, not a clear code defect.
