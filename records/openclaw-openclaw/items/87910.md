## Summary

Current main already fixes the reported chat session picker regression: blurring an empty picker search no longer clears/reloads picker state before a session option click, and cron sessions can be shown with the existing cron visibility toggle. The fix is current-main-only and is not contained in v2026.5.27 yet.

## What This Changes

The issue reports that OpenClaw v2026.5.27 cannot switch conversations from the blank session search picker and does not show cron sessions like v2026.5.22 did.

## Best Possible Solution

Keep the current main picker blur guard and cron-session visibility toggle, then ship a release containing the fix for v2026.5.27 users.

## Reproduction Assessment

Yes. Source comparison and the current regression test identify the released failure path: open the session picker, blur an empty search input, then click another session option; current main covers that click path successfully.

## Solution Assessment

Yes. Guarding empty blur preserves picker state during option clicks while keeping non-empty search behavior, and cron visibility stays under the existing explicit toggle.
