## Summary

Keep open: the linked Voice Call bug is source-reproducible on current main, but this PR still has merge-blocking patch issues and lacks after-fix real behavior proof.

## What This Changes

The PR changes generated Voice Call per-phone/per-call session keys to `agent:<agentId>:voice:*`, updates resolver tests, and refreshes Codex prompt snapshot fixtures.

## Best Possible Solution

Land a Voice Call-owned fix that uses shared normalized agent-id/session-key semantics, updates resolver and call-path coverage, avoids unrelated snapshot churn, and includes redacted restart proof.

## Reproduction Assessment

Yes, source-reproducible: current main generates raw `voice:*` keys while startup migration rewrites those keys to normalized `agent:<agentId>:voice:*` keys. I did not run a live telephony restart scenario in this read-only review.

## Solution Assessment

Partly: moving generated Voice Call keys to the canonical agent-scoped shape is the right owner-boundary fix. The patch should normalize agent ids through the shared contract, update the affected call-path tests, and drop unrelated Codex snapshot churn before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Normalize agent IDs before building session keys:** `extensions/voice-call/src/config.ts:729`
  - body: This interpolates raw `config.agentId` into the supposedly canonical key. Gateway migration and session-store helpers use `normalizeAgentId`, so configured ids with uppercase, surrounding spaces, or unsupported characters can still persist under a different key after restart. Use the shared normalizer before composing the Voice Call key.
  - confidence: 0.9
- **[P2] Align call-path tests with canonical generated keys:** `extensions/voice-call/src/config.ts:730-736`
  - body: The resolver behavior change affects inbound and outbound call records, but current manager/outbound tests still assert raw `voice:call:<callId>` keys. The broader Voice Call test lane is likely to fail until those expectations and related mocks are updated.
  - confidence: 0.88
- **[P2] Revert unrelated Codex prompt snapshot churn:** `test/fixtures/agents/prompt-snapshots/codex-runtime-happy-path/codex-dynamic-tools.telegram-direct.json:3`
  - body: The branch removes `namespace` and `deferLoading` from Codex dynamic-tool snapshots even though the PR does not change Codex tool generation, and current main still emits those fields for searchable tools. Keeping these fixture edits would bless an unrelated prompt shape change.
  - confidence: 0.84
