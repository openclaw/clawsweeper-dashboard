## Summary

Keep this PR open: it targets a valid Voice Call restart/session continuity bug, but the current patch still diverges from the gateway session-key canonicalization contract, leaves broader call-path tests on the old raw key shape, and lacks required real behavior proof.

## What This Changes

The PR changes the Voice Call session-key resolver and config tests to emit generated `agent:<agentId>:voice:*` keys, and also rewrites several Codex prompt snapshot fixtures.

## Best Possible Solution

Land a Voice Call-owned fix that uses the shared canonical session-key/agent-id normalization contract, updates resolver and call-path coverage together, removes unrelated snapshot churn, and includes redacted restart proof for per-phone continuity.

## Reproduction Assessment

Yes, source-reproducible: current main generates raw `voice:*` keys while startup migration canonicalizes raw keys to normalized `agent:<agentId>:voice:*` store keys. I did not establish a live telephony restart reproduction in this read-only pass.

## Solution Assessment

No, not as written: the direction is right, but the implementation should normalize/build keys through the shared session-key contract and update the full Voice Call test surface. The unrelated prompt snapshot changes should not ride along with this bug fix.
