## Summary

This is a plausible owner-boundary fix for a real Voice Call continuity bug, but it should stay open because the external PR lacks real restart proof and the patch still needs canonical agent-id handling plus adjacent test updates.

## What This Changes

The PR changes the Voice Call session resolver and config tests to emit generated per-phone and per-call keys as `agent:<agentId>:voice:*` while preserving explicit session keys.

## Best Possible Solution

Land a voice-call-owned fix that builds generated keys with the same normalized agent-id semantics as gateway migration, updates all affected resolver and call-path tests, and includes redacted restart proof for per-phone continuity.

## Reproduction Assessment

Yes, source-reproducible: current main generates raw `voice:*` keys while the startup migration rewrites raw keys to `agent:<agentId>:voice:*`, matching the linked restart mismatch. I did not run a live telephony restart scenario in this read-only pass.

## Solution Assessment

Partly: moving generated Voice Call keys to the canonical agent-scoped shape is the right owner-boundary fix. The patch should use the shared normalized agent-id contract, update the broader voice-call test expectations, and provide real behavior proof before merge.
