## Summary

Keep open. The Voice Call bug is valid and this PR is aimed at the right code path, but the branch still raw-interpolates `agentId`, leaves affected Voice Call call-path expectations stale, includes unrelated Codex prompt snapshot churn, and lacks after-fix real restart proof.

## What This Changes

This PR prefixes generated Voice Call per-phone/per-call session keys with `agent:<agentId>:voice:*`, updates resolver tests, and changes Codex prompt snapshot fixtures.

## Best Possible Solution

Land a Voice Call-owned fix that builds keys with the normalized plugin SDK agent-id contract, updates the affected Voice Call tests, drops unrelated Codex fixture churn, and includes redacted restart proof.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: current main generates raw `voice:*` keys while startup migration canonicalizes those records to normalized `agent:<agentId>:voice:*` keys, conflicting with the documented per-phone memory behavior. I did not run a live telephony restart scenario in this read-only review.

## Solution Assessment

No. The direction is right, but this branch should normalize `agentId`, update all affected Voice Call call-path tests, remove unrelated Codex snapshot changes, and add real restart proof before merge.
