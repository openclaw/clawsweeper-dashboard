## Summary

Keep open. The branch targets a real source-reproducible Voice Call continuity bug, but the submitted patch is incomplete because generated keys are not fully canonicalized and the broader Voice Call tests/mocks still exercise raw `voice:*` keys; real behavior proof is also unit-test-only.

## What This Changes

The PR changes the Voice Call session-key resolver to prefix generated per-phone and per-call keys with `agent:<agentId>:voice:*` and updates focused config resolver tests.

## Best Possible Solution

Merge a complete Voice Call resolver fix that reuses existing routing normalization, lowercases generated canonical keys, preserves explicit-key passthrough, updates all affected Voice Call tests/docs, and includes redacted restart or continuity proof.

## Reproduction Assessment

Yes, at source level. Current main generates raw `voice:*` keys while Gateway startup migration rewrites those keys into `agent:<agentId>:voice:*`, and the PR-specific normalization gap is visible with mixed-case `agentId` or carrier `callId` values.

## Solution Assessment

No, not as submitted. Prefixing generated keys is the right narrow direction, but the branch should normalize/lowercase generated keys, update the broader Voice Call expectations, and supply real restart or continuity proof before merge.
