## What This Changes

The PR prefixes generated Voice Call session keys with `agent:<agentId>:voice:*`, updates resolver tests, and regenerates Codex prompt snapshots.

## Best Possible Solution

Land a Voice Call-owned fix that uses shared normalized agent-id/session-key semantics, updates resolver and call-path coverage, removes unrelated Codex snapshot churn, and includes redacted restart proof.

## Reproduction Assessment

Yes, source-reproducible: current main generates raw `voice:*` keys while startup migration rewrites those keys to normalized `agent:<agentId>:voice:*`, and docs promise per-phone continuity. I did not run a live telephony restart scenario.

## Solution Assessment

Partly: canonical agent-scoped Voice Call keys are the right owner-boundary fix, but this PR should normalize agent ids through the shared plugin SDK contract, update affected call-path tests, and drop unrelated Codex snapshot changes.
