## Summary

Keep open: current main still drops the source-only TypeScript package-entry path, and an open PR already references this issue with closing syntax. The remaining action is to review or adjust that PR rather than close this issue early.

## What This Changes

Request that local TypeScript-source plugins with both `package.json` and `openclaw.plugin.json` not be silently dropped by strict compiled-runtime package validation.

## Best Possible Solution

Review and land, or maintainer-adjust, https://github.com/openclaw/openclaw/pull/80557 so managed installs stay strict while intentional untracked local source checkouts can load TypeScript safely.

## Reproduction Assessment

Yes. Current main is source-reproducible from the discovery path and the focused test: a global package entry pointing at TypeScript without built JavaScript is excluded with the compiled-runtime warning; I did not run a live gateway repro in this read-only review.

## Solution Assessment

Unclear until maintainer policy is accepted. The linked PR's install-record based classification is narrower than flipping manifest precedence, but it still changes global source-loading semantics and needs review.
