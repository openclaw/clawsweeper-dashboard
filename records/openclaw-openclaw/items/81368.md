## Summary

Keep this issue open: current main still does not classify `attempt_dispatch` as execution-started, and the issue now has an open focused fix PR with closing syntax. The linked beta.6 and beta.8 reports are useful current evidence, so the issue should close only after that PR lands or is replaced by an equivalent merged fix.

## What This Changes

The issue reports that isolated cron runs reaching `attempt_dispatch` are still killed by the pre-execution watchdog after about 60 seconds instead of honoring the configured job timeout.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81871 or an equivalent narrow classifier fix, then close this issue from the merge.

## Reproduction Assessment

Yes. The issue has live beta logs showing `last phase: attempt-dispatch`, and current main source shows that emitted phase does not clear the pre-execution watchdog.

## Solution Assessment

Yes. Treating the existing `attempt_dispatch` phase as execution-started in the private classifier, with focused regression coverage, is the narrowest maintainable fix.
