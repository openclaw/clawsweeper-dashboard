## Summary

Keep open: the reported `attempt_dispatch` watchdog gap is still present on current `main`, and an open linked PR is the active fix path.

## What This Changes

The issue asks cron to treat `attempt_dispatch` as an execution-start milestone so isolated runs are not killed by the 60s pre-execution watchdog after dispatch.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81371 if it passes, then close this issue with the merged commit and release provenance.

## Reproduction Assessment

Yes. The source-level reproduction is an isolated cron run that reports `runner_entered`, then `attempt_dispatch`, then runs past 60s; current `main` does not classify that phase as execution-started, matching the reporter's beta.4 output.

## Solution Assessment

Yes. Treating `attempt_dispatch` as execution-started in the private cron classifier, with a focused regression test, is the narrowest fix for the path where no later milestone arrives before the watchdog fires.
