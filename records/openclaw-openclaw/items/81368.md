## Summary

Keep open: current main still does not classify `attempt_dispatch` as cron execution-started, and an open focused PR already uses closing syntax for this issue, so the issue should remain open until that PR lands or is replaced.

## What This Changes

Long-running isolated cron agent-turns should stop being falsely aborted by the pre-execution watchdog after reaching `attempt_dispatch`.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/81371 or an equivalent narrow classifier/test change, then close this issue after the fix is merged and released or clearly available on main.

## Reproduction Assessment

Yes, at source level: current main can keep the pre-execution watchdog armed after an `attempt_dispatch` phase because that phase is not considered execution-started. I did not run a live cron job, but the issue comments provide matching beta.4 and beta.6 runtime logs.

## Solution Assessment

Yes. The linked PR's one-line classifier addition plus focused regression test is the narrowest maintainable fix for this false pre-execution stall; richer runner-side model-start telemetry can remain separate.
