## Summary

Keep open: current main still does not classify `attempt_dispatch` as cron execution-started, and a direct open fixing PR already references this issue with closing syntax. The issue should remain the canonical tracker until that PR lands or is replaced.

## What This Changes

Treat `attempt_dispatch` as an execution-start milestone so isolated cron jobs are not killed by the 60s pre-execution watchdog after dispatch has begun.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/81371, then close this issue after the fix reaches main and the release path is clear.

## Reproduction Assessment

Yes, as a source reproduction: current main can record `attempt_dispatch`, but `isCronAgentExecutionStarted` returns false for that phase, so a dispatched isolated cron run can still hit the 60s pre-execution timeout. I did not run a live cron smoke in this read-only review.

## Solution Assessment

Yes. The open PR's one-case classifier change plus focused regression test is the narrowest maintainable fix path and avoids changing watchdog duration, cron scheduling, or runner policy.
