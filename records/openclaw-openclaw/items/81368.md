## Summary

Keep open: current main still does not classify `attempt_dispatch` as an execution-start milestone, and an open fix PR already references this issue with closing syntax.

## What This Changes

The issue asks for isolated cron runs that reach `attempt_dispatch` to clear the pre-execution watchdog instead of failing after roughly 60 seconds.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81371 or an equivalent one-line classifier fix with the focused regression test, then close this issue after the fix reaches main.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: emit `runner_entered`, then only `attempt_dispatch`, then wait past the pre-execution watchdog window. The reporter also supplied beta.4 hotfix proof and a beta6 duplicate added matching Codex-harness evidence.

## Solution Assessment

Yes, adding `attempt_dispatch` to `isCronAgentExecutionStarted` with a regression test is the narrowest maintainable fix for this false pre-execution watchdog abort. A deeper runner-phase redesign can be separate, but it is not required to stop this failure mode.
