## Summary

Keep open: current main still does not classify `attempt_dispatch` as execution-started, and https://github.com/openclaw/openclaw/pull/81371 is an open closing PR for this exact remaining cron watchdog bug.

## What This Changes

The issue asks cron to stop treating isolated agent runs that reach `attempt_dispatch` as pre-execution stalls and let them continue until completion or the configured job timeout.

## Best Possible Solution

Review and land or revise https://github.com/openclaw/openclaw/pull/81371, then close this issue after the fix is merged and released or clearly present on main.

## Reproduction Assessment

Yes. Source inspection plus reporter live logs show an isolated agent cron run can record `attempt_dispatch`, which current main accepts as a phase but does not treat as watchdog-clearing execution progress; I did not run the live job, so this is source-reproducible rather than locally reproduced.

## Solution Assessment

Yes. The narrow maintainable solution is the linked focused PR that adds `attempt_dispatch` to the execution-start classifier with regression coverage, or an equivalent earlier runtime milestone that clears the same watchdog path.
