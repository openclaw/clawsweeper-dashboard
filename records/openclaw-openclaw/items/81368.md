## Summary

Keep open: current main still has the `attempt_dispatch` classifier gap, and an open PR already references this issue with the focused fix and regression coverage, so the issue should remain open until that PR lands or maintainers choose a different implementation.

## What This Changes

Treat `attempt_dispatch` as execution-started for isolated cron’s pre-execution watchdog so long-running cron jobs use the configured job timeout.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81871 if normal gates pass, then close this issue as fixed by that merged PR.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: an isolated cron run that reports `attempt_dispatch` but no later classified execution milestone leaves the pre-execution watchdog armed until it fires. The issue also includes live beta.4, beta.6, and beta.8 evidence matching that path.

## Solution Assessment

Yes. Adding the existing `attempt_dispatch` phase to the private execution-start classifier, with a focused timer regression, is the narrowest maintainable fix and avoids changing scheduler policy, watchdog duration, runner emissions, retries, or model behavior.
