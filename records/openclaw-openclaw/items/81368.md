## Summary

Keep open: current main still does not treat `attempt_dispatch` as an execution-started cron milestone, and an open PR already references this issue with the focused fix and regression coverage.

## What This Changes

The issue reports that isolated cron runs in the 2026.5.12 beta series can reach `attempt_dispatch` and still be killed by the 60s pre-execution watchdog.

## Best Possible Solution

Land or otherwise resolve the focused PR by treating `attempt_dispatch` as an execution-started milestone, or by guaranteeing all affected runners emit an earlier already-classified milestone with regression coverage.

## Reproduction Assessment

Yes, source-reproducible with high confidence: current main defines and emits `attempt_dispatch`, but the watchdog classifier does not clear on that phase. The reporter and duplicate issue provide live beta evidence for the same 60s false-stall path.

## Solution Assessment

Yes: the linked PR is the narrowest maintainable fix because it broadens only the private cron execution-start classifier and adds a targeted regression test. The issue should stay open until that PR is reviewed, landed, or replaced.
