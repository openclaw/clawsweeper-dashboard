## Summary

Keep open. Current main still has the Control UI model-picker path that patches the session and refreshes session options without reloading chat history, and the only linked fix PR was closed unmerged.

## What This Changes

The issue reports that older Control UI assistant replies become blank after switching the active chat model from the header dropdown.

## Best Possible Solution

Preserve existing assistant message text across a chat-header model change and add a regression test with multiple prior assistant replies before changing the runtime path.

## Reproduction Assessment

No, not with high confidence. The issue has concrete user steps and current-main source still shows the suspicious model-switch and render paths, but this read-only review did not run a browser repro or capture the exact blanking message payload.

## Solution Assessment

Unclear whether the closed PR's direct loadChatHistory call is the best final fix. A repair should first add failing coverage for the blank-history case, then choose the narrowest guarded fix that preserves visible messages without introducing stale history reloads.
