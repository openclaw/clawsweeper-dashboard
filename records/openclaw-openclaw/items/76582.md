## Summary

Keep open. This is a current, concrete Control UI regression report, and an open unmerged PR explicitly targets it with closing syntax, so the issue should remain open until that implementation is reviewed and either merged or closed.

## What This Changes

Switching the chat header model in Control UI should preserve all existing assistant message content instead of blanking older replies.

## Best Possible Solution

Review PR #76583 or an equivalent narrow fix that preserves existing chatMessages/rendered text across model override changes and adds regression coverage for multiple assistant replies before and after a header model switch.

## Reproduction Assessment

No high-confidence live reproduction was run. The report has concrete steps and a v2026.5.2 screenshot, and source inspection shows the same model-switch and renderer surfaces are still present on current main, so the issue is source-reproducible but not fully reproduced here.

## Solution Assessment

Unclear whether an explicit `chat.history` reload after every model switch is the best repair. The safest solution is to first add a regression test for the blanking case, then preserve or rehydrate message content without changing the model-picker contract.
