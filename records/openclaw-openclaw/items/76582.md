## Summary

Keep open. The report describes a current Control UI history-rendering regression on v2026.5.2, and current main still has the model-picker path and rendering behavior that make this plausible without an implemented fix or duplicate canonical issue.

## What This Changes

Switching the Control UI chat header model should preserve already-rendered assistant messages instead of blanking older responses.

## Best Possible Solution

Fix the Control UI model-switch/render path so session model overrides only affect future sends, and add regression coverage for multiple existing assistant replies across a model-picker change.

## Reproduction Assessment

No live high-confidence reproduction was run. The issue provides concrete steps and a screenshot, and source inspection shows the current model-switch path triggers a session-options refresh while the renderer can suppress normalized-empty assistant messages.

## Solution Assessment

Unclear whether an explicit chat.history reload is the best fix. The safest repair should first preserve existing chatMessages and rendered text across the model override change, then add a regression test that would fail on the reported blank-message behavior.
