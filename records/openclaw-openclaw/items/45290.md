## Review Findings

Overall confidence: 0.88

Full review comments:

- **[P1] Keep deferred silent deliveries drainable:** `src/infra/outbound/deliver.ts:501`
  - body: This returns after enqueueing the silent delivery, before any send, ack, fail, retry, or live drain path. Because recovery drains pending entries rather than watching this newly deferred item, overlap with a user-visible delivery can leave the follow-up pending until an unrelated later drain or restart.
  - confidence: 0.9
- **[P2] Fail explicit channel errors before fallback:** `src/infra/outbound/message.ts:120`
  - body: The mirrored fallback is passed even when the caller supplied an explicit channel. A typo or unavailable explicit channel can therefore deliver through the mirrored session channel instead of failing fast, which can route content to an unintended channel.
  - confidence: 0.86
- **[P2] Resolve Argus lock stores from configured paths:** `src/agents/session-write-lock.ts:317`
  - body: This hardcodes the store path to sessions.json beside the session file, but session.store supports configured paths. Those setups miss Argus metadata, so stale-lock cleanup and the watchdog can reclaim locks that should be preserved.
  - confidence: 0.84
