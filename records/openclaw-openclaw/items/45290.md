## Summary

Keep open: this external PR is not closeable as implemented or superseded because useful pieces are still absent from current main, but the current branch has unresolved correctness findings, a concrete outbound routing security concern, and only test/commit-hook proof.

## What This Changes

The PR changes outbound delivery fallback/recovery, mirrored session channel selection, Argus session metadata/protocol handling, context-overflow detection, plain-text sanitization, and Telegram polling logs.

## Best Possible Solution

Split this into focused owner-reviewed patches: fail closed for explicit channel selection, add a real live drain/retry path before deferring silent deliveries, resolve Argus through the configured session store or defer that contract, and land sanitizer/overflow/logging fixes separately with targeted proof.

## Reproduction Assessment

Yes, by source inspection. The blockers can be exercised by sending with an invalid explicit channel plus mirrored delivery context, enqueueing a silent delivery while user-visible work is pending, and using a configured session.store outside dirname(sessionFile)/sessions.json.

## Solution Assessment

No. The branch targets real reliability gaps, but the current diff is too broad and introduces routing, queue-drain, and configured-store regressions; narrower patches are the safer maintainable path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Avoid stranding deferred silent deliveries:** `src/infra/outbound/deliver.ts:496-501`
  - body: This returns after enqueueing the silent delivery, before any send, ack, fail, or live retry path. Since recovery drains pending snapshots rather than continuously watching this queue, normal runtime overlap with a user-visible delivery can leave the follow-up pending until an unrelated later drain.
  - confidence: 0.9
- **[P2] Reject explicit channel failures before fallback:** `src/infra/outbound/message.ts:113-120`
  - body: Passing the mirrored fallback into channel selection also applies it when params.channel is explicit. A typo or unavailable explicit channel can therefore deliver through the mirrored session channel instead of failing fast, which can route content to an unintended channel.
  - confidence: 0.86
- **[P2] Use the configured session store for Argus checks:** `src/agents/session-write-lock.ts:316-317`
  - body: resolveSessionStorePathForLock always reads sessions.json beside the session file, but session.store supports configured paths and templates. In those setups Argus metadata is missed, so stale-lock cleanup and the watchdog can still reclaim locks the new protection is meant to preserve.
  - confidence: 0.84

## Security Review

Status: needs_attention
