## Summary

Keep open. Current main does not fully supersede the PR's useful sanitizer/input_length/fallback/logging work, but the branch is not mergeable as-is because source inspection confirms delivery, channel fallback, and session-store regressions plus missing real behavior proof.

## What This Changes

The PR changes outbound fallback selection, delivery queue recovery, Argus session metadata, context-overflow detection, plain-text sanitization, Telegram polling logs, and related tests.

## Best Possible Solution

Split this into focused owner-reviewed patches that fail explicit channels closed, keep deferred deliveries drainable, resolve session state through configured stores, and land sanitizer/input_length/logging fixes separately with real proof.

## Reproduction Assessment

Yes, for the PR-introduced blockers by source inspection: invalid explicit channel plus mirrored context, silent delivery overlap with a pending user-visible queue item, and configured session.store outside dirname(sessionFile)/sessions.json exercise the risky paths.

## Solution Assessment

No. The patch targets real reliability gaps, but it mixes broad routing, protocol, queue, Argus, sanitizer, and Telegram changes while introducing regressions; narrower fail-closed patches with runtime proof are the maintainable path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Keep deferred silent deliveries drainable:** `src/infra/outbound/deliver.ts:496-501`
  - body: This returns after enqueueing the silent delivery, before any send, ack, fail, or retry path. Since recovery drains pending entries rather than watching this newly deferred item, overlap with a user-visible delivery can leave the follow-up pending until an unrelated later drain.
  - confidence: 0.9
- **[P2] Fail explicit channel errors before falling back:** `src/infra/outbound/message.ts:113-120`
  - body: The mirrored fallback is passed even when the caller supplied an explicit channel. A typo or unavailable explicit channel can therefore deliver through the mirrored session channel instead of failing fast, which can route content to an unintended channel.
  - confidence: 0.86
- **[P2] Resolve Argus locks from configured session stores:** `src/agents/session-write-lock.ts:316-317`
  - body: This hardcodes the store path to sessions.json beside the session file, but session.store supports configured paths. Those setups miss Argus metadata, so stale-lock cleanup and the watchdog can reclaim locks the new protection is meant to preserve.
  - confidence: 0.84

## Security Review

Status: needs_attention

Summary: The diff introduces a concrete misdelivery risk by allowing explicit outbound sends to fall back to mirrored session channels.

  - body: The proposed sendMessage fallback applies even when the caller supplied an explicit channel. If that channel is unavailable or malformed, message content can be routed through the mirrored session channel instead of failing, exposing content on an unintended channel or recipient path.
