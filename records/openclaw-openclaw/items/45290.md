Full review comments:

- **[P1] Keep deferred silent deliveries drainable:** `src/infra/outbound/deliver.ts:501`
  - body: This returns after enqueueing the silent delivery, before any send, ack, fail, retry, or live drain path.   - confidence: 0.9
- **[P2] Fail explicit channel errors before fallback:** `src/infra/outbound/message.ts:120`
  - body: The mirrored fallback is passed even when the caller supplied an explicit channel. A typo or unavailable explicit channel can therefore deliver through the mirrored session channel instead of failing fast, which can route content to an unintended channel.
  - confidence: 0.86

  - body: This hardcodes the store path to sessions.json beside the session file, but session.store supports configured paths. Those setups miss Argus metadata, so stale-lock cleanup and the watchdog can reclaim locks the new protection is meant to preserve.
  - confidence: 0.84
