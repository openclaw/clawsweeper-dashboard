## Summary

Keep open: current main and v2026.6.10 refute the narrow regex diagnosis because the shared timeout matcher already covers `operation was aborted`, but the reported custom-provider fallback failure is not proven fixed without logs showing whether the abort came from the provider timeout or an outer caller/run abort boundary.

## What This Changes

The issue reports that a provider transport `AbortError("This operation was aborted")` is treated as cancellation, so configured model fallbacks are not attempted after the primary model times out.

## Best Possible Solution

Keep the issue open until logs identify the abort owner, then route provider-originated transport timeouts through existing failover while preserving immediate stop behavior for caller/client/run aborts.

## Reproduction Assessment

## Solution Assessment
