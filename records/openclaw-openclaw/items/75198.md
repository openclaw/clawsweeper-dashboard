## What This Changes

## Best Possible Solution

Keep the server-side shared resolver direction, but preserve valid explicit provider/model refs first, use provider-qualified alias resolution only as a fallback for invalid canonical refs, remove the changelog edit, rebase, and add redacted real behavior proof.

## Reproduction Assessment

Yes, from source and issue evidence: current `sessions.patch` sends the picker value into `resolveAllowedModelRef`, while current `resolveModelRefFromString` only resolves whole-string aliases before parsing direct refs. ## Solution Assessment

No, not as submitted: shared server-side canonicalization is the right layer, but the alias-first precedence is not the safest fix. The narrow repair is to preserve exact allowed provider/model refs, then fall back to provider-qualified alias resolution for otherwise-invalid selections.
