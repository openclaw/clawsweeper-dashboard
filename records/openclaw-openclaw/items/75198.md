## What This Changes

The PR adds provider-qualified alias resolution for session model patches, adds resolver and gateway regression tests, and edits CHANGELOG.md.

## Best Possible Solution

Keep the shared resolver as the fix layer, but preserve exact provider/model refs first, use provider-scoped alias resolution only as a fallback for non-canonical display aliases, remove the changelog entry, rebase, and attach redacted real behavior proof before merge.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: Control UI sends the selected model value to sessions.patch, and current main resolves only exact bare aliases before parsing provider/model, so provider/Alias falls through as a raw model id.

## Solution Assessment

No. The shared resolver is the right layer, but this implementation should preserve exact provider/model refs before provider-scoped alias fallback and then prove the real Control UI/TUI workflow.
