## Best Possible Solution

Keep the PR open, make provider-qualified alias resolution exact-ref-first with collision coverage, remove `CHANGELOG.md`, and require redacted real Control UI/TUI or live `sessions.patch` proof before merge.

## Reproduction Assessment

Yes, source-reproducible: current main only exact-matches the full raw model string against aliases before parsing `provider/model`, so `lmstudio-moe/MoE` is parsed as model `MoE` instead of alias `MoE`. The linked issue provides matching LM Studio rejection logs.

## Solution Assessment

No, not as written. The right fix should resolve `provider/Alias` only after preserving valid direct provider/model refs, rather than adding alias precedence globally before parsing.
