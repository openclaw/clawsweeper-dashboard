## What This Changes

The issue reports that the memory-wiki `wiki_lint` agent tool can fail during tool result handling because it returns the raw lint result while the CLI lint path succeeds.

## Best Possible Solution

Keep lint core, CLI, and gateway contracts intact, and make only `createWikiLintTool` return relative, bounded, JSON-safe details with focused regression coverage and real direct-tool smoke proof.

## Reproduction Assessment

No executed high-confidence reproduction was established in this read-only pass. Source inspection plus the reporter's live validation show current main still has the raw absolute-path `wiki_lint` result shape that matches the reported failure surface.

## Solution Assessment

Yes. A wrapper-only fix that normalizes `reportPath` and bounds `details` in `createWikiLintTool` is the narrow maintainable solution; changing lint core or CLI behavior would broaden the blast radius unnecessarily.
