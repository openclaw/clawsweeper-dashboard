## What This Changes

The issue asks for `wiki_lint` to return path-safe, bounded tool output instead of raw absolute-path-heavy lint details when CLI lint succeeds.

## Best Possible Solution

Land one reviewed wrapper-only fix from https://github.com/openclaw/openclaw/pull/83439 or https://github.com/openclaw/openclaw/pull/83464, keep lint core and CLI behavior unchanged, then close this issue via the merged PR.

## Solution Assessment

Yes, the wrapper-only direction is the best fit. Normalize and bound the `wiki_lint` tool payload in `createWikiLintTool` while preserving `lintMemoryWikiVault` and CLI/file caller contracts.
