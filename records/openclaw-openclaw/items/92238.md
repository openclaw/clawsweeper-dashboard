## Summary

Keep open: current main still has a source-backed no-shell path for direct sandbox-off OpenAI/Codex turns, and the linked fix at https://github.com/openclaw/openclaw/pull/92294 is open rather than merged.

## What This Changes

OpenAI/Codex-backed isolated cron runs with sandbox off should retain an executable shell tool when the effective policy allows `exec`.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/92294 or an equivalent targeted fix, then close this issue after the fix reaches `main`.

## Reproduction Assessment

Yes, at source level. Current main filters `exec` and `process`, direct sandbox-off Codex turns do not register a Codex environment, and Codex `rust-v0.139.0` omits native shell tools without an environment.

## Solution Assessment

Yes. Restoring the existing OpenClaw shell tools only when Codex has no environment is the narrowest maintainable direction because it preserves sandbox-backed, node-host, memory-flush, and explicit-exclusion behavior.
