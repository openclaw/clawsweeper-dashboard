## Summary

Keep open. The PR addresses a real current-main Linux gateway/systemd gap, but the proposed env synthesis still forwards merged service-unit environment values into `systemctl` child processes, which is a concrete secret-exposure regression and blocks merge as-is.

## What This Changes

The PR updates daemon systemd helpers to synthesize missing Linux user-bus env for direct `systemctl --user` calls, prefer machine scope for root-effective non-root targets, narrow systemd-unavailable classification, and add focused tests plus a changelog entry.

## Best Possible Solution

Land a repaired and rebased implementation that keeps the direct user-bus recovery, target-user scope handling, classifier fix, tests, and changelog, but constructs the child env from inherited process env plus only the required `XDG_RUNTIME_DIR` and `DBUS_SESSION_BUS_ADDRESS` overrides, with a regression proving service-unit secrets are not forwarded.

## Reproduction Assessment

Yes. #63561 gives concrete Ubuntu status/install steps for the user-visible bug, and the PR finding is reproducible by following `readGatewayServiceState` merging parsed service env into systemd probes and then the PR's `resolveSystemctlUserBusEnv` spreading that env into `systemctl`.

## Solution Assessment

No, not as-is. The direction is the narrow maintainable fix for the missing user-bus env path, but the implementation should avoid carrying service-unit secrets into `systemctl` and the branch needs maintainer handling because it is currently dirty.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Avoid forwarding service secrets to systemctl:** `src/daemon/systemd.ts:395-403`
  - body: `resolveSystemctlUserBusEnv` builds the child environment from `{ ...process.env, ...env }`. Status paths pass an env merged with parsed service `Environment`/`EnvironmentFile` values, including gateway tokens and other credentials, so when bus vars are missing this forwards service-only secrets to `systemctl`. Build the child env from inherited process env plus only the DBus/XDG overrides needed for the probe.
  - confidence: 0.88
