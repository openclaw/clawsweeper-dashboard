## Summary

Keep open. The PR is aimed at a real current-main gateway/systemd gap, but it is not merge-ready because the synthesized `systemctl` child environment can carry parsed service-unit secrets into unrelated `systemctl` probes.

## What This Changes

The PR adds Linux user-bus environment synthesis for direct `systemctl --user` gateway status/install probes, preserves non-root target scope under root wrappers, narrows unavailable classification, adds regressions, and updates the changelog.

## Best Possible Solution

Keep the direct user-bus recovery and classifier direction, but construct the synthesized child environment from the inherited process env plus only `XDG_RUNTIME_DIR` and `DBUS_SESSION_BUS_ADDRESS`, with a regression proving parsed service-unit secrets are not forwarded.

## Reproduction Assessment

Yes. #63561 gives concrete Ubuntu status/install steps, and current-main source inspection shows direct `systemctl --user` calls have no env recovery; the PR-specific security regression can be reproduced with a service unit EnvironmentFile token plus missing bus vars.

## Solution Assessment

No, not as-is. The requested fix is appropriately narrow, but the implementation should not pass merged service command env to `systemctl`; limiting the child env to required DBus/XDG values is the safer maintainable path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Avoid forwarding service secrets to systemctl:** `src/daemon/systemd.ts:394-402`
  - body: `resolveSystemctlUserBusEnv` builds the child env from `{ ...process.env, ...env }`. Status paths pass an env merged with parsed service `Environment`/`EnvironmentFile` values, including gateway tokens and other credentials, so when bus vars are missing this forwards service-only secrets to every `systemctl` child. Build the child env from the inherited process env plus only the DBus/XDG overrides needed for the probe.
  - confidence: 0.88

## Security Review

Status: needs_attention

Summary: The patch introduces a concrete local secret-exposure risk by passing merged service-unit environment values to `systemctl` when synthesizing user-bus env.

Concerns:

- **[medium] Service secrets can be forwarded to systemctl:** `src/daemon/systemd.ts:394`
  - body: `readGatewayServiceState` merges parsed service env into systemd probes, and the PR then spreads that env into the child process environment. Service unit `EnvironmentFile` values can include gateway tokens or provider credentials, so the fix should only add the DBus/XDG variables instead of forwarding all service env.
  - confidence: 0.86
