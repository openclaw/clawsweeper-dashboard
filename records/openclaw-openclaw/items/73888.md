## Summary

Keep open. The branch addresses a real gateway/systemd bug from https://github.com/openclaw/openclaw/issues/63561, but the current head should not merge because its env recovery can forward parsed service credentials into `systemctl` child processes.

## What This Changes

The PR changes Linux gateway systemd probes to synthesize missing user DBus/XDG environment for `systemctl --user`, adjusts root/non-root scope handling, narrows unavailable classification, and adds regressions plus a changelog entry.

## Best Possible Solution

Keep the DBus/XDG recovery direction, but construct the `systemctl` child env from inherited process env plus only the required `XDG_RUNTIME_DIR` and `DBUS_SESSION_BUS_ADDRESS` overrides, with a regression proving service secrets are not forwarded.

## Reproduction Assessment

Yes, source-level. The linked issue gives concrete Ubuntu status/install logs, current main still has no env override for direct `systemctl --user`, and the PR-specific secret-forwarding bug follows from `{ ...process.env, ...env }` plus current service-env merging.

## Solution Assessment

No, not as-is. The env-recovery approach is the narrow maintainable fix, but this implementation should not pass parsed service `Environment` or `EnvironmentFile` values to `systemctl` children.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Avoid forwarding service secrets to systemctl:** `src/daemon/systemd.ts:394`
  - body: `resolveSystemctlUserBusEnv` builds the child env from `{ ...process.env, ...env }`. Status paths pass an env merged with parsed service `Environment`/`EnvironmentFile` values, including gateway tokens and provider credentials, so when bus vars are missing this forwards service-only secrets to `systemctl`. Build the child env from inherited process env plus only the DBus/XDG overrides needed for the probe.
  - confidence: 0.88

## Security Review

Status: needs_attention

Summary: The PR introduces a concrete local secret-exposure risk by forwarding merged service-unit environment values to `systemctl`.

Concerns:

- **[medium] Service secrets can be forwarded to systemctl:** `src/daemon/systemd.ts:394`

Service unit `EnvironmentFile` values can include gateway tokens or provider credentials, so the fix should only add DBus/XDG variables instead of forwarding all service env.
