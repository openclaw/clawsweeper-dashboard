## Summary

Keep open: the linked gateway/systemd bug is real and the PR direction is useful, but the patch forwards merged service-unit environment values to a `systemctl` child process, which can expose service secrets locally. The child env boundary needs repair before merge.

## What This Changes

The PR adds Linux `systemctl --user` DBus/XDG env recovery for gateway status/install, adjusts root target-user handling and systemd-unavailable classification, and adds daemon tests plus a changelog entry.

## Best Possible Solution

Keep the DBus/XDG recovery and target-user-scope behavior, but build the `systemctl` child env from inherited process env plus only the required `XDG_RUNTIME_DIR` and `DBUS_SESSION_BUS_ADDRESS` overrides, with a regression proving service secrets are not forwarded.

## Reproduction Assessment

Yes, source-level. The linked issue at https://github.com/openclaw/openclaw/issues/63561 provides concrete Ubuntu status/install logs, current main still has no env override for direct `systemctl --user`, and the PR-specific bug follows from `{ ...process.env, ...env }` after service-env merging.

## Solution Assessment

No, not as-is. The env-recovery approach is the narrow maintainable fix, but this implementation should not pass parsed service `Environment=` or `EnvironmentFile=` values to `systemctl` children.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Avoid forwarding service secrets to systemctl:** `src/daemon/systemd.ts:394`
  - body: `resolveSystemctlUserBusEnv` builds the child env from `{ ...process.env, ...env }`. Status reads the installed unit and merges parsed `Environment=`/`EnvironmentFile=` values into `env` before calling systemd probes; those values can include gateway tokens or provider credentials, so missing bus env causes service-only secrets to be forwarded to the local `systemctl` child. Build the child env from inherited process env plus only the DBus/XDG overrides needed for the probe.
  - confidence: 0.88

## Security Review

Service unit `EnvironmentFile` values can include gateway tokens or provider credentials, so the fix should only add DBus/XDG variables instead of forwarding all service env.
