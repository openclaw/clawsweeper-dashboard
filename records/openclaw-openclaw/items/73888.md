## Summary

Keep open: current main still lacks the user DBus/XDG recovery this PR is trying to add, but the proposed patch should not merge as-is because it forwards parsed service environment values to `systemctl` when recovering the bus env.

## What This Changes

The PR synthesizes missing Linux `XDG_RUNTIME_DIR` and `DBUS_SESSION_BUS_ADDRESS` for gateway `systemctl --user` probes, adjusts root/non-root user-scope handling, narrows systemd-unavailable classification, and adds daemon tests plus a changelog entry.

## Best Possible Solution

Keep the DBus/XDG recovery direction, but build the `systemctl` child env from inherited process env plus only the required `XDG_RUNTIME_DIR` and `DBUS_SESSION_BUS_ADDRESS` overrides, with a regression proving service secrets are not forwarded.

## Reproduction Assessment

Yes, source-level. The linked issue provides concrete Ubuntu status/install logs, current main still has no env override for direct `systemctl --user`, and the PR-specific bug follows from `{ ...process.env, ...env }` plus current service-env merging.

## Solution Assessment

No, not as-is. The env-recovery approach is the narrow maintainable fix, but this implementation should not pass parsed service `Environment=` or `EnvironmentFile=` values to `systemctl` children.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Avoid forwarding service secrets to systemctl:** `src/daemon/systemd.ts:394`
  - body: `resolveSystemctlUserBusEnv` builds the child env from `{ ...process.env, ...env }`. Status reads the installed unit and merges parsed `Environment=`/`EnvironmentFile=` values into `env` before calling systemd probes; those values can include gateway tokens or provider credentials, so missing bus env causes service-only secrets to be forwarded to the local `systemctl` child. Build the child env from inherited process env plus only the DBus/XDG overrides needed for the probe.
  - confidence: 0.88

## Security Review

Status: needs_attention

Summary: The patch introduces a concrete local secret-exposure risk by forwarding merged service-unit environment values to `systemctl`.

Concerns:

- **[medium] Service secrets can be forwarded to systemctl:** `src/daemon/systemd.ts:394`
  - body: `readGatewayServiceState` merges parsed service env into systemd probes, and the PR then spreads that env into the child process environment. Service unit `EnvironmentFile` values can include gateway tokens or provider credentials, so the fix should only add DBus/XDG variables instead of forwarding all service env.
  - confidence: 0.86
