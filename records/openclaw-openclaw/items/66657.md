## What This Changes

Handle Feishu messages whose runtime `message.content` is missing or empty so mention-only events do not crash before dispatch.

## Best Possible Solution

Land a narrow Feishu plugin fix that normalizes missing inbound content at the parser or receive boundary, preserves mention gating and empty-message skipping, and adds regression coverage for missing-content mention events.

Yes, at source level. Construct a Feishu text event with missing `message.content` and mention metadata, then exercise `getFeishuSequentialKey()` or the receive/debounce path; current `main` lacks a safe string-content boundary before string methods or payload rejection.
