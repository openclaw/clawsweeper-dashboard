## Summary

Keep open. Current `main` still has unsafe Feishu empty-content boundaries and rejects missing message content before dispatch, while the target-repo Feishu fix PRs were closed unmerged and the only exact open implementation reference is external.

## What This Changes

Handle Feishu group messages that contain only a bot mention or otherwise have missing/undefined content without throwing on `.trim()`.

## Best Possible Solution

Normalize Feishu inbound message content to a string at the parser or receive-handler boundary, preserve current group mention gating and sequential/debounce behavior, and add regression coverage for mention-only and missing-content events.

## Solution Assessment

Yes, a narrow Feishu plugin repair is the best path. Optional chaining only at the two reported trim sites is incomplete because parser, mention normalization, debounce, and final message handling all need a consistent string-content boundary.
