## Summary

Keep open. Current main still has Feishu missing-content paths that can drop or crash affected messages, and the linked fix PR is open rather than merged closure evidence.

## What This Changes

The issue reports a Feishu dispatch failure when group messages with only an @mention or missing text content leave `message.content` undefined and downstream Feishu code calls string methods on it.

## Best Possible Solution

Update and land the linked Feishu fix, or a narrow equivalent, so missing content is accepted or normalized at the receive/parser boundary, remaining string-method sites are guarded, and regression coverage proves the missing-content event path.

## Reproduction Assessment

Yes, at source level: current main either rejects missing-content receive payloads before dispatch or lets typed Feishu events reach parser/sequential-key string methods without a missing-content guard. I did not run a live Feishu tenant replay in this read-only review.
