## Summary

Keep this PR open: it correctly moves several returned exec/process/node-host tool-result surfaces toward source-boundary redaction, but the live exec update emitter still builds update text and details from raw session output, and the final head lacks real behavior proof for the later background-exec/live-update gap.

## What This Changes

The PR redacts exec, process, and node-host command output in returned tool-result text/details and adds focused regression coverage for those redaction paths.

## Best Possible Solution

Apply the same tool-payload redaction at every exec/process returned-result and live-update emission boundary, keep raw session buffers only for internal bookkeeping, and refresh final-head real behavior proof with a fake canary.

## Reproduction Assessment

Yes, at source level. Current main returns or emits stdout/stderr-derived `aggregated` and `tail` strings through foreground exec, background process, node-host, and live-update paths, and the linked security issue includes a fake-canary confirmation on OpenClaw 2026.5.7.

## Solution Assessment

No, not yet. The PR uses the right redaction helpers for several returned-result paths, but the maintainable fix also needs the exec live-update boundary covered and proven on the final head.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Redact live exec update payloads:** `src/agents/bash-tools.exec-runtime.ts:681-691`
  - body: The returned-result redaction does not cover live updates: `emitUpdate()` still renders `tailText` from raw session output and sends `details.tail: session.tail` through `opts.onUpdate`.
