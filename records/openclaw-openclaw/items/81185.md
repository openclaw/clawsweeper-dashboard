## Summary

Keep this PR open: it is a useful security fix with sufficient real-behavior proof for the covered surfaces, but it still misses the running/background exec tail path, so secret-shaped command output can still leave the exec tool result boundary.

## What This Changes

The PR redacts foreground exec, node-host exec, and process list/poll/log returned text/details, and adds focused redaction regression tests.

## Best Possible Solution

Use the existing result-boundary redaction helpers at every exec/process emission point that can expose stdout, stderr, aggregated output, update text, or session tails, while preserving raw registry buffers for process bookkeeping.

## Reproduction Assessment

Yes, at source level: current main returns raw exec/process output through foreground exec, process poll/log/list, running exec `details.tail`, and live exec update tails. A command that prints an API-key-shaped value before yielding or backgrounding can still expose that raw value.

## Solution Assessment

No. The PR uses the right redaction helpers and covers useful emitters, but it is incomplete until running/background exec tails and the live update tail boundary are redacted with regression coverage.
