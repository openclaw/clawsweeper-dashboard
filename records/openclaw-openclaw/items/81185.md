## Summary

Keep open: the follow-up redacts the returned running-session details tail, but PR head still leaves live exec update content/details built from raw session output, so the security fix is incomplete.

## What This Changes

The PR redacts exec, process, and node-host tool-result output surfaces that can contain secret-shaped stdout, stderr, errors, aggregated output, command text, or session tails.

## Best Possible Solution

Redact exec live update text and structured update details at the `emitUpdate()` boundary, keep raw session buffers for process bookkeeping, and add focused regression coverage for update payloads.

## Reproduction Assessment

Yes, at source level: current main and PR head still pass `session.tail || session.aggregated` and `details.tail` into live exec updates without local redaction. A command that prints an API-key-shaped value before an update fires exercises the remaining leak path.

## Solution Assessment

No. The PR uses the right helper for foreground, process, node-host, and returned background result paths, but it is not complete until live exec update content and details are redacted before `opts.onUpdate`.
