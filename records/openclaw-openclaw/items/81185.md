## Summary

Keep open: this is a useful security fix with sufficient proof, but the patch still leaves running exec session tails and live exec update tails unredacted, so secret-shaped output can still leak after an exec yields or backgrounds.

## What This Changes

The PR redacts exec foreground, node-host exec, and process list/poll/log tool-result text/details, and adds focused redaction regression tests.

## Best Possible Solution

Use the existing result-boundary redaction helpers at every exec/process emission point that can expose stdout, stderr, aggregated output, update text, or session tails, while preserving raw registry buffers for process bookkeeping.

## Reproduction Assessment

Yes, at source level: current main returns raw exec/process output through foreground exec, process poll/log/list, running exec `details.tail`, and live exec update tails. A command that prints an API-key-shaped value before yielding or backgrounding exercises the remaining leak path.

## Solution Assessment

No. The PR uses the right redaction helpers for the covered emitters, but it is incomplete until running/background exec tails and live update tail payloads are redacted with focused coverage.
