## What This Changes

Request configurable retry limits for provider model API calls so fallback chains can advance without waiting through provider SDK retry loops.

## Best Possible Solution

Choose one supported model-provider retry contract, then land a rebased implementation with schema, docs, focused tests, and real configured-provider proof while preserving current transport security, timeout, Retry-After, and failover behavior.
