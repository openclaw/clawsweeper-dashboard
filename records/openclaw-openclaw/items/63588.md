## Summary

Keep open: source inspection shows the Slack CDN redirect auth bug is still present on current main, and this PR targets the right user-visible failure, but the PR head still has merge-blocking test import and Slack media host-boundary issues plus no real Slack proof.

## What This Changes

The PR rewrites Slack media fetching to reattach bearer Authorization on Slack-owned redirect targets and updates Slack media regression tests.

## Best Possible Solution

Land a rebased or replacement Slack plugin fix that keeps Authorization only on Slack-owned HTTPS redirect hosts, preserves the hostnameAllowlist SSRF boundary and dispatcher path, fixes the tests, and includes redacted real Slack proof.

## Reproduction Assessment

Yes, at source level. Current main's fetchWithSlackAuth only keeps Authorization for same-origin redirects, and the current test fixture expects no Authorization on a cdn.slack-edge.com redirect that matches the reported failure path.

## Solution Assessment

No, not as-is. The direction is the narrow maintainable fix, but this PR needs a resolvable test import, a Slack-only redirect hostname boundary, a rebase, and real Slack proof before merge.
