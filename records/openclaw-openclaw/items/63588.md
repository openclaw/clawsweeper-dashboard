## Summary

Keep this PR open. The reported Slack download bug is still source-reproducible on current main and the PR is aimed at the right behavior, but the branch is not mergeable as-is because it has a broken test import, widens the Slack media redirect boundary, and still lacks real Slack proof.

## What This Changes

The PR changes Slack media download redirect handling and tests so Authorization is kept on Slack-owned CDN redirects and removed elsewhere.

## Best Possible Solution

Land a rebased or replacement Slack plugin fix that keeps Authorization only on Slack-owned HTTPS redirects, preserves the current Slack media SSRF policy and dispatcher path, updates focused tests, and includes redacted real Slack proof.

## Reproduction Assessment

Yes, at source level. Current main only keeps Authorization for same-origin redirects, while the current Slack media test fixture expects no Authorization on a cdn.slack-edge.com redirect that matches the reported failure path.

## Solution Assessment

No, not as-is. The direction is the narrow maintainable fix, but the branch needs a resolvable test import, a restored Slack-only redirect hostname boundary, a rebase, and real Slack proof before merge.
