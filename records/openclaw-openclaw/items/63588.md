## Summary

Keep open: the PR addresses a real Slack `download-file` bug that still exists on current main for Slack CDN cross-origin redirects, but the branch is not merge-ready because it has a broken test import, weakens the Slack redirect host boundary, and lacks contributor real-behavior proof.

The PR changes Slack media downloads to keep bearer auth on Slack-owned HTTPS redirect hosts, route auth fetches through SSRF guarding, strip auth on non-Slack redirects, and add redirect regression tests.

## Best Possible Solution

Land a rebased Slack plugin patch that retains Authorization across Slack-owned HTTPS redirect hosts, keeps non-Slack redirects outside the Slack media host boundary or strips credentials before a clearly approved fallback, preserves SSRF/runtime safeguards, and adds focused regression coverage.

## Solution Assessment
