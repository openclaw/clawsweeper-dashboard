## Summary

Keep open: current `main` and the latest release still exclude native OpenAI from the cache-TTL eligibility helper, and https://github.com/openclaw/openclaw/pull/95859 is an open closing PR rather than a merged fix. ## What This Changes

The issue reports that `contextPruning` with `mode: "cache-ttl"` never becomes eligible for native OpenAI providers, preventing idle-gap stale tool-result pruning for OpenAI agents.

## Best Possible Solution

Review and either land https://github.com/openclaw/openclaw/pull/95859 with real idle-gap OpenAI proof and agreed retention semantics, or replace it with a provider-agnostic pruning design if maintainers decide cache-TTL eligibility is the wrong abstraction.
