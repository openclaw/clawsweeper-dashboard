## Summary

Keep open. Current main and the latest release still expose browser SSRF as strict-by-default plus boolean private-network opt-in and hostname exceptions, but they do not implement the requested confirmation mode, temporary grant command, CIDR allow/block lists, logging contract, or non-overridable metadata policy; adjacent SSRF issues and PRs cover different egress surfaces rather than this browser approval design.

## What This Changes

The issue requests a browser SSRF policy mode that prompts for private-network access and supports allow/block lists, hard metadata blocking, logging, and temporary CLI enablement.

## Best Possible Solution

Define a maintainer-approved browser-plugin SSRF policy that keeps strict defaults, adds explicit per-host or per-session approval semantics, specifies metadata/link-local hard-block behavior, and updates schema, docs, and navigation/redirect tests together.
