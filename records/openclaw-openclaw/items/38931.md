## Summary

Keep open. Current main still has static browser SSRF allow/block behavior with hostname exceptions; it does not implement a per-request confirmation mode, CIDR/port allow/block policy, non-overridable metadata blocking under private-network opt-in, approval UI, audit/rate-limit/timeout flow, or temporary CLI controls.

## What This Changes

The issue requests a browser SSRF private-network policy that can ask the user for approval, support allow/block lists, hard-block metadata endpoints, and optionally expose temporary CLI controls.

## Best Possible Solution

Keep this as the canonical browser private-network confirmation/security design request, then implement an explicit policy contract that preserves fail-closed SSRF behavior while supporting operator-approved private-network access with documented precedence and tests for redirects, DNS rebinding, metadata endpoints, and browser/CDP edge cases.
