## Summary

Keep open: current main and v2026.6.10 still expose static boolean plus hostname-based browser SSRF controls, with no confirmation mode, temporary grant command, CIDR allow/block policy, or approval state. The remaining work changes a browser private-network security boundary and needs maintainer product/security review rather than conservative cleanup closure.

## What This Changes

Request adding browser SSRF private-network confirmation, allow/block lists, and temporary CLI grants so internal destinations can be approved without globally enabling private-network access.

## Best Possible Solution

Define a maintainer-approved browser SSRF policy that keeps strict defaults, specifies scoped approval and exception behavior, decides non-overridable metadata/link-local handling, and updates schema, docs, audit, and navigation tests together.
