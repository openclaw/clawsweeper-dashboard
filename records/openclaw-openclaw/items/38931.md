## Summary

Keep open: current main still has only static strict-or-broad browser SSRF behavior, while this asks for a new confirmation/allowlist/blocklist policy with security-sensitive UX and configuration semantics.

## What This Changes

Request a browser SSRF policy that can confirm private-network access per request, support allow/block lists and cloud metadata blocking, and optionally expose temporary CLI controls.

## Best Possible Solution

Keep this as the canonical browser private-network confirmation design request until maintainers approve a typed SSRF policy contract and matching browser/UI/CLI/docs/tests behavior.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/security design request. Source inspection on current main shows the requested confirm/blocklist fields are absent and browser navigation guards return, throw, continue, or abort rather than waiting for approval.

## Solution Assessment

Unclear whether the exact proposed JSON and CLI shape is the best final API. The narrow maintainable path is a maintainer-approved security design covering policy semantics, approval delivery, audit behavior, and regression tests before implementation.
