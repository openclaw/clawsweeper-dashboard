Keep open: current main and the latest release still only provide static browser SSRF allow/deny plus hostname exceptions, while the requested confirmation, timeout, allow/block-list, audit, and temporary-access semantics are missing and security-sensitive.

## Best Possible Solution

Define a typed browser SSRF policy contract with reviewed confirmation semantics, hard-deny precedence, approval routing, logging, docs, config schema/help updates, and regression coverage in the shared SSRF and browser navigation layers.
