## Summary

Keep open: current main and v2026.5.20 still expose browser SSRF as strict/default blocking plus boolean private-network opt-in and hostname exceptions, while confirmation, blocklist precedence, audit, and temporary access modes are new security-sensitive product/config work.

## What This Changes

Request adds a browser SSRF mode for private/internal network access that can ask for user approval, allow or block specific targets, and temporarily enable access instead of the current boolean allow/deny policy.

## Best Possible Solution

Define a reviewed typed browser SSRF policy contract with hard-deny precedence for metadata/link-local targets, explicit approval routing, timeout/rate-limit behavior, audit semantics, config schema/help/docs updates, and regression coverage across shared SSRF and browser navigation layers.
