## Summary

Keep open. Current main still sends OpenShell exec text into the shared SSH `/bin/sh -c` wrapper without rejecting generated placeholders or malformed shell syntax, and the generic repeated-call detector still returns warning-level results until the separate global no-progress breaker. Related #58726 and open loop-detection PRs overlap the hard-abort half, but they do not cover the OpenShell command-preflight gap.

## What This Changes

The issue requests OpenShell fail-fast validation for malformed/generated shell commands and hard-abort behavior for repeated identical tool calls.

## Best Possible Solution

Add a focused remote-shell exec preflight that rejects the reported generated placeholders and malformed shell constructs before SSH `/bin/sh -c` invocation, with targeted OpenShell/SSH tests. Resolve the repeated-call hard-abort through the existing loop-detection tracking so one threshold policy ships consistently.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: `workflow install <name>` or an unterminated quote flows from OpenShell `prepareExec` into `buildExecRemoteCommand` and then `/bin/sh -c`; current tests also explicitly assert generic repeated calls remain warn-only below the global breaker.

## Solution Assessment

Yes. The narrow maintainable fix is command preflight at the OpenShell/shared remote-shell boundary plus coordinated loop-detector escalation, rather than a broad command-obfuscation filter or a duplicate loop-policy PR.

## Repair Work Prompt

Fix the OpenShell malformed generated-command fail-fast half of openclaw/openclaw#72373. Add a focused preflight before OpenShell/SSH sandbox remote shell invocation, preferably in `src/agents/sandbox/ssh.ts` if the policy can stay generic and plugin-SDK compatible, otherwise at the OpenShell `prepareExec` boundary in `extensions/openshell/src/backend.ts`. Reject clearly non-runnable generated command text before SSH/session exec: literal placeholder tokens such as `<name>` or `<workflow-id>` in executable positions/args, unclosed single/double quotes, trailing backslash escapes, unterminated backticks, and unterminated command substitutions.
