## Summary

Keep open. Current main still forwards OpenShell exec text to the SSH `/bin/sh -c` wrapper without malformed-command preflight, while the repeated-tool hard-abort portion overlaps existing open loop-detection work.

## What This Changes

Request OpenShell preflight validation for malformed/generated commands and hard-abort behavior for repeated identical failed tool loops.

## Best Possible Solution

Add narrow OpenShell/SSH exec-command preflight before remote shell invocation, with regression coverage for generated placeholders and malformed shell syntax, and leave generic repeated-loop hard-abort policy to the existing loop-detection cluster.

## Reproduction Assessment

Yes for source reproduction: a command such as `workflow install <name>` or an unterminated quote flows from OpenShell `prepareExec` into `buildExecRemoteCommand` and then `/bin/sh -c` without validation. I did not run a live OpenShell session because this review is read-only.

## Solution Assessment

Yes, with a split boundary. The narrow maintainable fix is OpenShell/SSH exec preflight; generic repeated-tool hard-abort should be resolved through #58726, #56403, and #60248.
