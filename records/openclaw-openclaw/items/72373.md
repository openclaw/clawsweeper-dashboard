## Summary

Keep open. Current main now covers the generic repeated no-progress loop-abort half, but the OpenShell/SSH exec path still forwards malformed or placeholder command text into the remote shell wrapper without a preflight guard.

## What This Changes

OpenShell should reject malformed generated shell commands before execution, and repeated identical failed tool loops should hard-abort instead of warning indefinitely.

## Best Possible Solution

Add a narrow OpenShell/SSH exec preflight that rejects obviously generated placeholders and malformed shell syntax before remote shell invocation, while leaving broader loop-abort policy to the existing shared detector path.

## Reproduction Assessment

Yes for source reproduction: on current main, an OpenShell command such as `workflow install <name>` or an unterminated quote flows through `prepareExec` into the SSH `/bin/sh -c` wrapper without preflight rejection. I did not run a live OpenShell session because this review is read-only.

## Solution Assessment

Yes, with a split boundary. The narrow maintainable fix is OpenShell/SSH command preflight; the repeated no-progress loop half is already handled on current main by the shared loop detector.

## Repair Work Prompt

Fix the remaining OpenShell malformed-command preflight gap from https://github.com/openclaw/openclaw/issues/72373. Add a narrow validation step before OpenShell/SSH exec command text is wrapped into the remote `/bin/sh -c` body, likely around `buildExecRemoteCommand` or the OpenShell/SSH backend exec boundary. Reject obvious generated placeholder tokens such as unquoted `<name>`/`<workflow-id>` and malformed shell syntax called out in the issue: unclosed quotes, trailing backslash escapes, and unterminated backticks/command substitutions. Keep the detector conservative so valid shell redirection and quoted literal angle-bracket text are not rejected. Do not change broader generic loop-abort policy; current main already blocks identical no-progress outcomes at `criticalThreshold`. Add focused tests for OpenShell and the shared SSH helper, plus a changelog entry if user-visible behavior changes.
