## Summary

Keep open. Current main still lets OpenShell/SSH exec command text reach `/bin/sh -c` without malformed-command preflight, while the repeated-tool hard-abort request overlaps separate open loop-detection work and needs maintainer scope judgment.

## What This Changes

OpenShell should reject malformed generated shell commands before execution and repeated identical tool loops should hard-abort after a threshold.

## Best Possible Solution

Add a narrowly scoped OpenShell/SSH exec-command validation layer for clearly malformed generated command text, and resolve repeated-loop abort policy through the canonical loop-detection items.

## Reproduction Assessment

Yes for the current behavior by source inspection: `workflow install <name>` or an unterminated quote is accepted into OpenShell prepareExec and then wrapped for `/bin/sh -c`. I did not run a live OpenShell session because this review is read-only, but the call path is clear.

## Solution Assessment

Yes for splitting the fix boundary. A targeted OpenShell/SSH preflight is maintainable, while global repeated-tool hard-abort behavior should be handled through #58726, #56403, and #60248 rather than folded into this issue ad hoc.
