## Summary

Keep open. Current main fixed the generic repeated no-progress loop half, but the OpenShell/SSH exec path still forwards malformed or placeholder command text into the remote `/bin/sh -c` body without a preflight guard.

## What This Changes

Request OpenShell command preflight for malformed generated shell text and stronger repeated-tool-loop abort behavior.

## Best Possible Solution

Add a narrow OpenShell/SSH exec preflight that rejects obviously generated placeholders and malformed shell syntax before remote shell invocation, while leaving generic loop-abort policy to the existing loop-detection implementation and follow-ups.

## Reproduction Assessment

Yes for source reproduction: on current main, an OpenShell command such as `workflow install <name>` or an unterminated quote flows through `prepareExec` into the SSH `/bin/sh -c` wrapper without preflight rejection. I did not run a live OpenShell session because this review is read-only.

## Solution Assessment

Yes, with a split boundary. The narrow maintainable fix is OpenShell/SSH command preflight; the repeated no-progress loop part is already handled on current main by the shared loop detector, and broader abort policy should stay separate.

## Repair Work Prompt

Fix the OpenShell malformed generated-command preflight gap from https://github.com/openclaw/openclaw/issues/72373. Scope the change to the OpenShell/SSH exec boundary: reject obviously non-runnable generated placeholders such as literal `<name>` / `<workflow-id>` tokens and malformed shell syntax such as unclosed quotes, trailing escapes, and unterminated backticks or command substitutions before building the remote `/bin/sh -c` command. Keep the rule narrow enough to avoid blocking legitimate shell redirection or intentional literal angle-bracket data. Do not change global loop-detection defaults or broader repeated-tool policy; current main already has the generic no-progress critical block from https://github.com/openclaw/openclaw/pull/80668. Add focused tests for accepted normal commands and rejected malformed commands in the OpenShell/SSH helper tests, update changelog if the repair is user-visible, and keep core/plugin ownership boundaries intact.
