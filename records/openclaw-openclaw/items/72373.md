## Summary

Keep open. Current main has the repeated no-progress loop hard-block path, but OpenShell/SSH exec still interpolates raw generated command text into `/bin/sh -c` without rejecting placeholder tokens or malformed shell syntax first.

## What This Changes

The issue requests OpenShell command preflight rejection for malformed generated shell commands and hard aborts for repeated identical tool loops.

## Best Possible Solution

Keep the merged loop-detection behavior and add a focused validator at the shared SSH/OpenShell exec boundary with regression coverage.

## Reproduction Assessment

Yes, at source level: malformed commands still flow from OpenShell `prepareExec` into the shared SSH `/bin/sh -c` builder without a validation step. I did not run a live OpenShell session in this read-only review.

## Solution Assessment

Yes for the remaining gap. A shared SSH/OpenShell command preflight is narrower and more maintainable than changing loop-detection policy again, since generic no-progress loop blocking is already implemented.

## Repair Work Prompt

Fix the remaining OpenShell/SSH exec preflight gap for https://github.com/openclaw/openclaw/issues/72373. Current main already hard-blocks repeated generic no-progress loops via https://github.com/openclaw/openclaw/pull/80668, so do not change loop-detection defaults, thresholds, or policy. Add a focused validator at the shared SSH/OpenShell exec boundary that rejects clearly generated non-runnable commands such as literal placeholder executable/argument tokens (`<name>`, `<workflow-id>`) and malformed shell quoting/escaping such as unclosed quotes, trailing escapes, and unterminated backticks before building the remote `/bin/sh -c` command. Keep the rule narrow enough not to reject valid shell redirection, heredocs, process substitution, or legitimate quoted literal angle-bracket text. Add regression tests in `src/agents/sandbox/ssh.test.ts` and OpenShell coverage in `extensions/openshell/src/openshell-core.test.ts` or the closest existing OpenShell backend test seam. Add a user-facing changelog fix entry if behavior changes are shipped.
