## Summary

Keep open. Current main still sends OpenShell exec command text into the SSH `/bin/sh -c` wrapper without generated-placeholder or malformed-shell preflight, while the repeated-loop part remains covered by open loop-detection follow-ups.

## What This Changes

Request fail-fast validation for malformed OpenShell generated commands and hard-abort behavior for repeated identical tool loops.

## Best Possible Solution

Add a conservative preflight at the shared SSH exec/OpenShell boundary that rejects generated placeholder tokens and malformed shell syntax before remote shell invocation, while resolving generic loop-abort policy in the existing loop-detection items.

## Reproduction Assessment

Yes. Static source reproduction is high-confidence: a command such as `workflow install <name>` or an unterminated quote flows from OpenShell `prepareExec` into the shared SSH `/bin/sh -c` wrapper on current main without a rejection path.

## Solution Assessment

Yes, with a split boundary. The narrow maintainable fix is OpenShell/SSH exec preflight validation; generic repeated-tool hard-abort behavior should be handled through #58726, #56403, or #60248.

## Repair Work Prompt

Fix the OpenShell malformed generated-command preflight part of https://github.com/openclaw/openclaw/issues/72373 only. Add conservative validation before OpenShell/SSH exec commands are wrapped for remote `/bin/sh -c`, rejecting obvious generated placeholders such as literal `<name>` or `<workflow-id>` in command tokens/args and malformed shell syntax such as unclosed quotes, trailing escapes, and unterminated backticks/command substitutions. Prefer a shared helper near `src/agents/sandbox/ssh.ts` or the sandbox SDK export path used by OpenShell so both direct SSH sandbox exec and OpenShell exec get the same guard. Add focused tests for accepted normal commands and rejected examples in `src/agents/sandbox/ssh.test.ts` and/or `extensions/openshell/src/openshell-core.test.ts`. Do not change `tools.loopDetection` defaults, generic repeat thresholds, or hard-abort policy; #58726, #56403, and #60248 track that separately. Add a single-line `CHANGELOG.md` fix entry if runtime behavior changes.
