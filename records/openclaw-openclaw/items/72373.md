## Summary

Keep open. Current main covers the generic repeated no-progress loop half, but the OpenShell/SSH exec path still forwards raw command text into `/bin/sh -c` without rejecting obvious placeholder or malformed shell input first.

## What This Changes

Request OpenShell to reject obviously generated or malformed shell commands before execution and to hard-abort repeated identical tool loops.

## Best Possible Solution

Add a narrow validator in the shared SSH/OpenShell exec boundary that rejects obvious generated placeholders and malformed quoting or escaping before remote shell invocation, with regression tests; leave broader loop-abort policy in the existing loop detector.

## Reproduction Assessment

Yes for source reproduction: `workflow install <name>` or an unterminated quote still flows from OpenShell `prepareExec` into the shared `/bin/sh -c` command body on current main. I did not run a live OpenShell session because this review is read-only.

## Solution Assessment

Yes, with a split boundary. A focused OpenShell/SSH command preflight is the narrow maintainable fix for the remaining behavior, while the repeated no-progress loop half is already handled by the shared loop detector.
