## Summary

Keep open. Current main still lacks OpenShell/SSH exec preflight for malformed generated command text, while the repeated-tool hard-abort part overlaps related open loop-detection work in #58726, #56403, and #60248.

## What This Changes

Request fail-fast validation for malformed/generated OpenShell commands and hard-abort behavior for repeated identical tool loops.

## Best Possible Solution

Add narrow OpenShell/SSH exec-command validation before remote shell invocation, and leave generic repeated-tool abort policy to the existing loop-detection cluster.

## Reproduction Assessment

Yes for source reproduction: a command such as `workflow install <name>` or an unterminated quote flows through OpenShell prepareExec into the SSH `/bin/sh -c` wrapper without validation. I did not run a live OpenShell session because this review was read-only.

## Solution Assessment

Yes, with a split boundary. The narrow maintainable fix is OpenShell/SSH exec preflight; enabling or hard-aborting generic repeated tool calls should be resolved through the linked loop-detection items.

## Repair Work Prompt

Fix the OpenShell malformed generated-command preflight portion of https://github.com/openclaw/openclaw/issues/72373. Add narrow validation before OpenShell/SSH exec wraps model-generated commands for `/bin/sh -c`, likely in `src/agents/sandbox/ssh.ts` or a shared helper used by `extensions/openshell/src/backend.ts`/`buildExecRemoteCommand`. Reject only clearly non-runnable generated-command hazards such as standalone literal angle placeholders like `<name>` or `<workflow-id>`, unclosed single/double quotes, trailing backslash escapes, and unterminated backticks or command substitutions. Return a clear tool-visible error before creating or using the SSH exec command. Add regression coverage in `extensions/openshell/src/openshell-core.test.ts` and/or `src/agents/sandbox/ssh.test.ts`. Do not change global loop-detection defaults, generic_repeat hard-abort policy, provider behavior, or add a config option; leave #58726, #56403, and #60248 to the loop-detection cluster. Add a one-line `CHANGELOG.md` Fixes entry if the behavior change is user-facing.
