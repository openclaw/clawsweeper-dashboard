## Summary

Keep open. Current main still lacks OpenShell malformed/generated-command preflight before the SSH `/bin/sh -c` wrapper, and the generic repeated-call detector still warns rather than hard-aborting at its own critical threshold. The related loop-detection issue and open PRs overlap the hard-abort half, but they do not cover the OpenShell command-validation gap described by the reporter.

## What This Changes

The issue requests OpenShell preflight rejection for generated placeholder/malformed shell commands and hard-abort behavior for repeated identical tool-call loops.

## Best Possible Solution

Add a focused OpenShell/SSH exec preflight that rejects the reported generated placeholders and malformed shell syntax with clear tool-visible errors, then resolve the repeated identical-call hard-abort through the existing loop-detection tracking so one canonical threshold behavior ships with tests and docs.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: `workflow install <name>` or an unterminated quote flows from OpenShell `prepareExec` into the SSH `/bin/sh -c` wrapper, and current loop tests still assert warning-only generic repeats below the global breaker.

## Solution Assessment

Yes. The narrow maintainable solution is OpenShell/SSH command preflight plus coordinated loop-detector escalation, not a broad command-obfuscation layer or a duplicate generic loop-policy PR.
