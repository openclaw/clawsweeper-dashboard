## Summary

Keep open. Current main still lacks OpenShell/SSH exec preflight for malformed generated command text, and the repeated-tool hard-abort behavior is only partially covered by existing warning/global-breaker logic and related open loop-detection items.

## What This Changes

The issue requests OpenShell preflight rejection for malformed/generated shell commands and hard-abort behavior for repeated identical tool calls.

## Best Possible Solution

Add narrow OpenShell/SSH exec-command validation before remote shell invocation, while resolving generic repeated-tool abort policy through the existing loop-detection cluster.

## Reproduction Assessment

Yes. A static reproduction is enough: a command such as `workflow install <name>` or an unterminated quote flows from OpenShell `prepareExec` into the SSH `/bin/sh -c` wrapper without preflight rejection on current main.

## Solution Assessment

Yes, with a split boundary. The maintainable fix is targeted validation near the OpenShell/SSH exec wrapper; changing global loop-detection defaults or critical thresholds belongs in the related loop-detection items.

## Repair Work Prompt

Fix the OpenShell/SSH exec preflight gap from https://github.com/openclaw/openclaw/issues/72373. Add narrow validation before `buildExecRemoteCommand` is used for OpenShell/SSH exec commands so obviously generated or malformed commands fail fast before `/bin/sh -c`, including literal generated placeholder tokens in executable/argument positions such as `<name>` or `<workflow-id>`, unclosed quotes, trailing backslash escapes, and unterminated backticks/command substitutions. Keep the fix scoped to the OpenShell/SSH exec-command boundary; do not change global loop-detection defaults or generic-repeat hard-abort policy, which remains tracked by #58726, #56403, and #60248. Avoid blocking valid shell redirection/comparison/here-doc syntax. Add focused tests for rejection and for allowed valid shell commands. Add a user-facing `CHANGELOG.md` Fixes entry, without guessing contributor attribution. Before handoff run targeted tests/format checks and use Testbox for `pnpm check:changed` if code/tests changed.
