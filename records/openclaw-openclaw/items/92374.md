## Summary

Keep open: current main and latest release still bypass the documented `message_sending` hook when a channel supplies `beforeDeliver`, and the closing repair PR is open but unmerged. This issue should remain the canonical repair target until one linked fix lands or maintainers explicitly split the remaining preview-finalized gap.

## What This Changes

The issue reports that plugin `message_sending` hooks are silently skipped on channel agent-reply delivery paths when the channel provides its own `beforeDeliver` hook.

## Best Possible Solution

Land one shared dispatcher fix that composes channel `beforeDeliver`, `reply_payload_sending`, and `message_sending` exactly once before visible delivery, then explicitly verify or track the preview-finalized path.

## Reproduction Assessment

Yes. Source inspection shows any channel-provided `dispatcherOptions.beforeDeliver` deterministically selects a composition that excludes `message_sending`; the test suite currently asserts that bypass.

## Solution Assessment

Yes. The right fix belongs in the shared dispatcher hook chain, not a Telegram-only workaround, with tests for custom `beforeDeliver` ordering and exactly-once execution.
