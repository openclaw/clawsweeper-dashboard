## What This Changes

The issue asks for documented plugin `message_sending` hooks to run on channel agent-reply delivery paths even when a channel supplies its own `beforeDeliver`, including Telegram replies and preview-finalized streaming sends.

## Best Possible Solution

Land one canonical repair that applies `message_sending` exactly once across shared configured dispatchers and Telegram preview-finalized delivery, with regression tests and current-head Telegram proof.

## Reproduction Assessment

Yes, at source level. Current main deterministically drops `message_sending` whenever `dispatcherOptions.beforeDeliver` exists, and the adjacent test still asserts that `runMessageSending` is not called on that path.

## Solution Assessment

Yes for keeping this issue open as the canonical bug. The best fix is not another broad work item; it is review and landing of one candidate that preserves exactly-once hook execution across the shared dispatcher and Telegram streaming paths.
