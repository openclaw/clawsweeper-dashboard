## Label Justifications

- `P1`: A documented outbound interception contract is silently bypassed on primary channel agent-reply paths in current main and the latest release.
- `impact:security`: Safety, DLP, compliance, or content-filter plugins relying on `message_sending` can fail open without operator-visible errors.
- `impact:message-loss`: Plugin cancellation and rewrite decisions can be ignored, allowing unmodified or unintended outbound replies to be delivered.

## Summary

This issue is still necessary: current main and latest release `v2026.6.6` still compose channel-supplied `beforeDeliver` with only `reply_payload_sending`, leaving the documented `message_sending` gate out of configured dispatcher paths. It should remain the canonical issue while maintainers choose and land one of the open fix PRs, especially the broader Telegram preview-finalized coverage in https://github.com/openclaw/openclaw/pull/93216.

## What This Changes

The issue reports that plugin `message_sending` hooks are silently skipped on channel agent-reply delivery paths when a channel supplies `dispatcherOptions.beforeDeliver`, with Telegram as the concrete affected channel.

## Best Possible Solution

Land one canonical fix that runs `message_sending` exactly once after channel normalization and `reply_payload_sending` for every visible agent reply, including configured dispatchers and Telegram preview-finalized finals.

## Reproduction Assessment

Yes from source inspection: the configured dispatcher branches deterministically omit `buildMessageSendingBeforeDeliver`, and the adjacent test currently asserts that `runMessageSending` is not called when custom `beforeDeliver` is present. I did not run a live Telegram reproduction in this read-only review.

## Solution Assessment

Yes, the best fix is a shared exactly-once pre-send pipeline rather than a Telegram-only workaround or startup warning. The broader solution should cover both configured dispatchers and Telegram preview-finalized finals while preserving hook order and avoiding duplicate execution.
