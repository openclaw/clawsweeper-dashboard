## Summary

## What This Changes

The issue asks MiniMax usage polling to respect `models.providers.minimax.baseUrl` instead of always querying the CN host.

## Best Possible Solution

Keep the shipped baseUrl-derived MiniMax usage endpoint behavior and close obsolete duplicate PR attempts that targeted the old hardcoded host.

## Reproduction Assessment

Yes for the original bug from source and history: the old path hardcoded the CN usage host, while current main and v2026.5.18 now have tests that assert global baseUrl routing. I did not live-call MiniMax because no credential is needed to prove the endpoint-selection fix.

## Solution Assessment

Yes. The shipped implementation is the narrow maintainable fix: derive only the MiniMax usage polling origin from configured provider baseUrl and keep the CN fallback for existing or malformed configurations.
