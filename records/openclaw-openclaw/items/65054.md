## Summary

Current main and the shipped v2026.5.2 release already derive the MiniMax usage polling host from configured MiniMax baseUrl values, with regression coverage for the global endpoint and a CN fallback for compatibility.

## What This Changes

MiniMax usage polling should respect the configured MiniMax baseUrl instead of always using the CN usage host.

## Best Possible Solution

Keep the shipped baseUrl-derived MiniMax usage endpoint behavior and handle any remaining static-host PRs as separate superseded cleanup.

## Reproduction Assessment

No, current main no longer reproduces the hardcoded-host path. Source inspection shows the fetcher derives the usage origin from MiniMax config, and tests assert the global endpoint request URL.

## Solution Assessment

Yes. The current implementation is the narrow maintainable fix: derive only the usage endpoint origin from provider config while preserving the existing CN fallback for missing or malformed config.
