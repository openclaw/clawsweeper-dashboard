## Summary

Close as implemented: current main and shipped release v2026.5.2 now derive MiniMax Coding Plan usage polling from the configured MiniMax base URL, with regression coverage for global, CN, and fallback URLs. The earlier linked PRs were useful context, but the shipped commit already covers the requested behavior.

## What This Changes

The issue asks for MiniMax usage polling to respect the configured `models.providers.minimax.baseUrl` instead of always calling the CN usage host.

## Best Possible Solution

Keep the shipped baseUrl-aware MiniMax usage implementation and treat older open or unmerged endpoint-only PRs as superseded unless they add a distinct remaining fix.

## Reproduction Assessment

No. The original current-main reproduction no longer holds: source inspection shows the provider passes configured MiniMax base URLs into the usage fetcher, and regression tests assert the global URL for `https://api.minimax.io/anthropic`.

## Solution Assessment

Yes. The shipped fix is the narrow maintainable solution because it derives only the usage endpoint origin from configured MiniMax provider state while preserving the existing CN fallback for unset or malformed config.
