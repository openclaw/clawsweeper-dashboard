## Summary

The reported failure is in the separately installed TencentDB Agent Memory plugin: its runner intentionally selects an empty workspace while supplying its own extraction system/user prompts, and current OpenClaw’s embedded-agent API already supports the required explicit workspace and prompt inputs.

## What This Changes

Investigate why the memory TencentDB L1 extraction runner uses a temporary workspace without the expected agent context files and returns an empty model response.

## Best Possible Solution

Track and fix the empty-response compatibility problem in TencentDB Agent Memory by adjusting its explicit prompt/runtime invocation while preserving deliberate context isolation; open an OpenClaw issue only if that work demonstrates a missing generic embedded-agent API.

## Reproduction Assessment

Yes at the source-contract level: the logs match the external plugin’s intentional clean-workspace selection, while OpenClaw source confirms lightweight runs omit bootstrap files by design. The model-specific empty response still needs reproduction in the plugin repository.

## Solution Assessment

No; inheriting AGENTS.md, SOUL.md, and the entire main workspace would defeat the plugin’s documented clean-context isolation. The narrower fix is to repair its explicit extraction prompt/runtime compatibility upstream.
