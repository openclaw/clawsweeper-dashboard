## What This Changes

The branch adds a WARN log during isolated cron `agentTurn` preparation when the payload lacks `lightContext` and `toolsAllow`, and adjusts one model-override test fixture to set `lightContext`.

## Best Possible Solution

Land a narrow cron diagnostic with changelog coverage and provider-accurate wording or gating, without changing isolated cron timeout semantics.

## Reproduction Assessment

Yes. At code-path level, current main forwards `lightContext` and `toolsAllow` into embedded isolated cron runs, and the linked issue provides concrete cron-job settings that trigger the cold-start timeout.

## Solution Assessment

Partly. The warning is the narrowest maintainable diagnostic, but the current patch should be scoped or worded so CLI-provider jobs are not told to use embedded-only startup controls.
