## Summary

Keep open. The PR addresses a real isolated-cron diagnostic gap, but it still needs a required changelog entry and the new warning should avoid giving embedded-only advice to CLI-backed cron runs before merge.

## What This Changes

The branch adds a WARN log when isolated cron `agentTurn` payloads omit `lightContext` and `toolsAllow`, and updates one model-override test fixture to set `lightContext`.

## Best Possible Solution

Land a narrow cron diagnostic with changelog coverage and provider-accurate warning behavior, without changing isolated cron timeout semantics.

## Reproduction Assessment

Yes. At source level, isolated cron `agentTurn` jobs can omit both `lightContext` and `toolsAllow`, the linked issue gives concrete settings that time out on cold start, and current main lacks a warning for that state.

## Solution Assessment

Partly. A warning is a narrow maintainable fix for the diagnostic gap, but the current patch should add release-note coverage and scope or reword the warning so CLI-backed runs are not told to use embedded-only startup controls.
