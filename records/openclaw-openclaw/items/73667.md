## Summary

Keep open. This draft, conflicting external PR still has a P1 hook-timeout regression and lacks after-fix real behavior proof, while the linked Active Memory/QMD reliability issue remains broader than this branch.

## What This Changes

The branch lowers Active Memory default/setup-grace timeouts, changes before_prompt_build hook timeout sizing, adds jittered parallel QMD startup initialization, and updates docs/tests.

## Best Possible Solution

Repair or replace the branch so hook budgets preserve configured recall timeouts, keep current-main fast-fail/lazy QMD mitigations, and handle remaining nonblocking/default policy through maintainer-reviewed follow-ups.

## Reproduction Assessment

Yes, for the PR regression from source inspection: configure Active Memory with timeoutMs above 3000 ms on the branch, and the fixed 4000 ms hook timeout can fire before the configured recall budget. I did not run tests because this was a read-only review.

## Solution Assessment

No, not as proposed. The reliability direction is plausible, but the hook envelope must be sized from the resolved configured timeout or cap, and QMD startup/default-latency policy should be reconciled with current main before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9
