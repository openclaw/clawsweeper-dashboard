## Summary

Keep this draft PR open: the reliability direction is relevant to #72015, but the branch introduces a real hook-timeout regression for valid custom Active Memory timeouts and overlaps newer current-main QMD startup policy work.

## What This Changes

The PR lowers Active Memory default timeout/setup grace, updates docs/tests for shorter recall budgets, and adds jittered parallel QMD startup initialization for eager agents.

## Best Possible Solution

Land a maintainer-approved reliability change on top of current main that preserves the config-derived hook envelope, adds coverage for custom recall budgets above the default, updates changelog/docs, and fits the existing QMD startup opt-in/lazy-start policy.

## Reproduction Assessment

Yes for the PR regression: configure Active Memory with `timeoutMs` above 3000 ms, such as 5000 ms. The branch's 4000 ms hook timeout can fire before the plugin's configured recall watchdog returns.

## Solution Assessment

No, not as proposed. Lower default latency may be valid, but the hook timeout must be sized from the resolved configured budget or documented cap, and QMD startup changes should be reconciled with current main's opt-in/lazy startup policy.
