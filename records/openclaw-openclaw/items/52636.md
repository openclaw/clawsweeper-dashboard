Full review comments:

- **[P1] Enforce the loopback ceiling before normal lockout short-circuits it:** `src/gateway/auth-rate-limit.ts:214-215`
  - body: `isExempt()` allows loopback until `maxAttempts * 10`, but `recordFailure()` still sets `lockedUntil` at `maxAttempts`; after that, attempts stop growing while `check()` keeps returning allowed from the exempt path, so localhost failures can continue indefinitely instead of hitting the advertised ceiling.
  - confidence: 0.92
- **[P1] Keep same-origin microphone access unless policy changes:** `src/gateway/http-common.ts:18`
  - body: Current main intentionally sets and tests `microphone=(self)` for same-origin Control UI/webchat voice capture. Changing the default to `microphone=()` will break that existing behavior unless maintainers explicitly accept the policy break with docs, tests, and upgrade proof.
  - confidence: 0.9
- **[P1] Apply eval hardening to the shipped browser plugin path:** `src/browser/pw-tools-core.interactions.ts:305-306`
  - body: The PR adds the eval blocklist under `src/browser/...`, but current main's runtime implementation lives under `extensions/browser/...`; after rebase this either conflicts or misses the actual evaluate path, leaving the claimed hardening unenforced.
  - confidence: 0.88
- **[P2] Reject malformed webhook signatures without throwing:** `src/security/shield/webhook-dispatch.ts:63`
  - body: This compares hex string lengths before decoding, then calls `timingSafeEqual()` on decoded buffers. A same-length non-hex signature can decode to a shorter buffer and throw instead of returning `false`, turning a bad signature into an exception path.
  - confidence: 0.86
- **[P2] Recompute escalation when circuits recover:** `src/security/shield/gateway-shield.ts:174-175`
  - body: `recordSuccess()` updates the circuit map but never refreshes `this.escalation`, so status/summary readers can keep seeing a paused or degraded escalation state after the circuit has closed.
  - confidence: 0.82
- **[P2] Expire stale health scores when metrics leave the window:** `src/security/shield/gateway-shield.ts:287`
  - body: `computeHealthScores()` filters to recent metrics but only updates functions present in the new aggregate, so a function that drops out of the window can keep an old CRITICAL or CIRCUIT_OPEN score and continue being throttled from stale state.
  - confidence: 0.8
