## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Enforce the loopback ceiling before normal lockout:** `src/gateway/auth-rate-limit.ts:214-215`
  - body: `recordFailure()` still sets the normal `maxAttempts` lockout for loopback clients, but `check()` returns from `isExempt()` before honoring `lockedUntil` while the count is below `maxAttempts * 10`. Once the normal lockout is set, further failures stop being recorded, so localhost attempts remain allowed indefinitely instead of hitting the advertised 10x ceiling.
  - confidence: 0.9
- **[P1] Keep same-origin microphone access unless policy changes:** `src/gateway/http-common.ts:18`
  - body: Current main intentionally allows `microphone=(self)` and tests that header because `microphone=()` broke Control UI speech-to-text. This PR restores the fully-denying value without an explicit product/security decision or upgrade proof for the existing voice flow.
  - confidence: 0.88
- **[P1] Apply eval hardening to the shipped browser plugin path:** `src/browser/pw-tools-core.interactions.ts:305-306`
  - body: The PR adds the blocked-pattern filter under `src/browser/...`, but current main's shipped browser implementation is under `extensions/browser/src/browser/pw-tools-core.interactions.ts`. As written, the claimed evaluate hardening would not protect the current runtime path after a rebase.
  - confidence: 0.86
- **[P2] Reject malformed webhook signatures without throwing:** `src/security/shield/webhook-dispatch.ts:63`
  - body: `verifyWebhookSignature()` compares hex string lengths before decoding, then passes decoded buffers to `timingSafeEqual()`. A same-length non-hex signature can decode to a shorter buffer and throw instead of returning `false`, turning a bad signature into an exception path.
  - confidence: 0.86
- **[P2] Refresh escalation state when circuits recover:** `src/security/shield/gateway-shield.ts:174-175`
  - body: `recordSuccess()` updates the circuit state but never recomputes `this.escalation`. If a previously open circuit closes, status and summary calls can keep reporting `PARTIAL_PAUSE` or `FULL_PAUSE` until another failure opens a circuit.
  - confidence: 0.84
- **[P2] Expire stale health scores without recent metrics:** `src/security/shield/gateway-shield.ts:301`
  - body: `computeHealthScores()` filters metrics to the last five minutes but only updates functions present in the new aggregate. Functions with no recent metrics can keep old `CRITICAL` or `CIRCUIT_OPEN` scores and continue throttling after recovery or idle time.
  - confidence: 0.82
- **[P2] Collapse health windows before storing per-function scores:** `src/security/shield/gateway-shield.ts:287`
  - body: `aggregateByFunctionAndMinute()` returns buckets keyed by function and minute, but this loop stores them by function only. If bucket insertion order is not chronological, an older minute can overwrite the latest health score and make throttling depend on metric arrival order.
  - confidence: 0.78
