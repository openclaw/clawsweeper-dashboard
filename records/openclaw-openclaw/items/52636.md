## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Enforce the loopback cap before normal lockout:** `src/gateway/auth-rate-limit.ts:214-215`
  - body: `recordFailure()` still applies the normal `maxAttempts` lockout to loopback clients, then stops appending attempts while locked. Because `check()` allows loopback through `isExempt()` until `maxAttempts * 10`, the counter never reaches the intended 10x ceiling and localhost failures can continue being allowed.
  - confidence: 0.9
- **[P2] Rebase evaluate hardening onto the active browser plugin:** `src/browser/pw-tools-core.interactions.ts:303-309`
  - body: This filter is added under `src/browser`, but current main has moved `evaluateViaPlaywright()` to `extensions/browser/src/browser/pw-tools-core.interactions.ts`. As written, the security hardening does not affect the current browser runtime and needs to be rebased onto the active path.
  - confidence: 0.86
- **[P2] Wire Shield into gateway runtime paths:** `src/security/shield/index.ts:121`
  - body: `GatewayShield` is only exported from the new barrel; no gateway startup, auth, request, or WebSocket path constructs or calls it. The advertised circuit breaker, anomaly detection, health scoring, and escalation protections therefore do not run in production.
  - confidence: 0.87
- **[P2] Reject malformed webhook signatures without throwing:** `src/security/shield/webhook-dispatch.ts:63`
  - body: `verifyWebhookSignature()` checks the hex string length before decoding, then passes decoded buffers to `timingSafeEqual()`. A same-length non-hex signature can decode to a shorter buffer and throw instead of returning `false`.
  - confidence: 0.86
- **[P2] Refresh escalation state when circuits recover:** `src/security/shield/gateway-shield.ts:174-175`
  - body: `recordSuccess()` updates the circuit but never recomputes `this.escalation`. After an open circuit closes, status and summary APIs can keep reporting a pause state until some later failure recalculates escalation.
  - confidence: 0.84
- **[P2] Preserve same-origin microphone permission:** `src/gateway/http-common.ts:18`
  - body: Current main intentionally sets `microphone=(self)` and tests that default for Control UI voice behavior. This branch changes the shared default to `microphone=()`, which would block same-origin microphone access again.
  - confidence: 0.82
- **[P2] Expire stale health scores without recent metrics:** `src/security/shield/gateway-shield.ts:301`
  - body: `computeHealthScores()` filters to recent metrics but only updates functions present in the aggregate. Functions that leave the five-minute window keep prior `CRITICAL` or `CIRCUIT_OPEN` scores, so `shouldThrottle()` can keep throttling recovered or idle functions from stale state.
  - confidence: 0.8
