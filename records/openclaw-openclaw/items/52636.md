## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Enforce the loopback threshold before locking out:** `src/gateway/auth-rate-limit.ts:214-215`
  - body: `recordFailure()` still sets `lockedUntil` once loopback reaches the normal `maxAttempts`, but `check()` returns allowed from `isExempt()` until `maxAttempts * 10`. While locked, later failures are not appended, so the counter never reaches the intended 10x ceiling and localhost attempts remain effectively unbounded.
  - confidence: 0.9
- **[P2] Return false for malformed webhook signatures:** `src/security/shield/webhook-dispatch.ts:63`
  - body: This checks hex string lengths before decoding, then calls `timingSafeEqual()` on decoded buffers. A same-length non-hex signature decodes to a shorter buffer, making verification throw instead of returning `false`.
  - confidence: 0.86
- **[P2] Recompute escalation after circuit recovery:** `src/security/shield/gateway-shield.ts:174-175`
  - body: `recordSuccess()` stores the recovered circuit but never recalculates `this.escalation`. After an open circuit closes, `getStatus()` and `getSummary()` can keep reporting a pause state until another failure happens.
  - confidence: 0.84
- **[P2] Drop health scores that fall out of the metrics window:** `src/security/shield/gateway-shield.ts:301`
  - body: `computeHealthScores()` filters request metrics to the last five minutes but only updates functions present in the new aggregate. A function with no recent metrics keeps an old `CRITICAL` or `CIRCUIT_OPEN` score, so later throttling can use stale state.
  - confidence: 0.82
- **[P2] Collapse health buckets before storing per-function scores:** `src/security/shield/gateway-shield.ts:287`
  - body: `aggregateByFunctionAndMinute()` returns one bucket per function and minute, but this loop stores each bucket under only the function name. If metric arrival order is not chronological, an older minute can overwrite a newer score and make throttling depend on insertion order.
  - confidence: 0.78
- **[P2] Preserve same-origin microphone permission:** `src/gateway/http-common.ts:18`
  - body: Current main expects the default gateway `Permissions-Policy` to allow `microphone=(self)`, but this change sets `microphone=()`. That can break same-origin browser UI features that rely on microphone access.
  - confidence: 0.77
