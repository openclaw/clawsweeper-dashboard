## Summary

Keep open. Current main only covers the config file opener hardening slice, while this PR's broader Shield, loopback throttling, browser evaluate, and header changes are not fully implemented on main and the PR head still has merge-blocking security/correctness issues.

## What This Changes

The PR adds a new `src/security/shield` runtime subsystem and changes gateway config file opening, auth rate limiting, default HTTP security headers, and browser Playwright evaluate filtering.

## Best Possible Solution

Split the work into focused security PRs: keep current main's `config.openFile` hardening, then review loopback throttling, browser evaluate policy, headers, and any Shield subsystem separately with secops ownership, provenance, threat-model rationale, tests, and changelog entries.

## Reproduction Assessment

Yes. The blockers are reproducible by static inspection of PR head and current main: loopback lockout short-circuits the intended 10x cap, malformed webhook signatures can throw, the browser change targets a removed path, and Shield state can remain stale.

## Solution Assessment

No. This is not the best way to land the work because it combines a shipped hardening fix, stale-path edits, new runtime security policy, and third-party-attributed code in one broad branch; narrower secops-reviewed PRs are safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Enforce the loopback cap before returning exempt:** `src/gateway/auth-rate-limit.ts:156-157`
  - body: `check()` returns allowed for loopback entries before it considers `lockedUntil`, but `recordFailure()` still applies the normal `maxAttempts` lockout and then stops recording while locked. The attempt count never reaches the advertised 10x ceiling, so localhost failures continue to be allowed instead of being throttled at the new cap.
  - confidence: 0.9
- **[P2] Move evaluate filtering to the active browser plugin path:** `src/browser/pw-tools-core.interactions.ts:305-306`
  - body: Current main no longer has `src/browser`; the active `evaluateViaPlaywright()` implementation lives under `extensions/browser/src/browser`. Adding the blocked-pattern filter here either conflicts on rebase or leaves the active browser evaluate surface unchanged.
  - confidence: 0.88
- **[P2] Reject malformed webhook signatures without throwing:** `src/security/shield/webhook-dispatch.ts:63`
  - body: `verifyWebhookSignature()` compares hex string lengths before decoding, then passes decoded buffers to `timingSafeEqual()`. A same-length non-hex signature decodes to a shorter buffer and can throw instead of returning `false`, turning a bad signature into an exception path.
  - confidence: 0.86
- **[P2] Refresh escalation state when circuits recover:** `src/security/shield/gateway-shield.ts:174-175`
  - body: `recordSuccess()` updates the circuit state but never recomputes `this.escalation`. If an open circuit closes, `getStatus()` and `getSummary()` can keep reporting an escalated pause until another circuit opens.
  - confidence: 0.84
- **[P2] Drop stale health scores when metrics expire:** `src/security/shield/gateway-shield.ts:301`
  - body: `computeHealthScores()` filters to recent metrics but only updates functions present in the new aggregate. Functions with no recent metrics keep prior `CRITICAL` or `CIRCUIT_OPEN` health, so `shouldThrottle()` can continue throttling recovered or idle functions from stale state.
  - confidence: 0.82
- **[P2] Collapse health buckets before storing scores:** `src/security/shield/gateway-shield.ts:287`
  - body: `aggregateByFunctionAndMinute()` returns one bucket per function and minute, but this loop stores them in `healthScores` keyed only by function name. If metric insertion order is not chronological, an older minute can overwrite a newer health score and make throttling decisions depend on arrival order.
  - confidence: 0.78

  - body: Escalation and health-score state can persist after recovery or after metrics age out, which is risky for a subsystem that may throttle or pause gateway behavior.
  - confidence: 0.82
