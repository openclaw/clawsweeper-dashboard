## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Enforce the loopback cap before normal lockout:** `src/gateway/auth-rate-limit.ts:214-215`
  - body: `recordFailure()` still sets `lockedUntil` at `maxAttempts`, then stops appending attempts while locked. Because `check()` returns through the loopback exemption until `maxAttempts * 10`, localhost failures never reach the intended cap and remain allowed indefinitely.
  - confidence: 0.9
- **[P2] Wire the Shield into gateway runtime paths:** `src/security/shield/index.ts:121`
  - body: `GatewayShield` is only exported from the new barrel; the PR does not construct it from gateway startup, auth, request, or WebSocket paths. The circuit breaker, anomaly detection, health scoring, and escalation modules therefore never run in production.
  - confidence: 0.85
- **[P2] Reject malformed webhook signatures without throwing:** `src/security/shield/webhook-dispatch.ts:63`
  - body: This compares hex string lengths before decoding, then calls `timingSafeEqual()` on decoded buffers. A same-length non-hex signature can decode to a shorter buffer and throw instead of returning `false`.
  - confidence: 0.86
- **[P2] Refresh escalation state when circuits recover:** `src/security/shield/gateway-shield.ts:174-175`
  - body: `recordSuccess()` stores the updated circuit but never recomputes `this.escalation`. After an open circuit closes, status APIs can continue reporting a pause state until a later failure recalculates escalation.
  - confidence: 0.84
- **[P2] Drop stale health scores when metrics leave the window:** `src/security/shield/gateway-shield.ts:301`
  - body: `computeHealthScores()` filters to recent metrics but never removes functions that no longer appear in the aggregate. A stale `CRITICAL` or `CIRCUIT_OPEN` score can keep throttling an idle or recovered function.
  - confidence: 0.82
- **[P2] Preserve same-origin microphone permission:** `src/gateway/http-common.ts:18`
  - body: Current main intentionally allows `microphone=(self)` and has tests for that Control UI behavior. This branch changes the default Permissions-Policy to `microphone=()`, which would block same-origin voice paths again.
  - confidence: 0.78

  - body: The PR intends to replace full localhost exemption with a 10x cap, but normal lockout stops recording failures before that cap can be reached while `check()` continues allowing loopback through the exemption path.
  - confidence: 0.9

  - body: The new security subsystem is exported but not wired into gateway runtime paths, so it can give maintainers a false sense that circuit breaking and anomaly detection are protecting production traffic.
  - confidence: 0.85
