  - confidence: 0.91

  - body: `recordFailure()` still sets `lockedUntil` at `maxAttempts`, then stops appending failures while `check()` bypasses that lockout for loopback until `maxAttempts * 10`; the counter never reaches the intended loopback ceiling.
  - confidence: 0.9
- **[P1] Wire Shield into real gateway request paths:** `src/security/shield/index.ts:121`
  - body: The new Shield runtime is only exported from the new package index; no changed gateway, auth, WebSocket, or plugin request path calls it, so the advertised runtime protection never executes.
  - confidence: 0.86
- **[P2] Reject malformed webhook signatures without throwing:** `src/security/shield/webhook-dispatch.ts:63`
  - body: `verifyWebhookSignature()` checks hex string length before decoding, so a same-length non-hex signature can decode to a shorter buffer and make `timingSafeEqual()` throw instead of returning `false`.
  - confidence: 0.86
- **[P2] Recompute escalation when circuits recover:** `src/security/shield/gateway-shield.ts:174-175`
  - body: `recordSuccess()` stores the recovered circuit but never refreshes `this.escalation`, so status and summary can keep reporting pause states after the underlying circuit closes.
  - confidence: 0.84
- **[P2] Aggregate health metrics by function before storing:** `src/security/shield/gateway-shield.ts:287`
  - body: `aggregateByFunctionAndMinute()` returns one bucket per function and minute, but this writes each bucket into a map keyed only by function name, making the final health score depend on metric insertion order rather than the newest window.
  - confidence: 0.78
- **[P2] Expire stale health scores without recent metrics:** `src/security/shield/gateway-shield.ts:301`
  - body: `computeHealthScores()` filters metrics to the last five minutes but never removes functions missing from the new aggregate, so old CRITICAL or CIRCUIT_OPEN scores can keep throttling recovered or idle functions.
  - confidence: 0.82
