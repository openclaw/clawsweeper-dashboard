## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Enforce the loopback cap before normal lockout:** `src/gateway/auth-rate-limit.ts:214-215`
  - body: `recordFailure()` still sets `lockedUntil` at the normal `maxAttempts` threshold, then stops recording while locked. Because `check()` returns through the loopback exemption until `maxAttempts * 10`, localhost failures never reach the intended cap and remain allowed indefinitely.
  - confidence: 0.9
- **[P2] Wire Shield into gateway runtime paths:** `src/security/shield/index.ts:121`
  - body: `GatewayShield` is only exported from the new barrel; the PR does not construct it from gateway startup, auth, request, or WebSocket paths. The advertised circuit breaker, anomaly detection, health scoring, and escalation protections therefore never run in production.
  - confidence: 0.85
- **[P2] Reject malformed webhook signatures without throwing:** `src/security/shield/webhook-dispatch.ts:63`
  - body: `verifyWebhookSignature()` compares hex string lengths before decoding, then passes decoded buffers to `timingSafeEqual()`. A same-length non-hex signature can decode to a shorter buffer and make verification throw instead of returning `false`.
  - confidence: 0.86
- **[P2] Refresh escalation state when circuits recover:** `src/security/shield/gateway-shield.ts:174-175`
  - body: `recordSuccess()` stores the updated circuit but never recomputes `this.escalation`. After an open circuit closes, status APIs can keep reporting a pause state until a later failure recalculates escalation.
  - confidence: 0.84
- **[P2] Drop stale health scores when metrics leave the window:** `src/security/shield/gateway-shield.ts:265-287`
  - body: `computeHealthScores()` filters metrics to the last five minutes but only updates functions present in the aggregate. Functions with no recent metrics keep their previous `CRITICAL` or `CIRCUIT_OPEN` score, so `shouldThrottle()` can continue throttling idle or recovered functions from stale state.
  - confidence: 0.82
- **[P2] Preserve same-origin microphone permission:** `src/gateway/http-common.ts:18`
  - body: Current main intentionally allows `microphone=(self)` and tests that Control UI behavior. This branch sets the default Permissions-Policy to `microphone=()`, which would block same-origin voice paths again when reconciled with main.
  - confidence: 0.78
