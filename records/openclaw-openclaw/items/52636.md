## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Enforce the loopback threshold before setting lockout:** `src/gateway/auth-rate-limit.ts:214-215`
  - body: `recordFailure()` still sets the normal `maxAttempts` lockout for loopback clients, but `check()` returns allowed from `isExempt()` until attempts reach `maxAttempts * 10`. While `lockedUntil` is active, later failures are not appended, so localhost never reaches the advertised 10x ceiling.
  - confidence: 0.9
- **[P2] Return false for malformed webhook signatures:** `src/security/shield/webhook-dispatch.ts:63`
  - body: `verifyWebhookSignature()` compares hex string lengths before decoding, then passes decoded buffers to `timingSafeEqual()`. A same-length non-hex signature can decode to a shorter buffer and throw instead of returning `false`, turning bad input into an exception path.
  - confidence: 0.86
- **[P2] Recompute escalation after circuit recovery:** `src/security/shield/gateway-shield.ts:174-175`
  - body: `recordSuccess()` stores the updated circuit but never recalculates `this.escalation`. When an open circuit closes, `getStatus()` and `getSummary()` can keep reporting a pause state until another failure opens a circuit.
  - confidence: 0.84
- **[P2] Collapse health buckets before storing scores:** `src/security/shield/gateway-shield.ts:265-287`
  - body: `aggregateByFunctionAndMinute()` returns one bucket per function and minute, but the loop writes each bucket into `healthScores` keyed only by function. If metrics arrive out of chronological order, an older minute can overwrite the newer health score and make throttling depend on insertion order.
  - confidence: 0.78
- **[P2] Expire health scores when metrics age out:** `src/security/shield/gateway-shield.ts:259-301`
  - body: `computeHealthScores()` filters metrics to the last five minutes but leaves existing `healthScores` entries for functions absent from the new aggregate. Stale `CRITICAL` or `CIRCUIT_OPEN` scores can keep throttling a recovered or idle function indefinitely.
  - confidence: 0.82
- **[P2] Preserve same-origin microphone permission:** `src/gateway/http-common.ts:18`
  - body: Current main deliberately sets `Permissions-Policy` to `microphone=(self)` and tests that behavior. This shared header change switches to `microphone=()`, which can block same-origin gateway pages that need browser microphone access, such as realtime voice UI flows.
  - confidence: 0.8
