## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Enforce the loopback cap before normal lockout:** `src/gateway/auth-rate-limit.ts:214-215`
  - body: `recordFailure()` still sets `lockedUntil` at `maxAttempts` for loopback clients and then stops recording while locked, while `check()` continues allowing loopback until attempts reach `maxAttempts * 10`. The counter never reaches the intended cap, so localhost brute-force attempts remain effectively unbounded.
  - confidence: 0.9
- **[P2] Reject malformed webhook signatures without throwing:** `src/security/shield/webhook-dispatch.ts:63`
  - body: `verifyWebhookSignature()` compares hex string lengths before decoding and then calls `timingSafeEqual()` on decoded buffers. A same-length non-hex signature can decode to a shorter buffer and throw instead of returning `false`.
  - confidence: 0.86
- **[P2] Recompute escalation when circuits recover:** `src/security/shield/gateway-shield.ts:174-175`
  - body: `recordSuccess()` stores the updated circuit but never recomputes `this.escalation`. After an open circuit closes, `getStatus()` and `getSummary()` can keep reporting a pause state until another failure occurs.
  - confidence: 0.84
- **[P2] Expire health scores that leave the metrics window:** `src/security/shield/gateway-shield.ts:301`
  - body: `computeHealthScores()` filters metrics to the last five minutes but only updates functions present in the new aggregate. Functions with no recent metrics can keep old `CRITICAL` or `CIRCUIT_OPEN` scores and continue being throttled from stale state.
  - confidence: 0.82
- **[P2] Select the latest health bucket per function:** `src/security/shield/gateway-shield.ts:287`
  - body: `aggregateByFunctionAndMinute()` returns one bucket per function and minute, but this loop writes scores under only the function name. If metric arrival order is not chronological, an older minute can overwrite a newer score and make throttling depend on insertion order.
  - confidence: 0.78
- **[P2] Preserve same-origin microphone permission:** `src/gateway/http-common.ts:18`
  - body: Current main intentionally allows `microphone=(self)` because same-origin Control UI voice paths need browser microphone access. This branch changes it to `microphone=()`, which would regress that tested behavior.
  - confidence: 0.77
