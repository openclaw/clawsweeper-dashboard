## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Enforce the loopback 10x threshold in the lockout path:** `src/gateway/auth-rate-limit.ts:214-215`
  - body: `recordFailure()` still sets the normal `maxAttempts` lockout for loopback clients, but `check()` returns from `isExempt()` before honoring `lockedUntil` while the count is below `maxAttempts * 10`. Once the normal lockout is set, further failures stop being appended, so localhost failures remain allowed indefinitely instead of reaching the intended 10x cap.
  - confidence: 0.9
- **[P2] Reject malformed webhook signatures without throwing:** `src/security/shield/webhook-dispatch.ts:63`
  - body: `verifyWebhookSignature()` checks hex string lengths before decoding, then passes decoded buffers to `timingSafeEqual()`. A same-length non-hex signature decodes to a shorter buffer and can throw instead of returning `false`, turning invalid input into an exception path.
  - confidence: 0.86
- **[P2] Refresh escalation state when circuits recover:** `src/security/shield/gateway-shield.ts:174-175`
  - body: `recordSuccess()` stores the updated circuit but never recomputes `this.escalation`. After an open circuit closes, `getStatus()` and `getSummary()` can keep reporting a pause state until another failure happens.
  - confidence: 0.84
- **[P2] Drop stale health scores when metrics leave the window:** `src/security/shield/gateway-shield.ts:301`
  - body: `computeHealthScores()` filters metrics to the last five minutes but only updates functions present in the new aggregate. Functions with no recent metrics keep old `CRITICAL` or `CIRCUIT_OPEN` scores, so `shouldThrottle()` can keep throttling idle or recovered functions from stale state.
  - confidence: 0.82
- **[P2] Collapse health metrics by function before storing scores:** `src/security/shield/gateway-shield.ts:287`
  - body: `aggregateByFunctionAndMinute()` returns one bucket per function and minute, but this loop writes each bucket under only the function name. If metric arrival order is not chronological, an older minute can overwrite a newer score and make throttling decisions depend on insertion order.
  - confidence: 0.78
- **[P2] Preserve same-origin microphone permission:** `src/gateway/http-common.ts:18`
  - body: Current main intentionally allows `microphone=(self)` for the gateway because the Control UI voice path needs same-origin microphone access. Changing the shared header to `microphone=()` would regress that behavior and contradict the existing unit and fuzz tests.
  - confidence: 0.77
