## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Wire the Shield into the gateway runtime:** `src/security/shield/index.ts:120-121`
  - body: `GatewayShield` is only exported from the new barrel and no changed gateway startup, request, auth, or WebSocket path imports or constructs it. As written, the circuit breaker, anomaly detection, health scoring, escalation, and WS validation modules never run in production despite the PR presenting them as a runtime hardening layer.
  - confidence: 0.85
- **[P1] Enforce the loopback cap before normal lockout:** `src/gateway/auth-rate-limit.ts:214-215`
  - body: Loopback failures still enter the normal `maxAttempts` lockout path and then stop being recorded while locked, while `check()` keeps allowing loopback until `maxAttempts * 10`. The counter therefore never reaches the intended cap, so localhost brute-force attempts remain effectively unbounded.
  - confidence: 0.9
- **[P2] Reject malformed webhook signatures without throwing:** `src/security/shield/webhook-dispatch.ts:63`
  - body: `verifyWebhookSignature()` compares hex string lengths before decoding, then passes decoded buffers to `timingSafeEqual()`. A same-length non-hex signature can decode to a shorter buffer and throw instead of returning `false`, turning invalid input into an exception path.
  - confidence: 0.86
- **[P2] Recompute escalation when circuits recover:** `src/security/shield/gateway-shield.ts:174-175`
  - body: `recordSuccess()` stores the updated circuit but never recalculates `this.escalation`. After an open circuit closes, `getStatus()` and `getSummary()` can continue reporting a pause state until a later failure happens.
  - confidence: 0.84
- **[P2] Collapse health buckets before storing scores:** `src/security/shield/gateway-shield.ts:286-287`
  - body: `aggregateByFunctionAndMinute()` returns buckets keyed by function and minute, but this loop stores them by function name only. If metric arrival order is not chronological, an older minute can overwrite a newer score and make throttling depend on insertion order.
  - confidence: 0.78
- **[P2] Expire stale health scores without recent metrics:** `src/security/shield/gateway-shield.ts:301`
  - body: `computeHealthScores()` filters metrics to the last five minutes but only updates functions present in the new aggregate. Functions that leave the window can retain stale `CRITICAL` or `CIRCUIT_OPEN` scores and keep being throttled after recovery or idleness.
  - confidence: 0.82
- **[P2] Preserve same-origin microphone permission:** `src/gateway/http-common.ts:18`
  - body: Current main intentionally allows `microphone=(self)` and has tests for that behavior. This branch changes the default Permissions-Policy to `microphone=()`, which would block same-origin Control UI voice paths again.
  - confidence: 0.77
