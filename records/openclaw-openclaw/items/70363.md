## Summary

Keep this draft PR open. Current main still lacks the proposed generation fence, the related interruptibility issue remains open, and the PR is an active partial implementation rather than obsolete cleanup; the only concrete patch issue I found is a non-blocking test weakness that should be addressed during maintainer review.

## What This Changes

The PR adds per-session run-generation state, stamps reply operations with a captured generation, invalidates generation on abort paths, adds an optional block-reply stale-run fence, wires that fence into agent-runner execution, and adds focused tests.

## Best Possible Solution

Keep this draft open for maintainer review against the related interruptibility issue. The best end state is either a maintained generation-fence foundation with a corrected delivery-fence test and explicit follow-up tracking for deferred surfaces, or a maintainer decision to use a different unified cancellation architecture.

## Reproduction Assessment

Yes, partially: static source inspection of current main gives a high-confidence reproduction of the missing generation registry, ReplyOperation stamp, abort invalidation, and block-delivery fence. A live timing race was not run because this sweep is read-only, but the absent paths are concrete.

## Solution Assessment

Unclear/partial: the PR is a plausible narrow foundation, but it only wires one real emission site and leaves several guarantees from the related issue to follow-up work. The safer maintainer path is to fix the weak delivery-fence test and decide whether to land this as an explicit foundation or request a broader cancellation design.
