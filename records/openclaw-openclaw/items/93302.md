## Summary

Keep open: this PR fixes the earlier over-broad unit-path guard, but the new unit-missing fallback can still redirect sudo-to-root flows from the sudo caller's missing unit to root's bare `--user` manager. It needs a narrower root-owned stale-sudo predicate and live Linux systemd proof before merge.

## What This Changes

The PR adds a unit-missing fallback from sudo-to-root machine-scope `systemctl --user` calls to bare `--user`, plus a regression test for stale `SUDO_USER` restart behavior.

## Best Possible Solution

Keep the machine-scope default, add a root-owned stale-sudo fallback only when the current root manager is clearly the intended owner, and prove both root-owned and sudo-caller flows on live Linux.

## Reproduction Assessment

Yes for source-level reproduction: current main prefers non-root `SUDO_USER` under effective root, and PR head now falls back to bare `--user` on unit-missing. I did not run a live systemd reproduction in this read-only review.

## Solution Assessment

No. The fallback is a plausible mitigation for the root-owned stale-sudo report, but the best fix should preserve sudo caller scope unless root's own service is clearly the intended target.
