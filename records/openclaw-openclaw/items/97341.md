## Summary

Keep open. Current main and v2026.6.10 cover basic Slack thread session isolation and thread-history context, and the new context-engine workaround is plausible for prompt additions, but the requested per-thread prompt, skills, persona/tool, and routing customization semantics still need maintainer API/product design.

## What This Changes

Request Slack thread-aware context customization: isolate Slack threads and support per-thread history, prompt/skills/persona/tool, and routing behavior.

## Best Possible Solution

Define a maintainer-approved Slack plugin or SDK contract for thread-level overrides, reuse existing thread session keys and context-engine hooks where they fit, and keep business-specific rules in a plugin or ClawHub package.

## Solution Assessment
