Full review comments:

- **[P2] Mark fallback text as text in the transport path:** `src/agents/anthropic-transport-stream.ts:318-320`
  - body: `hasTextBlock` is only set for original text blocks. A resource_link/audio-only result now pushes a fallback text block here, but then still hits the later `if (!hasTextBlock)` branch and prepends `(see attached image)`, so the fixed path sends a false image hint before the actual resource text. Set the flag when fallback text is emitted or compute text presence from `blocks` like the provider converter.
  - confidence: 0.9
- **[P2] Do not stringify audio payloads into tool-result text:** `src/llm/providers/anthropic.ts:128`
  - body: MCP `AudioContent` carries base64 `data`; the generic JSON fallback will copy the full audio payload into Anthropic text tool results. For non-image audio results this can flood or overflow the prompt while giving the model no usable media, and it differs from the linked expected compact audio marker. Add an explicit audio branch, mirrored in both helpers, before `JSON.stringify`.
  - confidence: 0.84
