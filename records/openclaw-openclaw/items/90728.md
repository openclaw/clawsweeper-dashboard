## What This Changes

The PR converts bundle-MCP tool-result `ContentBlock` values into OpenClaw text/image `AgentToolResult.content` blocks and adds regression tests for resource, resource_link, audio, valid image, and malformed image cases.

## Best Possible Solution

Land this boundary normalization with the focused regression tests after normal maintainer gates; avoid duplicating provider-specific conversions unless a separate shipped-history repair is proven necessary.

## Reproduction Assessment

Yes. Current main source shows MCP content is cast into a text/image tool-result contract, while the MCP SDK allows resource_link/resource/audio blocks and Anthropic conversion can treat non-text blocks as images when an image is present; the PR body also supplies before/after real-runtime output.

## Solution Assessment

Yes. Normalizing at the bundle-MCP materialization boundary is the narrowest maintainable fix because it protects every provider that consumes `AgentToolResult.content` instead of patching one downstream converter.
