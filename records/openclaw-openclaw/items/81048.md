## Summary

Keep this PR open, but the current patch should not merge as-is: the useful Claude CLI resume/parser direction is plausible, yet the new deterministic transcript path duplicates a narrower workspace encoding than current main already uses for Claude project directories, and the latest v5 branch still lacks inspectable after-fix live proof.

## What This Changes

The PR changes Claude CLI resume validation to compute a workspace/session JSONL path with a 250 ms grace rescan, threads `workspaceDir` into both resume checks, and makes `parseCliJsonl` capture the last Claude `session_id`.

## Best Possible Solution

Extract or share one canonical Claude project-directory resolver for doctor and transcript probing, keep the last-wins parser fix if it remains covered, and require fresh redacted live Claude CLI proof for the latest branch before merge.

## Reproduction Assessment

Yes, at source level: current main returns false for missing or header-only Claude project transcripts and maps that false result to `transcript-missing` on resume. I did not run a live Claude CLI repro in this read-only review.

## Solution Assessment

No for the current patch: the deterministic-path direction and last-wins parser fix are plausible, but the path resolver should use the existing canonical workspace normalization rather than duplicating a narrower encoding rule.
