## Summary

Keep open: the Claude CLI context-loss bug is still real on current main and this PR contains useful work, but the current patch still has a blocking resolver mismatch in its deterministic transcript path.

## What This Changes

The PR changes Claude CLI resume transcript validation to probe a deterministic workspace/session JSONL path with a 250 ms grace retry, threads `workspaceDir` into resume checks, logs misses, and makes `parseCliJsonl` capture the last Claude `session_id`.

## Best Possible Solution

Share one canonical Claude project-dir resolver between doctor and transcript probing, keep the last-wins parser fix, and coordinate the final Claude CLI resume-validity shape with the overlapping draft PR.

## Reproduction Assessment

Yes, at source level: current main returns false when the Claude project JSONL exists but lacks an assistant record, and both resume callers convert that false result into `transcript-missing`. The contributor also supplied live Claude CLI logs showing the field symptom and the v5 soak result.

## Solution Assessment

No for the current patch: the deterministic-path direction and last-wins parser fix are reasonable, but the path helper should share the canonical Claude project-dir resolver before replacing the broad scan.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Reuse the canonical Claude project-dir resolver:** `src/agents/command/attempt-execution.helpers.ts:118-119`
  - body: The new deterministic probe encodes raw `workspaceDir` with only character replacement. Please share that resolver here instead of duplicating a narrower rule.
