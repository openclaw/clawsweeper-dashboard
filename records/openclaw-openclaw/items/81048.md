## What This Changes

The PR changes the Claude CLI transcript probe to retry once after 150ms when a matching JSONL exists without an assistant message, logs negative probe diagnostics, and adds focused Vitest coverage.

## Best Possible Solution

Land the bounded Claude CLI-only retry and diagnostics once maintainer review and the remaining CI checks are green, leaving the linked bug issue to close through the PR merge path.

## Reproduction Assessment

Yes, at source level: on current main, a matching Claude project JSONL that exists but lacks an assistant message makes the helper return false, and both resume callers treat that false result as `transcript-missing`. The PR body also supplies live before/after journal logs for the same race using an equivalent runtime patch.

## Solution Assessment

Yes, the proposed one-shot retry is the narrowest maintainable fix because it is gated on `fileExists`, preserves the fast no-session path, and keeps the policy inside the Claude CLI transcript probe instead of broadening session-reset behavior.
