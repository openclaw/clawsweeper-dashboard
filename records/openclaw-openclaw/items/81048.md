## Summary

Keep open: the bug is valid and the PR is aimed at the right Claude CLI resume path, but current PR head still computes the Claude project directory differently from the existing resolver and needs fresh after-fix proof for the latest v4 implementation.

## What This Changes

The PR changes Claude CLI transcript reuse to compute an expected JSONL path from workspace/session data, add a 250 ms grace rescan and warn logging, pass `workspaceDir` through callers, and update focused tests.

## Best Possible Solution

Extract or share one Claude project-dir resolver for doctor and transcript probing, keep the short grace rescan, cover symlink/long-path cases, then land with fresh redacted live Claude CLI proof.

## Reproduction Assessment

Yes, at source level: current main returns false for missing or header-only Claude transcript files, and both resume paths convert that false result into `transcript-missing`. The PR discussion adds live logs showing that path causing context loss, though I did not run a live Claude CLI setup in this read-only review.

## Solution Assessment

No for the current patch: deriving a deterministic transcript path is a good direction, but the implementation should reuse the existing canonical Claude project-dir resolver before it is the narrow maintainable fix. Fresh v4 real-behavior proof is also still needed.
