## Summary

Keep open: current main still has the Claude CLI transcript false-negative path and this PR addresses a real bug, but the latest patch still diverges from the existing Claude project-dir resolver and lacks fresh after-fix proof for the current v5 head.

## What This Changes

The PR changes Claude CLI resume validation to probe a deterministic workspace/session JSONL path with a 250 ms grace window, threads `workspaceDir` through callers, logs misses, and switches JSONL parsing to last-wins `session_id` handling with regression tests.

## Best Possible Solution

Share one Claude project-dir resolver between doctor and resume probes, keep the last-wins Claude session id parsing, and land only after current-head live proof and conflict resolution.

## Reproduction Assessment

Yes at source level: current main returns false for a matching Claude JSONL without assistant records, and both resume callers map that false result to `transcript-missing`. The live flush timing still rests on the reporter's gateway logs rather than a reproduction in this read-only review.

## Solution Assessment

No for the current patch: deterministic path probing and last-wins session capture are the right direction, but the path helper should share the existing canonical Claude project-dir contract before merge.
