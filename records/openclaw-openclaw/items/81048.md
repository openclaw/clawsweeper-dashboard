## Summary

Keep open: the PR is a focused fix for a real Claude CLI resume bug, but the current v4 patch can still false-miss valid transcripts because it computes the Claude project path differently from the existing resolver, and the latest implementation still lacks after-fix live proof.

## What This Changes

The PR changes the Claude CLI transcript probe to compute a deterministic workspace/session JSONL path, adds a 250 ms grace rescan and miss logging, passes `workspaceDir` from both resume callers, and updates focused tests.

## Best Possible Solution

Share or extract the existing Claude project-dir resolver for transcript probing, keep the bounded grace window and diagnostic miss log, add symlink/long-path regression coverage, and require fresh redacted live v4 proof before merge.

## Reproduction Assessment

Yes at source level: current main returns false for a matching Claude transcript that is absent or header-only at probe time, and both resume paths convert that false result into `transcript-missing` invalidation. I did not run a live Claude CLI timing reproduction in this read-only review.

## Solution Assessment

No for the current patch: deriving a deterministic transcript path is a good direction, but this implementation should reuse the existing canonical Claude project-dir resolver before it is the narrow maintainable fix.
