## Summary

Keep open: current main has the runner-side silent-empty gate, but cron still has no per-job `acceptSilentStop` contract and does not forward the existing runner flag for isolated agent-turn jobs.

## What This Changes

Request an opt-in cron job flag that treats clean `stopReason=stop` runs with zero payloads as successful silent cron executions.

## Best Possible Solution

Add a documented opt-in cron agent-turn flag that reuses the existing silent-empty runner gate, records an ok/no-delivery run, and leaves default empty-response, timeout, provider-error, and tool-error behavior unchanged.

## Reproduction Assessment

Yes. A focused test can mock an isolated cron agent turn ending in a clean empty assistant stop with zero payloads; current cron has no accepted job field and does not pass the runner silence flag, so the path remains classified as an incomplete/error turn.

## Solution Assessment

Yes. Reusing `allowEmptyAssistantReplyAsSilent` behind an explicit cron job contract is the narrowest maintainable fix because the runner helper already guards tool errors, side effects, timeouts, and error stop reasons.
