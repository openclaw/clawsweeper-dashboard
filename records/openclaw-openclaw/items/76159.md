## Summary

Keep open: current main still lacks the requested per-job `acceptSilentStop` cron contract, and the open related PR only covers a narrower `delivery.mode="none"` path that is not merged.

## What This Changes

Request to add an opt-in cron job flag that treats clean `stopReason="stop"` runs with zero payloads as successful silent output instead of cron errors.

## Best Possible Solution

Define a documented opt-in cron success contract that accepts intentional empty stops only in the approved job modes while preserving default incomplete-turn protection.

## Reproduction Assessment

Yes. Source inspection shows cron has no `acceptSilentStop` field, does not pass the existing silent-empty runner option, and still increments `consecutiveErrors` for error outcomes; I did not run a live cron because this review is read-only.

## Solution Assessment

Unclear. A per-job flag is plausible, but maintainers need to choose between a broad `acceptSilentStop` contract, the narrower `delivery.mode="none"` path in the open PR, or a different command/no-output cron mode.
