## Summary

Keep open. Current main has adjacent silent-token and runner-level support, but cron still lacks a per-job public contract and does not pass the existing empty-assistant silent gate into embedded cron runs.

## What This Changes

Request an opt-in cron job flag that treats a clean `stopReason=stop` run with zero payloads as a successful silent cron outcome instead of an error/backoff event.

## Best Possible Solution

Add an additive boolean on `agentTurn` cron payloads, using the existing runner guard for safety, and leave all default empty-response, timeout, provider-error, and tool-error behavior unchanged.

## Reproduction Assessment

Yes. A focused source-level test can mock an isolated cron embedded run ending with a clean empty assistant stop and zero payloads; current cron does not set the runner flag, so the finalizer still produces an error outcome that feeds cron backoff.

## Solution Assessment

Yes. Reusing `allowEmptyAssistantReplyAsSilent` behind an opt-in cron field is the narrowest maintainable path because the runner helper already contains the important safety gates.
