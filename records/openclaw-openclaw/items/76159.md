## Summary

Keep open: current main and the latest release still lack the requested per-job silent-stop contract, and the related open PR only covers a narrower no-delivery path while remaining unmerged and conflicting.

## What This Changes

The issue requests an opt-in cron job flag that treats clean `stopReason=stop` runs with zero payloads as successful silent cron completions.

## Best Possible Solution

Define one public cron silent-success contract, then align schema, runner classification, docs, and regression tests around that contract without silently weakening visible-session incomplete-turn detection.

## Reproduction Assessment

Yes at source level: a no-output isolated cron path can reach `runEmbeddedAgent` without the silent-empty flag, and error results increment cron `consecutiveErrors`. I did not run a live cron/provider reproduction in this read-only review.

## Solution Assessment

Unclear. The requested flag is plausible, but the best fix needs maintainer choice because existing docs already define `delivery.mode: "none"` and exact `NO_REPLY` silence, while current runner guards deliberately keep post-tool empty stops on the incomplete-turn path.
