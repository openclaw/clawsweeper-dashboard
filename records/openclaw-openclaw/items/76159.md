## Summary

Keep open. Current main has the runner gate needed for opt-in silent empty stops, but isolated cron jobs still do not expose or forward a per-job policy; the related #68452 fix only covers exact `NO_REPLY` tool-result paths.

## What This Changes

Request a per-job cron option that treats clean zero-payload assistant `stop` turns as successful silent runs.

## Best Possible Solution

Add a documented opt-in cron `agentTurn` silent-empty policy that flows through types, protocol schema, creation/edit surfaces, docs, and the isolated executor into the existing runner gate while preserving the current default failure behavior.

## Reproduction Assessment

Yes by source inspection: clean empty stops are accepted only when the runner receives `allowEmptyAssistantReplyAsSilent`, and isolated cron currently has no field or forwarding path for that option.

## Solution Assessment

Yes directionally: reusing the existing runner gate behind an explicit cron opt-in is the narrow maintainable path, but the public cron contract shape needs maintainer signoff.
