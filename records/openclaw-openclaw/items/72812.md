## Summary

Keep open for maintainer merge review: the patch appears correct and current main still has the shared-client startup teardown path, but the PR deliberately changes Codex auth/session lifecycle for spawned helper runs so maintainers should accept that risk before merge.

## What This Changes

The branch routes Codex app-server startup for non-empty `spawnedBy` helper runs through isolated clients, cleans those clients up on failure/abort/final exit, adds focused regressions, and records a changelog entry.

## Best Possible Solution

Land this PR, or an equivalent narrow spawned-helper isolation fix, after maintainer acceptance of the per-helper Codex app-server lifecycle cost; close the linked bug only after the fix reaches main.

## Reproduction Assessment

Yes, source-backed rather than personally live-reproduced: the linked issue gives concrete invalid-token steps/logs, and current main routes startup through the shared client before error cleanup can close it.

## Solution Assessment

Yes, the proposed fix is the narrow maintainable path: isolate only non-empty `spawnedBy` helper startup, preserve shared-client reuse for normal and subagent-key-only runs, and use the existing isolated-client helper with explicit cleanup.
