## Summary

Keep open for maintainer merge review: current main still routes spawned Codex helper startup through the shared app-server client, while this PR adds the scoped isolation and cleanup path with focused regression coverage and supplied terminal proof. ## What This Changes

This PR isolates Codex app-server startup for runs with non-empty `spawnedBy`, closes isolated clients during error/abort/final cleanup, adds focused regression tests, and adds a changelog entry.

## Best Possible Solution

Land the scoped `spawnedBy` isolation fix after maintainer acceptance of the isolated child-client lifecycle, keeping the boundary narrow and preserving the regression guard for subagent-shaped session keys without `spawnedBy`.

## Solution Assessment

Yes: using the existing isolated-client helper only when `spawnedBy` is non-empty is the narrowest maintainable fix I found for keeping helper auth failures local. The remaining question is lifecycle-risk acceptance, not a clear code defect.
