## What This Changes

The PR routes Codex app-server runs with non-empty `spawnedBy` through isolated startup clients, adds abort/error/final cleanup for those clients, and covers the behavior with focused tests plus a changelog entry.

## Best Possible Solution

Land the scoped `spawnedBy` isolation fix once maintainers accept the helper-run lifecycle boundary, preserving the regression guard that session-key shape alone does not trigger isolation.
