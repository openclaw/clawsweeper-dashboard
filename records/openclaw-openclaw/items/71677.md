## Summary

Keep open. Current main and v2026.6.10 still dispatch and dedupe Feishu card actions, but they do not return an instant WebSocket callback-response card/toast payload or define an ocf1 disable-state contract.

## What This Changes

The issue requests instant Feishu card button feedback by returning a disabled/update card payload from card.action.trigger and adding an ocf1 disable-state field.

## Best Possible Solution

Define one Feishu-plugin-owned typed callback response and ocf1 disable/update-state contract, then implement immediate disabled-card feedback with focused tests and live Feishu proof.

## Reproduction Assessment

Yes, at source level. A valid Feishu card.action.trigger reaches the current handler, but current main resolves through Promise<void> command dispatch with no returned WebSocket callback payload or disable-state contract.

## Solution Assessment

Unclear until maintainers choose the API shape. The maintainable direction is a typed Feishu callback-response contract coordinated with ocf1 metadata, not an ad hoc JSON return from the handler.
