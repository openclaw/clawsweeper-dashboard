## Summary

Keep open. Current main still routes Feishu card-action callbacks but does not return an immediate WebSocket callback payload for a transitional disabled card or toast; the merged Schema 2 work only fixed callback parsing, not this UX feature.

## What This Changes

Support immediate Feishu card callback responses that can disable clicked buttons while agent processing continues.

## Best Possible Solution

Implement a Feishu plugin-owned callback response that returns a typed transitional card/toast immediately, preserves dedupe and retry semantics while command dispatch continues, and extends ocf1/card helpers only as needed.

## Reproduction Assessment

Yes, at source level. A valid structured card.action.trigger callback reaches the registered handler, but current main resolves with undefined while the quick-action path awaits command dispatch and the tests do not assert any callback response payload.

## Solution Assessment

Yes. A Feishu plugin-owned handler return path is the narrowest maintainable direction; the merged https://github.com/openclaw/openclaw/pull/71787 parser fix should remain separate because it solved callback identity parsing, not immediate disabled-card feedback.
