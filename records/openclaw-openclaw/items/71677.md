## Summary

Keep open. Current main still accepts and routes Feishu card button callbacks, but it does not return an immediate card-action response payload for disabled/loading button feedback; the related Schema 2.0 work only covers callback parsing/routing.

## What This Changes

Request immediate Feishu interactive-card callback responses that disable or transition clicked buttons before the agent finishes processing.

## Best Possible Solution

Implement this in the Feishu plugin as a typed immediate card-action response path that returns a transitional disabled/loading card and optional toast while the normal command dispatch continues asynchronously.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to invoke the registered `card.action.trigger` handler with a valid structured quick-action payload and observe that it resolves to `undefined` while dispatch/dedupe still run.

## Solution Assessment

Yes. A plugin-owned handler return path is the narrowest maintainable solution because the SDK already carries handler return values through the callback response; core changes are only needed if a missing plugin contract appears during implementation.
