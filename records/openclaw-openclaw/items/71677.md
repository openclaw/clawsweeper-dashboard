## Summary

Keep open. Current main fixed adjacent Schema 2 identity parsing, but it still does not return a Feishu WebSocket callback response or expose a disable/update-state field for immediate button feedback.

## What This Changes

Support returning an immediate Feishu card.action.trigger WebSocket response so clicked Schema 2.0 card buttons can disable or show feedback before agent processing finishes.

## Best Possible Solution

Define one Feishu-plugin-owned typed callback response and `ocf1` disable/update-state contract, then prove immediate disabled-card feedback with focused tests and redacted live Feishu WebSocket proof.

## Reproduction Assessment

Yes, at source level. A valid Feishu `card.action.trigger` reaches the current handler, but current main resolves through `Promise<void>` command dispatch with no returned WebSocket callback payload or disable-state contract.

## Solution Assessment

Yes on direction, but not safe for autonomous implementation yet. The maintainable path is one typed Feishu callback-response contract coordinated with `ocf1` card metadata, not a one-off return payload.
