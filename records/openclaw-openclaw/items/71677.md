## Summary

Keep open. Current main and the latest release still return no `card.action.trigger` response, but the requested instant disabled-button update is a Feishu plugin UX/protocol feature that needs maintainer product direction before implementation.

## What This Changes

Request instant Feishu card-action callback responses that immediately disable or update clicked buttons while the agent continues processing.

## Best Possible Solution

Implement a Feishu-owned instant acknowledgement path that returns a transitional disabled card from `card.action.trigger`, preserves dedupe and final update semantics, and documents/tests any new `ocf1` field before generated cards depend on it.

## Solution Assessment

Unclear. Returning a transitional disabled card is plausible and matches the SDK plumbing, but maintainers still need to choose the `ocf1`/card-schema contract and retry/final-update behavior.
