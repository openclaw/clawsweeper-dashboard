## Summary

Keep open. Current main and v2026.6.10 still route and dedupe Feishu card actions, but they return no WebSocket callback-response payload and define no ocf1 disable-state/update contract, so the central request remains missing.

## What This Changes

The issue requests immediate Feishu card.action.trigger WebSocket callback responses that disable clicked buttons or show transitional feedback while the agent command continues asynchronously.

## Best Possible Solution

Define a Feishu-plugin-owned typed callback-response payload and ocf1 disable/update-state contract, then implement immediate disabled-card feedback with focused lifecycle tests and live Feishu proof.

## Reproduction Assessment

Yes, at source level. A valid Feishu card.action.trigger reaches a handler that resolves through Promise<void> command dispatch with no returned WebSocket callback payload or disable-state contract.

## Solution Assessment

Unclear until maintainers choose the API shape. The maintainable direction is a typed Feishu callback-response contract coordinated with ocf1 metadata, not an ad hoc JSON return from the handler.
