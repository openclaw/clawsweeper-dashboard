## Summary

Keep open. Current main still routes Feishu card actions through the existing async command path without returning a WebSocket callback payload, and the overlapping open Feishu card-update PRs do not yet provide a canonical instant callback response or disable-state contract.

## What This Changes

Request instant Feishu card updates from the WebSocket callback response so clicked buttons can show disabled feedback before agent processing completes.

## Best Possible Solution

Implement this in the Feishu plugin as the canonical card-action feedback path: return a typed instant callback response, define the `ocf1` disable/update-state contract, preserve token lifecycle semantics, and coordinate or replace the overlapping card-update PRs.

## Reproduction Assessment

Yes, at source level. A valid `card.action.trigger` payload reaches the handler, but current main resolves with `undefined` while command dispatch runs and the existing tests only prove routing, reply targeting, and dedupe behavior.

## Solution Assessment

Yes on direction, but not yet on implementation path. Returning a Feishu-plugin-owned callback response is the narrowest way to get instant feedback, while the response schema and overlap with open card-update PRs need maintainer coordination.
