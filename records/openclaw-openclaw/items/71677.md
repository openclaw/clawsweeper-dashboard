Keep open. Current main still handles Feishu card-action callbacks through a void async command path, and the related in-place card-update PRs are still open rather than merged as a canonical instant disable/update contract.

## What This Changes

Support instant Feishu card button feedback by returning a transitional disabled-card response from `card.action.trigger` instead of waiting for later agent processing.

## Best Possible Solution

Define one Feishu-plugin-owned typed callback response and `ocf1` disable/update-state contract, coordinate it with the open card-update PRs, and prove immediate button feedback with focused tests plus redacted live Feishu proof.

## Solution Assessment

Yes on product direction, but the implementation path still needs maintainer coordination. The maintainable fix is a single typed Feishu callback-response contract rather than an uncoordinated one-off patch beside the open card-update PRs.
