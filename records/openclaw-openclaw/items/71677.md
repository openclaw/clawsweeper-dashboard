## Summary

Keep open. Current main still parses and dispatches Feishu card actions but returns no WebSocket callback payload, and the merged Schema 2 fix only broadened callback identity parsing; this remains a narrow Feishu plugin UX feature.

## What This Changes

Add a Feishu WebSocket card-action callback return path that immediately returns a toast or disabled transitional card while dispatching the command asynchronously.

## Best Possible Solution

Implement this inside the Feishu plugin as a typed callback response path with transitional button/card state, focused tests, and no core SDK changes unless a missing plugin contract is proven.

## Reproduction Assessment

Yes, at source level. A valid structured `card.action.trigger` callback reaches the registered handler, but current main resolves with `undefined` while the command dispatch path runs and the tests only cover routing/dedupe behavior.

## Solution Assessment

Yes. A Feishu plugin-owned handler return path is the narrowest maintainable solution; the Schema 2 parser fix in https://github.com/openclaw/openclaw/pull/71787 should remain separate because it solved callback parsing, not immediate user feedback.
