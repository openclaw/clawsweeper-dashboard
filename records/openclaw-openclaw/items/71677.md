## Summary

Keep open. Current main still routes Feishu card actions but does not return an immediate WebSocket callback payload, and the merged Schema 2 parser work did not add disabled-card feedback or an ocf1 disable-state contract.

## What This Changes

Request immediate Feishu interactive-card callback responses so clicked buttons can show disabled/loading feedback before the agent finishes processing.

## Best Possible Solution

Implement this inside the Feishu plugin with a typed instant callback response, an ocf1 disable-state field, and focused lifecycle coverage before considering any core SDK seam.

## Reproduction Assessment

Yes, at source level. A valid card.action.trigger payload reaches the registered handler, but current main resolves with undefined while the command path runs, and the tests only prove routing/dedupe behavior.

## Solution Assessment

Yes. A plugin-owned handler return path is the narrowest maintainable solution because the Lark SDK already passes handler return values through the WebSocket response frame.
