## Summary

Keep open. Current main still parses and dispatches Feishu card actions but does not return a WebSocket callback payload, and the related merged Schema 2 work only fixed callback parsing, not instant disabled-card feedback.

## What This Changes

Support Feishu WebSocket card-action callback responses so interactive card buttons can show immediate disabled/loading feedback after a click.

## Best Possible Solution

Implement the immediate callback response inside the Feishu plugin with typed card/button state, focused lifecycle coverage, and no core SDK changes unless a missing plugin seam is proven.

## Reproduction Assessment

Yes, at source level. A valid structured card.action.trigger callback reaches the registered handler, but current main resolves with undefined while the command dispatch path runs and existing tests only cover routing and dedupe behavior.

## Solution Assessment

Yes. A Feishu plugin-owned handler return path is the narrowest maintainable solution; the merged Schema 2 parser fix should remain separate because it solved callback identity parsing, not immediate user feedback.
