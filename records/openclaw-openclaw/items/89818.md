## Summary

Keep open for maintainer review: the patch is a narrow bug fix for three bundled Anthropic-family provider transports, and current main still drops caller-provided stop sequences on those paths. I found no blocking code defect, but merging intentionally changes outbound provider payloads for existing callers who set non-empty stop sequences.

## What This Changes

The PR forwards `StreamOptions.stop` into Anthropic Vertex, Amazon Bedrock, and Bedrock Mantle request builders and adds focused transport tests for set and unset stop sequences.

## Best Possible Solution

Merge the narrow provider-payload fix after required checks complete and maintainers intentionally accept the guarded native stop-field compatibility change; keep OpenAI Responses stop semantics as a separate API-contract decision.

## Reproduction Assessment

## Solution Assessment

Yes. The patch fixes the owner-local request builders, reuses the existing shared Anthropic mapper for Vertex and Mantle, maps Bedrock to its native `inferenceConfig.stopSequences`, and leaves unresolved OpenAI Responses semantics out of scope.
