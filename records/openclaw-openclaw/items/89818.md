## What This Changes

The PR forwards `StreamOptions.stop` into Amazon Bedrock `inferenceConfig.stopSequences` and through Anthropic Vertex and Bedrock Mantle to the shared Anthropic transport, with focused tests for set and unset paths.

## Best Possible Solution

Merge the narrow provider-payload fix after required checks are clean and maintainers accept the guarded native stop-field compatibility change; keep OpenAI Responses stop semantics as a separate API-contract decision.

## Solution Assessment

Yes. The patch fixes the owner-local request builders, reuses the existing shared Anthropic mapper for Vertex and Mantle, maps Bedrock to its native field, and leaves unresolved OpenAI Responses semantics out of scope.
