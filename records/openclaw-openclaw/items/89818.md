## Summary

Keep open; this PR is still necessary because current main and v2026.6.10 still drop non-empty `stop` values in the Amazon Bedrock, Anthropic Vertex, and Bedrock Mantle request builders. ## What This Changes

The branch forwards `StreamOptions.stop` into Amazon Bedrock `inferenceConfig.stopSequences` and through Anthropic Vertex and Bedrock Mantle option bags, with focused tests for set and unset cases.

## Best Possible Solution

Merge the narrow provider-payload fix after required checks are clean and maintainers accept the guarded native stop-field compatibility change; keep OpenAI Responses stop semantics as a separate API-contract decision.

## Solution Assessment

Yes. The PR fixes the owner-local request builders, reuses the existing shared Anthropic mapper for Vertex and Mantle, maps Bedrock to its documented native field, and keeps unresolved OpenAI Responses semantics out of scope.
