## Summary

Keep open: this is a plausible fix area for the linked runtime-routing report, but the PR is not mergeable because the new `agent`/`defaults` policy sources are not propagated to downstream runtime-source types and the external contributor has not supplied after-fix real behavior proof. It is also not a clean duplicate close because related PR https://github.com/openclaw/openclaw/pull/81421 covers the defaults path with green CI/proof, while this PR has the unique per-agent runtime angle noted in review discussion.

## What This Changes

The PR teaches `resolveModelRuntimePolicy()` to read `agents.list[].agentRuntime` and `agents.defaults.agentRuntime`, and updates harness selection tests to expect defaults-level runtime selection.

## Best Possible Solution

Finalize one runtime-resolution contract, propagate its source metadata through harness and agent-list surfaces, update docs/tests to match that contract, and require redacted real CLI or resolver output showing the intended subagent route after the fix.

## Reproduction Assessment

Yes, at source level: current `resolveModelRuntimePolicy()` falls through without reading `agents.defaults.agentRuntime` or `agents.list[].agentRuntime`. I did not run a live Claude billing/auth reproduction in this read-only review.

## Solution Assessment

No, not as written. The PR needs downstream source-type propagation and real behavior proof, and maintainers should settle whether whole-agent runtime policy should override or only fall back behind provider/model policy.
