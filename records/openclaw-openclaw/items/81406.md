## Summary

## What This Changes

This PR adds agent-level and agents.defaults agentRuntime resolution before provider/model runtime fallback and updates selection tests to expect defaults-driven harness selection.

## Best Possible Solution

Land one canonical runtime-policy fix that propagates any new source values through harness metadata, documents the chosen precedence, includes focused regression tests, and supplies redacted real subagent runtime proof.

## Reproduction Assessment

Yes, at source level: current main's resolver falls through without checking agents.defaults.agentRuntime or agents.list[].agentRuntime, and the existing selection test asserts the defaults value is ignored. I did not run a live subagent because this was a read-only review.

## Solution Assessment

No, not yet: the direction is plausible, but this patch does not compile and it changes a documented legacy/ignored runtime path without completing downstream type and metadata propagation. The safer path is to align this with the canonical related work at https://github.com/openclaw/openclaw/pull/81421 and explicitly decide the per-agent/default precedence.
