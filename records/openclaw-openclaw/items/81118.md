## Summary

Keep open: the TDZ fix and supplied live proof are useful, but the second commit likely regresses bundled Codex native subagent task mirroring by resolving the private task runtime with a late native createRequire call that falls outside the plugin-loader alias scope.

## What This Changes

The PR moves Codex app-server client-factory override state into client-factory.ts, adds focused tests, and changes native subagent task runtime resolution and SDK aliasing for npm-installed Codex modules.

## Best Possible Solution

Keep the client-factory leaf split, but make the native task-runtime resolver preserve real task-registry wiring for bundled Codex and fall back only when the private helper is genuinely unavailable.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Source-level yes for the blocking PR regression: native createRequire cannot resolve the private SDK subpath through package exports, while the PR calls it after loader alias scoping has ended; the original TDZ live repro was not rerun in this read-only pass.

## Solution Assessment

Is this the best way to solve the issue? No: moving client-factory state out of run-attempt.ts is narrow, but replacing the static private runtime import with late createRequire silently disables bundled native subagent task mirroring.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Keep aliases in scope when resolving the task runtime:** `extensions/codex/src/app-server/native-subagent-task-mirror.ts:45-47`
  - body: The new default runtime resolver calls a createRequire-created native require after the plugin module has loaded. The private `openclaw/plugin-sdk/codex-native-task-runtime` subpath is intentionally not exported by `package.json`, and the loader's native alias patch is only scoped around the top-level module load, so this catch path will permanently cache the no-op runtime for bundled Codex too. That means native subagent task runs stop being created or updated instead of only tolerating the npm-installed plugin case.
  - confidence: 0.88
