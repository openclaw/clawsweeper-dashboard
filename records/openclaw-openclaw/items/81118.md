## Summary

This PR should stay open: the TDZ fix is useful and now has live proof, but the native task-runtime change appears to disable bundled Codex subagent task mirroring by bypassing the plugin loader alias map.

## What This Changes

The PR moves Codex app-server client factory override resolution into client-factory.ts, changes native subagent task-runtime loading, and expands SDK alias detection for npm-installed @openclaw/codex modules.

## Best Possible Solution

Keep the client-factory split, but make the task-runtime fallback preserve the bundled alias-aware runtime path and only no-op when the private helper is genuinely unavailable in the npm-installed plugin layout.

## Reproduction Assessment

Yes for the introduced task-mirror regression: a native createRequire from the affected file cannot resolve the private SDK subpath, while current tests show that subpath is provided through the plugin loader alias map. The original TDZ report is source-reproducible from current main, but I did not run a live re-entrant app-server failure path.

## Solution Assessment

No as written: moving factory state into client-factory.ts is the right narrow fix, but the task-runtime fallback should not turn off bundled Codex subagent task mirroring by leaving the alias-aware loader path.
