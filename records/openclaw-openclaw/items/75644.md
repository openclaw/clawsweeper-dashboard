## Summary

Keep open. Current main still lacks the proposed `plugins.bundledMode` config key and still uses bundled-provider compat modes that can widen runtime loading beyond a restrictive plugin allowlist; the PR is a useful narrow implementation but needs public docs and changelog coverage before merge.

## What This Changes

The PR adds a default-preserving `plugins.bundledMode` config/schema/type surface, tests, and runtime wiring so bundled provider/extractor paths can opt into respecting plugin allowlists, plus a small WhatsApp/test cleanup commit.

## Best Possible Solution

Land the default-preserving `plugins.bundledMode: respect-allow` implementation after adding the public docs row and changelog entry, then keep broader manifest-first runtime-loader cleanup separate.

## Reproduction Assessment

Yes. Source-level reproduction on current main is clear: the strict schema rejects `plugins.bundledMode`, and provider/extractor runtime paths still pass compat modes that can load bundled plugins beyond a restrictive allowlist.

## Solution Assessment

Mostly yes. The config opt-in is the narrow compatible fix because it preserves existing behavior by default while giving operators an allowlist-respecting mode; the safer completion path is docs/changelog coverage plus exact-head validation.
