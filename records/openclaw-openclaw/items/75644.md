## What This Changes

The PR adds a `plugins.bundledMode` config/schema/type surface and routes bundled provider, web provider, and extractor runtime compat behavior through a `respect-allow` opt-in while enabling web-provider loader caching in that mode.

## Best Possible Solution

Land the default-preserving `plugins.bundledMode: "respect-allow"` implementation after adding the required release-note credit and, if maintainers want the config table to be complete, a short plugin docs row; leave the broader manifest-only runtime-loader architecture cleanup for a separate follow-up.

## Solution Assessment

Mostly yes. The opt-in mode is the narrowest maintainable near-term fix because it preserves current bundled-plugin defaults, while the larger manifest-only enumeration approach is a broader architectural follow-up rather than a safer fix for this PR.
