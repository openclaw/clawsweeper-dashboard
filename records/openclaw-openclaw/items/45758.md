## What This Changes

Add YAML/YML as alternative root config files with extension-based parsing while keeping JSON5 as the default.

## Best Possible Solution

Make a maintainer config-contract decision: either keep JSON5-only and strengthen CLI/config.patch editing guidance, or accept YAML with a complete discovery, parsing, validation, reload, write, docs, and test plan.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source, tests, docs, and the latest release show OpenClaw currently treats the root config as JSON5/JSON only.

## Solution Assessment

Unclear. YAML would satisfy the literal request, but the discussion identifies comment-safe mutation as the sharper problem, so maintainers should decide whether the durable solution is a second root config format or stronger config.patch/CLI guidance.
