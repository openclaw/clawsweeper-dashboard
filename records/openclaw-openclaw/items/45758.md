## Summary

Keep open: current main and the latest stable release still expose the root config as JSON/JSON5 only, while the discussion usefully reframes part of the pain toward comment-safe `config.patch`/CLI editing rather than recording a maintainer decision to add or reject a second root config format.

## What This Changes

The issue requests `openclaw.yaml`/`openclaw.yml` support as an alternative root config format while keeping `openclaw.json` as the default.

## Best Possible Solution

Maintainers should make the root config contract explicit: keep JSON5-only and strengthen `openclaw config`/`config patch` guidance, or accept YAML with complete discovery, precedence, parser, validation, write, docs, and regression-test coverage.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source, tests, docs, and the latest stable release show OpenClaw currently treats the root config as JSON/JSON5 only.

## Solution Assessment

Unclear. YAML would satisfy the literal request, but the discussion identifies comment-safe mutation as the sharper problem, so maintainers should decide whether the durable solution is a second root config format or stronger `config patch`/CLI guidance.
