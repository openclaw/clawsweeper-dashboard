## Summary

Keep open. Current main still treats the root OpenClaw config as JSON5/JSON, and the existing `config.patch` / `config set` mutation paths address the comment-preservation discussion without implementing the requested `openclaw.yaml` or `openclaw.yml` loading behavior.

## What This Changes

Add YAML as an alternative root config file format alongside the current JSON5 `openclaw.json` config.

## Best Possible Solution

Make an explicit maintainer decision for the root config contract. If YAML is accepted, implement it in core discovery, parser selection, include semantics, validation, hot reload, write behavior, tests, and docs; if rejected, document the JSON5-only stance and strengthen guidance around `openclaw config set`, Control UI, and `config.patch` for comment-safe scripted edits.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Current-main verification is high confidence: discovery, parsing, include handling, writes, tests, and docs still point to JSON5/JSON config behavior only.

## Solution Assessment

Unclear. YAML support could satisfy the literal request, but the issue discussion identifies comment-safe mutation as the sharper problem, and the narrowest maintainable path may be documenting and improving `config.patch` / CLI config workflows instead of adding a second root config format.
