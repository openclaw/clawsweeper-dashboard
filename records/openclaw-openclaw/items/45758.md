## Summary

Current main and the latest shipped config-path code still do not implement YAML root config loading. The discussion usefully narrowed the practical pain to comment-safe config mutation, but there is no maintainer decision that makes `openclaw.yaml` / `openclaw.yml` support accepted, rejected, implemented, or superseded.

## What This Changes

Add `openclaw.yaml` / `openclaw.yml` as alternate root config files alongside the current JSON5 `openclaw.json` config.

## Best Possible Solution

Make a maintainer decision on the root config contract: either implement YAML across discovery, parsing, includes, validation, reload, writes, docs, and tests, or document the JSON5-only stance while strengthening `config.patch` and CLI config-editing guidance.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source and docs checks show current main still recognizes `openclaw.json` and legacy JSON candidates and parses root config as JSON5.

## Solution Assessment

Unclear. YAML would satisfy the literal request, but the issue discussion shows comment-safe mutation may be better solved through `config.patch`, CLI config commands, and docs rather than expanding the root config contract.
