## Summary

Current main still does not implement YAML root config loading, and the request remains a product-level config contract decision rather than a conservative cleanup close. The discussion usefully narrowed the underlying pain to comment-safe config mutation, and current docs now steer tooling toward CLI/RPC mutation paths, but that does not make the requested `openclaw.yaml` / `openclaw.yml` support implemented or obsolete.

## What This Changes

The issue requests support for `~/.openclaw/openclaw.yaml` and `~/.openclaw/openclaw.yml` as alternatives to the current JSON5 root config file.

## Best Possible Solution

Make a maintainer decision on the root config contract: either implement YAML support across discovery, parsing, includes, validation, reload, writes, docs, and tests, or document the JSON5-only stance while strengthening the `config.patch` / CLI config-editing guidance.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Current-main source and docs verification is high confidence that root config discovery, parsing, tests, and writes still cover JSON5/JSON behavior only.

## Solution Assessment

Unclear. YAML would satisfy the literal request, but the issue discussion shows comment-safe mutation may be better solved through `config.patch`, CLI config commands, and docs rather than adding a second root config format.
