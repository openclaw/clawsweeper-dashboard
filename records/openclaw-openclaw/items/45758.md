## Summary

Keep open: current main and v2026.5.12 still expose a JSON5-only root config contract, and YAML/YML support would be a new config-format product decision rather than conservative cleanup. The discussion usefully narrowed the pain toward comment-safe mutation via CLI/config.patch, but no canonical issue or maintainer decision supersedes the YAML request.

## What This Changes

Request support for `openclaw.yaml` or `openclaw.yml` as alternative root config files alongside the current JSON5 `openclaw.json`.

## Best Possible Solution

Make a maintainer config-contract decision: either keep JSON5-only and strengthen CLI/config.patch editing guidance, or accept YAML with a complete discovery, parsing, validation, reload, write, docs, and test plan.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source, docs, tests, and release inspection show current OpenClaw recognizes JSON root config candidates and parses the root config as JSON5.

## Solution Assessment

Unclear. YAML would satisfy the literal request, but the discussion identifies comment-safe mutation as the sharper problem, so maintainers should decide whether the durable fix is a second root config format or stronger config.patch/CLI guidance.
