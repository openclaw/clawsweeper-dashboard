## Summary

Keep open. Current main remains JSON5-only for root config discovery, parsing, patch inputs, includes, and writes; the newer CLI/RPC guidance helps with comment-safe mutation but does not implement YAML root config support. This is a product-level config contract decision, not a conservative cleanup close.

## What This Changes

Support `~/.openclaw/openclaw.yaml` and `~/.openclaw/openclaw.yml` as alternatives to the existing JSON5 `openclaw.json` root config.

## Best Possible Solution

Make a maintainer decision on the root config contract: either implement YAML support across discovery, parsing, includes, validation, reload, writes, docs, and tests, or document the JSON5-only stance while continuing to steer programmatic edits through CLI/RPC patch paths.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source and docs inspection gives high confidence that current main still recognizes JSON5/JSON root config behavior only.

## Solution Assessment

Unclear. YAML would satisfy the literal request, but the discussion shows the underlying comment-preservation pain may be better addressed through `config.patch`, CLI config commands, and docs instead of a second root config format.
