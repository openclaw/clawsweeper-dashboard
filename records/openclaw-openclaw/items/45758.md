## Summary

Keep open: current main and the latest release still use a JSON5-only root config contract, and adding YAML/YML support would be a new config-format product decision rather than conservative cleanup.

## What This Changes

The issue requests support for `~/.openclaw/openclaw.yaml` and `~/.openclaw/openclaw.yml` as alternatives to the current JSON5 root config file.

## Best Possible Solution

Make a maintainer config-contract decision: either keep JSON5-only and document safe `openclaw config`/`config.patch` editing guidance, or accept YAML with a complete discovery, parsing, validation, reload, write, docs, and test plan.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source, tests, docs, and the latest release show OpenClaw currently recognizes JSON config candidates and parses root config as JSON5.

## Solution Assessment

Unclear. YAML satisfies the literal request, but the issue discussion identifies comment-safe mutation as the sharper problem, so maintainers should decide whether the durable fix is a second root config format or stronger `config.patch`/CLI guidance.
