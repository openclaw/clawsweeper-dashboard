## Summary

Keep open for maintainer product review: current main still documents and implements JSON/JSON5 config as the canonical core config file format, while the discussion reframes the strongest pain as safe programmatic mutation rather than YAML itself.

## What This Changes

Request support for `~/.openclaw/openclaw.yaml` or `.yml` as an alternative config file format alongside the current JSON5 `openclaw.json`.

## Best Possible Solution

Maintain the JSON5 config contract unless maintainers explicitly approve a second core config format; otherwise keep improving `openclaw config` and `config.patch` as the safe mutation path.

## Reproduction Assessment

Not applicable: this is a request for a new config file format, not a report of broken existing behavior. Source inspection confirms YAML config files are not currently part of the main config path/parser contract.

## Solution Assessment

Unclear: YAML is a plausible ergonomics feature, but current code and discussion point to `openclaw config` and `config.patch` as the narrower solution for comment-preserving scripted edits. A maintainer should decide whether a second config format is worth the added core compatibility surface.
