## Summary

Keep open: current main still does not load YAML/YML root configs, and the discussion narrows the tradeoff without a maintainer decision that accepts, rejects, implements, or supersedes the YAML request.

## What This Changes

The issue requests `~/.openclaw/openclaw.yaml` or `.yml` support alongside the current JSON5 `openclaw.json` root config.

## Best Possible Solution

Make a maintainer config-contract decision: either implement YAML across discovery, parsing, includes, validation, reload, writes, docs, and tests, or document JSON5-only while strengthening `config.patch` and CLI editing guidance.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source and docs inspection show current main still recognizes JSON config candidates and parses root config as JSON5.

## Solution Assessment

Unclear. YAML satisfies the literal request, but the issue discussion points to comment-safe mutation as the root problem, so maintainers should decide whether the durable solution is a second root config format or stronger `config.patch`/CLI guidance.
