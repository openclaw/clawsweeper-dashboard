## Summary

Keep open for maintainer product review: current main and the latest release still define JSON/JSON5 `openclaw.json` as the core config contract, while the discussion identifies comment-safe mutation as the stronger underlying pain rather than proving YAML is the best core format.

## What This Changes

The issue requests support for `~/.openclaw/openclaw.yaml` or `.yml` as an alternative to the existing JSON5 `openclaw.json` config file.

## Best Possible Solution

Maintainers should either reaffirm JSON5 plus CLI/RPC config mutation as canonical, or explicitly approve YAML with precedence, write semantics, validation, docs, and tests.

## Reproduction Assessment

Not applicable: this is a request for a new config file format, and source/docs confirm YAML config files are not part of the current main or latest-release contract.

## Solution Assessment

Unclear: YAML is a plausible ergonomics feature, but the current implementation and issue discussion point to `openclaw config` and `config.patch` as the narrower solution for comment-safe scripted edits.
