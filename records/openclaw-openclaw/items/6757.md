## Summary

Keep open: current main still has manual, admin RPC/SDK, automatic, and plugin/context-engine compaction paths, but it does not expose the requested in-session agent-triggered self-compaction action; #43401 is closed unmerged and remains useful prior art.

## What This Changes

Add an agent-callable way for a running session to trigger its own context compaction without an authorized channel sender or active-run deadlock.

## Best Possible Solution

Implement a first-class agent-visible self-compaction action that preserves session visibility, active-run lane safety, compaction accounting, token snapshots, docs, and focused regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows session_status has no action parameter, the agent gateway tool cannot call sessions.compact, and the existing sessions.compact RPC is admin-scoped rather than an in-run self-compaction path.

## Solution Assessment

Yes. The session_status(action="compact") direction is the narrowest maintainable path because session_status is already agent-visible for current-session state and can be made explicitly mutating with focused tests.

## Repair Work Prompt

Implement agent-triggered self-compaction for https://github.com/openclaw/openclaw/issues/6757. Prefer `session_status(action="compact")` as prior art from #43401, while preserving existing session_status status/model behavior by default. Add an `action` parameter with a compact action, resolve the same visible/current session target as status calls, reject unauthorized targets before mutation, call the direct active-run-safe compaction path rather than the queued wrapper when running inside the agent tool path, update session store compaction count/sessionFile/sessionId/token freshness from the compaction result, and return a compact result payload that is small and auditable. Mark `session_status(action="compact")` as mutating in `src/agents/tool-mutation.ts` while keeping status-only calls non-mutating. Add focused tests for the schema/execution behavior, visibility-before-mutation, mutation tracking, and active-run-safe compaction wiring. Update `docs/concepts/session-tool.md` and `CHANGELOG.md` because this is user-facing. Do not relax `sessions.compact` admin scope, expose it through the agent gateway tool, or change unrelated compaction policy/config defaults.
