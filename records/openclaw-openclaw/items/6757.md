## Summary

Keep open: current main and v2026.5.20 still have manual, automatic, admin RPC, and compaction-hook paths, but not the requested in-session agent-triggered own-session compaction action. Prior implementation attempts are closed unmerged, and the remaining work is a product and security-boundary decision rather than conservative cleanup to close.

## What This Changes

Request an agent-callable way, such as `session_status(action="compact")` or an equivalent Gateway/CLI surface, for an active agent to compact its own session without authorized sender input.

## Best Possible Solution

Implement a first-class own-session compaction action, likely `session_status(action="compact")` or an equivalent mutating agent tool, preserving visibility authorization, active-run semantics, mutation tracking, compaction counters, token snapshots, and the admin RPC boundary.

## Reproduction Assessment

Yes, at source level. Current main and `v2026.5.20` lack a `session_status` compact action, model-facing Gateway compaction action, or dedicated compact CLI command, while existing compaction remains manual, automatic, plugin/context-engine, SDK, or admin/control-plane scoped.

## Solution Assessment

Unclear until maintainers choose the product and security boundary. A constrained own-session `session_status(action="compact")` or equivalent mutating tool looks narrow, but forwarding the existing admin RPC unchanged would not be the right solution.
