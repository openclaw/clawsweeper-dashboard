## Summary

Keep open: current main has manual `/compact`, automatic/runtime compaction, plugin/context-engine hooks, and an operator-admin `sessions.compact` RPC, but it still lacks the requested agent-callable in-session self-compaction action.

## What This Changes

Add a way for an agent to trigger compaction on its own active session without a user-authorized `/compact` message.

## Best Possible Solution

Implement a first-class in-session self-compaction action, preferably `session_status(action="compact")` or an equivalent agent tool, with explicit permission semantics, mutating-tool tracking, active-run-safe direct compaction, counter/token updates, and focused tests/docs.

## Reproduction Assessment

Yes. Source inspection on current main shows no `session_status` compact action, no agent gateway `sessions.compact` action, and only manual/admin/automatic/plugin-adjacent compaction paths.

## Solution Assessment

The requested direction is not implemented; the likely best fix is a direct in-run tool action with explicit permission and mutation semantics, not forwarding to the existing admin RPC that interrupts active sessions.
