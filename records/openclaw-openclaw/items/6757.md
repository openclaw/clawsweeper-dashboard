## Summary

Keep open. Current main still has adjacent manual, admin, automatic, SDK, and plugin compaction paths, but it does not expose the requested in-session agent self-compaction action; the matching prior implementation PRs were closed unmerged and the latest maintainer review kept this issue open.

## What This Changes

Request an agent-callable way to compact its own active session, such as a `session_status` action, Gateway RPC, or dedicated CLI command.

## Best Possible Solution

Implement a first-class self-session compact action with explicit own-session visibility checks, direct active-run compaction, mutating-tool tracking, counter/token updates, and focused regression tests.

## Reproduction Assessment

Yes, for the missing capability. Source inspection on current main shows no `session_status` compact action, no agent-facing gateway compaction action, and no `sessions_manage` implementation while adjacent admin/manual/automatic compaction paths exist.

## Solution Assessment

Unclear until maintainers choose the product and security boundary. A `session_status(action="compact")` or equivalent self-session tool looks like the narrowest maintainable direction, but it should not expose the existing admin RPC unchanged.
