## Summary

Keep open: current main still has manual, automatic, SDK/control-plane, plugin/context-engine, and admin Gateway compaction paths, but no agent-facing in-session self-compaction action. The directly related implementation attempts remain closed unmerged, and the latest maintainer-facing review also kept this open.

## What This Changes

Request an agent-callable way for a running OpenClaw session to compact its own context, such as `session_status(action="compact")`, a Gateway RPC, or a dedicated CLI surface.

## Best Possible Solution

Add a first-class mutating in-session compaction action with explicit authorization, visibility, replay/mutation semantics, active-run-safe direct compaction, counter/token snapshot updates, docs, changelog, and focused regression tests.

## Reproduction Assessment

Not applicable as a feature request rather than broken documented behavior. Source inspection on current main gives high confidence that the requested surface is absent: `session_status` has no compact action and the agent-facing `gateway` tool cannot call `sessions.compact`.

## Solution Assessment

Unclear until a maintainer chooses the public surface and permission model. `session_status(action="compact")` matches the closed prior implementation and looks like the narrowest candidate, but it still needs API and safety sign-off.
