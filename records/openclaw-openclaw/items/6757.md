## Summary

Keep open: current main still lacks an agent-callable self-compaction path through `session_status`, the agent-facing `gateway` tool, or a dedicated CLI command; the useful prior implementation at https://github.com/openclaw/openclaw/pull/43401 is closed unmerged.

## What This Changes

The issue requests a first-class way for an agent to trigger compaction of its own active session without a user sending `/compact`.

## Best Possible Solution

Add a first-class mutating in-session compaction action with explicit visibility and authorization checks, active-run-safe direct compaction, counter/token snapshot updates, docs, changelog, and focused regression tests.

## Reproduction Assessment

Not applicable as a feature request rather than broken documented behavior. Source inspection on current main gives high confidence that the requested `session_status` or agent-gateway self-compaction surface is absent.

## Solution Assessment

Unclear until maintainers choose the public API and permission model. `session_status(action="compact")` remains the narrowest prior-art direction, but it creates a new mutating agent-facing control path that needs explicit approval.
