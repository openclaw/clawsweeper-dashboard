## Summary

Keep open: current main and v2026.5.22 still lack the requested agent-facing own-session compaction action. Existing `/compact`, automatic/runtime compaction, context-engine hooks, and admin-scoped `sessions.compact` are adjacent but do not provide a model-callable self-compaction surface inside the active session.

## What This Changes

The issue requests a first-class way for an agent to trigger compaction of its own active session, such as `session_status(action="compact")`, an equivalent Gateway action, or a dedicated CLI command.

## Best Possible Solution

Implement a constrained own-session compact action through `session_status` or an equivalent mutating agent tool that uses direct in-run compaction, updates compaction counters/token snapshots, and keeps arbitrary target compaction behind the admin RPC boundary.

## Reproduction Assessment

Yes, at source level. Current main and `v2026.5.22` lack a `session_status` compact action, model-facing Gateway compaction action, or dedicated compact CLI command, while existing compaction remains manual, automatic/runtime, SDK/plugin, or admin/control-plane scoped.

## Solution Assessment

Unclear until maintainers choose the product and security boundary. A constrained own-session `session_status(action="compact")` or equivalent mutating tool looks narrow; forwarding the existing admin RPC unchanged would not be the right solution.
