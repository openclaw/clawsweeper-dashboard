## Summary

Keep open: current main and the latest release still lack an agent-facing self-compaction action, and the existing manual, automatic, and admin/control-plane compaction paths do not satisfy the requested in-session tool boundary.

## What This Changes

The issue requests a first-class way for an agent to compact its own active session without a user sending /compact.

## Best Possible Solution

Implement a current-session-only mutating agent action, preferably session_status(action="compact") or an equivalent narrow tool, backed by direct in-run compaction and focused security/session-state tests.

## Reproduction Assessment

Yes. Source inspection shows session_status has no compact action, the model-facing gateway tool cannot call sessions.compact, and the existing sessions.compact RPC is admin-scoped rather than an in-session self-compaction surface.

## Solution Assessment

Yes, conditionally. A current-session-only session_status(action="compact") or equivalent mutating agent tool is the best-fit direction; exposing the existing admin gateway RPC directly would be the wrong layer.
