## Summary

Keep open: current main still lacks an agent-callable self-compaction surface; existing compaction paths are manual chat commands, admin/control-plane RPC, automatic runtime behavior, or plugin/context-engine internals rather than the requested in-session action.

## What This Changes

Add an agent-callable way to compact the current session, preferably `session_status(action="compact")`, without requiring an authorized human `/compact` sender.

## Best Possible Solution

Implement a first-class agent-visible self-compaction action on `session_status` that reuses direct compaction for in-run calls and keeps `sessions.compact` as the admin/control-plane surface.

## Reproduction Assessment

Yes. Source inspection on current main shows the requested action is absent from `session_status`, unavailable through the agent gateway tool, and only present as an admin-scoped RPC.

## Solution Assessment

Yes. `session_status(action="compact")` is the narrowest maintainable direction because that tool already resolves current-session state and can be marked mutating with focused regression coverage.
