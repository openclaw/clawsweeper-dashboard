## Summary

Keep open: current main still lacks the requested agent-callable self-compaction surface, and the prior implementation attempts are closed unmerged.

## What This Changes

The issue requests an agent-callable way to trigger compaction for the current session, such as `session_status(action="compact")`, a Gateway RPC, or a dedicated CLI command.

## Best Possible Solution

Add a first-class mutating in-session compaction action with explicit visibility and authorization checks, active-run-safe compaction, counter/token snapshot updates, docs, changelog, and focused regression tests.

## Reproduction Assessment

Not applicable as a feature request rather than broken documented behavior. Source inspection on current main gives high confidence that the requested `session_status` or agent-gateway self-compaction surface is absent.

## Solution Assessment

Unclear until maintainers choose the public API and permission model. `session_status(action="compact")` remains narrow prior art, but it creates a new mutating agent-facing control path that needs explicit product/security approval.
