## Summary

Keep open: current main and the latest release still have manual `/compact`, admin `sessions.compact`, automatic compaction, and context-engine hooks, but no in-session agent-callable self-compaction action through `session_status`, the agent-facing `gateway` tool, or a dedicated CLI surface.

## What This Changes

Requested an agent-callable way to compact the current session without user intervention.

## Best Possible Solution

Implement a first-class self-compaction surface, likely `session_status(action="compact")` or an equivalent agent tool, using direct in-run compaction, explicit mutation tracking, session visibility checks, counter/token updates, and focused active-run tests.

## Reproduction Assessment

Yes. Source inspection on current main shows `session_status` has no compact action, the agent-facing `gateway` tool cannot call `sessions.compact`, and the only Gateway compaction method is admin-scoped and interrupts active runs.

## Solution Assessment

Unclear. `session_status(action="compact")` is a plausible narrow solution, but maintainers still need to confirm the product/security boundary; forwarding the admin RPC as-is is not the best fix because it has different active-run and authorization semantics.
