## Summary

Keep open: current main still has adjacent manual, automatic, SDK/control-plane, and admin Gateway compaction paths, but it does not expose the requested in-session agent self-compaction action. The directly related implementation attempt at https://github.com/openclaw/openclaw/pull/43401 is closed unmerged, and recent review comments reached the same keep-open conclusion.

## What This Changes

Request to add an agent-callable way, such as `session_status(action="compact")`, a Gateway RPC, or a CLI command, to compact the agent's own active session without user intervention.

## Best Possible Solution

Add a first-class mutating in-session compaction action with explicit authorization, visibility, replay/mutation semantics, active-run-safe direct compaction, counter/token snapshot updates, docs, changelog, and focused regression tests.

## Reproduction Assessment

Not applicable as a feature request rather than broken documented behavior. Source inspection on current main gives high confidence that the requested surface is absent: `session_status` has no compact action and the agent-facing `gateway` tool cannot call `sessions.compact`.

## Solution Assessment

Unclear until a maintainer chooses the public surface and safety model. `session_status(action="compact")` matches the closed prior implementation and looks like the narrowest candidate, but it still needs API and permission sign-off.
