## Summary

Keep open. Current main still has manual, admin RPC/SDK, automatic, and plugin/context-engine compaction paths, but it does not expose the requested in-session agent-triggered self-compaction action; the exact prior implementation PR was closed unmerged, and the maintainer-facing discussion already kept this issue open for follow-up.

## What This Changes

The issue requests an agent-callable way for a running session to trigger its own context compaction without user `/compact` authorization.

## Best Possible Solution

Implement a first-class agent self-compaction action, preferably `session_status(action="compact")` or an equivalent agent-visible tool path, using direct compaction inside an active run, preserving visibility and mutation semantics, updating compaction accounting, and documenting the new tool behavior.

## Reproduction Assessment

Yes. A high-confidence static reproduction is available on current main: the `session_status` schema has no `action` parameter, the agent `gateway` tool cannot call `sessions.compact`, and the existing `sessions.compact` RPC is admin/control-plane scoped rather than an in-run self-compaction path.

## Solution Assessment

Yes, the proposed `session_status(action="compact")` direction looks like the narrowest maintainable fix because `session_status` is already agent-visible for the current session and can be made mutating with focused tests. The Gateway/SDK `sessions.compact` surface is useful adjacent infrastructure but does not solve the in-session active-run case by itself.
