## Summary

Keep open: current main still lacks an agent-callable self-compaction surface. Existing manual, SDK, and admin RPC compaction paths do not satisfy in-run self-triggering through `session_status`, the agent-facing `gateway` tool, or a dedicated compact command, and related PR #43401 was closed unmerged.

## What This Changes

Request a first-class way for an agent to trigger compaction on its own active session without an authorized sender or external user command.

## Best Possible Solution

Add a mutating in-session compaction action with clear permission semantics, direct active-run compaction to avoid lane deadlocks, counter/token updates, docs, and focused tests.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main gives high confidence: `session_status` has no `action` parameter and the agent-facing `gateway` tool does not list `sessions.compact`.

## Solution Assessment

Unclear until a maintainer confirms the public surface, but `session_status(action="compact")` is the narrowest candidate because it matches the existing session tool and prior closed PR #43401.
