## Summary

Keep open: current main still lacks an agent-callable self-compaction surface; existing compaction paths are manual, admin/control-plane, or automatic runtime behavior rather than the requested in-session action.

## What This Changes

Request to let agents compact their own active session through `session_status`, a gateway RPC, or a dedicated CLI command without relying on an authorized sender command.

## Best Possible Solution

Implement a first-class agent-visible `session_status(action="compact")` path that reuses direct compaction, preserves visibility and runtime semantics, and keeps the existing admin RPC as a control-plane surface.

## Reproduction Assessment

Yes. Source inspection on current main shows the requested action is absent from `session_status`, unavailable through the agent gateway tool, and present only as an admin-scoped RPC plus manual command path.

## Solution Assessment

Yes. `session_status(action="compact")` is the narrowest maintainable direction because that tool already resolves current-session state and can be explicitly marked mutating with focused regression tests.

Implement the feature requested by https://github.com/openclaw/openclaw/issues/6757. Add a first-class `session_status(action="compact")` action so an agent can compact its own visible session from inside the current run without routing through `/compact` sender authorization or the admin `sessions.compact` RPC interruption path. Reuse the direct embedded compaction path where appropriate, preserve session visibility checks, permission semantics, compaction counters/token snapshots, hook/runtime behavior, and active-run safety. Mark `session_status` compact calls as mutating in `src/agents/tool-mutation.ts`. Update the tool schema/description, focused tests, docs, and CHANGELOG for a user-facing feature. Treat PR #43401 as prior art, but do not copy stale code blindly; stop if the implementation would require a broad product decision, new config option, or incompatible gateway protocol change.
