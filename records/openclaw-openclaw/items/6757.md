## Summary

Keep open: current main still lacks an agent-callable self-compaction surface; existing compaction paths are manual, admin/control-plane, or automatic runtime behavior rather than the requested in-session action.

## What This Changes

Request an agent-triggered context compaction mechanism, preferably `session_status(action="compact")`, so long-running agents can compact their own active session without user intervention.

## Best Possible Solution

Implement a first-class agent-visible self-compaction action on `session_status` that reuses the direct compaction path and keeps the existing admin RPC as a control-plane surface.

## Reproduction Assessment

Yes. Source inspection on current main shows the requested action is absent from `session_status`, unavailable through the agent gateway tool, and only present as an admin-scoped RPC.

## Solution Assessment

Yes. `session_status(action="compact")` is the narrowest maintainable direction because that tool already resolves current-session state and can be explicitly marked mutating with focused regression tests.
