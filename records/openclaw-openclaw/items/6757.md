## Summary

Keep open: current main and the latest v2026.5.20 release still lack a first-class in-session, agent-triggered own-session compaction action. Existing manual `/compact`, automatic/runtime compaction, SDK/admin RPC, and plugin/context-engine compaction paths do not satisfy the requested model-facing tool boundary, while the focused prior implementation PR remains closed unmerged.

## What This Changes

The issue requests an agent-callable way for a running session to compact its own context without an authorized user sending `/compact`.

## Best Possible Solution

Implement a constrained own-session compact action through `session_status` or an equivalent mutating agent tool that uses direct in-run compaction, updates compaction counters and token snapshots, and preserves the admin RPC boundary for arbitrary session targets.

## Reproduction Assessment

Yes, at source level. Current main and v2026.5.20 lack a `session_status` compact action, model-facing Gateway compaction action, or dedicated compact CLI command, while existing compaction remains manual, automatic/runtime, plugin/context-engine, SDK, or admin/control-plane scoped.

## Solution Assessment

Unclear until maintainers choose the product and security boundary. A constrained own-session `session_status(action="compact")` or equivalent mutating tool looks narrow, but forwarding the existing admin RPC unchanged would not be the right solution.
