## Summary

Keep open: current main still lacks an agent-callable self-compaction path. Existing manual and Gateway compaction paths are human/admin control surfaces, while the agent-facing `session_status` and `gateway` tools do not expose a compact action.

## What This Changes

Request an agent-triggered self-compaction surface so long-running agents can compact their current session without user intervention.

## Best Possible Solution

Add a first-class mutating in-session compaction action with explicit permission semantics, direct active-run compaction, session counter/token updates, docs, and focused tests.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main gives high confidence that the requested surface is absent: `session_status` has no action parameter and the agent-facing `gateway` tool cannot call `sessions.compact`.

## Solution Assessment

Unclear until a maintainer chooses the public surface. `session_status(action="compact")` is the narrowest candidate because it matches the existing session tool and PR #43401 prior art, but the mutating permission model needs product/safety sign-off.
