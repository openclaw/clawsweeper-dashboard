## Summary

Keep open: current main still lacks an agent-callable self-compaction action, and the existing manual slash command, admin RPC, automatic compaction, and context-engine paths do not satisfy the requested in-session active-run behavior.

## What This Changes

The issue requests an agent-triggered way to compact its own active session, such as `session_status(action="compact")`, an agent-safe Gateway RPC, or a dedicated CLI command.

## Best Possible Solution

Add a first-class mutating in-session compaction action with explicit visibility and authorization checks, active-run-safe direct compaction, counter/token snapshot updates, docs, changelog, and focused regression tests.

## Reproduction Assessment

Not applicable as a feature request rather than broken documented behavior. Source inspection on current main gives high confidence that the requested `session_status` or agent-gateway self-compaction surface is absent.

## Solution Assessment

Unclear until maintainers choose the public API and permission model. `session_status(action="compact")` remains the narrowest apparent prior art, but it creates a new mutating agent-facing control path that needs explicit approval.
