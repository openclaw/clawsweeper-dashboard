## What This Changes

The issue requests a model-callable way for an agent to compact its own active session without an authorized user sending `/compact`.

## Best Possible Solution

Implement a first-class own-session compaction action, likely `session_status(action="compact")` or an equivalent agent tool, constrained by session visibility, active-run-safe direct compaction, mutating-tool tracking, and the existing admin RPC boundary.

## Reproduction Assessment

Yes, at source level. Current main and `v2026.5.19` lack a `session_status` compact action, model-facing gateway compaction action, or dedicated compact CLI command, while existing compaction remains manual, automatic, or admin/control-plane scoped.

## Solution Assessment

Unclear until maintainers choose the product and security boundary. A constrained own-session `session_status(action="compact")` or equivalent mutating tool looks narrow, but exposing the existing admin RPC unchanged would not be the right solution.
