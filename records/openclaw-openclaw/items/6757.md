## Summary

Keep open. Current main and v2026.6.6 still have manual, admin, and automatic compaction paths, but not the requested in-session agent-triggered self-compaction action; the remaining work needs a product/security decision about agent-owned session mutation authority.

## What This Changes

The request asks for an agent-facing way to trigger compaction of its own active session, such as a `session_status` action, gateway RPC, or CLI command.

## Best Possible Solution

Implement a current-session-only agent action, likely `session_status(action="compact")` or an equivalent tool, using active-run-safe compaction semantics, explicit mutation tracking, and focused tests after maintainer approval.

## Reproduction Assessment

Yes from source inspection, with the caveat that this is a missing feature rather than a broken existing contract. Current main shows `session_status` lacks a compact action, the agent gateway tool does not expose `sessions.compact`, and the only compact RPC is admin/control-plane scoped.

## Solution Assessment

Unclear until maintainers choose the product and security boundary. A current-session-only agent action looks like the narrowest maintainable direction, while exposing the admin compact RPC or arbitrary session targets to agents would be the wrong layer.
