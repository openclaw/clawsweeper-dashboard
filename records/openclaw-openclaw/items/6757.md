## Summary

Keep open. Current main and the latest release still lack a first-class in-session agent self-compaction action; the prior narrow implementation was closed unmerged, and the newer broad continuation PR is still open with review blockers rather than shipped.

## What This Changes

The issue requests a model-facing way for an agent to compact its own active session without a user sending `/compact`.

## Best Possible Solution

Implement a first-class own-session compaction action with explicit visibility checks, active-run-safe execution, mutating-tool tracking, compaction counter/token updates, and focused regression coverage while preserving the admin RPC boundary.

## Reproduction Assessment

Yes, at source level. Current main has no compact action in `session_status`, no model-facing Gateway compaction action, no `request_compaction` tool, and docs only describe adjacent manual/admin/status paths.

## Solution Assessment

Unclear until maintainers choose the product and security boundary. An own-session `session_status(action="compact")` or equivalent mutating tool looks narrow, but exposing the existing admin RPC unchanged would not be the right solution.
