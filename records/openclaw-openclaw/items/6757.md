## Summary

Keep open: current main still has manual, automatic, SDK/admin RPC, and context-engine compaction paths, but no first-class in-session agent self-compaction tool; prior implementation attempts were closed unmerged.

## What This Changes

Request an agent-callable way for an agent to compact its own active session without manual sender intervention.

## Best Possible Solution

Implement a first-class own-session compact action with visibility checks, active-run-safe direct compaction, mutating-tool tracking, compaction counter/token updates, and focused regression coverage while preserving the admin RPC boundary.

## Reproduction Assessment

Yes, at source level. Current main has no compact action in session_status, no model-facing gateway compaction action, and docs only describe adjacent manual, automatic, SDK/admin, and status paths.

## Solution Assessment

Unclear until maintainers choose the product and security boundary. An own-session session_status(action="compact") or equivalent mutating tool looks narrow, but exposing the existing admin RPC unchanged would not be the right solution.
