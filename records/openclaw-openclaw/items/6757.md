## Summary

Keep open: current main still lacks the requested agent-callable self-compaction surface; existing compaction paths are manual, admin/control-plane, automatic runtime maintenance, or CLI lifecycle behavior rather than an in-session agent action.

## What This Changes

Request an agent-triggered self-compaction mechanism, preferably `session_status(action="compact")`, so long-running agents can compact their own active session without user intervention.

## Best Possible Solution

Implement `session_status(action="compact")` as a first-class agent-visible action that reuses direct compaction, preserves existing visibility checks and active-run behavior, marks compact calls as mutating, and updates docs plus regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows the requested action is absent from `session_status`, unavailable through the agent gateway tool, and present only as an admin-scoped RPC plus manual/automatic compaction paths.

## Solution Assessment

Yes. `session_status(action="compact")` is the narrowest maintainable direction because that tool already resolves current-session state and visibility, while exposing `sessions.compact` through the agent gateway would broaden a control-plane surface.
