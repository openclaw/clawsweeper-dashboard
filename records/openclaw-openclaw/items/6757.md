## Summary

Keep open. Current main still lacks the requested in-session, agent-triggered self-compaction action; existing auto-compaction, manual `/compact`, plugin/context-engine hooks, and admin `sessions.compact` do not provide the same agent-facing active-run API, and the remaining work needs maintainer product/security direction.

## What This Changes

The issue requests an agent-callable way to trigger context compaction for the agent's own session, such as a `session_status` action, gateway RPC, or CLI command.

## Best Possible Solution

Keep this open until maintainers sponsor a first-class in-session self-compaction API that uses the direct compaction path, preserves visibility and authorization semantics, marks the action as mutating, updates compaction counters/token snapshots, and adds focused tests.

## Reproduction Assessment

Yes. Source inspection against current main establishes the gap: `session_status` has no compact action, the agent `gateway` tool cannot call `sessions.compact`, and existing compaction surfaces are auto/manual/admin rather than in-session agent self-compaction.

## Solution Assessment

Unclear until maintainers choose the API shape. A direct `session_status(action="compact")` or equivalent agent tool looks narrower than exposing the admin RPC because it can preserve active-run behavior, permission checks, and mutation tracking.
