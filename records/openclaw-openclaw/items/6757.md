## Summary

Keep open: current main and the latest release still lack an in-session, agent-triggered own-session compaction action. Existing paths are manual, automatic/runtime, plugin/context-engine, or admin/control-plane scoped, and the related implementation attempts are closed unmerged.

## What This Changes

Add a first-class way for an agent to compact its own active session without requiring an authorized sender to issue `/compact`.

## Best Possible Solution

Implement a constrained own-session compact action through `session_status` or an equivalent mutating agent tool that uses direct in-run compaction, updates counters/tokens, and preserves the admin RPC boundary.

## Reproduction Assessment

Yes, at source level. Current main and `v2026.5.20` lack a `session_status` compact action, model-facing Gateway compaction action, or dedicated compact CLI command, while existing compaction remains manual, automatic/runtime, plugin/context-engine, SDK, or admin/control-plane scoped.

## Solution Assessment

Unclear until maintainers choose the product and security boundary. A constrained own-session `session_status(action="compact")` or equivalent mutating tool looks narrow, but forwarding the existing admin RPC unchanged would not be the right solution.
