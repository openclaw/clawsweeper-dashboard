## Summary

Keep open. Current main still has adjacent manual, admin/control-plane, automatic, SDK, and plugin compaction paths, but it does not expose the requested in-session agent self-compaction action; prior implementation attempts were closed unmerged and no open canonical replacement now owns the remaining work.

## What This Changes

The issue requests an agent-callable way for a running session to compact its own context without user intervention.

## Best Possible Solution

Implement a first-class self-session compact action with explicit own-session visibility checks, direct active-run compaction, mutating-tool tracking, counter/token updates, and focused regression tests.

## Reproduction Assessment

Yes, for the missing capability. Source inspection on current main shows no `session_status` compact action, no agent-facing Gateway compaction action, and no `sessions_manage` implementation while adjacent admin/manual/automatic compaction paths exist.

## Solution Assessment

Unclear until maintainers choose the product and security boundary. A `session_status(action="compact")` or equivalent own-session tool looks like the narrowest maintainable direction, but it should not expose the existing admin RPC unchanged.
