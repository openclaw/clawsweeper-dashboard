## Summary

Keep open: current main still lacks an agent-callable self-compaction action, and the existing manual slash command, admin RPC, automatic compaction, and context-engine hooks do not satisfy the requested in-session active-run behavior. The prior implementation attempt is closed unmerged, and no canonical open replacement was found.

## What This Changes

Request an agent-triggered context compaction surface, preferably through `session_status(action="compact")`, an agent-facing gateway call, or a dedicated CLI command.

## Best Possible Solution

Add a first-class mutating in-session compaction action that preserves session visibility and authorization, uses active-run-safe direct compaction, updates counters/token snapshots, and includes focused tests and docs.

## Reproduction Assessment

Not applicable as a feature request rather than broken documented behavior. Source inspection on current main gives high confidence that the requested `session_status` or agent-gateway self-compaction surface is absent.

## Solution Assessment

Unclear until maintainers choose the public API and permission model. `session_status(action="compact")` remains the narrowest apparent prior art, but it creates a new mutating agent-facing control path that needs explicit approval.
