## Summary

Keep open: current main still has manual, automatic, admin/control-plane, ACP, and context-engine compaction paths, but no agent-facing in-session self-compaction action, and the directly related implementation attempt is closed unmerged.

## What This Changes

Request to let an agent trigger compaction on its own active session through `session_status`, a Gateway RPC surface, or a dedicated CLI command.

## Best Possible Solution

Add a first-class mutating in-session compaction action that uses the direct compaction path, preserves session visibility and authorization, updates compaction counters/token snapshots, and documents the approved agent API.

## Reproduction Assessment

Not applicable as a feature request rather than broken documented behavior. Source inspection on current main gives high confidence that the requested surface is absent: `session_status` has no compact action and the agent-facing `gateway` tool cannot call `sessions.compact`.

## Solution Assessment

Unclear until maintainers choose the public API and permission model. `session_status(action="compact")` is plausible and has useful prior art, but a mutating self-compaction control needs explicit safety and product approval before implementation.
