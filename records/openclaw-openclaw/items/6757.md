## Summary

Keep open: current main still does not provide an agent-callable self-compaction action, and the existing manual, admin RPC, automatic, and context-engine compaction paths do not satisfy in-session active-run self-compaction.

## What This Changes

The issue requests a first-class way for an agent to trigger context compaction on its own session without user-issued `/compact` authorization.

## Best Possible Solution

Add a first-class mutating in-session compaction action with explicit authorization and visibility checks, active-run-safe direct compaction, counter/token snapshot updates, docs, changelog, and focused regression tests.

## Reproduction Assessment

Not applicable as a feature request rather than a broken documented behavior. Source inspection on current main gives high confidence that the requested `session_status` or agent gateway self-compaction surface is absent.

## Solution Assessment

Unclear until maintainers choose the public surface and permission model. `session_status(action="compact")` remains the narrowest apparent candidate, but it creates a new mutating agent-facing control path that needs explicit approval.
