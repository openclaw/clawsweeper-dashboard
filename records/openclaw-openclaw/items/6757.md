## Summary

Keep open. Current main still has manual, admin RPC, automatic, and context-engine compaction paths, but it does not expose an in-session agent-triggered self-compaction action; the remaining work needs product and security judgment because the existing Gateway RPC is admin-scoped.

## What This Changes

The issue requests an agent-callable way to compact its own active session, such as `session_status(action="compact")`, an agent-facing Gateway action, or a dedicated CLI command.

## Best Possible Solution

Implement a first-class own-session compaction tool with explicit visibility checks, active-run direct compaction, mutating-tool tracking, counter/token updates, and focused regression coverage.

## Reproduction Assessment

Yes, for the missing capability. Source inspection on current main shows no compact action in `session_status`, no agent-facing Gateway compaction action, and no equivalent documented self-compaction tool while adjacent manual/admin/automatic paths exist.

## Solution Assessment

Unclear until maintainers choose the product and security boundary. `session_status(action="compact")` or an equivalent own-session tool looks like the narrowest maintainable direction, but it should not expose `sessions.compact` unchanged.
