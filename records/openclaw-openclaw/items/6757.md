## Summary

Keep open: current main still has manual, automatic, and admin/control-plane compaction paths, but it does not expose the requested in-session agent-triggered self-compaction action. The prior implementation attempt was closed unmerged and is not an ancestor of current main.

## What This Changes

Add an agent-callable way for a running session to compact its own context without a user-authorized `/compact` command.

## Best Possible Solution

Implement a first-class own-session compaction action, likely `session_status(action="compact")` or an equivalent mutating agent tool, constrained by session visibility, active-run-safe direct compaction, counters/token updates, and the existing admin RPC boundary.

## Reproduction Assessment

Yes, at source level. Current main lacks a `session_status` compact action, model-facing gateway compaction action, or dedicated `openclaw compact --session` command while compaction remains manual, automatic, or admin/control-plane scoped.

## Solution Assessment

Unclear until maintainers choose the product and security boundary. A constrained own-session mutating tool looks narrow, but forwarding the existing admin RPC unchanged would not be the right solution.
