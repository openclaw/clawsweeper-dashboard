## Summary

Keep open. Current main and the latest shipped release still lack a first-class in-session agent self-compaction action; the remaining work is a model-triggered session mutation surface that needs product and authorization-boundary review.

## What This Changes

The issue requests an agent-callable way for a running session to compact its own context without an authorized user sending `/compact`.

## Best Possible Solution

Implement a first-class own-session compaction action with explicit visibility checks, active-run-safe execution, mutating-tool tracking, compaction counter/token updates, and focused regression coverage while preserving the admin RPC boundary.

## Reproduction Assessment

Yes, at source level for the missing capability. Current main has no compact action in `session_status`, no model-facing Gateway compaction action, no `sessions_manage` tool, and docs only describe adjacent manual/admin/automatic compaction paths.

## Solution Assessment

Unclear until maintainers choose the product and security boundary. An own-session `session_status(action="compact")` or equivalent mutating tool looks narrow, but it should not expose the existing admin RPC unchanged.
