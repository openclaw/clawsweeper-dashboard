## Summary

Keep open: current main still has manual chat compaction, automatic compaction, SDK/control-plane `sessions.compact`, and admin-scoped Gateway compaction, but it does not provide the requested in-session agent self-compaction action through `session_status`, the agent `gateway` tool, or a dedicated CLI command.

## What This Changes

The issue requests an agent-callable way to trigger context compaction for its own active session, such as `session_status(action="compact")`, a Gateway RPC, or a dedicated CLI command.

## Best Possible Solution

Add a first-class mutating in-session compaction action with explicit authorization, visibility, mutation/replay semantics, lane-safe direct compaction, counter/token snapshot updates, docs, changelog, and focused regression tests.

## Reproduction Assessment

Not applicable as a feature request rather than broken documented behavior. Source inspection on current main gives high confidence that the requested surface is absent: `session_status` has no compact action and the agent-facing `gateway` tool cannot call `sessions.compact`.

## Solution Assessment

Unclear until a maintainer chooses the public surface and safety model. `session_status(action="compact")` matches the closed prior implementation and looks like the narrowest candidate, but it still needs product and permission sign-off.
