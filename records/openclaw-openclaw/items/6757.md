## Summary

Keep open: current main still lacks the requested agent-callable self-compaction path. Existing code covers manual chat compaction, automatic compaction, SDK/control-plane `sessions.compact`, and admin-scoped Gateway compaction, but not an in-session agent action through `session_status`, the agent `gateway` tool, or a dedicated CLI command.

## What This Changes

The issue requests a way for an agent to trigger context compaction on its own active session without a human sender command.

## Best Possible Solution

Add a first-class mutating in-session compaction action with explicit authorization, visibility, mutation/replay semantics, lane-safe direct compaction, counter/token snapshot updates, docs, changelog, and focused regression tests.

## Reproduction Assessment

Not applicable as a feature request rather than a broken documented behavior. Source inspection on current main gives high confidence that the requested surface is absent: `session_status` has no compact action and the agent-facing `gateway` tool cannot call `sessions.compact`.

## Solution Assessment

Unclear until a maintainer chooses the public surface and safety model. `session_status(action="compact")` matches the closed prior implementation and looks like the narrowest candidate, but it still needs product and permission sign-off.
