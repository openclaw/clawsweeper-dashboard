## Summary

Current main still provides manual `/compact`, automatic/runtime compaction, context-engine hooks, and the administrative `sessions.compact` RPC, but no agent-facing action can compact the active session. The directly proposed implementation at https://github.com/openclaw/openclaw/pull/43401 closed unmerged, and no newer canonical item owns this exact capability.

## What This Changes

Requests an agent-callable action to compact its own active session without an authorized external sender.

## Best Possible Solution

Add a self-only `session_status(action="compact")` path that invokes canonical direct compaction, persists counters and token snapshots, and defines deterministic active-run return or continuation behavior.

## Reproduction Assessment

Yes, at the source-contract level. Current main has no compaction action in `session_status`, and the agent-facing gateway tool does not expose the existing administrative `sessions.compact` RPC.

## Solution Assessment

Yes, conditionally: a self-only `session_status` action is narrower and safer than exposing general administrative compaction, but maintainers must first settle authorization and post-compaction active-run semantics.
