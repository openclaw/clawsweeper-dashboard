## Summary

Keep open: current main and the latest release still lack a model-callable own-session compaction action. Existing manual `/compact`, automatic/runtime compaction, SDK/admin `sessions.compact`, and the closed unmerged prior PR are adjacent but do not provide the requested in-session self-compaction surface.

## What This Changes

The issue requests a first-class way for an agent to trigger context compaction on its own active session without user intervention.

## Best Possible Solution

Implement a constrained own-session compact action through `session_status` or an equivalent mutating agent tool that uses direct in-run compaction, updates counters/token snapshots, and keeps arbitrary target compaction behind the admin RPC boundary.

## Reproduction Assessment

Yes, at source level. Current main and `v2026.5.22` lack a `session_status` compact action, model-facing Gateway compaction action, or dedicated compact CLI command, while existing compaction remains manual, automatic/runtime, SDK/plugin, or admin/control-plane scoped.

## Solution Assessment

Unclear until maintainers choose the product and security boundary. A constrained own-session `session_status(action="compact")` or equivalent mutating tool looks narrow; forwarding the existing admin RPC unchanged would not be the right solution.
