## Summary

Keep open: current main has automatic, manual, and admin/control-plane compaction paths, but it still does not expose the requested in-session agent self-compaction action, and the prior implementation PRs were closed unmerged.

## What This Changes

Add an agent-callable way for a running agent to compact its own session without external sender intervention.

## Best Possible Solution

Implement a first-class own-session compaction action with explicit visibility checks, active-run-safe execution, mutating-tool tracking, compaction counter/token updates, and focused regression coverage while preserving the admin RPC boundary.

## Reproduction Assessment

Yes, at source level. Current main has no compact action in session_status, no model-facing gateway compaction action, and docs only describe adjacent manual/admin/status paths.

## Solution Assessment

Unclear until maintainers choose the product and security boundary. An own-session session_status(action="compact") or equivalent mutating tool looks narrow, but exposing the existing admin RPC unchanged would not be the right solution.
