## Summary

Keep this issue open: current main still lacks an agent-callable self-compaction action, while the available manual and gateway paths require external authority or interrupt the active run. The request fits OpenClaw’s agent-harness direction, but maintainers must approve the same-session authorization and post-compaction continuation contract before implementation.

## What This Changes

Add a first-class agent-callable action that compacts the agent’s own active session without requiring an authorized external sender.

## Best Possible Solution

Add a mutating, same-active-session-only agent tool action backed by the canonical direct compaction runtime, with no general gateway-admin authority and with focused accounting, visibility, active-run, and repeated-invocation coverage.

## Reproduction Assessment

Yes, at source level. Current main’s agent tool schema lacks a compaction action, and the only direct gateway compaction path interrupts active runs, so the requested capability gap is unambiguous.

## Solution Assessment

Yes in overall direction: a constrained `session_status(action="compact")`-style action is narrower and safer than exposing admin gateway compaction. The permanent contract still needs maintainer judgment on invocation authority, repeated calls, and whether the current turn resumes after compaction.

## Risks / Open Questions

- A self-compaction action could deadlock, abort, or corrupt the active run unless it uses the canonical direct compaction path and preserves transcript locking, checkpoints, token snapshots, counters, and continuation semantics.
- Accepting arbitrary `sessionKey` targets would widen agent authority beyond self-management and could permit one session to mutate another session’s context.
- The model may invoke compaction unexpectedly or repeatedly, so tool policy, mutation tracking, loop protection, and observable result semantics require an explicit product decision.
