## Summary

Keep open. Current main has manual `/compact`, admin/control-plane `sessions.compact`, automatic compaction, and compaction-provider hooks, but it still does not expose the requested in-session agent self-compaction action; the prior maintainer review also kept this open, and the linked implementation PR was closed unmerged.

## What This Changes

The issue requests an agent-callable way to compact its own active session, such as `session_status(action="compact")`, a gateway call, or a dedicated CLI command.

## Best Possible Solution

Implement a first-class in-session self-compaction action, preferably through `session_status` or an equivalent agent tool, using direct compaction inside an active run, explicit self-session permission checks, mutating-tool tracking, counter/token updates, and focused regression tests.

## Reproduction Assessment

Yes, for the requested missing capability. Source inspection on current main shows the requested `session_status` action, agent-facing gateway action, and CLI surface are absent while adjacent manual/admin/auto compaction paths exist.

## Solution Assessment

Unclear until maintainers choose the product boundary. A first-class self-session compact action is likely the maintainable direction, but the exact API and authorization model need explicit review before implementation.
