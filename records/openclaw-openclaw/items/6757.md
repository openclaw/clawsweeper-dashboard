## Summary

Current main still lacks an agent-callable self-compaction action, and the directly linked implementation was closed unmerged; the request remains useful but needs a maintainer decision on the canonical API, authorization boundary, and Pi/Codex runtime ownership.

## What This Changes

Add a first-class way for an agent to compact its own active session without requiring an authorized sender command.

## Best Possible Solution

Define one permission-aware agent/session self-compaction action that targets only a visible authorized session, delegates to each runtime or active context engine’s native compaction path, and preserves canonical transcript and accounting invariants.

## Reproduction Assessment

Yes, at the source level: current main exposes manual, automatic, context-engine, and admin compaction, while `session_status` and the agent-facing gateway tool still lack a self-compaction action.

## Solution Assessment

Not yet: `session_status(action="compact")` is plausible prior art, but the best fix must define shared authorization and active-run semantics and support both Pi/context-engine and Codex-native compaction rather than adding parallel CLI, gateway, and tool paths.
