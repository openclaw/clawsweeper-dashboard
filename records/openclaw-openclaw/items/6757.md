## Summary

Keep open: current main still lacks the requested agent-callable self-compaction path. Existing paths cover manual `/compact`, automatic compaction, SDK/control-plane `sessions.compact`, and admin-scoped Gateway maintenance, while the focused prior implementation PR remains closed and unmerged.

## What This Changes

The issue requests a first-class way for an agent to compact its own active session, such as `session_status(action="compact")`, a Gateway RPC, or a dedicated CLI command.

## Best Possible Solution

Add a first-class mutating in-session compaction action with explicit authorization, visibility, mutation/replay semantics, lane-safe direct compaction, docs, changelog, and focused regression tests.

## Reproduction Assessment

Not applicable as a feature request rather than a broken documented behavior. Source inspection on current main gives high confidence that the requested surface is absent: `session_status` has no compact action and the agent-facing `gateway` tool cannot call `sessions.compact`.

## Solution Assessment

Unclear until a maintainer chooses the public surface and safety model. `session_status(action="compact")` matches the closed prior PR and looks like the narrowest candidate, but it still needs product and permission sign-off.
