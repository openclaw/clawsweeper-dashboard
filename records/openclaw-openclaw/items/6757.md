## Summary

Keep open: current main still lacks the requested agent-callable self-compaction surface, and the existing manual/admin/automatic compaction paths do not satisfy in-session active-run self-compaction.

## What This Changes

The issue requests a way for an agent to trigger context compaction for its own active session without requiring an authorized user command.

## Best Possible Solution

Add a first-class mutating in-session compaction action with explicit authorization, visibility, replay/mutation semantics, active-run-safe compaction, counter/token snapshot updates, docs, changelog, and focused regression tests.

## Reproduction Assessment

Not applicable as a feature request rather than broken documented behavior. Source inspection on current main gives high confidence that the requested `session_status` or agent gateway self-compaction surface is absent.

## Solution Assessment

Unclear until maintainers choose the public surface and permission model. `session_status(action="compact")` remains the narrowest apparent candidate, but it creates a new mutating agent-facing control path that needs explicit product and safety approval.
