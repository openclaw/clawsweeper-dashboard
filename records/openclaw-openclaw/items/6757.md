## Summary

Keep open. Current main still has adjacent manual, admin, automatic, SDK, and plugin compaction paths, but it does not expose the requested in-session agent self-compaction action; a matching implementation PR was closed unmerged and a prior maintainer review also kept the issue open.

## What This Changes

Request an agent-accessible way to trigger compaction for the agent's own active session, such as `session_status(action: "compact")`, a gateway RPC, or a CLI command.

## Best Possible Solution

Implement a first-class self-session compaction action with explicit permission checks, direct active-run compaction behavior, mutation tracking, counter/token updates, and focused regression tests.

## Reproduction Assessment

Yes, for the requested missing capability. Source inspection on current main shows the requested `session_status` action, agent-facing gateway action, and dedicated CLI surface are absent while adjacent manual/admin/automatic compaction paths exist.

## Solution Assessment

Unclear until maintainers choose the product and security boundary. A `session_status(action="compact")` or equivalent self-session tool appears to be the narrowest maintainable direction, but it must not expose the existing admin compaction surface unchanged.

## Security Review

Status: needs_attention

Summary: The requested feature is security-sensitive because it would let a model mutate session state across a boundary currently represented by an admin-scoped RPC.

Concerns:

- **[medium] Preserve compaction authorization boundaries:** `src/gateway/methods/core-descriptors.ts:143`
