## Summary

Keep open. Current main and the latest release still have manual/admin/automatic compaction paths, but not the requested agent-facing self-compaction action; the related implementation PR was closed unmerged and the live maintainer-side review also kept this issue open.

## What This Changes

The issue requests an agent-callable way to trigger context compaction for its own active session without requiring an authorized human sender.

## Best Possible Solution

Keep this issue open until maintainers approve and implement a first-class current-session-only self-compaction action that preserves visibility/permission semantics, uses an active-run-safe compaction path, marks the action as mutating, updates compaction counters/token snapshots, and adds focused tests.

## Reproduction Assessment

Yes from source inspection, with the caveat that this is a missing feature rather than a broken existing contract. Current main shows `session_status` lacks a compact action, the agent gateway tool does not expose `sessions.compact`, and the only compact RPC is admin/control-plane scoped.

## Solution Assessment

Unclear until maintainers choose the product boundary. A current-session-only `session_status(action="compact")` or equivalent agent tool looks like the narrowest maintainable direction, while exposing the admin `sessions.compact` RPC to agents would be the wrong layer.
