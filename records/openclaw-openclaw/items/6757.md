## Summary

Keep open: current main still lacks the requested in-session agent-triggered self-compaction action. Existing support covers manual `/compact`, admin/control-plane `sessions.compact`, automatic/runtime compaction, and plugin/context-engine hooks; #43401 is closed unmerged prior art rather than a shipped fix.

## What This Changes

The issue requests an agent-callable way for a session to compact itself without requiring user sender authorization.

## Best Possible Solution

Implement a first-class agent-visible self-compaction action, preferably on session_status, that reuses the direct compaction path and preserves visibility checks, mutation tracking, counters, token snapshots, docs, and regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows the requested action is absent from session_status, the agent gateway tool cannot call sessions.compact, and the existing compact RPC is admin/control-plane scoped.

## Solution Assessment

Yes. session_status(action="compact") is still the narrowest maintainable path because session_status already exposes current-session state to agents and can be marked explicitly mutating with focused tests.

Keep the change narrow: add the action schema/runtime path in `src/agents/tools/session-status-tool.ts`, reuse direct/current-session compaction behavior without routing through an active-run-interrupting admin RPC, preserve session visibility checks, mark `session_status` compact calls as mutating in `src/agents/tool-mutation.ts`, update counters/token snapshots after successful compaction, and add focused tests.
