## Summary

Keep open. Current main and the latest release still lack a first-class in-session agent self-compaction action; the remaining work needs product and authorization-boundary review rather than cleanup closure.

## What This Changes

The issue requests an agent-callable way to compact its own active session without requiring a user-issued slash command.

## Best Possible Solution

Implement a first-class own-session compaction action with explicit visibility checks, active-run-safe execution, mutating-tool tracking, compaction counter/token updates, and focused regression coverage while preserving the admin RPC boundary.

## Reproduction Assessment

Yes, at source level for the missing capability. Current main has no compact action in `session_status`, no model-facing Gateway compaction action, no `sessions_manage` or `request_compaction` tool, and docs only describe adjacent manual/admin/status paths.

## Solution Assessment

Unclear until maintainers choose the product and security boundary. An own-session `session_status(action="compact")` or equivalent mutating tool looks narrow, but exposing the existing admin RPC unchanged would not be the right solution.
