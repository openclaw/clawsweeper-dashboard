## Summary

Keep open: current main still lacks the requested agent-callable self-compaction surface, the closest implementation PR is closed unmerged, and linked duplicate issues point back here as the canonical tracker.

## What This Changes

Requests an agent-callable way to compact its own active session, such as `session_status(action="compact")`, a gateway RPC, or a dedicated CLI command.

## Best Possible Solution

Add a first-class mutating in-session compaction action with explicit visibility and authorization checks, active-run-safe direct compaction, counter/token snapshot updates, docs, changelog, and focused regression tests.

## Reproduction Assessment

Not applicable as a feature request rather than broken documented behavior. Source inspection on current main gives high confidence that the requested `session_status` or agent-gateway self-compaction surface is absent.

## Solution Assessment

Unclear until maintainers choose the public API and permission model. `session_status(action="compact")` remains useful prior art, but it creates a new mutating agent-facing control path that needs explicit product and security review.
