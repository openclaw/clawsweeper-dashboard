## Summary

Keep open: current main and v2026.6.6 still lack an agent-facing self-compaction action, while the existing manual/admin/native compaction paths do not satisfy the requested in-session tool boundary and related implementation PRs are closed unmerged.

## What This Changes

Issue requests an agent-callable way to compact its own active session without a user-sent /compact command.

## Best Possible Solution

Keep this open until maintainers approve and implement a first-class current-session self-compaction action, preferably session_status(action="compact") or an equivalent agent tool that uses direct in-run compaction and preserves permission semantics.

## Reproduction Assessment

Yes. Source inspection shows session_status has no compact action, the model-facing gateway tool cannot call sessions.compact, and the existing sessions.compact RPC is admin-scoped rather than an in-session self-compaction surface.

## Solution Assessment

Yes, conditionally: a current-session-only session_status(action="compact") or equivalent mutating agent tool is the best-fit direction, while exposing the existing admin gateway RPC directly would be the wrong layer.
