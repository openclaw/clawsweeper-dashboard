## Summary

Current main still provides manual, automatic, control-plane, and plugin-backed compaction, but no agent-facing action that lets the running agent compact its own active session. The closed prior implementation remains useful design evidence but was never merged, and the latest substantive maintainer discussion explicitly preserves this issue for product and security follow-up.

## What This Changes

Add a first-class agent-triggered action for proactively compacting the agent’s current active session.

## Best Possible Solution

Add a maintainer-approved, current-run/current-session-only `session_status(action="compact")` operation that uses canonical direct compaction, is classified as mutating, preserves session visibility and operator control, updates persisted compaction metadata, and has focused active-run regression coverage.

## Reproduction Assessment

Yes, at high confidence from current source and docs: manual, automatic, and control-plane compaction exist, while the agent-facing `session_status` schema and execution path have no compact action. This establishes the missing capability without needing a failing runtime reproduction.

## Solution Assessment

Yes, `session_status(action="compact")` is the narrowest maintainable direction because it can reuse existing session visibility and avoid broadly exposing an administrative RPC. Maintainers still need to approve the destructive-tool permission and active-run lifecycle contract.
