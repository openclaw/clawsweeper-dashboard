## Summary

Keep open: current main and v2026.6.8 still lack a first-class in-session agent self-compaction surface; the available paths are manual `/compact`, automatic/runtime compaction, plugin hooks, and admin/control-plane `sessions.compact`, none of which satisfy the requested agent-callable self-compact action.

## What This Changes

The issue requests an agent-callable way for a running session to compact its own context, preferably through `session_status(action="compact")`.

## Best Possible Solution

Implement a first-class self-compaction tool action that only targets the caller's visible/current session, uses an in-run compaction path, records mutation state, updates compaction counters/token snapshots, and has explicit security semantics.

## Reproduction Assessment

Yes. Source inspection on current main and v2026.6.8 shows `session_status` has no `action` field, the agent `gateway` tool cannot call `sessions.compact`, and the existing compact RPC is admin-scoped and active-run interrupting.

## Solution Assessment

Unclear until maintainers choose the API boundary. `session_status(action="compact")` is the narrowest prior-art shape, while exposing the admin RPC directly is not the best fix because it would import control-plane and active-run interruption semantics into an agent tool.
