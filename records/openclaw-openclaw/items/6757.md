## Summary

Keep open. Current main still has adjacent manual, admin RPC, automatic, and context-engine compaction paths, but it does not expose the requested in-session agent-triggered self-compaction action; prior implementation attempts are closed unmerged, and the remaining boundary needs maintainer product and security judgment.

## What This Changes

The issue requests an agent-callable way to compact its own session without requiring user intervention.

## Best Possible Solution

Implement a first-class own-session compaction action with explicit visibility checks, active-run-safe compaction, mutating-tool tracking, compaction counter/token updates, and focused regression coverage.

## Reproduction Assessment

Yes, at source level for the missing capability. Current main has no compact action in `session_status`, no agent-facing Gateway compaction action, no `sessions_manage` tool, and no documented self-compaction surface while adjacent compaction paths exist.

## Solution Assessment

Unclear until maintainers choose the product and security boundary. `session_status(action="compact")` or an equivalent own-session mutating tool looks like the narrowest direction, but it should not expose the existing admin RPC unchanged.
