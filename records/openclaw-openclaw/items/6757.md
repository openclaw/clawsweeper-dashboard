## Summary

Keep open. Current main still has adjacent manual, admin RPC, automatic, and context-engine compaction paths, but it does not expose the requested in-session agent-triggered self-compaction action; the remaining work needs product and security judgment because the existing Gateway compaction RPC is admin-scoped.

## What This Changes

Request an agent-callable way to compact its own active session, such as `session_status(action="compact")`, a Gateway action, or a dedicated CLI command.

## Best Possible Solution

Implement a first-class own-session compaction tool with explicit visibility checks, active-run direct compaction, mutating-tool tracking, counter/token updates, and focused regression coverage.

## Reproduction Assessment

Yes, at source level for the missing capability. Current main has no compact action in `session_status`, no agent-facing Gateway compaction action, and no equivalent documented self-compaction tool while adjacent manual/admin/automatic paths exist.

## Solution Assessment

Unclear until maintainers choose the product and security boundary. `session_status(action="compact")` or an equivalent own-session tool looks like the narrowest maintainable direction, but it should not expose `sessions.compact` unchanged.
