## Summary

Keep open: current main still lacks an agent-callable self-compaction surface. This issue remains the canonical tracker; the focused prior implementation was closed unmerged, and the newer broad continuation PR is still open and unmerged.

## What This Changes

Request an agent-facing way to compact its own active session without requiring an authorized external sender.

## Best Possible Solution

Add a first-class mutating in-session compaction action with explicit visibility checks, active-run-safe compaction, counter/token snapshot updates, docs, changelog, and focused regression tests.

## Reproduction Assessment

Not applicable as a feature request rather than broken documented behavior. Source inspection on current main gives high confidence that `session_status`, the agent gateway tool, and current docs still do not expose self-compaction.

## Solution Assessment

Unclear until maintainers choose the API and permission model. A narrow `session_status(action="compact")` or `request_compaction()` path is plausible, but this needs an explicit mutating agent-control design rather than reuse of the admin RPC.
