## Summary

Current main still does not expose the requested in-session agent self-compaction action. The existing paths are manual `/compact`, admin/control-plane `sessions.compact`, automatic compaction, and context-engine/plugin hooks; the closed unmerged PR remains useful prior art, and related duplicates point back to this issue as the canonical thread.

## What This Changes

The issue requests a first-class way for an agent to compact its own active session, such as `session_status(action="compact")`, a gateway call, or a dedicated CLI command.

## Best Possible Solution

Implement an explicit in-session self-compaction path in core, with visibility checks, mutating-tool tracking, direct active-run compaction, compaction counters/token snapshots, and focused regression coverage.

## Reproduction Assessment

Not applicable; this is a feature request for a new agent-callable compaction surface. Source inspection confirms current main has adjacent auto/manual/admin compaction paths but no in-session self-compaction action.

## Solution Assessment

Unclear until maintainers choose the public API, but `session_status(action="compact")` or an equivalent agent tool is the narrowest direction. Reusing the admin RPC directly would not be the best solution because it has different scope and active-run behavior.
