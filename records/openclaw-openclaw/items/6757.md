## Summary

Keep open: current main still lacks an agent-callable self-compaction surface, and the closest implementation PR was closed unmerged. Existing manual, automatic, plugin-hook, and admin RPC compaction paths do not satisfy the requested in-session mutating tool behavior.

## What This Changes

The issue requests a way for an agent to trigger compaction of its own active session without user intervention.

## Best Possible Solution

Add a first-class mutating in-session compaction action with explicit visibility and authorization checks, active-run-safe direct compaction, counter/token snapshot updates, docs, changelog, and focused regression tests.

## Reproduction Assessment

Not applicable as a feature request rather than broken documented behavior. Source inspection on current main gives high confidence that `session_status` and the agent-facing `gateway` tool still do not expose self-compaction.

## Solution Assessment

Unclear until maintainers choose the API and permission model. `session_status(action="compact")` is plausible prior art, but it creates a new mutating agent-facing control path that needs explicit product and security review.
