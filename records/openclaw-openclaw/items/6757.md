## Summary

Current main still lacks an agent-facing self-compaction action: `session_status` remains status/model-only, while existing compaction paths are manual, automatic, plugin-driven, or control-plane scoped. The prior focused implementation was closed unmerged for contributor queue limits rather than because the capability was obsolete, so this issue remains the canonical product/security decision.

## What This Changes

Add a first-class in-session action that lets an agent proactively compact its own active context.

## Best Possible Solution

Expose one current-session-only agent action that delegates to the canonical embedded compaction runtime, with explicit authorization semantics and focused tests for active-run sequencing, mutation tracking, transcript integrity, failure behavior, and compaction accounting.

## Reproduction Assessment

Yes. Current main’s `session_status` schema and execution path directly show that the requested agent-facing compaction action is absent.

## Solution Assessment

Yes, conditionally: `session_status(action="compact")` is a narrow and discoverable shape if restricted to the current session and delegated to the existing compaction owner; a generic arbitrary-session RPC or duplicate runtime path would be less safe.
