## Summary

Keep open: current main and the latest release still lack a model-callable own-session compaction action; existing manual, automatic, plugin, SDK, and admin compaction paths are adjacent but do not satisfy the requested in-session self-compaction surface.

## What This Changes

Request a first-class way for an agent to compact its own active session without user intervention, such as `session_status(action="compact")`, a Gateway method, or a dedicated CLI command.

## Best Possible Solution

Implement a constrained own-session compact action through `session_status` or an equivalent mutating agent tool that uses direct in-run compaction, updates counters/token snapshots, and keeps arbitrary target compaction behind the admin RPC boundary.

## Reproduction Assessment

## Solution Assessment
