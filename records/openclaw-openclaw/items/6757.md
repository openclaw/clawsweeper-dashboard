## Summary

Current main still lacks an agent-facing self-compaction action, and the only direct implementation PR was closed unmerged; this remains a valid agent-harness feature needing maintainer approval for its session-mutation and authorization contract.

## What This Changes

Add a first-class action that lets an agent compact its own active session without requiring an authorized external sender.

## Best Possible Solution

Add a narrowly scoped `session_status(action="compact")` or equivalent current-session tool action that calls the canonical direct compaction runtime, enforces session visibility and authorization, records mutation/accounting state, and has focused active-run continuation and failure tests.

## Reproduction Assessment

Yes at source level. Current main exposes manual, automatic, embedded-runtime, and control-plane compaction, but neither `session_status` nor the agent-facing gateway tool can trigger self-compaction.

## Solution Assessment

Yes in broad direction. Extending the existing agent session tool is narrower and more maintainable than adding another CLI or exposing the broader control-plane RPC, but maintainers must approve the active-run and authorization contract.
