## Summary

Keep open: current main still lacks an agent-callable self-compaction action, and the existing manual, admin RPC, automatic, and context-engine compaction paths do not satisfy the requested in-session active-run behavior.

## What This Changes

Request for agents to trigger compaction of their own active session through `session_status(action="compact")`, an equivalent gateway method, or a dedicated CLI command.

## Best Possible Solution

Add a first-class mutating in-session compaction action with explicit visibility and authorization checks, active-run-safe direct compaction, counter/token snapshot updates, docs, changelog, and focused regression tests.

## Reproduction Assessment

Not applicable as a feature request rather than broken documented behavior. Source inspection on current main gives high confidence that the requested `session_status` or agent-gateway self-compaction surface is absent.

## Solution Assessment

Unclear until maintainers choose the public API and permission model. `session_status(action="compact")` remains the narrowest apparent candidate, but it creates a new mutating agent-facing control path that needs explicit product and safety approval.
