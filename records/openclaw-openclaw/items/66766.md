## Summary

Keep open: current main still has the reported synchronous 200k path for Anthropic Opus/Sonnet 4.6 without `context1m`, and an open unmerged closing PR is already the implementation candidate.

## What This Changes

The issue requests that session/status and Control UI context limits use the Anthropic Opus/Sonnet 4 1M window without requiring `params.context1m: true`.

## Best Possible Solution

Land PR #66862 or an equivalent narrow patch that aligns resolver, session/status persistence, Control UI denominator, tests, and Anthropic docs while preserving explicit opt-out or auth-specific fallback semantics.

## Reproduction Assessment

Yes. A source-level reproduction is present on current main: `anthropic/claude-opus-4-6` with empty params and `allowAsyncLoad: false` is still expected to resolve to 200k, and status/session persistence use that same resolver path.

## Solution Assessment

Unclear for the issue body's unconditional removal. Resolver-level alignment with provider-known 1M context is the right direction, but the final fix should keep beta-header/auth behavior and any explicit opt-out from diverging from the displayed limit.
