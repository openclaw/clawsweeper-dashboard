## Summary

Keep this issue open. Current main still has the reported Anthropic Opus/Sonnet 4.6 no-`context1m` path returning 200k in the synchronous session/status resolver, and open PR #66862 is an unmerged implementation candidate for the same behavior.

## What This Changes

The issue asks OpenClaw to report Anthropic Claude Opus/Sonnet 4.6 sessions as roughly 1M/977k context without requiring `params.context1m: true` when runtime discovery already knows the larger model window.

## Best Possible Solution

Land PR #66862 or an equivalent narrow patch that aligns resolver, session/status persistence, Control UI context display, docs, and Anthropic beta-header/auth semantics for eligible Opus/Sonnet 4.6 models without `context1m`, while preserving an explicit opt-out or auth-specific fallback where needed.

## Reproduction Assessment

Yes. Current main has a focused unit fixture for `anthropic/claude-opus-4-6` with empty params that expects 200k, and the `/status` path calls the same synchronous resolver with `allowAsyncLoad: false`.

## Solution Assessment

Unclear until the open implementation candidate is reviewed. The right direction is resolver-level alignment with provider-discovered 1M context, but the final fix should also keep Anthropic beta-header and OAuth/legacy-token behavior consistent so display and request semantics do not diverge.
