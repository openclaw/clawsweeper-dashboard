## Summary

Keep open: current main still has the 200k synchronous session/status resolver path for Anthropic Opus/Sonnet 4.6 when `context1m` is absent, and #66862 remains the open unmerged closing PR for this issue.

## What This Changes

The issue requests that session/status and Control UI context limits use the 1M Anthropic Opus/Sonnet 4.6 window without requiring `params.context1m: true`.

## Best Possible Solution

Land #66862 or an equivalent narrow patch that aligns the resolver, session/status denominator, Control UI display, tests, and Anthropic docs while preserving explicit opt-out and auth-specific fallback semantics.

## Reproduction Assessment

Yes. Current source has a focused resolver fixture for `anthropic/claude-opus-4-6` with empty params and `allowAsyncLoad: false` expecting 200k, and `/status` plus session persistence call that same synchronous resolver path.

## Solution Assessment

Yes for the direction, but the exact patch should be reviewed. The maintainable fix is a narrow resolver/status alignment that does not accidentally promise 1M request behavior for auth modes where the Anthropic beta header is unavailable.
