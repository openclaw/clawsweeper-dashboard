## Summary

Keep open. Current main still reproduces the reported 200k synchronous session/status resolver path for Anthropic Opus/Sonnet 4.6 without `context1m`, and the provided GitHub context shows an open, unmerged closing PR for the same behavior.

## What This Changes

The issue requests that Anthropic Claude Opus/Sonnet 4 session and Control UI context limits use the 1M-capable model window without requiring `params.context1m: true`.

## Best Possible Solution

Land PR #66862 or an equivalent narrow patch that aligns the resolver, session/status persistence, Control UI denominator, tests, and Anthropic docs for eligible Opus/Sonnet 4.6 models while preserving explicit opt-out or auth-specific fallback semantics.

## Reproduction Assessment

Yes. A source-level repro is already encoded in `src/agents/context.test.ts`: `anthropic/claude-opus-4-6` with empty params and `allowAsyncLoad: false` returns 200k, and `/status` plus session persistence use that same synchronous resolver path.

## Solution Assessment

Unclear for the exact proposed fix. Resolver-level alignment with provider-known 1M context is the right direction, but the final patch should also keep beta-header/auth behavior and any `context1m: false` opt-out from diverging from the displayed limit.
