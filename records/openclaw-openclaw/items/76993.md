## Summary

Keep open: the classifier is the right direction for #76499, but the patch still leaves the watchdog path able to recreate the advertiser after a no-valid-addresses service stays non-announced, so it does not fully prevent the restart loop.

## What This Changes

The PR adds a Bonjour ciao classifier for the "Could not find valid addresses for interface" assertion, suppresses that classification in advertiser recovery paths, adds regression tests, and updates the changelog.

## Best Possible Solution

Treat no-valid-addresses as a terminal Bonjour-disabled state for the active advertiser cycle, with regression coverage that advances the watchdog timer and proves no recreate/re-advertise loop occurs.

## Reproduction Assessment

Yes, from source inspection. Current main misses the exact upstream ciao assertion, and the PR's remaining loop is source-reproducible because the watchdog restarts any non-announced service after 20 seconds.

## Solution Assessment

No. The classifier and immediate recovery suppression are necessary, but the patch also needs to prevent watchdog recovery for the same stable no-valid-addresses condition.

## Review Findings

- **[P2] Stop the watchdog for no-valid-addresses:** `extensions/bonjour/src/advertiser.ts:544-548`
