## Summary

Keep this PR open: it addresses a real Bonjour crash path and adds useful classifier coverage, but the patch is incomplete because the new `no-valid-addresses` kind still flows into advertiser recovery paths.

## What This Changes

The PR adds a Bonjour ciao `no-valid-addresses` classifier, regression tests for direct and wrapped suppression, and a changelog entry for IPv6-only interface assertion crashes.

## Best Possible Solution

Keep the classifier and changelog, then add explicit `no-valid-addresses` no-recovery handling in the Bonjour advertiser process and advertise-failure paths with advertiser regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: ciao v1.3.7 can throw the reported assertion, current main does not classify it, and unclassified ciao rejections can reach the fatal gateway path.

## Solution Assessment

No. The classifier is necessary, but the proposed patch is not the complete narrow fix because the new stable environment condition still triggers Bonjour recovery instead of being suppressed without restart.
