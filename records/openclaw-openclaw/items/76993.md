## Summary

Keep open. The reported bug is valid and the classifier addition is useful, but this PR is incomplete because the new `no-valid-addresses` kind still falls through Bonjour advertiser recovery paths and can churn the advertiser instead of suppressing the stable interface condition.

## What This Changes

The PR adds a Bonjour ciao `no-valid-addresses` classifier, regression tests for direct and wrapped errors, and a changelog fix entry for IPv6-only interface crashes.

## Best Possible Solution

Keep the classifier, but add explicit `no-valid-addresses` handling in the Bonjour advertiser process and advertise-failure paths that logs and suppresses without `requestCiaoRecovery`, with advertiser regression coverage and the changelog retained.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: ciao v1.3.7 can throw the reported assertion, current main does not classify it, and unclassified ciao rejections fall through to the fatal unhandled-rejection path.

## Solution Assessment

No. Adding the classifier is necessary, but this PR is not the narrow complete fix because the new kind still enters advertiser recovery paths instead of being treated like a stable environment condition.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Suppress no-valid-addresses without recovery:** `extensions/bonjour/src/ciao.ts:87`
  - body: Returning the new `no-valid-addresses` kind here still sends it through the advertiser default handling: the process handler only skips recovery for `interface-enumeration-failure`, and `handleAdvertiseFailure` calls `requestCiaoRecovery` for every classification. On an IPv6-only interface this condition will not change after recreating ciao, so the patch can replace the systemd crash loop with repeated Bonjour restarts until the advertiser disables itself. Please add explicit no-recovery handling in `advertiser.ts` and cover both rejection and advertise-failure paths.
  - confidence: 0.91
