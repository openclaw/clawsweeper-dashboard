## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Use derived paths for envelope checks:** `src/agents/session-runtime-envelope.ts:297-307`
  - body: `allowedPaths` and `deniedPaths` only inspect strings found under generic path-like keys. Opaque tools such as `apply_patch` carry their target files inside the patch text, and current main already derives those targets for policy hooks; without feeding those facts into the envelope check, an envelope like `{ allowedPaths: ["/repo/src/**"] }` can still allow an `apply_patch` payload that edits `/repo/secrets/token.txt`.
  - confidence: 0.9
- **[P2] Remove per-call session-store polling:** `src/agents/pi-tools.before-tool-call.ts:418`
  - body: `runBeforeToolCallHook` calls `readSessionRuntimeEnvelope` for each session-keyed tool call, and that helper synchronously reads and parses the session store JSON before enforcement. This puts file I/O on the tool hot path and can fail closed for unrelated calls when the store is temporarily unreadable or malformed; carry the envelope through owned session/run state or a bounded update path instead.
  - confidence: 0.86
