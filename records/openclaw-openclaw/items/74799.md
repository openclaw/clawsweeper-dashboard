## Summary

Keep open: the PR addresses a real current-main gap, but the latest head still introduces a security-sensitive path-policy bypass in the new envelope matcher, so it is not ready to merge without a fix and maintainer/security review.

## What This Changes

The PR adds optional session runtime envelopes to Gateway session create/patch, persists and enforces tool/path/command constraints before tool execution, updates protocol docs/generated Swift models/tests, and includes small config/script guardrail changes.

## Best Possible Solution

Land a narrowed envelope implementation only after matching executable path canonicalization, adding regression coverage for equivalent path spellings, and getting owner/security acceptance of the deferred network-scope semantics.

## Reproduction Assessment

Yes. Current main source shows `envelope` is absent from the session schemas/store entry, and the PR-head bypass is source-reproducible by comparing the matcher with the executing file-tool path resolver.

## Solution Assessment

No, not yet. Persisting the envelope and enforcing it through before_tool_call is the right seam, but the path matcher must canonicalize with the same workspace and home-directory semantics as the tools it is authorizing.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Resolve envelope paths like the executing tools:** `src/agents/session-runtime-envelope.ts:45`
  - body: The envelope matcher checks raw path params with only `path.resolve`/`path.normalize`, but read/write/edit resolve relative paths against the workspace cwd and expand `~` before touching the filesystem. A session with `deniedPaths: ["/workspace/secrets/**"]` can still read or write `secrets/token.txt`, because the matcher sees a relative string that does not match the deny glob while the tool later resolves it inside the denied directory. Please canonicalize path params with the same resolver used by the executing tool before applying allow/deny patterns.
  - confidence: 0.88

## Security Review

Status: needs_attention

Summary: The PR adds a new authorization boundary, and the current path matching can be bypassed by alternate spellings of the same filesystem target.

Concerns:

- **[high] Path deny rules can be bypassed:** `src/agents/session-runtime-envelope.ts:45`
  - body: Denied absolute path globs are evaluated against unanchored relative params, while the underlying file tools later resolve those params against the workspace cwd and expand home paths. This makes the persisted envelope look restrictive even though equivalent inputs can reach the denied target.
  - confidence: 0.87
