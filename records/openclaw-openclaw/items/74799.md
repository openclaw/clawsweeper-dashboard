## Summary

Keep open: current main still lacks the session-envelope protocol surface, and this PR is the active implementation for #74632, but it still needs Gateway/session/tool-authorization owner review and exact-head CI is failing.

## What This Changes

The branch adds an optional session runtime envelope to Gateway sessions.create/patch, persists it on session entries, enforces tool/path/command constraints before tool execution, and updates docs, generated Swift models, changelog, tests, and related validation/bundle guardrails.

## Best Possible Solution

Land this after Gateway/session and tool-authorization owners approve the additive contract and the exact head is green, with network constraints left to a sandbox/proxy-backed follow-up.

## Reproduction Assessment

Yes. A source-level reproduction is to send envelope to sessions.create or sessions.patch on current main; the closed schemas reject it, while the PR adds focused create/patch and enforcement tests for that behavior.

## Solution Assessment

Yes, with owner-review caveats. A session-owned schema enforced in the existing before_tool_call path is the narrow maintainable core path, and deferring network controls avoids pretending hook-level URL scanning is a reliable boundary.
