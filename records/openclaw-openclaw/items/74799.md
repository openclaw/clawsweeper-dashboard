## Summary

Keep open: this PR implements the requested Gateway session envelope feature, but the current head still has two concrete policy-enforcement defects that should block merge.

## What This Changes

The PR adds an optional `envelope` field to `sessions.create` and `sessions.patch`, persists it on session entries, enforces tool/path/command constraints before tool execution, updates protocol docs/models, and adds tests.

## Best Possible Solution

Keep the additive session envelope direction, fix native cwd propagation and envelope lookup freshness, then have Gateway/session/tool-authorization owners review the public semantics before merge.

## Reproduction Assessment

Yes. By source inspection, a Codex native PreToolUse payload with `cwd: "/repo"` and `path: "secrets/token.txt"` can miss an absolute `deniedPaths` envelope, and mixed canonical/legacy store keys can make the first envelope-bearing entry differ from the freshest session entry.

## Solution Assessment

No. The existing before-tool-call enforcement seam is plausible, but this head is not the best mergeable form until it covers native relay cwd and uses the same freshest-session targeting semantics as Gateway session reads.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Pass native cwd into envelope path checks:** `src/agents/pi-tools.before-tool-call.ts:433-434`
  - body: The envelope matcher resolves relative paths only when `ctx.workspaceDir` is populated, but the Codex native PreToolUse relay calls this hook without forwarding `invocation.cwd`. A native file-tool input like `{ path: "secrets/token.txt" }` is checked as a relative string, so `deniedPaths: ["/repo/secrets/**"]` does not match and the provider-native tool can run. Please thread the native cwd into this context and add native-relay coverage.
  - confidence: 0.9
- **[P2] Resolve the freshest session before reading envelopes:** `src/agents/session-runtime-envelope.ts:263-269`
  - body: This loop returns the first target store key that has an envelope, but Gateway session resolution chooses the freshest canonical/legacy match by `updatedAt`. If a newer alias clears or tightens the envelope while an older canonical entry still has one, enforcement can use stale policy. Resolve the freshest matching entry first, then read its envelope.
  - confidence: 0.86
