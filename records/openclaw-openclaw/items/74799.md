## Summary

## What This Changes

The PR adds an optional persisted session runtime envelope to Gateway session create/patch, enforces tool/path/command constraints before tool execution, updates protocol docs/generated Swift models, and adds focused tests.

## Best Possible Solution

Land after Gateway/session and tool-authorization owners approve the additive contract and exact-head CI is green, with network enforcement left for a sandbox/proxy follow-up.

## Reproduction Assessment

## Solution Assessment

Yes, with maintainer-review caveats. A session-owned schema enforced in the existing before_tool_call path is the narrow maintainable core path, and deferring network constraints to sandbox/proxy enforcement is safer than hook-level URL scanning.
