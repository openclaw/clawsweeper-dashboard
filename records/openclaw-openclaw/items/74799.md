## Summary

Keep open: this PR is the active implementation for the open session-envelope request, but the current patch still has blocking issues on a Gateway/tool-authorization surface. The direction is useful and the latest head added the changelog entry requested by the prior ClawSweeper comment, but the patch imports an undeclared root runtime dependency and the advertised path envelope can be bypassed by command-style tools such as Bash.

## What This Changes

The PR adds an optional persisted session runtime envelope to `sessions.create` and `sessions.patch`, enforces tool/path/command constraints in the before-tool-call path, updates protocol docs, adds focused tests, and adds a changelog entry.

## Best Possible Solution

Keep the additive session-envelope design, but land it only after core runtime dependencies are declared or avoided, command-style tools fail closed under path-scoped envelopes unless an enforceable sandbox/proxy applies, and Gateway/tool-authorization owners complete security review. Network constraints should remain a follow-up until enforced below hook-level URL scanning.

## Reproduction Assessment

Yes. Current main’s schemas reject `envelope` because the create/patch schemas omit it under `additionalProperties: false`; the PR’s command/path bypass is also reproducible by evaluating `allowedTools: ["Bash"]`, `allowedPaths: ["/repo/src/**"]`, and `bashCommandAllowlist: ["cat"]` against `Bash({ command: "cat /etc/passwd" })`, which contains no collected path-like parameter.

## Solution Assessment

No, not as currently patched. Persisting and enforcing a declarative session envelope in the before-tool-call path is the right narrow seam, but the patch must not introduce an undeclared core dependency and must not advertise path constraints that command-style tools can bypass.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Add minimatch to the root runtime dependencies:** `src/agents/session-runtime-envelope.ts:3`
  - body: `session-runtime-envelope.ts` imports `minimatch`, but the PR does not add it to the root package dependencies; at the PR head, `package.json` only pins it in `pnpm.overrides`, while the only direct dependency is the file-transfer plugin package. A clean root install or published package can fail to build or load this new core module. Add a root dependency and lockfile update, or avoid the external package here.
  - confidence: 0.9
- **[P1] Fail closed for path-scoped Bash envelopes:** `src/agents/session-runtime-envelope.ts:223-225`
  - body: When `allowedPaths` is set, this code only rejects path values that were found in path-like parameters. A Bash call carries only `command`, so `allowedTools: ["Bash"]`, `allowedPaths: ["/repo/src/**"]`, and `bashCommandAllowlist: ["cat"]` would still allow `cat /etc/passwd`. Command-style tools need to fail closed under path envelopes or run under a real sandbox boundary before this is a reliable session policy.
  - confidence: 0.88

## Security Review

Status: needs_attention

Summary: The diff adds a security-sensitive authorization surface, but currently has a concrete path-policy bypass for Bash-style commands and an undeclared core runtime dependency.

Concerns:

- **[high] Path envelope can be bypassed by Bash commands:** `src/agents/session-runtime-envelope.ts:223`
  - body: The new evaluator only applies `allowedPaths` and `deniedPaths` to structured path-like params. Bash commands have a `command` string instead, so a path-scoped session can still execute allowlisted shell commands against paths outside the envelope.
  - confidence: 0.88
- **[medium] Core imports an undeclared package:** `src/agents/session-runtime-envelope.ts:3`
  - body: The new core module imports `minimatch` without adding it to the root runtime dependencies or lockfile. That makes package resolution depend on incidental workspace/transitive layout and can break clean or published installs.
