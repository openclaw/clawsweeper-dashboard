## What This Changes

The PR adds optional per-session runtime envelopes to Gateway session create/patch schemas and persistence, enforces tool/path/command constraints in before-tool-call paths, and updates protocol docs, changelog, and focused tests.

## Best Possible Solution

Land the additive session-envelope implementation after Gateway/tool-authorization owner and security review plus changed-gate confirmation; keep network constraints and an external authorization callback as explicit follow-up work rather than adding hook-level URL scanning here.

## Solution Assessment

Yes, with merge prerequisites. Persisting a declarative session envelope and enforcing it through the existing before-tool-call path is a narrow maintainable core seam for tool/path/command constraints, while deferring network controls to sandbox/proxy enforcement is the safer boundary.
