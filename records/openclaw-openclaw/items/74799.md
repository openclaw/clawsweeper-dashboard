## What This Changes

The PR adds an optional session runtime envelope to Gateway session create/patch, persists it on session entries, enforces tool/path/command constraints before tool execution, and updates docs/generated clients/tests.

## Best Possible Solution

Adopt an additive session-owned policy contract after owner review, with network and external callback enforcement tracked separately.

Yes. Current main's closed sessions.create and sessions.patch schemas reject an envelope field, and the PR includes focused create/patch persistence plus before_tool_call enforcement tests for the new behavior.

## Solution Assessment

Yes, with maintainer-review caveats. A session-owned schema plus enforcement in the existing before_tool_call path is the narrow maintainable core path, while network constraints are better deferred to sandbox or proxy controls.
