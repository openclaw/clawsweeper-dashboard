## Summary

Keep open for maintainer review. Current main still lacks the requested Gateway session envelope, and this PR implements the expected create/patch persistence plus before-tool-call enforcement with no concrete blocking finding found; the remaining question is owner approval for a new persisted authorization boundary.

## What This Changes

This PR adds an optional session runtime envelope to Gateway session create/patch schemas, stores it on session entries, enforces tool/path/command constraints before tool execution, updates docs/changelog/generated Swift models, and adds focused tests plus small validation/bundle guardrail fixes.

## Best Possible Solution

Review and, if Gateway/session/tool-policy owners agree with the semantics, land this PR as the core session-envelope implementation while tracking network runtime enforcement separately.

## Reproduction Assessment

Yes. On current main, sending envelope to sessions.create or sessions.patch targets closed schemas that do not define the field; the PR adds focused create, patch, lookup, matcher, and before-tool-call regression coverage.

## Solution Assessment

Yes, with owner-review caveats. Persisting an additive envelope on SessionEntry and enforcing it through the existing before-tool-call path is the narrowest maintainable core path, while deferring network scope avoids a misleading hook-level boundary.
