## Summary

Keep open: the PR implements the requested session-envelope protocol and enforcement surface, but the latest diff still has one blocking package-resolution issue. Core agent runtime code now imports `minimatch` without declaring it as a root OpenClaw dependency, so strict or published installs may fail even though the monorepo can see the package through an extension dependency.

## What This Changes

The PR adds an optional `envelope` field to Gateway `sessions.create` and `sessions.patch`, persists it on session entries, enforces tool/path/command constraints before tool execution, and updates protocol docs, changelog, and focused tests.

## Best Possible Solution

Land the additive session-envelope implementation after declaring the root runtime dependency, rerunning the focused Gateway/agent tests plus changed checks, and completing owner/security review; keep network controls and external authorization callbacks as separate follow-up designs.

## Reproduction Assessment

Yes. Current main rejects `envelope` in `sessions.create` or `sessions.patch` because the schemas omit that field under `additionalProperties: false`; the PR adds focused create/patch/enforcement tests for the desired behavior.

## Solution Assessment

Yes, after the package metadata fix. Persisting a declarative session envelope and enforcing it through the existing before-tool-call path is the narrow maintainable core seam, and deferring network constraints avoids pretending hook-level URL scanning is a real sandbox boundary.
