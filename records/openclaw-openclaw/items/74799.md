## Summary

Keep open. The latest head addresses the prior ClawSweeper native-cwd and freshest-store issues, but this security-sensitive feature still has a concrete path-envelope bypass because allow/deny matching is based on lexical path normalization rather than real filesystem targets.

## What This Changes

The PR adds optional session runtime envelopes to Gateway session create/patch, persists them on session entries, enforces tool/path/command constraints in before-tool-call handling, updates docs/generated models, and adds related tests plus small guardrail changes.

## Best Possible Solution

Enforce path envelopes against real filesystem targets or nearest existing parents at the same boundary used by file tools, then rerun the focused envelope, Gateway, protocol, and changed checks before maintainer/security review.

## Reproduction Assessment

Yes, by source inspection. A session with `allowedPaths: ["/repo/src/**"]` can approve `/repo/src/link/secret` lexically even when `link` resolves outside the allowed tree, because the PR does not realpath targets or parents before matching.

## Solution Assessment

No. The feature direction is plausible, but the current hook-level path matching is not the best mergeable boundary for a security policy because it can be bypassed with symlink aliases.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P1] Resolve real paths before matching path envelopes:** `src/agents/session-runtime-envelope.ts:297-299`
  - body: The envelope check only normalizes the path string before applying `allowedPaths`/`deniedPaths`. With host file tools, a path such as `/repo/src/link/secret` is approved by `allowedPaths: ["/repo/src/**"]` even if `link` points outside `/repo/src`, and the tool then follows that symlink. Please realpath existing targets or nearest existing parents before matching, or enforce this at the file-tool boundary, and add symlink coverage.
  - confidence: 0.84

## Security Review

Status: needs_attention

Summary: The PR adds a security-sensitive per-session policy boundary, and the current path matcher can be bypassed through filesystem aliases.

Concerns:

- **[high] Symlink aliases bypass path envelopes:** `src/agents/session-runtime-envelope.ts:297`
  - body: `allowedPaths` and `deniedPaths` are matched against lexical canonical paths only. Host file tools can follow symlinks after the hook approves the lexical path, so an allowed subtree can alias a denied or out-of-scope target.
  - confidence: 0.84
