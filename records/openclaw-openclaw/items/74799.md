## Summary

Keep open: the PR is meaningful feature work for the linked Gateway session-envelope request, but it currently needs changes before merge because the new path policy boundary can be bypassed through actual filesystem targets, the branch is conflicting against current main, and the external PR has only test/CI proof rather than real behavior proof.

## What This Changes

The PR adds optional Gateway session runtime envelopes to `sessions.create` and `sessions.patch`, persists them on session entries, enforces tool/path/Bash constraints before tool execution, and updates docs, generated Swift models, and focused tests.

## Best Possible Solution

Rebase onto current main, preserve current session/tool-hook protocol changes, harden path envelope enforcement against actual file targets, and land only after Gateway/tool-authorization owners accept the policy semantics.

## Reproduction Assessment

Yes, by source inspection. Current main omits `envelope` from the closed `sessions.create` and `sessions.patch` schemas, while the PR adds the field and enforcement path.

## Solution Assessment

No, not yet. The additive protocol direction is plausible, but the path enforcement needs target-aware hardening and the branch must be rebased before maintainers can judge the final implementation.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P1] Resolve path rules against actual file targets:** `src/agents/session-runtime-envelope.ts:297-301`
  - body: The envelope currently checks only the lexical path strings collected from tool params. A session with `allowedPaths: ["/repo/src/**"]` can allow a call like `Read` on `/repo/src/link-to-secret`; in the normal non-`workspaceOnly` host tools, the read/write/edit operation can then follow that symlink to an out-of-envelope target. Since this PR exposes path rules as a runtime enforcement boundary, evaluate filesystem params through the same realpath/fs-safe-aware path resolution, or restrict these rules to sandbox/workspace-only enforcement, before allowing the call.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The diff adds a security-sensitive policy boundary, and the path constraint implementation can allow symlink-based out-of-envelope filesystem access.

Concerns:

- **[high] Path envelope can be bypassed through symlink targets:** `src/agents/session-runtime-envelope.ts:297`
