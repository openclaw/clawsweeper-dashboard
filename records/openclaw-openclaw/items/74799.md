## Summary

Keep open: current main still lacks the session envelope capability, and this PR is the active implementation for #74632, but the patch leaves a policy bypass in the Codex native PreToolUse path because native invocation cwd is not passed into envelope path canonicalization.

## What This Changes

The PR adds optional runtime envelopes to `sessions.create` and `sessions.patch`, persists them on session entries, enforces tool/path/command constraints before tool execution, updates protocol docs/generated Swift models, and adds focused tests plus related guardrail fixes.

## Best Possible Solution

Carry this PR forward after threading native cwd/workspace context through every before-tool-call entry point and adding a native relay regression, then complete maintainer/security review of the envelope contract.

## Reproduction Assessment

Yes for the review finding: source inspection shows native Codex PreToolUse includes `cwd` but the hook context omits `workspaceDir`, while the new envelope matcher relies on `workspaceDir` for relative path canonicalization. I did not run tests because this was a read-only review and `pnpm` is unavailable in this checkout.

## Solution Assessment

No. The general enforcement point is appropriate, but the current patch is incomplete until native hook relay calls pass cwd/workspace context into envelope evaluation so relative native tool paths cannot bypass absolute path constraints.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Pass native cwd into envelope path checks:** `src/agents/pi-tools.before-tool-call.ts:433-434`
  - body: The envelope evaluator now relies on `ctx.workspaceDir` to resolve relative paths before matching absolute `allowedPaths`/`deniedPaths`, but the Codex native PreToolUse relay calls `runBeforeToolCallHook` without that context even though the native invocation carries `cwd`. In native mode, an envelope such as `deniedPaths: ["/repo/secrets/**"]` will not match tool input like `{ path: "secrets/token.txt" }`, so the new filesystem policy can be bypassed for Codex native tools. Please thread the native cwd into the hook context and add a native relay regression.
  - confidence: 0.86
