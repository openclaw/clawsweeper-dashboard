## Summary

Keep open: current main still lacks the requested session envelope field, but this PR should not merge as-is because the new enforcement can read the wrong session store and skip persisted restrictions for global-scoped sessions.

## What This Changes

This PR adds an optional session runtime envelope to Gateway session create/patch, persists it on session entries, enforces tool/path/command constraints before tool execution, and updates docs, tests, and changelog.

## Best Possible Solution

Keep the declarative envelope direction, but route envelope reads through the same canonical Gateway session-store target/lookup semantics as create and patch, then add a regression for global scope with a non-main default agent and templated stores.

## Reproduction Assessment

Yes. Configure session.scope as global with a non-main default agent such as ops and a {agentId}-templated store, persist an envelope on global, then run the before-tool-call path with sessionKey global and observe the PR reader looks in the main store instead of the ops store.

## Solution Assessment

No, not as currently patched. The protocol and before-tool-call placement are a reasonable narrow design, but enforcement must find the same persisted row that Gateway mutations write before this is safe.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Use canonical session-store targeting for envelope reads:** `src/agents/session-runtime-envelope.ts:177-178`
  - body: sessions.create and sessions.patch persist global session rows through resolveGatewaySessionStoreTarget, which stores them under the configured default agent's store. This reader derives the store with resolveAgentIdFromSessionKey(sessionKey), so sessionKey global falls back to main; with a default agent like ops and a templated store, tool execution sees no envelope and allows calls that should be blocked. Use the same canonical Gateway/session-store target lookup as the mutation paths and cover that config.
  - confidence: 0.88
