## Summary

Keep open: the PR is a useful implementation path for #74632, but source review found two enforcement gaps that can bypass or misapply the new session envelope policy before merge.

## What This Changes

The PR adds an optional Gateway session runtime envelope to `sessions.create` and `sessions.patch`, persists it on session entries, enforces tool/path/command constraints in before-tool-call handling, updates protocol docs/generated models, and adds related tests plus small validation/bundle guard changes.

## Best Possible Solution

Fix the two policy gaps, then have Gateway/session/tool-authorization owners review the public envelope semantics before merge.

## Reproduction Assessment

Yes, by source inspection. `evaluateSessionRuntimeEnvelope` only resolves relative paths with `ctx.workspaceDir`, but the native `pre_tool_use` relay omits `invocation.cwd`; envelope reads also visibly iterate store keys instead of using the freshest matching session entry.

## Solution Assessment

No. The feature direction is plausible, but the current patch is not the best mergeable form because it misses a provider-native path enforcement path and can choose stale envelope data from mixed canonical/legacy session store entries.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Pass native hook cwd into envelope path checks:** `src/agents/pi-tools.before-tool-call.ts:433-434`
  - body: The new path matcher only resolves relative paths against `ctx.workspaceDir`, but the Codex native `pre_tool_use` relay calls this hook without forwarding `invocation.cwd`. A native file-tool input like `{ path: "secrets/token.txt" }` is then checked as `secrets/token.txt`, so an envelope such as `deniedPaths: ["/repo/secrets/**"]` does not match and the provider-native tool can run. Please thread the native hook cwd into this context and cover that relay path.
  - confidence: 0.9
- **[P2] Use the freshest session entry for envelope reads:** `src/agents/session-runtime-envelope.ts:263-270`
  - body: This loop returns the first matching store key that has an envelope, but Gateway session resolution chooses the freshest canonical/legacy match by `updatedAt`. If a newer legacy entry tightens or clears an envelope while an older canonical entry still has one, tool enforcement can use stale policy. Resolve the freshest matching session entry first, then read its envelope.
  - confidence: 0.84
