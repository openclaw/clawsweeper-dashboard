## Summary

Keep open. The PR is useful, but the latest head still has merge-blocking policy gaps: native relative-path tool calls can bypass path envelopes, and envelope reads can use stale alias data instead of the freshest Gateway session entry.

## What This Changes

The PR adds an optional Gateway session `envelope` to create/patch schemas, persists it, enforces tool/path/command constraints in before-tool-call handling, updates docs/generated Swift protocol models, and adds tests plus small validation/bundle guard changes.

## Best Possible Solution

Make envelope enforcement share Gateway session targeting/freshest-entry semantics and pass native hook cwd into the envelope path context before maintainer/security review of the public policy contract.

## Reproduction Assessment

Yes, by source inspection of PR head ae170e7ba511. Native PreToolUse payloads carry `cwd` but the relay omits it from hook ctx, and the new envelope lookup visibly returns the first keyed envelope instead of selecting the freshest target entry.

## Solution Assessment

No. The feature direction is plausible, but the current patch is not the best mergeable form until native cwd propagation and freshest-entry envelope lookup are fixed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Pass native cwd into envelope path checks:** `src/agents/pi-tools.before-tool-call.ts:433-434`
  - body: The new path matcher resolves relative tool paths only when hook ctx provides `workspaceDir`/`containerWorkdir`, but Codex native `pre_tool_use` still calls this hook without copying `invocation.cwd` into that ctx. A native file tool input like `{ path: "secrets/token.txt" }` under `/repo` will not match an absolute deny glob such as `/repo/secrets/**`, so provider-native tools can bypass the persisted envelope. Please thread the native cwd into this context and add a native-relay regression.
  - confidence: 0.9
- **[P2] Read the freshest session entry for envelopes:** `src/agents/session-runtime-envelope.ts:263-269`
  - body: This loop returns the first target store key that has an envelope, but Gateway session writes promote the freshest canonical/legacy match by `updatedAt`. If an older canonical row has a permissive envelope and a newer alias row tightens it, enforcement can keep using the stale permissive policy. Resolve the freshest target entry first, then read its envelope.
  - confidence: 0.86
