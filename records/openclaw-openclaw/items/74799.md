## Summary

Keep open: current main still lacks this Gateway session-envelope feature and this PR contains useful implementation work, but the branch is currently conflicting and the proposed security-sensitive tool policy boundary still has a symlink-based path enforcement blocker plus missing real behavior proof.

## What This Changes

Adds an optional Gateway session runtime envelope to `sessions.create` and `sessions.patch`, persists it on session entries, enforces tool/path/Bash constraints before tool execution, and updates docs, generated Swift protocol models, tests, and small config/bundle guardrails.

## Best Possible Solution

Rebase onto current main, preserve the current session-create and hook path-fact contracts, make path checks target-aware or explicitly sandbox/workspace-scoped, then land only after maintainer/security review and real Gateway proof.

## Reproduction Assessment

Yes. By source inspection, current main rejects `envelope` on `sessions.create`/`sessions.patch` through closed schemas, and PR-head path matching remains lexical while host file operations follow symlinks.

## Solution Assessment

No, not yet. The Gateway/session direction is plausible, but this implementation is not the narrowest safe policy boundary until target-aware path enforcement, current-main rebase integration, real behavior proof, and owner/security review are complete.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P1] Resolve envelope paths against real targets:** `src/agents/session-runtime-envelope.ts:297-301`
  - body: The envelope still authorizes only the canonical string collected from tool params. With `allowedPaths: ["/repo/src/**"]`, a host file-tool call to `/repo/src/link-to-secret` can pass even when that symlink targets `/repo/secrets/token.txt`; non-workspace host read/write/edit then use normal `fs` operations and follow the symlink. Resolve existing targets or safe parent targets with the same realpath-aware boundary used by the operation, or scope path envelopes to sandbox/workspace-only execution before allowing the call.
  - confidence: 0.84
