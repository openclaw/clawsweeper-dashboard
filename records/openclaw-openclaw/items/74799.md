## Summary

Keep open. Current main still lacks the requested Gateway session envelope, but this active external PR is not merge-ready because the path policy still authorizes lexical path strings rather than actual filesystem targets, and it also lacks after-fix real behavior proof for the new runtime authorization boundary.

## What This Changes

The PR adds an optional Gateway session runtime envelope to session create/patch, persists and enforces tool/path/Bash constraints before tool execution, updates docs/generated Swift/tests, and carries small config validation and bundled-plugin guardrail fixes.

## Best Possible Solution

Rebase onto current main, integrate the current hook path-fact model, harden path checks against actual targets or scope them to sandbox/workspace-only enforcement, then land only after maintainer/security review and real Gateway proof.

## Reproduction Assessment

Yes, by source inspection. Current main rejects the new envelope field through closed session create/patch schemas, and the PR-head path bypass is visible from lexical matching combined with host fs operations that follow symlinks.

## Solution Assessment

No, not yet. The additive Gateway/session direction is plausible, but path enforcement must be target-aware or explicitly narrower before this can serve as a runtime policy boundary.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P1] Resolve envelope paths against real targets:** `src/agents/session-runtime-envelope.ts:297-301`
  - body: The envelope currently authorizes only the canonical string collected from tool params. With `allowedPaths: ["/repo/src/**"]`, a host file-tool call to `/repo/src/link-to-secret` can pass even if that symlink targets `/repo/secrets/token.txt`; non-workspace host read/write/edit then use normal fs operations and follow the symlink. Resolve existing targets or safe parent targets with the same fs-safe/realpath-aware boundary used by the actual operation, or restrict path envelopes to sandbox/workspace-only execution before allowing the call.
  - confidence: 0.82
