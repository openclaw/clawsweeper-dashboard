## Summary

Keep open. Current main still lacks a browser backend or driver registration API: profile config, schema, CLI/client, route, runtime resolution, and tests all remain limited to built-in drivers, and the maintainer discussion treats this as a real Plugin SDK/API design gap rather than a ClawHub-only plugin request.

## What This Changes

The issue requests a pluggable browser backend interface so third-party MCP/custom adapters can register profile drivers without modifying core.

## Best Possible Solution

Design an additive, versioned browser backend registration contract in core and the Plugin SDK, then broaden profile config/schema/help/client/CLI/control routes/runtime dispatch and docs around that contract with focused contract tests.

## Reproduction Assessment

Not applicable as a feature/API request. Source inspection gives high-confidence proof of the current limitation because config, schema, route, client, runtime, and tests accept only built-in browser drivers.

## Solution Assessment

Yes, keeping this open for maintainer design is the best current outcome. Replacing the whole bundled browser plugin is possible today, but it is not the requested stable driver/backend registration seam for the bundled browser profile surface.
