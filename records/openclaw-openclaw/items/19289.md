## Summary

Keep open. Current main still lacks a browser backend or driver registration API: config, schema, route, CLI, runtime resolution, Plugin SDK surface, and tests remain limited to built-in browser drivers, and the related discussion treats this as a real Plugin SDK/API gap rather than a ClawHub-only plugin request.

## What This Changes

The issue requests a pluggable browser backend registration interface so third-party MCP/custom drivers can bind without OpenClaw core forks.

## Best Possible Solution

Design an additive, versioned browser backend registration contract in core and the Plugin SDK, then broaden profile config/schema/help/client/CLI/control routes/runtime dispatch and docs around that contract with focused contract tests.

## Reproduction Assessment

Not applicable as a feature/API request. Source inspection gives high-confidence proof of the current limitation because config, schema, route, CLI, runtime, SDK, and tests accept only built-in browser drivers.

## Solution Assessment

Yes, keeping this open for maintainer design is the best current outcome. Replacing the whole bundled browser plugin is possible today, but it is not the requested stable driver/backend registration seam for the bundled browser profile surface.
