## Summary

Keep open: current main and v2026.6.8 still expose only built-in browser drivers and no public Plugin SDK/browser registry for third-party MCP or custom browser backends, so this remains a real core/plugin API design gap.

## What This Changes

Request to add a pluggable browser backend/driver registration interface so third-party MCP or custom adapters can bind without modifying OpenClaw core.

## Best Possible Solution

Define an additive, versioned browser backend Plugin SDK contract with explicit driver registration, runtime dispatch, security/capability rules, compatibility behavior, docs, and contract tests.

## Reproduction Assessment

Yes, at source level: current main and v2026.6.8 keep browser driver config, schema, CLI/route validation, runtime resolution, and Plugin SDK surfaces closed to third-party driver IDs. No live runtime setup is needed to prove the missing registration interface because the public contracts reject arbitrary drivers before dispatch.

## Solution Assessment

Yes on direction, but not as a narrow automated patch. The maintainable solution is a maintainer-approved typed backend/adapter contract with security, lifecycle, compatibility, docs, and contract-test coverage rather than simply accepting new driver strings.
