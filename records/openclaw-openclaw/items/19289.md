## Summary

Keep open. Current main still lacks a browser backend or driver registration API: browser profile config, schema, CLI/client, route, runtime, and contract tests accept only built-in drivers, and the Plugin SDK exposes no browser-driver registration seam; the maintainer comment and related open relay issues treat this as a real API design gap rather than a duplicate or ClawHub-only item.

## What This Changes

Add a stable Plugin SDK/browser profile extension point so third-party MCP or custom browser backend drivers can be registered and selected without core patches.

## Best Possible Solution

Design an additive, versioned browser backend registration contract in core and the Plugin SDK, then broaden profile config/schema/CLI/client/routes/runtime dispatch and docs around that contract with focused contract tests.

## Reproduction Assessment

Not applicable as a feature/API request. Source inspection gives high-confidence proof of the current limitation: config, schema, route, client, runtime, and tests accept only built-in browser drivers.

## Solution Assessment

Yes, keeping this open for maintainer design is the best current outcome. A full replacement browser plugin is possible today, but it is not the same as a stable driver/backend registration seam for the bundled browser profile surface.
