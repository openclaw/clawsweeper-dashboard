## Summary

Keep open: current main and v2026.6.6 still keep browser drivers closed and the public Plugin SDK lacks a browser-backend registration contract. Related browser issues are narrower demand signals, not replacements for this security-sensitive SDK design work.

## What This Changes

The issue requests a pluggable browser backend interface so third-party MCP/custom browser drivers can be registered without modifying OpenClaw core.

## Best Possible Solution

Define an additive, versioned browser backend Plugin SDK contract with explicit capability, security, lifecycle, compatibility, documentation, and contract-test coverage.

## Reproduction Assessment

Yes, at source level: current main has closed config, schema, CLI, route, runtime, client, and SDK surfaces for browser profile drivers. No live runtime test was needed because the missing capability is visible in the public contracts and route validation.

## Solution Assessment

Yes, the extension-point direction is appropriate, but it is not a trivial string/config change. The maintainable solution is a maintainer-approved typed backend/adapter contract with security, lifecycle, and compatibility rules.
