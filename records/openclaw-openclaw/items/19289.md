## Summary

Keep open: current main and the latest release still have a closed browser driver set, and the public Plugin SDK does not expose a browser-backend registration contract. The related relay and Patchright issues are useful context, but they do not replace this broader security-sensitive core/plugin API design gap.

## What This Changes

The issue requests a browser backend plugin interface so third-party MCP or custom browser drivers can register without modifying OpenClaw core.

## Best Possible Solution

Define an additive, versioned browser backend Plugin SDK contract with capability scoping, lifecycle, runtime dispatch, compatibility, documentation, security review, and contract tests.

## Reproduction Assessment

Yes, at source level: current main and v2026.6.8 keep browser profile drivers closed in config, schema, route, CLI, and runtime code. No live runtime test is needed to prove the missing registration interface because arbitrary driver IDs are rejected before any adapter could bind.

## Solution Assessment

Yes for the requested direction, but not as a small enum/config patch. The maintainable solution is a maintainer-approved typed backend contract with lifecycle, dispatch, docs, compatibility, and security rules.
