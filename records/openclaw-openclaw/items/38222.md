## Summary

Keep open: the PR targets a real source-reproducible prompt inflation issue, but the current patch should not merge as-is because it drops current prompt mutation fields, only clamps a subset of prompt aggregation paths, and lacks after-fix real behavior proof.

## What This Changes

The branch adds a 50,000-character clamp for hook-provided systemPrompt/prependContext, clamps one embedded-runner prependContext join, and adds hook truncation tests.

## Best Possible Solution

Implement a shared post-aggregation prompt-context budget in prompt preparation that preserves and caps every public prompt mutation field across embedded, CLI, harness, queued injection, turn-prepare, and heartbeat sources.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: current main joins queued, turn-prepare, heartbeat, before_prompt_build, and legacy hook text into prompt-visible fields without a shared cap. I did not run a live oversized-plugin flow in this read-only review.

## Solution Assessment

No. The PR is a useful hardening direction, but it clamps too early and too narrowly; the maintainable fix is a shared prompt-preparation budget that preserves all current prompt mutation fields.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Return every prompt mutation field:** `src/plugins/hooks.ts:276-279`
  - body: After clamping, runBeforePromptBuild returns only systemPrompt and prependContext. Current main exposes appendContext, prependSystemContext, and appendSystemContext, so rebasing this branch as-is would silently drop those public prompt mutations before the resolver can apply them.
  - confidence: 0.94
- **[P2] Clamp after shared prompt aggregation:** `src/agents/pi-embedded-runner/run/attempt.ts:911-917`
  - body: This cap covers selected hook fields and one old embedded prepend join, but current prompt assembly also accepts append context, system-context fields, queued injections, agent_turn_prepare, heartbeat contributions, and the CLI consumer. Oversized prompt-visible text can still bypass the mitigation unless the budget is enforced on the full resolved result.
  - confidence: 0.9
