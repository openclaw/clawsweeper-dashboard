## Summary

Keep open: the unbounded plugin prompt-context path is a valid hardening target, but this PR is not merge-ready because it drops current prompt mutation fields after rebase, caps only part of the shared prompt aggregation path, and lacks required real behavior proof.

## What This Changes

The PR adds a 50,000-character clamp for plugin hook `systemPrompt` and `prependContext`, applies it in hook runners plus one embedded-runner concatenation point, and adds truncation tests.

## Best Possible Solution

Implement one shared post-aggregation prompt-context budget in prompt preparation that preserves every public prompt mutation field across embedded, CLI, queued injection, turn-prepare, heartbeat, `before_prompt_build`, and legacy `before_agent_start` sources.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main joins plugin-provided prompt-visible text from multiple hook and queued-injection sources without a shared cap; I did not run a live oversized-plugin flow in this read-only review.

## Solution Assessment

No. The PR is a useful hardening direction, but the current diff clamps too early and too narrowly; the maintainable fix should enforce a shared prompt-preparation budget after all prompt-visible sources are aggregated.

## Review Findings
