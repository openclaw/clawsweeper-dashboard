## Summary

Keep open: the prompt-size hardening target is valid on current main, but this PR is incomplete as a fix, touches security-sensitive prompt assembly, and still lacks after-fix real behavior proof.

## What This Changes

The PR adds a 50,000-character clamp and warning for selected plugin hook prompt fields, plus phase and legacy hook-runner truncation tests.

## Best Possible Solution

Implement one shared post-aggregation prompt-context budget at the resolver or prompt-preparation boundary for embedded and CLI paths while preserving every public prompt mutation field.

## Reproduction Assessment

Yes, by source inspection: current main concatenates plugin-controlled prompt-visible text from hook, queued, turn-prepare, and heartbeat sources without a shared cap. I did not run a live failing scenario in this read-only review.

## Solution Assessment

No. The direction is useful, but this PR clamps too early and too narrowly; the maintainable fix belongs after full resolver aggregation and must preserve the complete prompt mutation contract.
