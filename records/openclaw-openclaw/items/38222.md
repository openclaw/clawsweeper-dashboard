## Summary

Keep open for maintainer/security review: the reported prompt-context hardening gap is still real on current main, but this PR is incomplete and would regress public prompt mutation fields.

## What This Changes

The PR adds a 50,000-character truncation helper for selected plugin hook prompt strings, tests it for phase and legacy hooks, and clamps one embedded-runner combined prependContext join.

## Best Possible Solution

Implement a shared post-aggregation cap in the prompt-build resolver or immediately after it, covering every prompt-visible hook and queued contribution field on embedded and CLI paths while preserving the public hook contract.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence reproduction path: register a plugin that returns oversized prompt-visible text through appendContext, system-context fields, agent_turn_prepare, heartbeat_prompt_contribution, or queued injections and the resolver passes it to the model prompt without a shared cap.

## Solution Assessment

No. The PR direction is useful, but the narrowest maintainable fix is a resolver-level cap over the complete aggregated prompt-context result, not selected hook-runner fields plus one embedded-only join.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Preserve all prompt-build mutation fields:** `src/plugins/hooks.ts:276-279`
  - body: runBeforePromptBuild now returns only systemPrompt and prependContext after clamping. Current public hooks also expose appendContext, prependSystemContext, and appendSystemContext, and the resolver consumes those fields, so plugins using them would silently lose prompt mutations.
  - confidence: 0.94
- **[P1] Clamp every prompt-visible contribution:** `src/plugins/hooks.ts:277-278`
  - body: The clamp covers systemPrompt, prependContext, and one embedded combined prependContext path, but oversized text can still flow through appendContext, system-context fields, queued injections, agent_turn_prepare, heartbeat contributions, and the CLI resolver path.
  - confidence: 0.9
