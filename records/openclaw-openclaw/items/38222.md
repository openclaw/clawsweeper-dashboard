## Summary

Keep open: this is plausible security hardening, but the branch is not mergeable as-is against current main because it preserves only part of the prompt mutation contract and still leaves shared prompt-visible aggregation paths uncapped.

## What This Changes

The PR adds a 50,000-character truncation helper for plugin hook `systemPrompt`/`prependContext`, applies it in hook runners and one embedded-runner prepend join, and adds truncation tests.

## Best Possible Solution

Implement a shared post-aggregation prompt-context budget in prompt preparation that preserves every public prompt mutation field across embedded, CLI, queued injection, turn-prepare, heartbeat, and legacy hook sources.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: current main joins plugin-provided prompt-visible fields from multiple hook sources without a shared cap. I did not run a live oversized-plugin flow in this read-only review.

## Solution Assessment

No. The PR is a useful hardening direction, but it clamps too early and too narrowly; the safer solution is a shared prompt-preparation budget that preserves all current prompt mutation fields.
