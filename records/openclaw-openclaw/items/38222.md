## Summary

This PR should stay open as a useful security-hardening proposal, but it is not merge-ready because current main's prompt mutation contract and shared aggregation path have moved beyond the patch.

## What This Changes

The PR adds a 50,000-character clamp for plugin hook prompt-context strings, tests truncation for phase and legacy hooks, and clamps an old embedded-runner combined prepend-context join.

## Best Possible Solution

Implement one shared post-aggregation prompt-context budget in the current prompt preparation path, preserving every public prompt mutation field and covering embedded plus CLI consumers.

## Reproduction Assessment

Yes, at source level. Current main aggregates plugin-provided prompt-visible text from multiple hook and queued-injection sources without a shared cap, but this read-only review did not run a live oversized-plugin flow.

## Solution Assessment

No. The PR is useful hardening work, but the patch clamps too early and too narrowly; the maintainable fix belongs at the shared resolved prompt-context aggregation boundary while preserving all public fields.
