## Summary

Keep open: the PR addresses a real security-hardening gap, but the current branch is incomplete because it drops public prompt mutation fields and still leaves uncapped prompt-visible paths; it also lacks contributor real-behavior proof.

## What This Changes

The branch adds a 50,000-character clamp and warning for selected plugin hook prompt fields, related hook tests, and one embedded prepend-context join clamp.

## Best Possible Solution

Implement one shared post-aggregation prompt-context budget at the resolver or prompt-preparation boundary, preserving all five public prompt mutation fields and covering embedded and CLI consumers.

## Reproduction Assessment

Yes, source-reproducible: current main joins hook, queued, turn-prepare, and heartbeat prompt-visible strings without a shared cap, then embedded and CLI paths apply them. I did not run a live failing setup in this read-only review.

## Solution Assessment

No. The direction is useful, but the branch clamps too early and too narrowly; the maintainable fix is a shared resolver/preparation budget that preserves the full prompt mutation contract.

## Review Findings

Overall correctness: patch is incorrect

  - confidence: 0.9
