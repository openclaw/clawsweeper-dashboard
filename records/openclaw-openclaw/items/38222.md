## Summary

Keep open: the hardening goal is valid, but this PR is not safe to merge as-is because it only clamps selected prompt-visible fields, drops part of the public hook mutation contract, and lacks the required changelog entry.

## What This Changes

The PR adds a 50,000-character hook-context clamp for selected plugin hook prompt strings, clamps one embedded combined prependContext path, and adds phase/legacy hook truncation tests.

## Best Possible Solution

Land a refreshed patch or replacement that preserves the full public hook mutation contract, applies one shared post-aggregation cap to every prompt-visible plugin contribution across embedded, CLI, and harness paths, adds focused regression coverage, and includes a changelog entry.

## Reproduction Assessment

Yes. Source inspection is enough: current main concatenates plugin-controlled prompt text through queued, turn-prepare, heartbeat, phase, and legacy paths before embedded and CLI prompt assembly without a cap.

## Solution Assessment

No. The proposed direction is useful, but this implementation is not the best fix because it clamps only selected fields and one embedded join instead of enforcing a shared limit over the complete prompt-visible hook surface.
