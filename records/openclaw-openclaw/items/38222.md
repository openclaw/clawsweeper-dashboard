## Summary

Keep open for maintainer review: the hardening goal is valid, but the branch is incomplete because it drops public prompt mutation fields, leaves uncapped prompt-visible paths, and still lacks after-fix runtime proof.

## What This Changes

The PR branch adds a 50,000-character clamp and warning for selected plugin hook prompt strings, plus phase and legacy hook-runner truncation tests and an embedded prepend clamp.

## Best Possible Solution

Implement one shared post-aggregation prompt-context budget at the resolver or prompt-preparation boundary for embedded, CLI, and harness paths while preserving the complete public prompt mutation contract.

## Reproduction Assessment

Yes. By source inspection, current main concatenates plugin-controlled prompt-visible text from multiple hook and queued-contribution paths without a shared cap; I did not run a live failing setup in this read-only review.

## Solution Assessment

No. The direction is useful, but this PR clamps too early and too narrowly; the maintainable fix is a shared post-aggregation cap that preserves all public mutation fields.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Preserve all prompt mutation fields:** `src/plugins/hooks.ts:276-279`
  - body: After clamping, `runBeforePromptBuild` returns only `systemPrompt` and `prependContext`. Current main exposes `appendContext`, `prependSystemContext`, and `appendSystemContext`, so plugins using those public fields would silently lose prompt mutations before callers can apply them.
  - confidence: 0.93
- **[P2] Clamp the complete aggregated prompt context:** `src/agents/pi-embedded-runner/run/attempt.ts:911-917`
  - body: This clamps only the old embedded combined prepend path. Current main also accepts append context, system-context fields, queued injections, `agent_turn_prepare`, heartbeat contributions, and the CLI/harness paths, so oversized prompt text can still bypass unless the cap is applied after shared aggregation.
  - confidence: 0.9
