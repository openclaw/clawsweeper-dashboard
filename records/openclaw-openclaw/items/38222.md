## Summary

Keep open: current main still has unbounded prompt-visible plugin context, but this PR is not mergeable as-is because it drops public mutation fields and leaves other prompt-injection surfaces unclamped.

## What This Changes

The PR adds a 50,000-character clamp and warning for selected plugin hook prompt strings, including one embedded combined prependContext join, plus phase and legacy hook truncation tests.

## Best Possible Solution

Implement a shared post-aggregation prompt-context limit over the complete public hook and queued contribution surface, preserve all mutation fields, cover embedded and CLI paths, and document the user-facing fix in the changelog.

## Reproduction Assessment

Yes. Source inspection on current main shows resolver and runner paths concatenate plugin-controlled prompt text from before_prompt_build, before_agent_start, agent_turn_prepare, heartbeat contributions, and queued injections without a cap.

## Solution Assessment

No. The direction is useful, but the diff clamps selected fields in the hook runner and one embedded join instead of enforcing one shared cap over the full prompt-visible resolver output.
