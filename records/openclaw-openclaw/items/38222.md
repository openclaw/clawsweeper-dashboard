## Summary

Keep open. The hardening direction is useful and the earlier bot review issues were addressed, but current main still has unbounded hook-injected prompt context, and this branch is now stale, conflicting, incomplete for the current hook surface, and missing required real behavior proof.

## What This Changes

This PR adds a 50,000-character truncation helper for hook-injected `systemPrompt` and `prependContext`, applies it in the phase and legacy hook runners plus the old embedded attempt merge point, and adds hook merger tests.

## Best Possible Solution

Land a replacement or rebased patch that defines the plugin prompt-context limit, applies it after final prompt-hook aggregation for every model-visible hook context field, preserves existing prompt-injection policy, and includes focused tests plus real plugin-run proof.

## Reproduction Assessment

Yes, from source inspection: current main concatenates hook-returned prompt context and system context into the submitted prompt/system prompt without a length cap. I did not run tests because this review is required to keep the checkout read-only.

## Solution Assessment

No. The idea is plausible, but the branch is not the best current form because the final cap needs to live on the current `resolvePromptBuildHookResult` aggregation path and cover all current model-visible hook fields, not only the old `systemPrompt`/`prependContext` pair.
