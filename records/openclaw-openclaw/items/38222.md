## Summary

Keep open: current main still lacks a shared enforcement point, but this branch is not mergeable as-is because it drops current prompt mutation fields, caps only selected paths, and lacks after-fix real behavior proof.

## What This Changes

The PR adds a 50,000-character clamp for selected before_prompt_build and before_agent_start prompt strings, clamps one embedded prepend-context join, and adds hook truncation tests.

## Best Possible Solution

Implement a shared post-aggregation prompt-context budget in prompt preparation that preserves and caps every public prompt mutation field across embedded, CLI, legacy, queued, turn-prepare, and heartbeat sources.

## Reproduction Assessment

Yes, source-reproducible: current main joins multiple prompt-visible hook/context sources without a shared cap before CLI and embedded consumers apply them. I did not execute a live oversized-plugin run in this read-only review.

## Solution Assessment

No. The PR is a useful hardening direction, but it clamps too early and too narrowly and would drop current prompt mutation fields; the safer fix is a shared resolver/preparation budget that preserves all fields.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Preserve all prompt mutation fields:** `src/plugins/hooks.ts:276-279`
  - body: After clamping, runBeforePromptBuild returns only systemPrompt and prependContext. Current main exposes appendContext, prependSystemContext, and appendSystemContext, so rebasing this branch as-is would silently drop those public prompt mutations before the resolver can apply them.
  - confidence: 0.94
- **[P2] Clamp after full prompt aggregation:** `src/agents/pi-embedded-runner/run/attempt.ts:911-917`
  - body: This cap covers selected hook fields and one old embedded prepend join, but current prompt assembly also accepts append context, system-context fields, queued injections, agent_turn_prepare, heartbeat contributions, and the CLI consumer. Oversized prompt-visible text can still bypass the mitigation unless the budget is enforced on the full resolved result.
  - confidence: 0.9
