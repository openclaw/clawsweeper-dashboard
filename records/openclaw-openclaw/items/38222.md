## Summary

Keep open: the prompt-size hardening is still relevant on current main, but this PR is incomplete because it drops current public prompt mutation fields and only caps selected hook paths while other prompt-visible contributions remain uncapped.

## What This Changes

The PR adds a 50,000-character clamp and warning for selected `before_prompt_build` and `before_agent_start` prompt fields, plus hook-runner tests.

## Best Possible Solution

Implement one shared post-aggregation prompt-context limit over the full resolver output for embedded and CLI paths, preserving all public mutation fields and covering queued, turn-prepare, heartbeat, phase, and legacy contributions.

## Reproduction Assessment

Yes, by source inspection: current main concatenates plugin-controlled prompt-visible text from multiple hook and queued-contribution paths without a shared cap. I did not run a failing live scenario in this read-only review.

## Solution Assessment

No. The direction is useful, but the PR clamps too early and too narrowly; the safer fix is a resolver or prompt-preparation boundary cap that preserves the complete public hook result shape.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Preserve all prompt mutation fields:** `src/plugins/hooks.ts:276-279`
  - body: `runBeforePromptBuild` now returns only `systemPrompt` and `prependContext` after clamping. Current main exposes and consumes `appendContext`, `prependSystemContext`, and `appendSystemContext`, so plugins using those public fields would silently lose prompt mutations before the resolver can apply them.
  - confidence: 0.93
- **[P2] Clamp the full resolver output:** `src/plugins/hooks.ts:277-278`
  - body: The clamp covers selected hook fields and one embedded prepend join, but current prompt assembly also accepts `appendContext`, system-context fields, queued injections, `agent_turn_prepare`, heartbeat contributions, and the CLI resolver path. Oversized prompt text can still bypass this mitigation unless the cap is applied after the full resolver aggregation.
  - confidence: 0.9
