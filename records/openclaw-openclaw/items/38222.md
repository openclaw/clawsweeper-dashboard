## Summary

Keep open: later commits addressed the initial truncation-marker, warning-wording, and legacy-hook test comments, but current main has since expanded the prompt mutation contract and shared prompt resolver, so this branch would drop fields and still leave uncapped prompt-visible sources. The external PR also lacks after-fix real behavior proof.

## What This Changes

The branch adds a fixed 50,000-character clamp for hook-provided prompt context/system strings, clamps one embedded-runner join, and adds hook truncation tests.

## Best Possible Solution

Implement a shared post-aggregation prompt-context budget in prompt preparation that preserves and caps every public prompt mutation field across embedded, CLI, harness, queued injection, turn-prepare, and heartbeat sources.

## Reproduction Assessment

Yes, source-reproducible: current main joins queued, turn-prepare, heartbeat, before_prompt_build, and legacy hook text into prompt-visible fields with no shared cap before embedded and CLI consumers apply them. I did not run a live oversized-plugin flow in this read-only review.

## Solution Assessment

No. The PR is a useful hardening direction, but it clamps too early and too narrowly; the safer fix is a shared prompt-preparation budget that preserves all current prompt mutation fields.
