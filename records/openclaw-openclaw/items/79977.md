## Summary

Keep open: current main still has warn-level logging in the benign `active_embedded_run` recovery-skip branch, and the open closing PR at https://github.com/openclaw/openclaw/pull/79979 should be reviewed or replaced before this issue closes.

## What This Changes

The report asks for diagnostic stuck-session logging to stop flooding warn logs during healthy long-running embedded agent runs.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/79979, or apply an equivalent diagnostic-only patch, so benign `active_embedded_run` observe-only recovery logs become debug while real stuck, aborted, released, and failed recovery paths remain warn-level.

## Reproduction Assessment

Yes. Current source and tests directly exercise the `active_embedded_run` observe-only recovery branch and expect `diag.warn`; I did not run a long cron session, so this is source-reproducible rather than live-reproduced.

## Solution Assessment

Yes. The narrow maintainable fix is to demote only the benign `active_embedded_run` observe-only recovery logs to debug, and the open linked PR now targets exactly that runtime and test surface.
