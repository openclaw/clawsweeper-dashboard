## Summary

Keep open: current main still emits warn-level logs for the `active_embedded_run` observe-only recovery branch, and the focused fix PR was closed unmerged. The issue is narrower than the original 2026.4.29 report because current main now classifies actively progressing work separately, but the exact benign recovery-skip warning path remains in source and tests.

## What This Changes

The issue asks to stop warn-level diagnostic log flooding when stuck-session recovery skips a healthy active embedded run.

## Best Possible Solution

Land a narrow diagnostic-only change that demotes `active_embedded_run` observe-only recovery skip/outcome logs to debug while preserving warn-level logging for real stuck, stalled, mutating, aborted, released, or failed recovery states.
