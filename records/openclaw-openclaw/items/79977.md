Keep open: current `main` still has the warn-level `active_embedded_run` observe-only recovery path, and https://github.com/openclaw/openclaw/pull/79979 is an open linked fix that should be reviewed or replaced before this issue closes.

## What This Changes

The issue reports warn-level diagnostic log flooding for healthy long-running embedded agent runs when stuck-session recovery skips because an active embedded run is still in progress.

## Best Possible Solution

Land a narrow diagnostic recovery change that demotes only the benign `active_embedded_run` observe-only recovery-skip logs to debug while preserving warnings for real stuck, stalled, aborted, released, or failed recovery outcomes.

## Reproduction Assessment

Yes from current source and tests. The active embedded-run observe-only branch still logs through `diag.warn`, and the unit test currently asserts that warn-level behavior; I did not run the reported long cron flow live.

## Solution Assessment

Yes. The narrow maintainable fix is to demote the confirmed benign `active_embedded_run` recovery-skip logs to debug, with https://github.com/openclaw/openclaw/pull/79979 as the current landing path if it passes review.
