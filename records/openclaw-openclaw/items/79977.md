## Summary

Keep open: current main still has a warn-level `active_embedded_run` observe-only recovery path, and the open linked fix PR should be reviewed and landed or replaced before this issue closes.

## What This Changes

The issue reports warn-level diagnostic log flooding for healthy long-running embedded agent sessions that are marked as stuck and then immediately skipped with `active_embedded_run action=observe_only`.

## Best Possible Solution

Land the focused diagnostic change that demotes only benign `active_embedded_run` observe-only recovery skip/outcome logs to debug while preserving warnings for real stalled, aborting, failed, or stale recovery states.

## Reproduction Assessment

Yes, source-reproducible: current main directly logs the `active_embedded_run` observe-only recovery branch with `diag.warn`, and the colocated unit test asserts that warn-level behavior. I did not live-run a long cron session in this read-only review.

## Solution Assessment

Yes, the narrowest maintainable fix is the linked PR's log-level demotion for the benign observe-only branch, not a broad threshold increase or suppression of all stuck-session diagnostics. The issue should remain open until that PR or an equivalent patch lands.
