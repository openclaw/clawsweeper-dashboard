## What This Changes

The issue asks to stop warn-level diagnostic noise when healthy long-running embedded agent runs are recognized as observe-only recovery skips.

## Best Possible Solution

Land a focused change that demotes only benign `active_embedded_run` observe-only recovery skip/outcome logs to debug while keeping warnings for abort, release, failure, and genuinely stalled or stuck recovery states.
