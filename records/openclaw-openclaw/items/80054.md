## Summary

Keep open for maintainer review and landing. The patch is a narrow ACPX adapter fix for a source-proven Gemini ACP timeout failure, the supplied terminal proof is sufficient for the adapter boundary, CI is green, and I found no blocking review findings.

## What This Changes

The PR updates ACPX runtime command detection so Gemini ACP timeout config options are ignored, with regression coverage for `gemini --acp` and `gemini --experimental-acp`.

## Best Possible Solution

Land the small ACPX adapter fix that keeps the manager generic, no-ops only unsupported Gemini ACP timeout controls, and continues forwarding other Gemini and non-Gemini config options.

## Reproduction Assessment

Yes from source inspection. Current main converts persisted `timeoutSeconds` into `timeout` or `timeout_seconds` and ACPX forwards it for Gemini, while the upstream Gemini ACP dispatcher lacks a config-option timeout handler.

## Solution Assessment

Yes. Handling this at the ACPX adapter boundary matches the existing Codex ACP timeout no-op and keeps the generic control-plane contract intact while using the known agent command identity.
