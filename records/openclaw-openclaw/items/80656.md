## Summary

Keep this PR open for maintainer review: the compatibility bug is well supported by current source and the supplied live proof, but landing it is a compatibility and security-boundary policy decision because Swift GatewayChannel would default back to v2 signing and the new reset APIs/sibling native scope need explicit acceptance.

## What This Changes

The PR adds a Swift v2 device-auth payload builder, switches OpenClawKit GatewayChannel signing from v3 to v2, exposes Swift device identity/auth reset helpers, and adds a v2 payload vector test.

## Best Possible Solution

Land a maintainer-approved device-auth compatibility policy that keeps older deployed gateways reachable while explicitly scoping the v2 signing window, reset API surface, and sibling native client behavior.
