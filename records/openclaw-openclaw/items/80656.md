## Summary

Keep open: the compatibility direction is plausible, but the patch is incomplete because another Swift gateway connect signer still uses the v3 payload, and the external PR does not include auditable real behavior proof.

## What This Changes

The PR adds a Swift v2 device-auth payload builder, switches OpenClawKit GatewayChannel signing from v3 to v2, adds device auth reset helpers, and adds a v2 payload unit vector.

## Best Possible Solution

Update or centralize the Swift device-auth signer so every affected Swift gateway connect path uses the compatibility payload, then attach redacted real-run proof against the older gateway behavior.

## Reproduction Assessment

No high-confidence live reproduction was established in this review. Source and history make the compatibility path clear: Swift signers currently emit v3, current gateway code still knows the v2 canonical payload, and the PR only switches one Swift signer.

## Solution Assessment

No. Signing v2 is a reasonable compatibility fix because current gateways verify both v3 and v2, but this patch leaves the macOS wizard Swift connect path on v3 instead of covering the full affected Swift surface.
