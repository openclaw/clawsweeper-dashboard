## Summary

Keep open for maintainer review: the compatibility gap is still present on current main, the PR has sufficient real-world proof, and I found no blocking code-review finding, but the auth compatibility tradeoff needs explicit owner sign-off.

## What This Changes

The PR adds a Swift v2 device-auth payload builder, switches Swift gateway connect signing from v3 to v2 for older gateway compatibility, adds reset helpers, and adds a Swift payload-vector test.

## Best Possible Solution

Land a maintainer-reviewed version of this compatibility fix with the Swift v2 payload-vector regression test, while keeping v3 as the preferred target once deployed gateways are upgraded.

## Reproduction Assessment

Yes. Current-main source shows Swift signs only v3 while the gateway still has a v2 verifier fallback, and the PR body supplies redacted before/after logs from an older v2-only Fly gateway path.

## Solution Assessment

Yes, with maintainer sign-off. The patch uses an existing server-supported v2 compatibility contract in Swift instead of changing the gateway protocol, but the temporary loss of v3 metadata signature binding should be an explicit compatibility decision.
