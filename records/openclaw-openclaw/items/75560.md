## Summary

Keep open for merge review: current main still lacks standalone channel lifecycle CLI controls and `channels.restart`, while this PR adds the narrow CLI/Gateway/protocol/docs path with no discrete correctness findings from this review.

## What This Changes

The PR adds `openclaw channels start|stop|restart`, wires `channels.restart` through Gateway protocol and admin scopes, updates Swift models/docs/changelog, and adds focused CLI/Gateway tests.

## Best Possible Solution

Land the additive lifecycle controls after normal maintainer merge review, preserving non-destructive start/stop/restart semantics and leaving automatic retry/backoff policy to a separate follow-up.

## Reproduction Assessment

Yes. The linked report gives concrete stopped/disconnected WhatsApp recovery steps after a transient DNS failure, and current-main inspection confirms the standalone CLI controls and `channels.restart` method are absent while `channels.start` already exists.

## Solution Assessment

Yes. Additive CLI commands backed by admin-scoped Gateway RPCs are the narrow maintainable fix for operator-initiated recovery; changing health-monitor retry policy would be a separate product decision.
