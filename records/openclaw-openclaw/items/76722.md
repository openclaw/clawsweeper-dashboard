## Summary

Keep open: current main still has no live WhatsApp group discovery. The existing WhatsApp directory path is deliberately config-backed today, so this is a product/feature gap around live group listing rather than a narrow broken-contract bug.

## What This Changes

The issue requests live WhatsApp group discovery from `openclaw directory groups list --channel whatsapp` instead of listing only configured group allowlist keys.

## Best Possible Solution

Add a WhatsApp-owned live `listGroupsLive` adapter backed by account-scoped runtime/Baileys group metadata, while preserving the config-backed list as the offline fallback and updating tests/docs.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: the CLI chooses `listGroupsLive ?? listGroups`, WhatsApp only defines config-backed `listGroups`, and that helper returns keys from `channels.whatsapp.groups`.

## Solution Assessment

Unclear. Dynamic discovery is the useful user outcome, but it should be implemented through the existing live-directory contract in the WhatsApp plugin rather than by adding Baileys calls to the generic config helper.
