## Summary

Keep open: current main still has the reported WhatsApp directory gap because group listing is config-backed and no WhatsApp live group directory adapter is registered.

## What This Changes

The issue requests live discovery or clearer handling for WhatsApp groups when the linked account belongs to groups that are not already configured in `channels.whatsapp.groups`.

## Best Possible Solution

Add a WhatsApp-owned `listGroupsLive` path backed by account-scoped runtime metadata or an explicit Baileys discovery query, while preserving the existing config-backed lister as the offline fallback.

## Reproduction Assessment

Yes. Source inspection on current main shows the CLI would prefer `listGroupsLive`, WhatsApp only registers config-backed `listGroups`, and that lister returns configured `channels.whatsapp.groups` keys only.

## Solution Assessment

Unclear. Dynamic discovery is likely the right user outcome, but the maintainable fix should use a WhatsApp-owned live directory adapter and privacy-aware runtime boundary rather than adding Baileys calls to the shared config helper.
