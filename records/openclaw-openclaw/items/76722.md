## Summary

Keep open: current main still does not expose WhatsApp live group membership through the directory command; the empty result is source-explained and the existing live-directory seam needs a WhatsApp-owned implementation.

## What This Changes

The issue requests that WhatsApp `directory groups list` discover live group membership for a linked account instead of returning only configured group JIDs.

## Best Possible Solution

Add a WhatsApp-owned live group directory adapter through the existing `listGroupsLive` contract, backed by active runtime or gateway-owned group metadata, with config-backed groups as the offline fallback.

## Reproduction Assessment

Yes, by source inspection rather than a live account run. Current main routes WhatsApp group listing to the config-backed `listGroups` helper, which can only return keys already present in `channels.whatsapp.groups`.

## Solution Assessment

Unclear. Dynamic discovery is likely the right user outcome, but the maintainable fix should use `listGroupsLive` plus a WhatsApp-owned runtime/cache seam rather than calling Baileys from the config directory helper.
