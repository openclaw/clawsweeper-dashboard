## Summary

Keep open: current main and the latest release still expose WhatsApp group directory results from configured group allowlist keys only; account group membership is hydrated internally by the WhatsApp runtime but is not wired to `openclaw directory groups list`.

## What This Changes

The issue requests WhatsApp group directory discovery for groups the linked account belongs to instead of returning only the configured group allowlist.

## Best Possible Solution

Add a WhatsApp-owned live directory path that lists participating groups from the active WhatsApp runtime or gateway-owned cache, includes group names when available, and keeps the config-backed allowlist as the fallback.

## Reproduction Assessment

Yes, source-level: current main routes WhatsApp group directory listing to `channels.whatsapp.groups` map keys only, so account membership cannot affect the result. I did not perform a live WhatsApp account test in this read-only review.

## Solution Assessment

Unclear: dynamic discovery is likely the right user outcome, but the maintainable implementation should use the existing live-directory seam and gateway-owned WhatsApp runtime/cache rather than adding provider calls to the config helper without owner review.
