## Summary

Keep open: current `main` and v2026.6.6 still expose WhatsApp group directory as config-backed only, so a linked account's unconfigured group memberships remain undiscoverable from `openclaw directory groups list --channel whatsapp`. The requested live discovery is additive and appears feasible through the existing plugin-owned live directory seam and Baileys API.

## What This Changes

Request live WhatsApp group discovery so `openclaw directory groups list --channel whatsapp` can show groups the linked account belongs to before those JIDs are already configured.

## Best Possible Solution

Add a WhatsApp plugin-owned `listGroupsLive` directory reader that uses the active linked socket and preserves the existing config-backed fallback when no live socket is available.

## Reproduction Assessment

Yes at source level: the current WhatsApp directory contract only reads configured group keys, so an account with memberships but no configured `channels.whatsapp.groups` entries returns an empty list.

## Solution Assessment

Yes. A plugin-local `listGroupsLive` implementation is the narrowest maintainable solution because the CLI already prefers live directory readers and Baileys already exposes participating-group metadata; changing allowlist semantics would be the wrong layer.
