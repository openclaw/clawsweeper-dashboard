## Summary

Keep open: the prior ClawSweeper comment still matches current main. WhatsApp group directory remains config-backed, while live participating-group metadata is hydrated only inside the inbound runtime path and is not exposed through a WhatsApp live directory adapter.

## What This Changes

Request dynamic WhatsApp group discovery for `openclaw directory groups list --channel whatsapp` instead of returning only configured allowlist keys.

## Best Possible Solution

Expose WhatsApp group membership through a WhatsApp-owned live directory path, using runtime or cached group metadata with the existing config-backed list as the offline fallback.

## Reproduction Assessment

Yes. Source inspection shows an empty or wildcard-only `channels.whatsapp.groups` map produces no configured group IDs, and WhatsApp provides no `listGroupsLive` adapter for the CLI to use.

## Solution Assessment

Unclear. Dynamic discovery is likely the right user outcome, but the maintainable fix should use the existing live-directory contract and a WhatsApp-owned runtime/cache boundary rather than adding provider calls to the config helper.
