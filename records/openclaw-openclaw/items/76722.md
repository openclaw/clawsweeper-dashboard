## Summary

Keep open: current main still only lists WhatsApp groups from configuration and lacks a WhatsApp live group directory adapter, so the reported setup gap remains.

## What This Changes

Request live discovery or clearer handling for WhatsApp group directory listings when the linked account belongs to groups that are not already configured.

## Best Possible Solution

Add a WhatsApp-owned live group directory backed by gateway/runtime metadata or an owner-approved Baileys query path, keeping the config-backed list as an offline fallback.

## Reproduction Assessment

Yes. Source inspection shows `openclaw directory groups list` would choose `listGroupsLive` if present, but WhatsApp registers only `listGroups`, which resolves `channels.whatsapp.groups` map keys.

## Solution Assessment

Unclear. The user outcome is right, but the implementation should be owner-chosen at the WhatsApp runtime/live-directory boundary rather than adding ad hoc discovery to the config helper.
