## Summary

Keep open: current main still exposes WhatsApp group directory as a config-backed list, so the reported empty result remains valid when the linked account belongs to groups that are not already present in `channels.whatsapp.groups`.

## What This Changes

The issue requests live discovery or clearer handling for WhatsApp groups so users can find group JIDs before configuring group access.

## Best Possible Solution

Add a WhatsApp-owned live group directory adapter that returns participating groups with names when available, while preserving config-backed groups as an offline fallback.

## Reproduction Assessment

Yes. Source inspection shows the CLI falls back to WhatsApp's config-backed `listGroups` path because no `listGroupsLive` adapter is registered, and that path can only return configured `channels.whatsapp.groups` keys.

## Solution Assessment

Unclear. Dynamic discovery is likely the right user outcome, but the maintainable fix should use a WhatsApp live-directory/runtime seam rather than adding provider calls to the generic config directory helper.
