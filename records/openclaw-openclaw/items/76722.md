## Summary

Keep open: current main still lacks WhatsApp live group discovery. The directory command can use a live adapter, but WhatsApp only registers the config-backed group lister, so an empty `channels.whatsapp.groups` map still produces an empty result even when the linked account belongs to groups.

## What This Changes

Request live WhatsApp group discovery so `openclaw directory groups list --channel whatsapp` can return member groups instead of only configured allowlist keys.

## Best Possible Solution

Add a WhatsApp-owned `listGroupsLive` adapter backed by an account-scoped runtime metadata path, with the current config-backed lister kept as the offline fallback.

## Reproduction Assessment

Yes. Source inspection on current main shows the CLI would use `listGroupsLive`, but WhatsApp only registers `listGroups`, and that lister reads configured `account.groups` map keys.

## Solution Assessment

Unclear. Dynamic discovery is the right user outcome, but the best implementation needs a maintainer choice about whether the directory reads active runtime state, an inbound metadata cache, or a dedicated Baileys query path.
