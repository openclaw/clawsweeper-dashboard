## Summary

Keep open: current main and v2026.5.2 still route WhatsApp group directory listing through configured allowlist keys only, while the existing Baileys participating-group hydration is not exposed through the directory adapter.

## What This Changes

Request dynamic WhatsApp group discovery so `openclaw directory groups list --channel whatsapp` returns groups for the linked account instead of only configured `channels.whatsapp.groups` keys.

## Best Possible Solution

Add a WhatsApp-owned live `listGroupsLive` directory path backed by the active gateway socket or maintained metadata cache, with the existing config allowlist as fallback and a helpful empty-state diagnostic.

## Reproduction Assessment

Yes, source-level: with `channels.whatsapp.groups` empty or omitted, the current reader enumerates only config map keys and therefore returns no groups regardless of WhatsApp account membership. I did not perform a live WhatsApp account test in this read-only review.

## Solution Assessment

Unclear: dynamic discovery is the right user outcome, but the maintainable implementation should be chosen by the WhatsApp/plugin owners because it crosses the live gateway socket/cache boundary.
