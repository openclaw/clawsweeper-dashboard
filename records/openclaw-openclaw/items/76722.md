## Summary

Keep open: current main still has the reported WhatsApp directory gap. The directory command can only enumerate configured WhatsApp group JID keys today, while participating group metadata is hydrated inside the WhatsApp runtime and is not exposed through a live directory adapter.

## What This Changes

The issue requests that WhatsApp group directory listing discover groups the linked account belongs to instead of returning only configured group JIDs.

## Best Possible Solution

Add a WhatsApp-owned live group directory adapter that reads participating groups from the active runtime or gateway-owned metadata cache, returns group IDs and names when available, and keeps the config-backed list as the offline fallback.

## Reproduction Assessment

Yes. Source inspection reproduces the observed empty result path: WhatsApp registers only config-backed `listGroups`, and that helper can only return configured `channels.whatsapp.groups` keys.

## Solution Assessment

Unclear. Dynamic discovery is the right user outcome, but the best implementation should use the existing `listGroupsLive` directory seam and WhatsApp runtime/cache ownership instead of adding Baileys calls to the config helper.
