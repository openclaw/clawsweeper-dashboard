## Summary

Keep open: current main and the latest release still expose only the config-backed WhatsApp group directory, while https://github.com/openclaw/openclaw/pull/93996 is an open candidate fix that has not merged yet.

## What This Changes

The issue requests that WhatsApp group directory listing discover the linked account's real WhatsApp group memberships instead of only returning configured allowlist keys.

## Best Possible Solution

Land a WhatsApp-plugin `listGroupsLive` implementation that uses the existing Baileys socket membership data, returns searchable group IDs and names, keeps config fallback behavior for unavailable sockets, and proves the real CLI output before closing this issue.

## Reproduction Assessment

Yes, at source level. Current main has the CLI prefer `listGroupsLive` when present, but WhatsApp only registers config-backed `listGroups`, and that lister reads `channels.whatsapp.groups` keys, so empty config produces an empty group directory without live membership discovery.

## Solution Assessment

Yes for the issue direction. The maintainable fix belongs in the WhatsApp plugin directory adapter through the existing `listGroupsLive` seam, not in core CLI behavior or a new config option.
