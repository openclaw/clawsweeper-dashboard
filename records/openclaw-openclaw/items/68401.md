## Summary

Keep open. Current main still serializes normal same-session runs and compaction through the session lane, rejects low-latency steering while a run reports compacting, and queues busy-session followups for later instead of processing or explicitly acknowledging Telegram-style inbound messages during compaction.

## What This Changes

The issue requests that OpenClaw keep accepting or processing channel messages while a session is being compacted, instead of delaying Telegram-style real-time interactions until compaction completes.

## Best Possible Solution

Design a core session-runtime contract for nonblocking compaction or bounded busy-session acknowledgement/offload, then add Telegram and Discord coverage for inbound messages arriving during compaction without corrupting transcript order.

## Reproduction Assessment

Yes, source-level. Send a same-session inbound message while compaction is active: the session lane remains occupied, steering returns reason compacting, and the message is stored as a later followup rather than processed during compaction.

## Solution Assessment

No. Current main has useful queueing, status reactions, and a narrower Discord listener handoff, but the requested behavior needs an explicit nonblocking compaction or busy-session contract rather than a config-only switch.
