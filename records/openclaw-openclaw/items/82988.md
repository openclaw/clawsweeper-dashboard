## Summary

Keep open: current main still drops delta-only assistant chunks before Gateway and embedded TUI live chat projection, while existing SDK/event contracts already accept string `data.delta` as assistant streaming output. This is a narrow source-level bug, not a duplicate of the adjacent watchdog/commentary/session-observer issues.

## What This Changes

The issue requests live Gateway and embedded TUI chat projection support for assistant events that carry `data.delta` without cumulative `data.text`.

## Best Possible Solution

Normalize assistant `text` and `delta` once in both projection callers, accept non-empty string delta when text is absent, and reuse the existing merge/project/suppression helpers with focused regression coverage.

## Reproduction Assessment

Yes, at source level. On current main, a `stream: "assistant"` event with only `data.delta` fails the Gateway and embedded TUI `data.text` guards before the projection buffer is updated, though I did not run a live provider session.

## Solution Assessment

Yes. The narrow maintainable fix is to relax the projection input gate and pass `text: ""` plus the string delta into the existing merge/project helpers, without adding protocol fields, config, or a new UI policy.
