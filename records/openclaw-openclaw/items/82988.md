## Summary

Keep open: current main and the latest release still require cumulative `data.text` before Gateway and embedded TUI live-chat projection, while the linked closing PR remains open and not ready to land.

## What This Changes

Fix Gateway and embedded TUI live chat projection so assistant events with `data.delta` but no `data.text` produce live `chat` delta updates.

## Best Possible Solution

Land or replace the linked focused fix so both Gateway and embedded TUI accept sanitized assistant `text` or `delta` before using the existing live-chat merge helper.

## Reproduction Assessment

Yes. At source level, an assistant event with `data.delta` but no `data.text` fails the current Gateway and embedded TUI projection gates before chat delta emission.

## Solution Assessment

Yes, the narrow maintainable solution is to admit sanitized assistant `text` or `delta` at the two projection gates and reuse the existing merge helper. The open linked PR is the current implementation path, but it must be cleaned up and merged before this issue closes.
