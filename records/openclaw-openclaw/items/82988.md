## Summary

Keep open: current main still drops assistant live-chat projection when an assistant event has `data.delta` but no `data.text`, while the internal event contract and tests already treat delta-only assistant events as valid streaming deltas.

## What This Changes

Allow Gateway and embedded TUI live chat projection to treat string `data.delta` without `data.text` as an assistant incremental chunk.

## Best Possible Solution

Normalize assistant projection input in both Gateway and embedded TUI so `text` or `delta` can advance the same live-chat buffer, then add focused delta-only regression coverage.

## Reproduction Assessment

Yes at source level: an assistant event with `data.delta` but no `data.text` fails both current projection gates before reaching chat delta emission. I did not run a live Gateway/TUI scenario in this read-only pass.

## Solution Assessment

Yes: accepting sanitized `text` or `delta` at the two projection gates is the narrow maintainable fix because the shared merge helper already supports delta-only chunks. The fix should avoid broad SDK contract changes and keep commentary suppression intact.
