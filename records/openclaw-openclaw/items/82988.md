## Summary

Keep open: current `main` still has the reported text-only gates in both Gateway and embedded TUI projection, and open fix PRs already target this issue. The issue should close only after one focused fix PR lands.

## What This Changes

The issue requests that Gateway and embedded TUI live chat projection accept assistant stream events carrying `data.delta` without `data.text`.

## Best Possible Solution

Review and land one focused fix PR, preferably https://github.com/openclaw/openclaw/pull/83038 if CI and maintainer review pass, then close this issue as implemented.

## Reproduction Assessment

Yes at source level: an assistant event with `data.delta` but no `data.text` fails both current projection gates before reaching chat delta emission. I did not run a live Gateway/TUI scenario in this read-only pass.

## Solution Assessment

Yes: accepting sanitized `text` or `delta` at the two projection gates is the narrowest maintainable fix because the shared merge helper already supports delta-only chunks. The fix should preserve commentary suppression and existing cumulative-text behavior.
