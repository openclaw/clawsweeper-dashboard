## Label Justifications

- `P2`: This is a normal-priority live streaming bug with limited blast radius: final answers still arrive, but live progress can be missing.
- `impact:message-loss`: Valid assistant delta chunks are broadcast as raw agent events but not promoted into live `chat` delta frames.
- `impact:session-state`: Downstream live chat observers can show stale or quiet run state until final because the projection buffer is not advanced.

## Summary

Keep open: current main and the latest release still gate Gateway and embedded TUI chat projection on cumulative `data.text`, and the open fixing PR at https://github.com/openclaw/openclaw/pull/83000 has not merged.

## What This Changes

Fix Gateway and embedded TUI live chat projection so assistant events with `data.delta` but no cumulative `data.text` produce streaming chat deltas.

## Best Possible Solution

Land a narrow fix that lets both Gateway and embedded TUI projection gates accept sanitized assistant `text` or `delta`, while keeping commentary suppression and adding focused regression coverage.

## Reproduction Assessment

Yes at source level: emitting an assistant event with `data.delta` but no `data.text` fails both current projection gates before chat delta emission. I did not execute a live Gateway/TUI run in this read-only pass.

## Solution Assessment

Yes: the narrow maintainable fix is to normalize assistant projection input at the two gates and reuse the existing live-chat merge helper. The open linked PR is the current implementation path, but it must become clean and merged before this issue closes.
