## Label Justifications

- `P2`: This is a normal-priority live streaming bug with limited blast radius: final responses still arrive, but live progress can disappear for delta-only streams.
- `impact:message-loss`: Supported assistant delta events can be dropped from the projected `chat` delta stream, suppressing intermediate live messages.
- `impact:session-state`: Live UI/session observers can show stale or quiet running state until final because the active stream state is not updated.

## Summary

Keep open: current main still has text-only projection gates in both Gateway and embedded TUI, while the SDK and existing tests already accept delta-only assistant events; open fixing PRs reference this issue, so it should close only after a fix merges.

## What This Changes

The issue asks Gateway and embedded TUI chat projection to treat assistant events with `data.delta` but no `data.text` as live chat deltas.

## Best Possible Solution

Land a focused text-or-delta projection fix, preferably the clean open PR with supplied proof at https://github.com/openclaw/openclaw/pull/83038, then close this issue after merge.

## Reproduction Assessment

Yes at source level: an assistant event with `data.delta` but no `data.text` fails both current projection gates before reaching chat delta emission. I did not run a live Gateway/TUI scenario in this read-only pass.

## Solution Assessment

Yes: accepting sanitized `text` or `delta` at the two projection gates is the narrow maintainable fix because the shared merge helper already supports delta-only chunks. The fix should preserve commentary suppression and avoid broader SDK contract changes.
