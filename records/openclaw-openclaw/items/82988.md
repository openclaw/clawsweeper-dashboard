## Summary

Keep open: current main still has the reported text-only projection gate in both Gateway and embedded TUI, and open implementation PRs already reference this issue with closing syntax. This issue should stay open until one fix PR merges, then close as implemented.

## What This Changes

Gateway and embedded TUI should promote assistant live-chat events that contain `data.delta` even when `data.text` is absent.

## Best Possible Solution

Land one narrow fix that widens the two projection gates to accept sanitized `text` or `delta`, keeps commentary suppression unchanged, and adds focused Gateway/TUI regression coverage.

## Reproduction Assessment

Yes at source level: an assistant event with `data.delta` but no `data.text` fails both current projection gates before reaching `emitChatDelta` or the embedded TUI buffer. I did not run a live Gateway/TUI scenario in this read-only review.

## Solution Assessment

Yes: widening the two live-chat projection gates to accept a string delta while passing empty text into the existing merge helper is the narrow maintainable fix. The open implementation PRs are the right place to review and land that change.
