## Label Justifications

- `P2`: The bug affects live streaming feedback for Gateway/TUI users but final answers can still arrive, so it is normal-priority with limited blast radius.
- `impact:message-loss`: Intermediate projected `chat` delta frames are dropped for delta-only assistant events.
- `impact:session-state`: Live consumers can show stale or quiet streaming state while the run is still active.

## Summary

Keep open: current main still gates both Gateway and embedded TUI live-chat projection on `data.text`, so valid delta-only assistant events are not promoted into chat deltas. Two open closing PRs already target this issue, so the best cleanup path is to review and land one canonical fix rather than close or queue another repair.

## What This Changes

The issue reports that Gateway and embedded TUI chat projection drop assistant stream events that carry `data.delta` without cumulative `data.text`.

## Best Possible Solution

Review and land one canonical fix, preferably the open PR with sufficient proof at https://github.com/openclaw/openclaw/pull/83038, then close this issue after the fix merges.

## Reproduction Assessment

Yes at source level: an assistant event with `data.delta` but no `data.text` fails both current projection gates before reaching chat delta emission. I did not run a live Gateway/TUI scenario in this read-only pass.

## Solution Assessment

Yes: accepting sanitized `text` or `delta` at the two projection gates is the narrow maintainable fix because the shared merge helper already handles delta-only chunks. The fix should keep commentary suppression and existing cumulative-text behavior intact.
