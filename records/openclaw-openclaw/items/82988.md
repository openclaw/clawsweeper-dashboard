## Label Justifications

- `P2`: This is a normal-priority live streaming bug: final answers still arrive, but active runs can lose visible progress deltas.
- `impact:message-loss`: Valid assistant delta chunks are broadcast as raw agent events but not promoted into live `chat` delta frames.
- `impact:session-state`: Live observers can show stale or quiet run state because the chat projection buffer is not advanced for delta-only assistant chunks.

## Summary

Keep open: current main and the latest release still gate Gateway and embedded TUI live chat projection on cumulative `data.text`, while the linked fixing PR is open and unmerged.

## What This Changes

The issue requests that Gateway and embedded TUI live chat projection accept assistant events carrying `data.delta` without cumulative `data.text`.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/83000 with a narrow Gateway/TUI gate fix that reuses the existing merge helper and includes focused regression coverage.

## Reproduction Assessment

Yes at source level: emitting an assistant event with `data.delta` but no `data.text` fails both current projection gates before chat delta emission. I did not execute a live Gateway/TUI run in this read-only pass.

## Solution Assessment

Yes: the narrow maintainable solution is to admit sanitized assistant `text` or `delta` at the two projection gates and reuse the existing live-chat merge helper. The open linked PR is the current implementation path, but it must become clean and merged before this issue closes.
