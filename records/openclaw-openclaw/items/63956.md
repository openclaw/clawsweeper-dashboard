## Summary

Keep open: current main still matches the reported Control UI streaming layout bug. The streaming state commits pre-tool text into segments, the chat item builder emits each segment plus the live tail as separate stream items, and the renderer wraps each stream item in its own assistant chat group; the linked layout fix PR was closed unmerged.

## What This Changes

The issue asks for Control UI streaming assistant responses to remain in one updating assistant group instead of showing multiple bubbles during an in-flight response.

## Best Possible Solution

Implement a Control UI-only rendering fix that models one active assistant run as one outer assistant group, interleaving committed stream segments, visible tool cards, and the live tail inside that group. When tool calls are hidden, merge segment text and live text into one updating bubble; do not change gateway streaming protocol, provider behavior, or persisted transcript shape.

## Reproduction Assessment

Yes. A source-level reproduction is to call buildChatItems with at least one streamSegments entry and a non-empty live stream, then render those items: current main produces multiple stream items and each renders as its own .chat-group.assistant.

## Solution Assessment

Yes. A UI-only grouped stream-run or equivalent single outer assistant group is the narrowest maintainable fix; changing gateway deltas or transcript persistence would be broader and risks other consumers.
