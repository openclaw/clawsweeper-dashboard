## What This Changes

The issue requests Control UI webchat streaming to keep one stable assistant bubble/group during an in-flight response instead of flashing multiple bubbles that collapse after completion.

## Best Possible Solution

Keep this issue as the canonical tracker while maintainers review https://github.com/openclaw/openclaw/pull/92063; land it if the one-group outcome is accepted, or adjust it if the desired UX is a stricter single-bubble merge.

## Reproduction Assessment

Yes. Source inspection on current main shows buildChatItems emits one stream item per committed segment plus the live tail, and renderChat maps each stream item to renderStreamingGroup, whose markup is a top-level .chat-group.assistant.

## Solution Assessment

Unclear until maintainers decide the UX target. The open render-layer coalescing PR is the narrowest fix for duplicate assistant groups/avatars because it avoids gateway protocol or persisted-state changes, but it does not implement a stricter single text bubble.
