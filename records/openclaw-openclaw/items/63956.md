## What This Changes

The issue reports that Control UI streaming responses temporarily render as multiple assistant bubbles/groups before collapsing into one completed assistant message.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/92063 or an equivalent render-layer coalescing fix, then track any stricter literal single-bubble streaming UX separately if maintainers still want it.

## Reproduction Assessment

Yes, at source level. Current main emits separate stream items for committed segments and the live tail, then wraps each stream item in its own assistant group.

## Solution Assessment

Yes for the duplicate group/avatar problem. The best current path is the render-layer coalescing approach in https://github.com/openclaw/openclaw/pull/92063 because it avoids changing gateway payloads, stream reconciliation, persisted chat data, or adding config surface.
