## What This Changes

The issue asks for Control UI streaming assistant replies to update within one assistant bubble instead of rendering multiple temporary bubbles before final merge.

## Best Possible Solution

Implement a Control UI-only grouped rendering path for one active assistant run, interleaving committed segments, visible tool cards, and the live tail inside one outer assistant group.

Yes. A source-level reproduction is to call buildChatItems with at least one streamSegments entry and a non-empty stream, then render those items: current main produces multiple kind:"stream" items and each becomes its own .chat-group.assistant.

## Solution Assessment

Yes. A UI-only stream-run item or equivalent single outer assistant group is the narrowest maintainable fix; changing gateway deltas, provider behavior, or persisted transcript shape would be broader than the reported layout bug.
