## Summary

Keep open: current main still stores pre-tool streaming text as stream segments, emits each segment plus the live tail as separate stream items, and renders every stream item as its own assistant group. The linked layout-only fix PR #63986 was closed unmerged, and a search found no open PR directly referencing this issue.

## What This Changes

The issue requests that Control UI streaming assistant replies remain in one in-place assistant bubble or group instead of rendering multiple temporary bubbles that collapse on completion.

## Best Possible Solution

Implement a Control UI-only rendering fix that represents one in-flight assistant run as one outer assistant group, interleaving committed stream segments, visible tool cards, and the live tail inside that group. When tool calls are hidden, merge committed segment text and live text into one updating assistant bubble without changing gateway streaming protocol or persisted transcript shapes.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to call buildChatItems with at least one streamSegments entry plus a non-empty live stream, then render those items: current main produces multiple kind: "stream" items and each maps to a separate .chat-group.assistant.

## Solution Assessment

Yes, a UI-only grouping fix is the narrowest maintainable solution. The prior #63986 direction, or an equivalent stream-run/grouped item, addresses the layout symptom without changing gateway deltas, provider behavior, or transcript persistence.
