## What This Changes

The issue reports Control UI streaming assistant responses appearing as multiple bubbles/groups during a run and collapsing after completion instead of updating in place in one assistant bubble.

## Best Possible Solution

Review and land or adjust https://github.com/openclaw/openclaw/pull/92063, with an explicit maintainer decision on one assistant group versus a fully merged single bubble.

## Reproduction Assessment

Yes, at source level: current main builds separate stream entries for committed segments and the live tail, then renders each stream entry through a helper that creates a full assistant group. I did not run a live browser/model reproduction.

## Solution Assessment

Unclear until maintainer scope is decided: the open render-layer PR is the right layer for repeated outer groups without gateway changes, but it intentionally does not merge all segment bubbles into one bubble.
