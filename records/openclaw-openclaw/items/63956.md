## Summary

Keep open: current main still has the reported Control UI streaming layout path, where committed stream segments and the live tail become separate stream items and each stream item renders as its own assistant chat group. The direct layout PR is closed unmerged, and the open adjacent PRs address duplicate text rather than the single-bubble grouping request.

## What This Changes

Request that Control UI webchat keep one assistant bubble/group visible throughout an in-flight streaming response instead of rendering streamed fragments as separate bubbles.

## Best Possible Solution

Implement a Control UI-only grouped rendering path for one active assistant run, keeping committed text segments, visible tool cards, and the live tail inside one outer assistant group without changing gateway streaming protocol or persisted transcript shape.

## Reproduction Assessment

Yes. A source-level reproduction is to build chat items with at least one streamSegments entry and a non-empty stream, then render them: current main produces multiple kind:"stream" items and each becomes its own .chat-group.assistant.

## Solution Assessment

Yes. A UI-only stream-run item or equivalent single outer assistant group is the narrowest maintainable fix; changing gateway deltas, provider behavior, transcript persistence, or adding config would be broader than the reported layout bug.

Fix the Control UI webchat streaming fragmentation reported in https://github.com/openclaw/openclaw/issues/63956. Keep the fix UI-only: do not change gateway streaming protocol, provider behavior, persisted transcript shape, or add a config option. Implement one grouped in-flight assistant rendering path, such as a stream-run ChatItem, so committed streamSegments, visible tool cards, the live stream tail, and the reading indicator render under one outer `.chat-group.assistant`. Add focused regression coverage for segments + live stream producing one assistant group, visible tools interleaved inside that group, hidden-tool merged text behavior, and existing heartbeat/sender metadata stream sanitization.
