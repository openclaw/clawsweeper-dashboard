## What This Changes

The issue reports that Control UI webchat shows multiple assistant bubbles/groups during a streaming response before collapsing into one completed message.

## Best Possible Solution

Review and land the active render-layer grouping fix, or an equivalent replacement, that coalesces adjacent in-flight stream/reading-indicator items while leaving gateway payloads and persisted chat state unchanged.

## Reproduction Assessment

Yes. Source inspection shows `buildChatItems` can emit multiple adjacent `stream` items and `renderChat` currently renders each with its own assistant group, matching the reported visual behavior.

## Solution Assessment

Yes. The best fix is to coalesce adjacent in-flight stream items in the render layer; changing gateway streaming payloads, persistence, or adding a config knob would be broader than needed.
