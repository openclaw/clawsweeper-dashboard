## Summary

Keep open: current main and the latest release still render segmented Control UI streams as separate assistant groups, and https://github.com/openclaw/openclaw/pull/92063 is the active fix candidate rather than a merged fix.

## What This Changes

Control UI streaming assistant replies should update in one stable assistant bubble/group instead of showing multiple in-flight bubbles that collapse after completion.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/92063 or an equivalent render-layer coalescing fix, then track a stricter literal single-bubble merge separately if maintainers want that UX.

## Reproduction Assessment

Yes. Source inspection shows tool-boundary text is stored as stream segments, `buildChatItems` emits separate stream items, and the chat view renders each stream item through an assistant group wrapper.

## Solution Assessment

Yes for the duplicate assistant group/avatar problem: a render-layer coalescing fix is narrower than changing gateway payloads, transcript state, or config. The only open decision is whether maintainers also want one literal text bubble instead of one grouped assistant turn.
