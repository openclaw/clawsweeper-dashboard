Keep open. The report is fresh, concrete, and not closeable: current main intentionally keeps Microsoft out of the gateway startup plugin count and resolves speech providers lazily, so the remaining problem is the reported failed TTS path after upgrade, not startup activation itself.

## What This Changes

## Best Possible Solution
