## Summary

Keep open. The report has concrete versioned regression evidence, current `main` still lacks a gateway startup-planner path from configured TTS speech providers to their owning plugins, and PR #76540 is an open closing PR for the same issue.

## What This Changes

The issue asks for Microsoft speech TTS to activate and deliver Telegram voice output after upgrading to 2026.5.2 when `messages.tts.provider` is set to `microsoft`.

## Best Possible Solution

Review and land the linked startup-planner fix if it narrowly maps configured speech providers to their owning plugins while preserving explicit disablement, allow/deny behavior, and lazy provider resolution for non-gateway capability calls.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? No live reproduction was run in this read-only review, but the reporter supplied strong versioned steps and current source makes the missing startup activation path source-reproducible.

## Solution Assessment

Is this the best way to solve the issue? Likely yes, but it should be confirmed through PR review: adding configured speech-provider owners to gateway startup is narrower than forcing all speech plugins on or changing synthesis and Telegram delivery behavior.
