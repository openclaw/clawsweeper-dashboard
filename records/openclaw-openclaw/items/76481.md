Keep open. Current main still lacks a gateway startup-planning path from configured `messages.tts` speech providers to their owning plugins, and an open linked PR already targets this regression with closing syntax, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

Restore Microsoft TTS after 2026.5.2 by activating the configured Microsoft speech-provider plugin for gateway-started Telegram voice replies.

## Best Possible Solution

Review and land PR #76540 or an equivalent startup-planner fix that activates the owning bundled speech-provider plugin for configured `messages.tts` providers while preserving plugin disablement and lazy provider behavior where appropriate.
