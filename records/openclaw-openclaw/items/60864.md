## Summary

Keep open. Current main has adjacent restart drain, one-shot restart continuations, Task Flow durability, session-delivery recovery, and main-session recovery, but it still lacks this RFC's generic checkpoint lifecycle, plan-cursor schema, atomic at-most-once claim, and original-to-resumed run linkage.

## What This Changes

Request a first-class checkpoint and auto-resume mechanism that persists task progress before gateway restarts and starts one seeded follow-up run after startup.

## Best Possible Solution

Define a versioned restart-checkpoint contract that builds on Task Flow, restart sentinels, session delivery, and main/subagent recovery, then split implementation into focused storage, claim/resume, delivery/UI, and regression-test changes.

## Reproduction Assessment

Not applicable: this is a feature/RFC rather than a broken existing contract. Source inspection gives high confidence that current main has adjacent recovery pieces but not the requested generic checkpoint lifecycle or run-linkage contract.

## Solution Assessment

Unclear because there is no patch to assess. The seeded follow-up-run direction is plausible, but the maintainable solution needs maintainer design across task state, restart orchestration, persistence, idempotency, and delivery boundaries.
