## Summary

Keep open. Current main has adjacent durability for Task Flow state, restart sentinels, session-delivery recovery, main-session restart recovery, and subagent orphan recovery, but it still lacks the RFC's generic task checkpoint lifecycle, at-most-once checkpoint claim model, plan-cursor checkpoint schema, and original-to-resumed run linkage.

## What This Changes

The issue requests a first-class checkpoint and auto-resume mechanism for in-progress tasks across gateway restarts.

## Best Possible Solution

Define a versioned core restart-checkpoint contract that persists secret-safe task/flow progress, atomically claims resume work on startup, and links original and resumed runs while preserving the existing narrower recovery paths.

## Reproduction Assessment

Not applicable as a feature RFC. Static inspection of current main gives a high-confidence gap because the adjacent recovery paths exist but the requested checkpoint statuses, schema, claim transition, and generic startup resume flow do not.

## Solution Assessment

Unclear because no patch is proposed. The seeded follow-up-run direction is plausible, but the maintainable solution needs design review across task state, Task Flow progress, restart sentinels, session delivery, subagent recovery, and secret-safe persistence.
