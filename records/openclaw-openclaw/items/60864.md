## Summary

Keep open: current main has adjacent Task Flow durability, restart sentinels, session-delivery recovery, and main/subagent restart recovery, but it still lacks this RFC's generic checkpoint lifecycle with restart-specific task states, atomic claim/at-most-once resume, plan cursor, artifact references, and original-to-resumed run linkage. The issue remains the canonical checkpoint design thread: a newer near-duplicate was closed back to this issue, while the broader graceful-restart tracker covers a larger reliability umbrella.

## What This Changes

Request a first-class checkpoint and auto-resume lifecycle so in-progress tasks can continue once across gateway restarts via a seeded follow-up run.

## Best Possible Solution

Define a versioned restart-checkpoint contract that composes with Task Flow, restart sentinels, session delivery, and main/subagent recovery, then split storage, claim/resume, delivery/UI, and regression coverage into focused changes.

## Reproduction Assessment

Not applicable: this is a feature/RFC rather than a broken existing contract. Source inspection confirms adjacent recovery pieces exist, but the requested generic checkpoint/resume lifecycle is not present on current main.

## Solution Assessment

Unclear: the seeded follow-up-run direction is plausible, but the best maintainable solution needs maintainer-scoped contracts across task state, restart orchestration, persistence, idempotency, and delivery/UI behavior.
