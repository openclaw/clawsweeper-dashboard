## Summary

Keep open: current main has adjacent Task Flow state, restart sentinels, durable restart-continuation delivery, and main/subagent restart recovery, but it still lacks the RFC's generic checkpoint lifecycle, atomic claim/at-most-once resume contract, plan cursor, artifact references, and original-to-resumed run linkage. The latest real-world comment reinforces that queued follow-ups and active-run checkpointing are separate outage windows.

## What This Changes

The issue requests a first-class checkpoint and exactly-once auto-resume lifecycle for in-progress runs/tasks across gateway restarts.

## Best Possible Solution

Define a versioned restart-checkpoint contract that composes with Task Flow, restart sentinels, durable session delivery, main-session recovery, and subagent recovery, then split storage, claim/resume, delivery/UI, and regression coverage into focused PRs.
