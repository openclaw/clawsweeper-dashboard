## Summary

Keep open: current main has important adjacent restart-drain, one-shot continuation, durable delivery, main-session recovery, subagent recovery, and Task Flow state pieces, but it still does not implement the RFC's generic restart checkpoint lifecycle with paused/resuming/resumed states, atomic checkpoint claiming, plan cursor/artifact references, and original-to-resumed run linkage.

## What This Changes

The issue requests a first-class checkpoint and auto-resume mechanism so in-progress tasks can continue across gateway restarts without a new user prompt.

## Best Possible Solution

Define a versioned restart-checkpoint contract that composes with Task Flow, restart sentinels, durable session delivery, main-session recovery, and subagent recovery, then split storage, claim/resume, delivery/UI, and regression coverage into focused PRs.
