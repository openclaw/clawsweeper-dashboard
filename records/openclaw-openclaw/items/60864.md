## Summary

Keep open: current main has adjacent durable Task Flow state, restart sentinels, durable restart-continuation delivery, and main/subagent restart recovery, but it still lacks the RFC's generic checkpoint lifecycle with restart-specific task states, atomic claim/at-most-once resume, plan cursor, artifact references, and original-to-resumed run linkage.

## What This Changes

The issue requests a first-class checkpoint and auto-resume mechanism so in-progress tasks can continue across gateway restarts as seeded follow-up runs without manual prompting.

## Best Possible Solution

Define a versioned restart-checkpoint contract that composes with Task Flow, restart sentinels, durable session delivery, main-session recovery, and subagent recovery, then split storage, claim/resume, delivery/UI, and regression coverage into focused PRs.

Not applicable: this is a feature/RFC rather than a broken existing contract. Source inspection confirms adjacent recovery pieces exist, but the requested generic checkpoint/resume lifecycle is not present on current main.

## Solution Assessment

Unclear: the seeded follow-up-run direction is plausible, but the best maintainable solution needs maintainer-scoped contracts across task state, restart orchestration, persistence, idempotency, and delivery/UI behavior.
