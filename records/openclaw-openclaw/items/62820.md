## Summary

Keep this open. Current main has useful WhatsApp-specific outbound stripping, but the central `sanitizeUserFacingText` path named in the report still only removes `<final>` tags before delivery-facing callers, and the focused fix PR that addressed that path was closed unmerged.

## What This Changes

Strip text-form tool-call XML such as MiniMax `<minimax:tool_call><invoke ...>` blocks from user-facing messaging output when providers emit them as plain text instead of structured tool calls.

## Best Possible Solution

Add a narrow shared sanitizer fix so `sanitizeUserFacingText` removes assistant-internal tool-call text using the existing assistant-visible text helpers, including MiniMax `<invoke>` handling, while preserving normal prose and not attempting to execute plain-text tool calls.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: pass the issue's XML text through `sanitizeUserFacingText`; the current implementation only strips `<final>` tags before returning through the delivery-facing sanitizer path, so tool-call XML is not removed there.

## Solution Assessment

The reporter's direction is mostly right, but the safest implementation should reuse the full assistant-visible sanitizer/profile rather than only `stripToolCallXmlTags`, because current MiniMax cleanup is handled by `stripMinimaxToolCallXml` before the generic XML block stripper.
