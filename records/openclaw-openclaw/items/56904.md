  - body: Current main imports `PluginHookName` and `PluginHookHandlerMap` from `src/plugins/hook-types.ts`, so adding the new public hook names only to `src/plugins/types.ts` leaves the branch on an obsolete contract owner.   - body: `runBeforeToolCallHook` treats `decision: "escalate"` without approval as a hard block, but this reducer keeps running lower-priority hooks because `shouldStop` only stops on `block` or `deny`. A later plugin can attach approval handling and weaken a stricter guard into user-approvable execution.
  - confidence: 0.9
- **[P2] Honor empty subagent result rewrites:** `src/agents/subagent-announce.ts:561-565`
  - body: These truthy checks skip intentional empty-string rewrites. A guard hook that redacts with `resultText: ""` or clears `replyInstruction` leaves the original child findings or instruction in place, defeating the new hook's suppression/redaction contract.
  - confidence: 0.88
- **[P2] Define or remove the redact guard decision:** `src/plugins/types.ts:2162`
  - body: The PR exports `redact` as a guard decision, but the delivery call sites only act on `warn`, `deny`, and `escalate`. A plugin that returns `decision: "redact"` without also rewriting the payload gets no redaction behavior, which is unsafe for a security-facing API.
  - confidence: 0.84
