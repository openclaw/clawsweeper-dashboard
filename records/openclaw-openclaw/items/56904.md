Full review comments:

- **[P1] Stop escalation before lower-priority guard hooks run:** `src/plugins/hooks.ts:794`
  - body: `runBeforeToolCallHook` later treats `decision: "escalate"` without approval as a hard block, but this reducer keeps running lower-priority hooks because `shouldStop` only stops on `block` or `deny`. A later plugin can attach `requireApproval`, weakening a stricter guard into approvable execution.
  - confidence: 0.9
- **[P2] Honor empty subagent result rewrites:** `src/agents/subagent-announce.ts:561-565`
  - body: These truthy checks skip intentional empty-string rewrites. A guard hook that redacts with `resultText: ""` or clears `replyInstruction` leaves the original child findings or instruction in place, defeating the new hook's suppression/redaction contract.
  - confidence: 0.88
- **[P2] Define or remove the redact guard decision:** `src/plugins/types.ts:2162`
  - body: The PR exports `redact` as a guard decision and ranks it in the reducer, but the delivery call sites only act on `warn`, `deny`, and `escalate`. A plugin that returns `decision: "redact"` without also knowing to rewrite the payload gets no redaction behavior, which is unsafe for a public security-facing contract.
  - confidence: 0.84
- **[P2] Document the new public hook contract:** `src/plugins/types.ts:1796-1804`
  - body: The PR exports new hook names and guard decision values, but the plugin docs still describe only the existing hook catalog. External plugin authors need the accepted payloads, ordering, sync-only behavior, and failure semantics before this becomes public API.
  - confidence: 0.82
