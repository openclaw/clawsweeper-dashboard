## Summary

Keep open with a blocking finding: the proxy/root-namespace intent is useful, but changing every initial asset link to document-relative regresses direct deep-route entrypoints before JavaScript rewrites the links.

## What This Changes

The PR changes four Control UI `ui/index.html` public asset `<link>` hrefs from root-absolute paths to document-relative paths.

## Best Possible Solution

Generate initial public asset hrefs from the request/base path, or use an equivalent early path-aware mechanism, with coverage for root, default namespace, configured base path, and deep-route entrypoints.

## Reproduction Assessment

Yes for the PR-introduced regression: browser URL resolution of `href="manifest.webmanifest"` from `/skills/workshop` yields `/skills/manifest.webmanifest`, while current tests and helpers keep root-mounted deep routes on `/manifest.webmanifest`. I did not run an end-to-end oauth2-proxy setup.

## Solution Assessment

No. The requested outcome is valid, but raw document-relative HTML links are too broad; the safer fix is request-aware asset URL preparation that preserves existing root/deep-route behavior.
