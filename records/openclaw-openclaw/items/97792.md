## Summary

Keep open for a narrow contributor-doc clarification: current main and the latest release intentionally require Node >=22.19.0, but CONTRIBUTING.md does not make that source-checkout prerequisite obvious for contributors starting from the contribution guide.

## What This Changes

The issue reports that source-checkout `pnpm install` is blocked on Node v22.17.0 because the repository declares Node >=22.19.0.

## Best Possible Solution

Keep Node 22.19+ as the supported floor and add a concise contributor-facing prerequisite note to CONTRIBUTING.md or the source-development README section.

## Reproduction Assessment

Partly. Source and tests prove Node 22.17 is below the supported floor, but the exact hard `pnpm install` failure was not reproduced in a Node 22.17 environment.

## Solution Assessment

Lowering the engine floor is not the best fix without runtime proof; the best narrow fix is to make the existing Node 24 recommended / Node 22.19+ minimum requirement clear in contributor setup guidance.
