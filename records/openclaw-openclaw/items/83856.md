## Repair Work Prompt

Fix the unbounded pagination loop in `loadCronJobForShow` for `openclaw cron show`. Add a `CRON_SHOW_LOOKUP_MAX_PAGES` guard and reject non-advancing `nextOffset` values, matching the existing `loadCronJobForEditSchedulePatch` pattern without changing cron gateway contracts or adding configuration. Add focused regression tests in `src/cli/cron-cli.test.ts` for a stagnant numeric `nextOffset` and for excessive advancing pages, asserting the command exits with the pagination error instead of looping. Add a single-line active-version changelog fix entry because this is a user-visible CLI hang fix.
