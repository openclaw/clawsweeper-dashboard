## What This Changes

The issue requests fixing stale-but-preserved session token totals so `/context`, session lists, and the Control UI show the preserved value instead of treating it as unavailable.

## Best Possible Solution

Land or replace the linked PR with a narrow fix that exposes preserved totals to display/session-list consumers while keeping explicit fresh-only guards for compaction, memory, fork, and compact logic.

## Reproduction Assessment

Yes. Source inspection on current main shows finite `totalTokens` with `totalTokensFresh: false` is filtered to `undefined`, and current tests assert `/context`, session list, and Control UI hide the value; no live UI run was performed in this read-only review.

## Solution Assessment

No, not as the issue's blanket helper-only edit. The maintainable fix should separate preserved-value display semantics from fresh-only runtime semantics and update only the consumers that should show stale preserved totals.
