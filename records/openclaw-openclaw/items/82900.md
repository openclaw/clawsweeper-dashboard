## Summary

Keep this open: current main still filters preserved stale totals to undefined, and the issue is paired with an open closing PR that should be reviewed or replaced before the issue closes.

## What This Changes

The issue reports that stale-but-preserved session totalTokens are hidden, causing /context and Control UI context usage to show unavailable or reset.

## Best Possible Solution

Land a narrow fix through the linked PR or a replacement that exposes preserved totals for display while preserving fresh-only semantics for compaction-sensitive paths.

## Reproduction Assessment

Yes. Source inspection on current main shows a finite totalTokens row with totalTokensFresh false is filtered to undefined by resolveFreshSessionTotalTokens and by session-list/UI consumers.

## Solution Assessment

No, not as a one-line helper-only change. The maintainable fix should separate preserved-total display semantics from fresh-only compaction and memory decisions.
