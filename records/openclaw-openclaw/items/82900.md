## Summary

Keep open: current main and the latest release still hide finite preserved `totalTokens` whenever `totalTokensFresh` is false, and https://github.com/openclaw/openclaw/pull/82966 remains the open closing implementation path rather than a merged fix.

## What This Changes

The issue requests correcting session-token freshness handling so preserved `totalTokens` values remain visible to context and status displays when `totalTokensFresh` is false.

## Best Possible Solution

Expose preserved totals to display and session-list consumers while keeping compaction, memory flush, fork, compact, and goal decisions on explicitly fresh token state.

## Reproduction Assessment

Yes, at source level: current main can preserve a finite `totalTokens` while `totalTokensFresh` is false, and the helper still maps that state to `undefined`. I did not run a live UI session in this read-only review.

## Solution Assessment

No for the blanket helper-only suggestion: the display direction is right, but the best fix must keep fresh-only runtime decisions explicitly guarded or split into a separate helper.
