## Summary

Keep open: current main and v2026.6.10 still hide finite preserved `totalTokens` whenever `totalTokensFresh` is false, and the linked implementation PR is still open, unmerged, and conflicting.

## What This Changes

The issue requests fixing stale preserved session token totals so `/context`, Gateway session projection, and the Control UI do not show unknown or reset context usage when a numeric `totalTokens` is still available.

## Best Possible Solution

Keep this issue open until the linked PR or a focused replacement exposes preserved stale totals to display/session-list consumers while keeping compaction, memory flush, fork, compact, and goal decisions on fresh-token state.

## Reproduction Assessment

Yes, at source level: current main can preserve a finite `totalTokens` while marking `totalTokensFresh` false, and the helper still maps that state to `undefined`. I did not run a live UI session in this read-only review.

## Solution Assessment

No, closing is not the best way to solve this issue yet. The maintainable path is to review and land, or replace, the open closing PR while preserving fresh-only runtime guards.
