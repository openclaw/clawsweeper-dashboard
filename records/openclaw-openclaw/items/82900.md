## Summary

Keep open. Current `main` and the latest release still hide preserved stale totals at the source level, and https://github.com/openclaw/openclaw/pull/82966 is an open closing PR, so this issue should stay open until that PR or a replacement fix lands.

## What This Changes

The issue requests that preserved `totalTokens` remain available to display/session-list consumers even when `totalTokensFresh` is false.

## Best Possible Solution

Land a corrected version of https://github.com/openclaw/openclaw/pull/82966, or a replacement, that exposes preserved totals to display/status/session-list consumers while keeping compaction, memory flush, and fork decisions on fresh totals or transcript estimates.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: a finite `totalTokens` with `totalTokensFresh: false` is still filtered to `undefined`, and current tests encode that stale totals are hidden. I did not run a live UI session because this review is read-only and tests may write artifacts.

## Solution Assessment

No, not as a blanket helper-only change. The maintainable fix should split preserved-total display semantics from fresh-only compaction semantics, or add explicit freshness guards to every compaction-sensitive caller.
