## Summary

Keep open: current main still hides finite preserved `totalTokens` when `totalTokensFresh` is false, and the open linked PR is the active implementation path rather than a reason to close this issue before merge.

## What This Changes

The issue requests that stale-but-preserved session token totals remain visible to `/context`, session-list, and Control UI display consumers without breaking fresh-only compaction semantics.

## Best Possible Solution

Update and land the linked PR with a narrow preserved-total display path while keeping compaction, memory-flush, fork, and compact gates on fresh totals or explicit transcript estimates.

## Reproduction Assessment

Yes, at source level: a `SessionEntry` with finite `totalTokens` and `totalTokensFresh: false` is filtered to `undefined` by the helper, and `/context`, gateway rows, and Control UI display paths consume that stale-hidden state. I did not run a live UI session because this review is read-only.

## Solution Assessment

No, not as a helper-only one-line change: current preflight and memory paths still rely on fresh-only semantics. The best fix is to update https://github.com/openclaw/openclaw/pull/82966 so display consumers can show preserved stale totals while freshness-sensitive consumers keep explicit guards.
