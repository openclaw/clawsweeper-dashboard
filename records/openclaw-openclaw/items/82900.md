## Summary

Keeping this issue open: current main still filters finite preserved `totalTokens` values when `totalTokensFresh` is false, and an open linked pull request already uses closing syntax for the fix. The report is valid by source inspection, but the issue should close only after a narrow equivalent change lands.

## What This Changes

The issue requests that preserved session `totalTokens` remain available for display and estimation even when the session row is marked not fresh.

## Best Possible Solution

Land a narrow fix that exposes preserved finite totals to display/session-list consumers while retaining an explicit fresh-only path for compaction, memory, fork, and compact decisions.

## Reproduction Assessment

Yes. A current-main row with finite `totalTokens` and `totalTokensFresh: false` is enough to make `resolveFreshSessionTotalTokens` return `undefined`, and existing session-list tests encode that behavior.

## Solution Assessment

Partly. Showing preserved totals is the right end state, but the safest implementation is to separate display-safe preserved totals from fresh-only helpers rather than changing every caller’s semantics at once.
