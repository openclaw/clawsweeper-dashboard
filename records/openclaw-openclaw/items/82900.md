## Summary

Keep this open: current `main` and the latest release still hide finite preserved totals when `totalTokensFresh` is false, and https://github.com/openclaw/openclaw/pull/82966 is an open closing PR for the same bug.

## What This Changes

The issue requests that preserved session `totalTokens` remain visible to `/context`, session-list, and Control UI consumers even when `totalTokensFresh` is false.

## Best Possible Solution

Expose preserved finite totals to display/session-list consumers while keeping compaction, memory, fork, and compact decisions on explicitly fresh totals or transcript-derived estimates.

## Reproduction Assessment

Yes. A finite `totalTokens` with `totalTokensFresh: false` is filtered to `undefined` by the helper, and current tests/UI paths still encode that behavior.

## Solution Assessment

No, not as a blanket helper-only change. The safer solution is to use preserved totals for display paths while preserving fresh-only semantics for compaction-sensitive callers.
