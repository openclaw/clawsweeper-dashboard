## Summary

Keep open: current main and the latest release still filter a finite preserved `totalTokens` when `totalTokensFresh` is false, and the linked fix PR at https://github.com/openclaw/openclaw/pull/82966 is still open, conflicting, and previously reviewed with a freshness-safety blocker.

## What This Changes

The issue reports a display/session-state regression where preserved session `totalTokens` become unavailable to `/context` and session-list consumers when `totalTokensFresh` is false.

## Best Possible Solution

Land a safe fix in the linked PR or a replacement that exposes preserved totals to display/session-list consumers while keeping compaction, memory, and fork decisions on fresh totals.

## Reproduction Assessment

Yes, at source level: a `SessionEntry` with finite `totalTokens` and `totalTokensFresh: false` still flows through `resolveFreshSessionTotalTokens` to `undefined`, and current tests encode that stale projection. I did not run a live UI session in this read-only review.

## Solution Assessment

No, not as the blanket helper-only change suggested in the issue. The maintainable fix should either split preserved-total display from fresh-only runtime decisions or explicitly guard every compaction-sensitive caller.
