## Best Possible Solution

Maintainers should explicitly approve or revise the public config surface and session-id egress semantics, then land only the wrapper/tests/docs that match that decision.

## Reproduction Assessment

## Solution Assessment

The provider-owned stream wrapper is an acceptable implementation boundary, but the final config location/name and raw session-id egress semantics are maintainer decisions. A safer alternative, if maintainers reject raw ids, is a derived provider-affinity key rather than forwarding OpenClaw's session id directly.
