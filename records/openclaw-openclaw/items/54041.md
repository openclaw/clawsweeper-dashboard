## Summary

Keep open. Current main still lacks the requested Anthropic `context_management` request shaping, compaction/context-editing stream semantics, and a direct `memory_20250818` provider-defined tool path; related trackers cover important slices but do not fully replace this broader provider-design request.

## What This Changes

The issue requests Anthropic context-management beta support, including server-side compaction, context editing, beta header/body passthrough, and Memory Tool support.

## Best Possible Solution

Define one maintainer-approved Anthropic context-management path that validates the current beta contracts, adds opt-in provider/config docs, round-trips compaction/context-editing metadata correctly, and keeps Memory Tool support on the plugin/ClawHub path unless a concrete SDK seam is missing.

## Solution Assessment

Unclear for autonomous implementation. Header passthrough alone is not enough because compaction blocks, `compaction` stop reasons, context-edit metadata, prompt-cache behavior, and Memory Tool filesystem boundaries need provider/product design.
