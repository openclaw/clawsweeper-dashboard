## Summary

Keep open. Current main still lacks the requested Anthropic context-management implementation beyond configurable Anthropic beta-header append and existing OpenAI-only server compaction; the issue discussion also shows production demand for compaction blocks, pause/replay behavior, and Memory Tool support rather than a stale cleanup item.

## What This Changes

Request native or passthrough support for Anthropic Memory Tool, server-side compaction, context editing, required beta headers, and related provider-defined tool/request-body handling.

## Best Possible Solution

Keep this open for bundled Anthropic provider design. The end state should be a documented opt-in Anthropic context-management surface that validates and sends `context_management.edits`, adds required beta headers only when enabled, preserves compaction blocks through streaming and replay, handles `pause_after_compaction` continuation behavior, records context-editing metadata, and deliberately supports `memory_20250818` either as provider-owned memory behavior or through a generic provider-defined-tool SDK seam.
