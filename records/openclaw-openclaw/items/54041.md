## Summary

Keep open. Current main still lacks Anthropic `context_management` request-body wiring, compaction/context-editing response handling, and a provider-defined Memory Tool path; the Memory Tool-only request at https://github.com/openclaw/openclaw/issues/48558 was closed to the ClawHub/plugin path, but the open compaction trackers do not cover this whole provider design.

## What This Changes

Add native or explicit passthrough support for Anthropic context-management beta features, including server-side compaction, context editing, beta headers, and Memory Tool/provider-defined tools.

## Best Possible Solution

Keep this open for a maintainer-approved Anthropic provider design: documented opt-in params and beta handling, validated `context_management.edits` for compaction and context editing, stream/replay support for compaction and applied-edit state, and Memory Tool only through a ClawHub/plugin path or an explicit filesystem security boundary.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Current-main source, docs, tests, release-tag, and related-issue inspection give a high-confidence absence check for the requested Anthropic provider support.

## Solution Assessment

Unclear for final implementation. Header passthrough alone is not the best complete fix because request validation, compaction blocks, context-edit metadata, stop reasons, replay behavior, post-compaction reinjection, and Memory Tool filesystem execution all need provider/product design.
