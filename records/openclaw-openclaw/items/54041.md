Keep open. Current main still lacks the requested Anthropic request-body context-management semantics, and the related items cover only parts of the broader provider design rather than superseding it.

## What This Changes

Request Anthropic-native or passthrough support for beta headers, `context_management` compaction/context editing, and `memory_20250818` provider-defined tool handling.

## Best Possible Solution

Define a maintainer-approved Anthropic provider surface that opt-in validates/sends context-management edits and beta headers, round-trips compaction/context-editing stream state, and leaves Memory Tool filesystem behavior behind a deliberate provider-owned or plugin-owned security boundary.
