## Summary

The reported OpenClaw behavior matches the current source contract: Anthropic extra-usage responses from the Claude CLI backend are classified as billing and placed on the default billing cooldown. The remaining action is Anthropic account or Extra Usage eligibility, unless a new report includes logs showing OpenClaw selected the wrong credential or produced a different upstream error.

## What This Changes

The issue reports that Claude CLI direct auth works, but OpenClaw receives an Anthropic extra-usage response and disables the `anthropic:claude-cli` profile as billing for five hours.

## Best Possible Solution

Keep the billing classification/backoff for Anthropic extra-usage responses and direct affected users to enable Anthropic Extra Usage or use Anthropic API-key auth for predictable billing.

## Reproduction Assessment

No high-confidence live reproduction is available without the reporter's Anthropic account. Source inspection does reproduce the OpenClaw side: when Claude CLI returns the reported extra-usage text, current main classifies it as billing and applies the default five-hour disabled state.

## Solution Assessment

Not applicable as a source fix: OpenClaw is applying its documented billing response path to a vendor extra-usage response. A code change would need new logs proving wrong credential selection, stale blocked prompt metadata, or another OpenClaw-controlled request difference.
