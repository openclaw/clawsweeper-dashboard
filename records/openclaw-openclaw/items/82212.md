## Summary

Keep open. The fresh v2026.5.12 report matches a real current-main path: `claude-cli` result errors containing “out of extra usage” classify as `billing`, and billing failures put the shared auth profile into a long disabled window. Related open issues cover helper-lane poisoning, stale recovery, and warning-event variants, but none clearly owns this first-turn direct-CLI-vs-OpenClaw mismatch.

## What This Changes

Investigate and fix Claude CLI OAuth runs where OpenClaw receives Anthropic extra-usage errors and disables `anthropic:claude-cli` even though direct `claude -p` succeeds.

## Best Possible Solution

Keep this open and isolate the current Claude CLI first-turn failure with a raw stream-json fixture or live redacted proof, then adjust only the Claude CLI/OAuth failure handling needed to avoid false 5-hour shared disables while preserving true billing backoff.
