## Summary

This is an Anthropic Extra Usage/account eligibility response rather than an OpenClaw source bug: current code and tests intentionally classify Claude CLI “extra usage” responses as billing and current docs warn that OpenClaw-shaped Claude CLI turns can hit extra-usage billing even after a direct `claude -p` smoke succeeds.

## What This Changes

The issue reports that direct Claude CLI auth works, but OpenClaw’s `anthropic:claude-cli` path receives an Anthropic extra-usage error and disables the profile as billing for five hours.

## Best Possible Solution

Keep the billing classification/backoff for Anthropic extra-usage responses and direct affected users to enable Anthropic Extra Usage or use Anthropic API-key auth; reopen only with logs showing OpenClaw selected the wrong credential or produced a different upstream error.

## Reproduction Assessment

No high-confidence live reproduction was possible without the reporter’s Anthropic account. Source and tests do show that when Claude CLI returns the reported “out of extra usage” text, current main classifies it as billing and applies the default five-hour disable.

## Solution Assessment

Not applicable as a source fix: OpenClaw is applying its documented billing response path to a vendor extra-usage response. The safer path is account setup guidance or a new actionable report with logs proving wrong credential selection.
