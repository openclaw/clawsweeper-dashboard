## Summary

The remaining maintainer decision is compatibility-sensitive because malformed `--timeout` values and repeated routed value flags can now fail or resolve differently instead of silently using the old fast-path behavior.

## What This Changes

The PR updates shared CLI argv value helpers and tests so routed `status`/`health --timeout` invalid values decline fast routing and reach Commander validation.

## Best Possible Solution

Land the narrow validation fix and regression coverage if maintainers accept fail-fast invalid timeout handling across routed and Commander CLI paths.

## Reproduction Assessment

Yes, source-reproducible. Current main and v2026.6.8 treat invalid present routed timeout values as absent, while the registered Commander path already rejects the same invalid timeout input.

## Solution Assessment

Yes. The PR fixes the ambiguity at the shared positive-integer flag helper used by the status/health fast routes, adds targeted regression coverage, and defers invalid shapes back to the existing Commander error path instead of duplicating error handling.
