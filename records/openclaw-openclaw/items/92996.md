## What This Changes

The PR changes shared CLI argv parsing and route-args tests so `status` and `health` fast routes reject present-but-invalid `--timeout` values and repeated scalar flags use the final value.

## Best Possible Solution

Land the narrow validation fix if maintainers accept fail-fast invalid timeout handling across routed and Commander paths; otherwise ask for a narrower status/health-only parser adjustment.

## Reproduction Assessment

Yes, source-reproducible. Current main and `v2026.6.8` treat invalid present routed timeout values as absent, while the registered Commander path already rejects the same invalid timeout input.

## Solution Assessment

Yes. Fixing the shared positive-integer flag helper keeps the status/health fast routes aligned with Commander and avoids duplicating raw-token validation in each routed parser.
