## Summary

## What This Changes

The PR updates Control UI config form coercion to choose a const-matching object variant for `anyOf`/`oneOf` unions and adds regression tests.

## Best Possible Solution

Land the localized `coerceFormValues()` variant-selection fix with its regression tests after proof is added and exact-head checks are green.

## Reproduction Assessment

Yes. Source inspection on current main shows an object-valued `anyOf`/`oneOf` branch still recurses into the first object variant, so a later const-matching variant can have nested fields coerced with the wrong schema.

## Solution Assessment

Yes for the code change. The PR keeps the fix local to config form coercion, preserves primitive and array union behavior, and covers the earlier unconstrained-variant edge case; the remaining blockers are proof and exact-head validation.
