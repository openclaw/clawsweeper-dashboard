The PR changes Control UI config form coercion to prefer const-matching object variants in `anyOf`/`oneOf` unions and adds regression tests.

## Best Possible Solution

Yes, at source level. Current `main` recurses into the first object variant for object-valued `anyOf`/`oneOf` schemas, so a numeric first variant and later string const match can coerce `"03"` to `3`.

## Solution Assessment

Yes for the code direction. The PR keeps the fix local to config form coercion, preserves primitive and array union handling, and includes the unconstrained-variant edge case; merge should wait for proof and green checks.
