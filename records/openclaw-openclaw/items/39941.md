## Summary

Keep open: current main still has the first-object-variant behavior, and this PR is a focused useful fix; the remaining blocker is missing real Control UI config-save proof plus refreshed validation.

## What This Changes

The branch adds const-aware object-variant selection to Control UI config form coercion and regression tests for `anyOf`/`oneOf` object unions.

## Best Possible Solution

Merge the const-aware object-variant selector with its regression tests after redacted real Control UI config-save proof and refreshed focused validation are available.

## Reproduction Assessment

Yes, at source level: current main returns the first object-valued `anyOf`/`oneOf` variant for object values, so a const-discriminated later variant can have nested fields coerced with the wrong schema. I did not run tests because this review is read-only.

## Solution Assessment

Yes: preferring variants whose top-level property `const` constraints match, then falling back to the first object variant, is the narrow maintainable fix for this coercion bug. Merge should wait for real behavior proof and refreshed validation.
