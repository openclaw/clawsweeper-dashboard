## Summary

Keep open. The reported duplication bug is real, but this patch filters too broadly: it would also drop assistant rows that are the only persisted visible reply in session-memory summaries, and the PR only includes test proof rather than after-fix real behavior proof.

## What This Changes

The PR changes bundled session-memory transcript rendering to skip OpenClaw delivery-mirror and gateway-injected assistant rows and adds regression coverage.

## Best Possible Solution

Session-memory should filter only redundant transcript bookkeeping while preserving assistant text that is the only user-visible reply, matching existing visible-history invariants.

## Reproduction Assessment

Yes from source inspection: current main’s session-memory reader extracts text from any assistant message, while the PR would skip all OpenClaw delivery-mirror and gateway-injected assistant rows before extraction. I did not run tests because this review is explicitly read-only.

## Solution Assessment

No. The right shape is a duplicate-aware filter or shared visible-conversation projection that preserves standalone delivery mirrors and non-duplicate gateway-injected text instead of dropping every transcript-only OpenClaw assistant row.
