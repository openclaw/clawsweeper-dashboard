## Summary

Keep open: current main still routes `openclaw status` through pre-Gateway config/SecretRef, update, agent, summary, and task-inspection work without a command-wide timeout, and there is no post-v2026.5.18 source change in the implicated status files.

## What This Changes

The issue reports that `openclaw status` and its JSON/all variants hang on macOS v2026.5.18 before any gateway WebSocket connection, while other diagnostic commands succeed.

## Best Possible Solution

Make the status command honor a command-wide timeout across config SecretRef resolution, pre-Gateway scans, task summary imports, and gateway calls, then prove the fix against the released npm package on macOS.

## Reproduction Assessment

No high-confidence live reproduction yet. The reporter gives a concrete v2026.5.18 macOS/npm command matrix, and source inspection shows unbounded pre-Gateway status work, but I did not run the packaged macOS repro.

## Solution Assessment

Unclear until package reproduction identifies the blocking step. The narrow maintainable direction is to bound the status startup/import/summary path, not to close this as implemented.
