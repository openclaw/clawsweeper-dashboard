## Summary

Keep open. Current main still leaves the Feishu setup/status probe on top-level credentials, and open PR #74060 is already the linked implementation candidate for this account-scoped credentials bug.

## What This Changes

Feishu health/status should treat account-scoped `channels.feishu.accounts.*.appId/appSecret` credentials as configured and probeable instead of requiring top-level credentials.

## Best Possible Solution

Land PR #74060 or an equivalent narrow Feishu plugin change that resolves the selected account before setup/status probing, with regression coverage for account-scoped credentials.

## Reproduction Assessment

Yes. A config with `channels.feishu.defaultAccount` and credentials only under `channels.feishu.accounts.<id>` exercises the source-level mismatch: configured detection sees the account, while `resolveStatusLines` still probes only top-level credentials.

## Solution Assessment

Yes. Routing the Feishu setup/status probe through the existing resolved-account path is the narrow plugin-owned fix and avoids adding a core health special case.
