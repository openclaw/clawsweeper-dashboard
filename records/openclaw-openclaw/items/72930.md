## Summary

Keep open. Current main has account-aware Feishu health/runtime resolution, but the setup/status fallback still probes only top-level Feishu credentials, and open PR #74060 is already the closing implementation candidate.

## What This Changes

Make Feishu health/status recognize account-scoped app credentials in multi-account configs instead of reporting the channel as unconfigured.

## Best Possible Solution

Land PR #74060 or an equivalent Feishu plugin change that resolves the selected account before setup/status probing and keeps regression coverage.

## Reproduction Assessment

Yes for the remaining source-level mismatch: a config with `defaultAccount` and credentials only under `accounts.<id>` leaves `resolveStatusLines` without probe credentials because it inspects the top-level Feishu config. The exact old OFF/not configured output appears partly addressed in current main's health path, so closure should wait for the open closing PR.

## Solution Assessment

Yes. Routing the Feishu setup/status probe through the existing resolved-account path is the narrow plugin-owned fix and avoids adding a core health special case.
