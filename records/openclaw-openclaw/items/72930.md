## Summary

Keep open. The Feishu multi-account credential gap remains source-valid on current main, and open PR #74060 is already the linked closing fix candidate.

## What This Changes

The issue asks Feishu health/status output to treat `channels.feishu.defaultAccount` plus `channels.feishu.accounts.*.appId/appSecret` as configured.

## Best Possible Solution

Land PR #74060 or an equivalent Feishu plugin change that resolves the selected account before setup/status probing and keeps regression coverage for account-scoped credentials.

## Reproduction Assessment

Yes. A config with `channels.feishu.defaultAccount` and credentials only under `channels.feishu.accounts.<id>` exercises the source-level mismatch: setup/status can classify Feishu as configured, but the probe still inspects only the top-level Feishu config.

## Solution Assessment

Yes. Reusing the existing Feishu account resolver for the setup/status probe is the narrow plugin-owned fix, while core health already has generic account-aware seams.
