## Summary

Keep open. Current main still has a Feishu setup/status probe path that starts from top-level credentials instead of the resolved account, and open PR #74060 already targets this issue with closing syntax.

## What This Changes

The issue reports that Feishu health/status shows not configured when credentials are stored under `channels.feishu.accounts` in a multi-account config.

## Best Possible Solution

Review and land PR #74060, or an equivalent narrow Feishu plugin change, so setup/status probing resolves the selected/default Feishu account before probing.

## Reproduction Assessment

Yes. A config with `channels.feishu.defaultAccount` and credentials only under `channels.feishu.accounts.<id>` exercises the current source-level mismatch: configured detection can see the account, but setup/status probing still starts from top-level credentials.

## Solution Assessment

Yes. Reusing the existing Feishu resolved-account path is the narrowest maintainable fix and keeps the behavior inside the Feishu plugin rather than adding a core health special case.
