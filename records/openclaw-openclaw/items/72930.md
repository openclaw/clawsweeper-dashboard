## Summary

Keep open. The report is valid enough to preserve, and open PR #74060 already targets the remaining Feishu setup/status account-credential gap with closing syntax, so this issue should wait for that PR to land or be closed by maintainers.

## What This Changes

Health/status should recognize Feishu credentials stored under `channels.feishu.accounts.*` instead of requiring top-level `channels.feishu.appId/appSecret`.

## Best Possible Solution

Land PR #74060 or an equivalent narrow Feishu plugin change that resolves the selected Feishu account before setup/status probing, with regression coverage for account-scoped credentials.

## Reproduction Assessment

Yes. A config with `channels.feishu.defaultAccount` and credentials only under `channels.feishu.accounts.<id>` exercises the source-level mismatch: setup/status can classify the channel as configured, but its probe still starts from top-level credentials.

## Solution Assessment

Yes. Resolving the Feishu setup/status probe through the existing account resolver is the narrow plugin-owned fix and avoids adding Feishu-specific behavior to core health.
