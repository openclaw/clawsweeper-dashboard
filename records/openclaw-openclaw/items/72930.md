## Summary

Keep open. The issue is still paired with open PR #74060 using closing syntax, and current main still has a Feishu setup/status probe path that checks only top-level credentials even though the gateway health path itself now resolves account-scoped credentials.

## What This Changes

The issue asks for Feishu health/status output to treat multi-account credentials under `channels.feishu.accounts.*.appId/appSecret` as configured instead of reporting OFF/not configured.

## Best Possible Solution

Review and either land PR #74060 or close it if maintainers decide the current gateway health behavior is sufficient. The desired end state is one consistent Feishu status path that resolves the selected account before checking or probing credentials, with regression coverage for account-scoped credentials.

## Reproduction Assessment

Unclear for the exact live OFF table on current main without real Feishu credentials. The source-level path is clear: gateway health now resolves selected Feishu accounts, while the setup/status verification probe still reads top-level credentials before probing.

## Solution Assessment

Yes. Routing Feishu setup/status probing through the existing Feishu account resolver is the narrow plugin-owned fix and avoids adding any core Feishu special case.
