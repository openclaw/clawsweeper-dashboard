## Summary

Keep open. Current main still has the Feishu setup/status credential mismatch for account-scoped credentials, and open PR #74060 already targets this issue with closing syntax, so the issue should wait for that PR to land or be closed by maintainers.

## What This Changes

The issue asks for `openclaw status --deep` health output to treat Feishu as configured when credentials live under `channels.feishu.accounts.<id>` with `defaultAccount`.

## Best Possible Solution

Land PR #74060 or an equivalent narrow Feishu plugin change that resolves the selected Feishu account before setup/status probing, with regression coverage for account-scoped credentials.

## Reproduction Assessment

Yes. A config with `channels.feishu.defaultAccount` and credentials only under `channels.feishu.accounts.<id>.appId/appSecret` exercises the source-level mismatch between account-aware configuration detection and top-level-only setup/status probing.

## Solution Assessment

Yes. Routing the Feishu setup/status probe through the existing account resolver is the narrowest maintainable fix and keeps Feishu-specific behavior inside the Feishu plugin.
