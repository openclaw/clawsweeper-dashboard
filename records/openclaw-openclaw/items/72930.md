## Summary

Keep open. Current main is account-aware in the gateway health path, but the issue is still paired with open PR #74060 using closing syntax, and the Feishu setup/status verification surface still has a top-level credential probe gap that the PR is intended to fix.

## What This Changes

Feishu health/status should treat account-scoped `channels.feishu.accounts.*.appId/appSecret` credentials as configured and probeable instead of requiring top-level Feishu credentials.

## Best Possible Solution

Review and land PR #74060, or an equivalent plugin-owned fix, so Feishu setup/status verification resolves the selected Feishu account before probing and keeps regression coverage for account-scoped credentials.

## Reproduction Assessment

Unclear for the exact live OFF output without real Feishu credentials. The remaining source-level mismatch is deterministic: account-scoped credentials are recognized as configured, while the setup/status probe still reads top-level credentials before verification.

## Solution Assessment

Yes. Routing Feishu setup/status probing through the existing resolved-account path is the narrowest maintainable fix and avoids adding core-specific Feishu exceptions.
