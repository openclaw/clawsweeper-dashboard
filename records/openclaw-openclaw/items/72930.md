## Summary

Keep open. Current main has account-aware Feishu gateway/runtime health, but the Feishu setup/status fallback still probes top-level Feishu credentials only, and open PR #74060 is the unmerged closing implementation candidate.

## What This Changes

The issue asks Feishu health/status to recognize multi-account credentials under `channels.feishu.accounts.*.appId/appSecret` instead of reporting Feishu as OFF/not configured.

## Best Possible Solution

Land PR #74060 or an equivalent Feishu plugin change that resolves the selected account before setup/status probing and keeps regression coverage for account-scoped credentials.

## Reproduction Assessment

Yes. A config with `channels.feishu.defaultAccount` and credentials only under `channels.feishu.accounts.<id>` exercises the source-level mismatch: configured detection can see the account, while setup/status probing still starts from top-level credentials.

## Solution Assessment

Yes. Resolving the selected Feishu account inside the plugin-owned setup/status probe is the narrow maintainable fix and avoids adding a Feishu special case to core health.
