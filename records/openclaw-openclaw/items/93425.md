## Summary

Keep open: current main still lacks a supported `agents.list[].env` contract, while the closest implementation PR was closed unmerged because ACPX, MCP, docs, and security-contract coverage were incomplete.

## What This Changes

Request native per-agent environment variables from `openclaw.json` to be injected into agent-owned subprocesses such as exec, git, CLI backends, ACP/ACPX, and MCP servers.

## Best Possible Solution

Design one canonical sanitized agent-child environment contract across schema/resolver, exec, CLI, ACP/ACPX, MCP, docs/help/redaction, and regression proof before implementation.

## Reproduction Assessment

Yes. Current-main source shows no `agents.list[].env` field in the agent type/schema/resolver, while exec and CLI spawn paths compose env from host, request, plugin, backend, or prepared-runtime sources instead.

## Solution Assessment

Unclear. The requested capability is coherent, but the best fix needs maintainer decisions on security filtering, precedence, docs/redaction, MCP behavior, and ACP/ACPX support rather than a narrow schema-only patch.
