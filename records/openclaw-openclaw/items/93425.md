## Summary

Keep open: current `main` and the latest release still lack an `agents.list[].env` config contract and do not propagate agent-owned env through all requested subprocess boundaries. A closed OpenClaw PR attempted part of this, but a member explicitly closed it as incomplete and kept this issue open for a correctly scoped implementation.

## What This Changes

Request native per-agent environment injection from `openclaw.json` into agent-owned subprocesses such as exec, CLI runtimes, ACP, MCP, and git.

## Best Possible Solution

Define a maintainer-approved canonical sanitized agent-child environment contract, land the ACPX support, then implement schema/docs plus exec, CLI, MCP, and ACP propagation with end-to-end child-process proof.

## Reproduction Assessment

Yes, at source level. Current `main` has no `agents.list[].env` schema/type/resolver path, and the inspected exec, CLI, ACP, and MCP launch paths do not consume agent-owned env.

## Solution Assessment

No single implementation path is ready yet. The requested direction is plausible, but the best fix needs a maintainer-approved sanitized contract and upstream ACPX support before adding a public config surface.
