## Summary

Keep this open: current main and the latest release still lack a supported per-agent child-process environment contract, and the direct implementation PR was closed unmerged after maintainer review found incomplete subprocess and security coverage.

## What This Changes

The issue requests an optional `agents.list[].env` config surface whose values are injected into agent-owned subprocesses such as exec, CLI backends, MCP servers, and ACP sessions.

## Best Possible Solution

Design one canonical sanitized agent-child environment contract across config schema/resolution, exec, CLI, MCP, ACP/ACPX, docs, redaction, and regression proof before implementation.

## Reproduction Assessment

Yes, from source inspection: current main has no `agents.list[].env` field in `AgentConfig`, `AgentEntrySchema`, or `ResolvedAgentConfig`, and the exec/CLI/MCP launch paths do not merge an agent-level env map.

## Solution Assessment

Unclear as a final implementation path; the capability is useful, but the best fix needs maintainer approval for security, precedence, redaction, docs, and ACP/MCP subprocess contracts before code should land.
