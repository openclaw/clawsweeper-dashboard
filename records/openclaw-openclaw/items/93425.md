## Summary

Keep open: current main and the latest release still lack `agents.list[].env`, and current child-spawn paths only compose parent, request, backend, plugin, or sandbox env. The prior implementation attempt was closed by a maintainer as incomplete, with this issue kept open for a correctly scoped contract.

## What This Changes

Request support for an optional per-agent env map in `openclaw.json` that is inherited by spawned subprocesses.

## Best Possible Solution

Keep this issue open for a maintainer-designed, documented, sanitized agent-child environment contract spanning exec, CLI, MCP, and ACP, backed by ACPX support and real child-process proof.

## Reproduction Assessment

Yes, source-backed for the missing behavior: current main lacks `agents.list[].env`, and the inspected spawn paths do not read per-agent env. This is a feature absence rather than a broken existing contract.

## Solution Assessment

Unclear; raw per-agent env injection is plausible but not yet proven to be the best permanent shape. The safer direction is one canonical sanitized child-env contract across exec, CLI, MCP, and ACP with docs and real subprocess proof.
