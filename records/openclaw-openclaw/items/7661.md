## Summary

Keep open because current main still lacks the requested end-to-end channel creation path: Discord has partial provider/runtime support, but the CLI create surface is absent and Slack has no channel-create discovery, dispatch, scope, or gate support.

## What This Changes

The issue requests a native `openclaw message channel create` command or message-tool action for Slack, Discord, and other messaging providers.

## Best Possible Solution

Agree on a provider-owned channel-management authority model, then add Slack channel creation with explicit disabled-by-default gates, required scopes, SecretRef-compatible credentials, runtime dispatch, docs, and tests while landing or replacing the focused Discord CLI work separately.

Unclear until maintainers decide the authority model. The maintainable solution is provider-owned channel management with explicit gates, scope documentation, requester authorization, and focused tests rather than a generic core shortcut.
