## Summary

Keep open. Current main and the latest release still lack the requested end-to-end channel creation path: the shared action name and partial Discord backend exist, but the CLI exposes only channel info/list and Slack has no channel-create discovery, runtime handler, action gate, setup scope, or docs.

## What This Changes

The issue requests native message-tool and CLI support for creating Slack, Discord, and similar provider channels.

## Best Possible Solution

Agree the channel-management authority model first, then add provider-owned Slack channel creation with explicit scopes/gates/docs/runtime/tests while landing or superseding the narrower Discord CLI wiring separately.

## Reproduction Assessment

Yes, by source inspection. Current main exposes the shared channel-create action name and partial Discord backend support, but it lacks the requested Slack/provider end-to-end path and CLI command.

## Solution Assessment

Unclear until maintainers decide the authority model. The maintainable direction is provider-owned implementation with explicit action gates, required provider scopes, docs, and focused tests rather than a generic core shortcut.
