## Summary

Keep open. Current main and the latest release still lack the requested end-to-end channel creation path: generic action names and partial Discord backend support exist, but the CLI only exposes channel info/list and Slack lacks discovery, runtime, gate, scope, and docs support; this needs maintainer product/security review before implementation.

## What This Changes

Request to add native `message channel create` / `channel-create` support for Slack, Discord, and similar messaging providers.

## Best Possible Solution

Agree the channel-management authority model first, then add provider-owned Slack support with explicit scopes, gates, docs, runtime handling, and tests while landing or superseding the narrower Discord CLI work separately.

## Reproduction Assessment

Yes, by source inspection. Current main has the shared action name and partial Discord backend, but lacks the requested Slack/provider end-to-end path and the CLI create command.

## Solution Assessment

Unclear until maintainers decide the authority model. The maintainable direction is provider-owned implementation with explicit action gates, required provider scopes, docs, and focused tests rather than a generic core shortcut.
