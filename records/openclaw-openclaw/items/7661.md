## Summary

Keep open. Current main still lacks the requested end-to-end channel creation path: the shared action name and Discord backend support exist, but the CLI still exposes only channel info/list and Slack lacks channel-create discovery, runtime dispatch, action gates, scope docs, and tests.

## What This Changes

The issue requests native OpenClaw message-tool support for creating channels, especially Slack `conversations.create` and comparable Discord channel creation flows.

## Best Possible Solution

Agree an explicit provider-owned channel-management authority model, then add Slack channel creation with required scopes, default-off gates, SecretRef-compatible credentials, runtime dispatch, docs, and tests while landing or superseding the focused Discord CLI work separately.

## Reproduction Assessment

Yes, by source inspection. Current main has the shared action name and partial Discord backend support, but lacks the requested Slack end-to-end path and the CLI channel create command.

## Solution Assessment

Unclear until maintainers decide the authority model. The maintainable path is provider-owned implementation with explicit gates, provider scopes, requester authorization behavior, docs, and focused tests rather than a generic core shortcut.
