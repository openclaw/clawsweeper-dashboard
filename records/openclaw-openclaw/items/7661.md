Keep open. Current main still lacks the requested end-to-end channel creation path: Discord has provider-side `channel-create` support but no merged CLI command, and Slack still does not advertise or execute channel creation; adding it also needs maintainer product/security review because it grants agents workspace/server administration authority.

## What This Changes

Request to add native `message channel create` / `channel-create` support for Slack, Discord, and similar messaging providers.

## Best Possible Solution

Add provider-owned channel creation after maintainer product/security review: Slack should get an explicit action gate, `channels:manage` setup/docs, discovery/runtime support backed by Slack `conversations.create`, and targeted tests, while the narrower Discord CLI gap can land through the existing focused PR or an equivalent replacement.
