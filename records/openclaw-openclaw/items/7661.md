## Summary

Keep open. Current main still lacks the requested end-to-end channel creation surface: Slack has no `channel-create` discovery, dispatch, action gate, or scope docs, and the CLI still exposes only `message channel info|list`; Discord has partial backend support and an open PR only covers the Discord CLI slice. This is high-privilege workspace/server mutation, so maintainer product/security review is needed before implementation.

## What This Changes

Request to add native `message channel create` / `channel-create` support for Slack, Discord, and similar message providers.

## Best Possible Solution

Approve a provider-owned channel-management model, then add Slack channel creation behind explicit gates/scopes/docs/tests while landing or replacing the focused Discord CLI wiring separately.

## Reproduction Assessment

Not applicable as a runtime bug reproduction because this is a feature request. The feature gap is high-confidence from source and docs: shared action naming exists, Discord is partially implemented, and Slack lacks discovery, dispatch, config, and docs for channel creation.

## Solution Assessment

Unclear until maintainer product/security review. A provider-owned, explicitly gated implementation is the maintainable direction; an unconditional channel-create command or tool would grant broad workspace/server mutation authority.
