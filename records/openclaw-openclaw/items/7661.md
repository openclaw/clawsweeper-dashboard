## What This Changes

Add explicitly gated Slack and Discord channel creation through the message tool and CLI.

## Best Possible Solution

Approve an explicit administrative-action policy, then implement provider-owned channel creation with secure opt-in gates, account-aware overrides, least-privilege Slack scopes, aligned discovery/runtime/CLI/docs, and focused provider tests.

## Reproduction Assessment

Yes. Current-main source provides a high-confidence verification path: the CLI registers only channel information/listing, Slack does not advertise or implement creation, and Discord demonstrates the existing gated backend pattern.

## Solution Assessment

Yes at the architectural level, but implementation should wait for the authority decision. Provider-owned actions behind explicit account-aware gates are the narrowest maintainable solution, rather than a generic ungated core operation.
