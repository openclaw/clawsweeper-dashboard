## What This Changes

The PR adds an internal `nativeHarnessCompaction` signal so queued canonical OpenAI sessions pinned to the Codex runtime invoke native Codex compaction while preserving custom OpenAI-compatible base URL behavior.

## Best Possible Solution

Queued manual and budget compaction for canonical OpenAI+Codex sessions should use the native Codex compaction path without direct API keys, while custom OpenAI-compatible base URLs remain on the direct OpenClaw compaction path.

## Reproduction Assessment

## Solution Assessment

Yes. An explicit internal `nativeHarnessCompaction` fact is the narrowest maintainable fix because it separates the native Codex compaction decision from provider rerouting display fields, and the PR keeps custom OpenAI-compatible base URLs on the existing path.
