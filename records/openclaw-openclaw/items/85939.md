## What This Changes

The issue requests faster Control UI full-page reloads by caching selected API/RPC responses, persisting lightweight view state, and avoiding unbounded session-list reloads.

## Best Possible Solution

Keep this as a security-aware Control UI reload design item: preserve the current no-cache API boundary until scoped caching is approved, persist only harmless per-view UI state, and keep the shipped bounded sessions-list behavior.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a full document reload rebuilds Lit state, the service worker skips /api and /rpc responses, and active-tab refresh paths call gateway loaders again.

## Solution Assessment

Unclear as requested. Persisting harmless UI state is likely maintainable, but stale-while-revalidate caching for authenticated Control UI API data needs a maintainer-approved security and invalidation policy first.
