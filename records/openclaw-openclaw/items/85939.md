## Summary

Keep open: current main and v2026.6.9 include the related navigation-auth fix and bounded sessions-list behavior, but the central reload request remains because API/RPC/plugin responses are still intentionally uncached and several sessions-view controls still reset to component defaults on full browser reload.

## What This Changes

The issue requests faster Control UI full-page reloads by caching selected API/RPC responses, preserving lightweight UI state across F5 reloads, and avoiding unbounded session-list reloads.

## Best Possible Solution

Keep this as a security-aware Control UI reload design item: preserve the no-cache authenticated API boundary until scoped caching is approved, persist only harmless reload UI state, and keep the shipped bounded sessions-list behavior.

## Reproduction Assessment

Yes for source reproduction: a full document reload rebuilds <openclaw-app>, the service worker skips /api, /rpc, and /plugins responses, and active-tab refresh paths call Gateway loaders again. I did not run a live large-session-store timing reproduction.

## Solution Assessment

Unclear as requested. Persisting harmless UI state is likely maintainable, but stale-while-revalidate caching for authenticated Control UI API/RPC data needs a maintainer-approved security and invalidation policy first.
