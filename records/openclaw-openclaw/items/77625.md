## Summary

Keep open: current main mitigates the default MiniMax M2.x path by defaulting thinking off, but it still preserves explicit MiniMax M2.x thinking and does not strip non-Claude Anthropic-path reasoning from replay, which matches the reporter's later `thinking level: high` context.

## What This Changes

The issue requests fixing a `reasoningDefault=stream` feedback loop where streamed MiniMax reasoning can be replayed into agent context and cause repetitive recursion.

## Best Possible Solution

Make a maintainer decision on whether explicit MiniMax-M2.x thinking should remain supported; if not, force-disable it or strip its reasoning from replay with live MiniMax proof and focused regression coverage.

## Reproduction Assessment

Yes at source level, but not as a live provider reproduction here: current main preserves explicit MiniMax-M2.7 thinking payloads and does not drop non-Claude Anthropic-path reasoning history. A live MiniMax credentialed run would still be needed to prove the exact infinite loop end to end.

## Solution Assessment

Unclear: the narrowest maintainable fix depends on whether maintainers want to preserve explicit MiniMax-M2.x thinking despite the reported loop. The safer product direction is a provider-owned policy decision plus replay/payload tests and live MiniMax proof.
