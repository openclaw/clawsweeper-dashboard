## What This Changes

The issue asks OpenClaw to prevent `reasoningDefault: "stream"` with MiniMax-M2.7 and high thinking from streaming repetitive reasoning that appears to be replayed into later context.

## Best Possible Solution

Harden the MiniMax Anthropic streaming/replay boundary so streamed or unsigned reasoning cannot become future assistant-visible context, backed by a MiniMax regression and live proof when credentials are available.

## Reproduction Assessment

No high-confidence live reproduction is available. Source and pinned dependency inspection show a credible reproduction path where non-off thinking enables MiniMax Anthropic `thinking`, while MiniMax-M2.7 replay policy does not force-drop thinking blocks.

## Solution Assessment

Unclear because there is no proposed patch yet. The narrowest maintainable direction is a MiniMax-owned or shared hybrid replay/stream fix, not a global disable of reasoning stream mode.
