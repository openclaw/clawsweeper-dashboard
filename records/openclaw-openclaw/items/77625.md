## Summary

Keep open: current main and the latest release still preserve explicit MiniMax thinking on the Anthropic-compatible stream path, the Anthropic replay policy still lacks reasoning-history dropping for non-Claude MiniMax, and the only linked fix PR was closed unmerged.

## What This Changes

The issue reports that `reasoningDefault: "stream"` with MiniMax-M2.7 can replay streamed reasoning into later context and create repetitive reasoning loops.

## Best Possible Solution

Capture a redacted current-main MiniMax-M2.7 stream-mode trajectory, then align MiniMax stream/replay policy, tests, docs, and release notes so non-native `reasoning_content` cannot become visible or replayed.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection does reproduce the plausible MiniMax leak/replay path: explicit thinking can bypass the stream guard, and the MiniMax Anthropic replay policy does not drop reasoning history.

## Solution Assessment

Unclear. The closed PR's runtime direction is plausible, but the final fix should follow live MiniMax proof and a decision on suppressing explicit MiniMax thinking versus normalizing its non-native reasoning stream output.
