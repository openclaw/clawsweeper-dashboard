## Summary

Keep open: current main mitigates the default MiniMax M2.x path by defaulting thinking off, but it still preserves explicit MiniMax-M2.7 thinking and the MiniMax Anthropic-compatible replay policy still does not drop reasoning history. The exact linked repair PR is closed unmerged, and the related loop issue is closed rather than a viable canonical replacement.

## What This Changes

The issue reports that `reasoningDefault: "stream"` with MiniMax-M2.7 can feed streamed reasoning back into later context and trap the agent in repetitive reasoning.

## Best Possible Solution

Keep this open until maintainers choose the MiniMax M2.x thinking/replay policy, then validate the chosen behavior with focused payload/replay tests and live MiniMax proof.

## Reproduction Assessment

Yes at source level: current main still preserves explicit MiniMax-M2.7 thinking and lacks reasoning-history dropping on the MiniMax Anthropic-compatible replay policy. No live MiniMax provider run was done in this read-only review.

## Solution Assessment

Unclear: the best fix depends on maintainer policy for explicit MiniMax-M2.x thinking. Force-disabling thinking and preserving it with replay stripping are both plausible but have different provider-behavior tradeoffs.
