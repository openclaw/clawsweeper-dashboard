## Summary

Keep open: current main and the latest release mitigate the default MiniMax M2.x path by defaulting thinking off, but the reported high-thinking MiniMax-M2.7 path still has source-level risk because explicit thinking is preserved and the Anthropic-compatible MiniMax replay policy does not drop reasoning history. ## What This Changes

The issue reports that `reasoningDefault: "stream"` with MiniMax-M2.7 can feed streamed reasoning back into later context and cause an unbounded reasoning loop.

## Best Possible Solution

Keep this open until maintainers choose the MiniMax M2.x explicit-thinking policy, then land focused payload/replay tests, docs, and live MiniMax proof for the chosen behavior.

## Reproduction Assessment

No for live current-main reproduction; yes for a source-level path. The reporter supplied MiniMax-M2.7 high-thinking output, and current source still preserves explicit M2.x thinking while the Anthropic MiniMax replay policy does not drop reasoning history.

## Solution Assessment

Unclear: current main's default-off mitigation is not a complete fix for the reported high-thinking path. The maintainable solution depends on whether maintainers want to force-disable MiniMax M2.x thinking or support it with replay stripping and live provider validation.
