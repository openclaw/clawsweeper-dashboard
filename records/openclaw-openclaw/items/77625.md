## Summary

Keep open: current main mitigates the default MiniMax M2.x path by defaulting thinking off, but it still preserves explicit MiniMax M2.x thinking and does not strip non-Claude Anthropic-family reasoning from replay, matching the reporter's later `thinking level: high` context. The open reasoning-leak work at https://github.com/openclaw/openclaw/issues/89473 overlaps but does not fully replace this feedback-loop/replay issue.

## What This Changes

The issue reports that `reasoningDefault: "stream"` plus MiniMax reasoning can feed streamed reasoning back into later context and create an infinite reasoning loop.

## Best Possible Solution

Keep this open until maintainers choose the explicit MiniMax M2.x thinking contract and land either safe explicit-thinking rejection/disablement or replay filtering that prevents streamed reasoning from re-entering future model context.

## Reproduction Assessment

No, not as a high-confidence current-main live reproduction. Source inspection shows the default MiniMax M2.x mitigation does not cover explicit thinking, which matches the reporter's `thinking level: high` update, but a live MiniMax-M2.7 run was not performed.

## Solution Assessment

Unclear. The closed PR's unconditional-disable approach is plausible, but current docs and tests preserve explicit MiniMax thinking, so the best fix needs a maintainer decision between changing that contract and adding replay filtering.
