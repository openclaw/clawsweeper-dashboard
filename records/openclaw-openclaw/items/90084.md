## Summary

Keep open for maintainer review: this PR changes the released daemon service runtime policy from supported system Node preference to active version-manager Node preference, while current docs, audit code, and tests still treat version-manager service runtimes as upgrade-fragile.

## What This Changes

The branch changes daemon Node resolution and tests so a supported active version-manager Node path wins over a supported system Node for generated service commands.

## Best Possible Solution

Keep the supported-system-Node default unless maintainers approve an explicit validated runtime-path opt-in; if the default changes, update docs/audit/tests and provide live Linux systemd upgrade proof.

## Reproduction Assessment

## Solution Assessment

No: unconditional active-node preference is not the best fix because it conflicts with current service audit, docs, tests, and released behavior. The safer maintainable path is explicit validated runtime pinning or a clear maintainer-approved default-policy change with upgrade proof.
