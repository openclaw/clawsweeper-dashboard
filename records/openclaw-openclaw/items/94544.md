## Summary

Keep open: this is a useful narrow Control UI fix, but it is not merge-ready because the shared display helper can now render a raw session key from `derivedTitle`, and the PR still lacks exact-head real Control UI proof or a maintainer proof override.

## What This Changes

The PR makes Control UI session lists and the chat session picker request `includeDerivedTitles` and render `derivedTitle` when explicit session labels are absent.

## Best Possible Solution

Land the narrow UI wiring only after `derivedTitle` preserves the existing key-equality guard and exact-head sidebar plus picker proof is supplied, or intentionally choose https://github.com/openclaw/openclaw/pull/96267 as the broader canonical path.

## Reproduction Assessment

Yes, source-level. Current main exposes the gateway option while Control UI does not consume it, and the PR defect is traceable because `deriveSessionTitle()` can propagate a raw-key displayName into `derivedTitle`.

## Solution Assessment

No as submitted. Reusing existing gateway `includeDerivedTitles` support is the right layer, but the shared display helper should ignore key-equal `derivedTitle` values just like `label` and `displayName`.
