## Summary

Keep open: the PR is a useful targeted bug fix, but it is not ready as written. The new prefix-only branch covers the submitted truncated-snapshot case, but it still misses the related single-entry phase-tagged stream shape already documented in https://github.com/openclaw/openclaw/issues/80458, and the supplied proof is only unit-test output rather than live Discord/gateway behavior.

## What This Changes

The PR adds a single-snapshot prefix heuristic in `buildEmbeddedRunPayloads` plus regression tests so a truncated accumulated assistant text can be replaced with the canonical `final_answer`.

## Best Possible Solution

Prefer the canonical phase-filtered `final_answer` for phase-tagged assistant messages in `buildEmbeddedRunPayloads`, preserve existing media/directive behavior, and cover both truncated-prefix and commentary-plus-final single-snapshot cases.

## Reproduction Assessment

Yes for helper-level source reproduction: current main returns a single non-empty accumulated `assistantTexts` entry before using the phase-filtered `final_answer`. I did not run a live Discord reproduction in this read-only review.

## Solution Assessment

No, not as currently written. The prefix heuristic fixes the submitted truncated-prefix case, but the narrower maintainable fix should key off explicit phase metadata/final-answer availability so the known non-prefix single-stream shape is covered too.
