## Summary

Keep open. Current main still drops PDF page images when the optional canvas module is unavailable, and this PR is the active linked implementation for that behavior; the latest branch is directionally correct but still needs the required changelog/docs/test follow-through before merge.

## What This Changes

The PR adds a Poppler `pdftoppm` fallback path for PDF page image rendering when `@napi-rs/canvas` cannot load.

## Best Possible Solution

Land the plugin-local Poppler fallback after adding a single-line changelog entry, a docs note for the optional system renderer, and focused regression tests around fallback rendering and cleanup.

## Solution Assessment

Yes, directionally. Keeping the fallback inside `extensions/document-extract` is the narrow owner-module fix; the remaining work is release/docs/test follow-through, not a different product direction.
