## What This Changes

The PR adds a `pdftoppm`-based PDF page image rendering fallback in the bundled `document-extract` plugin when `@napi-rs/canvas` cannot be loaded.

## Best Possible Solution

Land a bounded, documented, and tested Poppler fallback in `document-extract` that preserves the existing trimmed-text threshold and records the user-facing PDF fix in the changelog.

## Reproduction Assessment

Yes. Current main has a high-confidence source-level reproduction: low-text PDFs attempt canvas rendering, a failed `@napi-rs/canvas` import is caught, and the extractor returns text with `images: []`; #75358 adds concrete Ubuntu 24.04 setup evidence.

## Solution Assessment

No, not as-is. The fallback belongs in the bundled plugin, but this implementation should preserve the current threshold semantics and add changelog and regression coverage before merge.
