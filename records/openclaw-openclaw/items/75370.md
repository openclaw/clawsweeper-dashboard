## Summary

Keep open. Current main still lacks the requested Poppler fallback, so the PR is not obsolete, but the latest branch is not merge-ready because it drops low-volume extracted text, can bypass the render pixel budget, and introduces synchronous shell execution of an external PDF renderer without the required docs, changelog, or focused tests.

## What This Changes

The branch modifies the bundled `document-extract` PDF extractor to add a `pdftoppm` fallback when `@napi-rs/canvas` cannot load, while refactoring text and image extraction into helper functions.

## Best Possible Solution

Keep the fix in the `document-extract` plugin only if maintainers accept Poppler as an optional renderer, then preserve partial extracted text, enforce pixel/output/time bounds with an argv-based async process API, document the system dependency, and add focused regression coverage plus the required changelog entry.

## Reproduction Assessment

Yes. Current main has a high-confidence static reproduction: a low-text PDF reaches image rendering, `loadCanvasModule()` throws when `@napi-rs/canvas` is absent, and the extractor returns no images; #75358 supplies a concrete Ubuntu setup with `pdftoppm` present.

## Solution Assessment

No. The fallback direction is plausible, but this implementation is not the best fix until it preserves existing text behavior, avoids blocking shell execution, respects the pixel budget, and updates the user-facing contract and tests.

  - confidence: 0.91

  - body: The fallback only works when `pdftoppm` is installed and on `PATH`, but the PDF docs still describe only the canvas-backed image-rendering path. Add the install and behavior notes with this user-facing change.

## Security Review

Status: needs_attention

Summary: The diff introduces external rendering of untrusted PDFs and still needs safer process execution plus hard resource-bound enforcement before merge.

Concerns:

- **[medium] Synchronous shell execution of PDF renderer:** `extensions/document-extract/document-extractor.ts:187`
  - body: The PR invokes `pdftoppm` with `execSync()` and a command string. Even with a timeout, hostile or expensive PDFs can block the Gateway event loop per page, and argv-based execution would avoid shell parsing of generated temp paths.
  - confidence: 0.91
- **[medium] Pixel budget bypass amplifies render work:** `extensions/document-extract/document-extractor.ts:175`
  - body: The Poppler fallback preserves a 72-DPI floor after the render planner may have selected a smaller scale, allowing oversized PNG renders and larger memory or prompt payloads for attacker-supplied PDFs.
  - confidence: 0.92
