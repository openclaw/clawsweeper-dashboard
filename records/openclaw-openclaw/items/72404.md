## Summary

Keep open: this is a plausible fix for the linked Bedrock vision bug and the author addressed the initial Greptile regex concerns, but the current head still regresses explicit input overrides, can rewrite unchanged provider config, and lacks inspectable real Bedrock proof.

## What This Changes

The branch adds core merge-time defaults that infer `input: ["text", "image"]` for explicit-only models matching Claude/OpenAI/Gemini/Llama vision ID patterns, plus regression tests for `mergeProviderModels` and `mergeProviders`.

## Best Possible Solution

Land a revised stale-catalog repair that restores missing image metadata for Bedrock Claude rows, preserves explicit `input` overrides, keeps models.json planning stable, uses conservative provider-backed capability inference, and includes redacted real Bedrock proof.

## Reproduction Assessment

Yes, at source level: current main preserves an explicit Bedrock Claude row without `input` when discovery has no matching model, and `modelSupportsImages()` treats undefined input as text-only. I did not run a live AWS Bedrock reproduction.

## Solution Assessment

No, not as currently written. The merge-helper surface is plausible, but the patch must preserve explicit input overrides, avoid unnecessary config rewrites, and keep capability defaults conservative and provider-backed.
