Full review comments:

- **[P1] Guard inferred image input against custom endpoints:** `src/agents/models-config.merge.ts:164-167`
  - body: `builtInProvider` is derived only from the provider key. A user can configure a built-in provider key with a custom `baseUrl` or proxy endpoint, and this branch would still infer `input: ["text", "image"]` and forward images to an endpoint that never advertised image support. Make the default depend on provider-owned endpoint evidence, or skip it when provider/model transport is user-controlled.
  - confidence: 0.91
- **[P1] Thread provider identity into dynamic catalog merges:** `src/agents/models-config.merge.ts:152`
  - body: The new provider-key gate defaults to false when callers omit it, but the dynamic provider-model merge path still calls `mergeProviderModels(implicit, existing)` without the provider id. That leaves an intended stale-catalog merge path unable to apply the built-in Bedrock default.
  - confidence: 0.9
- **[P2] Align explicit-input default tests with provider scoping:** `src/agents/models-config.merge.explicit-input-default.test.ts:33`
  - body: These new tests call `mergeProviderModels` without a `providerKey` while expecting image defaults. Under this branch's own code, missing `providerKey` means `builtInProvider` is false, so the assertions no longer describe the intended contract and should fail.
  - confidence: 0.93
