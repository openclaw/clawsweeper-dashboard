## Summary

Keep open. The PR targets a real stale-catalog image capability bug, but the current head is not merge-ready because it can overwrite explicit `input` overrides and can infer image support for Venice `openai-gpt-52`, which current metadata marks text-only.

## What This Changes

The PR adds merge-time `input: ["text", "image"]` defaults for explicit-only vision-looking model IDs plus regression tests for Claude, OpenAI, Gemini, Llama, and new-provider merge paths.

## Best Possible Solution

Keep the merge-time stale-catalog repair, but preserve explicit `input` whenever configured, narrow vision inference to current provider metadata, add negative regression coverage, and add the required changelog entry.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main leaves explicit-only Bedrock Claude rows without `input`, and the image gates treat missing `input` as text-only; no live run was performed because this review was read-only.

## Solution Assessment

No, not as currently written. The merge helper is the right narrow seam, but the patch must keep explicit capability overrides and make the vision-family inference conservative enough to match current provider metadata.
