The PR adds merge-time `input: ["text", "image"]` defaults for explicit-only model entries whose IDs look vision-capable, plus focused merge-helper regression tests.

## Best Possible Solution

Land a revised merge-helper repair that restores missing image metadata for stale explicit Bedrock Claude rows, preserves configured `input` overrides, uses conservative/provider-backed capability inference, and includes redacted real Bedrock proof.

## Reproduction Assessment

Yes, at source level. Current main preserves explicit-only Bedrock rows without `input`, and `modelSupportsImages()` treats missing `input` as text-only; I did not run live AWS Bedrock in this read-only review.

## Solution Assessment

No, not as currently written. The merge helper is the right narrow surface, but the PR must preserve explicit input overrides and avoid inferring image support from broad `gpt-5` substrings such as Venice `openai-gpt-52`.
