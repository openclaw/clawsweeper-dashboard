Full review comments:

- **[P1] Guard inferred image input against custom endpoints:** `src/agents/models-config.merge.ts:164-167`
  - body: `builtInProvider` is derived only from the provider key. Users can configure `models.providers.anthropic.baseUrl`, `models.providers.openai.baseUrl`, or a model-level endpoint to route a known model id through a proxy/custom endpoint; this branch would still infer `input: ["text", "image"]` and forward images to an endpoint that never advertised image support. Make the default depend on provider-owned endpoint evidence, or skip it when provider/model transport is user-controlled.
  - confidence: 0.91
- **[P1] Thread provider identity into dynamic catalog merges:** `src/agents/models-config.merge.ts:152`
  - body: The new `providerKey` gate defaults to false when callers omit it, but `mergeImplicitProviderConfig` still calls `mergeProviderModels(implicit, existing)` without the provider id for dynamic provider models. That leaves one intended stale-catalog merge path unable to apply the built-in Bedrock default.
  - confidence: 0.9
- **[P2] Align direct default tests with provider scoping:** `src/agents/models-config.merge.explicit-input-default.test.ts:33`
  - body: These new tests call `mergeProviderModels` without a `providerKey` while expecting image defaults, and the later `custom-anthropic` case expects a custom provider to default too. Under this branch's own code, missing or custom provider keys suppress the default, so the assertions no longer describe the intended contract.
  - confidence: 0.93
