## Summary

Keep open: the Bedrock stale-catalog bug is still valid, but this PR has blocking capability/serialization regressions and lacks inspectable after-fix Bedrock proof.

## What This Changes

The PR adds merge-time input defaults for explicit-only vision-looking model IDs and regression tests covering Claude, OpenAI, Gemini, Llama, and brand-new providers.

## Best Possible Solution

Land a revised stale-catalog repair that restores missing Bedrock Claude image metadata without overriding explicit input choices, without no-op config rewrites, and with redacted real Bedrock proof.

## Reproduction Assessment

Yes. At source level, current main preserves an explicit Bedrock Claude row without input when discovery has no matching model, and modelSupportsImages() then treats undefined input as text-only; I did not run live AWS Bedrock.

## Solution Assessment

No. The merge helper is a plausible surface, but this patch overwrites explicit input overrides, can churn no-op models.json writes, and should keep capability defaults more provider-backed or narrowly scoped.

## Review Findings

Overall correctness: patch is incorrect

- **[P2] Preserve explicit input overrides:** `src/agents/models-config.merge.ts:158`
  - body: This same-ID merge branch writes implicitModel.input unconditionally. Current main intentionally preserves explicit input when present, so a configured text-only opt-out or explicit image override can be lost when discovery has the same model ID, changing attachment forwarding against user/provider metadata.
