## Summary

Keep open. The linked Bedrock stale-catalog bug is real, but the current patch still broad-matches `gpt-5` and would infer image support for `openai-gpt-52`, which current Venice metadata and docs mark text-only.

## What This Changes

The PR adds merge-time `input: ["text", "image"]` defaults for explicit-only model entries matching selected vision-capable ID patterns, with new regression tests for Claude, GPT/OpenAI, Gemini, Llama, and brand-new provider merge paths.

## Best Possible Solution

Keep the merge-time stale-catalog repair, but replace substring-based GPT-5 inference with exact or provider-aware vision matching that preserves known text-only metadata and adds a negative regression for `openai-gpt-52`.

## Reproduction Assessment

Yes. On current main, an explicit-only Bedrock Claude model without `input` remains without `input` and is treated as text-only; on the PR head, the added GPT regex matches `openai-gpt-52`.

## Solution Assessment

No, not as currently written. The merge point is a reasonable narrow place to repair stale explicit catalogs, but capability inference should not override known provider metadata through an unqualified substring match.
