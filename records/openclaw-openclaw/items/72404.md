## What This Changes

The PR adds merge-time `input: ["text", "image"]` defaults for explicit-only vision-capable model IDs and regression tests for those merge paths.

## Best Possible Solution

Keep the merge-time stale-catalog repair, but make capability inference exact or provider-aware, add negative coverage for `openai-gpt-52`, and include the required changelog entry.

Yes. Current main preserves explicit-only models without `input` and the image gate treats that as text-only; the PR head regex can also be reproduced by evaluating it against `openai-gpt-52`.
