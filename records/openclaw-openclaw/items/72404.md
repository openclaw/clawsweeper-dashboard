## Summary

Keep open: the underlying Bedrock stale-catalog bug is source-reproducible, but this PR still has blocking correctness/security issues and lacks inspectable real Bedrock proof.

## What This Changes

The PR adds merge-time defaults that infer `input: ["text", "image"]` for explicit-only vision-looking model IDs, plus regression tests for `mergeProviderModels` and `mergeProviders`.

## Best Possible Solution

Land a revised stale-catalog repair that restores missing Bedrock Claude image metadata without overriding explicit input choices, without no-op config rewrites, and with redacted real Bedrock proof.

## Reproduction Assessment

Yes, at source level: current main preserves an explicit Bedrock Claude row without `input` when discovery has no matching model, then `modelSupportsImages()` treats undefined input as text-only. I did not run a live AWS Bedrock reproduction.

## Solution Assessment

No, not as currently written. The merge-helper surface is plausible, but the patch must preserve explicit input overrides, avoid unnecessary config rewrites, and keep capability defaults conservative or provider-backed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Preserve explicit input overrides:** `src/agents/models-config.merge.ts:158`
  - body: This same-ID merge branch now always writes `implicitModel.input`, so it overwrites configured explicit `input` values. Current main deliberately preserves explicit input overrides; losing that can ignore a text-only opt-out or remove an explicit image-capable override, changing attachment forwarding against user/provider metadata.
  - confidence: 0.95
- **[P2] Preserve no-op provider serialization:** `src/agents/models-config.merge.ts:202-204`
  - body: Routing providers with no implicit entry through a synthetic `{ models: [] }` object inserts `models` before the provider's existing keys even when the model list is unchanged. Because `planOpenClawModelsJson` compares JSON bytes, unchanged providers like the covered `kilocode` case become writes instead of no-ops and churn `models.json`.
  - confidence: 0.9
