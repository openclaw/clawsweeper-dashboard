## What This Changes

Request to add a persistent `gateway.nodeMaxOldSpaceMb`/generated `NODE_OPTIONS` heap setting for macOS LaunchAgent installs so service regeneration preserves the V8 heap limit.

## Best Possible Solution

Keep #42853 as the canonical implementation path and ensure it covers typed config, constrained heap-only `NODE_OPTIONS` generation, macOS launchd env-file regeneration, systemd/schtasks, and node-service inheritance.

## Reproduction Assessment

No live OOM reproduction was established in this read-only pass, but the missing supported persistence path is source-reproducible because current main lacks the typed config key and generated service `NODE_OPTIONS` path while still blocking arbitrary `NODE_OPTIONS`.

## Solution Assessment

Is this the best way to solve the issue? Yes, a typed numeric heap config plus constrained generated `NODE_OPTIONS` is safer than allowing arbitrary `NODE_OPTIONS` passthrough; the remaining implementation should be reviewed on #42853.
