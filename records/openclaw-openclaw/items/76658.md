## Summary

Keep open. Current main still lacks the typed gateway heap configuration and generated service `NODE_OPTIONS` path, while the linked #42853 remains the canonical open implementation candidate and this issue adds useful macOS LaunchAgent regeneration evidence.

## What This Changes

The issue requests a durable `openclaw.json` gateway heap setting that regenerates macOS LaunchAgent `NODE_OPTIONS` instead of relying on fragile manual plist edits.

## Best Possible Solution

Review and land or replace #42853 with a typed numeric heap config, generated heap-only `NODE_OPTIONS` for service installs, and regression coverage for launchd regeneration, systemd, Windows tasks, and node-host service inheritance.

## Reproduction Assessment

No high-confidence live OOM reproduction was established. The supported-persistence gap is source-reproducible because current main lacks `gateway.nodeMaxOldSpaceMb`, does not synthesize service `NODE_OPTIONS`, and still blocks arbitrary durable `NODE_OPTIONS`.

## Solution Assessment

Yes, the typed heap-size config plus generated heap-only `NODE_OPTIONS` is safer than permitting arbitrary `NODE_OPTIONS` passthrough. The fix should be completed through #42853 or a narrow replacement that covers the macOS launchd regeneration path.
