## Summary

Keep open. Current main still lacks a typed gateway heap config and does not synthesize NODE_OPTIONS for installed gateway services, while the linked #42853 remains an open implementation candidate with macOS-specific validation still useful.

## What This Changes

Request to add durable gateway V8 heap sizing via openclaw.json so regenerated macOS LaunchAgent installs keep a heap-derived NODE_OPTIONS value.

## Best Possible Solution

Review and land or replace #42853 with a typed numeric heap config, generated heap-only NODE_OPTIONS for service installs, and regression coverage for launchd, systemd, Windows tasks, and node-host service inheritance.

## Reproduction Assessment

No high-confidence live OOM reproduction was established. The missing supported persistence path is source-reproducible on current main because the gateway config/schema lack nodeMaxOldSpaceMb, service env generation omits NODE_OPTIONS, and arbitrary NODE_OPTIONS is still blocked from durable config env.

## Solution Assessment

Yes, the requested typed heap-size config plus auto-detected default is safer than allowing arbitrary NODE_OPTIONS passthrough. The implementation should be checked against macOS launchd regeneration and node-host service env threading before merge.
