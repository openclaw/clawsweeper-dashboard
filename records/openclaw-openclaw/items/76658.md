## Summary

Keep open: current main still lacks a supported service-level heap configuration path, and open PR #42853 remains the implementation candidate rather than evidence that the request is done.

## What This Changes

Request a durable `openclaw.json` gateway heap setting that generates `NODE_OPTIONS` for the macOS LaunchAgent across service regeneration.

## Best Possible Solution

Land or replace PR #42853 with a typed numeric gateway heap config, safe generated `NODE_OPTIONS`, and cross-supervisor coverage for launchd, systemd, Windows tasks, and node-host service inheritance.

## Reproduction Assessment

Yes for the requested config gap: current main has no `gateway.nodeMaxOldSpaceMb` type/schema and the service install path never generates `NODE_OPTIONS`. No live 36-hour OOM reproduction was run.

## Solution Assessment

Yes, a typed heap-size config plus generated service-owned `NODE_OPTIONS` is safer than allowing arbitrary `NODE_OPTIONS` passthrough. The implementation should validate launchd regeneration and child/node-host inheritance before merge.
