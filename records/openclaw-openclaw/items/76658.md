## Summary

Keep open. Current main still lacks a typed gateway heap config and does not synthesize NODE_OPTIONS for installed gateway services, while open PR #42853 is the canonical implementation candidate and the macOS LaunchAgent evidence here remains useful for that review.

## What This Changes

The issue requests persistent gateway Node/V8 heap configuration through openclaw.json so regenerated macOS LaunchAgents keep the intended NODE_OPTIONS heap limit.

## Best Possible Solution

Review and land or replace PR #42853 with a typed numeric heap config, generated heap-only NODE_OPTIONS for managed gateway and node services, and regression coverage for launchd regeneration, systemd, Windows tasks, and node-host inheritance.

## Reproduction Assessment

No high-confidence live reproduction of the 36-hour OOM was established. The missing supported persistence path is source-reproducible because current main lacks nodeMaxOldSpaceMb/NODE_OPTIONS service generation while durable config env still blocks arbitrary NODE_OPTIONS.

## Solution Assessment

Yes, a typed heap-size config plus generated heap-only NODE_OPTIONS is safer than allowing arbitrary durable NODE_OPTIONS passthrough. The open PR should still be reviewed against macOS launchd regeneration and node-host service inheritance before merge.
