## Summary

Keep open. Current main still lacks a typed gateway heap config and does not generate a heap-only `NODE_OPTIONS` value for installed gateway services; open PR #42853 remains the canonical implementation candidate, and this macOS LaunchAgent report adds useful validation coverage.

## What This Changes

Requests durable `openclaw.json` configuration for the gateway V8 heap so macOS LaunchAgent regeneration preserves a generated max-old-space setting.

## Best Possible Solution

Land or replace #42853 with a typed numeric heap config, generated heap-only `NODE_OPTIONS`, and regression coverage across launchd, systemd, Windows tasks, and node-host services.

## Reproduction Assessment

No high-confidence live OOM reproduction was established. The missing supported persistence path is source-reproducible because current main lacks the requested config/schema field and generated service `NODE_OPTIONS`, while arbitrary `NODE_OPTIONS` remains intentionally blocked.

## Solution Assessment

Yes, the requested typed heap-size config plus generated heap-only service environment is safer than allowing arbitrary `NODE_OPTIONS` passthrough. The open PR should be reviewed against the macOS LaunchAgent regeneration and node-host inheritance paths before merge.
