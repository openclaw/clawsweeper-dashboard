## Summary

Keep open. Current main still lacks a typed gateway heap config and does not synthesize NODE_OPTIONS for installed gateway services, while the linked #42853 remains an open implementation candidate and the macOS LaunchAgent repro is useful review evidence.

## What This Changes

Request adding a durable openclaw.json gateway heap setting that regenerates macOS LaunchAgent service NODE_OPTIONS instead of relying on manual plist or env edits.

## Best Possible Solution

Land or replace #42853 with typed gateway.nodeMaxOldSpaceMb, an auto-detected default, generated service NODE_OPTIONS across gateway and node service installers, and regression coverage for launchd regeneration.

## Reproduction Assessment

No high-confidence live OOM reproduction was run in this read-only pass. The missing supported persistence path is source-reproducible on current main because config/schema omit the heap setting, service env generation omits NODE_OPTIONS, and durable config env blocks arbitrary NODE_OPTIONS.

## Solution Assessment

Yes, the requested typed heap-size config plus generated heap-only NODE_OPTIONS is safer than allowing arbitrary NODE_OPTIONS passthrough. The implementation should be reviewed against macOS launchd env-file regeneration and node-service inheritance before merge.
