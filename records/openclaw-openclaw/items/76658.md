## Summary

Keep open. Current main still lacks a typed gateway heap setting and does not synthesize NODE_OPTIONS for installed gateway services, while #42853 remains the open implementation candidate and this macOS LaunchAgent reproduction adds useful validation scope.

## What This Changes

Request a supported openclaw.json gateway heap setting that persists synthesized NODE_OPTIONS through macOS LaunchAgent regeneration.

## Best Possible Solution

Review and land or replace #42853 with a typed numeric gateway heap config, generated heap-only NODE_OPTIONS, and regression coverage for launchd, systemd, Windows tasks, and node-host inheritance.

## Reproduction Assessment

No high-confidence live OOM reproduction was established. The missing supported persistence path is source-reproducible on current main because config/schema lack nodeMaxOldSpaceMb, service env generation omits NODE_OPTIONS, and durable env filtering blocks arbitrary NODE_OPTIONS.

## Solution Assessment

Yes. A typed heap-size config with generated heap-only NODE_OPTIONS is narrower and safer than allowing arbitrary NODE_OPTIONS passthrough, but the implementation needs maintainer review across the service installers before merge.
