## Summary

Current main still appears vulnerable to the reported allowlist handoff: `alsoAllow`-only policy preserves implicit allow-all for normal filtering, but the optional media factory receives a reduced allowlist and can skip generation tools before `tools.effective` is built.

## What This Changes

The issue asks OpenClaw to keep media generation tools live when `tools.alsoAllow` is configured without `tools.allow`.

## Best Possible Solution

Preserve implicit allow-all semantics for built-in optional media factory planning while keeping the narrower plugin discovery behavior that avoids loading unrelated optional plugins.

## Reproduction Assessment

Yes, source-reproducible with high confidence: an `alsoAllow`-only policy becomes implicit `*`, but that `*` is intentionally stripped before the same list is reused to pre-gate media tool construction. I did not run a live Gateway reproduction in this read-only pass.

## Solution Assessment

Yes, the narrowest maintainable fix is to separate core optional-media allow-all planning from plugin discovery allowlisting, then add regression coverage for `tools.alsoAllow` without `tools.allow`.

Preserve existing plugin discovery behavior that strips the implicit `*` and avoids loading unrelated optional plugin tools, but ensure built-in optional media factory planning treats `tools.alsoAllow` without `tools.allow` as implicit allow-all so explicit/default-backed `image_generate`, `video_generate`, and `music_generate` are constructed unless denied.
