## Summary

Keep open. Current main handles the related `exec-approvals.json` rename-overwrite EPERM path, but this issue reports `spawn EPERM`; the Windows host exec spawn path can still propagate a spawn-time EPERM, and there is no native Windows log tying this report to the already-merged approvals fix.

## What This Changes

The issue reports that OpenClaw exec commands launched through Feishu or the web UI fail immediately with `spawn EPERM` on Windows 11.

## Best Possible Solution

Capture native Windows exec diagnostics first, then add a constrained host-spawn diagnostic or fallback only if the failing shell/path and original spawn error prove the root cause.

## Reproduction Assessment

No. Source inspection shows a spawn-time EPERM can still propagate through the current Windows host-exec path, but there is no native Windows run or log proving the all-commands failure on current main.

## Solution Assessment

Unclear. The merged approvals fallback is the right fix for the separate `exec-approvals.json` rename EPERM, but this issue needs targeted Windows evidence before changing shell selection or spawn retry behavior.
