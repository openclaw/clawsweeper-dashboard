## Summary

Keep open. Current main still has a plausible Windows host-exec spawn-time failure path matching `spawn EPERM`, and the related Microsoft tracker is an open draft tracker rather than a merged fix; the issue is fresh enough to need native Windows reproduction before changing host command-spawn fallback behavior.

## What This Changes

The issue reports that every agent `exec` command on Windows 11 fails immediately with `spawn EPERM` while the OpenClaw CLI itself still runs.

## Best Possible Solution

Windows host exec should either use a verified constrained fallback for the specific spawn mode that fails or return a diagnostic that identifies selected shell, argv, cwd, service/user context, and original spawn error fields.

## Reproduction Assessment

No. The issue gives plausible Windows 11 steps and current main still has a matching spawn-time path, but there is no native Windows log or selected-shell evidence to reproduce the exact `EPERM` source with high confidence.

## Solution Assessment

Unclear. There is no proposed patch, and a blanket `EPERM` retry is not the safest fix for host exec because it could hide real permission denials; native Windows proof should decide between a constrained fallback and clearer diagnostics.
