## What This Changes

The issue reports Windows 11 `exec` calls from Feishu or the web UI failing immediately with `spawn EPERM` while `openclaw --version` still works.

## Best Possible Solution

Capture native Windows exec diagnostics first, then add either a constrained validated fallback or a clearer diagnostic on the host spawn path with focused regression coverage.

## Reproduction Assessment

No. Source inspection shows an `EPERM` emitted during PowerShell spawn would propagate through current main, but there is no native Windows run or log proving the all-commands failure on current main.

## Solution Assessment

Unclear. A targeted diagnostic or constrained fallback may be the right fix, but choosing that safely needs native Windows spawn details rather than a generic `EPERM` retry.
