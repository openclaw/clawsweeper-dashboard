Current main still has a plausible native Windows host-exec spawn failure path: Windows exec resolves to PowerShell and runs through the child supervisor adapter, while the shared spawn fallback still retries only EBADF by default and not EPERM. The related Microsoft tracker is open inventory, not a merged or shipped fix.
