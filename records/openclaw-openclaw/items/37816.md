## Summary

Keep open. Current main has Telegram preview/progress/reasoning streaming and fixes for narrower delivery bugs, but it still lacks the requested Web UI-parity Telegram mode that renders each activity step as a persistent separate message; the remaining work needs UX/API/privacy direction tied to the broader observer-mode tracker.

## What This Changes

Request a Telegram mode/config option that renders every agent activity step as persistent separate messages, while excluding housekeeping turns such as NO_REPLY.

## Best Possible Solution

Keep this as the Telegram-specific parity tracker and design a privacy-safe persistent activity-message mode that reuses shared observer/event plumbing while keeping housekeeping and NO_REPLY turns invisible.

## Solution Assessment

No ready implementation is present yet. A new Telegram mode or config may be viable, but the maintainable path needs maintainer-approved privacy, UX, and observer-contract design coordinated with https://github.com/openclaw/openclaw/issues/12831.
