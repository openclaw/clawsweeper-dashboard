## Summary

Keep open. Current main and v2026.5.20 have Telegram preview/progress drafts, visible/suppressible tool-progress controls, and reasoning preview behavior, but they still do not provide the requested opt-in durable per-step Telegram activity transcript; the remaining work needs product and security scoping.

## What This Changes

Request an opt-in Telegram mode that renders agent thinking, tool calls, memory writes, and the final answer as persistent separate Telegram messages with Web UI-like activity visibility.

## Best Possible Solution

Keep this as the Telegram-specific parity tracker and design an opt-in, redacted durable activity-message mode over shared observer/session-event plumbing while preserving current preview and progress streaming semantics.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source, docs, tests, related issue context, and latest-release inspection give high confidence that OpenClaw lacks the requested persistent per-step Telegram activity-message mode.

## Solution Assessment

Unclear until maintainers settle the UX, privacy, retention, and redaction contract. The maintainable direction is to reuse shared observer/session-event plumbing rather than add a competing Telegram-only event pipeline.
