## Summary

Keep open: current main has Telegram preview/progress drafts and the narrow verbose fallback was fixed, but it still does not provide durable, separate Telegram messages for each agent activity step; the broader observer work is open/draft and does not supersede this Telegram-specific UX request.

## What This Changes

Request a Telegram mode or config option that renders every agent activity step as persistent chat messages with Web UI-like visibility.

## Best Possible Solution

Define an explicit opt-in activity visibility design with redaction, NO_REPLY exclusion, rate and volume limits, and a clear choice between a Telegram-owned persistent-message mode and shared observer plumbing.

## Reproduction Assessment

Not applicable. This is a feature request, and source/docs inspection shows current main supports edited preview/progress drafts and verbose tool-progress bubbles rather than durable per-step Telegram activity messages.

## Solution Assessment

No complete solution is on main yet. The suggested config direction is plausible, but the maintainable path needs product/security design and coordination with the broader observer-mode work before implementation.
