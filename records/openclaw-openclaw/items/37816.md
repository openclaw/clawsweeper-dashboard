## Summary

Keep open. Current main has Telegram preview/progress/reasoning streaming plus shared tool-event plumbing, but it still does not provide the requested durable per-step Telegram activity transcript; this needs maintainer UX/API/privacy scoping with https://github.com/openclaw/openclaw/issues/12831 and the open draft https://github.com/openclaw/openclaw/pull/60814.

## What This Changes

Request a Telegram mode that renders each user-facing agent activity step as a persistent separate message, closer to Web UI activity visibility.

## Best Possible Solution

Keep this as the Telegram-specific parity tracker and design an opt-in, privacy-reviewed activity-message mode over shared observer events, with housekeeping and NO_REPLY turns suppressed.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source, docs, and tests give high confidence current main lacks a persistent per-step Telegram activity-message mode.

## Solution Assessment

Unclear until maintainers settle the UX/API/privacy contract. The maintainable path is to reuse shared tool-events/session.tool observer plumbing rather than add a channel-only competing event pipeline.
