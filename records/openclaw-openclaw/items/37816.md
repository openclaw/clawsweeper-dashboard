## Summary

Keep open. Current main has Telegram preview/progress drafts and backend observer plumbing, but it does not provide the requested persistent, per-step Telegram activity-message mode; the remaining work needs maintainer UX, privacy, and config/API direction.

## What This Changes

The issue requests a Telegram mode or config that renders reasoning, tool calls, memory operations, and the final answer as persistent separate messages with Web UI-like visibility.

## Best Possible Solution

Design a Telegram activity-message mode that reuses existing tool-events, session.tool, and progress-rendering contracts, with explicit privacy filtering, NO_REPLY exclusion, and coordination with the broader observer-mode work.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source and docs inspection gives high confidence current main lacks the requested persistent Telegram activity-message mode: existing Telegram progress uses preview edits/status drafts, and off mode suppresses generic progress chatter.

## Solution Assessment

No current implementation is the complete solution yet. The maintainable path is a maintainer-approved Telegram activity mode built on the existing observer/progress plumbing rather than a channel-only parallel event system.
