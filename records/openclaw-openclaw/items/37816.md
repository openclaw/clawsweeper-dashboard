## Summary

Keep open. Current main has Telegram progress previews and shared `tool-events`/`session.tool` plumbing, but it still lacks the requested durable per-step Telegram activity transcript, and the current labels plus related tracker show this needs UX/privacy scoping rather than cleanup closure.

## What This Changes

The issue requests an opt-in Telegram mode that renders agent activity steps as persistent separate messages with Web UI-like visibility.

## Best Possible Solution

Keep this as the Telegram-specific parity tracker and design an opt-in, redacted durable activity-message mode over the shared observer/session tool event stream, with housekeeping and `NO_REPLY` turns suppressed.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source, docs, and tests give high confidence that current main lacks a persistent per-step Telegram activity-message mode.

## Solution Assessment

Unclear until maintainers settle the UX, API, privacy, retention, and redaction contract. The maintainable direction is to reuse shared `tool-events` and `session.tool` plumbing rather than add a competing Telegram-only event pipeline.
