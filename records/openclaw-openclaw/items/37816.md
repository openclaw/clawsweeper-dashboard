## Summary

Keep open: current main has Telegram verbose/tool-progress previews and the narrow verbose fallback was fixed, but it still lacks the requested durable per-step Telegram activity transcript; the broader observer work remains open/draft and does not supersede this Telegram-specific UX request.

## What This Changes

Request an opt-in Telegram mode that renders each agent activity step as persistent separate messages with Web UI-style visibility.

## Best Possible Solution

Define an explicit opt-in activity visibility design with redaction, NO_REPLY exclusion, rate and volume limits, and a clear choice between a Telegram-owned persistent-message mode and a shared observer contract.

## Reproduction Assessment

Not applicable. This is a feature request, and source/docs inspection shows current main supports edited preview/progress drafts and verbose tool bubbles rather than durable per-step Telegram activity messages.

## Solution Assessment

No complete implementation is on main yet. The suggested config direction is plausible, but the maintainable solution needs product/security design and coordination with broader observer-mode work before code changes.

## Risks / Open Questions

- Persistent Telegram activity messages could expose reasoning, tool arguments, command output, memory details, or local paths without a clear redaction and retention policy.
- A per-step Telegram transcript could create message volume and Bot API rate-limit pressure unless batching, truncation, and opt-in controls are designed first.
- The architecture overlaps the open observer-mode tracker and draft Control UI activity PR, so maintainers should choose whether Telegram gets a channel-local mode or a shared observer contract.
