## Summary

Keep open: current main has compact Telegram tool-progress preview/progress drafts, but it does not implement the requested durable per-step Telegram activity transcript or a full Web UI parity mode.

## What This Changes

The issue requests an opt-in Telegram mode that renders each agent activity step as a persistent visible message instead of only final replies or edited previews.

## Best Possible Solution

Define an explicit opt-in activity visibility design, likely either a Telegram-owned persistent activity-message mode or a shared observer contract aligned with #12831, with clear redaction, NO_REPLY, and rate/volume rules.

## Reproduction Assessment

Not applicable. This is a feature request, and source/docs inspection shows current main supports edited preview/progress drafts rather than durable per-step activity messages.

## Solution Assessment

No. The suggested config direction is plausible, but the best fix needs maintainer product/security direction because the current compact preview implementation is intentionally different from a persistent full-activity transcript.
