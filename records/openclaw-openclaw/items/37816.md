## Summary

Keep open: current main and the latest release still do not provide the requested persistent per-step Telegram activity history; existing Telegram progress is a transient draft, while the merged Control UI Activity work explicitly left Telegram durable activity as separate scope.

## What This Changes

The issue requests an opt-in Telegram mode that renders agent activity as persistent per-step messages with Web UI-like visibility.

## Best Possible Solution

Define a default-off Telegram durable activity mode backed by a typed, redacted activity-summary contract with explicit reasoning, volume, truncation, NO_REPLY, and failure semantics.

## Reproduction Assessment

Not applicable: this is a feature request for a new durable Telegram activity mode. Current source, docs, and tests confirm the existing Telegram activity surface is preview/verbose/reasoning behavior rather than persistent per-step history.

## Solution Assessment

No: literal Web UI parity with raw reasoning and unrestricted tool results is not the safest maintainable solution. A shared, typed, redacted activity-summary contract with Telegram-specific volume controls is the better boundary.

Summary: The requested feature is security-sensitive because it would persist internal agent activity into Telegram unless redaction, scope, and volume rules are defined first.
