## Summary

Keep open: current main and v2026.6.8 still do not provide the requested persistent per-step Telegram activity history. The existing surfaces are narrower: Telegram has verbose/tool-progress preview behavior, while the merged Control UI Activity work is browser-local and explicitly left Telegram durable activity as separate scope.

## What This Changes

The issue requests a Telegram mode that renders persistent per-step agent activity messages comparable to the Web UI Activity surface.

## Best Possible Solution

Define a default-off Telegram durable activity mode backed by a typed, redacted activity-summary contract with explicit reasoning, volume, truncation, NO_REPLY/internal-turn, and failure semantics.

## Reproduction Assessment

Not applicable: this is a request for a new durable Telegram activity mode, not a broken current-main behavior. Current source and docs show only verbose/tool-progress previews, persistent `/reasoning on`, and browser-local Activity summaries.

## Solution Assessment

No: literal Web UI parity with raw per-step reasoning/tool/memory messages is not the safest maintainable solution. The better path is a shared redacted summary contract plus Telegram-specific volume, truncation, and privacy controls.
