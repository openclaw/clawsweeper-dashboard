## Summary

Keep open: current main has partial Telegram observability through `/verbose`, `/reasoning`, and preview/progress streaming, but it does not provide the requested opt-in durable Web UI-parity activity mode where every agent step remains as separate persistent Telegram messages. The remaining work needs maintainer product and privacy scoping, not conservative cleanup closure.

## What This Changes

The issue requests an opt-in Telegram mode that persistently renders agent activity steps such as reasoning, tool calls, memory actions, and final answers as separate visible messages.

## Best Possible Solution

Define a privacy-reviewed, opt-in Telegram activity mode on top of the existing progress, verbose, and reasoning hooks, with persistent per-step messages, explicit exclusion of `NO_REPLY` or housekeeping turns, focused Telegram tests, docs, and a clear relationship to the broader observer-mode tracker.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature request. Source and docs inspection give high confidence that current main has partial `/verbose`, `/reasoning`, and preview-progress behavior but not the requested durable full activity mode.

## Solution Assessment

Unclear as a one-shot implementation because maintainers still need to choose the UX, config naming, privacy defaults, and relationship to the broader observer-mode work. The safest direction is a staged opt-in Telegram mode built on the existing event hooks rather than widening default progress visibility.
