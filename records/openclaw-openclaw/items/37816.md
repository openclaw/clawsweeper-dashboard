## Summary

Keep open. Current main and the latest release support Telegram preview/progress drafts and shared tool-event plumbing, but they still do not implement the requested opt-in, durable, separate Telegram activity-message transcript; the remaining work needs product and security scoping.

## What This Changes

The issue requests a Telegram mode/config option that renders each agent activity step as a persistent message with Web UI-style visibility.

## Best Possible Solution

Keep this as the Telegram-specific parity tracker and design an opt-in, redacted durable activity-message mode over shared observer/session event plumbing while preserving existing preview streaming semantics.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source, docs, tests, related issue context, and latest-release inspection give high confidence that OpenClaw lacks the requested persistent per-step Telegram activity-message mode.

## Solution Assessment

Unclear until maintainers settle the UX, privacy, retention, and redaction contract. The maintainable direction is to reuse shared tool-events/session.tool plumbing rather than add a competing Telegram-only event pipeline.
