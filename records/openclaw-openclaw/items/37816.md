## Summary

Keep open. Current main has Telegram preview tool-progress and reasoning-preview support, but the requested persistent, separate per-step Web UI parity mode is still missing and remains a product/config design decision.

## What This Changes

Request a Telegram mode/config that renders agent activity steps as persistent separate messages, including tool activity and reasoning, while excluding NO_REPLY housekeeping turns.

## Best Possible Solution

Define a sanitized activity-event contract and Telegram delivery mode that preserves final delivery semantics, keeps NO_REPLY/internal turns silent, and aligns with #12831's cross-surface observer policy.

## Reproduction Assessment

Not applicable as a feature request. There is a high-confidence source/docs inspection path showing current Telegram supports preview edits and progress lines, but not persistent per-step activity messages.

## Solution Assessment

Unclear. A Telegram-only persistent-message mode could satisfy the surface request, but the best implementation needs maintainer decisions on reasoning exposure, event sanitization, and whether the shared observer work in #12831 should provide the core contract.
