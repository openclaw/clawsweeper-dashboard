## Summary

Keep open. Current main has Telegram preview/progress/reasoning streaming and shared tool-event plumbing, but it still does not provide the requested durable per-step Telegram activity transcript; the open related tracker and current labels point to a product and privacy decision rather than cleanup.

## What This Changes

The issue requests a Telegram mode that renders each agent activity step as persistent chat messages for Web UI parity.

## Best Possible Solution

Keep this as the Telegram-specific parity tracker and design an opt-in, redacted durable activity-message mode over the shared observer/session.tool event stream, with housekeeping and NO_REPLY turns suppressed.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source, docs, and existing tests give high confidence current main lacks a persistent per-step Telegram activity-message mode.

## Solution Assessment

Unclear until maintainers settle the UX, API, privacy, and retention contract. The maintainable direction is to reuse shared observer/session.tool events rather than add a Telegram-only competing event pipeline.
