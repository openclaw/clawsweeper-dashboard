## Summary

Keep open. Current main has partial Telegram visibility through preview/progress drafts and a fixed verbose-progress fallback, but it still does not provide the requested durable, per-step Telegram activity messages; the broader observer-mode tracker and dashboard draft are related, not a replacement for this Telegram-specific feature.

## What This Changes

Request a Telegram mode/config that renders full agent activity as persistent separate messages with Web UI-like visibility.

## Best Possible Solution

Keep this as the Telegram-specific activity-visibility tracker, reuse existing progress/event plumbing where possible, and define a privacy-safe durable message mode in coordination with the broad observer-mode work.

## Reproduction Assessment

Not applicable as a bug reproduction. Source and docs review gives high confidence current main only supports preview/progress visibility, not the requested persistent per-step Telegram activity stream.

## Solution Assessment

No complete solution is on main yet. The maintainable path is a maintainer-approved Telegram activity mode that defines what can be persisted, how it relates to existing preview/progress modes, and how it coordinates with https://github.com/openclaw/openclaw/issues/12831 and https://github.com/openclaw/openclaw/pull/60814.
