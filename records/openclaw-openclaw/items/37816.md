## Summary

Keep open: current main and the latest release have useful Telegram preview, progress, verbose, and reasoning visibility, but they still do not provide the requested opt-in durable Telegram activity transcript where each agent step is sent as its own persistent message. The remaining work needs maintainer product and privacy scoping before implementation.

## What This Changes

Request a Telegram mode that mirrors Web UI activity visibility by rendering thinking, tool calls, memory operations, progress, and the final answer as persistent per-step chat messages.

## Best Possible Solution

Define a privacy-reviewed opt-in Telegram activity mode that builds on existing progress, verbose, and reasoning hooks, excludes housekeeping/NO_REPLY turns, persists only approved per-step summaries, and ships with focused Telegram tests, docs, and live proof.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature request. Source and docs inspection give high confidence that current main has partial Telegram observability but not the requested durable full-activity message mode.

## Solution Assessment

Unclear as a one-shot implementation because maintainers still need to decide Telegram UX, config naming, redaction, retention, and overlap with observer-mode work. A staged opt-in mode is safer than widening default channel visibility.
