## Summary

Keep open. Current main has Telegram preview/progress drafts, tool-progress controls, native nonpersistent draft support, and shared Web UI tool-event plumbing, but it still lacks the requested opt-in durable per-step Telegram activity transcript; the remaining scope needs product and security decisions.

## What This Changes

The issue requests an opt-in Telegram mode that renders durable per-step agent activity messages comparable to the Web UI activity visibility.

## Best Possible Solution

Keep this as the Telegram-specific parity tracker and design an opt-in, redacted durable activity-message mode over shared observer/session-event plumbing while preserving current preview, progress, and native-draft semantics.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source, docs, tests, and related issue context give a high-confidence check that OpenClaw lacks the requested persistent per-step Telegram activity-message mode.

## Solution Assessment

Unclear until maintainers settle the UX, privacy, retention, and redaction contract. The maintainable direction is to reuse shared observer/session-event plumbing rather than add a competing Telegram-only event pipeline.
