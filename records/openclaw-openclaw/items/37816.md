## Summary

Keep open. Current main has Telegram preview/progress/reasoning streaming and a narrow verbose-progress fix, but it still does not provide a Web UI-parity mode where every agent activity step is rendered as persistent separate Telegram messages; this remains a product/API/privacy decision tied to the broader observer-mode work.

## What This Changes

Request a Telegram mode or config that renders thinking, tool, memory, and final-answer activity as persistent separate messages.

## Best Possible Solution

Keep this as the canonical Telegram parity tracker and design a privacy-safe persistent activity-message mode that reuses shared observer/event plumbing while keeping housekeeping and NO_REPLY turns invisible.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source, docs, and tests give high confidence that current main lacks the requested persistent full-activity Telegram mode.

## Solution Assessment

No ready implementation is present yet. A new Telegram config/mode may be viable, but the maintainable path needs maintainer-approved privacy, UX, and observer-contract design coordinated with the broader tracker.
