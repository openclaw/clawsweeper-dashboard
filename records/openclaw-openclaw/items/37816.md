## Summary

Keep open: current main has Telegram preview/progress streaming and a browser-local Control UI Activity tab, but it still does not provide the requested persistent Telegram activity messages; the merged Activity tab PR explicitly left this issue as a distinct Telegram follow-up.

## What This Changes

Request a Telegram mode that renders agent activity, reasoning, tool, memory, and final-answer steps as persistent separate chat messages for Web UI parity.

## Best Possible Solution

Keep this as the canonical Telegram durable activity tracker and design an opt-in, redacted activity mode that reuses sanitized activity plumbing where possible and is proven with live Telegram behavior.
