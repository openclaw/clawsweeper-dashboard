## Summary

Keep open. Current main and the latest release have Telegram preview/progress drafts plus shared tool-event plumbing, but they still do not provide the requested opt-in, durable, per-step Telegram activity transcript, and the feature needs product and security scoping before implementation.

## What This Changes

Request a Telegram mode or config option that renders each agent activity step as a persistent visible message, closer to Web UI activity visibility.

## Best Possible Solution

Keep this as the Telegram-specific parity tracker and design an opt-in, redacted durable activity-message mode over the shared observer/session event stream while preserving existing preview streaming and suppressing housekeeping or NO_REPLY turns.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source, docs, tests, and latest-release inspection give high confidence that OpenClaw lacks a persistent per-step Telegram activity-message mode.

## Solution Assessment

Unclear until maintainers settle the UX, privacy, retention, and redaction contract. The maintainable direction is to reuse shared tool-events/session.tool plumbing rather than adding a competing Telegram-only event pipeline.
