## Summary

Keep open: current main and v2026.6.10 provide Telegram preview/progress drafts and browser-local Control UI Activity, but not the requested persistent per-step Telegram activity history. This remains a product and security design item because exposing reasoning, tool details, and memory activity in Telegram changes disclosure and retention boundaries.

## What This Changes

The issue requests a Telegram mode that renders every agent activity step as persistent chat messages, matching Control UI-style activity visibility.

## Best Possible Solution

Keep this open for a default-off Telegram activity design that defines disclosure, redaction, retention, message-volume policy, and the config/API boundary before implementation.

## Reproduction Assessment

Yes at source level: current main documents and implements Telegram preview/tool-progress drafts and browser-local Activity, but not a durable Telegram activity-message mode. The gap is visible in the accepted streaming modes, Telegram dispatch path, and Control UI Activity scope.

## Solution Assessment

No, not as proposed yet: a new `full` mode or `activityMessages` flag is plausible, but maintainers first need to choose the disclosure taxonomy, redaction rules, retention behavior, and message-volume policy. The safest direction is a default-off, redaction-aware Telegram activity design rather than simply mirroring raw Web UI activity into chat.
