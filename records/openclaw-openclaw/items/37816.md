## Summary

Keep this open. Current main has Telegram preview streaming, default tool-progress preview lines, and a separate reasoning preview lane, but it still lacks the requested opt-in mode that posts each agent activity step as a durable separate Telegram message; the related broader observer issue remains open and does not replace this Telegram-specific request.

## What This Changes

Request an opt-in Telegram mode/config that renders each agent activity step as persistent separate messages with Web UI-style visibility while suppressing NO_REPLY/internal housekeeping.

## Best Possible Solution

Add an opt-in Telegram activity-message mode with defaults unchanged, NO_REPLY/internal-run suppression, redaction and truncation, rate limiting/coalescing, config schema/UI/docs coverage, and focused Telegram dispatch tests; add only a narrow generic channel activity seam if Telegram cannot express the needed events locally.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a bug reproduction because this is a feature request, but source inspection on current main verifies the requested durable Telegram activity-message mode is absent.

## Solution Assessment

Is this the best way to solve the issue? Unclear until maintainers decide the privacy/product semantics; the requested direction is plausible, but the narrow maintainable path should be opt-in, redacted, rate-limited, and Telegram-owned unless a generic activity event seam is required.
