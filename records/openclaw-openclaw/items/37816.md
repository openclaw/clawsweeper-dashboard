## Summary

Keep open: current main has useful Telegram observability through verbose, reasoning, and editable preview/progress drafts, but it still lacks the requested opt-in durable Telegram mode where each agent activity step persists as its own message. The remaining work needs maintainer product and security scoping before implementation.

## What This Changes

Request an opt-in Telegram activity mode that mirrors Web UI agent activity visibility with persistent per-step messages and no preview deletion/replacement.

## Best Possible Solution

Define a privacy-reviewed opt-in Telegram activity mode on top of existing progress, verbose, and reasoning hooks, with persistent per-step messages, explicit housekeeping and NO_REPLY exclusions, focused Telegram tests, docs, and coordination with the broader observer-mode tracker.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature request. Source and docs inspection give high confidence that current main has partial observability but not the requested durable full-activity Telegram mode.

## Solution Assessment

Unclear as a one-shot implementation because maintainers still need to decide UX, config naming, privacy defaults, redaction, and overlap with observer-mode work. A staged opt-in Telegram mode is safer than widening default progress visibility.
