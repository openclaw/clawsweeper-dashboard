## Summary

Keep open. Current main has Telegram preview/progress/reasoning streaming and backend tool-event plumbing, but it still lacks the requested mode where every user-facing agent activity step becomes a persistent separate Telegram message; the remaining work needs product, API, and privacy scoping with https://github.com/openclaw/openclaw/issues/12831 and the open draft https://github.com/openclaw/openclaw/pull/60814.

## What This Changes

Request a Telegram mode or config that renders each user-facing agent activity step as persistent, separate messages for Web UI parity.

## Best Possible Solution

Keep this as the Telegram-specific parity tracker and design a privacy-safe persistent activity-message mode over shared observer events, with housekeeping and NO_REPLY turns suppressed.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source, docs, and tests give high confidence that current main lacks the requested persistent per-step Telegram activity mode.

## Solution Assessment

Unclear until maintainers settle the UX/API/privacy contract. The maintainable path is a scoped Telegram mode that reuses shared observer/event plumbing rather than a channel-only one-off.
