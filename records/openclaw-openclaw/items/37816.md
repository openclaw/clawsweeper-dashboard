## Summary

Keep open: current main has compact Telegram tool-progress previews and progress drafts, but it still does not provide the requested durable per-step Telegram activity transcript. The broader observer-mode tracker at https://github.com/openclaw/openclaw/issues/12831 is relevant, but this issue remains the Telegram-specific product/design surface.

## What This Changes

Request adds a Telegram mode or config that renders each agent activity step as persistent separate messages with Web UI-style visibility.

## Best Possible Solution

Define an explicit opt-in Telegram activity visibility design, aligned with https://github.com/openclaw/openclaw/issues/12831, with redaction, NO_REPLY filtering, retention, and rate/volume rules before adding config.

## Reproduction Assessment

Not applicable. This is a feature request, and source/docs inspection shows current main supports edited preview/progress drafts rather than durable per-step Telegram activity messages.

## Solution Assessment

No complete solution is on main yet. The suggested config direction is plausible, but the maintainable fix needs product/security design around persistent reasoning/tool output and coordination with the broader observer-mode tracker.

## Risks / Open Questions

- Persistent Telegram activity messages could expose reasoning, commands, tool output, memory paths, or local file details without a redaction policy.
- Telegram per-step messages could create rate-limit, notification-spam, and retention problems unless volume and grouping rules are designed first.
- Scope overlaps with the broader observer-mode work at https://github.com/openclaw/openclaw/issues/12831, so maintainers should choose whether this is channel-specific or shared infrastructure.
