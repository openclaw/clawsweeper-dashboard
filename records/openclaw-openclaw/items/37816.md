## Summary

Keep this issue open for maintainer product review. Current main now exposes Telegram tool/item progress and persistent reasoning, but progress remains a draft that is cleared before final delivery, so the requested permanent per-step activity history is still missing; the merged Control UI Activity work also explicitly left Telegram durable activity as separate scope.

## What This Changes

Add an opt-in Telegram mode that preserves agent activity steps as separate messages instead of replacing or deleting them before the final response.

## Best Possible Solution

Define an opt-in Telegram activity mode that sends separate persistent messages only from a typed, redacted activity-summary contract, excludes raw chain-of-thought and housekeeping turns, and specifies truncation, coalescing, and message-volume limits.

## Reproduction Assessment

Not applicable: this requests a new durable Telegram activity mode. Current main can show progress, but source and documentation confirm that its progress draft is cleared rather than retained per step.

## Solution Assessment

No. Literal Web UI parity with raw reasoning and unrestricted tool results is not the safest maintainable design; a shared, typed, sanitized activity-summary contract is the better boundary.
