## Summary

Keep open: current main has useful partial Telegram visibility through `/verbose`, `/reasoning`, preview edits, and progress drafts, but it still does not provide the requested opt-in durable Web UI-parity activity mode where every agent step remains visible as its own Telegram message. The remaining decision is product and privacy scope, not conservative cleanup closure.

## What This Changes

The issue requests an opt-in Telegram mode that renders full agent activity, including thinking, tool calls, intermediate results, and memory operations, as persistent separate messages like the Web UI.

## Best Possible Solution

Define a privacy-reviewed, opt-in Telegram activity mode on top of the existing progress, verbose, and reasoning hooks, with persistent per-step messages, explicit exclusion of `NO_REPLY` or housekeeping turns, focused Telegram tests, docs, and alignment with the broader observer-mode tracker.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature request. Source and docs inspection give high confidence that current main has partial `/verbose`, `/reasoning`, and preview-progress behavior but not the requested durable full activity mode.

## Solution Assessment

Unclear as a one-shot implementation because maintainers still need to choose the UX, config naming, privacy defaults, and relationship to the broader observer-mode work. The safer direction is a staged opt-in Telegram mode built on existing event hooks rather than widening default progress visibility.
