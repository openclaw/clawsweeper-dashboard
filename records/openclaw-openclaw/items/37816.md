## Summary

Keep open. Current main and the latest release provide Telegram preview/progress drafts plus shared `tool-events` and `session.tool` plumbing, but they still do not provide the requested opt-in, durable per-step Telegram activity transcript; the request also needs privacy, redaction, retention, and UX scoping before implementation.

## What This Changes

Request a Telegram mode/configuration that renders each agent activity step as its own persistent visible Telegram message with final-answer delivery afterward.

## Best Possible Solution

Keep this as the Telegram-specific parity tracker and design an opt-in, redacted durable activity-message mode over the shared observer/session event stream, while preserving existing preview streaming semantics and suppressing housekeeping/`NO_REPLY` turns.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source, docs, tests, and latest-release inspection give high confidence that current OpenClaw lacks a persistent per-step Telegram activity-message mode.

## Solution Assessment

Unclear until maintainers settle the UX, API, privacy, retention, and redaction contract. The maintainable direction is to reuse shared `tool-events` and `session.tool` plumbing instead of adding a competing Telegram-only event pipeline.
