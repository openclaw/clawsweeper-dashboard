## Summary

Keep open. Current main still lacks visible Telegram callback acknowledgements, and an open focused PR is already linked with closing syntax, so this issue should wait for that implementation decision rather than being cleaned up early.

## What This Changes

The issue requests immediate visible Telegram feedback for inline keyboard clicks, either through configurable callback acknowledgement text or by pre-editing the message.

## Best Possible Solution

Resolve the focused Telegram PR by landing a plugin-local opt-in acknowledgement config with schema, docs, tests, and changelog if maintainers accept the UX, or close that PR if visible callback feedback is not desired.

## Reproduction Assessment

Yes at source level. Any Telegram `callback_query` reaching the current handler is acknowledged through `ctx.answerCallbackQuery()` or `bot.api.answerCallbackQuery(callback.id)` without `text` or `show_alert`; I did not run a live Telegram client check.

## Solution Assessment

Unclear. A configurable acknowledgement payload is a narrow plugin-local approach, but maintainers still need to choose the user-visible Telegram UX before treating the linked PR as the right fix.
