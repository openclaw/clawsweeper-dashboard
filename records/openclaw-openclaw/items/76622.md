## Summary

Keep open. Current main and v2026.5.2 still acknowledge Telegram callbacks without toast text, and adding visible callback feedback is new Telegram UX/config behavior with related but non-equivalent work in #42156 and #76717.

## What This Changes

The issue requests immediate visible Telegram feedback when an inline keyboard callback is acknowledged, either through configurable callback acknowledgement text or a pre-edit loading state.

## Best Possible Solution

Decide the Telegram callback-feedback UX, then implement it in the Telegram plugin with schema/type/docs/test coverage and coordination with the callback timing work in #42156.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: any Telegram inline-button callback that reaches the current handler is acknowledged without `text`, and the Telegram/grammY contract says omitted text shows no notification. I did not run a live Telegram client check.

## Solution Assessment

Unclear. Configurable `answerCallbackQuery` text is a narrow technical path, but maintainers need to choose whether toast text, alerts, pre-editing, ack reactions/stickers, or a combination is the intended Telegram UX.
