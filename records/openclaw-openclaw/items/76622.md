## Summary

Keep open. Current main still acknowledges Telegram callback queries without visible text and has no `callbackAck` config surface, but #76975 is an open PR with closing syntax for this issue, so cleanup should wait for that PR to be reviewed, merged, or closed.

## What This Changes

Allow Telegram inline-button callback acknowledgements to include immediate visible toast or alert feedback instead of only clearing the spinner.

## Best Possible Solution

Review and either land or revise #76975's opt-in `callbackAck` implementation, while leaving #42156 to track the separate queued-update timeout behavior.

## Reproduction Assessment

Yes. A source-level reproduction is any Telegram inline-button callback reaching the current handler: it calls `answerCallbackQuery` without a payload, and the dependency contract says omitted `text` shows no notification; I did not run a live Telegram client.

## Solution Assessment

Yes, if maintainers accept the UX/config direction. The open PR's opt-in `callbackAck` schema/type/docs/test approach is a narrow implementation that preserves default behavior, but it should be reviewed before this issue closes.
