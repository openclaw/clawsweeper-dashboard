## Summary

Keep open. Current main and the latest release still acknowledge Telegram callback queries without any toast text, while the requested visible acknowledgement is a new Telegram UX/config behavior that needs maintainer product direction rather than cleanup closure.

## What This Changes

Requests Telegram inline button clicks to provide immediate visible feedback, either through configurable callback acknowledgement text/alert behavior or by pre-editing the message before model processing.

## Best Possible Solution

Decide the Telegram UX policy first, then implement it inside the Telegram plugin with schema/type/docs/test coverage and coordination with #42156 so acknowledgement timing and visible feedback do not diverge.

## Reproduction Assessment

Yes. A source-level reproduction is any Telegram inline-button callback that reaches the current handler: OpenClaw acknowledges the callback without `text`, and the upstream API contract says omitted text shows no notification; I did not run a live Telegram client check.

## Solution Assessment

Unclear. Configurable `answerCallbackQuery` text is a narrow technical path, but maintainers need to decide whether a toast, alert, pre-edit, default copy, or per-account setting is the right user-facing behavior.
