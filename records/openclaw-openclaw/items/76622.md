## Summary

Keep open. Current main and v2026.5.2 still acknowledge Telegram callback queries without `text`, and open PR #76975 already targets this request, so this issue should remain open until that PR lands, is replaced, or is closed.

## What This Changes

Request visible Telegram inline-button click acknowledgement, likely through configurable `answerCallbackQuery` text/showAlert or an immediate message edit.

## Best Possible Solution

Resolve this through PR #76975 or a maintainer-approved equivalent that adds an opt-in Telegram acknowledgement payload with schema, types, docs, changelog, and regression coverage while preserving default no-toast behavior.

## Reproduction Assessment

Yes. A source-level reproduction is any Telegram inline-button callback reaching the current handler: it acknowledges with no `text`, and Telegram documents that omitted `text` shows nothing to the user; I did not run a live Telegram client check.

## Solution Assessment

Unclear. The open PR’s opt-in config is a narrow implementation path, but maintainers still need to approve whether toast text/showAlert is the desired Telegram UX versus pre-editing or another policy.
