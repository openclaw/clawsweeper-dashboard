## Summary

Keep open. Current main and the latest release still acknowledge Telegram callback queries without `text`, and the requested visible acknowledgement is new Telegram UX/config behavior that needs maintainer direction rather than cleanup closure.

## What This Changes

Request immediate visible feedback for Telegram inline-button clicks, either by adding configurable callback acknowledgement text or pre-editing the message before model processing.

## Best Possible Solution

Decide the Telegram acknowledgement UX policy first, then implement it inside the Telegram plugin with schema/types/docs/tests and coordination with #42156 so timing and visible feedback stay consistent.

## Reproduction Assessment

Yes, source-level. Any Telegram inline-button callback reaching the current handler is acknowledged with no `text`, and Telegram's API contract says omitted `text` shows nothing to the user; I did not run a live client check.

## Solution Assessment

Unclear. Configurable acknowledgement text is a narrow technical option, but maintainers still need to choose the desired Telegram UX: toast, alert, pre-edit, default copy, or per-account setting.
