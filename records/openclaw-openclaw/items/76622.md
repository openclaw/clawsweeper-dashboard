## Summary

Keep open. Current main still acknowledges Telegram callback queries without visible toast/alert text, and #76975 is an open closing PR for the focused implementation, so this issue should not be closed before that PR is reviewed or resolved.

## What This Changes

Request optional immediate Telegram callback acknowledgement text/alert so inline button clicks show visible feedback before model processing.

## Best Possible Solution

Review #76975 as the implementation candidate; if accepted, keep the behavior Telegram-plugin-local, default-compatible, documented, and tested, while leaving #42156 to handle callback timing separately.

## Reproduction Assessment

Yes, at source level. Any Telegram inline-button callback that reaches `extensions/telegram/src/bot-handlers.runtime.ts` is acknowledged without a text/options payload, and the grammY/Telegram type contract says omitted `text` shows no notification.

## Solution Assessment

Unclear until maintainers review the linked PR. The opt-in `callbackAck` approach is narrow and default-compatible, but it still adds a new Telegram-visible config/UX policy and should be coordinated with the separate timing work in #42156.
