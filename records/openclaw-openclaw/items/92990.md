## What This Changes

The PR changes the doctor empty-allowlist scanner to suppress a top-level groupAllowFrom warning when every enabled account has its own allowlist.

## Best Possible Solution

Keep the scanner’s top-level diagnostic pass and suppress only the false default groupAllowFrom advisory when account-level group routing is fully covered.

## Reproduction Assessment

Yes. Current main checks the top-level channel record before account records, and the proposed head then skips that whole top-level path when account allowlists are present.

## Solution Assessment

No. The fix should be narrower: suppress only the false default groupAllowFrom warning while keeping top-level DM and plugin-specific warning collection intact.
