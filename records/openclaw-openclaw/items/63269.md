## Summary

Keep open. Current main still has the nominal Mattermost WebSocket `posted` parser and channel/group dispatch path, but I found no source, test, changelog, release, or related-PR proof that the reported public/group `oncall` @mention regression is fixed; #72282 was closed back to this as the canonical tracker, while the latest Mattermost changelog entries address adjacent keepalive, inbound-envelope, and threading/kind issues.

## What This Changes

Mattermost should receive and process public/group WebSocket messages, at least @mentioned `oncall` posts with `groupPolicy: "open"`, instead of only DMs working.

## Best Possible Solution

Fix the Mattermost bundled plugin so allowed public, private, and group `posted` WebSocket frames reach the inbound dispatch path, including @mentioned `oncall` posts with open group policy, and add focused regression coverage plus useful redacted diagnostics for frames that are filtered or never parsed.
