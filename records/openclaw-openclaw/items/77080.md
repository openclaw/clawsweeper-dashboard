## Summary

Keep open: current main exposes WhatsApp quoted stanza IDs as reply metadata but does not use them for session route selection, and the requested WhatsApp thread-binding behavior is a new channel feature that needs maintainer design review.

## What This Changes

Request to route WhatsApp reply-quoted inbound messages to the session that sent the quoted outbound stanza by adding WhatsApp threadBindings support, persistence, config, and CLI/admin visibility.

## Best Possible Solution

Implement a WhatsApp-owned DM-first reply-quote binding layer that records outbound stanza IDs, resolves quoted inbound stanza IDs before dispatch, tests fallback/expiry behavior, and updates WhatsApp config/docs through existing channel/plugin seams.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a bug reproduction because this is a feature request; source inspection on current main shows quoted stanza IDs are available as `ReplyToId` but never used to override WhatsApp session routing.

## Solution Assessment

Is this the best way to solve the issue? Unclear until maintainer design review; deterministic reply-quote lookup in the WhatsApp plugin is plausible, but persistence, defaults, DM/group scope, fallback, and CLI behavior should be decided first.
