## Summary

Keep open: current main still treats WhatsApp quote stanza IDs as reply metadata only, with no WhatsApp conversation-binding or `threadBindings` route override. This is a new WhatsApp channel feature with config, persistence, CLI/admin, and DM/group semantics that need maintainer design review.

## What This Changes

Add WhatsApp reply-quote based thread binding so quoted inbound messages route to the session that originally sent the quoted outbound stanza.

## Best Possible Solution

Add a WhatsApp-owned conversation-binding implementation that records outbound stanza IDs, resolves quoted inbound stanza IDs before dispatch, and documents/tests deterministic DM-first fallback behavior through existing plugin/session-binding seams.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken documented WhatsApp behavior. Source inspection gives high confidence that current main exposes quoted stanza IDs as context but does not use them for session routing.

## Solution Assessment

Unclear until maintainer design review: the proposed direction fits the WhatsApp plugin and existing conversation-binding concepts, but persistence, defaults, CLI surface, and group semantics are product choices.
