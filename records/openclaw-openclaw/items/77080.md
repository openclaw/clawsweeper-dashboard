## Summary

Keep open: current main does not implement WhatsApp reply-quote routing into thread-bound sessions. WhatsApp already extracts `contextInfo.stanzaId` as reply metadata, but inbound routing still resolves by the normal WhatsApp peer route and the WhatsApp plugin has no conversation-binding/thread-binding support or config surface.

## What This Changes

Request to route WhatsApp reply-quoted inbound messages to the session that sent the quoted outbound message by binding outbound stanza IDs to session keys.

## Best Possible Solution

Add a WhatsApp-owned conversation binding implementation that records outbound stanza IDs for session-originated replies, resolves quoted inbound stanza IDs before dispatch, and documents/tests the supported DM-first behavior with clear fallback semantics.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a new feature request. Source and docs give a high-confidence current-main gap because quoted stanza IDs are exposed as context only, not as a session-routing primitive.

## Solution Assessment

Unclear until maintainer design review: the proposed direction is plausible and belongs in the WhatsApp plugin plus shared conversation-binding seams, but storage, config defaults, and group behavior need owner judgment before implementation.
