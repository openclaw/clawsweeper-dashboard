## Summary

Keep open. This is a new WhatsApp plugin feature, not broken documented behavior: current main extracts quoted stanza IDs and exposes reply context, but inbound routing still resolves by WhatsApp peer before dispatch, and the WhatsApp plugin does not yet expose a conversation binding/thread-binding manager. The requested storage, config, CLI, and DM/group rollout need maintainer design review.

## What This Changes

Add WhatsApp reply-quote based thread-bound session routing so replies quoting an outbound stanza ID route back to the session that sent it.

## Best Possible Solution

Add a WhatsApp plugin-owned conversation binding manager that reuses the public thread-binding/conversation SDK seams, with docs/tests and a deliberate DM-first or default-off rollout if maintainers accept the product shape.

## Reproduction Assessment

Not applicable as a feature request. Source inspection confirms the current behavior: quoted stanza IDs become reply context, while WhatsApp inbound session routing remains peer-based.

## Solution Assessment

Unclear. The deterministic reply-quote mapping is plausible, but the proposed persistence model, config defaults, CLI surface, slash escape, and group-scope behavior need maintainer agreement before implementation.
