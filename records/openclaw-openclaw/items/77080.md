## Summary

Keep open: current main still exposes WhatsApp reply-quote stanza IDs only as message context, while session routing remains peer-based and WhatsApp has no thread-binding config or conversation-binding implementation.

## What This Changes

Add WhatsApp reply-quote based thread-bound session routing by binding outbound stanza IDs to session keys and resolving quoted inbound replies before normal DM routing.

## Best Possible Solution

Add a WhatsApp-owned conversation-binding implementation that records outbound stanza IDs for session-originated messages, resolves quoted inbound IDs before dispatch, and documents/tests DM-first fallback behavior after maintainers decide storage, config, and admin scope.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a new WhatsApp routing capability. Source inspection gives high confidence that current main keeps stanza IDs in reply metadata and does not use them to override `SessionKey`.

## Solution Assessment

Unclear until maintainer design review: the proposed reply-quote mapping is plausible and belongs in the WhatsApp plugin plus channel-binding surfaces, but storage, TTL defaults, CLI/admin scope, and DM/group semantics need owner judgment.
