## Summary

Keep open: current main exposes WhatsApp quoted stanza IDs as reply metadata, but there is no WhatsApp conversation-binding adapter, route override, config schema, persistence, or CLI surface for routing quoted replies back to the originating session. This is a new WhatsApp thread-binding feature that needs maintainer design review.

## What This Changes

Route inbound WhatsApp reply-quotes to the session that sent the quoted outbound message by recording outbound stanza IDs and resolving `contextInfo.quotedMessage.stanzaId` through thread bindings.

## Best Possible Solution

Implement a WhatsApp-owned conversation-binding layer that records accepted outbound message IDs, resolves quoted inbound IDs before dispatch, and documents/tests DM-first fallback behavior through the existing channel/plugin seams.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a bug reproduction because this is a feature request; source inspection gives high confidence that quoted stanza IDs are available as metadata but do not affect current-main route selection.

## Solution Assessment

Is this the best way to solve the issue? Unclear until maintainer design review; the direction fits the WhatsApp plugin and generic conversation-binding seams, but defaults, persistence, group scope, escape behavior, and CLI/admin surface need maintainer decisions first.
