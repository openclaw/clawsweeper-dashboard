## Summary

Keep open: current main still treats WhatsApp quoted stanza IDs as reply metadata, not session-routing input, and the request is a new WhatsApp thread-binding feature that needs maintainer design for persistence, defaults, config, CLI/admin surface, and DM/group semantics.

## What This Changes

Request to add WhatsApp reply-quote based thread-bound session routing by recording outbound stanza IDs and resolving inbound quoted stanza IDs to existing sessions.

## Best Possible Solution

Implement a WhatsApp-owned quote-stanza binding layer through the existing conversation/session binding seams, with DM-first routing, explicit fallback behavior, docs, and tests after maintainers settle config and admin scope.
