## Summary

Keep open: current main still exposes WhatsApp quote stanza IDs only as reply metadata, and no merged PR or narrower canonical item appears to implement stanza-to-session routing. The request is a new WhatsApp channel feature with config, persistence, fallback, and admin/UX choices that need maintainer design review.

## What This Changes

Add WhatsApp reply-quote based thread binding so inbound messages quoting an outbound stanza route back to that outbound message's session.

## Best Possible Solution

Add a WhatsApp-owned conversation-binding layer through the existing channel/plugin seams: record outbound stanza IDs, resolve quoted inbound stanza IDs before dispatch, document DM-first fallback behavior, and cover it with focused routing/config tests.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a bug reproduction; source inspection on current main shows quote stanza IDs are extracted and carried as ReplyToId while the session route is still resolved from the WhatsApp peer.

## Solution Assessment

Is this the best way to solve the issue? Unclear until maintainer design review; reply-quote mapping fits the existing binding direction, but persistence, defaults, group scope, and admin/escape behavior need product decisions first.
