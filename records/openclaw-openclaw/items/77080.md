## Summary

Keep open: current main still treats WhatsApp quoted stanza IDs as reply context metadata, not as a session-binding route key, and the requested routing mode is a new WhatsApp channel feature that needs maintainer design decisions.

## What This Changes

Request to add WhatsApp reply-quote based thread-bound session routing by binding outbound stanza IDs to session keys and resolving quoted inbound replies back to those sessions.

## Best Possible Solution

Add a WhatsApp-owned reply-quote conversation-binding implementation that records outbound stanza IDs, resolves quoted inbound stanza IDs before dispatch, and documents/tests DM-first fallback behavior through existing plugin/channel seams.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a new feature request. Source inspection gives a high-confidence current-main gap because quoted stanza IDs are exposed as context metadata but are not used for session routing.

## Solution Assessment

Unclear until maintainer design review: the proposed reply-quote binding direction fits the WhatsApp plugin boundary, but the persistence, defaults, CLI scope, and group behavior need product decisions.
