## Summary

Keep open: the prior discussion is useful and the current main tree still exposes WhatsApp quoted stanza IDs only as reply metadata, not as session-routing input. This is a new WhatsApp channel feature with routing, persistence, config, and admin-surface choices that should go through maintainer design review.

## What This Changes

Request to add WhatsApp thread-bound session routing by recording outbound stanza IDs and routing quoted inbound replies via `contextInfo.quotedMessage.stanzaId`/reply context metadata.

## Best Possible Solution

Add a WhatsApp-owned conversation-binding implementation that records outbound stanza IDs, resolves quoted inbound stanza IDs before dispatch, and documents/tests DM-first fallback behavior through the existing plugin/channel seams.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a new feature request. Source inspection gives a high-confidence current-main gap because quoted stanza IDs are exposed as reply metadata but are not used for session routing.

## Solution Assessment

Unclear until maintainer design review: the proposed direction is plausible and fits WhatsApp-owned routing, but storage durability, config defaults, CLI scope, and group-chat behavior are product choices.
