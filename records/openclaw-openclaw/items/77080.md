## Summary

Keep open: current main still lacks WhatsApp reply-quote routing into thread-bound sessions. WhatsApp extracts quoted stanza IDs and carries them as reply metadata, but inbound routing still resolves from the normal WhatsApp peer route, and the plugin has no conversation-binding/thread-binding manager or config surface for the requested behavior.

## What This Changes

Request to route WhatsApp quoted replies to the session that sent the quoted outbound message by binding outbound stanza IDs to thread-bound session keys.

## Best Possible Solution

Add a WhatsApp-owned conversation binding implementation that records outbound stanza IDs for session-originated sends, resolves quoted inbound stanza IDs before dispatch, and documents/tests the DM-first fallback behavior through the existing thread-binding policy seams.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a new feature request. Current main gives a high-confidence source check because quoted stanza IDs are exposed as reply metadata while session routing remains based on the WhatsApp peer route.

## Solution Assessment

Unclear until maintainer design review: the reply-quote binding direction is plausible and should live in the WhatsApp plugin plus generic conversation-binding seams, but storage, config defaults, CLI/admin scope, and group behavior need owner judgment first.
