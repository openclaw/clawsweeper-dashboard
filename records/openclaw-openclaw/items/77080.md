## Summary

Keep open. Current main has adjacent WhatsApp configured ACP binding and quote-context work, but it still does not bind reply-quoted WhatsApp outbound stanza IDs back to the session that sent them, so this remains a product/API design item rather than cleanup-closeable.

## What This Changes

The issue requests WhatsApp reply-quote stanza IDs to route inbound replies back to the session that sent the quoted outbound message.

## Best Possible Solution

Design this in the WhatsApp plugin and shared session-binding seam, using durable canonical state ownership, explicit DM/group scope, config/docs alignment, and regression coverage for quoted replies into spawned sessions.

## Reproduction Assessment

Source-reproducible as a current capability gap: WhatsApp quote ids are extracted, but current routing does not resolve quoted outbound stanza ids through a session-binding lookup. It is not a failing existing contract; it is a requested new routing mode.

## Solution Assessment

Unclear as proposed. The stanzaId primitive is plausible, but the maintainable solution should reuse the existing session-binding/conversation-binding machinery rather than introduce a separate ad hoc store and CLI surface without maintainer direction.
