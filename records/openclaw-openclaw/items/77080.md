## Summary

Keep open: current main still treats WhatsApp quoted stanza IDs as reply metadata, not as session-routing keys, and no merged PR or canonical related item appears to implement the requested stanza-to-session binding. This is a new WhatsApp channel feature with config, persistence, fallback, and UX choices that need maintainer design review.

## What This Changes

Request routing WhatsApp reply-quoted inbound messages to the session that originally sent the quoted outbound stanza via `threadBindings.spawnSessions`.

## Best Possible Solution

Implement a WhatsApp-owned DM-first quote-binding layer through existing channel conversation-binding seams, with documented fallback behavior, account scoping, TTL handling, and focused routing/config tests.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a bug reproduction; source inspection on current main shows quoted stanza IDs are extracted and carried as `ReplyToId` while the session route is resolved from the WhatsApp peer.

## Solution Assessment

Is this the best way to solve the issue? Unclear until maintainer design review; reply-quote mapping fits the existing binding direction, but persistence, defaults, group scope, and escape behavior need product decisions first.
