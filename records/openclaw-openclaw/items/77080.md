## Summary

Keep open: current main still exposes WhatsApp quoted stanza IDs only as reply metadata and does not have WhatsApp conversation-binding routing or `threadBindings` config, so this requested routing mode remains unimplemented and needs maintainer design review.

## What This Changes

Add WhatsApp reply-quote based thread-bound session routing by binding outbound stanza IDs to session keys and resolving quoted inbound stanza IDs before default DM routing.

## Best Possible Solution

Add a WhatsApp-owned conversation-binding implementation that records outbound stanza IDs, resolves quoted inbound stanza IDs before dispatch, and documents/tests conservative DM-first fallback behavior through existing generic binding seams where possible.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a new feature request, not a documented broken behavior. Source inspection gives high-confidence evidence of the current-main gap because quoted stanza IDs become `ReplyToId` but are never consulted for session routing.

## Solution Assessment

Unclear until maintainer design review: the proposed direction fits the WhatsApp plugin and existing conversation-binding seams, but storage durability, defaults, CLI scope, and group-chat behavior are product choices.
