## Summary

Keep open: current main lacks WhatsApp reply-quote session routing, and implementing it is a new WhatsApp plugin/thread-binding feature with config, storage, and product choices still open.

## What This Changes

The issue requests WhatsApp reply-quote based routing so inbound replies quoting an outbound stanza continue in the session that sent the quoted message.

## Best Possible Solution

Add a WhatsApp-owned conversation binding implementation that records accepted outbound message IDs to target session keys, resolves inbound `ReplyToId` before dispatch for DM support first, uses the existing session-binding policy helpers where possible, and documents fallback behavior.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a new routing feature request. Current-main source inspection is high confidence that quoted stanza ids are exposed as context metadata but do not alter `route.sessionKey`.

## Solution Assessment

Unclear until maintainer design review. The proposed reply-quote mapping is plausible and likely belongs in the WhatsApp plugin, but config defaults, persistence, CLI/admin behavior, and group-chat semantics need owner judgment before implementation.
