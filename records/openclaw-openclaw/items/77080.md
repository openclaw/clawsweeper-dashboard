## Summary

Keep open: current main still exposes WhatsApp reply-quote metadata only as inbound context, while session routing continues to use the normal WhatsApp peer route. The request is a plausible WhatsApp thread-binding feature, but it adds new routing behavior, config, storage, fallback, and CLI/admin choices that need maintainer design review before implementation.

## What This Changes

The issue requests WhatsApp reply-quote based routing for thread-bound spawned sessions by binding outbound stanza IDs to session keys and resolving quoted inbound stanza IDs before default DM routing.

## Best Possible Solution

Add a WhatsApp-owned conversation-binding design that records accepted outbound stanza IDs for session-originated messages, resolves quoted inbound stanza IDs before dispatch, and documents/tests DM-first fallback semantics.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a new feature request. Source and docs provide a high-confidence current-main gap because quoted stanza IDs are context metadata, not a session-routing primitive.

## Solution Assessment

Unclear until maintainer design review: the proposed stanzaId-to-session binding direction is plausible and should live in the WhatsApp channel/plugin area, but storage, config defaults, group behavior, and recovery semantics need owner judgment first.
