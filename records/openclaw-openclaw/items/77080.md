## Summary

Keep open: current main still exposes WhatsApp quoted stanza IDs only as reply metadata, not as a session-routing primitive. This is a new WhatsApp thread-binding feature with storage, config, CLI, and DM/group scope decisions, so it is not a narrow automated bug fix.

## What This Changes

The issue requests WhatsApp reply-quote based thread-bound session routing by binding outbound stanza IDs to sessions and resolving inbound quoted stanza IDs before normal WhatsApp routing.

## Best Possible Solution

Add a WhatsApp-owned conversation-binding implementation that records outbound stanza IDs for session-originated messages, resolves quoted inbound stanza IDs before dispatch, and documents/tests DM-first fallback behavior.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a new feature request. Source inspection gives high confidence that current main exposes quoted stanza IDs as `ReplyToId` metadata but does not use them for session routing.

## Solution Assessment

Unclear until maintainer design review: the proposed direction is plausible and likely belongs in the WhatsApp plugin using the existing conversation-binding seams, but storage, defaults, CLI, and group-chat semantics need owner judgment first.
