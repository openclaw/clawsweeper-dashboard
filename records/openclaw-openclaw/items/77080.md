## Summary

Keep open: current main still lacks WhatsApp reply-quote routing into thread-bound sessions. WhatsApp extracts quoted stanza IDs as reply metadata, but inbound routing still uses the normal peer route and the WhatsApp plugin has no conversation-binding manager or `threadBindings` config surface.

## What This Changes

The issue requests routing WhatsApp quoted replies by `contextInfo.quotedMessage.stanzaId` so replies to session-originated messages return to the original thread-bound/non-main session.

## Best Possible Solution

Add a WhatsApp-owned, DM-first conversation-binding implementation that records outbound stanza IDs for session-originated messages, resolves quoted inbound stanza IDs before dispatch, and documents/tests fallback behavior.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a new feature request. Source inspection gives high confidence in the current-main gap because quoted stanza IDs become `ReplyToId` metadata but are not used to override WhatsApp session routing.

## Solution Assessment

Unclear until maintainer design review: using quoted stanza IDs is plausible and belongs in the WhatsApp channel/plugin layer, but persistence, expiry, defaults, CLI surface, and group-chat scope need owner decisions first.
