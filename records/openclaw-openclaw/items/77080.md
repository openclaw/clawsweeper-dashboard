## Summary

Keep open: current main still does not route WhatsApp reply-quoted inbound messages to thread-bound sessions, and the request is a new WhatsApp channel feature with storage, config, CLI, and DM/group behavior choices that need maintainer design review.

## What This Changes

Request wiring WhatsApp reply-quoted inbound messages into thread-bound session routing by binding outbound stanza IDs to sessions and looking them up from quoted stanza IDs.

## Best Possible Solution

Add a WhatsApp-owned conversation-binding implementation that records outbound stanza IDs, resolves quoted inbound stanza IDs before dispatch, and documents/tests DM-first fallback behavior without hardcoding WhatsApp policy into core.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a new WhatsApp channel feature request. Source inspection gives a high-confidence current-main gap because quoted stanza IDs populate reply metadata but do not influence `route.sessionKey`.

## Solution Assessment

Unclear until maintainer design review: the proposed reply-quote binding direction fits the channel-owned adapter model, but persistence, config defaults, admin commands, and group behavior need product decisions before implementation.
