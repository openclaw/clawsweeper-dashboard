## Summary

Keep open. Current main and the latest release already expose WhatsApp quote metadata, but they do not route reply-quoted inbound messages through thread/conversation bindings, and the remaining work is a new WhatsApp-owned feature with config, persistence, and group-safety decisions.

## What This Changes

Request to route WhatsApp reply-quoted messages back to the spawned session that originally sent the quoted outbound message, using `contextInfo.stanzaId` as the native correlation key.

## Best Possible Solution

Keep this open for an owner-reviewed WhatsApp plugin conversation-binding design that reuses the existing session-binding infrastructure, persists the stanzaId-to-session mapping safely, and ships focused WhatsApp tests plus docs for the supported DM/group scope.

## Reproduction Assessment

Not applicable: this is a feature request, not a failing existing contract. Source and docs show current main extracts WhatsApp quote ids but does not use them for thread-bound session routing.

## Solution Assessment

Unclear as proposed: using `stanzaId` as the correlation key is reasonable, but a separate binding store plus CLI is likely over-scoped unless maintainers decide it should extend the existing plugin `conversationBindings` and session-binding service.
