## Summary

Keep open. Current main captures WhatsApp quote metadata and has adjacent configured ACP binding support, but it still does not bind outbound WhatsApp stanza IDs back to the session that sent them, so the requested reply-quote session routing remains unimplemented and needs product/API review.

## What This Changes

Request to route WhatsApp reply-quoted inbound messages to the session that sent the quoted outbound message by using `contextInfo.quotedMessage.stanzaId` with thread-bound session routing.

## Best Possible Solution

Design a DM-first WhatsApp reply-quote routing path inside the WhatsApp plugin and shared session-binding seam, with canonical durable state, explicit config/docs, and focused tests; treat group routing and active-window follow-ups as separate decisions.

## Reproduction Assessment

Yes, source inspection gives a high-confidence capability-gap path: WhatsApp extracts quoted stanza IDs and carries quote context, but current routing only resolves configured conversation bindings and does not look up quoted outbound stanza IDs. I did not run a live WhatsApp reproduction in this read-only review.

## Solution Assessment

Unclear as proposed. The stanzaId primitive is plausible, but the best implementation should reuse the existing session-binding and plugin-state machinery rather than adding a separate ad hoc JSON store and new CLI surface without maintainer direction.
