## Summary

Keep open: current main exposes WhatsApp quoted stanza IDs as reply metadata, but no WhatsApp conversation-binding/threadBindings surface or stanzaId lookup participates in inbound session routing; this is a new channel routing feature that needs maintainer design review.

## What This Changes

Request to add WhatsApp reply-quote based thread-bound session routing by binding outbound stanza IDs to session keys and resolving inbound quoted stanza IDs before default DM/group routing.

## Best Possible Solution

Add a WhatsApp-owned DM-first quote-binding implementation behind explicit WhatsApp threadBindings config, with outbound message-id recording, inbound quoted-stanza lookup before dispatch, lifecycle pruning, docs, and focused tests for fallback behavior.

## Solution Assessment
