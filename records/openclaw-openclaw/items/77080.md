## Summary

Keep open. Current main extracts WhatsApp quote stanza IDs and has adjacent configured ACP binding support, but it still does not bind outbound WhatsApp message IDs back to the originating session, so the requested routing remains a product/API decision.

## What This Changes

The issue requests WhatsApp reply-quote based thread-bound session routing using `contextInfo.quotedMessage.stanzaId`.

## Best Possible Solution

Ship a maintainer-approved DM-first WhatsApp plugin design that reuses the shared session-binding/state machinery, documents explicit config behavior, and leaves group routing plus active-window follow-ups as separate decisions.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence capability-gap path: WhatsApp extracts quoted stanza IDs and carries quote context, but current routing only resolves configured conversation bindings and does not look up quoted outbound stanza IDs.

## Solution Assessment

Unclear as proposed. The stanza-id primitive is plausible, but the best implementation should reuse the existing session-binding and SQLite-backed state machinery rather than add an ad hoc JSON store or broad CLI surface without maintainer direction.
