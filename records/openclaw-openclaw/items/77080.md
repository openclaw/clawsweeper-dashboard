## Summary

Keep open: current main still treats WhatsApp quoted stanza IDs as reply metadata, not a session-routing key, and the request adds a new WhatsApp routing mode with persistence, config, CLI, and DM/group behavior choices that need maintainer design.

## What This Changes

Add WhatsApp reply-quote based thread-bound session routing that maps outbound stanza IDs to session keys and routes quoted inbound replies back to the bound session.

## Best Possible Solution

Add a WhatsApp-owned conversation-binding implementation that records outbound stanza IDs, resolves quoted inbound stanza IDs before dispatch, and documents/tests DM-first fallback behavior without hardcoding WhatsApp-specific policy into core.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a new feature request. Source inspection gives a high-confidence current-main gap because quoted stanza IDs are exposed as context metadata but are not used for route/session resolution.

## Solution Assessment

Unclear until maintainer design review: the proposed direction is plausible and fits the WhatsApp plugin plus shared conversation-binding seams, but storage durability, config defaults, CLI scope, and group behavior are product choices.
