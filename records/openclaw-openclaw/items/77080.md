## Summary

Keep open: current main exposes WhatsApp quote stanza IDs as reply metadata but does not use them for session routing, and the requested behavior is a new WhatsApp thread-binding feature with persistence, config, CLI, and DM/group semantics that need maintainer design review.

## What This Changes

Add WhatsApp thread-bound session routing by binding outbound stanza IDs to sessions and routing quoted inbound replies through `contextInfo.quotedMessage.stanzaId`.

## Best Possible Solution

Implement a WhatsApp-owned conversation-binding layer that records outbound stanza IDs, resolves quoted inbound stanza IDs before dispatch, and documents/tests DM-first fallback behavior through existing channel/plugin seams.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a bug reproduction because this is a feature request; source inspection gives high confidence that current main has the gap because quote IDs are exposed as metadata but never feed route selection.

## Solution Assessment

Is this the best way to solve the issue? Unclear until maintainer design review; the proposed direction fits the WhatsApp plugin and existing conversation-binding seams, but persistence, defaults, group scope, and CLI/admin surface should be decided before implementation.
