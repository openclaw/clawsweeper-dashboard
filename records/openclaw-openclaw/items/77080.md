## Summary

Keep open: current main still exposes WhatsApp quoted stanza IDs only as reply metadata, while WhatsApp inbound routing is resolved from the normal peer route and there is no WhatsApp thread-binding storage, routing override, or config surface. This is a valid user-facing feature request that needs maintainer design review, not conservative cleanup.

## What This Changes

The issue requests using WhatsApp quoted-reply stanza IDs to route inbound replies back to the session that sent the quoted outbound message.

## Best Possible Solution

Add a WhatsApp-owned conversation-binding path that records outbound stanza IDs for session-originated sends, resolves quoted inbound stanza IDs before dispatch, and documents/tests a DM-first fallback model.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a new feature request. Source and docs provide a high-confidence current-main gap because quoted stanza IDs are metadata only and do not affect WhatsApp session routing.

## Solution Assessment

Unclear until maintainer design review: the proposed reply-quote routing direction is plausible and belongs in the WhatsApp plugin plus existing conversation-binding seams, but storage, config defaults, CLI surface, and group semantics need owner judgment.
