## Summary

Keep open: current main still exposes WhatsApp reply-quote stanza IDs only as reply metadata, while inbound routing resolves the WhatsApp peer route before dispatch and does not consult a stanza-to-session binding. This is a new WhatsApp thread-binding feature with storage, config, CLI, and DM/group rollout choices, so it needs maintainer design review before implementation.

## What This Changes

Add WhatsApp reply-quote based thread binding so inbound messages quoting an outbound stanza route to the session that sent the quoted message.

## Best Possible Solution

Add a WhatsApp-owned conversation-binding adapter that records outbound stanza IDs, resolves quoted inbound ReplyToId before dispatch, and documents/tests DM-first fallback behavior.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a new feature request. Source inspection on current main gives high-confidence evidence that quoted stanza IDs are exposed as context but not used for session routing.

## Solution Assessment

Unclear until maintainer design review. The proposed reply-quote mapping is plausible and belongs in the WhatsApp plugin/conversation-binding path, but persistence, defaults, admin commands, and group behavior need product and owner decisions.
