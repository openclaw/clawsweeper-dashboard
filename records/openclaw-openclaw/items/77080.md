## Summary

Keep open: current main still does not implement WhatsApp reply-quote routing into thread-bound sessions. Quoted stanza IDs are exposed as reply metadata, but WhatsApp inbound routing resolves by the normal peer route and the WhatsApp plugin/config/docs lack conversation-binding or `threadBindings` support.

## What This Changes

Add WhatsApp thread-bound session routing that records outbound stanza IDs and uses inbound quoted stanza IDs to route replies to the originating session, with config, storage, and CLI behavior.

## Best Possible Solution

Add a WhatsApp-owned conversation-binding design that records outbound stanza IDs, resolves quoted inbound stanza IDs before dispatch, and documents/tests the supported DM-first fallback semantics.

## Reproduction Assessment

Not applicable. This is a feature request, and source inspection gives a high-confidence current-main gap because quoted stanza IDs are exposed only as context metadata, not as a session-routing primitive.

## Solution Assessment

Unclear until maintainer design review. The proposed reply-quote binding direction is plausible and belongs in the WhatsApp plugin plus shared conversation-binding seams, but the storage, defaults, CLI surface, and group behavior need owner judgment first.
