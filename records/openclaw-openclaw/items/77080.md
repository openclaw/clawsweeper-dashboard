## Summary

Keep open: current main still does not route WhatsApp reply-quoted inbound messages to thread-bound sessions, and the request is a new WhatsApp channel feature with config, storage, CLI, and DM/group semantics that need maintainer design review.

## What This Changes

Add WhatsApp reply-quote based thread binding so inbound messages quoting an outbound stanza can route to the session that sent the quoted message.

## Best Possible Solution

Add a WhatsApp-owned conversation-binding implementation that records outbound stanza IDs, resolves quoted inbound stanza IDs before dispatch, and documents/tests DM-first fallback behavior without hardcoding WhatsApp-specific policy into core.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a new feature request. Source inspection gives a high-confidence current-main gap because quoted stanza IDs are exposed as context metadata but are not used for session routing.

## Solution Assessment

Unclear until maintainer design review: the proposed direction is plausible and fits the WhatsApp plugin plus shared conversation-binding seams, but storage durability, config defaults, CLI scope, and group behavior are product choices.
