## Summary

Keep open for maintainer review: the production fix is narrow and matches the source-reproducible doctor bug, but the added regression test mirrors the detector instead of exercising the production code path.

## What This Changes

This PR canonicalizes doctor-update git-root path comparison with `fs.realpathSync` fallback and adds doctor-update tests for symlink, direct-checkout, and non-git roots.

## Best Possible Solution

Land this narrow doctor-update fix after making the regression test exercise the real production detector path, with no broader update-flow changes.

## Reproduction Assessment

Yes. Current `main` compares the git top-level path to the supplied root as raw strings, so a symlinked or junction-backed root that resolves to the same checkout can still fall into `not-git`; the PR body also supplies terminal proof from a real symlink setup.

## Solution Assessment

Mostly yes. Canonicalizing both paths with a fallback is the narrow maintainable production fix and matches existing update-runner behavior, but the regression test should call the real doctor-update code instead of a mirrored helper.
