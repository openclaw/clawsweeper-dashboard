## Best Possible Solution

Land an inline-only auth-profile contract with doctor-owned legacy sidecar migration after maintainer review, green relevant CI, and explicit acceptance of the credential-storage tradeoff.

## Reproduction Assessment

Not applicable: this is an auth-storage cleanup PR rather than a standalone bug report. The current-main and PR-head source paths show the before/after `oauthRef` behavior, and the PR includes focused regression coverage for the migration path.

## Solution Assessment

Yes, with maintainer sign-off: moving compatibility into `openclaw doctor --fix` while keeping broad runtime inline-only is the narrow maintainable direction for retiring sidecar support. The safer alternative would be to keep runtime sidecar reads, but that would preserve the old credential contract this PR is intentionally removing.
