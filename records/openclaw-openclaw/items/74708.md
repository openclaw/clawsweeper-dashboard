## Summary

Keep open. This is a member-authored issue with the protected `maintainer` label, and current `main` still lacks the requested Gateway RPCs: the SDK environment namespace explicitly throws unsupported errors, the SDK docs call these methods unsupported, and the Gateway method registry has no `environments.list` or `environments.status` entry. The external SDK proposal/dogfood issue matches the same read-only discovery scope, so this needs maintainer API review rather than cleanup closure. ([github.com](https://github.com/openclaw/openclaw/issues/74708))

## What This Changes

Request to add read-only `environments.list` and `environments.status` Gateway RPCs so SDK clients can discover environment candidates, status, and capabilities while leaving provisioning unsupported.

## Best Possible Solution

Land an additive Gateway protocol change for read-only environment discovery: typed params/results, core handlers, method advertisement, least-privilege read scoping, SDK namespace calls, docs/generated client updates, and tests proving create/delete plus unsupported per-run runtime/environment options still fail loudly.
