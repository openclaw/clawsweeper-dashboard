## Summary

Keep open: current main and v2026.6.8 still build the Skill Workshop apply approval from fixed action text, the Control UI renders only generic plugin approval title/description/severity fields, and there is no documented Skill Workshop approval timeout knob. The reported 60s TTL is outdated because current plugin approvals default to 120s, but the central request for proposal-aware approval details and a longer or policy-backed approval window remains valid.

## What This Changes

The issue requests proposal metadata, a body preview, a details link, and a longer or configurable timeout in Skill Workshop apply approval prompts.

## Best Possible Solution

Define a proposal-aware Skill Workshop approval presentation and apply-specific timeout policy that keeps pending approval safe, avoids unnecessary config if a longer default is enough, and coordinates with the existing timeout PRs instead of duplicating them.

## Reproduction Assessment

Yes at source level: current main builds the apply approval from fixed action text, forwards only the generic plugin approval payload, and documents no Skill Workshop approval timeout setting. I did not run a live Control UI approval popup in this read-only review.

## Solution Assessment

Unclear as requested. Enriching the approval prompt is the right direction, but maintainers need to choose the durable approval payload/UI shape and whether timeout policy should be a longer default, an apply-specific timeoutMs, or a new config surface.
