## Summary

Keep open: current main and v2026.6.8 still build Skill Workshop lifecycle approvals from fixed action text and do not expose a proposal-aware popup or Skill Workshop-specific/configurable approval timeout. The reported 60s value is now partly outdated because generic plugin approvals default to 120s, but the central proposal-detail and timeout-policy request remains unresolved.

## What This Changes

Request to make Skill Workshop apply approvals show proposal details and use a longer or configurable approval timeout.

## Best Possible Solution

Define a bounded Skill Workshop approval presentation contract that carries safe proposal metadata and choose an approval-window policy that stays aligned with tool watchdogs and approval security defaults.

## Reproduction Assessment

Yes, at source level: `skill_workshop` lifecycle apply currently produces fixed generic approval text, the UI renders only generic plugin approval fields, and Skill Workshop config has no approval timeout field. I did not run a live Control UI approval popup in this read-only review.

## Solution Assessment

Unclear until maintainers choose the UX/security contract. A narrow fix should enrich the existing approval path rather than bypassing `approvalPolicy: pending`, and any longer timeout must be coordinated with the Codex/tool watchdog boundary.
