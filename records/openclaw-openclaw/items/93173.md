## Summary

Keep open: current main and v2026.6.8 still build Skill Workshop lifecycle approvals from static title/description text, the Control UI only parses/renders generic plugin approval fields, and `skills.workshop` has no approval-timeout field. Related open issues and PRs cover approval evidence or timeout pieces, but not the distinct proposal-details popup request.

## What This Changes

The issue asks for Skill Workshop apply approvals to include proposal context and for the approval timeout to be longer or configurable.

## Best Possible Solution

Define a narrow approval presentation contract for Skill Workshop lifecycle actions, enrich apply approvals with bounded proposal id/skill/description/preview context, and choose a longer default or approved timeout policy without weakening the approval boundary.

## Reproduction Assessment

Yes, from source inspection: lifecycle approvals are generated from fixed text, forwarded as generic plugin approval fields, and rendered by the UI without proposal metadata. I did not run a live UI reproduction in this read-only review.

## Solution Assessment

Unclear as requested. Enriching approval context is the right direction, but the exact timeout knob and deep-link/preview presentation need maintainer UX/API judgment before implementation.
