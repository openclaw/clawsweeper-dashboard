## Summary

Keep open: current main still builds Skill Workshop lifecycle approvals from fixed action text, the Control UI only renders the generic plugin approval title/description fields, and neither current config nor the latest release exposes a Skill Workshop approval timeout knob. The exact 60s timeout claim is partly outdated because current plugin approval defaults to 120s, but the central request for proposal-aware approval details and a longer/policy-backed approval window remains valid.

## What This Changes

Request to enrich Skill Workshop lifecycle approval popups with proposal details and make their approval timeout longer or configurable.

## Best Possible Solution

Define a narrow approval presentation contract for Skill Workshop lifecycle actions, enrich the approval with bounded proposal context, and choose a timeout policy that improves review time without adding config unless maintainers decide the knob is warranted.

## Reproduction Assessment

Yes. Source inspection shows `skill_workshop` lifecycle approvals are built from static text and the adjacent test locks that generic title/description behavior; I did not run a live UI repro because this review is read-only.

## Solution Assessment

Unclear as requested. Enriching the approval is the right direction, but the exact timeout knob and deep-link/preview presentation need maintainer UX/API judgment before an implementation PR.
