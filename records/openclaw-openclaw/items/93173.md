## Summary

Keep open: current main and v2026.6.8 still build Skill Workshop apply approvals from static action text, the approval transport/UI only exposes generic plugin approval fields, and there is no Skill Workshop approval timeout config. The related timeout PRs cover only part of the request, not the proposal-detail popup gap.

## What This Changes

Request to enrich Skill Workshop lifecycle approval popups with proposal context and provide a longer or configurable approval timeout.

## Best Possible Solution

Define a bounded Skill Workshop approval presentation contract, include proposal id/name/description plus a safe preview or inspect affordance, and choose one coordinated timeout policy that preserves pending approval.

## Reproduction Assessment

Yes, at source level. Current main builds the approval from static action text and the UI/transport only expose generic plugin approval fields; I did not run a live UI approval flow in this read-only review.

## Solution Assessment

Unclear as requested. Enriching the approval context is the right direction, but the exact timeout knob, default duration, preview size, and inspect affordance need maintainer UX/security/API judgment.
