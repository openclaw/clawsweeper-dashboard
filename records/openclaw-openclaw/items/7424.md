## Summary

Keep open. Current main still only has the global `agents.defaults.models` allowlist plus per-agent primary/fallback defaults, and the maintainer discussion keeps this narrower request separate from the broader account/provider policy work in #65557.

## What This Changes

Add a per-agent model allowlist so a family or specialized agent cannot list or select expensive or disallowed models.

## Best Possible Solution

Add an additive, documented per-agent model catalog/allowlist with explicit composition rules against `agents.defaults.models`, then enforce the effective allowed set consistently across model listing, model selection, session patching, and UI/TUI selectors.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is source and docs inspection showing current main only supports global `agents.defaults.models` plus per-agent primary/fallback defaults.

## Solution Assessment

Unclear because there is no proposed patch. The maintainable path needs a maintainer-approved additive config/protocol surface with explicit inheritance and enforcement semantics, not ad hoc overloading of the current strict per-agent `model` object.
