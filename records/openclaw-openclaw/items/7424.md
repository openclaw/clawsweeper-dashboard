## Summary

Keep open. Current main still has a global `agents.defaults.models` allowlist plus per-agent model defaults and metadata, but not an agent-scoped allowlist enforced across model listing, model switching, or session patch validation; related implementation attempts are closed unmerged and the broader account/provider policy issue does not supersede this narrower request.

## What This Changes

Request a per-agent model allowlist so an agent can only list or select approved models instead of inheriting the global model catalog.

## Best Possible Solution

Define one documented effective per-agent model policy with explicit inheritance semantics, then enforce that shared resolver consistently in listing, selection, session patching, tools, UI, and regression tests.

## Reproduction Assessment

Not applicable as a feature request. Source inspection provides a high-confidence verification path: the enforced allowlist is still derived from global `agents.defaults.models`, while per-agent `models` is metadata/runtime override shape only.

## Solution Assessment

Unclear until maintainers choose the public config and protocol semantics. The maintainable direction is a shared effective model-policy resolver rather than filtering only one picker or command path.
