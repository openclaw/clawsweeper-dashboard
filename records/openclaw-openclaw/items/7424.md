## Summary

Current main still lacks the requested per-agent model visibility and enforcement allowlist. The model catalog/allowlist path is still global, while the active implementation attempt at https://github.com/openclaw/openclaw/pull/80394 remains open with compatibility and UI-scope blockers, so this issue should stay open for maintainer/API follow-up rather than be closed as implemented or superseded.

## What This Changes

Request a per-agent model allowlist so one agent can be prevented from selecting expensive or inappropriate models while sharing the same gateway.

## Best Possible Solution

Define a documented per-agent model visibility contract with explicit inheritance/replacement semantics, then enforce the effective allowlist consistently in config schema/types, `models.list`, `/model`, session patching, UI/TUI model pickers, and regression tests.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction of the gap: current main derives configured model visibility from `agents.defaults.models`, and both gateway/UI catalog requests lack agent-scoped parameters. This is a feature gap rather than a broken existing bug contract.

## Solution Assessment

The direction is right, but the exact solution is not settled. The active PR should preserve existing runtime-metadata semantics and close the supplemental picker leak before it can be considered the best implementation.
