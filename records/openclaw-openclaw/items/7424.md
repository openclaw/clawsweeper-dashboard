## Summary

Current main still has only a global model visibility allowlist; its per-agent `models` field supplies metadata/runtime overrides rather than restricting selection. The issue remains necessary, and the open implementation PR needs maintainer reconciliation because it is currently conflicting and defines a new public config/protocol contract.

## What This Changes

Add a per-agent model allowlist so each agent can expose and select only an administrator-approved subset of configured models.

## Best Possible Solution

Define one typed effective per-agent model policy with explicit inheritance semantics, then use it consistently for Gateway catalog listing, `/model` selection, session patching, live execution, and UI/TUI selectors with focused compatibility tests.

## Reproduction Assessment

Yes at source level: current main’s allowed-set builder, Gateway catalog method, and Control UI request remain global, while the per-agent field is only a metadata/runtime override. This is a feature gap rather than a regression in documented behavior.

## Solution Assessment

Unclear until maintainers choose the inheritance semantics; the narrowest maintainable solution is one shared effective-policy computation rather than separate filtering in each UI or command path.
