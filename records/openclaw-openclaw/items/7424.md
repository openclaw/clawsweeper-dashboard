## Summary

Keep open. Current main still has only the global `agents.defaults.models` visibility allowlist, while the open implementation in https://github.com/openclaw/openclaw/pull/80394 is unmerged and currently conflicting; the closed duplicate and broader account-policy request do not supersede this narrower per-agent model policy request.

## What This Changes

Request to add a per-agent model allowlist so individual agents can hide or reject expensive or disallowed models instead of sharing the global configured model catalog.

## Best Possible Solution

Adopt one documented per-agent model visibility policy with explicit composition against `agents.defaults.models`, then enforce the effective policy consistently across Gateway model listing, `/model`, `sessions.patch`, UI/TUI selectors, docs, and regression tests while leaving broader account policy to https://github.com/openclaw/openclaw/issues/65557.

## Reproduction Assessment

Yes for the source-level feature gap: current main reads configured visibility from global `agents.defaults.models`, `models.list` has no `agentId`, and `sessions.patch` validates against the global allowed set. This is a feature request rather than a failing runtime bug.

## Solution Assessment

Unclear until maintainers settle the public API semantics. A shared effective policy resolver threaded through Gateway, model selection, session patching, and UI/TUI callers is the maintainable direction.
