## Summary

Keep open. Current main has global `agents.defaults.models` allowlist enforcement plus adjacent per-agent model metadata, but it still does not define or enforce an agent-scoped model allowlist across listing and selection paths.

## What This Changes

The issue requests an agent-scoped model allowlist so a given agent can restrict which configured models users may select or spend credits on.

## Best Possible Solution

Define a documented per-agent effective model policy with explicit replace/compose/fallback semantics, then enforce that shared policy everywhere users can list or select models.

## Reproduction Assessment

Not applicable as a feature request. Source and docs inspection on current main gives a high-confidence verification path showing model allowlist enforcement remains global rather than agent-scoped.

## Solution Assessment

Unclear until maintainers choose the public config/protocol semantics. The maintainable fix should be a shared effective model-policy resolver reused by all model listing and selection paths, not a local-only picker filter.
