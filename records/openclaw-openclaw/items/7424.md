## Summary

Keep open. Current main still lacks an agent-scoped model visibility allowlist, and the related implementation in https://github.com/openclaw/openclaw/pull/80394 is open and unmerged; the closed duplicate and broader account-policy issue do not supersede this narrower request.

## What This Changes

The issue requests a per-agent model allowlist so one agent can expose/select only its permitted models instead of the global model catalog.

## Best Possible Solution

Adopt one documented per-agent model visibility policy with explicit fallback/composition rules, then enforce that effective policy consistently across Gateway model listing, `/model`, session patching, UI/TUI selectors, docs, and regression tests.

## Reproduction Assessment

Yes for the source-level feature gap: current main reads configured visibility from global `agents.defaults.models`, `models.list` has no `agentId`, and `sessions.patch` validates against the global allowed set. This is a feature request rather than a failing runtime bug.

## Solution Assessment

Unclear until maintainers settle the public API semantics. A shared effective policy resolver threaded through Gateway, model selection, session patching, and UI/TUI callers is the maintainable direction.
