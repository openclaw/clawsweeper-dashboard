## Summary

Keep open. Current main still supports a global `agents.defaults.models` allowlist plus per-agent primary/fallback defaults, but not a typed per-agent model allowlist; the provided discussion also keeps this narrower request separate from closed duplicate #9010 and broader account/provider policy #65557.

## What This Changes

Request an additive per-agent model allowlist so users of a specific agent cannot switch that agent to disallowed or expensive models.

## Best Possible Solution

Add a documented, typed per-agent model allowlist/catalog with explicit composition rules against `agents.defaults.models`, then enforce the effective set consistently across model listing, model selection, session patching, HTTP model overrides, and UI/TUI pickers.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is source and docs inspection showing current main lacks a typed per-agent model allowlist while documenting and enforcing only the global `agents.defaults.models` allowlist.

## Solution Assessment

Unclear because there is no proposed patch. The maintainable path needs a maintainer-approved additive config/protocol surface with explicit inheritance and enforcement semantics, not ad hoc overloading of the current strict per-agent `model` object.
