## Summary

Keep open. Current main still only has a global model catalog/allowlist plus per-agent primary/fallback defaults, and the recent issue discussion already keeps this as the narrower canonical request rather than folding it into broader account/provider policy work.

## What This Changes

The issue requests a per-agent model allowlist so one agent can hide or reject expensive or inappropriate models while sharing the same gateway.

## Best Possible Solution

Add an additive, documented per-agent model catalog/allowlist with explicit composition rules against agents.defaults.models, then enforce the effective allowed set consistently across selection and listing surfaces.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is source and docs inspection showing only a global agents.defaults.models allowlist plus per-agent primary/fallback defaults.

## Solution Assessment

Unclear because there is no proposed patch. The maintainable direction is a maintainer-approved config/protocol addition with explicit inheritance and enforcement semantics, not ad hoc overloading of the current strict primary/fallback model object.
