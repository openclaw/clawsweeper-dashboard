## Summary

Keep open. Current main still has global `agents.defaults.models` enforcement plus per-agent model/runtime metadata, but no documented per-agent model allowlist that scopes listing, selection, and session patch validation; https://github.com/openclaw/openclaw/issues/9010 is already closed as a duplicate, https://github.com/openclaw/openclaw/issues/65557 is broader account/provider policy work, and the related implementation PRs are closed unmerged.

## What This Changes

Add a per-agent model allowlist so cost-sensitive or specialized agents can only list and select approved models.

## Best Possible Solution

Design and document one effective per-agent model policy, then enforce it through shared resolver code for catalog listing, model selection, session patching, UI/TUI pickers, and regression tests.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence verification path: current model allowlist enforcement still comes from global `agents.defaults.models`, not an agent-scoped policy.

## Solution Assessment

Unclear until maintainers choose the public config and protocol semantics. The maintainable direction is a shared effective model-policy resolver rather than filtering only one picker or command path.
