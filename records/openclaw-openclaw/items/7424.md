- `impact:auth-provider`: The requested policy controls which configured provider/model routes each agent may expose and select.

## Summary

Keep this issue open: current main has per-agent model metadata but still builds and serves the selectable model catalog from global defaults, while the only active implementation is the open, conflicting https://github.com/openclaw/openclaw/pull/80394.

## What This Changes

Add a per-agent model allowlist that restricts each agent’s visible and selectable models to an operator-approved subset.

## Best Possible Solution

Define one documented agent-level model-policy contract with explicit global inheritance semantics, then derive a single effective allowed set used consistently by Gateway catalogs, UI and command pickers, session patches, and runtime model selection.

## Reproduction Assessment

Yes, at source level: current main’s catalog and alias paths still use global `agents.defaults.models`, and UI requests do not send an agent identity. A live failing run is not needed to establish that the requested capability is absent.

## Solution Assessment

The shared config and model-selection boundary is the correct place to solve this, but it is not ready for autonomous implementation because maintainers must decide whether the existing agent `models` map replaces, intersects with, or inherits the global catalog.
