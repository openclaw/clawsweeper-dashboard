## Summary

Keep open. Current `origin/main` and the latest release still use the global `agents.defaults.models` visibility source, `models.list` has no agent-scoping parameter, and the active implementation at https://github.com/openclaw/openclaw/pull/80394 remains open rather than merged.

## What This Changes

The issue requests a per-agent model allowlist so individual agents can be restricted from selecting costly or unsuitable provider models.

## Best Possible Solution

Adopt one documented agent-scoped model visibility policy that composes with `agents.defaults.models`, enforce it through Gateway, model selection, session patching, commands, UI/TUI selectors, docs, and tests, and leave broader per-account/provider policy to https://github.com/openclaw/openclaw/issues/65557.

## Reproduction Assessment

Yes for the source-level feature gap: current main reads configured visibility from `agents.defaults.models`, `models.list` rejects agent-scoping params, and `sessions.patch` validates model changes without an agent-specific visibility input. This is a feature request rather than a failing runtime bug.

## Solution Assessment

Unclear until maintainers choose the public config and protocol semantics. The maintainable direction is a shared effective policy resolver threaded through Gateway, model selection, session patching, commands, and UI/TUI callers.
