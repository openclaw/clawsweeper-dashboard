## Summary

Keep open. Current main still exposes global model visibility from `agents.defaults.models` and per-agent model metadata/defaults, but it does not yet provide a documented per-agent model allowlist enforced through `models.list`, model switching, and session patch validation; an open implementation PR exists and needs maintainer/API review rather than a separate cleanup close.

## What This Changes

Add a per-agent model allowlist so different agents can only see or select approved provider/model choices.

## Best Possible Solution

Review and reshape https://github.com/openclaw/openclaw/pull/80394 or a successor into one documented per-agent model visibility policy that preserves existing runtime metadata behavior and enforces the effective allowlist across listing, selection, session patching, UI/TUI, docs, and tests.

## Reproduction Assessment

Yes for the source-level feature gap: current main reads configured visibility from global `agents.defaults.models`, `models.list` has no `agentId`, and UI/session patch paths do not enforce an agent-scoped visibility policy. This is a feature gap, not a runtime bug reproduction.

## Solution Assessment

Unclear until maintainers settle the public API semantics. A shared effective policy resolver threaded through Gateway, model selection, session patching, and UI/TUI callers is the maintainable direction, but `agents.list[].models` metadata compatibility needs an explicit decision.
