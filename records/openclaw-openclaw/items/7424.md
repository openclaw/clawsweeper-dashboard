## Summary

Keep open. Current main still has a global model visibility allowlist driven by `agents.defaults.models`; per-agent `models` is documented and typed as metadata/runtime overrides, and the open implementation PR still needs maintainer/API review before this tracker can close.

## What This Changes

Allow each configured agent to have its own model allowlist so users cannot select costly or inappropriate models for that agent.

## Best Possible Solution

Review and reshape https://github.com/openclaw/openclaw/pull/80394 or a successor into one documented per-agent model visibility policy that preserves existing `agents.list[].models` metadata behavior and enforces the effective allowlist across listing, selection, session patching, UI/TUI, docs, and tests.

## Reproduction Assessment

Yes for the source-level feature gap: current main reads configured model visibility from global `agents.defaults.models`, `models.list` has no `agentId`, and `sessions.patch` validates against the global allowed set. This is a feature request rather than a failing runtime bug.

## Solution Assessment

Unclear until maintainers settle the public API semantics. A shared effective policy resolver threaded through Gateway, model selection, session patching, and UI/TUI callers is the maintainable direction.
