## Summary

Keep open. Current main still lacks an enforced per-agent model allowlist: agent-level `models` is documented and typed as metadata/runtime overrides, `models.list` is not agent-scoped, and the active implementation PR remains unmerged with review blockers around config/API semantics.

## What This Changes

Request a per-agent model allowlist so one gateway can prevent specific agents from seeing or selecting costly or inappropriate models.

## Best Possible Solution

Resolve the config/API semantics, then enforce one effective per-agent model set across model listing, `/model`, session patching, UI/TUI pickers, docs, and regression tests while preserving existing runtime metadata behavior.

## Reproduction Assessment

Yes. Source inspection on current main shows the model visibility parser reads only `agents.defaults.models`, `models.list` has no `agentId`, and session model validation has no agent-scoped allowlist input.

## Solution Assessment

Unclear until maintainers choose the public config/protocol semantics. The active PR threads `agentId` in the right direction, but prior review found compatibility and picker-leakage blockers that should be resolved before this issue closes.
