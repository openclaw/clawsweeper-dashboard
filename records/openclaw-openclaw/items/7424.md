## Summary

Keep open. Current main still lacks an enforced per-agent model allowlist: the effective visibility set is derived from global `agents.defaults.models`, while per-agent `models` is documented and implemented as metadata/runtime overrides. An open implementation PR exists, but it is not merged and already has maintainer-review blockers, so this issue remains the canonical tracker.

## What This Changes

The issue requests a per-agent model allowlist to prevent a specific agent from selecting expensive or inappropriate models.

## Best Possible Solution

Define a documented effective per-agent model policy, preserve existing per-agent model metadata behavior, and enforce the same policy in listing, selection, session patching, UI/TUI pickers, and regression tests.

## Reproduction Assessment

Not applicable as a feature request rather than a broken existing contract. Source inspection gives high confidence that current main still lacks agent-scoped model allowlist enforcement.

## Solution Assessment

Unclear until maintainers choose the public config and inheritance semantics. The maintainable direction is a shared effective model-policy resolver, not filtering only one picker or command path.
