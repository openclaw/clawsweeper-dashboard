Keep open. Current main still has only a global configured model visibility policy, while an open implementation PR exists but needs maintainer/API review before this tracker can close.

## What This Changes

The issue requests a per-agent model allowlist so a specific agent cannot show or select disallowed provider models.

## Best Possible Solution

Review and reshape https://github.com/openclaw/openclaw/pull/80394 or a successor so OpenClaw has one documented effective per-agent model visibility policy that preserves existing runtime metadata behavior and is enforced across listing, selection, session patching, UI/TUI, docs, and tests.

## Reproduction Assessment

Yes for the source-level feature gap: current main reads visibility from global `agents.defaults.models`, `models.list` has no `agentId`, and session patch validation does not pass agent scope. This is not a runtime bug reproduction.

## Solution Assessment

Unclear until maintainers settle the public API semantics. Threading an effective per-agent visibility policy through model listing and selection is the right direction, but the existing `agents.list[].models` runtime metadata contract must be preserved or explicitly migrated.
