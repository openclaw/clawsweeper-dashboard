## Summary

Keep open. Current main still lacks an enforced per-agent model allowlist: model visibility is still derived from global defaults, the Gateway model-list protocol is not agent-scoped, and the active implementation PR remains unmerged with compatibility review blockers.

## What This Changes

Request a per-agent model allowlist so individual agents can see and select only configured model subsets instead of the global model catalog.

## Best Possible Solution

Define one documented effective per-agent model policy, preserve existing runtime metadata behavior, and enforce the same policy in shared model selection, Gateway `models.list`, `/model`, `sessions.patch`, UI/TUI pickers, docs, and regression tests.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request for a new config/protocol policy. Source inspection gives high confidence that current main still only enforces global model visibility.

## Solution Assessment

Unclear until maintainers choose the public config and inheritance semantics. Threading `agentId` through a shared policy resolver is the right direction, but the current open PR still needs compatibility and picker-filtering fixes before it is the best solution.
