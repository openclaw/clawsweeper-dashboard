## Summary

Keep open. Current main still lacks an enforced per-agent model allowlist: agent-level model metadata exists, but the visibility policy, `models.list`, and session model validation remain based on the global configured model catalog, so the requested cost-control guardrail is not implemented.

## What This Changes

The issue requests a per-agent model allowlist so specific agents cannot expose or select expensive or inappropriate models.

## Best Possible Solution

Define a typed effective per-agent model policy with explicit inheritance against `agents.defaults.models`, then enforce it consistently in model listing, model selection, session patching, UI/TUI pickers, runtime paths, docs, and regression tests.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a runtime bug, but the feature gap is source-reproducible: current main only parses global model visibility, `models.list` has no agent parameter, and session patch validation does not receive an effective per-agent allowlist.

## Solution Assessment

Is this the best way to solve the issue? Unclear until maintainers settle the public API semantics; the maintainable direction is an explicit effective per-agent model policy rather than overloading existing metadata without documented inheritance rules.
