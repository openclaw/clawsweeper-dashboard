## Summary

Keep open. Current main has an adjacent `agents.list[].models` metadata/runtime override, but model allowlist enforcement, aliases, Gateway model listing, and session patch validation still resolve against global `agents.defaults.models`; the closed PR attempts are not merged and the broader account-policy issue does not supersede this narrower per-agent request.

## What This Changes

Request to let each agent define its own model allowlist so users cannot switch that agent to disallowed or expensive models.

## Best Possible Solution

Define a documented per-agent effective model policy with explicit replace/compose/fallback semantics, then enforce that shared policy everywhere users can list or select models.

## Reproduction Assessment

Not applicable as a feature request. Source/docs inspection on current main gives a high-confidence verification path showing model allowlist enforcement remains global rather than agent-scoped.

## Solution Assessment

Unclear until maintainers choose the public config/protocol semantics. The maintainable fix should be a shared effective model-policy resolver, not a local-only picker filter.
