## Summary

Keep open. Current main has per-agent model defaults/fallbacks and per-agent model metadata/runtime overrides, but it still does not define or enforce an agent-scoped model allowlist for catalog listing, model switching, or session patch validation.

## What This Changes

Request a per-agent model allowlist so one configured agent can be prevented from selecting expensive or inappropriate models.

## Best Possible Solution

Define one typed effective per-agent model policy with explicit inheritance/replacement semantics, then enforce it through shared listing and validation helpers with focused config, Gateway, command, and UI regression coverage.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence verification path: current main still derives the enforced allowlist from global agents.defaults.models, not an agent-scoped policy.

## Solution Assessment

Unclear until maintainers choose the public config and protocol semantics. The maintainable direction is a shared effective model-policy resolver rather than filtering only one picker or command path.
