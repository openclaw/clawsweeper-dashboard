## Summary

Keep open. Current main still implements only a global `agents.defaults.models` allowlist plus per-agent primary/fallback defaults; there is no typed per-agent model catalog or agent-scoped enforcement path, and the related broader account-policy issue does not supersede this narrower request.

## What This Changes

The issue requests a per-agent model allowlist so one agent can be restricted from selecting costly or unsuitable models.

## Best Possible Solution

Define a documented additive per-agent model catalog/allowlist with explicit composition rules against `agents.defaults.models`, then enforce the effective set consistently anywhere users can list or select models.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is source and docs inspection showing current main documents and enforces only the global `agents.defaults.models` allowlist.

## Solution Assessment

Unclear until maintainers choose the public API and inheritance semantics. The requested direction is useful, but the maintainable solution needs a typed config/protocol/UI design rather than a local-only guard.
