## Summary

Keep open. Current main still uses a global `agents.defaults.models` visibility catalog, while the related implementation work in https://github.com/openclaw/openclaw/pull/80394 is still open and not yet a merged fix; https://github.com/openclaw/openclaw/issues/9010 is a closed duplicate and https://github.com/openclaw/openclaw/issues/65557 is broader account/provider policy work.

## What This Changes

Add a per-agent model allowlist so family or specialized agents can only see or select approved provider/model entries.

## Best Possible Solution

Land one documented per-agent model visibility policy with explicit fallback/composition rules and shared enforcement across model listing, `/model`, session patching, UI/TUI selectors, docs, and regression tests.

## Reproduction Assessment

Yes for the source-level feature gap: current main reads configured model visibility from global `agents.defaults.models`, `models.list` has no `agentId`, and `sessions.patch` validates against the global allowed set. This is a feature request rather than a failing runtime bug.

## Solution Assessment

Unclear until maintainers settle the public API semantics. A shared effective policy resolver threaded through Gateway, model selection, session patching, and UI/TUI callers is the maintainable direction.
