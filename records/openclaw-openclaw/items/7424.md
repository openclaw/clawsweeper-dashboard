## Summary

Keep open. Current `main` still does not provide an enforced per-agent model allowlist: visibility is sourced from the global `agents.defaults.models`, `models.list` has no request-level agent scope, and the open implementation PR still needs maintainer API/product review before this issue can close.

## What This Changes

The issue requests a per-agent model allowlist so a shared gateway can prevent selected agents from seeing or switching to costly or inappropriate models.

## Best Possible Solution

Define a documented per-agent model visibility policy that preserves existing runtime metadata semantics, then enforce the effective allowed set consistently in `models.list`, `/model` selection, session patching, UI/TUI pickers, runtime paths, docs, and regression tests.

## Reproduction Assessment

Not applicable as a runtime bug, but the feature gap is source-reproducible: current main parses visibility from `agents.defaults.models`, `models.list` has no `agentId`, and session model validation is not scoped to an effective per-agent allowlist.

## Solution Assessment

Unclear until maintainers settle the public API semantics. Threading agent scope through model visibility is the right general direction, but the open implementation still needs review because the existing `agents.list[].models` field already has runtime metadata semantics.
