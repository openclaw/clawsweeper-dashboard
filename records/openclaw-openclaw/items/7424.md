## Summary

Keep open. Current main still lacks an enforced per-agent model allowlist: visibility and model patch validation are still driven by the global `agents.defaults.models` policy, while the existing per-agent `models` field is documented and implemented as model metadata/runtime overrides. The open implementation PR targets this request, but it is conflicting and already has compatibility/API review findings, so the issue should stay open for maintainer review rather than be closed or duplicated by a new repair lane.

## What This Changes

The issue requests a per-agent model allowlist so one agent cannot expose or select costly or inappropriate models configured for another agent.

## Best Possible Solution

Review and reshape https://github.com/openclaw/openclaw/pull/80394 or a successor so OpenClaw has a documented per-agent visible model policy that preserves existing runtime metadata behavior and enforces one effective policy across model listing, `/model`, session patching, UI/TUI selectors, and docs.

## Reproduction Assessment

Yes for the feature gap by source inspection: current main uses global `agents.defaults.models` for visibility, `models.list` has no `agentId`, and session patch validation does not pass agent scope. This is not a runtime bug reproduction.

## Solution Assessment

Unclear until maintainers settle the public API semantics. Threading an effective per-agent visibility policy through model listing and selection is the right direction, but the existing `agents.list[].models` runtime metadata contract must be preserved or explicitly migrated.
