## Summary

Keep open. Current main still lacks an enforced per-agent model allowlist: `agents.list[].models` is documented and typed as model metadata/runtime overrides, while listing, selection, and session patch validation still center on the global `agents.defaults.models` allowlist.

## What This Changes

Request a per-agent model allowlist so each agent can expose and accept only approved models.

## Best Possible Solution

Define one typed effective per-agent model policy with explicit inheritance or replacement semantics, then enforce it through shared model listing, model selection, session patch validation, UI/TUI pickers, and regression tests.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main gives a high-confidence verification path: the enforced allowlist still comes from global `agents.defaults.models`, not an agent-scoped policy.

## Solution Assessment

Unclear until maintainers choose the public config and protocol semantics. The maintainable direction is a shared effective model-policy resolver rather than filtering only one picker or command path.
