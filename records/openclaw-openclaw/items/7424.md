Keep open. Current main still lacks an enforced per-agent model allowlist; it has global `agents.defaults.models` visibility plus per-agent model metadata/runtime overrides, and the active implementation PR still needs maintainer/API direction before this issue should close.

## What This Changes

Add a per-agent model allowlist so each agent can expose and accept only approved models without running separate gateways.

## Best Possible Solution

Resolve the config/API semantics, update the active PR or a replacement to preserve runtime-only metadata behavior, then enforce the effective per-agent model set in model listing, `/model` selection, session patching, UI pickers, docs, and regression tests.

Yes. Source inspection shows current main has a global allowlist and per-agent runtime metadata, but `models.list`, model visibility parsing, and session model validation are not agent-scoped.

## Solution Assessment
