## Summary

Current main still lacks per-agent model authorization: `agents.defaults.models` remains the effective allowlist, while `agents.list[].models` only supplies per-model metadata/runtime overrides. This remains the canonical focused request because the related issues are closed and the linked implementation PR was closed administratively without merging.

## What This Changes

Add a per-agent model allowlist so each agent exposes and accepts only operator-approved models.

## Best Possible Solution

Define a distinct, documented effective per-agent allowed-model contract that separates authorization from metadata, specifies composition with the global catalog and fallbacks, and is enforced across every model listing and selection path.

## Reproduction Assessment

Yes at source level. Current main has global model authorization plus per-agent defaults and metadata overrides, but no agent-specific permitted-model set used consistently by listing and selection enforcement.

## Solution Assessment

The requested outcome is appropriate, but no proposed config shape is yet proven best. Maintainers should first decide the authorization field and replace-versus-intersect inheritance semantics, especially because `agents.list[].models` already has a metadata meaning.
