## Summary

Keep this issue open. Current main has per-agent model metadata/runtime overrides, but the actual visibility allowlist still comes from the global `agents.defaults.models` map, so the requested per-agent cost guardrail is not implemented. The open implementation at https://github.com/openclaw/openclaw/pull/80394 owns this work but is conflicting and still needs a maintainer decision on the public policy and inheritance contract.

## What This Changes

Add an administrator-defined per-agent model allowlist that prevents an agent from exposing or selecting models outside its permitted set.

## Best Possible Solution

Adopt one documented per-agent authorization policy—likely a separate `modelAllowlist` that preserves existing metadata semantics—and enforce its effective set across model listing, commands, session mutation, persisted overrides, and runtime execution.

## Reproduction Assessment

Yes, at source level: current main derives the allowed model catalog exclusively from global configuration, while the existing per-agent model map has a different documented contract. This read-only review did not run a live gateway configuration.

## Solution Assessment

The requested outcome is appropriate, but overloading `agents.list[].models` is not the best approach because that field already owns metadata/runtime overrides. A distinct policy field is safer if maintainers explicitly define inheritance, empty-value behavior, and complete enforcement coverage.
