## Summary

Keep open. Current main still lacks an enforced per-agent model allowlist; the issue remains the canonical narrow feature request, with an open implementation PR to review rather than a merged fix to close against.

## What This Changes

The issue requests a per-agent model allowlist so a family or specialized agent can only expose and select models from its own configured set.

## Best Possible Solution

Review and reshape https://github.com/openclaw/openclaw/pull/80394 or a successor so the final contract documents a per-agent visible model policy, preserves existing runtime metadata compatibility, and enforces one effective policy across `models.list`, `/model`, `sessions.patch`, UI/TUI selectors, and tests.

## Reproduction Assessment

Yes for the feature gap by source inspection: current main still derives visible models from `agents.defaults.models`, `models.list` has no agent scope, and session patch validation does not receive the session agent id. This is not a runtime bug reproduction.

## Solution Assessment

Unclear until maintainers settle the public API semantics. Threading agent scope through model visibility is the right direction, but treating existing `agents.list[].models` metadata as the allowlist needs an explicit compatibility decision.
