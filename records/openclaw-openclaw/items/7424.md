## Summary

Keep open. Current main still has a global `agents.defaults.models` visibility policy, while the per-agent allowlist request remains unresolved and an open conflicting implementation PR is still under review.

## What This Changes

Request a per-agent model allowlist so one agent can be prevented from seeing or selecting expensive or inappropriate models.

## Best Possible Solution

Adopt one documented per-agent model visibility policy that composes with `agents.defaults.models`, then enforce it consistently across Gateway `models.list`, `/model`, session patching, UI/TUI selectors, docs, and tests while leaving broader account policy to https://github.com/openclaw/openclaw/issues/65557.

## Reproduction Assessment

Yes for the source-level feature gap: current main reads configured visibility from `agents.defaults.models`, rejects agent-scoped `models.list` params, and validates model changes without an agent-specific visibility input. This is a feature request rather than a failing runtime bug.

## Solution Assessment

Unclear until maintainers choose the public API semantics. A shared effective policy resolver threaded through Gateway, model selection, session patching, commands, and UI/TUI callers is the maintainable direction.
