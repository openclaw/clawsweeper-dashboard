## Summary

Keep open. Current main still has only a global model visibility allowlist from `agents.defaults.models`; per-agent `models` is documented and typed as metadata/runtime overrides, `models.list` is not agent-scoped, and the active implementation PR remains open with product/API review blockers.

## What This Changes

Request a per-agent model allowlist so each agent can expose and select only an approved subset of configured provider models.

## Best Possible Solution

Define a typed effective per-agent model policy, document its inheritance rules against `agents.defaults.models`, and enforce it consistently in model listing, `/model`, session patching, UI/TUI pickers, runtime selection, and regression tests.

## Reproduction Assessment

Not applicable as a runtime bug, but yes at source level: current main reads model visibility from `agents.defaults.models`, `models.list` has no agent-scoped request parameter, and session model validation has no agent allowlist input.

## Solution Assessment

Unclear until maintainers settle the public config/protocol semantics. The active PR moves in the right direction by threading `agentId`, but prior review found it must preserve existing runtime-only `agents.list[].models` behavior and apply policy to all picker/session paths.
