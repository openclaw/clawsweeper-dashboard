## Summary

Keep this issue open: current main has per-agent model metadata but no per-agent visibility allowlist, and https://github.com/openclaw/openclaw/pull/80394 is the active implementation candidate; that PR currently needs conflict resolution, refreshed proof, and maintainer agreement on the public config semantics.

## What This Changes

Add a per-agent model allowlist so each agent exposes and accepts only its permitted models.

## Best Possible Solution

Land one explicit, documented per-agent visibility policy without overloading `agents.list[].models`, and apply its effective catalog consistently to Gateway listing, `/model`, session validation, Control UI selection, aliases, wildcards, and fallback behavior.

## Reproduction Assessment

Yes, at source level: current main derives model visibility from `agents.defaults.models`, while each agent has only model metadata and primary/fallback configuration, not an independent visibility allowlist.

## Solution Assessment

Yes directionally, but the best implementation is the active PR's revised explicit allowlist field rather than assigning new restriction semantics to the existing per-agent metadata map; inheritance and empty-list behavior still require maintainer approval.
