## Summary

Keep open. Current main still has only a global enforced model allowlist from `agents.defaults.models`; the per-agent `models` field is typed and documented as per-model metadata/runtime overrides, not as an agent-scoped selection policy.

## What This Changes

The issue requests a per-agent model allowlist so one agent can be restricted from selecting costly or inappropriate models.

## Best Possible Solution

Define one typed effective per-agent model policy with explicit inheritance/replacement semantics, then enforce it through shared model listing and validation helpers with focused regression coverage.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence verification path: the enforced allowlist still comes from global `agents.defaults.models`, not an agent-scoped policy.

## Solution Assessment

Unclear until maintainers choose the public config and protocol semantics. The maintainable direction is a shared effective model-policy resolver rather than filtering only one picker or command path.
