## Summary

Keep open. Current main still has only a global model visibility allowlist from `agents.defaults.models`; per-agent `models` is a metadata/runtime override surface, `models.list` is not agent-scoped, and session model validation is not wired to an effective per-agent allowlist.

## What This Changes

Request a per-agent model allowlist so individual agents can restrict visible and selectable models, especially for cost-sensitive agents.

## Best Possible Solution

Define a typed effective per-agent model policy with explicit inheritance against `agents.defaults.models`, then enforce it consistently in model listing, selection, session patching, UI/TUI pickers, runtime paths, docs, and regression tests.

## Reproduction Assessment

Not applicable as a runtime bug, but the gap is source-reproducible: current main only parses `agents.defaults.models`, `models.list` accepts no agent parameter, and session model validation has no agent allowlist input.

## Solution Assessment

Unclear until maintainers settle the public API semantics. The safest direction is an explicit effective policy that preserves the existing `agents.list[].models` metadata/runtime contract.
