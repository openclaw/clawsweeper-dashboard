## Summary

Keep open. Current main and the latest release still implement model visibility through the global `agents.defaults.models` allowlist; per-agent `models` remains metadata/runtime override surface, `models.list` has no agent-scoping parameter, and the related implementation at https://github.com/openclaw/openclaw/pull/80394 is still open rather than merged.

## What This Changes

The issue requests a per-agent model allowlist so specific agents cannot expose or select expensive or inappropriate provider-backed models.

## Best Possible Solution

Adopt one documented per-agent model visibility and enforcement policy that composes with `agents.defaults.models`, enforce it across `models.list`, `/model`, `sessions.patch`, UI/TUI selectors, docs, and tests, and leave broader account/provider policy to https://github.com/openclaw/openclaw/issues/65557.

## Reproduction Assessment

Yes for the source-level feature gap: current main reads configured visibility from `agents.defaults.models`, `models.list` rejects agent-scoping params, and `sessions.patch` validates model changes without an agent-specific visibility input. This is a feature request rather than a failing runtime bug.

## Solution Assessment

Unclear until maintainers choose the public config and protocol semantics. The maintainable direction is a shared effective policy resolver threaded through Gateway, model selection, session patching, commands, and UI/TUI callers.
