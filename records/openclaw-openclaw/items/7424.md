## What This Changes

Add a per-agent model allowlist so one agent can restrict which models are visible or selectable independently of the global model catalog.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/80394 or a narrower successor after maintainers settle the public config semantics and one effective allowed-set policy covers listing, slash commands, session patching, UI/TUI selectors, scheduling, and runtime execution.

## Reproduction Assessment

Yes. Current-main source deterministically shows per-agent `models` is metadata/runtime override data while effective visibility and validation still derive from the global `agents.defaults.models` policy.

## Solution Assessment

The per-agent configuration boundary is the right direction, but the best fix needs maintainer-selected composition and enforcement semantics; one canonical effective-policy helper is safer than parallel checks in individual selectors.
