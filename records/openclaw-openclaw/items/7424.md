## Summary

Keep open. Current main still lacks a complete enforced per-agent model visibility allowlist, and the matching open implementation PR is still blocked on public config/API compatibility review rather than being ready to close this issue.

## What This Changes

The issue requests a per-agent model allowlist so individual agents can be restricted from selecting costly or inappropriate models.

## Best Possible Solution

Review and reshape https://github.com/openclaw/openclaw/pull/80394 or a successor so the final contract preserves runtime-only per-agent model metadata while adding an explicit, documented, consistently enforced per-agent visibility policy.

## Reproduction Assessment

Yes for the feature gap by source inspection: current main still derives visibility from `agents.defaults.models`, has no `models.list` agent scope, and does not pass the session agent id into model patch validation. This is not a runtime bug reproduction; it is a source-reproducible missing capability.

## Solution Assessment

Unclear until maintainers settle the public API semantics. Threading agent scope through model visibility is the right general direction, but reusing the existing per-agent `models` runtime metadata map as the allowlist needs an explicit compatibility decision.
