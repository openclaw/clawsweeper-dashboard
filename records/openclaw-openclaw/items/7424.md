## Summary

Keep open. Current main and the latest release still expose only the global model visibility allowlist, while the active implementation remains open and needs maintainer decisions around the public config and protocol shape.

## What This Changes

The issue requests a per-agent model allowlist so one agent can be restricted from selecting expensive or inappropriate models.

## Best Possible Solution

Land one documented agent-scoped model visibility policy, ideally through the existing open implementation after maintainer review, and enforce it consistently across config schema, Gateway, slash commands, session patching, UI/TUI, docs, and tests.

## Reproduction Assessment

Yes for the source-level feature gap: current main still reads configured visibility from agents.defaults.models, models.list has no agent-scoping parameter, and sessions.patch validates model changes without an agent-specific visibility policy. This is a feature request rather than a failing runtime bug.

## Solution Assessment

Unclear until maintainers choose the public config and protocol semantics. The maintainable direction is a shared effective policy resolver threaded through Gateway, model selection, session patching, commands, UI/TUI callers, and docs.
