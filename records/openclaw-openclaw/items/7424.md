## Summary

Keep open. Current main still lacks a per-agent model allowlist: config and schema only expose per-agent primary/fallback defaults, model listing and session patch validation still derive allowed models from global `agents.defaults.models`, and the prior discussion keeps this as narrower than #65557 while #9010 is already closed as a duplicate.

## What This Changes

Add a per-agent model allowlist so individual agents can hide and reject disallowed models independently of the global catalog.

## Best Possible Solution

Add an additive, documented per-agent model catalog/allowlist with explicit composition rules against `agents.defaults.models`, then enforce the effective allowed set consistently across model listing, `/model` selection, `sessions.patch`, and UI/TUI selectors with regression coverage.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is source and docs inspection showing current main only supports global `agents.defaults.models` plus per-agent primary/fallback defaults.

## Solution Assessment

Unclear because there is no proposed patch. The maintainable path needs a maintainer-approved config/protocol addition with explicit inheritance and enforcement semantics, not ad hoc overloading of the current strict per-agent `model` object.
