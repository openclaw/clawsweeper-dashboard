## Summary

Keep open. Current main still lacks a typed per-agent model allowlist: it has a global `agents.defaults.models` catalog/allowlist plus per-agent primary/fallback defaults, and the prior discussion keeps this narrower request separate from closed duplicate #9010 and broader account/provider policy #65557.

## What This Changes

The issue asks for a per-agent model allowlist so one agent can be restricted to a cheaper or safer model set without running a separate gateway.

## Best Possible Solution

Add a documented, typed per-agent model allowlist/catalog with explicit composition rules against `agents.defaults.models`, then enforce the effective allowed set consistently across model listing and model override paths.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is source and docs inspection showing current main lacks a typed per-agent model allowlist while documenting and enforcing only the global `agents.defaults.models` allowlist.

## Solution Assessment

Unclear because there is no proposed patch and this adds public config/protocol semantics. The maintainable path needs a maintainer-approved additive surface with explicit inheritance and enforcement rules.
