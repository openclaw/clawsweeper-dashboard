## Summary

Keep open. Current main has adjacent per-agent model metadata, but the effective allowlist used by model selection, Gateway model listing, and session patch validation is still global, and the related implementation PRs are closed unmerged.

## What This Changes

The issue requests a per-agent model allowlist so one agent can be restricted to cheaper or appropriate models without running a separate gateway.

## Best Possible Solution

Define one documented effective per-agent model policy, then enforce it through shared resolver code for listing, selection, session patching, UI/TUI pickers, and tests.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main gives a high-confidence verification path showing model allowlist enforcement still comes from global `agents.defaults.models`, not an agent-scoped policy.

## Solution Assessment

Unclear until maintainers choose the public config and protocol semantics. The maintainable direction is a shared effective model-policy resolver rather than filtering only one picker or command path.
