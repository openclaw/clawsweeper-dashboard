## Summary

Keep open. Current main still exposes only the global `agents.defaults.models` allowlist plus per-agent primary/fallback defaults; there is no typed per-agent model allowlist or agent-scoped Gateway/UI model catalog, and the related closed duplicate and broader account/provider policy issue do not supersede this narrower request.

## What This Changes

Add a per-agent model allowlist so each configured agent can expose and select only approved models instead of relying on the global allowlist.

## Best Possible Solution

Add a documented, additive per-agent model catalog/allowlist with explicit composition rules against `agents.defaults.models`, then enforce the effective set across model listing, `/model` selection, session patching, UI/TUI pickers, and regression tests.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is source and docs inspection showing current main lacks a typed per-agent model allowlist while documenting and enforcing only the global `agents.defaults.models` allowlist.

## Solution Assessment
