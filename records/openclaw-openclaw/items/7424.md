## Summary

Keep open. Current main still has a global `agents.defaults.models` allowlist plus per-agent primary/fallback defaults, but no typed per-agent model allowlist or agent-scoped Gateway/UI catalog enforcement.

## What This Changes

Request a per-agent model allowlist so individual agents can hide and reject expensive or inappropriate models.

## Best Possible Solution

Define a documented additive per-agent model catalog/allowlist, with explicit composition rules against `agents.defaults.models`, and enforce the effective set anywhere users can list or select models.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is source and docs inspection showing current main documents and enforces only the global `agents.defaults.models` allowlist.

## Solution Assessment

Unclear until maintainers choose the public API and inheritance semantics. The requested direction is useful, but the narrow maintainable path needs a typed config/protocol/UI design rather than a local-only guard.
