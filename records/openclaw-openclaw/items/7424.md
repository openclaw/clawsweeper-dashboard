## Summary

Keep open. Current main and v2026.5.22 still use the global `agents.defaults.models` visibility source, `models.list` has no agent-scoping parameter, and the active implementation at https://github.com/openclaw/openclaw/pull/80394 remains open rather than merged.

## What This Changes

Request to add a per-agent model allowlist so one agent can be prevented from selecting or showing expensive or inappropriate models.

## Best Possible Solution

Use https://github.com/openclaw/openclaw/pull/80394 as the implementation lane, choose one documented agent-scoped visibility policy that composes with `agents.defaults.models`, and enforce it consistently across Gateway, model selection, session patching, commands, UI/TUI selectors, docs, and tests while leaving broader account/provider policy to https://github.com/openclaw/openclaw/issues/65557.

## Reproduction Assessment

Yes for the source-level feature gap: current main reads configured visibility from `agents.defaults.models`, `models.list` rejects agent-scoping params, and `sessions.patch` validates model changes without an agent-specific visibility input. This is a feature request rather than a failing runtime bug.

## Solution Assessment

Unclear until maintainers choose the public config and protocol semantics. The maintainable direction is a shared effective policy resolver threaded through Gateway, model selection, session patching, commands, and UI/TUI callers, and the existing open PR is the right place to finish that review.
