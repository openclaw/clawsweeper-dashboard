## Summary

Keep open. Current main still lacks an enforced per-agent model allowlist; the existing per-agent `models` field is documented and used as model runtime metadata, and the open implementation PR needs maintainer compatibility/API review before this issue can close.

## What This Changes

Add a per-agent model allowlist so family or specialized agents cannot select costly or unsuitable models from the global catalog.

## Best Possible Solution

Review and reshape https://github.com/openclaw/openclaw/pull/80394 or a successor so OpenClaw has a documented per-agent visible model policy that preserves existing runtime metadata behavior and enforces one effective policy across model listing, `/model`, session patching, and UI/TUI selectors.

## Reproduction Assessment

Yes for the feature gap by source inspection: current main uses global `agents.defaults.models` for visibility, `models.list` has no `agentId`, and session patch validation does not pass agent scope. This is not a runtime bug reproduction.

## Solution Assessment

Unclear until maintainers settle the public API semantics. Threading an effective per-agent visibility policy through model listing and selection is the right direction, but the existing `agents.list[].models` runtime metadata contract must be preserved or explicitly migrated.
