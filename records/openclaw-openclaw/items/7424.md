## Summary

Keep open. Current main still has only the global `agents.defaults.models` allowlist plus per-agent primary/fallback defaults, while the maintainer discussion keeps this as the narrower per-agent request rather than the broader account/provider policy tracked in #65557.

## What This Changes

Requested an agent-scoped model allowlist so administrators can prevent specific agents from selecting expensive or inappropriate models.

## Best Possible Solution

Add an additive, documented per-agent model catalog/allowlist with explicit composition rules against `agents.defaults.models`, then enforce the effective allowed set consistently across model listing, `/model` selection, `sessions.patch`, and UI/TUI selectors.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is source and docs inspection showing current main only supports global `agents.defaults.models` plus per-agent primary/fallback defaults.

## Solution Assessment

Unclear because there is no proposed patch. The maintainable path needs a maintainer-approved config/protocol addition with explicit inheritance and enforcement semantics, not ad hoc overloading of the current strict per-agent `model` object.
