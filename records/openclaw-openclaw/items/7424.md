## Summary

Keep open. Current main still lacks an enforced per-agent model allowlist; https://github.com/openclaw/openclaw/issues/9010 is already closed as a duplicate, https://github.com/openclaw/openclaw/issues/65557 is broader account/provider policy work, and the open implementation PR at https://github.com/openclaw/openclaw/pull/80394 still needs maintainer API/product review before this issue can close.

## What This Changes

Request per-agent model allowlists so an agent can only show or select an approved subset of configured models.

## Best Possible Solution

Define a documented per-agent model visibility policy that preserves existing runtime metadata semantics, then enforce the effective allowed set consistently in `models.list`, `/model` selection, session patching, UI/TUI pickers, runtime paths, docs, and regression tests.

## Reproduction Assessment

Not applicable as a runtime bug, but the feature gap is source-reproducible: current main parses visibility from `agents.defaults.models`, `models.list` has no `agentId`, and session model validation is not scoped to an effective per-agent allowlist.

## Solution Assessment

Unclear until maintainers settle the public API semantics. Threading `agentId` is the right general direction, but the existing open PR shows why overloading the current `agents.list[].models` runtime metadata map needs explicit maintainer approval.
