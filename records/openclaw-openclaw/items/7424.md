## What This Changes

Request a per-agent model allowlist so one agent can be restricted to a configured subset of provider-backed models without duplicating gateways.

## Best Possible Solution

Use https://github.com/openclaw/openclaw/pull/80394 as the implementation lane, settle the additive config/protocol semantics, and enforce one effective per-agent policy across Gateway, model selection, session patching, commands, UI/TUI, docs, and tests.

## Reproduction Assessment

Yes, by source inspection: current main reads visibility from agents.defaults.models, models.list has no client agent scope, and per-agent models are runtime metadata rather than a visibility allowlist.

## Solution Assessment

Unclear until maintainers choose the public config and protocol shape. The maintainable direction is a shared effective policy resolver used by Gateway, model selection, sessions.patch, commands, UI/TUI callers, docs, and tests.
