## Summary

Keep open. Current main still has a global model catalog/allowlist plus per-agent primary and fallback defaults, but no typed per-agent model allowlist, and the latest issue discussion already kept this as the narrower canonical request rather than folding it into broader account/provider policy work.

## What This Changes

Add a per-agent model allowlist so specific agents can only show and select approved models.

## Best Possible Solution

Add an additive, documented per-agent model catalog/allowlist with explicit composition rules against agents.defaults.models, then enforce the effective allowed set in model listing, /model selection, sessions.patch, UI/TUI pickers, and regression tests while leaving broader account/provider administration to #65557.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is source, docs, and test inspection showing only a global agents.defaults.models allowlist plus per-agent primary/fallback defaults.

## Solution Assessment

Unclear because there is no proposed patch. The best direction is a maintainer-approved config/protocol addition with explicit inheritance and enforcement semantics, not overloading the current strict primary/fallback model object ad hoc.
