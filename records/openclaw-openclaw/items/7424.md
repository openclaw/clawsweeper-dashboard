## Summary

Keep open. Current main still exposes a global `agents.defaults.models` visibility policy rather than a per-agent model allowlist, and the active implementation in https://github.com/openclaw/openclaw/pull/80394 is still open and conflicting, so this issue remains the canonical tracker rather than a closeable duplicate.

## What This Changes

Request a per-agent model allowlist so individual agents can be restricted from selecting expensive or inappropriate provider models.

## Best Possible Solution

Adopt one documented per-agent model visibility and enforcement policy that composes with `agents.defaults.models`, then enforce it across `models.list`, `/model`, `sessions.patch`, UI/TUI selectors, docs, and regression tests while keeping broader account policy in https://github.com/openclaw/openclaw/issues/65557.

## Reproduction Assessment

Yes for the source-level feature gap: current main reads configured visibility from `agents.defaults.models`, `models.list` rejects agent-scoping params, and `sessions.patch` validates model changes without agent-specific visibility input. This is a feature request rather than a failing runtime bug.

## Solution Assessment

Unclear until maintainers choose the public API semantics. A shared effective policy resolver threaded through Gateway, model selection, session patching, commands, and UI/TUI callers is the maintainable direction.
