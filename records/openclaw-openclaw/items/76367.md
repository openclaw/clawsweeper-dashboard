## Summary

Keep open: current main still has the source-visible doctor/status false-warning path, and the linked unmerged PR #76393 explicitly tracks the narrow fix for this issue.

## What This Changes

The issue reports that `openclaw doctor --non-interactive` and status can say no active memory plugin is registered even when bundled `memory-core` is selected, loaded, and memory search works.

## Best Possible Solution

Review and land #76393, or an equivalent narrow fix, so standalone doctor/status perform a scoped configured memory-slot runtime load before claiming no active memory plugin exists.

## Reproduction Assessment

Yes. Source-reproducible: with `plugins.slots.memory = "memory-core"` and no active runtime registry in a standalone CLI process, `ensureMemoryRuntime` only checks for an already-loaded registry, so backend resolution can return null and doctor emits the exact warning.

## Solution Assessment

Yes. The linked PR direction is the narrow maintainable fix: align standalone doctor/status runtime bootstrap with the selected memory slot, without adding config or changing memory search policy.
