## Summary

Keep open: current main still has the source-visible doctor/status false-warning path, and open PR #76393 explicitly references this issue as the fix vehicle, so the issue should not close before that PR lands or is replaced.

## What This Changes

The issue asks doctor/status to recognize the selected and working bundled memory-core plugin instead of reporting that no active memory plugin is registered.

## Best Possible Solution

Review and land or replace #76393 with a narrow fix that loads only the configured memory-slot runtime for cold doctor/status processes, with regression coverage for memory-core and status JSON.

## Reproduction Assessment

Yes. Source inspection on current main shows the exact warning is emitted when the active memory backend lookup returns null, and that lookup can remain null in a standalone CLI process because it only checks an already-loaded runtime registry.

## Solution Assessment

Yes. The narrow maintainable direction is to align doctor/status runtime bootstrap with the selected memory-slot plugin instead of adding config, hardcoding memory-core in doctor, or treating working memory search as absent.
