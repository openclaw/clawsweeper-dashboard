  - confidence: 0.96

  - body: The OpenAI plugin has the same regression: the current code caps inline image JSON through `readProviderJsonResponse`, but this line switches to raw `response.json()`. A large or hostile provider response can bypass the image JSON size limit, so this should use the bounded reader with the existing inline-image cap.
  - confidence: 0.95
- **[P1] Remove the root native dependency:** `package.json:2031`
  - body: This PR still adds `node-llama-cpp` to root devDependencies even though the stated QMD and image URL fixes do not need it. The llama.cpp plugin docs say the external provider plugin owns this native runtime, and the package has postinstall/native optional dependency surface that should not ride along with this fix.
  - confidence: 0.99
- **[P1] Preserve the QMD package launcher:** `src/plugin-sdk/windows-spawn.ts:276-279`
  - body: This special-case returns `dist/cli/qmd.js` instead of the published `bin/qmd` launcher. The launcher seeds native llama/ggml environment and Node/Bun/source-mode behavior before invoking the JS entrypoint, so bypassing it can break real installs even if a direct unit fixture passes.
  - confidence: 0.96
- **[P2] Keep memory_search on the tool deadline:** `extensions/memory-core/src/tools.ts:435-439`
  - body: This changes `memory_search` from the established 15-second tool deadline to the configured QMD backend timeout, which can hold agent tool calls open much longer and is unrelated to Windows shim resolution or image URL downloads.   - confidence: 0.93
