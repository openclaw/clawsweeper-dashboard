- **[P1] Keep shared image JSON responses byte-capped:** `src/image-generation/openai-compatible-image-provider.ts:289`
  - body: This path currently uses a bounded provider JSON reader on main, but the PR calls `response.json()` before parsing URL entries. A provider-controlled image response can again be buffered without the configured cap, so restore the bounded read before adding URL fallback handling.
  - confidence: 0.96
- **[P1] Keep OpenAI image JSON responses byte-capped:** `extensions/openai/image-generation-provider.ts:1051`
  - body: The OpenAI plugin has the same regression: the current code caps inline image JSON through `readProviderJsonResponse`, but the PR switches to raw `response.json()`. Use the existing bounded reader with the inline-image cap here too.
  - confidence: 0.95
- **[P1] Remove the root native dependency:** `package.json:2031`
  - body: The PR still adds `node-llama-cpp` to root devDependencies even though the QMD and image URL fixes do not need it. That package has postinstall and native optional dependency surface, and llama.cpp runtime ownership belongs to the plugin/package that uses it.
  - confidence: 0.99
- **[P1] Preserve the QMD package launcher:** `src/plugin-sdk/windows-spawn.ts:276-279`
  - body: This special-case returns `dist/cli/qmd.js` instead of the published `bin/qmd` launcher. The launcher seeds native log/runtime environment and Node/Bun/source-mode behavior before invoking the JS entrypoint, so bypassing it can break real installs even if the unit fixture passes.
  - confidence: 0.95
- **[P2] Keep memory_search on the tool deadline:** `extensions/memory-core/src/tools.ts:435-439`
  - body: This changes `memory_search` from the established 15-second tool deadline to the configured QMD backend timeout, which can hold agent tool calls open much longer and is outside the stated Windows shim and image URL fixes. Restore the fixed tool deadline unless maintainers approve the policy change.
  - confidence: 0.91
- **[P2] Avoid exporting new SDK helpers by default:** `src/plugin-sdk/image-generation.ts:19-26`
  - body: These new exports add plugin-facing API surface for the async parser and URL downloader types. Keep the URL response parsing internal to the shared provider path, or get explicit SDK approval and focused compatibility coverage before making these helpers public.
  - confidence: 0.88
