## Summary

Keep this PR open: it fixes real source-backed Windows QMD and OpenAI-compatible image URL gaps, but the new provider-controlled URL image download path still buffers the whole response without a byte cap before merge.

## What This Changes

The branch adds Windows `.cmd` shim support for absolute QMD entrypoints, switches QMD package fallback to `@tobilu/qmd`, and adds async OpenAI-compatible `data[].url` image response handling through guarded provider HTTP.

## Best Possible Solution

Land the QMD and URL-response compatibility direction after URL-backed generated-image downloads use a bounded response reader with oversized-response coverage.

## Solution Assessment

No, not yet: the QMD resolver and URL-response support are the right maintainable direction, but URL downloads need bounded response reading before this is the best mergeable fix.
