## Summary

Keep open: the PR is a useful focused fix, but the latest head still buffers provider-controlled image URL responses with an unbounded `arrayBuffer()` after the guarded fetch, which is a concrete availability/security blocker before merge.

## What This Changes

The PR adds Windows `.cmd` shim handling for absolute QMD entrypoints, switches QMD fallback package resolution to `@tobilu/qmd`, and adds guarded OpenAI-compatible URL image parsing/downloading with tests.

## Best Possible Solution

Keep the QMD and URL-response direction, but land only after URL-backed generated-image downloads use a bounded response reader or equivalent max-byte policy with oversized-response coverage.
