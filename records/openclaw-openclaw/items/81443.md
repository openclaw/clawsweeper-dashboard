## Summary

The PR is useful and should stay open, but it is not merge-ready because the new provider-controlled image URL path still buffers the whole response body after the guarded fetch.

## What This Changes

The branch adds Windows QMD `.cmd` shim support for quoted absolute entrypoints, switches QMD fallback resolution to `@tobilu/qmd`, and adds OpenAI-compatible `data[].url` image parsing/downloading.

## Best Possible Solution

Land the QMD and URL-response support only after URL-backed generated-image downloads use the existing bounded response reader or equivalent max-byte policy with oversized-response coverage.

## Reproduction Assessment

Yes, at source level: current main only parses `b64_json` image entries and only resolves `%~dp0`-relative quoted `.cmd` shim targets, while QMD docs install `@tobilu/qmd`. I did not rerun Windows or a live provider in this read-only review.

## Solution Assessment

No, not yet: the QMD resolver and URL-response support are the right maintainable direction, but URL downloads need bounded response reading before the patch is merge-ready.
