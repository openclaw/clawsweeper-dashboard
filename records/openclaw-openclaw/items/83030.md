Keep open: current main can route an explicit OpenRouter image model ref, but it does not advertise ReCraft V4.1 models, expose SVG as a public image output format, or model ReCraft-specific capabilities. The remaining work is a feature/API decision rather than conservative cleanup.

Request explicit ReCraft V4.1 Standard, Utility, and Vector image-generation support through the existing OpenRouter image provider.

## Best Possible Solution

Decide the public image-generation contract first, then either document explicit OpenRouter ReCraft refs or add model-specific OpenRouter catalog/capability handling with SVG-safe output behavior.
