## Summary

Keep open: this is a current, concrete feature request against the bundled OpenRouter image-generation plugin, and neither current main nor the latest release expose ReCraft in the OpenRouter image catalog. It should not be swept to ClawHub by default because OpenRouter is already an included provider plugin, but the SVG/output contract and ReCraft-specific request limits need maintainer product review before implementation.

## What This Changes

Request support for the ReCraft V4.1 Standard, Utility, and Vector image model family through OpenRouter image generation.

## Best Possible Solution

Add ReCraft support in the OpenRouter provider plugin only after deciding the public SVG/output-format contract, with model-specific capabilities, docs, and focused provider/tool tests.

## Reproduction Assessment

Not applicable: this is a feature request, not a bug. Source inspection shows current main and v2026.5.12 only expose three OpenRouter image shortcuts, while OpenRouter's per-model pages for the requested ReCraft slugs are live.

## Solution Assessment

No proposed patch is under review. A blind catalog append is probably not enough because ReCraft has one-input-image limits, extra image_config parameters, and SVG/vector behavior that should be modeled deliberately.
