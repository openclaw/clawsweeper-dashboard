## Summary

Keep open. Current main still lacks the requested working-context capsule, and this draft PR remains a plausible implementation path, but the patch is not merge-ready because the write path uses a colliding temp filename, stores local context with default file permissions, and has no real behavior proof.

## What This Changes

Adds `src/agents/session-working-context.ts` and tests for a versioned sibling `.context.json` session working-context capsule, without runtime, status, compaction, or subagent wiring.

## Best Possible Solution

Keep the draft open for maintainer schema and storage-boundary guidance, then land a corrected private and collision-safe persistence primitive as part of or just before the first runtime/status/compaction integration.

## Reproduction Assessment

Yes, for the PR review findings: static inspection of the PR head shows the non-unique temp filename and missing private file mode. Current-main source search also confirms the requested capsule is not already implemented.

## Solution Assessment

No, not as-is. The storage primitive is a reasonable MVP direction, but it should use unique private writes and wait for maintainer agreement on schema, location, retention, and integration lifecycle.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Use a unique temp path for each context write:** `src/agents/session-working-context.ts:171`
  - body: `writeWorkingContext` builds the temp filename from the capsule path, process id, and `Date.now()`. Two writes to the same session in the same process and millisecond can collide, so one write/rename/cleanup can fail or clobber the other; add a per-call random component or use the existing atomic writer.
  - confidence: 0.87
- **[P2] Write context capsules with private permissions:** `src/agents/session-working-context.ts:174`
  - body: The capsule can persist local cwd/repo paths, branch and push targets, sandbox state, and free-form notes, but this write uses default file permissions. Match transcript/session-state behavior by writing the temp file with `0o600` before runtime code starts populating it.
  - confidence: 0.8
