## Summary

Keep open. Current main still lacks the proposed working-context capsule, and this draft PR remains a plausible feature direction but has blocking write-safety/privacy defects plus unresolved schema, storage-location, and lifecycle decisions.

## What This Changes

Adds `src/agents/session-working-context.ts` and tests for a versioned sibling `.context.json` capsule with resolve, normalize, read, write, and clear helpers.

## Best Possible Solution

Keep this draft open until maintainers approve the schema and storage boundary, then land a corrected private/unique-write primitive as part of or immediately before the first runtime/status/compaction integration.

## Reproduction Assessment

Yes, for the PR review findings: static inspection of the PR diff shows the temp filename collision and missing private file mode. Current-main source search also confirms the requested capsule is not already implemented.

## Solution Assessment

No, not as-is. The storage primitive is a reasonable MVP direction, but it should use unique private writes and should wait for maintainer agreement on schema, location, retention, and integration lifecycle.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Use a unique temp path for each context write:** `src/agents/session-working-context.ts:171`
  - body: `writeWorkingContext` derives the temp file from the capsule path, process id, and `Date.now()`. Two writes to the same session in the same process and millisecond can collide, so one rename or cleanup can clobber the other write; add a per-call random component or reuse the existing atomic writer.
  - confidence: 0.87
- **[P2] Write context capsules with private permissions:** `src/agents/session-working-context.ts:174`
  - body: The capsule can persist local cwd/repo paths, branch and push targets, sandbox state, and notes, but this write uses default file permissions. Match the existing session transcript/store behavior by writing the temp file with `0o600` before runtime code starts populating it.
  - confidence: 0.8
