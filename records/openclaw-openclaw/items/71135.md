Full review comments:

- **[P1] Store working context in the canonical state owner:** `src/agents/session-working-context.ts:63`
  - body: This resolver makes `<session>.context.json` the new persisted location for cwd, repo root, branch, temp clones, remotes, sandbox state, and notes. Current repo policy says OpenClaw-owned runtime state belongs in SQLite/per-agent state unless there is an explicit file-artifact exception, so landing this sidecar creates an unmanaged state store without migration, doctor, cleanup, or redaction ownership.
  - confidence: 0.9
- **[P2] Write context capsules with private permissions:** `src/agents/session-working-context.ts:174`
  - body: The capsule can persist private working paths, remotes, sandbox state, and free-form notes, but this `fs.writeFile` call uses default creation permissions. On permissive umasks the renamed capsule can be group/world-readable; use the repo private atomic writer or pass an explicit private mode before any runtime code populates it.
  - confidence: 0.87
- **[P2] Use a unique temp path for each write:** `src/agents/session-working-context.ts:171`
  - body: `writeWorkingContext` builds the temp filename from the target path, process id, and `Date.now()`. Two same-process writes to the same session in the same millisecond can pick the same temp file, letting one writer rename or unlink the other's payload; use the existing atomic replace helper or add a per-call random suffix.
  - confidence: 0.86
