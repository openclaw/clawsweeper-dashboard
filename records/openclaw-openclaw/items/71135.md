## Summary

Keep this PR open. Current main still lacks the requested session working-context capsule, and the PR is a draft storage-only MVP tied to open product/API work in #67511; the proposed patch also has blocking correctness and privacy issues in its atomic write path before it should merge.

## What This Changes

The PR adds a new `src/agents/session-working-context.ts` utility and colocated tests for resolving, normalizing, reading, writing, and clearing a versioned sibling `.context.json` capsule next to a session transcript.

## Best Possible Solution

Keep #67511 as the product/API tracker, have maintainers decide the durable schema/location/privacy boundary, then land a corrected primitive together with or immediately before the first runtime integration that updates and displays working context across status, compaction/new-session, and spawn flows.

## Reproduction Assessment

Yes. Static review of the PR head reproduces the temp-file collision and missing private-permission issues, and targeted current-main source inspection reproduces the absence of a working-context capsule or status/runtime integration.

## Solution Assessment

No, not as proposed. A small storage primitive could be a reasonable first step, but it should use a unique private temp file and should not become the durable contract until maintainers accept the schema, location, retention, and integration plan.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Use a unique temp file for each write:** `src/agents/session-working-context.ts:171`
  - body: `writeWorkingContext` builds the temp filename from only `process.pid` and `Date.now()`. Two same-process writes to the same session within the same millisecond can target the same temp path, so one write/rename/unlink can race the other and lose or corrupt the capsule. Add `randomUUID()` and use exclusive creation for each write.
  - confidence: 0.87
- **[P2] Write context capsules with private permissions:** `src/agents/session-working-context.ts:174`
  - body: The capsule can store local repo paths, temp clone paths, push targets, sandbox state, and free-form notes, but this write uses default file permissions. Existing session files are written `0o600`, so the sibling capsule should also be created privately before runtime code starts persisting it.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The diff adds no dependencies, workflows, scripts, or downloaded code, but it introduces a durable local capsule that may be world-readable under default file creation permissions.
