## Summary

Keep this draft PR open. Current main still lacks the working-context capsule or any `/status`, compaction, or spawn integration, and the PR is relevant source work for #67511, but the proposed write path is not merge-ready because it can reuse a temp filename under concurrent writes and does not pin private permissions for the durable local capsule.

## What This Changes

The PR adds a new `src/agents/session-working-context.ts` utility and colocated Vitest tests for resolving, normalizing, reading, writing, and clearing a versioned sibling `.context.json` working-context capsule.

## Best Possible Solution

Decide the durable schema, retention, permissions, and location first; then land either this corrected primitive or the same storage contract inside the first integration PR that actually updates and surfaces working context across session status, compaction/new-session transitions, and spawn flows.

## Reproduction Assessment

Yes. Static review of the PR head reproduces the temp-file collision and permission gaps, while focused current-main inspection shows the working-context capsule and integration points are still absent.

## Solution Assessment

No, not as proposed. A small storage primitive is a reasonable first step, but it should use a unique private temp file and should not become the durable contract until maintainers accept the schema/privacy boundary and integration plan.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Use a unique temp path for each context write:** `src/agents/session-working-context.ts:171`
  - body: `writeWorkingContext` builds the temp filename from only `process.pid` and `Date.now()`. Two same-process writes to the same session in the same millisecond can target the same temp file, so one rename/unlink can race the other and lose or corrupt the persisted capsule. Add `randomUUID()` or reuse the existing atomic JSON pattern so each call gets a unique temp path.
  - confidence: 0.87
- **[P2] Write context capsules with private file permissions:** `src/agents/session-working-context.ts:174`
  - body: The new capsule can persist cwd, repo roots, temp clone paths, push targets, and free-form notes, but this write uses the default file mode. On a normal umask that can create a world-readable sibling file, unlike existing session transcripts and session stores that are written as `0o600`. Pass `mode: 0o600` and keep the final capsule private.
  - confidence: 0.8
