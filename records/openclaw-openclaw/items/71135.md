## Summary

Keep this draft PR open, but it is not merge-ready. Current main still lacks the working-context capsule from #67511, while the PR's storage primitive has concrete temp-file collision and private-permission issues plus unresolved schema/privacy/location decisions.

## What This Changes

The PR adds a new `src/agents/session-working-context.ts` utility and tests for resolving, normalizing, reading, writing, and clearing a versioned sibling `.context.json` working-context capsule.

## Best Possible Solution

Decide the durable schema, retention, permissions, and storage location first; then land a corrected primitive or include the same contract in the first integration PR for status, compaction/new-session, and spawn flows.

## Reproduction Assessment

Yes. Static review of the PR head reproduces the temp-path collision and missing private file mode, and current-main source inspection shows the requested capsule is still absent.

## Solution Assessment

No, not as proposed. A small storage primitive is a reasonable MVP, but it should use unique private writes and should not become the durable contract until maintainers accept the schema, privacy boundary, location, and integration plan.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Use a unique temp path for each context write:** `src/agents/session-working-context.ts:171`
  - body: `writeWorkingContext` builds the temp filename from only the target path, process id, and `Date.now()`. Two same-process writes to the same session in the same millisecond can target the same temp file, so one rename/unlink can race the other and lose or corrupt the persisted capsule. Add `randomUUID()` or reuse the existing atomic JSON writer so every call gets a unique temp path.
  - confidence: 0.87
- **[P2] Write context capsules with private file permissions:** `src/agents/session-working-context.ts:174`
  - body: The new capsule can persist cwd, repo roots, temp clone paths, push targets, sandbox state, and free-form notes, but this write uses the default file mode. Existing session transcripts and the session store request `0o600`; pass `mode: 0o600` for the temp write so the final capsule stays private.
  - confidence: 0.8
