## Label Justifications

- `P2`: This is a meaningful compaction extensibility request for long-session users, but it is a feature/design item rather than an urgent regression.
- `impact:session-state`: The request changes how compaction summaries are produced and therefore affects transcript continuity and session state quality.

## Summary

Keep open: current main still only supports compaction providers registered through the plugin API, and the compaction/context owner reopened the issue saying this is not a ClawHub cleanup close and needs design discussion.

## What This Changes

Request to let `agents.defaults.compaction.provider` route to configured MCP server tools as compaction engines instead of only plugin-registered compaction providers.

## Best Possible Solution

Have compaction/context owners decide whether this belongs as a core MCP adapter, a plugin SDK extension, or a documented plugin/ClawHub pattern, then implement that chosen design with config, security, docs, and regression tests together.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing behavior report. Source and docs inspection confirm current main and `v2026.6.9` support registered plugin compaction providers, not the proposed direct MCP provider config.

## Solution Assessment

Unclear: the direct MCP config may be useful, but VISION prefers plugin and ClawHub paths unless there is a clear missing API, and the compaction/context owner has explicitly asked for design discussion.
