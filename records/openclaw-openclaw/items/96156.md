## Label Justifications

- `P2`: This is a bounded but non-trivial compaction capability request affecting long-session workflows, not an urgent regression or outage.
- `impact:session-state`: The requested adapter would change how session transcripts are summarized and preserved during compaction.
- `impact:security`: The proposed MCP tool path would expose conversation context to external MCP servers, so transcript disclosure and trust boundaries matter.

## Summary

Keep open. Current main still only supports a string `agents.defaults.compaction.provider` that resolves to a plugin-registered compaction provider; the requested direct MCP server/tool adapter is absent, and the listed compaction/context owner explicitly reopened the issue for discussion.

## What This Changes

The issue requests allowing `agents.defaults.compaction.provider` to target MCP server/tool references so MCP summarizers can act as compaction engines without a plugin wrapper.

## Best Possible Solution

Hold a compaction/context-engine design discussion that either sponsors a narrow core MCP compaction adapter with explicit config and security semantics or documents the plugin/API extension needed to build it outside core.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source inspection confirms current main lacks the proposed `mcpServer`/`mcpTool` compaction shape and only delegates to registered plugin providers.

## Solution Assessment

Unclear; a direct MCP adapter is plausible, but VISION.md warns against duplicating plugin/ClawHub paths and the compaction owner has asked for discussion rather than an automatic scope close.
