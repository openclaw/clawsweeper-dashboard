## Label Justifications

- `P2`: This is a normal-priority feature request affecting long-session compaction workflows without evidence of a current regression breaking all users.
- `impact:session-state`: The request directly changes transcript compaction summaries and successor session context state.
- `impact:security`: The proposed adapter would pass session transcript content to configured MCP tools, which is a sensitive data and policy boundary.

## Summary

Keep open: current main still resolves compaction providers through the plugin-registered provider registry, while MCP tools remain a separate runtime tool surface, and the reopened discussion explicitly asks for maintainer design review rather than ClawHub closure.

## What This Changes

Allow `agents.defaults.compaction.provider` to reference an MCP server/tool as the compaction summarizer instead of only a plugin-registered provider.

## Best Possible Solution

Keep this open for maintainer design review; if accepted, define a narrow MCP compaction adapter contract with explicit config schema, transcript security policy, failure semantics, docs, and regression coverage.

## Reproduction Assessment

Yes for the missing capability: current source shows compaction provider lookup only targets the plugin provider registry, while MCP tool execution is a separate runtime path. This is source-reproducible rather than a failing bug repro.

## Solution Assessment

Unclear; the adapter is plausible, but the best design needs maintainer direction because it adds config surface and makes MCP tool execution part of the compaction/session-state boundary.
