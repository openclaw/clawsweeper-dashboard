## Summary

Keep open: current main still supports compaction providers only through the plugin-registered provider API, while MCP tools are a separate runtime/tool surface, and a contributor reopened the issue specifically saying this needs discussion rather than ClawHub closure.

## What This Changes

The issue asks OpenClaw to let `agents.defaults.compaction.provider` reference an MCP server tool so MCP summarizers can run compaction without a code-plugin wrapper.

## Best Possible Solution

Keep the issue open for maintainer design review; if accepted, define a narrow MCP compaction adapter contract with config schema/docs/tests and explicit transcript/tool-policy behavior, otherwise document the plugin-wrapper path as the supported boundary.

## Reproduction Assessment

Yes for the missing capability: current source shows `provider` resolves only to a plugin-registered compaction provider, while MCP server tools live on a separate session MCP tool runtime. This is source-reproducible rather than a failing bug repro.

## Solution Assessment

Unclear; the requested adapter is plausible, but the best solution needs maintainer direction because it adds config surface and makes MCP tool execution part of the compaction/session-state boundary.
