## What This Changes

The PR adds a `fastMode` parameter to `sessions_spawn`, persists it for native subagent and ACP child sessions, maps ACP runtime fast-mode config options, and updates docs, tests, prompt snapshots, and changelog text.

## Best Possible Solution

Keep the feature, but parse ACP fast-mode values through the shared fast-mode normalizer, reject invalid values, rebase, remove release-owned changelog edits, and add current-head real spawn proof before merge.

## Reproduction Assessment

Yes, from source: current ACP presentation and bridge tests use `fast_mode` values `on`/`off`, while PR head maps only literal `true` to enabled. No tests were run because this review is read-only.

## Solution Assessment

Not yet: `sessions_spawn` is a reasonable feature boundary and the ACP runtime application path is needed, but the config-option parser must use the existing fast-mode normalizer before this is the best fix.
