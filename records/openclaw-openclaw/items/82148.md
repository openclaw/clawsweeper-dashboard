## Best Possible Solution

Land a refreshed PR that keeps the existing spawn and ACP runtime-option seams, fixes ACP fast-mode value parsing with the shared `normalizeFastMode` contract plus on/off regression coverage, removes the changelog edit, and includes redacted live terminal/log proof of a real fast-mode spawn.

## Reproduction Assessment

Yes for the PR defect: source inspection shows `/acp set` persists inferred runtime options through `setSessionConfigOption`, and the PR's new `fast_mode` branch would map `on` to false. The broader PR is a feature addition, not a current-main bug, because `sessions_spawn.fastMode` does not exist on main.

## Solution Assessment

No as currently patched: the spawn/API seam is plausible, but ACP runtime-option parsing should reuse the existing fast-mode normalization contract before merge. The safer repair is to parse `on`/`off` and `true`/`false` consistently and add focused regression coverage.
