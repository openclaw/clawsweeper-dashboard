## Best Possible Solution

## Reproduction Assessment

Not yet: `sessions_spawn` is the right feature boundary, and Codex dynamic-tool schema handling is compatible, but ACP config-option parsing must use the shared fast-mode normalizer and the PR needs current-head real spawn proof before this is the best fix.
