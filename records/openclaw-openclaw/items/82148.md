## Summary

Keep this PR open: the native subagent part is plausible, but the ACP path still appears incorrect because the new `fastMode` value is only persisted on the local session entry while ACP turns apply runtime controls from ACP runtime options.

## What This Changes

The PR adds a `sessions_spawn.fastMode` tool parameter, forwards it into native subagent and ACP spawn calls, and updates docs, tests, prompt snapshots, and changelog.

## Best Possible Solution

Keep the native subagent session override, but route ACP fast-mode overrides through ACP runtime options or backendExtras before the first ACP child turn, or withhold the ACP-facing parameter until that contract is implemented.

## Reproduction Assessment

Yes for the review finding. Source inspection shows the PR-added ACP change stores `fastMode` on the local session entry, while ACP initialization and runtime controls consume `AcpSessionRuntimeOptions` instead; I did not execute the branch in this read-only review.

## Solution Assessment

No. Persisting `SessionEntry.fastMode` is the narrow native-subagent solution, but ACP needs the override applied through its runtime-control path or omitted from ACP docs/schema.
