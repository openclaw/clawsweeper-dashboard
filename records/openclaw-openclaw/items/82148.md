## Summary

Keep open: the native subagent side is aligned with existing session fast-mode resolution, but the ACP half is incomplete because the PR only stores `fastMode` on the OpenClaw session row while ACP turns are driven through ACP runtime options before `runTurn`. The PR also needs real behavior proof beyond focused tests and static source output.

## What This Changes

The branch adds a `fastMode` parameter to `sessions_spawn`, threads it into native subagent and ACP spawn code, and updates focused tests, docs, and the changelog.

## Best Possible Solution

Keep the PR open, map the override through both native session persistence and ACP runtime config before the first child turn, then add regression coverage and real spawn proof.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? For this feature PR, no user-reported bug reproduction is applicable, but the ACP gap is source-reproducible: ACP spawn dispatch uses `acpManager.runTurn` and runtime controls from `acp.runtimeOptions`, not `SessionEntry.fastMode`.

## Solution Assessment

Is this the best way to solve the issue? No; storing `fastMode` on the child session is enough for native subagents, but ACP spawns also need the override mapped into the ACP runtime option/config-option path before dispatch.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Propagate ACP fast mode to the runtime:** `src/agents/acp-spawn.ts:1327`

A caller setting `sessions_spawn.fastMode=false` for an ACP child can still get the backend default on the first turn, so map the override into the ACP runtime option/config-option path before dispatch and cover that path.
