## Summary

Keep open: the native subagent path is plausible, but the ACP path still only persists `fastMode` onto the local session entry and does not route it through ACP runtime options/config controls before the first child turn. The PR also still needs real after-fix behavior proof rather than tests plus source assertions.

## What This Changes

The PR adds a `sessions_spawn.fastMode` boolean to native subagent and ACP spawn parameters, docs, prompt snapshots, changelog, and focused tests.

## Best Possible Solution

Keep the native subagent support, but map ACP fast-mode overrides into ACP runtime options or backendExtras so `session/set_config_option fast_mode` runs before the first ACP child turn, with regression coverage for that control path.

## Reproduction Assessment

Yes for the review finding: source inspection shows the PR-added ACP line patches only `SessionEntry.fastMode`, while ACP dispatch derives runtime controls from `AcpSessionRuntimeOptions` and backendExtras. I did not execute the PR branch in this read-only review.

## Solution Assessment

No: persisting `SessionEntry.fastMode` is the right shape for native subagents, but ACP spawns need the override applied through ACP runtime config controls or the ACP-facing parameter/docs should be withheld.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Apply fastMode through ACP runtime options:** `src/agents/acp-spawn.ts:1324`
  - body: In the ACP branch, this added patch only persists `fastMode` into OpenClaw's local session entry. ACP child turns are initialized through `initializeAcpSpawnRuntime`, and runtime controls derive `session/set_config_option` pairs from `AcpSessionRuntimeOptions`/`backendExtras`, which does not include this field, so the first ACP child turn can still use the backend default despite the new tool/docs promise. Please pass the override through the ACP runtime-control path before dispatch, or keep `fastMode` native-only.
  - confidence: 0.89
