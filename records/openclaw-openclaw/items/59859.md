## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve non-cron schedules when editing jobs:** `apps/linux/src/section_cron.c:274-275`
  - body: The edit flow initializes and saves every job as a cron schedule, but existing jobs can validly use `at` or `every`; saving unrelated metadata can fail validation or change the job's schedule shape.
  - confidence: 0.93
- **[P2] Keep session-scoped cron targets on edit:** `apps/linux/src/test_seams.c:42-47`
  - body: Valid `session:<id>` targets fall through to index 0 and are later saved as `isolated`, so editing a session-scoped cron job can silently retarget it away from the intended session.
  - confidence: 0.9
- **[P2] Send supported verbose levels:** `apps/linux/src/section_sessions.c:176-184`
  - body: The sessions UI sends `none`, `low`, and `high`, but `sessions.patch` accepts the current verbose override values `off`, `on`, and `full`; verbose changes from this app will be rejected.
  - confidence: 0.92
- **[P2] Resolve local session logs from session IDs:** `apps/linux/src/section_sessions.c:265-266`
  - body: The Log action builds a key-based `<dirname>/<session-key>/transcript.jsonl` path, while current transcript lookup is sessionId/sessionFile based, so existing local transcripts will usually be missed.
  - confidence: 0.88
- **[P2] Bracket IPv6 hosts before composing URLs:** `apps/linux/src/gateway_config.c:1153-1155`
  - body: Dashboard URLs interpolate the raw host into `scheme://host:port`; IPv6 literals such as `::1` produce malformed URLs and break dashboard or log handoffs on IPv6 setups.
  - confidence: 0.86
- **[P2] Read channel account labels from name:** `apps/linux/src/gateway_data.c:262`
  - body: Channel account snapshots expose the friendly label as `name`, but this parser reads `displayName`, so connected account rows fall back to account IDs instead of showing user labels.
  - confidence: 0.91
