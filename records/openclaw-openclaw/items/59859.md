## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve non-cron schedules when saving edits:** `apps/linux/src/section_cron.c:270-275`
  - body: The edit save path always sends `patch.schedule` as `{ kind: "cron", expr: ... }`. Existing jobs may have `kind: "every"` or `kind: "at"`, so saving unrelated edits can fail validation or change the job's schedule. Carry the original schedule variant through the dialog or block editing unsupported variants.
  - confidence: 0.9
- **[P2] Send supported verbose levels:** `apps/linux/src/section_sessions.c:175-184`
  - body: The Verbose combo emits `none`, `low`, and `high`, but `sessions.patch` normalizes only `off`, `on`, and `full` aliases. Every selection from this UI path is rejected, so users cannot change session verbosity from the Linux app.
  - confidence: 0.9
- **[P2] Keep session-scoped cron targets on edit:** `apps/linux/src/test_seams.c:42-47`
  - body: `session_target_to_index()` maps any unrecognized target, including valid `session:<id>` targets, to the New Session row; saving maps that row back to `isolated`. Editing a session-scoped cron job can silently retarget it even when the user changes only metadata.
  - confidence: 0.88
- **[P2] Resolve local session logs from session IDs:** `apps/linux/src/section_sessions.c:264-267`
  - body: This builds the local transcript path as `<storeDir>/<session-key>/transcript.jsonl`, but the gateway resolves transcripts from `sessionId` and optional `sessionFile` as direct candidates under the sessions directory. Normal local transcripts will be missed and the Log action will fall back to the dashboard.
  - confidence: 0.88
- **[P2] Resolve cron logs with the session transcript layout:** `apps/linux/src/section_cron.c:186-190`
  - body: Cron transcript lookup assumes `<dirname(storePath)>/sessions/<sessionKey>/transcript.jsonl`, but cron store paths are not session transcript directories and gateway transcript resolution is session-id/sessionFile based. Cron Log will usually miss local transcripts and fall back to the dashboard.
  - confidence: 0.84
- **[P2] Bracket IPv6 hosts before composing URLs:** `apps/linux/src/gateway_config.c:1153-1155`
  - body: The dashboard URL builder interpolates `config->host` directly into `scheme://host:port`. For an IPv6 literal such as `::1`, this produces malformed URLs like `http://::1:18789/`, breaking dashboard and log handoffs on valid IPv6/custom-bind setups.
  - confidence: 0.84
