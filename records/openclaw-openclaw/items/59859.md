## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve existing cron schedule variants:** `apps/linux/src/section_cron.c:270-279`
  - body: The edit save path always sends schedule_kind = "cron" and mutation_cron_update serializes schedule.expr, so saving an existing every/at job rewrites or invalidates its schedule even when the user edits only metadata. Preserve the original schedule variant unless the user explicitly changes it.
  - confidence: 0.94
- **[P1] Preserve session-scoped cron targets:** `apps/linux/src/section_cron.c:351`
  - body: sessionTarget allows session:<id>, but the edit flow maps any unknown target to index 0 and then saves it back as isolated. Editing a session-scoped cron job can silently retarget it away from the intended session.
  - confidence: 0.91
- **[P2] Send gateway-supported verbose levels:** `apps/linux/src/section_sessions.c:175-184`
  - body: The verbose combo sends none/low/high through sessions.patch, but the gateway accepts off/on/full. These updates are rejected at runtime, so users cannot change verbose behavior from the Linux app.
  - confidence: 0.93
- **[P2] Use session IDs for local transcript lookup:** `apps/linux/src/section_sessions.c:264-267`
  - body: The Log action builds <store-dir>/<sessionKey>/transcript.jsonl, while the current resolver derives candidates from sessionId/sessionFile such as <sessionsDir>/<sessionId>.jsonl. This usually misses local transcripts before falling back to the dashboard.
  - confidence: 0.89
- **[P2] Require installId before calling skills.install:** `apps/linux/src/gateway_mutations.c:66-69`
  - body: The non-ClawHub skills.install schema requires installId, but this mutation omits it when install_id is null. That can turn the Install button into a validation failure instead of a skill install request.
  - confidence: 0.88
- **[P2] Bracket IPv6 hosts before composing URLs:** `apps/linux/src/gateway_config.c:1031-1040`
  - body: HTTP, WebSocket, and dashboard URL builders interpolate config->host directly, so an IPv6 literal like ::1 becomes http://::1:18789. Bracket IPv6 literals before formatting endpoint URLs.
  - confidence: 0.85
