## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Do not block connected manual gateways on systemd state:** `apps/linux/src/state.c:123-124`
  - body: This marks chat blocked whenever the user systemd service is inactive before checking whether HTTP, WebSocket, and RPC are already healthy. Users running `openclaw gateway run` manually can be fully connected but still have the Linux app disable chat as `SERVICE_INACTIVE`.
  - confidence: 0.9
- **[P1] Send verbose levels accepted by sessions.patch:** `apps/linux/src/section_sessions.c:176-178`
  - body: The verbose selector sends `none`/`low`/`high`, but the current `sessions.patch` parser accepts only `off`/`on`/`full`. These UI updates will be rejected at runtime, so users cannot change session verbose behavior from the Linux app.
  - confidence: 0.92
- **[P1] Preserve non-cron schedules when saving edits:** `apps/linux/src/section_cron.c:274-275`
  - body: The edit path always sends a `cron` schedule patch, so saving an existing `every` or `at` job rewrites or invalidates its schedule even when the user only edits metadata. Omit schedule when unchanged or preserve the original schedule variant.
  - confidence: 0.9
- **[P2] Allow empty skill values to clear secrets:** `apps/linux/src/section_skills.c:133-134`
  - body: Returning on an empty entry prevents the app from sending the delete operation that the gateway already supports for empty `apiKey` and env values. This leaves stale skill secrets or env overrides impossible to remove from the Linux UI.
  - confidence: 0.88
