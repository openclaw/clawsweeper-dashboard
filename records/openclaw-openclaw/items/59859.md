## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve cron schedule variants when editing:** `apps/linux/src/section_cron.c:223-224`
  - body: The edit save path always sends `schedule_kind = "cron"` and serializes the textbox as an `expr`. Current cron jobs can use `kind: "every"` or `kind: "at"`, so saving one of those jobs after an unrelated edit rewrites the schedule shape or fails validation; carry the original variant through the dialog or block unsupported edits.
  - confidence: 0.9
- **[P2] Send supported verbose levels in session patches:** `apps/linux/src/section_sessions.c:141-144`
  - body: The Verbose combo sends `none`/`low`/`high` through `sessions.patch`, but the gateway normalizes verbose levels to `off`/`on`/`full` and rejects unsupported strings. As written, changing verbose behavior from the Linux app returns a validation error instead of updating the session.
  - confidence: 0.88
- **[P2] Resolve local logs from session IDs:** `apps/linux/src/section_sessions.c:213`
  - body: This local transcript probe builds `<store-dir>/<session-key>/transcript.jsonl`, while the gateway resolves transcripts from session ID/session file candidates under the sessions directory. The PR parser already exposes `session_id`; using the key path makes local Log actions miss existing transcript files and fall back to the dashboard unnecessarily.
  - confidence: 0.86
- **[P2] Bracket IPv6 hosts before composing URLs:** `apps/linux/src/gateway_config.c:939-944`
  - body: The HTTP, WebSocket, and dashboard URL builders interpolate `config->host` directly into `scheme://host:port`. For IPv6 literals such as `::1`, that produces malformed URLs like `http://::1:18789`, breaking health checks, WebSocket connections, and dashboard handoffs on valid IPv6 setups.
  - confidence: 0.84
