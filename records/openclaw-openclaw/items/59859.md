## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Move Linux durable state into the shared state DB:** `apps/linux/src/product_state.c:33-36`
  - body: This adds a new steady-state `product-state.ini` under the Linux companion state directory for connection mode, onboarding, and heartbeat intent. OpenClaw’s current runtime policy requires app state/cache to live in SQLite and treats sidecar state files as migration debt; keeping this separate store creates another persisted source of truth that doctor/shared state cannot migrate or inspect consistently.
  - confidence: 0.87
- **[P1] Do not store Linux device auth in legacy JSON files:** `apps/linux/src/device_auth_store.c:22-29`
  - body: The Linux companion writes device auth tokens to `identity/device-auth.json`, but current main’s database-first contract says device auth tokens use typed `state/openclaw.sqlite` rows and legacy JSON payloads are doctor import/export shapes only. Merging this would reintroduce a runtime token store that can diverge from the shared device-auth migration and security model.
  - confidence: 0.9
- **[P1] Do not create a second device identity store:** `apps/linux/src/device_identity.c:24-25`
  - body: This stores the Linux companion private device identity in `identity/device.json`; current main documents that runtime device identity now uses shared SQLite rows and leftover legacy `identity/device.json` blocks startup until doctor imports it. A new client should use the shared identity store instead of creating a parallel JSON identity that can rotate or conflict during migration.
  - confidence: 0.88
- **[P1] Use the canonical exec approval store:** `apps/linux/src/exec_approval_store.c:23-24`
  - body: This adds a JSON-backed Linux `exec-approvals.json` runtime store, but current main says exec approval config now lives in the shared SQLite `exec_approvals_config` row and runtime should not create or report the JSON file as the active store. Reintroducing the JSON path for the Linux app would split operator policy across two stores.
  - confidence: 0.86
