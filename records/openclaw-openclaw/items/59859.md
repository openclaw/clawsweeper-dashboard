## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve non-cron schedules when editing jobs:** `apps/linux/src/section_cron.c:274-275`
  - body: The edit save path always sends `schedule.kind = "cron"` and puts the text field into `expr`. Current jobs can use `kind: "at"` or `kind: "every"`, so saving an existing non-cron job after changing unrelated fields can fail validation or change when it runs.
  - confidence: 0.93
- **[P2] Resolve local session logs from session IDs:** `apps/linux/src/section_sessions.c:264-267`
  - body: This builds the local transcript path as `<store_dir>/<session-key>/transcript.jsonl`, but the gateway resolves transcripts from session IDs/session files under the sessions directory. In normal installs the Log action will miss existing local transcripts and fall back to the dashboard.
  - confidence: 0.88
- **[P2] Send supported verbose levels from the session selector:** `apps/linux/src/section_sessions.c:176-184`
  - body: The selector sends `none`, `low`, and `high`, but `sessions.patch` accepts verbose levels normalized to `off`, `on`, or `full`. Changing verbose from this UI path will be rejected by the gateway.
  - confidence: 0.92
- **[P2] Bracket IPv6 hosts before composing dashboard URLs:** `apps/linux/src/gateway_config.c:1153-1155`
  - body: Dashboard URLs interpolate `config->host` directly into `scheme://host:port`. For an IPv6 literal such as `::1`, this produces malformed URIs like `http://::1:18789/`, breaking dashboard and log handoffs on valid IPv6 setups.
  - confidence: 0.85
