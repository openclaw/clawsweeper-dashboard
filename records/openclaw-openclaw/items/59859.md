## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve schedule variants when editing cron jobs:** `apps/linux/src/section_cron.c:274-275`
  - body: The edit path always sends `patch.schedule` as `{kind:"cron", expr:...}`. Current cron jobs can be `every` or `at`, so saving one after changing unrelated fields rewrites the schedule shape or fails validation; carry the original variant through the dialog, or omit schedule unless it is explicitly edited.
  - confidence: 0.92
- **[P2] Send supported verbose levels:** `apps/linux/src/section_sessions.c:176-184`
  - body: The verbose selector sends `none`/`low`/`high`, but `sessions.patch` accepts `off`/`on`/`full`. Changing verbose from this UI path will be rejected by the gateway, so users cannot update session verbosity.
  - confidence: 0.93
- **[P2] Resolve local session logs from transcript candidates:** `apps/linux/src/section_sessions.c:265-266`
  - body: The Log action builds `<storeDir>/<sessionKey>/transcript.jsonl`, but session transcripts are resolved from `sessionId`/`sessionFile` candidates such as `<sessionsDir>/<sessionId>.jsonl`. Normal local transcripts are missed and the action falls back to the dashboard even when the file exists.
  - confidence: 0.88
- **[P2] Bracket IPv6 hosts before composing URLs:** `apps/linux/src/gateway_config.c:1153-1155`
  - body: Interpolating `config->host` directly into `scheme://host:port` produces malformed URLs for IPv6 literals such as `::1`. Health checks, WebSocket connection, and dashboard/log handoff break on valid IPv6 or custom-bind setups.
  - confidence: 0.86
- **[P2] Keep session-scoped cron targets on edit:** `apps/linux/src/test_seams.c:42-47`
  - body: `session:<id>` is a valid cron `sessionTarget`, but the edit combo maps unknown targets to index 0 and later saves that back as `isolated`. Editing a session-scoped job can silently retarget it away from the intended session.
  - confidence: 0.85
- **[P3] Add labeler coverage for the Linux app surface:** `apps/linux/meson.build:11`
  - body: This adds a new `apps/linux` app surface, but `.github/labeler.yml` still only routes Android, iOS, macOS, and web UI app changes. Add Linux app labeler coverage so future changes route to the right review surface.
  - confidence: 0.82
