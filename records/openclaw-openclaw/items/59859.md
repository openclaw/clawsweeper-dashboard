## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve non-cron schedules when editing cron jobs:** `apps/linux/src/section_cron.c:274-275`
  - body: Saving an edited job always sends `schedule.kind = "cron"` with an `expr`, but the gateway accepts `at`, `every`, and `cron` schedules. Editing an existing `at` or `every` job can fail validation or silently change when it runs, even if the user only edits metadata.
  - confidence: 0.95
- **[P2] Keep session-scoped cron targets on save:** `apps/linux/src/section_cron.c:264-266`
  - body: The save path serializes the combo index back through `session_target_from_index`, while unknown existing targets are selected as index 0. Valid `session:<id>` cron targets therefore get rewritten to `isolated` on save, retargeting jobs away from their intended session.
  - confidence: 0.9
- **[P2] Send supported verbose levels from the session selector:** `apps/linux/src/section_sessions.c:176-184`
  - body: The verbose selector sends `none`, `low`, and `high`, but `sessions.patch` validates verbose levels as `off`, `on`, or `full`. Changing verbose from this UI path will be rejected by the gateway.
  - confidence: 0.96
- **[P2] Resolve session logs with session transcript candidates:** `apps/linux/src/section_sessions.c:264-267`
  - body: The local log action builds `<store_dir>/<session-key>/transcript.jsonl`, but the gateway transcript resolver uses the session id/sessionFile layout and direct files in the sessions directory. This will miss normal local transcripts and fall back to the dashboard unnecessarily.
  - confidence: 0.88
- **[P2] Read channel account labels from name:** `apps/linux/src/gateway_data.c:262`
  - body: `channels.status` account snapshots expose the friendly account label as `name`, not `displayName`. Reading `displayName` leaves the Linux UI without normal account labels and makes rows fall back to account ids.
  - confidence: 0.93
- **[P2] Allow empty skill values to clear stored credentials:** `apps/linux/src/section_skills.c:133-135`
  - body: The env/API-key dialog returns early on an empty entry, but the gateway accepts empty `apiKey` or env values as update payloads. Users cannot clear stale skill secrets or env overrides once set from the Linux UI.
  - confidence: 0.88
- **[P2] Require an install id before sending skills.install:** `apps/linux/src/section_skills.c:80-88`
  - body: The install button can call `mutation_skills_install` with a null `installId`, and the mutation then omits the field. The non-ClawHub `skills.install` schema requires a non-empty `installId`, so this path can produce a validation failure instead of a useful install action.
  - confidence: 0.82
- **[P2] Bracket IPv6 hosts when composing dashboard URLs:** `apps/linux/src/gateway_config.c:1151-1155`
  - body: Dashboard URLs interpolate the raw host into `scheme://host:port`, which produces malformed URLs for IPv6 literals such as `::1`. Bracket IPv6 literals before composing the URI so dashboard and log handoffs work on IPv6 setups.
  - confidence: 0.9

## Security Review

Status: needs_attention

Summary: The diff introduces new security-sensitive desktop gateway/auth/exec surfaces that need focused review before merge.

Concerns:

- **[medium] Review durable device-token storage:** `apps/linux/src/device_auth_store.c:222`
  - body: The Linux app writes durable device auth tokens under an identity/device-auth.json store. The code uses 0700/0600 permissions, but token lifetime, rotation, failure modes, and parity with the TypeScript store should be explicitly reviewed before shipping a first-party desktop app.
  - confidence: 0.86
- **[high] Review exec approval quick-mode behavior:** `apps/linux/src/exec_approval_prompter.c:136`
  - body: The app can persist quick-mode approval policy and send exec approval resolve RPCs. Because this affects command execution authorization, maintainers should review policy mapping, downgrade behavior, UI affordances, and stored defaults as a security boundary.
  - confidence: 0.88
- **[medium] Review native dependency and packaging surface:** `apps/linux/meson.build:15`
  - body: The Meson build adds GTK/libadwaita/AppIndicator/libsoup/libsodium and a private tray helper. These are reasonable distro dependencies, but they expand the first-party native runtime and packaging attack surface and need owner/security signoff.
  - confidence: 0.82
