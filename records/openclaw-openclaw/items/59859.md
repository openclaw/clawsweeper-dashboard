## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P1] Preserve non-cron schedules when saving edits:** `apps/linux/src/section_cron.c:270-274`
  - body: The edit dialog always sends `schedule.kind: "cron"`, so saving an existing `every` or `at` job can rewrite or invalidate its schedule even when the user only changes unrelated metadata. Carry the original schedule variant through the edit flow, or add an explicit schedule-type control before sending `cron.update`.
  - confidence: 0.91
- **[P1] Keep session-scoped cron targets intact:** `apps/linux/src/section_cron.c:284-286`
  - body: Unknown targets are selected as index 0 and saved back as `isolated`, but valid cron targets include `session:<id>`. Editing a job with a session-scoped target can silently retarget future runs away from the intended session, so the UI needs to preserve those values unless the user explicitly changes them.
  - confidence: 0.84
- **[P1] Register the Linux client id in the canonical package:** `src/gateway/protocol/client-info.ts:1-12`
  - body: The branch adds `openclaw-linux` under the old `src/gateway/protocol` path, but current main derives `GatewayClientIdSchema` from `packages/gateway-protocol/src/client-info.ts`. Without moving this addition to the canonical package after rebase, Linux handshakes using that client id will fail validation.
  - confidence: 0.79
