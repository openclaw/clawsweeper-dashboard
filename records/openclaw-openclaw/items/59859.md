## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Preserve cron schedule variants on edit:** `apps/linux/src/section_cron.c:270-275`
  - body: The edit save path always sends `schedule_kind = "cron"` and serializes an `expr`, so saving a valid `every` or `at` job rewrites its schedule shape and can fail validation or change when it runs. Carry the original schedule variant through the dialog or block edits for unsupported variants.
  - confidence: 0.9
- **[P2] Resolve session logs from session IDs:** `apps/linux/src/section_sessions.c:264-267`
  - body: This probes `<store_dir>/<session-key>/transcript.jsonl`, but the gateway resolves transcripts from `sessionId`/`sessionFile` candidates such as `<sessionsDir>/<sessionId>.jsonl`. In normal installs the local Log action will miss existing transcripts and fall back to the dashboard even when a local file is available.
  - confidence: 0.87
- **[P2] Bracket IPv6 hosts before URL formatting:** `apps/linux/src/gateway_config.c:1151-1155`
  - body: Interpolating an IPv6 literal directly into `scheme://host:port` produces malformed URLs like `http://::1:18789/`, breaking dashboard, health, and handoff URLs for valid IPv6/custom-bind setups. Format IPv6 literals as `[host]` before composing URLs.
  - confidence: 0.84
- **[P2] Require an install option before enabling Install:** `apps/linux/src/section_skills.c:340-350`
  - body: The UI shows the Install button for any eligible uninstalled skill, but it only attaches `install-id` when the first option has an id. `skills.install` requires a non-empty `installId`, so clicking this button without one sends a request the gateway rejects; hide/disable the button or choose a valid install option first.
  - confidence: 0.83
- **[P2] Allow empty skill values to clear stored secrets:** `apps/linux/src/section_skills.c:133-134`
  - body: The env/API-key dialog returns early for an empty entry, but the gateway intentionally treats empty `apiKey` and env values as delete operations. This blocks users from removing stale skill credentials or env overrides from the Linux app.
  - confidence: 0.86
