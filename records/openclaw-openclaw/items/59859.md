Full review comments:

- **[P1] Use shared SQLite for Linux identity and auth state:** `apps/linux/src/device_identity.c:135-136`
  - body: This new Linux store writes the device private key to identity/device.json, and sibling code writes device-auth.json, exec-approvals.json, and product-state.ini sidecars. Current main already has shared SQLite tables for device identities, auth tokens, and exec approvals, and repo policy requires OpenClaw-owned runtime state to use that canonical store. Please move these Linux stores to the shared SQLite/Kysely path or get an explicit maintainer-approved artifact and migration design before merge.
  - confidence: 0.9
- **[P1] Preserve cron schedule and session target variants:** `apps/linux/src/section_cron.c:274-276`
  - body: Saving an edit always sends schedule.kind = "cron" and maps the target through a combo helper that turns unknown values such as session:<id> into isolated. Existing gateway jobs can be at/every/cron and can target a specific session, so a no-op Linux edit can change timing or routing. Preserve the original schedule and target unless the user explicitly changes them, and cover cron, every, at, and session:* cases.
  - confidence: 0.91
- **[P3] Register the Linux app labeler surface:** `.github/labeler.yml:184-208`
  - body: The PR adds apps/linux/**, but the app labeler still only routes Android, iOS, macOS, and web-ui surfaces. New app surfaces are supposed to update labeler routing, otherwise this companion work will be misrouted and harder for owners to find in future reviews.
  - confidence: 0.82
