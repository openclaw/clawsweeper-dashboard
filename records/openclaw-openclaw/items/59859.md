## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Preserve existing cron schedule variants:** `apps/linux/src/section_cron.c:274-275`
  - body: Current cron jobs may have `schedule.kind` of `at`, `every`, or `cron`, but the edit save path always sends a cron schedule with `expr`. Saving an existing non-cron job after changing unrelated fields can fail validation or change when the job runs.
  - confidence: 0.93
- **[P2] Keep session-scoped cron targets on save:** `apps/linux/src/test_seams.c:42-47`
  - body: Valid cron targets can be `session:<id>`, but unknown targets map to combo index 0 and are saved back as `isolated`. Editing a session-scoped job therefore retargets it away from the intended session even when the user changes only another field.
  - confidence: 0.9
- **[P2] Send supported verbose levels:** `apps/linux/src/section_sessions.c:176-184`
  - body: The verbose selector sends `none`, `low`, and `high`, while `sessions.patch` accepts `off`, `on`, or `full`. Changing verbose from this UI path will be rejected by the gateway.
  - confidence: 0.92
- **[P2] Resolve session logs from session IDs:** `apps/linux/src/section_sessions.c:265-267`
  - body: The local transcript lookup builds `<store-dir>/<session-key>/transcript.jsonl`, but gateway transcript files are resolved from the session id/sessionFile under the sessions directory. The Log action will usually miss existing local transcripts and fall back to the dashboard.
  - confidence: 0.87
- **[P2] Bracket IPv6 hosts before composing dashboard URLs:** `apps/linux/src/gateway_config.c:1153-1155`
  - body: Dashboard URLs interpolate `config->host` directly into `scheme://host:port`. For IPv6 literals such as `::1`, this produces malformed URLs like `http://::1:18789/`, breaking dashboard and transcript handoffs on valid IPv6 setups.
  - confidence: 0.86
- **[P2] Allow clearing skill credential values:** `apps/linux/src/section_skills.c:133-134`
  - body: The env/API-key dialog returns early on an empty entry, but `skills.update` treats empty `apiKey` and env values as delete operations. Users cannot remove stale secrets or env overrides from the Linux app once they are set.
  - confidence: 0.89
- **[P2] Require an install option before showing Install:** `apps/linux/src/section_skills.c:340-349`
  - body: The Install button is shown for eligible uninstalled skills even when no install option id is available, and the handler then sends `skills.install` without `installId`. The current gateway schema requires `installId` for the non-ClawHub install path, so that button can only fail validation.
  - confidence: 0.84
- **[P2] Register Linux in the native app UI allowlist:** `src/gateway/protocol/client-info.ts:13`
  - body: Adding `GATEWAY_CLIENT_IDS.LINUX_APP` makes the id valid, but current-main handshake logic still treats only macOS, iOS, and Android as first-party native app UI clients. Linux local native reconnects will miss native-app metadata-upgrade behavior unless that allowlist is updated with the new id.
  - confidence: 0.82
