## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Rebase the client id into the protocol package:** `src/gateway/protocol/client-info.ts:13`
  - body: Current main's gateway client-id registry lives in packages/gateway-protocol/src/client-info.ts, but this branch adds LINUX_APP under the old src/gateway/protocol path. After rebase, the Linux app must update the canonical protocol package and matching tests or the gateway handshake contract will not include the new client id.
  - confidence: 0.91
- **[P1] Send supported verbose override values:** `apps/linux/src/section_sessions.c:176-178`
  - body: The Sessions verbose combo sends none/low/high to sessions.patch, while the current parser accepts only off/on/full. Changing verbose from the Linux app will be rejected by the gateway instead of updating the session.
  - confidence: 0.94
- **[P1] Preserve non-cron schedules when saving jobs:** `apps/linux/src/section_cron.c:274-275`
  - body: The cron edit path always writes schedule.kind as cron and puts the textbox into expr. Existing jobs can be every or at schedules, so saving an unrelated edit can fail validation or rewrite the job's schedule semantics.
  - confidence: 0.92
- **[P1] Do not block connected gateways on systemd state:** `apps/linux/src/state.c:123-124`
  - body: Chat readiness is blocked when service_active is false even though compute_state treats HTTP+WS+RPC as runtime truth. A user running openclaw gateway run, a non-systemd setup, or a remote reachable gateway can be connected but still have Linux chat disabled.
  - confidence: 0.83
- **[P2] Allow empty skill values to clear settings:** `apps/linux/src/section_skills.c:133-134`
  - body: The dialog returns before sending skills.update when the entered value is empty, but the gateway uses empty apiKey/env values to delete stored secrets or env overrides. Users would be unable to clear stale skill credentials from the app.
  - confidence: 0.89
- **[P2] Resolve local transcripts from session ids:** `apps/linux/src/section_sessions.c:264-266`
  - body: The Log action probes <store-dir>/<session-key>/transcript.jsonl, but current transcript resolution uses sessionId/sessionFile candidates such as <sessionsDir>/<sessionId>.jsonl. This will usually miss local transcripts and unnecessarily fall back to the dashboard.
  - confidence: 0.86
