## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve non-cron schedules when editing jobs:** `apps/linux/src/section_cron.c:274-275`
  - body: The edit save path always sends `patch.schedule` as `{ kind: "cron", expr: ... }`. Current cron jobs may have `kind: "every"` or `kind: "at"`, so saving an existing non-cron job after changing unrelated fields rewrites its schedule shape and can fail validation or change when the job runs.
  - confidence: 0.9
- **[P2] Keep session-scoped cron targets on edit:** `apps/linux/src/test_seams.c:42-47`
  - body: `session:<id>` is a valid cron `sessionTarget`, but this mapping treats every unknown target as index 0 and the edit flow saves that back as `isolated`. Editing a session-scoped job can silently retarget it away from the intended session.
  - confidence: 0.87
- **[P2] Send supported verbose levels:** `apps/linux/src/section_sessions.c:176-184`
  - body: The verbose selector sends `none`/`low`/`high`, but `sessions.patch` accepts `off`/`on`/`full`. Changing verbose from this UI path will be rejected by the gateway, so users cannot update session verbosity.
  - confidence: 0.93
- **[P2] Resolve local session logs from session IDs:** `apps/linux/src/section_sessions.c:265-266`
  - body: This builds the local transcript path as `<storeDir>/<session-key>/transcript.jsonl`, but the gateway resolves transcripts from `sessionId` and `sessionFile` candidates such as `<sessionsDir>/<sessionId>.jsonl`. Normal local transcripts are missed and the action falls back to the dashboard even when the file exists.
  - confidence: 0.88
- **[P2] Bracket IPv6 hosts before composing URLs:** `apps/linux/src/gateway_config.c:1153-1155`
  - body: `gateway_config_dashboard_url` interpolates `config->host` directly into `scheme://host:port`. For an IPv6 literal such as `::1`, that produces malformed URLs like `http://::1:18789/`, breaking dashboard and log handoff on valid IPv6/custom-bind setups.
  - confidence: 0.84
- **[P2] Read channel account labels from the current field:** `apps/linux/src/gateway_data.c:262`
  - body: `channels.status` account snapshots expose the human label as `name`, but this parser reads `displayName`. Normal connected accounts therefore fall back to account IDs instead of showing friendly labels in the Linux UI.
  - confidence: 0.88
- **[P2] Allow clearing skill env and API key values:** `apps/linux/src/section_skills.c:133-134`
  - body: The env/API-key dialog returns early for an empty string, but `skills.update` treats empty `apiKey` or env values as delete operations. This blocks users from clearing stale secrets or env overrides from the Linux app.
  - confidence: 0.9
- **[P2] Classify the Linux app wherever native UI clients are gated:** `src/gateway/protocol/client-info.ts:13`
  - body: Adding `LINUX_APP` makes `openclaw-linux` a valid client ID, but current main's native-app UI policy still hardcodes macOS/iOS/Android. Without updating that allowlist and tests, Linux app reconnect metadata and pairing-policy paths will not follow the intended first-party native app behavior.
  - confidence: 0.78
