## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve non-cron schedules when editing jobs:** `apps/linux/src/section_cron.c:274-275`
  - body: The edit save path always sends `patch.schedule` as `{ kind: "cron", expr: ... }`. Current cron jobs may validly use `kind: "every"` or `kind: "at"`, so saving one after changing unrelated fields can rewrite its schedule shape, fail validation, or change when it runs; carry the original schedule variant or block unsupported variants in this dialog.
  - confidence: 0.92
- **[P2] Keep session-scoped cron targets on edit:** `apps/linux/src/test_seams.c:42-47`
  - body: `session:<id>` is a valid cron target, but this mapper treats unknown values as index 0, and the save path turns index 0 into `isolated`. Editing a session-scoped job can therefore retarget it away from the intended session even when the user changes another field.
  - confidence: 0.9
- **[P2] Send supported verbose levels from the session selector:** `apps/linux/src/section_sessions.c:176-184`
  - body: The verbose selector sends `none`, `low`, and `high` as `verboseLevel`, but current gateway normalization accepts `off`, `on`, and `full`. Changing verbose from this UI path will be rejected instead of updating the session.
  - confidence: 0.91
- **[P2] Bracket IPv6 hosts before composing dashboard URLs:** `apps/linux/src/gateway_config.c:1153-1155`
  - body: Dashboard URLs interpolate `config->host` directly into `scheme://host:port`. For an IPv6 literal such as `::1`, that produces malformed URLs like `http://::1:18789/`, breaking dashboard and log handoffs on valid IPv6/custom-bind setups.
  - confidence: 0.84
- **[P2] Resolve local session logs from transcript files:** `apps/linux/src/section_sessions.c:264-266`
  - body: The local log opener probes `<store-dir>/<session-key>/transcript.jsonl`, but the gateway resolves transcripts as session files under the sessions directory via `sessionId` or `sessionFile`. This usually misses existing local transcripts and falls back to the dashboard even when a local file is available.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: The PR adds a new native operator surface that handles credentials, pairing, remote access, local files, and exec approvals, so security review is required before merge.

Concerns:

- **[medium] Review durable device identity and token storage:** `apps/linux/src/device_identity.c:135`
  - body: The Linux app persists an Ed25519 private identity and device token records under the state directory. File modes are restrictive, but token lifetime, rotation, revocation, backup/migration behavior, and parity with other clients need explicit security signoff before this ships.
  - confidence: 0.86
- **[high] Review exec approval quick-mode auto-resolution:** `apps/linux/src/exec_approval_prompter.c:176`
  - body: The exec approval flow can persist a quick mode that maps requests to automatic allow or deny behavior and then resolves gateway requests without prompting. That changes a security-sensitive operator approval path and needs product/security signoff.
  - confidence: 0.84
