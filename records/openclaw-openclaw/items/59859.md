## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Read skills.status ids from skillKey:** `apps/linux/src/gateway_data.c:345`
  - body: Current skills.status entries expose `skillKey`, but this parser reads `key`. The action buttons then store a null target and Skills enable/install/update/env mutations cannot reliably address the selected skill.
  - confidence: 0.93
- **[P1] Send supported verbose levels:** `apps/linux/src/section_sessions.c:176-178`
  - body: The gateway validates session `verboseLevel` as `off`, `on`, or `full`; sending `none`, `low`, or `high` makes normal verbose changes from this dropdown fail at `sessions.patch`.
  - confidence: 0.94
- **[P1] Preserve cron schedule and target variants:** `apps/linux/src/section_cron.c:270-276`
  - body: Editing an existing cron job always serializes the patch as `kind: "cron"` and re-derives the target from the limited combo, so `every`/`at` schedules and `session:*` targets created elsewhere are overwritten on save.
  - confidence: 0.9
- **[P1] Copy the unit name before clearing the cache:** `apps/linux/src/systemd.c:488-490`
  - body: `current_unit` points at cached storage, but `clear_cached_service_config()` frees that storage before `g_strdup(current_unit)` uses it, creating a unit-retarget use-after-free path.
  - confidence: 0.87
- **[P1] Do not gate connected chat on systemd active:** `apps/linux/src/state.c:123-124`
  - body: A manually started gateway can have HTTP, WebSocket, RPC, and auth all healthy while no user systemd unit is active; this branch still blocks chat in that valid runtime mode.
  - confidence: 0.82
- **[P2] Allow empty skill values to clear config:** `apps/linux/src/section_skills.c:133-134`
  - body: The gateway treats empty `apiKey` and env values as delete operations, but this dialog returns before sending the mutation, so users cannot clear stale skill secrets or env overrides from the app.
  - confidence: 0.9
- **[P2] Require an installId for skills.install:** `apps/linux/src/gateway_mutations.c:66-69`
  - body: The non-ClawHub `skills.install` schema requires `installId`; conditionally omitting it sends a request the server rejects instead of disabling the action or choosing a valid install option.
  - confidence: 0.84
