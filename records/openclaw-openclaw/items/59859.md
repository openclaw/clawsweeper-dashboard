## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve non-cron schedules when editing jobs:** `apps/linux/src/section_cron.c:270-275`
  - body: The edit save path always sends `patch.schedule` as `{ kind: "cron", expr: ... }`. Current cron jobs can also use `kind: "every"` or `kind: "at"`, so saving an existing non-cron job after changing unrelated fields can fail validation or change when the job runs.
  - confidence: 0.93
- **[P2] Keep session-scoped cron targets on edit:** `apps/linux/src/test_seams.c:42-47`
  - body: Unknown targets, including valid `session:<id>` values, are mapped to index 0 and later saved back as `isolated`. A no-op edit can silently retarget a job away from its intended session.
  - confidence: 0.91
- **[P2] Send supported verbose values in session patches:** `apps/linux/src/section_sessions.c:175-184`
  - body: The UI sends verbose levels `none`, `low`, and `high`, but `sessions.patch` validation accepts `off`, `on`, or `full`. Verbose changes from the Linux app will be rejected by the gateway.
  - confidence: 0.94
- **[P2] Resolve local logs from session IDs:** `apps/linux/src/section_sessions.c:264-267`
  - body: This builds `<storeDir>/<session-key>/transcript.jsonl`, but gateway transcripts are resolved from `sessionId` or `sessionFile` as direct files under the sessions directory. The Log action will usually miss existing local transcripts and fall back to the dashboard.
  - confidence: 0.88
- **[P2] Bracket IPv6 hosts before composing URLs:** `apps/linux/src/gateway_config.c:1151-1155`
  - body: Dashboard URLs interpolate `config->host` directly as `scheme://host:port`; IPv6 literals such as `::1` need brackets. Without normalization, valid IPv6 gateway configs produce malformed health, WebSocket, and dashboard URLs.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: The diff adds a new credentialed native desktop app with device auth, exec approvals, remote SSH tunneling, and local gateway control paths that need focused security review before merge.

Concerns:

- **[medium] Review durable device auth storage:** `apps/linux/src/device_auth_store.c:4`
  - body: The Linux app persists operator device-token records under `identity/device-auth.json`; permissions, lifecycle, redaction, and compatibility with the shared device-auth contract should be explicitly reviewed before shipping.
  - confidence: 0.82
- **[medium] Review exec approval dispatch from the desktop app:** `apps/linux/src/exec_approval_prompter.c:4`
  - body: The new exec approval prompter subscribes to gateway approval events and resolves operator decisions over RPC. That is a sensitive command-execution authorization path and needs focused authorization/lifetime review.
  - confidence: 0.84
- **[medium] Review SSH tunnel command boundaries:** `apps/linux/src/remote_tunnel_command.c:15`
  - body: The remote tunnel helper builds an `ssh` argv from user/config input. The branch includes option-smuggling guards, but host/user/identity trust boundaries and spawned-process lifecycle still need security review as part of accepting remote mode.
  - confidence: 0.78
