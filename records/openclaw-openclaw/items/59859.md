## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve non-cron schedules when saving edited jobs:** `apps/linux/src/section_cron.c:274-275`
  - body: The edit save path always sends `schedule_kind = "cron"` with `schedule_expr`, but current cron jobs can validly use `kind: "every"` or `kind: "at"`; saving an existing non-cron job after changing unrelated fields can fail validation or change when it runs. Carry the original schedule variant through the dialog or block editing unsupported variants.
  - confidence: 0.92
- **[P2] Keep session-scoped cron targets on edit:** `apps/linux/src/test_seams.c:42-47`
  - body: Valid `session:<id>` targets fall through to index 0 and are later saved as `isolated`, so editing a session-scoped cron job can silently retarget it away from the intended session. Preserve explicit session targets or present them as a non-lossy read-only/current selection.
  - confidence: 0.9
- **[P2] Send supported verbose levels:** `apps/linux/src/section_sessions.c:176-184`
  - body: The sessions UI sends `none`, `low`, and `high`, but `sessions.patch` normalizes verbose overrides to `off`, `on`, and `full`; verbose changes from this app will be rejected. Map the combo values to the server contract before dispatching `sessions.patch`.
  - confidence: 0.92
- **[P2] Resolve local session logs from session IDs:** `apps/linux/src/section_sessions.c:264-266`
  - body: This builds the local transcript path as `<storeDir>/<session-key>/transcript.jsonl`, but the gateway stores and resolves transcripts by `sessionId` or `sessionFile`; the Log action will usually miss existing local transcripts and fall back to the dashboard. Use the parsed `session_id` and the shared transcript layout.
  - confidence: 0.87
- **[P2] Bracket IPv6 hosts before composing URLs:** `apps/linux/src/gateway_config.c:1153-1155`
  - body: The URL builder interpolates `config->host` directly into `scheme://host:port`. For IPv6 literals such as `::1`, this produces malformed URLs like `http://::1:18789/`, breaking dashboard and log handoff on valid IPv6 setups. Normalize IPv6 literals to `[host]` before formatting.
  - confidence: 0.86
- **[P2] Read channel account labels from name:** `apps/linux/src/gateway_data.c:262`
  - body: Channel account snapshots expose the friendly label as `name`, but this parser reads `displayName`; normal connected account rows fall back to account IDs instead of showing user labels. Parse `name` into the display label field.
  - confidence: 0.91
