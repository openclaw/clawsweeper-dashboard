## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve non-cron schedules when editing jobs:** `apps/linux/src/section_cron.c:270-275`
  - body: The edit save path always serializes the patch as a cron schedule, but the gateway accepts `at`, `every`, and `cron` schedules. Opening an `at` or `every` job and saving unrelated fields can fail validation or rewrite the schedule shape unexpectedly.
  - confidence: 0.94
- **[P2] Keep session-scoped cron targets on edit:** `apps/linux/src/test_seams.c:42-48`
  - body: Unknown targets, including valid `session:<id>` values, are mapped to index 0 and later saved back as `isolated`. A no-op edit can silently retarget a job away from its intended session.
  - confidence: 0.92
- **[P2] Send supported verbose values in session patches:** `apps/linux/src/section_sessions.c:175-184`
  - body: The UI sends verbose levels `none`, `low`, and `high`, but `sessions.patch` validation accepts `off`, `on`, or `full`. Verbose changes from the Linux app will be rejected by the gateway.
  - confidence: 0.93
- **[P2] Resolve local transcripts by session ID:** `apps/linux/src/section_sessions.c:264-267`
  - body: The local Log action builds `<store-dir>/<session-key>/transcript.jsonl`, but transcript storage is resolved from the session ID or sessionFile as direct files under the sessions directory. This misses existing local transcripts and unnecessarily falls back to the dashboard.
  - confidence: 0.9
- **[P2] Bracket IPv6 literals in dashboard URLs:** `apps/linux/src/gateway_config.c:1151-1155`
  - body: Dashboard URLs interpolate the host directly as `scheme://host:port`; IPv6 literals such as `::1` need brackets. Without normalization, otherwise valid IPv6 gateway configs produce malformed URLs.
  - confidence: 0.86
