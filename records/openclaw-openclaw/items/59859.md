- **[P1] Preserve cron schedule variants when editing jobs:** `apps/linux/src/section_cron.c:274-275`
  - body: The edit save path always sends `schedule_kind = "cron"` and serializes the textbox as `expr`, but the current protocol has valid `at`, `every`, and `cron` schedule shapes.   - confidence: 0.94

  - body: `session:<id>` is a valid cron target, but the combo mapping only recognizes `main`, `current`, and `isolated`; unknown values select index 0 and save back as `isolated`. That silently retargets session-scoped jobs when the user saves unrelated edits. Omit unchanged custom targets or model them explicitly.
  - confidence: 0.9
