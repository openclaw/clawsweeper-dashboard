## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve non-cron schedules when editing jobs:** `apps/linux/src/section_cron.c:274-275`
  - body: The edit save path always sends `schedule_kind = "cron"` with `schedule_expr`, but the gateway cron contract also supports `kind: "every"` and `kind: "at"`. Saving an existing non-cron job after editing unrelated fields can rewrite the schedule shape, fail validation, or change when the job runs; carry the original schedule variant through the dialog or disable unsupported edits.
  - confidence: 0.9
- **[P2] Keep session-scoped cron targets when saving edits:** `apps/linux/src/test_seams.c:42-47`
  - body: `session_target_to_index()` maps any unknown target to index 0, and index 0 is later serialized as `"isolated"`. Since valid cron targets include `session:<id>`, editing a job created elsewhere with a session-scoped target silently retargets it away from that session even if the user changes only another field.
  - confidence: 0.86
- **[P2] Resolve local session logs from session IDs:** `apps/linux/src/section_sessions.c:264-266`
  - body: The Log action builds `<store_dir>/<session-key>/transcript.jsonl`, but the gateway resolves transcript files from `sessionId` and optional `sessionFile` under the session store directory. The PR parser already exposes `session_id`; use that contract or the Log action will usually miss existing local transcripts and fall back to the dashboard.
  - confidence: 0.88
- **[P2] Bracket IPv6 hosts before formatting URLs:** `apps/linux/src/gateway_config.c:1034-1040`
  - body: These URL builders interpolate `config->host` directly into `scheme://host:port`. For an IPv6 literal such as `::1`, this produces malformed URLs like `http://::1:18789`, breaking health, WebSocket, and dashboard handoff paths on valid IPv6/custom-bind setups.
  - confidence: 0.84
- **[P3] Add labeler coverage for the Linux app:** `apps/linux/meson.build:11`
  - body: This PR introduces a new app surface under `apps/linux`, but current labeler coverage only knows Android, iOS, macOS, and web UI app paths. Add a Linux app label rule so future changes to this surface are routed and triaged consistently.
  - confidence: 0.82
