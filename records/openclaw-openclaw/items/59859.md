## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve non-cron schedules when editing jobs:** `apps/linux/src/section_cron.c:274-275`
  - body: The edit save path always sends `patch.schedule` as `{ kind: "cron", expr: ... }`. Current cron jobs may also use `kind: "every"` or `kind: "at"`, so saving an existing non-cron job after changing unrelated fields can rewrite or reject its schedule.
  - confidence: 0.92
- **[P2] Send supported verbose values:** `apps/linux/src/section_sessions.c:176-184`
  - body: The verbose selector sends `none`, `low`, and `high`, but `sessions.patch` validates verbose overrides as `off`, `on`, or `full`. Changing verbose from the Linux app will be rejected by the gateway.
  - confidence: 0.93
- **[P2] Resolve local logs from session IDs:** `apps/linux/src/section_sessions.c:265-266`
  - body: This builds `<store_dir>/<session-key>/transcript.jsonl`, but gateway transcript resolution is keyed by `sessionId` and optional `sessionFile`. Normal local transcripts are missed, so the Log action falls back to the dashboard even when the file exists.
  - confidence: 0.88
- **[P2] Bracket IPv6 hosts before composing URLs:** `apps/linux/src/gateway_config.c:1153-1155`
  - body: The URL builder interpolates `config->host` directly into `scheme://host:port`. For IPv6 literals such as `::1`, this produces malformed URLs like `http://::1:18789/`, breaking health checks, WebSocket connection, and dashboard handoff on valid IPv6 setups.
  - confidence: 0.86
- **[P3] Add labeler coverage for the Linux app surface:** `apps/linux/meson.build:11`
  - body: This adds a new `apps/linux` app surface, but the repository labeler currently covers Android, iOS, macOS, and web UI only. Add `.github/labeler.yml` coverage so future Linux app changes route correctly.
  - confidence: 0.82
