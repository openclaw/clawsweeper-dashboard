## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Preserve non-cron schedules when editing jobs:** `apps/linux/src/section_cron.c:274-275`
  - body: The edit save path always rebuilds the schedule as `cron`. Current gateway cron jobs can be `at`, `every`, or `cron`, so saving an existing `at`/`every` job after unrelated edits can rewrite or reject its schedule. Carry the original schedule variant through the edit flow or block unsupported variants.
  - confidence: 0.91
- **[P2] Resolve local session logs from session ids:** `apps/linux/src/section_sessions.c:265-266`
  - body: This probes `<storeDir>/<sessionKey>/transcript.jsonl`, but current transcript resolution uses `sessionId`/`sessionFile` direct files such as `<sessionsDir>/<sessionId>.jsonl`. The Log action will miss local transcripts and fall back to the dashboard, which fails for offline or no-dashboard-access cases.
  - confidence: 0.9
- **[P2] Bracket IPv6 hosts before composing URLs:** `apps/linux/src/gateway_config.c:1153-1155`
  - body: Host validation accepts IP literals, but the URL builders interpolate the host directly. An IPv6 literal like `::1` becomes malformed as `http://::1:18789/...`, breaking health checks, WebSocket URLs, and dashboard handoff; bracket IPv6 literals before formatting URLs.
  - confidence: 0.87
