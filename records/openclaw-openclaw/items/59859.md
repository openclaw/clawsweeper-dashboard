Overall confidence: 0.9

Full review comments:

- **[P1] Send supported session verbose levels:** `apps/linux/src/section_sessions.c:176-178`
  - body: The verbose combo maps selections to `none`/`low`/`high`, then `mutation_sessions_patch` sends those as `verboseLevel`. Current main rejects anything other than `off`, `on`, `full`, or null, so changing Verbose from the Linux app fails instead of updating the session.
  - confidence: 0.92
- **[P1] Preserve cron schedule and target variants:** `apps/linux/src/section_cron.c:274-276`
  - body: The edit save path always writes `schedule.kind = "cron"` and collapses the target through a fixed combo. Existing `every`/`at` jobs or `session:<id>` targets from current gateway contracts can be rejected or rewritten when the user saves unrelated edits.
  - confidence: 0.91
- **[P2] Allow empty skill values to clear settings:** `apps/linux/src/section_skills.c:133-134`
  - body: The dialog returns immediately when the entry is empty, so the Linux app can never send the empty API key/env value that `skills.update` uses to delete stored secrets or env overrides. Users can set a value but cannot clear it from this UI.
  - confidence: 0.88
- **[P2] Resolve local session logs from session ids:** `apps/linux/src/section_sessions.c:266`
  - body: This probes `<storeDir>/<session-key>/transcript.jsonl`, but current transcript resolution uses the session id or explicit `sessionFile` under the sessions directory. The Log action will usually miss local transcripts and fall back to the dashboard even when a local file exists.
  - confidence: 0.86
