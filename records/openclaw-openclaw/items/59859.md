## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve existing cron schedule variants:** `apps/linux/src/section_cron.c:270-276`
  - body: The edit save path always sends `.schedule_kind = "cron"` and `.schedule_expr = schedule`, so saving a job created elsewhere with `kind: "every"` or `kind: "at"` rewrites it as a cron schedule. Current `CronScheduleSchema` supports `at`, `every`, and `cron`, so editing unrelated fields can fail validation or change when the job runs.
  - confidence: 0.92
- **[P2] Keep session-scoped cron targets on edit:** `apps/linux/src/test_seams.c:42-47`
  - body: The mapper collapses any unrecognized target to index 0, which `session_target_from_index` later saves as `isolated`. Valid cron targets include `session:<id>`, so opening and saving a session-scoped job retargets it away from that session even when no target change was intended.
  - confidence: 0.9
- **[P2] Send supported verbose levels:** `apps/linux/src/section_sessions.c:175-184`
  - body: The verbose combo sends `none`/`low`/`high` through `sessions.patch`, but the gateway accepts only `off`/`on`/`full` for `verboseLevel`. Verbose changes from the Linux UI will be rejected instead of updating session behavior.
  - confidence: 0.88
- **[P2] Resolve local session logs from session IDs:** `apps/linux/src/section_sessions.c:264-266`
  - body: The local log probe builds `<store-dir>/<session-key>/transcript.jsonl`, but OpenClaw transcripts are direct files under the sessions directory, usually based on `sessionId` or `sessionFile`. This means Log usually misses an existing local transcript and falls back to the dashboard, which breaks offline or no-dashboard access.
  - confidence: 0.87
- **[P2] Bracket IPv6 hosts before composing dashboard URLs:** `apps/linux/src/gateway_config.c:1153-1155`
  - body: The dashboard URL builder interpolates `config->host` directly into `scheme://host:port`; an IPv6 literal like `::1` becomes `http://::1:18789/`, which is not a valid URI and breaks dashboard or log handoffs for IPv6/custom-bind setups. Normalize IPv6 literals to `[::1]` before formatting.
  - confidence: 0.86
