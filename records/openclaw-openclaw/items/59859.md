## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve non-cron schedules on edit:** `apps/linux/src/section_cron.c:270-276`
  - body: The edit save path always sends `schedule_kind = "cron"` and puts the textbox into `schedule_expr`. Existing jobs created with `kind: "every"` or `kind: "at"` will be rewritten or rejected when the user saves unrelated edits, so the dialog needs to carry the original schedule variant or block editing unsupported variants.
  - confidence: 0.9
- **[P1] Keep session-scoped cron targets:** `apps/linux/src/test_seams.c:42-47`
  - body: `session:<id>` is a valid gateway cron target, but this mapping treats unknown targets as index 0 and later saves that back as `isolated`. Editing a job targeted at a specific session can silently retarget future runs away from the intended session.
  - confidence: 0.88
- **[P2] Send supported verbose levels:** `apps/linux/src/section_sessions.c:175-184`
  - body: The verbose dropdown sends `none`, `low`, or `high`, but `sessions.patch` only accepts verbose levels normalized to `off`, `on`, or `full`. As written, changing verbose behavior from the Linux app is rejected by the gateway.
  - confidence: 0.9
- **[P2] Bracket IPv6 hosts before composing URLs:** `apps/linux/src/gateway_config.c:1151-1155`
  - body: The URL builder interpolates `config->host` directly into `scheme://host:port`. IPv6 literals such as `::1` become malformed URLs like `http://::1:18789/`, breaking health checks, WebSocket connection, and dashboard handoff on valid IPv6/custom-bind setups.
  - confidence: 0.84
