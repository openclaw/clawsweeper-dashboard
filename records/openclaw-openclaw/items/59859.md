## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve non-cron schedules when editing jobs:** `apps/linux/src/section_cron.c:270-275`
  - body: The edit save path always serializes the schedule as `{ kind: "cron", expr: ... }`. Current cron jobs can use `kind: "every"` or `kind: "at"`, so saving an existing non-cron job after changing unrelated fields can rewrite or invalidate its schedule; carry the original variant through the dialog or prevent unsupported edits.
  - confidence: 0.91
- **[P2] Keep session-scoped cron targets on edit:** `apps/linux/src/section_cron.c:264-266`
  - body: The target combo maps unrecognized persisted targets to index `0`, which is later saved as `isolated`. Since current cron targets can be `session:<id>`, editing a session-scoped job can silently retarget it away from the intended session even when only metadata changed.
  - confidence: 0.86
- **[P2] Send supported verbose levels:** `apps/linux/src/section_sessions.c:176-184`
  - body: The verbose dropdown sends `none`, `low`, and `high`, but `sessions.patch` validates verbose overrides as `off`, `on`, or `full`. These updates will be rejected by the gateway, so users cannot change verbose behavior from this UI path.
  - confidence: 0.89
- **[P2] Resolve local session logs from session IDs:** `apps/linux/src/section_sessions.c:265-266`
  - body: The Log action looks for `<storeDir>/<session-key>/transcript.jsonl`, but gateway transcript resolution is based on `sessionId` and optional `sessionFile`. This will miss normal local transcripts and fall back to dashboard even when the file exists locally.
  - confidence: 0.88
- **[P2] Bracket IPv6 hosts before composing URLs:** `apps/linux/src/gateway_config.c:1153-1155`
  - body: The dashboard URL builder interpolates `config->host` directly into `scheme://host:port`. For IPv6 literals such as `::1`, this produces malformed URLs like `http://::1:18789/`; bracket IPv6 literals before formatting.
  - confidence: 0.84
- **[P2] Read channel account labels from name:** `apps/linux/src/gateway_data.c:262`
  - body: `channels.status` account snapshots expose the friendly label as `name`, but this parser reads `displayName`. Normal channel account rows will lose their human-readable labels and fall back to less useful identifiers.
  - confidence: 0.86
