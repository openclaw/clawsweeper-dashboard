## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P1] Preserve existing cron schedule variants on save:** `apps/linux/src/section_cron.c:274-275`
  - body: Saving an edited job always sends `schedule_kind = "cron"`, but the gateway contract supports `at`, `every`, and `cron` schedules and `cron.update` accepts an optional schedule patch. Editing a non-cron job can therefore rewrite the schedule shape or fail validation even when the user only changed metadata.
  - confidence: 0.9
- **[P2] Resolve session logs through transcript candidates:** `apps/linux/src/section_sessions.c:265-266`
  - body: The local Log action probes `<storeDir>/<session-key>/transcript.jsonl`, while current transcript resolution is based on sessionId/sessionFile, storePath, agentId, and legacy candidates. Normal local transcripts will be missed and the app can fall back to the dashboard even when the file exists.
  - confidence: 0.86
- **[P2] Bracket IPv6 hosts before composing dashboard URLs:** `apps/linux/src/gateway_config.c:1153-1155`
  - body: `gateway_config_dashboard_url()` interpolates `config->host` directly into `scheme://host:port`; an IPv6 literal such as `::1` produces an invalid URI like `http://::1:18789/`, breaking dashboard and transcript handoffs for IPv6 setups.
  - confidence: 0.84
