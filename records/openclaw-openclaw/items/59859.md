## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve existing cron schedule variants:** `apps/linux/src/section_cron.c:270-276`
  - body: The edit save path always sends `schedule.kind: "cron"` with `expr`, even when an existing job uses `at` or `every`. Saving unrelated edits on those jobs can fail validation or change when they run; carry the original schedule variant or block editing unsupported variants.
  - confidence: 0.92
- **[P2] Keep session-scoped cron targets on edit:** `apps/linux/src/test_seams.c:42-47`
  - body: `sessionTarget` values such as `session:<id>` are valid, but this mapper treats unknown values as index 0, and the save path turns that back into `isolated`. Editing a session-scoped job can retarget it away from the intended session even when the user changes another field.
  - confidence: 0.9
- **[P2] Send supported verbose levels:** `apps/linux/src/section_sessions.c:175-184`
  - body: The verbose selector sends `none`, `low`, and `high` as `verboseLevel`, while current gateway normalization accepts `off`, `on`, and `full`. Changing verbose from this UI path will be rejected instead of updating the session.
  - confidence: 0.91
- **[P2] Resolve local logs from session IDs:** `apps/linux/src/section_sessions.c:264-267`
  - body: The Log action builds `<store-dir>/<session-key>/transcript.jsonl`, but gateway transcript resolution is based on `sessionId` and `sessionFile` candidates. This usually misses existing local transcripts and falls back to the dashboard, which breaks offline or no-dashboard log access.
  - confidence: 0.86
- **[P3] Add labeler coverage for the Linux app surface:** `apps/linux/meson.build:11`
  - body: This PR creates a new `apps/linux` app surface, but `.github/labeler.yml` has no Linux app rule. Future Linux companion changes will miss the app ownership label required for new app surfaces.
  - confidence: 0.8

## Security Review

Status: needs_attention

Summary: The diff adds credentialed native gateway access, durable identity/token storage, remote SSH handling, and exec approval auto-resolution, so it needs focused security review before merge.

Concerns:

- **[high] Review exec approval quick-mode auto-resolution:** `apps/linux/src/exec_approval_prompter.c:176`
  - body: A persisted quick mode can automatically allow or deny gateway exec approval requests without prompting. That changes a security-sensitive operator approval path and needs explicit product/security signoff before shipping.
  - confidence: 0.84
- **[medium] Review durable device identity and token storage:** `apps/linux/src/device_identity.c:126`
  - body: The Linux companion writes an Ed25519 private identity and device token records under the state directory. File modes are restrictive, but token lifetime, rotation, revocation, backup/migration behavior, and parity with other clients need security review.
