## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve non-cron schedules when editing jobs:** `apps/linux/src/section_cron.c:274-275`
  - body: The edit save path always sends the textbox as `schedule.kind = "cron"`. Existing jobs can be `at` or `every`, so saving one after changing unrelated fields rewrites the schedule shape and can fail validation or change when the job runs.
  - confidence: 0.93
- **[P2] Keep session-scoped cron targets on save:** `apps/linux/src/section_cron.c:351`
  - body: `session:<id>` targets are treated as unknown, selected as index 0, and then saved back as `isolated`. Editing a session-scoped job therefore retargets it away from the intended session even when the user changes another field.
  - confidence: 0.9
- **[P2] Send supported verbose levels from the session selector:** `apps/linux/src/section_sessions.c:176-184`
  - body: The verbose selector sends `none`, `low`, and `high`, but `sessions.patch` accepts `off`, `on`, and `full`. Changing verbose from this UI path will be rejected by the gateway.
  - confidence: 0.91
- **[P2] Bracket IPv6 hosts before composing dashboard URLs:** `apps/linux/src/gateway_config.c:1153-1155`
  - body: Dashboard URLs interpolate `config->host` directly into `scheme://host:port`. IPv6 literals such as `::1` produce malformed URLs like `http://::1:18789/`, breaking dashboard and log handoffs on valid IPv6 setups.
  - confidence: 0.84
- **[P2] Resolve local session logs from session IDs:** `apps/linux/src/section_sessions.c:265-266`
  - body: The local log path is built as `<store-dir>/<session-key>/transcript.jsonl`, but current transcript files are resolved from the session id/sessionFile under the sessions directory. The Log action will usually miss existing local transcripts and fall back to the dashboard.
  - confidence: 0.86
