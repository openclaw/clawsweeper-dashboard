## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve non-cron schedule variants:** `apps/linux/src/section_cron.c:274-275`
  - body: The edit-save path always sends `.schedule_kind = "cron"`, and the mutation helper serializes that as `{ kind: "cron", expr: ... }`. Current cron jobs may use `kind: "every"` or `kind: "at"`, so saving an existing non-cron job after changing unrelated fields can fail validation or change when the job runs; carry the original schedule variant through the dialog or block editing unsupported variants.
  - confidence: 0.9
- **[P2] Resolve session logs from session IDs:** `apps/linux/src/section_sessions.c:265-266`
  - body: This local transcript probe builds `<storeDir>/<session-key>/transcript.jsonl`, but the gateway resolves transcripts from `sessionId` and `sessionFile` candidates under the sessions directory, and the PR parser already stores `session_id`. As written, the Log action usually misses existing local transcripts and falls back to the dashboard, which breaks offline or no-dashboard access.
  - confidence: 0.87
- **[P2] Bracket IPv6 hosts before composing URLs:** `apps/linux/src/gateway_config.c:1031-1040`
  - body: The HTTP and WebSocket builders interpolate `config->host` directly into `scheme://host:port`. For an IPv6 literal such as `::1`, that produces malformed URLs like `http://::1:18789`, breaking health checks and WebSocket connection on valid IPv6/custom-bind setups; normalize IPv6 literals to `[host]` before formatting all gateway URLs.
  - confidence: 0.84
