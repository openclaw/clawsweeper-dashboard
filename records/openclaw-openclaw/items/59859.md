## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve non-cron schedules when editing jobs:** `apps/linux/src/section_cron.c:270-275`
  - body: The edit save path always sends `schedule_kind = "cron"` and serializes the textbox as `expr`. Existing gateway jobs can use `kind: "every"` or `kind: "at"`, so saving an unrelated field can fail validation or change when the job runs. Carry the original schedule variant through the dialog or block editing unsupported variants.
  - confidence: 0.9
- **[P2] Preserve session-scoped cron targets when saving jobs:** `apps/linux/src/test_seams.c:42-47`
  - body: `session_target_to_index()` maps any unrecognized target, including valid `session:<id>` targets, to the New Session row, and save maps that row back to `isolated`. Editing a session-scoped cron job can silently retarget it away from the intended session even when the user changes only metadata.
  - confidence: 0.86
- **[P2] Resolve local session logs from session IDs:** `apps/linux/src/section_sessions.c:264-266`
  - body: This builds the local transcript path as `<storeDir>/<session-key>/transcript.jsonl`, but the gateway stores and resolves transcripts from `sessionId` and optional `sessionFile` under the sessions directory. Normal local transcripts will be missed, so the Log action falls back to the dashboard even when a local file exists.
  - confidence: 0.87
- **[P2] Bracket IPv6 hosts before composing URLs:** `apps/linux/src/gateway_config.c:1151-1155`
  - body: The dashboard URL builder interpolates `config->host` directly into `scheme://host:port`. For an IPv6 literal such as `::1`, that produces malformed URLs like `http://::1:18789`, breaking dashboard/log handoffs on valid IPv6/custom-bind setups. Normalize IPv6 literals to `[host]` before formatting these URLs.
  - confidence: 0.84
