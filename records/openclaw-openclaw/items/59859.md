## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve non-cron schedules when editing jobs:** `apps/linux/src/section_cron.c:274-275`
  - body: The edit save path always sends `schedule_kind = "cron"` with `schedule_expr`, but current jobs can validly use `kind: "every"` or `kind: "at"`; saving an existing non-cron job after changing unrelated fields can rewrite or reject its schedule.
  - confidence: 0.92
- **[P2] Keep session-scoped cron targets on edit:** `apps/linux/src/test_seams.c:42-47`
  - body: Valid `session:<id>` targets fall through to index 0 and are later saved as `isolated`, so editing a session-scoped cron job can silently retarget it away from the intended session.
  - confidence: 0.9
- **[P2] Send supported verbose levels:** `apps/linux/src/section_sessions.c:176-184`
  - body: The sessions UI sends `none`, `low`, and `high`, but `sessions.patch` normalizes verbose overrides to `off`, `on`, and `full`; verbose changes from this app will be rejected.
  - confidence: 0.92
- **[P2] Resolve local session logs from session IDs:** `apps/linux/src/section_sessions.c:265-266`
  - body: The Log action builds `<storeDir>/<session-key>/transcript.jsonl`, while the gateway resolves transcripts from `sessionId`/`sessionFile` candidates; existing local transcripts will usually be missed and the app will fall back to the dashboard.
  - confidence: 0.88
- **[P2] Resolve cron logs through session transcript candidates:** `apps/linux/src/section_cron.c:187-190`
  - body: Cron log opening assumes `<cronStoreDir>/sessions/<sessionKey>/transcript.jsonl`, but cron store paths point at cron storage and session transcripts are resolved via the session transcript layout, so Cron Log often misses local files.
  - confidence: 0.83
- **[P2] Bracket IPv6 hosts before composing dashboard URLs:** `apps/linux/src/gateway_config.c:1153-1155`
  - body: Dashboard URLs interpolate `config->host` directly into `scheme://host:port`; IPv6 literals such as `::1` produce malformed URLs and break dashboard or log handoffs on IPv6 setups.
  - confidence: 0.86
- **[P2] Read channel account labels from name:** `apps/linux/src/gateway_data.c:262`
  - body: Channel account snapshots expose the friendly label as `name`, but this parser reads `displayName`, so connected account rows fall back to account IDs instead of showing user labels.
  - confidence: 0.91
