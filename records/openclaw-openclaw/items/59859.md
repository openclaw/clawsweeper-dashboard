## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Preserve cron schedule variants on edit:** `apps/linux/src/section_cron.c:274-275`
  - body: Saving an existing cron job always sends `schedule.kind = "cron"` with an `expr`. Current jobs may be `at` or `every`, and `applyJobPatch` replaces the stored schedule whenever `patch.schedule` is present, so an unrelated edit can rewrite or invalidate non-cron schedules.
  - confidence: 0.96
- **[P1] Keep session-scoped cron targets intact:** `apps/linux/src/test_seams.c:42-48`
  - body: Valid `session:<id>` cron targets are treated as unknown, displayed as the first combo option, and then saved back as `isolated`. Editing a job can silently move it away from its intended session even when the user changed another field.
  - confidence: 0.93
- **[P2] Send supported verbose levels:** `apps/linux/src/section_sessions.c:176-184`
  - body: The sessions UI sends `none`, `low`, and `high` as `verboseLevel`, but the gateway parser accepts `off`, `on`, and `full`. Verbose changes from this UI path will be rejected instead of updating the session.
  - confidence: 0.94
- **[P2] Resolve transcripts through the session file contract:** `apps/linux/src/section_sessions.c:264-266`
  - body: The local log path is built as `<store-dir>/<session-key>/transcript.jsonl`, but current transcript resolution uses `sessionId`, optional `sessionFile`, and per-agent session directories. This will usually miss local transcripts and force a dashboard fallback, which can fail offline or without dashboard access.
  - confidence: 0.91
- **[P2] Require an install option before calling skills.install:** `apps/linux/src/section_skills.c:87-88`
  - body: The install button can call `mutation_skills_install` without an `installId`, and that helper omits the field when it is missing. The current `skills.install` protocol requires a non-empty `installId`, so eligible skills without an attached option produce a schema failure instead of a useful install action.
  - confidence: 0.88
