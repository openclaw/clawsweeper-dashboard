## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve non-cron schedules when editing jobs:** `apps/linux/src/section_cron.c:274-275`
  - body: The edit save path always sends `schedule.kind = "cron"` and writes the textbox as `expr`. Current cron jobs may have `kind: "every"` or `kind: "at"`, so saving one after changing unrelated fields can rewrite or invalidate its schedule; carry the original schedule variant or prevent editing unsupported variants.
  - confidence: 0.92
- **[P2] Keep session-scoped cron targets on save:** `apps/linux/src/section_cron.c:351-352`
  - body: `sessionTarget` values like `session:<id>` are treated as unknown, selected as index 0, and then saved back as `isolated`. Editing a session-scoped job can therefore retarget it away from the intended session even when the user changes another field.
  - confidence: 0.9
- **[P2] Send supported verbose levels from the session selector:** `apps/linux/src/section_sessions.c:176-184`
  - body: The verbose selector sends `none`, `low`, and `high`, but `sessions.patch` accepts `off`, `on`, and `full`. Changing verbose from this UI path will be rejected by the gateway instead of updating the session.
  - confidence: 0.91
- **[P2] Resolve local session logs from session IDs:** `apps/linux/src/section_sessions.c:265-266`
  - body: This builds the local transcript path as `<storeDir>/<session-key>/transcript.jsonl`, but the gateway resolves transcripts from `sessionId`/`sessionFile` under the sessions directory. The Log action will usually miss existing local transcripts and fall back to the dashboard.
  - confidence: 0.85
- **[P2] Bracket IPv6 hosts before composing dashboard URLs:** `apps/linux/src/gateway_config.c:1153-1155`
  - body: Dashboard URLs interpolate `config->host` directly into `scheme://host:port`. IPv6 literals such as `::1` produce malformed URLs like `http://::1:18789/`, breaking dashboard and log handoffs on valid IPv6 setups.
  - confidence: 0.84

## Security Review

Status: needs_attention

Summary: This PR adds a first-party desktop surface for gateway auth, durable device identity/tokens, remote credentials, and exec approval decisions, so it needs focused platform/security review before merge.

Concerns:

- **[medium] Review durable device identity and token storage:** `apps/linux/src/device_identity.c:135`
  - body: The Linux app persists an Ed25519 private identity and gateway device tokens under the state directory with restrictive file modes; token lifetime, rotation, revocation, backup/migration behavior, and parity with other clients should receive explicit security review before shipping.
  - confidence: 0.86
- **[high] Review exec approval quick-mode auto-resolution:** `apps/linux/src/exec_approval_prompter.c:176`
  - body: The exec approval flow can persist a quick mode that auto-resolves requests as allow or deny. That changes a security-sensitive operator approval path and should have product/security signoff.
  - confidence: 0.84
