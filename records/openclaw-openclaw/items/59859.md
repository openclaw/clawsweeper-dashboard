## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve non-cron schedules when editing jobs:** `apps/linux/src/section_cron.c:274-275`
  - body: The edit save path always sends `patch.schedule` as `{ kind: "cron", expr: ... }`. Current cron jobs can be `every` or `at`, so saving one after changing unrelated fields rewrites the schedule shape or fails validation; carry the original variant through the dialog, or omit schedule unless it is explicitly edited.
  - confidence: 0.9
- **[P2] Keep session-scoped cron targets on edit:** `apps/linux/src/test_seams.c:42-47`
  - body: `session:<id>` is a valid cron `sessionTarget`, but the combo mapping treats every unknown target as index 0 and later saves that back as `isolated`. Editing a session-scoped job can silently retarget it away from the intended session.
  - confidence: 0.87
- **[P2] Send supported verbose levels:** `apps/linux/src/section_sessions.c:176-184`
  - body: The verbose selector sends `none`/`low`/`high`, but `sessions.patch` accepts `off`/`on`/`full`. Changing verbose from this UI path will be rejected by the gateway, so users cannot update session verbosity.
  - confidence: 0.93
- **[P2] Resolve local session logs from session IDs:** `apps/linux/src/section_sessions.c:265-266`
  - body: The Log action builds `<storeDir>/<sessionKey>/transcript.jsonl`, but the gateway stores transcripts from `sessionId` and `sessionFile` candidates such as `<sessionsDir>/<sessionId>.jsonl`. Normal local transcripts are missed and the action falls back to the dashboard even when the file exists.
  - confidence: 0.89
- **[P2] Bracket IPv6 hosts before composing URLs:** `apps/linux/src/gateway_config.c:1153-1155`
  - body: `gateway_config_dashboard_url` interpolates `config->host` directly into `scheme://host:port`. For an IPv6 literal such as `::1`, that produces malformed URLs like `http://::1:18789`, breaking dashboard and log handoff on valid IPv6/custom-bind setups.
  - confidence: 0.86
- **[P2] Read channel account labels from the current field:** `apps/linux/src/gateway_data.c:262`
  - body: `channels.status` account snapshots expose the human label as `name`, but this parser reads `displayName`. Normal connected accounts therefore fall back to account IDs instead of showing friendly labels in the Linux UI.
  - confidence: 0.88
- **[P2] Allow clearing skill env and API key values:** `apps/linux/src/section_skills.c:133-134`
  - body: The env/API-key dialog returns early for an empty string, but `skills.update` treats empty `apiKey` or env values as delete operations. This blocks users from clearing stale secrets or env overrides from the Linux app.
  - confidence: 0.9

## Security Review

Status: needs_attention

Summary: This PR introduces a security-sensitive native desktop client and needs focused security and supply-chain review before merge.

  - body: The app introduces durable Linux device-token storage under `identity/device-auth.json`; permissions are present, but token rotation, redaction, mismatch handling, and failure behavior need security-owner review before shipping.
  - confidence: 0.86
- **[high] Review device identity key persistence:** `apps/linux/src/device_identity.c:24`
  - body: The branch adds Ed25519 identity generation and persistence through libsodium; private key storage, migration, backup, and error modes are part of the device trust boundary and need security sign-off.
  - confidence: 0.84
- **[high] Review exec approval controls:** `apps/linux/src/exec_approval_prompter.c:4`
  - body: The branch subscribes to exec approval events and resolves operator decisions through gateway RPC; approval binding, expiry, replay resistance, and safe-denial behavior need focused review.
  - confidence: 0.85
- **[medium] Review SSH tunnel spawning:** `apps/linux/src/remote_tunnel_command.c:15`
  - body: Remote mode builds and spawns SSH tunnel argv from user-controlled connection settings; argv hardening exists, but this remains a command-execution boundary.
  - confidence: 0.82

  - body: The Meson build adds GTK/libadwaita/AppIndicator/libsoup/JSON-GLib/libsodium dependencies plus helper binaries; packaging, provenance, update flow, and helper isolation need supply-chain review.
