## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve non-cron schedules when saving edited jobs:** `apps/linux/src/section_cron.c:270-276`
  - body: The edit save path always serializes the schedule as `{ kind: "cron", expr: ... }`. Jobs created elsewhere can validly use `kind: "every"` or `kind: "at"`, so saving one after editing unrelated metadata can rewrite the schedule shape, fail validation, or change when it runs; carry the original schedule variant or block unsupported variants in this dialog.
  - confidence: 0.92
- **[P2] Keep session-scoped cron targets on edit:** `apps/linux/src/test_seams.c:42-47`
  - body: `sessionTarget` values such as `session:<id>` are valid, but this mapper treats unknown values as index 0, and the save path turns index 0 into `isolated`. Editing a session-scoped job can therefore retarget it away from the intended session even when the user changes another field.
  - confidence: 0.9
- **[P2] Send supported verbose levels from the session selector:** `apps/linux/src/section_sessions.c:175-184`
  - body: The verbose selector sends `none`, `low`, and `high` as `verboseLevel`, but current gateway normalization accepts `off`, `on`, and `full`. Changing verbose from this UI path will be rejected instead of updating the session.
  - confidence: 0.91
- **[P2] Bracket IPv6 hosts before composing dashboard URLs:** `apps/linux/src/gateway_config.c:1151-1155`
  - body: Dashboard URLs interpolate `config->host` directly into `scheme://host:port`. For an IPv6 literal such as `::1`, that produces malformed URLs like `http://::1:18789/`, breaking dashboard and log handoffs on valid IPv6/custom-bind setups.
  - confidence: 0.84
