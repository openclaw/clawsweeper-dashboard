## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve non-cron schedules when saving jobs:** `apps/linux/src/section_cron.c:274-275`
  - body: The edit save path always sends `schedule.kind: "cron"` with `expr`, but current jobs can use `kind: "every"` or `kind: "at"`. Saving an existing non-cron job after changing unrelated fields can fail validation or rewrite when the job runs, so carry the original schedule variant or block editing unsupported variants.
  - confidence: 0.94
- **[P2] Keep session-scoped cron targets on edit:** `apps/linux/src/test_seams.c:42-47`
  - body: Unknown targets, including valid `session:<id>` targets, map to combo index 0 and are later saved back as `isolated`. A no-op edit can silently retarget a job away from its intended session, so preserve session-scoped values or expose them explicitly.
  - confidence: 0.92
- **[P2] Send supported verbose values in session patches:** `apps/linux/src/section_sessions.c:176-184`
  - body: The verbose dropdown sends `none`, `low`, and `high`, but the gateway accepts `off`, `on`, or `full`. Verbose changes from the Linux app will be rejected by `sessions.patch`; map the UI choices to the supported values before dispatch.
  - confidence: 0.94
- **[P2] Resolve local session logs from session IDs:** `apps/linux/src/section_sessions.c:264-267`
  - body: The local Log action builds `<storeDir>/<session-key>/transcript.jsonl`, but the gateway resolves transcripts from `sessionId` or `sessionFile` as files under the sessions directory. This misses existing local transcripts and unnecessarily falls back to the dashboard.
  - confidence: 0.9
- **[P2] Bracket IPv6 hosts before composing dashboard URLs:** `apps/linux/src/gateway_config.c:1153-1155`
  - body: The URL builder interpolates `config->host` directly into `scheme://host:port`. IPv6 literals such as `::1` need brackets, otherwise health/dashboard handoff URLs are malformed for valid IPv6 gateway configs.
  - confidence: 0.86
- **[P2] Allow empty skill secret values to clear stored config:** `apps/linux/src/section_skills.c:133-134`
  - body: The env/API-key dialog returns early for empty input, but the gateway intentionally treats empty `apiKey` and env values as delete operations. This prevents users from clearing stale skill credentials or env overrides from the Linux app.
  - confidence: 0.88
- **[P2] Require an install option before enabling Install:** `apps/linux/src/section_skills.c:340-350`
  - body: The Install button is shown for any eligible uninstalled skill even when no `installId` is attached, but the non-ClawHub `skills.install` schema requires a non-empty `installId`. Hide/disable the button without an install option or send a supported install source.
  - confidence: 0.84

## Security Review

Status: needs_attention

Summary: The PR adds a new credentialed native desktop client with persistent device auth, exec-approval resolution, SSH tunnel control, and local file/gateway mutation surfaces that require focused security review.

Concerns:

- **[medium] Review persistent device-token storage:** `apps/linux/src/device_auth_store.c:4`
  - body: `device_auth_store.c` adds a durable operator token store under the Linux companion state directory. Permissions, token lifecycle, redaction, rotation, and parity with the shared device-auth contract need explicit security review before shipping.
  - confidence: 0.84
- **[medium] Review exec approval dispatch from the desktop app:** `apps/linux/src/exec_approval_prompter.c:4`
  - body: `exec_approval_prompter.c` subscribes to exec approval events and resolves operator decisions back to the gateway. This is a sensitive command-execution control path and should get focused authorization, expiry, and replay/lifetime review.
  - confidence: 0.86
- **[medium] Review SSH tunnel command boundaries:** `apps/linux/src/remote_tunnel_command.c:15`
  - body: `remote_tunnel_command.c` builds and launches SSH forwarding arguments from user/config input. The current dash-leading guards are useful, but host/user/identity trust boundaries and launch policy still need security review as part of accepting remote mode.
  - confidence: 0.78
