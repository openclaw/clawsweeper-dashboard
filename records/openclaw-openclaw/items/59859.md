## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve non-cron schedules when editing jobs:** `apps/linux/src/section_cron.c:274-275`
  - body: Saving an edit always sends `schedule.kind: "cron"` with `expr`, but current cron jobs can be `at`, `every`, or `cron`; opening an existing non-cron job and saving metadata can rewrite or invalidate its schedule. Preserve the original schedule variant unless the user explicitly changes it.
  - confidence: 0.91
- **[P2] Resolve local session logs from session IDs:** `apps/linux/src/section_sessions.c:265-266`
  - body: This lookup builds `<store_dir>/<session-key>/transcript.jsonl`, but current transcript files are resolved by `sessionId` as `<sessionsDir>/<sessionId>.jsonl` or topic variants. The local Log action will usually miss existing transcripts and fall back to the dashboard even when the file is present.
  - confidence: 0.9
- **[P2] Bracket IPv6 hosts before composing URLs:** `apps/linux/src/gateway_config.c:1031-1040`
  - body: The config parser accepts IPv6 host literals, but these URL builders interpolate `config->host` directly into `scheme://host:port`. For hosts like `::1`, this creates malformed URIs for health, WebSocket, and dashboard/log handoffs; bracket IPv6 literals before formatting URLs.
  - confidence: 0.86
