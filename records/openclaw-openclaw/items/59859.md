## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve non-cron schedules when saving edits:** `apps/linux/src/section_cron.c:274-275`
  - body: The edit path always sends `schedule.kind = "cron"` and an `expr`, but the current gateway schema also supports `at` and `every` schedules. Opening an existing non-cron job and saving unrelated fields can rewrite or invalidate its schedule, so the app must preserve the original schedule variant unless the user explicitly changes it.
  - confidence: 0.94
- **[P2] Resolve session logs from sessionId transcript paths:** `apps/linux/src/section_sessions.c:264-267`
  - body: The Log action builds `<store-dir>/<session-key>/transcript.jsonl`, while the gateway resolver uses `sessionId`/`sessionFile` candidates such as `<sessionsDir>/<sessionId>.jsonl`. Normal local transcripts will be missed, forcing a dashboard fallback even when the file is available locally.
  - confidence: 0.9
- **[P2] Bracket IPv6 hosts before composing URLs:** `apps/linux/src/gateway_config.c:1031-1040`
  - body: These URL builders interpolate `config->host` directly, so an IPv6 literal like `::1` becomes `http://::1:18789` instead of a valid bracketed URI. The sibling macOS dashboard code already has IPv6-origin coverage, so the Linux app should normalize host literals before building HTTP, WebSocket, and dashboard URLs.
  - confidence: 0.88
