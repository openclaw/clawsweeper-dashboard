## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve non-cron schedules when editing jobs:** `apps/linux/src/section_cron.c:274-275`
  - body: The edit save path always serializes `patch.schedule` as `{ kind: "cron", expr: ... }`. Current cron jobs can be `at` or `every`, and the gateway update path replaces `job.schedule`, so saving an existing non-cron job from this UI can fail validation or change when it runs. Carry the original schedule variant through the dialog or block editing unsupported variants.
  - confidence: 0.91
- **[P1] Keep session-scoped cron targets intact:** `apps/linux/src/test_seams.c:42-47`
  - body: `session_target_to_index()` maps every unrecognized target to 0, and index 0 is later saved as `isolated`. Since valid cron targets include `session:<id>`, editing a session-scoped job can silently retarget it away from the intended session even when the user changes an unrelated field.
  - confidence: 0.9
- **[P2] Resolve local logs from session transcript candidates:** `apps/linux/src/section_sessions.c:265-266`
  - body: The Log action probes `<storeDir>/<session-key>/transcript.jsonl`, but current transcript resolution uses the row `sessionId` and direct files under the sessions directory. This usually misses existing local transcripts and falls back to the dashboard, which breaks offline or no-dashboard-access setups.
  - confidence: 0.87
- **[P2] Bracket IPv6 hosts before composing dashboard URLs:** `apps/linux/src/gateway_config.c:1153-1155`
  - body: `gateway_config_dashboard_url()` interpolates the host directly into `scheme://host:port`, so an IPv6 literal such as `::1` becomes an invalid URL like `http://::1:18789/`. Normalize IPv6 literals to bracketed form before formatting dashboard and route URLs.
  - confidence: 0.84
- **[P3] Add Linux app labeler coverage:** `apps/linux/README.md:1`
  - body: This PR creates a new `apps/linux` app surface, but the repository labeler only has app rules for Android, iOS, macOS, and web UI. Add an `app: linux` rule and matching label follow-through so future Linux companion changes route to the right maintainer queue.
  - confidence: 0.82
