## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve the unit name before clearing the cache:** `apps/linux/src/systemd.c:490`
  - body: `current_unit` is a borrowed pointer from the cached canonical unit name. When the retarget branch calls `clear_cached_service_config()`, that cache can be freed before `g_strdup(current_unit)` uses it, so a unit retarget can corrupt the name or crash the app.
  - confidence: 0.93
- **[P1] Do not block connected manual gateways on systemd state:** `apps/linux/src/state.c:123-124`
  - body: This marks chat blocked whenever the user systemd service is inactive, before checking whether the gateway is already reachable over HTTP/WS/RPC. Users running `openclaw gateway run` manually can be fully connected but still have chat disabled by `CHAT_BLOCK_SERVICE_INACTIVE`.
  - confidence: 0.9
- **[P1] Send verbose levels accepted by sessions.patch:** `apps/linux/src/section_sessions.c:176-178`
  - body: The UI sends `none`/`low`/`high`, but the gateway verbose override parser accepts `off`/`on`/`full`. Session verbose changes from the Linux app will be rejected instead of updating the session.
  - confidence: 0.94
- **[P1] Preserve non-cron schedules when editing jobs:** `apps/linux/src/section_cron.c:419-420`
  - body: The edit path always serializes the schedule as `kind: cron` with an `expr`, so editing an existing `every` or `at` job can change its schedule shape or fail validation even when the user only edits unrelated fields.
  - confidence: 0.9
- **[P2] Read channel account labels from name:** `apps/linux/src/gateway_data.c:262`
  - body: Gateway channel account snapshots expose the friendly label as `name`, not `displayName`. Normal status responses will therefore lose account labels and fall back to raw account ids in the Linux UI.
  - confidence: 0.88
- **[P2] Allow empty skill values to clear stored secrets:** `apps/linux/src/section_skills.c:134`
  - body: The dialog returns before sending empty API key or env values, but the gateway treats empty strings as delete operations. That makes it impossible to clear stale skill secrets or env overrides from the app.
  - confidence: 0.87
