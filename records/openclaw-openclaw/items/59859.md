## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve non-cron schedules when editing jobs:** `apps/linux/src/section_cron.c:274-275`
  - body: The edit save path always writes `patch.schedule` as `{ kind: "cron", expr: ... }`, but current cron jobs may be `kind: "every"` or `kind: "at"`. Saving an existing non-cron job after changing unrelated fields can fail validation or change when it runs.
  - confidence: 0.92
- **[P2] Keep session-scoped cron targets on save:** `apps/linux/src/section_cron.c:264-266`
  - body: Unknown existing targets are selected as index 0, and the save path serializes that index back as `isolated`. Valid `session:<id>` cron targets can therefore be silently retargeted away from their intended session on edit save.
  - confidence: 0.9
- **[P2] Send supported verbose levels from the session selector:** `apps/linux/src/section_sessions.c:176-184`
  - body: The verbose selector sends `none`, `low`, and `high`, but `sessions.patch` validates verbose levels as `off`, `on`, or `full`. Changing verbose from this UI path will be rejected by the gateway.
  - confidence: 0.96
- **[P2] Resolve local session logs from session transcript files:** `apps/linux/src/section_sessions.c:265-266`
  - body: The Log action probes `<storeDir>/<session-key>/transcript.jsonl`, while the gateway resolves transcripts from the session id/sessionFile layout under the sessions directory. Normal local transcripts will be missed and fall back to the dashboard unnecessarily.
  - confidence: 0.87
- **[P2] Bracket IPv6 hosts before composing dashboard URLs:** `apps/linux/src/gateway_config.c:1153-1155`
  - body: Dashboard URLs interpolate `config->host` directly into `scheme://host:port`. For IPv6 literals such as `::1`, that creates malformed URLs and breaks health, WebSocket, or dashboard handoff paths on valid IPv6/custom-bind setups.
  - confidence: 0.84
- **[P2] Allow empty skill values to clear stored credentials:** `apps/linux/src/section_skills.c:133-134`
  - body: The env/API-key dialog returns early for an empty entry, but the gateway treats empty `apiKey` and env values as delete operations. Users cannot clear stale skill secrets or env overrides from the Linux UI.
  - confidence: 0.88
- **[P2] Require an install id before sending skills.install:** `apps/linux/src/section_skills.c:87-88`
  - body: The install button can call `mutation_skills_install` with a null `installId`, and the mutation then omits the field. The non-ClawHub `skills.install` schema requires a non-empty `installId`, so this path can produce a validation failure instead of a useful install action.
  - confidence: 0.82
