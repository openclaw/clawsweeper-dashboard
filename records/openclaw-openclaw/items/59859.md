Full review comments:

- **[P1] Send valid sessions.patch verbose levels:** `apps/linux/src/section_sessions.c:176-178`
  - body: The verbose dropdown sends `none`, `low`, and `high`, but the current gateway parser only accepts `off`, `on`, and `full`. Any verbose change from the Linux Sessions UI will be rejected instead of updating the session.
  - confidence: 0.96
- **[P1] Parse skills.status using current field names:** `apps/linux/src/gateway_data.c:345-351`
  - body: `skills.status` entries expose `skillKey` and `disabled`/eligibility fields, not `key`, `enabled`, or `installed`. This leaves skill keys null and button/filter state wrong, so enable/install/update actions either no-op or render misleading state.
  - confidence: 0.94
- **[P1] Preserve cron schedule and session target on edit:** `apps/linux/src/section_cron.c:264-275`
  - body: Saving an edited job always rebuilds the patch as `schedule.kind = "cron"` and maps unsupported combo targets back through the limited index set. Existing `at`/`every` jobs or `session:<id>` targets can be silently rewritten when the user saves unrelated fields.
  - confidence: 0.92
- **[P2] Allow empty skill secrets to clear stored values:** `apps/linux/src/section_skills.c:133-134`
  - body: The dialog returns before sending empty values, but the gateway intentionally treats empty `apiKey` and env values as deletes. Users cannot remove stale skill credentials or env overrides from the Linux app once set.
  - confidence: 0.9
