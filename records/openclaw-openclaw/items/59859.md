## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve non-cron schedules when editing jobs:** `apps/linux/src/section_cron.c:274-275`
  - body: The edit save path always sends `schedule_kind = "cron"` and writes the textbox as a cron expression. Current cron jobs can be `kind: "every"` or `kind: "at"`, so saving an existing non-cron job after changing unrelated fields rewrites its schedule shape and can fail validation or change when it runs.
  - confidence: 0.9
- **[P2] Resolve local session logs from session IDs:** `apps/linux/src/section_sessions.c:265-266`
  - body: This builds the local transcript path as `<storeDir>/<session-key>/transcript.jsonl`, but the gateway stores and resolves transcripts as direct session-id files under the sessions directory. The Log action will usually miss local transcripts and fall back to the dashboard, which breaks offline or no-dashboard-access cases.
  - confidence: 0.87
- **[P2] Bracket IPv6 hosts before composing URLs:** `apps/linux/src/gateway_config.c:1153-1155`
  - body: The URL builder interpolates `config->host` directly into `scheme://host:port`. For IPv6 literals such as `::1`, that produces malformed URLs like `http://::1:18789`, breaking health checks, WebSocket connection, and dashboard handoff on valid IPv6/custom-bind setups.
  - confidence: 0.84
