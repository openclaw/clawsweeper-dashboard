  - body: The Linux app sends `openclaw-linux`, but this PR adds `LINUX_APP` under `src/gateway/protocol/client-info.ts` while current main imports and validates client ids from `packages/gateway-protocol/src/client-info.ts`.   - confidence: 0.93
- **[P1] Send supported verboseLevel values:** `apps/linux/src/section_sessions.c:176-178`
  - body: The verbose combo sends `none`, `low`, or `high` to `sessions.patch`, but the current gateway parser only accepts `off`, `on`, or `full`. These UI updates will be rejected instead of changing the session; map the combo to the supported values before sending the mutation.
  - confidence: 0.91
- **[P1] Preserve existing cron schedule variants:** `apps/linux/src/section_cron.c:274-275`
  - body: Saving the edit dialog always writes `patch.schedule` as `{kind:"cron", expr:...}` even for jobs listed with `kind:"every"` or `kind:"at"`. Editing metadata on those jobs can rewrite or reject their schedule, so the dialog needs to preserve the original schedule shape unless the user explicitly changes it.
  - confidence: 0.9
- **[P2] Keep session-scoped cron targets on edit:** `apps/linux/src/test_seams.c:47`
  - body: Valid `session:<id>` cron targets are treated as unknown, displayed as index 0, and then saved back as `isolated`. That silently retargets session-scoped jobs when another field is edited; preserve the original session target or make it explicit in the UI.
  - confidence: 0.86
