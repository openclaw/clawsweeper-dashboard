Full review comments:

- **[P1] Preserve non-cron schedules when editing jobs:** `apps/linux/src/section_cron.c:274-275`
  - body: The edit path always sends schedule_kind = "cron" with the textbox as expr. Current cron jobs may be kind "at" or "every", so saving an unrelated edit can rewrite or invalidate an existing job schedule.
  - confidence: 0.94
- **[P1] Send supported verbose override values:** `apps/linux/src/section_sessions.c:176-178`
  - body: The verbose dropdown sends "none", "low", or "high", but current sessions.patch parsing accepts only "off", "on", or "full". User attempts to change verbose behavior from the Linux app will be rejected by the gateway.
  - confidence: 0.93
- **[P2] Resolve local logs from session transcript candidates:** `apps/linux/src/section_sessions.c:265-266`
  - body: The local log opener builds <store_dir>/<session-key>/transcript.jsonl, but current transcript resolution is sessionId/sessionFile based. This will miss normal local transcript files and fall back to the dashboard even when a local log exists.
  - confidence: 0.88
- **[P2] Bracket IPv6 hosts before formatting dashboard URLs:** `apps/linux/src/gateway_config.c:1153-1155`
  - body: gateway_config_dashboard_url interpolates the raw host into scheme://host:port, so IPv6 literals such as ::1 produce malformed URLs like http://::1:18789/. Dashboard and transcript handoffs break for IPv6-based local gateway configs.
  - confidence: 0.86
