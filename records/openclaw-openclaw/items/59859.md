## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P1] Preserve non-cron schedule variants on edit:** `apps/linux/src/section_cron.c:270-275`
  - body: Current cron.update accepts `at`, `every`, and `cron` schedules, but this edit path always sends `patch.schedule` as `{ kind: "cron", expr: ... }`. Opening a valid `every` or `at` job and saving an unrelated change can rewrite the schedule shape or fail validation.
  - confidence: 0.93
- **[P1] Keep custom cron session targets through edits:** `apps/linux/src/test_seams.c:42-47`
  - body: Valid cron jobs may use `session:<id>` targets, but this mapper treats any unrecognized target as index 0, which the edit save path later writes back as `isolated`. Editing a session-scoped job can silently retarget future runs away from the intended session.
  - confidence: 0.9
- **[P2] Move Linux product state to canonical storage:** `apps/linux/src/product_state.c:33-40`
  - body: This adds OpenClaw-owned app state in `product-state.ini` plus an `onboarding_version` marker under a sidecar state directory. The repo storage policy says app state belongs in SQLite or another designed state owner, so landing this creates a second migration surface for connection mode, onboarding, and heartbeat preferences.
  - confidence: 0.82
- **[P3] Add Linux app labeler coverage:** `apps/linux/README.md:1`
  - body: Adding `apps/linux/**` creates a new app surface, but the PR does not update `.github/labeler.yml` with a Linux app label rule. Future Linux companion changes will not route through the app labeler coverage required by the repo policy.
  - confidence: 0.88

## Security Review
