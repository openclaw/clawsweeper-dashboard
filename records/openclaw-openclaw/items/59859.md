- **[P1] Move the Linux client id to the current protocol package:** `src/gateway/protocol/client-info.ts:13`
  - body: This adds `LINUX_APP` only in `src/gateway/protocol/client-info.ts`, but current main's accepted client-id registry now lives in `packages/gateway-protocol/src/client-info.ts`. After rebase, the Linux app would still miss the live handshake schema unless the new id is added to the current package and its tests/docs are updated there.
  - confidence: 0.91
- **[P1] Send supported session verbose levels:** `apps/linux/src/section_sessions.c:176-184`
  - body: The sessions UI sends `none`/`low`/`high` to `sessions.patch`, while the current parser accepts only `off`/`on`/`full`. Verbose changes from the Linux app will be rejected at runtime instead of updating the session.
  - confidence: 0.94
- **[P1] Preserve cron schedule and target variants on edit:** `apps/linux/src/section_cron.c:270-279`
  - body: Saving an edited job always serializes the schedule as `{kind: "cron", expr: ...}` and the target combo cannot preserve `session:<id>` targets. Editing an existing `at`/`every` or session-scoped job can fail validation or silently rewrite scheduling/routing behavior.
  - confidence: 0.92
- **[P2] Resolve local transcripts from session ids:** `apps/linux/src/section_sessions.c:264-267`
  - body: The local log probe builds `<store-dir>/<session-key>/transcript.jsonl`, but current gateway transcript resolution is based on `sessionId`/`sessionFile` candidates. This will miss real local transcripts and force dashboard fallback, including cases where offline or no-dashboard access should still open the local log.
  - confidence: 0.88
- **[P2] Allow empty skill settings to clear values:** `apps/linux/src/section_skills.c:133-134`
  - body: The dialog returns before sending an empty env/API-key value, but current `skills.update` treats empty strings as delete operations for stored `apiKey` and env entries. Users cannot clear stale skill secrets or overrides from this UI once set.
  - confidence: 0.9
