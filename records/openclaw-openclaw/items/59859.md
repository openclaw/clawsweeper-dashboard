## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Preserve non-cron schedules when editing jobs:** `apps/linux/src/section_cron.c:274-275`
  - body: The edit save path always sends `schedule.kind = "cron"`, but current cron jobs may use `kind: "every"` or `kind: "at"`. Saving an existing non-cron job after changing unrelated fields can rewrite its schedule shape, fail validation, or change when it runs; carry the original schedule variant through the dialog or block unsupported variants.
  - confidence: 0.91
- **[P2] Resolve local session logs from session IDs:** `apps/linux/src/section_sessions.c:265-266`
  - body: This probes `<storeDir>/<session-key>/transcript.jsonl`, but the gateway transcript contract resolves direct session-id/sessionFile candidates and the PR parser already exposes `session_id`. The Log action will usually miss existing local transcripts and fall back to the dashboard, which breaks offline or no-dashboard-access use.
  - confidence: 0.88
- **[P2] Bracket IPv6 hosts before composing URLs:** `apps/linux/src/gateway_config.c:1153-1155`
  - body: Interpolating `config->host` directly into `scheme://host:port` creates malformed URLs for IPv6 literals such as `::1`. Normalize literal IPv6 hosts to `[::1]` before building dashboard, health, or WebSocket URLs.
  - confidence: 0.84
- **[P2] Move product state out of the INI sidecar:** `apps/linux/src/product_state.c:33-36`
  - body: The PR persists OpenClaw-owned app state in `product-state.ini` under the user state dir. Root policy requires runtime/app state to use the shared state DB, plugin KV, agent DB, or a designed SQLite schema, so shipping this sidecar creates a new migration and upgrade contract outside the canonical store.
  - confidence: 0.82
