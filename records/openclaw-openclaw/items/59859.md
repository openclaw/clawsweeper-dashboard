## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve cron schedule variants when saving edits:** `apps/linux/src/section_cron.c:270-275`
  - body: This save path always serializes edited jobs as `schedule.kind = "cron"`. Existing jobs with `at` or `every` schedules can be rewritten or rejected when the user saves unrelated edits, so preserve the original schedule variant unless the user explicitly changes it.
  - confidence: 0.9
- **[P2] Resolve session logs through transcript candidates:** `apps/linux/src/section_sessions.c:264-267`
  - body: The local log opener assumes transcripts live under `<store_dir>/<session-key>/transcript.jsonl`, but the gateway resolver uses sessionId/sessionFile candidate paths. In normal installs this misses existing local transcripts and falls back to the dashboard or fails offline.
  - confidence: 0.86
- **[P2] Bracket IPv6 literals before composing dashboard URLs:** `apps/linux/src/gateway_config.c:1153-1155`
  - body: Interpolating an IPv6 literal directly into `scheme://host:port` produces malformed URIs such as `http://::1:18789/`. Bracket IPv6 hosts before launching dashboard or transcript URLs.
  - confidence: 0.84
- **[P2] Use canonical SQLite storage for companion state:** `apps/linux/src/product_state.c:33-40`
  - body: This adds OpenClaw-owned runtime state in INI sidecar files under the user state directory. Repository policy requires runtime app state to live in the shared/plugin/agent SQLite stores or a justified SQLite owner, so this should not introduce a new sidecar state path.
  - confidence: 0.82
