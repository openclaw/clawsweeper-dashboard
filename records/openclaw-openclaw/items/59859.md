## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve non-cron schedules when editing jobs:** `apps/linux/src/section_cron.c:274-275`
  - body: The edit-save path always sends `patch.schedule` as `{ kind: "cron", expr: ... }`. Current cron jobs may also be `every` or `at`, so saving an existing non-cron job after changing unrelated fields rewrites its schedule shape and can fail validation or change when the job runs.
  - confidence: 0.91
- **[P1] Send supported verbose levels from Sessions:** `apps/linux/src/section_sessions.c:176-178`
  - body: The Sessions verbose combo sends `none`, `low`, or `high`, but `sessions.patch` accepts verbose overrides as `off`, `on`, or `full`. Selecting any value from this UI path will be rejected, so users cannot change session verbose behavior from the Linux app.
  - confidence: 0.94
- **[P2] Resolve local session logs from session IDs:** `apps/linux/src/section_sessions.c:265-266`
  - body: This builds the local transcript path as `<storeDir>/<sessionKey>/transcript.jsonl`, but gateway transcript lookup uses session IDs and direct session files under the sessions directory. The PR parser already keeps `session_id`, so the Log action should use that contract instead of the session key path.
  - confidence: 0.88
- **[P2] Bracket IPv6 hosts before composing URLs:** `apps/linux/src/gateway_config.c:1153-1155`
  - body: The URL builder interpolates `config->host` directly into `scheme://host:port`. For an IPv6 literal such as `::1`, this produces malformed URLs like `http://::1:18789`, breaking health checks, WebSocket connection, and dashboard handoff on valid IPv6/custom-bind setups.
  - confidence: 0.84
