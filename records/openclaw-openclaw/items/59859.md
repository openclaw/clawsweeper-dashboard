## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve existing cron schedule variants on edit:** `apps/linux/src/section_cron.c:274-275`
  - body: The edit dialog always sends `patch.schedule` as `{kind: "cron", expr: ...}`. Current cron jobs may use `kind: "at"` or `kind: "every"`, so saving an edit for an existing non-cron job rewrites the schedule shape or fails validation even when the user only changed metadata. Preserve the existing schedule variant unless the user explicitly changes it.
  - confidence: 0.91
- **[P1] Preserve session-scoped cron targets on edit:** `apps/linux/src/test_seams.c:47`
  - body: Unknown `sessionTarget` values are mapped to index 0 and later serialized as `isolated`. The current gateway contract accepts opaque `session:<id>` targets, so opening and saving a session-scoped cron job silently retargets it away from the intended session.
  - confidence: 0.9
- **[P2] Resolve local transcripts using sessionId candidates:** `apps/linux/src/section_sessions.c:266`
  - body: The local Log probe builds `<dirname(storePath)>/<sessionKey>/transcript.jsonl`, but current transcript resolution is based on `sessionId`, optional `sessionFile`, `storePath`, and `agentId`. This will miss normal local transcript files and fall back to the dashboard even when a local transcript exists.
  - confidence: 0.86
