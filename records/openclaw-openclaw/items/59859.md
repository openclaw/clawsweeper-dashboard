## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve cron schedule variants when saving edits:** `apps/linux/src/section_cron.c:270-275`
  - body: The edit path always sends `schedule_kind = "cron"`, so opening an existing `every` or `at` job and saving unrelated fields rewrites its schedule into a cron-shaped patch. Current cron jobs support non-cron variants, so this can fail validation or silently change scheduled behavior.
  - confidence: 0.9
- **[P2] Resolve local session logs from session IDs:** `apps/linux/src/section_sessions.c:264-266`
  - body: This builds local transcript paths as `<store_dir>/<session key>/transcript.jsonl`, but the gateway resolves transcripts from `sessionId`, `sessionFile`, and store-path candidates. The Log action will usually miss existing local transcripts and fall back to the dashboard or fail offline.
  - confidence: 0.88
- **[P2] Bracket IPv6 literals before composing URLs:** `apps/linux/src/gateway_config.c:1153-1155`
  - body: Interpolating `config->host` directly into `scheme://host:port` creates malformed URLs for IPv6 literals such as `::1`. IPv6 gateway setups can therefore break dashboard and related URL handoffs even when the gateway is reachable.
  - confidence: 0.84
- **[P2] Use canonical storage for companion product state:** `apps/linux/src/product_state.c:33-40`
  - body: The companion writes persistent OpenClaw-owned product state to an INI file under the user state dir. Repository policy requires runtime state to use the canonical SQLite-backed stores unless a named product artifact contract exists, so landing this creates new migration debt before the app ships.
  - confidence: 0.72
