Overall confidence: 0.88

Full review comments:

- **[P1] Register the Linux id in the shipped protocol package:** `src/gateway/protocol/client-info.ts:13`
  - body: This adds `LINUX_APP` only to `src/gateway/protocol/client-info.ts`, but current main validates connect params against the closed enum from `packages/gateway-protocol/src/client-info.ts`. After rebase, `openclaw-linux` is still absent from the canonical package, so the Linux app's first WebSocket connect frame is rejected.
  - confidence: 0.91
- **[P1] Preserve cron schedule variants when editing jobs:** `apps/linux/src/section_cron.c:274-275`
  - body: The edit save path always sends `schedule_kind = "cron"` and puts the textbox into `schedule_expr`. Existing jobs may have `kind: "at"` or `kind: "every"`, so saving unrelated edits can rewrite or invalidate their schedule instead of preserving the current variant.
  - confidence: 0.9
- **[P1] Preserve session-scoped cron targets on edit:** `apps/linux/src/test_seams.c:42-47`
  - body: The current cron schema accepts `session:<id>`, but this mapper treats any unknown value as index 0, which is saved back as `isolated`. Editing a session-scoped job can silently retarget it away from its intended session.
  - confidence: 0.89
- **[P1] Send supported verbose override values:** `apps/linux/src/section_sessions.c:175-184`
  - body: The Sessions UI sends `none`, `low`, or `high` as `verboseLevel`, while current `sessions.patch` parsing accepts `off`, `on`, or `full`. Verbose changes from this UI path will be rejected at runtime.
  - confidence: 0.88
- **[P2] Resolve local session logs from session ids:** `apps/linux/src/section_sessions.c:265-266`
  - body: This probes `<storeDir>/<session-key>/transcript.jsonl`, but gateway transcript resolution uses session ids/sessionFile candidates such as direct `<sessionsDir>/<sessionId>.jsonl` files. The Log action will usually miss existing local transcripts and fall back to the dashboard.
  - confidence: 0.87
- **[P2] Bracket IPv6 hosts before composing URLs:** `apps/linux/src/gateway_config.c:1153-1155`
  - body: The URL builder interpolates `config->host` directly into `scheme://host:port`. IPv6 literals such as `::1` produce malformed URLs like `http://::1:18789/`, breaking health checks, WebSocket connection, and dashboard handoff for valid IPv6/custom-bind setups.
  - confidence: 0.84
- **[P3] Add labeler coverage for the Linux app:** `apps/linux/README.md:1`
  - body: The PR introduces a new top-level app surface under `apps/linux`, but `.github/labeler.yml` is unchanged and currently has app labels for Android, iOS, and macOS only. Add Linux app routing so future changes reach the right maintainer queue.
  - confidence: 0.8
