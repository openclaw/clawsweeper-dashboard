## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Resolve SecretRef credentials instead of rejecting them:** `apps/linux/src/gateway_config.c:369-390`
  - body: Current gateway auth supports SecretInput/SecretRef values for token/password, but this loader turns any SecretRef-backed gateway config into `GW_CFG_ERR_SECRET_REF_UNSUPPORTED`. Existing users with SecretRef auth would have to weaken their setup or lose Linux app connectivity, so the app needs to resolve the supported credential contract or defer the flow behind an approved UX.
  - confidence: 0.9
- **[P1] Send the verbose values accepted by sessions.patch:** `apps/linux/src/section_sessions.c:176-184`
  - body: The UI serializes verbose choices as `none`/`low`/`high`, while the server-side override parser accepts `off`/`on`/`full`. As a result, changing verbose from the Linux Sessions view is rejected at runtime instead of updating the session.
  - confidence: 0.92
- **[P1] Preserve existing cron schedule variants when saving:** `apps/linux/src/section_cron.c:270-276`
  - body: The edit path always writes `schedule.kind = "cron"` and serializes the textbox as `expr`, so saving an existing `every` or `at` job can rewrite its schedule shape or fail validation even when the user only edited metadata. Keep the original schedule variant unless the UI explicitly supports changing it.
  - confidence: 0.86
- **[P2] Resolve local logs with session transcript candidates:** `apps/linux/src/section_sessions.c:264-267`
  - body: This local path assumes transcripts live under `<sessions-store-dir>/<session-key>/transcript.jsonl`, but current gateway resolution uses session id/session file candidates under the sessions directory. The Log action will miss real local transcripts and fall back to the dashboard or fail offline.
  - confidence: 0.84
- **[P2] Bracket IPv6 hosts before composing URLs:** `apps/linux/src/gateway_config.c:1153-1155`
  - body: The dashboard URL builder interpolates `config->host` directly into `scheme://host:port`, so an IPv6 literal such as `::1` becomes malformed. Normalize IPv6 literals to bracketed form before building dashboard, route, health, and WebSocket URLs that use this host.
  - confidence: 0.84
- **[P3] Add labeler coverage for the Linux app surface:** `apps/linux/README.md:1`
  - body: This PR introduces `apps/linux/**`, but current labeler coverage only routes existing app surfaces. Add the Linux app labeler/label wiring required for new app surfaces so future PRs and issues are triaged to the right maintainers.
  - confidence: 0.74
