## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Preserve schedule variants on cron edit save:** `apps/linux/src/section_cron.c:270-275`
  - body: The edit path always sends `patch.schedule` as `{ kind: "cron", expr: ... }`. Current cron jobs can be `at`, `every`, or `cron`, so saving an existing non-cron job to change only its name/description rewrites or rejects its schedule. Preserve the existing schedule variant unless the user explicitly changes schedule type.
  - confidence: 0.91
- **[P2] Allow empty skill values to clear secrets:** `apps/linux/src/section_skills.c:133-134`
  - body: The gateway intentionally treats empty `apiKey` and env strings as delete requests, but this handler returns before dispatching `skills.update` when the entry is empty. That leaves users unable to clear stale API keys or env overrides from the Linux app.
  - confidence: 0.9
- **[P2] Align local usage dates with server date mode:** `apps/linux/src/section_usage.c:101-108`
  - body: This builds `startDate` and `endDate` from local dates but omits `mode`; the server defaults explicit dates to UTC. Users outside UTC near day boundaries can see the wrong usage window, so either build UTC dates or send an explicit gateway/specific date mode with offset.
  - confidence: 0.86
- **[P2] Read channel account labels from name:** `apps/linux/src/gateway_data.c:262`
  - body: `channels.status` account snapshots expose the friendly account label as `name`, not `displayName`. Reading only `displayName` drops normal account labels and makes the UI fall back to account ids for configured channels.
  - confidence: 0.88
