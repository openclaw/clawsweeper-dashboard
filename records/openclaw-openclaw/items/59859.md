Full review comments:

- **[P1] Register the Linux client id in the canonical package:** `src/gateway/protocol/client-info.ts:13`
  - body: Current main validates gateway handshakes against `packages/gateway-protocol/src/client-info.ts`, but this PR adds `LINUX_APP` only under the stale `src/gateway/protocol` path. After rebasing, `openclaw-linux` remains unknown to the real gateway protocol registry, so the Linux app can fail to connect.
  - confidence: 0.92
- **[P1] Preserve non-cron schedules when editing jobs:** `apps/linux/src/section_cron.c:274-275`
  - body: The edit save path always sends `schedule_kind = "cron"` and serializes the textbox as `expr`, but current cron jobs can be `at`, `every`, or `cron`. Saving an existing `every` or `at` job, even just to rename it, will rewrite or fail its schedule.
  - confidence: 0.94
- **[P1] Preserve session-scoped cron targets on edit:** `apps/linux/src/test_seams.c:42-48`
  - body: `session:<id>` is a valid cron target, but this mapping treats any unknown value as index 0 and the edit path later saves it back as `isolated`. That silently retargets session-scoped jobs when the user saves unrelated edits.
  - confidence: 0.9
- **[P2] Resolve local session logs from session ids:** `apps/linux/src/section_sessions.c:264-267`
  - body: The local log probe builds `<storeDir>/<session-key>/transcript.jsonl`, while current main resolves transcripts from `sessionId` and `sessionFile` candidates. The Log action will miss real local transcript files and unnecessarily fall back to the dashboard.
  - confidence: 0.86
- **[P2] Bracket IPv6 hosts before composing URLs:** `apps/linux/src/gateway_config.c:1153-1155`
  - body: `gateway_config_dashboard_url` interpolates `host` directly into `scheme://host:port`, so IPv6 literals like `::1` produce malformed URLs such as `http://::1:18789/`. Bracket IPv6 literals before composing dashboard, health, or WebSocket URLs.
  - confidence: 0.82
