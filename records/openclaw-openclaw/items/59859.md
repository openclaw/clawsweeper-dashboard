## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve existing cron schedule variants:** `apps/linux/src/section_cron.c:270-275`
  - body: The edit save path always sends `schedule.kind: "cron"` with `expr`, but current cron jobs may use `kind: "every"` or `kind: "at"`. Saving an existing non-cron job after changing unrelated fields can fail validation or change when it runs, so carry the original schedule variant or block unsupported edits.
  - confidence: 0.94
- **[P2] Send supported verbose override values:** `apps/linux/src/section_sessions.c:175-184`
  - body: The verbose dropdown sends `none`, `low`, and `high`, but `sessions.patch` validation accepts `off`, `on`, or `full`. Verbose changes from the Linux app will be rejected until the UI maps to the supported wire values.
  - confidence: 0.94
- **[P2] Resolve local session logs from session IDs:** `apps/linux/src/section_sessions.c:264-266`
  - body: This builds the local transcript path as `<storeDir>/<session-key>/transcript.jsonl`, while the gateway resolves transcripts from `sessionId` or `sessionFile` under the sessions directory. Use the parsed `session_id`/session file contract so Log does not miss local transcripts and fall back unnecessarily.
  - confidence: 0.9
- **[P2] Bracket IPv6 hosts before composing URLs:** `apps/linux/src/gateway_config.c:1151-1155`
  - body: The dashboard URL builder interpolates `config->host` directly into `scheme://host:port`. IPv6 literals such as `::1` produce malformed URLs like `http://::1:18789/`, breaking dashboard and handoff links for valid IPv6 gateway configs.
  - confidence: 0.86
- **[P2] Allow empty skill values to clear config:** `apps/linux/src/section_skills.c:133-134`
  - body: The env/API-key dialog returns early for empty input, but the gateway intentionally treats empty `apiKey` and env values as delete operations. This prevents users from clearing stale skill credentials or env overrides from the Linux app.
  - confidence: 0.88
- **[P2] Require an install option before enabling Install:** `apps/linux/src/section_skills.c:339-350`
  - body: The Install button is shown for any eligible uninstalled skill even when no `installId` is attached, but the non-ClawHub `skills.install` schema requires a non-empty `installId`. Hide or disable the button without an install option, or send a supported install source.
  - confidence: 0.84
- **[P2] Read channel account labels from `name`:** `apps/linux/src/gateway_data.c:260-263`
  - body: `channels.status` account snapshots expose the friendly label as `name`, but this parser reads `displayName`. Normal connected accounts will lose their human-readable label and fall back to account IDs in the Linux UI.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The PR adds a credentialed native desktop client with persistent auth material, exec-approval decisions, SSH tunnel control, local file opening, and gateway mutations, so focused security review is required before merge.

Concerns:

- **[medium] Review persistent device auth storage:** `apps/linux/src/device_auth_store.c:222`
  - body: The Linux companion persists operator device tokens in a local `device-auth.json` store. The code sets restrictive permissions, but token lifecycle, rotation, redaction, backup exposure, and parity with the shared device-auth contract need security sign-off before shipping.
  - confidence: 0.86
- **[medium] Review exec approval resolution from the app:** `apps/linux/src/exec_approval_prompter.c:88`
  - body: The companion subscribes to exec approval requests and resolves command-execution decisions through `exec.approval.resolve`. Authorization, expiry, replay handling, prompt spoofing, and cross-client resolution semantics need focused review.
  - confidence: 0.86
- **[medium] Review SSH tunnel control behavior:** `apps/linux/src/remote_tunnel_command.c:15`
  - body: The app builds and runs SSH local-forwarding commands for remote connection mode. Even with argv-smuggling guards present, host-key policy, identity handling, lifecycle cleanup, and remote endpoint trust need security review.
  - confidence: 0.82
