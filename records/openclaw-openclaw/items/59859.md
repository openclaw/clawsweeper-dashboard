## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve cron schedule variants when editing:** `apps/linux/src/section_cron.c:274-275`
  - body: The edit dialog always saves `.schedule_kind = "cron"`, but gateway cron jobs can be `at`, `every`, or `cron`. Saving an existing non-cron job from the Linux app would rewrite or invalidate its schedule even when the user only edits metadata.
  - confidence: 0.92
- **[P1] Preserve session-scoped cron targets:** `apps/linux/src/section_cron.c:351-352`
  - body: The edit dialog maps the existing target through `session_target_to_index`, which treats unknown values as New Session, but the gateway accepts `session:<id>` targets. Editing a session-scoped job would silently retarget it away from the intended session.
  - confidence: 0.88
- **[P2] Send supported verbose levels in sessions.patch:** `apps/linux/src/section_sessions.c:175-184`
  - body: The verbose combo sends `none`, `low`, or `high`, while current gateway parsing accepts `off`, `on`, or `full`. This makes verbose updates from the Linux Sessions view fail validation at runtime.
  - confidence: 0.9
- **[P2] Resolve local transcripts with session ids:** `apps/linux/src/section_sessions.c:264-267`
  - body: The local log opener builds `<storeDir>/<sessionKey>/transcript.jsonl`, but current gateway transcript resolution is based on `sessionId` plus optional `sessionFile` candidates. Normal local transcripts can be missed, causing unnecessary dashboard fallback or offline failure.
  - confidence: 0.86
- **[P2] Bracket IPv6 hosts before composing URLs:** `apps/linux/src/gateway_config.c:1151-1154`
  - body: The URL builder interpolates `config->host` directly into `scheme://host:port`, so IPv6 literals produce invalid URLs such as `http://::1:18789/`. Normalize IPv6 literals to `[::1]` before building dashboard, HTTP, and WebSocket URLs.
  - confidence: 0.86
- **[P2] Bracket IPv6 hosts before composing URLs:** `apps/linux/src/gateway_config.c:939-944`
  - body: The HTTP, WebSocket, and dashboard URL builders interpolate `config->host` directly into `scheme://host:port`. For IPv6 literals such as `::1`, that produces malformed URLs like `http://::1:18789`, breaking health checks, WebSocket connections, and dashboard handoffs on valid IPv6 setups.
  - confidence: 0.84
