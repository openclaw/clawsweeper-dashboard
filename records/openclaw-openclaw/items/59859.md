## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve non-cron schedules when editing jobs:** `apps/linux/src/section_cron.c:274-275`
  - body: The edit save path always sends `schedule_kind = "cron"` and writes the entry text as `expr`. Jobs created elsewhere can use `kind: "every"` or `kind: "at"`, so saving an unrelated edit can rewrite or invalidate the schedule instead of preserving the existing variant.
  - confidence: 0.91
- **[P2] Keep session-scoped cron targets on edit:** `apps/linux/src/test_seams.c:42-47`
  - body: Valid cron targets include `session:<id>`, but this mapper treats any unrecognized value as index 0, which the save path sends back as `isolated`. Editing a session-scoped job can silently retarget it away from the intended session even when the user changes another field.
  - confidence: 0.9
- **[P2] Send supported verbose levels:** `apps/linux/src/section_sessions.c:176-184`
  - body: The sessions UI sends `none`, `low`, and `high` for `verboseLevel`, but the server normalizer accepts `off`, `on`, and `full`. Verbose changes from this app will be rejected until the combo values are mapped to the server contract.
  - confidence: 0.92
- **[P2] Resolve local session logs from session IDs:** `apps/linux/src/section_sessions.c:265-266`
  - body: This builds the local transcript path as `<storeDir>/<session-key>/transcript.jsonl`, but gateway transcript lookup is based on `sessionId` or `sessionFile` candidates. The Log action will usually miss existing local transcripts and fall back to the dashboard, which breaks offline or no-dashboard access.
  - confidence: 0.87
- **[P2] Bracket IPv6 hosts before composing dashboard URLs:** `apps/linux/src/gateway_config.c:1153-1155`
  - body: The dashboard URL builder interpolates `config->host` directly into `scheme://host:port`. For an IPv6 literal such as `::1`, this produces malformed URLs like `http://::1:18789/`, breaking dashboard and log handoff on valid IPv6/custom-bind setups.
  - confidence: 0.84
- **[P2] Read channel account labels from name:** `apps/linux/src/gateway_data.c:262`
  - body: `channels.status` account snapshots expose the human-readable account label as `name`, but this parser reads `displayName`. Normal connected account rows therefore fall back to account IDs instead of showing configured friendly labels.
  - confidence: 0.9
- **[P3] Add labeler coverage for the Linux app surface:** `apps/linux/meson.build:11`
  - body: This PR creates a new first-party app tree under `apps/linux`, but it does not add a matching `.github/labeler.yml` app label entry. Future Linux app changes will be mislabeled or unlabeled unless the new app surface is added to the repo metadata.
  - confidence: 0.86
