## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve non-cron schedules when editing jobs:** `apps/linux/src/section_cron.c:274-275`
  - body: The edit save path always sends `schedule.kind: "cron"` with an `expr`. Current jobs can also use `kind: "every"` or `kind: "at"`, so saving an existing non-cron job after changing unrelated fields can fail validation or rewrite when it runs. Carry the original schedule variant through the dialog or block editing unsupported variants.
  - confidence: 0.94
- **[P2] Preserve session-scoped cron targets:** `apps/linux/src/test_seams.c:42-48`
  - body: Unknown targets, including valid `session:<id>` targets, map to combo index 0 and are later saved back as `isolated`. A no-op edit can silently retarget a job away from its intended session, so preserve session-scoped values or expose them explicitly.
  - confidence: 0.92
- **[P2] Send gateway-supported verbose values:** `apps/linux/src/section_sessions.c:176-184`
  - body: The verbose dropdown sends `none`, `low`, and `high`, but `sessions.patch` validation accepts `off`, `on`, or `full`. Verbose changes from the Linux app will be rejected until the UI maps to the supported wire values.
  - confidence: 0.94
- **[P2] Resolve session logs from session IDs:** `apps/linux/src/section_sessions.c:264-267`
  - body: This builds the local transcript path as `<storeDir>/<session-key>/transcript.jsonl`, but the gateway resolves transcripts from `sessionId` or `sessionFile` as files under the sessions directory. This usually misses existing local transcripts and unnecessarily falls back to the dashboard.
  - confidence: 0.9
- **[P2] Bracket IPv6 hosts in dashboard URLs:** `apps/linux/src/gateway_config.c:1153-1155`
  - body: The URL builder interpolates `config->host` directly into `scheme://host:port`. IPv6 literals such as `::1` need brackets, otherwise dashboard and log handoff URLs are malformed for valid IPv6 gateway configs.
  - confidence: 0.86
- **[P2] Allow empty skill values to clear config:** `apps/linux/src/section_skills.c:133-134`
  - body: The env/API-key dialog returns early for empty input, but the gateway intentionally treats empty `apiKey` and env values as delete operations. This prevents users from clearing stale skill credentials or env overrides from the Linux app.
  - confidence: 0.88
- **[P2] Require an install option before enabling Install:** `apps/linux/src/section_skills.c:340-350`
  - body: The Install button is shown for any eligible uninstalled skill even when no `installId` is attached, but the non-ClawHub `skills.install` schema requires a non-empty `installId`. Hide or disable the button without an install option, or send a supported install source.
  - confidence: 0.84

## Security Review

Status: needs_attention

Summary: The diff adds a credentialed native desktop client with persistent auth material, exec-approval decisions, SSH tunnel control, local file opening, and gateway mutations, so focused security review is still required.

Concerns:

- **[medium] Review persistent device auth and identity storage:** `apps/linux/src/device_auth_store.c:4`
  - body: The PR persists operator device tokens and an Ed25519 device identity under the Linux state directory. Permissions are addressed in code, but token lifecycle, rotation, redaction, backup exposure, and parity with the shared device-auth contract need security sign-off before shipping.
  - confidence: 0.86
- **[medium] Review exec approval resolution from the desktop app:** `apps/linux/src/exec_approval_prompter.c:4`
  - body: The Linux companion subscribes to exec approval events and dispatches operator decisions through `exec.approval.resolve`. This is a command-execution control path, so authorization, expiry, replay handling, prompt spoofing, and cross-client resolution semantics should be reviewed.
  - confidence: 0.86
- **[medium] Review SSH tunnel command boundaries:** `apps/linux/src/remote_tunnel_command.c:15`
  - body: Remote mode builds and launches `/usr/bin/ssh` argv for local forwarding. The latest head adds dash-leading destination guards, but host/user/identity trust boundaries, strict SSH options, and remote endpoint credential handling still need security review as part of the new native app surface.
  - confidence: 0.8
