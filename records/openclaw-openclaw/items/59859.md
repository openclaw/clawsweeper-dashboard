## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Preserve schedule variants when saving cron edits:** `apps/linux/src/section_cron.c:274-275`
  - body: The edit save path always sends `schedule_kind = "cron"` and writes the entry text as `expr`. Existing jobs can validly use `kind: "every"` or `kind: "at"`, so saving an existing non-cron job after changing another field rewrites its schedule shape and can fail validation or change when it runs. Carry the original schedule variant through the dialog or block editing unsupported variants.
  - confidence: 0.91
- **[P2] Resolve local session logs from session IDs:** `apps/linux/src/section_sessions.c:266`
  - body: This builds a local transcript path as `<storeDir>/<session-key>/transcript.jsonl`, but the gateway resolves transcripts from session IDs as direct files under the sessions directory, and the PR parser already stores `session_id`. In normal installs the Log action can miss existing local transcripts and fall back to dashboard access instead of opening the file.
  - confidence: 0.87
- **[P2] Bracket IPv6 hosts before formatting URLs:** `apps/linux/src/gateway_config.c:1153-1155`
  - body: The URL builder inserts `config->host` directly into `scheme://host:port`. For an IPv6 literal such as `::1`, that creates malformed URLs like `http://::1:18789`, breaking dashboard, health, and WebSocket handoffs on valid IPv6/custom-host setups. Format IPv6 literals as `[host]` before composing these URLs.
  - confidence: 0.84
