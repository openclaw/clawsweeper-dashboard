## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Preserve existing cron schedule variants:** `apps/linux/src/section_cron.c:274-275`
  - body: The edit-save path always sends `schedule.kind = "cron"` with the textbox as `expr`. Existing jobs can be `at` or `every`, so saving an unrelated edit can rewrite the schedule shape, fail validation, or change when the job runs.
  - confidence: 0.93
- **[P2] Send supported verbose levels:** `apps/linux/src/section_sessions.c:176-184`
  - body: The session verbose selector sends `none`, `low`, and `high`, but the gateway accepts `off`, `on`, or `full`. Changing verbose from this UI path will be rejected by `sessions.patch`.
  - confidence: 0.91
- **[P2] Resolve local session logs from session IDs:** `apps/linux/src/section_sessions.c:264-266`
  - body: The local log path is built as `<store-dir>/<session-key>/transcript.jsonl`, while the gateway resolves transcripts as direct session-id/sessionFile candidates under the sessions directory. This usually misses existing local transcripts and falls back to the dashboard.
  - confidence: 0.88
- **[P2] Bracket IPv6 hosts before composing URLs:** `apps/linux/src/gateway_config.c:1153-1155`
  - body: Dashboard URLs interpolate `config->host` directly into `scheme://host:port`. IPv6 literals such as `::1` produce malformed URLs like `http://::1:18789/`, breaking dashboard and log handoffs on valid IPv6 setups.
  - confidence: 0.84
