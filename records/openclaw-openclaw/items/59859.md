- **[P1] Preserve non-cron schedules when editing cron jobs:** `apps/linux/src/section_cron.c:274-275`
  - body: This edit path always serializes the patch as a cron schedule, but gateway cron jobs can also be `at` or `every`. Saving an existing non-cron job after changing unrelated fields can rewrite or invalidate its schedule, so the dialog needs to carry the original schedule kind/fields or block unsupported edits.
  - confidence: 0.9
- **[P2] Resolve local logs from session files, not session keys:** `apps/linux/src/section_sessions.c:264-266`
  - body: The local log probe builds `<storeDir>/<session-key>/transcript.jsonl`, while current session transcripts resolve from `sessionId`/`sessionFile` as direct files under the sessions directory. This will usually miss local transcripts and fall back to the dashboard, which breaks offline or no-dashboard access.
  - confidence: 0.87
- **[P2] Bracket IPv6 literals when composing gateway URLs:** `apps/linux/src/gateway_config.c:1153-1155`
  - body: The URL builder interpolates `config->host` directly into `scheme://host:port`; a valid IPv6 literal like `::1` becomes malformed as `http://::1:18789`. Normalize bare IPv6 hosts to bracketed form before formatting dashboard, health, and WebSocket URLs.
  - confidence: 0.84
- **[P3] Add labeler coverage for the Linux app surface:** `apps/linux/meson.build:11`
  - body: This PR adds a new `apps/linux` app, but current labeler app coverage only recognizes Android, iOS, macOS, and web UI. Add Linux app/docs globs so future changes route to the right review labels instead of being misclassified.
