## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Preserve cron schedule variants on edit:** `apps/linux/src/section_cron.c:274-275`
  - body: The edit save path always sends `schedule_kind = "cron"` with the text entry. The gateway supports `at`, `every`, and `cron`, and `applyJobPatch` replaces `job.schedule` when a schedule is provided, so saving an existing `at` or `every` job from the Linux UI can reject or rewrite its schedule even when the user changed only metadata.
  - confidence: 0.92
- **[P2] Send supported verbose levels from sessions:** `apps/linux/src/section_sessions.c:176-184`
  - body: The verbose selector sends `none`/`low`/`high`, but `sessions.patch` validates `verboseLevel` through `parseVerboseOverride`, which accepts only `off`/`on`/`full`. Selecting a verbose option from this UI will fail validation instead of updating the session.
  - confidence: 0.95
- **[P2] Resolve logs with the transcript file contract:** `apps/linux/src/section_sessions.c:264-267`
  - body: The local log probe builds `<dirname(storePath)>/<sessionKey>/transcript.jsonl`, but current transcript resolution is based on `sessionId` plus the store directory and stores direct transcript files under `sessions/`. Normal local transcripts will be missed, forcing dashboard fallback and failing in offline/no-dashboard cases.
  - confidence: 0.88
