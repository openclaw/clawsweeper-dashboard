## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve non-cron schedules when saving edits:** `apps/linux/src/section_cron.c:274-275`
  - body: The edit save path always sends `schedule_kind = "cron"` and writes the textbox as a cron expression. Current gateway jobs may use `kind: "every"` or `kind: "at"`, so saving an existing non-cron job after changing unrelated fields can fail validation or change when it runs; carry the original schedule variant through the dialog or make unsupported variants read-only.
  - confidence: 0.9
- **[P2] Resolve local logs from session transcript candidates:** `apps/linux/src/section_sessions.c:265-266`
  - body: This builds the local transcript path as `<store_dir>/<session-key>/transcript.jsonl`, but the gateway resolves transcripts from `sessionId`, optional `sessionFile`, and the sessions directory. As written, the Log action will often miss existing local transcripts and fall back to the dashboard, which breaks offline or no-dashboard access.
  - confidence: 0.87
- **[P2] Bracket IPv6 hosts before composing dashboard URLs:** `apps/linux/src/gateway_config.c:1153-1155`
  - body: The URL builder interpolates `config->host` directly into `scheme://host:port`. For an IPv6 literal such as `::1`, this produces malformed URLs like `http://::1:18789`, breaking health/dashboard handoffs on valid IPv6 or custom-bind setups; normalize IPv6 literals to `[host]` before formatting.
  - confidence: 0.84
- **[P3] Add Linux app labeler coverage:** `apps/linux/meson.build:11`
  - body: This introduces a new app surface under `apps/linux`, but `.github/labeler.yml` still only covers Android, iOS, macOS, and web UI app labels. Add a Linux app label path so ownership and review automation can route future companion changes correctly.
  - confidence: 0.82
