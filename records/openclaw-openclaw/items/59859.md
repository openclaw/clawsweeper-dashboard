## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Preserve existing cron schedule variants:** `apps/linux/src/section_cron.c:270-276`
  - body: The edit save path always serializes `patch.schedule` as `{ kind: "cron", expr: ... }`. Current cron jobs can also use `at` or `every`, so saving an existing non-cron job from this UI can invalidate or rewrite its schedule even when the user changed only metadata.
  - confidence: 0.93
- **[P2] Preserve session-scoped cron targets:** `apps/linux/src/test_seams.c:42-47`
  - body: Unknown `sessionTarget` values map to index 0, which later saves back as `isolated`. Current cron targets allow `session:<id>`, so editing a session-scoped job silently retargets it away from the intended session.
  - confidence: 0.9
- **[P2] Send supported verbose values:** `apps/linux/src/section_sessions.c:175-184`
  - body: The verbose combo sends `none`, `low`, and `high`, but `sessions.patch` validates verbose overrides as `off`, `on`, or `full`. Changing verbose level from the Linux app will be rejected by the gateway.
  - confidence: 0.92
- **[P2] Resolve local logs from session IDs:** `apps/linux/src/section_sessions.c:264-267`
  - body: The log action builds `<store-dir>/<session-key>/transcript.jsonl`, but current transcript resolution is keyed by `sessionId` and optional `sessionFile`. Normal local transcripts are missed, so the Log action falls back to the dashboard even when the file exists.
  - confidence: 0.86
- **[P2] Bracket IPv6 hosts when composing gateway URLs:** `apps/linux/src/gateway_config.c:1151-1155`
  - body: Dashboard, HTTP, and WebSocket URLs interpolate `config->host` directly into `scheme://host:port`. For IPv6 literals such as `::1`, this produces malformed URLs like `http://::1:18789/`.
  - confidence: 0.87
- **[P2] Allow empty skill values to clear settings:** `apps/linux/src/section_skills.c:133-134`
  - body: The env/API-key dialog returns early on empty input. The gateway treats empty `apiKey` and env values as delete operations, so the Linux UI currently prevents users from clearing stale skill secrets or overrides.
  - confidence: 0.91
- **[P2] Require an install option before showing Install:** `apps/linux/src/section_skills.c:339-350`
  - body: The UI shows Install for any eligible uninstalled skill, but only attaches `install-id` when an option exists. Because `skills.install` requires `installId`, this button can send an invalid request and appear to do nothing.
  - confidence: 0.86
