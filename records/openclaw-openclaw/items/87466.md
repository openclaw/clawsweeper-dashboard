## Summary

Keep open: current main and v2026.6.6 still make tagged TTS directive-driven, while the merged inbound-audio fix only covers the narrower message-tool auto=inbound bug. The broader runtime-owned Telegram voice policy, stale-media race proof, and delivery-error behavior still need maintainer product direction plus live Telegram/local-TTS validation.

## What This Changes

The issue requests a runtime-owned Telegram voice delivery path that converts final text to fresh TTS media per turn instead of relying on model-generated TTS/media tags.

## Best Possible Solution

Define the canonical runtime-owned Telegram/TTS voice policy, reuse existing speech-core and Telegram delivery seams where possible, bind generated audio to the current turn or delivery id, and prove tagged, inbound, always, and failure paths with live Telegram voice delivery.

## Solution Assessment
