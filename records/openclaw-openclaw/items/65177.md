## Summary

Keep open. Current main still rejects legacy Feishu per-account `botName`, runtime account resolution reads only `name`, and the Feishu plugin still lacks a doctor compatibility normalizer; the active fix path is the open PR at https://github.com/openclaw/openclaw/pull/65384, while the earlier closing PR at https://github.com/openclaw/openclaw/pull/65202 was closed unmerged.

## What This Changes

The issue requests that `openclaw doctor --fix` migrate legacy Feishu `channels.feishu.accounts.*.botName` fields to `name` so upgraded configs validate and Gateway can start.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/65384 or an equivalent Feishu-owned doctor compatibility migration, with regression coverage, then close this issue after the fix merges.

## Reproduction Assessment

Yes. A config containing `channels.feishu.accounts.main.botName` is source-reproducible on current main because the account schema is strict and lacks `botName`, while doctor has no Feishu normalizer to rewrite it before validation.

## Solution Assessment

Yes. A Feishu plugin doctor compatibility normalizer via the existing bundled channel doctor contract is the narrowest maintainable fix; adding a core Feishu special case would cross the plugin boundary unnecessarily.
