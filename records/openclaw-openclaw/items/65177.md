## Summary

Keep open. Current main still rejects legacy Feishu account `botName` via the strict account schema, and Feishu still lacks a doctor compatibility normalizer; the active fix path is https://github.com/openclaw/openclaw/pull/65384 while https://github.com/openclaw/openclaw/pull/65202 was closed unmerged.

## What This Changes

The issue asks for `openclaw doctor --fix` to migrate legacy Feishu account `botName` fields to `name` after upgrade.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/65384 or an equivalent Feishu-owned doctor compatibility normalizer with regression coverage, then close this issue after the fix merges and ships.

## Reproduction Assessment

Yes. Source inspection on current main shows `channels.feishu.accounts.*` uses a strict account schema with `name` but not `botName`, and `doctor --fix` has no Feishu normalizer to rewrite the legacy key before validation.

## Solution Assessment

Yes. A Feishu plugin doctor compatibility normalizer wired through the existing bundled channel doctor contract is the narrowest maintainable fix; adding a core Feishu special case would cross the plugin boundary unnecessarily.
