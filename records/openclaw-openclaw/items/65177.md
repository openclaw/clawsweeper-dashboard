## Summary

Keep open. Current main still rejects legacy Feishu per-account `botName` under the strict account schema, runtime account resolution reads only `name`, and Feishu still has no doctor compatibility normalizer on main; the focused fix path is the still-open, unmerged #65384, while #65202 is closed unmerged.

## What This Changes

Request that `openclaw doctor --fix` migrate legacy `channels.feishu.accounts.*.botName` to `name` after the 2026.4.11 upgrade so Gateway validation succeeds.

## Best Possible Solution

Review and land #65384, or an equivalent Feishu-owned doctor compatibility migration, so `doctor --fix` runs before strict validation, moves legacy per-account `botName` into `name`, preserves an existing nonblank `name`, removes the stale key, and includes regression coverage plus the required user-facing changelog entry.

## Reproduction Assessment

Yes. The high-confidence source reproduction is a config with `channels.feishu.accounts.main.botName`: current main's strict Feishu account schema rejects the extra key, runtime reads only `name`, and current doctor compatibility has no Feishu normalizer to rewrite it.

## Solution Assessment

Yes. A Feishu-owned doctor compatibility normalizer exposed through the existing bundled channel doctor contract is the narrowest maintainable solution; adding a core special case for Feishu would cross the plugin ownership boundary unnecessarily.
