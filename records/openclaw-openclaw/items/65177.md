## Summary

Keep open. Current main still rejects legacy Feishu per-account `botName`, runtime account resolution reads only `name`, and no Feishu doctor compatibility normalizer is present; #65384 is the active open fix path while #65202 is closed unmerged.

## What This Changes

The issue requests that `openclaw doctor --fix` migrate legacy Feishu per-account `botName` fields to `name` after upgrade so Gateway validation succeeds.

## Best Possible Solution

Land #65384 or an equivalent Feishu-owned doctor compatibility migration that rewrites `channels.feishu.accounts.*.botName` to `name` before strict validation, preserves existing `name`, removes the stale key, and adds regression coverage plus a changelog entry.

## Reproduction Assessment

Yes. A config with `channels.feishu.accounts.main.botName` is a high-confidence source reproduction because the current per-account schema is strict and lacks `botName`, and doctor has no Feishu normalizer to rewrite it.

## Solution Assessment

Yes. A Feishu-owned doctor compatibility normalizer via the existing bundled channel doctor contract is the narrowest maintainable fix; a core Feishu special case would cross the ownership boundary unnecessarily.
