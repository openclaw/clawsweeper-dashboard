## Summary

Keep open. Current main still has the reported Feishu upgrade gap: legacy per-account `botName` is rejected by the active strict Feishu account schema, runtime resolution reads only `name`, and Feishu still does not expose a doctor compatibility normalizer for this rename. The focused implementation candidate #65384 is open and unmerged; the earlier closing-syntax PR #65202 is closed unmerged.

## What This Changes

The issue requests that `openclaw doctor --fix` migrate legacy Feishu account `botName` fields to `name` so Gateway startup is not blocked after upgrade.

## Best Possible Solution

Review and land #65384, or an equivalent Feishu-owned doctor compatibility migration, so `doctor --fix` runs before strict validation, moves legacy `channels.feishu.accounts.*.botName` into `name`, preserves an existing nonblank `name`, removes the stale key, and adds regression coverage.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: a legacy `channels.feishu.accounts.main.botName` key reaches the active strict Feishu account schema, is rejected as an additional property, and current doctor compatibility has no Feishu normalizer to remove or rename it.

## Solution Assessment

Yes. A Feishu-owned doctor compatibility normalizer is the narrow maintainable fix because the schema/runtime behavior belongs to the Feishu plugin boundary while the generic doctor seam already exists.
