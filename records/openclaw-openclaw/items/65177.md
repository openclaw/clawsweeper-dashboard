## What This Changes

Migrate legacy `channels.feishu.accounts.*.botName` to `name` during `openclaw doctor --fix` so upgraded Feishu configs can validate and the Gateway can start.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/65384 or an equivalent Feishu-owned doctor compatibility normalizer with regression coverage, then close this issue after the fix merges and ships.

## Reproduction Assessment

Yes, source-backed. Current main has a strict Feishu account schema with `name` but no `botName`, and the doctor compatibility path has no Feishu normalizer to rewrite the legacy key.

Yes. A Feishu plugin doctor compatibility normalizer wired through the existing bundled channel doctor contract is the narrow maintainable fix; a core Feishu special case would cross the plugin boundary.
