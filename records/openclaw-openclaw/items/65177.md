## Summary

Keep open. Current main still rejects legacy Feishu per-account `botName`, runtime account resolution reads only `name`, and Feishu has no doctor compatibility normalizer on main; #65384 remains the active open fix path.

## What This Changes

The issue requests that `openclaw doctor --fix` migrate legacy Feishu `channels.feishu.accounts.*.botName` fields to `name` so upgraded gateways can validate and start.

## Best Possible Solution

Land #65384 or an equivalent Feishu-owned doctor compatibility migration that rewrites `botName` to `name`, preserves existing `name`, removes the stale key, and adds regression coverage plus a changelog entry.

## Reproduction Assessment

Yes. A config with `channels.feishu.accounts.main.botName` is a high-confidence source reproduction because the current schema is strict and lacks `botName`, and doctor has no Feishu normalizer to rewrite it.

## Solution Assessment

Yes. A Feishu-owned doctor compatibility normalizer through the existing bundled channel doctor contract is the narrowest maintainable fix; a core Feishu special case would cross the ownership boundary unnecessarily.
