## What This Changes

The PR changes the embedded runner allowlist guard to ignore inherited config allowlists for intentional no-tool model runs while keeping explicit runtime `toolsAllow` fail-closed.

## Best Possible Solution

Land the shared guard fix with targeted regression coverage and changed-gate proof, keeping inherited config allowlists ignored only for intentional no-tool embedded runs while explicit runtime allowlists remain fail-closed.

## Reproduction Assessment

Yes, from source inspection: current main can be reproduced at the guard helper by passing non-empty inherited sources, no callable tools, and `disableTools: true`; the embedded `modelRun` and `llm-task` paths supply the matching no-tool conditions.

## Solution Assessment

Yes; tagging runtime allowlist sources and filtering inherited sources only for disabled-tool runs is narrower than clearing allowlists at each caller, and it preserves explicit `toolsAllow` failures.
