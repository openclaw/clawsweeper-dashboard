## Summary

Keep open: current main and v2026.6.10 still use a top-level-only visibility check for agent payloads, and the linked same-author fix PR is open rather than merged. This issue should close only after https://github.com/openclaw/openclaw/pull/98207 or an equivalent fix lands.

## What This Changes

The issue reports that subagent announce delivery can drop a visible reply when the only user-visible text is nested under wrapper fields such as content, result, output, or message.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/98207, or an equivalent narrow fix that lets only the subagent announce path treat nested wrapper text as visible while preserving sibling caller behavior.

## Reproduction Assessment

Yes at source level: a payload whose only visible text is nested under content/result/output/message is not considered visible by current hasVisibleAgentPayload, and subagent announce delivery delegates to that helper. I did not run a full live subagent/channel turn in this read-only pass.

## Solution Assessment

Yes, the best path is the existing linked PR direction: add a narrow opt-in nested-text visibility check for subagent announce delivery while leaving the other top-level-only callers unchanged.
