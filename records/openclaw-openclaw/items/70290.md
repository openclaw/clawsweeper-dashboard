## What This Changes

This PR replaces the default TUI footer with compact model, token ratio, progress bar, percent, agent, and elapsed-time segments plus compact formatter helpers.

## Best Possible Solution

Preserve the documented footer by default; if compact rendering is accepted, land it behind a reviewed footer mode/config path or the status-provider direction, with focused tests and real TUI proof.

## Reproduction Assessment

Not applicable as a user bug. For the PR blockers, yes: source and diff inspection show the default footer fields being replaced, the active timers not refreshing the footer elapsed value, and unknown token pressure using the success-color path.

## Solution Assessment

No, not as currently patched. A compact footer should be opt-in or explicitly approved as a default UX change, and the elapsed/unknown-pressure issues need fixing before merge.
