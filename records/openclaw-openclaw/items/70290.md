## Summary

Keep this PR open. Current main still uses the documented verbose TUI footer, but the proposed compact footer has a real elapsed-time refresh bug, an unknown-pressure color issue, and a missing user-facing changelog entry before it should merge.

## What This Changes

The PR changes the TUI footer to a compact branded line with model, compact token ratio, pressure-colored progress bar, percent, agent label, and elapsed time, and adds compact footer formatter helpers.

## Best Possible Solution

Decide whether compact footer rendering should be the default, a selectable mode, or part of a configurable/plugin-contributed footer design; if accepted, update the timer refresh path, unknown-state coloring, tests, docs, and changelog together.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable for the feature request itself, but yes for the main PR defect: source inspection shows the proposed elapsed footer depends on `statusStartedAt` while the existing busy timers do not call `updateFooter()`.

## Solution Assessment

Is this the best way to solve the issue? Unclear. A compact footer is plausible, but a hard replacement of documented session and mode fields is riskier than a maintainer-approved mode/config path aligned with #53774 and #64086.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Refresh the footer while showing elapsed time:** `src/tui/tui.ts:844-853`
  - body: The new footer renders `elapsedLabel` from `statusStartedAt`, but the existing busy timers only call `updateBusyStatusMessage()`, not `updateFooter()`. During a normal waiting or streaming run the footer elapsed segment will stay at its initial value, often `0s`, instead of reflecting run duration. Trigger `updateFooter()` from the same timer/activity path or remove elapsed time from the footer.
  - confidence: 0.87
- **[P3] Use neutral color for unknown token pressure:** `src/tui/tui.ts:834-849`
  - body: When token data is unavailable, `percentUsed` is `null`, but `renderCompactProgressBar(null, 12)` returns percent `0`, which selects `theme.success` for the `?%` label. Unknown pressure should not look like an all-clear state; use a neutral or dim color whenever the percent is unknown.
  - confidence: 0.78

  - body: This is a user-facing TUI feature change, but the PR only changes TUI source files. Repo policy requires user-facing `feat` changes to add a `CHANGELOG.md` entry, otherwise the default footer behavior change will be missing from release notes.
