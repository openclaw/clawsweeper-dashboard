## What This Changes

The branch adds compact TUI footer formatting helpers and replaces the default footer parts with a compact model/tokens/progress/percent/agent/elapsed line.

## Best Possible Solution

Have maintainers decide whether compact footer rendering should be default or opt-in; if accepted, preserve required session/mode visibility or update the contract, fix elapsed and unknown-pressure behavior, and require real TUI proof.

## Reproduction Assessment

Not applicable as a user bug. Yes for the review findings: PR diff and source inspection show the default footer fields are removed, elapsed text is only recomputed by `updateFooter()`, and unknown token pressure flows through a success-colored `0` progress value.

## Solution Assessment

Unclear. Hard-replacing the default footer is not clearly the narrowest maintainable solution while footer customization and plugin status-provider work remain open; a reviewed default-or-opt-in UX decision is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Preserve documented footer state:** `src/tui/tui.ts:843-851`
  - body: The replacement `footerParts` list omits the current session and mode/reasoning indicators that current main and the TUI docs expose in the default footer. Users switching sessions or toggling think/fast/verbose/reasoning lose that visible state unless compact rendering is opt-in or the contract is deliberately changed.
  - confidence: 0.86
- **[P2] Refresh the footer elapsed timer:** `src/tui/tui.ts:841-849`
  - body: `elapsedLabel` is computed inside `updateFooter()`, but the busy timers only refresh the status message. During waiting or streaming the footer elapsed value will stay stale, often at `0s`, unless the footer is refreshed on the same timer or the elapsed segment is removed.
  - confidence: 0.88
- **[P3] Use a neutral color for unknown pressure:** `src/tui/tui.ts:847`
  - body: When token data is unavailable, `percentUsed` is null but `renderCompactProgressBar()` returns percent `0`, so the `?%` label uses the success color. Unknown pressure should render neutral or dim instead of looking like an all-clear state.
  - confidence: 0.78
