## What This Changes

The branch adds compact TUI footer formatter helpers and replaces the default footer with a compact model, token, progress, percent, agent, and elapsed-time line.

## Best Possible Solution

Have maintainers decide whether compact footer rendering should be default or opt-in; if accepted, preserve or deliberately revise the documented footer contract, fix elapsed and unknown-pressure behavior, and require real TUI proof.

## Reproduction Assessment

Not applicable as a user bug. Yes for the PR findings: source and diff inspection show the removed default footer fields, elapsed value recomputed only by `updateFooter()`, and unknown token pressure flowing through the success-color path.

## Solution Assessment

No, not as currently patched. Hard-replacing the documented default footer is not the narrowest maintainable path while opt-in footer configuration and plugin status-provider work remain unresolved.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve documented footer state:** `src/tui/tui.ts:845-852`
  - body: The new `footerParts` list drops the session and think/fast/verbose/reasoning indicators that current main and the TUI docs expose in the default footer. Users switching sessions or toggling modes lose visible state unless compact rendering is opt-in or the footer contract is deliberately changed.
  - confidence: 0.88
- **[P2] Refresh elapsed footer while runs are active:** `src/tui/tui.ts:843`
  - body: `elapsedLabel` is computed inside `updateFooter()`, but the busy timers only refresh the status message. During waiting or streaming, the footer elapsed value can stay stale, often at `0s`, unless the footer is refreshed on the same timer or the segment is removed.
  - confidence: 0.89
- **[P3] Use a neutral color for unknown pressure:** `src/tui/tui.ts:849`
  - body: When token data is unavailable, `renderCompactProgressBar(null, 12)` returns percent `0`, and that success color is reused for the `?%` label. Unknown pressure should render neutral or dim instead of looking like an all-clear state.
  - confidence: 0.8
