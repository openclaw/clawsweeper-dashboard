## Summary

Keep open: the PR needs maintainer UX direction because it replaces the documented default TUI footer, drops visible session/mode state, adds an elapsed footer value that will not refresh during active runs, and still lacks real TUI behavior proof from the external contributor.

## What This Changes

The branch adds compact TUI footer formatter helpers and replaces the default footer with a compact model, token, progress, percent, agent, and elapsed-time status line.

## Best Possible Solution

Have maintainers decide whether compact footer rendering should be default or opt-in; if accepted, preserve or deliberately revise the documented footer contract, fix elapsed and unknown-pressure behavior, add focused coverage, and require real TUI proof.

## Reproduction Assessment

Not applicable as a user bug. Yes for the PR findings: source and diff inspection show the removed default footer fields, elapsed value recomputed only by updateFooter, and unknown token pressure flowing through the success-color path.

## Solution Assessment

No, not as currently patched. Hard-replacing the documented default footer is not the narrowest maintainable path while opt-in footer configuration and plugin status-provider work remain unresolved.
