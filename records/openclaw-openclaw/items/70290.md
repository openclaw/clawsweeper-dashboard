## Best Possible Solution

Have maintainers decide whether compact footer rendering should be the default, a selectable mode, or part of the footer customization/status-provider design; if accepted, fix elapsed refresh and unknown-state coloring, then align tests/docs with real TUI proof.

## Reproduction Assessment

Not applicable for the requested feature as a user bug. The main PR defect is source-reproducible: the diff adds an elapsed footer segment from statusStartedAt while the busy timer path only updates the status message, not the footer.

## Solution Assessment
