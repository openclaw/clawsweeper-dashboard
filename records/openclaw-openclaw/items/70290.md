## Summary

Keep open. The compact footer idea is plausible, but this branch should not merge yet because it drops documented footer session visibility, adds a stale elapsed timer path, colors unknown token pressure as success, and still lacks real TUI behavior proof.

## What This Changes

The PR replaces the current verbose TUI footer with a compact branded line showing model, token ratio, progress bar, percent used, agent label, and elapsed time.

## Best Possible Solution

Have maintainers choose whether compact footer rendering is the default, an opt-in footer mode, or part of the broader footer customization/plugin-status design; if accepted, preserve required session visibility or update the contract, fix the elapsed and unknown-pressure behavior, and add real TUI proof.

## Reproduction Assessment

Not applicable for the requested feature as a user bug. Yes for the review findings: source inspection shows the PR derives footer elapsed from statusStartedAt without a timer-driven footer refresh, and docs/source show the session-visible footer contract being removed.

## Solution Assessment

Unclear. Hard-replacing the default footer is not clearly the best maintainable solution while footer customization and plugin status-provider work are still open; a reviewed default-or-opt-in UX decision is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Keep the documented session visible:** `src/tui/tui.ts:845-851`
  - body: Current docs say the current agent and session are always visible in the footer, but the replacement footer renders the agent without the session. Users switching or resuming sessions lose that footer identity, so preserve the session in the default footer or make compact mode opt-in with matching docs.
  - confidence: 0.86
- **[P2] Refresh the footer elapsed timer:** `src/tui/tui.ts:843-851`
  - body: The new elapsedLabel comes from statusStartedAt, but the busy timers only call updateBusyStatusMessage(), not updateFooter(). During waiting/streaming the footer elapsed value will stay stale, often at 0s, unless the footer is refreshed on the same timer or the elapsed segment is removed.
  - confidence: 0.88
- **[P3] Use a neutral color for unknown pressure:** `src/tui/tui.ts:833-849`
  - body: When token data is unavailable, percentUsed is null but renderCompactProgressBar returns percent 0, which makes the ?% label use the success color. Unknown pressure should render neutral/dim instead of looking like an all-clear state.
  - confidence: 0.78
