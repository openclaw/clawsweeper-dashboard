## Summary

Keep open. Current main has not implemented this compact footer, and the PR still needs maintainer UX direction, real TUI proof, and fixes for session visibility, elapsed refresh, and unknown-pressure coloring before it is merge-ready.

## What This Changes

The branch replaces the verbose TUI footer with a compact model/tokens/progress/percent/agent/elapsed status line and adds compact footer formatting helpers.

## Best Possible Solution

Have maintainers choose whether compact footer rendering is the default or opt-in; if accepted, preserve required session visibility or update the contract, fix elapsed and unknown-pressure behavior, and require real TUI proof.

## Reproduction Assessment

Not applicable as a user bug. Yes for the review findings: source inspection shows the PR removes documented session visibility and derives elapsed footer text from updateFooter() without a timer-driven footer refresh.

## Solution Assessment

Unclear. Hard-replacing the default footer is not clearly the best maintainable path while footer customization and plugin status-provider work remain open; a reviewed default-or-opt-in UX decision is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Keep the documented session visible:** `src/tui/tui.ts:845-851`
  - body: Current docs and current main keep the active session visible in the footer, but this replacement renders only the agent label. Users switching or resuming sessions lose that identity, so preserve the session in the default footer or make compact mode opt-in with matching docs.
  - confidence: 0.86
- **[P2] Refresh the footer elapsed timer:** `src/tui/tui.ts:843-851`
  - body: The new elapsed label is computed inside updateFooter(), while the busy timers only update the status message. During waiting or streaming the footer elapsed value will stay stale, often at 0s, unless the footer refreshes on the same timer or the elapsed segment is removed.
  - confidence: 0.88
- **[P3] Use a neutral color for unknown pressure:** `src/tui/tui.ts:833-849`
  - body: When token data is unavailable, percentUsed is null but renderCompactProgressBar() returns percent 0, so the ?% label uses the success color. Unknown pressure should render neutral or dim instead of looking like an all-clear state.
  - confidence: 0.78
