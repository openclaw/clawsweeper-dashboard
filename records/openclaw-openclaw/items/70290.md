## Summary

Keep open: this PR is not merge-ready because it changes the documented default TUI footer contract without a compatibility/product decision, leaves the new elapsed segment stale during active runs, colors unknown pressure as success, and lacks after-fix real TUI proof.

## What This Changes

This PR changes the TUI footer renderer to use new compact token, percent, and progress helpers and show model, token ratio, progress bar, percent, agent label, and elapsed time.

## Best Possible Solution

Preserve the current default footer until maintainers choose the compact-footer UX; if accepted, make compact rendering opt-in or deliberately update the documented contract, fix elapsed and unknown-pressure behavior, and require real TUI proof.

## Reproduction Assessment

Not applicable as a user bug. Source and diff inspection give a high-confidence check of the PR findings: PR head removes documented footer fields, computes elapsed only inside updateFooter(), and routes unknown token pressure through the success-color path.

## Solution Assessment

No, not as currently patched. Hard-replacing the documented default footer is not the narrowest maintainable solution while footer customization and plugin/status-provider design are still open.

Overall correctness: patch is incorrect

Full review comments:

- **[P2] Preserve documented footer state:** `src/tui/tui.ts:845-852`
  - body: The new `footerParts` list no longer includes session or think/fast/verbose/reasoning indicators, even though current main and the TUI docs describe those as default footer state. Existing users switching sessions or toggling modes lose visible state unless compact rendering is opt-in or the footer contract deliberately changes.
  - confidence: 0.9
- **[P2] Refresh elapsed footer while runs are active:** `src/tui/tui.ts:843`
  - body: `elapsedLabel` is computed inside `updateFooter()`, but the active-run timers only call `updateBusyStatusMessage()`. During waiting or streaming, the footer elapsed value can remain stuck at `0s` or a stale value unless the footer is refreshed on the same cadence or the segment is removed.
  - confidence: 0.9
- **[P3] Render unknown pressure neutrally:** `src/tui/tui.ts:849`
  - body: When token data is unavailable, `renderCompactProgressBar(null, 12)` returns percent `0`, which selects the success color that is then applied to `?%`. Unknown pressure should render neutral or dim instead of visually signaling an all-clear state.
  - confidence: 0.82
