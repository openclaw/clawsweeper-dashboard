## Summary

Keep open: current main and the latest release still lack same-provider auth-profile failover in compaction, and the linked open PR is the active candidate fix but has not merged.

## What This Changes

Compaction should use same-provider auth-profile rotation/failover like normal runs when an auto-selected profile is rate-limited or in cooldown.

## Best Possible Solution

Land or replace the linked fixing PR with an idempotent compaction retry boundary that rotates only auto-selected same-provider profiles and applies exactly one successful compaction result.

## Reproduction Assessment

Yes at source level: current main compaction resolves one selected auth profile for a compaction request, while normal embedded runs have explicit profile advancement for retryable failures. I did not run a live provider reproduction in this read-only review.

## Solution Assessment

Unclear as currently implemented: same-provider rotation is the right target, but the safest fix needs the linked PR or replacement to preserve user-locked profiles and prove snapshot/apply-once semantics.
