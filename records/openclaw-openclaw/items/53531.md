## Summary

Keep open: current `main` has normal assistant auth-profile rotation and some compaction model/timeout fallback plumbing, but the compaction summarization attempt still uses one selected auth profile and does not retry eligible same-provider profiles after a compaction rate-limit failure.

## What This Changes

The issue requests compaction to reuse same-provider auth-profile rotation and failover when the active profile is rate-limited or cooling down.

## Best Possible Solution

Add a compaction job boundary that snapshots the transcript, tries eligible same-provider auth profiles against that snapshot, and commits exactly one successful summary/checkpoint.

## Reproduction Assessment

Yes at source level: normal assistant turns rotate through auth-profile candidates, while compaction resolves and uses one selected `authProfileId` for the summarization attempt. I did not run a live provider repro in this read-only review.

## Solution Assessment

No final implementation shape is settled yet: same-provider rotation is the right direction, but the best fix must add an idempotent snapshot/apply-once boundary before retrying compaction across profiles.
