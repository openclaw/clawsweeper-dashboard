## Summary

Keep open: current main still lacks same-provider auth-profile failover in the direct compaction path, and the linked fixing PR is open but unmerged.

## What This Changes

The issue requests that compaction rotate or fail over across same-provider auth profiles when the selected profile is rate-limited or in cooldown.

## Best Possible Solution

Land or replace the linked fixing PR with same-provider auto-profile rotation that preserves user-locked profiles and applies exactly one compaction result from a stable transcript snapshot.

## Reproduction Assessment

Yes at source level: current main resolves one selected auth profile for direct compaction, while normal embedded runs have explicit auth-profile advancement for retryable failures. I did not run a live provider reproduction in this read-only review.

## Solution Assessment

Yes on direction, but the final fix needs maintainer-approved idempotency semantics: rotate only auto-selected same-provider profiles, preserve user-locked profiles, and apply one result from a stable transcript snapshot.
