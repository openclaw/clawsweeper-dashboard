## Summary

Keep open: current main still lacks same-provider auth-profile failover inside the direct compaction path, and the only matching fix is the open, unmerged https://github.com/openclaw/openclaw/pull/95650. The issue should stay open until that PR lands or maintainers choose a replacement.

## What This Changes

Compaction should retry eligible same-provider auth profiles when the selected profile is rate-limited or cooling down, while preserving session-state correctness.

## Best Possible Solution

Land or replace the linked fixing PR with an idempotent compaction retry boundary that rotates only auto-selected same-provider profiles and applies exactly one successful compaction result.

## Reproduction Assessment

Yes at source level: current main compaction resolves one selected auth profile per compaction attempt, while normal embedded runs have explicit auth-profile advancement for retryable failures. I did not run a live provider reproduction in this read-only review.

## Solution Assessment

Unclear as implemented today: same-provider rotation is the right target, but the safest fix needs maintainer-approved snapshot/apply-once semantics before the issue closes.
