## Summary

Keep open for maintainer review: this protected, security-sensitive installer/workflow PR has a concrete supply-chain issue in the new Windows portable Git bootstrap.

## What This Changes

The PR adds a website installer sync workflow, introduces OpenClaw-owned `install-cli.sh` and `install.cmd`, rewrites `install.ps1`, and expands installer smoke routing/tests.

## Best Possible Solution

Keep the source-owned installer sync direction, but remove or pin/hash-check the portable Git bootstrap and require clean exact-head CI before merge.

## Reproduction Assessment

Yes for the review finding by source inspection: Windows without Git reaches `Ensure-Git`, downloads the latest MinGit ZIP, and extracts it without verification. I did not run a real Windows install.

## Solution Assessment

No. The sync workflow is a plausible maintainer-owned direction, but the Windows bootstrap should not trust a mutable third-party archive without pinning and verification.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Verify portable Git before extracting it:** `scripts/install.ps1:287-289`
  - body: This adds a Windows fallback that resolves the latest `git-for-windows` release, downloads the selected MinGit ZIP, and extracts it without pinning a release or checking a digest/signature.
