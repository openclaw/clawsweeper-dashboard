## Summary

Keep open: this is useful installer ownership work, but the latest head still has blocking installer safety issues and the PR carries a protected `maintainer` label. The main blockers are an unverified portable Git bootstrap on Windows and an unscoped cleanup call in the new CLI installer.

## What This Changes

The PR makes OpenClaw the source for website installer scripts, adds `install-cli.sh` and `install.cmd`, rewrites `install.ps1`, adds a website installer sync workflow, and expands installer scope/tests.

## Best Possible Solution

Keep the source-owned installer sync direction, but remove or pin/hash-check the portable Git bootstrap, scope legacy cleanup to the selected git checkout, and merge only after exact-head CI plus maintainer/security review.

## Reproduction Assessment

Yes for the review findings by source inspection: Windows without Git reaches the portable Git download/extract path, and `install-cli.sh` calls cleanup without the parsed `GIT_DIR`. I did not run a real Windows install.

## Solution Assessment

No. The source-owned installer sync direction is reasonable, but this PR should not add a mutable binary trust path or unscoped cleanup before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Verify portable Git before extracting it:** `scripts/install.ps1:287-289`
  - body: This fallback resolves the latest Git-for-Windows release, downloads a MinGit ZIP, and extracts it without pinning the release or checking a digest/signature. Fresh Windows installs without Git would trust a mutable third-party binary path that CI only dry-runs, so either keep the manual/package-manager path or pin and verify the archive before extraction.
  - confidence: 0.86
- **[P2] Pass the parsed git dir to legacy cleanup:** `scripts/install-cli.sh:711`
  - body: `main` invokes `cleanup_legacy_submodules` without an argument, so the helper falls back to `OPENCLAW_GIT_DIR` or `$HOME/openclaw` instead of the parsed `--git-dir` value in `GIT_DIR`. A custom git install can delete `Peekaboo` under the default checkout path, and npm installs run this cleanup too; move it into the git path and pass the selected repo directory.
  - confidence: 0.9
