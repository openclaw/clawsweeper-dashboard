## Summary

Keep open: current main still has the Docker setup path that can trigger a BuildKit Dockerfile frontend pull before the image build starts. The report's root-cause wording is off, because `# syntax=...` is a Docker parser directive rather than a shell command, but the reported WSL timeout against `auth.docker.io` is a plausible source-backed setup failure.

## What This Changes

The issue asks Docker setup to stop failing on WSL when the root Dockerfile syntax directive requires fetching the Dockerfile frontend from Docker Hub.

## Best Possible Solution

Make Docker setup avoid a mandatory external Dockerfile frontend pull on supported Docker versions, or add a narrow fallback, while preserving BuildKit `RUN --mount` support and Docker image coverage.
