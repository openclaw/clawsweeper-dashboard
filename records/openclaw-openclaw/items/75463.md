## Summary

Keep open. Current main still has the bare/prefixed in-flight alias race, and this PR is a narrow BlueBubbles plugin repair with focused regression coverage, a changelog entry, no blocking review finding, and no concrete security concern.

## What This Changes

The PR updates BlueBubbles reply-context fallback to cache validated `p:n/<guid>` aliases for callers that join an existing bare-message in-flight fetch, adds a regression test, and adds a changelog entry.

## Best Possible Solution

Land the narrow BlueBubbles plugin fix after maintainer review and normal changed-gate validation, keeping alias handling local to the reply-context fallback cache path.

## Reproduction Assessment

Yes. On current main, start a bare `msg-race` fallback fetch and concurrently call `p:0/msg-race` for the same account; the prefixed caller returns the existing promise before alias preservation, leaving the exact prefixed cache key absent. The PR's added regression test models that path directly.

## Solution Assessment

Yes. Caching the validated part-index alias from a success continuation on the shared in-flight promise preserves single-request dedupe while recording the joining caller's alias, which is narrower than changing fetch keying or redesigning the reply cache.
