## What This Changes

The PR installs the Codex runtime plugin during OpenAI onboarding and starts bundled Codex for implicit OpenAI model routes while preserving explicit PI runtime pins in gateway startup planning.

## Best Possible Solution

Update the Codex install decision to skip explicit PI runtime policies, keep the remote Codex install/load behavior for implicit official OpenAI routes, then rerun the targeted onboarding and startup tests.

## Reproduction Assessment

Yes, at source level. Current main has a clear Codex setup gap for implicit OpenAI routes, and the related issue plus PR body provide Docker/Testbox behavior evidence, but this read-only review did not run a live repro.

## Solution Assessment

No as written. The startup-planning direction is narrow and matches the documented runtime policy, but the new noninteractive install call should use the same runtime-policy check so explicit PI setups do not download or enable Codex unnecessarily.
