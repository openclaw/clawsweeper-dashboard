## Summary

The PR targets the right cron/MCP cleanup bug, but it should not merge yet because the new disposal timeout can still return before streamable HTTP transports are actually aborted, and the external proof is still test-only.

## What This Changes

Adds MCP runtime retirement to isolated cron timeout/dispose cleanup, wraps bundle MCP session disposal in a 5-second race, and updates focused cron/MCP tests.

## Best Possible Solution

Land a bounded cron/MCP cleanup fix that retires runtimes on timeout/dispose, actively aborts or closes transports when a disposal deadline wins, and includes focused regression coverage plus redacted runtime proof or maintainer override.

## Reproduction Assessment

No for the full live gateway wedge; the linked issue and current source make the cleanup leak source-reproducible, but no 8-hour soak or live cron/MCP gateway run was verified here.

## Solution Assessment

No. The retirement call sites are aimed at the right surface, but the timeout implementation should actively abort or close streamable HTTP transports when the deadline wins, and the test should cover a path that fails on current main.
