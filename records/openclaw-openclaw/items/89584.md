Full review comments:

- **[P1] Keep rerank registration behind the manifest gate:** `src/plugin-sdk/memory-core-host-runtime-core.ts:58`
  - body: This public SDK subpath now re-exports the raw `registerMemoryRerankProvider(pluginId, provider)` mutator. A plugin can import that function directly and register an undeclared reranker, bypassing the `contracts.memoryRerankProviders` check in `registry.ts` and potentially squatting the exclusive slot before the gate-enforced `api.registerMemoryRerankProvider` path runs. Remove or gate the raw public mutator and keep plugin registration on the API method documented in this PR.
  - confidence: 0.91
- **[P3] Update the plugin test API for the new method:** `src/plugins/types.ts:2887-2889`
  - body: Adding `registerMemoryRerankProvider` to `OpenClawPluginApi` also needs the default noop in `src/plugin-sdk/plugin-test-api.ts`. Today `createTestPluginApi()` still returns an object without this method, so reranker plugins using the repo's test helper will fail at runtime even though the helper casts the object to the full API type.
  - confidence: 0.82
- **[P3] Check empty rerank scores before generic validation:** `extensions/memory-core/src/memory/manager.ts:748-750`
  - body: For a non-empty candidate pool, `[]` fails `isValidRerankScores` at line 748 before the explicit empty-score branch can run, so a kill-switched provider records the generic invalid/duplicate-ref reason instead of the intended no-scores reason. Move the empty check before validation or fold the reason into validation so degraded observability matches the documented state.
  - confidence: 0.78
