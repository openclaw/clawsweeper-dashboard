## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Avoid reusing stale rerank state for empty searches:** `extensions/memory-core/src/memory/manager.ts:722`
  - body: When a registered reranker sees an empty `merged` pool, this branch emits `this.rerankState` without changing it. After a prior successful or failed search, a no-hit search can report `debug.rerank: "active"` or `"degraded"` even though no provider call or scores happened, contradicting the new docs that `active` means scores were applied. Reset to a non-stale state or add an explicit skipped state, and cover active-to-empty and degraded-to-empty.
  - confidence: 0.9
- **[P2] Document candidate.source as MemorySource:** `docs/plugins/memory-reranker.md:38`
  - body: The new SDK docs tell plugin authors that `candidate.source` is a `string` backend tag such as `"sqlite-vec"`, but the exported `MemoryRerankCandidate` type is `source: MemorySource` and the manager passes `r.source` (`"memory" | "sessions"`). A reranker written from this example would branch on a value it never receives, so the docs should import or describe the actual `MemorySource` contract.
  - confidence: 0.92
