## Review Findings

Overall correctness: patch is incorrect

Full review comments:

- **[P2] Do not report a registered reranker as disabled:** `extensions/memory-core/src/memory/manager.ts:1172`
  - body: `status().custom.rerank.state` currently mirrors `this.rerankState`, which starts as `"disabled"` and is only updated after `applyRerankStage` runs on a non-empty candidate pool. If a plugin registers a reranker and the operator checks memory status before the first candidate-bearing search, status reports `disabled` even though registration succeeded, contradicting the new contract that disabled means no reranker is registered. Derive disabled from the registry or add an explicit registered/idle state and cover that status path.
