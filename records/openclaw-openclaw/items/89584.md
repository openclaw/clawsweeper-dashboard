## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Avoid reporting stale rerank state:** `extensions/memory-core/src/memory/manager.ts:722-723`
  - body: When a registered reranker has previously set `this.rerankState` to `active` or `degraded`, a later search with an empty merged candidate pool emits that old state without calling the provider or applying scores. That makes `debug.rerank` and `status().custom.rerank` report a stale active/degraded state for a search where reranking did not happen, contradicting the new observability contract. Set an explicit non-stale state/debug value for the empty-pool path or add a separate skipped state.
  - confidence: 0.9
- **[P2] Document candidate.source as MemorySource:** `docs/plugins/memory-reranker.md:38`
  - body: The runtime fills `candidate.source` from `MemorySearchResult.source`, whose SDK type is `"memory" | "sessions"`, but this new docs example describes it as a backend tag such as `"sqlite-vec"`. A reranker following the docs could special-case a value it never receives and miss the actual memory/session distinction, so the guide should match the exported `MemorySource` contract.
  - confidence: 0.92
