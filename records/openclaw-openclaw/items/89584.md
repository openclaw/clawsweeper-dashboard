## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Reset rerank state for empty candidate searches:** `extensions/memory-core/src/memory/manager.ts:719-723`
  - body: When a reranker is registered but `merged.length === 0`, this branch emits the previous `rerankState`. After any successful rerank, a later no-hit search will report `debug.rerank: "active"` even though no scores were applied; after a failure it can keep reporting `"degraded"` for a no-op search. That contradicts the new docs that say `active` reliably means reranking happened, so set an explicit non-active state for this path and cover the active-then-empty transition.
  - confidence: 0.91
- **[P2] Describe candidate.source as MemorySource:** `docs/plugins/memory-reranker.md:35-39`
  - body: The docs present `MemoryRerankCandidate.source` as an arbitrary `string`, but the exported SDK type is `source: MemorySource`, currently limited to `"memory" | "sessions"`. Plugin authors following this snippet could implement against a wider contract than TypeScript accepts, so the docs should match the exported type or the type should intentionally widen.
  - confidence: 0.93
