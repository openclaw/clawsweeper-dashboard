## Summary

Keep open: current main still lacks the requested Control UI derived-title adoption. Related duplicate issues were closed in favor of this issue, and the open broader UI PR is not merged and is currently conflicting, so it does not resolve the canonical tracker.

## What This Changes

Request the Control UI session picker to call `sessions.list` with `includeDerivedTitles` and show `derivedTitle` before falling back to raw session keys.

## Best Possible Solution

Add a small Control UI adoption change: include `derivedTitle` in the UI row contract, request bounded derived titles, and resolve picker labels as label/displayName/derivedTitle/key while leaving AI title quality and manual renaming to their existing tracks.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not as a bug repro, but yes by source inspection: current main's Control UI sessions loader omits `includeDerivedTitles` and the picker label resolver ignores `derivedTitle`.

## Solution Assessment

Is this the best way to solve the issue? Yes for the narrow request: consume the existing Gateway `includeDerivedTitles` contract in the Control UI rather than adding new title generation or rename behavior here.
