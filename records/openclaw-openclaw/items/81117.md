## Summary

Keep open: current main and the latest release still omit `derivedTitle` from the Control UI session picker path, and the open linked PR is the active but unmerged implementation candidate.

## What This Changes

Request the Control UI session picker to call `sessions.list` with `includeDerivedTitles` and show `derivedTitle` before falling back to raw session keys.

## Best Possible Solution

Land a corrected version of https://github.com/openclaw/openclaw/pull/92805 that updates the actual picker label path and proves the rendered UI behavior, while leaving manual renaming and AI title generation to separate title-management issues.

## Reproduction Assessment

Yes. Source inspection of current main shows the picker request params omit `includeDerivedTitles` and the display resolver ignores `derivedTitle`, while Gateway can populate it when asked.

## Solution Assessment

Yes for the narrow direction. Reusing the existing Gateway `includeDerivedTitles` contract is the maintainable path; manual renaming or AI title generation should stay separate.
