## Summary

Keep open: current `main` and the latest release still do not wire derived titles into the Control UI sidebar or picker, and the remaining implementation is already represented by an open candidate PR that needs review rather than a duplicate fix lane.

## What This Changes

The issue requests that the Control UI session picker request and display derived session titles, falling back to raw session keys only when no readable title exists.

## Best Possible Solution

Land or replace the open candidate PR with a small UI change that requests `includeDerivedTitles`, preserves label/displayName precedence, updates both sidebar and picker label paths, and includes real Control UI proof.

## Reproduction Assessment

Yes. Source inspection shows current `main` exposes the Gateway option but the Control UI sidebar and picker do not request derived titles, and the shared display resolver does not render `derivedTitle`.

## Solution Assessment

Yes for the narrow direction, but it is not implemented yet. Reusing the existing Gateway `includeDerivedTitles` contract is the maintainable fix; manual renaming and AI title-generation policy should remain separate.
