## Summary

Keep open: current main still exposes derived titles in Gateway but the Control UI session picker neither requests `includeDerivedTitles` nor uses `derivedTitle` when building option labels. The related closed issue was explicitly closed in favor of this canonical issue, and the broader auto-title issue/PR does not supersede this narrower UI adoption request.

## What This Changes

The issue requests that the Control UI session picker call `sessions.list` with derived-title hydration and display readable titles instead of raw session keys.

## Best Possible Solution

Add `derivedTitle` to the Control UI session row contract, request bounded derived titles from `sessions.list`, and use them after manual labels/display names but before raw key fallbacks; leave manual rename and AI summarization policy to their existing tracks.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a bug reproduction, but source inspection gives high confidence: the Control UI omits `includeDerivedTitles` and the picker label path ignores `derivedTitle`.

## Solution Assessment

Is this the best way to solve the issue? Yes for the narrow request: consuming the existing Gateway `includeDerivedTitles` contract is the smallest maintainable UI fix, while manual renaming and AI title generation should remain separate.
