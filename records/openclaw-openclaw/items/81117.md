## Summary

Keep open: current Control UI still does not request or render `derivedTitle` for the chat session picker, while Gateway already exposes bounded derived-title support. The related AI auto-title issue and PR are broader and do not supersede this narrow UI adoption request.

## What This Changes

Request Control UI to show readable derived session titles in its session picker instead of raw session keys.

## Best Possible Solution

Add `derivedTitle` to the UI session row contract, request bounded derived titles for the Control UI session picker, and render them after manual labels/display names but before raw key fallbacks.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a bug reproduction, but source inspection gives high confidence: the Control UI omits `includeDerivedTitles` and the picker label path ignores `derivedTitle`.

## Solution Assessment

Is this the best way to solve the issue? Yes for the narrow request: use the existing Gateway `includeDerivedTitles` contract with existing list bounds instead of adding AI title generation, new config, or manual renaming here.
