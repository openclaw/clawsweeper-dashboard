## Summary

Keep open: current main still exposes Gateway-derived titles but the Control UI session picker does not request `includeDerivedTitles`, does not type `derivedTitle`, and does not use it in option labels. Related title-generation issues and PRs are broader or overlapping, but they do not supersede this narrow UI adoption request.

## What This Changes

Show derived session titles in the Control UI session picker instead of falling back to opaque session IDs.

## Best Possible Solution

Add `derivedTitle` to the UI session row contract, request bounded derived titles from `sessions.list`, and use them after manual `label`/`displayName` but before parsed-key fallbacks; leave AI summarization and manual renaming to their separate tracks.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a bug reproduction because this is a feature request, but source inspection gives high confidence that current main omits `includeDerivedTitles` in the Control UI request and ignores `derivedTitle` in picker labels.

## Solution Assessment

Is this the best way to solve the issue? Yes for the narrow request: consume the existing Gateway-derived-title contract in the Control UI picker instead of adding new AI generation, config, or rename behavior here.
