## Summary

Keep open: current Control UI code still does not request or render `derivedTitle` for the chat session picker, while Gateway already exposes the field. Related broader auto-title work exists, but it does not supersede this narrower UI adoption request.

## What This Changes

Request to show readable derived conversation titles in the Control UI session picker instead of raw session keys.

## Best Possible Solution

Add `derivedTitle` to the UI session row contract, request bounded derived titles from `sessions.list`, and use them after manual labels but before raw key fallbacks; leave manual renaming and AI title generation to separate title-management work.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a bug repro, but source inspection gives high confidence: the Control UI omits `includeDerivedTitles` and the picker label path ignores `derivedTitle`.

## Solution Assessment

Is this the best way to solve the issue? Yes for the narrow request: use the existing Gateway `includeDerivedTitles` contract in the bounded Control UI session list path instead of adding new generation, config, or manual rename behavior here.
