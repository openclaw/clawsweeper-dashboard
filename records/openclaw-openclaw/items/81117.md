## Summary

Keep open: current main still has the Gateway derived-title contract, but the Control UI session picker does not request or render `derivedTitle`; the broader AI-title work does not supersede this narrower UI adoption request.

## What This Changes

The issue requests that the Control UI session picker use `sessions.list` derived titles as readable labels instead of raw session keys.

## Best Possible Solution

Update Control UI to request `includeDerivedTitles`, carry `derivedTitle` in its session row type, and prefer manual labels/display names before derived titles, falling back to parsed keys only when no readable title exists.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high confidence that current main omits `includeDerivedTitles` in the Control UI request and ignores `derivedTitle` in picker label resolution.

## Solution Assessment

Yes for the narrow request: use the existing Gateway `includeDerivedTitles` contract in the bounded Control UI session-list path instead of adding AI title generation, new config, or manual rename behavior here.
