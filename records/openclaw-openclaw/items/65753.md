## Summary

Keep open: current main still has the docs link-audit tilde-fence gap, but this PR is not merge-ready because its closer detection treats marker-prefixed content as a closing fence and the external contributor has not supplied real after-fix proof.

## What This Changes

The PR updates the docs link audit to recognize backtick and tilde fence delimiters and adds parser unit tests.

## Best Possible Solution

Separate opener and closer parsing, require closing fences to match marker and length with only trailing whitespace, add audit-loop regressions for tilde fences and marker-prefixed content, then provide real after-fix proof.

## Reproduction Assessment

Yes, at source level. Current main only skips backtick fences, and the prior ClawSweeper review recorded an in-memory check showing tilde-fence links are still audited while the PR logic prematurely closes on `~~~not-a-close`.

## Solution Assessment

No. The PR targets the right script, but using one prefix parser for both openers and closers is not sufficient because closing fences must reject non-whitespace suffix text.
