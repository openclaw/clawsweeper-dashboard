## Summary

Keep open: the underlying docs-audit bug is real, but the patch is not mergeable because it treats any matching fence prefix as a closer and still lacks contributor-owned real behavior proof.

## What This Changes

The PR changes the docs link-audit script to track backtick and tilde fence delimiters and adds parser unit tests.

## Best Possible Solution

Separate opener and closer parsing, require closing fences to match marker and length with only trailing whitespace, add audit-loop regressions for tilde fences and marker-prefixed content, then provide real after-fix proof.

## Reproduction Assessment

Yes, at source level. A read-only in-memory run of the current audit loop reports a route inside a `~~~md` block, and the same check against the PR logic reports a route after `~~~not-a-close` even though it should still be inside code.

## Solution Assessment

No. The PR targets the right script, but using one prefix parser for both openers and closers is not sufficient because closing fences must reject non-whitespace suffix text.
