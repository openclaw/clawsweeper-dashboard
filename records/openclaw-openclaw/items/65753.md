## Summary

Keep this PR open. It addresses a real current-main docs audit bug, but the proposed close-detection still treats fenced content like `~~~not-a-close` as a closing fence and can resume link auditing too early.

## What This Changes

The PR adds a fence delimiter parser to `scripts/docs-link-audit.mjs` so docs link auditing skips backtick and tilde fenced code blocks, plus parser unit tests.

## Best Possible Solution

Land a narrow docs tooling fix that parses opening and closing fence rules separately, preserves marker and length semantics, and adds audit-loop regressions for tilde fences and content lines that begin with fence markers.

## Reproduction Assessment

Yes. A read-only Node simulation of the current main audit loop reports `/missing-openclaw-route` from inside a `~~~md` block because current main only toggles code-fence state for backticks.

## Solution Assessment

No. The PR direction is right, but closing fences need same marker, sufficient length, and only trailing whitespace after the marker; the proposed parser accepts `~~~not-a-close` as a close.
