## Summary

Keep open because current main and the latest shipped tag still render cleanup commands for the profile-owned gateway after detecting extra gateway-like services, and the linked implementation PR is still open and conflicting rather than merged.

## What This Changes

Doctor should stop suggesting cleanup commands for the active macOS gateway LaunchAgent and should handle exec-backed SecretRefs in doctor without exiting incorrectly.

## Best Possible Solution

Land a revised https://github.com/openclaw/openclaw/pull/52515 or replacement that carries detected service metadata into doctor/status cleanup hints, suppresses active gateway cleanup guidance on macOS, honors service scope, and keeps strict SecretRef runtime helpers fail-closed.

## Reproduction Assessment

Yes, at source level for the cleanup-hint symptom: current tests show custom LaunchAgents executing openclaw gateway are reported as extra services, and current doctor/status code renders cleanup hints without detected service context. The exec:keychain symptom was not live-reproduced here.

## Solution Assessment

No, current main is not a complete solution and the linked PR is not ready as-is. The maintainable fix is to render cleanup hints from actual detected service metadata and use explicit SecretRef inspection semantics rather than weakening strict helpers.
