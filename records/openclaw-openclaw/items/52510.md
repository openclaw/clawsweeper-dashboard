## Summary

Keep open. Current main has tightened the broad macOS LaunchAgent detection, but doctor and status still render cleanup commands from the active profile instead of the detected extra service, and the paired implementation PR is still open and conflicting with review blockers.

## What This Changes

The issue reports that `openclaw doctor` can suggest removing the active gateway LaunchAgent plist and can fail on runtime-only `exec:keychain:` SecretRefs.

## Best Possible Solution

Keep this issue open until https://github.com/openclaw/openclaw/pull/52515 or a replacement lands a service-scoped cleanup-hint fix for doctor and status while preserving strict SecretRef behavior outside explicit read-only inspection.

## Reproduction Assessment

Yes, source-level. Current tests can produce a custom `openclaw gateway` LaunchAgent as an extra service, and current doctor/status code still renders cleanup hints without the detected service context; I did not run macOS launchd in this read-only review.

## Solution Assessment

Yes for the cleanup-hint direction, but not as currently merged because it is not on main. The safest fix is to render commands from actual detected service metadata in both doctor and status, and handle the SecretRef part through the existing inspect-mode contract rather than weakening strict helpers.
