## Summary

Keep open. Current main and v2026.5.18 still render cleanup commands from the active profile after detecting extra gateway services, and the linked implementation PR remains open and conflicting rather than merged.

## What This Changes

Doctor should stop suggesting cleanup commands that remove the active gateway LaunchAgent and should treat exec SecretRef values as non-fatal read-only diagnostics.

## Best Possible Solution

Land a revised version of https://github.com/openclaw/openclaw/pull/52515 or a replacement that carries detected service metadata into doctor/status cleanup hints, suppresses active gateway cleanup guidance, and handles doctor SecretRef inspection without weakening strict runtime helpers.

## Reproduction Assessment

Yes, source-level. Current tests show custom LaunchAgents executing openclaw gateway are reported as extra services, and current doctor/status code then renders cleanup hints without that service context.

## Solution Assessment

No, current main is not a complete solution. The maintainable fix is to render cleanup hints from actual detected service metadata and use explicit SecretRef inspection semantics instead of changing strict helper behavior.
