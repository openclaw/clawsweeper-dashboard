## Summary

Keep open. Current main still has the no-context cleanup-hint path: detected extra gateway-like services in doctor/status can still produce cleanup commands for the active profile gateway, and the open PR that references this issue has not merged.

## What This Changes

The issue asks doctor to stop suggesting removal of the active macOS gateway LaunchAgent plist and to treat unresolved exec SecretRefs during doctor as warnings rather than fatal errors.

## Best Possible Solution

Land a revised or replacement fix that renders cleanup hints from actual detected service metadata in doctor and status, honors user versus system scope, avoids targeting the active profile gateway, preserves legacy manual cleanup guidance, and handles doctor SecretRef checks through an explicit read-only warning path.

Yes, source-level. Current tests show a custom macOS LaunchAgent that executes `openclaw gateway` is reported as an extra service, and current doctor/status callers then render cleanup hints without passing that service context.

## Solution Assessment
