## Summary

Current main still has the cleanup-hint mismatch: doctor/status can list a detected extra service and then print canonical gateway removal commands, so this issue remains necessary while the linked fix PR is open and unmerged.

## What This Changes

The issue reports that doctor/status cleanup hints can target the active macOS gateway LaunchAgent and that doctor exits non-zero on exec SecretRef values.

## Best Possible Solution

Keep the issue open and resolve it through the linked fix path or a narrower replacement that makes cleanup hints target only the detected stale service while preserving SecretRef-aware read-only doctor behavior.

## Reproduction Assessment

Yes by source inspection for the cleanup-hint half: current main can report a custom macOS `openclaw gateway` plist while both doctor and status still print canonical gateway cleanup commands. I did not run a live macOS launchd scenario in this read-only review.

## Solution Assessment

The requested cleanup-hint fix direction is the right shape, but the exact linked PR still needs review because its SecretRef handling may be broader than the remaining current-main gap. The safest solution is a focused target-aware cleanup hint plus scoped read-only SecretRef degradation.
