## Summary

Keep open. Current main has partially tightened macOS LaunchAgent detection, but the doctor cleanup hint still renders canonical gateway removal commands without knowing which service was detected, and open unmerged PRs already reference this issue with closing syntax.

## What This Changes

`openclaw doctor` should stop suggesting removal of the active gateway LaunchAgent for false-positive or extra LaunchAgent detections, and should avoid a hard failure for runtime-only `exec:` SecretRefs in doctor paths.

## Best Possible Solution

Keep this issue open and review PR #52515 or a smaller rebased equivalent until cleanup hints are detected-service-specific or suppressed when unsafe, with focused regression coverage for macOS LaunchAgent targeting and the reported doctor `exec:` SecretRef behavior.

## Reproduction Assessment

Yes for the cleanup-hint class: current source and tests show a macOS extra gateway-like LaunchAgent can still trigger canonical gateway removal hints. The exact `exec:keychain:` exit-code path is not fully reproduced from current-main evidence and needs a targeted doctor-path test.

## Solution Assessment

No current-main solution is complete. The scanner tightening is useful, but the safer fix is to pass detected service context into cleanup hint rendering or suppress active canonical gateway cleanup, while preserving strict SecretRef behavior outside read-only doctor/status paths.
