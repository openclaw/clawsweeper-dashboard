## Summary

Keep open: current main fixed part of the report by tightening macOS LaunchAgent detection and by making exec-backed Gateway SecretRefs non-fatal by default, but the cleanup hint path still emits static commands for the canonical active gateway whenever any extra service remains. The linked fix PR is still open and conflicting, so this issue should stay open until that PR, or a narrower replacement, lands.

## What This Changes

The issue asks doctor to stop suggesting cleanup commands that remove the active macOS gateway LaunchAgent and to treat runtime-only exec SecretRefs as non-fatal during doctor checks.

## Best Possible Solution

Review, repair, or replace https://github.com/openclaw/openclaw/pull/52515 so cleanup hints are derived from the detected stale/legacy services, while preserving the current stricter LaunchAgent detection and exec-SecretRef read-only behavior.

## Reproduction Assessment

Yes. Source inspection shows that when `findExtraGatewayServices` returns any remaining extra service, doctor still calls a no-argument cleanup helper that targets the canonical gateway profile; no live macOS launchd run was performed.

## Solution Assessment

No, current main is only a partial solution. The maintainable fix is to keep the stricter detector and make cleanup hints target the detected stale/legacy service, or suppress them when OpenClaw cannot prove a safe cleanup target.
