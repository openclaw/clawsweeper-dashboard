## Summary

Keep open. Current main and the latest shipped release still render cleanup commands from the active profile gateway instead of the detected extra service, and the issue is paired with an open conflicting implementation PR that has not landed.

## What This Changes

The issue reports that `openclaw doctor` can suggest removing the active macOS gateway LaunchAgent and can treat `exec:keychain:` SecretRef config as a hard doctor failure.

## Best Possible Solution

Land a revised or replacement fix that renders doctor/status cleanup hints from actual detected service metadata, suppresses active gateway cleanup guidance, honors service scope, and handles read-only SecretRef inspection without weakening strict runtime helpers.

## Reproduction Assessment

Yes for the cleanup-hint path at source level: current tests can produce extra macOS gateway-like services, and both doctor and status still render cleanup hints without detected service context. The `exec:keychain:` part is not live-reproduced in this review.

## Solution Assessment

No, current main is not a complete solution and the linked PR is not ready as-is. The maintainable path is a scoped cleanup-hint fix plus explicit read-only SecretRef inspection semantics, not process-wide leniency in strict helpers.
