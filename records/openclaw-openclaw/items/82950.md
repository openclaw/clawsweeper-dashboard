## Summary

Keep this PR open: current main and the latest release still compile exec approval `argPattern` strings directly, while this branch is a narrow candidate fix with sufficient proof and a maintainer-visible fail-closed compatibility decision.

## What This Changes

The branch adds a `hasNestedRepetition` guard before exec approval `argPattern` regex compilation and adds regression coverage for unsafe and safe allowlist patterns.

## Best Possible Solution

Land this PR or an equivalent narrow guard after maintainers explicitly accept the fail-closed behavior for unsafe stored `argPattern` entries.

## Reproduction Assessment

## Solution Assessment

Yes, subject to maintainer compatibility acceptance. Guarding with `hasNestedRepetition` immediately before compiling the original untrimmed pattern is narrower than switching this matcher to `compileSafeRegex` and preserves current whitespace semantics.
