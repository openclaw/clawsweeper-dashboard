## Summary

## What This Changes

This PR changes exec-approval `argPattern` matching to use `compileSafeRegex` and adds a changelog bullet for ReDoS hardening.

## Best Possible Solution

Validate nested repetition without normalizing the policy regex, compile the original `argPattern`, add focused regression coverage, and require redacted real allowlist proof before merge.

## Reproduction Assessment

Yes for the source path: current main passes `argPattern` directly to `new RegExp`, and the existing safe-regex parser rejects nested repetition patterns. I did not establish a live OpenClaw allowlist hang in this read-only review.

## Solution Assessment

No, not as patched: the narrow fix should reject nested repetition while compiling the original untrimmed `argPattern`. The current helper changes policy regex text and the PR still needs committed regression coverage plus real allowlist proof.
