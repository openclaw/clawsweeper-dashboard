## Summary

## What This Changes

The PR adds a nested-repetition guard before compiling exec approval argPattern regexes and expands coverage for unsafe and safe patterns.

## Best Possible Solution

Land the guard with maintainer-approved operator behavior for rejected unsafe argPatterns: either surface a safe diagnostic/docs path with tests, or explicitly accept and document silent fail-closed semantics for stored policies.

## Solution Assessment

No, not fully: the guard is a good narrow mitigation and preserves original pattern text, but silently returning false is not proven to be the best shipped operator behavior for persisted allowlist policy.
