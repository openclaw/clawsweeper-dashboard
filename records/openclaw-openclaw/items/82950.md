## Summary

Keep this PR open as the canonical fix candidate: current main and the latest release still compile exec approval `argPattern` values directly, while the PR is narrowly scoped and has sufficient terminal proof. ## What This Changes

The PR adds a nested-repetition guard before exec approval `argPattern` regex compilation and expands tests for unsafe and safe allowlist patterns.

## Best Possible Solution

Land this PR or an equivalent narrow guard after maintainers explicitly accept the fail-closed behavior for unsafe stored `argPattern` entries.

## Reproduction Assessment

Yes from source. Stored `entry.argPattern` reaches direct `new RegExp(argPattern).test(...)` through `matchAllowlist`; this read-only review did not run a deliberate hang case.

## Solution Assessment

Yes, subject to maintainer compatibility acceptance. Guarding with `hasNestedRepetition` immediately before compiling the original untrimmed pattern is narrower than switching this matcher to `compileSafeRegex` and preserves current whitespace semantics.
