## Summary

## What This Changes

The PR adds a hasNestedRepetition guard before exec approval argPattern regex compilation and adds ReDoS/safe-pattern regression tests.

## Best Possible Solution

Land this PR or an equivalent narrow guard after maintainers explicitly accept the fail-closed behavior for unsafe stored argPattern entries.

## Reproduction Assessment

## Solution Assessment

Yes, subject to maintainer compatibility acceptance. Guarding with hasNestedRepetition immediately before compiling the original untrimmed pattern is narrower than switching this matcher to compileSafeRegex, which would trim and alter pattern semantics.
