## Summary

Keep open: current main and v2026.6.10 still have a Feishu tool resolver path where a non-contextual tool call can choose the global Feishu default before any binding-aware account selection, and the account-precedence change is security- and compatibility-sensitive.

## What This Changes

The issue asks OpenClaw to route Feishu resource tool calls from non-inbound turns through the agent's bound Feishu account instead of the global Feishu default account.

## Best Possible Solution

Define a maintainer-approved Feishu tool account precedence rule that keeps explicit and valid contextual accounts first, uses a safe binding-derived or enabled-family account before global default, and covers cron, relay, and heartbeat paths with regression plus live or redacted runtime proof.

## Reproduction Assessment

Yes at the resolver level: a Feishu tool invocation with no valid ctx.agentAccountId reaches resolveImplicitToolAccountId and can return channels.feishu.defaultAccount before any enabled-family fallback. The full cron, sessions_send, and heartbeat trigger matrix still needs live or focused runtime proof.

## Solution Assessment

Unclear. Binding-aware precedence is likely the right direction, but the exact ordering across explicit tool account, turn context, session-derived account, agent binding, enabled-family fallback, and global default is security- and compatibility-sensitive.
