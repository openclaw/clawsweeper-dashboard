Full review comments:

- **[P1] Do not treat auth profiles as OAuth limits:** `src/infra/provider-usage.limits.ts:31`
  - body: `authMode` is only the mechanism: current embedded/CLI runners set it to `auth-profile` whenever a profile is selected, and profiles can be `api_key`, `token`, or `oauth`. Mapping `auth-profile` and missing values to OAuth means an OpenAI API-key profile can still resolve the OAuth usage provider and attach ChatGPT limit windows to an API-key turn; thread the actual credential type/profile id or omit limits when only the mechanism is known.
  - confidence: 0.91
- **[P2] Keep usageState scoped to the exact run:** `src/auto-reply/dispatch.ts:343`
  - body: This call falls back to `sessionKey`, while `consumeReplyUsageState` never deletes the stored snapshot and dispatch can install the hook without a current `runId`. A later command or non-agent reply in the same session within five minutes can receive the previous turn's model/token/limit data; only attach usageState on an exact live run match or clear the fallback after the matching delivery.
  - confidence: 0.87
