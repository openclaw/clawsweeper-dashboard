Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Carry usageState through durable reply delivery:** `src/auto-reply/dispatch.ts:343`
  - body: This only passes `usageState` when the in-memory dispatcher fires the hook. Routed replies go through `routeReply` and `sendDurableMessageBatch`, whose queued hook stores only kind/channel/sessionKey/runId/context before `deliver.ts` re-runs `reply_payload_sending`, so routed or recovered deliveries still get no `usageState` even though the hook contract includes those replies. Extend the queued hook carrier and replay path too.
  - confidence: 0.9
- **[P2] Respect the active auth mode for limits:** `src/auto-reply/reply/agent-runner.ts:1756`
  - body: `getProviderUsageLimitsCached(providerUsed)` defaults the helper to OAuth, while `resolveUsageProviderId` intentionally returns no OpenAI usage provider for API-key auth. On a gateway with both OpenAI API-key and OAuth credentials, an API-key turn can expose OAuth/ChatGPT usage windows to plugins instead of leaving `limits` undefined. Pass the resolved run auth mode into the limits lookup.
  - confidence: 0.86
