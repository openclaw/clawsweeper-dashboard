## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Stop falling back usageState by session key:** `src/auto-reply/dispatch.ts:343`
  - body: This attaches `usageState` by `sessionKey` whenever `runId` is absent. Current live reply paths can omit `replyOptions.runId`, so a later same-session delivery can pick up the previous turn's model/usage/cost/limits for up to five minutes. Thread the exact run id into the live delivery path or leave `usageState` undefined when the active run cannot be proven.
  - confidence: 0.91
- **[P1] Scope OpenAI limits to the resolved credential profile:** `src/infra/provider-usage.limits.ts:40`
  - body: `auth-profile` is a mechanism, not a credential type: current runner metadata emits it for any selected profile, and OpenAI profiles may be API-key or OAuth. Mapping all `auth-profile` turns to OAuth, then caching by provider only, lets an API-key OpenAI turn borrow OAuth/ChatGPT usage windows. Carry the resolved credential/profile identity into the limits resolver and cache key, or omit limits when it is unavailable.
  - confidence: 0.89
- **[P3] Document the new usageState hook contract:** `src/plugins/hook-types.ts:568-577`
  - body: This adds an exported plugin hook contract, but the public hook docs still only describe payload rewriting and existing correlation fields. Document the `usageState` fields, best-effort live-delivery scope, routed/replay absence cases, and graceful degradation expectations before shipping it as plugin API.
  - confidence: 0.86
