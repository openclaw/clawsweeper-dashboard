## Summary

Keep open. Current main still does not expose a per-spawn tool selector or tool-less mode for native `sessions_spawn`: the public tool schema, native spawn params, child `agent` RPC schema, spawn handoff, and session metadata all lack a field that can carry `tools`, `toolsAllow`, `disableTools`, or deny-list semantics. The existing lower-level embedded-run allowlist is useful implementation substrate, but the issue's local-model and plugin/MCP use cases remain unresolved API work alongside #15032 and #62695.

## What This Changes

Add a per-spawn tool override to `sessions_spawn` so spawned subagents can receive all tools, no tools, or a selected subset for local-model and least-privilege workflows.

## Best Possible Solution

Keep this open until maintainers choose a canonical per-spawn tool-policy API across this issue, #15032, and #62695. The eventual implementation should be backward-compatible, document the public `sessions_spawn` field, carry it through native spawn params and the gateway child-run protocol, map tool-less runs onto the existing embedded-run no-tools path, map subsets onto the existing allowlist primitive, and define plugin/MCP resolution and policy precedence explicitly.
