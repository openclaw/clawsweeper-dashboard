## Summary

Keep this issue open. Current main fixed the shared `resolveSubagentConfiguredModelSelection` helper, but the isolated cron/shared agent-scope path still selects the agent model before the global subagent default, and linked open PRs already target the remaining work.

## What This Changes

The issue asks to restore subagent model precedence so per-agent subagent config wins first, global subagent defaults win second, and the agent's own model is only the fallback.

## Best Possible Solution

Land or supersede one canonical fix that makes every subagent selection path honor the documented precedence and updates the regression tests that currently pin the old order.

## Reproduction Assessment

Yes, at source level. Current main still orders `agentConfig.model` before `agents.defaults.subagents.model` in `resolveSubagentModelConfigSelectionResult`, and a checked-in cron test asserts that old order.

## Solution Assessment

Yes for the reported bug boundary. The maintainable solution is to align agent-scope, cron, runtime session behavior, docs, and tests around the documented subagent default precedence.
