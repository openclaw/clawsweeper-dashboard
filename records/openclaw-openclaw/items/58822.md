## Label Justifications

- `P2`: This is a real model-selection regression with clear current-main source evidence, but it affects configured subagent/default model setups rather than making the core runtime unusable.
- `impact:session-state`: The remaining paths can keep or inherit the wrong model choice through subagent or parent session state.
- `impact:auth-provider`: The wrong model selection can route subagent work to the wrong provider/auth profile path.

## Summary

Keep this issue open. Current main fixed the direct subagent spawn helper, but the shared `agent-scope` helper used by isolated cron still checks the agent model before the global subagent default, and open closing PRs already target the remaining fix path.

## What This Changes

The issue asks OpenClaw to restore subagent model precedence so per-agent subagent config wins first, then `agents.defaults.subagents.model`, with the agent's own model only as a fallback.

## Best Possible Solution

Land or supersede one canonical fix that aligns `agent-scope`, isolated cron selection, runtime parent override handling, and regression tests with the documented subagent default precedence.

## Reproduction Assessment

Yes, at source level. Current main's `resolveSubagentModelConfigSelectionResult` returns the agent model before the global subagent default, cron consumes that helper, and a checked-in cron test still expects the old order.

## Solution Assessment

Yes for the reported bug boundary. The maintainable fix is to make every subagent model-selection path honor per-agent subagent config first, then the global subagent default, then agent or parent fallback only when no subagent default applies.
