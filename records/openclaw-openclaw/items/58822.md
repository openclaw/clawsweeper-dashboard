## Label Justifications

- `P2`: This is a real model-selection regression with a clear affected surface, but it is limited to configured subagent/default model setups rather than a core outage.
- `impact:session-state`: The remaining runtime path can persist or inherit the wrong model through parent session override state for subagent sessions.
- `impact:auth-provider`: The bug routes subagent runs to the wrong configured provider/model and can change auth or provider usage unexpectedly.

## Summary

Keep this issue open. Current main and v2026.5.18 fix the shared subagent helper precedence, but cron still has a source-visible path that prefers the agent model before the global subagent default, and runtime session model override handling can still let parent session state win before any subagent-default exception. Open PRs https://github.com/openclaw/openclaw/pull/58823 and https://github.com/openclaw/openclaw/pull/72984 already target the remaining work.

## What This Changes

This issue reports a regression where subagents spawned from a main agent use the parent agent model instead of the configured global subagent default.

## Best Possible Solution

Land or supersede one canonical fix that aligns helper precedence, isolated cron selection, runtime subagent session behavior, docs, and regression tests around the documented subagent default contract.

## Reproduction Assessment

Yes, at source level. Current main's cron selection still orders the agent model before the global subagent default, and the stored parent override path can still return a parent model before any subagent-default exception.

## Solution Assessment

Yes for the reported bug boundary. The maintainable fix is to make every subagent selection path honor explicit per-agent subagent config first, then the global subagent default, then parent or agent fallback.
