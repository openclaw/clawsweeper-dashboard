## Label Justifications

- `P2`: The report is a normal-priority model routing regression with limited but real impact on subagent runs.
- `impact:session-state`: The issue is about subagent session model selection drifting from the configured subagent default.
- `impact:auth-provider`: The wrong resolved model can send subagent work through the wrong provider or auth profile.

## Summary

Keep open: current main only fixed the native model-selection resolver; the shared agent-scope helper still evaluates the agent primary before the global subagent default, the isolated cron path consumes that helper, and the open clean PR https://github.com/openclaw/openclaw/pull/58823 is the direct closing fix.

## What This Changes

The issue asks OpenClaw to make subagent model selection prefer per-agent subagent config, then `agents.defaults.subagents.model`, and use the parent agent's own model only as a last-resort fallback.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/58823, or explicitly reject that precedence and update source, docs, and tests to one canonical contract.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: configure an agent primary and `agents.defaults.subagents.model`, then call the shared subagent model helper or isolated cron model selection path; the helper still evaluates the agent primary before the global subagent default.

## Solution Assessment

Yes, pending maintainer acceptance of the compatibility tradeoff. Applying the same precedence to the shared agent-scope helper and cron tests is the narrowest maintainable fix, and the open linked PR already targets that exact path.
