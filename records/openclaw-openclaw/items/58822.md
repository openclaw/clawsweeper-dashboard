## Label Justifications

- `P2`: The bug can route subagent or cron runs to the wrong configured model, but the affected surface is bounded to subagent model precedence.
- `impact:session-state`: The issue affects spawned subagent session model state and can make child runs inherit the parent model instead of the configured subagent default.
- `impact:auth-provider`: Selecting the wrong model can also select the wrong provider, auth profile, runtime policy, fallback chain, or cost profile for subagent execution.

## Summary

Keep open: current main and the latest release still leave the shared agent-scope and isolated-cron selection path with the parent agent model ahead of the global subagent default, and an open closing PR already owns the narrow fix.

## What This Changes

Restore subagent model precedence so per-agent subagent config wins first, then the global subagent default, with the parent agent model only as a fallback.

## Best Possible Solution

Land one focused fix, preferably https://github.com/openclaw/openclaw/pull/58823 or a reconciled equivalent, that aligns `resolveSubagentModelConfigSelectionResult` and cron tests with the already-corrected native helper, then close this issue from the merge.

## Reproduction Assessment

Yes, at source level. Configure an agent primary model plus a different `agents.defaults.subagents.model`; current `agent-scope` checks the agent primary before the global subagent default, and isolated cron consumes that helper.

## Solution Assessment

Yes, the narrow maintainable fix is to reorder the shared helper and update the cron/shared-helper tests so they match the documented precedence and the already-corrected native helper. Maintainers still need to coordinate this with the broader runtime-persistence PR before landing.
