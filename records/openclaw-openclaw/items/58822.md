## Label Justifications

- `P2`: This is a bounded but real agents model-selection bug with provider-routing impact and an open fix candidate, not an outage or emergency.
- `impact:session-state`: The wrong selected subagent model can be carried into child session/runtime state and make spawned or cron subagent behavior drift from configured defaults.
- `impact:auth-provider`: The bug can select the parent agent's provider/model instead of the configured global subagent provider/model, changing model choice and auth routing.

## Summary

Keep open: current main and the latest release still leave the shared `agent-scope`/cron path with agent-primary precedence ahead of the global subagent default, and an open closing PR already owns the narrow fix.

## What This Changes

Restore subagent model precedence so per-agent `subagents.model` wins first, global `agents.defaults.subagents.model` wins next, and the agent's own model is only the last-resort fallback.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/58823 or a maintainer-selected narrow equivalent for the shared helper and cron precedence, then close this issue from the merged PR while coordinating broader runtime-persistence work separately.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction: configure an agent with its own model plus `agents.defaults.subagents.model`, and the shared `agent-scope` helper still chooses the agent model before the global subagent default; I did not run tests because this review is read-only.

## Solution Assessment

Yes, the narrow helper-order fix is the best code direction for the reported bug because direct spawn selection already has the documented order and cron consumes the shared helper. Maintainers still need to coordinate the open broader runtime-persistence PR separately.
