## Summary

Keep this issue open. Current main and the latest release still check the agent's own model before `agents.defaults.subagents.model` in the native and isolated cron subagent model paths, while the docs and a user confirmation support the reported global subagent default precedence; open implementation PRs already cover the fix path.

## What This Changes

The issue requests restoring global `agents.defaults.subagents.model` precedence over an agent's own model for spawned subagents in native and isolated cron selection.

## Best Possible Solution

Land or supersede the open implementation with one canonical fix that aligns helper precedence, isolated cron selection, runtime subagent session behavior, docs, and regression tests around the documented subagent model default contract.

## Reproduction Assessment

Yes. The report's YAML maps directly to current main: both resolver paths check the agent's own model before `agents.defaults.subagents.model`, and checked-in tests assert that same agent-over-global result.

## Solution Assessment

Yes for the reported bug boundary: swapping the global subagent default ahead of the agent's own model in both resolver paths is the narrow maintainable fix. The broader runtime persistence behavior should be handled by the canonical open PR if maintainers choose that path.
