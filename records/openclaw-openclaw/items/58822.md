## Summary

Keep open. Current main and the latest release still put the target agent's own model ahead of `agents.defaults.subagents.model` in both native subagent selection and isolated cron selection, while the docs and confirmed reproduction support the requested global subagent default precedence. Open PR #58823 remains the paired implementation candidate, so the issue should stay open until that PR lands or is superseded.

## What This Changes

The issue requests restoring subagent model precedence so per-agent `subagents.model` wins first, then `agents.defaults.subagents.model`, with the parent agent's own model used only as a last-resort fallback in native and isolated cron paths.

## Best Possible Solution

Review and land or explicitly supersede #58823 with a narrow fix that changes the native and isolated cron precedence plus their regression tests, preserving explicit spawn/job overrides and per-agent `subagents.model` as the highest-priority configured override.

## Reproduction Assessment

Yes. The report's YAML maps directly to the current fallback chain: `resolveAgentConfig` returns the agent entry model, and both resolvers check that model before `agents.defaults.subagents.model`; existing tests assert the same agent-over-global result.

## Solution Assessment

Yes. The narrow maintainable solution is to swap the global subagent default ahead of the agent's own model in both resolver paths and update the tests to match the documented contract, while leaving explicit overrides untouched.
