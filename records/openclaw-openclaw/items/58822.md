## Summary

Keep this issue open. Current main fixed the direct `sessions_spawn` helper precedence, but the shared `agent-scope` helper still places the agent primary model before `agents.defaults.subagents.model`, and isolated cron consumes that helper, so the central bug is not fully implemented on main.

## What This Changes

The issue reports a regression where subagents can inherit the parent agent's primary model instead of the configured global subagent default.

## Best Possible Solution

Land or replace the linked fix PR so the shared helper and cron path use the same subagent model precedence as the direct spawn helper.

## Reproduction Assessment

Yes from source: current main's `agent-scope` helper still selects the agent primary before the global subagent default, and cron calls that helper. I did not run a live repro because this review is read-only.

## Solution Assessment

Yes for the requested fix direction, but no to closing now. The narrow maintainable solution is to align the shared helper and cron tests with the direct spawn helper precedence.
