## Label Justifications

- `P2`: The bug can route subagent or cron child runs to the wrong configured model, but it is bounded to model-selection paths rather than a core outage.
- `impact:session-state`: The reported behavior affects the effective model stored or used for spawned subagent session state.
- `impact:auth-provider`: The wrong model precedence can route subagent runs through a different provider/auth profile than the configured subagent default.

## Summary

Keep this issue open: current main fixed the direct spawn resolver, but the shared agent-scope selection helper still lets the agent primary model beat the global subagent default, and cron uses that helper. The issue also has open closing PRs, especially https://github.com/openclaw/openclaw/pull/58823, so it should close only after a canonical fix lands.

## What This Changes

The issue asks OpenClaw to make `agents.defaults.subagents.model` take precedence over an agent's own primary model for subagent model selection, except when a per-agent `subagents.model` is set.

## Best Possible Solution

Keep the issue open until maintainers land either the narrow shared-helper precedence fix in https://github.com/openclaw/openclaw/pull/58823 or a broader canonical replacement such as https://github.com/openclaw/openclaw/pull/72984, with release-note context for the provider-routing change.

## Reproduction Assessment

Yes. Source inspection on current main shows `resolveSubagentModelConfigSelectionResult` still picks the agent primary before the global subagent default, and the cron path consumes that helper.

## Solution Assessment

Yes on the issue direction, but unclear which open PR should be canonical. The maintainable end state is one shared precedence contract across direct spawn, agent-scope, cron, and tests, plus explicit upgrade/release context for the provider-routing behavior change.
