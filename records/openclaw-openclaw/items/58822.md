- `P2`: This is a real model-selection regression with clear code evidence, but it affects configured subagent/default model setups rather than making the core runtime unusable.
- `impact:session-state`: The remaining paths can persist or inherit the wrong model choice through subagent or parent session state.
- `impact:auth-provider`: The bug can route subagent runs to the wrong configured provider/model and therefore the wrong auth/provider path.

## Summary

Keep this issue open. Current main fixed the shared subagent helper, but the `agent-scope` helper still checks the agent model before `agents.defaults.subagents.model`, cron consumes that helper, and open PRs already target the remaining work.

## What This Changes

The issue requests restoring subagent model precedence so per-agent subagent config wins first, then the global subagent default, then the parent or agent model only as fallback.

## Best Possible Solution

Land or supersede one canonical fix that aligns `agent-scope`, isolated cron selection, runtime parent-override handling, and regression tests with the documented subagent default contract.

## Reproduction Assessment

Yes, at source level. Current main's `resolveSubagentModelConfigSelectionResult` returns the agent model before the global subagent default, cron consumes that result, and an existing cron test still expects the old order.

## Solution Assessment

Yes for the reported bug boundary. The maintainable fix is to make every subagent model-selection path honor per-agent subagent config first, then the global subagent default, then agent or parent fallback only when no subagent default applies.
