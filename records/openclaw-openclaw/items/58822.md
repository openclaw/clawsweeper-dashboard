## Summary

Keep open: current main and the latest release still leave the shared agent-scope/isolated-cron selection path with the parent agent model ahead of the global subagent default, and an open closing PR already owns the narrow fix.

## What This Changes

The issue asks to restore subagent model precedence so per-agent subagent overrides win first, then global subagent defaults, then the parent agent primary model as a fallback.

## Best Possible Solution

Land one focused fix, preferably the existing closing PR or a reconciled replacement, that aligns `resolveSubagentModelConfigSelectionResult` and cron tests with the already-corrected native helper, then close this issue from the merge.
