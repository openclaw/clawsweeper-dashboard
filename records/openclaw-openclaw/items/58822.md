## Summary

Keep open: current main still has the shared agent-scope selection path and isolated cron consumer using the agent primary before the global subagent default, while the direct closing PR remains open and clean.

## What This Changes

The issue reports that subagents spawned from an agent with its own model can ignore `agents.defaults.subagents.model` and use the parent agent model instead.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/58823 if maintainers accept the precedence contract; otherwise update source, tests, and docs to one canonical model-routing rule.

## Reproduction Assessment

Yes. Source inspection on current main shows `resolveSubagentModelConfigSelectionResult` evaluates `agentConfig.model` before `agents.defaults.subagents.model`, and isolated cron calls that helper.

## Solution Assessment

Yes, pending maintainer acceptance of the compatibility tradeoff. Applying the same precedence to the shared helper and cron tests is the narrowest maintainable fix, and the linked PR already targets that path.
