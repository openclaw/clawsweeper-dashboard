## Summary

Keep this issue open. Current main and the latest release have the native helper precedence fixed by https://github.com/openclaw/openclaw/pull/81783, but isolated cron and runtime session paths still have source-visible ways to prefer the agent or parent model before the configured subagent default, and open implementation PRs already target the remaining work.

## What This Changes

The issue reports that spawned subagents should honor `agents.defaults.subagents.model` before falling back to the parent agent's own model.

## Best Possible Solution

Land or supersede one canonical fix that aligns helper precedence, isolated cron selection, runtime subagent session behavior, docs, and regression tests around the documented subagent default contract.

## Reproduction Assessment

Yes, at source level. Current main's isolated cron resolver still chooses the agent override before `agents.defaults.subagents.model`, and the stored parent override path still has no subagent-default exception before returning the parent model.

## Solution Assessment

Yes for the reported bug boundary: the maintainable fix is to make all subagent selection paths honor explicit per-agent subagent config first, then the global subagent default, then parent/agent fallback. The safest next step is maintainer review of the existing canonical PRs rather than opening another repair branch.
