## Summary

Keep open: current main still runs Active Memory recall through an embedded sub-agent on the parent agent/session path and the Codex app-server/native-hook surfaces implicated by the report, while recent issue comments show recurrence on stable builds and the related open items only cover narrower slices of the problem.

## What This Changes

The issue asks OpenClaw to investigate and fix Active Memory latency/timeouts when Codex app-server routing, context-engine projection, auth prewarm, and native hook relay interact with embedded recall runs.

## Best Possible Solution

A maintainer-owned fix should prove the current-main failure mode in a real Codex-backed gateway, then either move Active Memory recall off the embedded Codex/app-server path or isolate app-server startup/native-hook state so recall cannot stall the parent turn, with clear model-routing diagnostics.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Partly: the source path and multiple stable-version logs show the implicated embedded recall/Codex path, but this review did not run a current-main live gateway repro after the latest post-release changes.

## Solution Assessment

Is this the best way to solve the issue? Unclear: a single-shot Active Memory path may be the cleanest architectural fix, but maintainers still need to choose between non-embedded recall, Codex startup/native-hook isolation, or narrower repairs to the current embedded runner path.
