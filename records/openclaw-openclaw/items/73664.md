## Summary

Keep this issue open because current main still lacks llama.cpp `/props.n_ctx` discovery and the open closing PR has not merged yet.

## What This Changes

The report asks the llama.cpp self-hosted provider path to use the actual runtime context window for session token percentages instead of the training context window.

## Best Possible Solution

Review and land #74057, or an equivalent narrow discovery fix that records llama.cpp `/props.n_ctx` as `contextTokens` while keeping `contextWindow` as native metadata.

## Reproduction Assessment

Yes, by source inspection: a llama.cpp server whose `/v1/models` reports a larger training window while `/props` reports a smaller loaded `n_ctx` follows a current-main path that never reads `/props`. I did not run a live llama.cpp server in this read-only review.

## Solution Assessment

Yes. Capturing the runtime cap during self-hosted provider discovery is the narrow maintainable fix; the sessions table should continue consuming stored session/model metadata instead of doing live provider-specific HTTP calls.
