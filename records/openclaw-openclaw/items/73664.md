## Summary

Keep this open because current main still does not discover llama.cpp's loaded runtime `n_ctx`, and #74057 is the open, unmerged closing PR for the reported sessions-table behavior.

## What This Changes

Use the actual llama.cpp runtime context window in session usage displays instead of the model's training context metadata.

## Best Possible Solution

Review and land #74057, or an equivalent narrow self-hosted discovery fix that records llama.cpp `/props.n_ctx` as `contextTokens` while preserving native `contextWindow` metadata.

## Reproduction Assessment

Yes, by source inspection: current main's self-hosted discovery path does not read `/props.n_ctx`, and the sessions table uses stored/configured `contextTokens` for the denominator. I did not run a live llama.cpp server in this read-only review.

## Solution Assessment

Yes. Capturing the runtime cap during self-hosted provider discovery is the narrow maintainable fix; probing llama.cpp from the sessions display path would spread provider-specific behavior into a command surface.
