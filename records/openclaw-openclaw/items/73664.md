## Summary

Keep this issue open: current main still has the reported source path, and PR #74057 is the open, unmerged implementation candidate linked to this issue.

## What This Changes

The issue asks OpenClaw to show llama.cpp's loaded runtime context (`n_ctx`) rather than training context metadata (`n_ctx_train`) in session token percentages.

## Best Possible Solution

Review and land PR #74057 or an equivalent narrow discovery fix that records llama.cpp `/props.n_ctx` as `contextTokens` while leaving native `contextWindow` metadata intact.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: a llama.cpp-compatible server whose `/v1/models` metadata exposes a larger training window while `/props` exposes a smaller loaded `n_ctx` will still go through the current `/models`-only discovery path. I did not run a live llama.cpp server in this read-only review.

## Solution Assessment

Yes. Capturing the loaded runtime cap during self-hosted provider discovery as `contextTokens` is the narrow maintainable fix; the sessions table should continue consuming generic stored/configured model metadata.
