## Summary

Keep this open because current main still lacks llama.cpp runtime `n_ctx` discovery, and the linked closing PR is open and unmerged.

## What This Changes

The issue requests that llama.cpp self-hosted provider sessions show the loaded runtime context (`n_ctx`) rather than the model training context (`n_ctx_train`).

## Best Possible Solution

Review and land #74057, or an equivalent narrow provider-discovery fix that records llama.cpp `/props.n_ctx` as `contextTokens` while preserving native `contextWindow` metadata.

## Reproduction Assessment

Yes, by source inspection: current self-hosted discovery does not read `/props.n_ctx`, and the sessions table uses stored/configured `contextTokens` for its denominator. I did not run a live llama.cpp server in this read-only review.

## Solution Assessment

Yes. Capturing the runtime cap during self-hosted provider discovery is the narrow maintainable fix; probing llama.cpp from the sessions display path would spread provider-specific behavior into a command surface.
