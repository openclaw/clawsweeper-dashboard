## Summary

Keep this issue open because current main still does not discover llama.cpp's runtime `n_ctx`, and #74057 is an open, unmerged closing PR for the same report.

## What This Changes

The issue requests that llama.cpp self-hosted provider context reporting use the loaded runtime `n_ctx` instead of the trained/native context window for session table percentages.

## Best Possible Solution

Review and land #74057, or an equivalent narrow provider-discovery fix that stores llama.cpp `/props.n_ctx` as `contextTokens` while preserving native `contextWindow` metadata.

## Reproduction Assessment

Yes, by source inspection: current self-hosted discovery never reads `/props.n_ctx`, while sessions use stored/configured `contextTokens` for the denominator. I did not run a live llama.cpp server in this read-only review.

## Solution Assessment

Yes. Capturing the runtime cap during self-hosted provider discovery is the narrow maintainable fix; putting llama.cpp HTTP probing in the sessions display path would spread provider-specific behavior into a command surface.
