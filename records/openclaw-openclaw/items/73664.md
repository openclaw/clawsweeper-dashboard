## Summary

Keep this issue open because current main still does not discover llama.cpp's loaded runtime `n_ctx`, and #74057 is an open, unmerged closing PR for the reported behavior.

## What This Changes

The issue asks the llama.cpp/OpenAI-compatible provider path to show the loaded runtime context window in session context percentages instead of the training/native context ceiling.

## Best Possible Solution

Review and land #74057, or an equivalent narrow provider-discovery fix that records llama.cpp `/props.n_ctx` as `contextTokens` while preserving native `contextWindow` metadata.

## Reproduction Assessment

Yes, by source inspection: self-hosted discovery does not read `/props.n_ctx`, and the sessions table uses stored/configured `contextTokens` for its denominator. I did not run a live llama.cpp server in this read-only review.

## Solution Assessment

Yes. Capturing the runtime cap during self-hosted provider discovery is the narrow maintainable fix; probing llama.cpp from the sessions display path would spread provider-specific behavior into a command surface.
