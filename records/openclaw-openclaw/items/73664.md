## Summary

Keep open. Current main still lacks llama.cpp runtime-context discovery in the self-hosted OpenAI-compatible provider path, and the linked #74057 is open and unmerged as the implementation candidate with closing syntax.

## What This Changes

The issue asks OpenClaw to show the actual llama.cpp runtime context window for session token percentages instead of the model training context metadata.

## Best Possible Solution

Land or revise #74057, or an equivalent narrow provider-discovery change, so self-hosted llama.cpp-compatible servers can report the loaded runtime window as `contextTokens` while preserving native `contextWindow` metadata and explicit user configuration.

Yes. The source-level reproduction is high confidence: current main has no `/props` or `n_ctx` lookup in the self-hosted provider discovery path, while session listing relies on stored/model `contextTokens` rather than probing llama.cpp at display time.
