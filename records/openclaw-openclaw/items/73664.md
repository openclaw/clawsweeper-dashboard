## Summary

Keep this issue open because current main still does not discover llama.cpp's loaded runtime `n_ctx`, and #74057 is an open, unmerged closing PR for the reported behavior.

## What This Changes

The report asks for llama.cpp-backed OpenAI-compatible sessions to display the loaded runtime `n_ctx` as the context denominator instead of the model's trained/native window.

## Best Possible Solution

Land #74057 or an equivalent narrow provider-discovery change that records llama.cpp `/props.n_ctx` as `contextTokens` while preserving native `contextWindow` metadata and configured overrides.

## Reproduction Assessment

Yes, by source inspection: current self-hosted discovery fetches only `/models` and never `/props.n_ctx`, while session display formats percentages from `contextTokens`. I did not run a live llama.cpp server in this read-only review.

## Solution Assessment

Yes. Capturing the runtime cap during self-hosted provider discovery is the narrow maintainable fix; probing llama.cpp from the sessions display path would spread provider-specific behavior into a command surface.
