## Summary

Keep this issue open: current main still discovers self-hosted OpenAI-compatible models from `/models` only, does not read llama.cpp `/props.n_ctx`, and the linked fix PR #74057 is still open rather than merged.

## What This Changes

Use the actual llama.cpp runtime context window for self-hosted OpenAI-compatible session context percentages instead of the model training-context metadata.

## Best Possible Solution

Land #74057 or an equivalent narrow discovery fix that records llama.cpp `/props.n_ctx` as effective `contextTokens` while keeping `contextWindow` as native metadata.

## Reproduction Assessment

Yes. A llama.cpp OpenAI-compatible server whose `/v1/models` reports a larger training window while `/props` reports a smaller loaded `n_ctx` should hit this path because current main never reads `/props` for self-hosted discovery.

## Solution Assessment

Yes. Capturing the runtime cap during provider discovery is the narrow maintainable fix; the sessions table should keep consuming stored session/model metadata rather than doing live provider-specific HTTP calls.
