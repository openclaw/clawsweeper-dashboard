## Summary

Keep this issue open. Current main has the config-backed context cap machinery, but it still does not implement llama.cpp `/props.n_ctx` runtime-context discovery, and #74057 is an open implementation candidate with closing syntax for this issue.

## What This Changes

Request that llamacpp/self-hosted OpenAI-compatible session context display use the loaded runtime context window (`n_ctx`) or configured cap instead of the model training context (`n_ctx_train`).

## Best Possible Solution

Land or revise #74057, or an equivalent narrow provider-discovery change, so explicit configured caps remain authoritative while llama.cpp-compatible servers can report their smaller loaded runtime context through `/props.n_ctx` as the effective `contextTokens` cap.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: current main has no `/props.n_ctx` lookup in the self-hosted discovery/session listing path, while the session denominator is derived from stored/configured context tokens rather than a live llama.cpp runtime-context probe.

## Solution Assessment

Yes. Keeping this issue open while maintainers review #74057 is the best path; a duplicate ClawSweeper repair branch would add churn before the existing candidate is resolved.
