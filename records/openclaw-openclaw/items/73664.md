## Summary

Keep this issue open because current main still has no llama.cpp `/props.n_ctx` runtime-context discovery in the self-hosted OpenAI-compatible provider path, and #74057 is an open implementation candidate with closing syntax for this report.

## What This Changes

The issue requests that llama.cpp-backed session context accounting use the server runtime `n_ctx` instead of the model training/native context window.

## Best Possible Solution

Land or revise #74057, or an equivalent narrow self-hosted provider discovery change, so llama.cpp-compatible servers store `/props.n_ctx` as effective `contextTokens` while retaining native model metadata separately.

## Reproduction Assessment

Yes. A llama.cpp OpenAI-compatible server whose `/v1/models` exposes only `meta.n_ctx_train` and whose `/props` exposes a smaller `n_ctx` should reproduce because current main has no `/props` or `n_ctx` handling.

## Solution Assessment

Yes. The narrowest maintainable fix belongs in self-hosted provider discovery so all session, status, and model-list consumers receive the same effective runtime cap.
