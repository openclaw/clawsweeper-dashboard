## Summary

Keep this issue open because current main still lacks llama.cpp runtime `n_ctx` discovery for the self-hosted OpenAI-compatible path, and #74057 is an open closing PR rather than merged code.

## What This Changes

The issue requests that llama.cpp/self-hosted provider context budgeting use the runtime context window (`n_ctx`) instead of the trained context ceiling (`n_ctx_train`) in session usage displays.

## Best Possible Solution

Review and land #74057, or an equivalent narrow discovery fix that stores llama.cpp runtime `n_ctx` as `contextTokens` while keeping the sessions table provider-agnostic.

## Reproduction Assessment

Yes, by source inspection: current main has the session display path that consumes `contextTokens`, and the self-hosted discovery path does not read `/props.n_ctx`. I did not run a live llama.cpp server in this read-only review.

## Solution Assessment

Yes. Capturing the runtime cap during provider discovery is the narrow maintainable solution; adding llama.cpp-specific HTTP probing to the sessions command would spread provider-specific behavior into a display surface.
