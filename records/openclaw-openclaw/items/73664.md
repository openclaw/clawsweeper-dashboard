## Summary

Keep this issue open. Current main still has no llama.cpp `/props.n_ctx` discovery in the self-hosted OpenAI-compatible provider path, and the provided GitHub context shows #74057 is an open, unmerged implementation candidate with closing syntax for this issue.

## What This Changes

The issue asks OpenClaw to show a llama.cpp server's runtime context window (`n_ctx`) in session context percentages instead of the model's trained context ceiling (`n_ctx_train`).

## Best Possible Solution

Land or revise #74057, or an equivalent narrow provider-discovery change, so explicit configured caps remain authoritative while llama.cpp-compatible servers can report their loaded runtime `n_ctx` as the effective session `contextTokens` cap.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: current main only discovers self-hosted OpenAI-compatible models through `/models`, has no `/props.n_ctx` lookup, and the sessions table consumes stored/configured context tokens rather than querying llama.cpp during display.

## Solution Assessment

Yes. A focused self-hosted provider discovery fix that reads llama.cpp `/props.n_ctx` while preserving explicit configured `contextWindow`/`contextTokens` is the narrowest maintainable solution; because #74057 is already open, this issue should wait for that PR or an equivalent fix.
