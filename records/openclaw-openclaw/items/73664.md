## Summary

Keep this issue open: current main still does not discover llama.cpp `/props.n_ctx`, and the linked closing PR is open rather than merged.

## What This Changes

The issue asks OpenClaw to show llama.cpp's loaded runtime context window for session token percentages instead of the model's training context ceiling.

## Best Possible Solution

Review and land #74057 or an equivalent narrow discovery fix that stores llama.cpp `/props.n_ctx` as effective `contextTokens` while keeping `contextWindow` as native metadata.

## Reproduction Assessment

Yes. A llama.cpp OpenAI-compatible server whose `/v1/models` reports a larger training context while `/props` reports a smaller loaded `n_ctx` should reproduce because current main only discovers `/models` and the sessions table uses stored/resolved context metadata.

## Solution Assessment

Yes. Fixing provider discovery is the narrowest maintainable direction; the sessions table should keep consuming stored model/session metadata rather than adding live llama.cpp-specific HTTP calls.
