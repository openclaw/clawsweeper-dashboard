Keep this issue open. Current main still lacks llama.cpp `/props.n_ctx` runtime-context discovery in the self-hosted OpenAI-compatible provider path, and #74057 is an open, unmerged implementation candidate with closing syntax for this issue.

## What This Changes

The issue asks OpenClaw to show llama.cpp's actual loaded runtime context window (`n_ctx`) in session context denominators instead of the model training context limit (`n_ctx_train`).

## Best Possible Solution
