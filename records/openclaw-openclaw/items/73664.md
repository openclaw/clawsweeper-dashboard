## Summary

Keep this issue open. Current main still lacks llama.cpp `/props` runtime-context discovery in the self-hosted OpenAI-compatible provider path, and the linked open PR is an implementation candidate with closing syntax rather than a merged fix.

## What This Changes

Request that llama.cpp/self-hosted OpenAI-compatible model discovery use the loaded runtime context `n_ctx`, not training `n_ctx_train`, for session-table context usage denominators.

## Best Possible Solution

Revise and land #74057, or an equivalent narrow provider-discovery fix, so explicit configured caps remain authoritative while llama.cpp-compatible servers can store the loaded runtime context as effective `contextTokens`.

Yes. A source-level reproduction is high confidence: current main has no `/props.n_ctx` lookup in self-hosted discovery, while session display uses stored or resolved context caps as the denominator.

## Solution Assessment

Yes. The maintainable direction is a narrow self-hosted provider discovery change that records runtime context separately from native `contextWindow`; the open PR should be corrected and reviewed rather than duplicating the issue lane.
