## Summary

Keep this issue open because current main still lacks llama.cpp runtime context discovery in the self-hosted OpenAI-compatible provider path, and #74057 is an open, unmerged implementation candidate with closing syntax for this report.

## What This Changes

The issue requests that llama.cpp self-hosted OpenAI-compatible models use the loaded runtime context window for session usage denominators instead of the model training context window.

## Best Possible Solution

Land or revise #74057, or an equivalent narrow self-hosted provider discovery change, so llama.cpp-compatible servers can store `/props.n_ctx` as effective `contextTokens` while preserving configured `contextWindow` as native model metadata.

## Reproduction Assessment

Yes. A llama.cpp OpenAI-compatible server whose `/v1/models` reports training metadata but whose `/props.n_ctx` differs should reproduce the wrong denominator because current discovery never reads `/props` and session listing consumes stored/configured/model context tokens only.

## Solution Assessment

Yes. The best fix belongs in the self-hosted provider discovery boundary, not in the sessions table, so all consumers receive the same effective runtime context cap.
