## Summary

Keep open: current main still does not read llama.cpp `/props.n_ctx` for self-hosted OpenAI-compatible discovery, and #74057 remains open and unmerged as the linked fix candidate.

## What This Changes

Request that llama.cpp OpenAI-compatible model discovery and session budgeting use the loaded runtime context (`n_ctx`) instead of the larger training context (`n_ctx_train`).

## Best Possible Solution

Land #74057 or an equivalent narrow discovery fix that records llama.cpp `/props.n_ctx` as effective `contextTokens` while preserving native `contextWindow` metadata.

## Reproduction Assessment

Yes. A llama.cpp OpenAI-compatible server whose `/v1/models` reports a larger training context and whose `/props` reports a smaller loaded `n_ctx` should reproduce the wrong denominator because current main never reads `/props` on this path.

## Solution Assessment

Yes. Capturing the runtime cap during provider discovery is the narrowest maintainable direction; the sessions table should keep consuming stored session/model metadata instead of doing live provider-specific HTTP calls.
