## Summary

Keep this issue open. Current main still lacks llama.cpp runtime-context discovery in the self-hosted OpenAI-compatible provider path, and the linked #74057 is open, unmerged, and still under review as the implementation candidate for this report.

## What This Changes

The issue asks the llama.cpp-compatible self-hosted provider path to use the loaded runtime context window for session usage denominators instead of the model training context window.

## Best Possible Solution

Land or revise #74057, or an equivalent narrow provider-discovery change, so explicit configured caps remain authoritative while llama.cpp-compatible servers can report the loaded runtime window as `contextTokens`; include focused tests and a changelog entry before closing this issue.
