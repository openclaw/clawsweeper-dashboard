## Summary

Keep this issue open because current main still lacks llama.cpp `/props.n_ctx` runtime-context discovery in the self-hosted OpenAI-compatible provider path, and #74057 is an open implementation candidate with closing syntax for this report.

## What This Changes

The issue requests that llama.cpp-backed self-hosted provider context accounting use the runtime context window instead of the model training context window.

## Best Possible Solution

Land or revise #74057, or an equivalent narrow self-hosted provider discovery change, so llama.cpp-compatible servers store `/props.n_ctx` as effective `contextTokens` while retaining native model metadata separately.

## Reproduction Assessment

Yes. A llama.cpp OpenAI-compatible server whose `/v1/models` reports `meta.n_ctx_train` and whose `/props` reports a smaller `n_ctx` should reproduce the wrong denominator because current main never reads `/props`.

## Solution Assessment

Yes. The best fix belongs in self-hosted provider discovery, so the runtime cap is captured once and all session/model consumers receive consistent `contextTokens`.
