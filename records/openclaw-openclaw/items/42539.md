## Summary

Keep open. Current main still lacks a configurable Telegram auto-TTS split-delivery mode: speech-core attaches generated audio to the existing visible reply payload, and Telegram sends that text as a voice/audio caption except for the existing overflow fallback.

## What This Changes

The request is to add a Telegram TTS delivery option that sends the text reply and generated voice note as separate messages instead of one voice message with a caption.

## Best Possible Solution

Define a backward-compatible TTS delivery policy that preserves bundled caption delivery as the default, adds a separate text-plus-voice mode for Telegram or shared voice-note channels, and documents ordering, failure fallback, and overflow behavior.

## Reproduction Assessment

Not applicable as a strict bug reproduction. Source inspection gives high confidence that current main still sends Telegram auto-TTS text through the caption path unless the caption exceeds Telegram's 1024-character caption limit.

## Solution Assessment

Unclear for the exact proposed JSON shape. The direction is sound, but maintainers should choose whether this belongs as a shared auto-TTS text-delivery policy or a Telegram-specific delivery mode before implementation.
