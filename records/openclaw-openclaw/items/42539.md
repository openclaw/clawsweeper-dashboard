## Summary

Keep open. Current main still lacks a configurable Telegram TTS mode that always sends readable text separately from the voice note, and related open work points in the opposite bundled-caption direction, so this needs a maintainer product/API decision rather than cleanup closure.

## What This Changes

Request a Telegram TTS delivery option that sends normal readable text and a voice note as separate messages instead of using the text as a voice caption.

## Best Possible Solution

Define a backward-compatible TTS text-delivery policy with the current bundled behavior as default and an explicit Telegram-supported separate mode that composes with streaming previews, caption overflow, and TTS failure fallback.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature request. Source inspection gives high confidence that current main lacks a delivery/textDelivery policy and still uses caption-or-overflow-only Telegram behavior.

## Solution Assessment

Unclear for the exact proposed JSON shape. The user need is valid, but maintainers should decide whether this belongs in shared TTS config, Telegram plugin config, or a broader text-delivery policy covering separate, suppress, and caption modes.
