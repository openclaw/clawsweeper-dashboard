## Summary

Keep open. Current main partially avoids caption duplication when Telegram preview streaming has already shown the final answer, but there is still no public TTS delivery-mode config and the normal/direct Telegram media path still uses captions except for overflow or video-note cases.

## What This Changes

Request a configurable Telegram Auto-TTS mode that sends readable text and the voice note as separate messages instead of one captioned voice message.

## Best Possible Solution

Define a backward-compatible TTS text-delivery policy that composes with Telegram preview streaming, non-streaming delivery, caption overflow, and voice-upload failure fallback.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature request. Source inspection gives high confidence that the requested config mode is absent and that only some Telegram streaming cases already split visible text from late voice media.

## Solution Assessment

Unclear for the exact proposed JSON shape. The maintainable direction is valid, but maintainers should choose whether this belongs in shared `messages.tts` policy, Telegram config, or a broader text-delivery policy that also covers suppress and caption-only requests.
