Keep open. Current main still lacks a configurable Telegram TTS mode that always sends readable text separately from the voice note, and related TTS delivery threads show this needs a maintainer product/API decision rather than cleanup closure.

## What This Changes

Request a configurable Telegram auto-TTS delivery mode that sends readable text separately from the voice message instead of using a caption.

## Best Possible Solution

Define a backward-compatible TTS text-delivery policy with the current bundled behavior as default and an explicit Telegram-supported separate mode that composes with streaming, caption overflow, and voice-send fallback.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature request. Source inspection gives high confidence that current main lacks a delivery/textDelivery policy and still uses caption-or-overflow-only Telegram behavior.

## Solution Assessment

Unclear for the exact proposed JSON shape. The user need is valid, but maintainers should decide whether this belongs in shared TTS config, Telegram plugin config, or a broader text-delivery policy covering separate, suppress, and caption modes.
