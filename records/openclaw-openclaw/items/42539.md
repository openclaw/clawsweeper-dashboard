## Summary

This should stay open: current main and the latest release still have no `messages.tts.delivery` or equivalent text/voice delivery mode, and the remaining choice is a real Telegram/TTS product and config-contract decision rather than stale cleanup.

## What This Changes

Request adds a Telegram TTS delivery mode so text can be sent as a normal message and the voice note follows separately instead of using a voice caption by default.

## Best Possible Solution

Choose one canonical auto-TTS text/voice delivery policy across Telegram and other voice-note channels, then implement the accepted mode through TTS config/schema/docs and focused Telegram plus speech-core tests.

## Reproduction Assessment

Yes, from source inspection: current main has no delivery-mode config key, Auto-TTS attaches audio to the reply, and Telegram uses reply text as a voice/media caption except for overflow or error fallback. I did not run a live Telegram client scenario in this read-only review.

## Solution Assessment

Unclear until maintainers decide the product policy. A delivery-mode option is plausible, but the best shape needs to account for existing streamed-text behavior, voice-only requests, and open captioned-voice fixes before adding another public config surface.
