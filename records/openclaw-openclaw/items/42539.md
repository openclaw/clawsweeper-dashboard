## Summary

Keep open. Current main and v2026.5.22 still only support bundled/captioned Telegram TTS with overflow fallback, and the related open TTS text-delivery requests are adjacent rather than a replacement for this separate text-plus-voice mode.

## What This Changes

Request a configurable Telegram TTS delivery mode that sends readable text as a normal message and voice as a separate voice message instead of using a voice caption.

## Best Possible Solution

Define a backward-compatible TTS text-delivery policy with current bundled behavior as default and an explicit Telegram-supported separate mode that composes with overflow, final-mode TTS, and voice-send fallback.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature request. Source inspection gives high confidence that current main lacks the requested delivery policy and still uses caption-or-overflow Telegram behavior.

## Solution Assessment

Unclear for the exact messages.tts.delivery spelling. The user need is valid, but maintainers should decide whether this belongs in shared TTS config, Telegram plugin config, or a broader text-delivery policy covering separate, suppress, preserve, and caption modes.
