## Summary

Keep open. Current main still lacks a configurable Telegram split-delivery mode for auto-TTS: speech-core attaches audio to the visible reply payload, and Telegram sends that text as a media caption unless the existing overflow fallback is triggered.

## What This Changes

Request a TTS delivery option so Telegram can send the text reply and generated voice note as separate messages instead of one captioned voice message.

## Best Possible Solution

Define a backward-compatible auto-TTS delivery policy that keeps bundled caption delivery as the default, adds a separate text-plus-voice mode for Telegram or voice-note channels, and documents ordering, failure fallback, and overflow behavior.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature request. Source inspection gives high confidence that current main still sends Telegram auto-TTS text through the caption path unless the caption exceeds the overflow fallback limit.

## Solution Assessment

Unclear for the exact proposed JSON shape. The direction is sound, but maintainers should choose whether this belongs as a shared auto-TTS delivery policy or a Telegram-specific mode before implementation.
