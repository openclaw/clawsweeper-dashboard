## Summary

Keep open. Current main and v2026.5.12 still lack a configurable Telegram TTS delivery mode for always sending readable full-width text plus a separate voice note; existing behavior only splits captions on overflow and partially separates late TTS media in Telegram preview-streaming paths.

## What This Changes

Request a TTS delivery option that makes Telegram send text and voice as separate messages instead of a voice caption.

## Best Possible Solution

Define a backward-compatible TTS text-delivery policy that composes with Telegram streaming previews, non-streaming media captions, overflow fallback, and TTS failure fallback.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature request. Source inspection gives high confidence that the requested config mode is absent and that current main only separates text from voice in overflow or specific Telegram preview-streaming paths.

## Solution Assessment

Unclear for the exact proposed JSON shape. The maintainable direction is valid, but maintainers should choose whether this belongs in shared TTS config, Telegram plugin config, or a broader text-delivery policy.
