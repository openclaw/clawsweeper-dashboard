## Summary

Keep open: current main still lacks an always-separate Telegram TTS delivery mode; source and tests show normal-length Telegram voice replies still carry visible text as a media caption, with split text only on overflow or other channel-specific paths. This is a feature/config product decision rather than stale cleanup.

## What This Changes

Request a configurable Telegram TTS delivery mode that sends readable text and a separate voice message instead of bundling text as a voice caption.

## Best Possible Solution

Define a backward-compatible TTS delivery policy that keeps the current bundled default while allowing Telegram to send full-width text and a separate voice note, with schema, docs, and channel tests covering the selected behavior.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/config request. Source and tests give high confidence for current behavior: normal Telegram TTS media uses captions and separate follow-up text only appears for overflow or specific media/channel cases.

## Solution Assessment

Unclear until maintainers choose the product contract. A backward-compatible messages.tts delivery policy is plausible, but it should be aligned with async/text-first TTS work in https://github.com/openclaw/openclaw/issues/69315 and Telegram-specific ordering semantics.
