## Summary

Keep open. Current main and v2026.5.20 still lack a Telegram TTS delivery policy that always sends readable text separately from the voice note, and the related threads leave this as the canonical product/API decision rather than a cleanup close.

## What This Changes

The issue requests a configurable Telegram TTS mode that sends normal text and a voice note as separate messages instead of using the voice caption by default.

## Best Possible Solution

Define a backward-compatible TTS text-delivery policy with bundled as the default and an explicit Telegram-supported separate mode that composes with overflow, streaming/final-mode behavior, and voice-send fallback.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature request. Source inspection gives high confidence that current main still lacks a delivery/textDelivery policy and still uses caption-or-overflow-only Telegram behavior.

## Solution Assessment

Unclear for the exact proposed JSON shape. The user need is valid, but maintainers should decide whether this belongs in shared TTS config, Telegram plugin config, or a broader text-delivery policy covering separate, suppress, preserve, and caption modes.
