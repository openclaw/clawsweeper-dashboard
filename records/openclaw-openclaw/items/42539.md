## Summary

Keep open. Current main and the latest release still lack a TTS delivery/text-delivery option, Telegram still uses captions except for overflow, and the related open issues are adjacent rather than a complete replacement for this Telegram split-delivery request.

## What This Changes

The issue requests a Telegram TTS delivery option that sends readable text separately from the generated voice note instead of using a voice caption.

## Best Possible Solution

Define a backward-compatible TTS text-delivery policy with current bundled behavior as default and an explicit Telegram-supported separate mode that composes with caption overflow, final-mode TTS, and voice-send fallback.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature request. Source and release inspection give high confidence that current main lacks the requested delivery policy and still uses caption-or-overflow-only Telegram behavior.

## Solution Assessment

Unclear for the exact messages.tts.delivery spelling. The user need is valid, but maintainers should decide whether this belongs in shared TTS config, Telegram plugin config, or a broader text-delivery policy covering separate, suppress, preserve, and caption modes.
