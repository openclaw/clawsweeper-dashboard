## Summary

Keep open: current main still only supports bundled Telegram TTS delivery plus overflow/error fallback splitting, and the requested always-separate mode would add a new config/product surface rather than fix a broken existing contract.

## What This Changes

Add a configurable Telegram TTS delivery mode so text can be sent as a normal Telegram message and the voice note separately instead of using a voice caption.

## Best Possible Solution

Define an explicit TTS delivery policy, likely in the effective `messages.tts`/channel override shape, that preserves the current default while letting Telegram send readable text and voice as separate messages in coordination with the async TTS design.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/config request. Source inspection does confirm the current limitation with high confidence: Telegram receives one text-plus-audio payload and renders the text as a media caption unless the overflow/error fallback fires.

## Solution Assessment

Unclear until maintainers choose the product contract. The requested direction is plausible, but the maintainable solution should decide whether this is a generic TTS delivery mode, a Telegram channel override, or part of #69315's async text-first delivery work.
