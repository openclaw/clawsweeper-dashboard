## Summary

Keep open. Current main and the latest shipped tag still lack a Telegram TTS delivery policy that always sends readable text separately from voice, and related TTS delivery threads show this needs maintainer product/API direction rather than cleanup closure.

## What This Changes

Add a configurable Telegram TTS delivery mode that keeps the current bundled caption behavior by default but can send text and voice as separate messages.

## Best Possible Solution

Define a backward-compatible TTS text-delivery policy with the current bundled behavior as default and an explicit Telegram-supported separate mode that composes with streaming, caption overflow, and voice-send fallback.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature request. Source inspection gives high confidence that current main lacks a delivery/textDelivery policy and still uses caption-or-overflow-only Telegram behavior.

## Solution Assessment

Unclear for the exact proposed JSON shape. The user need is valid, but maintainers should decide whether this belongs in shared TTS config, Telegram plugin config, or a broader text-delivery policy covering separate, suppress, preserve, and caption modes.
