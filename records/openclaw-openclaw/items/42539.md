## Summary

Keep open. Current main and v2026.5.20 still lack a TTS delivery/text-delivery policy, Telegram still captions voice media except for caption overflow, and the related issues are adjacent rather than a complete canonical replacement.

## What This Changes

Request adding a configurable Telegram TTS mode that sends readable text and the generated voice note as separate messages instead of one captioned voice message.

## Best Possible Solution

Define a backward-compatible TTS text-delivery policy with the current bundled behavior as default and an explicit Telegram-supported separate mode that composes with caption overflow, streaming/final mode, and voice-send fallback.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature request. Source and test inspection give high confidence that current main lacks a delivery/textDelivery policy and still uses caption-or-overflow-only Telegram behavior.

## Solution Assessment

Unclear for the exact proposed messages.tts.delivery shape. The user need is valid, but maintainers should decide whether this belongs in shared TTS config, Telegram plugin config, or a broader text-delivery policy covering separate, suppress, preserve, and caption modes.
