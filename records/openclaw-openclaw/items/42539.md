## Summary

Keep open. Current main still bundles Telegram TTS text as a voice caption by default, only splits text on caption overflow or voice-send fallback, and the TTS config/schema has no delivery-mode option.

## What This Changes

The issue requests a Telegram TTS delivery mode that sends readable text as a normal message and the generated voice note as a separate follow-up message instead of using the text as a voice caption.

## Best Possible Solution

Design a shared, backwards-compatible TTS delivery policy that keeps bundled caption behavior as the default while allowing Telegram to send text-first plus voice-follow-up with schema, docs, and regression tests.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high confidence that current main still follows the reported bundled-caption behavior and only separates text on overflow/fallback paths.

## Solution Assessment

Unclear. The requested direction is reasonable, but the maintainable solution should be designed alongside the related async and voice-only TTS delivery requests rather than adding an isolated Telegram-only switch without policy review.
