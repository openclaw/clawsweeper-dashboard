## Summary

Keep open. Current `main` still keeps successful auto-TTS as a media-bearing reply with text on the caption path unless Telegram caption overflow forces a split, and the requested always-separate mode needs maintainer product/API direction.

## What This Changes

Request a Telegram TTS delivery option that sends readable text and a voice note as separate messages instead of bundling text as the voice caption.

## Best Possible Solution

Define a backward-compatible auto-TTS text-delivery policy that keeps bundled caption delivery as the default, adds an explicit separate text-plus-voice mode, and documents Telegram overflow and failure ordering.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature request. Source inspection gives high confidence that current `main` still bundles successful auto-TTS text with voice media unless Telegram caption overflow forces a split.

## Solution Assessment

Unclear for the exact proposed JSON shape. The direction is valid, but maintainers should choose whether this is shared `messages.tts` delivery policy or a Telegram-specific option before implementation.
