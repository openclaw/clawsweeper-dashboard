## Summary

Keep open. Current main still builds auto-TTS as one media-bearing reply and Telegram renders the visible text as a voice/media caption except for overflow, while the requested delivery-mode config is not implemented or fully superseded by related broader TTS policy issues.

## What This Changes

Request to add a `messages.tts` delivery mode so Telegram auto-TTS can send readable text and a voice note as separate messages.

## Best Possible Solution

Add a backward-compatible, documented TTS delivery/text policy that defaults to current behavior and lets Telegram send full-width text plus a follow-up voice note with focused speech-core, config, Telegram, and docs coverage.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives high confidence that current main still captions Telegram voice/media sends by default and only splits text on caption overflow.

## Solution Assessment

Unclear. The requested separate mode is plausible, but the maintainable fix should likely be a shared TTS delivery/text policy that Telegram honors rather than a Telegram-only special case.
