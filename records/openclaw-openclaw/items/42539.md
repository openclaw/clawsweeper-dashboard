## Summary

Keep open. Current main and v2026.5.19 still lack a configurable Telegram TTS delivery mode for always sending readable text plus a separate voice note; the remaining work is a product/API decision, especially because related open work currently argues for bundled caption behavior in some streaming paths.

## What This Changes

Request adding a configurable Telegram TTS delivery mode that sends full-width text and a separate voice note instead of using a voice caption.

## Best Possible Solution

Define a backward-compatible TTS text-delivery policy with the current bundled behavior as default and an explicit Telegram-supported separate mode that composes with streaming previews, caption overflow, and TTS failure fallback.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature request. Source and docs inspection give high confidence that current main and v2026.5.19 lack a delivery/textDelivery policy and still use caption or overflow-only split behavior.

## Solution Assessment

Unclear for the exact proposed JSON shape. The user need is valid, but maintainers should decide whether this belongs in shared TTS config, Telegram plugin config, or a broader text-delivery policy covering separate, suppress, and caption modes.
