## Summary

Keep open. Current main and the latest shipped tag still lack a public TTS text/voice delivery policy, and Telegram still uses caption-or-overflow/fallback behavior rather than an opt-in separate text plus voice mode.

## What This Changes

The issue requests a Telegram TTS delivery option that sends text as a normal message and voice as a separate voice note instead of using the text as the voice-note caption.

## Best Possible Solution

Define a backward-compatible TTS text-delivery policy with the current bundled/caption behavior as default and an explicit Telegram-supported separate mode that composes with streaming, caption overflow, and voice-send fallback.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature request. Source inspection gives high confidence that current main lacks a delivery/textDelivery policy and still uses caption-or-overflow behavior for Telegram voice payloads.

## Solution Assessment

Unclear for the exact proposed JSON shape. The user need is valid, but maintainers should decide whether this belongs in shared TTS config, Telegram plugin config, or a broader text-delivery policy covering separate, suppress, preserve, and caption modes.
