## Summary

Keep open. Current main and the latest tagged release still route successful auto-TTS as one media-bearing reply, so Telegram keeps the visible text on the voice message caption path unless the existing overflow fallback splits it.

## What This Changes

Request adding a Telegram TTS delivery mode/config that sends the normal text message separately from the generated voice note instead of using a caption.

## Best Possible Solution

Define one backward-compatible auto-TTS text-delivery policy that keeps bundled caption delivery as the default, adds an explicit separate text-plus-voice mode, and documents Telegram overflow/failure ordering.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature request. Source inspection gives high confidence that current main still routes Telegram auto-TTS text into the caption path unless the overflow fallback splits it.

## Solution Assessment

Unclear for the exact proposed JSON shape. The direction is valid, but maintainers should choose whether this is a shared auto-TTS text-delivery policy or a Telegram-specific option before implementation.
