## Summary

Keep open. Current main and v2026.5.22 still lack a TTS delivery/text-delivery policy, auto-TTS still attaches generated audio to the existing visible reply payload, and Telegram still uses caption-or-overflow behavior rather than a configurable separate text plus voice mode.

## What This Changes

Request a Telegram TTS delivery option that sends readable text and voice as separate messages instead of using the voice-message caption by default.

## Best Possible Solution

Define a backward-compatible TTS text-delivery policy with the current bundled behavior as default and an explicit Telegram-supported separate mode that composes with caption overflow, streaming/final mode, and voice-send fallback.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature request. Source and release inspection give high confidence that current main lacks a delivery/textDelivery policy and still uses caption-or-overflow-only Telegram behavior.

## Solution Assessment

Unclear for the exact proposed messages.tts.delivery shape. The user need is valid, but maintainers should decide whether this belongs in shared TTS config, Telegram plugin config, or a broader text-delivery policy covering separate, suppress, preserve, and caption modes.
