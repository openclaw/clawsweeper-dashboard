## Summary

Keep open. Current main and the latest release still attach automatic TTS audio to one reply payload, Telegram still sends short media text as the voice caption, and the requested split delivery requires a new config/product contract.

## What This Changes

Request adding a `messages.tts` delivery mode that makes Telegram send readable text and the generated voice note as separate messages instead of one captioned voice message.

## Best Possible Solution

Add a backward-compatible, schema-documented TTS delivery policy that keeps bundled caption delivery as the default and lets Telegram opt into separate text plus follow-up voice delivery with focused Telegram proof.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source inspection gives high confidence that automatic Telegram TTS still sends one media payload whose short text becomes the voice caption except when the caption exceeds 1024 characters.

## Solution Assessment

Unclear. The requested behavior is reasonable, but the maintainable fix needs a shared TTS delivery policy and Telegram ordering/fallback decisions rather than a hidden Telegram-only workaround.
