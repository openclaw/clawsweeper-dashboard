## Summary

Keep open: current main and v2026.6.5 still expose caption/overflow behavior for Telegram TTS, and always-separate text plus voice delivery is a new config/product decision rather than a strict bug fix.

## What This Changes

The issue requests a Telegram TTS delivery option that sends readable full-width text and the voice note as separate messages instead of one captioned voice bubble.

## Best Possible Solution

Define one typed TTS text/audio delivery policy, keep the current caption behavior as the default, and add Telegram-focused source tests, docs, and config schema coverage if maintainers approve the new mode.

## Reproduction Assessment

Yes, at source level. Current main attaches TTS audio to the existing reply payload and the Telegram send path uses that text as the media caption, with separate text only for caption overflow.

## Solution Assessment

Unclear as proposed. A separate mode is plausible, but adding `messages.tts.delivery` needs a maintainer decision on the permanent config name, default behavior, and how it relates to broader auto-TTS text-delivery options.
