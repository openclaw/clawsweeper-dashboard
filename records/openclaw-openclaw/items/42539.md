## Summary

Keep open. Current main still bundles automatic TTS audio back onto the visible reply payload, and Telegram sends that text as the voice-message caption except for the existing caption-overflow fallback; the requested default split delivery is a new feature/config policy, not already implemented.

## What This Changes

The issue requests a Telegram TTS delivery option that sends readable text and the voice note as separate messages instead of a voice-note caption.

## Best Possible Solution

Add a backward-compatible, config-backed TTS delivery policy that keeps bundled caption delivery as the default and lets Telegram send normal text plus a follow-up voice note when explicitly selected.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that enabling automatic TTS on Telegram still produces one media-bearing reply where short visible text is used as the voice-message caption.

## Solution Assessment

Unclear. The proposed user-facing behavior is reasonable, but the maintainable fix needs a product/API choice for shared TTS delivery policy, ordering, and interaction with async/background TTS rather than a hidden Telegram-only workaround.
