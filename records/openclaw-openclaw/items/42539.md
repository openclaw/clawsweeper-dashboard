## Summary

Keep open: this is a clear Telegram/TTS feature request rather than stale cleanup. Current main still bundles normal auto-TTS text and voice into one Telegram payload except for overflow and streaming-specific paths, and adding a configurable split mode needs maintainer product judgment.

## What This Changes

Add a TTS delivery mode option so Telegram can send readable text and the generated voice note as separate messages by default.

## Best Possible Solution

Define a typed TTS delivery policy with the current bundled behavior as default, allow a Telegram-appropriate separate text/voice mode at the right scope, and add Telegram delivery tests plus docs for ordering and fallbacks.

## Reproduction Assessment

Yes for current behavior from source: normal final replies pass text plus TTS media as one payload, and Telegram uses reply text as the media/voice caption. No live Telegram run was performed, and this is a feature request rather than a broken existing contract.

## Solution Assessment

Unclear. The requested split mode is plausible, but the best implementation needs maintainer choice between a global TTS delivery policy, a Telegram-specific override, or integration with the async TTS work.
