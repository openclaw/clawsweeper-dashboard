## Summary

Keep open. Current main still forwards the built-in compaction reserve token budget into Pi summarization without clamping it to the selected model output cap, and the paired same-author implementation PR remains open and unmerged.

## What This Changes

The issue requests that built-in LLM compaction clamp Anthropic 1M-context summarization output budgets to the model output limit instead of requesting 240000 tokens against a 128000-token cap.

## Best Possible Solution

Land PR #54392 or an equivalent maintainer-approved patch that clamps the built-in LLM compaction reserve/output budget against the resolved summarization model's output cap before Pi can request `max_tokens`, with regression coverage for a 300000 reserve and 128000 output cap.
