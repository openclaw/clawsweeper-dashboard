## Summary

Keep open: this PR is the canonical fix path for stale managed external CLI OAuth profiles and has sufficient terminal proof, but the latest head still lets no-external-discovery callers run persisted CLI sync and mutate `auth-profiles.json`. The protected `maintainer` and `clawsweeper:human-review` labels also require maintainer handling before merge.

## What This Changes

The PR persists fresher managed external CLI OAuth credentials into `auth-profiles.json`, keeps bootstrap-only Codex CLI credentials runtime-only, adds auth-store coverage, and clears related test-helper lint.

## Best Possible Solution

Gate persisted external CLI sync behind external-discovery paths, or pass `syncExternalCli: false` from no-external loaders, while preserving the lock/cache guards and Codex bootstrap-only runtime behavior.

## Reproduction Assessment

Yes. Source inspection and the PR's terminal proof show a stale `anthropic:claude-cli` profile plus fresher Claude CLI credentials exercises the intended runtime path; I did not run tests in this read-only sweep.

## Solution Assessment

No. Persisting managed non-bootstrap CLI refreshes is the right narrow direction, but the current implementation must first preserve callers that explicitly requested no external CLI discovery.
