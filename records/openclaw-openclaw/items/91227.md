## Summary

Keep this PR open for normal review/landing: the shipped and current memory-core helper can leave the target SQLite base path absent during reindex publication, and this patch addresses that root window with focused coverage and no blocking review findings.

## What This Changes

The PR changes memory-core atomic reindex publication to rename the temp SQLite base directly over the existing base while moving WAL/SHM sidecars separately, and adds a regression test for target-base reachability.

## Best Possible Solution

Land a narrow memory-core atomic publication fix that keeps the base index reachable throughout reindex publication, preserves missing-metadata stale-row protection, and avoids config, schema, migration, or public API changes.

## Reproduction Assessment

Yes. Source inspection shows the shipped/current helper removes `main.sqlite` before publishing the temp base, and the linked issue plus Node SQLite contract explain how an opener during that window can create a metadata-missing database.

## Solution Assessment

Yes. Fixing the owner publication helper is the best layer because it prevents the absent-path race at the source, while gateway-side recovery would only mitigate a bad handle after SQLite has already created it.
