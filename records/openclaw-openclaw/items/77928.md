The PR adds Telegram `/handoff` and `/resume latest`, inflight reliability tracking, failure/interrupted notices, long-input protection, stale diagnostic-lane recovery, docs, tests, and changelog entries.

## Best Possible Solution

Fix the topic/thread regressions, then run trusted Telegram QA or equivalent redacted live proof for the command and notice behavior before a maintainer merge decision.

## Reproduction Assessment

Yes for the review findings: source inspection at PR head shows interrupted notices bypass the central Telegram thread helper and handoff scope ignores `MessageThreadId`. The broader feature behavior is not fully reproduced because the proof does not exercise a live Telegram chat path.

## Solution Assessment

No, not yet. The maintainable path is to reuse the central Telegram thread helper for interrupted notices, include `MessageThreadId` in handoff scope, and get trusted Telegram proof before merge.
