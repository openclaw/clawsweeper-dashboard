## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Store handoff packets in SQLite-backed state:** `src/auto-reply/reply/commands-handoff.ts:306-318`
  - body: This writes the handoff packet and metadata as Markdown/JSON files under the OpenClaw state directory, then reads those files as steady-state runtime state. Current policy and Telegram precedent put runtime/plugin state in SQLite-backed stores with doctor-owned migrations, so this sidecar path creates upgrade, retention, privacy, and concurrency risk before the new command can ship.
  - confidence: 0.91
- **[P1] Store Telegram inflight records in plugin state:** `extensions/telegram/src/telegram-reliability-store.ts:199-235`
  - body: The new inflight tracker persists request lifecycle rows through `loadJsonFile`/`saveJsonFile` at `state/telegram/reliability-inflight-*.json`. Telegram's current caches and bindings already use SQLite plugin state, so adding a new JSON sidecar for runtime message/session state reopens the migration and corruption surface this codebase has been removing.
  - confidence: 0.9
- **[P1] Preserve current main's stuck-lane recovery guard:** `src/logging/diagnostic-stuck-session-recovery.runtime.ts:166-180`
  - body: This branch's `recoverableStaleLane` path is based on stale queue-depth/lane snapshots and can report an effective release even when `resetCommandLane` released nothing. Current main already has the later merged recovery fix with active task-id protection; resolving the dirty PR by keeping this older path would risk resetting or misreporting live session work.
  - confidence: 0.86
- **[P2] Do not terminalize unnotified interrupted Telegram turns:** `extensions/telegram/src/bot-core.ts:204`
  - body: `markStaleTelegramInflightInterrupted` marks all matching records interrupted, but startup only sends notices for the first five. Any later records are terminalized and will not be picked up on the next startup, so those users can still see a silently lost Telegram turn despite the PR's interruption-notice guarantee.
  - confidence: 0.88
