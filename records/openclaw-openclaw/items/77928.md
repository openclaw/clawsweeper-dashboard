Full review comments:

- **[P2] Notify every interrupted request or keep it pending:** `extensions/telegram/src/bot-core.ts:203`
  - body: `markStaleTelegramInflightInterrupted` has already moved every stale dispatched record to `interrupted`, but this loop sends notices only for `interrupted.slice(0, 5)`. Any sixth and later record is terminal and will not be picked up on the next startup, so those chats never receive the promised user-facing interruption notice. Cap the marking batch too, or leave unsent records eligible for a later pass.
  - confidence: 0.88

  - body: This adds a new runtime state file under `state/telegram/reliability-inflight-*.json`. Current OpenClaw policy and the existing plugin-state facade keep plugin-owned runtime state in the shared SQLite/plugin-state path; adding a fresh JSON sidecar creates a second persistence model with no doctor or migration owner. Move these inflight records to plugin state or shared SQLite before merge.
  - confidence: 0.8
