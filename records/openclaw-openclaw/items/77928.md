## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Use the Telegram thread helper for interrupted notices:** `extensions/telegram/src/bot-core.ts:205-208`
  - body: The startup recovery notice builds its own send options, so it only preserves numeric forum topics and will also pass forum General id 1. Current delivery code preserves DM topics and skips General; this path can send interrupted notices to the wrong thread or fail for General-topic records.
  - confidence: 0.91
- **[P2] Include MessageThreadId in the handoff scope:** `src/auto-reply/reply/commands-handoff.ts:143-147`
  - body: `/handoff` is documented as scoped to the current sender/thread, but this scope ignores `params.ctx.MessageThreadId` and falls back to the group chat id when there is no reply/root id. In Telegram forum topics, a packet saved in one topic can become the `/resume latest` packet for another topic in the same group.
  - confidence: 0.88
