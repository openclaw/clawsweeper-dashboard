## Summary

Current main still appears to have the reported Feishu named-account routing bug: dynamic agent creation writes account-less DM bindings, and the generic routing contract treats account-less bindings as default-account-only. The linked PR #42886 described the right narrow fix shape but is closed and unmerged, so there is no implemented or canonical merged fix to close against.

## What This Changes

The issue reports that Feishu dynamicAgentCreation creates agents and bindings, but DMs on named Feishu accounts still route to agent:main instead of per-user agents.

## Best Possible Solution

Make Feishu dynamic agent binding creation account-aware while preserving default-account compatibility for legacy account-less bindings.

## Reproduction Assessment

Yes from source inspection. A named-account Feishu DM with dynamic creation enabled creates an account-less direct binding, and the generic routing contract then excludes that binding from the named-account route path.

## Solution Assessment

Yes. The narrow maintainable fix is to pass the Feishu account id into dynamic binding creation, include it in new named-account bindings, and keep legacy account-less bindings valid only for default-account traffic.

## Repair Work Prompt

Fix Feishu dynamicAgentCreation routing for named accounts. Current main passes `account.accountId` into `resolveAgentRoute`, but `maybeCreateDynamicAgent` creates and detects direct bindings without `match.accountId`, so named-account DMs keep falling back to `agent:main` because account-less bindings match only the default account. Update `extensions/feishu/src/dynamic-agent.ts` and the call site in `extensions/feishu/src/bot.ts` to pass the Feishu account id, persist account-scoped bindings for named accounts, and preserve legacy account-less binding behavior for default account traffic. Add focused regression coverage for named non-default accounts, default accounts, and existing-agent/missing-binding cases. Do not change group routing, broadcast behavior, or generic routing semantics. Add a user-facing changelog fix entry crediting the human reporter or contributor when appropriate.
