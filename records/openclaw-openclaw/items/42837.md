## What This Changes

The issue asks to fix Feishu dynamicAgentCreation so named-account DM users route to independent per-user dynamic agents instead of agent:main.

## Best Possible Solution

Land a focused Feishu plugin runtime patch that passes normalized accountId into dynamic-agent binding detection and creation for DM and comment flows, writes account-scoped bindings for named accounts, and keeps legacy default-account behavior intact.

## Reproduction Assessment

Yes, at source level. A named Feishu account resolves routes with account.accountId, but dynamicAgentCreation writes an account-less binding that current routing tests define as default-account-only, so re-resolution can still fall back to agent:main.

## Solution Assessment

Yes. The maintainable fix belongs in the Feishu plugin by making maybeCreateDynamicAgent account-aware through the existing account-id normalization contract and updating both DM and comment callers without changing core routing semantics.
