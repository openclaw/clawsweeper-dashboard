## Summary

Keep open: current main and v2026.5.20 still create Feishu dynamic-agent bindings without account scope while named-account DM/comment routing re-resolves with the active account id, so traffic can still fall back to agent:main. ## What This Changes

The issue reports that Feishu dynamicAgentCreation creates agents and bindings but named-account DM users still route to agent:main instead of per-user agents.

## Best Possible Solution

Land a focused Feishu plugin runtime patch that passes normalized accountId into dynamic-agent binding detection and creation for DM and comment flows, writes account-scoped bindings for named accounts, and keeps default-account legacy behavior intact.

## Reproduction Assessment

Yes, at source level. A named Feishu account resolves routes with account.accountId, but dynamicAgentCreation writes an account-less binding that current routing tests define as default-account-only, so re-resolution can fall back to agent:main.

## Solution Assessment

Yes. The maintainable fix belongs in the Feishu plugin by making maybeCreateDynamicAgent account-aware through the existing account-id normalization contract and updating both DM and comment callers without changing core routing semantics.
