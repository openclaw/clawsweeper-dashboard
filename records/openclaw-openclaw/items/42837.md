## Summary

Keep open: current main still creates Feishu dynamic-agent direct bindings without `match.accountId`, while named-account DM and comment flows resolve and re-resolve routes with `account.accountId`, so generated bindings can miss and traffic can fall back to `agent:main`. The closed unmerged fix PR identified the right account-aware shape, and no current code PR owns the remaining regression.

## What This Changes

The issue asks Feishu dynamicAgentCreation to route each DM user to the generated per-user agent/session instead of `agent:main`, including named Feishu account setups.

## Best Possible Solution

Land a focused Feishu plugin fix that passes normalized `accountId` into dynamic-agent binding detection and creation, preserves default-account legacy binding semantics, and updates targeted regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows named-account Feishu DM/comment traffic creates account-less direct bindings, then route resolution looks for that same peer under the named account id and misses the generated binding.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the Feishu plugin: make dynamic-agent binding detection and creation account-aware using the public account-id normalizer while leaving core routing and default-account semantics unchanged.

## Repair Work Prompt

Fix the Feishu dynamicAgentCreation named-account routing regression for https://github.com/openclaw/openclaw/issues/42837. On current main, `maybeCreateDynamicAgent` is account-blind and writes direct Feishu bindings without `match.accountId`, while DM and comment callers resolve and re-resolve with `account.accountId`; named accounts can therefore fall back to `agent:main`. Update the Feishu plugin helper to accept the runtime account id, use `DEFAULT_ACCOUNT_ID`/`normalizeAccountId` from `openclaw/plugin-sdk/account-id`, treat missing binding account ids as default-account-only and `*` as any-account, and write normalized `match.accountId` for both existing-agent missing-binding and brand-new-agent paths. Preserve `configWritesAllowed` behavior, maxAgents ordering, default-account legacy behavior, and core routing semantics; do not change dynamic path-template behavior or broaden non-Feishu surfaces. Add focused regression coverage in dynamic-agent, DM, and comment-handler tests, and add a changelog entry if this becomes a user-facing fix PR.
