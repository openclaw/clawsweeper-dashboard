## Summary

Keep this open: current main still has the Feishu named-account dynamic-agent routing gap, and the directly related fix PR was closed unmerged.

## What This Changes

Fix Feishu dynamicAgentCreation so named/non-default account DMs route to per-user dynamic agents instead of falling back to `agent:main`.

## Best Possible Solution

Make Feishu dynamic-agent creation account-aware in the Feishu plugin while preserving default-account legacy bindings and leaving core routing semantics unchanged.

## Reproduction Assessment

Yes, by source inspection: enable Feishu `dynamicAgentCreation` on a named account, send a new DM, and the helper writes an account-less direct-peer binding that named-account route resolution cannot match.

## Solution Assessment

Yes. The narrow maintainable fix is to pass the normalized Feishu runtime `accountId` into the plugin helper and include it in generated bindings; loosening core routing would violate the documented default-account semantics.
