## Summary

Keep open: current main and latest release still create Feishu dynamic-agent bindings without `match.accountId`, while named-account routing re-resolves with the active account ID and can fall back to the main agent. The related account-aware repair was closed unmerged, and the merged docs PR documents the remaining limitation rather than fixing it.

## What This Changes

The issue requests that Feishu `dynamicAgentCreation` route each DM user to an independent per-user agent/session instead of falling back to `agent:main`.

## Best Possible Solution

Carry the active Feishu account ID into dynamic binding lookup and creation, while preserving default-account behavior for legacy account-less bindings.
