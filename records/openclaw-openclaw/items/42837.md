## Repair Work Prompt

Fix Feishu `dynamicAgentCreation` routing for named/non-default accounts. On current main, `maybeCreateDynamicAgent` has no account-id input and writes direct Feishu bindings without `match.accountId`; `bot.ts` and `comment-handler.ts` resolve routes with `account.accountId` before and after creation, so named-account traffic misses the generated binding and stays on `agent:main`. Add an `accountId` parameter to the Feishu dynamic-agent helper, pass the active account id from both DM and comment call sites, normalize it with the public account-id helper, and write `match.accountId` on generated direct bindings. Make existing-binding detection account-aware so account-less legacy bindings remain default-account-only, `accountId: "*"` behavior remains intact, and account-scoped bindings are not duplicated. Update focused regression coverage in `extensions/feishu/src/dynamic-agent.test.ts`, `extensions/feishu/src/bot.test.ts`, and `extensions/feishu/src/comment-handler.test.ts`; adapt the closed reference PR to current `configWritesAllowed` helper shape rather than cherry-picking blindly. Do not change core routing semantics, group routing, broadcast behavior, or Feishu path-template/sender-id filesystem behavior.
