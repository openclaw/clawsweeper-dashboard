## Summary

Current main still appears to have the reported Feishu named-account routing gap: dynamic agent creation writes account-less direct bindings, while route matching treats account-less bindings as default-account-only. The linked fix PR diagnosed the same root cause but was closed unmerged, so this issue should stay open for a narrow replacement fix.

## What This Changes

Feishu dynamic agent creation should route each direct-message user to an independent per-user agent/session when enabled, including named Feishu accounts.

## Best Possible Solution

Patch the Feishu-owned dynamic-agent helper and DM/comment call sites to carry the normalized active account id into generated bindings, while preserving legacy account-less bindings as default-account-only and `*` as any-account.

## Reproduction Assessment

Yes, by source inspection: current main generates Feishu direct bindings without `accountId`, and current routing only considers account-less bindings for the default account. I did not execute a live Feishu flow or tests in this read-only review.

## Solution Assessment

Yes. The narrow maintainable fix is to pass the active Feishu account id into dynamic-agent creation, normalize it with the shared plugin SDK account-id helper, write it into new bindings, and make existing-binding detection account-aware.
