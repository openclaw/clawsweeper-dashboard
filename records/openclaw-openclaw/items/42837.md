## Summary

Keep open: current main still omits `accountId` from Feishu dynamic-agent bindings while named-account ingress re-resolves routes with `account.accountId`, so the regression remains source-reproducible; the directly related fix PR #42886 is closed unmerged.

## What This Changes

Feishu `dynamicAgentCreation` should create account-scoped per-user DM bindings so named Feishu accounts route to independent sessions instead of `agent:main`.

## Best Possible Solution

Make Feishu dynamic-agent creation account-aware inside the Feishu plugin while preserving default-account legacy bindings and leaving core routing semantics unchanged.

## Reproduction Assessment

Yes, by source inspection: enable Feishu `dynamicAgentCreation` on a named account and send a new DM; the helper writes an account-less direct-peer binding that named-account route resolution cannot match.

## Solution Assessment

Yes. The narrow maintainable repair is to pass the normalized Feishu runtime account id into the dynamic-agent helper, make existing-binding detection account-aware, and include the account id in generated non-default bindings.
