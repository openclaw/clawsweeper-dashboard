## Summary

Keep open: current main still creates Feishu dynamic DM bindings without `match.accountId`, while Feishu message and comment routing re-resolves with the active account id. The related fix PR at https://github.com/openclaw/openclaw/pull/42886 identified the same named-account root cause but is closed unmerged, so the regression remains actionable.

## What This Changes

The issue reports Feishu `dynamicAgentCreation` creating per-user agent bindings but continuing to route named-account DMs to `agent:main`.

## Best Possible Solution

Make Feishu dynamic-agent binding creation account-aware inside the Feishu plugin, preserving default-account legacy behavior and wildcard routing semantics with focused regression coverage.

## Reproduction Assessment

Yes, source-level. With a non-default Feishu account, the helper writes or recognizes direct bindings without `match.accountId`, then message/comment flows re-resolve with that non-default account and the routing contract treats the unscoped binding as default-only.

## Solution Assessment

Yes. The narrow fix is to pass the active Feishu account id into `maybeCreateDynamicAgent`, include account scope in existing-binding checks and generated bindings, and add regression coverage without changing core routing policy.

## Repair Work Prompt

Fix the Feishu `dynamicAgentCreation` named-account regression from https://github.com/openclaw/openclaw/issues/42837. Current main resolves Feishu DM/comment routes with `accountId: account.accountId`, but `maybeCreateDynamicAgent` detects and writes dynamic direct bindings without `match.accountId`, so non-default Feishu accounts fall back to `agent:main`. Update the Feishu plugin-owned helper to accept the active account id, account-scope existing-binding detection and generated bindings, preserve default-account legacy behavior for bindings with omitted accountId, preserve wildcard `accountId: "*"` semantics, and do not change core routing policy. Add focused regression coverage for non-default DM creation, default legacy behavior, missing-binding add-path, and comment/message call sites. Add a single-line `CHANGELOG.md` fix entry because this is user-visible.
