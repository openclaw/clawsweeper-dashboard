## Summary

Keep open: current main and `v2026.5.18` still create Feishu dynamic-agent bindings without `match.accountId` while DM/comment routing re-resolves with the active account id, so named-account traffic can still fall back to `agent:main`. The closed runtime fix at https://github.com/openclaw/openclaw/pull/42886 had the right account-aware shape; the open docs PR at https://github.com/openclaw/openclaw/pull/82793 does not fix the runtime path.

## What This Changes

The issue requests that Feishu `dynamicAgentCreation` route named-account DM users to their generated per-user agents instead of `agent:main`.

## Best Possible Solution

Land a focused Feishu plugin fix that passes normalized account id into dynamic-agent binding detection and creation for DM and comment flows, preserves account-less default-account semantics, and adds regression coverage.

## Reproduction Assessment

Yes, at source level. Configure a named Feishu account with `dynamicAgentCreation.enabled`, trigger a DM or comment without an account-scoped binding, and current code writes an account-less binding that the named-account route re-resolution will not match.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the Feishu plugin by making the dynamic-agent helper account-aware through the existing account-id normalizer and updating both DM and comment callers, without changing core routing semantics.
