## Summary

Keep open: current main and latest release still create Feishu dynamic-agent bindings without `match.accountId`, while DM and comment routing re-resolve with the active account id, so named-account traffic can still fall back to `agent:main`. The closed runtime PR had the right account-aware shape, and the open docs PR does not fix this path.

## What This Changes

The issue asks Feishu `dynamicAgentCreation` to route each DM user to an independent per-user agent/session instead of falling back to `agent:main` for named-account traffic.

## Best Possible Solution

Land a focused Feishu plugin fix that passes normalized account id into dynamic-agent binding detection and creation for DM and comment flows, preserves account-less default-account semantics, and adds regression coverage.

## Reproduction Assessment

Yes, at source level. Current main and `v2026.5.18` create account-less Feishu dynamic bindings, then re-run route resolution with a named account id that the routing contract will not match against account-less bindings.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the Feishu plugin by making dynamic-agent matching and writes account-aware through the existing account-id normalizer, without changing core routing or default-account compatibility.
