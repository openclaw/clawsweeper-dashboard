## Summary

Keep open: current `main` and the latest release still create Feishu dynamic-agent bindings without account scope, while named-account DM/comment routing re-resolves with the active account id. The closed runtime fix at https://github.com/openclaw/openclaw/pull/42886 is unmerged, and the merged docs PR only documents the limitation.

## What This Changes

The issue asks for Feishu `dynamicAgentCreation` to route each named-account DM user to its generated per-user agent/session instead of falling back to `agent:main`.

## Best Possible Solution

Land a focused Feishu plugin runtime fix that makes dynamic-agent binding detection and creation account-aware for DM and comment flows while preserving default-account legacy behavior.

## Reproduction Assessment

Yes, at source level. The helper writes account-less bindings while DM/comment routing resolves named accounts with `account.accountId`, and the routing tests define account-less bindings as default-account-only.

## Solution Assessment

Yes. The maintainable fix belongs in the Feishu plugin by passing normalized `accountId` into dynamic-agent binding detection and creation, without changing core routing semantics.
