## Summary

## What This Changes

The issue asks for Feishu `dynamicAgentCreation` to route each named-account DM user to the generated per-user agent instead of `agent:main`.

## Best Possible Solution

Port the account-aware dynamic-agent binding fix into the Feishu plugin, covering message and comment flows plus regression tests for named-account and legacy default-account behavior.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a named Feishu account with `dynamicAgentCreation.enabled=true` writes a direct binding without `match.accountId`, and current routing ignores that binding for the non-default account.

## Solution Assessment

Yes. The narrow maintainable fix is to make dynamic-agent binding detection and creation account-aware, using the existing account-id normalizer and preserving legacy account-less bindings as default-only.
