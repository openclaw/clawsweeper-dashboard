## Summary

Keep open. Current main still performs stale daily/idle rollover by minting a new session and archiving the previous transcript without a reset-time compaction memory flush, and the requested reset policy controls remain absent.

## What This Changes

The issue requests reset-time context preservation for daily/idle session rollover, including a compaction-style memory flush before archive plus flush/archive-only and off/manual-only reset controls.

## Best Possible Solution

Define the reset contract explicitly: add a core stale-rollover preservation path that can run the existing memoryFlush silent turn before archive when enabled, document how it relates to hooks, and align schema/types/docs/tests for the chosen reset policy controls.

## Reproduction Assessment

Yes. A high-confidence source path is to seed a stale daily or idle session and call initSessionState; source inspection shows it creates a new session and archives the previous transcript without invoking the compaction memory-flush turn.

## Solution Assessment

Unclear. The proposed reset-time memoryFlush direction is plausible, but the default behavior, failure semantics, and overlap with hook-based PRs require maintainer product judgment before a narrow implementation can be selected.
