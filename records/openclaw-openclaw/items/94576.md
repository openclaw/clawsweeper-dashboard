Full review comments:

- **[P1] Wrap resolved .cmd shims before spawning:** `src/process/supervisor/adapters/child.ts:17`
  - body: This changes `claude` and `codex` into `.cmd` names, but `createChildAdapter` still passes the result to `spawnWithFallback`, which calls `spawn()` directly. Node's Windows contract says `.cmd`/`.bat` files are not executable on their own, so the reported ENOENT can become a batch-launch failure instead of a working agent run.
  - confidence: 0.93
- **[P2] Keep CLI shim resolution out of a core name allowlist:** `src/process/supervisor/adapters/child.ts:17`
  - body: Adding provider command names to the core supervisor means every bundled or external npm-backed CLI runtime needs another core allowlist edit to work on Windows, and this patch already misses the bundled Gemini CLI backend. The command ownership lives in CLI backend descriptors, so this should be a generic supervisor resolution path instead.
  - confidence: 0.82
