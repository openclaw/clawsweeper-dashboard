## Summary

The branch has useful live Copilot evidence, but it changes the established failure contract: disabled or failed live discovery no longer falls back to bundled Copilot rows. The linked issue asks for live metadata to win over static duplicates while retaining a recovery catalog, so the PR needs a focused precedence repair and rebase rather than the proposed fail-empty behavior.

## What This Changes

The PR removes GitHub Copilot’s bundled manifest catalog and static model overrides, updates thinking-policy fallback behavior, and adds tests using live `/models` metadata as the catalog source.

## Best Possible Solution

Keep the bundled catalog strictly as recovery data, make successful live rows replace duplicate provider/model ids with their complete metadata, and preserve explicit configuration precedence plus the winning row’s compatibility and thinking capabilities.

## Reproduction Assessment

Yes. Current source exposes separate bundled and live catalog sources, and the linked report provides a real Copilot account where a newer entitled model is absent from the visible catalog despite working when selected directly.

## Solution Assessment

No. Preferring live metadata is the right goal, but deleting the documented recovery catalog is broader than fixing duplicate precedence and introduces a separate compatibility regression.
