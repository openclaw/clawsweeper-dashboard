## Summary

Keep open for maintainer review: the patch appears to solve the Copilot live/static catalog bug through the existing provider-owned catalog seam, but the live-first union intentionally keeps fallback-only model IDs visible and needs explicit maintainer acceptance before merge.

## What This Changes

The PR changes the GitHub Copilot provider to prefer live `/models` metadata, append bounded static fallback rows, expose a static catalog hook, and add regression coverage for disabled, failed, and duplicate discovery cases.

## Best Possible Solution

Land the provider-owned live-first catalog merge if maintainers accept fallback-only visibility as the intended degraded/offline compatibility policy; otherwise narrow successful live responses to live rows only and keep static rows for disabled, unauthenticated, or failed discovery.

## Reproduction Assessment

## Solution Assessment

Yes for the implementation boundary: the patch uses the existing provider-owned live/static catalog contract and keeps live metadata authoritative for duplicate IDs. The remaining question is the maintainer product policy for fallback-only visibility after successful live discovery.
