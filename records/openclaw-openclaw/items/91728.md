## Summary

Keep open for maintainer review: the patch is a focused Copilot provider fix and is not superseded by current main, but it deliberately changes live/static model visibility and needs the maintainer fallback-policy choice before merge.

## What This Changes

The PR changes the GitHub Copilot provider to merge live `/models` catalog rows before bounded static fallback rows, add a static catalog hook, and cover disabled, failed, and duplicate discovery cases in tests.

## Best Possible Solution

Land the live-first catalog merge if maintainers accept fallback-only visibility; otherwise restrict static rows to disabled, unauthenticated, failed, or offline discovery paths while keeping live successes authoritative.

## Reproduction Assessment

Yes at source level: current main can return a non-null empty Copilot runtime catalog, and the list/picker consumers do not run static fallback after a non-null runtime result. The PR body also supplies redacted live Copilot `/models` output showing corrected model metadata after the patch.

## Solution Assessment

Yes with a policy caveat: the patch uses the provider-owned catalog seam and makes live metadata win on duplicates. The only unresolved question is whether fallback-only IDs should remain visible after successful live discovery.
