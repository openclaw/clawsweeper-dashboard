## Summary

The refreshed PR is a focused, mergeable fix for the linked Copilot catalog bug: it uses the existing hybrid catalog contract, preserves degraded fallback visibility, gives live rows precedence by model ID, has strong real-service proof, and all current checks pass. The remaining merge consideration is intentional policy rather than a patch defect: successful live discovery still appends bounded fallback-only IDs that may not be included in the account response.

## What This Changes

The PR changes the GitHub Copilot provider to merge live-discovered models ahead of bounded static fallback rows, adds a static catalog hook, and expands regression coverage for disabled, failed, and duplicate discovery cases.

## Best Possible Solution

Land the existing provider-SDK hybrid design with live rows authoritative for duplicate IDs and bounded static rows retained for degraded or missing-ID visibility, while treating the fallback-only visibility policy as an explicit provider product choice.
