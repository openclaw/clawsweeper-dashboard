## Summary

Keep this PR open: the new proof is useful for the patched `initSessionState` path, but the PR is not merge-ready because it adds a config/session-state contract while only one of the shared session resolver paths honors it.

## What This Changes

The PR adds `session.restartContinuation` to config/schema/help/type metadata and changes `initSessionState` so an aborted fresh main session can keep its existing session id when the terminal transcript is newer than the registry entry.

## Best Possible Solution

Land one maintainer-approved restart-continuation contract shared by auto-reply, agent command, and Gateway agent session resolution, with focused regression coverage for enabled and default-off behavior.

## Reproduction Assessment

Yes at source level: current main rotates when terminal main transcript mtime is newer than the registry timestamp, and the linked issue gives the lossless-claw restart scenario. I did not run a live Gateway/lossless-claw reproduction in this read-only review.

## Solution Assessment

No. The config-gated approach may be acceptable, but this patch is too narrow because sibling session resolvers still ignore the new opt-in.
