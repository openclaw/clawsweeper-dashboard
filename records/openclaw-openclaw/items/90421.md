## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Keep read-scoped listing from writing proposal state:** `src/skills/workshop/service.ts:91`
  - body: This call runs inside `listSkillProposals`, which backs `skills.proposals.list`; that gateway method is exposed with `operator.read`, while the docs reserve mutating proposal methods for `operator.admin`. Because `reconcileStaleProposals` reaches `markProposalStale` and `updateSkillProposalRecord`, a read-only UI or CLI list can permanently change a pending proposal to `stale` and refresh the manifest. Move the durable transition behind an admin-scoped lifecycle action, or return a derived warning/status without persisting from the list path.
  - confidence: 0.91
