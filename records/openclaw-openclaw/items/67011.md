## Summary

Keep open because current main and the latest release still use traversal-wide `WeakSet` tracking in diagnostic payload redaction, so the shared-reference false positive remains. The patch is the right narrow code shape, but the external draft PR still needs after-fix real OpenClaw cache-trace proof before merge.

## What This Changes

The PR changes diagnostic payload redaction to use recursion-path cycle tracking with a `WeakMap` memo and adds cache-trace regression coverage for shared object references.

## Best Possible Solution

Merge a ready PR that keeps credential and image redaction intact while switching object cycle detection to recursion-path semantics with memoized completed objects, backed by regression tests and redacted real cache-trace output proof.

## Reproduction Assessment

Yes, from source inspection rather than execution: current main uses one traversal-wide `WeakSet`, so a payload with sibling fields sharing the same object will visit that object once and then emit `[Circular]` for the later sibling.

## Solution Assessment

Yes for the code path: active recursion-path tracking plus memoized completed objects is the narrow maintainable fix. The remaining blocker is merge readiness evidence, not a different implementation layer.
