## Summary

Keep this PR open for maintainer review. Current main still has the shared-reference false positive, and the latest PR patch is focused, addresses the prior memoization review concern, and has no blocking correctness or security findings from this read-only pass.

## What This Changes

The PR changes diagnostic payload redaction from traversal-wide seen tracking to active-path plus WeakMap memoization and adds cache-trace regression coverage for repeated shared references.

## Best Possible Solution

Land or lightly adapt the active-path plus memoized sanitizer fix after maintainer review, keeping the change scoped to diagnostic payload traversal and preserving existing credential and image redaction behavior.

## Reproduction Assessment

Yes. On current main, a payload shaped like `{ left: shared, right: shared }` causes the second sibling to hit the traversal-wide `WeakSet` and become `[Circular]`; the affected call path is direct through cache-trace diagnostic payload sanitization.

## Solution Assessment

Yes. Active recursion-path tracking is the correct cycle criterion, and pairing it with `WeakMap` memoization avoids repeated work on DAG-shaped payloads without changing the redaction rules.
