## Summary

Keep open. The PR targets a real source-reproducible gateway heredoc approval bug, but current main has not absorbed the fix and the branch still has a security-sensitive P1 issue plus missing contributor real behavior proof.

## What This Changes

The PR changes gateway exec approval follow-up so heredoc-required approvals execute the original `params.command` instead of the rebuilt allowlist `enforcedCommand`.

## Best Possible Solution

Revise the branch so raw heredoc text is used only after explicit `allow-once` or `allow-always`, keep timeout fallback on `enforcedCommand`, and add focused tests plus redacted real gateway proof.

## Reproduction Assessment

Yes, source-reproducible: current main marks allowlist-satisfied heredocs as requiring approval, then after approval still passes the rebuilt `enforcedCommand` to execution. I did not run a live gateway approval flow in this read-only review.

## Solution Assessment

No. The proposed patch keys raw execution only on `requiresHeredocApproval`, but timeout fallback approval reaches the same path without an explicit human allow decision.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92
