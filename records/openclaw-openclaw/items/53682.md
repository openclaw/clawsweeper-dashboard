## Summary

Keep open: the PR targets a real heredoc approval bug, but the current patch would also run raw shell text for allowlist timeout fallback approvals and it lacks after-fix real behavior proof.

## What This Changes

The PR changes gateway exec approval follow-up so heredoc-required approvals execute the original command text instead of the rebuilt enforced allowlist command.

## Best Possible Solution

Land a narrow gateway approval fix that uses raw heredoc command text only after explicit `allow-once` or `allow-always`, keeps `enforcedCommand` for fallback approvals, and adds focused regression tests.

## Reproduction Assessment

Yes, source-reproducible. Current main and the PR diff show the timeout fallback path reaches the same execution call, and the PR selector would choose raw `params.command` for heredoc-required approvals.

## Solution Assessment

No. The fix should key off an explicit approval decision, not `requiresHeredocApproval`, so timeout fallback keeps the enforced allowlist execution boundary.
