## Summary

Keep this PR open: the reported heredoc approval bug is plausible and current main has not absorbed the change, but the branch is not mergeable as-is because it changes a host exec approval boundary for timeout fallback approvals and lacks after-fix real behavior proof.

## What This Changes

The PR changes gateway exec approval follow-up so heredoc-required approvals execute the original command text instead of the rebuilt enforced allowlist command.

## Best Possible Solution

Land a narrow gateway approval fix that uses raw heredoc command text only after explicit `allow-once` or `allow-always`, keeps `enforcedCommand` for allowlist timeout fallback approvals, and adds focused regression tests.

## Reproduction Assessment

Yes, source-reproducible. Current main always passes `enforcedCommand` after gateway approval while `buildEnforcedShellCommand` rebuilds from argv, and the PR diff shows timeout fallback approvals would also switch to raw `params.command`.

## Solution Assessment

No. The fix should key off an explicit approval decision, not `requiresHeredocApproval`, so timeout fallback keeps the enforced allowlist execution boundary.
