## Summary

Keep open for revision: current main still has the heredoc approval problem, but this PR would also switch allowlist timeout fallback approvals to raw command execution, so it is not safe to merge as written.

## What This Changes

The PR changes gateway heredoc approval follow-up to execute the original command text instead of the rebuilt allowlist command when heredoc-specific approval is required.

## Best Possible Solution

Land a narrow gateway approval fix that uses raw heredoc command text only for explicit `allow-once` or `allow-always`, keeps timeout fallback on `enforcedCommand`, and adds focused regression coverage plus real gateway proof.

## Reproduction Assessment

Yes, source-reproducible: current main requires approval for allowlist-satisfied heredoc commands and then passes the rebuilt `enforcedCommand` to execution. I did not run a live gateway approval flow in this read-only review.

## Solution Assessment

No. The patch keys raw command execution only on `requiresHeredocApproval`, so timeout fallback approval reaches the same branch without an explicit human allow decision; the safer fix should distinguish explicit decisions from fallback approvals.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92
