## Summary

Keep open: the PR targets a real gateway heredoc approval bug, but current main has not implemented the change and the proposed patch introduces a security-sensitive fallback path regression.

## What This Changes

The PR changes gateway exec approval follow-up so heredoc-required approvals execute the original command text instead of the rebuilt allowlist command.

## Best Possible Solution

Preserve raw heredoc text only after explicit `allow-once` or `allow-always`, keep `enforcedCommand` for allowlist timeout fallback, and add focused gateway approval tests.

## Reproduction Assessment

Yes, source-reproducible. A focused unit test can mock an allowlist-satisfied heredoc command, a timed-out approval decision, and `askFallback: "allowlist"`; this patch would select raw `params.command` while current main keeps `enforcedCommand`.

## Solution Assessment

No. The selector should depend on an explicit approval decision, not only `requiresHeredocApproval`, so fallback approvals retain the enforced allowlist execution boundary.

## Review Findings
