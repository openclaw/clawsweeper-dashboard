## Summary

Keep this PR open, but it should not merge as-is: the reported heredoc approval failure is source-reproducible on current main, while the proposed fix bypasses the enforced allowlist command for timeout fallback approvals and still lacks contributor-supplied real behavior proof.

## What This Changes

The PR changes gateway exec approval follow-up so heredoc-required approvals execute the original command text instead of the rebuilt allowlist `enforcedCommand`.

## Best Possible Solution

Land a narrow gateway approval fix that preserves raw heredoc command text only after explicit `allow-once` or `allow-always`, keeps timeout fallback on `enforcedCommand`, and adds focused regression coverage.

## Reproduction Assessment

Yes, source-reproducible: current main detects allowlist-satisfied heredoc segments, requires approval, then still passes the rebuilt `enforcedCommand` to execution. I did not run a live gateway approval flow in this read-only review.

## Solution Assessment

No. The proposed patch keys raw execution only on `requiresHeredocApproval`, but timeout fallback approval reaches the same path without an explicit human allow decision.

## Review Findings
