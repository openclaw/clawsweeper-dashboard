## Summary

Keep open: the PR targets a real gateway heredoc approval bug, but the current patch would run raw command text for timeout fallback approvals and still lacks after-fix real behavior proof.

## What This Changes

The PR changes gateway exec approval follow-up so heredoc-required approvals execute the original command text instead of the rebuilt enforced allowlist command.

## Best Possible Solution

Land a narrow gateway approval fix that preserves raw heredoc text only after explicit `allow-once` or `allow-always`, keeps `enforcedCommand` for allowlist timeout fallback approvals, and adds focused tests for both branches.

## Reproduction Assessment

Yes, source-reproducible. Current main rebuilds allowlist-satisfied heredoc commands into `enforcedCommand` and uses that after approval, while analyzer tests show heredoc bodies are not part of the planned argv that the enforced command renders.

## Solution Assessment

No. The PR keys raw execution only on `requiresHeredocApproval`, but timeout fallback approvals share that follow-up path; the narrower fix is to use raw command text only for explicit allow decisions.
