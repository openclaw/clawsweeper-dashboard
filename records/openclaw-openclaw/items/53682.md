## Summary

Keep open: the reported heredoc approval bug is plausible and not implemented on current main, but this branch should not merge because it would run raw heredoc shell text for allowlist timeout fallback approvals and still lacks after-fix real behavior proof.

## What This Changes

The PR changes gateway exec approval follow-up so heredoc-required approvals execute the original command text instead of the rebuilt enforced allowlist command.

## Best Possible Solution

Land a narrow gateway approval fix that uses raw heredoc command text only after explicit `allow-once` or `allow-always`, keeps timeout fallback on `enforcedCommand`, and adds focused tests plus redacted real gateway proof.

## Reproduction Assessment

Yes. Source inspection shows current main requires approval for allowlist-satisfied heredocs but still passes the rebuilt `enforcedCommand` to execution; I did not run a live gateway approval flow in this read-only review.

## Solution Assessment

No. The proposed fix uses raw command text for every heredoc-required approval, but the same path is used by allowlist timeout fallback where no explicit `allow-once` or `allow-always` decision occurred.
