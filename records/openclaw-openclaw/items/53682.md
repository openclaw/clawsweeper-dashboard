## Summary

Keep this PR open: the source bug is valid, but the patch is not mergeable as written because heredoc timeout fallback approvals would execute raw shell text instead of the enforced allowlist command.

## What This Changes

The PR changes gateway exec approval follow-up so heredoc-required approvals execute the original command text instead of the rebuilt allowlist command.

## Best Possible Solution

Preserve raw heredoc text only for explicit `allow-once` or `allow-always` decisions, keep enforced execution for allowlist timeout fallback, and cover both branches with focused gateway approval tests.

## Reproduction Assessment

Yes, source-reproducible. A focused unit test can mock an allowlist-satisfied heredoc command, a timed-out approval decision, and `askFallback: "allowlist"`; this patch selects raw `params.command` while current main keeps `enforcedCommand`.

## Solution Assessment

No. The selector should be the explicit approval decision, not `requiresHeredocApproval`, so fallback approvals keep the enforced allowlist execution boundary.
