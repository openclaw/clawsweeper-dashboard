## Summary

Keep open. The heredoc failure is plausible and current main has not implemented this PR, but the branch is not mergeable as written because it also switches allowlist timeout fallback approvals to raw shell text without an explicit human decision.

## What This Changes

The PR changes gateway exec approval follow-up so heredoc-required approvals execute the original command text instead of the rebuilt allowlist command.

## Best Possible Solution

Land a narrow gateway approval fix that preserves raw heredoc command text only after explicit `allow-once` or `allow-always`, keeps the enforced command for allowlist timeout fallback approvals, and adds focused regression tests plus the required changelog entry.

## Reproduction Assessment

Yes. A focused unit test can mock an allowlist-satisfied heredoc command, a timed-out approval decision, and `askFallback: "allowlist"`; this PR would select raw `params.command` while current main keeps `enforcedCommand`.

## Solution Assessment

No. The proposed selector is `requiresHeredocApproval`, but the safer maintainable selector is an explicit approval decision rather than a timeout fallback.
