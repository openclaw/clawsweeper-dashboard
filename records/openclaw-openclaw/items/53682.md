## Summary

Keep this PR open, but not mergeable as-is: current main has not absorbed the heredoc approval execution fix, while the branch would run raw heredoc text on allowlist timeout fallback and lacks required real behavior proof.

## What This Changes

The PR changes gateway exec approval follow-up so heredoc-required approvals pass the original command text to `runExecProcess` instead of the rebuilt allowlist command.

## Best Possible Solution

Land a narrow gateway approval fix that uses raw heredoc command text only after explicit `allow-once` or `allow-always`, keeps fallback approvals on `enforcedCommand`, and covers explicit, fallback, and non-heredoc paths.

## Reproduction Assessment

Yes, source-reproducible: current main detects allowlist-satisfied heredoc segments, requires approval, then still passes the rebuilt `enforcedCommand` to execution. I did not run a live gateway approval flow in this read-only review.

## Solution Assessment

No. The proposed patch keys raw execution only on `requiresHeredocApproval`, but timeout fallback approval reaches the same path; the safer solution is to gate raw command execution on explicit allow decisions.
