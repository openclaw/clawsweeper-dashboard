## Summary

Keep open. The PR addresses a real source-reproducible heredoc approval bug, but the current patch would also run raw shell text for allowlist timeout fallback approvals and the contributor has not posted after-fix real behavior proof.

## What This Changes

The branch changes gateway exec approval follow-up to pass the original command text instead of the rebuilt enforced allowlist command when `requiresHeredocApproval` is true.

## Best Possible Solution

Land a narrow gateway approval fix that preserves raw heredoc command text only after explicit `allow-once` or `allow-always`, keeps the enforced command for allowlist timeout fallback approvals, and adds focused tests for both branches.

## Reproduction Assessment

Yes, source-reproducible. Current main requires approval for allowlist heredocs but the follow-up still executes the rebuilt `enforcedCommand`, and the PR diff shows timeout fallback approvals would also switch to raw `params.command`.

## Solution Assessment

No. The fix should key off an explicit approval decision rather than `requiresHeredocApproval`, so timeout fallback keeps the enforced allowlist execution boundary.
