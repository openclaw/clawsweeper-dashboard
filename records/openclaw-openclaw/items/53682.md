## Summary

Keep this PR open, but do not merge it as-is: current main still has the reported heredoc approval handoff problem, while the PR introduces a security-sensitive timeout-fallback bypass and lacks after-fix real behavior proof.

## What This Changes

The PR changes gateway exec approval follow-up so heredoc-required approvals execute the original command text instead of the rebuilt allowlist enforced command.

## Best Possible Solution

Land a narrow gateway approval fix that preserves raw heredoc command text only after explicit `allow-once` or `allow-always`, keeps the enforced command for allowlist timeout fallback approvals, and adds focused regression tests.

## Reproduction Assessment

Yes, source-reproducible. Current main marks allowlist-satisfied heredoc commands as requiring approval, then still passes the rebuilt `enforcedCommand` to `runExecProcess`, while analyzer tests show heredoc bodies are not represented in the planned argv.

## Solution Assessment

No. The PR keys raw execution only on `requiresHeredocApproval`, but timeout fallback approvals share the same follow-up path; the safer fix is to use raw command text only for explicit allow decisions.
