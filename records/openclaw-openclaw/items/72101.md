## Summary

Keep open. Current main already shipped the ACP-only `streamTo` tolerance, but the native `sessions_spawn` `model:"default"` behavior still remains a real gap; this PR is a focused implementation candidate with a low-priority changelog policy issue and one ACP semantics point for maintainer review.

## What This Changes

The PR adds shared model-override normalization, applies it to `sessions_spawn` and `session_status`, gates `streamTo` to ACP runtime, and adds `sessions_spawn` regression coverage for `model:"default"`.

## Best Possible Solution

Land a rebased, focused version that normalizes native `sessions_spawn` `model:"default"` at the tool boundary, keeps `session_status` semantics aligned through a shared helper if that contract is intentional, adds the required changelog entry, and explicitly preserves or documents the chosen ACP model-sentinel behavior.

## Reproduction Assessment

Yes. A native `sessions_spawn` call with `model:"default"` is enough to trace the issue on current main: the tool forwards the literal string and subagent model selection prefers any non-empty override before fallback defaults.

## Solution Assessment

Mostly yes. Tool-boundary normalization is the narrow maintainable fix; the only safer refinement is an explicit maintainer decision on whether ACP `model:"default"` should also mean unset rather than an explicit harness model string.
