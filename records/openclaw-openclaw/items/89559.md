## Summary

Keep open: the PR is a maintainer-labeled draft and the hardening direction is useful, but the patch still rereads a potentially hostile tool name after validation and can let a stateful unreadable-name tool through as an empty-name tool.

## What This Changes

The PR wraps plugin tool shape reads in guarded accessors, skips unsafe descriptor-cache captures, and adds regressions for unreadable plugin tool name and parameter getters.

## Best Possible Solution

Centralize plugin tool shape capture so each required field is read once under try/catch, malformed tools are diagnosed and skipped, and descriptor capture consumes the same safe name/schema data.

## Reproduction Assessment

Yes. Source inspection shows current main directly reads plugin tool shape fields, and the PR diff demonstrates the intended failing path with focused regressions for unreadable name and parameters getters.

## Solution Assessment

No, not yet. The fix should avoid or revalidate the second post-validation name read so hostile/stateful name getters cannot leak a malformed tool into the resolved tool list.
