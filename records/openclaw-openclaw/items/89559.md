## Summary

## What This Changes

The branch guards plugin tool shape reads and descriptor-cache capture, then adds optional plugin-tool regressions for throwing getters.

## Best Possible Solution

Resolve plugin tool shape reads into stable checked fields or a sanitized wrapper before registration so downstream consumers never reread hostile getters.

## Reproduction Assessment

Yes. Source inspection shows a getter can return a valid name during `describeMalformedPluginTool`, throw during the later registration read, and still leave the original tool in the returned array.

## Solution Assessment

No, not yet. Guarding individual reads is a good direction, but the maintainable fix must carry stable checked fields forward or sanitize the tool before registration.
