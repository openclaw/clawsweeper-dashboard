## Summary

Keep open: current main has internal cron single-flight and removal-preservation behavior, but it still does not expose a public claim/lease, CAS remove, claim-aware exit code, or documented precondition contract. This is a core cron API/product decision, not an already-fixed cleanup item.

## What This Changes

Request adding atomic cron dispatch/remove primitives so external code can safely retire scheduled jobs without double-fires or lost removals.

## Best Possible Solution

Define the smallest additive public cron precondition contract, then update the protocol schema, Gateway handler, CLI help/docs, plugin-facing type if needed, and focused concurrency tests around that contract.

## Reproduction Assessment

Not applicable as a feature/API request. Source inspection shows the requested public claim/lease/CAS surface is absent, while current tests already cover some internal overlap and removal cases.

## Solution Assessment

Unclear until maintainers choose the public contract. A narrow additive CAS or if-unclaimed remove path may be simpler than a full lease-token system, but that is a product/API decision.
