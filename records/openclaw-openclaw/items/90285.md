## Summary

Keep open: current main and v2026.6.6 still lack a public claim/lease, CAS remove, claim-aware exit code, or documented cron precondition contract. Current code has internal runningAtMs reservation and removal-preservation behavior, but that does not satisfy the requested external cron API contract.

## What This Changes

The issue requests atomic cron job claim/lease, compare-and-set remove, or documented single-flight primitives for concurrent dispatch and removal.

## Best Possible Solution

Have maintainers choose the smallest additive public cron concurrency contract, then align the service, Gateway protocol schema, CLI help, docs, and focused concurrency tests around that contract.

## Reproduction Assessment

Not applicable as a feature/API request. Source inspection against current main and v2026.6.6 shows the requested public claim/lease/CAS fields and CLI flags are absent.

## Solution Assessment

Unclear until maintainers choose the public contract. A narrow additive CAS or `--if-unclaimed` remove path may be simpler than a full lease-token system, but either choice is an API/product decision.
