## Summary

Keep open: current main and v2026.6.8 still lack a public cron claim/lease, CAS remove flag, claim-aware exit code, or documented single-flight precondition contract. The scheduler has internal `runningAtMs` reservation and process-local locking, but that is not the requested external API for safely coordinating dispatch with `cron rm`.

## What This Changes

The issue requests a public cron concurrency primitive, such as lease/claim dispatch or CAS-aware remove flags, so external automation can safely coordinate job dispatch and removal.

## Best Possible Solution

Choose one additive public cron precondition contract, then align the Gateway protocol, service result type, CLI flags/help, docs, and focused concurrency tests around that contract.

## Reproduction Assessment

Not applicable as a feature/API request. Source inspection against current main and v2026.6.8 confirms the requested public claim/lease/CAS fields, CLI flags, and documented exit-code contract are absent.

## Solution Assessment

Unclear until maintainers choose the public contract. A narrow additive CAS or `--if-unclaimed` remove path may be smaller than a full lease-token system, but either choice is an API/product decision.
