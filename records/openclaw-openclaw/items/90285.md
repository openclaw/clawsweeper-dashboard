## Summary

Keep open: current main and v2026.6.9 still expose unconditional cron removal plus internal `runningAtMs` reservation, but not the requested public claim/lease, CAS remove, claim-aware exit code, or documented single-flight contract.

## What This Changes

The issue requests an additive cron concurrency primitive for safely retiring jobs during dispatch, such as claim/lease state, CAS removal, or an enforced single-flight contract.

## Best Possible Solution

Choose one additive public cron precondition contract, then align the Gateway schema, service result type, CLI flags/help, docs, and focused concurrency tests around that contract.

## Reproduction Assessment

Not applicable as a feature/API request. Source inspection against current main and v2026.6.9 confirms the requested public claim/lease/CAS fields, CLI flags, and documented exit-code contract are absent.

## Solution Assessment

Unclear until maintainers choose the public contract. A narrow additive CAS or `--if-unclaimed` remove path may be smaller than a full lease-token system, but either choice is an API/product decision.
