## What This Changes

Request an additive cron dispatch/removal precondition contract, such as claim/lease, CAS remove, or enforced single-flight semantics, so external automation can retire jobs safely.

## Best Possible Solution

Choose one additive public cron precondition contract, then align the Gateway schema, service result type, CLI flags/help, docs, and focused concurrency tests around that contract.

## Solution Assessment
