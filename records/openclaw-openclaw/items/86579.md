## Label Justifications

- `P1`: The report describes a current update regression that breaks a real macOS gateway/native-approval workflow for self-signed TLS users.
- `impact:security`: The affected behavior controls TLS trust roots and self-signed gateway certificate handling, which is a security boundary even when the bug is availability-facing.

## Summary

Keep open: current main and the latest release still rebuild the managed gateway service environment from installer inputs, filter an existing NODE_EXTRA_CA_CERTS value as a dangerous override, and default macOS LaunchAgent trust back to /etc/ssl/cert.pem. The related native-approvals path also does not currently reuse the gateway client's local TLS fingerprint pinning, so the reported self-signed gateway TLS failure remains plausible and not superseded by a merged fix.

## What This Changes

The issue asks OpenClaw to preserve or durably configure custom NODE_EXTRA_CA_CERTS for macOS gateway reinstalls so self-signed gateway TLS trust survives updates.

## Best Possible Solution

Keep the generated service env authoritative, then add a durable narrow trust path: preferably make native/operator gateway clients reuse local gateway TLS fingerprint pinning, and add a documented config or service-env persistence surface only if maintainers decide process-wide CA trust must be supported.

## Reproduction Assessment

Yes by source inspection, though not by a live macOS run: current main and v2026.5.22 filter an existing NODE_EXTRA_CA_CERTS value during forced install and rewrite the generated LaunchAgent env file from the default plan. The native-approvals client path also lacks the TLS fingerprint option used elsewhere for local self-signed gateway TLS.

## Solution Assessment

No, preserving arbitrary edits in the generated service env is probably not the best permanent fix because the file is generated and NODE_EXTRA_CA_CERTS is intentionally treated as a dangerous override. A narrower fix is to reuse existing local TLS fingerprint pinning for native approvals, with a maintainer-approved durable CA config only if process-wide trust remains necessary.
