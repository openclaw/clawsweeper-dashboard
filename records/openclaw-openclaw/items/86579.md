## Label Justifications

## Summary

Keep open: current main and v2026.6.8 still lack a durable managed-service extra-CA trust setting, and forced gateway service regeneration still filters edited `NODE_EXTRA_CA_CERTS` values as dangerous overrides.

## What This Changes

The issue requests a reinstall-safe way for macOS managed gateway updates to preserve or configure extra Node CA certificates for self-signed/private-root TLS deployments.

## Best Possible Solution

Land a maintainer/security-approved durable trust path for managed gateway installs, preferably explicit and config-owned or otherwise narrowly scoped, instead of preserving arbitrary generated service-env edits.

## Reproduction Assessment

Yes, from source: forced install rebuilds the service environment, filters `NODE_EXTRA_CA_CERTS` as a dangerous override, and falls back to `/etc/ssl/cert.pem` on macOS when no installer env value is supplied. I did not run a live macOS LaunchAgent reinstall in this read-only review.

## Solution Assessment

Unclear until maintainers choose the trust boundary. Preserving arbitrary generated env edits conflicts with current host-env hardening, while a config-owned extra-CA path or narrower internal TLS trust mechanism needs maintainer/security approval.
