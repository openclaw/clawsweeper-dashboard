The issue asks OpenClaw clients and forwarded session methods to preserve terminal chat.send acknowledgement statuses instead of treating every returned runId as an active run.

Land https://github.com/openclaw/openclaw/pull/91049 or an equivalent bounded replacement, then close this issue through the merge that preserves terminal ACK status across the affected clients and forwarders.

## Reproduction Assessment

Yes. Source inspection shows current main can return terminal chat.send ACKs, while current Web, TUI, ACP, Android, and forwarded-schema consumers still collapse or ignore terminal statuses in ways that can keep pending state active.

## Solution Assessment
