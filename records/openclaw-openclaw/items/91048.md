Keep open: current main still emits terminal `chat.send` acknowledgements while several consumers collapse or ignore terminal status, and the open closing PR at https://github.com/openclaw/openclaw/pull/91049 still owns the fix.

## What This Changes

Terminal `chat.send` acknowledgement consumers should preserve and branch on terminal statuses so clients do not track completed or failed runs as active.

## Best Possible Solution

Keep this issue open until https://github.com/openclaw/openclaw/pull/91049 or an equivalent status-preserving ACK fix lands across the affected clients and forwarders.

## Solution Assessment

Yes in direction. Preserving ACK status through the affected consumers is the narrow maintainable fix, and the best current path is to review and land the linked PR or replace it with an equivalent bounded implementation.
