## Summary

Keep open: current main and v2026.6.8 still collapse or drop terminal `chat.send` acknowledgement statuses in several clients, and the open closing PR is the right implementation candidate rather than a reason to close before merge.

## What This Changes

The issue asks clients and forwarded session helpers to treat terminal `chat.send` acknowledgements as terminal instead of tracking their `runId` as active.

## Best Possible Solution

Preserve terminal ACK status through each client/helper boundary, then close this issue after https://github.com/openclaw/openclaw/pull/91049 or an equivalent implementation merges.

## Reproduction Assessment

Yes. Source inspection on current main shows terminal Gateway ACKs with `runId` while Control UI, TUI, ACP, and forwarded schemas still collapse or ignore terminal status; no runtime test was run in this read-only review.

## Solution Assessment

Yes in direction. Preserving ACK status at each client/helper boundary is the maintainable fix shape, and the best current path is to review and land the linked PR or an equivalent bounded implementation.
