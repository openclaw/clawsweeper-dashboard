## Summary

Keep open: current main partially addresses MiMo reasoning replay, but Kimi/Moonshot `kimi-k2.6` and custom Kimi-compatible Chat Completions replay still have source-visible paths that drop or strip historical `reasoning_content` despite the upstream preserved-thinking contract.

## What This Changes

OpenAI-compatible Chat Completions replay should preserve historical assistant `reasoning_content` for MiMo/Kimi tool-call turns when the provider requires it.

## Best Possible Solution

Keep the stock OpenAI/Qwen stripping behavior, but add a narrow provider/model replay contract so Kimi preserved-thinking and MiMo-compatible tool-call histories keep valid `reasoning_content` through both transcript replay and Chat Completions payload sanitization.

## Reproduction Assessment

Yes, source-reproducible: current Moonshot/Kimi replay policy still drops historical reasoning before follow-up turns, and the transport sanitizer does not classify Kimi model IDs as `reasoning_content` pass-back models. I did not live-replay the provider HTTP 400 without credentials.

## Solution Assessment

Yes, a narrow code fix is the right direction: preserve reasoning only for documented provider/model contracts such as Kimi preserved thinking and MiMo-compatible reasoning models, instead of reverting the broader stock OpenAI stripping change.
