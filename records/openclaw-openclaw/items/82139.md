## Summary

Keep open: current main has partial MiMo replay work, but the reported Kimi K2.6 and MiMo V2.6-style `reasoning_content` replay gap is not fully fixed. Source and current tests still show Kimi history configured to drop prior reasoning, and the MiMo preservation allowlist does not include the reported V2.6 Pro model.

## What This Changes

The issue asks OpenAI-compatible Chat Completions replay to preserve `reasoning_content` on historical assistant tool-call messages for MiMo/kimi thinking models.

## Best Possible Solution

Add provider/model-specific replay preservation for Kimi K2.6 and MiMo V2.6 tool-call history when thinking is enabled, while keeping the stock OpenAI/Qwen reasoning-strip default intact.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: an OpenAI-compatible Kimi or unrecognized MiMo V2.6 follow-up turn passes through replay policy that drops prior assistant reasoning before transport can preserve `reasoning_content`. I did not run a live provider request in this read-only review.

## Solution Assessment

Yes, the best fix is narrow provider/model compatibility logic plus regression coverage, not a global revert of the Qwen/oMLX/vLLM strip behavior.
