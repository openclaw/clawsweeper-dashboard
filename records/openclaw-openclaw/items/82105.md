## What This Changes

This PR changes Dockerfile and setup.sh defaults so Docker builds include `discord`, `feishu`, and `voice-call` plugin runtime dependencies by default, updates Docker docs/changelog, and adds focused tests for default inclusion plus empty opt-out.

## Best Possible Solution

Land the focused Docker default only after maintainer acceptance and real Docker build/run proof showing the default image path includes these plugin dependencies while `OPENCLAW_EXTENSIONS=` remains an opt-out.

## Reproduction Assessment

Not applicable as a strict bug reproduction; this is a default Docker packaging behavior change. Source inspection on current main confirms both `Dockerfile` and `scripts/docker/setup.sh` currently default `OPENCLAW_EXTENSIONS` to empty.

## Solution Assessment

Yes, if maintainers accept the larger default image footprint: the patch updates both direct Docker builds and setup.sh, preserves explicit empty opt-out, and adds focused tests. The safer product alternative is to keep optional plugin dependencies opt-in for a slimmer default image.
