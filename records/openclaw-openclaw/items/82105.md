## Summary

Keep open: this is a focused and useful Docker packaging PR, but it is not ready because official workflow builds already pass explicit `OPENCLAW_EXTENSIONS` values that bypass the new Dockerfile default, and the attached proof is harness-only rather than a real Docker image build/run.

## What This Changes

The PR changes Dockerfile, Docker setup docs/tests, `scripts/docker/setup.sh`, and changelog so Docker builds default `OPENCLAW_EXTENSIONS` to `discord feishu voice-call` while preserving an explicit empty opt-out.

## Best Possible Solution

Centralize the Docker bundled-plugin keep-list across direct Dockerfile builds, setup.sh, release/smoke/live workflow build args, docs, and tests, preserving `OPENCLAW_EXTENSIONS=` as the slim-image opt-out.

## Reproduction Assessment

Not applicable as a strict bug reproduction. Source inspection on current `main` confirms the baseline: optional Docker plugin dependency inclusion is controlled by `OPENCLAW_EXTENSIONS`, and explicit workflow build args bypass Dockerfile defaults.

## Solution Assessment

No as written. The Dockerfile/setup default is a reasonable direction, but the patch must update or centralize official workflow build args so release, smoke, and live images get the same default list plus their extra plugins.
