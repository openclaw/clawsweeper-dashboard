## Summary

Keep open. Current main still retries the same bounded Gateway `agent` resume path for restart-aborted main sessions and records failures without a transcript-backed attach/import or explicit recovery handoff, so the reported orphaned-context path remains.

## What This Changes

Recover interrupted main chat context after Gateway restart timeouts, or expose/import the existing transcript history instead of leaving the user in a fresh session.

## Best Possible Solution

Fold this into the graceful Gateway restart design with readiness-aware retry, preserved interrupted-session binding, and a bounded transcript-backed attach/import or clear recovery-failure notice.

## Reproduction Assessment

Yes. A high-confidence source reproduction is a running main-session store entry with `abortedLastRun: true`, a resumable transcript tail, and a Gateway `agent` resume call that times out; current main records failure without importing the transcript into a new active session.

## Solution Assessment

No. The current retry loop is useful but not sufficient; increasing the timeout alone would still leave no safe transcript attach/import path or explicit user-visible recovery failure state.
