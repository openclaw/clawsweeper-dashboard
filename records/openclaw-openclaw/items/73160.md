## Summary

Keep open. Current main still routes restart-aborted main sessions through a bounded synthetic `agent` resume; a timeout only records a failed recovery attempt, and there is still no transcript attach/import path that would keep the old context available to the next active session.

## What This Changes

The issue reports that main-session restart recovery times out after a gateway restart and leaves existing transcript history orphaned instead of preserving or importing context.

## Best Possible Solution

Add readiness-aware restart recovery that preserves the interrupted session binding until recovery is definitive and ends with either safe bounded transcript-backed attach/import or a visible recovery-failure notice.

## Reproduction Assessment

Yes. A high-confidence source reproduction is a running main-session store entry with `abortedLastRun: true`, a resumable transcript tail, and a Gateway `agent` resume call that times out; current main records the failed attempt without importing or attaching the transcript.

## Solution Assessment

Unclear for the exact transcript-import mechanism. Retrying the same 10-second synthetic resume narrows some timing cases but does not solve the orphaned-transcript failure mode; the safer fix needs readiness-aware recovery plus explicit transcript preservation or user-visible failure handling.
