## Summary

Keep open. Current main still routes restart-aborted main sessions through a bounded 10s synthetic Gateway agent resume, and repeated failure records failed recovery without transcript-backed attach/import or explicit handoff; the workspace-first workaround in the discussion is useful operational context but does not close the core recovery gap.

## What This Changes

The issue asks OpenClaw to recover or surface main-session context when restart recovery times out instead of silently orphaning the previous transcript.

## Best Possible Solution

Design a restart recovery path that waits for Gateway readiness, preserves the interrupted session binding through retry/failure, and either safely reattaches/imports the transcript or surfaces a clear recovery-failure action.

## Reproduction Assessment

Yes. A source-level reproduction is a running main-session entry with `abortedLastRun: true`, a resumable transcript tail, and the Gateway `agent` resume call timing out; current main increments `failed` and leaves the old transcript unimported.

## Solution Assessment

No. The current retry of the same 10s synthetic `agent` resume is not sufficient; a maintainable fix should be readiness-aware and define safe transcript attach/import behavior.
