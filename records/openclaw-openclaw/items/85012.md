## Label Justifications

- `P1`: The report describes a broken multi-node agent shell workflow where explicit node execution still runs on the gateway/container.
- `impact:security`: The issue concerns a shell execution boundary because commands can run on the gateway despite an explicit trusted-node target.
- `impact:session-state`: The session accepts and persists exec host/node defaults, but Codex native shell/runtime selection appears not to honor those session defaults.

## Summary

Keep open: current main still has separate placement decisions for OpenClaw-managed exec and Codex native shell execution, so `/exec host=node` is not yet a single authoritative shell boundary. A related open PR is a plausible mitigation for implicit Codex routing, but it is unmerged, needs stronger proof, and a reviewer already raised that the root app-server placement policy may need a broader maintainer decision.

## What This Changes

The issue reports that `/exec host=node node=<remote-macos-node>` is accepted but agent shell commands still execute in the Linux gateway/container instead of the selected macOS node.

## Best Possible Solution

Make session exec placement authoritative across all agent shell surfaces: either route Codex native shell through the selected node-backed OpenClaw exec path or disable/fail closed on native shell when `host=node`, with regression tests and real Linux-gateway-to-macOS-node proof.

## Reproduction Assessment

Yes, at source level. Current main persists `/exec host=node`, managed OpenClaw exec can route to a node, but the Codex native shell surface and implicit Codex runtime policy do not take the exec host override as an input; I did not live-run the required Linux gateway plus macOS node setup.

## Solution Assessment

No, current main is not the complete solution because it lets Codex native shell/runtime placement remain separate from the session exec target. The maintainable fix should make node exec placement a single routing contract rather than relying on model/tool choice between native Codex shell and OpenClaw exec.
