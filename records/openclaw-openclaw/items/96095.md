## Summary

Keep this PR open, but the current patch is not merge-ready: it can treat an intentional Telegram silent final as a failed visible delivery and send a user-visible fallback message.

## What This Changes

The branch adds delivered-count tracking to the shared reply dispatcher and uses it to require confirmed Telegram final delivery before completing dispatcher or routed final replies.

## Best Possible Solution

Keep the delivered-count approach only after the visible-attempt signal is based on normalized sendable final content or delivery evidence, with regression coverage for NO_REPLY/empty finals and redacted live Telegram proof.

## Reproduction Assessment

Yes, source-reproducible. On the PR head, an exact NO_REPLY final sets `attemptedFinalDelivery` before normalization drops it, then Telegram treats the added `attemptedVisibleFinalDelivery` result as a missing visible final and emits the failure fallback.

## Solution Assessment

No as submitted. Confirming delivered Telegram finals is the right direction, but the attempt signal must distinguish normalized visible sends from intentional silent or empty finals.
