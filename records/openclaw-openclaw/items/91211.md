## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Reuse the runtime policy resolver for CLI labels:** `src/agents/model-selection-cli.ts:47`
  - body: This only checks an exact `agents.defaults.models[provider/model].agentRuntime` entry. Current runtime selection also honors agent-specific model maps, provider wildcards, provider model config, and provider-level `agentRuntime`; models routed through those supported paths will be labeled OAuth/API or blank instead of `CLI`. Use the existing runtime policy/harness resolver with the relevant agent context instead of a direct map lookup.
  - confidence: 0.91
- **[P2] Thread model API into auth labels:** `src/gateway/server-methods/models-list-result.ts:228-233`
  - body: The label helper only receives provider and model id, so every model under a provider gets the same OAuth/API label. `models.list` availability already filters auth by `entry.api` and treats OpenAI platform vs ChatGPT Responses differently; with both OpenAI OAuth and API-key profiles, platform-only entries can be shown as `OAuth` even when they require API-key auth. Pass the entry API/model auth mode through and reuse the same per-entry compatibility logic for the label.
  - confidence: 0.88
- **[P2] Default the new Swift initializer parameter:** `apps/shared/OpenClawKit/Sources/OpenClawProtocol/GatewayModels.swift:4705`
  - body: `ModelChoice` is public, and adding `runtimelabel` as a required argument to its public initializer is a source break for existing Swift callers that construct this type. Since the JSON field is optional, give the initializer parameter a `nil` default or add a compatibility overload.
  - confidence: 0.85
