## What This Changes

The PR adds a `runtimeLabel` model-choice field across TUI, Gateway protocol/Swift models, and tests, then populates it with CLI/OAuth/API labels for the `/model` picker.

## Best Possible Solution

Keep the display label as optional metadata, but derive it from the same runtime-policy and per-model auth availability path used by execution and `models.list`, preserve Swift source compatibility, and reconcile with https://github.com/openclaw/openclaw/pull/90328 if maintainers choose that broader API shape.

## Solution Assessment

No. The maintainable fix is to reuse the canonical runtime-policy and `models.list` auth availability decisions instead of adding a parallel provider-level classifier, while keeping generated Swift APIs source-compatible.
