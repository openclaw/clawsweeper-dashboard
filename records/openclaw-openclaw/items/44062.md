## Summary

Keep open. The PR is aimed at a real outbound media-root bug, but current main now routes media through the shared outbound media-access resolver, while the PR still patches older direct root call sites; strict Telegram and Feishu config schemas also still reject the advertised key, and the PR lacks required real media-send proof.

## What This Changes

The PR merges configured channel/account `mediaLocalRoots` into outbound delivery and plugin message-action send paths, adds a Telegram config type field, and adds focused outbound tests.

## Best Possible Solution

Rebase the fix around the shared media-access resolver, validate `mediaLocalRoots` in the strict channel/account schemas, resolve account-scoped roots through canonical account lookup, and attach redacted live media-send proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure channel/account `mediaLocalRoots`, send a local media payload through outbound delivery, and current main has no configured-root input in the shared resolver while strict Telegram/Feishu schemas omit the key.

## Solution Assessment

No. The direction is useful, but this PR should be rebased to the canonical shared media-access resolver and schema/account-resolution contracts instead of extending stale direct root call sites.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Thread configured roots through the shared resolver:** `src/infra/outbound/deliver.ts:548-552`
  - body: Current main computes outbound media roots through `resolveAgentScopedOutboundMediaAccess`, but this patch only adds channel/account context to the older direct `getAgentScopedMediaLocalRoots` call. After rebase, generic delivery can still drop configured roots unless the shared resolver accepts and applies that context.
  - confidence: 0.9
- **[P1] Accept mediaLocalRoots in runtime config schemas:** `src/config/types.telegram.ts:150-151`
  - body: Adding the Telegram TypeScript field is not enough for production configs. The strict Telegram and Feishu Zod schemas still omit `mediaLocalRoots`, so the advertised channel/account keys can be rejected before the helper ever reads them.
  - confidence: 0.9
- **[P2] Resolve account roots through canonical account lookup:** `src/media/local-roots.ts:66`
  - body: The helper reads `accounts[accountId]` directly. Sends that use a configured default account, or an account id whose casing differs from the config key, can still resolve/send on that account while losing its account-scoped media roots.
  - confidence: 0.84
