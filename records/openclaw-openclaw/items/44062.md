## Summary

Keep open. This PR targets a real outbound-media bug, but current main moved the active delivery path to shared media access, production schemas still reject the advertised configured roots, and the PR only has mock/test proof.

## What This Changes

The branch adds configured channel/account `mediaLocalRoots` merging to outbound delivery/message actions, a Telegram config type field, and focused regression tests.

## Best Possible Solution

Rebase onto current main, resolve configured roots in the shared outbound media-access layer, accept the chosen config contract in schemas, use canonical account resolution, and provide focused tests plus redacted live media-send proof.

## Reproduction Assessment

Yes. Source inspection is enough: configure channel/account `mediaLocalRoots`, send local media through outbound delivery, and current main has no configured-root input in the shared resolver while strict Telegram/Feishu schemas omit the key.

## Solution Assessment

No. The PR direction is useful, but the current patch targets older call sites and TypeScript typing instead of the shared resolver, runtime schemas, and canonical account lookup now needed on main.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Thread configured roots through the shared resolver:** `src/infra/outbound/deliver.ts:548`
  - body: Current main builds handler roots with `resolveAgentScopedOutboundMediaAccess`, but this patch only adds context to older direct `getAgentScopedMediaLocalRoots` calls. After rebase, generic outbound delivery and plugin actions can still drop configured channel/account roots unless the shared resolver receives and applies that context.
  - confidence: 0.9
- **[P1] Accept mediaLocalRoots in runtime config schemas:** `src/config/types.telegram.ts:150`
  - body: Adding the Telegram TypeScript field does not make production configs load. The strict Telegram and Feishu Zod schemas still omit `mediaLocalRoots`, so the advertised channel/account config keys can be rejected before the helper ever reads them.
  - confidence: 0.9
- **[P2] Resolve account roots through canonical account lookup:** `src/media/local-roots.ts:63`
  - body: The helper reads `accountsObj[accountId]` directly. Sends that use a configured default account, or an id whose casing differs from the config key, can still resolve/send on the account while losing account-scoped media roots.
  - confidence: 0.84
