## Summary

Keep open. The PR targets a real outbound media-root gap, but it is not merge-ready because current main now routes outbound media through `mediaAccess`, the PR still patches older direct root lists, strict runtime schemas still omit the advertised config key, and the contributor has not provided real media-send proof.

## What This Changes

The PR merges channel/account `mediaLocalRoots` into outbound delivery helper call sites, adds a Telegram config type field, and adds focused regression tests.

## Best Possible Solution

Define one canonical configured local-media-root contract, validate and normalize it in runtime schemas, resolve account roots canonically, merge them into `resolveAgentScopedOutboundMediaAccess`, and require redacted real media-send proof.

## Reproduction Assessment

Yes, by source inspection. Current main's shared media-access resolver and strict Telegram/Feishu schemas do not consume channel/account `mediaLocalRoots`, while outbound local media sends depend on the resolved roots.

## Solution Assessment

No. The direction is useful, but this patch should rebase onto the shared `mediaAccess` resolver, validate real runtime schemas, use canonical account selection, and include real behavior proof before merge.
