## Summary

Keep open. The PR is a useful attempt at the configured outbound media-root problem, but it is not mergeable against current main: current delivery now flows through the shared media-access resolver, the strict Telegram and Feishu schemas still do not accept the advertised config key, account-root lookup is not canonical, and real after-fix behavior proof is missing.

## What This Changes

The PR merges channel/account `mediaLocalRoots` into outbound delivery and plugin message-action media roots, adds a Telegram config type field, and adds focused outbound tests.

## Best Possible Solution

Define the canonical configured local-media root contract, validate it in strict channel/account schemas, merge it through `resolveAgentScopedOutboundMediaAccess`, use canonical account resolution, and require redacted live media-send proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure channel/account `mediaLocalRoots`, send local media through outbound delivery, and current main has no configured-root input in the shared resolver while strict Telegram/Feishu schemas omit the advertised key; I did not run tests because this review is read-only.

## Solution Assessment

No. The direction is useful, but the patch should be rebased around the shared media-access resolver, strict schema validation, and canonical account lookup rather than extending stale direct root call sites.
