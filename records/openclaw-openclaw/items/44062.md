## Summary

The PR is useful but still needs changes before merge: current main resolves outbound media roots through the shared media-access resolver, while this branch patches older direct call sites and still misses schema and account-resolution details.

## What This Changes

The PR attempts to merge channel-level and account-level `mediaLocalRoots` into outbound delivery, plugin message actions, outbound send service calls, Telegram config typing, and related regression tests.

## Best Possible Solution

Rebase the fix onto current main by resolving configured roots in the shared media-access layer, accepting the config keys through real channel schemas/metadata, and using canonical/default account resolution before selecting account roots.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: configure channel/account `mediaLocalRoots`, send a local media path through generic outbound delivery, and current main resolves media roots without any channel/account root inputs.

## Solution Assessment

No, this PR points in the right direction but is not the best mergeable fix on current main; the fix needs to move to `resolveAgentScopedOutboundMediaAccess` and align schema plus account resolution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Thread configured roots through the central resolver:** `src/infra/outbound/deliver.ts:548`
  - body: Current main builds outbound media roots through `resolveAgentScopedOutboundMediaAccess`, not the older direct delivery helper call this PR patches. Rebase the change so delivery, send-service, and message-action paths pass channel/account context into the shared resolver, otherwise generic outbound delivery will still miss configured roots.
  - confidence: 0.9
- **[P1] Accept mediaLocalRoots in real channel schemas:** `src/media/local-roots.ts:65-71`
  - body: The new helper reads channel/account `mediaLocalRoots`, but real Telegram and Feishu config schemas are strict and still do not accept those keys. Production configs for the advertised Telegram/Feishu cases will be rejected before these values reach the helper, so add schema and metadata coverage with the runtime change.
  - confidence: 0.9
- **[P2] Resolve default accounts before reading account roots:** `src/media/local-roots.ts:63`
  - body: Account roots are loaded only when an explicit `accountId` is passed. Default-account sends can resolve the account later in the channel path, so `channels.<channel>.accounts.<default>.mediaLocalRoots` remains omitted for common sends that do not set an account id.
  - confidence: 0.84
- **[P2] Normalize account ids before account-root lookup:** `src/media/local-roots.ts:63`
  - body: The helper indexes `accounts[accountId]` directly, making root lookup case-sensitive even though channel account resolution canonicalizes ids. Passing a differently cased account id can still send on the account while silently dropping its trusted media roots.
  - confidence: 0.8
