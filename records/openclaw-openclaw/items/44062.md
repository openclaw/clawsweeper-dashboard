## Summary

Keep open: current main still does not honor configured channel/account `mediaLocalRoots`, so the PR is not obsolete, but the branch is dirty and the patch is not mergeable as-is because it reads new config keys that the real strict channel schemas still reject and it bypasses normal account fallback/canonicalization.

## What This Changes

This PR threads channel/account context into outbound media-root resolution, adds configured `mediaLocalRoots` merging, declares the Telegram config type field, and adds outbound regression tests.

## Best Possible Solution

Rebase or replace the PR on current main, implement configured channel/account root merging in the current central media access resolver, accept the same fields in the real channel/account schemas and generated config metadata, resolve default/canonical account ids with shared account helpers, and add parser-level plus outbound-path regression coverage for Telegram and Feishu without changing media upload protocols.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: current main's root resolver ignores channel/account `mediaLocalRoots`, while strict Telegram and Feishu schemas do not accept the PR's advertised config fields, so a real configured Feishu/Telegram local media send can still miss those roots.

## Solution Assessment

No. The PR direction is right, but this is not the best mergeable shape because it patches older call sites and raw config lookup instead of the current central media-access path, parser schemas, and shared account resolution semantics.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Accept mediaLocalRoots in parsed channel schemas:** `src/media/local-roots.ts:65-69`
  - body: The new helper reads `channelObj?.mediaLocalRoots` and account-level `mediaLocalRoots`, but production configs go through strict Telegram and Feishu schemas that still do not declare this key. Real user configs for the advertised Feishu/Telegram cases will be rejected or never reach this helper, so add the field to the relevant channel/account schemas and cover the parser path.
  - confidence: 0.9
- **[P2] Use the default account before reading account roots:** `src/media/local-roots.ts:62`
  - body: Account roots are only loaded when `options.accountId` is explicitly set. Telegram and Feishu send paths can resolve `defaultAccount` later when no account id is passed, so default-account sends will still omit `accounts.<default>.mediaLocalRoots` and can degrade local media to text.
  - confidence: 0.84
- **[P2] Normalize account ids before indexing accounts:** `src/media/local-roots.ts:62`
  - body: The raw `accountsObj[accountId]` lookup is case-sensitive, while shared account resolution canonicalizes ids. Passing an account id with different casing can still resolve the channel account but silently skip its configured media roots here.
  - confidence: 0.8
