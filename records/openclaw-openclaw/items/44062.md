## Summary

Keep open. The reported outbound media-root bug appears real, but the branch is stale against current main's shared media-access path and still lacks runtime schema validation, canonical account-root resolution, and real behavior proof.

## What This Changes

The PR threads channel/account `mediaLocalRoots` through outbound delivery call sites, adds a Telegram config type field, and adds focused regression tests.

## Best Possible Solution

Define and validate one configured local-media-root contract in shared outbound media access, resolve channel/account roots canonically, and require redacted real media-send proof before merge.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main's shared outbound media resolver and strict schemas do not consume channel/account `mediaLocalRoots`, while local media sends depend on those resolved roots.

## Solution Assessment

No. The PR direction is useful, but the patch should rebase onto `resolveAgentScopedOutboundMediaAccess`, validate runtime schemas, resolve accounts through the canonical account-selection path, and include real behavior proof.
