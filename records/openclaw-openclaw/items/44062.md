## Summary

Keep open: the PR targets a real outbound media bug, but current main has moved the decisive delivery path to the shared media-access resolver, so this branch misses important current-main behavior and still lacks after-fix real behavior proof.

## What This Changes

This PR adds channel/account `mediaLocalRoots` merging to outbound media root handling, message-action sends, Telegram config typing, and focused regression tests.

## Best Possible Solution

Resolve configured media roots once in the shared outbound media-access layer, accept the key in real strict channel/account schemas, use canonical/default account lookup, and back it with focused tests plus redacted real behavior proof.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: configure channel/account `mediaLocalRoots`, send local media through generic outbound delivery, and current main's shared resolver has no channel/account root inputs.

## Solution Assessment

No. The PR direction is right, but the best current-main fix should target the shared resolver, schema acceptance, and canonical/default account lookup rather than only the older direct call sites.

Full review comments:

- **[P1] Thread configured roots through the shared resolver:** `src/infra/outbound/deliver.ts:548`
  - body: Current main now builds handler media roots through `resolveAgentScopedOutboundMediaAccess`; this added argument only updates the older direct root call path.   - confidence: 0.9

  - body: Adding the Telegram TypeScript field is not enough for production config loading. Telegram and Feishu still use strict schemas that lack `mediaLocalRoots`, so the advertised channel/account keys are rejected before the new helper can read them.
  - confidence: 0.9
- **[P2] Resolve default accounts before reading roots:** `src/media/local-roots.ts:63`
  - body: The helper only reads account roots when an explicit `accountId` is passed. Sends that later fall back to `channels.<channel>.defaultAccount` or a channel default can still omit account-scoped roots and degrade local media sends.
  - confidence: 0.84
- **[P2] Use canonical account lookup for account roots:** `src/media/local-roots.ts:63`
  - body: The direct `accountsObj[accountId]` lookup is case-sensitive, while channel account resolution normalizes account ids. A valid differently-cased account id can send through the account but lose its configured media roots.
  - confidence: 0.8
