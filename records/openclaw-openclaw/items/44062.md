## Summary

Keep open. The PR is aimed at a real outbound media-root bug, but current main now routes media roots through the shared outbound media-access resolver, strict channel schemas still reject the advertised config keys, account-root lookup has default/canonical lookup gaps, and the PR still lacks real behavior proof.

## What This Changes

The PR adds helper logic and call-site arguments to merge channel/account `mediaLocalRoots` into outbound delivery and message-action media roots, plus a Telegram type addition and focused regression tests.

## Best Possible Solution

Resolve configured roots in the shared outbound media-access layer, use canonical/default account resolution, settle the schema surface, and verify with focused tests plus redacted live Feishu or Telegram media-send proof.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: configure channel/account `mediaLocalRoots`, send local media through generic outbound delivery, and current main has no channel/account configured-root input in the shared resolver while strict Telegram/Feishu schemas omit the key.

## Solution Assessment

No. The PR direction is useful, but the current patch targets older call sites and TypeScript typing instead of the shared resolver, runtime schemas, and canonical account resolution now needed on main.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

- **[P1] Thread configured roots through the shared resolver:** `src/infra/outbound/deliver.ts:548`
  - body: Current main builds handler roots with `resolveAgentScopedOutboundMediaAccess`, but this patch only adds context to the older direct `getAgentScopedMediaLocalRoots` call.   - confidence: 0.9

  - confidence: 0.9

  - confidence: 0.84
