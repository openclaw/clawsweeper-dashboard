## Summary

Keep open: the reported bug is still valid on current main, but this PR is not mergeable as-is because it targets stale outbound call sites and misses the current shared media-access resolver, real config schema acceptance, and canonical account lookup behavior.

## What This Changes

The PR adds configured channel/account `mediaLocalRoots` merging to outbound delivery, plugin message actions, outbound send service plumbing, and related regression tests.

## Best Possible Solution

Rework the fix on current main so configured roots are resolved once in the shared media-access layer, accepted by real channel/account schemas, selected through canonical/default account resolution, covered by focused tests, and backed by redacted real behavior proof.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: configure channel/account `mediaLocalRoots` and send local media through generic outbound delivery; current main routes through `resolveAgentScopedOutboundMediaAccess`, which has no channel/account root inputs. I did not run tests because this was a read-only review.

## Solution Assessment

No, the PR direction is right but the patch is not the best current-main solution; the fix should target the shared resolver, schema/metadata acceptance, and canonical/default account lookup rather than stale direct call sites.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Thread configured roots through the shared resolver:** `src/infra/outbound/deliver.ts:548`
  - body: Current main builds outbound media access through `resolveAgentScopedOutboundMediaAccess` before creating the channel handler. This patch only threads channel/account context into older `getAgentScopedMediaLocalRoots` call sites, so after rebase generic outbound delivery still misses configured roots unless the shared resolver carries that context.
  - confidence: 0.9
- **[P1] Accept mediaLocalRoots in real config schemas:** `src/media/local-roots.ts:65-71`
  - body: The helper reads `channels.<channel>.mediaLocalRoots`, but strict Telegram and Feishu schemas still reject the advertised keys, so normal parsed configs never reach this code. Add schema and generated metadata coverage for the channels being fixed.
  - confidence: 0.9
- **[P2] Resolve default accounts before root lookup:** `src/media/local-roots.ts:63`
  - body: Account roots are only loaded when `accountId` is explicitly provided. Sends that later fall back to `channels.<channel>.defaultAccount` or the channel default will still omit account-scoped roots.
  - confidence: 0.84
- **[P2] Use canonical account lookup for account roots:** `src/media/local-roots.ts:63`
  - body: The direct `accountsObj[accountId]` lookup is case-sensitive, while channel account resolution normalizes ids. An otherwise valid account id with different casing can send through that account but lose its media roots.
  - confidence: 0.8
