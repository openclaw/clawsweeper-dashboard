## Summary

Keep open. The linked Feishu/media-root bug is still valid on current main, and this PR is useful source work, but the patch is not mergeable as-is because it reads new config keys that the real strict channel schemas still reject and it does account-root lookup outside the normal default/canonical account resolution path.

## What This Changes

This PR threads channel/account context into outbound media-root resolution, adds configured `mediaLocalRoots` merging, declares the Telegram config type field, and adds outbound regression tests.

## Best Possible Solution

Update or replace the PR against current main by implementing configured channel/account root merging in the central media access resolver, accepting the same fields in the real Telegram and Feishu channel/account schemas plus generated metadata, and resolving default/canonical account ids through shared account helpers while preserving existing default roots, agent workspace roots, and media upload protocols.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: configure `channels.feishu.mediaLocalRoots` or account-level roots and send local Feishu media; current main's resolver ignores those configured roots, while the strict Feishu schema does not accept the advertised field.

## Solution Assessment

No. The PR direction is right, but the current patch is not the best mergeable shape because it patches stale call sites and raw config lookup instead of the current central media-access path, real parser schemas, and shared account resolution semantics.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Accept mediaLocalRoots in parsed channel schemas:** `src/media/local-roots.ts:65-69`
  - body: The new helper reads `channelObj?.mediaLocalRoots` and account-level `mediaLocalRoots`, but production configs go through strict Telegram and Feishu schemas that still do not declare this key. Real user configs for the advertised Feishu/Telegram cases will be rejected or never reach this helper, so add the field to the relevant channel/account schemas and cover the parser path.
  - confidence: 0.9
- **[P2] Resolve default account before reading account roots:** `src/media/local-roots.ts:62`
  - body: Account roots are only loaded when `options.accountId` is explicitly set. Telegram and Feishu send paths can resolve `defaultAccount` later when no account id is passed, so default-account sends will still omit `accounts.<default>.mediaLocalRoots` and can degrade local media to text.
  - confidence: 0.84
- **[P2] Normalize account ids before indexing accounts:** `src/media/local-roots.ts:62`
  - body: The raw `accountsObj[accountId]` lookup is case-sensitive, while shared account resolution canonicalizes ids. Passing an account id with different casing can still resolve the channel account but silently skip its configured media roots here.
  - confidence: 0.8
