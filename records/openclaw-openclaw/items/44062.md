## Summary

Keep open. The PR is a useful direction, but it is stale/conflicting against current main’s shared outbound media-access path, still has blocking schema/account-resolution issues, and lacks after-fix real behavior proof.

## What This Changes

The PR merges channel/account `mediaLocalRoots` into outbound delivery root lists, declares a Telegram config type field, and adds focused outbound media-root regression tests.

## Best Possible Solution

Define one canonical configured local-media-root contract in shared outbound media access, validate it in relevant channel/account schemas, resolve accounts canonically, and require a real media-send proof after the fix.

## Reproduction Assessment

Yes, by source inspection. Configure channel/account `mediaLocalRoots`, send a local media payload through outbound delivery, and current main’s shared resolver has no channel/account configured-root input while strict schemas omit the key.

## Solution Assessment

No. The PR direction is useful, but the patch should route configured roots through `resolveAgentScopedOutboundMediaAccess`, validate schemas, resolve accounts canonically, and include real behavior proof before merge.
