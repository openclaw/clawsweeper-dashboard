## Summary

Keep open: the PR addresses a real outbound media-root bug linked from #44050, but the branch is stale against current main and misses the current shared resolver, real config schemas, and account-resolution behavior.

## What This Changes

The PR adds channel/account mediaLocalRoots merging to outbound delivery, plugin message actions, outbound send service calls, and related regression tests.

## Best Possible Solution

Rework the fix on current main so configured roots are resolved once in the shared media-access layer, validated by real channel/account schemas, and selected through canonical/default account resolution.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: configure channel/account mediaLocalRoots and send local media through generic outbound delivery; current main routes through resolveAgentScopedOutboundMediaAccess, which has no channel/account root inputs.

## Solution Assessment

No, the PR direction is right but the patch is not the best mergeable form on current main; the fix should target the shared resolver, schema/metadata acceptance, and canonical account lookup rather than stale direct call sites.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Thread configured roots through the central resolver:** `src/infra/outbound/deliver.ts:548`
  - body: Current main no longer builds outbound roots at this direct delivery call site; it routes media through resolveAgentScopedOutboundMediaAccess. Rebase the fix into that shared resolver and pass channel/account context from delivery, send-service, and message-action paths, or generic outbound delivery will still miss configured roots.
  - confidence: 0.9
- **[P1] Accept mediaLocalRoots in parsed channel schemas:** `src/media/local-roots.ts:65-71`
  - body: The helper reads channel/account mediaLocalRoots, but the real Telegram and Feishu schemas are strict and this PR only adds the Telegram TypeScript type. User configs for the advertised cases will be rejected before these values reach the helper, so add schema and config-metadata coverage.
  - confidence: 0.9
- **[P2] Resolve the default account before reading account roots:** `src/media/local-roots.ts:63`
  - body: Account roots are only loaded when accountId is explicitly set. Default-account sends can resolve the account later in the channel path, so channels.<channel>.accounts.<default>.mediaLocalRoots remains omitted for common sends without an explicit account id.
  - confidence: 0.84
- **[P2] Normalize account ids before account-root lookup:** `src/media/local-roots.ts:63`
  - body: The direct accountsObj[accountId] lookup is case-sensitive, while channel account resolution canonicalizes ids. Passing WORK for a configured work account can still select the account but silently skip its trusted media roots here.
  - confidence: 0.8
