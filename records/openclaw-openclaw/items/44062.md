## Summary

Keep open. The reported bug is still valid, but this PR is stale against current main and would not yet make configured channel/account `mediaLocalRoots` usable through the current central media-access path or strict channel schemas.

## What This Changes

The PR merges raw channel/account `mediaLocalRoots` into outbound media roots, declares a Telegram config type field, and adds outbound regression coverage.

## Best Possible Solution

Rework the fix on current main so configured roots are resolved once in the shared media-access layer, accepted by real channel/account schemas, and account ids follow the same default/canonical resolution as outbound send paths.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: configure Feishu or Telegram `mediaLocalRoots`, send local media through generic outbound delivery, and the current central resolver has no path to include those configured roots while the strict schemas reject the advertised keys.

## Solution Assessment

No. The PR direction is right, but the mergeable solution should target `resolveAgentScopedOutboundMediaAccess`, parser schemas, generated metadata, and shared account-resolution helpers instead of older direct call sites and raw account lookup.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Thread configured roots through the central resolver:** `src/infra/outbound/deliver.ts:548`
  - body: Current main no longer builds outbound roots at this direct delivery call site; it routes media through `resolveAgentScopedOutboundMediaAccess`. Rebase the fix into that shared resolver and pass channel/account context from delivery, send-service, and message-action paths, or generic outbound delivery will still miss configured roots.
  - confidence: 0.9
- **[P1] Accept mediaLocalRoots in parsed channel schemas:** `src/media/local-roots.ts:65-71`
  - body: The helper reads channel/account `mediaLocalRoots`, but the real Telegram and Feishu schemas are strict and this PR only adds the Telegram TypeScript type. User configs for the advertised Feishu/Telegram cases will be rejected before these values reach the helper, so add schema and config-metadata coverage.
  - confidence: 0.9
- **[P2] Resolve the default account before reading account roots:** `src/media/local-roots.ts:63`
  - body: Account roots are only loaded when `options.accountId` is explicitly set. Default-account sends can resolve the account later in the channel path, so `channels.<channel>.accounts.<default>.mediaLocalRoots` remains omitted for common sends without an explicit account id.
  - confidence: 0.84
- **[P2] Normalize account ids before account-root lookup:** `src/media/local-roots.ts:63`
  - body: The direct `accountsObj[accountId]` lookup is case-sensitive, while channel account resolution canonicalizes ids. Passing `WORK` for a configured `work` account can still select the account but silently skip its trusted media roots here.
  - confidence: 0.8
