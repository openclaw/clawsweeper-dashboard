## Summary

Keep open. The reported outbound media-root bug is source-reproducible, but this PR is not merge-ready because current main now routes media through the shared media-access resolver, the strict Telegram/Feishu schemas still omit the advertised config key, account roots bypass canonical account lookup, and the contributor proof is tests-only.

## What This Changes

The PR merges channel/account `mediaLocalRoots` into outbound delivery root lists, declares a Telegram config type field, and adds focused outbound media-root regression tests.

## Best Possible Solution

Define one canonical configured local-media-root contract in the shared outbound media-access resolver, validate it in the relevant strict channel/account schemas, use canonical account resolution, and prove a real media send after the fix.

## Reproduction Assessment

Yes, by source inspection. Configure channel/account `mediaLocalRoots`, send a local media payload through outbound delivery, and current main has no configured-root input in the shared resolver while strict Telegram/Feishu schemas omit the key.

## Solution Assessment

No. The direction is useful, but the current patch is not the narrow maintainable fix until it routes configured roots through `resolveAgentScopedOutboundMediaAccess`, validates the config contract, and resolves accounts canonically.
