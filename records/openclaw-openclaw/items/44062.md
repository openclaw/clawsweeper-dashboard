## Summary

Keep open. The branch targets a real outbound-media gap documented by https://github.com/openclaw/openclaw/issues/44050, but current main now routes delivery and plugin actions through `resolveAgentScopedOutboundMediaAccess`; this PR still patches older direct root call sites, omits the strict Telegram/Feishu runtime schemas, and does not use canonical account resolution. It also still needs after-fix real behavior proof before merge.

## What This Changes

The branch adds channel/account `mediaLocalRoots` merging in `src/media/local-roots.ts`, threads it through outbound send call sites, adds a Telegram config type field, and adds focused outbound tests.

## Best Possible Solution

Rebase onto current main, choose the canonical local-media root config contract, validate it in strict runtime schemas, merge it inside `resolveAgentScopedOutboundMediaAccess`, resolve account roots through the canonical account resolver, and add focused schema/resolver/delivery tests plus redacted live media-send proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure channel/account `mediaLocalRoots`, send local media through outbound delivery, and current main has no configured-root input in the shared resolver while strict Telegram/Feishu schemas omit the key.

## Solution Assessment

No. The direction is useful, but the current patch is not the best fix on main because it patches older call sites and TypeScript shape instead of the shared media-access resolver, runtime schemas, and canonical account lookup.
