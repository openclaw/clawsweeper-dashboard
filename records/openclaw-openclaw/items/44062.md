## Summary

Keep open. The PR addresses a real outbound media-root gap, but current main moved delivery through the shared media-access resolver, the strict Telegram and Feishu config schemas still reject the advertised keys, and the PR lacks real after-fix media-send proof.

## What This Changes

The branch merges channel/account `mediaLocalRoots` into outbound media roots, threads them through outbound delivery/plugin-action call sites, adds a Telegram config type field, and adds focused tests.

## Best Possible Solution

Define the canonical configured local-media root contract, validate it in strict runtime schemas, merge it through `resolveAgentScopedOutboundMediaAccess` with canonical account resolution, and require redacted live media-send proof before merge.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure channel/account `mediaLocalRoots`, send local media through outbound delivery, and current main has no configured-root input in the shared resolver while strict Telegram/Feishu schemas omit the key.

## Solution Assessment

No. The direction is useful, but the current PR is not the best main-branch fix because it patches older call sites and TypeScript shape instead of the shared media-access resolver, runtime schemas, and canonical account lookup.
