## Summary

Keep open: the branch is a plausible fix direction, but current main has moved outbound media through a shared `mediaAccess` resolver, while this PR still targets older direct root-list calls and does not validate the advertised config surface or provide real media-send proof.

## What This Changes

The PR attempts to merge channel/account `mediaLocalRoots` into outbound delivery, add a Telegram config type field, and add focused outbound media-root regression tests.

## Best Possible Solution

Define and validate one configured local-media-root contract in shared outbound media access, resolve account roots canonically, and prove a real configured-root media send after rebase.

## Reproduction Assessment

Yes. By source inspection, configuring channel/account `mediaLocalRoots` and sending local media still reaches current main's shared resolver without configured-root input, while strict schemas omit the advertised keys.

## Solution Assessment

No. The PR direction is useful, but it should rebase onto `resolveAgentScopedOutboundMediaAccess`, validate runtime schemas, resolve accounts through canonical account selection, and include real behavior proof before merge.
