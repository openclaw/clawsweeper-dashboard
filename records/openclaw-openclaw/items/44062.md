## Summary

Keep open: the underlying bug is still source-reproducible, but this PR is not mergeable as-is because current main now routes outbound media through the shared media-access resolver, real channel schemas still reject the advertised config keys, and the PR only has mock/test proof.

## What This Changes

This PR adds channel/account `mediaLocalRoots` merging to outbound delivery/message-action media root plumbing, Telegram config typing, and focused outbound regression tests.

## Best Possible Solution

Rework the fix on current main so configured roots are resolved once in the shared media-access layer, accepted by real channel/account schemas, selected through canonical/default account resolution, covered by focused tests, and backed by redacted real behavior proof.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: configure channel/account `mediaLocalRoots` and send local media through generic outbound delivery; current main routes through a shared resolver that has no channel/account configured-root inputs. I did not run tests because this was a read-only review.

## Solution Assessment

No, the direction is right but this patch is not the best current-main solution; the fix should target the shared resolver, schema/metadata acceptance, and canonical/default account lookup rather than stale direct call sites.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Thread configured roots through the shared resolver:** `src/infra/outbound/deliver.ts:548`
  - body: Current main builds outbound media access with `resolveAgentScopedOutboundMediaAccess` before handler creation. This patch only threads channel/account context into older `getAgentScopedMediaLocalRoots` call sites, so after rebase generic outbound delivery and plugin actions can still omit configured roots unless the shared resolver carries that context.
  - confidence: 0.9
- **[P1] Accept mediaLocalRoots in parsed channel schemas:** `src/media/local-roots.ts:65-71`
  - body: The new helper reads `channels.<channel>.mediaLocalRoots` and account roots from config objects, but strict Telegram and Feishu schemas still do not declare those keys. Normal parsed configs can fail validation before these values ever reach the helper, so the advertised fix remains unusable for those real channel configs.
  - confidence: 0.9
- **[P2] Use canonical account selection for account roots:** `src/media/local-roots.ts:63`
  - body: Account roots are loaded only with an explicit exact-case `accountId` via direct map access. Sends that rely on a channel default account, or pass an account id that normal channel routing resolves case-insensitively, can send through the intended account while still omitting its configured media roots.
  - confidence: 0.84
