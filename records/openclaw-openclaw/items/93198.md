Yes. Current main persists CLI runtime metadata as `claude-cli`, then resolves context tokens with that provider while the resolver does not inspect canonical Anthropic config before the fixed 1M fallback.

## Solution Assessment

No, not fully. The resolver/session-store boundary is the right fix layer, but the compaction regression and proof should match the normal Claude CLI `ownsNativeCompaction` deferral contract.
