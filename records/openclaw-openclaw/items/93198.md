## Summary

Keep open: this is the live candidate fix for the linked Claude CLI configured-budget bug, and current main still lacks the canonical Anthropic fallback for claude-cli context-token persistence.

## What This Changes

The PR teaches context-token resolution to fall back from the claude-cli runtime id to canonical anthropic model config and adds resolver, session-store, and CLI-compaction regression coverage.

## Best Possible Solution

Land this PR after maintainer acceptance of the persisted-budget semantics and current CI, or replace the narrow mapping with an explicitly approved generic backend modelProvider contract before merge.

## Reproduction Assessment

Yes. Source inspection shows Claude CLI runs persist provider metadata as claude-cli, session-store resolves contextTokens through that provider, and current main does not inspect canonical anthropic config before fallback; I did not run a live OAuth compaction session.

## Solution Assessment

Yes, with maintainer acceptance. Moving the configured-budget lookup into the context-token resolver before session-store persistence is the narrow root-cause fix; the remaining choice is whether the claude-cli-to-anthropic mapping should stay narrow or become a generic backend contract.
