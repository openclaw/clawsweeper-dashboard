## Summary

Keep this PR open: current main still has the Claude CLI configured `contextTokens` resolver gap, and this PR remains the live candidate fix for https://github.com/openclaw/openclaw/issues/80933. The patch appears correct, but the hard-coded runtime-to-canonical provider mapping and changed persisted session budget are compatibility-sensitive and still need maintainer acceptance before merge.

## What This Changes

This PR makes `claude-cli` context-token resolution consult canonical Anthropic model config before fixed fallback and adds resolver, session-store, and CLI compaction regression coverage.

## Best Possible Solution

Land a resolver/session-store fix after maintainers choose whether the Claude CLI canonical-provider mapping belongs locally in context resolution or should be sourced from the CLI backend binding contract.

## Reproduction Assessment

Yes. Source inspection shows current main persists context budgets through `providerUsed`/`modelUsed`, so `claude-cli` misses canonical `models.providers.anthropic` `contextTokens` before falling back; the contributor also supplied deterministic after-fix terminal proof.

## Solution Assessment

Unclear as a best fix until maintainers choose the provider-alias source of truth. The patch is a narrow, correct-looking fix, but the alternative is to source the mapping from the existing CLI backend `modelProvider` binding rather than a local special case.
