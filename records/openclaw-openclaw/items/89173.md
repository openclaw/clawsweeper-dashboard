## Summary

Current main and the v2026.6.6 release already route manifest-declared external plugin tools into the agent tool surface, including the path-based plugin cold-load case described in this report.

## What This Changes

The issue reports that external plugin management tools such as memory_store and memory_recall do not appear in the agent tool list after OpenClaw v2026.5.27.

## Best Possible Solution

Keep the shipped generic plugin-tool routing path and ask for a new focused report only if a specific external plugin still fails on v2026.6.6 or newer.

## Reproduction Assessment

Yes, the source path is clear: plugin tools must flow from contracts.tools through registry loading into createOpenClawTools. I did not install the external package because this was a read-only review, but current source and shipped tests cover the reported routing boundary.

## Solution Assessment

Yes. The implemented generic manifest-contract and cold-load path is the maintainable fix because it routes all enabled external plugin tools without special-casing the memory plugin.
