## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Trim media-token prefix padding from retained text:** `src/media/parse.ts:218-222`
  - body: When a line starts with spaces before a valid MEDIA token, keeping raw `pieces.join("")` preserves whitespace that belonged to the stripped directive. Input like `  MEDIA:https://x.test/a.png caption` can return `  caption`, introducing artificial Markdown indentation.
  - confidence: 0.9
- **[P2] Collapse gaps after stripped multiline directives:** `src/utils/directive-tags.ts:32-36`
  - body: For multiline input, this cleanup only removes spaces before newlines. A directive on its own line or followed by separator space can leave an extra blank line or leading space introduced by the stripped tag, changing outbound message formatting.
  - confidence: 0.86
- **[P2] Collapse gaps after audio tag removal:** `src/media/parse.ts:233`
  - body: Changing audio cleanup to `audioTagResult.text.trim()` preserves the newline pair left when `[[audio_as_voice]]` is on a line by itself, so outbound text can gain an extra blank paragraph after the directive is stripped.
  - confidence: 0.88
