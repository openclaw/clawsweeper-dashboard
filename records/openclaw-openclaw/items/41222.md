Full review comments:

- **[P2] Trim media-token indentation before keeping captions:** `src/media/parse.ts:218-222`
  - body: Keeping raw `pieces.join("")` after removing a valid MEDIA token preserves indentation that belonged to the removed token. Input like `  MEDIA:https://x.test/a.png caption` can return `  caption`, changing the caption into an indented block.
  - confidence: 0.9
- **[P2] Collapse directive-only gaps in multiline replies:** `src/utils/directive-tags.ts:32-36`
  - body: For multiline input, this cleanup only removes spaces before newlines. A directive on its own line or followed by separator space can leave an extra blank line or leading space introduced by the stripped tag, changing outbound message formatting.
  - confidence: 0.86
- **[P2] Collapse audio-only blank lines after stripping tags:** `src/media/parse.ts:233`
  - body: Changing audio cleanup to `audioTagResult.text.trim()` preserves the newline pair left when `[[audio_as_voice]]` is on a line by itself, so outbound text can gain an extra blank paragraph after the directive is stripped.
  - confidence: 0.88
