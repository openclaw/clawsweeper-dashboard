## What This Changes

The PR changes outbound directive and media parsing plus focused tests so multiline assistant reply text keeps indentation after reply, audio, and media tag stripping.

## Best Possible Solution

Refresh the branch into a narrow current-main parser fix that preserves authored indentation while trimming only whitespace introduced by stripped directive or media syntax, then add focused regression coverage and redacted real OpenClaw proof.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: current main routes outbound payload text through parseReplyDirectives, and retained text after MEDIA extraction still reaches cleanLineText plus final whitespace folding; I did not run tests because this review is read-only.

## Solution Assessment

No. The direction is right, but the submitted patch removes cleanup too broadly and leaves artificial caption indentation plus blank-line or leading-space artifacts after directive and audio tag stripping.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Trim media-token prefix padding from retained text:** `src/media/parse.ts:218-222`
  - body: When a line starts with spaces before a valid MEDIA token, keeping raw `pieces.join("")` after token removal preserves whitespace from the stripped directive. Input like `  MEDIA:https://x.test/a.png caption` can return `  caption`, introducing artificial Markdown indentation.
  - confidence: 0.9
- **[P2] Collapse gaps after stripped multiline directives:** `src/utils/directive-tags.ts:32-36`
  - body: For multiline input, this cleanup only removes spaces before newlines. A directive on its own line, or followed by separator space, can leave an extra blank line or leading space introduced by the stripped tag.
  - confidence: 0.86
- **[P2] Collapse gaps after audio tag removal:** `src/media/parse.ts:233`
  - body: Changing audio cleanup to `audioTagResult.text.trim()` preserves the newline pair left when `[[audio_as_voice]]` is on a line by itself, so outbound text can gain an extra blank paragraph after the directive is stripped.
  - confidence: 0.88
