## What This Changes

The PR changes outbound directive and media parsing so multiline reply text keeps indentation after stripping reply, audio, and media tags, with focused parser and outbound JSON regression tests.

## Best Possible Solution

Refresh this branch or replace it with a narrow parser patch that preserves authored body indentation while trimming only whitespace introduced by removed reply/audio/media syntax.

## Reproduction Assessment

Yes. Source inspection shows retained text after splitMediaFromOutput("MEDIA:https://x.test/a.png\n  hello\n    world") still reaches cleanLineText and final whitespace folding on current main, which collapses meaningful indentation.

## Solution Assessment

No. The direction is right, but this PR head removes cleanup too broadly around stripped directive/media syntax, so the best fix is a narrower refresh that preserves indentation without introducing artificial spaces or blank paragraphs.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Trim media-token prefix padding from captions:** `src/media/parse.ts:218`
  - body: When a line starts with spaces before MEDIA:, returning raw pieces.join("") preserves directive-prefix padding in the remaining caption. For input like "  MEDIA:https://x.test/a.png caption", the visible text becomes "  caption", introducing indentation that was not authored body text.
  - confidence: 0.9
- **[P2] Normalize gaps after stripped multiline directives:** `src/utils/directive-tags.ts:36`
  - body: For multiline input, this cleanup only removes spaces before newlines. A directive on its own line, or a directive followed by a separator space, can leave an extra blank line or leading space introduced by the stripped tag.
  - confidence: 0.86
- **[P2] Collapse gaps after audio tag removal:** `src/media/parse.ts:233`
  - body: Changing audio cleanup to audioTagResult.text.trim() preserves newline pairs left when [[audio_as_voice]] is on its own line, so outbound text gains an extra blank paragraph after the directive is stripped.
  - confidence: 0.88
