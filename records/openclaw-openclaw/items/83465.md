The issue requests that Lossless/LCM defaults or doctor repair include the canonical mini summary model ref so `openai/gpt-5.4-mini` requests are not denied by a legacy `openai-codex/*` plugin LLM allowlist.

## Best Possible Solution

Add a targeted Lossless doctor/default normalization that preserves explicit allowlists while adding the canonical `openai/gpt-5.4-mini` ref when the configured Lossless summary path can request it.
