## Summary

Keep open: this is an active fix attempt for a real session-state bug, but it is not merge-ready because the mechanical guard only reaches embedded OpenClaw `sessions_spawn` while the implicated CLI/Codex-native spawn path remains available.

## What This Changes

The PR adds fallback-run metadata, completion-announcement provenance detection, announcement prompt hardening, an embedded `sessions_spawn` fallback-announcement guard, and focused regression tests.

## Best Possible Solution

Represent fallback-completion handling as a per-turn runtime mode and mechanically reject or remove every spawn-capable surface for that mode, including Codex-native collaboration, while preserving normal spawning for unrelated fallback and resume turns.

## Reproduction Assessment

## Solution Assessment

No. The embedded `sessions_spawn` guard is a useful slice, but the best fix must also cover or explicitly split the CLI/Codex-native spawn path before the shared invariant is true.
