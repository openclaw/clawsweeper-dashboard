## Summary

The PR therefore remains useful but is not yet a complete fix for the reported behavior.

## What This Changes

The PR marks non-primary model attempts, detects completion-announcement provenance, blocks embedded `sessions_spawn` for that combined state, and strengthens completed-work instructions across harnesses.

## Best Possible Solution

Represent the forbidden fallback-completion condition as a closed run state carried through harness selection, mechanically suppress both embedded and Codex-native spawning for that turn, and verify it using a real fallback completion-announcement session.
