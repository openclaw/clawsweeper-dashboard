## Summary

Keep open: current main still matches the report because doctor is allowed to continue with invalid config, final validation only prints `Invalid config:` diagnostics, and the command wrapper exits 0 after a resolved doctor run. Related reports #77802 and #77803 cover adjacent repair/migration failures, but this exit-status problem is distinct and narrow.

## What This Changes

The issue requests that `openclaw doctor` return a non-zero exit code when it reports invalid configuration errors.

## Best Possible Solution

Propagate final invalid-config validation as a doctor failure state, use a failure outro, and exit non-zero while preserving doctor's ability to inspect and repair invalid configs.

## Reproduction Assessment

Yes, source-reproducible on current main: invalid config is allowlisted for `doctor`, final validation prints errors through `runtime.error`, and the wrapper exits 0 when `doctorCommand` resolves. A live CLI run was not attempted because this review is read-only.

## Solution Assessment

Yes, the narrow maintainable fix is to propagate final invalid-config status to the doctor result/outro and command exit code, not to make preflight block doctor startup. Keep config repair and bailian enum migration work in the related issues.
