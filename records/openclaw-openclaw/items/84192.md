Keep open: current `main` still has separate missing-plugin warning paths for `plugins.entries` and `plugins.allow`, and the shipped `v2026.5.18` tag contains the same behavior. The exact Windows/Bun default-startup path was not live-run, but the duplicated warning shape is source-reproducible and narrow enough to fix.

## What This Changes

The issue requests that a missing `yuanbao` plugin produce at most one actionable startup/config warning and only when the user has actually configured or enabled it.

## Best Possible Solution

Deduplicate missing official-plugin warnings by normalized plugin id while preserving one actionable path/hint, and ensure a truly default config with no material `yuanbao` reference stays quiet.

## Reproduction Assessment

Yes for the duplicate warning shape via source inspection: a config that references the same missing official plugin in both `plugins.entries` and `plugins.allow` will emit both warnings. The reported default-startup path still needs live confirmation.

## Solution Assessment

Yes, a narrow config-validation fix is the best path: dedupe the repeated missing-plugin warning and add regression coverage for `yuanbao` or another official external plugin across both config surfaces.
