## What This Changes

The PR changes OpenAI reasoning-block replay sanitization to drop signed `thinking` blocks with empty, whitespace-only, or non-string `thinking` text and adds focused sanitizer tests.

## Best Possible Solution

Revise the sanitizer to drop only truly non-replayable empty reasoning while preserving parsed signed payloads with replayable OpenAI state such as encrypted content, then add regression coverage and redacted real runtime proof.

## Reproduction Assessment

Yes, at source level: the changed condition would drop a signed thinking block solely because `record.thinking` is empty, while current docs and pi-ai source preserve signed empty thinking for encrypted OpenAI reasoning replay. I did not run a live provider reproduction.

## Solution Assessment

No, not as written: the narrow idea is plausible, but the implementation uses visible thinking text as the replayability gate and therefore removes a documented valid replay form. The safer fix is to inspect the signed reasoning payload and preserve encrypted replay state.
