## Summary

Keep open for maintainer review: current main still preserves signed whitespace-only reasoning before later assistant content, while this small PR cleanly narrows replay to meaningful thinking text or explicit Responses replay state. ## What This Changes

The PR changes the OpenAI Responses replay sanitizer to drop signed reasoning blocks with empty, whitespace-only, or non-string `thinking` unless the signature carries `content`, `encrypted_content`, or `summary` replay state, with focused regression tests.

## Best Possible Solution

Land this sanitizer after maintainer acceptance that empty or invalid visible reasoning without content, encrypted_content, or summary state is non-replayable; require live provider replay proof only if source-level proof is not enough for this compatibility-sensitive path.

## Reproduction Assessment

Yes for source reproduction: current main preserves signed whitespace-only reasoning when a following tool or text block exists, and the PR adds fixtures that show the changed sanitizer output. I did not run a live provider continuation.

## Solution Assessment

Yes. The sanitizer is the right boundary because OpenAI Responses replay history already centralizes reasoning cleanup before transport assembly, and the PR preserves encrypted/content/summary replay state instead of dropping all empty visible thinking.
