## What This Changes

The PR changes OpenAI Responses reasoning-block replay sanitization to drop signed `thinking` blocks whose persisted `thinking` text is empty, whitespace-only, or non-string, with focused sanitizer tests.

## Best Possible Solution

Revise the sanitizer to drop only truly non-replayable empty reasoning while preserving parsed signed payloads with replayable OpenAI state such as `encrypted_content`, then add regression coverage and redacted real runtime proof.

## Reproduction Assessment

Yes, at source level: the changed condition would drop a signed thinking block solely because `record.thinking` is empty, even when `thinkingSignature` carries replayable encrypted OpenAI state. I did not run a live provider reproduction.

## Solution Assessment

No, not as written: the narrow idea is plausible, but visible `thinking` text is the wrong replayability gate for encrypted empty-summary reasoning. The safer fix is to inspect the parsed signed reasoning payload and preserve replayable provider state.
