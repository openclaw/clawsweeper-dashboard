## What This Changes

The PR changes the OpenAI Responses replay sanitizer and regression tests so empty or invalid signed reasoning blocks are dropped unless their signature contains replayable Responses state.

## Best Possible Solution

Land the focused sanitizer after maintainer acceptance that empty or invalid visible reasoning without replay state is non-replayable, while encrypted/content/summary reasoning state remains preserved.

## Reproduction Assessment

## Solution Assessment

Yes, with a compatibility caveat. The sanitizer is the narrow owner boundary because replay history already centralizes reasoning and tool-call cleanup there, and the PR avoids over-dropping by preserving encrypted/content/summary replay state.
