## Summary

Keep open for maintainer review. The patch is a focused, source-supported bug fix with sufficient terminal proof, but it intentionally changes persisted OpenAI Responses replay/session behavior and current main still has the old preservation behavior.

## What This Changes

The PR updates the OpenAI Responses replay sanitizer to drop signed reasoning blocks with empty, whitespace-only, or non-string `thinking` unless the signature carries replayable `content`, `encrypted_content`, or `summary`, and adds focused sanitizer tests.

## Best Possible Solution

Land the sanitizer after maintainer acceptance that empty or invalid visible reasoning without replay state is non-replayable, while encrypted/content/summary reasoning state remains preserved.

## Reproduction Assessment

Yes. Source inspection shows current main preserves signed whitespace-only reasoning when a following tool or text block exists, and the PR adds focused fixtures plus terminal proof for the empty, non-string, and replayable-state cases.

## Solution Assessment

Yes, with maintainer acceptance of the compatibility risk. The sanitizer is the right boundary because OpenAI Responses replay history already centralizes reasoning and tool-call cleanup before transport assembly; requiring live provider continuation is the safer proof alternative, not a different code fix.
