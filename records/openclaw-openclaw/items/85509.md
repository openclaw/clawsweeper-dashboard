## What This Changes

The PR updates the OpenAI Responses history sanitizer and regression tests to drop signed reasoning blocks whose `thinking` text is empty or non-string unless the signature includes replayable Responses state.

## Best Possible Solution

Land the focused sanitizer after maintainer acceptance that source-level proof is enough for this compatibility-sensitive replay path, or require a live OpenAI/Azure continuation proof first.

## Reproduction Assessment

## Solution Assessment

Yes, with a compatibility caveat. The sanitizer is the narrow owner boundary before Responses payload conversion, and the PR avoids blanket dropping by preserving content, encrypted_content, and summary replay state.
