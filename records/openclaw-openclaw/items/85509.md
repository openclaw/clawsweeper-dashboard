## What This Changes

The PR changes the OpenAI Responses replay sanitizer to drop signed reasoning blocks with empty, whitespace-only, or non-string visible thinking unless the stored reasoning item has replayable content, encrypted_content, or summary state.

## Best Possible Solution

Land the sanitizer after maintainer acceptance that empty or invalid visible reasoning without replay state is non-replayable, while encrypted/content/summary reasoning state remains preserved.

## Reproduction Assessment

Yes. Source inspection shows current main preserves signed whitespace-only reasoning when a following tool or text block exists, and the PR adds focused fixtures for the empty, non-string, and replayable-state cases.

## Solution Assessment

Yes. The sanitizer is the right boundary because OpenAI Responses replay history already centralizes reasoning and tool-call cleanup before transport assembly; the PR also preserves replayable encrypted/content/summary state instead of dropping all empty visible thinking.
