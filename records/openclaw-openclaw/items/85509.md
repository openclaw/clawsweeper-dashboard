## Summary

Keep open: the focused sanitizer bugfix idea is plausible and the PR now includes source-level proof, but the patch would also drop valid OpenAI Responses reasoning replay state when visible `thinking` text is empty and `thinkingSignature` carries the reusable reasoning item.

## What This Changes

This PR changes OpenAI Responses transcript sanitization to remove signed reasoning `thinking` blocks with empty, whitespace-only, or non-string visible text and adds focused sanitizer tests.

## Best Possible Solution

Keep the sanitizer narrow: drop only signed empty reasoning blocks that lack replayable Responses state, preserve encrypted/content/summary reasoning signatures, and cover both valid and invalid empty-text cases in tests.

## Reproduction Assessment

Yes, at source level: pi-ai can persist a Responses reasoning block with `thinking: ""` while storing the full replay item in `thinkingSignature`, and the PR condition would drop it solely because visible text is empty. The claimed provider rejection path still lacks live provider proof.

## Solution Assessment

No, not as written: visible thinking text is the wrong replayability gate for OpenAI Responses items. The safer fix is to inspect the parsed signature and preserve replayable encrypted/content/summary state while dropping only truly non-replayable empty blocks.
