## Summary

Keep open for changes. The patch is aimed at the right user report, but it changes the host send cleanup instead of the composer replay path that can reintroduce the submitted text, and it removes a guard that protects newer composer state.

## What This Changes

The PR changes `clearSubmittedComposerState` so the Control UI host draft and attachments are always cleared after a send.

## Best Possible Solution

Keep the send-side match guard and add a focused stale-replay guard at the Control UI composer draft mirror or draft-change boundary, with regression tests and real browser proof.

## Reproduction Assessment

Yes, at source level. Current main still lets a later composer input event mirror the submitted textarea value back into draft state after send-side clearing, and the linked issue describes that visible Control UI symptom.

## Solution Assessment

No. This is not the best fix because it changes host cleanup while the stale replay happens in the composer mirror/draft-change path, and it removes a useful submitted-state guard.
