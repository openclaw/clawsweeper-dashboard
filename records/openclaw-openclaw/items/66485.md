## Summary

Keep this PR open: current main already has the `/usage` context-percentage alignment with safer freshness semantics, but it still lacks a scoped compaction-complete chat-history refresh. The proposed branch is not merge-ready because it would hide `/usage` context for session rows missing `totalTokensFresh` and would reload the active chat for sessionless compaction events without proving ownership.

## What This Changes

The PR changes Control UI slash-command usage percentage calculation and reloads chat history after completed compaction agent events.

## Best Possible Solution

Rebase or replace with a narrow Control UI patch that keeps current `/usage` semantics, adds a compaction-complete history refresh only when the event matches the active session or active run, and covers matching, unrelated, and missing-`sessionKey` event cases.

## Reproduction Assessment

Yes. A static reproduction exists for the PR's `/usage` regression with valid `totalTokens` and omitted `totalTokensFresh`, and an event-path reproduction exists for a sessionless completed compaction event triggering a visible-chat reload.

## Solution Assessment

No. The direction is close, but the best fix is a scoped reload using the same session/run ownership discipline already used in Control UI event handling, not treating missing `sessionKey` as sufficient.
