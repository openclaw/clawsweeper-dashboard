## Summary

This PR should stay open because the useful direction is real, but the branch regresses current `/usage` freshness semantics, reloads chat history for sessionless compaction events, and has no after-fix real behavior proof.

## What This Changes

The PR changes Control UI `/usage` context percentage calculation and adds an agent compaction-complete handler that reloads chat history.

## Best Possible Solution

Keep current main's `/usage` snapshot semantics, and if compaction history still needs immediate refresh, add an active-session/run-scoped refresh path with tests for matching, unrelated, and missing-sessionKey cases.

## Reproduction Assessment

Yes. Source inspection reproduces both issues: the PR guard drops context for rows with `totalTokens` but missing `totalTokensFresh`, and the compaction handler reloads on sessionless compaction events.

## Solution Assessment

No. The `/usage` direction is already implemented more compatibly on main, and the compaction refresh needs ownership scoping instead of accepting all sessionless events.

## Review Findings

- **[P2] Handle missing freshness as usable in /usage:** `ui/src/ui/chat/slash-command-executor.ts:384-387`
  - body: The new guard requires `totalTokensFresh === true`, but current UI semantics treat only explicit `false` as stale. Older or compatibility-shaped session rows with valid `totalTokens` and no freshness flag would lose the context percentage even though current main and the chat notice display it.
  - confidence: 0.92
