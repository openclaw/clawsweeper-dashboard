## Summary

Keep open: current main and the latest release still reject the reported MSYS2/MINGW `install.sh` path and still provide bash/zsh-only PATH persistence guidance, so this remains a Windows support-boundary feature decision rather than cleanup.

## What This Changes

Support running the Unix installer in Windows 11 MSYS2/MINGW and add fish-shell-aware PATH persistence or guidance.

## Best Possible Solution

Maintain one explicit Windows Unix-shell policy: either support MSYS2/MINGW in the installer with fish-aware PATH handling, docs, tests, and real Windows smoke proof, or document PowerShell/WSL2 as the supported Windows install paths.

## Reproduction Assessment

Yes, at source level: current `install.sh` accepts only Darwin, Linux, or WSL and sends MSYS/MINGW-style `OSTYPE` values to the unsupported-OS branch; the fish PATH concern is also visible because only bash/zsh files and syntax are handled.

## Solution Assessment

Unclear until maintainers decide the support boundary. The maintainable solution is either a tested MSYS2/MINGW plus fish implementation in the installer/docs/test surface, or explicit documentation that Windows users should use PowerShell or WSL2 instead.
