## Summary

Keep open. Current main still rejects MSYS2/MINGW in `install.sh` and still gives bash/zsh-only PATH persistence and warning guidance; the related Microsoft tracker preserves the work but does not implement it.

## What This Changes

Request support for running the Unix installer in Windows 11 MSYS2/MINGW and for Fish-compatible PATH persistence and guidance.

## Best Possible Solution

Define the Windows Unix-like shell support policy, then either implement tested MSYS2/MINGW installer handling with Fish-aware PATH persistence and docs, or document PowerShell/WSL2 as the supported Windows routes.

## Reproduction Assessment

Yes for the feature gap: source inspection and a read-only `detect_os_or_die` call show MSYS2/MINGW still exits through the unsupported `install.sh` path, and Fish PATH handling is absent. This is source-reproducible rather than a broken documented-behavior repro because MSYS2/Fish installer support is not currently in the support contract.

## Solution Assessment

No for the proposed workaround. Removing the unsupported-OS guard would leave `OS=unknown` behavior without MSYS-safe install decisions, Fish-specific PATH syntax, docs, and Windows smoke coverage.
