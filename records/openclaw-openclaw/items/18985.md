## Summary

Keep open. Current main and the latest release still keep `install.sh` scoped to macOS/Linux/WSL and still provide bash/zsh-only PATH persistence and warning text, while the related Microsoft tracker only tracks the work rather than implementing it.

## What This Changes

Request support for installing and running OpenClaw from Windows 11 MSYS2/MINGW-style Unix shells, plus fish-shell PATH guidance.

## Best Possible Solution

Maintainers should either approve tested MSYS2/MINGW plus fish installer support with docs and Windows smoke proof, or document that Windows Unix-like shells should use WSL2 or the PowerShell installer instead.

## Reproduction Assessment

Yes, source-level reproduction is high-confidence: current `detect_os_or_die` exits for non-Darwin, non-Linux, non-WSL shells, and the PATH warning still emits bash/zsh-only guidance. I did not run a live Windows 11 MSYS2 smoke test in this read-only review.

## Solution Assessment

No, the proposed guard removal is not the best complete fix by itself. The maintainable path needs explicit MSYS2/MINGW detection, Windows-safe install decisions, fish-aware PATH handling or guidance, docs, and real Windows smoke coverage, or a deliberate unsupported-policy docs update.
