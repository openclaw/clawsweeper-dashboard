## Summary

Keep open. Current main still rejects the reported MSYS2/MINGW `install.sh` path and still emits bash/zsh-only PATH guidance, while the open Microsoft tracker only lists this work rather than implementing it.

## What This Changes

Request to support `install.sh` and OpenClaw under Windows 11 MSYS2/MINGW and add fish-specific PATH persistence or guidance.

## Best Possible Solution

Maintainers should either approve and implement tested MSYS2/MINGW plus fish PATH support in the installer/docs, or explicitly document that Windows Unix-like environments should use WSL2 or `install.ps1`.

## Reproduction Assessment

Yes, source-level reproduction is high-confidence: `detect_os_or_die` exits for `OSTYPE=msys` and `OSTYPE=mingw64`, and the PATH warning still prints bash/zsh-only guidance. I did not run a real Windows 11 MSYS2 smoke test.

## Solution Assessment

No, the proposed guard removal is not the best complete fix by itself. The maintainable path needs explicit MSYS2/MINGW detection, Windows-safe install behavior, fish-aware PATH handling or guidance, docs, and real Windows smoke coverage, or a deliberate unsupported-policy doc update.
