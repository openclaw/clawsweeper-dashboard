## Summary

Keep open. Current main still excludes the reported MSYS2/MINGW install path from `install.sh`, still gives bash/zsh-only PATH persistence guidance, and the related Microsoft tracker is open tracking platform work rather than implementing this request.

## What This Changes

Request support for installing and running OpenClaw from a Windows 11 MSYS2/MINGW Unix-like environment, including fish shell PATH guidance.

## Best Possible Solution

Keep this open until maintainers either explicitly support MSYS2/MINGW in `scripts/install.sh` with Windows/MSYS-safe install behavior, fish-aware PATH handling, docs, and smoke coverage, or document that Windows Unix-like environments should use WSL2 or the PowerShell installer.

## Reproduction Assessment

Yes. The deterministic source path is clear: under `OSTYPE=msys` or MINGW-like environments, `detect_os_or_die` leaves `OS=unknown` and exits before install work; the PATH helpers and warning text also deterministically omit fish.

## Solution Assessment

No for the proposed workaround. Commenting out the unsupported-OS guard proves the npm path can advance, but it bypasses the platform contract and does not handle Windows/MSYS safety, fish PATH persistence, docs, or smoke coverage.
