Keep open. Current main still treats MSYS2/MINGW as outside the supported install.sh OS contract, Fish PATH persistence is still absent, and the open Microsoft tracker preserves this as remaining Windows platform work rather than superseding it.

The request asks OpenClaw to support installing and running from Windows 11 MSYS2/MINGW Unix-like shells and to provide Fish shell PATH setup guidance.

## Best Possible Solution

Define the Windows Unix-like shell policy, then either implement tested MSYS2/MINGW install.sh support with Fish-aware PATH persistence and docs, or document PowerShell/WSL2 as the supported Windows routes.

## Solution Assessment

No for the proposed workaround. Removing the unsupported-OS guard would leave OS=unknown behavior without MSYS-safe install decisions, Fish-specific PATH syntax, docs, and Windows smoke coverage.
