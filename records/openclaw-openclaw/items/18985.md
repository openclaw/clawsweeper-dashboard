Keep open. Current main and the latest release still route `install.sh` to macOS/Linux/WSL only, still lack Fish-specific PATH persistence/guidance, and the open Microsoft tracker records this as remaining Windows-platform work rather than a fix.

Request support for running the Unix installer in Windows 11 MSYS2/MINGW and for Fish shell PATH persistence or guidance.

## Best Possible Solution

Define the Windows Unix-like shell policy, then either implement tested MSYS2/MINGW `install.sh` support with Fish-aware PATH handling and docs, or document PowerShell/WSL2 as the supported Windows routes.
