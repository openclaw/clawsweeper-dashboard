## Summary

Keep open. Current main still excludes MSYS2/MINGW from the bash installer and still provides bash/zsh-only PATH persistence and warning guidance, while the related Microsoft tracker is coordination rather than an implementation.

## What This Changes

The issue requests Windows 11 MSYS2/MINGW support for `install.sh` and Fish shell PATH configuration guidance.

## Best Possible Solution

Define the Windows Unix-like shell support policy, then either add tested MSYS2/MINGW installer handling with Fish-aware PATH persistence and docs, or explicitly document WSL2/PowerShell as the supported Windows paths.

## Reproduction Assessment

Yes for the requested feature gap: current-main source and a read-only `detect_os_or_die` invocation show MSYS/MINGW remains unsupported, while Fish PATH handling is absent. This is not a broken existing contract, so it is source-reproducible rather than a reproduced bug.

## Solution Assessment

No for the proposed workaround. Removing the unsupported-OS guard would bypass the platform contract without MSYS-safe install behavior, Fish PATH syntax, docs, or Windows smoke coverage.
