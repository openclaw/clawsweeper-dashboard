## Summary

Keep open. Current main and latest release still reject MSYS/MINGW in the bash installer and still provide bash/zsh-only PATH persistence and warning guidance, so the remaining decision is whether MSYS2/MINGW and fish become supported installer targets.

## What This Changes

Support running the bash installer under Windows 11 MSYS2/MINGW and add fish-shell PATH handling or guidance.

## Best Possible Solution

Maintainers should either explicitly support MSYS2/MINGW with tested OS detection, Windows-safe install behavior, fish-aware PATH handling, docs, and smoke coverage, or document PowerShell/WSL2 as the supported Windows routes.

No. Commenting out the platform guard is not the best solution because it broadens the Windows support contract without platform-safe install behavior, fish handling, docs, or Windows smoke proof.
