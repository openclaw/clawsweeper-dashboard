## Summary

Keep open. Current main and v2026.6.5 still reject the MSYS2/MINGW `install.sh` path, Fish PATH handling remains bash/zsh-only, and the related Microsoft tracker is still tracking work rather than implementing this support.

## What This Changes

The issue requests Windows 11 MSYS2/MINGW support for the Unix installer and Fish-aware PATH persistence or guidance.

## Best Possible Solution

Define the Windows Unix-like shell policy, then either implement tested MSYS2/MINGW `install.sh` support with Fish-aware PATH handling and docs, or explicitly document PowerShell/WSL2 as the supported Windows routes.

## Reproduction Assessment

Yes for the source-level feature gap: current `detect_os_or_die` exits under `OSTYPE=msys MSYSTEM=MINGW64`, and installer PATH handling remains bash/zsh-only. No live Windows 11 MSYS2 smoke was run in this read-only review.

## Solution Assessment

No for the proposed workaround. Commenting out the unsupported-OS guard would leave `OS=unknown` install behavior without MSYS-safe install decisions, Fish syntax, docs, or Windows smoke coverage.
