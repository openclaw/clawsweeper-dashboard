## Summary

Keep open. Current main and the latest release still reject the reported MSYS2/MINGW `install.sh` path and still provide bash/zsh-only PATH persistence and warning guidance; the remaining blocker is the maintainer decision about whether MSYS2/MINGW becomes a supported Windows installer target.

## What This Changes

The issue requests support for running the Unix installer in Windows 11 MSYS2/MINGW and fish-aware PATH guidance.

## Best Possible Solution

Keep this open until maintainers decide whether MSYS2/MINGW is supported, then either implement tested installer/fish/docs support or document PowerShell/WSL2 as the supported Windows paths.

## Reproduction Assessment

Yes. Source inspection and a read-only shell check show `OSTYPE=msys` still exits through the unsupported OS branch while `OSTYPE=linux-gnu` succeeds; I did not run a real Windows MSYS2 smoke in this review.

## Solution Assessment

No automatic fix should choose the final behavior yet. Supporting MSYS2/MINGW is a platform-support decision, and the maintainable solution needs explicit installer logic, fish-specific PATH handling, docs, tests, and real Windows proof if approved.
