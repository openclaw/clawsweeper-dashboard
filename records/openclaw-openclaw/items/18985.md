## Summary

Keep open: current main and the latest release still route Windows users away from the Unix shell installer, and fish-specific PATH persistence remains missing, so the central request is not implemented and still needs a maintainer support-boundary decision.

## What This Changes

The issue requests support for installing and running OpenClaw from Windows 11 MSYS2/MINGW and fish shell environments.

## Best Possible Solution

Maintain one explicit Windows Unix-shell policy: either support MSYS2/MINGW in the installer with fish-aware PATH handling, docs, tests, and Windows smoke proof, or document PowerShell/WSL2 as the supported Windows paths.

## Reproduction Assessment

Yes, at source level: current install.sh sends MSYS/MINGW-style OSTYPE values to the unsupported-OS branch, and the fish PATH concern is visible in the bash/zsh-only warning. I did not run a real Windows 11 MSYS2 smoke test in this read-only review.

## Solution Assessment

Unclear until maintainers decide the support boundary. The maintainable solution is either tested MSYS2/MINGW plus fish support in the installer/docs/test surface, or explicit documentation that Windows users should use PowerShell or WSL2 instead.
