## Summary

Keep open. Current main still excludes MSYS2/MINGW from `install.sh`, Fish PATH persistence/guidance is still absent, and the related Microsoft tracker is an open coordination PR rather than an implementation.

## What This Changes

Request support for installing and running OpenClaw from Windows 11 MSYS2/MINGW Unix-like shells, including Fish shell PATH guidance.

## Best Possible Solution

Define the Windows Unix-like shell support policy, then either implement tested MSYS2/MINGW installer handling with Fish-aware PATH persistence and docs, or document WSL2/PowerShell as the supported Windows paths.

## Reproduction Assessment

Yes for the requested feature gap: current-main source and a read-only `detect_os_or_die` invocation show MSYS/MINGW remains unsupported, while Fish PATH handling is absent. This is not a broken existing contract; it reproduces the missing requested support.

## Solution Assessment

No for the proposed workaround. Removing the unsupported-OS guard would bypass the platform contract without MSYS-safe install behavior, Fish PATH syntax, docs, or Windows smoke coverage.
