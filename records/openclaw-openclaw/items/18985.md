## Summary

Keep open: current main and the latest release still route Windows native installs to PowerShell/WSL2, reject MSYS2/MINGW-style `install.sh` detection, and give bash/zsh-only PATH guidance, so this remains a concrete Windows shell support-boundary decision.

## What This Changes

Request support for running the Unix installer and OpenClaw under Windows 11 MSYS2/MINGW with Fish-aware PATH setup guidance.

## Best Possible Solution

Maintain one explicit Windows Unix-shell policy: either support MSYS2/MINGW in the installer with fish-aware PATH handling, docs, tests, and real Windows smoke proof, or document PowerShell/WSL2 as the supported Windows paths.

## Reproduction Assessment

Yes, at source level: current `install.sh` accepts only Darwin, Linux, or WSL and sends MSYS/MINGW-style `OSTYPE` values to the unsupported-OS branch; the Fish PATH concern is also visible in the bash/zsh-only warning.

## Solution Assessment

Unclear until maintainers decide the support boundary. The maintainable solution is either a tested MSYS2/MINGW plus Fish implementation in the installer/docs/test surface, or explicit documentation that Windows users should use PowerShell or WSL2 instead.
