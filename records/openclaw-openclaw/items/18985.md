## Summary

Keep open. Current main still rejects the reported MSYS2/MINGW `install.sh` path and still gives bash/zsh-only PATH persistence guidance, while the related Microsoft tracker remains an open coordination item rather than an implementation.

## What This Changes

Support running the Unix installer and Fish shell PATH setup guidance in Windows 11 MSYS2-like environments.

## Best Possible Solution

Define the support boundary, then either implement tested MSYS2/MINGW installer handling with fish-aware PATH persistence and docs, or document that Windows Unix-like environments should use WSL2 or the PowerShell installer.

## Reproduction Assessment

Yes. The deterministic source path is clear: under `OSTYPE=msys` with `MSYSTEM=MINGW64`, `detect_os_or_die` exits before install work, and the PATH helpers/warning text omit fish.

## Solution Assessment

No for the proposed workaround. Commenting out the unsupported-OS guard proves the npm path can advance, but it bypasses the platform contract and does not cover Windows/MSYS safety, fish PATH persistence, docs, or smoke coverage.
