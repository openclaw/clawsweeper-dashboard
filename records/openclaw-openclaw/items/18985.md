## Summary

Keep open. Current main still treats MSYS2/MINGW as unsupported for `install.sh`, still gives bash/zsh-only PATH persistence guidance, and the related Microsoft tracker is open tracking platform work rather than implementing this support.

## What This Changes

Support installing and running OpenClaw from Windows 11 MSYS2/MINGW with fish-aware PATH setup in the Unix installer.

## Best Possible Solution

Maintainers should either explicitly support MSYS2/MINGW with installer detection, Windows-safe install behavior, fish PATH handling, docs, and smoke coverage, or document PowerShell/WSL2 as the supported Windows paths.

## Reproduction Assessment

Yes for the observable current behavior: the source path and a read-only invocation show `OSTYPE=msys` exits as unsupported, and PATH helpers omit fish. This is still a feature request because current docs do not define MSYS2/MINGW as a supported `install.sh` target.

## Solution Assessment

No for the proposed workaround. Commenting out the guard proves the npm path can advance, but the maintainable fix needs an explicit Windows/MSYS support decision, safe installer behavior, fish-specific PATH handling, docs, and smoke coverage.
