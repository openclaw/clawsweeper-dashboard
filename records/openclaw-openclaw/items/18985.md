## Summary

Keep open: current main and the latest release still reject the reported MSYS2/MINGW `install.sh` path and still give bash/zsh-only PATH persistence guidance, so the remaining question is a Windows support-boundary decision rather than cleanup.

## What This Changes

Support installing and running OpenClaw from Windows 11 MSYS2/MINGW with `install.sh`, and add Fish shell-aware PATH guidance or persistence.

## Best Possible Solution

Maintain one explicit Windows Unix-shell policy: either support MSYS2/MINGW in the installer with fish-aware PATH handling, docs, tests, and real Windows smoke proof, or document PowerShell/WSL2 as the supported Windows install paths.

## Reproduction Assessment

Yes, at source level: current `install.sh` accepts only Darwin, Linux, or WSL and sends MSYS/MINGW-style `OSTYPE` values to the unsupported-OS branch; the Fish PATH concern is also visible because only bash/zsh files and syntax are handled. I did not run a real Windows MSYS2 smoke in this read-only review.

## Solution Assessment

Unclear until maintainers decide the support boundary. The maintainable solution is either a tested MSYS2/MINGW plus Fish implementation in the installer/docs/test surface, or explicit documentation that Windows users should use PowerShell or WSL2 instead.
