## Summary

Keep open. Current main and the latest release still exclude MSYS2/MINGW from the supported Unix installer path, Fish-specific PATH handling is absent, and the related Microsoft tracker records this as remaining Windows-platform work rather than a fix.

## What This Changes

Request support for running the Unix installer in Windows 11 MSYS2/MINGW and add Fish-aware PATH persistence or guidance.

## Best Possible Solution

Define the Windows Unix-like shell policy, then either implement tested MSYS2/MINGW `install.sh` support with Fish-aware PATH handling and docs, or document PowerShell/WSL2 as the supported Windows routes.

## Reproduction Assessment

Yes for the feature gap: source inspection and a read-only source invocation show MSYS2/MINGW still falls through the unsupported `install.sh` branch, while Fish PATH persistence is absent. No live Windows smoke was run in this read-only review.

## Solution Assessment

No for the proposed workaround. Removing the unsupported-OS guard would leave `OS=unknown` install behavior without MSYS-safe decisions, Fish syntax, docs, and Windows smoke coverage.
