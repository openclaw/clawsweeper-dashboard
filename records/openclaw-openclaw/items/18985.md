## Summary

Keep open. Current main and the latest release still keep the Unix installer scoped to macOS/Linux/WSL, fish-specific PATH handling remains missing, and the open Microsoft tracker only tracks Windows platform work rather than implementing this support.

## What This Changes

The issue requests support for installing and running OpenClaw from Windows 11 MSYS2/MINGW and fish shell, including installer OS detection and PATH guidance.

## Best Possible Solution

Maintainers should either approve tested MSYS2/MINGW plus fish installer support with docs and Windows smoke proof, or document that Windows Unix-like shells should use WSL2 or the PowerShell installer instead.

## Reproduction Assessment

Yes. Source inspection on current main shows `detect_os_or_die` exits for non-Darwin, non-Linux, non-WSL shells, and the PATH warning still emits bash/zsh-only guidance; I did not run a live Windows 11 MSYS2 smoke in this read-only review.

## Solution Assessment

No. The proposed guard removal is not the best complete fix by itself; the maintainable path needs explicit MSYS2/MINGW detection, Windows-safe install decisions, fish-aware PATH handling or guidance, docs, and real Windows smoke coverage.
