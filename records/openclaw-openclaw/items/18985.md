## Summary

Keep open. Current main still treats MSYS2/MINGW as unsupported for `install.sh`, Fish-specific PATH persistence and warning guidance are still absent, and the linked Microsoft tracker is an open coordination PR rather than an implementation.

## What This Changes

Request support for running the Unix installer and OpenClaw under Windows 11 MSYS2/MINGW plus Fish shell PATH guidance.

## Best Possible Solution

Define the Windows Unix-like shell support policy, then either implement tested MSYS2/MINGW installer handling with Fish-aware PATH persistence and docs, or document WSL2/PowerShell as the supported Windows paths.

Yes for the feature gap: current-main source shows MSYS2/MINGW remains unsupported by `install.sh` and Fish PATH handling is absent. This is source-reproducible rather than a reproduced bug because the request adds a supported platform/shell mode.

## Solution Assessment

No for the proposed workaround. Removing the unsupported-OS guard is not the best fix without MSYS-safe install decisions, Fish-specific PATH syntax, docs, and Windows smoke coverage.
