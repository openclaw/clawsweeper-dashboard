## Summary

Keep open. Current main still treats MSYS2/MINGW as unsupported for `install.sh`, the installer PATH guidance remains bash/zsh-only, and the related Microsoft tracker #74163 is an open coordination item rather than a merged implementation.

## What This Changes

The issue requests support for installing and running OpenClaw from Windows 11 MSYS2 and Fish shell environments.

## Best Possible Solution

Define the Windows Unix-like shell support policy, then either implement tested MSYS2/MINGW installer handling with Fish-aware PATH persistence and docs or document WSL2/PowerShell as the supported Windows paths.

## Reproduction Assessment

Yes. Source inspection on current main shows `OSTYPE=msys`/MINGW would stay `OS=unknown` in `detect_os_or_die`, and the PATH helper text only covers bash/zsh.

## Solution Assessment

No for the proposed workaround. Removing the unsupported-OS guard would bypass the platform contract without covering MSYS-safe install behavior, Fish PATH syntax, docs, or smoke coverage.
