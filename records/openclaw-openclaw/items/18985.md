## Summary

Keep open. Current main and the latest shipped release still route Windows users to PowerShell or WSL2, while MSYS2/MINGW and Fish PATH handling remain unsupported installer behavior that needs an explicit Windows support-boundary decision.

## What This Changes

The issue requests support for running the Unix installer in Windows 11 MSYS2/MINGW and better Fish shell PATH setup guidance.

## Best Possible Solution

Define the Windows Unix-like shell policy, then either implement tested MSYS2/MINGW installer handling with Fish-aware PATH persistence and docs, or document PowerShell/WSL2 as the supported Windows routes.

## Reproduction Assessment

Yes for the feature gap: source inspection shows MSYS2/MINGW still falls through to the unsupported install.sh branch, and Fish PATH handling is absent. This is source-reproducible rather than a broken documented-behavior repro because MSYS2/Fish installer support is not in the current support contract.

## Solution Assessment

No for the proposed workaround. Removing the unsupported-OS guard would leave OS=unknown behavior without MSYS-safe install decisions, Fish-specific PATH syntax, docs, and Windows smoke coverage.
