## Summary

Keep open. Current main and v2026.5.20 still route Windows users away from `install.sh`, Fish-specific PATH persistence is still absent, and the open Microsoft tracker lists this as remaining Windows-platform work rather than a merged fix.

## What This Changes

Request support for installing and running OpenClaw from Windows 11 MSYS2/Unix-like shells and provide Fish shell PATH handling.

## Best Possible Solution

Define the Windows Unix-like shell policy, then either implement tested MSYS2/MINGW `install.sh` support with Fish-aware PATH handling and docs, or document PowerShell/WSL2 as the supported Windows routes.

## Reproduction Assessment

Yes for the source-level feature gap: current installer detection excludes MSYS2/MINGW, and PATH persistence/warnings remain bash/zsh-only. No live Windows 11 MSYS2 smoke was run in this read-only review.

## Solution Assessment

No for the proposed workaround. Removing the unsupported-OS guard would leave `OS=unknown` install behavior without MSYS-safe install decisions, Fish syntax, docs, or Windows smoke coverage.
