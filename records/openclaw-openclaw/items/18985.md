## Summary

Keep open. Current main still rejects the reported MSYS2/MINGW `install.sh` path, Fish-specific PATH persistence and warning guidance are absent, and the related Microsoft tracker at https://github.com/openclaw/openclaw/pull/74163 is open coordination rather than an implementation.

## What This Changes

Request support for running OpenClaw installation from Windows 11 MSYS2/MINGW Unix-like shells and Fish shell.

## Best Possible Solution

Define the Windows Unix-like shell support policy, then either implement tested MSYS2/MINGW installer handling with Fish-aware PATH persistence and docs, or document PowerShell/WSL2 as the supported Windows paths.

## Reproduction Assessment

Yes for the feature gap: source inspection and a no-run function check show MSYS2/MINGW still exits through the unsupported `install.sh` path, and Fish PATH handling is absent. This is source-reproducible rather than a broken existing-behavior repro because it asks to add a supported platform/shell mode.

## Solution Assessment

No for the proposed workaround. Removing the unsupported-OS guard is not the narrow maintainable fix without MSYS-safe install decisions, Fish-specific PATH syntax, docs, and Windows smoke coverage.
