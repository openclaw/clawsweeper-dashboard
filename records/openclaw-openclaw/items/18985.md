## Summary

Keep open. Current main still excludes the reported MSYS2/MINGW `install.sh` path, Fish installer PATH handling is still bash/zsh-only, and the related Microsoft tracker is open coordination rather than an implementation.

## What This Changes

The issue requests support for installing and running OpenClaw from Windows 11 MSYS2/MINGW and Fish shell environments.

## Best Possible Solution

Define the Windows Unix-like shell support policy, then either implement tested MSYS2/MINGW installer handling with Fish-aware PATH persistence and docs, or document PowerShell/WSL2 as the supported Windows routes.

## Reproduction Assessment

Yes for the feature gap: source inspection shows MSYS2/MINGW still exits through the unsupported `install.sh` path and Fish PATH handling is absent. This is source-reproducible rather than a broken existing-behavior repro because MSYS2/MINGW support is not currently a documented contract.

## Solution Assessment

No for the proposed workaround. Removing the unsupported-OS guard is not the narrow maintainable fix without MSYS-safe install decisions, Fish-specific PATH syntax, docs, and Windows smoke coverage.
