## Summary

Keep open. Current main still rejects the reported MSYS2/MINGW `install.sh` path, installer PATH persistence and warnings remain bash/zsh-only, and the related Microsoft tracker is open coordination rather than an implementation.

## What This Changes

The issue requests support for running the Unix installer under Windows 11 MSYS2/MINGW and Fish shell PATH guidance.

## Best Possible Solution

Define the Windows Unix-like shell support policy, then either implement tested MSYS2/MINGW installer handling with Fish-aware PATH persistence and docs or document WSL2/PowerShell as the supported Windows paths.

## Reproduction Assessment

Yes for the current requested gap: a read-only current-main source check shows `OSTYPE=msys`/MINGW exits as unsupported and the PATH warning remains bash/zsh-only. Because this is a feature/support-boundary request, that reproduces the missing support rather than a broken existing contract.

## Solution Assessment

No for the proposed workaround. Removing the unsupported-OS guard would bypass the platform contract without MSYS-safe install behavior, Fish PATH syntax, docs, or Windows smoke coverage.
