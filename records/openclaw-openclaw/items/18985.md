## Summary

Keep open. Current main still excludes MSYS2/MINGW from the supported `install.sh` OS path, still gives bash/zsh-only PATH persistence guidance, and the open Microsoft tracker preserves this as remaining Windows-platform work rather than implementing or superseding it.

## What This Changes

The issue requests Windows 11 MSYS2/MINGW support for `install.sh` plus Fish shell PATH persistence or guidance.

## Best Possible Solution

Define the Windows Unix-like shell policy, then either implement tested MSYS2/MINGW `install.sh` support with Fish-aware PATH persistence and docs, or document PowerShell/WSL2 as the supported Windows routes.

## Reproduction Assessment

Yes for the feature gap: source inspection and a read-only shell probe show MSYS2/MINGW still falls through to the unsupported `install.sh` branch, and Fish PATH persistence is absent. This is source-reproducible rather than a broken documented-behavior repro because MSYS2/Fish installer support is not currently part of the documented contract.

## Solution Assessment

No for the proposed workaround. Removing the unsupported-OS guard would leave `OS=unknown` behavior without MSYS-safe install decisions, Fish-specific PATH syntax, docs, and Windows smoke coverage.
