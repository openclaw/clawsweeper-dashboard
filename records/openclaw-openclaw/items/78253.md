## What This Changes

The branch adds `QQBOT_DATA_DIR` handling to QQBot data-path helpers, helper tests, and QQBot channel documentation.

## Best Possible Solution

Land a corrected QQBot-owned helper fix that trims and validates the override, aligns diagnostics with the effective data directory, keeps focused tests/docs in sync, and includes redacted real setup proof.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: set `QQBOT_DATA_DIR`, then diagnostics and persistent QQBot state still resolve through helpers that always use `getHomeDir()/.openclaw/qqbot`.

## Solution Assessment

No, not as submitted. Centralizing the override in the QQBot helper is the right direction, but the branch must reject blank overrides, update diagnostics behavior, and provide real behavior proof before merge.
