  - body: The linked report is about QQBot diagnostics still showing the problematic Windows home data path after `QQBOT_DATA_DIR` is set.   - confidence: 0.9
- **[P1] Keep legacy imports anchored to the OS-home path:** `extensions/qqbot/src/engine/utils/platform.ts:86`
  - body: On current main this helper is documented as the legacy `~/.openclaw/qqbot` import/remap root, and session, known-user, ref-index, and credential-backup migrations read from it. Pointing the helper at `QQBOT_DATA_DIR` means users who set the new env var during upgrade can skip existing OS-home JSON state/backups. Preserve a separate legacy OS-home path or add an explicit tested migration before switching the effective writable data root.
  - confidence: 0.87
- **[P2] Reject blank QQBOT_DATA_DIR before joining paths:** `extensions/qqbot/src/engine/utils/platform.ts:92-93`
  - body: `if (envDataDir)` accepts strings made only of spaces, then `normalizePath()` trims them to an empty string. That makes calls like `getQQBotDataDir("downloads")` create or read a relative `downloads` path under the Gateway process cwd. Trim first and treat blank/null-like values as unset or fail with a clear diagnostic.
  - confidence: 0.86
