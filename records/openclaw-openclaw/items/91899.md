## Summary

Keep open: current main and v2026.6.6 address the older one-shot Feishu reconnect failure, but the silent-health part remains because Feishu still does not publish connected or transport-activity fields into the channel runtime status. The open fix path is https://github.com/openclaw/openclaw/pull/90966, which targets that remaining gap but still needs maintainer review and stronger patched-runtime proof.

## What This Changes

The issue asks OpenClaw to prevent silent Feishu WebSocket disconnects by keeping reconnects alive and exposing transport liveness/status for health checks and logging.

## Best Possible Solution

Land a narrow Feishu plugin fix that publishes connected/disconnected state from SDK lifecycle callbacks and only publishes transport activity from real transport or heartbeat proof, then validate it with focused tests and redacted live Feishu status/health-monitor evidence.

## Reproduction Assessment

Yes for the remaining source-level health gap: Feishu publishes only port/configuration status while the gateway health policy needs connected and transport-activity fields to detect disconnected or stale sockets. No live Feishu WebSocket kill test was run in this read-only review.

## Solution Assessment

No close is appropriate yet: the persistent reconnect part is fixed, but the current implementation still does not solve the status/health-monitor detection gap. The best fix path is the existing Feishu plugin status-sink direction, corrected to use SDK lifecycle or true transport proof and backed by live evidence.
