Keep open: current main and v2026.6.6 address the older one-shot Feishu reconnect failure, but the silent-health part of this issue remains because Feishu still does not publish connected or transport-liveness fields for gateway status. ## What This Changes

Report a Feishu WebSocket reliability bug where dead connections and weak status health can cause silent inbound message loss.

## Best Possible Solution

Land a Feishu plugin-owned transport-status fix that publishes connected/disconnected state from SDK lifecycle or heartbeat-backed proof, then validate it with focused tests plus redacted live Feishu status and health-monitor evidence.

## Solution Assessment

Yes, keeping this issue open is the best outcome until the related Feishu transport-status PR or an equivalent fix lands. The implementation should use SDK lifecycle or true transport activity rather than app-event timestamps as socket liveness.
