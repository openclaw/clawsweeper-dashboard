## Summary

Keep open: the one-shot Feishu WebSocket reconnect problem is already fixed and shipped, but current main and v2026.6.6 still do not publish Feishu transport liveness into channel status, so the gateway health monitor cannot reliably detect the dead-socket state reported here. The related open PR targets that remaining gap, but it still has proof and transport-status semantics blockers.

## What This Changes

The issue asks OpenClaw to recover from silent Feishu WebSocket disconnects and make status/health checks detect dead Feishu transport state.

## Best Possible Solution

Ship a Feishu plugin-owned transport-status fix that feeds real connected/disconnected and transport-activity state into the existing gateway health monitor while keeping the shipped reconnect supervisor.

## Reproduction Assessment

Yes for the status/health-monitor gap by source inspection: Feishu only publishes port status, while the gateway stale-socket policy needs connected plus transport-activity fields. I did not live-reproduce the reported Feishu WebSocket drop.

## Solution Assessment

Yes to keeping this open; current main only solves the reconnect-supervisor part. The best fix is the Feishu-owned status/liveness path, likely by repairing or replacing the related open PR rather than closing the issue now.
