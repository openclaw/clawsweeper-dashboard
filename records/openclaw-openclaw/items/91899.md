## Summary

The issue remains necessary: v2026.6.5 and current main already retry terminal Feishu SDK failures indefinitely, but both still omit transport liveness from account status, so gateway health cannot detect the reported silently dead connection. The paired open PR targets that gap but is not yet a safe resolution because it lacks real Feishu proof and appears to use inbound message timestamps as transport activity.

## What This Changes

Report silent Feishu WebSocket message loss where reconnect supervision does not recover an undetected dead transport and gateway status remains healthy.

## Best Possible Solution

Preserve the shipped reconnect supervisor and make Feishu publish SDK-proven connected, disconnected, and heartbeat activity through the existing account status sink, with gateway-policy tests and real Feishu idle/disconnect proof.

## Reproduction Assessment

Yes for the status blind spot: current source and tests show Feishu supplies no transport liveness and gateway health skips stale recovery without it. No live reproduction yet proves the exact v2026.6.5 silent socket failure.

## Solution Assessment

No, the issue's full suggested list is broader than the best fix because indefinite backoff supervision already shipped; the narrow maintainable solution is genuine SDK transport-state publication, not inactivity inferred only from inbound messages.
