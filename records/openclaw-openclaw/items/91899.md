## Summary

Keep open: the latest release and current main have the persistent Feishu WebSocket reconnect loop, but the status/health-monitor gap remains because Feishu still does not publish connected or transport-liveness fields. The related open fix candidate https://github.com/openclaw/openclaw/pull/90966 targets the remaining gap but has not landed.

## What This Changes

The issue reports Feishu WebSocket silent disconnects that lead to missed inbound messages, with reconnect and status/health checks not surfacing the dead transport.

## Best Possible Solution

Land a Feishu plugin-owned transport-status fix that publishes connected/disconnected and last transport activity from true SDK, heartbeat, or transport-liveness signals, with focused tests and redacted runtime proof, then close this issue.

## Reproduction Assessment

Yes for the remaining status/health-monitor gap by source inspection: Feishu still publishes only port status while the gateway health policy needs connected and transport-activity fields. I did not live-reproduce the reported Feishu WebSocket outage.

## Solution Assessment

Yes, keeping this issue open is the best current outcome because the remaining fix is targeted by an open PR but not landed. The implementation should stay in the Feishu plugin boundary and use true transport or SDK lifecycle signals rather than broad core fallback behavior.
