## What This Changes

The issue reports Feishu WebSocket silent disconnects, one-shot reconnect behavior, stale status output, and missing logging/liveness checks after prolonged message loss.

## Best Possible Solution

Ship a Feishu plugin-owned transport-status fix that reports connected/disconnected state from real WebSocket lifecycle signals and only reports transport activity from heartbeat, SDK lifecycle, or equivalent transport proof.

## Reproduction Assessment

Yes for the status/health-monitor gap by source inspection: Feishu only publishes port status, while the gateway health policy needs connected and transport-activity fields. No live Feishu WebSocket drop was reproduced in this read-only review.

## Solution Assessment

No close yet: the current reconnect loop is the right shipped base, but the status/liveness part still needs a Feishu-owned fix and maintainer review of the exact transport-health semantics.
