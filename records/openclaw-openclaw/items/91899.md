## Summary

Keep open: current main and the latest release cover the older Feishu SDK retry-exhaustion reconnect failure, but Feishu still does not publish connected or transport-activity status for the gateway health monitor. The open candidate https://github.com/openclaw/openclaw/pull/90966 targets the remaining gap, but it has not landed and still needs maintainer liveness-semantics review plus patched-runtime proof.

## What This Changes

The issue reports Feishu WebSocket silent disconnects, weak reconnect/status detection, missing liveness logging, and hours of missed inbound messages.

## Best Possible Solution

Land a Feishu plugin-owned status/liveness fix that publishes connected state and transport activity from true SDK readiness, reconnect, heartbeat, or poll signals, then close this issue after focused tests and redacted runtime evidence.

## Reproduction Assessment

Source-reproducible for the remaining status/health gap: current main Feishu startup publishes only port status, while gateway health needs connected and transport-activity fields. I did not live-reproduce the reported Feishu WebSocket outage.

## Solution Assessment

Yes, keeping this open and routing through the existing Feishu plugin status-sink path is the best current shape. The older reconnect loop is already shipped, but the remaining fix needs true Feishu transport-liveness signals rather than broad core fallback behavior.
