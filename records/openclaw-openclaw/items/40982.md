## Summary

Keep open: current main and the latest release only partially mitigate the watchdog cap. Explicit cron/configured timeout resumed runs can now escape the 180s resume default, but ordinary inherited resumed CLI runs still resolve to 180s, and the broader default/config fix is still an open conflicting PR.

## What This Changes

The issue requests raising or removing the 3-minute CLI no-output watchdog cap so long-running silent CLI requests are not killed prematurely.

## Best Possible Solution

Land a focused maintainer-approved watchdog fix that keeps explicit timeout safety, uses the existing backend watchdog config surface where possible, and allows legitimate long silent CLI runs without silently killing resumed sessions at 180s.
