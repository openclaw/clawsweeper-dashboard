## Summary

Keep open: current main still has the 180s resumed CLI no-output cap, and https://github.com/openclaw/openclaw/pull/41022 is the active unmerged closing PR rather than a reason to close this issue early.

## What This Changes

Raise or remove the CLI no-output watchdog's 3-minute cap so long-running CLI-backed requests can run past 180 seconds without being terminated.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/41022 for the default-cap mitigation, then handle Monitor-specific expected-idle behavior in https://github.com/openclaw/openclaw/issues/71803 if maintainers want that deeper contract.

## Reproduction Assessment

Yes. A resumed non-cron CLI backend run with default watchdog settings and no stdout or stderr for more than 180 seconds resolves to `noOutputTimeoutMs=180000` and follows the existing no-output termination path; I did not run a live three-minute CLI hang in this read-only review.

## Solution Assessment

Yes for the narrow mitigation: the linked PR's default-only increase is the smallest path to remove the hard 3-minute resumed cap while preserving the outer timeout guard. Full no-cap semantics and Monitor expected-idle behavior should stay separate because they change the watchdog contract more broadly.
