## Label Justifications

## Summary

Keep open: current main and the latest release still keep the ordinary resumed CLI no-output watchdog capped at 180 seconds, and the remaining fix needs maintainer timeout-policy review rather than cleanup closure.

## What This Changes

The issue requests raising, removing, or making configurable the hard 3-minute no-output watchdog cap for long-running CLI-backed agent sessions.

## Best Possible Solution

Land a maintainer-approved CLI watchdog policy that prevents false 180s aborts for healthy long-running sessions while keeping an intentional stuck-process guard and focused resolver coverage.

## Reproduction Assessment

Yes, at source level: current main still resolves an inherited resumed user CLI run with a 600s outer timeout to a 180s no-output watchdog, and the discussion includes matching live logs. I did not run a live 180-second stall in this read-only review.

## Solution Assessment

Unclear as an implementation direction: raising or disabling the cap is plausible, but the exact default, no-cap semantics, and state-gated behavior need maintainer policy review. The current open PR is not the best fix as submitted because it is conflicting and still needs proof and policy cleanup.
