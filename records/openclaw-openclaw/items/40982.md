## Summary

Keep open: current main and latest v2026.5.7 still keep the resumed CLI no-output watchdog capped at 180 seconds, and the unmerged linked PR with closing syntax remains the active implementation path.

## What This Changes

The issue asks to raise or remove the CLI no-output watchdog's 3-minute defaults and optionally let operators tune or disable the cap.

## Best Possible Solution

Review, land, or replace https://github.com/openclaw/openclaw/pull/41022 for the narrow default increase, then handle no-cap and Monitor expected-idle semantics as deliberate follow-up work.

## Reproduction Assessment

Yes. Source inspection shows a default resumed non-cron CLI run with a long global timeout resolves through CLI_RESUME_WATCHDOG_DEFAULTS.maxMs=180000 and then follows the existing no-output termination path; I did not run a live three-minute hang in this read-only review.

## Solution Assessment

Yes for the immediate mitigation: raising shared defaults is the smallest first step for this issue. Full disable/null semantics and Monitor-aware expected-idle handling should be separate maintainer decisions.
