## Label Justifications

- `P1`: The issue reports real CLI-backed agent turns being terminated mid-request after 180 seconds of valid silence, breaking active user workflows.
- `impact:session-state`: The failure kills the active CLI-backed agent session/turn and can leave users without the in-flight conversation state.

## Summary

Keep open: current main and the latest release still preserve the ordinary resumed-session 180s no-output cap, and the same-repo implementation path is still an open PR needing maintainer review rather than a merged fix.

## What This Changes

The issue requests raising or removing the 3-minute no-output watchdog cap for CLI requests so long silent agent runs are not terminated prematurely.

## Best Possible Solution

Keep this issue open while maintainers choose a bounded CLI watchdog policy and land a refreshed implementation with focused resolver tests plus real behavior proof for long silent CLI work.

## Reproduction Assessment

Yes from source: current main still resolves ordinary resumed CLI requests to the 180_000 ms resume cap, and the current regression test explicitly expects that behavior. I did not run a live long-silent CLI process, so this is source-reproducible rather than runtime-reproduced.

## Solution Assessment

No final fix is on main yet. Raising the default cap is a plausible fix, but the exact cap, no-cap semantics, and whether to bundle TUI watchdog config changes need maintainer policy and upgrade-safety review.
