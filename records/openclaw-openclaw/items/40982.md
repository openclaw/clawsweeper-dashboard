## Label Justifications

- `P1`: The issue has source and user-log evidence that long-running CLI agent requests can be terminated mid-session and surface as all-model failure.

## Summary

Keep open: current main and the latest release still preserve the 180-second resumed CLI no-output cap, and an open linked PR is still the active implementation path rather than a landed fix.

## What This Changes

The issue asks to raise, remove, or make configurable the 3-minute no-output watchdog cap for CLI requests so long-running silent agent work is not killed prematurely.

## Best Possible Solution

Keep this issue open while maintainers review the linked watchdog PR path and choose a default/config policy that allows legitimate long silent resumed CLI runs without removing runaway-process protection.

## Reproduction Assessment

Yes. Source inspection confirms the default resumed-session cap is still 180 seconds and the runtime uses the resolved no-output timeout to terminate silent CLI runs; I did not run a live 180-second CLI reproduction in this read-only review.

## Solution Assessment

Unclear. Raising the default cap is a plausible mitigation, but removing or disabling the watchdog is a product/default decision; the existing backend watchdog config surface may cover per-backend tuning without adding a new key.
