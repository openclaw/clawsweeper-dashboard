## Summary

Keep open: current main still clamps resumed CLI no-output watchdogs to 180 seconds, and an open PR with closing syntax is already tracking the narrow fix.

## What This Changes

Request raises or removes the CLI no-output watchdog cap so long-running resumed CLI requests are not terminated at about 180 seconds.

## Best Possible Solution

Review and land #41022 or a narrow replacement, then handle any explicit no-cap or Monitor-aware watchdog behavior as a separate maintainer decision if needed.

## Reproduction Assessment

Yes, source-reproducible: a resumed CLI backend with a global timeout above 180 seconds still resolves to a 180-second no-output timeout, and the runner terminates on that watchdog. I did not run a live long-idle CLI session in this read-only review.

## Solution Assessment

Partly: raising the defaults as in #41022 is the narrowest immediate relief for the reported cap, but removing or disabling the watchdog is a separate product-policy decision.
