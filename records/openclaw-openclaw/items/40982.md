## Summary

Keep open: current main and latest release still preserve the ordinary resumed CLI no-output cap at 180 seconds, and the active implementation path is still an unmerged, dirty, proof-blocked PR rather than a completed fix.

## What This Changes

The issue requests raising, removing, or making configurable the 3-minute CLI no-output watchdog cap that terminates long silent CLI-backed agent requests.

## Best Possible Solution

Keep this issue as the canonical watchdog policy tracker until maintainers either refresh and land https://github.com/openclaw/openclaw/pull/79997 or replace it with a narrower fix that raises the unsafe default while preserving stuck-process protection and proving the long-silent CLI path.

## Reproduction Assessment

Yes, at source level. Current main still resolves ordinary inherited resumed CLI runs with a 600-second outer timeout to `180_000`, and the CLI runner passes that value into the no-output termination path; I did not run a live 180-second CLI stall in this read-only review.

## Solution Assessment

Unclear as an implementation choice. Raising the cap is a plausible mitigation, but the best fix still needs a maintainer decision on default length, disablement semantics, and how much stuck-process protection to preserve.
