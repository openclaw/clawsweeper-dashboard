## Summary

Keep open: current main and latest release still preserve the ordinary resumed CLI 180s no-output cap, while the open implementation path needs maintainer policy review and proof before it can close the issue.

## What This Changes

Request to raise, remove, or make configurable the 180s resumed CLI no-output watchdog cap so long silent CLI agent turns are not terminated before the outer timeout.

## Best Possible Solution

Land a maintainer-approved watchdog policy change that lets legitimate long-silent resumed CLI work honor explicit timeouts or overrides while preserving bounded stuck-process cleanup, with focused resolver/runtime tests and redacted real behavior proof.

## Reproduction Assessment

Yes, at source level: ordinary inherited resumed CLI runs still resolve a 600s outer timeout to a 180_000 ms no-output watchdog. I did not run a live three-minute silent CLI turn in this read-only review.

## Solution Assessment

Unclear as an implementation direction. Raising defaults is a plausible mitigation, but removing or disabling the cap needs maintainer policy judgment because the watchdog also protects stuck CLI processes.
