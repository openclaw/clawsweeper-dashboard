## Summary

Keep open: current main and the latest release still preserve the 180-second inherited resumed-user CLI no-output cap, while the main open implementation path is unmerged and still needs maintainer policy review/proof.

## What This Changes

The issue asks to raise, remove, or make tunable the hard 3-minute no-output watchdog cap for CLI-backed agent requests.

## Best Possible Solution

Choose a maintainer-approved watchdog policy that changes the inherited resumed-user default or disable behavior, uses the existing backend watchdog tuning surface where possible, and lands with real CLI behavior proof plus focused resolver coverage.

## Reproduction Assessment

Yes. Current main still defines the resumed CLI watchdog max as 180,000 ms, and the resolver test explicitly expects an inherited resumed user run with a 600-second outer timeout to resolve to 180,000 ms.

## Solution Assessment

Unclear. Raising the cap is a plausible mitigation, but the exact default, disable behavior, and stuck-process protection policy need maintainer approval before a final implementation shape is best.
