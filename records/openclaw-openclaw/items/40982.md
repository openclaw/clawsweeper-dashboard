## Summary

Keep open: current main and latest release still preserve the ordinary resumed CLI 180s no-output cap, and the open implementation path needs maintainer decision and proof before it can own the fix.

## What This Changes

The issue requests raising, removing, or making configurable the 3-minute no-output watchdog cap that can terminate long-running CLI-backed agent requests.

## Best Possible Solution

Land a maintainer-approved CLI watchdog policy that avoids false 180s aborts for healthy long-running CLI sessions while preserving a clear stuck-process guard, with focused resolver tests and release-note context.

## Reproduction Assessment

Yes, at source level: current main still resolves an inherited resumed user CLI run with a 600s outer timeout to a 180s no-output watchdog, and the issue discussion includes matching live logs. I did not run a live 180-second stall in this read-only review.

## Solution Assessment

Unclear as an implementation direction: raising or disabling the cap is plausible, but the exact default, no-cap semantics, and state-gated behavior need maintainer policy review. The current open PR is not the best fix as submitted because it is conflicting and still needs proof/policy cleanup.
