## Summary

Keep open: current main and v2026.6.10 still cap ordinary resumed CLI no-output watchdogs at 180 seconds, and the open related PR at https://github.com/openclaw/openclaw/pull/79997 has not landed or resolved the maintainer policy/proof blockers.

## What This Changes

The issue requests raising, removing, or making configurable the CLI no-output watchdog cap that can terminate silent resumed CLI requests after about three minutes.

## Best Possible Solution

Land a maintainer-approved, proof-backed watchdog policy that raises the remaining resumed-user cap or supports bounded per-backend tuning while preserving stuck-process protection and updating focused tests/docs.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: current main still resolves ordinary resumed user CLI runs without explicit override to the 180,000 ms ceiling. I did not run a live three-minute silent CLI process in this read-only review.
