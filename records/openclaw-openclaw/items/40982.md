## Summary

Keep open: current main still caps default resumed CLI no-output watchdogs at 180 seconds, and https://github.com/openclaw/openclaw/pull/41022 is an open closing PR for this issue. That PR is currently conflicting and still marked as needing real behavior proof, so the issue should remain open until maintainers land, replace, or close that implementation path.

## What This Changes

The issue asks to raise or remove the 3-minute no-output watchdog cap for resumed CLI sessions and optionally add tunable or no-cap semantics.

## Best Possible Solution

Review, fix, and land or replace the active watchdog-default PR with passing regression coverage and redacted real CLI proof, while tracking Monitor expected-idle and no-cap policy separately.

## Reproduction Assessment

Yes, at source level. Current main still resolves default resumed non-cron CLI runs through CLI_RESUME_WATCHDOG_DEFAULTS.maxMs = 180_000 and passes that value into the timeout enforcement path; reporter logs match the 180-second failure.

## Solution Assessment

Unclear as a full solution. Raising the default cap is a plausible narrow mitigation, but removing the cap or changing expected-idle semantics needs maintainer policy review and should be reconciled with the open watchdog PRs.
