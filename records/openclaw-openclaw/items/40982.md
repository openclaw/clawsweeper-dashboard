## Summary

Keep this issue open: current main and the latest release still cap ordinary inherited resumed CLI runs at 180,000 ms, while the related open PR is unmerged and still needs maintainer review for the watchdog default/config policy.

## What This Changes

The issue asks to raise or remove the 180,000 ms CLI no-output watchdog cap so long-running silent resumed CLI requests are not killed prematurely.

## Best Possible Solution

Keep this issue as the canonical CLI watchdog cap tracker and land a maintainer-approved fix through the open PR or a narrower replacement that raises or removes the ordinary resumed-session cap with focused resolver tests and real CLI behavior proof.

## Reproduction Assessment

Yes. Source inspection shows ordinary inherited resumed CLI runs still resolve through the 180,000 ms resume default, and the issue has a user log comment reporting repeated 180s failures on a released OpenClaw version; I did not run a live 3-minute CLI stall in this read-only review.

No final fix is merged yet. Raising defaults is a plausible repair, but the best solution needs a maintainer-approved watchdog policy and upgrade proof because it changes default stall detection behavior for existing users.
