## Summary

Keep open: current main and v2026.6.8 still keep the 180s inherited resumed-CLI no-output cap, while https://github.com/openclaw/openclaw/pull/90912 only lifted configured cron/explicit timeout paths and https://github.com/openclaw/openclaw/pull/79997 remains unmerged.

## What This Changes

Request to raise or remove the CLI no-output watchdog cap for resumed CLI sessions and optionally allow per-backend tuning or disablement.

## Best Possible Solution

Land a maintainer-approved CLI no-output watchdog policy that removes the harmful inherited 180s resume ceiling for legitimate long-silent work while preserving an explicit availability guard and focused regression coverage.

## Reproduction Assessment

Yes, at source level: current main still resolves ordinary inherited resumed CLI runs to 180_000 ms and the runtime emits the reported no-output termination text. I did not run a live long-silent CLI session in this read-only review.

## Solution Assessment

Unclear as proposed: raising/removing the cap is plausible, but the best fix needs a maintainer timeout policy that balances long legitimate silence against stuck-process recovery and uses the existing per-backend watchdog config surface deliberately.
