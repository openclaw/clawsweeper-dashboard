## Label Justifications

- `P1`: The issue reports real CLI-backed agent turns being killed mid-run and surfacing as failed model/session workflows.

## Summary

Keep open: current main and latest release v2026.6.10 still preserve the 180-second no-output cap for ordinary resumed CLI user runs, while the open implementation candidate at https://github.com/openclaw/openclaw/pull/79997 is conflicting, proof-blocked, and broader than this issue.

## What This Changes

This issue requests raising, removing, or making configurable the hard 3-minute CLI no-output watchdog cap that terminates long-running resumed CLI sessions.

## Best Possible Solution

Keep this canonical issue open until maintainers choose a proof-backed watchdog policy, preferably by refreshing or replacing the open PR with a narrow default change using the existing watchdog config surface plus upgrade and real-behavior evidence.

## Reproduction Assessment

Yes. Source inspection shows an ordinary resumed user run with a 600_000 ms overall timeout still resolves to a 180_000 ms no-output watchdog, and the runner passes that value into the termination path; I did not run a live 180-second stall in this read-only review.

## Solution Assessment

No complete solution is on main yet. Raising the bounded defaults through the existing per-backend watchdog surface is the narrowest likely code fix, while disable/null semantics need explicit maintainer product approval and upgrade proof.
