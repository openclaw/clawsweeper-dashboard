## Summary

Keep open: current main and the latest release still keep the default resumed CLI no-output watchdog capped at 180 seconds, and an open closing PR at https://github.com/openclaw/openclaw/pull/41022 is the active implementation path. That PR is still conflicting and marked as needing real behavior proof, so this issue should stay open until maintainers land, replace, or close that path.

## What This Changes

The issue requests raising or removing the hard 180-second default CLI no-output watchdog cap for long-running CLI-backed agent requests.

## Best Possible Solution

Review, fix, and land or replace the active watchdog-default PR with updated tests and redacted real CLI proof, while tracking no-cap and Monitor expected-idle semantics as separate policy work if needed.

## Reproduction Assessment

Yes, at source level. Current main still resolves default resumed non-cron CLI runs through CLI_RESUME_WATCHDOG_DEFAULTS.maxMs = 180000 and passes that value into both timeout enforcement paths; reporter logs match the 180-second failure.

## Solution Assessment

Unclear as a full solution. Raising the default cap is a plausible narrow mitigation, but removing or disabling the cap and coordinating Monitor expected-idle semantics need maintainer timeout-policy judgment.
