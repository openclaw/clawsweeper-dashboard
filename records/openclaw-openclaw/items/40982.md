## Summary

Keep open: current main still has the 180s resumed CLI no-output cap, and an open linked PR with closing syntax is the active implementation path rather than a reason to close this issue before merge.

## What This Changes

Raise or remove the CLI no-output watchdog defaults so long-running resumed CLI requests are not terminated after about three minutes of stdout silence.

## Best Possible Solution

Review and land or replace #41022 for the immediate default-cap fix, while keeping #71803 separate if maintainers want Monitor-specific idle semantics.

## Reproduction Assessment

Yes, source-reproducible with high confidence: a resumed CLI run with default watchdog settings and no stdout for more than 180 seconds resolves to `noOutputTimeoutMs=180000` and follows the existing no-output termination path. I did not run a live three-minute CLI hang in this read-only review.

## Solution Assessment

Yes for the immediate cap: the linked PR’s described default increase is the narrowest first fix for this report. A fuller watchdog disable mode or Monitor-aware pause should be handled deliberately as follow-up scope rather than hidden in this issue.
