## Summary

Keep open: current main and v2026.6.6 still keep the ordinary resumed CLI no-output cap at 180 seconds, and the open related PR is an unmerged implementation candidate rather than a completed fix.

## What This Changes

The issue requests raising, removing, or making configurable the 3-minute no-output watchdog cap for resumed CLI sessions so long silent agent work is not terminated prematurely.

## Best Possible Solution

Keep this issue as the canonical tracker, then land a refreshed, focused watchdog policy change that either raises the inherited resume cap safely or exposes a documented operator-controlled override without weakening runaway-process protection by accident.

## Reproduction Assessment

Yes at source level: ordinary inherited resumed CLI runs still resolve to the 180-second no-output cap, and the test suite currently asserts that behavior. I did not run a live 180-second CLI stall in this read-only review.

## Solution Assessment

Unclear as a final product choice: raising the default cap is plausible, but removing or disabling the watchdog and adding config surface need maintainer agreement on the availability versus stuck-process tradeoff.
