## Summary

Keep open. Current main has pending approval list/resolve backend methods and an event-driven Control UI modal queue, but it still lacks the requested discoverable Web dashboard or panel for recovering missed exec approvals; the CLI slice remains separately tracked.

## What This Changes

Add a Web or CLI surface to list live pending exec approvals and approve or deny them when popup delivery is missed.

## Best Possible Solution

Add a first-class Pending approvals Control UI page or panel backed by `exec.approval.list` and `plugin.approval.list`, resolving through the existing scoped methods while coordinating CLI work through https://github.com/openclaw/openclaw/issues/9987.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that backend list/resolve APIs exist while the Web UI remains modal/event-driven, and the intermittent popup delivery failure itself is not reproducible without channel logs.

## Solution Assessment

Unclear until maintainers choose the Web/CLI scope, audit model, timeout behavior, and stale-resolution UX. The narrow maintainable implementation should reuse the existing approvals-scoped list and resolve RPCs rather than adding a second pending-approval store.
