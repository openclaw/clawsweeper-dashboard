## Summary

Keep open. Current main has reusable pending approval backend APIs and an event-driven Control UI approval modal queue, but it still lacks the requested discoverable Web dashboard or panel for finding and resolving missed exec approvals; the CLI-only slice remains open separately.

## What This Changes

Request a Web GUI, and optionally CLI, surface to list pending exec approvals and approve or deny them without relying on popup delivery.

## Best Possible Solution

Add a first-class Pending approvals Control UI page or panel backed by the existing `exec.approval.list`, `plugin.approval.list`, and scoped resolve methods, while keeping the CLI command tracked through https://github.com/openclaw/openclaw/issues/9987.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that pending-list/resolve plumbing exists, while the requested Web recovery surface is absent and the intermittent popup-delivery failure lacks channel logs.

## Solution Assessment

Unclear until maintainers choose the Web/CLI split, audit model, timeout behavior, and stale-resolution UX. The narrow maintainable direction is to reuse the existing approvals-scoped list and resolve RPCs rather than adding a second pending-approval store.
