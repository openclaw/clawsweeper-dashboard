## Summary

Keep open. Current main has backend pending-approval RPCs, but it still lacks a discoverable Control UI dashboard or CLI live queue for recovering approvals missed by popup delivery, and the remaining work needs maintainer scope and security decisions.

## What This Changes

Request a Web GUI and/or CLI surface to list pending exec approvals and approve or deny them without relying on transient popups.

## Best Possible Solution

Add a first-class Pending approvals Control UI page or panel that loads `exec.approval.list` and `plugin.approval.list` on connect/reconnect, updates from approval events, resolves through the existing scoped RPCs, and aligns any CLI pending command with #9987.

## Reproduction Assessment

Yes for the central missing surface: source inspection shows pending-list RPCs and tests exist while the Web UI and CLI do not expose a browsable live queue. The intermittent popup delivery failure itself remains unclear without live channel logs.

## Solution Assessment

Not applicable because there is no patch. The narrow maintainable direction is to reuse the existing `exec.approval.*` and `plugin.approval.*` RPCs behind `operator.approvals`, not introduce a second approval store.
