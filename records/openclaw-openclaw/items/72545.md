## Summary

Keep open: current main and v2026.6.8 have pending approval list/resolve plumbing and a Control UI modal queue, but they still do not provide the requested discoverable Web dashboard or panel for recovering missed exec approvals.

## What This Changes

Request a Web GUI interface, with possible CLI fallback, to list pending exec approvals and approve or deny them when pop-up prompts are missed.

## Best Possible Solution

Add a first-class Pending approvals Control UI page or panel backed by the existing exec/plugin approval list and resolve RPCs, preserving the `operator.approvals` boundary and keeping the CLI-only command tracked separately.

## Reproduction Assessment

Not applicable as a feature request. Source and release inspection show backend pending-list RPCs and a modal queue exist, while the requested discoverable Web recovery dashboard is absent from current main and v2026.6.8.

## Solution Assessment

Unclear until maintainers choose the Web UX, audit model, timeout behavior, and stale-resolution semantics. The narrow maintainable direction is to reuse the existing approvals-scoped list and resolve RPCs rather than adding a second pending-approval store.
