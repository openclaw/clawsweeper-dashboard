## Summary

Keep open. Current main has backend pending-approval list/resolve RPCs and an event-driven Web modal queue, but it still lacks the requested discoverable Web dashboard or panel for recovering approvals missed by popup delivery; the CLI-only slice remains tracked separately at https://github.com/openclaw/openclaw/issues/9987.

## What This Changes

Request to add a Web GUI, and possibly CLI, surface that lists pending exec approvals and lets operators approve or deny them without relying on popup delivery.

## Best Possible Solution

Add a first-class Pending approvals Control UI page or panel backed by `exec.approval.list` and `plugin.approval.list`, resolving through the existing scoped RPCs and coordinating any CLI work with https://github.com/openclaw/openclaw/issues/9987.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that backend list/resolve APIs exist while the Web UI remains modal/event-driven and the CLI has no live pending command; the intermittent popup delivery failure itself is not reproduced without channel logs.

## Solution Assessment

Unclear until maintainers choose the Web/CLI scope, audit model, and stale/timeout behavior. The narrow maintainable direction is to reuse the existing `exec.approval.list`, `plugin.approval.list`, and resolve methods behind `operator.approvals`, not add a second pending-approval store.
