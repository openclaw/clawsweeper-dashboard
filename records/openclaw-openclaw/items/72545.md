## Summary

Keep open. Current main has pending exec/plugin approval RPCs and event-driven Control UI modals, but it still lacks a discoverable Web dashboard or CLI queue for listing missed live approvals and resolving them outside popup delivery.

## What This Changes

Add a Web or CLI surface to list pending live exec approvals and approve or deny them when popup delivery is missed.

## Best Possible Solution

Add a first-class Pending approvals Control UI page or panel that loads `exec.approval.list` and `plugin.approval.list` on connect/reconnect, updates from approval events, resolves through the existing scoped RPCs, and leaves the CLI-only pending command to #9987 unless maintainers merge the scopes.

## Reproduction Assessment

Yes for the central missing surface: source inspection shows pending-list RPCs exist while the Control UI only renders event-driven modals and the CLI only manages approval policy files. The intermittent popup non-delivery path remains unclear without a live failing run or channel logs.

## Solution Assessment

Not applicable because there is no patch. The maintainable direction is to reuse the existing `exec.approval.*` and `plugin.approval.*` RPCs under `operator.approvals`, not add a second approval store.
