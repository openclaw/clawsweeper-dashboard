## Summary

Keep open. Current main has backend pending exec/plugin approval list and resolve RPCs, plus event-driven Control UI approval modals, but it still lacks a discoverable Web dashboard or CLI queue for finding missed live approvals and acting on them outside popup delivery.

## What This Changes

Request a Web GUI and/or CLI surface to list pending exec approvals and approve or deny them when popups are missed.

## Best Possible Solution

Add a first-class Pending approvals Control UI page or panel that hydrates `exec.approval.list` and `plugin.approval.list` on connect/reconnect, updates from approval events, resolves through the existing scoped RPCs, and coordinates any CLI pending command with #9987.

## Reproduction Assessment

Yes for the central missing surface: source inspection shows pending-list RPCs exist while the Control UI only renders event-driven modals and the CLI only manages approval policy snapshots. The intermittent popup failure itself remains unclear without a live failing run or channel logs.

## Solution Assessment

Not applicable because there is no patch. The maintainable direction is to reuse the existing `exec.approval.*` and `plugin.approval.*` RPCs under `operator.approvals`, not introduce a second approval store.
