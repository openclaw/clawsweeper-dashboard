## Summary

Keep open. Current main has the backend approval queue primitives needed for a dashboard, but the Control UI still exposes approval requests as an event-driven modal and the approvals CLI still manages policy rather than pending live work; the remaining scope is security-sensitive and needs maintainer decisions.

## What This Changes

Add a Web GUI, and possibly CLI, surface to list pending exec approvals and approve or deny them when popup delivery is missed.

## Best Possible Solution

Add a first-class Pending approvals Control UI page or panel backed by `exec.approval.list` and `plugin.approval.list`, resolve through the existing scoped RPCs, and align any CLI command with #9987 while preserving `operator.approvals` authorization.

## Reproduction Assessment

Yes for the central missing surface: source inspection shows pending-list RPCs and event-driven modal handling, but no Web dashboard or CLI pending queue caller. The intermittent popup delivery failure remains unclear without live channel logs.

## Solution Assessment

Not applicable because there is no patch. The narrow maintainable direction is to reuse the existing `exec.approval.*` and `plugin.approval.*` queue/resolve RPCs rather than introduce a second approval store.
