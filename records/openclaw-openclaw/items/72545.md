## Summary

Keep open. Current main has backend pending-approval RPCs and modal event handling, but it still lacks the requested discoverable Web dashboard or live CLI queue for recovering approvals missed by popup delivery; the CLI-only slice remains tracked separately at https://github.com/openclaw/openclaw/issues/9987.

## What This Changes

Request a Web GUI or CLI surface to list pending exec approvals and approve or deny them without relying on channel popups.

## Best Possible Solution

Add a first-class Pending approvals Control UI page or panel that loads `exec.approval.list` and `plugin.approval.list`, updates from approval events, resolves through the existing scoped RPCs, and coordinates any CLI surface with https://github.com/openclaw/openclaw/issues/9987.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that the backend list RPCs exist while the Web UI remains event/modal-driven and the CLI has no live pending command; the intermittent popup delivery failure itself is not reproduced without live channel logs.

## Solution Assessment

Unclear until maintainers choose the Web/CLI scope and audit model. The narrow maintainable direction is to reuse `exec.approval.list`, `plugin.approval.list`, and existing resolve methods behind `operator.approvals`, not add a second pending-approval store.
