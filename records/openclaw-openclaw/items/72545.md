## Summary

Keep open. Current main has backend pending exec/plugin approval RPCs and event-driven Control UI approval modals, but it still lacks a discoverable Web dashboard or CLI queue for finding missed live approvals and resolving them outside popup delivery; the related #9987 only covers the CLI subset.

## What This Changes

Add a Web GUI or CLI surface for listing and resolving live exec approval requests when popup delivery is missed.

## Best Possible Solution

Add a first-class Pending approvals Control UI page or panel that loads existing `exec.approval.list` and `plugin.approval.list` data on connect/reconnect, resolves through the existing scoped RPCs, and align or split the CLI subset with #9987.

## Reproduction Assessment

Yes for the central missing surface: source inspection shows pending-list RPCs exist while the Control UI only renders event-driven modals and the CLI only manages approval policy files. The intermittent popup failure itself remains unclear without live channel logs.

## Solution Assessment

Not applicable because there is no patch. The maintainable direction is to reuse the existing `exec.approval.*` and `plugin.approval.*` RPCs under `operator.approvals`, rather than adding a second approval store.
