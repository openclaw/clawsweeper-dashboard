## Summary

Keep open. Current main has scoped pending-approval list/resolve RPCs and event-driven Control UI modals, but it still lacks the requested discoverable Web dashboard or CLI queue for finding missed live exec approvals and acting on them outside popup delivery.

## What This Changes

Add a discoverable Web GUI surface, and possibly CLI commands, to list and resolve live pending exec approvals when popup delivery is missed.

## Best Possible Solution

Add a first-class Pending approvals surface in Control UI that loads `exec.approval.list` and `plugin.approval.list` on connect/reconnect, updates from approval events, resolves through the existing scoped RPCs, and documents the recovery workflow; add CLI pending/approve/deny commands only after maintainers confirm terminal recovery scope.

## Reproduction Assessment

Yes for the central missing surface: source inspection shows pending-list backend RPCs and event-driven modal handling, but no Web dashboard, connect-time pending-list replay, or CLI pending queue commands. The intermittent popup non-delivery path itself remains unclear without logs from a failing channel/session.

## Solution Assessment

Not applicable because there is no patch. The maintainable direction is to reuse the existing `exec.approval.*` and `plugin.approval.*` RPCs under `operator.approvals`, rather than introducing a second approval store.
