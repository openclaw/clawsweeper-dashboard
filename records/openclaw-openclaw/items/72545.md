## Summary

Keep open: current main and v2026.6.8 have backend pending-approval list/resolve plumbing and an event-driven Control UI modal queue, but they still do not provide the requested discoverable Web dashboard or panel for recovering missed approvals.

## What This Changes

The issue requests a Web GUI surface to list pending exec approval requests and approve or deny them when channel popups are missed.

## Best Possible Solution

Add a first-class Pending approvals Control UI page or panel backed by the existing exec/plugin approval list and resolve RPCs, preserve the `operator.approvals` boundary, and keep the CLI-only pending command tracked at https://github.com/openclaw/openclaw/issues/9987.

## Reproduction Assessment

Not applicable as a feature request. Source and docs inspection show backend list/resolve plumbing exists, while the requested discoverable Web recovery dashboard is absent from current main and v2026.6.8.

## Solution Assessment

Unclear until maintainers choose the Web UX, audit model, timeout behavior, and stale-resolution semantics. The narrow maintainable direction is to reuse the existing approvals-scoped list and resolve RPCs rather than adding a second pending-approval store.
