## Summary

Keep open. Current main has pending approval list/resolve RPCs, but the requested recoverable Web GUI dashboard and live CLI queue are still missing, and exposing approval decisions needs maintainer security/product scope.

## What This Changes

Request to add a Web GUI and/or CLI surface for listing pending exec approvals and approving or denying them without relying only on popup delivery.

## Best Possible Solution

Add a first-class Pending approvals Control UI page or panel that loads `exec.approval.list` and `plugin.approval.list` on connect/reconnect, updates from approval events, resolves through the existing scoped RPCs, and coordinates any CLI live-queue work with #9987.

## Reproduction Assessment

Yes for the central missing surface: source inspection shows backend pending-list RPCs and tests exist, while the Web UI and CLI do not expose a browsable live approval queue. The intermittent popup delivery failure itself remains unclear without live channel logs.

## Solution Assessment

Not applicable because there is no patch. The narrow maintainable direction is to reuse the existing `exec.approval.*` and `plugin.approval.*` RPCs behind `operator.approvals`, not introduce a second approval store.
