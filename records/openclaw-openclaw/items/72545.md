## Summary

Keep open. Current main has backend pending-approval list/resolve RPCs and event-driven Control UI modals, but it still lacks a discoverable Web dashboard or CLI queue for finding missed live exec/plugin approvals and acting outside popup delivery.

## What This Changes

The issue requests a Web GUI and possibly CLI interface to list and approve or deny live exec approvals when popups are missed.

## Best Possible Solution

Add a first-class Pending approvals Control UI surface that loads `exec.approval.list` and `plugin.approval.list` on connect/reconnect, updates from approval events, resolves through the existing scoped RPCs, and documents the recovery workflow; add CLI queue commands only after maintainers choose that scope.

## Reproduction Assessment

Yes for the central missing surface: source inspection shows pending-list backend RPCs exist while the Control UI only renders event-driven modals and the CLI only manages policy/allowlists. The intermittent popup non-delivery case remains unclear without a live failing run or channel logs.

## Solution Assessment

Not applicable because there is no patch. The maintainable direction is to reuse the existing `exec.approval.*` and `plugin.approval.*` RPCs under `operator.approvals`, rather than introducing a second approval store.
