## Summary

Keep open. Current main has pending approval list/resolve backend APIs and an event-driven Control UI modal queue, but it still lacks the requested discoverable Web dashboard or panel for finding and resolving missed exec approvals.

## What This Changes

Request a Web GUI surface, and optionally CLI support, to list pending exec approvals and approve or deny them without relying on popups.

## Best Possible Solution

Add a first-class Pending approvals Control UI page or panel backed by exec.approval.list, plugin.approval.list, and scoped resolve methods, while keeping the CLI command tracked separately in https://github.com/openclaw/openclaw/issues/9987.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that backend list/resolve plumbing exists while the requested discoverable Web recovery surface is absent.

## Solution Assessment

Unclear until maintainers choose the Web UX, audit model, timeout behavior, and stale-resolution semantics. The narrow maintainable direction is to reuse the existing approvals-scoped list and resolve RPCs rather than adding a second pending-approval store.
