## Summary

Keep open. Current main and the latest release already have backend pending-approval list/resolve plumbing and an event-driven Control UI modal queue, but they still lack the requested discoverable Web dashboard or panel for finding and resolving missed exec approvals.

## What This Changes

Request adds a discoverable Web GUI dashboard or panel to list pending exec approvals and approve or deny them when popup approvals are missed.

## Best Possible Solution

Add a first-class Pending approvals Control UI page or panel backed by the existing exec/plugin approval list and resolve RPCs, preserving operator.approvals and keeping CLI-only work tracked separately at https://github.com/openclaw/openclaw/issues/9987.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that backend list/resolve plumbing exists while the requested discoverable Web recovery surface is absent.

## Solution Assessment

Unclear until maintainers choose the Web UX, audit model, timeout behavior, and stale-resolution semantics. The narrow maintainable direction is to reuse existing approvals-scoped list and resolve RPCs rather than adding a second pending-approval store.
