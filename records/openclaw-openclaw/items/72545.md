## Summary

Keep open. Current main and v2026.5.19 have backend pending-approval APIs and an event-driven modal queue, but they still lack the requested discoverable Web dashboard or panel for finding and resolving missed exec approvals.

## What This Changes

The issue requests a Web GUI surface, and secondarily CLI support, to list pending exec approvals and approve or deny them when popups are missed.

## Best Possible Solution

Add a first-class Pending approvals Control UI page or panel backed by `exec.approval.list`, `plugin.approval.list`, and the existing scoped resolve methods; keep the CLI-only path with https://github.com/openclaw/openclaw/issues/9987 or its implementation PR.

## Reproduction Assessment

Not applicable as a feature request. Source and release inspection give high confidence that backend list/resolve plumbing exists while the discoverable Web dashboard or panel is absent.

## Solution Assessment

Unclear until maintainers choose the Web UX, audit model, timeout behavior, and stale-resolution semantics. The narrow maintainable direction is to reuse the existing approvals-scoped list and resolve RPCs rather than adding a second pending-approval store.
