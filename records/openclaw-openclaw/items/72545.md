## Summary

Keep open. Current main has backend pending approval list/resolve APIs and event-driven approval modals, but it still lacks a discoverable Control UI dashboard or CLI pending-queue action surface; the related #9987 only covers the narrower CLI listing request.

## What This Changes

The issue requests a Web GUI and/or CLI surface to list pending exec approvals and approve or deny them without relying on popup delivery.

## Best Possible Solution

Add a first-class Pending approvals Control UI surface that loads `exec.approval.list` and `plugin.approval.list` on connect/reconnect, updates from approval events, resolves through scoped RPCs, and coordinates CLI queue work with #9987.

## Reproduction Assessment

Yes for the central missing surface: source inspection shows pending-list backend RPCs exist while Web only renders event-driven modals and the CLI only manages policy. The intermittent popup non-delivery path is unclear without live logs.

## Solution Assessment

Not applicable because there is no patch. The maintainable direction is to reuse the existing `exec.approval.*` and `plugin.approval.*` RPCs under `operator.approvals`, not add a second approval store.
