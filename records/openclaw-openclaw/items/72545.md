## Summary

Keep open. Current main has pending exec/plugin approval list and resolve RPCs plus event-driven Control UI approval modals, but it still lacks the requested discoverable Web dashboard or CLI live queue for finding missed approvals and acting outside popup delivery.

## What This Changes

Request a Web GUI, and optionally CLI, surface to list live pending exec approvals and approve or deny them when popups are missed.

## Best Possible Solution

Add a first-class Pending approvals Control UI view that loads `exec.approval.list` and `plugin.approval.list` on connect/reconnect, updates from approval events, resolves through the existing scoped RPCs, and documents the recovery workflow; add CLI pending/approve/deny commands only after maintainers settle terminal authorization and audit semantics.

## Reproduction Assessment

Yes for the central missing surface: source inspection shows pending-list backend RPCs exist while Control UI only handles live events and renders a first-item modal, and the CLI only manages policy snapshots/allowlists. The intermittent popup failure itself remains unclear without channel logs or a live failing run.

## Solution Assessment

Not applicable because there is no patch. The maintainable direction is to reuse the existing `exec.approval.*` and `plugin.approval.*` RPCs under `operator.approvals`, rather than introducing a second approval store.
