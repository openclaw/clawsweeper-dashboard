## Summary

Keep open: current main has broad operator/admin cron mutation and an owner-only model-facing cron tool with a narrow self-cleanup grant, but it still lacks the requested constrained agent scheduling contract with gateway-injected provenance, playbook allowlisting, quota/lifetime policy, and mutation audit.

## What This Changes

The issue requests an agent-facing scheduling API that creates constrained cron jobs with gateway-injected provenance, playbook allowlisting, quotas, lifetime policy, and audit logging.

## Best Possible Solution

Add an additive gateway scheduling capability that derives actor and timestamp from authenticated request context, accepts only registered playbooks with validated params, enforces quota and lifetime policy, records schedule mutation audit events, and keeps unrestricted operator cron separate.

## Reproduction Assessment

Not applicable as a runtime bug; this is a security-sensitive feature/RFC. Source inspection gives high confidence that schedule_task, gateway-injected createdBy, playbooks, per-agent quotas, recurring lifetime policy, and schedule-mutation audit do not exist on current main.

## Solution Assessment

Unclear until maintainers choose the policy contract. The requested direction is safer than exposing broad cron.add to non-owner agents, but the maintainable fix should be a designed Gateway/cron/audit capability rather than a one-off schema patch.
