## Summary

Keep open. Current main has broad agent-visible cron scheduling and admin-scoped Gateway cron RPC, but it does not implement the proposed gateway-enforced playbook allowlist, non-forgeable creator provenance, recurring expiration requirement, per-agent quota, or dedicated audit contract; related open issues cover adjacent pieces rather than a merged canonical replacement.

## What This Changes

The issue requests a new agent-facing scheduling API that lets agents create bounded cron tasks through the Gateway with non-forgeable provenance and security guardrails.

## Best Possible Solution

Keep this as the focused scheduler-provenance RFC until maintainers define one additive Gateway contract for agent-created schedules, playbook registration, creator provenance, expiration policy, quota enforcement, audit records, and migration/docs/tests.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a runtime bug, but yes at source level: current main lacks the requested schedule_task/playbook/provenance/activeUntil/quota fields in the cron tool, cron job model, Gateway schema, and docs.

## Solution Assessment

Is this the best way to solve the issue? Unclear until maintainer design review; the direction is plausible, but the best fix needs a single approved scheduler security contract rather than a one-off field addition.
