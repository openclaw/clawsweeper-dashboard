## Summary

Keep open: current main still lacks the requested constrained agent-facing scheduling API, and the RFC is recent, security-sensitive, and backed by AS76's production-use context, so this needs maintainer design review rather than cleanup closure.

## What This Changes

Add an agent-facing scheduling API that creates or updates cron jobs through gateway-enforced provenance, playbook allowlists, quotas, recurring lifetime controls, and audit logging.

## Best Possible Solution

Create a short core design/security record, then add an additive Gateway scheduling capability that derives actor and timestamp from authenticated request context, accepts only registered playbook identifiers with validated params, enforces quota and lifetime policy, records schedule create/update audit events, and leaves unrestricted operator cron separate.

## Reproduction Assessment

Not applicable as a feature/security RFC rather than a runtime bug. The high-confidence verification path is source and docs inspection on current main showing no schedule_task surface, no playbook allowlist, no injected creator provenance, no per-agent schedule quota, no recurring lifetime policy, and no durable schedule-creation audit record.

## Solution Assessment

Unclear until maintainers choose the product and security model. The requested direction is safer than exposing the full cron.add shape to non-owner agents, but the maintainable solution should be a designed Gateway/cron/audit capability rather than a one-off schema patch.
