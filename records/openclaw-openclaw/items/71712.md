## Summary

Keep open: https://github.com/openclaw/openclaw/pull/96883 added useful agent-tool cron scoping, but current main still lacks the requested Gateway-owned schedule-authority API with non-forgeable provenance, registered playbook allowlists, quotas, activeUntil/permanent policy, and execution-lineage receipts.

## What This Changes

The issue requests an agent-facing scheduling API that creates or modifies recurring cron jobs through the Gateway with injected provenance, allowlisted playbooks, quotas, expiry policy, and audit lineage.

## Best Possible Solution

Keep this as a maintainer-owned security/product RFC and design one Gateway schedule-authority contract that covers creation provenance, execution lineage, playbook registration/versioning, quotas, activeUntil/permanent policy, audit records, and migration semantics.

## Reproduction Assessment

Yes for source-level gap proof, not as a failing bug reproduction. Current main has agent-scoped cron CRUD, but the agent tool, Gateway protocol, persisted cron schema, and docs still lack the requested playbook/provenance/quota/expiry/execution-lineage contract.

## Solution Assessment

Unclear until maintainers decide the security and product contract. The best path is a designed Gateway schedule-authority surface, not further widening the existing generic cron tool.
