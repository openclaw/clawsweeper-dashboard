## Summary

Keep open: current main still lacks the requested constrained agent-facing scheduling API and the RFC is recent, security-sensitive, and supported by production-use context from AS76, so it needs maintainer design review rather than cleanup closure.

## What This Changes

Request a Gateway-exposed agent scheduling API that creates or modifies recurring jobs through allowlisted playbooks with gateway-injected provenance, quotas, audit records, and lifetime controls.

## Best Possible Solution

Create a short core design/security record, then add an additive Gateway scheduling capability that derives actor and timestamp from authenticated request context, accepts only registered playbook identifiers plus validated params, enforces quota and lifetime policy, records schedule create/update audit events, and leaves the unrestricted operator cron CLI separate.

## Reproduction Assessment

Not applicable as a feature/security RFC rather than a runtime bug. The high-confidence verification path is source and docs inspection on current main showing the absence of schedule_task, playbook allowlists, injected createdBy provenance, per-agent quotas, activeUntil/permanent policy, and schedule-creation audit records.

## Solution Assessment

Unclear until maintainers choose the product and security model. The requested direction is safer than exposing full cron.add to non-owner agents, but the maintainable fix should be a designed Gateway/cron/audit capability rather than a one-off schema patch.

- **[medium] Persistent schedules lack non-forgeable creator provenance:** `src/cron/types-shared.ts:1`
  - body: Cron jobs currently store timestamps and routing metadata, but not a gateway-injected actor envelope that distinguishes operator-created jobs from agent-created jobs in a non-forgeable way.
  - confidence: 0.88

  - body: The current cron.add schema accepts the full systemEvent or agentTurn job create shape. Exposing that shape to non-owner agents would not enforce playbook allowlists, parameter validation, quotas, or recurring lifetime guardrails.
  - confidence: 0.87
- **[low] Schedule creation is not a durable task audit event:** `src/gateway/server-methods/cron.ts:308`
  - body: The current handler logs job creation to Gateway logs, while tasks audit tracks detached execution health. That does not provide the requested durable schedule create/update audit ledger.
  - confidence: 0.78
