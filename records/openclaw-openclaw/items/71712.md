## Summary

Keep open: current main still lacks the requested constrained agent-facing scheduling API, and the RFC is recent, security-sensitive, and supported by AS76's production-use context, so it needs maintainer design review rather than cleanup closure.

## What This Changes

Request a constrained agent-facing scheduling API that lets agents create allowlisted playbook schedules with gateway-injected provenance, lifetime policy, quotas, and audit records.

## Best Possible Solution

Create a short core design/security record, then add an additive Gateway scheduling capability that derives actor and timestamp from authenticated request context, accepts only registered playbook identifiers plus validated params, enforces quota and lifetime policy, records schedule create/update audit events, and leaves unrestricted operator cron separate.

## Reproduction Assessment

Not applicable as a feature/security RFC rather than a runtime bug. The high-confidence verification path is source and docs inspection on current main showing the absence of schedule_task, playbook allowlists, injected createdBy provenance, per-agent quotas, activeUntil/permanent policy, and schedule-creation audit records.

## Solution Assessment

Unclear until maintainers choose the product and security model. The requested direction is safer than exposing full cron.add to non-owner agents, but the maintainable fix should be a designed Gateway/cron/audit capability rather than a one-off schema patch.
