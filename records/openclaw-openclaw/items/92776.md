## Summary

## What This Changes

The issue asks OpenClaw to stop auto-created fallback session model pins from persisting indefinitely when fallback-origin metadata no longer matches the configured primary.

## Best Possible Solution

Review and land the guarded stale-origin cleanup in https://github.com/openclaw/openclaw/pull/92819 if maintainers accept the session/auth reset semantics, then let the merge close this issue.

## Reproduction Assessment

Yes. Source inspection shows a modelOverrideSource:auto session entry with an origin model different from the current primary is still eligible as stored session model state on normal turns, while resolveAutoFallbackPrimaryProbe returns undefined.

## Solution Assessment

Yes, with maintainer acceptance of the reset semantics. The narrow fix is guarded stale-origin cleanup in the model-selection and direct agent-command paths, not a broad new fallback persistence policy.
