## Summary

Keep open. Current main and the latest release still honor origin-mismatched auto-fallback session overrides on normal turns, and the linked fix candidate is open and unmerged, so this issue is still necessary as the canonical bug report.

## What This Changes

The issue reports that polluted auto-fallback origin metadata can leave sessions pinned to a fallback model indefinitely instead of retrying the configured primary.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/92819 or an equivalent rebased fix that clears stale auto-fallback origins consistently across reply and direct-agent selection while preserving user-owned overrides and newer session/auth changes.

## Reproduction Assessment

Yes. Source inspection on current main shows an auto fallback entry whose origin differs from the current primary is still applied as a stored override on normal turns while the primary probe returns undefined.

## Solution Assessment

Yes. The narrow fix belongs in the shared reply and direct-agent model-selection paths that decide whether a stored auto fallback override is stale; a new fallback persistence mode is broader product work tracked separately.
