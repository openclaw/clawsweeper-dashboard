## Summary

Keep open. Current main still applies polluted-origin auto fallback pins as stored session model overrides on normal turns, and the active fix candidate is the open linked PR rather than merged code.

## What This Changes

The issue reports that auto fallback session entries can record stale fallback-origin provider/model metadata, preventing recovery to the configured primary and leaving sessions pinned to fallback models.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/92819, or an equivalent rebased fix, so stale auto fallback origins are ignored or cleared consistently across reply and direct-agent selection while preserving user-owned model overrides.

## Reproduction Assessment

Yes. A current-main session entry with `modelOverrideSource: "auto"`, fallback provider/model overrides, and origin fields that differ from the current primary is applied as a stored override while `resolveAutoFallbackPrimaryProbe` returns `undefined`.

## Solution Assessment

Yes. A shared stale-origin cleanup in the existing reply and direct-agent model-selection paths is the narrow maintainable fix; adding a new fallback persistence mode would be broader product work tracked separately.
