## What This Changes

The issue reports that auto-created fallback model pins with polluted fallback-origin fields keep sessions stuck on fallback models because the primary snap-back probe never runs.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/92819, or an equivalent core fix, so stale auto-fallback origin rows are classified consistently and cleared safely without overwriting newer user or session state.

## Reproduction Assessment

Yes. A current-main auto fallback session entry whose origin fields differ from the current primary makes resolveAutoFallbackPrimaryProbe return undefined, leaving the stored fallback override active before any snap-back probe can run.

## Solution Assessment

No. Current main is not the best end state because it still lacks generic stale-origin cleanup; the best path is the linked fix PR or an equivalent source change with concurrency-safe persistence guards.
