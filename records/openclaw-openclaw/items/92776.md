## What This Changes

The issue reports auto fallback session pins whose stored origin model no longer matches the configured primary, causing later turns to stay on a fallback model indefinitely.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/92819, or replace it with an equivalent shared stale-origin repair that covers normal reply and direct agent-command selection without broadening fallback policy.

## Reproduction Assessment

Yes. Source inspection shows an auto session entry with provider/model overrides and mismatched fallback-origin fields is still honored as a stored model on normal paths, while the primary probe exits on the origin mismatch.

## Solution Assessment

Yes. The best fix is a targeted stale-origin repair in the shared session model-selection boundary used by reply and direct agent runs, not a new config option or broad fallback-policy mode.
