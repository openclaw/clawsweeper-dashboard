## What This Changes

The issue reports that auto-created session fallback model pins with polluted fallback-origin fields stay active indefinitely instead of retrying the configured primary.

## Best Possible Solution

Merge or replace the linked fix PR with a complete stale-origin cleanup across the normal reply and direct agent-command model-selection paths, then close this issue as implemented.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: a session auto fallback entry with origin fields that differ from the current primary is still accepted as a stored override, while the primary probe resolver returns undefined for that same mismatch.

## Solution Assessment

Yes, if scoped across the shared entry points. The best fix is a bounded stale-origin cleanup for auto-owned fallback pins in both normal reply and direct agent-command model selection, while preserving user overrides and valid auto fallback provenance.
