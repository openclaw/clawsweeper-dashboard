## Summary

Keep open: current main and latest release still have the polluted-origin gap on normal session turns, and the issue is now paired with an open closing PR that should be reviewed or landed before the issue closes.

## What This Changes

The issue reports that auto-created fallback model pins can persist indefinitely when fallback-origin metadata no longer matches the configured primary, preventing the primary snap-back probe from clearing the session override.

## Best Possible Solution

Land a narrow, guarded stale-origin repair that makes reply and direct agent-command selection treat mismatched auto fallback origin pins as stale while preserving user overrides and newer concurrent session changes.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to seed an auto session entry with fallback provider/model overrides and fallback-origin fields that do not match the current primary; current main can still honor the stored fallback on normal turns while `resolveAutoFallbackPrimaryProbe` returns no probe.

## Solution Assessment

Yes, the best fix shape is the linked narrow stale-origin cleanup shared by reply and direct agent-command selection, with exact-row persistence guards. The issue should close only after that or an equivalent PR lands.
