## Summary

Keep open. Current main still has a source-reproducible gap for generic polluted auto-fallback origin rows: the primary probe refuses to run when the stored origin no longer matches the current primary, and the linked open fix PR is the right path to review rather than opening a second repair lane.

## What This Changes

The issue requests fixing auto-created fallback session pins whose recorded origin provider/model can be polluted, preventing primary snap-back and keeping sessions on fallback models indefinitely.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/92819, or an equivalent core fix, so stale auto-fallback origin rows are classified consistently and cleared safely at the run boundary without overwriting newer user or session state.

## Reproduction Assessment

Yes. A current-main auto fallback session entry with origin fields that do not match the current primary makes resolveAutoFallbackPrimaryProbe return undefined, so the stored fallback override remains active before any primary snap-back attempt can run.

## Solution Assessment

No. Current main is not the best end state because it still lacks generic stale-origin cleanup; the best path is the linked fix PR or equivalent source change with concurrency-safe persistence guards.
