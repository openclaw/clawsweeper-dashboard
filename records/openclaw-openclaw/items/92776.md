## Summary

Keep open. Current main and the latest release still honor origin-mismatched auto-fallback session pins on normal turns, and the active fix path is the open linked PR rather than merged code.

## What This Changes

Report requests a fix for auto-fallback session entries whose polluted fallback-origin metadata prevents primary-probe cleanup and leaves sessions pinned to fallback models.

## Best Possible Solution

Yes. Source inspection shows an auto fallback entry with origin fields that differ from the current primary is still treated as a stored override while the primary probe returns undefined; I did not run a live provider-outage reproduction.

## Solution Assessment

Yes. A shared stale-origin cleanup in the existing reply and direct-agent model-selection paths is the narrow maintainable fix; adding a new fallback persistence mode is broader product work tracked separately.
