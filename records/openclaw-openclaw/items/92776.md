## Summary

Keep open. Current main and the latest release still allow polluted-origin auto fallback pins to remain effective stored session model overrides, and the live linked fix PR is open rather than merged.

## What This Changes

The issue asks OpenClaw to stop auto fallback session model pins from persisting indefinitely when recorded fallback-origin metadata no longer matches the configured primary model.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/92819, or an equivalent rebased fix, with coverage for stale-origin cleanup across reply and direct-agent selection plus the hot-reload/wrong-fallback reproducer.

## Reproduction Assessment

Yes. Source inspection shows a current-main session entry with auto fallback provenance and origin fields that differ from the current primary remains an effective stored override while resolveAutoFallbackPrimaryProbe returns undefined.

## Solution Assessment

Yes. A shared stale-origin repair in the existing reply and direct-agent model-selection paths is the narrow maintainable fix; a new fallback persistence mode is broader product work tracked separately.
