## Label Justifications

- `P1`: The issue reports current-release sessions staying pinned to degraded fallback models indefinitely, requiring manual session-store edits to restore normal provider routing.
- `impact:session-state`: The failure is persisted session metadata keeping an auto-created model override active after it should be recoverable.
- `impact:auth-provider`: The bug changes provider/model routing and can keep sessions on the wrong provider or model despite configured primary recovery.

## Summary

Keep open. Current `main` and v2026.6.6 still skip the primary snap-back probe when an auto fallback entry has origin fields that do not match the current primary, and normal turn selection only treats origin-mismatched auto fallback pins as stale for heartbeat cleanup, so this is not already implemented.

## What This Changes

The issue asks OpenClaw to stop indefinitely honoring auto-created session fallback model pins when polluted fallback-origin metadata prevents the configured-primary recovery probe from running.

## Best Possible Solution

Add a bounded session-state repair so auto-owned fallback pins with an origin that no longer matches the current primary are treated as stale or re-originated through the existing recovery path on normal turns, with user overrides and valid primary-probe cadence preserved.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: an auto session entry with provider/model overrides and origin fields that differ from the current primary is honored as the stored model, while `resolveAutoFallbackPrimaryProbe` returns `undefined`.

## Solution Assessment

Yes, if scoped narrowly. The best fix is not a broad new persistence policy, but a focused cleanup/probe repair for auto-owned mismatched-origin pins with regression coverage for normal turns, direct agent runs, and queued followups.

Fix the OpenClaw session auto-fallback polluted-origin bug reported at https://github.com/openclaw/openclaw/issues/92776. Current `main` treats auto fallback entries with origin fields that do not match the current primary as ineligible for `resolveAutoFallbackPrimaryProbe`, but normal turn selection still honors the stored fallback override, so sessions can stay pinned forever. Implement a bounded repair in the existing session model-selection/probe cleanup path so auto-owned mismatched-origin pins are cleared or re-originated to the current configured primary on normal turns, while preserving explicit `modelOverrideSource: "user"` selections, valid auto fallback pins whose origin still matches the primary, the five-minute probe cadence, parent-session behavior, and direct `openclaw agent` paths. Do not add config flags or edit CHANGELOG.md; include user-visible release-note context in the PR body if behavior changes. Add focused regression coverage for the reported present-but-wrong origin shape in normal reply selection, direct agent command selection, and queued/runner recovery where applicable, and stop for maintainer review if the fix would require a broader product decision about auto fallback lifetime.
