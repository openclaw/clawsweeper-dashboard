## Summary

Keep open. Current main still has the source-reproducible polluted-origin gap, and the issue is already paired with an open linked fix PR that should be reviewed or landed before closing the report.

## What This Changes

Fix auto-fallback session model pins whose polluted origin metadata prevents the primary snap-back probe from clearing stale provider/model overrides.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/92819, or an equivalent rebased fix, so stale auto fallback origins are ignored or cleared consistently across reply and direct-agent selection while preserving user-owned model overrides.

## Reproduction Assessment

Yes. A current-main session entry with `modelOverrideSource: "auto"`, fallback provider/model overrides, and origin fields that differ from the current primary is still applied as a stored override while the snap-back probe returns `undefined`.

## Solution Assessment

Yes. A shared stale-origin cleanup in the existing reply and direct-agent model-selection paths is the narrow maintainable fix; adding a new fallback persistence mode would be broader product work tracked separately.
