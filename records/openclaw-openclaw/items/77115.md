## Summary

Keep open: current main still leaves processing diagnostic entries active after recovery no-ops and classifies stalled model calls as non-recoverable, matching the reported ghost-session paths.

## What This Changes

The issue requests automatic cleanup and recovery for gateway sessions stuck in diagnostic processing state that can degrade event-loop health and make CLI commands time out.

## Best Possible Solution

Add recovery-side reconciliation that clears or idles stale diagnostic processing states once active-work checks prove no live work remains, and handle stalled model-call activity without clearing genuinely active calls.

## Reproduction Assessment

Yes, source-reproducible: set a diagnostic session state to `processing` with no active work and let the heartbeat exceed the stuck threshold; recovery can release or no-op without clearing the diagnostic state. The model-call variant is also source-reproducible by recording `model.call.started` without a terminal model/run event, which classifies as stalled active work with `recovery=none`.

## Solution Assessment

No, current main is only a partial fix: warning backoff and lane reset exist, but diagnostic state cleanup and stalled model-call reconciliation remain incomplete. The narrow maintainable repair belongs in the diagnostic recovery/state modules with regression tests around active-work safeguards.

## Repair Work Prompt

Fix the gateway diagnostic ghost-session bug reported at https://github.com/openclaw/openclaw/issues/77115. Current main leaves stale `processing` entries in `diagnosticSessionStates` after stuck-session recovery releases or no-ops a lane, and stalled `model_call` activity can remain non-recoverable when no terminal diagnostic event arrives. Implement a narrow recovery/reconciliation path in the diagnostic session state/recovery code that marks or removes stale processing entries only after checks prove there is no active embedded run, reply work, lane task, queued work that must be preserved, or genuinely active model call. Add regression coverage for no-active-work cleanup, lane-release cleanup, active-work preservation, and the stalled model-call path. Do not change trajectory writer caps/timeouts/yielding here; the related trajectory-flush surface is tracked by https://github.com/openclaw/openclaw/pull/77133. Add a single-line `CHANGELOG.md` fixes entry unless maintainers classify the final patch as purely internal diagnostics.
