## Summary

Keep open: the stale-drop idea is still useful and is not implemented on current main, but this branch is not merge-ready because it can silently consume legitimate non-completion exec notices and still lacks real behavior proof.

## What This Changes

## Best Possible Solution

Rebase onto current main's selected-consumption heartbeat path and drop only marked stale transient completion events while preserving cron, non-completion exec, and durable failure notices, then prove the behavior in a real Telegram or equivalent transport setup.

## Solution Assessment
