## Summary

Keep open: current main still lacks the stale-policy behavior, but this PR head can drop durable exec notices on the broad exec-event null-prompt path and still has only test output as proof.

## What This Changes

The PR adds an in-memory stale policy for transient async system events, filters stale entries after session activity advances, adjusts heartbeat exec-event skipping, and adds focused regression tests.

## Best Possible Solution

Keep the stale-policy direction, but rebase and narrow heartbeat skipping/consumption so only marked stale transient completions are dropped while durable started, denied, failed, timeout, cron, and routed events stay deliverable.

## Reproduction Assessment

Yes. Source inspection of the PR head gives a high-confidence path: enqueue an Exec denied or Exec started notice, run an exec-event heartbeat with no completion or cron event, and the null-prompt path can consume that queued notice without a reply turn.

## Solution Assessment

No. The stale-policy seam is maintainable, but the current exec-event skip condition is too broad and should be scoped to stale-policy-marked transient completion notices after session advance.
