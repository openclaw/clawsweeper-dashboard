## Summary

Keep open: the report matches the documented sub-agent completion contract and current source still has a gateway-dependent announce path with give-up behavior, while the long-uptime gateway stall is not fully diagnosed or superseded by a merged fix.

## What This Changes

The issue reports that completed sub-agents fail to announce results back to parent sessions after clustered gateway timeouts, leaving Discord/Slack sessions without the child output or a visible reply.

## Best Possible Solution

Keep the issue open, capture gateway event-loop and queue diagnostics under the reported workload, then land or replace the related sub-agent delivery fix that preserves completed child output through transient gateway stalls.

## Reproduction Assessment

No, not with high confidence end-to-end on current main. The source path from gateway announce timeout to failed delivery and retry give-up is clear, but the after-30-minute macOS gateway cascade still needs live reproduction or diagnostic traces.

## Solution Assessment

Unclear. Queue-first completion handoff and timeout tuning look relevant, but maintainers should confirm whether the root cause is requester-session queuing, event-loop starvation, gateway transport timeout budgeting, or a combination before closing this issue.
