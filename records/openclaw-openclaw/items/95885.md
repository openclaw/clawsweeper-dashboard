## What This Changes

This PR adds persisted SessionEntry byte-compaction retry metadata, suppresses repeat transcript-byte preflight compactions until threshold/file/material growth changes, reserves the new session keys, and adds regression tests.

## Best Possible Solution

Land this bounded byte-trigger hysteresis after the contributor adds redacted real runtime evidence, while keeping the stale transcript-usage loop work on its separate linked PR path.

## Reproduction Assessment

No high-confidence real setup reproduction was established in this read-only review. Source inspection shows current main can re-trigger byte compaction for the same unchanged over-threshold transcript, and the PR adds focused source tests for that path.

## Solution Assessment

Yes, subject to proof: recording retry state on SessionEntry is narrower than changing threshold semantics, and it preserves token-triggered compaction behavior. The stale transcript-usage loop is a separate issue already tracked by https://github.com/openclaw/openclaw/pull/81916.
