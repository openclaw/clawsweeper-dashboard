## Summary

Keep open: current main has no byte-trigger retry hysteresis, and this PR is a scoped bugfix, but the external contributor proof is still source/test-only and the persisted SessionEntry retry gate needs maintainer review before merge.

## What This Changes

The PR records byte-triggered compaction retry metadata on SessionEntry and suppresses repeated same-file byte-triggered preflight compactions until the threshold or file changes, or the transcript grows materially.

## Best Possible Solution

Land this after a maintainer accepts the SessionEntry-based retry state and the contributor adds redacted real-turn proof covering the first byte compaction and the suppressed next-turn byte retry.

## Reproduction Assessment

Yes. Source inspection shows current main repeats the byte-trigger condition whenever the active transcript remains above maxActiveTranscriptBytes, and the PR tests model the same-file retry after byte compaction without needing a product decision.

## Solution Assessment

Yes, with proof still missing. Storing last byte-trigger size/file/threshold next to compaction count is a narrow fix in the OpenClaw preflight owner path, and it preserves token-triggered compaction precedence.
