The PR adds Control UI chat-item projection and unit tests so successful `message(action="send")` tool calls can appear as assistant chat bubbles.

## Best Possible Solution

Keep the fix in `buildChatItems`, but project before hidden-tool filtering, recognize all supported successful send result shapes, dedupe existing assistant/delivery-mirror rows, and cover both `showToolCalls` modes.

## Reproduction Assessment

Yes. Source inspection shows current main has no successful message-send projection and that the PR's new projection is bypassed for raw assistant tool-call turns when `showToolCalls` is false.

## Solution Assessment

No, not yet. `buildChatItems` is the right boundary, but the implementation needs to run before tool hiding, use the current message-tool result contract, and dedupe existing mirror/final assistant rows.

  - confidence: 0.83
