## Summary

Keep this PR open: the requested Feishu keyword-trigger feature is not implemented on current main, but the proposed patch is not ready because it misses broadcast addressing, accepts blank triggers, and has no after-fix Feishu runtime proof.

## What This Changes

The PR adds a Feishu `mentionTriggers` config field and lets keyword-matched group messages pass the existing `requireMention` gate.

## Best Possible Solution

Define validated non-empty Feishu keyword triggers with clear account/group inheritance, carry one effective addressed state through ingress and broadcast dispatch, document the contract, add focused tests, and require real Feishu proof.

## Reproduction Assessment

Yes for the PR defects by source inspection: keyword-only turns can pass the new gate while `ctx.mentionedBot` remains false, and `ctx.content.includes("")` would match every message. I did not run a live Feishu setup.

## Solution Assessment

No. The patch is a useful start, but the maintainable fix needs non-empty trigger validation, a settled config scope, and a shared addressed-state decision used by both single-agent and broadcast paths.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Propagate keyword-triggered state into broadcast dispatch:** `extensions/feishu/src/bot.ts:1057`
  - body: When a group message passes this new gate through a keyword only, `ctx.mentionedBot` remains false. The later broadcast path still derives `activeAgentId` and `WasMentioned` from `ctx.mentionedBot`, so broadcast groups dispatch every agent as an observer and create no real Feishu reply for the triggered turn.
  - confidence: 0.91
- **[P2] Reject empty mention triggers before substring matching:** `extensions/feishu/src/config-schema.ts:160`
  - body: The schema accepts `""`, and `ctx.content.includes("")` is true for every message. A blank `mentionTriggers` entry therefore bypasses `requireMention` for all Feishu group messages instead of requiring a real keyword.
  - confidence: 0.92
