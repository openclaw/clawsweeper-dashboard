## Summary

Keep open: current main still does not implement Feishu keyword-based mention triggers, but this PR is not ready because the patch drops keyword-triggered turns in broadcast groups, permits blank triggers that bypass mention gating, and still lacks real Feishu behavior proof.

## What This Changes

The PR adds a Feishu `mentionTriggers` config field and treats configured text substrings as satisfying `requireMention` for group messages.

## Best Possible Solution

Define validated non-empty Feishu keyword triggers with clear account/group inheritance, feed one effective addressed state through ingress and single/broadcast dispatch, document the contract, and add focused tests plus redacted real Feishu proof.

## Reproduction Assessment

Yes for the PR defects by source inspection: keyword-only messages can pass the new gate with `ctx.mentionedBot` false while broadcast routing still uses `ctx.mentionedBot`, and blank strings satisfy substring matching. I did not run a live Feishu setup.

## Solution Assessment

No. The patch is a useful start, but current main routes Feishu mention gating through ingress policy and broadcast active-agent state, so the maintainable fix needs validated triggers, settled config scope, and one effective addressed decision shared across dispatch paths.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Propagate keyword-triggered state into broadcast dispatch:** `extensions/feishu/src/bot.ts:1057`
  - body: When this new gate lets a keyword-only group message through, `ctx.mentionedBot` remains false. The broadcast path still computes `activeAgentId` and `WasMentioned` from `ctx.mentionedBot`, so broadcast groups treat every agent as an observer and create no Feishu reply dispatcher for the triggered turn.
  - confidence: 0.91
- **[P2] Reject empty mention triggers before matching:** `extensions/feishu/src/config-schema.ts:160`
  - body: The schema accepts `""`, and the runtime uses `ctx.content.includes(t)`. A blank trigger therefore matches every group message and bypasses `requireMention`, so the config should reject or normalize empty trigger entries before matching.
  - confidence: 0.92
