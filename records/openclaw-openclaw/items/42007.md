## Summary

Keep this PR open: the Feishu keyword-trigger feature is not implemented on current main and no open canonical replacement owns the same work, but this patch still has correctness, security, product-contract, mergeability, and real-behavior-proof blockers.

## What This Changes

The PR adds a top-level Feishu `mentionTriggers` config array and lets group messages pass `requireMention` when their text contains a configured keyword.

## Best Possible Solution

Define one Feishu keyword-trigger contract with non-empty validation and group/account inheritance, carry an effective addressed state through ingress and broadcast dispatch, and add focused tests plus user-facing docs.

## Reproduction Assessment

Yes for the PR defects by source inspection: a keyword-only group message can pass the new gate while `ctx.mentionedBot` remains false, and `mentionTriggers: [""]` matches every message through `includes("")`. I did not run a live Feishu setup in this read-only review.

## Solution Assessment

No. The patch is a useful start, but the maintainable solution needs validated non-empty triggers, a resolved public config contract, and one effective addressed decision shared by single-agent and broadcast dispatch.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Mark keyword-triggered broadcasts active:** `extensions/feishu/src/bot.ts:1057`
  - body: This new gate lets keyword-only group messages continue with `requireMention` true while `ctx.mentionedBot` remains false. The later broadcast path derives `activeAgentId` and `WasMentioned` only from `ctx.mentionedBot`, so every broadcast agent is treated as an observer and no Feishu reply dispatcher is created for the keyword-triggered turn.
  - confidence: 0.9
- **[P2] Reject empty keyword triggers:** `extensions/feishu/src/config-schema.ts:160`
  - body: `z.array(z.string())` permits empty strings, and the runtime calls `ctx.content.includes(t)`. Because every string includes `""`, `mentionTriggers: [""]` makes all group messages pass `requireMention`, defeating the mention gate.
  - confidence: 0.92

## Security Review
