## Summary

Keep open: current main still lacks Feishu text keyword trigger support, and this PR is a plausible feature attempt, but the patch is not mergeable as-is because keyword-triggered broadcast groups would be accepted and then dispatched only as observers, blank triggers bypass the mention gate, and real Feishu after-fix proof is missing.

## What This Changes

The branch adds a top-level Feishu `mentionTriggers` array and uses substring matching in `handleFeishuMessage` to let keyword-only group messages pass `requireMention`.

## Best Possible Solution

Define one Feishu keyword-trigger contract with non-empty validation and group/account inheritance, feed a single effective addressed state through ingress and broadcast dispatch, then add focused tests and Feishu docs.

## Reproduction Assessment

Yes for the PR defects by source inspection: keyword-only group messages can pass the new gate while `ctx.mentionedBot` remains false, and `mentionTriggers: [""]` matches every message through `includes("")`. I did not run a live Feishu setup in this read-only review.

## Solution Assessment

No. The patch is a useful start, but the maintainable solution needs validated non-empty triggers, a resolved public config scope, and one effective addressed decision shared by single-agent and broadcast dispatch.

  - confidence: 0.92
