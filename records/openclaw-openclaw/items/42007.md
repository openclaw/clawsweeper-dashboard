## Summary

Keep open: current main still lacks Feishu keyword-trigger support, and this external PR is a useful feature attempt, but the patch is not mergeable because keyword-triggered broadcast messages can be accepted without producing a reply, empty triggers bypass mention gating, and real Feishu behavior proof is missing.

## What This Changes

The PR adds a Feishu `mentionTriggers` config array and lets matching group text bypass `requireMention`.

## Best Possible Solution

Define validated non-empty Feishu keyword triggers with clear top-level/account/group inheritance, feed one effective addressed state through ingress and broadcast dispatch, document the contract, and add focused tests plus real Feishu proof.

## Reproduction Assessment

Yes for the PR defects by source inspection: keyword-only messages can pass the new local gate while `ctx.mentionedBot` remains false, and `ctx.content.includes("")` would match every message if empty strings are accepted. I did not run a live Feishu setup.

## Solution Assessment

No. The patch is a useful start, but the maintainable fix needs non-empty trigger validation, a settled config scope, and one effective addressed decision shared by Feishu ingress and broadcast paths.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Propagate keyword-triggered state into broadcast dispatch:** `extensions/feishu/src/bot.ts:1057`
  - body: When this gate lets a keyword-only group message through, `ctx.mentionedBot` stays false. The broadcast path still computes `activeAgentId` and `WasMentioned` from `ctx.mentionedBot`, so broadcast groups treat every agent as an observer and create no Feishu reply dispatcher for the triggered turn.
  - confidence: 0.91
- **[P2] Reject empty mention triggers before substring matching:** `extensions/feishu/src/config-schema.ts:160`
  - body: The schema accepts `""`, and the runtime uses `ctx.content.includes(t)`. A blank trigger therefore matches every group message and bypasses `requireMention`, so the config should reject or normalize empty trigger entries before matching.
  - confidence: 0.92
