## Summary

Keep open: current main still lacks Feishu keyword-trigger support, but this PR is not ready because keyword-only matches do not propagate into broadcast active dispatch, blank triggers can bypass mention gating, and the external PR has no real Feishu behavior proof.

## What This Changes

The branch adds a Feishu `mentionTriggers` config array and uses substring matches to let keyword-only group messages pass the `requireMention` gate.

## Best Possible Solution

Define validated non-empty Feishu keyword triggers with clear account/group inheritance, feed one effective addressed state through ingress and broadcast dispatch, document the contract, and add focused tests plus real Feishu proof.

## Reproduction Assessment

Yes for the PR defects by source inspection: keyword-only turns can pass the new gate while `ctx.mentionedBot` remains false, and `ctx.content.includes("")` would match every message. I did not run a live Feishu setup.

## Solution Assessment

No. The patch is a useful start, but the maintainable fix needs non-empty trigger validation, a settled config scope, and a single effective addressed decision shared by single-agent and broadcast paths.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Propagate keyword-triggered state into broadcast dispatch:** `extensions/feishu/src/bot.ts:1057`
  - body: When a group message passes this new gate through `mentionTriggers` only, `ctx.mentionedBot` remains false. The later broadcast path still derives `activeAgentId` and `WasMentioned` from `ctx.mentionedBot`, so broadcast groups dispatch every agent as an observer and create no real Feishu reply for the triggered turn.
  - confidence: 0.91
- **[P2] Reject empty mention triggers before substring matching:** `extensions/feishu/src/config-schema.ts:160`
  - body: The schema accepts `""`, and `ctx.content.includes("")` is true for every message. A blank `mentionTriggers` entry therefore bypasses `requireMention` for all Feishu group messages instead of requiring a real keyword.
  - confidence: 0.92
