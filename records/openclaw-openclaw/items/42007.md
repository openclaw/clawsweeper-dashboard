## Summary

Keep open: current main does not implement Feishu keyword triggers, and the PR remains blocked by source-level correctness issues, a blank-trigger security concern, and missing real Feishu behavior proof.

## What This Changes

The PR adds a Feishu `mentionTriggers` config field and treats configured text keywords as satisfying group `requireMention` gating.

## Best Possible Solution

Define validated non-empty Feishu keyword triggers with clear account/group inheritance, then feed one effective addressed state through ingress, single-agent dispatch, and broadcast dispatch with focused tests and redacted Feishu proof.

## Reproduction Assessment

Yes for the PR defects by source inspection: keyword-only messages can pass the new gate while later broadcast dispatch still keys off `ctx.mentionedBot`, and blank string triggers satisfy substring matching. I did not run a live Feishu setup.

## Solution Assessment

No. The patch is a useful start, but the maintainable fix needs validated trigger values, settled config scope, and one effective addressed decision shared across Feishu activation and dispatch paths.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Propagate keyword-triggered state into broadcast dispatch:** `extensions/feishu/src/bot.ts:1050-1057`
  - body: When this new gate lets a keyword-only group message through, `ctx.mentionedBot` remains false. The broadcast path still computes `activeAgentId` and `WasMentioned` from `ctx.mentionedBot`, so broadcast groups dispatch every agent as an observer and create no Feishu reply dispatcher for the triggered turn.
  - confidence: 0.91
- **[P2] Reject empty mention triggers before matching:** `extensions/feishu/src/config-schema.ts:160`
  - body: The schema accepts `""`, and the runtime uses `ctx.content.includes(t)`. A blank trigger therefore matches every group message and bypasses `requireMention`, so empty or whitespace-only entries need to be rejected or normalized before matching.
  - confidence: 0.92

## Security Review

Status: needs_attention

Summary: Blank Feishu keyword triggers can make every group message satisfy mention gating.

Concerns:

- **[medium] Blank triggers bypass group mention gating:** `extensions/feishu/src/config-schema.ts:160`
  - body: Because the schema permits empty strings and runtime matching uses substring `includes`, an accidental blank trigger makes every Feishu group message pass the mention gate even when `requireMention` is meant to restrict processing.
  - confidence: 0.88
