## Summary

Keep open: current main still has no Feishu keyword-trigger support, but this PR is not merge-ready because the branch leaves broadcast keyword turns passive, permits blank triggers to bypass mention gating, and has no real Feishu runtime proof.

## What This Changes

The PR adds a top-level Feishu `mentionTriggers` config array and uses substring matching so keyword-only group messages can pass `requireMention`.

## Best Possible Solution

Define a validated non-empty Feishu keyword-trigger contract with clear inheritance, propagate one effective addressed state through ingress and broadcast dispatch, and cover it with focused tests, docs, and real Feishu proof.

## Reproduction Assessment

Yes for the PR defects by source inspection: a keyword-only message can pass the new gate while `ctx.mentionedBot` remains false, and `mentionTriggers: [""]` matches every message through `includes("")`. I did not run a live Feishu setup.

## Solution Assessment

No. The patch is a useful start, but the maintainable fix needs non-empty trigger validation, a settled config scope, and a single effective addressed decision shared by single-agent and broadcast paths.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Mark keyword-triggered broadcasts active:** `extensions/feishu/src/bot.ts:1057`
  - body: When `requireMention` is true and a message passes only through `mentionTriggers`, `ctx.mentionedBot` remains false. The later broadcast path still derives `activeAgentId` and `WasMentioned` from `ctx.mentionedBot`, so broadcast groups dispatch every agent as passive and do not create a real Feishu reply for the triggered turn.
  - confidence: 0.91
- **[P2] Reject empty mention triggers:** `extensions/feishu/src/config-schema.ts:160`
  - body: `z.array(z.string())` accepts an empty string, and `ctx.content.includes("")` is true for every message. A blank entry would therefore bypass `requireMention` for all Feishu group messages instead of requiring a real keyword.
  - confidence: 0.92
