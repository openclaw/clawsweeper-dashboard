## Summary

Keep this PR open, but do not merge it as-is. Current main still has no Feishu keyword-trigger support, and the PR addresses a real gap; however, the proposed patch only changes the first mention gate, leaving broadcast dispatch tied to native Feishu mentions, and it accepts blank trigger strings that can make every group message pass mention gating. Prior automated review comments already identified the broadcast bug, and related PR #33800 shows this is a recurring Feishu feature request rather than something implemented elsewhere.

## What This Changes

The PR adds a `channels.feishu.mentionTriggers` array and uses substring matches to let Feishu group text pass `requireMention` when no native bot mention is present.

## Best Possible Solution

Implement a Feishu-owned keyword or pattern trigger contract that validates non-empty entries, resolves the effective addressed/mentioned state once, carries that state through single-agent and broadcast dispatch including `WasMentioned`, and documents the final config shape with focused regression coverage.

## Reproduction Assessment

Yes. A source-level reproduction is clear: configure a Feishu broadcast group with `requireMention: true` and a keyword trigger, send a group text containing the keyword without a native bot mention, and the PR admits the message while broadcast activation still depends on `ctx.mentionedBot`; `mentionTriggers: [""]` also reproduces the blank-trigger bypass because `includes("")` is always true.

## Solution Assessment

No. The proposed patch is a useful start, but the maintainable fix should validate trigger entries and propagate one effective mention decision through the whole Feishu reply path instead of adding a local bypass before later paths that still require native mentions.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Mark keyword-triggered broadcasts active:** `extensions/feishu/src/bot.ts:1057`
  - body: When a keyword match admits a `requireMention` group message, `ctx.mentionedBot` remains false. The broadcast path still computes `activeAgentId` only from `ctx.mentionedBot || !requireMention`, so keyword-triggered broadcast messages run every agent as passive and create no Feishu reply dispatcher.
  - confidence: 0.9
- **[P2] Reject blank keyword triggers:** `extensions/feishu/src/config-schema.ts:160`
  - body: `mentionTriggers` accepts empty strings, and the runtime checks `ctx.content.includes(t)`. Since `includes("")` is always true, `mentionTriggers: [""]` makes every non-empty group message pass the `requireMention` gate.
  - confidence: 0.92

## Security Review

Status: needs_attention

Summary: No dependency, workflow, or supply-chain changes were found, but the diff introduces a config-controlled Feishu mention-gate bypass for blank trigger strings.

Concerns:

- **[medium] Blank triggers bypass group mention gating:** `extensions/feishu/src/config-schema.ts:160`
  - body: Because `z.array(z.string())` permits empty strings and `includes("")` always matches, an accidental blank trigger can make every Feishu group message eligible even when `requireMention` is intended to limit processing to addressed messages.
  - confidence: 0.88
