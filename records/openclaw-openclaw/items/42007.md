## Summary

Keep open: current main and the latest release still do not implement Feishu keyword mention triggers, but the PR’s current diff is incomplete and still lacks required real Feishu behavior proof.

## What This Changes

The PR adds a Feishu `mentionTriggers` config field and treats matching text keywords as satisfying group `requireMention` gating.

## Best Possible Solution

Implement validated non-empty Feishu keyword triggers with clear account/group inheritance, feed one effective addressed state through ingress and broadcast dispatch, and add focused tests plus redacted live Feishu proof.

## Reproduction Assessment

Yes for the PR defects by source inspection: keyword-only messages can pass the new gate while broadcast dispatch still keys off `ctx.mentionedBot`, and empty string triggers satisfy `includes`. I did not run a live Feishu setup in this read-only review.

## Solution Assessment

No. The patch is a useful start, but the maintainable fix needs validated trigger values, settled config scope, and one effective addressed decision shared across Feishu activation and dispatch paths.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Propagate keyword-triggered state into broadcast dispatch:** `extensions/feishu/src/bot.ts:1053-1057`
  - body: When this new gate lets a keyword-only group message through, `ctx.mentionedBot` remains false. The broadcast path still computes `activeAgentId` and `WasMentioned` from `ctx.mentionedBot`, so broadcast groups dispatch every agent as an observer and create no Feishu reply dispatcher for the triggered turn.
  - confidence: 0.91
- **[P2] Reject empty mention triggers before matching:** `extensions/feishu/src/config-schema.ts:160`
  - body: The schema accepts `""`, and the runtime uses `ctx.content.includes(t)`. A blank trigger therefore matches every group message and bypasses `requireMention`, so empty or whitespace-only entries need to be rejected or normalized before matching.
  - confidence: 0.92

## Security Review
