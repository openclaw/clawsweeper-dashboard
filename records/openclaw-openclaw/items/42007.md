## Summary

Keep open. The requested Feishu keyword-trigger capability is still absent from current main, but this PR is not mergeable as-is because it misses the broadcast addressed-state path, accepts blank trigger strings, lacks real Feishu proof, and still needs a maintainer choice on the public config contract.

## What This Changes

The PR adds a Feishu `mentionTriggers` config array and lets group messages pass `requireMention` when message text contains one of those configured keywords.

## Best Possible Solution

Define one Feishu keyword-trigger config contract with non-empty validation and group/account inheritance, carry a single effective addressed state through ingress and broadcast dispatch, and cover it with focused tests, docs, and changelog.

## Reproduction Assessment

Yes. Source inspection of the PR branch gives a high-confidence path: a keyword-only Feishu group message with `requireMention: true` passes the new gate while `ctx.mentionedBot` stays false, and `mentionTriggers: [""]` matches every message via `includes("")`.

## Solution Assessment

No. The patch is a useful start, but the maintainable solution should validate trigger entries, resolve the public config contract, and propagate one effective addressed decision through single-agent and broadcast dispatch.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Mark keyword-triggered broadcasts active:** `extensions/feishu/src/bot.ts:1057`
  - body: This new gate lets a keyword-only group message continue with `requireMention` true while `ctx.mentionedBot` remains false. The later broadcast path derives `activeAgentId` and `WasMentioned` only from `ctx.mentionedBot`, so every broadcast agent is treated as an observer with a no-op dispatcher and no Feishu reply is created for the keyword-triggered turn.
  - confidence: 0.9
- **[P2] Reject empty keyword triggers:** `extensions/feishu/src/config-schema.ts:160`
  - body: `z.array(z.string())` admits `""`, and the runtime calls `ctx.content.includes(t)`. Because every string includes the empty string, `mentionTriggers: [""]` makes all group messages pass `requireMention`, defeating the mention gate.
  - confidence: 0.92
