Full review comments:

- **[P1] Propagate keyword triggers into broadcast dispatch:** `extensions/feishu/src/bot.ts:1057`
  - body: When this gate lets a keyword-only group message through, `ctx.mentionedBot` remains false.   - confidence: 0.91

  - body: The schema accepts `""`, and runtime matching uses `ctx.content.includes(t)`. A blank trigger therefore matches every group message and bypasses `requireMention`, so empty or whitespace-only entries need to be rejected or normalized before matching.
  - confidence: 0.92
- **[P1] Avoid a second mention-trigger config path:** `extensions/feishu/src/config-schema.ts:160`
  - body: OpenClaw already documents `groupChat.mentionPatterns` for text-pattern mention triggers. Adding Feishu-only `mentionTriggers` creates a competing source of truth unless the PR proves the existing contract is insufficient and updates docs, inheritance, and tests for the new channel-specific field.
  - confidence: 0.82
