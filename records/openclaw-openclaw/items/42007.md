## Summary

Keep open: current main still lacks Feishu keyword trigger support, and this PR has merge-blocking broadcast and validation issues plus an unsettled public config contract.

## What This Changes

The PR adds a Feishu `mentionTriggers` config array and lets `requireMention` group text pass when message content contains a configured keyword.

## Best Possible Solution

Choose one Feishu trigger contract, validate non-empty trigger entries, carry a single effective addressed state through single-agent and broadcast dispatch, and add focused tests, docs, and a changelog entry.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: on the PR branch, a Feishu broadcast group with `requireMention: true` and a keyword-only trigger passes the new gate, then the unchanged broadcast path keeps `activeAgentId` null; `mentionTriggers: [""]` also matches every message because `includes("")` is always true.

## Solution Assessment

No. The proposed patch is a useful start, but the maintainable fix should validate trigger entries and propagate one effective mention/addressed decision through all Feishu dispatch paths after the public config shape is chosen.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Mark keyword-triggered broadcasts active:** `extensions/feishu/src/bot.ts:1057`
  - body: This new gate lets a keyword-only group message continue with `requireMention` true while `ctx.mentionedBot` remains false. The later broadcast path derives `activeAgentId` and `WasMentioned` only from `ctx.mentionedBot`, so every broadcast agent is treated as an observer with a no-op dispatcher and no Feishu reply is created for the keyword-triggered turn.
  - confidence: 0.9
- **[P2] Reject empty keyword triggers:** `extensions/feishu/src/config-schema.ts:160`
  - body: `z.array(z.string())` admits `""`, and the runtime calls `ctx.content.includes(t)`. Because every string includes the empty string, `mentionTriggers: [""]` makes all group messages pass `requireMention`, which defeats the mention gate.
  - confidence: 0.92

## Security Review
