## Summary

Keep open: current main has not implemented Feishu keyword triggers, and the PR still has blocking broadcast and blank-trigger issues plus a public config contract decision.

## What This Changes

The PR adds a top-level `channels.feishu.mentionTriggers` array and uses substring matching to let Feishu group text pass `requireMention` without a native bot mention.

## Best Possible Solution

Choose one Feishu trigger contract, validate non-empty entries, carry one effective addressed state through single-agent and broadcast dispatch, and document/test the final config shape.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: on the PR branch, a Feishu broadcast group with `requireMention: true` and a keyword-only trigger passes the new gate, then the unchanged broadcast path keeps `activeAgentId` null; `mentionTriggers: [""]` also matches every message because `includes("")` is always true.

## Solution Assessment

No. The proposed patch is a useful start, but the maintainable fix should validate trigger entries and propagate a single effective mention/addressed decision through all Feishu dispatch paths after the public config shape is chosen.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Mark keyword-triggered broadcasts active:** `extensions/feishu/src/bot.ts:1057`
  - body: When `mentionTriggers` admits a `requireMention` group message without a native @mention, `ctx.mentionedBot` remains false. The broadcast path still computes the active agent and `WasMentioned` from `ctx.mentionedBot`, so every broadcast agent runs as an observer with a no-op Feishu dispatcher and no visible reply.
  - confidence: 0.9
- **[P2] Reject empty keyword triggers:** `extensions/feishu/src/config-schema.ts:160`
  - body: `mentionTriggers` accepts empty strings, and the runtime checks `ctx.content.includes(t)`. Since `includes("")` is always true, `mentionTriggers: [""]` makes every group message pass `requireMention` instead of requiring an addressed trigger.
  - confidence: 0.92

## Security Review

Status: needs_attention

Summary: The diff introduces a config-controlled Feishu mention-gate bypass for blank trigger strings.

Concerns:

- **[medium] Blank triggers bypass group mention gating:** `extensions/feishu/src/config-schema.ts:160`
  - body: Because `z.array(z.string())` permits empty values and the runtime calls `includes(t)`, an accidental blank trigger would make every Feishu group text look addressed even when `requireMention` is intended to limit processing.
  - confidence: 0.88
