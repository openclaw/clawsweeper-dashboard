## Label Justifications

- `P2`: The report concerns a real but limited active-run control behavior with an existing hard-restart alternative.
- `impact:session-state`: The issue is about when steering text enters the active run transcript/context and whether the active session is restarted.

## Summary

Keep open. Current main still implements `/steer` as soft, boundary-based steering and keeps hard abort/restart behind `/redirect`, so this needs a maintainer behavior/UX decision rather than a cleanup close.

## What This Changes

Request that `/steer` interrupt the currently streaming run and restart with the steering message injected immediately.

## Best Possible Solution

Make the command contract explicit: keep `/steer` as boundary-based soft steering and route immediate restarts to `/redirect`, or deliberately change/rename the commands with matching docs, tests, and upgrade messaging.

## Reproduction Assessment

Yes, from source inspection: `/steer` is wired to `chat.send` with `deliver: false`, while hard restart is separately implemented by `/redirect` through `sessions.steer`. I did not run a live streaming provider scenario, so this is source-reproducible rather than live reproduced.

## Solution Assessment

No clear best fix yet: directly making `/steer` abort would override an intentional soft-steering contract and duplicate `/redirect`. Maintainers should first decide whether the issue is command semantics or misleading UI/help wording.
