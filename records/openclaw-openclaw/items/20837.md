## Summary

Keep open. Current main still exposes channel metadata in normal inbound/agent-run prompts, but `/context` and `/export-session` prompt reconstruction still drops the already-derived command channel and related runtime metadata; the linked implementation PR is closed unmerged.

## What This Changes

The issue requests exposing the originating communication channel to the agent, with optional selective reply routing by channel.

## Best Possible Solution

Thread existing channel/account runtime facts through `resolveCommandsSystemPromptBundle()` so command prompt reconstruction matches normal agent runs, while leaving delivery-channel routing host-controlled.

## Reproduction Assessment

Yes for the scoped remaining gap. Source inspection on current main shows `buildCommandContext()` derives a channel, while `resolveCommandsSystemPromptBundle()` drops that channel and related runtime metadata before prompt reconstruction.

## Solution Assessment

Yes for the awareness portion. The narrow maintainable solution is to reuse or extract the embedded-runner runtime prompt assembly for command reconstruction, without adding prompt-driven delivery-channel policy.
