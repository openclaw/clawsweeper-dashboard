## Summary

Keep open. Current main and v2026.5.12 still have the same scoped gap: normal inbound/agent-run prompts carry channel metadata, but `/context` and `/export-session` prompt reconstruction drops the already-derived command channel and related runtime metadata; the linked implementation PR is closed unmerged.

## What This Changes

The issue requests source-channel awareness in agent context and optionally channel-selective replies.

## Best Possible Solution

Thread existing channel/account runtime facts through `resolveCommandsSystemPromptBundle()` so reconstructed command prompts match normal agent runs, while leaving delivery-channel selection under host routing or a separate product follow-up.

## Reproduction Assessment

Yes for the scoped remaining gap. Source inspection on current main shows `buildCommandContext()` derives a channel, while `resolveCommandsSystemPromptBundle()` drops that channel and related runtime metadata before `/context` and `/export-session` prompt reconstruction.

## Solution Assessment

Yes for the awareness portion. The narrow maintainable solution is to reuse or extract the normal embedded-runner runtime prompt assembly for command reconstruction, without adding prompt-driven delivery-channel policy.
