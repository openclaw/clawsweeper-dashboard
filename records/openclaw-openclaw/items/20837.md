## Summary

Keep open. Current main still has the scoped slash-command prompt reconstruction gap: normal agent runs can pass channel/runtime metadata, but the command bundle used by `/context` and `/export-session` still builds runtime info with `unknown` host fields and no channel, shell, capabilities, or channel actions.

## What This Changes

The issue requests that agents receive communication-channel context and, separately, explores channel-specific reply behavior.

## Best Possible Solution

Thread the already-derived channel and account-scoped runtime facts through `resolveCommandsSystemPromptBundle()` so `/context` and `/export-session` match normal agent-run prompts, while leaving selective reply routing for separate product/API work.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path for the scoped remaining gap: build a command context with `OriginatingChannel`, then follow `/context` or `/export-session` into `resolveCommandsSystemPromptBundle()`, where that channel is dropped before runtime prompt assembly.

## Solution Assessment

Yes for the awareness portion. The narrow maintainable fix is to reuse or extract the normal embedded-runner runtime metadata assembly for command reconstruction, not to add new reply-routing behavior in this issue.
