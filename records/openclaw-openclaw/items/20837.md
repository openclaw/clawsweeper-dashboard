## Summary

Keep open. Current main already exposes channel metadata in the normal inbound run path, but slash-command prompt reconstruction still omits runtime channel, capabilities, shell, and channel actions; the related fix PR #21271 was closed unmerged.

## What This Changes

Request to make agents aware of the source communication channel and optionally support channel-selective replies.

## Best Possible Solution

Populate runtime metadata in resolveCommandsSystemPromptBundle() using the existing embedded-runner pattern, and leave channel-selective replies to a separate API/product discussion.

## Reproduction Assessment

Yes for the remaining command-path gap. Source inspection on current main shows command prompt reconstruction omits channel metadata while buildCommandContext() and the normal embedded runner already have it; I did not run tests in this read-only review.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse or extract the embedded-runner runtime prompt assembly for slash-command reconstruction, including account-aware capabilities, without changing reply-routing policy.

## Repair Work Prompt

Fix the remaining channel-awareness gap for this issue only. In src/auto-reply/reply/commands-system-prompt.ts, update resolveCommandsSystemPromptBundle() so slash-command prompt reconstruction passes real runtime metadata to buildSystemPromptParams(): host, os, arch, shell, normalized channel, account-aware capabilities, and supported channel actions/message guidance where available. Reuse or extract the existing embedded-runner pattern from src/agents/pi-embedded-runner/run/attempt.ts and use ctx.AccountId when resolving account-scoped channel capabilities. Do not implement selective reply routing, do not let the model choose delivery channels, and keep volatile per-message identifiers out of the system prompt. Add focused regression coverage in commands-system-prompt.test.ts and related inbound-meta tests if needed. Add a single-line CHANGELOG.md Fixes entry for slash-command context/export channel metadata if the change is user-facing.
