## Summary

Keep open. Current main still exposes channel metadata in the normal inbound/runtime prompt path, but the slash-command reconstruction bundle used by `/context` and `/export-session` still omits channel, capabilities, shell, and channel-action runtime metadata; the linked implementation PR is closed unmerged.

## What This Changes

Request that agents receive trusted source-channel context for inbound messages and, optionally, support channel-selective replies.

## Best Possible Solution

Keep this issue open until slash-command prompt reconstruction carries the same prepared channel/runtime metadata as normal agent runs, while leaving channel selection policy unchanged.

## Reproduction Assessment

Yes for the scoped remaining gap. Source inspection on current main shows `buildCommandContext()` derives a channel while `resolveCommandsSystemPromptBundle()` still omits channel and related runtime metadata from reconstructed prompts.

## Solution Assessment

Yes for the scoped remainder. The narrow maintainable fix is to reuse or extract the embedded-runner runtime prompt assembly in `resolveCommandsSystemPromptBundle()` without adding prompt-driven selective reply routing.
