## Summary

Keep open: current main still exposes channel metadata in the normal inbound/runner flow, but `/context` and `/export-session` prompt reconstruction still drops channel/runtime metadata; the linked repair PR is closed unmerged.

## What This Changes

The issue asks for agents to receive source-channel context and possibly support channel-selective replies.

## Best Possible Solution

Thread the already-derived channel and account-scoped runtime facts through `resolveCommandsSystemPromptBundle()` so command-rebuilt prompts match normal agent-run prompts; keep selective reply routing as a separate product/API follow-up.

## Reproduction Assessment

Yes, at source level. Build a command context with `OriginatingChannel`, then follow `/context` or `/export-session` into `resolveCommandsSystemPromptBundle()`, where that channel is dropped before runtime prompt assembly.

## Solution Assessment

Yes for the awareness portion. The narrow maintainable fix is to reuse or extract the normal embedded-runner runtime metadata assembly for command reconstruction, not to add new reply-routing behavior here.
