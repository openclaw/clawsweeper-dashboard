## Summary

Keep open: current main and v2026.6.8 expose channel metadata in the normal inbound and runner paths, but `/context` and `/export-session` still rebuild command system prompts through a bundle that omits channel/capability/shell/action runtime metadata. The only directly related repair, https://github.com/openclaw/openclaw/pull/21271, is closed unmerged, and the selective-reply-routing part still needs a maintainer product split.

## What This Changes

The issue asks OpenClaw to expose the source communication channel to agents and optionally support channel-selective replies.

## Best Possible Solution

Split the work: repair `resolveCommandsSystemPromptBundle()` to mirror or extract the embedded-runner runtime metadata assembly, and leave selective reply routing for a separate maintainer-designed channel/API follow-up.
