## Summary

Keep this open: current main already covers normal inbound channel metadata, but the slash-command prompt reconstruction path still omits channel-related runtime metadata, and the selective-reply-routing half remains a product/API decision.

## What This Changes

The issue requests agent awareness of the inbound communication channel and possible channel-selective reply routing.

## Best Possible Solution

Implement the remaining command prompt metadata repair in `resolveCommandsSystemPromptBundle()` and leave selective reply routing to a separate maintainer-sponsored API decision.

## Solution Assessment

Yes for the first part: reusing or extracting the embedded-runner runtime metadata pattern is the narrowest maintainable repair. No for selective reply routing as part of the same change; that needs separate product/API direction.
