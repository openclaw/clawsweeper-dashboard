## Summary

Keep open. Current main exposes channel metadata in the normal inbound/runtime path, but slash-command prompt reconstruction used by `/context` and `/export-session` still omits channel, capabilities, shell, and channel-specific prompt guidance; the linked implementation PR at https://github.com/openclaw/openclaw/pull/21271 is closed unmerged.

## What This Changes

Expose the inbound communication channel to agent context and, as a later optional direction, support channel-aware reply behavior.

## Best Possible Solution

Thread the existing channel/account runtime facts through `resolveCommandsSystemPromptBundle()` so `/context` and `/export-session` match normal run prompt metadata while keeping reply routing host-controlled.

## Reproduction Assessment

Yes for the scoped remaining gap. Current main shows the command context derives a channel, while `resolveCommandsSystemPromptBundle()` drops that channel and related runtime metadata before prompt reconstruction.

## Solution Assessment

Yes for the scoped remainder. The narrow maintainable fix is to reuse or extract the embedded-runner runtime prompt assembly for command reconstruction, with account-aware capabilities and no delivery-routing policy change.
