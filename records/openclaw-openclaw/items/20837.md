## Summary

Keep open. The discussion correctly identified that normal inbound channel metadata exists, but current main still drops runtime channel, capabilities, shell, and channel-action guidance when /context and /export-session reconstruct the system prompt; the related fix PR #21271 is closed unmerged.

## What This Changes

Request to expose the source communication channel to the agent and possibly allow channel-selective replies.

## Best Possible Solution

Populate command prompt runtime metadata from the existing embedded-runner pattern, including account-aware channel capabilities, and leave selective reply routing to a separate API/product discussion.

Yes for the remaining command-path gap. Source inspection on current main shows command context and normal runs carry channel metadata, while resolveCommandsSystemPromptBundle() still omits it from reconstructed prompts; I did not run tests in this read-only review.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse or extract the embedded-runner runtime prompt assembly for slash-command reconstruction, including account-aware capabilities, without changing reply-routing policy.
