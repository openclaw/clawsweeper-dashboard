## Summary

Keep open. Current main has channel metadata in the normal inbound/agent-run path, but slash-command prompt reconstruction still drops runtime channel, capabilities, shell, channel actions, and message-tool guidance for /context and /export-session; the related PR #21271 is closed unmerged.

## What This Changes

The issue requests exposing the source communication channel to the agent, with selective channel replies left as an optional follow-up idea.

## Best Possible Solution

Populate command prompt runtime metadata from the existing embedded-runner pattern, including account-aware channel capabilities, and leave reply-channel selection policy unchanged.

## Reproduction Assessment

Yes. Source inspection on current main shows command context already has a channel, while resolveCommandsSystemPromptBundle() still omits that channel and related runtime metadata from reconstructed prompts.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse or extract the embedded-runner runtime prompt assembly for slash-command reconstruction without adding new routing behavior.
