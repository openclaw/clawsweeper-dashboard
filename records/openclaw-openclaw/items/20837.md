## Summary

Keep open: current main and the latest release still expose channel metadata in the normal inbound and agent-run paths, but `/context` and `/export-session` prompt reconstruction still rebuild runtime metadata with `unknown` host fields and no channel, shell, capabilities, reaction guidance, or message-tool hints; the linked repair PR is closed unmerged.

## What This Changes

The issue requests agent-visible incoming communication-channel context, with optional future control over which channel receives replies.

## Best Possible Solution

Thread the already-derived channel and account-scoped runtime facts through resolveCommandsSystemPromptBundle so reconstructed command prompts match normal agent-run prompts, while leaving force-origin or selective reply routing to the separate routing-design track.

## Reproduction Assessment

Yes, at source level. Build a command context with OriginatingChannel, then follow /context or /export-session into resolveCommandsSystemPromptBundle, where the channel fact is dropped before runtime prompt assembly.

## Solution Assessment

Yes for the channel-awareness portion. The narrow maintainable fix is to reuse or extract the normal embedded-runner runtime metadata assembly for command prompt reconstruction, not to add prompt-directed selective reply routing here.
