## Summary

Keep open. Current main already exposes channel metadata in the normal inbound/runtime prompt path, but the slash-command reconstruction bundle used by `/context` and `/export-session` still rebuilds runtime info without channel, capabilities, shell, or channel-specific prompt guidance; the linked implementation PR is closed unmerged.

## What This Changes

The issue requests making agents aware of the inbound communication channel and, optionally, enabling channel-aware reply behavior.

## Best Possible Solution

Thread the existing prepared channel/account runtime facts through `resolveCommandsSystemPromptBundle()` so `/context` and `/export-session` match normal run prompt metadata, while keeping reply routing deterministic and host-controlled.

## Reproduction Assessment

Yes for the scoped remaining gap. Source inspection on current main shows the command context derives a channel, while `resolveCommandsSystemPromptBundle()` drops that channel and related runtime metadata before prompt reconstruction.

## Solution Assessment

Yes for the scoped remainder. The narrow maintainable fix is to reuse or extract the embedded-runner runtime prompt assembly for command reconstruction, without adding prompt-driven delivery-channel policy.
