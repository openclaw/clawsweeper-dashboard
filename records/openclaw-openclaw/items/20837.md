## Summary

Keep open: current main and v2026.6.6 still expose channel metadata in the normal inbound/runner paths but omit it when `/context` and `/export-session` rebuild the command system prompt; the only linked fix PR is closed unmerged, and selective reply routing still needs a maintainer product split.

## What This Changes

Request to expose incoming communication channel metadata to the agent and optionally let prompts or rules restrict replies to a specific channel.

## Best Possible Solution

Split the work: repair `resolveCommandsSystemPromptBundle()` to mirror or extract the embedded-runner runtime metadata assembly, and leave selective reply routing for a separate maintainer-designed channel/API follow-up.

## Solution Assessment
