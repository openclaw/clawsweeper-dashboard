## Summary

Keep open. Current main and the latest release still omit channel/capabilities/shell/action metadata from slash-command prompt reconstruction, while the normal inbound and embedded-runner paths already carry channel metadata; the linked fix PR is closed unmerged and no other canonical fix was found.

## What This Changes

The issue requests making the agent aware of the inbound communication channel and optionally supporting channel-selective replies.

## Best Possible Solution

Implement the missing runtime metadata in `resolveCommandsSystemPromptBundle()` using the existing embedded-runner/CLI runtime metadata pattern, and handle channel-selective reply routing as a separate product/API decision.

## Reproduction Assessment

Yes, at source level: the command context has a channel, but `resolveCommandsSystemPromptBundle()` still omits it when rebuilding prompts for `/context` and `/export-session`. A live Telegram/webchat run was not attempted because this review is read-only.

## Solution Assessment

Partly. The narrow metadata repair is the best first fix because it reuses existing runner-owned runtime prompt composition; channel-selective reply routing should stay out of that repair until maintainers decide the product/API shape.
