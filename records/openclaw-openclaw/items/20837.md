## Summary

Keep open. Current main still carries channel metadata in the normal inbound/run path, but `/context` and `/export-session` prompt reconstruction still drops the already-derived command channel and related runtime metadata; the linked implementation PR is closed unmerged.

## What This Changes

Expose the source communication channel to the agent, with optional follow-up behavior for channel-selective replies.

## Best Possible Solution

Thread existing channel/account runtime facts through `resolveCommandsSystemPromptBundle()` so `/context` and `/export-session` match normal run prompt metadata while leaving reply delivery routing host-controlled.

## Reproduction Assessment

Yes for the scoped remaining gap: source inspection on current main shows `buildCommandContext()` derives a channel, while `resolveCommandsSystemPromptBundle()` drops that channel and related runtime metadata before prompt reconstruction.

## Solution Assessment

Yes for the awareness portion. The narrow maintainable solution is to reuse or extract the embedded-runner runtime prompt assembly for command reconstruction, without adding prompt-driven delivery-channel policy.
