## Summary

Keep open. Current main still drops channel/runtime metadata in command prompt reconstruction even though the normal runner and prompt contract support it; the linked fix PR is closed unmerged and no canonical replacement was found.

## What This Changes

The issue requests that agents know which communication channel a message came from and optionally route replies by channel.

## Best Possible Solution

Thread the already-derived channel/account runtime facts through `resolveCommandsSystemPromptBundle()` so `/context` and `/export-session` prompt reconstruction matches normal agent runs, without changing reply routing policy.

## Reproduction Assessment

Yes for the scoped remaining gap. Source inspection on current main shows `buildCommandContext()` derives a channel, while `resolveCommandsSystemPromptBundle()` drops that channel and related runtime metadata before `/context` and `/export-session` prompt reconstruction.

## Solution Assessment

Yes for the awareness portion. The narrow maintainable fix is to reuse or extract the normal embedded-runner runtime prompt assembly for command reconstruction, while leaving prompt-directed delivery-channel policy to a separate product/API discussion.
