## Summary

Keep open. The PR targets a real current-main timeout gap, but it is blocked because supplied expired timeout budgets can turn into unbounded remote HTTP requests.

## What This Changes

The PR threads the configured remote embedding batch timeout through memory-host SDK HTTP helpers and OpenAI, Gemini, and Voyage batch upload/create/status/error-read/download phases, with docs, changelog, exports, and regression tests.

## Best Possible Solution

Keep the narrow budget-threading design and #49981 credit, but make explicitly supplied expired budgets fail closed before any request or retry proceeds without a timeout.

## Reproduction Assessment

Yes. Current-main source shows `remote.batch.timeoutMinutes` reaches provider batch execution but is not forwarded into upload/create/status/error-read/download HTTP calls; the PR-specific regression is source-reproducible by passing `postJsonWithRetry` a timeout function that returns `0`.

## Solution Assessment

No, not as-is. The implementation direction is maintainable, but the shared resolver/retry path must distinguish an omitted timeout from an exhausted supplied budget and fail closed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Fail closed when supplied timeout budgets expire:** `packages/memory-host-sdk/src/host/remote-http.ts:18-20`
  - body: `resolveRemoteHttpTimeoutMs` maps a supplied timeout function or number that resolves to `0` or less to `undefined`. In this PR, `undefined` means no timeout: `postJsonWithRetry` skips the budget-aware loop and `postJson` calls remote HTTP without a timeout, so an exhausted batch budget can still start an unbounded provider request.
  - confidence: 0.88

## Security Review

Status: needs_attention

Summary: The patch introduces a shared fail-open timeout-budget path that can turn an exhausted supplied budget into an unbounded remote HTTP request.

Concerns:

- **[medium] Expired supplied budget can disable HTTP timeout:** `packages/memory-host-sdk/src/host/remote-http.ts:18`
  - body: A supplied timeout function returning `0` or a numeric value `<= 0` resolves to `undefined`; callers then pass no timeout to guarded fetch or fall back to generic retry, which can let stalled provider requests run without the configured batch budget.
  - confidence: 0.86
