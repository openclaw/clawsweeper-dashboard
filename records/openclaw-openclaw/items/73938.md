## Summary

Keep open: the timeout plumbing is still useful and not present on current main, but this PR currently introduces a fail-open timeout resolver that can turn an exhausted supplied budget into an unbounded remote HTTP request.

## What This Changes

Threads remote embedding batch timeout budgets through memory-host SDK HTTP helpers and OpenAI, Gemini, and Voyage batch upload, create, polling, error-read, retry, and download paths with docs and tests.

## Best Possible Solution

Keep the budget plumbing, but make explicitly supplied expired budgets fail closed before any request or retry proceeds without a timeout, with zero/negative budget regression coverage.

## Reproduction Assessment

Yes from source. Current main does not forward the configured batch timeout into remote HTTP calls, and PR head is source-reproducible by supplying a timeout function that returns 0 before postJsonWithRetry runs.

## Solution Assessment

No, not as-is. The implementation direction is maintainable, but the shared resolver/retry path must distinguish omitted timeouts from exhausted supplied budgets and fail closed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Fail closed when supplied timeout budgets expire:** `packages/memory-host-sdk/src/host/remote-http.ts:18-20`
  - body: `resolveRemoteHttpTimeoutMs` maps a supplied timeout function or number that resolves to `0` or less to `undefined`. In this PR, `undefined` means no timeout, so an exhausted supplied budget can still start an unbounded provider request through postJsonWithRetry/postJson instead of failing closed.
  - confidence: 0.88

## Security Review

Status: needs_attention

Summary: The patch introduces a fail-open timeout-budget path that can disable remote HTTP request timeouts when a supplied budget is exhausted.

Concerns:

- **[medium] Expired supplied budget can disable HTTP timeout:** `packages/memory-host-sdk/src/host/remote-http.ts:18`
  - body: A supplied timeout function returning 0, or a numeric value <= 0, resolves to undefined; callers then pass no timeout to guarded fetch or fall back to generic retry, which can let stalled provider requests run without the configured batch budget.
  - confidence: 0.86
