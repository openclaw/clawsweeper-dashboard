## What This Changes

This PR updates the BlueBubbles webhook message normalizer to synthesize senderId `me` for `fromMe` group message payloads without handles, adds unit coverage, and adds a changelog entry.

## Best Possible Solution

Merge the narrow BlueBubbles message-normalizer fix after targeted validation and normal CI; keep broader unknown-sender group and reaction fallback semantics separate unless maintainers explicitly want that wider behavior.

## Reproduction Assessment

Yes. On current main, call `normalizeWebhookMessage` with a `new-message` payload containing `isGroup: true`, `isFromMe: true`, `handle: null`, and a group `chatGuid`; the normalizer has no group self fallback and returns null.

## Solution Assessment

Yes. Synthesizing `me` only for self-authored group messages preserves local outbound events without inventing senders for non-self group traffic, and the updated PR includes the required regression test and changelog entry.
