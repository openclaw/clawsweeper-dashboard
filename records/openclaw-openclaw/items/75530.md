## What This Changes

The PR changes BlueBubbles webhook message normalization to synthesize sender `me` for self-authored group messages without a sender handle, adds a focused regression test, and adds a changelog entry.

## Best Possible Solution

Merge the focused normalizer, regression test, and changelog entry after exact-head validation is green; keep broader sender-less group/reaction handling as separate BlueBubbles product work if maintainers want to revive it.

## Reproduction Assessment

Yes. On current main, a `normalizeWebhookMessage` payload with `type: "new-message"`, `isGroup: true`, `isFromMe: true`, `handle: null`, and a group `chatGuid` reaches the empty-sender path and returns null.

## Solution Assessment

Yes. The proposed fallback is the narrowest maintainable fix for the stated bug because it only synthesizes `me` for explicit self-authored group messages and preserves the existing guard against inferring non-self group senders from group chat GUIDs.
