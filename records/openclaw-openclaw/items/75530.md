## What This Changes

The PR changes the BlueBubbles webhook message normalizer to synthesize sender `me` for self-authored group messages without a handle, and adds a focused regression test plus a changelog entry.

## Best Possible Solution

Merge this focused normalizer fix after the failing preflight check is understood and targeted BlueBubbles test or CI proof is green; keep broader sender-less group/reaction behavior as separate product work unless maintainers intentionally expand scope.

## Reproduction Assessment

Yes. On current main, a `normalizeWebhookMessage` call with `type: "new-message"`, `isGroup: true`, `isFromMe: true`, `handle: null`, and a group `chatGuid` takes the no-group-fallback path and returns null.

## Solution Assessment

Yes. The PR applies the `me` fallback only for explicit self-authored group messages after explicit sender and DM chatGuid recovery fail, preserving the existing non-self missing-sender guard and matching downstream `fromMe` cache labeling.
