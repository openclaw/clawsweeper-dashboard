## Summary

Keep open: this PR targets a real Telegram streaming reply artifact and current main still has the source path that reuses reply metadata across preview rotations, but the current branch is not mergeable as-is because it conflicts with later Telegram streaming protections and still lacks after-fix live Telegram proof.

## What This Changes

The branch gates Telegram stream-preview reply metadata so single-use replies apply `reply_to_message_id` only to the first rotated preview, passes `replyToMode` into draft streaming, adds regression tests, and adds a changelog entry.

## Best Possible Solution

Land a current-main port of the single-use reply gating that preserves current topic, overflow, transport, and `replyToMode: "all"` contracts, backed by redacted live Telegram proof.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection shows current main still reuses reply params across `forceNewMessage()` rotations, and the linked bug report includes a Telegram screenshot of the visible `Deleted message` header.

## Solution Assessment

No for this branch as-is. The reply-gating idea is the right narrow direction, but it must be ported without reintroducing native draft transport, threadless DM-topic retry, or lost over-limit preview chaining.
