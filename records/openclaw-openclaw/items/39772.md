## What This Changes

The PR gates Telegram draft-stream preview reply metadata by `replyToMode`, passes `replyToMode` into dispatch, adds draft-stream regression tests, and adds a changelog entry.

## Best Possible Solution

## Reproduction Assessment

Yes, at source level: current main builds one `replyParams` object with `reply_to_message_id` and `forceNewMessage()` resets stream state without changing it, so a rotated preview can reuse the reply target. I did not establish a live Telegram reproduction.

## Solution Assessment
