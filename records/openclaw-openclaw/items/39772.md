## Summary

Keep this PR open: the single-use Telegram streaming reply gate is still a meaningful unique fix direction, but the current head should not merge because it reintroduces stale Telegram streaming behavior and still lacks real Telegram proof.

## What This Changes

The branch gates Telegram streaming preview `reply_to_message_id` use to the first single-use preview, passes `replyToMode` into draft stream creation, and adds Telegram regression tests plus a changelog entry.

## Best Possible Solution

Re-port only the single-use reply gating onto current main while preserving persistent send/edit streaming, DM topic fail-closed sends, overflow chaining, and `replyToMode: "all"`, then attach redacted live Telegram proof.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection shows current main can rotate Telegram preview messages through `forceNewMessage()` while reusing reply metadata, but the visible Telegram `Deleted message` artifact still needs live proof.

## Solution Assessment

No for the current PR head. The single-use reply gate is the right narrow direction, but this branch must be re-ported without reintroducing stale topic fallback, native draft transport, or stopped overflow chaining.
