## Summary

## What This Changes

The PR adds a commitment-only heartbeat mode, runs it after successful due HEARTBEAT.md task turns and scheduler commitment-session wakes, and updates the commitments regression test to expect both the task delivery and no-tools commitment delivery.

## Best Possible Solution

Keep the separate-turn design: HEARTBEAT.md tasks should stay tool-capable, then due inferred commitments for the same session should run in a commitment-only, no-tools heartbeat turn and update commitment status independently. Add the missing changelog entry before final review/merge.

## Solution Assessment

Yes functionally. The PR’s commitment-only follow-up pass is the narrowest maintainable fix because it preserves task tools while keeping commitment metadata out of tool-capable prompts; it is not complete for merge until the changelog entry is added.
