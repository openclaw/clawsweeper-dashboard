## What This Changes

The PR adds a commitment-only heartbeat path so due inferred commitments run after due tool-capable HEARTBEAT.md task turns, and updates the commitments regression test to expect both deliveries.

## Best Possible Solution

Land the separate-turn design after maintainer review: due HEARTBEAT.md tasks should keep normal tool capability, then same-session due commitments should run in a commitment-only heartbeat turn with tools disabled and independent status updates.

## Solution Assessment

Yes. The PR's commitmentOnly pass is the narrowest maintainable fix I found because it avoids mixing untrusted commitment metadata into the tool-capable task prompt while preventing recurring tasks from starving due commitments.
