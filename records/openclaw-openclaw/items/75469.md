## Summary

Keep open. Current main still has the same-session starvation path, and this PR is a narrow repair that separates tool-capable heartbeat task turns from no-tools commitment delivery without adding a blocking correctness finding.

## What This Changes

The PR adds a commitment-only heartbeat path, runs due commitments after successful tool-capable HEARTBEAT.md task turns, and updates the commitment regression test to expect both deliveries.

## Best Possible Solution

Keep the separate-turn design: due HEARTBEAT.md tasks should run with normal tool capability, then due inferred commitments for the same session should run in a commitment-only, no-tools heartbeat turn and update commitment status independently.

## Reproduction Assessment

Yes. A current-main code path with commitments enabled, heartbeat target not none, a due same-session commitment, and a due HEARTBEAT.md task reproduces the starvation by selecting the task prompt and leaving the commitment pending.

## Solution Assessment

Yes. The PR’s separate commitment-only pass is the narrow maintainable fix because it preserves task tools while keeping untrusted commitment metadata out of tool-capable prompts.
