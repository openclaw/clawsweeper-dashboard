## Summary

Keep open. Current main still has the same-session commitment starvation path when a due HEARTBEAT.md task wins prompt selection, and this PR is a narrow functional fix for that behavior. The patch looks correct, but it is missing the required user-facing changelog entry before merge.

## What This Changes

The PR adds a commitment-only heartbeat path, runs same-session due commitments after successful tool-capable heartbeat task turns, updates scheduler commitment runs to use that path, and extends the commitments regression test to expect both deliveries.

## Best Possible Solution

Keep the separate-turn design, add the missing Unreleased changelog fix entry, then merge after exact-head CI or the normal changed-gate proof passes.

## Reproduction Assessment

Yes. A current-main run with commitments enabled, heartbeat target last, a due same-session commitment, and a recurring due HEARTBEAT.md task follows the task-only prompt path and leaves the commitment pending; the existing test currently asserts that pending state.

## Solution Assessment

Yes, functionally. A commitment-only no-tools follow-up run after the task turn is the narrow maintainable fix because it preserves tool-capable tasks while keeping commitment metadata out of tool-capable prompts; the remaining issue is release-note coverage.
