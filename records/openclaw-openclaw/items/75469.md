## Summary

Keep open. Current main still has the reported same-session commitment starvation path, and the PR’s separate no-tools commitment pass is the right functional direction, but the user-facing fix is missing the required changelog entry and still needs exact-head merge checks.

## What This Changes

The PR adds a commitment-only heartbeat path after successful due HEARTBEAT.md task turns, routes scheduler commitment wakeups through that no-tools path, and updates the commitment regression test to expect both the task delivery and follow-up delivery.

## Best Possible Solution

Land the separate commitment-only, no-tools post-task heartbeat path with the regression coverage, add the missing Unreleased Fixes changelog bullet, and let exact-head CI decide merge readiness.

## Reproduction Assessment

Yes. On current main, commitments enabled with target last, a due same-session commitment, and a due HEARTBEAT.md task reproduces the issue because the task prompt path sets hasDueCommitments=false and the scheduler skips the default session follow-up pass.

## Solution Assessment

Yes for the code path. Running due commitments as a separate no-tools heartbeat turn after successful task turns is the narrow maintainable fix because it preserves normal task tools while keeping untrusted commitment metadata out of tool-capable prompts.
