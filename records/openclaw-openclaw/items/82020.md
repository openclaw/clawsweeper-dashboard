Land https://github.com/openclaw/openclaw/pull/90689 or an equivalent focused fix that makes status auth labels follow runtime provider-entry credential precedence for literal custom provider keys.

## Reproduction Assessment

Yes, at source level. Current main's display helper can choose env auth before literal custom-provider config while runtime auth resolves the provider-entry key first; I did not run a live Telegram flow in this read-only review.

## Solution Assessment
