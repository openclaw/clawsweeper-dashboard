## Summary

Keep open: current main still lacks downstream SSE heartbeats for the two OpenAI-compatible streaming endpoints, and this PR is relevant, but PR head still has a finite-interval bug, fake-timer cleanup gap, missing real behavior proof, and an unresolved stream-lifetime security policy question.

## What This Changes

The PR adds a `startSseHeartbeat` helper and wires SSE `: ping` comments into streaming `/v1/chat/completions` and `/v1/responses` responses.

## Best Possible Solution

Land an SSE-comment heartbeat only after finite interval validation, deterministic fake-timer cleanup, redacted proxy/curl proof, and an explicit stream lifetime or concurrency-limit decision.

## Reproduction Assessment

Yes, source-level. Current main has SSE streaming branches for both OpenAI-compatible endpoints with no periodic downstream bytes during quiet model windows, so an idle-timeout proxy can close before `[DONE]` or `response.completed`; I did not run a live proxy repro in this read-only review.

## Solution Assessment

No, not as-is. SSE comments are the right transport shape and match the existing session-history precedent, but this patch still needs finite interval normalization, test cleanup, real behavior proof, and maintainer agreement on stream lifetime or concurrency policy.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Validate the heartbeat interval before scheduling:** `src/gateway/http-common.ts:147`
  - body: `Math.max(1000, options?.intervalMs ?? 30_000)` still lets `NaN`, `Infinity`, and `-Infinity` reach `setInterval`; Node 22 normalizes those to a 1ms timer, so a future malformed override can create a high-frequency heartbeat per open stream. Normalize with `Number.isFinite` before clamping.
  - confidence: 0.91
- **[P3] Restore real timers after heartbeat tests:** `src/gateway/http-common.test.ts:302-304`
  - body: This suite enables fake timers in `beforeEach` but never restores real timers. In shared-worker or `--isolate=false` runs, later tests can inherit fake timers and stop exercising real timer behavior, so clear pending timers and call `vi.useRealTimers()` in cleanup.
  - confidence: 0.72
