## Summary

Keep open: this PR addresses a real, source-reproducible gateway streaming gap, but the current head still needs interval hardening, test timer cleanup, contributor real-behavior proof, and maintainer judgment on stream lifetime limits before merge.

## What This Changes

The PR adds a `startSseHeartbeat` helper, wires it into streaming `/v1/chat/completions` and `/v1/responses`, and adds focused unit tests for interval, backpressure, and error cleanup behavior.

## Best Possible Solution

Land an SSE-comment heartbeat only after finite interval normalization, fake-timer cleanup, redacted proxy/curl proof, and an explicit stream-limit policy decision.

## Reproduction Assessment

Yes. Current `main` has SSE streaming branches for both OpenAI-compatible endpoints with no periodic downstream bytes during quiet model windows, so an idle-timeout proxy can close the stream before `[DONE]` or `response.completed`; I did not run a live proxy repro in this read-only review.

## Solution Assessment

No, not as-is. SSE comments are the right transport shape and match existing session-history precedent, but this patch still needs interval validation, test cleanup, real behavior proof, and maintainer agreement on stream lifetime/concurrency policy.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Validate the heartbeat interval before scheduling:** `src/gateway/http-common.ts:147`
  - body: `Math.max(1000, options?.intervalMs ?? 30_000)` still lets `NaN`, `Infinity`, and `-Infinity` reach `setInterval`; Node 22 coerces those to 1ms, so a future malformed override can create a per-stream high-frequency heartbeat. Normalize with `Number.isFinite` before clamping.
  - confidence: 0.91
- **[P3] Restore real timers after heartbeat tests:** `src/gateway/http-common.test.ts:302-304`
  - body: This suite calls `vi.useFakeTimers()` in `beforeEach` and never restores real timers. In shared-worker or `--isolate=false` runs, later tests can inherit fake timers and stop exercising real timer behavior, so clear pending timers and call `vi.useRealTimers()` in cleanup.
  - confidence: 0.72

## Security Review

Status: needs_attention
