## Summary

Keep open. Current main still lacks downstream SSE heartbeats on the two OpenAI-compatible streaming routes, so the PR addresses a real gap, but the proposed patch still has a blocking interval-validation issue, a test timer leak, a missing user-facing changelog entry, and an unresolved stream-lifetime security posture decision.

## What This Changes

The PR adds a reusable `startSseHeartbeat()` helper with tests and wires it into the streaming `/v1/chat/completions` and `/v1/responses` gateway paths.

## Best Possible Solution

Ship an SSE-comment heartbeat for the two streaming endpoints with finite interval normalization, deterministic fake-timer cleanup in tests, the required changelog entry, preserved backpressure/error handling, and an explicit gateway-layer decision on whether stream duration or concurrency limits should accompany the transport keepalive.

## Reproduction Assessment

Yes. Current main is source-reproducible: both streaming endpoints set SSE headers and then only write model/final events, so a proxy idle timeout shorter than a silent reasoning window can truncate the stream. The PR's interval bug is also directly reproducible with Node 22 coercing non-finite timer delays to 1 ms.

## Solution Assessment

Unclear as-is. SSE comment heartbeats are the right transport-level direction and match existing session-history precedent, but this patch needs small hardening fixes plus maintainer security/product judgment before it is the narrowest maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Validate heartbeat intervals before scheduling:** `src/gateway/http-common.ts:147`
  - body: `Math.max(1000, options?.intervalMs ?? 30_000)` leaves `NaN`, `Infinity`, and `-Infinity` unnormalized. Node coerces those delays to a 1 ms interval, so a malformed override can turn each open SSE stream into a high-frequency write loop despite the intended clamp. Normalize with `Number.isFinite` and fall back to the default before calling `setInterval`.
  - confidence: 0.9
- **[P3] Restore real timers after heartbeat tests:** `src/gateway/http-common.test.ts:300-302`
  - body: The new heartbeat suite enables fake timers in `beforeEach` but never restores real timers. In shared-worker Vitest runs, later tests in this file can inherit fake timers and stop exercising real timer behavior or hang. Add cleanup that clears pending timers and calls `vi.useRealTimers()`.
  - confidence: 0.68
- **[P3] Add the required changelog entry:** `src/gateway/openai-http.ts:639`
  - body: This is a user-facing gateway reliability fix, but the PR only changes gateway source and tests. Repo policy requires a `CHANGELOG.md` entry for user-facing fixes, credited to the contributor, before merge.
  - confidence: 0.86
