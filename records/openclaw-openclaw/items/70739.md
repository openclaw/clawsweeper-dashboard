## Summary

Keep open. Current main still lacks downstream SSE heartbeats for the OpenAI-compatible streaming endpoints, and the PR is useful source work, but the patch is not merge-ready because the new interval clamp accepts non-finite values, the tests leave fake timers active, and the stream-lifetime security posture needs maintainer review before proxy idle caps are bypassed.

## What This Changes

The PR adds a reusable SSE heartbeat helper with unit tests and calls it from the streaming `/v1/chat/completions` and `/v1/responses` gateway paths.

## Best Possible Solution

Land the SSE-comment heartbeat after normalizing the interval to a finite bounded value, restoring real timers in the new tests, preserving the backpressure/error cleanup, and making an explicit maintainer call on whether streaming endpoints also need lifetime or concurrency limits.

## Reproduction Assessment

Yes. Current-main source shows both target streaming endpoints can go quiet after setting SSE headers, and the PR's interval bug is reproducible with Node 22 by passing non-finite timer values that become 1ms intervals.

## Solution Assessment

Unclear as-is. SSE comments are the right transport-level direction and match the existing session-history pattern, but this patch needs the interval/test fixes and a stream-limit security decision before it is the narrowest safe mergeable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Validate the heartbeat interval before scheduling:** `src/gateway/http-common.ts:147`
  - body: `Math.max(1000, options?.intervalMs ?? 30_000)` does not normalize `NaN` or infinities, and Node coerces those durations to a 1ms interval. If a future caller builds the override from config or env, each open stream can spin at 1ms despite the intended clamp; validate with `Number.isFinite` and fall back to the default before calling `setInterval`.
  - confidence: 0.9
- **[P3] Restore real timers after heartbeat tests:** `src/gateway/http-common.test.ts:302-303`
  - body: This suite switches to fake timers in `beforeEach` but never restores real timers. In shared Vitest workers, later tests can inherit fake timers and stop exercising real timeout behavior, so add cleanup that clears pending timers and calls `vi.useRealTimers()`.
  - confidence: 0.68

## Security Review

Status: needs_attention

Summary: The diff has no supply-chain changes, but it changes stream lifetime behavior in a way maintainers should review before merge.

Concerns:

- **[medium] Heartbeat bypasses proxy idle caps without stream limits:** `src/gateway/openai-http.ts:646`
  - body: The PR starts heartbeats unconditionally on authenticated SSE streams. Backpressure handling prevents heartbeat buffering, but current gateway code shows auth/body checks and failed-auth rate limiting rather than a maximum stream lifetime or per-subject/IP concurrent-stream cap, so exposed or `auth.mode: none` deployments lose proxy idle timeout as a fallback resource limiter.
  - confidence: 0.74
