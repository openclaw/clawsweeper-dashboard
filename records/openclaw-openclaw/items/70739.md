## Summary

Keep open: this PR targets a real source-reproducible SSE idle-timeout gap, but the current head still has a finite-interval normalization bug, a fake-timer cleanup leak in tests, missing after-fix real behavior proof, and a stream lifetime/concurrency policy concern that needs maintainer judgment.

## What This Changes

The PR adds a `startSseHeartbeat` helper and wires SSE `: ping` comments into the streaming `/v1/chat/completions` and `/v1/responses` gateway handlers with unit coverage.

## Best Possible Solution

Land a hardened SSE-comment heartbeat that normalizes invalid intervals, cleans test timer state, includes redacted real proxy proof, and fits an explicit stream-limit policy.

## Reproduction Assessment

Yes. Current main sets SSE headers for both OpenAI-compatible streaming endpoints but emits no periodic bytes during quiet upstream windows, so a proxy with a shorter idle timeout can close the stream before `[DONE]` or `response.completed`.

## Solution Assessment

Unclear as-is. SSE comments are the right transport shape and match current session-history precedent, but this patch needs interval hardening, test cleanup, real proxy proof, and a stream-limit policy decision before it is the maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Validate the heartbeat interval before scheduling:** `src/gateway/http-common.ts:147`
  - body: `Math.max(1000, options?.intervalMs ?? 30_000)` leaves `NaN`, `Infinity`, and `-Infinity` unsafe; Node coerces those delays to a 1ms interval, so a malformed override can turn each open stream into a high-frequency write loop. Normalize with `Number.isFinite` before clamping and fall back to the default when invalid.
  - confidence: 0.9
- **[P3] Restore real timers after heartbeat tests:** `src/gateway/http-common.test.ts:302-304`
  - body: This suite enables fake timers in `beforeEach` but never restores real timers. In shared-worker or `--isolate=false` runs, later tests in the file can inherit fake timers and stop exercising real timer behavior, so add cleanup that clears pending timers and calls `vi.useRealTimers()`.
  - confidence: 0.68

## Security Review

Status: needs_attention

Summary: No supply-chain changes were found, but the diff extends streaming connection lifetime without a visible max-duration or concurrent-stream policy.
