## Summary

Keep open: the gateway heartbeat fix is plausible and current main still lacks downstream SSE keepalives, but this PR still has a timer-normalization bug, fake-timer cleanup issue, missing observable real behavior proof, and an unresolved stream-lifetime/concurrency security decision.

## What This Changes

Adds an SSE heartbeat helper and wires `: ping` comments into the streaming `/v1/chat/completions` and `/v1/responses` gateway paths, with unit coverage.

## Best Possible Solution

Land a hardened SSE-comment heartbeat after finite interval normalization, timer cleanup, observable proxy/curl proof, and an explicit decision on stream lifetime or concurrency limits.

## Reproduction Assessment

Yes. This is source-reproducible: current main’s streaming `/v1/chat/completions` and `/v1/responses` branches set SSE headers but do not emit periodic bytes during quiet model windows, so an idle-timeout proxy can close the stream before `[DONE]` or `response.completed`.

## Solution Assessment

Unclear as-is. SSE comments match the right transport shape and existing session-history precedent, but the patch needs finite interval validation, test timer cleanup, real behavior proof, and a maintainer security decision before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Validate the heartbeat interval before scheduling:** `src/gateway/http-common.ts:147`
  - body: `Math.max(1000, options?.intervalMs ?? 30_000)` leaves `NaN`, `Infinity`, and `-Infinity` unchanged; Node coerces those delays to 1ms, so any malformed override can turn each open SSE stream into a high-frequency write loop. Normalize with `Number.isFinite` before clamping and scheduling.
  - confidence: 0.9
- **[P3] Restore real timers after heartbeat tests:** `src/gateway/http-common.test.ts:301-304`
  - body: This suite enables fake timers in `beforeEach` but never restores real timers. In shared-worker runs, later tests in the file can inherit fake timers and stop exercising real timer behavior, so add cleanup that clears pending timers and calls `vi.useRealTimers()`.
  - confidence: 0.68

## Security Review

Status: needs_attention

Summary: The diff has no supply-chain changes, but it changes stream lifetime behavior without a visible stream-duration or concurrency policy.
