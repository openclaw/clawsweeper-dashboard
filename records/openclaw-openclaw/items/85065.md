## Summary

Current main and shipped release evidence show the reported CLI-runtime alias path no longer treats `claude-cli` as a missing embedded harness; the remaining non-atomic registry lifecycle race is tracked separately.

## What This Changes

The issue asks for `claude-cli` and related CLI runtime aliases to stop throwing `MissingAgentHarnessError` when harness selection sees them during resume or compaction paths.

## Best Possible Solution

Keep the shipped CLI-runtime alias passthrough, and leave the separate registry lifecycle race tracked in https://github.com/openclaw/openclaw/issues/86342.

## Reproduction Assessment

Yes at source/tag level: `v2026.5.22` still threw for an explicit `claude-cli` runtime alias in selector code, while `v2026.5.26` and current main route matching CLI aliases before the throw. I did not rerun the live idle-resume race in this read-only review.

## Solution Assessment

Yes. Reusing provider-owned CLI runtime alias metadata at the shared harness-selection boundary is the narrow maintainable fix, while unknown and provider-mismatched runtime IDs still fail closed.
