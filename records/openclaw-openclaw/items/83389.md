## Summary

Keep open: current `main` still maps the documented code-mode timeout and worker-runtime-unavailable paths to `internal_error`, and the paired open fix PR should be reviewed or landed before this issue closes.

## What This Changes

The issue asks code mode to return documented `timeout` and `runtime_unavailable` error codes for known worker timeout and runtime-load failures instead of collapsing them to `internal_error`.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/83444 or an equivalent narrow fix that maps known timeout and worker-runtime failures to the documented codes, then close this issue with merge and test proof.

## Reproduction Assessment

Yes. Source inspection on current `main` shows both the timeout and worker-error paths still return `internal_error` despite the documented `timeout` and `runtime_unavailable` codes.

## Solution Assessment

Yes. The best fix is narrow: preserve the existing failure behavior while classifying these known paths with their documented codes and adding regression assertions for `details.code`.
