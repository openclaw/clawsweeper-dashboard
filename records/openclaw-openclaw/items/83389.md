## Summary

This looks like a valid, narrow code-mode bug on current main: the documented error-code contract includes `timeout` and `runtime_unavailable`, but the relevant worker timeout and worker-load failure paths still return `internal_error`.

## What This Changes

The issue requests that known code-mode timeout and worker-load failures surface their documented `CodeModeErrorCode` values instead of `internal_error`.

## Best Possible Solution

Map known code-mode failure classes to explicit documented codes and add focused regression assertions for timeout and worker-runtime-unavailable results, leaving `internal_error` only for unexpected failures.
