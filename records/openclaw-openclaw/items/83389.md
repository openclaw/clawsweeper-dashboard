## What This Changes

The issue requests that code mode return documented `timeout` and `runtime_unavailable` error codes for known worker timeout and load/unavailable failures instead of `internal_error`.

## Best Possible Solution

Land one canonical focused fix that preserves existing failure behavior but maps known worker timeout, load failure, and unavailable-runtime paths to the documented codes with regression tests.

## Reproduction Assessment

Yes. Source inspection on current main shows the infinite-loop timeout and worker load/error paths are known failure branches that still serialize as `internal_error`.

## Solution Assessment

Yes. The narrow maintainable fix is to preserve the same failed-result behavior while classifying these known paths as `timeout` or `runtime_unavailable` and leaving unexpected failures as `internal_error`.
