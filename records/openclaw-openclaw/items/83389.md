## Summary

Keep open: current `main` and the latest release still return `internal_error` for the known code-mode timeout and worker-runtime failure paths, and the open fixing pull request https://github.com/openclaw/openclaw/pull/83444 has closing syntax for this issue, so the issue should wait for that PR to land or be superseded.

## What This Changes

The issue asks code mode to return documented `timeout` and `runtime_unavailable` error codes for known worker timeout and runtime-load failure paths instead of collapsing them to `internal_error`.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/83444, or an equivalent narrow patch, so the documented structured codes are emitted with regression coverage before this issue closes.

## Reproduction Assessment

Yes. Source inspection on current main shows `while (true) {}` reaches a timeout path that is serialized as `internal_error`, and worker construction/load errors are also returned as `internal_error`; the existing test covers the timeout message but not the structured code.

## Solution Assessment

Yes. The maintainable fix is to preserve the existing failure behavior while mapping known timeout and runtime-unavailable worker failures to the documented codes, leaving `internal_error` for unexpected failures.
