## Summary

Keep open: current main still emits `internal_error` for the reported code-mode timeout and worker-runtime failure paths, and two open PRs explicitly close this issue but have not merged yet.

## What This Changes

The issue asks code mode to return documented `timeout` and `runtime_unavailable` error codes instead of `internal_error` for known worker failure paths.

## Best Possible Solution

Land one focused fix that maps known worker timeout and runtime-unavailable failures to the documented codes with regression coverage, then close this issue and supersede the duplicate PR.

## Reproduction Assessment

Yes. Source inspection on current main shows `while (true) {}` reaches timeout handling that is serialized as `internal_error`, and worker load errors also return `internal_error` despite the documented `runtime_unavailable` code.

## Solution Assessment

Yes. The narrow maintainable solution is to preserve existing failure behavior while classifying these known paths with documented codes and leaving `internal_error` for unexpected failures.
