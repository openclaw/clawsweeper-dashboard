## What This Changes

The issue asks code mode to return documented `timeout` and `runtime_unavailable` error codes for known worker timeout and runtime-load failure paths instead of collapsing them to `internal_error`.

## Best Possible Solution

## Reproduction Assessment

Yes. Source inspection on current `main` shows the timeout and worker-error paths still serialize known failures as `internal_error`, and the existing infinite-loop test checks only timeout wording rather than the structured code.

## Solution Assessment

Yes. The narrow maintainable fix is to preserve the existing failed-result behavior while classifying known timeout and runtime-unavailable cases with the documented codes and adding regression assertions.
