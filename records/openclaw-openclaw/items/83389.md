## Summary

Keep open: current main still routes known code-mode timeout and worker-runtime failures through `internal_error`, while the docs expose specific `timeout` and `runtime_unavailable` codes.

## What This Changes

The issue requests that code-mode `exec` return documented structured error codes for known timeout and runtime-unavailable failures instead of collapsing them to `internal_error`.

## Best Possible Solution

Map known timeout and worker-runtime-unavailable failures to the documented code-mode error codes and add focused regression assertions for the structured `code` field.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `while (true) {}` reaches the QuickJS timeout branch and is serialized as `internal_error`, while worker load failures are handled by the parent worker error handler that also returns `internal_error`.

## Solution Assessment

Yes. The narrow fix is to preserve existing failure behavior but classify these known failure paths with the documented codes, leaving `internal_error` for unexpected worker responses, exits, and uncaught failures.
