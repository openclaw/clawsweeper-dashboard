## Summary

Keep open. Current main still accepts and preserves `audio` and `video` in configured model input metadata while contributor/operator docs and the normalized model-catalog type only describe text/image/document behavior, so this remains an unresolved config/API contract decision rather than conservative cleanup.

## What This Changes

Request to add validation guardrails, tests, CI failure, and docs for unsupported `models.providers.*.models[].input` modality configurations.

## Best Possible Solution

Define one configured-model input contract, then enforce it consistently across schema, exported types, catalog normalization, tests, generated/public docs, and CI.

## Reproduction Assessment

Yes. Source inspection on current main shows `input: ["audio"]` or `input: ["video"]` is accepted by the schema/type path and preserved by configured-catalog normalization.

## Solution Assessment

Unclear. The requested guardrail is plausible, but a direct schema rejection would conflict with the current public type and preservation test, so maintainers should choose the contract first.
