## Best Possible Solution

Define one explicit configured-model input contract, then align schema, exported types, catalog normalization, runtime capability checks, tests, generated/public docs, and CI around that contract.

Yes, source inspection gives a high-confidence current-main reproduction path: configure `models.providers.*.models[*].input` with `audio` or `video`, and the schema/type/catalog path accepts and preserves it rather than rejecting it as unsupported.

## Solution Assessment

Unclear. The requested guardrail is plausible, but direct schema rejection is not the best fix until maintainers choose whether shipped `audio`/`video` values are invalid, future-compatible metadata, or a separate capability contract.
