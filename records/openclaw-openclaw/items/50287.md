## Summary

Keep open. Current main and v2026.6.8 still accept or preserve `audio`/`video` configured model-input metadata, and the remaining guardrail needs a maintainer product/API decision before it is safe to tighten validation.

## What This Changes

The issue requests schema, CI, tests, and docs guardrails that reject unsupported `models.providers.*.models[*].input` modality combinations before deployment.

## Best Possible Solution

Define one explicit configured-model input contract, then enforce it consistently across schema, exported types, generated schema, catalog normalization, tests, docs, and any needed doctor migration.

## Reproduction Assessment

Yes, at source level: current main and `v2026.6.8` accept or preserve `audio` and `video` in configured model input metadata. I did not run tests or mutate the checkout because this review is read-only.

## Solution Assessment

Unclear as an implementation: the guardrail is aligned with safer validation, but maintainers need to decide whether `audio`/`video` should be rejected, migrated, or given a supported runtime meaning before code changes are safe.
