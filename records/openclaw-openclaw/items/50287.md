## What This Changes

## Best Possible Solution

Define one explicit configured-model input contract, then align schema, exported types, generated schema/docs, catalog normalization, tests, and any needed doctor/migration diagnostics around that contract.

## Solution Assessment

Unclear. A validation guardrail is directionally useful, but a blanket rejection is not the best fix until maintainers decide whether `audio` and `video` are invalid chat inputs, valid custom/future metadata, or a separate media-understanding contract.
