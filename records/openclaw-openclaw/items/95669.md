## Summary

Keep open: the PR targets a real Mattermost session split, but the current patch treats the resolver's broad `group` bucket as authoritative and can re-key public channels into group sessions; it also lacks real Mattermost behavior proof.

## What This Changes

The branch changes Mattermost outbound session routing to use `resolvedTarget.kind === "group"` for peer/chatType/from and adds a unit test for that resolved-group route.

## Best Possible Solution

Mattermost session routing should preserve public `O` channels as `channel` while using authoritative type metadata or plugin-owned target resolution to route private/group channels as `group`, with tests and redacted live proof for both paths.

## Reproduction Assessment

Yes for the PR defect by source inspection: Mattermost directory resolution can return `kind: "group"` for public channels, while inbound public channels still map to `channel`. I did not run a live Mattermost instance.

## Solution Assessment

No. The fix belongs in the Mattermost plugin boundary, but this implementation uses a broad resolver kind instead of authoritative channel type, so it is too broad for the reported private-channel bug.
