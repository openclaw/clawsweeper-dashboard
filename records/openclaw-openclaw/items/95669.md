## Summary

## What This Changes

The PR changes Mattermost outbound session routing so resolved group targets use the `group` session namespace while keeping the Mattermost delivery target as `channel:<id>`.

## Best Possible Solution

Use Mattermost-owned authoritative channel type or equivalent resolved metadata so only proven `P`/`G` channels become `group`, public `O` channels remain `channel`, and the fix has focused tests plus redacted real behavior proof.

## Reproduction Assessment

Yes. Source inspection shows current main can produce `resolvedTarget.kind: "group"` for public Mattermost directory targets while inbound `O` channels map to `channel`; the PR then keys those public sends as `group`.

## Solution Assessment

No. Preserving resolved `group` is only safe after the resolver has proved Mattermost channel type `P` or `G`; as submitted, the route trusts ambiguous metadata.
