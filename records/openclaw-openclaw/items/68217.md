## Summary

Keep this PR open. Current main has not adopted the proposed scheme-guard hardening, the PR diff is narrow and appears correct, and the remaining action is ordinary maintainer review rather than conservative cleanup or a replacement repair.

## What This Changes

The PR updates the docs-i18n localized link rewriter to use bounded case-insensitive scheme prefix checks, adds `vbscript:` to the explicit scheme guard, and adds regression cases for `vbscript:` and mixed-case `Javascript:` hrefs.

## Best Possible Solution

Merge this PR or an equivalent tiny patch if maintainers want the docs-i18n scheme classifier to explicitly handle case-insensitive schemes and `vbscript:`, keeping the change confined to `scripts/docs-i18n` with focused regression coverage.

## Reproduction Assessment

Yes. The source-level reproduction is `routeIndex.localizeURL` calling the current case-sensitive `hasURLScheme`; local tests were not run because this was a read-only review, but the PR head has green check-runs for the relevant repository gates.

## Solution Assessment

Yes. The bounded `strings.EqualFold` prefix helper is a narrow maintainable fix for the stated hardening and avoids the full-string lowercase allocation discussed in review; a broader URL parser would be more invasive for this script.
