## What This Changes

The PR updates `scripts/docs-i18n` localized URL scheme detection to use bounded case-insensitive prefix checks, adds `vbscript:`, and adds two localized-link cases.

## Best Possible Solution

Merge this tiny helper hardening after contributor proof is added; direct `hasURLScheme` table coverage would be a useful landing polish but is not a blocking correctness defect.

## Reproduction Assessment

Yes at source level: current main's `hasURLScheme` is case-sensitive and omits `vbscript:`. A user-visible bad rewrite path is not established because non-route targets are returned unchanged later in `localizeURL`.

## Solution Assessment

Yes: bounded `strings.EqualFold` prefix checks are a narrow maintainable way to harden this classifier without lowercasing full URL targets. A direct helper-level table test would improve coverage but is not a blocking patch defect.
