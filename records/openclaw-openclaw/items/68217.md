## Best Possible Solution

## Reproduction Assessment

Yes at source level: current main's `hasURLScheme` is case-sensitive and omits `vbscript:`. A high-confidence user-visible bad rewrite path is not established because `localizeURL` returns non-route targets unchanged later.

## Solution Assessment

Yes: bounded `strings.EqualFold` prefix checks are a narrow maintainable hardening and avoid lowercasing full URL targets. Direct helper-level tests would improve proof quality but are not a blocking correctness defect.
