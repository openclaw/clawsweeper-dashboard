## What This Changes

The branch changes `scripts/docs-i18n` to classify URL schemes with bounded case-insensitive prefix checks, add `vbscript:`, and add localized-link regression cases.

## Best Possible Solution

Merge this PR or an equivalent tiny patch if maintainers want the docs-i18n scheme classifier itself to be case-insensitive and explicitly recognize `vbscript:`, after contributor proof is added.

## Reproduction Assessment

Yes at source level: current main's `hasURLScheme` is case-sensitive and lacks `vbscript:`. No high-confidence bad rewritten docs output is demonstrated because `localizeURL` already returns the cited non-route hrefs unchanged through the later path guard.

## Solution Assessment

Yes for the proposed code shape: a bounded `strings.EqualFold` prefix helper is a narrow maintainable way to harden this classifier without lowercasing full link targets. A broader URL parser would be more invasive for this script.
