## What This Changes

The PR changes `scripts/docs-i18n` to use bounded case-insensitive URL scheme prefix checks, adds `vbscript:`, and adds localized-link regression cases.

## Best Possible Solution

Merge this patch or an equivalent tiny helper hardening after contributor proof is added; a direct helper-level table test would strengthen coverage without broadening the script.

Yes at source level: current main's scheme helper is case-sensitive and omits `vbscript:`. I did not establish a user-visible bad rewrite because the later non-route fallback already returns those link targets unchanged.

## Solution Assessment

Yes: bounded `strings.EqualFold` prefix checks are a narrow maintainable way to harden this classifier without lowercasing the full URL target. A direct `hasURLScheme` test would be stronger, but I do not see a blocking correctness defect in the patch.
