## Summary

Keep this PR open for maintainer review. Current main still has the older case-sensitive scheme helper, while the PR is a narrow docs-i18n script/test change that addresses the earlier review feedback with bounded `EqualFold` prefix checks, adds `vbscript:`, and has green PR check-runs; I found no blocking correctness or security issue.

## What This Changes

The PR updates the docs-i18n localized link rewriter to use bounded case-insensitive scheme prefix checks, adds `vbscript:` to the scheme guard, and adds regression cases for `vbscript:` and mixed-case `Javascript:` hrefs.

## Best Possible Solution

Adopt this PR or an equivalent tiny patch if maintainers want the docs-i18n scheme guard to be case-insensitive and explicitly include `vbscript:`, keeping the change confined to `scripts/docs-i18n` with the added regression coverage.

## Reproduction Assessment

Yes. The relevant path is `routeIndex.localizeURL` in `scripts/docs-i18n/localized_links.go`; current main shows the case-sensitive helper, and the PR adds focused table tests for the new scheme cases, with PR check-runs green on the head SHA.

## Solution Assessment

Yes. The current PR is a narrow maintainable way to harden the existing allowlisted scheme helper, and the bounded `EqualFold` helper is safer than lowercasing the full URL candidate; a broader URI parser would be more invasive for this script.
