## What This Changes

The PR changes `scripts/docs-i18n` to use bounded case-insensitive URL-scheme prefix checks, add `vbscript:`, and add localized-link regression cases.

## Best Possible Solution

Merge this patch or an equivalent tiny helper hardening after contributor proof is added; direct helper coverage would make the regression test stronger without broadening the script.

## Reproduction Assessment

Yes at source level: current main's scheme helper is case-sensitive and omits `vbscript:`. No high-confidence user-visible bad rewrite is demonstrated because the later non-route path guard already returns those link targets unchanged.

## Solution Assessment

Yes: bounded `strings.EqualFold` prefix checks are a narrow maintainable way to harden this classifier without lowercasing the full URL target. A direct `hasURLScheme` table test would strengthen coverage, but I do not see a blocking correctness defect in the patch.
