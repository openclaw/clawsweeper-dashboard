## Summary

Keep this open. Current main still clears Control UI/WebChat active run state after reconnect, the Gateway hello snapshot still has no active-run recovery contract, and open PR #41067 remains the relevant implementation candidate rather than a merged fix.

## What This Changes

Control UI/WebChat should preserve or recover in-flight AI session state across WebSocket reconnects instead of leaving active runs terminated or lost.

## Best Possible Solution

Keep this issue as the canonical Control UI/WebChat active-run reconnect recovery report. The desired end state is an additive Gateway hello/snapshot or reconnect RPC contract that exposes active chat runs, lets the UI reconcile chatRunId and busy/stream state after reconnect, avoids clearing active state until the run is known terminal, and adds regression coverage for WebSocket close/reconnect during an active chat.send.
