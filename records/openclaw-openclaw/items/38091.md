## Summary

Keep open: current main only partially addresses the reconnect family. The merged Stop/keepalive work helps connection stability and abort availability, but the Control UI reconnect hello path still clears local chat run state and the Gateway hello snapshot still lacks active run metadata; open PR https://github.com/openclaw/openclaw/pull/41067 remains the focused candidate for that recovery path.

## What This Changes

The issue reports Control UI WebSocket reconnects terminating or losing in-flight WebChat sessions during active AI runs.

## Best Possible Solution

Land or replace PR 41067 with a rebased narrow fix that exposes active chat run metadata in the Gateway hello snapshot and reconciles Control UI in-flight chat state across reconnects, while keeping outbound queue/ACK work in https://github.com/openclaw/openclaw/issues/45952 separate.
