## What This Changes

The issue reports that OpenClaw Control UI WebSocket reconnects can terminate or lose state for an active AI chat session.

## Best Possible Solution

Repair or replace the active-run recovery design so reconnect hello reconciles authoritative Gateway-owned run state without falsely killing sessions, then prove it in a real Control UI reconnect flow.

## Reproduction Assessment

Yes, at source level: current main still clears `chatRunId`/stream state and marks the session killed during reconnect hello, and the adjacent test asserts that behavior. No live WSL2/macOS reconnect E2E was run in this read-only review.
