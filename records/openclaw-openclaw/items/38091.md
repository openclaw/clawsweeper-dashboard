## Summary

Keep open: current main still clears active Control UI/WebChat run state during reconnect hello handling, and the Gateway hello snapshot still lacks active-run recovery metadata. The issue has concrete reporter logs plus a second cross-platform data point; open PR #41067 remains the focused implementation candidate, while #45952 tracks a related but different outbound queue/ACK loss problem.

## What This Changes

Preserve active OpenClaw WebChat/Control UI session state across WebSocket reconnects instead of terminating or clearing in-flight AI runs.

## Best Possible Solution

Land or refresh #41067, or an equivalent narrow implementation, so Gateway exposes active run metadata on reconnect and Control UI reconciles busy/stream/run state until a terminal event is known.

## Reproduction Assessment

## Solution Assessment

Yes for keeping this open, but not for opening a separate repair branch. The narrow maintainable fix is an additive reconnect recovery contract plus Control UI reconciliation, and #41067 already targets that path.
