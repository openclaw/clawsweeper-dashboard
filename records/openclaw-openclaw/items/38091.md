## Summary

Keep open: current main still drops Control UI/WebChat active-run state during reconnect hello handling, and the Gateway hello snapshot does not carry enough active-run metadata for recovery. Existing keepalive and Stop-button improvements reduce adjacent symptoms, but they do not solve the reported in-flight session loss; open PR #41067 remains the focused implementation candidate.

## What This Changes

The issue reports Control UI WebSocket reconnects during WebChat causing in-flight AI sessions to terminate or lose state, and asks reconnects to preserve active session state.

## Best Possible Solution

Land or refresh #41067, or an equivalent narrow fix, so Gateway exposes active run metadata on reconnect and Control UI preserves busy/stream/run state until a terminal event is known; keep #45952's outbound queue/ACK work separate.

## Reproduction Assessment

## Solution Assessment
