WebChat reconnects during an active Control UI run can clear local run/session state and surface terminated or interrupted behavior instead of preserving the in-flight session.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/41067 with an additive Gateway active-run snapshot/reconciliation contract, while keeping outbound reconnect send replay tracked in https://github.com/openclaw/openclaw/issues/45952.
