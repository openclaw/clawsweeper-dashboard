## Summary

Keep open: current main and the latest release still clear local active WebChat run state on reconnect hello, the Gateway snapshot has no active-run identity recovery contract, and the focused recovery PR at https://github.com/openclaw/openclaw/pull/41067 remains open rather than shipped.

## What This Changes

The issue reports that WebChat WebSocket reconnects can terminate or lose an in-flight AI session instead of preserving active run state.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/41067 with deterministic active-run snapshot/reconciliation, while leaving outbound reconnect send replay tracked in https://github.com/openclaw/openclaw/issues/45952.

## Solution Assessment

Yes for keeping this issue open. The maintainable fix is an additive Gateway active-run recovery contract plus Control UI reconciliation, with the currently open recovery PR as the implementation candidate and outbound ACK/replay tracked separately.
