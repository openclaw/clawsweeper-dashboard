## Summary

Keep open: current main still has the provider default overwrite paths, and this PR targets the right bug, but the PR head is not mergeable as-is because it preserves defaults unconditionally in the raw merge helper, violates the generic fixture boundary, and duplicates the changelog entry.

## What This Changes

The PR makes provider preset/auth patch defaults additive, updates OpenAI/Moonshot behavior and tests, and documents that explicit default-setting commands still replace the model.

## Best Possible Solution

Keep the additive provider-default behavior, but make preservation explicitly option-gated, keep core tests provider-neutral, remove the duplicate changelog note, and validate before merge.

## Reproduction Assessment

Yes. Current main can reproduce the original overwrite by applying a provider preset or auth configPatch over an existing custom primary/fallback, and the PR-head defects are directly visible in the diff plus the boundary test contract.

## Solution Assessment

No, not as implemented. The direction is maintainable, but raw merge preservation must not affect explicit replacement/migration paths, and generic core fixtures must stay provider-neutral.
