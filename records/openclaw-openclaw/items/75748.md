## Summary

Keep this PR open. The branch targets a real current-main provider/auth default-model overwrite bug from #75720, but the proposed auth-patch helper still lets provider fallback suggestions replace an existing explicit empty or primary-only fallback selection, so it is not ready to merge as-is.

## What This Changes

The branch adds preserve-existing-default-model options across provider preset/auth helpers, updates OpenAI and Moonshot setup behavior, and adds docs, changelog, and targeted tests for #75720.

## Best Possible Solution

Keep the additive provider-default behavior, but make preserveExistingDefaultModel preserve the existing model selection exactly enough that provider patches cannot add or replace fallback behavior when a user already has a configured default. Keep explicit replacement limited to openclaw models set and models auth login --set-default, and add regression coverage for empty fallback and primary-only existing configs.

## Reproduction Assessment

Yes. Current main can be reproduced at helper level with applyProviderConfigWithDefaultModelPreset/applyOpenAIConfig or a ProviderAuthResult.configPatch against an existing agents.defaults.model; the PR-specific bug is reproducible by combining an existing { primary, fallbacks: [] } with a provider patch that supplies fallback entries.

## Solution Assessment

No, not as-is. The direction is the narrow maintainable fix, but the auth-patch preservation logic should restore the fallback selection for empty and primary-only existing configs before the PR is mergeable.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Preserve empty fallback selections in auth patches:** `src/plugins/provider-auth-choice-helpers.ts:125-127`
  - body: When preserveExistingDefaultModel is true and the existing model has fallback selection like `fallbacks: []` or a primary-only config, this block only restores fallbacks when `existingFallbacks.length > 0`. A provider configPatch that suggests fallbacks therefore still adds those fallbacks while the CLI/docs promise that auth without `--set-default` keeps the user's default model selection. Preserve the existing fallback field/absence when keeping an existing default, and add a regression for the empty fallback case.
  - confidence: 0.9
