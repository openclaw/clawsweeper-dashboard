## Label Justifications

- `P1`: The PR fixes a shipped Mattermost channel reply-routing regression that can move agent replies out of the user’s active thread.

## Summary

Keep open for maintainer review: current main and the latest release violate the documented Mattermost threading contract, while this PR is a narrow source/test fix with adequate manual Mattermost proof and no blocking findings.

## What This Changes

The PR changes Mattermost reply targeting so inbound channel/group posts that already have a thread root keep that root when `replyToMode` is `off`, and updates focused tests for that behavior.

## Best Possible Solution

Land the narrow Mattermost helper/test fix after normal maintainer review and required checks, preserving the documented default while leaving DMs and top-level `off` messages non-threaded.

## Reproduction Assessment

Yes from source: current `main` and `v2026.6.1` return no reply target for a non-direct existing `threadRootId` when `replyToMode` is `off`, while the docs and type comment say the default should reply inside an existing thread. I did not run a live Mattermost repro in this read-only review.

## Solution Assessment

Yes: changing the single effective-reply helper is the narrowest owner-boundary fix because that value feeds session scoping, typing, preview, and final delivery. A delivery-layer-only patch would leave session/history and interactive paths drifting.
