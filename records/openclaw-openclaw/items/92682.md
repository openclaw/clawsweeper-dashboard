The PR wires the built-in read tool text branch to `decodeWindowsOutputBuffer` so Windows invalid-UTF-8 text can fall back to the existing codepage-aware decoder.

## Best Possible Solution

Land the narrow dependency-free read-tool fallback using the existing Windows codepage decoder, with redacted real-run proof and focused GBK read regression coverage before merge.

## Reproduction Assessment
