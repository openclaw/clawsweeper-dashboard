## Summary

Keep open: the diagnostic direction is useful, but the patch still warns from a cumulative overflow counter and can mislabel later independent overflows; it also lacks real-run proof of the new warning.

## What This Changes

The branch adds a context-overflow hint log, compaction troubleshooting docs/changelog text, and two overflow-loop unit tests.

## Best Possible Solution

Keep the additive hint/docs, but trigger the warning from an immediate post-compaction retry marker that is cleared by intervening non-overflow retry paths, then add redacted real-run proof.

## Reproduction Assessment

Yes for the review finding: source inspection shows `overflowCompactionAttempts` is run-scoped and only initialized/incremented, while non-overflow retry branches can continue without clearing it. I did not run tests because this review is read-only.

## Solution Assessment

No. The operator hint is a good direction, but `overflowCompactionAttempts > 0` is not the narrowest maintainable signal for an overflow that directly persisted after compaction.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Gate overflow hint to immediate post-compaction retries:** `src/agents/pi-embedded-runner/run.ts:1169-1171`
  - body: `overflowCompactionAttempts` is cumulative for the whole run loop, so after one compaction a later planning-only, reasoning-only, or empty-response retry can continue without clearing it and a later unrelated overflow will still emit `[context-overflow-hint]`. Track a dedicated immediate-post-compaction retry marker, and clear it on intervening non-overflow retry paths before warning.
