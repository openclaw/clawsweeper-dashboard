## Summary

Keep open: the operator hint is useful, but the current gate can fire for later unrelated overflows after any prior compaction, and the external PR still lacks real behavior proof from an actual overflow run.

## What This Changes

The PR adds a context-overflow warning pointing at `agents.defaults.contextLimits.toolResultMaxChars`, plus compaction docs, changelog text, and overflow-loop unit tests.

## Best Possible Solution

Keep the hint and docs, but gate the warning on an immediate-post-compaction retry marker that is cleared by any intervening non-overflow retry path, with a regression for that sequence.

## Reproduction Assessment

Yes for the PR-introduced bug by source inspection: a compacted overflow can be followed by planning-only, reasoning-only, or empty-response retries before a later overflow sees the stale nonzero counter. I did not run tests because this was a read-only review.

## Solution Assessment

No. The hint is a good direction, but `overflowCompactionAttempts > 0` is not the narrowest maintainable signal for an overflow that directly persisted after compaction.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Gate hint to direct post-compaction retries:** `src/agents/pi-embedded-runner/run.ts:1169-1171`
  - body: `overflowCompactionAttempts` is cumulative for the run loop, so after one compaction any later independent overflow can log `[context-overflow-hint]` even if it did not immediately persist after compaction. Track a dedicated immediate-post-compaction marker and clear it on intervening retry paths before emitting the hint.
