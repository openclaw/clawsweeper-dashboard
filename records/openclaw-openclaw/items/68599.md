## Summary

Keep open: the diagnostic hint is useful, but the PR currently gates it on a run-scoped overflow counter, so it can emit for a later independent overflow rather than only an immediate post-compaction retry. The PR also needs after-fix real behavior proof before merge.

## What This Changes

The PR adds a context-overflow warning hint, compaction troubleshooting docs/changelog text, and two overflow-loop regression tests around `toolResultMaxChars`.

## Best Possible Solution

Keep the hint, but drive it from an immediate-post-compaction retry marker that is cleared by any intervening non-overflow retry path, and add a regression for that later-overflow case.

## Reproduction Assessment

Yes for the PR-introduced regression: source inspection shows a compacted overflow can be followed by a planning-only, reasoning-only, empty-response, or silent-error retry before a later overflow sees the stale nonzero counter. I did not run tests because this was a read-only review.

## Solution Assessment

No. The operator hint is a good direction, but `overflowCompactionAttempts > 0` is not the narrowest maintainable signal for “immediately persisted after compaction.”

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Gate hint to direct post-compaction retries:** `src/agents/pi-embedded-runner/run.ts:1171`

Track a separate immediate-post-compaction retry marker and clear it on intervening retry paths.
