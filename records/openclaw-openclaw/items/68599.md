## Summary

Keep open: the diagnostic hint is useful, but the PR still gates it on a run-scoped compaction counter that can over-fire after later unrelated retry paths, and the external real behavior proof gate is still missing.

## What This Changes

The PR adds a `[context-overflow-hint]` warning for post-compaction overflows, compaction troubleshooting docs, a changelog entry, and overflow-loop tests.

## Best Possible Solution

Keep the hint and docs, but gate the warning on an immediate-post-compaction retry marker that is cleared by intervening non-overflow retry paths, with a regression for that sequence and redacted real overflow proof.

## Reproduction Assessment

Yes for the PR-introduced defect by source inspection: after one compaction, planning-only, reasoning-only, or empty-response retries can continue before a later overflow sees the stale nonzero counter. I did not run tests because this is a read-only review.

## Solution Assessment

No, not as written. The hint is a good direction, but `overflowCompactionAttempts > 0` is not the narrowest maintainable signal for an overflow that directly persisted after compaction.

## Review Findings

Overall correctness: patch is incorrect
