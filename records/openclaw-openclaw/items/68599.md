## Summary

This PR should stay open: the diagnostic hint is useful, but the current warning gate is broader than the documented compact-then-overflow case, and the external real-behavior proof gate is still unmet.

## What This Changes

The PR adds a context-overflow hint log, compaction troubleshooting docs, changelog text, and two overflow-loop regression tests.

## Best Possible Solution

Keep the additive hint/docs direction, but drive the warning from an immediate post-compaction retry marker that is cleared by intervening non-overflow retry paths, then add redacted real-run proof.

## Reproduction Assessment

Yes for the review finding: source inspection shows `overflowCompactionAttempts` is run-scoped and non-overflow retry branches can continue without clearing it. I did not run tests because this review is read-only.

## Solution Assessment

No. The operator hint is a good direction, but `overflowCompactionAttempts > 0` is not the narrowest maintainable signal for an overflow that directly persisted after compaction.
