## Summary

Keep open. The PR targets a real operator diagnostics gap, but its guard uses a cumulative run-loop counter, so the new hint can fire on a later independent overflow after unrelated non-overflow continuations rather than only on an immediate post-compaction retry overflow.

## What This Changes

The PR adds a `[context-overflow-hint]` warning keyed on `overflowCompactionAttempts > 0`, updates compaction troubleshooting docs and the changelog, and adds two overflow-compaction regression tests.

## Best Possible Solution

Keep the additive hint and docs, but drive the log from explicit immediate post-compaction retry state that is cleared before unrelated continuations, then add a negative regression for a later independent overflow in the same run.

## Reproduction Assessment

Yes for the PR-introduced bug. The existing harness can model overflow, successful compaction, a non-overflow continuation such as planning-only or empty response, and then a later overflow; with the current PR guard, the stale counter still emits the hint.

## Solution Assessment

No. The log/docs/test direction is narrow and useful, but the implementation should key the hint to an immediate compact-then-retry overflow or reset the marker after any non-overflow continuation.
