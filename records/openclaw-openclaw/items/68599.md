## Summary

Keep this PR open: the diagnostic hint is useful and not implemented on current main, but the current patch still has a blocking over-broad warning gate and lacks after-fix real behavior proof.

## What This Changes

The PR adds a context-overflow hint log, compaction troubleshooting docs, a changelog entry, and two overflow-loop regression tests.

## Best Possible Solution

Keep the additive diagnostic/docs direction, but gate the warning on an immediate post-compaction retry marker that is cleared by intervening retry paths, then add redacted real-run proof.

## Reproduction Assessment

Yes for the review finding: source inspection shows `overflowCompactionAttempts` is run-scoped and planning-only, reasoning-only, and empty-response retries continue without clearing it. The PR's claimed real overflow scenario is not proven by an after-fix run artifact.

## Solution Assessment

No. The operator hint is useful, but `overflowCompactionAttempts > 0` is not the narrowest maintainable signal for overflow that directly persisted after compaction.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Gate the overflow hint to immediate post-compaction retries:** `src/agents/pi-embedded-runner/run.ts:1169-1171`
