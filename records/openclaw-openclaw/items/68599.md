## Summary

Keep open: this PR adds a useful compaction diagnostic direction, but the warning gate is broader than the stated compact-then-overflow case and external real behavior proof is still missing.

## What This Changes

The branch adds a `[context-overflow-hint]` warning, compaction troubleshooting docs, changelog text, and overflow-loop regression tests for post-compaction context overflow.

## Best Possible Solution

Keep the diagnostic and docs, but gate the warn line on a direct post-compaction retry marker that is cleared by intervening continuation paths, with regression coverage for that sequence.

## Reproduction Assessment

Yes for the PR-introduced issue by source inspection: after one overflow compaction, planning-only, reasoning-only, or empty-response continuations can keep the same loop alive before a later overflow sees the cumulative counter. I did not run tests in this read-only review.

## Solution Assessment

No. The hint points at the right troubleshooting knob, but `overflowCompactionAttempts > 0` is not the narrowest maintainable signal for an overflow that directly persisted after compaction.

## Review Findings
