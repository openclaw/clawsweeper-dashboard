## Summary

Keep open. The operator hint is useful and current main does not already provide it, but the PR still gates the hint on a cumulative run-loop counter, so it can fire after unrelated retry paths rather than only on a direct post-compaction overflow.

## What This Changes

The PR adds a context-overflow hint log for persistent overflow after compaction, compaction troubleshooting docs, changelog text, and two overflow-loop regression tests.

## Best Possible Solution

Keep the operator hint, but drive it from an immediate post-compaction retry marker that is cleared by any intervening non-overflow attempt or retry path while preserving the cumulative overflow retry limit.

## Reproduction Assessment

Yes for the PR regression. The existing overflow-loop harness can mock overflow, successful compaction, an intervening non-overflow retry path such as planning-only or empty-response retry, then a later overflow that would still see `overflowCompactionAttempts > 0`.

## Solution Assessment

No. The additive log and docs direction are useful, but `overflowCompactionAttempts > 0` is not the narrowest maintainable signal for an overflow that directly persisted after compaction.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Gate the hint to direct post-compaction retries:** `src/agents/pi-embedded-runner/run.ts:1171`
  - body: `overflowCompactionAttempts` is initialized once for the whole run loop and is not reset when the loop continues through planning-only, reasoning-only, empty-response, or silent-error retries. A later independent overflow can therefore emit `[context-overflow-hint]` even though it did not directly persist after compaction, so please use an immediate post-compaction retry marker and clear it on intervening paths.
  - confidence: 0.89
