## Summary

Keep open: current main and the latest release still return `Math.floor(n * factor)` from the cron-specific duration parser without checking the computed result for finiteness, and the direct fixing PR at https://github.com/openclaw/openclaw/pull/84047 was closed unmerged.

## What This Changes

Reject non-finite computed millisecond durations from the cron CLI duration parser before they can reach cron schedule payloads.

## Best Possible Solution

Apply the small finite-result guard with focused parser regression coverage, using the closed unmerged PR as useful source context rather than leaving the issue tied to stale linked-PR state.

## Reproduction Assessment

Yes, at source level. The current parser still multiplies a finite parsed long-digit value by the days factor and returns `Infinity`, and the direct `--every` caller does not reject that truthy result.

## Solution Assessment

Yes. The narrow maintainable fix is to reject non-finite computed milliseconds in `src/cli/cron-cli/shared.ts` and add regression tests for the reachable long-digit overflow case plus normal valid durations.
