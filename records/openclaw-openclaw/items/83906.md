## Summary

Keep open: current main and the latest release still let the cron-specific duration parser return `Infinity` after unit multiplication, and the linked fix PR was closed unmerged, so there is no active implementation path now.

## What This Changes

The issue requests that cron CLI duration parsing reject non-finite millisecond results instead of allowing `Infinity` through `--every` and related options.

## Best Possible Solution

Add a finite-result check after computing cron duration milliseconds, preserve the existing null-on-invalid parser contract, and cover the long pure-digit overflow case with focused regression tests.

## Reproduction Assessment

Yes, at source level. Replaying the current parser regex, parse, unit multiplication, and caller truthiness check with a long pure-digit days input produces `Infinity`; the literal `1e308d` example is rejected earlier by the regex.

## Solution Assessment

Yes. The narrowest maintainable repair is to reject non-finite computed milliseconds in `src/cli/cron-cli/shared.ts`, or to reuse the central duration parser while preserving the cron CLI error contract.
