## Summary

Keep open: current main and the latest release still return `Math.floor(n * factor)` without a finite-result check, and the linked fix PR was closed unmerged.

## What This Changes

The issue reports that the cron CLI duration parser can return non-finite millisecond values for very large duration inputs.

## Best Possible Solution

Add a finite-result guard to the cron-specific parser, or switch this cron path to the central duration parser, with focused regression coverage for long digit overflow inputs and ordinary valid durations.

## Reproduction Assessment

Yes, at source level. A long pure-digit days input follows the current parser regex/math path into a finite parsed number, overflows during multiplication, and returns truthy `Infinity`; the literal `1e308d` example is rejected before that path.

## Solution Assessment

Yes. Rejecting non-finite computed milliseconds after multiplication is the narrowest maintainable fix for the existing null-on-invalid parser contract; reusing the central parser is also acceptable if caller error behavior stays friendly.
