## Summary

Keep open: current main and v2026.5.20 still allow the cron-specific duration parser to return a non-finite value, and the linked fix PR was closed unmerged, leaving no open implementation candidate.

## What This Changes

The issue asks OpenClaw to reject cron duration inputs whose finite parsed number overflows to Infinity after unit multiplication.

## Best Possible Solution

Land a focused fix that rejects non-finite computed cron durations, ideally with regression coverage for the long pure-digit overflow case and without broadening cron duration syntax unnecessarily.

## Reproduction Assessment

Yes. Source inspection plus a direct arithmetic replay show a long pure-digit days input reaches `Math.floor(n * factor) === Infinity` on current main, while the reporter's literal `1e308d` example is rejected earlier by the regex.

## Solution Assessment

Yes. The narrow maintainable repair is to reject non-finite computed milliseconds in the cron parser or migrate that cron path to the central parser that already enforces this contract.
