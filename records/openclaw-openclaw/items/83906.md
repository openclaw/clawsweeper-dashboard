Add a finite-result guard in the cron parser, cover the overflow case with focused regression tests, and keep caller behavior rejecting invalid durations before schedule payloads reach the gateway.

## Repair Work Prompt

Fix the cron CLI duration overflow reported at https://github.com/openclaw/openclaw/issues/83906. Current `src/cli/cron-cli/shared.ts` validates the parsed number before multiplying by the unit factor, but a long pure-digit value such as `1` plus 302 zeros plus `d` produces `Math.floor(Infinity)` and callers in `schedule-options.ts` and `register.cron-edit.ts` treat it as valid. Add the narrow guard so non-finite computed milliseconds are rejected as invalid, or reuse the central `src/cli/parse-duration.ts` parser without widening behavior unexpectedly. Add focused regression tests for ordinary durations, invalid/zero values, parseFloat-Infinity input, and the finite-parse/multiply-overflow case. Preserve the existing user-facing invalid-duration messages unless a test-supported change is necessary. Include a single changelog Fixes entry if the repair lane owns changelog updates.
