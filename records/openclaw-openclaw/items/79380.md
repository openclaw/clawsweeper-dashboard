## What This Changes

Investigate and fix a Raspberry Pi 4 ARM64 Docker gateway regression where versions 2026.4.25 and later pin CPU and stall Telegram polling after upgrading from 2026.4.23.

## Best Possible Solution

Land a corrected heartbeat scheduler fix that repairs stale `nextDueMs` after terminal skipped/disabled outcomes, adds focused regression coverage, and includes redacted real ARM64 Docker or long-running gateway proof before closing this issue.

## Reproduction Assessment

Yes for the scheduler defect by source inspection: current main can leave `nextDueMs` stale after skipped/disabled outcomes and then re-arm `scheduleNext` with a zero-delay timer. I did not establish a live Raspberry Pi Docker reproduction in this read-only review.

## Solution Assessment

No for the current linked PR direction as reviewed: a fixed timer floor avoids `setTimeout(0)` but does not repair the stale schedule state. The narrower maintainable fix is to advance or clear stale heartbeat due state so the runner returns to configured cadence.
