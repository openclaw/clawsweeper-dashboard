Keep open. The report is recent, detailed, and current main still contains a plausible heartbeat scheduler path that can re-arm with a zero-delay timer; an open linked PR claims to close this issue but remains unmerged and already has review/proof gaps.

## What This Changes

The issue reports a Raspberry Pi 4 ARM64 Docker regression where OpenClaw 2026.4.25 through 2026.5.7 pin gateway CPU and stall Telegram polling after 2026.4.23 worked.

## Best Possible Solution

Fix stale heartbeat nextDueMs handling so skipped or deferred scheduled outcomes settle back to the configured cadence, add focused scheduler regression coverage, and require redacted real runtime proof before merging the linked or replacement fix.

## Reproduction Assessment

Yes for the scheduler defect by source inspection: current main can leave nextDueMs stale and past-due, then schedule a zero-delay interval wake. No for the full Raspberry Pi ARM64 Docker CPU-starvation scenario because this read-only review did not run that environment.

## Solution Assessment

No for closing now. The open linked PR is the right place to continue, but the current proposed timer-floor direction should be reviewed against a schedule-state repair and backed by real gateway proof.
