## What This Changes

The issue requests priority-aware command queue scheduling plus an age-based starvation guard so urgent work can bypass background work within a lane while equal-priority entries remain FIFO.

## Best Possible Solution

Review and land one maintainer-approved starvation policy in the canonical command queue, with regression coverage for priority ordering, equal-priority FIFO, and low-priority aging.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path for the remaining starvation case: hold a lane busy, enqueue background work, keep adding foreground work, and static priority insertion plus `shift()` keeps the background entry from aging upward.

## Solution Assessment

Unclear. The queue is the right owner boundary, but maintainers still need to choose whether aged background work may outrank fresh foreground work before the linked implementation should land.
