Keep open: current main and v2026.6.10 have static foreground/normal/background command-queue priority, but the requested age-based starvation guard is still missing and open closing PRs already own the remaining scheduler-policy decision.

## What This Changes

The issue requests command-queue priorities plus an aging guard so latency-sensitive work can overtake background work within a lane while preserving FIFO among equal-priority entries.

## Best Possible Solution

Review and land one maintainer-approved command-queue aging policy with regression coverage for static priority, equal-priority FIFO, and low-priority aging; keep this issue open until that PR merges or a canonical replacement is chosen.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: hold a lane busy, enqueue background work, keep adding foreground work, and current static priority insertion plus shift-based draining provides no age-based promotion.

Unclear. The command queue is the right owner boundary, but maintainers still need to decide the aged-background-versus-fresh-foreground contract before either linked PR is the best fix.
