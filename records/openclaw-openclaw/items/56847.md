## Summary

Keep open. The contributor addressed the earlier flush-assertion note, but the branch still suppresses the missing-path warning for `file`/`filePath` while current read execution and tests require canonical `path`, so it can hide an invalid tool call rather than fix the contract.

## Best Possible Solution

Define one coherent read-tool parameter contract across schema, validation, diagnostics, display/fingerprint helpers, docs, and tests; either support aliases end to end or keep diagnostics aligned with canonical `path`.
