## Summary

Keep open: current main and v2026.6.6 still run provider tool-schema normalization on each embedded attempt, and the active cache fix is still an open, conflicting PR rather than merged behavior.

## What This Changes

The issue asks OpenClaw to stop repeatedly reloading/normalizing provider tool schemas for Google Antigravity/Gemini because it can trigger anti-abuse flags and gateway allocation pressure.

## Best Possible Solution

Rebase and land a bounded provider-normalized schema-parameter cache for bundled pure compat normalizers, avoid new config/session-persistence knobs, then close this after the fix merges.

## Reproduction Assessment

No, not for the full Google ban in a live account. The repeated normalization path is source-reproducible on current main, and the issue has reporter logs plus an independent heap-profile data point.

## Solution Assessment

Yes in direction: an internal bounded cache at the provider runtime boundary is the narrowest maintainable fix. New session-persistence config or backoff should not be the first fix unless the cache path proves insufficient.
