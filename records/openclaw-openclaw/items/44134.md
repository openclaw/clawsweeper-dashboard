## Summary

Keep open: current main and the latest release still route embedded attempts through provider tool-schema normalization without a cache, while the focused cache implementation remains an open PR rather than a merged fix.

## What This Changes

The issue asks OpenClaw to avoid repeatedly reloading or normalizing Google/Gemini provider tool schemas across active sessions because users reported provider bans, hangs, and memory pressure.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/78664, or an equivalent provider-owned schema-parameter cache that avoids re-normalizing stable schemas while preserving fresh tool objects, execute closures, approval state, runtime handles, and guards.

## Reproduction Assessment

Yes for the source-level root behavior: current main and v2026.5.20 still route embedded attempts into provider schema normalization without a cache. The Google ban itself remains external provider enforcement and is not locally reproducible from the repository alone.

## Solution Assessment

Yes for the fix direction: provider-owned schema-parameter caching is narrower than session persistence, retry backoff, or broad tool-loading rewrites. This issue should close only after the open cache PR or an equivalent implementation lands and ships.
