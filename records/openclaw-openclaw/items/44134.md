## Summary

Keep open: current main and v2026.6.6 still run provider tool-schema normalization during embedded attempts without a cache, and the direct cache fix is still an open dirty PR rather than merged behavior.

## What This Changes

The issue asks OpenClaw to stop repeatedly reloading or re-normalizing Google Antigravity/Gemini tool schemas, with caching/backoff/session-persistence considered to avoid provider anti-abuse false positives and memory pressure.

## Best Possible Solution

Land a bounded internal provider-normalized schema cache at the provider tool-schema boundary, preserving custom provider hook behavior and avoiding new user-facing config unless the cache proves insufficient.

## Reproduction Assessment

No for the full Google ban outcome; yes at source level for repeated normalization. Current main calls normalizeAgentRuntimeTools inside embedded attempts, and the Google hook routes through the shared Gemini schema normalizer each time.

## Solution Assessment

Yes in direction: a bounded internal cache at the provider runtime/tool-schema boundary is the narrowest maintainable fix. Session-persistence config or backoff should remain secondary unless the cache path is insufficient.
