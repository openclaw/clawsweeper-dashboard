## Summary

Keep open: current main and v2026.6.6 still normalize provider tool schemas on each embedded attempt, while the cache implementation that would address the central problem is still an open, conflicting PR rather than merged behavior.

## What This Changes

The issue asks OpenClaw to stop repeatedly reloading or normalizing Google Antigravity/Gemini tool schemas in a way that can trigger provider anti-abuse checks and gateway memory pressure.

## Best Possible Solution

Rebase and review the bounded internal provider-normalized schema cache from https://github.com/openclaw/openclaw/pull/78664, or replace it with an equivalent cache at the same owner boundary, then close this issue after the fix merges.

## Reproduction Assessment

No for the full Google ban outcome; yes at source level for the repeated normalization path. Current main calls normalizeAgentRuntimeTools on each embedded attempt and the Gemini provider normalizer maps tool schemas each time.

## Solution Assessment

Yes in direction: an internal bounded cache at the provider runtime/tool-schema boundary is the narrowest maintainable fix. Session-persistence config or backoff should remain secondary unless the cache path proves insufficient.
