## Summary

Keep open: current main still normalizes provider tool schemas on each embedded attempt with no cache, and the Google plugin routes Antigravity/Gemini-style providers through the Gemini normalization hook. An open implementation PR now targets this exact hotspot, but it has not merged, so the issue remains valid rather than implemented.

## What This Changes

The issue asks OpenClaw to avoid repeatedly loading or normalizing Google/Antigravity tool schemas during active sessions because repeated snapshots may trigger provider anti-abuse handling and gateway memory pressure.

## Best Possible Solution

Land the open provider-normalization cache PR or an equivalent narrow fix that caches cloned normalized schema parameters with provider-owned invalidation while keeping tool objects, execute closures, approvals, and runtime policy fresh per turn.

## Reproduction Assessment

Yes, at source level. Current main calls provider tool-schema normalization from each embedded attempt and the helper has no cache, while Google/Antigravity routes through the Gemini normalization hook.

## Solution Assessment

Yes for the current direction: a conservative provider-owned cache for normalized schema parameters is the narrow maintainable fix. This issue should stay open until https://github.com/openclaw/openclaw/pull/78664 or an equivalent patch lands.
