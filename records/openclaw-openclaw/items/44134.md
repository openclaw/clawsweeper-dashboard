## Summary

Keep this issue open: current main still constructs attempt-local tools and runs provider tool-schema normalization during each embedded attempt, while the active cache implementation at https://github.com/openclaw/openclaw/pull/78664 remains open and unmerged.

## What This Changes

Avoid repeatedly reloading or re-normalizing provider tool schemas during active Google/Gemini CLI sessions to reduce anti-abuse false positives and memory pressure.

## Best Possible Solution

Land or replace the conservative provider-boundary cache that reuses normalized schema parameters while preserving fresh per-attempt tool objects, execute closures, approval state, and guards.

## Reproduction Assessment

Yes for the source-level root behavior: current main builds tools inside runEmbeddedAttempt and passes them through provider normalization on each attempt. No live reproduction was established for the external Google ban because that depends on provider-side enforcement.

## Solution Assessment

Yes: a conservative provider-boundary reuse layer for normalized schema parameters is narrower than session persistence or backoff. The open implementation candidate should be reviewed, repaired if needed, and landed or replaced before this issue closes.
