## Summary

Keep this open: current main and the latest release still run provider tool-schema normalization during embedded attempts without a cache, and the dedicated cache PR remains open rather than merged.

## What This Changes

The issue reports repeated provider tool-schema reload/normalization during agent attempts, with Google Antigravity anti-abuse bans and gateway memory pressure as observed impacts.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/78664, or an equivalent provider-owned schema-parameter cache that avoids re-normalizing stable provider schemas while preserving fresh tool objects, execute closures, approval state, runtime handles, and guards.

## Reproduction Assessment

Yes for the source-level root behavior: current main still normalizes provider tool schemas during embedded attempts without a provider-normalized cache. The external Google ban itself is not a local source reproduction because it depends on provider-side anti-abuse enforcement.

## Solution Assessment

Yes for the fix direction: a provider-owned cache for normalized schema parameters is narrower and safer than session-persistence or retry/backoff policy changes. The current open PR should be reviewed, repaired if needed, or replaced before this issue closes.
