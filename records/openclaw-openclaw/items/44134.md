## Summary

Keep this open: current main still runs provider tool-schema normalization on the embedded-attempt path and has no provider-normalized schema cache, while the dedicated cache PR remains open and currently conflicting.

## What This Changes

Investigate and mitigate repeated provider tool schema reloading/normalization that may trigger Google anti-abuse bans and cause gateway memory pressure.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/78664 after resolving the conflict, or replace it with an equivalent provider-owned cache that preserves fresh tool objects, execute closures, approval state, and guards.

## Reproduction Assessment

Yes for the source-level root behavior: current main still normalizes provider tool schemas during each embedded attempt and maps OpenAI-compatible tools without a cache. The Google ban itself is not locally reproduced because it depends on provider-side anti-abuse enforcement.

## Solution Assessment

Yes: a provider-owned cache for normalized schema parameters is narrower than session persistence or retry/backoff changes. The existing PR should be reviewed, repaired, or replaced before this issue closes.
