## Summary

Keep this open: current main still sends embedded-agent tools through provider schema normalization without a provider-normalized schema cache, and the matching cache work remains an open PR rather than merged code.

## What This Changes

The issue asks OpenClaw to avoid repeatedly reloading or re-normalizing provider tool schemas for Google Antigravity/Gemini CLI sessions to reduce anti-abuse false positives, hangs, and allocation pressure.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/78664, or an equivalent provider-owned schema-parameter cache that preserves fresh tool objects, execute closures, approval state, runtime handles, and guards; close this issue only after that mitigation is merged or superseded.

## Reproduction Assessment

Yes for the source-level root behavior: current main still normalizes provider tool schemas during each embedded attempt and has no cache layer on that path. The Google ban itself is not locally reproducible from source because it depends on provider-side anti-abuse enforcement.

## Solution Assessment

Yes: a provider-owned cache for normalized schema parameters is the narrowest maintainable fix compared with session persistence or retry/backoff policy changes. The current open PR should be reviewed, repaired if needed, or replaced before this issue closes.
