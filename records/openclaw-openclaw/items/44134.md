## Summary

Keep this issue open: current main still rebuilds attempt-local tools and runs provider tool-schema normalization on each embedded attempt, while the active cache implementation candidate at https://github.com/openclaw/openclaw/pull/78664 has not merged.

## What This Changes

The issue asks OpenClaw to avoid frequent provider tool-schema reloads for Google Antigravity/gemini-cli by reusing schemas or improving session persistence.

## Best Possible Solution

Land a conservative provider-runtime cache/reuse path that stores only normalized schema parameters while preserving fresh per-attempt tool objects, execute closures, approval state, and guards.

## Reproduction Assessment

Yes for the source-level root behavior: current main builds tools inside runEmbeddedAttempt and passes them through provider normalization on each attempt. No live reproduction was established for the external Google ban because that depends on provider-side enforcement.

## Solution Assessment

Yes: a conservative provider-boundary reuse layer for normalized schema parameters is narrower than session persistence or backoff. The open implementation candidate should be reviewed, fixed if needed, and landed or replaced before this issue closes.
