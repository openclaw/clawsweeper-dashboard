## Summary

Keep open: current main and the latest release still show per-attempt provider tool-schema normalization on the Google/OpenAI provider-hook path, while the issue has two independent user reports for Google account lockout/hangs and gateway OOM pressure. ## What This Changes

The issue asks OpenClaw to avoid repeatedly reloading or re-normalizing Google/Gemini provider tool schemas because repeated snapshots can trigger anti-abuse flags, hangs, and memory pressure.

## Best Possible Solution

Keep this as the canonical issue for a maintainer-approved provider-tool schema reuse fix that reduces repeated Google/Gemini snapshot pressure without adding session-persistence configuration by default.

## Reproduction Assessment

No live high-confidence reproduction was established. Source inspection does show a current-main path that normalizes provider tool schemas per invocation, and the issue discussion provides concrete logs and a heap-profile call path.

## Solution Assessment

Unclear as an implementation choice. A cache/reuse fix is likely the right direction, but maintainers should choose the owner boundary and avoid adding session-persistence config unless they explicitly want that product surface.
