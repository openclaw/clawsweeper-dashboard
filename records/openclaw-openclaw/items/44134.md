## Summary

Keep this issue open. Current main still rebuilds the per-attempt tool catalog and delegates provider-owned tool-schema normalization on each embedded attempt, while the issue discussion includes concrete Google account-ban symptoms and a later allocation-pressure report pointing at the same normalization path.

## What This Changes

The issue asks OpenClaw to avoid repeated Google/Gemini tool-schema reloads by caching or persisting normalized tool schemas and reducing session recreation churn.

## Best Possible Solution

Add a conservative normalized-tool-schema reuse layer at the runtime/provider-schema boundary: reuse unchanged normalized parameter schemas across attempts, rebuild executable AgentTool objects each attempt, and invalidate on tool schema or provider/model context changes.
