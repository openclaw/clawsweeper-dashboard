## Summary

Keep open: current main still runs Gemini provider tool-schema normalization from the embedded attempt path without a visible cache, and the closest related items address adjacent schema compatibility or failover behavior rather than this repeated Google schema snapshot/cache problem.

## What This Changes

The issue asks OpenClaw to stop repeatedly reloading or re-normalizing Google Antigravity / google-gemini-cli tool schemas because the behavior may trigger Google anti-abuse bans and gateway memory pressure.

## Best Possible Solution

Keep a canonical provider/runtime issue open and fix the hot path by reusing a prepared or cached Gemini-normalized tool catalog across attempts, adding new session-persistence config only if live proof shows it is necessary.

## Reproduction Assessment

No for the full external failure: this review did not reproduce Google's anti-abuse ban. Yes for the internal hot path: current main still calls Gemini tool normalization from runEmbeddedAttempt on each attempt, and that normalizer has no visible cache.

## Solution Assessment

Unclear: caching or preparing the normalized tool catalog earlier is the narrow maintainable direction, but backoff and session-persistence config need maintainer choice and live provider evidence before implementation.
