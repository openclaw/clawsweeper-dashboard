## Summary

Keep open: current main and the latest release still run provider tool-schema normalization during embedded attempts without an internal schema cache, and an open related PR already owns the likely fix but still needs maintainer/product judgment.

## What This Changes

The issue requests reducing repeated provider tool-schema loading/normalization for Google Antigravity/google-gemini-cli sessions to avoid anti-abuse flags and memory pressure.

## Best Possible Solution

Land or replace the bounded provider-normalized schema cache at the provider tool-schema boundary, preserving custom provider hook behavior and avoiding new user-facing session-persistence config unless caching proves insufficient.

## Reproduction Assessment

No for the full Google ban outcome; yes at source level for repeated normalization. Current main calls normalizeAgentRuntimeTools inside embedded attempts, then routes Google provider tools through the shared Gemini normalizer without a provider-normalized schema cache.

## Solution Assessment

Yes in direction, but not ready to close: a bounded internal cache at the provider runtime/tool-schema boundary is the narrowest maintainable fix, while session persistence config or backoff should remain secondary unless caching is proven insufficient.
