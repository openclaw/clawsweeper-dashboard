## Summary

Keep this open: current main and the latest release still normalize provider tool schemas on embedded attempts without a cache, and the focused cache fix remains an open PR rather than merged.

## What This Changes

The issue asks OpenClaw to stop repeatedly reloading or re-normalizing stable provider tool schemas for Google/Gemini-style sessions because it can trigger provider abuse detection and gateway memory pressure.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/78664, or an equivalent provider-owned schema-parameter cache that avoids re-normalizing stable schemas while preserving fresh tool objects, execute closures, approval state, runtime handles, and guards.

## Reproduction Assessment

Yes for the source-level root behavior: current main still normalizes provider tool schemas during embedded attempts without a cache. The external Google ban itself is not locally reproducible because it depends on provider anti-abuse enforcement.

## Solution Assessment

Yes for the fix direction: provider-owned schema-parameter caching is narrower than session persistence, retry backoff, or broad tool-loading rewrites. The open PR should be reviewed, repaired if needed, or replaced before this issue closes.
