## Summary

Keep open: current main and the latest release still run provider tool-schema normalization during each embedded attempt, and the discussion has credible Google CLI hang plus heap/OOM evidence. I did not find a merged fix or safe canonical replacement that settles the cache/session-lifecycle decision.

## What This Changes

The issue asks OpenClaw to stop repeatedly reloading or normalizing provider tool schemas for Google Antigravity/Gemini CLI sessions and to consider caching, backoff, or session persistence.

## Best Possible Solution

Add a lifecycle-owned provider/runtime cache or earlier prepared tool snapshot that reuses normalized schemas for stable tool catalogs, with focused Google CLI/Gemini and provider-hook tests, and avoid a new session-persistence config unless maintainers explicitly choose that policy.

## Reproduction Assessment

No for the full provider ban: I did not live-reproduce Google anti-abuse behavior. Yes for the source-level hot path: current main and v2026.6.8 still route each embedded attempt through provider tool-schema normalization.

## Solution Assessment

Unclear as an implementation decision: caching stable normalized provider tool catalogs looks like the narrowest likely fix, but session persistence and backoff policy need maintainer choice before automation should patch it.
