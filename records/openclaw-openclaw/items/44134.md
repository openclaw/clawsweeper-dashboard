## Summary

Keep this open: current main still runs provider tool-schema normalization on the embedded-attempt path without the cache requested here, and the only matching implementation candidate is still the open, conflicting PR at https://github.com/openclaw/openclaw/pull/78664.

## What This Changes

Investigate and reduce repeated tool-schema loading/normalization for Google Antigravity or Gemini CLI sessions to avoid provider anti-abuse flags and allocation pressure.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/78664 after review/rebase, or replace it with an equivalent provider-owned cache that preserves fresh tool objects, execute closures, approval state, and runtime guards.

## Solution Assessment
