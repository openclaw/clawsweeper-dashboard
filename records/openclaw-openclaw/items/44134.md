## Summary

Keep this open: current main still normalizes provider tool schemas on the embedded-attempt path without a provider-normalized schema cache, and the dedicated cache PR is still open rather than merged.

## What This Changes

The issue asks OpenClaw to stop repeatedly reloading or normalizing provider tool schemas during Google Antigravity/Gemini runs because the repeated snapshots can trigger provider anti-abuse handling and heavy gateway allocation pressure.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/78664 after resolving its merge state and any remaining review blockers, or replace it with an equivalent provider-owned cache that preserves fresh tool objects, execute closures, approval state, runtime handles, and guards.

## Reproduction Assessment

Yes for the source-level root behavior: current main still normalizes provider tool schemas during each embedded attempt and maps OpenAI-compatible tools without a cache. The Google ban itself is not locally reproduced because it depends on provider-side anti-abuse enforcement.

## Solution Assessment

Yes: a provider-owned cache for normalized schema parameters is narrower than session persistence or retry/backoff changes. The open cache PR should be reviewed, repaired if needed, or replaced before this issue closes.
