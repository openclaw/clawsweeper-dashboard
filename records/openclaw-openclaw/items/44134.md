## Summary

Keep this issue open: current main still constructs fresh attempt-local tools and delegates provider tool-schema normalization on each embedded attempt, while https://github.com/openclaw/openclaw/pull/78664 remains open as the active cache implementation candidate.

## What This Changes

The issue asks OpenClaw to avoid repeated provider tool-schema loading or normalization for Google Antigravity/Gemini sessions because it can trigger anti-abuse bans and memory pressure.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/78664 after review, or replace it with an equally conservative provider-runtime cache that stores only cloned normalized schema parameters while preserving fresh tool objects, execute closures, approval state, and guards.

## Reproduction Assessment

Yes for the source-level root behavior: current main builds attempt-local tools and passes them through provider normalization on each embedded attempt. No live reproduction was established for the external Google ban because that depends on provider-side enforcement.

## Solution Assessment

Yes: a conservative provider-boundary reuse layer for normalized schema parameters is narrower than session persistence or retry backoff. The open implementation candidate should be reviewed, fixed if needed, and landed or replaced before this issue closes.
