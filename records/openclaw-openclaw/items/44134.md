## What This Changes

The issue asks OpenClaw to stop repeatedly reloading or re-normalizing Google Antigravity/Gemini tool schemas because users report anti-abuse bans, hangs, and memory pressure.

## Best Possible Solution

Land a bounded internal provider-normalized-parameter cache with tests for cache hits, custom-provider bypasses, and closure freshness; avoid new session-persistence config unless maintainers explicitly choose that product surface.

## Reproduction Assessment

No full live repro for the Google ban/OOM was established in this review. The source path is clear: each embedded attempt can call provider tool-schema normalization and current main has no cache for the Gemini compatibility normalizer.

## Solution Assessment

Unclear until the open candidate PR is reviewed. An internal cache for bundled provider-normalized parameters appears to be the narrowest maintainable direction, while backoff or session-persistence config needs maintainer product direction.
