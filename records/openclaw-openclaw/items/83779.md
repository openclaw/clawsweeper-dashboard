Fix the WhatsApp group-routing diagnostics reported by https://github.com/openclaw/openclaw/issues/83779 and https://github.com/openclaw/openclaw/issues/83777. Keep the current `channels.whatsapp.groups` admission semantics; do not make `groupAllowFrom` a group registry and do not add a new config option. Update the WhatsApp startup log so it does not imply DM-only listening and includes configured group scope/count where practical. Update `docs/channels/whatsapp.md` and, if needed, `docs/channels/groups.md` to distinguish group admission via `groups` from sender authorization via `groupAllowFrom`. Add focused regression coverage for the new diagnostic path and preserve existing tests for the drop behavior. Add a changelog fix entry if the repair changes user-visible logging/docs.
