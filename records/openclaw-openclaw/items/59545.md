## Summary

Keep open. Current `main` still contains the macOS General settings path described in the report: entering General in local mode constructs the Tailscale settings section, that section hydrates SwiftUI state from config, and the same state values are watched by save-on-change handlers that can route into a config rewrite. The existing tests are smoke coverage and do not prove initial hydration is read-only.

## What This Changes

The issue reports that opening the macOS app General settings page rewrites `~/.openclaw/openclaw.json` without explicit user changes.

## Best Possible Solution

Keep the fix in the macOS app settings owner path: distinguish initial Tailscale/General config hydration from user-driven edits, avoid `ConfigStore.save` on view entry, and add a regression proving existing `gateway`, `meta`, `wizard`, and auth fields remain unchanged until the user changes a control.

## Reproduction Assessment

Yes. The report gives a concrete mtime/diff-based macOS UI reproduction path, and current-main source inspection still shows the General-tab hydration and save-on-change route that can produce the write.

## Solution Assessment

Yes. The narrow maintainable solution is to dirty-track or gate the Tailscale settings save handlers during initial config hydration; changing core config stamping, gateway defaults, or auth policy would be broader than the reported UI side effect.
