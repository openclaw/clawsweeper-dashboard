## Summary

Keep open: current main still only supports global session age/count/disk cleanup, while per-label count retention would add new session-maintenance semantics that need maintainer API and product review.

## What This Changes

Request adds per-label retention to `openclaw sessions cleanup` and config so the last N sessions can be kept per cron label instead of only by global thresholds.

## Best Possible Solution

Define an additive `session.maintenance` retention shape, then implement grouping in the shared session maintenance path with CLI, Gateway protocol, docs, and regression tests kept in sync.

## Reproduction Assessment

Not applicable; this is a feature request for new retention semantics, not a broken documented behavior. Source inspection confirms current main has global cap cleanup and existing cron labels that could serve as a grouping key.

## Solution Assessment

Unclear; per-label retention is plausible, but the exact CLI/config/protocol design is a maintainer product/API decision before implementation should start.
