## Summary

Keep open: current main and latest release still only support global session maintenance age/count limits, not per-label keep-N grouping. The open per-pattern retention discussion at https://github.com/openclaw/openclaw/issues/50900 overlaps, but it does not fully own this issue's count-based grouped retention behavior.

## What This Changes

Add per-label keep-N retention to `openclaw sessions cleanup` so cron/job labels retain balanced history instead of relying only on global caps.

## Best Possible Solution

Design one additive core session-retention rule contract that can express label/type grouped keep-N or deliberately fold this use case into the broader retention-rule design, then align schema, CLI/Gateway preview output, docs, and pruning tests.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high confidence that current main cannot express per-label keep-N cleanup.

## Solution Assessment

Unclear until maintainers choose the retention contract. Per-label keep-N is plausible, but it should not be added as a parallel destructive cleanup knob without coordinating with the broader retention-rule design.
