## Summary

Keep open: current main still only supports global session cleanup limits, while this request adds new per-label retention behavior that needs maintainer API/product review before implementation.

## What This Changes

Add per-label retention to `openclaw sessions cleanup` so the last N sessions can be kept per label or cron job type instead of only by global age/count limits.

## Best Possible Solution

Define an additive `session.maintenance` retention policy first, then implement grouping in the shared session maintenance path with CLI, Gateway protocol, docs, and tests kept in sync.

## Reproduction Assessment

Not applicable; this is a feature request for new retention semantics, not a report of broken documented behavior. Source inspection does confirm current main lacks per-label grouping while cron sessions already carry labels.

## Solution Assessment

Unclear; per-label retention is plausible, but the exact CLI/config/protocol shape is a maintainer product/API decision. If accepted, the shared session maintenance path is the narrowest maintainable implementation point.
