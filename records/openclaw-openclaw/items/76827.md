## Summary

Keep open: current main still uses global session maintenance age/count caps plus cron-run age retention, while this request adds new per-label keep-N grouping semantics that are not implemented and need maintainer API design.

## What This Changes

Request adding per-label retention to `openclaw sessions cleanup` so cleanup can keep the last N sessions per job/session label instead of applying only global caps.

## Best Possible Solution

Design an additive retention-policy contract first, then implement label-aware grouping in the central session maintenance path with schema, CLI/Gateway parameters, docs, and focused pruning coverage kept in sync.

## Reproduction Assessment

Not applicable; this is a feature request for new retention semantics, not a broken documented behavior. Source inspection confirms labels exist on session entries, but current cleanup and capping do not group by label.

## Solution Assessment

Unclear pending maintainer design. Per-label keep-N retention is plausible, but the narrow maintainable solution should be coordinated with the broader retention-rule contract and destructive cleanup safeguards.
