## Summary

Keep open: current main and latest release still only support global session cleanup age/count retention, and the related open per-pattern discussion does not fully own this issue's per-label keep-N behavior.

## What This Changes

The issue requests per-label session cleanup so `openclaw sessions cleanup` can keep the last N sessions per cron/job label instead of only applying global age and count limits.

## Best Possible Solution

Design one additive core session-retention contract that can express grouped keep-N by label or intentionally fold this use case into the broader retention-rule engine, then align schema, CLI/Gateway preview output, docs, and pruning tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main cannot express per-label keep-N cleanup.

## Solution Assessment

Unclear until maintainers choose the retention contract. Per-label keep-N is plausible, but it should not be added as a parallel destructive cleanup knob without coordinating with the broader retention-rule design.
