## Summary

Keep open: current main still applies session cleanup globally, while this request adds new per-label retention behavior plus config, CLI, and Gateway API surface that needs maintainer design before implementation.

## What This Changes

Request adds per-label session cleanup retention so `openclaw sessions cleanup` can keep the last N sessions per label/job type instead of only applying global age/count limits.

## Best Possible Solution

Decide the additive `session.maintenance` retention contract first, then implement grouping in the shared session maintenance path with CLI, Gateway schema, docs, and regression tests kept in sync.

## Reproduction Assessment

Not applicable; this is a feature request for new retention semantics rather than broken documented behavior. Source inspection does confirm current main lacks per-label grouping while cron sessions already have labels.

## Solution Assessment

Unclear; per-label retention is plausible, but the exact config/CLI/Gateway shape is a maintainer product and API decision before a narrow implementation should start.
