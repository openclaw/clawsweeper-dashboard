## Summary

Keep open: current main and v2026.6.6 still implement global session cleanup age/count retention, not per-label grouped keep-N retention. The related per-pattern retention issue overlaps, but it does not explicitly own count-based grouping by label, so maintainer product/config design is still needed.

## What This Changes

The issue requests `openclaw sessions cleanup` support for keeping the last N sessions per session label or job type instead of only applying global age/count caps.

## Best Possible Solution

Define one additive session-retention policy that can express label/type grouped keep-N or deliberately folds this use case into the broader retention-rule contract, then implement it centrally with schema, CLI/Gateway, docs, and pruning tests aligned.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high confidence that current main cannot express per-label keep-N cleanup.

## Solution Assessment

Unclear until maintainers choose the public retention contract. Per-label keep-N is plausible, but it should be coordinated with the broader retention-rule design rather than added as a parallel destructive cleanup knob.
