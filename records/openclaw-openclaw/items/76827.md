## Summary

Keep open: current main still lacks grouped keep-N retention by session label. The merged dry-run label summary and model-run cleanup are useful adjacent work, but neither implements the requested destructive retention behavior, so the remaining work needs maintainer product/API direction.

## What This Changes

Add session cleanup behavior that groups sessions by label and keeps the most recent N entries per label instead of relying only on global age/count thresholds.

## Best Possible Solution

Choose one maintainer-approved session-retention contract that either supports grouped keep-N by label directly or folds it into broader retention rules, then update schema, cleanup paths, docs, and tests together.

## Reproduction Assessment

## Solution Assessment

Unclear. Per-label keep-N is plausible, but the best fix should be integrated with the broader session-retention policy instead of adding an independent destructive cleanup knob.
