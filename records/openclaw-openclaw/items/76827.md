## Summary

Keep open: current main and the latest release still implement global session-maintenance age/count caps, not per-label keep-N grouping. The related per-pattern retention issue overlaps, but it does not fully cover this issue's count-based grouped retention behavior, so duplicate closure would be premature.

## What This Changes

Request per-label session cleanup retention so `openclaw sessions cleanup` can keep the last N sessions per cron/job label instead of only applying global age/count thresholds.

## Best Possible Solution

Design one additive session retention policy that can either express label/type grouped keep-N or explicitly fold this use case into the broader per-pattern retention contract, then implement it centrally with schema, CLI/Gateway, docs, and pruning tests aligned.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high confidence that current main cannot express per-label keep-N cleanup: labels exist on session rows, but cleanup/capping do not group by label.

## Solution Assessment

Unclear until maintainers choose the public retention contract. Per-label keep-N is plausible, but it should be coordinated with the broader retention-rule design rather than added as a parallel destructive cleanup knob.
