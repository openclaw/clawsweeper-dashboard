## Summary

Keep open: current main still caps session entries by global updatedAt order, while this request adds new per-label retention behavior and config/CLI semantics that need maintainer API review.

## What This Changes

Add per-label retention to `openclaw sessions cleanup` so it can keep the most recent N sessions for each cron/job label instead of only applying global age/count caps.

## Best Possible Solution

Have maintainers choose the additive `session.maintenance` API shape, then implement grouping in the shared session maintenance path with CLI, schema/help/docs, dry-run output, Gateway cleanup, and focused pruning tests kept aligned.

## Reproduction Assessment

Not applicable; this is a feature request for new retention semantics, not a report of broken documented behavior. Source inspection confirms current main has global `maxEntries` cleanup and no per-label CLI/config field.

## Solution Assessment

Unclear; grouping by label is plausible, but the requested CLI/config shape is not yet established and must fit existing protected-entry, disk-budget, and Gateway cleanup behavior.
