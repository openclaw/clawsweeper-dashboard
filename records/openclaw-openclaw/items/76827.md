## Summary

Keep open: current main only supports global session cleanup limits, while this request adds new per-label retention behavior that needs maintainer API/product review before implementation.

## What This Changes

Add a per-label retention mode to `openclaw sessions cleanup` so it can keep the newest N sessions independently for each cron/job label instead of relying only on global age/count limits.

## Best Possible Solution

Decide the additive `session.maintenance` API first, then implement per-label retention in the shared session maintenance path with CLI/Gateway/docs/tests kept in sync.

## Reproduction Assessment

Not applicable; this is a feature request for new retention semantics, not a report of broken documented behavior. Source inspection does confirm current main lacks per-label grouping while cron labels exist.

## Solution Assessment
