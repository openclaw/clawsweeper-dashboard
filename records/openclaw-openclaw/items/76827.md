## Summary

Keep open: current main and v2026.6.8 still only support global session cleanup retention, plus label-aware dry-run reporting; grouped keep-N retention by session label remains a real feature gap that needs maintainer product/API direction.

## What This Changes

Add grouped session cleanup retention so `openclaw sessions cleanup` can keep the newest N sessions per label instead of pruning only by global age/count thresholds.

## Best Possible Solution

Design one additive core session-retention contract that can express grouped keep-N by label or deliberately fold this use case into the broader retention-rule engine, then align schema, CLI/Gateway cleanup, docs, and pruning tests.

## Reproduction Assessment

Yes at source level for the missing capability: current main exposes only global session-maintenance age/count/disk-budget fields and cleanup functions, while grouped keep-N by label is absent. This is a feature-gap reproduction rather than a failing bug repro.

## Solution Assessment

Unclear until maintainers choose the retention contract. Per-label keep-N is plausible, but it should not be added as a parallel destructive cleanup knob without coordinating with the broader retention-rule design.
