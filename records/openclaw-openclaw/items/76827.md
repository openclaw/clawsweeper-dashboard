## Summary

Keep open: current main still only supports global session cleanup limits, while this request asks for new per-label count retention across CLI/config/Gateway behavior and needs maintainer API review before implementation.

## What This Changes

Add per-label retention to session cleanup so the last N sessions can be kept per cron/job label instead of only by global age/count thresholds.

## Best Possible Solution

Decide the additive `session.maintenance` retention API first, then implement label-grouped count cleanup in the shared maintenance path with CLI, Gateway protocol, docs, and regression tests kept in sync.

## Reproduction Assessment

Not applicable; this is a feature request for new retention semantics, not a broken documented behavior. Source inspection confirms current main has cron labels but no per-label cleanup grouping.

## Solution Assessment

Unclear; per-label retention is plausible, but the exact CLI/config/Gateway contract is the maintainer decision. The safest implementation would be additive under the existing session maintenance surface rather than a parallel cleanup path.
