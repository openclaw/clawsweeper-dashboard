Full review comments:

- **[P2] Deduplicate overlapping page-group scans:** `extensions/memory-wiki/src/query.ts:267-268`
  - body: The README now documents `dir: "."`, but this path builds the scan list directly from every page group. If a user sets a root page group, `listWikiMarkdownFiles` recursively reads the whole vault and then the default-filled group dirs are scanned again, so the same nested pages can appear twice in search/index data and counts. Build a canonical non-overlapping path set or dedupe relative paths before reading pages.
  - confidence: 0.88
- **[P2] Keep manifest validation in sync with runtime:** `extensions/memory-wiki/openclaw.plugin.json:215-218`
  - body: Runtime `WikiPageGroupSchema` rejects absolute and traversal dirs, but the manifest schema only requires a non-empty string. That means discovery/setup validation can accept config the runtime silently drops or falls back from, which is exactly the kind of config-contract drift this new surface should avoid.
  - confidence: 0.82
- **[P3] Document page groups in published plugin docs:** `extensions/memory-wiki/README.md:74-80`
  - body: This adds the new pageGroups surface only to the extension README, while the published memory-wiki docs still describe fixed English directories. Please update the public docs source for the new config, merge precedence, validation limits, and migration expectations so docs.openclaw.ai matches the shipped behavior.
  - confidence: 0.83
