- **[P1] Update markdown helpers before passing pageGroups:** `extensions/memory-wiki/src/query.ts:282`
  - body: `toWikiPageSummary` and `inferWikiPageKind` still have the old signatures in this branch, but the new query/status code passes `pageGroups` into them. That will fail type checking, and even if ignored, custom directories cannot be classified because the helper still checks only the hard-coded English prefixes.
  - confidence: 0.97
- **[P1] Sync pageGroups into the manifest schema:** `extensions/memory-wiki/src/config.ts:199`
  - body: The PR documents and parses `plugins.entries.memory-wiki.config.pageGroups`, but the static `openclaw.plugin.json` config schema still has `additionalProperties: false` and no `pageGroups` property. The plugin loader validates config against that manifest schema before runtime activation, so the documented openclaw.json config path is rejected.
  - confidence: 0.96
- **[P1] Thread pageGroups through compile and indexes:** `extensions/memory-wiki/README.md:76-77`
  - body: The README says custom dirs are scanned and indexed, but `compile.ts` still uses hard-coded `COMPILE_PAGE_GROUPS` and non-recursive per-dir collection. `openclaw wiki compile` will ignore pages that live only in custom or nested page-group directories, so generated indexes and digests will not match the new config.
  - confidence: 0.94
- **[P1] Preserve QMD session visibility fallbacks:** `extensions/memory-wiki/src/query.ts:1322-1335`
  - body: This rewrite drops the current `liveStem` and `allowQmdSlugFallback` handling used for QMD exported session paths, while the memory-core sibling still preserves that same fallback. Existing allowed session transcript reads such as archived or slugified QMD exports can disappear after upgrade.
  - confidence: 0.92
- **[P2] Restore finite integer normalization:** `extensions/memory-wiki/src/query.ts:1473`
  - body: Replacing `normalizePositiveInteger` with `Math.max(1, value ?? fallback)` lets `NaN` stay `NaN`, leaves fractional values fractional, and allows `Infinity` through. Current tests cover this behavior for wiki_get line windows, so the PR should keep the finite integer normalization helper or equivalent logic.
  - confidence: 0.9
- **[P1] Create custom directories through the safe root:** `extensions/memory-wiki/src/vault.ts:95-100`
  - body: `buildVaultDirectories(config.pageGroups)` now makes the directory list vault/config-controlled, but initialization still calls raw `fs.mkdir(path.join(rootDir, relativeDir))`. String validation blocks `..`, but it does not prove symlink-parent behavior; use the existing safe-root/within-root helpers or add equivalent security proof before accepting vault-local directory config.
  - confidence: 0.78
