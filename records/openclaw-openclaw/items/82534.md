  - body: This replaces `fsRoot(rootDir).create(relativePath, ...)` with `fs.access` plus raw `fs.writeFile(filePath, ...)` for plugin-owned vault files.   - confidence: 0.9

  - body: Removing `extends: "../tsconfig.package-boundary.base.json"` makes memory-wiki fall out of the opt-in extension package-boundary set, and current contracts assert every extension tsconfig uses that shared boundary plus the standard include/exclude shape. Keep the boundary config and add only the minimal local override needed for this plugin.
  - confidence: 0.96
- **[P2] Walk query directories recursively too:** `extensions/memory-wiki/src/query.ts:240-245`
  - body: The compile path now recurses, but wiki search/get fallback still uses `fs.readdir` and filters only direct `.md` children of each page-group dir. Nested pages under custom directories will be indexed by compile but missed by query fallback when the digest does not provide an exact candidate path, so share the recursive walk behavior here as well.
  - confidence: 0.87
- **[P2] Pass page groups into status source summaries:** `extensions/memory-wiki/src/status.ts:119-123`
  - body: `collectVaultCounts` detects the configured source dir, but then parses each source page without passing `config.pageGroups`; for a custom source directory, `toWikiPageSummary` returns null and source-type counts stay at zero. Thread the page groups through this call the same way the page-count path does.
