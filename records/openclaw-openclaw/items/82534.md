  - confidence: 0.96

  - body: Defaulting `pageGroupDirs` to `[]` makes existing callers that do not pass config see an empty vault; for example, `wiki_apply update_metadata` calls the reader without config before resolving the target page. Make the config explicit at all call sites or fall back to the built-in groups.
  - confidence: 0.94
- **[P1] Let custom groups override built-in directories:** `extensions/memory-wiki/src/config.ts:347`
  - body: The resolved list prepends `DEFAULT_PAGE_GROUPS`, and `getDefaultDirForKind` returns the first matching kind, so writers still choose `sources`, `syntheses`, and `reports` before the user's localized directories. That contradicts the advertised custom directory behavior and leaves mixed vault layouts.
  - confidence: 0.93
- **[P1] Keep page writes behind fs-safe:** `extensions/memory-wiki/src/apply.ts:168-169`
  - body: This replaces the rooted fs-safe writer with `fs.mkdir`/`fs.writeFile` on an absolute path. Vault writes should stay behind `fsRoot(...).write/create` or an equivalent rooted helper so symlink, hardlink, and root-boundary protections are preserved.
  - confidence: 0.92
- **[P2] Restore the extension package-boundary tsconfig:** `extensions/memory-wiki/tsconfig.json:2-9`
  - body: Dropping `extends: "../tsconfig.package-boundary.base.json"` removes the package-boundary compile contract that keeps plugins on public SDK seams. Keep the base config and only add the minimum local options needed for memory-wiki.
  - confidence: 0.88
