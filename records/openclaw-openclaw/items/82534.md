## Summary

Keep this PR open for revision: the localized vault idea is useful and the discussion links a strong narrow recursion alternative, but the current merge result does not compile, compile/index generation still depends on fixed English directories, and the static plugin manifest does not accept the new configuration.

## What This Changes

This PR adds configurable memory-wiki page groups through plugin and vault-local configuration, recursively scans page directories, and routes selected query, compile, apply, and import paths through configured directories.

## Best Possible Solution

Choose one canonical, manifest-declared page-group contract; wire it through every vault reader, writer, compiler, index, status, lint, and import path; reject overlapping or unbounded groups unless explicitly approved; and land the separately proved recursion fix independently if the broader configuration decision is deferred.

## Solution Assessment

No. Configurable localized directories are reasonable, but this implementation creates parallel configuration authorities and changes only part of the whole read/write/index surface; the narrower recursion fix is already isolated in https://github.com/openclaw/openclaw/pull/90870.
