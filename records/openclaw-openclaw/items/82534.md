  - confidence: 0.91
- **[P1] Write dashboards under the configured report group:** `extensions/memory-wiki/src/compile.ts:83`
  - body: Generated dashboard definitions still point at `reports/*.md`. If a user configures the `report` group to another directory, compile will keep using or recreating the English `reports` path instead of the configured report location, so reports are split or missing from the configured vault view.
  - confidence: 0.9
- **[P2] Make page group scans recursive in query and compile:** `extensions/memory-wiki/src/query.ts:251-256`
  - body: The changed scanner still filters only immediate files returned by `readdir`; there is no directory recursion here, and compile uses the same one-level pattern. That contradicts the PR’s recursive scanning behavior and means nested custom pages are omitted from search, digest/index generation, and follow-on get paths while status can count them.
  - confidence: 0.88
- **[P2] Mirror dir validation in the manifest schema:** `extensions/memory-wiki/openclaw.plugin.json:215-218`
  - body: Runtime validation rejects absolute paths and traversal for `pageGroups[].dir`, but the public manifest schema only requires a non-empty string. Metadata-based config validation can therefore accept unsafe values that runtime later strips or rejects, which is a config contract mismatch for a filesystem path surface.
  - confidence: 0.84
