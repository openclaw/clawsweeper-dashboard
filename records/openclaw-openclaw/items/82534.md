## Summary

Keep open: the feature direction is plausible and the author added live-output proof plus follow-up commits, but the latest head still breaks wiki query/default-reader paths and bypasses fs-safe vault writes.

## What This Changes

The branch adds configurable memory-wiki page groups, vault-local `.wiki-page-groups.json` loading, recursive compile scanning, dynamic directory resolution, docs, and tests.

## Best Possible Solution

Ship this only after the config contract is owner-reviewed, all read/search/write/status paths use the resolved page groups consistently, vault writes stay behind fs-safe, and package-boundary checks remain enabled.

## Reproduction Assessment

Yes for the PR regressions by source inspection: the latest head still passes empty page groups or omits config in wiki query readers, while current-main callers still rely on the default reader contract. I did not run tests in this read-only review.

## Solution Assessment

No. The requested capability is useful, but this branch is not the best mergeable shape until it preserves defaults, fs-safe writes, recursive query coverage, package-boundary checks, and clear precedence semantics.

  - body: `searchWikiCorpus` accepts `pageGroups`, but the actual candidate and remaining corpus reads still pass `[]`, and the no-digest path still calls `readQueryableWikiPages` without config. That means `wiki_search` can return no wiki pages even after the caller provides resolved groups.

  - confidence: 0.96
- **[P1] Keep vault writes behind fs-safe:** `extensions/memory-wiki/src/apply.ts:168-169`
  - body: This change replaces fs-safe rooted writes with absolute `fs.mkdir`/`fs.writeFile` calls. Config validation rejects simple `..` dirs, but direct filesystem writes still bypass the root-relative fs-safe behavior that current memory-wiki write paths use, including symlink/root checks.
  - confidence: 0.9
- **[P2] Let configured dirs override built-ins:** `extensions/memory-wiki/src/config.ts:337-349`
  - body: Resolved groups append user and vault groups after the defaults, while `findDirForKind` returns the first match. As a result, `getDefaultDirForKind` keeps choosing `sources`/`syntheses`/`reports`, so custom dirs are scanned but do not become the default write targets promised by the feature.
  - confidence: 0.91
- **[P2] Walk query directories recursively:** `extensions/memory-wiki/src/query.ts:238-245`
  - body: Compile now walks nested directories, but the query reader still only lists immediate markdown files in each configured dir. Nested pages can appear in generated indexes while `wiki_search` and `wiki_get` miss them.
  - confidence: 0.9
- **[P2] Keep memory-wiki on package-boundary tsconfig:** `extensions/memory-wiki/tsconfig.json:1-14`
  - body: Dropping the shared package-boundary base config opts this bundled plugin out of the checks that keep extension code on public plugin contracts. Keep the shared `extends` and adjust includes/excludes inside that contract.
  - confidence: 0.88
