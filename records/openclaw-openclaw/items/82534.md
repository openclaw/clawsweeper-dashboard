Full review comments:

- **[P1] Pass page groups through wiki search:** `extensions/memory-wiki/src/query.ts:1330-1331`
  - body: `searchWikiCorpus` accepts `pageGroups` but still reads candidate and fallback pages with `[]` or no config. Custom-group vaults can produce status/index output while `wiki_search` misses the same pages; thread the resolved groups through all three reads.
  - confidence: 0.96
- **[P1] Preserve defaults for config-less page readers:** `extensions/memory-wiki/src/query.ts:256-258`
  - body: The exported `readQueryableWikiPages(rootDir)` now turns missing config into an empty directory list. Existing callers such as `wiki_apply`, memory-palace, and import-insights still use that form, so they will read zero pages unless this falls back to the default groups or those callers pass config.
  - confidence: 0.95
- **[P1] Keep wiki writes behind fs-safe:** `extensions/memory-wiki/src/apply.ts:168-169`
  - body: This replaces the current fs-safe root write with direct `fs.mkdir`/`fs.writeFile` on vault-derived paths. Keep vault-relative writes through the established fs-safe helper so configured dirs and existing vault entries do not bypass the symlink/path protections current main relies on.
  - confidence: 0.88
- **[P2] Honor configured groups before defaults:** `extensions/memory-wiki/src/config.ts:337`
  - body: The resolver prepends built-in defaults before user and vault groups, but `findDirForKind` returns the first match. A configured source directory like `资料` therefore cannot become the default write target, which contradicts the feature’s custom-directory behavior.
  - confidence: 0.9
- **[P2] Walk query page groups recursively:** `extensions/memory-wiki/src/query.ts:240-245`
  - body: Compile now recursively walks nested directories, but the query reader still keeps only direct Markdown children. Nested pages can appear in generated indexes while `wiki_search` and lookup miss them.
  - confidence: 0.9
- **[P2] Keep memory-wiki on the package-boundary tsconfig:** `extensions/memory-wiki/tsconfig.json:2-9`
  - body: Removing the shared package-boundary `extends` opts memory-wiki out of the contract that keeps bundled plugins compiled against the public SDK surface and also includes tests/helpers in this package tsconfig. Restore the shared base config and adjust only through that contract.
  - confidence: 0.9
