## Summary

Keep open: the feature direction is useful and the PR body now has real behavior proof, but the branch still introduces blocking memory-wiki correctness and security regressions around page-group propagation, session visibility, vault-relative path safety, reports, and plugin package-boundary checks.

## What This Changes

The PR adds configurable memory-wiki page groups, vault-local `.wiki-page-groups.json` loading, recursive compile scanning, dynamic directory resolution, docs, and related tests.

## Best Possible Solution

Ship page groups only after validating vault-relative dirs, preserving default reports/search behavior, passing resolved groups through every reader and writer, keeping sandboxed/fs-safe guards, and retaining the shared plugin-boundary tsconfig contract.

## Reproduction Assessment

Yes for the PR regressions by source inspection: the PR head defaults missing page groups to no dirs, passes empty groups through search, removes sandboxed propagation, and accepts unvalidated dirs that reach filesystem writes. I did not run tests because this review is read-only.

## Solution Assessment

No. The requested capability is reasonable, but this branch is not the narrowest maintainable merge path until it preserves existing defaults, security boundaries, package-boundary checks, and the full wiki read/search/status contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.95

Full review comments:

- **[P1] Pass page groups into wiki search:** `extensions/memory-wiki/src/query.ts:1329-1330`
  - body: `searchMemoryWiki` resolves `effectiveConfig.pageGroups`, but `searchWikiCorpus` still receives only the vault path and then calls the page readers with `[]` or no config. With custom groups, ordinary `wiki_search` can return no wiki pages or skip the configured directories.
  - confidence: 0.96
- **[P1] Preserve default groups for existing readers:** `extensions/memory-wiki/src/query.ts:256-257`
  - body: The shared `readQueryableWikiPages(rootDir)` fallback now maps missing config to an empty directory list. Existing callers such as wiki apply, memory-palace, and import-insights still call it without page groups, so they can read zero pages on the default path.
  - confidence: 0.95
- **[P1] Keep sandboxed visibility on wiki tools:** `extensions/memory-wiki/src/tool.ts:143`
  - body: The tool calls no longer forward `memoryContext.sandboxed` into `searchMemoryWiki` or `getMemoryWikiPage`. That bypasses the current shared-memory visibility guard for sandboxed wiki calls that do not also carry a session key.
  - confidence: 0.96
- **[P1] Reject dirs that escape the vault:** `extensions/memory-wiki/src/config.ts:133`
  - body: `dir` accepts any non-empty string and later flows into direct `path.join` plus `fs.mkdir`/`fs.writeFile` calls. Values like `../outside` need to be rejected, or writes need to stay behind the fs-safe root wrapper, before this config can be safe.
  - confidence: 0.94
- **[P2] Walk query page groups recursively:** `extensions/memory-wiki/src/query.ts:238-245`
  - body: Compile now recursively walks nested directories, but the query reader still lists only immediate files in each group dir. Nested pages can appear in generated indexes while `wiki_search` and lookup miss them.
  - confidence: 0.9
- **[P2] Keep reports in the default page groups:** `extensions/memory-wiki/src/config.ts:118-123`
  - body: Current main indexes and counts `reports/`, and this branch still writes dashboards there as a fallback. Because the default page groups omit `report`, fresh default vaults can create reports that are omitted from summaries, counts, search, and indexes.
  - confidence: 0.89
- **[P2] Honor configured dirs before built-in defaults:** `extensions/memory-wiki/src/config.ts:315`
  - body: The resolved pageGroups prepend built-in defaults before user and vault groups, while `findDirForKind` returns the first match. A configured source dir such as `资料` therefore cannot become the default write target even though the feature promises configurable directories.
  - confidence: 0.88
- **[P2] Keep memory-wiki in package-boundary checks:** `extensions/memory-wiki/tsconfig.json:2-9`
  - body: Dropping `../tsconfig.package-boundary.base.json` opts this bundled plugin out of the shared extension boundary contract and changes test/source inclusion at the same time. Keep the shared base and adjust includes through that contract.
  - confidence: 0.9
