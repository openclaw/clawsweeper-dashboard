## Label Justifications

- `P1`: If confirmed in native/plugin harness runs, the missing fail-closed input gate affects real external-channel agent workflows that depend on security plugins.
- `impact:security`: The report is about security and compliance plugins being unable to inspect or block complete input context before model consumption.

## Summary

Keep open. The PI embedded runner and CLI paths already call `before_agent_run`, but the agent-harness/Codex app-server path exported through `agent-harness-runtime` still wires `before_prompt_build` and starts the model turn without the input gate, so a meaningful cross-runtime security gap remains.

## What This Changes

Add `before_agent_run` input-gate support to embedded/native agent harness runs so external channel messages cannot bypass security plugins before model submission.

## Best Possible Solution

Add a shared agent-harness `before_agent_run` helper and call it in Codex/native harness attempts after the final prompt and developer instructions are assembled but before `llm_input` and native turn start, preserving the PI/CLI redacted block semantics.

## Reproduction Assessment

Yes. Source inspection shows the Codex/native agent-harness path applies `before_prompt_build` and then starts the model turn without any `before_agent_run` call, while PI and CLI do have that gate; I did not execute a live channel run in this read-only review.

## Solution Assessment

Yes. The maintainable fix is to add one shared agent-harness gate helper and use it from native harness attempts, rather than duplicating policy inside individual channels or closing this as already covered by the PI-only path.
