## Summary

Keep this PR open: the linked bug is still present on current main and the PR is directionally useful, but the proposed fallback can leak existing silent control tokens into user-visible background-task completion text and still needs the required changelog entry.

## What This Changes

The PR changes ACP background-task handling to return accumulated progress as `terminalSummary` for non-blocked completions and raises the progress-summary cap from 240 to 1200 characters.

## Best Possible Solution

Land a narrow ACP manager fix that copies cleaned non-blocked progress into `terminalSummary`, strips exact or trailing silent control tokens, preserves blocked-outcome detection, and adds focused regression coverage plus the required changelog entry.

## Reproduction Assessment

Yes. On current main, a parent-owned ACP run that emits output `text_delta` progress and then `done` stores `progressSummary` but a null `terminalSummary`; with this PR as written, the same path ending in `ANNOUNCE_SKIP` would deliver that control token as visible terminal text.

## Solution Assessment

No, not as submitted. The fallback-and-cap direction is the narrow maintainable fix for the linked bug, but the fallback needs silent-token cleanup and regression coverage before merge.
