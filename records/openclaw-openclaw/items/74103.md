## Best Possible Solution

Land a narrow ACP manager fix that preserves the progress-to-terminal fallback and larger cap, strips non-deliverable trailing tokens before fallback delivery, and adds focused regression coverage.

## Reproduction Assessment

Yes. Current-main source shows ACP progress is accumulated, the success path only stores resolver output as `terminalSummary`, terminal delivery only formats `terminalSummary`, and the resolver still returns `{}` for ordinary non-blocked progress.

## Solution Assessment
