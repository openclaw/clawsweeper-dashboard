## Summary

Keep open. The PR is useful source work for the linked ACP background-task bug, but it is not merge-ready because the new fallback can surface silent control tokens as visible completion text, and the user-facing fix also needs the required changelog entry.

## What This Changes

The PR changes `src/acp/control-plane/manager.core.ts` to raise the ACP background-task progress cap from 240 to 1200 characters and return normalized non-blocked progress as `terminalSummary`.

## Best Possible Solution

Land a narrow ACP manager fix that copies cleaned non-blocked progress into `terminalSummary`, preserves blocked-outcome detection, strips exact or trailing silent-control tokens before terminal delivery, covers the ordinary success and token cleanup cases in tests, and adds a single-line Unreleased changelog entry.

## Reproduction Assessment

Yes. The source path is high-confidence: ACP `text_delta` output accumulates into `taskProgressSummary`, `resolveBackgroundTaskTerminalResult()` currently returns `{}` for ordinary non-blocked progress, terminal storage records `null`, and the delivery formatter only emits `terminalSummary`.

## Solution Assessment

No, not as submitted. The fallback-and-cap direction is the narrow maintainable fix, but returning raw normalized progress should be adjusted to strip silent control tokens and accompanied by regression tests plus the required changelog entry.
