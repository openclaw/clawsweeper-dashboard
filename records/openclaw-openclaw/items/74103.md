## Summary

Keep open; this PR addresses a real current-main ACP background-task completion bug, but the submitted fallback is not merge-ready because it can expose existing silent control tokens as visible terminal text and it lacks the required regression coverage/changelog follow-through.

## What This Changes

The PR raises the ACP background-task progress cap from 240 to 1200 characters and returns non-blocked progressSummary as terminalSummary from resolveBackgroundTaskTerminalResult().

## Best Possible Solution

Land a narrow ACP manager fix that copies cleaned non-blocked progress into terminalSummary, strips exact/trailing silent control tokens, preserves blocked-outcome detection, and adds focused regression coverage plus the required changelog entry.

## Reproduction Assessment

Yes. On current main, a parent-owned ACP run that emits output text_delta progress and then done stores progressSummary but null terminalSummary; in this PR, the same path with progress ending in ANNOUNCE_SKIP would format that control token as visible terminal text.

## Solution Assessment

No, not as submitted. The fallback-and-cap direction is the narrow maintainable fix for #74070, but the fallback needs silent-token cleanup and regression coverage before merge.
