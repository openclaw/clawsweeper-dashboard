## Summary

Keep open: current main still has the reported ACP terminal-summary gap, but this PR should not merge as submitted because the fallback can expose silent control tokens and the external contributor has not supplied real after-fix behavior proof.

## What This Changes

The PR raises the ACP background-task progress cap from 240 to 1200 and returns normalized progress as `terminalSummary` for non-blocked completions.

## Best Possible Solution

Land a narrow ACP manager fix that keeps the streamed-whitespace behavior on main, strips/suppresses silent control tokens before fallback, chooses the cap deliberately, and adds focused regression tests.

## Reproduction Assessment

Yes. Source inspection shows ACP progress is accumulated, ordinary successful progress resolves to `{}`, the success path persists `terminalSummary: null`, and terminal delivery formats only `terminalSummary`.

## Solution Assessment

No, not as submitted. Returning progress as a fallback terminal summary is the right narrow direction, but the PR needs silent-token stripping and regression coverage before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Strip silent tokens before terminal fallback:** `src/acp/control-plane/manager.core.ts:144`

Clean exact/trailing silent-control tokens before returning the fallback summary and cover that case.
