## Summary

Keep open. Current main still drops `agentId` and `sessionId` from embedded compaction hook callbacks, while this PR's narrow forwarding matches the existing params and hook context contract; merge should wait for real behavior proof and refreshed validation.

## What This Changes

The PR forwards existing `agentId`, `sessionId`, and `sessionKey` values into embedded before/after compaction plugin hook contexts and updates the compaction hook wiring test.

## Best Possible Solution

Land a rebased version of this focused embedded compaction hook context fix after contributor real behavior proof, while leaving broader hook trace-context design to https://github.com/openclaw/openclaw/issues/50291.
