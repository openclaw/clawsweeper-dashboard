## Summary

Keep this PR open. Current main still drops `agentId` and `sessionId` from embedded compaction plugin hook contexts even though the subscription params already carry them; the PR is a narrow fix but still needs a changelog entry and current-main validation before merge.

## What This Changes

The PR forwards embedded compaction `agentId`, `sessionId`, and `sessionKey` into before/after compaction plugin hook callbacks and updates the compaction hook wiring assertions.

## Best Possible Solution

Rebase or adapt the PR to current main, forward the existing scoped fields at both embedded compaction hook sites, add the single-line `CHANGELOG.md` fix entry, and validate the focused wiring test plus changed gate.

## Reproduction Assessment

Yes. On current main, an embedded compaction context with `ctx.params.agentId` and `ctx.params.sessionId` still results in `runBeforeCompaction` and `runAfterCompaction` receiving only `sessionKey`.

## Solution Assessment

Yes, with changelog and validation caveats. Forwarding fields already present on `SubscribeEmbeddedPiSessionParams` is the narrowest maintainable fix and avoids expanding the plugin hook API beyond existing context fields.
