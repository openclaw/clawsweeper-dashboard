## Summary

Keep open: current main still drops embedded compaction `agentId` and `sessionId` before plugin hooks, and this PR is a narrow fix, but it still needs the required changelog entry before merge.

## What This Changes

The PR forwards `agentId`, `sessionId`, and `sessionKey` into embedded before/after compaction plugin hook callbacks and updates compaction hook wiring tests.

## Best Possible Solution

Forward the existing scoped fields at both embedded compaction hook sites, keep the focused wiring assertions, add one single-line `CHANGELOG.md` fix entry, and validate the focused test plus changed gate.

## Reproduction Assessment

Yes, from source inspection. On current main, embedded compaction handlers receive `ctx.params.agentId` and `ctx.params.sessionId` but pass only `sessionKey` into `runBeforeCompaction` and `runAfterCompaction`.

## Solution Assessment

Yes, with the changelog caveat. Forwarding fields already present on `SubscribeEmbeddedPiSessionParams` is the narrowest maintainable fix and keeps parity with the direct compaction helper without expanding the plugin hook contract.
