## Summary

Keep open. Current main still omits `agentId` and `sessionId` from embedded compaction hook contexts even though the subscribe params and hook context type already carry them; this PR is a narrow fix direction, but it still needs real behavior proof and a rebase that preserves the merged `sessionFile` behavior.

## What This Changes

The PR forwards existing `agentId` and `sessionId` values into embedded before/after compaction plugin hook contexts and updates compaction hook wiring tests.

## Best Possible Solution

Land a refreshed version of this focused embedded compaction hook context fix after real behavior proof, preserving the existing `sessionFile` behavior and leaving broader hook observability fields to https://github.com/openclaw/openclaw/issues/50291.

## Reproduction Assessment

Yes. Source inspection on current main shows subscribe params carry `agentId` and `sessionId`, while embedded before/after compaction handlers only pass `sessionKey`; I did not run a live plugin reproduction in this read-only review.

## Solution Assessment

Yes, with proof and rebase. Forwarding already-scoped fields through `PluginHookAgentContext` is the narrowest maintainable fix, but the branch should refresh to current main and preserve the merged `sessionFile` hook behavior.
