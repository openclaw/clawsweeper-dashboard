## Summary

Keep this PR open, but it still needs changes before merge: the additive API idea is useful, but the current branch does not enforce the deny override on CLI-backed runs and newly documents at least one tool group more broadly than the runtime expands it.

## What This Changes

The PR adds an OpenClaw-specific `/v1/responses` `tool_deny` array, threads it into gateway and embedded-agent tool filtering, and documents per-request tool denial.

## Best Possible Solution

Implement the request field through a single run-scoped policy contract that is honored by embedded runs and either enforced or explicitly rejected by CLI loopback runs, with docs describing exact group expansion.

## Reproduction Assessment

Yes. Source inspection shows current main rejects `tool_deny`, and the PR blocker is reproducible from the PR-added `runCliAgent` argument plus the missing `RunCliAgentParams` and loopback deny path.

## Solution Assessment

No, not as-is. Reusing the policy pipeline is a reasonable embedded-run seam, but the feature must cover or fail closed on CLI-backed runs and its docs must match actual group expansion before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Enforce the deny override in CLI runs:** `src/agents/command/attempt-execution.ts:477`
  - body: The PR passes `toolDenyOverride` into `runCliAgent`, but `RunCliAgentParams` does not declare or consume that field. That should fail excess-property checking, and CLI-backed OpenResponses runs would still expose denied tools through the loopback MCP tool list unless the deny list is wired into that path or the CLI runner fails closed for this request.
  - confidence: 0.93
- **[P2] Document the actual openclaw group membership:** `docs/gateway/openresponses-http-api.md:145`
  - body: The new docs say `group:openclaw` disables all built-in tools, but the runtime builds that group only from tools marked `includeInOpenClawGroup`, excluding groups such as `group:fs` and `group:runtime`. A caller using `tool_deny: ["group:openclaw"]` to remove every built-in tool would still expose file/runtime tools, so this line should describe the actual group or point to the canonical group table.
  - confidence: 0.84
