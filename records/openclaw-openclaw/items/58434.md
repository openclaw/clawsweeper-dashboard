The PR adds a `tool_deny` request field to `/v1/responses`, forwards it through agent command options into embedded PI tool filtering, and documents per-request built-in tool denial.

## Best Possible Solution

Define a run-scoped deny contract that every supported `/v1/responses` backend either enforces or rejects fail-closed, document group semantics from the canonical catalog, and require redacted live gateway proof before merge.
