## Summary

Keep open: the feature is not on current main and the PR is useful, but it is not mergeable as-is because CLI-backed runs would not enforce the deny override, the new docs overstate `group:openclaw`, and external real behavior proof is still missing.

## What This Changes

This PR adds an OpenClaw-specific `tool_deny` array to `/v1/responses`, threads it through gateway and embedded-agent tool filtering, and documents per-request tool denial.

## Best Possible Solution

Add a single run-scoped deny policy for `/v1/responses` that is enforced or fail-closed across embedded and CLI-backed runs, with canonical group documentation and focused coverage.

## Reproduction Assessment

Yes. Source inspection verifies current main rejects `tool_deny` via the strict schema, and the PR blocker is source-reproducible because CLI runner params and preparation have only `toolsAllow` fail-closed behavior with no deny override enforcement.

## Solution Assessment

No, not as-is. Reusing the embedded policy pipeline is reasonable, but the feature must either enforce or reject the deny override for CLI-backed runs and document exact group expansion before merge.
