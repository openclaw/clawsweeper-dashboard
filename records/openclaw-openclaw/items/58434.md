## What This Changes

The branch adds an optional OpenClaw-specific `tool_deny` field to `/v1/responses`, forwards it through agent command options into embedded PI tool filtering, and documents per-request tool denial.

## Best Possible Solution

Define a run-scoped deny contract that every supported `/v1/responses` execution backend either enforces or rejects fail-closed, document group semantics from the canonical catalog, and require redacted live gateway proof before merge.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature PR. The review blockers are source-reproducible because the diff threads the override into embedded PI while current CLI and gateway loopback tool-resolution paths have no matching request-scoped deny input.

## Solution Assessment

No, not as-is. Reusing the embedded tool-policy pipeline is a reasonable component, but the public API needs backend-wide enforcement or fail-closed rejection and canonical group documentation before it is the maintainable solution.
