## Summary

Keep open: current main and the latest release still emit dotted provider IDs in Telegram model-list callbacks while the parser rejects dots, and the direct fix PRs are closed unmerged.

## What This Changes

Telegram `/models` should keep inline provider/model buttons working when a provider ID contains a dot, such as `hf.co`.

## Best Possible Solution

Land a narrow Telegram plugin fix that accepts dots in `mdl_list_{provider}_{page}` provider IDs, adds `hf.co` parser and keyboard regression coverage, and records a user-facing changelog fix entry.

## Reproduction Assessment

Yes. Source inspection plus a direct regex check show current main accepts `mdl_list_open-ai_1` but rejects `mdl_list_hf.co_1`, while the provider keyboard emits exactly that rejected callback for a dotted provider id.

## Solution Assessment

Yes. The narrowest maintainable fix is to expand only the model-list provider capture and add regression tests; no core model catalog or provider configuration behavior needs to change.
