## Summary

Current main still has the reported Telegram model-list callback mismatch: provider buttons emit raw provider IDs, but the list parser rejects dots, so a callback such as `mdl_list_hf.co_1` does not enter the model-list path. The latest release tag in this checkout has the same behavior, and the two direct fix PRs are closed unmerged, so this should remain open as a narrow repair candidate.

## What This Changes

Allow Telegram `/models` provider-list callbacks to handle dotted provider IDs such as `hf.co` so selecting that provider opens the inline model keyboard instead of falling back to text.

## Best Possible Solution

Land a small Telegram plugin fix that accepts dotted provider IDs in `mdl_list_{provider}_{page}` callbacks, preserves existing model-selection callback semantics, adds `hf.co` parser/provider-keyboard regression coverage, and records a credited user-facing changelog fix.
