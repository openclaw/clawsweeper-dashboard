## Reproduction Assessment

Yes. Current main’s plain CLI path leaves `loadPluginSecurityCollectors` undefined while the audit context defaults it on, and collector discovery imports plugin registry/metadata loader paths; the PR’s new regression test covers the intended non-deep behavior.

## Solution Assessment

Yes. Defaulting collector loading to `opts.loadPluginSecurityCollectors ?? deep` is the narrow maintainable fix because it preserves deep audits and explicit internal callers while keeping routine audits on the cold path.
