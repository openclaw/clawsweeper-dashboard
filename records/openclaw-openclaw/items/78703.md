## What This Changes

The issue reports cron jobs being silently removed around a runtime config reload after a failed and then successful cron.add sequence.

## Best Possible Solution

## Solution Assessment

Unclear. Backup restore alone is not the best fix until maintainers know whether the remover was reload/timer code, an agent or plugin cleanup path, an operator action, or another explicit cron.remove caller.
