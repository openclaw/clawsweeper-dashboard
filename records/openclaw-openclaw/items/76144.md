## Summary

Keep open: current main still has the ownership-blocked plugin diagnostic gap, and the targeted repair PR #76151 is open rather than merged. The ownership block is intentional, but blocked-present plugins can still be reported as stale or missing config entries.

## What This Changes

The issue asks OpenClaw to report ownership-blocked plugins as present-but-blocked, deduplicate ownership warnings, document the ownership constraint, and address Docker restart ownership durability.

## Best Possible Solution

Preserve the plugin ownership safety gate, carry blocked-present state into config diagnostics, deduplicate ownership warnings per boot, document durable ownership guidance, and handle any Docker ownership-policy change separately.

## Reproduction Assessment

Yes. Source inspection on current main shows a mismatched non-bundled plugin is blocked before registration and later explicit config references use the missing-plugin warning branch.

## Solution Assessment

Yes for the diagnostic/docs portion: distinguishing blocked-present plugins from absent plugins is the narrow maintainable fix. The container ownership-reset portion needs separate confirmation and maintainer policy review.
