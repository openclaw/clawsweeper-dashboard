## Summary

Keep open: current main still has the reported diagnostic gap, and the directly related repair PR #76151 is closed unmerged. The ownership block is intentional, but blocked-present plugins still fall through to stale missing-plugin warnings and duplicate ownership diagnostics can still be emitted.

## What This Changes

The issue asks OpenClaw to distinguish ownership-blocked plugin directories from stale config entries, dedupe suspicious-ownership warnings, document the ownership constraint, and consider durable Docker container ownership behavior.

## Best Possible Solution

Keep the ownership safety gate, propagate blocked-present plugin id/path state into config validation, dedupe ownership diagnostics per boot, document the constraint, and split any Docker ownership-policy change into a separately confirmed follow-up.

## Reproduction Assessment

Yes by source inspection. A non-bundled uid-mismatch candidate is blocked before registration, lacks a diagnostic plugin id, and configured entries or allowlist references still flow to the stale missing-plugin warning branch.

## Solution Assessment

Yes for the diagnostics and docs portion: propagating blocked-present state from discovery into validation and deduping discovery diagnostics is the narrow maintainable fix. Docker chown behavior should be handled separately if maintainers want to change that policy.
