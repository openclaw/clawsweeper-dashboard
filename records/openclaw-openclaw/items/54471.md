## Summary

Keep open: this is a real ACP parent-streaming bug, but the PR is incomplete because current ACP status text is dropped before the added relay branch can see it, the new path does not update stall/resume state, and external after-fix real behavior proof is missing.

## What This Changes

The PR adds a `system_event` branch to `startAcpSpawnParentStreamRelay()` to emit non-empty system-event text to the parent session.

## Best Possible Solution

Map non-empty ACP runtime `status` text into untrusted parent-session system events, treat relayed system text as progress for stall/resume state, preserve assistant/lifecycle behavior, add focused tests, and require after-fix real behavior proof.

## Reproduction Assessment

Yes, at source level. A `sessions_spawn({ runtime: "acp", streamTo: "parent" })` run can receive `AcpRuntimeEvent` status text, but `agent-command.ts` returns for non-`text_delta` events before the parent relay sees it.

## Solution Assessment

No. Listening for an agent event stream named `system_event` alone is not sufficient because current main does not emit that stream for ACP status text, and the added branch also needs to update stall/resume progress state.
