## Summary

Keep open: the reported ACP parent-streaming bug is valid, but this PR is incomplete because current ACP `status` events are dropped before the proposed relay branch can see them, and the new path would not reset the relay stall watcher.

## What This Changes

The PR adds a `system_event` handling branch in `startAcpSpawnParentStreamRelay()` to emit non-empty event text to the parent session with a fallback ACP context key.

## Best Possible Solution

Map non-empty ACP runtime status/progress text into untrusted parent-session system events, count that text as progress for stall/resume state, preserve existing assistant/lifecycle behavior, and add focused tests plus a changelog fix.

## Reproduction Assessment

Yes, at source level. An ACP `status` event from `runTurn()` reaches `agent-command.ts`, but the callback returns for `event.type !== "text_delta"`, so no parent-relay event is emitted.

## Solution Assessment

No. A relay branch for `event.stream === "system_event"` alone does not wire the actual ACP status source into agent events, and it should also update the same stall/resume state as visible assistant progress.
