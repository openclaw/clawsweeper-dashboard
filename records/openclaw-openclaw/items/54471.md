The branch adds a `system_event` branch in `startAcpSpawnParentStreamRelay()` to emit non-empty event text to the parent ACP session.

## Best Possible Solution

Update this PR in place to relay sanitized ACP `runtime_event` status/progress text from the current `acp` stream, count it as visible progress for stall/resume state, preserve existing assistant/lifecycle behavior, and add focused relay coverage plus contributor-supplied real behavior proof.

## Solution Assessment
