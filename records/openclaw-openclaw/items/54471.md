## Summary

Keep open. The PR targets a real ACP parent-streaming gap, but the patch is incomplete: current ACP turns expose progress/clarifying text as runtime `status` events that are dropped before the relay sees them, and the new relay branch would also fail to update stall/resume progress accounting.

## What This Changes

The PR adds a `system_event` branch to `startAcpSpawnParentStreamRelay()` so non-empty system-event text can be emitted to the parent session with a fallback ACP context key.

## Best Possible Solution

Map the actual ACP runtime status/progress source into untrusted parent-session system events, count relayed status text as progress for stall/resume state, preserve existing assistant/lifecycle behavior, and add focused relay coverage plus a user-facing changelog fix if merged.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is an ACP `runTurn` event with `type: "status"` during a `streamTo: "parent"` spawn: current main returns before emitting an agent event, so the parent relay never receives that text.

## Solution Assessment

No. The proposed branch listens for a `system_event` stream at the relay boundary, but current main does not produce that stream from ACP status events; the safer fix is to translate the current ACP `status` source and update relay progress accounting.
