## Summary

Keep open: the reported ACP parent-streaming bug is source-reproducible, but this branch does not wire current ACP `status` events into the agent-event relay, still leaves system text out of stall/resume accounting, and lacks contributor-supplied real behavior proof.

## What This Changes

The PR adds a `system_event` branch to `startAcpSpawnParentStreamRelay()` that emits non-empty system-event text to the parent session with a fallback ACP context key.

## Best Possible Solution

Update this PR in place to translate ACP runtime status/progress text into parent-session system events, treat that text as progress/resume activity, preserve existing assistant/lifecycle behavior, and add focused regression coverage plus real behavior proof.

## Reproduction Assessment

Yes, at source level. Current ACP runtime `status` events carry text, but the ACP turn callback returns before emitting anything for non-`text_delta` events, so that text cannot reach the parent relay on current main.

## Solution Assessment

No. Listening only for an agent-event stream named `system_event` is too late for current ACP `status` events, and relayed system text also needs to refresh stall/resume state.
