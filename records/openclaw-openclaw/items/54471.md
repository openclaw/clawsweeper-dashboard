## Summary

Keep open. The reported ACP parent-streaming bug is source-reproducible and the PR touches the right relay area, but the diff is incomplete because current ACP `status` events are dropped before this relay, the new branch does not count system text as progress, and the PR still lacks after-fix real behavior proof.

## What This Changes

The branch adds a `system_event` branch to `startAcpSpawnParentStreamRelay()` so non-empty text is emitted to the parent session with a fallback ACP context key.

## Best Possible Solution

Update this PR in place to translate ACP runtime status/progress text into parent-session system events, treat relayed system text as progress/resume activity, preserve existing assistant/lifecycle behavior, and add focused regression coverage.

## Reproduction Assessment

Yes, at source level. Current ACP runtime `status` events carry text, but `agent-command.ts` returns before emitting anything for non-`text_delta` events, so that text cannot reach the parent relay.

## Solution Assessment

No. Listening for an agent-event stream named `system_event` inside the relay is too late for current ACP status events, and any relayed system text also needs to refresh stall/resume accounting.
