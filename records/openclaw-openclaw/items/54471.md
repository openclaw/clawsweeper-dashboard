## Best Possible Solution

Update this PR in place to relay sanitized current ACP runtime status text from the `acp` stream, count that text as visible progress/resume state, add focused relay coverage, and keep broader tool/result/thinking visibility in the separate canonical issue.

## Solution Assessment

No. A separate `system_event` branch does not match the current ACP event source; the narrower maintainable fix is to relay sanitized ACP runtime status text from the existing `acp` event contract while updating stall/resume state.
