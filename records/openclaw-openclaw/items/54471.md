## Summary

Keep open: the reported ACP parent-stream bug is valid, but this patch is not merge-ready because current ACP status/progress events are dropped before this relay handler and the new branch does not update stall/resume accounting; the external PR also lacks real behavior proof.

## What This Changes

The PR adds a `system_event` handler in `src/agents/acp-spawn-parent-stream.ts` to emit non-empty event text to the parent ACP spawn session with a fallback context key.

## Best Possible Solution

Map non-empty ACP runtime status/progress text into untrusted parent-session system events at the ACP command boundary, count visible system text as relay progress/resume activity, preserve assistant/lifecycle behavior, and add focused regression coverage.

## Reproduction Assessment

Yes, at source level. Current ACP runtime `status` events carry text, but `agent-command.ts` drops every non-`text_delta` event before anything reaches the parent relay.

## Solution Assessment

No. Listening for an agent-event stream named `system_event` inside the relay is not enough when the actual ACP status source is never emitted there, and relayed system text should also refresh stall/resume state.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Wire ACP status events into the relay:** `src/agents/acp-spawn-parent-stream.ts:315`
  - body: This branch listens for `event.stream === "system_event"` in the parent relay, but current ACP turns return from `agent-command.ts` for every runtime event except `text_delta`. The `status` events that carry clarifying/progress text still never reach this handler, so the reported parent updates can remain dropped.
  - confidence: 0.9
- **[P2] Count system events as relay progress:** `src/agents/acp-spawn-parent-stream.ts:320`
  - body: This branch emits the system text without refreshing `lastProgressAt` or clearing `stallNotified`. A run that only emits progress or clarifying system text can still trigger a later no-output warning, producing contradictory parent-session updates.
  - confidence: 0.9
