## Summary

Keep this PR open: it targets a real ACP parent-streaming gap, but the current patch does not wire the actual ACP runtime status/progress source into the relay and still leaves false stall notices possible. It also lacks contributor-supplied real behavior proof from a real ACP spawn after the patch.

## What This Changes

The PR adds a `system_event` branch to `startAcpSpawnParentStreamRelay()` that emits non-empty event text to the parent session with a fallback ACP context key.

## Best Possible Solution

Update this PR in place to translate ACP runtime status/progress text into parent-session system events, count that text as progress/resume activity, preserve existing assistant/lifecycle behavior, and add focused regression coverage plus real behavior proof.

## Reproduction Assessment

Yes, at source level. ACP runtime `status` events carry text, but current `agent-command.ts` ignores non-`text_delta` events before the parent relay can see them.

## Solution Assessment

No. Listening only for an agent-event stream named `system_event` is too late for current ACP status events, and relayed system text also needs to refresh stall/resume state.
