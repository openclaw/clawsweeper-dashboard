## Label Justifications

- `P1`: The PR touches agent exec process lifecycle behavior where a regression can unexpectedly terminate live background work.
- `merge-risk: 🚨 compatibility`: Existing sessions that use a non-empty session key can be treated differently after merge and lose a previously valid background run.
- `merge-risk: 🚨 session-state`: The diff changes reconciliation between supervisor run state and bash session state, and currently compares mismatched ownership identifiers.
- `merge-risk: 🚨 availability`: A false orphan decision can cancel legitimate process trees and interrupt active agent workflows.

## Summary

Keep open: current main still leaves supervisor orphan reconciliation as a no-op, but this PR should not merge as written because the new predicate checks supervisor session-key values against a registry keyed by generated process session ids, so valid session-key-owned exec runs can be cancelled; the external PR also lacks required real behavior proof.

## What This Changes

The PR adds `ProcessSupervisor.reconcileOrphans(params)`, a bash session existence predicate, exec/process-tool reconciliation calls, and focused tests for cancelling missing-session runs while preserving owned runs.

## Best Possible Solution

Repair reconciliation so supervisor ownership is checked against the actual running `ProcessSession` identity/session-key contract, add regression coverage for non-empty `sessionKey`, resolve conflicts, and keep durable startup orphan reaping tracked by the linked issue unless maintainers choose to expand this PR.

## Reproduction Assessment

Yes, at source level: current main stores bash sessions by generated `session.id`, while supervisor records can store `opts.sessionKey` as `sessionId`; the PR's `hasSession(record.sessionId)` check therefore returns false for valid session-key-owned runs. I did not run a live process proof because this was a read-only review.

## Solution Assessment

No, not as written. In-process orphan reconciliation is a reasonable partial layer, but the predicate must use the correct ownership identifier before this can be the best maintainable fix.
