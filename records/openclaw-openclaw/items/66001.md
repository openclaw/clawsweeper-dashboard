## Review Findings

Full review comments:

- **[P1] Make orphan checks session-key aware:** `src/agents/bash-tools.exec-runtime.ts:556`
  - body: `hasSession` only checks the bash registry by generated session id, but `runExecProcess` records supervisor ownership as `opts.sessionKey?.trim() || sessionId`. Any live exec run started with a non-empty session key is therefore treated as untracked and canceled the next time reconciliation runs, such as when another exec starts or the process tool is invoked.
  - confidence: 0.95
- **[P3] Restrict orphan cancellation reasons:** `src/process/supervisor/types.ts:14`
  - body: `ReconcileOrphansParams.reason` accepts the full `TerminationReason` union, including natural completion reasons like `exit`, `signal`, and `spawn-error`. A future caller could record a forced orphan cancellation as a natural or spawn exit, so this contract should be narrowed to forced-cancellation reasons or the parameter should be removed until needed.
  - confidence: 0.82
