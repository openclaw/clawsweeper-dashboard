## Summary

This PR addresses a real lifecycle gap, but it is not ready to merge because the proposed reconciliation can cancel valid sessionKey-owned background runs and the external contributor has not provided real behavior proof. Current main still has a no-op orphan reconciliation path, so the work is not obsolete.

## What This Changes

The PR adds a typed process-supervisor orphan reconciliation API, calls it from bash exec/process paths, exports the new types, and adds focused tests for orphan cancellation and preservation.

## Best Possible Solution

Revise this PR or replace it with ownership-aware in-process reconciliation that preserves sessionKey-owned runs, updates affected runtime mocks/tests, narrows forced-cancel reasons, and leaves durable restart orphan policy tracked by https://github.com/openclaw/openclaw/issues/65983.

## Reproduction Assessment

Yes, from source inspection: start a background exec with a non-empty sessionKey, then trigger reconciliation by starting another exec or invoking the process tool. The supervisor record is owned by that sessionKey, while the PR's predicate checks only the generated registry id, so the active run can be misclassified as orphaned.

## Solution Assessment

No. In-process reconciliation is a reasonable smaller fix direction, but this patch compares the wrong ownership key, leaves the cancellation reason contract too broad, and should not close the broader restart-orphan issue without maintainer scope agreement.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Use ownership-aware session checks:** `src/agents/bash-process-registry.ts:100-102`
  - body: `hasSession` only checks `runningSessions` by generated process-session id, but `runExecProcess` stores supervisor ownership as `opts.sessionKey?.trim() || sessionId`. Any active exec launched with a non-empty `sessionKey` can be misclassified as untracked and cancelled the next time reconciliation runs.
  - confidence: 0.94
- **[P2] Update the exec-runtime supervisor mock:** `src/agents/bash-tools.exec-runtime.ts:557-559`
  - body: `bash-tools.exec-runtime.test.ts` mocks `getProcessSupervisor()` with only `spawn` in existing cases. This diff makes `runExecProcess` call `reconcileOrphans` unconditionally, so those tests can throw before reaching the behavior under test.
  - confidence: 0.86
- **[P3] Restrict reconciliation reasons to forced cancels:** `src/process/supervisor/types.ts:14`
  - body: `ReconcileOrphansParams.reason` accepts natural terminal reasons such as `exit`, `signal`, and `spawn-error`, but reconciliation force-cancels live runs. Narrow the type so callers cannot record a killed orphan as a natural exit.
  - confidence: 0.8
