## Summary

Keep open. The PR targets a real process-lifecycle gap, but the current patch can cancel legitimate sessionKey-owned runs, misses an affected runtime test mock, lacks after-fix real behavior proof, and would prematurely close the broader restart-orphan issue.

## What This Changes

The PR adds a typed `reconcileOrphans` contract, calls it from exec/process paths using bash session registry membership, exports the new types, and adds focused supervisor/process-tool tests.

## Best Possible Solution

Revise this PR or replace it with ownership-aware reconciliation that preserves sessionKey-owned runs, updates affected runtime tests, narrows forced-cancel reasons, and leaves durable restart orphan policy explicit for https://github.com/openclaw/openclaw/issues/65983.

## Reproduction Assessment

Yes, from source inspection: start a background exec with a non-empty `sessionKey`, then trigger reconciliation. The supervisor record is owned by that `sessionKey`, while the PR's predicate checks only the generated registry id, so the run can be misclassified as orphaned.

## Solution Assessment

No. In-process reconciliation is a reasonable smaller direction, but this patch compares the wrong ownership key, leaves the cancellation reason contract too broad, and should not close the broader restart-orphan issue without maintainer scope agreement.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Use ownership-aware session checks:** `src/agents/bash-process-registry.ts:100-102`
  - body: `hasSession` only checks the generated process-session id, but `runExecProcess` stores supervisor ownership as `opts.sessionKey?.trim() || sessionId`. Any active exec launched with a non-empty `sessionKey` can be misclassified as untracked and cancelled the next time reconciliation runs.
  - confidence: 0.94
- **[P2] Update the exec-runtime supervisor mock:** `src/agents/bash-tools.exec-runtime.ts:557-559`
  - body: `bash-tools.exec-runtime.test.ts` currently mocks `getProcessSupervisor()` with only `spawn`. This diff makes `runExecProcess` call `reconcileOrphans` unconditionally, so existing runtime tests that exercise `runExecProcess` can throw before reaching the behavior under test.
  - confidence: 0.86
- **[P3] Restrict reconciliation reasons to forced cancels:** `src/process/supervisor/types.ts:13`
  - body: `ReconcileOrphansParams.reason` accepts natural terminal reasons such as `exit`, `signal`, and `spawn-error`, but reconciliation force-cancels live runs. Narrow the type so callers cannot record a killed orphan as a natural exit.
  - confidence: 0.8
