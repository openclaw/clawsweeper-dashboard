## Summary

Keep open. The PR targets a real process-lifecycle gap, but the current patch can cancel legitimate sessionKey-owned exec runs, leaves the reconciliation reason contract too broad, lacks required real behavior proof, and still does not resolve the broader restart-orphan issue it closes.

## What This Changes

The PR adds process-supervisor orphan reconciliation, calls it from exec/process-tool paths, exports the new contract, and adds focused supervisor/process-tool tests.

## Best Possible Solution

Revise the PR to reconcile against the actual ownership contract, preserve sessionKey-owned runs, narrow forced-cancel reasons, update current-main regression coverage, and leave durable restart orphan policy explicit rather than silently closing it.

## Reproduction Assessment

Yes for the blocking PR regression from source inspection: start a background exec with a non-empty sessionKey, then trigger reconciliation. The supervisor record is keyed by sessionKey while the PR's hasSession predicate checks only the generated registry id, so the active run can be treated as orphaned.

## Solution Assessment

No. In-process reconciliation is a reasonable smaller direction, but this implementation compares the wrong ownership key, leaves the cancellation reason contract too broad, and should not close the broader restart-orphan issue without maintainer scope agreement.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Use ownership-aware session checks:** `src/agents/bash-process-registry.ts:100`
  - body: `hasSession` only checks the generated process-session id, but `runExecProcess` stores supervisor ownership as `opts.sessionKey?.trim() || sessionId`. Any active exec launched with a non-empty `sessionKey` can be misclassified as untracked and cancelled by the new reconciliation path.
  - confidence: 0.94
- **[P2] Update the exec-runtime supervisor mock:** `src/agents/bash-tools.exec-runtime.ts:557-559`
  - body: On current main, `bash-tools.exec-runtime.test.ts` exercises `runExecProcess` with `getProcessSupervisor()` mocked to expose only `spawn`. The new unconditional `reconcileOrphans` call will throw in that coverage unless the mock is extended and the sessionKey ownership case is tested.
  - confidence: 0.86
- **[P3] Restrict reconciliation reasons to forced cancellations:** `src/process/supervisor/types.ts:13`
  - body: `ReconcileOrphansParams.reason` accepts natural terminal reasons such as `exit`, `signal`, and `spawn-error`, but reconciliation force-cancels live runs. Narrow this to forced-cancellation reasons so callers cannot record a killed orphan as a natural exit.
  - confidence: 0.8
