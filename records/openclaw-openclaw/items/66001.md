Full review comments:

- **[P1] Restrict reconciliation to bash-owned runs:** `src/process/supervisor/supervisor.ts:276`
  - body: `reconcileOrphans` iterates every active supervisor record, but these callers pass a bash-process-registry predicate. CLI runner and live-session runs also use the same singleton supervisor and are not in `runningSessions`, so the next exec/process call can cancel a legitimate agent backend run. Filter to bash-owned records or pass a record-aware ownership predicate before canceling.
  - confidence: 0.9
- **[P1] Track the generated process session id:** `src/process/supervisor/supervisor.ts:280`
  - body: `runExecProcess` stores the bash `ProcessSession` under the generated `session.id`, while the supervisor record can use `opts.sessionKey` as `sessionId`. Checking `hasSession(record.sessionId)` therefore returns false for valid session-keyed exec runs and the next reconciliation cancels them. Preserve/check the generated process session id as the ownership key.
  - confidence: 0.92
- **[P2] Narrow orphan termination reasons:** `src/process/supervisor/types.ts:14`
  - body: `reason?: TerminationReason` allows natural terminal values such as `exit`, `signal`, and `spawn-error` for a forced orphan cancellation. Restrict this API to cancellation/timeout reasons so future callers cannot record a killed orphan as a natural exit.
  - confidence: 0.78
