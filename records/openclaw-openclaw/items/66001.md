## Summary

Keep open: the fix direction is useful, but this branch can misclassify sessionKey-owned background exec runs as orphaned, lacks real behavior proof, and is currently conflicting against main.

## What This Changes

The PR adds active `ProcessSupervisor.reconcileOrphans(...)` cancellation, calls it from exec/process paths, exports the new types, and adds focused supervisor/process tests.

## Best Possible Solution

Revise or replace the branch with ownership-aware in-process reconciliation that preserves `sessionKey`-owned runs, updates tests/types, and leaves cold-start stale PTY policy tracked separately until maintainers decide it.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path for the PR regression: start a background exec with a non-empty `sessionKey`, then trigger reconciliation by starting another exec or invoking the process tool. The read-only review did not run a live process proof.

## Solution Assessment

No. In-process reconciliation is a reasonable smaller fix direction, but this implementation compares the wrong ownership key, leaves the reason contract too broad, and should not close the broader restart-orphan issue without scope agreement.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Use ownership-aware session checks:** `src/agents/bash-tools.exec-runtime.ts:557-559`
  - body: This passes `hasSession`, which checks generated process session ids, into reconciliation, but `runExecProcess` stores supervisor ownership as `opts.sessionKey?.trim() || sessionId`. A background exec with a non-empty `sessionKey` can be treated as orphaned and killed on the next reconciliation.
  - confidence: 0.94
- **[P2] Update the exec-runtime supervisor mock:** `src/agents/bash-tools.exec-runtime.ts:557-559`
  - body: Adding this unconditional method call means `bash-tools.exec-runtime.test.ts`'s mocked supervisor, which currently only exposes `spawn`, can throw before the existing `runExecProcess` assertions run. Add `reconcileOrphans` to that mock and reset it with the rest of the supervisor mock state.
  - confidence: 0.86
- **[P3] Restrict reconciliation reasons to forced cancels:** `src/process/supervisor/types.ts:14`
  - body: `ReconcileOrphansParams.reason` accepts natural terminal reasons such as `exit`, `signal`, and `spawn-error`, but reconciliation force-cancels live runs. Narrow the type so callers cannot record a killed orphan as a natural exit.
  - confidence: 0.8
