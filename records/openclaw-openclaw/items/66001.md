## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Use session-key-aware ownership checks:** `src/agents/bash-process-registry.ts:99-100`
  - body: When `opts.sessionKey` is set, `runExecProcess` registers the supervisor record with `sessionId: opts.sessionKey?.trim() || sessionId`, but this predicate only checks the registry by generated session id. The new reconciliation calls will treat those legitimate runs as orphans and cancel them on the next exec or process tool request, so the predicate needs to recognize the `ProcessSession.sessionKey` owner and add regression coverage.
  - confidence: 0.93
- **[P2] Restrict orphan reconciliation reasons:** `src/process/supervisor/types.ts:13`
  - body: `reason?: TerminationReason` allows natural terminal reasons such as `exit`, `signal`, and `spawn-error`; `cancel()` writes that value before killing, and `registry.finalize()` preserves the first terminal reason. A caller could therefore record a killed orphan as a normal exit, so this API should accept only forced-cancellation reasons.
  - confidence: 0.82
