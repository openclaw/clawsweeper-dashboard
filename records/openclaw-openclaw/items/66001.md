## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Use session-key-aware ownership checks:** `src/agents/bash-process-registry.ts:100`
  - body: `hasSession` only checks the bash registry by `ProcessSession.id`, but `runExecProcess` stores the supervisor record's `sessionId` as `opts.sessionKey?.trim() || sessionId`. A run started with a non-empty `sessionKey` will be treated as untracked and cancelled on the next reconciliation even though its bash session still exists. Reconcile against the canonical owner or teach the predicate to recognize both the session slug and session key.
  - confidence: 0.93
- **[P3] Restrict orphan cancellation reasons:** `src/process/supervisor/types.ts:14`
  - body: `ReconcileOrphansParams.reason` accepts the full `TerminationReason` union, including natural terminal states like `exit`, `signal`, and `spawn-error`. Since reconciliation force-cancels active runs, keep this API limited to forced-cancellation reasons so callers cannot record an orphan kill as a natural exit.
  - confidence: 0.78
