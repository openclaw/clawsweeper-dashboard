## Summary

Keep open. Current main still has no-op reconciliation, and this PR remains a real fix attempt, but the proposed patch has blocking correctness issues around sessionKey-owned runs and test mocks, plus no after-fix real behavior proof from a live setup.

## What This Changes

The PR adds an in-process orphan reconciliation contract for process-supervisor runs, calls it from bash exec/process-tool paths, and adds focused supervisor/process-tool tests.

## Best Possible Solution

Revise this PR to use ownership-aware reconciliation that preserves both generated session ids and sessionKey-owned runs, narrow forced-cancel reasons, update regression coverage, and leave durable restart orphan policy tracked separately unless it is implemented here.

## Reproduction Assessment

Yes, from source inspection for the PR-introduced bug: launch a background exec with a non-empty sessionKey, then trigger the PR's reconciliation path. The supervisor record is owned by sessionKey while the registry lookup is keyed by generated session id, so the live run is misclassified as untracked.

## Solution Assessment

No. The smaller in-process reconciliation direction is reasonable, but this implementation compares the wrong ownership key, leaves the reason contract too broad, and lacks real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Use ownership-aware session checks:** `src/agents/bash-tools.exec-runtime.ts:607`
  - body: hasSession only checks runningSessions by generated session id, but runExecProcess stores supervisor ownership as opts.sessionKey?.trim() || sessionId. Any active run launched with a non-empty sessionKey can be treated as orphaned and cancelled on the next reconciliation; preserve sessions by matching either session.id or session.sessionKey.
  - confidence: 0.94
- **[P2] Update the exec-runtime supervisor mock:** `src/agents/bash-tools.exec-runtime.ts:606-608`
  - body: Existing bash-tools.exec-runtime.test.ts mocks getProcessSupervisor() with only spawn and calls runExecProcess() in background-exit tests. This new unconditional reconcileOrphans call will throw before spawn unless the mock gets a reconcileOrphans stub and sessionKey ownership coverage.
  - confidence: 0.88
- **[P3] Restrict reconciliation reasons to forced cancellations:** `src/process/supervisor/types.ts:13`
  - body: ReconcileOrphansParams.reason accepts natural terminal reasons such as exit, signal, and spawn-error, but reconciliation force-cancels live runs. Narrow the type to forced cancellation reasons so callers cannot record a killed orphan as a natural exit.
  - confidence: 0.8
