## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Use a session-key-aware ownership check before cancelling runs:** `src/agents/bash-process-registry.ts:100`
  - body: `hasSession` only checks `runningSessions` by the generated process session id, but `runExecProcess` can store supervisor records under `opts.sessionKey?.trim() || sessionId`. Any active run with a non-empty `sessionKey` will look untracked to `reconcileOrphans` and can be cancelled the next time another exec starts or the process tool runs.
  - confidence: 0.95
- **[P2] Restrict orphan reconciliation reasons to forced exits:** `src/process/supervisor/types.ts:11-14`
  - body: The new public params accept the full `TerminationReason` union, including natural outcomes like `exit`, `signal`, and `spawn-error`. Passing one of those while force-cancelling an orphan would record a misleading terminal reason, so the type should only allow forced cancellation reasons or keep the reason internal.
  - confidence: 0.82
