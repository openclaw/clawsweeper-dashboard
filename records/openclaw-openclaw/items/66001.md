## Summary

Keep open: the PR addresses a real process-supervision gap, but the current implementation can cancel legitimate sessionKey-owned background execs, needs current-main test coverage updates, and lacks contributor-owned real behavior proof.

## What This Changes

The PR adds process-supervisor orphan reconciliation, calls it from bash exec/process paths, exports the new contract, and adds focused supervisor/process-tool tests.

## Best Possible Solution

Revise the PR to reconcile against an ownership-aware session predicate that preserves generated session ids and sessionKey-owned runs, add regression coverage, narrow forced-cancel reason typing, and keep durable restart orphan policy tracked separately unless implemented here.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path for the PR-introduced regression: start a background exec with a non-empty sessionKey, then trigger process list or another exec so reconciliation checks that sessionKey against a registry keyed only by generated session id.

## Solution Assessment

No. In-process reconciliation is a reasonable narrow direction, but this implementation must use ownership-aware matching, update current-main tests, and avoid closing the broader restart/cold-start issue accidentally.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Use ownership-aware session checks:** `src/agents/bash-tools.exec-runtime.ts:557-559`
  - body: `hasSession` only checks the registry by generated session id, but `runExecProcess` stores supervisor ownership as `opts.sessionKey?.trim() || sessionId`. A live background exec with a non-empty sessionKey can be treated as orphaned and killed when reconciliation runs; preserve runs by matching generated id and sessionKey ownership.
  - confidence: 0.94
- **[P2] Update the exec-runtime supervisor mock:** `src/agents/bash-tools.exec-runtime.ts:557-559`
  - body: Current main has exec-runtime tests that mock `getProcessSupervisor()` with only `spawn` and call `runExecProcess()` with a sessionKey. This new unconditional `reconcileOrphans` call will throw before spawn after rebasing unless the mock gets a reconcile stub and sessionKey ownership coverage.
  - confidence: 0.88
- **[P3] Restrict reconciliation reasons to forced cancellations:** `src/process/supervisor/types.ts:13`
  - body: `ReconcileOrphansParams.reason` accepts natural terminal reasons such as `exit`, `signal`, and `spawn-error`, but reconciliation force-cancels active runs. Narrow this to forced-cancel reasons so callers cannot record a killed orphan as a natural exit.
  - confidence: 0.8
