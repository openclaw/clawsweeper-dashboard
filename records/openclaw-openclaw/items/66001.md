- `P1`: The PR touches active exec/process lifecycle behavior and currently has a blocking path that can kill valid background jobs.
- `merge-risk: 🚨 compatibility`: Existing sessionKey-owned background exec workflows can stop working because the new reconciliation check uses a different ownership key.

## Summary

Keep open: current main still has no active orphan reconciliation, but this PR is not merge-ready because the new reconciliation predicate can cancel valid sessionKey-owned background runs and contributor real behavior proof is still missing.

## What This Changes

The branch adds a `ProcessSupervisor.reconcileOrphans(...)` contract, calls it from exec/process paths using the bash process registry, exports the new types, and adds focused supervisor/process tests.

## Best Possible Solution

Revise or replace the branch with ownership-aware in-process reconciliation that preserves sessionKey-owned runs, updates tests/types, and keeps cold-start PTY reaping tracked under https://github.com/openclaw/openclaw/issues/65983 until maintainers choose that policy.

## Reproduction Assessment

Yes for the PR regression by source inspection: start a background exec with a non-empty `sessionKey`, then trigger reconciliation by starting another exec or invoking the process tool. I did not run live process proof in this read-only review.

## Solution Assessment

No. In-process reconciliation is a reasonable smaller fix direction, but this implementation compares the wrong ownership key, leaves the reason contract too broad, and should not close the broader restart-orphan issue without scope agreement.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Use ownership-aware session checks:** `src/agents/bash-tools.exec-runtime.ts:557-559`
  - body: This passes `hasSession` into reconciliation, but `hasSession` checks generated process session ids while `runExecProcess` stores supervisor ownership as `opts.sessionKey?.trim() || sessionId`. A background exec with a non-empty `sessionKey` can be treated as orphaned and killed on the next reconciliation.
  - confidence: 0.94
- **[P2] Update the exec-runtime supervisor mock:** `src/agents/bash-tools.exec-runtime.ts:557-559`
  - body: Adding this unconditional `reconcileOrphans` call means current `bash-tools.exec-runtime.test.ts`'s mocked supervisor, which only exposes `spawn`, can throw before existing `runExecProcess` assertions run. Add `reconcileOrphans` to that mock and reset it with the rest of the supervisor mock state.
  - confidence: 0.86
- **[P3] Restrict reconciliation reasons to forced cancels:** `src/process/supervisor/types.ts:13`
  - body: `ReconcileOrphansParams.reason` accepts natural terminal reasons such as `exit`, `signal`, and `spawn-error`, but reconciliation force-cancels live runs. Narrow the type so callers cannot record a killed orphan as a natural exit.
  - confidence: 0.8
