## Summary

Keep open. Current main still has the no-op supervisor reconciliation that this PR is trying to address, but the PR is not mergeable as written: its slug-only session check would cancel legitimate runs owned by a `sessionKey`, and the new unconditional reconciliation call would break existing exec-runtime tests that mock only `spawn`.

## What This Changes

The PR adds a process-supervisor orphan reconciliation contract, exports its types, adds a registry `hasSession` helper, calls reconciliation from bash exec and process-tool paths, and adds focused supervisor/process-tool tests.

## Best Possible Solution

Keep the in-process reconciliation direction, but make ownership checks aware of both generated process-session ids and `sessionKey` ownership, narrow reconciliation reasons to forced cancellations, add regression coverage for sessionKey-owned background execs, and leave the broader durable restart orphan work tracked separately unless that scope is implemented.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to run a background exec with a non-empty `sessionKey`, then trigger the PR's reconciliation path; the supervisor record is keyed by `sessionKey` while the registry is keyed by generated session id, so `hasSession(sessionKey)` returns false and the live run is cancelled.

## Solution Assessment

No. The smaller in-process reconciliation approach is maintainable, but this implementation compares against the wrong ownership key and leaves the new contract too broad; an ownership-aware registry helper plus focused tests is the safer narrow fix.

## Review Findings

Full review comments:

- **[P1] Use an ownership-aware session check:** `src/agents/bash-tools.exec-runtime.ts:607`
  - body: `hasSession` only checks the registry by generated session id, but `runExecProcess` stores supervisor ownership as `opts.sessionKey?.trim() || sessionId`. Any live run launched with a non-empty `sessionKey` is treated as untracked and cancelled on the next reconciliation; preserve sessions by matching either `session.id` or `session.sessionKey`.
  - confidence: 0.94
- **[P2] Mock reconciliation in exec-runtime tests:** `src/agents/bash-tools.exec-runtime.ts:606-608`
  - body: Existing `bash-tools.exec-runtime.test.ts` mocks `getProcessSupervisor()` with only `spawn` and calls `runExecProcess()` in background-exit tests. This new unconditional call will throw before spawn unless the mock gets a `reconcileOrphans` stub and sessionKey ownership coverage.
  - confidence: 0.88
- **[P3] Restrict orphan reconciliation to forced reasons:** `src/process/supervisor/types.ts:13`
  - body: `ReconcileOrphansParams.reason` accepts natural terminal reasons such as `exit`, `signal`, and `spawn-error`, but reconciliation force-cancels live runs. Narrow this to forced cancellation reasons so callers cannot record a killed orphan as a natural exit.
  - confidence: 0.8
