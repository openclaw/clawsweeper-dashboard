## Summary

Keep open. Current main still has no-op orphan reconciliation, the PR is not superseded by a merged fix, and the proposed branch has source-proven correctness issues plus missing real behavior proof.

## What This Changes

The PR adds an in-process `reconcileOrphans` supervisor contract, calls it from exec/process-tool paths, exports its types, and adds focused process-supervisor tests.

## Best Possible Solution

Revise the PR to use ownership-aware reconciliation that preserves generated session ids and sessionKey-owned runs, narrow forced-cancel reasons, update exec-runtime mocks/regression coverage, and keep durable restart orphan policy tracked separately unless implemented here.

## Reproduction Assessment

Yes, from source inspection for the PR-introduced bug: launch a background exec with a non-empty `sessionKey`, then trigger reconciliation. The supervisor record is owned by `sessionKey` while the PR's registry lookup is keyed by generated session id, so the live run can be misclassified as untracked.

## Solution Assessment

No. The in-process reconciliation direction is reasonable, but this implementation compares the wrong ownership key, leaves the reason contract too broad, misses an exec-runtime mock update, and lacks real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Use ownership-aware session checks:** `src/agents/bash-tools.exec-runtime.ts:557-559`
  - body: `hasSession` only checks the running-session map by generated session id, but `runExecProcess()` stores supervisor ownership as `opts.sessionKey?.trim() || sessionId`. Any active run launched with a non-empty `sessionKey` can be treated as orphaned and cancelled on the next reconciliation; preserve sessions by matching either `session.id` or `session.sessionKey`.
  - confidence: 0.94
- **[P2] Update the exec-runtime supervisor mock:** `src/agents/bash-tools.exec-runtime.ts:557-559`
  - body: The existing exec-runtime test mock for `getProcessSupervisor()` exposes only `spawn`. This new unconditional `reconcileOrphans()` call will throw before `spawn` in those tests unless the mock gets a `reconcileOrphans` stub and sessionKey ownership coverage.
  - confidence: 0.88
- **[P3] Restrict reconciliation reasons to forced cancellations:** `src/process/supervisor/types.ts:13`
  - body: `ReconcileOrphansParams.reason` accepts natural terminal reasons such as `exit`, `signal`, and `spawn-error`, but reconciliation force-cancels live runs. Narrow the type to forced cancellation reasons so callers cannot record a killed orphan as a natural exit.
  - confidence: 0.8
