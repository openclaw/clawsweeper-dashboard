## Summary

Keep this PR open. Current main still has the no-op reconciliation gap the PR is trying to address, but the proposed implementation would misclassify valid sessionKey-owned exec runs as orphans and cancel them, so it needs changes before merge.

## What This Changes

The PR adds process-supervisor orphan reconciliation, wires it into exec and process-tool paths with a bash-session registry predicate, exports the new types, and adds focused supervisor/process-tool tests.

## Best Possible Solution

Revise the PR to use an ownership-aware registry predicate that preserves runs tracked by either generated session id or matching sessionKey, narrow orphan reconciliation to forced-cancellation reasons, add regression coverage for sessionKey-owned runs, and keep the durable restart/cold-start reaping portion tracked separately.

## Reproduction Assessment

Yes. A high-confidence code-path reproduction is to start a background exec with a non-empty sessionKey, then trigger the PR's reconciliation; the supervisor record is owned by sessionKey while the registry is keyed by generated session id, so hasSession(sessionKey) returns false and the run is cancelled.

## Solution Assessment

No. The smaller in-process reconciliation direction is reasonable, but this implementation compares the supervisor owner against the wrong registry key and exposes an overbroad reason contract; an ownership-aware helper plus forced-reason type is the narrower maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Use session-key-aware orphan ownership checks:** `src/agents/bash-tools.exec-runtime.ts:607`
  - body: `hasSession` only checks `runningSessions` by generated session id, but `runExecProcess` records supervisor ownership as `opts.sessionKey?.trim() || sessionId`. Any active exec launched with a non-empty sessionKey is therefore treated as untracked and cancelled the next time reconciliation runs. Use a registry helper that preserves sessions by id or matching `sessionKey`.
  - confidence: 0.93
- **[P3] Restrict reconciliation reasons to forced cancellations:** `src/process/supervisor/types.ts:13`
  - body: `ReconcileOrphansParams.reason` accepts the full `TerminationReason` union, including `exit`, `signal`, and `spawn-error`, even though reconciliation kills a live run through `cancel`. Narrow it to forced-cancellation reasons so orphan kills cannot be recorded as natural exits.
  - confidence: 0.78
