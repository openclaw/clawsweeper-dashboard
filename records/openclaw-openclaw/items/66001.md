- **[P1] Use the same owner key for reconciliation:** `src/agents/bash-tools.exec-runtime.ts:558`
  - body: `hasSession` checks the bash registry by `ProcessSession.id`, but supervisor records for exec runs use `opts.sessionKey?.trim() || sessionId` as their `sessionId`. Any active run with a non-empty sessionKey will therefore look untracked and be cancelled on the next reconciliation pass even though its generated process session is still registered.
  - confidence: 0.91

  - body: The new params type accepts the full `TerminationReason` union, including natural terminal states like `exit`, `signal`, and `spawn-error`. Passing one of those values would record a forced orphan cancellation as if it exited naturally, so the type should be narrowed to cancellation/timeout reasons or the override should stay private until needed.
  - confidence: 0.78
