## Summary

Keep open: the latest head fixes the earlier targetProvider cache-scope and per-model transport findings, but the stacked cache-fingerprint code still has source-reproducible drift-detection bugs and the external PR only shows test/typecheck output, not real behavior proof.

## What This Changes

The branch adds content-hashed models/config cache state, targetProvider/targetModel options with a disk-vs-config short-circuit, focused tests, plugin SDK baseline hash updates, and changelog entries.

## Best Possible Solution

Keep the optimization, but distinguish absent files from unhashable/unsafe files in cache state, make oversized auth profile changes invalidate or miss the cache, then add real gateway or embedded-runner proof before merge.

## Reproduction Assessment

Yes, from source inspection. The cache can compare null to null for unhashable models.json states, and oversized auth-profiles.json always contributes null to the fingerprint once over the cap.

## Solution Assessment

No. The targetProvider validation direction is reasonable, but the current cache-null handling is not the narrowest safe solution because it treats unsafe/unhashable state as stable cache state.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Treat unhashable models.json as cache drift:** `src/agents/models-config.ts:812`
  - body: `readModelsJsonContentHash` returns `null` for symlinks, non-regular files, oversize files, and I/O errors, but the cache hit accepts `null === null`. If a previous skip/noop captured a null hash, later unsafe `models.json` states can still skip planning instead of invalidating the cache.
  - confidence: 0.88
- **[P2] Distinguish oversized auth profiles from a stable fingerprint:** `src/agents/models-config.ts:120-125`
  - body: Returning `null` for every auth-profiles.json over 8 MiB means all oversized variants produce the same fingerprint input. Once the file is already oversized, credential/profile changes can keep hitting the ready cache instead of forcing provider rediscovery.
  - confidence: 0.84
