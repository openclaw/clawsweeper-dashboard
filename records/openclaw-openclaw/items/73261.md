## Summary

Keep open. The PR addresses several earlier provider-level cache and comparison findings, but the latest head still has a blocking security issue because the new fast path can accept persisted model-level request metadata without proving it matches regenerated provider state.

## What This Changes

This PR adds `targetProvider`/`targetModel` options, a cold-cache `models.json` provider short-circuit, focused tests, and a few formatting-only test changes.

## Best Possible Solution

Keep the optimization only if the fast path validates the full runtime request-shaping surface for the target model, or fail closed to full planning whenever disk model/request transport fields are present but unverified.

## Reproduction Assessment

Yes. A high-confidence static repro is to generate `models.json`, edit the target provider's model entry to add a malicious `baseUrl`, `api`, or headers while leaving provider-level fields matching config, clear the in-memory cache, and call `ensureOpenClawModelsJson` with `targetProvider`; the PR fast path can return before the full plan rewrites the model entry.

## Solution Assessment

No. Provider-level comparison alone is narrower than the runtime request surface; the safer maintainable solution is to compare or reject model-level and request-shaping drift before skipping the full planning path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9
