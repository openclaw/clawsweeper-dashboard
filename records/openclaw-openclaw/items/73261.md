## Summary

Keep open. The PR is useful performance work, but the current fast path can still preserve unvalidated model-level transport drift and can throw on malformed disk-controlled comparison fields before falling back to full planning; it also needs the required changelog entry.

## What This Changes

Adds `targetProvider`/`targetModel` options to `ensureOpenClawModelsJson`, a cold-cache `models.json` short-circuit with provider-level structural checks, focused tests, and a few formatting-only edits.

## Best Possible Solution

Keep the optimization only if the cold-cache path validates or rejects the complete target provider/model request-shaping surface with bounded disk parsing, then add the required changelog entry before merge.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to generate `models.json`, edit the target provider's model entry to change `api`, `baseUrl`, or headers while leaving provider-level fields matching config, clear `MODELS_JSON_STATE.readyCache`, and call `ensureOpenClawModelsJson` with `targetProvider`; the PR path can return before full planning rewrites the entry.

## Solution Assessment

No. Provider-level comparison is narrower than the runtime request surface; the safer fix is to compare the target model/request transport fields or fail closed to full planning, with bounded comparison handling for malformed disk JSON.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Reject model-level transport drift before short-circuiting:** `src/agents/models-config.ts:456-462`
  - body: The fast path returns after comparing only provider-level `apiKey`, `baseUrl`, `headers`, and `auth`, but runtime request construction can still use `discoveredModel.api`, `discoveredModel.baseUrl`, and discovered model headers from `models.json`. A disk edit that keeps the provider row matching but changes a model transport field will skip the full plan that would regenerate the entry, so requests can keep using the tampered destination or headers. Compare the target model's request-shaping fields or fail closed when unvalidated disk model entries are present.
  - confidence: 0.9
- **[P2] Fail closed when disk comparison cannot be stringified:** `src/agents/models-config.ts:371-375`
  - body: `stableEqual` recursively stringifies disk-controlled `diskProvider.headers` and `diskProvider.auth` with no depth guard or catch. A small but deeply nested `models.json` value can throw during the short-circuit check instead of falling through to full planning, turning malformed disk state into an agent startup/request DoS. Validate these fields as bounded JSON shapes, or catch comparison failures and return false.
  - confidence: 0.78
- **[P3] Add the required changelog entry:** `src/agents/models-config.ts:428-442`
  - body: This is a user-visible perf change to the agent/model hot path, but the branch does not update `CHANGELOG.md`. Add a single-line entry in the active release section with appropriate contributor credit before merge.
  - confidence: 0.83

## Security Review

Status: needs_attention

Summary: Security review needs attention because the new fast path can preserve or crash on disk-controlled model transport metadata from `models.json`.

Concerns:

- **[high] Unvalidated model transport can survive the fast path:** `src/agents/models-config.ts:456`
  - body: Because `models[]` is not compared before returning from the short-circuit, a tampered per-model `baseUrl`, `api`, or header-bearing model entry can remain on disk and later shape embedded-runner requests even though a full plan would regenerate that provider entry.
  - confidence: 0.9
- **[medium] Recursive disk comparison can fail as a DoS:** `src/agents/models-config.ts:371`
  - body: The new disk-vs-config comparison recursively stringifies disk-controlled `headers` and `auth` without bounding depth or catching errors, so malformed `models.json` can throw before the intended fail-closed planning path runs.
  - confidence: 0.78
