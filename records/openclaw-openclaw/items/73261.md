## Summary

Keep this PR open. Current main does not already implement the proposed target-provider fast path, but the patch should not merge as-is because the new short-circuit can accept a disk provider entry after checking only provider-level fields while runtime request construction still consumes model-level transport fields from `models.json`.

## What This Changes

This PR adds `targetProvider`/`targetModel` options to `ensureOpenClawModelsJson`, a cold-cache disk-vs-config provider short-circuit, and targeted tests for match, drift, and cache behavior.

## Best Possible Solution

Keep the optimization only if the short-circuit proves the full runtime request-shaping surface is consistent with the planned provider state, or make it fail closed whenever unvalidated disk model/request fields could affect transport. Add focused regression coverage for tampered `models[].baseUrl`, `models[].api`, and model/header/request fields before merge.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to generate `models.json`, edit the target provider's model entry to add a malicious `baseUrl`, `api`, or header while leaving provider-level `apiKey`/`baseUrl`/`headers`/`auth` matching config, clear the in-memory cache, and call `ensureOpenClawModelsJson` with `targetProvider`.

## Solution Assessment

No. The proposed provider-level comparison is narrower than the runtime request surface; the safer maintainable solution is to validate or reject model-level and request-shaping drift before skipping the full planning path.
