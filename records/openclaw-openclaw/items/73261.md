## Summary

Keep this PR open: current main does not implement the new `targetProvider` fast path, but the proposed fast path still needs changes before merge because it can preserve unvalidated model-level request transport from `models.json`.

## What This Changes

Adds `targetProvider`/`targetModel` options to `ensureOpenClawModelsJson`, a cold-cache `models.json` structural-match fast path, focused tests, and a few formatting-only test changes.

## Best Possible Solution

Keep the optimization only if the fast path validates the full runtime request-shaping surface for the target model, or fails closed to full planning whenever disk model/request transport fields are present but unverified.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to generate `models.json`, edit the target provider's model entry to add a malicious `baseUrl`, `api`, or headers while leaving provider-level fields matching config, clear the in-memory cache, and call `ensureOpenClawModelsJson` with `targetProvider`.

## Solution Assessment

No. Provider-level comparison alone is narrower than the runtime request surface; the safer maintainable solution is to compare model-level/request-shaping drift or reject the short-circuit before skipping full planning.
