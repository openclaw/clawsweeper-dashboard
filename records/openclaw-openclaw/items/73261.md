## Summary

Keep open. The optimization is useful, but the current PR still trusts a provider-level match while runtime requests can be shaped by persisted model-level transport fields, and malformed disk-controlled headers/auth can throw before the intended full-planning fallback.

## What This Changes

The PR adds targetProvider/targetModel options to ensureOpenClawModelsJson, a cold-cache models.json short-circuit with provider-level structural checks, focused tests, and a few formatting-only test changes.

## Best Possible Solution

Keep the optimization only if the cold-cache path validates the complete runtime request-shaping surface for the target provider/model with bounded comparisons; otherwise fail closed to full planning and add the required changelog entry.

## Reproduction Assessment

Yes. Generate models.json, edit the target provider's model entry to change baseUrl, api, or headers while leaving provider-level fields matching config, clear the ready cache, then call ensureOpenClawModelsJson with targetProvider; the PR fast path can return before the full plan rewrites the entry.

## Solution Assessment

No. Provider-level comparison is narrower than the current runtime request surface; the safer maintainable fix is to compare or reject target model/request transport drift and make malformed disk values fall through rather than throw.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Reject model-level transport drift before short-circuiting:** `src/agents/models-config.ts:456-462`
  - body: The fast path returns after comparing only provider-level apiKey, baseUrl, headers, and auth, but runtime request construction can still use discoveredModel.api, discoveredModel.baseUrl, and discovered model headers from models.json. A disk edit that keeps the provider row matching but changes a model transport field will skip the full plan that would regenerate the entry, so requests can keep using the tampered destination or headers. Compare the target model's request-shaping fields or fail closed when unvalidated disk model entries are present.
  - confidence: 0.9
- **[P2] Bound disk comparisons before trusting the fast path:** `src/agents/models-config.ts:371-375`
  - body: stableEqual recursively stringifies diskProvider.headers and diskProvider.auth with no depth guard or catch. A small but deeply nested models.json value can throw during the short-circuit check instead of falling through to full planning, turning a malformed disk file into an agent startup/request DoS. Validate these fields as bounded JSON shapes, or catch comparison failures and return false.
  - confidence: 0.78
- **[P3] Add the required changelog entry:** `src/agents/models-config.ts:428-442`
  - body: This is a user-visible perf change to the agent/model hot path, but the branch does not update CHANGELOG.md. Add a single-line Unreleased entry with appropriate contributor credit before merge.
  - confidence: 0.83

## Security Review

  - body: Because models[] is not compared before returning from the short-circuit, a tampered per-model baseUrl, api, or header-bearing model entry can remain on disk and later shape embedded-runner requests even though a full plan would regenerate that provider entry.
