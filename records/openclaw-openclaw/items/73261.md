## Summary

Keep open: this PR is not implemented on current main, and latest head still has a cache-integrity bug where the scoped short-circuit caches a later `models.json` read than the one that passed structural validation.

## What This Changes

This PR adds a provider/model-scoped `ensureOpenClawModelsJson` fast path, content-hash cache validation, pi embedded caller wiring, regression tests, and a changelog entry.

## Best Possible Solution

Bind the scoped cache entry and return path to the exact validated `models.json` snapshot, then refresh latest-head runtime proof for the target-provider fast path.

## Reproduction Assessment

Yes. Source inspection of PR head shows the validator reads one `models.json` snapshot, while the cache entry is populated from a second path read after validation.

## Solution Assessment

No. The fast path should return the validated content outcome from `readExistingProviderMatchesConfig`, or compare any second read against the validated hash before caching or returning.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Cache only the validated models.json snapshot:** `src/agents/models-config.ts:1397-1401`

  - confidence: 0.84
