## Summary

Keep open. The latest head still has a security-sensitive fast path that can accept provider-level matches while preserving unvalidated model-level request-shaping data, so it should not merge without maintainer/author review of the models.json trust boundary.

## What This Changes

This PR adds targetProvider/targetModel options and a cold-cache models.json provider-level disk-vs-config short-circuit for ensureOpenClawModelsJson, adds focused tests, and includes formatting-only updates in several unrelated files.

## Best Possible Solution

Keep the optimization only if the cold-cache fast path proves that all runtime request-shaping state is consistent with regenerated provider state, or fails closed when disk model/request fields could affect transport. Add regression coverage for tampered models[].baseUrl, models[].api, and model-level headers/request metadata before relying on the shortcut.

## Reproduction Assessment

Yes. Generate models.json, edit the target provider's model entry to add a malicious baseUrl, api, or headers while leaving provider-level fields matching config, clear the ready cache, and call ensureOpenClawModelsJson with targetProvider; the PR fast path can return before the full plan rewrites the entry.

## Solution Assessment

No. The proposed provider-level comparison is narrower than the runtime request surface; the safer maintainable solution is to compare the target model's transport/request fields or reject the short-circuit whenever unvalidated disk model entries are present.
