## Summary

Keep this PR open: the current head still has blocking correctness and security issues in the new persisted models.json fast path, and the external contributor has not provided after-fix real behavior proof.

## What This Changes

The PR adds targetProvider/targetModel options to ensureOpenClawModelsJson with a disk-vs-config short-circuit, focused tests, plugin SDK baseline hash updates, and formatting-only test changes.

## Best Possible Solution

Keep the optimization only after it scopes cache readiness to the validated mode, validates or rejects target model/request-shaping drift, bounds disk comparisons, adds a changelog entry, and includes real gateway or embedded-runner proof.

## Reproduction Assessment

Yes, from source inspection. A targeted call can validate one matching provider and populate the global readyCache, then a later non-targeted call with the same fingerprint can skip full planning; separately, disk model transport drift survives because models[] is excluded from the short-circuit comparison.

## Solution Assessment

No. The proposed provider-level comparison and global cache write are narrower than the current runtime request surface; the safer fix is provider/mode-scoped cache readiness plus fail-closed validation of target model transport fields.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Scope cache entries to validated target provider:** `src/agents/models-config.ts:461`
  - body: After a targetProvider short-circuit succeeds, this writes a normal readyCache entry keyed only by fingerprint. A later ensureOpenClawModelsJson call without targetProvider can hit that cache and skip full planning even though only one provider was validated, leaving other providers stale on disk.
  - confidence: 0.92
- **[P1] Reject model transport drift before short-circuiting:** `src/agents/models-config.ts:278-282`
  - body: The fast path explicitly excludes models[] from validation, but runtime model resolution can use discoveredModel.api, discoveredModel.baseUrl, and discovered model headers from models.json. A disk edit that keeps provider-level fields matching can therefore preserve stale or tampered request transport instead of falling through to full planning.
  - confidence: 0.9
- **[P2] Fail closed on malformed disk apiKey values:** `src/agents/models-config.ts:367-368`
  - body: When config has no apiKey, this only rejects non-empty string disk apiKey values. A non-string apiKey in models.json can still be treated as a match and avoid the full plan that would drop malformed provider state.
  - confidence: 0.81
- **[P2] Bound recursive disk comparisons:** `src/agents/models-config.ts:371-374`
  - body: stableEqual recursively stringifies disk-controlled headers/auth without a depth guard or catch. A small deeply nested models.json value can throw during the short-circuit check instead of falling through to the normal planning path.
  - confidence: 0.8
- **[P3] Add the required changelog entry:** `src/agents/models-config.ts:428-442`
  - body: This adds a user-facing performance/API change on the model preparation path, but the branch does not add an Unreleased CHANGELOG.md entry. Add one single-line entry with appropriate human contributor credit before merge.
  - confidence: 0.82
