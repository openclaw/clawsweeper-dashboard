## Summary

Keep open: current main does not implement this targetProvider fast path, and the PR still has security/correctness blockers plus missing current-head real behavior proof.

## What This Changes

The PR adds targetProvider/targetModel options to ensureOpenClawModelsJson, a cold-cache models.json short-circuit with provider-level disk/config checks, focused tests, generated SDK baseline updates, and small formatting-only test changes.

## Best Possible Solution

Keep the optimization only if the cold-cache path validates the full target provider/model request-shaping surface with bounded fail-closed comparisons, includes a changelog entry, and is backed by real runtime proof.

## Reproduction Assessment

Yes, from source inspection. Generate models.json, edit the target provider's model entry to change baseUrl, api, or headers while provider-level fields still match config, clear readyCache, and call ensureOpenClawModelsJson with targetProvider; the PR fast path can return before full planning rewrites the entry.

## Solution Assessment

No. Provider-level comparison is narrower than the runtime request surface; the safer maintainable fix is to compare or reject target model/request transport drift and make malformed disk comparisons fall through instead of throwing.
