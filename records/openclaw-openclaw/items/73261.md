## Summary

Keep this PR open. Current main does not already implement the target-provider API, and the proposed fast path still needs changes before merge because it can accept provider-level disk state while preserving unchecked model-level transport metadata; the branch also does not wire the new option into the embedded-runner call sites that motivated the performance work.

## What This Changes

The PR adds `targetProvider`/`targetModel` options, a disk-vs-config short-circuit in `ensureOpenClawModelsJson`, a focused target-provider test file, and a few formatting-only edits in unrelated files.

## Best Possible Solution

Keep the optimization only after the cold-cache fast path proves the full runtime request-shaping surface is consistent with regenerated provider state, or fails closed when unchecked disk model/request fields could affect transport. Then thread the option through the embedded-runner and compaction call sites that already know the provider/model, and add regression coverage plus the required changelog entry.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to generate `models.json`, edit the target provider's model entry to add a malicious `baseUrl`, `api`, or headers while leaving provider-level fields matching config, clear the in-memory cache, and call `ensureOpenClawModelsJson` with `targetProvider`; the PR fast path can return before full planning rewrites the model entry.

## Solution Assessment

No. Provider-level comparison is narrower than the runtime request surface, and the current branch leaves the new hint test-only rather than applying it to the motivating embedded-runner paths. A safer solution validates or rejects model-level transport drift before skipping planning, then wires the option through known-provider call sites.

## Security Review

Status: needs_attention

Summary: Security review needs attention because the new `models.json` short-circuit can preserve unchecked model-level transport metadata and uses a race-prone path validation/read sequence.

Concerns:

- **[high] Unvalidated model transport can survive the fast path:** `src/agents/models-config.ts:456`
  - body: Because `models[]` is not compared before returning from the short-circuit, a tampered per-model `baseUrl`, `api`, or header-bearing model entry can remain on disk and later shape embedded-runner requests even though a full plan would regenerate that provider entry.
  - confidence: 0.92

  - body: The PR's `lstat` checks can be invalidated before `fs.readFile` reopens the path, so the claimed symlink and size-cap protections are not race-resistant for the new cold-cache trust decision.
