## Summary

Keep open: the latest PR head still misses the stated production performance path, because the embedded callers do not pass the new hint and the disk/config comparison rejects a common planner-produced env marker state.

## What This Changes

The PR adds content-hash-backed models/auth cache validation plus a provider-scoped `targetProvider` short-circuit for `ensureOpenClawModelsJson`, with tests and a changelog entry.

## Best Possible Solution

Keep the hardened cache/file validation design, then thread provider/model hints through the embedded run and compact paths and compare against the same normalized provider auth shape the planner writes.

## Reproduction Assessment

Yes. Source inspection of the PR head shows both production embedded callers omit `targetProvider`, and the planner can write env-derived apiKey markers that the new compare rejects when raw config omits apiKey.

## Solution Assessment

No. The hardening approach is maintainable, but the current merge shape is not the best fix until the stated production callers reach the fast path and common env-auth disk state can match safely.
