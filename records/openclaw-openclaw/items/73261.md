## Summary

Keep open. The optimization is not implemented on current main, but the proposed fast path still trusts on-disk model-level transport fields after comparing only provider-level fields, so it can preserve a tampered request destination that the full plan would otherwise regenerate.

## What This Changes

The branch adds a `targetProvider`/`targetModel` fast path to `ensureOpenClawModelsJson` that skips implicit provider discovery when the on-disk provider entry matches selected config fields, plus focused tests for match, mismatch, and cache behavior.

## Best Possible Solution

Keep the optimization only if the fast path proves the full runtime request-shaping surface is consistent with the planned provider state, or make it fail closed whenever unvalidated disk model/request fields could affect transport. Add regression coverage for tampered `models[].baseUrl`, `models[].api`, model headers, and relevant request overrides before merge.

## Reproduction Assessment

Yes. Generate `models.json`, edit the target provider's model entry to add a malicious `baseUrl` or `api` while leaving provider-level `apiKey`/`baseUrl`/`headers`/`auth` matching config, clear the in-memory cache, and call `ensureOpenClawModelsJson` with `targetProvider`; the PR fast path can return before the full plan rewrites that entry.

## Solution Assessment

No. The provider-level comparison is narrower than the runtime request surface; the safer maintainable approach is to validate or reject model-level and request-shaping drift before skipping planning.
