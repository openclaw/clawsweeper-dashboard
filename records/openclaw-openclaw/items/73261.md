## Summary

Keep open: this is useful performance/security work, but the latest patch is not merge-ready because the new embedded-runner hint can validate one provider while the next model resolution uses a different runtime provider.

## What This Changes

The branch replaces mtime-based models.json readiness caching with content-hash drift checks, adds targetProvider/targetModel short-circuit validation for ensureOpenClawModelsJson, threads those hints through embedded callers, and adds focused tests plus a runtime proof script.

## Best Possible Solution

Land after the target hint follows the provider that resolveModelAsync will actually use, or disables the short-circuit when configured and runtime providers diverge, with OpenAI Codex-routed regression coverage.

## Reproduction Assessment

Yes, by source inspection: compaction can compute a runtimeProvider that differs from provider, but the PR passes provider as targetProvider before resolving runtimeProvider. I did not run a live failing compaction scenario in this read-only review.

## Solution Assessment

No; the cache and validation direction is reasonable, but the caller hint must be scoped to the actual runtime provider or disabled when provider/runtime diverge. That is the narrowest fix before merge.
