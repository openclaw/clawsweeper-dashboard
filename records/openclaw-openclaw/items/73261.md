## Summary

Keep open. The branch is useful, but it is not merge-ready because the new short-circuit can preserve tampered model-level request transport fields while current runtime still consumes those fields from `models.json`; the PR branch is also dirty against current `main`.

## What This Changes

This PR adds `targetProvider`/`targetModel` options to `ensureOpenClawModelsJson`, a provider-level disk-vs-config fast path before implicit provider discovery, and a new targeted test file for the fast path and cache behavior.

## Best Possible Solution

Keep the fast path only after it validates every disk field that can affect runtime requests, or make it fail closed when disk model/request-shaping fields are present and unverified. Add regression coverage for tampered `models[].baseUrl` and `models[].api`, then rebase onto current `main`.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to generate `models.json`, edit the target provider’s model entry to add a malicious `baseUrl` while keeping provider-level fields matching config, clear the ready cache or restart, then call `ensureOpenClawModelsJson` with `targetProvider`; the PR fast path can return before full planning rewrites the model entry.

## Solution Assessment

No. The proposed provider-level comparison is narrower than the runtime request surface; the safer fix is to compare or reject model-level and request-shaping drift before skipping the full planning path.
