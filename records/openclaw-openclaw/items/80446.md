## Summary

Keep open: current main still routes Telegram Bot API fetches through undici/proxy fetches without stamping a User-Agent, and the only linked fix PR was closed unmerged after review found proof/version-source blockers.

## What This Changes

The issue asks OpenClaw to stamp Telegram Bot API outbound requests with a stable User-Agent instead of relying on undici's default.

## Best Possible Solution

Add a Telegram-plugin-owned fetch wrapper that stamps a versioned OpenClaw User-Agent on every Bot API outbound path while preserving caller headers, Request/init semantics, proxy dispatchers, abort behavior, and fallback retry behavior.

## Reproduction Assessment

No for the ISP throttling end-to-end because it depends on a specific network path. Yes at source level: current main dispatches Telegram transport requests without a User-Agent and undici v8.2.0 supplies the bare default when absent.

## Solution Assessment

Yes: stamping at `resolveTelegramTransport` is the narrow owner-boundary fix because that module owns Bot API fetch selection, proxy dispatchers, and fallback behavior. The implementation should use a real service/runtime version source and should not strip extra headers unless maintainers explicitly choose that broader policy.

## Repair Work Prompt

Fix the Telegram Bot API outbound User-Agent issue reported in https://github.com/openclaw/openclaw/issues/80446. In `extensions/telegram/src/fetch.ts`, wrap the selected `sourceFetch` inside `resolveTelegramTransport` so every Telegram Bot API request gets a stable versioned User-Agent such as `OpenClawBot/<version>` when the caller did not already provide one. Preserve caller-provided User-Agent, Request-input headers, init.headers precedence, dispatcher/proxy handling, abort behavior, transport fallback retry behavior, and `sourceFetch` export semantics. Use a plugin-legal version source that covers managed services, including `OPENCLAW_SERVICE_VERSION`, instead of only `OPENCLAW_VERSION`; do not import core internals from the plugin. Do not bundle optional stripping of `Sec-Fetch-Mode` or `Accept-Language` unless separately justified. Add focused tests in `extensions/telegram/src/fetch.test.ts` for default stamping, caller User-Agent preservation, Request header preservation, dispatcher/fallback paths, and managed service version fallback. Add a single-line `CHANGELOG.md` fix entry because this is user-visible. Validation should run `pnpm test extensions/telegram/src/fetch.test.ts -- --reporter=verbose`; include redacted real behavior proof through the patched transport or a clearly documented blocker if live Telegram credentials are unavailable.
