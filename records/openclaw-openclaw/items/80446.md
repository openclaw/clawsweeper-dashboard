## Summary

Keep open: current main still routes Telegram Bot API traffic through the Telegram transport without stamping a User-Agent, and the only linked fix PR was closed unmerged. The network throttling is ISP-specific, but the missing-header source path and undici default are clear enough for a focused repair.

## What This Changes

Request to stamp Telegram Bot API outbound requests with a stable versioned User-Agent instead of relying on undici's default.

## Best Possible Solution

Add a Telegram-plugin-owned transport wrapper that stamps a stable versioned OpenClaw User-Agent on every Bot API outbound path while preserving existing proxy, dispatcher, fallback, abort, and caller-header semantics.

## Reproduction Assessment

No, not for the ISP throttling end-to-end because it depends on a specific network path. Yes at source level: current main sends Telegram transport requests without a User-Agent, and undici 8.2.0 supplies the `undici` default when the header is absent.

## Solution Assessment

Yes for the core issue: stamping the header in `resolveTelegramTransport` is the narrow maintainable fix because that module owns Telegram Bot API fetch, proxy, dispatcher, and fallback behavior. Stripping extra headers should stay out of scope unless separately justified by maintainer review.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/80446 with a narrow Telegram transport change. In `extensions/telegram/src/fetch.ts`, wrap the selected Telegram `sourceFetch` so every Bot API outbound path gets `User-Agent: OpenClawBot/<version>` when the caller did not provide a User-Agent. Preserve caller-provided User-Agent values, Request-input headers, init headers, custom proxy/apiRoot behavior, EnvHttpProxyAgent/ProxyAgent dispatchers, abort/capture behavior, sticky fallback, and `transport.sourceFetch` media behavior. Use a plugin-legal version source: prefer `OPENCLAW_VERSION`, then `OPENCLAW_SERVICE_VERSION`, then `npm_package_version` or the plugin/package version if available without importing core internals; avoid making managed installs stamp `OpenClawBot/unknown` unless there is no safe version source. Do not strip `Sec-Fetch-Mode` or `Accept-Language` unless a maintainer explicitly expands scope. Add focused regression tests in `extensions/telegram/src/fetch.test.ts` for default stamping, caller-provided UA preservation for Request and init headers, dispatcher/fallback stamping, proxy fetch behavior, and service-version env fallback. Validate with `pnpm test extensions/telegram/src/fetch.test.ts -- --reporter=verbose`; include redacted real transport proof in the PR body when feasible.
