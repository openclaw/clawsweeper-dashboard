The PR changes `parseConfigValue` to accept exponent notation as numeric input and adds colocated Vitest coverage for the parser value types.

## Best Possible Solution

## Solution Assessment

A broader alternative would be to align this parser with the JSON5-based `openclaw config set` value parser, but that would change more syntax than this PR needs.
