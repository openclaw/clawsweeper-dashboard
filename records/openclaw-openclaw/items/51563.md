## Summary

Keep this PR open: the underlying status bug still exists on current main, but the PR is not merge-ready because it misses supported custom Bun global roots, is now conflicting, and lacks real after-fix global-install proof.

## What This Changes

The PR adds a path-heuristic global install package-manager detector, uses it for package installs in `checkUpdateStatus`, and adds detector tests.

## Best Possible Solution

Wire package-install status reporting through the existing global-install detection contract, including npm Homebrew, pnpm, and custom Bun roots, while preserving current status behavior and adding focused regression coverage plus real global-install proof.

## Reproduction Assessment

Yes by source inspection: current main reads `package.json.packageManager` through `detectPackageManager(root)`, the root metadata is pnpm, and status renders that value for package installs. I did not run a live npm-global install because this review was read-only.

## Solution Assessment

No. The PR fixes the npm Homebrew case directionally, but the new path-only detector is narrower than the existing `detectGlobalInstallManagerForRoot()` contract and misses custom Bun global roots.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Detect Bun installs from custom BUN_INSTALL roots:** `src/infra/detect-package-manager.ts:51-52`
  - body: `detectGlobalInstallManager` only returns `bun` when the path contains `/.bun/`, then falls back to `npm`. Bun supports custom `BUN_INSTALL` roots such as `/opt/bun/install/global/node_modules/openclaw`, and current update code already resolves that layout via `BUN_INSTALL`; this PR would still report those installs as `npm`. Please reuse the existing global-install detector or cover Bun's configured global root.
  - confidence: 0.9
