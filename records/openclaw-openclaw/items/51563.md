## Summary

Keep open. The PR targets a real current-main bug, but the patch should not merge as-is because it adds a weaker duplicate package-root detector and misses supported custom Bun global roots; current main still reports package installs from repository `packageManager` metadata.

## What This Changes

The PR adds a path-heuristic global install manager detector, switches package-install update status to use it, and adds unit tests for pnpm, Bun, and npm global path examples.

## Best Possible Solution

Route package-root status detection through the existing global-install ownership logic, keep repository package-manager detection for git checkouts, add regressions for npm-global OpenClaw metadata declaring pnpm and custom `BUN_INSTALL` Bun roots, and include the user-facing changelog credit when the fix lands.

## Reproduction Assessment

Yes. A source-level reproduction is clear: use a non-git OpenClaw package root whose `package.json.packageManager` is pnpm, as the package metadata is today, and current `checkUpdateStatus` returns that metadata-derived manager while `openclaw update status` renders it as `Install`.

## Solution Assessment

No. The requested direction is right, but this patch is not the best implementation because it duplicates and weakens `detectGlobalInstallManagerForRoot` instead of reusing or adapting the existing detector that handles symlinks, npm prefixes, and `BUN_INSTALL`.
