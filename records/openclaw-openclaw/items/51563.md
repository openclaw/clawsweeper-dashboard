## What This Changes

The PR changes update status package-install detection to infer npm, pnpm, or Bun from the resolved global package path rather than the installed package metadata.

## Best Possible Solution

Route package-install status through the existing global-install ownership detector or a shared status-safe wrapper, keep git/repo package-manager detection intact, cover custom Bun/pnpm and Homebrew npm roots, and attach redacted real status output from global installs.

## Reproduction Assessment

Yes, from source inspection: current main routes package roots through detectPackageManager(), that helper reads package.json packageManager first, and status renders the returned packageManager. I did not run a live global install in this read-only review.

## Solution Assessment

No. The PR targets the right status boundary, but the new helper is non-buildable as written and duplicates a narrower version of the existing global-install detector instead of sharing the broader ownership checks.
