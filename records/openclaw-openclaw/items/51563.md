## Summary

Keep open: the PR targets a real status bug that still exists on current main, but the patch should not merge as-is because its new heuristic misses supported custom Bun global installs and the PR lacks after-fix real behavior proof.

## What This Changes

The PR adds a global-install package-manager detector for update status, switches package installs away from package.json.packageManager, and adds tests for npm, pnpm, and Bun path heuristics.

## Best Possible Solution

Rebase the PR and make status package installs use the existing robust global-manager detection contract, with regression tests and redacted real `openclaw update status` proof for npm-global behavior.

## Reproduction Assessment

Yes, at source level. Current main still reads package.json.packageManager for checkUpdateStatus() and renders that value as the package install label; I did not run a real Homebrew npm global install in this read-only review.

## Solution Assessment

No, not as written. The direction is right, but the maintainable fix should reuse or align with the existing global-install detector so custom Bun roots and other supported global layouts are not regressed.
