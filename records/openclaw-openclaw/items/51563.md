## Summary

Keep open. Current main still has the status mis-detection path, and this PR targets the right bug, but the branch should not merge as-is because it adds a narrower path-only detector instead of reusing the existing global-install detector and it only provides unit-test proof.

## What This Changes

The PR adds a path-based global package-manager detector, uses it for package-install update status, and adds detector tests for npm, pnpm, and Bun paths.

## Best Possible Solution

Wire package-install status through `detectGlobalInstallManagerForRoot()` or a shared equivalent while preserving current realpath and registry behavior, add focused regressions, and attach redacted real global-install proof.

## Reproduction Assessment

Yes. Source inspection against current main shows a non-git package root flows through `checkUpdateStatus()` to `detectPackageManager()`, which reads `package.json.packageManager`, and status renders that value as `Install`; I did not run a live global install in this read-only review.

## Solution Assessment

No. The PR fixes the right boundary, but the proposed detector is narrower than the existing global-install detector and would miss supported custom pnpm and Bun roots.
