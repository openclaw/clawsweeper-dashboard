## Summary

## What This Changes

The PR adds a path-based global install package-manager detector, uses it for non-git update status, and adds detector unit tests for npm, pnpm, and Bun paths.

## Best Possible Solution

Wire package-install status to the existing global-install detector or a shared status-safe wrapper, preserve git/repo package-manager behavior, cover Homebrew npm, custom pnpm, and configured Bun roots, then collect real global-install status output.

## Reproduction Assessment

Yes. Source inspection shows current main routes package roots through detectPackageManager(), that helper reads package.json packageManager first, and status renders the returned packageManager as Install; I did not run a live global install in this read-only review.

## Solution Assessment

No. The PR targets the right status boundary, but the new path-only helper duplicates and narrows the existing global-install detector instead of sharing the broader npm, pnpm, and Bun ownership checks.
