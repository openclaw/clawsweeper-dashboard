## Summary

Keep open: the reported status bug is still present on current main and this PR is the canonical implementation path for the closed related issue, but the patch needs a tighter fix and real behavior proof before merge.

## What This Changes

The PR adds a path-based global install manager detector, uses it for package-install update status, and adds unit tests for pnpm, Bun, and npm path examples.

## Best Possible Solution

Route package-root status detection through the existing global-install manager logic or a shared injectable equivalent, preserve repo package-manager detection for git checkouts, and cover npm Homebrew, custom Bun, and custom pnpm roots.

## Reproduction Assessment

Yes, at source level: a non-git package root whose package.json declares pnpm will make current checkUpdateStatus return pnpm as packageManager, and status renders that value as Install. I did not run a live global install in this read-only review.

## Solution Assessment

No. The PR targets the right bug, but its new path heuristics duplicate a weaker detector and miss supported custom Bun and pnpm global roots already handled by src/infra/update-global.ts.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Reuse the existing global-root detector:** `src/infra/detect-package-manager.ts:45-52`
  - body: The new detector only recognizes pnpm/Bun from hard-coded path substrings, so package installs under a custom BUN_INSTALL or custom pnpm global dir fall through to npm even though update-global already supports those roots. Route status through that detector or share its root-comparison logic instead of adding a narrower heuristic.
