## Summary

Keep open. The PR targets a real status bug and the linked report was closed in favor of this PR, but the patch is not merge-ready because it misclassifies supported custom Bun global installs as npm and lacks the required changelog entry.

## What This Changes

The PR adds a path-heuristic global install manager detector, switches package-install status checks to use it, and adds detector tests for pnpm, Bun, and npm global paths.

## Best Possible Solution

Land a rebased version that separates repo package metadata from actual global install ownership, reuses or mirrors the existing global installer semantics including custom `BUN_INSTALL`, and adds focused regression coverage plus a changelog entry.

## Reproduction Assessment

Yes from source inspection and the linked report's macOS Homebrew npm-global steps: current main reads `package.json.packageManager` for package installs and status renders that value. I did not execute the CLI because this review is read-only.

## Solution Assessment

No, not as written. The direction is right, but the detector should preserve the npm-global fix while covering Bun's `BUN_INSTALL` global layout and preserving current-main status/update-check behavior after rebase.
