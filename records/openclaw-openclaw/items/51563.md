## Summary

Keep open: current main still has the status mis-detection path, but this PR is not merge-ready because it duplicates a broader existing global-install detector with narrower path heuristics, lacks real behavior proof, and is currently merge-conflicting.

## What This Changes

The PR adds a path-based global install package-manager detector, uses it for package-install update status, and adds npm, pnpm, and Bun detector tests.

## Best Possible Solution

Route package-install status through detectGlobalInstallManagerForRoot() or a shared status-safe wrapper, keep git/repo package-manager detection intact, add custom Bun, pnpm, and Homebrew npm regressions, and attach redacted real global-install status proof.

## Reproduction Assessment

Yes, from source inspection: current main routes package roots through detectPackageManager(), that helper reads package.json packageManager first, and status renders the returned packageManager. I did not run a live global install in this read-only review.

## Solution Assessment

No. The PR targets the right status boundary, but the new path-only detector duplicates and narrows the existing global-install detector instead of sharing the broader npm, pnpm, and Bun ownership checks.
