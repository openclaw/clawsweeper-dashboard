## Summary

## What This Changes

The PR adds a path-heuristic global install manager detector, routes package-install update status through it, and adds detector tests for npm, pnpm, and Bun paths.

## Best Possible Solution

Route package-install status through detectGlobalInstallManagerForRoot() or a shared status-safe wrapper, keep git/repo package-manager detection intact, add custom Bun and pnpm regressions, and attach redacted real global-install status proof.

## Reproduction Assessment

Yes, from source inspection: current main routes package roots through detectPackageManager(), that helper reads package.json packageManager first, and status renders the returned packageManager as Install. I did not run a live global install in this read-only review.

## Solution Assessment
