## Summary

Keep open: current main still has the reported update-status bug, but this PR's path-only detector would misreport supported custom Bun global installs and the user-facing fix is missing a changelog entry.

## What This Changes

The PR adds a path-based global install manager detector for non-git update status and unit tests for pnpm, Bun, and npm global path examples.

## Best Possible Solution

Route package-root status detection through the existing global-install manager logic or a shared equivalent, preserve repo package-manager detection for git checkouts, and add regressions plus a changelog entry.

## Reproduction Assessment

Yes. A source-level reproduction is clear: a non-git package root with OpenClaw's package.json declaring pnpm will make current checkUpdateStatus return pnpm as the package install manager, which status then displays as Install.

## Solution Assessment

No. The PR targets the right bug, but the implementation duplicates weaker path heuristics instead of reusing the existing global-install detection that already handles real roots, npm prefixes, and BUN_INSTALL.
