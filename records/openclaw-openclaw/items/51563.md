## Summary

Keep open: current main still has the npm-global status mis-detection, but this PR is not merge-ready because the branch introduces a missing `path` import and uses a narrower path heuristic that misses supported custom Bun global roots.

## What This Changes

The PR adds a path-heuristic global install manager detector, switches package-install update status to use it, and adds detector tests.

## Best Possible Solution

Repair this PR to use or share a global-install detector aligned with the existing updater contract, cover npm Homebrew, pnpm, and configured Bun roots, preserve git-install behavior, and add real global-install proof.

## Reproduction Assessment

Yes. By source inspection, current main reads `package.json.packageManager` for package roots, the root metadata is pnpm, and `openclaw update status` renders that value for package installs.

## Solution Assessment

No. The intended status boundary is right, but this patch needs to import or avoid `path` and should align with `detectGlobalInstallManagerForRoot()` so configured Bun roots are handled.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.95

Full review comments:

- **[P1] Import path before using path.sep:** `src/infra/detect-package-manager.ts:42`
  - body: `detectGlobalInstallManager` calls `path.sep`, but this module does not import `node:path`. Because `checkUpdateStatus` now calls this function for package installs, `openclaw update status` would fail before reporting the install manager.
  - confidence: 0.96
- **[P2] Handle custom Bun install roots:** `src/infra/detect-package-manager.ts:51-52`
  - body: This detector only returns `bun` when the path contains `/.bun/`, then falls back to `npm`. Bun supports configured `BUN_INSTALL` roots, and current update code already resolves that root, so custom Bun installs would still be reported as npm.
  - confidence: 0.9
