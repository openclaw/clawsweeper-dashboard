## Summary

Keep open. Current main and v2026.5.22 still have the status mis-detection, but this PR is not merge-ready because it adds a second, path-only global-install detector that can drift from the existing updater detector, is merge-conflicting, and has only test/CI proof rather than real global-install output.

## What This Changes

The branch adds a path-heuristic global install manager detector, uses it in update status for package installs, and adds detector tests for npm, pnpm, and Bun global paths.

## Best Possible Solution

Route package-install status through detectGlobalInstallManagerForRoot() or a shared status-safe wrapper, preserve git/repo package-manager detection, cover custom roots, refresh the branch, and attach redacted real status output.

## Reproduction Assessment

Yes, from source inspection. Current main and v2026.5.22 call detectPackageManager(root), that helper reads package.json packageManager first, and status renders packageManager as the package Install label; I did not run a live global install in this read-only review.

## Solution Assessment

No. The PR targets the right status boundary, but the maintainable fix should reuse the existing global-install ownership detector or a shared wrapper instead of adding a path-only source of truth.
