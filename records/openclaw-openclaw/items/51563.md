## Summary

Keep this PR open: the underlying status bug is still present on current main and the linked bug report was closed in favor of this PR, but the branch is not merge-ready because it is conflicting, lacks real global-install proof, and its new path-only detector is narrower than the existing global install-manager contract.

## What This Changes

The branch adds a path-based global install package-manager detector, uses it for package-install update status, and adds detector tests for pnpm, Bun, and npm global paths.

## Best Possible Solution

Use the existing global install-manager ownership logic, or an equivalent shared helper, for package-install status while preserving current git and registry behavior, then add focused regression tests and real installed-CLI proof.

## Reproduction Assessment

Yes. Source inspection of current main shows a package root flows through `checkUpdateStatus()` to `detectPackageManager()`, then `openclaw update status` renders that value as `Install`; I did not run a live global install in this read-only review.

## Solution Assessment

No. The PR targets the right status path, but the durable fix should align with `detectGlobalInstallManagerForRoot()` or equivalent behavior, preserve current-main `update-check` changes during rebase, and include real global-install proof.
