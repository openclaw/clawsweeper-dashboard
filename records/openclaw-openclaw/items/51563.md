## Summary

Keep open. Current main still has the source-reproducible status mis-detection, and this PR targets the right bug, but it is not merge-ready because it adds a second, narrower global-install detector, is merge-conflicting, and has only unit-test proof.

## What This Changes

The PR adds a path-substring global install manager detector for status reporting, switches package installs in checkUpdateStatus() to use it, and adds npm, pnpm, and Bun detector unit tests.

## Best Possible Solution

Route package-install status through the existing global-install ownership detector or a shared status-safe wrapper, preserve git/repo package-manager detection, cover custom roots, refresh the branch, and attach redacted real status output.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: package-root status calls detectPackageManager(), that helper reads package.json packageManager first, and status renders the result as the package Install label; I did not run a live global install in this read-only review.

## Solution Assessment

No. The PR targets the right status boundary, but the maintainable fix should reuse the existing global-install ownership detector or a shared wrapper instead of adding a path-only source of truth.
