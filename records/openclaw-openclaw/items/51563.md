## Summary

Keep open: the PR targets a real current-main status bug, but its path-only detector is weaker than the existing global-install detector and would misclassify supported custom Bun global installs; it also lacks the required changelog entry.

## What This Changes

The PR adds path-based global package-manager detection for non-git update status and tests for pnpm, Bun, and npm global install paths.

## Best Possible Solution

Route package-root status detection through the existing global-install manager logic or a shared equivalent, preserving repo package-manager detection for git checkouts and adding regressions plus changelog coverage.

## Reproduction Assessment

Yes. A source-level reproduction is clear: a non-git package root whose installed package.json declares pnpm makes current checkUpdateStatus return pnpm, and status renders that value as Install.

## Solution Assessment

No. The PR points at the right bug, but duplicating path heuristics is not the narrowest maintainable fix because current update-global code already handles npm prefixes, pnpm roots, and BUN_INSTALL-aware Bun roots.
