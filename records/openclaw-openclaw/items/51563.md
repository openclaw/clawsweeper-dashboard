## Summary

Keep this PR open because it targets a real current-main status bug, but the patch is not mergeable as proposed: the new helper references an unimported `path` binding and duplicates weaker global-install heuristics than the existing updater path.

## What This Changes

The PR adds a path-based global-install package-manager detector, wires non-git update status to use it, and adds unit tests for pnpm, Bun, and npm global path examples.

## Best Possible Solution

Route package-install status through the existing global-install manager detection or a shared equivalent, with regressions for npm Homebrew/global paths plus custom Bun and pnpm roots while preserving repo package-manager detection for git checkouts.

## Reproduction Assessment

Yes, source-level reproduction is high confidence. On current main, a non-git installed package root whose `package.json` declares OpenClaw's `pnpm` build tool will flow through `detectPackageManager(root)` and render that value as `Install`.

## Solution Assessment

No. The PR targets the right bug, but the proposed path-only helper has an immediate missing-import defect and is weaker than the existing global-install detection that already accounts for `BUN_INSTALL`, npm prefixes, and pnpm layouts.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.95

Full review comments:

- **[P1] Import path before using path.sep:** `src/infra/detect-package-manager.ts:37`
  - body: The new helper calls `resolvedRoot.split(path.sep)`, but this file never imports `node:path`. Package-install status now reaches this helper, so the changed path throws `ReferenceError: path is not defined` instead of reporting the install manager.
  - confidence: 0.98
- **[P2] Honor custom Bun global roots:** `src/infra/detect-package-manager.ts:45-47`
  - body: The new detector only returns Bun when the resolved package path contains `/.bun/`. OpenClaw's updater already supports custom `BUN_INSTALL` roots, so installs under paths like `/opt/bun/install/global/node_modules/openclaw` fall through to npm and report the wrong manager.
  - confidence: 0.9
