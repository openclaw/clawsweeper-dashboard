## Summary

The current PR addresses a real status bug, but it is not merge-ready because it duplicates a broader existing global-install detector with narrower path heuristics, lacks real global-install proof, and is currently merge-conflicting.

## What This Changes

The PR adds a path-based global install manager detector, uses it for package-install update status, and adds detector unit tests for npm, pnpm, and Bun global paths.

## Best Possible Solution

Route package-install status through the existing global-install ownership detector or a shared status-safe wrapper, keep git/repo package-manager detection intact, cover custom Bun and pnpm roots, and attach redacted real status output from global installs.
