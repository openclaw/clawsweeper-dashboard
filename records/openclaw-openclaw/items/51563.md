## Summary

Keep open: current main still has the status mis-detection path and this PR targets the right bug, but it is not merge-ready because it duplicates a narrower global-install detector, lacks real global-install proof, and is merge-conflicting.

## What This Changes

The PR adds path-heuristic global install manager detection, uses it for package-install update status, and adds detector tests for npm, pnpm, and Bun global paths.

## Best Possible Solution

Route package-install status through detectGlobalInstallManagerForRoot() or a shared status-safe wrapper, preserve git/repo package-manager detection, add custom Bun/pnpm/npm regressions, and prove real global-install status output.

## Reproduction Assessment

Yes, from source inspection: current main routes package roots through detectPackageManager(), that helper reads package.json packageManager first, and status renders the returned packageManager. I did not run a live global install in this read-only review.

## Solution Assessment

No. The PR targets the right status boundary, but the new path-only detector duplicates and narrows the existing global-install detector instead of sharing the broader npm, pnpm, and Bun ownership checks.
