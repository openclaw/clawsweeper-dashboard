## Summary

Keep this PR open: current main still has the reported status mis-detection and the linked issue was closed in favor of this PR, but the patch is not ready because it adds a narrower path-only detector instead of reusing the broader global-install detector and it still lacks real global-install proof.

## What This Changes

The PR adds a path-based global install package-manager detector, uses it for package-install update status, and adds detector tests for npm, pnpm, and Bun global paths.

## Best Possible Solution

Use `detectGlobalInstallManagerForRoot()` or a shared equivalent for package-install status, preserve current realpath and registry-channel behavior, cover npm Homebrew, custom pnpm, and configured Bun roots, then attach redacted real installed-CLI proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a package root flows through `checkUpdateStatus()` to `detectPackageManager()`, which reads package metadata, and status renders that value as `Install`; I did not run a live global install in this read-only review.

## Solution Assessment

No. The PR fixes the right boundary but adds a narrower path-only detector instead of sharing the existing global-install detector that already handles configured roots and manager ownership checks.
