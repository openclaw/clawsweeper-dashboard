## Summary

Keep open: current main still has the reported status mis-detection, and this PR targets the right boundary, but the proposed patch is not mergeable as-is because it introduces a missing `path` import bug, misses the existing custom Bun global-root behavior, and lacks real global-install proof.

## What This Changes

The PR adds a path-heuristic global-install package-manager detector, switches package-install update status to use it, and adds detector tests.

## Best Possible Solution

Repair this PR or replace it with a narrow fix that aligns status detection with the existing global-install detector, preserves git-install behavior, covers npm Homebrew, pnpm, and configured Bun roots, and includes real global-install proof.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: a non-git package root whose `package.json` declares pnpm will return pnpm from `checkUpdateStatus()`, and `openclaw update status` renders that value as `Install`. I did not run a live global install in this read-only review.

## Solution Assessment

No. The direction is right, but the current patch should first fix the missing import and align with the existing global-install-manager contract so configured Bun roots and other supported layouts are not misreported.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.95

Full review comments:

- **[P1] Import `path` before using `path.sep`:** `src/infra/detect-package-manager.ts:42`
  - body: `detectGlobalInstallManager()` calls `path.sep`, but this module does not import `node:path`. Because `checkUpdateStatus()` calls this for package installs, `openclaw update status` would fail before it can report the install manager.
  - confidence: 0.96
- **[P2] Handle Bun installs outside `~/.bun`:** `src/infra/detect-package-manager.ts:51-52`
  - body: The detector only returns `bun` when the package path contains `/.bun/`, then falls back to `npm`. Existing updater code resolves Bun's global root, so a configured `BUN_INSTALL` under another directory would still be reported as npm.
  - confidence: 0.9
