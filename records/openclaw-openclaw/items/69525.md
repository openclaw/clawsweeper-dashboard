## Summary

Keep open. Current main still resolves an implicit source-session `defaultAccountId` before cross-provider policy enforcement and forwards that account id into target plugin dispatch; the linked fix remains the open, unmerged https://github.com/openclaw/openclaw/pull/69529.

## What This Changes

Prevent implicit source-session `defaultAccountId` values from being passed to target channel plugins during allowed cross-provider message sends.

## Best Possible Solution

Land or replace the focused outbound runner fix so implicit source defaults are omitted for allowed cross-provider sends while explicit account ids and same-provider defaults continue to work.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: call `runMessageAction` with a source `defaultAccountId`, `allowAcrossProviders` enabled, and a different target provider, then assert target plugin dispatch does not receive the implicit source account id.

## Solution Assessment

Yes for the core direction. The narrow maintainable fix belongs in the shared outbound runner using the same effective cross-context policy as enforcement; plugin-side validation is useful defense-in-depth but should not be required.
