## What This Changes

The issue asks core outbound messaging to avoid forwarding a source session default account id into a different target provider during allowed cross-provider sends.

## Best Possible Solution

Land or replace the focused outbound runner fix so implicit source defaults are omitted for allowed cross-provider sends while explicit account ids and same-provider defaults continue to work.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: call `runMessageAction` with a source `defaultAccountId`, effective cross-provider policy enabled, and a different target provider, then assert the target plugin dispatch does not receive the implicit source account id.

## Solution Assessment

Yes for the core direction. The narrow maintainable fix belongs in the shared outbound runner using the same effective cross-context policy as enforcement; the linked open PR should be landed after cleanup or replaced with an equivalent focused fix.
