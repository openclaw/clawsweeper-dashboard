## Summary

Current `main` already fixes the reported npm sibling-pruning behavior by maintaining a package manifest in the managed npm root before every npm plugin install/update, so npm has the full intended dependency set instead of treating each install as a single-package tree.

## What This Changes

The issue asks for `openclaw plugins install --pin --force` and `openclaw plugins update --all` to preserve previously installed `@openclaw/*` plugin siblings under the managed npm root.

## Best Possible Solution

Keep the current managed npm-root manifest design and ship it in the next release so affected `v2026.5.2` users get the preservation behavior without changing package managers.

## Reproduction Assessment

No current-main failing reproduction is established; source inspection shows the old no-`package.json` npm-root path has been replaced by a manifest-preserving install path used by both install and update.

## Solution Assessment

Yes; maintaining the npm root `package.json` is the narrowest fix for npm pruning siblings, and it avoids switching the plugin installer to another package manager or adding a new user-facing policy.
