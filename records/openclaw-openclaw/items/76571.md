## Summary

Current main addresses the reported npm sibling overwrite by maintaining a managed `~/.openclaw/npm/package.json` dependency ledger before each npm plugin install, so npm resolves the full intended plugin set instead of treating each install as a single-package tree.

## What This Changes

Sequential npm-backed plugin installs and `plugins update --all` should preserve previously installed `@openclaw/*` sibling packages under the managed npm root.

## Best Possible Solution

Keep the managed npm-root manifest behavior on main and ship it in the next release; reopen only if a current-main or post-2026.5.2 build still loses sibling plugin packages.

## Reproduction Assessment

Yes for the affected release: the report gives a deterministic two-install and update-all path, and current source directly addresses the reported missing-manifest root cause. I did not run a live npm mutation because this review is read-only.

## Solution Assessment

Yes. Keeping the full intended plugin set in `~/.openclaw/npm/package.json` is the narrow maintainable fix because npm then resolves the managed root as a project rather than pruning around one transient install argument.
