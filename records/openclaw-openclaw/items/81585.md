## Summary

Keep open. The linked cron bug is real, but this PR's any-success predicate is too broad: it turns existing fatal trailing canvas/provider errors into successful runs whenever any earlier text payload exists, and the external PR only supplies helper/test proof rather than a real isolated cron run.

## What This Changes

The PR changes the isolated cron payload classifier to treat any non-error text payload in a turn as recovery from structured tool errors and updates regression tests accordingly.

## Best Possible Solution

Implement a narrower order-agnostic recovered-tool signal that preserves current fatal trailing-error coverage, then prove it with regression tests and a real isolated cron run.

## Reproduction Assessment

Yes. Source inspection shows the linked bug path in current main, and the PR diff source-reproduces a new regression by changing existing fatal trailing-error cases to nonfatal.

## Solution Assessment

No. The proposed predicate is the smallest textual change, but it is not the safest fix because it treats any earlier successful payload as recovery from any later structured error; a narrower recovered-tool signal should keep the existing fatal cases intact.
