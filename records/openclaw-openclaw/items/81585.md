## Summary

Keep open. The PR addresses a valid isolated-cron classifier bug, but the current patch still treats final assistant text plus any earlier successful payload as enough to clear a trailing exec warning, which would mark claimed-success exec failures as `ok` despite current main intentionally surfacing those failures.

## What This Changes

The PR changes isolated cron payload outcome classification to suppress recovered trailing exec warnings when final assistant-visible text exists and adds regression tests for recovered exec and fatal boundary cases.

## Best Possible Solution

Keep trailing exec warnings fatal unless the classifier receives an explicit recovery signal, acknowledgement, or validated sentinel, and cover both recovered fallback and claimed-success failure cases.

## Reproduction Assessment

Yes. Source inspection of the PR head shows `[assistant success text, Exec failed]` with `finalAssistantVisibleText` and an earlier non-error payload would clear `hasFatalStructuredErrorPayload`, while current main intentionally emits that exec warning for claimed-success failures.

## Solution Assessment

No. The patch is narrower than the first broad approach, but final assistant text alone is not a strong enough recovery signal for trailing exec failures.
