## Summary

Keep open. The PR targets a valid cron status bug, but the current patch still clears a trailing `Exec failed` payload whenever final assistant text and an earlier non-error payload exist, which can hide the existing claimed-success exec failure path.

## What This Changes

The PR changes isolated cron outcome classification so selected trailing exec warnings are treated as recovered, and adds focused regression tests for recovered exec and fatal trailing-error boundaries.

## Best Possible Solution

Define an explicit recovery or acknowledgement signal for isolated cron tool failures, then update the helper and regression tests so only that signal clears trailing exec warnings while claimed-success exec failures stay fatal.

## Reproduction Assessment

Yes. Source inspection shows `buildEmbeddedRunPayloads` can produce `[assistant success text, Exec failed]`, and the PR predicate would clear that trailing exec error when `finalAssistantVisibleText` and an earlier non-error payload exist.

## Solution Assessment

No. The PR is narrower than the original issue suggestion, but final assistant text plus an earlier successful payload is still too weak; the safer fix needs a concrete recovery or acknowledgement signal.
