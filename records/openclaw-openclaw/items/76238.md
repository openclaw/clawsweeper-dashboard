## What This Changes

The PR changes Telegram reply delivery to derive inline keyboards from `reply.interactive` when Telegram-specific buttons are absent, adds a delivery regression test, and adds an Unreleased changelog entry.

## Best Possible Solution

Land this PR after it is mergeable and exact-head checks pass, preserving Telegram-specific button precedence while using the shared interactive fallback for reply delivery.

## Solution Assessment

Yes. Reusing `resolveTelegramInlineButtons` in `deliverReplies` is the narrowest maintainable fix because it preserves Telegram-specific precedence and aligns reply delivery with the existing direct outbound path.
