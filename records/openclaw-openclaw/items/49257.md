## Summary

Keep open: current main has in-memory Control UI connection timing plus connected-only Debug and Logs tabs, but it still does not provide the requested pre-auth, copyable, ephemeral diagnostic drawer for connection failures. The linked implementation PR is closed unmerged and was not a safe canonical replacement.

## What This Changes

The issue requests an ephemeral Control UI diagnostic mode for gateway WebSocket lifecycle logs, handshake/error details, close codes, and copyable support output without persistent browser storage.

## Best Possible Solution

Design a small Control UI-owned, memory-only diagnostics surface that reuses the existing connect timing and log-redaction contracts, exposes only sanitized pre-auth status/close/error facts, and adds copy/export only after maintainer security approval.

## Reproduction Assessment

No; this is not a reproducible bug report, it is a feature request for a new diagnostics mode. Source inspection confirms current main lacks the requested pre-auth, copyable diagnostic surface.

## Solution Assessment

Unclear; the requested direction is plausible, but the best implementation depends on a maintainer decision about which pre-auth handshake, close, and gateway details may be displayed or copied. The current connect timing and Logs tab are useful building blocks but do not solve the issue by themselves.
