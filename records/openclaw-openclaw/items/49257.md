## Summary

Keep open: current main and v2026.5.22 have connected-only log tailing, a post-auth Debug event log/RPC view, and login failure hints, but not the requested session-only, copyable pre-auth WebSocket timeline. The remaining work needs maintainer product and security review before exposing handshake/auth details in the browser.

## What This Changes

Add a session-only Control UI debug mode that captures WebSocket connection lifecycle, handshake, close/error details, and copyable diagnostic logs without durable browser storage.

## Best Possible Solution

Design a small explicit Control UI/Gateway diagnostic contract that records redacted pre-auth lifecycle facts in memory, gives users a copyable session-only timeline, and keeps raw gateway log tailing behind the existing authenticated read-scope path unless maintainers approve a stricter pre-auth alternative.
