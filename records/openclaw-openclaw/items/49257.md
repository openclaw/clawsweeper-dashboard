## Summary

Keep this open. Current main improved WebSocket constructor failure guidance and has authenticated gateway log tailing, but it still lacks the requested session-scoped, copyable Control UI diagnostic timeline for pre-auth handshake, WebSocket lifecycle, close details, and errors.

## What This Changes

The issue asks for an ephemeral Control UI debug mode that captures and exposes copyable WebSocket connection, pre-auth handshake, close-code, and error diagnostics for the current browser session.

## Best Possible Solution

Keep this as a Control UI product/security design task: add a redacted in-memory diagnostic timeline at the GatewayBrowserClient boundary, then expose it through a focused debug surface with copy support.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a new diagnostic capability request. Source inspection shows current main still lacks the requested pre-auth/connection timeline and copyable session diagnostic surface.

## Solution Assessment

Unclear until maintainers settle UX and redaction rules. The narrow maintainable direction is to capture sanitized connection diagnostics in memory at `GatewayBrowserClient` and expose only redacted copy/export data in the Control UI.
