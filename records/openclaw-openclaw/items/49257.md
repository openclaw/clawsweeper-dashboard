## Summary

Keep this open: current main and the latest release still have targeted connection-error guidance, delivered-event Debug logs, and connected-only gateway log tailing, but not the requested ephemeral, copyable Control UI timeline for pre-auth WebSocket lifecycle, challenge, close, and error details.

## What This Changes

Add an ephemeral Control UI debug mode that captures gateway WebSocket connection lifecycle, handshake, close, and error details for the current browser session.

## Best Possible Solution

Capture sanitized connection diagnostics in memory at GatewayBrowserClient, expose a focused Control UI debug drawer with redacted copy/export, and keep server-side phase logging on the related gateway diagnostics track.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a new diagnostic capability request. Source inspection gives high confidence that current main lacks the requested pre-auth connection timeline and copyable session diagnostic surface.

## Solution Assessment

Unclear until maintainers settle UX and redaction rules. The narrow maintainable direction is an in-memory, sanitized GatewayBrowserClient diagnostic stream surfaced through Control UI copy/export, not raw log dumping.
