## Summary

Keep this open. Current main has a Debug tab, gateway event log, structured connect-error messages, and a connected-only `logs.tail` view, but it does not implement the requested session-scoped Control UI diagnostics for pre-auth WebSocket lifecycle, handshake frames, close details, or a copyable ephemeral drawer.

## What This Changes

Add a Control UI debug/diagnostic mode that captures ephemeral connection and handshake logs, displays them in the browser, and supports copying them for support without persistent storage.

## Best Possible Solution

Define a redacted, session-memory-only connection diagnostics model in the Control UI gateway client, then expose it through the existing Debug/Gateway Access surfaces with copy support and focused regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a new diagnostic capability request. Source inspection confirms the current UI has only a brief connection error, post-auth event logging, and connected-only gateway log tailing.

## Solution Assessment

Unclear until maintainers settle the redaction and UX boundary. The narrowest maintainable direction is a browser-memory diagnostic collector attached to the Control UI WebSocket client, with sensitive connect/auth fields summarized rather than copied raw.
