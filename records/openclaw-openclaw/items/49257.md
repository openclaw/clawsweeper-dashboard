## Summary

Keep this open. Current main and the latest shipped tag still provide only partial connection-error guidance, a normal post-connect event log, and connected-only log tailing; they do not provide the requested ephemeral, copyable Control UI timeline for pre-auth WebSocket lifecycle, challenge, close, and error details.

Add a Control UI debug mode that captures ephemeral, copyable WebSocket connection diagnostics and error details for the current browser session.

## Best Possible Solution

Keep this as a Control UI product/security design task: capture sanitized connection diagnostics in memory at the `GatewayBrowserClient` boundary, then expose a focused debug surface with copy support and redaction tests.

## Solution Assessment
