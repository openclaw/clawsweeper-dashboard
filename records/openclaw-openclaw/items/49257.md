## What This Changes

The issue requests a session-only Control UI debug mode that exposes websocket connection lifecycle logs, handshake/error details, close codes, and copyable diagnostics without durable browser storage.

## Best Possible Solution

Define a security-reviewed Control UI support diagnostics surface that extends the existing bounded connect-timing/event-log model, redacts sensitive auth/device/network data, works while disconnected, and keeps gateway log tailing behind authenticated operator scope.

## Reproduction Assessment

Not applicable as a bug repro; this is a new diagnostic UI request. Source and release inspection give high confidence that current main lacks the requested pre-auth, copyable diagnostic drawer.

## Solution Assessment

Unclear as proposed. The useful direction is not raw frame dumping by default, but a bounded, redacted support diagnostic surface that reuses existing connect timing and only exposes post-auth logs after authorization.
