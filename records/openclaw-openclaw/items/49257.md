## Summary

Keep this open. Current main has better Control UI connection failure guidance and gateway log tailing after connection, but it still lacks a session-scoped, copyable WebSocket/pre-auth diagnostic timeline, and the requested export needs maintainer UX and redaction decisions before implementation.

## What This Changes

Request adds a session-only Control UI debug mode that captures and copies WebSocket lifecycle, handshake, close, and error details without persistent browser storage.

## Best Possible Solution

Add a redacted, in-memory diagnostic timeline at the GatewayBrowserClient boundary and expose it through a Control UI connection/debug surface with copy support, focused tests, and explicit redaction rules.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a new diagnostic capability request. Source inspection still shows the missing surface because pre-auth frames are consumed internally and failures collapse to lastError/lastErrorCode.

## Solution Assessment

Unclear until maintainers settle UX and redaction policy. The narrow maintainable direction is a browser-memory diagnostic collector with sanitized summaries before any copy/export path.
