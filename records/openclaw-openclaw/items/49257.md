## Summary

Keep this open. Current main has improved connection failure guidance, but it still does not implement a session-scoped, copyable Control UI diagnostic timeline for WebSocket lifecycle, pre-auth handshake frames, close details, and errors.

## What This Changes

Request a Control UI debug mode that keeps ephemeral WebSocket connection and handshake diagnostics in the browser and lets users copy them for support.

## Best Possible Solution

Add a redacted, in-memory diagnostic timeline at the GatewayBrowserClient boundary and expose it through a Control UI connection/debug surface with copy support and focused tests.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a new diagnostic capability request. Source inspection on current main shows the missing surface because pre-auth frames are consumed internally and failures collapse to lastError/lastErrorCode.

## Solution Assessment

Unclear until maintainers settle UX and redaction rules. The narrow maintainable direction is a browser-memory diagnostic timeline with raw auth, device, URL, and handshake values summarized or redacted before copy/export.
