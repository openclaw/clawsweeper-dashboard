## Summary

Keep this open. Current main still has only structured login hints, a post-auth Debug event log, and connected-only log tailing, not the requested session-scoped, copyable pre-auth WebSocket lifecycle and handshake diagnostics; the linked implementation PR was closed unmerged after build-breaking review feedback.

## What This Changes

Request a Control UI debug mode that keeps ephemeral WebSocket connection logs, handshake/error details, close reasons, and a copyable support view for the current browser session.

## Best Possible Solution

Add a redacted, in-memory Control UI connection diagnostics collector at the GatewayBrowserClient boundary, then expose it through the existing login failure, Gateway Access, or Debug surfaces with copy support and focused tests.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a new diagnostic capability request. Source inspection on current main shows the gap by code path: connection failures collapse to lastError/lastErrorCode, while event logs and logs.tail only help after the relevant pre-auth frames are already unavailable.

## Solution Assessment

Unclear until maintainers settle the UX and redaction boundary. The narrow maintainable solution is a browser-memory diagnostic timeline attached to the Control UI WebSocket client, with raw auth/device fields summarized or redacted before copy/export.
