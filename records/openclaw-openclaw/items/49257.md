## Summary

Keep this open. Current main has better Control UI connection error guidance and authenticated gateway log tailing, but it still lacks the requested session-scoped, copyable pre-auth connection diagnostic timeline.

## What This Changes

The issue requests a session-only Control UI debug mode that exposes copyable connection lifecycle logs, pre-auth handshake details, close codes, and errors.

## Best Possible Solution

Add a redacted, in-memory diagnostic timeline at the GatewayBrowserClient boundary and expose it through a Control UI connection/debug surface with copy support and focused tests.

## Reproduction Assessment

Not applicable: this is a feature request, not a bug report. Source inspection shows the missing capability because pre-auth frames are consumed internally and failures still surface as lastError/lastErrorCode plus guidance.

## Solution Assessment

Unclear as proposed until maintainers settle UX and redaction rules. The maintainable direction is a browser-memory timeline that summarizes or redacts raw auth, device, URL, nonce, and close-detail values before copy/export.
