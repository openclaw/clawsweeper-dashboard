Keep this open. Current main has partial connection-error guidance and connected-only log tailing, but it still lacks the requested ephemeral, copyable Control UI diagnostic timeline for pre-auth handshake, WebSocket lifecycle, close details, and browser/gateway errors.

## What This Changes

The issue requests a Control UI debug mode that captures session-scoped connection diagnostics and copyable error detail without persisting logs beyond the browser session.

## Best Possible Solution

Keep this as a Control UI product/security design task: add a redacted in-memory diagnostic timeline at the GatewayBrowserClient boundary, then expose it through a focused debug surface with copy support.

Not applicable as a bug reproduction: this is a new diagnostic capability request. Source inspection gives high confidence that current main still lacks the requested pre-auth/connection timeline and copyable session diagnostic surface.

## Solution Assessment

Unclear until maintainers settle UX and redaction rules. The narrow maintainable direction is to capture sanitized connection diagnostics in memory at GatewayBrowserClient and expose only redacted copy/export data in the Control UI.
