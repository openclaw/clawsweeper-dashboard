## Summary

Keep this open. Current main improved some connection failure guidance, but it still does not provide the requested session-scoped, copyable Control UI diagnostics for WebSocket lifecycle, handshake frames, close details, and pre-auth failures.

## What This Changes

Add an ephemeral Control UI debug mode that captures and displays browser-side Gateway WebSocket connection logs and error details without durable storage.

## Best Possible Solution

Add a redacted, in-memory diagnostics collector at the GatewayBrowserClient boundary and expose it through an existing Control UI connection/debug surface with copy support and focused tests.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a new diagnostic capability request. Source inspection on current main shows the gap because failures collapse to lastError/lastErrorCode while event logs and logs.tail only help after the relevant pre-auth frames are unavailable.

## Solution Assessment

Unclear until maintainers settle the UX and redaction boundary. The narrow maintainable direction is a browser-memory diagnostic timeline attached to the Control UI WebSocket client, with raw auth and device fields summarized or redacted before copy/export.
