## Summary

Keep open: current main still has the raw chat-buffer cleanup mismatch described in the report. The stale session-list CPU side is partly mitigated on main by bounded listing and yielding, but the raw-buffer leak has a clear source-level fix path.

## What This Changes

The issue reports gateway memory retention from orphaned chat raw buffers and CPU stalls from stale session index growth.

## Best Possible Solution

Thread the raw chat buffer map through the existing gateway runtime, abort, request-context, and maintenance cleanup contracts, then add regression tests that assert every run-keyed chat buffer map is cleared together.

## Reproduction Assessment

Yes for the raw-buffer leak from source inspection: current main writes unprojected stream text to `rawBuffers` but abort and stale maintenance cleanup only receive `chatRunBuffers`. I did not live-reproduce the reported heap/RPC stall behavior.

## Solution Assessment

Yes for the raw-buffer part: extending the existing cleanup invariant to include `rawBuffers` is the narrowest maintainable fix. Broader stale active-session pruning should be measured and handled separately if existing `sessions.cleanup` and bounded `sessions.list` behavior are not enough.

## Repair Work Prompt

Fix the gateway chat run raw-buffer leak reported in https://github.com/openclaw/openclaw/issues/80282. On current main, `createChatRunState()` has both `rawBuffers` and projected `buffers`; `server-chat.ts` writes both and normal finalization clears both, but `chat-abort.ts`, `server-maintenance.ts`, `server-runtime-state.ts`, `server.impl.ts`, `server-request-context.ts`, `server-methods/shared-types.ts`, and `server-methods/chat.ts` only thread and clear `chatRunBuffers`. Add a typed `chatRunRawBuffers` or equivalent runtime/request/abort/maintenance contract and delete raw buffers everywhere stale, aborted, or timed-out run buffers are deleted. Add focused regression tests that fail before the fix, especially maintenance stale orphan cleanup, aborted-run aging, and `abortChatRunById()` cleanup. Do not add a new config option, default pruning policy, or broad session-store migration in this PR; if stale session-index cleanup needs behavior beyond existing `sessions.cleanup` and bounded `sessions.list`, leave it for a follow-up. Add a changelog entry because this is a user-visible gateway stability fix.
