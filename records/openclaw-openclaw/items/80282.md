## Summary

Keep open: current main still has the raw-buffer cleanup gap reported here. The gateway writes unprojected assistant stream text into `rawBuffers`, but abort and stale-run maintenance cleanup only receive and delete the projected `chatRunBuffers`/delta maps.

## What This Changes

The issue reports gateway heap/CPU growth from orphaned raw chat buffers and stale active session entries during long-lived chat sessions.

## Best Possible Solution

Patch the gateway cleanup contract so every run-keyed chat map, including `rawBuffers`, is cleared on abort, timeout, aged-aborted cleanup, orphaned stale cleanup, and clean finalization, with focused regression tests.

## Reproduction Assessment

Yes at source level: seed a stale or aborted run with both `rawBuffers` and `buffers`, then exercise maintenance or abort cleanup; current main has cleanup handles for `buffers` and delta maps but no handle for `rawBuffers`. I did not run a live long-lived gateway repro in this read-only review.

## Solution Assessment

Yes: the narrow maintainable fix is to thread the raw-buffer map through the existing cleanup contracts and delete it wherever projected run buffers are deleted. Broader session-index policy should remain separate unless a focused current-main repro proves it is still needed after the cleanup fix.
