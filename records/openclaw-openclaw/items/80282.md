## Summary

Keep open: current main still has the raw chat-buffer cleanup mismatch described in the report. There is already a focused open fix PR at https://github.com/openclaw/openclaw/pull/75089, but it has not merged, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue reports gateway memory and CPU growth from orphaned raw chat stream buffers and stale active session index entries.

## Best Possible Solution

Land or replace the focused raw-buffer cleanup in https://github.com/openclaw/openclaw/pull/75089, then handle broader stale-session pruning only with separate measurement and policy review.

## Reproduction Assessment

Yes for the raw-buffer leak from source inspection: current main writes unprojected stream text to rawBuffers while abort and maintenance cleanup paths only receive chatRunBuffers. The reported heap/RPC stall was not live-reproduced in this read-only pass.

## Solution Assessment

Yes for the raw-buffer part: centralizing per-run cleanup or threading rawBuffers through the existing cleanup contracts is the narrow maintainable fix. The stale session-index cleanup side should remain separate unless maintainers want to change pruning behavior.
