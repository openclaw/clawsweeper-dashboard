## Summary

Keep open. Current `main` still ignores `agents.defaults.agentRuntime`, and three open PRs reference this issue with closing syntax, so the issue should stay paired with the active fix discussion until one PR merges or maintainers decide to keep the provider/model-only runtime contract.

## What This Changes

The issue requests that subagent runtime resolution inherit `agents.defaults.agentRuntime` so Claude subagents use `claude-cli` instead of falling back to embedded Anthropic HTTP.

## Best Possible Solution

Review the linked fix PRs against the current provider/model runtime-policy contract, then either merge one deliberate compatibility fix with regression coverage or close the PRs and improve doctor/startup guidance for legacy `agents.defaults.agentRuntime` configs.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence current-main path: set only `agents.defaults.agentRuntime`, resolve an Anthropic harness policy, and the resolver falls through with no configured runtime because it only reads model/provider runtime policy.

## Solution Assessment
