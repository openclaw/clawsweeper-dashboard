## Summary

Keep open. Current main still ignores `agents.defaults.agentRuntime` under the documented provider/model runtime contract, but three open implementation PRs already close this issue, so the next step is maintainer review rather than cleanup closure.

## What This Changes

Request that spawned subagents inherit `agents.defaults.agentRuntime`, especially `claude-cli`, and warn before falling back to embedded HTTP routing.

## Best Possible Solution

Pick one supported runtime contract: land a canonical PR that intentionally restores default-level runtime inheritance with tests/docs/doctor updates, or keep provider/model-only runtime policy and improve diagnostics for legacy config.

## Reproduction Assessment

Yes, by source inspection. Current main ignores `agents.defaults.agentRuntime` in the runtime resolver and the checked-in harness test asserts that behavior, though I did not run the live Claude billing flow.

## Solution Assessment

Unclear. Inheriting the legacy whole-agent runtime could solve the reported setup, but it conflicts with the current documented provider/model runtime contract, so maintainers should choose one linked PR or reaffirm the current contract with better diagnostics.
