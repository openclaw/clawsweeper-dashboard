## Summary

Keep open: current main still intentionally ignores whole-agent `agents.defaults.agentRuntime` per source, tests, and docs, while three open PRs reference this issue as their fix target. This needs maintainer review of the linked runtime-policy change rather than cleanup closure.

## What This Changes

The issue asks for subagent runtime resolution to inherit `agents.defaults.agentRuntime`, especially `claude-cli`, and to warn when that legacy default would otherwise be bypassed.

## Best Possible Solution

Keep this issue open while maintainers decide whether one of the linked PRs should intentionally change the runtime-policy contract, or close it after documenting that provider/model-scoped `agentRuntime` remains the supported path.

## Reproduction Assessment

Yes, from source inspection: current main does not inherit `agents.defaults.agentRuntime`, and an existing harness test asserts that legacy whole-agent runtime is ignored. I did not run a live subagent because this read-only review could not create runtime state and the selection path is directly visible in source.

## Solution Assessment

Unclear as proposed: inheriting `agents.defaults.agentRuntime` is not the narrowest bug fix under the current provider/model runtime contract, but it may be a valid product change if maintainers choose to reverse or soften that legacy behavior.
