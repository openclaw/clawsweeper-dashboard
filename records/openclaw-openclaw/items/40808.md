## Summary

This should stay open for maintainer review, not cleanup closure: current `main` does not implement `sessions_send_concurrent`, and the PR still introduces a new core tool with source-reproducible correctness and security-sensitive contract drift from current `sessions_send`.

## What This Changes

The PR adds a core `sessions_send_concurrent` tool with A2A support, docs, tests, tool registration, and gateway Nested-lane concurrency changes.

## Best Possible Solution

If maintainers want multi-target sending, extract or reuse current `sessions_send` internals so validation, provenance, run-wait, per-session lanes, and A2A guards stay single-source, then require focused regression coverage and redacted live proof.

## Reproduction Assessment

Yes for the PR blockers: source comparison of PR head against current main shows stale package imports plus mismatched validation, lane, provenance, wait, and A2A contracts. This is not a current-main bug reproduction because the PR proposes a new core tool.

## Solution Assessment

No. The PR forks current `sessions_send` behavior and changes shared lane policy; the narrower maintainable solution is a wrapper or extraction over current single-send internals while preserving per-session isolation.

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Import tool types from the current packages:** `src/agents/tools/sessions-send-concurrent-tool.ts:2-3`
  - body: Current main depends on `@earendil-works/pi-agent-core` and `typebox`, but this new file imports `@mariozechner/pi-agent-core` and `@sinclair/typebox`. After rebase those packages are not declared, so the tool will not resolve until it uses the current package names.
  - confidence: 0.93
- **[P2] Keep the shared nested lane serial:** `src/gateway/server-lanes.ts:10`
  - body: Current main keeps `CommandLane.Nested` at default serial concurrency and uses session-scoped nested lanes for safe parallelism. Widening the shared lane lets unscoped nested work interleave and contradicts the current lane contract.
  - confidence: 0.9
- **[P2] Use per-session lanes for each target run:** `src/agents/tools/sessions-send-concurrent-tool.ts:364`
  - body: Every target dispatch uses `AGENT_LANE_NESTED`, so two sends to the same target can run concurrently. Current `sessions_send` dispatches through `resolveNestedAgentLaneForSession(resolvedKey)` to preserve same-session serialization.
  - confidence: 0.92
- **[P2] Annotate inter-session messages before dispatch:** `src/agents/tools/sessions-send-concurrent-tool.ts:359`
  - body: The primary dispatch sends `target.message` directly. Current inter-session sends wrap the text with `annotateInterSessionPromptText`, so the target agent receives the prompt-level warning that this content came from another session/tool rather than a direct user instruction.
  - confidence: 0.88
- **[P2] Read replies with the run-wait helper:** `src/agents/tools/sessions-send-concurrent-tool.ts:520-528`
  - body: After `agent.wait`, this reads the latest assistant message from `chat.history`; if no fresh reply is produced or another message lands first, it can return a stale or wrong reply. Use the current baseline-aware `waitForAgentRunAndReadUpdatedAssistantReply` path.
  - confidence: 0.88
- **[P2] Reject targets that set both sessionKey and label:** `src/agents/tools/sessions-send-concurrent-tool.ts:131-134`
  - body: The parser stores both fields and later silently prefers `sessionKey`. Current `sessions_send` rejects that ambiguous input, so generated bulk payloads can send to the wrong session without a validation error.
  - confidence: 0.87
- **[P2] Reject thread-session targets before dispatch:** `src/agents/tools/sessions-send-concurrent-tool.ts:335-336`
  - body: After resolving the visible session, this proceeds to dispatch without the thread-session guard current `sessions_send` applies. Bulk inter-agent sends can therefore target a thread session that the single-send tool blocks.
  - confidence: 0.84
- **[P2] Skip A2A for parent-owned child sessions:** `src/agents/tools/sessions-send-concurrent-tool.ts:547-559`
  - body: Successful sends always start the concurrent A2A helper. Current `sessions_send` skips follow-up for parent-owned ACP/native child sessions because another result path already reports the reply; omitting that guard can duplicate output or start reply loops.
  - confidence: 0.86
- **[P2] Filter non-deliverable replies during A2A:** `src/agents/tools/sessions-send-concurrent-a2a.ts:184`
  - body: The copied A2A flow only checks `REPLY_SKIP` and `ANNOUNCE_SKIP`. Current `sessions_send` also suppresses non-deliverable control replies before ping-pong and announce, so this helper can feed or post control replies that existing behavior filters out.
  - confidence: 0.82
