Overall confidence: 0.95

Full review comments:

- **[P1] Import Type from the current typebox package:** `src/agents/tools/sessions-send-concurrent-tool.ts:3`
  - body: Current main declares `typebox` and existing core tools import `Type` from that package. This new import uses undeclared `@sinclair/typebox`, so a rebased branch will fail dependency resolution.
  - confidence: 0.96
- **[P1] Gate the new send tool from HTTP invocation:** `src/agents/tool-catalog.ts:141-147`
  - body: This registers a new cross-session message-injection tool, but the existing `/tools/invoke` deny list only blocks `sessions_send`. Add this tool to the same high-risk gate and tests before exposing it.
  - confidence: 0.9
- **[P2] Annotate forwarded inter-session prompts:** `src/agents/tools/sessions-send-concurrent-tool.ts:359`
  - body: The primary send passes `target.message` directly even though the metadata says this is inter-session data. Use the current `sessions_send` prompt annotation so the target model does not treat another agent's text as direct user input.
  - confidence: 0.94
- **[P2] Use per-session nested lanes for fan-out:** `src/agents/tools/sessions-send-concurrent-tool.ts:364`
  - body: All primary sends use the shared nested lane. Current main scopes nested agent work by target session, so each target should use `resolveNestedAgentLaneForSession(resolvedKey)` instead of the bare shared lane.
  - confidence: 0.93
- **[P2] Keep the shared nested lane at default concurrency:** `src/gateway/server-lanes.ts:10`
  - body: Current tests intentionally keep `CommandLane.Nested` serial while per-session nested lanes provide parallelism. Widening the shared lane globally changes unrelated nested workflows and is not needed for concurrent fan-out.
  - confidence: 0.92
- **[P2] Reject ambiguous target identity fields:** `src/agents/tools/sessions-send-concurrent-tool.ts:131-134`
  - body: The parser copies both `sessionKey` and `label` and later silently prefers `sessionKey`. Current `sessions_send` rejects this ambiguous input, so generated calls can otherwise be misrouted without a validation error.
  - confidence: 0.92
- **[P2] Reject thread-scoped targets before dispatch:** `src/agents/tools/sessions-send-concurrent-tool.ts:335-338`
  - body: After resolving the visible session, this path dispatches without the current `sessions_send` thread-session guard. That allows inter-agent coordination messages to be injected into active human-facing thread sessions.
  - confidence: 0.9
- **[P2] Keep delayed replies alive after soft wait timeouts:** `src/agents/tools/sessions-send-concurrent-tool.ts:493-502`
  - body: Current `sessions_send` treats a non-terminal wait timeout as accepted work and starts background A2A so a late target reply can still reach the requester. This branch returns `timeout` immediately and drops the follow-up flow.
  - confidence: 0.9
- **[P2] Read only the reply produced by this run:** `src/agents/tools/sessions-send-concurrent-tool.ts:520-528`
  - body: After `agent.wait`, this reads the last assistant message from the whole session history. Current `sessions_send` captures a pre-run baseline and only returns a newer assistant reply, avoiding stale or nearby unrelated replies.
  - confidence: 0.9
- **[P2] Suppress non-deliverable A2A control replies:** `src/agents/tools/sessions-send-concurrent-a2a.ts:184`
  - body: The concurrent A2A loop only checks `REPLY_SKIP`, while current A2A also suppresses non-deliverable control replies such as `NO_REPLY` and `HEARTBEAT_OK`. Those controls can otherwise be reinjected or delivered outward.
  - confidence: 0.86
- **[P2] Keep announce bookkeeping out of transcripts:** `src/agents/tools/sessions-send-concurrent-a2a.ts:231-240`
  - body: Current `sessions_send` passes `transcriptMessage: ""` for the announce step so the internal announce prompt is not recorded as a visible user turn. This announce call omits that guard.
  - confidence: 0.84
- **[P2] Preserve parent-owned subagent A2A skip behavior:** `src/agents/tools/sessions-send-concurrent-tool.ts:547-559`
  - body: Current `sessions_send` skips A2A when the requester is the parent of a child session that already has a parent-visible result path. This branch always starts A2A, which can duplicate delivery or create bounded ping-pong loops.
  - confidence: 0.82

## Security Review

Status: needs_attention
