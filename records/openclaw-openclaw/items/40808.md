Full review comments:

- **[P1] Import Type from the declared typebox package:** `src/agents/tools/sessions-send-concurrent-tool.ts:3`
  - body: The new file imports `Type` from `@sinclair/typebox`, but current package metadata declares `typebox` and existing source imports from `typebox`. On a strict install this module specifier can fail resolution before the tool runs.
  - confidence: 0.93
- **[P2] Use per-session nested lanes for target runs:** `src/agents/tools/sessions-send-concurrent-tool.ts:364`
  - body: This enqueues every target on the shared `AGENT_LANE_NESTED` lane while the PR also widens that lane globally. Current `sessions_send` uses `resolveNestedAgentLaneForSession(resolvedKey)` so independent sessions do not bypass the per-session lane contract.
  - confidence: 0.9
- **[P2] Use per-session lanes for A2A turns:** `src/agents/tools/sessions-send-concurrent-a2a.ts:176`
  - body: The A2A helper runs ping-pong and announce turns on the shared nested lane. Current `sessions_send` scopes those turns with `resolveNestedAgentLaneForSession(...)`, so this can block unrelated targets and bypass the isolation contract.
  - confidence: 0.89
- **[P2] Read replies through the run-wait helper:** `src/agents/tools/sessions-send-concurrent-tool.ts:520-528`
  - body: After `agent.wait`, this reads the latest assistant message from session history without a pre-run baseline. If an older or concurrent assistant reply is still last, the concurrent tool can return stale text that current `sessions_send` filters out.
  - confidence: 0.88
- **[P2] Reject targets with both sessionKey and label:** `src/agents/tools/sessions-send-concurrent-tool.ts:182`
  - body: Because label resolution only runs when `sessionKey` is absent, a target that supplies both fields silently ignores `label`/`agentId`. Current `sessions_send` rejects that ambiguous input to avoid sending to the wrong session.
  - confidence: 0.87
- **[P2] Mirror the parent-owned child A2A skip guard:** `src/agents/tools/sessions-send-concurrent-tool.ts:547`
  - body: Current `sessions_send` skips A2A follow-up when the requester owns the target ACP/native child session, preventing duplicate output and parent/child echo loops. This starts the concurrent A2A helper unconditionally after successful sends.
  - confidence: 0.85
- **[P2] Filter non-deliverable A2A replies:** `src/agents/tools/sessions-send-concurrent-a2a.ts:184`
  - body: The copied A2A flow only checks `REPLY_SKIP` and `ANNOUNCE_SKIP`. Current `sessions_send` also suppresses silent and heartbeat tokens before ping-pong and announce delivery, so this can feed or post control text.
  - confidence: 0.8
- **[P2] Reject thread-session targets before dispatch:** `src/agents/tools/sessions-send-concurrent-tool.ts:335-336`
  - body: Current `sessions_send` rejects direct or label-resolved thread session keys before starting a run. This path proceeds after resolving the visible session, so bulk sends can target thread sessions despite the single-target tool's guard.
  - confidence: 0.82
