## Summary

Keep open: current `main` does not contain the proposed concurrent send tool, but the PR still has blocking correctness, security, and proof gaps; it needs maintainer architecture review rather than cleanup closure.

## What This Changes

The branch adds a core `sessions_send_concurrent` tool with docs, tests, catalog registration, and gateway Nested-lane concurrency changes for multi-target session messaging.

## Best Possible Solution

If multi-target sending belongs in core, extract or reuse `sessions_send` dispatch/A2A helpers so validation, lane isolation, wait semantics, ownership guards, tests, and docs remain single-source.

## Reproduction Assessment

Yes for the PR blockers: source comparison of PR head `2b2d220d5d333823ef05c439f5560d11b88d8c48` with current main shows unresolved import/package drift and behavior divergences in lane isolation, target validation, reply reading, and A2A guards. This is not a current-main bug reproduction because the PR proposes a new feature.

## Solution Assessment

No. Forking `sessions_send` and widening the shared Nested lane is not the narrowest maintainable approach; a safer solution would reuse existing single-send internals and preserve per-session lane isolation.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Import tool types from current packages:** `src/agents/tools/sessions-send-concurrent-tool.ts:2-3`
  - body: Current main declares `@earendil-works/pi-agent-core` and `typebox`, but this new tool imports the old `@mariozechner/pi-agent-core` and `@sinclair/typebox` packages. After rebasing, these modules will not resolve, so the branch cannot compile as written.
  - confidence: 0.95
- **[P2] Keep the shared Nested lane serial:** `src/gateway/server-lanes.ts:10`
  - body: Current main explicitly keeps the shared `CommandLane.Nested` lane at default serial concurrency and uses scoped nested lanes for safe parallelism. Widening the shared lane lets unscoped nested work interleave and contradicts the current gateway lane contract.
  - confidence: 0.92
- **[P2] Dispatch each target on its scoped lane:** `src/agents/tools/sessions-send-concurrent-tool.ts:364`
  - body: Every target dispatch uses `AGENT_LANE_NESTED`, so once the shared lane is widened, two sends to the same target can run concurrently. Use `resolveNestedAgentLaneForSession(resolvedKey)` so different sessions can proceed while each target session stays serialized.
  - confidence: 0.92
- **[P2] Validate target identity before resolution:** `src/agents/tools/sessions-send-concurrent-tool.ts:131-134`
  - body: The parser accepts and stores `sessionKey`, `label`, and `agentId` together, then silently prefers `sessionKey`; it also errors later for `agentId`-only targets even though the tool advertises agent IDs. Match `sessions_send` by rejecting ambiguous identity and resolving configured agent main sessions.
  - confidence: 0.88
- **[P2] Reject thread-session targets before dispatch:** `src/agents/tools/sessions-send-concurrent-tool.ts:335-336`
  - body: After resolving the visible session, this proceeds directly to `agent` without the thread-session guard that current `sessions_send` applies. Bulk inter-agent sends can therefore target thread sessions that the single-send tool blocks for coordination safety.
  - confidence: 0.84
- **[P2] Read replies through the run-wait helper:** `src/agents/tools/sessions-send-concurrent-tool.ts:520-528`
  - body: After `agent.wait`, this reads the latest assistant message from `chat.history`; if another message lands first or no fresh reply is produced, it can return stale or wrong text. Use the current baseline-aware `waitForAgentRunAndReadUpdatedAssistantReply` path.
  - confidence: 0.89
- **[P2] Skip A2A for parent-owned child sessions:** `src/agents/tools/sessions-send-concurrent-tool.ts:547-558`
  - body: Successful sends always start the concurrent A2A helper. Current `sessions_send` skips follow-up for requester-owned ACP/native child sessions because another result path already reports the reply; omitting that guard can duplicate output or start reply loops.
  - confidence: 0.87
- **[P2] Filter non-deliverable A2A replies:** `src/agents/tools/sessions-send-concurrent-a2a.ts:184`
  - body: The copied A2A flow only checks `REPLY_SKIP` and `ANNOUNCE_SKIP`. Current `sessions_send` also suppresses non-deliverable control replies before ping-pong and announce, so this helper can feed or post control replies that existing behavior filters out.
  - confidence: 0.84
- **[P2] Use scoped lanes inside A2A steps:** `src/agents/tools/sessions-send-concurrent-a2a.ts:176`
  - body: Ping-pong and announce steps use the shared `AGENT_LANE_NESTED` lane. Current A2A steps use `resolveNestedAgentLaneForSession(...)`, preserving same-session serialization while allowing different sessions to proceed independently.
  - confidence: 0.87
