Full review comments:

- **[P1] Import Type from the repo package:** `src/agents/tools/sessions-send-concurrent-tool.ts:3`
  - body: The new file imports `Type` from `@sinclair/typebox`, but current `package.json` only declares `typebox` and current source imports from `typebox`. On current main this will fail module resolution before the tool can build.
  - confidence: 0.91
- **[P2] Use per-session nested lanes for each target:** `src/agents/tools/sessions-send-concurrent-tool.ts:364`
  - body: This schedules every target on the shared nested lane. Current main keeps `CommandLane.Nested` serial and routes nested agent work through `resolveNestedAgentLaneForSession`, so this either serializes the new concurrent tool or depends on widening a shared lane that current tests intentionally keep serial.
  - confidence: 0.9
- **[P2] Use the shared wait helper before reading replies:** `src/agents/tools/sessions-send-concurrent-tool.ts:520-523`
  - body: The wait path reads the latest assistant message after `agent.wait` without a baseline or `waitForAgentRunAndReadUpdatedAssistantReply`. If the target already had an assistant message, or the run is still pending after a non-terminal timeout, the tool can return stale text or a terminal timeout where `sessions_send` would return `accepted` and let A2A continue.
  - confidence: 0.86
- **[P2] Reject targets with both sessionKey and label:** `src/agents/tools/sessions-send-concurrent-tool.ts:179-182`
  - body: Current `sessions_send` treats both identifiers as invalid. Here `sessionKey` is assigned first and label resolution only runs when it is absent, so ambiguous generated input silently ignores `label`/`agentId` and can send to the wrong session.
  - confidence: 0.87
- **[P2] Mirror the current A2A skip guard:** `src/agents/tools/sessions-send-concurrent-tool.ts:547-559`
  - body: Current `sessions_send` skips A2A follow-up for parent-owned ACP/native child sessions to avoid duplicate output and reply loops. This tool unconditionally starts `runConcurrentA2AFlowForTarget` after each successful target, so concurrent sends reintroduce that loop path for the same session types.
  - confidence: 0.82
