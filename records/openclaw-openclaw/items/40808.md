## Review Findings

Overall correctness: not a patch

Overall confidence: 0.9

Full review comments:

- **[P2] Keep nested work on per-session lanes:** `src/gateway/server-lanes.ts:10`
  - body: Current main deliberately keeps the shared `nested` lane serialized and scopes agent work with `resolveNestedAgentLaneForSession`. Setting `CommandLane.Nested` concurrency here, and in hot reload, conflicts with that contract; remove the shared-lane widening and run each target, reply, and announce step on the target session's nested lane.
  - confidence: 0.9
- **[P1] Honor the parent-owned child A2A skip:** `src/agents/tools/sessions-send-concurrent-tool.ts:404-416`
  - body: The new tool always starts `runConcurrentA2AFlowForTarget`, but current `sessions_send` skips A2A for requester-owned ACP/native child sessions because the normal completion path already reports results and the peer loop can wake the parent and feed a reply back to the child. Add the same target-entry and requester-ownership guard before both fire-and-forget and waited A2A starts.
  - confidence: 0.88
- **[P2] Read only a reply newer than the baseline:** `src/agents/tools/sessions-send-concurrent-tool.ts:520-528`
  - body: After `agent.wait`, this reads the latest assistant message for the whole session. If no new reply was produced, or another concurrent send to that session wins the race, the tool can return and announce a stale or wrong reply; snapshot the pre-run reply and use the existing baseline-aware wait/read helper.
  - confidence: 0.88
- **[P2] Reject ambiguous target selectors:** `src/agents/tools/sessions-send-concurrent-tool.ts:131-134`
  - body: Targets that include both `sessionKey` and `label` silently prefer `sessionKey` and ignore `label`/`agentId`. `sessions_send` rejects that combination, and this tool should keep the same validation to avoid generated payloads being routed to the wrong session without an error.
  - confidence: 0.84
- **[P2] Reject thread-session targets:** `src/agents/tools/sessions-send-concurrent-tool.ts:335-336`
  - body: Current `sessions_send` rejects thread sessions for inter-agent coordination and asks callers to use the parent channel session key. This tool resolves a visible session and then dispatches to it without that guard, so thread targets can enter A2A flows that the canonical tool forbids.
  - confidence: 0.8
- **[P2] Suppress non-deliverable control replies:** `src/agents/tools/sessions-send-concurrent-a2a.ts:243`
  - body: The concurrent A2A helper checks only `REPLY_SKIP` and `ANNOUNCE_SKIP`, while current `sessions_send` also suppresses `NO_REPLY` and heartbeat control replies before continuing ping-pong or sending to a channel. Reuse `isNonDeliverableSessionsReply` in the same three places as the canonical A2A helper.
  - confidence: 0.82
