## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.95

Full review comments:

- **[P1] Import Type from the declared typebox package:** `src/agents/tools/sessions-send-concurrent-tool.ts:3`
  - body: Current main declares and imports `typebox`, not `@sinclair/typebox`. This new undeclared import will fail dependency/type resolution when the branch is rebased into the current package graph.
  - confidence: 0.96
- **[P1] Gate the concurrent send tool on the HTTP deny list:** `src/agents/tool-catalog.ts:141-146`
  - body: This registers another cross-session message-injection tool, but the default `/tools/invoke` hard deny list only blocks `sessions_send`. A gateway surface that intentionally rejects `sessions_send` could expose `sessions_send_concurrent` instead.
  - confidence: 0.9
- **[P2] Preserve inter-session prompt provenance:** `src/agents/tools/sessions-send-concurrent-tool.ts:359`
  - body: The primary dispatch sends `target.message` directly even though current `sessions_send` marks forwarded text with the inter-session prompt marker before it reaches the target model. Without that marker, another agent's text can look like direct user input.
  - confidence: 0.94
- **[P2] Use per-session nested lanes for target fan-out:** `src/agents/tools/sessions-send-concurrent-tool.ts:364`
  - body: All primary sends use the shared nested lane. Current main scopes nested agent work by target session, so concurrent fan-out should use the target's per-session nested lane instead of relying on a global shared-lane widening.
  - confidence: 0.93
- **[P2] Keep the shared nested lane at default concurrency:** `src/gateway/server-lanes.ts:10`
  - body: Current `server-lanes.test.ts` explicitly preserves default concurrency for `CommandLane.Nested`. Widening the shared lane changes unrelated nested workflows and is not needed once target sends use per-session nested lanes.
  - confidence: 0.92
- **[P2] Reject targets that provide both sessionKey and label:** `src/agents/tools/sessions-send-concurrent-tool.ts:131-134`
  - body: The parser copies both fields and later silently prefers `sessionKey`. Current `sessions_send` rejects that ambiguous input, so generated calls can be misrouted without a validation error.
  - confidence: 0.92
- **[P2] Block thread-scoped targets before dispatch:** `src/agents/tools/sessions-send-concurrent-tool.ts:335-359`
  - body: After resolving visibility, this path dispatches to the resolved key without the current `sessions_send` guard for thread-scoped sessions. That allows inter-agent coordination messages to be injected into active human-facing thread sessions.
  - confidence: 0.9
- **[P2] Keep delayed replies alive after soft wait timeouts:** `src/agents/tools/sessions-send-concurrent-tool.ts:493-502`
  - body: On timeout this returns `timeout` immediately and never starts the follow-up A2A flow. Current `sessions_send` treats non-terminal wait timeouts as accepted work so a late target reply can still reach the requester.
  - confidence: 0.9
- **[P2] Read only the reply produced by this run:** `src/agents/tools/sessions-send-concurrent-tool.ts:520-528`
  - body: After `agent.wait`, this reads the last assistant message from the whole session history. Current `sessions_send` captures a pre-run baseline and only returns a newer assistant reply, avoiding stale or nearby unrelated replies.
  - confidence: 0.9
- **[P2] Suppress non-deliverable A2A control replies:** `src/agents/tools/sessions-send-concurrent-a2a.ts:183-184`
  - body: The concurrent A2A loop only checks `REPLY_SKIP`, while current A2A also suppresses non-deliverable control replies such as silent or heartbeat tokens. Those control replies can otherwise be reinjected or delivered outward.
  - confidence: 0.86
- **[P2] Keep announce bookkeeping out of transcripts:** `src/agents/tools/sessions-send-concurrent-a2a.ts:231-240`
  - body: Current `sessions_send` passes an empty transcript message for the announce step so the internal announce prompt is not recorded as a visible user turn. This announce call omits that guard, so A2A bookkeeping can leak into the target transcript.
  - confidence: 0.84

## Security Review

Status: needs_attention

Summary: The diff introduces a new cross-session message-injection tool without preserving all current provenance and high-risk HTTP/tool-policy gates.

Concerns:

- **[high] Gate the new cross-session tool like sessions_send:** `src/agents/tool-catalog.ts:141`
  - body: `sessions_send` is denied by default over `/tools/invoke` because it injects messages across sessions. Registering `sessions_send_concurrent` without adding it to the same deny list and policy coverage creates a bypass for the existing restriction.
  - confidence: 0.9
- **[medium] Preserve inter-session prompt provenance:** `src/agents/tools/sessions-send-concurrent-tool.ts:359`
  - body: The PR sends raw target messages to another session instead of using the current inter-session prompt annotation, so another agent's output can be treated by the target model as direct user-authored instruction.
  - confidence: 0.9
- **[medium] Reject thread-scoped send targets:** `src/agents/tools/sessions-send-concurrent-tool.ts:335`
  - body: The current single-send tool blocks thread-scoped targets to avoid placing tool-routed coordination messages into active human-facing threads; the concurrent path lacks that guard before dispatch.
  - confidence: 0.86
