Full review comments:

- **[P1] Import current tool packages:** `src/agents/tools/sessions-send-concurrent-tool.ts:2-3`
  - body: Current main depends on `@earendil-works/pi-agent-core` and imports schemas from `typebox`; this new tool imports the old package names. After rebase, these modules will not resolve, so the branch cannot compile as written.
  - confidence: 0.96
- **[P2] Keep the shared Nested lane serial:** `src/gateway/server-lanes.ts:10`
  - body: Current main keeps the shared `CommandLane.Nested` lane at default serial concurrency and uses scoped nested lanes for safe parallelism. Widening the shared lane lets unscoped nested work interleave and contradicts the gateway lane contract now covered by `server-lanes.test.ts`.
  - confidence: 0.93
- **[P2] Dispatch each target on a scoped lane:** `src/agents/tools/sessions-send-concurrent-tool.ts:364`
  - body: Every target dispatch uses `AGENT_LANE_NESTED`, so once the shared lane is widened, two sends to the same target can run concurrently. Use the current `resolveNestedAgentLaneForSession(resolvedKey)` pattern so different sessions can proceed while each target session stays serialized.
  - confidence: 0.93
- **[P2] Reject ambiguous target identities:** `src/agents/tools/sessions-send-concurrent-tool.ts:131-134`
  - body: Targets may provide `sessionKey`, `label`, and `agentId` together, after which the tool silently prefers `sessionKey`. Current `sessions_send` rejects `sessionKey` plus `label`, so generated bulk payloads can send to the wrong session without an error.
  - confidence: 0.88
- **[P2] Reject thread-session targets before dispatch:** `src/agents/tools/sessions-send-concurrent-tool.ts:335-336`
  - body: After resolving the visible session, the new tool proceeds toward `agent` without the thread-session guard that current `sessions_send` applies. Bulk inter-agent sends can therefore target thread sessions that the single-send tool blocks for coordination safety.
  - confidence: 0.86
- **[P2] Read replies through the run-wait helper:** `src/agents/tools/sessions-send-concurrent-tool.ts:520-528`
  - body: After `agent.wait`, this reads the latest assistant message from `chat.history`; if another message lands first or no fresh reply is produced, it can return stale or wrong text. Use the current baseline-aware `waitForAgentRunAndReadUpdatedAssistantReply` path.
  - confidence: 0.91
- **[P2] Preserve parent-owned child A2A skips:** `src/agents/tools/sessions-send-concurrent-tool.ts:404-416`
  - body: The new tool starts the A2A flow for every accepted target, but current `sessions_send` skips parent-owned ACP/native child flows because another parent-visible result path already exists. Without that guard, concurrent sends can wake the parent and produce duplicate follow-up output.
  - confidence: 0.86
- **[P2] Filter non-deliverable A2A replies:** `src/agents/tools/sessions-send-concurrent-a2a.ts:184`
  - body: Current `sessions_send` stops on non-deliverable session control replies before ping-pong or announce. This helper only checks `REPLY_SKIP`, so internal control text can be forwarded through the concurrent A2A flow.
  - confidence: 0.84
