## Summary

Keep this PR open, but it is not merge-ready. Current main does not implement the new `sessions_send_concurrent` tool, so the PR is not obsolete; however, the diff forks sensitive `sessions_send` behavior and conflicts with current main’s per-session nested-lane, reply-correlation, ACP-boundary, provenance, and dependency contracts.

## What This Changes

The PR adds a built-in `sessions_send_concurrent` tool with docs, tests, catalog/tool registration, concurrent batch sends to 1-20 sessions, A2A follow-up handling, and gateway nested-lane concurrency changes.

## Best Possible Solution

Implement any batch-send surface as a thin orchestrator over shared `sessions_send` and run-wait helpers, preserving ambiguous-selector rejection, inter-session prompt annotation, session-scoped nested lanes, reply baseline/correlation, ACP child-session A2A guards, and focused regression coverage before adding public docs.

## Reproduction Assessment

Yes. The blocking findings are reproducible by comparing the latest PR diff against current main’s `sessions_send`, A2A, run-wait, lane, ACP, provenance, dependency, and regression-test contracts; live runtime reproduction was not attempted in this read-only review.

## Solution Assessment

No. The proposed capability is useful, but forking the sensitive `sessions_send` path and widening the shared nested lane is not the narrowest maintainable solution; shared helpers or an extracted batch orchestrator would be safer.

Full review comments:

- **[P2] Use the repo's typebox package:** `src/agents/tools/sessions-send-concurrent-tool.ts:3`
  - body: This new file imports `Type` from `@sinclair/typebox`, but current main depends on and imports the direct `typebox` package. A clean install/build can fail to resolve this module unless the import follows the repo dependency.
  - confidence: 0.92
- **[P2] Reject ambiguous target selectors:** `src/agents/tools/sessions-send-concurrent-tool.ts:120-123`
  - body: A target can provide both `sessionKey` and `label`; this parser keeps both and later silently prefers `sessionKey`. `sessions_send` rejects that combination so generated payloads do not route to the wrong session without an error.
  - confidence: 0.91
- **[P2] Preserve inter-session prompt annotation:** `src/agents/tools/sessions-send-concurrent-tool.ts:333`
  - body: The target run sends the raw message while only attaching provenance metadata. Current `sessions_send` also prefixes inter-session text so the target model sees that the content came from another session, which this new path drops.
  - confidence: 0.86
- **[P2] Use session-scoped lanes for target sends:** `src/agents/tools/sessions-send-concurrent-tool.ts:338`
  - body: The primary target run is enqueued on the bare nested lane. Current `sessions_send` uses `resolveNestedAgentLaneForSession(resolvedKey)` so different target sessions can proceed independently while repeated sends to the same session stay ordered.
  - confidence: 0.9
- **[P2] Keep the shared nested lane serialized:** `src/gateway/server-lanes.ts:10`
  - body: Raising `CommandLane.Nested` concurrency conflicts with current main's per-session nested-lane design and its regression test that the shared bare lane stays at default concurrency. The batch path should use session-scoped lanes instead of widening the shared lane.
  - confidence: 0.88
- **[P2] Use session-scoped lanes for A2A steps:** `src/agents/tools/sessions-send-concurrent-a2a.ts:158`
  - body: The concurrent A2A helper passes `AGENT_LANE_NESTED` to `runAgentStep`, overriding that helper's current session-scoped default. Ping-pong and announce work should run on the active session's nested lane.
  - confidence: 0.88
- **[P2] Correlate replies to the waited run:** `src/agents/tools/sessions-send-concurrent-tool.ts:489-490`
  - body: After `agent.wait`, the code reads the latest assistant history item without a baseline or run correlation. If the session already had a reply or another turn finishes nearby, the tool and A2A announce can use unrelated text.
  - confidence: 0.86
- **[P2] Preserve the ACP child A2A guard:** `src/agents/tools/sessions-send-concurrent-tool.ts:373`
  - body: Current `sessions_send` skips A2A follow-up when the requester owns a background ACP child session. This tool starts follow-up unconditionally, which can re-enter the parent/child completion path and produce a visible loop.
  - confidence: 0.84
