## Summary

Keep this PR open, but do not merge it as-is. The earlier review discussion correctly caught and the branch later fixed the initial A2A fire-and-forget wait bug, but the current head still conflicts with current main’s sessions_send, typebox, nested-lane, reply-baseline, and changelog contracts.

## What This Changes

This PR adds a new sessions_send_concurrent tool with docs/tests/tool registration and changes gateway nested-lane concurrency for concurrent A2A session sends.

## Best Possible Solution

Rebase the feature onto current main and implement it as a thin concurrent wrapper around the current sessions_send run, wait, visibility, thread-target, reply-baseline, and per-session lane helpers, with focused regression tests, docs, and a changelog entry.

Yes. Source inspection is enough: the PR head and current main show a deterministic dependency mismatch and direct contract conflicts with sessions_send and server-lanes tests.

## Solution Assessment

No. The feature direction may be useful, but this patch forks stale sessions_send behavior and changes global nested-lane semantics instead of reusing current main’s narrower helper contracts.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Import Type from typebox:** `src/agents/tools/sessions-send-concurrent-tool.ts:3`
  - body: Current main has migrated tool schemas to `typebox` and no longer declares `@sinclair/typebox` in `package.json`. This new source file will fail type resolution after the branch is rebased or merged; import `Type` from `typebox` like the existing tools.
  - confidence: 0.95
- **[P2] Keep nested concurrency scoped per session:** `src/gateway/server-lanes.ts:10`
  - body: Current main deliberately keeps the shared `CommandLane.Nested` lane at its default single concurrency and uses per-session nested lanes for agent work. Widening the shared lane changes unrelated nested workflows and contradicts `server-lanes.test.ts`; route each target through `resolveNestedAgentLaneForSession` instead.
  - confidence: 0.9
- **[P2] Reject ambiguous session targets:** `src/agents/tools/sessions-send-concurrent-tool.ts:182`
  - body: The docs say `sessionKey` and `label` are mutually exclusive, and current `sessions_send` rejects that ambiguous input. This implementation only resolves labels when `sessionKey` is absent, so a payload with both fields silently ignores `label` and can send to the wrong session.
  - confidence: 0.93
- **[P2] Reject thread session targets before sending:** `src/agents/tools/sessions-send-concurrent-tool.ts:357-365`
  - body: Current `sessions_send` rejects session keys that include thread info because thread sessions are not valid inter-agent coordination targets. The concurrent tool resolves visibility and then sends without that guard, reopening a target class that main now blocks.
  - confidence: 0.88
- **[P2] Read the reply produced by the waited run:** `src/agents/tools/sessions-send-concurrent-tool.ts:520-528`
  - body: After `agent.wait` succeeds, this code reads the latest assistant message from the whole session history. Current `sessions_send` captures a pre-run baseline and only returns a newer assistant reply, which avoids stale reply carryover when the run produces no assistant text or another message lands nearby.
  - confidence: 0.86
- **[P3] Add the required changelog entry:** `src/agents/openclaw-tools.ts:171`
  - body: This registers a new user-facing tool, so repo policy requires a `CHANGELOG.md` entry for the feature. Please add one under the active release section before this PR is ready to merge.
  - confidence: 0.9
