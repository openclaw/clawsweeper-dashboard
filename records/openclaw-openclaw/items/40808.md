## Summary

Keep open: current main still lacks the requested tool, but this external feature PR is not merge-ready because the branch adds a new core inter-session messaging surface with source-reproducible contract mismatches, missing real behavior proof, and a maintainer product/architecture decision still needed.

## What This Changes

Adds a core `sessions_send_concurrent` tool with docs, tests, tool registration, an A2A helper, and gateway Nested-lane concurrency changes for multi-session sends.

## Best Possible Solution

If maintainers want this core capability, rebase it as a thin multi-target wrapper over current `sessions_send` start/wait/A2A helpers, preserving per-session nested lanes, child-session A2A guards, provenance, and focused regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction for the feature request. The merge blockers are source-reproducible by comparing the PR head with current main's `sessions_send`, run-wait, A2A, dependency, and nested-lane contracts.

## Solution Assessment

No. The useful capability should reuse or extend current `sessions_send` internals and per-session nested lanes instead of copying stale wait/A2A logic and widening the shared Nested lane.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Import Type from the repo package:** `src/agents/tools/sessions-send-concurrent-tool.ts:3`
  - body: The new file imports `Type` from `@sinclair/typebox`, but current `package.json` declares `typebox` and current source imports from `typebox`. This will fail module resolution on current main before the tool can build.
  - confidence: 0.91
- **[P2] Use per-session nested lanes for each target:** `src/agents/tools/sessions-send-concurrent-tool.ts:364`
  - body: This schedules target runs on the shared nested lane while the PR also widens that shared lane globally. Current main keeps `CommandLane.Nested` serial and routes nested agent work through per-session lanes, so this either conflicts with current tests or bypasses the isolation intended by `resolveNestedAgentLaneForSession`.
  - confidence: 0.9
- **[P2] Use the shared wait helper before reading replies:** `src/agents/tools/sessions-send-concurrent-tool.ts:520-523`
  - body: After `agent.wait`, this reads the latest assistant message from session history without the pre-run baseline or `waitForAgentRunAndReadUpdatedAssistantReply`. If the session already had an assistant reply, or the wait times out without a terminal result, the tool can return stale text or a terminal timeout where `sessions_send` would preserve delayed delivery.
  - confidence: 0.87
- **[P2] Reject targets with both sessionKey and label:** `src/agents/tools/sessions-send-concurrent-tool.ts:131-134`
  - body: Current `sessions_send` treats `sessionKey` plus `label` as invalid. This parser accepts both and then silently prefers `sessionKey`, which can misroute generated input that intended the label target.
  - confidence: 0.87
- **[P2] Mirror the current A2A child-session skip guard:** `src/agents/tools/sessions-send-concurrent-tool.ts:547-559`
  - body: Current `sessions_send` skips A2A follow-up for parent-owned ACP/native child sessions because the completion path already reports the result and A2A can create duplicate output or reply loops. This new tool starts the A2A helper after successful sends without that guard, reopening the loop path for the concurrent tool.
  - confidence: 0.83
