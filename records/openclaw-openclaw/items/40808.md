## Summary

Keep this PR open, but it is not ready to merge. Current main does not implement the new `sessions_send_concurrent` tool, and source inspection confirms the branch still has concrete correctness and security blockers against today’s `sessions_send` and gateway policy contracts.

## What This Changes

The PR adds a built-in `sessions_send_concurrent` tool with docs, tests, registration, concurrent fan-out, A2A handling, and gateway nested-lane concurrency changes.

## Best Possible Solution

Land a smaller rebased implementation only if maintainers want this new core tool, and make it reuse current `sessions_send` validation, provenance, run/wait, A2A, policy, and per-session lane helpers with real concurrent-session proof.

## Solution Assessment

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.95

Full review comments:

- **[P1] Import Type from the current typebox package:** `src/agents/tools/sessions-send-concurrent-tool.ts:3`
  - body: Current main depends on `typebox` and existing tools import `Type` from that package. This new import uses undeclared `@sinclair/typebox`, so the branch will fail dependency/type resolution when rebased into the current dependency graph.
  - confidence: 0.96
- **[P1] Wire the new send tool into high-risk policy gates:** `src/agents/tool-catalog.ts:137-144`
  - body: This registers a new cross-session message-injection tool, but the existing HTTP `/tools/invoke` deny list and related policy tests only block `sessions_send`. Without adding this new tool id to the same high-risk gates, a surface that intentionally rejects `sessions_send` can expose `sessions_send_concurrent` instead.
  - confidence: 0.88
- **[P2] Keep inter-session provenance on forwarded prompts:** `src/agents/tools/sessions-send-concurrent-tool.ts:359`
  - body: Current `sessions_send` annotates forwarded text with the inter-session marker before it reaches the target model. Sending `target.message` directly makes another agent's text look like ordinary user input even though metadata says it is inter-session data.
  - confidence: 0.94
- **[P2] Use per-session nested lanes for target fan-out:** `src/agents/tools/sessions-send-concurrent-tool.ts:364`
  - body: All primary sends use the shared nested lane. Current main scopes nested agent work by target session, so concurrent fan-out should use `resolveNestedAgentLaneForSession(resolvedKey)` instead of sending every target through the bare shared lane.
  - confidence: 0.93
- **[P2] Do not widen the shared nested lane globally:** `src/gateway/server-lanes.ts:10`
  - body: Current `server-lanes.test.ts` explicitly keeps `CommandLane.Nested` at default concurrency. Setting the shared lane to agent max concurrency changes unrelated nested workflows and is not needed once the tool uses per-session nested lanes.
  - confidence: 0.92
- **[P2] Reject targets that provide both sessionKey and label:** `src/agents/tools/sessions-send-concurrent-tool.ts:132-134`
  - body: The target parser copies both fields into the target object and later silently prefers `sessionKey`. Current `sessions_send` rejects this ambiguous input, so generated calls can be misrouted without a validation error.
  - confidence: 0.92
- **[P2] Block thread-scoped targets before dispatch:** `src/agents/tools/sessions-send-concurrent-tool.ts:335-338`
  - body: After resolving the visible session, this path dispatches without the current `sessions_send` thread-session guard. That allows inter-agent coordination messages to be injected into active human-facing thread sessions.
  - confidence: 0.9
- **[P2] Keep delayed replies alive after soft wait timeouts:** `src/agents/tools/sessions-send-concurrent-tool.ts:493-502`
  - body: Current `sessions_send` treats a non-terminal wait timeout as accepted work and starts background A2A so a late target reply can still reach the requester. This branch returns `timeout` immediately and never starts the follow-up flow.
  - confidence: 0.9
- **[P2] Read only the reply produced by this run:** `src/agents/tools/sessions-send-concurrent-tool.ts:520-528`
  - body: After `agent.wait`, this reads the last assistant message from the entire session history. Current `sessions_send` captures a pre-run baseline and only returns a newer assistant reply, avoiding stale or nearby unrelated replies.
  - confidence: 0.9
- **[P2] Suppress non-deliverable A2A control replies:** `src/agents/tools/sessions-send-concurrent-a2a.ts:184`
  - body: The concurrent A2A loop only checks `REPLY_SKIP`, while current A2A also suppresses exact non-deliverable control replies such as `NO_REPLY` and `HEARTBEAT_OK`. Those control replies can otherwise be reinjected into another agent or delivered outward.
  - confidence: 0.86
- **[P2] Keep announce bookkeeping out of transcripts:** `src/agents/tools/sessions-send-concurrent-a2a.ts:231-240`
  - body: Current `sessions_send` passes `transcriptMessage: ""` for the announce step so the internal announce prompt is not recorded as a visible user turn. This announce call omits that guard, so A2A bookkeeping can leak into the target transcript.
  - confidence: 0.84
