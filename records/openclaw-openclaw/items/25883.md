## Summary

Keep open: current main and v2026.6.6 still lack the requested Redis/BullMQ-style distributed job backend, worker claiming, dead-letter semantics, and horizontal worker coordination. Open related issues cover narrower cron and session-storage slices, so this remains broad architecture input rather than closeable cleanup.

## What This Changes

The issue proposes a Redis/BullMQ-backed horizontal-scaling architecture for background jobs, delayed work, scheduled jobs, retries, failure handling, and queue observability.

## Best Possible Solution

Define a backend-neutral scheduler/job orchestration boundary first, then decide whether implementation belongs in core, a plugin API, or documented external deployment guidance while coordinating https://github.com/openclaw/openclaw/issues/13613 and https://github.com/openclaw/openclaw/issues/63492.

## Solution Assessment

No for direct Redis/BullMQ core integration as written. The maintainable path is a maintainer-approved backend-neutral boundary that fits the existing cron, task, session, plugin, and config ownership model.
