## Summary

Keep open: current main and the latest stable release still lack the requested Redis/BullMQ-style distributed job orchestration, shared worker claiming, dead-letter semantics, and horizontal worker coordination. The open cron HA and external session-storage issues cover narrower slices, while VISION.md makes this a maintainer architecture/product decision rather than a cleanup close or automated fix.

## What This Changes

The issue requests a Redis/BullMQ-style distributed job orchestration layer for horizontally scaled OpenClaw deployments.

## Best Possible Solution

Keep this as the broad horizontal-scaling architecture tracker, preserve ScalyClaw as prior art, and have maintainers decide a backend-neutral scheduler/job boundary that coordinates with the narrower cron HA and shared session-storage issues.

## Reproduction Assessment

No strict bug reproduction is applicable; this is an architecture feature proposal. Source and release inspection give high confidence that current main and v2026.6.10 lack the requested distributed queue/backend contract.

## Solution Assessment

No for direct Redis/BullMQ core integration as written. The maintainable path is a maintainer-approved backend-neutral boundary that fits the existing cron, task, session, plugin, config, and security ownership model.
