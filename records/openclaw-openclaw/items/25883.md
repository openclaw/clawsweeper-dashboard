## Summary

Keep open: current main and v2026.6.6 still lack a distributed job backend, shared worker claiming, dead-letter semantics, and horizontal worker coordination; the issue remains broad architecture input, with narrower cron HA and session-storage slices tracked separately.

## What This Changes

Request to add Redis/BullMQ-style distributed job orchestration so OpenClaw can coordinate background work, cron, retries, workers, and observability across horizontally scaled deployments.

## Best Possible Solution

Define a backend-neutral scheduler/job orchestration boundary first, then decide whether implementation belongs in core, a plugin API, or documented external deployment guidance while coordinating the narrower cron HA and session-storage scaling issues.

## Reproduction Assessment

No bug reproduction is applicable; this is an architecture feature proposal. Source and release inspection gives high confidence that current main lacks the requested distributed queue/backend contract.

## Solution Assessment

No for direct Redis/BullMQ core integration as written. The maintainable path is a maintainer-approved backend-neutral boundary that fits the existing cron, task, session, plugin, and config ownership model.
