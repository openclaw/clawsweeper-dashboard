## Summary

Keep open: current main and the latest release still drop Feishu card-action sibling fields, and the issue is paired with active same-author implementation work plus a broader related payload-contract issue.

## What This Changes

Request support for Feishu card action callbacks to preserve `action.option`, `action.options`, and `action.form_value` so plugins can receive select, multi-select, and form data.

## Best Possible Solution

Review and land one canonical Feishu plugin patch that preserves supported action sibling fields at the parser boundary, carries them through fallback dispatch, adds focused regression coverage, includes redacted live Feishu proof, and then closes the related Feishu issues/PRs cleanly.

## Reproduction Assessment

Yes, source-level. A raw `card.action.trigger` payload with `action.option`, `action.options`, or `action.form_value` is normalized into only `action.value` and `action.tag`, then fallback dispatch serializes only `action.value`.

## Solution Assessment

Unclear as a standalone handler-only patch. The maintainable path is to align parser, event type, decoder/fallback behavior, tests, and the broader Feishu callback payload contract through the open canonical PR path.
