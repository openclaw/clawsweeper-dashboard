## Summary

Keep open: current main and the latest release still drop Feishu card-action fields outside `action.value`, and the same-author implementation path remains open at https://github.com/openclaw/openclaw/pull/42908.

## What This Changes

The issue requests preserving Feishu `action.option`, `action.options`, and `action.form_value` callback fields so plugins can receive dropdown, multi-select, and form data.

## Best Possible Solution

Review, land, or supersede https://github.com/openclaw/openclaw/pull/42908 with one Feishu-plugin fix that preserves these fields from raw parsing through fallback dispatch and regression tests, coordinated with the broader callback-field follow-up.

## Reproduction Assessment

Yes, at source level. A raw `card.action.trigger` payload with `action.value` plus sibling `action.option`, `action.options`, or `action.form_value` is normalized into an event containing only `value` and `tag`, then fallback text serializes only `action.value`.

## Solution Assessment

Yes at the direction level: the narrow maintainable solution is to preserve these fields at the Feishu parser/event boundary and carry them through fallback dispatch with tests. The serialized content shape should be settled in the open PR review.
