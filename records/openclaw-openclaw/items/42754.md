## Label Justifications

- `P2`: This is a normal-priority Feishu channel data-preservation improvement with limited blast radius and active implementation candidates.
- `impact:data-loss`: Feishu callback fields carrying user selections and form data are silently dropped before plugins can read them.
- `impact:message-loss`: The channel callback message delivered to the agent omits part of the user interaction payload.

## Summary

Keep open: current main and the latest release still drop Feishu card-action sibling fields, and open Feishu PRs already cover overlapping fixes that need a maintainer-selected payload contract.

## What This Changes

The issue requests preserving `action.option`, `action.options`, and `action.form_value` from Feishu card callbacks so plugins can receive dropdown, multi-select, and form data.

## Best Possible Solution

Land one canonical Feishu plugin patch that preserves selected callback sibling fields from parser through fallback dispatch, covers empty payloads, includes focused tests and redacted live Feishu proof, then resolve the overlapping items.

## Reproduction Assessment

Yes, source-level. A raw `card.action.trigger` payload with `action.option`, `action.options`, or `action.form_value` is normalized into only `action.value` and `action.tag`, then fallback dispatch serializes only `action.value`.

## Solution Assessment

Not as a handler-only patch. The maintainable fix should start at `parseFeishuCardActionEventPayload` and align the event type, decoder/fallback behavior, handler dispatch, tests, and selected payload shape.
