## Summary

Keep open: current main and the latest tagged release still drop Feishu card-action sibling fields, and there are open Feishu PRs covering the same payload-contract work that need maintainer selection rather than a new cleanup close.

## What This Changes

The issue requests preserving Feishu `action.option`, `action.options`, and `action.form_value` callback fields so plugins can receive dropdown, multi-select, and form submission data.

## Best Possible Solution

Preserve the selected Feishu action sibling fields at `parseFeishuCardActionEventPayload`, carry them through decoder/fallback/handler dispatch under one payload contract, add focused regression coverage, and require redacted live Feishu proof before landing.

## Reproduction Assessment

Yes, source-level. A raw `card.action.trigger` payload with `action.option`, `action.options`, or `action.form_value` is normalized into only `action.value` and `action.tag`, then fallback dispatch serializes only `action.value`.

## Solution Assessment

Not as a handler-only patch. The maintainable fix should start at `parseFeishuCardActionEventPayload` and align the event type, decoder/fallback behavior, handler dispatch, tests, and selected payload shape.
