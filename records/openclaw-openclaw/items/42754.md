## Summary

Keep open: current main and the latest release still drop Feishu card-action sibling fields, and this issue is paired with open Feishu implementation work rather than ready for cleanup closure.

## What This Changes

Request support for preserving Feishu card-action `option`, `options`, and `form_value` fields so plugins can receive select, multi-select, and form submission data.

## Best Possible Solution

Land or replace one canonical Feishu plugin patch that preserves supported action sibling fields at the parser boundary, carries them through fallback dispatch, and adds focused regression plus redacted live Feishu proof.

## Reproduction Assessment

Yes, source-level. A raw `card.action.trigger` payload with `action.option`, `action.options`, or `action.form_value` is normalized into only `action.value` and `action.tag`, then fallback dispatch serializes only `action.value`.

## Solution Assessment

Yes directionally, but the maintainable solution should preserve these fields at the parser/decoder contract boundary rather than only patching one handler branch. The overlapping Feishu PRs should converge on one payload contract before merge.
