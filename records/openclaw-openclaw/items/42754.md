## Summary

Keep open: current main and the latest release still discard Feishu card-action sibling fields before fallback dispatch, and the issue is paired with open Feishu implementation work that needs one canonical payload contract.

## What This Changes

Request support for forwarding Feishu card callback `action.option`, `action.options`, and `action.form_value` to plugins instead of only forwarding `action.value`.

## Best Possible Solution

Select and land one canonical Feishu plugin patch that preserves supported sibling action fields at the parser boundary, carries them through fallback dispatch, adds focused regression coverage, and includes redacted live Feishu proof.

## Reproduction Assessment

Yes, source-level. A raw `card.action.trigger` payload with `action.option`, `action.options`, or `action.form_value` is normalized into only `action.value` and `action.tag`, then fallback dispatch serializes only `action.value`.

## Solution Assessment

Not as a handler-only patch. The maintainable fix should preserve these fields in `parseFeishuCardActionEventPayload`, then align the event type, decoder/fallback behavior, handler dispatch, and tests around one payload contract.
