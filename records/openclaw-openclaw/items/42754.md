## Summary

Keep open: current main and the latest release still drop Feishu card-action sibling fields, and the active implementation path is already covered by https://github.com/openclaw/openclaw/pull/42908 with broader overlap at https://github.com/openclaw/openclaw/issues/43202 and https://github.com/openclaw/openclaw/pull/43953.

## What This Changes

Preserve Feishu card action callback fields `action.option`, `action.options`, and `action.form_value` so plugins can receive select, multi-select, and form submission data.

## Best Possible Solution

Land or supersede one canonical Feishu plugin fix, likely through https://github.com/openclaw/openclaw/pull/42908 for the narrowed fields, preserving sibling fields from raw parsing through fallback dispatch with regression tests and redacted live Feishu proof.

## Reproduction Assessment

Yes, at source level. A raw `card.action.trigger` payload with sibling `action.option`, `action.options`, or `action.form_value` is normalized into an event containing only `value` and `tag`, then fallback text serializes only `action.value`.

## Solution Assessment

Yes directionally: the maintainable fix is to preserve these fields at the Feishu parser/event boundary and carry them through fallback dispatch with tests. The payload contract should be settled while reviewing the linked implementation PR and the broader callback-field follow-up.
