## Summary

Keep open: current `main` still drops the sibling Feishu card-action fields before the handler can use them, and open PR #42908 by the reporter is the active implementation path for this exact request.

## What This Changes

The issue requests preserving Feishu card callback fields `action.option`, `action.options`, and `action.form_value` so card selections and form submissions reach plugins/agents.

## Best Possible Solution

Land or replace the paired Feishu plugin PR with a focused change that preserves these callback fields through the parser, decoder/fallback, handler, and regression tests while keeping existing button flows unchanged.

## Reproduction Assessment

Yes, from source inspection: a callback with `action.value` plus sibling `action.option`, `action.options`, or `action.form_value` is parsed into an event that retains only `value` and `tag`, and the fallback text builder only uses `value`. I did not run tests because this review is read-only.

## Solution Assessment

Yes, the requested direction is narrow and maintainable if implemented at the Feishu parser/event boundary with focused regression coverage. The open paired PR is the right place to settle the exact serialized content format.
