## Label Justifications

- `P2`: This is a normal-priority Feishu channel data-preservation improvement with limited blast radius and active implementation candidates.
- `impact:data-loss`: Feishu callback fields carrying user selections and form data are discarded before plugins or agents can use them.
- `impact:message-loss`: The synthetic channel message delivered from the card callback omits part of the user interaction payload.

## Summary

Keep open: current main and release v2026.5.19 still strip the requested Feishu card-action sibling fields, and open implementation PRs at https://github.com/openclaw/openclaw/pull/42908 and https://github.com/openclaw/openclaw/pull/43953 already cover overlapping fixes that need a maintainer-selected payload contract.

## What This Changes

Request Feishu card-action callbacks preserve `action.option`, `action.options`, and `action.form_value` so plugins can receive select, multi-select, and form data.

## Best Possible Solution

Land one canonical Feishu plugin patch that preserves selected sibling action fields at parse time, carries them through fallback dispatch, covers the payload contract with focused tests, and includes redacted live Feishu proof.

## Reproduction Assessment

Yes, source-level. A raw `card.action.trigger` payload with `action.option`, `action.options`, or `action.form_value` would be normalized into only `action.value` and `action.tag`, then fallback dispatch serializes only `action.value`.

## Solution Assessment

No, not as a handler-only patch. The durable fix should start at `parseFeishuCardActionEventPayload` and align the event type, decoder/fallback behavior, handler dispatch, tests, and selected payload shape.
