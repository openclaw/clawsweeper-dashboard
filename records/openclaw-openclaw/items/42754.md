## Summary

Keep open: current main and the latest release still drop Feishu card-action sibling fields, while active related PRs already cover the repair path and need a maintainer payload-contract decision.

## What This Changes

Preserve Feishu `action.option`, `action.options`, and `action.form_value` data so plugins can receive dropdown, multi-select, and form submissions from card callbacks.

## Best Possible Solution

Land or replace one canonical Feishu plugin patch that preserves sibling action fields at the parser boundary, carries them through fallback/dispatch, and uses a maintainer-approved payload encoding with regression coverage and redacted live Feishu proof.

## Reproduction Assessment

Yes, at source level. A raw `card.action.trigger` payload with `action.option`, `action.options`, or `action.form_value` is normalized into only `value` and `tag`, then fallback dispatch serializes only `action.value`.

## Solution Assessment

Yes directionally, but not as a handler-only patch. The maintainable fix should preserve the fields in `parseFeishuCardActionEventPayload`, align decoder/fallback behavior, and decide JSON versus command-text encoding across the overlapping Feishu PRs.
