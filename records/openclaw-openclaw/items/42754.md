## Summary

Current main still loses the reported Feishu card fields, and two open linked PRs already own overlapping fixes; keep the issue open until maintainers choose one canonical payload contract and land it.

## What This Changes

Preserve Feishu card `option`, `options`, and `form_value` callback data through parsing and synthetic message dispatch.

## Best Possible Solution

Land one plugin-owned Feishu patch that preserves selected fields at the parser boundary, defines one stable structured payload contract, carries it through decoder/fallback/handler dispatch, retains existing button behavior, and includes focused tests plus redacted live Feishu proof.

## Reproduction Assessment

Yes, source-level with high confidence: a callback containing these sibling action fields is normalized to only `value` and `tag`, and fallback dispatch reads only `value`. This read-only review did not run a live Feishu callback.

## Solution Assessment

No final implementation is on main. A handler-only merge is not the best solution; the fix should start at the parser boundary and align the event type, decoder/fallback, handler behavior, tests, and one maintainer-approved payload shape.
