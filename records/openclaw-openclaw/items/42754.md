## Summary

Keep open: current main still drops the sibling Feishu card-action fields before the handler can use them, and the same-author PR #42908 is the active implementation path for this request.

## What This Changes

Support Feishu interactive card callbacks by preserving `action.option`, `action.options`, and `action.form_value` for plugin and agent handling.

## Best Possible Solution

Land or replace PR #42908 with a focused Feishu plugin change that preserves these fields through parsing, decoding/fallback text, handler dispatch, and regression tests while keeping button actions compatible.

## Reproduction Assessment

Yes. From source inspection, a callback with `action.value` plus sibling `action.option`, `action.options`, or `action.form_value` is parsed into an event that retains only `value` and `tag`, and the fallback text builder only uses `value`.

## Solution Assessment

Yes at the direction level: preserving the fields at the Feishu parser/event boundary with focused regression coverage is the narrow maintainable fix. The exact serialized content format should be settled in PR #42908 review.
