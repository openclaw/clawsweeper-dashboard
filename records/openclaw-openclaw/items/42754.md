## Summary

Keep open: current main and the latest release still strip the Feishu card action sibling fields, and this issue is paired with open implementation work rather than being ready for cleanup closure.

## What This Changes

The issue requests preserving Feishu card callback `option`, `options`, and `form_value` fields so plugins can receive select, multi-select, and form submission data.

## Best Possible Solution

Review, refresh, and land or replace one canonical Feishu plugin patch that preserves selected sibling fields at the parser boundary, carries them through fallback dispatch, adds regression coverage, and includes redacted live Feishu proof.

## Reproduction Assessment

Yes, source-level. A raw `card.action.trigger` payload with `action.option`, `action.options`, or `action.form_value` is normalized into only `action.value` and `action.tag`, then fallback dispatch serializes only `action.value`.

## Solution Assessment

Yes directionally, but it should be solved at the parser/decoder contract boundary rather than as a handler-only workaround. The overlapping Feishu PRs should converge on one payload shape before merge.
