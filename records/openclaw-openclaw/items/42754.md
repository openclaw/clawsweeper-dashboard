## Label Justifications

- `P2`: This is a normal-priority Feishu channel payload-preservation issue with real interactive-card workflow impact but limited blast radius.
- `impact:data-loss`: Feishu callback fields carrying user selections and form values are discarded before plugins or agents can use them.
- `impact:message-loss`: The synthetic channel message created from the card callback omits part of the user interaction payload.

## Summary

Keep open: current main and release v2026.5.20 still drop Feishu card-action sibling fields before agent dispatch, and the open implementation work at https://github.com/openclaw/openclaw/pull/42908 plus https://github.com/openclaw/openclaw/pull/43953 needs a maintainer-selected payload contract rather than cleanup closure.

## What This Changes

The issue asks Feishu card action callbacks to preserve action.option, action.options, and action.form_value so dropdown, multi-select, and form data can reach plugins and agents.

## Best Possible Solution

Land one canonical Feishu plugin patch that preserves selected sibling action fields at the parser boundary, carries them through decode/fallback/handler dispatch, keeps legacy command handling compatible, covers the chosen payload contract with tests, and includes redacted live Feishu proof.

## Reproduction Assessment

Yes, source-level. A raw card.action.trigger payload with action.option, action.options, or action.form_value is normalized into only action.value and action.tag, then fallback dispatch serializes only action.value.

## Solution Assessment

No final implementation is on main yet. A handler-only change would be incomplete; the maintainable fix should start at parseFeishuCardActionEventPayload and align the event type, decoder/fallback behavior, handler dispatch, tests, and selected payload shape.
