## Summary

Keep open: current main and the latest release still drop Feishu card-action sibling fields before dispatch, and active open PRs already cover the fix path.

## What This Changes

Request to preserve Feishu `action.option`, `action.options`, and `action.form_value` so card dropdowns, multi-select choices, and form submissions reach plugins/agents.

## Best Possible Solution

Review, land, or replace one canonical Feishu plugin patch that preserves sibling fields at the parser boundary and fallback dispatch, with regression tests and redacted live Feishu proof.

## Reproduction Assessment

Yes, at source level. A raw `card.action.trigger` payload with `action.option`, `action.options`, or `action.form_value` is normalized into only `value` and `tag`, then fallback dispatch serializes only `action.value`.

## Solution Assessment

Yes directionally, but the best fix should start at `parseFeishuCardActionEventPayload` and use one selected payload contract across the overlapping Feishu PRs.
