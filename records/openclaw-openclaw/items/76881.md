## Summary

Keep open: current main still awaits temp-file and parent-directory fsync in the session-store writer path, and this PR is a narrow fix direction, but it still has concrete review blockers around durability wording, test proof, and changelog coverage.

## What This Changes

Adds a default-on `durable` option to atomic JSON/text writes, opts the session-store registry write out with `durable: false`, and adds focused `json-files` tests.

## Best Possible Solution

Land the narrow patch after rewording the durability rationale, adding the required changelog entry, and tightening the default fsync regression test while keeping all other atomic writes durable by default.

## Reproduction Assessment

Yes, from source inspection: current main serializes session-store writes through `runExclusiveSessionStoreWrite` and awaits temp-file plus parent-directory fsync in `writeTextAtomic`. The linked field reports provide the heavy-cron load evidence, though I did not run that live reproduction in this read-only pass.

## Solution Assessment

Mostly yes: a default-on internal `durable` option with a session-registry-only opt-out is the narrowest maintainable shape. The patch should avoid claiming fsync-backed transcript recovery and should add direct test and changelog proof before merge.
