## Summary

Keep this PR open: the implementation is the right narrow shape, but it still has actionable merge blockers around overstated durability wording, missing changelog coverage, and a weak default-fsync test.

## What This Changes

Adds a default-on `durable` option to atomic JSON/text writes, opts the session-store registry write out of fsync, and adds `json-files` tests for durable modes.

## Best Possible Solution

Land a narrow version that keeps default durable atomic writes, opts only the session-store registry out, accurately documents crash-loss behavior, includes a changelog entry, and directly tests both durable modes.

## Reproduction Assessment

Yes, from source inspection: current main awaits temp-file and parent-directory fsync inside the queued session-store writer path, and linked field reports show that path causing long lock holds under cron load. I did not run a live heavy-cron reproduction in this read-only review.

## Solution Assessment

Mostly yes: a default-on `durable` option with a session-registry-only opt-out is the narrowest maintainable shape. The patch should reword the durability rationale, add the changelog entry, and strengthen the default fsync assertion before merge.
