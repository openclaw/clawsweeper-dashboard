## Summary

Keep open. Current main still runs live memory-search query embeddings as a single provider call with no retry for the reported socket/TLS transport failures, and the related open PR is scoped to batch indexing rather than this live query path.

## What This Changes

Live memory search query embeddings intermittently fail on transient socket/TLS fetch errors against remote embedding providers.

## Best Possible Solution

Add a bounded retry/backoff path for live query embeddings using the shared memory embedding transport-error classifier, while keeping auth, configuration, and non-retryable HTTP failures fail-fast.

## Reproduction Assessment

Yes. The issue includes concrete live repro loops across providers, and current main has a deterministic unit seam: make `provider.embedQuery()` fail once with a socket-style transport error, then assert `MemoryManager.search()` retries and returns results.

## Solution Assessment

Yes. A scoped live-query retry is the narrowest maintainable fix for the user-visible failure; dispatcher tuning or a new config knob can remain follow-up work if field data shows retries are insufficient.

## Repair Work Prompt

Fix the live memory-search query embedding transport retry gap reported at https://github.com/openclaw/openclaw/issues/71784. Current main retries batch embeddings but `MemoryManager.search()` still calls `embedQueryWithTimeout()` once, so transient remote embedding socket/TLS failures such as `fetch failed`, `other side closed`, `ECONNRESET`, `EPIPE`, `UND_ERR_SOCKET`, and `Client network socket disconnected before secure TLS connection was established` abort live search. Add a bounded retry/backoff path for query embeddings, preferably reusing/extending the memory embedding retry classifier shared with batch retries. Keep auth/config/provider HTTP status failures fail-fast, do not add provider-specific branches or new config knobs, and do not mix in QMD SQLite lock contention work from #66339. Coordinate with #44167 if its batch classifier changes have landed or are still open. Add focused regression tests for a query embed that fails once with a retryable transport error and then succeeds, plus classifier coverage for the reported transport strings. This is user-facing, so include a single-line CHANGELOG.md fix entry with appropriate non-bot attribution if known.
