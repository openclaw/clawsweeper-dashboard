## Summary

Keep open. Current main still has a single-shot live memory-search query embedding path; the existing retry loop is used for batch embeddings only, and the retry classifier still omits the socket/TLS transport failures reported by multiple commenters.

## What This Changes

The issue requests reliable live memory-search embedding queries when remote providers hit transient guarded-fetch or Undici socket failures.

## Best Possible Solution

Live query embeddings should reuse or lightly generalize the memory embedding retry/backoff policy for narrow transient transport errors while preserving fail-fast behavior for auth, config, schema, and non-retryable provider responses.

## Reproduction Assessment

Yes. The issue and follow-up comments provide concrete OpenAI, Gemini, and OpenRouter live search loops with matching socket errors, and current source gives a deterministic unit seam by making provider.embedQuery() fail once with a transient transport error and then succeed.

## Solution Assessment

Yes for the scoped repair direction. A bounded retry around live query embeddings is narrower and more maintainable than first changing the global guarded-fetch dispatcher or adding user-facing retry configuration.

## Repair Work Prompt

Fix the live memory-search embedding reliability bug reported in https://github.com/openclaw/openclaw/issues/71784. Current main retries batch embeddings but `MemoryManagerEmbeddingOps.embedQueryWithTimeout()` directly wraps `provider.embedQuery(text)` with a timeout, so `openclaw memory search` can fail on transient guarded-fetch/Undici socket errors such as `fetch failed`, `other side closed`, `ECONNRESET`, `EPIPE`, `socket hang up`, and `Client network socket disconnected before secure TLS connection was established`. Keep the fix scoped to memory live-query embeddings: reuse or lightly generalize the retry helper in `extensions/memory-core/src/memory/manager-embedding-policy.ts` and `extensions/memory-core/src/memory/manager-embedding-ops.ts`; classify only narrow transient transport/socket failures as retryable; do not change provider selection, auth/config/schema, timeout constants, cache format, SSRF policy, global Undici dispatcher behavior, or QMD SQLite lock handling. Add focused tests proving query embedding retries once and succeeds for a transport error while non-retryable auth/config/provider HTTP errors fail fast. Coordinate with #44167/#44166 if the batch retry work lands first, but ensure the live query path is covered. Add a single-line user-facing `CHANGELOG.md` fix entry crediting @kevinheinrichs and the confirming reporters where appropriate, and do not credit bot or maintainer handles. Validate with focused memory tests and then `pnpm check:changed` via Blacksmith Testbox before handoff.
