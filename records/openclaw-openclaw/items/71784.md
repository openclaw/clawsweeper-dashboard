## Summary

Keep open. Current main still lets live memory-search query embeddings fail on a single remote provider call, while the related open PR only addresses batch indexing behavior and does not close the live query path reported here.

## What This Changes

The issue asks OpenClaw to make builtin memory live semantic search resilient to transient remote embedding socket/TLS failures such as `fetch failed`, `other side closed`, `ECONNRESET`, and `EPIPE`.

## Best Possible Solution

Add a narrow bounded retry/backoff path for live query embeddings, reuse or extend the existing memory embedding retry policy for transient transport/socket errors, and keep auth/config/provider HTTP status failures fail-fast.

## Reproduction Assessment

Yes. The issue has concrete live repro loops across providers, and current main gives a deterministic unit seam: make `provider.embedQuery()` throw one transient socket-style error before succeeding, then assert `MemoryManager.search()` retries and returns results.

## Solution Assessment

Yes. A scoped live-query retry using the existing memory embedding retry helper is the narrowest maintainable fix; global Undici dispatcher tuning or new config knobs can remain follow-up work if the retry does not fully address the field reports.
