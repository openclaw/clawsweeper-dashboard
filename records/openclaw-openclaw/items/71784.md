## Summary

Keep open. Current main still leaves live memory-search query embeddings on a single provider call, while existing retry behavior covers batch/probe paths and the open related PR is batch-scoped rather than a fix for this live query failure.

## What This Changes

The report asks OpenClaw to make live memory search query embeddings resilient to transient remote embedding transport/socket failures across providers.

## Best Possible Solution

Add a narrow bounded retry/backoff for live query embeddings using the existing memory embedding retry policy, expand transport-error classification, and keep auth/config/provider HTTP status failures fail-fast.

## Reproduction Assessment

Yes. The issue includes concrete live loops and provider-diverse confirmations; current main also has a deterministic unit seam by making `provider.embedQuery()` fail once with a socket-style `fetch failed` error and then succeed.

## Solution Assessment

Yes. Reusing the existing memory embedding retry helper for live query embeddings is narrower and more maintainable than first changing global Undici dispatcher behavior or adding new config knobs.
