## Summary

Keep open. Current main already covers the rapid same-sender batching part through `messages.inbound` and channel debouncers, and the related coalescing RFC was closed as implemented, but this issue also asks for semantic multi-topic splitting and adaptive timing that are not implemented or tracked by a narrower canonical item.

## What This Changes

Request to add semantic preprocessing that coalesces rapid short chat messages, splits or structures long multi-topic messages, and optionally adapts the batching window.

## Best Possible Solution

Keep the shipped inbound debounce/coalescing path for rapid bursts, and define any remaining semantic topic-splitting or adaptive-timing work as a narrower product RFC with clear config and channel/API boundaries.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source review shows rapid same-sender batching exists, while semantic multi-topic splitting and adaptive windows have no current code path.

## Solution Assessment

Unclear as proposed: a broad semantic preprocessing layer is not yet the narrowest maintainable solution. The safer path is to split the already-implemented debounce behavior from a separate maintainer decision on semantic topic handling.
