## Summary

Keep open: current main already implements configurable text-only inbound debouncing for the rapid short-message subset, and related narrower issues cover media and channel-specific batching gaps, but this issue still uniquely asks for semantic long-message splitting and adaptive timing. That remaining work is a cross-channel product/API design rather than conservative cleanup closure.

## What This Changes

The issue requests semantic preprocessing to merge rapid short chat bursts, split or structure long multi-topic messages, and optionally adapt aggregation timing to user behavior.

## Best Possible Solution

Use this issue as the product/API design thread for whether OpenClaw should add semantic long-message splitting, adaptive timing, or a plugin/SDK preprocessing hook on top of the existing `messages.inbound` debounce baseline.
