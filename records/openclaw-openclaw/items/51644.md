## Summary

Keep open: current main already covers the rapid short-message coalescing part through inbound debouncing and channel-specific coalescers, but the requested semantic long-message splitting and adaptive behavior are still a new cross-channel product/API design.

## What This Changes

The issue requests semantic preprocessing for inbound chat messages: batching rapid short bursts, detecting multi-topic long messages, and optionally adapting the batching window.

## Best Possible Solution

Keep the existing `messages.inbound` coalescing baseline and use this issue for a maintainer design decision on whether OpenClaw should add semantic long-message splitting, adaptive timing, or a plugin/SDK hook for that preprocessing.

## Reproduction Assessment

Not applicable. This is a feature request, and current source/docs show existing debounce behavior rather than a broken existing semantic-splitting contract.

## Solution Assessment

No, closing as implemented would be too broad: current main implements time-window coalescing, but not the requested semantic multi-topic splitting or adaptive behavior. The maintainable path is a product/API decision layered on top of the existing inbound debounce surface.
