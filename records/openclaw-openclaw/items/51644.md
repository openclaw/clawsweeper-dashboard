## Summary

Keep open: current main and v2026.5.22 implement fixed same-sender inbound debouncing and channel-side concatenation, but semantic long-message splitting and adaptive timing remain absent. The implemented rapid-batching subset is covered by closed related work, while adaptive timing is still tracked at https://github.com/openclaw/openclaw/issues/57123.

## What This Changes

The issue requests semantic inbound message preprocessing that merges rapid short messages, splits or structures long multi-topic messages, and optionally adapts the aggregation window to user cadence.

## Best Possible Solution

Keep fixed messages.inbound as the baseline, design semantic splitting as an explicit opt-in product/API contract, and coordinate adaptive timing through https://github.com/openclaw/openclaw/issues/57123 unless maintainers merge the scopes.

## Reproduction Assessment

Not applicable; this is a feature request rather than a broken existing behavior. Source, schema, docs, and release inspection show fixed text batching, not an existing semantic-splitting contract to reproduce.

## Solution Assessment

No for closure or a single automated bugfix. The maintainable path needs an explicit product/API decision for semantic splitting and adaptive timing while preserving current fixed-window behavior unless maintainers choose a migration.
