## Summary

Keep open: current main and the latest release cover fixed same-sender inbound debouncing, but this issue still asks for semantic long-message splitting and adaptive aggregation behavior that are absent from the current config, schema, docs, and debouncer contract. The closed coalescing RFC at https://github.com/openclaw/openclaw/issues/51361 covers the implemented rapid-batching subset, while adaptive timing remains separately tracked at https://github.com/openclaw/openclaw/issues/57123.

## What This Changes

Request a semantic message preprocessing layer that merges rapid short messages, splits or structures long multi-topic messages, and adapts aggregation timing.

## Best Possible Solution

Keep fixed `messages.inbound` as the current baseline and use this issue for product/API design of opt-in semantic splitting, with adaptive timing coordinated through https://github.com/openclaw/openclaw/issues/57123 unless maintainers merge the scopes.

## Solution Assessment
