## Summary

Keep open: current main and v2026.5.22 cover fixed same-sender inbound debouncing and iMessage split-send coalescing, but the requested semantic long-message topic splitting and adaptive timing remain missing. Adaptive timing is already tracked at https://github.com/openclaw/openclaw/issues/57123, while no narrower canonical item I found covers the multi-topic splitting part.

## What This Changes

The issue asks for semantic inbound preprocessing that merges rapid short-message bursts, splits or structures long multi-topic messages, and adapts timing to user cadence.

## Best Possible Solution

Keep the shipped fixed-debounce behavior, handle adaptive timing through the existing focused issue, and decide whether long-message topic splitting belongs in a generic inbound runtime/API hook, a plugin, or prompt-layer behavior before implementation.

## Reproduction Assessment

Not applicable; this is a feature request, not a report of broken existing behavior. Source inspection confirms the existing behavior is fixed-window same-sender debouncing rather than semantic aggregation or topic splitting.

## Solution Assessment

Unclear; the all-in-one semantic preprocessing layer is broader than the shipped debounce model and needs maintainer product/API direction. A narrower maintainable path is to keep fixed debouncing stable, route adaptive timing through https://github.com/openclaw/openclaw/issues/57123, and separately design any topic-splitting hook.
