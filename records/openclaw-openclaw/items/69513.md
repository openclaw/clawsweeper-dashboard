## Summary

Keep open. Current main still validates cron announce delivery channels and schema shape, but not explicit primary or failure-destination account IDs before persistence; open PR #69562 and fork PR #2 already target the same narrow bug.

## What This Changes

Validate cron create/update announce delivery so explicit account IDs are rejected when they are not configured for the selected channel.

## Best Possible Solution

Land one focused gateway validation fix covering `cron.add` and `cron.update` for primary and failure-destination announce account IDs, with regression tests and a changelog entry.

## Reproduction Assessment

Yes. The reported CLI path maps directly to current source: the CLI forwards `--account`, the protocol accepts any non-empty string, the gateway validates only channel shape/configuration, and persistence follows that incomplete guard.

## Solution Assessment

Yes. Gateway-side validation against the selected channel's configured account IDs is the narrowest maintainable fix because it covers CLI and RPC callers before jobs are stored.
