## Summary

Keep open. Current main still validates cron announce channels but does not validate explicit primary or failure-destination account IDs against configured channel accounts before persistence, and there is already open candidate PR context for this same bug.

## What This Changes

The issue requests rejecting cron announce creation/update when an explicit delivery accountId is not configured for the selected channel.

## Best Possible Solution

Land one focused gateway-side validation change that rejects unknown explicit announce account IDs for both cron.add and cron.update, including failure destinations, with regression tests and a user-facing changelog entry.

## Reproduction Assessment

Yes. The issue's CLI steps match current source inspection: the CLI forwards the accountId, gateway validation only checks channels, and persistence follows that incomplete guard.

## Solution Assessment

Yes. Gateway-side add/update validation against configured channel accounts is the narrowest maintainable fix because it covers CLI and RPC callers before persistence while leaving implicit account resolution and delivery execution unchanged.
