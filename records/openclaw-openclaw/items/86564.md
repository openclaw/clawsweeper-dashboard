## What This Changes

The branch makes gateway provider-auth startup prewarm opt-in with `enabled: true`, keeps hook-only auth failure invalidation and delayed rewarm when `providerAuthPrewarm` is supplied without `enabled`, and adds a gateway regression test.

## Best Possible Solution

Land the narrow hook-only default only after maintainers explicitly accept the upgrade tradeoff, and keep the broader WSL2/Telegram event-loop diagnostics tracked outside this PR.

## Solution Assessment
