## Summary

Keep open: current main and the latest release still schedule provider-auth startup warming by default, and this PR carries a useful hook-only/default-off fix. It is not a cleanup close because it conflicts with current main and overlaps the still-open maintainer branch at https://github.com/openclaw/openclaw/pull/88514, so maintainers need to pick and refresh one branch and accept the auth-startup compatibility tradeoff.

## What This Changes

The PR changes gateway post-attach provider-auth sidecar scheduling so omitted `providerAuthPrewarm.enabled` registers the auth-failure rewarm hook without running startup warm, and adds regression coverage.

## Best Possible Solution

Choose one canonical branch, refresh it onto current main while preserving current lazy imports and agent-runtime-plugin prewarm, and land only after maintainers accept lazy/default-off provider-auth startup; track broader startup/event-loop symptoms separately.
