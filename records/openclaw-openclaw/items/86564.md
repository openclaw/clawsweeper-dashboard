This PR changes gateway post-attach provider-auth scheduling so omitted `providerAuthPrewarm.enabled` keeps the failure rewarm sidecar without running startup warm, and adds a focused regression test.

## Best Possible Solution

Consolidate on one mergeable hook-only/default-off provider-auth startup branch, preserve explicit startup warming and failure-hook rewarm coverage, and track broader startup/event-loop symptoms separately.
