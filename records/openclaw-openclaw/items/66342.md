## Summary

## What This Changes

The PR adds a terminal channel error type, changes Slack terminal auth/reconnect failures to throw it, suppresses gateway auto-restart for that signal, and adds Slack/gateway regression tests.

## Best Possible Solution

Keep the terminal channel failure signal only after moving supervisor coverage to a core or public-test surface, fixing CI and SDK baseline alignment, and adding redacted real Slack/gateway proof.

## Reproduction Assessment

Yes. Source inspection of current main shows Slack non-recoverable auth failures still throw from monitorSlackProvider, and server-channels catches the failure then schedules bounded restarts because it lacks a terminal-error signal.

## Solution Assessment

Mostly yes, but not as submitted. A typed terminal channel failure signal is a narrow maintainable direction, but the current branch needs test-boundary, type, lint, API-baseline, and real-proof fixes first.
