## Summary

Keep this PR open, but it is not mergeable as-is. Current main already ships bounded Slack/gateway crash-loop handling, while this branch adds a useful terminal no-restart refinement that still has two concrete branch defects and lacks real behavior proof.

## What This Changes

This PR adds a TerminalChannelError signal for Slack auth/reconnect terminal failures, suppresses gateway auto-restart for that signal, exports the error through the Plugin SDK, and adds Slack/gateway regression coverage.

## Best Possible Solution

Keep the terminal failure/no-restart lifecycle handling only after the test uses the current manager API, the Plugin SDK contract is aligned, and the contributor adds redacted after-fix real behavior proof.

## Reproduction Assessment

Yes. Source inspection shows current main can reproduce the remaining behavior by making Slack Bolt app.start reject with account_inactive: the Slack monitor throws, the gateway records the error, and the non-terminal supervisor path schedules bounded auto-restarts.

## Solution Assessment

Mostly yes. A typed terminal channel failure signal is a narrow maintainable direction, but this branch must compile against the current manager API and keep the public SDK baseline in sync before it is a good fix.
