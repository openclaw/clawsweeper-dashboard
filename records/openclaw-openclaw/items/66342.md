## Summary

Keep open: the PR is a plausible narrow Slack/gateway lifecycle improvement, but latest head still has blocking type, lint, extension-boundary, and SDK-baseline problems, and external contributor real behavior proof is missing.

## What This Changes

The PR adds a TerminalChannelError signal, makes Slack terminal auth/reconnect failures throw it, suppresses gateway restarts for that signal, and adds Slack/gateway regression tests.

## Best Possible Solution

Keep the terminal-failure/no-restart direction only after moving supervisor coverage to an allowed core or public-test surface, fixing CI/API-baseline issues, and adding redacted real Slack/gateway proof.

## Reproduction Assessment

Yes. Source inspection of current main shows Slack non-recoverable auth failures still throw from monitorSlackProvider, and server-channels catches the failure then schedules bounded restarts because it lacks a terminal-error signal.

## Solution Assessment

Mostly yes, but not as submitted. A typed terminal channel failure signal is a narrow maintainable direction, but this branch must respect test boundaries, current manager APIs, lint, public SDK baseline requirements, and real proof before merge.
