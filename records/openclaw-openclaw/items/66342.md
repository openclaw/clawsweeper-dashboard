## Summary

Keep this PR open, but it is not merge-ready: current main already shipped the broad gateway crash-loop isolation for the linked report, while this branch still offers a narrower terminal Slack failure/no-restart improvement that currently breaks type and boundary checks and lacks real behavior proof.

## What This Changes

The PR adds a TerminalChannelError signal for Slack terminal auth/reconnect failures, suppresses gateway auto-restart for that signal, exports the error through the Plugin SDK runtime-env surface, and adds Slack/gateway regression coverage.

## Best Possible Solution

Keep the terminal-failure/no-restart lifecycle direction only after moving gateway supervisor coverage to an allowed core or public-test surface, fixing current API/type errors, updating the SDK baseline if the export remains public, and adding redacted after-fix real behavior proof.

## Reproduction Assessment

Yes. Source inspection shows current main can reproduce the remaining behavior by making Slack Bolt startup fail with account_inactive or invalid_auth: Slack throws, the gateway records lastError, and the non-terminal supervisor path schedules bounded restart attempts.

## Solution Assessment

Mostly yes. A typed terminal channel failure signal is a narrow maintainable way to stop useless auth-error restarts, but this branch must respect extension test boundaries, current manager APIs, and the public SDK baseline before it is the best implementation.
