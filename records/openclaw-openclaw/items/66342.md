## Summary

Keep this PR open: the terminal Slack auth-failure direction is plausible and not fully present on current main, but the branch still has CI-blocking test/API contract issues and lacks external contributor real behavior proof.

## What This Changes

This PR adds a TerminalChannelError path for Slack terminal auth/reconnect failures, suppresses gateway auto-restart for that signal, exports it through plugin-sdk/runtime-env, and adds Slack/gateway regression coverage.

## Best Possible Solution

Land a rebased branch that keeps the terminal failure signal, puts supervisor coverage on allowed test boundaries, updates the Plugin SDK baseline if the export stays public, and includes redacted live gateway/Slack proof.

## Reproduction Assessment

Yes, by source inspection. On current main, a Slack Bolt startup failure with account_inactive reaches monitorSlackProvider's non-recoverable auth branch, throws as an ordinary error, and then the gateway supervisor records it and schedules the normal bounded restart path.

## Solution Assessment

Mostly yes. A typed terminal channel failure is a narrow maintainable way to preserve lastError while suppressing restarts, but this branch must fix its test boundary/API contract issues and add real behavior proof before merge.
