Keep open: the PR addresses a still-source-visible Slack terminal auth/restart path, but the current head is not mergeable because CI reports type, lint, and boundary failures, and the external real-behavior proof gate is still missing.

## What This Changes

The PR adds a terminal channel error signal, changes Slack terminal auth/reconnect failures to throw it, suppresses gateway restarts for that signal, and adds Slack/gateway regression tests.

## Best Possible Solution

Keep the terminal failure signal only after moving supervisor coverage to a core or public-test surface, fixing CI and SDK contract alignment, and adding redacted real Slack/gateway proof.

## Reproduction Assessment

Yes, by source inspection. Current main throws for non-recoverable Slack auth failures and the gateway then treats that as a restartable channel failure because no terminal-error signal exists.

## Solution Assessment

Mostly yes, but not as submitted. A typed terminal channel failure signal is a narrow maintainable direction, but this branch needs test-boundary, type, lint, API-baseline, and real-proof fixes first.
