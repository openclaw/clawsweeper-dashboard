## Summary

Keep open: current main does not expose a public per-run workdir override, and the existing sandbox scopes cannot express a shared-per-run but isolated remote workspace across multiple agents. This is a real feature request, but it needs maintainer API and security design before implementation.

## What This Changes

Request a dynamic per-invocation or per-session sandbox workdir so multiple agents can share one run workspace while keeping separate agent identities and run isolation.

## Best Possible Solution

Design a narrow Gateway/CLI/SDK workdir override for sandboxed agent turns that validates remote path ownership, preserves the agent workspace for identity/bootstrap context, and documents/tests SSH, OpenShell, and Docker behavior without adding an orchestration framework to core.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that current main lacks the requested CLI/Gateway/protocol workdir override, but there is no broken existing behavior contract to reproduce.

## Solution Assessment

Unclear: `--workdir` is a plausible user-facing shape, but it is not yet proven to be the best solution until maintainers choose the API, authorization, local-vs-remote path semantics, and sandbox lifecycle behavior.
