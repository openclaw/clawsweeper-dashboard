## Summary

Keep open. Current main still has session-oriented routing, session pickers, and partial single-session/global-session affordances, but it does not provide the requested first-class linear persistent workspace mode for blind users; the open related picker PR is useful partial overlap, not a complete fix.

## What This Changes

Request to add an accessibility-oriented linear persistent workspace mode with one main timeline, task sections/bookmarks, quick navigation, and screen-reader-friendly session recovery.

## Best Possible Solution

Design a maintainer-approved accessibility workflow that preserves session isolation while adding a screen-reader-friendly linear timeline, index, task-section/bookmark navigation, and recovery behavior across the primary clients.

## Reproduction Assessment

Not applicable; this is an accessibility workflow feature request, not a broken existing behavior report. Source and docs inspection confirm current main remains session/picker-oriented rather than linear-workspace-oriented.

## Solution Assessment

Unclear until maintainers choose the product shape. A linear persistent workspace mode is plausible, but the maintainable solution must define session isolation, timeline/bookmark semantics, client scope, and VoiceOver proof before implementation.
