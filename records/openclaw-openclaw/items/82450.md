## Summary

Keep open. Current main still exposes a session-oriented workflow with session pickers, session lists, filters, labels, checkpoints, and some ARIA affordances, but it does not provide the requested first-class linear persistent workspace mode for blind users.

## What This Changes

The issue requests a linear persistent workspace mode with one main workspace, task grouping, bookmarkable sections, jump navigation, timeline navigation, and a screen-reader-friendly session index for blind users.

## Best Possible Solution

Design an accessibility-first linear workspace workflow that preserves session isolation while adding screen-reader-friendly timeline, task-section, jump, index, and recovery behavior in the primary clients.

## Reproduction Assessment

Not applicable; this is an accessibility workflow feature request, not broken existing behavior with a defined expected result. Current docs, source, and tests show the session-oriented workflow the reporter wants to improve.

## Solution Assessment

Unclear; the requested direction is plausible, but the final UX/API shape needs maintainer accessibility/product design with blind-user feedback before implementation.
