## Summary

Keep open. Current main has session lists, session switching, per-session overrides, search/filtering, checkpoint controls, and persistent session tooling, but it does not provide the requested first-class linear persistent workspace mode for blind users; this needs accessibility/product design across session model and client UIs.

## What This Changes

Requests an accessibility-focused mode that keeps work in one persistent linear workspace with task sections, jump navigation, and recovery instead of many independent sessions.

## Best Possible Solution

Design an accessibility-first linear workspace workflow that preserves existing session isolation APIs while adding screen-reader-friendly timeline, index, jump, task-section, and recovery behavior in the primary chat clients.

## Reproduction Assessment

Not applicable; this is an accessibility workflow feature request, not a broken existing behavior with a defined expected result. Source and docs inspection show the current session-oriented workflow that the request wants to change.

## Solution Assessment

Unclear; the requested direction is plausible, but the issue intentionally leaves the UX/API shape open. The best next step is maintainer accessibility/product design with blind-user feedback before implementation.
