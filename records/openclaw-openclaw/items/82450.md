## Summary

Keep open. Current main still exposes session-oriented Control UI, TUI, and Gateway session surfaces, with no first-class linear persistent workspace mode; this is a valid accessibility feature request needing product/design review rather than cleanup.

## What This Changes

Request a screen-reader-friendly linear persistent workspace mode with one main workspace, task sections, bookmarks, quick jumps, and easier context recovery.

## Best Possible Solution

Design an accessibility-first linear workspace workflow that preserves session isolation while adding screen-reader-friendly timeline, index, jump, task-section, and context recovery behavior in the primary clients.
