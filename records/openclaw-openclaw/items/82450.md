## Summary

Keep open. Current main still exposes a session-oriented workflow with session pickers, per-session overrides, session lists, and cross-session tools, but it does not provide the requested first-class linear persistent workspace mode for blind users; this needs accessibility and product design rather than conservative cleanup.

## What This Changes

Requests a first-class accessibility mode with one persistent workspace, task sections, quick-jump/timeline navigation, and screen-reader-friendly recovery of older context.

## Best Possible Solution

Design an accessibility-first linear workspace workflow that preserves session isolation while adding screen-reader-friendly timeline, index, jump, task-section, and context recovery behavior in the primary clients.

## Reproduction Assessment

Not applicable; this is an accessibility workflow feature request, not broken existing behavior with a defined expected result. Source and docs inspection show the current session-oriented workflow that the request wants to change.

## Solution Assessment
