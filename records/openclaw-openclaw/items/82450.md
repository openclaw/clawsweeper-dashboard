## Summary

Keep open. Current main still exposes a session-oriented workflow with session lists, pickers, per-session overrides, search, and history tools, but it does not provide the requested first-class linear persistent workspace mode for blind users.

## What This Changes

The issue requests an accessibility-focused linear workspace mode for blind users, with persistent timeline-style task navigation instead of fragmented session switching.

## Best Possible Solution

Design an accessibility-first linear workspace workflow that preserves session isolation while adding screen-reader-friendly timeline, index, jump, task-section, and recovery behavior in primary clients.

## Reproduction Assessment

Not applicable; this is an accessibility workflow feature request, not broken existing behavior with a defined expected result. Source and docs inspection show the current session-oriented workflow that the request wants to change.

## Solution Assessment

Unclear; the requested direction is plausible, but the issue intentionally leaves the UX/API shape open. The best next step is maintainer accessibility and product design with blind-user feedback before implementation.
