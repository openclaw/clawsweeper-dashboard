## Summary

Keep open. Current main still implements a session-oriented workflow with session pickers, per-session history, session creation/reset, and recent-session navigation, but it does not provide the requested first-class linear persistent workspace mode for blind users; this remains a product/accessibility design item, not conservative cleanup.

## What This Changes

The issue requests an accessibility-oriented linear persistent workspace mode that reduces session fragmentation for blind users.

## Best Possible Solution

Design an accessibility-first linear workspace workflow that preserves session isolation while adding screen-reader-friendly timeline, index, jump, task-section, and context recovery behavior in the primary clients.
