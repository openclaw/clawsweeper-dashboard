## Summary

Keep open. Current main still has session routing, session pickers, recent-session shortcuts, and a configurable global session scope, but it does not provide the requested first-class linear persistent workspace mode for blind users; this needs accessibility/product design rather than cleanup closure.

## What This Changes

Request a blind-user-friendly linear persistent workspace mode with timeline, bookmarks, jump navigation, and reduced cross-session fragmentation.

## Best Possible Solution

Design an accessibility-first linear workspace workflow that preserves session isolation while adding timeline, task-section, jump, bookmark, and recovery affordances in the primary clients.

## Reproduction Assessment

Not applicable; this is an accessibility workflow feature request, not a report of broken behavior with a defined expected result. Source and docs inspection show the current session-oriented workflow the request wants to change.

## Solution Assessment

Unclear; the requested direction is plausible, but the specific UX and session-contract shape need maintainer accessibility/product design. Existing global session scope and recent-session shortcuts are useful partial affordances, not the full solution.
