## Summary

Keep open. Current main still has a session-oriented workflow with session pickers, session creation/reset, per-session overrides, and a newer session workspace file rail, but it does not provide the requested first-class linear persistent workspace mode for blind users.

## What This Changes

The issue requests an accessibility-oriented linear persistent workspace mode with one main workspace/chat, task sections, timeline navigation, bookmarks, and easier context recovery for blind users.

## Best Possible Solution

Design an accessibility-first linear workspace workflow that preserves session isolation while adding screen-reader-friendly timeline, index, jump, bookmark/task-section, and context-recovery behavior across the primary clients.

## Reproduction Assessment

Not applicable; this is an accessibility workflow feature request, not broken existing behavior with a defined expected result. Source and docs inspection show the session-oriented behavior the request wants to augment.

## Solution Assessment

Unclear. A linear persistent workspace mode is a plausible direction, but the right solution needs maintainer product/accessibility design before implementation because the issue leaves session isolation, bookmarks, timeline behavior, and client UX open.
