## Summary

Keep open. Current main still uses session-based navigation, session pickers, `/new`/`/reset`, and bounded session indexes; it does not provide the requested first-class linear persistent workspace mode, so this is an accessibility/product feature request rather than conservative cleanup.

## What This Changes

The issue requests an accessibility-focused linear persistent workspace mode with task sections, jump navigation, and screen-reader-friendly session recovery for blind users.

## Best Possible Solution

Design an accessibility-first workspace workflow that preserves session isolation while adding screen-reader-friendly timeline, task-section, bookmark, jump, and recovery behavior in the primary clients.

## Reproduction Assessment

Not applicable; this is an accessibility workflow feature request, not broken existing behavior with a defined expected result. Source and docs inspection shows the current session-oriented workflow that the request wants to change.

## Solution Assessment
