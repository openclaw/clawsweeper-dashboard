## Summary

Keep open. Current main has session lists, session switching, per-session overrides, search/filtering, checkpoint controls, and persistent session tooling, but it does not provide the requested first-class linear workspace mode for blind users; this needs accessibility and product design across the session model and client UIs.

## What This Changes

The issue requests a first-class linear persistent workspace mode for blind users, reducing multi-session context fragmentation with timeline, jump, section, and session-index affordances.

## Best Possible Solution

Design an accessibility-first linear workspace workflow that preserves session isolation while adding screen-reader-friendly timeline, index, jump, task-section, and recovery behavior in the primary clients.

## Reproduction Assessment

Not applicable; this is an accessibility workflow feature request, not broken existing behavior with a defined expected result. Source and docs inspection show the current session-oriented workflow that the request wants to change.

## Solution Assessment

Unclear; the requested direction is plausible, but the issue intentionally leaves the UX/API shape open. The best next step is maintainer accessibility and product design with blind-user feedback before implementation.
