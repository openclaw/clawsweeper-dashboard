## Summary

Keep open. Current main has session lists, session switching, per-session overrides, search/filtering, pins, checkpoints, and persistent named cron sessions, but it does not provide the requested first-class linear persistent workspace mode for blind users; this needs accessibility/product design across the session model and client UIs.

## What This Changes

Requests an accessibility-oriented persistent linear workspace mode that reduces session/context fragmentation for blind users.

## Best Possible Solution

Design an accessibility-first linear workspace workflow that preserves the existing session APIs while adding screen-reader-friendly timeline, index, jump, and recovery behavior in the primary chat clients.
