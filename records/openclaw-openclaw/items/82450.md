## Summary

Keep open. Current main has persistent sessions, a Control UI session picker, New Chat, and session APIs, but it does not provide the requested first-class linear workspace/timeline mode for blind users; this is a recent accessibility feature request that needs product and UX design.

## What This Changes

Request a first-class Accessibility Linear Workspace Mode with one persistent workspace, task sections/bookmarks, quick jumps, and a screen-reader-friendly timeline/session index.

## Best Possible Solution

Treat this as core accessibility/product design: specify a persistent workspace/timeline experience on top of existing session APIs, validate it with screen-reader users, and then implement it in focused UI/API/docs increments.

## Reproduction Assessment

Not applicable as a bug reproduction. The report describes a workflow accessibility barrier, and source/docs confirm the current product is session-picker/New Chat oriented rather than a first-class linear workspace mode.

## Solution Assessment

Unclear until maintainers choose the UX. Existing session APIs are a good foundation, but the best solution should be a designed accessibility workflow rather than an ad hoc session-list tweak.
