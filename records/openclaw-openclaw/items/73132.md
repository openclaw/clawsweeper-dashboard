## Summary

Keep open. Current main still has the reported macOS managed-stop gap: the LaunchAgent stop path verifies launchd state, but the loaded-service stop flow does not verify that the OpenClaw listener exited or that the configured gateway port became free before reporting success.

## What This Changes

Request that `openclaw gateway stop` on macOS terminate or fail on a lingering gateway listener after launchd stop or bootout, including graceful cleanup, force cleanup when safe, and port-free verification.

## Best Possible Solution

Add a macOS managed-stop postcondition that resolves the effective gateway port, waits for verified OpenClaw listener PIDs to exit after launchd stop or bootout, escalates only for safely attributed gateway processes, and fails with clear diagnostics if the port remains busy or cannot be safely attributed.

## Reproduction Assessment

Yes. The issue gives concrete macOS steps, and current main’s loaded LaunchAgent stop path can be verified from source to skip listener and port checks; a live macOS LaunchAgent smoke is the final runtime confirmation.

## Solution Assessment

Yes. The narrowest maintainable fix is to reuse the existing verified gateway-process and port-inspection helpers after managed LaunchAgent stop rather than redesigning launchd handling or killing arbitrary `node` processes.

## Repair Work Prompt

Fix the macOS managed `openclaw gateway stop` path for https://github.com/openclaw/openclaw/issues/73132. Current main stops the LaunchAgent and verifies launchd state, but does not verify that the OpenClaw gateway listener exited or that the configured port is free before returning success. Add a narrow postcondition for loaded LaunchAgent stop: resolve the effective gateway port from the installed service command/env/config, inspect safely attributed OpenClaw listener PIDs, wait for graceful exit after launchd stop/bootout, escalate only for verified OpenClaw gateway processes if needed, and return a non-zero failure with clear diagnostics if the port remains busy or cannot be safely attributed.
