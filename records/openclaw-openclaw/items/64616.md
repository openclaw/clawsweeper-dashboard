## Summary

## What This Changes

The PR adds ACP task-maintenance logic and regression tests to mark stale running ACP session-store rows lost when their transcript metadata is missing or dangling.

## Best Possible Solution

Update the PR to resolve ACP transcript liveness with resolveSessionFilePathOptions/resolveSessionFilePath using ACP store/session context, cover derived and relative paths, then merge after redacted real behavior proof is added.

## Reproduction Assessment

Yes, at source level. Current main's ACP branch returns Boolean(acpEntry.entry), so an old running ACP session-store row can keep a grace-expired task active.

## Solution Assessment

No, not yet. The direction is narrow, but the implementation should resolve canonical transcript paths instead of treating missing or raw sessionFile strings as definitive.
