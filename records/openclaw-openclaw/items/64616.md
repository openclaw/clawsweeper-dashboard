## Summary

Keep open. Current main still trusts ACP session-store row existence for active ACP tasks, so the PR targets a real remaining gap, but the proposed predicate only catches missing or empty `sessionFile` metadata. It still treats a non-empty path to a deleted transcript as live, which leaves the stated “transcript file is gone” zombie case unresolved.

## What This Changes

The PR adds ACP zombie-session detection in task-registry maintenance for old running ACP metadata with no `sessionFile`, plus regression tests for missing and present `sessionFile` cases.

## Best Possible Solution

Land a narrow task-maintenance fix that treats a stale running ACP row as backing state only when the stored ACP metadata is non-terminal and the resolved transcript backing file still exists or is readable, while preserving existing fail-open behavior for unreadable session stores and leaving non-ACP maintenance paths unchanged.

## Reproduction Assessment

Yes. A focused regression can create an ACP task older than the maintenance grace period with `acp.state: "running"` and a `sessionFile` path pointing to a deleted or nonexistent transcript; the PR head keeps it `running` because it only checks that the path string is present.

## Solution Assessment

No. The proposed change is directionally correct for undefined `sessionFile`, but the narrow maintainable fix must validate the resolved transcript backing file rather than using `sessionFile` truthiness as the liveness signal.
