## Summary

This PR is still a useful focused fix path for ACP zombie task reconciliation, but it should not merge as-is because the transcript liveness check bypasses the existing session-file resolver and real after-fix behavior proof is still missing.

## What This Changes

The PR changes task-registry maintenance to mark stale running ACP session-store rows with missing transcript files as lost and adds regression coverage for stale and live ACP cases.

## Best Possible Solution

Keep this PR open as the canonical fix path, update the implementation to use the existing session-file path helpers, add derived/relative path coverage, then merge only after redacted real behavior proof is added.

## Reproduction Assessment

Yes, at source level. Current main keeps ACP tasks active whenever readAcpSessionEntry returns an entry, while the task contract says active tasks should become lost when authoritative backing state is gone after the grace period.

## Solution Assessment

No, not yet. The fix direction is narrow, but the implementation should resolve transcript liveness through resolveSessionFilePath/resolveSessionFilePathOptions instead of treating missing or raw sessionFile strings as definitive.
