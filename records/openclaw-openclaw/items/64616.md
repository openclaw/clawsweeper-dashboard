## What This Changes

The branch updates task-registry maintenance so stale running ACP tasks with missing or dangling transcript files are treated as lacking backing sessions, and adds regression tests for zombie and live ACP cases.

## Best Possible Solution

## Solution Assessment

Yes for the narrow ACP zombie-session bug. The PR preserves fail-open store-read behavior, gates the new lost transition to stale running ACP metadata, and checks missing/dangling transcript files before treating backing state as absent.
