## Summary

Keep open: current main still has the ACP stale-session bug, and this PR is the focused implementation path, but the latest head is not merge-ready because it bypasses the session-file resolver and lacks real behavior proof.

## What This Changes

The PR updates task-registry maintenance to treat stale running ACP session-store rows with missing transcript files as missing backing sessions and adds regression tests for lost/live ACP cases.

## Best Possible Solution

Resolve ACP transcript liveness through the existing session-file path helpers, add derived/relative path regression coverage, then land after redacted real behavior proof.

## Reproduction Assessment

Yes, at source level. Current main keeps ACP tasks active whenever a session-store entry exists, while the documented lost contract says active tasks should lose status when authoritative backing state is gone after the grace period.

## Solution Assessment

No, not yet. The fix direction is narrow, but the implementation should resolve transcript liveness through resolveSessionFilePath/resolveSessionFilePathOptions instead of treating missing or raw sessionFile strings as definitive.
