## Summary

Keep open. Current main still treats any readable ACP session-store row as backing state, and this PR is a focused, source-backed fix for the stale ACP zombie case; it still needs contributor real-behavior proof and clean exact-head validation before merge.

## What This Changes

The PR changes task-registry maintenance to treat stale running ACP metadata with no existing transcript file as missing backing state and adds regression tests for missing, dangling, and live transcript files.

## Best Possible Solution

Land the narrow ACP task-maintenance fix after the contributor adds real behavior proof and maintainers rerun exact-head checks, while keeping broader CLI zombie-task work tracked at https://github.com/openclaw/openclaw/issues/73121.

## Reproduction Assessment

Yes, source-level. On current main, a stale active ACP task whose ACP store row still exists keeps `hasBackingSession()` true via `Boolean(acpEntry.entry)`, even when transcript metadata is missing or dangling; the PR tests encode that failure path.

## Solution Assessment

Yes for the narrow ACP zombie-session case. The PR preserves the fail-open store-read behavior, gates zombie handling to stale `running` ACP metadata, and treats missing or dangling transcript files as absent backing state; broader CLI liveness remains separate.
