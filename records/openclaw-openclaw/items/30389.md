## Summary

Keep open. Current main already has WebChat file upload and a chat model picker, and it exposes commands/skills through slash commands plus the command palette, but it still lacks the requested categorized composer skill/capability menu and the routing semantics behind categories like finance, writing, image generation, or deep reasoning.

## What This Changes

The issue requests WebChat composer controls for file upload, model switching, and a categorized skill/capability menu.

## Best Possible Solution

Keep the shipped upload and model-picker behavior, then either document slash commands, command palette, and Skills as the intended WebChat capability UX or design a narrow composer menu backed by explicit runtime command/skill metadata.

## Reproduction Assessment

Not applicable as a feature request. Source, docs, tests, and prior discussion give a high-confidence current-state check: upload and model picker exist, while the categorized composer capability menu remains undefined.

## Solution Assessment

Unclear. A composer capability menu could address the remaining UX gap, but maintainers first need to decide the category model, metadata source, routing behavior, and config surface before implementation is clearly the best path.
