## Summary

Keep open. Current main covers the WebChat attachment picker and chat model dropdown, and it has slash-command/command-palette discovery for commands and direct skills, but it still does not provide the requested categorized composer skill/capability menu or an agreed product model for task categories and routing.

## What This Changes

The issue requests WebChat composer UI for file uploads, model selection, and a categorized skill/capability menu.

## Best Possible Solution

Keep the shipped upload and model-picker behavior, then either document the existing slash-command, command-palette, and Skills surfaces as the intended WebChat UX or design a narrow composer capability menu backed by explicit runtime command/skill metadata.

## Solution Assessment
