## What This Changes

The issue reports that configured model fallbacks stop after a model timeout because the request terminates before later fallback models are attempted.

## Best Possible Solution

Add a focused regression for the WebChat or agent-command embedded timeout path and keep fallback attempts invisible to terminal chat state until a candidate succeeds or the full configured chain is exhausted.
