## Summary

Keep open: the ACP diagnostic change is useful and has real log proof, but the PR still has a merge-blocking redaction gap where `formatAcpErrorChain` returns top-level non-Error input before masking secrets.

## What This Changes

The PR preserves ACP runtime error cause chains across lifecycle, gateway, task, and reply surfaces, adds redacted formatting helpers, focused tests, and a changelog entry.

## Best Possible Solution

Keep the narrow ACP diagnostic fix, but route the non-Error `formatAcpErrorChain` branch through safe stringification and `redactSensitiveText`, with a regression test for secret-shaped top-level string input.

## Reproduction Assessment

Yes from source inspection: current main stores or emits only `acpError.message`/`String(err)` on the ACP lifecycle, task, and gateway failure paths, which drops nested cause detail. I did not run a live current-main ACP repro in this read-only review.

## Solution Assessment

No, not quite: the PR’s narrow ACP-boundary approach is the right shape, but every public formatter exit must pass through redaction before merge.
