## Summary

Keep open: this PR targets the right fallback classifier, but it still does not fix the reported `claude-cli/claude-sonnet-4-6` path because the new generic-failure branch sits after the non-GPT model guard; the submitted proof is focused tests/CI rather than live or runtime proof of the exhausted CLI fallback flow.

## What This Changes

The PR imports the generic external-runner failure matcher, bypasses the visible-payload guard for all-generic failure text, treats that case as an empty-result fallback, and adds classifier tests.

## Best Possible Solution

Classify the exact generic external-runner failure before the non-GPT guard, add regression coverage using `claude-cli/claude-sonnet-4-6`, and keep the existing no-retry guards for delivered or policy-blocked output.
