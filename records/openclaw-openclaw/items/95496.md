## Summary

Keep open, but the current patch is not merge-ready: it targets the right classifier area, yet the new generic-failure `empty_result` path still runs after the GPT-5-only guard, so the reported `claude-cli/claude-sonnet-4-6` failure would still return `null` and bypass fallback.

## What This Changes

The branch modifies `classifyEmbeddedAgentRunResultForModelFallback` to treat exact generic external runner failure text as fallback-eligible and adds classifier regression tests.

## Best Possible Solution

Handle exact generic external failure text before the non-GPT guard, or shape CLI generic failures as fallback-safe error/incomplete results, then cover `claude-cli/claude-sonnet-4-6` and add redacted real behavior proof.
