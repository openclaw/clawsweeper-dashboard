## Summary

Keep open: this is a focused, useful bug-fix PR, but the patch is not merge-ready because the reported `claude-cli/claude-sonnet-4-6` path still falls through the existing non-GPT guard and returns `null` instead of advancing fallback.

## What This Changes

The PR changes embedded-agent fallback classification so generic external run failure text can be treated as fallback-eligible, with three regression tests.

## Best Possible Solution

Handle exact generic external runner failure text before the GPT-5-only empty-result guard, or shape the CLI result as an error/incomplete result, and add regression coverage using `claude-cli/claude-sonnet-4-6`.

## Reproduction Assessment

Yes. The issue is source-reproducible: current main wraps CLI `output.text` as a normal payload and the fallback classifier treats visible non-error text as a completed attempt before fallback classification.

## Solution Assessment

No. The intended boundary is plausible, but this implementation does not solve the reported Claude model because the new empty-result path is still after the GPT-5-only guard.
