Land a focused Matrix-only implementation that is default-off, uses supported SDK/runtime seams, preserves command and /steer exemptions, documents the privacy model, covers the #45184-style multi-bot case with tests, and splits release workflow/updater/live-test work into owner-reviewed PRs.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Default AI gates to explicit opt-in:** `extensions/matrix/src/matrix/monitor/index.ts:246-256`
  - body: `participationEnabled` and `freshnessEnabled` resolve to true unless explicitly set false, with AI-backed defaults and zero thresholds. Existing Matrix rooms can therefore start sending room text/context to the configured model without adding the new config blocks, so these controls should default off or require an explicit migration/approval path.
  - confidence: 0.9
- **[P2] Let revise and auto continue after holdback:** `extensions/matrix/src/matrix/monitor/handler.ts:2360-2370`
  - body: When the draft holdback sees newer room activity, every mode except `send-as-is` is suppressed before drafting. That makes `revise` and `auto` skip the freshness resolver entirely, contrary to their documented semantics; restrict the early return to `freshnessMode === "suppress"`.
  - confidence: 0.88
- **[P2] Use SDK runtime imports:** `extensions/matrix/src/matrix/monitor/handler.ts:11-17`
  - body: The Matrix plugin production handler deep-imports core `src/**` modules for simple completion and reply generation even though SDK subpaths already expose these seams. Bundled plugins should stay on `openclaw/plugin-sdk/*` or injected runtime helpers before this can merge.
  - confidence: 0.9
- **[P2] Fix the AI participation schema prompt:** `extensions/matrix/src/matrix/monitor/participation-policy.ts:314`
  - body: The schema string given to the model has malformed optional array field names for `includeAgentIds?` and `excludeAgentIds?`. Because this prompt asks for structured JSON, the malformed keys can degrade or corrupt targeted participation decisions.
  - confidence: 0.8
- **[P2] Remove the deterministic ai-first fallback:** `extensions/matrix/src/matrix/monitor/participation-policy.ts:371-382`
  - body: `resolveParticipationDecision` can parse raw `text` deterministically when `currentDirective` is absent. That leaves a silent fallback path despite the PR contract saying ai-first must not silently fall back to deterministic behavior; remove the raw-text fallback and require callers to pass a resolved directive.
  - confidence: 0.84
