## Summary

This PR adds Matrix participation-policy parsing, room-scoped participation state, latest-visible/freshness tracking, Matrix config/schema/tests/docs, and unrelated release workflow, changelog, updater, gateway-harness, and live-test changes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Default the Matrix AI gates to explicit opt-in:** `extensions/matrix/src/matrix/monitor/index.ts:246-255`
  - body: `participationEnabled` and `freshnessEnabled` both resolve to true unless explicitly set false, with AI-backed defaults and zero member thresholds. Existing Matrix rooms can therefore start sending room text/context to the configured model without adding the new config blocks. Default these controls off unless the operator opts in.
  - confidence: 0.88
- **[P2] Let revise and auto reach the freshness resolver:** `extensions/matrix/src/matrix/monitor/handler.ts:2360-2370`
  - body: When the holdback sees newer room activity, every mode except `send-as-is` is suppressed before drafting. That makes `revise` and `auto` skip the freshness resolver entirely, contrary to their documented semantics. Restrict this early return to `freshnessMode === "suppress"`.
  - confidence: 0.87
- **[P2] Keep Matrix runtime inside plugin SDK seams:** `extensions/matrix/src/matrix/monitor/handler.ts:11-17`
  - body: The Matrix plugin production handler deep-imports core `src/**` simple-completion and reply modules. Bundled plugin code should use `openclaw/plugin-sdk/*` seams or injected runtime helpers, so this needs a documented SDK/runtime boundary before merge.
  - confidence: 0.9
- **[P2] Fix the AI participation schema prompt:** `extensions/matrix/src/matrix/monitor/participation-policy.ts:314`
  - body: The schema string given to the model has malformed optional array field names for `includeAgentIds?` and `excludeAgentIds?`. Because this prompt asks for structured JSON, the malformed keys can reduce or corrupt model output for targeted participation decisions.
  - confidence: 0.78
- **[P2] Remove the hidden deterministic participation fallback:** `extensions/matrix/src/matrix/monitor/participation-policy.ts:378-382`
  - body: `resolveParticipationDecision` can parse raw `text` deterministically when `currentDirective` is absent. That leaves an accidental fallback path despite the PR contract saying ai-first must not silently fall back to deterministic behavior. Remove the raw-text fallback from this resolver.
  - confidence: 0.83
