Overall confidence: 0.86

Full review comments:

- **[P1] Default the AI gates to explicit opt-in:** `extensions/matrix/src/matrix/monitor/index.ts:246-256`
  - body: `participationEnabled` and `freshnessEnabled` are true unless explicitly set to false, with AI-backed defaults and zero thresholds. Existing Matrix rooms can therefore send room context to the configured model without the operator adding the new blocks; make both controls default-off or add an explicit migration/approval path.
  - confidence: 0.88
- **[P2] Let revise and auto pass the holdback gate:** `extensions/matrix/src/matrix/monitor/handler.ts:2360-2371`
  - body: When `draftHoldbackMs` observes newer room activity, this branch suppresses every mode except `send-as-is`. That makes `revise` and `auto` skip drafting instead of letting the freshness resolver revise or choose an allowed action; restrict this early suppression to `freshnessMode === "suppress"`.
  - confidence: 0.87
- **[P2] Import reply and model helpers through the SDK:** `extensions/matrix/src/matrix/monitor/handler.ts:11-17`
  - body: The Matrix plugin production handler imports simple-completion and reply helpers through relative `src/**` paths. Bundled plugin code should use supported `openclaw/plugin-sdk/*` seams, and current main already exposes `simple-completion-runtime` and `reply-runtime` subpaths for these helpers.
  - confidence: 0.9
- **[P2] Fix the AI participation schema prompt:** `extensions/matrix/src/matrix/monitor/participation-policy.ts:314`
  - body: The schema string sent to the model has malformed optional field names for `includeAgentIds?` and `excludeAgentIds?`, leaving the quoted keys unterminated. Since the model is being asked to return JSON, keep the schema text syntactically clear so those arrays are not under-produced or misread.
  - confidence: 0.8
- **[P2] Remove the raw-text deterministic fallback:** `extensions/matrix/src/matrix/monitor/participation-policy.ts:378-382`
  - body: `resolveParticipationDecision` can parse raw `text` deterministically when no `currentDirective` is supplied. That leaves a silent fallback path in the added helper even though the PR contract says `ai-first` must not fall back to deterministic parsing; require callers to pass a resolved directive instead.
  - confidence: 0.78
