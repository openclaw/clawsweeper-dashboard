Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Default Matrix AI gates to explicit opt-in:** `extensions/matrix/src/matrix/monitor/index.ts:246-256`
  - body: `participationEnabled` and `freshnessEnabled` resolve to true unless explicitly set false, with AI-backed defaults and zero thresholds. Existing Matrix rooms could start sending room text/context to the configured model without adding the new config blocks, so these controls need to be default-off or have an explicit maintainer-approved migration path.
  - confidence: 0.9
- **[P2] Let revise and auto continue past holdback:** `extensions/matrix/src/matrix/monitor/handler.ts:2360-2370`
  - body: When holdback sees newer room activity, every mode except `send-as-is` is suppressed before drafting. That makes `revise` and `auto` skip the resolver/final-action path documented by the config; restrict the early return to `freshnessMode === "suppress"`.
  - confidence: 0.88
- **[P2] Use plugin SDK runtime imports:** `extensions/matrix/src/matrix/monitor/handler.ts:11-17`
  - body: The Matrix plugin production handler deep-imports core `src/**` modules for simple completion and reply generation. Bundled plugins should use `openclaw/plugin-sdk/*` seams or injected runtime helpers, so this needs to move behind the documented SDK/runtime boundary before merge.
  - confidence: 0.9
- **[P2] Fix the participation schema prompt:** `extensions/matrix/src/matrix/monitor/participation-policy.ts:314`
  - body: The schema string given to the model has malformed optional array field syntax for `includeAgentIds?` and `excludeAgentIds?`. Because this prompt requests structured JSON, the malformed keys can degrade targeted participation decisions or cause those arrays to be omitted.
  - confidence: 0.8
- **[P2] Remove the deterministic ai-first fallback:** `extensions/matrix/src/matrix/monitor/participation-policy.ts:378-382`
  - body: `resolveParticipationDecision` still parses raw `text` deterministically when `currentDirective` is absent. That leaves a silent fallback path despite the PR contract saying ai-first must not fall back to deterministic behavior; require callers to pass a resolved directive instead.
  - confidence: 0.84
