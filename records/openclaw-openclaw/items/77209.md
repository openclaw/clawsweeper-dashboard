## Summary

Keep this PR open, but it is not merge-ready: the implementation is broader than the advertised image/PDF-only behavior and the config surface is incomplete for generated metadata/changelog policy.

## What This Changes

Adds an opt-in `agents.defaults.revertAfterImageModel` setting and post-run logic intended to clear auto-sourced session model overrides after image or PDF analysis.

## Best Possible Solution

Gate the opt-in revert on an explicit media-analysis completion signal, leave normal auto fallback recovery untouched, and document the new config contract with regression coverage.

## Reproduction Assessment

Yes for the review finding by source inspection: any successful non-image fallback that persists `modelOverrideSource: "auto"` would satisfy the added reset condition when the new flag is enabled. I did not establish a live current-main failure path because this is a proposed feature patch review.

## Solution Assessment

No. The proposed direction needs a narrower implementation that knows the completed turn was image/PDF analysis, then regenerated config metadata, docs/changelog, and focused tests.
