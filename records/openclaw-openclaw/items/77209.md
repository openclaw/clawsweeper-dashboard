## Summary

Keep open: this feature PR still has blocking correctness issues in the revert path and its proof shows schema/status output rather than an after-fix image or PDF run that actually reverts the session model.

## What This Changes

Adds an opt-in `agents.defaults.revertAfterImageModel` config key plus post-run session logic intended to clear auto model overrides after image/PDF analysis.

## Best Possible Solution

Land a narrow, product-approved implementation that reverts only after verified image/PDF analysis, clears all fallback-owned session state, and updates docs, changelog, tests, and runtime proof.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a feature PR; source and docs establish the existing fallback persistence behavior, but there is no current-main bug report with a failing reproduction path.

## Solution Assessment

Is this the best way to solve the issue? No. The opt-in direction may be viable, but this patch needs to scope the revert to actual image/PDF analysis and persist a complete clear of fallback-owned model and auth state.
