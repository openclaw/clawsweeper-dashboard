## Summary

Keep open: the PR targets a real linked feature request, but the implementation would clear all auto-sourced model fallbacks when the new option is enabled, not only image/PDF analysis fallbacks, which conflicts with the documented failover contract.

## What This Changes

Adds an opt-in `agents.defaults.revertAfterImageModel` config surface and post-run session logic intended to clear automatic model overrides after image/PDF analysis.

## Best Possible Solution

Keep the PR open, but require the revert to be driven by a media/PDF-specific fallback signal and keep generic model failover persistence unchanged.

## Reproduction Assessment

Yes for the PR regression: source inspection shows the new condition matches any `modelOverrideSource: "auto"` session, while current fallback code uses that source for ordinary model failover. A live image/PDF run was not needed to establish that overmatch.

## Solution Assessment

No; this is not the best implementation because it clears generic auto fallback state instead of proving that the override came from image/PDF analysis. A safer fix would carry or derive a media-analysis-specific marker and add regression coverage for both media and non-media fallbacks.
