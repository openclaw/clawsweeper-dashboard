## Summary

Keep open: this is a useful implementation attempt for #77090, but the runtime guard is too broad and would undo ordinary auto failover state after non-media turns; the PR also misses required config-doc baseline and changelog follow-through.

## What This Changes

The PR adds an opt-in `agents.defaults.revertAfterImageModel` setting and runtime logic to clear auto-sourced session model overrides after a turn completes, with config schema/help/label updates.

## Best Possible Solution

If maintainers want this feature, make the revert media-specific by recording or consulting fallback origin, preserve normal text failover pinning, and add regression coverage plus the required config docs and changelog updates.

## Reproduction Assessment

Yes, for the PR defect from source inspection: any successful turn with `agents.defaults.revertAfterImageModel === true` and an auto model override reaches the new revert branch, with no image/PDF guard. I did not execute the PR branch, but current docs/source make the non-media failover regression clear.

## Solution Assessment

No. The narrow maintainable solution would only clear fallbacks owned by image/PDF analysis, likely by tagging fallback origin or checking the actual media-analysis turn context, then updating generated config baseline, changelog, and tests.
