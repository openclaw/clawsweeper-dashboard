## Summary

Keep this PR open for maintainer review: it targets a real requested workflow from #77090, but the current patch changes the broader model-fallback contract and has schema drift that should block merge.

## What This Changes

The PR adds an opt-in `agents.defaults.revertAfterImageModel` config field and a post-run hook that clears auto-sourced session model overrides, plus config schema labels/help.

## Best Possible Solution

If maintainers want this option, implement it as a media-specific fallback policy that records why an auto override was selected, reverts only after image/PDF analysis, and keeps schema, docs, tests, and changelog aligned.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Source inspection and docs confirm the current behavior: auto fallback overrides remain selected across turns, but I did not run a live image/PDF session in this read-only review.

## Solution Assessment

Is this the best way to solve the issue? No; the patch clears every auto-sourced fallback override when enabled, not only image/PDF analysis overrides, and it introduces schema drift.
