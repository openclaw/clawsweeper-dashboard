## Summary

Keep open for maintainer review: current main does not implement the requested auto-revert, and this PR has blocking issues because it clears all auto-sourced fallback overrides instead of only media-analysis fallbacks and leaves config schema drift.

## What This Changes

The PR adds an opt-in `agents.defaults.revertAfterImageModel` setting, post-run session override clearing, and config schema label/help entries.

## Best Possible Solution

If maintainers want this option, record why an auto fallback was selected and revert only image/PDF-analysis fallbacks, with schema, docs, tests, and changelog aligned.

## Reproduction Assessment

Yes, at source level: current fallback code persists auto model overrides and docs/tests say they remain selected across turns until reset. I did not run a live media-analysis session in this read-only review.

## Solution Assessment

No; this PR clears every auto-sourced fallback override when enabled, not just image/PDF analysis fallbacks, and it introduces config schema drift. A safer fix needs media-specific fallback provenance before clearing the override.
