## Summary

Keep open for maintainer review: the PR targets a real friction point, but the current patch is not merge-ready because it clears all auto fallback overrides, introduces generated-schema drift, and lacks the required changelog entry.

## What This Changes

Adds an opt-in `agents.defaults.revertAfterImageModel` config field and post-run logic to clear auto-sourced session model overrides, plus config schema labels/help.

## Best Possible Solution

If maintainers want this option, record why an auto fallback was selected and revert only media-analysis fallbacks, with generated schema, docs, tests, and changelog kept in sync.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Partially: source and docs confirm auto fallback overrides persist across later turns, but I did not establish a live image/PDF path that sets the session override.

## Solution Assessment

Is this the best way to solve the issue? No; the patch treats every auto fallback as an image/PDF fallback and adds an unsynced schema field, so the safer path is a media-specific fallback marker plus complete config-surface updates.
