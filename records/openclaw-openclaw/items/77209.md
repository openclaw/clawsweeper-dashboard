## Summary

Keep this PR open: the opt-in feature is plausible, but the current patch still clears fallback state too broadly, leaves persisted auth override state behind, and only proves schema output rather than the image/PDF runtime behavior it changes.

## What This Changes

The PR adds an `agents.defaults.revertAfterImageModel` config key, schema metadata, and post-run session logic intended to clear auto model overrides after image/PDF analysis.

## Best Possible Solution

Land a narrow, product-approved implementation that reverts only after verified image/PDF analysis, clears all fallback-owned session state, and updates docs, changelog, tests, and runtime proof.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a feature PR; source and docs establish the existing fallback persistence behavior, but there is no existing bug report with a failing current-main reproduction path.

## Solution Assessment

Is this the best way to solve the issue? No. The opt-in direction may be viable, but the patch needs to scope the revert to actual image/PDF analysis and persist a complete clear of fallback-owned model and auth state.
