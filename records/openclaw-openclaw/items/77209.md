## Summary

Keep open; the PR targets a useful feature request, but the current implementation is not mergeable because it clears generic automatic model-fallback state, not only image/PDF-analysis fallback state.

## What This Changes

The PR adds an opt-in `agents.defaults.revertAfterImageModel` setting and reply-run logic to clear automatic session model overrides after a run, plus config schema label/help metadata.

## Best Possible Solution

Add a media-scoped revert path that only clears auto overrides tied to successful image/PDF analysis, while preserving generic model fallback behavior, with docs, changelog, and regression tests.

## Reproduction Assessment

Yes, at source level. With the new option enabled, any non-media model fallback that writes `modelOverrideSource: "auto"` would be cleared by the added generic end-of-run branch because it has no image/PDF guard.

## Solution Assessment

No. The requested behavior should be implemented with a media/PDF-specific signal rather than treating all automatic fallback overrides as image-analysis overrides.
