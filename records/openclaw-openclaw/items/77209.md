## Summary

Keep open: the feature direction may be useful, but the current patch clears auto fallback state too broadly, leaves persisted auth fallback state behind, and lacks runtime image/PDF proof plus required docs/changelog coverage.

## What This Changes

Adds an `agents.defaults.revertAfterImageModel` config option and post-run session logic intended to clear auto model overrides after image/PDF analysis.

## Best Possible Solution

Land a narrow, product-approved implementation that reverts only after verified image/PDF analysis, clears all fallback-owned session state, and updates docs, changelog, tests, and live proof.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a feature PR; current source and docs establish the existing auto fallback persistence contract, but there is no existing bug report with runtime reproduction steps.

## Solution Assessment

Is this the best way to solve the issue? No. The opt-in direction may be viable, but the patch must be scoped to actual image/PDF analysis and persist a complete clear of fallback-owned model and auth state.
