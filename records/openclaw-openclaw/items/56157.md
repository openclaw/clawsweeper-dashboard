Full review comments:

- **[P2] Document announceTimeoutMs as config, not a tool arg:** `docs/tools/subagents.md:85`
  - body: The added `announceTimeoutMs` bullet presents this as a `sessions_spawn` parameter and gives a `90000ms` default, but current main resolves it from config and falls back to `120000` ms. Publishing this would document the wrong API surface and default.
  - confidence: 0.94
- **[P2] Keep streamTo scoped to ACP spawns:** `docs/tools/subagents.md:86`
  - body: The new `streamTo` bullet reads as a general sub-agent option, but current `sessions_spawn` only exposes it for ACP and says native sub-agent spawns ignore it. Mark it ACP-only or keep it in ACP-specific docs.
  - confidence: 0.92
- **[P2] Apply the depth rule to depth 2 too:** `docs/tools/subagents.md:174-176`
  - body: The updated table adds a general `maxSpawnDepth > current depth` rule for deeper rows while leaving depth 2 as `Never`. Current runtime and tests allow a depth-2 caller to spawn when `maxSpawnDepth=3`, so the table remains contradictory.
  - confidence: 0.93
- **[P2] List always-denied tools separately:** `docs/tools/subagents.md:248`
  - body: The revised tool-policy text still only enumerates session tools, but current policy always denies `gateway`, `agents_list`, `session_status`, `cron`, and `sessions_send`, then applies leaf-only child-control denies. Readers need those sets separated.
  - confidence: 0.9
- **[P2] Remove the nonexistent session info command:** `docs/tools/index.md:157`
  - body: The owner-only tools section tells readers to run `openclaw session info`, but current CLI descriptors expose `openclaw sessions` and repository search found no singular `session info` command. This setup instruction would fail as written.
  - confidence: 0.86
