## What This Changes

## Best Possible Solution

- **[P1] Do not suspend autonomy for missing tokens:** `src/infra/aama-spine-controls.ts:682-690`
  - body: When approvalToken or approvalNonce is absent, this branch calls suspendAutonomy and persists the global suspension state. A stale client, local dev call, or misconfiguration can therefore permanently block later valid approved sends; block and audit the current action instead, and reserve suspension for explicit bypass or invalid-token cases.
  - confidence: 0.9
- **[P1] Forward approval context to gateway sends:** `src/infra/outbound/message.ts:194`
  - body: sendMessage validates the AAMA token locally, but the later gateway send request still carries only the existing message fields. With gateway-side enforcement enabled, an approved gateway send can consume the one-time token locally and then fail at the gateway for missing approvalToken/approvalNonce, so the RPC params need the approval and actor context.
  - confidence: 0.88
- **[P1] Forward approval context to gateway polls:** `src/infra/outbound/message.ts:327`
  - body: sendPoll adds local AAMA enforcement but does not add agentId, approvalToken, or approvalNonce to the gateway poll params. Gateway-side enforcement will see the request as system/no-token and reject otherwise valid approved polls, so the poll RPC needs the same approval context propagation.
  - confidence: 0.88
- **[P2] Restrict AAMA gating to outbound actions:** `src/infra/outbound/message-action-runner.ts:721-735`
  - body: handlePluginAction applies the AAMA outbound guard to every plugin action, while buildActionType maps unknown actions to external_followup_send. Read-oriented actions such as search/list-style channel tools can unexpectedly require external-send approval and trigger fail-closed behavior; gate only side-effecting outbound actions or use explicit per-action policy mapping.
  - confidence: 0.82
- **[P3] Honor actionBlocked before writing memory:** `src/hooks/bundled/session-memory/handler.ts:337-348`
  - body: enforceAamaMemoryGateWrite can return actionBlocked for the contradiction hard-limit path, but this caller only checks quarantineRequired and then writes the file. Even though this specific record currently sets contradiction_group_id to null, the exported decision contract should be respected before more callers depend on it.
