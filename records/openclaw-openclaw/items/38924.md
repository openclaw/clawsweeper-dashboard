## Summary

Keep open. The branch adds a broad security/policy feature that is not already on main, but it has blocking correctness and security issues, failed protocol validation, merge conflicts, and no after-fix real behavior proof.

## What This Changes

This PR adds AAMA Phase 1 enforcement with a durable HMAC/SQLite approval-token policy gate wired into outbound send/poll/message-action and session-memory paths, plus gateway protocol fields and regression fixtures/tests.

## Best Possible Solution

Land this only after maintainer-approved AAMA scope is documented, missing approval is a non-destructive per-action block, approval context is propagated through gateway clients, only outbound-mutating actions are gated, generated protocol/docs/tests are synced, and real behavior proof is attached.

## Reproduction Assessment

Yes for the blocking PR findings by source inspection: the missing-token path persists suspension, and send/poll gateway calls omit approval context after local enforcement. I did not run branch tests because this review is read-only.

## Solution Assessment

No. The proposed direction needs a narrower security design and protocol follow-through before it is a maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Avoid suspending autonomy for a missing approval token:** `src/infra/aama-spine-controls.ts:682-690`
  - body: When enforcement is enabled, an omitted approval token or nonce calls `suspendAutonomy` and persists `autonomy_suspended` before throwing. The added test confirms this makes a later request with a valid token fail, so a misconfigured caller can permanently block outbound traffic instead of only blocking that one action.
  - confidence: 0.94
- **[P1] Forward approval context on gateway sends:** `src/infra/outbound/message.ts:197-203`
  - body: `sendMessage` enforces and consumes the approval token before reaching the gateway path, but the `callMessageGateway({ method: "send" })` payload later omits the token and nonce. Gateway-delivery channels with Phase 1 enabled will reject the already-approved send, and the one-time token has already been consumed locally.
  - confidence: 0.9
- **[P1] Forward poll actor and approval context to the gateway:** `src/infra/outbound/message.ts:330-337`
  - body: The poll path performs the local AAMA check with `actor` and `actionParams`, then sends a gateway poll request without `agentId`, approval token, or approval nonce. A gateway-side check will default the actor to `system` or report the token missing, breaking approved polls.
  - confidence: 0.9
- **[P2] Gate only actions that actually send externally:** `src/infra/outbound/message-action-runner.ts:728-738`
  - body: `handlePluginAction` now runs the outbound guard for every plugin action, while `buildActionType` maps every non-send/broadcast/poll action to `external_followup_send`. Read-only actions such as `search`, `list-pins`, or `member-info` will unexpectedly require send approval and can trigger the suspension path on a missing token.
  - confidence: 0.86
- **[P2] Regenerate the gateway protocol artifacts:** `src/gateway/protocol/schema/agent.ts:48-59`
  - body: The PR adds new send/poll protocol fields, but the live PR status shows `pnpm protocol:check` failed. Protocol schema changes need the generated/client protocol outputs kept in sync before this can merge.
  - confidence: 0.78

## Security Review

Status: needs_attention

Summary: The diff introduces security-sensitive approval enforcement and has concrete availability and token-propagation risks.

Concerns:

- **[high] Missing approval can persistently suspend all autonomy:** `src/infra/aama-spine-controls.ts:682`
  - body: A missing token or nonce is treated like an approval-token bypass and persists global suspension state, creating a denial-of-service path from ordinary misconfiguration or outdated callers.
  - confidence: 0.94
- **[medium] One-time approval tokens are consumed before gateway forwarding:** `src/infra/outbound/message.ts:197`
  - body: Local send/poll enforcement can consume an approval token before making a gateway call that lacks the same approval context, causing the remote enforcement point to fail and burning the token.
  - confidence: 0.9
