Full review comments:

- **[P1] Stop suspending autonomy when tokens are absent:** `src/infra/aama-spine-controls.ts:682-690`
  - body: A missing approvalToken or approvalNonce calls suspendAutonomy, so caller drift or a stale client can durably block later valid outbound actions. Block and audit the current action, and reserve global suspension for explicit bypass or invalid-token cases.
  - confidence: 0.9
- **[P1] Forward approval fields on gateway sends:** `src/infra/outbound/message.ts:197-203`
  - body: sendMessage consumes the AAMA context locally, but the later gateway send payload omits approvalToken and approvalNonce. Gateway-side enforcement can then reject an approved send after the one-time token has already been consumed.
  - confidence: 0.88
- **[P1] Forward poll actor and approval fields to the gateway:** `src/infra/outbound/message.ts:330-336`
  - body: sendPoll validates the local AAMA context, but the gateway poll request omits agentId, approvalToken, and approvalNonce. Gateway-side enforcement will default the actor to system or see no token, rejecting otherwise approved polls.
  - confidence: 0.88
- **[P2] Scope AAMA gating to outbound side effects:** `src/infra/outbound/message-action-runner.ts:728-738`
  - body: handlePluginAction now runs the outbound AAMA guard for every plugin action while unknown actions map to external_followup_send. Read-oriented plugin actions can unexpectedly require external-send approval and fail closed.
  - confidence: 0.82

  - confidence: 0.86
