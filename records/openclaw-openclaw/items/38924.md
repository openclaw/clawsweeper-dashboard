## Summary

Keep open. Current main still lacks AAMA, but this PR is not mergeable as written because source review shows durable suspension on missing approval tokens, approval context not propagated through gateway send/poll calls, broad plugin-action gating, and no after-fix real behavior proof.

## What This Changes

The PR adds AAMA Phase 1 approval-token enforcement across outbound send/poll paths, gateway schemas/handlers, session-memory writes, durable SQLite policy state, fixtures, and tests.

## Best Possible Solution

Revise AAMA as a maintainer-owned security/protocol design, then land a smaller implementation that avoids global suspension for missing tokens, forwards approval context end-to-end, scopes enforcement to outbound side effects, documents the contract, and includes real runtime proof.

## Reproduction Assessment

Yes for the PR findings from source inspection of head 0d0a566af87749153d71a46c0985cd5522a1ae1d. The control flow and branch tests show missing-token suspension, omitted gateway approval fields, broad plugin-action gating, and unhandled memory actionBlocked semantics.

## Solution Assessment

No. The current branch combines a broad new security feature, durable state, gateway protocol changes, and unsettled enforcement semantics, so a narrower maintainer-reviewed design is the safer path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Stop suspending autonomy when approval tokens are absent:** `src/infra/aama-spine-controls.ts:682-690`
  - body: A missing approvalToken or approvalNonce calls suspendAutonomy, so a stale client or misconfiguration can durably block later valid outbound actions. Block and audit the current action, and reserve global suspension for explicit bypass or invalid-token cases.
  - confidence: 0.9
- **[P1] Forward approval fields on gateway sends:** `src/infra/outbound/message.ts:197-203`
  - body: sendMessage enforces AAMA locally, but gateway-delivery sends still omit approvalToken and approvalNonce from the RPC payload. With gateway-side enforcement enabled, an approved send can consume the local one-time token and then fail at the gateway as missing approval context.
  - confidence: 0.88
- **[P1] Forward poll actor and approval fields to the gateway:** `src/infra/outbound/message.ts:330-336`
  - body: sendPoll validates the local AAMA policy context, but the later gateway poll request omits agentId, approvalToken, and approvalNonce. Gateway-side enforcement will treat the request as system/no-token and reject otherwise approved polls.
  - confidence: 0.88
- **[P2] Scope AAMA gating to outbound side effects:** `src/infra/outbound/message-action-runner.ts:728-734`
  - body: handlePluginAction now runs the outbound AAMA guard for every plugin action, while unknown actions map to external_followup_send. Read-oriented plugin actions can unexpectedly require external-send approval and trigger fail-closed behavior.
  - confidence: 0.82
- **[P3] Honor actionBlocked before writing memory:** `src/hooks/bundled/session-memory/handler.ts:338-347`
  - body: enforceAamaMemoryGateWrite can return actionBlocked for hard-limit contradiction backlog cases, but the session-memory caller only checks quarantineRequired and then writes the file. Respect the block decision before this exported decision contract gains additional callers.
  - confidence: 0.72
