## Summary

Keep this PR open: current main does not contain the AAMA surface, so the branch is not obsolete, but the proposed patch still has blocking correctness and security-sensitive policy issues that make it unsafe to merge as-is.

## What This Changes

The PR adds an AAMA Phase 1 approval-token policy gate across outbound send/poll paths, gateway protocol/server handlers, session-memory writes, durable policy state, fixtures, and regression tests.

## Best Possible Solution

Define AAMA as a maintainer-owned security and protocol contract, then land a smaller revised branch that compiles, avoids durable suspension for missing tokens, propagates approval context end-to-end, scopes enforcement to intended outbound side effects, and documents/tests the public contract.

## Reproduction Assessment

Yes for the PR findings. The diff shows a TypeScript-shape mismatch in executePollAction, missing-token suspension, and missing gateway approval-context propagation; I did not run tests because this review was read-only.

## Solution Assessment

No. The current PR is not the best mergeable solution because it combines a broad new security feature, durable state, and protocol changes while leaving blocking compile-time, availability, and policy-integrity issues unresolved.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Use the resolved poll payload before enforcing AAMA:** `src/infra/outbound/outbound-send-service.ts:211-218`
  - body: The new executePollAction guard reads params.to, params.question, params.options, and related fields, but this function only receives ctx and resolveCorePoll. This should fail strict TypeScript and prevents the PR from building until the guard uses the resolved corePoll values.
  - confidence: 0.92
- **[P1] Do not suspend autonomy for missing tokens:** `src/infra/aama-spine-controls.ts:682-690`
  - body: When approvalToken or approvalNonce is absent, this branch persists global autonomy suspension. A stale client or misconfiguration can therefore block later valid approved sends; block and audit only the current action, and reserve suspension for explicit bypass or invalid-token cases.
  - confidence: 0.9
- **[P1] Forward approval context to gateway sends:** `src/infra/outbound/message.ts:194`
  - body: sendMessage validates AAMA locally, but the later gateway send request still omits approvalToken and approvalNonce. With gateway-side enforcement enabled, an approved send can consume the one-time token locally and then fail at the gateway for missing approval context.
  - confidence: 0.88
- **[P1] Forward approval context to gateway polls:** `src/infra/outbound/message.ts:327`
  - body: sendPoll adds local AAMA enforcement but does not include agentId, approvalToken, or approvalNonce in the gateway poll params. Gateway-side enforcement will see the request as system/no-token and reject otherwise valid approved polls.
  - confidence: 0.88
- **[P2] Restrict AAMA gating to outbound actions:** `src/infra/outbound/message-action-runner.ts:721-735`
  - body: handlePluginAction applies the AAMA outbound guard to every plugin action while unknown actions map to external_followup_send. Read-oriented channel actions can unexpectedly require external-send approval and trigger fail-closed behavior.
  - confidence: 0.82
- **[P3] Honor actionBlocked before writing memory:** `src/hooks/bundled/session-memory/handler.ts:337-348`
  - body: enforceAamaMemoryGateWrite can return actionBlocked, but this caller only checks quarantineRequired before writing. Respect the block decision before future callers depend on that exported decision contract.
  - confidence: 0.72

## Security Review

Status: needs_attention

Summary: The diff introduces security-sensitive approval enforcement and durable suspension state with concrete availability and policy-integrity concerns.

Concerns:

- **[high] Missing token can permanently suspend autonomy:** `src/infra/aama-spine-controls.ts:682`
  - body: A missing approval token is treated like an approval-token bypass and persists global suspension, allowing client drift or misconfiguration to create durable denial of outbound actions until a recovery path exists.
  - confidence: 0.9
- **[medium] Gateway token propagation breaks policy integrity:** `src/infra/outbound/message.ts:194`
  - body: The PR adds gateway-side approval fields and local token consumption, but gateway client calls do not propagate those fields, so valid approved sends or polls can be rejected after consuming a one-time token.
  - confidence: 0.86
