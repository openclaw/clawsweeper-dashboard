## Summary

Keep open: current main still lacks AAMA, and the PR remains a broad security/protocol change with unresolved availability, propagation, generated-protocol, and proof blockers.

## What This Changes

The PR adds AAMA Phase 1 approval-token enforcement across outbound send/poll paths, gateway schema/server handlers, session-memory writes, durable SQLite-backed policy state, fixtures, and regression tests.

## Best Possible Solution

Define AAMA as a maintainer-owned security/protocol contract, then land a smaller revised implementation that avoids durable suspension for missing tokens, propagates approval context end-to-end, updates generated protocol artifacts/docs, scopes enforcement to side-effecting outbound actions, and includes real runtime proof.

Yes for the PR findings from source inspection. PR head 0d0a566af87749153d71a46c0985cd5522a1ae1d shows missing-token suspension, missing gateway approval-context propagation, broad plugin-action gating, unhandled memory actionBlocked semantics, and a failed protocol generated-artifact check.

## Solution Assessment

No. The current branch is not the best mergeable solution because it combines a broad new security feature, durable state, and protocol changes while leaving blocking availability, policy-integrity, generated-protocol, and proof issues unresolved.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Stop suspending autonomy when approval tokens are absent:** `src/infra/aama-spine-controls.ts:682-690`
  - body: A missing approvalToken or approvalNonce calls suspendAutonomy, so a stale client or misconfiguration can durably block later valid outbound actions. Block and audit the current action, and reserve global suspension for explicit bypass or invalid-token cases with a recovery story.
  - confidence: 0.9
- **[P1] Forward approval fields on gateway sends:** `src/infra/outbound/message.ts:197-203`
  - body: sendMessage enforces AAMA locally, but gateway-delivery sends still omit approvalToken and approvalNonce from the RPC payload. With gateway-side enforcement enabled, an approved send can consume the local one-time token and then fail at the gateway as missing approval context.
  - confidence: 0.88
- **[P1] Forward poll actor and approval fields to the gateway:** `src/infra/outbound/message.ts:330-336`
  - body: sendPoll validates the local AAMA policy context, but the later gateway poll request omits agentId, approvalToken, and approvalNonce. Gateway-side enforcement will treat the request as system/no-token and reject otherwise approved polls.
  - confidence: 0.88
- **[P2] Refresh generated protocol artifacts:** `src/gateway/protocol/schema/agent.ts:48-59`
  - body: Adding fields to SendParamsSchema and PollParamsSchema changes the Gateway wire contract, but the generated JSON and Swift protocol artifacts are not part of the PR; the protocol check is already failing and native clients will not see the new fields.
  - confidence: 0.86
- **[P2] Scope AAMA gating to outbound side effects:** `src/infra/outbound/message-action-runner.ts:728-734`
  - body: handlePluginAction now runs the outbound AAMA guard for every plugin action, while unknown actions map to external_followup_send. Read-oriented plugin actions can unexpectedly require external-send approval and trigger fail-closed behavior.
  - confidence: 0.82
- **[P3] Honor actionBlocked before writing memory:** `src/hooks/bundled/session-memory/handler.ts:338-347`
  - body: enforceAamaMemoryGateWrite can return actionBlocked for hard-limit contradiction backlog cases, but the session-memory caller only checks quarantineRequired and then writes the file. Respect the block decision before this exported decision contract gains additional callers.
  - confidence: 0.72
