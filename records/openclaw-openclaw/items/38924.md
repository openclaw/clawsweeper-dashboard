## Summary

Keep this PR open, but it is not mergeable as-is: it adds a broad security/gateway protocol contract with blocking availability and policy-propagation bugs, red protocol/secrets checks, and no real behavior proof from a live setup.

## What This Changes

This PR adds AAMA Phase 1 approval-token enforcement for outbound sends, gateway send/poll requests, session-memory writes, durable policy state, fixtures, and regression tests.

## Best Possible Solution

Define AAMA as a maintainer-owned security and gateway protocol contract, then land a smaller revised implementation that avoids durable suspension for missing tokens, propagates approval context end to end, scopes enforcement to side-effecting outbound actions, commits generated protocol artifacts, and includes real runtime proof.
