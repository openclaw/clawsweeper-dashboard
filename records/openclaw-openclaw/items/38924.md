  - body: When approvalToken or approvalNonce is absent, this branch calls suspendAutonomy and persists the global autonomy_suspended flag. A stale client, local dev call, or misconfiguration can therefore permanently block later valid approved sends; block and audit the current action, and reserve suspension for explicit bypass or invalid-token cases.
  - confidence: 0.9

  - body: sendMessage now validates the AAMA token locally, but the later gateway send request still carries only the existing message fields. With gateway-side enforcement enabled, an approved gateway send can consume the one-time token locally and then fail at the gateway for missing approvalToken/approvalNonce, so the RPC params need to include the approval and actor context.
  - confidence: 0.88

  - body: sendPoll adds local AAMA enforcement but does not add agentId, approvalToken, or approvalNonce to the gateway poll params. Gateway-side enforcement will see the request as system/no-token and reject otherwise valid approved polls, so the poll RPC needs the same approval context propagation.
  - confidence: 0.88
- **[P2] Restrict AAMA gating to outbound actions:** `src/infra/outbound/message-action-runner.ts:721-735`
  - body: handlePluginAction applies the AAMA outbound guard to every plugin action, while buildActionType maps unknown actions to external_followup_send. Read-oriented actions such as search/list-style channel tools can unexpectedly require external-send approval and trigger fail-closed behavior; gate only side-effecting outbound actions or use explicit per-action policy mapping.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The diff introduces security-sensitive approval enforcement and durable suspension state, and it has concrete availability/integrity concerns before merge.

Concerns:

- **[high] Missing token can permanently suspend autonomy:** `src/infra/aama-spine-controls.ts:682`
  - body: A missing approval token is treated like an approval-token bypass and persists global suspension, which allows ordinary client drift or misconfiguration to create a durable denial of outbound actions until an unspecified recovery path exists.
  - confidence: 0.9
- **[medium] Gateway token propagation breaks policy integrity:** `src/infra/outbound/message.ts:194`
  - body: The PR adds gateway-side approval fields and local token consumption, but the gateway client calls do not propagate those fields, so mixed local/gateway enforcement can reject valid approved sends or polls after consuming a one-time token.
  - confidence: 0.86
