## Summary

Keep open: current main still has a source-level path where heartbeat fallback text made only of the internal stream-error sentinel is normalized as ordinary user-visible output and can be delivered to the configured channel.

## What This Changes

Reports that a heartbeat fallback model can echo repeated internal stream-error placeholder text as a visible Feishu DM instead of completing the heartbeat check or staying non-visible.

## Best Possible Solution

Keep this issue open and add a narrow heartbeat/output guard that treats placeholder-only STREAM_ERROR_FALLBACK_TEXT replies as non-deliverable, with regression coverage that preserves real heartbeat alerts and media.

## Reproduction Assessment

Yes, at source level. A heartbeat reply consisting only of repeated STREAM_ERROR_FALLBACK_TEXT is not recognized by the current heartbeat suppression filters and can flow to sendDurableMessageBatch as normalized.text.

## Solution Assessment

Yes, the best narrow fix is to suppress only placeholder-only internal stream-error heartbeat output near heartbeat/output normalization while preserving real alerts, media, and the persisted sentinel contract.
