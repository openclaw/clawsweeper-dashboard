## What This Changes

The issue asks OpenClaw to prevent repeated internal stream-error sentinel text from being delivered as visible heartbeat output after model fallback.

## Best Possible Solution

Add a narrow heartbeat-runner guard that treats placeholder-only stream-error sentinel output as internal empty or failed heartbeat output before durable channel delivery, while preserving the sentinel for session replay safety.

Yes at source level: a mocked heartbeat reply containing only repeated STREAM_ERROR_FALLBACK_TEXT would pass current normalization and reach durable channel delivery. I did not establish a live Feishu/provider fallback reproduction.

## Solution Assessment

Yes: the narrowest maintainable fix is to classify placeholder-only heartbeat output before delivery, not remove the persisted sentinel or add channel/provider-specific behavior.
