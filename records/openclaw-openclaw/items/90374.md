## What This Changes

The branch changes the browser wait-for-download test suite by replacing three unconditional Windows skips with symlink/hardlink capability probes and Windows junction symlink creation.

## Best Possible Solution

Land the capability-based gating after the probes allocate their own temporary directory and remove it with best-effort cleanup in `finally`.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not by execution in this read-only review; the cleanup problem is source-reproducible because `tempFile` is created before `linkSync`, and the `catch` path returns without deleting it if `linkSync` throws.

## Solution Assessment

Is this the best way to solve the issue? Yes, capability-based skips are a better test boundary than skipping all Windows runs, but the final patch should make the probes allocate and clean up temporary paths robustly.
