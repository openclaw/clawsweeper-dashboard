## Summary

This PR should stay open: current main still has the empty-success chat gap, and the branch is a narrow useful fix, but it still needs real behavior proof and docs/help alignment before merge.

## What This Changes

The PR changes background exec creation so reply-capable chat delivery contexts default `notifyOnExitEmptySuccess` on, while preserving explicit false opt-out and non-chat silent behavior.

## Best Possible Solution

Merge a rebased update that documents the conditional chat default, explicit false opt-out, and unchanged non-chat behavior, with redacted real behavior proof from a chat background exec run.

## Reproduction Assessment

Yes, by source inspection and related real repro context. Current main computes the flag only for explicit true and then drops completed empty-output notifications unless that flag reaches the runtime as true.

## Solution Assessment

Mostly yes. The PR's latest runtime shape is narrower than defaulting all background execs on because it keys off reply-capable delivery context and preserves explicit false, but the user-facing docs/help and real behavior proof still need to catch up.
