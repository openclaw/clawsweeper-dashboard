## Summary

Keep open: current main still has the quiet-success chat gap, and the PR is a narrow, mostly correct implementation, but it is not merge-ready because real behavior proof is missing and the user-facing docs/help still describe the old default.

## What This Changes

The PR defaults empty-success background exec notifications on for reply-capable chat delivery contexts, preserves explicit opt-out and non-chat silence, and adds regression tests plus a changelog entry.

## Best Possible Solution

Merge a rebased update that documents the reply-capable chat default, explicit false opt-out, and unchanged non-chat behavior, with redacted real behavior proof from a chat background exec run.

## Reproduction Assessment

Yes, by source inspection. Current main computes notifyOnExitEmptySuccess only for explicit true and the runtime drops completed empty-output notifications unless that session flag is true.

## Solution Assessment

Mostly yes. The PR's latest runtime change is narrower than a global default-on policy because it keys off reply-capable delivery context and preserves explicit false, but it needs docs/help alignment and real behavior proof before merge.
