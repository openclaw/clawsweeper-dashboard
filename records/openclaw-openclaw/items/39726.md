## What This Changes

The PR makes empty-success background exec completion notifications default on for reply-capable chat delivery contexts, preserves explicit opt-out and non-chat silence, adds exec regression tests, and adds a changelog line.

## Best Possible Solution

Land a rebased version only after maintainers accept the conditional chat default, docs/help describe it with the explicit opt-out, and redacted real chat proof shows completion routing; otherwise keep empty-success notifications explicit-only.

## Reproduction Assessment

Yes, at source level. Current main sets empty-success notifications only when notifyOnExitEmptySuccess is explicitly true, and the runtime suppresses empty successful completions otherwise; the PR changes that default for routed chat contexts.

## Solution Assessment

Not as-is. The narrow conditional default with explicit false preserved is maintainable, but it needs docs/config alignment, conflict resolution, maintainer acceptance of the default change, and real chat proof before merge.
