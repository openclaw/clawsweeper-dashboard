## Summary

Keep open: current main still lacks the chat-scoped empty-success notification default, and this branch remains the focused implementation, but it is not merge-ready because real behavior proof is absent and docs/config help still describe the old default.

## What This Changes

The PR changes exec defaulting and tests so reply-capable chat delivery contexts emit empty-success background completion notifications by default while preserving explicit opt-out and non-chat silence.

## Best Possible Solution

Land a rebased chat-scoped default that preserves explicit false and non-chat silence, updates docs/config help, and includes redacted real chat proof.

## Reproduction Assessment

Yes, source-level. Current main computes notifyOnExitEmptySuccess only from explicit true and the runtime suppresses completed empty-output notifications unless that flag is true; no live chat path was run in this read-only review.

## Solution Assessment

Mostly yes. The chat-scoped default is narrower than the closed global-default direction in https://github.com/openclaw/openclaw/pull/46926 and preserves explicit opt-out/non-chat behavior, but docs/help, rebase, and real behavior proof still need to land.
