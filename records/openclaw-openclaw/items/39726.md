## Summary

Keep open: current main still suppresses empty successful background exec notifications unless explicitly enabled, and this PR is the active narrow chat-scoped fix path, but it still lacks after-fix real chat proof and docs/config-help alignment.

## What This Changes

This PR defaults `notifyOnExitEmptySuccess` on for reply-capable chat delivery contexts, keeps explicit false and non-chat contexts silent, adds regression coverage, and adds a changelog entry.

## Best Possible Solution

Ship a rebased chat-scoped default that respects explicit false/non-chat silence, updates docs/help/type comments, and includes redacted real chat proof for an empty `exit 0` background exec.

## Reproduction Assessment

Yes, source-level: current main defaults `notifyOnExitEmptySuccess` to true only when explicitly configured, and the runtime suppresses completed empty-output notifications otherwise. I did not run a live chat session in this read-only review.

## Solution Assessment

Mostly yes: the chat-scoped default is narrower than the closed global-default PR and preserves explicit false/non-chat silence. It still needs docs/help alignment and real behavior proof before merge.
