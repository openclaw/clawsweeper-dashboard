## Summary

Keep open. Current main and the latest release still route Linux node-host exec through `/bin/sh -lc` and directly return spawn errors, while the open closing-reference PR is the active fix candidate and has not merged.

## What This Changes

Linux headless node exec is reported to fail with `spawn /bin/sh ENOENT` even though connected nodes have a usable shell.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/85543, or an equivalent node-host fallback that keeps approval, allowlist, durable trust, and completion metadata bound to the argv that actually executes.

## Reproduction Assessment
