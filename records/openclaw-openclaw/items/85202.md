## What This Changes

The issue reports Linux headless node `exec host=node` failing with `spawn /bin/sh ENOENT` on connected nodes where `/bin/sh` and `/usr/bin/sh` exist.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/85543 or an equivalent node-host shell fallback that preserves approval, allowlist, audit, and completion metadata for the argv that actually executes.
