## Summary

Keep open. Current main and the latest release still route Linux node exec through `/bin/sh -lc` and directly return the spawn error, while https://github.com/openclaw/openclaw/pull/85543 is the open closing-reference fix candidate and has not merged.

## What This Changes

The issue requests a fix for Linux headless `exec host=node` failing with `spawn /bin/sh ENOENT` on connected nodes that advertise `system.run` and have `/bin/sh` present.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/85543, or an equivalent node-host fallback that binds the executed shell argv before approval, policy, and audit metadata are finalized.

## Reproduction Assessment

Yes for triage, but not as a fresh local run: reporter live evidence plus current source and latest-release inspection show the same `/bin/sh -lc` direct-spawn path that returns `spawn /bin/sh ENOENT`.

## Solution Assessment

Yes: keeping this issue open for the linked fix path is the best current resolution. The implementation should preserve approval and audit metadata parity while adding the shell fallback.
