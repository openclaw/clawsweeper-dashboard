## Summary

Keep open. Current main and the latest release still build Linux node exec as `/bin/sh -lc` and directly spawn that argv, while https://github.com/openclaw/openclaw/pull/85543 is the open closing-reference fix candidate and still needs review/repair before this issue can close.

## What This Changes

The issue reports Linux headless `exec host=node` failing with `spawn /bin/sh ENOENT` on connected nodes that advertise `system.run` and have `/bin/sh` present.

## Best Possible Solution

Repair and land the open canonical fallback PR, or an equivalent node-host fix that binds the selected shell argv before policy, approval, audit, and completion metadata are finalized; then close this issue as implemented.

## Reproduction Assessment

Yes for triage, but not as a fresh local run: the reporter supplied live Linux headless node evidence, and current main plus `v2026.6.6` still show the `/bin/sh -lc` direct-spawn path that matches the error.
