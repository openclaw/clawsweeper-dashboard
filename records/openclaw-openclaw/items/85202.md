## Summary

Keep open. Current main and v2026.6.8 still build Linux node exec as `/bin/sh -lc` and directly spawn that argv, while https://github.com/openclaw/openclaw/pull/85543 is the open unmerged fix candidate and still has a security-boundary review blocker.

## What This Changes

The issue reports Linux headless `exec host=node` failing with `spawn /bin/sh ENOENT` even when connected nodes advertise `system.run` and have `/bin/sh` and `/usr/bin/sh` available.

## Best Possible Solution

Land a narrow node-host shell fallback fix that preserves approval and audit parity, then close this issue as implemented.

## Reproduction Assessment

Yes for triage/source: the reporter live-reproduced this on three Linux user-systemd nodes, and current main still builds `/bin/sh -lc` and directly spawns `argv[0]`. I did not run a fresh paired-node environment in this read-only review.

## Solution Assessment

No for closure: current main and the latest release still lack the fallback. The right path is to repair and land https://github.com/openclaw/openclaw/pull/85543 or an equivalent node-host shell selection fix that preserves approval/audit parity.
