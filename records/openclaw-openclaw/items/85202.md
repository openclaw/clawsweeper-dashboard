## Summary

Keep open. Current main and the latest release still build Linux node exec as `/bin/sh -lc` and return spawn errors without a shell fallback; https://github.com/openclaw/openclaw/pull/85543 is the linked closing fix candidate but is still open and unmerged.

## What This Changes

Linux headless node exec should run commands on connected nodes instead of failing with `spawn /bin/sh ENOENT` when a usable shell exists.

## Best Possible Solution

Land a security-reviewed node-host fallback that retries trusted absolute POSIX shell paths without PATH resolution, then close this issue from the merged PR.

## Reproduction Assessment

Yes. The reporter supplied live paired Linux node commands and current main source still routes Linux node exec through `/bin/sh` with no retry path when spawn returns ENOENT.

## Solution Assessment

Yes. Keeping this issue open for the linked fix PR is the narrowest path because current main and the latest release still lack the fallback, while https://github.com/openclaw/openclaw/pull/85543 directly targets the failing node-host path.
