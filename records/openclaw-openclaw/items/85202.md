## Best Possible Solution

Keep the issue open, reproduce it on a Linux headless node against current main or the latest release, then fix the node-owned execution path with either robust shell selection or clearer spawn/cwd diagnostics while preserving exec approvals and cwd safety.

## Solution Assessment

Unclear until the failing node environment is inspected. The safest direction is to diagnose whether ENOENT is shell-path, cwd, or service-environment related before adding fallback behavior that could weaken existing approval or cwd guarantees.
