## Summary

Keep open: the requested alias-resolution fix is valid, but the patch currently introduces a TypeScript contract error by passing `modelManifestContext` as a nested property and it lacks required after-fix real behavior proof.

## What This Changes

The PR resolves explicit agent run model-only overrides through configured model aliases, updates agent command test mocks, and adds an alias-only regression case.

## Best Possible Solution

Resolve explicit model-only overrides through the existing model alias index using the shared resolver/context shape, preserve provider+model override behavior, and prove the alias CLI path with real redacted output plus focused regression coverage.

## Reproduction Assessment

Yes, from source inspection: configure `agents.defaults.models` with an alias like `groktop`, then run an agent command with only that model override. Current main parses it as the current provider's model instead of resolving the alias index, so the failure path is clear even though this read-only review did not execute tests.

## Solution Assessment

No as submitted. Resolving through the alias index is the right direction, but the helper calls need `...modelManifestContext` and the PR needs real behavior proof before merge.
