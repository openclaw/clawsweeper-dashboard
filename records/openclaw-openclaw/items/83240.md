## Summary

Keep open: the alias-run-override fix targets a real existing model-routing gap, but the current PR head fails type checking because the new resolver calls pass `modelManifestContext` as a nested property, and the external PR still lacks after-fix real behavior proof.

## What This Changes

The branch resolves explicit agent model-only run overrides through the configured model alias index and adds an alias-only regression case in the agent command tests.

## Best Possible Solution

Use the existing alias resolver with the shared manifest context shape for explicit model-only overrides, preserve provider-plus-model behavior, and merge once type checks and real alias-run proof are clean.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: configure `agents.defaults.models` with an alias like `groktop`, then run an agent command with only that model override. Current main parses that value as the current provider's model instead of resolving the alias index; this review did not execute the live command because the checkout is read-only.

## Solution Assessment

No as submitted. Resolving through the alias index is the right direction, but the helper calls must spread `...modelManifestContext` and the PR needs real after-fix behavior proof before merge.
