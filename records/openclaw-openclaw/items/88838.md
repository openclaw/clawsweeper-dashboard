## Summary

Keep open: this protected maintainer tracker still owns an active, upgrade-sensitive core migration. Current main has the file-backed accessor seam and one command/cron adoption slice, but the typed SQLite schema/store, most caller migration, doctor/import flip, and false-takeover fence removal remain unresolved.

## What This Changes

Track a staged migration of core session/transcript runtime state from file-backed JSON/JSONL paths to a permanent SQLite-backed accessor seam.

## Best Possible Solution

Continue the ordered stack: land file-backed caller slices with ratchets, add the typed SQLite store behind the same seam, then release-gate the doctor/import flip and fence removal after beta soak.

## Reproduction Assessment

Yes, source-level. Current main still contains the JSONL fingerprint fence and many direct file-store callers, and linked user reports document the false-takeover/message-loss class, but I did not run a live repro in this read-only review.

## Solution Assessment

Yes, the branch-by-abstraction plan is the best current direction for this broad migration because it separates storage, caller shape, SDK compatibility, and release migration risk. Current main is only partially through that plan, so closing would be premature.
