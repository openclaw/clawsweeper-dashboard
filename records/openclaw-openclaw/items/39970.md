## Summary

Keep open: this is a plausible iMessage outbound bug fix, but the submitted branch is stale and would not fix current main because live iMessage delivery now lives under `extensions/imessage` and still does not carry `audioAsVoice` through that runtime path.

## What This Changes

The PR modifies the former core iMessage outbound adapter/sender and shared delivery tests to route media-bearing `audioAsVoice` payloads through `sendPayload`, suppress audio placeholders, and restore iMessage text chunking.

## Best Possible Solution

Port the behavior into `extensions/imessage`: carry `audioAsVoice` through the live outbound/runtime/sender path, suppress the generic audio placeholder for voice audio, send captions through the configured chunked text path, and add focused regression tests plus redacted live proof.

## Reproduction Assessment

Yes, source-level. Current main can propagate `audioAsVoice` in shared delivery, but live iMessage lacks `sendPayload` and drops the option before `sendMessageIMessage`; I did not run live Messages delivery.

## Solution Assessment

No, not as submitted. The branch edits stale core files and includes a text-only `sendPayload` path that can bypass configured chunking; the maintainable fix is a narrow port to the live iMessage plugin using the existing payload helper semantics.
