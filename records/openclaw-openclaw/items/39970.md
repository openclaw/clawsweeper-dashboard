## Summary

Keep open: the reported iMessage behavior still needs a fix, but this PR targets stale core iMessage files and does not prove real Messages delivery after the change.

## What This Changes

The PR modifies old core iMessage outbound/sender code and shared delivery tests to route media-bearing `audioAsVoice` payloads through payload-adapter semantics, suppress audio placeholders, and restore text chunking.

## Best Possible Solution

Port the behavior into `extensions/imessage`, carry `audioAsVoice` through the live outbound runtime to the sender, suppress voice-audio placeholder text, keep captions on the chunked text path, and add focused tests plus real Messages proof.

## Reproduction Assessment

Yes, source-level. Current main can route `audioAsVoice` to `sendPayload` only for handlers that implement it, while live iMessage has no `sendPayload` handler and the runtime/sender do not accept `audioAsVoice`.

## Solution Assessment

No, not as submitted. The maintainable fix is a narrow port to the live iMessage plugin using the current SDK payload helper semantics rather than reviving the removed core adapter path.
