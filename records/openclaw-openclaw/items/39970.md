## Summary

Keep open: the PR addresses a real iMessage outbound regression, but it targets the old pre-plugin file layout and still has a blocking chunking issue in its custom `sendPayload` path. Current main has related generic `audioAsVoice` delivery plumbing, but legacy iMessage still does not forward `audioAsVoice` into its send runtime.

## What This Changes

The PR adds a custom legacy iMessage payload adapter for `audioAsVoice` media, restores text chunking, suppresses audio placeholder text, and adds outbound regression tests.

## Best Possible Solution

Port the useful behavior to the current `extensions/imessage` plugin adapter, preserving chunking with the shared payload helper style, adding focused regression tests and a changelog entry, without adding native private-API voice-bubble support.

## Reproduction Assessment

Yes, at source level. Current main carries `audioAsVoice` through generic delivery, but the iMessage outbound runtime ignores it and the sender still turns blank audio media into `<media:audio>`.

## Solution Assessment

No, not as currently written. The intended direction is sound, but the PR needs to be ported to the current plugin layout and its text-only `sendPayload` path must preserve chunking.
