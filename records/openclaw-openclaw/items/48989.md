## Summary

Keep this PR open, but do not merge it as-is. The docs-only direction is useful, but the patch still points readers to a disabled issue tracker and overstates the maintained fork as the live source/support home for an npm package whose metadata still points elsewhere.

## What This Changes

The PR updates the claude-max-api-proxy provider docs to distinguish the unchanged npm package from the maintained GitHub fork and issue tracker.

## Best Possible Solution

Revise the provider guide so it accurately separates the npm package, the maintained community fork, and any available support channel, without linking a disabled issue tracker or editing generated locale docs.

## Reproduction Assessment

Yes. Static inspection plus GitHub and npm metadata reproduce the review findings: the PR links to wende issues even though that repo has Issues disabled, and npm metadata still points to the original repository.

## Solution Assessment

No. The narrow docs-only approach is right, but this patch should first remove or replace the disabled issue-tracker link and soften the provenance wording so the fork is not presented as the canonical package source/support home.
