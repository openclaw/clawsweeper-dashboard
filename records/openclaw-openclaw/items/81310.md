## Repair Work Prompt

Update the plugin maintainer documentation for `PluginHookSessionEndReason` shutdown/restart behavior. In `docs/plugins/hooks.md`, explicitly state that `shutdown` and `restart` fire during gateway termination/restart for active sessions under a 2 s total drain budget, that slow handlers can be cut off by process exit, and that plugins should fast-path/early-return, run bounded finalization, or make shutdown writes crash-consistent and partial-write tolerant. Mirror the same contract where `docs/automation/hooks.md` mentions the shutdown drain if needed, and consider a short JSDoc note near `PluginHookSessionEndReason` in `src/plugins/hook-types.ts` if that feeds API docs. Preserve the existing runtime warning behavior unless source review shows it is insufficient; do not add a changelog entry for a docs-only change, because `CHANGELOG.md` already has the migration note.
