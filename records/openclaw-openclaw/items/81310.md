## Repair Work Prompt

Update the public plugin-maintainer documentation for the `PluginHookSessionEndReason` shutdown/restart contract. Document that `shutdown` and `restart` are emitted during gateway termination for still-active sessions under a 2 s total shutdown drain budget; plugin maintainers should fast-path these reasons, perform only bounded finalization, or make writes crash-consistent/partial-write tolerant because slow handlers may be cut off. Prefer `docs/plugins/hooks.md` as the canonical user-facing page and add a concise JSDoc note near `PluginHookSessionEndReason` in `src/plugins/hook-types.ts` if that is treated as the API reference surface. Do not change runtime behavior unless maintainers separately request per-plugin timeout attribution; the current aggregate warning and changelog note already exist. No changelog entry is needed for a docs-only clarification unless maintainers ask for a forward migration note.
