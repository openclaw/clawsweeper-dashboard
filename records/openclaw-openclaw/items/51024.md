## What This Changes

The PR changes cron maintenance to clear stale `runningAtMs` markers using `resolveCronJobTimeoutMs(job)` plus grace/minimum thresholds and adds a short-timeout `agentTurn` regression test.

## Best Possible Solution

Land one rebased timeout-aware stale-marker cleanup that reuses `resolveCronJobTimeoutMs`, preserves current post-clear skip behavior and the conservative unlimited-timeout fallback, and includes redacted real cron/gateway recovery proof.

## Reproduction Assessment

Yes, by source inspection. On current main, an `agentTurn` cron job with `timeoutSeconds: 900` and a persisted `runningAtMs` older than about 16 minutes but younger than two hours remains blocked because cleanup still waits for the fixed two-hour threshold.

## Solution Assessment

Yes for the code direction. Reusing the existing cron timeout policy is the narrow maintainable fix, but this branch is not merge-ready until it is rebased, backed by inspectable real behavior proof, and revalidated.
