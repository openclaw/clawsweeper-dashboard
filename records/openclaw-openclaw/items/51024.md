## Summary

Keep this PR open: current main still uses the fixed two-hour stale running-marker threshold, while this branch is a focused timeout-aware fix with no blocking code finding; it still needs inspectable real behavior proof and maintainer acceptance of the shorter effective stale windows before merge.

## What This Changes

The PR derives cron stale running-marker cleanup from each job's effective timeout plus grace/minimum/fallback handling, logs the resolved threshold, and adds a short-timeout agentTurn regression test.

## Best Possible Solution

Land one rebased timeout-aware stale-marker cleanup that reuses resolveCronJobTimeoutMs, preserves the unlimited-timeout fallback and post-clear skip behavior, and includes redacted real cron/gateway recovery proof.
