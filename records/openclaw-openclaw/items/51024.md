## What This Changes

The PR changes cron maintenance to clear stale `runningAtMs` markers after each job’s resolved timeout plus grace/minimum handling instead of the fixed two-hour threshold, and adds a short-timeout `agentTurn` regression test.

## Best Possible Solution

## Reproduction Assessment

Yes, from source inspection. Current `main` still clears `runningAtMs` only after a fixed two-hour threshold, so an `agentTurn` job with `timeoutSeconds: 900` can remain blocked after its configured timeout window has elapsed.

## Solution Assessment

No, not as currently mergeable. Timeout-aware cleanup is a maintainable direction, but this branch needs a current-main rebase that preserves post-clear skip handling plus maintainer acceptance of the shorter default-timeout recovery windows.
