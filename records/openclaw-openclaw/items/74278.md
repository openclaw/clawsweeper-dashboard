## Summary

Keep open. The patch is a useful loader backstop, but it still does not fix the production image-tool path that resolves managed media URIs under `workspaceDir`, and the new fail-fast branch can reject valid whitespace-prefixed inbound refs before the shared normalizer can resolve them.

## What This Changes

The PR adds a loader-side guard for unresolved `media://` refs, skips loader workspace joins for scheme-bearing inputs, and adds `src/media/web-media.test.ts` regression coverage.

## Best Possible Solution

Normalize managed media references through the shared helper before any workspace join, exempt `media://` refs from the image-tool workspace path branch, keep sandbox bridge handling coordinated with #74186, and add focused regression tests plus an Unreleased changelog fix entry.

## Reproduction Assessment

Yes. The static reproduction is high-confidence: production tool construction passes `workspaceDir`, the image tool resolves `media://inbound/<id>` as a relative workspace path, and Node collapses that URI to a `media:/` directory component before the loader can resolve the media store ref.

## Solution Assessment

No. A loader guard is a useful defense-in-depth change, but the narrow maintainable fix also needs to stop the image tool from pre-joining managed refs and let trimmed valid inbound refs resolve before treating them as stranded.
