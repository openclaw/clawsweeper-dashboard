## Summary

Keep open. The loader changes are useful defense-in-depth, but the normal image-tool path can still workspace-join managed `media://` refs before the loader sees them, so the linked user-facing bug is not fixed yet.

## What This Changes

The PR adds `loadWebMediaInternal()` guards and regression tests to reject unresolved `media://` and other scheme-bearing media references before workspace path resolution.

## Best Possible Solution

Carry one normalized managed-media path through image-tool resolution, the direct loader, and sandbox bridge handling, with regressions for `workspaceDir` and a changelog entry for the user-facing fix.

## Reproduction Assessment

Yes. Source inspection plus the `path.resolve` check give a high-confidence current-main path: `media://inbound/<id>` can be joined under `workspaceDir` and collapse to `media:/inbound/<id>` before image loading.

## Solution Assessment

No. The PR’s loader guard is useful, but the narrow maintainable fix also needs to keep managed media refs out of the image-tool workspace join and normalize inbound refs before declaring them stranded.
