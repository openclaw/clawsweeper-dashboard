## Summary

This PR should stay open: it improves the loader error path and has addressed the earlier Greptile cleanup comments, but it still misses the normal image-tool workspaceDir path for the linked bug and can reject whitespace-prefixed inbound refs before the shared media resolver normalizes them.

## What This Changes

This PR modifies `src/media/web-media.ts` and its tests to fail fast on unresolved `media://` and other scheme-bearing references before loader-level workspace resolution.

## Best Possible Solution

Resolve managed `media://` refs through the shared media-reference contract before any workspace or sandbox path join, with focused loader, image-tool, and sandbox bridge coverage.

## Reproduction Assessment

Yes, at source level. `createOpenClawTools()` passes `workspaceDir` into `createImageTool()`, and the current image-tool branch can still workspace-join `media://inbound/<id>` before the loader sees the managed URI.

## Solution Assessment

No. The loader guard is useful defense-in-depth, but the fix should also prevent image-tool and sandbox pre-resolution and should normalize through the shared resolver before declaring a media URI stranded.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Keep managed media refs out of workspace joins:** `src/media/web-media.ts:387-388`
  - body: The new loader guard only runs if `loadWebMedia()` receives the raw URI. In the normal non-sandbox image-tool path, `workspaceDir` is supplied and `media://inbound/<id>` is resolved as a relative path before the loader, so the malformed `<workspace>/media:/...` path can still occur; exclude media-store refs or schemes from that join and add `workspaceDir` coverage.
  - confidence: 0.9
- **[P2] Resolve normalized inbound refs before failing fast:** `src/media/web-media.ts:387-388`
  - body: This branch runs after the strict raw `resolveMediaStoreUriToPath()` check, so an existing `  media://inbound/<id>` is reported as `not-found` even though `resolveInboundMediaReference()` trims the same value and can hydrate it. Normalize or call the shared resolver before declaring the ref stranded.
  - confidence: 0.88
