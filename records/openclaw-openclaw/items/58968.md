## Summary

Keep open. Current main and the latest release still lack a timeout on the remote-CDP Playwright tab enumeration path, and https://github.com/openclaw/openclaw/pull/80147 is an open closing PR for this issue, so this should remain open until that PR lands or is replaced.

## What This Changes

The issue requests timeout protection and graceful recovery for remote-CDP Playwright page enumeration so gateway browser requests do not hang or return 500 when CDP stalls.

## Best Possible Solution

Review and land, or explicitly reject and replace, the open browser-plugin fix that reuses existing remote-CDP timeout settings and evicts the scoped Playwright connection on timeout.

## Reproduction Assessment

Yes, source-level. A Playwright CDP session whose `newCDPSession`, `send`, or title read never resolves can leave `listPagesViaPlaywright` pending because current main has no operation-level timeout around that path.

## Solution Assessment

Yes. The narrow maintainable solution is the active PR's direction: pass the existing remote-CDP timeout budget into Playwright tab enumeration and drop only the scoped cached connection when that budget expires.
