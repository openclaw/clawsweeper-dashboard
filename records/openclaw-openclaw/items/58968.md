## Summary

Keep open. Current main and the latest release still await Playwright page enumeration operations directly on the remote-CDP tab listing path, and the only PR that added a timeout was closed unmerged. This is a narrow, valid browser-plugin repair candidate rather than a cleanup close.

## What This Changes

The issue reports that remote-CDP browser tab listing can hang in `listPagesViaPlaywright` without a timeout, causing gateway HTTP requests that depend on tab listing to fail.

## Best Possible Solution

Add a bounded timeout around persistent-Playwright page enumeration in the browser plugin, and on timeout evict or disconnect the affected cached Playwright CDP connection so later requests can reconnect instead of repeatedly using the same stuck transport. Keep the fix local to `extensions/browser`, preserve existing SSRF checks and remote-CDP configuration semantics, and add a regression test for a never-resolving CDP session.

## Reproduction Assessment

Yes. The high-confidence reproduction path is a mocked browser whose Playwright CDP session never resolves during `listPagesViaPlaywright`; current main has no timeout around that await, and the closed related PR demonstrated the same scenario with fake timers.

## Solution Assessment

Yes, with one refinement. A timeout wrapper is the right narrow fix, but it should also clear or disconnect the cached Playwright browser connection on timeout to avoid the stuck connection surviving behind a fast rejection.
