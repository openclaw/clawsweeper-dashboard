## Summary

Keep open. Current main and the latest release still route remote Playwright tab listing through `listPagesViaPlaywright` without an operation-level timeout, and an open closing PR already owns the narrow fix path.

## What This Changes

The issue asks for timeout protection around remote-CDP Playwright tab enumeration so a hung CDP connection does not stall gateway HTTP requests.

## Best Possible Solution

Review and land, or explicitly reject and replace, https://github.com/openclaw/openclaw/pull/80147 so the browser plugin reuses existing remote-CDP timeout settings and evicts only the scoped Playwright connection on timeout.

## Reproduction Assessment

Yes, source-level. A Playwright CDP session whose `newCDPSession`, `send`, or title read never resolves can leave `listPagesViaPlaywright` pending because current main has no operation-level timeout around that path.

## Solution Assessment

Yes. The narrow maintainable fix is the open PR's direction: pass the existing remote-CDP timeout budget into Playwright tab enumeration and drop only the scoped cached connection when that budget expires.
