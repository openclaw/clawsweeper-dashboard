## Summary

Keep open: current main and the latest shipped release still route remote Playwright tab listing through an unbounded enumeration path, and the open linked PR is the focused fix path.

## What This Changes

The issue reports that remote CDP page enumeration can hang without a timeout and cause gateway/browser HTTP requests to fail.

## Best Possible Solution

Review, refresh if needed, and land or replace https://github.com/openclaw/openclaw/pull/80147 with a scoped timeout around remote Playwright tab enumeration that reuses existing remote CDP timeout settings and evicts only the stuck scoped connection.

## Reproduction Assessment

Yes, source-level. Remote CDP profiles use the persistent Playwright tab-listing path, and current main still awaits connection, page enumeration, target lookup, title, and URL reads without an operation-level timeout.

## Solution Assessment

Yes. Reusing the existing remote CDP timeout budget and clearing only the scoped Playwright connection on timeout is the narrow maintainable fix, and the linked open PR is already pointed at that solution.
