## Summary

Keep this issue open: current main and the latest release still leave persistent Playwright remote tab enumeration without an operation-level timeout, and the open linked PR is the current fix path rather than a reason to close before merge.

## What This Changes

Remote CDP browser profiles need listPagesViaPlaywright to fail fast when Playwright tab enumeration stalls instead of blocking Gateway HTTP handling.

## Best Possible Solution

Land or replace the linked browser-plugin fix so persistent Playwright remote tab enumeration uses the existing remote CDP timeout budget, evicts a stuck scoped connection, and documents the expanded timeout semantics.

## Reproduction Assessment

Yes, at source level. Current main awaits Playwright CDP target and title reads during persistent remote tab listing without an operation timeout, so a stalled CDP session can keep the request path waiting.

## Solution Assessment

Yes for the fix direction, but it is not landed yet. Reusing the existing remote CDP timeout budget and evicting the scoped stuck Playwright connection is the narrowest maintainable path; the open PR should be reviewed, refreshed, and merged or replaced.
