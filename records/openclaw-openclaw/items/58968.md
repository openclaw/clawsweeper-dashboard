## Summary

Keep open. Current main and the latest release still route remote Playwright tab listing through `listPagesViaPlaywright` without an operation-level timeout, while an open closing PR already owns the narrow fix path.

## What This Changes

Bound browser remote CDP Playwright tab enumeration so a hung CDP connection cannot leave gateway HTTP requests returning 500.

## Best Possible Solution

Review and land, or explicitly reject and replace, https://github.com/openclaw/openclaw/pull/80147 so remote Playwright tab enumeration reuses existing remote-CDP timeout settings and evicts only the scoped Playwright connection on timeout.

## Reproduction Assessment

Yes, source-level. A Playwright CDP session whose `newCDPSession`, `send`, or title read never resolves can leave `listPagesViaPlaywright` pending because current main has no operation-level timeout around that path.

## Solution Assessment

Yes, with one required refresh correction. Reusing existing remote CDP timeout settings and dropping only the scoped Playwright connection is the narrow maintainable fix, but the linked PR must be rebased without losing current-main internal target filtering.
