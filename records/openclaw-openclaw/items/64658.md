## Summary

Keep open. Current main still lacks a documented native `sessions_spawn` caller policy for silent or parent-only completion delivery, while tests and docs preserve top-level external channel delivery; related #13911 is relevant but tracks channel-level suppression rather than this per-spawn control.

## What This Changes

The issue requests per-spawn control over sub-agent completion announcements so channel-bound parent sessions can suppress or internalize child completion delivery.

## Best Possible Solution

Add a documented native sub-agent completion delivery policy that keeps current channel delivery as the default while supporting parent-only internal injection and no-announce completion handling, coordinated with the channel-level policy in #13911.

## Reproduction Assessment

Yes. Current source and regression tests give a high-confidence reproduction path: a top-level channel requester such as BlueBubbles still produces an external `agent` delivery with `deliver=true`, and native `streamTo: "parent"` is explicitly ignored.

## Solution Assessment

Unclear. A per-spawn enum is a plausible narrow solution, but the exact names, interaction with the hidden boolean, and precedence relative to #13911 need maintainer API design before implementation.
