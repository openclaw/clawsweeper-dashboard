## Summary

Keep open. Current main now has an internal `expectsCompletionMessage: false` escape hatch for silent completion cleanup, but `sessions_spawn` still lacks a documented/schema-advertised `announce` or `completionDelivery` policy and has no parent-only mode; related #13911 is adjacent channel-level suppression, not an exact duplicate.

## What This Changes

Request to add per-call `sessions_spawn` control that can suppress or reroute sub-agent completion announcements instead of always delivering them to the inherited channel.

## Best Possible Solution

Expose a documented native sub-agent completion delivery policy that preserves channel delivery as the default, supports silent completion handling, and adds parent-only internal delivery if maintainers accept that API shape.

## Reproduction Assessment

Yes, source-reproducible. Current main shows no public `sessions_spawn` announce policy in schema/docs, while source and tests preserve default external requester-channel delivery and only expose an internal boolean suppression path.

## Solution Assessment

Unclear. The existing `expectsCompletionMessage: false` path is a useful silent-mode building block, but deciding whether to document it, rename it to an enum, and add parent-only semantics should be a maintainer API decision coordinated with #13911.
