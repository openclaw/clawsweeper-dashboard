## Summary

Keep open. Current main still lacks a documented native `sessions_spawn` policy for silent or parent-only sub-agent completion delivery, and the related open #13911 is adjacent channel-level suppression work rather than a terminal duplicate for this per-spawn caller-control request.

## What This Changes

The issue requests a per-call `sessions_spawn` announcement control so sub-agent completions can be suppressed, routed only to the parent session, or delivered to the inherited channel.

## Best Possible Solution

Add a documented native sub-agent completion delivery policy to `sessions_spawn` that preserves current channel delivery as the default while supporting parent-only internal injection and silent completion handling, then align its precedence with any channel-level suppression design tracked in #13911.

## Reproduction Assessment

Yes. Current-main source and regression tests provide a high-confidence reproduction path: native `sessions_spawn` has no public `announce`/`completionDelivery` enum, and top-level BlueBubbles/iMessage-like completion delivery is still expected to use external `deliver: true`.

## Solution Assessment

Unclear. A per-spawn enum is a plausible narrow solution, but the exact naming and precedence relative to #13911's broader channel-level suppression should be settled by a maintainer before implementation.
