## Summary

Keep open. Current main has a partial hidden `expectsCompletionMessage: false` suppression path, but the public `sessions_spawn` schema and docs still lack the requested per-spawn delivery policy, and top-level channel-bound completion delivery still defaults to external channel delivery.

## What This Changes

Add documented per-spawn `sessions_spawn` completion delivery control so channel-bound sub-agents can suppress channel auto-announces or route completion only to the parent session.

## Best Possible Solution

Promote the partial suppression seam into a documented `sessions_spawn` completion delivery policy that preserves channel delivery by default, supports silent completion handling, and deliberately defines parent-only internal delivery in coordination with #13911.

## Reproduction Assessment

Yes. Source inspection and regression tests show native `sessions_spawn` still defaults completion delivery on, top-level channel-bound requesters still resolve to `deliver: true`, and the advertised schema/docs do not expose a `none | parent | channel` policy.

## Solution Assessment

Unclear. A per-spawn enum is a plausible narrow solution, but current main's hidden boolean only covers silent suppression and the public API shape should be decided with #13911 in mind.
