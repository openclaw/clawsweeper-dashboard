## Summary

Keep open. Current main still has no documented native `sessions_spawn` policy for silent or parent-only completion delivery; the existing implementation keeps requester-channel announce as the public default, and #13911 is related but tracks channel-level suppression rather than this per-spawn API control.

## What This Changes

Add caller-controlled completion announcement delivery for native `sessions_spawn`, such as silent, parent-only, or channel delivery modes.

## Best Possible Solution

Add a documented native sub-agent completion delivery policy that keeps current channel delivery as the default while supporting parent-only internal injection and silent completion handling, coordinated with #13911.

## Reproduction Assessment

Yes. Source and regression tests give a high-confidence path: native `sessions_spawn` has no public announce policy, defaults completion delivery on, and current BlueBubbles-like coverage asserts external `deliver: true`.

## Solution Assessment

Unclear. A per-spawn enum is a plausible narrow solution, but naming and precedence with the broader channel-level suppression request in #13911 should be decided by maintainers first.
