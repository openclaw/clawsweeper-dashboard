## Summary

Keep open. Current main still lacks a documented native `sessions_spawn` policy for silent or parent-only completion delivery, while source, docs, and tests preserve external requester-channel delivery; related #13911 is adjacent but tracks channel-level suppression rather than this per-spawn API control.

## What This Changes

The issue requests a per-`sessions_spawn` option to suppress or reroute sub-agent completion announcements away from the inherited channel context.

## Best Possible Solution

Add a documented native sub-agent completion delivery policy that preserves channel delivery as the default while supporting parent-only internal injection and silent completion handling, coordinated with #13911.

## Reproduction Assessment

Yes. Source and regression tests give a high-confidence path: native `sessions_spawn` has no public announce policy, defaults completion delivery on, and current tests assert external `deliver=true` for BlueBubbles-like requester origins.

## Solution Assessment

Unclear. A per-spawn enum is a plausible narrow solution, but the exact names and precedence with channel-level suppression in #13911 need maintainer API design first.
