## Summary

Keep open. Current main still splits a LINE text-plus-table reply into a Reply API text call followed by a Push API Flex message, and the only linked fix PR was closed unmerged after review found it was a no-op.

## What This Changes

The issue requests that LINE text plus Markdown-table Flex output be delivered through the Reply API when it fits, instead of forcing the table through Push API.

## Best Possible Solution

Batch eligible generated Flex/media messages into the first LINE Reply API call before the reply token is consumed, then push only true overflow while preserving quick-reply behavior.

## Reproduction Assessment

Yes. A high-confidence source-level path is to pass text containing a Markdown table through deliverLineAutoReply: processLineMessage emits cleaned text plus a Flex table, current main replies with the text, and then pushes the Flex message separately.

## Solution Assessment

Yes. The narrow maintainable fix is to let sendLineReplyChunks accept extra Flex/media messages for unused Reply API slots and report how many extras were consumed; the closed PR's allowReplyToken-only change was not enough because the token was already consumed.
