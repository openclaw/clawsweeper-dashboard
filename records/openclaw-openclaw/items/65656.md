## Summary

Keep open. Current main and the latest release still split text onto the Reply API and table Flex content onto Push API, while the open closing PR at https://github.com/openclaw/openclaw/pull/81299 owns the focused fix and has not merged.

## What This Changes

A LINE channel bug report where table Flex messages are delivered via Push API instead of the still-available Reply API when text and a Markdown table are returned together.

## Best Possible Solution

Review and land or explicitly reject the focused batching PR at https://github.com/openclaw/openclaw/pull/81299, then close this issue only after the fix is on main.

## Reproduction Assessment

Yes, at source level. Current main extracts Markdown tables into Flex messages, replies with text first, then sends the Flex messages through the Push API path; a live exhausted-quota LINE account would be needed for full end-to-end reproduction.

## Solution Assessment

Yes, the narrow maintainable fix is to batch eligible rich/media messages into the same Reply API call before the reply token is consumed and push only overflow. The open linked PR appears to own that direction, so this issue should stay open for that review path.
