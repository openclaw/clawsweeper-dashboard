## Summary

Keep this open. Current main still converts Markdown tables into separate LINE Flex messages and sends the text reply before forcing the Flex table through Push API, while the only linked fix PR was closed unmerged after review showed it did not change the reachable behavior.

## What This Changes

The issue requests that LINE text-plus-table auto-replies deliver the generated Flex table with the Reply API when reply slots are still available instead of always using Push API.

## Best Possible Solution

Update the LINE plugin delivery path so eligible generated Flex/media messages consume unused Reply API slots before the reply token is spent, then push only overflow while keeping quick-reply ordering intact.

## Reproduction Assessment

Yes, at source level: call deliverLineAutoReply with a reply token and payload text that processLineMessage splits into cleaned text plus a table Flex message. Current main replies with the text, marks the token used, and pushes the Flex table separately; I did not run a live exhausted-quota LINE account.

## Solution Assessment

Yes. The narrow maintainable fix is to batch eligible rich/media extras into sendLineReplyChunks before it consumes the reply token, with overflow pushed afterward; the closed allowReplyToken-only PR was not sufficient because the token was already marked used.
