## Summary

Keep open. Current main and the latest release still send extracted LINE table Flex messages on the Push path after text uses the Reply API path, and the focused linked fix PR remains open and unmerged.

## What This Changes

The issue requests that LINE auto-replies deliver explanatory text plus extracted Markdown table Flex messages without dropping the table through a quota-bound Push API 429 path.

## Best Possible Solution

Review and land or explicitly reject https://github.com/openclaw/openclaw/pull/81299; close this issue only after the selected fix is on main.

## Reproduction Assessment

Yes, at source level: current main extracts Markdown tables into flexMessages, sends text chunks through replyMessageLine, then routes rich/table messages through sendLineMessages with allowReplyToken=false. I did not establish a live exhausted-quota LINE account reproduction.

## Solution Assessment

Yes. The narrow maintainable fix is to batch eligible rich/media messages into the same Reply API call before the reply token is consumed, then push only overflow; the open focused PR owns that direction.
