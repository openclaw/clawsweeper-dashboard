## Summary

Keep open. Current main still has the source-level LINE text-plus-table Reply/Push split, and https://github.com/openclaw/openclaw/pull/81299 is an open closing PR for the same bug, so this issue should stay open until that fix lands or is explicitly rejected.

## What This Changes

The issue asks for LINE replies containing both explanatory text and Markdown-table Flex output to use available Reply API slots before falling back to Push API overflow.

## Best Possible Solution

Review and land or explicitly close https://github.com/openclaw/openclaw/pull/81299; the end state should batch eligible rich/media/text into one Reply API call up to LINE's five-message limit and push only overflow.

## Reproduction Assessment

Yes, by source inspection: current main converts Markdown tables into Flex messages, sends text through sendLineReplyChunks first, and then forces rich/media through Push API. I did not run a live exhausted-quota LINE account.

## Solution Assessment

Yes. The narrow maintainable fix is the direction in https://github.com/openclaw/openclaw/pull/81299: batch eligible rich/media/text before the reply token is consumed and push only overflow, while preserving quick-reply behavior.
