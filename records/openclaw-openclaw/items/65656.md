## Summary

Keep open. Current main still routes LINE text-plus-table replies as a Reply API text batch followed by a Push API Flex batch, and the only linked fix PR was closed unmerged after review showed it did not change the reachable path.

## What This Changes

LINE text-plus-table auto-replies should deliver the generated table Flex message without depending on the Push API quota.

## Best Possible Solution

Update the LINE plugin delivery path so eligible Flex/media extras fill unused Reply API slots before the token is consumed, then push only overflow while preserving quick-reply behavior.

## Reproduction Assessment

Yes, by source inspection: processLineMessage splits table content into flexMessages, deliverLineAutoReply sends text through sendLineReplyChunks first, and current tests expect the Flex message to be pushed separately. I did not run a live exhausted-quota LINE account.

## Solution Assessment

Yes. The narrow maintainable fix is to batch eligible rich/media messages into the same Reply API call before sendLineReplyChunks consumes the token, then push only overflow; the closed allowReplyToken-only PR was not sufficient.
