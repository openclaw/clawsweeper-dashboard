## Summary

Keep open. Current main and v2026.6.6 still split LINE text-plus-table replies into Reply API text followed by Push API Flex/table delivery, and the open closing PR at https://github.com/openclaw/openclaw/pull/81299 is the active repair path.

## What This Changes

The issue reports LINE Markdown table Flex messages being dropped when an agent response contains both explanatory text and a table after Push API quota is exhausted.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81299, or an equivalent narrow LINE delivery fix, so eligible text plus Flex/table messages consume Reply API slots before Push overflow while preserving quick-reply behavior.

## Reproduction Assessment

Yes. Source inspection shows Markdown tables become Flex messages, and current LINE auto-reply delivery sends text through Reply API before pushing rich/table messages, matching the reporter's 429 loss path; I did not run a live exhausted-quota LINE account.

## Solution Assessment

Yes. The narrow maintainable fix is to batch eligible rich/media plus text into the same Reply API opportunity before Push overflow; the open linked PR implements that direction and should be reviewed rather than creating a second fix lane.
