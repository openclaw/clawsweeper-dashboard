## Summary

Keep open. Current main and the latest release still route LINE text-plus-table replies through a Reply API text call followed by Push API rich-message delivery, and https://github.com/openclaw/openclaw/pull/81299 is an open closing PR for the same bug.

## What This Changes

The issue requests that LINE replies containing explanatory text plus a Markdown table deliver the generated Flex table with the text through the Reply API up to LINE's five-message limit instead of pushing the table separately.

## Best Possible Solution

Review and land or explicitly reject https://github.com/openclaw/openclaw/pull/81299; the desired end state is one Reply API batch for eligible rich/media/text up to five messages, with only overflow sent through Push API.

## Reproduction Assessment

Yes, by source inspection: current main extracts Markdown tables into Flex messages, sends text chunks through the Reply API first, then forces rich/media through Push API. A live exhausted-quota LINE account was not exercised in this read-only review.

## Solution Assessment

Yes. The narrow maintainable direction is to batch eligible rich/media/text before the reply token is consumed and push only overflow, which is the approach in the open linked PR.
