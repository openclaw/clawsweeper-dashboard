Full review comments:

- **[P1] Include local settings in provider identity:** `packages/memory-host-sdk/src/host/embeddings.ts:150-153`
  - body: The PR starts using `gpu`, `gpuLayers`, `contextSize`, and `flashAttention` when creating the local embedding runtime, but memory index identity is still based on provider runtime cache data. Current main hashes that cache data to decide whether an existing vector index is valid, and the llama.cpp runtime currently records only provider and model, so changing these settings can leave old document vectors marked valid while new query embeddings use different runtime parameters. Add the embedding-affecting local settings to the stable provider/cache key or add an explicit tested policy for why they do not require reindexing.
  - confidence: 0.88
- **[P1] Preserve the current contextSize contract:** `packages/memory-host-sdk/src/host/embeddings.ts:74-77`
  - body: Current main already supports `memorySearch.local.contextSize` as `number | "auto"` and defaults it to `4096` before creating the embedding context. This PR's added type is `number` only and the implementation only passes a context size when the user sets one, so a rebase that takes this shape would drop the existing `"auto"` value and default behavior. Keep the current contract while adding the new GPU and flash-attention knobs.
  - confidence: 0.82
- **[P3] Remove the source-tree localized docs edit:** `docs/zh-CN/concepts/memory.md:209-231`
  - body: `docs/AGENTS.md` says localized docs under `docs/<locale>/**` are not maintained in this repository; English docs are the source and localization is handled by the publish/i18n pipeline. Please drop the `docs/zh-CN` change from this PR and keep the source docs update in English.
