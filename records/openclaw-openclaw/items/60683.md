## Summary

Keep open. Current main still lacks the requested GPU and GPU-layer local embedding knobs, but this PR is not mergeable as-is because it exposes an unsupported `flashAttention` embedding-context option, regresses current `contextSize` defaults and `"auto"` support, edits localized docs, conflicts with main, and still lacks real runtime proof.

## What This Changes

The branch adds `memorySearch.local` GPU, `gpuLayers`, `contextSize`, and `flashAttention` settings, carries them into the local node-llama-cpp embedding provider, and updates config metadata, docs, and tests.

## Best Possible Solution

Land a rebased implementation that exposes only upstream-supported local llama tuning, preserves the current `contextSize` default and `"auto"` semantics, updates English config/docs/tests together, and includes redacted real runtime proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Remove unsupported flashAttention passthrough:** `packages/memory-host-sdk/src/host/embeddings.ts:152`
  - body: `node-llama-cpp` 3.18.1 does not define `flashAttention` on `LlamaEmbeddingContextOptions`; the upstream flag is `defaultContextFlashAttention` on model-load options. Passing this object key to `createEmbeddingContext` creates a public config setting that the embedding path does not support, so remove it or wire it through a supported upstream option with runtime proof.
  - confidence: 0.91
- **[P2] Preserve contextSize default and auto support:** `packages/memory-host-sdk/src/host/embeddings.ts:116`
  - body: Current main defaults local embeddings to `contextSize: 4096` and accepts explicit `"auto"`, with tests covering both. This branch makes `contextSize` number-only and omits the default when unset, so rebasing as-is would restore node-llama-cpp's auto default and the VRAM spike the current config was added to avoid.
  - confidence: 0.9
- **[P3] Remove the localized docs edit:** `docs/zh-CN/concepts/memory.md:209`
  - body: `docs/AGENTS.md` says localized docs under `docs/<locale>/**` are not maintained here. Keep the English docs/config reference changes in this repo and let the docs publish/i18n pipeline regenerate the Chinese page.
  - confidence: 0.96
