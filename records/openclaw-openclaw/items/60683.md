## Review Findings

Overall confidence: 0.89

Full review comments:

- **[P1] Include local tuning in the provider identity:** `src/plugins/memory-embedding-providers.ts:47-50`
  - body: These new fields change how local embeddings are produced, but memory reindex/cache invalidation keys off `providerRuntime.cacheKeyData`, and the current llama.cpp runtime key on main only includes provider/model. Changing `gpuLayers`, `contextSize`, or `flashAttention` after an index exists can leave stored document vectors and cached embeddings under the old settings while queries use the new runtime. Add the local tuning values to the llama.cpp runtime cache key or otherwise force a full reindex.
  - confidence: 0.9
- **[P2] Pass flash attention to the supported API:** `packages/memory-host-sdk/src/host/embeddings.ts:152`
  - body: `node-llama-cpp@3.18.1` does not expose `flashAttention` on `createEmbeddingContext`; the model option is `defaultContextFlashAttention`. Passing this setting into the embedding context means the new public config knob is likely ignored or relies on an undocumented property, so move it to model load or remove the option.
  - confidence: 0.88
- **[P2] Preserve contextSize auto and default behavior:** `src/config/zod-schema.agent-runtime.ts:669`
  - body: Current main already accepts `local.contextSize: "auto"` and the SDK defaults unset values to `4096`. This branch's schema only accepts positive numbers, so taking this version during rebase would reject existing current-main configs and lose the bounded default. Keep `"auto"` plus the 4096 default while adding the GPU knobs.
  - confidence: 0.86
