## Summary

Keep this PR open with changes requested: the operation-aware resolver is the right fix shape, but the gateway poll handler still uses the shared resolver and can select a setup-only or text-only plugin before a poll-capable runtime fallback.

## What This Changes

The PR adds operation-aware outbound delivery plugin resolution, routes message/direct delivery through it, and drains matching QQBot pending deliveries on gateway ready/resumed callbacks.

## Best Possible Solution
