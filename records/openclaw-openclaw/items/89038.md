## Summary

Keep open: the PR is a credible fix for the linked QQBot reconnect message-loss bug, but the current head still uses one delivery-capability predicate for both text sends and polls, so one-operation plugins can shadow the correct runtime plugin for the other operation.

## What This Changes

The branch adds a delivery-only outbound channel resolver, switches outbound send/poll preflight to it, drains QQBot pending deliveries on ready/resumed reconnect, and adds regression tests.

## Best Possible Solution

Keep the QQBot reconnect drain and setup-shell fix, but make delivery resolution operation-specific before merge.

## Solution Assessment

No. The QQBot drain and setup-shell direction is good, but the best fix is operation-specific delivery resolution rather than one shared send-capability predicate.
