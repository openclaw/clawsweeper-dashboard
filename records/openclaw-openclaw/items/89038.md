## Summary

## What This Changes

The branch adds operation-aware outbound delivery resolution, routes text, poll, direct delivery, and gateway poll paths through it, adds QQBot reconnect pending-delivery drains, and expands regression coverage.

## Best Possible Solution

Land this PR or an equivalent owner-approved fix after maintainers accept the resolver first-match semantics and QQBot reconnect replay timing.

## Reproduction Assessment

Yes at source level: current main still has capability-blind resolver and direct-delivery paths that can select a non-send shell and later throw Outbound not configured, and the linked issue provides live QQBot 4009 logs. I did not run a fresh live QQBot timeout in this read-only review.

## Solution Assessment

Yes, this appears to be the best maintainable fix shape: keep the shared resolver for action/read callers and add a delivery-only operation-aware resolver for send/poll paths. Gating the shared resolver would risk action-only surfaces, while a QQBot-only drain would not fix the generic first-match delivery drift.
