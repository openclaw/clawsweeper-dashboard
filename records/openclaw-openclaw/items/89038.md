## Summary

Keep open: the PR is a useful candidate fix for the QQBot reconnect message-loss regression, but the shared delivery resolver still accepts either text or poll capability for both send operations, so it can select a plugin that cannot perform the requested send before a later usable sender is considered.

## Best Possible Solution
