## Summary

## What This Changes

The PR adds a send-capability-gated outbound delivery resolver, routes message send paths through it, and adds QQBot reconnect pending-delivery drains with regression tests.

## Best Possible Solution

## Solution Assessment

Yes; splitting shared lookup from delivery-only send-capable lookup is the narrowest maintainable fix because action and target paths keep the existing resolver while send paths avoid setup shells. The QQBot drain reuses the existing SDK delivery-queue runtime instead of adding plugin-local queue logic.
