## Summary

This PR should stay open: the central QQBot reconnect/text-delivery fix is well supported, but the latest head still leaves the gateway poll sibling path on the shared resolver, so setup-only or text-only entries can still shadow a poll-capable fallback there before merge.

## What This Changes

The PR adds operation-aware outbound delivery plugin resolution, routes message/direct delivery through it, and adds QQBot reconnect pending-delivery drains with regression tests.

## Best Possible Solution

Land this after the gateway poll server method uses the same poll-capable delivery resolver and has a focused regression test, while keeping the shared resolver ungated for action/read callers and retaining the QQBot reconnect drain as an owner-accepted recovery behavior.

## Solution Assessment
