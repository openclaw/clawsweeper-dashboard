## Summary

## What This Changes

## Best Possible Solution

## Reproduction Assessment

Yes, from source inspection: PR head calls the send-capability-gated resolver before the delivery layer's lazy bootstrap runs, so setup-only loaded channels can now fail at message preflight. I did not run tests because this review is read-only.

## Solution Assessment

No, not yet: the PR is close, but the current shape moves send-capability gating too early for direct send paths. The better fix keeps shared resolution ungated, applies send filtering for delivery, and preserves the existing bootstrapped sender path with regression coverage.
