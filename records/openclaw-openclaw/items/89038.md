The PR adds operation-aware outbound delivery resolution, routes message/direct delivery through it, drains QQBot pending deliveries on reconnect, and adds outbound/QQBot regression tests.

## Best Possible Solution

Route every send/poll delivery caller that requires send capability through the operation-aware resolver, keep the shared resolver for action/read callers, and land the QQBot reconnect drain only with the targeted retry-timing behavior covered by tests.
