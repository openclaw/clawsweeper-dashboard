## Summary

Keep open for maintainer review: the branch is a well-evidenced candidate fix for the linked QQBot reconnect outbound-delivery regression, but it intentionally changes delivery resolver first-match semantics and QQBot queued-message replay timing, which repository policy treats as compatibility and message-delivery sensitive.

## What This Changes

The PR adds an operation-aware outbound delivery resolver, routes text, poll, direct delivery, and gateway poll paths through it, and drains QQBot pending deliveries on ready/resumed reconnect callbacks.

## Best Possible Solution

Adopt the operation-aware delivery resolver and QQBot reconnect drain only after outbound/QQBot owners explicitly accept the compatibility and replay-timing behavior, or narrow those behaviors before merge.
