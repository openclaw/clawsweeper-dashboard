## Summary

Keep this PR open, but it is not merge-ready: the latest patch still applies send-capability gating inside the shared plugin resolver used by message-action dispatch, and the live proof covers outbound resolution but not the new reconnect-drain behavior.

This PR adds a send-capability filter to `resolveOutboundChannelPlugin`, drains QQBot pending deliveries on gateway ready/resumed callbacks, and adds focused resolver and QQBot drain unit tests.

## Best Possible Solution
