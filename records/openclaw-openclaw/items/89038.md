## Summary

Keep open: this is a useful P1 QQBot/outbound delivery fix candidate, but the current head is not merge-ready because the new delivery resolver still conflates text and poll capabilities and can let the wrong registration shadow the usable sender.

## What This Changes

The PR adds a send-capability-gated outbound resolver for message delivery, drains QQBot pending deliveries on ready/resumed reconnect, and adds outbound/QQBot regression tests.

## Best Possible Solution

Keep the QQBot reconnect drain and setup-shell fix, but split delivery resolution by operation so text sends require text or gateway capability and polls require poll or gateway capability before merge.

## Solution Assessment

No. The direction is right, but the best fix is operation-specific delivery resolution rather than one send-capability predicate shared by text and poll paths.
