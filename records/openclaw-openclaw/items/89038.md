## Summary

Keep open: this is a useful fix for the linked QQBot reconnect delivery regression with solid real-behavior proof, but current head still leaves the gateway poll dispatch path on the shared resolver while the PR makes poll delivery operation-aware elsewhere.

## What This Changes

The PR adds a send-operation-aware outbound delivery resolver, routes message and direct delivery through it, drains QQBot pending deliveries on reconnect, and expands outbound/QQBot regression coverage.

## Best Possible Solution

Use one operation-aware delivery resolver for all send and poll delivery entry points while leaving the shared resolver for action/read callers, then land the QQBot reconnect drain if maintainers accept the replay timing.
