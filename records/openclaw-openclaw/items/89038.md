## Summary

Keep open: this is a credible QQBot reconnect/message-loss fix PR with strong real-behavior proof, but the latest head still has a blocking generic outbound resolver bug before merge.

## What This Changes

The PR adds a delivery-only outbound channel resolver, routes message send preflight through it, drains QQBot pending deliveries on ready/resumed reconnect callbacks, and adds outbound/QQBot regression tests.

## Best Possible Solution

Keep the QQBot reconnect drain and setup-shell filtering, but make delivery resolution intent-specific so text sends require text/gateway capability while polls require poll/gateway capability.

## Solution Assessment

No: the QQBot drain plus delivery-only setup-shell filtering are the right direction, but the latest patch is not the best final fix until text and poll capability checks are separated at the resolver boundary.
