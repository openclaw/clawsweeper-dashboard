## What This Changes

The PR adds a delivery-only send-capable outbound resolver, routes message send paths through it, and drains QQBot pending deliveries from ready/resumed reconnect callbacks.

## Best Possible Solution

Land the resolver split and QQBot reconnect drain after maintainer review confirms the shared-versus-delivery resolver boundary and reconnect recovery behavior are acceptable.

## Solution Assessment

Yes. Splitting shared lookup from delivery-only send-capable lookup is the narrow maintainable fix, and the QQBot drain reuses the existing SDK delivery-queue runtime instead of adding a plugin-local retry queue.
