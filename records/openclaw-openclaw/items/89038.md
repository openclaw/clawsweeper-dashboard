## Summary

## What This Changes

Adds a send-capability-filtered outbound channel resolver for delivery paths, switches message send/poll resolution to it, and drains matching qqbot pending deliveries on gateway ready/resumed with focused regression tests.

## Best Possible Solution

## Solution Assessment

Yes. The delivery-only resolver is the narrowest maintainable fix because it avoids changing action/metadata callers, and the qqbot reconnect drain reuses the existing delivery queue runtime as an upgrade safeguard.
