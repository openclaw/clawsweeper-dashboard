## Summary

Keep this PR open, but it is not merge-ready: the useful Control UI math-rendering idea currently uses global single-dollar MathJax parsing that can corrupt ordinary chat text, and it vendors a large opaque browser script outside dependency tracking without sufficient real behavior proof.

## What This Changes

The PR adds a self-hosted MathJax SVG bundle, initializes global MathJax typesetting from `ui/index.html`, and extends a Gateway HTTP asset-serving test for the new asset.

## Best Possible Solution

Implement math rendering through a tracked UI dependency or approved vendor process, keep single-dollar inline parsing off by default, add focused markdown/UI coverage, and provide real Control UI proof.

## Reproduction Assessment

Yes from source for the current behavior: the Control UI markdown renderer uses markdown-it without a math extension, so LaTeX delimiters are rendered as ordinary sanitized text. I did not run a browser reproduction in this read-only review.

## Solution Assessment

No; the current PR is a plausible prototype but not the best fix because global post-render MathJax with `$...$` parsing and an opaque vendored bundle creates avoidable compatibility and supply-chain risk.

## Review Findings

Full review comments:

- **[P2] Remove the single-dollar inline delimiter:** `ui/index.html:202`
  - body: MathJax intentionally leaves `$...$` disabled by default because dollar signs are common in normal text; this PR typesets the whole chat DOM, so a message like `cost is $2.50 ... $2.00` can have the middle treated as math. Keep `\(...\)`/display delimiters or add a markdown-stage opt-in instead. ([docs.mathjax.org](https://docs.mathjax.org/en/v3.1/input/tex/delimiters.html?utm_source=openai))
  - confidence: 0.93
- **[P1] Replace the opaque bundled MathJax blob:** `ui/public/assets/mathjax-tex-svg.js:1`
  - body: This adds a 2.1 MB minified third-party script that is not declared in `ui/package.json` or the lockfile and has no source/license/build provenance in the repo. Because it runs under the Control UI origin, reviewers and dependency tooling need a tracked package/build path or explicit approved vendoring before merge.
  - confidence: 0.91
