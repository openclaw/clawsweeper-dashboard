## Summary

Keep open for maintainer review: current main and the latest release still have the unguarded isolated-cron `sourceDelivery` dereference, and this PR is the active candidate fix with no blocking code-review findings. The remaining decision is whether maintainers want to accept runtime tolerance for stale executor caller shapes and the reconstructed message-delivery semantics.

## What This Changes

The branch makes `sourceDelivery` optional at the isolated cron executor boundary, adds a shared cron source-delivery planner/fallback helper, refactors the normal cron planner to use it, and adds focused guard/parity tests.

## Best Possible Solution

Merge the shared guarded source-delivery planner after maintainers accept the version-skew fallback contract and normal required checks are clean.

## Reproduction Assessment

Yes at source level: current main requires `sourceDelivery` and dereferences it before runner dispatch, matching the linked crash when a stale executor caller omits it. I did not run a live packaged cron repro in this read-only review.

## Solution Assessment

Yes, if maintainers accept the compatibility contract: sharing the normal cron source-delivery planner with the fallback path is the narrowest maintainable code shape. The alternative is a product decision to reject runtime skew tolerance and fix packaging, restart, or doctor behavior instead.
