## Summary

Keep open: the PR targets a plausible isolated-cron regression, but the fallback still changes legacy delivery policy and the submitted proof is only a Docker-run unit simulation.

## What This Changes

The PR makes `sourceDelivery` optional in the isolated cron executor, reconstructs a fallback delivery plan, logs when the fallback is used, and adds a source-delivery guard test.

## Best Possible Solution

Keep the normal `SourceDeliveryPlan` contract for current callers, and make the version-skew fallback preserve legacy delivery fields with live isolated-cron proof for `none` and `announce` delivery.

## Reproduction Assessment

Yes from source for the skewed-boundary failure: if the executor receives params without `sourceDelivery`, current main dereferences it immediately. I did not establish a live installed or stale-artifact cron reproduction in this read-only review.

## Solution Assessment

No. The guard avoids the immediate TypeError, but the fallback should preserve legacy delivery policy instead of changing message-tool forcing and announce source-reply behavior.
