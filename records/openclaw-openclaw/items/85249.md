## Summary

The patch is internally coherent and strongly proven at the executor boundary, but it deliberately adds steady-state recovery for mixed build artifacts and reconstructs visible delivery policy; that compatibility choice needs explicit owner approval before merge.

## What This Changes

The PR makes the isolated cron executor’s `sourceDelivery` input optional, reconstructs it from the job and resolved target when absent, shares that planner with the normal path, and adds regression tests.

## Best Possible Solution

If mixed-artifact recovery is an accepted production contract, keep the single canonical planner, obtain source-delivery owner approval, and prove one exactly-once isolated announce on a real channel; otherwise retain the required boundary and fix the packaging or rebuild lifecycle.

## Reproduction Assessment

Yes. Omitting `sourceDelivery` at the executor boundary reaches an unconditional current-main dereference, and the supplied live-source harness reproduces the exact reported TypeError.

## Solution Assessment

Unclear. The shared planner is the narrowest implementation if runtime artifact skew must be supported, but OpenClaw policy otherwise favors canonical runtime state and lifecycle repair over new fallback compatibility paths.
