## Summary

## What This Changes

Makes `sourceDelivery` optional at the isolated cron executor boundary, rebuilds a fallback `SourceDeliveryPlan` from the cron job delivery config, shares the cron planner with `run.ts`, and adds guard/parity tests.

## Best Possible Solution

Merge only after real stale-artifact Gateway cron or live transport proof shows the patched path avoids the crash and preserves single final delivery, or after a maintainer explicitly accepts that proof gap.

## Reproduction Assessment

Yes at source level: current main unconditionally dereferences `params.sourceDelivery`, so a stale caller that omits the field reaches the reported crash path. I did not run an actual packaged stale-artifact Gateway repro.

## Solution Assessment

Yes, the shared planner is the right code shape because it avoids legacy field sniffing and keeps normal and fallback cron delivery semantics together. The remaining blocker is proof of the real upgrade path, not an obvious code alternative.
