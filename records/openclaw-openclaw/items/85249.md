## Summary

This PR addresses a real isolated-cron crash and has strong focused regression coverage, but it changes a required internal handoff into steady-state recovery for mixed runtime artifacts. That compatibility policy needs explicit owner judgment, and the branch still lacks real-channel proof that the recovery path preserves exactly-once announce delivery.

## What This Changes

The PR makes the isolated cron executor’s `sourceDelivery` input optional, reconstructs a plan from the job and resolved target when absent, shares that planner with the normal path, and adds focused regression tests.

## Best Possible Solution

Keep one canonical source-delivery plan prepared before the lazy boundary; either repair the upgrade/restart lifecycle that permits mixed artifacts, or explicitly approve a narrowly defined skew-recovery contract and prove its exact-once behavior across supported delivery modes.

## Solution Assessment
