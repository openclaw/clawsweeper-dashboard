## Summary

Keep open: this is a focused, non-obsolete fix PR for the open isolated-cron `sourceDelivery` crash, and current `main` plus the latest release still have the raw executor dereference. ## What This Changes

The PR makes isolated cron executor `sourceDelivery` optional, rebuilds missing source-delivery plans through a shared cron helper, routes the normal planner through that helper, and adds guard/parity tests.

## Best Possible Solution

Merge the shared-planner fallback after maintainers accept current-job-config recovery semantics; live transport proof would reduce the remaining delivery-path uncertainty.

## Reproduction Assessment

Yes at source level: current main and v2026.6.6 still dereference `params.sourceDelivery`, and the PR body supplies Docker/host terminal output for a stale caller omitting `sourceDelivery`. I did not run a live packaged stale-module cron job in this read-only review.

## Solution Assessment

Yes, this is the best maintainable shape if maintainers accept runtime compatibility fallback because it shares the normal planner instead of preserving duplicated legacy field sniffing. The safer alternative is to reject runtime fallback and handle affected installs through rebuild or upgrade guidance, but that leaves the reported crash path unresolved.
