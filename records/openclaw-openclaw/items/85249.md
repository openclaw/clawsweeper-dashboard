## What This Changes

The branch makes the isolated cron executor tolerate a missing `sourceDelivery` plan by rebuilding it from job delivery config, moves the cron source-delivery planner into a shared helper, and adds guard/parity tests.

## Best Possible Solution

Keep the shared planner guard, then merge after redacted real cron proof shows the missing-`sourceDelivery` path no longer crashes and announce/none/webhook delivery do not double-post or misroute.

## Solution Assessment

Yes for the code shape: rebuilding from canonical job delivery config and sharing one planner is cleaner than sniffing stale legacy fields. It is not merge-ready until real after-fix cron behavior proof is added.
