## Summary

Keep open: current main and v2026.6.10 still have the unguarded isolated-cron `sourceDelivery` dereference, and this PR is the live candidate fix with no blocking code findings found. The remaining question is maintainer acceptance of the stale-executor compatibility fallback and its message-delivery semantics.

## What This Changes

This PR makes isolated cron executor `sourceDelivery` optional, reconstructs a missing plan from cron delivery config through a shared helper, rewires normal cron planning to that helper, and adds regression coverage.

## Best Possible Solution

Land or deliberately replace the shared-helper fallback after maintainers accept the stale-executor upgrade behavior, then close https://github.com/openclaw/openclaw/issues/85051 once the fix reaches main.

## Reproduction Assessment

Yes for the source-level stale-caller shape: current main and v2026.6.10 directly dereference `params.sourceDelivery.sourceReplyDeliveryMode`, and the PR body supplies terminal proof for an omitted `sourceDelivery`. A clean normal cron path still passes `sourceDelivery`, so live end-to-end reproduction depends on stale artifacts or version skew.

## Solution Assessment

Yes, with maintainer acceptance: sharing the normal planner for fallback reconstruction is cleaner than legacy field sniffing and reduces drift. The unresolved part is upgrade policy rather than a narrow code defect.
