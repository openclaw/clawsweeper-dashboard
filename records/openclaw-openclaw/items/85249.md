## Summary

Keep open: the crash guard is focused and well supported, but the patch currently forwards a broad `string` legacy delivery mode into APIs typed for the closed `SourceReplyDeliveryMode` union, which should be fixed before merge.

## What This Changes

This PR makes isolated cron executor `sourceDelivery` optional, reconstructs a legacy delivery plan when it is missing, logs that fallback, and adds focused guard tests.

## Best Possible Solution

Merge this narrow crash guard after narrowing the legacy reply mode to the `SourceReplyDeliveryMode` union, leaving broader cron delivery ownership changes to the separate canonical PR.

## Reproduction Assessment

Yes. Current main requires `sourceDelivery` and dereferences `params.sourceDelivery.sourceReplyDeliveryMode`; a stale executor caller that omits the field reproduces the undefined-property crash described in the linked issue.

## Solution Assessment

No, not quite yet. Reconstructing the old boundary is the right narrow fix, but the legacy reply mode must be narrowed to the closed `SourceReplyDeliveryMode` contract before merge.
