## Summary

Keep open: current main and the latest inspected release still have the unguarded isolated-cron `sourceDelivery` dereference, and this PR is the active candidate fix for the open isolated cron crash report. The patch looks mechanically correct, but merge should remain a maintainer decision because it intentionally tolerates omitted stale/version-skew executor input and affects visible cron delivery fallback behavior.

## What This Changes

The branch makes the isolated cron executor tolerate omitted `sourceDelivery`, extracts a shared cron source-delivery planner, rewires the normal planner to use it, and adds guard/parity tests.

## Best Possible Solution

Land after maintainers explicitly accept the stale-artifact compatibility behavior, with optional live announce-transport proof if they want visible delivery confirmation before merge.

## Solution Assessment
