## Summary

Keep open: current main still has the unsafe executor dereference, while this PR appears to address the reported stale-caller crash without a discrete correctness finding; the remaining question is maintainer acceptance of the compatibility and cron delivery policy.

## What This Changes

The PR makes the isolated cron executor tolerate a missing `sourceDelivery` by rebuilding a `SourceDeliveryPlan` from job delivery config and shares the cron source-delivery planner between the normal and fallback paths.

## Best Possible Solution

Land only after maintainers accept the stale-artifact compatibility behavior and cron delivery policy; otherwise replace it with an explicit fail-closed or upgrade/doctor path.

## Reproduction Assessment

Yes from source inspection: current main dereferences `params.sourceDelivery.sourceReplyDeliveryMode`, so a stale caller that omits `sourceDelivery` will throw before the agent run. I did not execute the repro because this review was read-only.

## Solution Assessment

Yes as an implementation shape: sharing the planner avoids a second cron delivery policy while guarding the stale boundary. The unresolved part is not code shape but whether maintainers want this compatibility fallback instead of a fail-closed upgrade policy.
