## What This Changes

The PR makes isolated cron executor `sourceDelivery` optional, reconstructs a cron source-delivery plan from job delivery config when absent, moves the planner into a shared helper, and adds guard/parity tests.

## Best Possible Solution

Land the shared current-semantics fallback if maintainers want runtime skew tolerance; otherwise request one packaged skew smoke or replace it with an explicit fail-fast policy.

## Reproduction Assessment

No high-confidence live current-main reproduction was established in this read-only review. The source path is reproducible when a stale caller omits `sourceDelivery`: current main has the raw dereference, and the PR body includes Docker/host terminal proof for that stale-caller path.

## Solution Assessment

Yes, this appears to be the best observed fix shape: the normal path still prepares `sourceDelivery`, while stale callers are rebuilt through the same cron planner instead of a second legacy-field implementation. The remaining question is maintainer acceptance of supporting that skew in runtime code.
