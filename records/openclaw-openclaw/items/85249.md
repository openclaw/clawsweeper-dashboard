## Summary

The PR is still necessary: current main and the latest release still dereference `params.sourceDelivery` without a guard, while this branch adds the missing stale-boundary fallback. ## What This Changes

This PR makes isolated cron `sourceDelivery` optional, extracts a shared cron source-delivery planner/fallback helper, updates the normal cron path to use it, and adds guard/parity tests for omitted `sourceDelivery` across delivery modes.

## Best Possible Solution

Land only after a maintainer accepts the version-skew compatibility contract and, ideally, records or requests one live announce cron transport proof for the visible delivery path.

## Reproduction Assessment

I did not run a live cron job in this read-only review.
