## Summary

## Best Possible Solution

Yes, likely: deriving stale-caller fallback policy from the job delivery config through one shared helper is the narrowest maintainable code shape. The remaining blocker is real behavior proof for the upgrade/message-delivery path, not a line-level code defect I can point to.
