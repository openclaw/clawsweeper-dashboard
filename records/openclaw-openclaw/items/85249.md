## Summary

Keep open: current main and the latest release still have the unguarded isolated-cron `sourceDelivery` dereference, while this PR is the open candidate fix for the linked crash report and remains mergeable with no concrete patch defect found.

## What This Changes

The branch makes the isolated cron executor tolerate omitted `sourceDelivery` by rebuilding a `SourceDeliveryPlan` from cron delivery config, sharing the planner with the normal cron path, and adding targeted regression tests.

## Best Possible Solution

Land this guard after maintainers accept the version-skew runtime fallback contract; otherwise replace it with an explicit rebuild or doctor-owned remediation path while keeping one canonical cron source-delivery planner.

## Reproduction Assessment

Yes, at source level: current main and `v2026.6.9` still directly read `params.sourceDelivery.sourceReplyDeliveryMode`, so an omitted `sourceDelivery` call reproduces the reported crash shape. I did not run a live packaged cron job in this read-only review.

## Solution Assessment

Yes, if maintainers accept runtime compatibility for version skew: sharing the planner avoids policy drift and the tests cover the missing-boundary modes. The stricter alternative is a rebuild, reinstall, or doctor remediation path, which is a maintainer product choice rather than an obvious patch defect.
