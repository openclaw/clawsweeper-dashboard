## Summary

Keep open: this PR is still the active candidate fix for the isolated cron `sourceDelivery` crash, and current main has not implemented the guard. The patch looks coherent with no blocking code findings, but it intentionally accepts a stale/version-skew executor input and reconstructs message-delivery policy, so maintainers should make that compatibility decision before merge.

## What This Changes

The PR makes isolated cron `sourceDelivery` optional at the executor boundary, extracts a shared cron source-delivery planner/fallback helper, and adds guard/parity tests.

## Best Possible Solution

Land this once maintainers accept the version-skew fallback contract; otherwise replace it with an explicit remediation or fail-fast path.

## Reproduction Assessment

Partially. Current main directly dereferences `params.sourceDelivery.sourceReplyDeliveryMode` when `sourceDelivery` is omitted, and the PR proof exercises that omitted-argument path, but I did not reproduce a live packaged stale-artifact cron run.

## Solution Assessment

Unclear as a product decision, but the code shape is sound if runtime fallback is desired. The safer alternative is to fail fast with explicit rebuild/doctor remediation if maintainers do not want stale executor compatibility.
