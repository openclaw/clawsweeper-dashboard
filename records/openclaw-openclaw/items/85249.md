The PR makes isolated cron executor `sourceDelivery` optional, rebuilds missing source-delivery plans from cron job delivery config through a shared helper, routes the normal planner through that helper, and adds guard/parity tests.

## Best Possible Solution

Merge the shared-planner fallback once maintainers accept current-job-config recovery semantics and normal required checks pass; optional live transport proof would further reduce delivery-path uncertainty.

The safer alternative is to reject runtime fallback and handle affected installs through rebuild or upgrade guidance, but that leaves the reported crash path unresolved.
