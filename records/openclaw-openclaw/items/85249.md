## Summary

Keep open for maintainer review: the central fix is not on current main, the patch has no concrete code finding, and the remaining decision is whether OpenClaw should accept this version-skew fallback and its current cron message-delivery semantics.

## What This Changes

The PR makes the isolated cron executor tolerate an omitted `sourceDelivery` plan by reconstructing one from the cron job delivery config, moves cron source-delivery planning into a shared helper, and adds focused guard tests.

## Best Possible Solution

Land the shared planner and guarded executor fallback after maintainers accept the version-skew contract, then let the merge close the linked cron crash report.

## Reproduction Assessment

Partially. Current main has the raw `params.sourceDelivery` dereference and the PR proof shows a stale-caller omitted-parameter harness hitting the exact TypeError; I did not independently reproduce a packaged gateway skew.

## Solution Assessment

Yes, this is the best observed fix shape: sharing the cron planner avoids a second delivery-policy implementation, while the normal path still passes prepared `sourceDelivery`. The remaining question is maintainer acceptance of steady-state runtime-skew compatibility.
