## Label Justifications

- `P1`: The PR targets a reported regression where isolated cron agent jobs can fail before the agent turn starts.
- `merge-risk: 🚨 compatibility`: The patch changes the executor contract by accepting missing `sourceDelivery` at runtime and deriving behavior instead of failing on version skew.
- `merge-risk: 🚨 message-delivery`: The fallback can affect whether cron uses message tools, direct fallback delivery, and duplicate suppression for stale executor callers.

## Summary

Keep open for maintainer review: the patch appears correct and has no discrete blocking code finding, but merging it means accepting runtime compatibility for a stale/version-skew executor input and its reconstructed message-delivery semantics.

## What This Changes

This PR makes isolated cron `sourceDelivery` optional at the executor boundary, reconstructs missing source-delivery plans from cron job delivery config through a shared helper, and adds guard/parity tests.

## Best Possible Solution

Land this only after maintainers accept the compatibility fallback contract, or replace it with an explicit rebuild/doctor remediation path if runtime fallback is not desired.

## Reproduction Assessment

Partially. Source inspection shows the current-main crash point for omitted `sourceDelivery`, and the PR body includes a source-level stale-caller harness, but this review did not run a live packaged cron with stale executor artifacts.

## Solution Assessment

Unclear. The implementation is a coherent narrow fallback, but the best product fix depends on whether maintainers want runtime compatibility for stale build artifacts or a remediation-first failure mode.
