This PR makes isolated cron executor `sourceDelivery` optional at the lazy execution boundary, reconstructs missing plans from job delivery config through a shared helper, and adds guard/parity tests.

## Best Possible Solution

Keep the shared planner fallback and merge after maintainers accept current-job-config recovery semantics; optionally capture a live Telegram cron smoke to reduce message-delivery uncertainty.

## Solution Assessment

Yes, this is the best maintainable shape if maintainers accept a version-skew fallback: it shares the normal planner instead of preserving legacy field sniffing.
