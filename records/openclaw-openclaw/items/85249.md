## Summary

Keep open for maintainer review: the PR appears to fix a real isolated-cron regression and I found no concrete line-level correctness defect, but it intentionally turns an omitted executor `sourceDelivery` into runtime compatibility fallback behavior with message-delivery semantics that maintainers need to accept before merge.

## What This Changes

The PR makes isolated cron source-delivery planning shared between the normal execution path and a fallback path, then allows the isolated executor to rebuild a missing `sourceDelivery` plan from the cron job and resolved delivery target.

## Best Possible Solution

Land the shared-planner guard after maintainers accept the version-skew fallback contract; otherwise keep one canonical planner and replace the runtime fallback with explicit rebuild, reinstall, or doctor-owned remediation.

## Reproduction Assessment

Yes, at source level: current main and v2026.6.9 still directly read `params.sourceDelivery.sourceReplyDeliveryMode`, so an omitted executor argument matches the reported crash shape. I did not run a live packaged cron job in this read-only review.

## Solution Assessment
