## Summary

This PR should stay open as the active candidate fix for the isolated cron `sourceDelivery` crash; the implementation looks correct, but landing it still requires maintainer acceptance of a runtime version-skew fallback and its delivery semantics.

## What This Changes

The PR makes isolated cron executor `sourceDelivery` optional, rebuilds a cron source-delivery plan from job delivery config when it is missing, shares the planner with the normal path, and adds focused guard tests.

## Best Possible Solution

Land the shared fallback only if maintainers intentionally want runtime tolerance for stale executor artifacts with defaults tied to current cron delivery semantics; otherwise choose rebuild or reinstall remediation instead.

## Reproduction Assessment

## Solution Assessment

Unclear as product direction. The shared helper is the cleanest runtime fallback implementation I found if version-skew tolerance is desired, but maintainers still need to choose that policy over rebuild or reinstall remediation.
