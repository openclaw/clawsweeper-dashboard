## What This Changes

The PR makes isolated cron execution tolerate a missing `sourceDelivery` boundary param by rebuilding the cron source-delivery plan from job delivery config and sharing that planner with the normal cron path.

## Best Possible Solution

Land one canonical cron source-delivery planner only if maintainers explicitly accept the version-skew fallback policy and the final proof bar for visible channel delivery is met.

## Reproduction Assessment

Yes. Current main dereferences `params.sourceDelivery.sourceReplyDeliveryMode`, and the PR body supplies before/after terminal output for an omitted-`sourceDelivery` executor call; I did not run it locally because this review is read-only.

## Solution Assessment

Acceptable mitigation, but not a pure no-decision bug fix. Sharing the planner is the right shape if maintainers accept runtime tolerance for mixed artifacts; otherwise the safer policy would be a fail-closed diagnostic and rebuild/restart guidance.
