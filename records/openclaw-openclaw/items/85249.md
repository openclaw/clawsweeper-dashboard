## Summary

Keep this PR open for maintainer review: the patch is a plausible and code-backed fix for the isolated cron executor crash, and I found no blocking correctness finding, but it deliberately changes the lazy executor boundary from fail-closed to reconstructed delivery policy.

## What This Changes

The PR makes the isolated cron executor tolerate missing `sourceDelivery` by sharing a cron source-delivery planner between the normal path and a fallback path, with regression coverage for omitted `sourceDelivery`.

## Best Possible Solution

Keep one canonical cron source-delivery planner shared by normal and fallback paths, with explicit maintainer acceptance that stale executor calls should be reconstructed from job delivery config instead of crashing.

## Reproduction Assessment

Yes. Current main has a direct dereference of `params.sourceDelivery.sourceReplyDeliveryMode`, so a stale caller that omits `sourceDelivery` hits the reported TypeError; the linked issue supplies installed-version symptoms.

## Solution Assessment

Yes, with maintainer acceptance. Sharing the planner is a cleaner fix than re-reading legacy fields, but the policy choice is whether this version-skew fallback should be supported at runtime.
