## Summary

Keep open for maintainer review: the patch addresses the open isolated-cron `sourceDelivery` crash with a shared fallback planner and focused tests, but merging it consciously accepts a stale/version-skew executor compatibility fallback that can affect visible cron delivery semantics.

## What This Changes

The PR makes isolated cron `sourceDelivery` optional at the executor boundary, adds a shared cron source-delivery planner/fallback helper, reuses it from the normal cron path, and adds guard/parity tests.

## Best Possible Solution

Land after maintainers explicitly accept the stale-artifact compatibility behavior, with optional live announce-transport proof if they want visible delivery confirmation before merge.

## Solution Assessment
