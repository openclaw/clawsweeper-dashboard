## Summary

Keep open for maintainer review: the PR addresses a real isolated-cron crash that is still present on current main and the latest release, and I did not find a blocking patch defect. The remaining decision is whether OpenClaw should accept a stale/version-skew executor fallback that reconstructs message-delivery policy at runtime.

## What This Changes

The PR makes `sourceDelivery` optional at the isolated cron executor boundary, adds a shared cron source-delivery planner/fallback helper, rewires the normal path to use that helper, and adds guard/parity regression tests.

## Best Possible Solution

Land after maintainers explicitly accept the stale-artifact compatibility behavior, with optional live announce-transport proof if they want visible delivery confirmation before merge.

## Reproduction Assessment

## Solution Assessment

Yes mechanically if maintainers accept fallback compatibility: the patch shares one planner between normal and fallback paths and avoids legacy field sniffing. The remaining question is whether stale executor inputs should be tolerated or handled by rebuild/doctor remediation.
