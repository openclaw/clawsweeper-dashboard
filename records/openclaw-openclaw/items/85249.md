## Summary

Keep open: this remains a useful candidate fix for the isolated cron crash, and current main plus the latest release still have the unguarded `sourceDelivery` dereference. The patch looks correct from source review, but merging it still needs maintainer acceptance of a runtime version-skew fallback that can affect cron message delivery.

## What This Changes

This PR makes cron `sourceDelivery` optional at the isolated executor boundary, adds a shared cron source-delivery fallback planner used by `run.ts` and `run-executor.ts`, and adds guard/parity tests.

## Best Possible Solution

Land or replace the shared fallback once maintainers explicitly accept omitted `sourceDelivery` as version-skew recovery and normal merge gates pass.

## Solution Assessment
