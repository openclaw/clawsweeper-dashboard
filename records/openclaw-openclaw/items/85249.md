## Summary

Keep open: current main still has the crashable raw `sourceDelivery` dereference and the linked regression remains open, while this PR now uses a shared cron delivery planner with sufficient terminal proof and no blocking code findings. ## What This Changes

The PR makes isolated cron execution tolerate a missing `sourceDelivery` by reconstructing the source-delivery plan from the cron job delivery plan and sharing that planner with the normal path.

## Best Possible Solution

Merge a green version of the shared-planner fallback after maintainers accept the compatibility tradeoff, then let the linked crash reports close from the merged fix.

## Reproduction Assessment

Yes from source inspection, but not from an independently run live repro: current main still dereferences `params.sourceDelivery.sourceReplyDeliveryMode`, which throws the reported TypeError when the executor is reached without `sourceDelivery`. The contributorтАЩs terminal proof demonstrates that exact omitted-field path before and after the patch.

## Solution Assessment

Yes, this is the best maintainable shape I found: reconstructing from the canonical cron delivery plan and sharing the planner avoids stale legacy-field sniffing and delivery-policy drift. The alternative strict failure path would preserve the crash for reported version-skew users, and duplicating planner logic would be riskier.
