## What This Changes

The PR makes `sourceDelivery` optional at the isolated cron executor boundary, adds a shared cron source-delivery fallback/planner helper, rewires the normal planner to use it, and adds guard/parity tests.

## Best Possible Solution

Land this or an equivalent shared-planner guard after maintainers accept the version-skew compatibility fallback and the cron message-delivery semantics it preserves.

## Solution Assessment

Yes, this is an acceptable best fix shape: sharing the normal cron planner with the fallback avoids a separate legacy policy path. The remaining question is maintainer acceptance of a runtime compatibility guard for stale/version-skew artifacts.
