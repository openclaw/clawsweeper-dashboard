## Label Justifications

- `P1`: The linked report is an active regression where isolated cron jobs fail before the agent starts, affecting a real channel workflow.

## Summary

Keep open: this targets a real isolated-cron crash and the PR body now includes before/after Docker terminal proof, but the fallback still ignores the legacy source-delivery fields from the exact stale caller shape it is trying to support, creating compatibility, delivery, and tool-permission drift.

## What This Changes

The PR makes isolated cron executor source delivery optional, adds a fallback source delivery plan when it is missing, logs that fallback, and adds guard tests.

## Best Possible Solution

Keep the new SourceDeliveryPlan contract for current callers, but make the missing-sourceDelivery compatibility path reconstruct from legacy sourceReplyDeliveryMode and toolPolicy when present and cover announce plus webhook stale-caller cases.

## Reproduction Assessment

Yes from source and contributor terminal proof. Current main dereferences sourceDelivery at executor creation, and the pre-refactor lazy caller shape omitted sourceDelivery while carrying sourceReplyDeliveryMode and toolPolicy.

## Solution Assessment

No. The crash guard is needed, but an unconditional cron_none fallback is not the best repair because the compatibility path should preserve the legacy delivery and tool-policy fields it is replacing.
