## Summary

Keep this PR open, but not merge-ready: the contributor supplied strong before/after runtime proof, while the patch turns an unspecified partial-build or module-cache mismatch into a permanent runtime compatibility path even though current source and the inspected shipped caller provide the required plan. ## What This Changes

The PR makes `sourceDelivery` optional in the isolated cron executor, reconstructs a delivery plan from the job and resolved target when it is absent, shares that planner with the normal preparation path, and adds regression tests.

## Best Possible Solution

Keep `SourceDeliveryPlan` as the canonical required prepared fact and fix update/build/module invalidation so old and new cron runtime artifacts cannot execute together; if maintainers intentionally support a tagged skew window, add a narrowly detected, fully specified compatibility path with exact semantics and a removal point.

## Reproduction Assessment

A full stack trace plus caller/executor artifact versions from an affected installation would establish the real lifecycle.

## Solution Assessment

No, not as a permanent fallback: it prevents the immediate dereference but normalizes an unproven mixed-artifact state inside the consumer. The narrower maintainable solution is coherent artifact invalidation/restart, unless maintainers explicitly define a bounded shipped compatibility contract.
