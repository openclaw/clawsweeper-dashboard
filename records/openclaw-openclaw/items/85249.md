## Summary

The PR remains useful and reviewable: current main still has the unguarded isolated-cron `sourceDelivery` dereference, while this branch rebuilds a plan from cron delivery config when a stale executor caller omits it. No new line-level blocker showed up in this review, but the runtime fallback and cron delivery ownership are compatibility-sensitive maintainer decisions before merge.

## What This Changes

The PR makes cron isolated-agent `sourceDelivery` optional at the executor boundary, rebuilds a delivery plan from cron job delivery config when absent, moves the normal cron planner into a shared helper, and adds focused guard/parity tests.

## Best Possible Solution

Land the shared guarded planner if maintainers accept stale-caller recovery from current cron delivery config, then use it as the canonical fix path for the linked cron crash reports.

## Reproduction Assessment

Yes from source inspection and supplied proof: current main still has the raw `sourceDelivery` dereference, and the linked reports describe the same isolated cron crash when that argument is missing. I did not run a live cron job in this read-only review.

## Solution Assessment

Yes as a code shape, pending maintainer policy acceptance: sharing the normal cron planner with the fallback avoids a second delivery-policy implementation. The unresolved choice is whether stale executor callers should be repaired at runtime or forced through rebuild/reinstall recovery.
