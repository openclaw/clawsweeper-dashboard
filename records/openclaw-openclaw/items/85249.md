## Summary

Keep open. Current main still has the required `sourceDelivery` dereference that matches the linked isolated-cron crash, and this PR appears to fix it cleanly, but merging it means accepting runtime recovery for stale or partially rebuilt executor artifacts instead of failing closed.

## What This Changes

The branch makes isolated cron execution tolerate missing `sourceDelivery` by rebuilding the cron source-delivery plan through a shared helper and adding focused regression coverage.

## Best Possible Solution

Merge this shared fallback only after maintainers explicitly accept runtime recovery for stale isolated-executor artifacts and the exact head stays green.

## Reproduction Assessment

## Solution Assessment

Mostly yes: the shared planner is the cleanest code shape because it avoids drift between normal and fallback cron delivery semantics. The remaining question is policy, namely whether stale-artifact recovery is preferred over a fail-closed restart diagnostic.
