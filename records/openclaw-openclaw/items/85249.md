## Summary

Keep open: this PR is the active candidate fix for the isolated cron `sourceDelivery` crash tracked at https://github.com/openclaw/openclaw/issues/85051. Current `main` and v2026.6.10 still have the unguarded executor dereference, while the PR head adds a focused fallback and tests; the remaining blocker is maintainer acceptance of the runtime version-skew fallback contract.

## What This Changes

The PR makes `sourceDelivery` optional at isolated cron executor boundaries, reconstructs a cron source-delivery plan from job config and resolved delivery when missing, extracts a shared planner, and adds regression coverage.

## Best Possible Solution

Land this guard if maintainers accept runtime compatibility for stale executor callers; otherwise replace it with an explicit rebuild or doctor-owned remediation path while keeping one canonical cron source-delivery planner.

## Reproduction Assessment

Yes at source level: current `main` directly dereferences `params.sourceDelivery.sourceReplyDeliveryMode`, so an omitted-`sourceDelivery` executor call reproduces the reported TypeError shape. I did not run a live packaged cron job in this read-only review.

## Solution Assessment

Yes if maintainers accept runtime compatibility for version skew: sharing the planner avoids policy drift and the regression coverage exercises the affected delivery modes. The stricter alternative is rebuild/reinstall or doctor-owned remediation, which is a maintainer product choice rather than a line-level patch defect.
