## What This Changes

The PR makes sourceDelivery optional at the isolated cron executor boundary, reconstructs a SourceDeliveryPlan from cron delivery config when it is absent, and shares the normal/fallback cron source-delivery planner.

## Best Possible Solution

Land the narrow compatibility guard if maintainers accept this shipped-upgrade edge case, keeping normal and fallback cron source-delivery planning in one helper with the new regression coverage.

## Solution Assessment

Yes. The shared helper is a better fix than re-reading stale toolPolicy fields because it keeps fallback behavior aligned with current cron delivery semantics and avoids a second policy implementation.
