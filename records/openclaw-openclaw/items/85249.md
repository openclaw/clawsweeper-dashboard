## Summary

Keep open: current main still has the crash-prone required `sourceDelivery` executor contract, and this PR is a focused candidate fix, but accepting a runtime version-skew fallback for stale executor callers is a maintainer compatibility and delivery-semantics decision.

## What This Changes

This PR makes isolated cron executor `sourceDelivery` optional, reconstructs a cron source-delivery plan from job delivery config when missing, shares the planner with the normal path, and adds regression tests.

## Best Possible Solution

Land the shared fallback only if maintainers intentionally want runtime tolerance for stale executor artifacts, with defaults tied to current cron delivery semantics; otherwise route users to rebuild or reinstall remediation instead of adding this runtime compatibility path.

## Reproduction Assessment

Yes. Current main still dereferences `params.sourceDelivery.sourceReplyDeliveryMode`, and the linked issue plus PR terminal proof show the omitted-field stale-caller path; I did not run a live cron repro in this read-only review.

## Solution Assessment

Unclear as product direction. The shared helper is a clean implementation if runtime version-skew tolerance is desired, but maintainers still need to decide whether that fallback is preferable to rebuild or doctor-style remediation.
