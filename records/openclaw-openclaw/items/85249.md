## Summary

No blocking code finding surfaced in this review; the patch fixes the current crash shape by rebuilding the cron source-delivery plan through one shared helper. The remaining blocker is maintainer acceptance of a runtime compatibility fallback for stale or partially rebuilt executor artifacts, because that fallback reconstructs message-delivery policy instead of failing closed.

## What This Changes

The PR makes isolated cron execution tolerate missing `sourceDelivery`, extracts cron source-delivery planning into a shared helper, and adds regression coverage for stale-caller fallback behavior.

## Best Possible Solution

Land the shared planner fallback only if maintainers intentionally accept the stale-artifact compatibility policy; otherwise replace it with a fail-closed diagnostic or upgrade/doctor path.

## Reproduction Assessment

Yes. Current main dereferences `params.sourceDelivery.sourceReplyDeliveryMode`, and a stale caller that omits `sourceDelivery` reaches the exact undefined-property crash described in the linked report.

## Solution Assessment

Yes for implementation shape, with a policy caveat. Centralizing the normal and fallback planner is the cleanest code path I found, but maintainers still need to accept runtime stale-artifact compatibility as product behavior.
