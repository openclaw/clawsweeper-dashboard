## What This Changes

The PR makes cron source-delivery planning shared between the normal cron path and an isolated-executor fallback, then tolerates omitted `sourceDelivery` at that executor boundary.

## Best Possible Solution

Land this guard after maintainers accept the version-skew fallback contract; if they do not, replace it with an explicit rebuild/doctor remediation path while keeping one canonical source-delivery planner.

## Reproduction Assessment

Yes, at source level: current main and v2026.6.9 still directly read `params.sourceDelivery.sourceReplyDeliveryMode`, so an omitted `sourceDelivery` call reproduces the crash shape. This review did not run a live packaged cron job.

## Solution Assessment

Yes, if maintainers accept runtime compatibility for version skew: sharing one planner avoids policy drift and the tests cover the missing-boundary modes. The alternative is a stricter rebuild/doctor remediation path, which is a product choice rather than a code correctness defect.
