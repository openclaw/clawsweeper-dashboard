## Summary

Keep open for maintainer review: the patch appears source-correct and proof-backed, but it intentionally accepts a stale executor caller shape and owns cron delivery defaults in that fallback, so the remaining blocker is maintainer acceptance of that compatibility behavior.

## What This Changes

The PR makes isolated cron `sourceDelivery` optional at the lazy executor boundary, reconstructs missing plans through a shared cron source-delivery helper, and adds guard/parity tests.

## Best Possible Solution

Land the shared fallback only if maintainers accept runtime tolerance for stale executor artifacts with defaults tied to current cron delivery semantics; otherwise prefer rebuild, reinstall, or doctor remediation.

## Reproduction Assessment

Yes at the source boundary: current main directly dereferences `params.sourceDelivery.sourceReplyDeliveryMode`, and the linked issue reports the same omitted-field crash for isolated cron `agentTurn` jobs. I did not run a live packaged cron repro in this read-only review.

## Solution Assessment

Unclear as product policy. The shared helper is the cleanest runtime fallback I found if version-skew tolerance is desired, but maintainers still need to choose that policy over rebuild, reinstall, or doctor remediation.
