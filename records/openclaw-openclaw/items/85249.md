## Summary

Keep open: the PR addresses a real source-reproducible isolated cron crash, but merging it intentionally accepts a version-skew runtime fallback that reconstructs delivery policy, so maintainer review is still needed before landing.

## What This Changes

This PR makes isolated cron `sourceDelivery` optional at the executor boundary, adds a shared cron source-delivery planner/fallback helper, and adds guard/parity tests.

## Best Possible Solution

Land this only after maintainers accept the compatibility fallback contract, or replace it with an explicit rebuild/doctor remediation path if runtime fallback is not desired.

## Reproduction Assessment

Partially. Current main directly dereferences `params.sourceDelivery.sourceReplyDeliveryMode` when `sourceDelivery` is omitted, matching the reported TypeError, but this read-only review did not reproduce a packaged stale-artifact cron run.

## Solution Assessment

Unclear. The code is a coherent narrow fallback that shares the current planner, but the best product fix depends on whether maintainers want runtime compatibility for stale executor artifacts or a remediation-first failure mode.
