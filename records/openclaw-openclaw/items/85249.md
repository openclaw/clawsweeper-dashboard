## Summary

Keep open for maintainer judgment: the patch appears to correctly guard the missing `sourceDelivery` path and includes useful proof, but merging it accepts runtime recovery for stale or partially rebuilt isolated-executor artifacts and can affect visible cron delivery behavior.

## What This Changes

The PR makes `sourceDelivery` optional at the isolated cron executor boundary, rebuilds a fallback `SourceDeliveryPlan` from cron job delivery config, shares the normal-path planner, and adds focused regression coverage.

## Best Possible Solution

Land this only after an owner explicitly accepts stale-artifact runtime recovery as the desired upgrade behavior; otherwise replace it with the maintainer-approved rebuild or diagnostic path.

## Solution Assessment

Unclear as a product policy, but the implementation is a clean mitigation if maintainers want runtime recovery. The safer alternative, if they reject runtime fallback, is a fail-loud rebuild or doctor/install repair path.
