## Summary

The crash fix is still useful, but the current head is not merge-ready: the fallback path still preserves stale legacy message-tool policy and the proof is unit-test/mock evidence rather than current-head real cron behavior.

## What This Changes

The PR makes `sourceDelivery` optional in the isolated cron executor, synthesizes a fallback delivery plan from legacy fields when it is missing, and adds guard tests for the executor path.

## Best Possible Solution

Rebuild missing `sourceDelivery` through one helper that derives webhook/none/announce behavior from `resolveCronDeliveryPlan(job)` plus `resolvedDelivery`, ignores legacy delivery-policy fields, and proves the current head with a real isolated cron run.
