## Summary

Keep this PR open, but do not merge the current fallback: the crash is real and the supplied proof is strong, while making prepared delivery state optional turns mismatched runtime artifacts into an implicit compatibility contract that can silently change message ownership and duplicate suppression.

## What This Changes

The PR makes isolated cron `sourceDelivery` optional, reconstructs missing plans from the cron job and resolved target, shares plan construction with the normal path, and adds focused regression tests.

## Best Possible Solution

Keep `sourceDelivery` required and repair or reject incoherent runtime artifacts at the lazy-loader/build lifecycle boundary; retain reconstruction only if maintainers explicitly define a bounded cross-version contract and add upgrade proof for targeting, ownership, and duplicate suppression.

## Reproduction Assessment

Yes for the crash: omitting `sourceDelivery` reaches a direct property dereference on current main, and the supplied Docker proof reproduces the exact exception. The supported installation or update sequence that creates mixed artifacts remains unproven.

## Solution Assessment

No. Reconstructing lost prepared state is only the best solution if maintainers intentionally support mixed caller/executor artifacts; otherwise artifact coherence is the narrower owner-boundary fix.
