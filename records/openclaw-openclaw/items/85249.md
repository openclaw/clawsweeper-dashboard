## Summary

## What This Changes

This PR makes isolated cron executor `sourceDelivery` optional, adds a shared cron source-delivery fallback planner, rewires normal cron planning through that helper, and adds guard/parity tests.

## Best Possible Solution

Land after maintainers explicitly accept the stale-artifact compatibility behavior, with optional live announce-transport proof if they want visible delivery confirmation before merge.

## Reproduction Assessment

## Solution Assessment

Yes mechanically if maintainers accept fallback compatibility. The patch shares one planner between normal and fallback paths and avoids legacy field sniffing; if maintainers do not want runtime tolerance, the safer alternative is rebuild, cache, installer, or doctor remediation instead of this fallback.
