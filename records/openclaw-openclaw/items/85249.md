## Summary

Keep this PR open: it targets a real isolated-cron crash, but the current fallback still changes delivery semantics for the legacy/version-skew caller shape and the after-fix proof is only a unit-test simulation.

## What This Changes

The PR makes `sourceDelivery` optional in the isolated cron executor, constructs a fallback `SourceDeliveryPlan`, logs a warning, and adds focused regression tests.

## Best Possible Solution

Keep the normal `SourceDeliveryPlan` contract for current callers, and make the version-skew fallback preserve or reconstruct the legacy delivery fields with real isolated-cron runtime proof.

## Reproduction Assessment

Yes from source for the version-skew case: a stale caller can omit `sourceDelivery` while current `run-executor.ts` dereferences it. I did not establish a live stale-build or installed-package reproduction in this read-only review.

## Solution Assessment

No. The guard avoids the immediate TypeError, but the fallback should preserve the legacy delivery policy instead of changing message-tool and source-reply behavior.
