## Summary

Keep open: current main and the latest release still have the unguarded isolated-cron `sourceDelivery` dereference, while this PR is the active linked candidate fix with strong source-level proof and no blocking line-level findings. The remaining decision is whether maintainers accept the runtime version-skew fallback and its message-delivery reconstruction contract before merge.

## What This Changes

The PR makes `sourceDelivery` optional at the isolated cron executor boundary, adds a shared cron source-delivery fallback planner, moves the normal planner to that helper, and adds regression tests.

## Best Possible Solution

Merge the shared guarded cron source-delivery planner if maintainers accept the version-skew fallback contract; otherwise reject runtime-skew support and pursue packaging, restart, or doctor remediation instead.

## Reproduction Assessment

## Solution Assessment

Yes, if maintainers accept the fallback contract: sharing the normal cron source-delivery planner for the fallback is the narrowest maintainable code shape I found. The safer alternative is a product decision to reject runtime skew support and fix packaging or restart behavior instead.
