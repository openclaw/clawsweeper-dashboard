## Summary

The PR makes the isolated cron executor recover from a missing `sourceDelivery` by rebuilding the canonical plan from job delivery configuration and resolved target data, while sharing the planner with the normal path.

## Best Possible Solution

Keep one canonical planner, but merge the fallback only after an exact-head, supported in-place-update reproduction shows an old loaded caller importing the new executor and completing none, webhook, and announce jobs with exactly-once delivery.

## Reproduction Assessment

## Solution Assessment

Unclear. The shared planner is the best implementation if runtime skew is an accepted shipped compatibility boundary, but otherwise the better fix is update/startup lifecycle consistency rather than a steady-state executor fallback.
