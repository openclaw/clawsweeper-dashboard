## Summary

Keep open: current main partially handles routed restart continuations as internal agent turns, but the no-target WebChat/session-only path can still fall back to a visible `System:` event and miss the intended agent wake. The latest issue comment is useful because it ties the confusing UI symptom to a concrete gateway dispatch failure on OpenClaw 2026.6.1.

## What This Changes

The issue asks Control UI/WebChat to keep `continuationMessage` and runtime handoff prompts out of ordinary user-visible chat, or render them as clearly system/introspection content.

## Best Possible Solution

Fix restart `agentTurn` continuations so WebChat/session-only recovery dispatches an internal agent turn without enqueuing the raw continuation as a visible system event, while leaving broader system-message styling to the canonical UI/system-event tracker.

## Reproduction Assessment

Yes by source, medium confidence: current main still enqueues a restart continuation as a system event when no routable `to` exists, matching the WebChat no-target failure described in the latest comment. I did not run a live restart/WebChat reproduction in this read-only review.

## Solution Assessment

Yes for the narrow dispatch bug: the best fix is in the restart-sentinel continuation route, not in a UI-only label change, because hiding the line without dispatching the agent turn would keep recovery silent.
