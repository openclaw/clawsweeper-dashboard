## Summary

Keep open: current main still turns restart continuation text into an internal-provenance agent turn while carrying the raw text in normal message body fields, and the WebChat/Control UI display projection does not hide or reclassify restart-sentinel `internal_system` messages.

## What This Changes

The issue requests that restart/recovery `continuationMessage` agent turns be treated as internal runtime context instead of rendered as ordinary user chat in Control UI/WebChat.

## Best Possible Solution

Use the existing internal provenance or a narrow continuation audience contract at the transcript/display boundary so the agent receives the continuation while WebChat either suppresses it or renders it through one shared runtime/system treatment.

## Reproduction Assessment

Yes, at source level. A restart sentinel `continuationMessage` becomes an `agentTurn` whose text is copied into `Body`/`RawBody`, and the display projection has no branch for `internal_system` restart-sentinel provenance.

## Solution Assessment

Unclear as an implementation choice. The current direction should align the continuation path with the broader internal-message and audience-classification work before choosing hide versus system-block rendering.
