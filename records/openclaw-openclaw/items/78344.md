## Summary

Keep open: current main now tags routed restart continuations with `InputProvenance.kind = "internal_system"`, but it still dispatches and persists the raw continuation text as a user-role agent turn and the Control UI/WebChat projection does not hide or restyle that provenance as a runtime block.

## What This Changes

Treat restart/update `continuationMessage` and similar runtime continuation prompts as agent-internal or clearly runtime-authored content instead of visible user-authored chat.

## Best Possible Solution

Use the existing `internal_system` provenance on restart continuations to keep the raw continuation prompt agent-visible while removing it from normal user-authored chat display, or render it through a dedicated runtime block if maintainers want transparency.

## Reproduction Assessment

Yes, at source level: a routed restart/update with `continuationMessage` becomes an `agentTurn` whose raw text is used as `Body`/`RawBody`, and current tests assert that shape with `InputProvenance.kind = "internal_system"`. I did not run a live browser scenario.

## Solution Assessment

No, current main is not the best final solution: provenance tagging is a useful step, but the display/history paths still do not honor it for this continuation type. The safer maintainer direction is to decide whether `internal_system` turns are hidden or rendered as a distinct runtime lane, then update gateway projection and Control UI filtering consistently.
