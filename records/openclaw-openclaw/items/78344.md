## Summary

Keep open: current main tags routed restart continuations as `internal_system`, but still sends the raw continuation text through the normal agent-turn `Body`/`RawBody`/`BodyForAgent` path, and the visible-history projection does not hide or distinctly render `internal_system` user turns. Related audience/system-message work remains open and does not cover this restart-continuation path yet.

## What This Changes

Request that Control UI/WebChat treat `continuationMessage` and runtime continuation text as agent-only runtime context or a distinct runtime block instead of visible user-authored chat.

## Best Possible Solution

Route restart continuation text to the agent as internal runtime context with a hidden or empty transcript-facing prompt; if maintainers want transparency, render it through a shared runtime/system block instead of a user bubble.

## Reproduction Assessment

Yes, at source level: a restart/update with `continuationMessage` and a routable session is converted to an `agentTurn`, queued, and dispatched with the continuation text in `Body`, `RawBody`, and `BodyForAgent`. I did not establish a live browser repro in this read-only review.

## Solution Assessment

No, the current path is not the best solution because it tags provenance but still models the runtime continuation as an inbound user-shaped turn. The maintainable fix should separate agent-visible continuation input from transcript/display provenance before choosing hidden versus distinct visible treatment.
