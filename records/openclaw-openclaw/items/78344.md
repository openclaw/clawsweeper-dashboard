## Summary

Keep open: current main still models restart continuations as inbound agent turns whose raw text is placed in normal `Body` and `RawBody` fields, and the Gateway/WebChat display projection does not hide or reclassify `internal_system` provenance. The related system-message and audience-classification tracks remain useful coordination points, but they are open or partial rather than a merged continuation-specific fix.

## What This Changes

The issue asks for restart/runtime continuation prompts to reach the agent without being rendered as ordinary human-authored chat in Control UI/WebChat.

## Best Possible Solution

Use existing internal provenance or a narrow continuation audience contract at the transcript/display boundary so the agent still receives the continuation while WebChat either suppresses it or renders it through one shared runtime/system treatment.

## Reproduction Assessment

Yes, at source level. A restart sentinel with `continuationMessage` becomes an `agentTurn` whose text is dispatched in `Body`/`RawBody`, and the display projection has no handling for `internal_system` provenance.

## Solution Assessment

No, current main is not the best solution because it marks the turn as internal only in provenance while still carrying the raw continuation through normal visible message fields. The requested audience distinction is plausible, but the exact hide-versus-system-block policy needs maintainer alignment with the broader internal-message work.
