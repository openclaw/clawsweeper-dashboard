## Summary

Keep open: current main still routes restart continuation text as an internal-provenance inbound agent turn while carrying the raw continuation in normal message body fields, and the WebChat/Control UI display projection does not hide or reclassify that `internal_system` provenance. The related audience and system-block tracks are active coordination points, but they are still open rather than shipped fixes for this continuation-specific path.

## What This Changes

The issue requests that Control UI/WebChat stop rendering restart/runtime continuation prompts as ordinary user-authored chat and instead treat them as hidden or explicitly system-scoped agent continuity state.

## Best Possible Solution

Use the existing internal provenance or a narrow continuation audience contract at the transcript/display boundary so the agent receives the continuation while WebChat either suppresses it or renders it through one shared runtime/system treatment.

## Reproduction Assessment

Yes, at source level. A restart sentinel `continuationMessage` becomes an `agentTurn` whose text is copied into `Body`/`RawBody`, and the display projection has no branch for `internal_system` restart-sentinel provenance.

## Solution Assessment

Unclear as an implementation direction because current main is not solving the UI boundary, and the exact hide-versus-system-block policy should align with the broader internal-message and audience-classification work.
