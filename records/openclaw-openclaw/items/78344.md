## Summary

Keep open: current main still dispatches restart continuations as inbound agent turns whose raw continuation text is present in normal message body fields, and WebChat/Control UI display projection does not hide or reclassify `internal_system` provenance. The related system-message and audience-classification issues are useful coordination points, but they are broader open tracks rather than a merged fix for this continuation-specific report.

## What This Changes

The issue requests that restart/runtime continuation messages reach the agent without appearing as ordinary user-authored chat in Control UI/WebChat.

## Best Possible Solution

Use the existing internal provenance or a narrow continuation audience contract at the transcript/display boundary so the agent receives the continuation while WebChat either suppresses it or renders it through one shared runtime/system treatment.

## Reproduction Assessment

Yes, at source level. A restart sentinel with `continuationMessage` becomes an `agentTurn` whose continuation text is dispatched in `Body`/`RawBody`, and the display projection has no handling for `internal_system` provenance.

## Solution Assessment

No, current main is not the best solution because it marks the turn as internal only in provenance while still carrying raw continuation text through normal visible message fields. The issue's audience distinction is a plausible direction, but the exact hide-versus-system-block policy needs maintainer alignment with the broader internal-message work.
