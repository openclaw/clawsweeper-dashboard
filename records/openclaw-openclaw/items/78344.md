## Summary

Keep open: current main still turns a restart continuation into an internal-provenance agent turn while preserving the raw continuation text in normal message body fields, and the display projection does not hide or reclassify `internal_system` restart-sentinel provenance. The broader system-block and event-audience tracks are still open, so this continuation-specific path is not implemented or superseded yet.

## What This Changes

The issue requests that restart continuation and runtime handoff messages be delivered to the agent without appearing as ordinary user-authored chat in Control UI/WebChat.

## Best Possible Solution

Use existing `internal_system` provenance or a shared audience contract at the transcript/display boundary so the agent receives the continuation while WebChat either suppresses it or renders it through one consistent runtime/system treatment.

## Reproduction Assessment

Yes, at source level. A restart sentinel `continuationMessage` becomes an `agentTurn`, is copied into normal body fields with `internal_system` provenance, and the display projection has no branch that hides or separates that provenance.

## Solution Assessment

Unclear until maintainers choose the UX/API boundary. Hiding `internal_system` restart continuations at projection time is narrow, but the durable solution should align with the broader system-message block and event-audience work.
