## Summary

Keep open: current main still turns a restart continuation into an internal-system agent turn while preserving the raw continuation text in normal message fields, and neither the gateway display projection nor Control UI hides or separately renders that provenance. The related system-block and audience-classification tracks remain open, so this is not implemented or superseded.

## What This Changes

Request that Control UI/WebChat stop rendering restart `continuationMessage` and runtime continuation prompts as ordinary user chat, either by hiding them from users or showing them as distinct runtime/system content.

## Best Possible Solution

Use existing `internal_system` provenance or a shared audience contract at the transcript/display boundary so the agent still receives the continuation while WebChat hides it or renders it through one consistent runtime/system treatment.

## Reproduction Assessment

Yes, at source level. A restart sentinel `continuationMessage` becomes an `agentTurn`, is copied into normal body fields with `internal_system` provenance, and current display projection has no branch that hides or separates that provenance.

## Solution Assessment

Unclear until maintainers choose the UX/API boundary. Hiding `internal_system` restart continuations at projection time is narrow, but the durable solution should align with the broader system-message block and event-audience work.
