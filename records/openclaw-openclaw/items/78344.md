## Summary

Keep open: current main still routes explicit restart/update continuations through the normal inbound agent-turn path with the continuation text as the message body, while the transcript hygiene docs treat runtime context as non-user-authored and related audience-classification work is still open.

## What This Changes

The issue requests that Control UI/WebChat stop rendering restart/runtime continuation prompts as ordinary user-authored chat and instead hide or clearly classify them as runtime/agent-only context.

## Best Possible Solution

Add typed provenance/audience for restart continuations so the agent receives the prompt without creating a visible user-authored turn; if transparency is desired, render it as a distinct runtime block.

## Reproduction Assessment

Yes for source-level reproduction: a restart or update with continuationMessage and a routable session goes through the tested restart-continuation path, where the continuation text is used as the inbound Body/RawBody. I did not run a live browser scenario in this read-only review.

## Solution Assessment

No, the current path is not the best solution because it models runtime continuation text as a normal inbound message. A safer direction is to separate agent-visible continuation input from transcript/display provenance and align it with the existing runtime-context hygiene contract.
