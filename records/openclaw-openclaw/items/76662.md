## Summary

Keep open. Current main still has the source-backed macOS Voice Wake gap for non-final text-only SpeechRecognition updates without usable timings, and the issue is now paired with open closing PR #77032, so it should remain open until that PR lands or is superseded.

## What This Changes

Make macOS Voice Wake trigger from SpeechRecognition text-only or interim results when usable word timings are absent, including Talk Mode activation while preserving false-positive guards.

## Best Possible Solution

Ship a bounded macOS runtime fallback that matches only newly appended text-only deltas when timings are absent, with regression coverage for trigger-only Talk Mode and boundary rejection.

## Reproduction Assessment

Yes, source-reproducible: on current main, a non-final update with trigger text and empty/no usable segments gets nil from WakeWordGate.match and skips the immediate text-only fallback because update.isFinal is false. I did not live-run Apple SpeechRecognition.

## Solution Assessment

Yes in direction: a narrow delta-aware text-only fallback in the macOS runtime is the maintainable fix, while broad whole-transcript substring matching would weaken the existing false-positive guards.
