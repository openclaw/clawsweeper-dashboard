## Summary

Keep open. Current main still has a source-backed macOS Voice Wake gap for non-final SpeechRecognition updates with trigger text but no usable word timing: segment matching cannot accept them, immediate text-only fallback is final-only, and the delayed fallback can miss trigger-to-Talk Mode activation while interim text keeps changing.

## What This Changes

Voice Wake should trigger from macOS SpeechRecognition text-only or interim results that contain a configured wake trigger even when usable word timings are missing.

## Best Possible Solution

Add a bounded macOS runtime fallback for no-usable-timing interim text, preferably based on newly appended transcript text, while preserving word-boundary and ordinary-mention rejection behavior.

## Reproduction Assessment

Yes, source-reproducible: a non-final recognition update with trigger text and empty or zero-timed segments leaves `WakeWordGate.match` nil and skips immediate `textOnlyFallbackMatch`. I did not establish a live Apple SpeechRecognition run in this read-only review.

## Solution Assessment

Yes, the maintainable fix is a narrow macOS Voice Wake fallback for segmentless/no-timing interim text rather than weakening the shared segment-gap gate globally.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/76662. Current main misses macOS Voice Wake triggers when SpeechRecognition emits non-final text with no usable word timing: `WakeWordGate.match` returns nil and `VoiceWakeRuntime` only runs immediate text-only fallback for final results. Add a bounded fallback in the macOS Voice Wake runtime/helper layer for interim no-timing text, preferably using newly appended transcript text so accumulated ordinary speech does not become a broad substring match. Preserve existing false-positive protections for filler prefixes, ordinary mentions, word boundaries, and suffixes like `computers`. Do not add a new setting or weaken shared `WakeWordGate` timing semantics unless tests show the shared contract needs it. Add focused Swift regression coverage for trigger-only Talk Mode activation, appended interim wake phrase, no usable timing, and suffix rejection, plus a user-facing `CHANGELOG.md` fix entry.
