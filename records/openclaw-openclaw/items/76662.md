## Summary

Keep open. Current main still has a source-backed macOS Voice Wake gap for segmentless interim SpeechRecognition updates, and open PR #77032 explicitly targets this issue, so the issue should stay open until that PR lands or is superseded.

## What This Changes

Voice Wake should activate on macOS when SpeechRecognition emits interim or text-only transcripts containing a configured trigger but no usable word timings.

## Best Possible Solution

Land or replace #77032 with a bounded delta-aware text-only fallback for segmentless interim recognition, with regression tests for trigger-only Talk Mode and false-positive boundaries.

## Reproduction Assessment

Yes, source-reproducible: a non-final recognition update with trigger text and zero usable segments leaves WakeWordGate.match nil and skips the immediate text-only fallback. I did not live-run Apple SpeechRecognition in this read-only review.

## Solution Assessment

Yes, the narrow maintainable direction is a delta-aware text-only fallback that reuses the existing text-boundary guards; broad substring matching would be riskier. Current main has not implemented that yet, while #77032 is the active candidate.
