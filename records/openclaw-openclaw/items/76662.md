## Summary

Keep open. Current main still has a source-backed macOS Voice Wake gap: segmentless non-final recognition updates can carry trigger text, but the runtime only applies the text-only fallback on final or delayed-silence paths.

## What This Changes

Fix macOS Voice Wake so text-only/interim SpeechRecognition results without usable word timings can still trigger Talk Mode or wake capture when they contain a configured trigger.

## Best Possible Solution

Add a regression-covered macOS Voice Wake fallback for segmentless interim updates, preferably using newly appended transcript text while leaving the segment-timing gate intact for normal results.

## Reproduction Assessment

Yes, source-reproducible: on current main, a non-final update with trigger text and empty/unusable segments leaves WakeWordGate.match nil and skips immediate text-only fallback. I did not live-run Apple SpeechRecognition, so this is source-backed rather than live reproduced.

## Solution Assessment

No. The current implementation relies on final text or delayed silence fallback; a bounded delta-aware fallback for no-timing interim transcripts is the narrower maintainable repair.

## Repair Work Prompt

Fix the macOS Voice Wake bug where Apple SpeechRecognition can emit interim/text-only results containing a configured wake trigger but no usable segment timings. Keep the fix scoped to the macOS Voice Wake runtime/helper surface unless a small Swabble helper is clearly necessary. Prefer a bounded fallback based on newly appended transcript text for no-segment interim updates, and preserve existing rejection of ordinary mentions and suffix matches such as `computers`. Add or update regression tests for: trigger-only Talk Mode activation from text-only interim input, appended wake phrase after prior interim text, text-only command capture without usable timings, and false-positive rejection. Include a user-facing CHANGELOG.md fix entry. Do not add a new config option or loosen WakeWordGate's segment-timing behavior globally without focused tests showing that is the intended contract.
