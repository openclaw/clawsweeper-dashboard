## Summary

Keep open. Current main still has a source-backed macOS Voice Wake gap for segmentless or unusably timed interim SpeechRecognition updates, especially when the wake phrase appears as newly appended text rather than as the whole transcript.

## What This Changes

The issue asks macOS Voice Wake to trigger from text-only SpeechRecognition updates when usable word timings are missing, including Talk Mode activation.

## Best Possible Solution

Add a bounded macOS runtime fallback for no-timing interim transcript deltas, preserving word-boundary checks and ordinary-mention rejection, with regression coverage for Talk Mode activation.

## Reproduction Assessment

Yes, source-reproducible: a non-final transcript containing an appended trigger with empty or unusable segments bypasses WakeWordGate.match and the immediate final-only fallback, then the existing text-only helper rejects ordinary-prefix accumulated text. I did not live-run the macOS SpeechRecognition path.

## Solution Assessment

No, the current implementation is not the best solution because it relies on final results or whole-transcript fallback checks. A narrow delta-aware fallback in VoiceWakeRuntime is the safer repair than loosening WakeWordGate's timing contract globally.
