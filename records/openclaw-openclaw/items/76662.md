## Summary

Keep open. Current main still has a source-backed macOS Voice Wake gap: text-only/interim SpeechRecognition updates without usable segment timings can miss the wake trigger before Talk Mode or capture starts.

## What This Changes

The issue requests macOS Voice Wake activation from text-only SpeechRecognition results when word timings are unavailable, including interim/appended transcripts and Trigger Talk Mode.

## Best Possible Solution

Add a narrow macOS Voice Wake text-only interim/delta fallback with regression coverage, while preserving the segment-aware `WakeWordGate` timing semantics and existing false-positive guards.

## Reproduction Assessment

Yes, source-reproducible: an interim recognition update with a trigger in text and empty or zero-timed segments deterministically misses the segment gate, skips the final-only fallback, and can be rejected by the existing full-transcript text-only guard. I did not live-reproduce it with Apple SpeechRecognition in this checkout.

## Solution Assessment

Yes, the best path is a narrow runtime/helper fallback for text-only interim deltas, backed by tests for no usable timings, appended trigger text, Trigger Talk Mode, and suffix false positives. Changing the shared segment gate to ignore timing would be a broader and riskier fix.
