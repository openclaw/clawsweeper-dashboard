## Summary

Keep open. Current main still has a source-backed macOS Voice Wake gap for segmentless interim SpeechRecognition updates: the segment-aware gate returns no match without segments, immediate text-only fallback is final-only, and the delayed paths can miss trigger-to-Talk Mode activation while interim text changes.

## What This Changes

The issue requests macOS Voice Wake activation when Apple SpeechRecognition returns trigger text without usable word timings, including Talk Mode activation when that setting is enabled.

## Best Possible Solution

Add a bounded macOS runtime fallback for segmentless interim text, preferably based on newly appended transcript text, while preserving word-boundary and ordinary-mention rejection behavior.

## Reproduction Assessment

Yes, source-reproducible: a non-final recognition update with trigger text and `segments=[]` leaves `WakeWordGate.match` nil and skips immediate text-only fallback. I did not live-run Apple SpeechRecognition.

## Solution Assessment

Yes, a narrow macOS runtime fallback for segmentless interim updates is the best direction; changing the shared segment-aware gate or using broad substring matching would risk false positives.

## Repair Work Prompt

Fix macOS Voice Wake so Apple SpeechRecognition interim updates with transcript text but no usable word timings can still activate a configured wake trigger, including `voiceWakeTriggersTalkMode=true`. Keep the fix in the macOS Voice Wake runtime/helper layer; do not weaken `WakeWordGate`'s segment-aware gap semantics or add config knobs. Prefer a bounded fallback based on newly appended transcript text so ordinary earlier speech plus an appended wake trigger can be handled without accepting suffix matches like `computers` or broad ordinary mentions. Add regression coverage in `apps/macos/Tests/OpenClawIPCTests/VoiceWakeRuntimeTests.swift` for no-segment non-final trigger-only Talk Mode activation, appended interim trigger text after earlier non-wake speech, and suffix/ordinary-mention rejection. Update `CHANGELOG.md` because this is user-facing. Related refs: https://github.com/openclaw/openclaw/issues/76662 and https://github.com/openclaw/openclaw/pull/58490.
