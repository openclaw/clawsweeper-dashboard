## Summary

Keep open. Current main still has a source-backed macOS Voice Wake gap for text-only/interim recognition updates without usable word timings, and open PR #77032 is already linked as the fix path.

## What This Changes

Voice Wake should trigger on macOS when SpeechRecognition emits a configured wake phrase as text-only/interim results without usable word timings.

## Best Possible Solution

Review and land PR #77032 or an equivalent delta-aware macOS fallback with regression coverage for no-timing interim updates and boundary rejection cases.

## Reproduction Assessment

Yes, source-reproducible: a non-final recognition update containing a trigger with zero usable segments leaves WakeWordGate.match nil and skips the immediate text-only fallback. I did not live-run Apple SpeechRecognition in this read-only review.

## Solution Assessment

Yes, a bounded text-only delta fallback in the macOS runtime is the narrow maintainable repair; relaxing WakeWordGate globally would risk more false positives.
