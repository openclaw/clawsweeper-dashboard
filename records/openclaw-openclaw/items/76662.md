## Summary

Keep open. Current main still has a source-backed macOS Voice Wake gap: segment matching needs usable SpeechRecognition segments, the immediate text-only fallback is final-only, and the delayed fallback can miss interim or appended wake-trigger text before Talk Mode or capture starts.

## What This Changes

Voice Wake should trigger from text-only/interim macOS SpeechRecognition results when no usable word timing data is available.

## Best Possible Solution

Add a bounded macOS Voice Wake fallback for segmentless interim recognition, preferably based on newly appended transcript text, while preserving trigger boundary, filler-prefix, suffix-rejection, and Talk Mode activation behavior.

## Reproduction Assessment

Yes, source-reproducible: a non-final recognition update with trigger text and zero usable segments leaves WakeWordGate.match nil and skips the immediate text-only fallback. The currently tested helpers do not cover the appended/interim no-segment sequence described by the reporter.

## Solution Assessment

No, the current implementation is not the best way to solve the reported behavior because it relies on final results or delayed silence checks for text-only recognition. A narrow delta-aware text-only fallback in VoiceWakeRuntime is the safer repair than broad substring matching.

## Repair Work Prompt

Fix the macOS Voice Wake bug reported in https://github.com/openclaw/openclaw/issues/76662: when Apple SpeechRecognition returns interim/text-only results with zero usable word timings, a configured wake trigger should still activate capture or Trigger Talk Mode. Keep the change bounded to the macOS Voice Wake detection path and supporting helper tests. Preserve existing false-positive protections for ordinary mentions, filler-only prefixes, and suffix matches such as `computers`. Add or update regression tests for: text-only wake trigger with no usable timings, accumulated/interim transcript where a wake phrase is appended after earlier non-wake speech, trigger-only Talk Mode activation when `voiceWakeTriggersTalkMode=true`, and suffix/ordinary-mention rejection.
