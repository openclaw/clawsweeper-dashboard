## Summary

Keep open. Current main still has the source-backed macOS Voice Wake gap for segmentless interim SpeechRecognition updates, and the open fix PR #77032 explicitly references this issue with closing syntax, so this issue should remain open until that PR lands or is replaced.

## What This Changes

Make macOS Voice Wake trigger from text-only/interim SpeechRecognition updates when Apple does not provide usable word timings, including trigger-only Talk Mode activation.

## Best Possible Solution

Review and land PR #77032, or an equivalent bounded appended-delta fallback in the macOS runtime with regression tests for no-timing interim updates and false-positive boundaries.

## Reproduction Assessment

Yes, source-reproducible. A non-final recognition update with transcript text and zero usable segments leaves `WakeWordGate.match` nil, and current main only runs immediate text-only fallback for final updates.

## Solution Assessment

Yes, the linked PR's appended-delta fallback is the narrow maintainable direction because it keeps the existing text-only guards while matching only newly appended interim text. It still needs normal PR review and validation before this issue should close.
