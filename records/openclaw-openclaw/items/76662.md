## Summary

Keep open: current main still has the source-backed no-timing/interim Voice Wake gap, and #77032 is an open closing PR that should be reviewed before this issue closes.

## What This Changes

macOS Voice Wake should trigger when SpeechRecognition produces text-only or interim results containing a configured wake word but no usable word timings.

## Best Possible Solution

Review and land #77032, or an equivalent narrow macOS runtime fallback that matches newly appended no-timing interim transcript text while preserving word-boundary and Talk Mode trigger-only coverage.

## Reproduction Assessment

Yes, source-reproducible: on current main, a non-final update with trigger text and empty or zero-timing segments makes `WakeWordGate.match` return nil and skips the final-only text fallback. I did not live-run macOS SpeechRecognition in this read-only review.

## Solution Assessment

Yes, the narrow delta-aware fallback proposed by #77032 is the right repair boundary because it targets segmentless interim text without broad substring matching. The issue should stay open until that PR or an equivalent fix merges.
