## Summary

Keep open. Current main still has a source-backed Voice Wake gap for segmentless/interim SpeechRecognition updates: segment matching needs usable segments, immediate text-only fallback is final-only, and the delayed fallback can miss trigger-to-Talk Mode activation while interim text keeps changing.

## What This Changes

The issue requests that macOS Voice Wake trigger from SpeechRecognition text-only/interim results when word timing data is unavailable or unusable.

## Best Possible Solution

Add a bounded macOS Voice Wake fallback for text-only/interim no-timing updates, preferably based on newly appended transcript text, while preserving word-boundary and ordinary-mention rejection tests.

## Reproduction Assessment

Yes, source-reproducible: a non-final recognition update with trigger text and zero usable segments leaves `WakeWordGate.match` nil and skips immediate text-only fallback, while later interim text can cancel the trigger-only pause path. I did not live-run Apple SpeechRecognition in this read-only review.

## Solution Assessment

No, the current implementation is not the best fix for the reported behavior because it relies on final results or delayed silence after interim text. A narrow delta-aware text-only fallback in the macOS runtime is the safer repair than broad substring matching.
