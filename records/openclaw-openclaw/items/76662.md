## Summary

Keep open. Current main still has a source-backed macOS Voice Wake gap for segmentless interim recognition updates, and the linked fix PR #77032 is open with closing linkage, so this issue should stay open until that PR or an equivalent fix lands.

## What This Changes

Make macOS Voice Wake activate from text-only/interim SpeechRecognition results that contain a configured wake trigger but no usable word timings.

## Best Possible Solution

Review and land PR #77032, or an equivalent narrow delta-aware text-only fallback with regression tests, then let the issue close through the merged fix.

## Reproduction Assessment

Yes, source-reproducible: a non-final recognition update with trigger text and zero usable segments leaves `WakeWordGate.match` nil and skips the immediate text-only fallback. Continued interim text can then reset the delayed trigger-only path before Talk Mode activation.

## Solution Assessment

Yes, the right repair boundary is a bounded macOS runtime fallback for newly appended text-only wake deltas while preserving word-boundary and ordinary-mention rejection. The linked PR should be reviewed rather than opening duplicate work.
