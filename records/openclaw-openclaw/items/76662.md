## Summary

Keep open. Current main still has the source-backed macOS Voice Wake gap for segmentless interim SpeechRecognition updates, and the linked open PR #77032 is the right implementation candidate to review before this issue closes.

## What This Changes

Voice Wake should trigger Talk Mode or wake capture from text-only macOS SpeechRecognition results when configured trigger text is present but word timings are unavailable.

## Best Possible Solution

Land #77032 or an equivalent narrow macOS runtime fallback that matches newly appended text-only interim transcript deltas while preserving word-boundary and false-positive guards.

## Reproduction Assessment

Yes, source-reproducible: on current main, a non-final update with transcript text and no usable segments gets nil from `WakeWordGate.match` and skips immediate text-only fallback because `update.isFinal` is false. I did not live-run Apple SpeechRecognition.

## Solution Assessment

Yes, the linked delta-aware fallback direction is the narrowest maintainable fix because it keeps the timing-sensitive gate intact and handles only macOS text-only interim recognition. Broadly relaxing `WakeWordGate` would raise false-positive risk across callers.
