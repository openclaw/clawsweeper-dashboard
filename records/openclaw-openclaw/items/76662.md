## Summary

Keep open. Current main still has the source-backed macOS Voice Wake gap for interim/text-only SpeechRecognition updates without usable word timings, and an open closing PR (#77032) is already paired with this issue, so it should remain open until that PR or an equivalent fix merges.

## What This Changes

The issue asks macOS Voice Wake to trigger from text-only SpeechRecognition updates when configured wake words appear but segment timing data is unavailable.

## Best Possible Solution

Review and land #77032 or an equivalent delta-aware macOS Voice Wake fallback that matches newly appended text-only interim transcripts while preserving word-boundary and false-positive protections.

## Reproduction Assessment

Yes, source-reproducible: a non-final recognition update containing a trigger with zero usable segments leaves WakeWordGate.match nil, skips immediate fallback, and can be delayed or cancelled by later interim text. I did not perform live macOS SpeechRecognition validation in this read-only pass.

## Solution Assessment

Yes, the narrowest maintainable fix is a bounded macOS runtime fallback for newly appended text-only interim recognition, not broad substring matching. The open #77032 diff appears aimed at that boundary and should be reviewed or replaced with equivalent coverage.
