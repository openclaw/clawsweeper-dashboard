  - body: The new status, dispatch, and recovery code reads and patches replyTurnState, replyTurnStartedAt, replyTurnUpdatedAt, replyTurnRunId, and replyTurnLastError as SessionEntry fields, but the branch never adds them to the SessionEntry contract. - **[P2] Only mark actually delivered progress as visible:** `src/auto-reply/reply/dispatch-from-config.ts:1487`

  - confidence: 0.83

  - body: This removes the existing isError exception from the suppressed-progress guard. With suppressDefaultToolProgressMessages active, a text-only tool error is swallowed the same as routine progress, so direct and preview-suppressed flows can hide the only useful failure text.
  - confidence: 0.8
