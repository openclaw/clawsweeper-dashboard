## What This Changes

The PR changes ACP spawn label persistence to write the requested label into the spawned ACP target session store and adds a regression test for requester-store versus target-store mismatch.

## Best Possible Solution

## Reproduction Assessment

Yes. Current main has a high-confidence source reproduction: a cross-agent `/acp spawn ... --label` creates an `agent:codex:acp:*` target session, but label persistence still writes through the requester `storePath`; the PR regression models that requester/target store split.

## Solution Assessment

Yes for the code direction. Resolving the target store from the spawned ACP `sessionKey` is the narrow maintainable fix because it reuses the same store-selection helper used by ACP metadata, while same-store in-memory mutation remains guarded by path equality.
