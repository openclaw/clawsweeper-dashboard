## Summary

Keep open. Current main already avoids repeatedly reading full SKILL.md bodies and reuses a skill snapshot, but it still injects a compact eligible-skills catalog and explicitly tells the model to scan it before replying when skills are available; there is no current semantic on-demand suppression for casual chat turns.

## What This Changes

Request support for on-demand skill catalog injection so pure chat turns can skip scanning or receiving the full available-skills list.

## Best Possible Solution

Design a core-owned, opt-in skill catalog policy that can suppress or narrow the prompt catalog for low-tool or casual turns while preserving explicit skill commands, per-agent allowlists, prompt-cache determinism, and current SKILL.md on-demand reads; document the tradeoff and add focused prompt/snapshot tests.

## Reproduction Assessment

Yes. A high-confidence reproduction is any run with an eligible skill snapshot: `buildAgentSystemPrompt` receives a nonempty `skillsPrompt`, emits `## Skills (mandatory)`, and includes `<available_skills>` even for messages like a greeting.

## Solution Assessment

Unclear. The requested direction addresses real token overhead, but the best implementation needs maintainer design because naive semantic filtering can hide needed skills and destabilize prompt caching; existing allowlists and prompt limits are only manual mitigations.
