## Summary

This PR is still necessary: current main does not load memory embedding provider owners from `agents.*.memorySearch.provider`, while the branch adds that startup-planning path and warning coverage without a blocking code defect found. It remains compatibility-sensitive because existing configured setups may now activate a provider plugin earlier at Gateway startup.

## What This Changes

The branch updates Gateway startup plugin planning and tests to load plugins that own configured memory embedding providers, including custom provider API owners, and adds a startup warning for unregistered configured memory embedding providers.

## Best Possible Solution

Land a bounded startup-planning fix that loads only explicitly configured, enabled memory embedding provider owners under existing allow/deny rules, while tracking the runtime provider-visibility race separately in https://github.com/openclaw/openclaw/issues/89691.

## Solution Assessment
