## Summary

Keep this PR open: it is a focused bug fix for a current-main startup planning gap, and I did not find a blocking code defect. It still needs maintainer review because explicit `memorySearch.provider` or `memorySearch.fallback` configs can now load and activate an owning provider plugin during Gateway startup, which is compatibility-sensitive even when it fixes silent FTS-only recall.

## What This Changes

The branch adds memory embedding provider and fallback owner collection to Gateway startup plugin planning, extracts shared embedding-provider config lookup, warns on unregistered configured memory embedding providers, and adds regression coverage.

## Best Possible Solution

Land this startup-planning fix after maintainer acceptance of the compatibility behavior, and keep the runtime provider-loss work with https://github.com/openclaw/openclaw/issues/89691 and https://github.com/openclaw/openclaw/pull/90336.
