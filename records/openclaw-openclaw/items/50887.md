## Best Possible Solution

Keep this issue open until an upstream fix makes default UTF-8 stream decoding stateful, preserves Windows legacy-codepage fallback, adds split-CJK regression coverage, and resolves the linked fork PR by landing or superseding it.

## Reproduction Assessment

Yes on direction, but not via the current fork PR as-is. The maintainable fix point is the shared decoder and any verified socket route, with tests for split CJK bytes and preservation of the Windows fallback behavior.
