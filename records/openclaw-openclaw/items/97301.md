## Summary

Keep open: current main still has the embedded context-engine afterTurn path without a Codex-style transcript mirror or explicit flush before the hook, while the suppressed-user-message path can leave the durable JSONL missing the current user turn.

## What This Changes

The issue requests that the embedded context-engine host guarantee the session transcript is durable before afterTurn, or explicitly document that afterTurn has no such guarantee.

## Best Possible Solution

Make the embedded context-engine host use one durable transcript-before-afterTurn contract that matches the Codex harness, with focused regression coverage for suppressed user-message turns.

## Reproduction Assessment

Yes from source, but not from a live run: current main still has the embedded afterTurn call without a Codex-style mirror or explicit flush, and the suppressed-user-message path can skip the runtime-event write.

## Solution Assessment

Yes, the best fix is likely a narrow embedded-host persistence guarantee before afterTurn rather than asking plugins to special-case a host timing gap that the Codex harness avoids.
