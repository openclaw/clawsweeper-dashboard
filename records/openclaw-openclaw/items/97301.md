## Label Justifications

- `P2`: This is a normal-priority core runtime consistency bug with limited but real impact on embedded context-engine consumers.
- `impact:session-state`: A context-engine `afterTurn` consumer can reconcile against stale or incomplete durable session state.
- `impact:message-loss`: Suppressed first-turn user messages can be absent from the durable transcript that a reconciling plugin reads, causing duplicate import or continuity loss.

## Summary

Keep open: current main still leaves the embedded context-engine `afterTurn` path without the Codex app-server host's transcript mirror/readback step, and the suppressed room-event path can leave the durable transcript missing the current user turn for `sessionFile` consumers.

## What This Changes

The issue asks OpenClaw to guarantee that the embedded context-engine host flushes the session JSONL before `afterTurn`, or to document that no such guarantee exists.

## Best Possible Solution

Make the embedded host establish the same durable transcript-before-`afterTurn` contract as the Codex host, without adding a config flag or channel-specific policy.

## Reproduction Assessment

Yes at source level: the embedded finalizer passes memory messages to `afterTurn` without a file mirror/readback, `persistRecord` can defer first user-only writes, and suppressed room events skip user persistence. I did not run a live gateway/plugin reproduction.

## Solution Assessment

Yes: a narrow embedded-host ordering fix is the best path because the Codex host already establishes post-mirror finalization and context-engine plugins receive `sessionFile` during `afterTurn`.
