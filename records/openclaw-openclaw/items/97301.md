## Label Justifications

- `impact:session-state`: The affected behavior is whether context-engine lifecycle hooks can reconcile against the durable session transcript for the current turn.

## Summary

Keep open: current main still finalizes embedded context-engine turns from an in-memory snapshot before any explicit transcript mirror/flush, while room-event turns can intentionally suppress the durable user-message write.

## What This Changes

The issue asks OpenClaw to make embedded context-engine `afterTurn` see a durable session transcript for the completed turn, or to document that no such guarantee exists.

## Best Possible Solution

Give the embedded context-engine host one durable transcript-before-afterTurn contract, matching the Codex app-server host where feasible, with regression coverage for suppressed room-event turns.

## Reproduction Assessment

Yes from source, but not from a live run: current main still calls embedded `afterTurn` without a pre-hook transcript mirror/flush, and room-event suppression can skip the durable user-message write.

## Solution Assessment

Yes: the narrow maintainable fix is to make embedded finalization materialize the current turn before `afterTurn`, rather than requiring every reconciling plugin to special-case a host timing gap.
