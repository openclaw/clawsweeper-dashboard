## Summary

## What This Changes

The issue asks OpenClaw to diagnose or fix `TypeError: fetch failed` during QR login for the external `@tencent-weixin/openclaw-weixin` channel plugin.

## Best Possible Solution

Track and ship the fix in `Tencent/openclaw-weixin`, then have affected users update or reinstall the plugin; OpenClaw core should stay on the generic channel contract unless upstream identifies a missing SDK/API seam.

## Reproduction Assessment

Yes, for the external plugin path: multiple users reproduced `openclaw channels login --channel openclaw-weixin` with `@tencent-weixin/openclaw-weixin@2.4.1`, and the upstream issue/PR identifies the package-owned fetch wrapper. I did not reproduce the full Gateway flow inside this read-only OpenClaw checkout.

## Solution Assessment

Yes: routing the fix to the external Weixin plugin is the narrowest maintainable solution because OpenClaw core only delegates channel login to plugin `auth.login` and documents Weixin API/login behavior as plugin-owned.
