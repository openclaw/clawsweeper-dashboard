## What This Changes

The issue reports that `openclaw channels login --channel openclaw-weixin` fails with `TypeError: fetch failed` for `@tencent-weixin/openclaw-weixin@2.4.1` on OpenClaw 2026.5.4/2026.5.5.

## Best Possible Solution

Keep the current main lazy-loading fix and ship it in the next release; the plugin upstream should still remove its manual `Content-Length`, but OpenClaw core no longer needs a separate change for this report.

## Reproduction Assessment

Yes for the affected releases: OpenClaw 2026.5.4/2026.5.5 with `@tencent-weixin/openclaw-weixin@2.4.1` reproduces via `openclaw channels login --channel openclaw-weixin`, and the merged related fix includes before/after live output. I did not establish a failing current-main path because current main contains the lazy-loading fix.

## Solution Assessment

Yes. Lazy-loading Undici only when proxy or dispatcher work is actually needed is the narrow maintainable OpenClaw fix because it preserves the configured proxy paths while avoiding no-proxy side effects for external plugin `fetch()`.
