Keep open. Current main still appears to forward read-tool path arguments exactly as received, while the only adjacent repairs handle malformed tool-call framing rather than a valid JSON string whose path text was already mutated.

## Best Possible Solution

Add focused regression coverage for mixed CJK+ASCII read-tool arguments and the local-provider tool-call boundary, then fix argument preservation or emit precise diagnostics without broad filesystem guessing.
