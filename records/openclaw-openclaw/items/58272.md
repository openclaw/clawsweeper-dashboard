## Summary

Close as ClawHub/plugin-scope cleanup: the remaining behavior is opt-in CJK/ASCII path de-spacing for model-mutated tool arguments, while current core read paths preserve exact path strings and the existing plugin hook surface can rewrite parameters before execution.

## What This Changes

The issue requests that OpenClaw read mixed CJK+ASCII paths on macOS without ENOENT when model or tool-call text inserts spaces at script boundaries.

## Best Possible Solution

Keep core path handling literal and route optional CJK/ASCII de-spacing to a ClawHub code plugin using `before_tool_call`, with validation that the original path is missing and the corrected path exists.

## Reproduction Assessment

No. The issue includes plausible ENOENT evidence, but current source preserves the path string passed to the read tool and there is no current-main macOS/oMLX trace showing whether the model/parser or the read tool mutated the argument.

## Solution Assessment

Yes for the remaining auto-correction behavior: an opt-in ClawHub plugin is the narrowest maintainable solution because a core heuristic could silently rewrite legitimate filenames or command text.
