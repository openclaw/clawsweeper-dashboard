Keep open. The PR targets a real current-main gateway auth bug, but it is not merge-ready because it lacks after-fix real behavior proof and leaves route-level trusted-operator expectations inconsistent with the resolver change.

## What This Changes

The PR changes trusted-operator plugin route scope resolution so shared-secret auth receives only operator.write and updates the focused resolver unit test.

## Best Possible Solution

Keep the write-only shared-secret resolver direction, update the dispatch-level trusted-operator tests to match it, and merge only after contributor-supplied real behavior proof plus normal security review.

Yes. Current-main source inspection shows token/password auth sets trustDeclaredOperatorScopes false and trusted-operator route resolution converts that path into default scopes containing operator.admin; existing integration tests assert the same behavior.

## Solution Assessment

No, not yet. Changing the central resolver is the narrow fix direction, but this PR must also update route-dispatch expectations and provide real runtime proof before it is a mergeable solution.
