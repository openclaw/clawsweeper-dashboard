## Summary

Keep open: current main still creates and updates the native Windows Gateway Scheduled Task without disabling Task Scheduler battery restrictions, and no open or merged related PR owns this remaining fix.

## What This Changes

The issue reports the native Windows npm Gateway daemon stopping or crashing when a laptop switches from AC power to battery.

## Best Possible Solution

Set both Task Scheduler battery settings to false for new and existing Gateway Scheduled Tasks, then add focused coverage for create, update, and fallback behavior.

## Reproduction Assessment

Yes, source-reproducible: install the Gateway as a native Windows Scheduled Task, then switch the laptop from AC power to battery. I did not run a live battery test, but current code and the Task Scheduler power-setting contract expose the reported failure path.

## Solution Assessment

Yes: the narrow fix belongs in the Windows `schtasks` install/update owner path, not the generic Gateway loop. The implementation should preserve the `/Change` behavior from the related merged PR while explicitly repairing the two power settings.
