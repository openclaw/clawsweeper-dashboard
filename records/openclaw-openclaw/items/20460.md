## Summary

Keep open: current main and the latest release still store config backups beside the active config as `.bak` plus numbered `.bak.N` files. The direct directory-layout PR is closed unmerged, and the open related timestamp PR only partially overlaps because it keeps backups adjacent.

## What This Changes

The issue requests moving OpenClaw config backup files into a dedicated backup directory and replacing numeric backup suffixes with datetime-based names.

## Best Possible Solution

Define one canonical config-backup layout, then implement directory placement and datetime naming together with retention, permissions, legacy handling, recovery/audit/doc updates, and focused tests.

## Reproduction Assessment

Yes for the current behavior claim: source and focused tests show adjacent `.bak` plus numbered `.bak.1` through `.bak.4` files. This is a feature request rather than a broken existing behavior report.

## Solution Assessment

Unclear until maintainers pick the canonical backup layout and legacy handling. A timestamp-only suffix change is not the full solution because this issue also asks to move backups into their own directory and current recovery/audit paths assume an adjacent primary backup.
