## Summary

Keep open: current main and the latest release still store config backups beside the active config as `.bak` plus numbered `.bak.N` files. The direct directory-plus-datetime PR is closed unmerged, while the open related timestamp work does not satisfy this issue's dedicated-directory request.

## What This Changes

The issue requests moving OpenClaw configuration backups into a dedicated backup directory and replacing numbered backup suffixes with datetime-based names.

## Best Possible Solution

Define one canonical config-backup layout, then implement directory placement and collision-safe datetime naming with retention, permissions, legacy handling, recovery selection, audit output, doctor docs, and focused upgrade tests.

## Reproduction Assessment

Yes for the current behavior claim: current-main source and focused tests show adjacent `.bak` plus numbered `.bak.1` through `.bak.4` backups. This is a feature request rather than a broken existing behavior report.

## Solution Assessment

Unclear until maintainers choose the canonical backup layout and legacy handling. A timestamp-only suffix change is not the full solution because this issue also asks to move backups into their own directory and current recovery/audit paths assume an adjacent primary backup.
