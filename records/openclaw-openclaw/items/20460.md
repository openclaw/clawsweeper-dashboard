## Summary

Keep open. Current main and v2026.6.5 still create adjacent `.bak` plus numbered `.bak.N` config backups; the direct directory PR closed unmerged, and the open timestamp-only PR does not implement the requested dedicated directory or settle the upgrade contract.

The issue requests moving OpenClaw config backups into a dedicated backup directory with datetime-stamped filenames.

## Best Possible Solution

Implement one core-owned backup storage contract that covers directory location, sortable collision-safe timestamps, bounded retention, owner-only permissions, legacy adjacent backup handling through doctor, recovery lookup, audit output, docs, and focused tests.

## Reproduction Assessment

## Solution Assessment

No, a simple file move or suffix rename is not enough. The best solution is a coordinated backup-layout contract that updates creation, cleanup, recovery, security, legacy migration, audit output, and docs together.
