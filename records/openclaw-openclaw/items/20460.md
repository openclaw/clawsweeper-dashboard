## Summary

Keep open: current main and the latest release still use adjacent numbered config backups, while the related open work only covers timestamp suffixes and does not implement the requested dedicated backup directory.

## What This Changes

The issue requests moving config backup files into a dedicated backup directory with datetime-based backup names instead of adjacent numbered files.

## Best Possible Solution

Adopt one unified backup-layout design that moves managed config backups into a dedicated directory with collision-safe datetime names, safe legacy migration, preserved permissions/retention, and coordinated recovery, audit, docs, changelog, and tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main and `v2026.5.7` still create adjacent `.bak` and `.bak.N` files through `maintainConfigBackups`.

## Solution Assessment

No current implementation solves the whole issue. The maintainable path is a maintainer-chosen backup-layout contract that updates creation, retention, permissions, recovery, audit, docs, and focused tests together.
