## Summary

Keep open: current main and v2026.5.12 still use adjacent `.bak` plus numbered `.bak.N` config backups. ## What This Changes

The issue requests moving config backups into a dedicated `config-backup/` directory and naming them with datetime-stamped suffixes.

## Best Possible Solution

Adopt one coherent config-backup layout with a dedicated directory, collision-safe datetime names, bounded retention, owner-only permissions, safe legacy handling, and matching recovery, audit, docs, tests, and changelog coverage.

Not applicable as a bug reproduction; this is a feature request. Source inspection of current main and `v2026.5.12` gives high confidence that config writes still create adjacent `.bak` and numbered `.bak.N` files through `maintainConfigBackups`.

Unclear until maintainers choose the backup-layout contract. A dedicated directory may be the right end state, but the maintainable fix must update creation, retention, permissions, legacy handling, recovery, audit output, docs, and tests together.
