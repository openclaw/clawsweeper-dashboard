## Summary

Keep open: current main and the latest release still store config backups as adjacent `.bak` plus numbered `.bak.N` files, while this request asks for a dedicated backup directory with datetime names. The direct implementation PR was closed unmerged, and the active related PRs only cover timestamp suffixes beside the config file.

## What This Changes

Move managed OpenClaw config backups into a dedicated directory using datetime-based names instead of adjacent numbered `.bak` files.

## Best Possible Solution

Adopt one coherent config-backup layout that moves managed backups into a dedicated directory with collision-safe datetime names, bounded retention, owner-only permissions, safe legacy handling, recovery/audit/doctor updates, focused tests, and changelog coverage.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main and `v2026.5.12` still create adjacent `.bak` and `.bak.N` files through `maintainConfigBackups`.

## Solution Assessment

No current implementation solves the whole issue. The maintainable path is a maintainer-chosen backup-layout contract that updates creation, retention, permissions, legacy handling, recovery, audit, docs, and focused tests together.
