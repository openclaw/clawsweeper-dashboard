## Summary

Keep open. Current main still stores config backups beside the active config as a numbered `.bak` ring, and the related open PR only addresses datetime suffixes for adjacent backups rather than the requested dedicated backup directory.

## What This Changes

Store OpenClaw config backups under a dedicated `config-backup/` directory and name them with sortable datetime suffixes instead of numeric adjacent files.

## Best Possible Solution

Implement one shared config-backup path helper used by write, rotation, recovery, audit, and docs; keep managed backups in `config-backup/` with sortable UTC datetime names, collision handling, bounded retention, owner-only permissions, and safe legacy adjacent-backup handling.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: any existing config write calls `maintainConfigBackups`, which creates `${configPath}.bak` and rotates numbered adjacent backups that current tests explicitly expect.

## Solution Assessment

No. Current main does not solve the requested dedicated-directory behavior, and PR #40392 is only a partial datetime-suffix solution unless it also moves backups and updates recovery, audit, docs, legacy handling, and security hardening.
