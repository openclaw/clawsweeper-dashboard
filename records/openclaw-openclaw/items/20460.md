## Summary

Keep open: current main still uses adjacent numbered config backups, and recovery, audit, docs, and tests still assume that layout. The related implementation attempts are either closed unmerged or only cover timestamp suffixes, so the broader directory-plus-datetime request still needs maintainer design direction.

## What This Changes

Request to store OpenClaw config backup files in a dedicated backup directory with datetime-based names instead of adjacent numbered `.bak` files.

## Best Possible Solution

Adopt one unified backup-layout design that moves managed config backups into a dedicated directory with collision-safe datetime names, preserves or migrates legacy adjacent backups safely, and updates recovery, audit, docs, changelog, and focused tests together.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection on current main gives high confidence that config writes still create adjacent `.bak` and `.bak.N` files through `maintainConfigBackups`.

## Solution Assessment

No current implementation solves the whole issue. The maintainable path is a maintainer-chosen backup-layout contract that updates creation, retention, permissions, recovery, audit, docs, and tests together.
