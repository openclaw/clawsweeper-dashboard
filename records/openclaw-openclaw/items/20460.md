## Summary

Keep open: current main and the latest release still use adjacent `.bak` plus numbered `.bak.N` config backups, and the direct directory-plus-datetime PR was closed unmerged. The related open PRs only address timestamp suffixes beside the config file, so this broader dedicated-directory request still needs maintainer design.

## What This Changes

Request to store OpenClaw config backup files under a dedicated `config-backup/` directory using datetime-based names instead of adjacent numbered `.bak.N` files.

## Best Possible Solution

Adopt one unified backup-layout design that moves managed config backups into a dedicated directory with collision-safe datetime names, safe legacy migration, owner-only permissions, bounded retention, and coordinated recovery, audit, docs, changelog, and tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main and `v2026.5.12` still create adjacent `.bak` and `.bak.N` files through `maintainConfigBackups`.

## Solution Assessment

No current implementation solves the whole issue. The maintainable path is a maintainer-chosen backup-layout contract that updates creation, retention, permissions, legacy migration, recovery, audit, docs, and focused tests together.
