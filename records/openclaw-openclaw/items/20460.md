## Summary

Keep open. Current `main` still writes config backups beside the active config as `${configPath}.bak` plus numbered `.bak.1` through `.bak.4` files, and tests, recovery, audit, mutation, and doctor docs still depend on that adjacent layout. The direct directory implementation PR was closed unmerged, while the newer open related PR only addresses datetime suffixes for adjacent backup files, so the requested dedicated `config-backup/` directory has not landed.

## What This Changes

Request to move OpenClaw config backup files out of the main `.openclaw` directory into a dedicated backup subdirectory and replace numeric backup suffixes with datetime-based names.

## Best Possible Solution

Land a coordinated config-backup design that writes generated backups into a dedicated directory, uses UTC datetime-based names with collision handling, preserves the current retention and permission guarantees, handles legacy adjacent `.bak` files safely, and updates recovery, audit, doctor docs, and focused config tests.
