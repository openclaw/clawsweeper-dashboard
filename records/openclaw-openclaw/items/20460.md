## Summary

Keep open. Current main still stores config backups beside the active config as a numbered `.bak` ring, while the direct directory-and-datetime PR was closed unmerged and the open related work only covers datetime suffixes for adjacent backups.

## What This Changes

Store OpenClaw configuration backup files in a dedicated `config-backup/` directory using datetime-based backup names instead of adjacent numbered `.bak` files.

## Best Possible Solution

Implement one shared config-backup path and retention helper used by write, rotation, recovery, audit, and docs, with `config-backup/` datetime names, collision handling, bounded retention, owner-only permissions, and safe legacy adjacent-backup handling.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: existing config writes call `maintainConfigBackups`, which creates `${configPath}.bak` and rotates numbered adjacent backups that current tests explicitly expect.

## Solution Assessment

No. Current main does not solve the requested dedicated-directory behavior, and the open related PR is only a partial datetime-suffix solution unless it also moves backups and updates recovery, audit, docs, legacy handling, and security hardening.
