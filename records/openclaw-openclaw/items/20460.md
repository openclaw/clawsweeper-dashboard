## Summary

Keep open: current main still writes adjacent `openclaw.json.bak` plus numbered `.bak.N` backups, and the requested dedicated `config-backup/` directory with datetime names has not landed. The direct implementation at https://github.com/openclaw/openclaw/pull/20877 was closed unmerged, while active related work only addresses timestamp suffixes beside the config file.

## What This Changes

Store configuration backup files in a dedicated `config-backup/` directory with datetime-based names instead of adjacent numbered `.bak` files.

## Best Possible Solution

Adopt one unified backup-layout design that moves managed config backups into a dedicated directory, uses collision-safe datetime names, safely handles legacy adjacent backups, preserves bounded retention and owner-only permissions, and updates recovery, audit, docs, changelog, and focused tests together.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main still creates adjacent `.bak` and `.bak.N` files through `maintainConfigBackups`.

## Solution Assessment

No current implementation solves the whole issue. The open timestamp pull requests are plausible partial work, but the best maintainable path needs a maintainer-chosen directory, migration, recovery, and security contract before merge.
