## Summary

Keep open: current main and the latest release still write config backups beside the active config as `.bak` plus numbered `.bak.N` files. The open timestamp-only tracker and PR only partially overlap, while the direct directory-layout PR is closed unmerged.

## What This Changes

The issue requests moving OpenClaw configuration backup files into a dedicated backup directory and switching from numbered backup names to datetime-based names.

## Best Possible Solution

Define one canonical config-backup layout, then implement directory placement and collision-safe datetime naming together with bounded retention, owner-only permissions, legacy handling, recovery selection, audit output, doctor docs, and focused upgrade tests.

## Reproduction Assessment

Yes for the current behavior claim: source and focused tests on current main show `.bak` plus numbered `.bak.1` through `.bak.4` backups. This is a feature request rather than a broken existing behavior report.

## Solution Assessment

Unclear until maintainers choose the canonical backup layout and legacy handling. A timestamp-only suffix change is not the full solution because this issue also asks to move backups into their own directory and current recovery/audit paths assume an adjacent primary backup.
