## Summary

Keep open: current main and the latest release still use adjacent `.bak` plus numbered `.bak.N` config backups, while this request is for a dedicated backup directory with datetime names. The direct directory PR was closed unmerged, and the active related PRs only cover timestamp suffixes beside the config file.

## What This Changes

Request to move OpenClaw configuration backup files into a dedicated backup directory and use datetime-based filenames instead of numbered adjacent `.bak.N` files.

## Best Possible Solution

Adopt one coherent config-backup layout that moves managed backups into a dedicated directory with collision-safe datetime names, bounded retention, owner-only permissions, safe legacy handling, and matching recovery, audit, doctor docs, tests, and changelog coverage.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main and `v2026.5.12` still create adjacent `.bak` and `.bak.N` files through `maintainConfigBackups`.

## Solution Assessment

No current implementation solves the whole issue. The maintainable path is a maintainer-chosen backup-layout contract that updates creation, retention, permissions, legacy migration, recovery, audit, docs, and focused tests together.
