## What This Changes

Request to move OpenClaw config backup files from adjacent numbered `.bak` files into a dedicated `config-backup` directory using datetime-stamped names.

## Best Possible Solution

Adopt one coherent config-backup layout with a dedicated directory, collision-safe datetime names, bounded retention, owner-only permissions, safe legacy handling, and matching recovery, audit, doctor docs, tests, and changelog coverage.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection of current `main` and `v2026.5.12` gives high confidence that config writes still create adjacent `.bak` and numbered `.bak.N` files through `maintainConfigBackups`.

## Solution Assessment

No current implementation solves the full issue. The maintainable path is a maintainer-chosen backup-layout contract that updates creation, retention, permissions, legacy handling, recovery, audit, docs, and focused tests together.
