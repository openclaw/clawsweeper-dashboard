## Summary

Keep open: current main still writes adjacent numbered backups, the direct directory-and-datetime implementation was closed unmerged, and the active related PRs only cover timestamp suffixes beside the config file. This remains a valid feature/design item because moving credential-bearing backups changes recovery, retention, permissions, and legacy migration behavior.

## What This Changes

Move OpenClaw config backup files into a dedicated `config-backup/` directory and name them with datetimes instead of numeric rotation slots.

## Best Possible Solution

Adopt one unified backup-layout design that moves managed config backups into a dedicated directory with collision-safe datetime names, preserves or migrates legacy adjacent backups safely, and updates recovery, audit, docs, changelog, and focused tests together.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection on current main gives high confidence that config writes still create adjacent `.bak` and `.bak.N` files through `maintainConfigBackups`.

## Solution Assessment

No current implementation solves the whole issue. The timestamp-only PRs are plausible partial work, but the maintainable solution needs a chosen directory, migration, recovery, retention, and permission contract before merge.
