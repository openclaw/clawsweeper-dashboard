## Summary

Keep open: current main and v2026.6.5 still use adjacent `.bak` plus numbered `.bak.N` config backups, while the direct dedicated-directory implementation was closed unmerged and the remaining open related work only covers timestamp suffixes beside the config file.

## What This Changes

This issue requests moving OpenClaw config backup files into a dedicated backup directory and naming them with datetimes instead of numeric rotation suffixes.

## Best Possible Solution

Adopt one coherent config-backup layout that uses a dedicated directory, collision-safe datetime names, bounded retention, owner-only permissions, safe legacy handling, and matching recovery, audit, docs, and focused tests.

## Risks / Open Questions

- Moving backups without an explicit legacy handling plan could make existing adjacent `.bak` and `.bak.N` rollback files harder to find or accidentally prune useful recovery points.
- Config backups can contain credentials, so the new directory and timestamped files need owner-only permissions, bounded retention, and cleanup semantics at least as strong as current main.
- Recovery, audit output, startup hints, and doctor documentation currently assume `${configPath}.bak`; changing only the rotation helper would leave diagnostics and restore behavior inconsistent.
