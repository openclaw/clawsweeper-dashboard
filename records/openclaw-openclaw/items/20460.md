## What This Changes

The issue requests moving OpenClaw config backup files into a dedicated `config-backup/` directory with collision-safe datetime-based names instead of adjacent numbered `.bak` files.

## Best Possible Solution

Define one canonical config-backup layout, then implement directory placement and datetime naming together with collision handling, bounded retention, owner-only permissions, legacy adjacent-backup handling, recovery/audit updates, doctor docs, and focused tests.

## Reproduction Assessment

## Solution Assessment

Unclear until maintainers decide the backup-layout policy. A dedicated directory with datetime names is plausible, but the best fix must include migration, recovery, audit, permissions, and retention rather than only changing file names.
