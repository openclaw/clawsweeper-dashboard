## Summary

Keep open. Current main still creates adjacent `.bak` / `.bak.N` config backups, the direct directory-and-datetime PR was closed unmerged, and the open related PR only covers timestamp suffixes rather than the requested dedicated backup directory.

## What This Changes

Request stores OpenClaw config backups under a dedicated `.openclaw/config-backup/` directory using datetime-stamped filenames instead of adjacent numbered `.bak` files.

## Best Possible Solution

Implement one shared config-backup path and retention helper used by write, rotation, recovery, audit, and docs, with `config-backup/` datetime names, collision handling, bounded retention, owner-only permissions, and safe legacy handling.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence in the current behavior: config writes call `maintainConfigBackups`, which writes adjacent `.bak` files that tests and docs still expect.

## Solution Assessment

No. Current main does not solve the requested dedicated-directory behavior, and a timestamp-only rename would be incomplete unless recovery, audit, docs, retention, legacy cleanup, and permission hardening move to the same helper.
