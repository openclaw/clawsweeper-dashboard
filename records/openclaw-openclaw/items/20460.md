## Risks / Open Questions

- Moving backups without a legacy plan could make existing adjacent `.bak` and `.bak.N` rollback files harder to find or accidentally prune useful recovery points.
- Config backups can contain credentials, so any directory and timestamp migration must preserve owner-only permissions, bounded retention, and stale sensitive-copy cleanup.
- Recovery, audit output, startup hints, and doctor documentation currently assume `${configPath}.bak`; changing only rotation would leave restore behavior and diagnostics inconsistent.
- Open related timestamp-only work at https://github.com/openclaw/openclaw/pull/40392 needs coordination so backup-format changes do not conflict.
