## Summary

Keep open. Current main has partial usage surfaces, but no Telegram/chat command returns the requested per-session or per-sub-agent token and API/model-call breakdown, and the exact `/usage` and `/status` command names are reserved core surfaces that need maintainer product judgment.

## What This Changes

Request a Telegram-visible `/usage` command or status panel showing per-session/sub-agent token consumption and API call counts.

## Best Possible Solution

Add a compact, authorized chat usage summary backed by `sessions.usage`, with documented call-count semantics and the Control UI remaining the full-detail surface.

## Reproduction Assessment

Not applicable; this is a feature request. Current source inspection gives a high-confidence state check: `/usage`, `/status`, and `/subagents list` do not expose the requested per-session/sub-agent token and API/model-call breakdown.

## Solution Assessment

Unclear until maintainers choose the command shape, visibility boundary, and call-count semantics. Reusing `sessions.usage` from a core chat command is likely narrower than building Telegram-only tracking.
