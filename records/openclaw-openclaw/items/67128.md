## Summary

Keep open. Current main and the latest shipped command surface still provide partial usage/status visibility, but no Telegram/chat command returns the requested per-session or per-sub-agent token and API/model-call breakdown, and the remaining design depends on command shape and usage-visibility semantics.

## What This Changes

The issue asks for a Telegram `/usage` command or session stats panel that shows per-session/sub-agent API call counts and token consumption.

## Best Possible Solution

Add a compact, authorized chat usage summary backed by `sessions.usage`, while keeping the Control UI as the full-detail surface and documenting call-count semantics.

## Reproduction Assessment

Not applicable; this is a feature request. Source and docs inspection gives a high-confidence current-state check that `/usage`, `/status`, and `/subagents list` do not expose the requested per-session/sub-agent token and API/model-call breakdown.

## Solution Assessment

Unclear until maintainers choose the command shape, visibility boundary, and call-count semantics. Reusing `sessions.usage` from a core chat command is likely narrower than building Telegram-only tracking.
