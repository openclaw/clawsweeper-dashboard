## Summary

Keep open. Current main still has Telegram-compatible `/status`, `/usage cost`, `/subagents list`, and a richer Gateway/Control UI usage backend, but no chat command returns the requested per-session or per-sub-agent usage breakdown with token consumption and API/model-call counts.

## What This Changes

Request a Telegram-accessible `/usage` or `/status` surface that breaks down token and API-call usage by session or sub-agent.

## Best Possible Solution

Expose a compact, chat-safe usage summary over existing `sessions.usage` data, with clear call-count semantics and the Control UI remaining the full-detail surface.

## Reproduction Assessment

Not applicable because this is a feature request. Source inspection gives a high-confidence current-state check: `/usage` is footer/cost-only, `/status` only summarizes subagent counts, and richer per-agent/tool-call usage is Gateway/Control UI-only.

## Solution Assessment

Unclear until maintainers choose the command shape, visibility boundary, and call-count semantics. Reusing `sessions.usage` from `/usage` or `/status` is likely narrower than building a Telegram-only usage subsystem.
