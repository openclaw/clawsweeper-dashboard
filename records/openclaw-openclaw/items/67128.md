## Summary

Keep open. Current main still exposes only partial chat usage visibility, and the related `/total_usage` PR is open, unmerged, and does not fully settle the requested per-session/sub-agent token and API-call breakdown for Telegram.

## What This Changes

Add a Telegram chat command or status panel that shows per-session/sub-agent token consumption and API call counts.

## Best Possible Solution

Add a compact authorized chat usage summary backed by `sessions.usage`, document the chosen `/usage`, `/status`, or `/total_usage` semantics, and keep the dashboard as the full-detail surface.

## Reproduction Assessment

Not applicable; this is a feature request. Source and docs inspection gives a high-confidence current-state check that Telegram/chat commands do not expose the requested per-session or per-sub-agent token and API/model-call breakdown.

## Solution Assessment

Unclear until maintainers choose the command shape, visibility boundary, and call-count definition. Reusing `sessions.usage` from a core chat command looks narrower than building Telegram-only tracking.
