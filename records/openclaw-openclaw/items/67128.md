## Summary

Keep open: current main and the latest release still provide `/usage` as a footer/current-session/today/30-day cost summary and `/status` as runtime/provider quota plus subagent liveness, but they do not expose the requested Telegram-facing per-session or per-sub-agent token/API-call breakdown. The related https://github.com/openclaw/openclaw/pull/40828 was closed unmerged and is only a partial overlap, so this issue remains the canonical product-direction item.

## What This Changes

The issue requests a Telegram `/usage` command or stats panel that reports per-session and per-sub-agent token consumption and API-call counts.

## Best Possible Solution

Reuse the existing Gateway session usage aggregation as the data source, then add a narrow authorized Telegram command or `/status` extension after maintainers choose naming, privacy scope, and API-call semantics.

## Reproduction Assessment

Not applicable: this is a feature request rather than a reproducible bug. Source, docs, related-PR, and release checks show the requested Telegram command or stats panel is absent today.

## Solution Assessment

No: the requested direction is plausible, but adding it directly is not yet the best fix until maintainers define command shape, privacy scope, and API-call semantics. The narrow maintainable path is to present existing Gateway session usage data instead of creating a separate ad hoc data source.
