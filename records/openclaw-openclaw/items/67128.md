## Summary

Keep open. Current main still exposes Telegram-compatible `/status`, `/usage cost`, and `/subagents list`, plus a richer Gateway/Control UI usage backend, but there is still no chat command that returns the requested per-session or per-sub-agent usage breakdown with call counts.

## What This Changes

Request a Telegram-visible `/usage` command or status panel that breaks usage down by session/sub-agent, API/model call counts, and token consumption.

## Best Possible Solution

Add a compact, chat-safe Telegram usage summary that reuses `sessions.usage`, defines the call-count semantics, and keeps the Control UI as the full-detail surface.

## Reproduction Assessment

Not applicable as a feature request. Current-state inspection gives a high-confidence check: `/usage cost` only returns session/today/last-30-day totals, while `/subagents list` is partial current-session visibility.

## Solution Assessment

Unclear until maintainers choose the command shape and visibility boundary. Extending the existing `/usage` or related usage-command path is likely narrower than building a Telegram-only stats subsystem.
