## Summary

Keep open. Current main has Telegram `/status`, `/usage cost`, and `/subagents list` coverage, plus a richer Control UI usage backend, but the Telegram chat command still does not provide the requested per-agent or per-session usage breakdown with call counts.

## What This Changes

Request a Telegram-visible `/usage` command or status panel that breaks usage down by session/sub-agent, API/model call counts, and token consumption.

## Best Possible Solution

Add a compact, chat-safe Telegram usage summary that reuses the existing session usage aggregation backend, with clear command syntax and conservative authorization, while keeping the Control UI as the full-detail surface.

## Reproduction Assessment

Not applicable as a feature request. Code and docs inspection give a high-confidence current-state check: `/usage cost` only returns current session/today/last-30-day totals, and `/subagents list` is only a partial current-session token view.

## Solution Assessment

Unclear until maintainers choose the command shape and visibility boundary. The narrowest maintainable path appears to be extending the existing `/usage` handler rather than building a Telegram-only stats subsystem.
