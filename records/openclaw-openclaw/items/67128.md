## Summary

Keep open. Current main and the latest release still provide only partial usage visibility in chat: `/usage` toggles footers or shows a session/today/30-day cost summary, `/status` reports subagent counts, and the richer per-agent aggregation remains in the gateway/UI rather than a Telegram command.

## What This Changes

Request a Telegram-accessible `/usage` or `/status` surface that shows per-session or per-sub-agent token consumption and API/model-call counts.

## Best Possible Solution

Add a compact, authorized chat usage summary backed by `sessions.usage`, keep the Control UI as the full-detail surface, and document the call-count semantics.

## Reproduction Assessment

Not applicable; this is a feature request. Source and docs inspection gives a high-confidence current-state check that Telegram/chat commands do not expose the requested per-session or per-sub-agent token and API/model-call breakdown.

## Solution Assessment

Unclear until maintainers choose the command shape, visibility boundary, and call-count definition. Reusing `sessions.usage` from a core chat command is likely narrower than building Telegram-only tracking.
