## What This Changes

The issue requests a Telegram-facing `/usage` command or stats panel showing per-session or per-sub-agent token consumption and API-call counts.

## Best Possible Solution

Use the existing Gateway/session usage aggregation as the data source, then add a narrow authorized Telegram command or `/status` extension once maintainers choose naming, privacy scope, and API-call semantics.
