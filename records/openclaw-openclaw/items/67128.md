## Summary

Keep open. Current main has Telegram-compatible `/status`, `/usage cost`, `/subagents list`, and a richer Gateway/Control UI usage backend, but no chat command returns the requested per-session or per-sub-agent breakdown with token consumption and API/model-call counts.

## What This Changes

Add Telegram chat visibility for per-session or per-sub-agent usage, including token consumption and API call counts.

## Best Possible Solution

Add a compact, chat-safe usage summary that reuses `sessions.usage`, defines the call-count semantics, and keeps the Control UI as the full-detail surface.

## Reproduction Assessment

Not applicable because this is a feature request. Source inspection gives a high-confidence current-state check: `/usage` only supports footer/cost modes, `/status` only shows provider/current-session status plus sub-agent counts, and richer agent/tool usage is currently web/Gateway-only.

## Solution Assessment

Unclear until maintainers choose the command shape and visibility boundary. Extending the existing `/usage` or `/status` path over `sessions.usage` is likely narrower than building a Telegram-only stats subsystem.
