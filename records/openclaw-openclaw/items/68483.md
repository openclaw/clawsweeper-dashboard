## Summary

Keep open. Current main and the latest release still lack an agent-visible redaction marker, and this PR remains the relevant implementation path, but the PR head still has a post-truncation false-positive notice defect, missing real behavior proof, and a dirty merge state.

## What This Changes

The branch adds `[OPENCLAW-REDACTED]` constants and a `redactSensitiveTextForAgent` helper, emits `sessions_history.notice` when returned content is redacted, adds regression coverage, and adds a changelog entry.

## Best Possible Solution

Land a corrected version that gates the notice on final visible redaction or updates the wording contract, documents the marker, resolves conflicts, and includes real `sessions_history` proof.

## Reproduction Assessment

Yes, source-reproducible. Current main masks sk-style tokens in `sessions_history` and sets `contentRedacted` without a marker, and the PR defect follows from `truncateHistoryText` setting `redacted` before the returned text is cut.

## Solution Assessment

No, not as-is. The additive marker and notice are the right narrow direction, but the notice must reflect final visible redaction or have a different documented contract before merge.
