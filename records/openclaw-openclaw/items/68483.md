## Summary

## Best Possible Solution

Gate the marker or notice on redaction visible in the final returned agent-visible payload, or revise the wording, document the contract, and merge only after real `sessions_history` proof is posted.

## Reproduction Assessment

Yes, source-level. A `sessions_history` message with more than 4,000 safe characters before an sk-style token is redacted before truncation, then truncated before the placeholder, while `contentRedacted` still drives the new notice.

## Solution Assessment

No, not as-is. The additive marker direction is appropriate, but the notice must reflect the final returned payload or use safer wording, and the new stable contract needs docs plus real behavior proof.
