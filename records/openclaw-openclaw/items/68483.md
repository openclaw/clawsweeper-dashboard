## Summary

Keep open. The PR is the active implementation path for the linked redaction-marker issue, but the latest head still has a source-reproducible false-positive notice case after per-text truncation, lacks public docs for the new contract, and has only test/lint evidence rather than real behavior proof.

## What This Changes

Adds redaction marker constants, a reusable agent-facing redaction helper, a conditional `sessions_history.notice` field, tests, and a changelog entry.

## Best Possible Solution

Gate the notice on redaction visible in the final returned payload, document the public marker/notice contract, and require redacted real-behavior proof before merge.

## Reproduction Assessment

Yes. On PR head, a retained `sessions_history` text block with more than 4,000 safe characters followed by an sk-style token records redaction before the text cutoff, so the returned text can lack a masked placeholder while `notice` is still emitted.

## Solution Assessment

No, not as-is. The additive marker is a narrow maintainable hardening direction, but the notice condition must reflect the final returned payload and the stable response contract should be documented.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Gate the notice on visible redaction:** `src/agents/tools/sessions-history-tool.ts:291-292`
  - body: `contentRedacted` is still derived from `entry.redacted`, which is computed before `truncateHistoryText` applies the 4,000-character cut. If the only secret is beyond that cutoff, the returned messages contain no masked placeholder while this branch still emits a notice saying redaction happened in this output; gate the notice on the final visible text or adjust the contract.
  - confidence: 0.91

  - body: The new constants and `sessions_history.notice` field create a stable downstream contract, but the public session-tool docs still only mention `contentRedacted`. Add the `notice` field and `[OPENCLAW-REDACTED]` guidance to the public docs so agents and tooling have a durable contract to follow.
  - confidence: 0.8
