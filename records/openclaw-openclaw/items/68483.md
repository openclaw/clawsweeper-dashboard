## Summary

Keep this PR open because current main still lacks the requested marker, but the branch is not ready to merge: the notice can still be emitted for a retained message whose secret was removed by per-text truncation, the new stable contract is undocumented, and the contributor has not provided real after-fix behavior proof.

## What This Changes

The PR adds redaction marker constants and a helper, conditionally adds `sessions_history.notice`, expands redaction/session-history tests, and adds a changelog entry.

## Best Possible Solution

Gate the notice on redaction visible in the final returned payload, document the `sessions_history.notice` marker contract, and add redacted real-behavior proof before merge.

## Reproduction Assessment

Yes, source-reproducible: return a retained `sessions_history` text block with more than 4,000 safe characters followed by an sk-style token. PR head records redaction before the 4,000-character cut, so it can emit `notice` even though the returned text has no redacted placeholder.

## Solution Assessment

No, not as-is. The additive marker is a narrow maintainable hardening direction, but the notice must describe the final returned payload and the stable response contract needs docs.

## Review Findings

Overall correctness: patch is incorrect

Full review comments:

- **[P2] Gate the notice on visible redaction:** `src/agents/tools/sessions-history-tool.ts:291-292`
  - body: `contentRedacted` is now limited to messages that survive byte/hard caps, but it still uses `entry.redacted`, which is computed before `truncateHistoryText` applies the 4,000-character cut. A retained message with its only secret after that cutoff returns no masked placeholder while this branch still emits the machine-readable marker, so downstream agents can receive a false-positive redaction signal. Recompute the notice condition from the final returned text or change the contract wording.
  - confidence: 0.91
- **[P3] Document the redaction notice contract:** `src/logging/redact.ts:208-214`
  - body: The new constants define a stable downstream contract and the PR adds `sessions_history.notice`, but the public session-tool docs still only list `contentRedacted` among summary flags. Document the `notice` field and `[OPENCLAW-REDACTED]` guidance so agents and tooling have a durable contract to follow.
  - confidence: 0.8
