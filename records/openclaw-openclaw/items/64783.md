## Summary

Keep open. Current main still routes Feishu group-message handling through static imports from the heavy `bot.ts` and `card-action.ts` modules, while the parser remains in `bot.ts`; the open PR #64820 is already the explicit closing implementation candidate for this crash, so this issue should remain open until that PR lands or is replaced.

## What This Changes

The issue reports a Feishu/Lark group mention crash where multi-bot group routing throws `ReferenceError: Cannot access 'utils_1' before initialization` and prevents the mentioned bot from responding.

## Best Possible Solution

Land a narrow Feishu plugin change that extracts message parsing into a lightweight module, lazy-loads heavy message and card handlers from monitor paths, keeps compatibility exports stable, and adds focused parser/module-isolation coverage plus a practical group-mention validation path.
