## Summary

Keep open. Current main still leaves the embedded Pi SDK retry path enabled, the generic tool-loop guard is opt-in, and #74434 plus #74503 are open unmerged implementation candidates for this issue.

## What This Changes

The issue reports runtime-level replay of previous tool calls after tool failure, causing repeated exec/tool loops across Feishu and Telegram sessions.

## Best Possible Solution

Land one canonical agents-runtime fix that disables embedded Pi SDK auto-retry for OpenClaw-managed runs, preserves failed tool context for the next model turn, and adds focused failed-tool replay regression coverage while leaving broader transcript replay analysis under #69208.

## Reproduction Assessment

Unclear. The report and the Telegram follow-up give high-signal replay patterns, and source/dependency inspection shows a plausible retry path, but this read-only review did not run a deterministic current-main fixture or live Feishu/Qwen/Telegram reproduction.
