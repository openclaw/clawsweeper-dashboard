## Summary

Keep open. Current main still leaves the embedded Pi SDK auto-retry path enabled, generic tool-loop detection remains opt-in, and open closing PRs #74434 and #74503 are already tracking candidate fixes.

## What This Changes

The issue reports that failed or empty tool-call results can cause embedded agent sessions to replay prior tool calls indefinitely after the 2026.4.25 release.

## Best Possible Solution

Land or consolidate one canonical agents-runtime fix that disables Pi SDK auto-retry for OpenClaw-managed sessions, preserves failed-tool context, and adds focused replay-loop regression coverage while keeping broader transcript/session replay work under #69208.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? No. Source inspection shows a plausible current-main retry/recontinue path and the issue plus Telegram comment give concrete patterns, but this review did not run the live Feishu/Qwen or Telegram route and found no current regression fixture for the exact loop.

## Solution Assessment

Is this the best way to solve the issue? Unclear. Disabling Pi SDK auto-retry appears to be the narrow first fix, while the broader thinking-repetition detector in #74503 needs maintainer judgment because it changes model-context steering beyond the described failed-tool replay loop.
