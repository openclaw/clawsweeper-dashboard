## Summary

Keep open. Current main still leaves the embedded Pi SDK retry path enabled, generic tool-loop detection remains opt-in, and the issue is actively paired with two open closing PRs plus a newer Telegram replay observation.

## What This Changes

The issue asks OpenClaw to stop runtime-level replay of a prior failed tool call so the agent can reason or choose a different next action.

## Best Possible Solution

Land or consolidate one canonical agents-runtime fix that disables embedded Pi SDK auto-retry for OpenClaw-managed sessions, preserves failed tool context, and adds focused failed-tool replay coverage while tracking broader transcript replay under #69208.

## Reproduction Assessment

Unclear. The reports provide concrete replay patterns and the pinned SDK gives a strong source-level reproduction path, but this review did not run the live Feishu/Qwen or Telegram route and did not find a current-main regression fixture for the exact sequence.

## Solution Assessment

Unclear. Disabling Pi SDK auto-retry appears to be the narrowest first fix, while #74503's broader thinking-repetition detector needs maintainer judgment because it changes model-context steering beyond the described failed-tool replay.
