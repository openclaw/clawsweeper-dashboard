## Summary

Keep open. Current main still leaves the embedded Pi SDK retry path enabled, the generic tool-loop guard is opt-in, and two open PRs already reference this issue as active implementation candidates.

## What This Changes

The issue reports that failed tool calls in OpenClaw 2026.4.25 can be replayed by runtime retry logic, causing repeated identical exec calls and infinite loops.

## Best Possible Solution

Land one canonical agents-runtime fix that disables embedded pi-coding-agent auto-retry for OpenClaw-managed runs, preserves failed tool context for the next turn, and adds a focused failed-exec replay regression test.

## Reproduction Assessment
