## Summary

Keep open. Current main still leaves the embedded Pi SDK auto-retry path enabled, generic tool-loop detection remains opt-in, and open PRs #74434 and #74503 already track candidate fixes for this issue.

## What This Changes

The issue reports runtime-level replay of prior agent tool calls after failed or stale runs, causing repeated exec/process calls in Feishu/Qwen and a similar Telegram restart replay observation.

## Best Possible Solution

Land or consolidate one canonical agents-runtime fix that disables Pi SDK auto-retry for OpenClaw-managed sessions, preserves failed-tool context, and adds focused replay-loop regression coverage while keeping broader transcript/session replay work under #69208.

## Reproduction Assessment
