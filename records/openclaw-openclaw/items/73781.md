## Summary

Keep open. Current main still preserves the embedded Pi SDK retry setting and leaves generic tool-loop detection opt-in, while two open unmerged PRs already reference this issue with closing syntax and a newer Telegram observation keeps the replay symptom active.

## What This Changes

The issue reports a runtime-level replay loop where failed or stale prior tool calls are re-emitted instead of letting the agent reason and choose a new action.

## Best Possible Solution

Land or consolidate one canonical agents-runtime fix that disables embedded Pi SDK auto-retry for OpenClaw-managed sessions, preserves failed tool context, adds a focused failed-tool replay regression test, and leaves broader transcript replay tracking under #69208.

## Reproduction Assessment

Unclear. The issue and Telegram follow-up give concrete replay patterns and current source leaves a plausible retry path intact, but this review did not run the live Feishu/Qwen or Telegram route and did not find a current-main regression fixture for the exact sequence.
