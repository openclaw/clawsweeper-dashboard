## Summary

Keep open. Current main still leaves the embedded Pi SDK retry path enabled, the generic tool-loop guard remains opt-in, and #74434 plus #74503 are open implementation candidates that explicitly close this issue.

## What This Changes

The issue reports that OpenClaw 2026.4.25 can replay a failed tool call at runtime, causing repeated identical exec calls and infinite loops.

## Best Possible Solution

Land one canonical agents-runtime fix that disables embedded pi-coding-agent auto-retry for OpenClaw-managed runs, preserves failed tool context for the next turn, and adds a focused failed-exec replay regression test.
