## Summary

Keep open. Current main still leaves the embedded Pi SDK retry path enabled and the generic tool-loop guard disabled by default, while open closing PRs #74434 and #74503 already target this report and should be reviewed or consolidated before closing.

## What This Changes

The issue reports a regression where failed or stale agent tool calls are replayed by the runtime in Feishu/Qwen and later Telegram sessions instead of letting the model take a new action.

## Best Possible Solution

Land or consolidate one canonical agents-runtime fix that disables embedded Pi SDK auto-retry for OpenClaw-managed sessions, preserves failed-tool context, and adds focused failed-tool replay regression coverage while leaving broader transcript/session replay under #69208.

## Reproduction Assessment

Unclear. The issue and Telegram follow-up provide concrete replay patterns and the pinned SDK shows a plausible retry path, but this review did not run the live channel/model route and did not find an exact current-main regression fixture.

## Solution Assessment

Unclear. Disabling the embedded Pi SDK auto-retry appears to be the narrowest first fix, while the broader thinking-repetition detector in #74503 needs maintainer judgment because it changes model-context steering beyond the reported failure replay.
