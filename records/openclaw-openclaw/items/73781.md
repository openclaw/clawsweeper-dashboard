## Summary

Keep open. Current main still leaves the embedded PI settings manager without a retry override, the pinned pi-coding-agent dependency defaults SDK auto-retry on, and the generic tool-loop detector remains opt-in; two open PRs still reference this issue with closing syntax, so the issue should stay open until one canonical fix lands or both candidates are rejected.

## What This Changes

Prevent the agents runtime from replaying a previous failed tool call and causing repeated or infinite tool-call loops.

## Best Possible Solution

Land one canonical agents-runtime fix that disables the inner pi-coding-agent auto-retry path for embedded OpenClaw runs without persisting user settings, preserves failed tool-result/error context for the next model turn, and adds focused regression coverage for failed `exec` followed by a changed or no-op assistant action; handle broader reasoning-loop hints separately if maintainers want that behavior.
