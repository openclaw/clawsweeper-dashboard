## Summary

Keep open. Current main still leaves the embedded Pi SDK retry path enabled, the generic repeated-tool guard is opt-in, there is a fresh 2026.4.29 replay observation, and two open unmerged PRs already reference this issue as active fix candidates.

## What This Changes

The issue reports runtime-level replay of previous failed tool calls, especially repeated `exec` calls, causing loops and transcript/context growth after upgrading to OpenClaw 2026.4.25.

## Best Possible Solution

Land one canonical agents-runtime fix that disables embedded pi-coding-agent auto-retry for OpenClaw-managed runs, preserves failed tool context for the next model turn, and adds focused regression coverage for failed tool-call replay.
