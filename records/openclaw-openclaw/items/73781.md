## Summary

Keep open. Current main still leaves the embedded Pi SDK retry behavior enabled, the generic loop guard remains opt-in, and two open pull requests already reference this issue with closing syntax as active implementation candidates.

## What This Changes

The issue reports a regression where the agents runtime repeats a previous failed tool call instead of letting the model reason and choose a new action.

## Best Possible Solution

Ship one canonical agents-runtime fix that disables the embedded pi-coding-agent auto-retry path for OpenClaw-managed runs, preserves failed tool/error context for the next model turn, and adds focused regression coverage for a failed exec followed by a changed or no-op assistant action.

## Reproduction Assessment

Unclear. The report gives a concrete repeated failed-exec pattern and dependency inspection shows a plausible replay path, but this review did not run the live Feishu/Qwen route and current main does not contain a regression fixture for that sequence.

## Solution Assessment

Unclear. Disabling the embedded SDK retry appears to be the narrowest first fix, but the broader thinking-repetition detector proposed by one linked PR needs maintainer judgment before becoming part of the canonical solution.
