## Summary

Keep open. Current main still leaves the embedded Pi SDK retry path enabled, the generic repeat/ping-pong loop guard is opt-in, and two unmerged PRs already reference this issue as active implementation candidates.

## What This Changes

The issue reports runtime-level replay of prior tool calls after failed or stale tool activity, observed on Feishu/Qwen after 2026.4.25 and later on Telegram in 2026.4.29.

## Best Possible Solution

Review and land one canonical agents-runtime fix that disables embedded Pi SDK auto-retry for OpenClaw-managed runs, preserves failed-tool context for the next model turn, and adds a focused failed-tool replay regression test.

## Reproduction Assessment

Unclear. The issue provides a concrete repeated failed-exec pattern plus a fresh Telegram observation on 2026.4.29, and the current dependency path plausibly explains replay, but this review did not run the live Feishu/Qwen or Telegram route.

## Solution Assessment

Unclear. Disabling embedded Pi SDK auto-retry appears to be the narrowest first fix for the failed-tool replay path, while broader thinking-repetition or transcript-replay handling should be decided through #74434, #74503, and the #69208 umbrella.
