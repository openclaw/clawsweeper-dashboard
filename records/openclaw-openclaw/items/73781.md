## Summary

Keep open. Current main still leaves the embedded pi-coding-agent retry path enabled in the prepared runtime settings, the generic loop guard remains disabled by default, and two open PRs still reference this issue with closing syntax as active fix candidates.

## What This Changes

The issue asks OpenClaw to stop replaying a previous failed tool call at runtime, especially for Feishu/Qwen workflows using repeated `exec` calls.

## Best Possible Solution

Ship one canonical agents-runtime fix that disables the embedded pi-coding-agent auto-retry path for OpenClaw-managed runs, preserves failed tool/error context for the next model turn, and adds focused regression coverage for failed `exec` followed by a changed or no-op assistant action.

## Reproduction Assessment

Unclear. The report gives a concrete repeated failed-`exec` pattern and the pinned SDK source shows a plausible replay mechanism, but this review did not run the live Feishu/Qwen route and current main lacks a deterministic regression fixture for that sequence.

## Solution Assessment

Unclear. Disabling the embedded SDK retry appears to be the narrow first fix, while the broader thinking-repetition detector proposed by one linked PR needs maintainer judgment before becoming part of the canonical solution.
