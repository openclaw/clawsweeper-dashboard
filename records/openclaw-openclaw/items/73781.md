## Summary

Keep open. Current main still preserves the embedded Pi retry setting in prepared runtime settings, generic tool-loop detection remains disabled by default, and two open PRs already reference this issue with closing fixes that need maintainer review or consolidation.

## What This Changes

The issue reports a runtime-level replay loop where failed or stale prior tool calls are re-emitted in Feishu/Qwen and later Telegram sessions instead of letting the agent choose a new action.

## Best Possible Solution

Review and land or consolidate one canonical agents-runtime fix that disables embedded Pi retry for OpenClaw-managed turns and adds a focused failed-tool replay regression test, while keeping broader cross-channel replay cleanup tracked under #69208.

## Reproduction Assessment

Unclear. The issue and later Telegram comment provide concrete observed replay sequences, and current source shows plausible retry and default-off guard paths, but this read-only review did not run Feishu/Qwen or Telegram live and found no current-main regression fixture for the exact loop.

## Solution Assessment

Yes for keeping this issue open; the repair path is still unsettled. Disabling the Pi SDK retry path appears to be the narrowest first fix, while the broader thinking/session replay behavior should be reviewed separately or folded into #69208.
