## Summary

Keep open. Current main still does not disable the embedded pi-coding-agent auto-retry path, the built-in tool-loop detector is opt-in, and two open PRs already use closing syntax for this issue as active implementation candidates.

## What This Changes

The issue reports a regression where failed or stale tool calls are replayed by runtime/session recovery instead of letting the agent choose the next action.

## Best Possible Solution

Land one canonical agents-runtime fix that disables embedded Pi SDK auto-retry for OpenClaw-managed runs, preserves failed tool context for the next turn, and adds focused regression coverage for failed/stale tool replay.

## Reproduction Assessment

Unclear. The issue and later Telegram comment give concrete replay-like sequences, and dependency inspection shows a plausible retry path, but this review did not run the live Feishu/Qwen or Telegram route or find a current-main fixture for the exact loop.

## Solution Assessment

Unclear. Disabling the embedded SDK retry appears to be the narrowest first fix, while the broader thinking-repetition detector in #74503 needs maintainer judgment before it becomes the canonical solution.
