## Summary

Keep open. Current main still preserves the embedded Pi SDK retry setting and default tool-loop detection remains off; the issue is also paired with two open closing PRs plus a newer Telegram replay observation, so it remains active runtime work rather than cleanup.

## What This Changes

Stop runtime/session recovery from replaying a previous failed tool call as if the agent issued it again, especially in Feishu/Qwen and Telegram routes.

## Best Possible Solution

Land or consolidate one canonical agents-runtime fix that disables embedded Pi SDK auto-retry for OpenClaw-managed sessions, preserves failed-tool context, and adds a focused failed-tool replay regression test while tracking broader transcript replay under #69208.
