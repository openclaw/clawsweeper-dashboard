## Summary

Keep open. Current main still leaves the embedded Pi SDK auto-retry path enabled, generic tool-loop detection remains opt-in, and two open closing PRs plus the newer Telegram observation keep this as active runtime work.

## What This Changes

The issue asks OpenClaw to stop replaying an agent's prior failed tool call at the runtime layer and let the agent reason or choose a new action on the next turn.

## Best Possible Solution

Land one canonical agents-runtime fix that disables embedded Pi SDK auto-retry for OpenClaw-managed sessions, preserves failed tool context, and adds a focused failed-tool replay regression test while tracking broader transcript replay under #69208.

## Reproduction Assessment

Unclear. The issue and later Telegram comment provide concrete replay patterns and the pinned SDK shows a plausible retry path, but this read-only review did not run the live Feishu/Qwen or Telegram route and did not find a current-main regression fixture for this exact sequence.

## Solution Assessment

Unclear. Disabling Pi SDK auto-retry appears to be the narrowest first fix, while #74503's broader thinking-repetition detector needs maintainer judgment because it changes model-context steering beyond the described failure.
