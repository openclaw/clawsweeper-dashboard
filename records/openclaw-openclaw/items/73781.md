## Summary

Keep open. Current main still preserves the Pi SDK auto-retry path and leaves generic tool-loop detection opt-in, while open closing PRs #74434 and #74503 already track candidate fixes for the reported replay loop.

## What This Changes

The issue reports a regression where failed or stale tool calls are replayed by the runtime instead of letting the agent reason and choose the next action.

## Best Possible Solution

Land or consolidate one canonical agents-runtime fix that disables Pi SDK auto-retry for OpenClaw-managed sessions, preserves failed-tool context, and adds focused replay-loop regression coverage while keeping broader transcript replay work under #69208.

## Reproduction Assessment
