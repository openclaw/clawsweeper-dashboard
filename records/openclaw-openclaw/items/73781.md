## Summary

Keep open. Current main still leaves the embedded Pi SDK retry path enabled, the normal repeated-tool-call guard remains opt-in, and two open unmerged PRs already reference this issue with closing syntax as implementation candidates.

## What This Changes

The report asks OpenClaw to stop replaying the agent runtime's previous identical tool call after a failed tool invocation, especially in Qwen/Feishu `exec` workflows.

## Best Possible Solution

Land one canonical agents-runtime fix that disables the inner Pi SDK auto-retry for embedded OpenClaw runs, preserves failed tool-result/error context for the next model turn, and adds focused regression coverage for failed `exec` followed by a changed or no-op assistant action. Treat broader thinking-repetition steering as a separate maintainer-reviewed decision if needed.

## Reproduction Assessment

Unclear. The issue gives a concrete failed-`exec` replay pattern and the pinned SDK source corroborates a plausible replay path, but there is no deterministic current-main test or live Qwen/Feishu reproduction in this review.

## Solution Assessment

Unclear. The narrow disable-SDK-retry path in #74434 appears to be the smallest maintainable fix boundary for the reported runtime replay, while #74503's broader thinking-repetition detector changes model-steering behavior and should get maintainer judgment.
