## Summary

Keep open: current main still exposes and consumes the reported exec policy fields, and #76370 is an open linked fix candidate that should be reviewed or landed before this issue closes.

## What This Changes

Prevent model-generated `exec` calls from setting operator-only controls such as `security`, `ask`, `host`, `node`, and elevated routing.

## Best Possible Solution

Land a narrow fix like #76370 that keeps the model-facing exec schema to command/runtime inputs and resolves policy only from config, session defaults, and trusted command paths, with regression tests and a changelog entry.

## Reproduction Assessment

Yes, at source level: current main exposes the operator-only fields in `execSchema` and then consumes those same tool-call args when resolving policy. I did not run the reporter's live Hermes/vLLM sampling harness.

## Solution Assessment

Yes, the linked PR direction is the narrow maintainable fix: remove the policy fields from the model schema and ignore any model-supplied values at execution. The issue should remain open until that PR or an equivalent fix merges.
