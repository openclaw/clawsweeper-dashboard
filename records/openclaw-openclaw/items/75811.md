## Summary

Keep open: current main still exposes and consumes model-supplied `exec` policy fields, and open PR #76370 explicitly targets this issue with closing syntax, so this issue should remain open until that PR is reviewed and merged or closed.

## What This Changes

The issue requests moving `exec` policy controls such as `security`, `ask`, `host`, `elevated`, and `node` out of the model-controllable tool-call surface.

## Best Possible Solution

Review and land or replace #76370 so model-facing schemas omit operator-only exec controls and dispatch derives those controls from trusted config/session defaults with regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows the model-facing schema exposes the fields, and the exec dispatcher reads those same fields into effective host/security/ask/elevated/node behavior.

## Solution Assessment

Yes. Keeping this issue open while #76370 is reviewed is the narrowest maintainable path because the requested boundary fix belongs in the exec schema and dispatch code already touched by that PR.
