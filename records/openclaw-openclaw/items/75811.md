## Summary

Keep open. Current main still exposes and consumes model-supplied exec policy controls, and #76370 is the active open fix path rather than a reason to close before merge.

## What This Changes

The issue requests that exec policy, routing, node, approval, and elevation controls be removed from or ignored in model-supplied tool-call arguments.

## Best Possible Solution

Land #76370 or an equivalent reviewed patch that makes exec policy, routing, node selection, approval, and elevation trusted configuration only while preserving model control over command/runtime inputs.

## Reproduction Assessment

Yes by source inspection. Current main exposes the reported policy fields in execSchema, forwards the full schema to model tool parameters, and consumes params.security/params.ask in exec dispatch; I did not run the external vLLM/Hermes stack.

## Solution Assessment

Yes. The narrow maintainable fix is to split or normalize model-facing exec args from trusted operator/session policy, which matches #76370's stated direction.
