## Summary

Keep this open. Current main partially fixed the confirmed `security` self-denial path, but the broader model-facing exec policy boundary remains source-visible and an open PR already references this issue with closing syntax.

## What This Changes

The issue asks OpenClaw to stop exposing exec policy controls as model-controllable tool-call arguments.

## Best Possible Solution

Review and land or replace the linked fix PR so the model-facing exec schema contains only command/runtime inputs, with host, node, ask, security, and elevated routing controlled by config or trusted session defaults.

## Reproduction Assessment

Yes, from source for the remaining boundary: current main still exposes and consumes several operator-control fields from exec tool-call arguments. The originally confirmed `security` self-denial path is partially fixed on main by ignoring model-supplied `security`.

## Solution Assessment

Yes in direction. The linked PR is the narrowest current fix path because it hides operator-only exec controls from the model schema and moves policy decisions back to configured defaults while preserving the partial security fix.
