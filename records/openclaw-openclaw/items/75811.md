## Summary

Keep this issue open: current main still exposes and consumes exec policy controls from tool-call arguments, and open PR #76370 explicitly references this issue as its fix candidate. The issue should not be closed before that PR is reviewed and merged or rejected.

## What This Changes

The issue requests separating model-controllable exec inputs from operator-only exec policy controls so model tool calls cannot set security, approval, host routing, elevation, or node selection.

## Best Possible Solution

Land a reviewed fix that removes or normalizes operator-only exec controls at the model/schema boundary while preserving configured defaults and regression coverage; #76370 is the current candidate.

## Reproduction Assessment

Yes, from source inspection: current main exposes the reported fields in `execSchema` and consumes them from `createExecTool` arguments. I did not run the reporter's Hermes/vLLM harness, so this is source-reproducible rather than live-reproduced.

## Solution Assessment

Yes. Hiding or ignoring model-supplied operator-only fields while keeping command/runtime inputs model-facing is the narrowest maintainable fix, and it matches the direction of open PR #76370.
