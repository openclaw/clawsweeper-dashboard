## Summary

Keep open: current main still exposes and consumes model-supplied exec policy fields, and an open linked PR already targets the fix with closing syntax, so the issue should remain open until that PR lands or is rejected.

## What This Changes

The issue asks OpenClaw to hide or ignore model-supplied exec policy controls such as `security`, `ask`, `host`, `elevated`, and `node`.

## Best Possible Solution

Review and land #76370 or an equivalent core fix that removes operator-only exec policy fields from model-facing args and ignores them at dispatch.

## Reproduction Assessment

Yes, at source level: current main exposes the reported exec policy fields in the model-facing schema and uses those same fields when deriving host, security, ask, elevated mode, and node routing. I did not run the reporter's local-model harness.

## Solution Assessment

Yes: the maintainable fix is to keep exec policy in configured defaults and approval state, not in model tool-call arguments. The linked PR is the right review target, but this issue should not close before that work merges.
