## Summary

Keep open. Current main still exposes and consumes model-supplied exec policy controls, and the open linked PR is the active fix path rather than a reason to close before merge.

## What This Changes

The issue asks OpenClaw to remove operator-only exec policy fields from the model-controllable tool schema and dispatch arguments.

## Best Possible Solution

Land #76370 or an equivalent patch that splits model-visible exec command inputs from trusted operator/session policy, with regression coverage for schema exposure and dispatch normalization.

## Reproduction Assessment

Yes. Source inspection on current main shows a tool call can supply `security: "deny"` or `security: "allowlist"`, those values feed effective exec policy, and the host approval path emits the reported denial family.

## Solution Assessment

Yes. The narrow maintainable fix is to keep policy fields out of model-facing args and preserve trusted config/session defaults at the dispatch boundary; changing only errors or docs would leave model-controllable policy intact.

## Security Review

Status: needs_attention

Summary: Current main still lets model-visible exec arguments influence host exec policy, which can create denials that look operator-enforced.

Concerns:

- **[medium] Model-visible exec policy controls:** `src/agents/bash-tools.schemas.ts:27`
  - body: `security`, `ask`, `host`, `node`, and `elevated` are present in the exec schema exposed through tool parameters even though they control policy, routing, and elevation rather than command content.
  - confidence: 0.93
- **[medium] Model args feed effective exec policy:** `src/agents/bash-tools.exec.ts:1560`
  - body: `params.security` and `params.ask` are normalized into the effective security and approval modes, so model-supplied values can tighten policy and trigger the reported self-denial path.
  - confidence: 0.92
- **[medium] Derived policy can deny host exec:** `src/agents/bash-tools.exec-host-shared.ts:205`
  - body: The host approval helper derives `hostSecurity` from the supplied security value and throws the reported `exec denied: host=... security=deny` error when it resolves to `deny`.
  - confidence: 0.9
