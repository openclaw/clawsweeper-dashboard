## Summary

Keep open. Current main still exposes model-visible exec policy controls and consumes `security`/`ask` during dispatch, while #76370 is the active open fix path and should not close this issue before merge.

## What This Changes

Prevent model-supplied exec arguments from controlling operator-owned policy fields such as `security`, `elevated`, `ask`, `host`, and `node`.

## Best Possible Solution

Land #76370 or an equivalent patch that separates model command/runtime inputs from operator/session policy inputs and adds regression coverage for model-supplied policy fields.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: current main exposes the policy fields in the exec tool schema, forwards that schema to OpenAI-format tools, and consumes `security`/`ask` during dispatch.

## Solution Assessment

Yes. Removing or ignoring model-supplied policy controls at the schema/dispatch boundary is the narrow maintainable fix; docs or error-message changes would leave the boundary intact.
