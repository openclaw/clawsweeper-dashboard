## Summary

Keep this issue open: current main still exposes and honors model-supplied exec policy fields, and the linked fix PR #76370 is open but not merged.

## What This Changes

The issue asks OpenClaw to stop exposing and honoring operator-only exec controls (`security`, `ask`, `host`, `elevated`, `node`) as model-supplied exec tool arguments.

## Best Possible Solution

Review, validate, and land PR #76370 or an equivalent patch that removes or strips model-supplied policy fields while keeping exec policy sourced from trusted config and session defaults.

## Reproduction Assessment

Yes from source inspection, not from a live model harness. Current main publishes the policy fields in the exec tool schema and then reads those same fields from tool-call args, so `security: "deny"` can flow into the reported denial path.

## Solution Assessment

Yes. The narrow maintainable fix is to remove or ignore operator-only controls at the model/dispatch boundary while preserving trusted `/exec` and config defaults; the linked PR is the active implementation candidate.
