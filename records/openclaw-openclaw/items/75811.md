## Summary

Keep open. Current main still exposes and consumes model-supplied `exec` policy controls, and the linked open PR is the active closing path rather than a reason to close this issue before merge.

## What This Changes

The issue asks OpenClaw to keep `exec` operator policy controls out of the model-controllable tool-call argument surface.

## Best Possible Solution

Review and land #76370 or an equivalent patch that removes or ignores model-supplied exec policy controls while preserving trusted config/session-default policy paths with regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows the policy fields are exposed in the exec schema, forwarded into OpenAI tool parameters, and consumed during exec dispatch into the reported denial path.

## Solution Assessment

Yes. Splitting model-facing command/runtime args from operator/session policy, as #76370 proposes, is the narrow maintainable fix; docs or error-message changes alone would leave the boundary intact.
