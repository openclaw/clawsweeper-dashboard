## Summary

Keep open. Current main still exposes and consumes model-supplied exec policy controls, and the open PR that says it fixes this issue is the active path rather than a reason to close before merge.

## What This Changes

The issue reports that the model-facing `exec` tool schema includes operator-only policy fields, letting a model trigger denial by supplying values such as `security`, `ask`, `host`, `elevated`, or `node`.

## Best Possible Solution

Review and land #76370 or an equivalent patch that removes model visibility of exec policy controls, ignores model-supplied policy fields at dispatch, preserves trusted config/session defaults, and keeps regression coverage for the boundary.

## Reproduction Assessment

Yes. Source inspection on current main shows a model tool call can supply `security: "deny"`, that value is normalized into effective exec policy, and the host approval path throws the reported denial.

## Solution Assessment

Yes. Splitting model-call arguments from operator/session policy is the narrow maintainable fix; a docs-only change would leave the policy boundary exposed.
