## Summary

Keep this open: current main still exposes and consumes model-supplied exec policy fields, and the linked open PR with closing syntax should be reviewed or landed before this issue closes.

## What This Changes

The issue asks OpenClaw to remove or ignore operator-only exec controls from model-visible tool-call arguments.

## Best Possible Solution

Land a reviewed fix that removes operator-only exec policy fields from the model schema and ignores or normalizes any such fields before dispatch, while preserving configured defaults and adding regression tests.

## Reproduction Assessment

Yes. Source inspection on current main shows a high-confidence path: model-visible `execSchema` includes the policy fields, and `createExecTool` applies `security: "deny"` into the same denial path reported.

## Solution Assessment

Yes. The best direction is the open PR #76370 shape: remove or ignore model-supplied operator controls and keep session/config defaults as the policy source; the issue should remain open until that PR lands or is replaced.
