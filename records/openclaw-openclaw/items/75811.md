## Summary

Keep open. Current main still exposes exec policy controls to the model-facing schema and consumes model-supplied values into effective exec policy; #76370 is the active closing PR, so the issue should remain open until that or an equivalent fix lands.

## What This Changes

The issue requests separating model-controllable exec command arguments from operator/session-only policy controls such as security, ask, host, elevated, and node.

## Best Possible Solution

Land #76370 or an equivalent patch that removes or ignores model-supplied exec policy controls while preserving trusted config, session-default, and slash-command policy paths with regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows the policy fields are exposed in the exec schema, forwarded to OpenAI tool parameters, and consumed during exec dispatch into the reported denial path.

## Solution Assessment

Yes. The narrow maintainable fix is to split model-facing command/runtime args from operator/session policy and normalize or ignore model-supplied policy controls; docs-only messaging would leave the boundary intact.
