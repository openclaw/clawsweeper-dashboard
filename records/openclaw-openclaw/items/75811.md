## Summary

Keep open: current main still exposes and consumes exec policy fields from model-call args, and the open #76370 PR is the active closing implementation path.

## What This Changes

The issue reports that model-facing exec tool arguments include operator-controlled policy fields, allowing the model to accidentally deny or alter exec behavior.

## Best Possible Solution

Review and land #76370, or replace it with an equivalent narrow fix that removes operator-only exec controls from model-visible args and derives them from config/session defaults.

## Reproduction Assessment

Yes. Current main source exposes the reported policy fields in `execSchema` and consumes them in `createExecTool`; no live model harness was needed to confirm that boundary.

## Solution Assessment

Yes. Hiding or ignoring model-supplied `security`, `ask`, `host`, `node`, and `elevated` is the narrow maintainable fix; trusted overrides should flow through configured exec defaults instead.
