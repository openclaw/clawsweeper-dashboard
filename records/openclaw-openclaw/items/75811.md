## Summary

Keep open. Current main still exposes exec policy controls through the model-facing tool schema and still consumes those model-supplied values during dispatch; the linked open PR is the active fix path, not a reason to close before merge.

## What This Changes

The issue requests splitting exec command/runtime arguments from operator-only policy fields so model tool calls cannot set `security`, `ask`, `host`, `elevated`, or `node`.

## Best Possible Solution

Review and land #76370 or an equivalent patch that removes model control over exec policy while preserving trusted config/session defaults and regression coverage.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: a tool call can provide `security: "deny"` or `security: "allowlist"`, those args feed the effective exec policy, and the host approval path can emit the reported denial.

## Solution Assessment

Yes. Splitting model-visible command/runtime inputs from operator-owned policy at the schema and dispatch boundary is the narrow maintainable fix; message-copy or docs changes would leave the boundary bug intact.
