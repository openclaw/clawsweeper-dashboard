## Summary

Keep open: current main still exposes and honors model-supplied exec policy controls, and an open linked PR is already the active fix path, so the issue should stay open until that PR lands or is replaced.

## What This Changes

The issue asks OpenClaw to remove operator-controlled exec policy fields from the model-facing exec tool-call surface.

## Best Possible Solution

Review and land PR #76370 or an equivalent patch that hides and ignores exec operator-policy fields at the model boundary, then close this issue after merge.

## Reproduction Assessment

Yes. Current main source shows the exec tool schema advertises the policy fields and the runtime consumes those same fields from tool-call args; I did not need a live Hermes run to verify the boundary bug exists.

## Solution Assessment

Yes. Removing these fields from the model-facing schema and deriving them only from configured/session/host policy is the narrowest maintainable fix, with PR #76370 already targeting that path.
