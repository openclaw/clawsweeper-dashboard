## Summary

Keep open: current main still drops config env vars from sandbox exec, so this PR targets a real gap, but the branch is not merge-ready because it forwards the raw collected config map into a security-sensitive sandbox env path without preserving runtime guards or documenting the sandbox contract change.

## What This Changes

The PR adds a `configEnvVars` exec default, layers it into sandbox env construction, wires it from `createOpenClawCodingTools`, and adds env propagation tests.

## Best Possible Solution

Implement an approved sandbox config-env layer that derives the effective runtime env with the same existing skip and placeholder guards, preserves sandbox/per-call precedence, and documents/tests exactly which configured env sources enter sandbox exec.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: configure `env.vars` and run sandbox exec; current main’s sandbox branch builds env from defaults, sandbox env, and per-call env only, while gateway exec inherits sanitized `process.env` after config loading.

## Solution Assessment

No. Threading config env vars into sandbox exec is plausible, but this patch uses the collector directly instead of an effective guarded runtime env and leaves the sandbox env policy undocumented.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve runtime env.vars guards:** `src/agents/pi-tools.ts:408`
  - body: This forwards `collectConfigRuntimeEnvVars(options?.config)` directly into exec defaults. That collector filters names only; it does not skip keys already present in the runtime env or unresolved `${VAR}` placeholders like `applyConfigEnvVars` does, so sandbox exec can now see stale config values or literal placeholders that gateway exec intentionally omits.
  - confidence: 0.93

## Security Review

Status: needs_attention

Summary: The diff expands config-level environment variables into sandbox command environments and needs explicit sandbox/security contract approval before merge.

  - body: Forwarding `env.vars` into every sandbox exec environment changes which configured secrets can reach sandboxed commands and does so through exec-time env serialization rather than the Docker `sanitizeEnvVars` container-env path.
