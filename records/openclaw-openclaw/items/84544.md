## Summary

Keep open: current main preserves the origin/external route for webchat views of an existing channel session, but the session tools still do not expose an unambiguous active-vs-origin channel distinction for agents.

## What This Changes

The issue asks OpenClaw to clarify or expose metadata distinguishing a session's origin/stored channel from the active/current channel when a channel-created session is continued through webchat.

## Best Possible Solution

Keep stable origin-based session keys and external route preservation, then add additive session-tool metadata and docs that clearly separate origin/stored delivery channel from active inbound channel.

## Reproduction Assessment

Yes, at source level: current main preserves the external/stored route for webchat views while session_status and sessions_list do not expose a distinct active/current channel field. I did not run a live Discord-to-webchat E2E.

## Solution Assessment

Yes: an additive tool-output and docs change is the maintainable path. Changing session-key semantics or overwriting preserved routes would regress existing delivery behavior.
