## Summary

Keep open: current main still lacks a clear active-vs-origin session-tool/docs distinction, and this issue is paired with an open closing PR that should be reviewed or rejected before the issue is closed.

## What This Changes

The issue asks OpenClaw to make clear that channel-shaped session keys reflect the origin channel, while an agent's inbound context may reflect a different active channel such as webchat.

## Best Possible Solution

Preserve stable origin-based session keys and external route preservation, land the docs clarification if maintainers accept it, and add only additive tool metadata in a follow-up if structured fields are still needed.

## Reproduction Assessment

Yes, at source level: current main preserves external/stored route metadata for webchat views while session_status and sessions_list do not expose a separate active/current channel field, and the docs do not explain the mismatch. I did not run a live Discord-to-webchat E2E.

## Solution Assessment

Yes for the direction: changing session-key semantics or overwriting preserved routes would regress delivery behavior, so the maintainable path is documentation plus additive metadata only if maintainers still want structured fields.
