- `impact:session-state`: The report concerns agents mis-associating session identity and history when origin and active channel signals differ.

## Summary

Keep open because the source-level mismatch is still present on current main and the linked closing PR is open, not merged. Current code intentionally preserves origin/external routing for webchat views, while the session tools and docs still do not expose or explain a clear active-vs-origin distinction on main.

## What This Changes

The issue asks OpenClaw to clarify that session-key channel segments represent the origin channel, not necessarily the active inbound channel, either through session tool metadata or documentation.

## Best Possible Solution

Keep the origin-based session key and preserved external route semantics, then land the linked docs clarification or add narrowly named tool metadata if maintainers want machine-readable active/origin channel separation.

## Reproduction Assessment

Yes, at source level: current main preserves external/stored route metadata for webchat views while session_status, sessions_list, and the main docs do not clearly distinguish active/current channel from origin/stored channel. I did not run a live Discord-to-webchat E2E.

## Solution Assessment

Yes for the current linked direction: a docs clarification is a narrow maintainable fix for the documented confusion, while additive tool fields can be tracked separately if maintainers want stronger machine-readable semantics.
