## Summary

The PR still carries a focused Slack bug fix that is not on current main: current routing computes normalized DM state, but the thread resolver still cannot use it, so self-thread DM roots in non-`all` reply modes can lose `MessageThreadId`. The latest branch addresses the earlier raw-`channel_type` and non-DM regression concerns; the remaining decision is maintainer acceptance of resolver-level proof versus requesting live Slack transport proof.

## What This Changes

The PR passes normalized Slack direct-message state into Slack thread-context resolution, adds regression coverage for DM assistant self-thread roots and non-DM guards, and adds a changelog fix entry.

## Best Possible Solution

Land this PR, or an equivalent Slack-owned patch, preserving `MessageThreadId` only for normalized DM assistant self-thread roots while keeping non-DM self-thread roots unset.

## Reproduction Assessment

Yes, at source level. Current main normalizes D-prefixed Slack DMs, but `resolveSlackThreadContext` cannot see `isDirectMessage`, so normalized DM roots with `thread_ts === ts` and non-`all` reply modes leave `MessageThreadId` undefined.

## Solution Assessment

Yes. Passing the already-normalized direct-message classification into the existing thread resolver is the narrow maintainable fix, and the branch keeps the necessary non-DM self-thread guard.
