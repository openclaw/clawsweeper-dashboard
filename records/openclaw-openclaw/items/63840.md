## Summary

Keep open: current main still drops the Slack DM assistant-root `MessageThreadId` outside `replyToMode: "all"`, while this PR uses the existing normalized direct-message classification and adds focused regression coverage with sufficient real behavior proof.

## What This Changes

The PR passes normalized Slack direct-message state into thread-context resolution so DM assistant self-thread roots preserve `MessageThreadId`, adds Slack regression tests, and updates the changelog.

## Best Possible Solution

Land this PR or an equivalent Slack-owned patch that preserves `MessageThreadId` only for normalized DM assistant roots while keeping non-DM self-thread roots top-level.

## Reproduction Assessment

Yes, at source level. Current main has a concrete path where a normalized D-prefix DM root with `thread_ts === ts`, `isDirectMessage: true`, and reply mode `off`, `first`, or `batched` leaves `MessageThreadId` undefined.

## Solution Assessment

Yes. Passing the existing normalized direct-message flag into the Slack thread resolver is the narrow maintainable fix because it avoids trusting raw `channel_type` and does not broaden channel/group self-thread behavior.
