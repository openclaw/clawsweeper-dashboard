## Summary

Keep open: current main and the latest release still drop `MessageThreadId` for normalized Slack DM self-thread roots outside `replyToMode=all`, while the latest PR head uses normalized DM state and has no blocking patch defect found.

## What This Changes

The PR passes normalized Slack DM state into thread-context resolution, preserves `MessageThreadId` for assistant DM self-thread roots, adds Slack regression tests, and updates the changelog.

## Best Possible Solution

Land this PR or an equivalent Slack-owned patch that preserves `MessageThreadId` only for normalized DM assistant roots while leaving non-DM self-thread roots top-level.

## Reproduction Assessment

Yes, at source level. Current main has a concrete path where a normalized D-prefix DM root with `thread_ts === ts` and reply mode `off`, `first`, or `batched` leaves `MessageThreadId` undefined; I did not run tests because this was a read-only review.

## Solution Assessment

Yes. Passing the existing normalized direct-message flag into the Slack thread resolver is the narrow maintainable fix because it avoids trusting raw `channel_type` and does not broaden channel/group self-thread behavior.
