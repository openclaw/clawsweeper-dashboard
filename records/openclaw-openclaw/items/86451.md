## Summary

Keep open: current main and latest release still allow equivalent cron jobs to be appended or imported as separate enabled rows, and the open duplicate-name PR only covers a narrower name-collision slice of this update/migration report.

## What This Changes

The issue reports that `openclaw update` or cron migration can recreate equivalent cron jobs without deduplication, causing duplicate enabled jobs and agent/session routing drift.

## Best Possible Solution

Add maintainer-approved cron duplicate audit, repair, and deduplication behavior across runtime add/update and legacy import, with upgrade proof for existing duplicate rows before closing this tracker.

## Reproduction Assessment

Yes for a source-level reproduction: current main accepts repeated equivalent `cron.add` calls as distinct jobs and the legacy migration merge deduplicates only matching ids/jobIds. No full end-to-end `openclaw update` replay was run in this read-only review.
