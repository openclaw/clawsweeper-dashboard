## Summary

Keep open: the PR is a plausible small compaction tweak, but the fixed 1.5 margin does not cover the stated CJK chars/4 underestimation and the supplied proof is only tests/typecheck output, not an after-fix real compaction run.

## What This Changes

This PR raises the shared compaction `SAFETY_MARGIN` from 1.2 to 1.5 and adds an Unreleased changelog entry.

## Best Possible Solution

Use the existing CJK-aware token helper, or an equivalent content-aware estimator, in the compaction token path while keeping explicit headroom for special-token noise, then prove it with a real CJK-heavy compaction run.

## Reproduction Assessment

No high-confidence live reproduction was established. The source path is clear because the dependency estimator divides text chars by 4 while OpenClaw's CJK helper documents a 2-4x CJK undercount, but the PR did not show an actual CJK compaction run.

## Solution Assessment

No. A margin-only change is not the narrowest maintainable fix for the stated CJK failure because 1.5x still undercounts CJK-heavy text and the repo already has a CJK-aware estimator to reuse or adapt.
