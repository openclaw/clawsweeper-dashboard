## What This Changes

The PR raises the agent compaction token-estimation safety margin from 1.2 to 1.5, adds regression tests for the new chunk budget, and adds a changelog entry.

## Best Possible Solution

Use content-aware compaction token estimates that reuse the existing CJK weighting helper, keep separate constants for chunk headroom versus pruning tolerance, and prove the change with CJK-heavy regression coverage plus redacted real behavior proof.

## Reproduction Assessment

No high-confidence live reproduction is established. Source inspection does show the underestimation path: OpenClaw feeds messages into a dependency estimator that divides raw character counts by 4, while existing docs/code comments acknowledge CJK can be closer to 1 token per character.

## Solution Assessment

No. A global 1.5 margin is not the best fix for the stated CJK failure because it still materially undercounts CJK-heavy content and also changes pruning behavior through other SAFETY_MARGIN call sites.
