## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Honor explicit completion routes from internal requesters:** `src/agents/subagent-announce.ts:780`
  - body: `directOriginIsInternal` is derived from the original requester route, not the effective completion route. When a webchat requester has a valid external `completionDirectOrigin`, this forces `deliver=false` and drops that explicit completion delivery.
  - confidence: 0.9
- **[P2] Filter hidden mirror entries before applying the history limit:** `src/gateway/server-methods/chat.ts:746-747`
  - body: The PR filters delivery-mirror messages after slicing the raw tail. If the newest entries are mirrors, `chat.history` can return fewer visible messages than requested even though older visible messages exist.
  - confidence: 0.88
- **[P2] Require install intent before silent plugin shadowing:** `src/plugins/manifest-registry.ts:238`
  - body: The origin-rank branch suppresses duplicate diagnostics and lets a different physical plugin copy win whenever origins differ. Current main only suppresses installed or same-physical-root cases, so this can silently change plugin code precedence.
  - confidence: 0.84

## Security Review

Status: needs_attention

Summary: The diff introduces a plugin supply-chain concern by allowing silent plugin shadowing based only on origin rank.
