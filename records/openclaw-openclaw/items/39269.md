## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Honor explicit completion routes from internal requesters:** `src/agents/subagent-announce.ts:780`
  - body: `directOriginIsInternal` is computed from the original requester route, not the effective completion route. When a webchat requester has a valid `completionDirectOrigin` from a binding or hook, this new guard forces `deliver=false` and silently drops the explicit external completion delivery.
  - confidence: 0.9
- **[P2] Filter mirror entries before applying the history limit:** `src/gateway/server-methods/chat.ts:746-747`
  - body: This removes `delivery-mirror` messages after slicing the raw tail to `max`. If the newest entries are mirrors, chat.history can return fewer visible messages than requested even though older visible messages exist; project/filter first, then limit visible messages.
  - confidence: 0.88
- **[P2] Require install intent before silent plugin shadowing:** `src/plugins/manifest-registry.ts:238`
  - body: The new origin-rank branch suppresses duplicate warnings and lets a different physical plugin copy win whenever origins differ. For bundled plugin duplicates, current main only treats installed globals as intentional overrides; this would silently change plugin code precedence for auto-discovered copies.
  - confidence: 0.84
