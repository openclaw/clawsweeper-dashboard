## Summary

Keep open: current main still resolves the Claude CLI runtime path through `claude-cli` before checking canonical Anthropic `contextTokens` caps, so a configured 100k budget can still be bypassed by the 1M Anthropic fallback. The closed companion PR is useful evidence for a narrow fix, but it was not merged.

## What This Changes

The issue asks OpenClaw to honor configured `contextTokens` caps for canonical Anthropic models executed through the `claude-cli` runtime so CLI compaction triggers at the configured budget.

## Best Possible Solution

Resolve CLI-runtime aliases to canonical provider config for explicit `contextTokens` cap lookup before the Anthropic 1M fallback, persist that cap into `SessionEntry.contextTokens`, and cover resolver plus session-store behavior with focused regressions.

## Reproduction Assessment

Yes. There is a high-confidence source-level reproduction path: `providerUsed` becomes `claude-cli`, `modelUsed` can be `anthropic/claude-opus-4-7`, and current resolver ordering returns the 1M fallback before checking `models.providers.anthropic.models[].contextTokens`.

## Solution Assessment

Yes. The narrow maintainable fix is alias-aware configured cap lookup before 1M fallback, preferably reusing `src/agents/model-runtime-aliases.ts`; https://github.com/openclaw/openclaw/pull/80937 is useful prior art but is closed and unmerged.
