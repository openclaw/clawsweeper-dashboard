## Summary

Keep open: current main still resolves the Claude CLI runtime path through the Anthropic 1M shortcut before checking the configured Anthropic `contextTokens` cap, so the reported compaction budget can still be bypassed.

## What This Changes

The issue reports that Claude CLI runtime sessions ignore configured `contextTokens` budgets and therefore skip CLI transcript compaction.

## Best Possible Solution

Resolve explicit configured `contextTokens` caps through the canonical provider/runtime alias before the Anthropic 1M fallback, persist that cap into `SessionEntry.contextTokens`, and add focused resolver/session-store/compaction regressions.

## Reproduction Assessment

Yes. There is a high-confidence source-level reproduction path: Claude CLI runs can persist `providerUsed = "claude-cli"`, the resolver returns the 1M fallback before configured caps, and CLI compaction gates on the persisted session budget.

## Solution Assessment

Yes. The narrow maintainable solution is alias-aware configured cap lookup before the 1M fallback, reusing `src/agents/model-runtime-aliases.ts`; the closed unmerged companion PR is useful prior art.
