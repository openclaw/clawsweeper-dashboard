## Summary

This should stay open: current main appears to fix the missing session field part from the related status work, but the reported configured-budget path is still source-reproducible when a canonical Anthropic model is executed through the Claude CLI runtime.

## What This Changes

The issue requests that Claude CLI runtime sessions honor the configured `contextTokens` budget so CLI transcript compaction triggers at the expected threshold.

## Best Possible Solution

Carry the selected canonical provider/model context budget into session persistence for CLI runtime turns, then cover Claude CLI compaction with a regression using an Anthropic model-level `contextTokens` cap.

## Reproduction Assessment

Yes, there is a high-confidence source-level reproduction path: canonical `anthropic/claude-opus-4-7` with `agentRuntime.id: "claude-cli"` returns CLI metadata as provider `claude-cli`, so current session-store budget lookup misses the Anthropic model-level cap. I did not execute a live Claude CLI run during this read-only review.

## Solution Assessment

No, the current related status fix is not the full solution for this issue because it can populate a default or 1M runtime budget while still ignoring the configured 100k cap. The narrow maintainable fix is to resolve the budget from the selected canonical model config, or explicitly canonicalize the CLI runtime alias for context-token lookup before compaction.
