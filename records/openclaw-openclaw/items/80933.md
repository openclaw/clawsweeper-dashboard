Keep this open: current main still has the reported claude-cli configured-budget resolver gap, and https://github.com/openclaw/openclaw/pull/93198 is the live closing PR candidate rather than a merged fix.

## What This Changes

The issue reports that claude-cli runtime sessions ignore configured contextTokens budgets, causing compaction decisions to use the wrong persisted session budget.

## Best Possible Solution

Review and land the linked fix PR, or close that PR with rationale and keep this issue as the canonical source-backed bug until another resolver/session-store fix lands.

## Reproduction Assessment

Yes, by source inspection: Claude CLI runs persist provider metadata as claude-cli, session-store resolves contextTokens using that runtime provider, and the resolver does not inspect canonical anthropic config before fallback. I did not run a live Claude CLI OAuth compaction session.

## Solution Assessment

Yes. The narrow maintainable fix belongs at the context-token resolver/session-store boundary so the existing configured Anthropic cap is persisted, not in a new config option or compaction threshold change.
