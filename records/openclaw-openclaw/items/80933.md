- `impact:session-state`: The issue concerns persisted session budget state that controls transcript compaction and context growth.

## Summary

Keep open: current main still resolves explicit `claude-cli` Anthropic 1M models to the 1M fallback before honoring the configured canonical `anthropic` `contextTokens` cap. The closed companion fix is useful prior art, but it is unmerged and not contained in `HEAD`.

## What This Changes

The issue requests that Claude CLI runtime sessions persist configured Anthropic `contextTokens` caps and use them to trigger CLI transcript compaction.

## Best Possible Solution

Resolve explicit `contextTokens` caps through the existing runtime-provider alias table before Anthropic 1M/default fallbacks, then cover resolver, session-store, and CLI compaction behavior with focused regressions.

## Reproduction Assessment

Yes, by source inspection. A Claude CLI run using canonical `anthropic/claude-opus-4-7` reaches budget resolution with `provider: claude-cli`, where current main can return the Anthropic 1M fallback before checking the configured canonical Anthropic `contextTokens` cap.

## Solution Assessment

Yes. The narrow maintainable fix is alias-aware configured-cap lookup before fallback windows, using the existing runtime alias table instead of adding a new config option or changing compaction policy.
