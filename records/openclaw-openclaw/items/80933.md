## Summary

Keep this open: current main appears to have moved past the original missing-field symptom, but it still ignores an Anthropic model-level `contextTokens` cap when the same canonical model is executed through the `claude-cli` runtime, so compaction can still evaluate against the wrong budget.

## What This Changes

The issue reports that Claude CLI runtime sessions ignore configured Anthropic `contextTokens` caps, preventing CLI transcript compaction from triggering at the intended budget.

## Best Possible Solution

Resolve configured `contextTokens` for CLI runtime aliases from the canonical provider/model config before Anthropic 1M fallback, persist that budget into the session entry, and add regression coverage for Claude CLI compaction with an Anthropic model cap.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: a canonical Anthropic model routed through `claude-cli` records CLI runtime metadata, then current budget resolution returns the Anthropic 1M runtime window before checking the configured 100k Anthropic model cap. I did not execute a live Claude CLI run in this read-only review.

## Solution Assessment

Yes, the best fix is a narrow alias-aware budget resolution change, not a broader compaction rewrite. The closed related PR demonstrates the intended direction, but current main still needs the fix landed or reimplemented.
