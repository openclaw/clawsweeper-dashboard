## Label Justifications

## Summary

Keep open: current main still ignores the configured Claude CLI context budget for the reported route because the resolver returns the Anthropic 1M fallback before checking the canonical `anthropic` `contextTokens` cap. The targeted fix in https://github.com/openclaw/openclaw/pull/80937 is useful prior art, but that PR is closed and unmerged.

## What This Changes

The issue asks OpenClaw to honor configured `contextTokens` caps for Anthropic models routed through `claude-cli` so CLI compaction uses the user’s budget.

## Best Possible Solution

Make explicit `contextTokens` caps authoritative for canonical Anthropic models run via `claude-cli` by resolving runtime aliases before fallback windows, with regression coverage for 1M and non-1M routes.

## Reproduction Assessment

Yes, by source inspection rather than a live run. On current main, a `claude-cli` route for `anthropic/claude-opus-4-7` reaches `resolveContextTokensForModel` with provider `claude-cli` and returns the 1M fallback before the configured `anthropic` cap can be checked.

## Solution Assessment

Yes. The narrow fix is to reuse the existing runtime-provider alias seam for configured `contextTokens` lookup before Anthropic fallback defaults, without adding config surface or changing compaction policy.
