## Summary

Keep open: current main still resolves the Claude CLI runtime path to the Anthropic 1M fallback before checking the reporter's canonical `anthropic` model `contextTokens` cap, so the configured 100k budget can still be bypassed. The closed companion fix is useful prior art, but it was not merged.

## What This Changes

The issue asks OpenClaw to honor configured `contextTokens` budgets for Anthropic models routed through the `claude-cli` runtime so CLI transcript compaction triggers at the intended threshold.

## Best Possible Solution

Resolve CLI runtime aliases to canonical provider config for explicit `contextTokens` lookup before the Anthropic 1M fallback, persist that cap into `SessionEntry.contextTokens`, and add focused resolver/session-store/compaction regressions.

## Reproduction Assessment

Yes, by source inspection. A `claude-cli` run using `anthropic/claude-opus-4-7` reaches session-store budget resolution with provider `claude-cli`, and the resolver returns the Anthropic 1M fallback before checking the canonical configured 100k cap.

## Solution Assessment

Yes. The narrow maintainable fix is alias-aware configured-cap lookup before the 1M fallback, reusing `src/agents/model-runtime-aliases.ts`; https://github.com/openclaw/openclaw/pull/80937 is useful prior art but is closed and unmerged.
