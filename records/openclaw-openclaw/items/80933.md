## Summary

Keep this open: current main still ignores the configured Claude CLI runtime budget for canonical Anthropic model config, although the exact failure mode has shifted from an unset budget to an over-large 1M budget. The closed companion PR is useful prior art, but it was not merged and no open canonical fix remains.

## What This Changes

The issue reports that Claude CLI runtime sessions ignore configured `contextTokens` budgets, preventing CLI transcript compaction from triggering at the intended cap.

## Best Possible Solution

Honor explicit canonical provider `contextTokens` caps for runtime-provider aliases before Anthropic fallback windows, persist that cap into `SessionEntry.contextTokens`, and cover resolver plus session-store behavior with focused regressions.

## Reproduction Assessment

Yes, by source inspection. A Claude CLI run can persist its session budget via `resolveContextTokensForModel` with provider `claude-cli`, where current resolver ordering returns the Anthropic 1M fallback before checking canonical `anthropic` configured `contextTokens` caps.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse the existing runtime alias mapping for configured-cap lookup before the Anthropic 1M fallback, without changing compaction policy or adding configuration.
