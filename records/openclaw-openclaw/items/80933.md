## What This Changes

The issue reports that Claude CLI runtime sessions ignore configured model `contextTokens` budgets, preventing CLI transcript compaction from triggering at the intended threshold.

## Best Possible Solution

Resolve configured `contextTokens` through the existing runtime-provider alias table before Anthropic fallback defaults, persist the configured cap into `SessionEntry.contextTokens`, and add focused resolver, session-store, and compaction regressions.

## Solution Assessment

Yes. The narrow maintainable fix is alias-aware configured-cap lookup before fallback defaults, reusing the existing `claude-cli` to `anthropic` alias seam rather than changing compaction policy or adding config.
