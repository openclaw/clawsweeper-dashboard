## Summary

Current main still appears to bypass the configured Claude CLI budget: session persistence asks the context resolver for `provider: claude-cli`, and the resolver can return the Anthropic 1M fallback before consulting the canonical `anthropic` model `contextTokens` cap. The closed companion fix is useful prior art, but it was not merged, so this should stay open as a narrow bug-fix candidate.

## What This Changes

The issue requests that Claude CLI runtime sessions honor configured model `contextTokens` budgets so CLI transcript compaction triggers at the configured cap.

## Best Possible Solution

Resolve runtime-provider aliases to canonical provider configuration for explicit `contextTokens` lookup before Anthropic fallback defaults, persist that cap into `SessionEntry.contextTokens`, and cover resolver, session-store, and CLI compaction behavior with focused regressions.

## Reproduction Assessment

Yes, by source inspection. A Claude CLI run using a canonical Anthropic model can reach budget resolution with provider `claude-cli`, where current resolver ordering can return the 1M Anthropic fallback before checking the configured canonical `anthropic` `contextTokens` cap.

## Solution Assessment

Yes. The narrow maintainable solution is to reuse the existing runtime alias seam for configured-cap lookup before fallback defaults, rather than adding a new config option or changing compaction policy.
