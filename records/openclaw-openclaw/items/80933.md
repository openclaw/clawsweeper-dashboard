## Summary

This should stay open: current main appears to have fixed the missing-session-field part of the original report, but it still ignores an explicit `models.providers.anthropic.models[].contextTokens` cap for Anthropic models executed through the `claude-cli` runtime. The closed companion PR had a plausible narrow fix and proof, but it was closed unmerged, so there is still actionable source-level work here.

## What This Changes

The issue asks OpenClaw to honor configured `contextTokens` budgets for canonical Anthropic models routed through the `claude-cli` runtime so CLI transcript compaction triggers at the configured cap.

## Best Possible Solution

Land a narrow resolver/session-store fix that treats explicit configured `contextTokens` caps as authoritative for canonical Anthropic models executed by `claude-cli`, while preserving the 1M Anthropic runtime window when no explicit cap is configured.

## Reproduction Assessment

Yes. A high-confidence source reproduction is `anthropic/claude-opus-4-7` with model-scoped `agentRuntime.id: "claude-cli"` and `models.providers.anthropic.models[].contextTokens = 100000`; current main resolves the session budget from `claude-cli` runtime metadata and returns 1M before consulting the configured Anthropic cap.

## Solution Assessment

Yes. The maintainable fix is to resolve explicit configured caps through the existing canonical provider/runtime alias relationship before the Anthropic 1M fallback, with focused resolver and session-store regression coverage.
