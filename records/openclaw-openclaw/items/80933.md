## Summary

Keep open: current main still can bypass the reporter's configured 100k `contextTokens` cap for Claude CLI compaction by resolving the route to the Anthropic 1M fallback first. The related display fix is useful context, and the direct fix attempt was closed unmerged, so the core bug remains actionable.

## What This Changes

The issue asks OpenClaw to honor configured `contextTokens` budgets for `claude-cli` runtime sessions so CLI transcript compaction triggers at the configured threshold.

## Best Possible Solution

Resolve CLI runtime aliases to canonical provider config before applying the Anthropic 1M fallback, persist the explicit configured cap into `SessionEntry.contextTokens`, and add focused resolver, session-store, and compaction regressions.

## Reproduction Assessment

Yes, by source inspection plus the reporter's live fleet output. A `claude-cli` run can persist its compaction budget through `resolveContextTokensForModel`, where the current ordering can return the Anthropic 1M fallback before honoring the canonical configured 100k cap.

## Solution Assessment

Yes. The narrowest maintainable fix is to reuse the existing runtime alias mapping so explicit canonical `contextTokens` caps win before the Anthropic 1M fallback, rather than adding a new config surface or a compaction-only special case.
