## Summary

Keep open: current main still source-reproduces the configured-budget failure for canonical Anthropic models executed through the Claude CLI runtime, and the linked fix PR was closed without merging.

## What This Changes

Ensure Claude CLI runtime sessions honor configured canonical Anthropic model `contextTokens` budgets when persisting the session budget used by CLI compaction.

## Best Possible Solution

Make canonical provider/model `contextTokens` caps authoritative for CLI runtime aliases before the Anthropic 1M fallback, while preserving the 1M default when no explicit cap is configured.

## Reproduction Assessment

Yes, source-reproducible: `claude-cli` runtime metadata feeds the session-store resolver as provider `claude-cli`, and the resolver returns the 1M Anthropic window before checking the canonical `anthropic` model cap. I did not execute a live Claude CLI run.

## Solution Assessment

No, current main is not the best solution because it still bypasses configured caps for this route. The narrow maintainable fix is alias-aware configured-cap lookup before the 1M fallback, with regression coverage for session persistence and CLI compaction budget use.
