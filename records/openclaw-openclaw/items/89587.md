The PR wraps plugin JSON-schema validation internals so unreadable schema objects report sanitized invalid-schema errors and adds cached and uncached regression coverage.

## Best Possible Solution

## Reproduction Assessment

## Solution Assessment

Yes, this is the best fix shape: the shared validator is the common owner for plugin/channel JSON Schema validation, and wrapping only individual callers would miss SDK, install, and runtime validation surfaces.
