The PR wraps plugin JSON-schema validation operations so unreadable schema objects become sanitized invalid-schema errors and adds cached/uncached regression coverage.

## Best Possible Solution

Land the shared validator mitigation once the draft/protected maintainer review is satisfied, keeping unreadable plugin schema normalization centralized in `validateJsonSchemaValue`.

## Solution Assessment

Yes, this is the best fix shape: the shared validator is the common owner for plugin/channel JSON Schema validation, while per-caller wrapping would miss SDK, install, config, and runtime validation surfaces.
