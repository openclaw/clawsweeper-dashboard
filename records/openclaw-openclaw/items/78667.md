## Summary

Close as ClawHub scope: OCI GenAI is an optional provider plugin that can be published through the existing external provider-plugin path, and this branch still does not wire its advertised OpenClaw chat and memory-registration paths correctly.

## What This Changes

The PR adds a bundled Oracle Cloud Infrastructure Generative AI provider plugin with OCI docs, tests, onboarding/catalog code, a memory embedding adapter, request-signing helpers, and lockfile metadata.

## Best Possible Solution

Publish OCI GenAI as an external ClawHub provider plugin, using a provider-owned signed chat transport and declaring the memory embedding contract before publication; core should only change if a generic plugin API gap is identified.

## Reproduction Assessment

Not applicable as a current-main bug; this is a feature PR. The merge blockers are source-reproducible from the PR head plus current provider transport and memory contract code.

## Solution Assessment

No; bundling a new OCI provider is not the narrowest maintainable path under the current plugin/ClawHub policy, and the branch also needs provider-owned signing and manifest ownership fixes before it can work as advertised.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P2] Wire OCI chat through signed requests:** `extensions/oci-genai/provider-catalog.ts:31`
  - body: The catalog registers OCI chat as the generic `openai-completions` transport, but the current OpenAI-compatible transport builds a normal OpenAI SDK client with the captured value as `apiKey` and `buildGuardedModelFetch`. That means catalog-driven OCI chat never uses `createOciSignedFetch`, so real OpenClaw chat requests will not carry OCI RSA `Signature` auth.
  - confidence: 0.94
- **[P2] Declare the memory embedding provider contract:** `extensions/oci-genai/index.ts:96`
  - body: This plugin calls `api.registerMemoryEmbeddingProvider(ociMemoryEmbeddingProviderAdapter)`, but its manifest does not declare `contracts.memoryEmbeddingProviders`. The current registry rejects non-memory plugins that register an undeclared memory adapter, so the OCI memory provider will not be registered.
  - confidence: 0.93
