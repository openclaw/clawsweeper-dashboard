## Summary

The updated proof does not change that scope decision, and the proposed bundled patch also has source-proven chat-signing and memory-contract blockers.

## What This Changes

The PR adds an in-repo bundled `extensions/oci-genai` provider plugin with OCI docs, tests, a changelog entry, and a workspace lockfile entry.

## Best Possible Solution

Publish OCI GenAI as a ClawHub provider plugin, and return to OpenClaw core only for a generic missing SDK seam or explicit maintainer-approved bundled promotion.

## Reproduction Assessment

Not applicable; this is a feature PR, not a bug report. The scope decision and patch blockers are high-confidence from current main source/docs plus the PR body and diff context.

## Solution Assessment

No; bundling a new optional OCI provider in core is not the narrowest maintainable solution under the current VISION and plugin docs. External ClawHub publication uses the existing provider and memory embedding APIs without expanding core.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Wire signed fetch into chat transport:** `extensions/oci-genai/provider-catalog.ts:31`
  - body: `buildOciProvider()` registers OCI chat as a normal `openai-completions` provider. Current main turns that into an OpenAI SDK client using the resolved value as `apiKey` plus the generic guarded fetch, so chat requests will not receive OCI's per-request RSA `Signature` Authorization header, date, content length, or body hash.
  - confidence: 0.9
- **[P2] Declare the memory embedding contract:** `extensions/oci-genai/openclaw.plugin.json:8`
  - body: The plugin registers an OCI memory embedding adapter, but the manifest does not declare `contracts.memoryEmbeddingProviders`. Current plugin loading emits an error and skips non-memory plugin adapters without that ownership declaration, so the advertised Cohere V3 memory adapter will not register.
  - confidence: 0.87
