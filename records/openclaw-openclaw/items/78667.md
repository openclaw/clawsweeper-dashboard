## Summary

Keep open for maintainer review: the ClawHub scope policy is real, but the author raised a concrete bundled-provider precedent question that needs explicit product judgment, and the current patch still has source-proven chat-auth and memory-contract blockers.

## What This Changes

The PR adds a bundled OCI Generative AI provider plugin with docs, tests, changelog, and workspace lockfile metadata.

## Best Possible Solution

Have a maintainer decide whether OCI should be bundled or published through ClawHub; if bundled is approved, require a signed chat transport, the memory contract declaration, focused regression coverage, docs/changelog alignment, and redacted real OpenClaw behavior proof.

## Reproduction Assessment

Not applicable; this is a feature PR rather than a bug report. The merge blockers are source-reproducible from PR head plus current main contracts: OCI chat is routed through generic OpenAI transport, and the memory adapter lacks its manifest contract.

## Solution Assessment

No; the current patch is not mergeable because the default chat path will not use RSA signing and memory embedding registration is rejected. If maintainers approve bundling, the narrow technical path is a provider-owned `wrapStreamFn` or `createStreamFn` plus the manifest contract; otherwise ClawHub publication is sufficient.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Wire OCI signing into the chat transport:** `extensions/oci-genai/provider-catalog.ts:31`
  - body: `buildOciProvider` registers OCI chat as `openai-completions`, but current main's OpenAI-completions path builds a normal OpenAI SDK client with `apiKey` and `buildGuardedModelFetch`. The PR's `createOciSignedFetch` is never used for catalog-driven chat, so OCI requests will be sent as Bearer-token OpenAI-compatible calls instead of RSA `Signature` requests and the advertised default chat path will fail authentication.
  - confidence: 0.94
- **[P2] Declare the memory embedding provider contract:** `extensions/oci-genai/openclaw.plugin.json:7`
  - body: `index.ts` registers `ociMemoryEmbeddingProviderAdapter`, but this manifest declares only `providers` and omits `contracts.memoryEmbeddingProviders`. Current plugin loading emits an error and returns for non-memory plugins that register an undeclared memory adapter, so the OCI embedding provider will not be registered.
  - confidence: 0.92
