## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Wire OCI signing into the chat transport:** `extensions/oci-genai/provider-catalog.ts:31`
  - body: `buildOciProvider` registers OCI chat as `openai-completions`, but current main's OpenAI-completions path constructs a normal OpenAI SDK client with `apiKey` and `buildGuardedModelFetch`. `createOciSignedFetch` is never used for catalog-driven chat, so OCI requests will be sent through the Bearer-token path instead of RSA `Signature` auth.
  - confidence: 0.94
- **[P2] Declare the memory embedding provider contract:** `extensions/oci-genai/openclaw.plugin.json:7`
  - body: `index.ts` registers `ociMemoryEmbeddingProviderAdapter`, but this manifest declares only `providers` and omits `contracts.memoryEmbeddingProviders`. Current plugin loading rejects non-memory plugins that register undeclared memory adapters, so OCI embeddings will not be registered.
  - confidence: 0.92
- **[P2] Honor plugin config before probing OCI credentials:** `extensions/oci-genai/memory-embedding-adapter.ts:18`
  - body: The adapter calls `hasOciCredentials()` before `createOciEmbeddingProvider()` can read `plugins.oci-genai.profileName` or `configFile`, and the probe only checks env/default config. Users who configure a non-default profile through plugin config can be rejected before the provider reads that config.
  - confidence: 0.83
