## Summary

Keep this PR open, but do not merge it as-is. Current main still lacks the fallback-aware typing TTL and timeout-specific final reply behavior, and the PR is a narrow implementation path, but its timeout copy advertises `/retry` even though OpenClaw does not register or document that slash command.

## What This Changes

This PR adds fallback-aware auto-reply typing TTL budgeting, timeout-specific final reply copy, regression tests, and a changelog entry.

## Best Possible Solution

Keep the PR's fallback-chain TTL approach, but change the timeout final reply so it only suggests supported actions, such as sending a new message. A real `/retry` command should be a separate, intentional feature with command registry, handler, docs, and tests.
