## Summary

Keep open for maintainer handling. The PR is paired with the still-open packaged-install report #75685, and the diff is not safe to merge as written because it routes built plugin manifests through a root dependency mirror check even though current main treats Telegram, Discord, and similar plugins as package-local staged runtime-dependency owners.

## What This Changes

The PR adds a release-check pass over built `dist/extensions/*/package.json` manifests, labels those gaps as built-root dependency drift, and adds a Telegram dependency regression unit test.

## Best Possible Solution

Resolve #75685 against current main first. If a release guard is still missing, add it to the current staged-runtime-deps and packaged/post-publish verification helpers rather than reviving a built-manifest root dependency mirror check for all bundled plugin manifests.

## Reproduction Assessment

Yes. #75685 provides concrete 2026.4.29 packaged-install missing-module logs, and the PR's false-positive path is statically reproducible by comparing its new built-manifest root check with current Telegram/Discord staged runtime-dependency manifests.

## Solution Assessment

No. The PR's approach conflicts with current main's staged plugin runtime-dependency model; the maintainable solution is to validate staged payloads under each plugin or package-level runtime-deps root, while using root mirror checks only for root-owned dist importers.
