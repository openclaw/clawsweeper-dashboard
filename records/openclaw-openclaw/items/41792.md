The branch adds an `openclaw/plugin-sdk/backup` package export, SDK entrypoint inventory item, and facade that re-exports backup plan/archive helpers and backup-related types.

## Best Possible Solution

Publish a documented, metadata-backed backup SDK facade with an explicit trusted-plugin path-data contract, or expose a narrower helper if maintainers do not want full backup-plan paths public.

## Reproduction Assessment

## Solution Assessment

No, not as-is. A narrow backup facade may be reasonable, but it should land only after maintainer approval of the data boundary, SDK docs/metadata/API baseline updates, and real import-path proof.
