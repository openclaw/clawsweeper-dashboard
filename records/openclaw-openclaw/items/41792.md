## Summary

Keep open: current main still does not expose a dedicated backup SDK subpath, but this PR should not merge as-is because the new public API exposes local backup-plan path data without the required SDK docs, metadata/API-baseline, and changelog alignment.

## What This Changes

The PR adds an `openclaw/plugin-sdk/backup` facade that re-exports backup archive and plan helpers, then wires that subpath into package exports and the plugin SDK entrypoint list.

## Best Possible Solution

Approve or reject the backup-planning SDK boundary explicitly; if approved, land it as a documented trusted-plugin surface with metadata, API baseline, export checks, and changelog updated together.

## Reproduction Assessment

Not applicable as a runtime bug. The high-confidence review path is static: current main lacks the subpath, the PR adds a public SDK facade, and the missing docs/API/changelog plus path-exposure boundary are visible in source.

## Solution Assessment

No, not as-is. The facade is a plausible narrow shape, but the maintainable solution needs maintainer API/security approval and the public SDK documentation, metadata, baseline, and changelog updates before merge.
