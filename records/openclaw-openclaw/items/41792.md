The branch adds a public `openclaw/plugin-sdk/backup` subpath via `package.json`, plugin SDK entrypoint metadata, and a new facade that re-exports backup archive helpers, backup plan resolver/types, and backup create option/result types.

## Best Possible Solution

Land a rebased version only if maintainers want backup planning to become a supported plugin API; keep the subpath narrow, document the trusted-plugin local data boundary, add docs/API metadata, regenerate the SDK API hash, and run the focused plugin SDK export and baseline checks.
