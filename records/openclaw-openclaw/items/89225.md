## Reproduction Assessment

Yes, source-reproducible: doctor schema projection constructs coding tools, current main eagerly starts `fsRoot(root)` for workspace-only write/edit operations, and the pinned fs-safe dependency rejects missing roots with the reported error. I did not run the CLI because this review is read-only.

## Solution Assessment

Yes. Lazy, memoized root resolution at the host workspace operation boundary is the narrow maintainable fix because schema-only construction stops touching the filesystem while real read/write/access calls still use fs-safe enforcement.
