## What This Changes

The issue reports that `openclaw doctor` can abort with an unhandled `FsSafeError: root dir not found` when schema-only tool projection constructs workspace-scoped write/edit tools for a missing agent workspace.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/89226, or an equivalent maintainer-owned lazy-root fix, then close this issue after the fix reaches main.

## Reproduction Assessment

Yes. Source inspection shows doctor schema projection constructs the coding toolset, and current workspace-scoped write/edit factories eagerly call `fsRoot(root)` even though schema inspection does not invoke filesystem operations.

## Solution Assessment

Yes. Lazy, memoized root resolution in the linked PR is the narrowest maintainable direction because it removes filesystem access from schema-only construction while keeping workspace-boundary checks on first real tool use.
