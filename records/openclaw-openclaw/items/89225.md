## What This Changes

The issue reports that `openclaw doctor` can abort with an unhandled `FsSafeError` when schema projection constructs workspace-scoped tools for an agent whose workspace directory is missing.

## Best Possible Solution

Land a narrow fix through the linked PR or an equivalent maintainer-owned patch that makes schema-only doctor inspection workspace-independent while keeping workspace-boundary enforcement on first real tool use.

## Reproduction Assessment

Yes. Source inspection shows doctor schema projection constructs the coding toolset, and current workspace-scoped edit/write factories eagerly call `fsRoot(root)` even though schema inspection never invokes the filesystem operations.

## Solution Assessment

Yes, the linked lazy-root direction appears to be the narrowest maintainable fix because it preserves the existing workspace checks when tools run while removing filesystem access from schema-only construction.
