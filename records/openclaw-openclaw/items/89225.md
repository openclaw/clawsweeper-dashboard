## Reproduction Assessment

Yes. Source inspection shows doctor schema projection constructs the coding toolset, current main eagerly opens workspace fs-safe roots during host write/edit operation construction, and fs-safe rejects missing roots with the reported error.

## Solution Assessment

Yes. Lazy, memoized root resolution at the host workspace operation boundary is the narrowest maintainable fix because schema-only construction stops touching the filesystem while real operations still validate paths before fs-safe access.
