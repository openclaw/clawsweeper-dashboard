## Summary

Keep open. Current main and the latest release still batch the classic Voice Call STT -> embedded-agent -> telephony TTS path; the full-duplex realtime mode is a separate bypass, and the prior streaming-TTS implementation at https://github.com/openclaw/openclaw/pull/42187 was closed unmerged.

## What This Changes

Request to add sentence-level streaming from LLM output through telephony TTS to audio playback for classic voice-call providers.

## Best Possible Solution

Add a provider-generic streaming telephony TTS contract and wire classic Voice Call through embedded-agent partial replies, sentence splitting, ordered provider audio streaming, Twilio/Telnyx playback adapters, barge-in/abort handling, fallback behavior, and focused tests.

## Reproduction Assessment

Yes, source-level. Current main awaits one generated text result, one telephony audio buffer, and then chunks the completed buffer to the media stream; no live carrier setup was run.

## Solution Assessment

Yes at the product direction level, but it needs maintainer design before implementation. The maintainable route is a shared streaming telephony speech contract plus provider adapters, not treating the separate realtime voice bypass as the fix.
