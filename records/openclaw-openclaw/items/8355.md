## Summary

Keep open: current main still batches the classic Voice Call STT-to-LLM-to-TTS path through a completed embedded-agent text result and completed telephony TTS buffer before Twilio chunking. Realtime voice and generic ElevenLabs streaming TTS are adjacent but do not implement this provider-generic sentence-level telephony pipeline.

## What This Changes

Request sentence-level streaming for voice calls so LLM output can feed TTS and caller audio before the full response is generated and synthesized.

## Best Possible Solution

Design an additive streaming telephony speech contract and Voice Call wiring that can consume embedded-agent partial replies, split complete sentences, stream provider audio chunks when available, and preserve barge-in, abort, fallback, and ordering behavior across providers.

## Reproduction Assessment

Yes. Source inspection on current main shows the classic Voice Call path awaits a complete embedded-agent result, a complete telephony TTS buffer, and then Twilio chunking; no live carrier call was placed.

## Solution Assessment

Yes. Keeping this open is the best current outcome because realtime voice is a separate bypass, while this request needs a provider-generic streaming telephony speech contract rather than a Twilio- or ElevenLabs-only shortcut.
