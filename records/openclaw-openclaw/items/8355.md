## Summary

Keep open: current main still batches the classic Voice Call STT-to-agent-to-telephony-TTS path, while `realtime.enabled` is a separate full-duplex bypass rather than the requested sentence-level TTS pipeline.

## What This Changes

The issue requests a provider-generic voice-call pipeline that streams LLM output into TTS sentence by sentence and begins telephony audio playback before the full reply is generated.

## Best Possible Solution

Design and land an additive streaming telephony speech contract plus Voice Call wiring that consumes embedded-agent partial replies, splits complete sentences, streams provider audio chunks where available, and preserves barge-in/fallback semantics across providers.

## Reproduction Assessment

Yes. Source inspection against current main shows the classic Voice Call path awaits a complete embedded-agent result, then a complete telephony TTS buffer, then Twilio chunking; no live carrier call was placed.

## Solution Assessment

Yes. Keeping this open is the best current outcome because realtime voice is a separate bypass, and the maintainable fix is a provider-generic telephony streaming contract plus Voice Call integration rather than a Twilio-only shortcut.
