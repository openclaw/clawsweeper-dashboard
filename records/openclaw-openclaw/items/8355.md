## Summary

Keep open: current main and latest release still batch the classic Voice Call STT -> LLM -> TTS path, while realtime voice and normal ElevenLabs streaming TTS are separate paths and prior sentence-level streaming PRs closed unmerged.

## What This Changes

Request sentence-level streaming for classic voice calls so partial LLM output can feed TTS and audio playback before the full response is generated.

## Best Possible Solution

Add an additive provider-generic streaming telephony speech contract and wire classic Voice Call through partial replies, sentence splitting, ordered audio streaming, playback adapters, barge-in/abort handling, buffered fallback, and focused tests.

## Reproduction Assessment

Yes, source-level: current main awaits one embedded-agent result, one telephony audio buffer, and then chunks the completed buffer to the media stream. I did not run a live carrier latency test.

## Solution Assessment

Yes at the product direction level, but not as a one-off automated patch. The maintainable route is a shared streaming telephony speech contract plus classic voice-call adapters, with realtime voice kept as a separate bypass mode.
