## Summary

Keep open: current main and the latest release still batch the classic Voice Call STT-to-agent-to-telephony-TTS path, while realtime voice is a separate full-duplex bypass and the prior streaming implementations were closed unmerged.

## What This Changes

The issue requests sentence-level streaming from LLM output through telephony TTS so voice-call callers hear audio before the full response is generated and synthesized.

## Best Possible Solution

Design and land an additive provider-generic streaming telephony speech contract plus Voice Call wiring that consumes embedded-agent partial replies, splits complete sentences, streams provider audio chunks where available, and preserves barge-in, fallback, ordering, and Twilio/Telnyx behavior.

## Reproduction Assessment

Yes, by source inspection: current main awaits a complete embedded-agent result, then a complete telephony TTS buffer, then Twilio chunking. I did not place a live carrier call.

## Solution Assessment

Yes, keeping this open is the right outcome; realtime voice is a separate bypass, and a maintainable fix should be provider-generic rather than a Twilio-only or ElevenLabs-only shortcut.
