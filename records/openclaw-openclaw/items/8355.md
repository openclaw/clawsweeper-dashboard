## Summary

Keep open: current main still batches the classic Voice Call path from complete agent response to complete telephony TTS buffer before Twilio chunking, while realtime voice is a separate bypass and the previous implementation PR is closed unmerged.

## What This Changes

Request a provider-generic sentence-level streaming pipeline for the classic Voice Call STT -> LLM -> TTS path so audio can begin before full text and full TTS synthesis complete.

## Best Possible Solution

Keep this as the canonical provider-generic classic Voice Call latency feature and design an additive streaming speech contract plus voice-call adapters that preserve barge-in, abort, fallback, ordering, and provider capability boundaries.

## Reproduction Assessment

Yes. Source inspection on current main shows the classic path waits for a complete agent result, a complete telephony TTS Buffer, and then Twilio chunking; I did not place a live carrier call.

## Solution Assessment

Yes. Keeping this open is the right outcome because realtime voice solves a separate full-duplex bypass, while this request needs an additive provider-generic streaming TTS contract rather than a Twilio- or ElevenLabs-only shortcut.
