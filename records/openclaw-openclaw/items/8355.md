## Summary

Keep open. Current main still buffers the classic Voice Call STT to embedded-agent to telephony TTS path before Twilio playback; realtime voice is documented as a separate full-duplex bypass, and the previous sentence-level streaming PRs are closed unmerged.

## What This Changes

Request to add a provider-generic sentence-level streaming TTS pipeline for classic voice calls so playback can begin before the full LLM response and full TTS buffer are complete.

## Best Possible Solution

Add an additive provider-generic streaming telephony TTS contract and wire classic Voice Call through embedded-agent partial replies, sentence splitting, ordered provider audio streaming, Twilio/Telnyx playback adapters, barge-in/abort handling, fallback behavior, and focused tests.

## Reproduction Assessment

Yes, source-level. Current main awaits one embedded-agent result, one telephony audio buffer, and then chunks the completed buffer to the media stream; no live carrier setup was run.

## Solution Assessment

Yes at the product direction level, but not as a one-off automated patch. The maintainable route is a shared streaming telephony speech contract plus voice-call adapters, while realtime voice remains a separate bypass architecture.
