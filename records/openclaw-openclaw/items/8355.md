## Summary

Keep open: current main still waits for a complete agent reply and a complete telephony TTS buffer before Twilio chunking; realtime voice and generic ElevenLabs stream synthesis are adjacent work, but they do not implement the requested classic sentence-level telephony pipeline.

## What This Changes

Add provider-generic sentence-level streaming to the classic Voice Call STT -> LLM -> telephony TTS pipeline to reduce first-audio latency.

## Best Possible Solution

Design and land an additive provider-generic streaming telephony speech contract plus Voice Call wiring that consumes partial agent replies, splits complete sentences, streams provider audio chunks where available, and preserves barge-in, fallback, and ordering across Twilio and Telnyx.

## Reproduction Assessment

Yes, by source inspection: the inbound path awaits a complete generated response, then manager.speak, then Twilio awaits a complete telephony TTS Buffer before chunking. I did not place a live carrier call.

## Solution Assessment

Yes. Keeping this open is the right current outcome because realtime voice is a separate bypass, and the maintainable fix is a provider-generic telephony streaming contract plus Voice Call integration rather than a Twilio-only shortcut.
