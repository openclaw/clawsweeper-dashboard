## Summary

Keep open: current main still batches the classic Voice Call STT-to-LLM-to-TTS path, and the realtime voice mode plus generic speech streaming support do not implement the requested provider-generic sentence-level telephony pipeline.

## What This Changes

The issue requests a provider-generic streaming voice-call pipeline that starts telephony TTS from sentence-level LLM output before the full response and full audio buffer are complete.

## Best Possible Solution

Design an additive streaming telephony speech contract and Voice Call wiring that can consume embedded-agent partial replies, split complete sentences, stream provider audio chunks when available, and preserve barge-in, abort, fallback, and ordering behavior across providers.

## Reproduction Assessment

Yes. Source inspection against current main shows the classic Voice Call path awaits a complete embedded-agent result, a complete telephony TTS buffer, and then Twilio chunking; no live carrier call was placed.

## Solution Assessment

Yes. Keeping this open is the best current outcome because realtime voice is a separate bypass and the existing generic speech stream does not cover the telephony pipeline; the maintainable fix is provider-generic contract and Voice Call wiring, not a Twilio-only shortcut.
