## Summary

Keep open: current main still batches the classic Voice Call path through a complete agent response and complete telephony TTS buffer before Twilio chunking; realtime voice is a separate full-duplex bypass, and the prior implementation attempt at https://github.com/openclaw/openclaw/pull/42187 was closed unmerged.

## What This Changes

Request to add a sentence-level streaming LLM-to-TTS-to-audio pipeline for classic Voice Call responses.

## Best Possible Solution

Design an additive streaming telephony speech contract plus Voice Call wiring that uses embedded-agent partial replies, sentence splitting, provider audio chunk streaming where available, and provider adapters with regression coverage for ordering, abort, fallback, and barge-in.

## Reproduction Assessment

Yes. Source inspection on current main shows the classic Voice Call path awaits a complete agent result, a complete telephony TTS buffer, and then Twilio chunking; I did not place a live carrier call.

## Solution Assessment

Yes. Keeping this open is the best current outcome because realtime voice is a separate bypass, while this request needs a provider-generic streaming TTS/speech contract rather than a Twilio- or ElevenLabs-only shortcut.
