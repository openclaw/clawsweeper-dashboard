## Summary

Keep open. Current main and v2026.6.8 still batch the classic Voice Call STT -> agent -> telephony TTS path; realtime voice and ordinary ElevenLabs streaming do not implement the requested provider-generic sentence-level telephony pipeline.

## What This Changes

Requests a provider-generic sentence-level LLM-to-TTS-to-audio streaming pipeline for classic voice-call flows.

## Best Possible Solution

Keep this as the canonical classic voice-call latency feature and implement it with an additive streaming telephony/speech contract, sentence segmentation, transport playback adapters, and ordering, abort, fallback, and provider compatibility tests.

## Reproduction Assessment

Yes, at source level: current main and v2026.6.8 still await a completed embedded-agent result and completed telephony audio Buffer before Twilio sends audio chunks. I did not run a live Twilio or Telnyx call to measure the exact latency numbers.

## Solution Assessment

Yes, keeping this open as canonical remaining work is the best triage outcome. The maintainable fix is provider-generic streaming telephony plus classic voice-call wiring, not treating realtime voice or ordinary non-telephony ElevenLabs streaming as the fix.
