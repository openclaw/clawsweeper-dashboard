## Summary

Keep open. Current main has fixed several high-cost future Dreaming paths, including `dreaming.model`, accepted narrative-session deletion, orphan/stale Dreaming scrub, plugin-owned session cleanup scope, and a hard cap on detached narrative concurrency, but the broader operator cleanup request is not complete: there is still no Dreaming-specific purge/retention knob, no `sessions purge --kind dreaming`, and no shipped Dreaming filter in the session picker/list surface.

## What This Changes

Request that Dreaming avoid spawning and retaining large numbers of background sessions, support cheaper model overrides, bound burst fan-out, hide/filter Dreaming artifacts, and provide a bulk purge path.

## Best Possible Solution

Keep the shipped future-run fixes, then explicitly scope and land the remaining operator-facing path: Dreaming-aware retention or purge for legacy referenced sessions plus a session-picker/list filter, or split those leftovers into narrower canonical issues.

## Reproduction Assessment

Yes for the remaining gaps: current source inspection shows no `sessions purge --kind dreaming`, no Dreaming-specific retention knob, and no shipped Dreaming session picker/list filter. A live replay of the original 184-session deployment is not needed to verify those missing operator surfaces.

## Solution Assessment

No, current main is a useful partial solution but not the complete best solution for the full report. The narrow maintainable path needs maintainer scoping across memory-core retention, generic session purge semantics, and Control UI filtering.
