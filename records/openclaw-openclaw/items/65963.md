## Summary

Keep open. Current main has already addressed important parts of the report, including `dreaming.model`, model routing into Dream Diary subagents, best-effort narrative session deletion, orphan cleanup, and a fixed detached-narrative concurrency cap, but the remaining operator request is still live: no Dreaming retention knob, no `sessions purge --kind dreaming`, and no shipped Dreaming/session-kind filter in the Control UI/session list.

## What This Changes

The issue requests controls to reduce Dreaming session sprawl: automatic cleanup or purgeability, a cheaper model override, burst control, and hiding or grouping Dreaming sessions in session lists and the picker.

## Best Possible Solution

Keep the shipped Dreaming safeguards, resolve the focused cleanup and UI-filter follow-ups on their merits, and make an explicit maintainer decision on whether generic purge, Dreaming retention, and configurable burst controls should be split into separate canonical issues.

## Reproduction Assessment

A live replay of the original 184-session deployment is not required to verify those missing operator surfaces.

## Solution Assessment

No complete solution is on main yet. The maintainable path is to finish or replace the focused memory-core cleanup and UI filtering follow-ups, then decide whether generic purge, retention, and burst controls belong in core, memory-core config, or separate issues.
