## Summary

Keep open. Current main has fixed the highest-cost pieces of the Dreaming report for future runs, including `dreaming.model`, Dream Diary session deletion, stale/orphan artifact scrubbing, and a hard-coded detached narrative concurrency cap, but it still does not provide the requested operator-facing purge/retention/filter path for legacy referenced Dreaming sessions and transcripts.

## What This Changes

The issue requests controls for memory-core Dreaming session sprawl: automatic cleanup or purge, a Dreaming model override, burst control, session picker filtering, and a `sessions purge --kind dreaming` CLI path.

## Best Possible Solution

Keep the shipped model override, future-session deletion, scrubber, and bounded queue, then explicitly scope the remaining UX and operator cleanup path: either add a Dreaming-aware retention/purge flow for referenced legacy narrative sessions plus picker/list filtering, or split those leftovers into narrower canonical issues before closing this umbrella report.

## Reproduction Assessment

Yes for the remaining gaps: current source inspection shows no `sessions purge --kind dreaming`, no Dreaming-specific retention knob, and no Dreaming-key session picker filter. Replaying the original 184-session deployment is not necessary to verify those missing operator surfaces.

## Solution Assessment

No, current main is a useful partial solution but not the complete best solution for the full issue. The remaining fix needs maintainer scoping because generic purge semantics, plugin-owned retention, and UI filtering each have different ownership and user-facing behavior.
