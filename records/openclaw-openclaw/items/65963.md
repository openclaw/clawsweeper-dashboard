## Summary

Current main and the shipped v2026.5.12 release cover the central dreaming sprawl and cost problem: Dream Diary subagent sessions are deleted/scrubbed, `dreaming.model` is supported, the default cadence is daily, and detached narrative fan-out is capped. The follow-up deployment comment was useful confirmation that cadence reduction helped, and the remaining generic purge/session-picker ideas are no longer needed to solve the reported dreaming artifacts.

## What This Changes

The issue asks for dreaming session cleanup, a cheaper dreaming model override, burst control, session-list filtering, and bulk purge ergonomics.

## Best Possible Solution

Keep the shipped Dreaming-specific cleanup/model/concurrency behavior and open a new focused request only if a generalized session purge or picker policy is still needed.

## Reproduction Assessment

No high-confidence current-main reproduction remains. The original v2026.4.11/v2026.4.12 data shows the old problem, but current source and tests show accepted Dream Diary sessions are deleted, stale artifacts are scrubbed, model override is supported, and detached narrative fan-out is capped.

## Solution Assessment

Yes. The shipped combination is narrower than adding a broad purge CLI or picker policy for internal Dreaming sessions, because Dreaming now avoids leaving those sessions behind in the first place.
