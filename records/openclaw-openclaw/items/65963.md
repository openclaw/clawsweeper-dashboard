## Summary

Keep open. Current main has addressed the highest-cost parts of the report, including `dreaming.model`, future Dream Diary session deletion, stale/orphan artifact scrubbing, and bounded detached narrative fan-out, but it still lacks a terminal operator-facing cleanup path for referenced legacy dreaming sessions and transcripts, a dreaming-specific retention knob, a `sessions purge --kind dreaming` command, or a dreaming-key session picker/list filter.

## What This Changes

The issue requests reducing memory-core dreaming session sprawl with automatic cleanup, a cheaper model override, concurrency control, session filtering, and bulk purge tooling.

## Best Possible Solution

Keep the shipped model override, future-session deletion, scrubber, and bounded detached queue, then scope the remaining cleanup UX explicitly: either add a memory-core retention/purge path for `dreaming-narrative-` store entries and transcripts plus a session picker/list filter, or split those leftovers into narrower canonical issues before closing this umbrella report.

## Reproduction Assessment

Yes for the remaining gaps: current source inspection reproduces the absence of purge-by-kind, dreaming-specific retention, and dreaming-key UI filtering. Recreating the original 184-session deployment is not necessary to see that referenced legacy dreaming sessions with transcripts still lack a first-class cleanup path.

## Solution Assessment

No, current main is not yet the complete best solution for the full report. The partial fixes are valuable, but the safest remaining path needs maintainer scoping because it crosses memory-core artifact policy, generic sessions CLI semantics, and Control UI session filtering.
