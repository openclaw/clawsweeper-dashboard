Keep this PR open. Current `main` still has the hot-mode warn-and-return behavior, but the PR is not merge-ready because the new public config field is not carried through the generated schema/help/docs surfaces and the user-facing behavior lacks changelog coverage.

## What This Changes

The PR adds `gateway.reload.fallbackToRestart`, defaults it to `true`, routes restart-required hot-mode config reloads to the restart path unless disabled, and adds gateway reload schema/type/test coverage.

## Best Possible Solution

Keep the PR as an implementation candidate, decide whether hot-mode restart fallback should default on or require opt-in, then update the full public config contract: runtime types, zod schema, generated base schema, help/labels, Gateway docs, changelog, and focused reload tests.
