## Summary

Keep open: current main still has the hot-mode warning-and-return path this PR is trying to change, but the branch is not ready to merge because it changes the documented hot-mode default, leaves the new public config key undocumented in metadata/docs, and has no contributor-supplied real gateway reload proof.

## What This Changes

The PR adds a `gateway.reload.fallbackToRestart` option and changes hot-mode restart-required file-watcher reloads to queue a restart by default, with focused reload tests.

## Best Possible Solution

Land only a maintainer-approved version that preserves or intentionally revises the hot-mode contract, aligns runtime/schema/help/labels/docs/tests, and includes redacted real gateway reload proof.

## Reproduction Assessment

Yes. Source inspection on current main shows `startGatewayConfigReloader` warns and returns for restart-required plans in `hot` mode; the disputed part is whether that documented behavior should change.

## Solution Assessment

No for the current branch as-is. The runtime idea is narrow, but the patch changes the default hot-mode contract, omits public config metadata/docs, and lacks real behavior proof.
