## Summary

Keep open: this PR targets a real CLI watchdog false positive, but the submitted patch is too broad for current main, misses an empty-stdin edge case, and still lacks real behavior proof.

## What This Changes

The branch changes `src/agents/cli-runner/execute.ts` to skip `noOutputTimeoutMs` when `stdinPayload` is truthy and to make no-output timeout messages tolerate an optional timeout.

## Best Possible Solution

Land a rebased or replacement fix that disables the no-output watchdog only for normal closed-stdin child spawns, preserves Claude live-session timers, handles empty stdin, fixes fallback wording, and adds focused regression coverage plus real CLI proof.

## Reproduction Assessment

Yes, by source inspection. Current main can pass both `input: stdinPayload` and a numeric `noOutputTimeoutMs` to a normal child spawn, while the child adapter closes stdin whenever input is present.

## Solution Assessment

No, not as submitted. The fix should be scoped to the closed-stdin child spawn path rather than the shared pre-routing timeout value, and it should guard on stdin presence instead of payload truthiness.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Scope watchdog disabling to closed-stdin child runs:** `src/agents/cli-runner/execute.ts:169-175`
  - body: This guard is computed before current-main Claude live-session routing, but live sessions keep stdin open and require a numeric turn watchdog. Apply the skip only where the normal child spawn will close stdin after writing input.
  - confidence: 0.88
- **[P2] Check stdin presence instead of payload truthiness:** `src/agents/cli-runner/execute.ts:169-175`
  - body: `stdinPayload` is `stdin ?? ""`, so an empty stdin-mode prompt still closes stdin but leaves this guard false. Use raw stdin presence or an explicit closed-stdin state.
  - confidence: 0.9
- **[P3] Avoid zero-second watchdog timeout text:** `src/agents/cli-runner/execute.ts:219-225`
  - body: When the watchdog is disabled, the `?? 0` fallback can surface a misleading `0s` no-output timeout message if that result reaches this branch. Use separate text for an undefined watchdog duration.
  - confidence: 0.78
