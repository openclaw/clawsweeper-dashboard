## Summary

Keep open. The PR addresses a real current-main CLI runner bug, but the submitted patch still has blocking correctness issues: it disables the shared watchdog before Claude live-session routing, misses empty stdin, and can emit a misleading 0s timeout.

## What This Changes

This PR changes `src/agents/cli-runner/execute.ts` to omit the CLI no-output watchdog when a stdin payload is present and to make watchdog timeout messages tolerate an optional timeout.

## Best Possible Solution

Land a rebased or replacement fix that disables the no-output watchdog only for normal closed-stdin child spawns, preserves Claude live-session timers, handles empty stdin and timeout text, and adds focused regression coverage.

## Reproduction Assessment

Yes. Source inspection shows current main can pass a numeric no-output watchdog and closed child stdin together, so a silent stdin-mode child can still be terminated by the watchdog even though stdin has already been closed.

## Solution Assessment

No, not as submitted. The direction is right, but the skip needs to be scoped to the normal closed-stdin child spawn path and must not remove the numeric watchdog from Claude live sessions.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Scope the watchdog skip to closed-stdin child runs:** `src/agents/cli-runner/execute.ts:169-175`
  - body: This value is computed before Claude live-session routing, but Claude live sessions use stdin mode with an open pipe and require a numeric watchdog for turn timers. Setting the shared value to `undefined` for stdin backends can disable or break live-session stall protection, so apply the skip only to the normal child spawn path where stdin is actually closed after writing.
  - confidence: 0.88
- **[P2] Use stdin presence for the watchdog guard:** `src/agents/cli-runner/execute.ts:169-175`
  - body: `stdinPayload` is `stdin ?? ""`, so an empty stdin-mode prompt still closes stdin but leaves this truthiness guard false. Guard on raw stdin presence or an explicit closed-stdin state so empty stdin gets the same watchdog behavior as non-empty stdin.
  - confidence: 0.9
- **[P3] Avoid reporting a 0s no-output timeout:** `src/agents/cli-runner/execute.ts:219-225`
  - body: When the watchdog is disabled, `noOutputTimeoutMs ?? 0` can produce a user-facing `0s` timeout if an unexpected no-output result reaches this branch. Build separate enabled and disabled watchdog messages instead of inventing a zero-second duration.
  - confidence: 0.78
