## Summary

## What This Changes

The PR changes `src/agents/cli-runner/execute.ts` so CLI spawns with a truthy stdin payload omit `noOutputTimeoutMs` and adjusts no-output timeout messages for an optional timeout.

## Best Possible Solution

Rebase and implement the skip only at the normal closed-stdin child-spawn boundary, preserve numeric live-session turn watchdogs, handle empty stdin, and attach real CLI proof.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: current main passes a numeric no-output timeout to `supervisor.spawn` while the child adapter closes stdin whenever `input` is defined. I did not run a live long-silent CLI command in this read-only review.

## Solution Assessment

No, not as submitted. The safer fix is to disable the supervisor watchdog only for the normal closed-stdin spawn path, keep Claude live-session no-output timers numeric, use stdin/input presence rather than payload truthiness, and avoid `0s` fallback text.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Keep live-session watchdogs numeric:** `src/agents/cli-runner/execute.ts:169-175`
  - body: Current main passes this same value into `runClaudeLiveSessionTurn`, where `noOutputTimeoutMs` is a required number and the Claude process keeps stdin open via `stdinMode: "pipe-open"`. Rebase the fix so only the normal closed-stdin `supervisor.spawn` path disables the supervisor watchdog; otherwise live turns lose or fail their per-turn stall protection.
  - confidence: 0.9
- **[P2] Check stdin presence instead of payload truthiness:** `src/agents/cli-runner/execute.ts:169-175`
  - body: `stdinPayload` is derived from `stdin ?? ""`, so an empty stdin-mode prompt is falsy even though the child receives defined input and closes stdin. Use raw stdin/input presence or an explicit closed-stdin condition so empty stdin gets the same behavior.
  - confidence: 0.9
- **[P3] Avoid zero-second watchdog timeout text:** `src/agents/cli-runner/execute.ts:219-225`
  - body: If a no-output result reaches this branch while the timeout value is unknown, the fallback reports `0s`, which is misleading diagnostic text. Build separate wording for an unknown watchdog duration instead of formatting it as zero seconds.
  - confidence: 0.78
