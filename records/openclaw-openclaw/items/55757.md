## Summary

Keep open: the PR targets a valid CLI runner availability bug, but it is not merge-ready because the branch is conflicting against the current live-session watchdog boundary, has two concrete patch defects, and still lacks after-fix real behavior proof.

## What This Changes

The branch changes `src/agents/cli-runner/execute.ts` to skip `noOutputTimeoutMs` when `stdinPayload` is truthy and to tolerate optional watchdog values in timeout messages.

## Best Possible Solution

Rebase against current main and disable the supervisor no-output watchdog only for normal closed-stdin child spawns, while preserving numeric Claude live-session watchdogs and overall timeouts.

## Reproduction Assessment

Yes, at source level. Current main passes defined closed stdin and a numeric no-output watchdog to the child supervisor, and the supervisor cancels silent children when that watchdog fires.

## Solution Assessment

No. The intended fix is plausible, but this branch should be reworked so only the normal closed-stdin child-spawn path disables the supervisor watchdog while Claude live-session turn watchdogs remain numeric.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Keep live-session watchdogs numeric:** `src/agents/cli-runner/execute.ts:169-175`
  - body: On current main the same `noOutputTimeoutMs` value is also passed into `runClaudeLiveSessionTurn`, whose contract requires a number and keeps stdin open. Rebase this by disabling the supervisor watchdog only on the normal closed-stdin child spawn, not by making the shared live-session value optional.
  - confidence: 0.86
- **[P2] Check stdin presence, not payload truthiness:** `src/agents/cli-runner/execute.ts:169-175`
  - body: `stdinPayload` is `stdin ?? ""`, so an stdin-mode backend with an empty prompt still closes stdin in the child adapter but keeps the no-output watchdog enabled. Use `stdin != null` or an explicit closed-stdin boolean for the normal child path.
  - confidence: 0.82
- **[P3] Avoid zero-second watchdog messages:** `src/agents/cli-runner/execute.ts:219-225`
  - body: If a no-output result reaches this branch while the watchdog value is undefined, the new fallback reports `0s`, which points users at a bogus timeout. Build a generic message for the undefined case instead of formatting zero seconds.
  - confidence: 0.78
