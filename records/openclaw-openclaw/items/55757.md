## Summary

Keep open: current main still has a source-reproducible closed-stdin watchdog problem, but this PR should not merge as-is because the shared timeout value now feeds Claude live sessions, the guard misses empty stdin, and contributor real behavior proof is still missing.

## What This Changes

The PR changes `src/agents/cli-runner/execute.ts` to make `noOutputTimeoutMs` undefined when `stdinPayload` is truthy and adjusts timeout message formatting for optional values.

## Best Possible Solution

Scope watchdog disabling to normal closed-stdin child spawns, keep Claude live-session watchdogs numeric, handle empty stdin, add regression coverage, and require redacted real CLI proof.

## Reproduction Assessment

Yes. Current main computes a numeric no-output watchdog, passes it with defined `input` to the child supervisor spawn, and the child adapter closes stdin whenever `input` is defined; I did not run a live long-silent CLI command in this read-only review.

## Solution Assessment

No, not as submitted. The safer fix is to move the watchdog skip to the non-live closed-stdin child-spawn boundary, use raw stdin presence rather than payload truthiness, and avoid formatting unknown watchdog duration as zero seconds.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Keep live-session watchdogs numeric:** `src/agents/cli-runner/execute.ts:169-175`
  - body: On current main this shared `noOutputTimeoutMs` value is also passed to `runClaudeLiveSessionTurn`, while Claude live sessions keep stdin open and use that number for per-turn no-output timers. Rebase the fix so only the normal closed-stdin child spawn disables the supervisor watchdog.
  - confidence: 0.9
- **[P2] Check stdin presence instead of payload truthiness:** `src/agents/cli-runner/execute.ts:169-175`
  - body: `stdinPayload` is derived from `stdin ?? ""`, so an empty stdin-mode prompt is falsy even though the child adapter receives defined input and closes stdin. Use raw stdin presence or an explicit closed-stdin condition so empty stdin gets the same behavior.
  - confidence: 0.9
- **[P3] Avoid zero-second watchdog timeout text:** `src/agents/cli-runner/execute.ts:219-225`
  - body: If a no-output result reaches this branch while the timeout value is unknown, the fallback reports `0s`, which is misleading diagnostic text. Build separate wording for an unknown watchdog duration instead of formatting it as zero seconds.
  - confidence: 0.78
