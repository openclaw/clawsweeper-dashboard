## Summary

Keep open: the reported CLI runner bug is real on current main, but this PR needs changes before merge because the watchdog skip is too broad/implicit and the timeout messaging can become misleading.

## What This Changes

The PR makes `executePreparedCliRun` skip `noOutputTimeoutMs` when `stdinPayload` is truthy and adjusts no-output timeout messages for an optional watchdog value.

## Best Possible Solution

Land a rebased or replacement fix that omits the supervisor no-output watchdog only for normal closed-stdin child spawns, preserves Claude live-session watchdogs, handles empty stdin, and adds focused tests plus the required changelog entry.

## Reproduction Assessment

Yes. Source inspection shows current main can pass both closed child stdin and `noOutputTimeoutMs` to `supervisor.spawn`; a silent stdin-mode child can hit the no-output timer even though stdin has already been closed.

## Solution Assessment

No, not as submitted. The direction is appropriate, but the guard should be based on explicit closed-stdin child state, must preserve numeric watchdogs for Claude live sessions, and should avoid synthetic `0s` timeout text.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Scope the watchdog skip to closed child spawns:** `src/agents/cli-runner/execute.ts:169-175`
  - body: On current main, the watchdog value is computed before the Claude live-session branch and passed into `runClaudeLiveSessionTurn`, whose API and timers require a number. If this change is rebased as-is, stdin/jsonl Claude live sessions can receive `undefined` and lose or break stall protection; keep the numeric value for live sessions and omit it only on the normal child spawn where stdin is actually closed.
  - confidence: 0.86
- **[P2] Use stdin presence for the watchdog guard:** `src/agents/cli-runner/execute.ts:169-175`
  - body: `stdinPayload` is `stdin ?? ""`, so an empty stdin-mode prompt still closes stdin but leaves this truthiness guard false. Guard on raw stdin presence or an explicit closed-stdin state so empty stdin gets the same watchdog behavior as non-empty stdin.
  - confidence: 0.9
- **[P3] Avoid reporting a zero-second watchdog timeout:** `src/agents/cli-runner/execute.ts:219-225`
  - body: When the watchdog is disabled, `noOutputTimeoutMs ?? 0` can produce a user-facing `0s` timeout if an unexpected no-output result reaches this branch. Build a generic message for disabled or unknown watchdog duration instead of inventing a zero-second duration.
  - confidence: 0.78
