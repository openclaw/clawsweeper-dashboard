## Summary

Keep open: this PR targets a real source-level CLI availability problem, but it is merge-conflicting, lacks real behavior proof, and needs a current-main watchdog-boundary rework before it is safe to merge.

## What This Changes

The PR changes `src/agents/cli-runner/execute.ts` to skip the supervisor no-output timeout when a stdin payload is present and to handle optional no-output timeout diagnostics.

## Best Possible Solution

Rebase against current main and disable the supervisor no-output watchdog only for normal child spawns with defined closed stdin, while preserving numeric Claude live-session per-turn watchdogs and proving both long-silent completion and overall timeout behavior.

## Reproduction Assessment

Yes, at source level. Current main passes a numeric `noOutputTimeoutMs` and defined `input` to the child supervisor, while the child adapter closes stdin for any defined input; I did not run a live long-silent CLI process in this read-only review.

## Solution Assessment

No. The direction is plausible, but the patch makes the shared watchdog value optional too early and uses payload truthiness; the safer fix is to keep a numeric live-session watchdog and pass `undefined` only to normal closed-stdin child spawns.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Keep live-session watchdogs numeric:** `src/agents/cli-runner/execute.ts:169-175`
  - body: On current main, the same `noOutputTimeoutMs` value is also passed into `runClaudeLiveSessionTurn`, whose contract requires a number and whose child process keeps stdin open. Rebase this by disabling the supervisor watchdog only on the normal closed-stdin child spawn, not by making the shared live-session value optional.
  - confidence: 0.86
- **[P2] Check stdin presence, not payload truthiness:** `src/agents/cli-runner/execute.ts:169-175`
  - body: `stdinPayload` is `stdin ?? ""`, so an stdin-mode backend with an empty prompt still closes stdin in the child adapter but keeps the no-output watchdog enabled. Use `stdin != null` or an explicit closed-stdin boolean for the normal child path.
  - confidence: 0.82
- **[P3] Avoid zero-second watchdog messages:** `src/agents/cli-runner/execute.ts:219-225`
  - body: If a no-output result reaches this branch while the watchdog value is undefined, the new fallback reports `0s`, which points users at a bogus timeout. Build a generic message for the undefined case instead of formatting zero seconds.
  - confidence: 0.78
