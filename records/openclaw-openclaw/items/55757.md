## Summary

Keep open: current main still has the closed-stdin/no-output watchdog path, but this PR is not ready to merge because the submitted guard is too broad for current Claude live sessions, misses empty stdin, and lacks real after-fix CLI proof.

## What This Changes

The PR changes `src/agents/cli-runner/execute.ts` to skip `noOutputTimeoutMs` when `stdinPayload` is truthy and makes watchdog timeout messages tolerate an undefined timeout.

## Best Possible Solution

Land a rebased or replacement fix that disables the no-output watchdog only for normal closed-stdin child runs, preserves Claude live-session watchdogs, handles empty stdin, and adds focused regression coverage plus real CLI proof.

## Reproduction Assessment

Yes, by source inspection. Current main can pass a numeric no-output watchdog with `input: stdinPayload`, and the child adapter closes stdin whenever input is defined.

## Solution Assessment

No, not as submitted. The fix should be scoped to the normal child closed-stdin path instead of the shared pre-routing timeout value, and it should test stdin presence or an explicit closed-stdin state rather than payload truthiness.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Keep live-session watchdogs numeric:** `src/agents/cli-runner/execute.ts:169-175`
  - body: This guard makes `noOutputTimeoutMs` optional before current-main Claude live-session routing, but live sessions keep stdin open and require a numeric turn watchdog. Move the skip to the normal closed-stdin child spawn path so live sessions keep their protection after rebase.
  - confidence: 0.88
- **[P2] Check stdin presence instead of payload truthiness:** `src/agents/cli-runner/execute.ts:169-175`
  - body: `stdinPayload` is derived from `stdin ?? ""`, so an empty stdin-mode prompt still closes stdin but leaves this guard false. Use raw stdin presence or an explicit closed-stdin child condition instead of testing the payload string.
  - confidence: 0.9
- **[P3] Avoid zero-second watchdog timeout text:** `src/agents/cli-runner/execute.ts:219-225`
  - body: If a no-output result reaches this branch while the timeout value is undefined, the new fallback reports `0s`, which is misleading diagnostic text. Use separate wording for an unknown watchdog duration.
  - confidence: 0.78
