## Summary

Keep open: the underlying closed-stdin watchdog bug is still source-reproducible on current main, but this PR is not mergeable as submitted because the watchdog skip is too broad for current Claude live sessions, misses empty stdin-mode prompts, and lacks real CLI proof.

## What This Changes

The PR updates `src/agents/cli-runner/execute.ts` to skip the CLI no-output watchdog when `stdinPayload` is truthy and to tolerate an undefined watchdog timeout in error text.

## Best Possible Solution

Land a rebased or replacement fix that disables the no-output watchdog only for closed-stdin normal child spawns, preserves Claude live-session watchdogs, handles empty stdin, keeps the overall timeout, and adds focused regression coverage plus real CLI proof.

## Reproduction Assessment

Yes, by source inspection rather than a live run. Current main sends both a numeric `noOutputTimeoutMs` and `input: stdinPayload` into the child spawn path, and the child adapter closes stdin whenever input is defined.

## Solution Assessment

No, not as submitted. The fix should be scoped to closed-stdin normal child runs and should use stdin presence or explicit closed-stdin state instead of payload truthiness.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Scope watchdog skip to closed-stdin child runs:** `src/agents/cli-runner/execute.ts:169-175`
  - body: This guard is computed before current-main Claude live-session routing, but live sessions keep stdin open and require a numeric turn watchdog. Move the skip into the normal child spawn path or compute a child-only timeout so Claude live sessions keep their watchdog.
  - confidence: 0.88
- **[P2] Check stdin presence instead of payload truthiness:** `src/agents/cli-runner/execute.ts:169-175`
  - body: `stdinPayload` is `stdin ?? ""`, so an empty stdin-mode prompt still closes stdin but leaves this guard false. Use raw stdin presence or an explicit closed-stdin state instead of testing the payload string.
  - confidence: 0.9
- **[P3] Avoid zero-second watchdog timeout text:** `src/agents/cli-runner/execute.ts:219-225`
  - body: When the watchdog is disabled, the `?? 0` fallback can surface a misleading `0s` no-output timeout message if this result path is reached. Use separate text for an undefined watchdog duration.
  - confidence: 0.78
