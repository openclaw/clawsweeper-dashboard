## Summary

Keep open: the reported watchdog bug is source-reproducible and the direction is useful, but this PR is not merge-ready because it was written before current Claude live-session routing, still has concrete watchdog-scoping bugs, and lacks real CLI behavior proof.

## What This Changes

The PR changes `src/agents/cli-runner/execute.ts` so CLI runs with a truthy stdin payload skip `noOutputTimeoutMs`, and adjusts no-output timeout text for an optional timeout.

## Best Possible Solution

Land a rebased or replacement fix that scopes watchdog disabling to closed-stdin normal child spawns, preserves Claude live-session timers, handles empty stdin, fixes fallback wording, and adds focused regression plus real CLI proof.

## Reproduction Assessment

Yes. Source inspection on current main shows a normal CLI child spawn can receive both `input: stdinPayload` and a numeric `noOutputTimeoutMs`, while the child adapter closes stdin whenever input is present, so a silent closed-stdin child can still hit the no-output watchdog.

## Solution Assessment

No, not as submitted. The fix should disable the no-output watchdog only for normal closed-stdin child spawns, keep Claude live-session turn timers numeric, guard on stdin presence rather than payload truthiness, and avoid zero-second fallback text.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Scope watchdog disabling to closed stdin:** `src/agents/cli-runner/execute.ts:169-175`
  - body: This guard is computed before current-main Claude live-session routing, but live sessions also use stdin and keep it open with `stdinMode: "pipe-open"`; carrying `undefined` into that path would remove or break the numeric turn timer. Apply the skip only where the normal child spawn will close stdin after writing input.
  - confidence: 0.88
- **[P2] Guard on stdin presence, not payload truthiness:** `src/agents/cli-runner/execute.ts:169-175`
  - body: `stdinPayload` is `stdin ?? ""`, so an empty stdin-mode prompt still closes stdin but leaves this guard false. That preserves the original false-positive watchdog for the empty-prompt case; use raw stdin presence or an explicit closed-stdin state.
  - confidence: 0.9
- **[P3] Avoid zero-second no-output timeout messages:** `src/agents/cli-runner/execute.ts:219-225`
  - body: When the watchdog is disabled, the added `?? 0` fallback can surface a `0s` timeout notice if a no-output result reaches this branch. Build a separate undefined-timeout message instead of inventing a zero-second duration.
  - confidence: 0.78
