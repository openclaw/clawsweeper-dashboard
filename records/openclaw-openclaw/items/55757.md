## Summary

The PR targets a real current-main CLI runner problem, but the submitted approach is not ready because it makes the watchdog decision from a truthy payload value and would not survive current live-session watchdog requirements after rebase.

## What This Changes

The PR changes `src/agents/cli-runner/execute.ts` to skip `noOutputTimeoutMs` when a CLI runner stdin payload is present and to tolerate an optional watchdog value in timeout messages.

## Best Possible Solution

Land a rebased or replacement fix that disables the supervisor no-output watchdog only for normal closed-stdin child spawns, preserves live-session watchdog timers, handles empty stdin explicitly, avoids misleading `0s` messages, and adds focused regression coverage plus the required changelog entry.

## Reproduction Assessment

Yes. Source inspection shows current main can pass both closed child stdin and `noOutputTimeoutMs` to `supervisor.spawn`; a stdin-mode backend that stays silent longer than the watchdog will hit the no-output timer even though stdin has already been closed.

## Solution Assessment

No, not as submitted. The direction is narrow and appropriate, but the skip should be based on explicit closed-stdin child state rather than truthy payload and must not remove the numeric watchdog used by Claude live sessions.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Scope the watchdog skip to closed-stdin child runs:** `src/agents/cli-runner/execute.ts:169-175`
  - body: This value is computed before live-session routing on current main, but Claude live sessions use stdin mode with an open pipe and pass a numeric watchdog into `runClaudeLiveSessionTurn`. Making the shared value `undefined` for stdin backends can disable or break live-session stall protection, so apply the skip only to the normal child spawn path where stdin is actually closed after writing.
  - confidence: 0.86
- **[P2] Use stdin presence for the watchdog guard:** `src/agents/cli-runner/execute.ts:169-175`
  - body: `stdinPayload` is `stdin ?? ""`, so an empty stdin-mode prompt still closes stdin but leaves this truthiness guard false. Guard on raw stdin presence or an explicit closed-stdin state so empty stdin gets the same watchdog behavior as non-empty stdin.
  - confidence: 0.9
- **[P3] Avoid reporting a 0s no-output timeout:** `src/agents/cli-runner/execute.ts:219-225`
  - body: When the watchdog is disabled, `noOutputTimeoutMs ?? 0` can produce a user-facing `0s` timeout if an unexpected no-output result reaches this branch. Build separate enabled and disabled watchdog messages instead of inventing a zero-second duration.
  - confidence: 0.78
