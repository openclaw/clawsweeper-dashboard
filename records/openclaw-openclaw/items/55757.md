## Summary

Keep open: the closed-stdin watchdog problem is source-reproducible, but the submitted branch is stale against current main, would drop Claude live-session no-output protection if rebased directly, misses the empty-stdin case, and has no real CLI behavior proof.

## What This Changes

The branch changes `src/agents/cli-runner/execute.ts` to make `noOutputTimeoutMs` optional when `stdinPayload` is truthy and updates no-output timeout message formatting.

## Best Possible Solution

Rework the fix on current main so only normal closed-stdin child spawns skip the supervisor no-output watchdog, Claude live sessions keep numeric per-turn watchdogs, empty stdin is covered, and real CLI proof is supplied.

## Reproduction Assessment

Yes. Current main passes a numeric no-output timeout to `supervisor.spawn` with defined `input`, and the child adapter closes stdin whenever input is defined; I did not run a live long-silent CLI command in this read-only review.

## Solution Assessment

No, not as submitted. The safer solution is to move the watchdog skip to the normal closed-stdin child-spawn boundary, check stdin/input presence rather than payload truthiness, and avoid formatting unknown timeout duration as `0s`.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Keep live-session watchdogs numeric:** `src/agents/cli-runner/execute.ts:169-175`
  - body: On current main this shared timeout value is also passed to `runClaudeLiveSessionTurn`, while Claude live sessions keep stdin open and require a numeric per-turn no-output timer. Rework the change so only normal closed-stdin child spawns disable the supervisor watchdog.
  - confidence: 0.9
- **[P2] Check stdin presence instead of payload truthiness:** `src/agents/cli-runner/execute.ts:169-175`
  - body: `stdinPayload` is derived from `stdin ?? ""`, so an empty stdin-mode prompt is falsy even though the child receives defined input and closes stdin. Use raw stdin/input presence or an explicit closed-stdin condition so empty stdin gets the same behavior.
  - confidence: 0.9
- **[P3] Avoid zero-second watchdog timeout text:** `src/agents/cli-runner/execute.ts:219-225`
  - body: If a no-output result reaches this branch while the timeout value is unknown, the fallback reports `0s`, which is misleading diagnostic text. Build separate wording for an unknown watchdog duration instead of formatting it as zero seconds.
  - confidence: 0.78
