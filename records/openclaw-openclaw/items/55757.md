## Summary

Keep open: current main still has the closed-stdin CLI watchdog path this PR targets, but the submitted patch is not ready because it disables the shared timeout too early, misses empty stdin, and lacks real behavior proof.

## What This Changes

The branch changes `src/agents/cli-runner/execute.ts` to skip the CLI no-output watchdog when a stdin payload is present and to allow optional watchdog durations in timeout messages.

## Best Possible Solution

Land a rebased fix that disables the watchdog only for normal child spawns whose stdin will be closed, preserves Claude live-session watchdogs, handles empty stdin, and includes focused regression coverage plus real CLI proof.

## Reproduction Assessment

Yes, by source inspection. Current main can pass a numeric `noOutputTimeoutMs` together with `input: stdinPayload` to a normal child spawn, while the child adapter closes stdin whenever input is present.

## Solution Assessment

No, not as submitted. The skip should be scoped to the normal closed-stdin child-spawn path and keyed off stdin presence or explicit closed-stdin state, with safer fallback timeout wording.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Scope watchdog disabling to closed-stdin child runs:** `src/agents/cli-runner/execute.ts:169-175`
  - body: This guard is computed before current-main Claude live-session routing, but live sessions keep stdin open and require a numeric turn watchdog. Move the skip to the normal child-spawn path so live sessions retain their no-output timer after rebase.
  - confidence: 0.88
- **[P2] Check stdin presence instead of payload truthiness:** `src/agents/cli-runner/execute.ts:169-175`
  - body: `stdinPayload` is `stdin ?? ""`, so an empty stdin-mode prompt still closes stdin but leaves this condition false. Use raw stdin presence or an explicit closed-stdin flag so empty stdin is covered.
  - confidence: 0.9
- **[P3] Avoid zero-second watchdog timeout text:** `src/agents/cli-runner/execute.ts:219-225`
  - body: When the watchdog duration is undefined, the `?? 0` fallback can surface `0s` if a no-output result reaches this branch. Use separate fallback copy instead of formatting an invented duration.
  - confidence: 0.78
