## Summary

This PR should stay open: it targets a real closed-stdin watchdog bug, but the branch is conflicting, the submitted guard misses empty stdin, a naive rebase can break Claude live-session watchdog protection, and the required real CLI behavior proof is still absent.

## What This Changes

The branch changes `src/agents/cli-runner/execute.ts` to omit `noOutputTimeoutMs` when `stdinPayload` is truthy and adjusts no-output timeout messages for an optional timeout value.

## Best Possible Solution

Rebase against current main, disable the supervisor no-output watchdog only for normal child spawns with defined closed stdin, keep Claude live-session timers numeric, handle empty stdin, improve unknown-duration diagnostics, and attach redacted real CLI proof.

## Reproduction Assessment

Yes. Source inspection shows current main passes a numeric no-output timeout to a normal child spawn whose adapter closes stdin whenever `input` is defined; I did not run a live long-silent CLI command in this read-only review.

## Solution Assessment

No, not as submitted. The safer fix is to move the skip to the normal closed-stdin child-spawn boundary, preserve Claude live-session no-output timers, use stdin presence rather than payload truthiness, and avoid `0s` fallback text.
