## Summary

Keep open. Current main and the latest release still pass a CLI no-output watchdog into normal closed-stdin child spawns, so the PR is useful, but the submitted patch is dirty against main and needs a small revision before merge.

## What This Changes

The PR changes `src/agents/cli-runner/execute.ts` to skip `noOutputTimeoutMs` when `stdinPayload` is truthy and adjusts no-output timeout messages for an optional watchdog value.

## Best Possible Solution

Land a rebased fix that disables the supervisor no-output watchdog only for normal closed-stdin child spawns, uses raw stdin presence or explicit closed-stdin state, preserves Claude live-session watchdog timers and the overall timeout, avoids misleading timeout text, and adds focused tests plus the required changelog entry.

## Reproduction Assessment

Yes. A code-level reproduction is clear: use a stdin-mode or long-prompt CLI backend that produces no output longer than the watchdog; current main passes both `input` and `noOutputTimeoutMs` to the child supervisor even though the child adapter closes stdin.

## Solution Assessment

No, not as submitted. The direction is right, but the guard should be scoped to non-live closed-stdin child spawns, should use raw stdin presence instead of `stdinPayload` truthiness, and should avoid `0s` watchdog messages when the watchdog is disabled.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Scope the skip to non-live child spawns:** `src/agents/cli-runner/execute.ts:169-175`
  - body: On current main, `noOutputTimeoutMs` is also passed into `runClaudeLiveSessionTurn`, and Claude CLI live sessions are `input: "stdin"` while keeping stdin open through `stdinMode: "pipe-open"`. A straight rebase of this shared guard would disable or break the live-session no-output timer, so keep the skip in the normal child-spawn path only.
  - confidence: 0.86
- **[P2] Use stdin presence for the watchdog guard:** `src/agents/cli-runner/execute.ts:169-175`
  - body: `stdinPayload` is `stdin ?? ""`, so a stdin-mode backend with an empty prompt still closes stdin but leaves this truthiness guard false. Guard on raw stdin presence or an explicit closed-stdin state so empty stdin is handled the same as non-empty stdin.
  - confidence: 0.9
- **[P3] Avoid reporting a 0s no-output timeout:** `src/agents/cli-runner/execute.ts:219-225`
  - body: When the watchdog is disabled, `noOutputTimeoutMs ?? 0` can produce a user-facing `0s` timeout message if an unexpected no-output result reaches this branch. Split the message construction so the disabled-watchdog path does not report a bogus duration.
  - confidence: 0.78
