## Summary

Keep open: the bug is source-reproducible, but this branch is stale against current main, the submitted guard would interfere with the Claude live-session watchdog path after rebase, and real CLI behavior proof is still absent.

## What This Changes

The PR makes `src/agents/cli-runner/execute.ts` skip the no-output watchdog when `stdinPayload` is truthy and adjusts watchdog timeout messages for optional timeout values.

## Best Possible Solution

Disable the no-output watchdog only for normal closed-stdin child spawns, keep numeric live-session watchdogs, handle empty stdin, add focused regression coverage, and require redacted real CLI proof.

## Reproduction Assessment

Yes, by source inspection: current main passes a numeric no-output watchdog with defined `input` to the child supervisor spawn, and the child adapter closes stdin whenever `input` is defined. I did not run a live long-silent CLI reproduction in this read-only review.

## Solution Assessment

No, not as submitted. The safer fix is to move the skip to the normal closed-stdin child-spawn boundary, preserve numeric watchdogs for Claude live sessions, and avoid treating empty stdin or unknown timeout duration as zero.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Keep live-session watchdogs numeric:** `src/agents/cli-runner/execute.ts:169-175`
  - body: On current main, this shared value is also passed to `runClaudeLiveSessionTurn`, while Claude live sessions keep stdin open and use their own per-turn no-output timer. Making it `undefined` for stdin-mode prompts would disable or invalidate those guards after rebase, so the skip needs to be scoped to the normal closed-stdin child spawn path.
  - confidence: 0.9
- **[P2] Check stdin presence instead of payload truthiness:** `src/agents/cli-runner/execute.ts:169-175`
  - body: `stdinPayload` is derived from `stdin ?? ""`, so an empty stdin-mode prompt still closes stdin in the child adapter but leaves this guard false. Use raw stdin presence or an explicit closed-stdin child condition so empty prompts get the same watchdog behavior.
  - confidence: 0.9
- **[P3] Avoid zero-second watchdog timeout text:** `src/agents/cli-runner/execute.ts:219-225`
  - body: If a no-output result reaches this branch while the timeout value is unknown, the fallback reports `0s`, which is misleading diagnostic text. Build separate wording for an unknown watchdog duration instead of formatting it as zero seconds.
  - confidence: 0.78
