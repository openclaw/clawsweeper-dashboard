## Summary

Keep open: the underlying closed-stdin watchdog bug is source-reproducible on current main, but this branch is not mergeable as submitted because it disables the shared watchdog before Claude live-session routing, misses empty stdin, and lacks real CLI behavior proof.

## What This Changes

The PR changes `src/agents/cli-runner/execute.ts` to make `noOutputTimeoutMs` optional when the CLI stdin payload is truthy and updates watchdog timeout messages for that optional value.

## Best Possible Solution

Land a rebased fix that disables the no-output watchdog only for normal closed-stdin child spawns, preserves numeric watchdogs for Claude live sessions, handles empty stdin, and adds focused regression coverage plus redacted real CLI proof.

## Reproduction Assessment

Yes, by source inspection. Current main can pass a numeric no-output watchdog with `input: stdinPayload`, and the child adapter closes stdin whenever `input` is defined.

## Solution Assessment

No, not as submitted. The safer fix is to decide at the normal closed-stdin child-spawn boundary, keep live-session watchdogs numeric, use stdin presence rather than payload truthiness, and avoid misleading unknown-duration text.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Keep live-session watchdogs numeric:** `src/agents/cli-runner/execute.ts:169-175`
  - body: On current main this value is also passed to `runClaudeLiveSessionTurn`, and Claude live sessions keep stdin open with their own per-turn no-output timer. Making the shared value `undefined` for stdin-mode prompts disables or invalidates those live-session guards after rebase; move the skip to the normal closed-stdin child spawn path.
  - confidence: 0.9
- **[P2] Check stdin presence instead of payload truthiness:** `src/agents/cli-runner/execute.ts:169-175`
  - body: `stdinPayload` is derived from `stdin ?? ""`, so an empty stdin-mode prompt still closes stdin in the child adapter but leaves this guard false. Use raw stdin presence or an explicit closed-stdin child condition so empty prompts get the same watchdog behavior.
  - confidence: 0.9
- **[P3] Avoid zero-second watchdog timeout text:** `src/agents/cli-runner/execute.ts:219-225`
  - body: If a no-output result reaches this branch while the timeout value is unknown, the fallback reports `0s`, which is misleading diagnostic text. Build separate wording for an unknown watchdog duration instead of formatting it as zero seconds.
  - confidence: 0.78
