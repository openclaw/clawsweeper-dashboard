## Summary

Keep open: the underlying bug is source-reproducible on current main, but this PR is not mergeable as submitted because the watchdog skip is too broad for current Claude live-session behavior and still lacks after-fix real CLI proof.

## What This Changes

The PR makes `executePreparedCliRun` skip the CLI no-output watchdog when stdin payload text is present and adjusts timeout messages for an optional watchdog value.

## Best Possible Solution

Rebase or replace the PR with a focused fix that disables the no-output watchdog only for closed-stdin one-shot child spawns, keeps Claude live-session turn timers, covers empty stdin, adds regression coverage, and includes redacted real CLI proof.

## Reproduction Assessment

Yes. Source inspection shows current main can pass a numeric `noOutputTimeoutMs` together with `input: stdinPayload`, and the child adapter closes stdin whenever input is defined, so a silent closed-stdin child can still hit the no-output watchdog.

## Solution Assessment

No, not as submitted. The direction is right, but the guard needs to be scoped to the normal closed-stdin child spawn path, preserve numeric Claude live-session timers, handle empty stdin, and avoid misleading `0s` timeout text.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Scope the watchdog skip to closed-stdin child spawns:** `src/agents/cli-runner/execute.ts:169-175`
  - body: Current main routes Claude live sessions through stdin mode with an open pipe and numeric per-turn watchdog timers. Applying this shared `undefined` value to stdin payloads after rebase would remove or break that protection, so the skip needs to be applied only at the normal child spawn where stdin is actually closed after writing.
  - confidence: 0.88
- **[P2] Guard on stdin presence instead of payload truthiness:** `src/agents/cli-runner/execute.ts:169-175`
  - body: `stdinPayload` is `stdin ?? ""`, so an empty stdin-mode prompt still closes stdin but leaves this guard false. Use raw stdin presence or an explicit closed-stdin state so empty input gets the same watchdog behavior as non-empty input.
  - confidence: 0.9
- **[P3] Avoid inventing a zero-second watchdog duration:** `src/agents/cli-runner/execute.ts:216-225`
  - body: If a no-output result reaches this branch while the watchdog value is disabled, the new `?? 0` path reports that the CLI stalled for `0s`. Build a separate disabled-watchdog message or keep duration formatting only for numeric watchdog values.
  - confidence: 0.78
