## Summary

Keep open: current main still has a source-reproducible closed-stdin/no-output watchdog path, but the submitted PR has correctness blockers against current main and lacks required real behavior proof.

## What This Changes

The PR changes `src/agents/cli-runner/execute.ts` to make `noOutputTimeoutMs` optional when `stdinPayload` is truthy and updates no-output timeout messages to tolerate an undefined timeout.

## Best Possible Solution

Land a rebased or replacement fix that disables the no-output watchdog only for closed-stdin normal child runs, preserves Claude live-session watchdogs, handles empty stdin, fixes timeout copy, and adds focused regression coverage plus real CLI proof.

## Reproduction Assessment

Yes, by source inspection. Current main can pass a numeric `noOutputTimeoutMs` together with `input: stdinPayload` into the normal child spawn path, and the child adapter closes stdin whenever input is defined.

## Solution Assessment

No, not as submitted. The fix should be scoped to the closed-stdin child spawn path rather than the shared pre-routing timeout value, and it should guard on stdin presence or explicit closed-stdin state instead of payload truthiness.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Scope watchdog skip to child runs with closed stdin:** `src/agents/cli-runner/execute.ts:169-175`
  - body: This guard is computed before current-main Claude live-session routing, but live sessions keep stdin open and require a numeric turn watchdog. Move the skip into the normal child spawn path or compute a child-only timeout so Claude live sessions keep their watchdog.
  - confidence: 0.88
- **[P2] Check stdin presence instead of payload truthiness:** `src/agents/cli-runner/execute.ts:169-175`
  - body: `stdinPayload` is `stdin ?? ""`, so an empty stdin-mode prompt still closes stdin but leaves this guard false. Use raw stdin presence or an explicit closed-stdin state instead of testing the payload string.
  - confidence: 0.9
- **[P3] Avoid zero-second watchdog timeout text:** `src/agents/cli-runner/execute.ts:219-225`
  - body: When the watchdog is disabled, the `?? 0` fallback can surface a misleading `0s` no-output timeout message if this result path is reached. Use separate text for an undefined watchdog duration.
  - confidence: 0.78
