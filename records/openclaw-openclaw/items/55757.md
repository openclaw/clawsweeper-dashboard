## Summary

Keep this PR open: it targets a real closed-stdin CLI watchdog bug that current main still has, but the branch is conflicting, lacks real behavior proof, and needs watchdog-boundary rework before merge.

## What This Changes

The PR changes `src/agents/cli-runner/execute.ts` so CLI spawns with stdin payloads omit the supervisor no-output timeout and adjusts timeout diagnostics for an optional watchdog value.

## Best Possible Solution

Rebase against current main, disable the supervisor no-output watchdog only for normal child spawns with defined closed stdin, keep Claude live-session timers numeric, handle empty stdin, improve unknown-duration diagnostics, and attach redacted real CLI proof.

## Reproduction Assessment

Yes, from source inspection. Current main passes a numeric no-output timeout to a normal child spawn whose adapter closes stdin whenever `input` is defined, although I did not run a live long-silent CLI command in this read-only review.

## Solution Assessment

No, not as submitted. The safer fix is to move the skip to the normal closed-stdin child-spawn boundary, preserve Claude live-session no-output timers, use stdin presence rather than payload truthiness, and avoid `0s` fallback text.
