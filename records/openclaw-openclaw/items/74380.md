## Summary

Keep open. Current main still has the reported installer metadata and formula-only brew command path, and the provided GitHub context shows open PR #74408 references this issue with closing syntax but has not merged. The issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue reports that `openclaw onboard` fails to install bundled skill dependencies because `model-usage` invokes the CodexBar Homebrew cask as a formula and `summarize` points at a removed Homebrew formula.

## Best Possible Solution

Land a revised PR #74408 or equivalent narrow fix that makes brew casks first-class in skill install metadata and command generation, switches `summarize` to a validated maintained installer source, updates security/status contracts, adds regression tests, and records the user-facing fix in the changelog.

## Reproduction Assessment

Yes. On macOS with Homebrew and missing `codexbar` or `summarize`, current main surfaces these bundled skills during `openclaw onboard` and runs the formula-only brew install path that matches the reporter's failures.

## Solution Assessment

Yes with review. Correcting the skill install metadata and installer command builder is the narrow maintainable path; the issue should stay open until the linked PR or equivalent code lands on main.
