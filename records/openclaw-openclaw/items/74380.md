## Summary

Keep open. Current main still has the reported formula-only Homebrew install path and outdated bundled skill metadata, and open PR #74408 is already the active implementation candidate for this issue.

## What This Changes

The issue reports that `openclaw onboard` fails installing bundled skill dependencies because CodexBar needs Homebrew cask installation and `summarize` no longer exists as a Homebrew formula.

## Best Possible Solution

Land PR #74408 or an equivalent narrow fix that preserves cask metadata through parsing/status/installation, moves `summarize` to a maintained installer source, adds focused tests, and records the user-facing fix in the changelog.

## Reproduction Assessment

Yes. On macOS with Homebrew and missing `codexbar`/`summarize`, `openclaw onboard` selects missing skills and current main still calls formula-only `brew install <formula>` for the reported bundled specs.

## Solution Assessment

Yes for the linked fix direction, but it is not implemented on main yet. The narrow maintainable solution is to add first-class cask handling and update the stale `summarize` install metadata rather than changing broad onboarding behavior.
