## Summary

Keep open. Current main still has the reported formula-only Homebrew installer path and affected bundled skill metadata, and open PR #74408 already references this issue as the implementation candidate, so the issue should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue reports that `openclaw onboard` fails to install bundled skill dependencies because `model-usage` uses a Homebrew cask through a formula-only path and `summarize` points at a removed brew formula.

## Best Possible Solution

Ship a narrow installer and metadata fix that preserves cask semantics for CodexBar, moves `summarize` to a maintained install source, adds focused regression coverage, and records the user-facing fix in the changelog.

## Reproduction Assessment

Yes. On macOS with Homebrew and missing `codexbar` or `summarize`, selecting the affected skills in `openclaw onboard` reaches the current `brew install <formula>` path shown in the reporter logs.

## Solution Assessment

Yes for the linked fix direction. Adding first-class cask handling and updating `summarize` to a maintained package source is narrower than changing onboarding broadly.
