## Summary

Keep open. Current `main` still has the reported installer behavior, and open PR #74408 already references this issue with closing syntax, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue reports `openclaw onboard` failing to install bundled skills because CodexBar is treated as a Homebrew formula instead of a cask and `summarize` points at a removed Homebrew formula.

## Best Possible Solution

Review and land PR #74408, or an equivalent narrow fix that preserves brew cask metadata, switches `summarize` to a maintained installer source, adds focused regression tests, and records the user-facing fix in the changelog.

## Reproduction Assessment

Yes. On macOS with Homebrew and missing `codexbar` or `summarize`, selecting those bundled skills in `openclaw onboard` reaches the current formula-only brew command path shown in the reporter logs.

## Solution Assessment

Yes for the linked fix direction. Adding first-class cask handling and updating `summarize` to a maintained package source is narrower than changing onboarding broadly.
