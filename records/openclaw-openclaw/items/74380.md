## Summary

Keep open. Current main still routes the reported bundled skill installs through the failing metadata/command paths, and open PR #74408 already references this issue with closing syntax, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue reports that `openclaw onboard` fails installing bundled skill dependencies because `model-usage` needs a Homebrew cask install and `summarize` points at a removed Homebrew formula.

## Best Possible Solution

Review and land PR #74408, or an equivalent narrow fix that adds real brew cask handling, switches `summarize` to a maintained installer source, adds focused regression coverage, and records the user-facing fix in the changelog.

## Reproduction Assessment

Yes. Source inspection plus the reporter logs give a high-confidence path: on macOS with Homebrew, selecting `model-usage` or `summarize` in `openclaw onboard` reaches the current formula-only brew command path for a cask and a removed formula.

## Solution Assessment

Yes for the linked fix direction. Preserving cask metadata for brew installs and updating `summarize` to a maintained installer source is narrower than changing onboarding broadly.
