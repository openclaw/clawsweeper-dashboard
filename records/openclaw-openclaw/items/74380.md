## Summary

Keep open. Current main still contains the reported skill install failure path, and PR #74408 is an open, unmerged implementation candidate that references this issue with closing syntax.

## What This Changes

Multiple bundled skill dependency installs fail during `openclaw onboard` because CodexBar is treated as a Homebrew formula and `summarize` points at a removed Homebrew formula.

## Best Possible Solution

Review and land PR #74408, or an equivalent narrow fix that preserves brew cask metadata, runs `brew install --cask` for CodexBar, switches `summarize` to a maintained installer source, adds focused regression coverage, and records the user-facing fix in the changelog.

## Reproduction Assessment

Yes. Source inspection on current main plus the reporter logs give a high-confidence path: on macOS with Homebrew, selecting `model-usage` or `summarize` in `openclaw onboard` reaches the current formula-only brew command path for a cask and a removed formula.

## Solution Assessment

Yes for the linked fix direction. Preserving a cask flag through the skill install contract and updating only the stale `summarize` installer metadata is narrower than changing onboarding broadly.
