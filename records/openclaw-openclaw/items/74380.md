## Summary

Keep open. Current main still matches the reported onboarding failure: bundled `model-usage` is labeled as a cask but parsed/executed as a Homebrew formula, `summarize` still points at the removed Homebrew formula, and the open closing PR has not landed.

## What This Changes

The issue asks for `openclaw onboard` to install bundled skill dependencies successfully instead of failing on `model-usage`/CodexBar and `summarize` Homebrew metadata.

## Best Possible Solution

Land PR #74408 or an equivalent narrow fix that adds first-class cask metadata through parsing, types, installer command generation, status/security metadata, and tests, while switching `summarize` away from the removed tap formula to a validated maintained installer source and adding a user-facing changelog entry.

Yes. On macOS with Homebrew installed and missing `codexbar`/`summarize`, select those missing skill dependencies in `openclaw onboard`; current main will route them through formula-only `brew install` commands that match the reporter's failures.

## Solution Assessment
