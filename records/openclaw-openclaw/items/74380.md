## Summary

Keep open. Current main still routes the reported bundled skill installs through a formula-only Homebrew path and stale installer metadata, and open PR #74408 references this issue with closing syntax, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue reports `openclaw onboard` failing to install bundled skill dependencies because `model-usage` needs Homebrew cask handling and `summarize` still points at a removed Homebrew formula.

## Best Possible Solution

Review and land PR #74408, or an equivalent narrow fix that preserves Homebrew cask semantics, switches `summarize` to a maintained installer source, adds focused regression coverage, and records the user-facing fix in the changelog.

## Reproduction Assessment

Yes. Source inspection plus the reporter logs give a high-confidence path: on macOS with Homebrew, selecting `model-usage` or `summarize` in `openclaw onboard` reaches the current formula-only brew command for a cask and a removed formula.

## Solution Assessment

Yes for the linked fix direction. Preserving cask semantics for brew installs and updating `summarize` to a maintained installer source is narrower than changing onboarding broadly.
