## Summary

Keep open. Current main still sends the reported onboarding path through formula-only Homebrew installs, the bundled metadata still points at the affected CodexBar and summarize install specs, and open PR #74408 is the unmerged implementation candidate for this issue.

## What This Changes

The issue reports `openclaw onboard` failing to install bundled skill dependencies because CodexBar needs a brew cask path and summarize points at a removed Homebrew formula.

## Best Possible Solution

Review and land PR #74408, or an equivalent narrow fix that adds cask-aware brew installer metadata, moves summarize to a maintained installer source, adds focused regression coverage, and records the user-facing fix in the changelog.

## Reproduction Assessment

Yes. On macOS with Homebrew and missing `codexbar` or `summarize`, selecting those bundled skills in `openclaw onboard` reaches the current formula-only brew install path shown in the reporter logs.

## Solution Assessment

Yes for the linked fix direction. Adding first-class cask handling and updating summarize to a maintained package source is narrower than changing onboarding broadly, but the issue should remain open until that fix lands.
