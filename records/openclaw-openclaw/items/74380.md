## Summary

Keep open. Current main still routes the reported bundled skill installs through the failing Homebrew paths, and PR #74408 is open with closing syntax for this issue, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue reports that `openclaw onboard` fails when installing bundled skill dependencies because CodexBar is treated as a Homebrew formula and summarize points at a removed Homebrew formula.

## Best Possible Solution

Land PR #74408 or an equivalent narrow fix that preserves brew cask semantics through parsing, types, status display, installer command generation, and security-scan metadata; updates `model-usage` and `summarize` to maintained install sources; adds focused regression coverage; and records the user-facing fix in the changelog.

## Reproduction Assessment

Yes. On macOS with Homebrew and missing `codexbar`/`summarize`, current main will surface these bundled skills during `openclaw onboard` and invoke the formula-only brew install commands that match the reporter's failures.

## Solution Assessment

Yes, with revisions. Correcting skill install metadata and command generation is the narrow maintainable path; special-casing onboarding would leave the underlying skill installer contract wrong.
