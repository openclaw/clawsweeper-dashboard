## Summary

Keep open. Current main still contains the formula-only Homebrew skill installer and the two bundled skill manifests that match the reported `model-usage` and `summarize` failures, and PR #74408 is already the open unmerged closing fix candidate.

## What This Changes

The issue reports that `openclaw onboard` fails while installing bundled skill dependencies because `model-usage` invokes CodexBar through Homebrew as a formula and `summarize` points at a removed Homebrew formula.

## Best Possible Solution

Land PR #74408 or an equivalent narrow fix that adds a real cask install path for CodexBar, moves `summarize` to the maintained npm package, and keeps installer/status/security metadata plus focused coverage aligned.

## Reproduction Assessment

Yes. On macOS with missing `codexbar` and `summarize`, current main will surface these bundled install options through onboarding and run the same formula-only Homebrew commands shown in the report.

## Solution Assessment

Yes. The linked repair direction is the narrowest maintainable solution because it fixes the installer contract for casks and corrects the stale `summarize` package source instead of adding onboarding-specific special cases.
