## Summary

Keep open. Current main still lacks the requested Jina task configuration and request serialization, and the issue is explicitly paired with open, unmerged PR #74444, so it should remain open until that implementation lands or is replaced.

## What This Changes

Request adding configurable Jina Embeddings v5 task parameters for memory search embedding requests, preferably with separate query and document task settings.

## Best Possible Solution

Land PR #74444 or a narrow replacement that adds typed query/document task settings through memory search config, resolved provider options, OpenAI-compatible query and batch embedding request bodies, cache identity, tests, docs, and changelog.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: configure a Jina task value under memorySearch and current main has no accepted schema field or request-body path to carry it, while OpenAI-compatible embedding requests still serialize only `model`, `input`, and optional `input_type`.

## Solution Assessment

Unclear until PR #74444 is reviewed, but the typed `queryTask`/`documentTask` direction is safer than an unrestricted `extraBody` pass-through because memory search has separate query/document paths and task values need to participate in cache identity.
