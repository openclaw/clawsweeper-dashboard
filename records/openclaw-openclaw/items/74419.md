## Summary

Keep open. Current main still has no schema, resolved config, provider option, request-body, or batch-cache path for Jina/OpenAI-compatible embedding `task` values, and open PR #74444 is explicitly paired with this issue via closing syntax.

## What This Changes

Add configurable Jina/OpenAI-compatible embedding `task` support, preferably with separate query and document task settings, for memory search embeddings.

## Best Possible Solution

Review and land PR #74444, or a narrow replacement, that adds typed query/document task settings through config schema, resolved config, provider options, OpenAI request bodies, batch cache identity, docs, and changelog.

## Solution Assessment

Unclear until PR #74444 is reviewed. A typed query/document task surface is safer than unrestricted `extraBody` because query embeddings, document indexing, and cache identity need separate values.
