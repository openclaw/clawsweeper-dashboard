## Summary

Keep open. Current main still lacks a configurable Jina `task` payload for OpenAI-compatible memory embeddings, and the issue is already paired with open PR #74444 using closing syntax, so the issue should remain open until that PR lands or is replaced.

## What This Changes

Allow OpenAI-compatible memory embedding requests to send task-specific Jina v5 adapter parameters, ideally differentiated between query and document embeddings.

## Best Possible Solution

Land or revise PR #74444, or an equivalent narrow implementation, so the config schema, resolved memorySearch state, provider options, OpenAI inline and batch request bodies, cache identity, docs, and changelog all handle query/document task values consistently.

## Reproduction Assessment

Yes. A source inspection against current main is sufficient: the proposed keys are absent from the strict memorySearch config schema, and the OpenAI embedding request body has no path to emit `task`.

## Solution Assessment

Unclear until the linked PR is reviewed. A typed `queryTask`/`documentTask` surface looks more maintainable than unconstrained `extraBody` for asymmetric retrieval, provided cache identity and docs are updated with it.
