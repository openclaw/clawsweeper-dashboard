Keep open. Current main still exposes only OpenAI-compatible `input_type` memory embedding config and request bodies, while PR #74444 remains open, unmerged, and explicitly targets this issue.

## What This Changes

Request support for Jina Embeddings v5 task-specific `task` parameters in OpenAI-compatible memory embedding requests, likely through typed query/document task config or a request-body passthrough.

## Best Possible Solution

Review and land PR #74444 or a narrow replacement that carries typed query/document task settings through schema, resolved config, provider requests, batch bodies, cache identity, docs, and changelog.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: configure the requested `memorySearch.queryTask`, `memorySearch.documentTask`, or embedding `extraBody.task`, and current main has no schema, resolved-config, provider-option, request-body, or cache-identity path that can carry it.

## Solution Assessment

Unclear until PR #74444 is reviewed. A typed query/document task surface looks more maintainable than unrestricted `extraBody` because query embeddings, document indexing, batch requests, and cache identity need separate values.
