## Summary

Keep open. Current main still only exposes and forwards OpenAI-compatible `input_type` settings for memory embeddings, not Jina `task` values, and PR #74444 explicitly closes this issue but remains open and unmerged.

## What This Changes

Request to add configurable Jina/OpenAI-compatible embedding `task` parameters for query and document memory embeddings.

## Best Possible Solution

Review and land PR #74444, or a narrow replacement, adding typed query/document task settings through config, provider options, request bodies, batch cache identity, docs, and changelog.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: configure `memorySearch.queryTask`, `memorySearch.documentTask`, or `memorySearch.extraBody.task` and current main has no schema, resolved-config, provider-option, or request-body path that can carry it.

## Solution Assessment

Unclear until PR #74444 is reviewed. A typed query/document task surface is safer than unrestricted embedding `extraBody` because query embeddings, document indexing, and cache identity need separate values.
