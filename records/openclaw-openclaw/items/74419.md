## Summary

Keep open. Current main still only carries OpenAI-compatible embedding `input_type` fields through config resolution, provider options, request bodies, and batch cache identity, and open PR #74444 is already linked as the implementation candidate.

## What This Changes

The issue requests configurable Jina v5 embedding task parameters for OpenAI-compatible memory embeddings.

## Best Possible Solution

Review and land PR #74444 or a narrow replacement that adds typed query/document task settings through config schema, resolved config, provider options, direct and batch request bodies, cache identity, docs, and changelog.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: configure the requested `memorySearch.queryTask`, `memorySearch.documentTask`, or `extraBody.task`, and current main has no schema, resolved-config, provider-option, or request-body path that can carry it.

## Solution Assessment

Unclear until PR #74444 is reviewed. A typed query/document task surface looks more maintainable than unrestricted `extraBody` because query embeddings, document indexing, and cache identity need separate values.
