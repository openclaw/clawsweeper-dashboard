## Summary

Keep open. Current main still lacks configurable Jina v5 task forwarding for memory search embeddings, and the issue is paired with open, unmerged PR #74444, so it should remain open until that implementation lands or is replaced.

## What This Changes

Request support for Jina Embeddings v5 task-specific adapter parameters in memory search embedding requests, preferably with typed query/document task configuration or a safe extra body mechanism.

## Best Possible Solution

Land PR #74444 or a narrow replacement that adds typed query/document task settings through memory search config, resolved provider options, OpenAI-compatible query and batch embedding request bodies, cache identity, docs, changelog, and focused tests.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: current main has no accepted memorySearch task fields and both query and batch OpenAI-compatible embedding requests serialize only model/input plus optional input_type.

## Solution Assessment

Unclear until PR #74444 is reviewed, but a typed queryTask/documentTask solution is the most maintainable direction because memory search has separate query and document embedding paths and the task values must affect cache identity.
