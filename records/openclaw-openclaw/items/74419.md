## Summary

Keep open. Current main still has only OpenAI-compatible `input_type` support for memory embeddings, not Jina v5 `task` settings, and open unmerged PR #74444 explicitly tracks the requested implementation with closing syntax.

## What This Changes

Allow `memorySearch` configuration to pass Jina v5 task parameters, ideally distinguishing query and document embedding requests.

## Best Possible Solution

Review and land PR #74444 or a narrow replacement that adds typed query/document task settings through schema, resolved provider options, request bodies, cache identity, docs, and changelog.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: configure a Jina task value under `memorySearch` and current main has no schema field or request-body path that can carry it.

## Solution Assessment

Unclear until PR #74444 is reviewed. The typed `queryTask`/`documentTask` direction is safer than unrestricted `extraBody` because query/document embeddings and cache identity need separate handling.
