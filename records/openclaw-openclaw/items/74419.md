## Summary

Keep open. Current main still only accepts and forwards OpenAI-compatible `input_type` fields for memory embeddings, not Jina v5 `task` settings, and open PR #74444 is explicitly paired with this issue via closing syntax.

## What This Changes

Add configurable Jina v5 task parameters for OpenAI-compatible memory embeddings, preferably distinguishing query and document requests.

## Best Possible Solution

Review and land PR #74444 or a narrow replacement that adds typed query/document task settings through schema, provider options, OpenAI request bodies, batch cache identity, docs, and changelog.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: configure `memorySearch.queryTask`, `memorySearch.documentTask`, or `extraBody.task` and current main has no schema, resolved-config, provider-option, or request-body path that can carry it.

## Solution Assessment

Unclear until PR #74444 is reviewed. A typed query/document task surface is safer than unrestricted `extraBody` because query embeddings, document indexing, and cache identity need separate values.
