## Summary

Keep open. Current main still lacks a configurable Jina `task` path for memory-search embeddings, and this issue is paired with open, unmerged PR #74444 using closing syntax, so it should remain open until that implementation lands or is replaced.

## What This Changes

The issue requests configurable embedding request body support for Jina v5 task-specific adapters, preferably through typed query/document task settings or a controlled extra-body mechanism.

## Best Possible Solution

Review and land PR #74444, or a narrow replacement, that adds typed query/document task settings through memory-search config, provider request plumbing, OpenAI-compatible query and batch embedding bodies, cache identity, tests, docs, and changelog.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: configure a Jina task value under `memorySearch`, and current main has no schema field, resolved config field, provider option, request-body serialization, or cache identity slot to carry it.

## Solution Assessment

Unclear until PR #74444 is reviewed as a patch. The typed `queryTask`/`documentTask` direction appears safer than unrestricted `extraBody` because memory search has distinct query/document paths and task values need to affect cache identity.
