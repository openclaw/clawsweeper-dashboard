## Summary

Keep open. Current main still has only OpenAI-compatible `input_type` plumbing for memory embeddings, with no `task`, `queryTask`, `documentTask`, or embedding `extraBody` path, and PR #74444 is the active unmerged closing implementation candidate.

## What This Changes

Add a configurable Jina Embeddings v5 `task` request field for OpenAI-compatible memory embeddings, ideally with separate query and document settings.

## Best Possible Solution

Carry typed query/document task settings through config schema, resolved memory config, provider options, direct and batch request bodies, cache identity, docs, and changelog.

## Reproduction Assessment

Yes. Source inspection is enough: configuring `memorySearch.queryTask`, `memorySearch.documentTask`, or embedding `extraBody.task` on current main has no schema, resolver, provider-option, request-body, or cache-identity path to carry it.

## Solution Assessment

Unclear until PR #74444 is reviewed. A typed query/document task surface looks more maintainable than unrestricted `extraBody` because query embeddings, document indexing, batch requests, and cache identity need separate values.
