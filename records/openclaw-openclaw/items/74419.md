## Summary

Keep open. Current main still lacks a `task`, `queryTask`, `documentTask`, or memory-search embedding `extraBody` path for OpenAI-compatible embedding requests, and the issue is paired with open, unmerged PR #74444 using closing syntax, so it should remain open until that PR lands or is replaced.

## What This Changes

Allow OpenAI-compatible memory embeddings to send Jina v5 task parameters, preferably with separate query/document task configuration or a controlled extra-body mechanism.

## Best Possible Solution

Land PR #74444 or a narrow replacement that adds typed query/document task settings through the memorySearch schema, resolved config, provider request contract, OpenAI query and batch request bodies, embedding cache identity, tests, docs, and changelog.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: configure a Jina task value under `memorySearch` and current main has no accepted config field or request-body path to carry it into OpenAI-compatible embedding calls.

## Solution Assessment

Unclear until PR #74444 is reviewed. The typed `queryTask`/`documentTask` direction is more maintainable than unrestricted `extraBody` because memory search has distinct query/document embedding paths and task values need to participate in cache identity.
