## Summary

Keep open. Current main still supports only OpenAI-compatible `input_type` memory embedding options, not Jina v5 `task` parameters, and open unmerged PR #74444 is explicitly paired with this issue by closing syntax.

## What This Changes

Request to add configurable Jina v5 `task` parameters for OpenAI-compatible memory embeddings, preferably with query/document-specific settings.

## Best Possible Solution

Review and land PR #74444 or a narrow replacement that adds typed query/document task settings through schema, resolved config, request bodies, cache identity, docs, and changelog.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: configure a Jina task value under `memorySearch`, and current main has no schema field or request-body path that can carry it.

## Solution Assessment

Unclear until PR #74444 is reviewed. The typed `queryTask`/`documentTask` direction is safer than unrestricted `extraBody` because query/document embeddings and cache identity need separate handling.
