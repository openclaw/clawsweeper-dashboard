## Summary

Keep open. Current main still lacks a configurable embedding `task` path for Jina/OpenAI-compatible embeddings, and the issue is paired with open PR #74444 using closing syntax, so it should remain open until that PR lands or is replaced.

## What This Changes

The request is to add a configurable Jina Embeddings v5 task parameter for memory-search embedding requests, preferably with separate query and document task settings.

## Best Possible Solution

Land PR #74444 or a replacement that adds a typed `queryTask`/`documentTask` memory-search surface, threads it through resolved config, provider creation, query embeddings, document/batch embeddings, cache identity, tests, docs, and changelog without introducing unrestricted request-body pass-through.

## Reproduction Assessment

Yes. The high-confidence reproduction is source-level: current main has no accepted config field or request-body path for a Jina task value, while the OpenAI-compatible embedding body only serializes `model`, `input`, and optional `input_type`.

## Solution Assessment

Unclear until the linked PR is reviewed. A typed query/document task API looks more maintainable than unrestricted `extraBody` because memory search has separate query and document embedding paths and cache identity needs to account for task changes.
