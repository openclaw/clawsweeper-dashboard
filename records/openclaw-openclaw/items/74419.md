## Summary

Keep open. Current main still lacks the requested Jina task configuration and request serialization, and the issue is paired with open, unmerged PR #74444, so it should remain open until that implementation lands or is replaced.

## What This Changes

The issue asks for configurable Jina Embeddings v5 task parameters in OpenAI-compatible memory search embedding requests.

## Best Possible Solution

Land PR #74444 or a narrow replacement that adds typed query/document task settings through the memorySearch schema, resolved config, provider request contract, OpenAI query and batch request bodies, embedding cache identity, tests, docs, and changelog.

## Reproduction Assessment

Yes. The high-confidence reproduction path is source-level: configure a Jina task value under memorySearch and current main has no accepted schema field or request-body path to carry it, while OpenAI-compatible embedding requests still serialize only `model`, `input`, and optional `input_type`.

## Solution Assessment

Unclear until PR #74444 is reviewed, but the typed `queryTask`/`documentTask` direction is safer than an unrestricted `extraBody` pass-through because memory search has separate query/document paths and cache identity must change when task values change.
