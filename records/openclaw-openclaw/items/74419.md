## Summary

Keep open. Current main still only carries OpenAI-compatible embedding `input_type` settings through config resolution, provider options, direct requests, batch requests, and cache identity, while linked PR #74444 remains open and is the active implementation candidate.

## What This Changes

Allow OpenAI-compatible memory embedding config to pass Jina v5 task-specific adapter values, preferably with separate query and document task settings.

## Best Possible Solution

Review and land PR #74444 or a narrow replacement that carries typed query/document task settings through schema, resolved config, provider requests, direct and batch bodies, cache identity, docs, and changelog.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: configure the requested `memorySearch.queryTask`, `memorySearch.documentTask`, or embedding `extraBody.task`, and current main has no schema, resolved-config, provider-option, request-body, or cache-identity path that can carry it.

## Solution Assessment

Unclear until PR #74444 is reviewed. A typed query/document task surface looks more maintainable than unrestricted `extraBody` because query embeddings, document indexing, batch requests, and cache identity need separate values.
