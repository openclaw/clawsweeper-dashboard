## Summary

Keep open. Current main still has no schema, resolved-config, provider-option, request-body, or batch-cache path for Jina-style embedding `task` values, and open PR #74444 is explicitly linked as the implementation candidate.

## What This Changes

Request adding configurable Jina v5 embedding task parameters for memory search, preferably with separate query and document task values for OpenAI-compatible embedding requests.

## Best Possible Solution

Review and land PR #74444 or a narrow replacement that adds typed query/document task settings through config schema/resolution, provider options, OpenAI query and batch request bodies, cache identity, docs, and changelog.

Yes. Source-level reproduction is high confidence: configure `memorySearch.queryTask`, `memorySearch.documentTask`, or `extraBody.task`, and current main has no schema, resolved-config, provider-option, or request-body path that can carry it.

## Solution Assessment

Unclear until PR #74444 is reviewed. A typed query/document task surface is more maintainable than unrestricted `extraBody` because query embeddings, document indexing, and cache identity need separate values.
