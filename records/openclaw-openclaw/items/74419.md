Keep open. Current main still has no config, resolved-options, OpenAI request-body, batch-cache, docs, or changelog path for Jina-style embedding `task` values, and open PR #74444 is the paired implementation candidate.

## What This Changes

Add support for configuring Jina v5 embedding task adapters, such as query/document `task` values, in OpenAI-compatible memory embedding requests.

## Best Possible Solution

Review and land PR #74444, or a narrow replacement, with typed query/document task settings carried through schema, resolved memory options, provider requests, batch indexing, cache identity, docs, and changelog.

Yes. A source-level reproduction is high confidence: configure `memorySearch.queryTask`, `memorySearch.documentTask`, or `extraBody.task`, and current main has no accepted config or OpenAI embedding request path that can carry it.

## Solution Assessment

Unclear until PR #74444 is reviewed. A typed query/document task surface appears more maintainable than unrestricted `extraBody` because query embeddings, document embeddings, and cache identity need separate values.
