## Summary

Keep open. Current main still only carries OpenAI-compatible `input_type` through memory-search config and embedding request bodies; the Jina-style `task` request field has no schema, resolved-config, provider-option, cache-key, or request-body path. The linked open PR #74444 is the active implementation candidate, so this issue should remain open until that PR lands or is replaced.

## What This Changes

Request adding configurable Jina/OpenAI-compatible embedding `task` parameters, preferably query/document-specific, to memory-search embedding requests.

## Best Possible Solution

Review and land PR #74444 or a narrow replacement that adds typed query/document task settings through config schema/resolution, provider options, query and batch request bodies, cache identity, docs, and changelog.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: configure `memorySearch.queryTask`, `memorySearch.documentTask`, or `extraBody.task`, and current main has no schema, resolved-config, provider-option, or request-body path that can carry it.

## Solution Assessment

Unclear until PR #74444 is reviewed. A typed query/document task surface is likely more maintainable than unrestricted `extraBody` because query embeddings, document indexing, and cache identity need separate values.
