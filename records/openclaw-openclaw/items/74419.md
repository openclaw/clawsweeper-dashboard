## Summary

Keep open. Current `main` still has no schema, resolved-config, provider-option, direct request, batch request, or cache-identity path for Jina `task` values, and PR #74444 remains the linked open implementation candidate.

## What This Changes

The issue requests configurable Jina v5 embedding task parameters for OpenAI-compatible memory embedding requests.

## Best Possible Solution

Review and land PR #74444 or a narrow replacement that adds optional query/document task settings through schema, resolution, provider options, inline and batch OpenAI payloads, cache identity, docs, metadata, tests, and changelog.

## Reproduction Assessment

Yes. Source inspection on current `main` is enough: configuring `memorySearch.queryTask`, `memorySearch.documentTask`, or `extraBody.task` has no accepted schema or request-body path to reach the OpenAI-compatible embedding endpoint.

## Solution Assessment

Unclear until PR #74444 is reviewed. A typed query/document task surface appears more maintainable than unrestricted `extraBody` because query embeddings, document indexing, batch payloads, and cache identity need separate values.
