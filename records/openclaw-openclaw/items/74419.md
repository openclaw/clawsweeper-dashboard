## Summary

Keep open. Current main still has only the OpenAI-compatible `input_type` path for memory embeddings, and PR #74444 is still open with closing syntax for this issue. ([github.com](https://github.com/openclaw/openclaw/pull/74444))

## What This Changes

Allow memory-search embedding requests to send Jina v5 task-specific adapter values, preferably with separate query and document task settings.

## Best Possible Solution

Review and land PR #74444 or a narrow replacement that adds typed query/document task settings through schema, resolved config, provider options, request bodies, cache identity, docs, tests, and changelog.

## Reproduction Assessment

Yes. Static reproduction is high confidence: configure `memorySearch.queryTask`, `memorySearch.documentTask`, or `extraBody.task` and current main has no schema, resolved-config, provider-option, or request-body path that can carry it.

## Solution Assessment

Unclear until the paired PR is reviewed. A typed query/document task surface is likely safer than unrestricted `extraBody` because query embeddings, document indexing, and cache identity need separate values.
