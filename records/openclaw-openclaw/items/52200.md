## Summary

Keep open: this PR fixes a real Windows skill prompt path bug and now has real behavior proof, but the current diff normalizes backslashes for every home-compacted path and can corrupt valid POSIX skill paths that contain a literal backslash.

## What This Changes

The PR changes skill path compaction to replace backslashes with forward slashes after home-prefix compaction, adds regression coverage, and adds a changelog entry.

## Best Possible Solution

Keep the normalization at the skill prompt compaction boundary, but gate it to Windows/native backslash paths and add coverage that preserves POSIX literal-backslash paths.

## Reproduction Assessment

Yes. Source inspection shows current main compacts home-relative skill paths by prepending `~/` while preserving a Windows backslash suffix, then emits that value directly in `<location>`; I did not run a live Windows repro in this read-only pass.

## Solution Assessment

No, not as patched. The normalization direction is the narrow fix for Windows, but it should only rewrite Windows separators so POSIX filenames containing literal backslashes remain addressable.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Gate separator normalization to Windows paths:** `src/agents/skills/workspace.ts:82`
  - body: The helper now replaces any backslash in every home-compacted skill path. On POSIX, backslash is a valid filename character rather than a separator, so a skill under `~/foo\bar/SKILL.md` would be rendered as `~/foo/bar/SKILL.md` and point the model at a different path. Keep the replacement behind the Windows/native-separator case or otherwise only normalize Windows paths.
  - confidence: 0.83
