## Summary

Keep open: current main still has the Windows separator bug in the skills prompt path flow, and this PR remains useful source work. It should not be closed as implemented because the compaction helper still returns mixed `~/...\...` paths, but the branch is stale against the current helper shape and needs adaptation before merge.

## What This Changes

This PR normalizes Windows backslashes in compacted skill prompt paths and adds a regression test asserting rendered skill `<location>` values do not contain backslashes.

## Best Possible Solution

Normalize prompt-facing Windows skill file paths once at the skills path compaction boundary, preserving current home-prefix and realpath-prefix behavior, so both full and compact available-skills prompt formats receive slash-consistent `<location>` values.

## Reproduction Assessment

Yes. On Windows, `path.sep` is `\`, `compactHomePath()` prepends `~/` while leaving the sliced remainder unchanged, and the prompt formatters emit that value directly into `<location>`.

## Solution Assessment

Yes in direction, but not as a merge-ready patch. The narrow fix is still the right one, but it should be rebased onto current main by placing normalization in or immediately after the current `compactHomePath()`/`compactSkillPaths()` flow.
