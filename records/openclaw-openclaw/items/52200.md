## Summary

Keep open: the Windows skill prompt bug is valid and after-fix terminal proof is present, but the latest PR head still normalizes backslashes unconditionally after home compaction, which can misadvertise POSIX paths that contain literal backslashes.

## What This Changes

The PR normalizes backslashes in home-compacted skill prompt paths, adds regression coverage and a changelog entry, and adjusts a provider-validation test fixture type.

## Best Possible Solution

Keep the home-prefix compaction fix, but gate backslash-to-slash normalization to Windows/native backslash-separated paths and add POSIX literal-backslash coverage before merge.

## Reproduction Assessment

Yes. Current main source shows home-relative skill paths are compacted by prepending `~/` while preserving the suffix, and the prompt renderer emits that file path directly; the PR-introduced POSIX risk is also source-reproducible because Node on Linux preserves backslashes as literal path characters.

## Solution Assessment

No, not as currently patched. Normalizing after compaction is the right narrow Windows fix, but this implementation should not rewrite POSIX literal backslashes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Gate skill path backslash normalization to Windows:** `src/agents/skills/workspace.ts:82`
  - body: The new helper replaces backslashes in every home-compacted path. On POSIX, `\\` can be a literal filename character, so a skill under `~/.openclaw-test-skills\\win-skill/SKILL.md` would be advertised as `~/.openclaw-test-skills/win-skill/SKILL.md`, pointing the model at a different path. Gate this replacement to Windows/native backslash paths or preserve POSIX literal backslashes.
  - confidence: 0.86
