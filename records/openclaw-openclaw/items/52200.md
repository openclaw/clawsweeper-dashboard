## Summary

Keep open. The PR targets a valid Windows skill-path bug and includes real prompt-builder proof, but the live diff normalizes every backslash in a home-compacted path, which can corrupt valid POSIX skill paths that contain a literal backslash.

## What This Changes

The PR changes skill path compaction to replace backslashes with forward slashes after home-prefix compaction, adds regression coverage, and adds a changelog entry.

## Best Possible Solution

Keep normalization at the skill prompt compaction boundary, but restrict it to Windows/native separator paths and preserve POSIX literal-backslash skill names.

## Reproduction Assessment

Yes. Source inspection on current main shows `compactHomePath()` prepends `~/` while preserving a Windows-style backslash suffix, and the prompt renderer emits that value in `<location>`; the linked report also includes Windows 11 examples.

## Solution Assessment

No, not as patched. The normalization belongs at this compaction boundary, but the current helper is too broad because POSIX backslashes can be literal filename characters.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Gate separator normalization to Windows paths:** `src/agents/skills/workspace.ts:82`
  - body: The helper now replaces every backslash after home compaction. On POSIX, backslash is a valid filename character, so a skill under `~/foo\bar/SKILL.md` would be rendered as `~/foo/bar/SKILL.md` and point the model at a different path. Keep the replacement behind the Windows/native-separator case and add preservation coverage for POSIX literal backslashes.
  - confidence: 0.86
