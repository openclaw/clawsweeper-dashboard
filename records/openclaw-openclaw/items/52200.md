## Summary

Keep open. Current main still returns `~/` plus the unchanged Windows remainder from skill home compaction, and both available-skills prompt renderers emit that `filePath` directly. The PR targets a real Windows prompt-path bug, but it still needs the required changelog entry and should be adapted to current main's home-prefix helper shape before merge.

## What This Changes

The PR normalizes Windows backslashes in compacted prompt-facing skill paths and adds a regression assertion for rendered skill `<location>` values.

## Best Possible Solution

Normalize prompt-facing Windows paths at the compaction boundary while preserving current multi-home and realpath-prefix behavior, add focused rendered-`<location>` coverage, and include a single-line changelog fix entry crediting the contributor.
