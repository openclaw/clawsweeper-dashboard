## Summary

Keep open. The Windows bug is valid and real behavior proof is supplied, but the latest patch still rewrites every backslash in every home-compacted skill path, which can corrupt valid POSIX paths containing literal backslashes.

## What This Changes

The PR adds prompt-facing skill path normalization after home-prefix compaction, regression coverage, a changelog entry, and a provider-validation test fixture type adjustment.

## Best Possible Solution

Gate backslash replacement to Windows/native backslash-separated paths at the skill prompt compaction boundary and add POSIX literal-backslash coverage before merging.

## Reproduction Assessment

Yes. Source inspection against current main shows home-relative skill paths are compacted by prepending `~/` while preserving the suffix, and a focused Node path probe confirms POSIX treats backslash as a literal path character.

## Solution Assessment

No, not as patched. Normalizing after compaction is the right narrow Windows fix, but this implementation must not rewrite POSIX literal backslashes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Gate separator normalization to Windows paths:** `src/agents/skills/workspace.ts:82`
  - body: The new helper rewrites backslashes in every home-compacted path. On POSIX, `\\` is a valid filename character, so a skill under `~/.openclaw-test-skills\\win-skill/SKILL.md` would be rendered as `~/.openclaw-test-skills/win-skill/SKILL.md` and point the model at a different path. Gate this replacement to Windows/native backslash paths or preserve POSIX literal backslashes.
  - confidence: 0.86
