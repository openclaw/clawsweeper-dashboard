## Summary

Keep open. Current main still compacts prompt-facing skill paths by prepending `~/` and returning the remaining Windows path segment unchanged, and both full and compact available-skills renderers emit `skill.filePath` directly. The PR addresses a real Windows prompt-path bug, but it is missing the required user-facing changelog entry and should be adapted to the current `resolveCompactHomePrefixes()` / `compactHomePath()` helper shape before merge.

## What This Changes

The PR normalizes Windows backslashes in compacted prompt-facing skill paths and adds a regression assertion that rendered skill `<location>` values do not contain backslashes.

## Best Possible Solution

Normalize prompt-facing Windows paths at the current compaction boundary, preferably inside `compactHomePath()` or immediately after `compactSkillPaths()` maps paths, while preserving current multi-home and realpath-prefix behavior. Add focused Windows-style regression coverage for rendered `<location>` values and include the required changelog entry crediting the contributor.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: on Windows, `path.sep` is `\`, `compactHomePath()` slices after the home prefix and prepends `~/`, and the prompt formatters emit that returned `filePath` without normalization.

## Solution Assessment

No, not as-is. The PR’s direction is the narrow maintainable fix, but the final patch should be adapted to current main’s helper split and include the required changelog entry before merge.
