## What This Changes

The branch adds an age-only fallback deletion branch to `pruneDiagnosticSessionStates()` so any diagnostic session state older than `SESSION_STATE_TTL_MS` is removed even when it is non-idle or still queued.

## Best Possible Solution

Land one canonical recovery-aware cleanup that prunes failed-recovery ghost entries only after active and queued recovery can no longer use the state, with focused regression coverage and real behavior proof.

## Reproduction Assessment

Yes, source-reproducible: failed recovery outcomes do not mutate diagnostic state, while current main and the latest release only TTL-prune idle empty entries. I did not run a live gateway repro in this read-only review.

## Solution Assessment

No. The age-only fallback is a plausible leak mitigation, but it is not the best fix because heartbeat pruning runs before recovery and ignores configured abort windows and queued-idle recovery state.
