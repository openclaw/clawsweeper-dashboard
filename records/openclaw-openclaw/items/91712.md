## What This Changes

The PR broadens diagnostic session-state pruning so entries older than the existing TTL are deleted even when they are not idle.

## Best Possible Solution

No. The proposed fallback is a plausible mitigation for ghost entries, but it is too broad; the safer fix must preserve recoverable queued state and configured recovery thresholds.
