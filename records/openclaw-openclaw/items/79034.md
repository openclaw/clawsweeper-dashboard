## Summary

Keep open: current main and the latest release still render several Control UI chat/tool-card metadata labels as hard-coded English, and the open linked PR is the right active fix path rather than a reason to close this issue early.

## What This Changes

Non-English Control UI users report that chat metadata and tool-card labels render as hard-coded English/raw technical labels instead of using the existing locale system.

## Best Possible Solution

Land or replace the focused linked PR so chat/tool-card metadata labels use existing Control UI locale keys while leaving transcript storage, gateway routing, and arbitrary debug payloads unchanged.

## Reproduction Assessment

Yes, by source inspection: rendering an expanded Control UI tool card or grouped tool message under a non-English locale still reaches hard-coded English labels in `ui/src/ui/chat/tool-cards.ts` and `ui/src/ui/chat/grouped-render.ts`. I did not run the UI because this review is read-only, so this is source-reproducible rather than locally executed.

## Solution Assessment

Yes in shape: routing the finite metadata labels through the existing `t()` helper and generated locale pipeline is the narrow maintainable fix. The issue should stay open until the linked PR is reviewed and merged or replaced.
