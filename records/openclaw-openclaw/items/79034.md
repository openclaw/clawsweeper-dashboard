## Summary

Keep open: current main still renders several Control UI chat metadata labels as hard-coded English, and a focused open PR already references this issue with closing syntax. The issue should remain open until that PR lands or maintainers choose a different fix path.

## What This Changes

The issue asks for Control UI chat metadata and technical labels to respect the selected non-English locale instead of showing raw English metadata in the chat view.

## Best Possible Solution

Land a focused fix equivalent to https://github.com/openclaw/openclaw/pull/79049: route finite chat metadata labels through the existing Control UI i18n tree, regenerate locale metadata, and add locale regression coverage without changing gateway routing or transcript storage.

## Solution Assessment

Yes. The narrow maintainable solution is to localize the finite user-facing chat metadata chrome through the existing i18n system; collapsing arbitrary raw debug JSON can be handled separately if maintainers want that UX change.
