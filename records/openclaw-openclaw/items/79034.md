## Summary

Current main already fixes the reported Control UI problem by stripping AI-facing inbound metadata from user-visible chat text rather than trying to localize raw internal JSON; the fix was merged and shipped in v2026.2.21.

## What This Changes

The issue asks for Control UI chat metadata such as sender labels and channel routing JSON to be localized or hidden for non-English users.

## Best Possible Solution

Keep AI-facing inbound metadata hidden from Control UI history; any future debug metadata view should be a separate explicit product design, not localized raw JSON in chat.

## Reproduction Assessment

No current-main failing reproduction: the Control UI user-message display path strips the exact inbound metadata blocks described here. Older release reports for the same raw JSON envelope were reproducible and are linked to the merged fix.

## Solution Assessment

Yes. Hiding AI-facing routing metadata at display boundaries is the narrow maintainable fix; localizing internal JSON labels would preserve a user-visible implementation detail that the code now treats as non-display content.
