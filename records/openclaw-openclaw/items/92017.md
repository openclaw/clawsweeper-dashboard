Correct the memory status diagnostic so migrated SQLite dreaming-ingestion state is recognized instead of reported absent.

## Best Possible Solution

Have the memory-core audit return an explicit ingestion-state source derived from the legacy file or workspace rows in the daily/session SQLite namespaces, then keep CLI and doctor projections aligned with regression coverage.

## Solution Assessment

Yes in direction: reuse memory-core’s existing workspace-scoped plugin-state API and represent the detected source explicitly. Raw SQL, restoring the archived sidecar, or adding a steady-state JSON fallback would conflict with the canonical SQLite ownership model.
