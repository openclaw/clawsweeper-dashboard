## Summary

Keep open. Current main still lacks a runner-side pre-prompt active-tool reapply, and open PR #75496 explicitly targets this issue with closing syntax but has not merged.

## What This Changes

The issue reports that Telegram-triggered direct Anthropic runs send an empty provider tools array even after the embedded runner computes the expected tool set.

## Best Possible Solution

Land PR #75496, or an equivalent runner-side pre-prompt active-tool restoration with regression coverage and live Telegram/Anthropic proof.

## Reproduction Assessment

Yes. The reporter provided concrete Telegram/direct-Anthropic steps and debug logs, and current main still builds tools before session setup while dispatching prompts without a reapply guard; I did not run a live Telegram repro, so this is source-reproducible rather than live-reproduced.

## Solution Assessment

Yes. Reapplying the runner-owned allowlist before prompt dispatch is the narrow maintainable fix; adding an Anthropic transport fallback for empty tools would mask legitimate no-tool runs.
