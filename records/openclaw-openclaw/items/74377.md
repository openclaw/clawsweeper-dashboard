## Summary

Keep open. Current main still has the runner/provider shape described by the report, and https://github.com/openclaw/openclaw/pull/75496 is an open closing PR for the focused fix path.

## What This Changes

Telegram Anthropic runs are reported to send `tools: []` even after the embedded runner computes the coding tool allowlist.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/75496 or an equivalent runner-side prompt-boundary reapply of the policy-filtered active tool allowlist while preserving raw no-tool probes.

## Reproduction Assessment

Yes, source-reproducible. The reporter provided concrete Telegram/direct-Anthropic steps and logs, and current source still shows the allowlist set before prompt dispatch while Anthropic forwards `context.tools`; I did not run the live Telegram setup.

## Solution Assessment

Yes. Reapplying the already policy-filtered Pi allowlist immediately before prompt dispatch is the narrow maintainable fix; a provider transport fallback for empty tools would not restore real `tool_use` blocks and could hide legitimate no-tool runs.
