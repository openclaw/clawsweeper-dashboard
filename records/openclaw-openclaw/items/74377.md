## Summary

Keep open. Current main still shows the Telegram-to-embedded-runner-to-Anthropic tools gap, and open PR #75496 explicitly targets this issue but has not merged, so closing now would race the active fix path.

## What This Changes

The issue reports that Telegram-triggered Anthropic runs send `tools: []` even though the embedded runner computed the expected coding tools.

## Best Possible Solution

Review and land PR #75496, or an equivalent narrow runner fix that reapplies the active tool allowlist before each prompt dispatch while preserving legitimate raw no-tool probes.

## Reproduction Assessment

Yes. The reporter provided concrete Telegram/direct-Anthropic steps and debug logs, and current source still shows tools built and allowlisted before `session.prompt()` while Anthropic forwards `context.tools` as received; I did not run a live Telegram repro, so this is source-reproducible rather than live-reproduced.

## Solution Assessment

Yes. A runner-side pre-prompt allowlist reapply is the narrow maintainable fix; adding an Anthropic transport fallback from empty tools would mask legitimate no-tool runs.
