## Summary

Keep open. Current main still has the reported Telegram to embedded-runner to Anthropic tool-delivery gap, and open PR #75496 explicitly closes this issue but has not merged, so this issue should remain open for that review and validation path.

## What This Changes

Telegram inbound runs using Anthropic should pass the runner's computed OpenClaw tools through to the provider instead of sending an empty tools array.

## Best Possible Solution

Review and land PR #75496, or an equivalent narrow runner fix that reapplies the active tool allowlist immediately before prompt dispatch while preserving legitimate raw no-tool probes.

## Reproduction Assessment

Yes. The reporter provided concrete Telegram/direct-Anthropic configuration and debug logs showing tools built in `attempt.ts` but empty at provider dispatch, and current main still has the same static gap before `session.prompt()`.

## Solution Assessment

Unclear until the open PR is reviewed and validated live. A runner-side pre-prompt allowlist reapply is the narrow plausible fix; adding an Anthropic transport fallback would risk masking legitimate no-tool runs.
