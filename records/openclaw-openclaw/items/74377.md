## What This Changes

The issue reports that Telegram runs with direct Anthropic lose the computed OpenClaw tools before provider dispatch, causing Anthropic to receive an empty tools array.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/75496, or an equivalent runner-side prompt-boundary reapply of the policy-filtered active tool allowlist, with redacted live Telegram/direct-Anthropic proof.

## Reproduction Assessment

Yes, source-reproducible. The reporter provided concrete Telegram/direct-Anthropic steps and logs, and current main still shows setup-time tool activation followed by direct context.tools forwarding; I did not run the live Telegram setup.

## Solution Assessment

Yes. Reapplying the already policy-filtered Pi allowlist immediately before prompt dispatch is the narrow maintainable fix; a provider transport fallback for empty tools would hide legitimate no-tool runs and would not restore real tool_use blocks.
