## Summary

Keep open. Current main still applies the Pi tool allowlist only at session creation and then calls `prompt()` without reapplying it; Anthropic serializes only the `context.tools` it receives. Open PR #75496 explicitly targets this report and is unmerged, so this issue should stay open until that fix or an equivalent one is validated and merged.

## What This Changes

The issue reports that Telegram/direct-Anthropic runs build 17 runner tools but send an empty tools array to Anthropic, preventing real `tool_use` blocks.

## Best Possible Solution

Review and land PR #75496 or an equivalent narrow runner fix that reapplies a non-empty session tool allowlist before Pi prompt dispatch while preserving legitimate raw no-tool runs.

## Reproduction Assessment

Yes. The issue gives version, channel, provider, config, and debug logs showing tools built in the runner but empty at provider dispatch; current main still statically matches that propagation gap.

## Solution Assessment

Unclear until the linked PR is validated live. Reapplying the session allowlist immediately before prompt dispatch is the narrow plausible fix, while an Anthropic transport fallback would risk masking legitimate no-tool probes.
