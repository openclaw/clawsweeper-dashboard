Keep open. Current main still has the reported Telegram/direct-Anthropic tool propagation gap, and an open linked PR already uses closing syntax for this issue, so the issue should remain open until that PR or an equivalent fix merges.

## What This Changes

The issue reports that Telegram-origin Anthropic runs send an empty provider tool array even though the embedded runner computed 17 OpenClaw tools.

## Best Possible Solution

Review and land PR #75496, or an equivalent focused fix, with regression coverage that preserves active Pi tools from Telegram embedded runner setup through Anthropic prompt dispatch without changing legitimate no-tool raw model probes.

## Reproduction Assessment

Yes. The issue includes version, platform, Telegram channel, direct Anthropic provider setup, config, and debug probes showing non-empty runner tools followed by empty Anthropic `context.tools`; current main inspection matches that vulnerable boundary.

## Solution Assessment

Yes. The proposed direction in the linked PR is the narrow maintainable path: restore the Pi active tool allowlist immediately before prompt dispatch and add targeted diagnostics/tests, rather than adding a broad Anthropic transport fallback that would mask legitimate no-tool calls.
