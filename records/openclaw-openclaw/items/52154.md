## Summary

Keep open because current main still lacks the requested config surface and this PR is the canonical implementation thread, but the patch still has source-visible merge blockers in shell defaults and strict-module cleanliness.

## What This Changes

The branch adds an optional `commands.shellProfile` config key and threads profile sourcing through exec runtime, PTY supervisor, chat bash, config schema/help, generated config docs, and shell utility tests.

## Best Possible Solution

Land a maintainer-approved opt-in profile overlay that preserves current deterministic shell args and fallbacks, keeps allowlist-enforced exec isolated from profile code, and refreshes schema/docs/tests after rebase.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not as a current-main bug; this is a feature PR. The blocking PR defects are source-reproducible from PR head: the unset-profile fallback drops current shell defaults and the supervisor imports an unused symbol.

## Solution Assessment

Is this the best way to solve the issue? No. The direction is plausible, but this patch should layer profile loading on top of current shell resolution/default args, remove stale code, rebase/regenerate cleanly, and receive product/security approval.
