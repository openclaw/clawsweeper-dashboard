## Summary

Keep open. Current main still does not implement `commands.shellProfile`, and this PR remains the canonical implementation thread, but the patch is not mergeable as-is because it changes default exec shell startup behavior, has a strict TypeScript unused import, and needs maintainer product/security approval for the new core exec config surface.

## What This Changes

The PR adds an optional `commands.shellProfile` config key plus schema/help artifacts, exec/chat/supervisor wiring, and shell utility tests to source a configured profile before host shell commands.

## Best Possible Solution

Land an opt-in host-only profile overlay only if maintainers accept the config and security model, while preserving existing no-profile/no-startup defaults and clearly documenting excluded sandbox, node, and enforced-allowlist paths.

## Reproduction Assessment

No, not as a current-main bug; this is a feature PR. The blocking defects are source-reproducible by comparing PR head's shell fallback and unused import with current main's deterministic shell defaults and strict TypeScript settings.

## Solution Assessment

No, not as-is. The direction is plausible, but the patch should preserve current shell defaults, remove stale code, rebase onto current main, and receive maintainer product/security approval for the new exec surface.
