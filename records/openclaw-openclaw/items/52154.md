Keep this PR open: the feature is not on current main, but the branch is not mergeable as-is because it regresses current shell startup defaults and still has a failing check from an unused import.

## Best Possible Solution

Land a rebased implementation that sources an explicit profile only on supported host-shell exec paths while preserving current no-profile defaults, allowlist enforcement, and documented sandbox/node exclusions.
