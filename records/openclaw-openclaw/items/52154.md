## Summary

Keep this PR open: current main still lacks the requested shell-profile configuration and the linked feature issues were closed in favor of this PR, but the current patch is not merge-ready because it regresses default shell startup behavior and contains an unused strict-TypeScript import.

## What This Changes

The PR adds a `commands.shellProfile` config option, schema/help/docs artifacts, exec/chat/supervisor wiring, and shell utility tests to source a configured profile before host shell commands.

## Best Possible Solution

Land an opt-in profile overlay only if maintainers approve the config and security model, preserving current deterministic defaults and documenting the host-only scope.

## Reproduction Assessment

No, not as a current-main bug; this is a feature PR. The blocking patch defects are source-reproducible from PR head by comparing its `getShellConfig(shellProfile?)` fallback with current main's shell defaults and by inspecting the unused supervisor import.

## Solution Assessment

No, not as-is. The direction is plausible, but the patch should preserve current default shell args outside the opt-in path, remove stale code, rebase cleanly, and receive product/security approval for the new exec config surface.
