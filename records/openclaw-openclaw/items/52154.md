## Summary

Keep open: current main still lacks `commands.shellProfile`, and the PR has blocking source-level defects in default shell behavior plus an unused import that keeps the patch from being merge-ready.

## What This Changes

The branch adds an optional `commands.shellProfile` config key, schema/help entries, tests, and exec/PTY/chat wiring to source a user-selected shell profile before host command execution.

## Best Possible Solution

Implement profile loading as an explicit overlay on current deterministic shell defaults, keep allowlist/sandbox/node boundaries documented, rebase generated config docs, and land only after focused tests plus maintainer security approval.

## Reproduction Assessment

Not applicable as a bug report because current main simply lacks `commands.shellProfile`. The blocking PR defects are source-reproducible from the PR-head code and a focused bash alias probe.

## Solution Assessment

No. The feature direction is plausible, but this patch is not the best merge shape until it preserves current shell defaults, fixes advertised alias behavior or narrows the docs, drops the unused import, and rebases onto current main.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Preserve current shell defaults when unset:** `src/agents/shell-utils.ts:108`
  - body: Current main suppresses bash/zsh/fish startup files and avoids placeholder `SHELL` values like `false` and `nologin`. This fallback returns the raw env shell or `sh` with bare `-c` when `shellProfile` is unset, so users who do not opt in lose existing deterministic exec behavior; build profile loading on top of the current defaults instead.
  - confidence: 0.9
- **[P2] Do not claim alias support with same-line sourcing:** `src/agents/shell-utils.ts:102`
  - body: For bash, aliases defined by the sourced profile are expanded before the profile is executed when the source prefix and user command are concatenated into the same `-c` string, so even a profile that enables `expand_aliases` leaves those aliases unavailable. Use a shell-specific pre-read hook or narrow the advertised support to env/functions.
  - confidence: 0.84
- **[P2] Drop the unused supervisor import:** `src/process/supervisor/supervisor.ts:3`
  - body: `createSubsystemLogger` is imported but never referenced, and the latest relevant `check` job fails on that unused identifier. Remove it or use it intentionally before rerunning CI.
  - confidence: 0.99
