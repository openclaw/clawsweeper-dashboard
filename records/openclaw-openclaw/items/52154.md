## Summary

Keep open: current main does not have this config option, and this PR is still the active implementation path. The branch needs rework because it drops current deterministic shell defaults when no profile is configured, leaves an unused import, and introduces a command-execution config that needs maintainer product/security review.

## What This Changes

Adds an optional `commands.shellProfile` config field and threads it through host exec, PTY, chat bash, config metadata, generated docs, and shell utility tests to source a profile before commands.

## Best Possible Solution

Land a maintainer-approved opt-in profile overlay that preserves deterministic default shell args and fallback behavior, with clear documented exclusions for sandbox, node host, and enforced allowlist paths.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature PR. The blocking PR defects are high-confidence from source inspection of PR head against current main's shell defaults and supervisor imports.

## Solution Assessment

No. The direction is plausible, but this patch should layer profile sourcing on top of current `getShellConfig()` defaults, remove the unused import and stale generated artifacts, and get maintainer product/security approval before merge.
