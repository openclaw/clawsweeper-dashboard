## Summary

## What This Changes

Adds a `commands.shellProfile` config option and wires it through exec, PTY supervisor, chat bash, config metadata, generated docs, and shell utility tests to source an absolute shell profile before host shell commands.

## Best Possible Solution

Land a maintainer-approved design that sources an explicit profile only as an opt-in overlay on the current deterministic shell defaults and documented host boundaries.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature PR. The blocking PR defects are source-reproducible by comparing PR head against current main's shell defaults and supervisor imports.

## Solution Assessment

No. The feature direction is plausible, but this patch should preserve unset shell defaults, remove the unused import, rebase generated docs, and receive maintainer product/security review before merge.
