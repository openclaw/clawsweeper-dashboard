## Summary

The feature is not implemented on current main, but this PR should stay open rather than close because it still contains source-proven P2 blockers in the shell startup path and supervisor import cleanup.

## What This Changes

The PR adds optional `commands.shellProfile` config and threads profile sourcing through exec, PTY supervisor, chat bash, config schema/help, and shell utility tests.

## Best Possible Solution

Land a maintainer-approved opt-in profile overlay that preserves deterministic default shell args and fallbacks, keeps allowlist-enforced exec from sourcing profile code, documents host exclusions clearly, and updates schema/docs/tests without stale generated artifacts.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not as a user bug; this is a feature PR. The blocking defects are source-reproducible by comparing PR head `getShellConfig()` and supervisor imports against current main.

## Solution Assessment

Is this the best way to solve the issue? No. The feature direction is plausible, but the patch should preserve current shell defaults when unset, remove the unused import, rebase away stale generated artifacts, and get maintainer product/security approval before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve current shell defaults when profile is unset:** `src/agents/shell-utils.ts:108`
  - body: When `shellProfile` is unset or unsupported, this fallback returns the selected shell with bare `-c`, dropping current main's bash/zsh/fish startup-file suppression and placeholder-shell fallback.
