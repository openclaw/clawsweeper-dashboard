## Summary

Keep open: current main still lacks the requested opt-in shell profile capability, but this PR is not mergeable as-is because PR head regresses default shell startup behavior and has an unused import in a strict TypeScript path.

## What This Changes

The PR adds a `commands.shellProfile` config option plus schema/help artifacts, exec/chat/supervisor wiring, and shell utility tests to source a configured shell profile before host shell commands.

## Best Possible Solution

Land an opt-in host-only profile overlay only if maintainers accept the config and security model, while preserving existing no-profile/no-startup defaults and documenting excluded paths such as enforced allowlist, sandbox, and node execution.

## Reproduction Assessment

No, not as a current-main bug; this is a feature PR. The merge-blocking defects are source-reproducible by comparing PR head's shell fallback with current main and by inspecting the unused supervisor import.

## Solution Assessment

No, not as-is. The direction is plausible, but the patch should preserve current shell defaults, remove stale code, rebase onto current main, and receive maintainer product/security approval for the new exec surface.
