## Summary

Keep open: current main still does not implement `commands.shellProfile`, and this PR remains the canonical implementation thread, but the PR head has blocking exec-surface regressions and is currently conflicting/dirty.

## What This Changes

The PR adds an opt-in `commands.shellProfile` config key, wires it through host command execution paths, and updates config schema/help plus exec tests.

## Best Possible Solution

Rebase the feature onto current main, keep profile loading opt-in and host-only, preserve existing no-profile/no-rc and placeholder-shell safeguards when unset, and rerun focused exec/config proof.

## Reproduction Assessment

Not applicable; this PR adds a new opt-in exec configuration feature, not a documented current-main bug. Source inspection shows current main has no `shellProfile` config key, while the PR screenshots show the intended Windows/PowerShell before/after behavior.

## Solution Assessment

No. Opt-in profile loading is plausible, but this branch is not merge-ready because it drops current default shell hardening and carries an unused supervisor import.
