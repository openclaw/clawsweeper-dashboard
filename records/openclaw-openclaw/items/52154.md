## Summary

Keep this PR open: current main still lacks the requested `commands.shellProfile` config surface, and the linked feature issue was already closed in favor of this PR as the canonical thread. The patch is not merge-ready because PR head still changes default shell startup behavior for users who did not opt in and has a failing `check` job from an unused import.

## What This Changes

The PR adds an optional `commands.shellProfile` config key and threads profile loading through host exec, chat bash, supervisor PTY, schema/help/generated baselines, and shell utility tests.

## Best Possible Solution

Land a rebased implementation that sources an explicitly configured profile only on supported host shell paths while preserving main's no-profile/no-config defaults, allowlist enforcement, and documented sandbox/node exclusions.

## Solution Assessment
