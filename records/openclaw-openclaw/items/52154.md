## Summary

Keep open: current main still does not implement `commands.shellProfile`, and the linked feature issue was already closed in favor of this PR, but the PR head still has merge-blocking defects in the shell startup path and CI.

## What This Changes

The PR adds an optional `commands.shellProfile` config key and wires explicit profile sourcing through host exec, chat bash, PTY spawning, schema/help, generated baselines, and shell utility tests.

## Best Possible Solution

Land a rebased implementation that sources an explicit profile only on supported host-shell paths while preserving current no-profile defaults, allowlist bypass protection, and documented sandbox/node exclusions.
