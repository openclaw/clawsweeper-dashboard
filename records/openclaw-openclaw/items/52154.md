Keep this PR open: current main does not implement `commands.shellProfile`, and the related feature issues point to this PR as the active thread, but the PR head still has blocking correctness/security issues and needs maintainer review for the new command-execution config surface.

## What This Changes

The PR adds an optional `commands.shellProfile` config key, schema/help entries, tests, and exec/PTY/chat wiring to source a user-selected shell profile before host command execution.

## Best Possible Solution

Rebase this PR onto current main, implement profile loading as an opt-in overlay on the existing `getShellConfig()` defaults, and land it only after focused tests, schema/docs alignment, and maintainer security review.
