## Summary

Keep open: current main does not implement `commands.shellProfile`, and this PR has useful Windows proof, but the patch still changes default shell startup behavior for users who did not opt in and includes an unused import. The new core command profile sourcing surface also needs maintainer product/security approval before merge.

## What This Changes

The PR adds an optional `commands.shellProfile` config key with schema/help updates, shell profile prefixing helpers, exec/supervisor/chat bash plumbing, generated config docs, and tests.

## Best Possible Solution

Land a maintainer-approved opt-in profile overlay that composes with current shell argument selection and fallbacks, preserves allowlist-enforced exec safety, documents host exclusions, and regenerates config docs from current main.

## Reproduction Assessment

Yes for the PR defects, not as a user bug: comparing the PR diff with current main shows the fallback drops default shell args and adds an unused supervisor import. The requested feature itself has Windows screenshot proof but no Linux/macOS real behavior proof.

## Solution Assessment

No. The feature direction is plausible, but the patch is not mergeable until it preserves current default shell behavior, removes the unused import, rebases away stale generated artifacts, and receives maintainer approval for the command-execution policy change.
