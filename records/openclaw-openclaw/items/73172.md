## Summary

Keep open. Current main still has the iMessage DM-history contract mismatch described in discussion: config and docs expose `channels.imessage.dmHistoryLimit`, but the iMessage runtime history path is group-only, and the linked docs-honest PR was closed unmerged.

## What This Changes

The issue asks iMessage `dmHistoryLimit` to provide decoded direct-message history instead of blank outbound history entries caused by relying on `chat.db` text.

## Best Possible Solution

Make the iMessage contract honest: either implement decoded direct-message history through verified `imsg` history/RPC output with outbound-message coverage, or remove iMessage from the documented supported list while keeping schema compatibility.

## Reproduction Assessment

Yes for the current-main contract mismatch: source inspection shows `channels.imessage.dmHistoryLimit` is accepted and documented, while iMessage only builds history for groups. The narrower blank outbound `attributedBody` symptom still needs live macOS or dependency-backed verification.

## Solution Assessment

Unclear. A decoded `imsg` history/RPC adapter is the right runtime fix if iMessage DM history is intended, but the docs-honest correction is the narrower maintainable path if that runtime support is not intended yet.
