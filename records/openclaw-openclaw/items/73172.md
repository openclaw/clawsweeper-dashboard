## Summary

Keep open. Current main still has the advertised iMessage DM-history contract/runtime mismatch, and the only linked fix PR was closed unmerged; the original attributedBody observation remains useful implementation guidance rather than proof of an already-fixed path.

## What This Changes

The issue asks for iMessage `dmHistoryLimit` to inject decoded direct-message history, including outbound Messages content stored in `attributedBody`, instead of blank context.

## Best Possible Solution

Make the iMessage contract explicit: either implement decoded direct-message history in `extensions/imessage` using verified `imsg` history/RPC output, or remove iMessage from supported DM-history docs while keeping the config field backward-compatible.

## Reproduction Assessment

Yes for the current-main gap: source inspection shows iMessage config/docs expose `dmHistoryLimit`, while the runtime history key and `InboundHistory` construction are group-only. The original macOS database-level `attributedBody` symptom still needs live imsg verification.

## Solution Assessment

Unclear. A decoded `imsg` adapter is the right fix if iMessage DM history is intended, but a docs/contract correction is the narrower maintainable path if maintainers do not want to support that runtime behavior yet.
