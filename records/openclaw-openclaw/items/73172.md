## Summary

Keep open. Current main still accepts and documents iMessage direct-message history, but the iMessage runtime only builds group history; the related docs-honest PR was closed unmerged, so neither acceptable fix path has landed.

## What This Changes

The issue requests that iMessage dmHistoryLimit use decoded imsg history/RPC output so outbound direct-message history is not blank.

## Best Possible Solution

Make the iMessage contract honest: either implement decoded direct-message history through verified imsg history/RPC output, or remove iMessage from the documented supported list while preserving schema compatibility.

## Reproduction Assessment

Yes for the current-main contract mismatch: configure channels.imessage.dmHistoryLimit, then source inspection shows the field is accepted and documented but not used by the iMessage monitor for direct-message InboundHistory. The exact macOS attributedBody storage behavior still needs live or dependency-backed verification.

## Solution Assessment

Unclear. A decoded imsg history adapter is the best runtime fix if iMessage DM history is intended, while a docs-honest correction is the narrowest maintainable path if it is not intended.
