## Summary

Keep open. Current main still exposes and documents iMessage `dmHistoryLimit`, but the iMessage runtime only resolves/builds group-history context; the prior docs-only fix PR #73775 was closed unmerged.

## What This Changes

The issue requests fixing iMessage `dmHistoryLimit` so direct-message history includes decoded outbound Messages content instead of blank entries.

## Best Possible Solution

Make the iMessage DM-history contract honest: implement decoded direct-message history through verified `imsg` history/RPC output, or remove iMessage from the documented supported list while preserving schema compatibility.

## Reproduction Assessment

Yes for the current-main gap: configure `channels.imessage.dmHistoryLimit`, then source inspection shows the value is accepted and documented but never used by the iMessage monitor for direct-message `InboundHistory`. The original database-level `attributedBody` claim still needs a macOS/imsg live check.

## Solution Assessment

Unclear. A decoded `imsg` adapter is the right solution if legacy iMessage DM history is intended, while a docs/support correction is the narrower maintainable path if it is not intended.
