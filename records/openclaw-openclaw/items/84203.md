## Summary

Keep open: current main and the latest stable release still have the reported 60s process-local `models.authStatus` cold path that reads Codex CLI auth and awaits OpenAI/Codex usage enrichment, while the reporter’s 2026.5.19 follow-up says latency improved but still remains double-digit and Telegram can multiply it.

## What This Changes

The issue asks OpenClaw to eliminate Windows Codex-backed cold CLI sessions where `models.authStatus` adds 10-24s before a trivial agent response.

## Best Possible Solution

Keep this as the canonical Windows/Codex `models.authStatus` latency issue, profile the current Windows OAuth path, then make normal status reads cheap through bounded coalescing or background refresh while preserving explicit refresh and invalidation.

## Reproduction Assessment

No high-confidence live reproduction was run, but the reporter provided concrete Windows steps and timings, current source still contains the matching cold path, and upstream Codex confirms the Windows daemon workaround is unavailable.

## Solution Assessment

Keeping it open is the best current outcome; the repair should be trace-guided because the safe cache/coalescing shape depends on auth freshness, usage enrichment, and explicit refresh behavior.
