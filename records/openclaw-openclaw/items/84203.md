## Summary

Keep open: the reporter’s 2026.5.21 update says the newer OpenClaw/Codex combination substantially improves the cold session regression, but not to the requested under-10s behavior and Telegram remains a large latency multiplier. Current main still has the same gateway `models.authStatus` cache/cold path, so there is not enough evidence to close this as implemented or superseded.

## What This Changes

The issue reports Windows Codex-backed CLI cold sessions spending 10-24s in `models.authStatus` and 27-45s total on 2026.5.18, with a later note that 2026.5.19 plus `@openclaw/codex` 0.132.0 improves but does not fully remove the latency when Telegram is enabled.

## Best Possible Solution

Profile current main on Windows with Codex OAuth and Telegram enabled/disabled, then make normal `models.authStatus` and startup metadata reads cheap through coalescing or background refresh while preserving explicit refresh and invalidation.

## Reproduction Assessment

No, not as a high-confidence live reproduction. The issue has concrete Windows commands and timings, and current source confirms the same cache/cold auth-status path, but this review did not run a Windows/Codex OAuth/Telegram reproduction against current main.

## Solution Assessment

Yes for keeping it open, but no current fix is proven complete. The maintainable path is trace-guided cache/coalescing/background refresh work rather than closing based on the Codex dependency improvement or only raising the TTL.
