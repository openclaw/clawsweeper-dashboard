Keep open: this is a recent external Windows/Codex regression report with concrete timings, and current `main` plus the latest shipped tag still show the same blocking `models.authStatus` cold path and 60s in-process cache shape. ## What This Changes

The issue asks to fix Windows cold CLI sessions where `models.authStatus` adds 10-24s of auth/provider latency on each Codex-backed CLI run.

## Best Possible Solution

Make normal `models.authStatus` reads cheap by serving a usable cached or stale snapshot while coalescing/background-refreshing heavy auth and usage work, preserve explicit refresh and logout/repair invalidation, and prove the Windows Codex OAuth CLI path before landing.
