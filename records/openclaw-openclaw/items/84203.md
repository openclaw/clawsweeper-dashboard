## Summary

Keep open: the issue is recent and detailed, current main still has the cold `models.authStatus` cache-miss path, and the reporter's 2026.5.19/Codex 0.132.0 update says the regression is improved but not back under the expected latency, with Telegram still multiplying the delay.

## What This Changes

The issue asks OpenClaw to stop cold Windows Codex-backed CLI sessions from spending 10-24s in `models.authStatus` on each session.

## Best Possible Solution

Keep the issue open, profile the Windows Codex OAuth CLI path with and without Telegram enabled, then make normal `models.authStatus` reads cheap by coalescing or background-refreshing heavy auth/usage work while preserving explicit refresh and invalidation.

## Reproduction Assessment

No, we do not have a high-confidence live reproduction. The reporter supplied concrete Windows timings and current source shows the cold auth-status path, but this review did not run a Windows 11 npm-global Codex OAuth session.

## Solution Assessment

Yes for keeping it open. The best repair is not simply increasing a constant; it should remove blocking heavy work from normal status reads while preserving freshness for explicit refresh, auth logout, and token changes.
