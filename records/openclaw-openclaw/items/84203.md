## Summary

Keep open: this is a fresh, detailed external regression report, and current `main` still has the same cold `models.authStatus` cache and auth/usage enrichment path that matches the reported timing pattern. A related Control UI latency issue exists, but this report has distinct Windows CLI plus Codex OAuth evidence and should not be closed as a duplicate.

## What This Changes

The issue reports a Windows 11 regression where Codex-backed CLI agent invocations spend 10-24s in cold `models.authStatus`, making trivial runs take 27-45s.

## Best Possible Solution

Profile the Windows Codex OAuth CLI path, then make normal `models.authStatus` reads cheap through coalescing, bounded enrichment, or background refresh while preserving explicit refresh/logout invalidation.

## Reproduction Assessment

No, we do not have a high-confidence live reproduction. The reporter provided concrete Windows steps and timings, and current source confirms the 60s cache plus cold auth/usage path, but no current-main Windows/Codex OAuth run was performed here.

## Solution Assessment

Unclear as a code fix because there is no patch and the bottleneck has not been isolated. The maintainable direction is to remove blocking cold `models.authStatus` work from normal reads, not just raise a TTL constant without freshness proof.
