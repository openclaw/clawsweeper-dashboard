The issue asks OpenClaw to restore fast Windows Codex CLI cold-session latency by removing 10-24s `models.authStatus` cold auth/status work.

## Best Possible Solution

Keep the issue open and make normal `models.authStatus` reads cheap through coalesced or background auth/usage refresh with explicit refresh/invalidation preserved, then profile the Windows Telegram multiplier separately.

No, there is not a high-confidence live reproduction from this review. The reporter supplied concrete Windows timings and current source confirms the same 60s cache plus cold auth/usage path, but the Windows Codex OAuth and Telegram multiplier path still needs live profiling.
