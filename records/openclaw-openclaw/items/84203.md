## Summary

Keep open: current main and the latest stable release still keep `models.authStatus` on a 60s process-local cache and the cold path still waits on auth-profile loading plus provider usage enrichment. The reporter’s 2026-05-21 update shows partial improvement after OpenClaw 2026.5.19 plus `@openclaw/codex` 0.132.0, not a full resolution, and the remaining Windows/Codex/Telegram latency split still needs live profiling and a maintainer product call.

## What This Changes

The issue asks OpenClaw to fix Windows cold `models.authStatus` latency that makes each Codex-backed CLI agent session take tens of seconds after the cache expires.

## Best Possible Solution

Keep `models.authStatus` fast for cold reads by separating mandatory auth health from optional provider usage enrichment or by warming/coalescing the Codex usage snapshot, with Windows and Telegram before/after proof.

## Reproduction Assessment

No high-confidence live repro was established in this read-only review; source inspection confirms the cold cache-miss path and the reporter supplied concrete Windows timings, but current-main Windows profiling is still missing.

## Solution Assessment

Unclear; a timeout or TTL tweak alone is probably not the best fix until maintainers decide whether usage enrichment should block auth status or move to a warmer/background path.
