## Summary

Keep open: the original cold-session regression is substantially improved per the reporter’s 2026-05-21 update, but current `main` still has the same 60s `models.authStatus` cache and cold auth/usage enrichment path, and the remaining Windows Codex plus Telegram latency is not isolated or proven fixed.

## What This Changes

The issue reports a Windows 11 regression where cold OpenClaw CLI agent sessions using the Codex plugin and OAuth wait on slow `models.authStatus` work before producing a trivial response.

## Best Possible Solution

Profile current `main` on Windows with Codex OAuth and Telegram enabled/disabled, then make normal `models.authStatus` reads cheap through coalescing or background refresh while preserving explicit refresh and auth invalidation semantics.

## Reproduction Assessment

No, not at high confidence from this review alone. The reporter provided concrete Windows steps and timings, and current source confirms the cold cache/auth/usage path, but no current-main Windows Codex OAuth run was performed.

## Solution Assessment

Unclear because there is no patch and the remaining bottleneck is not isolated. A TTL-only change would be risky; the better direction is trace-guided coalescing/background refresh plus Telegram on/off proof.
