## Summary

Keep open: current main still has the 60s `models.authStatus` cache and cache-miss path that synchronously builds auth status and awaits usage enrichment, and the related open reports do not supersede this Windows Codex CLI regression.

## What This Changes

Reports Windows 11 cold Codex-backed CLI sessions where `models.authStatus` takes 10-24s and trivial `openclaw agent` runs take 27-45s after upgrading to 2026.5.18.

## Best Possible Solution

Profile the current Windows Codex OAuth CLI path, then make routine `models.authStatus` reads cheap by coalescing or background-refreshing heavy auth/usage work while preserving explicit refresh and invalidation.

## Reproduction Assessment

No, we do not have a high-confidence live reproduction. The reporter provided concrete Windows timings and current source confirms the cache-miss auth/usage path, but no current-main Windows Codex OAuth run was profiled here.

## Solution Assessment

Unclear until live profiling isolates the dominant cold phase. The likely maintainable direction is to remove blocking heavy work from normal status reads rather than only raising the TTL.
