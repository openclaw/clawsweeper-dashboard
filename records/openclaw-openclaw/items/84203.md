## What This Changes

The issue reports a Windows Codex OAuth regression where each cold CLI session waits 10-24s in `models.authStatus`, causing trivial agent runs to take 27-45s.

## Best Possible Solution

Keep this open, profile the Windows Codex OAuth CLI path, and make normal `models.authStatus` reads cheap through coalescing or background refresh while preserving explicit refresh, logout invalidation, and auth-freshness semantics.

## Solution Assessment
