## What This Changes

The issue reports a Windows 11 regression where cold Codex-backed CLI sessions spend 10-24s in `models.authStatus`, making trivial `openclaw agent` runs take 27-45s.

## Best Possible Solution

Make normal `models.authStatus` reads cheap through coalesced or background auth/usage refresh with explicit refresh/invalidation, then profile the Windows Codex CLI path with Telegram enabled and disabled.

## Reproduction Assessment

No, not with high confidence. The reporter supplied concrete Windows timings and a 2026.5.19 workaround update, and current source confirms the cache/cold path, but this review did not run a Windows/Codex OAuth CLI reproduction.

## Solution Assessment

Yes for keeping it open. The narrow maintainable fix is to stop ordinary status reads from blocking on heavy cold auth/usage work, while preserving explicit refresh and separately validating the Telegram multiplier.
