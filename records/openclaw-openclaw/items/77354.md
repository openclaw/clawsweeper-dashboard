## Summary

The reported broken Discord package/manifest state does not match current main or the live `@openclaw/discord@2026.5.3` npm artifact, and the current startup planner does not require `activation.onStartup=true` for configured channel owners.

## What This Changes

The issue reports that `@openclaw/discord@2026.5.3` loads as a plugin but never starts its Discord channel because package and manifest metadata appear incomplete.

## Best Possible Solution

Keep the current runtime-extension packaging, generated channel metadata, and configured-channel startup path; if this reappears, triage it as a stale local managed npm install or cache mismatch with the exact installed package files.

## Reproduction Assessment

No. The reported broken artifact conditions are absent from the live `@openclaw/discord@2026.5.3` tarball, and current source selects configured channel owners through manifest `channels` before considering `activation.onStartup`.

## Solution Assessment

Yes. Closing as not reproducible is the narrow maintainer outcome because changing `activation.onStartup` is not required for a configured channel owner and the current artifact already carries the runtime and channel metadata.
