## What This Changes

The branch changes isolated cron finalization to keep successful agent runs at `status: "ok"` when delivery dispatch returns an error, preserves delivery failure metadata, and adds a regression test.

## Best Possible Solution

Narrow the reclassification to real post-run delivery-send failures, keep `delivery-target` refusals as errors, add real cron/gateway or CLI proof, and get maintainer acceptance of the interim `ok` plus `not-delivered` semantics.
