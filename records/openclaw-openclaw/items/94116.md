## What This Changes

The PR changes isolated cron finalization so a successful agentTurn followed by a delivery dispatch error returns `status: "ok"` with delivery failure metadata, and adds a regression test for that path.

## Best Possible Solution

Narrow the finalizer change to real post-run delivery-send failures, keep `delivery-target` refusals as errors, preserve delivery error metadata, and add real cron/gateway or CLI proof before maintainers choose the interim ok/not-delivered model.
