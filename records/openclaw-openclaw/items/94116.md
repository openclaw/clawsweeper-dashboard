## Summary

## What This Changes

The PR changes isolated cron finalization so a post-run delivery error can preserve an otherwise successful isolated agent run as `ok` while carrying delivery error metadata and adds a regression test for that branch.

## Best Possible Solution

Land the focused finalization fix only after contributor-provided real cron proof and maintainer acceptance of the interim `ok` plus delivery metadata model, or defer to the broader warning-status contract if that is the preferred product shape.

## Reproduction Assessment

Yes, source-reproducible: current main can return a dispatcher `status: "error"` after successful isolated payload resolution, and the scheduler persists that returned status as the parent run status. I did not live-reproduce the original WSL2/systemd/Feishu scheduled run.

## Solution Assessment

Yes as an acceptable narrow mitigation, but not the full product solution. The best long-term model may be the separate public warning status tracked by https://github.com/openclaw/openclaw/issues/49190; this PR is a smaller fix that preserves existing status values.
