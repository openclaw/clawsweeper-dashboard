## Summary

## What This Changes

The PR changes isolated cron finalization so successful agent execution stays `status: "ok"` when post-run delivery dispatch fails, preserves delivery failure metadata, and adds a regression test.

## Best Possible Solution

Narrow the finalizer reclassification to non-target delivery-send failures, preserve delivery error text through persisted cron state and run logs, add real cron/gateway or CLI proof, and have maintainers choose between this interim `ok` plus `not-delivered` model, the stronger https://github.com/openclaw/openclaw/pull/95419 path, or the broader warning-status contract.
