## Summary

Keep this PR open, but not merge-ready: it targets a real doctor duplicate-entry problem, yet the proposed skip conflicts with current plugin trust policy and can leave configured external channel owners inactive after doctor runs.

## What This Changes

The branch changes doctor plugin auto-enable to treat global/workspace manifest-channel plugins as already enabled and adds origin-aware helper support plus channel auto-enable tests.

## Best Possible Solution

Change doctor so it avoids duplicate config-selected plugin records while still preserving explicit allowlist/activation trust for external manifest-channel owners, then prove both doctor output and startup owner resolution.

## Reproduction Assessment

Yes, by source inspection: with the proposed branch, a configured global/workspace manifest-channel plugin can skip entry and allowlist creation while current startup ownership requires that explicit trust. I did not run a writable doctor/gateway setup because this review is read-only.

## Solution Assessment

No. Avoiding duplicate entries is the right target, but this implementation skips the trust materialization needed by current plugin activation policy; the safer fix is to avoid the duplicate entry while preserving allowlist or equivalent activation trust.
