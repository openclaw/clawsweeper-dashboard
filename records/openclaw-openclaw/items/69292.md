## Summary

Keep this PR open, but the proposed patch is not ready. The duplicate-entry bug is core doctor behavior and the PR targets the right function, but the skip also drops the explicit enablement/trust signal that current startup and channel-presence policies require for non-bundled channel owners.

## What This Changes

The PR changes doctor plugin auto-enable to skip writing `plugins.entries` for global/workspace manifest-channel plugins and adds origin-aware tests.

## Best Possible Solution

Fix doctor in core by preventing duplicate registry materialization while preserving a durable explicit trust/activation signal for non-bundled channel owners, then cover doctor output plus startup/presence behavior in regression tests.

## Reproduction Assessment

Yes, by source inspection: apply plugin auto-enable for a configured global/workspace manifest-channel plugin, then evaluate current startup and channel-presence policy against the resulting config. The PR’s skipped entry leaves no current explicit trust signal for those non-bundled owners.

## Solution Assessment

No. The PR fixes the right layer, but configured channel presence is not currently equivalent to explicit plugin trust; the safer solution must separate duplicate-entry avoidance from trust preservation.
