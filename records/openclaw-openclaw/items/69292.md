## Summary

Keep open: this targets a real doctor duplicate-entry bug, but the current patch can skip the explicit trust signal that non-bundled channel owners need at startup, and it still lacks real doctor/gateway proof.

## What This Changes

The PR changes doctor auto-enable to skip writing `plugins.entries` for global/workspace manifest-channel plugins and adds helper/test coverage for global, workspace, and bundled origins.

## Best Possible Solution

Fix doctor’s duplicate materialization for installed manifest-channel plugins while preserving explicit activation or allowlist trust, then cover both doctor output and startup owner resolution.

## Reproduction Assessment

Yes. By source inspection, the PR branch can return `alreadyEnabled` and skip entry registration/allowlisting, while current startup/channel policies reject non-bundled configured-channel owners without explicit trust; I did not run a writable doctor/gateway setup in this read-only review.

## Solution Assessment

No. Doctor is the right layer, but this implementation should avoid duplicate materialization without removing explicit enablement or allowlist trust; changing channel config into a trust signal would need policy/docs/tests as an intentional design change.
