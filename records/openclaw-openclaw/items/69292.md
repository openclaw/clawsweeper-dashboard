## Summary

Keep open: this is a plausible core doctor fix, but the current patch is not merge-ready because current startup policy still requires explicit trust for non-bundled channel owners and the branch skips the path that records that trust.

## What This Changes

The PR changes doctor plugin auto-enable so global or workspace manifest-channel plugins with configured channels are treated as already enabled and no longer get a generated `plugins.entries` entry.

## Best Possible Solution

Fix duplicate doctor materialization while preserving explicit plugin trust, or make a deliberate startup trust-policy change with docs and tests; the adjacent silent-channel diagnostic work is already tracked at https://github.com/openclaw/openclaw/issues/83212.

## Reproduction Assessment

Yes, by source inspection: the new `alreadyEnabled` path can skip both entry creation and allowlisting, while current startup code and tests reject non-bundled configured-channel owners without explicit selection. I did not run a writable doctor/gateway setup because this review is read-only.

## Solution Assessment

No. Doctor is the right layer, but this branch treats configured global/workspace channels as already safe without preserving the explicit activation or allowlist trust that current startup planning requires.
