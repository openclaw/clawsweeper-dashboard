## Summary

Keep open. Current main still has only the shallow HTTP liveness/readiness probes plus separate model/status RPC surfaces; it does not provide the requested native auth-gated no-inference model-contract endpoint, and the proposed route is recent, concrete, and security/API-sensitive enough to need maintainer direction before implementation.

## What This Changes

Request to add a native auth-gated Gateway HTTP `/health/deep` probe that reports the active backend model, provider, fallbacks, uptime, and future routing/error telemetry without invoking inference.

## Best Possible Solution

Keep this open for maintainer API and security direction. The likely end state is an additive Gateway diagnostics/readiness surface with a narrow no-inference model-resolution helper and dedicated auth/no-leak tests, or a documented plugin/runtime seam if maintainers decide this should remain external without reaching through Gateway internals.

## Reproduction Assessment

Not applicable as a feature/API proposal rather than a bug. Current-main code inspection gives a high-confidence verification path: `/health/deep` is absent from the probe map, readiness/model HTTP payloads do not expose the requested fields, and unhandled paths fall through to 404.

## Solution Assessment

Unclear. A native deep-health endpoint may be the right shape, but maintainers still need to choose the path, auth semantics, field contract, telemetry wiring, and whether the implementation should be core-owned or enabled by a plugin/API seam.
