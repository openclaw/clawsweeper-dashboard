## Summary

Keep open. Current main still lacks a native no-inference model-contract probe, and the remaining choices are API/security contract decisions that should be made by maintainers before code lands.

## What This Changes

Request a native authenticated no-inference Gateway probe, proposed as `/health/deep`, that returns active provider/model/fallback and routing/error metadata for external model-contract checks.

## Best Possible Solution

Keep this as a maintainer design item and land an additive Gateway diagnostics/readiness contract only after the route, auth, disclosure, redaction, telemetry storage, and field contract are agreed.

## Reproduction Assessment

Not applicable as a feature/API proposal. Source inspection gives a high-confidence verification path: `/health/deep` is absent, unknown probe paths fall through to 404, and current readiness/model surfaces do not expose the requested payload.

## Solution Assessment

Unclear. A native endpoint may be the right direction, but the safest implementation depends on maintainer decisions about route naming, disclosure policy, response fields, telemetry wiring, and whether startup logging should be split out.
