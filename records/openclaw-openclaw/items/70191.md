## Summary

Keep open. Current main still lacks a native no-inference model-contract HTTP probe, and the downstream overlay is useful prior art rather than an implementation in OpenClaw core; the remaining work needs Gateway API and disclosure-policy decisions before code should land.

## What This Changes

The issue proposes a native Gateway HTTP endpoint that reports active model, provider, fallbacks, uptime, and routing/error diagnostics without invoking inference.

## Best Possible Solution

Keep this as a maintainer design item and, if accepted, land an additive Gateway diagnostics/readiness contract with explicit auth, redaction, no-inference behavior, tests, and docs.

## Reproduction Assessment

Not applicable as a feature/API proposal rather than a bug. Source inspection gives a high-confidence verification path: the requested endpoint and fields are absent from current main, and existing probe routes fall through to the normal 404 path when unrecognized.

## Solution Assessment

Unclear. A native endpoint may be reasonable, but the best implementation depends on maintainer decisions about route placement, disclosure policy, response schema, telemetry wiring, and whether this belongs under readiness or a separate diagnostics surface.
