## Summary

Keep open. Current main still lacks the requested auth-gated, no-inference `/health/deep` model-contract endpoint, and the proposal is recent, concrete, and security/API-sensitive enough to need maintainer direction rather than cleanup closure.

## What This Changes

The issue requests a native Gateway `/health/deep` endpoint that exposes the active backend model, provider, fallbacks, uptime, and bounded routing/error telemetry without invoking inference.

## Best Possible Solution

Keep the issue open for maintainer API and security direction. The likely end state is either an additive Gateway probe/readiness-detail extension with a narrow no-inference model-resolution helper and auth/no-leak tests, or a documented plugin SDK/runtime seam that lets this be implemented externally without reaching into Gateway internals.

## Reproduction Assessment

Not applicable as a feature/API proposal rather than a bug. The current-main verification path is high-confidence code inspection: `/health/deep` is absent from the probe map, and readiness/model HTTP payloads do not expose the requested contract fields.

## Solution Assessment

Unclear. The user problem is concrete, but the best shape still needs maintainer choice between a native `/health/deep`, extending `/ready` details, an observability/diagnostics route, or first adding a plugin/API seam with equivalent auth and model-resolution semantics.
