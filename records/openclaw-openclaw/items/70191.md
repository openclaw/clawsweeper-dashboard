## Summary

Keep open. Current main still lacks a native no-inference model-contract probe, and the remaining work is a Gateway API/security design decision rather than conservative cleanup.

## What This Changes

The issue proposes a native auth-gated `/health/deep` Gateway probe that reports active provider/model/fallback state without invoking inference.

## Best Possible Solution

Keep this as a maintainer design item, then land an additive Gateway diagnostics/readiness contract with tests and docs after route, auth, disclosure, and telemetry semantics are settled.

## Reproduction Assessment

Not applicable as a feature/API proposal rather than a bug. Source inspection on current main gives a high-confidence verification path: `/health/deep` is absent, existing readiness/model surfaces do not expose the requested payload, and unknown paths fall through to 404.

## Solution Assessment

Unclear. A native endpoint may be the right direction, but the safest solution depends on maintainer decisions about route naming, localhost/bearer disclosure policy, field contract, telemetry wiring, and storage.
