## Summary

Keep open. Current main still lacks the native no-inference model-contract endpoint, and the unresolved route, auth/disclosure, schema, and telemetry choices need maintainer design review.

## What This Changes

The issue proposes an auth-gated `/health/deep` Gateway endpoint that reports active model, provider, fallbacks, uptime, and routing/error diagnostics without invoking inference.

## Best Possible Solution

Keep this as a Gateway diagnostics/readiness design item and land an additive contract only after route naming, auth/redaction, field stability, and telemetry storage semantics are agreed.

## Reproduction Assessment

Not applicable as a feature/API proposal rather than a bug. Current-main source inspection gives a high-confidence verification path: `/health/deep` is absent, existing readiness/model surfaces do not expose the requested payload, and unknown paths fall through to 404.

## Solution Assessment

Unclear. A native endpoint may be the right direction, but the safest implementation depends on maintainer decisions about route naming, disclosure policy, field contract, telemetry wiring, and whether a plugin/API seam should own part of the behavior.
