## Summary

Keep open. Current main still lacks a native no-inference model-contract HTTP probe, and the remaining work is an additive Gateway API/security design decision rather than conservative cleanup.

## What This Changes

The issue proposes a native authenticated `/health/deep` Gateway endpoint that reports active model/provider/fallback contract state without invoking inference.

## Best Possible Solution

Keep this as a maintainer design item and land an additive Gateway diagnostics/readiness contract only after route, disclosure, redaction, schema, and telemetry semantics are agreed.

## Reproduction Assessment

Not applicable as a feature/API proposal rather than a bug. Source inspection on current main gives a high-confidence verification path: `/health/deep` is absent, existing readiness/model surfaces do not expose the requested payload, and unknown paths fall through to `404 Not Found`.

## Solution Assessment

Unclear. A native endpoint may be the right direction, but the best implementation depends on maintainer decisions about route naming, disclosure policy, response contract, telemetry wiring, and whether this belongs under the existing readiness surface.
