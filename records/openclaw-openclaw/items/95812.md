## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Preserve update restart deferral bypass:** `src/gateway/server-methods/update.ts:369-370`
  - body: This update path still passes `skipDeferral: true`, but `requestSafeGatewayRestart` now downgrades that flag whenever active work exists unless `activeWorkInterruptApproved` is set. Current main explicitly bypasses deferral for package swaps and managed handoffs because the new code is already staged; after this change a successful npm/handoff update can keep the old gateway process running behind active tasks for the deferral window instead of restarting immediately. Keep the existing update bypass semantics here, or add an explicit owner-approved interrupt flag plus coverage for the active-work case.
  - confidence: 0.86
- **[P2] Keep timeout-specific ACP spawns distinct:** `src/agents/acp-spawn.ts:342`
  - body: The duplicate-attach predicate treats a second spawn with `runTimeoutSeconds` set as equivalent to the existing active task. Since the existing task record does not store the timeout to compare against, a user asking for the same task with a different timeout will silently attach to the old run and ignore the new timeout. Exclude explicit `runTimeoutSeconds` from this fast-path attach, or persist and compare it before reusing the run.
  - confidence: 0.82
