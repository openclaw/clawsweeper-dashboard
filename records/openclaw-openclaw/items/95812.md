- **[P1] Preserve the documented skip-deferral restart path:** `src/gateway/server-methods/restart.ts:42`
  - body: `openclaw gateway restart --safe --skip-deferral` currently sends only `skipDeferral: true`, and the docs promise that this bypasses active-work deferral when blockers are reported. This new gate requires `interruptActiveWork === true`, but the CLI and docs are not updated, so the documented operator escape hatch will silently become a deferred restart whenever active work exists. Either keep the existing `skipDeferral` behavior for trusted operator RPCs or update the CLI/docs/tests to send an explicit interrupt approval.
  - confidence: 0.94

  - body: The new duplicate-attach shortcut treats a request as a plain duplicate without checking `runTimeoutSeconds`. A second spawn with the same task/label but a different timeout will be accepted onto the existing run while the response reports the new timeout, even though the existing run's timeout was not changed. Reject attachment when `runTimeoutSeconds` is explicitly set or store and compare the original timeout before reusing the run.
  - confidence: 0.86
