## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Emit restart before writing the audit row:** `src/infra/restart.ts:473-477`
  - body: `emitGatewayRestart()` now writes the best-effort `emit_requested` row before authorizing and emitting SIGUSR1. Because this write uses the shared state DB synchronously, a locked DB can delay an immediate restart even though audit persistence should not sit on the signal path.
  - confidence: 0.9
- **[P2] Re-arm the replacement timer before auditing reschedules:** `src/infra/restart.ts:1058-1069`
  - body: In this reschedule branch the old timer has already been cleared, but the synchronous audit write runs before `pendingRestartDueAt` is updated and the new timer is armed. A shared-state DB lock can therefore leave the requested restart unarmed until the write returns or times out.
  - confidence: 0.9
