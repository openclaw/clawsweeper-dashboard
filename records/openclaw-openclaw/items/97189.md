## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Emit restart before writing the audit row:** `src/infra/restart.ts:503-507`
  - body: `emitGatewayRestart()` now waits for a shared-state SQLite transaction before authorizing and emitting SIGUSR1 or the supervisor restart. Because the shared DB can wait on locks for up to the configured busy timeout, a busy database can delay an immediate restart that should already be in progress; move this audit after emission or queue it off the critical path.
  - confidence: 0.9
- **[P2] Bypass deferral before auditing:** `src/infra/restart.ts:1028-1039`
  - body: The forced deferral-bypass path writes the `bypassed_deferral` audit row before clearing active deferral polls and emitting the prepared restart. If the state DB is busy, an operator's explicit bypass still waits on audit persistence, which defeats this path's purpose.
  - confidence: 0.88
- **[P2] Re-arm rescheduled restarts before auditing:** `src/infra/restart.ts:1088-1099`
  - body: The earlier-reschedule paths write audit rows before the replacement restart state is fully armed; in the dropped-continuation branch the old timer is cleared first. A busy audit write can therefore leave the earlier restart delayed or unarmed until SQLite returns.
  - confidence: 0.89
- **[P2] Keep truncated audit payloads valid JSON:** `src/infra/restart.ts:376`
  - body: `audit_json` and `preflight_json` are stored as JSON fields, but values over 20 KB become a raw serialized prefix plus an ellipsis. Truncate before `JSON.stringify` or store a valid envelope with a `truncated` marker so forensic readers can still parse large rows.
  - confidence: 0.83
