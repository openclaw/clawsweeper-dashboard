Full review comments:

- **[P2] Emit restart before persisting the audit row:** `src/infra/restart.ts:503-507`
  - body: `emitGatewayRestart()` now waits for a shared-state SQLite write before authorizing and emitting SIGUSR1 or the supervisor restart. Since shared state writes are synchronous and can wait on SQLite locks, a busy database can delay an immediate restart that should already be in progress; move this audit after emission or queue it off the critical path.
  - confidence: 0.9
- **[P2] Bypass deferral before writing the audit row:** `src/infra/restart.ts:1028-1039`
  - body: The forced deferral-bypass path writes `bypassed_deferral` before clearing active deferral polls and emitting the prepared restart. A busy state DB can therefore delay an operator's explicit bypass, which defeats the purpose of this path cutting through the safe-restart delay.
  - confidence: 0.88
- **[P2] Re-arm the rescheduled restart before auditing:** `src/infra/restart.ts:1118-1129`
  - body: In the reschedule path, the synchronous audit write runs before the replacement earlier timer is armed, and the adjacent dropped-hook branch even clears the old timer first. If the state DB is busy, the earlier restart remains delayed or unarmed until the audit write returns.
  - confidence: 0.89
