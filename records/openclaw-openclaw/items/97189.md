## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Emit restart before persisting the audit row:** `src/infra/restart.ts:503-507`
  - body: `emitGatewayRestart()` waits for a shared-state SQLite write before authorizing and emitting SIGUSR1 or the supervisor restart. Because shared state writes can wait on SQLite locks, a busy database can delay an immediate restart that should already be in progress; move this audit after emission or queue it off the critical path.
  - confidence: 0.9
- **[P2] Bypass deferral before writing the audit row:** `src/infra/restart.ts:1028-1039`
  - body: The forced deferral-bypass path writes `bypassed_deferral` before clearing active deferral polls and emitting the prepared restart. A busy state DB can therefore delay an operator's explicit bypass, which defeats the purpose of this path cutting through the safe-restart delay.
  - confidence: 0.88
- **[P2] Re-arm rescheduled restarts before auditing:** `src/infra/restart.ts:1088-1129`
  - body: The reschedule paths write the audit row before the earlier replacement timer is armed; in the dropped-continuation branch the old timer is cleared first. If the state DB is busy, the earlier restart remains delayed or unarmed until the audit write returns.
  - confidence: 0.89
- **[P2] Keep stored audit payloads valid JSON:** `src/infra/restart.ts:376`
  - body: `audit_json` and `preflight_json` are stored as JSON fields, but when the serialized payload exceeds 20 KB this returns a raw prefix plus an ellipsis, which is no longer parseable JSON. Truncate before `JSON.stringify` or store a valid envelope with a `truncated` marker so forensic readers can still parse large rows.
  - confidence: 0.83
