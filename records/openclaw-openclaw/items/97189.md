Full review comments:

- **[P2] Re-arm rescheduled restarts before auditing them:** `src/infra/restart.ts:1058-1099`
  - body: These reschedule audit writes run while the existing timer is cleared or before the pending restart state is replaced. Because the audit helper uses the shared state DB's synchronous immediate transaction path, a locked DB can leave the restart unarmed or stale until the busy timeout or catch path returns; update/re-arm first or make auditing off-path.
  - confidence: 0.9

  - body: emitGatewayRestart() writes the emit_requested audit row before authorizing and emitting the restart signal. A shared-state DB lock can therefore delay an immediate restart even though the audit row is best-effort; emit first or make this write bounded/off-path.
  - confidence: 0.88
