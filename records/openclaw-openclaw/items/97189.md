Full review comments:

- **[P1] Do not persist raw restart session identifiers:** `src/infra/restart.ts:370-401`
  - body: The warning log redaction does not protect the new durable state: this still serializes the full audit payload and writes `session_key` directly into the shared SQLite table. Hash or omit the identifier, or get an explicit maintainer-approved storage contract before merging.
  - confidence: 0.9
- **[P2] Emit restart before writing the audit row:** `src/infra/restart.ts:473-477`
  - body: `emitGatewayRestart()` now waits for a synchronous shared-state insert before authorizing and emitting the restart signal. If SQLite is busy, audit persistence can delay an immediate restart that should already be in progress.
  - confidence: 0.9
- **[P2] Bypass deferral before writing the audit row:** `src/infra/restart.ts:998-1009`
  - body: The deferral-bypass path writes `bypassed_deferral` before clearing active deferral polls and emitting the prepared restart. A busy state DB can therefore delay an operator's explicit bypass through the safe-restart delay.
  - confidence: 0.88
- **[P2] Re-arm the replacement timer before auditing reschedules:** `src/infra/restart.ts:1058-1069`
  - body: This branch clears the old timer, performs the synchronous audit write, and only then updates `pendingRestartDueAt` and arms the replacement timer. If the shared state DB is busy, the earlier restart remains unarmed until the audit write returns.
  - confidence: 0.88
