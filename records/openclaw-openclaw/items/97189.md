Full review comments:

- **[P2] Re-arm rescheduled restarts before auditing them:** `src/infra/restart.ts:1058-1069`
  - body: In this reschedule branch, the existing timer has already been cleared, but `writeGatewayRestartAuditEventSync()` runs before `pendingRestartDueAt` is updated and `armPendingRestartTimer()` is called. Because that audit helper uses the shared state DB's synchronous immediate transaction path, a locked DB can leave the restart unarmed until the busy timeout or catch path returns.
  - confidence: 0.9
- **[P2] Emit prepared restarts before audit persistence:** `src/infra/restart.ts:473-477`
  - body: `emitGatewayRestart()` now writes the `emit_requested` audit row before authorizing and emitting the restart signal. A shared-state DB lock can therefore delay an immediate restart even though the audit row is best-effort; emit first or make this write bounded/non-blocking.
  - confidence: 0.88
