Full review comments:

- **[P2] Emit restart before writing the audit row:** `src/infra/restart.ts:473-477`
  - body: `emitGatewayRestart()` now writes the `emit_requested` row through the shared state DB before authorizing and emitting SIGUSR1 or the supervisor restart. Because that synchronous write can wait on SQLite locks, audit persistence can delay an immediate restart that should already be in progress.
  - confidence: 0.9
- **[P2] Bypass deferral before writing the audit row:** `src/infra/restart.ts:998-1009`
  - body: The deferral-bypass path writes `bypassed_deferral` before clearing the active deferral polls and emitting the prepared restart. A locked state DB can therefore delay the operator's explicit bypass even though this path is supposed to cut through the safe-restart delay.
  - confidence: 0.88
- **[P2] Re-arm the replacement timer before auditing reschedules:** `src/infra/restart.ts:1058-1069`
  - body: In this reschedule branch the old timer is cleared, then the synchronous audit write runs before `pendingRestartDueAt` is updated and the replacement timer is armed. If the shared state DB is busy, the requested earlier restart remains unarmed until the write returns or times out.
  - confidence: 0.9
