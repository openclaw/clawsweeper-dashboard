## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Emit restart before persisting the audit row:** `src/infra/restart.ts:503-507`
  - body: `emitGatewayRestart()` now waits for a shared-state SQLite write before authorizing and emitting SIGUSR1 or the supervisor restart. Because the shared DB can wait on locks, a busy database can delay an immediate restart; move this audit after emission or queue it off the critical path.
  - confidence: 0.9
- **[P2] Bypass deferral before writing the audit row:** `src/infra/restart.ts:1028-1039`
  - body: The forced deferral-bypass path writes `bypassed_deferral` before clearing active deferral polls and emitting the prepared restart. A busy state DB can therefore delay an operator's explicit bypass, which defeats the purpose of this escape hatch.
  - confidence: 0.88
- **[P2] Re-arm rescheduled restarts before auditing:** `src/infra/restart.ts:1088-1129`
  - body: The reschedule paths write audit rows before the earlier replacement timer is armed; in one branch the old timer has already been cleared. If the state DB is busy, the earlier restart remains delayed or unarmed until the audit write returns.
  - confidence: 0.88
- **[P2] Keep stored audit payloads valid JSON:** `src/infra/restart.ts:376`
  - body: `audit_json` and `preflight_json` are stored as JSON fields, but oversized payloads are truncated after `JSON.stringify`, producing an invalid JSON prefix plus an ellipsis. Truncate before serialization or store a valid envelope with a `truncated` marker.
  - confidence: 0.84
