## Summary

Keep this open. Current main still does not provide the requested cross-platform argument-level exec allowlist contract: non-Windows matching ignores `argPattern`, the proposed `argsPattern` field is not accepted by the protocol schema, and the public docs/UI/CLI surfaces still describe executable-pattern allowlists.

## What This Changes

Request argument-level exec allowlist matching so a restricted agent can run one specific interpreter/script invocation without allowing the whole binary.

## Best Possible Solution

Define and document a secure argv-level allowlist contract, then implement it consistently across runtime matching, persistence, protocol schema, CLI, Control UI, docs, and regression tests while preserving existing path-only entries.

## Reproduction Assessment

Not applicable as a feature request. The missing behavior is high-confidence from source: on non-Windows a path-matched entry bypasses `argPattern`, and the requested `argsPattern` field is rejected by the schema.

## Solution Assessment

Unclear for the exact proposed `argsPattern` glob spelling. The capability is valid, but the maintainable path is to reconcile it with existing regex-shaped `argPattern` metadata and define secure argv parsing semantics first.
