## Summary

Keep this open. Current main still lacks the requested public, cross-platform argument-level exec allowlist contract: non-Windows matching ignores `argPattern`, the proposed `argsPattern` field is not accepted by the protocol schema, and the docs/UI/CLI still present allowlists as executable-pattern entries.

## What This Changes

The issue requests argument-level exec approval allowlist matching so one interpreter invocation can be allowed without allowing all uses of that interpreter.

## Best Possible Solution

Define and document a secure argv-level allowlist contract, then implement it consistently across runtime matching, persistence, protocol schema, CLI, Control UI, docs, and regression tests while preserving existing path-only entries.

## Reproduction Assessment

Not applicable as a feature request. The missing behavior is high-confidence from source: non-Windows path matches return before argv checking, and the requested `argsPattern` field is rejected by the schema.

## Solution Assessment

Unclear for the exact proposed `argsPattern` glob spelling. The capability is valid, but the maintainable path is to reconcile it with existing regex-shaped `argPattern` metadata and define secure argv parsing semantics first.
