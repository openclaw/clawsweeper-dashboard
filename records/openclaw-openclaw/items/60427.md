## Summary

Current main now implements the requested binary-plus-arguments allowlist capability under the documented `argPattern` field, with cross-platform matcher coverage, protocol schema support, docs, and a beta release containing the fix.

## What This Changes

Request to let exec allowlist entries restrict a matched executable by its argv shape so one trusted interpreter/script invocation can be allowed without allowing the whole interpreter.

## Best Possible Solution

Keep the shipped `argPattern` regex contract for this narrow use case and leave broader permission-learning or denylist policy design to https://github.com/openclaw/openclaw/issues/48532.

## Reproduction Assessment

Not applicable as a feature request. Source and tests now give a high-confidence current-main state check: an argPattern allowlist entry can permit one argv shape and reject another for the same interpreter.

## Solution Assessment

Yes. Current main solves the narrow request with the existing singular `argPattern` regex field, which is documented and tested; the exact proposed `argsPattern` glob spelling was not adopted, but the requested authorization boundary is available.
