## Summary

Keep open. Current main has several relevant primitives, including policy-gated session tools, parent-owned subagent send/steer/status surfaces, task-state reporting, restart-abort recovery, WebChat session routing, and an external WeChat plugin path, but the issue asks for a broader continuity model across channels and agent identities that remains only partially covered and privacy-sensitive.

## What This Changes

The issue requests smoother cross-session and cross-agent continuity between WebChat/WeChat and parent/child agents, including parent-to-child messaging/status/history and recovery after aborted child runs.

## Best Possible Solution

Define an owner-approved continuity model that keeps isolation as the default while specifying which identities/sessions may link, how parents observe and control child runs, what failed or aborted child states can be retried or resumed, and how WebChat/QClaw plus the WeChat plugin expose those choices through explicit policy and UX.

Unclear. The report describes plausible workflows, but it does not provide versions, config, effective tool lists, logs, transcript snippets, or an abort trigger needed to reproduce the exact QClaw/WebChat/WeChat behavior against current main.

## Solution Assessment

Unclear. The existing primitives are the right building blocks, but an unconditional cross-session or cross-agent bridge would not be the safest solution without explicit privacy, identity-linking, plugin-boundary, and client UX decisions.
