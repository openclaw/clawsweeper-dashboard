## Summary

Keep open. Current main has useful session tools, subagent send/steer/status surfaces, restart-abort recovery, WebChat routing behavior, and external WeChat plugin documentation, but the broader cross-channel and parent-child continuity model is still only partially covered and remains privacy-sensitive product architecture rather than a terminal cleanup close.

## What This Changes

The issue requests cross-session and cross-agent continuity between WebChat, WeChat/QClaw, parent agents, and spawned subagents, including communication, status, abort recovery, and resume/retry behavior.

## Best Possible Solution

Define an owner-approved continuity model that preserves default isolation: which identities and sessions may link, how parents observe/control child runs, what failed or aborted child states can be retried or resumed, and how WebChat/QClaw plus openclaw-weixin expose those flows through explicit policy and UX.

## Solution Assessment
