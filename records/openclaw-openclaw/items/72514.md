## Summary

Keep open. Current main has useful session tools, subagent control, task tracking, identity-linking, and restart-abort recovery, but the requested WebChat-WeChat plus parent-child continuity model is only partially covered and remains privacy-sensitive architecture work.

## What This Changes

The issue requests opt-in cross-session and cross-agent continuity so WebChat/WeChat and parent/subagent workflows can share context, status, communication, and recovery.

## Best Possible Solution

Define an owner-approved continuity model that preserves default isolation while specifying explicit identity linking, parent-child observation/control, failed child retry/resume semantics, and WebChat/QClaw plus WeChat-plugin UX.
