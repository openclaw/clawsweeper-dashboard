## Summary

Keep open: current main has several scoped primitives, but it does not yet provide the requested cross-session, cross-channel, and parent-child continuity model. The remaining work changes privacy and agent-to-agent authority, so it needs maintainer product/security design before implementation.

## What This Changes

Request cross-session and cross-agent continuity for WebChat/WeChat/QClaw workflows, including parent-child communication, abort recovery, and shared context/history across channels.

## Best Possible Solution

Define an owner-approved continuity model that preserves default isolation while specifying explicit identity/session linking, parent-child observation/control, failed child retry/resume behavior, and WebChat/QClaw plus WeChat UX.

## Reproduction Assessment

Not applicable: this is a feature/design request rather than a broken existing contract. Source inspection shows partial primitives on current main, but the report does not provide enough versions, config, transcripts, or an abort trigger to replay the exact workflow.

## Solution Assessment

Unclear. Current gated session and subagent primitives are the right safety baseline, but an unconditional bridge across sessions, channels, and agents is not the best maintainable solution without product and privacy design.
