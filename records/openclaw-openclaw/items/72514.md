## Summary

Keep open because current main has useful session, subagent, task-status, identity-linking, and gated agent-to-agent primitives, but the requested cross-channel and parent-child continuity model remains only partially covered and is security/privacy-sensitive architecture work.

## What This Changes

Request a continuity model that lets WebChat/WeChat/QClaw sessions and parent/child agents share or inspect context, status, and recover failed sub-agent work.

## Best Possible Solution

Define an owner-approved continuity model that preserves default isolation while specifying explicit identity/session linking, parent-child observation/control, failed child retry/resume behavior, and WebChat/QClaw plus WeChat-plugin UX.
