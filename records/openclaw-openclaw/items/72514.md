## Summary

Keep open. Current main has scoped session tools, identity-link/docking primitives, and subagent list/steer/send controls, but it does not provide the complete cross-channel, cross-agent continuity and retry/resume model requested, and broadening those boundaries is privacy-sensitive architecture work.

## What This Changes

The issue requests WebChat/WeChat continuity, parent-child subagent communication/status, and abort/retry/resume behavior across agents and sessions.

## Best Possible Solution

Define an owner-approved continuity model that preserves default isolation while specifying explicit identity/session linking, parent-child observation/control, failed child retry/resume behavior, and WebChat/QClaw plus WeChat-plugin UX.

## Solution Assessment

Unclear. The existing explicit session tools, identity links, and subagent controls are the right foundation, but an unconditional bridge across sessions, channels, and agents would not be the narrowest maintainable solution without product and privacy design.
