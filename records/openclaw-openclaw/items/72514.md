## Summary

Keep open. Current main has useful session, subagent, recovery, and cross-channel continuity primitives, but the broad QClaw/WebChat-WeChat and parent-child agent UX described here is not fully proven or product-shaped enough to close: model-facing subagent control is still narrower than the requested send/status/retry/resume surface, restart recovery is bounded, and WeChat behavior lives behind an external plugin/client boundary.

## What This Changes

The issue requests seamless cross-session WebChat-WeChat/QClaw continuity plus stronger parent-child agent communication, status, abort recovery, and history visibility.
