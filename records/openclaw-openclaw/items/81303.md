## What This Changes

The PR adds a `session.maintenance.preserveKeys` field to session maintenance config types and returns it from maintenance config resolution.

## Best Possible Solution

Define the public config contract, then wire configured exact-key preservation through schema, resolved config, save-time pruning/capping, sessions cleanup, disk-budget entry eviction, docs/help, and focused regression tests.

## Reproduction Assessment

Not applicable as a current-main bug reproduction because this is a new config field. The PR defects are source-reproducible: the strict config schema still lacks the key, and the deletion paths do not consume the resolved `preserveKeys`.

## Solution Assessment

No. The requested direction may be maintainable, but this patch is incomplete because it does not add the strict schema field, thread configured keys through all removal paths, or document/test the public contract.
