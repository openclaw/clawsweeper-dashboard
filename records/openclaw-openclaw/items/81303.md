## What This Changes

Adds a `session.maintenance.preserveKeys` field to session maintenance types and the resolved maintenance config.

## Best Possible Solution

Define the public config contract first, then add schema/help/docs/tests and thread configured keys through every destructive session-maintenance path.

## Reproduction Assessment

Yes for the PR defects: source inspection shows the strict schema lacks `preserveKeys`, and runtime cleanup still only preserves the active key or no explicit preserve set. I did not run mutating CLI proof in this read-only review.

## Solution Assessment

No. Adding type fields and a resolved property is not enough; the maintainable solution must align the public config contract and pass a merged preserve set into all prune, cap, cleanup, load, and disk-budget paths that should honor it.
