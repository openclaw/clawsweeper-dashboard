The branch adds `preserveKeys?: string[]` to session maintenance config types and returns it from the maintenance config resolver.

## Best Possible Solution

If maintainers accept the new config surface, implement exact-key preservation as a full config contract across schema, runtime cleanup paths, docs/help, and regression tests.
