## Summary

Keep open: the GitHub Copilot device-flow bug still appears valid on current main, but this PR is not mergeable as-is because it patches an obsolete core provider path and the new `--wait` flag is not wired through the current auth contract.

## What This Changes

The branch changes GitHub Copilot device-flow polling/backoff timing in the old core provider and adds a `--wait` option to the CLI login command for WSL or remote terminals.

## Best Possible Solution

Port the polling/backoff fix into `extensions/github-copilot/login.ts` with focused coverage, then consider any `--wait` UX through the provider auth contract rather than the obsolete core provider path.

## Reproduction Assessment

Yes. Source inspection shows current main's active GitHub Copilot plugin polls before the initial interval and handles `slow_down` with the old fixed-offset behavior, while GitHub's device-flow docs require respecting the minimum interval; I did not run a live WSL auth flow.

## Solution Assessment

No. The polling fix direction is sound, but this branch patches an obsolete file and exposes `--wait` outside the current typed auth contract; the maintainable path is a provider-local fix plus an approved contract/CLI surface if the flag stays.
