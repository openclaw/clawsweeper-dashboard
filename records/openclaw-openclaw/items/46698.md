## Summary

Keep open: the GitHub Copilot device-flow problem still appears valid, but this branch is not merge-ready because it patches the retired core provider path while current main routes auth through the plugin helper, and the PR still lacks inspectable real behavior proof.

## What This Changes

The PR changes GitHub Copilot device-flow polling cadence and adds a `--wait` CLI flag for WSL/remote-terminal login.

## Best Possible Solution

Move the polling/backoff fix into `extensions/github-copilot/login.ts`, add focused timing/backoff coverage, and add typed wait plumbing only if maintainers want that CLI UX.

## Reproduction Assessment

Yes, from source inspection. Current main's active GitHub Copilot login helper still polls before an initial delay and uses the old `slow_down` timing, matching the closed WSL hang report linked to this PR.

## Solution Assessment

No. The polling direction matches GitHub's device-flow contract, but the patch targets a removed core provider file and the new wait option is not part of the current provider-auth/plugin contract.
