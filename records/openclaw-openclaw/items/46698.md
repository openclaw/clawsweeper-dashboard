## Summary

Keep open. The GitHub Copilot device-flow polling bug still appears valid on current main, but this PR is stale against the provider-plugin refactor and its useful fix is not applied to the active code path.

## What This Changes

The PR changes GitHub Copilot device-flow polling/backoff behavior and adds a `--wait` flag to the CLI login command for WSL or remote terminals.

## Best Possible Solution

Port the polling/backoff fix into `extensions/github-copilot/login.ts` with focused timing/backoff coverage, and add typed `--wait` plumbing only if maintainers approve that CLI UX.

## Reproduction Assessment

Yes, from source inspection. Current main's active GitHub Copilot polling loop sends the first token request before waiting and still handles `slow_down` with a fixed offset, matching the linked WSL hang symptoms.

## Solution Assessment

No. The polling direction matches the GitHub device-flow contract, but this branch edits an obsolete core provider file and does not wire `--wait` through the current provider-auth/plugin contract.
