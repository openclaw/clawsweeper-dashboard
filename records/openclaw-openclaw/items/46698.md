## What This Changes

The PR adds a `--wait` flag to GitHub Copilot login and changes device-flow polling/backoff timing in an old core provider auth file.

## Best Possible Solution

Port the polling/backoff fix into `extensions/github-copilot/login.ts`, add typed wait plumbing through the models auth command path if the flag remains desired, and cover the plugin, CLI, docs, changelog, and focused tests without restoring `src/providers`.

Yes. Source inspection of current main shows the active token poll fires before any interval delay, `slow_down` does not persist a server-returned interval, and the direct GitHub Copilot login command has no `--wait` option.

## Solution Assessment

No. The timing/backoff direction is sound, but this patch edits a removed core provider file and does not forward the new wait option into the active plugin-owned auth flow.
