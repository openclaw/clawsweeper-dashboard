## Summary

Keep open. The GitHub device-flow polling problem is still present on current main, but this PR is stale as-is because it changes the removed core provider path and does not wire the new wait option into the active plugin-owned auth flow.

## What This Changes

The PR adds a GitHub Copilot device-login `--wait` flag and changes device-flow polling to delay before the first token poll and honor `slow_down` intervals.

## Best Possible Solution

Move the timing/backoff fix into `extensions/github-copilot/login.ts`, add typed direct-login wait plumbing through the provider-auth command path, preserve the #73290 GUI/RPC prompter flow, and add focused tests, docs, and changelog coverage.

## Reproduction Assessment

Yes. Static reproduction is high-confidence: current main posts the token request before sleeping, handles `slow_down` with a fixed non-persistent delay, and does not register or forward `--wait`.

## Solution Assessment

No. The polling/backoff direction is correct, but the proposed patch is not the best current implementation because it targets a removed core path and omits typed option plumbing into the active plugin-owned auth flow.
