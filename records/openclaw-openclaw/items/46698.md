## Summary

Keep open for rework: the reported GitHub Copilot device-flow problem still exists on current main, but this branch edits an obsolete core provider path and the advertised wait flag does not reach the active plugin auth flow.

## What This Changes

The PR adds a GitHub Copilot login `--wait` option and changes device-flow polling to delay before polling and handle `slow_down` intervals differently.

## Best Possible Solution

Move the polling/backoff fix into `extensions/github-copilot/login.ts`, add focused timing/backoff coverage, and add typed wait plumbing only if maintainers want that CLI UX.

## Reproduction Assessment

Yes, from source inspection. Current main's active plugin login path polls before any delay and keeps the old `slow_down` handling, while the related WSL issue reports the same stuck authorization symptom.

## Solution Assessment

No. The polling direction is valid, but this patch edits the removed core provider file and does not route the new wait option through the current provider-auth/plugin contract.
