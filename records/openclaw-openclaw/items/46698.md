## Summary

Keep open: the underlying GitHub Copilot device-flow polling problem is still visible on current main and a related WSL issue was closed as superseded by this PR, but the branch is not mergeable as-is because it targets the old core auth file, does not wire `--wait` through the current provider-auth path, and lacks real behavior proof.

## What This Changes

The branch changes GitHub Copilot device-flow polling cadence and adds a `--wait` CLI flag with plain-text URL/code output for WSL or remote terminals.

## Best Possible Solution

Port the polling cadence fix to `extensions/github-copilot/login.ts`, decide whether a public `--wait` flag is needed or the provider auth UX should handle remote terminals automatically, then add focused regression coverage and real WSL/terminal proof.

## Reproduction Assessment

Source-reproducible for the polling root cause: current main sends the first token request before any delay and uses `intervalMs + 2000` for `slow_down`. I did not run a live WSL/GitHub authorization flow in this read-only review.

## Solution Assessment

No, not as-is. The polling fix direction matches GitHub's device-flow contract, but the implementation needs to move to the current provider-owned code and the new CLI flag needs a real provider-auth contract or removal.
