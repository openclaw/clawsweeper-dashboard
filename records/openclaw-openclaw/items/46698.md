## Summary

Keep this PR open, but it is not mergeable as-is: the reported GitHub Copilot device-flow polling bug still exists on current main, while this branch applies the core fix to an obsolete provider file and lacks real after-fix behavior proof.

## What This Changes

The PR changes GitHub Copilot device-flow polling/backoff timing and adds a `--wait` CLI option for WSL or remote-terminal login.

## Best Possible Solution

Port the polling/backoff fix into `extensions/github-copilot/login.ts` with focused timing/backoff coverage, and add typed `--wait` plumbing only if maintainers approve that CLI UX.

## Reproduction Assessment

Yes, by source inspection: current main's active `extensions/github-copilot/login.ts` token loop posts before any initial delay and uses `intervalMs + 2000` for `slow_down`, while GitHub's device-flow contract requires honoring the returned interval. I did not run a live WSL auth flow.

## Solution Assessment

No. The polling fix direction matches the GitHub device-flow contract, but this branch edits an obsolete core provider file and does not carry `--wait` through the current provider-auth plugin contract.
