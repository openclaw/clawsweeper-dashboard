## Summary

Keep open: the reported polling bug still appears valid on current main, but this PR is not mergeable as-is because it edits an obsolete core provider file and the new flag is outside the current plugin auth contract.

## What This Changes

The PR changes GitHub Copilot device-flow polling/backoff timing and adds a CLI `--wait` option for WSL or remote terminal login.

## Best Possible Solution

Port the polling/backoff fix into `extensions/github-copilot/login.ts` with focused coverage, then handle any `--wait` UX as an explicit provider/auth contract change after maintainer approval and real WSL proof.

## Reproduction Assessment

Yes, source inspection gives a high-confidence current-main path: the active GitHub Copilot helper polls before the initial interval and keeps the old fixed `slow_down` timing. I did not run a live WSL auth flow.

## Solution Assessment

No. The polling fix direction is sound, but this branch patches an obsolete core path and the `--wait` option is not part of the current typed auth contract; the safer fix is provider-local plus an approved contract/CLI surface if the flag remains.
