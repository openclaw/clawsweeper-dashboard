## Summary

Keep open. Current main still enables Gateway model-pricing refresh from `models.pricing.enabled` alone, so `models.mode: "replace"` can still reach OpenRouter and LiteLLM pricing fetches; open closing PR #74038 should be reviewed, landed, rejected, or replaced before this issue closes.

## What This Changes

The issue requests skipping remote Gateway model-pricing catalog fetches when `models.mode` is `replace`.

## Best Possible Solution

Update the shared Gateway pricing-enable helper so startup, direct refresh, and scheduled refresh derive the same default from both `models.pricing.enabled` and `models.mode`, then align regression tests, schema/help/docs, generated config surfaces, and changelog.

## Reproduction Assessment

Yes. Configure `models.mode: "replace"` with configured model refs and no `models.pricing.enabled: false`; current main still returns pricing enabled and can reach the OpenRouter/LiteLLM fetch path.

## Solution Assessment

Yes. The narrow maintainable fix is a shared helper default used by every pricing entry point; caller-only guards would leave refresh paths and docs/schema semantics inconsistent.
