## Summary

Keep open. Current main still derives Gateway pricing enablement only from `models.pricing.enabled`, and #74038 is an open unmerged closing PR for this behavior, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

Skip Gateway OpenRouter/LiteLLM pricing bootstrap fetches by default when `models.mode` is `replace`.

## Best Possible Solution

Review and land or replace #74038 with a shared pricing-enable helper fix plus aligned regression coverage, docs/schema surfaces, and changelog.

## Reproduction Assessment

Yes. A config with `models.mode: "replace"` and no explicit `models.pricing.enabled: false` still passes the current helper and can reach the OpenRouter/LiteLLM pricing fetch path by source inspection.

## Solution Assessment

Yes. The narrow maintainable direction is to derive pricing enablement in the shared helper used by startup and refresh paths, preserving explicit opt-in/opt-out behavior while changing the omitted-pricing default for replace mode.
