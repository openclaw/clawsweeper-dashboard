## Summary

Keep open. Current main still derives Gateway pricing enablement only from `models.pricing.enabled`, so `models.mode: "replace"` can still enter the startup and refresh paths that fetch OpenRouter and LiteLLM pricing; the open closing PR #74038 should be reviewed, landed, rejected, or replaced before this issue closes.

## What This Changes

The issue asks Gateway startup to skip OpenRouter/LiteLLM pricing fetches by default when `models.mode` is `replace`.

## Best Possible Solution

Update the shared Gateway pricing enablement helper so startup, direct refresh, and scheduled refresh all derive the same default from both `models.pricing.enabled` and `models.mode`, with regression tests, schema/help/docs, generated config surfaces, and changelog aligned.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: configure `models.mode: "replace"` without `models.pricing.enabled: false`, and the current helper still returns true, allowing startup and refresh to reach remote pricing fetches when external refs exist.

## Solution Assessment

Yes. The narrow maintainable solution is a shared pricing-enable helper default that all pricing entry points use; a caller-only guard would leave refresh paths and docs/schema semantics inconsistent.
