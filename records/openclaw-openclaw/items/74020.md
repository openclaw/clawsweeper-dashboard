## Summary

Keep this issue open. Current main still gates Gateway pricing only on `models.pricing.enabled !== false`, so `models.mode: "replace"` alone can still enter the OpenRouter/LiteLLM pricing bootstrap path; the open closing PR #74038 remains the right implementation candidate to review or replace.

## What This Changes

The issue requests that Gateway startup skip remote OpenRouter/LiteLLM pricing fetches when `models.mode` is `replace`.

## Best Possible Solution

Review and land #74038, or replace it with an equivalent focused change that derives Gateway pricing enablement from both `models.pricing.enabled` and `models.mode`, keeps explicit opt-in semantics clear, and updates startup/cache tests plus config docs/changelog together.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: with `models.mode: "replace"` and no `models.pricing.enabled: false`, the helper still returns true, so startup and refresh can proceed into OpenRouter/LiteLLM pricing fetches when external model refs are present.

## Solution Assessment

Yes. The narrow maintainable fix is the shared pricing enablement helper used by startup and direct refresh, with docs and regression tests updated; a one-off startup-only guard would leave other pricing entry points inconsistent.
