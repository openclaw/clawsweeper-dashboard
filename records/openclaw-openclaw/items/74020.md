## Summary

Keep open. Current main still enables Gateway pricing whenever `models.pricing.enabled` is not explicitly false, so `models.mode: "replace"` can still enter the OpenRouter/LiteLLM pricing fetch path; open PR #74038 links this issue but is unmerged.

## What This Changes

Request Gateway startup skip OpenRouter and LiteLLM pricing-catalog fetches when `models.mode` is `replace` and models are manually configured.

## Best Possible Solution

Land or replace #74038 with a shared pricing-enable helper that defaults omitted pricing to disabled in replace mode while preserving explicit opt-in, then align tests, docs, schema/help, generated config, and changelog.

## Reproduction Assessment

Yes. A config with `models.mode: "replace"`, an OpenRouter/LiteLLM-backed model, and no `models.pricing.enabled: false` still returns pricing enabled and can reach the refresh import/fetch path in current main.

## Solution Assessment

Yes. The narrow maintainable direction is to update the shared Gateway pricing-enable helper and its startup/cache callers so every pricing entry point uses the same replace-mode default.
