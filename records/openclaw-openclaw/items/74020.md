## Summary

Keep open. Current main still enables Gateway model-pricing refresh from `models.pricing.enabled` alone, so `models.mode: "replace"` can still reach the OpenRouter/LiteLLM pricing fetch path; open closing PR #74038 targets this issue but is unmerged.

## What This Changes

Request `models.mode: "replace"` to skip Gateway startup pricing fetches for manually configured model catalogs.

## Best Possible Solution

Land or replace #74038 with a shared pricing-enable helper fix plus aligned tests, docs, schema/help, generated config surfaces, and changelog.

## Reproduction Assessment

Yes. Configure `models.mode: "replace"` with an OpenRouter/LiteLLM-backed model and omit `models.pricing.enabled: false`; current main still treats pricing as enabled and can reach the remote catalog fetch path.

## Solution Assessment

Yes. The narrow maintainable direction is to derive pricing enablement in the shared helper used by startup and refresh paths, preserving explicit opt-in/opt-out behavior while changing only the omitted-pricing default for replace mode.
