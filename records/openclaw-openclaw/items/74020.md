## Summary

Keep open. Current main and the latest release still enable Gateway model pricing unless `models.pricing.enabled` is explicitly false, so a `models.mode: "replace"` config can still reach the OpenRouter/LiteLLM pricing fetch path; the open closing PR has not merged.

## What This Changes

Skip Gateway pricing catalog fetches by default when `models.mode` is `replace`.

## Best Possible Solution

Review, rebase, and land or replace https://github.com/openclaw/openclaw/pull/74038 with the shared helper default change, aligned regression coverage, docs/schema updates, and real startup proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure `models.mode: "replace"` with an OpenRouter/LiteLLM-backed model and omit `models.pricing.enabled`; current main still treats pricing as enabled and can reach the remote catalog fetch path.

## Solution Assessment

Yes. The narrow maintainable fix is to derive pricing enablement in the shared helper used by startup and refresh paths, preserving explicit true/false overrides while changing the omitted-pricing default for replace mode.
