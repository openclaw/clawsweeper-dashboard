## Summary

Keep open. Current main still enables Gateway model-pricing refresh from `models.pricing.enabled` alone, so `models.mode: "replace"` can still reach OpenRouter/LiteLLM pricing fetches; open closing PR #74038 targets this issue but has not merged.

## What This Changes

The issue asks Gateway model-pricing refresh to skip OpenRouter/LiteLLM pricing fetches when `models.mode` is `replace`.

## Best Possible Solution

Land or replace #74038 with shared helper behavior, replace-mode startup/cache regression tests, aligned docs/schema/help, and a changelog entry.

## Reproduction Assessment

Yes. Source inspection shows a replace-mode config with an OpenRouter/LiteLLM-backed model and omitted `models.pricing.enabled` still passes the pricing gate and can reach the remote catalog fetch path.

## Solution Assessment

Yes. The narrow maintainable fix is to derive pricing enablement in the shared helper used by startup and refresh paths, preserving explicit opt-in/opt-out behavior while changing only the omitted-pricing default for replace mode.
