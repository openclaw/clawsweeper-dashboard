## Summary

Keep open. Current main still derives Gateway pricing enablement only from `models.pricing.enabled`, so `models.mode: "replace"` can still reach the OpenRouter and LiteLLM pricing bootstrap; open PR #74038 is the linked implementation candidate but is unmerged.

## What This Changes

The issue asks Gateway startup to skip remote model-pricing catalog fetches when `models.mode` is `replace`.

## Best Possible Solution

Update the shared Gateway pricing-enable helper so omitted pricing defaults to disabled in replace mode while preserving an explicit opt-in, then align startup/cache tests, schema/help, docs, generated config surfaces, and changelog.

## Reproduction Assessment

Yes. Configure `models.mode: "replace"` with an OpenRouter- or LiteLLM-backed model and omit `models.pricing.enabled`; source tracing shows current main still enables pricing and can enter the remote catalog fetch path.

## Solution Assessment

Yes. The narrow maintainable fix is a shared helper used by startup, direct refresh, and scheduled refresh; #74038 follows that direction but remains unmerged.
