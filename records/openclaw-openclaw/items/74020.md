## Summary

Keep open. Current main still enables Gateway pricing bootstrap whenever `models.pricing.enabled` is not explicitly `false`, so `models.mode: "replace"` alone can still enter the startup and refresh paths that fetch OpenRouter and LiteLLM pricing. The open closing PR #74038 is the right place to review, land, reject, or replace the fix before this issue closes.

## What This Changes

Skip remote Gateway model-pricing catalog fetches during startup and refresh when `models.mode` is `replace`, because models are manually configured in that mode.

## Best Possible Solution

Update the shared Gateway pricing enablement helper so startup, direct refresh, and scheduled refresh all derive the same default from both `models.pricing.enabled` and `models.mode`, then land matching regression tests, config help/docs, generated config surfaces, and a changelog entry.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: with `models.mode: "replace"` and no `models.pricing.enabled: false`, the current helper still returns true, allowing startup and refresh to enter the remote pricing paths when external pricing refs exist.

## Solution Assessment

Yes, the narrow maintainable fix boundary is the shared pricing enablement helper used by startup and refresh paths. A one-off caller guard would leave other pricing entry points inconsistent.
