## Summary

Keep this issue open. Current main still enables Gateway pricing bootstrap unless `models.pricing.enabled` is explicitly false, so `models.mode: "replace"` alone can still reach the OpenRouter/LiteLLM pricing paths; the open closing PR #74038 remains the right place to review or replace the fix.

## What This Changes

The issue requests that Gateway startup skip OpenRouter/LiteLLM pricing fetches when `models.mode` is set to `replace` and models are manually configured.

## Best Possible Solution

Review and land #74038, or replace it with an equivalent focused change that derives Gateway pricing enablement from both `models.pricing.enabled` and `models.mode`, updates startup/refresh regression tests, and aligns config help/docs/changelog before closing this issue.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: with `models.mode: "replace"` and no `models.pricing.enabled: false`, the current helper still returns true, so startup and refresh can proceed into OpenRouter/LiteLLM pricing fetches when external refs are present.

## Solution Assessment

Yes. The best fix boundary is the shared pricing enablement helper used by startup and direct refresh, with tests and user-facing config docs updated together; a one-off caller guard would leave other pricing entry points inconsistent.
