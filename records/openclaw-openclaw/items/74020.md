## Summary

Keep open. Current main still enables Gateway pricing bootstrap solely from `models.pricing.enabled`, so `models.mode: "replace"` alone can still reach OpenRouter and LiteLLM pricing fetches; the open closing PR #74038 remains the right place to review or replace the fix before this issue is closed.

## What This Changes

Skip Gateway OpenRouter and LiteLLM pricing bootstrap/fetches when `models.mode` is `replace` and models are manually configured.

## Best Possible Solution

Review and land #74038, or replace it with an equivalent focused change that derives Gateway pricing enablement from both `models.pricing.enabled` and `models.mode`, updates startup/refresh regression tests, and aligns schema help/docs/changelog before closing this issue.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: with `models.mode: "replace"` and no `models.pricing.enabled: false`, current main's helper still returns true, so startup and refresh can proceed into OpenRouter/LiteLLM pricing fetches for configured external refs; no tests were run because this review was read-only.

## Solution Assessment

Yes. The best fix boundary is the shared pricing enablement/default helper used by startup and direct refresh, with tests and docs updated together; a one-off caller guard would leave other pricing entry points inconsistent.
