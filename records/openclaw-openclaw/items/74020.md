## Summary

Keep open. Current main still enables Gateway model-pricing refresh from `models.pricing.enabled` alone, so `models.mode: "replace"` can still reach OpenRouter and LiteLLM pricing fetches; open PR #74038 targets the issue but is unmerged and currently reports a dirty merge state.

## What This Changes

Request Gateway model-pricing bootstrap skip OpenRouter/LiteLLM pricing fetches when `models.mode` is `replace`.

## Best Possible Solution

Update the shared Gateway pricing-enable helper so omitted pricing defaults to disabled in replace mode while preserving an explicit opt-in, then align startup/cache tests, schema/help, docs, generated config surfaces, and changelog.

## Reproduction Assessment

Yes. Configure `models.mode: "replace"` with an OpenRouter/LiteLLM-backed model and no `models.pricing.enabled: false`; current main still returns pricing enabled and can reach the remote pricing fetch path.

## Solution Assessment

Yes. The narrow maintainable fix is a shared helper used by startup, direct refresh, and scheduled refresh; #74038 follows that direction but needs maintainer review and an update against current main.
