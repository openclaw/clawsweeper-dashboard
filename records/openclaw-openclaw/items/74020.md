## Summary

Keep open. Current main still gates Gateway pricing bootstrap only on `models.pricing.enabled !== false`, so `models.mode: "replace"` alone can still enter the startup and refresh paths; the open closing PR is the right place to review or replace the fix.

## What This Changes

The issue requests that Gateway startup skip OpenRouter/LiteLLM pricing fetches when `models.mode` is `replace`.

## Best Possible Solution

Review and land #74038, or replace it with an equivalent focused change that derives Gateway pricing enablement from both the explicit pricing switch and `models.mode`, then updates startup/refresh tests, config help/docs, and changelog together.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: with `models.mode: "replace"` and no `models.pricing.enabled: false`, the current helper still returns true, allowing startup and refresh to enter the OpenRouter/LiteLLM pricing paths when external refs exist.

## Solution Assessment

Yes. The best fix boundary is the shared pricing enablement helper used by startup and direct refresh; a one-off caller guard would leave other pricing entry points inconsistent.
