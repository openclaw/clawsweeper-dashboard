## Summary

Keep open. Current main still enables Gateway pricing from `models.pricing.enabled` alone, so replace-mode configs can still reach the OpenRouter/LiteLLM pricing fetch path; open closing PR #74038 should be reviewed, landed, rejected, or replaced before this issue closes.

## What This Changes

The issue requests that Gateway startup skip OpenRouter and LiteLLM pricing fetches when `models.mode` is `replace`.

## Best Possible Solution

Update the shared Gateway pricing-enable helper so startup, direct refresh, and scheduled refresh derive the same default from both `models.pricing.enabled` and `models.mode`, then align regression tests, docs/schema/help, generated config surfaces, and changelog.

## Reproduction Assessment

Yes. Configure `models.mode: "replace"` with an explicit model ref lacking local `cost` and omit `models.pricing.enabled`; current main still returns pricing enabled and can call the OpenRouter/LiteLLM refresh path.

## Solution Assessment

Yes. The narrow maintainable fix is a shared helper default used by every pricing entry point, with explicit `models.pricing.enabled: true` semantics documented if maintainers want that opt-in.
