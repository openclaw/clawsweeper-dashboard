## Summary

Keep open. Current main still derives Gateway pricing refresh enablement only from `models.pricing.enabled`, and the linked open closing PR #74038 should be reviewed, landed, rejected, or replaced before this issue closes.

## What This Changes

Gateway startup should skip OpenRouter and LiteLLM pricing fetches by default when `models.mode` is `replace`.

## Best Possible Solution

Review and land or replace #74038 with a shared helper default that disables pricing in replace mode when the explicit pricing toggle is omitted, with docs/schema/help and regression tests kept aligned.

## Reproduction Assessment

Yes. Configure `models.mode: "replace"` with configured hosted model refs and no `models.pricing.enabled: false`; current main still treats pricing as enabled and can enter the OpenRouter/LiteLLM fetch path.

## Solution Assessment

Yes. The narrow maintainable fix is to centralize the derived pricing enablement in the shared helper so startup, direct refresh, scheduled refresh, docs, schema help, and tests all agree.
