## Summary

Keep open. Current main still gates Gateway pricing refresh only on `models.pricing.enabled`, so `models.mode: "replace"` can still start the OpenRouter/LiteLLM pricing path; open PR #74038 already targets this issue with closing syntax.

## What This Changes

Request that Gateway skip OpenRouter/LiteLLM pricing bootstrap when `models.mode` is `replace` and pricing is not explicitly enabled.

## Best Possible Solution

Review and land or replace #74038 with a shared pricing-enable helper fix plus aligned tests, docs/schema surfaces, and changelog.

## Reproduction Assessment

Yes. Configure `models.mode: "replace"` with an OpenRouter/LiteLLM-backed model and omit `models.pricing.enabled: false`; source inspection shows current main still treats pricing as enabled and can reach the remote catalog fetch path.

## Solution Assessment

Yes. The narrow maintainable direction is to derive pricing enablement in the shared helper used by startup and refresh paths, preserving explicit opt-in/opt-out behavior while changing only the omitted-pricing default for replace mode.
