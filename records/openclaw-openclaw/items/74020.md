## Summary

Keep open. Current main still derives Gateway pricing enablement only from `models.pricing.enabled`, so `models.mode: "replace"` can still enter the startup and refresh paths that fetch OpenRouter and LiteLLM pricing; open PR #74038 remains the right item to review, land, reject, or replace.

## What This Changes

Request Gateway startup to avoid remote pricing catalog fetches when manually configured models use `models.mode: "replace"`.

## Best Possible Solution

Land or replace the #74038-style fix that changes the shared pricing enablement default, preserves intentional explicit opt-in semantics, and aligns tests, config help, docs, generated surfaces, and changelog.

## Reproduction Assessment

Yes. Configure `models.mode: "replace"` with an OpenRouter or LiteLLM-backed model ref and omit `models.pricing.enabled: false`; source inspection shows the helper still returns true, startup imports pricing refresh, and refresh can reach both remote catalog fetches when external refs remain.

## Solution Assessment

Yes. The narrow maintainable fix is to update the shared pricing-enable helper/default because startup, direct refresh, and scheduled refresh already funnel through it; a caller-only guard would leave inconsistent behavior.
