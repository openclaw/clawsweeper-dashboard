## Summary

Keep open. Current main still derives Gateway model-pricing refresh only from `models.pricing.enabled`, so `models.mode: "replace"` does not suppress the OpenRouter/LiteLLM pricing path; open PR #74038 already targets this issue.

## What This Changes

The issue requests skipping Gateway OpenRouter/LiteLLM pricing fetches when `models.mode` is `replace`.

## Best Possible Solution

Land or replace #74038 with a shared pricing-enable helper change plus aligned tests, docs/schema generated surfaces, and changelog coverage.

## Reproduction Assessment

Yes. From source inspection, configure `models.mode: "replace"` with an OpenRouter/LiteLLM-backed model and omit `models.pricing.enabled: false`; current main still treats pricing as enabled and can reach the remote catalog fetch path.

## Solution Assessment

Yes. The narrow maintainable fix is to derive pricing enablement in the shared helper used by startup and refresh paths, preserving explicit opt-in/opt-out behavior while changing the omitted-pricing default for replace mode.
