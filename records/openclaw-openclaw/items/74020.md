## Summary

Keep open. Current main still derives Gateway pricing bootstrap solely from `models.pricing.enabled`, so `models.mode: "replace"` can still reach the OpenRouter/LiteLLM pricing refresh path. The linked #74038 is open and explicitly tied to closing this issue, so this should remain open until that PR or an equivalent focused fix lands.

## What This Changes

The issue requests that Gateway startup skip remote model-pricing catalog fetches when `models.mode` is `replace` and models are manually configured.

## Best Possible Solution

Review and land #74038, or replace it with an equivalent focused change that derives Gateway pricing enablement from both `models.pricing.enabled` and `models.mode`, updates startup and direct refresh regression tests, and aligns schema help, docs, and changelog before closing this issue.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: configure `models.mode: "replace"` without `models.pricing.enabled: false`, and the current helper still returns true, allowing startup and refresh to enter the OpenRouter/LiteLLM pricing fetch path when external refs remain.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the shared pricing enablement/default helper used by both startup and direct refresh, not in a one-off startup guard; the only open design detail is explicit opt-in behavior in replace mode.
