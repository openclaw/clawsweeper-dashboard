## Summary

Keep open. Current main still derives Gateway pricing bootstrap solely from `models.pricing.enabled`, so `models.mode: "replace"` can still reach the OpenRouter/LiteLLM pricing fetch path; open unmerged PR #74038 already targets this issue and should be reviewed, landed, rejected, or replaced before the issue closes.

## What This Changes

Skip Gateway OpenRouter/LiteLLM pricing bootstrap when `models.mode` is `replace` and models are manually configured.

## Best Possible Solution

Land or replace #74038 with a shared pricing-enable default that disables Gateway pricing bootstrap in replace mode unless explicitly opted in, with startup/cache tests plus docs, schema/help, generated config, and changelog aligned.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: configure `models.mode: "replace"` without `models.pricing.enabled: false`, and the current helper still returns true, allowing startup and refresh to enter remote pricing fetch paths when external refs exist.

## Solution Assessment

Yes. The narrow maintainable fix is to change the shared Gateway pricing enablement helper so startup, direct refresh, and scheduled refresh use the same derived default; caller-only guards would leave other entry points inconsistent.
