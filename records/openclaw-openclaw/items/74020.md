## Summary

Keep open. Current main still derives Gateway pricing bootstrap solely from `models.pricing.enabled`, so `models.mode: "replace"` can still enter the OpenRouter/LiteLLM pricing refresh path; the open closing PR #74038 is the right place to review or replace the fix before this issue closes.

## What This Changes

The issue requests that Gateway startup skip OpenRouter/LiteLLM pricing fetches when `models.mode` is `replace` and models are manually configured.

## Best Possible Solution

Review and land #74038, or replace it with an equivalent focused change that derives Gateway pricing enablement from both `models.pricing.enabled` and `models.mode`, keeps explicit opt-in semantics intentional, and updates startup/refresh regression tests plus schema help/docs/changelog before closing this issue.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: with `models.mode: "replace"` and no `models.pricing.enabled: false`, the current helper still returns true, allowing startup and refresh to proceed into remote pricing fetches for external refs; no tests were run because this review was read-only.

## Solution Assessment

Yes for the linked implementation direction. The narrow maintainable fix is the shared pricing enablement/default helper used by both startup and direct refresh; a caller-only guard would leave other pricing entry points inconsistent.
