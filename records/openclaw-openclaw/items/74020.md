## Summary

Keep this issue open. Current main still gates Gateway pricing bootstrap only on `models.pricing.enabled !== false`, so `models.mode: "replace"` alone can still reach the OpenRouter/LiteLLM pricing fetch path; the open closing PR #74038 remains the right implementation candidate to review, land, or replace.

## What This Changes

The issue requests that Gateway startup skip remote model pricing fetches when `models.mode` is set to `replace`.

## Best Possible Solution

Derive Gateway pricing enablement in the shared pricing config helper from both the explicit pricing override and `models.mode`, then keep startup, direct refresh, config help/docs, changelog, and focused regression tests aligned before closing this issue.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: with `models.mode: "replace"` and no `models.pricing.enabled: false`, the current helper still returns true, allowing startup and refresh to proceed into remote pricing fetches when external refs are present.

## Solution Assessment

Yes. The best fix boundary is the shared pricing enablement helper used by startup and refresh, not a one-off caller guard; #74038 appears to target that boundary and should be reviewed or replaced with an equivalent focused change.
