## Summary

Keep open. Current main still enables Gateway pricing bootstrap whenever `models.pricing.enabled` is not explicitly `false`, so `models.mode: "replace"` alone can still start the OpenRouter/LiteLLM pricing path. The open closing PR #74038 is the right implementation path to review, land, reject, or replace before this issue closes.

## What This Changes

The issue requests that Gateway startup skip remote OpenRouter/LiteLLM pricing fetches by default when `models.mode` is `replace`.

## Best Possible Solution

Land or replace the focused Gateway pricing fix so the shared pricing enablement helper derives the default from both `models.pricing.enabled` and `models.mode`, with matching startup/refresh tests, docs/schema/help updates, and a changelog entry.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: with `models.mode: "replace"` and no `models.pricing.enabled: false`, the current helper returns true, allowing startup and refresh to enter the remote pricing paths when external pricing refs exist.

## Solution Assessment

Yes, the best fix boundary is the shared pricing enablement helper used by startup and refresh paths. A one-off caller guard would leave other pricing entry points inconsistent.
