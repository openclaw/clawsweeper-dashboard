## Summary

Keep open. Current main still enables Gateway pricing bootstrap whenever `models.pricing.enabled` is not explicitly false, so `models.mode: "replace"` alone can still reach the startup/cache pricing fetch path. The open closing PR #74038 is the correct item to review or replace before this issue closes.

## What This Changes

The issue requests that Gateway startup skip OpenRouter and LiteLLM pricing fetches by default when `models.mode` is `replace`.

## Best Possible Solution

Review and land #74038, or replace it with an equivalent focused change that derives Gateway pricing enablement from both `models.pricing.enabled` and `models.mode`, preserves any intended explicit opt-in behavior, and updates regression tests, schema/help docs, and changelog together.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: with `models.mode: "replace"` and no `models.pricing.enabled: false`, the current helper still returns true, so Gateway startup and refresh can proceed into OpenRouter/LiteLLM pricing fetches; no runtime test was run because this review was read-only.

## Solution Assessment

Yes, the best fix boundary is the shared pricing enablement/default helper used by startup and direct refresh, with docs and regression coverage aligned. A one-off caller guard would leave other pricing entry points inconsistent.
