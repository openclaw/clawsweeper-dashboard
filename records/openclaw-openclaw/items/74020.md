## Summary

Keep open. Current main still enables Gateway model-pricing refresh whenever `models.pricing.enabled` is not explicitly false, and the open closing PR #74038 has not landed, so the issue should remain open until that PR or an equivalent fix merges.

## What This Changes

Gateway should skip OpenRouter and LiteLLM pricing catalog fetches when `models.mode` is `replace` and pricing is not explicitly enabled.

## Best Possible Solution

Review and land or replace #74038 with a shared pricing-enable helper change plus aligned Gateway tests, docs/schema help, generated config surfaces, and changelog.

## Reproduction Assessment

Yes. Configure `models.mode: "replace"` with an OpenRouter or LiteLLM-backed configured model and omit `models.pricing.enabled: false`; source inspection shows the helper still returns enabled and the refresh path can reach the remote catalog fetches.

## Solution Assessment

Yes. The narrow maintainable fix is to derive default pricing enablement in the shared helper from `models.mode`, while preserving explicit pricing overrides and covering startup/cache behavior plus user-facing config docs.
