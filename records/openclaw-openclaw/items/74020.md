## Summary

Keep open. Current main still treats Gateway model pricing as enabled unless `models.pricing.enabled` is explicitly false, so replace-mode configs can still start the OpenRouter/LiteLLM pricing path; the open closing PR at https://github.com/openclaw/openclaw/pull/74038 is the right implementation path but has not merged.

## What This Changes

The issue asks Gateway startup to skip OpenRouter/LiteLLM pricing fetches when `models.mode` is `replace`.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/74038 with the shared helper default change plus aligned regression coverage and docs/schema updates.

## Reproduction Assessment

Yes. Configure `models.mode: "replace"` with an OpenRouter/LiteLLM-backed model and omit `models.pricing.enabled`; source inspection shows current main still treats pricing as enabled and can reach the remote catalog fetch path.

## Solution Assessment

Yes. The narrow maintainable fix is to derive pricing enablement in the shared helper used by startup and refresh paths, preserving explicit true/false overrides while changing the omitted-pricing default for replace mode.
