## Summary

Keep open: current main still has a source-reproducible Telegram forum-topic leak path, and https://github.com/openclaw/openclaw/pull/81885 is open rather than merged.

## What This Changes

Ensure Telegram forum-topic replies and send fallbacks keep the topic `message_thread_id` instead of retrying or delivering in General or another topic.

## Best Possible Solution

Review and land or supersede https://github.com/openclaw/openclaw/pull/81885 for fail-closed explicit topic sends, while coordinating broader async and cron route preservation in https://github.com/openclaw/openclaw/pull/76920.

## Reproduction Assessment

Yes at source level: current main retries Telegram forum-topic sends and draft previews without `message_thread_id` after `message thread not found`. I did not run a live Telegram forum probe in this read-only review.

## Solution Assessment

Yes: the narrow maintainable fix is to fail closed for explicit non-General topic sends instead of stripping topic metadata, while preserving the General-topic exception. The active implementation path is the linked open PR, not current main.
