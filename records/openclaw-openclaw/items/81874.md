## Summary

Keep open: current main still has source-reproducible Telegram forum-topic send paths that can retry without `message_thread_id`, and https://github.com/openclaw/openclaw/pull/81885 is an open closing PR rather than a merged fix.

## What This Changes

The issue asks OpenClaw to preserve Telegram forum-topic targeting for replies, previews, final sends, tool sends, and fallbacks so topic-qualified sessions do not answer in General or the wrong topic.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/81885 or an equivalent Telegram transport fix that preserves non-General topic IDs or fails visibly, with focused regressions and live Telegram topic proof.

## Reproduction Assessment

Yes, source-reproducible: a forum-topic preview or generic send with `message_thread_id` can hit `message thread not found`, delete the topic id, and retry into the parent chat. I did not run a live Telegram forum-topic probe.

## Solution Assessment

Yes, the best direction is narrow: remove threadless retry for topic-qualified Telegram sends while preserving the documented General-topic `threadId=1` omission, then update regression tests and live proof.
