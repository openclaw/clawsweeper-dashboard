Keep open: current main still has a source-reproducible Telegram forum-topic leak path, and the linked closing PR is open rather than merged. The issue should stay open while https://github.com/openclaw/openclaw/pull/81885 is reviewed, landed, or superseded.

## What This Changes

The issue reports Telegram forum-topic replies appearing in General or another topic despite topic-qualified Telegram sessions.

## Best Possible Solution

Review and land or supersede https://github.com/openclaw/openclaw/pull/81885, preserving the General-topic exception while failing closed for explicit non-General topic sends; leave broader async/cron route preservation to https://github.com/openclaw/openclaw/pull/76920 or a follow-up.

## Reproduction Assessment

Yes, at source level: current main retries Telegram forum-topic sends and draft previews without `message_thread_id` after `message thread not found`. I did not run a live Telegram forum probe in this read-only review.

## Solution Assessment

Yes: the narrow maintainable fix is to fail closed for explicit non-General topic sends instead of stripping topic metadata, while preserving the documented General-topic exception. The active implementation path is the linked open PR, not current main.
