## Summary

Keep open: current main still has source-reproducible Telegram topic send paths that can retry without `message_thread_id`, and the linked closing PR is still open rather than merged.

## What This Changes

Telegram forum-topic replies should remain in the source topic for all assistant, preview, tool-result, and fallback send paths.

## Best Possible Solution

Review and land or revise https://github.com/openclaw/openclaw/pull/81885, preserving the General-topic exception and adding live Telegram topic proof before closing this issue as fixed.

Yes, at source level: current main retries several forum-topic send paths after `message thread not found` by deleting `message_thread_id`. I did not run live Telegram, so this is source-reproducible rather than live-reproduced.

## Solution Assessment

Yes: the narrow maintainable fix is to remove threadless retry for topic-qualified Telegram sends while preserving the documented General-topic omission and adding regression coverage.
