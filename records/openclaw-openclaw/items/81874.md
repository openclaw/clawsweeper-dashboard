## Summary

Keep open: current main still has a source-reproducible Telegram forum-topic leak, and the linked fix PR is open rather than merged. Policy says this issue should stay open while the closing PR is reviewed or replaced.

## What This Changes

Telegram forum-topic replies should preserve the topic `message_thread_id` on all send paths instead of falling back into General or another topic.

## Best Possible Solution

Review and land or supersede https://github.com/openclaw/openclaw/pull/81885, with live Telegram proof or maintainer override as needed, then close this issue from the merged fix.

## Reproduction Assessment

Yes at source level: current main retries forum-topic sends and draft previews without `message_thread_id` after Telegram reports `message thread not found`. I did not run a live Telegram forum probe in this read-only review.

## Solution Assessment

Yes: the narrow maintainable fix is to fail closed for explicit non-General topic sends instead of stripping topic metadata, while preserving the documented General-topic exception. The linked PR is the active implementation path but is not merged.
