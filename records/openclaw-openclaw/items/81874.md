## Label Justifications

- `P1`: The report covered a real Telegram channel workflow where replies could surface in the wrong forum topic.
- `impact:message-loss`: The core symptom was Telegram replies being misrouted to General or another topic.
- `impact:session-state`: The visible reply left the topic-qualified session context, making the conversation appear to lose state.
- `impact:security`: A wrong-topic Telegram reply can expose conversation content to the wrong audience.

## Summary

Current `main` and the `v2026.5.18` release now remove the unsafe Telegram thread-not-found retry path and add regression coverage for topic-qualified sends, so the reported forum-topic leak is fixed well enough to close.

## What This Changes

The issue reports Telegram forum-topic replies sometimes being sent to General or the wrong topic even when the inbound session is topic-qualified.

## Best Possible Solution

Keep the shipped fail-closed topic-send behavior, and track any remaining Telegram topic edge cases in narrower follow-up issues or PRs rather than reopening this fixed report.

## Reproduction Assessment

No against current `main`: the old source-level path that retried without `message_thread_id` is gone, and tests now assert one topic-qualified call plus a surfaced error. The original report was high quality and matched the pre-fix source behavior.

## Solution Assessment

Yes: failing closed for explicit non-General topic sends is the narrow maintainable fix, while preserving the documented General-topic exception. Current `main` also has adjacent topic-routing hardening for broader follow-up cases.
