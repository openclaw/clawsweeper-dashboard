## Summary

Keep open: the real Feishu/Lark proof is now adequate and the security pass is clear, but the amended patch appears to regress normal Feishu group `root_id` replies by sending to the root instead of the triggering message.

## What This Changes

The PR updates the Feishu plugin, docs, changelog, and regression tests to preserve topic reply anchors across assistant replies, message-tool sends, media/cards, reactions, and subagent completion notices.

## Best Possible Solution

Keep the Feishu-plugin topic-anchor fix, but preserve current normal `group` `root_id` behavior while applying threaded delivery only to topic, direct-topic, or explicitly configured thread paths.

## Reproduction Assessment

Yes, source-reproducible: current main shows the topic/configured-thread routing problem in the Feishu reply-target code, and the amended PR body includes before/after Feishu live proof. I did not perform a live Feishu run in this read-only review.

## Solution Assessment

No, not yet: the Feishu-plugin boundary and proof are appropriate, but the current patch also changes the established normal-group `root_id` behavior. The safer fix is to keep the new topic anchoring while excluding plain `groupSessionScope: "group"` `root_id` events from root-targeted threaded delivery.
