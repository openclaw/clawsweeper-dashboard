## Summary

Keep open. Current main still has the Feishu gateway per-account startup fan-out that can enter bot-identity probes concurrently, and this PR is a focused fix path, but it needs real behavior proof, a rebase/re-review, and coordination with related Feishu startup-timeout work before merge.

## What This Changes

The branch adds a shared Feishu startup bot-identity probe queue and regression tests for serialization, failure isolation, pre-aborted callers, and aborted queued callers.

## Best Possible Solution

Land a Feishu-owned startup probe queue that covers gateway account fan-out while preserving abort behavior and monitor lifecycle, then reconcile timeout policy with the related Feishu startup reports before merge.

## Reproduction Assessment

Yes, at source level: current main can run up to four Feishu account start tasks concurrently, and the accountId path fetches bot identity directly without a shared queue. Live Feishu credentials would be needed to re-exercise the external vendor timeout itself.

## Solution Assessment

Yes, with proof and rebase. Queuing at `fetchBotIdentityForMonitor` is the narrow Feishu-owned fix that covers gateway fan-out and direct callers without serializing all channel startup in core.
