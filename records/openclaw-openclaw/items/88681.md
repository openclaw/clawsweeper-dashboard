This PR adds runtime plugin load progress diagnostics and appends attempted, completed, and in-flight plugin details to isolated cron runtime-plugin timeout messages.

## Best Possible Solution

Land the diagnostic-only loader/watchdog path after redacted real cron or gateway timeout output confirms attempted, completed, and in-flight plugin details; keep the broader follow-ups in https://github.com/openclaw/openclaw/issues/87327 separate.

## Solution Assessment

Is this the best way to solve the issue? Yes for the first ask: adding diagnostic-only progress at the loader and formatting it in the existing cron watchdog is narrower than changing plugin load semantics; per-plugin timeouts, retries, and metrics remain separate work in https://github.com/openclaw/openclaw/issues/87327.
