## Summary

Keep this PR open, but it should not merge yet: the patch adds a new gateway/UI feature while leaving the RPC undiscoverable, breaking Overview test type coverage, bypassing the Control UI i18n pipeline, and still lacking inspectable visible UI proof.

## What This Changes

The PR adds a read-only `system.workspaceBoundaryStatus` gateway method and a Control UI Overview card that displays a conservative workspace-boundary unavailable state.

## Best Possible Solution

If maintainers want this surface, land it as a documented and discoverable read-scoped gateway method with protocol/list/test coverage, localized Overview strings, updated render-test defaults, and inspectable real UI proof.

## Reproduction Assessment

Yes for the review blockers: source inspection shows the new RPC is absent from feature discovery and the card strings bypass i18n, while CI reproduces the required `OverviewProps` type break. I did not run local tests because this review is read-only.

## Solution Assessment

No, not as currently proposed. The narrow maintainable path is to make the new RPC discoverable and covered, keep Overview test props valid, route strings through Control UI i18n, and get maintainer approval for the new gateway/UI status concept.
