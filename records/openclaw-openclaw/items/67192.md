## Summary

Keep open. Current main still has the reported non-string stream-content failure mode in the OpenAI-compatible completions stream parser, and the issue is already paired with open PR #67203, which remains the active implementation candidate rather than a landed fix.

## What This Changes

Fix the Mistral `/think` streaming crash by normalizing Mistral reasoning/content block payloads instead of letting object or array chunks become text output.

## Best Possible Solution

Land or replace #67203 with a focused parser change in `src/agents/openai-transport-stream.ts` that only treats string `delta.content` as text, normalizes Mistral reasoning/content block arrays or objects into thinking/text deltas, and adds regression tests plus the required user-facing changelog entry.

## Reproduction Assessment

Yes. A deterministic completions-stream fixture containing object/array `choice.delta.content` or array/object `reasoning_content` would hit the current text append or string-only fallback path; the reporter's `/think high` Mistral steps align with the enabled Mistral thinking route.

## Solution Assessment

Yes. The best fix is a narrow OpenAI-compatible stream parser normalization with regression coverage; channel-side filtering or session cleanup would only hide already-corrupted output.
