## Summary

Keep open. Current main still has the reported non-string stream-content failure mode: completions streaming accepts any truthy `delta.content` as text before reasoning parsing, while fallback reasoning fields only accept strings. The issue is also paired with open PR #67203, which remains the active implementation candidate.

## What This Changes

The issue reports that Mistral `mistral-small-latest` with `/think high` can stream non-string reasoning/content blocks that OpenClaw coerces to `[object Object]`, corrupting channel output and session state.

## Best Possible Solution

Land or replace #67203 with a focused parser fix in `src/agents/openai-transport-stream.ts` that treats only string `delta.content` as text, normalizes Mistral block-array/object reasoning payloads into thinking/text deltas, and adds regression coverage for those chunk shapes.

## Reproduction Assessment

Yes. A deterministic completions-stream fixture with object/array `choice.delta.content` or block-array `reasoning_content` would hit the current text append or string-only fallback path; the reporter’s `/think high` Mistral steps are consistent with that source path.

## Solution Assessment

Yes. The narrow maintainable solution is to fix the OpenAI-compatible stream parser and its tests; channel-side cleanup or session repair would only hide already-corrupted output.
