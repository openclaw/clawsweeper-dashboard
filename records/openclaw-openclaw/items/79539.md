## Summary

Keep open: the reported ACPX diagnostic bug is valid, but the latest PR head writes a parallel projected stream from the OpenClaw wrapper instead of honoring ACPX's own event-log contract, so it needs changes before merge.

## What This Changes

The PR adds an ACPX `runTurn` wrapper that reads `eventLog.active_path`, validates it under the ACPX sessions directory, appends projected NDJSON frames, and adds focused event-log stream tests.

## Best Possible Solution

Persist real ACP JSON-RPC `session/update` notifications through ACPX's own event-log writer and then consume that dependency update from the OpenClaw ACPX plugin.

## Reproduction Assessment

Yes, at source level: current main advertises ACPX `eventLog.active_path` through `acpx` but the OpenClaw wrapper only delegates `runTurn` without writing that path. The PR also includes a before-fix deployed-container `ls` repro, but I did not run a live ACP session in this read-only review.

## Solution Assessment

No. The wrapper-level projected append path is not the best fix because ACPX 0.7.0 already defines a JSON-RPC event-log writer with locking, rotation, metadata, and checkpoint semantics; the fix should be dependency-owned and then consumed here.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Write ACP JSON-RPC frames to the ACPX event log:** `extensions/acpx/src/runtime.ts:938-944`
  - body: This appends projected OpenClaw events shaped as `{ _format, params: { update } }` into the advertised ACPX stream. ACPX's event-log contract accepts real JSON-RPC messages and its session-update parser requires `jsonrpc: "2.0"`, `method: "session/update"`, and `params.sessionId`, so consumers following `eventLog.active_path` will not get a valid ACPX event stream. Move the persistence to the ACPX runtime/writer or reuse a dependency seam that writes the raw notification.
  - confidence: 0.9
- **[P2] Use the ACPX event writer instead of appendFile:** `extensions/acpx/src/runtime.ts:916-947`
  - body: The new writer bypasses ACPX's stream lock, segment rotation, `lastSeq` and `eventLog.last_write_at` updates, and checkpoint behavior. Long sessions can exceed `max_segment_bytes`, concurrent writers can lose ordering guarantees, and the session record metadata stays stale even though the advertised stream is changing.
  - confidence: 0.87
