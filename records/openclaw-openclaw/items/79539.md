## Summary

Keep open: current main appears to have the advertised-path bug and the PR is in the right ACPX extension area, but the patch still writes an unconstrained persisted path, emits partial/lossy frames, and only supplies mock-based after-fix proof.

## What This Changes

This PR wraps ACPX `runTurn` events with a fire-and-forget writer that creates the parent directory and appends projected `AcpRuntimeEvent` NDJSON frames to the session record's `eventLog.active_path`, plus a new unit test file.

## Best Possible Solution

Implement the ACPX writer with a derived or constrained stream path, complete session/update JSONL semantics or explicit maintainer-approved lossy semantics, a changelog entry, and real ACP run proof.

## Reproduction Assessment

Yes, at source level: current main advertises `eventLog.active_path` through `acpx@0.6.1` while `AcpxRuntime.runTurn` only delegates events and never writes that path. The contributor also supplied a deployed-container before-fix repro; I did not run a live ACP session in this read-only review.

## Solution Assessment

No. The extension boundary is plausible, but this patch should constrain or derive the write path and either emit complete `session/update` JSONL frames or get explicit maintainer acceptance for the lossy projected format.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Validate the event log path before appending:** `extensions/acpx/src/runtime.ts:895-896`
  - body: `eventLog.active_path` is loaded from the persisted ACPX session record and passed to `mkdir`/`appendFile` unchanged. A stale or corrupted record can redirect event data to an unintended writable path as the Gateway user, so derive the path from the trusted ACPX record id or constrain it to the expected ACPX sessions directory before writing.
  - confidence: 0.84
- **[P2] Emit complete session/update JSONL frames:** `extensions/acpx/src/runtime.ts:892-894`
  - body: The stream lines only contain `params.update` and omit `jsonrpc`, `method: "session/update"`, and `params.sessionId`. The acpx runtime converts real updates from full JSON-RPC notifications and OpenClaw's ACP schema requires `sessionId`, so consumers tailing the advertised path cannot reliably parse these as ACP session updates.
  - confidence: 0.82
- **[P3] Add the required changelog entry:** `extensions/acpx/src/runtime.ts:879`
  - body: This is a user-facing ACPX diagnostic fix, but the PR only changes runtime code and tests. Repo policy requires user-facing fixes to add a `CHANGELOG.md` entry under the active release fixes section.
  - confidence: 0.88

## Security Review

Concerns:

- **[medium] Unvalidated event log path write:** `extensions/acpx/src/runtime.ts:895`
