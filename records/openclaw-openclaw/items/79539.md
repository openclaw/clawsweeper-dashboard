## Summary

Keep this PR open, but not merge-ready: the reported ACPX diagnostic gap is real, while the branch writes a projected OpenClaw-side stream on the legacy `runTurn` path instead of the current `startTurn` path or the ACPX-owned raw event-log layer.

## What This Changes

The PR adds an ACPX runtime wrapper writer that appends projected per-event NDJSON frames to the session record's advertised `eventLog.active_path` and adds tests around that `runTurn` behavior.

## Best Possible Solution

Move the event-log write into ACPX or consume a released ACPX writer from this plugin; if maintainers intentionally accept an OpenClaw-side mitigation, it should wrap the current startTurn path, define the projected format contract, update metadata expectations, and include real ACP proof.

## Solution Assessment

No. The current PR is a plausible mitigation for the old runTurn-only surface, but the best fix is ACPX-owned raw notification logging or an explicitly approved OpenClaw-side wrapper that covers current `startTurn` behavior and defines the projected format.

Full review comments:

- **[P1] Cover the current startTurn path:** `extensions/acpx/src/runtime.ts:926-946`
  - body: This writer only runs while `runTurn` is consumed. On current main, `consumeAcpTurnStream` calls `runtime.startTurn` whenever the backend provides it, and `AcpxRuntime` now does provide `startTurn`, so normal ACP manager turns would bypass this loop after the branch is rebased. Put the writer on the `startTurn(...).events` path or a shared wrapper used by both turn APIs.
  - confidence: 0.89
- **[P1] Do not write projected frames as the ACPX stream contract:** `extensions/acpx/src/runtime.ts:938-940`
  - body: The advertised path belongs to ACPX session metadata, but these frames are `_format: "projected"` OpenClaw envelopes and omit the raw JSON-RPC `jsonrpc`, `method: "session/update"`, and `params.sessionId` shape that ACPX emits internally. This creates a second diagnostic format at the canonical ACPX path; move the write into ACPX's raw notification layer or require an explicit projected-stream contract before merge.
  - confidence: 0.86
- **[P2] Keep event-log metadata consistent with writes:** `extensions/acpx/src/runtime.ts:946`
  - body: The dependency record advertises segment count, max segment bytes, max segments, and last write fields, but this direct `appendFile` path never updates the record or rotates/checkpoints the stream. Operators inspecting the session record can still see stale metadata even while the stream grows, and long sessions can violate the advertised caps.
  - confidence: 0.78
