## Summary

Keep open: this PR addresses a real ACPX diagnostic bug, and the latest head fixed the earlier path-validation concern, but it is not merge-ready because after-fix proof is still test/mock-only and the appended frame shape is not a parseable `session/update` JSON-RPC frame.

## What This Changes

The PR wraps ACPX `runTurn` to validate `eventLog.active_path`, create its parent directory once, append projected JSONL frames for yielded runtime events, and add event-log stream tests.

## Best Possible Solution

Land a constrained ACPX event-log writer only after the stream format is parseable or explicitly documented as projected, with a changelog entry and redacted real ACP run proof; use an upstream ACPX callback if raw wire fidelity is required.

## Reproduction Assessment

Yes at source level: current main advertises ACPX `eventLog.active_path` through the pinned `acpx` package while OpenClaw's ACPX wrapper delegates `runTurn` without writing that path. The PR also includes a deployed-container before-fix repro, but I did not run a live ACP session in this read-only review.

## Solution Assessment

No. The extension-level write hook is plausible and the latest head constrains the path, but the frame shape should be parseable/complete or explicitly accepted as projected, and the PR still needs real after-fix behavior proof.
