## Summary

Keep open: the bug is valid, but this PR should not merge as-is because it writes projected, non-ACP JSON-RPC frames via a direct append path, bypasses ACPX event-writer mechanics, has a Windows path-containment gap, and lacks real after-fix ACP run proof.

## What This Changes

The PR adds an ACPX `runTurn` event-log writer and tests that append projected JSONL frames to the session record's advertised `eventLog.active_path`.

## Best Possible Solution

Move the stream persistence into ACPX or expose/reuse its raw-message event writer, then update the OpenClaw ACPX dependency and keep this wrapper focused on integration tests and wiring.

## Reproduction Assessment

Yes, at source level: current main advertises `eventLog.active_path` through `acpx@0.7.0`, while OpenClaw's wrapper still only delegates `runTurn`. The PR also includes before-fix deployed-container evidence, but I did not run a live ACP session in this read-only review.

## Solution Assessment

No. The wrapper hook is not the best fix because it writes a parallel projected stream and bypasses ACPX's event-log writer; the maintainable path is dependency-owned raw event persistence or a public ACPX seam reused from OpenClaw.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Use a platform-safe path containment check:** `extensions/acpx/src/runtime.ts:902`
  - body: This guard only rejects relative paths starting with `..` or `/`. On Windows, `path.relative()` returns an absolute different-drive path such as `D:\escape\file.ndjson`, which passes both checks and then reaches the new `fs.appendFile` sink. Use `path.isAbsolute(rel)` or an equivalent platform-safe containment check before trusting the session record path.
  - confidence: 0.88
- **[P2] Write valid ACP session/update frames:** `extensions/acpx/src/runtime.ts:938-941`
  - body: The stream line written here is a projected object with `_format` and nested `params.update`, but ACPX's event-log contract validates ACP JSON-RPC messages and its session-update extractor requires `method: "session/update"` plus `params.sessionId`. Consumers replaying the advertised event log will not treat these as session/update frames.
  - confidence: 0.91
- **[P2] Use ACPX event writer mechanics for the stream:** `extensions/acpx/src/runtime.ts:946`
  - body: Direct `fs.appendFile` bypasses ACPX's `SessionEventWriter`, so these writes do not take the stream lock, rotate segments, increment `lastSeq`, update `last_write_at`, or checkpoint the session record. Long or concurrent sessions can leave the advertised event-log metadata stale or out of order.
  - confidence: 0.86

## Security Review
