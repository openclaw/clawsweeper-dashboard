## Summary

Keep open: the underlying ACPX diagnostic bug is real on current main, but this PR should not merge as-is because it writes a wrapper-side projected stream that does not match the ACPX event-log contract and bypasses the dependency's canonical event-writer mechanics.

## What This Changes

The PR changes the ACPX wrapper to read `eventLog.active_path`, append projected NDJSON for each yielded runtime event, validate the path under the ACPX sessions directory, and add regression tests.

## Best Possible Solution

Move persistence into ACPX's dependency-owned event writer or expose that writer/callback through ACPX, then consume the released package from OpenClaw with a real ACP smoke showing the advertised stream file and contents.

## Reproduction Assessment

Yes at source level: current main advertises `eventLog.active_path` through `acpx@0.7.0`, while OpenClaw's ACPX wrapper `runTurn` only delegates and does not write that path. I did not run a live ACP session in this read-only review.

## Solution Assessment

No. The wrapper-side direct append is not the best fix because it bypasses ACPX's JSON-RPC validation, stream lock, segment rotation, sequence metadata, and checkpointing; the narrower maintainable fix belongs in ACPX or an ACPX-exposed seam.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Write ACP JSON-RPC frames to the advertised log:** `extensions/acpx/src/runtime.ts:938-940`
  - body: The advertised ACPX event log is a JSON-RPC stream, but this line writes only `{ _format, params: { update } }`. Consumers and ACPX helpers that expect `jsonrpc: "2.0"`, `method: "session/update"`, and `params.sessionId` will ignore or reject these entries, so the diagnostic path is still not a faithful event log.
  - confidence: 0.88
- **[P2] Use the ACPX event writer instead of direct appends:** `extensions/acpx/src/runtime.ts:946`
  - body: The direct `fs.appendFile` path bypasses ACPX's stream lock, segment rotation, `lastSeq`, `last_write_at`, and record checkpointing. Long or concurrent sessions can leave the advertised metadata stale and exceed the segment limits that the session record promises.
  - confidence: 0.86
