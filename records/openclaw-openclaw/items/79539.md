## Summary

Keep open: this is a focused ACPX bug-fix PR for a real diagnostic gap, but the current patch still has blocking correctness/security issues and lacks after-fix real ACP run proof.

## What This Changes

The PR loads the ACPX session record during `runTurn`, validates `eventLog.active_path`, creates its parent directory, appends projected JSONL event frames, and adds ACPX runtime tests.

## Best Possible Solution

Land an ACPX event-log writer only after it emits a parser-readable projected or full `session/update` shape, has cross-platform path containment, and includes redacted real ACP run proof.

## Reproduction Assessment

Yes, at source level: current main pins `acpx` 0.7.0, whose session record advertises `eventLog.active_path`, while OpenClaw's ACPX `runTurn` only delegates and has no writer. I did not run a live ACP session because this review is read-only.

## Solution Assessment

No, not yet: the `runTurn` hook is plausible, but the current frame shape is not parser-readable for the advertised event log and the path guard is incomplete on Windows. The PR also needs real after-fix ACP run proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Emit a parser-readable projected event frame:** `extensions/acpx/src/runtime.ts:938-940`
  - body: The appended line has `_format` plus nested `params.update`, but ACPX only reads nested updates when the object is a `method: "session/update"` notification; otherwise it looks for top-level `sessionUpdate` or `type`. As written, consumers following the advertised stream will ignore this frame, and text/status events would also be mislabeled as `sessionUpdate: "text_delta"` or `"status"`. Emit a full `session/update` notification or a top-level projected `{ type, tag, ...event }` shape.
  - confidence: 0.87
- **[P2] Reject Windows drive escapes in the path guard:** `extensions/acpx/src/runtime.ts:901-902`
  - body: The containment check only rejects relative paths beginning with `..` or `/`. On Windows, `path.relative()` for a different drive can return an absolute drive path such as `D:\tmp\x.stream.ndjson`, which passes this predicate and lets a manipulated session record write outside the trusted ACPX sessions directory. Use `path.isAbsolute(rel)` or equivalent drive-aware validation before accepting the path.
  - confidence: 0.9

## Security Review
