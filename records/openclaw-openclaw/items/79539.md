## Summary

Keep open: the underlying ACPX event-log bug is valid, but this PR should not land as-is because it writes a parallel projected stream instead of the ACPX event-log contract and still lacks real after-fix ACP run proof.

## What This Changes

The PR adds an ACPX `runTurn` wrapper that reads `eventLog.active_path`, validates it under the trusted ACPX sessions directory, creates the parent directory, fire-and-forget appends projected NDJSON frames, and adds focused tests.

## Best Possible Solution

Persist raw ACP `session/update` messages through ACPX's event-log writer or an exposed ACPX runtime seam, then bump the OpenClaw ACPX plugin to that released package with real ACP smoke proof.

## Reproduction Assessment

Yes at source level: current main advertises ACPX `eventLog.active_path` through the pinned `acpx` dependency while the OpenClaw wrapper does not write that stream. I did not run a live ACP session in this read-only review, but the PR body includes a before-fix deployed-container repro.

## Solution Assessment

No. The extension-level hook makes the path non-empty, but it bypasses ACPX's event-log contract and writes projected OpenClaw events where standard ACP JSON-RPC frames are expected.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Write valid ACP session/update frames:** `extensions/acpx/src/runtime.ts:938-940`
  - body: The advertised ACPX event log is expected to contain ACP JSON-RPC, but this appends `{ _format, params: { update } }` without `jsonrpc`, `method: "session/update"`, or `params.sessionId`. ACPX readers that extract session updates from the stream will ignore these lines, so the fix should persist the raw notification or use a separate documented projected-log path.
  - confidence: 0.9
- **[P2] Use ACPX's event writer instead of parallel appends:** `extensions/acpx/src/runtime.ts:946`
  - body: This `appendFile` path never takes ACPX's stream lock, rotation, `lastSeq`, `last_write_at`, or checkpoint logic. Long or concurrent sessions can drift from the advertised event-log metadata; move the persistence into ACPX or expose/reuse its writer rather than maintaining a second stream writer in the wrapper.
  - confidence: 0.88
