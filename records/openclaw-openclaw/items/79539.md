## Summary

Keep open: the PR targets a real acpx diagnostic bug, but the current patch should not merge because the writer can still leave the advertised stream missing, emits a non-JSON-RPC frame shape, trusts persisted path metadata, and still lacks observed after-fix behavior proof.

## What This Changes

The PR adds an acpx `runTurn` wrapper that appends one JSONL frame per yielded runtime event to the session record's advertised `eventLog.active_path`, plus a new event-log stream test file.

## Best Possible Solution

Implement the writer at the acpx extension boundary with a derived or validated session stream path, parent-directory creation, valid ACP JSON-RPC `session/update` JSONL frames, and explicit maintainer acceptance of the lossy-versus-raw capture semantics.

## Reproduction Assessment

Yes, at source level: current main advertises acpx event-log metadata through the pinned dependency, while `AcpxRuntime.runTurn` only delegates events and never writes the advertised stream. I did not run a live ACP session in this read-only review.

## Solution Assessment

No. The PR is aimed at the right extension boundary, but the current implementation is incomplete because it misses directory creation, emits a partial frame shape, trusts persisted path metadata, and still needs real after-fix proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Create the event stream directory before appending:** `extensions/acpx/src/runtime.ts:893`
  - body: The PR's own repro starts with `/home/codeclaw/.acpx/sessions/` missing, but this calls `appendFile` on the advertised file directly. `appendFile` fails with `ENOENT` when the parent directory is absent, and the catch only logs, so the advertised stream can remain missing while the mocked unit test passes.
  - confidence: 0.93
- **[P2] Emit JSON-RPC session/update frames:** `extensions/acpx/src/runtime.ts:890-892`
  - body: The advertised stream is an ACP event log, and acpx queues session updates as JSON-RPC notifications before translating them. This writes only `{ params: { update: ... } }`, omitting `jsonrpc: "2.0"` and `method: "session/update"`, so consumers expecting ACP JSON-RPC lines cannot identify these frames.
  - confidence: 0.9
- **[P2] Constrain the event log path before writing:** `extensions/acpx/src/runtime.ts:881-893`
  - body: This appends to whatever `eventLog.active_path` was persisted in the session record. A stale or corrupted record can redirect event data to an unintended writable path as the gateway user; derive the stream path from the trusted session id or validate it stays under the expected acpx session directory.
  - confidence: 0.8

## Security Review

Status: needs_attention

Summary: The diff introduces a new file-write sink whose destination is read directly from persisted ACP session metadata.

Concerns:
