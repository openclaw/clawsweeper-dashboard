## Summary

Keep open: the PR targets a real ACPX diagnostic bug that is still present on current main and the latest release, but the proposed patch still has blocking frame-shape and cross-platform path-validation issues, plus only mock/unit after-fix proof.

## What This Changes

The PR wraps ACPX runTurn to read and validate the session record eventLog.active_path, create its parent directory, append projected NDJSON frames for yielded runtime events, and add focused runtime tests.

## Best Possible Solution

Keep the event-log path honest with an owner-accepted parseable/projected format, hardened path validation on every supported OS, and proof from a real ACP run.

## Reproduction Assessment

Yes at source level: current main and v2026.5.7 advertise the ACPX event-log path via the acpx session record but delegate runTurn without any writer. The PR discussion also supplies a before-fix container ENOENT repro, but I did not run a live ACP session in this read-only review.

## Solution Assessment

No: wrapping runTurn is a plausible owner-local hook, but the current patch still emits a frame shape the existing ACPX parser ignores and leaves a Windows trusted-base bypass in the path guard. Maintainers should also explicitly accept projected/lossy semantics or require an upstream raw-message ACPX seam.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Reject Windows absolute relative paths:** `extensions/acpx/src/runtime.ts:902`
  - body: The guard rejects `..` and `/`, but on Windows `path.relative()` can return a different-drive absolute path like `D:\...`, which does not start with either string. A manipulated session record could still drive the new mkdir/appendFile sink outside the trusted ACPX sessions base; reject `path.isAbsolute(rel)` or the platform-equivalent absolute/UNC case before writing.
  - confidence: 0.87
- **[P2] Emit a parser-recognized event-log frame:** `extensions/acpx/src/runtime.ts:938-940`
  - body: The writer appends `{ _format, params: { update } }`, but the ACPX parser recognizes `method: "session/update"` with `params.update` or a top-level `sessionUpdate`. Consumers using the existing parser will ignore this advertised stream, so emit a parser-recognized projected frame or document/use a top-level projected event shape.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: Needs attention: the diff adds filesystem writes from session metadata, and the latest trusted-base guard misses Windows different-drive absolute paths.
