## Summary

Keep open: the reported bug is source-reproducible, but this PR still has blocking correctness/security issues and only mock-level after-fix proof.

## What This Changes

The PR adds an ACPX `runTurn` wrapper that appends yielded runtime events to the session record's `eventLog.active_path`, plus a new event-log-stream test file.

## Best Possible Solution

Implement the writer at the ACPX extension boundary with a derived or validated stream path, parent-directory creation, complete `session/update` JSONL frames or explicit maintainer-approved lossy semantics, and real ACP run proof.

## Reproduction Assessment

Yes, at source level: current main advertises `eventLog.active_path` through `acpx@0.6.1`, while `AcpxRuntime.runTurn` only delegates events and never writes that path. I did not run a live ACP session in this read-only review.

## Solution Assessment

No. The extension boundary is plausible, but the proposed implementation should create and constrain the target path and write a complete event frame, or get explicit maintainer acceptance for the lossy format.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Create the event stream directory before appending:** `extensions/acpx/src/runtime.ts:893`
  - body: The PR writes directly to `eventLogActivePath`, but the reported failing case has `/home/codeclaw/.acpx/sessions/` missing and OpenClaw's acpx session store only creates the OpenClaw state `sessions` directory. `appendFile` will fail with `ENOENT`, the catch only logs, and the advertised stream can remain missing while the unit spy passes.
  - confidence: 0.94
- **[P2] Write complete session/update JSON-RPC frames:** `extensions/acpx/src/runtime.ts:890-892`
  - body: This frame omits `jsonrpc`, `method: "session/update"`, and `params.sessionId`. The upstream runtime parses session updates from full JSON-RPC notifications, and OpenClaw's ACP schema requires `sessionId` in the notification params, so consumers of the advertised event stream cannot reliably treat these lines as ACP session/update frames.
  - confidence: 0.88
- **[P2] Validate the event log path before writing:** `extensions/acpx/src/runtime.ts:893`
  - body: The new writer appends to whatever `eventLog.active_path` was persisted in the session record. A stale or corrupted record can redirect event data to an unintended writable path as the gateway user; derive the path from the trusted ACPX record id or constrain it to the expected ACPX sessions directory before appending.
  - confidence: 0.8

## Security Review

Status: needs_attention

Summary: The diff introduces a file-write sink whose destination is read from persisted ACPX session metadata without validation.

Concerns:

- **[medium] Unvalidated event log path write:** `extensions/acpx/src/runtime.ts:893`
