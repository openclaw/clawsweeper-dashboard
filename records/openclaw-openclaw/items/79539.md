## Summary

Keep open. The underlying ACPX diagnostic bug is real and the contributor provided useful before-fix container evidence, but the current PR should not merge as-is because it writes a parallel projected stream from the OpenClaw wrapper instead of preserving the ACPX event-log contract, and it still lacks after-fix real ACP run proof.

## What This Changes

The PR adds an ACPX `runTurn` wrapper that reads `eventLog.active_path`, validates it against a trusted sessions directory, appends projected NDJSON frames for yielded events, and adds unit tests.

## Best Possible Solution

Move raw ACP JSON-RPC event persistence into `openclaw/acpx` or expose its writer/callback, then update OpenClaw to a published ACPX release with redacted live ACP proof.

## Reproduction Assessment

Yes at source level: current main advertises the ACPX package's event-log path through `acpx@0.7.0`, while OpenClaw's wrapper delegates `runTurn` without writing that path. I did not run a live ACP session in this read-only review.

## Solution Assessment

No. The wrapper-level write hook addresses the dead path superficially, but it bypasses the ACPX event-log writer contract and writes frames that dependency consumers will not recognize as `session/update` JSON-RPC.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Write ACP event-log frames through ACPX:** `extensions/acpx/src/runtime.ts:938-940`
  - body: The advertised event log is an ACPX session log, but this appends `_format` plus `params.update` without `jsonrpc: "2.0"`, `method: "session/update"`, or `params.sessionId`. ACPX's own validators/parsers expect real ACP JSON-RPC, so consumers following this path cannot replay or parse the stream as the dependency contract promises.
  - confidence: 0.9
- **[P2] Preserve ACPX event-log writer semantics:** `extensions/acpx/src/runtime.ts:942-946`
  - body: Direct `fs.appendFile` bypasses the dependency writer's lock, segment rotation, `lastSeq` increment, `last_write_at` update, and session-record checkpoint. Long or concurrent turns can leave the advertised log and metadata inconsistent; this should use or expose the ACPX event writer instead.
  - confidence: 0.87
- **[P2] Reject cross-drive event-log paths:** `extensions/acpx/src/runtime.ts:901-902`
  - body: The traversal guard checks `rel.startsWith("..") || rel.startsWith("/")`, but on Windows `path.relative(trustedBase, "D:\\...")` can return a drive-qualified absolute path that passes both checks. Use `path.isAbsolute(rel)` or an equivalent same-root check before enabling writes.
  - confidence: 0.78
