## Repair Work Prompt

Fix Telegram inbound single-message document/media loss during gateway restart windows. Source inspection shows durable isolated ingress is enabled by default, but extensions/telegram/src/bot-handlers.runtime.ts catches non-size media download failures, attempts a best-effort warning reply, releases dispatch dedupe, and returns; extensions/telegram/src/bot-core.ts then treats the update as completed, so a shutdown-window download or warning-send interruption can delete/ack the spooled update without media, assistant dispatch, or durable notice. Keep the fix in the Telegram plugin boundary. Make transient media download or shutdown-window failures retryable for spooled replay updates until the media is downloaded and dispatched, or until an explicit user-visible failure is durably delivered; preserve permanent oversize handling. Do not add a new config option or broad fallback. Add focused regression tests around spooled replay/single document media failure and update completion semantics. Do not edit CHANGELOG.md.
