## Summary

Current main implements the default Telegram polling durability path well enough for this report: Bot API updates are written to the durable ingress queue before media handling, stalled claims are recoverable after restart, and document media is downloaded during replay. The fix landed after v2026.6.10, so this is current-main-only rather than shipped in the latest release.

## What This Changes

Telegram document uploads sent during a gateway restart or upgrade window can be silently lost instead of being durably queued or surfaced as failed.

## Best Possible Solution

Keep the current-main polling-spool recovery and leave separate webhook ACK-before-durable-processing work tracked in its own issue.

## Reproduction Assessment

No live current-main failing reproduction was run. The reported v2026.6.10 polling-window failure is source-plausible, and current main has source and regression-test evidence for the fixed durable replay path.

## Solution Assessment

Yes for the default polling-mode document upload path. Current main persists updates before media download and recovers stalled claims, while webhook restart ACK durability remains a separate adjacent issue.
