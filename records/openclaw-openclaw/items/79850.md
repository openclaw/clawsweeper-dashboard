## What This Changes

The issue asks for model-provider requests to `host.docker.internal` and other `.internal` hostnames to work under the provider SSRF private-network/allowlist policy instead of being blocked.

## Best Possible Solution

Keep this issue open while maintainers review https://github.com/openclaw/openclaw/pull/79862 or replace it with a narrower fix that preserves explicit private-network trust boundaries.

## Solution Assessment
