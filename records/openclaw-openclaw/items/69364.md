## Summary

Keep open. Current main still exposes per-agent/session primitives and selected-agent Control UI panels, but no first-class cross-agent topology or Studio surface; the linked openclaw-studio prototype is useful evidence, while accepting this in core needs maintainer product and security direction rather than conservative cleanup.

## What This Changes

The issue requests a first-class Control UI cross-agent topology view for multi-agent teams, covering delegate, dataflow, sequence, config editing, tools/skills controls, and chat slices.

## Best Possible Solution

If maintainers accept this for core, start with a read-only, typed, allowlisted topology endpoint plus Control UI graph, then split config editing, tools/skills mutation, and chat into separate security-reviewed follow-ups.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives a high-confidence feature-gap check: current main has per-agent, session, task, file, tools, and skills primitives, but no joined topology route or Studio surface.

## Solution Assessment

Unclear. The direction is coherent, but the maintainable first slice should be a read-only topology MVP with explicit auth and filesystem boundaries before adding mutation or chat controls.
