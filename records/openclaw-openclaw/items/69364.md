## Summary

Keep open. Current main still lacks a joined Control UI topology/debugging surface for durable multi-agent teams, and the linked openclaw-studio prototype/RFC plus the adjacent orchestration issue make this a product/API/security design decision rather than conservative cleanup.

## What This Changes

Add a Control UI Studio/topology tab that visualizes delegate, dataflow, and sequence relationships across durable agents and supports per-agent config editing.

## Best Possible Solution

Keep this as a product/RFC item; if accepted for core, start with a read-only allowlisted topology contract and defer config/tools/skills/chat mutation to separate reviewed follow-ups.

Not applicable as a bug reproduction because this is a feature request. Source inspection gives a high-confidence feature-gap check: current main has separate Agents, Sessions, file, tools, skills, and task surfaces, but no joined team topology route or Gateway topology contract.

## Solution Assessment

Unclear. The requested direction is useful but broad; the narrowest maintainable path would be a read-only typed topology MVP first, with editing and chat controls split behind later product/security review.
