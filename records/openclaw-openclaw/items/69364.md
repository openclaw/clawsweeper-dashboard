## Summary

Keep open. Current main still lacks the requested joined Control UI topology/debugging surface, and the discussion plus related #52803 show this is a broader product/API direction question rather than a stale duplicate or a narrow missing widget.

## What This Changes

The issue requests a native Control UI cross-agent/team view that joins delegation, shared-workspace dataflow, task sequence, per-agent configuration, tools/skills overrides, and per-agent chat/status into one debugging and configuration surface.

## Best Possible Solution

Keep this open for maintainer product review. If maintainers want this in core, the safest first milestone is a read-only, allowlisted topology contract that joins existing agent/session metadata without scanning arbitrary symlinks or mutating config; editing, tools/skills changes, and per-agent chat should be split into later issues after API and security review.

## Reproduction Assessment

Not applicable as a bug reproduction. Source and docs inspection give a high-confidence verification path: current main has separate Agents, Sessions, Config, Skills, and Chat surfaces, but no cross-agent topology route, panel, RPC, or documented feature.

## Solution Assessment

Unclear. The full five-slice proposal is too broad to call the best implementation before maintainers choose the product direction, but a read-only topology MVP would be the narrowest maintainable core path if the direction is accepted; otherwise the linked openclaw-studio prototype is useful community surface evidence.
