## Summary

Keep open. Current main and v2026.6.8 still expose separate agent, file, session, config, and chat primitives but no joined command/dataflow/sequence topology surface; the linked Studio prototype/RFC makes this a maintainer product and security-boundary decision rather than conservative cleanup.

## What This Changes

The issue requests a Control UI relationship view for multi-agent teams that joins delegation, workspace dataflow, task sequence, per-session status, and later per-agent editing/chat flows.

## Best Possible Solution

Keep this open for maintainers to decide whether a core Control UI topology surface belongs in the default product and, if accepted, define the smallest gateway/API and symlink-security contract before UI work starts.

## Reproduction Assessment

Yes for the source-level feature gap: current main exposes per-agent files, flat sessions, and gateway primitives but no joined topology view. This is not a bug reproduction because no existing documented UI contract promises that cross-agent relationship surface.

## Solution Assessment

Unclear: the proposed Studio direction is coherent and backed by a prototype, but it is not yet proven to be the best core product shape. Maintainers should first decide core-vs-external scope and the gateway/symlink security boundary.
