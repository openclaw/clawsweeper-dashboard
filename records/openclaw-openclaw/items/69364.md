## Summary

Keep open. Current main still exposes per-agent/session primitives and a flat selected-agent Control UI surface rather than a first-class cross-agent topology or Studio view; the linked openclaw-studio prototype is useful evidence, but accepting this in core needs maintainer product/API/security direction rather than conservative cleanup.

Request a Control UI surface that visualizes and later edits relationships across multi-agent teams, including delegation, shared-workspace dataflow, task sequencing, per-agent config, tools, skills, and agent chat.

## Best Possible Solution

If accepted for core, start with a read-only, typed, allowlisted topology endpoint and Control UI graph, then split config editing, tools/skills mutation, and chat into separate security-reviewed follow-ups.
