## Summary

## What This Changes

The PR moves core doctor lint health checks to ordered doctor health contributions, appends extension registry checks afterward, adds inline contribution health-check support, updates docs/tests, and removes the temporary conversion-plan ledger.

## Best Possible Solution

Land the contribution-owned ordering only after the reserved core-ID collision rule is enforced consistently in both lint and `doctor --fix`, then have maintainers explicitly accept the compatibility and auth-provider behavior.

## Solution Assessment
