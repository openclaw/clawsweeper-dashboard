## Summary

Keep open. The ordering direction is useful and the terminal proof is strong, but the PR still leaves a repair-path namespace hole: `doctor --lint` rejects extension checks reusing core IDs, while `doctor --fix` extension repairs can still run a non-core check under a reserved `core/doctor/*` ID.

## What This Changes

This PR moves core doctor lint checks to ordered doctor health contributions, appends registered extension checks afterward, updates doctor docs/tests, and removes the temporary doctor health conversion-plan ledger.

## Best Possible Solution

Add the same reserved core-ID guard to the structured repair path, keep the contribution-owned ordering model, and then let maintainers explicitly accept the doctor ordering/auth-provider compatibility risk before merge.
