  - body: This drops `maybeRepairCanonicalApiKeyFieldAlias()` from the normal auth profile doctor contribution. Current main runs that repair before the rest of auth health, and the repair code says runtime auth parsing only reads canonical `key`/`keyRef`; without it, existing profiles that still contain `api_key` are no longer reported or rewritten by `doctor --fix`, so those provider credentials remain silently skipped.
  - confidence: 0.9
- **[P1] Keep resolving Gateway token SecretRefs:** `src/flows/doctor-health-contributions.ts:318-326`
  - body: The new Gateway auth path checks `auth.token` and then treats any `gatewayTokenRef` as unavailable without calling `resolveGatewayAuthToken()`. Current main resolves env/SecretRef-backed tokens, skips exec refs unless `--allow-exec` is present, and reports the detailed unresolved reason; this head would give wrong doctor output for existing SecretRef-backed Gateway token setups.
  - confidence: 0.91
- **[P1] Preserve exec SecretRef probe skipping:** `src/flows/doctor-health-contributions.ts:843-850`
  - body: `runGatewayHealthChecks()` now calls `checkGatewayHealth()` unconditionally. Current main first detects active exec SecretRefs and skips Gateway health probes unless `openclaw doctor --allow-exec` was requested, so this change crosses the command-execution boundary that currently protects configured secret resolvers.
  - confidence: 0.9
- **[P1] Forward allowExec to Gateway service repair:** `src/flows/doctor-health-contributions.ts:616-621`
