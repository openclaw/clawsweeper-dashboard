## Summary

Keep open: the fix is aimed at the right realtime payload boundary, but the Google validator still lets 65-128 character function names through even though Google caps FunctionDeclaration names at 64, so malformed Google tools can still poison Live config creation.

## What This Changes

This PR adds shared and provider-local realtime voice tool-name filtering for OpenAI and Google, with regression tests for malformed custom realtime tools.

## Best Possible Solution

Cap the Google realtime tool-name regex at 64 characters, add a boundary test for a 65-character Google name, then review the draft with the focused provider tests and any feasible live provider proof.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: current main reads custom tool names directly into shared resolver and provider payloads, and malformed provider-incompatible names reach OpenAI or Google session construction.

## Solution Assessment

No, not yet: provider-local quarantine is the right shape, but the Google implementation must match the 64-character FunctionDeclaration contract before this is the best fix.
