## Summary

This PR still appears necessary and the patch is focused: current main can preserve retry grace for per-attempt lifecycle errors but has no distinct final-fallback marker for the gateway to finalize immediately after provider fallback is exhausted. I found no blocking correctness or security findings; the remaining decision is maintainer acceptance of the session-state timing risk and normal CI.

## What This Changes

The PR emits `finalFailure: true` on final embedded provider failure, lets the gateway finalize that marked lifecycle error immediately, and adds focused runner and gateway regression tests.

## Best Possible Solution

Land the focused lifecycle-marker fix after CI and maintainer acceptance of the session-state timing change; keep https://github.com/openclaw/openclaw/issues/91730 open until this PR merges or a replacement lands.

## Reproduction Assessment

No executed current-main repro was run in this read-only review, but the failure is source-reproducible: current main has per-attempt lifecycle error deferral without a final fallback-exhausted marker, and the linked issue plus PR proof describe the stuck webchat/session state path.

## Solution Assessment

Yes, this is the best fix shape I found: the runner owns knowledge that fallback is exhausted, while the gateway only changes cleanup timing when that explicit marker is present.
