## Summary

Keep open. The Discord display bug is valid, but this PR is not merge-ready because it mixes the display-only fix with guild-slug-first `GroupSpace` changes that can bypass ID-keyed Discord policy lookup, and its fallback still does not preserve mixed ASCII/non-ASCII channel names.

## What This Changes

The PR adds a Unicode-preserving fallback for Discord display channel slugs when ASCII normalization returns empty, and changes Discord message/component `GroupSpace` construction to prefer guild slugs over guild IDs.

## Best Possible Solution

Implement a display-only Unicode-preserving Discord channel label helper on current main, cover pure and mixed non-ASCII names with regression tests, and leave config matching plus ID-first guild policy identity unchanged.

## Reproduction Assessment

Yes. A static reproduction is high-confidence: current normalization maps `실험` to an empty display slug and `baseline-검증` to `baseline`, and the process context only emits `GroupChannel` when that slug is non-empty.

## Solution Assessment

No. The display-fallback idea belongs in this area, but the proposed patch is not the best solution because it changes policy identity and only preserves Unicode when the ASCII slug is completely empty.
