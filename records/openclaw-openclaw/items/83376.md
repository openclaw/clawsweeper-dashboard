## Best Possible Solution

Resolve the conflict, implement a final-request-payload budget helper that clamps only configured or model caps to the remaining context, preserve omitted-field semantics, add focused regression coverage, and provide redacted provider proof.

## Solution Assessment

No. Dynamic per-call clamping is the right direction, but this branch currently cannot parse and the budgeting source should be the final provider payload, not pre-conversion context messages.
