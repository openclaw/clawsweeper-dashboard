## What This Changes

The PR moves isolated cron delete-after-run session cleanup into a shared helper, calls it from the isolated runner finally path, preserves delivery cleanup, and adds a regression test for delivery-none jobs.

## Best Possible Solution

Land this PR after ordinary maintainer review so https://github.com/openclaw/openclaw/issues/84707 can close with the merged fix, while leaving broader hook/session archival policy to https://github.com/openclaw/openclaw/issues/10253.

## Reproduction Assessment

Yes. Source inspection on current main shows `delivery.mode: "none"` makes delivery unrequested, while delete-after-run cleanup is only reached from delivery paths and not the runner finally block.

## Solution Assessment

Yes. The shared helper plus runner-final invocation is the narrowest maintainable fix because it preserves direct-delivery cleanup while covering delivery-none and terminal paths at the run boundary.
