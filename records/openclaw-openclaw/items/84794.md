## Summary

Keep open: this is a focused candidate fix for the linked isolated cron `deleteAfterRun` leak, and current `main` has not already implemented the same finalizer cleanup. I found no blocking patch defect, but the PR intentionally broadens session/transcript deletion timing, so the existing human-review gate should be cleared only after a maintainer accepts that session-state behavior.

## What This Changes

The PR extracts isolated cron delete-after-run session cleanup into a shared helper and calls it from the isolated runner finalizer while preserving delivery-dispatch cleanup.

## Best Possible Solution

Land this PR after maintainer acceptance of the broader delete-after-run cleanup semantics, while leaving adjacent failed-job deletion and hook cleanup policy to their separate issues.

## Reproduction Assessment

Yes. Source inspection shows delivery-none isolated cron runs bypass current delivery-local cleanup, and the PR includes live Gateway terminal proof of a retained session before cleanup and an empty sessions list afterward.

## Solution Assessment

Yes. Reusing the existing `sessions.delete` cleanup from the isolated runner finalizer is the narrowest maintainable fix because delivery-local cleanup cannot cover delivery-none and early terminal paths, and `sessions.delete` is already idempotent for missing sessions.
