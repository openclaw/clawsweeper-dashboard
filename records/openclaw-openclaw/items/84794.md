## Summary

Keep open for maintainer review: the patch appears correct with no blocking review findings and sufficient real Gateway proof, but it intentionally changes when persisted isolated cron session/transcript state is deleted, so the existing human-review gate should be cleared or kept by a maintainer before automerge proceeds.

## What This Changes

The PR extracts isolated cron delete-after-run cleanup into a shared helper, calls it from the isolated runner finalizer, and keeps delivery-dispatch cleanup routed through that helper.

## Best Possible Solution

Land this focused finalizer cleanup once maintainers accept deleteAfterRun as explicit permission to remove isolated cron run sessions and transcripts; keep hook cleanup, archival policy, and failed-job retention work separate.

## Reproduction Assessment

Yes. Current main creates run-scoped isolated cron sessions, delivery-none skips delivery-dispatch cleanup, and the runner finalizer currently disposes context without delete-after-run session cleanup.

## Solution Assessment

Yes. Moving the shared cleanup into the isolated runner finalizer is the narrow maintainable fix because delivery dispatch cannot cover delivery-none or runner-error terminal paths, while service-level reaping would be broader and delayed.
