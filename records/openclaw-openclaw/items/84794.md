## Summary

Keep open for maintainer review: the patch looks correct and the supplied terminal proof is sufficient, but it intentionally broadens `deleteAfterRun` so delivery-none and terminal isolated cron paths delete persisted session/transcript state.

## What This Changes

This PR extracts isolated cron delete-after-run cleanup into a shared helper, calls it from the isolated runner finalizer, preserves delivery-dispatch cleanup, and adds a delivery-none regression test.

## Best Possible Solution

Land this focused finalizer cleanup once maintainers accept `deleteAfterRun` as explicit permission to delete isolated cron run sessions and transcripts; keep broader hook and failed-job retention policy separate.

## Reproduction Assessment

Yes. Source inspection shows current main creates run-scoped isolated cron sessions, `delivery.mode: "none"` bypasses delivery cleanup, and the runner finalizer currently disposes context without delete-after-run session cleanup.

## Solution Assessment

Yes, with maintainer acceptance of the retention semantics. Reusing the existing Gateway-backed cron cleanup from the runner finalizer is the narrow maintainable fix because delivery dispatch cannot cover delivery-none or runner-error terminal paths.
