## Summary

Keep open for maintainer review: the patch appears correct and well focused, but it intentionally changes isolated cron `deleteAfterRun` terminal paths from retaining run sessions/transcripts to deleting them, so the human-review gate should be cleared before merge.

## What This Changes

The PR extracts isolated cron delete-after-run session cleanup into a shared helper, calls it from the isolated runner finalizer, keeps delivery-dispatch cleanup, and adds delivery-none regression coverage.

## Best Possible Solution

Land the focused finalizer cleanup after a maintainer accepts that `deleteAfterRun` should delete isolated cron sessions and transcripts across terminal paths, while leaving broader hook/session retention policy to separate tracked work.

## Reproduction Assessment

Yes, at source level. Current main creates run-scoped isolated cron sessions and has delivery-local cleanup, but delivery-none and some terminal paths bypass that cleanup before the runner finalizer disposes context.

## Solution Assessment

Yes. Reusing the existing cron-scoped `sessions.delete` cleanup contract from a shared helper and invoking it from the isolated runner finalizer is the narrow maintainable fix; the remaining question is maintainer acceptance of the session-state semantics.
