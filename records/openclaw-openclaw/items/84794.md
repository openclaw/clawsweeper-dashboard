## What This Changes

The PR moves isolated cron delete-after-run cleanup into a shared helper and invokes it from the isolated runner finalizer while preserving the delivery-dispatch cleanup path and adding delivery-none regression coverage.

## Best Possible Solution

Land or rework the focused finalizer cleanup once maintainers explicitly accept deleteAfterRun as permission to delete isolated cron run session and transcript state across terminal paths.

## Reproduction Assessment

Yes, at source level. Current main creates run-scoped isolated cron sessions, explicit delivery.mode none bypasses delivery-owned cleanup, and the runner finalizer currently disposes the run context without delete-after-run session cleanup.

## Solution Assessment

Yes, pending maintainer acceptance of retention semantics. Reusing the existing Gateway-backed cleanup from the isolated runner finalizer is the narrow owner-boundary fix because delivery dispatch cannot cover delivery-none or runner-error terminal paths.
