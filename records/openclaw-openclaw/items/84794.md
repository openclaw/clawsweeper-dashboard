Land the focused finalizer cleanup after a maintainer accepts that `deleteAfterRun` should delete isolated cron run sessions and transcripts across terminal paths; keep broader retention policy in separate issues.

## Solution Assessment

Yes. Reusing the existing cron-scoped `sessions.delete` cleanup in a shared helper and invoking it from the isolated runner finalizer is the narrow maintainable fix; broader retention/archival policy belongs outside this PR.
