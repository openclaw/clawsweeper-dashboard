## What This Changes

The PR extracts isolated cron delete-after-run cleanup into a shared helper, calls it from the isolated runner `finally` path, preserves delivery-dispatch cleanup, and adds a delivery-none regression test.

## Best Possible Solution

Land this focused finalizer cleanup once maintainers accept the existing `deleteAfterRun` flag as permission to delete isolated cron run sessions and transcripts; keep broader hook cleanup and failed-job retention policy in separate items.

## Reproduction Assessment

## Solution Assessment

Yes. Reusing the existing Gateway-backed cron cleanup from the runner finalizer is the narrow maintainable fix because delivery dispatch cannot cover delivery-none or runner-error terminal paths.
