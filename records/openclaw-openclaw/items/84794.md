Keep open for normal maintainer review: current main still lacks delete-after-run cleanup for isolated cron runs that do not enter delivery dispatch, while this PR moves the cleanup to run finalization, preserves the existing dispatch cleanup path, and has no blocking review findings.

## What This Changes

The branch extracts isolated cron delete-after-run session cleanup into a shared helper, invokes it from isolated runner finalization, and adds regression coverage for `delivery.mode: "none"`.

## Best Possible Solution

Land this PR after normal maintainer and CI gating, keeping delete-after-run cleanup in the cron runner finalization path while retaining dispatch cleanup for direct-delivery exits.

## Reproduction Assessment

Yes, source-reproducible: current main’s runner finalization has no delete-after-run cleanup, and delivery mode `none` does not enter the existing direct-delivery cleanup path. I did not execute the bug locally, but the PR includes live Gateway before/after proof and a focused regression test.

## Solution Assessment

Yes, this is the best fix shape: delivery dispatch only owns delivery exits, while runner finalization owns terminal run cleanup for success, errors, and no-delivery paths. Keeping the dispatch call through the shared helper preserves the existing direct-delivery behavior without duplicating the deletion logic.
