## What This Changes

The request asks for auto-TTS generation and delivery to stop blocking Telegram inbound message processing, optionally by delivering text first or adding an async delivery mode.

## Best Possible Solution

Keep the remaining async/background TTS design consolidated under https://github.com/openclaw/openclaw/issues/48179, and coordinate any text-first or separate voice-note delivery API with https://github.com/openclaw/openclaw/issues/42539.

## Reproduction Assessment

## Solution Assessment
