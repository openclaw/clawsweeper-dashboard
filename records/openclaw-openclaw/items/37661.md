## Summary

Keep open. Current main still has exact duplicate suppression and tool-call loop detection, but not a generic guard for assistant text that keeps growing by repeating the same phrase; the April follow-up also reports pure text repetition outside the original subagent-timeout trigger.

## What This Changes

The issue requests a runtime guard that stops assistant streaming/final output from repeating the same phrase many times and flooding user channels.

## Best Possible Solution

Add a conservative assistant-output repetition guard in the generic streaming/final reply path, with shared outbound duplicate/message-cap protection before channel delivery and focused regression coverage.

## Solution Assessment

## Repair Work Prompt

Fix the open assistant text repetition bug by adding a conservative guard in the generic assistant streaming/final reply path. Detect repeated phrase or n-gram growth across text_delta/text_end/message_end for one turn, stop or suppress further repeated assistant output before channel delivery, and add a shared per-turn outbound duplicate/message cap where block replies are queued. Keep tool-call loop detection scoped to tools, avoid Telegram/QQ-specific fixes, avoid adding a user-facing config option unless maintainers request it, and do not suppress legitimate non-consecutive repeated content such as code blocks, tables, logs, or quoted snippets. Add regression tests that simulate repeated text_delta chunks and repeated block/final delivery, and add a single-line CHANGELOG.md fix entry crediting the human reporters if attribution is clear.
