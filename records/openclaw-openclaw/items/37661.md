## Summary

Keep open. Current main still has tool-call loop detection, exact duplicate resend suppression, and final-payload dedupe, but it does not implement a generic guard for assistant text that keeps growing by appending the same phrase; the April 11 follow-up reports the same pure text repetition on a newer build and outside the original subagent-timeout trigger.

## What This Changes

The issue requests a safeguard that detects repeated assistant text output during streaming/final delivery and prevents repeated outbound messages in the same turn.

## Best Possible Solution

Add a conservative assistant-output repetition guard in the streaming/final reply path, plus a shared per-turn outbound duplicate/message cap before channel delivery, while leaving tool-call loop detection scoped to tool behavior.

## Reproduction Assessment

Yes for the missing guard behavior: a deterministic harness can emit repeated text_delta chunks such as the same sentence fragment many times and current main will treat them as growing assistant output. The exact intermittent upstream model loop is not live-reproduced here because the report’s custom endpoint and logs are not available.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the generic assistant streaming/final delivery path and block reply pipeline, not in Telegram/QQ-specific delivery code or the existing tool-call loop detector alone.
