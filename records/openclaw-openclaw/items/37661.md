## Summary

Keep open. Current main has exact duplicate suppression and tool-call loop detection, but it still treats repeated assistant text as valid growing output; the April follow-up also reports pure text repetition outside the original subagent-timeout trigger.

## What This Changes

Detect and stop LLM text-output repetition loops that spam repeated phrases or duplicate messages during assistant streaming and reply delivery.

## Best Possible Solution

Add a conservative generic assistant-output repetition guard in the streaming/final reply path, plus outbound duplicate/message-cap protection before channel delivery and focused regression tests.

## Reproduction Assessment

Yes for the source-level guard gap: a deterministic harness can emit repeated text_delta chunks and current main will append and emit them as growing assistant output. The intermittent custom endpoint trigger itself was not live-reproduced.

## Solution Assessment

Yes. The narrow maintainable fix belongs in generic assistant streaming/final delivery and the shared block reply pipeline, not in Telegram/QQ-specific delivery code or the existing tool-call loop detector alone.
