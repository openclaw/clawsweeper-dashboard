## Summary

Keep open. Current main still has tool-call loop detection and exact duplicate resend suppression, but the reported pure assistant-text repetition loop remains unguarded in the streaming/final reply path and outbound block delivery has no generic per-turn repetition/message cap.

## What This Changes

The issue requests protection against LLM assistant text repetition loops and duplicate outbound channel messages when streaming or final output repeats the same phrase many times.

## Best Possible Solution

Add a conservative output-level guard that detects repeated assistant text growth during streaming/finalization, halts or suppresses further delivery for that turn with one bounded notice, and adds shared outbound duplicate/message-cap protection before channel dispatch while keeping tool-call loop detection scoped to tools.

## Repair Work Prompt

Implement a conservative fix for the assistant text repetition loop tracked by this issue. Add output-level repetition protection in the assistant streaming/final reply path so repeated appended sentence fragments or N-grams are detected before `onPartialReply`/block reply delivery can spam a channel. Add a shared per-turn outbound duplicate/message cap in the block reply delivery path so one runaway model turn cannot send dozens of logical messages. Keep #34574 tool-call loop detection scoped to tools; do not replace it with text logic or change provider/channel defaults unnecessarily.
