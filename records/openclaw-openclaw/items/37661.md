## Summary

Keep open. Current main has tool-call loop detection plus exact/final payload dedupe, but I did not find a text-output repetition guard that would stop repeated phrases within one assistant message or repeated outbound text blocks; the April 11 follow-up also reports the same symptom on 2026.4.10 outside subagent timeouts.

## What This Changes

Detect and stop pathological repeated LLM text output and duplicate channel messages when model generation loops.

## Best Possible Solution

Add a generic output-level repetition guard in the auto-reply/block streaming and final payload path that catches pathological consecutive repeated text while preserving legitimate repeated content, with targeted regression tests and updated streaming/loop-detection docs.

## Reproduction Assessment

No high-confidence live reproduction path was established. Source inspection does show the missing guard: repeated text that survives normalization can flow through block/final delivery, while existing loop detection is scoped to tools.

## Solution Assessment

Unclear. The likely direction is an output-level repetition circuit breaker, but maintainers should decide the thresholds, scope, and whether it belongs with existing loopDetection or as an always-on delivery safety guard.
