## Summary

Keep open. Current main still lacks a privacy-safe OpenClaw-to-OpenClaw package/profile export-import surface; existing backup and migrate commands solve adjacent full-state backup and external-provider import cases, and the overlapping open PR/work shows the product and security boundary is still unsettled.

## What This Changes

Request a standardized portable OpenClaw agent/package format or commands for sharing stable persona, skills, rules, and knowledge while excluding secrets and runtime state.

## Best Possible Solution

Define a maintainer-approved portability model separate from full backup: versioned manifest/archive, strict secret and runtime-state exclusion, dependency metadata for skills/plugins, preview-first non-destructive import, and docs that distinguish profile sharing from disaster recovery.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature gap. Source and docs show no shipped profile/package export-import command, while `migrate` imports external providers and `backup` archives broad local state.

## Solution Assessment

Unclear for the exact Docker-like agent-image proposal. The user problem is valid, but the narrow maintainable solution needs maintainer product/security choice between profile portability, safer backup/restore, provider migration extensions, and plugin or ClawHub distribution.
