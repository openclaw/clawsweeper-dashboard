## Summary

Keep open. Current main still lacks a privacy-safe agent or instance packaging surface: `migrate` is provider-owned import from other agent systems, `backup` creates/verifies broad local-state archives, and the migration guide still tells users to copy the full state directory manually. Related work confirms the request is real but unsettled: PR #53520 was closed unmerged after maintainer product/security concerns, #45993 and #13616 remain open as overlapping trackers, and PR #73668 is an active but unmerged narrower profile-portability proposal.

## What This Changes

Request a standardized portable OpenClaw agent or instance package for sharing/migration that includes stable persona, skills, rules, and knowledge while excluding runtime state, logs, secrets, and machine-local paths.

## Best Possible Solution

Define a maintainer-approved profile or agent portability model separate from full backup: a versioned manifest/archive, shared sensitive-data redaction, explicit dependency metadata for skills/plugins, preview/non-destructive import semantics, and docs that distinguish profile sharing from full disaster recovery. Then align #45993, #13616, and PR #73668 under that chosen boundary.

## Reproduction Assessment

Not applicable as a runtime bug. The feature-gap check is high-confidence: source, CLI registration, docs, and related GitHub discussion show no shipped safe agent/instance package export/import on current main.

## Solution Assessment

Unclear for the exact Docker-like `agent-image` proposal. The user problem is valid, but the best implementation needs maintainer product/security choice between a narrow profile-portability command, safer backup conversion/manual restore, provider migration extensions, or plugin/ClawHub distribution.
