## Summary

Current main still has backup create/verify and external-system migration, but no shipped privacy-safe OpenClaw profile/package export-import surface; the closest implementation remains an open, conflicting PR that needs maintainer product and security review before this issue can close.

## What This Changes

Request a standardized, privacy-safe OpenClaw agent/profile packaging or export-import workflow that excludes runtime state and secrets while supporting sharing and migration.

## Best Possible Solution

Land a maintainer-approved profile portability model with a versioned manifest, strict secret/runtime-state exclusion, non-destructive preview-first import, conflict handling, plugin/skill dependency metadata, and docs that distinguish profile sharing from full backup/restore.

Not applicable as a runtime bug; this is a feature gap. Source, docs, live PR status, and related search show current main lacks a shipped privacy-safe profile/package export-import command.

## Solution Assessment

Unclear for the exact Docker-like agent-image proposal. The active candidate is https://github.com/openclaw/openclaw/pull/73668, but the final solution needs maintainer product and security decisions before this issue should close.
