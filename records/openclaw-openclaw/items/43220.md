## Summary

Keep open. Current main still lacks a privacy-safe OpenClaw-to-OpenClaw profile or package export/import surface; the shipped backup and migrate paths cover adjacent full-state backup and external-provider import cases, while the active related PR remains unmerged and security/product boundaries are still unsettled.

## What This Changes

The issue requests a standardized, privacy-safe agent packaging or migration model for sharing OpenClaw behavior without copying secrets, logs, sessions, or other runtime-local state.

## Best Possible Solution

Define and land a maintainer-approved portability model with a versioned manifest, strict secret/runtime-state exclusion, plugin and skill dependency metadata, preview-first non-destructive import, conflict handling, and docs that clearly distinguish profile sharing from disaster recovery backups.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature gap. Current source and docs show backup, external migration, and manual full-state copy paths, but no shipped profile/package export-import workflow.

## Solution Assessment

Unclear for the exact Docker-like agent-image proposal. The user problem is valid, but the narrow maintainable solution needs maintainer choice between profile portability, safer backup/restore, and plugin or ClawHub distribution; PR #73668 is the active candidate to review first.
