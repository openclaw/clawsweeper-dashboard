## Summary

This should close as a concrete duplicate of the open durable-fallback delivery contract work: current main still has a source-level path where a sanitized incomplete-turn error can be produced, but message-tool-only source suppression can keep it invisible, and https://github.com/openclaw/openclaw/issues/87561 already owns that remaining channel-contract fix across Discord and other channels.

## What This Changes

The issue requests that Discord source-channel turns ending with `non_deliverable_terminal_turn` and no `message.send` produce a visible fallback, retry, or channel-specific diagnostic instead of leaving only progress feedback.

## Best Possible Solution
