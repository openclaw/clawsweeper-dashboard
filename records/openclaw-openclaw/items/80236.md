## Best Possible Solution

No high-confidence current-main reproduction was run. The provided harness output plus current source make the path source-reproducible: the scenario expects a real `read`, the mock model plans `read`, and Codex currently filters `read` out of OpenClaw dynamic tools as app-server-owned.

Unclear. The right fix should be in the Codex app-server/native-tool or private-QA mock contract so `read` parity works without undoing the intentional native-tool boundary.
