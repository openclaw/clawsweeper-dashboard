## Summary

Keep open. Current main still applies one Feishu group mention policy to normal group messages and topic/thread replies, and PR #76020 is an open closing implementation candidate that should be reviewed before this issue closes.

## What This Changes

The issue requests a Feishu config split so normal group messages can require @mentions while topic/thread replies can bypass the mention requirement.

## Best Possible Solution

Review and land or replace PR #76020 with an additive Feishu-owned config override covering schema, policy, inbound gating, docs, tests, and changelog while preserving existing defaults when unset.

## Reproduction Assessment

Yes. Source inspection of current main shows Feishu topic messages expose `threadId`, but the group mention gate uses only `requireMention`; a group with `requireMention: true` will drop an unmentioned thread reply before dispatch.

## Solution Assessment

Yes, pending review. The maintainable direction is an additive Feishu-owned config override with schema, policy, bot, docs, tests, and changelog coverage, which is what PR #76020 is attempting.
