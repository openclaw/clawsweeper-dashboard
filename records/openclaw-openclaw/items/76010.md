## Summary

Keep open. Current main still applies one Feishu group mention policy to both normal group messages and topic/thread replies, `topicRequireMention` is absent from schema/docs, and open PR #76020 uses closing syntax for this feature, so the issue should wait for that PR to land or be replaced.

## What This Changes

Request a Feishu config option to require bot mentions in normal group messages while allowing topic/thread replies without repeated mentions.

## Best Possible Solution

Land or replace PR #76020 with Feishu-owned additive `topicRequireMention` handling across schema, policy, bot gate, docs, tests, and changelog while preserving current defaults when unset.

## Reproduction Assessment

Yes. Source inspection shows a Feishu group with `requireMention: true` and an unmentioned `topic_group` or `thread_id` reply reaches the same `resolveFeishuReplyPolicy` gate and returns before dispatch.

## Solution Assessment

Yes. An additive Feishu-owned config override is the narrow maintainable direction, and the linked open PR should be reviewed before this issue closes.
