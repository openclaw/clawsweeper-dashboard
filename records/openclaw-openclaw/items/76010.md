## Summary

Keep open. Current main still applies one Feishu group mention policy to both normal group messages and topic/thread replies, and open PR #76020 explicitly targets this issue, so the issue should wait for that PR to merge or be replaced.

## What This Changes

Request separate Feishu mention requirements for normal group messages and topic/thread replies, likely via an additive `topicRequireMention` config option.

## Best Possible Solution

Land or replace PR #76020 with a Feishu-owned additive `topicRequireMention` implementation covering schema, policy, bot dispatch, docs, tests, and changelog while preserving current defaults when unset.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure a Feishu group with `requireMention: true`, send a `thread_id` topic reply without a bot mention, and current main returns at the shared mention gate because no topic context reaches `resolveFeishuReplyPolicy`.

## Solution Assessment

Yes. A Feishu-owned additive `topicRequireMention` override is the narrow maintainable direction; because PR #76020 already targets this issue with closing syntax, that PR should be reviewed, landed, or replaced before this issue closes.
