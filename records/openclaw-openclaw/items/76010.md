## Summary

Keep open. Current main still applies one Feishu group mention gate to both normal group messages and topic/thread replies, and open PR #76020 already targets this issue with closing syntax, so the issue should wait for that PR to merge or be replaced.

## What This Changes

The issue requests a Feishu `topicRequireMention`-style configuration so normal group messages can require @mentions while topic/thread replies do not.

## Best Possible Solution

Land or replace PR #76020 with a Feishu-owned additive topic mention policy covering schema, policy resolution, bot dispatch, docs, tests, and changelog while preserving current behavior when unset.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure a Feishu group with `requireMention: true`, send an unmentioned event with `chat_type: "topic_group"` or `thread_id`, and current main reaches the same shared mention gate as a normal group message.

## Solution Assessment

Yes. A Feishu-owned additive config override that falls back to existing `requireMention` is the narrowest maintainable direction; the open closing PR is the current implementation candidate to review.
