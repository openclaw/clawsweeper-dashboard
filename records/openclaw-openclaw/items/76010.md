## Summary

Keep open. Current main still has only one Feishu group mention policy, `topicRequireMention` is absent from schema/docs, and open PR #76020 uses closing syntax for this feature, so the issue should wait for that PR to land or be replaced.

## What This Changes

Request separate Feishu mention-gating configuration so normal group messages can require @mention while topic/thread replies can continue without one.

## Best Possible Solution

Land or replace PR #76020 with an additive Feishu-owned `topicRequireMention` implementation covering schema, policy resolution, inbound gating, docs, tests, and changelog while preserving existing defaults when unset.

## Reproduction Assessment

Yes. Source inspection shows a Feishu topic event can carry `chatType: "topic_group"` and `threadId`, but a group with `requireMention: true` still reaches the same mention gate and returns when `ctx.mentionedBot` is false.

## Solution Assessment

Yes. An additive Feishu plugin config override is the narrow maintainable direction; because open PR #76020 already references this issue with closing syntax, the issue should remain open until that PR lands or is replaced.
