## Summary

Keep open. Current main still has a single Feishu group mention gate for normal group messages and topic/thread replies, and the open closing PR #76020 is the right implementation candidate to review before this issue closes.

## What This Changes

Add a Feishu config option so normal group messages can require an @mention while topic/thread replies can continue without one.

## Best Possible Solution

Feishu should expose an optional topic/thread mention override inside the plugin, with schema, docs, tests, and changelog coverage, while preserving existing group mention behavior by default.

## Reproduction Assessment

Yes. A source-level reproduction is clear: configure a Feishu group with `requireMention: true`, send a `topic_group` or `thread_id` reply event without a bot mention, and current main drops it at the shared group mention gate.

## Solution Assessment

Yes. A Feishu-owned additive config and policy change is the narrow maintainable solution; core channel APIs do not need to change for this channel-specific option.
