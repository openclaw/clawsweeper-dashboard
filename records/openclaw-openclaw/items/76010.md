## Summary

Keep open. Current main still applies one Feishu group mention policy to both normal group messages and topic/thread replies, so the requested split remains a valid Feishu channel enhancement.

## What This Changes

Allow Feishu group configuration to require @mentions for normal group messages while allowing topic/thread replies to continue without repeated @mentions.

## Best Possible Solution

Implement this inside the Feishu plugin by adding an optional topic/thread mention override that is used only for Feishu topic/thread replies and otherwise preserves existing group mention behavior.

## Reproduction Assessment

Yes. A source-level reproduction is clear: configure a Feishu group with `requireMention: true`, send a synthetic `topic_group` or `thread_id` reply event without a bot mention, and current main returns before dispatch because the shared group mention gate is true.

## Solution Assessment

Yes. A Feishu-owned additive config and policy change is the narrow maintainable path; core channel behavior does not need a new generic seam for this channel-specific option.
