## Summary

Keep open. Current main still applies one Feishu group mention policy to both normal group messages and topic/thread replies, the schema and docs do not expose `topicRequireMention`, and open PR #76020 is already linked with closing syntax for this feature.

## What This Changes

Request separate Feishu mention-gating config so normal group messages can require @mentions while topic/thread replies can omit them.

## Best Possible Solution

Land or replace PR #76020 with an additive Feishu-owned `topicRequireMention` config path across schema, policy resolution, bot gate, docs, tests, and changelog while preserving existing defaults when unset.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a Feishu group with `requireMention: true` and an unmentioned event carrying `thread_id` reaches the same `resolveFeishuReplyPolicy` gate and is dropped before dispatch.

## Solution Assessment

Yes. An additive Feishu-owned config override is the narrow maintainable direction, but this issue should stay open until the linked implementation PR lands or is replaced.
