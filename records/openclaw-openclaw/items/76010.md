## Summary

Keep open: current main and the latest release still use one Feishu group mention gate for both normal group messages and topic-thread replies, while the linked closing implementation PR remains open and conflicting.

## What This Changes

Request support for separate Feishu mention requirements so normal group messages can require an @mention while topic/thread replies can continue without repeated mentions.

## Best Possible Solution

Review, refresh, and either land or replace the linked Feishu-owned implementation with an additive topic-thread mention policy that keeps existing requireMention behavior as the fallback and includes schema, generated metadata, docs, tests, and real Feishu proof.

## Reproduction Assessment

Yes, at source level. Current main preserves Feishu topic/thread identifiers but resolves one requireMention value and applies it before dispatch, so an unmentioned topic reply in a mention-required group follows the normal drop path.

## Solution Assessment

Not yet. The explicit Feishu-owned topicRequireMention direction is plausible and narrower than older stateful follow-up proposals, but maintainers still need to accept the public config semantics and land a refreshed, proven implementation.
