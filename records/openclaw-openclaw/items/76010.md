## Summary

Keep open. Current main still uses one Feishu group mention gate for normal group messages and topic/thread replies, while open PR #76020 is the active closing implementation candidate.

## What This Changes

The issue requests a Feishu `topicRequireMention` override so normal group messages can require @mention while topic/thread replies can continue without one.

## Best Possible Solution

Feishu should add an optional, backwards-compatible topic/thread mention override in the plugin, with schema, docs, tests, and changelog coverage; #76020 is the current implementation candidate to review.

## Reproduction Assessment

Yes. Configure a Feishu group with `requireMention: true`, send a `topic_group` or `thread_id` reply event without a bot mention, and current main reaches the shared group mention gate and drops it before dispatch.

## Solution Assessment

Yes. An additive Feishu-owned config option with fallback to the existing `requireMention` cascade is the narrowest maintainable solution; core channel APIs do not need to change.
