## Summary

Keep open. Current main still applies one Feishu group mention policy to both normal group messages and topic/thread replies, the requested config key is absent from schema/docs, and open PR #76020 is already the closing implementation candidate.

## What This Changes

Request separate Feishu mention-gating config for normal group messages and topic/thread replies, such as keeping `requireMention` for main chat while allowing unmentioned topic replies with `topicRequireMention`.

## Best Possible Solution

Review and land or replace PR #76020 with a Feishu-owned additive `topicRequireMention` implementation across schema, policy, bot gate, docs, tests, and changelog while preserving current defaults when unset.

## Reproduction Assessment

Yes. Source inspection shows a Feishu group with `requireMention: true` and an unmentioned `topic_group` or `thread_id` reply reaches the same `resolveFeishuReplyPolicy` gate and is dropped before dispatch.

## Solution Assessment

Yes, directionally. An additive Feishu-owned config override is the narrow maintainable solution, and the open closing PR should be reviewed rather than starting a parallel fix lane.
