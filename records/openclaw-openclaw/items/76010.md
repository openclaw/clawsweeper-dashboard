## Summary

Keep open. Current main still has one Feishu group mention gate for both normal group messages and topic/thread replies, and open closing PR #76020 is the implementation candidate to review before this issue closes.

## What This Changes

The issue requests a Feishu configuration option to require @mentions in normal group messages while allowing topic/thread replies without repeated mentions.

## Best Possible Solution

Land PR #76020 or an equivalent Feishu-owned additive implementation with schema, docs, tests, and changelog coverage, preserving existing mention behavior when the new option is unset.

## Reproduction Assessment

Yes. Configure a Feishu group with `requireMention: true`, send a `topic_group` or `thread_id` reply without a bot mention, and current main reaches the same shared `requireMention` drop path used for normal group messages.

## Solution Assessment

Yes. A Feishu-local additive topic/thread override that falls back to the existing mention cascade is the narrow maintainable solution; no core channel API change is indicated by the current code.
