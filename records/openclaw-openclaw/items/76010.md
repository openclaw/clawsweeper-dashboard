## Summary

Keep open. Current main still applies one Feishu group mention policy to both normal group messages and topic/thread replies, `topicRequireMention` is absent from schema/docs, and open PR #76020 is already the closing implementation candidate.

## What This Changes

Request adds a Feishu topic/thread mention policy so normal group messages can require @mention while topic replies can opt out.

## Best Possible Solution

Review and land or replace PR #76020 with additive Feishu-owned `topicRequireMention` handling across schema, policy, inbound gate, docs, tests, and changelog while preserving existing defaults when unset.

## Reproduction Assessment

Yes for the current limitation, by source inspection rather than live Feishu: current main parses `threadId`, but the mention resolver only receives account/group/policy inputs, so an unmentioned topic reply with `requireMention: true` reaches the same drop gate as a normal group message.

## Solution Assessment

Yes. An additive Feishu-owned config override is the narrow maintainable direction, and because PR #76020 uses closing syntax, this issue should remain open until that PR is reviewed and merged or replaced.
