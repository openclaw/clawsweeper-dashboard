## Summary

Keep open. Current main still applies one Feishu group `requireMention` policy to normal group messages and topic/thread replies, and open PR #76020 is already the linked implementation candidate for the requested split.

## What This Changes

Allow Feishu operators to configure separate mention requirements for normal group messages and topic/thread replies.

## Best Possible Solution

Review and land or replace PR #76020 with a Feishu-owned additive `topicRequireMention` implementation that preserves existing defaults when unset.

## Reproduction Assessment

Yes. Source inspection shows an unmentioned Feishu topic/thread reply with `requireMention: true` reaches the same group mention gate as a normal group message and is dropped before dispatch.

## Solution Assessment

Yes. An additive Feishu config override resolved inside the Feishu-owned policy path is the narrow maintainable direction; current main does not implement it yet, so the linked PR should be reviewed before this issue closes.
