## Summary

Keep open. Current main and the latest release still use one shared Feishu `requireMention` gate for normal group messages and topic/thread replies, and the open closing PR is the active implementation path, so closing the issue before that PR or a successor lands would be premature.

## What This Changes

The issue requests a Feishu topic/thread-specific mention requirement so main group messages can require @mention while topic replies can continue without repeated mentions.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/76020 or a successor that adds Feishu-owned topic mention policy across schema, runtime, docs, generated metadata if needed, and focused tests while preserving existing defaults when unset.

## Reproduction Assessment

Yes, at source level. Current main preserves Feishu thread context, but the sender activation gate still receives only the shared `requireMention` result, so an unmentioned topic reply with mention gating enabled follows the normal drop path.

## Solution Assessment

Yes for the direction, but it is not complete yet. The maintainer-selected declarative `topicRequireMention` approach is the narrowest maintainable shape, and the open closing PR should be reviewed, repaired, or replaced before this issue closes.
