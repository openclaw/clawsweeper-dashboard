Keep open. Current main still lacks a Feishu topic/thread-specific mention policy, and the open closing PR is the active implementation path, so closing this issue before that work lands would be premature.

## What This Changes

The issue requests a Feishu config split so normal group messages can require @mentions while topic/thread replies can skip them.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/76020 or a successor that adds Feishu-owned topic mention policy across runtime, schema/generated metadata, docs, and tests while preserving existing defaults when unset.

## Reproduction Assessment

Yes, at source level. Current main preserves Feishu thread context but still sends group messages through one shared `requireMention` gate, so an unmentioned topic reply with mention gating enabled follows the normal drop path.

## Solution Assessment

Yes for the direction, but not complete yet. The additive Feishu-owned `topicRequireMention` policy is the narrow maintainable shape, and the open closing PR should be reviewed or replaced before this issue closes.
