## Summary

Keep open. Current main still applies one Feishu `requireMention` policy to normal group messages and topic/thread replies, and https://github.com/openclaw/openclaw/pull/76020 is still the open closing implementation path.

## What This Changes

Request separate Feishu mention gating for normal group messages and topic/thread replies, likely through a `topicRequireMention` config option.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/76020 or a successor that adds Feishu-owned topic mention policy across schema, generated config metadata, docs, tests, and runtime while preserving existing defaults when unset.

## Reproduction Assessment

Yes, at source level. Current main preserves Feishu thread context but sends all group messages through the same `requireMention` gate before dispatch, so an unmentioned topic reply with mention gating enabled is still dropped.

## Solution Assessment

Yes for the direction, but not complete yet. An additive Feishu-owned topic mention policy is the narrow maintainable shape, and the open closing PR should be reviewed or replaced before this issue closes.
