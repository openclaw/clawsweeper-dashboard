## Summary

Keep open. Current main still applies a single Feishu group mention policy to normal messages and topic/thread replies, `topicRequireMention` is absent from schema/docs, and open PR #76020 is the active closing implementation candidate.

## What This Changes

The issue requests a separate Feishu topic/thread mention requirement so normal group messages can require @mention while topic replies can continue without repeated @mentions.

## Best Possible Solution

Land or replace PR #76020 with an additive Feishu-owned `topicRequireMention` implementation covering schema, policy, bot gating, docs, tests, and changelog while preserving existing defaults when unset.

## Reproduction Assessment

Yes. Source inspection shows a Feishu group with `requireMention: true` and an unmentioned `topic_group` or `thread_id` reply reaches the same `resolveFeishuReplyPolicy` gate and returns before dispatch.

## Solution Assessment

Yes. An additive Feishu-owned config override is the narrow maintainable direction, but the issue should stay open until #76020 lands or is replaced.
