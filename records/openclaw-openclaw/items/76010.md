## Summary

Keep open. Current main and the latest release still have a single Feishu `requireMention` gate for group messages, and the remaining work is already owned by the open `topicRequireMention` implementation candidate rather than something this cleanup pass should close.

## What This Changes

The issue requests separate Feishu mention requirements so normal group messages can require an @mention while topic-thread replies can continue without repeated @mentions.

## Best Possible Solution

Land or replace the open `topicRequireMention` implementation after maintainer product review, rebase, changelog cleanup, schema/docs/tests alignment, and live Feishu proof.

## Reproduction Assessment

Yes, at source level. Configure a Feishu group with `requireMention: true`, send an unmentioned topic-thread reply, and current main still passes the same `requireMention` decision into sender activation before dispatch.

## Solution Assessment

Yes for the current direction, with caveats. The explicit `topicRequireMention` design has maintainer-routing support over the competing stateful `threadFollowUp` design, but it still needs product review and merge-ready proof in the linked PR.
