## Summary

Keep open. Current main still applies one Feishu group `requireMention` gate to both normal group messages and topic/thread replies, and open PR #76020 is already the implementation candidate linked from the issue discussion.

## What This Changes

The issue requests a Feishu config split so normal group messages can require @mentions while topic/thread replies can continue without repeated @mentions.

## Best Possible Solution

Feishu should add an optional topic/thread mention override inside the plugin, default it to the existing `requireMention` cascade when unset, and cover schema, policy, bot dispatch, docs, tests, and changelog together.

## Reproduction Assessment

Yes. Configure a Feishu group with `requireMention: true`, send a `topic_group` or `thread_id` reply event without mentioning the bot, and current main drops it at the shared group mention gate before dispatch.

## Solution Assessment

Yes. A Feishu-owned additive config and policy change is the narrow maintainable solution; the current source shows the needed topic/thread context is already available without changing core channel APIs.
