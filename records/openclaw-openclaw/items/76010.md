## Summary

Keep open. Current main still has a single Feishu `requireMention` policy for normal group messages and topic/thread replies, and open PR #76020 is the active closing implementation candidate, so this issue should wait for that PR to land or be replaced.

## What This Changes

Request an additive Feishu configuration that can require @mentions in normal group messages while allowing unmentioned topic/thread replies.

## Best Possible Solution

Feishu should own an additive topic/thread mention override in schema, policy resolution, bot gating, docs, tests, and changelog while preserving existing defaults when unset.

## Reproduction Assessment

Yes. Source inspection shows a Feishu group with `requireMention: true` and an unmentioned `topic_group` or `thread_id` reply reaches the same shared mention gate and is dropped before dispatch.

## Solution Assessment

Yes. An additive Feishu-owned `topicRequireMention` override is the narrow maintainable direction, and the linked open PR should be reviewed before this issue closes.
