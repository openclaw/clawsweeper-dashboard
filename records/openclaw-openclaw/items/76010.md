## Summary

Keep open. Current main and v2026.6.10 still use one Feishu group `requireMention` decision for both normal group messages and topic-thread replies, and the open linked PR is the active implementation candidate rather than a reason to close this issue before merge.

## What This Changes

The issue requests a Feishu topic/thread-specific mention requirement so main group messages can require `@bot` while topic-thread replies can continue without repeated mentions.

## Best Possible Solution

Ship a Feishu-owned topic mention policy only after maintainers settle the public config shape and land a refreshed implementation with schema metadata, docs/tests, and redacted live Feishu proof.

## Reproduction Assessment

Yes, at source level. Configure a Feishu group with `requireMention: true`, send an unmentioned topic-thread reply, and current main still passes the same shared `requireMention` decision into sender activation before dispatch.

## Solution Assessment

Unclear. The explicit `topicRequireMention` shape is a narrow fit for the request, but because it adds public Feishu config and changes message admission, maintainers should confirm the final direction through the linked PR before this issue closes.
