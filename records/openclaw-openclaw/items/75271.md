## Summary

Keep open. Current main still does not persistently repair a session JSONL that ends on a normal assistant message, and the linked fix PR is open and unmerged, so this issue should stay open for that PR review path.

## What This Changes

The issue asks embedded-agent session repair to prevent repaired session JSONL files from ending on assistant turns that Anthropic rejects as assistant prefill.

## Best Possible Solution

Land a focused persistent repair that drops trailing non-tool-call assistant message entries from `repairSessionFileIfNeeded`, preserves mid-transcript assistant turns, and keeps regression coverage plus transcript-hygiene docs aligned.

## Reproduction Assessment

Yes. A session JSONL with a valid header/user turn followed by a normal `message.role: "assistant"` entry will currently pass through `repairSessionFileIfNeeded` without any repair counter changing, so the file remains assistant-terminated.

## Solution Assessment

Yes. Trimming trailing assistant entries during file repair is narrower and more maintainable than appending a synthetic user turn because it removes stale provider-rejected replay state without inventing transcript content.
