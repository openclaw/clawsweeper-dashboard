## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve callback lifecycle completion:** `src/infra/net/undici-global-dispatcher.ts:190-201`
  - body: For callback-style `close`/`destroy`, the wrapper passes the callback only to the EnvHttpProxyAgent and starts the bypass Agent without one. Undici's callback overload is supposed to report completion only after owned dispatchers settle, but here the original callback can fire before the bypass Agent finishes; returning a Promise does not help callback callers. Coordinate both lifecycle results and invoke the single callback after both have completed.
  - confidence: 0.9
- **[P2] Keep bootstrap bypass timeouts symmetric:** `src/infra/net/undici-global-dispatcher.ts:326-327`
  - body: The bootstrap path creates the EnvHttpProxyAgent without an explicit timeout, but NO_PROXY-matched requests are intercepted and sent to a new direct Agent with `DEFAULT_UNDICI_STREAM_TIMEOUT_MS`. Existing NO_PROXY users therefore move from Undici's normal no-proxy Agent timeout behavior to a 30-minute timeout on upgrade; make the bypass Agent use the same timeout inputs as the EnvHttpProxyAgent path, or add an explicit tested compatibility decision. The same issue exists in the force-reset path.
  - confidence: 0.84
