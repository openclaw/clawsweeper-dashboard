Full review comments:

- **[P2] Wait for bypass lifecycle before signaling close:** `src/infra/net/undici-global-dispatcher.ts:193-196`
  - body: For callback or promise close/destroy, the wrapper starts the bypass Agent lifecycle with callbacks stripped and then returns only the EnvHttpProxyAgent result. Undici treats these APIs as shutdown-completion signals, so callers can observe the global dispatcher as closed while direct bypass requests are still draining.
  - confidence: 0.9
- **[P2] Keep bootstrap bypass timeouts consistent:** `src/infra/net/undici-global-dispatcher.ts:318-320`
  - body: Bootstrap and force-reset give NO_PROXY-matched direct requests the 30-minute OpenClaw timeout while the paired EnvHttpProxyAgent is constructed without that explicit timeout. Existing proxy deployments can therefore get different timeout behavior only for bypassed requests after upgrade.
  - confidence: 0.84
- **[P3] Make the bypass lifecycle test observe the bypass agent:** `src/infra/net/undici-global-dispatcher.test.ts:946-956`
  - body: This test replaces the wrapper's close/destroy methods with spies, so it no longer exercises the wrapper path that should close both dispatchers; the unused bypassClosed flag is another sign it never observes the bypass Agent.
  - confidence: 0.86
