## Summary

Keep open: the PR has sufficient live proxy proof and is aimed at the right core networking surface, but the current head still returns lifecycle completion before the direct bypass Agent has closed and changes timeout behavior for bootstrap NO_PROXY-matched requests.

## What This Changes

The PR wraps global undici EnvHttpProxyAgent dispatchers with OpenClaw's enhanced NO_PROXY matcher so matched requests use a direct dispatcher while nonmatches keep using the env proxy dispatcher.

## Best Possible Solution

Land one generic global-dispatcher fix that preserves EnvHttpProxyAgent no-proxy timeout and lifecycle contracts, proves both proxy and bypass routes, and then use that merged PR as the canonical resolution for the linked regression.

## Reproduction Assessment

Yes for the source-level path: current main installs a global EnvHttpProxyAgent while the enhanced OpenClaw NO_PROXY matcher is only used by helper-level fetch paths. I did not rerun the external qqbot/COS upload flow in this read-only review.

## Solution Assessment

No: the wrapper is in the right core network surface, but this implementation is not the best merge path until timeout policy and lifecycle completion match the dispatcher contract. A safer version keeps the generic wrapper and makes both owned dispatchers share lifecycle and timeout semantics.

- **[P2] Wait for bypass cleanup before lifecycle completion:** `src/infra/net/undici-global-dispatcher.ts:191-197`
  - body: The wrapper starts `close()`/`destroy()` on `bypassAgent` but immediately returns the EnvHttpProxyAgent lifecycle result. Undici lifecycle promises and callbacks are expected to complete after the owned dispatcher has closed or destroyed, so global resets and shutdowns can finish while bypass sockets are still closing.   - confidence: 0.91

That changes existing NO_PROXY-matched requests from undici's default no-proxy Agent behavior to a 30-minute timeout after upgrade; use the same timeout inputs as the env-proxy path or add an explicit, tested compatibility decision.
