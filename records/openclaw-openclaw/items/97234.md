## Summary

Keep open: this is a useful candidate fix for the linked proxy regression, but the latest head still returns EnvHttpProxyAgent lifecycle completion before the added bypass Agent is known closed and still changes NO_PROXY-bypassed bootstrap/reset timeout behavior. Current main does not already contain this wrapper, and several sibling PRs still need one canonical resolution.

## What This Changes

The PR wraps the global undici EnvHttpProxyAgent so OpenClaw's enhanced NO_PROXY matcher can route matched requests through a direct bypass dispatcher while nonmatches continue through the environment proxy dispatcher.

## Best Possible Solution

Land one generic global-dispatcher fix that preserves EnvHttpProxyAgent timeout and lifecycle contracts, proves both proxy and bypass routes, and then use the merged PR as the canonical resolution for the linked regression.

## Reproduction Assessment

Yes for the source-level path: current main installs a global EnvHttpProxyAgent while the enhanced OpenClaw NO_PROXY matcher is only used by helper-level fetch paths. I did not rerun the external qqbot/COS upload flow in this read-only review.

## Solution Assessment

No: the wrapper is in the right core network surface, but this implementation is not the best merge path until timeout policy and lifecycle completion match the dispatcher contract. A safer version keeps the generic wrapper and makes both owned dispatchers share lifecycle and timeout semantics.
