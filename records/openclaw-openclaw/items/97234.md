Full review comments:

- **[P2] Wait for bypass shutdown before returning:** `src/infra/net/undici-global-dispatcher.ts:191-196`
  - body: For close/destroy, the wrapper starts the bypass Agent lifecycle without a callback or returned promise join, then returns only the EnvHttpProxyAgent result. Undici treats these APIs as shutdown-completion signals, so callers can observe the global dispatcher as closed while the direct bypass Agent is still draining.
  - confidence: 0.9
- **[P2] Keep bootstrap bypass timeouts consistent:** `src/infra/net/undici-global-dispatcher.ts:318-320`
  - body: Bootstrap gives NO_PROXY-matched direct requests the 30-minute OpenClaw timeout while the paired EnvHttpProxyAgent is constructed without that explicit timeout. Existing proxy deployments can therefore get different timeout behavior only for bypassed requests after upgrade.
  - confidence: 0.84
- **[P3] Make the lifecycle test observe the bypass agent:** `src/infra/net/undici-global-dispatcher.test.ts:948-955`
  - body: This test replaces the wrapper's close/destroy methods with spies, so it no longer exercises the wrapper path that should close both dispatchers. It can pass even when the hidden bypass Agent is never closed or destroyed.
  - confidence: 0.86
