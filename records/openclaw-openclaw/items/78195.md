The PR removes `hostReadCapability` from the shared outbound media-load options helper, sets it only for auto-reply attachment normalization, and updates explicit outbound media tests.

## Best Possible Solution

## Solution Assessment

No, not as submitted. Moving `hostReadCapability` to the auto-reply path is plausible, but the patch leaves stale tests behind and needs maintainer agreement on the broader all-MIME explicit-send policy versus the narrower #78073 approach.
