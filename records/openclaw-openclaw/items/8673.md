## Summary

Keep open: current main and latest shipped release still refresh OAuth credentials with a single provider/plugin call for ordinary transient failures, while the only second-attempt behavior is scoped to refresh-token-reuse/adoption recovery. The related retry PRs are closed and unmerged, and prior review discussion points to a remaining owner-reviewed retry-classification decision rather than a completed fix.

## What This Changes

The issue requests bounded retry logic for transient OAuth token refresh failures so agents do not fail on a single temporary provider/API error.

## Best Possible Solution

Add an owner-reviewed retryAsync integration around OAuth refresh with a transient-only classifier, preserving the existing lock, timeout, adoption, fallback, plugin refresh, and refresh-token-reuse behavior with focused regression tests.

## Reproduction Assessment

Yes, at source level: current main calls the provider/plugin OAuth refresh path once, and HTTP 500/network timeout are outside the existing refresh-token-reuse recovery branch. I did not run a live transient provider outage in this read-only review.

## Solution Assessment

No for the issue body's exact local withRetry helper: retrying every thrown refresh error would retry permanent auth failures and duplicate src/infra/retry.ts. The safer solution is an owner-reviewed retryAsync integration with preserved error causes, lock safety, and a transient-only classifier.

## Risks / Open Questions

- A blanket retry around every refresh error could repeat permanent OAuth failures such as invalid_grant, revoked credentials, or sign-in-required states and delay correct re-auth guidance.
- Retry placement is auth-sensitive because the path already combines global locks, hard timeouts, main-store/bootstrap adoption, fallback credentials, provider-plugin hooks, and refresh-token-reuse recovery.
- Provider plugins may implement their own refreshOAuth behavior, so shared retry policy needs to avoid surprising double retries or extended lock/timeout windows.
