## Summary

Keep open: this is a valid core setup UX feature request, not a cleanup close. Current main still has English-only CLI wizard strings, while the existing UI/docs i18n systems are separate and the right CLI locale source, generation policy, and prompt coverage need maintainer direction.

## What This Changes

Request locale-aware translations for `openclaw setup` prompts, initially via a lightweight translation map and `t()` helper.

## Best Possible Solution

Add a maintainable CLI i18n layer for core setup prompts that aligns with the existing generated UI/docs localization model, keeps English fallback, defines locale resolution/override behavior, and adds coverage for locale mapping plus wizard prompt output.

## Reproduction Assessment

Not applicable; this is a feature request rather than a bug. Source inspection gives high-confidence evidence that current main still renders the interactive setup wizard through literal English strings in `src/wizard/setup.ts`, `src/wizard/setup.finalize.ts`, and related prompt helpers.

## Solution Assessment

Unclear; the proposed single `t()` map is a plausible first slice, but maintainers should first choose the CLI translation source, generation workflow, fallback behavior, and coverage boundary for plugin-provided and security-sensitive prompts.
