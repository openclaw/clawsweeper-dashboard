## Summary

Keep open; latest head is not merge-ready because it exposes Codex goal RPCs while OpenClaw still supports Codex app-server 0.125.0, and dependency/protocol inspection shows those RPCs are only available in newer app-server builds.

## What This Changes

The PR adds Codex `/goal` and `/codex goal` controls, docs/changelog entries, and terminal-goal bridge events for bound Codex app-server conversations.

## Best Possible Solution

Keep the feature, but align the goal RPC contract with supported app-server versions or return a friendly unsupported response before merge.

## Reproduction Assessment

Yes for the review finding: use a supported custom Codex app-server 0.125.0 and run `/goal status` on a bound conversation; package inspection shows that build lacks `thread/goal/get`, so the call reaches an unknown-method path. The PR itself is a feature, so current-main bug reproduction is otherwise not applicable.

## Solution Assessment

No; the plugin boundary, docs, changelog, and untrusted bridge handling are reasonable, but the patch is not the best maintainable solution until the new RPCs are version-gated or gracefully handled as unsupported.
