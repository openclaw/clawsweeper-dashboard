## Summary

Keep open: the feature is useful and owner-local, but the PR still calls new Codex goal RPCs while current OpenClaw supports older Codex app-server builds that do not expose those methods, so `/goal` can fail as an unknown RPC instead of a friendly unsupported path.

## What This Changes

The PR adds `/goal` and `/codex goal` Codex thread-goal controls, terminal-goal system-event/heartbeat bridging, docs/changelog entries, and command/binding tests.

## Best Possible Solution

Keep the Codex goal feature, but align it with the app-server capability contract by either raising/syncing the supported minimum and protocol or returning a clear unsupported response for older supported binaries.

## Reproduction Assessment

Not applicable; this is a feature PR rather than a bug report. Source and dependency inspection give a high-confidence compatibility path: the PR calls `thread/goal/*`, but the current supported `0.125.0` app-server package lacks those RPCs.

## Solution Assessment

No; the plugin boundary, docs/changelog, and untrusted system-event handling are appropriate, but the PR is not the best maintainable solution until unsupported goal RPCs are gated or the minimum/protocol contract is updated.
