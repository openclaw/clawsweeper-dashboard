## Summary

Keep open: the PR adds a useful Codex goal-control feature, but it still exposes new goal RPCs without guarding older app-server builds that OpenClaw currently accepts.

## What This Changes

The PR adds `/goal` and `/codex goal` commands, documents them, and bridges terminal Codex thread-goal state into an untrusted OpenClaw system event plus heartbeat wake.

## Best Possible Solution

Keep the feature, but align the goal RPC contract with supported app-server versions or return the existing friendly unsupported response before merge.

## Reproduction Assessment

Not applicable; this is a feature PR rather than a bug report. Source and dependency inspection give a high-confidence compatibility failure path for older supported Codex app-server 0.125.0 builds.

## Solution Assessment

No; the plugin boundary, docs, changelog, and untrusted system-event handling are appropriate, but the PR is not merge-ready until unsupported goal RPCs are gated or the minimum/protocol contract is updated.
