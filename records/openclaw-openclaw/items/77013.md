## Summary

This PR should stay open because current main does not implement the Codex `/goal` command or terminal-goal bridge, and the PR still has blocking correctness/security review findings plus insufficient real bound-flow proof.

## What This Changes

The PR adds Codex `/goal` and `/codex goal` controls, bridges terminal Codex goal states into plugin system events, documents the new command surface, and adds focused Codex command/binding tests.

## Best Possible Solution

Keep the plugin-local goal command direction, but make terminal-goal polling non-blocking or tightly timeout-bound, escape all displayed goal fields, and require redacted real bound-flow proof before merge.

## Reproduction Assessment

Yes for the PR review findings: the blocking poll and unescaped output are visible directly in PR-head source against current-main timeout and formatting contracts. The PR itself is a feature addition rather than a current-main bug report.

## Solution Assessment

No; the command surface is plausible, but best-effort goal polling should not delay reply delivery and app-server/user-supplied goal text should follow the existing Codex display sanitizer path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Do not block replies on goal polling:** `extensions/codex/src/conversation-binding.ts:449-459`
  - body: After the Codex turn has already produced `replyText`, this awaits the best-effort goal bridge before returning the reply. That helper calls `thread/goal/get` with the normal app-server request timeout, so a slow or stuck goal RPC can delay every bound conversation reply even though the bridge is only observational.
  - confidence: 0.9
- **[P2] Escape goal response text before returning it:** `extensions/codex/src/command-handlers.ts:1691-1694`
  - body: The new goal formatter reflects `threadId`, `objective`, and `status` from the app-server directly into chat output. Existing Codex command replies escape app-server text with `formatCodexDisplayText`; without that, a goal objective such as an @mention or markdown link can be echoed as trusted-looking bot output.
  - confidence: 0.86
