## Summary

Keep this PR open: the feature is useful and current main does not already provide `/goal`, but the patch still has a merge-blocking compatibility defect because it adds `thread/goal/*` calls while OpenClaw still supports Codex app-server `0.125.0`, whose package does not expose those methods.

## What This Changes

The PR adds Codex `/goal` and `/codex goal` controls, registers the command/docs/changelog surface, and bridges terminal native Codex goal state into untrusted OpenClaw system events after bound turns.

## Best Possible Solution

Keep the Codex plugin feature, but land it only after goal RPCs either return a friendly unsupported response on older supported app-server builds or the minimum/protocol contract is deliberately updated.

## Reproduction Assessment

Not applicable; this is a feature PR rather than a bug report. The compatibility issue is source- and dependency-backed: current main supports app-server `0.125.0`, and that package lacks the new `thread/goal/*` RPCs.

## Solution Assessment

No; the plugin boundary, docs, changelog, and untrusted system-event handling look appropriate, but the PR is not the best maintainable solution until unsupported goal RPCs are gated or the app-server minimum/protocol contract is updated.
