## Summary

Keep this open. Current main has partial cron diagnostics, including run-state counters, failure alerts, durable run-history JSONL, and generic `/health`/`/ready` probes, but it still lacks the central requested cron observability: structured per-attempt logs for all scheduled/manual outcomes, cron-aware health/readiness reporting, and `openclaw cron list` warnings for overdue or repeatedly failing/skipped jobs.

## What This Changes

The issue requests cron execution observability: per-attempt logs, skipped/failure warnings, cron health reporting, and unhealthy-job annotations in `openclaw cron list`.

## Best Possible Solution

Land a coordinated cron-observability change that logs bounded structured outcomes for scheduled, startup catch-up, queued, and manual runs; warns on skipped and non-throwing failures; exposes cron health through the chosen probe surface; and annotates `openclaw cron list` for overdue jobs and repeated failures/skips while preserving existing counters, failure alerts, and run history.

## Reproduction Assessment

Not applicable as a strict bug reproduction: this is an observability feature request layered on top of the already-fixed scheduler root cause. Source inspection gives high confidence that the requested log, health, and list-warning surfaces are still incomplete on current main.

## Solution Assessment

Unclear without maintainer product direction. The desired endpoint/readiness shape should be decided together with the overlapping diagnostics PR, while the logging/list-warning parts can be implemented narrowly once that direction is set.
