## Summary

Keep open: current main still only partially covers cron observability. It has cron state, run history, failure counters, and generic probes, but the requested per-attempt success/skipped runtime logs, cron-aware health/readiness behavior, and `openclaw cron list` warning annotations remain missing; the prior related implementation PRs are closed unmerged.

## What This Changes

Request structured logging, health/probe reporting, and CLI warnings when cron executions fail, skip, or become overdue.

## Best Possible Solution

Define the cron observability contract, then implement bounded structured outcome logs, cron-aware probe/readiness reporting, and operator-facing cron list warnings on top of the existing counters, alerts, diagnostics, and run history.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not as a strict runtime bug repro; this is an observability enhancement. Source inspection of current main gives high confidence that success logging, CLI warning annotations, and cron-aware probe/readiness behavior are still missing.

## Solution Assessment

Is this the best way to solve the issue? Unclear. Structured logging and CLI warning annotations are narrow, but the health/readiness endpoint behavior needs maintainer product/API direction before one implementation can be called the best fix.
