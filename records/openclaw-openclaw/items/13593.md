## Summary

Keep open: current main has useful cron state, failure alerts, and durable run history, but the requested per-attempt success/skipped logging, cron-aware health/readiness, and `openclaw cron list` warning annotations are still incomplete. The overlapping diagnostics PR at https://github.com/openclaw/openclaw/pull/44363 is still open, while https://github.com/openclaw/openclaw/pull/46501 was closed unmerged.

## What This Changes

The issue requests per-attempt cron execution logging, skipped/failure warnings, cron health reporting, and `openclaw cron list` annotations for unhealthy jobs.

## Best Possible Solution

Land one coordinated cron-observability design that logs bounded scheduled/startup/manual outcomes, warns on skipped and non-throwing error results, exposes cron health through the chosen probe surface, and annotates list output from existing counters.

## Reproduction Assessment

Not applicable as a strict bug reproduction: this is an observability enhancement layered on cron reliability work. Source inspection on current main shows the requested logging, health, and list-warning surfaces remain incomplete.

## Solution Assessment

Unclear until maintainers choose the cron health/readiness API shape and reconcile the overlapping diagnostics work. The logging and list-warning pieces can be implemented narrowly after that product surface is settled.
