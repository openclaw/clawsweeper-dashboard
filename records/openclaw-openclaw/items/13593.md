## Summary

Keep open: current main still has only partial cron observability. Cron state, counters, failure alerts, durable run history, and generic probes exist, but source inspection shows the requested per-attempt success/skipped runtime logs, cron-aware health/readiness surface, and `openclaw cron list` warning annotations are still missing; the related implementation PRs were closed unmerged.

## What This Changes

The issue requests structured cron execution logging, cron health reporting, and CLI warnings when scheduled jobs fail, skip, or become overdue.

## Best Possible Solution

Define the cron observability contract, then implement bounded structured outcome logs, cron-aware probe/readiness reporting, and operator-facing cron list warnings on top of the existing counters, alerts, diagnostics, and run history.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not as a strict runtime bug repro; this is an observability enhancement. Source inspection on current main gives high confidence that success/skipped runtime logs, cron list warnings, and cron-aware probe/readiness behavior are still missing.

## Solution Assessment

Is this the best way to solve the issue? Unclear until maintainers define the cron health/readiness contract. Structured logging and list warnings are narrow, but the probe/API shape needs owner direction before a repair lane should implement it.
