## Summary

Keep open: current main has useful partial cron diagnostics, including error-state tracking, failure alerts, durable run history, and generic probes, but it still lacks the requested success/skipped per-attempt runtime logs, cron list warning annotations, and a cron-aware health/readiness contract.

## What This Changes

Add cron execution observability for failed or skipped jobs through structured logs, health/readiness reporting, and `openclaw cron list` warnings.

## Best Possible Solution

Define the cron observability contract, then implement bounded structured outcome logs, cron-aware probe/readiness reporting, and operator-facing cron list warnings on top of the existing counters, alerts, diagnostics, and run history.

Not applicable as a strict runtime bug reproduction because this is an observability enhancement. Source inspection of current main gives high confidence that success/skipped logging, cron list warnings, and cron-aware health/readiness are still missing.

## Solution Assessment

Unclear. The logging and list-warning pieces are narrow and maintainable, but the health/readiness endpoint behavior needs maintainer product/API direction before one implementation can be called the best fix.
