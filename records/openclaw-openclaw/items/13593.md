## Summary

Keep open. Current main has partial cron diagnostics, including run-state counters, failure alerts, durable JSONL run history, and generic liveness/readiness probes, but it still lacks the requested per-attempt runtime logs for all outcomes, cron-aware health/readiness reporting, and `openclaw cron list` warning annotations. The overlapping diagnostics PR #44363 is still open, while #46501 was closed unmerged.

## What This Changes

The issue requests per-attempt cron execution logging, skipped/failure warnings, cron health reporting, and `openclaw cron list` annotations for unhealthy or overdue jobs.

## Best Possible Solution

Land a coordinated cron-observability change that records bounded structured logs for scheduled, startup catch-up, and manual run outcomes; warns on skipped and non-throwing error results; exposes cron health through the maintainer-chosen probe surface; and annotates `openclaw cron list` for overdue jobs and repeated failures or skips while preserving existing run history, counters, and failure-alert behavior.

## Reproduction Assessment

Yes for the missing observability surfaces. Source inspection on current main shows non-throwing scheduled/manual outcomes can complete through state updates, events, and run history without per-attempt logger calls; `/health/cron` is absent; and `printCronList` has no warning annotation path.

## Solution Assessment

Unclear as an autonomous fix, but the requested direction is sound. The narrow implementation should be coordinated with #44363 and the existing gateway probe/readiness contract before adding a cron-specific health surface.
