## Summary

Keep this issue open. Current main has durable run state, consecutive failure/skip counters, failure alerts, and structured logging for error-status outcomes, but the requested success/skipped attempt logs, unhealthy-job CLI warnings, and cron-aware health diagnostics remain absent. Both prior implementation PRs closed unmerged, and no newer canonical item owns the remaining work.

## What This Changes

Request structured cron execution logging, cron-aware health signals, and CLI warnings for overdue or repeatedly failing jobs.

## Best Possible Solution

Add concise structured outcomes at the canonical cron completion boundary and derive CLI warnings from existing scheduler state, while exposing cron degradation through an explicitly designed diagnostic surface rather than implicitly redefining shallow liveness.

## Reproduction Assessment

Yes for the missing observability surfaces: current-main source clearly lacks success/skipped attempt logs, unhealthy-job CLI annotations, and cron-aware readiness. The original historical scheduler execution failure was not reproduced live on current main.

## Solution Assessment

Partly. Canonical completion logging and CLI annotations are the narrowest maintainable direction, but returning unhealthy probe status for cron failures requires an explicit operator-impact decision and upgrade proof.
