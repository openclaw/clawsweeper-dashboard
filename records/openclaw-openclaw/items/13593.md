## Summary

Keep open: current main has partial cron diagnostics, but it still lacks the requested per-attempt success/skipped runtime logs, cron-aware health/readiness behavior, and `openclaw cron list` warning annotations. The overlapping diagnostics PR at https://github.com/openclaw/openclaw/pull/44363 remains open and broad, while the narrower prior implementation at https://github.com/openclaw/openclaw/pull/46501 was closed unmerged.

## What This Changes

The issue asks for cron execution outcome logging, cron health reporting, and warnings in cron list output when scheduled jobs fail, skip, or become overdue.

## Best Possible Solution

Land a coordinated cron-observability design that preserves current run history, counters, and failure alerts while adding bounded outcome logs, the chosen health/readiness contract, and operator-facing list warnings.

## Reproduction Assessment

Not applicable as a strict bug reproduction: this is an observability enhancement layered on cron reliability work. Source inspection on current main shows the requested logging, health, and list-warning surfaces remain incomplete.

## Solution Assessment

Unclear. The logging and list-warning pieces can be implemented narrowly, but the health/readiness endpoint shape and the overlapping diagnostics PR need maintainer coordination before this is the best maintainable fix.
