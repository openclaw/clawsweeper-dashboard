## Summary

Keep open: current main and v2026.5.22 still only partially cover cron diagnostics. Cron state now tracks outcomes, counters, diagnostics, failure alerts, and run history, but the requested all-outcome execution logs, cron-aware health/readiness signal, and `openclaw cron list` warning annotations are still missing; the two related implementation PRs were closed unmerged.

## What This Changes

The issue requests cron execution observability: per-attempt logs and warnings, a cron health signal, and CLI warnings when scheduled jobs fail, skip, or become overdue.

## Best Possible Solution

Define the cron observability contract, then add bounded scheduled/manual outcome logs, cron-aware probe/readiness reporting, and list warnings while preserving existing counters, failure alerts, task records, diagnostics, and run history.

## Reproduction Assessment

Yes at source level: current main still lacks all-outcome runtime logs, cron list warning annotations, and cron-aware probe/readiness checks. I did not run a live cron scenario because this was a read-only review.

## Solution Assessment

Unclear until maintainers choose the health/readiness and log-level contract. The implementation should build on existing cron counters, diagnostics, alerts, task records, and run history rather than resurrecting closed stale PRs unchanged.
