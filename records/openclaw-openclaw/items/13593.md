## Summary

Keep open. Current main has useful cron state counters, failure-alert support, durable run history, and generic gateway probes, but it still does not implement the requested per-attempt runtime logging, cron-specific health/readiness signal, or warning annotations in `openclaw cron list`; the overlapping diagnostics PR #44363 is still open and #46501 closed unmerged.

## What This Changes

Requests cron execution observability through per-attempt logs, skipped/failure warnings, cron health reporting, and `openclaw cron list` warnings.

## Best Possible Solution

Land one coordinated cron-observability design that adds bounded structured run outcome logs, warning treatment for skipped and non-throwing failures, cron-aware probe/readiness reporting through the chosen API surface, and list annotations for overdue or repeatedly failing/skipped jobs.

## Reproduction Assessment

Yes for the missing observability surfaces. Source inspection on current main shows scheduled/manual outcomes can update state and history without per-attempt runtime logs, `/health/cron` is absent, readiness has no cron dependency, and `printCronList` has no warning path.

## Solution Assessment

Unclear as an autonomous fix, but the requested direction is sound. The maintainable path is to coordinate with #44363 and decide the cron health/readiness API before adding code.
