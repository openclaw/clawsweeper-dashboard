## Summary

Keep open: current main still has only partial cron diagnostics, while the requested per-attempt success/skipped runtime logs, cron-aware health/readiness reporting, and `openclaw cron list` warnings remain incomplete. The open diagnostics PR at https://github.com/openclaw/openclaw/pull/44363 overlaps the area, so this needs maintainer coordination rather than closure or an automated standalone repair.

## What This Changes

Request logging every cron execution outcome, warnings for skipped/failing/overdue jobs, a cron-aware health endpoint, and CLI list warnings.

## Best Possible Solution

Land one coordinated cron-observability design that logs bounded scheduled/startup/manual outcomes, warns on skipped and non-throwing error results, exposes cron health through the chosen probe surface, and annotates list output from existing counters.

## Reproduction Assessment

Not applicable as a strict bug reproduction: this is an observability enhancement layered on cron reliability work. Source inspection on current main shows the requested logging, health, and list-warning surfaces remain incomplete.

## Solution Assessment

Unclear until maintainers choose the cron health/readiness API shape and reconcile the overlapping open diagnostics PR. The logging and list-warning pieces can be implemented narrowly once that product surface is settled.
