## Summary

Keep open: current main has partial cron diagnostics, but the requested per-attempt success/skipped logging, cron-aware health/readiness reporting, and unhealthy-job warnings in `openclaw cron list` are still incomplete. The overlapping diagnostics PR at https://github.com/openclaw/openclaw/pull/44363 is still open and conflicting, so this needs maintainer coordination rather than cleanup closure.

## What This Changes

The issue requests cron execution observability: structured run-attempt logs, skipped-run warnings, cron health/probe reporting, and `openclaw cron list` warnings for unhealthy jobs.

## Best Possible Solution

Land one coordinated cron observability design that logs bounded scheduled, startup catch-up, and manual run outcomes; warns on skipped outcomes; exposes cron health through the chosen probe surface; and annotates list output from existing state counters.
