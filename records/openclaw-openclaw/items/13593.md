## Summary

Keep open. Current main still has partial cron diagnostics, including execution state counters, failure alerts, durable run-history JSONL, and generic health/readiness probes, but it still lacks the requested per-attempt runtime logging for all cron outcomes, cron-aware health/readiness, and warning annotations in `openclaw cron list`; #44363 remains open and overlapping, while #46501 was closed unmerged.

## What This Changes

The issue requests cron observability for failed or skipped job execution through structured logs, cron health reporting, and CLI status warnings.

## Best Possible Solution

Land a coordinated cron-observability change that records bounded structured logs for scheduled, startup catch-up, and manual outcomes; warns on skipped and non-throwing error results; exposes cron health through the chosen probe surface; and annotates `openclaw cron list` for overdue jobs and repeated failures or skips while preserving existing counters, run history, and failure alerts.

## Reproduction Assessment

Yes for the missing observability surfaces. Source inspection on current main shows ordinary scheduled/manual outcomes can update state and history without per-attempt runtime logger calls, `/health/cron` is absent, readiness has no cron dependency, and `printCronList` has no warning path.

## Solution Assessment

Unclear as an autonomous fix, but the requested direction is sound. The best implementation should be coordinated with #44363 and the existing gateway probe/readiness contract before adding a cron-specific health surface.
