## Summary

Keep open: current main has partial cron diagnostics, including counters, failure alerts, durable run history, and generic probes, but it still lacks all-outcome per-attempt logs, cron-aware health/readiness, and `openclaw cron list` warnings. The related implementation PRs at https://github.com/openclaw/openclaw/pull/44363 and https://github.com/openclaw/openclaw/pull/46501 are closed unmerged, and no open related PR owns the central request.

## What This Changes

The issue requests cron observability for scheduler execution failures: log every scheduled/manual attempt, expose cron health, and show unhealthy-job warnings in `openclaw cron list`.

## Best Possible Solution

Define the cron observability contract, then add bounded structured logs for scheduled/manual outcomes, cron-aware probe or readiness reporting, and `openclaw cron list` warnings while preserving existing counters, failure alerts, run history, and narrower ghost-run diagnostics.
