## Summary

Keep open: current main and the latest release still only partially cover this request. Cron now tracks run status, counters, diagnostics, failure alerts, and durable run logs, but it still lacks all-outcome runtime logging, cron-aware health/readiness, and `openclaw cron list` warning annotations.

## Best Possible Solution

Land one coordinated cron-observability change that preserves existing run history, counters, and failure alerts while adding bounded structured logs, unhealthy CLI annotations, and a maintainer-approved cron health/readiness surface.
