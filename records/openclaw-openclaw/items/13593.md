## Summary

Keep open: current main has partial cron diagnostics, but it still lacks the requested per-attempt runtime logging, cron-aware health/readiness reporting, and unhealthy-job warnings in `openclaw cron list`.

## What This Changes

Request adds cron execution logging, cron-aware health reporting, and CLI warnings for failed, skipped, or overdue cron jobs.

## Best Possible Solution

Land one coordinated cron observability design that logs bounded structured outcomes, warns on skipped and non-throwing failures, exposes cron health through the chosen probe surface, and annotates cron list output from existing state counters.

## Solution Assessment
