## Summary

Keep this open. Current main has partial cron diagnostics, including run-state counters, failure alerts, durable run history, and generic probes, but it still lacks the requested per-attempt runtime logging, cron-aware health/readiness reporting, and `openclaw cron list` warnings for unhealthy jobs.

## What This Changes

Request structured cron execution logging, cron-aware health reporting, and CLI warnings for failed, skipped, or overdue scheduled jobs.

## Best Possible Solution

Land one coordinated cron-observability design that logs bounded structured outcomes, warns on skipped/non-throwing failures, exposes cron health through the chosen probe surface, and annotates cron list output using existing state counters.

## Reproduction Assessment

Not applicable as a strict bug reproduction: this is an observability feature request layered on top of cron reliability work. Source inspection gives high confidence that the requested logging, health, and CLI warning surfaces are still incomplete on current main.

## Solution Assessment

Unclear until maintainers choose the cron health/readiness API shape and reconcile PR #44363. The logging and list-warning pieces can be implemented narrowly once that surface decision is made.
