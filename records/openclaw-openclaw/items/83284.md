## What This Changes

The issue asks for migrate text previews to keep config detail redaction while reporting aggregate conflict and sensitive counts accurately when hidden config items block apply.

## Best Possible Solution

Keep config rows hidden, but make the preview header use authoritative aggregate conflict and sensitive counts, or an equivalent hidden-conflict aggregate, without changing apply behavior.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a plan whose only conflict is kind=config is filtered out of visibleConflicts, while the same plan still fails assertConflictFreePlan through summary.conflicts.

## Solution Assessment

Yes. The narrow maintainable fix is to keep HIDDEN_KINDS for item rendering while using plan.summary.conflicts and plan.summary.sensitive, or explicit hidden aggregates, for header counts.
