## What This Changes

The issue asks for migration text previews to keep config details redacted while reporting accurate conflict and sensitive counts for hidden config plan items.

## Best Possible Solution

Land one focused fix that keeps hidden config rows redacted but derives header conflict and sensitive counts from the authoritative migration summary, with regression coverage for hidden config conflicts.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a plan with only a hidden kind=config conflict is filtered out before header counts are computed, while assertConflictFreePlan still blocks from plan.summary.conflicts.

## Solution Assessment

Yes. The narrow maintainable fix is to keep HIDDEN_KINDS for detail rendering and use plan.summary.conflicts and plan.summary.sensitive, or equivalent hidden aggregates, for the preview header.
