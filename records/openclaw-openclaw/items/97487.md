## Summary

Keep open: current main still builds managed npm generation project names from a readable artifact-generation key, and the linked open pull request is the active fix candidate rather than a merged implementation. The issue is actionable and source-reproducible against the current plugin install path.

## What This Changes

The issue asks OpenClaw to keep managed npm plugin generation roots short enough that nested Codex native binaries remain spawnable on Windows after plugin install or update.

## Best Possible Solution

The desired end state is a bounded managed-npm install-path fix that shortens only the generation fingerprint suffix while preserving prefix-based recovery and leaving the broader stale-gateway lifecycle work tracked separately.

## Reproduction Assessment

Yes for source-level reproduction: current main still composes generated managed npm project roots from the full readable generation key, and upstream Codex source confirms the nested win32-x64 codex.exe path the report measured. I did not run a live Windows current-main upgrade repro in this read-only review.

## Solution Assessment

Yes, the compact deterministic hash suffix proposed by the linked PR appears to be the narrow owner-boundary fix because prefix-based generation discovery can remain intact while the long fingerprint portion is bounded.
