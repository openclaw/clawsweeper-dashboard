## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Publish generated titles without overriding labels:** `src/auto-reply/reply/session-title-generator.ts:162`
  - body: `maybeGenerateSessionTitle` stores the generated value in `autoTitle`, but then emits that same text as `label`. `sessions.changed` applies `label` directly and the UI resolves labels before display names or derived titles, so a session with a manual label/displayName/subject can be overwritten in the live picker. Skip manually named sessions and notify the UI through derived-title metadata instead of the label field.
  - confidence: 0.89
- **[P2] Use derived titles in the picker fallback:** `ui/src/ui/chat/session-controls.ts:542-543`
  - body: Adding `derivedTitle` to `resolveSessionDisplayName` does not make persisted generated titles show in the session picker, because `resolveSessionScopedOptionLabel` only calls that helper when `label` or `displayName` is already present. A reloaded row with only `autoTitle`/`derivedTitle` still falls back to the key/rest. Treat `derivedTitle` as a scoped option display candidate and cover it with a UI test.
  - confidence: 0.84
