## Summary

Keep open. The branch implements the requested AI session-title feature, but the proposed generator can launch duplicate in-flight model calls, can miss the turn threshold when early transcript content exceeds 8 KiB, and still lacks real after-fix behavior proof beyond test/typecheck claims.

## What This Changes

Adds async AI-generated session titles with new `sessionTitle` config, persisted `autoTitle`, gateway/UI `derivedTitle` display, tests, and a changelog entry.

## Best Possible Solution

If maintainers want this feature, merge a product-approved version with per-session in-flight dedupe, a bounded transcript reader that reaches the configured threshold, aligned config/docs metadata, and real Control UI or gateway proof.

## Solution Assessment

No. The feature direction may be useful, but this patch is not the best mergeable shape until generation is deduped, transcript scanning reaches the threshold reliably, and the new default-on config/runtime behavior is approved and demonstrated.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Add an in-flight title-generation guard:** `src/auto-reply/reply/session-title-generator.ts:48-59`
  - body: Every successful reply after the threshold can call `generateAndPersistTitle` while the first summarization is still running, because the only guard checks the stale `sessionEntry` passed into this function. That can launch duplicate paid model calls and later completions can overwrite earlier titles; add a per-session in-flight guard and re-check the current store entry before persisting.
  - confidence: 0.87
- **[P2] Read enough transcript to reach the threshold:** `src/auto-reply/reply/session-title-generator.ts:161-181`
  - body: The reader samples only the first 8192 bytes and counts user turns inside that sample. If an early user message or transcript entry pushes the configured third user turn past 8 KiB, `userMessageCount` stays below the threshold forever and the title never generates; stream or continue reading until the threshold is reached within a bounded byte cap.
  - confidence: 0.79
