## Summary

Keep open: this active feature PR is not implemented on current main and still has source-level blockers around manual-label precedence, scoped picker display, and public config documentation/metadata, plus missing real behavior proof.

## What This Changes

The PR adds configurable AI-generated session titles after replies, persists an `autoTitle`, propagates derived title data through Gateway and Control UI session rows, and updates config, tests, generated hashes, and changelog text.

## Best Possible Solution

Land one product-approved implementation that stores AI titles as derived metadata, preserves manual names, updates config/docs metadata, and proves live title creation plus sidebar rendering.

## Reproduction Assessment

Not applicable as a current-main bug because this is a feature PR. The review blockers are source-reproducible from PR head: generated titles are emitted as labels and scoped picker labels ignore derived-only rows.

## Solution Assessment

No. The direction is plausible, but this branch is not yet the best implementation because it routes generated titles through manual-label semantics, misses a scoped picker path, and leaves config/privacy documentation incomplete.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Emit generated titles as derived metadata:** `src/auto-reply/reply/session-title-generator.ts:162`
  - body: `maybeGenerateSessionTitle` persists `autoTitle` but then broadcasts the generated text as `label`. The Gateway snapshot applies `params.label` through the manual label precedence path, so live Control UI rows can treat an AI title as a user label; emit a derived-title field or reload the row after persist, and avoid generating for manually named sessions.
  - confidence: 0.9
- **[P2] Include derived-only rows in scoped picker labels:** `ui/src/ui/chat/session-controls.ts:827-828`
  - body: The PR adds `derivedTitle` to `resolveSessionDisplayName`, but `resolveSessionScopedOptionLabel` only calls that helper when `label` or `displayName` exists. Agent-scoped rows with only `derivedTitle` still fall back to the raw scoped key, so the picker can keep showing unreadable ids; include `derivedTitle` in this predicate and cover it with a UI test.
  - confidence: 0.86
- **[P2] Document the new sessionTitle config surface:** `src/config/zod-schema.ts:683-690`
  - body: This adds a public `sessionTitle` config object, but the config help/label metadata and user docs are not updated for `enabled`, `turnsBeforeTitle`, or `maxChars`. That leaves the default-on background model-call behavior without the normal config guidance, including privacy and cost semantics.
  - confidence: 0.78
