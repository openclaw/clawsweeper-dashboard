## Best Possible Solution

Consolidate around one product-approved implementation that stores generated titles as derived session metadata, preserves manual labels/display names/subjects, documents config and privacy semantics, and includes redacted runtime proof.

No. The implementation direction is plausible, but the patch should publish generated values through the derived-title path, skip manually named sessions, and get product/privacy approval plus real behavior proof before merge.
