## Summary

Keep open. The feature is not on current main, but this PR is not mergeable yet: CI fails on the new `SessionEntry.autoTitle` key, the generated-title write does not notify the Control UI, public config metadata/docs are incomplete, real behavior proof is missing, and default-on transcript summarization needs maintainer product/privacy approval.

## What This Changes

The PR adds AI-generated session titles after replies, persists an `autoTitle` field, adds `sessionTitle` config/schema plumbing, exposes derived titles through gateway session rows, updates Control UI display types, tests title precedence, and adds a changelog entry.

## Best Possible Solution

A mergeable version should reserve the new session metadata key, broadcast an updated session row after title persistence, complete config help/labels/docs, and gate model-based summarization behind an approved privacy/cost policy.

## Reproduction Assessment

Yes for the PR defects: CI logs reproduce the missing `autoTitle` reserved-key failure, and source inspection shows the generated title is persisted without a follow-up `sessions.changed` event. The linked request itself is a feature request, so current-main bug reproduction is not applicable.

## Solution Assessment

No. The feature direction is plausible, but this branch is not the best mergeable form until it fixes the build contract, updates the live session row after persistence, completes public config metadata/docs, and resolves default-on summarization policy.
