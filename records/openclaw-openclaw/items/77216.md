## Summary

Keep open. The feature direction is plausible, but this patch does not yet make the generated title reliably visible in the live sessions UI and it lacks acceptable real behavior proof.

## What This Changes

The PR adds async AI-generated session titles with new session metadata/config plumbing, gateway and Control UI title rendering changes, tests, generated config hash updates, and a changelog entry.

## Best Possible Solution

Define generated titles as gateway-owned session metadata, emit a live session update after persistence, handle long transcripts with existing bounded transcript helpers, document the new config contract, and prove the flow in a real multi-turn session.

## Reproduction Assessment

Yes for the review findings from source inspection: the PR persists `autoTitle` after the reply path without a later session broadcast, and the new parser only scans the first 8 KiB. I did not run a live E2E test because this was a read-only review.

## Solution Assessment

No. The requested feature is reasonable, but this implementation should first use the existing live session-update contract and a less brittle transcript scan before it is a maintainable fix.

Full review comments:

- **[P2] Broadcast title updates after persistence:** `src/auto-reply/reply/session-title-generator.ts:132-136`
  - body: The title is written to the session store after the reply has already completed, but this path never emits a new `sessions.changed` payload. Since the Control UI only updates session rows from gateway events, the sidebar can keep showing the old derived title until a later refresh or unrelated message.
  - confidence: 0.91
- **[P2] Read enough transcript data before deciding no title is due:** `src/auto-reply/reply/session-title-generator.ts:170-176`
  - body: `readUserMessagesFromTranscriptHead` reads only the first 8192 bytes and then counts user turns from whatever complete JSONL records happen to fit there. A long initial prompt or early assistant output can keep `userMessageCount` below the threshold forever, so those sessions never get an auto-generated title.
  - confidence: 0.87
