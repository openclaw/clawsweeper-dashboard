## What This Changes

The PR adds asynchronous AI-generated session titles after replies, a `sessionTitle` config surface, `autoTitle` session metadata, Gateway/UI derived-title propagation, generated config hash updates, focused session-title tests, and a changelog entry.

## Best Possible Solution

Land a product-approved implementation that stores generated titles as derived session metadata, preserves manual `label`/`displayName`/`subject` values, documents config/privacy semantics, and includes redacted runtime proof.

## Reproduction Assessment

Not applicable as a current-main bug; this is a feature PR. The PR defect is source-reproducible from the head diff: the generated title is emitted as a `label`, and the UI prioritizes labels over manual display names and derived titles.

## Solution Assessment

No. The implementation direction is reasonable, but the patch should publish generated values through the derived-title path, avoid work for manually named sessions, and provide real behavior proof plus product/privacy approval before merge.
