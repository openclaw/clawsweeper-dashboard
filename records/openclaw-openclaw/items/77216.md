## Summary

Keep open: the PR addresses the linked feature request, but the current head is not mergeable because the generated title is persisted without a live session update, public config metadata is incomplete, and default-on transcript summarization needs maintainer product/privacy approval.

## What This Changes

The PR branch adds default-on AI-generated session titles using a new `sessionTitle` config, an async reply-time title generator, `autoTitle` persistence, session row/UI display plumbing, generated config baseline changes, tests, and a changelog entry.

## Best Possible Solution

Land the feature only after maintainer approval of the default behavior, with an explicit live session update after title persistence and complete config help/docs alignment.

## Reproduction Assessment

Yes for the PR defects. Source inspection of PR head shows `updateSessionStoreEntry` persists `autoTitle` after `runReplyAgent` returns without any session update, while normal Control UI sends rely on session events unless an explicit refresh was requested.

## Solution Assessment

No. The PR direction matches the linked feature request, but the mergeable solution should push a live session update, align public config help/docs, and settle the default-on privacy/cost policy before shipping.
