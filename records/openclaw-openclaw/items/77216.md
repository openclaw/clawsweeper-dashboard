## Summary

Keep open. The PR implements the requested feature direction, but the final head still has blocking model-privacy, live update, and generated config-doc drift issues before it can merge.

## What This Changes

The PR adds default-on AI-generated session titles with a new `sessionTitle` config, async title generation after replies, stored `autoTitle`/`derivedTitle` display plumbing, tests, generated schema output, and a changelog entry.

## Best Possible Solution

If maintainers accept the feature, implement it as gateway session metadata that uses the active session model/runtime, emits and merges `derivedTitle` in live session events, refreshes config artifacts/docs, and adds focused regression coverage.

## Reproduction Assessment

Yes for the PR defects: source inspection of the final PR head and current main shows the default-model label path, missing `derivedTitle` event projection/merge, and unchanged config baseline. A runtime reproduction is not needed to see these blockers.

## Solution Assessment

No. The PR direction matches the request, but the implementation is not the narrowest maintainable solution until it preserves the active model boundary, updates live session events, and refreshes generated config-doc artifacts after product/privacy approval.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Use the active session model for title generation:** `src/auto-reply/reply/session-title-generator.ts:100-105`
  - body: This background job calls `generateConversationLabel` without the provider/model selected for the reply run. That helper resolves the agent default model, so sessions using a local/private or alternate model can have their first messages summarized by another configured provider.
  - confidence: 0.88
- **[P2] Propagate generated titles through session events:** `ui/src/ui/chat/session-controls.ts:450-451`
  - body: Returning `row.derivedTitle` only helps after a fresh `sessions.list` load. The session event projection and Control UI merge field list still omit `derivedTitle`, so the sidebar can keep showing the old fallback title after `autoTitle` is persisted until a full reload.
  - confidence: 0.84
- **[P2] Refresh the config docs baseline:** `src/config/zod-schema.ts:620-627`
  - body: Adding top-level `sessionTitle` changes the public config schema. The PR updates `schema.base.generated.ts`, but leaves `docs/.generated/config-baseline.sha256` unchanged, and `pnpm config:docs:check` is designed to fail on that hash drift.
  - confidence: 0.9

## Security Review
