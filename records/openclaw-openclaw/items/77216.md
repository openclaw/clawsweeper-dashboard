## Summary

Keep open. The feature is not implemented on current main, and this PR still has merge-blocking issues: CI fails on the new session metadata field, the generated title is persisted without a live session update, the new public config surface is under-documented, real behavior proof is missing, and default-on transcript summarization needs maintainer privacy/cost approval.

## What This Changes

The PR adds default-on AI-generated session titles, `sessionTitle` config, `autoTitle` session metadata, session row/UI display plumbing, tests, generated config hash updates, and a changelog entry.

## Best Possible Solution

Consolidate on one reviewed implementation that makes generated title metadata gateway-owned, reserves the session field, emits a live session update, documents the config contract, adds focused regression coverage, and includes redacted real behavior proof after maintainer privacy/cost approval.

## Reproduction Assessment

Yes for the PR defects. CI reproduces the type/lint failure on the latest head, and source inspection shows the async `autoTitle` write has no corresponding `sessions.changed` broadcast path for ordinary Control UI chat sends.

## Solution Assessment

No. The feature direction matches the linked request, but this implementation is not the best mergeable form until it fixes the build invariant, pushes live session updates, completes config docs/help, provides real behavior proof, and gets maintainer approval for default-on transcript summarization.
