## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Deduplicate title generation before calling the model:** `src/auto-reply/reply/session-title-generator.ts:47-58`
  - body: `maybeGenerateSessionTitle` runs after each successful reply and only checks the stale `sessionEntry.autoTitle` snapshot. While the first summarization is still in flight, later replies can launch duplicate paid model calls and race the store write; add a per-session in-flight guard and re-check the fresh store entry before persisting.
  - confidence: 0.88
- **[P2] Read past the first chunk before rejecting the threshold:** `src/auto-reply/reply/session-title-generator.ts:170-176`
  - body: The transcript reader counts user turns only inside the first 8192 bytes. If early transcript content pushes the configured third user turn past that head chunk, title generation never starts; keep reading until the threshold is reached or a bounded byte cap is hit.
  - confidence: 0.84
- **[P2] Notify session listeners after storing the generated title:** `src/auto-reply/reply/session-title-generator.ts:132-136`
  - body: Persisting `autoTitle` only updates the session store. The final transcript `sessions.changed` event has already fired before this async write completes, so the Control UI sidebar will not show the generated title until another refresh or event happens; emit an appropriate session change after a successful title write.
  - confidence: 0.78
