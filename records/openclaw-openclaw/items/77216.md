## Summary

Keep open. This is an active feature PR and current main does not implement AI-generated session titles; the branch still has a source-reproducible Control UI update defect, missing real behavior proof, and a default-on transcript summarization policy question.

## What This Changes

The PR adds default-on AI-generated session titles after replies, persists `autoTitle`, adds `sessionTitle` config/schema, and threads derived titles into Gateway and Control UI session rows.

## Best Possible Solution

Consolidate around one reviewed design that treats generated titles as gateway-owned session metadata, emits reliable session updates after persistence, documents default/privacy semantics, and includes redacted real Control UI proof.

## Reproduction Assessment

Yes for the PR defects: source inspection at PR head shows generated title persistence happens after reply lifecycle events and uses `updateSessionStoreEntry` without broadcasting a session update. For the original request, reproduction is not applicable because it is a new feature.

## Solution Assessment

No. The feature direction is plausible, but this branch is not the best mergeable form until it emits session updates, completes config metadata/docs, provides real proof, and receives maintainer policy approval for default-on model calls.
