## Summary

Keep open: the branch is useful source work, but it should not merge as-is because the generated title is persisted without a live session update, the transcript scan can miss valid turn counts, and default-on background summarization needs maintainer privacy/cost approval.

## What This Changes

The branch adds default-on AI-generated session title generation, `sessionTitle` config/schema plumbing, persisted `autoTitle` display in gateway/UI session rows, focused tests, generated config baseline hashes, and a changelog entry.

## Best Possible Solution

Land the feature only after maintainer approval, with a bounded transcript reader that can reliably reach the configured turn threshold, an explicit live session update after `autoTitle` persistence, and complete config help/docs alignment.

## Reproduction Assessment

Yes for the PR defects: source inspection shows `autoTitle` persistence happens after the reply without a session event, and the transcript reader counts only one fixed head chunk. The normal Control UI path only reloads sessions after reset-triggered sends, so the live sidebar can miss the new title until refresh or another event.

## Solution Assessment

No. The direction matches the linked feature request, but the mergeable solution needs reliable turn detection, a pushed session update or equivalent refresh path, config documentation/help parity, and maintainer approval for default-on summarization.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Emit a session update after saving auto titles:** `src/auto-reply/reply/session-title-generator.ts:131-136`
  - body: Normal Control UI chat sends do not reload the session list after completion; they rely on `sessions.changed` updates. This async path writes `autoTitle` only after the reply has already finished and emits no event, so the sidebar can keep showing the fallback title until a manual refresh or later unrelated session event.
  - confidence: 0.9
- **[P2] Read enough transcript data to count configured turns:** `src/auto-reply/reply/session-title-generator.ts:147-151`
  - body: The helper reads only the first 8192 bytes and counts user messages parsed from that slice. If an early JSONL message is large enough to push later turns outside the chunk, `userMessageCount` stays below the threshold on every run and the session never gets an auto title despite having enough exchanges.
  - confidence: 0.84
- **[P3] Document the public sessionTitle config:** `src/config/zod-schema.ts:617-624`
  - body: This adds a user-facing top-level config object, but the PR does not add matching schema labels/help or user docs. The generated config schema therefore exposes raw `sessionTitle` fields without titles/descriptions, leaving the config UI/docs contract incomplete.
  - confidence: 0.78
