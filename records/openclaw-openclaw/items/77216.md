## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Make title generation opt-in by default:** `src/auto-reply/reply/session-title-generator.ts:37-38`
  - body: `maybeGenerateSessionTitle` treats missing `sessionTitle` config as enabled and is invoked after every successful reply. That means existing installs start sending transcript excerpts to a model provider and incurring model cost without an explicit privacy setting, so preserve the current no-extra-model-call behavior unless maintainers intentionally approve and document default-on behavior.
  - confidence: 0.93
- **[P1] Preserve the selected auth profile for title calls:** `src/auto-reply/reply/get-reply-run.ts:1225-1226`
  - body: The background title call receives only `provider` and `model`, while normal agent runs thread the selected auth profile through fallback/user overrides. Sessions pinned to a specific auth profile can therefore generate titles with the wrong account or fail even though the reply itself succeeded.
  - confidence: 0.9
- **[P2] Render derived titles through the session display helper:** `ui/src/ui/types.ts:425`
  - body: Adding `derivedTitle` to the UI row type is not enough because `resolveSessionDisplayName` still considers only `label` and `displayName`. Any refreshed list or shared-helper render path can keep showing the fallback session key/display name instead of the generated title.
  - confidence: 0.84
