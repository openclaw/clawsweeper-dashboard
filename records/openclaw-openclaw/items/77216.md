## What This Changes

Adds configurable AI-generated session titles by summarizing early transcript messages after replies, persisting `autoTitle`, projecting it as `derivedTitle`, and updating config, schema, tests, changelog, and UI row types.

## Best Possible Solution

Consolidate one canonical auto-title design that has maintainer-approved default/privacy semantics, preserves reply auth/model context, stores derived metadata without competing fields, displays it through shared session label helpers, and includes redacted live proof.

## Reproduction Assessment

Not applicable as a current-main bug; this is a feature PR. The blocking defects are source-reproducible from the PR diff: default-enabled title calls, missing auth-profile propagation, and UI display helpers that do not consume `derivedTitle`.

## Solution Assessment

No. The maintainable path is to settle the default/privacy policy first, reuse the resolved reply auth/model context, and make the shared session display helper consume persisted derived titles before merge.
