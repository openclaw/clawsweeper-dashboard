## Summary

Keep open. The feature is still absent from current main, but this PR has a blocking source-reproducible display-name regression, lacks real runtime proof, and needs maintainer product/privacy approval before merge.

## What This Changes

This PR adds AI-generated session title generation after replies, `sessionTitle` config, `autoTitle` session metadata, Gateway/UI propagation, generated config baseline hashes, tests, and changelog text.

## Best Possible Solution

Land one product-approved implementation that stores generated titles as derived session metadata, preserves manual labels/display names/subjects, documents config/privacy semantics, and includes redacted live runtime proof.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a current-main bug because this is a feature PR; the PR defect is source-reproducible by following the generated title lifecycle event into the Gateway session-change payload and then the Control UI label-priority path.

## Solution Assessment

Is this the best way to solve the issue? No. The direction is plausible, but generated titles should be published as derived metadata, manual titles should suppress generation, and product/privacy semantics need approval before merge.
