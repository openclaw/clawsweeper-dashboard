## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Add an in-flight title-generation guard:** `src/auto-reply/reply/session-title-generator.ts:48-58`
  - body: Every successful reply after the threshold can call `generateAndPersistTitle` while the first summarization is still running, because the only guard checks the stale `sessionEntry` passed into this function. That can launch duplicate paid model calls and later completions can overwrite earlier titles; add a per-session in-flight guard and re-check the current store entry before persisting.
  - confidence: 0.87
- **[P2] Read enough transcript to reach the threshold:** `src/auto-reply/reply/session-title-generator.ts:170-176`
  - body: The reader samples only the first 8192 bytes and counts user messages inside that sample. If early transcript content pushes the configured third user turn past 8 KiB, `userMessageCount` stays below the threshold forever and the title never generates; stream or continue reading until the threshold is reached within a bounded byte cap.
  - confidence: 0.81
