## Summary

Keep open: current main still lacks AI-generated session titles, and this active feature PR has blocking correctness findings plus mock-only proof. The branch is useful directionally, but it needs contributor runtime evidence and maintainer product/privacy review before merge.

## What This Changes

This PR adds configurable AI-generated session titles, persists an `autoTitle` field, pushes derived title data through Gateway and Control UI session rows, and updates config, tests, generated hashes, and changelog text.

## Best Possible Solution

Land one product-approved implementation that stores generated titles as derived session metadata, preserves manual names, uses existing provider/runtime auth contracts, documents config and privacy semantics, and proves live UI rendering.

## Reproduction Assessment

Not applicable as a current-main bug because this is a feature PR for AI-generated session titles. Source inspection gives high confidence current main only supports heuristic first-message-derived titles, while the PR defects are source-reproducible from the PR head title event and picker-label paths.

## Solution Assessment

No. The direction is plausible, but this branch currently sends generated titles through the manual label path, misses derived-only scoped picker labels, and still needs maintainer product/privacy approval plus real runtime proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Avoid publishing generated titles as manual labels:** `src/auto-reply/reply/session-title-generator.ts:162`
  - body: `maybeGenerateSessionTitle` persists the generated value as `autoTitle`, but then emits it as `label`. `sessions.changed` applies that label through the manual label/display-name precedence path, so a generated title can overwrite a manual name in the live picker. Publish the generated value as derived session metadata instead, and avoid generating for manually named sessions.
  - confidence: 0.89
- **[P2] Use derived titles in scoped picker labels:** `ui/src/ui/chat/session-controls.ts:542-543`
  - body: The new `derivedTitle` branch only helps when `resolveSessionDisplayName` is called. The scoped picker helper still calls it only for rows with `label` or `displayName`, so a reloaded row with only `derivedTitle` falls back to the scoped key text. Treat `derivedTitle` as a scoped option display candidate and cover it with a UI test.
  - confidence: 0.84
