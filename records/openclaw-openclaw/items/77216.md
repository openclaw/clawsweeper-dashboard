## Summary

Keep this PR open, but it is not mergeable as-is: the patch adds useful session-title work, yet the title job can use a different model than the active session, live session events do not carry the new title field, and the generated config schema artifacts are stale.

## What This Changes

Adds default-on AI-generated session titles with `sessionTitle` config, async persistence, sidebar display wiring, tests, and a changelog entry.

## Best Possible Solution

If maintainers accept the feature, route title generation through the active session model/runtime, emit and merge `derivedTitle` in live session events, regenerate public config artifacts, and keep focused regression coverage.

## Reproduction Assessment

Yes for the PR defects: source inspection of the PR diff and current main shows the model-selection, event-projection, event-merge, and generated-schema gaps. A runtime reproduction is not needed to see these merge blockers.

## Solution Assessment

No, this is not the best implementation as-is. The narrower maintainable path is to preserve the active session model boundary, update live session event contracts, and refresh generated config artifacts after product/privacy approval.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Use the active session model for title generation:** `src/auto-reply/reply/session-title-generator.ts:100-106`
  - body: This background job calls `generateConversationLabel`, whose current implementation resolves the agent default model. Because the call does not pass the active `provider`/`model` from the reply run or session override, a session using a local/private or alternate model can have its first messages summarized by a different configured provider.
  - confidence: 0.88
- **[P2] Propagate generated titles through session events:** `ui/src/ui/chat/session-controls.ts:450-452`
  - body: Using `row.derivedTitle` in the display name only helps after a full `sessions.list` reload. The live `session.message`/`sessions.changed` projection and merge paths omit `derivedTitle`, so normal chat sends can leave the sidebar showing the old fallback title after `autoTitle` is persisted.
  - confidence: 0.83
- **[P2] Regenerate the config schema artifacts:** `src/config/zod-schema.ts:617-627`
  - body: Adding the top-level `sessionTitle` schema changes the public config surface, but this PR does not update `src/config/schema.base.generated.ts` or `docs/.generated/config-baseline.sha256`. The generated schema comparison and config docs checks will report drift until those artifacts are regenerated.
  - confidence: 0.91

## Security Review

Status: needs_attention

Summary: The patch introduces a concrete privacy-boundary concern because background title generation can send transcript snippets to a different configured provider than the active session model.

Concerns:

- **[medium] Preserve the session model privacy boundary:** `src/auto-reply/reply/session-title-generator.ts:100`
  - body: The new title generator calls the existing conversation-label helper without the active run model, and that helper resolves the agent default model. For sessions using a local/private or alternate model override, the title job can disclose the first user messages to another configured provider.
