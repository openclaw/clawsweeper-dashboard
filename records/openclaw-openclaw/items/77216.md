## Summary

Keep open. The feature is not implemented on current main, but this PR still has a blocking source issue: generated titles can override manually named sessions in the live UI, and the external PR still lacks real after-fix behavior proof for an actual title-generation run.

## What This Changes

The PR adds AI-generated session titles after successful replies, a `sessionTitle` config surface, `autoTitle` persistence, Gateway/UI derived-title propagation, tests, generated config hash changes, and a changelog entry.

## Best Possible Solution

Consolidate around one product-approved implementation that stores generated titles as derived session metadata, preserves manual titles, documents the config semantics, and includes redacted real runtime proof.

## Reproduction Assessment

Not applicable as a current-main bug; this is a feature PR. The PR defect is source-reproducible from the head branch: manual `label`/`displayName`/`subject` values do not block generation, and the generated value is emitted as a `label` that the UI prioritizes.

## Solution Assessment

No. The direction is plausible, but this patch should preserve manual titles, publish generated values through the derived-title path instead of a label override, and provide real behavior proof plus maintainer product/privacy approval before merge.
