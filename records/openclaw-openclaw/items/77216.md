## Summary

Keep open. This PR targets a valid feature request for https://github.com/openclaw/openclaw/issues/77165, but the current patch has source-reproducible bugs in title-generation gating and manual-title handling, and the external PR still lacks accepted real behavior proof.

## What This Changes

Adds configurable AI-generated session titles that run asynchronously after replies, persist an `autoTitle` session field, and surface `derivedTitle` through gateway and Control UI session rows.

## Best Possible Solution

Land one consolidated implementation after product approval that counts turns robustly, respects manual titles, documents the `sessionTitle` config, includes focused tests, and provides real multi-turn runtime proof.

## Reproduction Assessment

Not applicable as a feature request. The review findings are source-reproducible from the PR head by reading the fixed-head transcript counter and the generation skip conditions.

## Solution Assessment

No. The feature direction is plausible, but this patch can miss long early conversations and spends a model call on manually named sessions; those need fixing before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Scan past the fixed head before gating title generation:** `src/auto-reply/reply/session-title-generator.ts:170-171`
  - body: `userMessageCount` is computed only from the first 8KB read at offset 0 and then compared with `turnsBeforeTitle`. If an early assistant/tool response pushes the second or third user message past that buffer, every later call still sees too few turns and the title is never generated for that session. Count user turns from the transcript index or scan until the threshold is reached while keeping a separate prompt budget.
  - confidence: 0.89
- **[P2] Skip sessions that already have a manual title:** `src/auto-reply/reply/session-title-generator.ts:37-40`
  - body: The generator only skips when `autoTitle` exists. Sessions can already carry a user-visible `label`, `displayName`, or `subject`, and those take precedence in the gateway/UI, so this still makes a background model call and stores a hidden auto title for a session the user already named. Return early for existing manual titles unless regeneration is explicit.
  - confidence: 0.82
