## Summary

Keep open: the PR addresses a useful feature request, but it is not merge-ready because the title job can use the wrong model/provider, live session updates do not carry the new title field, and the public config schema artifacts are stale.

## What This Changes

The PR adds default-on AI-generated session titles stored as `autoTitle`, a `sessionTitle` config block, gateway/UI title display wiring, tests, and a changelog entry.

## Best Possible Solution

If maintainers accept the feature, route title generation through the active session model/runtime, emit and merge `derivedTitle` in live session events, and regenerate the config schema/docs baselines with focused regression coverage.

## Reproduction Assessment

Yes for the PR defects: source inspection shows the new title path calls a helper that resolves the agent default model, while event projection/merge paths omit `derivedTitle` and generated config artifacts are unchanged. A runtime reproduction is not needed to see those merge blockers.

## Solution Assessment

No, this is not the best implementation as-is. The safer path is to preserve the active session model boundary, update live session event contracts, and refresh generated config artifacts before maintainer product review.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Use the active session model for title generation:** `src/auto-reply/reply/session-title-generator.ts:91-98`
  - body: This background job calls `generateConversationLabel`, whose current implementation resolves the agent default model. Since this call does not pass the active `provider`/`model` from `runPreparedReply` or the session override, a session intentionally using a local/private or alternate model can have its first user messages summarized by a different default provider, contrary to the requested current-model behavior.
  - confidence: 0.88
- **[P2] Propagate generated titles through session events:** `ui/src/ui/chat/session-controls.ts:450-452`
  - body: Using `row.derivedTitle` in the display name only helps after a full `sessions.list` reload. The live `session.message`/`sessions.changed` paths do not project or merge `derivedTitle`, and normal chat sends do not refresh the sessions list, so the sidebar can keep showing the old fallback title after `autoTitle` is persisted.
  - confidence: 0.83
- **[P2] Regenerate the config schema artifacts:** `src/config/zod-schema.ts:620-627`
  - body: Adding the top-level `sessionTitle` schema changes the public config surface, but this PR does not update `src/config/schema.base.generated.ts` or `docs/.generated/config-baseline.sha256`. Runtime schema responses are based on the generated module, and the config schema/docs checks will report drift until those artifacts are regenerated.
  - confidence: 0.91
