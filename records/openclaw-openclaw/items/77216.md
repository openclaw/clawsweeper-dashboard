## Summary

Keep open. The PR is active and linked to an open feature request, but the current head still has source-reproducible UI update and transcript-read defects, incomplete config metadata, insufficient real behavior proof, and an unresolved product/privacy question for default-on summarization.

## What This Changes

The PR adds AI-generated session titles after replies, a new `sessionTitle` config surface, session row/title plumbing through the gateway and Control UI, generated config baseline hashes, tests, and a changelog entry.

## Best Possible Solution

Land the feature only after maintainer-approved defaults/policy, an explicit session-row update when `autoTitle` is persisted, robust transcript scanning, complete config hints/docs, focused regression coverage, and redacted Control UI proof.

## Reproduction Assessment

Yes for the review defects: source inspection shows `autoTitle` is written after reply completion without a follow-up `sessions.changed` event, and the new transcript reader parses only a fixed 8 KiB prefix as complete JSONL records. I did not run tests because this was a read-only review.

## Solution Assessment

No. The feature direction matches the linked request, but this PR is not the best mergeable form until it updates subscribers reliably, reads transcript records safely, completes the public config contract, provides real proof, and settles the default-on summarization policy.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Emit a session update after saving auto titles:** `src/auto-reply/reply/session-title-generator.ts:132-136`
  - body: The generator writes `autoTitle` after `runReplyAgent` has completed, but `updateSessionStoreEntry` only persists the store entry and no `sessions.changed` payload is emitted afterward. The Control UI can keep showing the old fallback title until a manual refresh or unrelated event.
  - confidence: 0.91
- **[P2] Scan complete transcript records before counting title turns:** `src/auto-reply/reply/session-title-generator.ts:170-176`
  - body: This reads only the first 8 KiB and parses split lines as JSON. If the first transcript record is longer than that, the partial JSON is skipped and `userMessageCount` stays below the threshold, so common long initial prompts can prevent auto title generation entirely.
  - confidence: 0.84
- **[P3] Add config hints for sessionTitle:** `src/config/zod-schema.ts:683-690`
  - body: This introduces a public `sessionTitle` config object, but there are no matching `FIELD_LABELS` or `FIELD_HELP` entries or docs coverage. Config UI/docs surfaces can expose raw fields without useful descriptions.
  - confidence: 0.78
