## Summary

Keep open. The latest PR head is still not merge-ready: it persists generated titles without a session update broadcast, leaves the new public config surface under-documented, lacks real after-fix behavior proof, and needs maintainer approval for default-on transcript summarization.

## What This Changes

The PR adds default-on AI-generated session titles after successful replies, stores `autoTitle`, wires `sessionTitle` config/schema, updates generated config baseline hashes, and passes derived titles through gateway and Control UI session rows.

## Best Possible Solution

Land one consolidated implementation that treats generated titles as gateway-owned session metadata, emits subscriber updates after persistence, preserves manual-title precedence, documents the config contract, and includes redacted real Control UI proof after product/privacy approval.

## Reproduction Assessment

Yes for the PR defects: source inspection of PR head shows `autoTitle` is persisted through `updateSessionStoreEntry`, while current main's store helper only writes state and the Control UI depends on `sessions.changed` or explicit reloads. The requested feature itself is not a bug reproduction path.

## Solution Assessment

No. The feature direction is plausible, but this is not the best mergeable form until it broadcasts the persisted title, completes config docs/metadata, supplies real behavior proof, and settles the default-on privacy/cost policy.
