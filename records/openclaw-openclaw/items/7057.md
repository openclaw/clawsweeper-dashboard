## Best Possible Solution

Make the affected tests deterministic by reusing shared retrying temp-dir helpers and applying explicit WSL `skipIf`/`describe.skipIf` only where child-process behavior cannot be made reliable, with production behavior unchanged.

## Reproduction Assessment

## Solution Assessment

Unclear for the reporter's broad skip request; the safer solution is a narrow test-only hardening pass that reuses shared temp helpers first and skips WSL only for cases that remain unreliable.
