## Summary

Current main still has the reported WSL-sensitive test harness shape: workspace-paths uses a local temp helper with plain cleanup, and safe-bins skips only native Windows.

## What This Changes

The issue asks for the Pi tools workspace-paths and safe-bins tests to stop timing out or failing with temp-dir ENOENT under Windows/WSL.

## Best Possible Solution

Make the two affected tests deterministic on supported hosts by reusing shared retrying temp-dir helpers and adding only narrowly scoped WSL guards where child-process behavior cannot be made reliable.

## Reproduction Assessment

Yes, with WSL access: the issue provides focused commands and failure excerpts, and current main still exercises the same unguarded WSL paths. I verified the source path in current main but did not run WSL live.

## Solution Assessment

Yes. A narrow test-only repair using the existing temp-dir helper and WSL detection utilities is the maintainable path; PR #45974 is the closest bounded direction, while PR #7085 should not be revived as-is because it bundled unrelated changes.
