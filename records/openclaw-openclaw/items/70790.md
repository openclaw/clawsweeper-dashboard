## Summary

## What This Changes

The PR adds an embedded-runner stream wrapper that removes empty `tools` arrays and orphaned `tool_choice` fields from outgoing payloads, plus a test expectation update for custom stream wrapping.

## Best Possible Solution

Land a narrow embedded-runner egress sanitizer that composes with provider-owned, boundary-aware, session HTTP fallback, and custom streams, preserves existing auth/signal/cache-boundary behavior, keeps non-empty tools intact, and includes focused regression tests plus a changelog entry.

## Solution Assessment

No, not as currently proposed. A payload-level guard is the right maintainable direction, but this branch needs to cover provider-owned streams and the required release-note surface before it is ready.
