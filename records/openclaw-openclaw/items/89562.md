## Summary

Keep open: current main and the latest release still route voice-call agent work through the `voice` lane, propagate parent abort into embedded attempts and tool execution, and `finalizeCall` still only tears down call state without defining an accepted-spawn detach boundary.

## What This Changes

The issue asks for voice-call hangup to stop aborting independent `sessions_spawn` child work once spawn execution has begun.

## Best Possible Solution

Define and test a narrow accepted-spawn lifecycle boundary so child sessions accepted by `sessions_spawn` survive parent voice-call teardown while pre-accept failures and explicit cancellations still clean up normally.

## Reproduction Assessment

No, not with high-confidence live reproduction. Source inspection shows the current voice-call lane, embedded-run abort propagation, and `sessions_spawn` pre-accept dispatch path that make the reported failure plausible.

## Solution Assessment

Unclear. A narrow accepted-spawn lifecycle boundary is likely the maintainable direction, but maintainers need to decide how it interacts with normal voice hangup, explicit cancellation, and timeout behavior.
