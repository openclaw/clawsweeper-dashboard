## Summary

Keep open: current main and v2026.6.6 still route voice-call consult runs through the `voice` lane, run abort signals still wrap tool execution, and `finalizeCall` still ends call state without coordinating in-flight `sessions_spawn` acceptance. No canonical merged fix or duplicate appears to own the remaining work.

## What This Changes

The issue asks for voice-call hangup not to abort independent `sessions_spawn` child work once spawn execution has begun.

## Best Possible Solution

Define and test a bounded accepted-spawn lifecycle boundary so child sessions accepted by `sessions_spawn` survive parent voice-call teardown while pre-accept failures and explicit cancellations still clean up normally.

## Reproduction Assessment

No high-confidence live phone-call reproduction was run. Source inspection shows the same path remains: voice consults use the `voice` lane, runner abort signals wrap tools, and `call.ended` finalizes call state without coordinating in-flight `sessions_spawn`.

## Solution Assessment

Unclear. A narrow lifecycle boundary for accepted or initializing `sessions_spawn` child work is preferable to adding a broad config knob, but maintainers need to decide how this interacts with normal voice hangup and explicit cancellation.
