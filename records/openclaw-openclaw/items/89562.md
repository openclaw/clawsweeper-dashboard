## Label Justifications

- `P2`: The report describes a bounded session-lifecycle bug affecting voice-call background work, not an emergency outage.
- `impact:session-state`: The central failure is abort propagation from the call session into spawned child session state.

## Summary

Keep open: current main and the latest release still route voice-call agent work through the `voice` lane, propagate embedded-run abort into attempt/tool execution, and leave `sessions_spawn`/ACP child acceptance tied to a post-dispatch registration point rather than an explicit detach boundary.

## What This Changes

The issue asks for voice-call hangup to avoid aborting already-started independent session spawn work.

## Best Possible Solution

Define and test a narrow accepted-spawn lifecycle boundary so child sessions accepted by `sessions_spawn` survive parent voice-call teardown while pre-accept failures and explicit cancellations still clean up normally.

## Reproduction Assessment

No high-confidence live reproduction has been run. Source inspection shows the plausible path: voice responses use the `voice` lane, embedded attempts propagate abort, and subagent/ACP spawn registration happens only after the child gateway call returns.

## Solution Assessment

Unclear. A narrow accepted-spawn lifecycle boundary is likely more maintainable than a broad grace timeout or new config knob, but maintainers need to define how hangup, timeout, and explicit cancellation interact.
