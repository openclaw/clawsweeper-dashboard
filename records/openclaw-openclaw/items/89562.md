## Summary

Keep open: current main and the latest release still route voice-call responses through the `voice` embedded-run lane, propagate run aborts into tool execution, and lack a defined detach boundary for accepted `sessions_spawn` work.

## What This Changes

The issue asks for voice-call hangup not to abort independent child sessions once `sessions_spawn` has begun execution.

## Best Possible Solution

Define and test a narrow accepted-spawn lifecycle boundary so child sessions accepted by `sessions_spawn` survive parent voice-call teardown while pre-accept failures and explicit cancellations still clean up normally.

## Reproduction Assessment

No, not with high-confidence live reproduction. Source inspection shows the voice-call lane, embedded-run abort propagation, and `sessions_spawn` pre-accept dispatch path that make the reported failure plausible, but no real call flow was replayed.

## Solution Assessment

Unclear. A narrow accepted-spawn detach boundary is likely the maintainable direction, but maintainers need to decide how it interacts with hangup, timeout, and explicit cancellation semantics.
