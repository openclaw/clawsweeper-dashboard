## Summary

Keep open: current main still has the same plausible Voice Call and embedded-agent abort shape, and I did not find a merged fix or canonical replacement item for the hangup-versus-independent-child-session contract.

## What This Changes

The issue asks for Voice Call hangup teardown not to abort independent background work spawned through `sessions_spawn`.

## Best Possible Solution

Define and test a lifecycle contract where hangup stops call-owned audio/consult work but does not kill an accepted independent child session, then implement it at the narrow Voice Call/gateway boundary with live or focused regression proof.

## Reproduction Assessment

No. Source inspection shows a plausible abort-before-dispatch race and a Voice Call consult path on the `voice` lane, but I did not establish a high-confidence current-main path where `call.ended` aborts `sessions_spawn` before child acceptance.

## Solution Assessment

Unclear. A gateway carve-out for independent spawn work and a Voice Call grace-drain are both plausible, but the best fix depends on the maintainer-approved hangup-versus-background-work lifecycle contract.
