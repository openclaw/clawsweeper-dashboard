## Summary

Keep open: current main still has voice-call paths that run embedded agent work on the voice lane and realtime relay paths that abort active agent runs on close/cancel, while `sessions_spawn` only reports acceptance after child initialization and registration. I did not find a merged fix or canonical issue that decouples an accepted spawn from voice-call teardown, and the remaining choice needs maintainer/product review plus live voice proof.

## What This Changes

The issue reports that ending a voice call aborts gateway lane-managed embedded runs before `sessions_spawn` can finish initializing child sessions.

## Best Possible Solution

Keep this open for a maintainer-owned lifecycle decision: define a generic contract for when `sessions_spawn` is safely decoupled from parent voice-call teardown, then prove it with focused voice-call/talk relay regression coverage and live voice proof.

## Reproduction Assessment

No, not yet with high confidence. Source inspection supports the reported voice-run abort and spawn-initialization race, but a live voice-call hangup repro on current main is still missing.

## Solution Assessment

Unclear. A plugin grace timeout is plausible but may be the wrong layer; the maintainable fix likely needs a generic gateway/agent handoff rule for accepted child work rather than a voice-only config knob.
