## What This Changes

The issue reports that ending a Voice Call aborts an in-flight `sessions_spawn` before the spawned child session is initialized, and asks for independent child sessions to decouple from the call lifecycle once spawning starts.

## Best Possible Solution

Define and test a Voice Call consult cancellation contract where hangup stops call audio but does not kill accepted independent child sessions, then implement it at the correct plugin/gateway boundary with regression and live proof.

## Reproduction Assessment

No. The issue has a plausible source trail through Voice Call realtime consults, gateway agent abort tracking, and `sessions_spawn`, but I did not establish a high-confidence current-main failing path or live voice-call reproduction.
