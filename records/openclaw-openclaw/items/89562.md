## Summary

Keep open: current main and v2026.6.9 still route voice-call consults through abortable voice/Talk run lifecycles, and `sessions_spawn` only becomes durable after the child spawn is accepted.

## What This Changes

Hanging up a voice-call session can abort the active voice consult before an in-flight `sessions_spawn` child session has accepted and detached.

## Best Possible Solution

Define a durable detach contract for starting or accepted `sessions_spawn` work from voice consults, then prove with focused source coverage and a live voice/Talk scenario that hangup no longer kills independent child work.

## Reproduction Assessment

Source-reproducible, but not live-reproduced here. The code path shows voice consults run with `lane: "voice"` under abortable Talk/embedded-run lifecycle, while `sessions_spawn` only records an accepted child after the child Gateway call returns.

## Solution Assessment

Unclear until maintainers choose the lifecycle contract. A plugin-only grace timeout is plausible, but a shared detach boundary for child spawns may be cleaner so voice, Talk, and other aborting parent surfaces do not diverge.
