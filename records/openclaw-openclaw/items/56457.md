  - body: `splitRemainingChunk` can increment `safeBreakIdx` to make progress, but the caller discards the returned `breakIdx` and slices with the original index.   - confidence: 0.91

  - body: Current main documents and applies a 17-line Discord soft cap to avoid very tall messages; this changes omitted `maxLines` to unlimited character-only chunking. Existing users will get different message counts and taller Discord posts, so this needs an explicit compatibility decision or opt-in path before merge.
  - confidence: 0.88
- **[P2] Preserve current CJK and codepoint-safe splits:** `extensions/discord/src/chunk.ts:173-174`
  - body: Current main has a merged repair that clamps hard splits to code-point boundaries and uses CJK punctuation before hard cuts. This branch's line splitter slices raw UTF-16 limits and whitespace only, so rebasing or merging as written would regress those Discord message-splitting cases.
  - confidence: 0.84
- **[P2] Preserve Thinking-prefixed reasoning italics:** `extensions/discord/src/chunk.ts:365-366`

  - confidence: 0.9

Keep it Discord-local or add the documented SDK contract and compatibility coverage after the algorithm is stable.
