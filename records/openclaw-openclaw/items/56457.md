  - body: Changing `DEFAULT_MAX_LINES` to `Number.MAX_SAFE_INTEGER` silently removes the documented 17-line Discord default for existing users whose config omits `maxLinesPerMessage`. Preserve the old default, or make this an explicit maintainer-approved upgrade with docs and upgrade proof.

  - confidence: 0.88
- **[P1] Use the safe split index returned by the helper:** `src/markdown/chunk.ts:299-303`
  - body: `splitRemainingChunk` can increment `safeBreakIdx`, but the caller ignores the returned `breakIdx` and slices with the original value. When that branch fires, one input character is in neither the emitted chunk nor `nextRemaining`, so delivered content can be silently dropped.
  - confidence: 0.9
- **[P2] Do not drop fence state when carry-over cannot shrink:** `src/markdown/chunk.ts:253-258`
  - body: The fallback that assigns `nextRemaining = tail` drops the active fence opener, so later chunks can render fenced content as plain text. Keep or reconstruct the fence context instead of switching to a bare tail.
  - confidence: 0.84
- **[P2] Avoid shrinking the break into the fence opener:** `src/markdown/chunk.ts:217-223`
  - body: When reserve adjustment shrinks the effective limit into the opener line, the helper can emit an unbalanced opener chunk even though a balanced split fits within `maxChars`. Choose a balanced opener break before falling back inside the opener.
  - confidence: 0.83
