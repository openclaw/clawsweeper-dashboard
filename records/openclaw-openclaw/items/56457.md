  - confidence: 0.9
- **[P2] Keep fence context when the reopened tail cannot shrink:** `src/markdown/chunk.ts:257`
  - body: When reopening the fence would not reduce the remaining buffer, this branch assigns `nextRemaining = tail`, so the rest of a fenced block can be delivered as plain text. Preserve or reconstruct the active opener instead of discarding fence context.
  - confidence: 0.86
- **[P2] Avoid shrinking the split into the fence opener:** `src/markdown/chunk.ts:217-223`
  - body: The reserve adjustment can pick a break inside the opening fence line; `splitRemainingChunk` then disables carry-over and emits an unbalanced opener chunk even when a balanced split fits. Keep the break after the opener or choose a different balanced boundary.
  - confidence: 0.84
- **[P1] Do not publish the helper before the SDK contract is settled:** `src/plugin-sdk/reply-chunking.ts:4`
  - body: This exports a new Markdown chunking helper through the public plugin SDK while the algorithm still has blocking rendering defects and no maintainer-visible contract/docs decision. Keep it internal or complete the SDK contract work before exposing it to third-party plugins.
  - confidence: 0.82
