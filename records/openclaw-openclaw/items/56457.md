Full review comments:

- **[P1] Use the adjusted split index when slicing chunks:** `src/markdown/chunk.ts:303`
  - body: `splitRemainingChunk` can increment `safeBreakIdx` to make progress, but the caller discards the returned `breakIdx` and slices with the original index. When that branch fires, one character can be omitted from both the emitted chunk and `nextRemaining`, so Discord/plugin SDK callers silently lose fenced content.
  - confidence: 0.9

Existing users will get different message counts and taller Discord posts, so this needs an explicit compatibility decision or opt-in path before merge.
  - confidence: 0.86

  - body: Current main has a merged repair that clamps hard splits to code-point boundaries and uses CJK punctuation before hard cuts. This branch's line splitter still slices raw UTF-16 limits and whitespace only, so rebasing or merging as written would regress those Discord message-splitting cases.
  - confidence: 0.82

Keep it Discord-local or add the documented SDK contract and compatibility coverage after the algorithm is stable.
  - confidence: 0.78
