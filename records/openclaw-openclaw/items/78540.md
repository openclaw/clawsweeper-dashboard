## Summary

Keep this open. Current main still has a source-reproducible false doctor warning for a documented `memory-lancedb` active memory slot, and the live issue is already tied to open fix work rather than a merged implementation.

## What This Changes

The issue reports a false `openclaw doctor` missing-memory-plugin warning when `memory-lancedb` is installed and selected as the active memory slot.

## Best Possible Solution

Land a focused `memory-lancedb` runtime/capability registration fix, preferably the direction in https://github.com/openclaw/openclaw/pull/80211 after its review finding is fixed, then close this issue once the fix is merged.

## Reproduction Assessment

Yes, at source level: current main warns when `resolveActiveMemoryBackendConfig` returns null, and `memory-lancedb` is documented as an active slot plugin while not registering the runtime capability that function reads.

## Solution Assessment

No patch is attached to this issue, so the best solution is to finish one of the open fix PRs rather than close it now. The runtime-registration approach is the more complete direction because it makes doctor and status discover `memory-lancedb` through the existing SDK contract.
