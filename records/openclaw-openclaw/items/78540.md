## Summary

This looks like a valid current-main false warning: `doctor` treats a selected memory plugin without a registered memory runtime as “no active memory plugin,” while `memory-lancedb` is documented and declared as the active memory slot plugin but does not register the runtime capability that this check expects.

## What This Changes

The issue requests that `openclaw doctor` stop warning that no active memory plugin is registered when `plugins.slots.memory` is configured for `memory-lancedb`.

## Best Possible Solution

Keep this open and fix the doctor/memory-runtime path so a valid `memory-lancedb` memory slot is recognized without the false “no active memory plugin” warning, with a regression test for the documented config shape.

## Reproduction Assessment

Yes, at source level: `noteMemorySearchHealth` warns when `resolveActiveMemoryBackendConfig` returns null, and `memory-lancedb` does not register the memory runtime capability needed for that call to succeed. I did not run a live macOS/npm-global doctor session in this read-only review.

## Solution Assessment

Unclear because no patch is proposed, but the narrow maintainable direction is to align doctor’s active-memory-plugin detection with the plugin slot/runtime contract rather than leaving a memory-lancedb-specific false warning.
