## Summary

Keep open: current main still has a source-reproducible false doctor warning for the documented memory-lancedb slot, and an open closing PR already tracks this issue. The stronger runtime-registration approach is also open, so this should be resolved by reviewing/landing one of those PRs rather than closing the issue early.

## What This Changes

The issue asks for `openclaw doctor` to stop reporting a missing active memory plugin when `memory-lancedb` is installed, enabled, and selected as `plugins.slots.memory`.

## Best Possible Solution

Land a narrow fix that makes memory-lancedb visible through the generic memory runtime/capability contract, with doctor/status regression coverage for the documented enabled slot configuration.

## Reproduction Assessment

Yes, at source level: current main returns null from `resolveActiveMemoryBackendConfig` when memory-lancedb owns the slot because that plugin does not register the memory runtime, and doctor warns on that null path. I did not run the full npm-global macOS repro locally.

## Solution Assessment

Unclear until the open PRs are resolved; the runtime-registration approach in https://github.com/openclaw/openclaw/pull/80211 looks more maintainable than a broad doctor-only suppression in https://github.com/openclaw/openclaw/pull/78557, but it still has a focused review blocker.
