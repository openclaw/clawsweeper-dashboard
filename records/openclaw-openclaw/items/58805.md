## What This Changes

The branch attempts to add `skills` to `PluginRecord`, forward manifest-declared skills through plugin loader and CLI registry record construction, default non-manifest records to `[]`, and add loader regression coverage.

## Best Possible Solution

Reapply the metadata preservation on current main by adding `skills` to the canonical registry types and record factory, forwarding `manifestRecord.skills` at manifest-backed loader and CLI registry call sites, defaulting non-manifest records to `[]`, and keeping focused regression coverage without changing skill discovery semantics.

## Solution Assessment

No for this PR as submitted. Preserving manifest metadata is the right narrow fix, but the branch needs to be rebased and repaired against the current `registry-types`/`loader-records` split and then backed by real behavior proof before merge.
