## Summary

Keep open: current main still drops manifest-declared plugin `skills` from loaded `PluginRecord`s, and this PR targets a real bug, but the PR head is syntactically broken, updates a stale registry surface, has failing CI, and still lacks contributor real-behavior proof.

## What This Changes

This PR adds `skills` to loaded plugin records, forwards manifest-declared skills through loader-created records and CLI registry paths, defaults non-manifest records to an empty array, and adds regression coverage.

## Best Possible Solution

Reapply the metadata preservation on current main by adding `skills` to `registry-types` and `loader-records`, forwarding `manifestRecord.skills` at all manifest-backed record creation sites, defaulting non-manifest records to `[]`, and adding focused loader/status coverage without changing skill discovery semantics.
