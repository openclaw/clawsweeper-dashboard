## Summary

Keep open. Current main still drops manifest-declared plugin skills from runtime plugin records, so the intended bug fix remains useful, but this PR head is stale against the split loader/type files and currently contains blocking syntax/type-surface defects.

## What This Changes

The PR adds `skills` to plugin records, forwards manifest-declared skills through loader-created records, defaults non-manifest records to an empty skills array, and adds regression coverage.

## Best Possible Solution

Reapply the metadata preservation narrowly on current main's `registry-types`, `loader-records`, and loader call sites, with focused tests and a single changelog entry, while leaving plugin discovery and skill-loading semantics unchanged.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: create a plugin manifest with `skills: ["skills"]`, load it through `loadOpenClawPlugins()`, and the current runtime `PluginRecord` shape has no place where those manifest skills are copied.

## Solution Assessment

No for this PR as submitted. Preserving manifest metadata on the loaded record is the right narrow fix, but this branch must be rebased or replaced because it edits stale surfaces and leaves unparseable TypeScript.
