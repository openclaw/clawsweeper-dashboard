## What This Changes

The PR raises `ORPHAN_LOCK_PAYLOAD_GRACE_MS` from 5s to 30s, rewrites session write-lock acquisition/release away from `@openclaw/fs-safe`, updates tests, and adds a changelog entry.

## Best Possible Solution

Rebase to current main and change only the orphan payload grace/comment plus focused tests, while preserving `createFileLockManager`, false-live owner cleanup, and snapshot-safe release semantics; merge only after patched-build proof.

## Reproduction Assessment

Yes from source: current main treats an empty lock payload as missing pid/invalid createdAt and then reclaims by mtime after the 5s grace when `staleMs` is larger. I did not run the focused test because this review is read-only.

## Solution Assessment

No for this PR as written. Raising the grace is a narrow maintainable fix, but the branch should preserve the current fs-safe-backed lock manager and cleanup behavior instead of replacing them.
