## What This Changes

The PR raises the session write-lock payload-less orphan grace from 5 seconds to 30 seconds, updates adjacent boundary tests, and adds a changelog entry.

## Best Possible Solution

Apply the payload-less grace consistently to both contended acquisition and cleanup/removal paths, add focused cleanup-path regression coverage, then land with redacted runtime proof from a patched setup.

## Reproduction Assessment

Yes by source inspection. fs-safe creates an empty sidecar before metadata is written, current main reclaims a 10-second payload-less lock during acquisition, and the startup cleanup path can still remove a payload-less `.jsonl.lock` without consulting the grace.

## Solution Assessment

No, not yet. Raising the contended-acquire grace is the right direction, but the cleanup path should share the same payload-less mtime grace so all OpenClaw reclaim paths preserve a suspended legitimate owner.
