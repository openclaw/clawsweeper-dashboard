## Summary

Keep this PR open for maintainer review: the race fix is narrow and proof-backed, but the global 30s orphan grace changes recovery behavior for abandoned empty locks and can turn short configured acquire timeouts into busy-session failures.

## What This Changes

Raises the payload-less session write-lock orphan grace from 5s to 30s and updates tests for preserving a 10s empty lock while reclaiming one past 30s.

## Best Possible Solution

Land only after maintainers choose the orphan empty-lock recovery policy and the code/tests/docs encode both the delayed-owner case and the permanently abandoned-lock case.

## Reproduction Assessment

Yes. Source inspection shows current main reclaims payload-less locks older than 5s, and the PR body supplies a two-process before/after fixture showing the contender acquiring before the stalled owner writes metadata on current main.

## Solution Assessment

Unclear. Raising the grace to 30s addresses the demonstrated delayed-owner race, but the best mergeable fix needs maintainer agreement on the availability tradeoff for abandoned empty locks and short acquire timeouts.
