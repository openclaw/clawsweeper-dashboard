## Summary

Keep open: the underlying lock bug is plausible and source-reproducible, but the latest branch is not mergeable because it imports a missing module, drops the owner-argv cleanup contract used by session-lock doctor repair, and still lacks after-fix proof from a patched build.

## What This Changes

The PR raises payload-less session write-lock orphan grace from 5s to 30s, rewrites session-write-lock ownership state around a process-scoped map, updates focused tests, and adds a changelog entry.

## Best Possible Solution

Keep the existing fs-safe-backed lock manager and owner-argv cleanup contract, apply only the bounded grace/comment/regression-test change, then merge after patched-build behavior proof.

## Reproduction Assessment

Yes from source: current main treats a payload-less lock as missing pid/invalid createdAt and reclaims it by mtime after the 5s grace when staleMs is larger. I did not execute tests because this review is read-only.

## Solution Assessment

No as submitted: raising the bounded grace is a good narrow fix, but this branch also introduces a missing import and removes existing doctor lock-cleanup behavior. The safer solution is a minimal constant/comment/test change that preserves the current contracts.
