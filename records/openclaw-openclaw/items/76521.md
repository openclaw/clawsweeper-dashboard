## Summary

Close as implemented on current main. The reported disabled-flag path is now handled by a standalone `active-memory` plugin with explicit activation and live-config gates, so `enabled: false` skips recall before any embedded memory subagent is run; this is present on `main` but not in the latest `v2026.5.2` release tag.

## What This Changes

The issue asks for Active Memory to be skipped entirely when `plugins.entries.active-memory.enabled` or its plugin config disables it, avoiding 30s blocking recall timeouts.

## Best Possible Solution

Keep the current standalone Active Memory plugin gates and make sure the next release includes this main-only fix for users still on 2026.4.x or `v2026.5.2`.

## Reproduction Assessment

No. On current main, source and regression tests show disabled Active Memory exits before `runEmbeddedPiAgent`, so the reported blocking recall path should not reproduce with `enabled:false`.

## Solution Assessment

Yes. The current fix addresses the narrow bug at both plugin activation and live hook execution, which is more maintainable than adding another memory-core special case.
