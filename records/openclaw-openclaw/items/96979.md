## Summary

Keep open: the patch targets a real TUI duplicate-message bug, but its suppression boundary is too broad and can drop the late final chat event that is still needed after history reload re-adopts an in-flight run.

## What This Changes

The PR adds a TUI surrendered-run-id set so late chat delta/final/error events are skipped after a matching `sessions.changed` history reload.

## Best Possible Solution

Narrow the dedupe so live finals remain authoritative for in-flight runs, dedupe only once a persisted or already-displayed final is known, and add focused regression coverage plus redacted real behavior proof.

## Reproduction Assessment

Yes at source level: current main reloads history on matching `sessions.changed`, and history replay can rebuild assistant rows without run IDs. I did not run a live Telegram/TUI reproduction in this read-only review.

## Solution Assessment

No. Suppressing all late events for pre-reload run IDs is too broad because `loadHistory()` can re-adopt an in-flight run and still require its subsequent final chat event to render the completed answer.
