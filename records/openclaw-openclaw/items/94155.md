## Summary

Keep open: current main and v2026.6.9 still count all loaded SQLite cron jobs in status while default list/listPage return enabled jobs only, and docs say plain `cron list` shows all matching jobs. The related open PR is useful context but only adds a disabled-job hint and explicitly preserves the default that causes this mismatch.

## What This Changes

The issue reports that after JSON-to-SQLite cron migration, `cron list` returns only enabled jobs while `cron status` and SQLite show all migrated jobs.

## Best Possible Solution

Maintainers should choose the plain CLI/RPC `cron list` contract, then align service behavior, CLI flags, docs, tests, and the related hint PR with that decision.

## Reproduction Assessment

Yes, at source level. With disabled migrated rows present, `cron.status()` counts every loaded job while default `cron.list()` and `cron.listPage()` filter to enabled jobs before returning `total`.

## Solution Assessment

Unclear until maintainers choose the contract. If plain list should mean all jobs, the best fix is to change defaults and regression coverage; if enabled-only is intentional, docs and user guidance need to be corrected instead.
