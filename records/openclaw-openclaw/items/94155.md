## Summary

Keep open: current main and v2026.6.8 still have the reported cron visibility mismatch when stored SQLite rows include disabled jobs. `cron.status` counts every loaded job, while plain `cron.list` defaults to enabled-only; docs and the open related PR leave that default contract unsettled.

## What This Changes

The issue asks for plain `cron list` to return all migrated SQLite cron jobs so its count matches `cron status` and the persisted store.

## Best Possible Solution

Maintainers should decide the plain `cron list` contract first, then align the service/RPC default, CLI help, docs, tests, and related hint PR to that single behavior.

## Reproduction Assessment

Yes, at source level: with 41 loaded SQLite rows and only 9 enabled rows, `cron.status` counts all loaded jobs while default `cron.list` and `listPage` filter to enabled jobs unless `includeDisabled` or `--all` is used.

## Solution Assessment

Unclear until maintainers choose the contract. The mechanical code change is small, but the best fix depends on whether plain `cron list` should mean all jobs or enabled jobs plus an explicit `--all` operator view.
