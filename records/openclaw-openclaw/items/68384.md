## What This Changes

The PR changes exec SecretRef parsing so a single-id `jsonOnly: false` provider accepts valid JSON or exactly one non-empty plaintext line, and adds a POSIX regression test for multiline noisy stdout rejection.

## Solution Assessment

Acceptable mitigation for ambiguous multiline plaintext, but not a complete fix for the broader hot-reload corruption report. The best path is secops-approved fail-closed parsing plus proof and user-visible contract context.
