## Summary

Keep open: the patch is focused and source-correct for the narrow multiline plaintext parser issue, but it still needs real behavior proof and secops acceptance of the fail-closed compatibility change before merge.

## What This Changes

The PR changes exec SecretRef parsing so single-id `jsonOnly: false` stdout must be JSON or exactly one non-empty plaintext line, with a regression test for multiline noisy output.

## Best Possible Solution

Land only after redacted real resolver proof, visible one-line plaintext contract context, and secops acceptance that ambiguous multiline plaintext should fail while the broader hot-reload issue remains tracked.

## Reproduction Assessment

Yes for the narrow parser behavior: current `main` returns full trimmed stdout after JSON parse failure for a single-id `jsonOnly: false` provider, so a two-line stdout payload is source-reproducible. I did not establish a live gateway hot-reload reproduction for the broader linked issue.

## Solution Assessment

Acceptable mitigation for ambiguous multiline plaintext, but not a complete fix for the broader hot-reload corruption report. The best path is secops-approved fail-closed parsing plus proof and user-visible contract context.
