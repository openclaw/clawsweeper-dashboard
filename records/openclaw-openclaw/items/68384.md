## Summary

Keep open: the patch is focused and source-correct for rejecting multiline noisy plaintext, but it changes fail-closed SecretRef behavior, lacks real after-fix proof, and does not fully resolve the broader hot-reload corruption report.

## What This Changes

This PR changes exec SecretRef parsing so single-id `jsonOnly: false` stdout must be valid JSON or exactly one non-empty plaintext line, and adds a resolver regression test.

## Best Possible Solution

Continue normal maintainer review; ClawSweeper found no patch-correctness issue.

## Solution Assessment

Acceptable mitigation for ambiguous multiline plaintext, but not a complete fix for the broader hot-reload corruption report. The safest path is secops-approved fail-closed parsing plus documentation/proof while leaving the broader issue open.
