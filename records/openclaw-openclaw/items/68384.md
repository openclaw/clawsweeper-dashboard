## What This Changes

The PR updates the exec secret parser to reject multi-line non-JSON stdout for single-id `jsonOnly: false` providers and adds a POSIX resolver regression test.

## Best Possible Solution

Land this or an equivalent focused resolver guard after secops approves the plaintext stdout contract, document the `jsonOnly: false` one-line requirement, and keep the related security issue open until same-line corruption is fixed or separately tracked.

## Solution Assessment

Unclear. The fail-closed parser guard is narrow and maintainable for newline-separated noise, but secops should approve the compatibility change and it does not resolve the same-line corruption shown in the related security issue.
