## Summary

## What This Changes

The branch makes single-id `jsonOnly: false` exec SecretRef plaintext output valid only when it is JSON or exactly one non-empty line, and adds a regression test for polluted multi-line stdout.

## Best Possible Solution

Land a resolver-boundary fix only after secops accepts the fail-closed plaintext contract, add public upgrade or release-note context for `jsonOnly: false`, attach redacted real exec-provider proof, and keep the linked security issue open until any same-line corruption path is resolved or separately tracked.
