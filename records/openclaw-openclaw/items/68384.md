## Summary

Keep open: this is a focused secrets hardening PR, but it changes a CODEOWNERS-protected parser contract and needs secops review plus contributor-provided real behavior proof before merge.

## What This Changes

The PR changes `src/secrets/resolve.ts` to reject multi-line non-JSON stdout for single-id `jsonOnly: false` exec SecretRefs and adds a POSIX regression test in `src/secrets/resolve.test.ts`.

## Best Possible Solution

Have secops approve the plaintext exec stdout contract, land this or an equivalent focused resolver guard with public docs/release guidance, and keep https://github.com/openclaw/openclaw/issues/64821 open until same-line corruption is resolved or separately tracked.

## Reproduction Assessment

Yes, from source inspection. On current main, a single-id exec SecretRef with `jsonOnly: false` and two non-empty non-JSON stdout lines reaches `src/secrets/resolve.ts:556-560` and returns the full trimmed stdout; this read-only pass did not execute tests.

## Solution Assessment

Unclear. The fail-closed parser guard is narrow and defensible for multiline noise, but secops should decide compatibility and it does not prove resolution of the same-line corruption reported in the linked security issue.
