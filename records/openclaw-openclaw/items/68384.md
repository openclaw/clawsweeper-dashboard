## What This Changes

The PR changes `parseExecValues()` so single-id `jsonOnly: false` exec stdout must be valid JSON or exactly one non-empty plaintext line, and adds a POSIX regression test.

## Best Possible Solution

Land a secops-approved parser hardening with redacted real exec-provider proof and release/docs context for the plaintext contract, while keeping the related same-line corruption issue open until that path is resolved.

## Solution Assessment
