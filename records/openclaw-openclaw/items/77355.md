## Summary

Keep open: current main still preserves group-chat sender data only as current-turn/prompt-local metadata while durable user transcript text remains the clean body, so the continuation-skip attribution gap is still source-reproducible and needs a generic design decision.

## What This Changes

The issue requests durable sender attribution on group-chat user messages so prior turns remain attributable when continuation-skip relies on the rolling LLM context.

## Best Possible Solution

Define a generic persisted sender-attribution field for user transcript messages, map it safely at provider boundaries, and add continuation-skip group-chat regression coverage while preserving current-turn metadata.

## Reproduction Assessment

Yes, by source inspection. Group turns keep BodyForAgent/transcript text clean, carry sender metadata in currentTurnContext, and continuation-skip can reuse prior transcript messages after historical runtime-context metadata has been stripped.

## Solution Assessment

Unclear. Message-array-level attribution is a plausible durable layer, but the transcript schema, provider conversion, and untrusted-sender semantics need maintainer design before a narrow repair PR.
