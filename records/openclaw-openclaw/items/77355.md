## Summary

Keep open: current main and the latest release still treat group sender attribution as current-turn inbound context only, while persisted/replayed user messages have no durable sender field and historical metadata is stripped before model replay.

## What This Changes

The issue requests durable per-message sender attribution for group-chat user turns so sender identity survives continuation-skip rolling context.

## Best Possible Solution

Define a provider-agnostic durable sender-attribution contract for user turns and serialize it consistently through transcript replay, compaction, and provider adapters without polluting BodyForAgent.

## Reproduction Assessment

No live high-confidence reproduction was run. Source inspection gives a clear current-main path: sender metadata is current-turn context while persisted user messages and replay conversion lack a durable sender field.

## Solution Assessment

Unclear without maintainer design input. The per-message metadata direction is cleaner than inline BodyForAgent prefixes, but it requires a durable transcript/provider contract rather than a narrow channel-only patch.
