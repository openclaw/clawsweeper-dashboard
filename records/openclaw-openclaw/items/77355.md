## Summary

Keep open: current main and v2026.6.8 still preserve group sender identity only as current-turn or display metadata, not as a durable per-message model-history contract, and the closest related PR was closed by a maintainer as the wrong boundary.

## What This Changes

The issue asks for group-chat sender attribution to be attached to durable LLM/user-message history entries so attribution survives continuation-skip rolling context.

## Best Possible Solution

Define one provider-agnostic durable sender attribution contract for user turns, persist it with transcript messages, and serialize it consistently through replay, compaction, and provider adapters without polluting BodyForAgent.

## Reproduction Assessment

Yes, at source level. Current main stores and serializes historical user turns without durable sender metadata while stripping current-turn inbound Sender/Conversation blocks before historical replay; no live multi-user channel repro was run.

## Solution Assessment

Unclear without maintainer product/API input. Per-message sender metadata is the right direction, but the maintainable fix is a durable transcript/provider contract rather than an inline body prefix or channel-only workaround.
