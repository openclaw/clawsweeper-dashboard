## Summary

Keep open: current main still keeps group sender attribution in prompt-local/current-turn metadata while the durable user transcript entry remains plain body text, so the reported continuation-skip attribution gap remains source-reproducible. The linked Slack PR is open, protected, and Slack-specific rather than a merged generic message-array attribution fix.

## What This Changes

The issue requests durable per-message sender attribution for group-chat user turns so attribution survives the rolling LLM context window under continuation-skip.

## Best Possible Solution

Define a generic persisted sender-attribution contract for user transcript messages, map it safely at provider boundaries, and add continuation-skip group-chat regression coverage.

## Reproduction Assessment

Yes, by source inspection. SenderName and InboundHistory reach current-turn prompt context, while transcriptPrompt/transcriptBody remain clean user text and continuation-skip can rely on prior transcript entries without re-injecting prior metadata blocks.

## Solution Assessment

Unclear. Message-array-level attribution is the right layer for durable sender identity, but the transcript schema, provider mapping, and untrusted-sender semantics need maintainer design before implementation.
