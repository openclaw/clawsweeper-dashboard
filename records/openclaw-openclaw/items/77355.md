## Summary

Keep open: current main attributes group senders through prompt-local untrusted context, but the persisted transcript and provider replay path still carry prior user turns as content without durable sender metadata.

## What This Changes

The issue requests durable sender attribution on persisted LLM message entries for group chats so attribution survives continuation-skip rolling context replay.

## Best Possible Solution

Add a generic untrusted sender-attribution field to persisted user transcript messages and map it at provider boundaries with regression coverage for continuation-skip group chats.

## Reproduction Assessment

Yes, by source inspection rather than a live run: group sender fields reach the current prompt, but normal-turn `transcriptPrompt` stores the clean body and hidden runtime-context entries are stripped before LLM replay.

## Solution Assessment

Unclear: message-array-level attribution is the right layer, but maintainers should choose a durable transcript schema and provider mapping before implementing OpenAI `name` or provider-specific prefixes.
