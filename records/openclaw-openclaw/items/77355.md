## Summary

Keep open: current main still provides group sender data as prompt-local metadata, but the persisted transcript/user message text remains the clean body, so continuation-skip can leave earlier group turns anonymous in the rolling LLM context.

## What This Changes

Request to persist group-chat sender attribution on each LLM/transcript user message entry so prior turns remain attributable under continuation-skip.

## Best Possible Solution

Define a generic persisted sender-attribution field for user transcript messages, map it safely at provider boundaries, and cover continuation-skip group replay with regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: group SenderName reaches the current prompt metadata, but persisted transcript/user message content remains the clean body and continuation-skip later relies on that rolling transcript.

## Solution Assessment

Unclear. Message-array-level attribution is likely the right direction compared with inline body pollution, but maintainers should first choose the durable transcript schema and provider-specific mapping rules.
