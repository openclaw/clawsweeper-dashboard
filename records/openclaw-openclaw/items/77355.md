## Summary

Keep open: current main still separates durable transcript text from prompt-local sender metadata, so group sender attribution is available for the current turn but is not persisted on prior user message entries for continuation-skip replay.

## What This Changes

The issue requests durable sender attribution on group-chat transcript or LLM message entries so prior turns remain attributable when continuation-skip relies on the rolling context window.

## Best Possible Solution

Define a generic persisted sender-attribution field for user transcript messages, map it safely at the LLM boundary per provider, and add continuation-skip group-chat regression coverage.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: group SenderName reaches the current prompt, but the persisted transcript prompt remains clean user text and hidden runtime-context custom messages are stripped from LLM replay. I did not run a live group-channel reproduction.

## Solution Assessment

Unclear: message-array-level attribution is a better direction than inline BodyForAgent prefixes, but maintainers should first choose the durable transcript schema and provider-specific mapping rules.
