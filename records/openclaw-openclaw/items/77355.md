## Summary

Keep open: current main still keeps sender attribution in prompt-local/current-turn metadata while the durable transcript/user message remains plain body text, so the reported continuation-skip attribution gap is still source-reproducible. The related Slack PR is open and Slack-specific, not a merged generic message-array attribution fix.

## What This Changes

Request to persist group-chat sender attribution with LLM message-array entries so prior group turns remain attributable under continuation-skip.

## Best Possible Solution

Define a generic persisted sender-attribution field for user transcript messages, map it safely at provider boundaries, and add continuation-skip group-chat regression coverage.

## Reproduction Assessment

Yes, by source inspection. SenderName and InboundHistory are available for current-turn context, but transcriptPrompt remains the clean body and continuation-skip can rely on prior transcript entries without reinjecting prior metadata blocks.

## Solution Assessment

Unclear. Message-array-level attribution is the right layer for durable sender identity, but the durable schema, provider conversion, and untrusted-sender semantics need maintainer design before implementation.
