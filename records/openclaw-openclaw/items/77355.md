## Summary

Keep open: current main and the latest release still have current-turn sender metadata, but historical/replayed user messages do not carry a durable provider-bound sender attribution contract.

## What This Changes

Request durable sender attribution on group-chat user turns so model history can distinguish speakers after continuation-window replay.

## Best Possible Solution

Define one additive provider-agnostic sender attribution contract for user turns, persist it safely, and serialize it consistently through replay, compaction, and provider adapters without inline body pollution.

## Reproduction Assessment

Yes, at source level: current code carries sender metadata in active-turn context, but the normalized model user-message type and historical replay boundary have no durable sender field and strip historical inbound Sender metadata. A live multi-user continuation-skip run was not performed in this read-only review.

## Solution Assessment

Partly: per-message sender metadata is the right class of solution, but the permanent field shape, trust boundary, and provider mapping need maintainer approval; inline body prefixes and channel-only workarounds are not the best fix.
