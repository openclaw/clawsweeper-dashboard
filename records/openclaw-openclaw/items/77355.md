## Summary

Keep open: current main attributes the current group-chat turn through untrusted prompt metadata, but the persisted session message remains the clean prompt body and runtime-context custom messages are stripped before provider replay, so durable per-message sender attribution is still missing.

## What This Changes

Request durable per-message sender attribution for group-chat LLM history so prior turns remain attributed under continuation-skip.

## Best Possible Solution

Define a generic persisted sender-attribution field for user transcript messages, map it safely at the provider boundary, and add group-session continuation-skip regression coverage.

## Reproduction Assessment

Yes, from source inspection rather than live execution: group sender data reaches current-turn prompt metadata, while the session transcript prompt remains the clean body and runtime-context custom messages are stripped before LLM replay.

## Solution Assessment

Unclear: messages-array-level attribution is the right direction, but maintainers should choose the durable transcript schema and provider mapping before implementing it; inlining `BodyForAgent` would conflict with the clean-body contract.
