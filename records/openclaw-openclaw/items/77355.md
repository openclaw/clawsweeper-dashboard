## Summary

Keep open: current main still carries sender attribution as prompt-local untrusted context for the current turn, while the durable transcript prompt remains the clean user body, so continuation-skip can replay older group turns without sender identity.

## What This Changes

Persist group-chat sender attribution with each LLM user-message entry so rolling transcript context remains attributable across continuation turns.

## Best Possible Solution

Define a generic durable sender-attribution field for user transcript messages, map it safely at the provider boundary, and add continuation-skip group-chat regression coverage.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: channel senders reach prompt-local metadata, but transcriptPrompt persists the clean body and continuation-skip relies on that rolling transcript. I did not run a live group-channel scenario.

## Solution Assessment

Unclear: message-array-level attribution is a better direction than inline BodyForAgent prefixes, but maintainers should first choose the durable transcript schema and provider-specific mapping rules.
