## Summary

Keep open: current main still keeps group sender identity in prompt-local metadata and channel payload fields, but prior transcript user messages remain clean body text under continuation-skip, so durable per-message sender attribution is still missing.

## What This Changes

Request durable sender attribution on group-chat LLM message entries so sender identity survives continuation-skip rolling-context replay.

## Best Possible Solution

Define a generic persisted sender-attribution field for user transcript messages, carry it through replay/compaction, map it safely per provider, and add continuation-skip group-chat regression coverage.

## Reproduction Assessment

Yes. Source inspection shows group SenderName and InboundHistory.sender reach current-turn untrusted context, while the stored transcript prompt remains baseBodyFinal and continuation-skip relies on rolling user messages; I did not run a live group-channel scenario.

## Solution Assessment

Unclear. Message-array-level attribution is a better direction than inline BodyForAgent prefixes, but maintainers should first choose the durable transcript schema, provider mappings, and untrusted-sender semantics.
