## Summary

Keep open: current main still treats group sender attribution as prompt-local untrusted context rather than durable metadata on persisted user transcript messages, so prior turns can remain anonymous when continuation-skip relies on the rolling context window.

## What This Changes

Request durable group sender attribution on persisted LLM user-message entries so prior turns keep sender identity in continuation-skip sessions.

## Best Possible Solution

Persist sanitized sender attribution as a generic user-message metadata field and map it at the provider boundary, with continuation-skip group-chat regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: group SenderName and InboundHistory.sender reach current-turn untrusted context, while transcriptCommandBody persists the clean BodyForAgent/baseBodyFinal and continuation-skip relies on the rolling transcript.

## Solution Assessment

Unclear. Message-array-level attribution is a better direction than inline BodyForAgent prefixes, but maintainers should first choose the durable transcript schema, provider-specific mapping, and untrusted sender handling.
