## Label Justifications

- `impact:session-state`: The issue is about sender identity being lost from durable conversation state when historical user turns are replayed to the model.

## Summary

Keep open: current main still treats sender attribution as current-turn metadata and strips that metadata from historical user turns before model replay, while the persisted user-message type has no sender/name field to carry attribution through continuation-skip history.

## What This Changes

Group chat user turns should retain sender attribution in durable model message history so rolling context remains attributable under continuation-skip.

## Best Possible Solution

Define a provider-agnostic sender-attribution field for durable user turns, preserve it through transcript replay/compaction, and serialize it per provider without polluting BodyForAgent.

## Reproduction Assessment

No live high-confidence reproduction was run. Source inspection gives a clear current-main path: historical user turns have inbound Sender metadata stripped before model replay and the persisted UserMessage shape has no sender/name field.

## Solution Assessment

Unclear without maintainer design input. The per-message metadata direction is cleaner than inline BodyForAgent prefixes, but it requires a durable transcript/provider contract rather than a narrow channel-only patch.
