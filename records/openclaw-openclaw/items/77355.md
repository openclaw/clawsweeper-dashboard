## Summary

Keep open: current main still keeps group sender attribution in prompt-local/current-turn metadata while the durable transcript prompt remains plain text, so prior turns can remain unattributed under continuation-skip. The related Slack PR is open and Slack-specific, not a merged generic message-array attribution fix.

## What This Changes

Persist group-chat sender attribution on individual LLM transcript/message entries so prior turns remain attributable when continuation-skip relies on the rolling context window.

## Best Possible Solution

Define a generic persisted sender-attribution field for user transcript messages, map it safely at each provider boundary, and add continuation-skip group-chat regression coverage.

## Reproduction Assessment

Yes. Source inspection shows SenderName reaches currentTurnContext, transcriptCommandBody remains the clean user body, and continuation-skip can rely on prior transcript entries without reinjecting those prompt-local metadata blocks; I did not run a live group chat.

## Solution Assessment

Unclear. Message-array-level attribution is the right layer for the reported durable attribution problem, but the exact transcript schema, provider mapping, and untrusted-sender handling need maintainer design before implementation.
