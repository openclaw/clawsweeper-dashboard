## Summary

Keep open: current main still keeps group sender identity in prompt-local untrusted context while the durable transcript user message remains clean body text, so continuation-skip rolling history can lose who said prior turns.

## What This Changes

Request durable per-user sender attribution on persisted LLM message entries for group chats so continuation-skip history retains who said each prior turn.

## Best Possible Solution

Define a generic persisted sender-attribution field for user transcript messages, map it safely at the LLM boundary per provider, and add continuation-skip group-chat regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: group SenderName reaches current-turn prompt metadata, but the persisted transcript prompt remains clean user text and continuation-skip relies on that rolling transcript.

## Solution Assessment

Unclear. Message-array-level attribution is a stronger direction than inline BodyForAgent prefixes, but maintainers should first choose the durable transcript schema, provider mapping, and trust semantics.
