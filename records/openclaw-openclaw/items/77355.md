## Summary

Keep open: current main still keeps sender attribution in prompt-local metadata while persisted user transcript entries remain clean body text, so prior group turns can lose attribution under continuation-skip.

## What This Changes

Requests durable sender attribution on group-chat LLM transcript/message entries so previous turns retain who said them under continuation-skip.

## Best Possible Solution

Add a generic persisted sender-attribution field for user transcript messages, map it safely at the LLM boundary per provider, and add continuation-skip group-chat regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: group `SenderName` reaches current-turn metadata, but persisted transcript prompts remain clean user text and continuation-skip relies on that rolling transcript.

## Solution Assessment

Unclear. Message-array-level attribution is a better direction than inline `BodyForAgent` prefixes, but maintainers should first choose the durable transcript schema and provider-specific mapping rules.
