## Summary

Keep open: current main handles sender attribution for the current turn through untrusted context blocks, but the transcript/replay path still stores the clean user prompt and strips OpenClaw runtime-context custom messages before LLM replay, so durable per-message group sender attribution is not implemented.

## What This Changes

Add durable sender attribution to group-chat LLM message entries so prior turns keep participant identity across rolling context and continuation-skip turns.

## Best Possible Solution

Define a generic persisted sender-attribution field for user transcript messages, map it safely at the LLM boundary per provider, and add group-session regression coverage for continuation-skip replay.

## Reproduction Assessment

No live reproduction was run, but source inspection gives a clear path: group SenderName is available for the current prompt, while the persisted transcript prompt is clean and OpenClaw runtime-context custom messages are stripped from LLM replay.

## Solution Assessment

Unclear: message-array-level metadata is the right direction, but maintainers should first choose the durable session-message schema and provider mapping rather than patching individual channel BodyForAgent strings.
