## Summary

Keep open: current main still leaves Docker sandbox containers without a default PID cap, and open PR #44098 references this issue with closing syntax but has not merged.

## What This Changes

Add a safe default Docker sandbox `pidsLimit` so sandbox containers are not left without a process cap when the option is omitted.

## Best Possible Solution

Land PR #44098 or a narrow replacement that adds an agreed PID cap, preserves explicit overrides, defines opt-out semantics, updates schema/docs/tests/changelog, and coordinates broader resource defaults with #60827.

## Reproduction Assessment

Yes. Source inspection on current main shows an omitted `pidsLimit` resolves to `undefined`, and Docker create args omit `--pids-limit` unless the value is positive.

## Solution Assessment

Unclear as submitted. The resolver fallback in #44098 is the narrow implementation boundary, but the maintainable fix also needs approved opt-out semantics plus schema, docs, tests, and changelog updates.
