## Summary

Keep open. Current main and the latest shipped tag still leave Docker sandbox pidsLimit unset unless configured, and an open PR directly references this issue with the proposed default-limit fix.

## What This Changes

The issue asks OpenClaw to add a default Docker sandbox pidsLimit so sandbox containers are not created without an OpenClaw-managed process-count cap.

## Best Possible Solution

Land PR #44098 or an equivalent secops-reviewed change that adds the resolver default, preserves explicit overrides, proves Docker create args include the resolved default, and updates docs if the runtime default changes.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: resolve Docker sandbox config without global or per-agent pidsLimit, then build Docker create args; current main leaves the value undefined and emits no --pids-limit flag.

## Solution Assessment

Yes, pending secops approval of the default value. Adding the fallback in resolveSandboxDockerConfig is the narrowest maintainable fix; the WASM-runtime idea raised in discussion is broader architecture work and does not replace hardening the current Docker backend.
