## Summary

Keep open. Current main still resolves Docker sandbox pidsLimit only from per-agent or global config, Docker create args only emit --pids-limit when that resolved value is positive, and the latest release has the same behavior. The issue is also paired with open, unmerged PR #44098, so it should remain open until that PR or an equivalent security-approved fix lands.

## What This Changes

The issue requests a default Docker sandbox pidsLimit and clearer sandbox/security docs so containers are not left without a process-count limit against fork-bomb style exhaustion.

## Best Possible Solution

Add a maintainer-approved default PID limit in the sandbox Docker config resolver, keep explicit global and per-agent overrides taking precedence, add resolver/create-args coverage, and update the public sandbox/config/security docs to describe the default and how to override or recreate containers.
