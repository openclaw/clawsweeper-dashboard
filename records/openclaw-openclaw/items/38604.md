## Summary

Keep open: current main still leaves Docker sandbox containers without a default PID cap, and open PR #44098 is the canonical linked implementation candidate that should be reviewed or landed before this issue closes.

## What This Changes

The issue requests a default Docker sandbox pidsLimit plus documentation guidance so sandbox containers are not left without a process cap.

## Best Possible Solution

A security-reviewed PR should set a safe default in the sandbox Docker resolver, preserve explicit overrides, add regression coverage for unset/global/agent cases, and align docs/JSDoc with the shipped default.

## Reproduction Assessment

Yes. Source inspection on current main shows the unset path returns undefined for pidsLimit and buildSandboxCreateArgs then omits --pids-limit; I did not run a live Docker fork-bomb test.

## Solution Assessment

Yes for the core direction. Adding the default in resolveSandboxDockerConfig while preserving explicit overrides is the narrowest maintainable fix, with final default value and docs coverage reviewed on the linked PR.
