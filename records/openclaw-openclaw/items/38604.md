## Summary

Keep open: current main still leaves Docker sandbox containers without a default PID cap, and https://github.com/openclaw/openclaw/pull/44098 references this issue with closing syntax but has not merged.

## What This Changes

Add a default PID cap for Docker sandbox containers so omitted `pidsLimit` settings do not leave sandboxed tool execution vulnerable to process exhaustion.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/44098 or a narrow replacement after approving the PID default and opt-out contract, with schema, docs, tests, and changelog follow-through.

## Reproduction Assessment

Yes. Source inspection on current main shows an omitted `pidsLimit` resolves to `undefined`, and Docker create args omit `--pids-limit` unless the value is positive; Docker documents PID limiting as an explicit runtime flag. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/container/run?utm_source=openai))

## Solution Assessment

Unclear as currently tracked. The resolver fallback is the narrow implementation point, but the default value and opt-out semantics need maintainer/security approval plus public contract updates before merge.
