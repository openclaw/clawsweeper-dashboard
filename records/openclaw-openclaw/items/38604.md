## Summary

Keep open: current main and the latest tagged release still leave Docker sandbox containers without a default PID cap, and the open implementation PR that references this issue has not merged.

## What This Changes

The issue asks OpenClaw to add a default Docker sandbox PID limit so unconfigured sandbox containers are not vulnerable to process exhaustion.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/44098 or a narrow replacement that adds an approved PID cap, preserves explicit overrides, defines opt-out semantics, and updates schema, docs, tests, and changelog.

## Reproduction Assessment

Yes. A safe source-level reproduction is to resolve Docker sandbox config with no global or per-agent `pidsLimit` and observe that the value remains `undefined`, then follow `buildSandboxCreateArgs` and see that `--pids-limit` is omitted.

## Solution Assessment

Unclear as submitted. Adding the fallback in the Docker sandbox resolver is the narrow implementation point, but the default value and explicit opt-out contract need maintainer approval plus schema/docs/test follow-through.
