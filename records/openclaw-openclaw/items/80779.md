The PR changes `devices approve` local fallback to throw an actionable stale requestId diagnostic with pending-entry counts and `openclaw devices list` routing, plus focused CLI regression tests.

## Best Possible Solution

## Reproduction Assessment

## Solution Assessment

Yes. Improving the CLI fallback diagnostic is the narrowest maintainable fix; changing reconciliation ids or auto-approving an alternate pending id would alter approval semantics and carry more operator-safety risk.
