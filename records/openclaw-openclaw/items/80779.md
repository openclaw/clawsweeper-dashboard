## Summary

## What This Changes

The PR changes the devices CLI local approve fallback to throw an actionable stale requestId diagnostic that routes operators to `openclaw devices list`, and adds regressions for zero, one, and multiple visible pending requests.

## Best Possible Solution

Merge the CLI-layer diagnostic after standard maintainer validation, preserving exact requestId approval semantics and operator inspection through `openclaw devices list`.

## Reproduction Assessment

Yes, at source level. On current main, a bare pairing-required gateway error plus a missing local pending id reaches the `return null` branch and the caller prints only `unknown requestId`.

## Solution Assessment

Yes. Improving the CLI fallback diagnostic is the narrowest maintainable fix; it avoids changing reconciliation semantics or auto-approving a different requestId.
