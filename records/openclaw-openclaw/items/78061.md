## What This Changes

Slack thread sessions are reported to generate assistant replies that appear in the transcript but are not delivered back to the originating Slack thread.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/78536 or an equivalent narrow Slack native-stream pending-text fallback, then close this issue after the fix reaches main.

## Reproduction Assessment

Yes at the source level. A short ChatStreamer-buffered Slack reply followed by an unexpected stopSlackStream failure can leave pendingText outside the threaded deliverReplies fallback path on current main.

## Solution Assessment

Yes. The linked PR direction is the narrowest maintainable fix because it reuses the existing fallback delivery seam and preserves the original Slack thread timestamp without changing routing, config, or streaming defaults.
