## Best Possible Solution

Review and land one canonical MS Teams PR that carries per-call placement metadata through the established outbound/send-context path while preserving omitted-parameter behavior.

## Reproduction Assessment

## Solution Assessment

Yes, the per-call override direction is the narrow maintainable solution because it avoids flipping static `replyStyle` for all sends. The implementation should use the existing Teams outbound/send-context path rather than a partial side path.
