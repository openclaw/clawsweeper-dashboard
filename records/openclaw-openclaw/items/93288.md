## Summary

Keep open: current main and the latest release still lack a Microsoft Teams per-call `topLevel` override for proactive channel sends, and the same-author open PR at https://github.com/openclaw/openclaw/pull/93292 is now the active implementation candidate. This issue should stay open until that PR or a replacement lands, or maintainers close the implementation path.

## What This Changes

Request adds a per-call `topLevel` override to Microsoft Teams `message(action="send")` so proactive channel announcements can post as new root posts without changing static `replyStyle` config.

## Best Possible Solution

Review, repair, and land the linked PR or a narrow replacement that routes existing `topLevel` message-tool semantics through the MSTeams send context for all supported send payloads while preserving configured `replyStyle` when omitted.

## Reproduction Assessment

Yes, at source level: current Teams sends resolve proactive placement from static `replyStyle` config only, while shared message-action code already recognizes `topLevel` as thread suppression. I did not run a live Teams tenant send in this read-only review.

## Solution Assessment

Yes for the requested direction: reusing existing per-call `topLevel` semantics is narrower than changing static Teams config. The current open PR still needs repair/proof before it is the best mergeable implementation.
