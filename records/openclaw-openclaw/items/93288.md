## What This Changes

The issue requests a per-call Microsoft Teams `topLevel` send-action override so proactive announcements can create a new root channel post while normal thread replies keep using static `replyStyle` config.

## Best Possible Solution

Review and land a focused Teams plugin PR that implements the shared `topLevel` send-action semantics without changing configured `replyStyle` defaults.

## Reproduction Assessment

Yes. Source inspection on current main shows the Teams message tool has no `topLevel` send schema and the Teams send context has no per-call reply-style override, matching the report.

## Solution Assessment

Yes. The requested shape is the narrow maintainable path because it reuses existing shared `topLevel` message-action semantics inside the Teams plugin rather than adding config or changing static defaults.
