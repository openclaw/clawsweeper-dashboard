Keep open: current main and v2026.6.6 still lack the requested Microsoft Teams per-call top-level send override, and the same reporter has an open implementation PR that should own the fix until it is repaired and merged.

## What This Changes

The issue requests a per-call `topLevel` send option for Microsoft Teams so proactive channel sends can create a new root thread while normal thread replies continue using static `replyStyle: "thread"`.

## Best Possible Solution

Land a repaired Microsoft Teams plugin change that carries existing `topLevel` send intent through the normal outbound/send context path without changing static `replyStyle` defaults.

## Reproduction Assessment

Yes, at source level: current main and v2026.6.6 resolve Teams proactive send placement from static `replyStyle` only, while shared message-action semantics already recognize `topLevel`. I did not run a live Teams tenant send in this read-only review.

## Solution Assessment

Yes in direction, but not by closing this issue now: the maintainable solution is a Teams plugin-local per-call override that reuses existing `topLevel` semantics and preserves normal send payload handling.
