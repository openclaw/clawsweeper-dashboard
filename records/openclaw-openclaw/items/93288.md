## Summary

Keep open: current `main` and the latest release still lack the Microsoft Teams per-call `topLevel` send override, and open linked PRs already own the implementation path.

## What This Changes

The issue requests a per-call Microsoft Teams `topLevel` override for the message `send` action so proactive channel sends can create a root post without changing static `replyStyle` configuration.

## Best Possible Solution

Land one narrow Teams plugin change that exposes send-scoped `topLevel` and carries placement intent through the established Teams outbound/send-context path while preserving omitted-parameter behavior.

## Reproduction Assessment

Yes, source-level: current `main` exposes no Teams `topLevel` send parameter and derives proactive placement only from static `replyStyle` config. I did not run a live Teams tenant send in this read-only review.

## Solution Assessment

Yes. The per-call override is the narrow maintainable direction because static `replyStyle` cannot express both normal in-thread replies and proactive root posts in the same channel; the fix should use the existing Teams outbound/send-context path rather than add another config knob.
