## What This Changes

Request to add a Microsoft Teams `send` action `topLevel` override so proactive channel announcements can create a new root post without changing static `replyStyle` config.

## Best Possible Solution

Review and land one narrow Teams PR that exposes `topLevel` for `send` and carries that intent through the Teams outbound/send-context path while preserving omitted-parameter behavior.

## Reproduction Assessment

Yes, source-level: current main exposes no Teams `topLevel` send parameter and derives proactive placement only from static `replyStyle` config. I did not run a live Teams tenant send in this read-only review.

## Solution Assessment

Yes, the per-call override is the narrow maintainable direction because static `replyStyle` cannot express both in-thread replies and proactive root posts in the same channel. The final implementation should reuse established outbound/send-context paths rather than add another config knob.
