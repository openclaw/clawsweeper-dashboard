## Summary

Keep open: current main and the latest release still lack a Microsoft Teams per-call `topLevel` override, and an open fixing PR now explicitly references this issue. The issue should stay open until a candidate lands or maintainers close the linked implementation path.

## What This Changes

Request Microsoft Teams `message` send calls support a per-call `topLevel` override so proactive channel announcements can create root posts while normal thread replies keep using configured `replyStyle`.

## Best Possible Solution

Land a narrow fix that carries `topLevel` intent through the existing Teams outbound/send-context path, preserves default `replyStyle` behavior when omitted, and then close this issue from the merged PR.

## Reproduction Assessment

Yes, at source level: current main and v2026.6.8 expose no Teams `topLevel` send schema and resolve proactive placement only from static `replyStyle`. I did not run a live Teams tenant send in this read-only review.

## Solution Assessment

Yes on direction: reusing existing `topLevel` message-action semantics is the narrowest maintainable fix. The implementation should use the existing outbound/send-context path, not a separate send branch that bypasses normal Teams delivery behavior.
