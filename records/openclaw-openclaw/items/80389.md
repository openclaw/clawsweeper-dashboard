## Summary

Keep open. Current main still sends Telegram video media through `sendVideo` and probes video dimensions before the document fallback even when `forceDocument` is true, and an open closing PR is already the active fix path.

## What This Changes

The issue asks for Telegram MP4 media sends to honor `--force-document` by using `sendDocument`, with possible fallback and failure exit-code handling.

## Best Possible Solution

Review and land the focused PR that makes forced-document Telegram videos skip video-specific probing/routing and use `sendDocument`; split fallback and exit-code follow-ups if they remain reproducible afterward.

## Reproduction Assessment

Yes for the routing bug: source inspection on current main shows a `forceDocument: true` video still selects video-specific probing and `sendVideo`. I did not perform live Telegram delivery in this read-only review.

## Solution Assessment

Yes for the explicit override: adding `!forceDocument` guards around video-note selection, video probing, and the `sendVideo` branch is the narrow maintainable fix. Automatic fallback from `sendVideo` failure should be considered separately because ambiguous network failures can risk duplicate sends.
