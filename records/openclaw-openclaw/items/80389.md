## What This Changes

The issue reports that Telegram `.mp4` media sends ignore `--force-document`, still route through `sendVideo`, and can fail without a non-zero CLI exit.

## Best Possible Solution

Land one focused PR that makes explicit forced-document video sends use Telegram `sendDocument`, then track fallback and exit-code behavior separately if still reproducible.

## Reproduction Assessment

Yes for the routing defect by source inspection: with loaded `video/mp4` media and `forceDocument: true`, current main still enters the video branch before `sendDocument`. I did not live-reproduce the Telegram upload failure or exit-code symptom.

## Solution Assessment

Yes for the linked fix direction: adding a force-document guard around video probing and `sendVideo` routing is the narrow maintainable fix for the explicit override. The issue should not be closed until the linked PR lands or maintainers decide how to split any remaining fallback and exit-code work.
