- **[P1] Accept group-prefixed Telegram targets:** `ui/src/ui/controllers/cron.ts:130-132`
  - body: The regex allows `telegram:` / `tg:` prefixes but has no `group:` branch after them, while the Telegram runtime still supports `telegram:group:<target>` / `tg:group:<target>` and tests cover that normalizer path. Existing cron jobs using those runtime-supported targets could be blocked when saved or edited.
  - confidence: 0.93

  - body: The topic suffix branch accepts any digit string, but runtime only treats safe non-negative integers as topic IDs via `parseStrictNonNegativeInteger`; oversized suffixes become part of the chat target and fail later. Mirror the runtime parser so save-time validation does not accept targets Telegram delivery cannot route.
  - confidence: 0.88
