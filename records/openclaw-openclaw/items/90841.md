Full review comments:

- **[P1] Preserve group-prefixed Telegram targets:** `ui/src/ui/controllers/cron.ts:131`
  - body: The regex rejects `telegram:group:-100123` and `tg:group:-100123`, but the current Telegram normalizer keeps those legacy/internal group-prefixed forms and send-time parsing strips them before delivery. Existing cron jobs using that supported form would be blocked when edited or re-saved.
  - confidence: 0.9

  - body: The new `:\d+` and `:topic:\d+` suffix check accepts arbitrarily large topic IDs, while runtime only treats safe non-negative integers as thread IDs. Values that the UI saves can still fall through to a Telegram runtime failure, so mirror the parser's safe-integer rule here and cover it in tests.
  - confidence: 0.82
