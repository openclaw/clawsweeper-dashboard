## Summary

Keep open. Current main only partially addresses the report with health-monitor and heartbeat start logs; the Gateway startup summary itself is still compact and does not include the requested auth/source, full server URL, or structured dashboard format.

## What This Changes

Restore or make configurable a structured Gateway startup log summary with auth, server URL, agent model, plugin list, log file, and subsystem status fields.

## Best Possible Solution

Add a deterministic, redacted Gateway startup summary in the existing Gateway startup logging path, using the current logging subsystem and adding a format toggle only if maintainers want to keep both compact and structured modes.

## Reproduction Assessment

Yes. Source inspection on current main shows the startup helper and its focused test still assert compact output, while the requested auth/source and full server URL fields are absent from the startup summary.

## Solution Assessment

Unclear as filed. Restoring the exact old emoji banner may not be the best long-term style, but a structured, redacted summary in `src/gateway/server-startup-log.ts` is the narrow maintainable direction.
