## Summary

Keep open. Current main still emits the compact Gateway startup log and does not provide the requested auth, server URL, health-monitor, or heartbeat startup summary; this is a current, narrow Gateway logging gap rather than a stale or already-implemented request.

## What This Changes

The issue requests restoring a structured Gateway startup log that shows auth mode, server URL, agent model, loaded plugins, log file path, and health/heartbeat startup status.

## Best Possible Solution

Add a redacted, deterministic Gateway startup summary through the existing startup/post-attach/scheduled-service path, covering auth mode/source, bind URL, model, plugins, log file, and service activation status without introducing a new config option unless maintainers ask for dual formats.
