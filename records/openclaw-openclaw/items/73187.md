## Summary

Keep open. Current main still emits the compact Gateway startup log shape and has no structured auth/server/model/plugin/log-file/service-status startup snapshot, so the requested operator visibility remains missing.

## What This Changes

The issue requests restoring a structured Gateway startup log summary with auth mode/source, server URL, agent model, loaded plugins, log file, and health-monitor/heartbeat status.

## Best Possible Solution

Add a redacted, deterministic Gateway startup summary that gives operators auth, server, model, plugin, log-file, and service-start state without exposing secrets or requiring config-file inspection.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: current main's helper and tests assert compact output, while the post-attach and scheduled-service paths do not emit the requested structured status lines.

## Solution Assessment

Yes. The narrowest maintainable solution is to extend the Gateway-owned startup logging and activation paths with focused tests; a broad logging subsystem rewrite or new config key is not needed unless maintainers explicitly want dual formats.
