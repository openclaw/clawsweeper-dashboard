## Summary

Keep open: current main and v2026.6.9 still drop the reported WhatsApp DM/group access-control denials before dispatch with only verbose-gated diagnostics, and the remaining fix needs maintainer product/security direction on redaction, rate limiting, and WhatsApp-owned logging versus a shared unauthorized-inbound audit surface.

## What This Changes

Request a default-visible, redacted/rate-limited diagnostic for WhatsApp inbound DM/group access-control denials that are currently only logged when verbose mode is enabled.

## Best Possible Solution

Define a redacted, rate-limited WhatsApp denial diagnostic that reuses shared ingress reason metadata and preserves existing fail-closed authorization behavior, or deliberately route this into a shared unauthorized-inbound audit/status surface.

## Reproduction Assessment

Yes. Source inspection on current main shows WhatsApp `dmPolicy` or `groupPolicy` denial paths return before dispatch and only log through `logWhatsAppVerbose` when `verbose` is true; no live WhatsApp account was exercised in this read-only review.

## Solution Assessment

Unclear as a final implementation. The visibility goal is valid, but maintainers should choose between a WhatsApp-owned rate-limited info diagnostic and a structured/shared unauthorized-inbound audit surface with explicit redaction rules.
