## Summary

Keep open: current main and v2026.6.8 still gate WhatsApp inbound sender/policy denial diagnostics behind verbose logging, so default-config access-control drops remain silent. The report is source-reproducible, but the durable fix needs maintainer/security direction on redaction, rate limiting, and whether this should be a WhatsApp-local log or a shared unauthorized-inbound audit/status signal.

## What This Changes

Request a default-visible, rate-limited diagnostic for WhatsApp inbound access-control denials without changing authorization decisions.

## Best Possible Solution

Define a privacy-safe, rate-limited unauthorized-inbound diagnostic contract that reuses shared ingress reason metadata where practical, then add WhatsApp coverage while preserving the existing access decisions.

## Reproduction Assessment

Yes, from source. With WhatsApp verbose off, group or DM access-control denials call logWhatsAppVerbose and then return blocked access, and the monitor drops the message before dispatch.

## Solution Assessment

Unclear as a final implementation. The observability goal is sound, but maintainers should choose between a WhatsApp-owned rate-limited diagnostic and a structured/shared unauthorized-inbound audit event with explicit redaction rules.
