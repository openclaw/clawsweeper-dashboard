## Summary

Keep open: current main and v2026.6.9 still drop WhatsApp access-control denials before dispatch with only verbose-gated denial diagnostics. ## What This Changes

The issue requests default-visible, rate-limited observability for WhatsApp inbound messages denied by allowlist or DM/group policy checks.

## Best Possible Solution

Define a redacted, rate-limited WhatsApp denial diagnostic that reuses shared ingress reason metadata and preserves existing fail-closed authorization behavior, or deliberately route the work into a shared unauthorized-inbound audit/status surface.

Yes, source-reproducible for the current behavior: denied WhatsApp inbound messages return from checkInboundAccessControl before dispatch, and the denial diagnostics are gated on verbose. I did not run a live WhatsApp account in this read-only review.

## Solution Assessment

Unclear as a final implementation: the visibility goal is sound, but maintainers should choose between a WhatsApp-owned rate-limited diagnostic and a structured/shared unauthorized-inbound audit event with explicit redaction rules.
