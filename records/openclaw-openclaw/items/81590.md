- **[P1] Read protocol constants from gateway-protocol:** `scripts/gateway_health_watchdog.py:155-159`
  - body: Current main no longer has `src/gateway/protocol/version.ts`; the constants live under `packages/gateway-protocol`. Because this helper returns `None` when the old path is absent, `probe_ws_upgrade` reports `unable to resolve gateway protocol version from repo` before it can perform the advertised authenticated handshake after merge.
  - confidence: 0.94
- **[P1] Reuse the supported TLS-aware gateway probe:** `scripts/gateway_health_watchdog.py:495-499`
  - body: This raw socket path bypasses the existing `buildGatewayProbeConnectionDetails` and `probeGateway` behavior for `wss://`, TLS fingerprints, device identity, preauth timing, and probe mode. Secure local setups that the supported client can reach can therefore be marked unhealthy by the watchdog.
  - confidence: 0.9
- **[P1] Preserve shared SecretInput exec controls:** `scripts/gateway_health_watchdog.py:358-364`
  - body: The watchdog runs `SecretInput.exec` commands directly instead of going through the shared resolver, so it misses the secure-path check, no-output timeout, max-output/request-size bounds, and process cleanup policy. That creates a second credential execution boundary with weaker controls.
  - confidence: 0.88
- **[P1] Restart through the managed gateway lifecycle:** `scripts/gateway_health_watchdog.py:653-661`
  - body: When kickstart is enabled, this calls `launchctl kickstart -k` directly instead of the safe restart coordinator and restart-intent flow. That bypasses current preflights for pending replies, embedded runs, cron runs, and active tasks, so a watchdog tick can disrupt live work.
  - confidence: 0.86
- **[P2] Store watchdog state in the canonical state store:** `scripts/gateway_health_watchdog.py:66-69`
  - body: The script writes OpenClaw-owned runtime state to `gateway-watchdog-state.json` under the state directory. Repository policy and current runtime paths keep OpenClaw-owned state in SQLite, so this sidecar would add another persistence path with unclear migration and cleanup behavior.
  - confidence: 0.82
