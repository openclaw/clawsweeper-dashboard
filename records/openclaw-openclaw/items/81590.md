## Summary

Keep open: this is a plausible gateway-ops proposal, but the patch is not mergeable because the watchdog duplicates the gateway handshake with stale hardcoded protocol, auth, and port assumptions that can falsely mark a healthy gateway unhealthy.

## What This Changes

The PR adds a standalone Python gateway health watchdog that performs an authenticated WebSocket connect probe and can kickstart the local gateway, plus a one-line live model test return-type annotation.

## Best Possible Solution

Route any watchdog through the maintained TypeScript GatewayClient/probe/daemon health path, with coverage for protocol constants, token/password/SecretInput auth, and configured ports before adding restart policy.

## Reproduction Assessment

Yes. Source inspection is enough: the PR sends minProtocol/maxProtocol 3 while current main requires protocol 4, so a current gateway rejects the connect probe before reporting healthy.

## Solution Assessment

No. A standalone Python handshake duplicates established TypeScript probe/client contracts; the safer direction is to reuse or extend src/gateway/probe.ts and daemon restart health instead of hardcoding protocol, auth, and port behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Use the current gateway protocol version:** `scripts/gateway_health_watchdog.py:275-276`
  - body: This connect probe advertises protocol 3, but current main exports protocol 4 and the server rejects clients whose protocol range does not cover it. As written, the watchdog can mark a healthy gateway failed and kickstart it after debounce.
  - confidence: 0.95
- **[P2] Resolve the configured gateway auth mode:** `scripts/gateway_health_watchdog.py:206-215`
  - body: The probe only accepts OPENCLAW_GATEWAY_TOKEN or a raw gateway.auth.token string. Supported local gateways can use password auth, env password auth, or SecretInput-backed credentials, so those healthy configurations are reported as missing a token.
  - confidence: 0.86
- **[P2] Use the configured gateway port:** `scripts/gateway_health_watchdog.py:55-56`
  - body: The script always probes 127.0.0.1:18789 even though OpenClaw resolves the gateway port from CLI/env/config before the default. A healthy gateway on a configured non-default port will be misreported as down.
  - confidence: 0.82
