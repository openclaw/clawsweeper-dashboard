## Summary

Keep open: the refreshed branch fixes the earlier protocol, port, and SecretInput drift and now supplies current-head live proof, but the PR still adds a separate restart watchdog that misses current state/config and TLS contracts and bypasses the managed safe-restart path.

## What This Changes

Adds a Python gateway health watchdog script that performs an authenticated WebSocket connect probe and can kickstart the local launchd gateway, with Python/Vitest contract tests and a live-test return-type tweak.

## Best Possible Solution

Route the watchdog through the maintained gateway probe/auth/path/TLS and safe-restart primitives, or make it an explicitly macOS-only emergency script with owner-approved limits and coverage for profiles, TLS, and active-work deferral.

## Reproduction Assessment

Yes, by source inspection against current main. Setting OPENCLAW_STATE_DIR or enabling gateway.tls makes the PR script follow different path/probe contracts than OpenClaw's maintained status and gateway-call code.

## Solution Assessment

No. The safer maintainable direction is to reuse the existing GatewayClient/status probe and managed safe-restart coordinator instead of hand-mirroring protocol, auth, path, TLS, and launchd behavior in Python.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Honor OPENCLAW_STATE_DIR for watchdog paths:** `scripts/gateway_health_watchdog.py:58-67`
  - body: The script derives OPENCLAW_HOME, logs, state, and the default config path from ~/.openclaw, but current OpenClaw resolves mutable state/config from OPENCLAW_STATE_DIR and profile-aware paths. A managed/profile install can be healthy while the watchdog reads the wrong config/logs and probes with the wrong port or auth.
  - confidence: 0.91
- **[P2] Probe TLS-enabled gateways with wss:** `scripts/gateway_health_watchdog.py:482-497`
  - body: This opens a plain TCP socket and sends a cleartext WebSocket upgrade to 127.0.0.1. Current status/gateway call paths switch local probes to wss:// and load the TLS fingerprint when gateway.tls.enabled; a healthy TLS gateway will fail the watchdog probe twice and be treated as restartable.
  - confidence: 0.89
- **[P2] Restart through the managed safe path:** `scripts/gateway_health_watchdog.py:651-655`
  - body: When the trigger fires, the script calls launchctl kickstart -k directly. Current user-facing restart supports openclaw gateway restart --safe so active queued work, reply delivery, embedded runs, and task runs can defer restart; bypassing that path can interrupt live work.
  - confidence: 0.84
