  - body: The watchdog opens a raw socket and builds its own WebSocket frame/handshake, bypassing `probeGateway` and `GatewayClient` behavior for TLS fingerprints, proxy routing, device identity, token/password handling, and preauth timeouts. That creates a second gateway-health contract that can classify real supported setups differently from OpenClaw itself.
  - confidence: 0.9
- **[P1] Reuse shared SecretInput resolution:** `scripts/gateway_health_watchdog.py:358`
  - body: This exec SecretInput implementation duplicates the shared resolver instead of using the gateway credential path that enforces secure executable paths, output limits, no-output timeouts, cleanup, and controlled environment behavior. A watchdog that resolves secrets differently from the core path can weaken or misreport the same credential configuration it is meant to test.
  - confidence: 0.88
- **[P1] Restart through the managed gateway lifecycle:** `scripts/gateway_health_watchdog.py:653-661`
  - body: Even behind the opt-in env flag, calling `launchctl kickstart -k` directly bypasses the restart coordinator's pending reply, queue, embedded run, cron run, and active-task deferrals. This can interrupt live work or message delivery in exactly the cases the managed restart path is designed to protect.
  - confidence: 0.9
- **[P2] Store watchdog state in the canonical state store:** `scripts/gateway_health_watchdog.py:66-69`
  - body: The PR writes `gateway-watchdog-state.json` under OpenClaw's state area, adding a runtime state sidecar instead of using the shared SQLite state model. If this becomes OpenClaw-owned behavior, the watchdog state needs a clear SQLite owner or must stay outside the runtime state contract.
  - confidence: 0.82
