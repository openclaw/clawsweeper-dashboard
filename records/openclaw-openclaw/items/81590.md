## Summary

Keep open: the PR is active and adds useful authenticated gateway handshake proof, but the branch still duplicates hardened gateway path, TLS, restart, and SecretRef contracts with blocking correctness and security findings.

## What This Changes

This PR adds a Python gateway health watchdog script with an authenticated WebSocket connect probe, Python/Vitest contract tests, and a small live-test return-type tweak.

## Best Possible Solution

Route the watchdog through maintained gateway connection/auth/TLS/path and safe-restart primitives, or keep it out of core as an owner-approved macOS emergency script with explicit limitations.

## Reproduction Assessment

Yes, by source inspection against current main. Setting OPENCLAW_STATE_DIR, enabling gateway.tls, or using a SecretRef file/exec provider that core rejects exercises paths where the watchdog differs from maintained OpenClaw contracts.

## Solution Assessment

No. The safer maintainable direction is to reuse existing gateway connection, SecretRef, TLS, path, and safe-restart contracts rather than hand-mirroring them in a standalone Python watchdog.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Reuse the hardened SecretRef resolver:** `scripts/gateway_health_watchdog.py:325-365`
  - body: The watchdog reimplements SecretRef resolution and directly reads file providers or runs exec providers. Current OpenClaw validates absolute paths, symlinks, permissions, trustedDirs, exec ref ids, timeouts, and output limits before reading or executing; this script can accept configurations the core resolver rejects.
  - confidence: 0.88
- **[P2] Honor OPENCLAW_STATE_DIR for watchdog paths:** `scripts/gateway_health_watchdog.py:58-67`
  - body: The script derives the default log, state, and config paths from OPENCLAW_HOME. Managed/profile installs can put mutable state and openclaw.json under OPENCLAW_STATE_DIR, so the watchdog can read the wrong config/logs and probe with stale auth or port data.
  - confidence: 0.91
- **[P2] Probe TLS-enabled gateways with wss:** `scripts/gateway_health_watchdog.py:493-497`
  - body: The WebSocket probe always opens a cleartext socket and sends a ws upgrade to 127.0.0.1. A healthy gateway with gateway.tls.enabled uses wss locally, so the watchdog will count it as failed and can move toward restart.
  - confidence: 0.89
- **[P2] Restart through the managed safe path:** `scripts/gateway_health_watchdog.py:651-655`
  - body: When a trigger fires, the script calls launchctl kickstart -k directly. That bypasses openclaw gateway restart --safe, which defers or coalesces restarts while queued work, reply delivery, embedded runs, or task runs are active.
  - confidence: 0.84
