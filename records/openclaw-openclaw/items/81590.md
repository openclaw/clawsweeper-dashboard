## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Honor OPENCLAW_STATE_DIR for watchdog paths:** `scripts/gateway_health_watchdog.py:59-68`
  - body: The watchdog derives its default log, state, and config paths from OPENCLAW_HOME, while current OpenClaw supports OPENCLAW_STATE_DIR for mutable state and config lookup. Managed/profile installs can therefore have a healthy gateway while this script reads stale logs or the wrong config/auth/port.
  - confidence: 0.91
- **[P2] Probe TLS-enabled gateways with WSS:** `scripts/gateway_health_watchdog.py:495-499`
  - body: The probe always opens a plain TCP socket and sends a cleartext WebSocket upgrade to 127.0.0.1. Current gateway status switches local probes to wss:// and passes the TLS fingerprint when gateway.tls.enabled is true, so a healthy TLS gateway can be counted as failing and become restartable.
  - confidence: 0.89
- **[P2] Restart through the managed gateway path:** `scripts/gateway_health_watchdog.py:653-655`
  - body: Even after making kickstart opt-in, the enabled restart path still calls launchctl kickstart -k directly. That bypasses the safe restart active-work preflight/deferral and the managed restart recovery path for stale ports and launch-agent repair, so enabling the watchdog can still interrupt active work or miss existing recovery handling.
  - confidence: 0.85
