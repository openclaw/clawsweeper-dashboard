## Summary

Keep open: the PR is a risky new gateway restart watchdog and the proposed patch is currently incorrect against main. Source inspection shows the added Python probe hardcodes stale protocol, auth, and port assumptions, so it can mark a healthy gateway unhealthy.

## What This Changes

The PR adds a Python local gateway health watchdog that performs an authenticated WebSocket connect probe and launchd kickstart, plus a live model test return-type annotation cleanup.

## Best Possible Solution

If this watchdog is wanted, implement it through the maintained GatewayClient, gateway probe, or daemon health path with tests for protocol constants, token/password/SecretInput auth, configured ports, and restart safety.

## Reproduction Assessment

Yes. Source inspection is enough: the PR sends protocol 3 while current main requires protocol 4, so a current gateway will reject the connect probe.

## Solution Assessment

No. Duplicating the gateway handshake in Python with hardcoded protocol, auth, and port values is less maintainable than reusing the existing GatewayClient/probe or daemon health path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Use the current gateway protocol version:** `scripts/gateway_health_watchdog.py:275-276`
  - body: This connect probe advertises protocol 3, but current main requires protocol 4 and rejects clients whose protocol range does not cover it. The watchdog can therefore report a healthy current gateway as failed and kickstart it after debounce.
  - confidence: 0.95
- **[P2] Resolve the configured gateway auth mode:** `scripts/gateway_health_watchdog.py:206-215`
  - body: The probe only accepts OPENCLAW_GATEWAY_TOKEN or a raw gateway.auth.token string. Supported gateways can use password auth, env password auth, trusted-proxy headers, or SecretInput-backed credentials, so healthy configurations can be reported as missing auth.
  - confidence: 0.86
- **[P2] Use the configured gateway port:** `scripts/gateway_health_watchdog.py:55-56`
  - body: The script always probes port 18789 even though OpenClaw resolves the gateway port from configuration and environment before the default. A healthy gateway on a configured non-default port will be misreported as down.
  - confidence: 0.82
