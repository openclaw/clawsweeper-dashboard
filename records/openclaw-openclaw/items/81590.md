## Summary

Keep open: the PR has useful live proof for the basic authenticated loopback handshake, but the latest branch still diverges from current gateway contracts for state/config path resolution, TLS probes, and managed safe restarts.

## What This Changes

Adds a Python local gateway health watchdog that performs an authenticated WebSocket connect probe, can kickstart the macOS gateway via launchd, and adds Python/Vitest contract coverage plus a cron test fixture type update.

## Best Possible Solution

Route the watchdog through the maintained path/config/auth/TLS and safe-restart primitives, or narrow it as owner-approved macOS emergency tooling with coverage for state dir, TLS, profiles, and active-work deferral.

## Reproduction Assessment

Yes. By source inspection against the PR head and current main, setting OPENCLAW_STATE_DIR or gateway.tls.enabled makes the watchdog use different path/probe contracts, and any trigger calls launchctl directly instead of the safe restart path.

## Solution Assessment

No. Hand-mirroring gateway protocol, auth, path, TLS, and restart behavior in a Python watchdog is not the narrowest maintainable solution when OpenClaw already has GatewayClient/status and safe-restart primitives.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Honor OPENCLAW_STATE_DIR for watchdog paths:** `scripts/gateway_health_watchdog.py:58-67`
  - body: The script derives OPENCLAW_HOME, logs, state, and the default config path from ~/.openclaw, but current OpenClaw resolves mutable state/config from OPENCLAW_STATE_DIR and state-dir candidates. A managed/profile install can be healthy while the watchdog reads the wrong config or logs and probes with the wrong port/auth.
  - confidence: 0.91
- **[P2] Probe TLS-enabled gateways with wss:** `scripts/gateway_health_watchdog.py:493-497`
  - body: This opens a plain TCP socket and sends a cleartext WebSocket upgrade to 127.0.0.1. Current status and gateway-call paths switch local probes to wss:// and load the TLS fingerprint when gateway.tls.enabled is true, so a healthy TLS gateway can fail the watchdog twice and be treated as restartable.
  - confidence: 0.89
- **[P2] Restart through the managed safe path:** `scripts/gateway_health_watchdog.py:651-655`
  - body: When the trigger fires, the script calls launchctl kickstart -k directly. Current user-facing restart supports openclaw gateway restart --safe so active work can defer restart, and the managed launchd path handles enable, stale-port cleanup, and bootstrap fallback that this script skips.
  - confidence: 0.86
