## Summary

Keep open, but do not merge as-is: the watchdog still has source-level correctness blockers in protocol parsing, gateway port parsing, and SecretInput resolution, and the copied terminal proof does not demonstrate the latest head.

## What This Changes

This PR adds a Python `scripts/gateway_health_watchdog.py` restart watchdog that performs an authenticated gateway WebSocket connect probe and adjusts one live model test return type.

## Best Possible Solution

If maintainers want this watchdog, reuse maintained gateway protocol/auth/port helpers or generate a tiny shared contract, then add focused tests and fresh redacted live proof before merge.

## Reproduction Assessment

Yes. The source path is reproducible without a live gateway: the PR regex does not match current `PROTOCOL_VERSION = 4 as const`, so `_read_gateway_protocol_range()` returns `None` and the probe reports protocol resolution failure before a real connect.

## Solution Assessment

No. The PR hand-copies protocol, port, and secret resolution; a maintainable fix should reuse or generate from existing gateway contracts and cover the restart decision with tests and live proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Fix protocol version parsing before probing:** `scripts/gateway_health_watchdog.py:158`
  - body: The regex only matches a bare numeric assignment, but current main exports `PROTOCOL_VERSION = 4 as const`. That makes `_read_gateway_protocol_range()` return `None`, so every real probe reports protocol resolution failure before the WebSocket connect and can eventually restart a healthy gateway.
  - confidence: 0.96
- **[P2] Match the supported gateway port parser:** `scripts/gateway_health_watchdog.py:264-278`
  - body: This parser only accepts plain numeric `OPENCLAW_GATEWAY_PORT` values, while current `resolveGatewayPort` also accepts supported Compose-style values such as `127.0.0.1:18789` and `[::1]:18789`. In those setups the watchdog can probe the wrong port and treat a healthy gateway as down.
  - confidence: 0.88
- **[P2] Use the full SecretInput credential contract:** `scripts/gateway_health_watchdog.py:248-252`
  - body: The script resolves only inline strings, legacy env markers, and `source: env` dictionaries, but gateway auth accepts SecretInput refs resolved through the shared env/file/exec provider path. A file- or exec-backed gateway token/password can be reported as missing auth and trigger unnecessary recovery behavior.
  - confidence: 0.84
