Full review comments:

- **[P1] Read the current protocol version source:** `scripts/gateway_health_watchdog.py:155`
  - body: On the merge ref, `src/gateway/protocol/version.ts` is gone and the protocol constants live under `packages/gateway-protocol/src/version.ts`. Because `_read_protocol_version_from_repo()` returns `None`, every real watchdog probe fails before the authenticated connect can run.
  - confidence: 0.95
- **[P1] Restart through the managed gateway lifecycle:** `scripts/gateway_health_watchdog.py:655`
  - body: Even behind the new opt-in env gate, `kickstart_gateway()` calls `launchctl` directly, outside the SIGUSR1 restart coordinator that coalesces restarts and defers until active operations and pending replies drain. Enabling this can interrupt live replies or background work instead of using the existing restart lifecycle.
  - confidence: 0.91
- **[P2] Preserve exec SecretRef security policy:** `scripts/gateway_health_watchdog.py:358-367`
  - body: The Python exec SecretRef path spawns the configured command without the shared resolver's secure-path, trustedDirs, symlink, permission, no-output timeout, max-output, plugin-integration, and batch-limit checks. This creates a second auth-secret boundary that can accept configurations the runtime would reject or bound differently.
  - confidence: 0.88
- **[P2] Honor resolved state and config directories:** `scripts/gateway_health_watchdog.py:59-90`
  - body: The watchdog defaults config, error log, and state paths from `OPENCLAW_HOME`, but current gateway installs can use `OPENCLAW_STATE_DIR` with a separately resolved config path. Those supported installs can be probed against the wrong config, log, or state and reported unhealthy.
  - confidence: 0.86
- **[P2] Use TLS-aware gateway probing:** `scripts/gateway_health_watchdog.py:495-499`
  - body: `probe_ws_upgrade()` always opens a plaintext socket to 127.0.0.1. TLS-enabled local gateways are supported, so this probe can mark a healthy `wss` gateway as failed and trigger a restart when kickstart is enabled.
  - confidence: 0.84
- **[P2] Store watchdog state in canonical storage:** `scripts/gateway_health_watchdog.py:66-69`
  - body: The script adds a JSON state file under the OpenClaw home/data path for watchdog cooldown and escalation state. OpenClaw-owned runtime state and caches are SQLite-first, so this should use the canonical state owner or remain outside core as an operator-local artifact.
  - confidence: 0.8
