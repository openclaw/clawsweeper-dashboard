## Review Findings

Overall confidence: 0.93

Full review comments:

- **[P1] Read the current protocol package constants:** `scripts/gateway_health_watchdog.py:155`
  - body: The script reads `src/gateway/protocol/version.ts`, but current main exposes the gateway protocol constants from `packages/gateway-protocol/src/version.ts`. On current main this returns `None`, so the watchdog reports that it cannot resolve the protocol version before it can perform the authenticated handshake.
  - confidence: 0.96
- **[P1] Use the existing TLS-aware gateway probe path:** `scripts/gateway_health_watchdog.py:495-499`
  - body: This opens a plaintext socket to `127.0.0.1`, while current gateway connection details switch local probes to `wss://` when `gateway.tls.enabled` is set and also honor URL/config resolution. A healthy TLS-enabled or overridden gateway can therefore be marked unhealthy.
  - confidence: 0.91
- **[P1] Preserve the shared exec SecretInput policy:** `scripts/gateway_health_watchdog.py:358-366`
  - body: The Python resolver runs configured exec providers itself instead of using the shared SecretInput resolver, missing secure command path validation, output limits, no-output timeout handling, and resolver process controls. That creates a second security-sensitive gateway auth path that can drift from the runtime contract.
  - confidence: 0.89
- **[P1] Restart through the managed gateway lifecycle:** `scripts/gateway_health_watchdog.py:653-655`
  - body: Even behind the opt-in env var, this calls `launchctl kickstart -k` directly. That bypasses the existing `scheduleGatewaySigusr1Restart` deferral, cooldown, coalescing, and session-key protection, so enabled watchdog recovery can interrupt active gateway work outside the managed lifecycle.
  - confidence: 0.9
- **[P2] Store watchdog state through the canonical state owner:** `scripts/gateway_health_watchdog.py:66-68`
  - body: The script writes `gateway-watchdog-state.json` under `OPENCLAW_HOME/data` for cooldown and history. OpenClaw runtime state should use the canonical state directory/storage owner, so this sidecar can ignore `OPENCLAW_STATE_DIR` and create a second state source during upgrades.
  - confidence: 0.82
