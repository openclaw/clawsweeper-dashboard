Full review comments:

- **[P1] Read protocol constants from the current package:** `scripts/gateway_health_watchdog.py:155-160`
  - body: `_read_protocol_version_from_repo()` opens `src/gateway/protocol/version.ts`, but that directory does not exist on current main; the protocol constants now live under `packages/gateway-protocol/src/version.ts`. As merged, every non-simulated watchdog run will fail before the authenticated handshake with `unable to resolve gateway protocol version from repo`.
  - confidence: 0.96
- **[P1] Use the existing TLS-aware gateway probe path:** `scripts/gateway_health_watchdog.py:725`
  - body: The watchdog always calls `probe_ws_upgrade(GATEWAY_HOST, gateway_port, ...)`, so it ignores the current gateway probe target, URL, TLS fingerprint, and `GatewayClient` handshake behavior. Supported remote or TLS-pinned gateway setups can be reported unhealthy and, with kickstart enabled, driven into recovery even though the configured gateway is reachable through the supported probe path.
  - confidence: 0.9
- **[P1] Preserve the shared exec SecretInput policy:** `scripts/gateway_health_watchdog.py:358-366`
  - body: This Python resolver executes configured `exec` SecretInput providers directly with `subprocess.run`, bypassing the shared resolver's absolute/safe path checks, trustedDirs/symlink policy, output limits, no-output timeout, process-tree cleanup, and plugin integration support. That creates a second gateway credential execution path with weaker security semantics.
  - confidence: 0.93
- **[P1] Restart through the managed gateway lifecycle:** `scripts/gateway_health_watchdog.py:655-657`
  - body: Even behind `GATEWAY_WATCHDOG_ALLOW_KICKSTART`, direct `launchctl kickstart` bypasses `requestSafeGatewayRestart` and `scheduleGatewaySigusr1Restart`, which own active-work deferral, cooldown/coalescing, and restart continuation state. Enabling this watchdog can interrupt in-flight agent work or lose handoff state outside the current restart contract.
  - confidence: 0.88
- **[P2] Store watchdog state in the canonical state owner:** `scripts/gateway_health_watchdog.py:144-147`
  - body: The script writes runtime watchdog state to a new JSON file under the OpenClaw state area. Current repository policy and runtime state code use SQLite-backed state owners for OpenClaw-owned runtime state, so this adds a parallel persistent state path that doctor/runtime migrations will not own.
  - confidence: 0.78
