## Summary

Keep open: the PR has useful intent and current-head loopback proof, but the patch still hand-mirrors gateway path, TLS, restart, and SecretRef contracts in a new self-healing script, including one security-sensitive divergence from the maintained resolver.

## What This Changes

The branch adds a Python gateway health watchdog script with authenticated WebSocket probing, Python/Vitest contract tests, and a small cron test fixture type update.

## Best Possible Solution

Route the watchdog through maintained gateway path/auth/TLS probing and safe-restart primitives, or keep it as an explicitly owner-approved macOS emergency script with matching security and path-contract coverage.

## Reproduction Assessment

Yes, by source inspection: setting OPENCLAW_STATE_DIR, enabling gateway.tls, configuring guarded file/exec SecretRefs, or having active work during a trigger exercises paths where the PR diverges from current main contracts.

## Solution Assessment

No. The safer maintainable solution is to reuse the existing TypeScript GatewayClient/status probe, SecretRef resolver, and safe-restart coordinator instead of duplicating those contracts in Python.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Resolve SecretRefs through the guarded resolver:** `scripts/gateway_health_watchdog.py:320-365`
  - body: The watchdog reads file SecretRefs and executes exec SecretRefs directly, skipping the maintained resolver’s secure path checks, trustedDirs/allowSymlinkCommand policy, output limits, and no-output timeout. A SecretRef that OpenClaw would reject can be read or executed by this script, so the new watchdog weakens the secret-provider contract.
  - confidence: 0.86
- **[P2] Honor OPENCLAW_STATE_DIR for watchdog paths:** `scripts/gateway_health_watchdog.py:58-67`
  - body: The script derives the default config, state, and err-log paths from OPENCLAW_HOME/~/.openclaw. Managed/profile installs can set OPENCLAW_STATE_DIR, so the watchdog can read the wrong config/logs and probe with stale auth or port data while the real gateway is healthy.
  - confidence: 0.91
- **[P2] Probe TLS-enabled gateways over wss:** `scripts/gateway_health_watchdog.py:493-497`
  - body: This always opens a cleartext TCP socket and sends a ws upgrade to loopback. Current status probes switch local loopback to wss and pass the local TLS fingerprint when gateway.tls.enabled is true, so a healthy TLS gateway will look failed and can be restarted unnecessarily.
  - confidence: 0.89
- **[P2] Restart through the managed safe path:** `scripts/gateway_health_watchdog.py:651-655`
  - body: The trigger path calls launchctl kickstart -k directly. Current safe restart preflights queued work, reply delivery, embedded runs, and task runs; bypassing that path can interrupt live work the gateway would otherwise defer.
  - confidence: 0.84
