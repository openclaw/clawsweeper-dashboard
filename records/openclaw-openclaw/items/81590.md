- **[P1] Read the current protocol package constants:** `scripts/gateway_health_watchdog.py:155`
  - body: The watchdog reads `src/gateway/protocol/version.ts`, but the actual merge result keeps current main's protocol constants under `packages/gateway-protocol/src/version.ts` and does not contain that old path. As merged, `_read_gateway_protocol_range()` returns `None`, so the watchdog reports an unresolved protocol version before it can run the authenticated handshake.
  - confidence: 0.97
- **[P1] Use the existing TLS-aware gateway probe path:** `scripts/gateway_health_watchdog.py:495-499`
  - body: This code opens a plaintext socket to `127.0.0.1` and hand-rolls the WebSocket handshake instead of using the current gateway connection/probe path, which already handles `wss://`, TLS fingerprints, URL overrides, timeouts, and device/probe behavior. A healthy TLS-enabled or overridden gateway can be reported unhealthy.
  - confidence: 0.91
- **[P1] Preserve the shared exec SecretInput policy:** `scripts/gateway_health_watchdog.py:358-366`
  - body: The Python resolver executes configured SecretInput commands directly instead of using the shared resolver, so it lacks secure command path validation, max-output limits, no-output timeout, process-tree cleanup, and provider integration handling. That creates a second security-sensitive gateway auth path that can drift from runtime policy.
  - confidence: 0.89
- **[P1] Restart through the managed gateway lifecycle:** `scripts/gateway_health_watchdog.py:655`
  - body: Even behind the opt-in env var, this calls `launchctl kickstart -k` directly. That bypasses the existing restart deferral, cooldown, coalescing, and session-key protections, so enabled watchdog recovery can interrupt active gateway work outside the managed lifecycle.
  - confidence: 0.9
- **[P2] Store watchdog state through the canonical state owner:** `scripts/gateway_health_watchdog.py:66-68`
  - body: The script writes `gateway-watchdog-state.json` under `OPENCLAW_HOME/data` for cooldown and history. OpenClaw runtime state should use the canonical state directory/storage owner, so this sidecar can ignore `OPENCLAW_STATE_DIR` and add another upgrade-sensitive state source.
  - confidence: 0.83
