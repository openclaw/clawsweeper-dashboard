## Summary

Keep open: the PR is active and includes useful live proof, but the new watchdog still duplicates maintained gateway contracts and diverges on state-dir resolution, TLS probing, safe restart, and SecretRef security guardrails.

## What This Changes

The PR adds a Python gateway health watchdog script that performs an authenticated WebSocket handshake probe, can kickstart the macOS gateway service, and adds Python/Vitest contract tests.

## Best Possible Solution

Either reuse the maintained TypeScript gateway path/probe/auth/TLS/SecretRef and safe-restart primitives, or explicitly scope this as an owner-approved emergency macOS ops script with matching guardrails and coverage.

## Reproduction Assessment

Yes, by source inspection against current main. Setting OPENCLAW_STATE_DIR, enabling gateway.tls.enabled, using a protected SecretRef provider, or relying on safe restart semantics all exercise code paths where the PR script diverges from maintained contracts.

## Solution Assessment

No. The maintainable fix is to reuse or call the existing gateway probe/auth/TLS/path and restart machinery rather than hand-mirroring those contracts in a long-running Python watchdog.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Honor OPENCLAW_STATE_DIR for watchdog paths:** `scripts/gateway_health_watchdog.py:58-90`
  - body: The script derives its default config, state, and error-log paths from OPENCLAW_HOME only. Managed/profile installs that set OPENCLAW_STATE_DIR can have the real gateway config and logs elsewhere, so the watchdog may probe with the wrong port/auth and write state into the wrong tree.
  - confidence: 0.91
- **[P2] Use TLS-aware gateway probing:** `scripts/gateway_health_watchdog.py:493-497`
  - body: The probe always opens a plain socket and sends cleartext WebSocket bytes to 127.0.0.1. When gateway.tls.enabled is true, current status/call paths use wss:// with the local TLS fingerprint, so a healthy TLS gateway would be classified as failed and restartable here.
  - confidence: 0.9
- **[P2] Restart through the safe gateway path:** `scripts/gateway_health_watchdog.py:651-655`
  - body: The watchdog calls launchctl kickstart -k directly after triggers fire. That bypasses openclaw gateway restart --safe, which preflights active queued work, reply delivery, embedded runs, and task runs before restart, so this can interrupt live OpenClaw work.
  - confidence: 0.85
- **[P2] Preserve SecretRef provider safety checks:** `scripts/gateway_health_watchdog.py:320-365`
  - body: The Python SecretRef resolver reads file providers and executes exec providers directly, without the current resolver's secure path checks, trustedDirs/symlink policy, max output limits, no-output timeout, and provider limits. Because this script is meant to run automatically, it should reuse the maintained resolver or enforce the same guardrails before resolving gateway auth.
  - confidence: 0.86
