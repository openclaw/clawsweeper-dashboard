## Summary

This PR should stay open because the current head still adds a risky local self-healing gateway watchdog that duplicates maintained gateway path, TLS, SecretRef, and restart contracts. The supplied live proof covers the happy-path loopback token handshake, but it does not cover the drift cases that can restart a healthy gateway or bypass existing fail-closed secret handling.

## What This Changes

Adds a Python gateway health watchdog script with an authenticated WebSocket connect probe plus Python/Vitest contract tests and a live-test type cleanup.

## Best Possible Solution

Route the watchdog through existing OpenClaw gateway probe/auth/path/TLS and safe-restart primitives, or keep it as an explicitly owner-approved emergency script with matching coverage for those contracts.

## Reproduction Assessment

Yes. Source inspection against current main shows concrete drift: OPENCLAW_STATE_DIR/profile paths, gateway.tls, managed safe restart, and SecretRef file/exec security contracts are handled by current OpenClaw paths but not by the PR watchdog.

## Solution Assessment

No. The maintainable fix is to reuse the existing TypeScript gateway probe/auth/restart machinery or intentionally scope the script as an emergency-only macOS tool after owner approval, not to hand-mirror those contracts in Python.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Honor OPENCLAW_STATE_DIR for watchdog paths:** `scripts/gateway_health_watchdog.py:58-90`
  - body: The script derives its default config, state, and gateway.err.log paths from OPENCLAW_HOME/~/.openclaw. Current OpenClaw resolves mutable state and the active config from OPENCLAW_STATE_DIR/profile-aware paths, so a healthy managed/profile install can be probed with the wrong port/auth/logs and then restarted incorrectly.
  - confidence: 0.91
- **[P2] Probe TLS-enabled gateways with wss:** `scripts/gateway_health_watchdog.py:473-497`
  - body: The watchdog always opens a cleartext socket and sends a plain WebSocket upgrade to 127.0.0.1. Current status/probe paths switch local probes to wss:// when gateway.tls.enabled is true, so a healthy TLS gateway will fail this probe and can be treated as restartable.
  - confidence: 0.89
- **[P2] Restart through the managed safe path:** `scripts/gateway_health_watchdog.py:651-655`
  - body: When a trigger fires, the script calls launchctl kickstart -k directly. OpenClaw already exposes gateway restart --safe to preflight queued operations, reply delivery, embedded runs, and task runs; bypassing it can interrupt live work that the maintained path would defer.
  - confidence: 0.86
- **[P2] Keep SecretRef path security checks:** `scripts/gateway_health_watchdog.py:320-363`
  - body: The Python SecretRef resolver reads file providers and runs exec providers directly. Current OpenClaw fails closed on insecure file/exec paths, symlink/trustedDirs policy, no-output timeouts, and output byte limits, so the watchdog can accept and execute a secrets config that the runtime would reject.
  - confidence: 0.86

## Security Review
