Full review comments:

- **[P1] Read the current protocol version source:** `scripts/gateway_health_watchdog.py:155`
  - body: Current main no longer has `src/gateway/protocol/version.ts`; the constants live in `packages/gateway-protocol/src/version.ts`. As written, `_read_protocol_version_from_repo()` returns `None`, so real watchdog checks fail with `unable to resolve gateway protocol version from repo` before the authenticated connect probe can run.
  - confidence: 0.95
- **[P1] Restart through the managed gateway lifecycle:** `scripts/gateway_health_watchdog.py:655`
  - body: Even with `GATEWAY_WATCHDOG_ALLOW_KICKSTART=1`, calling `launchctl kickstart -k` directly bypasses the restart coordinator that checks pending replies, queues, embedded runs, cron runs, and tasks before SIGUSR1 restart. Enabling the watchdog can therefore interrupt live work instead of using the existing deferral path.
  - confidence: 0.91
- **[P2] Preserve shared exec SecretRef policy:** `scripts/gateway_health_watchdog.py:358-367`
  - body: The Python exec SecretRef path spawns the configured command itself, bypassing the shared resolver's secure path, trustedDirs, symlink, no-output timeout, max-output, plugin-integration, and batch-limit checks. Gateway auth secrets should resolve through the shared SecretRef boundary rather than a second implementation.
  - confidence: 0.88
- **[P2] Use the existing TLS-aware gateway probe:** `scripts/gateway_health_watchdog.py:495-499`
  - body: `probe_ws_upgrade()` always opens a plaintext socket to 127.0.0.1, while the current gateway probe goes through `GatewayClient` and passes `tlsFingerprint` for `wss://` daemon probes. A TLS-enabled local gateway can be marked unhealthy and, with kickstart enabled, restarted unnecessarily.
  - confidence: 0.84
- **[P2] Honor resolved config and state directories:** `scripts/gateway_health_watchdog.py:59-90`
  - body: The watchdog defaults config, log, and state paths from `OPENCLAW_HOME`, but current gateway deployments can use per-instance `OPENCLAW_CONFIG_PATH` and `OPENCLAW_STATE_DIR`. Supported multi-gateway installs can therefore be checked against the wrong config or logs and reported unhealthy.
  - confidence: 0.86
