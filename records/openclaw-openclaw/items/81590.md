## Summary

Keep open. The authenticated handshake probe is useful, but the latest branch still hand-mirrors gateway state/config, TLS, auth, SecretRef, and restart behavior in a new Python watchdog, leaving healthy state-dir, TLS, and password-mode installs at risk of false failures plus a secret-provider safety regression.

## What This Changes

This PR adds a Python gateway health watchdog with authenticated WebSocket handshake probing, opt-in launchd kickstart, Python/Vitest contract tests, and a small cron test fixture type update.

## Best Possible Solution

Route the watchdog through maintained gateway primitives for state/config paths, TLS/fingerprint probing, credential selection, SecretRef safety, and safe restart coordination, with live proof for the affected install modes.

## Reproduction Assessment

Yes for the PR review findings: source inspection of the latest head shows state-dir, TLS, auth-mode, and SecretRef safety divergence from current main. I did not run a live gateway because this is a read-only review.

## Solution Assessment

No. The authenticated connect probe is useful, but duplicating gateway runtime contracts in Python is not the narrowest maintainable solution for a self-healing gateway path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Honor state-dir config paths:** `scripts/gateway_health_watchdog.py:59-90`
  - body: The script treats OPENCLAW_HOME as the state directory and defaults config/log/state paths under it. Current OpenClaw resolves OPENCLAW_STATE_DIR separately and treats OPENCLAW_HOME as the effective home, so managed/profile installs can have a healthy gateway while the watchdog reads stale logs or the wrong config/auth/port.
  - confidence: 0.92
- **[P2] Use the maintained TLS probe path:** `scripts/gateway_health_watchdog.py:495-499`
  - body: The probe always opens a plain TCP socket and sends a cleartext WebSocket upgrade. For gateway.tls.enabled, current status probes use wss:// and pass the local TLS fingerprint; this watchdog will count a healthy TLS gateway as failed.
  - confidence: 0.9
- **[P2] Preserve auth-mode credential precedence:** `scripts/gateway_health_watchdog.py:455-462`
  - body: When OPENCLAW_GATEWAY_TOKEN is set, _load_gateway_auth returns that token before reading config. A password-mode gateway with a configured password and stale token env will be probed with token only, while the server requires connectAuth.password, so the watchdog can fail a healthy gateway.
  - confidence: 0.86
- **[P2] Use the safe SecretRef resolver:** `scripts/gateway_health_watchdog.py:322-390`
  - body: The script resolves file and exec SecretRefs itself, but current OpenClaw enforces secure file reads, permission checks, command path checks, and output/time limits. This path can accept or execute provider configs the core resolver would reject, so it should call a maintained resolver or narrow the supported auth sources.
  - confidence: 0.83

## Security Review

Status: needs_attention

Summary: The diff adds local SecretRef file reads and exec-provider command execution without the maintained resolver’s safety checks.

Concerns:

- **[medium] Bypass SecretRef file and exec safety checks:** `scripts/gateway_health_watchdog.py:322`
  - body: The Python resolver reads file providers directly and runs exec providers directly, omitting the core resolver’s secure-file permission checks, trusted command path checks, max output limits, and no-output timeout handling. That weakens the secrets contract for any operator who runs this watchdog with file or exec gateway auth refs.
