## Summary

Keep open: the PR has convincing live proof for the basic authenticated loopback handshake, but current head still diverges from OpenClaw's maintained state/config, TLS probe, and safe-restart contracts. Because this also overlaps the broader rescue-watchdog effort, it needs owner/product review rather than cleanup closure.

## What This Changes

The PR adds a Python gateway health watchdog with authenticated WebSocket handshake probing, Python/Vitest contract tests, and a one-line live model test return-type change.

## Best Possible Solution

Route the watchdog through maintained GatewayClient/status probe and safe-restart primitives, or explicitly defer the standalone script in favor of the first-class rescueWatchdog design in https://github.com/openclaw/openclaw/pull/46502.

## Reproduction Assessment

Yes, for the review findings: source inspection shows OPENCLAW_STATE_DIR and gateway.tls configurations follow different contracts on current main than the PR script does, and the restart path directly invokes launchctl instead of the safe restart RPC.

## Solution Assessment

No. The safer maintainable path is to reuse the existing gateway probe/auth/TLS/path and safe-restart surfaces, or align this work with the broader rescueWatchdog PR before adding another self-healing path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Honor OPENCLAW_STATE_DIR for watchdog state paths:** `scripts/gateway_health_watchdog.py:58-67`
  - body: The script still builds OPENCLAW_HOME from ~/.openclaw and derives the default log, state, and config paths from that value. Current OpenClaw resolves mutable state and the default config from OPENCLAW_STATE_DIR, so a profile or managed install can be healthy while this watchdog reads the wrong config/logs and probes with the wrong port or auth.
  - confidence: 0.91
- **[P2] Use TLS-aware gateway probes:** `scripts/gateway_health_watchdog.py:493-497`
  - body: This path always opens a plain TCP socket and sends a cleartext WebSocket upgrade. Current status and gateway-call probes switch local TLS gateways to wss:// with the local TLS fingerprint, so a healthy gateway.tls.enabled=true install will fail the watchdog probe and can be kickstarted.
  - confidence: 0.89
- **[P2] Restart through the managed safe path:** `scripts/gateway_health_watchdog.py:651-655`
  - body: When a trigger fires, the script calls launchctl kickstart -k directly. Current openclaw gateway restart --safe coordinates with the running Gateway and defers restart while reply delivery, embedded runs, or task runs are active; bypassing it can drop live work.
  - confidence: 0.84
