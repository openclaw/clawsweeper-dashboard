- **[P1] Read the current protocol package constants:** `scripts/gateway_health_watchdog.py:155`
  - body: On the current merge result, `src/gateway/protocol/version.ts` is absent; only `packages/gateway-protocol/src/version.ts` defines the protocol constants. `_read_gateway_protocol_range()` therefore returns `None`, so the real watchdog probe fails before the authenticated connect can run.
  - confidence: 0.95
- **[P1] Restart through the managed gateway lifecycle:** `scripts/gateway_health_watchdog.py:655`
  - body: Even with the new opt-in env var, `kickstart_gateway()` calls `launchctl kickstart -k` directly. That bypasses `scheduleGatewaySigusr1Restart()` and the restart deferral/drain path, so enabling the watchdog can interrupt active replies or background work.
  - confidence: 0.91
- **[P2] Preserve the shared exec SecretRef policy:** `scripts/gateway_health_watchdog.py:358-367`
  - body: The Python exec SecretRef path spawns the configured command without the shared resolver's secure-path, trustedDirs/symlink, no-output timeout, max-output, max-batch, and process-tree cleanup policies. Gateway auth secrets should not gain a second weaker resolver.
  - confidence: 0.88
- **[P2] Honor resolved state and config directories:** `scripts/gateway_health_watchdog.py:59-69`
  - body: The script derives log and state paths from `OPENCLAW_HOME`, while current gateway installs can use `OPENCLAW_STATE_DIR` with a separate `OPENCLAW_CONFIG_PATH`. Those supported multi-gateway layouts can be checked against the wrong log/state and reported unhealthy.
  - confidence: 0.86
- **[P2] Use the TLS-aware gateway probe path:** `scripts/gateway_health_watchdog.py:495-499`
  - body: `probe_ws_upgrade()` always opens a plaintext socket to `127.0.0.1`, but local gateway probe details can resolve to `wss://` with a TLS fingerprint when `gateway.tls.enabled` is set. A healthy TLS-enabled gateway can therefore be marked failed and restarted.
  - confidence: 0.84
- **[P2] Store watchdog state in canonical storage:** `scripts/gateway_health_watchdog.py:66-69`
  - body: The script adds a JSON state file under the OpenClaw home/data path for watchdog cooldown and escalation state. OpenClaw-owned runtime state and caches are SQLite-first, so this should use the canonical state owner or remain outside core as an operator-local artifact.
  - confidence: 0.8
