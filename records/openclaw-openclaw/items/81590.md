## Summary

Keep open: the current PR head has useful authenticated-handshake work and sufficient happy-path proof, but it still bypasses current gateway state-dir, TLS, and safe-restart contracts in a new self-healing restart surface.

## What This Changes

This PR adds a Python gateway health watchdog that performs an authenticated WebSocket connect probe, can kickstart the macOS gateway, and adds Python/Vitest contract coverage plus a small cron test fixture type update.

## Best Possible Solution

Route the watchdog through the maintained path/config/auth/TLS and safe-restart contracts, or narrow it as owner-approved macOS emergency tooling with explicit coverage for state dirs, TLS, profiles, and active-work deferral.

## Reproduction Assessment

Yes for the PR review findings: source inspection of the latest PR head against current main shows OPENCLAW_STATE_DIR, gateway.tls.enabled, and safe-restart behavior diverge. I did not run a live gateway because this review is read-only.

## Solution Assessment

No. The authenticated handshake probe is useful, but hand-mirroring gateway config, TLS, auth, and restart behavior in a Python watchdog is not the narrowest maintainable path unless the gateway owners explicitly approve that emergency-tool boundary.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Honor OPENCLAW_STATE_DIR for watchdog paths:** `scripts/gateway_health_watchdog.py:58-68`
  - body: The watchdog derives its default log, state, and config paths from OPENCLAW_HOME, while current OpenClaw supports OPENCLAW_STATE_DIR for mutable state and config candidates. Managed/profile installs can therefore have a healthy gateway while the watchdog reads stale logs or the wrong config/auth/port.
  - confidence: 0.91
- **[P2] Probe TLS-enabled gateways with WSS:** `scripts/gateway_health_watchdog.py:493-497`
  - body: The probe always opens a plain TCP socket and sends a cleartext WebSocket upgrade to 127.0.0.1. Current gateway status switches local probes to wss:// and passes the TLS fingerprint when gateway.tls.enabled is true, so a healthy TLS gateway can be counted as failing and become restartable.
  - confidence: 0.89
- **[P2] Restart through the managed safe path:** `scripts/gateway_health_watchdog.py:651-655`
  - body: The trigger path calls launchctl kickstart -k directly. That bypasses openclaw gateway restart --safe active-work preflight/deferral and the managed restart recovery path for stale ports and launch agent repair, so the watchdog can interrupt active work or miss existing recovery handling.
  - confidence: 0.86
