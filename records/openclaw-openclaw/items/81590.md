- **[P2] Honor the resolved state and config directories:** `scripts/gateway_health_watchdog.py:59-68`
  - body: The watchdog defaults logs, state, and config to paths under OPENCLAW_HOME, but current OpenClaw supports OPENCLAW_STATE_DIR and resolves the default config path from that state dir. Profiled or managed installs can therefore have a healthy gateway while this script reads stale logs, the wrong state file, or the wrong auth/port config.
  - confidence: 0.92
- **[P2] Use WSS for TLS-enabled local gateways:** `scripts/gateway_health_watchdog.py:495-499`
  - body: This probe always opens a plain TCP socket and sends a cleartext WebSocket upgrade. Current daemon status switches local probes to wss:// and forwards the local TLS fingerprint when gateway.tls.enabled is true, so a healthy TLS gateway can be counted as failed and restartable.
  - confidence: 0.9
- **[P2] Restart through the safe gateway restart path:** `scripts/gateway_health_watchdog.py:653-655`
  - body: Even with kickstart opt-in, the enabled path still calls launchctl kickstart directly. That bypasses gateway.restart.request preflight/deferral and the managed restart recovery path, so the watchdog can interrupt active work or miss existing service repair behavior.
  - confidence: 0.88
- **[P2] Preserve exec SecretRef security checks:** `scripts/gateway_health_watchdog.py:358-367`
  - body: The script runs configured exec SecretRef commands directly, while the shared resolver first asserts the command path security policy and bounds command output/timeouts. A gateway auth config that core would reject or constrain can be executed by this watchdog unless it routes through the maintained resolver or implements the same policy with tests.
  - confidence: 0.86
- **[P2] Parse the full SecretInput env shorthand:** `scripts/gateway_health_watchdog.py:246-247`
  - body: Current SecretInput parsing accepts both $NAME and ${NAME} env shorthand, but this resolver only treats ${NAME} as an env SecretRef before falling back to a literal string. A gateway.auth token configured with $OPENCLAW_GATEWAY_TOKEN will be sent literally and the healthy gateway probe will fail.
  - confidence: 0.84
