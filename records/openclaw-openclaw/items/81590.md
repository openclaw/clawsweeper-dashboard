## Summary

Keep open: the latest branch still has correctness blockers in the new restart watchdog, and the supplied real-behavior proof no longer matches the current source evidence.

## What This Changes

The PR adds a Python gateway health watchdog script that performs an authenticated WebSocket connect probe and kickstarts the launchd gateway service on failures, plus a live model test return-type annotation.

## Best Possible Solution

If this watchdog is wanted, build it on the maintained gateway client/status path or a shared generated protocol/auth/port helper, then cover the restart decision with focused tests and fresh redacted live proof.

## Reproduction Assessment

Yes. Source inspection plus a direct Python regex check show the latest script cannot read current `src/gateway/protocol/version.ts`, so its probe will report protocol resolution failure before a real connect handshake.

## Solution Assessment

No. Duplicating the gateway handshake, credential resolver, and port resolver in Python is fragile; the safer path is to reuse or generate from the existing gateway contracts instead of mirroring them by hand.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Fix protocol version parsing before probing:** `scripts/gateway_health_watchdog.py:158`
  - body: The regex used to read `PROTOCOL_VERSION` does not match the current `export const PROTOCOL_VERSION = 4 as const;` line, so `_read_gateway_protocol_range()` returns `None` and every real probe reports `unable to resolve gateway protocol version from repo`. After the debounce window, a healthy gateway can be treated as failed and kickstarted.
  - confidence: 0.96
- **[P2] Match the supported gateway port parser:** `scripts/gateway_health_watchdog.py:264-278`
  - body: This parser only accepts plain numeric `OPENCLAW_GATEWAY_PORT` values, but the repo's `resolveGatewayPort` also accepts supported Compose-style values like `127.0.0.1:18789` and `[::1]:28789`. In those setups the watchdog can probe the wrong port and report a healthy gateway as down.
  - confidence: 0.88
- **[P2] Use the full SecretInput credential contract:** `scripts/gateway_health_watchdog.py:248-252`
  - body: The script resolves only inline strings, legacy env markers, and `source: env` dictionaries, while gateway auth accepts structured SecretInput refs from env, file, and exec providers. A gateway using file/exec-backed token or password auth will be reported as missing auth and can be restarted unnecessarily.
  - confidence: 0.84
