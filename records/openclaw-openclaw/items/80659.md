## Summary

Keep open: the reported ghost-turn bug is plausible from current main, but this PR is not mergeable as-is because the patch drops existing timeout safeguards, the branch diff spans 588 files, and the attached after-fix proof is still insufficient.

## What This Changes

The branch changes the embedded Pi runner to always append an idle-timeout error payload on prompt-phase timeout while preserving prior payloads, but the PR diff also carries many unrelated release, workflow, docs, app, plugin, and UI changes.

## Best Possible Solution

Rebase and narrow the PR to the runner and focused tests, append the timeout error only after successful non-messaging tool/media payloads, preserve current delivery and partial-fragment safeguards, and add redacted after-fix proof.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: current main can skip timeout-error injection when `timedOutDuringPrompt` is true but non-messaging tool payloads make `payloadsWithToolMedia` non-empty. I did not establish a live provider timeout run.

## Solution Assessment

No, this patch is not the best fix as written. The maintainable fix should preserve current messaging-tool delivery behavior, suppress partial assistant fragments, and add a targeted regression test for successful tool payloads followed by prompt idle timeout.
