## Summary

Keep open. The PR is a valid narrow fix direction for Telegram DM-topic subagent announces, but it should not merge as written because the routing change still uses the generic target resolver, the added log can report a thread removed by fallback, and changelog coverage is missing.

## What This Changes

The PR changes subagent completion announce routing to preserve Telegram DM-topic thread IDs, adds Telegram sendMessage thread logging, and adds regression coverage for direct and queued announce paths.

## Best Possible Solution

Rebase the fix onto current main, derive the requester conversation id through a channel-aware target parser while keeping threadId as a delivery modifier, log the effective post-fallback Telegram thread, add focused regression coverage, and include the required changelog entry.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: Telegram records `OriginatingTo: telegram:<chatId>` plus a separate `MessageThreadId`, while the PR/current routing path uses the generic resolver that skips `telegram:` targets and can fall back to the topic id for bound-route matching.

## Solution Assessment

No as-is. Preferring the chat-level route over the topic id is the right direction, but the implementation needs the Telegram-aware parser, effective-thread logging, changelog coverage, and current-main rebase before it is the narrow maintainable fix.
