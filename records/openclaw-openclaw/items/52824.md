## Summary

Keep open. This PR targets a valid Telegram subagent completion announce bug, but it should not merge as-is because the routing change still misses real `telegram:<chatId>` requester targets, the new logging can report the pre-fallback thread, the changelog coverage is missing, and current main has since split the active helper into `src/agents/subagent-announce-delivery.ts`.

## What This Changes

The PR changes subagent completion announce routing to preserve Telegram topic `threadId`, adds Telegram sendMessage thread logging, and adds regression coverage.

## Best Possible Solution

Rebase or replace the patch on current main, derive the requester conversation id through the Telegram-aware target parser while keeping `threadId` as a delivery modifier, log the effective post-fallback thread, add regression coverage for real `telegram:<chatId>` targets, and add the required changelog entry.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: Telegram records `OriginatingTo: telegram:<chatId>` plus `MessageThreadId`, while current completion routing uses the topic `threadId` before a channel-aware target parse for bound-route lookup.

## Solution Assessment

No as-is. Preferring the chat-level route is the right direction, but this PR needs the Telegram-aware parser, effective-thread logging, current-main rebase, and changelog coverage before it is the narrow maintainable fix.
