## Summary

Keep open. The PR appears to be a focused and correct fix for current-main behavior where non-Windows allowlist entries with `argPattern` are accepted on path match before argv is checked, but the PR carries the protected `maintainer` label and touches the exec approval trust boundary, so it should go through maintainer review rather than cleanup closure.

## What This Changes

The PR removes the Windows-only gate from exec allowlist `argPattern` matching, expands matcher coverage across Linux, macOS, and Windows, and documents the `argPattern` allowlist field with a changelog entry.

## Best Possible Solution

Merge the focused cross-platform `argPattern` enforcement after maintainer review and completion of the relevant checks, while keeping broader deny-pattern or allow-always generation changes separate.

## Reproduction Assessment

Yes. On current main, a `matchAllowlist` call with an `argPattern` entry and `platform` of `linux` or `darwin` returns on path match before argument evaluation, which is directly visible in `src/infra/exec-command-resolution.ts`.

## Solution Assessment

Yes. Removing the platform gate is the narrowest maintainable fix for the observed mismatch, and the PR preserves path-only fallback semantics while adding cross-platform matcher coverage and docs.
