## Summary

Keep open: the linked bug is real and current main/latest release still fail closed, but this PR is not merge-ready because it widens plugin-root containment by trusting escaped package-name metadata and lacks real packaged Codex sub-agent proof.

## What This Changes

This PR modifies `src/plugins/loader.ts` to allow plugin-root boundary failures when the escaped realpath appears to be inside a package named `openclaw`, at runtime, setup-runtime, and CLI metadata load sites.

## Best Possible Solution

Repair host SDK peer imports through explicit SDK/native resolver or package-entry handling, keep plugin entry containment fail-closed, add negative containment tests, and provide packaged Codex sub-agent proof.

## Reproduction Assessment

Yes from source, but not from a live packaged run in this review. Current main and `v2026.6.10` keep the strict plugin-root rejection, strict runtime loading can throw, and the Codex plugin imports OpenClaw SDK subpaths through the documented host peer link shape.

## Solution Assessment

No. The patch is a plausible workaround, but it broadens plugin entry and CLI metadata containment; the safer fix belongs in explicit SDK resolver or package-entry handling.
