## Summary

## What This Changes

The branch modifies `src/plugins/loader.ts` to allow failed plugin-root safe-open checks to proceed when `fs.realpathSync.native()` resolves to a nearby package named `openclaw`.

## Best Possible Solution

Repair host SDK peer imports through explicit SDK/native resolver or package-entry handling, keep plugin entry containment fail-closed, add negative containment tests, and provide packaged Codex sub-agent proof.

Yes from source, but not from a live packaged run in this review. Current main still rejects escaped plugin entries, strict runtime loading can throw, and the Codex plugin imports OpenClaw SDK subpaths through the documented host peer link shape.

## Solution Assessment

No. The patch is a plausible workaround, but it broadens plugin entry and CLI metadata containment; the safer fix belongs in explicit SDK resolver or package-entry handling.
