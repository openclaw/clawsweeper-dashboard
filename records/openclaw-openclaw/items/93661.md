## Summary

Keep open: current main and the latest release still hard-deny Claude native `can_use_tool` requests under `allowlist`/`on-miss`, and https://github.com/openclaw/openclaw/pull/94424 is the open closing implementation candidate rather than a merged fix.

## What This Changes

The issue asks OpenClaw to stop treating Claude CLI native file-tool permission as all-or-nothing by routing restrictive `ask` modes to approval, adding native allowlist support, or documenting/fixing the root bypass dead end.

## Best Possible Solution

Land or revise https://github.com/openclaw/openclaw/pull/94424 so restrictive Claude native tool misses use the configured operator approval path without weakening `deny` or full/off semantics, while keeping broader allowlist parity tracked separately if needed.
