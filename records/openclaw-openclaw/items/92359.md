## Summary

Keep this PR open: current main and the latest release still have a strict `ModelCompatSchema` that omits `sendSessionAffinityHeaders`, while the type and runtime paths already recognize that compat flag. The patch is a narrow schema/test alignment and is not superseded by the related transport PR, which tracks a separate runtime forwarding gap.

## What This Changes

Adds `model.compat.sendSessionAffinityHeaders` as an optional boolean in the strict model config Zod schema and adds a regression test proving that config is accepted.

## Best Possible Solution

Land the focused schema/test fix for persisted custom model config, then coordinate with the separate transport PR if maintainers want full same-release session-affinity behavior across all OpenAI-compatible paths.

## Reproduction Assessment

Yes. Source inspection shows current main's strict `ModelCompatSchema` omits `sendSessionAffinityHeaders` while `ModelsConfigSchema` routes provider model `compat` through that schema, so the PR body's config shape is rejected before the patch.

## Solution Assessment

Yes. Adding the existing boolean compat field to the strict schema next to the other compat flags plus a regression parse test is the narrowest maintainable fix for this config-validation gap.
