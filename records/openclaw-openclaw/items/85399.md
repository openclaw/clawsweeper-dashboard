## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Honor ACP session caps during startup reconcile:** `src/acp/control-plane/manager.core.ts:317`
  - body: `reconcilePendingSessionIdentities` now starts up to four runtime-handle paths in parallel, but `acp.maxConcurrentSessions` is documented as the maximum concurrently active ACP sessions. Enforcement checks `runtimeCache.size()` before `runtime.ensureSession` records a handle, so with `maxConcurrentSessions: 1` several startup tasks can pass the check and open more runtime sessions than configured. Cap this fanout by the configured limit or add an atomic reservation path, and cover the low-limit case.
  - confidence: 0.86
