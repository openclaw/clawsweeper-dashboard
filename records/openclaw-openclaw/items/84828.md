## Summary

Keep open. The latest PR head is not merge-ready: it changes Telegram progress mode from the documented single edited draft into standalone messages that can enter transcripts, drops existing label-only progress behavior, can send late progress around final delivery, and the new context-token helper can preserve stale token windows across model switches. The PR also still lacks required real Telegram behavior proof.

## What This Changes

The PR adds a shared session context-token resolver and changes Telegram progress mode to send tool progress as standalone durable messages, with updated Telegram dispatch and session tests.

## Best Possible Solution

Keep session context preservation gated to the same provider/model or explicit configuration, and preserve Telegram progress draft semantics unless maintainers intentionally document and prove a standalone durable-progress migration.

## Reproduction Assessment

Yes, by source inspection: progress-mode Telegram tool events now reach `sendStandaloneToolProgress`, which uses the mirrored delivery path, and the helper can select stale fallback tokens. I did not run tests or a live Telegram probe because this review is read-only.

## Solution Assessment

No. The session-context idea needs tighter provider/model gating, and the Telegram progress transport change conflicts with current docs and maintainer review context unless it is treated as a deliberate migration with proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Stop mirroring progress into transcripts:** `extensions/telegram/src/bot-message-dispatch.ts:1198-1199`
  - body: `sendStandaloneToolProgress` spreads `deliveryBaseOptions`, so the existing `transcriptMirror` reaches `deliverReplies`. Since `deliverReplies` mirrors delivered text, progress lines like tool labels or commands become assistant transcript content and can affect later turns. Clear `transcriptMirror` for non-final progress or use a non-mirroring delivery path.
  - confidence: 0.94
- **[P1] Preserve label-only progress mode:** `extensions/telegram/src/bot-message-dispatch.ts:553-560`
  - body: Disabling draft lanes for all `progress` mode means `streaming.progress.toolProgress: false` now returns without sending standalone progress and without showing the documented label-only draft. Existing configs that intentionally hide tool lines lose their visible progress label, so keep a draft for this mode or make a documented migration.
  - confidence: 0.9
- **[P1] Suppress progress before final delivery awaits:** `extensions/telegram/src/bot-message-dispatch.ts:1272-1274`
  - body: The PR sets `streamToolProgressSuppressed` only after `sendPayload` returns for the final answer. A tool callback while final delivery is still pending can still enter the standalone progress branch and add progress around the final response; restore a delivery-started guard or suppress before awaiting final delivery.
  - confidence: 0.86
- **[P1] Ignore stale fallback tokens on model switches:** `src/agents/session-context-tokens.ts:75-78`
  - body: The helper adds `fallbackContextTokens` as a max candidate even when `sameRuntimeModel` rejects the existing entry. Callers pass the current entry context as that fallback, so switching from a large-context model to a smaller one can keep the old oversized context window and skew compaction/session state.
  - confidence: 0.87
