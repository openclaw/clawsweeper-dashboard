Full review comments:

- **[P1] Verify Homebrew before extracting it:** `Dockerfile:328`
  - body: This build step streams the Homebrew archive from GitHub straight into `tar`. Pinning the tag helps repeatability, but it does not detect a tampered archive before installing executable tooling into the image; add a pinned digest or equivalent checksum verification before extraction.
  - confidence: 0.91
- **[P1] Default the init bind sources:** `docker-compose.yml:13-14`
  - body: `openclaw-init` is now a required dependency for the gateway, but its config and workspace volumes do not use the fallback source expressions that gateway and CLI use. A plain `docker compose up` or documented manual Compose path without setup-exported variables can produce invalid bind mounts and prevent the gateway from starting.
  - confidence: 0.9
- **[P1] Restore the auth-profile secret mount:** `docker-compose.yml:69-71`
  - body: Current main mounts `OPENCLAW_AUTH_PROFILE_SECRET_DIR` into `/home/node/.config/openclaw` for both Docker services and setup preserves that directory outside the state mount. This branch drops that mount, so Docker users can lose the persistent auth-profile encryption key path on container replacement.
  - confidence: 0.9
- **[P1] Preserve current image package build args:** `Dockerfile:221-236`
  - body: Current main supports the Docker-neutral `OPENCLAW_IMAGE_APT_PACKAGES` path and the newer `OPENCLAW_IMAGE_PIP_PACKAGES` path, while this branch reverts the Dockerfile/setup surface to the older apt-only variable. Rebase those arguments forward so existing local-build settings do not become no-ops.
  - confidence: 0.86
