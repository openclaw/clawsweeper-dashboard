## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Verify the Homebrew archive before extraction:** `Dockerfile:328`
  - body: This still downloads the Homebrew tag archive and pipes it straight into `tar`. A tag name is not content verification, so a retagged or tampered archive would be accepted as executable runtime tooling in the image; add a pinned SHA256 or equivalent digest check before extraction.
  - confidence: 0.9
- **[P2] Default the init service volume sources:** `docker-compose.yml:13-14`
  - body: The new `openclaw-init` service uses `${OPENCLAW_CONFIG_DIR}` and `${OPENCLAW_WORKSPACE_DIR}` without the fallback expressions used by the gateway and CLI services. On the documented manual flow before setup has created `.env`, Compose sees empty volume sources for this service instead of falling back to `${HOME:-/tmp}/.openclaw`, so bare `docker compose` commands can fail during config parsing.
  - confidence: 0.86
