Full review comments:

- **[P1] Verify the Homebrew archive before extraction:** `Dockerfile:328`
  - body: This still downloads the Homebrew tag archive and pipes it directly into `tar`. A tag name is not content integrity, and this archive becomes executable runtime tooling in the image; add a pinned SHA256 or equivalent digest check before extraction.
  - confidence: 0.9
- **[P2] Default the init service volume sources:** `docker-compose.yml:13-14`
  - body: The new init service uses `${OPENCLAW_CONFIG_DIR}` and `${OPENCLAW_WORKSPACE_DIR}` without the fallback expressions used by gateway and CLI.   - confidence: 0.86
