## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Verify the Homebrew archive before extraction:** `Dockerfile:328`
  - body: The PR pins `HOMEBREW_BREW_TAG` but pipes the GitHub tag archive directly into `tar`. A tag name is not content integrity, so a compromised response or retagged archive would install executable runtime tooling in the image; add a pinned SHA256 or equivalent digest check before extraction.
  - confidence: 0.9
- **[P2] Default the init service volume sources:** `docker-compose.yml:13-14`
  - body: The new init service uses `${OPENCLAW_CONFIG_DIR}` and `${OPENCLAW_WORKSPACE_DIR}` without the fallback expressions used by the gateway and CLI mounts. The documented manual Docker flow can run before setup has written `.env`, so Compose can expand these to empty bind sources and fail before init or gateway starts; mirror the same defaults here.
  - confidence: 0.86
