## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Verify the Homebrew archive before extraction:** `Dockerfile:328`
  - body: This streams `HOMEBREW_BREW_TAG` from GitHub directly into `tar`. A tag is not content integrity, so a compromised response or retagged archive would install executable runtime tooling in the image; add a pinned SHA256 or equivalent digest check before extraction.
  - confidence: 0.9
- **[P2] Default the init service bind sources:** `docker-compose.yml:13-14`
  - body: The new init service uses `${OPENCLAW_CONFIG_DIR}` and `${OPENCLAW_WORKSPACE_DIR}` without the fallback expressions used by gateway and CLI. With those vars unset, Compose rejects the file with an empty bind source before manual/setup flows can start; mirror the same defaults here.
  - confidence: 0.9
