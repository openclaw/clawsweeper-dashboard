Keep open: current main still redacts `sessions_spawn.streamTo` to `__OPENCLAW_REDACTED__` even though the tool schema only accepts `"parent"`, so the reported validator failure remains source-backed and narrow.

## Best Possible Solution

Keep the ACP streaming contract and make `sessions_spawn` redaction schema-aware so safe enum routing fields do not become invalid sentinels while `resumeSessionId` and attachment content remain redacted.
