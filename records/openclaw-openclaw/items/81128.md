## Summary

Keep open: current main still has a source-backed mismatch where `sessions_spawn` accepts and forwards `streamTo: "parent"`, but the transcript/tool-call sanitizer rewrites that field to `__OPENCLAW_REDACTED__` before replay/persistence paths can reuse it.

## What This Changes

The issue reports intermittent invalidation of valid ACP `sessions_spawn` calls when `streamTo: "parent"` is rewritten by redaction before schema validation.

## Best Possible Solution

Keep the ACP streaming contract and make `sessions_spawn` redaction schema-aware so safe enum routing fields remain valid while sensitive resume IDs and attachment payloads stay redacted.

## Reproduction Assessment

Yes, at source level: a `sessions_spawn` tool-call block with `streamTo: "parent"` passes through `sanitizeToolCallInputs` and becomes `streamTo: "__OPENCLAW_REDACTED__"`. I did not run the live intermittent Telegram/ACP flow.

## Solution Assessment

Yes: the narrow maintainable fix is to stop redacting the non-secret `streamTo` enum in schema-visible tool-call plumbing while keeping `resumeSessionId` and attachment contents protected.
