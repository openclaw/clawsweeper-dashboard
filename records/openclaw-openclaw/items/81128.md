## Summary

Keep open: current main still has the reported contract mismatch, where `sessions_spawn` accepts and forwards `streamTo: "parent"` but the transcript/tool-call sanitizer rewrites that same field to `__OPENCLAW_REDACTED__`. This is a narrow source-backed bug with no implemented-on-main, duplicate, or stale-info close path.

## What This Changes

The issue asks OpenClaw to stop internal redaction from replacing `sessions_spawn` `streamTo: "parent"` with `__OPENCLAW_REDACTED__` before validation.

## Best Possible Solution

Make `sessions_spawn` redaction schema-aware so safe routing enums such as `streamTo: "parent"` are preserved or omitted in replay-visible payloads while sensitive resume IDs and attachment contents remain redacted.

## Reproduction Assessment

Yes, source-level: a `sessions_spawn` tool-call block containing `streamTo: "parent"` is deterministically sanitized to `"__OPENCLAW_REDACTED__"`, while the tool schema only accepts the legal enum value. I did not run the live Telegram/ACP flow.

## Solution Assessment

Yes, the best fix is narrow schema-aware sanitization for this field, not disabling transcript redaction broadly. The repair should keep `resumeSessionId` and attachment content protected and update the regression tests that currently expect the invalid sentinel.
