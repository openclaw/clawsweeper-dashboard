## Summary

Keep open: current main still has a source-backed `sessions_spawn` ACP redaction/replay mismatch, and the linked open fix PR explicitly closes this issue but has not merged yet.

## What This Changes

The issue reports that ACP `sessions_spawn` calls with `streamTo: "parent"` can lose parent streaming when transcript redaction rewrites or re-copies that field before replay.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/82203 after acceptable real ACP/provider-owned-thinking proof, keeping first-pass secret redaction while making already-redacted ACP fields idempotent.

## Reproduction Assessment

Yes, source-reproducible: current main re-copies already-redacted `resumeSessionId`/`streamTo`, and signed-thinking replay treats that sanitizer identity change as unsafe. The reporter also added five runtime failures over May 12-13, 2026, though I did not run a live ACP session here.

## Solution Assessment

Yes, the linked idempotency guard is the narrow maintainable fix because it preserves redaction of real values while avoiding identity changes for already-redacted values. The issue should remain open until that PR or an equivalent fix merges.
