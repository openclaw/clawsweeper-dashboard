## Summary

Keep open: current main still rewrites `sessions_spawn.streamTo` to `__OPENCLAW_REDACTED__` in sanitized tool-call blocks even though the tool schema accepts and forwards `streamTo: "parent"` for ACP spawns. The reporter also added fresh 2026-05-13 data showing 3/3 affected dispatches with no new stream log, so this remains a valid narrow bug rather than cleanup.

## What This Changes

The issue asks OpenClaw to stop rewriting legal `sessions_spawn` `streamTo: "parent"` tool arguments to the redaction sentinel before validation.

## Best Possible Solution

Make `sanitizeToolCallInputs` schema-aware for `sessions_spawn` so legal `streamTo: "parent"` survives tool-call plumbing while `resumeSessionId` and attachment content remain redacted, with focused regression coverage.

## Reproduction Assessment

Yes at source level: a `sessions_spawn` tool-call block with `streamTo: "parent"` passed through `sanitizeToolCallInputs` is deterministically rewritten to the sentinel, and current tests assert that behavior. I did not run a live Telegram/ACP dispatch in this read-only review.

## Solution Assessment

Yes: the narrow maintainable fix is to preserve the safe enum value in schema-visible tool-call arguments while keeping the existing transcript redaction for sensitive fields. Broadly disabling the sanitizer would be too risky.
