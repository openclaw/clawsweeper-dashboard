## Summary

Keep open: current `main` still has the reported contract mismatch. The ACP `sessions_spawn` tool accepts and forwards `streamTo: "parent"`, but the transcript/tool-call sanitizer still rewrites that field to `__OPENCLAW_REDACTED__` in `sessions_spawn` blocks, and fresh reporter data says the live stream log path is still being lost.

## What This Changes

The issue asks OpenClaw to preserve the legal `sessions_spawn({ runtime: "acp", streamTo: "parent" })` argument instead of rewriting it to the redaction sentinel before validation or execution.

## Best Possible Solution

Keep the ACP parent-stream contract and make the `sessions_spawn` sanitizer schema-aware so `streamTo: "parent"` remains valid while genuinely sensitive ACP fields and attachment contents stay redacted.

## Reproduction Assessment

No live high-confidence reproduction was run in this read-only pass. Yes for a source-backed reproduction path: current code deterministically rewrites `sessions_spawn.streamTo` to the redaction sentinel in sanitized tool-call blocks while the tool schema/executor only accept `"parent"`.

## Solution Assessment

Yes, the narrow maintainable fix is to stop redacting the safe `streamTo: "parent"` enum in schema-visible tool-call plumbing while continuing to redact `resumeSessionId` and attachment contents. Broadly disabling transcript hygiene would be the wrong repair.
