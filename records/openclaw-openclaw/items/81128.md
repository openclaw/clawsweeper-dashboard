## Summary

Keep open: current main still has the reported contract mismatch, because `sessions_spawn.streamTo` is documented and schema-accepted as `"parent"` but the transcript sanitizer still rewrites that field to `__OPENCLAW_REDACTED__`. The reporter’s fresh 5/5 failure data reinforces the source-backed root cause rather than making this stale cleanup.

## What This Changes

Fix `sessions_spawn({ runtime: "acp", streamTo: "parent" })` so transcript/tool-call sanitization does not rewrite the legal `streamTo` enum before validation or replay.

## Best Possible Solution

Make `sessions_spawn` redaction schema-aware: preserve or safely omit non-secret routing enums such as `streamTo`, while continuing to redact ACP resume IDs and attachment contents.

## Reproduction Assessment

Yes for a source-level reproduction: current main deterministically rewrites any `sessions_spawn.streamTo` key to the redaction sentinel in sanitizer paths that are used for replay. I did not run a live Telegram/ACP dispatch.

## Solution Assessment

Yes: the narrow maintainable fix is to stop replacing the legal `streamTo: "parent"` enum with a sentinel while preserving sensitive-field redaction. Broadly disabling transcript sanitization would be the wrong direction.
