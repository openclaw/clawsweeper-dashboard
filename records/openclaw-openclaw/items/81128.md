Keep open. Current main still has a source-level path that rewrites `sessions_spawn` `streamTo: "parent"` to the redaction sentinel, while the ACP tool contract treats that value as a valid non-secret enum.

## What This Changes

The issue reports intermittent rewriting of the valid `sessions_spawn({ runtime: "acp", streamTo: "parent" })` argument to `__OPENCLAW_REDACTED__` before validation.

## Best Possible Solution

Preserve legal `streamTo: "parent"` through sessions_spawn transcript/tool-call sanitization while continuing to redact `resumeSessionId` and attachment contents, backed by focused regression coverage.

## Reproduction Assessment

Yes for a source-level reproduction: current main's sanitizer rewrites `streamTo: "parent"` to the redaction sentinel, while the tool schema/docs treat `"parent"` as valid for ACP. I did not establish the full intermittent live Telegram/ACPX failure path.

## Solution Assessment

Yes: the narrow maintainable fix is to stop redacting the non-secret `streamTo` enum value while leaving secret-bearing ACP resume ids and attachment contents redacted. Bypassing validation or requiring callers to omit `streamTo` would lose the documented stream log behavior.
