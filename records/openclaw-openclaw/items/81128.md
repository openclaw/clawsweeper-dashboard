## Summary

Keep open: current `main` still has the reported contract mismatch. `sessions_spawn` accepts and forwards `streamTo: "parent"`, but the transcript sanitizer still rewrites that same field to `__OPENCLAW_REDACTED__` before replay/persistence paths, and no linked or searched PR currently fixes this issue.

## What This Changes

The issue reports that valid ACP `sessions_spawn` calls with `streamTo: "parent"` can be redacted to `__OPENCLAW_REDACTED__`, breaking schema validation and parent streaming.

## Best Possible Solution

Keep the ACP parent-streaming contract and make `sessions_spawn` transcript/tool-call sanitization schema-aware so `streamTo: "parent"` remains valid while sensitive resume IDs and attachment content stay redacted.

## Reproduction Assessment

Yes at source level: current `main` deterministically rewrites `sessions_spawn.streamTo` to the redaction sentinel in sanitized tool-call blocks, while the tool schema only accepts `parent`. I did not establish a live Telegram/ACP intermittent reproduction.

## Solution Assessment

Yes: the narrow maintainable fix is to stop redacting the non-secret `streamTo` enum in schema-visible tool-call plumbing while preserving redaction for `resumeSessionId` and attachments. Broadly disabling transcript redaction would be unsafe.
