## Summary

Keep open: current main and the latest release still show a source-backed mismatch between the ACP `sessions_spawn` contract and transcript replay sanitation, and there is no linked fix PR or duplicate that owns the remaining work.

## What This Changes

The issue asks OpenClaw to stop internal transcript redaction from turning legal ACP `sessions_spawn.streamTo: "parent"` arguments into `__OPENCLAW_REDACTED__` during replay/tool plumbing.

## Best Possible Solution

Make `sessions_spawn` ACP argument sanitation idempotent and schema-aware for safe routing enums while continuing to redact sensitive resume/session data, with regression coverage for provider-owned thinking replay.

## Reproduction Assessment

No live current-main Telegram/ACP reproduction was run in this read-only review. Source inspection gives a high-confidence reproduction path: already-redacted `sessions_spawn.streamTo` still produces a changed block, which provider-owned thinking replay treats as unsafe.

## Solution Assessment

Yes, the best fix is narrow: make the ACP redaction pass idempotent or otherwise avoid invalidating schema-visible safe enum fields while preserving redaction for sensitive values. Broadly disabling transcript redaction would be too risky.
