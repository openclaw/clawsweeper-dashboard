## Summary

## What This Changes

The PR changes `src/infra/outbound/internal-source-reply.ts` so the internal source-reply sink bypass can use a loaded channel plugin with `actions.handleAction` or `outbound` when outbound resolution returns no plugin.

## Best Possible Solution

Use one shared send-capability predicate for this sink escape, preserve the internal-source fallback for non-send action plugins, then add focused regression coverage and real external-channel proof.

## Reproduction Assessment

## Solution Assessment

No. The PR is in the right area, but the best fix should reuse the actual send-support contract instead of treating any `handleAction` or `outbound` object as sufficient.
