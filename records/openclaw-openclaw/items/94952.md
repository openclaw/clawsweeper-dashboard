## Summary

Keep open: this is a plausible narrow fix for the linked message-loss bug, but it is not merge-ready because the sink escape does not carry action-only capability through the full channel-selection path, also lets inactive outbound shells bypass the sink, and the supplied proof is only local tests.

## What This Changes

The PR broadens the internal source-reply sink predicate to treat loaded action-capable or outbound plugin channels as current-source delivery candidates and adds predicate-level Vitest coverage.

## Best Possible Solution

Keep the source-reply sink private unless a shared send-action-capable channel resolver can both select and execute the current plugin through `runMessageAction`, then cover that full path and attach redacted external-channel proof.

## Reproduction Assessment

## Solution Assessment

No. The PR is in the right area, but the best fix should use one send-action-capable current-plugin contract across the sink predicate and channel selection, keep inactive outbound shells on the sink, and prove the full runner path.
