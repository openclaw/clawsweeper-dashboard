## Summary

Keep open: current main still exposes the lowercased LINE recipient path, and the linked fix PR is open and unmerged. Because that PR explicitly closes this issue, the issue should stay open until the PR or an equivalent fix lands.

## What This Changes

The issue asks OpenClaw to preserve or canonicalize LINE recipient IDs for post-reply-token push and delivery-recovery paths so lowercased IDs or `line:*:` session-key prefixes are not sent as invalid LINE API recipients.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81670, or an equivalent LINE-owned send-boundary fix, after real LINE delivery proof and focused regression tests are available.

## Reproduction Assessment

Yes for the lowercased-recipient path: source inspection shows LINE peer IDs can be lowercased into session keys, cron fallback can copy that value to `delivery.to`, and recovery replays queued targets unchanged. No live LINE account was exercised in this review.

## Solution Assessment

Yes. A narrow LINE channel-boundary normalization fix is the best direction because LINE casing is plugin-owned behavior and core should remain plugin-agnostic.
