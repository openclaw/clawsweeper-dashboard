## Summary

Keep open: current main still has a credible LINE lowercased-recipient delivery bug, and https://github.com/openclaw/openclaw/pull/81670 is an open closing PR for the same issue, so this should wait for that PR to be reviewed and merged.

## What This Changes

The issue reports that LINE delivery recovery and post-reply-token push paths can send lowercased or still-prefixed recipients to the LINE Messaging API, causing silent push failures.

## Best Possible Solution

Keep this issue open and resolve it by landing a LINE plugin send-boundary normalization fix with real LINE behavior proof and focused regression coverage.

## Reproduction Assessment

Yes for the lowercase failure path by source inspection: LINE peer IDs can be lowercased into session keys, cron fallback can copy that fragment into delivery.to, and recovery replays entry.to unchanged into the LINE send path. I did not run a live LINE environment reproduction.

## Solution Assessment

Yes, a channel-owned LINE send-boundary fix is the narrowest maintainable direction because core should remain plugin-agnostic while the LINE plugin owns wire-format target normalization.
