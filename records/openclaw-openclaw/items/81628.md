## What This Changes

Fix LINE long-running and delivery-recovery sends so case-sensitive LINE chat IDs reach the Messaging API without lowercasing and without session-key prefixes.

## Best Possible Solution

Review and land one canonical fix that preserves case-exact LINE delivery context or adds a LINE-owned outbound guard while keeping core plugin-agnostic.

## Reproduction Assessment

Yes for source reproduction: current main lowercases peer IDs in session keys, cron fallback can copy that fragment into `delivery.to`, and recovery replays stored `entry.to`. No live LINE group run was performed in this read-only review.

## Solution Assessment

Yes, the narrow maintainable fix is to keep exact LINE recipients at the channel delivery boundary and avoid reconstructing unrecoverable LINE push targets from lowercased session-key fragments; the open linked PRs are the right place to finish that decision.
