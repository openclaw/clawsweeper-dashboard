## Summary

Keep open: current main still has a source-visible path that can lower-case LINE recipients during fallback delivery inference, and I found no merged PR or changelog entry that resolves this LINE-specific failure.

## What This Changes

The issue asks OpenClaw to preserve LINE recipient casing and strip `line:*:` prefixes for long-running reply recovery and post-reply-token push delivery.

## Best Possible Solution

Normalize LINE recipients at the channel-owned delivery boundary and preserve case-exact delivery context for durable recovery and session-key fallback paths, with focused LINE regression coverage.

## Reproduction Assessment

Yes at source level, but not with a live LINE run. Current main lowercases peer IDs in session keys and cron fallback can reuse that lowercased fragment as `delivery.to`, matching the reported failed push target.

## Solution Assessment

Yes, the narrow maintainable fix is routing/normalization coverage rather than a new setting. Keep core plugin-agnostic and use LINE-owned target normalization or stored case-preserving delivery context before retrying push sends.
