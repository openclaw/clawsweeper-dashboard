## Summary

Keep open. Current `main` and latest release still use a chokidar-backed skills watcher that includes `SKILL.md` files, and chokidar 5 registers file-level watchers for included files; the `SKILL.md` read helper itself closes its descriptor, so the remaining likely leak path is the watcher layer described by the report.

## What This Changes

Reports that the gateway's workspace skills watcher keeps one persistent read-only file descriptor per user `SKILL.md` on macOS.

## Best Possible Solution

Change the skills watcher so workspace skill roots still refresh on `SKILL.md` changes without opening persistent per-file watchers, then prove on macOS that descriptor count stays bounded.

## Reproduction Assessment

Yes for a source-level reproduction path, but not a live current-main `lsof` run in this review. Current code routes session snapshots through `ensureSkillsWatcher`, and chokidar registers file watchers for included `SKILL.md` files.

## Solution Assessment

Yes, the issue is aimed at the right layer: the direct `SKILL.md` read path closes its descriptor, while the watcher path remains capable of retaining one watcher per included file.
