Fix https://github.com/openclaw/openclaw/issues/40641. Current docs show unquoted bracket-notation config paths like `openclaw config set agents.list[0].tools.exec.node "node-id-or-name"`, which zsh/failglob-style shells treat as a glob and reject before OpenClaw receives argv. Update the English source docs in `docs/cli/config.md`, `docs/tools/exec.md`, and `docs/nodes/index.md` so bracket-notation CLI paths are shell-safe, either by quoting the full path or using dot-index notation where clearer; add a concise Paths note in `docs/cli/config.md` explaining that paths containing brackets must be quoted. Do not change `src/cli/config-cli.ts`, config schema behavior, runtime behavior, or generated/localized docs under `docs/<locale>/**`; `docs/AGENTS.md` says English docs are the source of truth and localization is handled by the publish/i18n pipeline. No changelog is expected for this docs-only correction unless a maintainer asks for one.
