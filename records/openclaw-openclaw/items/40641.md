Keep open: current main and v2026.5.12 still publish unquoted bracket-notation config examples, and the shell failure reproduces before OpenClaw receives the argument. ## What This Changes

The issue reports that the official docs show an unquoted `agents.list[0].tools.exec.node` config path that shells can reject as a glob before the CLI runs.

## Best Possible Solution

Update the English config, exec, and nodes docs to use shell-safe quoted bracket paths or dot-index paths, plus a short Paths note explaining when bracket paths need quoting.

## Solution Assessment
