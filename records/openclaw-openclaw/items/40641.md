## Summary

Keep open: current main and the latest release still publish the unquoted bracket-notation examples, and the shell-level failure is reproducible while the linked docs fix PR closed unmerged.

## What This Changes

The issue reports that the documented `openclaw config set agents.list[0].tools.exec.node "node-id-or-name"` command fails because shells treat the unquoted bracket path as a glob before OpenClaw receives it.

## Best Possible Solution

Update the English source docs with shell-safe path examples and a short Paths note that bracket notation must be quoted when used in shells.

## Reproduction Assessment

Yes. Current main documents the unquoted `agents.list[0].tools.exec.node` path, and `bash -O failglob` rejects that exact argument before OpenClaw receives argv; quoted bracket notation and dot-index notation pass through.

## Solution Assessment

Yes. A docs-only correction is the narrowest maintainable fix because the config parser already supports bracket notation after shell parsing.
