## Summary

Keep open: current main and the latest release still publish shell-unsafe unquoted bracket-notation config examples, and the shell failure is reproducible while the related docs PR is closed unmerged.

## What This Changes

The issue asks OpenClaw to fix official docs that show `openclaw config set agents.list[0].tools.exec.node ...`, which can fail in shells before OpenClaw receives the path.

## Best Possible Solution

Update the English config, exec, and nodes docs with shell-safe path examples and a short Paths note explaining that bracket notation must be quoted in shells, or use dot-index notation consistently.

## Reproduction Assessment

Yes. Current main documents the unquoted `agents.list[0].tools.exec.node` path, and `bash -O failglob` rejects that exact argument before OpenClaw receives argv; quoted bracket notation and dot-index notation pass through.

## Solution Assessment

Yes. A docs-only correction is the narrowest maintainable fix because the config parser already supports bracket notation after shell parsing, and the broken behavior is caused by shell glob expansion of the documented examples.
