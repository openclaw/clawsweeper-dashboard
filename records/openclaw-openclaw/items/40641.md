## Summary

Keep open: current main and v2026.5.7 still publish shell-unsafe unquoted bracket-notation config examples, the shell-level failure is reproducible, and the related docs fix PR is closed unmerged.

## What This Changes

The issue asks OpenClaw docs to stop recommending an unquoted `agents.list[0].tools.exec.node` config path that shells can treat as a glob.

## Best Possible Solution

Update the English config, exec, and nodes docs with shell-safe path examples and a short Paths note that bracket notation must be quoted, or use dot-index notation consistently.

## Reproduction Assessment

Yes. Current main documents the unquoted `agents.list[0].tools.exec.node` path, and `bash -O failglob` rejects that exact argument before OpenClaw receives argv while quoted bracket notation passes through.

## Solution Assessment

Yes. A docs-only correction is the narrowest maintainable fix because the config parser already accepts bracket paths after argv parsing; the broken behavior is shell glob expansion of the documented examples.
