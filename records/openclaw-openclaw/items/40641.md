## Summary

Keep open: current main and v2026.5.7 still publish shell-unsafe unquoted bracket-notation config examples, and the shell-level failure is reproducible while the related docs PR remains closed unmerged.

## What This Changes

This issue asks to fix OpenClaw docs examples that show unquoted `agents.list[0]...` config paths and fail in shells that treat brackets as globs.

## Best Possible Solution

Update the English config, exec, and nodes docs with shell-safe path examples and a short Paths note explaining that bracket notation must be quoted in shells, or use dot-index notation consistently.

## Reproduction Assessment

Yes. Current main documents the unquoted `agents.list[0].tools.exec.node` path, and `bash -O failglob` rejects that exact argument before OpenClaw receives argv while quoted bracket notation and dot-index notation pass through.

## Solution Assessment

Yes. A docs-only correction is the narrowest maintainable fix because the config parser already supports bracket paths after argv parsing, and the broken behavior is caused by shell glob expansion of the documented examples.
