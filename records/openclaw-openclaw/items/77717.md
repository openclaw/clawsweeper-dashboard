## Summary

Keep open. Current main still has the reported Feishu identity-recovery race at source level: an aborted lifecycle signal can return an empty bot identity, write it as unknown, and suppress the only background recovery path; the related open Feishu PRs cover adjacent startup serialization or latency work rather than this abort-gated recovery skip.

## What This Changes

The issue reports that Feishu WebSocket bot identity recovery can be skipped when a lifecycle abort arrives before the identity probe resolves, leaving the channel with an unknown bot identity.

## Best Possible Solution

Fix the Feishu plugin monitor startup/recovery path so an aborted old lifecycle cannot write or strand an unknown bot identity without a retry in the active lifecycle.

## Reproduction Assessment

Yes, at source level. Current main has a concrete path where fetchBotIdentityForMonitor returns an empty identity for an aborted signal and monitorSingleAccount then skips startBotIdentityRecovery because that same signal is aborted.

## Solution Assessment

Yes. The narrow maintainable direction is a Feishu-local lifecycle/recovery fix that preserves gateway abort shutdown semantics while ensuring unknown bot identity still has a current-lifecycle retry or is not written as a permanent unknown state.

## Repair Work Prompt

Fix the Feishu WebSocket bot identity recovery race reported in https://github.com/openclaw/openclaw/issues/77717. On current main, monitorSingleAccount applies an empty identity from fetchBotIdentityForMonitor and skips startBotIdentityRecovery when the lifecycle abortSignal is already aborted, leaving botOpenIds[accountId] as unknown with no retry. Keep the fix Feishu-local in the monitor startup/recovery path; preserve gateway lifecycle abort semantics and do not broaden into proxy, timeout, health-monitor, or startup parallelization policy changes. Add focused regression coverage proving an aborted stale probe cannot permanently strand unknown bot identity without a retry/current-lifecycle recovery path. Coordinate conceptually with, but do not depend on, the overlapping Feishu startup PRs https://github.com/openclaw/openclaw/pull/63481 and https://github.com/openclaw/openclaw/pull/67466.
