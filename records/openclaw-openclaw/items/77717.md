## Summary

Keep open. Current main and the latest release still contain the abort-gated Feishu identity recovery path, and the open linked fix PR is a candidate rather than a merged fix.

## What This Changes

The issue reports a Feishu WebSocket lifecycle race where an aborted startup identity probe leaves bot open_id unknown and skips recovery, causing dropped messages until restart.

## Best Possible Solution

Land a lifecycle-safe Feishu monitor fix through the linked candidate PR or a replacement, with redacted restart/reload proof or an explicit maintainer proof override, then close this issue from the merged fix.

## Reproduction Assessment

Yes at source level: an aborted Feishu identity probe can return an empty identity, current main applies it as unknown, and the recovery call is gated off by the already-aborted signal. I did not run a live Feishu tenant restart.

## Solution Assessment

Yes on owner boundary: the fix belongs in the Feishu monitor identity lifecycle, not in core gateway policy or a new config knob. The linked PR is only a candidate until it preserves lifecycle cancellation and supplies real reload proof or a maintainer proof override.
