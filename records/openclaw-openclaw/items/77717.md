## Repair Work Prompt

Fix the Feishu bot identity recovery race in https://github.com/openclaw/openclaw/issues/77717. Current main applies an empty bot identity, logs unknown, and calls startBotIdentityRecovery only when !abortSignal?.aborted; fetchBotIdentityForMonitor also returns {} for aborted probes, and the recovery loop exits on the same abort signal. Implement a Feishu-local lifecycle-safe repair so an unknown bot identity after startup/reload schedules recovery for the active monitor lifecycle, while a stopped lifecycle cannot keep retrying or overwrite replacement identity state. Do not add config/env options, do not move Feishu-owned behavior into core gateway code, and do not preserve a stale fallback path. Add focused regression coverage for a missing botOpenId with an already-aborted startup signal and for normal shutdown cleanup/cancellation.
