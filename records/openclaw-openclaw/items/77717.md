## Label Justifications

- `P1`: The issue describes a source-backed Feishu channel workflow failure where accounts can stop receiving messages until a gateway restart.
- `impact:message-loss`: The reported failure mode is silently dropped Feishu direct messages when bot identity remains unknown and recovery is not scheduled.

## Summary

Keep open. Current main and the latest release still contain the abort-gated Feishu bot identity recovery path, the reporter added a recent reproduction on 2026.5.22, and the related Feishu startup/concurrency items do not close this abort-specific recovery gap.

## What This Changes

The issue requests a Feishu WebSocket lifecycle fix so an aborted bot identity probe cannot leave an account with unknown bot open_id and no retry path.

## Best Possible Solution

Make Feishu identity recovery lifecycle-safe: stale aborted probes should not overwrite active identity state, and unknown identity in an active per-account lifecycle should schedule retry without reusing an already-aborted signal.

## Reproduction Assessment

Yes, at source level. Gateway startAccount passes a lifecycle abort signal into the per-account Feishu monitor, fetchBotIdentityForMonitor can return an empty identity after abort, and monitorSingleAccount then skips startBotIdentityRecovery because the same signal is aborted.

## Solution Assessment

Yes, the maintainable direction is a focused Feishu-local lifecycle/recovery fix rather than a new config option or broad gateway restart rewrite. The repair should preserve shutdown semantics while preventing stale aborted identity probes from suppressing recovery for the active lifecycle.

## Repair Work Prompt

Fix the Feishu bot identity recovery race reported in https://github.com/openclaw/openclaw/issues/77717. In the gateway per-account path, an abort delivered while fetchBotIdentityForMonitor is pending can produce an empty identity, apply unknown state, and skip startBotIdentityRecovery because the lifecycle abort signal is already aborted. Keep the fix inside the Feishu plugin unless source review proves a gateway lifecycle seam is required. Preserve shutdown semantics: do not keep retry work alive for a deliberately stopped old lifecycle, but ensure stale aborted probes cannot overwrite active identity state or suppress recovery for the new/current lifecycle. Do not add config options or environment variables. Coordinate conceptually with https://github.com/openclaw/openclaw/issues/63475 and https://github.com/openclaw/openclaw/pull/63481, but do not broaden this into startup serialization, proxy handling, health monitor policy, or unrelated Feishu changes. Add focused regression coverage for the per-account/gateway path where the identity probe resolves empty after abort and verify recovery/state behavior. Changelog is release-owned; do not edit CHANGELOG.md.
