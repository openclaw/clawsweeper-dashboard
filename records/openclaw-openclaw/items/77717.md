## Summary

Keep open. Current main still has the reported Feishu identity-recovery race at source level: an aborted lifecycle signal can produce an empty bot identity and then suppress the background recovery path, while related open Feishu PRs only cover adjacent startup serialization or fresh-probe behavior.

## What This Changes

The issue asks Feishu WebSocket monitoring to recover bot identity after startup or reload abort races instead of leaving accounts with an unknown bot open_id and no retry path.

## Best Possible Solution

Keep this issue open and fix the Feishu plugin identity startup/recovery path so an already-aborted lifecycle signal cannot permanently clear or strand bot identity without a retry, with focused regression coverage.

## Reproduction Assessment

Yes, at source level. Current main has a concrete path where an aborted lifecycle signal makes fetchBotIdentityForMonitor return an empty identity and monitorSingleAccount then skips startBotIdentityRecovery because that same signal is aborted.

## Solution Assessment

Yes, the best direction is a narrow Feishu-plugin fix that preserves gateway lifecycle abort semantics while ensuring unknown bot identity still has a current-lifecycle retry or is not written as a permanent unknown state. The adjacent open PRs do not supersede this specific abort race.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/77717 in the Feishu plugin only. Current main can apply an empty bot identity after an aborted startup/reload probe and then skip `startBotIdentityRecovery` because the same lifecycle `abortSignal` is already aborted. Preserve gateway channel lifecycle semantics: do not detach `monitorSingleAccount` from the gateway `startAccount` promise, do not broaden channel startup policy, and do not add config knobs. Implement the smallest Feishu-owned change that prevents a reload/abort race from leaving `botOpenIds` permanently unknown without a retry, either by avoiding state-clearing on aborted probes or by scheduling recovery only under the current live lifecycle. Add focused regression coverage for the identity probe returning unknown after abort and for recovery/retry behavior under a non-aborted current lifecycle. Check open adjacent work at https://github.com/openclaw/openclaw/pull/70471, https://github.com/openclaw/openclaw/pull/63481, and https://github.com/openclaw/openclaw/pull/67466 for conflicts, but do not fold in their broader startup/proxy/comment changes.
