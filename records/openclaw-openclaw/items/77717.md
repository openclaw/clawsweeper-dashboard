## Summary

Keep open: current main still gates Feishu bot identity recovery on the same aborted lifecycle signal, and existing Feishu behavior fails closed when bot identity is unknown.

## What This Changes

Feishu WebSocket accounts can lose bot identity recovery when a config reload or restart aborts the identity probe before recovery is scheduled.

## Best Possible Solution

Fix Feishu’s plugin-local identity lifecycle so an active account cannot remain with unknown bot identity and no retry, while explicit stop/reload cleanup still cancels stale monitors.

## Reproduction Assessment

No live high-confidence reproduction was run. The issue is source-reproducible by aborting the Feishu identity probe before it resolves and observing that the current code applies an unknown identity and skips `startBotIdentityRecovery`.

## Solution Assessment

Yes, a narrow Feishu plugin fix is the right direction. The implementation should avoid reviving explicitly stopped stale monitors while ensuring the active replacement lifecycle schedules or performs identity recovery.
