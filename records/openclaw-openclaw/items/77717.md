## Summary

Keep open. Current main and latest release still contain the Feishu abort-gated bot identity recovery path, and the linked fix PR is open but not merged, so the issue remains necessary until a lifecycle-safe fix lands.

## What This Changes

The issue reports a Feishu WebSocket restart race where an aborted startup identity probe leaves bot open_id unknown and prevents background recovery from being scheduled.

## Best Possible Solution

Land a Feishu-local, lifecycle-owned identity recovery fix with focused regression coverage and redacted live or runtime-log proof, then close this issue from the merged PR.

## Reproduction Assessment

Yes, at source level: the gateway passes a lifecycle abort signal into Feishu startAccount, fetchBotIdentityForMonitor can return an empty identity after abort, and monitorSingleAccount skips startBotIdentityRecovery when that signal is aborted. I did not run a live Feishu tenant restart.

## Solution Assessment

Yes in direction: the maintainable fix is a focused Feishu lifecycle/recovery change, not a new config option or broad gateway rewrite. The open PR is a candidate path but needs lifecycle ownership and proof resolved before it can close the issue.
