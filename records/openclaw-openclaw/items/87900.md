  - body: `resolveTarget` trusts `targetWorkspaceDir` from the model-facing stage tool before checking collected surfaces. That lets a staged suggestion store its JSONL record and later apply writes under an arbitrary workspace/path, even though the feature is supposed to target the supported durable memory surfaces only.
  - confidence: 0.93
- **[P1] Require hashes for range-changing suggestions:** `extensions/memory-core/src/memory-audit.ts:495`
  - body: For edit/delete/move, staging records `hash: input.source.hash ?? ""`, and apply only checks the hash when it is truthy. A tool call that omits `sourceHash` can therefore delete or replace a stale line range without the conflict guard promised by the docs.
  - confidence: 0.96
