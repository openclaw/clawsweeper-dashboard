## Summary

Keep open. Current main still has a source-level path where a configured Feishu channel can become a `feishu` plugin auto-enable candidate and be written into `plugins.allow` before a Feishu plugin manifest owner is known, matching the reported recurring validation warning.

## What This Changes

The issue asks OpenClaw to stop treating a configured Feishu channel as a missing plugin allowlist entry during gateway startup.

## Best Possible Solution

Require a discovered plugin manifest owner before channel-derived ids are written to `plugins.allow`, while preserving allowlisting for an installed Feishu plugin manifest.

## Reproduction Assessment

Yes at source level: a `channels.feishu` config with a restrictive allowlist can produce candidate `feishu`, persist it to `plugins.allow`, and later warn when the registry has no known Feishu plugin id. I did not run a live Windows gateway restart.

## Solution Assessment

Yes for a narrow fix, but not as a literal ban on `feishu` in `plugins.allow`. The maintainable solution is to skip allowlist mutation for catalog-only or unclaimed channel ids while keeping installed Feishu plugin behavior intact.
