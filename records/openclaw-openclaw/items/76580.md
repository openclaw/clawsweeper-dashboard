## Summary

Keep open. Current main still has a source-level path where a configured Feishu channel can be converted into a `feishu` plugin auto-enable candidate, persisted into `plugins.allow`, and then reported by plugin validation when no Feishu plugin manifest is installed.

## What This Changes

The issue asks OpenClaw to stop Gateway startup/wizard auto-enable logic from adding the Feishu channel id to `plugins.allow` when Feishu is configured only as a channel.

## Best Possible Solution

Require a discovered or installed plugin manifest owner before channel-derived ids are written to `plugins.allow`, while keeping installed Feishu and real third-party channel plugin setup working.

## Reproduction Assessment

Yes, at source level. A config with `channels.feishu.defaultAccount` counts as configured, can produce a `feishu` auto-enable candidate with an empty manifest registry, and the startup path can persist that id before validation warns because it is not a known plugin id.

## Solution Assessment

Yes for a narrow code fix, but not the literal “never add feishu” wording. The maintainable solution is to skip allowlist mutation for catalog-only or unclaimed channel ids while preserving allowlisting for an actually installed Feishu plugin manifest.
