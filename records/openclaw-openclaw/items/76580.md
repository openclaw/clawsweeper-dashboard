## Summary

Keep open. Current main still has a source-level path where a configured Feishu channel can be treated as a plugin auto-enable candidate and persisted into `plugins.allow` before an installed Feishu plugin manifest is present, which matches the reported recurring validation warning.

## What This Changes

The issue asks for Gateway auto-enable logic to stop adding a configured Feishu channel id to `plugins.allow` when Feishu is not an installed plugin.

## Best Possible Solution

Require an installed or discovered plugin manifest owner before channel-derived ids are written to `plugins.allow`, while preserving allowlisting for installed Feishu and other real channel plugin manifests.

## Reproduction Assessment

Yes at source level: a config with `channels.feishu` and a restrictive `plugins.allow` can produce a `feishu` auto-enable candidate, persist it to the allowlist, and then hit the missing-plugin validation warning when no Feishu plugin manifest is installed.

## Solution Assessment

Yes for a narrow code fix, but not for the literal “never add feishu” wording. The maintainable fix is to skip allowlist mutation for catalog-only or unclaimed channel ids, while still allowlisting a real installed Feishu plugin manifest.
