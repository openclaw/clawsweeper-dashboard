## Summary

Keep open. Current main still has a source-level path where a configured catalog-only Feishu channel can be materialized into `plugins.allow` before an installed Feishu plugin manifest exists, which matches the reported recurring validation warning in packaged installs.

## What This Changes

The issue asks OpenClaw to stop Gateway startup from adding the Feishu channel id to `plugins.allow` when Feishu is not an installed plugin.

## Best Possible Solution

Make plugin auto-enable skip `plugins.allow` mutations for catalog-only channel ids until an installed/discovered plugin manifest owns that channel, while keeping installed bundled/external channel plugins allowlisted as needed.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes at source level: a packaged config with `channels.feishu` and a restrictive `plugins.allow` can follow the catalog-only channel candidate path into `plugins.allow`, then validation warns because no Feishu plugin manifest is installed.

## Solution Assessment

Is this the best way to solve the issue? No for the literal “never add feishu” wording, because an installed Feishu plugin may need allowlisting; the maintainable fix is to require a real manifest/discovered plugin owner before writing a channel id into `plugins.allow`.
