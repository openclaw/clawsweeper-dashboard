## What This Changes

The PR imports `isApprovalNotFoundError`, suppresses approval-not-found followup dispatch failures in `sendExecApprovalFollowupResult`, adds a focused regression test, and records the fix in the changelog.

## Best Possible Solution

Land the narrow classifier guard and regression test after normal maintainer approval, while keeping broader restart false-failure and followup-delivery reliability work in the separate linked issues and PRs.

## Reproduction Assessment

Yes, at source level. On current main, injecting a `sendExecApprovalFollowup` rejection with `Error("unknown or expired approval id")` into `sendExecApprovalFollowupResult` reaches the generic warning path, while the existing classifier already marks that error class as approval-not-found.

## Solution Assessment

Yes. Reusing the existing approval-not-found classifier before the warning/dedupe path is the narrowest maintainable fix for this log-noise case and preserves warnings for real channel, transport, or configuration failures.
