## Summary

Keep open: current main still sends approval-not-found followup failures through the generic warning path, and this active PR provides a narrow, source-backed fix with sufficient terminal proof and no blocking review findings.

## What This Changes

The PR imports `isApprovalNotFoundError` in `sendExecApprovalFollowupResult`, suppresses that specific followup failure, adds focused regression coverage, and records the fix in `CHANGELOG.md`.

## Best Possible Solution

Land the narrow classifier guard and regression test after normal required checks, while leaving broader restart false-failure and followup-delivery reliability work to the linked follow-up issues and PRs.

## Reproduction Assessment

Yes, at source level. On current main, a `sendExecApprovalFollowup` rejection with `Error("unknown or expired approval id")` reaches the generic warning path, while the existing classifier already identifies that condition as approval-not-found.

## Solution Assessment

Yes. Reusing the existing approval-not-found classifier before the warning/dedupe path is the narrowest maintainable fix for this log-noise class, and it preserves warnings for real transport or channel failures.
