## Summary

Keep open: current main still logs approval-not-found followup dispatch failures, and this PR applies the existing benign-error classifier at the narrow warning boundary with a focused regression test and sufficient terminal proof. I found no blocking correctness or security issue in the diff.

## What This Changes

The PR imports the existing approval-not-found classifier, suppresses expired exec approval followup warning logs, adds a regression test, and records a changelog entry.

## Best Possible Solution

Land this classifier guard with its regression test and changelog, while leaving the broader restart false-failure investigation to https://github.com/openclaw/openclaw/issues/66675 and other followup-delivery work to their own threads.

## Reproduction Assessment

Yes, at source level. On current main, an injected `sendExecApprovalFollowup` rejection with `Error("unknown or expired approval id")` reaches the warning path, while the existing classifier already marks that error benign.

## Solution Assessment

Yes. Reusing the existing approval-not-found classifier before the warning/dedupe path is the narrowest maintainable fix for this log-noise class, and it keeps real transport failures on the existing warning path.
