## Best Possible Solution

Land the narrow classifier guard and regression test after normal maintainer merge handling, while leaving broader restart and delivery reliability fixes to the linked follow-up work.

## Solution Assessment

Yes. Reusing the existing approval-not-found classifier before the warning/dedupe path is the narrowest maintainable fix for this log-noise class and preserves warnings for real transport or channel failures.
