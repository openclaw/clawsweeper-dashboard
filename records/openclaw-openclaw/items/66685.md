The PR suppresses approval-not-found errors in exec approval followup result dispatch, adds a focused regression test, and records the fix in the changelog.

## Best Possible Solution

Land the narrow classifier guard and regression test, while leaving the broader gateway restart false-failure and followup-delivery reliability work to the linked follow-up items.

## Solution Assessment

Yes. Reusing the existing approval-not-found classifier before the warning/dedupe path is the narrowest maintainable fix for this log-noise class, and it preserves warnings for real transport or channel failures.
