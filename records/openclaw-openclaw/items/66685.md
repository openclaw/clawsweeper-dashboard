## Summary

Keep open: current main still logs expired approval followup dispatch failures, and this PR is a focused, non-obsolete fix with no blocking review findings.

## What This Changes

The PR imports `isApprovalNotFoundError` into `sendExecApprovalFollowupResult`, suppresses approval-not-found followup dispatch failures, adds a focused regression test, and adds a changelog entry.

## Best Possible Solution

Land this narrow classifier guard with its regression test after required checks finish, while leaving the broader restart false-failure investigation to https://github.com/openclaw/openclaw/issues/66675 and separate followup delivery reliability work to their own threads.

## Reproduction Assessment

Yes, at source level. On current main, an injected `sendExecApprovalFollowup` rejection with `Error("unknown or expired approval id")` reaches the warning path, while the existing classifier already marks that error benign.

## Solution Assessment

Yes. Reusing the existing approval-not-found classifier before the warning/dedupe path is the narrowest maintainable fix for this log-noise class, and it keeps real transport failures on the existing warning path.
