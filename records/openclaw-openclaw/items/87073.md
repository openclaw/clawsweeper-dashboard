## Summary

Keep open for maintainer judgment: current main and v2026.6.6 still lack the requested version-keyed root dependency sentinel or `openai`/`typebox` entrypoint validation, while existing guards cover package dist/manifest checks and documented startup remains verify-only.

## What This Changes

The issue requests a dep-guard pattern that validates root npm dependencies with a version-keyed sentinel and wipes/reinstalls a corrupt `node_modules` tree before Gateway startup.

## Best Possible Solution

Choose an explicit recovery boundary, likely update, doctor, installer documentation, release/package verification, or a supervised startup wrapper, and keep ordinary Gateway startup from silently deleting or reinstalling root dependency state unless maintainers approve that policy.

## Reproduction Assessment

No. The issue gives plausible real-world OpenClaw 2026.5.22 evidence, and current source shows the requested guard is absent, but I did not reproduce npm corrupting root dependency entrypoints on current main.

## Solution Assessment

No for the exact default startup reinstall pattern. A sentinel may be useful, but current docs and code keep package-manager mutation in explicit install/update/doctor paths, so the maintainable fix needs an approved owner boundary first.
