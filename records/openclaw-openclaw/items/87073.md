## Summary

Keep open: current main and the latest release still lack the requested version-keyed root dependency sentinel or `openai`/`typebox` entrypoint corruption probe. Existing code verifies packaged `dist` inventory and plugin dependency repair in explicit update/doctor paths, so automatic root `node_modules` self-healing still needs a maintainer product-boundary decision.

## What This Changes

The issue requests a version-keyed dependency sentinel plus entrypoint corruption checks that wipe and reinstall root `node_modules` after npm ENOTEMPTY-style corruption.

## Best Possible Solution

Maintainers should choose the owner boundary for root dependency-tree integrity, likely explicit install/update verification or doctor repair, before adding any default startup-time package-manager mutation.

## Reproduction Assessment

No. Source inspection confirms the requested guard is absent, but this read-only review did not reproduce npm corrupting root dependency entrypoints on current main.

## Solution Assessment

No for the exact default startup reinstall pattern. A sentinel may be useful, but current code and docs keep package-manager mutation in explicit install/update/doctor paths, so the maintainable fix needs an approved owner boundary first.
