## Summary

Keep open: current main and the latest release still provide package/dist verification and explicit install/update/doctor repair boundaries, but they do not implement the requested version-keyed root dependency sentinel or `openai`/`typebox` entrypoint corruption check. The remaining question is whether OpenClaw should own automatic root dependency-tree self-healing and which lifecycle should perform it.

## What This Changes

The issue requests a version-keyed dependency sentinel plus corruption detection and full reinstall recovery for npm `node_modules` corruption before Gateway startup.

## Best Possible Solution

Choose an explicit owner boundary for root dependency-tree repair, likely update/install verification, doctor repair, installer docs, or a supervised service wrapper, before adding startup-time reinstall behavior.

## Reproduction Assessment

No. Source inspection confirms the requested guard is absent, but this read-only review did not reproduce npm corrupting root dependency entrypoints on current main.

## Solution Assessment

No for the exact default startup reinstall pattern. A sentinel may be useful, but current code and docs keep package-manager mutation in explicit install/update/doctor paths, so the maintainable fix needs an approved owner boundary first.
