## Summary

Keep open: current main and the latest release still cover package/dist and plugin dependency integrity, but not the requested version-keyed root dependency sentinel or `openai`/`typebox` entrypoint corruption check. The remaining question is where, if anywhere, OpenClaw should own automatic root dependency-tree repair.

## What This Changes

The issue requests a built-in dependency guard that detects corrupted npm-installed dependency trees with a version-keyed sentinel and recovers via full reinstall.

## Best Possible Solution

Choose an explicit recovery boundary, likely update, doctor, installer documentation, release/package verification, or a supervised startup wrapper, and avoid ordinary Gateway startup silently deleting or reinstalling root dependency state unless maintainers approve that policy.

## Reproduction Assessment

No. Source inspection confirms the requested guard is absent, but this read-only review did not reproduce npm corrupting root dependency entrypoints on current main.

## Solution Assessment

No for the exact default startup reinstall pattern. A sentinel may be useful, but current code and docs keep package-manager mutation in explicit install/update/doctor paths, so the maintainable fix needs an approved owner boundary first.
