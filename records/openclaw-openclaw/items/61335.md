## Best Possible Solution

Rebase the PR onto current main, apply monotonic next-hour-boundary stepping inside the Usage hour-slice walker and selected-hour span fallback, preserve `sessionTouchesSelectedHours` token-bucket-first behavior, and add current-main DST coverage plus real Control UI proof.
