## Summary

Keep open. The PR addresses a real DST hour-boundary bug on current main, but the branch still routes selected-hour filtering through a span-only helper that would drop current main's token-bucket-first filtering, and the external PR only provides test/CI proof rather than after-fix real behavior proof.

## What This Changes

The PR replaces Usage hour slicing with monotonic next-hour-boundary stepping, routes selected-hour filtering through a shared span helper, and adds node-side DST regression tests.

## Best Possible Solution

Keep the monotonic DST-safe boundary helper, but integrate it into current main's hour-slice and selected-hour fallback walkers while preserving token-bucket-first filtering and focused regression coverage.

## Reproduction Assessment

Yes. Current main still uses the `setToHourEnd` cursor path, and a Sydney DST fallback Node probe shows the old local hour-end calculation resolving one millisecond before the cursor.

## Solution Assessment

No. Monotonic next-hour stepping is the right underlying fix, but this branch should keep `sessionTouchesSelectedHours` token-bucket semantics and make only the fallback span walker DST-safe.
