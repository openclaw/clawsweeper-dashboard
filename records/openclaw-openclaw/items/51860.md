## Summary

Keep open. Current main still manages the macOS gateway as a per-user LaunchAgent and only treats LaunchDaemons as best-effort extra-service discovery, so the boot-without-login install/status request is not implemented.

## What This Changes

The issue requests a macOS system LaunchDaemon mode for gateway install/status so the gateway can start at boot before user login and report system-domain services correctly.

## Best Possible Solution

Design an explicit macOS system-service mode that installs and manages /Library/LaunchDaemons/ai.openclaw.gateway.plist with matching status/start/stop/restart/uninstall/doctor/docs support, coordinated with #14251.

## Reproduction Assessment

Yes. The reporter's steps are concrete, and current-main source inspection shows install/status/lifecycle still target the LaunchAgent path and GUI launchd domain while LaunchDaemon discovery is only a --deep extra-service scan.

## Solution Assessment

Yes in direction, but it needs maintainer design. An explicit opt-in system mode is safer than automatic detection because it changes privilege, ownership, launchd domain behavior, and service secret handling.
