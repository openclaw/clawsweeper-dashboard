## Summary

Keep open. Current main still manages the macOS gateway as a per-user LaunchAgent, while LaunchDaemon support is only best-effort discovery for external services and the requested boot-without-login install mode remains unimplemented.

## What This Changes

The issue requests macOS gateway LaunchDaemon support for boot-before-login startup plus status/lifecycle awareness for system-level plists.

## Best Possible Solution

Design and implement an explicit opt-in macOS system-service mode that installs /Library/LaunchDaemons with UserName, durable TMPDIR/env handling, matching status/start/stop/restart/uninstall/doctor support, docs, and regression coverage.

## Reproduction Assessment

Yes, by source inspection rather than a local macOS reboot test: current main has no system install option and always selects the LaunchAgent/gui-domain manager for Darwin. The reporter's boot-without-login path remains unsupported by the managed install surface.

## Solution Assessment

Yes in direction, but not as automatic detection. An explicit opt-in LaunchDaemon mode is the safer maintainable path because this changes privilege, UserName, TMPDIR, launchd domain, and secret/env handling semantics.
