## Summary

Keep open. Current main and v2026.6.10 still manage the macOS gateway service as a user LaunchAgent, while system LaunchDaemon handling is only best-effort discovery and the requested boot-before-login install/status mode remains unresolved.

## What This Changes

The issue asks for macOS gateway install/status support for system LaunchDaemons so the gateway can start at boot before user login and report that mode accurately.

## Best Possible Solution

Define one explicit macOS user/system launchd scope contract for install, status, doctor, lifecycle, docs, and tests, keeping the user LaunchAgent default while adding an audited LaunchDaemon path or clear external-supervisor guidance.

## Reproduction Assessment

Yes at source level: current main still installs and reports the managed macOS gateway as a user LaunchAgent in the gui/<uid> launchd domain. I did not run a live macOS reboot or LaunchDaemon install in this read-only review.

## Solution Assessment

No, not as a small one-off flag only. The maintainable solution is a shared launchd user/system scope model covering install, status, lifecycle, doctor, docs, tests, and privilege/security behavior.
