## Summary

Keep open: current main still treats the macOS gateway service as a per-user LaunchAgent, and there is no supported LaunchDaemon install mode. The stale TMPDIR subproblem appears improved on main, but the boot-before-login install/status lifecycle remains unresolved and needs maintainer design.

## What This Changes

The issue requests macOS gateway install/status support for a system LaunchDaemon so the gateway can start before user login.

## Best Possible Solution

Add a maintainer-approved macOS system LaunchDaemon mode and update status/lifecycle commands to understand both gui and system launchd domains, with durable TMPDIR, docs, and coordination with #14251.

## Reproduction Assessment

Source-reproducible, not live-reproduced: current main has no --system/--daemon install option and the Darwin service path is LaunchAgent/gui-domain only. I did not run a macOS boot-without-login test in this checkout.

## Solution Assessment

No for an automatic narrow fix: adding privileged LaunchDaemon support is a new CLI/service mode with security, ownership, and lifecycle policy decisions. The narrower TMPDIR issue is partly fixed on current main, but the requested system launchd behavior still needs design and implementation.
