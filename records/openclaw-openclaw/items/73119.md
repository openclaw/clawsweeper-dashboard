## Summary

Keep open because current main still has the transcript rewrite path that branches at the first replacement and re-appends the active suffix. The `sessions_history` tool itself is read-only and bounded, but large history results can still enter the tool-result truncation path that causes raw JSONL replay duplication.

## What This Changes

Prevent large `sessions_history`/tool-result maintenance from writing duplicate raw JSONL transcript entries while preserving correct billing and model context behavior.

## Best Possible Solution

Keep this open until #69486, #72511, or an equivalent maintained fix lands and is verified against the raw JSONL `sessions_history` trigger. The end state should keep `sessions_history` read-only and bounded while preventing low-value suffix replay or safely cleaning abandoned rewrite entries without breaking DAG ancestry.
