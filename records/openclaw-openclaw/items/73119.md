## Summary

Keep open: current main still routes large tool-result truncation through branch-and-suffix transcript rewrite, so raw JSONL can still gain replayed duplicate rows even though `sessions_history` itself is read-only and chat history now follows the active branch.

## What This Changes

The issue asks OpenClaw to stop transcript-only duplicate rows after `sessions_history` or similar large tool-result rewrite paths without changing provider-call billing.

## Best Possible Solution

Land an ancestry-preserving transcript rewrite cleanup or equivalent invariant so abandoned suffix rows do not inflate raw JSONL transcripts, with regression coverage for large `sessions_history` tool results and repeated rewrites.

## Reproduction Assessment

Yes. The issue provides concrete JSONL/trajectory evidence, a follow-up comment reproduced the mechanism on main, and current source still contains the truncation-to-branch-and-suffix-rewrite path.

## Solution Assessment

No for fixing `sessions_history` directly: that tool is already read-only. The maintainable fix belongs in transcript rewrite/truncation cleanup or an equivalent raw-transcript active-branch invariant.
