## Summary

Keep open: current main still has the transcript rewrite path that can replay unchanged suffix entries after tool-result truncation. The issue discussion already localized the mechanism, and open PRs #69486 and #72511 target the same transcript-rewrite area but are not merged.

## What This Changes

The issue asks OpenClaw to stop large `sessions_history` or startup-triggered transcript maintenance from duplicating raw JSONL messages while preserving correct billing and model context.

## Best Possible Solution

Keep this open until #69486, #72511, or an equivalent maintained fix lands and is verified against the raw JSONL large-`sessions_history` case. The end state should preserve active-branch ancestry while preventing low-value suffix replay or safely reconciling abandoned rewrite entries.

Yes. The issue includes concrete raw JSONL evidence and provider trajectory counts, a follow-up comment reproduced the mechanism, and current main still contains the large-tool-result truncation to suffix-rewrite path; this review stayed read-only and did not create live artifacts.
