## Summary

Keep open: current main still sends large tool-result truncation through the branch-and-suffix transcript rewrite path and persists replayed suffix entries to JSONL. `sessions_history` itself is read-only, and active-branch history readers reduce visible duplication, but the raw transcript-write duplication reported here remains unresolved.

## What This Changes

The issue asks OpenClaw to stop `sessions_history`-triggered transcript rewrites from appending duplicate assistant and tool-result JSONL rows while preserving correct billing and model context.

## Best Possible Solution

Preserve the active DAG semantics while adding an ancestry-aware cleanup or equivalent rewrite invariant so abandoned suffix rows do not inflate raw transcripts/history, with regression coverage for large `sessions_history` tool results and sibling branches.

## Reproduction Assessment

Yes. The issue includes concrete JSONL and trajectory evidence, a follow-up comment reproduced the mechanism, and current source still contains the truncation-to-branch-and-suffix-rewrite path that appends replayed entries.

## Solution Assessment

Unclear. Fixing `sessions_history` itself is not the right boundary because it only reads; the maintainable fix belongs in transcript rewrite/truncation or an equivalent active-branch/raw-file invariant.
