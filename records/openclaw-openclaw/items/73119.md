## Summary

Keep open: current main still has the raw transcript duplication path through tool-result truncation and transcript rewrite; `sessions_history` is read-only, but large history results can trigger the existing rewrite behavior later.

## What This Changes

The issue asks OpenClaw to stop large `sessions_history`/tool-result truncation from re-appending duplicate prior transcript rows into session JSONL files.

## Best Possible Solution

Land one canonical ancestry-preserving or idempotent transcript rewrite fix with regression coverage for large history/tool-result rewrites, then close this as implemented.

## Reproduction Assessment

Yes. The issue has concrete JSONL and trajectory evidence, a comment reproduced the mechanism on main, and current source still contains the truncation-to-branch-rewrite-to-append path.

## Solution Assessment

Unclear. Fixing `sessions_history` itself is not the right boundary because it only reads; the maintainable fix belongs in transcript rewrite/truncation or an equivalent raw-transcript invariant.
