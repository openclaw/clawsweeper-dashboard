## Summary

Keep open. The underlying security bug is still present on current main, but this PR should not merge as submitted because it leaves already-connected nodes without commands after node-pair approval until reconnect. The PR is also currently dirty against main, so it needs an amended security-aware gateway fix before merge.

## What This Changes

The PR changes node connection reconciliation to hide commands for unpaired nodes and updates gateway tests to pre-approve invoke fixtures and assert pre-pair `node.invoke` is blocked.

## Best Possible Solution

Land a fail-closed node command gate that hides commands before node-pair approval, then deliberately makes approved commands available for the live connected node by updating the node session or having `node.list` and `node.invoke` consult approved pairing state. Tests should cover pre-approval blocking, post-approval availability without reconnect, and existing approved-node invoke flows.

## Reproduction Assessment

Yes. A high-confidence reproduction path exists from current main source and tests: connect a device-paired node that declares `system.run` before node-pair approval, observe `node.list` exposes the commands, then invoke through an `operator.write` session.

## Solution Assessment

No, not as submitted. Returning an empty command list for unpaired nodes is the right fail-closed boundary, but the patch needs a post-approval live-session refresh or equivalent paired-state authorization check with tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Refresh live commands after node pairing approval:** `src/gateway/node-connect-reconcile.ts:84`
  - body: With this change, the first unpaired connection is registered with `commands: []`. `node.pair.approve` only writes the paired record and broadcasts `node.pair.resolved`, while `node.list` and `node.invoke` keep using the live `NodeSession.commands`, so approving the pending request leaves that connected node unable to expose or run the newly approved commands until it reconnects. Please update the live session or have the invoke/list paths consult the approved pair, and add a post-approval assertion.
  - confidence: 0.9
