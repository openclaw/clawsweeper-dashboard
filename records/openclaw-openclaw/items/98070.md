## Summary

Keep open: current main has the Skill Workshop proposal service, built-in tool, CLI/Gateway surfaces, tool allowlists, and post-selection `before_tool_call` policy hooks, but it does not expose a skill-creation authority/routing model that can make a custom workflow run before `skill_workshop` is selected.

## What This Changes

Request to make Skill Workshop composable so custom skill-creation workflows can run before the built-in proposal/apply mechanism.

## Best Possible Solution

Define a maintainer-approved skill-creation authority model that lets custom tools prepare proposal content while keeping the existing Skill Workshop service as the single governed proposal/apply path.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/API-direction request. Source inspection verifies the current gap: `skill_workshop` is prompt-preferred when available, while current hooks are per-tool-call policy hooks rather than a pre-selection skill-creation authority interface.

## Solution Assessment

Unclear. A dedicated hook/config model may be the right API, but current main already has tool policy hooks, plugin tools, and an internal proposal-content primitive, so maintainers should choose the permanent product boundary before implementation.
