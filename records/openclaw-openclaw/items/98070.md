## Summary

Keep open: current main already has a Skill Workshop proposal primitive, generic plugin tools, tool allowlists, and `before_tool_call` policy hooks, but it does not expose a dedicated skill-creation authority/routing model that can make a custom workflow run before `skill_workshop` is selected.

## What This Changes

Request to make Skill Workshop composable so custom skill creation workflows can run before proposal creation and application.

## Best Possible Solution

Define a maintainer-approved skill-creation authority model that composes custom tools with the existing Skill Workshop proposal lifecycle and explicitly does not create a second skill-write path.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/API-direction request. Source inspection does verify the current gap: `skill_workshop` is prompt-preferred when available, while current hooks are per-tool-call policy hooks rather than a pre-selection skill-creation authority interface.

## Solution Assessment

Unclear. A dedicated hook/config model may be the right API, but current code already has tool policy hooks, optional plugin tools, and a proposal-content primitive, so maintainers should choose the permanent product boundary before implementation.
