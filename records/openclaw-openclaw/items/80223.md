## Summary

Keep open: the bug is real and the supplied live proof is sufficient, but the current patch still has a blocking correctness gap because it requires message-tool delivery without giving the completion agent message-tool-only delivery guidance.

## What This Changes

The PR adds `subagent_announce` to the mediated completion tool set and adds regression coverage for channel completions that do not deliver through the message tool.

## Best Possible Solution

Keep the mediated-delivery guard, but pair it with explicit message-tool-only guidance or an equivalent runtime mode for `subagent_announce` completion agents before landing.

## Reproduction Assessment

Yes, source-level: production subagent completions pass `sourceTool: "subagent_announce"`, current main excludes that source from the mediated set, and group/channel source policy can require message-tool delivery. I did not run a live Telegram reproduction in this read-only review.

## Solution Assessment

No. Adding the set entry is the right guard direction, but without message-tool-only guidance or an equivalent runtime mode it can convert normal completion final answers into missing-delivery failures.
