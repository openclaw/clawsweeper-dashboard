## Summary

Keep open: the guard is aimed at a real source-level gap and the added redacted live proof is now acceptable, but the patch still adds `subagent_announce` to the message-tool-mediated set without teaching the subagent completion agent that normal final replies are private on those routes.

## What This Changes

The PR adds `subagent_announce` to the mediated completion source set and adds regression coverage that a Slack channel completion does not raw-send child output when message-tool delivery is missing.

## Best Possible Solution

Land the mediated-delivery guard only with matching message-tool-only guidance or an equivalent runtime mode for subagent completion agents, plus focused regression coverage for both missing and successful message-tool delivery.

## Reproduction Assessment

Yes, source-level: production subagent completions pass `sourceTool: "subagent_announce"`, while current main excludes that source from the mediated set, allowing the raw-fallback behavior the PR is trying to block. I did not execute a live reproduction in this read-only review.

## Solution Assessment

No: adding the set entry alone blocks raw fallback but can convert group/channel completions into missing-delivery failures because the completion agent is still instructed to use a normal final reply. The safer narrow fix is to pair the set entry with message-tool-only guidance or equivalent runtime mode.
