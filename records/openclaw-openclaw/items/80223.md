## Summary

Needs changes before merge: the guard is directionally correct, but it requires message-tool delivery without giving subagent completion agents the matching message-tool-only guidance or runtime mode, and the supplied proof does not exercise the real channel path.

## What This Changes

The branch adds `subagent_announce` to the agent-mediated completion tool set and adds Slack channel regression coverage for missing message-tool delivery.

## Best Possible Solution

Keep the mediated-delivery guard, but pair it with message-tool-only guidance or equivalent runtime mode for subagent completion agents, then validate with a redacted live group/channel completion trace.

## Reproduction Assessment

Yes, source-level: production subagent completions pass `sourceTool: "subagent_announce"`, while current main excludes that source from the mediated completion set. I did not run a live group/channel reproduction in this read-only review.

## Solution Assessment

No: adding the set entry blocks raw fallback, but without message-tool-only guidance or mode it can turn visible completions into missing-delivery failures. The safer fix is to carry the same explicit delivery contract used by async media completions.

## Review Findings
