## Summary

Keep open: the reported bug is valid and current main still lacks the `subagent_announce` mediated-delivery guard, but this PR is incomplete because it requires message-tool delivery without adding subagent completion guidance or runtime mode that tells the completion agent to use the message tool.

## What This Changes

The PR adds `subagent_announce` to the mediated completion tool set and adds a Slack channel regression test that blocks raw child-result fallback when message-tool delivery is missing.

## Best Possible Solution

Land a narrow fix that keeps subagent completions mediated while also giving the completion agent explicit message-tool-only guidance or an equivalent runtime mode, with focused regression coverage for visible group/channel delivery.

## Reproduction Assessment

Yes. Source inspection shows current main sends production completions with `sourceTool: "subagent_announce"` while excluding that source from the mediated set, and group/channel completion policy resolves to `message_tool_only`; the PR comments also provide live recurrence logs on `openclaw 2026.5.12`.

## Solution Assessment

No, not as-is. Adding the set entry blocks raw fallback, but without prompt guidance or runtime `message_tool_only` mode it can turn a normal completion final answer into a missing-delivery failure instead of a user-visible mediated report.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Add message-tool guidance before requiring it:** `src/agents/subagent-announce-delivery.ts:61`

Add explicit message-tool-only guidance or wire the proper runtime mode before requiring the message-tool proof.
