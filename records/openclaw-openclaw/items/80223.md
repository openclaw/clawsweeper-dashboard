## Summary

Keep open: the patch targets a real source-level gap, but it is not merge-ready because it requires message-tool mediated delivery without adding matching subagent completion guidance, and the supplied proof does not exercise the group/channel path after the fix.

## What This Changes

The PR adds `subagent_announce` to the agent-mediated completion source set and adds regression coverage for channel completions that miss message-tool delivery.

## Best Possible Solution

Keep the mediated-delivery guard only if it is paired with message-tool-only guidance or equivalent runtime mode for subagent completion agents, then validate with redacted live group/channel proof.

## Reproduction Assessment

Yes, source-level: production completions pass `sourceTool: "subagent_announce"`, while current main excludes that source from the mediated set. I did not establish a live group/channel reproduction in this read-only review.

## Solution Assessment

No: adding the set entry blocks raw fallback, but without message-tool guidance or mode it can convert visible completions into missing-delivery failures. The safer fix is to carry the same explicit delivery contract used by async media completions.
