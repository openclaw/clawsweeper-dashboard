## Summary

Keep open: current main still lacks the `subagent_announce` mediated-delivery guard, but this PR is not merge-ready because it requires message-tool delivery without giving the completion agent message-tool-only instructions or an equivalent runtime mode.

## What This Changes

The PR adds `subagent_announce` to the agent-mediated completion source set and adds a regression test that channel completions without message-tool delivery fail instead of raw-sending child output.

## Best Possible Solution

Keep the mediated-delivery guard, but add explicit message-tool-only guidance or an equivalent runtime mode for `subagent_announce` completion agents, then land with the existing regression plus focused format/e2e coverage.

## Reproduction Assessment

Yes, source-level: current main sends `sourceTool: "subagent_announce"` but excludes that source from the mediated set, while group/channel policy defaults to `message_tool_only`. The contributor also supplied live installed-runtime evidence, but I did not run a live Telegram reproduction in this read-only review.

## Solution Assessment

No, not as-is. Adding the set entry blocks raw fallback, but without message-tool guidance or an equivalent runtime mode it can convert a normal completion final answer into a missing-delivery failure; the media-completion path already shows the safer pattern.
