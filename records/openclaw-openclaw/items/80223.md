## Summary

Keep open: the reported bug is valid and the supplied live proof is sufficient, but the PR still has a blocking correctness gap because it requires mediated message-tool delivery without adding the guidance or runtime mode that makes the completion agent use the message tool.

## What This Changes

The PR adds `subagent_announce` to the agent-mediated completion source set and adds a Slack channel regression test that prevents raw child-result fallback when message-tool delivery is missing.

## Best Possible Solution

Land a version that keeps `subagent_announce` mediated and also gives completion agents explicit message-tool-only delivery guidance or an equivalent runtime mode, with focused regression coverage.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: current main sends production completions with `sourceTool: "subagent_announce"`, group/channel policy defaults to `message_tool_only`, and the current mediated set excludes that source.

## Solution Assessment

No. The set addition blocks raw fallback, but without media-style message-tool guidance or an equivalent runtime mode, normal completion final answers can remain private and surface as missing-delivery failures.
