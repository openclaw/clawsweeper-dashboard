## Summary

Keep open: the reported subagent completion delivery problem is real and the PR has useful proof, but the branch still has a merge-blocking behavior gap because it requires message-tool delivery without adding explicit guidance or an equivalent runtime mode for the completion agent.

## What This Changes

The branch adds `subagent_announce` to the mediated completion-source set and adds a regression for channel completions that miss message-tool delivery.

## Best Possible Solution

Keep the mediated-delivery guard, but pair it with explicit message-tool-only guidance or equivalent runtime behavior so subagent completion results reach group/channel users through the orchestrator instead of raw fallback or silent loss.

## Reproduction Assessment

Yes, source-level. Current main passes `sourceTool: "subagent_announce"` from the production announce path while excluding that source from the mediated set, and the PR discussion supplies live Telegram evidence of the delivery failure class.

## Solution Assessment

No, not as-is. The guard is the right direction, but requiring message-tool evidence without adding message-tool-only guidance or runtime mode can turn a normal completion final answer into a missing-delivery failure.
