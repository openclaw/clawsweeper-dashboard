## Summary

Keep open: the updated proof is enough for the real-behavior gate, but the code diff still has the same correctness blocker. Adding `subagent_announce` to the mediated set makes group/channel completions require message-tool evidence while the subagent completion instruction still tells the requester to produce a normal final answer, which can become private and fail with missing mediated delivery.

## What This Changes

The PR adds `subagent_announce` to the mediated completion source set and adds regression coverage for channel completions that miss message-tool delivery.

## Best Possible Solution

Keep the mediated-delivery guard, but pair it with explicit message-tool-only guidance or an equivalent runtime mode for subagent completion agents, then land with focused regression coverage.

## Reproduction Assessment

Yes, source-level: production subagent completions pass `sourceTool: "subagent_announce"`, current main excludes that source from the mediated set, and group/channel completion policy defaults to `message_tool_only`. I did not run a live Telegram reproduction in this read-only review.

## Solution Assessment

No. Adding the set entry blocks raw fallback, but without message-tool-only guidance or an equivalent runtime mode it can turn a normal completion final answer into a missing-delivery failure.
