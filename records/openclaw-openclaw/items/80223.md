## Summary

Keep open: this targets a real source-level delivery gap, but the current patch only adds `subagent_announce` to the mediated-source set and a regression test. It still lacks message-tool-only guidance or equivalent runtime mode, and the supplied proof does not exercise the real group/channel completion path.

## What This Changes

The branch adds `subagent_announce` to the mediated completion tool set and adds Slack channel regression coverage for missing message-tool delivery.

## Best Possible Solution

Keep the mediated-delivery guard, add explicit message-tool-only guidance or equivalent runtime mode for subagent completions, coordinate with the adjacent routing fix, and require redacted live group/channel proof.

## Reproduction Assessment

Yes, source-level: current main emits production subagent completions with `sourceTool: "subagent_announce"` while the mediated set excludes that source. I did not run a live group/channel reproduction in this read-only review.

## Solution Assessment

No: adding the set entry blocks raw fallback, but without message-tool-only guidance or equivalent runtime mode it can turn visible completions into missing-delivery failures. The safer fix is to mirror the async media completion delivery contract.
