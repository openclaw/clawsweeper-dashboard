## Label Justifications

- `P1`: The report describes a current-release agent-to-agent workflow where messages can be lost after an accepted response, leaving real multi-agent users without retry signal.
- `impact:message-loss`: The central symptom is a sessions_send message being dropped after a target session write-lock timeout while the caller sees accepted/pending.
- `impact:session-state`: The failure is rooted in session JSONL write-lock contention and stale pending delivery state for the target/run session.
- `maturity:stable`: The affected Gateway agent/session RPC and delivery behavior maps to the M4 Stable gateway-runtime feature in the maturity shortlist.

## Summary

Keep open: current main and v2026.6.11 still have a sessions_send path that returns accepted with pending delivery before the target run actually writes, while later SessionWriteLockTimeoutError failures are only observable through agent.wait/logs. Existing related issues cover the lock and pending-metadata pieces, but none fully owns this exact post-accepted sessions_send failure contract.

## What This Changes

The issue asks OpenClaw to surface target-session write-lock failures from sessions_send instead of returning accepted/pending when delivery later fails.

## Best Possible Solution

Keep this open until maintainers choose and land a sessions_send delivery contract that either verifies queue/persist before accepted or exposes late lock failures, coordinated with the canonical session-lock fixes.

## Reproduction Assessment

Yes, at source level: the gateway returns accepted before dispatch, sessions_send returns pending for normal async A2A, and the session write-lock code still throws SessionWriteLockTimeoutError later. I did not run a live lock-contention reproduction against current main.

## Solution Assessment

Unclear. Surfacing this correctly needs a maintainer decision between fixing the underlying lock contention, changing sessions_send fire-and-forget semantics, or adding a durable delivery-status/error channel for post-accepted failures.
