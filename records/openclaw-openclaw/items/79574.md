The PR changes shared channel setup helpers to run contract-backed credential promotion for non-default account writes, adds a synthetic external-plugin regression fixture, and updates changelog plus rebase-related generated/script files.

## Best Possible Solution

## Reproduction Assessment

No high-confidence live reproduction was run here. Source inspection shows current main lacks shared-adapter credential promotion, and the published WeCom setup path ignores the selected account id, matching the reported overwrite mechanism.
