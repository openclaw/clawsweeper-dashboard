## Summary

Keep open: current main still has the synchronous fallback vector scan, and this focused PR addresses that exact path with bounded batching, event-loop yielding, regression tests, and strong supplied runtime proof. I found no blocking correctness or security findings.

## What This Changes

The PR changes memory-core fallback vector search to scan chunks in bounded rowid batches with `setImmediate` yields, updates regression tests, and adds a changelog entry.

## Best Possible Solution

Land this focused batching/yield fix after exact-head validation, keeping https://github.com/openclaw/openclaw/issues/81172 paired until the merged PR closes it.

## Reproduction Assessment

Yes, from source: current main enters the fallback vector path when the sqlite-vec index is unavailable or mismatched, then synchronously scans and scores all chunk embeddings. The PR body also supplies before/after harness output showing zero heartbeats on pre-fix code and interleaved heartbeats after the patch.

## Solution Assessment

Yes, this is the narrowest maintainable fix I see: it preserves the existing scoring algorithm and private API while adding cooperative scheduling in the one unbounded JS-side fallback loop. Worker threads or a new timeout/config surface would be larger follow-up work, not required for this patch.
