## Summary

Keep open for maintainer merge review. Current main still has the synchronous fallback scan that can starve the event loop, while this focused PR moves only that fallback path to batched yielding with regression tests and strong live-output proof; I found no blocking patch or security findings.

## What This Changes

The PR changes memory-core fallback vector search to scan chunks in rowid-limited batches with `setImmediate` yields, updates fallback vector tests, and adds a changelog entry.

## Best Possible Solution

Land the focused batching/yield fix with its regression tests after maintainer review, and keep https://github.com/openclaw/openclaw/issues/81172 paired until the merged PR closes it.

## Reproduction Assessment

Yes, source-reproducible: current main's fallback vector path synchronously iterates chunk rows and scores every embedding on the Node event loop. The PR body also supplies before/after harness output showing zero pre-fix heartbeats and interleaved heartbeats after the patch.

## Solution Assessment

Yes. Batching the existing fallback loop preserves the scoring contract while matching the repo's existing cooperative-yield pattern; worker-thread isolation can remain a larger follow-up if maintainers want deeper separation.
