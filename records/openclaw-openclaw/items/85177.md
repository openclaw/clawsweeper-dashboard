## Summary

Keep open: current main still source-reproduces the claude-cli continuity loss for auth-profile/auth-epoch invalidation when there is no compaction summary, and an open same-author replacement PR now addresses the safer root cause by narrowing cosmetic auth-profile resets instead of raw-replaying history across an auth boundary.

## What This Changes

The issue asks OpenClaw to preserve claude-cli conversation history when auth-profile or auth-epoch invalidation forces a fresh CLI session.

## Best Possible Solution

Review and either land or reject https://github.com/openclaw/openclaw/pull/85505 as the focused root-cause fix, while keeping raw transcript replay across real auth boundaries blocked unless maintainers define a summarized, redacted, or explicitly approved recovery policy.

## Reproduction Assessment

Yes for the current-main behavior by source and tests: an uncompacted session invalidated by auth-profile or auth-epoch cannot produce raw reseed messages, so buildCliSessionHistoryPrompt receives no history. I did not live-run the WSL2 billing-lockout scenario.

## Solution Assessment

No for the direct raw allow-list change because it conflicts with current docs/tests and may cross an auth identity boundary. The narrower maintainable direction is to avoid resetting claude-cli sessions for cosmetic OpenClaw auth-profile rotation, as proposed in the open replacement PR, or define a safer summarized recovery path.
