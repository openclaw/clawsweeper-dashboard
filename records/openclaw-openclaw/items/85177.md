## Summary

Keep open, but not as a straightforward bug-fix automation item: current main intentionally blocks raw transcript reseed for auth-profile and auth-epoch invalidations, while the issue identifies a real continuity gap when no compaction summary exists. The remaining work needs a maintainer product/security decision on how to preserve continuity without crossing auth identity boundaries unsafely.

## What This Changes

The issue asks OpenClaw to preserve claude-cli conversation continuity after auth-profile or auth-epoch invalidation instead of starting the fresh CLI run with no reseeded history.

## Best Possible Solution

Keep the current no-raw-cross-auth default until maintainers choose a safe continuity policy, then implement a focused path such as summarized/redacted Claude transcript recovery or a visible warning with tests around auth-boundary behavior.

## Reproduction Assessment

Yes for the current behavior: source and tests show an uncompacted OpenClaw session with auth-profile or auth-epoch invalidation produces no reseed messages, so the fresh CLI prompt has no history prompt. I did not live-run the WSL2 billing-lockout scenario.

## Solution Assessment

No for the direct allow-list fix as written: adding auth-profile/auth-epoch to raw reseed would contradict current docs/tests and may cross an auth identity boundary. A safer solution needs maintainer-approved summarized, redacted, opt-in, or warning-backed recovery semantics.
