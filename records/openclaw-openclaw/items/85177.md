Keep open: current main and the latest release still keep auth-profile/auth-epoch invalidations on the no-raw-reseed boundary, and the open fix path still needs maintainer/security agreement before this canonical context-loss bug can close.

## What This Changes

The issue asks OpenClaw to preserve claude-cli conversation continuity when auth-profile or auth-epoch changes force a fresh CLI session.

## Best Possible Solution

Land or replace the auth-identity fix so cosmetic Claude CLI profile rotation preserves session reuse while real or unproven credential changes still reset, keeping raw transcript replay blocked across auth boundaries.

## Reproduction Assessment

Yes. Source inspection shows current main can produce auth-profile/auth-epoch invalidations, pass those reasons into reseed loading, and return no raw history for those reasons; I did not run a live claude-cli gateway repro.

## Solution Assessment

No for the raw-replay suggestion as written. The better fix is a maintainer-approved Claude CLI auth identity path that avoids cosmetic profile resets without replaying raw transcript across real auth changes.
