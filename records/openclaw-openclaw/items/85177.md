## Summary

Keep open: current main intentionally blocks raw transcript reseed for auth-profile and auth-epoch invalidations, and docs/tests lock in that boundary, but the issue identifies a real claude-cli continuity gap when no compaction summary exists. The remaining fix needs maintainer product/security direction rather than a direct allow-list change.

## What This Changes

The issue requests preserving claude-cli conversation continuity when auth-profile or auth-epoch invalidation forces a fresh CLI session instead of producing an empty reseed prompt.

## Best Possible Solution

Keep the no-raw-cross-auth default until maintainers choose a safe continuity policy, then implement a focused recovery path such as summarized or redacted Claude transcript reseed, explicit opt-in semantics, or a visible warning with auth-boundary tests.

## Reproduction Assessment

Yes from source: an uncompacted OpenClaw transcript with `auth-profile` or `auth-epoch` invalidation produces no reseed messages, so `buildCliSessionHistoryPrompt()` has no history to render. I did not live-run the WSL2 billing-lockout path.

## Solution Assessment

No for the direct allow-list fix: adding auth-profile/auth-epoch to raw reseed would contradict current docs/tests and may cross an auth identity boundary. A safer solution needs maintainer-approved summarized, redacted, opt-in, or warning-backed recovery semantics.
