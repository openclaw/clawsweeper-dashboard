## Summary

Keep open: current main still intentionally blocks raw transcript reseed for auth-profile and auth-epoch invalidations, while the report identifies a real claude-cli continuity gap when no compaction summary exists. The remaining fix needs maintainer product/security direction, not an automatic allow-list change.

## What This Changes

The issue asks OpenClaw to preserve claude-cli conversation continuity after auth-profile or auth-epoch invalidation instead of starting a fresh CLI session with no prior history.

## Best Possible Solution

Keep the current no-raw-cross-auth default until maintainers choose a safe continuity policy, then add focused summarized/redacted recovery or a visible warning path with regression coverage for auth-profile and auth-epoch invalidations.

## Reproduction Assessment

Yes, from source inspection: an uncompacted session with auth-profile or auth-epoch invalidation reaches the reseed loader, which returns no messages because those reasons are excluded from raw reseed. I did not live-run the WSL2 billing-lockout scenario.

## Solution Assessment

No for the direct allow-list fix: adding auth-profile/auth-epoch to raw reseed would contradict current docs/tests and may cross an auth identity boundary. A safer solution needs maintainer-approved summarized, redacted, opt-in, or warning-backed recovery semantics.
