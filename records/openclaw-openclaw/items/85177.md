## Summary

Keep open: current main deliberately blocks raw transcript reseed for auth-profile and auth-epoch invalidations, but the report identifies a real claude-cli continuity gap when no compaction summary exists. The remaining decision is product/security policy for preserving context without replaying raw transcript text across an auth boundary.

## What This Changes

Request to preserve claude-cli conversation continuity by reseeding prior history after auth-profile or auth-epoch invalidation instead of starting a fresh contextless CLI session.

## Best Possible Solution

Keep the no-raw-cross-auth default until maintainers choose a safe continuity policy, then implement a focused summarized, redacted, opt-in, or warning-backed recovery path with tests around auth-profile and auth-epoch behavior.

## Reproduction Assessment

Yes from source: an auth-profile or auth-epoch invalidation with no compaction summary produces no reseed messages, which makes buildCliSessionHistoryPrompt return undefined. I did not live-run the WSL2 billing-lockout scenario.

## Solution Assessment

No for the direct allow-list fix: adding auth-profile/auth-epoch to raw reseed would contradict current docs/tests and may cross an auth identity boundary. A safer solution needs maintainer-approved summarized, redacted, opt-in, or warning-backed recovery semantics.
