## Summary

## What This Changes

The PR adds session-key ownership and an `emitHooksQueued`/`continuationQueued` reporting path for coalesced gateway restart continuations, with gateway-tool, `/restart`, restart-coordinator, infra, and QA test updates.

## Best Possible Solution

Preserve the accepted session-owned restart hooks, or explicitly report that they were displaced, across every forced/coalesced scheduler path before merging this security-sensitive restart continuation fix.

- **[P1] Preserve queued hooks when pulling a forced restart earlier:** `src/infra/restart.ts:823-825`
  - body: `requestSafeGatewayRestart(... skipDeferral)` now asks the scheduler to preserve pending emit hooks, but this preservation is only honored after the restart has entered the active-deferral bypass branch. If a session-owned restart with a continuation is still waiting on its timer/cooldown, a hookless forced restart takes the `shouldPullEarlier` path, passes the guard because `hooks` is undefined, calls `clearPendingScheduledRestart()`, and reschedules without the original hooks. The first caller already observed `emitHooksQueued: true`, so its continuation is still silently dropped. Please preserve the existing hooks in this earlier-reschedule path, or explicitly reject/report the displacement, and add a regression test for that sequence.
