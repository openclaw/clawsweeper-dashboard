## Summary

## What This Changes

## Best Possible Solution

Land this after threading the same restart owner key through every session-scoped emit-hook caller, especially chat `/restart`, while keeping the explicit queued-state reporting and regression coverage.

## Reproduction Assessment

Yes. Source inspection shows chat `/restart` creates a sentinel with `sessionKey`, then schedules emit hooks without passing that key while the new guard only rejects replacement when both keys are defined.

## Solution Assessment

No as-is. The gateway-tool fix and the in-flight preparation regression test are the right shape, but the best fix must also carry the session owner through the sibling `/restart` continuation path or make the scheduler reject session-scoped hooks that arrive without ownership.
