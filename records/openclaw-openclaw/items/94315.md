## Summary

Keep open, but this PR should not merge as-is: the new 300s apply approval window exceeds the current 90s Codex dynamic-tool watchdog, and also exceeds the 150s watchdog in the older open fix path, so agent-tool apply can still fail before approval resolves.

## What This Changes

The PR adds an apply-only `timeoutMs: 300_000` approval window in `src/skills/workshop/policy.ts` while leaving reject and quarantine on the default approval timeout.

## Best Possible Solution

Choose one timeout contract before merge: either keep apply approval within the active dynamic-tool watchdog or raise every affected watchdog above the new approval window with focused tests and live approval proof.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes at source level: current main requires approval for `skill_workshop apply`, plugin approval waits can run 120s plus gateway grace, and Codex dynamic-tool calls still use a 90s watchdog; linked issue comments also report live failures.

## Solution Assessment

Is this the best way to solve the issue? No. An apply-only 300s approval timeout is not the best fix because it exceeds the current and proposed Codex watchdogs; the safer path is to align the watchdog and approval TTL together.
