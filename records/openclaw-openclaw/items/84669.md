## Summary

Keep open: the final head still has a source-provenance bug that can leave the linked OpenAI `/think` path off-only when a configured model row omits `reasoning` and stale runtime catalog metadata supplies `reasoning:false`.

## What This Changes

The PR adds source-aware thinking catalog metadata, lets OpenAI/OpenAI-Codex provider profiles override stale runtime `reasoning:false` rows, and adds focused `/think` and resolver regressions.

## Best Possible Solution

Keep the provider-owned override approach, but track the source of the effective `reasoning` field itself and cover both omitted configured reasoning and explicit configured opt-out cases before merge.

## Reproduction Assessment

Yes, source-reproducible: the PR head can merge runtime `reasoning:false` with a configured row that omits `reasoning`, stamp it as configured, and then bypass the runtime-only override in `thinking.ts`. I did not run tests because this review is read-only.

## Solution Assessment

No. The provider-owned override is the right direction, but the current implementation loses field-level provenance when configured metadata overlays runtime catalog metadata.
