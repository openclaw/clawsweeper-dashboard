## Summary

Keep open: the PR is a plausible fix for the linked `/think` path and has useful proof, but the new provenance gate is only wired through one model-selection path, leaving other runtime catalog callers unable to honor the OpenAI/OpenAI-Codex override.

## What This Changes

The PR adds source-aware thinking catalog precedence so OpenAI/OpenAI-Codex provider profiles can override stale runtime `reasoning:false` rows while configured opt-outs remain off-only.

## Best Possible Solution

Centralize thinking catalog provenance or tag every runtime/configured catalog caller consistently, then keep the OpenAI/OpenAI-Codex override scoped to stale runtime rows with configured opt-out and sessions_spawn regression coverage.

## Reproduction Assessment

Yes, source-reproducible: current main short-circuits to off-only on catalog `reasoning:false`, while the issue gives a concrete Telegram `/think xhigh` rejection and the OpenAI provider profile advertises non-off levels. I did not run a live Telegram/Codex OAuth reproduction in this read-only review.

## Solution Assessment

No, not yet: the PR's source-aware override is the right shape for preserving configured opt-outs, but it only tags one caller path. The maintainable fix needs the same provenance contract anywhere runtime catalog rows can feed thinking validation.
