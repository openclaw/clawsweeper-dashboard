## Summary

Keep open: the reported bug is real and the branch is close, but current head still stamps runtime-loaded `reasoning:false` as configured whenever a configured model row exists, so configured OpenAI rows that omit `reasoning` can still reject `/think xhigh`.

## What This Changes

The PR adds source-aware thinking-catalog provenance and an OpenAI provider flag so OpenAI/OpenAI-Codex thinking profiles can override runtime `reasoning:false`, with resolver/directive regressions and a small CLI formatting cleanup.

## Best Possible Solution

Preserve provenance for the `reasoning` field itself: runtime `reasoning:false` should remain runtime-derived when config omits the field, while explicit configured `reasoning:false` remains off-only, with regressions for both shapes.

## Reproduction Assessment

Yes. Current main source shows `reasoning:false` short-circuits provider thinking profiles to off-only, and the PR body supplies after-fix resolver output for the stale runtime-catalog path; I did not run tests because this review is read-only.

## Solution Assessment

No, not yet. Provider-owned override is the right direction, but the current implementation needs field-level reasoning provenance so configured overlays do not accidentally turn stale runtime metadata into an explicit opt-out.
