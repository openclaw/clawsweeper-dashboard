## Summary

Needs changes: the provider-owned override is the right direction, but the latest patch still misclassifies inherited runtime `reasoning:false` as a configured opt-out for configured OpenAI/OpenAI-Codex rows that omit `reasoning`.

## What This Changes

The PR adds source-aware thinking catalog handling so OpenAI/OpenAI-Codex provider thinking profiles can override stale runtime `reasoning:false` rows while preserving explicit configured opt-outs.

## Best Possible Solution

Carry provenance for the `reasoning` field itself, or only mark reasoning as configured when config explicitly set it, while keeping explicit configured `reasoning:false` authoritative.

## Reproduction Assessment

Yes. Source inspection of current main shows `reasoning:false` short-circuits provider thinking profiles, and the remaining PR defect is source-reproducible from the overlay logic on PR head.

## Solution Assessment

No. The provider-owned override is the maintainable direction, but this implementation still needs field-level reasoning provenance so omitted config does not become an explicit opt-out.
