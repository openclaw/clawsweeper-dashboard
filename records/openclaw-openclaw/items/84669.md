## Summary

Keep open: the PR targets a real source-reproducible `/think` rejection and includes useful proof, but the patch is not merge-safe because it treats every OpenAI `reasoning:false` catalog row as stale and hard-codes provider policy in core.

## What This Changes

The PR changes the shared thinking resolver so OpenAI/OpenAI-Codex provider profiles can override `reasoning:false` catalog rows and adds resolver/directive regression tests for `gpt-5.5`.

## Best Possible Solution

Keep explicit catalog opt-outs authoritative unless the catalog source is known stale, and express the OpenAI exception through a provider-owned or source-aware contract rather than core provider-id checks.

## Reproduction Assessment

Yes. Source inspection on current main shows `reasoning:false` returns off-only before provider profile resolution, and directive handling validates `/think` through that resolver, matching the reported `Use one of: off` path.

## Solution Assessment

No. The PR fixes the rejection in targeted tests, but it conflates stale OpenAI catalog rows with explicit config opt-outs and hard-codes OpenAI policy in core; a source-aware provider-owned opt-in is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve explicit OpenAI thinking opt-outs:** `src/auto-reply/thinking.ts:181-182`
  - body: The new OpenAI override treats every `reasoning:false` row for `openai`/`openai-codex` as stale. `thinkingCatalog` can come directly from the configured or allowed model catalog, and docs define `reasoning` as whether that model exposes reasoning, so an existing OpenAI row intentionally set to `false` would now accept `/think xhigh` instead of staying off-only. Please make the override source-aware or provider-owned so explicit configured opt-outs remain authoritative.
  - confidence: 0.9

Put the opt-in on the provider profile/contract so core stays generic and future providers do not need core edits for the same stale-catalog case.
  - confidence: 0.86
