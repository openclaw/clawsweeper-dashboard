## Summary

Keep this PR open: the provider-owned override is the right direction and the contributor supplied useful proof, but the latest diff still has a blocking compatibility bug around configured catalog rows with omitted reasoning metadata.

## What This Changes

The PR adds source-aware thinking catalog handling so OpenAI/OpenAI-Codex provider thinking profiles can override stale runtime `reasoning:false` rows while preserving explicit configured opt-outs, with resolver/directive tests and small fixture/lint cleanups.

## Best Possible Solution

Carry provenance for the `reasoning` field itself, or only mark the reasoning source as configured when config explicitly set `reasoning`, so stale runtime rows can be overridden without weakening explicit opt-outs.

## Reproduction Assessment

Yes. Source inspection on current main shows `reasoning:false` short-circuits provider thinking profiles and directive validation emits the unsupported-level message; the remaining PR defect is also source-reproducible from the overlay logic.

## Solution Assessment

No. The provider-owned override is the maintainable direction, but the current patch needs field-level reasoning provenance so omitted config does not become an explicit opt-out.
