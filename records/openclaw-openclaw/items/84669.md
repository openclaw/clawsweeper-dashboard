## Summary

Keep open and request changes: the PR targets a real P1 thinking-level regression and has sufficient after-change proof, but latest head still misclassifies inherited runtime `reasoning:false` as a configured opt-out when a configured OpenAI model row omits `reasoning`.

## What This Changes

The PR adds provider-owned runtime-catalog override support for OpenAI thinking profiles, source-tagged thinking catalogs, `/think` regressions, and small test/lint fixture updates.

## Best Possible Solution

Carry provenance for the `reasoning` field itself, or only mark reasoning as configured when config explicitly set it, so explicit configured `false` stays off-only while stale runtime `false` remains overridable by the provider profile.

## Reproduction Assessment

Yes, source-reproducible. Current main short-circuits `reasoning:false` to off-only before provider profiles, and latest PR head has a clear omitted-config case where inherited runtime `reasoning:false` is marked configured; I did not run a live Telegram repro in this read-only review.

## Solution Assessment

No as written. The provider-owned override direction is maintainable, but the patch needs field-level reasoning provenance so omitted config does not become an explicit opt-out.
