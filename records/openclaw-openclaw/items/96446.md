## Summary

Keep open. Current main and the latest release still parse explicit reply directive IDs with raw `trim()`, while this PR applies the existing sanitizer contract at the parser boundary and adds focused coverage; I found no actionable patch defect.

## What This Changes

The PR updates `src/utils/directive-tags.ts` and its tests so explicit `[[reply_to:...]]` IDs use `sanitizeReplyDirectiveId()` and C1 control characters are stripped.

## Best Possible Solution

Land the focused sanitizer unification after normal maintainer approval and exact-head CI so inline parsing and directive emission share one reply-ID normalization contract.

## Reproduction Assessment

Yes, at source level. Current main stores explicit reply directive IDs with raw `trim()`, and the PR body shows an after-fix terminal probe normalizing the same malformed directive shape to `abcdef`.

## Solution Assessment

Yes. Reusing `sanitizeReplyDirectiveId()` at the parse boundary is the narrowest maintainable fix; re-sanitizing every downstream caller would duplicate the contract and still miss direct parser consumers.
