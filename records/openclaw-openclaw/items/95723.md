## Summary

Keep open for maintainer review: current main and v2026.6.9 still count disabled historical cron errors in Overview, while this PR is a clean candidate fix with sufficient terminal proof and no blocking findings.

## What This Changes

The PR adds a shared Control UI active-failure predicate for cron jobs, applies it to the Overview failed-count card and attention list, and adds a colocated unit test.

## Best Possible Solution

Land this or an equivalent small UI-only fix after normal gates so Overview active-failure summaries ignore disabled historical errors while detail and filter surfaces preserve historical run status.

## Reproduction Assessment

Yes, at source level: current main's Overview card and attention builder filter solely on resolved `lastRunStatus === "error"`, so a disabled `CronJob` retaining historical error state enters both summaries. I did not run tests locally because this review was required to keep the checkout read-only.

## Solution Assessment

Yes: centralizing `job.enabled && historical status is error` in `cron-status.ts` is the narrow maintainable fix for current-actionability surfaces. Changing scheduler state or the detail/filter status path would be broader and would hide useful historical status.
