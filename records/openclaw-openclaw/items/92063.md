## What This Changes

The PR changes Control UI chat rendering so adjacent streaming and reading-indicator items render as one assistant stream group with one avatar/footer while keeping segments as separate bubbles.

## Best Possible Solution

Land this render-layer coalescing fix if maintainers accept one assistant group as the right scope, and track any stricter single-bubble streaming merge separately.

## Reproduction Assessment

## Solution Assessment

Yes for the duplicate outer group/avatar bug: render-layer coalescing avoids changing gateway state or `buildChatItems`, and it avoids the broader prior PR's hidden-tool-call regression. Maintainers still need to decide whether a later single-bubble merge is required.
