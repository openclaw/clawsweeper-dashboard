## Summary

## What This Changes

Treats terminal Control UI `chat.send` acknowledgements as completed runs in direct and queued send paths and adds regression coverage for stale active `sessions.list` rows after ACK completion.

## Best Possible Solution

Land the narrow lifecycle-helper reuse with the targeted regression tests and live Control UI proof once normal PR checks are acceptable; keep the linked user issue open until this PR merges.

## Reproduction Assessment

Yes from source inspection: current `chat.send` ACK `ok` clears local run fields without arming the stale-session reconciliation window, and `loadSessions` can then apply a stale active selected-session row. I did not run the failing browser path in this read-only review.

## Solution Assessment

Yes; this is the best fix shape because terminal cleanup and stale-row suppression already live in `reconcileChatRunLifecycle`, while patching `hasAbortableSessionRun` or adding a second ACK-only cleanup path would duplicate the session-state invariant.
