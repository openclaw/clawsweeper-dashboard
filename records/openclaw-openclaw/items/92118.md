## Summary

Current main does not recognize the reported provider wording, and the missing match can misclassify an invalid model selection as a schema rejection. No linked repair PR or exact canonical duplicate currently owns the remaining fix.

## What This Changes

Classify OpenAI-compatible provider responses containing “Not supported model <model-id>” as model-not-found errors instead of schema errors.

## Best Possible Solution

Extend the shared model-not-found classifier with a narrowly scoped match for unsupported model identifiers, plus positive coverage for the reported serialized payload and a negative unsupported-capability case.

## Reproduction Assessment

Yes at source level. The supplied serialized payload contains no phrase accepted by the current model-not-found classifier, and the inspected runtime path can consequently present it as a schema failure; a live gateway call was not performed.

## Solution Assessment

Yes, with a precision safeguard. The shared classifier is the narrowest maintainable boundary, but the new match should specifically identify an unsupported model rather than broadly matching all unsupported behavior.
