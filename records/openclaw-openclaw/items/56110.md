## Summary

## What This Changes

The issue requests adding `STATE.md` to the built-in workspace bootstrap file set so ephemeral active-task state is injected after compaction.

## Best Possible Solution

Make an explicit maintainer decision on whether `STATE.md` should be a first-class session/handoff convention or handled through configurable/session-scoped extra files; if accepted, update loader, filters, docs, file surfaces, and focused tests together.
