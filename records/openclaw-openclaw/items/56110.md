## Best Possible Solution

Make an explicit maintainer decision on the bootstrap-state contract, reconcile it with https://github.com/openclaw/openclaw/issues/39120 and https://github.com/openclaw/openclaw/issues/46565, then implement one consistent path across loader, filters, gateway file APIs, docs, and tests if STATE.md is accepted.

## Reproduction Assessment

Yes for the capability gap: current source and docs show STATE.md is absent from the default bootstrap list, recognized basename set, gateway file allowlist, and latest release. The claimed post-compaction recovery benefit is a product behavior claim rather than a broken existing contract.

## Solution Assessment

Unclear until maintainers choose the product contract. A one-file addition is mechanically narrow if STATE.md becomes a core convention, but the safer direction may be a configurable or session-scoped context surface that avoids making a new global state file authoritative by default.
