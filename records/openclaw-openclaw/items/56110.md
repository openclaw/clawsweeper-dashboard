## Summary

Keep open. Current main and the latest release still omit `STATE.md` from the fixed workspace bootstrap contract, and the related custom-file work does not settle whether OpenClaw should bless a first-class session/handoff state file.

## What This Changes

The issue requests adding `STATE.md` as an auto-loaded ephemeral workspace bootstrap file for post-compaction and long-running-session continuity.

## Best Possible Solution

Make an explicit maintainer decision on whether `STATE.md` should be a first-class bootstrap convention or handled through configurable/session-scoped extra files; if accepted, update the loader, filters, docs, gateway file surface, and focused tests together.

## Reproduction Assessment

Yes for the capability gap: current main and `v2026.5.22` omit `STATE.md` from the recognized bootstrap names, default loader entries, and docs. This is not a broken documented behavior repro; it is a source-verifiable missing feature.

## Solution Assessment

Unclear until maintainers choose the product contract. A one-line loader addition is technically small, but the comments show unresolved global versus session-scoped lifetime and automatic-authority questions.
