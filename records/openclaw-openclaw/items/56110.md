## Summary

Keep open. Current main and the latest release still omit `STATE.md` from the fixed workspace bootstrap contract, and the related configurable/session-scoped bootstrap-file work does not decide whether OpenClaw should bless a first-class handoff state file.

## What This Changes

The issue requests adding `STATE.md` as an auto-loaded workspace bootstrap file for compact, ephemeral post-compaction working state.

## Best Possible Solution

Make an explicit maintainer decision on whether `STATE.md` should be a first-class bootstrap convention or handled through configurable/session-scoped extra files; if accepted, update the loader, session filters, gateway file surface, docs, and focused tests together.

## Solution Assessment
