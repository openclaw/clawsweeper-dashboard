## Summary

Keep open. Current main and the latest release still exclude `STATE.md` from the workspace bootstrap contract, and recent discussion adds real use-case support while highlighting unresolved global-vs-session and authority-after-compaction design questions.

## What This Changes

The issue requests adding `STATE.md` as an auto-loaded workspace bootstrap file for ephemeral agent handoff state after compaction.

## Best Possible Solution

Make an explicit bootstrap product decision: either add `STATE.md` as a first-class workspace file across constants, loader, templates, docs, tests, prompt-cache ordering, and session filtering, or document a session-scoped extra-file pattern as the supported handoff-state path.

## Solution Assessment

Unclear pending maintainer product review. A one-line loader addition is not enough because the durable contract needs aligned docs, templates, tests, prompt-budget/cache behavior, and global-versus-session semantics.
