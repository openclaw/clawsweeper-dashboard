## Summary

Keep open. Current main does not contain this PR's launcher-wide hard exit, and the proposed one-line fix is still too broad because it can skip asynchronous version output and truncate pending stdout/stderr writes before the Windows hang root cause is isolated.

## What This Changes

The branch adds a final `process.exit(process.exitCode ?? 0)` to `openclaw.mjs` after launcher help or entrypoint import completes.

## Best Possible Solution

Target the actual completed-command handle leak or add an awaited, drain-aware launcher shutdown path that preserves version/help/JSON output and long-running commands.

## Reproduction Assessment

Yes for the PR regression: source inspection plus local Node checks show the new final hard exit can run before asynchronous version output and stream drains complete. The original Windows hang has reporter evidence in #74378 but still needs Windows npm smoke proof for the repaired fix.

## Solution Assessment

No. A launcher-wide `process.exit()` is broader than the reported hang; the safer fix should clean up the leaking completion path or explicitly await safe shutdown and stream-drain boundaries.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve root version output:** `openclaw.mjs:317`
  - body: The final `process.exit(...)` can run immediately after the entry import resolves. Current `src/entry.ts` treats the root version fast path as handled once `tryHandleRootVersionFastPath` returns true, but that helper prints inside a later `resolveVersion().then(...)`, so `openclaw --version` can exit successfully with no output.
  - confidence: 0.91
- **[P2] Drain output before forcing launcher exit:** `openclaw.mjs:317`
  - body: This hard exit applies after help and routed command output. `process.exit()` can terminate before pending stdout/stderr writes drain, so piped help/status/JSON output can be truncated; fix the leaking handle or explicitly wait for stream completion before exiting.
  - confidence: 0.87
