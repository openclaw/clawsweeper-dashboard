## Summary

Keep this PR open for author or maintainer revision. It targets a real, still-open Windows CLI lifecycle regression in #74378, but the proposed launcher-wide hard exit is not safe to merge because it can preempt the async root version fast path and can truncate pending stdout/stderr output.

## What This Changes

The PR adds a final `process.exit(process.exitCode ?? 0)` to `openclaw.mjs` after launcher help handling or built entry import completes.

## Best Possible Solution

Revise the lifecycle fix so short-lived Windows npm commands terminate by cleaning up the leaking handle or by adding a narrow completion shutdown that awaits fast paths and drains output, while preserving `--version`, help/JSON output, and intentional foreground commands such as the gateway.

## Reproduction Assessment

Yes for the PR regression: the source path shows `openclaw --version` can finish entry module import before its async version output runs, then hit the new hard launcher exit. The underlying Windows hang also has a concrete reporter repro for npm-installed `status`, but it still needs Windows confirmation after revision.

## Solution Assessment

No. A launcher-wide `process.exit()` is broader than the bug and conflicts with Node's stream-drain guidance and OpenClaw's async version fast path; a narrower cleanup or drained shutdown boundary is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve the root version fast path:** `openclaw.mjs:317`
  - body: The new final `process.exit(...)` runs after `await import('./dist/entry.js')`, but the root `--version` path starts `resolveVersion().then(...)` and returns from module evaluation before that promise prints and exits. This can make `openclaw --version` exit 0 with no output; await that path or avoid a launcher-wide hard exit.
  - confidence: 0.91
- **[P2] Drain output before forcing launcher exit:** `openclaw.mjs:317`
  - body: This hard exit applies after help and routed command output. Node documents that `process.exit()` may terminate before pending stdout/stderr I/O completes, so piped help/status/JSON output can be truncated; fix the leaking handle or explicitly drain streams before exiting.
  - confidence: 0.86
