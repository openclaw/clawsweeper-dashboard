The issue reports that dynamic runtime system-prompt additions bypass the existing cache-boundary helper and destabilize Anthropic/OpenAI prompt-prefix caching.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/85375 or a narrower maintainer-approved equivalent, then close this issue once current main preserves cache-boundary placement for volatile system-prompt context with focused regression coverage.

Mostly yes: a focused boundary-aware system-prompt composition fix is the right shape, but the current open implementation still needs maintainer acceptance for the additive dynamic hook API and ordering contract.
