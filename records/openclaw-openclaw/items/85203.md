## Summary

Keep open. Current main still has the cache-boundary helper and Anthropic split policy, but several reported runtime paths still raw-join dynamic text; the fix needs scoping because some cited sites are current-turn prompt composition rather than actual system-prompt composition.

## What This Changes

The issue asks OpenClaw to route dynamic runtime prompt/system additions through cache-boundary-aware composition so Anthropic and OpenAI prompt-cache prefixes stay stable.

## Best Possible Solution

Make actual system-prompt additions boundary-aware, add focused regression coverage for Anthropic splitting and OpenAI payload stability, and keep current-turn prompt injections as user-message context unless maintainers deliberately change that contract.

## Reproduction Assessment

Yes, at source level. Current main still raw-joins several reported dynamic additions, including actual system-context and subagent system-prompt paths, but I did not run a live provider cached_tokens reproduction.

## Solution Assessment

Partly. Reusing the existing cache-boundary machinery for true system additions is the maintainable direction, but applying it to every listed current-prompt concat would be too broad without a maintainer decision.
