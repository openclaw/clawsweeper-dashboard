## Summary

Keep this open: current main still has cache-boundary-aware prompt infrastructure, but several runtime prompt-composition paths still append or join dynamic system-prompt context without consulting that boundary. The report is actionable, though the fix should be narrower than a mechanical replacement of every listed concat because some listed paths are intentionally current-turn prompt-body text rather than system-prompt text.

## What This Changes

The issue asks OpenClaw to route dynamic system-prompt additions through the cache-boundary helper so Anthropic and OpenAI prompt caching stay byte-stable.

## Best Possible Solution

Audit the remaining runtime prompt-composition sites, route confirmed dynamic system-prompt additions through a boundary-aware helper, and preserve intentional current-turn prompt-body appends with regression coverage for Anthropic and OpenAI payload shaping.

## Reproduction Assessment

Yes, at source level: building or modifying a system prompt through composeSystemPromptWithHookContext or the subagent attachment suffix path currently appends without checking SYSTEM_PROMPT_CACHE_BOUNDARY. I did not run an end-to-end provider-cache telemetry reproduction in this read-only sweep.

## Solution Assessment

The broad proposal is not the best literal fix because some listed concat paths are intentionally current-turn prompt text; the best implementation is a focused boundary-aware system-prompt composition audit with tests that preserve the documented prompt-body exceptions.
