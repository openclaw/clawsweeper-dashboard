## Summary

Keep open for maintainer design review: current main has partial estimated pre-prompt overflow checks and context-engine seams, but it does not implement the requested default budget tracker, estimated status fallback for missing usage, or stable threshold/strategy config surface.

## What This Changes

Request runtime token budget tracking with estimated-usage fallback, proactive threshold compaction, configurable compaction strategy knobs, and agent/status visibility.

## Best Possible Solution

Define a maintainer-approved design that extends the existing precheck, context-engine, status/UI, and config seams with explicit estimated-vs-reported accounting and stable compaction thresholds.

## Reproduction Assessment

Not applicable: this is a feature/design request rather than a broken existing behavior with a current-main failing reproduction. Source inspection confirms partial budget-aware behavior exists, while the remaining gaps are new status, config, and runtime policy surfaces.

## Solution Assessment

Unclear: adding a standalone ContextBudget class may not be the best boundary because current main already has precheck and context-engine seams. A safer direction is to design one coherent accounting contract that UI, status, runtime compaction, and context engines can share.
