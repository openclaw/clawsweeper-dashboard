## Summary

Keep open for maintainer design review: current main has partial preflight compaction and context-engine budget seams, but it still lacks the requested default budget tracker, estimated status fallback for missing provider usage, configurable threshold/strategy surface, and settled proactive compaction policy.

## What This Changes

Request runtime token-budget tracking with estimation fallback, proactive threshold-based compaction, configurable compaction strategy, and agent/status visibility.

## Best Possible Solution

Define a maintainer-approved design that extends the existing precheck, status, and context-engine seams, separates estimated from reported usage, and splits any remaining narrow bugs into canonical follow-ups.
