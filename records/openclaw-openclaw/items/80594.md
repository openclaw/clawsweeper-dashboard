## Summary

Keep open for maintainer design review: current main has estimated pre-prompt overflow checks and context-engine budget seams, but it still lacks the requested default runtime budget tracker, estimated status fallback for missing usage, and configurable threshold/strategy surface.

## What This Changes

Request runtime token-budget awareness, proactive compaction before overflow, estimated `/status`/UI usage fallback, agent visibility, and new compaction configuration knobs.

## Best Possible Solution

Define a maintainer-approved design that reuses the existing precheck, context-engine, and status/UI seams, labels estimated versus provider-reported usage, and coordinates overlapping follow-ups instead of adding a parallel budget system.

## Reproduction Assessment

Not applicable: this is a feature/design request, not a broken existing behavior with a current-main failing reproduction. Source inspection gives high confidence that current main has partial budget-aware prechecks but not the requested default estimated budget/status/config behavior.

## Solution Assessment

Unclear: the requested direction is plausible, but a standalone `ContextBudget` class may not be the best boundary because OpenClaw already has precheck, compaction, and context-engine APIs. Maintainers should decide the stable default policy and public config/API shape first.
