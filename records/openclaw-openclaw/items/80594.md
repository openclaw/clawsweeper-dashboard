## What This Changes

The issue requests runtime token-budget awareness, estimated usage fallback when providers omit usage, proactive threshold-based compaction, and related status/API/config surfaces.

Make a maintainer-approved design that extends the existing precheck, transcript-usage, ContextEngine, and status/UI seams; then split implementation into narrow follow-ups for fresh-token preflight, assemble-time headroom, estimated status labeling, and any approved config contract.
