## Summary

Keep open for maintainer design review: current main already has estimated pre-prompt overflow checks and context-engine extension points, but it does not fully provide the requested default budget tracker, estimated status fallback for missing provider usage, or configurable threshold/strategy surface.

## What This Changes

Request runtime token-budget tracking with estimation fallback, proactive compaction thresholds, configurable compaction behavior, and agent/status visibility.

## Best Possible Solution

Define a maintainer-approved design that reuses the existing context-engine and legacy compaction seams, separates estimated versus provider-reported usage in status/UI, and adds only stable config knobs.

## Reproduction Assessment

Not applicable as a feature request. Source inspection shows current main has some budget-aware prechecks, while status/UI still rely on fresh token snapshots and do not estimate missing provider usage.

## Solution Assessment

Unclear: a single new core ContextBudget class may not be the best boundary because OpenClaw already has context-engine plugin APIs and legacy prechecks; maintainers should decide which parts belong in default core policy versus plugin/context-engine behavior.
