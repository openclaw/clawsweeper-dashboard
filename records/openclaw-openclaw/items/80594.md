## Summary

Keep open for maintainer design review: current main has partial budget-aware prechecks and context-engine seams, but it still lacks the requested default runtime budget tracker, estimated status fallback when usage is unavailable, and stable threshold/strategy config surface.

## What This Changes

Request runtime token budget awareness with proactive context compaction, estimated usage fallback, agent/status visibility, and configurable compaction thresholds.

## Best Possible Solution

Define a maintainer-approved design that extends the existing precheck, context-engine, status, and config contracts with explicit estimated-vs-reported semantics instead of adding a parallel budget system ad hoc.

## Reproduction Assessment

Not applicable: this is a feature/design request, not a current-main bug with a failing reproduction. Source inspection confirms partial budget-aware behavior exists, while the requested default estimated fallback and threshold policy remain design work.

## Solution Assessment

Unclear: a single new ContextBudget class may not be the best boundary because OpenClaw already has precheck, context-engine, compaction, and status freshness contracts. Maintainers should decide which pieces become default core policy and which stay plugin/context-engine behavior.
