## Summary

Keep open for maintainer design review: current main has several budget-aware and preflight compaction pieces, but it does not fully provide the requested framework-level budget tracker, estimated status fallback for missing provider usage, configurable threshold/strategy surface, or agent-visible context budget API.

## What This Changes

The issue requests runtime token-budget tracking, estimated usage fallback, proactive context compaction thresholds, configurable compaction strategy, and status/API visibility for context budget.

## Best Possible Solution

Define a maintainer-approved design that reuses the existing precheck, context-engine, session-usage, and status/UI contracts while adding only stable estimated-usage and threshold configuration semantics.

## Reproduction Assessment

Not applicable: this is a feature/design request rather than a broken existing behavior. Source inspection gives high confidence that current main has partial precheck support but still lacks the requested status fallback, config, and agent budget visibility.

## Solution Assessment

Unclear: the requested direction is valid, but a new parallel ContextBudget class may not be the narrowest maintainable boundary because OpenClaw already has preflight compaction, context-engine runtime context, and session/status usage contracts.
