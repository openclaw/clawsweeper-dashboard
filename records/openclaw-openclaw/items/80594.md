## Summary

Keep open for maintainer design review: current main has partial budget-aware preflight and context-engine seams, but the requested default estimated status fallback, public threshold/strategy config, and agent-visible budget API are still not fully present.

## What This Changes

Requests framework-level runtime token budget awareness, estimated usage fallback when providers omit usage, proactive compaction thresholds, new compaction config, and agent/status budget visibility.

## Best Possible Solution

Define a shared budget snapshot contract that labels estimated versus provider-reported values, feeds status/UI and compaction consistently, and exposes only maintainer-approved config and agent API hooks.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence current-behavior check: main has partial preflight estimation, but missing status/UI fallback, config, and agent API pieces remain.

## Solution Assessment

Unclear: the requested direction is useful, but a standalone core ContextBudget class may not be the best boundary because OpenClaw already has preflight, context-engine, session-status, and UI contracts that should be unified first.
