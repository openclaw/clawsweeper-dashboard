## Summary

Keep open for maintainer design review: current main has partial context-pressure prechecks and context-engine budget seams, but it still lacks the requested default runtime budget tracker, estimated status fallback for missing provider usage, and stable threshold/strategy config surface.

## What This Changes

Request runtime token-budget tracking, proactive threshold-based compaction, estimated status fallback when provider usage is missing, and agent-visible context-budget status/config.

## Best Possible Solution

Define a maintainer-approved design that reuses the existing precheck, context-engine, status, and compaction config seams while explicitly modeling estimated versus provider-reported token data.

## Reproduction Assessment

Not applicable: this is a feature/design request, not a broken existing behavior with a current-main failing reproduction. Source inspection confirms current main has partial prechecks but not the requested estimated status/config/API behavior.

## Solution Assessment

Unclear: adding a standalone ContextBudget class may not be the best boundary because OpenClaw already has context-engine hooks, pre-prompt pressure checks, and compaction config. Maintainers should choose whether this belongs in default core policy, context-engine behavior, or a narrower status/UI fallback.
