## Summary

Keep open for maintainer design review: current main has partial reserve-based prechecks, transcript-estimation paths, context-engine budget fields, and UI notices, but it does not implement the requested default runtime budget tracker, estimated `/status` fallback for missing usage, configurable threshold/strategy surface, agent-visible budget API, or settled proactive compaction policy.

## What This Changes

The issue requests framework-level token budget awareness, estimated usage fallback, proactive compaction before overflow, configurable compaction thresholds/strategy, and agent/status visibility into context budget.

## Best Possible Solution

Define a maintainer-approved design that extends the existing precheck, context-engine, `/status`, UI, and config seams, then land narrower bug fixes like fresh-token preflight rechecks separately.

## Reproduction Assessment

Not applicable: this is a feature/design request, not a broken existing contract with a current-main failing reproduction path. Source inspection confirms partial existing mechanisms and missing requested behavior.

## Solution Assessment

Unclear: a new standalone `ContextBudget` class may duplicate existing precheck and context-engine seams. The maintainable solution should be a reviewed design that separates reported versus estimated usage and reuses current runtime contracts where possible.
