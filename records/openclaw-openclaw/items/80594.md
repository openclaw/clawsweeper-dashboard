## Summary

Keep open for maintainer design review. Current main has reserve-based pre-prompt checks, an optional mid-turn pressure precheck, and context-engine budget seams, but it still lacks the requested default estimated budget/status contract, threshold/strategy config shape, and agreed runtime policy for proactive compaction.

## What This Changes

Request adds framework-level context budget tracking with estimated token fallback, proactive threshold compaction, configurable compaction strategy, and status/agent visibility.

## Best Possible Solution

Define one shared estimated context-budget contract that reuses existing Pi precheck and context-engine seams, labels estimate provenance in status/UI, and adds only maintainer-approved config knobs.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing behavior with a current-main failing reproduction. Source inspection confirms partial existing budget checks and the missing requested status/config/API surfaces.

## Solution Assessment

Unclear: a standalone ContextBudget class may not be the best boundary because OpenClaw already has Pi reserve settings, precheck logic, and context-engine runtime context. The safer direction is a maintainer-approved shared budget snapshot contract that those existing surfaces consume.
