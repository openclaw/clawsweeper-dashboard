## What This Changes

Adds transient cooldown/backoff handling for Telegram sendChatAction failures, adjusts shared Telegram network-error tests/helpers, and adds a changelog entry.

## Best Possible Solution

Layer transient cooldown onto the current Telegram sendChatAction handler while preserving coalescing and the broad shared rate-limit helper, then land with focused tests and redacted real Telegram proof.

## Reproduction Assessment

Yes, at source level. Current main rethrows non-401 sendChatAction failures after the 401-only branch, and the current tests still expect repeated 500 failures to reject without suspending or cooling down the handler.

## Solution Assessment

No, not as currently diffed. Handler-local transient cooldown is the right boundary, but this head must preserve chat-action coalescing and shared wrapped rate-limit matching before it is a maintainable fix.
