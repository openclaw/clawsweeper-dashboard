The PR adds transient cooldown handling for Telegram sendChatAction failures, adjusts Telegram network-error detection/tests, and adds a changelog entry.

## Best Possible Solution

Layer transient cooldown into the current Telegram sendChatAction handler while preserving minInterval coalescing and shared retry classifiers, then land with focused tests and redacted live Telegram transport proof.

## Reproduction Assessment

## Solution Assessment

No, not as currently diffed. Handler-local cooldown is the right boundary, but this head must preserve chat-action coalescing and the broad shared rate-limit classifier before it is a maintainable fix.
