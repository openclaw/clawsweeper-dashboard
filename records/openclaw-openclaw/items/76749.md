## Summary

Keep open. The report matches current main by source inspection, but current source and tests intentionally couple `deny: ["write"]` to `apply_patch`, so changing it needs maintainer policy and security review rather than conservative cleanup.

## What This Changes

Request to let `tools.deny: ["write"]` hide only the raw `write` tool while leaving `apply_patch` available when otherwise enabled.

## Best Possible Solution

Maintainers should decide the intended policy semantics; if decoupling is accepted, update the matcher, regression tests, and docs while preserving explicit `apply_patch` and `group:fs` denial.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: `isToolAllowedByPolicyName("apply_patch", { deny: ["write"] })` returns false, and current tests assert that behavior.

## Solution Assessment

Unclear. Treating this as a straight bug fix is not the best path until maintainers decide whether `deny: ["write"]` should block only the raw write tool or all file-mutation capabilities.
