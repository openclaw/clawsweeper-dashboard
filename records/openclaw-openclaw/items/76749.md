## Summary

Keep open. Current main still hides `apply_patch` when `tools.deny` contains `write`, the checked-in tests assert that behavior, and an open unmerged closing PR already targets the requested policy change.

## What This Changes

Allow `tools.deny: ["write"]` to block raw writes while keeping `apply_patch` visible when otherwise enabled.

## Best Possible Solution

Maintainers should decide the intended write-deny semantics, then land #76795 or an equivalent patch with matcher, regression test, and docs updates if decoupling is accepted.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `isToolAllowedByPolicyName("apply_patch", { deny: ["write"] })` returns false via the matcher, and current tests assert that result.

## Solution Assessment

Unclear. The requested behavior is plausible, but treating it as a straight bug fix is not the best path until maintainers decide whether `deny: ["write"]` means only the raw write tool or all file-writing capabilities.
