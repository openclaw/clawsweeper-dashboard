The PR resolves bare explicit agent `--model` overrides through the configured alias index before validation, keeps provider-qualified overrides on the normalized path, and adds regression/mock coverage.

## Best Possible Solution

Land the narrow resolver reuse with the regression test if maintainers accept alias-first explicit routing, and keep any future multi-alias config shape as separate product work.
