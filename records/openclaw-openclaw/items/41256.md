## Summary

Keep open. Current main still drops MIME type when constructing Discord upload Blobs and still sends WebP images through an optimizer that returns PNG or JPEG; the issue is also paired with open closing PR #41265, so the maintainer path is review or consolidation rather than cleanup closure.

## What This Changes

The report asks Discord media uploads to preserve MIME content type and WebP/animated image fidelity instead of converting files or stripping animation frames.

## Best Possible Solution

Keep this issue open until #41265 or an equivalent maintainer-selected patch lands. The final fix should thread detected content types into normal sends, component sends, and native-command replies, preserve WebP and animated media without destructive optimization, and add focused regression coverage for those paths.

## Reproduction Assessment

Yes. The source path is high-confidence: sending WebP or animated image media through Discord reaches untyped Blob construction, and shared `loadWebMedia` still optimizes non-GIF images into PNG/JPEG unless optimization is disabled.

## Solution Assessment

Yes in direction. The narrow maintainable solution is to fix the Discord plugin upload paths plus the shared web-media preservation behavior, while keeping existing cap and access checks intact; #41265 is the current review target for that direction.
