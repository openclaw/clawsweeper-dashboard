## What This Changes

Request to preserve MIME content types for Discord media uploads and avoid silently converting WebP or animated image bytes into still optimized images.

## Best Possible Solution

Refresh and land, or replace, the open linked PR so Discord sends typed file Blobs and the media loader preserves WebP/animated sources with focused regression tests and real behavior proof.

## Reproduction Assessment

Yes, from source inspection: current main drops media.contentType before Discord upload Blob construction and can still optimize non-GIF/non-preserved image inputs. I did not run a live Discord upload in this read-only review.

## Solution Assessment

Yes, the narrow maintainable fix is to thread the loaded content type into Discord file payloads and preserve WebP/animated media before optimizer fallback. The existing open PR is the right starting point, but it needs conflict refresh and real behavior proof.
