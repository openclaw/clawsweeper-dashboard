- `impact:message-loss`: Discord attachments can be delivered with missing or generic MIME metadata, causing recipients or bots to lose the intended media interpretation.

## Summary

Keep open: current main and the latest release still show the reported Discord/media behavior, and the open closing PR has not merged. This issue should stay paired with https://github.com/openclaw/openclaw/pull/41265 or a current-main replacement until the fix lands.

## What This Changes

Discord media uploads should preserve MIME content types and WebP/animated media bytes instead of uploading typeless files or lossy still/JPEG outputs.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/41265 or a current-main replacement that threads MIME metadata through all active Discord upload paths and preserves WebP/animated media with explicit size-cap behavior.

Yes, at source level: current main still constructs typeless Discord upload blobs and still sends WebP/non-GIF images through the optimizer path. I did not run live Discord proof in this read-only review.

## Solution Assessment
