## Summary

Keep open: current main still matches the reported Discord/WebP media bug, and open closing PR #41265 remains the active implementation candidate rather than a cleanup reason to close this issue.

## What This Changes

The issue requests preserving MIME content types for Discord media uploads and preventing WebP or animated media from being destructively optimized into still JPEG/PNG output.

## Best Possible Solution

Review and land #41265 or an equivalent narrow patch that threads MIME types into Discord upload payloads, preserves WebP and animated media, and adds regression coverage around those paths.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: Discord media sends still construct untyped Blobs from loaded buffers, and WebP media still reaches the optimizer path that emits PNG or JPEG.

## Solution Assessment

Yes in direction. The maintainable fix is a narrow Discord/media patch that carries detected MIME types into upload files and skips destructive optimization for WebP or animated media while preserving the existing safety checks.
