## Summary

Keep open: current main still matches the reported Discord/WebP media bug, and open closing PR #41265 is the active implementation candidate rather than a cleanup reason to close this issue.

## What This Changes

The issue asks OpenClaw to preserve MIME content types on Discord media uploads and avoid destructive optimization of WebP or animated images.

## Best Possible Solution

Review and land #41265 or an equivalent narrow patch that threads content types through Discord file payloads, preserves WebP and animated media, and adds focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: Discord media sends still build untyped Blobs from loaded buffers, and WebP image media still reaches the optimizer path that emits PNG or JPEG.

## Solution Assessment

Yes in direction. The best fix is a narrow Discord/media patch that carries detected MIME types into upload files and skips destructive optimization for WebP or animated media while keeping the existing media safety checks intact.
