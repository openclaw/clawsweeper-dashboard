## What This Changes

The issue reports that Discord media uploads lose MIME type and that WebP or animated media are optimized in a way that loses format fidelity.

## Best Possible Solution

Review, repair, and land https://github.com/openclaw/openclaw/pull/41265 or an equivalent narrow patch that threads content type through Discord payloads, preserves WebP and animated media, and adds focused regression coverage.

## Solution Assessment

Yes in direction. The narrow maintainable fix is to thread `contentType` through the Discord file payload and bypass lossy optimization for WebP or animated media while preserving existing size limits.
