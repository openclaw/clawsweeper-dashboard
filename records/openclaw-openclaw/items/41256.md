## What This Changes

Discord media uploads should preserve attachment MIME type and avoid silently converting WebP or animated images into still JPEG/PNG output.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/41265 or an equivalent current-main patch that threads MIME metadata through all Discord upload paths and preserves WebP/animated media within the existing size limits.

Yes. The current source path is high-confidence: Discord upload builders still construct typeless blobs, and WebP/non-GIF images still reach an optimizer path that emits PNG or JPEG.

## Solution Assessment

Yes in direction, but not complete yet. The maintainable fix is to thread existing MIME metadata through the Discord payload contract and preserve animated/WebP bytes without weakening size limits; the linked PR still needs repair/proof before this can close.
