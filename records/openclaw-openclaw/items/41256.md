## Summary

Keep this issue open. Current main still matches the reported Discord/WebP media bugs: normal sends, component sends, and native-command replies discard loaded MIME types before upload, and shared web media still optimizes non-GIF images, including WebP, into JPEG or PNG. The issue is already paired with open closing PR #41265, so the maintainer path is review or consolidation rather than closure or a new repair job.

## What This Changes

Discord media uploads should preserve MIME content type, and shared web media handling should preserve WebP and animated images without silently converting or flattening them.

## Best Possible Solution

Land a focused fix, likely by reviewing or consolidating #41265, that threads detected `contentType` through every Discord file upload path and preserves WebP/animated media in `src/media/web-media.ts` while keeping current safety checks and adding regression coverage for normal sends, component sends, native-command replies, WebP preservation, and animated-media preservation.
