## Summary

Keep open: current main still does not implement the QQ Bot route-persistence fix, but this PR is not merge-ready because its direct-message guard excludes only `c2c` while QQ Bot also has `dm` direct-message events. The patch changes persistent session routing and later announce delivery, so the guard needs repair, focused coverage, and refreshed real behavior proof before merge.

## What This Changes

The PR adds QQ Bot metadata for session-backed announce-target lookup and writes QQ Bot last-route deliveryContext for non-`c2c` inbound turns.

## Best Possible Solution

Keep the PR open and revise it to persist only group/guild route metadata, with focused regression tests and redacted QQ Bot proof for the corrected route behavior.

## Reproduction Assessment

Yes. Source inspection shows `DIRECT_MESSAGE_CREATE` maps to `event.type === "dm"`, the access stage treats only `guild` and `group` as group chats, and the PR guard only excludes `c2c`.

## Solution Assessment

No, not as written. Plugin-owned route persistence plus metadata-gated announce lookup is the right layer, but the route write should be limited to `inbound.isGroupChat` or explicit `group`/`guild` events and covered by focused tests.
