## Summary

Keep open: current main still lacks the QQ Bot route persistence and announce-target opt-in, but this PR still writes last-route state for reachable `dm` direct-message turns and lacks focused route-persistence tests/proof.

## What This Changes

The PR adds QQ Bot announce-target session lookup metadata and persists QQ Bot inbound last-route deliveryContext for non-`c2c` outbound dispatch turns.

## Best Possible Solution

Revise the PR to persist only group/guild route metadata, add focused regression tests, and provide redacted live QQ Bot proof for the corrected route behavior.

## Reproduction Assessment

Yes. Source inspection shows `DIRECT_MESSAGE_CREATE` maps to `event.type === "dm"`, the access stage treats only `guild` and `group` as group chats, and the PR guard only excludes `c2c`.

## Solution Assessment

No, not as written. Plugin-owned route persistence plus metadata-gated announce lookup is the right layer, but the route write should be limited to `inbound.isGroupChat` or explicit `group`/`guild` events and covered by focused tests.
