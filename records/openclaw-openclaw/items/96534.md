## Summary

Keep open: current main still has the memory_search fallback recovery gap, and the linked open candidate fix at https://github.com/openclaw/openclaw/pull/96651 has not merged yet.

## What This Changes

The issue asks for Gateway memory_search to recover from a fallback embedding provider after the primary provider becomes healthy again, without requiring a full process restart.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/96651 or a replacement that makes every fallback-active entry path recoverable while preserving the index identity guard.

## Reproduction Assessment

Yes, at source level: createEmbeddingProvider can return a fallbackFrom result, the manager then marks providerInitialized true, and later searches do not re-resolve the primary provider. I did not run the reporter's live Ollama outage sequence in this read-only review.
