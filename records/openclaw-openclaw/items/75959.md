## Summary

Keep open. Current main still routes the bundled Kimi provider through the Anthropic Messages transport at the old coding base URL while advertising Kimi Code models as image-capable, and open PR #76002 is the active unmerged fix path.

## What This Changes

The issue requests Kimi Code Plan image attachments to work by routing the bundled Kimi provider through Kimi's OpenAI-compatible chat completions endpoint with the existing approved User-Agent.

## Best Possible Solution

Review and land PR #76002, or an equivalent provider-owned Kimi default change, switching defaults to `openai-completions` at `https://api.kimi.com/coding/v1` while preserving explicit overrides, the built-in User-Agent, tests, docs, and changelog coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows Kimi Code models advertise image input while default provider and onboarding config still select `anthropic-messages` at the old coding base URL; the reporter also supplied failing and working curl evidence.

## Solution Assessment
