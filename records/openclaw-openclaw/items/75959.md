## Summary

Keep open. Current main still advertises image-capable Kimi Code models while routing the bundled Kimi provider and onboarding defaults through `anthropic-messages` at `https://api.kimi.com/coding/`; open PR #76002 is the active closing implementation path, so this issue should remain open until that PR lands or is replaced.

## What This Changes

Support Kimi Code Plan image attachments by routing bundled Kimi requests through the OpenAI-compatible coding endpoint with the existing accepted User-Agent.

## Best Possible Solution
