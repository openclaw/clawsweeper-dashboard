## Summary

Current main and the shipped v2026.4.29 release already cover the reported Kimi image path: the bundled Kimi Coding provider advertises image input and sends the reporter's whitelisted User-Agent by default, while the OpenAI-compatible `/chat/completions` route remains available through the existing explicit provider API override.

## What This Changes

The issue asks OpenClaw to make `kimi/kimi-code` handle image attachments against the Kimi Code Plan API without 403 failures.

## Best Possible Solution

Keep the current bundled Kimi Coding default and document/use explicit `api: "openai-completions"` only for users who intentionally want Kimi's OpenAI-compatible route.

## Reproduction Assessment

Yes. The issue includes a concrete Kimi Code Plan curl failure for OpenClaw 2026.4.27, and current-main verification follows the provider catalog, image transport, and shipped release files that now avoid the reported default User-Agent gap.

## Solution Assessment

Yes. The current solution is narrower than switching the bundled default endpoint: it keeps the Kimi Coding provider on the documented Anthropic-compatible agent path with image input and a working default User-Agent, while preserving explicit OpenAI-compatible configuration for `/chat/completions`.
