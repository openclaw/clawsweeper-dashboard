## Summary

Keep open. Current main still routes the bundled Kimi provider through the Anthropic Messages transport while advertising image-capable Kimi models, and the open closing PR #76002 is the active implementation path rather than a reason to close this issue before merge.

## What This Changes

Support image analysis for the bundled Kimi Code Plan provider when users attach images to `kimi-code` conversations.

## Best Possible Solution

The Kimi plugin defaults should route image-capable `kimi-code` requests through Kimi's OpenAI-compatible coding endpoint while preserving explicit provider overrides and the existing whitelisted User-Agent.

## Reproduction Assessment

Yes. Source inspection on current main shows Kimi models declare image input while default Kimi configuration still selects `anthropic-messages` and the old coding base URL; the reporter also provided concrete failing and working curl evidence.

## Solution Assessment

Yes. A narrow provider-owned default `api`/`baseUrl` change, as proposed by #76002, is the maintainable fix; the issue should remain open until that PR is merged or rejected.
