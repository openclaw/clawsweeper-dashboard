## Summary

Keep open. Current main still advertises Kimi Code models as image-capable while routing the bundled Kimi provider through the Anthropic Messages transport and old coding base URL; open PR #76002 is the active closing implementation path, so this issue should stay open until that PR merges or is rejected.

## What This Changes

Route bundled Kimi Code Plan image requests through Kimi's OpenAI-compatible coding endpoint instead of the current Anthropic Messages default.

## Best Possible Solution

Review and land or close PR #76002; the desired end state is for Kimi defaults, onboarding, tests, docs, and changelog to agree on the OpenAI-compatible coding endpoint while preserving explicit user overrides.

## Reproduction Assessment

Yes. Source inspection on current main shows Kimi models marked image-capable while defaulting to the old Anthropic Messages route, and the reporter supplied concrete failing and working curl evidence; I did not run the paid live API path in this read-only review.

## Solution Assessment

Yes. A narrow provider-owned default `api` and `baseUrl` change, as proposed by PR #76002, is the maintainable fix because the OpenAI-compatible path already serializes images in the requested `image_url` format.
