## Summary

Keep open. Current main still advertises Kimi Coding models as image-capable while routing the bundled provider through `anthropic-messages` at `https://api.kimi.com/coding/`, and open PR #76002 references this issue with closing syntax as the active fix candidate.

## What This Changes

Request Kimi Coding image attachments to use Kimi Code Plan's OpenAI-compatible chat completions endpoint while keeping a whitelisted User-Agent.

## Best Possible Solution

Land or reject the narrow provider-owned fix in #76002; the desired end state is Kimi Coding defaults that use Kimi's OpenAI-compatible endpoint for image-capable models while preserving explicit config overrides and whitelisted headers.

## Reproduction Assessment

Yes. Current main's source and tests show the default Kimi Coding path still uses `anthropic-messages` while declaring image input, and the reporter supplied concrete curl evidence for the 403/200 split.

## Solution Assessment

Yes. A provider-owned default change is the narrowest maintainable direction because the generic `openai-completions` path already targets `/chat/completions` and serializes image payloads as `image_url` blocks.
