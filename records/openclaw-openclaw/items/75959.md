## Summary

Keep open. Current main still advertises `kimi-code` as image-capable while the bundled Kimi provider and onboarding defaults use the Anthropic Messages transport, and open PR #76002 is already the active closing implementation candidate.

## What This Changes

Enable Kimi Code Plan image analysis by routing the bundled Kimi provider through a Kimi-compatible OpenAI chat completions endpoint while preserving the accepted User-Agent.

## Best Possible Solution

Review and land PR #76002, or an equivalent provider-owned patch, that switches Kimi defaults to `openai-completions` and `https://api.kimi.com/coding/v1` while preserving user overrides, the accepted User-Agent, tests, and changelog coverage.

## Reproduction Assessment

Yes. Current main declares the Kimi Coding models image-capable while routing them through `anthropic-messages` at `https://api.kimi.com/coding/`, and the reporter supplied concrete curl evidence for the 403 image-request failure.

## Solution Assessment

Yes. A narrow Kimi provider default change is the maintainable fix boundary because the bug is provider-owned and the OpenAI-compatible image payload path already exists; no core API change is indicated by the current evidence.
