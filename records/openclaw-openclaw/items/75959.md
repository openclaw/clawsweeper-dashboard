## Summary

Keep open. Current main still advertises image-capable Kimi Code models while routing the bundled Kimi provider and onboarding defaults through `anthropic-messages` at the old `https://api.kimi.com/coding/` base URL; open PR #76002 is the active closing implementation path, so this issue should remain open until that PR lands or is replaced.

## What This Changes

The issue requests that the bundled Kimi provider handle Kimi Code Plan image attachments through the OpenAI-compatible `/chat/completions` endpoint with a Kimi-accepted User-Agent.

## Best Possible Solution

Review and land PR #76002 or an equivalent Kimi-plugin change that switches the bundled defaults to `openai-completions` at `https://api.kimi.com/coding/v1`, preserves explicit overrides, and updates focused Kimi tests plus the changelog.
