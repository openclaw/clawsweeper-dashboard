## Summary

Keep open. Current main and latest release still configure the bundled Kimi provider for `anthropic-messages` on `https://api.kimi.com/coding/` while advertising image input; open PR #76002 is the active closing implementation path but is not merged.

## What This Changes

Make the bundled Kimi Code Plan provider send image-capable requests through Kimi's OpenAI-compatible coding endpoint instead of the Anthropic Messages route.

## Best Possible Solution

Review and land or reject PR #76002; the desired end state is a provider-owned default switch to `openai-completions` and `https://api.kimi.com/coding/v1` while preserving explicit user overrides.

## Reproduction Assessment

Yes. Source inspection on current main shows an image-capable Kimi catalog routed through the Anthropic transport, and the reporter provided concrete failing and working curl evidence; live verification was not possible because no Kimi API key was present in the environment/profile check.

## Solution Assessment

Yes as a direction. The narrow provider-owned default `api` and `baseUrl` switch proposed by PR #76002 matches the existing OpenAI-compatible image serialization path; this issue should stay open until that PR merges or is replaced.
