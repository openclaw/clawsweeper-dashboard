## Summary

Keep open. Current main still publishes Kimi Code as image-capable while routing bundled defaults and onboarding through `anthropic-messages` at `https://api.kimi.com/coding/`; open PR #76002 is the active closing implementation and should be reviewed or rejected before this issue closes.

## What This Changes

The issue asks for the bundled Kimi Code Plan provider to handle image attachments by using Kimi's OpenAI-compatible coding endpoint while keeping the approved User-Agent and standard `image_url` payloads.

## Best Possible Solution

Review and land or reject PR #76002; the desired end state is a provider-owned Kimi default that sends image-capable `kimi-code` traffic through the OpenAI-compatible coding endpoint while preserving explicit user overrides and the existing approved User-Agent.

## Reproduction Assessment

Yes. Source inspection on current main shows Kimi models advertise image input while default catalog and onboarding still choose the Anthropic transport and old coding base URL; the reporter also supplied concrete failing and working curl evidence.

## Solution Assessment

Yes. The narrow provider-owned default `api` and `baseUrl` switch proposed by PR #76002 is the maintainable fix boundary; this issue should remain open until that PR merges or maintainers choose a different route.
