## Summary

Keep open. Current main still configures the bundled Kimi provider with the Anthropic Messages transport and the old coding base URL while advertising image input, and the linked open PR #76002 is the active implementation path rather than a reason to close this issue before merge.

## What This Changes

The issue asks for Kimi Code Plan image attachments to be sent through Kimi's OpenAI-compatible coding endpoint with an accepted User-Agent.

## Best Possible Solution

Review and either land or reject PR #76002; the desired end state is a provider-local Kimi default that routes image-capable `kimi-code` requests through the OpenAI-compatible coding endpoint while preserving explicit overrides and the existing accepted User-Agent.

Yes. Source inspection on current main shows Kimi models declare image input while default Kimi configuration still selects `anthropic-messages` and the old coding base URL; the reporter also supplied concrete failing and passing curl evidence.

## Solution Assessment
