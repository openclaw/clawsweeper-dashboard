## Summary

Keep open. Current main has useful mitigations for model prewarm, pricing startup, Telegram transport fallback, proxy dispatchers, and event-loop diagnostics, but there is still no proof that the broader native-Windows Gateway-process HTTP failure across Telegram REST, Discord REST, pricing/catalog fetches, and local handshakes is fixed or superseded.

The report says outbound HTTP/HTTPS fails from the running Gateway process on native Windows while WebSockets and standalone fetches work, and asks for a model-prewarm skip workaround.
