## Summary

Keep open. Current main contains useful mitigations for startup prewarm, model-pricing fetches, Telegram control calls, proxy handling, and event-loop diagnostics, but those are scoped workarounds and do not prove the reported native Windows gateway-process failure across Telegram REST, Discord REST, model/catalog HTTP, and local handshakes is fixed.

## What This Changes

The issue reports that native Windows 10 with Node 22 can lose outbound HTTP/HTTPS inside the running Gateway process while WebSocket paths keep working, and asks for a startup model-prewarm skip workaround.

## Best Possible Solution

Reproduce on native Windows with Node 22 and Node 24 using multiple Discord plus Telegram accounts, instrument undici/global fetch dispatcher state, DNS/connect/TLS timing, active handles, event-loop delay, and channel startup concurrency, then land a targeted Gateway or plugin-transport fix that preserves the existing proxy and SSRF boundaries. Keep the prewarm and pricing-disable workarounds as operator escape hatches.
