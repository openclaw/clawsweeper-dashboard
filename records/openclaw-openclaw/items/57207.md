## Summary

Keep open. Current main still checks the sibling module `package.json` before the bundled `dist/extensions/memory-lancedb/package.json`, and the PR targets a credible packaged `memory-lancedb` startup regression. The runtime change is narrow, but the PR should not merge as-is because it lacks the required changelog entry and its new test does not fail if the ordering fix is reverted.

## What This Changes

The PR reorders `memory-lancedb` manifest candidate lookup to prefer the bundled extension `package.json` before `dist/package.json` and adds a regression test for layouts where `dist/package.json` is absent.

## Best Possible Solution

Land a narrow memory-lancedb runtime fix that makes the bundled extension manifest the first packaged candidate, adds an order-sensitive regression test, and records the user-facing startup fix in `CHANGELOG.md`. No core plugin loader or package-root migration is needed for this follow-up.

## Solution Assessment

Is this the best way to solve the issue? No, not quite as submitted: the runtime change is the right narrow fix, but the PR should add the required changelog entry and strengthen the test so it fails when `dist/package.json` is consulted before the extension manifest.
