This PR restores `PluginRuntime.channel.turn` as a deprecated object-identical alias of `channel.inbound`, updates SDK types/mock/tests/API hash, and adjusts SDK docs.

## Best Possible Solution

Land the object-identical alias after maintainer acceptance and green exact-head checks, keeping `inbound.*` canonical and `turn.*` documented as deprecated compatibility only.

## Reproduction Assessment

## Solution Assessment

Yes, if maintainers accept the compatibility contract: restoring `turn` as the same object as `inbound` is the narrow core-boundary fix for all old compiled channel plugins. Updating only one plugin would leave other shipped old-plugin installs exposed.
