## What This Changes

## Best Possible Solution

Land the object-identical runtime alias after preserving both `inbound` and legacy `turn` mock override spellings, while keeping docs and new plugin guidance pointed at `channel.inbound`.

## Reproduction Assessment

Yes. Current main and `v2026.6.6` expose `channel.inbound` without `channel.turn`, while the published `@openclaw/bluebubbles@2026.5.7` package calls `core.channel.turn.run` and allows newer OpenClaw cores.

## Solution Assessment

No, not quite as-is. The production alias is the right layer for the shipped plugin API break, but the SDK mock should preserve legacy `turn` override spelling and maintainers should accept the restored deprecated contract before merge.
