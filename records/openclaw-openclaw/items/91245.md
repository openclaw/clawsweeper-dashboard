## Summary

Keep this PR open: the runtime alias is a plausible narrow compatibility fix for old shipped channel plugins, but the branch is currently conflicting and the SDK runtime mock drops overrides supplied through the restored deprecated alias.

## What This Changes

The PR restores `runtime.channel.turn`/`core.channel.turn` as a deprecated alias of `channel.inbound`, updates SDK types/docs/API baseline, and adds runtime/mock regression coverage.

## Best Possible Solution

Keep `channel.inbound` canonical, preserve `channel.turn` as the same runtime object for shipped-plugin compatibility, fix the mock so legacy `turn` overrides feed that same object, then rebase and land only with maintainer acceptance of the deprecated API contract.

## Reproduction Assessment

Yes from source and package inspection: current main and `v2026.6.6` lack `channel.turn`, while published old channel packages dereference `core.channel.turn.run`. I did not run a live Feishu or BlueBubbles credentialed repro.

## Solution Assessment

Mostly yes, but not ready as-is: the object-identical alias in `createRuntimeChannel()` is the narrow owner-boundary fix, while the SDK mock should also preserve legacy `turn` overrides and the deprecated contract needs maintainer acceptance.
