## Summary

Keep this PR open: current main and `v2026.6.6` still lack the legacy `channel.turn` runtime surface, and a published BlueBubbles package plus linked user report show old channel plugins still call it. ## What This Changes

Restores `PluginRuntime.channel.turn` as a deprecated alias of `channel.inbound`, with runtime/types, SDK mock tests, generated API hash, and SDK docs updates.

## Best Possible Solution

Preserve explicit `turn` mock overrides, rebase against current main, regenerate/check the Plugin SDK API baseline, then land the object-identical alias if maintainers accept it as the deprecated compatibility contract.

## Reproduction Assessment

Yes, from source and package inspection: current main and `v2026.6.6` expose `channel.inbound` without `channel.turn`, while `@openclaw/bluebubbles@2026.5.7` calls `core.channel.turn.run(...)`. I did not run a live credentialed old-plugin channel.

## Solution Assessment

No, not as-is. The runtime alias is the best production layer versus per-plugin updates, but the PR still needs the explicit `channel.turn` mock override edge fixed and a clean rebase with refreshed API baseline.
