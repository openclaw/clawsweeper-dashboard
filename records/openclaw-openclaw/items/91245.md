Keep open for maintainer review: current main and the latest release still lack the old runtime alias, a shipped Feishu package uses `core.channel.turn.run`, and the PR is a narrow object-identical compatibility repair with no blocking code findings. ## What This Changes

The PR restores `channel.turn` as a deprecated alias for `channel.inbound` in the plugin runtime and updates SDK docs, types, API baseline hashes, mocks, and regression tests.

## Best Possible Solution

Land the alias only if maintainers accept it as a deprecated shipped-plugin compatibility contract, keeping `channel.inbound` canonical and `channel.turn` object-identical with focused runtime, SDK API, docs, and formatting checks on the final head.

## Reproduction Assessment

Yes from source: current main and the latest release expose `channel.inbound` without `channel.turn`, while a published Feishu package dereferences `core.channel.turn.run`. This read-only review did not run a live Feishu credentialed repro.

## Solution Assessment

Yes, with maintainer acceptance: placing an object-identical alias in `createRuntimeChannel()` and reflecting it in the SDK type/mock is the narrow owner-boundary fix. Updating only the Feishu plugin would not protect other shipped third-party plugins compiled against the old runtime property.
