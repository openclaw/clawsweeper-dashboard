## Best Possible Solution

Land the object-identical alias after fixing mock override handling and explicitly treating `channel.turn` as a deprecated shipped-plugin compatibility surface while keeping new plugin docs on `channel.inbound`.

## Reproduction Assessment

## Solution Assessment

Not quite as-is. The production alias is the right core compatibility layer for the shipped plugin API break, but the SDK mock should preserve legacy `turn` override spelling before this is merge-ready.
