## Summary

Keep open. ## What This Changes

Restores `PluginRuntime.channel.turn` as a deprecated alias of `channel.inbound`, with runtime/types/mock/tests, API baseline hash, and SDK docs updated around the compatibility surface.

## Best Possible Solution

Land a rebased alias only if maintainers accept `channel.turn` as deprecated shipped-plugin compatibility, keeping `channel.inbound` canonical and object-identical.

## Reproduction Assessment

Yes, from source and package inspection: current main/latest release expose `channel.inbound` without `turn`, while older published Feishu and BlueBubbles plugin tarballs call `core.channel.turn.run`. I did not run a credentialed live Feishu repro.

## Solution Assessment

Yes, with maintainer acceptance: adding an object-identical alias in `createRuntimeChannel()` and reflecting it in the SDK type/mock/docs is the narrow core compatibility fix. Updating only Feishu would not protect other shipped plugins compiled against the old runtime property.
