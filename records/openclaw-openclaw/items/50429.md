## Summary

Keep this PR open, but not mergeable as-is: the reply-dispatcher fallback addresses a real Feishu table-limit failure, while the current patch still catches every card send error and can resend block/partial content or duplicate already-delivered chunks.

## What This Changes

This PR wraps Feishu reply-dispatcher non-streaming card delivery in a catch block that retries failed card sends as plain text and adds two fallback tests.

## Best Possible Solution

Constrain the reply-dispatcher fallback to final Feishu table-limit failures before any card chunk has been delivered, reusing a shared 230099 / 11310 classifier if #73651 lands first.

## Reproduction Assessment

Yes. #43690 provides concrete Feishu 230099 / 11310 logs, current main still has an uncaught reply card send path, and the PR test simulates the relevant `sendStructuredCardFeishu` rejection.

## Solution Assessment

No. The fallback direction is useful, but this patch is broader than the safe fix because it catches all card send failures and retries full text without final-payload or partial-delivery guards.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Avoid retrying after delivered card chunks:** `extensions/feishu/src/reply-dispatcher.ts:419-439`
  - body: If chunk 2 or later fails, earlier card chunks have already been sent by `sendChunkedTextReply`, but this catch retries the entire reply as text. That can duplicate content and first-chunk mentions; track whether any card chunk was delivered or resume from the failed chunk only.
  - confidence: 0.88
- **[P2] Limit fallback to final replies:** `extensions/feishu/src/reply-dispatcher.ts:439-448`
  - body: When streaming startup has failed, `info.kind === "block"` can still reach this catch and be resent through `sendMessageFeishu`. That bypasses the current block suppression and can expose partial or preview content before the final answer, so the fallback should drop or ignore non-final payloads.
  - confidence: 0.84
- **[P2] Classify table-limit errors before falling back:** `extensions/feishu/src/reply-dispatcher.ts:439-445`
  - body: The catch handles every `sendStructuredCardFeishu` failure. Auth, permission, malformed-card, and rate-limit errors should continue to fail normally; only the known Feishu 230099 / 11310 table-limit rejection should degrade to plain text.
  - confidence: 0.8
