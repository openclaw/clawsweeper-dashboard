## Summary

Keep this PR open. Current main still has the Feishu reply-dispatcher card-failure gap, but the proposed fallback is too broad because it retries every card error as full plain text, including after partial card delivery and for block preview payloads; #73651 is related but outbound-only and does not supersede this reply-dispatcher work.

## What This Changes

This PR wraps non-streaming Feishu card reply delivery in a plain-text fallback when `sendStructuredCardFeishu` throws and adds reply-dispatcher tests for fallback and successful card delivery.

## Best Possible Solution

Constrain the reply-dispatcher fallback to known Feishu table-limit failures, final user-visible payloads, and cases where no prior card chunk has been delivered; add regression coverage for non-table card errors, block payloads, and partial multi-chunk delivery.

## Reproduction Assessment

Yes. #43690 includes concrete Feishu 230099 / 11310 logs, current main still awaits `sendStructuredCardFeishu` without fallback in the reply path, and the PR's mocked rejection test exercises the intended card-failure path.

## Solution Assessment

No. The fallback direction is useful, but the current patch is not the narrowest maintainable solution because it catches all card send failures and retries the full reply as text without payload-kind or partial-delivery guards.

- **[P2] Avoid retrying after delivered card chunks:** `extensions/feishu/src/reply-dispatcher.ts:419-439`
  - body: If chunk 2 or later fails, `sendChunkedTextReply` has already delivered earlier card chunks, but this catch retries the entire reply as text, causing duplicated content and first-chunk mentions.   - confidence: 0.88
- **[P2] Limit fallback to final reply payloads:** `extensions/feishu/src/reply-dispatcher.ts:439-448`
  - body: When streaming startup has failed, `info.kind === "block"` can still reach this catch path. Sending those failed card chunks through `sendMessageFeishu` bypasses the current block suppression and can expose partial or preview text before the final answer.
  - confidence: 0.84
- **[P2] Classify table-limit errors before falling back:** `extensions/feishu/src/reply-dispatcher.ts:439-445`
  - body: The catch handles every `sendStructuredCardFeishu` failure. Auth, permission, malformed-card, and rate-limit failures should continue to fail normally; only the known Feishu 230099 / 11310 table-limit rejection should degrade to plain text.
  - confidence: 0.8
