## What This Changes

The PR wraps Feishu reply-dispatcher structured-card delivery with a plain-text fallback and adds dispatcher tests for failed and successful card sends.

## Best Possible Solution

Add a table-limit-only final-reply fallback that reuses the existing plain-text/table-conversion path only before any card chunk has been delivered, while coordinating outbound coverage through the related outbound PR and the linked issue.

## Reproduction Assessment

Yes, source-level. The linked issue includes concrete Feishu 230099/11310 logs, and current main still routes Markdown-table auto/card replies into sendStructuredCardFeishu without a table-limit fallback; I did not run a live Feishu send.

## Solution Assessment

No for the current PR head. Plain-text fallback is the right direction, but the patch should classify the Feishu table-limit error, limit fallback to final replies, and avoid replaying already-delivered card chunks.
