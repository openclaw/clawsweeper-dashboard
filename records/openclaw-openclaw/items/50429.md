## Summary

Keep open: current main still lacks a Feishu reply-dispatcher fallback for table-limit card failures, and the related outbound PR does not cover this surface, but this PR head is still too broad and lacks real after-fix Feishu proof.

## What This Changes

The PR wraps Feishu non-streaming reply card delivery in a catch and retries failed card sends as plain text, adding dispatcher tests for fallback and normal card success.

## Best Possible Solution

Add a table-limit-only final-reply fallback in the Feishu reply dispatcher that retries through the existing plain-text/table-conversion path only before any card chunk has been delivered, reusing a shared 230099 / 11310 classifier if the outbound PR lands first.

## Reproduction Assessment

Yes, at source level: current main routes table-heavy Feishu replies to sendStructuredCardFeishu without a catch, and the linked issue provides concrete 230099 / 11310 logs. I did not run a live Feishu send in this read-only review.

## Solution Assessment

No. Plain-text fallback is the right direction, but the current patch needs to classify the table-limit error, avoid replaying already-delivered chunks, and ignore non-final block payloads.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Avoid retrying after delivered card chunks:** `extensions/feishu/src/reply-dispatcher.ts:419-438`
  - body: If chunk 2 or later fails, sendChunkedTextReply has already delivered earlier card chunks, but the catch retries the whole reply as text. That can duplicate content and first-chunk mentions; only retry before any card chunk is sent, or resume from the failed chunk.
  - confidence: 0.88
- **[P2] Limit fallback to final replies:** `extensions/feishu/src/reply-dispatcher.ts:438-448`
  - body: When streaming startup has failed, block payloads can reach this catch and be sent as normal Feishu text. That bypasses the existing block suppression for internal preview content and can send a partial snippet before the final answer, so the fallback should ignore non-final payloads.
  - confidence: 0.84
- **[P2] Classify table-limit errors before falling back:** `extensions/feishu/src/reply-dispatcher.ts:438-445`
  - body: The catch handles every sendStructuredCardFeishu failure. Auth, permission, malformed-card, and rate-limit errors should continue to fail normally; only the known Feishu 230099 / 11310 table-limit rejection should degrade to plain text.
  - confidence: 0.82
