## Summary

Keep open. Current main still ignores structured `stdout` in Swift Chat UI tool-result formatting, and this PR is a focused useful fix, but it still needs the required changelog entry and PR issue-link cleanup before merge.

## What This Changes

This PR updates the OpenClawKit Chat UI formatter to prefer structured `stdout` in successful JSON tool results and adds Swift formatter tests for stdout precedence.

## Best Possible Solution

Land the stdout precedence behavior with its regression tests, add a single Unreleased changelog entry crediting @Narcooo, and correct the unrelated `Closes #38133` linkage without expanding the PR into stderr fallback policy.

## Reproduction Assessment

Yes. A source-level reproduction is clear on current main: `ToolResultTextFormatter.format` with a JSON payload containing only `stdout` reaches the empty fallback, while a payload with both `stdout` and `message` returns the generic message because `stdout` is never read.

## Solution Assessment

Yes, with the small metadata cleanup. The proposed stdout-before-message formatter precedence is the narrow maintainable fix, preserves error and nodes-summary behavior, and the reverted stderr fallback should stay out of scope.
