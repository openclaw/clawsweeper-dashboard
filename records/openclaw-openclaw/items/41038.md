## Summary

Keep open. Current main still hides stdout-only structured tool results, and this PR is a focused fix rather than obsolete cleanup; it should wait for the required changelog and metadata cleanup before merge.

## What This Changes

This PR makes the OpenClawKit Chat UI formatter prefer structured `stdout` in successful tool-result JSON and adds Swift tests for stdout precedence.

## Best Possible Solution

Land the stdout precedence behavior with its regression tests after adding a single Unreleased changelog entry and correcting the unrelated issue reference, without broadening the PR into stderr fallback policy.

## Reproduction Assessment

Yes. A source-level reproduction is clear on current main: `ToolResultTextFormatter.format` with stdout-only structured JSON returns an empty string, and a payload with both `stdout` and `message` returns the generic message because `stdout` is never read.

## Solution Assessment

Yes, with small cleanup. Preferring `stdout` before generic message fields is the narrow maintainable fix and preserves the existing error and nodes-summary paths.
