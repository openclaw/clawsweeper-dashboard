## Summary

Keep open. Current main still suppresses stdout-only structured tool results in OpenClawKit Chat UI, and this PR is a focused functional fix; it needs release-note and PR metadata cleanup before merge.

## What This Changes

This PR adds stdout precedence to the OpenClawKit Chat UI tool-result formatter and adds formatter tests for exec-style payloads.

## Best Possible Solution

Land the stdout precedence behavior with regression tests, add one Unreleased changelog entry crediting the contributor, simplify the redundant guard, and remove or correct the unrelated closing reference.

## Reproduction Assessment

Yes. A source-level reproduction is clear: on current main, stdout-only JSON has no handled field and returns an empty formatter result, while payloads with both `stdout` and `message` return the generic message because `stdout` is never read.

## Solution Assessment

Yes, with small cleanup. Prioritizing non-empty `stdout` after error and nodes-summary paths but before generic message fields is the narrow maintainable fix; stderr fallback should remain out of scope unless separately requested.
