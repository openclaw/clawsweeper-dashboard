## What This Changes

The PR updates the OpenClawKit Chat UI tool-result formatter to prefer structured `stdout` output for successful payloads and adds stdout-precedence formatter tests.

## Best Possible Solution

## Reproduction Assessment

Yes, at source level. Current main does not inspect `stdout` in `renderDictionary`, and `ToolResultCard` suppresses empty formatted output, so a stdout-only structured result can disappear from the visible Chat UI.

## Solution Assessment

Yes, with proof and metadata cleanup. Preferring trimmed `stdout` after error and nodes-summary handling but before generic message fields is the narrow maintainable fix for this formatter path.
