## Summary

Keep open. Current main still drops structured `stdout` tool output in OpenClawKit Chat UI, and this PR is a narrow useful fix, but it still needs release-note and PR-metadata cleanup before landing.

## What This Changes

The PR adds `stdout` extraction ahead of generic message/result/detail fallback in `ToolResultTextFormatter` and adds Swift tests for stdout precedence over message/stderr fields.

## Best Possible Solution

Land the stdout precedence behavior with its regression tests, add one Unreleased changelog entry crediting @Narcooo, and correct the unrelated closing reference without expanding scope into stderr fallback policy.

## Reproduction Assessment

Yes. On current main, a JSON payload containing only `stdout` has no matching key in `renderDictionary` and returns an empty string, while a payload containing both `stdout` and `message` returns the generic message because stdout is never read.

## Solution Assessment

Yes, with small cleanup. Prioritizing non-empty stdout before the generic message fallback is the narrow maintainable fix and preserves the existing error and nodes-summary paths.
