## Summary

Keep open. Current main still hides stdout-only structured tool results in the OpenClawKit Chat UI; the PR is a focused fix, but it still needs changelog and PR-metadata cleanup before merge.

## What This Changes

The PR adds stdout extraction before generic message fallback in `ToolResultTextFormatter` and adds Swift formatter tests for stdout precedence.

## Best Possible Solution

Land stdout precedence in the OpenClawKit formatter with regression tests, a single-line Unreleased changelog entry credited to @Narcooo, a simplified optional binding, and corrected PR closing metadata.

## Reproduction Assessment

Yes, at source level. Current main never reads `stdout` in `renderDictionary`, and `ToolResultCard` hides empty formatted output, so a stdout-only structured tool result has a clear missing-output path.

## Solution Assessment

Yes, with small cleanup. Prioritizing trimmed stdout after errors and nodes summaries but before generic message fields is the narrow maintainable fix; stderr fallback and unrelated issue closure should stay out of scope.
