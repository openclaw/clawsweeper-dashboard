## Summary

Keep open. Current main still drops stdout-only structured tool output in the OpenClawKit Chat UI, and this PR is a narrow, useful fix that only needs release-note and metadata cleanup before merge.

## What This Changes

The PR adds `stdout` extraction before the generic message fallback in the OpenClawKit Chat UI tool-result formatter and adds Swift tests for stdout precedence.

## Best Possible Solution

Land the stdout precedence behavior with regression tests, add one Unreleased changelog entry crediting @Narcooo, and remove or correct the unrelated closing reference without adding stderr fallback policy.

## Reproduction Assessment

Yes. A source-level reproduction is clear: on current main, `ToolResultTextFormatter.format` with stdout-only JSON returns an empty string, and a payload with both `stdout` and `message` returns the generic message because `stdout` is not read.

## Solution Assessment

Yes, with small cleanup. Prioritizing non-empty `stdout` after errors and nodes summaries but before generic message fields is the narrow maintainable fix; stderr fallback should stay out of scope unless a separate product decision requests it.
