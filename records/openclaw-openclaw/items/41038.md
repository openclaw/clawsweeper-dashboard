## Summary

Keep open. Current main still drops stdout-only structured tool results in OpenClawKit Chat UI, and this PR is a focused, useful fix; it needs small merge cleanup rather than closure.

## What This Changes

The PR adds `stdout` extraction to the OpenClawKit Chat UI tool-result formatter and adds Swift tests for exec-style stdout precedence.

## Best Possible Solution

Land the stdout precedence behavior with regression tests, an Unreleased changelog entry crediting the human contributor, and corrected PR metadata without adding stderr fallback policy.

## Reproduction Assessment

Yes. A source-level reproduction is clear: on current main, stdout-only JSON returns an empty formatter string, and JSON with both `stdout` and `message` returns the generic message because `stdout` is never read.

## Solution Assessment

Yes, with small cleanup. Prioritizing trimmed `stdout` after errors and nodes summaries but before generic message fields is the narrow maintainable fix; stderr fallback should remain out of scope unless separately requested.
