## What This Changes

This PR makes the OpenClawKit Chat UI formatter return trimmed `stdout` from successful structured tool-result JSON before generic message fields and adds stdout-precedence formatter tests.

## Best Possible Solution

Land the formatter-level stdout precedence with focused regression tests after the contributor adds real behavior proof and removes or corrects the unrelated closing reference.

## Reproduction Assessment

Yes, at source level. Current main does not inspect `stdout` in `renderDictionary`, and `ToolResultCard` suppresses empty formatted output, so a stdout-only structured result can disappear from the visible Chat UI.

## Solution Assessment

Yes, with proof and metadata cleanup. Preferring trimmed `stdout` after error and nodes-summary handling but before generic message fields is the narrow maintainable fix for this formatter path.
