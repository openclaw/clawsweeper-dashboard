## Summary

Keep open: current main still has a source-reproducible root-mounted Control UI logo path problem, but this PR needs changes because it removes CSP too broadly and targets stale chat avatar code.

## What This Changes

The PR makes Control UI logo paths absolute, renders assistant fallback avatars without a basePath guard, moves Control UI CSP from shared static headers to HTML/avatar responses, and updates focused tests.

## Best Possible Solution

Land a current-main port or replacement that fixes root-mounted logo/favicon paths and static SVG CSP behavior without weakening assistant media/avatar security, with focused regression tests and a changelog entry.

## Reproduction Assessment

Yes by source inspection. Current main returns route-relative `favicon.svg` for `agentLogoUrl("")`, the sidebar uses that helper for the header logo, and #55600 includes browser steps and screenshots; I did not run a browser or tests in this read-only pass.

## Solution Assessment

No. The absolute logo-path direction is useful, but the PR removes shared CSP too broadly and edits stale grouped-render avatar logic; a current-main port scoped to static assets and `renderChatAvatar` is safer.
