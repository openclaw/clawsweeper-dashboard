## Summary

Keep open. The regression is real and current main still has route-relative logo behavior, but this branch is not merge-ready because it drops CSP from assistant media, targets a stale chat-avatar path, and lacks the required changelog entry.

## What This Changes

This PR changes Control UI logo URL generation to use `/favicon.svg`, moves CSP from shared Control UI headers to HTML/avatar-specific responses, removes a basePath guard in grouped chat fallback rendering, and updates focused tests.

## Best Possible Solution

Land a current-main port or replacement that fixes root-mounted and non-root Control UI static asset paths, keeps restrictive CSP on HTML/avatar/assistant-media responses, omits CSP only for static favicon/SVG assets if needed, updates focused tests, and adds a single-line changelog entry.

## Reproduction Assessment

Yes. Current main still returns `favicon.svg` from `agentLogoUrl("")` while the sidebar uses that helper, and #55600 provides concrete upgrade/browser steps with screenshots. Static inspection also reproduces why the PR's grouped-render hunk no longer covers current avatar rendering.

## Solution Assessment

No. The absolute logo path direction is useful, but the patch is not the narrowest maintainable fix because it removes CSP too broadly and targets an old chat rendering block instead of the current `renderChatAvatar` helper.

  - confidence: 0.9
