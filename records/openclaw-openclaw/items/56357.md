## Summary

Keep open. Current main still has the root-mounted Control UI logo path regression, and this PR is useful source work, but it is not merge-ready because part of the diff targets an older chat avatar renderer, it omits required user-facing changelog coverage, and the CSP refactor drops CSP from the authenticated assistant media route as well as static UI assets.

## What This Changes

The PR changes Control UI logo URL generation to use an absolute `/favicon.svg` fallback, moves Control UI CSP from shared response headers to HTML/avatar-specific responses, removes an assistant avatar basePath guard in grouped chat rendering, and updates focused tests.

## Best Possible Solution

Land a current-main port or replacement that fixes root-mounted `/favicon.svg` and any current assistant fallback logo paths, omits CSP only for the bundled static SVG/favicon responses that need it, preserves restrictive CSP on HTML, avatars, and authenticated assistant media, updates focused UI/Gateway tests, and adds a single-line changelog entry crediting the non-bot contributor or reporter.

## Reproduction Assessment

Yes. The linked bug has concrete upgrade/browser steps, and current main source independently reproduces the core path issue: `agentLogoUrl("")` returns `favicon.svg` while the sidebar uses that helper for the logo.

## Solution Assessment

No, not as-is. The direction is narrow and useful, but the implementation must be ported to current rendering and must scope CSP removal more carefully so authenticated media endpoints do not lose script protections.
