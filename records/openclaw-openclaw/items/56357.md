## Summary

Keep open. The PR targets a valid Control UI logo/favicon regression, but the current diff is not safe to merge because it removes shared CSP too broadly and one UI hunk targets avatar rendering that has moved on current main.

## What This Changes

The PR changes Control UI logo URL generation, adjusts assistant avatar fallback rendering, scopes CSP away from static assets while adding avatar CSP, and updates focused UI/Gateway tests.

## Best Possible Solution

Land a current-main port or replacement that fixes root-mounted and base-mounted logo/favicon paths, preserves restrictive CSP on HTML/avatar/assistant-media responses, omits CSP only for static favicon/SVG assets if needed, and adds focused tests plus a one-line changelog entry.

## Reproduction Assessment

Yes. Static inspection of current main shows `agentLogoUrl("")` still returns `favicon.svg` while the sidebar consumes that helper, and related issue #55600 provides concrete upgrade/browser steps with screenshots.

## Solution Assessment

No. The absolute logo path direction is useful, but this PR is not the best mergeable fix because it removes CSP too broadly and edits an avatar rendering block that current main no longer uses.
