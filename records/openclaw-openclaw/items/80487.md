## Summary

Keep open: current main still drops the model-level `api` before synthetic auth hook discovery, and the paired open PR https://github.com/openclaw/openclaw/pull/80488 explicitly closes this issue.

## What This Changes

The report asks provider auth resolution to honor a per-model `api` override when discovering plugin synthetic auth hooks for custom providers.

## Best Possible Solution

Thread the effective `model.api` from `getApiKeyForModel` into provider synthetic auth hook discovery, add focused regression coverage for provider-level `openai-completions` with model-level `ollama`, then close this issue after the fix merges.

## Reproduction Assessment

Yes, at source level: current main preserves a model-level `api` in config/model resolution and stream registration, but the auth hook refs only read the provider-level API. I did not run a live Docker/Ollama repro in this read-only review.
