## What This Changes

The issue reports that a custom-provider model with a per-model `api: "ollama"` override fails auth/plugin hook resolution when the provider-level API is different.

## Best Possible Solution

Thread the effective resolved model API into provider synthetic auth hook discovery, add a focused regression test for provider-level `openai-completions` with model-level `ollama`, and close this issue after the linked PR or equivalent fix merges.
