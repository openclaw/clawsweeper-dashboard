## Summary

Current main and the latest release now mirror successful current-source message tool sends into assistant transcript/history, so the reported Telegram-visible reply and later Codex context drift is fixed well enough to close.

## What This Changes

The issue asks for Telegram-visible `message(action="send")` replies to be represented as normal assistant transcript/context instead of remaining hidden in tool cards and elided tool payloads.

## Best Possible Solution

Keep the shipped source-reply transcript mirror and treat any future stuck-running lifecycle report as a separate, narrower issue if it still reproduces.

## Reproduction Assessment

No current-main failing reproduction remains. The historical failure is source-reproducible from the old `message.action` tool-result-only path plus Codex tool-payload elision, and the current regression tests cover the fixed direct/topic source-send behavior.

## Solution Assessment

Yes. Mirroring only successful current-source sends at the gateway transcript boundary is the narrow maintainable fix because it preserves generic tool-payload elision and avoids showing external-target or failed sends as assistant replies.
