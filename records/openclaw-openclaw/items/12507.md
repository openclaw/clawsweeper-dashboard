## Summary

Keep open. Current main still lacks a trusted SKILL.md signature/integrity contract for native skills, and the issue remains a security/product design item rather than a stale cleanup candidate.

## What This Changes

The issue requests trusted integrity and signature verification for SKILL.md skills, including ClawHub metadata, runtime verification, and unsigned-skill policy.

## Best Possible Solution

Keep this as the canonical native skill-signature tracker and design one end-to-end contract across ClawHub metadata, install/update persistence, loader-time verification, status surfaces, docs, revocation, and unsigned-skill policy.

## Reproduction Assessment

Not applicable. This is a feature/security-design request rather than broken existing behavior, and current-main source inspection shows the requested signature and loader verification path is absent.

## Solution Assessment

Unclear until maintainers choose the trust model. The requested direction is valid, but a durable solution needs trusted publisher/signature metadata, loader-time verification, status reporting, revocation handling, and unsigned capability policy rather than a local hash-only check.
