## Summary

Keep open. Current main still lacks native SKILL.md signing or trusted integrity verification across ClawHub skill metadata, install persistence, and runtime skill loading; the recent issue discussion also treats this as canonical security architecture work rather than cleanup to close.

## What This Changes

The issue requests native integrity verification and cryptographic signatures for SKILL.md-based skills, with policy controls for unsigned skills.

## Best Possible Solution

Define a native skill trust model that adds trusted digest/signature/publisher metadata to ClawHub, verifies archives before extraction, persists expected integrity in origin/lock state, revalidates skill contents during snapshot/load, and documents the unsigned-skill policy across CLI/UI/status surfaces.

## Reproduction Assessment

Yes for the reported missing capability. Source inspection on current main shows native ClawHub skill install and runtime SKILL.md loading have no trusted signature or persisted integrity state, while only the separate plugin package path verifies trusted archive hashes.

## Solution Assessment

Unclear. The requested direction is valid security work, but the best fix needs maintainer decisions on publisher trust, ClawHub API metadata, migration, runtime enforcement, and unsigned capability policy before a narrow patch is safe.
