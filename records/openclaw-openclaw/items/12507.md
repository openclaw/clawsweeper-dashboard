## Summary

Keep open. Current main still lacks native SKILL.md signing or trusted integrity verification across ClawHub skill metadata, skill install persistence, and runtime skill loading; recent issue discussion also treats this as canonical security architecture work.

## What This Changes

The issue requests cryptographic signature/hash verification for SKILL.md skills, including ClawHub metadata, install verification, runtime enforcement, and restricted behavior for unsigned skills.

## Best Possible Solution

Define a native skill trust model that adds trusted digest/signature/publisher metadata to ClawHub, verifies archives before extraction, persists expected integrity in origin/lock state, revalidates skill contents during snapshot/load, and documents unsigned-skill policy across CLI/UI/status surfaces.

## Reproduction Assessment

Yes for the reported missing capability. Source inspection on current main shows native ClawHub skill install and runtime SKILL.md loading have no trusted signature or persisted integrity state, while the separate plugin package path verifies trusted archive metadata.

## Solution Assessment

Unclear. The signing/hash direction is appropriate security work, but the best fix needs maintainer decisions on publisher trust, ClawHub API metadata, migration, runtime enforcement, and unsigned capability policy before a narrow patch is safe.
