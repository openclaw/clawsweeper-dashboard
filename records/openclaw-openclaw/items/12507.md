## Summary

Keep open. Current main and v2026.5.22 still lack native trusted skill digest/signature metadata, persisted verification state, and loader-time SKILL.md enforcement; the closed SIE issue was consolidated here and the open SHA-256 PR is only a partial local tamper-detection proposal.

## What This Changes

Request a signed and integrity-verified SKILL.md trust model for ClawHub and local skill loading, including degraded behavior for unsigned skills.

## Best Possible Solution

Keep this as the canonical native skill-signature tracker and define one end-to-end trust contract across ClawHub metadata, archive verification, persisted origin/lock state, loader-time verification, status surfaces, docs, revocation, and unsigned-skill policy.

## Reproduction Assessment

Not applicable. This is a feature/security-design request rather than broken existing behavior, and source plus latest-release inspection confirms the requested native signature and loader verification path is absent.

## Solution Assessment

Unclear until maintainers choose the trust model. Cryptographic skill verification is a valid direction, but the maintainable solution needs trusted publisher/signature metadata, loader-time verification, status reporting, revocation handling, and unsigned capability policy rather than a local hash-only check.
