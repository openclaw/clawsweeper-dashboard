## Summary

Keep open. Current main and v2026.6.5 have ClawHub trust-card and `skills verify` surfaces, but SKILL.md files still enter skill snapshots without enforced signature, publisher, revocation, or unsigned capability policy.

## What This Changes

Request to add cryptographic code verification/signatures for SKILL.md skills, including loader verification and restricted behavior for unsigned skills.

## Best Possible Solution

Keep this as the canonical native skill-signature tracker and define one end-to-end trust contract across ClawHub metadata, install/update provenance, loader-time verification, status surfaces, docs, revocation, and unsigned-skill policy.

## Reproduction Assessment

Not applicable. This is a feature/security-design request rather than broken existing behavior, and source inspection confirms current main still lacks loader-time signature enforcement.

## Solution Assessment

Unclear until maintainers choose the trust model. Cryptographic verification is a valid direction, but the maintainable fix needs trusted publisher/signature metadata, loader-time enforcement, revocation handling, status reporting, and unsigned capability policy rather than a local hash-only or display-only check.
