## Summary

Keep open. Current main still lacks native SKILL.md signing or trusted integrity enforcement for ClawHub skills, and the issue discussion already treats this as canonical security architecture work rather than cleanup to close.

## What This Changes

The issue requests cryptographic signing and integrity verification for SKILL.md skills distributed through ClawHub and .skill packages.

## Best Possible Solution

Define a native skill trust model that adds trusted digest/signature/publisher metadata to ClawHub, verifies archives before extraction, persists expected integrity in origin/lock state, revalidates skill contents during load, and documents unsigned-skill policy across CLI/UI/status surfaces.

## Reproduction Assessment

Yes for the reported missing capability. Source inspection on current main shows native ClawHub skill install and runtime SKILL.md loading have no trusted signature or persisted integrity state, while the separate plugin package path verifies trusted archive hashes.

## Solution Assessment

Unclear. The requested signing/hash direction is credible, but the best fix needs maintainer decisions on publisher trust, ClawHub API metadata, migration, runtime enforcement, and unsigned-skill policy before a narrow patch is safe.
