## What This Changes

Requests cryptographic signature and integrity verification for SKILL.md skills, ClawHub distribution metadata, and unsigned-skill capability policy.

## Best Possible Solution

Keep this as the canonical native skill-signature tracker and define one end-to-end contract across ClawHub metadata, install/update persistence, loader-time verification, status surfaces, docs, revocation, and unsigned-skill policy.

## Solution Assessment

Unclear until maintainers choose the trust model.
