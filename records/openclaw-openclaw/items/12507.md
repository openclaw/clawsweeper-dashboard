## Summary

Keep open. Current main and v2026.6.5 added ClawHub skill verification/trust surfaces, but they still do not implement native loader-time SKILL.md signature enforcement, persisted trusted integrity state, or unsigned-skill capability policy.

## What This Changes

Request end-to-end code verification and signatures for SKILL.md skills, including ClawHub signing metadata, runtime verification, and degraded behavior for unsigned skills.

## Best Possible Solution

Define one maintainer-approved skill trust contract across ClawHub signing metadata, install/update verification, persisted origin/lock state, loader-time verification, status/UI/CLI reporting, revocation, and unsigned-skill policy.

## Reproduction Assessment

Not applicable. This is a feature/security-design request rather than broken existing behavior, and source inspection shows current main and v2026.6.5 still lack native loader-time signature enforcement.

## Solution Assessment

Unclear until maintainers choose the trust model. Cryptographic skill verification is a valid direction, but the maintainable solution needs trusted publisher/signature metadata, loader-time enforcement, revocation handling, status reporting, and unsigned capability policy rather than a local hash-only check.
