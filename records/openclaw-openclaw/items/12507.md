## Summary

Keep open: current main and v2026.6.10 have ClawHub verification, provenance, and status surfaces, but SKILL.md loading still consumes local bytes without trusted signature/digest enforcement or an unsigned-skill capability policy.

## What This Changes

Request cryptographic signing and integrity verification for SKILL.md skills, including runtime verification and restricted behavior for unsigned skills.

## Best Possible Solution

Extend existing ClawHub trust metadata into load-time runtime enforcement with a maintainer-approved policy for signed, unsigned, invalid, revoked, and locally modified skills.

## Reproduction Assessment

Yes, as a source-level feature gap: current main and v2026.6.10 load SKILL.md into runtime skill records and snapshots without comparing a trusted signature, persisted digest, revocation state, or unsigned capability tier.

## Solution Assessment

No, not as a one-shot Ed25519/degraded-mode patch without design. The maintainable path is to extend current ClawHub verification and install metadata into runtime enforcement after maintainers settle trust roots, revocation, compatibility, and operator override behavior.
