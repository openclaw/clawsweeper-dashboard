## Summary

Keep open. Current main still lacks trusted native signing or integrity verification for SKILL.md skills, and the issue discussion plus related open PR show this remains active security architecture work rather than implemented cleanup.

## What This Changes

The issue requests cryptographic integrity/signature verification for distributed skills, persisted trust metadata, and policy restrictions for unsigned skills.

## Best Possible Solution

Keep this as the canonical native skill trust design item and resolve ClawHub metadata, install verification, persisted trust state, runtime revalidation, status/audit visibility, migration, and unsigned-skill policy together.

## Solution Assessment

Unclear as a narrow patch. Content signing/integrity is the right direction, but the maintainable fix needs maintainer-approved trust roots, registry metadata, migration semantics, and signed/unsigned enforcement policy before implementation.
