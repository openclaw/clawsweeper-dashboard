## Summary

Keep open. Current main still lacks native SKILL.md signature or trusted integrity enforcement across ClawHub skill metadata, native skill installation, persisted skill origin/lock state, and runtime skill loading; the prior issue comments also keep this as canonical security architecture work.

## What This Changes

Request native skill code-signing or content-integrity verification for SKILL.md files and ClawHub/.skill distribution.

## Best Possible Solution

Define a native skill trust model that adds trusted digest/signature/publisher metadata to ClawHub, verifies skill archives before extraction, persists expected integrity in origin/lock state, revalidates skill contents at snapshot/load time, and surfaces unsigned-skill policy in CLI/UI/docs.

## Reproduction Assessment

Yes, for the reported missing capability. Source inspection on current main shows native ClawHub skill install and runtime skill loading have no trusted signature or persisted integrity state, while adjacent plugin archive verification is separate.

## Solution Assessment

Unclear as a narrow patch. The signing/hash direction is appropriate, but the best maintainable fix needs a security/product design for trusted metadata, migration, runtime enforcement, and unsigned-skill policy before implementation.
