## Summary

Keep open. Current main still has no trusted SKILL.md signature or integrity contract for native skills, and the related open hash-only PR does not resolve publisher identity, trusted registry metadata, loader-time verification, revocation, or unsigned-skill policy.

## What This Changes

The issue requests cryptographic integrity and signature verification for SKILL.md skills, including trusted metadata, loader enforcement, and unsigned-skill capability policy.

## Best Possible Solution

Keep this as the canonical native skill-signature tracker and design one end-to-end contract across ClawHub metadata, install/update persistence, loader-time verification, status surfaces, docs, and unsigned-skill policy.

## Reproduction Assessment

Not applicable. This is a feature/security-design request rather than broken existing behavior, and source inspection on current main shows the requested signature and loader verification path is absent.

## Solution Assessment

Unclear until maintainers choose the trust model. The requested direction is valid, but a durable solution needs trusted publisher/signature metadata, loader-time verification, status reporting, revocation handling, and unsigned capability policy rather than a local hash-only check.
