## Summary

Keep open. Current main still lacks native SKILL.md signing or trusted integrity verification across ClawHub skill metadata, install persistence, and runtime skill loading, and the recent issue discussion treats this as canonical security architecture work.

## What This Changes

Request adding cryptographic signing, integrity verification, and unsigned-skill policy for ClawHub/native SKILL.md skills.

## Best Possible Solution

Keep this as the canonical native skill trust-model issue and define end-to-end ClawHub metadata, install verification, persisted state, load-time validation, and unsigned-skill policy before implementation.

## Reproduction Assessment

Yes for the reported missing capability. Source inspection on current main shows native ClawHub skill install and runtime SKILL.md loading have no trusted signature or persisted integrity state, while only the separate plugin package path verifies trusted archive hashes.

## Solution Assessment

Unclear. The requested direction is valid security work, but the best fix needs maintainer decisions on publisher trust, ClawHub API metadata, migration, runtime enforcement, and unsigned capability policy before a narrow patch is safe.
