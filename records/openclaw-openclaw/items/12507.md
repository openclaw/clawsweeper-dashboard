## Summary

Keep open. Current main still lacks trusted signature or integrity verification for native SKILL.md skills across ClawHub skill metadata, native skill install records, and runtime skill loading, and the related open work is adjacent rather than a complete replacement.

## What This Changes

The issue requests cryptographic signing and integrity verification for SKILL.md skills, including trusted ClawHub metadata, loader verification, and unsigned-skill policy.

## Best Possible Solution

Keep this as the canonical native skill signing/integrity work and design trusted ClawHub metadata, install verification, persisted trust state, runtime revalidation, and visible unsigned-skill policy together.

## Solution Assessment

Unclear as a narrow patch. The signing and integrity direction is appropriate, but the best maintainable solution needs maintainer-approved trust roots, registry metadata, migration, and enforcement semantics before implementation.
