## Summary

Keep open. Current main still does not provide trusted native signature or integrity verification for SKILL.md skills; ClawHub skill metadata, install records, and runtime skill snapshots lack verification state, and the related open SHA-256 PR is only a partial/review-blocked implementation path.

## What This Changes

The issue requests cryptographic signing and integrity verification for SKILL.md-based skills, including trusted ClawHub metadata, runtime verification, and policy for unsigned skills.

## Best Possible Solution

Keep this as the canonical native skill signing/integrity design item and implement trusted ClawHub metadata, install-time verification, persisted trust state, runtime revalidation, status/audit visibility, migration behavior, and unsigned-skill policy together.

## Reproduction Assessment

Yes for the source-verifiable missing capability: current main has no trusted native skill digest/signature fields, no persisted verification state, and no loader-time signature decision path. This is not a runtime bug reproduction; it is verified by current source inspection.

## Solution Assessment

Unclear as a narrow patch. Content signing is the right direction, but the maintainable solution needs maintainer-approved trust roots, registry metadata, migration semantics, and signed/unsigned enforcement policy before implementation.
