## Summary

Keep open. Current main still lacks a trusted native SKILL.md signature/integrity contract, prior discussion keeps this as the canonical tracker, the closed SIE proposal points back here, and the open SHA-256 PR is partial hardening rather than the full trust model.

## What This Changes

Requests native SKILL.md integrity and signature verification for skills, including trusted metadata, loader enforcement, and policy for unsigned skills.

## Best Possible Solution

Keep this as the canonical native skill-signature tracker and define one end-to-end contract across ClawHub metadata, install/update persistence, loader-time verification, status surfaces, docs, revocation, and unsigned-skill policy.

## Solution Assessment

Unclear until maintainers choose the trust model. The direction is valid, but a maintainable fix needs trusted publisher/signature metadata, loader-time verification, status reporting, revocation handling, and unsigned capability policy rather than a local hash-only check.
