## Summary

Keep open. Current main still lacks native SKILL.md signature/integrity verification, trusted ClawHub skill metadata, persisted verification state, and loader-time unsigned policy; the related closed SIE issue was consolidated here and the open hash-only PR is partial review input, not a replacement for this trust-model tracker.

## What This Changes

Request to add cryptographic integrity/signature verification for SKILL.md skills, ClawHub skill distribution, and unsigned-skill capability policy.

## Best Possible Solution

Keep this as the canonical skill-signature trust tracker and design one end-to-end contract across ClawHub metadata, install/update persistence, loader-time verification, status/UI/CLI surfaces, docs, and unsigned-skill capability policy.

## Reproduction Assessment

Not applicable. This is a feature/security-design request rather than a broken existing behavior report, and current-main source inspection shows the requested signature and loader verification path is absent.

## Solution Assessment

Unclear until maintainers choose the trust model. A local hash/audit check is useful but incomplete; the durable solution needs trusted publisher/signature metadata, loader-time verification, status reporting, and explicit unsigned capability policy.
