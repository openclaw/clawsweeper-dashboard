## Summary

Keep open. Current main still loads SKILL.md without trusted signature/integrity state, ClawHub native skill metadata and lockfiles do not carry trusted digest or publisher verification, and the related closed SIE issue plus open hash-only PR leave this as the canonical skill trust-model tracker.

## What This Changes

Request cryptographic integrity and signature verification for SKILL.md skills across ClawHub distribution, runtime loading, and unsigned-skill capability policy.

## Best Possible Solution

Keep this as the canonical skill-signature trust tracker and design one end-to-end contract across ClawHub metadata, install/update persistence, loader-time verification, status surfaces, docs, and unsigned-skill policy.

## Reproduction Assessment

Not applicable. This is a feature/security-design request rather than a broken existing behavior report, and source inspection on current main shows the requested signature and loader verification path is absent.

## Solution Assessment

Unclear until maintainers choose the trust model. A hash-only audit or lockfile check is useful but incomplete; the durable solution needs trusted publisher/signature metadata, loader-time verification, status reporting, and unsigned capability policy.
