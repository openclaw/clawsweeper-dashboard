## Summary

Keep open: current main has useful ClawHub verification, provenance, and status surfaces, but runtime skill loading still consumes local SKILL.md bytes without trusted signature/digest enforcement or an unsigned-skill capability policy.

## What This Changes

Request to add cryptographic integrity/signature verification for SKILL.md skills and restrict unsigned or untrusted skills.

## Best Possible Solution

Extend existing ClawHub trust metadata into load-time runtime enforcement with a maintainer-approved policy for trusted, unsigned, invalid, revoked, and locally modified skills.

## Reproduction Assessment

Yes, source-reproducible: current main reads SKILL.md into Skill records and snapshots without comparing a trusted signature, persisted digest, revocation state, or unsigned capability tier. v2026.6.8 shows the same runtime loader gap.

## Solution Assessment

No, not as a one-shot Ed25519/degraded-mode patch without design. The maintainable path is to extend current ClawHub verification and install metadata into runtime enforcement after maintainers settle trust roots, revocation, compatibility, and operator override behavior.
