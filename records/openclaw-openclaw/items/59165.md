## Summary

Keep open: current main and latest release still have static SecretRef plus model-provider auth/profile machinery, but no cross-domain credential-provider lifecycle API, consumer request path, per-session credential contract, or plugin trust policy. Recent comments and adjacent PRs add useful implementation context, but they also confirm this needs maintainer security/API/product review rather than cleanup closure.

## What This Changes

The issue proposes a new Credential Provider Plugin capability for cross-domain credential acquisition, lifecycle management, scoping, and trust policy alongside SecretRef.

## Best Possible Solution

Keep the RFC open for maintainer and secops-led API design; if accepted, land an additive SDK/runtime contract with explicit trust policy, SecretRef/AuthProfileStore boundaries, per-session scoping, raw-value versus proxy/broker semantics, and a proof provider before moving concrete backends to plugins or ClawHub.

Summary: This issue is security-sensitive because it proposes a cross-plugin credential supply and lifecycle boundary, but there is no patch to audit.
