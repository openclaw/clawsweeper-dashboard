## Summary

Keep open. Current main still has static SecretRef support and model-provider-scoped auth hooks, but not the proposed cross-domain Credential Provider Plugin capability; the remaining work is a security-sensitive SDK/API and product design, not cleanup closure.

## What This Changes

Introduce a plugin SDK credential-provider capability that can acquire, refresh, rotate, and share typed credentials across models, channels, skills, tools, and plugins.

## Best Possible Solution

Keep the RFC open for secops and maintainer-led API design; if accepted, land an additive versioned SDK capability with explicit trust policy, per-session scoping, SecretRef/AuthProfileStore boundaries, and a proof provider, while treating https://github.com/openclaw/openclaw/pull/72548 as adjacent narrower work.

## Reproduction Assessment

Not applicable; this is an RFC for a new capability, not a broken existing behavior report. The current-main verification path is source and docs inspection showing no credential-provider registration API and static SecretRef semantics.

## Solution Assessment

No for automatic implementation. The direction is plausible, but the safe path is a maintainer-approved SDK and security contract first, not a one-shot repair PR.
