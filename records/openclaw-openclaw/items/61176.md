## Summary

Keep open. The PR is useful source work for the Control UI session-sidebar request, but current main does not already implement the sidebar and the proposed patch has concrete merge-blocking behavior bugs around preserved history, shortcut handling, and sidebar state.

## What This Changes

The PR adds a Control UI session sidebar with grouped recent sessions, context meters, New Chat controls, `/new` history preservation metadata, browser-local session-key imports, and related UI styling.

## Best Possible Solution

Repair and rebase the implementation on current main so the sidebar uses existing session APIs and browser-safe helpers, preserves stable transcript references, keeps tool-output sidebar state separate, and leaves broader #50404 features to follow-up work.

## Reproduction Assessment

Yes. The stale transcript bug is reproducible by preserving a session entry before reset and then following the current reset archive path that renames the old transcript; the shortcut bug is reproducible by pressing Ctrl/Cmd+` while focus is inside the chat component.

## Solution Assessment

No. The feature direction is reasonable, but the current patch is not the best merge path until the history row stores a valid transcript reference, the sidebar state is decoupled from tool output, and one shortcut owner handles the toggle.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve the archived transcript path for /new history:** `src/gateway/session-reset-service.ts:458-463`
  - body: The preserved history row copies `currentEntry.sessionFile` before the reset flow archives and renames that transcript. Selecting the old row can therefore load a path that no longer exists, so `/new` history can appear empty instead of reopening the prior conversation.
  - confidence: 0.92
- **[P2] Stop the session shortcut from toggling twice:** `ui/src/ui/views/chat.ts:1144-1151`
  - body: This handler toggles the session sidebar for Ctrl/Cmd+`, while the PR also adds a document-level handler for the same key. When focus is in the chat, the event bubbles and both handlers run, leaving the sidebar in its original state.
  - confidence: 0.9
- **[P2] Restore the prior sidebar after closing sessions:** `ui/src/ui/app.ts:793-797`
  - body: Opening the session list reuses `sidebarOpen`, but closing it forces `sidebarOpen = false`. If a markdown/tool-output sidebar was already open, closing the session list hides that existing sidebar instead of returning the user to it.
  - confidence: 0.78
