## Summary

Keep open. Current main still lacks the proposed in-chat session sidebar, but this PR is not ready to merge because the branch is conflicting, has source-proven regressions, and lacks inspectable real behavior proof.

## What This Changes

The PR adds a Control UI session sidebar/history list with context meter thresholds, New Chat styling, 7-day ended-session visibility, `/new` history metadata, a local UI session-key helper, and a Vite server-only stub.

## Best Possible Solution

Rebase a narrower sidebar/history implementation onto current main's session creation and switching flow, preserve archived transcript references correctly, decouple session-sidebar state from tool-output sidebar state, and align the remaining scope with https://github.com/openclaw/openclaw/issues/27526.

## Reproduction Assessment

Yes, for the PR-introduced defects by source inspection. The copied pre-archive `sessionFile`, duplicate shortcut handlers, and shared `sidebarOpen` close behavior are visible in the PR branch without needing a live runtime repro.

## Solution Assessment

No. The feature direction is plausible, but this branch is not the best merge path until it is rebased onto current main, fixes the transcript/shortcut/sidebar-state regressions, and supplies real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve the archived transcript path for history rows:** `src/gateway/session-reset-service.ts:454-456`
  - body: The preserved history entry spreads `currentEntry`, including `sessionFile`, before the reset flow archives and renames the old transcript. Selecting that row can point at a stale or missing transcript instead of the prior conversation.
  - confidence: 0.92
- **[P2] Stop the session shortcut from toggling twice:** `ui/src/ui/views/chat.ts:1141-1142`
  - body: This Cmd/Ctrl+` handler toggles the session sidebar inside chat while `app.ts` also registers a document-level handler for the same shortcut. When focus is in chat, the event bubbles and both handlers run, leaving the sidebar in its original state.
  - confidence: 0.9
- **[P2] Restore the previous sidebar after closing sessions:** `ui/src/ui/app.ts:797-800`
  - body: Closing the session sidebar also forces `sidebarOpen = false`, so a user who had markdown or tool-output content open loses that panel after briefly opening the session list. The session overlay should restore or decouple the previous sidebar state.
  - confidence: 0.82
