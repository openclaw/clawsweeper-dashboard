## Summary

Keep open. This is still a plausible Control UI sidebar implementation candidate, and related discussion points the broader product scope to https://github.com/openclaw/openclaw/issues/27526, but this PR is currently conflicting, lacks real behavior proof, and introduces source-provable bugs in `/new` history preservation, duplicate shortcut handling, and sidebar state restoration.

## What This Changes

The PR adds a Control UI session sidebar/history list with context meter threshold changes, a full-width New Chat action, 7-day ended-session visibility, `previousSessionKey` metadata, a copied UI session-key helper, and a Vite server-only stub plugin.

## Best Possible Solution

Rebase and narrow the session-sidebar slice onto current main's `sessions.create` New Chat flow, preserve archived transcript references correctly, decouple the session sidebar from tool-output sidebar state, and leave broader conversation-management scope to https://github.com/openclaw/openclaw/issues/27526.

## Reproduction Assessment

Yes, for the PR-introduced defects by source inspection. The `/new` history row copies `currentEntry` before transcript archival renames its `sessionFile`, and the shortcut/sidebar-state issues follow directly from duplicate handlers plus shared `sidebarOpen` state.

## Solution Assessment

No. The feature direction is plausible, but this patch is not the best merge path until it is rebased onto current main's `sessions.create` flow, fixes the transcript/shortcut/sidebar-state regressions, and supplies real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve the archived transcript path for /new history:** `src/gateway/session-reset-service.ts:454-456`
  - body: The preserved history row copies `currentEntry`, including `sessionFile`, before the reset flow archives and renames the old transcript. Selecting that row can point at a missing or stale transcript instead of the previous conversation.
  - confidence: 0.92
- **[P2] Stop the session shortcut from toggling twice:** `ui/src/ui/views/chat.ts:1141-1142`
  - body: This Ctrl/Cmd+` handler toggles the session sidebar inside chat while `app.ts` also registers a document-level handler for the same shortcut. When focus is in chat, the event bubbles and both handlers run, leaving the sidebar in its original state.
  - confidence: 0.9
- **[P2] Restore the previous sidebar after closing sessions:** `ui/src/ui/app.ts:797-800`
  - body: Closing the session sidebar also forces `sidebarOpen = false`, so a user who had markdown or tool-output content open loses that panel after briefly opening the session list. The session overlay should restore or decouple the previous sidebar state.
  - confidence: 0.8
