## Summary

Keep open. Current main still does not implement the proposed Control UI session sidebar, but this PR head should not merge as-is because the `/new` history preservation can point archived rows at a transcript path that the reset flow later renames, and the new sidebar shortcut is handled twice in the same bubbling event path.

## What This Changes

The PR adds a Control UI session sidebar with grouped recent sessions, context meters and a New Chat button, adds `/new` history preservation via `previousSessionKey`, moves UI session-key imports to a browser-local helper, and adds a Vite server-only-module stub plugin.

## Best Possible Solution

Keep the PR open as useful source work, but require a repaired implementation that preserves archived transcript references correctly, decouples session-list state from the existing markdown/tool sidebar, uses one shortcut owner, and rebases the UI helper work onto the browser-safe session-key helper already present on main.

## Reproduction Assessment

Yes. The `/new` breakage is reproducible by preserving a session entry before reset, then observing that the reset archive step renames the copied `sessionFile`; the shortcut bug is reproducible by pressing `Ctrl/Cmd+\`` while focus is inside the chat area so the component and document handlers both fire.

## Solution Assessment

No. The proposed direction is useful, but this implementation is not the best merge path until it stores a stable archived transcript path and separates session-sidebar behavior from the existing tool-output sidebar and duplicate key handling.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve the archived transcript path for /new history:** `src/gateway/session-reset-service.ts:458-463`
  - body: The preserved history row copies `currentEntry.sessionFile` before the reset flow archives and renames that transcript. Selecting the old row can therefore load a path that no longer exists, so `/new` history appears empty or starts a fresh transcript instead of opening the prior conversation.
  - confidence: 0.92
- **[P2] Stop the session shortcut from toggling twice:** `ui/src/ui/views/chat.ts:1144-1151`
  - body: This handler toggles the session sidebar for `Ctrl/Cmd+\``, and the PR also adds a document-level handler for the same key. When focus is in the chat, the event bubbles and both handlers run, leaving the sidebar in its original state.
  - confidence: 0.9
- **[P2] Restore the prior sidebar after closing sessions:** `ui/src/ui/app.ts:793-797`
  - body: Opening the session list reuses `sidebarOpen`, but closing it forces `sidebarOpen = false`. If a markdown/tool-output sidebar was already open, closing the session list hides that existing sidebar instead of returning the user to it; keep the states separate or restore the previous sidebar mode.
  - confidence: 0.78
