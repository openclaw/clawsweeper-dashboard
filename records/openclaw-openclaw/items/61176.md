## Summary

Keep open. Current main still lacks the Control UI session sidebar and the broader #50404 request remains open, while this PR head still has blocking correctness issues in session history preservation, shortcut handling, sidebar state, and required changelog coverage.

## What This Changes

The PR adds a Control UI session-sidebar experience with grouped recent sessions, context meters, New Chat controls, `/new` history metadata, browser-local session-key imports, Vite server-only stubs, and related styling.

## Best Possible Solution

Repair and rebase the session-sidebar implementation so it uses current session creation/list APIs, stores valid archived transcript references, keeps tool-output and session sidebars decoupled, has one shortcut owner, and leaves rename/search/delete scope to #50404.

## Reproduction Assessment

Yes. The stale transcript bug follows from copying `currentEntry.sessionFile` before the reset archive moves that file; the shortcut bug follows from two handlers for Cmd/Ctrl+` on bubbling events; the sidebar bug follows from sharing `sidebarOpen` with tool output.

## Solution Assessment

No. The feature direction is useful, but this is not the best merge path until the PR is rebased around current main and the blocking state/history/shortcut bugs plus changelog requirement are fixed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve the archived transcript path for history rows:** `src/gateway/session-reset-service.ts:458-463`
  - body: This copies `currentEntry`, including `sessionFile`, before the reset flow later archives and renames that same transcript. The preserved history row can then point at a missing file, so selecting the previous session opens empty history instead of the conversation.
  - confidence: 0.92
- **[P2] Stop the session shortcut from toggling twice:** `ui/src/ui/views/chat.ts:1144-1151`
  - body: This handler toggles the session sidebar for Cmd/Ctrl+`, while the PR also adds a document-level handler for the same key. With focus in the chat, the event bubbles and both handlers run, leaving the sidebar in its original state.
  - confidence: 0.9
- **[P2] Restore the prior sidebar after closing sessions:** `ui/src/ui/app.ts:793-797`
  - body: Opening the session list reuses `sidebarOpen`, but closing it forces `sidebarOpen = false`. If a markdown/tool-output sidebar was already open, closing the session list hides that existing sidebar instead of returning the user to it.
  - confidence: 0.78
- **[P3] Add the required changelog entry:** `ui/src/ui/components/session-sidebar.ts:1`
  - body: This user-facing feature changes Control UI behavior, but the PR does not touch `CHANGELOG.md`. Repo policy requires user-facing `feat` changes to add a single-line changelog entry.
  - confidence: 0.86
