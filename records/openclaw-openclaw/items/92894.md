## Summary

Keep open. The PR targets a real managed-skill prompt/read mismatch, but the patch is still incomplete because existing persisted session skill prompts can keep advertising the old unreadable format, and the new preservation rule also keeps longer absolute managed paths in ordinary home-root installs where tilde expansion would already work.

## What This Changes

The PR changes managed skill prompt path compaction so managed skill locations under OpenClaw’s resolved skills root can remain absolute instead of being compacted to a tilde path.

## Best Possible Solution

Add a narrow prompt-format refresh/version for persisted skill snapshots, preserve absolute managed paths only where tilde expansion would be wrong, and then prove the patched Docker prompt plus read flow.

## Reproduction Assessment

Yes at source/report level: the linked issue has live before evidence, current main compacts prompt skill locations, and the read helper expands tilde paths through `os.homedir()`. I did not replay the full AlphaClaw Docker/Telegram loop in this read-only review.

## Solution Assessment

No, not yet. The prompt-rendering change is a plausible new-session mitigation, but it misses persisted snapshot refresh and still needs maintainer acceptance of the absolute managed-path contract.

- **[P1] Refresh persisted prompts after changing their path format:** `src/skills/loading/workspace.ts:83-85`
  - body: This only changes newly rendered prompt paths.   - confidence: 0.9
- **[P2] Preserve absolute paths only when tilde cannot reach them:** `src/skills/loading/workspace.ts:83-85`
  - body: The preserved-root check applies to every managed skill under the resolved skills dir, including ordinary installs where read-tool tilde expansion already reaches the same file. That makes prompts longer without improving readability and can push large skill catalogs into compact or truncated output, so gate preservation on the managed skills root not being equivalent to the read tool's home expansion.
  - confidence: 0.78
