## Summary

Keep open. The PR is aimed at a real managed-skill path bug, but the current patch is not merge-ready because existing sessions can keep their persisted broken skill prompt and the absolute-path prompt contract still needs maintainer acceptance plus live runtime proof.

## What This Changes

The PR changes skill prompt path compaction so managed skills under the resolved OpenClaw skills root stay absolute, with an AlphaClaw-shaped regression test.

## Best Possible Solution

Refresh or version persisted skill prompts when this path format changes, preserve absolute managed paths only where tilde would not resolve to the same readable root, and then prove the live AlphaClaw/Docker prompt and read path together.

## Reproduction Assessment

Yes. The linked issue has live before evidence, and source inspection shows current main compacts prompt paths while the read helper expands '~' through os.homedir(); I did not run the full AlphaClaw Telegram/Docker loop in this read-only review.

## Solution Assessment

No, not yet. The proposed prompt-rendering change is a plausible narrow fix for newly built prompts, but it misses persisted snapshot refresh and still needs maintainer acceptance of the absolute managed-path contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Refresh persisted prompts after changing their path format:** `src/skills/loading/workspace.ts:84-86`
  - body: This only changes newly rendered prompt paths. Existing sessions can already have a persisted `skillsSnapshot.prompt` containing the broken `~/.openclaw/...` location, and `resolveReusableWorkspaceSkillSnapshot` reuses that prompt when the snapshot version remains at the default 0 and the filter still matches. Bump/version the prompt format or otherwise force a narrow snapshot refresh so upgraded AlphaClaw sessions stop advertising the stale path.
  - confidence: 0.9
- **[P2] Preserve absolutes only when tilde cannot read them:** `src/skills/loading/workspace.ts:84-86`
  - body: The preserved-root check applies to every managed skill under the resolved config skills dir, including ordinary installs such as `/home/alice/.openclaw/skills` where the read tool's `~` expansion already resolves correctly. That makes prompts longer without improving readability and can push large skill catalogs into compact/truncated output, so gate preservation on the config skills root not being equivalent to the read tool's home expansion.
  - confidence: 0.78
