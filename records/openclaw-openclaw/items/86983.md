## What This Changes

The issue requests an outbound Slack DM recipient allowlist such as `allowTo`/`dmAllowTo` to block `openclaw message send --target <user>` from DMing non-allowlisted users.

## Best Possible Solution

Define an opt-in outbound recipient policy with maintainer-approved owner boundary, schema/help/docs, logging, and tests, then enforce it on Slack explicit sends without overloading inbound `allowFrom` semantics.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source inspection gives high confidence that current main supports Slack user DM targets but has no outbound recipient allowlist to enforce.

## Solution Assessment

Unclear pending maintainer design. The requested `allowTo` direction is plausible, but the maintainable fix needs an approved outbound-recipient policy boundary rather than reusing inbound `allowFrom` semantics ad hoc.
