## Summary

Keep open: current main has inbound DM allowlist enforcement and message-action guardrails, but no outbound recipient allowlist for Slack user-target sends. The request fits OpenClaw security/safe-default priorities, but it needs a maintainer product/API decision because it adds a new config surface across shared outbound messaging behavior.

## What This Changes

Request an outbound DM recipient allowlist, such as `allowTo` or `dmAllowTo`, so Slack user-target message sends can be blocked unless the target is approved.

## Best Possible Solution

Design a shared outbound direct-recipient policy at the message action/outbound plugin seam, then add schema, docs, and focused tests for Slack and any other DM-capable channels selected by maintainers.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection confirms current main has inbound `allowFrom` controls and outbound user-target sends, but no outbound recipient allowlist.

## Solution Assessment

Unclear pending maintainer design. The requested `allowTo` direction is reasonable, but the maintainable fix should likely be a shared outbound recipient-policy contract instead of a Slack-only ad hoc check.
