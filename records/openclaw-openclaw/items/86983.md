## Summary

Keep open: current main still has inbound Slack DM allowlists and cross-context message-action guardrails, but no outbound direct-recipient allowlist for Slack user-target sends. This is a security-hardening feature that fits OpenClaw priorities, but it needs maintainer product/API design because it adds a new config surface across shared outbound messaging behavior.

## What This Changes

Request to add an outbound Slack DM recipient allowlist such as `allowTo`/`dmAllowTo` so `openclaw message send --target user:<id>` cannot DM arbitrary users.

## Best Possible Solution

Design a maintainer-approved shared outbound direct-recipient policy, then add schema/help/docs and focused tests for Slack plus any selected shared send paths without changing existing sends unless operators opt in.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection confirms current main accepts Slack user targets and lacks a configured outbound recipient allowlist.

## Solution Assessment

Unclear pending maintainer design. The requested `allowTo` direction is reasonable, but the maintainable fix should likely be a shared outbound recipient-policy contract rather than a Slack-only ad hoc check.
