## Summary

## What This Changes

The PR records auto-saved Playwright downloads in page state and returns optional `downloads` metadata from browser `/act` responses.

## Best Possible Solution

Use an action-owned download waiter or equivalent bounded event window that starts before the click and waits through save completion, then choose one canonical API shape and require real browser proof.

## Reproduction Assessment

Yes at source level: current main auto-saves unmanaged Playwright downloads but `/act` click responses do not include any download completion payload. I did not run a live browser download in this read-only review.

## Solution Assessment

No as written: draining queued saves fixes one race, but the `/act` code still needs an action-owned download event wait and maintainers still need to choose the final API shape.
