Keep open: current main and the latest release still lack agent-visible metadata for click-triggered browser downloads, and this PR is a plausible fix, but merge should wait for live OpenClaw `/act` proof and maintainer acceptance of the overlap semantics.

## What This Changes

The PR adds action-scoped Playwright download capture for browser `/act`, returns optional `downloads` metadata in `/act` and typed client responses, and adds `pw-session` coverage for capture timing, overlap, and disposal.

## Best Possible Solution

Define the intended `/act` download ownership contract, keep focused coverage for that contract, and require redacted live `/act` output showing the `downloads` payload before merge.

## Reproduction Assessment

## Solution Assessment

Unclear: augmenting `/act` is plausible and lower-friction than adding a new action, but maintainers still need to choose implicit metadata versus explicit download actions and define overlap semantics.
