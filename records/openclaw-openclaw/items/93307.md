## Summary

Keep open: this PR targets a real source-level browser download feedback gap, but it still lacks real Chromium/Gateway proof and the current page-global pending-save drain can stall unrelated later actions.

## What This Changes

The PR records Playwright auto-saved downloads in page state and returns optional `/act` `downloads` metadata for click, clickCoords, batch, and evaluate responses.

## Best Possible Solution

Land one canonical browser-download feedback path that scopes saved downloads to the triggering action, preserves download-root constraints, and includes live Chromium/Gateway proof.

## Reproduction Assessment

Yes, at source level: current main and v2026.6.10 auto-save unmanaged downloads but non-evaluate `/act` responses return no download payload. I did not establish a live Chromium/Gateway reproduction in this read-only review.

## Solution Assessment

Unclear. The cursor and save-drain path is plausible, but it must scope drains to action-window downloads and maintainers still need to choose this implicit `/act` metadata shape versus explicit download actions.
