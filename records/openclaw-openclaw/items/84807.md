## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.78

Full review comments:

- **[P2] Probe the same env used to launch systemd-run:** `src/process/supervisor/adapters/child.ts:96-109`
  - body: resolveSupervisorBoundary() decides that systemd-run is available from the gateway's process.env, but the later spawn runs systemd-run with launchEnv, which can come from the worker override. If that override sets PATH without systemd-run, this path chooses systemd-scope and then fails with ENOENT instead of falling back inline. Resolve the launcher command/path against the same env used for spawn, or use an absolute gateway-resolved systemd-run path and cover the PATH-override case.
  - confidence: 0.84
