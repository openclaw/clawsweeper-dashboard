## Summary

Keep open. The plugin-local Poppler fallback is the right direction, but this PR still has a concrete resource-bound bug in the new renderer path and lacks after-fix proof from a real canvas-absent setup.

## What This Changes

The PR adds a plugin-local `pdftoppm` PDF image-rendering fallback, focused tests, PDF docs, and a changelog entry for canvas-unavailable document extraction.

## Best Possible Solution

Keep the fallback in `extensions/document-extract`, but enforce actual Poppler output bounds with scaling or dimension accounting and require real headless-Linux proof before merge.

## Reproduction Assessment

Yes. Source inspection on current main shows the canvas-missing path returns empty images, and the linked issue gives matching Ubuntu/Poppler setup evidence; I did not run a live PDF extraction in this read-only review.

## Solution Assessment

No as written. A plugin-local `pdftoppm` fallback is the narrow owner-boundary fix, but the current DPI clamp must be replaced with Poppler scaling or actual output dimension accounting before merge.
