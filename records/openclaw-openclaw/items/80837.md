## Summary

Keep open. The Poppler fallback is the right plugin-local direction, but the latest head still fails CI on temp-path guardrails, test typing, lint, and the real behavior proof gate.

## What This Changes

The PR adds a plugin-local `pdftoppm` PDF image-rendering fallback for `document-extract` when `@napi-rs/canvas` cannot load, with regression tests, PDF docs, and changelog text.

## Best Possible Solution

Keep the Poppler fallback plugin-local, use the plugin SDK temp-path helper, fix the test typing/lint failures, and require redacted real OpenClaw proof from a canvas-absent Poppler setup before merge.

## Reproduction Assessment

Yes, source-reproducible. Current main reaches `loadCanvasModule()` for low-text PDFs and returns empty images when canvas is missing, and the linked issue provides matching Ubuntu/Poppler setup evidence.

## Solution Assessment

No, not yet. The plugin-local fallback is the narrow maintainable direction, but this branch must fix CI guardrails and provide sufficient real behavior proof before merge.

Overall correctness: patch is incorrect

Full review comments:

  - body: The new renderer stages the input PDF under `os.tmpdir()`, and exact-head CI rejects this in messaging/channel runtime sources. Use `resolvePreferredOpenClawTmpDir()` or a plugin SDK temp workspace helper so user PDF staging stays under the guarded OpenClaw temp root.
  - confidence: 0.9

  - body: `extract()` returns `DocumentExtractionResult | null`, so the new tests fail `check-test-types` when reading `result.images` without first proving `result` is non-null. Add the explicit non-null guard pattern before the image assertions, including the later empty-images assertion.
  - confidence: 0.94
- **[P2] Drop the unnecessary import result assertions:** `extensions/document-extract/document-extractor.test.ts:157`
  - body: The dynamic imports already infer the module type, so the two `as typeof import(...)` assertions trigger current lint failures. Removing the assertions should keep the same runtime behavior and clear `no-unnecessary-type-assertion`.
  - confidence: 0.9
