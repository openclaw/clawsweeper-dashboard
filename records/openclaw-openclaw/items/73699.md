## Summary

Keep open. Current main now has a useful `voice.agentSession.target` path that routes Discord voice turns through an existing Discord session, but the requested bridge still includes automatic speaker-attributed transcript posting and text-channel reply delivery, and those remain unimplemented product/security decisions.

## What This Changes

Request an opt-in Discord voice bridge that routes spoken turns through the existing text-channel session and mirrors transcripts/replies to that text channel while speaking replies in voice.

## Best Possible Solution

Decide whether `voice.agentSession.target` is the intended product boundary; if not, add an opt-in Discord voice text-mirroring contract with validated targets, speaker-attributed transcript posting, reply delivery, TTS fan-out, docs, changelog, and focused tests.

## Reproduction Assessment

Yes. Source inspection on current main shows a high-confidence path: voice turns can share a target session through `voice.agentSession`, but `runDiscordVoiceAgentTurn` still uses `deliver: false` and there is no automatic transcript/reply text posting path.

## Solution Assessment

Unclear. The current `voice.agentSession.target` implementation is a good partial answer for shared context, but completing the requested text-channel mirroring needs an explicit product and safety contract before a narrow implementation is clearly best.
