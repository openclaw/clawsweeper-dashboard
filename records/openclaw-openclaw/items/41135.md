## Summary

Keep open. Current main still provides auth profile ordering, session stickiness, cooldown/failover, and limited API-key/profile rotation controls, but it does not define the requested opt-in provider-profile pool routing contract, scheduler strategies, per-profile in-flight/quota state, or CLI/status policy surface.

The issue requests an opt-in provider-profile routing layer for multi-account OAuth/API pools with configurable strategies, stickiness, concurrency limits, cooldown/quota awareness, and observability.
