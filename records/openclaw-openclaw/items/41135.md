## Summary

Keep open. Current main still only offers auth order, cooldown/disabled failover, usage snapshots, probes, and session pinning; the requested opt-in pool scheduler and route policy/state surface remains absent, and related #38775 is closed rather than an active canonical tracker.

## What This Changes

The request asks for an opt-in provider-profile routing layer for multiple OAuth/API profiles with routing policies, concurrency/in-flight awareness, cooldown/quota signals, status JSON, and CLI inspection.

## Best Possible Solution

Keep this as an open core design item for an additive auth-routing contract and scheduler that preserves session-sticky defaults while exposing typed pool config, pool state, CLI inspection, and regression coverage.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that current main lacks the requested pool routing contract and scheduler, while the reported Gemini timeout was not live-reproduced in this read-only review.

## Solution Assessment

Unclear until maintainers settle the public contract. The direction plausibly belongs in core because it crosses auth config, runner selection, status JSON, and CLI UX, but the schema and strategy semantics need product/API review first.
