## Summary

Keep open: current main still has ordered auth profiles, last-used round-robin, session stickiness, cooldown/failover state, and probe/status visibility, but it does not have the requested opt-in provider-profile pool routing contract or scheduler state. ## What This Changes

Add an OpenClaw-native provider-profile pool routing layer with opt-in strategies, per-profile runtime state, concurrency limits, cooldown/fallback behavior, and CLI/status observability.
