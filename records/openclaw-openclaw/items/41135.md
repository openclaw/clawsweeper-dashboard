Keep open: current main still supports ordered auth profiles, cooldown/failover, and session-sticky round-robin, but it lacks the requested opt-in provider-profile pool routing config, scheduler state, and per-profile concurrency/quota policy. Related items narrow pieces of the same area rather than superseding this broader API design tracker.

## What This Changes

The issue requests an opt-in provider-profile routing layer for multi-account OAuth/API pools, including routing strategies, per-profile state, concurrency limits, cooldown/failover policy, and status/CLI observability.
