## Summary

Keep open. Current main and the latest shipped release still lack a Slack option or routing branch for tool-use-conditioned auto-threading, and the request is a new Slack config/UX policy rather than a broken existing contract. The related Slack misrouting reports are useful context, but they track distinct safety/idempotency bugs rather than superseding this feature request.

## What This Changes

Add an opt-in Slack setting that sends tool-using agent replies into the triggering thread while leaving purely conversational replies at top level.

## Best Possible Solution

Have maintainers approve the Slack UX semantics first, then implement a Slack-owned opt-in schema/types/docs/routing change that preserves the current default and covers DMs, channels, inherited threads, message-tool sends, previews, and native streaming.
