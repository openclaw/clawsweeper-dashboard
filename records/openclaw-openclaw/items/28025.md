## Summary

Keep open. Current main still lacks a Slack tool-use-based reply threading config or routing path; existing behavior is driven by `replyToMode` and thread context, so this is a new Slack plugin routing policy rather than an implemented bug fix.

## What This Changes

Requests an opt-in Slack config that threads replies only when the agent used at least one tool and keeps pure conversational replies top-level.

## Best Possible Solution

Approve the desired Slack semantics first, then add an opt-in Slack config/schema/docs/tested dispatch change that decides routing before streaming starts.
