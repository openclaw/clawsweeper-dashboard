## Summary

Keep open. Current main still lacks `channels.slack.thread.replyOnToolUse` or an equivalent Slack reply-routing mode; the Slack plugin still plans reply targets from `replyToMode` and incoming thread state, while tool callbacks only update status/progress after that plan exists.

## What This Changes

The issue requests a default-false Slack config option that sends tool-using agent responses to a thread while leaving purely conversational responses at the top level.

## Best Possible Solution

Keep this open for maintainer product review. If accepted, implement the option inside the Slack plugin with a typed default-false config key, a reliable per-turn tool-use signal available before the first Slack transport action, preserved explicit-thread and reply-mode semantics, updated Slack docs/generated config metadata, and focused tests across DMs, channels, existing threads, native streaming, draft preview streaming, progress callbacks, and reply-mode interactions.
