## Summary

Keep open. Current main has Slack reply threading controls, but it does not implement conditional thread-on-tool-use behavior; this is a new Slack routing/config feature rather than an already-fixed bug.

## What This Changes

Add an optional Slack setting that sends agent replies into a thread when a turn used tools and keeps no-tool conversational replies at top level.

## Best Possible Solution

Design this as an opt-in Slack plugin routing policy, with schema/docs/tests covering DMs, channels, existing replyToMode behavior, thread.inheritParent, and Slack streaming before implementation.

## Reproduction Assessment

Not applicable: this is a feature/config request, not a broken existing contract. Source inspection confirms current main routes Slack replies from existing replyToMode/thread context rather than from whether the turn used tools.

## Solution Assessment

Unclear. The proposed direction is plausible, but maintainers should decide the exact Slack semantics and timing before implementation because tool-use detection interacts with streaming and existing replyToMode behavior.
