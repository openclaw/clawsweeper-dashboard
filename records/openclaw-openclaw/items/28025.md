## Summary

Keep open. Current main still lacks a Slack tool-use-conditioned thread-routing option, and the remaining work is a new Slack UX/config policy rather than an already-defined bug fix.

## What This Changes

The issue requests an opt-in Slack config policy that posts agent replies in a thread when a session used tools and keeps purely conversational replies top-level.

## Best Possible Solution

Approve the Slack UX first, then add an opt-in Slack-owned schema/types/docs/tested dispatch change that preserves current defaults and handles normal replies, streaming, previews, DMs, channels, and inherited thread context consistently.

## Reproduction Assessment

Not applicable: this is a new Slack config/policy request, not a broken existing contract. Source inspection on current main verifies there is no tool-use-based reply-routing branch or config key to exercise.

## Solution Assessment

Unclear. The tool-use heuristic is plausible, but current Slack reply planning and native streaming choose thread targets before tool callbacks, so maintainers should decide the UX and timing semantics before implementation.
