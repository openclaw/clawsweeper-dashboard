## Summary

Keep open. Current main still supports Slack threading through fixed reply modes and existing thread context, but it does not implement conditional thread-on-tool-use behavior; the linked wrong-thread report is adjacent but tracks a different bug.

## What This Changes

The issue requests an opt-in Slack setting that sends tool-using agent turns to a Slack thread while leaving purely conversational turns at top level.

## Best Possible Solution

Define an opt-in Slack plugin routing policy with schema, docs, and tests for DMs, channels, existing reply modes, thread inheritance, and streaming before implementing it.

## Reproduction Assessment

Not applicable: this is a feature/config request, not a broken existing contract. Source inspection shows current Slack routing is driven by replyToMode and thread context, while tool hooks only update progress/status.

## Solution Assessment

Unclear. The proposed signal is plausible, but maintainers should decide the Slack semantics and timing before implementation because tool-use detection would change reply routing around streaming and existing reply modes.
