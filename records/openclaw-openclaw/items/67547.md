## Summary

Keep open. Current main has removed the old tracked `src/canvas-host/a2ui` dirty-artifact churn and stabilized hash inputs, but the active Canvas plugin bundle config still resolves `ui/node_modules` before root dependencies and still lacks the `signal-polyfill` singleton aliases covered by https://github.com/openclaw/openclaw/pull/68934.

## What This Changes

The issue asks for A2UI bundle and hash output to be reproducible regardless of whether dependencies are present under `ui/node_modules` or root `node_modules`.

## Best Possible Solution

Rebase or replace the related PR onto the current Canvas plugin paths, add the `signal-polyfill` singleton aliases, verify bundle output with and without `ui/node_modules`, and keep generated bundle/hash artifacts untracked.

## Reproduction Assessment

Yes, source-reproducible with high confidence: the old tracked-file dirty diff is addressed, but current main still has the install-layout-sensitive resolver and missing singleton aliases. I did not perform a write-producing build in this read-only review.

## Solution Assessment

Yes, the narrow maintainable fix is to update the current Canvas plugin resolver/singleton alias table and verify both install layouts. Removing tracked artifacts and stabilizing hash inputs solved only part of the reported reproducibility problem.
