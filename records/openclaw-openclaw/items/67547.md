## Summary

Keep open. Current main has fixed the tracked generated-artifact churn by ignoring and untracking A2UI bundle/hash outputs, but the active Canvas A2UI resolver still prefers `ui/node_modules` and still lacks `signal-polyfill` singleton aliases, so the remaining install-layout-sensitive bundle path is not resolved.

## What This Changes

The issue reports that A2UI bundle/hash outputs change when `ui/node_modules` exists, making the generated Canvas A2UI artifacts non-reproducible across install layouts.

## Best Possible Solution

Add `signal-polyfill` and `signal-polyfill/` singleton aliases to the active Canvas plugin Rolldown config, verify output with and without `ui/node_modules`, and keep generated A2UI bundle/hash artifacts ignored and untracked.

## Reproduction Assessment

Partly. The original tracked-artifact dirt no longer reproduces from source because generated outputs are untracked, but current source still shows the active resolver path depends on `ui/node_modules` presence and lacks the reported singleton aliases.

## Solution Assessment

Yes for the direction, but not via the stale related PR as submitted. The narrow maintainable fix belongs in `extensions/canvas/src/host/a2ui-app/rolldown.config.mjs`, not the obsolete pre-plugin config or old tracked hash artifacts.
