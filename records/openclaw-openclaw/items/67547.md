## Summary

Keep open. Current main has fixed the tracked generated-artifact churn by untracking and ignoring A2UI bundle/hash outputs, but the active Canvas plugin bundler still prefers `ui/node_modules` and still lacks `signal-polyfill` singleton aliases, so the remaining install-layout-sensitive bundle path is unresolved.

## What This Changes

The issue requests byte-stable A2UI bundle/hash generation regardless of whether `ui/node_modules` exists during source builds.

## Best Possible Solution

Add `signal-polyfill` and `signal-polyfill/` singleton aliases to the active Canvas plugin Rolldown config, verify output with and without `ui/node_modules`, and keep generated A2UI outputs ignored and untracked.

## Reproduction Assessment

Yes at source level. Current main's active A2UI resolver still changes dependency roots based on `ui/node_modules`, and dependency source/metadata shows both signal consumers share `signal-polyfill`; I did not run the full build in this read-only review.

## Solution Assessment

Yes for the direction, but not via the stale related PR as submitted. The narrow maintainable fix belongs in `extensions/canvas/src/host/a2ui-app/rolldown.config.mjs`, with the already-merged untracked generated-output behavior preserved.
