## Summary

Keep open. Current main fixed the old tracked A2UI bundle/hash artifact churn, but the active Canvas A2UI Rolldown config still chooses `ui/node_modules` before root dependencies and still lacks `signal-polyfill` singleton aliases, so the remaining install-layout-sensitive bundle path is not fully resolved.

## What This Changes

The issue reports that A2UI bundle/hash output changes depending on whether `ui/node_modules` exists during `pnpm build`.

## Best Possible Solution

Apply the singleton aliases to the active Canvas plugin Rolldown config, verify A2UI bundle output with and without `ui/node_modules`, and keep generated bundle/hash artifacts ignored rather than tracked.
