## Summary

Keep open. Current main has removed the tracked generated A2UI bundle/hash churn, but the active Canvas A2UI resolver still prefers `ui/node_modules` and still lacks `signal-polyfill` singleton aliases, so the remaining install-layout-sensitive bundle path is not fixed.

## What This Changes

The issue reports that Canvas A2UI bundling produces different bundle/hash output depending on whether `ui/node_modules` exists.

## Best Possible Solution

Add `signal-polyfill` and `signal-polyfill/` aliases to `extensions/canvas/src/host/a2ui-app/rolldown.config.mjs`, verify generated output with and without `ui/node_modules`, and keep A2UI generated bundle/hash artifacts ignored and untracked.

## Reproduction Assessment

Yes. Source inspection shows the active resolver still changes dependency roots based on `ui/node_modules`, and the lockfile shows the signal packages share `signal-polyfill`; the actual two-layout bundle comparison was not run because the checkout had to remain read-only.

## Solution Assessment

Yes for the direction, but not via the old paths in the related PR. The narrow maintainable fix belongs in the active Canvas plugin Rolldown config, followed by bundle proof across both install layouts.
