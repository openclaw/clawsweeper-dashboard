## Summary

Keep open. Current main has fixed the tracked generated-artifact churn by leaving Canvas A2UI bundle/hash outputs untracked and ignored, but the active Canvas A2UI Rolldown resolver still prefers `ui/node_modules` over root dependencies and still lacks `signal-polyfill` singleton aliases; the related open PR at https://github.com/openclaw/openclaw/pull/68934 targets the right idea but is stale, dirty, and aimed at obsolete paths.

## What This Changes

The issue reports non-reproducible Canvas A2UI generated bundle/hash output depending on whether `ui/node_modules` is present during a source build.

## Best Possible Solution

Add `signal-polyfill` and `signal-polyfill/` singleton aliases to the active Canvas plugin Rolldown config, verify generated A2UI output with and without `ui/node_modules`, and keep generated bundle/hash artifacts ignored and untracked.

## Reproduction Assessment

Partly. The original tracked-artifact dirt no longer reproduces on current main because the generated files are untracked, but the remaining install-layout-sensitive resolver path is source-reproducible in `extensions/canvas/src/host/a2ui-app/rolldown.config.mjs`.

## Solution Assessment

Yes for the direction, but not via the stale PR as submitted. The narrow maintainable fix is to put the singleton aliases in the active Canvas plugin config and verify both dependency layouts, rather than refreshing old generated artifacts.
