## Summary

Keep open: the reported transcript-archive bug is valid and current main still lacks the full archival behavior, but this PR branch is not mergeable as proposed because it regresses current cron persistence semantics, can skip reset archival when historical run rows still reference the prior session id, and lacks after-fix real behavior proof.

## What This Changes

The PR adds reset-archive helpers and wires cron and heartbeat forced rotations to rename prior isolated-session transcript files to `.reset.<ts>`, with unit and heartbeat sandbox tests.

## Best Possible Solution

Land a rebased archival fix that preserves current cron persistence, filters historical run rows out of reset reference checks, and coordinates the heartbeat slice with https://github.com/openclaw/openclaw/pull/73809.

## Reproduction Assessment

Yes, at source level. Current main overwrites isolated cron or heartbeat entries during rotation while only deleted stale entries are archived, and the archive primitive skips any id included in the caller-provided reference set.

## Solution Assessment

No, not as currently proposed. Capturing before overwrite and archiving after persistence is the right direction, but this branch must preserve current cron persistence semantics and filter active references before it can merge.

## Review Findings

This branch writes the same mutable entry to the distinct run key, restoring the older run-row shape and potentially reintroducing references that block reset archival.
