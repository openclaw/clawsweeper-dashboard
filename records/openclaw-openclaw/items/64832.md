## Summary

Keep this PR open: current main still lacks same-key reset archival for rotated isolated-session transcripts, but this branch is not merge-ready because its cron reference-set logic can skip the archive, its cron persistence code predates current main, and it still lacks after-fix real behavior proof.

## What This Changes

The PR adds helpers and cron/heartbeat wiring to archive prior isolated-session transcript files as `.reset.<ts>` files after session rotation.

## Best Possible Solution

Land a rebased archival fix that captures the prior entry before overwrite, archives after successful persist with `reason: "reset"` and `restrictToStoreDir: true`, excludes historical run rows from active-reference checks, and preserves current cron persistence semantics.

## Reproduction Assessment

Yes, from source inspection rather than a live run. Current main overwrites prior isolated cron or heartbeat session entries while only archiving stale-key removals, and the archive primitive skips ids still present in the reference set.

## Solution Assessment

No, not as currently proposed. The direction is right, but the branch should not merge until it computes active references correctly, is rebased onto current cron persistence behavior, and has after-fix real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Exclude run-history ids from reset archives:** `src/cron/isolated-agent/session.ts:143-146`
  - body: `referencedSessionIds` is built from every session-store row, but isolated cron runs can leave historical `...:run:<id>` rows that still reference the just-rotated `sessionId`. When that happens, `archiveRemovedSessionTranscripts` skips the rename, so the reset-archive path does not actually archive the orphaned prior transcript for the common cron flow.
  - confidence: 0.91
- **[P2] Preserve current cron persistence semantics:** `src/cron/isolated-agent/run-session-state.ts:55-63`
  - body: This branch writes the same mutable `sessionEntry` back to the per-run key, but current main now persists cron base rows through the non-resumable `persistedEntry` path and does not alias mutable run rows. Reintroducing the old shape can restore stale historical references that block reset archival and undo the later cron isolation fix.
  - confidence: 0.86
