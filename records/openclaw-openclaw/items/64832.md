## Summary

Keep open: the underlying transcript-archive bug is plausible, but this PR is not merge-ready because it is stale against current cron persistence, still has two concrete correctness blockers, and lacks after-fix real behavior proof from the contributor.

## What This Changes

The branch adds archival helpers plus cron and heartbeat wiring/tests to rename prior isolated-session transcript files to `.reset.<ts>` after rotation.

## Best Possible Solution

Land a rebased archival fix that preserves current cron persistence, computes active referenced session ids correctly, coordinates the heartbeat overlap with https://github.com/openclaw/openclaw/pull/73809, and includes after-fix real behavior proof.

## Reproduction Assessment

Yes, at source level. Current main can overwrite prior isolated cron or heartbeat session entries while only deleted stale-key files are archived, and the archive primitive skips ids still present in the caller-provided reference set.

## Solution Assessment

No, not as currently proposed. Capturing the prior entry before overwrite and archiving after persistence is the right direction, but this branch must preserve current cron persistence semantics and filter active references before it can merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve current cron persistence semantics:** `src/cron/isolated-agent/run-session-state.ts:62-63`
  - body: This branch still writes the same mutable `sessionEntry` to a distinct `runSessionKey`, but current main intentionally persists only the stable cron key and stores a non-resumable entry until a transcript exists. Rebasing this as-is would restore the stale per-run row shape removed by the merged cron isolation work and can reintroduce references that block reset archival.
  - confidence: 0.9
- **[P2] Exclude run-history ids from reset archives:** `src/cron/isolated-agent/session.ts:143-146`
  - body: `referencedSessionIds` is built from every row in the store, so legacy or historical `...:run:<id>` rows can still reference the prior session id being rotated. In that case `archiveRemovedSessionTranscripts` skips the rename and the orphaned transcript remains outside reset retention; build this set from active session rows only or ignore historical run rows for reset archival.
  - confidence: 0.88
