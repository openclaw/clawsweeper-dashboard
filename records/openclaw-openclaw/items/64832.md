## Summary

Keep open: the reported orphaned transcript bug is valid, but this PR is not merge-ready because its branch is conflicting, reintroduces obsolete cron run-key persistence, can skip reset archival when historical run rows still reference the prior session id, and still lacks after-fix real behavior proof.

## What This Changes

This PR adds cron/heartbeat reset-archive helpers and tests so prior isolated-session transcript files are renamed to `.reset.<ts>` after forced rotation.

## Best Possible Solution

Land a rebased archival fix that preserves current cron persistence, filters active referenced session ids correctly, coordinates the heartbeat slice with https://github.com/openclaw/openclaw/pull/73809, and includes after-fix real behavior proof.

## Reproduction Assessment

Yes, at source level. Current main overwrites isolated cron or heartbeat session entries during rotation while only stale deleted entries are archived, and the archive primitive skips ids still present in the caller-provided reference set.

## Solution Assessment

No, not as currently proposed. Capturing before overwrite and archiving after persistence is the right direction, but this branch must be rebased onto current cron persistence semantics and must filter active references before it can merge.
