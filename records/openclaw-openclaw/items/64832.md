## Summary

Keep open: current main still appears to need archive-on-rotation coverage, but this PR should not merge as-is because it can skip cron archives when historical run rows still reference the prior session id, it is stale relative to current-main cron persistence, and it lacks after-fix real behavior proof.

## What This Changes

The PR adds session archival helpers plus cron and heartbeat wiring/tests to rename prior isolated-session transcript files to `.reset.<ts>` archives after rotation.

## Best Possible Solution

Land a rebased archival fix that captures the prior entry before overwrite, archives it after successful persist with `reason: "reset"` and `restrictToStoreDir: true`, excludes historical run rows from active-reference checks, and preserves current-main cron persistence semantics.

## Reproduction Assessment

Yes, from source inspection rather than a live run. Current main overwrites prior isolated cron or heartbeat session entries while only archiving stale-key removals, and the archive primitive skips ids still present in the reference set.

## Solution Assessment

No, not as currently proposed. The direction is right, but the branch should not merge until it computes active references correctly and is rebased to preserve current-main cron-session persistence behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Exclude historical run rows from reset archival:** `src/cron/isolated-agent/session.ts:143-146`
  - body: `referencedSessionIds` is built from every session-store row, but isolated cron runs can leave historical `...:run:<id>` rows that still reference the just-rotated `sessionId`. When that happens, `archiveRemovedSessionTranscripts` skips the rename, so the new reset-archive path is a no-op for the common cron flow; build the reset reference set from active rows or exclude run-history keys for this path.
  - confidence: 0.91
- **[P2] Preserve current cron persistence semantics:** `src/cron/isolated-agent/run-session-state.ts:57-62`
  - body: This branch writes the same mutable `sessionEntry` back to the per-run key, but current main no longer follows that shape after https://github.com/openclaw/openclaw/pull/71340. Reintroducing the per-run alias can restore stale historical references that block the reset archive; rebase on current main and add only archive-on-rotation behavior.
  - confidence: 0.86
