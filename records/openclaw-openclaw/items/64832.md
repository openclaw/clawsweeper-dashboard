## Summary

Keep open: current main still lacks reset archival for the prior isolated-session transcript, so the underlying disk-growth bug is real, but this PR cannot merge as-is. The branch’s reset helper can skip the archive when historical cron run rows still reference the prior session id, and it is stale against current main’s cron-session persistence semantics.

## What This Changes

The branch adds helpers plus cron and heartbeat wiring/tests to rename prior isolated-session transcript files to `.reset.<ts>` archives after rotation.

## Best Possible Solution

Land a rebased archival fix that captures the prior entry before overwrite, archives it after a successful persist with `reason: "reset"` and `restrictToStoreDir: true`, excludes historical run rows from active-reference checks, and preserves current-main cron persistence behavior.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a prior isolated cron or heartbeat entry with a transcript is overwritten on rotation, current main does not archive that prior file, and the PR’s helper can skip its archive when an older `...:run:<id>` row still references the prior session id.

## Solution Assessment

No. The direction is right, but the branch should not merge until it computes active references correctly and is rebased to keep current-main cron-session persistence semantics.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Exclude historical run rows from reset archival:** `src/cron/isolated-agent/session.ts:143-146`
  - body: `referencedSessionIds` is built from every session-store row, but isolated cron runs can leave historical `...:run:<id>` rows that still reference the just-rotated `sessionId`. When that happens, `archiveRemovedSessionTranscripts` skips the rename, so the new reset-archive path is a no-op for the common cron flow. Build the reset reference set from active rows or exclude run-history keys for this rotation archive path.
  - confidence: 0.91
- **[P2] Preserve current cron persistence semantics:** `src/cron/isolated-agent/run-session-state.ts:57-62`
  - body: This branch writes the same mutable `sessionEntry` back to the per-run key, but current main no longer follows this shape after the isolated-cron state fixes in https://github.com/openclaw/openclaw/pull/71340. Reintroducing the per-run row aliases stale state and creates historical references that can block the reset archive. Rebase on current main and add only the archive-on-rotation behavior without restoring the stale run-key persistence path.
  - confidence: 0.86
