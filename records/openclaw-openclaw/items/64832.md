## Summary

Keep open: the bug class is real, but this PR is not merge-ready because it conflicts with current cron persistence semantics, can skip the intended reset archive when historical run rows still reference the prior session, lacks after-fix real behavior proof, and overlaps the active heartbeat-only fix PR.

## What This Changes

The PR adds helpers and call-site wiring to archive prior isolated-session transcript files on cron and heartbeat forced rotations, with unit and heartbeat sandbox tests.

## Best Possible Solution

Land a rebased archival fix that preserves current cron persistence, filters active reset references correctly, and coordinates or splits the heartbeat portion with https://github.com/openclaw/openclaw/pull/73809.

## Reproduction Assessment

Yes, at source level. Current main still has a heartbeat same-key overwrite path without reset archival, and the PR's cron helper failure is traceable from its reference-set construction plus the archive guard contract.

## Solution Assessment

No, not as currently proposed. Capturing before overwrite and archiving after persistence is the right direction, but this branch must preserve current cron persistence semantics and filter historical run rows out of reset reference checks before it can merge.
