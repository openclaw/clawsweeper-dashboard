## Summary

Keep open: current main still lacks the broader isolated-session reset-archive behavior, but this branch is not merge-ready because it conflicts with current cron persistence semantics and lacks after-fix real behavior proof.

## What This Changes

The PR adds helpers plus cron and heartbeat wiring to archive prior isolated-session transcript files as `.reset.<ts>` after rotation, with regression tests for the archival flow.

## Best Possible Solution

Land a rebased archival fix that captures the prior entry before overwrite, archives after successful persistence with `reason: "reset"`, excludes historical run rows from active-reference checks, preserves current cron persistence semantics, and coordinates the heartbeat-only overlap with https://github.com/openclaw/openclaw/pull/73809.

## Reproduction Assessment

Yes, at source level rather than a live run. Current main overwrites prior isolated cron or heartbeat session entries while only archiving stale-key removals, and the archive primitive skips ids still present in the caller-provided reference set.

## Solution Assessment

No, not as currently proposed. The direction is right, but the branch should not merge until it computes active references correctly, is rebased onto current cron persistence behavior, and has after-fix real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Exclude run-history ids from reset archives:** `src/cron/isolated-agent/session.ts:143-146`
  - body: `referencedSessionIds` is built from every session-store row, but isolated cron runs can leave historical `...:run:<id>` rows that still reference the just-rotated `sessionId`. When that happens, `archiveRemovedSessionTranscripts` skips the rename, so the reset-archive path does not actually archive the orphaned prior transcript for the common cron flow.
  - confidence: 0.91
- **[P2] Preserve current cron persistence semantics:** `src/cron/isolated-agent/run-session-state.ts:55-63`
  - body: This branch writes the same mutable `sessionEntry` back to the per-run key, but current main now persists cron base rows through the non-resumable `persistedEntry` path and does not alias mutable run rows. Reintroducing the old shape can restore stale historical references that block reset archival and undo the later cron isolation fix.
  - confidence: 0.86
