## What This Changes

The issue asks `openclaw update` to detect a system-level `openclaw.service` and avoid reporting restart success when the updater can only handle the user-scoped service path.

## Best Possible Solution

Use https://github.com/openclaw/openclaw/pull/57276, or a narrower replacement derived from it, as the canonical fix path while preserving this issue’s concrete 2026-05-07 cutover evidence.

## Reproduction Assessment

Yes. Source inspection shows that a package update with no user-scoped unit leaves `serviceState.installed` false, so the stop/restart helper is not prepared even if a system-level unit is active; I did not run a live systemd system-service update in this checkout.

## Solution Assessment

Yes for the requested direction: detecting system-scope service ownership and returning explicit sudo/manual restart guidance is the narrow maintainable fix. The remaining work should be handled in the canonical PR thread rather than by keeping this duplicate issue open.
