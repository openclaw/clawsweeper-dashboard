## Summary

Close as implemented on current main: the reported npm resolver failure is covered by the main-only Bedrock runtime dependency pin, with a package override, lockfile convergence, changelog note, and regression guard in place. The fix has not shipped to npm yet, so users on 2026.4.29 remain affected until the next release.

## What This Changes

The issue reports global and local npm installs of OpenClaw 2026.4.27 through 2026.4.29 failing during AWS SDK/Smithy dependency resolution on macOS with npm 11 and Node 25.

## Best Possible Solution

Keep the Bedrock runtime override and ship it in the next npm release; reopen only if the same macOS/npm resolver failure persists on a version newer than 2026.4.29.

## Reproduction Assessment

Yes for the affected releases: the report provides concrete npm commands, versions, and debug-log symptoms, and related #75963 shows the same AWS/Smithy resolver signature. I did not run a live macOS Node 25 install in this read-only review; current-main verification is source, lockfile, and guard-test based.

## Solution Assessment

Yes. Pinning the problematic Bedrock runtime dependency from the root package is the narrowest maintainable fix for an install-time npm resolver failure, and npm's own package-json docs confirm root overrides are the dependency-tree mechanism for this kind of resolution control.
