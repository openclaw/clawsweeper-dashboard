## Label Justifications

- `P1`: Automated zalouser media delivery is reported broken across CLI, agent, and cron workflows, making a real channel attachment workflow unusable.
- `impact:message-loss`: The reported failure leaves intended file/image messages undelivered and no platform message id is produced.
- `impact:crash-loop`: The affected non-interactive sends can hang until Node exits with code 13, and scheduled runs can remain stuck.

## Summary

Keep open; current main and the latest release still route non-dry-run zalouser media sends through the direct plugin adapter into zca-js attachment upload, and no merged or open PR search result appears to own the reported non-interactive hang.

## What This Changes

The issue reports that zalouser media/file sends hang and eventually exit with Node code 13 in non-interactive CLI, agent, and cron contexts while text-only sends and interactive media replies work.

## Best Possible Solution

Make non-interactive zalouser media sends use a valid live upload path and return a platform receipt, with a bounded actionable error when the required session is unavailable.

## Reproduction Assessment

Yes, with source-level confidence rather than an independent live repro: the report provides exact v2026.6.1 steps, and current main plus v2026.6.8 still have the same direct media upload path. I could not run a live Zalo Personal account here.

## Solution Assessment

No current-main solution is sufficient yet. The maintainable fix should address the zalouser media upload/session boundary and bounded failure behavior, not only local media validation or dry-run handling.
