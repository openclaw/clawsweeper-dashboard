## Summary

Keep open; current main and v2026.6.8 still send non-interactive zalouser media through the direct plugin adapter into zca-js attachment upload, and no merged/open related item clearly owns the reported hang.

## What This Changes

The issue reports that zalouser media/file sends hang and can exit with Node code 13 in non-interactive CLI, agent-message, and cron contexts, while text sends and inbound-triggered interactive media sends work.

## Best Possible Solution

Make non-interactive zalouser media sends use a valid live upload path and return a platform receipt, with a bounded actionable error when the required upload/listener session is unavailable.

## Reproduction Assessment

Yes, at source level rather than from an independent live run: the reporter provided exact v2026.6.1 commands, and current main plus v2026.6.8 still retain the same direct media upload path. I could not run a live Zalo Personal account in this read-only review.

## Solution Assessment

Unclear until maintainers choose the session ownership policy. The current direct upload path without bounded failure is not sufficient; the repair should address the zalouser upload/session boundary rather than local media validation or dry-run behavior.
