## Summary

Keep open: current main still has the TUI inline reasoning-tag leak, but this branch should not merge as-is because it rewrites non-assistant string content, diverges from existing nested-tag semantics, and is missing the required user-facing changelog entry.

## What This Changes

This PR adds shared inline reasoning-tag extraction and routes TUI string content through it so tagged assistant reasoning can be hidden or shown separately, with formatter regression tests.

## Best Possible Solution

Land a rebased TUI formatter fix that parses inline reasoning tags only for assistant string content, shares depth-aware tag extraction semantics with the existing stripper, preserves phase-aware assistant rendering, adds focused regressions, and records the user-facing privacy fix in the changelog.

## Reproduction Assessment

Yes. Source inspection shows `TuiStreamAssembler` receives assistant string content, `extractThinkingFromMessage()` returns no thinking for strings, and `extractContentFromMessage()` returns the sanitized full string, so `<think>hidden</think>Visible` remains visible when thinking display is off.

## Solution Assessment

No. The PR direction is right, but the implementation should be role-aware, reuse the existing depth/code-region semantics, keep current phase-aware assistant extraction intact, and include the required changelog entry.
