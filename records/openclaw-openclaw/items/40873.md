## What This Changes

The PR adds inline `<think>...</think>` extraction/stripping for TUI string message content and formatter regression tests for assistant string payloads.

## Best Possible Solution

Land a rebased, role-aware TUI formatter fix that parses inline reasoning tags only for assistant string content, reuses depth-aware shared tag semantics, preserves current phase-aware assistant extraction, and adds focused regression coverage.

## Reproduction Assessment

Yes. A high-confidence source path is `TuiStreamAssembler` receiving `{ role: "assistant", content: "<think>hidden</think>Visible" }` with thinking display off; current main passes the string through visible content while the docs say closed thinking blocks stay hidden.

## Solution Assessment

No. The PR direction is right, but the current patch should be rebased and narrowed so tagged parsing applies to assistant strings only and the extractor matches the existing depth-aware stripping semantics.
