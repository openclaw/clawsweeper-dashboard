## Summary

Keep open. Current `main` still has the TUI assistant-string reasoning-tag leak, and this PR is useful source work, but the proposed patch would also rewrite non-assistant transcript strings and needs a narrow repair before merge.

## What This Changes

The PR adds inline `<think>...</think>` extraction and stripping for TUI string message content plus formatter regression coverage.

## Best Possible Solution

Land a rebased TUI formatter fix that parses inline reasoning tags only for assistant string content, shares depth-aware tag semantics, preserves current phase-aware assistant extraction, adds focused regressions, and documents the user-facing fix.

## Reproduction Assessment

Yes. A high-confidence source path is `TuiStreamAssembler` receiving `{ role: "assistant", content: "<think>hidden</think>Visible" }` with thinking display off; current `main` keeps that string as visible content.

## Solution Assessment

No. The PR direction is right, but the current patch should be role-aware, keep nested/code-region parsing aligned with the existing stripper, preserve current phase-aware assistant handling, and add the required changelog entry.
