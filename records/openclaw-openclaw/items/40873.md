## Summary

Keep this PR open: it addresses a real TUI string-content reasoning leak that is still source-reproducible on current main, but the branch needs correctness fixes and after-fix real behavior proof before merge.

## What This Changes

The branch adds inline `<think>...</think>` extraction and stripping for TUI string content plus focused formatter regression tests.

## Best Possible Solution

Rebase and narrow the formatter change so only assistant string payloads are split into reasoning and visible text, use extraction semantics that match the depth-aware parser, preserve literal user and command text, and add focused regression coverage plus real TUI/session proof.

## Reproduction Assessment

Yes, by source inspection: feed an assistant message with string content like `<think>hidden</think>Visible` through `TuiStreamAssembler` with thinking disabled, and current main still routes it through `extractContentFromMessage()` without reasoning-tag stripping. I did not run a live TUI session because this review is read-only.

## Solution Assessment

No. The PR targets the right bug, but the current patch should be assistant-role scoped, depth-aware, rebased onto current TUI formatter semantics, and backed by after-fix real behavior proof.
