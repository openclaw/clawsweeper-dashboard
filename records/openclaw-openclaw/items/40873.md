## Summary

Keep open: current main still has a source-proved TUI formatter path for assistant string reasoning tags, but this external PR is not merge-ready because it lacks real behavior proof, conflicts with main, and introduces blocking review issues.

## What This Changes

The PR adds shared extraction for inline `<think>...</think>` reasoning tags, routes TUI string content through that splitter, and adds formatter regression tests.

## Best Possible Solution

Land a rebased, role-aware TUI formatter fix that strips inline reasoning tags only from assistant string content, reuses depth-aware shared tag semantics, and keeps focused regression coverage.

## Reproduction Assessment

Yes, from source inspection: `TuiStreamAssembler.updateRunState` uses helpers that currently return empty thinking and unstripped assistant string content for `<think>hidden</think>Visible`. I did not run a live TUI session in this read-only review.

## Solution Assessment

No. The PR targets the right gap, but the patch should be rebased and narrowed so tagged parsing applies only to assistant string content and the extractor matches the existing depth-aware stripping semantics.
