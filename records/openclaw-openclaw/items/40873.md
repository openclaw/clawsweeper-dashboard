## Summary

Keep open. The PR targets a real TUI formatter gap, but the proposed patch still has merge-blocking correctness issues: it parses non-assistant strings as reasoning, diverges from the existing depth-aware tag semantics, and omits the required changelog entry.

## What This Changes

The PR adds inline reasoning-tag extraction for TUI string content and formatter regression tests for tagged assistant payloads.

## Best Possible Solution

Land a rebased TUI formatter fix that applies inline reasoning-tag parsing only to assistant string content, shares the current depth/code-region/malformed-tag semantics, preserves user and command text, adds focused regressions, and includes a single-line changelog entry.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is an assistant message with string content like `<think>alpha</think>Visible`; current main's streaming assembler consumes formatter helpers that do not strip or separate those tags on the assistant string path.

## Solution Assessment

No. The PR's direction is right, but this patch is not the best final form because it is not role-aware, adds a divergent boolean extractor, and omits the required user-facing changelog entry.
