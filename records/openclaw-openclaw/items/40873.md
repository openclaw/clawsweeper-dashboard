## Summary

Keep open: the PR targets a real TUI string-content reasoning leak that is still source-reproducible on current main, but the branch is conflicting, lacks contributor real behavior proof, and introduces role-scope and nested-tag parsing defects.

## What This Changes

The branch adds a shared inline reasoning-tag extractor, applies tagged string parsing in TUI formatter helpers, and adds formatter regression tests for assistant string payloads.

## Best Possible Solution

Land a rebased, role-aware TUI formatter fix that parses inline reasoning only for assistant string content, shares depth-aware tag semantics with the existing stripper, preserves current phase-aware assistant rendering, and includes focused tests plus real TUI/session proof.

## Reproduction Assessment

Yes. A high-confidence source path is `TuiStreamAssembler` receiving an assistant message with `content: "<think>hidden</think>Visible"` while thinking display is off; current main routes the string through visible content while docs require closed thinking blocks to stay hidden.

## Solution Assessment

No. The PR direction is right, but the current patch should be rebased and narrowed so tagged parsing applies only to assistant string content and the new extraction logic matches the existing depth-aware parser.
