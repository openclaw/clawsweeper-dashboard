## Summary

Keep open: the PR targets a real TUI reasoning-tag leak, but the current patch is not mergeable as-is because it rewrites non-assistant string content, diverges from the existing nested-tag parser contract, conflicts with current main, and still lacks real behavior proof.

## What This Changes

The PR adds inline `<think>...</think>` extraction and stripping for TUI string content, plus formatter regression tests.

## Best Possible Solution

Rebase and revise the fix so only assistant string payloads split inline reasoning tags, the extractor mirrors the depth-aware shared reasoning-tag contract, and redacted real TUI/session proof is attached.

Yes. Source inspection shows that feeding `TuiStreamAssembler` an assistant string like `<think>hidden</think>Visible` with thinking disabled still routes through formatter helpers that do not strip inline tags on current main.

## Solution Assessment

No. The PR targets the right bug, but the implementation should be rebased, restricted to assistant strings, and aligned with the existing nested-tag parser before merge.
