## Summary

Keep open. The PR targets a real TUI helper gap, but the current patch can strip literal user/command text, diverges from the existing nested reasoning-tag parser, and still lacks required after-fix real TUI proof.

## What This Changes

The PR adds inline `<think>...</think>` extraction for TUI string content and regression tests for hiding or showing that reasoning text.

## Best Possible Solution

Rebase this as an assistant-only TUI/live-chat fix that reuses canonical reasoning-tag semantics, preserves literal user/command examples, adds nested and non-assistant regressions, and attaches redacted real TUI/session output proof.

## Reproduction Assessment

Yes, at source level. Current main forwards assistant string content through TUI helpers that do not extract or strip inline reasoning tags, and streamed TUI rendering uses those helpers.

## Solution Assessment

No. The PR targets the right bug, but the implementation should be constrained to assistant-visible strings and share the existing nested/depth parser semantics instead of adding a divergent broad string parser.
