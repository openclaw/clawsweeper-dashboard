## Summary

## What This Changes

The branch adds a shared inline reasoning-tag extractor, routes TUI string content through it, and adds formatter regressions for hiding or showing tagged thinking.

## Best Possible Solution

Rebase this as an assistant-only TUI/live-chat fix that reuses canonical reasoning-tag semantics, preserves literal user/command examples, adds nested and non-assistant regressions, and attaches redacted real TUI/session output proof.

## Reproduction Assessment

Yes, at source level. Current main still lets assistant string content pass through TUI helpers without extracting or stripping inline reasoning tags, and streamed TUI rendering uses those helpers.

## Solution Assessment

No. The PR targets the right bug, but the implementation should be constrained to assistant-visible strings and share the existing nested/depth parser semantics instead of adding a divergent broad string parser.
