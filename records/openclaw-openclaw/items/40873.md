## Summary

Keep open. The PR targets a real TUI formatter gap, but the proposed patch still has correctness blockers: it strips literal tagged text from non-assistant transcript strings, diverges from the existing nested reasoning-tag semantics, and misses the required changelog entry.

## What This Changes

The PR adds inline reasoning-tag extraction for TUI string content and formatter regression tests for tagged assistant payloads.

## Best Possible Solution

Land a rebased TUI formatter fix that splits inline reasoning only for assistant string content, shares current depth/code-region/malformed tag semantics, preserves user and command text, adds focused regression coverage, and includes the changelog entry.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is an assistant message with string content like `<think>alpha</think>Visible`; current main's TUI formatter string path still treats that as visible content while the live assembler consumes the same helper path.

## Solution Assessment

No. The PR's direction is right, but this patch is not the best final form because it is not role-aware, adds a divergent boolean extractor, and omits the required user-facing changelog entry.
