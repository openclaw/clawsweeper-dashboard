## Summary

Keep open: the TUI string-content leak remains source-reproducible, but this external PR should not merge as-is because it broadens parsing into non-assistant strings, diverges from existing nested-tag semantics, and still lacks real behavior proof.

## What This Changes

The PR adds inline reasoning-tag extraction for TUI string content and tests that assistant string payloads hide or surface `<think>...</think>` content according to the thinking display setting.

## Best Possible Solution

Rebase as an assistant-only TUI/live-chat fix that reuses the canonical reasoning-tag semantics, adds user-literal and nested-tag regressions, and attaches redacted real TUI output proof.

## Reproduction Assessment

Yes, at source level. Current main’s embedded TUI path forwards assistant text into live chat projection without reasoning-tag stripping, and the stream assembler then renders assistant string content via `extractContentFromMessage`.

## Solution Assessment

No. The PR targets the right bug, but it should be constrained to assistant-visible strings and should mirror the existing nested/depth parser semantics rather than adding a divergent extractor.
