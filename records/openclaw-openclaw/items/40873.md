## Summary

Keep open. The formatter bug is still real on current main, but this PR should not merge as written because it strips literal tags from non-assistant transcript strings and diverges from the existing nested reasoning-tag contract.

## What This Changes

The PR adds inline `<think>...</think>` extraction for TUI string content and formatter regression tests for tagged assistant payloads.

## Best Possible Solution

Land a current-main TUI hardening patch that splits inline reasoning only for assistant string content, preserves user and command strings literally, shares the depth-aware reasoning-tag semantics for nested, malformed, final-tag, and code-region cases, and includes regression coverage plus the required changelog entry.

## Reproduction Assessment

Yes. A high-confidence unit reproduction is an assistant message with string content like `<think>hidden</think>Visible`; current main returns no thinking text for string content and renders the assistant string through plain sanitization.

## Solution Assessment

No. The direction is right, but this patch is not the best final form because it is not role-aware, adds a divergent boolean extractor, and omits the required user-facing changelog entry.
