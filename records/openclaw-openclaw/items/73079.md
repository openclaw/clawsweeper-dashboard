## Summary

Keep open. Current main and the latest release still omit the explicit MiniMax TTS `stream: false` and `output_format: "hex"` request fields while decoding `data.audio` as hex, so this focused PR is still useful and not superseded.

## What This Changes

This PR adds `stream: false` and `output_format: "hex"` to MiniMax TTS requests and asserts those fields in the speech-provider test.

## Best Possible Solution

Land the narrow MiniMax plugin request-shape fix after maintainer review and proof-check refresh, while preserving the merged envelope-error handling from https://github.com/openclaw/openclaw/pull/93688.

## Reproduction Assessment

Yes from source and the upstream API contract, but not from an independently replayed live current-main call. Current main decodes `data.audio` as hex while omitting explicit output-shape fields, and the contributor supplied after-fix live output showing MiniMax accepts the fields and returns success.

## Solution Assessment

Yes. The MiniMax plugin owns this vendor request shape, so adding the fields in `minimaxTTS` is narrower than a core speech change and separate from the already-merged envelope-error fix in https://github.com/openclaw/openclaw/pull/93688.
