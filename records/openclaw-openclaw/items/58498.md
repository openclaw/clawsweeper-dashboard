The issue asks for the status usage/quota line to use the same OAuth profile that the session status card shows as active.

## Best Possible Solution

Review and land the existing paired fix, preserving the provider-ownership guard and provider-order fallback while carrying the session-selected OAuth profile into usage auth resolution.

## Solution Assessment
