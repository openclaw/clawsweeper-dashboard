## Summary

Keep open. Current main still disables Slack draft preview streaming for flat direct messages by requiring a thread timestamp for DMs, while the draft preview send path already supports posting to the DM channel without one. The PR is directionally correct, but it still needs docs/config help and changelog alignment before merge or replacement.

## What This Changes

The PR removes the Slack direct-message `threadTs` gate from preview-streaming eligibility and updates Slack dispatch streaming tests to expect preview streaming for top-level and threaded DMs.

## Best Possible Solution

Land or replace this with a Slack-owned fix that enables draft post-and-edit preview streaming for flat DMs, leaves native Slack streaming thread requirements unchanged, updates Slack docs/config help and `CHANGELOG.md`, and validates the focused Slack streaming paths.

## Reproduction Assessment

Yes. On current main, `shouldEnableSlackPreviewStreaming({ mode: "partial", isDirectMessage: true })` returns false, and the current eligibility test asserts that disabled flat-DM path.

## Solution Assessment

Mostly yes. Removing the DM-specific `threadTs` gate is the narrowest code fix for draft previews because `createSlackDraftStream` already posts to the target with optional thread metadata; the safer merge path is to align docs/config/changelog and rerun focused validation.
