## Summary

Keep open. Current main still disables Slack draft preview streaming for top-level DMs, while the PR’s core change matches the draft-stream path; the remaining work is to reapply it on current main and align Slack docs, config help, and changelog before merge.

## What This Changes

The PR removes Slack preview streaming’s direct-message `threadTs` gate and updates three Slack dispatch streaming eligibility tests to expect draft previews for flat DMs.

## Best Possible Solution

Land or replace this with a Slack-owned fix that enables draft post-and-edit previews for flat direct messages, keeps the native Slack streaming thread requirement separate, and updates operator-facing guidance plus the active changelog entry.

## Reproduction Assessment

Yes. Current main reproduces at helper level: `shouldEnableSlackPreviewStreaming({ mode: "partial", isDirectMessage: true })` returns `false`, and the current unit test asserts that disabled path.

## Solution Assessment

Mostly yes. Removing the preview-only direct-message `threadTs` gate is the narrow maintainable fix because `createSlackDraftStream` can send to the DM target with an optional `threadTs`; the safer merge path is to preserve the native-streaming gate and update docs/config/changelog.
