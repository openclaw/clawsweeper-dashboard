## Summary

Keep open. Current main still has reachable resolver paths that can turn `media://inbound/...` into a workspace `media:/inbound/...` path, and two open closing PRs already target the affected shared-loader and sandbox surfaces.

## What This Changes

The issue asks for `media://inbound/<image>` references to resolve to OpenClaw-managed inbound media instead of being treated as workspace-relative local paths.

## Best Possible Solution

Handle `media://` as managed media before any workspace or sandbox path join, with focused regression coverage for the shared loader, image tool with `workspaceDir`, and sandbox bridge entry point.

## Reproduction Assessment

Yes, at source level. Current main still has `media://` values flowing into workspace and sandbox path resolution, and Node's path primitive produces the reported `/workspace/media:/inbound/...` shape.

## Solution Assessment

Yes for triage. Keeping this issue open while maintainers review or consolidate the linked closing PRs is the narrowest maintainable path.
