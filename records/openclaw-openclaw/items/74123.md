## Summary

Keep open. Current main still has resolver paths that can turn a `media://inbound/...` image reference into a workspace `media:/inbound/...` path, and open closing PRs already target the affected areas.

## What This Changes

The issue requests that inbound `media://...` image references resolve to OpenClaw-managed media instead of becoming malformed workspace `media:/...` paths.

## Best Possible Solution

Land or consolidate a focused fix that treats `media://` as a media-store URI before workspace or sandbox bridge path resolution, with regression coverage for the image tool, shared loader, and sandbox helper paths.

## Reproduction Assessment

Yes. The source-level path is high-confidence: `media://inbound/example.png` can reach workspace-relative path resolution, and Node turns it into `/workspace/media:/inbound/example.png`.

## Solution Assessment

Yes for triage. Keeping this issue open while maintainers review or consolidate the open closing PRs is the narrowest maintainable path; the code fix should preserve existing local-root and inbound-media validation boundaries.
