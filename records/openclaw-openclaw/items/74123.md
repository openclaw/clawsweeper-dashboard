## Summary

Keep open. Current main still contains resolver paths that can turn `media://inbound/...` into a workspace `media:/...` path, and two open PRs already reference this issue with closing syntax, so the issue should stay open until a fix lands or those PRs are consolidated.

## What This Changes

The issue requests that inbound `media://inbound/<file>` image references resolve to OpenClaw-managed media files instead of malformed workspace paths.

## Best Possible Solution

Land or consolidate a focused fix that recognizes `media://` as a media-store URI before workspace or sandbox bridge path resolution, with regression coverage for image-tool, shared loader, and sandbox helper paths.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: `media://inbound/example.png` can reach a workspace `path.resolve` branch, and Node produces the reported `/workspace/media:/inbound/example.png` shape.

## Evidence
