## Summary

Keep open. Current main still has source-level paths that can convert unresolved `media://inbound/...` references into malformed workspace or sandbox paths, and two open PRs already reference this issue with closing syntax.

## What This Changes

The issue reports that inbound `media://inbound/...` image references resolve to malformed workspace paths like `media:/inbound/...` instead of the managed media file.

## Best Possible Solution

Treat `media://` as managed media before any workspace or sandbox path join, then land or consolidate the open fixes with regression coverage for the shared loader, image tool, and sandbox bridge seams.

## Reproduction Assessment

Yes, at source level. Current main still allows `media://inbound/example.png` to flow into workspace/sandbox path resolution, and Node resolves that shape as `/workspace/media:/inbound/example.png`.

## Solution Assessment

Yes for triage. Keeping this issue open while maintainers review or consolidate the linked closing PRs is the narrowest maintainable path because the remaining work is already paired with open implementation candidates.
