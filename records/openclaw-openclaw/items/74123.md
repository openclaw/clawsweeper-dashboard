## Summary

Keep open. Current main still has scheme-blind resolver paths that can turn `media://inbound/...` into a workspace `media:/inbound/...` path, and open closing PRs already target the affected surfaces.

## What This Changes

The issue reports that `media://inbound/...` image references are incorrectly resolved as workspace-local `media:/inbound/...` paths instead of media-store files.

## Best Possible Solution

Land or consolidate a focused fix that treats `media://` as a media-store URI before workspace or sandbox bridge path resolution, with regression coverage for the shared loader and image-tool/sandbox entry points.

## Reproduction Assessment

Yes. Source inspection shows `media://inbound/example.png` can still reach workspace or sandbox bridge path resolution, and the Node path primitive produces the reported `/workspace/media:/inbound/example.png` shape.

## Solution Assessment

Yes for triage. Keeping this issue open while maintainers review or consolidate the open closing PRs is the narrowest maintainable path; the code fix should preserve existing local-root and inbound-media validation boundaries.
