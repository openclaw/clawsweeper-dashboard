## Summary

Keep open. Current main still contains resolver paths that can turn a `media://inbound/...` image reference into an invalid workspace `media:/inbound/...` path, and open closing PRs already target the affected surfaces.

## What This Changes

The issue reports that inbound image `media://` URIs are converted to invalid workspace `media:/` paths instead of resolving as OpenClaw-managed media files.

## Best Possible Solution

Treat `media://` as managed media before any workspace or sandbox path join, with focused regression coverage for the shared loader plus image-tool and sandbox entry points.

## Reproduction Assessment

Yes at source level. `media://inbound/example.png` can still flow to workspace or sandbox path resolution on current main, and the path primitive produces the reported `/workspace/media:/inbound/example.png` shape.

## Solution Assessment

Yes for triage. Keeping this issue open while maintainers review or consolidate the linked closing PRs is the narrowest maintainable path.
