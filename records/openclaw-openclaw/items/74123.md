## Summary

Keep open. Current main still has source paths that can collapse `media://` into a workspace `media:/...` path, and open closing PRs already target the affected resolver surfaces.

## What This Changes

Make `media://inbound/...` image references resolve to OpenClaw-managed inbound media instead of being joined under the workspace as `media:/...`.

## Best Possible Solution

Land or consolidate a focused fix that recognizes `media://` as a media-store URI before workspace or sandbox bridge path resolution, with regression coverage for the image tool, shared media loader, and sandbox helper paths.

## Reproduction Assessment

Yes. The source-level path is high-confidence: `media://inbound/example.png` can reach `path.resolve(workspaceDir, ...)`, and Node produces `/workspace/media:/inbound/example.png`, matching the reported failure shape.

## Solution Assessment

Yes for triage. Keeping this open while maintainers review or consolidate #74186 and #74278 is the narrowest maintainable path; the code fix should preserve local-root and inbound media validation boundaries.
