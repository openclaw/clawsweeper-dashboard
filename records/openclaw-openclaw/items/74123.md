## Summary

Keep open. Current main still contains the `media://` to `media:/` workspace-path corruption path, and two open implementation PRs already reference this issue with closing syntax, so the issue should remain open until maintainers land or consolidate the fix.

## What This Changes

Fix inbound `media://inbound/...` image references so they resolve to OpenClaw-managed media files instead of malformed workspace paths like `workspace/media:/inbound/...`.

## Best Possible Solution

Land or replace the linked implementation work with one focused fix that preserves inbound media ID validation, prevents scheme-bearing values from being joined under `workspaceDir`, handles sandbox bridge ordering before direct path resolution, and adds regression tests for image-tool, sandbox resolver, and shared media-loader paths.

## Reproduction Assessment

Yes. The source-level reproduction is high-confidence: `media://inbound/example.png` reaches path resolution paths that treat it as a relative string, and Node resolves that shape to `/workspace/media:/inbound/example.png`.

## Solution Assessment

Yes. Keeping this issue open while maintainers review or consolidate the linked implementation PRs is the narrowest current path; a separate ClawSweeper repair job would duplicate already-open work.
