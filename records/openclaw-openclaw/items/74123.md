## Summary

Keep open. Current main still contains resolver paths that can collapse `media://` into malformed workspace or sandbox filesystem paths, and two open closing PRs already target the affected surfaces.

## What This Changes

The issue asks for `media://inbound/...` image references to resolve to OpenClaw-managed inbound media instead of malformed workspace paths like `media:/inbound/...`.

## Best Possible Solution

Land or consolidate a focused fix that recognizes `media://` as a media-store URI before image-tool, shared loader, and sandbox bridge workspace resolution, with regression coverage for each affected boundary.

## Reproduction Assessment

Yes. The source-level reproduction is high-confidence: `path.resolve('/workspace','media://inbound/example.png')` produces `/workspace/media:/inbound/example.png`, and current main can pass `media://` into that primitive when `workspaceDir` or sandbox cwd resolution is active.

## Solution Assessment

Yes for triage. Keeping this issue open while maintainers review or consolidate #74186 and #74278 is the narrowest maintainable path; the code fix should preserve the local-root and inbound media validation boundaries.
