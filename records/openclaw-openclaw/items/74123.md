## Summary

Keep open. Current main still has source-level paths that can turn unresolved `media://inbound/...` references into malformed workspace or sandbox media paths, and live GitHub state shows open closing PRs at https://github.com/openclaw/openclaw/pull/74186 and https://github.com/openclaw/openclaw/pull/74278.

## What This Changes

`media://inbound/<image>` image references should resolve through OpenClaw-managed media storage instead of being treated as workspace-relative local paths.

## Best Possible Solution

Treat `media://` as managed media before any workspace or sandbox path join, then land or consolidate the open fixes with regression coverage for the shared loader, image tool, and sandbox bridge seams.

## Reproduction Assessment

Yes, at source level. Current main still allows `media://inbound/example.png` to reach workspace or sandbox path resolution, and a direct Node check produces the reported `/workspace/media:/inbound/example.png` shape.

## Solution Assessment

Yes for triage. Keeping this issue open while maintainers review or consolidate the linked closing PRs is the narrowest maintainable path because the remaining work is already paired with implementation candidates.
