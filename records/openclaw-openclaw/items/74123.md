## Summary

Keep open. Current main still has resolver paths that can turn a supported `media://inbound/...` image reference into a malformed workspace `media:/inbound/...` path, and open closing PRs already target the affected surfaces.

## What This Changes

The issue requests that `media://inbound/...` image references resolve to OpenClaw-managed inbound media files instead of malformed workspace-local `media:/inbound/...` paths.

## Best Possible Solution

Review and land or consolidate the open linked fixes so `media://` is handled as a media-store URI before any workspace or sandbox bridge path resolution, with regression coverage for the shared loader, image tool, and sandbox helper paths.

## Reproduction Assessment

Yes. Source inspection on current main shows `media://inbound/example.png` can reach workspace or sandbox bridge path resolution, and the Node path primitive produces the reported `/workspace/media:/inbound/example.png` shape.

## Solution Assessment

Yes for triage. Keeping this issue open while maintainers review or consolidate the open closing PRs is the narrowest maintainable path; the code fix should preserve existing local-root and inbound-media validation boundaries.
