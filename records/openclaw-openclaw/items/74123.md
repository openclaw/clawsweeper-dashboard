## Summary

Keep open. Current main still contains the reported `media://` URI corruption paths, and open closing PRs #74186 and #74278 already target this issue, so maintainers should land or consolidate those fixes before closing it.

## What This Changes

The issue reports that `media://inbound/...` image references can be corrupted into workspace-local `media:/inbound/...` paths before image processing.

## Best Possible Solution

Land or consolidate a focused fix that treats `media://` as a media-store URI before workspace or sandbox path resolution, while preserving inbound media validation and adding regression coverage for the image tool, `loadWebMedia`, and sandbox bridge paths.

## Reproduction Assessment

Yes. Current main can feed `media://inbound/example.png` into workspace path resolution, and Node produces `/workspace/media:/inbound/example.png`, matching the reported malformed local media path.

## Solution Assessment

Yes for triage. Keeping this issue open while maintainers review #74186 and #74278 is the narrowest maintainable path because those PRs already target the affected resolver surfaces.
