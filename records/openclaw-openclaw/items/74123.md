## Summary

Keep open. Current main still has source-level paths that can corrupt `media://inbound/...` image references into workspace `media:/inbound/...` paths, and open closing PRs #74186 and #74278 already target the affected resolver paths.

## What This Changes

The issue reports that `media://inbound/...` image references are treated as workspace-relative paths and become malformed `media:/inbound/...` local paths instead of resolving through managed media storage.

## Best Possible Solution

Treat `media://` as managed media before any workspace or sandbox path join, then land or consolidate the linked focused PRs with regression coverage for the shared loader plus image-tool and sandbox entry points.

## Reproduction Assessment

Yes, at source level. Current main still allows `media://inbound/example.png` to flow into workspace or sandbox path resolution, and the path primitive produces the reported `<workspace>/media:/inbound/example.png` shape.

## Solution Assessment

Yes for triage. Keeping this issue open while maintainers review or consolidate #74186 and #74278 is the narrowest maintainable path because those PRs already target the concrete resolver boundaries.
