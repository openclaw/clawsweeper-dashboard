## Summary

Keep open: this is the canonical active PR for a useful per-agent memory backend override request, but it is not merge-ready because the branch still targets the old bridge path, its agent QMD schema is out of sync with current root QMD config, it reopens a per-agent QMD paths surface that was closed as not planned, and external real behavior proof is missing.

## What This Changes

The PR adds `agents.list[].memory` for per-agent memory backend and QMD overrides, updates memory docs/schema/types, and adds schema/runtime resolver coverage while keeping citations global.

## Best Possible Solution

Land this PR or an equivalent refreshed patch after moving the resolver and tests into the package host SDK boundary, aligning schema/types/generated config/docs with the approved per-agent QMD surface, and adding redacted real behavior proof.

## Reproduction Assessment

Yes for the requested capability. Current main's strict agent schema lacks `memory`, and the active resolver reads backend/QMD settings only from root `cfg.memory`, so the proposed per-agent override path is source-reproducible as absent.

## Solution Assessment

No, not as written. The feature direction matches the linked requester, but the patch must update the package-owned resolver/config mirror and settle the per-agent QMD surface before merge.

  - body: Current main makes `src/memory-host-sdk/host/backend-config.ts` a bridge to `packages/memory-host-sdk/src/host/backend-config.ts`, but this patch puts the merge logic in the old path.   - body: This accepts `agents.list[].memory.qmd.paths`, but current docs route agent-scoped QMD collections through `memorySearch.qmd.extraCollections`, and https://github.com/openclaw/openclaw/issues/19005 closed the separate per-agent QMD path surface as not planned. Remove it or get an explicit product decision reopening that config shape.
  - confidence: 0.84
