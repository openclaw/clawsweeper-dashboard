## Summary

Keep this PR open: current main still lacks per-agent memory backend overrides, the linked requester confirmed this PR matches the needed rollout shape, and the closed feature issue now points here as the canonical implementation track. The branch is not merge-ready because it still targets the old host-SDK path, has schema-boundary problems, and has only test proof rather than real behavior proof.

## What This Changes

The PR adds `agents.list[].memory` for per-agent backend and QMD overrides, with docs, config schema/types, resolver logic, and regression/runtime tests while keeping citations global.

## Best Possible Solution

Refresh this PR or replace it with an equivalent patch that implements the override in the package host SDK resolver/config mirror, aligns the approved per-agent QMD schema boundary, updates docs/generated config surfaces, and adds focused schema/runtime coverage plus redacted real behavior proof.

## Reproduction Assessment

Yes by source inspection: current main's strict agent schema lacks `memory`, and the active resolver reads backend/QMD settings only from root `cfg.memory`. I did not run a live config command because this review is read-only.

## Solution Assessment

No, not as written. The feature direction is useful, but the patch needs to target the package-owned resolver/config mirror and either remove or get explicit approval for per-agent QMD path overrides.
