## Summary

Keep open: the PR targets a real missing capability and has positive requester context, but it is not merge-ready against current main because the runtime resolver now lives in the package implementation, generated schema surfaces are missing, the per-agent QMD schema is incomplete, and the user-facing feature lacks a changelog entry.

## What This Changes

The PR adds `agents.list[].memory` so individual agents can override the root memory backend and QMD settings while keeping citations global, with docs and regression/runtime tests.

## Best Possible Solution

Land this PR or an equivalent focused patch after porting the override to the package-owned host SDK resolver/config mirror, adding the source and generated config surfaces, mirroring the full root QMD config shape except global-only citations, documenting the supported boundary, adding the changelog entry, and keeping targeted schema/runtime regressions for backend selection, QMD overlay behavior, agent isolation, and citations rejection.

## Reproduction Assessment

Yes. Current main can be verified by static source inspection: the strict agent schema lacks `memory`, the agent type lacks a memory backend override, and the runtime resolver reads only root `cfg.memory`; a focused regression should accept `agents.list[].memory` and resolve QMD only for the selected agent.

## Solution Assessment

No, not as written. The product direction is the narrow maintainable fix, but the implementation needs to move to the package-owned resolver/config mirror, regenerate schema output, include the missing QMD update fields, and add the required changelog entry before merge.
