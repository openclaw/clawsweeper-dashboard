Keep open: current main still lacks the requested per-agent memory backend override, but this PR is not mergeable as written because it targets the old resolver path, leaves the package-owned config/QMD schema boundary misaligned, and lacks real behavior proof.

## What This Changes

The PR adds per-agent `agents.list[].memory` backend/QMD overrides, docs, config schema/types, and regression/runtime tests while keeping `memory.citations` global.

## Best Possible Solution

Land this PR or an equivalent refreshed patch after moving the override into the package host SDK resolver/config mirror, explicitly constraining or approving the per-agent QMD subfield boundary, aligning docs/schema/tests, and adding redacted real runtime proof.

## Solution Assessment
