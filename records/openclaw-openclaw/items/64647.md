## Summary

Keep open: current main still lacks the config/runtime surface, and the requester confirmed this PR matches the staged-rollout need, but the patch is not merge-ready because it updates the old bridge path, omits supported QMD update fields from the per-agent schema, and lacks after-fix real behavior proof.

## What This Changes

The PR adds `agents.list[].memory` backend/QMD overrides, docs, schema/type changes, and schema/runtime tests while keeping `memory.citations` global.

## Best Possible Solution

Land this PR or an equivalent patch after moving the override into the package host SDK resolver/config mirror, mirroring the full root QMD update schema, keeping docs/schema surfaces aligned, and adding real terminal or log proof for a per-agent backend override.

## Reproduction Assessment

Yes. Source inspection on current main is high-confidence: the strict agent schema has no `memory` key and the active package resolver reads only root `params.cfg.memory`, so `agents.list[].memory` remains outside accepted/runtime behavior.

## Solution Assessment

No, not as written. The requested config shape is the narrow maintainable direction, but the branch must be refreshed to the package-owned resolver/config mirror and the per-agent QMD schema must match root-supported update controls before merge.
