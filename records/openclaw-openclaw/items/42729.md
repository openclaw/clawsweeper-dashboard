## What This Changes

The PR adds a cron timer guard that skips stale due slots whose last run is later than the scheduled slot, plus a restart catch-up regression test.

## Best Possible Solution

Port the strict stale-slot guard into current cron timer code, preserve same-millisecond retry behavior, keep focused restart coverage, and land after redacted restart proof plus fresh gates.

## Reproduction Assessment

Yes, by source inspection. On current main, startup calls runMissedJobs before the expired-slot maintenance repair, and isRunnableJob selects a past-due nextRunAtMs without first checking whether lastRunAtMs already passed that slot.

## Solution Assessment

Yes, after rebase and proof. Guarding the runnable-selection hot path with a strict greater-than check is the narrowest maintainable fix for the remaining stale due-slot case while preserving same-millisecond retry behavior.
