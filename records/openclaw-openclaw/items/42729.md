## What This Changes

The PR adds a strict `lastRunAtMs > nextRunAtMs` guard in cron timer runnable selection and a restart catch-up regression test for already-executed stale due slots.

## Best Possible Solution

## Solution Assessment

Yes, with rebase and proof. Guarding `isRunnableJob` is the narrowest maintainable fix for the remaining stale due-slot path, and the strict `>` comparison avoids the same-millisecond retry regression raised in prior review.
