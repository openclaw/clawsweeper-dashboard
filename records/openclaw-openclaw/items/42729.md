## What This Changes

The PR adds a strict `lastRunAtMs > nextRunAtMs` guard to cron runnable selection plus a restart catch-up regression test for already-executed stale slots.

## Best Possible Solution

Port the strict stale due-slot guard into current cron timer code, preserve same-millisecond retry behavior, keep focused stale-slot regression coverage, and land after proof plus rebased validation.

## Reproduction Assessment

Yes, by source inspection. On current main, startup calls `runMissedJobs` before maintenance repair, and `isRunnableJob` returns true for a due `nextRunAtMs` without first checking whether `lastRunAtMs` already passed that slot.

## Solution Assessment

Yes, after rebase and proof. Guarding `isRunnableJob` is the narrowest maintainable fix for the remaining stale past-due slot path, and the strict `>` comparison preserves same-millisecond retry behavior.
