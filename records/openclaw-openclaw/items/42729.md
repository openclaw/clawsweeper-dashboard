## Summary

Keep open. Current main and the latest release still select stale due cron slots before the maintenance repair path can advance them, while this PR targets that remaining restart duplicate-execution gap and appears to have addressed the prior same-timestamp retry review concern.

## What This Changes

This PR adds a stale scheduled-slot guard to cron runnable selection and a restart catch-up regression test for already-executed due slots.

## Best Possible Solution

Land the stale-slot guard in the runnable selection path with the regression test, preserving missed-slot catch-up and zero-backoff retry behavior while preventing already-executed persisted slots from being selected again on restart.

## Reproduction Assessment

Yes. The linked report and PR discussion provide concrete gateway-restart reproduction, and source inspection shows current main will collect a persisted stale due `nextRunAtMs` before the later maintenance recompute can advance it.

## Solution Assessment

Yes. Guarding `isRunnableJob` is the narrowest maintainable fix because it prevents duplicate selection before execution, while the strict `>` update keeps same-timestamp retry slots from being suppressed.
