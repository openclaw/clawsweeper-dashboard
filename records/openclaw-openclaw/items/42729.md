## What This Changes

This PR adds a strict `lastRunAtMs > nextRunAtMs` stale-slot guard to cron runnable selection and adds a restart catch-up regression test for an already-executed stale due slot.

## Best Possible Solution

Land the stale-slot guard in `isRunnableJob` with the regression test and a `CHANGELOG.md` Fixes entry, preserving the strict `>` behavior so same-timestamp retry slots remain runnable while already-executed stale due slots are skipped and repaired.
