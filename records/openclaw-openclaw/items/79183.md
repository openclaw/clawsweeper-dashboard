## Summary

Keep open: current main still lacks the requested Feishu direct-session account-name fallback, and https://github.com/openclaw/openclaw/pull/79251 is an open closing PR for the same behavior, so this issue should wait for that PR to land or be closed.

## What This Changes

Use configured Feishu channel account names as the `sessions_list` display fallback for direct multi-bot sessions instead of opaque `open_id` labels.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/79251 if it still preserves explicit display names, labels, group titles, meaningful sender labels, and search behavior against current main; then close this issue as implemented.

## Reproduction Assessment

Yes, as a source-level path: Feishu direct inbound context records `AccountId` without a direct `ConversationLabel`, while Gateway session rows fall through to `origin.label` and `sessions_list` passes the display name through. I did not run a live Feishu account.

## Solution Assessment

Yes. A Gateway-level account-name fallback shared by row display and search is the narrow maintainable path, provided explicit display names, labels, group titles, and meaningful origin labels keep precedence.
