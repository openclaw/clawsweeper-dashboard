## What This Changes

The issue requests that direct Feishu session rows prefer the configured channel account name over opaque open_id values when building sessions.list display names.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/96282 so the shared gateway sessions.list row and search projection reuse the existing account name contract while preserving stronger labels.

## Reproduction Assessment

Yes. At source level, current main returns origin.label for direct session rows/search when no explicit displayName or label exists, so a Feishu open_id remains visible despite a configured account name.

## Solution Assessment

Yes. The best direction is a shared gateway sessions.list display/search fallback using the existing channel account name field; a new displayLabel config surface or UI-only workaround would duplicate state and leave RPC search inconsistent.
