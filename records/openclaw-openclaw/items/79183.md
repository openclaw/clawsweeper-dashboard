## Summary

Keep open: current main and the latest release still fall back to origin.label for direct session rows/search, while https://github.com/openclaw/openclaw/pull/94359 is the open candidate fix and uses closing syntax for this issue.

## What This Changes

The issue requests using configured Feishu account names as the sessions.list display fallback for direct multi-bot sessions instead of opaque open_id values.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/94359 with a shared gateway row/search fallback that reuses channels.<channel>.accounts.<account>.name, respects defaultAccount/accounts.default, and preserves explicit displayName, labels, group names, and meaningful origin labels.

## Reproduction Assessment

Yes. At source level, a direct Feishu-style session whose origin.label is an opaque open_id and whose channel account has a configured name will still return origin.label because current main does not look up the account name.

## Solution Assessment

Yes, the best direction is the shared gateway session row/search fallback using the existing account name contract. A new displayLabel config surface or Feishu-only UI workaround would duplicate configuration and leave sessions.list/search inconsistent.
