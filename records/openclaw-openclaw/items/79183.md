## Summary

Keep open: current main still lacks the configured account-name fallback for Feishu direct session display, and the linked open PR explicitly closes this issue with a focused implementation, so this issue should remain open until that PR lands or is replaced.

## What This Changes

Request `sessions_list` direct Feishu sessions to show the configured account name instead of an opaque `open_id` when no explicit display name or label exists.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/79251, or an equivalent narrow Gateway session row/search fix that uses configured account names only when explicit display names, labels, group names, and meaningful origin labels are absent.

## Reproduction Assessment

Yes, as a source-level reproduction: current main can carry Feishu account metadata for direct sessions, but session row display/search does not derive `displayName` from configured account names. I did not live-run Feishu in this read-only review.

## Solution Assessment

Yes; resolving the fallback in Gateway session row/search helpers is the narrow maintainable path, and the linked open PR follows that shape while preserving existing precedence.
