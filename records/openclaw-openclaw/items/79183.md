## Summary

Keep this open: current main still lacks the requested account-name fallback for Feishu direct session rows, and an open linked fix PR already targets the issue with closing syntax.

## What This Changes

Request that Feishu multi-bot direct sessions in `sessions_list` prefer configured account names over opaque `open_id` values while preserving labels and falling back to the current value.

## Best Possible Solution

Review and land the linked PR if it preserves the current display precedence while adding a structured account-name fallback for direct channel-account sessions.

## Reproduction Assessment

Yes, source-level: current main shows Feishu direct sessions carry account metadata but no direct conversation label, and session row display falls back to `origin.label` without account-name lookup. I did not run a live Feishu tenant.

## Solution Assessment

Yes, the linked PR direction is the narrow maintainable fix: derive from configured channel account metadata and keep explicit display names, labels, group names, and meaningful origin labels ahead of the fallback.
