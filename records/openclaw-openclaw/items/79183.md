## Summary

Keep open: current main does not derive a Feishu direct-session display name from configured account names, so the requested multi-bot session-list UX remains unimplemented.

## What This Changes

Request that `sessions_list` prefer configured Feishu account names for direct multi-bot session display names while preserving labels and open_id fallback.

## Best Possible Solution

Add a narrow account-name display derivation for direct channel-account sessions using structured account metadata, without overriding explicit display names, labels, or group titles.

## Reproduction Assessment

Yes, as a source-level reproduction: Feishu direct inbound context records `AccountId` but no direct `ConversationLabel`, and `sessions.list` only returns existing display metadata. I did not run a live Feishu account in this read-only review.

## Solution Assessment

Yes; deriving from stored account metadata and configured account `name` is narrower and more maintainable than parsing an open_id out of the session key. Preserve the existing explicit displayName, label, and group-title precedence.
