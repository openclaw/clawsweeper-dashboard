## Summary

Keep open: current main and latest release still derive direct-session displayName from explicit metadata, labels, or origin.label, and the earlier fix PR was closed unmerged. The request remains a small, aligned gateway/session-list improvement rather than conservative cleanup to close.

## What This Changes

Use configured channel account names ahead of opaque Feishu open_id values when deriving sessions_list display names for direct sessions.

## Best Possible Solution

Add the fallback in the shared gateway row and search path using existing channels.<channel>.accounts.<account>.name, while preserving explicit displayName, session labels, group names, and meaningful origin labels.

## Reproduction Assessment

Yes, at source level: seed a direct Feishu SessionEntry with origin.label equal to an open_id, origin.accountId set, and cfg.channels.feishu.accounts.<id>.name configured; current buildGatewaySessionRow returns origin.label because there is no account-name lookup. I did not execute a live Feishu tenant in this read-only pass.

## Solution Assessment

Yes, the best direction is the narrow shared gateway row/search fallback using the existing account name contract. Adding a new displayLabel key or solving only in Feishu would duplicate config or leave Control UI and sessions_list inconsistent.
