## Summary

Keep open: current main still has a CLI compaction/persistence path that resolves the default context engine before registering the built-in `legacy` engine, matching the reported error path.

## What This Changes

The issue asks OpenClaw to persist CLI agent transcripts without failing on an unregistered default `legacy` context engine.

## Best Possible Solution

Patch the CLI compaction lifecycle to initialize built-in context engines before resolving the active slot, add regression coverage for the default `legacy` path, and include a user-facing changelog fix entry.

## Reproduction Assessment

Yes, at source level: a CLI-backed turn with a persisted transcript enters `runCliTurnCompactionLifecycle`, which calls `resolveContextEngine` while the registry can still be empty. I did not run the live CLI command in this read-only review.

## Solution Assessment

Yes; the narrowest maintainable fix is to call the existing `ensureContextEnginesInitialized()` helper before CLI compaction resolves the context engine, mirroring embedded runner and subagent paths without changing plugin slot defaults or resolver semantics.
