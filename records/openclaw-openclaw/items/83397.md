## Summary

Keep open: current main still has the reported source-level regression path, and the right fix needs maintainer judgment because ACP visibility overlaps with deliberate configured-agent session scoping.

## What This Changes

ACP-spawned sessions from dynamically keyed agents should remain visible in the Control UI without undoing configured-agent session isolation.

## Best Possible Solution

Add an explicit ACP visibility contract that keeps dynamically spawned ACP sessions discoverable in Control UI while preserving configured-agent data isolation, with focused backend and UI regression coverage.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: Control UI sends configuredAgentsOnly true, store discovery skips disk-only ACP agent stores, and key filtering excludes agent:claude:acp:* when claude is not configured. I did not run a live UI repro in this read-only review.

## Solution Assessment

Unclear: the report points at the right code path, but an unconditional ACP bypass is not obviously the best fix because it may conflict with deliberate data-scope hardening. The maintainable solution should define which ACP sessions are safe to surface and test both visibility and isolation.
