## Label Justifications

- `P1`: The report describes a regression hiding active ACP-spawned sessions from the Control UI, which breaks an agent/session workflow for real users.
- `impact:session-state`: The affected behavior is session-list visibility and lineage filtering for ACP/acpx and spawned runtime sessions.

## Summary

Current main already fixes the reported Control UI session-list regression: ACP/acpx and runtime child sessions are preserved in configured-only session lists when their owner or parent session belongs to a configured agent, with focused regression coverage. The fix is on main only and is not contained in the latest release tag provided.

## What This Changes

The issue asks for ACP/acpx-spawned sessions to stay visible in Control UI session lists when `configuredAgentsOnly` is enabled.

## Best Possible Solution

Keep the current-main session-list fix and ship it in the next release so configured-only Control UI lists include ACP/acpx and runtime child sessions with configured ownership.

## Reproduction Assessment

Yes from source and history: the reported beta path filtered configured-only session lists too narrowly, and current main now has regression coverage for ACP and spawned child visibility. I did not run a live Gateway because this review was read-only.

## Solution Assessment

Yes. The current fix addresses the narrow session-store discovery/filtering bug by preserving configured ACP agents and parent-owned spawned sessions, avoiding a new UI toggle or config option.
