## Summary

Close as implemented: current main now preserves configured ACP harness stores and parent-owned spawned sessions in configured-only session listings, with regression coverage for the reported `agent:claude:acp:*` shape. The fix is present on current main only; no tag contains it yet.

## What This Changes

The issue reports that ACP-spawned sessions disappear from the Control UI session list when `sessions.list` uses the default `configuredAgentsOnly` filtering after the agent-scoped gateway change.

## Best Possible Solution

Keep the current main implementation and ship a release containing commit 8483d0337530f11259559f29bc8940a786ffda8c so affected beta users receive the restored configured-only ACP session visibility.

## Reproduction Assessment

Yes, at source level. The current regression test creates the reported `agent:claude:acp:*` store shape with only `main` in `agents.list` and verifies `sessions.list` with `configuredAgentsOnly: true` includes it on current main.

## Solution Assessment

Yes. The main-branch fix is the narrow maintainable path: load the combined store broadly enough not to drop ACP stores, then filter by configured ACP agents and configured parentage rather than adding a new UI mode.
