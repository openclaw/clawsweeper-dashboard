## Summary

Keep open: current main still has the Control UI default and backend configured-agent filters that can hide dynamically keyed ACP sessions, and the fix needs security-aware product judgment because the related scoping work deliberately narrowed session data visibility.

## What This Changes

Restore Control UI visibility for ACP-spawned sessions when session listing is scoped to configured agents only.

## Best Possible Solution

Add an explicit ACP visibility path that preserves configured-agent data isolation, with focused backend and Control UI regression coverage for dynamic ACP sessions.

## Reproduction Assessment

Yes, from source inspection: the Control UI sends `configuredAgentsOnly: true`, and current backend discovery plus key filtering excludes `agent:claude:acp:*` when `claude` is not configured. I did not run a live UI repro in this read-only sweep.

## Solution Assessment

Unclear: the report identifies the right code path, but an unconditional ACP exception may conflict with the scoping hardening. The best fix should deliberately define which ACP sessions are safe to surface and prove both visibility and data-scope behavior.
