## Summary

Keep open: the timeout boundary is the right narrow direction for the linked regression, but this PR still has a correctness bug in its timeout warning detection and is missing the required user-facing changelog entry.

## What This Changes

The PR wraps provider usage auth resolution in `withTimeout`, emits a warning when it infers an auth timeout, and adds a unit test for a never-settling auth resolver.

## Best Possible Solution

Land the auth-resolution timeout with an explicit timeout sentinel or boolean, keep normal empty-auth results quiet, cover both timeout and no-auth paths, and add the active-release changelog entry.

## Reproduction Assessment

Yes. The linked report gives a concrete `spawnSync` non-TTY reproduction for the original hang, current main shows the unbounded auth await, and the PR-introduced warning bug is reproducible by resolving auths to `[]` normally and observing `console.warn`.

## Solution Assessment

No. The timeout boundary is the narrow maintainable fix, but the diagnostic must track the timeout outcome directly instead of inferring it after the promise has resolved.
