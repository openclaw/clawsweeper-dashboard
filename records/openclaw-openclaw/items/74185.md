## Summary

Keep open: the auth-resolution timeout is useful and source-backed, but the current branch also disables OAuth refresh for status usage, which can make expired-but-refreshable OAuth-backed usage providers disappear after upgrade.

## What This Changes

The branch wraps provider usage auth resolution in a timeout, adds focused tests, preserves scoped status usage lookup, and changes status usage auth to skip credential-less plugin auth while disabling OAuth refresh.

## Best Possible Solution

Land the auth-resolution timeout while preserving default OAuth refresh for status usage, unless provider/auth owners explicitly approve cached-only status usage with upgrade coverage.

## Solution Assessment

No: wrapping auth resolution in the existing timeout is the right layer, but suppressing OAuth refresh for status usage is broader than needed and can remove existing OAuth usage visibility.
