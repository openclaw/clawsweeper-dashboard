## Summary

Keep open: the timeout wrapper addresses a real current-main gap, but the branch also suppresses OAuth refresh for status usage, which can make expired-but-refreshable OAuth usage providers disappear after upgrade.

## What This Changes

The PR wraps provider usage auth resolution in a timeout for `status --usage --json`, adds timeout tests, changes status usage auth options, and adds cached-only OAuth behavior tests.

## Best Possible Solution

Land the auth-resolution timeout while preserving default OAuth refresh behavior for status usage unless provider/auth owners explicitly approve and test cached-only status usage.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes at source level: current main has no timeout around `resolveProviderAuths`, and the linked report gives a non-TTY subprocess command, though I did not run a live current-main hang reproduction in this read-only review.

## Solution Assessment

Is this the best way to solve the issue? No: wrapping auth resolution in the existing timeout is the right layer, but suppressing OAuth refresh for status usage is broader than needed and can remove existing OAuth usage visibility.
