## Summary

Keep open: the auth-resolution timeout is the right fix for the non-TTY status hang, but the branch also makes status usage skip expired OAuth refreshes, which can hide OpenAI/Codex usage data after upgrade.

## What This Changes

The PR wraps provider usage auth resolution in a timeout, adds regression tests, preserves scoped agent lookup, and changes status usage auth to skip credential-less plugin auth while disabling OAuth refresh.

## Best Possible Solution

Land the auth-resolution timeout while preserving default OAuth refresh behavior for status usage, unless provider/auth maintainers explicitly approve and test cached-only status usage.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes at source level: current main has no timeout around resolveProviderAuths and the linked issue gives a non-TTY subprocess command, though I did not run a live current-main hang reproduction in this checkout.

## Solution Assessment

Is this the best way to solve the issue? No: wrapping auth resolution in the existing timeout is the right layer, but suppressing OAuth refresh for status usage is broader than needed and can remove existing OAuth usage visibility.
