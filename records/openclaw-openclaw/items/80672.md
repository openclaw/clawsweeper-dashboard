## Summary

Current main already covers the reported failure: isolated cron announce delivery resolves explicit Feishu targets and sends final text through the real outbound adapter, with delivered state derived from the send result. The proof is current-main-only and not yet in the latest tagged release.

## What This Changes

The issue asks for isolated cron jobs using announce delivery to deliver their final Markdown output to the configured Feishu target instead of recording not-delivered.

## Best Possible Solution

Keep the current-main implementation and ship it in the next release; users on v2026.5.7 can use the direct message tool workaround until that release is available.

## Reproduction Assessment

No live current-main reproduction was run. Source inspection shows the reported path no longer follows the failing behavior: explicit isolated announce delivery resolves the Feishu target, sends through the outbound adapter, and records delivered from that result.

## Solution Assessment

Yes. The current implementation is the narrow maintainable solution because announce remains runner-owned, target resolution uses the existing delivery contract, and final output goes through the channel outbound adapter instead of a separate workaround path.
