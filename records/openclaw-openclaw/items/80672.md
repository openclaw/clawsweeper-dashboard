Keep open: this is a concrete delivery bug report, current main still builds isolated cron direct-delivery session context from the transient agent session key, and https://github.com/openclaw/openclaw/pull/81280 is an open closing PR for the same problem. ## What This Changes

The issue reports that isolated cron jobs using announce delivery can execute successfully but finish with not-delivered for Feishu targets.

## Best Possible Solution

Keep this issue open while maintainers review the paired fix PR; after it merges, close this issue with the merge commit and any remaining Feishu-specific evidence tracked separately.

## Reproduction Assessment

No high-confidence live reproduction was established because Feishu credentials were not available. Source inspection does show the current-main path that can reproduce the reported mismatch: isolated cron delivery uses a transient agent session key for outbound direct delivery while the stable announce path uses resolveCronNotificationSessionKey.

## Solution Assessment

The best direction is to fix the implicated session context construction rather than document the message-tool workaround. The linked PR appears scoped to that path, but it still needs normal PR review and stronger behavior proof before merge.
