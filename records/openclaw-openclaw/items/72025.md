Land a narrow Signal plugin fix that reuses the shared controller, adds remove/restore terminal behavior consistent with supported channels, and documents Signal as a supported status-reaction channel.

## Reproduction Assessment

Yes. Current main has a high-confidence static reproduction: a Signal inbound turn reaches dispatchInboundMessage without status-reaction callbacks, while the PR cleanup bug follows directly from the setDone/setError-only terminal path after a visible dispatch.

## Solution Assessment

No, not in the current PR form. The implementation direction is right, but terminal cleanup must honor removeAckAfterReply and the behavior needs docs/changelog updates before this is the narrowest maintainable fix.
