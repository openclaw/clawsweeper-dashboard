## Summary

Keep open. Current main still lacks Signal inbound status-reaction lifecycle wiring, so the PR is addressing a real gap, but the latest head should not merge yet because terminal Signal reactions are left stale and the user-facing config docs/changelog were not updated.

## What This Changes

The PR wires Signal inbound messages into the shared status-reaction controller and adds Signal regression coverage for direct, group, and no-final-reply status-reaction paths.

## Best Possible Solution

Land a narrow Signal plugin fix that reuses the shared status-reaction controller, routes direct and group reactions through existing Signal helpers, preserves mention gating and no-final restoration, implements terminal clear/restore behavior consistent with Slack, Discord, and Telegram, and updates the changelog plus public config docs.

Yes. The implementation direction is right and the latest head addresses the earlier no-final-reply and mention-gating review comments, but finalization still needs to honor the shared cleanup contract and the user-facing docs/changelog need to move with the behavior change.
