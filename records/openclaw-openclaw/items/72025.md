## Summary

Keep open. Current main still lacks Signal inbound status-reaction lifecycle wiring, and this PR is useful, but the latest head still has a correctness blocker around terminal cleanup plus missing user-facing docs/changelog updates.

## What This Changes

The PR adds Signal inbound status-reaction controller wiring and a new Signal status-reaction regression test for direct, group, and no-final-reply dispatch cases.

## Best Possible Solution

Land a narrow Signal plugin fix that reuses the shared status-reaction controller, routes direct and group reactions through existing Signal helpers, preserves mention gating and no-final restoration, implements terminal clear/restore behavior consistent with Slack/Discord/Telegram, and updates the changelog plus user-facing config docs.

Yes. ## Solution Assessment

No, not in the current PR form. The implementation direction is right and the latest head addresses the earlier no-final-reply race, but terminal cleanup still needs to honor removeAckAfterReply/restore semantics and the user-facing docs/changelog need to move with the behavior change.
