## Summary

Keep open. Current main still normalizes and hooks outbound reply payloads, but it does not provide the requested shared fail-closed validator for structured operator-facing payloads before dispatcher delivery or direct outbound delivery; merged PR #70118 only added the adjacent text-oriented `message_sending` hook seam.

## What This Changes

The issue asks for a universal pre-send structured-output validation gate at native reply-dispatch and outbound delivery boundaries.

## Best Possible Solution

Define a shared `ReplyPayload`/presentation validation contract, enforce it at the last common pre-send boundaries for dispatcher replies and direct outbound delivery, and document the reject-versus-degrade rules.

## Reproduction Assessment

Yes. A source-level unit check can send malformed structured fields through a `ReplyDispatcher` delivery spy or `deliverOutboundPayloads` fake channel and observe current main normalize/hooks before delivery rather than rejecting with one shared validator.

## Solution Assessment

Unclear. The requested dispatcher seam is useful but too narrow by itself; the maintainable fix needs one shared validator contract that covers both dispatcher and direct outbound paths after maintainer agreement on failure semantics.
