## Summary

Keep open. Current main added a useful `beforeDeliver`/`message_sending` seam, but it still does not provide the universal fail-closed structured-output validation gate requested here: dispatcher delivery can still call `options.deliver(...)` after normalization and an optional hook, the default inbound hook only handles text content, and direct outbound paths such as cron/CLI delivery use `deliverOutboundPayloads` outside the dispatcher seam.

## What This Changes

Request a universal pre-send validator that fail-closes invalid structured operator-facing reply payloads before native reply/outbound delivery while preserving simple text replies and existing hooks.

## Best Possible Solution

Define a shared structured reply-payload validation contract and enforce it at the last common pre-send boundaries: the reply dispatcher for native replies and the outbound delivery path for direct/cron/message-tool sends. Keep `message_sending` as a transform/cancel hook, but make malformed structured operator-facing payloads fail closed before channel adapters run.

## Reproduction Assessment

Yes at the unit/source level: construct a `ReplyDispatcher` with a delivery spy and send a payload with malformed structured fields; current code has no built-in validator before `options.deliver`. A live channel reproduction is not required to prove this missing seam, though it would help choose the final contract.

## Solution Assessment

No. The current before-deliver hook work is useful and should stay, but it is not the best complete solution for this issue because it is optional/text-hook based and does not cover direct outbound delivery paths; a shared validation helper with dispatcher and outbound integration is safer.
