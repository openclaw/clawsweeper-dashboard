## Summary

Keep open. Current main still has normalization and `message_sending` hook seams, including the merged `beforeDeliver` path, but it does not provide the requested shared fail-closed validator for structured operator-facing payloads before dispatcher and direct outbound delivery.

## What This Changes

The issue requests a shared fail-closed validation gate for structured operator-facing replies before native reply dispatch and outbound delivery.

## Best Possible Solution

Define a shared structured reply-payload validation contract, enforce it at the last common pre-send boundaries for dispatcher replies and direct outbound delivery, and cover it with contract tests plus docs.

## Reproduction Assessment

Yes. Source inspection shows a high-confidence reproduction path: a `ReplyDispatcher` delivery spy or `deliverOutboundPayloads` call can receive malformed structured fields and reach normalization/hooks without a canonical fail-closed validator.

## Solution Assessment

Unclear. A dispatcher-only patch is too narrow because direct outbound delivery bypasses that seam; the maintainable solution needs a shared contract that preserves documented degradation where intended.
