## Summary

Keep open. Current main has a useful `beforeDeliver` and `message_sending` hook path, but it still lacks the requested universal fail-closed structured-payload validation before dispatcher delivery and direct outbound delivery.

## What This Changes

Request a shared pre-send validator that blocks malformed structured operator-facing reply payloads across native dispatcher and direct outbound delivery paths.

## Best Possible Solution

Define a shared structured reply-payload validation contract, enforce it at the last common pre-send boundaries for dispatcher replies and direct outbound delivery, and cover it with contract tests plus docs.

## Reproduction Assessment

Yes. At the source/unit level, a `ReplyDispatcher` delivery spy or `deliverOutboundPayloads` call can receive malformed structured fields and show current main only normalizes/hooks before delivery.

## Solution Assessment

Unclear. The current hook seam is useful but not sufficient; a dispatcher-only patch would miss direct outbound paths, while a shared validator needs maintainer agreement on schema and degradation boundaries.
