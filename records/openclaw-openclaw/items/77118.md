## Summary

Keep open: current main still lacks an exact `agentId` filter for `cron.list`, and the requested default session scoping is a new multi-agent visibility policy rather than an already-defined bug fix.

## What This Changes

Add `agentId` filtering to `cron list` across the tool, CLI, gateway protocol, server handler, and cron service, with agent-session calls scoped to the calling agent by default.

## Best Possible Solution

Add an explicit server-side `agentId` filter through the protocol, service, CLI, and tool, then deliberately choose the default agent-session visibility while preserving a clear admin/all-jobs path.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection on current main is high confidence that `cron.list` cannot accept or apply an exact `agentId` filter today.

## Solution Assessment

Unclear until maintainers choose the default visibility policy. An optional exact filter is narrow, but automatically scoping agent-session list calls changes multi-agent and admin behavior.
