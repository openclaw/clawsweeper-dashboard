## Summary

Keep open: current main still lacks exact `agentId` filtering for `cron.list`, and two open closing PRs are active implementation candidates rather than grounds to close this issue now.

## What This Changes

The issue requests `cron list` support for filtering jobs by `agentId`, including a CLI `--agent` flag and agent-session default scoping in multi-agent setups.

## Best Possible Solution

Add an additive exact `agentId` filter through the cron service, gateway protocol, handler, CLI, and tool surfaces, with an explicit maintainer choice on whether agent-session list calls auto-scope or preserve all-jobs visibility by default.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection on current main gives high confidence that `cron.list` cannot currently accept or apply an exact `agentId` filter.

## Solution Assessment

Unclear until maintainers choose the default visibility policy. An optional exact server-side filter is narrow, but automatic agent-session scoping changes multi-agent and admin behavior.
