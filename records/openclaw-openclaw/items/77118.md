## Summary

Keep open: current main still has no exact `agentId` filter for `cron.list`, and two open PRs already reference this issue with closing syntax, so the issue should remain open until one implementation lands or maintainers choose a different policy.

## What This Changes

The issue requests `agentId` filtering for `cron list` across the cron tool, gateway protocol, cron service, and CLI in multi-agent setups.

## Best Possible Solution

Review and consolidate one linked implementation that adds an additive exact `agentId` filter through the protocol/service path, then deliberately choose whether tool and CLI defaults should auto-scope or remain explicit.

## Reproduction Assessment

Yes for the reported capability gap: source inspection on current main shows `cron.list` cannot accept or apply an exact `agentId` filter. This is not a strict bug reproduction because the request adds a new filter, CLI flag, and default visibility behavior.

## Solution Assessment

Unclear until maintainers choose the default visibility policy. An optional exact filter is a narrow additive contract change, but automatic agent-session scoping changes multi-agent and admin behavior.
