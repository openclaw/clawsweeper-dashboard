## Summary

Keep open: current main still lacks exact `agentId` filtering for `cron.list`, and two open PRs already reference this issue with closing syntax, so the issue should remain open until one implementation is reviewed and merged.

## What This Changes

Request optional `agentId` filtering for `cron list`, including protocol/service support, CLI `--agent`, and possible agent-session default scoping.

## Best Possible Solution

Review and land one canonical additive implementation that adds an exact server-side `agentId` filter, then deliberately chooses whether tool calls should auto-scope by session while keeping CLI all-jobs behavior explicit.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature request. Source inspection on current main does give high confidence that `cron.list` cannot currently accept or apply an exact `agentId` filter.

## Solution Assessment

Unclear until maintainers choose the default visibility policy. An optional exact filter is narrow and additive, but automatic agent-session scoping changes multi-agent/admin behavior and should be decided intentionally.
