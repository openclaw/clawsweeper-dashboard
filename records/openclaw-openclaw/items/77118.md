## Summary

Keep open: current main still lacks an exact `agentId` filter for `cron.list`, and two open closing PRs are already implementation candidates rather than evidence that this issue can be closed.

## What This Changes

Request support for scoping `cron list` results by `agentId` in multi-agent setups, including protocol/tool/CLI support and possible session-default filtering.

## Best Possible Solution

Land one focused implementation that adds an exact server-side `agentId` filter and deliberately defines the CLI/tool default visibility policy, then close or supersede the overlapping PR.

## Reproduction Assessment

Yes for the missing capability by source inspection, but not as a broken documented-contract bug. Current main has no `agentId` field in the `cron.list` protocol, tool call, CLI list command, or service list options.

## Solution Assessment

Unclear until maintainers choose the default visibility policy. An optional exact filter is narrow and additive, but automatic agent-session scoping and related mutation controls are product behavior decisions.
