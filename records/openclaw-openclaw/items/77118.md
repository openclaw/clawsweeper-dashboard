## Summary

Keep open: current main still lacks an exact `agentId` filter for `cron.list`, but the requested default agent-session filtering is a new feature and policy change rather than a broken documented contract.

## What This Changes

Request adds `agentId` filtering to `cron list` in multi-agent setups, including a CLI flag and agent-session default filtering for the cron tool.

## Best Possible Solution

Add an explicit server-side `agentId` filter through the cron protocol, service, CLI, and tool, then deliberately decide whether agent-session calls should default to scoped results or require an explicit filter.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection on current main is high confidence that `cron.list` cannot accept or apply an exact `agentId` filter today.

## Solution Assessment

Unclear until maintainers choose the default visibility policy. An optional exact filter is narrow, but automatically scoping agent-session list calls changes multi-agent/admin behavior.
