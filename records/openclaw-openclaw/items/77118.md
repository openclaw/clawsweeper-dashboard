## Summary

Keep open: current main still lacks an exact `agentId` filter for `cron.list`, and two open PRs already reference this issue with closing syntax, so the issue should wait for maintainer review/merge of the linked implementation work.

## What This Changes

Request `cron list` to accept an `agentId` filter, auto-scope agent-tool calls, and expose a CLI `--agent` flag while preserving the all-jobs CLI default.

## Best Possible Solution

Land or replace one canonical implementation that adds an explicit exact `agentId` filter through the protocol and service, then deliberately decide the tool default and CLI flag behavior.

## Reproduction Assessment

Not applicable as a bug reproduction, but source inspection gives high confidence for the missing capability: current main rejects `agentId` in `cron.list` params and never applies exact agent filtering in `listPage`.

## Solution Assessment

Unclear until maintainers choose the desired visibility policy. An optional exact filter is narrow and additive, but automatic agent-session scoping changes multi-agent/admin behavior.
