## Summary

Keep open: current main and the latest release still stage `.gemini` through unfiltered recursive `fs.cpSync`, and the open closing PR at https://github.com/openclaw/openclaw/pull/91907 now owns the fix path.

## What This Changes

The issue requests excluding generated browser cache/profile trees when live tests stage external auth directories such as `.gemini` into a temporary HOME.

## Best Possible Solution

Land one scoped external-auth staging fix that preserves auth/config files while excluding generated browser cache/profile/recording trees, then close this issue and the duplicate reports after that fix reaches main.

## Reproduction Assessment

Yes, source-level. With `OPENCLAW_LIVE_TEST=1`, current main reaches `stageLiveTestState`, copies `.gemini` from `LIVE_EXTERNAL_AUTH_DIRS`, and uses recursive `fs.cpSync` without a filter.

## Solution Assessment

Yes. A narrow live external-auth staging filter or allowlist with regression coverage is the maintainable fix; the best current path is to review and land the focused linked PR rather than queue another duplicate repair.
