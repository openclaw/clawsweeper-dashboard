## Summary

## What This Changes

The issue reports live test setup copying large `.gemini` browser cache/profile/recording trees into temporary HOME directories and exhausting disk space.

## Best Possible Solution

Land one scoped live external-auth staging fix that preserves auth/config files while excluding generated browser cache/profile/recording trees, then close this issue and duplicate attempts after the fix reaches main.

## Reproduction Assessment

Yes, source-level. With `OPENCLAW_LIVE_TEST=1`, current main reaches `stageLiveTestState`, copies `.gemini` from `LIVE_EXTERNAL_AUTH_DIRS`, and uses recursive `fs.cpSync` without a filter.

## Solution Assessment

Yes. A narrow live external-auth staging filter or allowlist with regression coverage is the maintainable fix; the best current path is to review and land or narrow the linked PR rather than queue another duplicate repair.
