## Summary

Keep open: the Helm chart is a real new deployment feature, not implemented on current main, and the branch still has correctness and security blockers that need maintainer product review before repair or merge.

## What This Changes

The branch adds a values-driven Helm chart under charts/openclaw/ for deploying the OpenClaw gateway on Kubernetes alongside the existing raw Kustomize manifests.

## Best Possible Solution

If OpenClaw owns a Helm chart in this repository, it should preserve the raw-manifest resource/data contract, avoid plaintext credentials in Helm values by default, restart pods on config and secret changes, validate unsupported value combinations, track current release images, and update Kubernetes docs under owner approval.

## Reproduction Assessment

Yes. Source inspection of the PR chart is enough to reproduce the review defects: config/secret changes do not alter the pod template, service.port diverges from config.openclawJson, secret.create=false can render a missing required Secret reference, and the default RWO PVC conflicts with replicaCount > 1.

## Solution Assessment

No. A Helm deployment path may be useful, but this branch is not the best merge path until maintainers decide to own that surface and the chart fixes credential handling, rollout behavior, resource equivalence, stale image defaults, validation, docs, and proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Trigger rollouts for chart config and secret changes:** `charts/openclaw/templates/deployment.tpl:15-17`
  - body: The pod template has only static labels and no checksum annotations for the rendered ConfigMap or Secret. Because the init container copies config into the PVC and credentials are consumed as env vars, helm upgrade can succeed while the running pod keeps stale config or tokens until a manual restart. Kubernetes documents that env-var sourced config does not update running Pods without replacement. ([kubernetes.io](https://kubernetes.io/docs/tutorials/configuration/updating-configuration-via-a-configmap/?utm_source=openai))
  - confidence: 0.9
- **[P1] Avoid storing credentials in Helm release values:** `charts/openclaw/README.md:8-11`
  - body: The install docs pass secret.gatewayToken and provider keys through --set, and the chart models those credentials under values. Helm exposes release values with helm get values, so long-lived gateway tokens and provider keys can persist outside the intended Secret-only workflow. ([helm.sh](https://helm.sh/docs/v3/helm/helm_get_values/?utm_source=openai))
  - confidence: 0.86
- **[P2] Default the chart to a current OpenClaw image:** `charts/openclaw/values.yaml:8`
  - body: The chart hardcodes 2026.3.12/2026.3.12-slim even though the latest stable release in the review context is v2026.5.7. Fresh Helm installs would deploy an old gateway by default.
  - confidence: 0.86
- **[P2] Preserve existing Kubernetes resource names:** `charts/openclaw/templates/_helpers.tpl:7-9`
  - body: The fullname helper always renders <release>-<chart>, so the documented release name openclaw creates resources like openclaw-openclaw and a new PVC. That breaks the stated equivalence with the existing openclaw objects and prevents an in-place migration from the raw manifests.
  - confidence: 0.91
- **[P2] Keep the gateway config port tied to service.port:** `charts/openclaw/values.yaml:83-84`
  - body: config.openclawJson hardcodes gateway port 18789 while the chart exposes service.port for the container, probes, and Service. Overriding service.port renders Kubernetes resources for one port while the gateway listens on another.
  - confidence: 0.88
- **[P2] Require existingSecret when secret creation is disabled:** `charts/openclaw/templates/secret.tpl:1`
  - body: When secret.create=false and secret.existingSecret is empty, the helper still generates a Secret name but this template creates no Secret. The Deployment then references a required token Secret that does not exist, leaving pods stuck at startup instead of failing at render time.
  - confidence: 0.88
- **[P2] Do not advertise unreachable Service URLs:** `charts/openclaw/templates/NOTES.txt:2-8`
  - body: The notes print NodePort and LoadBalancer access instructions, but the default config keeps the gateway bound to loopback. Those Service paths route to the pod IP, so the rendered instructions lead operators to a dead endpoint unless the chart also changes or warns about the bind setting.
  - confidence: 0.82
- **[P3] Guard replicaCount with the default RWO PVC:** `charts/openclaw/values.yaml:4`
  - body: The chart exposes replicaCount while the default PVC access mode is ReadWriteOnce. Setting more than one replica can leave extra pods pending on multi-node clusters, and strategy: Recreate does not prevent multiple steady-state replicas.
  - confidence: 0.78
