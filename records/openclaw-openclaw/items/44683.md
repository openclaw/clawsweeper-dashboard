## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Trigger rollouts when config or secrets change:** `charts/openclaw/templates/deployment.tpl:15-17`
  - body: The pod template has only static labels, so changing `config.openclawJson`, `config.agentsMd`, or `secret.*` during `helm upgrade` does not create a new ReplicaSet. Because the init container copies config at startup and credentials are env vars, upgrades can report success while pods keep stale config and tokens until a manual restart.
  - confidence: 0.93
- **[P1] Preserve existing Kubernetes object names:** `charts/openclaw/templates/_helpers.tpl:9`
  - body: The documented release name `openclaw` renders names like `openclaw-openclaw` and `openclaw-openclaw-home-pvc`, while current manifests use `openclaw` and `openclaw-home-pvc`. Users migrating from the supported manifests would get a second deployment and fresh PVC rather than adopting existing state.
  - confidence: 0.92
- **[P1] Avoid putting provider credentials in Helm values:** `charts/openclaw/README.md:8-11`
  - body: The primary install example sends the gateway token and provider API key through `--set`, which exposes sensitive values through shell/process history and Helm release state. For an official chart, the default guidance should prefer a pre-created Kubernetes Secret or a safer secret-management path.
  - confidence: 0.88
- **[P2] Validate external Secret configuration:** `charts/openclaw/templates/secret.tpl:1`
  - body: When `secret.create=false` and `secret.existingSecret` is empty, Helm still renders a Deployment that references the generated Secret name, but `secret.tpl` creates no Secret. That turns a template-time configuration error into pods stuck in `CreateContainerConfigError`.
  - confidence: 0.94
- **[P2] Keep service values synchronized with gateway config:** `charts/openclaw/values.yaml:79`
  - body: `service.port` drives the Service, container port, and probes, but the embedded gateway config still hardcodes port `18789` and loopback binding. Operators can render a custom Service, NodePort, or LoadBalancer that the gateway does not actually listen on.
  - confidence: 0.9
- **[P2] Guard replica counts with ReadWriteOnce persistence:** `charts/openclaw/values.yaml:4`
  - body: The chart exposes `replicaCount` while default persistence uses a ReadWriteOnce PVC. Setting replicas above one can leave pods pending on multi-node clusters, and `strategy: Recreate` does not prevent multiple steady-state replicas.
  - confidence: 0.86
