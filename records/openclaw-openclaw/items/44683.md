## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve existing Kubernetes resource names:** `charts/openclaw/templates/_helpers.tpl:15`
  - body: The documented `openclaw` release renders names like `openclaw-openclaw` and `openclaw-openclaw-home-pvc`, while the current manifests use `openclaw` and `openclaw-home-pvc`. Users moving from the documented Kustomize install would get a second deployment and fresh PVC/Secret instead of adopting their existing state, so the chart needs a migration-safe fullname pattern or explicit migration path.
  - confidence: 0.91
- **[P1] Restart pods when rendered config or secrets change:** `charts/openclaw/templates/deployment.tpl:21-23`
  - body: `config.openclawJson`, `config.agentsMd`, and `secret.*` values are consumed only at pod startup, but the pod template has no checksum annotations or other changing field tied to those rendered resources. `helm upgrade` can succeed while the running gateway keeps stale config or credentials until an operator manually restarts it.
  - confidence: 0.9
- **[P1] Do not put credentials in the primary Helm values path:** `charts/openclaw/README.md:10-11`
  - body: The install command tells users to pass the gateway token and provider key via `--set`, which can expose secrets through shell history, process listings, CI logs, and Helm release storage. A first-party chart should prefer an existing Kubernetes Secret or another documented secret-management path by default.
  - confidence: 0.88
- **[P2] Validate missing external Secret before rendering:** `charts/openclaw/templates/secret.tpl:1`
  - body: When `secret.create=false` and `secret.existingSecret` is empty, the helper still returns a generated Secret name while `secret.tpl` creates no Secret. The Deployment then references a required `OPENCLAW_GATEWAY_TOKEN` from a non-existent Secret and pods fail at runtime instead of Helm failing with an actionable error.
  - confidence: 0.9
- **[P2] Synchronize the gateway port with service.port:** `charts/openclaw/values.yaml:90`
  - body: `service.port` controls the container port and probes, but `config.openclawJson` still hardcodes the gateway to `18789`. If an operator changes `service.port`, Kubernetes checks and routes to a port the gateway is not listening on unless they also manually edit the JSON blob.
  - confidence: 0.88
- **[P2] Guard replicas against the default ReadWriteOnce PVC:** `charts/openclaw/values.yaml:4`
  - body: The chart exposes `replicaCount` while persistence defaults to a single ReadWriteOnce PVC. Setting replicas above one can leave pods pending or competing for one writer unless the chart validates the combination or documents a supported multi-replica storage mode.
  - confidence: 0.82
