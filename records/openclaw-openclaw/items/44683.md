## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Trigger rollouts when config or secrets change:** `charts/openclaw/templates/deployment.tpl:15-17`
  - body: The pod template has only static labels, so changing `config.openclawJson`, `config.agentsMd`, or `secret.*` will not create a new ReplicaSet. Those inputs are copied or injected only at startup, so `helm upgrade` can succeed while the gateway keeps stale config and credentials.
  - confidence: 0.91
- **[P1] Preserve existing Kubernetes object names:** `charts/openclaw/templates/_helpers.tpl:9`
  - body: The fullname helper always renders `<release>-openclaw`; with the documented release name `openclaw`, this creates `openclaw-openclaw` and a new PVC instead of adopting the current `openclaw` resources. That breaks the stated equivalence and can split existing deployment state during migration.
  - confidence: 0.9
- **[P1] Avoid passing provider secrets through Helm values:** `charts/openclaw/README.md:8-11`
  - body: The install example tells operators to pass the gateway token and provider API key through `helm --set`. An official path should prefer a pre-created Kubernetes Secret or another safer handoff and make inline values a warned local-testing path.
  - confidence: 0.86
- **[P2] Keep service.port and gateway.port in sync:** `charts/openclaw/values.yaml:84`
  - body: `service.port`, `containerPort`, and probes are templated, but `config.openclawJson` still hardcodes gateway port `18789`. If an operator changes only `service.port`, Kubernetes targets a port the gateway is not listening on and the pod fails readiness or becomes unreachable.
  - confidence: 0.88
- **[P2] Require an existing Secret when creation is disabled:** `charts/openclaw/templates/secret.tpl:1`
  - body: When `secret.create=false` and `secret.existingSecret` is empty, the chart renders no Secret but still points the Deployment at the generated secret name. Helm should fail at template time instead of installing pods that enter `CreateContainerConfigError`.
  - confidence: 0.89
- **[P2] Keep Service exposure aligned with gateway bind:** `charts/openclaw/templates/NOTES.txt:2-8`
  - body: The notes print NodePort and LoadBalancer access paths, but the default config still binds the gateway to loopback inside the pod. Users who change only `service.type` will get a Service that advertises an endpoint the gateway is not accepting through the pod IP.
  - confidence: 0.84
- **[P2] Guard replicaCount with the default PVC access mode:** `charts/openclaw/values.yaml:4`
  - body: The chart exposes `replicaCount`, but the default persistent volume access mode is `ReadWriteOnce`. Setting more than one replica can leave pods pending or competing for the same state unless the chart validates the unsupported combination or clearly requires a compatible storage mode.
  - confidence: 0.82
