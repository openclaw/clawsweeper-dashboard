## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Trigger rollouts when config or secrets change:** `charts/openclaw/templates/deployment.tpl:15-17`
  - body: The pod template has only static labels, so changing `config.openclawJson`, `config.agentsMd`, or `secret.*` will not create a new ReplicaSet. Those inputs are copied or injected only at startup, and Kubernetes rolls a Deployment only when `.spec.template` changes, so `helm upgrade` can succeed while the gateway keeps stale config and credentials. ([kubernetes.io](https://kubernetes.io/docs/concepts/workloads/controllers/deployment/?source=post_page-----8db6c301b18--------------------------------))
  - confidence: 0.91
- **[P1] Preserve existing Kubernetes object names:** `charts/openclaw/templates/_helpers.tpl:9`
  - body: The fullname helper always renders `<release>-openclaw`; with the documented release name `openclaw`, this creates `openclaw-openclaw` and a new PVC instead of adopting the current `openclaw` resources. That breaks the stated equivalence and can split existing deployment state during migration.
  - confidence: 0.9
- **[P1] Avoid passing provider secrets through Helm values:** `charts/openclaw/README.md:8-11`
  - body: The install example tells operators to pass the gateway token and provider API key through `helm --set`. Helm exposes release values through `helm get values`, so an official path should prefer a pre-created Secret or another safer handoff and make inline values a warned testing-only path. ([helm.sh](https://helm.sh/el/docs/helm/helm_get_values))
  - confidence: 0.86
- **[P2] Keep service.port and gateway.port in sync:** `charts/openclaw/values.yaml:84`
  - body: `service.port`, `containerPort`, and probes are templated, but `config.openclawJson` still hardcodes gateway port `18789`. If an operator changes only `service.port`, Kubernetes targets a port the gateway is not listening on and the pod fails readiness or becomes unreachable.
  - confidence: 0.88
- **[P2] Require an existing Secret when creation is disabled:** `charts/openclaw/templates/secret.tpl:1`
  - body: When `secret.create=false` and `secret.existingSecret` is empty, the chart renders no Secret but still points the Deployment at the generated secret name. Helm should fail at template time instead of installing pods that enter `CreateContainerConfigError`.
  - confidence: 0.89
- **[P2] Keep Service exposure aligned with gateway bind:** `charts/openclaw/templates/NOTES.txt:2-8`
  - body: The generated notes advertise NodePort and LoadBalancer access, but the default config still binds the gateway to loopback. Users who change only `service.type` will get an external Service that cannot reach the pod listener unless the chart fails, templates the bind, or warns in the generated notes.
  - confidence: 0.84
- **[P2] Guard replicaCount with the ReadWriteOnce PVC default:** `charts/openclaw/values.yaml:4`
  - body: The chart lets users set `replicaCount > 1` while persistence defaults to a single ReadWriteOnce claim. Kubernetes documents ReadWriteOnce as read-write mountable by a single node, so multi-node scheduling can leave extra replicas pending unless the chart validates or documents the unsupported combination. ([kubernetes.io](https://kubernetes.io/docs/concepts/storage/persistent-volumes/?source=post_page---------------------------))
  - confidence: 0.8
