## Summary

Keep open: current main does not contain a Helm chart, but this PR is not merge-ready because source review shows blocking chart correctness/security gaps and the external contributor has not supplied real install/upgrade proof.

## What This Changes

The PR adds an additive Helm chart under `charts/openclaw` with chart metadata, values, templates, NOTES, README, and `.helmignore` for a values-driven Kubernetes deployment.

## Best Possible Solution

If OpenClaw owns a Helm chart, it should align with the raw-manifest resource and data contract, avoid credential-in-values defaults, roll pods on config or Secret changes, validate unsupported values, update public Kubernetes docs, and include real install/upgrade proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Trigger rollouts for config and secret changes:** `charts/openclaw/templates/deployment.tpl:15-17`
  - body: The pod template has only static labels and no checksum annotations for the rendered ConfigMap or Secret. Because this chart copies config during init and injects API keys/tokens as env vars, `helm upgrade` can succeed while the running pod keeps stale config or credentials until a manual restart; Kubernetes documents that Secret env updates require a container restart. citeturn3search0.
  - confidence: 0.9
- **[P1] Avoid putting long-lived credentials in Helm values:** `charts/openclaw/README.md:10-11`
  - body: The README tells operators to pass the gateway token and provider API keys through `--set`, and the chart models those secrets as values. Helm release values are retrievable later with `helm get values`, so this stores long-lived credentials outside the intended Secret-only workflow. citeturn0search0.
  - confidence: 0.88
- **[P2] Default to a current OpenClaw image:** `charts/openclaw/values.yaml:8`
  - body: The chart hardcodes `2026.3.12-slim` while current main is `2026.5.10-beta.1` and the provided review context identifies `v2026.5.7` as the latest stable release. Fresh Helm installs would deploy an old gateway by default.
  - confidence: 0.86
- **[P2] Preserve raw-manifest resource names:** `charts/openclaw/templates/_helpers.tpl:7-9`
  - body: The `fullname` helper always renders `<release>-<chart>`, so the documented release name `openclaw` creates resources like `openclaw-openclaw` and a new PVC. That diverges from the existing `openclaw` resources and breaks the stated equivalent migration path.
  - confidence: 0.9
- **[P2] Tie the gateway config port to service.port:** `charts/openclaw/values.yaml:83-84`
  - body: `config.openclawJson` hardcodes gateway port `18789` while the Deployment, probes, and Service use `.Values.service.port`. Overriding `service.port` renders Kubernetes resources for one port while the gateway still listens on another.
  - confidence: 0.88
- **[P2] Require existingSecret when secret creation is disabled:** `charts/openclaw/templates/secret.tpl:1`
  - body: When `secret.create=false` and `secret.existingSecret` is empty, the Secret template renders nothing but the Deployment still references the generated Secret name. Pods then fail at startup instead of Helm failing with a clear render-time error.
  - confidence: 0.88
- **[P2] Do not print unreachable Service endpoints:** `charts/openclaw/templates/NOTES.txt:2-8`
  - body: The NOTES output advertises NodePort and LoadBalancer access paths, but the default config keeps the gateway bound to loopback. Kubernetes Services route to selected Pods and target ports, not the pod's loopback listener, so the rendered instructions can lead to a dead endpoint unless the bind setting is also changed or clearly warned about. citeturn0search3.
  - confidence: 0.82
- **[P3] Guard multiple replicas with the RWO PVC default:** `charts/openclaw/values.yaml:4`
  - body: The chart exposes `replicaCount` while the default PVC access mode is `ReadWriteOnce`, which Kubernetes defines as read-write by a single node. Setting more than one replica can leave extra pods pending on multi-node clusters, and `strategy: Recreate` does not prevent multiple steady-state replicas. citeturn0search6.
  - confidence: 0.78
