## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Restart pods when chart config or secrets change:** `charts/openclaw/templates/deployment.tpl:15-17`
  - body: The pod template has no checksum or rollout annotation, so `helm upgrade` changes to `config.openclawJson`, `config.agentsMd`, or `secret.*` only update the ConfigMap/Secret. Because the init container copies config once and gateway credentials are env vars, the running pods keep stale config or tokens until someone manually restarts the Deployment.
  - confidence: 0.91
- **[P2] Keep rendered names equivalent to existing objects:** `charts/openclaw/templates/_helpers.tpl:9`
  - body: The README installs the chart with release name `openclaw`, but this helper renders names as `openclaw-openclaw`. That creates a second Deployment/Service/PVC instead of matching the existing `openclaw` objects and `openclaw-home-pvc`, breaking the promised equivalent migration path.
  - confidence: 0.9
- **[P2] Tie the gateway listener port to service.port:** `charts/openclaw/values.yaml:84`
  - body: `service.port` drives the container port and probes, but `config.openclawJson` still hard-codes `gateway.port` to `18789`. Overriding `service.port` makes the probes and Service target a port the gateway is not listening on unless users also edit raw JSON by hand.
  - confidence: 0.89
- **[P2] Fail when an external Secret name is required:** `charts/openclaw/templates/_helpers.tpl:29-30`
  - body: With `secret.create=false` and no `secret.existingSecret`, the helper still returns a generated name while `secret.tpl` emits no Secret. The Deployment then references a missing non-optional `OPENCLAW_GATEWAY_TOKEN`, so pods fail at startup instead of Helm rejecting the invalid values.
  - confidence: 0.88
- **[P2] Warn before advertising non-port-forward Service URLs:** `charts/openclaw/templates/NOTES.txt:2-8`
  - body: The install notes print NodePort and LoadBalancer access instructions when `service.type` changes, but the default gateway config binds to loopback. Those URLs will refuse connections unless users also change the bind setting, so the chart should fail or clearly warn in the generated notes for non-port-forward exposure.
  - confidence: 0.8
- **[P3] Guard multi-replica installs that share the default PVC:** `charts/openclaw/values.yaml:4`
  - body: `replicaCount` can be raised while `persistence.enabled` and `ReadWriteOnce` remain the defaults, which asks multiple pods to share one RWO PVC. Add chart validation or a prominent values note so extra pods do not sit pending on multi-node clusters.
  - confidence: 0.72
