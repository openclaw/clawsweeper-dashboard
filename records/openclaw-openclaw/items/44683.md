## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Preserve existing Kubernetes resource names:** `charts/openclaw/templates/_helpers.tpl:9`
  - body: The documented release name `openclaw` renders resources like `openclaw-openclaw` and `openclaw-openclaw-home-pvc`. That is not equivalent to the shipped manifests and would create a second Deployment/PVC instead of migrating existing Kubernetes users; use the standard fullname pattern or explicit migration-safe names.
  - confidence: 0.93
- **[P1] Trigger pods when config or secrets change:** `charts/openclaw/templates/deployment.tpl:17`
  - body: The pod template has only static labels, while the chart's ConfigMap is copied by an init container and Secrets are read as env vars at startup. `helm upgrade` can therefore succeed after `config.*` or `secret.*` changes while the running pod keeps stale config or credentials until a manual restart.
  - confidence: 0.91
- **[P1] Keep credentials out of Helm values:** `charts/openclaw/README.md:10-11`
  - body: The primary install command passes the gateway token and provider key through `--set`. For an official chart, that risks leaking secrets through shell history, process listings, CI logs, and Helm release state; prefer pre-existing Kubernetes Secrets or a documented secret-management path.
  - confidence: 0.88
- **[P2] Validate external Secret configuration:** `charts/openclaw/templates/secret.tpl:1`
  - body: When `secret.create=false` and `secret.existingSecret` is empty, the chart renders no Secret but the Deployment still references the generated Secret name for the non-optional gateway token. Add a render-time failure so this does not become a pod `CreateContainerConfigError`.
  - confidence: 0.9
- **[P2] Synchronize the gateway port with service.port:** `charts/openclaw/values.yaml:84`
  - body: `service.port` drives the Service, containerPort, and probes, but `config.openclawJson` still hardcodes `gateway.port` to 18789. Overriding `service.port` alone makes the pod probe and Service target a port where the gateway is not listening.
  - confidence: 0.88
- **[P2] Guard unsupported replica and PVC settings:** `charts/openclaw/values.yaml:4`
  - body: The chart exposes `replicaCount`, but the default persistent volume is `ReadWriteOnce`; setting replicas above one can leave pods pending or contending for one PVC. Add a template validation or make the persistence/replica relationship explicit before exposing this as a supported value.
  - confidence: 0.82
