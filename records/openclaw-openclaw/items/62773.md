## Summary

Keep this PR open for maintainer/contributor follow-up: current main still lacks a first-class directory listing tool, but this implementation has sandbox, symlink metadata, deterministic truncation, stale import, and proof blockers.

## What This Changes

The branch adds a new `list` filesystem tool under `group:fs`, wires required params/catalog/tool construction, and returns structured directory entries with size and mtime.

## Best Possible Solution

Revise the feature around a sandbox-bridge directory listing contract, safe symlink metadata semantics, deterministic sorting/truncation, catalog/docs/test coverage, and redacted real-run proof.

## Reproduction Assessment

Yes. Source inspection of the PR diff reproduces the blockers: `list` is added as a host filesystem tool even when sandboxed, child metadata uses `fs.stat` after only parent validation, and truncation occurs before sorting.

## Solution Assessment

No. A first-class listing tool is a reasonable direction, but this patch is not the best implementation until it preserves sandbox boundaries, settles symlink metadata, rebases package imports, and adds focused docs/tests/proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Route sandbox listings through the sandbox bridge:** `src/agents/pi-tools.ts:549-552`
  - body: In sandboxed sessions, this adds `list` as a host filesystem tool instead of routing through `SandboxFsBridge` like the existing filesystem tools. If `tools.fs.workspaceOnly` is false, an agent can enumerate host paths outside sandbox mounts.
  - confidence: 0.91
- **[P2] Stop following symlink targets for entry metadata:** `src/agents/pi-tools.list.ts:116-118`
  - body: The parent directory is boundary-checked, but each child is inspected with `fs.stat(entryPath)`, which follows symlinks. A workspace symlink to an out-of-workspace target can expose target type, size, and mtime; use no-follow metadata or revalidate each child.
  - confidence: 0.88
- **[P2] Sort before applying the entry cap:** `src/agents/pi-tools.list.ts:105-107`
  - body: The loop stops after 1000 entries in raw filesystem order and sorts only that subset. Large directories therefore return an arbitrary page while reporting `truncated: true`; sort the full dirent list first, then slice.
  - confidence: 0.86
- **[P2] Return null for missing mtimes:** `src/agents/pi-tools.list.ts:113-123`
  - body: `ListEntry.mtime` is documented as an ISO-8601 string, but broken or unreadable entries return an empty string. Make the field nullable and return `null` when metadata is unavailable.
  - confidence: 0.82
- **[P2] Use the current PI package name:** `src/agents/pi-tools.list.ts:3`
  - body: The new file imports `AgentTool` from `@mariozechner/pi-agent-core`, but current main depends on the renamed `@earendil-works/pi-agent-core` package, so the rebased branch will not resolve this import.
  - confidence: 0.83
