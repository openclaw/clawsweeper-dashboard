Full review comments:

- **[P1] Route sandbox listings through the sandbox bridge:** `src/agents/pi-tools.ts:549-552`
  - body: The new tool is created with `createListTool({ cwd: sandboxRoot ?? workspaceRoot, workspaceOnly })`, and that implementation uses host filesystem APIs. In sandboxed sessions where `tools.fs.workspaceOnly` is false, absolute paths such as `/etc` enumerate the host instead of the sandbox.
  - confidence: 0.91
- **[P1] Use the current PI package name:** `src/agents/pi-tools.list.ts:3`
  - body: The added file imports `AgentTool` from `@mariozechner/pi-agent-core`, but current main depends on `@earendil-works/pi-agent-core`; rebasing this branch as-is leaves the import unresolved.
  - confidence: 0.86
- **[P2] Stop following symlink targets for entry metadata:** `src/agents/pi-tools.list.ts:116-118`
  - body: The parent directory is boundary-checked, but each child is inspected with `fs.stat(entryPath)`, which follows symlinks. A workspace symlink can expose target type, size, and mtime outside the workspace; use no-follow metadata or revalidate each child before exposing target metadata.
  - confidence: 0.88
- **[P2] Sort before applying the entry cap:** `src/agents/pi-tools.list.ts:105-107`
  - body: The loop stops once 1000 raw filesystem-order entries are collected and sorts only that subset. Large directories therefore return an arbitrary page while claiming `truncated: true`; sort the full dirent list first, then slice.
  - confidence: 0.86
- **[P2] Return null when mtime is unavailable:** `src/agents/pi-tools.list.ts:23-24`
  - body: `ListEntry.mtime` is typed and documented as an ISO-8601 timestamp, but failed entry stats return an empty string. Make the field nullable and return `null` so callers can distinguish missing metadata from a real timestamp.
  - confidence: 0.82
