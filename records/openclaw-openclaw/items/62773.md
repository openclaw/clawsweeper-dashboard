## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Reuse the existing ls tool instead of adding list:** `src/agents/pi-tools.list.ts:49-52`
  - body: Current main already has `createLsTool` and `ToolName` includes `ls`; adding a separate `list` implementation creates a second filesystem enumeration API with different limits, output, metadata, and tests. Wire the existing tool through the current `agent-tools.ts` path, or get an explicit tool-name decision before adding a parallel API.
  - confidence: 0.92
- **[P1] Keep sandboxed listings behind the sandbox bridge:** `src/agents/pi-tools.ts:546-549`
  - body: This appends `createListTool` as a host filesystem tool even when a sandbox is active. With `tools.fs.workspaceOnly` false or unset, absolute paths are resolved on the host, while current sandboxed read/write/edit go through `SandboxFsBridge`; directory listing needs the same bridge-backed boundary and tests.
  - confidence: 0.9
- **[P2] Avoid following symlink targets for entry metadata:** `src/agents/pi-tools.list.ts:116`
  - body: The boundary check validates the listed directory, but `fs.stat(entryPath)` follows symlinks, so a workspace symlink can expose target size/mtime and type information outside the intended boundary. Use bridge or `lstat`-style metadata, or omit unsafe symlink metadata.
  - confidence: 0.88
- **[P2] Sort before applying the entry limit:** `src/agents/pi-tools.list.ts:105-108`
  - body: The loop stops after the first 1000 raw `readdir` entries and sorts only that subset, so large directories return a filesystem-order slice masquerading as deterministic sorted output. Sort the full name list before slicing or add an explicit cursor contract.
  - confidence: 0.86
- **[P3] Make missing mtime nullable:** `src/agents/pi-tools.list.ts:26`
  - body: `ListEntry.mtime` is documented as an ISO-8601 string, but the error path returns an empty string for entries that cannot be statted. Use `string | null` and return `null` so callers do not have to special-case an invalid timestamp.
  - confidence: 0.82
