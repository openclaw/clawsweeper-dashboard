## Summary

Keep open: the PR targets a real filesystem-tool gap, but the current branch is not merge-ready because it bypasses the sandbox filesystem bridge, has unsafe symlink metadata behavior, and lacks real after-fix behavior proof.

## What This Changes

The PR adds a core `list` filesystem tool wired into `group:fs` and required path parameters, returning structured directory entries from a new host filesystem implementation.

## Best Possible Solution

Define a sandbox-bridge directory listing operation and safe symlink metadata contract, then wire `list` through the same host/sandbox policy model as `read`, `write`, and `edit` with docs and regression tests.

## Reproduction Assessment

Yes. Source inspection of the PR head shows a host `createListTool` added directly in `createOpenClawCodingTools`, while current sandboxed file tools use `SandboxFsBridge`, and the new implementation follows child symlinks with `fs.stat`.

## Solution Assessment

No. A first-class directory listing tool is a reasonable direction, but this implementation should not land until it follows the sandbox bridge model and has an explicit symlink metadata contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Route sandbox listings through the sandbox bridge:** `src/agents/pi-tools.ts:549-552`
  - body: The PR always registers `createListTool` as a host filesystem tool. In sandboxed sessions this bypasses `SandboxFsBridge`; when workspace-only mode is disabled, paths such as `/etc` can be listed on the host rather than through sandbox mounts.
  - confidence: 0.91
- **[P2] Stop following child symlink targets:** `src/agents/pi-tools.list.ts:116-118`
  - body: `fs.stat(entryPath)` follows symlinks after only the parent directory has been boundary-checked, so a workspace symlink can expose out-of-workspace target type, size, and mtime. Use no-follow metadata or revalidate each resolved child before returning metadata.
  - confidence: 0.88
- **[P2] Sort before applying the entry cap:** `src/agents/pi-tools.list.ts:105-109`
  - body: The loop stops at `MAX_ENTRIES` in raw filesystem order and sorts only the truncated subset, so large directories return an arbitrary page while claiming sorted output. Read and sort the directory entries first, then slice to the cap.
  - confidence: 0.86
- **[P2] Return null for unavailable mtimes:** `src/agents/pi-tools.list.ts:113-123`
  - body: `mtime` is documented as an ISO-8601 string, but the stat-failure path returns `""`, which violates the structured contract for broken symlinks or unreadable entries. Make the field nullable and return `null` when metadata is unavailable.
  - confidence: 0.82
