  - confidence: 0.9
- **[P1] Route sandbox listings through the sandbox bridge:** `src/agents/pi-tools.ts:549-552`
  - body: When a sandbox is active, this creates a host filesystem tool with `cwd: sandboxRoot` rather than delegating through `sandboxFsBridge` like read/write/edit. In non-workspace-only or mis-scoped runs, the model can enumerate host paths instead of the sandbox view.
  - confidence: 0.93
- **[P2] Stop following symlink targets for child metadata:** `src/agents/pi-tools.list.ts:116-118`
  - body: The parent directory boundary check does not validate every child target, but `fs.stat(entryPath)` follows symlinks. A workspace symlink to an external file can therefore leak the target's size and mtime through `list`; use `lstat` or omit symlink target metadata.
  - confidence: 0.91
- **[P2] Sort entries before enforcing the cap:** `src/agents/pi-tools.list.ts:105-137`
  - body: The loop stops at `MAX_ENTRIES` before sorting, so large directories return an arbitrary filesystem-order subset that is only sorted afterward. Read and sort the full directory entry list first, then slice to the cap so `truncated: true` has deterministic meaning.
  - confidence: 0.9
- **[P2] Return nullable mtime for unstatable entries:** `src/agents/pi-tools.list.ts:24`
  - body: `ListEntry.mtime` is documented as an ISO-8601 string, but the broken-symlink/error path returns an empty string. Make the contract `string | null` and return `null` when metadata is unavailable so callers do not special-case invalid timestamps.
  - confidence: 0.86
