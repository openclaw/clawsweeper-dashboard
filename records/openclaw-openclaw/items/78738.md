## Summary

Keep open: current main still has exec approval delivery paths where transport/follow-up failures are caught and logged instead of becoming an agent-visible approval failure. The related retry PR was closed unmerged, and the still-open warning-suppression PR only handles benign expired-id noise.

## What This Changes

The issue asks for exec approval delivery and follow-up dispatch failures to return visible errors to the agent instead of silently dropping commands.

## Best Possible Solution

Surface failed approval prompt delivery and failed async follow-up delivery through existing approval-unavailable or follow-up failure contracts, with focused regression coverage for forwarding, native delivery, and session follow-up failures.

## Reproduction Assessment

Yes, at source level: current main has injectable paths where outbound or session follow-up failures are caught and logged without producing an agent-visible result. I did not run a live Discord/WebSocket reproduction.

## Solution Assessment

Yes, the narrow maintainable direction is to reuse the existing approval failure/no-route/follow-up result mechanisms. A log-only suppression or retry-only patch would not fully solve failed prompt delivery or final delivery exhaustion.
