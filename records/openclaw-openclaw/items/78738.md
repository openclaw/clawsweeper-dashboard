## Summary

Keep open: current main still has source-visible paths where exec approval prompt or followup delivery failures are accepted/logged without becoming an agent-visible approval failure, and the linked fix PR was closed unmerged.

## What This Changes

The issue requests an explicit agent-visible error when exec approval dispatch or followup delivery fails instead of silently dropping the command.

## Best Possible Solution

Propagate failed approval prompt delivery and final followup delivery failures through existing visible approval failure or no-route mechanisms, with transient retry as a supplement rather than a log-only endpoint.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: injected outbound or session-followup failures flow into log-only catch paths instead of an agent-visible failure. I did not run the live Windows/Discord/WebSocket scenario.

## Solution Assessment

Yes, the best fix is to reuse the existing approval-request-failed or no-approval-route outcomes after delivery failure, with focused retries only where they improve transient recovery. A warning-only or log-suppression change would not solve the dropped-command behavior.
