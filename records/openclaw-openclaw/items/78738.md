## Summary

Keep open: current main still has exec approval delivery and follow-up paths where dispatch failures are accepted or caught and logged instead of becoming an agent-visible failure, and the related retry PR was closed unmerged.

## What This Changes

The issue reports that exec approval dispatch or follow-up failures can drop an exec command without returning an agent-visible error.

## Best Possible Solution

Surface failed approval prompt delivery and failed async follow-up delivery through existing approval-unavailable or follow-up failure contracts, with focused regression coverage for forwarding, native delivery, and session follow-up failures.

Yes at source level: current main has injectable outbound and session-follow-up failure paths that are caught and logged without producing an agent-visible failure. I did not run a live Discord/WebSocket reproduction.

## Solution Assessment

Yes: the narrow maintainable fix is to reuse existing approval failure, no-route, or follow-up result mechanisms. A log-only change or retry-only patch would not fully solve failed prompt delivery or final delivery exhaustion.
