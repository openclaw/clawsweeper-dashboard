## Summary

Keep open: current main still treats configured approval-target existence as successful delivery, then logs per-target dispatch failures asynchronously, so the approval-gated command can remain pending without an agent-visible error. The linked open fix PR is the right item to review or replace before closing this issue.

## What This Changes

Exec approval dispatch failures should return a visible agent error instead of leaving approval-gated commands pending silently.

## Best Possible Solution

Review, rebase, and land https://github.com/openclaw/openclaw/pull/82506 or replace it with a bounded fix that turns all-target approval delivery failure into an explicit agent-visible failure with regression coverage for exec and plugin approvals.
