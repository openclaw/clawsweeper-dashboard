## Summary

Keep open: this is a plausible narrow bug fix for the linked Tailscale startup-noise report, but the latest head is still draft, lacks real Tailscale/gateway proof, and contains a focused test expectation that does not match the retry implementation.

## What This Changes

The PR adds retry/backoff around `tailscale status --json`, a `runExec` option to suppress intermediate failure logs, new Tailscale unit coverage, and an injected repro script.

## Best Possible Solution

Merge a corrected narrow retry that keeps intermediate attempts quiet, preserves final failure logging and missing-binary fallback behavior, and includes focused tests plus redacted live Tailscale restart proof.

## Reproduction Assessment

Yes at source level: current main enables Tailscale Serve, immediately calls `getTailnetHostname`, and `runExec` logs failed status commands before the caller catch can swallow the optional hostname failure. I did not establish a live macOS Tailscale daemon reproduction.

## Solution Assessment

Mostly yes, but not as patched: retrying the optional status probe with final-attempt logging is the narrow maintainable fix, while the current head still needs the focused assertion corrected and real daemon proof added.
