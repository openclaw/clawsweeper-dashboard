## Summary

Keep open: the PR targets a valid source-level Tailscale startup log race, but the current diff can still emit the transient verbose failure before retry succeeds, and the supplied real behavior proof is mock-only.

## What This Changes

The PR adds a retrying `tailscale status --json` reader for `getTailnetHostname`, unit coverage for transient and missing-binary cases, and an injected-exec repro script.

## Best Possible Solution

Keep the localized retry, but make intermediate status attempts quiet or final-attempt-only and back it with focused tests plus redacted real restart output.

## Reproduction Assessment

Yes at source level: Serve startup immediately calls `getTailnetHostname` after `enableTailscaleServe`, and `runExec` logs a failed status command before rethrowing under verbose/debug logging. I did not run a live macOS Tailscale daemon restart.

## Solution Assessment

No. Retrying the status probe is the right shape, but the default exec path must stay quiet until the final failure, and this patch still needs real Tailscale restart proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Suppress transient status logs before retry succeeds:** `src/infra/tailscale.ts:140-145`
