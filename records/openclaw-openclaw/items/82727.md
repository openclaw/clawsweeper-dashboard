## Summary

Keep open: the linked Tailscale startup bug is source-reproducible and the PR is the right general shape, but the current draft head has mock-only real behavior proof and a failing test expectation in the newly added quiet-retry coverage.

## What This Changes

This PR adds retry handling around `tailscale status --json`, suppresses intermediate retry failure logs via `runExec`, adds Tailscale unit coverage, and includes an injected-exec repro script.

## Best Possible Solution

Finish the localized retry with quiet all-but-final status attempts, fix the focused unit expectation, and attach redacted real gateway/Tailscale restart output before merge.

## Reproduction Assessment

Yes at source level: current main enables Tailscale Serve, immediately calls `getTailnetHostname`, and `runExec` logs failed status commands before rethrowing under verbose logging. I did not establish a live macOS Tailscale daemon reproduction here.

## Solution Assessment

Mostly yes, but not merge-ready: retrying the status probe with final-attempt logging is the narrow maintainable direction, while the current patch needs the test expectation fixed and real daemon proof added.
