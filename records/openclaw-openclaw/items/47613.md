## Summary

Keep open. The PR targets a real callback-coupling gap, but it is dirty against current main and needs maintainer coordination with overlapping reasoning-stream PRs plus a rebase that preserves current visibility gates.

## What This Changes

The PR changes `src/agents/pi-embedded-subscribe.ts` so `reasoningMode: "stream"` can broadcast thinking events without requiring `onReasoningStream`, while invoking that callback only when present.

## Best Possible Solution

Land one canonical Gateway reasoning-stream implementation that decouples `emitAgentEvent` from channel callbacks, preserves explicit stream-mode, thinking-enabled, and silent-turn gates, and adds focused WebSocket regression coverage plus docs and changelog updates.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: Gateway `chat.send` passes no `onReasoningStream`, while `subscribeEmbeddedPiSession` still requires that callback before broadcasting live thinking events.

## Solution Assessment

No. The direction is plausible, but the PR is not the best mergeable patch as-is because a rebase must remove only the callback dependency while preserving current `canShowReasoning` and silent-turn protections.
