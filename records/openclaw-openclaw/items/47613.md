## Summary

Keep open: current main still has the no-callback reasoning broadcast gap, but this branch is not mergeable as-is because it drops a visibility gate and overlaps with other open Gateway reasoning-stream PRs.

## What This Changes

The PR changes `src/agents/pi-embedded-subscribe.ts` so `reasoningMode: "stream"` enables reasoning streaming without an `onReasoningStream` callback and only calls that callback when present.

## Best Possible Solution

Land one canonical Gateway reasoning-stream fix that decouples WebSocket broadcasts from channel callbacks, preserves thinking-level and silent-turn gates, adds no-callback Gateway coverage, and updates docs/changelog if Gateway streaming is supported.

## Reproduction Assessment

Yes. Source inspection shows `chat.send` supplies no `onReasoningStream`, current `emitReasoningStream()` returns before `emitAgentEvent`, and Gateway would already broadcast the non-tool event once emitted.

## Solution Assessment

No. The direction is plausible, but the patch should remove only the callback dependency while preserving visibility gates and should be reconciled with the overlapping open Gateway reasoning-stream PRs.
