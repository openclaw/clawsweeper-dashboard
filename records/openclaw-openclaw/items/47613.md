## Summary

Keep open: current main still has the no-callback Gateway reasoning gap, but this PR is not mergeable as the canonical fix because it is stale against newer visibility gates and does not address the current Gateway run path that still fails to forward the resolved reasoning level.

## What This Changes

The PR changes `src/agents/pi-embedded-subscribe.ts` so reasoning stream emission no longer requires `onReasoningStream`, while calling that callback only when present.

## Best Possible Solution

Land one canonical Gateway reasoning-stream implementation that forwards the resolved reasoning level, decouples the WebSocket `emitAgentEvent` path from channel callbacks, preserves thinking visibility and silent-turn gates, and updates focused tests plus docs/changelog.

## Reproduction Assessment

Yes. Source inspection shows a high-confidence path: Gateway `chat.send` supplies no `onReasoningStream`, `runAgentAttempt` does not forward the resolved reasoning level, and `subscribeEmbeddedPiSession` currently requires the callback before broadcasting live thinking events.

## Solution Assessment

No. The direction is plausible, but this PR is not the best mergeable fix because it removes the callback dependency without preserving current visibility gates and does not cover the current Gateway reasoning-level propagation gap.
