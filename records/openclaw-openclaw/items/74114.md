## Summary

Keep open. The PR targets a real concern from #72021, but the patch is not safe as-is because current main has tested and documented daily-ingestion and grounded-backfill promotion paths that can satisfy the default deep promotion gates without real recall hits. This needs a maintainer policy decision before changing the gate semantics.

## What This Changes

This PR changes memory-core short-term promotion ranking and application filters so `minRecallCount` is compared against `recallCount` alone while aggregate `signalCount` remains used for scoring.

## Best Possible Solution

Define the promotion contract explicitly: separate organic user recalls from automated daily/session evidence where #72021 needs protection, while preserving or deliberately retiring daily and grounded deep-promotion paths with matching tests and docs.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: with this PR diff applied, existing tests that expect `recallCount: 0, dailyCount: 3` and grounded-only `groundedCount: 3` candidates to pass default gates would fail the new recall-only threshold.

## Solution Assessment

No, not as-is. A recall-only gate is a plausible direction for #72021, but the maintainable fix must reconcile daily and grounded promotion semantics instead of contradicting current supported behavior.
