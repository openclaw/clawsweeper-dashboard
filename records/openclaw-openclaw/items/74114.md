## Summary

Keep open for maintainer review. The patch targets a real aggregate-gate concern, but it conflicts with current tests and docs that still treat daily/session and grounded evidence as valid promotion inputs, and the external PR still lacks after-fix real behavior proof.

## What This Changes

The PR changes memory-core short-term promotion ranking and apply filters so `minRecallCount` checks `recallCount` instead of aggregate `signalCount`.

## Best Possible Solution

Define one memory-core promotion signal contract for organic recalls, daily/session evidence, and grounded evidence, then update implementation, regression tests, docs, and changelog together.

## Reproduction Assessment

Yes, source-level reproduction is high-confidence. Current main aggregates recall, daily, and grounded counts for the gate, and applying this patch would make the current daily-only and grounded-only promotion tests fail; I did not run tests in this read-only review.

## Solution Assessment

No, not as-is. A recall-only gate addresses the reported false eligibility, but it silently disables current daily and grounded promotion paths unless maintainers intentionally retire or replace those contracts.
