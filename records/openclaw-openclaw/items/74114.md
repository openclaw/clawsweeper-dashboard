## Summary

Keep open: the PR targets a real signal-mixing concern, but the patch conflicts with current daily-only and grounded promotion behavior that is covered by tests and docs, so it needs maintainer policy review before merge.

## What This Changes

The PR changes memory-core short-term promotion so `minRecallCount` is checked against `recallCount` alone in ranking and apply filtering while aggregate `signalCount` remains used for scoring.

## Best Possible Solution

Define the promotion contract for organic recalls, daily ingestion, and grounded backfill, then update implementation, regression tests, docs, and changelog together so each supported lane is explicit.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: applying the PR diff would make the current daily-only and grounded-only promotion tests fail because those candidates have zero `recallCount` but rely on aggregate daily or grounded evidence.

## Solution Assessment

No. A recall-only gate is a plausible partial repair for the linked report, but this PR is not the best fix as-is because it silently retires current daily and grounded promotion paths instead of resolving the promotion policy contract.
