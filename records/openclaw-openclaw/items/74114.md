## Summary

Keep open. The PR targets a real memory-core promotion concern, but the current patch would remove supported daily-ingestion and grounded-backfill promotion paths that are covered by current tests and docs, so it needs maintainer policy review rather than conservative cleanup closure.

## What This Changes

This PR changes memory-core short-term promotion ranking and application so `minRecallCount` is checked against `recallCount` instead of aggregate `signalCount`, while leaving aggregate signal scoring in place.

## Best Possible Solution

Define the promotion contract for organic recalls, daily-ingestion evidence, and grounded backfill evidence, then update implementation, regression tests, docs, and changelog together.

## Reproduction Assessment

Yes. Source inspection shows applying the PR diff would make current daily-only and grounded-only promotion tests fail because both supported paths use zero organic `recallCount`.

## Solution Assessment

No, not as-is. A recall-only gate is a plausible part of a fix for #72021, but it contradicts existing tested and documented daily and grounded promotion lanes without retiring or replacing them.
