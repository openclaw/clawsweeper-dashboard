## Summary

Keep open. The PR targets a real memory-core promotion bug tracked by #72021, but the patch would remove currently tested and documented daily/grounded promotion paths from the default gate, so it is not ready for conservative merge or cleanup closure.

## What This Changes

The PR changes memory-core short-term promotion to compare `minRecallCount` against `recallCount` alone in ranking and apply filtering while keeping aggregate `signalCount` for scoring.

## Best Possible Solution

Define the promotion contract explicitly: separate organic user recalls from automated daily/session evidence where #72021 needs protection, while preserving or deliberately retiring daily and grounded deep-promotion paths with matching tests and docs.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: with this PR diff applied, current tests that expect `recallCount: 0, dailyCount: 3` and grounded-only `groundedCount: 3` candidates to pass default gates would fail the new recall-only threshold.

## Solution Assessment

No, not as-is. A recall-only gate is a plausible direction for #72021, but the maintainable fix must reconcile daily and grounded promotion semantics instead of silently contradicting supported behavior.
