## Summary

Keep open and do not merge as-is. The PR addresses the linked promotion-signal complaint, but the recall-only gate conflicts with current tested and documented daily and grounded promotion paths, so the remaining decision is promotion-policy design rather than conservative cleanup closure.

## What This Changes

The PR changes memory-core short-term promotion so `minRecallCount` is checked against `recallCount` alone in ranking and apply filtering while aggregate `signalCount` remains used for scoring.

## Best Possible Solution

Define the promotion contract for organic recalls versus daily and grounded evidence, then update implementation, regression tests, docs, and changelog together so supported promotion lanes remain explicit.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: applying the PR diff would make the current daily-only and grounded-only default-promotion tests fail because both have zero `recallCount` but rely on aggregate daily or grounded evidence.

## Solution Assessment

No, not as-is. A recall-only gate is a plausible partial fix for the linked issue, but this patch changes policy without reconciling the existing daily and grounded promotion contract.
