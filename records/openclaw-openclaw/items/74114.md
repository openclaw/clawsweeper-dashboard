## Summary

Keep open. The PR targets a real memory-core promotion bug tracked by #72021, but the patch removes currently tested and documented daily/grounded promotion paths from the default `minRecallCount` gate, so it needs maintainer policy resolution before merge.

## What This Changes

The PR changes memory-core short-term promotion so `minRecallCount` is checked against `recallCount` alone in ranking and apply filtering while aggregate `signalCount` remains the scoring denominator.

## Best Possible Solution

Define the promotion contract explicitly: separate organic user recalls from automated daily/session evidence where #72021 needs protection, while preserving or deliberately retiring the documented daily and grounded deep-promotion paths with matching tests and docs.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: with the PR diff applied, current tests that expect `recallCount: 0, dailyCount: 3` and grounded-only `groundedCount: 3` candidates to pass default gates would fail the new recall-only threshold.

## Solution Assessment

No, not as-is. The recall-only gate is a narrow fix for #72021, but the maintainable solution needs to reconcile daily and grounded signal semantics instead of silently removing those supported paths.
