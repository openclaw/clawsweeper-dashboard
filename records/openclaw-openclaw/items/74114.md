## What This Changes

This PR changes memory-core short-term promotion ranking and apply gates so `minRecallCount` uses `recallCount` instead of aggregate `signalCount` while leaving aggregate signal scoring intact.

## Best Possible Solution

Choose one explicit promotion-signal contract, then land rank/apply logic together with regression tests, CLI diagnostics, docs, and release notes so daily/session and grounded evidence are either preserved intentionally or retired deliberately.

## Reproduction Assessment

Yes. Source inspection shows current main gates on `recallCount + dailyCount + groundedCount`, and the linked issue comments include production candidates with `recallCount=0` and high `dailyCount` clearing the default ranking threshold; I did not run tests in this read-only review.

## Solution Assessment

Unclear as-is. A recall-only gate addresses the reported false eligibility, but the maintainable fix must explicitly preserve, replace, or retire the current daily and grounded promotion paths instead of leaving tests and docs in conflict.
