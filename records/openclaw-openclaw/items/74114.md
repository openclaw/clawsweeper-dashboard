## Summary

Keep open: the linked bug is real, but the existing ClawSweeper P2 concern still holds after rechecking current main because the patch would break tested and documented daily/grounded promotion behavior.

## What This Changes

The PR changes memory-core promotion ranking and application filters so `minRecallCount` uses `recallCount` only while scoring still uses aggregate `signalCount`.

## Best Possible Solution

Define the promotion contract explicitly: separate organic recalls from automated daily/session evidence where needed, while preserving or deliberately retiring daily and grounded deep-promotion paths with matching tests, docs, and changelog coverage.

## Reproduction Assessment

Yes. With this diff applied, `pnpm test extensions/memory-core/src/short-term-promotion.test.ts` would exercise current cases that expect zero-recall daily and grounded candidates to pass default gates, directly contradicting the new predicate.

## Solution Assessment

No, not as-is. A recall-only gate is a plausible direction for #72021, but the narrow patch changes supported promotion semantics without reconciling daily/grounded behavior or release-note coverage.
