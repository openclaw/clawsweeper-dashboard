## Summary

Keep open. The PR is a focused attempt to address the linked memory promotion bug, but as written it would remove current tested and documented daily/grounded promotion paths, and it still needs maintainer policy review plus contributor real-behavior proof before merge.

## What This Changes

The PR changes memory-core short-term promotion ranking and apply filters so `minRecallCount` checks `recallCount` instead of aggregate `signalCount`, while keeping aggregate `signalCount` for scoring.

## Best Possible Solution

Define the memory-core promotion signal contract for organic recalls, daily/session evidence, and grounded evidence, then update implementation, regression tests, docs, and changelog to match that decision.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: applying this diff would make the current daily-only and grounded-only default-promotion tests fail because they have zero `recallCount` and rely on daily or grounded evidence.

## Solution Assessment

No, not as-is. A recall-only gate addresses the reported false eligibility, but the maintainable fix must preserve or explicitly retire the current daily and grounded promotion paths instead of changing them silently.
