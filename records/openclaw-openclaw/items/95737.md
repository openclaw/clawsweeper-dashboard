## Summary

Keep open: current main and the latest release still lack a Teams conversation-id allowlist identity, and the open linked PR is the active candidate fix rather than a merged implementation.

## What This Changes

The issue reports that Microsoft Teams `groupAllowFrom` entries using group or channel conversation IDs fail to match runtime conversation IDs, causing allowlisted group messages to be dropped.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/95764, or an equivalent Teams plugin fix that adds a non-DM conversation-id allowlist alias with suffix normalization and regression coverage.

## Reproduction Assessment

Yes, source-reproducible: current main normalizes the Teams conversation id but never exposes it as a subject identity alias, while the shared matcher only checks declared identity fields. I did not run a live Teams tenant repro in this read-only review.

## Solution Assessment

Yes, the linked PR's direction is the narrowest maintainable fix because the Teams plugin owns platform identity normalization and can keep DM matching unchanged. The issue should remain open until that PR or an equivalent fix is merged.
