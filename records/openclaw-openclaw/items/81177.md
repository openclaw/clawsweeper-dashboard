Current main still lacks the requested WhatsApp outbound policy: normalized non-empty WhatsApp replies can proceed to durable/fallback delivery, current silent-token handling remains selective rather than robust suppress-if-present, and the requested explicit-send mode plus audit log are new security/product surfaces.

The issue requests stronger WhatsApp outbound gating so monitor auto-replies do not send non-empty text when NO_REPLY/no-reply intent, explicit-send mode, or audit logging should block delivery.
