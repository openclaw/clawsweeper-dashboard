Keep open: current main still drops the requested subagent lineage metadata from `openclaw sessions --json`, and the two related fixing PRs were closed without merging. The root cause is narrow and still present in the CLI row projection.

## What This Changes

Expose existing subagent lineage and lifecycle metadata from the session store in `openclaw sessions ... --json`.

## Best Possible Solution

Project the existing optional store/Gateway lineage fields into CLI JSON, preserve current table output and existing JSON keys, and add focused regression coverage for direct `--store` and configured-agent paths.

## Solution Assessment

Yes. The narrow maintainable solution is to project existing optional `SessionEntry` fields through the sessions JSON row, with tests; unrelated provider/runtime changes should stay out of scope.
