## Summary

Keep open: current main still reduces session store entries through a thin CLI display projection before JSON output, while the store type and Gateway session rows already carry the requested subagent lineage metadata. I found no duplicate or open fix PR for this exact CLI JSON surface.

## What This Changes

Expose optional subagent runtime lineage fields from `sessions.json` through `openclaw sessions ... --json`.

## Best Possible Solution

Expose the existing session store/Gateway lineage metadata as additive optional `openclaw sessions --json` fields while keeping text output and existing JSON fields stable.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a store row with `spawnedBy` or `sessionFile` is reduced by `toSessionDisplayRow` before `sessionsCommand` writes JSON, so those fields are dropped without needing a live deployment to see the cause.

## Solution Assessment

Yes. The narrow maintainable fix is to project the existing optional `SessionEntry` fields into the CLI JSON row, cover both `--store` and configured-agent paths, and avoid requiring consumers to read the persisted store directly.
