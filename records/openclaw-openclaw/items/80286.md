## Summary

Keep open: current main still drops the requested subagent lineage metadata from the CLI JSON projection, and the two linked fix PRs were closed unmerged. The store and Gateway row contracts already carry most of the fields, so this remains a narrow additive CLI JSON surface request.

## What This Changes

Expose existing subagent lineage and lifecycle metadata from session stores through `openclaw sessions ... --json`.

## Best Possible Solution

Project the existing optional store/Gateway lineage fields into CLI JSON while preserving existing keys and table output, then add focused regression coverage for explicit `--store` and configured-agent paths.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a store row with `spawnedBy` or `sessionFile` is reduced by `toSessionDisplayRow`, and `sessionsCommand` serializes that reduced row for `--json`.

## Solution Assessment

Yes. The narrow maintainable direction is to project existing optional `SessionEntry` fields into JSON without changing existing keys; a new opt-in flag is possible but not required by the current contracts.
