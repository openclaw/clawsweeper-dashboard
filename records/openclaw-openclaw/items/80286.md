## Summary

Keep open: current main still drops the requested subagent lineage metadata from `openclaw sessions --json`, and the only directly linked fix PRs were closed unmerged.

## What This Changes

The issue requests additive `openclaw sessions --json` fields for subagent runtime lineage and lifecycle metadata already present in session store entries.

## Best Possible Solution

Expose the existing session store lineage fields as optional properties in the CLI JSON row projection, preserve current table output and existing JSON keys, and add focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a store row with `spawnedBy` or `sessionFile` is reduced by `toSessionDisplayRow` before `sessionsCommand` writes JSON, and both current main and `v2026.5.12` omit those fields from the display row.

## Solution Assessment

Yes. The narrow maintainable solution is to project existing optional `SessionEntry` fields through `SessionDisplayRow` into JSON and test that `--store`/agent-backed `--json` output preserves them; no new persistence or Gateway contract is needed.
