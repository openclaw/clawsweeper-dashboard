## Summary

Keep open: current main still reduces `openclaw sessions --json` rows through a display projection that omits the requested lineage fields, and the linked closing PR is open but unmerged.

## What This Changes

Expose existing subagent lineage and lifecycle metadata from session stores through `openclaw sessions ... --json` so read-only consumers do not need direct store reads.

## Best Possible Solution

Review and land a sessions-only additive projection that exposes the existing optional store/Gateway metadata through `openclaw sessions --json`, with focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a store row with `spawnedBy` or `sessionFile` is converted through `toSessionDisplayRow`, which currently omits those fields before JSON output.

## Solution Assessment

Yes, the narrow maintainable direction is to project the existing optional session metadata into the CLI JSON row while preserving existing keys. The remaining maintainer choice is whether this should be default additive output or gated behind an opt-in flag.
