## Summary

Keep open: current main and the latest release still reduce `sessions --json` rows through a display projection that omits stored subagent lineage metadata, and the two related fix PRs were closed without merging.

## What This Changes

The issue requests additive `openclaw sessions --json` fields for subagent runtime parent metadata already present in session store and Gateway session rows.

## Best Possible Solution

Add optional projections for the existing store/Gateway lineage fields in `SessionDisplayRow`/`toSessionDisplayRow`, preserve table output and existing JSON keys, and cover both explicit `--store` and configured-agent JSON paths.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a store row with `spawnedBy` or `sessionFile` is reduced by `toSessionDisplayRow` before `sessionsCommand` writes JSON, so the field is dropped without needing a live store to see the cause.

## Solution Assessment

Yes. The narrow maintainable solution is an additive optional CLI JSON projection from the existing `SessionEntry` fields, with regression coverage and no Gateway/store schema change.
