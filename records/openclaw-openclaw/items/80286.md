## Summary

Keep open: current main still emits `openclaw sessions --json` from the reduced display row, so existing subagent lineage fields in the session store are not reachable through the CLI JSON surface. The related fix PR was closed unmerged, so it does not resolve the issue.

## What This Changes

Expose existing subagent runtime parent metadata from `SessionEntry` in `openclaw sessions --json` rows for configured-agent and explicit-store paths.

## Best Possible Solution

Add the existing store metadata as optional JSON row fields, keep text table output stable, and cover the projection with focused sessions command tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a store row with `spawnedBy` or `sessionFile` is loaded, reduced by `toSessionDisplayRow`, and then emitted through `toJsonSessionRow` without those fields.

## Solution Assessment

Yes. The narrowest maintainable solution is an additive pass-through from `SessionEntry` to CLI JSON rows with regression coverage; no store migration or Gateway change is needed.
