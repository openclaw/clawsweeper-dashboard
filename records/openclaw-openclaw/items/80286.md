## What This Changes

Expose existing subagent lineage and runtime metadata from stored session rows through the supported `openclaw sessions --json` CLI output.

## Best Possible Solution

Land one narrow, tested projection-path fix that carries existing optional `SessionEntry` lineage fields into CLI JSON, then close the issue after the merged implementation is on main.

## Reproduction Assessment

Yes. A store row with `spawnedBy` or `sessionFile` reaches `sessionsCommand`, but current main reduces it through `toSessionDisplayRow` before JSON emission, so the field cannot appear in `openclaw sessions --store <path> --json`.

## Solution Assessment

Yes, if maintainers accept the additive JSON contract. The narrowest implementation is to preserve already-persisted optional `SessionEntry` fields through the existing row adapter; an opt-in flag is the safer alternative if default JSON output must stay thin.
