## Summary

Keep open: current main still builds `sessions --json` output from the reduced display-row projection, so stored subagent lineage fields remain unavailable from the CLI; the two related fix PRs were closed unmerged.

## What This Changes

Expose existing stored subagent lineage, lifecycle, and transcript metadata as optional fields in `openclaw sessions --json` output.

## Best Possible Solution

Add optional pass-through fields from `SessionEntry` into `SessionDisplayRow`/JSON output, preserve table output and existing JSON keys, and cover the projection with focused sessions command tests.

## Solution Assessment

Yes. The narrow maintainable solution is to project existing optional store fields into CLI JSON without changing the store, Gateway row contract, or text table behavior.
