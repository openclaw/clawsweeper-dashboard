## Summary

Keep open: current main still drops the requested subagent lineage and lifecycle metadata from `openclaw sessions --json`, while the store and Gateway row contracts already carry most of it. The two linked fix PRs are both closed and unmerged, so there is no canonical landed fix to sweep this under.

## What This Changes

Request that `openclaw sessions --json` expose existing optional subagent lineage and lifecycle fields from session store rows.

## Best Possible Solution

Expose the existing store/Gateway subagent metadata as optional JSON fields while preserving current keys and table output.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a store row with `spawnedBy` or `sessionFile` is reduced by `toSessionDisplayRow` before `sessionsCommand` writes JSON, so those fields are dropped on current main.

## Solution Assessment

Yes. The narrow maintainable path is an additive optional projection in `SessionDisplayRow`/`toSessionDisplayRow` with focused JSON regression coverage; a new opt-in flag is possible but not necessary for compatibility because omitted fields stay optional.
