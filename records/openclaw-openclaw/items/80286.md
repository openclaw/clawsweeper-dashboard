## Summary

Keep open: current main still serializes `openclaw sessions --json` from a reduced display row that omits the requested subagent lineage fields, while the store and Gateway session rows already carry them. The two linked fix PRs are closed and unmerged, so the work remains actionable.

## What This Changes

Expose stored subagent runtime parent and lifecycle metadata as optional fields in `openclaw sessions --json`.

## Best Possible Solution

Add optional pass-through fields from `SessionEntry` into the sessions JSON row projection, preserve existing JSON keys and table output, and add focused regression coverage for explicit `--store` and configured-agent paths.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a store entry containing `spawnedBy` or `sessionFile` is reduced by `toSessionDisplayRow` before `sessionsCommand` writes JSON, and the reduced row does not contain those fields.

## Solution Assessment

Yes. The narrow maintainable solution is an additive optional projection from existing store fields into CLI JSON, with regression tests; a new flag is possible but not required by the current request.
