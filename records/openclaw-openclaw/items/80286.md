## Summary

Keep open: current main still drops the requested subagent lineage fields from `openclaw sessions ... --json`, and the two linked fix PRs are closed unmerged, so the user-facing gap remains.

## What This Changes

Expose existing subagent session lineage and lifecycle metadata from `SessionEntry` through `openclaw sessions ... --json`.

## Best Possible Solution

Add optional store-backed lineage fields to the CLI JSON projection, preserve existing JSON/table behavior, and cover both explicit `--store` and configured-agent paths with focused tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: a store row with `spawnedBy` or `sessionFile` is reduced by `toSessionDisplayRow` before `sessionsCommand` writes JSON, so those fields are dropped without needing a live deployment.

## Solution Assessment

Yes. The narrow maintainable solution is to project the existing optional `SessionEntry` fields into JSON rows while leaving table output and existing keys unchanged; a new flag is possible but not required by the issue.
