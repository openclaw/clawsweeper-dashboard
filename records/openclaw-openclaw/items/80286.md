## What This Changes

The issue requests additive `openclaw sessions --json` fields for subagent runtime lineage metadata already present in the session store.

## Best Possible Solution

Project the existing optional `SessionEntry` lineage and lifecycle fields into `sessions --json`, preserve existing keys and table output, and add focused regression coverage for store-path and configured-agent paths.

## Solution Assessment
