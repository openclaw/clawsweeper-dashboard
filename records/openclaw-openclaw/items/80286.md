## Summary

Keep open: current `main` still drops the requested subagent lineage/runtime fields from the CLI JSON projection, while the session store and Gateway row already carry them. The two linked fix PRs were closed unmerged, so there is still a focused additive CLI surface change to make.

## What This Changes

Expose optional subagent runtime parent metadata from stored session entries in `openclaw sessions --json`.

## Best Possible Solution

Add the existing optional lineage/lifecycle fields to the CLI JSON row projection, keep current keys and table output stable, and cover both direct `--store` and normal agent-store paths with focused regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: any store row containing `spawnedBy` or `sessionFile` is reduced by `toSessionDisplayRow` before `sessionsCommand` writes JSON, so those fields are omitted.

## Solution Assessment

Yes. The narrow maintainable fix is to project the existing optional `SessionEntry` fields through `SessionDisplayRow` for JSON output while preserving existing JSON fields and table formatting.
