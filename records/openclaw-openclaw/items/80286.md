## Summary

Keep open: current main and latest v2026.5.12 still serialize `openclaw sessions --json` from a reduced `SessionDisplayRow`, while the session store and Gateway row contracts already carry the requested subagent metadata. Two related fix PRs were opened for this issue, but both are closed and unmerged, so no canonical open implementation currently owns the remaining work.

## What This Changes

The issue requests adding optional subagent lineage and lifecycle fields from `SessionEntry` to `openclaw sessions --json` output.

## Best Possible Solution

Project the existing optional store metadata through the CLI JSON row, preserve existing keys and table output, and add focused regression coverage plus a small docs/changelog note for the API surface change.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: any store row with `spawnedBy` or `sessionFile` is reduced by `toSessionDisplayRow` before `sessionsCommand` writes JSON, so those fields are dropped from the supported CLI output.

## Solution Assessment

Yes. The narrow maintainable solution is to add the existing optional metadata fields to `SessionDisplayRow`/`toSessionDisplayRow` and assert they flow through JSON without changing table output or existing JSON keys.
