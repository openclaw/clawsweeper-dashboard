## Summary

Keep open. Current `main` and the latest release still route `openclaw sessions --json` through the minimal CLI display projection, while the store and Gateway row contracts already carry the requested lineage metadata.

## What This Changes

The issue requests additive session lineage and runtime metadata fields in `openclaw sessions --json` output.

## Best Possible Solution

Expose raw optional store-backed metadata in CLI JSON rows, preserve existing fields and text output, and document the additive JSON surface.

Yes, this is source-reproducible: a store row with `spawnedBy` or related fields flows through `toSessionDisplayRow`, whose type and mapper omit those fields before JSON output. I did not need to mutate the checkout to establish the path.

## Solution Assessment

Yes, the narrow maintainable fix is to extend the CLI JSON projection with optional raw store-backed fields and focused regression coverage. Avoid translating enum values or adding a flag unless a maintainer asks for an opt-in shape.
