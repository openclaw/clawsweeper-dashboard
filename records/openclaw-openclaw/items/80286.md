## Summary

Keep open: current main still drops the requested subagent lineage metadata from the CLI JSON projection even though the session store type and Gateway session rows already carry most of it. This is a narrow additive CLI/API surface request, not a stale or duplicate cleanup candidate.

## What This Changes

Expose optional subagent runtime lineage fields such as `spawnedBy`, `label`, `status`, `sessionFile`, timing, role, control scope, and spawned workspace metadata in `openclaw sessions ... --json`.

## Best Possible Solution

Add the existing store/Gateway lineage fields to the CLI JSON row projection as optional fields, preserve existing JSON keys and table output, and cover both `--store` and configured-agent paths with focused command tests.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: a session store row containing `spawnedBy` or `sessionFile` will be reduced by `toSessionDisplayRow` before `sessionsCommand` writes JSON. I did not execute the CLI because this review was read-only and avoided creating a throwaway store or test artifacts.

## Solution Assessment

Yes, the narrow maintainable fix is to project the existing optional fields from `SessionEntry` into CLI JSON while leaving existing fields unchanged. A new opt-in flag is possible, but the requested fields are already in the local store and Gateway row contract.
