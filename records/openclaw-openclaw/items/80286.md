## Summary

Keep open: current main still trims CLI JSON through `SessionDisplayRow`, which omits the requested subagent lineage and lifecycle fields even though `SessionEntry` and Gateway session rows already carry them. The two linked fix PRs are closed unmerged, and the merged ACP lineage PR only covers ACP `_meta`, so this remains a narrow additive CLI JSON projection item.

## What This Changes

Expose existing subagent lineage and lifecycle metadata from session store entries in `openclaw sessions ... --json` output.

## Best Possible Solution

Project the existing optional store/Gateway metadata into CLI JSON rows, keep table output unchanged, and add focused regression coverage for explicit-store and configured-agent paths.

## Reproduction Assessment

Yes. Source inspection shows a store row with `spawnedBy` or `sessionFile` is passed through `toSessionDisplayRow`, whose type and mapper omit those fields, before `sessionsCommand` serializes the JSON rows.

## Solution Assessment

Yes. The narrow maintainable fix is to add optional fields to `SessionDisplayRow` and `toSessionDisplayRow` and assert JSON output; no store migration or new runtime discovery is needed.
