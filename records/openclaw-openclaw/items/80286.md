## Summary

Keep open: current main still drops the requested subagent runtime metadata from the CLI JSON projection, and the only linked implementation PR was closed unmerged for queue policy rather than because the issue was solved.

## What This Changes

Expose existing subagent runtime parent and session metadata from the session store in `openclaw sessions ... --json` output.

## Best Possible Solution

Add the existing optional session-store lineage fields to the CLI JSON projection, keep table output and existing JSON keys stable, and cover both direct store and configured-agent paths with focused tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a `SessionEntry` containing `spawnedBy` or `sessionFile` is reduced by `toSessionDisplayRow` before JSON serialization, so those fields are absent from current `sessions --json` output.

## Solution Assessment

Yes. The narrow maintainable fix is to project the existing optional `SessionEntry` fields into CLI JSON while preserving existing fields and table rendering; a new flag is optional rather than necessary for compatibility.
