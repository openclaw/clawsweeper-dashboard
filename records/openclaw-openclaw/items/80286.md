## What This Changes

Expose existing subagent runtime parent and lifecycle metadata from `SessionEntry` in `openclaw sessions ... --json` output.

## Best Possible Solution

Add optional pass-through fields to the sessions JSON projection from the existing session store contract, preserve table output and existing JSON keys, and cover the shared projection with focused regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a store row with `spawnedBy` or `sessionFile` is reduced by `toSessionDisplayRow` before `toJsonSessionRow` writes the CLI JSON response.

## Solution Assessment

Yes. The narrow maintainable solution is to project the existing optional `SessionEntry` lineage/control/lifecycle fields through `SessionDisplayRow` for JSON output while leaving current keys and table formatting unchanged.
