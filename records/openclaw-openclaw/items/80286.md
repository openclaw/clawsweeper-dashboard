## Summary

Keep open: current main still omits the requested subagent lineage fields from `openclaw sessions --json`, while the session store and Gateway session row contracts already carry most of that metadata. The two exact fix PRs were closed unmerged, and the merged ACP lineage PR exposes `_meta` for ACP only, not the CLI sessions JSON output.

## What This Changes

Expose existing subagent/session runtime metadata from session store entries in the `openclaw sessions --json` CLI output.

## Best Possible Solution

Project the existing optional `SessionEntry` lineage fields through the CLI JSON row path, preserve existing keys and text table output, and add focused regression coverage for both explicit `--store` and configured-agent sessions.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a store row containing `spawnedBy` or `sessionFile` is reduced by `toSessionDisplayRow` before `sessionsCommand` serializes JSON, so the requested fields are dropped without needing a live deployment to see the code path.

## Solution Assessment

Yes. The narrow maintainable fix is to add optional pass-through fields to `SessionDisplayRow`/`toSessionDisplayRow` and cover the JSON output; adding a new flag is possible, but not necessary unless maintainers want path-like metadata gated.
