## Summary

Keep open: current main still drops the requested subagent lineage and lifecycle fields in the CLI display-row projection before writing `sessions --json`, while the session store and Gateway row contracts already carry the metadata. The two linked fix PRs are both closed and unmerged, so no canonical implementation currently resolves this issue.

## What This Changes

The issue requests that `openclaw sessions ... --json` expose optional subagent lineage and lifecycle fields from `SessionEntry` instead of dropping them in the CLI row projection.

## Best Possible Solution

Add the existing store/Gateway lineage fields to `SessionDisplayRow` and `toSessionDisplayRow`, preserve existing JSON keys and table output, document the additive JSON fields if needed, and cover both `--store` and configured-agent JSON paths with focused regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: a store entry containing `spawnedBy` or `sessionFile` is reduced by `toSessionDisplayRow` before `sessionsCommand` writes JSON, so the requested fields cannot appear.

## Solution Assessment

Yes. The narrow maintainable solution is an additive projection/test change that maps existing optional `SessionEntry` fields into CLI JSON without changing the store or Gateway contracts.
