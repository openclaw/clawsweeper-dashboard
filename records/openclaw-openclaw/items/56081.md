## Summary

Keep this PR open. Current main still does not show the answering provider/model in the per-response usage footer, and open issue #65477 tracks the same visible model-name request, so this remains a relevant implementation candidate rather than conservative cleanup.

## What This Changes

The PR adds optional provider/model inputs to `formatResponseUsageLine` and passes `runReplyAgent`'s resolved provider/model so the response usage footer can append a normalized model label.

## Best Possible Solution

Land a small rebased version of this change that reuses `formatProviderModelRef`, preserves the existing token/cache/cost/session footer behavior, adds focused response-footer tests, and updates the public token-use docs if the footer text changes.

## Reproduction Assessment

Yes. The display gap is reproducible by source inspection on current main: the formatter cannot receive provider/model data and the only call site does not pass it, even though `runReplyAgent` has already resolved those values.

## Solution Assessment

Yes, the proposed direction is the narrowest maintainable fix: feed the already-resolved provider/model into the existing usage-line formatter and normalize the label through the shared model-ref helper. The safer merge path is to rebase it over current main and add tests/docs rather than broaden the feature.
