## Summary

Keep open: the patch is a focused fix for a source-reproducible gateway session collision, but this external PR still lacks after-fix real gateway/CLI behavior proof and carries a session/message-delivery tradeoff maintainers should accept before merge.

## What This Changes

The PR propagates accepted gateway-run disconnects, routes that embedded fallback to a fresh `gateway-fallback-*` session, extends fallback metadata, and adds focused `agent-via-gateway` regression tests.

## Best Possible Solution

Land the fresh-session accepted-disconnect fallback after redacted real gateway proof and maintainer acceptance of extending the existing timeout-style two-run fallback boundary to this path.

## Reproduction Assessment

Yes at source level: current main records gateway acceptance, generic closed-transport fallback reuses the original local session options, and the gateway dedupe contract only protects gateway retries. I did not run a live WebSocket/proxy reproduction in this read-only review.

## Solution Assessment

Yes, the PR is the narrow maintainable fix shape: preserve never-accepted fallback, keep transient `1000` closes on gateway retry, and use the existing fresh-session timeout pattern for accepted non-transient disconnects. A no-local-result or wait-for-in-flight design would be a larger behavior change.
