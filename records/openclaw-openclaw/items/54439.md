## What This Changes

The issue reports a Control UI regression where navigating away from Chat and back can stop responding because a malformed Cron row reaches a render path that reads `payload.kind`.

## Best Possible Solution

Add a narrow Control UI/Cron robustness fix: validate or filter malformed `cron.list` rows before storing them in UI state, guard remaining Cron render helpers before reading `payload.kind`, and cover the malformed-row navigation case with a focused UI regression test.

## Reproduction Assessment

Yes. A source-level reproduction is clear: put a Cron row with a missing or non-object `payload` into `state.cronJobs` via `cron.list`, then render the app while returning to Chat; current `renderApp` will read `job.payload.kind`.

## Solution Assessment

Yes. The narrow maintainable fix is to harden the UI boundary against malformed or legacy Cron rows while preserving server-side create/update validation; broad core migration is not required for the reported UI crash.
