## Summary

Keep this issue open. Current main still has the reported Control UI crash shape: Cron rows are accepted into UI state without row validation, and the app render path still dereferences `job.payload.kind` while building model suggestions outside the Cron-only render branch. The two linked focused fixes, #54550 and #54552, are closed and unmerged, so there is no implemented-on-main close path.

## What This Changes

The issue reports that after opening another Control UI sidebar view such as Cron, clicking back to Chat fails because rendering dereferences an undefined `kind` property.

## Best Possible Solution

Implement a narrow Control UI robustness fix: validate or filter malformed `cron.list` rows before storing them in UI state, guard remaining Cron render helpers before reading `payload.kind`, and add regression coverage proving malformed Cron rows cannot block Chat navigation.
