## Summary

Keep open: current main still hardcodes durable exec approval actions in the macOS prompt and Control UI even though the gateway already computes and enforces allowed decisions. The PR is a focused fix with no blocking code-review findings, but the maintainer label, MEMBER-authored security-boundary surface, and non-green current checks make this a maintainer-review item rather than an automated close or repair lane.

## What This Changes

This PR threads `allowedDecisions` into macOS native exec approvals and the Control UI so unavailable approval buttons are hidden and cancellation does not send a disallowed denial.

## Best Possible Solution

Land the PR after maintainer review accepts the legacy fallback policy and current CI failures are cleared or confirmed unrelated, keeping gateway `allowedDecisions` as the source of truth for approval UIs.

## Reproduction Assessment

Yes. Current-main source shows native macOS and Control UI prompts still render `Always Allow` unconditionally while the gateway already computes and enforces disallowed `allow-always` for `ask=always`; I did not run the macOS UI locally.

## Solution Assessment

Yes, with one maintainer policy choice. Consuming the existing `allowedDecisions` contract in both prompt consumers is the narrow maintainable fix; if maintainers want stricter security posture, the remaining choice is to make nil or unknown native `ask` fall back to no durable approval.
