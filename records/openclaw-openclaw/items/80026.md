## Security Review

Status: needs_attention

Summary: The request would expose provider headers on narrower, potentially persisted surfaces, so validation and redaction policy must be explicit.

Concerns:

- **[medium] Define cron header redaction and bounds:** `src/cron/types.ts:171`
  - body: Job-scoped `payload.providers.*.request.headers` can carry credential-like values such as `Authorization` or `Cookie`, while cron payloads are persisted and listed. Any implementation should keep cron overrides headers-only and define validation, SecretRef, and redaction behavior for the new paths.
  - confidence: 0.84
- **[low] Register new credential paths deliberately:** `docs/reference/secretref-credential-surface.md:23`
  - body: The current documented credential surface includes only global `models.providers.*.request.headers.*`; adding agent or cron header paths should update the supported sensitive-path and documentation surface intentionally.
