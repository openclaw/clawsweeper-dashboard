## Security Review

Status: needs_attention

Summary: The request is security-sensitive because provider request headers may carry credentials or tenant-routing secrets, especially in mutable cron job payloads.

Concerns:

- **[medium] Keep cron request overrides headers-only and redacted:** `src/cron/types.ts:152`
  - body: Cron payloads are mutable scheduled-job data, so any implementation should keep them restricted to headers and avoid exposing auth, proxy, TLS, or private-network transport policy through job payloads.
  - confidence: 0.86
- **[medium] Preserve SecretRef handling for new header surfaces:** `docs/reference/secretref-credential-surface.md:27`
  - body: Existing provider request headers are treated as sensitive SecretRef-capable config; new agent or job header fields need matching schema hints, redaction, and docs so secrets are not leaked in config snapshots or diagnostics.
  - confidence: 0.82
