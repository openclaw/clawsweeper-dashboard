## Security Review

Status: needs_attention

Summary: The request introduces secret-bearing provider headers into per-agent and mutable cron-job configuration surfaces.

Concerns:

- **[medium] Keep cron headers bounded and redacted:** `src/cron/types.ts:152`
  - body: Job-scoped `payload.providers.*.request.headers` can carry auth-like values such as Authorization or Cookie, and cron payloads are persisted/listed. Any implementation should keep cron overrides headers-only, avoid exposing auth/proxy/TLS/private-network policy through job payloads, and define validation/redaction behavior.
