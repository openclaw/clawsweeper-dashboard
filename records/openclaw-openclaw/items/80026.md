## Security Review

Status: needs_attention

Summary: The request is security-sensitive because it would expose secret-bearing provider HTTP headers through new per-agent and mutable cron-job configuration surfaces.

Concerns:

- **[medium] Define cron header policy before exposing payload headers:** `src/cron/types.ts:152`
  - body: Cron payloads are persisted mutable job data, and arbitrary request headers can include auth-like values such as Authorization or Cookie. Any implementation should restrict the cron surface to headers only, preserve redaction/SecretRef behavior where applicable, and explicitly define whether sensitive header names are allowed.
