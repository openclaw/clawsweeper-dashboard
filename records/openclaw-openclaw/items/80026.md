## Security Review

Status: needs_attention

Summary: The request introduces secret-bearing provider headers into per-agent and mutable cron-job configuration surfaces.

Concerns:

- **[medium] Define cron header validation and redaction policy:** `src/cron/types.ts:171`
  - body: Job-scoped `payload.providers.*.request.headers` can carry auth-like values such as `Authorization` or `Cookie`, and cron payloads are persisted/listed.
