## Summary

Keep open: current main still lets the macOS Dashboard async path turn a native/stored device token into the browser Control UI `token` value, and the linked draft fix is the active closing candidate rather than a reason to close early.

## What This Changes

The issue asks for macOS Dashboard browser Control UI auth injection to use only configured shared gateway tokens and never native device tokens.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/98488 so the macOS Dashboard injects only configured shared gateway tokens into browser Control UI auth while native device tokens stay on native websocket clients.

## Reproduction Assessment

Yes for source reproduction: current main's Dashboard async path calls a helper that can return native/stored device tokens and then injects that value into the browser URL/native auth payload. No live macOS app run was attempted in this read-only review.

## Solution Assessment

Yes directionally: restricting browser Dashboard auth injection to configured shared gateway tokens is the narrow fix for the reported boundary. The open draft PR appears to implement that path, but it still needs normal review and validation before this issue is resolved.
