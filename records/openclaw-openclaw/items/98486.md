## What This Changes

The issue reports that the macOS Dashboard can inject a native device token into the browser Control UI as shared gateway authentication.

## Best Possible Solution

Keep the issue open while maintainers review and either land or reject the linked PR or an equivalent narrow fix that separates browser shared-auth injection from native websocket device-token auth.

## Reproduction Assessment

Yes from source inspection: current `main` passes `controlUiAutoAuthToken` into Dashboard URL generation, and that helper can return native device tokens. I did not run a live macOS Dashboard session in this read-only review.

## Solution Assessment

Yes: the narrow maintainable fix is to use only configured shared gateway tokens for browser Dashboard injection while keeping native device tokens scoped to native websocket clients.
