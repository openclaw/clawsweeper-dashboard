## What This Changes

## Best Possible Solution

Keep the MIME-inference fix, but route the normalizer through a single shared browser-safe media-core extension mapper, remove the release-owned changelog edit, and leave the gateway/download companion PRs to complete the browser Save As flow.

## Reproduction Assessment

Yes. Current main and v2026.6.6 both lack .xlsx/.docx/.pptx entries in the Control UI normalizer table, so an assistant `MEDIA:/tmp/openclaw/pricing.xlsx` path deterministically produces a document attachment with `mimeType` undefined.

## Solution Assessment

No. The target bug is valid, but the current patch is only a plausible fix because it extends a second MIME mapping table while media-core already owns extension MIME inference; the better fix centralizes that mapping for the UI and gateway paths.
