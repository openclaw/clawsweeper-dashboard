## Summary

Keep open: the PR adds UI MIME metadata for document extensions, but source inspection shows that metadata does not control the browser download response, and the PR lacks required real behavior proof.

## What This Changes

The PR extends Control UI chat attachment MIME inference for Office, OpenDocument, RTF, HTML/XML/YAML, and source-code extensions and adds normalizer tests.

## Best Possible Solution

Put the user-visible fix in the download/serving path with correct document `Content-Type` and `Content-Disposition`, and keep UI MIME metadata only if it is consumed or proven end-to-end.

## Reproduction Assessment

No high-confidence end-to-end reproduction was established. Source inspection shows current main omits UI `mimeType` for `.xlsx`, but the actual browser Save As response is governed by the serving path, which derives MIME independently.

## Solution Assessment

No. Adding UI MIME table entries is not the best complete fix for the reported Save As behavior because document links and response headers are controlled by the rendering/serving path, not the normalized `mimeType`.
