## Summary

Keep open. Current main still lacks the Mattermost read-only inspector needed for the reported status crash, but this PR head has a blocking credential-precedence regression and lacks after-fix real `openclaw status` proof.

## What This Changes

The PR adds a Mattermost read-only account inspector, wires it into config/status snapshots, and adds tests for unresolved SecretRef botToken status.

## Best Possible Solution

Land the Mattermost-owned read-only inspector/status fix after aligning credential precedence with strict runtime resolution, adding a SecretRef-plus-env regression test, and attaching redacted real CLI proof.

## Reproduction Assessment

Yes. The source path is clear: current main lacks Mattermost `inspectAccount`, the status inspection fallback can call strict `resolveAccount`, and `resolveMattermostAccount` throws on unresolved configured SecretRefs before env fallback; the linked issue includes the matching stack trace.

## Solution Assessment

No. The inspector/status design is the right maintainable direction, but this implementation lets env fallback override a configured-unavailable SecretRef in read-only metadata while runtime resolution remains strict.
