Keep open. This PR targets a real Mattermost status bug, but the current head still has a credential-precedence regression and lacks after-fix real CLI proof, so it should not merge yet.

## What This Changes

The PR adds Mattermost read-only account inspection/status metadata for unresolved SecretRef bot tokens and regression tests for the status snapshot path.

## Best Possible Solution

Land the Mattermost-owned read-only inspector/status fix after preserving configured SecretRef precedence over env fallback, adding a SecretRef-plus-env regression test, and attaching redacted real CLI proof.

## Reproduction Assessment

Yes. The linked issue gives a concrete Mattermost SecretRef config and stack trace, and current main shows the fallback from missing `inspectAccount` into strict `resolveMattermostAccount`; I did not run a live `openclaw status` command in this read-only review.

## Solution Assessment

No. The inspector/status design is the right maintainable direction, but this implementation lets env fallback override configured-unavailable SecretRef metadata while runtime resolution remains strict.
