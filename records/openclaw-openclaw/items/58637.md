## Summary

Keep open: the problem is still real and the branch is useful, but the proposed fallback still skips transcript reads for model-only recovery and the UI changes need current-main i18n/model-id semantics plus real behavior proof.

## What This Changes

The PR adds a Model column to the Control UI sessions table and changes buildGatewaySessionRow to prefer transcript model metadata when stored runtime and override model metadata are absent.

## Best Possible Solution

Add an i18n-backed model column that shows the exact effective model id and update session-row model recovery to read transcript data when runtime and override model identity are absent, even if usage fields are already persisted, with focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction: a session with complete persisted usage fields, no entry.model, no entry.modelOverride, and transcript assistant model metadata will not trigger the branch's transcript read for model-only recovery.

## Solution Assessment

No. The direction is useful, but the current patch is not the best fix until transcript reads are decoupled from missing usage data and the table column follows current i18n and model-id display semantics.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Load the transcript for model-only fallback:** `src/gateway/session-utils.ts:1359-1363`
  - body: The new fallback still depends on transcriptUsage, but transcriptUsage is only populated when token/context/cost data is missing. A session with complete persisted usage fields but no stored runtime model and no override will still never read the transcript, so the dashboard can keep showing the configured default instead of the model actually used.
  - confidence: 0.94
- **[P3] Preserve nested model ids in the table:** `ui/src/ui/views/sessions.ts:126-130`
  - body: row.model is already the model id returned by sessions.list, and OpenClaw preserves nested ids such as moonshotai/kimi-k2.5 under provider overrides. Splitting every slash makes the visible cell show only kimi-k2.5, hiding useful model namespace; render row.model as-is or only strip a matching provider prefix from a full provider/model ref.
  - confidence: 0.86
- **[P3] Use localized text for the Model header:** `ui/src/ui/views/sessions.ts:403`
  - body: The sessions view renders existing table headers through sessionsView.* locale strings, and current main already has a sessionsView.model key. Adding a literal Model header leaves the new column outside the Control UI localization path.
  - confidence: 0.84
