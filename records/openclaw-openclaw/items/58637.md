## Summary

Keep open: the PR addresses a real remaining dashboard/session-row gap, but the current patch is still incomplete and lacks required after-fix real behavior proof.

## What This Changes

The PR adds a sessions-table Model column and changes gateway session-row model fallback to prefer transcript model metadata when no runtime or override model is stored.

## Best Possible Solution

Rebase and implement bounded model-only transcript recovery that preserves lightweight list rows, then add a localized table column showing the exact effective model ID.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a row with persisted usage fields, no runtime model, no override, and transcript model metadata will not trigger the PR's transcript read for model recovery.

## Solution Assessment

No. The direction is useful, but the patch should decouple model-only transcript recovery from usage fallback and follow current exact-ID and localization conventions.
