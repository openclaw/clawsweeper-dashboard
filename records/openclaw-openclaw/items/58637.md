## Summary

Keep open: current main still lacks the requested Sessions table model column, and the PR's backend fallback still misses sessions whose usage is already persisted but whose runtime model is absent. The branch also lacks after-fix real behavior proof.

## What This Changes

The PR adds a Sessions dashboard Model column and changes gateway session-row model selection to prefer transcript model metadata when stored runtime and override model fields are absent.

## Best Possible Solution

Rebase the PR, decouple transcript model recovery from usage fallback while preserving bounded list behavior, and render a localized table column with the exact effective model ID.

## Reproduction Assessment

Yes, by source inspection: a session row with persisted usage fields, no runtime model, no override, and transcript model metadata skips the PR's transcript read for model recovery. I did not run a live dashboard repro in this read-only review.

## Solution Assessment

No. The direction is useful, but the patch should load transcript metadata for model-only recovery independently from usage fallback and should follow current exact-ID and localization conventions.
