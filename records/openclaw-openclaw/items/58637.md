## Summary

Keep open: the direction is useful, but the patch still misses a model-only transcript fallback case, is dirty against current main, and lacks required after-fix real behavior proof.

## What This Changes

This PR adds a Model column to the Control UI sessions table and changes `buildGatewaySessionRow` to prefer transcript model metadata when no runtime or override model is recorded.

## Best Possible Solution

Rebase onto current main, add an i18n-backed sessions-table model column that renders the exact effective model ID, and recover transcript model metadata through a bounded path that preserves lightweight `sessions.list` behavior.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a session with complete persisted usage fields, no stored runtime model, no override, and transcript assistant model metadata will not trigger the PR's transcript read for model-only recovery.

## Solution Assessment

No. The direction is useful, but the current patch is not the best fix until model-only transcript recovery is decoupled from usage fallback and the table column follows current exact-ID and localization conventions.

- **[P2] Load the transcript for model-only fallback:** `src/gateway/session-utils.ts:1359-1363`
  - body: `useTranscriptFallback` still depends on `transcriptUsage`, but `transcriptUsage` is only populated when token/context/cost data is missing.   - confidence: 0.86

  - body: The current Sessions view renders table headers through `sessionsView.*` locale strings, and current main already has `sessionsView.model`. Adding a literal `Model` header leaves the new column outside the Control UI localization path after rebase.
  - confidence: 0.84
