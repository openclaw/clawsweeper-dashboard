## Summary

Keep open. Current main still lacks the sessions-table model column and still cannot recover transcript model identity for no-runtime/no-override sessions when usage data is already persisted, but this PR is not ready to merge because its transcript fallback remains gated on missing usage fields and the UI side needs a few current-main adjustments.

## What This Changes

The PR adds a Model column to the Control UI sessions table and changes gateway session-row model fallback to prefer transcript model data over the configured default when stored runtime and override model metadata are absent.

## Best Possible Solution

Carry the product direction forward on current main: add an i18n-backed model column that preserves the exact effective model id, decouple transcript model recovery from missing usage data while preserving runtime, override, and live-subagent precedence, and cover both paths with focused gateway/UI tests plus a changelog fix entry.

## Reproduction Assessment

Yes. A high-confidence static reproduction is a session entry with persisted token/context/cost data, no entry.model, no entry.modelOverride, and a transcript assistant usage snapshot with model data; the proposed fallback would not read that transcript because the usage-needed gates are false.

## Solution Assessment

No. The requested behavior is right, but the current patch is not the best fix until transcript reads are triggered for model-only recovery and the table column follows current Control UI i18n and model-id display semantics.
