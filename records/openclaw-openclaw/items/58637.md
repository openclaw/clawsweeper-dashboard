## Summary

Keep open: the PR is aimed at a useful dashboard/session-model problem, but the current head still misses the model-only transcript fallback case and has two UI integration issues. The external contributor proof gate is also unmet because the PR body and comments only list tests and PAL checks.

## What This Changes

The PR adds a Model column to the Control UI sessions table and changes `buildGatewaySessionRow` to prefer transcript model metadata when runtime and override model fields are absent.

## Best Possible Solution

Rebase onto current main, add an i18n-backed sessions-table model column that renders the exact effective model ID, and recover transcript model metadata only through a bounded path that preserves the lightweight `sessions.list` performance contract.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a session with complete persisted usage fields, no stored runtime model, no override, and transcript assistant model metadata will not trigger the PR's transcript read for model-only recovery.

## Solution Assessment

No. The direction is useful, but the proposed patch is not the best fix until transcript model recovery is decoupled from missing usage data, the table preserves exact model IDs, and the header follows current localization conventions.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Load the transcript for model-only fallback:** `src/gateway/session-utils.ts:1359-1363`
  - body: The new fallback still depends on `transcriptUsage`, but `transcriptUsage` is only populated when token/context/cost data is missing. A session with complete persisted usage fields but no stored runtime model and no override will still never read the transcript, so the dashboard can keep showing the configured default instead of the model actually used.
  - confidence: 0.94
- **[P3] Preserve nested model IDs in the table:** `ui/src/ui/views/sessions.ts:126-130`
  - body: `row.model` can already be a nested model ID such as `moonshotai/kimi-k2.5` under a provider override. Splitting every slash makes the visible cell show only `kimi-k2.5`, hiding useful model namespace; render the effective model as-is or strip only a known provider prefix.
  - confidence: 0.86
- **[P3] Use localized text for the Model header:** `ui/src/ui/views/sessions.ts:403`
  - body: The sessions view renders table headers through `sessionsView.*` locale strings, and current main already has `sessionsView.model`. Adding a literal `Model` header leaves the new column outside the Control UI localization path.
  - confidence: 0.84
