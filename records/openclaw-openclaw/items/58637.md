## Summary

Keep open: this PR is useful but not merge-ready because the backend fallback still misses model-only recovery, the UI column needs to match current Control UI conventions, and the external real-behavior proof gate is unmet.

## What This Changes

The PR adds a Model column to the Control UI sessions table and changes `buildGatewaySessionRow` to prefer transcript model metadata when runtime and override model fields are absent.

## Best Possible Solution

Rebase onto current main, add an i18n-backed sessions-table model column that renders the exact effective model ID, and recover transcript model metadata through a bounded path that preserves the lightweight `sessions.list` contract.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a session with complete persisted usage fields, no stored runtime model, no override, and transcript assistant model metadata will not trigger the PR's transcript read for model-only recovery.

## Solution Assessment

No. The direction is useful, but the current patch is not the best fix until model-only transcript recovery is decoupled from usage fallback and the table column follows current exact-ID and localization conventions.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Load the transcript for model-only fallback:** `src/gateway/session-utils.ts:1359-1363`
  - body: The new fallback still depends on `transcriptUsage`, but `transcriptUsage` is only populated when token/context/cost data is missing. A session with complete persisted usage fields but no runtime model and no override will never read the transcript, so the dashboard can still show the configured default instead of the model actually used.
  - confidence: 0.94
- **[P3] Preserve nested model IDs in the table:** `ui/src/ui/views/sessions.ts:126-130`
  - body: `row.model` can already be a nested model ID such as `moonshotai/kimi-k2.5` under a provider override. Splitting every slash makes the visible cell show only the final segment, hiding useful model namespace; render the effective model as-is or strip only a known provider prefix.
  - confidence: 0.86
- **[P3] Use localized text for the Model header:** `ui/src/ui/views/sessions.ts:403`
  - body: The sessions view renders table headers through `sessionsView.*` locale strings, and current main already has `sessionsView.model`. Adding a literal `Model` header leaves the new column outside the Control UI localization path.
  - confidence: 0.84
