## Summary

Keep open: current main still lacks the sessions-table model column, and the PR addresses a real dashboard/session-row gap, but the branch is incomplete because model recovery still depends on usage fallback and the external PR has no real behavior proof.

## What This Changes

The branch adds a Model column to the sessions dashboard table and changes session-row model identity fallback to prefer transcript metadata when stored runtime or override model data is absent.

## Best Possible Solution

Rebase the PR into the current sessions table, add bounded model-only transcript recovery that preserves lightweight list behavior, and render a localized table column with the exact effective model ID.

## Reproduction Assessment

Yes, by source inspection: a session row with persisted usage fields, no runtime model, no override, and transcript model metadata skips the PR's transcript read for model recovery. I did not run a live dashboard repro in this read-only review.

## Solution Assessment

No. The direction is useful, but the patch should decouple model-only transcript recovery from usage fallback and follow current exact-ID and localization conventions.
