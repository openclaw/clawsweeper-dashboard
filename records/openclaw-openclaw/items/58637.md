## Summary

Keep open: the PR is aimed at a real dashboard/session metadata problem, but it is not merge-ready because it lacks real behavior proof, is conflicting with current main, and the backend fallback still misses the model-only case when usage data is already persisted.

## What This Changes

This PR adds a Model column to the Control UI Sessions table and changes gateway session-row model identity fallback to prefer transcript metadata when no stored runtime model or override is present.

## Best Possible Solution

Rebase onto current main, add a localized exact-model table column, and recover transcript model metadata only through a bounded path that preserves the lightweight list behavior.

## Reproduction Assessment

Yes, by source inspection: the PR only reads transcript usage when token/context/cost fallback is needed, so a row with persisted usage, no runtime model, no override, and transcript model metadata still misses model recovery. I did not run a live dashboard repro in this read-only review.

## Solution Assessment

No. The direction is useful, but the patch should decouple model recovery from usage fallback, preserve exact model IDs, follow the current localization/table structure, and include real behavior proof.
