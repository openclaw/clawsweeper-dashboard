## Summary

Keep open: the PR addresses a real sessions-model display gap, but the current patch still misses the model-only transcript fallback case it is meant to fix and has no after-fix real behavior proof.

## What This Changes

The PR adds a Model column to the Control UI sessions table and changes buildGatewaySessionRow to prefer transcript model metadata when stored runtime and override model metadata are absent.

## Best Possible Solution

Add an i18n-backed sessions-table model column that shows the exact effective model id, and update session-row model recovery to read transcript metadata when runtime and override identity are absent even if usage fields are already persisted.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a session with complete persisted usage fields, no entry.model, no entry.modelOverride, and transcript assistant model metadata will not trigger the branch's transcript read for model-only recovery.

## Solution Assessment

No. The direction is useful, but the patch is not the best fix until transcript reads are decoupled from missing usage data and the table column follows current i18n and model-id display semantics.

Full review comments:

- **[P2] Load the transcript for model-only fallback:** `src/gateway/session-utils.ts:1359-1363`
  - body: The new fallback still depends on transcriptUsage, but transcriptUsage is only populated when token/context/cost data is missing.   - body: The sessions view renders current table headers through sessionsView.* locale strings, and current main already has a sessionsView.model key. Adding a literal Model header leaves the new column outside the Control UI localization path.
