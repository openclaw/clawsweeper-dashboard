## Summary

Keep open: current main still lacks an absolute token-count memory-flush trigger, and adding one would be a new public config/runtime feature. The only exact duplicate found was closed in favor of this issue, while broader memory-durability work remains adjacent rather than superseding.

## What This Changes

Request adds an optional `absoluteThresholdTokens` sibling under `agents.defaults.compaction.memoryFlush` so memory flush can trigger at a model-independent token count.

## Best Possible Solution

Approve an additive absolute-threshold config contract, then wire it through schema/types/help, `MemoryFlushPlan`, token/preflight gates, docs, and focused tests while preserving existing `softThresholdTokens` behavior.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/config request. Source inspection verifies current main computes the token gate from context window minus reserve and soft threshold, with no absolute threshold field.

## Solution Assessment

Unclear until maintainer API/product review. The sibling field is a plausible narrow design, but maintainers need to define interactions with reserve floor, the existing relative threshold, transcript-byte forced flush, and config docs.
