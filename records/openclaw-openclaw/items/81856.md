## Summary

Keep open: current main still lacks an absolute token trigger, and the request is a public compaction config feature that needs maintainer API/product judgment before implementation.

## What This Changes

Request adds an optional `agents.defaults.compaction.memoryFlush.absoluteThresholdTokens` trigger that fires by absolute token usage rather than distance from the model context limit.

## Best Possible Solution

Choose a backward-compatible compaction threshold API, then wire the accepted absolute-token trigger through config types/schema/help, `MemoryFlushPlan`, preflight and runtime gates, docs, and focused tests.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request for a new config mode. Source inspection verifies current main is context-window-relative and has no absolute threshold field.

## Solution Assessment

Unclear until maintainer product/API review. The proposed sibling field is plausible and narrow, but it should be reconciled with active compaction-threshold scaling and share-based config work before changing the public contract.
