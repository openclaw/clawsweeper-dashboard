## Summary

Keep open: current main still only has the context-window-relative `softThresholdTokens`/transcript-size triggers, and the requested absolute token trigger would add a new public config option and runtime semantics.

## What This Changes

The issue requests a new `agents.defaults.compaction.memoryFlush.absoluteThresholdTokens` setting that triggers compaction or memory flush at a fixed conversation token count regardless of model context window.

## Best Possible Solution

Add an explicitly additive absolute threshold only after choosing the config API, then wire it through the schema, `MemoryFlushPlan`, token/preflight gates, docs, and focused regression tests while preserving existing `softThresholdTokens` behavior.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request for a new config mode. Source inspection does verify the current implementation is context-window-relative and has no absolute threshold field.

## Solution Assessment

Unclear until maintainer product/API review. The proposed sibling field is a plausible narrow additive design, but it should be weighed against existing context-window-relative threshold work before changing the public config contract.
