## Summary

Keep open: current main still lacks an absolute token-count trigger for memory flush, and adding one would be a new public config/API behavior rather than a cleanup close.

## What This Changes

The issue requests an additive `agents.defaults.compaction.memoryFlush.absoluteThresholdTokens` setting that triggers memory flush at an absolute conversation token count independent of the model context window.

## Best Possible Solution

Approve an additive config/API contract, then wire the absolute threshold through schema, types, memory-core plan resolution, runtime token gates, docs/help text, and focused regression coverage while preserving existing relative behavior.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a request for a new config mode. Source inspection gives high confidence that current main only supports the relative token-distance and transcript-byte triggers.

## Solution Assessment

Unclear until maintainer product/API review. The proposed sibling field is plausible and narrow, but the repository should first settle how it interacts with scaled relative defaults and plugin-owned memory-flush plans.
