## Summary

Keep open: current main still only supports the relative `softThresholdTokens` memory-flush trigger plus transcript-byte forcing; the requested absolute token trigger is a new public compaction config feature that needs maintainer API judgment.

## What This Changes

The issue requests an `absoluteThresholdTokens` memory-flush setting that triggers compaction-related memory flushing at a fixed token usage count independent of the active model context window.

## Best Possible Solution

Choose a backward-compatible compaction threshold API, then wire the accepted absolute-token trigger through schema, types/help, `MemoryFlushPlan`, preflight/runtime gates, docs, and focused tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a request for a new public config mode. Source inspection verifies current main has only relative `softThresholdTokens` and transcript-byte triggers.

## Solution Assessment

Unclear until maintainer API review. The proposed sibling field is plausible, but the maintainable fix should first settle precedence and overlap with existing compaction threshold controls.
