## Summary

Keep open: current main and the latest v2026.6.10 release still lack a model-independent absolute token trigger for memory flush, while the open related percentage/default-scaling work overlaps but does not replace this fixed total-token ceiling request.

## What This Changes

The issue requests a new `agents.defaults.compaction.memoryFlush.absoluteThresholdTokens` config field that triggers memory flush when total session usage reaches a fixed token count independent of the active model context window.

## Best Possible Solution

Maintainers should choose one public memory-flush threshold contract, then implement it consistently across schema/types, memory-core planning, runtime gates, docs/help metadata, and focused regression coverage.

## Reproduction Assessment

Yes at source level: current schema/types lack an absolute threshold field, and the runtime gate still derives its token threshold from context-window distance. I did not live-reproduce the reporter's 190k-token session scenario.

## Solution Assessment

Unclear as proposed: an additive absolute field is plausible, but maintainers should decide whether OpenClaw wants absolute ceilings, percentage/window-aware thresholds, model-aware defaults, or a combined contract before implementation.
