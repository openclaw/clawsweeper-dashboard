Keep open: current main still lacks a model-independent absolute token trigger, and the request would add a public compaction config option that needs maintainer API/product judgment before implementation.

## What This Changes

Request adds a model-independent `agents.defaults.compaction.memoryFlush.absoluteThresholdTokens` setting that triggers pre-compaction memory flush at an absolute token count.

## Best Possible Solution

Define a backward-compatible threshold API, then wire the accepted absolute-token gate through config schema/types/help, `MemoryFlushPlan`, preflight/runtime gating, docs, and focused tests.

## Solution Assessment

Unclear until maintainer API/product review. The proposed sibling field is plausible and narrow, but the public contract should be reconciled with existing relative and transcript-size gates before implementation.
