## Summary

Keep open: current main and v2026.6.8 still lack an absolute total-token memory-flush trigger, and the remaining decision is a public compaction config/API shape that overlaps but is not replaced by the open percentage-threshold and default-scaling work.

## What This Changes

The issue requests a new `agents.defaults.compaction.memoryFlush.absoluteThresholdTokens` setting that triggers memory flush at a fixed total token count independent of the active model context window.

## Best Possible Solution

Maintainers should choose one public memory-flush threshold contract, then implement the accepted semantics consistently across schema/types, memory-core planning, runtime preflight and flush gates, docs/help metadata, and focused regression tests.

## Reproduction Assessment

Yes, source-level: current main and v2026.6.8 lack `absoluteThresholdTokens`, and the token gate still derives the trigger from context-window distance. I did not live-reproduce the long-session cost scenario because the missing config surface and relative formula are already clear from source.

## Solution Assessment

Unclear as proposed: an additive absolute field is plausible, but maintainers should decide whether OpenClaw wants absolute ceilings, percentage/window-aware thresholds, model-aware defaults, or a combined contract before implementation.
