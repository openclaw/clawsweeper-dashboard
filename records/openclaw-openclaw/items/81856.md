## Summary

Keep open: current main still lacks an absolute token-count trigger for memory flush compaction, and the request adds a new public compaction config contract that needs maintainer API/product judgment rather than cleanup closure.

## What This Changes

Request adds `agents.defaults.compaction.memoryFlush.absoluteThresholdTokens` so memory flush can trigger at an absolute token count independent of the active model context window.

## Best Possible Solution

Choose one backward-compatible compaction threshold API, then wire the accepted absolute-token behavior through config types/schema/help, `MemoryFlushPlan`, preflight/runtime gates, docs, and focused regression tests.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request for a new public config mode. Source inspection gives high confidence current main remains context-window-relative and has no absolute threshold field.

## Solution Assessment

Unclear until maintainer API/product review. The proposed sibling field is plausible and narrow, but it should be reconciled with active share-based and default-scaling compaction work before changing the public contract.
