## Summary

Keep open: current main still exposes only the context-window-relative memory-flush token threshold plus transcript-size forcing, while this asks for a new absolute token trigger and public config semantics. The later duplicate was closed in favor of this canonical issue, and the remaining work needs maintainer API/product approval before implementation.

## What This Changes

Add a model-independent `agents.defaults.compaction.memoryFlush.absoluteThresholdTokens` option that triggers memory flush/compaction at an absolute token count.

## Best Possible Solution

Approve an additive absolute-threshold contract, then wire it through the schema, public types, `MemoryFlushPlan`, runtime token/preflight gates, help/docs, and focused regression tests while preserving existing `softThresholdTokens` behavior.

## Reproduction Assessment
