## Summary

Keep open: current main still only supports the context-window-relative `softThresholdTokens` gate plus transcript-byte forcing, while the requested absolute token trigger would add a new public config option and runtime semantics.

## What This Changes

Request an additive `absoluteThresholdTokens` memory-flush config that triggers at a model-independent token usage count.

## Best Possible Solution

Add an explicitly additive absolute threshold only after choosing the public config API, then wire schema, types, `MemoryFlushPlan`, memory-flush/preflight gates, docs, and focused regression tests while preserving existing relative behavior.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request for a new config mode. Source inspection does show current main has no absolute threshold field and evaluates existing token gates relative to the context window.

## Solution Assessment

Unclear pending maintainer API review. The sibling field is plausible and narrow, but it should be reconciled with `forceFlushTranscriptBytes`, existing relative semantics, and the broader memory durability config discussion.
