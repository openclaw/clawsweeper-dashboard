## What This Changes

This PR filters unsupported shared slash commands out of local TUI autocomplete/help, blocks them before `sendMessage(raw)`, and adds unit plus local PTY coverage.

## Best Possible Solution

Land the bounded TUI-local guard if maintainer review agrees, keeping gateway-connected command recovery unchanged and preserving the focused local PTY regression coverage.

## Reproduction Assessment

Yes, source-reproducible: current main advertises shared commands in the TUI, unhandled local commands fall through to `sendMessage(raw)`, and the embedded backend sends that text into `agentCommandFromIngress`. I did not run the PTY test locally because this review is read-only, but the PR body supplies after-fix PTY output.

## Solution Assessment

Yes, this is the narrowest maintainable fix I found: the TUI owns local autocomplete/help and local command fallback, while gateway-connected mode already has command recovery. Reimplementing gateway command dispatch inside the embedded backend would be broader and riskier.
