## Summary

Keep open: the patch is a useful start for the linked TUI local-mode bug, but it only blocks the two named shared commands while the TUI still advertises other shared commands that can fall through to the local embedded model path, and the PR still needs real local TUI proof.

## What This Changes

The branch adds local-mode handling for TUI `/status` and `/compact` plus a regression test asserting those two commands do not call `sendChat`.

## Best Possible Solution

Centralize local-mode handling for advertised shared commands: either route them through a real shared/local command adapter or block/remove every unsupported advertised command from local TUI, with focused tests and PTY/local TUI proof.

## Reproduction Assessment

Yes from source inspection, not a live run: current main routes slash submissions to `handleCommand`, sends unhandled names through `sendMessage`, and local embedded `sendChat` invokes `agentCommandFromIngress`. That establishes the reported fallthrough path for local mode.

## Solution Assessment

No. Blocking `/status` and `/compact` is a plausible partial fix for the named examples, but the best fix also covers or hides the other advertised shared commands that lack local execution.
