Keep open for normal maintainer merge; I found no blocking findings. Current main still has the local TUI fallthrough, and this PR now covers the command handler, local autocomplete/help, unit coverage, and a real local PTY proof showing blocked commands do not reach the model path.

## What This Changes

The PR blocks unsupported shared slash commands in local TUI mode, removes them from local autocomplete/help, and adds unit plus local PTY coverage for the behavior.

## Best Possible Solution

Keep this as a local TUI boundary fix: unsupported shared commands should be hidden or answered locally, while supported local commands, unknown slash text, and gateway forwarding keep their current behavior.

## Reproduction Assessment

Yes from source inspection: current main routes slash input through `handleCommand`, the default branch sends `raw`, and local `sendChat` reaches `agentCommandFromIngress`. I did not run current main because this review is read-only.

## Solution Assessment

Yes; this is the narrow maintainable fix for the allowed issue shape because TUI owns local command presentation and fallback, while a shared command execution adapter would be a broader follow-up rather than necessary for this bug.
