## Summary

Keep open: current main does not implement the requested run-scoped cross-tool error detector, and this PR is useful but still has a blocking ordering bug plus a duplicate unrelated changelog entry before it can merge.

## What This Changes

The PR adds a runId-scoped consecutive error loop detector, the `tools.loopDetection.consecutiveErrorThreshold` config/schema/docs surface, diagnostics type updates, tests, and a changelog entry.

## Best Possible Solution

Keep the narrow runId-scoped detector design, but make the critical consecutive-error check win before warning-only loop classifications and clean the changelog before merge.

## Reproduction Assessment

Yes. Source inspection of the PR head shows alternating same-run failing A/B calls can hit a `ping_pong` warning before `consecutive_errors` is evaluated, and the current hook path only vetoes critical loop results.

## Solution Assessment

No, not as currently ordered. The detector is a maintainable narrow direction, but the critical check must run before warning-only returns and the unrelated duplicate changelog entry should be removed.
