## What This Changes

The PR adds a runId-scoped consecutive tool-error loop detector, `tools.loopDetection.consecutiveErrorThreshold` config/docs/schema coverage, diagnostics typing, changelog, and regression tests.

## Best Possible Solution

Land a repaired version that evaluates the critical consecutive-error guard before warning-only returns, preserves loop-history recording for non-tool-loop vetoes, validates the effective history window, and keeps docs/changelog aligned.

## Reproduction Assessment

Yes. ## Solution Assessment

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Check consecutive errors before warning-only returns:** `src/agents/tool-loop-detection.ts:635-636`
  - body: The new critical check still runs after the warning-only ping-pong return. With the default threshold, ten same-run alternating failing A/B calls can return a `ping_pong` warning before `consecutive_errors` is evaluated, and the hook path only blocks critical loop results, so this guard does not stop that cascade at its threshold.
  - confidence: 0.92
- **[P2] Keep non-loop vetoes in loop history:** `src/agents/pi-tools.before-tool-call.ts:643`
  - body: This removes outcome recording for every veto. That fixes sticky tool-loop blocks, but it also stops repeated `plugin-before-tool-call` policy blocks from contributing stable blocked results to existing no-progress/global escalation; skip recording only `tool-loop` vetoes or keep recording non-loop vetoes.
  - confidence: 0.86
- **[P2] Reject thresholds above the effective history window:** `src/config/zod-schema.agent-runtime.ts:540-543`
  - body: The schema only rejects `historySize < consecutiveErrorThreshold` when `historySize` is explicitly set. If a user sets `consecutiveErrorThreshold` above the default history size of 30 without also setting `historySize`, history is truncated before the detector can ever reach the threshold, silently disabling the new guard.
  - confidence: 0.84
- **[P3] Remove the duplicate gcloud changelog entry:** `CHANGELOG.md:404`
  - body: This PR adds the gcloud `CLOUDSDK_PYTHON` fix bullet even though the same bullet remains later in the active changelog section, so the release notes would duplicate an unrelated fix after merge.
  - confidence: 0.9
