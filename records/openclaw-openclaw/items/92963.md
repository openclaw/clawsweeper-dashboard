## What This Changes

The PR broadens launchd service audit KeepAlive detection to count KeepAlive dictionaries containing `SuccessfulExit` or `Crashed` as present alongside `<true/>`.

## Best Possible Solution

Accept only value-correct dict forms such as `SuccessfulExit=false` and/or `Crashed=true`, add accepted/rejected service-audit coverage, require redacted real doctor or LaunchAgent proof, and leave generated plist lifecycle policy with the canonical related issue.
