## Summary

Keep open: current main and the latest release still select the user-scope systemd unit when same-name user and system units both exist, and two open linked PRs are already candidate fixes for this issue.

## What This Changes

The issue requests upgrade/doctor cleanup for stale user-scope Linux systemd gateway units when a newer system-scope unit also exists, preventing dueling services and gateway restart loops.

## Best Possible Solution

Land one focused systemd/doctor fix that detects same-name user+system units, retires or clearly blocks the stale user unit with operator-safe messaging, and preserves intentional multi-gateway setups through distinct unit names and ports.

## Reproduction Assessment

Yes, source-reproducible: current main has a test fixture where both user and system unit files exist and the resolver returns the user scope. I did not live-run systemd in this read-only review.

## Solution Assessment
