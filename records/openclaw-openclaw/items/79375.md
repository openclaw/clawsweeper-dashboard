## Best Possible Solution

On Linux, when both the canonical user unit (~/.config/systemd/user/openclaw-gateway.service) and a canonical system unit (/etc/systemd/system/openclaw-gateway.service or other system systemd dirs) exist for the same gateway/profile/port, current doctor/status paths fail to report or clean up the duplicate supervisor. Add detection for this canonical user+system systemd pair in doctor/status.
