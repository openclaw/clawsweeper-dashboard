## Summary

Keep open for maintainer review: the patch is a focused fix for the documented safe cache-trace config path, has sufficient terminal proof and no blocking review findings, but it relaxes an agent-facing gateway config write guard.

## What This Changes

The PR allows agent-facing gateway `config.patch`/`config.apply` to mutate safe `diagnostics.cacheTrace` leaves only when raw content capture remains disabled, updates cache-trace env precedence, adds regression tests, and adds a changelog entry.

## Best Possible Solution

Land this PR or an equivalent narrow resulting-config guard that preserves the fail-closed gateway write policy while allowing documented safe cache-trace observability.

## Reproduction Assessment

Yes. Source inspection on current main shows the reported safe `diagnostics.cacheTrace` patch changes non-allowlisted leaves, so the gateway tool rejects it before forwarding the config mutation.

## Solution Assessment

Yes. The patch keeps the default allowlist fail-closed, permits only exact cache-trace leaves after checking the resulting config, and adds runtime protection so config-disabled raw content capture stays disabled.
