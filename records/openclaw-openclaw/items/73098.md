The PR allows agent-facing gateway `config.patch` and `config.apply` to change safe `diagnostics.cacheTrace` leaves only when raw message, prompt, and system capture remain disabled, with tests and a changelog entry.

## Best Possible Solution

Land this PR or an equivalent narrow resulting-config guard that preserves the fail-closed gateway write policy while allowing documented safe cache-trace observability.

## Reproduction Assessment

Yes. Source inspection on current main shows a safe `diagnostics.cacheTrace` patch changes non-allowlisted leaves, so the gateway tool rejects it before forwarding the config mutation.

## Solution Assessment

Yes. The patch keeps the default allowlist fail-closed, allows only exact cache-trace leaves after checking the resulting config, and adds runtime protection so config-disabled raw content capture stays disabled.
