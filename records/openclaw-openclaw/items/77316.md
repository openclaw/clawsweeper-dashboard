## Summary

Current main and the shipped v2026.5.3 release already honor the effective tool denylist before optional PDF/media factories are constructed, so this 2026.5.2 report can close as implemented.

## What This Changes

Request that `tools.deny: ["pdf"]` prevent built-in PDF tool initialization/prep so non-PDF agent turns avoid the PDF setup cost.

## Best Possible Solution

Keep the shipped v2026.5.3 denylist-aware factory planning; users still seeing this on 2026.5.2 should upgrade, and any residual current-release latency should be filed with fresh traces.

## Reproduction Assessment

No current-main failure path: source and regression coverage show `tools.deny: ["pdf"]` makes optional PDF factory planning return false before `createPdfTool` is constructed. The historical 2026.5.2 behavior is corroborated by #76997 and the v2026.5.3 release fix.

## Solution Assessment

Yes. Honoring the existing effective denylist during optional factory planning is the narrowest maintainable fix and avoids adding a separate PDF-specific config switch.
