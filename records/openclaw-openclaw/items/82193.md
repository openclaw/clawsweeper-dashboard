## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Enforce session visibility before delivery:** `src/agents/tools/sessions-broadcast-tool.ts:242-245`
  - body: The broadcast loop queues a system event for each resolved row without applying the existing session visibility checker. Under the default `tree` or stricter `self` visibility, an unrelated same-agent row returned by `sessions.list` can still receive an injected event, bypassing the boundary current session tools enforce.
  - confidence: 0.93
- **[P2] Classify gateway rows before kind filtering:** `src/agents/tools/sessions-broadcast-tool.ts:200-202`
  - body: `sessions.list` gateway rows use raw kinds like `direct`, while the tool documents filters such as `main`, `cron`, and `hook`. Comparing `kindsRaw` to `session.kind` means `kinds: ["main"]` skips normal direct sessions, and rows with missing kinds bypass the filter entirely.
  - confidence: 0.9
- **[P2] Register broadcasts as side-effecting tools:** `src/agents/openclaw-tools.ts:230-236`
  - body: After adding `sessions_broadcast` to the runtime tool surface, mutation classification still only names `sessions_send` and `sessions_spawn`, and the fallback heuristic does not match `broadcast`. Replay/recovery handling can therefore treat a successful broadcast as safe to retry even though it queued external session events.
  - confidence: 0.86
