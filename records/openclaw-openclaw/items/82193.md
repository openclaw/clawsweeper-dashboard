## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Enforce session visibility before delivery:** `src/agents/tools/sessions-broadcast-tool.ts:242-245`
  - body: The broadcast loop queues a system event for each resolved row without applying the existing session visibility checker. Under `self`, default `tree`, or restricted cross-agent visibility, rows returned by the gateway can still receive injected events even though current session tools would hide or reject them.
  - confidence: 0.94
- **[P2] Classify rows before kind filtering:** `src/agents/tools/sessions-broadcast-tool.ts:200-202`
  - body: The tool compares requested `kinds` directly with `session.kind`, but gateway rows can use raw kinds that differ from the normalized tool-facing categories documented for session tools. Reuse `classifySessionKind` before filtering so `main`, `cron`, `hook`, and other filters select the same rows as `sessions_list`.
  - confidence: 0.9
- **[P2] Register broadcasts as side-effecting tools:** `src/agents/openclaw-tools.ts:233-239`
  - body: After adding `sessions_broadcast` to the runtime tool surface, mutation classification still only names `sessions_send` and `sessions_spawn`, and the fallback heuristic does not match `broadcast`. Replay/recovery handling can therefore treat a successful broadcast as safe to retry even though it queued external session events.
  - confidence: 0.87
