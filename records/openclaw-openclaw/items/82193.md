## Summary

Keep this PR open, but the current patch is not ready to merge: it introduces a new core broadcast tool with a session-visibility boundary bypass, kind-filter mismatches against the gateway row contract, missing side-effect classification, and no real behavior proof beyond tests.

## What This Changes

The PR adds a core `sessions_broadcast` agent tool, registers it in the OpenClaw tool surface/catalog, adds unit tests, and documents broadcast parameters and examples.

## Best Possible Solution

Land only after maintainer approval for the core broadcast API and after the implementation reuses existing session visibility checks, normalized session kind handling, side-effect classification, focused tests, and real behavior proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction: with `tools.sessions.visibility` at the default `tree` or `self` and `sessions.list` returning an unrelated same-agent row, the new loop still reaches `enqueueSystemEvent` because it never invokes the visibility checker.

## Solution Assessment

No. The maintainable path is to build this on the existing session visibility and classification helpers, then explicitly classify `sessions_broadcast` as a side-effecting messaging tool before documenting it as a core surface.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Enforce session visibility before delivery:** `src/agents/tools/sessions-broadcast-tool.ts:242-245`
  - body: The broadcast loop queues a system event for every resolved row after only checking the A2A allow-list. Current session tools also enforce `tools.sessions.visibility` via the shared visibility guard, so the default `tree`/`self` boundary is bypassed here and an agent can inject events into unrelated sessions returned by `sessions.list`.
  - confidence: 0.93
- **[P2] Classify gateway rows before kind filtering:** `src/agents/tools/sessions-broadcast-tool.ts:188-202`
  - body: `sessions.list` gateway rows use raw kinds such as `direct`, while this tool documents and filters on user-facing kinds such as `main`, `cron`, and `hook`. Comparing `kindsRaw` to the raw `session.kind` means `kinds: ["main"]` will skip normal direct sessions and rows with missing kinds can bypass the filter entirely.
  - confidence: 0.9
- **[P2] Register broadcasts as side-effecting tools:** `src/agents/openclaw-tools.ts:230-236`
  - body: After adding `sessions_broadcast` to the runtime tool surface, the runner still will not classify it as a mutating tool because `tool-mutation.ts` only lists `sessions_send`/`sessions_spawn` and name heuristics do not match `broadcast`. That can make replay and incomplete-turn handling treat a successful broadcast as safe to retry even though it enqueued external session events.
  - confidence: 0.86
