## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Enforce session visibility before delivery:** `src/agents/tools/sessions-broadcast-tool.ts:242-245`
  - body: The broadcast loop queues a system event for every resolved row without applying the shared session visibility checker. Under default `tree` or stricter `self` visibility, rows returned by `sessions.list` can still receive injected events outside the boundary enforced by current session tools.
  - confidence: 0.93
- **[P2] Normalize session kinds before filtering:** `src/agents/tools/sessions-broadcast-tool.ts:200-202`
  - body: The PR documents filters like `main`, `cron`, and `hook`, but compares requested values directly to `session.kind` and skips filtering when the raw kind is missing. Current `sessions_list` classifies rows with `classifySessionKind` first, so this can skip intended targets or deliver rows that should have been filtered out.
  - confidence: 0.9
- **[P2] Register broadcasts as mutating tools:** `src/agents/openclaw-tools.ts:233-239`
  - body: After adding `sessions_broadcast` to the runtime tool surface, mutation/replay classification still only names `sessions_send` and `sessions_spawn`, and the fallback heuristic does not match `broadcast`. Retry and failure-warning paths can therefore treat a successful broadcast as non-mutating even though it queued external session events.
  - confidence: 0.88
- **[P2] Add the factory name for narrow allowlists:** `src/agents/tool-catalog.ts:157-162`
  - body: The catalog exposes `sessions_broadcast`, but `OPENCLAW_TOOL_FACTORY_NAMES` is not updated. A configuration or inherited policy that allowlists only `sessions_broadcast` will not construct the OpenClaw tool factory, so the newly documented tool can be allowed but absent at runtime.
  - confidence: 0.86
