## Summary

Keep this PR open but do not merge it yet: the proposed core broadcast tool has high-confidence correctness and session-boundary defects, and the external contributor has not provided required real behavior proof.

## What This Changes

The PR adds a core `sessions_broadcast` tool, registers it in the agent tool catalog, adds unit coverage, and documents multi-session broadcast usage.

## Best Possible Solution

Land only after maintainer approval for the core broadcast API and after the implementation reuses existing session visibility checks, normalized session kind handling, mutating-tool classification, focused tests, and real behavior proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction: with `tools.sessions.visibility` at the default `tree` or `self` and a gateway list row outside the requester tree, the PR loop still reaches `enqueueSystemEvent` because it never invokes the visibility checker.

## Solution Assessment

No. The maintainable path is to build broadcast on the existing session visibility and classification helpers, explicitly classify it as side-effecting, and only then expose the core tool with real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Enforce session visibility before delivery:** `src/agents/tools/sessions-broadcast-tool.ts:242-245`
  - body: The broadcast loop queues a system event for every resolved row after only checking the A2A allow-list. Current session tools also enforce `tools.sessions.visibility`, so the default `tree` or `self` boundary is bypassed here and an agent can inject events into unrelated sessions returned by `sessions.list`.
  - confidence: 0.93
- **[P2] Normalize session kind before filtering:** `src/agents/tools/sessions-broadcast-tool.ts:201-202`
  - body: `sessions.list` gateway rows use raw kinds such as `direct`, while this tool documents and filters on user-facing kinds such as `main`, `cron`, and `hook`. Comparing the requested kinds to raw `session.kind` makes `kinds: ["main"]` skip normal direct sessions and lets rows with missing kinds bypass the filter.
  - confidence: 0.9
- **[P2] Mark broadcast as mutating:** `src/agents/openclaw-tools.ts:233-239`
  - body: After registering `sessions_broadcast`, the runner still will not classify it as a mutating tool because `tool-mutation.ts` only lists `sessions_send` and `sessions_spawn`, and the name heuristic does not match `broadcast`. Successful broadcasts can therefore be treated as safe to replay even though they enqueue session events.
  - confidence: 0.86
