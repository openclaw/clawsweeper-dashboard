  - body: The broadcast loop queues a system event for each resolved row without applying the shared session visibility checker.   - body: The PR documents filters such as `main`, `cron`, and `hook`, but compares requested values directly to `session.kind`. Current `sessions_list` normalizes gateway rows through `classifySessionKind` first, so this can skip normal sessions or let missing raw kinds bypass the filter.
  - confidence: 0.9
- **[P2] Register broadcasts as mutating actions:** `src/agents/openclaw-tools.ts:233-239`
  - body: After exposing `sessions_broadcast` in the runtime tool surface, mutation classification still only names `sessions_send` and `sessions_spawn`, and the fallback heuristic does not match `broadcast`. Retry/recovery paths can therefore treat a successful broadcast as non-mutating even though it queued external session events.
  - confidence: 0.87
