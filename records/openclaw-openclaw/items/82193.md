## Summary

Keep open: the proposed core fan-out tool is plausible, but the current patch has blocking session-boundary and replay-safety defects and still lacks required real behavior proof.

## What This Changes

Adds a new core `sessions_broadcast` agent tool with registry/catalog entries, tests, and session-tool documentation.

## Best Possible Solution

Land only after maintainer approval for the core broadcast API and after the implementation reuses existing visibility checks, normalized kind handling, side-effect classification, focused tests, and real behavior proof.

## Reproduction Assessment

Yes. Source inspection shows that with default `tools.sessions.visibility=tree` or `self`, a listed unrelated same-agent session can still reach `enqueueSystemEvent` because the PR delivery loop never calls the shared visibility checker.

## Solution Assessment

No. The maintainable path is to build broadcast on the same visibility and session-kind contracts used by existing session tools, then classify it as side-effecting before exposing it as a core tool.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Enforce session visibility before delivery:** `src/agents/tools/sessions-broadcast-tool.ts:242-245`
  - body: The broadcast loop queues a system event for each resolved row after only checking the A2A allow-list. Existing session tools also enforce `tools.sessions.visibility`, so the default `tree`/`self` boundary can be bypassed here for unrelated sessions returned by `sessions.list`.
  - confidence: 0.93
- **[P2] Classify gateway rows before kind filtering:** `src/agents/tools/sessions-broadcast-tool.ts:188-202`
  - body: `sessions.list` gateway rows use raw kinds such as `direct`, while this tool documents user-facing kinds such as `main`, `cron`, and `hook`. Comparing `kinds` to the raw row kind makes `kinds: ["main"]` skip normal direct sessions and lets rows with missing kinds bypass the filter.
  - confidence: 0.9
- **[P2] Register broadcasts as side-effecting tools:** `src/agents/openclaw-tools.ts:233-239`
  - body: After adding `sessions_broadcast` to the tool surface, replay and incomplete-turn handling still will not classify it as mutating because `tool-mutation.ts` only names `sessions_send` and `sessions_spawn`, and the heuristics do not match `broadcast`. A successful broadcast can enqueue external events and must not be treated as safe to retry.
  - confidence: 0.86
