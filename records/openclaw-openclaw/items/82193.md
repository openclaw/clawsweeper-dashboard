Full review comments:

- **[P1] Enforce session visibility before delivery:** `src/agents/tools/sessions-broadcast-tool.ts:242-245`
  - body: The broadcast loop queues a system event for each resolved row without applying the shared session visibility checker. Under default `tree` or stricter `self` visibility, rows returned by gateway listing can still receive injected events even when the caller should not be able to send to them.
  - confidence: 0.93
- **[P2] Normalize kind filters before comparing rows:** `src/agents/tools/sessions-broadcast-tool.ts:200-202`
  - body: The docs advertise filters such as `main`, `cron`, and `hook`, but the PR compares requested values directly to `session.kind`. Current `sessions_list` normalizes rows through `classifySessionKind`, so this can skip intended targets or include rows whose raw gateway kind does not match the documented tool kind.
  - confidence: 0.9
- **[P2] Classify broadcasts as mutating tools:** `src/agents/openclaw-tools.ts:233-239`
  - body: This registers a side-effecting tool, but mutation/replay classification still only names `sessions_send` and uses a `send` heuristic that does not match `sessions_broadcast`. Retry or recovery paths can therefore treat a successful broadcast as replay-safe after it already queued session events.
  - confidence: 0.87
