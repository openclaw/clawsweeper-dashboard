## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Enforce session visibility before queuing broadcasts:** `src/agents/tools/sessions-broadcast-tool.ts:242-245`
  - body: The broadcast loop queues a system event for each resolved row without applying the shared session visibility checker. Under default `tree` or stricter `self` visibility, unrelated rows returned by `sessions.list` can still receive injected events even though current session tools would hide or reject them.
  - confidence: 0.93
- **[P2] Normalize session kinds before filtering:** `src/agents/tools/sessions-broadcast-tool.ts:200-202`
  - body: The PR documents filters such as `main`, `cron`, and `hook`, but compares requested values directly to `session.kind`. Current `sessions_list` normalizes gateway rows through `classifySessionKind` first, so this can skip normal sessions or let missing raw kinds bypass a requested filter.
  - confidence: 0.9
- **[P2] Classify broadcasts as control-plane side effects:** `src/agents/openclaw-tools.ts:233-239`
  - body: After exposing `sessions_broadcast` in the runtime tool surface, mutation and ACP control-plane classification still only name existing session send/spawn/status tools, and the fallback heuristic does not match `broadcast`. Retry/recovery or approval paths can therefore treat a successful broadcast as non-mutating even though it queued external session events.
  - confidence: 0.87
