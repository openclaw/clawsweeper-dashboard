## Summary

Keep open: this is a new core API surface, and the current patch has blocking session-boundary and replay-safety defects plus missing real behavior proof.

## What This Changes

The PR adds a `sessions_broadcast` agent tool, runtime/tool-catalog registration, documentation, and unit tests for filtered multi-session fan-out.

## Best Possible Solution

Land a broadcast primitive only after maintainers approve the core API and the implementation reuses existing visibility, normalized session-kind, and side-effect classification contracts with focused tests and redacted real setup proof.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes, by source inspection: with default `tools.sessions.visibility` of `tree` or `self`, an unrelated same-agent row returned by `sessions.list` can still reach `enqueueSystemEvent` because the new loop never invokes the visibility checker.

## Solution Assessment

Is this the best way to solve the issue? No. The maintainable path is to build fan-out on the existing session visibility and classification helpers, then explicitly mark the tool as side-effecting before documenting it as a supported core surface.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Enforce session visibility before delivery:** `src/agents/tools/sessions-broadcast-tool.ts:242-245`
  - body: The broadcast loop queues a system event for every resolved row after only checking the A2A allow-list. Current session tools also enforce `tools.sessions.visibility`, so the default `tree`/`self` boundary can be bypassed and an agent can inject events into unrelated sessions returned by `sessions.list`.
  - confidence: 0.93
- **[P2] Classify gateway rows before kind filtering:** `src/agents/tools/sessions-broadcast-tool.ts:188-202`
  - body: `sessions.list` gateway rows use raw kinds such as `direct`, while this tool documents and filters on user-facing kinds such as `main`, `cron`, and `hook`. Comparing the filter to raw `session.kind` means `kinds: ["main"]` skips normal direct sessions, and rows with missing kinds can bypass the filter.
  - confidence: 0.9
- **[P2] Register broadcasts as side-effecting tools:** `src/agents/openclaw-tools.ts:233-239`
  - body: After adding `sessions_broadcast` to the runtime tool surface, the runner still will not classify it as mutating because `tool-mutation.ts` only lists `sessions_send` and `sessions_spawn`, and the name heuristics do not match `broadcast`. A successful broadcast could be treated as safe to retry even though it enqueued external session events.
  - confidence: 0.86
