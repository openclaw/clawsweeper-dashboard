## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Make the no-script fallback carrier-aware:** `src/agents/bash-tools.exec.ts:1338`
  - body: When recursive extraction finds a script through `sudo`/`env` plus a shell wrapper but every candidate is missing, outside the workdir, or unreadable, `checkedAnyScript` stays false and this branch rechecks the raw command. That raw heuristic does not unwrap the carrier, so a command like `sudo -EH bash -lc 'python missing.py'` can still execute instead of failing closed.
  - confidence: 0.87
- **[P1] Normalize carried interpreter executable tokens:** `src/agents/bash-tools.exec.ts:591`
  - body: After unwrapping a carrier, this direct-target check still delegates to a parser that compares the raw executable token, so `sudo -u openclaw /usr/bin/python bad.py` or `/usr/bin/node bad.js` will not be recognized and the unsafe script is never preflighted. Reuse the existing executable basename normalization before matching Python and Node executables.
  - confidence: 0.82
