## Summary

Keep open. Current main and v2026.6.10 still expose WhatsApp DM access through `dmPolicy`/`allowFrom` plus prompt/history maps, but not a per-contact policy that separates inbound handling, explicit outbound sends, and autonomous replies.

## What This Changes

Request per-contact WhatsApp config to independently allow or block inbound processing, explicit outbound sends, and autonomous replies at root and account scopes.

## Best Possible Solution

Define one maintainer-approved WhatsApp-owned per-contact policy model, then implement it consistently across schema/types, account merging, inbound admission, explicit outbound resolution, autonomous/queued reply paths, docs, generated metadata, and focused tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source inspection gives high confidence that current main lacks the requested per-contact inbound, outbound, and autonomous-reply policy.

## Solution Assessment

Unclear until maintainers choose the product and security contract. The likely maintainable shape is one plugin-owned WhatsApp contact-policy resolver reused by inbound, outbound, and autonomous paths rather than separate ad hoc checks.
