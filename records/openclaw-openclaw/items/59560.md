## Summary

Keep open. Current main and the latest release still lack a WhatsApp per-contact policy that independently controls inbound admission, explicit outbound sends, and autonomous replies, and the remaining work is a security-sensitive public config decision.

## What This Changes

The issue requests a new WhatsApp per-contact policy config for allowing or blocking inbound handling, explicit outbound sends, and autonomous replies per phone number.

## Best Possible Solution

Design one WhatsApp-owned contact-policy model with typed schema, root/account merge semantics, inbound admission enforcement, explicit outbound checks, autonomous-reply suppression, docs, and focused regression coverage.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that current main lacks the requested schema, inbound gate, outbound gate, and autonomous-reply guard.

## Solution Assessment

Unclear until maintainers choose the public config contract. The maintainable direction is a single WhatsApp-owned hard policy across inbound, explicit outbound, and autonomous reply paths rather than memory, dummy agents, or a deny-list alone.
