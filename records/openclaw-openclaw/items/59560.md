## Summary

Keep open. Current main still exposes only global/list-based WhatsApp DM controls plus prompt/history maps, while the requested per-contact inbound, outbound, and autonomous-reply policy remains unimplemented; the closed related issue confirms the same outbound-only-contact use case should stay centralized here.

## What This Changes

Add WhatsApp per-contact config so individual phone numbers can allow explicit outbound sends while blocking inbound handling and autonomous replies.

## Best Possible Solution

Design this as a WhatsApp-owned config/runtime feature: typed contact policy, root/account merge semantics, inbound admission enforcement before side effects, explicit outbound checks, autonomous-reply dispatch guard, docs/schema updates, and focused tests.

## Reproduction Assessment

Not applicable as a feature request, but there is a high-confidence static verification path. Current schema, docs, inbound admission, outbound authorization, and reply dispatch show no per-contact inbound/outbound/autonomous policy on current main.

## Solution Assessment

Unclear until a maintainer chooses the public config shape. The durable direction is hard WhatsApp runtime enforcement in config/schema/runtime, not prompts, memory, or the dummy-agent workaround from the related discussion.
