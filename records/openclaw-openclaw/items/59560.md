Keep open. Current main and v2026.6.10 still lack a WhatsApp per-contact policy that separates inbound handling, explicit outbound sends, and autonomous replies; this remains a security-sensitive feature/config decision rather than conservative cleanup.

## What This Changes

## Best Possible Solution

Define one maintainer-approved WhatsApp-owned contact-policy resolver and enforce it consistently across schema/types, account merging, inbound admission, explicit outbound resolution, autonomous or queued reply paths, docs, generated metadata, and tests.

## Solution Assessment
