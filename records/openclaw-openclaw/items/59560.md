## Summary

Keep open. Current main and v2026.6.10 still expose WhatsApp DM control through `dmPolicy`/`allowFrom`, prompt/history maps, and session `sendPolicy` reply suppression, but not a per-contact policy that can block inbound turns while allowing explicit outbound sends and preventing autonomous replies.

## What This Changes

Request adds WhatsApp per-contact config to independently control inbound handling, explicit outbound sends, and autonomous replies.

## Best Possible Solution

Define one maintainer-approved WhatsApp contact-policy resolver and reuse it consistently across schema/types, account merging, inbound admission, explicit outbound sends, autonomous or queued reply paths, docs, generated metadata, and focused tests.

## Reproduction Assessment

No runtime bug reproduction is applicable because this is a feature/config request. Source inspection gives high confidence that current main lacks the requested per-contact inbound, outbound, and autonomous-reply policy.

## Solution Assessment

Unclear until maintainers choose the product and security contract. The maintainable direction is a single WhatsApp-owned contact-policy resolver reused by all affected paths, not separate ad hoc inbound, outbound, and reply checks.
