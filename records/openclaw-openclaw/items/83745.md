## Summary

Keep open: current main still advertises `openclaw memory` in docs and plugin CLI metadata, while the command is plugin-owned and can be absent from runtime dispatch when the active memory slot is not `memory-core`. The reporter’s v2026.5.18 live repro with an `openclaw-honcho` memory slot matches the source path, and the related https://github.com/openclaw/openclaw/issues/81077 covers a different unknown-command help fallthrough case.

## What This Changes

The issue asks for the advertised and documented `openclaw memory` command to dispatch `status`, `search`, and related subcommands instead of failing as unknown.

## Best Possible Solution

Make the advertised `openclaw memory` contract and dispatch agree, with explicit coverage for the default `memory-core` slot and a non-default memory slot such as Honcho.

## Reproduction Assessment

Yes, at source level: docs and plugin metadata advertise `openclaw memory`, while dispatch only works if plugin CLI registration adds that root. The reporter also supplied a live v2026.5.18 repro with a non-default memory slot; I did not execute a packaged current-main command in this read-only review.

## Solution Assessment

Unclear: the maintainable fix depends on whether maintainers want `openclaw memory` to be a stable generic active-memory command or a `memory-core` plugin command that is only advertised when it can dispatch. Either path should include tests for default and non-default memory slots.
