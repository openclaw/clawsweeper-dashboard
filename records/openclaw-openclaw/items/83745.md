## Summary

Keep open: current main still advertises `openclaw memory` in docs and plugin metadata, while dispatch depends on the selected plugin actually registering that root. The reported Honcho setup is a credible mismatch that needs an owner decision rather than cleanup closure.

## What This Changes

The issue asks OpenClaw to make the documented `openclaw memory` CLI executable, or stop advertising it when the active memory plugin cannot provide it.

## Best Possible Solution

Make docs, top-level help, and dispatch agree for the active memory plugin: either reliably provide `openclaw memory` or only advertise the plugin-specific CLI that can run.

## Reproduction Assessment

Yes, source-level reproduction is high-confidence: docs and metadata advertise `openclaw memory`, while current dispatch only gets that root if the owning plugin is actually registered for the parsed primary. The reporter also supplied a sanitized live v2026.5.18 repro with a non-default Honcho memory slot.

## Solution Assessment

Unclear. The narrow end state is obvious, but maintainers need to choose whether `openclaw memory` is a stable active-memory abstraction or a `memory-core`-specific CLI that should disappear when another memory plugin owns the slot.
