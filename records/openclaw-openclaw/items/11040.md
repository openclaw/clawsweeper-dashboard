## Summary

Keep open: current main still implements only partial spawned-session lineage and does not provide the requested first-class rootSessionKey plus session-chain traceId contract across storage, Gateway, agent tooling, and spawn write paths.

## What This Changes

Add optional parent/root/trace/depth metadata for spawned sessions and expose it through session listing APIs so multi-hop task chains can be reconstructed.

## Best Possible Solution

Land a focused additive core contract that persists optional rootSessionKey and traceId, derives parent/root/trace/depth consistently for native and ACP child sessions, exposes the fields through Gateway and agent session-list projections, and documents the compatibility behavior for older entries.

## Reproduction Assessment

Yes. Source inspection on current main shows the existing partial lineage fields but no durable rootSessionKey or session-chain traceId in the model, schema, projections, or native/ACP spawn patches.

## Solution Assessment

Yes. The issue's Phase 1 optional metadata contract is the narrowest maintainable first step before adding traversal APIs or UI graphing, and it preserves backward compatibility.
