## Summary

Keep open: current main still has only partial spawned-session lineage and does not implement the requested first-class root/trace chain contract. The provided discussion already notes that related PR #11043 was closed unmerged, and current source still lacks rootSessionKey plus session-chain traceId in the durable model, gateway schema/projections, agent-facing sessions_list rows, and native/ACP spawn writes.

## What This Changes

Add first-class spawned-session chain metadata so parent, root, trace, and depth fields are persisted and exposed through session listing APIs.

## Best Possible Solution

Land a focused additive core contract that persists optional rootSessionKey and traceId, derives parent/root/trace/depth in native and ACP spawn paths, exposes the fields through gateway and agent session-list projections, documents them, and preserves compatibility for older session entries.

## Reproduction Assessment

Yes. Source inspection on current main shows the requested rootSessionKey and session-chain traceId are absent from the session model, gateway schema/projections, sessions_list output, and native/ACP spawn patches; a live reproduction would be spawning nested native/ACP sessions and observing those fields missing from sessions.list and sessions_list.

## Solution Assessment

Yes. The issue's Phase 1 optional metadata contract is the narrowest maintainable solution because deterministic DAG reconstruction needs durable lineage fields before adding traversal APIs or UI graphing.
