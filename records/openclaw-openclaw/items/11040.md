## Summary

Keep open: current main still only provides partial spawned-session lineage and does not implement the requested first-class `rootSessionKey` plus session-chain `traceId` contract. The direct implementation attempt at https://github.com/openclaw/openclaw/pull/11043 is closed unmerged, while the merged ACP projection at https://github.com/openclaw/openclaw/pull/73458 and open resolver-lineage work at https://github.com/openclaw/openclaw/pull/79962 are adjacent but do not replace this core spawn-chain request.

## What This Changes

Request adds first-class optional parent/root/trace/depth chain metadata to session storage, spawned-session writes, and Gateway plus agent-facing session listing rows.

## Best Possible Solution

Approve one additive core session-chain contract that persists optional root and trace metadata, derives it in native and ACP spawn paths, projects it through Gateway and agent session listings, and leaves traversal/UI work as separate follow-ups.

## Reproduction Assessment

Yes, source-level: current main’s session model, Gateway protocol schema, Gateway row projection, agent-facing `sessions_list`, and native/ACP spawn writes can be inspected directly and still lack `rootSessionKey` plus a session-chain `traceId`. I did not run a live spawn flow because this is a read-only feature-gap review.

## Solution Assessment

Unclear until maintainers choose the permanent trace semantics. The maintainable direction is a narrow additive session-chain metadata contract, but it should be aligned with diagnostic tracing and adjacent resolver-lineage work before another fix PR runs.
