## Summary

Keep open: current main and the latest release still provide only partial spawned-session lineage. `spawnedBy`, `parentSessionKey`, `spawnDepth`, and child rows exist in some surfaces, but the requested first-class `rootSessionKey` and session-chain `traceId` contract is still missing from the persisted session model, Gateway protocol, agent-facing `sessions_list`, and spawn write paths.

## What This Changes

Add first-class spawned-session chain metadata so parent, root, trace, and depth can be reconstructed from session APIs.

## Best Possible Solution

Land a narrow additive core session-chain contract that persists optional root and trace metadata, derives it consistently on native and ACP child creation, exposes it through Gateway and agent list projections, and documents the compatibility behavior for existing sessions.

## Reproduction Assessment

Yes, at source level. Current main and `v2026.6.6` expose partial parent/depth lineage but lack the requested `rootSessionKey` and stable session-chain `traceId` across the persistent model, protocol, projections, tool rows, and spawn writes.

## Solution Assessment

Unclear until maintainers choose the trace/root contract. The best shape is a narrow additive core metadata contract, not a broad agent-hierarchy framework or a wholesale resurrection of the closed PR.
