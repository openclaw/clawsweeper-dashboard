## Summary

Keep open: current main and v2026.6.10 still provide only partial spawned-session lineage. The requested persisted root session key and session-chain trace id contract remains missing, and the issue now needs maintainer API/product agreement before another implementation attempt.

## What This Changes

Request first-class persisted parent/root/trace/depth metadata for spawned sessions and exposure through Gateway and agent session-list APIs.

## Best Possible Solution

Approve a focused additive core session-chain contract that persists optional rootSessionKey and traceId, derives them consistently for native and ACP child sessions, exposes them through Gateway and agent list projections, and leaves traversal/UI work as follow-ups.

## Reproduction Assessment

Yes. Source inspection on current main shows the session model, Gateway protocol/projections, sessions_list tool rows, and native/ACP spawn writes still lack rootSessionKey plus a session-chain traceId.

## Solution Assessment

Unclear until maintainers choose trace semantics. A narrow additive metadata contract is the maintainable direction, but it should not become a broader agent-hierarchy framework or duplicate diagnostic trace context accidentally.
