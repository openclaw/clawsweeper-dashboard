## Summary

Keep open: current main still has only partial spawned-session lineage. It persists and exposes spawnedBy, parentSessionKey, spawnDepth, and childSessions in some paths, but it does not implement the requested first-class rootSessionKey/traceId contract; the direct source PR is closed unmerged, and the merged ACP work only serializes existing parent/depth metadata.

## What This Changes

The issue requests first-class persisted session-chain metadata for spawned sessions, including parent, root, trace, and depth fields in session APIs and tool output.

## Best Possible Solution

Land a focused additive core session-chain contract that persists optional rootSessionKey and traceId, derives them consistently for native and ACP child sessions, exposes them through Gateway and agent session-list projections, documents the fields, and proves multi-hop reconstruction while preserving older session entries.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction of the missing contract: current main’s storage type, protocol schema, row projections, agent-facing sessions_list output, and spawn write paths still lack durable rootSessionKey and session-chain traceId.

## Solution Assessment

Yes. The Phase 1 optional metadata contract is the narrow maintainable direction; https://github.com/openclaw/openclaw/pull/73458 only handled ACP serialization of existing metadata, and https://github.com/openclaw/openclaw/issues/79903 tracks a separate rotated-session family discovery seam.
