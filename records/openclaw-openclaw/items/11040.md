## Summary

Keep open: current main still has partial spawned-session lineage only. It persists and exposes `spawnedBy`, `parentSessionKey`, and `spawnDepth` in several paths, and https://github.com/openclaw/openclaw/pull/73458 added adjacent ACP serialization for those existing fields, but the requested persisted `rootSessionKey` and session-chain `traceId` contract is still absent; https://github.com/openclaw/openclaw/pull/11043 is closed and unmerged.

## What This Changes

Add optional parent/root/trace/depth metadata to persisted spawned sessions and expose it through Gateway and agent session-list APIs.

## Best Possible Solution

Land a focused additive session-chain contract that persists optional `rootSessionKey` and session-chain `traceId`, derives them in native and ACP spawn paths, projects them through Gateway and agent session-list rows, documents the fields, and preserves existing sessions.

## Reproduction Assessment

Yes, for a source-level feature-gap check: current main lacks `rootSessionKey` and session-chain `traceId` in the persistent model, Gateway schema/projections, agent `sessions_list` rows, and native/ACP spawn writes. Runtime bug reproduction is not applicable because this is an additive feature request.

## Solution Assessment

Yes. The proposed Phase 1 optional metadata contract is the narrow maintainable path; current parent/depth support and merged ACP serialization are useful foundations but do not replace the missing root/trace contract.
