## Summary

Keep open: current main still only exposes partial spawned-session lineage. The requested durable root/trace chain contract is absent from session storage, Gateway protocol/projections, agent-facing `sessions_list`, and native/ACP spawn writes; the linked implementation PR is closed and unmerged.

## What This Changes

The issue asks OpenClaw to persist, derive, and expose optional parent/root/trace/depth metadata for spawned session chains.

## Best Possible Solution

Land a focused additive core session-chain contract that persists and exposes optional root/trace metadata while preserving compatibility for existing sessions.

## Reproduction Assessment

Yes. Source inspection on current main is enough: storage, schema, row projection, agent tool output, and spawn-write surfaces expose only parent/depth pieces and no durable `rootSessionKey` or session-chain `traceId`.

## Solution Assessment

Yes. The Phase 1 optional metadata contract is the narrowest maintainable direction; traversal APIs, UI graphing, and descendant termination should remain separate follow-up work.
