## Summary

Keep open: current main still has only partial spawned-session lineage and does not implement the requested durable root/trace session-chain contract. The direct implementation PR is closed and unmerged, and the newer related ACP metadata PR only serializes existing lineage fields rather than adding persisted rootSessionKey/traceId.

## What This Changes

Add first-class optional session-chain metadata for spawned sessions, including parent/root/trace/depth fields, persisted on spawn and exposed through session listing APIs.

## Best Possible Solution

Land a focused additive core contract that persists optional rootSessionKey and traceId, derives them consistently for native and ACP child sessions, exposes them through Gateway and agent session-list projections, documents the fields, and covers multi-hop reconstruction while leaving UI graphing and termination policy for separate work.

## Reproduction Assessment

Yes. Source inspection on current main shows the storage type, Gateway schema/projections, agent-facing sessions_list output, and native/ACP spawn writes still expose only parent/depth pieces and no durable rootSessionKey or session-chain traceId.

## Solution Assessment

Yes. The Phase 1 optional metadata contract is the narrow maintainable direction; the ACP metadata PR only serializes existing fields, and descendant termination should remain a separate lifecycle fix.
