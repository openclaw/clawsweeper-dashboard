## Summary

Keep open: current main still only has partial spawned-session lineage. It persists and exposes spawnedBy, parentSessionKey, spawnDepth, and childSessions in some paths, but rootSessionKey and a session-chain traceId are still absent from the persistent model, Gateway schema/projections, agent-facing sessions_list rows, and native/ACP spawn writes.

## What This Changes

Request adding optional parent/root/trace/depth session-chain metadata to spawned sessions and exposing it through Gateway and agent session listing.

## Best Possible Solution

Land a focused additive core contract that persists optional rootSessionKey and traceId, derives them consistently for native and ACP child sessions, exposes them through Gateway and agent session-list projections, documents the fields, and proves multi-hop reconstruction without changing spawn policy.

## Reproduction Assessment

Yes, source-level. Current main lacks rootSessionKey and session-chain traceId in the storage type, protocol schema, projections, tool rows, and spawn writes, so the feature gap is directly visible without a live spawn.

## Solution Assessment

Yes. The proposed Phase 1 optional metadata contract is the narrow maintainable direction; the merged ACP metadata work only serializes existing parent/depth fields, while https://github.com/openclaw/openclaw/issues/79903 and https://github.com/openclaw/openclaw/pull/79962 cover rotated session-family discovery rather than spawn-chain root/trace metadata.

Add optional rootSessionKey and traceId to the persisted SessionEntry contract, Gateway protocol/schema and row projections, and agent-facing sessions_list rows; preserve existing parentSessionKey, spawnedBy, and spawnDepth behavior.
