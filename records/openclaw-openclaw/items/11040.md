## Summary

Keep open: current main still has only partial spawned-session lineage and does not implement the requested durable rootSessionKey/traceId session-chain contract. The direct implementation PR is closed and unmerged, while the merged ACP lineage PR only serializes existing Gateway lineage fields.

## What This Changes

Add first-class session/task chain tracking fields for parent/root/trace/depth so spawned-session chains can be reconstructed through OpenClaw session APIs.

## Best Possible Solution

Land a focused additive core contract that persists optional rootSessionKey and traceId, derives them for native and ACP child sessions, exposes them through Gateway and agent session-list projections, documents the fields, and covers multi-hop reconstruction.

## Solution Assessment
