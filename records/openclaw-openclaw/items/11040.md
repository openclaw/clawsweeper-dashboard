## Summary

Keep open: current main still provides only partial spawned-session lineage and does not implement the requested rootSessionKey plus session-chain traceId contract across storage, Gateway, agent tooling, and native/ACP spawn writes.

## What This Changes

Add optional first-class session chain metadata so spawned sessions expose parent, root, trace, and depth information for deterministic DAG reconstruction.

## Best Possible Solution

Land an additive session-chain contract that persists optional rootSessionKey and traceId, derives them consistently for native and ACP spawned sessions, exposes them through Gateway and sessions_list projections, and documents compatibility for older entries.

## Reproduction Assessment

Yes. Source inspection on current main shows the existing spawnedBy, parentSessionKey, spawnDepth, and childSessions pieces, but no durable rootSessionKey or session-chain traceId in the model, schema, projections, or spawn patches.

## Solution Assessment

Yes. The Phase 1 optional metadata contract is the narrowest maintainable first step; traversal APIs, UI graphing, and unrelated closed-PR changes should remain separate.
