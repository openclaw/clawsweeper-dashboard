## Summary

Keep open: current main still has only partial spawned-session lineage. The existing code carries `spawnedBy`, `parentSessionKey`, `spawnDepth`, and ACP projection for those fields, but the requested persisted `rootSessionKey` and session-chain `traceId` contract is still absent from storage, Gateway schema/projections, agent session-list rows, and native/ACP spawn writes; the implementation PR linked from the issue is closed and unmerged.

## What This Changes

Add optional first-class session/task chain metadata for spawned sessions so APIs can expose parent, root, trace, and depth fields for deterministic chain reconstruction.

## Best Possible Solution

Land a focused additive core session-chain contract that persists optional `rootSessionKey` and session-chain `traceId`, derives them in native and ACP spawn paths, projects them through Gateway and agent session-list rows, documents the fields, and preserves existing sessions.

## Solution Assessment

Yes. The proposed Phase 1 optional metadata contract is the narrow maintainable path; existing parent/depth support and merged ACP serialization are useful foundations but do not replace the missing root/trace contract.
