## Summary

Keep open: current main still has only partial spawned-session lineage. The requested durable root/trace session-chain contract is still absent from session storage, Gateway schemas/projections, agent-facing `sessions_list`, and native/ACP spawn writes; the linked implementation PR is closed and unmerged.

## What This Changes

Request optional first-class session-chain metadata so spawned sessions expose parent, root, trace, and depth information for deterministic DAG reconstruction.

## Best Possible Solution

Land a focused additive core contract: persist optional `rootSessionKey` and session-chain `traceId`, derive them consistently for native and ACP child sessions, expose them through Gateway and agent session-list projections, document the fields, and cover multi-hop reconstruction while preserving compatibility for older entries.

## Reproduction Assessment

Yes. Source inspection on current main is enough: the relevant storage, schema, projection, and spawn-write surfaces expose parent/depth pieces but not durable `rootSessionKey` or session-chain `traceId`.

## Solution Assessment

Yes. The proposed Phase 1 optional metadata contract is the narrowest maintainable first step; traversal APIs, UI graphing, and descendant termination should remain separate follow-up work.
