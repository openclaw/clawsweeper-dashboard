## Summary

Keep open: current main and the latest release still provide only partial spawned-session lineage, while the requested first-class rootSessionKey and session-chain traceId contract is missing from storage, protocol, projections, and spawn writes.

## What This Changes

Request to add first-class parent/root/trace/depth chain metadata for spawned sessions so tools can reconstruct multi-hop subagent execution DAGs.

## Best Possible Solution

Define an additive core session-chain contract that persists optional rootSessionKey and traceId, derives them consistently for native and ACP spawned sessions, exposes them through Gateway and agent listing projections, documents the fields, and proves multi-hop reconstruction without changing spawn permissions.

## Reproduction Assessment

Yes, at source level: current main exposes partial spawned-session lineage but lacks rootSessionKey and session-chain traceId in the persistent model, protocol schema, projections, and spawn writes. I did not run a live multi-hop spawn scenario because this read-only review could establish the missing contract from source.

## Solution Assessment

Unclear as final API design: the proposed optional metadata is a plausible narrow contract, but maintainers still need to decide trace semantics and how it relates to ACP lineage, usage-family lineage, and recursive listing surfaces.
