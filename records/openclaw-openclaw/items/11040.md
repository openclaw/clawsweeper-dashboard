## Summary

Keep open: current main and latest release still only provide partial spawned-session lineage. The stored and projected contracts include `spawnedBy`, `parentSessionKey`, `spawnDepth`, and child summaries in some paths, but the requested `rootSessionKey` plus session-chain `traceId` are still missing from the persistent model, Gateway protocol/projections, agent-facing `sessions_list`, and native/ACP spawn writes.

## What This Changes

The issue requests first-class parent/root/trace/depth metadata for spawned session chains so OpenClaw can reconstruct subagent DAGs from session APIs.

## Best Possible Solution

Land a focused additive core session-chain contract that persists optional `rootSessionKey` and `traceId`, derives them consistently for native and ACP child sessions, exposes them through Gateway and agent session-list projections, documents the fields, and leaves traversal/UI work to follow-up items.
