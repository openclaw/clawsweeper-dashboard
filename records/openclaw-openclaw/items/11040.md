## Summary

Keep open: current main and the latest release still provide only partial spawned-session lineage, while the requested rootSessionKey and session-chain traceId contract is absent from the persisted model, gateway schema/projections, agent-facing list output, and native/ACP spawn writes.

## What This Changes

The issue requests first-class optional session-chain metadata for spawned sessions so parent/root/trace/depth can be reconstructed from session APIs.

## Best Possible Solution

Land a focused additive core session-chain contract: persist optional rootSessionKey and traceId, derive them consistently for native and ACP child sessions, expose them through gateway and agent session-list projections, document the fields, and cover multi-hop reconstruction while preserving compatibility for older entries.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/API contract request. Source inspection confirms current main lacks rootSessionKey and session-chain traceId across the model, protocol, projections, and spawn writes.

## Solution Assessment

Unclear as a final API until maintainers choose traceId semantics, but the narrow additive model/projection/spawn-write contract is the maintainable direction. Recursive listing, chain traversal RPCs, and UI graphs should remain follow-up work.
