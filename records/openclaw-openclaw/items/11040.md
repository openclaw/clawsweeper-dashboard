## Summary

Keep open: current main still implements only partial spawned-session lineage. It has `spawnedBy`, `parentSessionKey`, `spawnDepth`, and ACP serialization for those existing fields, but not the requested persistent `rootSessionKey` or session-chain `traceId` contract in storage, Gateway rows, agent-facing `sessions_list`, or native/ACP spawn writes.

## What This Changes

The issue requests first-class parent/root/trace/depth metadata for spawned session chains so OpenClaw can reconstruct subagent execution DAGs through session APIs.

## Best Possible Solution

Land a focused additive core contract that persists optional `rootSessionKey` and session-chain `traceId`, derives them consistently for native and ACP child sessions, exposes them through Gateway and agent session-list projections, documents the fields, and proves multi-hop reconstruction without changing spawn policy.

## Reproduction Assessment

Yes, source-level. Current main lacks `rootSessionKey` and session-chain `traceId` in the storage type, Gateway schema/projections, agent `sessions_list` rows, and native/ACP spawn writes, so the feature gap is visible without a live spawn.

## Solution Assessment

Yes. The Phase 1 optional metadata contract is the narrow maintainable direction; merged ACP metadata work and open rotation-lineage work are adjacent but do not replace this spawn-chain root/trace contract.
