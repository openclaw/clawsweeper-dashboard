## Summary

Keep open: current main still has only partial spawned-session lineage. It persists and exposes `spawnedBy`, `parentSessionKey`, and `spawnDepth` in some paths, and the merged ACP bridge now serializes those existing fields, but the requested `rootSessionKey` and session-chain `traceId` contract is still absent from storage, Gateway/tool projections, and spawn writes.

## What This Changes

The issue requests optional first-class session-chain metadata for spawned sessions, including parent/root/trace/depth fields in storage and session list outputs.

## Best Possible Solution

Land a focused additive core session-chain contract: persist optional `rootSessionKey` and `traceId`, derive them consistently for native and ACP child sessions, expose them through Gateway and agent session-list projections, document the fields, and cover multi-hop reconstruction while preserving compatibility for older entries.

## Reproduction Assessment

No runtime bug reproduction is applicable because this is a feature request. There is a high-confidence source-level feature-gap check: current main lacks `rootSessionKey` and session-chain `traceId` in storage, Gateway schema/projections, agent `sessions_list` rows, and native/ACP spawn writes.

## Solution Assessment

Yes. The proposed Phase 1 optional metadata contract is the narrow maintainable direction; merged ACP metadata work and related UI/lifecycle issues are adjacent but do not replace the core root/trace spawn-chain contract.
