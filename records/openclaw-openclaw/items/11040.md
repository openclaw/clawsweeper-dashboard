## Summary

Keep open: current main and the latest release still provide only partial spawned-session lineage. The merged ACP lineage work exposes existing parent/depth fields to ACP clients, but the requested persisted rootSessionKey and session-chain traceId contract is still missing from the core session model, gateway projections, agent-facing sessions_list rows, and native/ACP spawn writes; the original implementation PR is closed unmerged.

## What This Changes

Request first-class spawned-session chain metadata so parent, root, trace, and depth can be reconstructed from session APIs.

## Best Possible Solution

Approve a focused additive core session-chain contract that persists optional rootSessionKey and traceId, derives them consistently for native and ACP child sessions, exposes them through gateway and agent session-list projections, documents the fields, and leaves traversal/UI features as follow-ups.

## Reproduction Assessment

Yes, source-level. Current main’s stored session model, gateway rows, agent-facing sessions_list output, and native/ACP spawn writes can be inspected directly and they still lack the requested rootSessionKey and session-chain traceId fields.

## Solution Assessment

Unclear until maintainers choose trace semantics. A narrow additive session-chain contract is the maintainable direction, but it should not proceed as automation until the API/product choice is settled.
