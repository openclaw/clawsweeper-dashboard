## Summary

Keep open: current main still implements only partial spawned-session lineage. The requested root/trace chain contract is absent from the persistent session model, Gateway protocol and rows, agent-facing session list rows, and both native and ACP spawn writes; the only linked implementation PR is closed and unmerged.

## What This Changes

The issue requests optional first-class session-chain metadata for spawned sessions, including parent/root/trace/depth fields and list-output exposure.

## Best Possible Solution

Land a focused additive core session-chain contract: persist optional rootSessionKey and traceId, derive them consistently for native and ACP child sessions, expose them through Gateway and agent session-list projections, document the fields, and cover multi-hop reconstruction while leaving traversal UI and parent-death cleanup separate.

## Reproduction Assessment

Yes. Source inspection on current main shows the missing contract: storage, schema, projections, and spawn writes have spawnedBy/parent/depth pieces but no durable rootSessionKey or session-chain traceId.

## Solution Assessment

Yes. The Phase 1 optional metadata contract is the narrowest maintainable first step; dedicated chain traversal, UI graphing, and descendant termination behavior should remain follow-up work.
