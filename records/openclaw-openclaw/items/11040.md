## Summary

Keep open: current main and v2026.6.8 still provide only partial spawned-session lineage. The code has spawnedBy, parentSessionKey, spawnDepth, childSessions, and ACP projection of those fields, but it still lacks the requested persisted rootSessionKey and session-chain traceId contract across the session model, gateway/tool projections, and native/ACP spawn writes.

## What This Changes

The issue requests first-class optional parent/root/trace/depth metadata for spawned sessions so tools can reconstruct multi-hop session chains deterministically.

## Best Possible Solution

Approve a focused additive core session-chain contract that persists optional rootSessionKey and traceId, derives them consistently for native and ACP child sessions, exposes them through gateway and agent session-list projections, documents the fields, and leaves traversal/UI work as follow-ups.

## Reproduction Assessment

Yes, source-level. Current main and v2026.6.8 can be inspected directly: the stored model, gateway rows, agent-facing sessions_list output, and native/ACP spawn writes still lack rootSessionKey and a session-chain traceId.

## Solution Assessment

Unclear until maintainers choose trace semantics. A narrow additive session-chain contract is the maintainable direction, but it should not proceed as automation while the root/trace API meaning is unsettled.
