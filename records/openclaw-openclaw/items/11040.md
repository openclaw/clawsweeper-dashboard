The session model, Gateway schema/projections, agent-facing session list, and native/ACP spawn paths still lack the requested `rootSessionKey` plus session-chain `traceId` contract, and the only merged related work exposes existing lineage to ACP rather than adding root/trace persistence.

Add first-class persisted session/task chain metadata so spawned sessions expose parent, root, trace, and depth information through session APIs.

## Best Possible Solution

Land a focused additive core session-chain contract: persist optional `rootSessionKey` and `traceId`, derive them consistently for native and ACP child sessions, expose them through Gateway and agent session-list projections, document the fields, and cover multi-hop reconstruction while preserving compatibility for older entries.
