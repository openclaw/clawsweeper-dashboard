## Summary

Keep open: current main still has only partial spawned-session lineage and does not implement the requested durable rootSessionKey/traceId session-chain contract. The direct implementation PR is closed and unmerged, while the newer ACP lineage PR only serializes existing Gateway fields.

## What This Changes

Add optional first-class session-chain metadata for spawned sessions, including parent/root lineage, a stable trace id, depth, and list output fields.

## Best Possible Solution

Land a focused additive core contract that persists optional rootSessionKey and traceId, derives them for native and ACP child sessions, exposes them through Gateway and agent session-list projections, documents the fields, and covers multi-hop reconstruction.
