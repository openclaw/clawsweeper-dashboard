## Summary

Keep open. Current main still has only partial spawned-session lineage: parent/depth metadata exists in some paths, and the merged ACP work exposes that partial lineage to ACP clients, but the requested persisted `rootSessionKey` and session-chain `traceId` contract is still absent from the core model, gateway schema/projections, agent-facing list output, and native/ACP spawn writes.

## What This Changes

Request to add first-class spawned-session chain metadata with parent, root, trace, and depth fields across persistence, spawn writes, and session listing APIs.

## Best Possible Solution

Land a focused additive core session-chain contract that persists optional `rootSessionKey` and `traceId`, derives them in native and ACP spawn paths, exposes them through gateway and agent session-list projections, documents the fields, and tests multi-hop reconstruction without adding traversal/UI work here.

## Reproduction Assessment

Yes for a source-level feature gap. Current main can be inspected in the session model, gateway protocol/schema, projections, tool row builder, and spawn writes, and those paths still lack `rootSessionKey` and a session-chain `traceId`.

## Solution Assessment

Unclear until maintainers choose the permanent root/trace semantics. A narrow additive metadata contract is the maintainable direction; recursive listing, traversal RPCs, and graph UI should remain separate follow-up work.
