## Summary

Keep open: current main and v2026.6.6 still expose only partial spawned-session lineage. The model, Gateway protocol/projections, agent-facing list rows, and native/ACP spawn writes have spawnedBy, parentSessionKey, and spawnDepth in places, but not the requested first-class rootSessionKey or session-chain traceId contract; the direct implementation PR is closed unmerged and the merged ACP lineage PR only projects existing Gateway fields.

## What This Changes

The issue requests additive first-class session/task chain tracking so spawned sessions carry parent, root, trace, and depth metadata through persistence and session-list APIs.

## Best Possible Solution

Land a focused additive core session-chain contract that persists optional rootSessionKey and traceId, derives them consistently for native and ACP child sessions, exposes them through Gateway and agent list projections, documents the fields, and preserves older entries as optional metadata.

## Reproduction Assessment

Yes, at source level. Current main and v2026.6.6 expose only spawnedBy, parentSessionKey, and spawnDepth in the checked session model/schema/projection/spawn paths, while rootSessionKey and a session-chain traceId remain absent.

## Solution Assessment

Unclear as a final API design. The core session model, spawn write paths, and Gateway/tool projections are the right owner boundary, but maintainers still need to choose stable root/trace semantics before implementation.
