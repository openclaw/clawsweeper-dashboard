## Summary

Keep open: current main still has only partial spawned-session lineage. The requested durable root/trace session-chain contract is absent from storage, Gateway schemas/projections, agent-facing `sessions_list`, and native/ACP spawn writes; the related implementation PR is closed and unmerged.

## What This Changes

The issue requests optional parent/root/trace/depth metadata for spawned sessions so session APIs can reconstruct multi-hop subagent DAGs.

## Best Possible Solution

Land a focused additive session-chain contract: persist optional `rootSessionKey` and `traceId`, derive them consistently for native and ACP child sessions, expose them through Gateway and agent session-list rows, document the fields, and cover multi-hop reconstruction with regression tests.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence reproduction path: the relevant storage, schema, projection, and spawn-write surfaces have parent/depth pieces but no durable root/trace chain fields.

## Solution Assessment

Yes. The proposed Phase 1 optional metadata contract is the narrowest maintainable first step; traversal APIs, UI graphing, and descendant termination should remain separate follow-up work.
