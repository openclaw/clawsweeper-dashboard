## Summary

Keep open: current main still provides only partial spawned-session lineage. It has spawnedBy, parentSessionKey, spawnDepth, child sessions, and merged ACP serialization of those existing fields, but it still lacks the requested first-class rootSessionKey and session-chain traceId contract in the persistent model, gateway/tool projections, and native/ACP spawn writes.

## What This Changes

The issue requests first-class session chain metadata for spawned sessions, including parent, root, trace, and depth fields exposed through session APIs.

## Best Possible Solution

Land a focused additive core session-chain contract: persist optional rootSessionKey and traceId, derive them consistently for native and ACP child sessions, expose them through gateway and agent session-list projections, document the fields, and cover multi-hop reconstruction while preserving compatibility for older entries.

## Reproduction Assessment

Yes, at source level. Current main exposes partial parent/depth lineage but lacks rootSessionKey and a session-chain traceId in the stored model, gateway projections, agent-facing rows, and native/ACP spawn writes.

## Solution Assessment

Mostly yes if kept narrow: additive persisted metadata plus read-path projection is the maintainable shape. The unresolved part is the trace/root API semantics, so this should not be automated until maintainers confirm that contract.
