## Summary

Keep open: current main still provides partial spawned-session lineage only. It has `spawnedBy`, `parentSessionKey`, `spawnDepth`, child rows, and ACP projection for those existing fields, but it does not persist or expose the requested `rootSessionKey` and session-chain `traceId`, and neither native nor ACP spawn derives them.

Add optional first-class spawned-session chain metadata so sessions can expose parent, root, trace, and depth for deterministic task-chain reconstruction.

## Best Possible Solution

Land a focused additive core session-chain contract that persists and projects root/trace lineage while preserving existing parent/depth behavior and compatibility for older session entries.
