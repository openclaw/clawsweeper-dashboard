## Summary

Keep open: current main and latest release still only provide partial session lineage. `spawnedBy`, `parentSessionKey`, `spawnDepth`, and child summaries exist in some paths, but the requested persisted `rootSessionKey` plus session-chain `traceId` contract is still missing from the session model, Gateway protocol/projections, agent-facing `sessions_list`, and native/ACP spawn writes.

## What This Changes

The issue requests optional first-class session-chain metadata for spawned sessions, including parent, root, trace, and depth fields exposed through session listing APIs.

## Best Possible Solution

Land a focused additive core session-chain contract: persist optional `rootSessionKey` and `traceId`, derive them consistently for native and ACP child sessions, expose them through Gateway and agent session-list projections, document the fields, and cover multi-hop reconstruction while preserving compatibility for older entries.

## Reproduction Assessment

Yes, source-level: current main and `v2026.6.5` lack `rootSessionKey` and session-chain `traceId` in the persistent model, Gateway/tool projections, and native/ACP spawn writes. Runtime bug reproduction is not applicable because this is an additive feature request.

## Solution Assessment

Yes, the requested Phase 1 optional metadata contract is the narrow maintainable direction if it stays metadata-only and keeps trace semantics explicit. The closed source PR is useful context but too broad to reuse directly.
