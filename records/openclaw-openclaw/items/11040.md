## Summary

Current main still provides only partial spawned-session lineage: parent/depth relationships are available, including through ACP, but sessions do not persist or expose a canonical root key or durable multi-hop chain identifier. Both direct implementation PRs are closed, and the remaining identity-contract choice requires maintainer judgment.

## What This Changes

Add durable parent, root, trace, and depth metadata for spawned session chains and expose it through session listing APIs.

## Best Possible Solution

Define one canonical, optional durable lineage contract across all session creation and spawn paths, expose it consistently through Gateway, agent-tool, and ACP projections, and document whether chain identity is session-owned or deliberately bridged to existing task or diagnostic tracing.

## Reproduction Assessment

Yes at source level. Current main deterministically exposes parent/depth lineage while lacking the requested persistent root and durable multi-hop chain identifier; no runtime execution is needed to establish that contract gap.

## Solution Assessment

Unclear pending one product/API decision. Optional persisted lineage fields are a bounded approach, but maintainers should first choose whether the stable chain identity is independent from or derived from existing task-run and diagnostic trace concepts.
