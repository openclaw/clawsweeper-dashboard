## Summary

Keep open: current main still has only partial spawned-session lineage and does not implement the requested durable rootSessionKey/traceId session-chain contract. The direct implementation PR is closed unmerged, while the merged ACP lineage PR only serializes existing Gateway parent/depth metadata.

## What This Changes

Add optional first-class session-chain metadata for spawned sessions and expose it through Gateway and agent session listing surfaces.

## Best Possible Solution

Land a focused additive core contract that persists optional rootSessionKey and traceId, derives them for native and ACP child sessions, exposes them through Gateway and agent session-list projections, documents the fields, and covers multi-hop reconstruction.

## Reproduction Assessment

Yes for the missing contract, by source inspection rather than a runtime failure: the storage type, Gateway schema/projections, agent-facing sessions_list output, and native/ACP spawn writes still expose only parent/depth pieces and no durable rootSessionKey or session-chain traceId.

## Solution Assessment

Yes. The Phase 1 optional metadata contract is the narrow maintainable path; broader lifecycle propagation remains separate in https://github.com/openclaw/openclaw/issues/77720 and sessionId rotation lineage remains separate in https://github.com/openclaw/openclaw/issues/79903.
