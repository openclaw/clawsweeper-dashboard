## Summary

Keep open: current main and the latest release still only provide partial spawned-session lineage, while the requested first-class rootSessionKey and session-chain traceId contract is missing from persistence, gateway/tool projections, and native/ACP spawn writes.

## What This Changes

Add optional first-class parent/root/trace/depth metadata to spawned session records and session-list outputs so multi-hop subagent chains can be reconstructed.

## Best Possible Solution

Define a narrow additive core session-chain contract, persist optional rootSessionKey and traceId, derive them once in native and ACP child-session creation, expose them through gateway and agent list projections, and cover multi-hop reconstruction without changing spawn permissions.

## Reproduction Assessment

Yes, at source level. Current main lacks rootSessionKey and session-chain traceId in the session model, protocol/projection rows, and native/ACP spawn writes, while the latest release lacks those fields too.

## Solution Assessment

Unclear. A narrow additive metadata contract is plausible, but maintainers still need to choose trace semantics and confirm the boundary relative to recursive listing and diagnostic tracing.
