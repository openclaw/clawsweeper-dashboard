## Summary

Keep open because current main still only implements partial spawned-session lineage; the requested first-class rootSessionKey and stable session-chain traceId contract is absent from the model, gateway schema/projections, tool output, and spawn write paths.

## What This Changes

Add first-class persisted session-chain metadata for spawned sessions, including parent/root/trace/depth fields exposed through session listing APIs.

## Best Possible Solution

Land a focused additive core session-chain contract after maintainers confirm the trace/root semantics: persist optional rootSessionKey and traceId, derive them consistently in native and ACP spawn paths, expose them through gateway and agent session-list projections, document the fields, and prove multi-hop reconstruction while preserving older entries.

## Reproduction Assessment

Yes, for a source-level gap: current main exposes spawnedBy, parentSessionKey, spawnDepth, and childSessions in some paths, but lacks rootSessionKey and a session-chain traceId contract. No runtime reproduction was run because this is a feature/API request rather than a broken existing behavior contract.

## Solution Assessment

Unclear until maintainers choose the exact trace/root contract. A narrow optional session metadata addition is the maintainable direction; recursive listing, traversal RPCs, and UI graph work should remain separate follow-up items.
