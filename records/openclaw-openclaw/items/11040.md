## Summary

Keep open: current main still has only partial spawned-session lineage. It persists and exposes spawnedBy, parentSessionKey, spawnDepth, and childSessions in some paths, but the requested rootSessionKey and session-chain traceId contract is still absent from the persistent model, Gateway schema/projections, agent-facing sessions_list rows, and native/ACP spawn writes.

## What This Changes

Add first-class optional parent/root/trace/depth metadata for spawned session chains and expose it through session APIs.

## Best Possible Solution

Land a focused additive core contract that persists optional rootSessionKey and traceId, derives them consistently for native and ACP child sessions, exposes them through Gateway and agent session-list projections, documents the fields, and proves multi-hop reconstruction without changing spawn policy.

## Reproduction Assessment

Yes, source-level. Current main lacks rootSessionKey and session-chain traceId in the storage type, protocol schema, projections, tool rows, and spawn writes, so the feature gap is directly visible without a live spawn.

## Solution Assessment

Yes. The requested Phase 1 optional metadata contract is the narrow maintainable direction; https://github.com/openclaw/openclaw/pull/73458 only serializes existing ACP lineage fields, while https://github.com/openclaw/openclaw/issues/79903 and https://github.com/openclaw/openclaw/pull/79962 track rotated session-family discovery rather than spawn-chain root/trace metadata.
