## Summary

Keep open: current main and the latest release still provide only partial parent/depth lineage, while the requested persisted rootSessionKey and stable session-chain traceId contract is still missing from the core session model, gateway protocol/projections, agent-facing session list output, and spawn write paths.

## What This Changes

The issue requests first-class optional parent/root/trace/depth metadata for spawned sessions so OpenClaw can reconstruct multi-hop subagent execution chains from session APIs.

## Best Possible Solution

Land a focused additive core session-chain contract: persist optional rootSessionKey and traceId, derive them consistently for native and ACP child sessions, expose them through gateway and agent session-list projections, and cover multi-hop reconstruction while preserving compatibility for older entries.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes, at source level: current main exposes partial spawned-session lineage but lacks rootSessionKey and a stable session-chain traceId across the persistent model, protocol schema, projections, tool rows, and spawn writes.

## Solution Assessment

Is this the best way to solve the issue? Unclear until maintainers choose the trace/root contract, but a narrow additive core metadata contract is the maintainable direction; recursive listing, chain traversal RPCs, and UI graphs should remain follow-up work.
