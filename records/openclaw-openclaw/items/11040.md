## Summary

Keep open: current main and the latest release still provide only partial spawned-session lineage, while the requested rootSessionKey and session-chain traceId contract is absent from the persistent model, gateway protocol/projections, agent-facing sessions_list rows, and native/ACP spawn writes.

## What This Changes

The issue requests first-class optional session-chain metadata for spawned sessions, including parent, root, trace, and depth fields exposed through session APIs.

## Best Possible Solution

Land a focused additive core session-chain contract that persists optional rootSessionKey and traceId, derives them consistently for native and ACP child sessions, exposes them through gateway and agent session-list projections, documents the fields, and proves multi-hop reconstruction without expanding spawn permissions.

## Reproduction Assessment

Yes, at source level. Current main lacks rootSessionKey and session-chain traceId in the persistent model, protocol schema, gateway/tool projections, and native/ACP spawn writes.

## Solution Assessment

Unclear until maintainers choose the traceId/root semantics. A narrow additive metadata contract is the maintainable shape, while recursive listing, traversal RPCs, and UI graphs should remain separate follow-up decisions.
