## Summary

Keep open: current main and the latest release still lack the requested first-class root/trace session-chain contract. Recent ACP lineage work exposes existing parent/depth metadata to ACP clients, but it does not add persisted rootSessionKey or session-chain traceId fields, and the direct implementation PR is closed unmerged.

## What This Changes

Request first-class session/task chain metadata so spawned sessions consistently expose parent, root, trace, and depth lineage through session APIs and tools.

## Best Possible Solution

Define and land a focused additive core session-chain contract: optional persisted rootSessionKey and traceId, consistent native/ACP spawn derivation, gateway and agent-list projection, docs, and multi-hop reconstruction tests.

## Reproduction Assessment

Yes, at source level. Current main and v2026.6.6 expose partial parent/depth lineage, but the inspected stored model, gateway schema/projections, agent-facing rows, and native/ACP spawn writes still lack rootSessionKey and a session-chain traceId.

## Solution Assessment

Unclear until maintainers choose the trace/root API semantics. The narrow maintainable path is additive metadata and projection coverage, not a traversal API or broader orchestration framework in the same change.
