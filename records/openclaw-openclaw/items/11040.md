## Summary

Current main still has only partial session lineage: parent/depth relationships are persisted and exposed, including through ACP, but there is no canonical root key or durable multi-hop chain identifier. The merged ACP projection PR improves consumers without solving that storage contract, both direct implementation PRs are closed, and the remaining identity/lifecycle choice needs maintainer judgment.

## What This Changes

Add optional canonical parent, root, trace, and depth metadata for spawned session chains and expose it through gateway and agent-facing session listings.

## Best Possible Solution

Define one canonical session-chain identity contract—especially its lifetime and relationship to run/task tracing—then persist it atomically at native and ACP spawn boundaries and project the same optional fields through gateway, tool, and ACP surfaces with multi-hop tests.

## Reproduction Assessment

Yes at source level: current main clearly persists and exposes parent/depth lineage while lacking the requested durable root and chain identity. This is a feature-gap verification rather than a failing runtime bug reproduction.

## Solution Assessment

Unclear until maintainers choose the identity semantics. Optional persisted metadata is a reasonable narrow shape, but adding traceId is not the best fix unless it is explicitly distinguished from or unified with existing run/task correlation identifiers.
