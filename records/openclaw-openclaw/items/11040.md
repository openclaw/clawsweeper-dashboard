## Summary

Keep open: current main still has only partial spawned-session lineage. It exposes `spawnedBy`, `parentSessionKey`, `spawnDepth`, child links, and ACP serialization for existing fields, but the requested persisted `rootSessionKey` and session-chain `traceId` contract is still missing from storage, Gateway/tool projections, and native/ACP spawn writes.

## What This Changes

The issue requests additive first-class session/task chain metadata so spawned sessions expose parent, root, trace, and depth information through storage and session listing APIs.

## Best Possible Solution

Land a focused additive session-chain contract that persists optional `rootSessionKey` and `traceId`, derives them consistently for native and ACP children, exposes them through Gateway and agent session-list projections, documents the fields, and covers multi-hop reconstruction without changing spawn policy.

## Reproduction Assessment

No runtime bug reproduction is applicable because this is a feature request, but there is a high-confidence source-level feature-gap check. Current main lacks `rootSessionKey` and session-chain `traceId` in the storage type, Gateway schema/projections, agent `sessions_list` rows, and native/ACP spawn writes.

## Solution Assessment

Yes. The proposed Phase 1 optional metadata contract is the narrow maintainable direction; merged ACP metadata work and the open `sessions.resolve` lineage PR are adjacent but do not replace the spawn-chain root/trace contract.
