## Summary

Keep open: current main still only implements partial spawned-session lineage. `spawnedBy`, `parentSessionKey`, and `spawnDepth` exist in some storage and projection paths, and the merged ACP bridge serializes those existing fields, but the requested persistent `rootSessionKey` and session-chain `traceId` contract is still absent from the model, Gateway/tool rows, and spawn write paths.

## What This Changes

The issue requests optional first-class spawned-session chain metadata so callers can reconstruct parent, root, trace, and depth relationships across multi-hop subagent sessions.

## Best Possible Solution

Land a focused additive core session-chain contract: persist optional `rootSessionKey` and `traceId`, derive them consistently for native and ACP child sessions, expose them through Gateway and agent session-list projections, document the fields, and cover multi-hop reconstruction while preserving compatibility for older entries.

## Solution Assessment

Yes. The proposed Phase 1 optional metadata contract is the narrow maintainable direction; merged ACP metadata work and related UI/lifecycle reports are adjacent but do not replace the core root/trace spawn-chain contract.
