## Summary

Keep open: current main and the latest release still provide only partial spawned-session lineage. `spawnedBy`, `parentSessionKey`, and `spawnDepth` exist in some paths, but the requested persisted `rootSessionKey` plus session-chain `traceId` contract is still missing from the session model, Gateway schema/projections, agent-facing list rows, and native/ACP spawn writes.

## What This Changes

Add optional first-class session/task chain metadata so spawned sessions expose parent, root, trace, and depth information consistently for DAG reconstruction.

## Best Possible Solution

Land a focused additive core contract that persists optional `rootSessionKey` and session-chain `traceId`, derives them for native and ACP spawned sessions, projects them through Gateway and agent session-list rows, documents the fields, and preserves existing sessions.

## Reproduction Assessment

Yes, source-level. Current main lacks `rootSessionKey` and session-chain `traceId` in the persistent model, Gateway/tool projections, and native/ACP spawn writes; runtime bug reproduction is not applicable because this is an additive feature request.

## Solution Assessment

Yes. The proposed Phase 1 optional metadata contract is the narrow maintainable path if it stays metadata-only and keeps session-chain `traceId` semantics separate from diagnostic tracing unless maintainers explicitly decide otherwise.
