## Summary

Keep open: current main still has only partial spawned-session lineage. It exposes `spawnedBy`, `parentSessionKey`, `spawnDepth`, child links, and ACP serialization for those existing fields, but it does not implement the requested persisted `rootSessionKey` or session-chain `traceId` contract in storage, Gateway rows, agent-facing `sessions_list`, or native/ACP spawn writes.

## What This Changes

The issue requests first-class session-chain metadata for spawned sessions, including parent/root lineage, trace id, and depth in persisted session data and listing APIs.

## Best Possible Solution

Land a focused additive core session-chain contract that persists optional `rootSessionKey` and `traceId`, derives them consistently for native and ACP child sessions, exposes them through Gateway and agent session-list projections, documents the fields, and proves multi-hop reconstruction without changing spawn policy.
