## Summary

Keep open: current main still provides only partial spawned-session lineage and does not implement the requested durable rootSessionKey/traceId session-chain contract. The direct implementation PR is closed and unmerged, while the merged ACP lineage PR only serializes the existing parent/depth Gateway fields.

## What This Changes

The issue requests optional first-class session-chain metadata on session entries, spawn write paths, Gateway session rows, and agent-facing session list output.

## Best Possible Solution

Land a focused additive core contract that persists optional rootSessionKey and traceId, derives them for native and ACP child sessions, exposes them through Gateway and agent session-list projections, documents the fields, and covers multi-hop reconstruction.
