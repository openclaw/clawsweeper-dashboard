## Summary

Keep this issue open for a maintainer product decision. Current main now provides substantial parent/child lineage, including persisted parent/depth data, child projections, and ACP metadata, but it still does not provide the requested durable root identity, chain identifier, or spawn depth in the agent-facing session-list row. The remaining work is narrower than the original proposal, but it is not implemented or safely superseded.

## What This Changes

Add canonical parent, root, trace, and depth metadata for spawned session chains and expose it through session APIs.

## Best Possible Solution

Define one canonical parent edge across native, ACP, and dashboard-created sessions; expose depth consistently and provide bounded traversal, while deriving root when possible and introducing a distinctly named durable chain identifier only if existing lineage and diagnostic tracing cannot satisfy the use case.

## Reproduction Assessment

Yes at the source-contract level: current main clearly exposes only partial lineage and lacks root identity, a durable chain identifier, and agent-facing spawn depth. This is a feature-gap verification rather than a runtime bug reproduction.

## Solution Assessment

No, the original four-field proposal is not yet the best defined solution: root may be derivable, while traceId already has diagnostic semantics. Maintainers should first define the minimal durable lineage contract and whether traversal or derivation avoids redundant persisted state.
