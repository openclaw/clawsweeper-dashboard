## Summary

This issue is still necessary: current main and the latest release still provide only partial spawned-session lineage, while the requested first-class root/trace session-chain contract is absent from the persisted session model, gateway/session projections, agent-facing sessions_list rows, and native/ACP spawn writes. The merged ACP lineage PR is useful adjacent work, but it serializes existing parent/depth metadata rather than implementing the parent/root/trace/depth contract requested here.

## What This Changes

Add optional first-class session-chain metadata for spawned sessions, including parent, root, trace, and depth fields, and expose it through session listing APIs.

## Best Possible Solution

Land a focused additive core session-chain contract: persist optional rootSessionKey and traceId, derive them consistently for native and ACP child sessions, expose them through gateway and agent session-list projections, document the fields, and cover multi-hop reconstruction while preserving compatibility for older entries.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes, at source level: current main exposes spawnedBy, parentSessionKey, spawnDepth, and childSessions in some paths, while rootSessionKey and traceId are absent from the session model, gateway schema/projections, sessions_list output, and spawn derivation paths.

## Solution Assessment

Is this the best way to solve the issue? Unclear until maintainers choose the root/trace contract, but the requested Phase 1 metadata-only boundary is the narrow maintainable path if that product decision is accepted.
