## Summary

Keep open. Current main and v2026.6.6 still provide partial parent/depth lineage and ACP serialization only; `rootSessionKey` and a session-chain `traceId` are missing from persisted session entries, gateway protocol/rows, agent list rows, and spawn writes, while the matching implementation PR is closed unmerged.

## What This Changes

The issue requests first-class parent/root/trace/depth session-chain metadata for spawned sessions so tools can reconstruct multi-hop subagent DAGs from session APIs.

## Best Possible Solution

Define a narrow additive core session-chain contract that persists optional `rootSessionKey` and `traceId`, derives them in native and ACP spawn writes, exposes them consistently in gateway/tool/ACP projections, documents the semantics, and tests multi-hop reconstruction.

## Reproduction Assessment

Yes, at source level. Current main exposes partial lineage (`spawnedBy`, `parentSessionKey`, `spawnDepth`) but the requested root and trace fields are absent from the persistent model, protocol, projections, and spawn write paths.

## Solution Assessment

Unclear until maintainers choose the trace semantics and API contract. The best implementation shape is a focused additive session-chain metadata contract, not a broad branch that mixes unrelated plugin/provider or lifecycle-hook changes.
