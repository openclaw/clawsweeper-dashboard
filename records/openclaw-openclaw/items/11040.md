## Summary

Keep open: current main and the latest release still provide only partial spawned-session lineage. Parent/depth fields and ACP serialization exist, but the requested persisted rootSessionKey and session-chain traceId contract is still missing from storage, gateway protocol/projections, agent-facing sessions_list rows, and native/ACP spawn writes.

## What This Changes

The issue requests first-class spawned-session chain metadata with parent, root, trace, and depth fields exposed through session APIs.

## Best Possible Solution

Add a narrow additive core session-chain contract: persist optional rootSessionKey and traceId, derive them consistently for native and ACP children, expose them through gateway and agent list projections, document the fields, and cover multi-hop reconstruction.

## Reproduction Assessment

Yes, at source level. Current main exposes partial spawned-session lineage, but the requested rootSessionKey and stable session-chain traceId are absent from the session model, gateway schema/projections, sessions_list rows, and spawn writes.

## Solution Assessment

Unclear until maintainers choose the trace/root contract. The best implementation shape is a narrow additive core metadata change, not a broad orchestration rewrite or wholesale resurrection of the closed PR.
