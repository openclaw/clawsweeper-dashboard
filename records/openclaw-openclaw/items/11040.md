## Summary

Keep open: current main still has partial spawned-session lineage, but not the requested first-class root/trace chain contract in persisted session entries, Gateway protocol rows, agent-facing session listing, or native/ACP spawn writes.

## What This Changes

Request optional parent/root/trace/depth metadata for spawned sessions so tools can reconstruct session/task chains from session APIs.

## Best Possible Solution

Land a focused additive core session-chain contract with optional persisted rootSessionKey and traceId, consistent native/ACP derivation, Gateway and agent-list projection, docs, changelog, and multi-hop regression coverage.

Yes. Source inspection on current main confirms the gap: storage, schema, projections, tests, and spawn writes have spawnedBy/parent/depth pieces but no durable rootSessionKey or session-chain traceId contract.

## Solution Assessment

Yes. The Phase 1 optional metadata contract is the narrowest maintainable first step; chain traversal APIs, UI graphing, and unrelated closed-PR changes should remain separate.
