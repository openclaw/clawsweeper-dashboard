## Summary

Keep open: current main still implements only partial spawned-session lineage. The model and projections carry `spawnedBy`, `parentSessionKey`, and `spawnDepth` in several paths, and the merged ACP work exposes those existing fields, but the requested persisted `rootSessionKey` plus session-chain `traceId` contract is still absent from the core session model, Gateway schema/projections, agent-facing list rows, and spawn write paths.

## What This Changes

The issue requests first-class optional session-chain metadata so spawned sessions expose parent, root, trace, and depth information through session APIs.

## Best Possible Solution

Land a focused additive session-chain contract that persists optional `rootSessionKey` and session-chain `traceId`, derives them in native and ACP spawn paths, projects them through Gateway and agent session-list rows, documents the fields, and preserves older session entries.

## Reproduction Assessment

Yes, source-level. Current main lacks `rootSessionKey` and session-chain `traceId` in the persistent session model, Gateway schema/projections, agent `sessions_list` rows, and native/ACP spawn writes; runtime bug reproduction is not applicable because this is an additive feature request.

## Solution Assessment

Yes. The proposed Phase 1 optional metadata contract is the narrow maintainable path; the merged parent/depth and ACP serialization work are useful foundations but do not replace the missing root/trace contract.
