## Summary

Keep open: current main still has only partial spawned-session lineage. `spawnedBy`, `parentSessionKey`, `spawnDepth`, `childSessions`, and ACP projection of those existing fields are present, but the requested persisted `rootSessionKey` and session-chain `traceId` contract is still missing across the session model, gateway/schema projections, agent-facing `sessions_list`, and native/ACP spawn writes.

## What This Changes

Request first-class spawned-session chain tracking with parent, root, trace, and depth metadata exposed through session APIs and tools.

## Best Possible Solution

Approve a focused additive core session-chain contract that persists optional `rootSessionKey` and `traceId`, derives them consistently for native and ACP child sessions, exposes them through gateway and agent session-list projections, documents the fields, and leaves traversal/UI work as follow-ups.

## Reproduction Assessment

Yes, source-level. Current main’s session entry type, gateway protocol schema, gateway row projection, agent-facing `sessions_list` row, and native/ACP spawn writes can be inspected directly and still lack `rootSessionKey` plus a session-chain `traceId`.

## Solution Assessment

Unclear until maintainers choose trace semantics. A narrow additive session-chain metadata contract is the maintainable direction, but it should align with existing diagnostic trace context and adjacent resolver-lineage work.
