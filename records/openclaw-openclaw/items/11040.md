## Summary

Keep open: current main still has only partial spawned-session lineage and does not implement the requested first-class root/trace chain contract across storage, Gateway protocol rows, agent-facing session listing, and native/ACP spawn writes.

## What This Changes

Add optional parent/root/trace/depth metadata to spawned sessions so OpenClaw can reconstruct multi-hop session chains from session APIs.

## Best Possible Solution

Land a focused additive core session-chain contract: persist optional rootSessionKey and traceId, derive them consistently for native and ACP child sessions, expose them through Gateway and agent session-list projections, document compatibility for older entries, and test multi-hop reconstruction.

## Reproduction Assessment

Yes. Source inspection on current main confirms the requested gap: the model, schema, projections, and spawn patches have spawnedBy/parent/depth pieces but no durable rootSessionKey or session-chain traceId contract.

## Solution Assessment

Yes. The proposed Phase 1 optional metadata is the narrowest maintainable first step; chain traversal APIs, UI graphs, and unrelated closed-PR changes should remain separate.
