## Summary

Keep open: current main and the latest release still provide only partial parent/depth lineage, while the requested rootSessionKey and session-chain traceId contract is absent from the persisted model, Gateway/session projections, agent-facing list output, and native/ACP spawn writes.

## What This Changes

This issue requests first-class spawned-session chain metadata with parent, root, trace, and depth fields exposed through session APIs and tool listings.

## Best Possible Solution

Define a focused additive core session-chain contract, derive it consistently in native and ACP spawn paths, expose it through Gateway and agent session-list projections, and document the trace/root semantics before implementation.

## Reproduction Assessment

Not applicable as a bug reproduction, but source inspection gives a high-confidence feature-gap check: current main lacks rootSessionKey and session-chain traceId in the model, protocol, projections, and spawn writes.

## Solution Assessment

Unclear as a final API design. The requested additive metadata is plausible and bounded, but maintainers still need to choose whether traceId is a new session-chain identifier or bridges existing diagnostic trace context.
