## Summary

Keep open: current main and v2026.6.5 still only provide partial spawned-session lineage. ## What This Changes

Add first-class spawned-session chain metadata so OpenClaw can reconstruct parent/root/trace/depth relationships through session APIs and tool listings.

## Best Possible Solution

Land a focused additive core session-chain contract: persist optional `rootSessionKey` and `traceId`, derive them consistently for native and ACP child sessions, expose them through Gateway and agent session-list projections, document the fields, and cover multi-hop reconstruction while preserving older entries.
