## Summary

Keep open: current main still has only partial spawned-session lineage. The requested durable `rootSessionKey` and session-chain `traceId` contract is absent from session storage, Gateway protocol/projections, agent-facing `sessions_list`, and native/ACP spawn writes; the directly linked implementation PR is closed and unmerged.

## What This Changes

Request adds optional parent/root/trace/depth session-chain metadata for spawned sessions and exposes it through session list APIs and tooling.

## Best Possible Solution

Land a focused additive session-chain contract that persists optional root/trace metadata, derives it consistently for native and ACP child sessions, and exposes it through Gateway and agent session-list projections while preserving compatibility for existing entries.

## Reproduction Assessment

Yes. Source inspection on current main is enough for this missing API feature: storage, schema, row projection, agent tool output, and spawn-write surfaces expose only parent/depth pieces and no durable `rootSessionKey` or session-chain `traceId`.

## Solution Assessment

Yes. The Phase 1 optional metadata contract is the narrowest maintainable direction; traversal APIs, UI graphing, and descendant termination should remain separate follow-up work.
