## Summary

Keep open: current main still implements only partial spawned-session lineage. The durable session model, Gateway protocol/schema projections, agent-facing sessions_list output, and native/ACP spawn write paths still lack the requested rootSessionKey and session-chain traceId contract, and the related implementation PR is closed and unmerged with no open replacement PR found.

## What This Changes

Add first-class, backward-compatible session/task chain metadata so spawned sessions expose parent, root, trace, and depth information for deterministic DAG reconstruction.

## Best Possible Solution

Land a focused additive core session-chain contract: persist optional rootSessionKey and traceId, derive parent/root/trace/depth consistently for native and ACP child sessions, expose the metadata through Gateway and agent session-list projections, document it, and cover multi-hop reconstruction while preserving compatibility for older entries.

## Reproduction Assessment

Yes. This is a source-level feature gap: inspecting the current session model, Gateway schema/projections, sessions_list row builder, and native/ACP spawn patches shows the requested rootSessionKey and session-chain traceId are absent.

## Solution Assessment

Yes. The issue's Phase 1 optional metadata contract is the narrowest maintainable fix because reliable reconstruction needs persisted lineage on session entries; recursive traversal APIs and UI graphing should remain follow-up work.
