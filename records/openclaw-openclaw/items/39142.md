## Summary

Keep open: current main still has structured fallback exhaustion handling and safer built-in user text, but it does not provide a supported configurable message for billing or rate-limit model exhaustion.

## What This Changes

Request to add a `messages.modelExhausted`-style config key so all-model billing/rate-limit exhaustion can send operator-defined friendly text.

## Best Possible Solution

Add a documented optional `messages.modelExhausted` string, validate and label it in the config schema, and apply it only at the terminal fallback-exhaustion collapse point for billing/rate-limit exhaustion while preserving current built-in text when unset.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: the proposed `messages` key is absent from the type and strict schema, and the terminal `FallbackSummaryError` branch currently returns built-in billing/rate-limit/generic text.

## Solution Assessment

Yes. A first-class `messages` config key applied where `FallbackSummaryError` is collapsed is the narrowest maintainable solution; a plugin hook rewrite would not have a supported reply path for this case.
