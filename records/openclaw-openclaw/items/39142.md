## Summary

Keep open: current main still has only built-in terminal fallback copy, and the strict `messages` config schema still rejects the proposed override, so the requested operator-configurable exhaustion message remains missing.

## What This Changes

Add a configurable `messages.modelExhausted`-style user-facing message for billing/rate-limit terminal model fallback exhaustion.

## Best Possible Solution

Add a documented optional `messages.modelExhausted` string, validate it in config, and apply it only at terminal billing/rate-limit model fallback exhaustion while preserving existing text when unset.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `messages.modelExhausted` is not accepted by the strict schema, and a terminal `FallbackSummaryError` currently returns built-in billing/rate-limit/generic text.

## Solution Assessment

Yes. A first-class `messages` config key at the terminal fallback collapse point is narrower than a plugin workaround because `agent_end` is observational and the reply runner already has structured fallback reasons.
