## Summary

Keep open. Current main no longer exposes the raw all-models-failed text for pure billing or rate-limit fallback summaries, but the requested configurable `messages.*` user-facing text is still not present and needs a maintainer product/config decision.

## What This Changes

The issue asks for configurable friendly user-facing text when all fallback models fail because of expected billing or rate-limit exhaustion.

## Best Possible Solution

Decide the product surface first, then implement a documented typed `messages.*` option or plugin-hook alternative that only overrides safe billing/rate-limit exhaustion replies and keeps raw provider errors out of user-facing chat.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection is high-confidence that current main has built-in billing/rate-limit fallback replies but no configurable `messages.modelExhausted`-style key in the type or strict schema.

## Solution Assessment

Unclear. A configurable core message is plausible, but current main already solves the raw-error exposure for pure billing/rate-limit summaries, so maintainers should decide whether customization belongs in core config or the evolving plugin hook surface.
