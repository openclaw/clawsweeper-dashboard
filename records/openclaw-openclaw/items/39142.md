## Summary

Keep open: current main still has no supported configurable model-exhaustion message, and the terminal fallback path still chooses built-in billing, rate-limit, or generic text.

## What This Changes

Request a supported `messages.modelExhausted`-style config option that replaces terminal all-models-exhausted billing/rate-limit failures with a configured friendly user-facing message.

## Best Possible Solution

Add a documented optional `messages.modelExhausted` string, validate it in the config schema/help/labels/docs, and apply it only at the terminal `FallbackSummaryError` collapse point for billing/rate-limit exhaustion while preserving current built-in text when unset.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: the proposed key is rejected by the strict `messages` schema, and an injected `FallbackSummaryError` currently returns built-in billing/rate-limit/generic text.

## Solution Assessment

Yes. A first-class `messages` config key is the narrow maintainable solution because the reply path already has structured fallback reasons; a plugin-only rewrite would be less precise and is not currently a supported reply surface for this failure.
