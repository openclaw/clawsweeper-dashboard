## Summary

Keep open. Current main now avoids the raw billing/rate-limit `All models failed` text in the main before-reply path, but it still uses fixed built-in copy and has no documented `messages.*` or plugin/API contract for operator-supplied model-exhaustion text.

## What This Changes

Request a configurable user-facing message, such as `messages.modelExhausted`, for all-models-exhausted billing or rate-limit failures.

## Best Possible Solution

Choose one public customization surface, then route billing/rate-limit all-models-exhausted replies through it with schema, exported types, docs/help, and focused regression coverage.

## Reproduction Assessment

Yes, source-reproducible. Current main has structured fallback summaries and fixed billing/rate-limit reply branches, while the strict messages schema and exported messages type contain no custom exhaustion-copy field.

## Solution Assessment

Unclear. A `messages.modelExhausted` key is plausible, but maintainers should decide whether core config or a terminal-failure plugin/reply hook is the durable customization surface.
