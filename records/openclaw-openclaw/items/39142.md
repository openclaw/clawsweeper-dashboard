## What This Changes

Add a public way to customize the user-facing message shown when all configured models are exhausted by billing or rate-limit failures.

## Best Possible Solution

Choose one public customization surface, then route billing/rate-limit all-models-exhausted replies through it with schema, exported types, docs/help, and focused regression coverage.

## Reproduction Assessment

Yes, source-level. Current main has structured fallback summaries and fixed billing/rate-limit reply branches, while the strict messages schema and exported messages type contain no custom exhaustion-copy field.

## Solution Assessment

Unclear. A `messages.modelExhausted` key is plausible, but maintainers should decide whether core config, the terminal-failure hook path, or a broader reply customization surface is the durable solution.
