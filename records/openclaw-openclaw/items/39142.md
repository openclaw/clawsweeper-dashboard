## Summary

Keep open: current main still lacks a supported configurable model-exhaustion message, and the terminal fallback path still selects built-in billing, rate-limit, or generic text instead of a `messages` override.

## What This Changes

The issue requests an optional user-facing `messages.modelExhausted`-style config value for all-models-exhausted billing or rate-limit fallback failures.

## Best Possible Solution

Add a documented optional `messages.modelExhausted` string, validate it in the config schema, and apply it only at the terminal `FallbackSummaryError` collapse point for billing/rate-limit exhaustion while preserving current built-in text when unset.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: the proposed config key is rejected by the strict `messages` schema, and a `FallbackSummaryError` injected into `runAgentTurnWithFallback` currently returns built-in billing/rate-limit/generic text.

## Solution Assessment

Yes. A first-class `messages` config key is the narrowest maintainable solution because the reply path already has structured fallback reasons; a plugin-only text rewrite would be less precise and would not provide a supported core config surface.
