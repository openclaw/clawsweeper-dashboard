## Summary

Keep open. Current main no longer blindly exposes the raw `All models failed` summary for pure billing or rate-limit exhaustion, but the requested operator-configurable user-facing exhaustion message does not exist in the strict config schema, typed config surface, or docs.

## What This Changes

The issue requests a configurable friendly user-facing reply when every configured model fails due to budget, billing, or rate-limit exhaustion.

## Best Possible Solution

Pick one public contract: either add a documented `messages.*` configuration field for terminal model-exhaustion copy with schema/types/docs/tests, or land a plugin hook path that lets operators provide custom copy without adding another core message knob.

## Reproduction Assessment

Yes, from current-main source inspection: the fallback exhaustion path is present and uses hard-coded billing/rate-limit text, while the strict `messages` schema and exported type have no requested custom-copy key. I did not run a live provider-quota exhaustion test.

## Solution Assessment

Unclear. The proposed `messages.modelExhausted` direction is plausible and narrow, but maintainers should first decide whether core config or the pending plugin terminal-failure hook is the better long-term customization surface.
