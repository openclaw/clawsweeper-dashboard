## Best Possible Solution

Choose one public customization surface, then route billing/rate-limit all-models-exhausted replies through it with schema, exported types, help/docs, and focused regression coverage.

Unclear. A `messages.modelExhausted` key is plausible, but maintainers should first decide whether core config or a terminal-failure plugin hook is the durable customization surface.
