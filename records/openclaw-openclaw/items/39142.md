## Summary

Keep open. Current main has better built-in billing and rate-limit fallback-exhaustion copy, but it still does not provide the requested operator-configurable user-facing message in the strict config schema, exported message type, or docs.

## What This Changes

Request a `messages.modelExhausted`-style config option to replace all-models-exhausted billing or rate-limit errors with operator-provided text.

## Best Possible Solution

Choose one public customization surface, then wire it through the agent-runner terminal failure path with schema, exported types, docs, and focused regression coverage.

## Reproduction Assessment

Yes. Current-main source inspection shows the fallback-exhaustion path is classifiable but uses hard-coded billing/rate-limit text, while the strict message config contract has no custom-copy field; I did not run a live quota-exhaustion provider scenario.

## Solution Assessment

Unclear. A `messages.modelExhausted` key is plausible, but maintainers should first decide whether core config or the pending plugin terminal-failure hook is the cleaner long-term customization surface.
