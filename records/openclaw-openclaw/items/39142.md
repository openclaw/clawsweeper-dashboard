## Summary

Keep open: current main has safer built-in fallback text for terminal all-model failures, but it still has no supported configurable `messages.modelExhausted`-style override and no canonical PR or issue implementing that config surface.

## What This Changes

Request a supported messages config key that replaces terminal billing/rate-limit all-model fallback failures with operator-defined user-facing text.

## Best Possible Solution

Add a documented optional messages key for terminal billing/rate-limit all-model exhaustion and apply it only after structured fallback classification, leaving existing sanitized built-in messages as the default.

## Reproduction Assessment

Yes. Source inspection on current main shows the proposed key is absent from the strict `messages` schema while the terminal fallback path still selects built-in text from structured `FallbackSummaryError` reasons.

## Solution Assessment

Yes in direction, but it needs maintainer product approval for the public config contract. A documented optional `messages` key applied only to terminal billing/rate-limit exhaustion is the narrowest maintainable path.
