## Summary

Keep open: current main has safer built-in fallback copy for billing/rate-limit exhaustion, but it still does not provide a supported configurable `messages.modelExhausted` user-facing override.

## What This Changes

The issue requests a config key for replacing all-models-exhausted billing/rate-limit failures with operator-provided friendly text.

## Best Possible Solution

Add a documented optional `messages.modelExhausted` string and apply it only at the terminal `FallbackSummaryError` collapse point for billing/rate-limit exhaustion, preserving current built-in text when unset or when failures are unrelated.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: the proposed key is rejected by the strict `messages` schema, and a terminal `FallbackSummaryError` currently returns built-in billing, rate-limit, or generic text rather than a config override.

## Solution Assessment

Yes. A first-class `messages` config key is the narrowest maintainable route because the reply path already has structured fallback reasons; an `agent_end` plugin cannot reliably replace the already-finalized user-facing reply.
