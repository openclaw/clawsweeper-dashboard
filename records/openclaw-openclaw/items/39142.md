## Summary

Keep open: current main has safer built-in fallback text for terminal billing/rate-limit exhaustion, but it still does not expose a supported configurable `messages.modelExhausted` override.

## What This Changes

Request a configurable `messages.modelExhausted` user-facing reply for terminal all-model billing or rate-limit exhaustion.

## Best Possible Solution

Add a documented optional `messages.modelExhausted` string and apply it only when terminal fallback attempts are exhaustion reasons such as billing or rate-limit, preserving current built-in text when unset or when failures are unrelated.

## Reproduction Assessment

Yes. Source inspection on current main shows the proposed key is absent from the strict `messages` schema, while terminal `FallbackSummaryError` handling chooses built-in billing, rate-limit, or generic text instead of a config override.

## Solution Assessment

Yes. A first-class `messages` config key is the narrowest maintainable path because the reply path already has structured fallback reasons and `agent_end` is documented as an observation hook after the turn.
