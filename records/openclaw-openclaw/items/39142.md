## Summary

Keep open. Current main still lacks a supported configurable message for all-model fallback exhaustion: the `messages` config contract has no `modelExhausted` key, the strict schema would reject it, and the before-reply failure path still selects built-in billing, rate-limit, or generic text. The report remains concrete, current, and narrow enough for a focused repair.

## What This Changes

The issue requests a supported `messages.modelExhausted` configuration key that replaces raw or built-in all-models-exhausted failure text when every failed model attempt is due to billing or rate-limit exhaustion.

## Best Possible Solution

Expose `messages.modelExhausted` as a first-class message config option and apply it only to all-model billing/rate-limit exhaustion, while preserving the current built-in messages when the key is unset or the failure reasons fall outside that exact condition.
