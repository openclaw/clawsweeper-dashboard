## Summary

Keep open: current main and v2026.6.8 still have setup-time realtime agent context and consult-time fast context, but no supported async per-call preload hook that can enrich realtime instructions from inbound caller metadata before the first spoken turn. The adjacent latency/prewarm issue is related but distinct, and the closed private-objective issue does not cover inbound caller/contact preload.

## What This Changes

The issue requests a first-class async pre-answer preload hook for realtime voice agents so per-call caller/contact context can be loaded before the first spoken model turn.

## Best Possible Solution

Design a maintainer-approved voice-call or plugin-SDK lifecycle seam with bounded metadata, a hard fail-open timeout, sanitized per-session context injection, redacted logging, and coordination with the adjacent provider prewarm work.

## Reproduction Assessment

No as a bug reproduction; this is a missing feature/API seam. Source inspection gives a high-confidence current-main path showing instructions are setup-time while caller metadata arrives later, so the requested pre-first-turn preload is absent.

## Solution Assessment

Yes directionally, but not as an automated fix. A stable hook is the maintainable replacement for compiled-runtime patching, while the API shape, timeout budget, privacy contract, and cross-channel scope need maintainer approval first.
