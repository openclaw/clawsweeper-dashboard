## Summary

Keep open: current main and v2026.6.8 still provide setup-time realtime agent context and consult-time fast context, but no supported async pre-answer hook that can use per-call metadata, preload caller/contact data, and inject sanitized session-scoped context before the first realtime voice turn.

## What This Changes

Requests a first-class async pre-answer context preload hook for realtime voice calls so caller/contact context can be loaded and injected before the first model turn.

## Best Possible Solution

Design an additive Voice Call or plugin-SDK lifecycle seam that receives bounded call/message metadata, runs under a hard fail-open timeout, injects sanitized per-session context before the first realtime response, and documents privacy, logging, and cross-channel behavior.

## Reproduction Assessment

No, not as a bug reproduction; this is a feature/API request. Source inspection gives a high-confidence current-main path showing the requested pre-first-turn lifecycle seam is absent.

## Solution Assessment

Unclear as an immediate implementation because maintainers still need to choose a Voice Call-only hook versus a generic pre-reply lifecycle seam. Directionally, a stable fail-open hook is the maintainable replacement for compiled-runtime patching.
