## Summary

Keep open: current main still has realtime voice context features, but not the requested async pre-answer/per-call preload lifecycle seam. The recent Discord side-agent comment broadens the same product/API question rather than making this stale or duplicative.

## What This Changes

Request an async pre-answer context preload hook for realtime voice agents so caller/contact context can be injected before the first spoken turn.

## Best Possible Solution

Design an additive, typed lifecycle seam that receives bounded call or channel metadata, runs under a hard timeout, injects sanitized per-session context before the first response, and documents privacy/logging behavior.

## Reproduction Assessment

Not applicable; this is a feature request, not a broken existing contract. Source inspection on current main gives a high-confidence path showing no pre-answer/preload hook in the voice-call config, TwiML flow, bridge request, or provider contract.

## Solution Assessment

Yes in direction, but not as an automated bug fix. A first-class lifecycle hook is the maintainable alternative to compiled-runtime patching, while the API shape, timeout budget, privacy boundary, and voice-vs-generic scope need maintainer review.
