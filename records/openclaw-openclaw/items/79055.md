## Summary

Keep open: current main has realtime voice, static `agentContext`, consult-time `fastContext`, and normal plugin prompt hooks, but it still lacks an async per-call pre-answer preload hook that can enrich realtime instructions before the first spoken turn. The newer Discord/active-memory comment broadens the same lifecycle/latency concern, so this needs maintainer API and privacy review rather than cleanup closure.

## What This Changes

Request an async pre-answer context preload hook for realtime voice agents that can load sanitized caller/contact context with a timeout before the first model turn.

## Best Possible Solution

Design an additive, default-off Voice Call or plugin-SDK lifecycle seam that receives bounded call metadata, runs under a hard timeout, injects sanitized per-call context into realtime instructions before first speech, and reuses or generalizes the pattern for Discord only if maintainers want a shared low-latency preload contract.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing behavior. Source inspection gives a high-confidence current-main check showing the requested pre-answer lifecycle seam is absent from config, webhook, runtime, docs, and provider contracts.

## Solution Assessment

Yes in direction, but not as an automated bug fix. A first-class preload hook is the maintainable replacement for compiled-runtime patching, but the API shape, timeout, privacy, logging, and cross-channel ownership need maintainer approval first.

## Risks / Open Questions

- The hook would run before answer/stream setup, so maintainers need to define a tight latency budget and fail-open behavior that does not degrade inbound call connection time.
- Caller/contact/property context can contain PII, so the API needs explicit sanitization, scoping, and log-redaction rules before implementation.
- The Discord comment suggests this may be broader than Voice Call alone, so maintainers should decide whether the seam is plugin-local, channel-generic, or a shared agent lifecycle contract.
