## Summary

Keep open: current main has realtime voice, startup agent context, consult-time fast context, and per-number routing, but no supported async per-call preload seam before the first realtime turn. The newer Discord comment reinforces that this is a cross-channel lifecycle/API design request rather than a stale or already-implemented voice-only bug.

## What This Changes

Request a first-class async pre-answer context preload hook for realtime voice agents so caller/contact context can be injected before the model's first turn.

## Best Possible Solution

Design an additive voice-call or plugin-SDK lifecycle seam that receives bounded call metadata, runs under a hard timeout, injects sanitized per-call context into realtime instructions, and logs only redacted success/failure state.

## Reproduction Assessment

Not applicable; this is a feature request rather than a bug report. Source inspection gives a high-confidence current-main path showing the requested pre-first-turn lifecycle seam is absent.

## Solution Assessment

Yes directionally, but not as an automated bug fix. A stable hook is the maintainable replacement for compiled-runtime patching, but the API shape, timeout budget, privacy contract, and cross-channel scope need maintainer approval first.
