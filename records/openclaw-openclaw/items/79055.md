## Summary

Keep open: current `main` still lacks an async, fail-open per-call preload hook before the first realtime voice turn, and the recent related pull request is only a partial/static caller-context proposal with unresolved privacy review notes.

## What This Changes

The issue requests an async, configurable pre-answer context preload hook for realtime voice agents so caller/contact/property context can be injected into the first session instructions without patching compiled runtime code.

## Best Possible Solution

Design an additive lifecycle seam that accepts bounded call/session metadata, runs a short-timeout async preload, injects only sanitized per-session context into first-turn realtime instructions, and uses https://github.com/openclaw/openclaw/pull/75592 only as partial input after its identity/privacy concerns are resolved.

## Reproduction Assessment

Not applicable; this is a feature request rather than a broken existing contract. Source inspection gives a high-confidence current-main path showing the requested pre-answer preload lifecycle seam is absent.

## Solution Assessment

Yes in direction, but not as an automated bug fix. A first-class hook is the maintainable alternative to compiled-runtime patching, while the API shape, timeout budget, privacy contract, and possible cross-agent scope need maintainer approval.

## Risks / Open Questions

- A preload hook can delay Twilio answer/stream setup, so timeout and fail-open semantics need an explicit product decision.

- The Discord-side comment suggests the desired seam may extend beyond Voice Call, so a voice-only API might be too narrow without maintainer review.
- The related static caller-context pull request shows useful direction but also demonstrates caller-ID assurance and transcript privacy hazards.
