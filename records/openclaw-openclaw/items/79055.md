## Summary

Keep open: current main has realtime voice, static startup agent context, consult-time fast context, and per-number routing, but it still lacks a first-class async per-call preload hook before the first realtime turn. The open caller-context PR is related but narrower and already has security/design blockers, so this needs maintainer API and privacy review rather than cleanup closure.

## What This Changes

The issue requests a configurable fail-open async preload hook for realtime voice calls so caller/contact context can be sanitized and injected before the first model response.

## Best Possible Solution

Design an additive Voice Call or plugin-SDK lifecycle seam that receives bounded call metadata, runs under a tight timeout, returns sanitized per-call context for realtime instructions, fails open, and coordinates or supersedes https://github.com/openclaw/openclaw/pull/75592 after security review.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives a high-confidence current-main path showing that the requested async pre-answer preload seam is absent.

## Solution Assessment

Yes in direction, but not as a narrow automated fix. A first-class bounded lifecycle hook is the maintainable alternative to compiled-runtime patching, while the exact API, timeout, caller-identity, privacy, and cross-channel semantics need maintainer approval.
