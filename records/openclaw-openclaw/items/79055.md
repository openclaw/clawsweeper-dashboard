## Summary

Keep open: current main still supports static realtime voice context, consult-time fast context, and per-number routing, but not an async per-call pre-answer preload hook. The recent Discord/active-memory comment broadens this into an API/product design question rather than a cleanup-close candidate.

## What This Changes

Request a first-class async pre-answer context preload hook for realtime voice calls that can enrich per-call instructions from caller metadata before the first model turn.

## Best Possible Solution

Design an additive plugin-facing lifecycle seam with bounded call/message metadata, hard timeout and fail-open behavior, sanitized per-session instruction injection, and redacted diagnostics.

## Reproduction Assessment

Not applicable; this is a feature request rather than a bug. Source inspection on current main gives a high-confidence path showing that the requested pre-answer lifecycle seam is absent.

## Solution Assessment

Yes in direction, but not as an automated fix. A stable hook is the maintainable alternative to compiled-runtime patching, but the voice-vs-generic API boundary, timeout budget, privacy rules, and logging contract need maintainer review first.
