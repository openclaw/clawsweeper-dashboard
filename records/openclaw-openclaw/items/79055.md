## Summary

Keep open: current main still supports static startup voice context and consult-time fast context, but not an async per-call pre-answer preload hook before the first realtime model turn. The fresh Discord/active-memory follow-up points to a broader lifecycle/API design question, so this needs maintainer product review rather than cleanup closure.

## What This Changes

Request an async pre-answer context preload hook so realtime voice calls can enrich session instructions before the first model turn.

## Best Possible Solution

Add an additive, typed lifecycle seam that receives bounded call metadata, runs with a hard timeout, injects sanitized per-call context into realtime instructions before the first turn, and logs only redacted success/failure state.

## Reproduction Assessment

Not applicable; this is a feature request rather than a broken existing contract. Source inspection gives a high-confidence current-main path showing the requested lifecycle seam is absent.

## Solution Assessment

Yes directionally, but not as an automated bug fix. A first-class hook is the maintainable alternative to compiled-runtime patching, but the API boundary, timeout budget, privacy behavior, and plugin ownership need maintainer approval first.
