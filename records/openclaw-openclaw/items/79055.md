## Summary

Keep open: current main has realtime voice, startup `agentContext`, consult-time `fastContext`, and generic prompt hooks, but it still lacks an async per-call preload hook before the first realtime voice turn. The latest Discord-side-agent comment reinforces this as a lifecycle/API design request, not a cleanup candidate.

## What This Changes

Request a pre-answer context preload lifecycle hook so realtime voice agents can fetch caller/contact context and inject sanitized per-call instructions before the first model turn.

## Best Possible Solution

Design an additive lifecycle seam that receives bounded call/message metadata, runs with a hard timeout, fails open, injects sanitized per-session context before the first response, and logs only redacted status.

## Reproduction Assessment

Not applicable; this is a feature request rather than a broken existing contract. Source inspection gives a high-confidence current-main path showing the requested pre-answer/per-call preload seam is absent.

## Solution Assessment

Yes in direction, but not as an automated repair. A first-class lifecycle hook is the maintainable alternative to compiled-runtime patching, while API shape, timeout budget, privacy, and cross-channel ownership need maintainer approval first.
