## Summary

Keep open: current main and v2026.6.8 still provide static realtime instructions, setup-time agent context, and consult-time fast context, but no supported async pre-answer hook that can use inbound call metadata to preload caller/contact context before the first realtime voice response.

## What This Changes

Request a first-class pre-answer context preload hook for realtime voice agents that can fail open, use per-call metadata, sanitize private context, and inject it before the first model turn.

## Best Possible Solution

Design an additive Voice Call or narrow plugin-SDK lifecycle seam that receives bounded call metadata, runs under a hard fail-open timeout, injects sanitized session-scoped context before first realtime speech, and documents privacy, logging, and cross-channel behavior.

## Reproduction Assessment

No, not as a bug reproduction; this is a feature/API request. Source inspection gives a high-confidence current-main path showing the requested pre-first-turn lifecycle seam is absent.

## Solution Assessment

Unclear as an immediate implementation because maintainers still need to choose a Voice Call-only hook versus a generic realtime pre-reply context seam. Directionally, a stable fail-open hook is the maintainable replacement for compiled-runtime patching.
