## Summary

Keep open: current main still supports static realtime instructions, startup agent context, and consult-time fast context, but not the requested async per-call preload before the first realtime voice turn. The newer Discord-side comment strengthens that this is a cross-surface lifecycle/API design request, not stale cleanup.

## What This Changes

The issue asks OpenClaw to run an async, timeout-bounded caller/contact context preload before a realtime voice agent's first spoken turn.

## Best Possible Solution

Design an additive voice/realtime lifecycle seam that receives bounded call or channel metadata, runs under a hard timeout, returns sanitized call-scoped context for first-turn instructions, and logs only redacted status.

## Reproduction Assessment

Not applicable; this is a feature request rather than a broken existing contract. Source inspection gives a high-confidence current-main path showing the requested pre-answer lifecycle seam is absent.

## Solution Assessment

Yes in direction, but not as a narrow automated fix. A first-class hook is the maintainable replacement for compiled-runtime patching, but the hook scope, timeout defaults, privacy rules, and voice-versus-shared ownership need maintainer design first.
