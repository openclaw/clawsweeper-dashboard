## Summary

Keep open: current main and the latest release have setup-time realtime agent context and consult-time fast context, but still no supported async per-call preload hook before the realtime voice session starts. A prior caller-context PR was closed unmerged, so the remaining work is a maintainer API, timeout, privacy, and cross-channel design decision rather than implemented or superseded work.

## What This Changes

Requests a stable async pre-answer context preload hook for realtime voice agents so caller metadata can enrich per-call session instructions before the first spoken turn.

## Best Possible Solution

Design an additive voice-call or plugin-SDK lifecycle seam that receives bounded call/message metadata, runs under a hard fail-open timeout, injects sanitized per-session context before first realtime response, and documents privacy, logging, and cross-channel behavior.

## Reproduction Assessment

No, not as a bug reproduction; this is a feature/API request. Source inspection gives a high-confidence current-main path showing that the requested pre-first-turn lifecycle seam is absent.

## Solution Assessment

Yes directionally, but not as an automated bug fix. A stable hook is the maintainable replacement for compiled-runtime patching, while the API shape, timeout budget, privacy contract, and cross-channel scope need maintainer approval first.
