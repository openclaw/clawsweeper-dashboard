## Summary

Keep open: current main has Voice Call realtime, startup agent context, consult-time fast context, and normal agent-turn hooks, but it still lacks a supported async per-call preload seam that can use caller metadata before the first realtime voice turn. The Discord follow-up makes this a broader lifecycle/API design request, not a stale voice-only bug or duplicate of the shipped fast-context consult path.

## What This Changes

Request a supported async lifecycle hook that preloads caller/contact context before a realtime voice agent speaks first.

## Best Possible Solution

Design an additive voice-call or plugin-SDK lifecycle seam that receives bounded call/message metadata, runs under a hard timeout, injects sanitized per-session context before first reply, and logs only redacted success/failure state.

## Reproduction Assessment

No, not as a bug reproduction; this is a feature request. Source inspection gives a high-confidence current-main path showing the requested pre-first-turn lifecycle seam is absent.

## Solution Assessment

Yes directionally, but not as an automated bug fix. A stable hook is the maintainable replacement for compiled-runtime patching, but the API shape, timeout budget, privacy contract, and cross-channel scope need maintainer approval first.
