## Summary

Keep open: current main supports realtime voice, static agent context, consult-time fast context, and per-number routing, but it does not expose an async per-call preload hook before the first realtime turn.

## What This Changes

The issue requests a first-class asynchronous preload hook so realtime voice agents can load caller/session context before the first spoken model turn.

## Best Possible Solution

Design an additive Voice Call or plugin-SDK lifecycle seam that receives bounded call metadata, runs with a hard timeout, injects sanitized per-call context into realtime instructions, and logs only redacted success/failure state.

## Reproduction Assessment

Not applicable; this is a feature request rather than a bug report. Source inspection gives a high-confidence current-main path showing the requested lifecycle seam is absent.

## Solution Assessment

Yes in direction, but not as an automated bug fix. A first-class voice-call lifecycle hook is the maintainable alternative to compiled-runtime patching, but the API, timeout, privacy, and plugin ownership shape need maintainer approval first.
