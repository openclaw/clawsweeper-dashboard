## Summary

Keep open: current main still lacks an async per-call pre-answer preload seam for realtime voice, and the related open PR is an implementation candidate with unresolved caller-identity and privacy concerns rather than a reason to close this issue.

## What This Changes

Request a first-class async pre-answer context preload hook for realtime voice agents so sanitized caller/session context can be injected before the first model turn.

## Best Possible Solution

Design an additive Voice Call or plugin-SDK lifecycle seam with bounded call metadata, a hard timeout, fail-open behavior, sanitized per-call instruction contribution, and redacted observability, then coordinate it with the related open PR’s trust and storage review.

## Reproduction Assessment

Not applicable; this is a feature request rather than a bug report. Source inspection gives a high-confidence current-main path showing realtime config, TwiML, and bridge setup do not expose an async caller preload seam.

## Solution Assessment

Unclear as an implementation, but the requested direction is reasonable. The best path is a maintainer-reviewed lifecycle/API design rather than ad hoc caller-ID-keyed context injection or compiled-runtime patching.
