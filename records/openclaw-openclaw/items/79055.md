## Summary

Keep open: current main still lacks an async, per-call pre-answer preload seam for realtime voice, and the related open Voice Call PR does not supersede the request because it is still unmerged and raises caller-identity/privacy design questions.

## What This Changes

Request an async, timeout-bounded lifecycle hook that can load caller/contact context and inject sanitized per-call instructions before realtime voice agents speak.

## Best Possible Solution

Design an additive Voice Call or plugin-SDK lifecycle seam that receives bounded call metadata, runs with a strict timeout, injects sanitized per-call context before realtime session creation, and documents identity, privacy, logging, and fail-open behavior.

## Reproduction Assessment

Not applicable; this is a feature/API request rather than broken documented behavior. Source inspection on current main shows the requested lifecycle seam is absent, so the need is source-verifiable but not a bug reproduction.

## Solution Assessment

Yes in direction, but not as an automated bug fix. A first-class hook is the maintainable alternative to compiled-runtime patching, while the API shape, timeout, caller-identity trust model, and privacy rules need maintainer approval first.
