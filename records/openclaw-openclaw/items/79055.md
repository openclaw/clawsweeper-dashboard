## Summary

Keep open: current main still supports static startup agent context, consult-time fast context, and direct Twilio realtime stream setup, but it does not expose an async per-call preload hook before the first realtime turn. The newer Discord/active-memory comment reinforces that this is a broader lifecycle/API design question rather than a stale or already-implemented request.

## What This Changes

Add a first-class async pre-answer context preload hook for realtime voice agents, with timeout/fail-open behavior and sanitized per-call context injection before the first response.

## Best Possible Solution

Design an additive plugin or Voice Call lifecycle seam that receives bounded call metadata, runs with a hard latency budget, injects sanitized per-call context into realtime instructions before the first provider turn, and records only redacted success/failure diagnostics.

## Reproduction Assessment

Not applicable; this is a feature request, not a broken existing contract. Source inspection gives a high-confidence current-main path showing the requested lifecycle seam is absent.

## Solution Assessment

Yes in direction, but not as an automated bug fix. A first-class lifecycle hook is the maintainable alternative to compiled-runtime patching, while service-specific CRM/PropertyMeld enrichment should remain outside core or behind the plugin API.
