## Summary

Keep open. Current main has related realtime agent-context and fast-context support, but not the requested async per-call pre-answer preload hook; this is a new voice-call lifecycle/API design rather than stale cleanup or a ClawHub-only item.

## What This Changes

The issue requests a supported async pre-answer preload hook so realtime voice calls can inject sanitized caller/contact context before the first model turn.

## Best Possible Solution

Design an additive voice-call-owned lifecycle seam that runs after authenticated call metadata is available but before realtime session creation, with bounded timeout, fail-open behavior, per-call scoping, sanitized diagnostics, schema/docs, and focused tests.

## Reproduction Assessment

Not applicable; this is a feature request, not a broken existing contract. Source inspection does confirm the requested lifecycle hook is absent: realtime instructions are built from config and the bridge starts directly once the call stream arrives.

## Solution Assessment

Yes in direction, but not as an automated cleanup fix. A stable hook is a maintainable direction only if it is added as an explicit voice-call/plugin lifecycle contract rather than another local compiled-runtime patch.
