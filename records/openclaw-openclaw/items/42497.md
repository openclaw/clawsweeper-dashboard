## Summary

Keep open. The underlying WS self-contention report still appears valid and current main has not added an in-process gateway dispatch path, but this PR’s local branch bypasses current gateway auth, scope, timeout, and transport-normalization contracts and should not merge as-is.

## What This Changes

This PR adds a gateway local-dispatch registry, registers it during gateway startup, and makes embedded agent gateway tool calls prefer in-process dispatch for non-`expectFinal` calls, with focused tests.

## Best Possible Solution

Rework this into an explicit in-process gateway request adapter that preserves normal option resolution, effective scopes, token intent, bounded timeout behavior, JSON-frame parameter normalization, and final-response semantics, with regression tests for cron, node, session, token, timeout, scope, and undefined-param cases.

## Reproduction Assessment

Yes. A focused unit test can register a local dispatcher and call `callGatewayTool` with `gatewayToken`, `timeoutMs`, explicit `scopes`, or optional `undefined` params to show the local path returns before the current WebSocket contract is applied.

## Solution Assessment

No. In-process dispatch is a plausible direction for the self-contention bug, but this implementation is not the best fix because it early-returns around the existing gateway transport and authorization contract instead of adapting the local path to match it.

  - body: The guard only excludes explicit `gatewayUrl` and `expectFinal`, so calls with `gatewayToken` or `timeoutMs` still take the in-process path. That ignores explicit credential intent and removes the bounded request timeout used by existing gateway tool callers.
  - confidence: 0.88
- **[P2] Normalize params before in-process dispatch:** `src/agents/tools/gateway.ts:154-155`
  - body: The local path forwards the live params object, while the WebSocket path sends a JSON frame. Optional properties set to `undefined` can therefore reach gateway validators locally even though they are omitted over WebSocket, making embedded calls fail that currently succeed.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The diff introduces a new in-process gateway control-plane path without preserving the existing authorization, credential, and timeout contract.

Concerns:

- **[high] Local dispatch bypasses caller authorization intent:** `src/agents/tools/gateway.ts:152`
  - body: The early local return can execute a gateway method before resolving explicit tokens, least-privilege scopes, or explicit scope overrides, so embedded calls may run under ambient or synthetic credentials rather than the caller’s intended authorization contract.
  - confidence: 0.88

  - body: Because `timeoutMs` is not applied on the local path, a stuck in-process handler can block an active agent turn instead of failing under the timeout behavior used by the WebSocket request path.
