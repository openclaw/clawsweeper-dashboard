Full review comments:

- **[P1] Pass resolved scopes into local gateway dispatch:** `src/agents/tools/gateway.ts:156-158`
  - body: This early local-dispatch return bypasses the current `resolveLeastPrivilegeOperatorScopesForMethod` and `extra.scopes` path. On current main, `cron.add` and `cron.run` require `operator.admin`, while the in-process dispatcher defaults synthetic callers to narrower scopes unless options are supplied, so a rebase of this patch either fails the exact cron operations it is meant to fix or has to widen authorization outside the normal scope contract. Build the local path on the current in-process dispatch options and cover read/write/admin/approval/dynamic scopes.
  - confidence: 0.9
- **[P1] Preserve timeout and token override semantics:** `src/agents/tools/gateway.ts:155-158`
  - body: The guard only skips local dispatch for `gatewayUrl` and `expectFinal`, so calls with `opts.timeoutMs` or an explicit `gatewayToken` no longer go through the normal gateway option/auth path. That can make embedded calls hang beyond their configured timeout or succeed/fail under ambient local credentials instead of the caller's requested token semantics. Either skip local dispatch for explicit auth/timeout overrides or pass equivalent behavior into the in-process dispatcher.
  - confidence: 0.88
- **[P2] Normalize params before local dispatch:** `src/agents/tools/gateway.ts:156-158`
  - body: The WebSocket path sends a JSON frame, which drops object properties whose value is `undefined`; this local path forwards the original object directly. Validators that accept omitted optional fields can reject explicitly `undefined` fields, so embedded calls can fail differently from the same call over WebSocket. Normalize through the same JSON-compatible request-frame shape before dispatching locally.
  - confidence: 0.82
