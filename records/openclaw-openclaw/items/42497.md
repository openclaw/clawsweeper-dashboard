## Summary

Keep open: the underlying gateway self-contention bug still appears valid and this PR is the canonical open implementation path, but the current branch is stale against main and introduces concrete gateway authorization, timeout, startup-registration, and frame-contract regressions. It also still lacks after-fix real behavior proof.

## What This Changes

The PR adds a gateway-local dispatch registry and makes agent gateway-tool calls prefer in-process dispatch over WebSocket when a dispatcher is registered.

## Best Possible Solution

Rebase the fix onto current main as an adapter over the existing in-process gateway dispatch contract, preserving startup registration, credentials, scopes, timeout, expectFinal, and JSON-frame semantics before proving the real embedded cron/gateway-tool path.

## Reproduction Assessment

Yes for the PR regressions: source inspection shows the new local branch runs before current option, scope, timeout, and frame-normalization handling. The original active-session cron timeout is supported by linked issue logs, but I did not live-run it in this read-only review.

## Solution Assessment

No. In-process dispatch is a plausible fix direction, but this implementation should be rebased onto the current dispatch contract and preserve existing gateway authorization and transport semantics before merge.
