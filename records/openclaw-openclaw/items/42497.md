## Summary

Keep this PR open, but do not merge it as-is: the linked gateway self-contention bug is real and the in-process dispatch direction is plausible, but the branch currently regresses gateway startup, authorization, timeout, and frame-normalization contracts against current main.

## What This Changes

This PR adds a gateway-local dispatch registry, registers it from gateway fallback setup, and makes embedded agent gateway tool calls prefer in-process dispatch over WebSocket.

## Best Possible Solution

Rework the local dispatch path as an adapter over the current in-process gateway dispatch contract, preserving resolver startup, credentials, caller scopes, timeout, expectFinal, and JSON-frame semantics, then prove a real embedded cron or gateway-tool run no longer times out.

## Reproduction Assessment

Yes for the PR regressions: source inspection shows the new local branch runs before current option, scope, timeout, and frame-normalization handling. The original active-session cron timeout is supported by linked issue logs but was not live-run in this read-only review.

## Solution Assessment

No. In-process dispatch is a plausible fix direction, but this implementation should be rebased onto the current dispatch contract and preserve existing gateway authorization and transport semantics before merge.

  - body: Current gateway startup installs fallback context with setFallbackGatewayContextResolver, but this PR registers the local dispatcher only from setFallbackGatewayContext. Embedded calls that rely on approval, pairing, admin, or dynamic plugin scopes can execute under fallback or synthetic credentials instead of the caller-specific authorization contract.
  - confidence: 0.9
- **[P1] Honor token and timeout options on the local path:** `src/agents/tools/gateway.ts:154-160`
  - body: Calls with gatewayToken or timeoutMs but no gatewayUrl still take the local path, so explicit credential intent is ignored and stuck in-process handlers lose the bounded timeout behavior provided by the WebSocket path.
  - confidence: 0.88
- **[P2] Normalize params through the gateway frame contract:** `src/agents/tools/gateway.ts:156-158`
  - body: The local path forwards raw params directly instead of the JSON request frame used by WebSocket. Optional fields set to undefined can reach validators as present invalid values even though the same call over WebSocket would omit them.
  - confidence: 0.83
