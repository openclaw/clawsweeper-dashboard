## Summary

Keep open: the target embedded gateway-tool timeout is still a plausible real bug, but this PR is not merge-ready because it conflicts with current main, misses real behavior proof, and bypasses the current gateway-tool auth/scope/timeout contract.

## What This Changes

The PR adds a process-global local dispatch registry, registers it from fallback context setup, and makes `callGatewayTool` use it for embedded calls when no explicit `gatewayUrl` or `expectFinal` is set.

## Best Possible Solution

Rebase onto current main and implement a scoped in-process gateway-tool path that preserves gateway options, tokens, scopes, approval runtime token, timeout, `expectFinal`, and request-frame semantics, then prove embedded cron no longer times out.

## Reproduction Assessment

Yes for the PR regressions by source inspection: the added local branch runs before current auth/scope/timeout handling and registers on an older startup path. I did not live-run the original embedded cron timeout in this read-only review.

## Solution Assessment

No. In-process dispatch is a plausible fix direction, but this implementation should be rebuilt on the current scoped gateway dispatch contract rather than adding a process-wide shortcut.
