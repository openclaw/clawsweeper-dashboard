## Summary

Keep open. Current main does not implement this in-process dispatch path, but this PR should not merge as-is because the local branch bypasses the existing gateway auth, scope, timeout, and JSON-frame contracts.

## What This Changes

Adds a gateway local-dispatch registry, registers `dispatchGatewayMethod` during gateway startup, and makes `callGatewayTool` prefer it for non-explicit-URL and non-`expectFinal` calls with focused tests.

## Best Possible Solution

Rework this as an explicit in-process gateway request adapter that preserves option resolution, effective credentials/scopes, bounded timeout behavior, JSON-frame parameter normalization, and final-response semantics, with regression coverage for cron, node, token, timeout, scope, and `undefined` parameter cases.

## Reproduction Assessment

Yes. A focused test can register the PR’s local dispatcher and call `callGatewayTool` with `gatewayToken`, `timeoutMs`, explicit `scopes`, or optional `undefined` params to show the local path returns before the current WebSocket contract is applied.

## Solution Assessment

No. In-process dispatch is a plausible direction for the self-contention bug, but this implementation is not the best fix because it early-returns around the established gateway transport and authorization contract instead of adapting the local path to match it.

## Security Review

Status: needs_attention

Summary: The diff introduces a new in-process gateway control-plane path without preserving the existing authorization, credential, and timeout contract.

Concerns:

- **[high] Local dispatch bypasses caller authorization intent:** `src/agents/tools/gateway.ts:152`
  - body: The early local return can execute a gateway method before resolving explicit tokens, least-privilege scopes, or explicit scope overrides, so embedded calls may run under ambient or synthetic credentials rather than the caller’s intended authorization contract.
  - confidence: 0.88
- **[medium] Local dispatch skips bounded request timeout:** `src/agents/tools/gateway.ts:152`
  - body: Because `timeoutMs` is not applied on the local path, a stuck in-process handler can block an active agent turn instead of failing under the timeout behavior used by the WebSocket request path.
  - confidence: 0.84
