## What This Changes

The PR adds a local gateway dispatch registry, registers it during fallback context setup, and makes agent gateway tool calls prefer in-process dispatch for embedded calls.

## Best Possible Solution

Rebase onto current main and implement a scoped in-process gateway-tool path that preserves gateway options, scopes, approval runtime token, timeout, `expectFinal`, and request-frame semantics, then prove embedded cron no longer times out.

## Reproduction Assessment

Yes for the PR regressions by source inspection: the added local branch runs before current auth/scope/timeout handling and registers on an older startup path. I did not live-run the original embedded cron timeout in this read-only review.

## Solution Assessment

No. In-process dispatch is a plausible fix direction, but this implementation should be rebuilt on the current scoped gateway dispatch contract rather than adding a process-wide shortcut.
