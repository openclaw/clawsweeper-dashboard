## Summary

Keep open: the embedded gateway-tool self-contention bug is still not implemented on current main, and this PR remains the canonical proposed fix, but the branch is stale against current gateway dispatch/auth contracts and is not safe to merge as-is.

## What This Changes

The PR adds a process-wide local gateway dispatch registry, registers it from gateway fallback context setup, and makes agent gateway tool calls prefer that in-process path with focused tests.

## Best Possible Solution

Rebase onto current main and implement a scoped, per-server in-process gateway-tool path that preserves gateway options, scopes, approval runtime token, timeout, expectFinal, and JSON-frame semantics, then prove embedded cron no longer times out.

## Reproduction Assessment

Yes for the PR regressions by source inspection: the added local branch runs before current auth/scope/timeout handling and registers on an older startup path. I did not live-run the original embedded cron timeout in this read-only review.

## Solution Assessment

No. In-process dispatch is still a plausible fix direction, but this implementation must be rebuilt on the current scoped gateway dispatch contract rather than adding a process-wide shortcut.

  - body: `_localDispatch` is a single module-global dispatcher with no per-server context.   - confidence: 0.91

  - body: The local branch returns before current `resolveGatewayOptions`, explicit/derived scopes, and approval-runtime token handling. That can execute auth-sensitive methods under a synthetic or ambient client instead of the same authorization contract as the WebSocket path.
  - confidence: 0.92

  - confidence: 0.88

  - body: The local path forwards raw params directly instead of the JSON request frame used by WebSocket. Optional properties set to `undefined` can reach validators as present invalid values even though the same call over WebSocket would omit them.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The diff introduces a local gateway control-plane path before preserving current scoped authorization, credential, context, and timeout semantics.

Concerns:

- **[high] Local dispatch bypasses caller authorization intent:** `src/agents/tools/gateway.ts:154`
  - body: The early local return can execute gateway methods before resolving explicit tokens, approval runtime identity, explicit scope overrides, or least-privilege scopes, so embedded calls may run under synthetic or ambient authority rather than the caller’s intended authorization contract.
  - confidence: 0.9
- **[high] Process-global dispatch can cross gateway contexts:** `src/gateway/local-dispatch.ts:18`
  - body: A singleton local dispatcher is not bound to the accepting gateway instance, which conflicts with current main's recent hardening around multi-gateway in-process dispatch context.
  - confidence: 0.84
- **[medium] Local dispatch skips bounded request timeout:** `src/agents/tools/gateway.ts:154`
  - body: Because `timeoutMs` is not applied on the local path, a stuck in-process handler can block an active agent turn instead of failing under the timeout behavior used by the WebSocket request path.
