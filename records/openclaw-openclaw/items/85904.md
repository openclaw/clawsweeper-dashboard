## Summary

## What This Changes

The PR adds routed thread ids to active reply operations, blocks active steering when the current turn targets a different known route thread, and expands Slack/auto-reply regression tests.

## Best Possible Solution

Land the routed-thread metadata guard after maintainers accept the generic rule that known different routeThreadId values should not steer into the active run, while keeping Slack's stable DM SessionKey contract and the new regression tests.

## Reproduction Assessment

Yes, at source level. Current main keeps Slack DMs on a stable session while carrying routed thread metadata, and active steering only checks the session lane; the PR body also reports the before-fix assertion failure for the same-session/different-thread case.

## Solution Assessment

Yes, with maintainer acceptance of the generic scope. The PR fixes the active-run routing boundary instead of changing Slack session-key generation, which preserves the shipped stable DM SessionKey contract.
