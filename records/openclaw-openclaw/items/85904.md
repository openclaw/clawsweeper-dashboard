## Summary

Keep open: this PR addresses a source-verifiable Slack DM active-run routing bug and has sufficient live Slack proof, but merge still needs maintainer acceptance of the Slack DM compatibility/session-state change.

## What This Changes

The PR records routed Slack thread IDs on reply operations and adds a Slack-scoped guard so same-session Slack DM turns with different Slack thread IDs queue/follow up instead of steering into the active run.

## Best Possible Solution

Land the Slack-scoped routed-thread guard once maintainers accept the compatibility change, preserving stable Slack DM SessionKey behavior and non-Slack same-session steering.

## Reproduction Assessment

Yes, at source level. Current main carries Slack routed thread metadata but shouldSteer only checks room/heartbeat/reset/queue mode, so two Slack DM threads under the same stable SessionKey can target the active run.

## Solution Assessment

Yes. Carrying routeThreadId into ReplyOperation and applying the comparison only to Slack direct routed-thread turns is a narrow fix that preserves the Slack DM session contract without changing non-Slack same-session steering.
