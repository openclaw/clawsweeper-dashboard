## Summary

Keep open for maintainer review: current main still makes active steering decisions from the stable session only, while this PR adds a Slack-scoped routed-thread guard with focused regression coverage and live Slack proof. I found no blocking patch defect, but the merge still changes Slack DM active-run behavior enough that maintainers should explicitly accept the compatibility and session-state tradeoff.

## What This Changes

The PR records routed thread ids on reply operations, prevents Slack direct-message turns from steering across different Slack threads, and adds Slack plus non-Slack steering regression tests.

## Best Possible Solution

Land the Slack-scoped routed-thread guard after maintainers accept the Slack DM steering compatibility change, keeping non-Slack steering unchanged and retaining the routeThreadId regression coverage.

## Reproduction Assessment

Yes, at source level. Current main preserves Slack DM route-thread metadata but computes active steering without comparing routed thread ids, and the PR body reports the before-fix unit failure for the same-session/different-thread case.

## Solution Assessment

Yes. Recording routeThreadId on reply operations and applying the guard only to Slack direct routed-thread turns is a narrow fix that preserves the stable Slack DM SessionKey contract and avoids the earlier generic non-Slack behavior change.
