## Summary

Keep open for maintainer review. I found no blocking correctness or security issue, and the supplied live Slack proof is sufficient, but the generic active-steering rule can affect other threaded channels that share session keys.

## What This Changes

The PR records routed thread IDs on reply operations, suppresses active steering when the current turn has a different routed thread ID, and adds Slack and auto-reply regression coverage.

## Best Possible Solution

Land the routed-thread metadata guard after maintainers accept the generic rule, or narrow it to Slack/threaded channel families if that generic contract is too broad.

## Reproduction Assessment

Yes, at source level. Current main keeps Slack DMs on a stable direct SessionKey while active steering is session-key based; the PR also reports a before-fix assertion failure for the same-session, different-thread case.

## Solution Assessment

Yes, if maintainers accept the generic routeThreadId rule. The fix preserves Slack's stable DM SessionKey contract and moves the separation check to the active-run routing boundary; the safer alternative is a Slack-scoped guard.
