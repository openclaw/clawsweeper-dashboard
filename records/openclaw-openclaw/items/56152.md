## Summary

Keep open: the macOS symlink-root bug still appears relevant and current main does not implement this PR's fix, but the branch is not merge-ready because it weakens the localRoots security boundary, changes the workspace sibling guard, conflicts with current main, and lacks inspectable Telegram/macOS proof.

## What This Changes

This PR changes local media root validation to compare lexical and realpath variants for media paths and allowed roots, and adds mocked macOS /tmp to /private/tmp regression tests.

## Best Possible Solution

Revise the validator to preserve canonical containment for existing media files, use lexical fallback only when realpath fails, compare allowed-root aliases safely, and cover both alias success and symlink-escape/workspace-sibling rejection in tests.

## Reproduction Assessment

Yes for the PR-introduced security bug: source inspection shows a symlink under an allowed root can pass via the lexical variant even when its realpath points outside. The original macOS Telegram false-negative is modeled by PR tests but still lacks inspectable live proof.

## Solution Assessment

No. The safer fix is to keep canonical containment for existing media paths and only fall back to lexical paths when realpath fails, while still handling root aliases such as /tmp and /private/tmp.
