## Summary

Keep open. The PR targets a plausible local-media false negative, but the current diff weakens the `localRoots` filesystem boundary by accepting lexical media-path variants after successful canonicalization and by moving the workspace-sibling guard off the canonical path.

## What This Changes

The PR changes `assertLocalMediaAllowed()` to compare lexical and realpath variants for media paths and roots, and adds Vitest coverage for macOS `/tmp` and `/private/tmp` alias cases.

## Best Possible Solution

Revise the PR so media paths use the canonical path whenever `fs.realpath(mediaPath)` succeeds, allow only safe configured-root alias handling for `/tmp` and `/private/tmp`, keep workspace-sibling checks on canonical paths, and add regression coverage for both alias acceptance and symlink escapes.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main can reject mismatched lexical/canonical `/tmp` aliases, and the PR regression follows from matching lexical media variants after `fs.realpath(mediaPath)` succeeds.

## Solution Assessment

No. The PR fixes the right class of false negative, but accepting lexical media variants after canonicalization is not the narrowest maintainable solution because it weakens containment.

## Review Findings

Overall correctness: patch is incorrect

Full review comments:

- **[P1] Preserve canonical-path enforcement for media symlinks:** `src/media/local-media-access.ts:89-97`
  - body: The new cross-product check accepts the lexical `path.resolve(mediaPath)` variant even when `fs.realpath(mediaPath)` resolves outside the allowed root. A symlink under an allowed root can therefore pass via the lexical variant, weakening the existing `localRoots` boundary.
  - confidence: 0.93
- **[P2] Keep workspace sibling checks on canonical paths:** `src/media/local-media-access.ts:60-62`
  - body: `resolved` now comes from the first variant, which is always the lexical path, so the default `workspace-*` sibling restriction no longer checks the canonical path when the media file exists. A symlinked path can miss the guard that previously ran against `fs.realpath(mediaPath)`.
  - confidence: 0.86
