## What This Changes

The PR changes local media root validation to compare lexical and canonical path variants for media paths and roots, and adds tests for macOS `/tmp` and `/private/tmp` aliases.

## Best Possible Solution

Revise `assertLocalMediaAllowed()` so existing files remain contained by canonical target, macOS root aliases are tolerated safely, workspace sibling checks stay canonical, and coverage includes out-of-root symlink rejection plus redacted Telegram proof.

## Reproduction Assessment

Yes, for the PR-introduced blocker at source level: a symlink inside an allowed root can match via `path.resolve(mediaPath)` while `fs.realpath(mediaPath)` points outside. The original Telegram false negative still needs attached real behavior proof.

## Solution Assessment

No. The PR addresses the right area, but the current cross-product approach is not the best fix because it trades away canonical containment instead of only tolerating safe root aliases.
