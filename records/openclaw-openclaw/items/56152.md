## Summary

Keep open: the PR addresses a valid macOS path-alias false negative, but the current diff weakens canonical local-media containment and still lacks contributor-owned real Telegram proof, so it is not safe to merge or close.

## What This Changes

The PR changes local media path validation to compare lexical and real path variants for media paths and allowed roots, plus adds targeted macOS `/tmp` and `/private/tmp` regression tests.

## Best Possible Solution

Revise `assertLocalMediaAllowed()` so existing files remain contained by canonical target, macOS root aliases are tolerated without symlink escapes, workspace sibling checks stay canonical, and regression coverage includes out-of-root symlink rejection plus real Telegram proof.

## Reproduction Assessment

Yes, at source level: current main can reject the mocked `/tmp` and `/private/tmp` alias case, and the PR's symlink escape follows from accepting lexical media variants after `fs.realpath(mediaPath)` succeeds.

## Solution Assessment

No. The PR targets the right false negative, but the maintainable fix must preserve canonical containment for existing files while adding safe alias tolerance and explicit escape coverage.
