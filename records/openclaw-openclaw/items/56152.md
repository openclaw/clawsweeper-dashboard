## Summary

Keep open: the PR targets a real local media false negative, but the proposed cross-product variant check weakens the local-root containment boundary and still lacks attached after-fix real Telegram proof.

## What This Changes

The PR changes local media root validation to accept symlinked `/tmp` and `/private/tmp` path variants and adds focused regression tests.

## Best Possible Solution

Revise `assertLocalMediaAllowed()` so existing files remain contained by canonical target, macOS root aliases are tolerated safely, workspace sibling checks stay canonical, and regression coverage includes out-of-root symlink rejection plus redacted Telegram proof.

## Reproduction Assessment

Yes, at source level. Current main can reject the `/tmp` versus `/private/tmp` alias case, and the PR's symlink escape follows from accepting lexical media variants after `fs.realpath(mediaPath)` succeeds.

## Solution Assessment

No. The PR targets the right false negative, but the maintainable fix must preserve canonical containment for existing files while adding safe alias tolerance and explicit escape coverage.
