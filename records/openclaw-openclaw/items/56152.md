## Summary

Keep open. Current main still lacks the PR's `/tmp` and `/private/tmp` alias handling, but the branch weakens the local media root containment check, so it needs revision before merge rather than cleanup closure.

## What This Changes

The PR adds lexical-plus-realpath variant matching to `assertLocalMediaAllowed()` and adds Vitest coverage for macOS `/tmp` to `/private/tmp` local media root combinations.

## Best Possible Solution

Revise the branch so the media path remains canonical whenever `fs.realpath(mediaPath)` succeeds, uses the lexical media path only when realpath fails, permits configured roots to match necessary lexical/canonical aliases, and adds regression coverage for the alias success case plus symlink escape and workspace-sibling cases.

## Reproduction Assessment

Yes. The PR finding is reproducible from the diff with a mocked or real symlink where `path.resolve(mediaPath)` is under an allowed root but `fs.realpath(mediaPath)` resolves outside it; the new cross-product check returns allowed in that case.

## Solution Assessment

No. Variant-aware root matching is the right direction for the macOS alias false negative, but accepting lexical media-path variants after a successful realpath is not the narrowest maintainable fix for a filesystem containment boundary.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve canonical containment when realpath succeeds:** `src/media/local-media-access.ts:89-97`
  - body: The new cross-product allows the lexical media path to satisfy an allowed root even when `fs.realpath(mediaPath)` resolves outside that root through a symlink. That weakens the localRoots boundary and can permit reads outside the configured media roots; only use the lexical media path when realpath fails.
  - confidence: 0.92
- **[P2] Check workspace siblings against canonical variants:** `src/media/local-media-access.ts:64-79`
  - body: `resolved` now comes from the lexical variant, so the default `workspace-*` sibling guard no longer sees a symlink target that canonicalizes into a sibling workspace. Preserve the existing canonical check, or fail when any resolved media variant lands in a blocked sibling workspace.
  - confidence: 0.84
