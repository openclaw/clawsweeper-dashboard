## Summary

Keep open: the PR targets a valid macOS `/tmp` versus `/private/tmp` local media bug, but the current diff weakens local media containment and lacks the required real Telegram behavior proof.

## What This Changes

The PR changes local media root validation to compare lexical and realpath variants for media paths and roots, and adds regression tests for macOS `/tmp` and `/private/tmp` alias cases.

## Best Possible Solution

Revise `assertLocalMediaAllowed()` so existing files remain contained by canonical target, macOS root aliases are tolerated safely, workspace sibling checks stay canonical, and regression coverage includes out-of-root symlink rejection plus redacted Telegram proof.

## Reproduction Assessment

Yes, at source level. Current main can reject the `/tmp` versus `/private/tmp` alias case, and the PR’s symlink escape follows directly from accepting lexical media variants after `fs.realpath(mediaPath)` succeeds.

## Solution Assessment

No. The PR targets the right false negative, but the maintainable fix must preserve canonical containment for existing files while adding safe root-alias tolerance and explicit escape coverage.
