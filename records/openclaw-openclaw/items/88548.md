## Summary

Keep open: current main still combines a refreshable bundled Copilot catalog with live discovery, so static IDs and metadata can shadow per-account results; the linked closing PR remains open and needs repair, upgrade-contract review, and live proof before it can resolve the issue.

## What This Changes

Make GitHub Copilot’s live per-account model catalog authoritative for model visibility and metadata instead of allowing bundled defaults to shadow it.

## Best Possible Solution

Use the authenticated live Copilot response as the canonical source for entitled model visibility and metadata, preserve explicit user overrides, derive request transport from supported endpoint contracts, and surface a clear discovery failure instead of silently serving stale bundled rows.

## Reproduction Assessment

Yes at source level and through the reporter’s concrete real-account evidence: current main materializes refreshable static rows alongside live discovery, and an entitled live-only model was absent from the visible catalog. This review did not independently authenticate to a Copilot account.

## Solution Assessment

The live-authoritative catalog is the best long-term ownership direction, but the discovery-failure contract and whether broader transport/thinking cleanup belongs in the same PR require explicit maintainer judgment.
