## Summary

Keep open: current main still carries bundled Copilot static/default model metadata, and the same author has an open linked implementation PR with closing syntax for the live-first catalog behavior, so this issue should track that review rather than be closed now.

## What This Changes

The issue asks for GitHub Copilot model listing and picker visibility to use live `/models` discovery as the authoritative source instead of being gated by bundled static defaults.

## Best Possible Solution

Review and land a provider-owned fix that makes Copilot visible catalog metadata come from live `/models` when available, preserves explicit user config overrides, and defines the degraded fallback behavior intentionally.

## Reproduction Assessment

Yes, source inspection plus the issue's live-account steps give a high-confidence reproduction path: use an entitled Copilot account whose `/models` response includes a newer model, then compare direct model routing with `openclaw models list --provider github-copilot`. ## Solution Assessment

Unclear until the linked PR is reviewed: live metadata should win, but maintainers still need to choose the exact fallback policy between live-only and live-plus-bounded-static behavior.
