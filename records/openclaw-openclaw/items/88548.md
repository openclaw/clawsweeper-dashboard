## Best Possible Solution

## Reproduction Assessment

## Solution Assessment

Yes, the best fix belongs in the provider-owned catalog assembly so live Copilot rows win for duplicate IDs and static rows are only fallback. The issue should stay open until the linked fix PR merges or maintainers choose a different policy.
