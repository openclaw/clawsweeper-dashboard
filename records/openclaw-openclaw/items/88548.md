## Summary

Close as implemented on current main: refreshable provider catalogs now force runtime discovery, and both model list and picker paths prefer the live provider catalog before static fallback. Remaining Copilot Claude context/thinking/session-budget metadata symptoms are narrower follow-up work, not a blocker for this visibility/source-of-truth issue.

## What This Changes

GitHub Copilot live `/models` discovery should be authoritative for model picker and `openclaw models list` visibility instead of being shadowed by static default catalog rows.

## Best Possible Solution

Close this tracker as fixed by current-main runtime-first refreshable catalog discovery, while keeping Copilot metadata/session-budget follow-up on the narrower open issue.

## Solution Assessment

Yes for the original picker/list visibility bug: making refreshable catalogs load the runtime provider is the correct owner-boundary fix, and the live Copilot mapper remains responsible for provider metadata.
