## Summary

Keep open: the report is plausible and touches real subagent, exec, and compaction boundaries, but current main already scopes background processes by session key and exposes `process list`, so I did not find enough evidence to close it as implemented or not reproducible.

## What This Changes

The issue asks for running `exec` process references to survive subagent compaction, or for a reliable reattach path after compaction.

## Best Possible Solution

Add a deterministic runtime-owned active-process handoff after compaction, likely a bounded scoped process-session summary or equivalent state bridge, plus regression coverage that a subagent can poll/log/write a background exec started before compaction.

## Reproduction Assessment

Unclear. Source inspection shows the relevant runtime paths and the `No session found` failure branch, but I did not live-run a subagent compaction on current main to prove the reported failure still occurs.

## Solution Assessment

Unclear. Relying on compaction summaries to retain process session ids is fragile, but maintainers should choose between documenting `process list` as the recovery path and injecting active process references as runtime-owned post-compaction context.
