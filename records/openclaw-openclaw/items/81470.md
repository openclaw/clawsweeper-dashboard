## What This Changes

The PR changes WebChat final broadcast assembly to keep delivered final payloads and adds an idempotency key to a transcript append in `src/gateway/server-methods/chat.ts`.

## Best Possible Solution

Move the fix to the agent-run final broadcast path so WebChat receives the same audio content already appended to the transcript, with regression coverage for final chat payload media and duplicate-transcript behavior.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection shows the media supplement only happens after `agentRunStarted`, while the proposed edit is in the non-agent branch and the current agent final broadcast remains text-only.

## Solution Assessment

No. The PR targets the named payload-clearing code, but that branch is not reached for the reported agent-run TTS path; the safer fix is to bridge the appended media into the agent-run final broadcast and handle transcript idempotency explicitly.
