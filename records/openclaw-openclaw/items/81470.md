## Summary

Keep open: the linked WebChat TTS bug appears plausible from source, but this PR edits the non-agent final assembly path and does not reach the reported agent-run media path, so it should not merge as-is.

## What This Changes

The PR changes WebChat final broadcast handling to keep delivered final payloads when media was already appended and adds an idempotency key to the follow-up transcript append.

## Best Possible Solution

Move the fix to the agent-run final broadcast path so WebChat receives the same audio content that was already appended to the transcript, with regression coverage for the final chat payload and duplicate-transcript behavior.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection shows the current agent-run media path persists an audio transcript update while the final chat event is built from buffered text only, but this review did not execute WebChat/TTS end to end.

## Solution Assessment

No. The PR edits the non-agent final assembly branch, while the reported media append condition requires `agentRunStarted`; the safer fix is to bridge appended agent media into the agent-run final broadcast path.
