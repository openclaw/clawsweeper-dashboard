## Summary

## What This Changes

The PR updates `src/agents/cli-output.ts` and parser tests so CLI JSONL output prefers `task_complete.last_agent_message`, then `final_answer`, and skips explicit `commentary` item text.

## Best Possible Solution

Land the focused parser-boundary fix after maintainer review accepts the explicit phase semantics, while keeping the broader channel-leak work at https://github.com/openclaw/openclaw/issues/25592 and the separate Codex final-message artifact work at https://github.com/openclaw/openclaw/pull/66081 on their own tracks.

## Reproduction Assessment

Yes, source-level. Current main's buffered and streaming JSONL parsers collect generic `item.text` without terminal-field priority, and the PR fixtures demonstrate the wrong commentary or generic selection that the patch corrects.

## Solution Assessment

Yes. The patch fixes the narrow parser boundary in both buffered and streaming paths, preserves Claude-specific handling, and avoids expanding into channel dispatch or Codex artifact policy.
