## Summary

This PR should stay open: current main still lacks terminal-field precedence, and the patch is a scoped parser fix with real CLI proof, but merging needs maintainer acceptance of the commentary-only JSONL compatibility change and normal check refresh.

## What This Changes

The PR updates `src/agents/cli-output.ts` and parser tests so full-buffer and streaming JSONL parsing prefer `task_complete.last_agent_message` and `final_answer` text over commentary or generic item text.

## Best Possible Solution

Land this parser fix after maintainers accept terminal-field and `final_answer` precedence for phase-aware JSONL, while keeping the broader channel-leak policy on https://github.com/openclaw/openclaw/issues/25592 separate.

## Reproduction Assessment

Yes, source-reproducible. The linked report supplies a concrete JSONL shape, and current main still ignores `task_complete.last_agent_message` and `phase:"final_answer"` while collecting generic `item.text`.

## Solution Assessment

Yes for the narrow parser bug, with a compatibility caveat. The shared CLI JSONL parser is the right boundary because both streamed and completed CLI runner paths consume it before final text is delivered.
