## Summary

Keep open: the parser fix is useful but incomplete against current main because JSONL streaming output can bypass the updated full-buffer parser, and the external PR still lacks real Codex/OpenAI CLI JSONL proof.

## What This Changes

The branch adds terminal-field extraction and parser tests so `parseCliJsonl()` prefers `task_complete.last_agent_message` and `final_answer` over commentary, plus a changelog entry.

## Best Possible Solution

Share the terminal-field priority logic between full-buffer and streaming JSONL parsing, add streaming or runner-path regression coverage, rebase the branch, and land after real Codex/OpenAI CLI JSONL proof is attached.

## Reproduction Assessment

Yes, source-level: current main's JSONL runner prefers `createCliJsonlStreamingParser().getOutput()` before `parseCliOutput()`, and the streaming parser can still return generic `item.text` without terminal-field priority. I did not run a live provider flow in this read-only review.

## Solution Assessment

No, not as-is. The direction is right, but the fix should apply the same terminal-field selection to the streaming parser or runner path so `task_complete` and `final_answer` cannot be bypassed.
