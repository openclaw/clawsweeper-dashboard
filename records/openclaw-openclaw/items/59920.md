## Summary

Keep this PR open: it is the narrow parser-layer fix for the linked CLI JSONL terminal-answer bug, but the new terminal-field preference currently keeps the first terminal record instead of the latest one, which can surface stale final text when later terminal records correct it.

## What This Changes

The PR changes `src/agents/cli-output.ts` and parser tests so CLI JSONL output prefers `task_complete.last_agent_message`, then `phase:"final_answer"`, while skipping explicit commentary-phase fallback text.

## Best Possible Solution

Keep the parser-layer fix, but first make terminal-field selection last-wins in both buffered and streaming JSONL parsers, then let maintainers explicitly accept the commentary-suppression compatibility tradeoff.

## Reproduction Assessment

Yes. Current main is source-reproducible because `parseCliJsonl()` ignores `task_complete.last_agent_message` and `phase:"final_answer"` for generic non-Claude JSONL while the linked fixture contains those terminal records.

## Solution Assessment

No, not yet. The parser layer is the right owner for this bug, but terminal records should be last-wins to match Codex SDK turn-result behavior before the PR is merge-ready.
