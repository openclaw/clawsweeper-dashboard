## Summary

Keep open: the PR is a focused bug fix, but it only updates the full-buffer JSONL parser while the runtime can return streaming JSONL output first, and the PR still lacks real Codex/OpenAI CLI behavior proof.

## What This Changes

The branch teaches `parseCliJsonl()` to prefer `task_complete.last_agent_message` and `final_answer` over commentary, adds parser tests, and adds a changelog entry.

## Best Possible Solution

Share the terminal-field priority logic between full-buffer and streaming JSONL parsing, add streaming/runner-path regression coverage, then land after real Codex/OpenAI CLI JSONL proof is attached.

## Reproduction Assessment

Yes, source-level. Current main uses `createCliJsonlStreamingParser().getOutput()` before `parseCliOutput()` for JSONL backends, and that streaming parser can still return generic `item.text` without terminal-field priority.

## Solution Assessment

No, not as-is. The direction is right, but the maintainable fix should make terminal-field selection common to both JSONL parsing paths or ensure the runner uses the terminal-aware result after process completion.
