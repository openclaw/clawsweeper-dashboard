## Summary

Keep this PR open for maintainer review: current main and the latest release still lack the terminal-field precedence reported in the linked bug, and this remains the canonical narrow implementation candidate. The patch looks correct for the Codex/OpenAI-style parser path, but it intentionally changes generic JSONL commentary-only fallback behavior, so the remaining decision is compatibility acceptance rather than cleanup closure.

## What This Changes

The PR updates the full-buffer and streaming CLI JSONL parsers to prefer `task_complete.last_agent_message`, then `final_answer` content, before generic item text, with focused parser tests.

## Best Possible Solution

Land this parser fix after maintainers accept terminal-field and `final_answer` precedence, while keeping the broader inter-tool channel-leak policy on https://github.com/openclaw/openclaw/issues/25592 separate.

## Reproduction Assessment

Yes. The linked report provides a sanitized JSONL shape, and current main plus the latest release show the parser ignores `task_complete.last_agent_message` and `phase:"final_answer"` while collecting generic item text.

## Solution Assessment

Yes for the narrow parser bug, with one compatibility caveat. The shared CLI JSONL parser is the right boundary because both streaming and completed CLI runner paths consume it before final text is delivered; if maintainers do not accept generic commentary-only empty output, the safer alternative is to gate that behavior to known phase-aware dialects.
