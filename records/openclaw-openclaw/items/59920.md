## What This Changes

The PR updates the shared CLI JSONL parser and parser tests so terminal reply fields win over commentary-like fallback text in both full-buffer and streaming parser paths.

## Best Possible Solution

Land the focused parser fix after a maintainer explicitly accepts the commentary-only JSONL compatibility tradeoff, keeping generic non-commentary fallback behavior intact.

## Reproduction Assessment

Yes. Current main source still uses the generic non-Claude item.text fallback, and the linked bug plus PR fixtures provide a concrete JSONL shape where commentary can beat terminal final-answer semantics.

## Solution Assessment

Yes, with a compatibility caveat. The shared CLI JSONL parser is the narrowest maintainable layer because both streaming and full-buffer CLI runner paths consume it, while channel dispatch and provider failover do not need to change for this parser-specific bug.
