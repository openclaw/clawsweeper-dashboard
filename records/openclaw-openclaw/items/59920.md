## Summary

Keep open. Current main still has the reported CLI JSONL terminal-field gap, and this PR only fixes the full-buffer parser while current main can return `createCliJsonlStreamingParser().getOutput()` before the new `parseCliJsonl()` priority rules run; the PR also still lacks after-fix real behavior proof.

## What This Changes

This PR adds terminal JSONL field extraction to `parseCliJsonl()`, parser tests for Codex/OpenAI `task_complete` and `final_answer` precedence over commentary, and a changelog entry.

## Best Possible Solution

Share terminal-field priority and commentary suppression between the full-buffer and streaming JSONL parsers, keep Claude handling intact, add a streaming parser or runner-path regression, and land only after redacted real Codex/OpenAI CLI JSONL proof is attached.

## Reproduction Assessment

Yes, source-level. Current main still accumulates message-like `item.text` in both JSONL parser paths, and the runner uses streaming output before `parseCliOutput()`, so a stream with commentary `item.text` plus later terminal fields can still return commentary; I did not run tests in this read-only review.

## Solution Assessment

No, not as-is. The full-buffer parser direction is right, but the maintainable fix needs the same terminal-field priority in the streaming parser or a runner path that re-parses final stdout before accepting streaming fallback text.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Apply terminal priority to streaming JSONL output:** `src/agents/cli-output.ts:554-555`
  - body: Current main uses `createCliJsonlStreamingParser().getOutput()` before `parseCliOutput()` for JSONL backends. This patch only records `task_complete` and `final_answer` inside `parseCliJsonl()`, so a stream that first accumulates commentary `item.text` can return that streaming output and never reach the new terminal-field priority. Please share this extraction with the streaming parser or make the runner re-parse final stdout, with a regression for that path.
