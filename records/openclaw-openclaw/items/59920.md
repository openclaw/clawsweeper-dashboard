## Summary

Keep open: current main and the latest release still have the phase-blind CLI JSONL fallback, and this PR remains the canonical open fix for the narrow terminal-field parser gap. The patch looks correct, but merge needs maintainer acceptance of the generic JSONL commentary-only compatibility change and the currently failing checks need normal pre-merge handling.

## What This Changes

The PR updates the CLI JSONL full-buffer and streaming parsers to prefer `task_complete.last_agent_message`, then `final_answer` content, before generic non-commentary item text.

## Best Possible Solution

Land this parser fix after maintainers accept terminal-field and `final_answer` precedence, while keeping the broader inter-tool channel-leak policy on https://github.com/openclaw/openclaw/issues/25592 separate.

## Reproduction Assessment

Yes. The linked report supplies a concrete JSONL shape, and current main plus the latest release show the parser ignores `task_complete.last_agent_message` and `phase:"final_answer"` while collecting generic `item.text`.

## Solution Assessment

Yes for the narrow parser bug, with one compatibility caveat. The shared CLI JSONL parser is the right boundary because both streaming and completed CLI runner paths consume it before final text is delivered; if maintainers do not accept generic commentary-only empty output, the safer alternative is to gate that behavior to known phase-aware dialects.
