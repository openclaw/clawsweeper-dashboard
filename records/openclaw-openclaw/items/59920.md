## Summary

Keep open: current main still lacks the terminal-field selection this PR adds to the CLI JSONL parser. The patch is focused, includes regression coverage and a changelog entry, and the adjacent phase-aware fixes in related PRs do not cover this parser path.

## What This Changes

The PR updates `parseCliJsonl()` to prefer `task_complete.last_agent_message` and `final_answer` payloads over commentary text, adds parser regression tests, and adds a changelog entry.

## Best Possible Solution

Land the narrow parser fix after maintainer review, preserving Claude JSONL behavior while making Codex/OpenAI-style terminal reply fields authoritative and keeping the regression tests colocated with the parser.

## Reproduction Assessment

Yes. A synthetic JSONL stream with commentary, `phase: "final_answer"`, and `task_complete.last_agent_message` can be fed directly to `parseCliJsonl()`; current main lacks the extraction path and the PR adds those fixtures.

## Solution Assessment

Yes. The patch solves the loss of terminal semantics at the parser boundary without changing channel dispatch or provider failover, which is the narrowest maintainable fix for the reported path.
