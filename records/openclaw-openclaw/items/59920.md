## Summary

Keep open: the bug is valid and the branch is useful, but it only updates the full-buffer JSONL parser while current main can return streaming-parser output first, so the reported path can still bypass the new terminal-field priority. The external PR also still lacks real behavior proof from a Codex/OpenAI CLI JSONL run.

## What This Changes

The PR teaches the CLI JSONL parser to prefer `task_complete.last_agent_message` and `final_answer` over commentary, adds parser tests, and adds a changelog entry.

## Best Possible Solution

Share terminal-field priority across full-buffer and streaming JSONL parsing, add streaming or runner-path regression coverage, then land after redacted real Codex/OpenAI CLI JSONL output is attached.

## Reproduction Assessment

Yes, source-level. Current main pushes JSONL stdout into `createCliJsonlStreamingParser()`, reads `getOutput()` before `parseCliOutput()`, and that streaming parser can still collect commentary `item.text` without terminal-field priority.

## Solution Assessment

No, not as-is. The direction is right, but the maintainable fix should use one terminal-aware JSONL extraction path for both parser modes or otherwise ensure runner completion uses the terminal-aware result.
