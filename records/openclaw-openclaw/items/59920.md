## Summary

## What This Changes

This PR updates `src/agents/cli-output.ts` and `src/agents/cli-output.test.ts` so CLI JSONL parsing prefers `task_complete.last_agent_message`, then `phase:"final_answer"`, and suppresses commentary-phase fallback text.

## Best Possible Solution

Merge the focused parser fix once maintainers accept that explicit commentary-phase JSONL should not be treated as the user-visible final reply.
