## Summary

Keep open: current main still lacks terminal-field and phase-aware extraction for generic JSONL CLI output, and this PR is the narrow canonical parser fix. ## What This Changes

The branch updates `src/agents/cli-output.ts` and parser tests so full-buffer and streaming JSONL parsing prefer `task_complete.last_agent_message` and `final_answer` records over commentary or generic item text.

## Best Possible Solution

Land this narrow parser fix after maintainers explicitly accept the commentary-only compatibility tradeoff, keeping the terminal-field precedence in both parser paths and leaving broader channel-leak work on the existing canonical issue.

## Reproduction Assessment

Yes. Source inspection on current main shows the JSONL parser still takes generic `item.text` without terminal-field or phase preference, while the linked report and PR fixtures provide a concrete JSONL shape with `commentary`, `final_answer`, and `task_complete.last_agent_message`.

## Solution Assessment

Yes, this is the right layer after the rebase: the parser owns JSONL final-reply semantics, and the PR covers both full-buffer and streaming paths without changing provider failover or channel dispatch. The remaining question is maintainer acceptance of the commentary-only compatibility change.
