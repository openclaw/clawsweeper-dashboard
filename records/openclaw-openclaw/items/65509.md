## Summary

Keep this PR open. Current main still routes the default agent fallback through the home/profile workspace and does not implement the requested `OPENCLAW_STATE_DIR` workspace base behavior, but the submitted patch is not merge-ready because it uses the raw env value and is stale against the current resolver split.

## What This Changes

The PR changes default agent workspace resolution so `OPENCLAW_STATE_DIR` becomes the base directory for `workspace` and profile-suffixed workspace paths when set.

## Best Possible Solution

Either update current main so the default workspace base comes from the normalized state-dir contract, with focused tests and docs, or keep workspace roots as a separate explicit config knob and document that decision consistently.

## Reproduction Assessment

Yes. A source-level reproduction is clear: on current main, default-agent fallback calls `resolveDefaultAgentWorkspaceDir(env)`, which ignores `OPENCLAW_STATE_DIR`, while non-default fallback uses `resolveStateDir(env)`.

## Solution Assessment

No. The product direction may be acceptable after maintainer review, but this patch is not the best implementation because it uses raw env input, lacks matching tests/docs, and no longer targets the current resolver file split.
