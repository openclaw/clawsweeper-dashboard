## Summary

Keep this PR open: current main still has the default-agent workspace gap, but the proposed patch is not merge-ready because it uses unnormalized env input and is dirty against current main.

## What This Changes

The PR changes default agent workspace resolution so `OPENCLAW_STATE_DIR` becomes the base for `workspace` and profile-suffixed workspace paths when set.

## Best Possible Solution

Make a deliberate workspace isolation contract: either derive default workspace roots from the normalized state-dir helper everywhere, with tests/docs/changelog and SDK mirror alignment, or document that workspaces remain an explicit separate knob.

## Reproduction Assessment

Yes. A source-level reproduction is clear: on current main, default-agent fallback calls `resolveDefaultAgentWorkspaceDir(env)`, which ignores `OPENCLAW_STATE_DIR`, while non-default fallback uses `resolveStateDir(env)`.

## Solution Assessment

No. The requested direction may be acceptable, but this patch is not the best implementation because it uses raw env input, lacks matching tests/docs, misses the current file split, and may need SDK mirror alignment.
