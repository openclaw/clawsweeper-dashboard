## Summary

Keep this PR open. Current main still does not implement the requested default-agent behavior, but the proposed patch is not ready to merge because it uses `OPENCLAW_STATE_DIR` without the repository's normal path expansion and targets the older resolver location rather than the current `workspace-default.ts` split.

## What This Changes

The PR changes default agent workspace resolution so `OPENCLAW_STATE_DIR` becomes the base directory for `workspace` and `workspace-<profile>` paths when set.

## Best Possible Solution

Either revise current main so the default workspace base is derived from the normalized state-dir contract, with tests and docs updated, or explicitly keep workspace roots as a separate `agents.defaults.workspace` setting and document that product decision.

## Reproduction Assessment

Yes. A source-level reproduction is clear: on current main, the default-agent fallback calls `resolveDefaultAgentWorkspaceDir(env)`, which ignores `OPENCLAW_STATE_DIR`, while non-default fallback uses `resolveStateDir(env)`.

## Solution Assessment

No. The direction may be acceptable after maintainer product review, but this patch is not the best implementation because it uses raw env input, misses tests/docs, and is stale against the current resolver file split.
