## Summary

Keep open. The requested workspace isolation behavior is not already on current `main`, but this PR still uses raw `OPENCLAW_STATE_DIR` input, lacks real behavior proof, and changes a documented state/workspace contract that needs maintainer direction before merge.

## What This Changes

The PR changes default agent workspace resolution so `OPENCLAW_STATE_DIR` can replace the default `.openclaw` base before appending the workspace/profile suffix.

## Best Possible Solution

Decide the workspace isolation contract, then implement normalized state-dir path handling in the current split helper with core tests, docs, and SDK mirror alignment if maintainers accept the behavior.

## Reproduction Assessment

Yes, at source level. Current `main` routes the default agent through a helper that ignores `OPENCLAW_STATE_DIR`, and the PR's changed code would use raw env input without expanding the `~` form shown in its own example.

## Solution Assessment

No. The requested direction may be acceptable, but this patch skips existing path normalization, lacks tests/docs/SDK alignment, and needs a branch refresh plus real behavior proof before it is a maintainable fix.
