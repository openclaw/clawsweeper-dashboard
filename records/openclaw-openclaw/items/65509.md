## Summary

Keep open: current main still has a real default-workspace isolation gap, but this PR is not ready to merge because it uses raw `OPENCLAW_STATE_DIR`, targets an older helper location, and does not cover the SDK mirror/docs/tests needed for this user-facing contract change.

## What This Changes

The PR changes default agent workspace resolution so `OPENCLAW_STATE_DIR` becomes the base for default and profile-suffixed workspace paths when set.

## Best Possible Solution

If maintainers want one-env state/workspace isolation, derive the default workspace from the normalized state-dir helper while preserving explicit `agents.defaults.workspace`, update the memory-host SDK mirror, add focused workspace tests for tilde/profile/default-agent behavior, and document the changed default with a changelog entry.

## Reproduction Assessment

Yes. A source-level reproduction is clear: current main ignores `OPENCLAW_STATE_DIR` for the default agent workspace, while the PR as written would turn `OPENCLAW_STATE_DIR=~/.openclaw-work` into a literal `~/...` workspace path instead of the normalized state directory.

## Solution Assessment

No. The direction may be acceptable, but this patch is not the best implementation because it skips existing path normalization, misses the current helper split and SDK mirror, and changes a documented state/workspace contract without tests or docs.
