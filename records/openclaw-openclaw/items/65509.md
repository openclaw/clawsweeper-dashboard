## Summary

Keep open. The requested default-workspace isolation behavior is plausible, but this PR is not mergeable as-is because it skips existing path normalization, lacks after-fix real behavior proof, and needs maintainer confirmation that OPENCLAW_STATE_DIR should also imply the default workspace root.

## What This Changes

The branch changes default agent workspace resolution to use OPENCLAW_STATE_DIR as the base before appending the workspace or profile suffix.

## Best Possible Solution

Decide the workspace isolation contract, then implement it in the current workspace-default helper using normalized state-dir resolution with focused tests and SDK/docs alignment if accepted.

## Reproduction Assessment

Yes, at source level. Current main routes the default agent without agents.defaults.workspace through a helper that ignores OPENCLAW_STATE_DIR, and the PR diff would use raw env input without expanding the tilde form shown in its own example.

## Solution Assessment

No. The requested direction may be acceptable, but this patch skips existing path normalization, lacks tests/docs/SDK alignment, targets an older helper shape, and lacks real behavior proof.
