## Summary

Current main and the shipped v2026.5.18 release cover the reported scope-upgrade approval loop: reconnects pause on non-retry `PAIRING_REQUIRED` errors, pending request ids are resolved from live pairing state, and the paired-token scope upgrade fix from the related report is present.

## What This Changes

The issue reports that the embedded TUI rapidly retries scope-upgrade pairing failures, causing pending approval requestIds to rotate before a user can approve them.

## Best Possible Solution

Keep the released generic gateway reconnect and device-pairing token fixes; preserve `--latest` as preview-only so implicit approval cannot approve the wrong pending requester.
