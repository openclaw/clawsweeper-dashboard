The PR lowers billing auth cooldown defaults from 5h/24h to about 5m/15m, clamps stale billing `disabledUntil` during auth profile ordering, and updates auth-profile tests plus docs.

## Best Possible Solution

Land a stable upgrade path that shortens billing recovery without moving persisted windows forward, with focused regression coverage for fresh store reload after the new cap and maintainer acceptance of the new defaults.

## Reproduction Assessment
