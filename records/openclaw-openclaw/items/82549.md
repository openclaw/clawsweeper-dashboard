## Summary

Keep open: this is a protected maintainer PR, and the latest head still has a concrete P2 secret-handling blocker. The restoration direction fits the linked update regression, but the handoff still serializes env-substituted full config data to temporary storage.

## What This Changes

This PR preserves pre-update channel config through update finalization and post-core plugin repair, changes pre-update snapshot overwrite behavior, and adds update CLI plus Docker upgrade survivor coverage.

## Best Possible Solution

Restore dropped channels from a minimized channel-only handoff or fresh legacy snapshot while keeping env-resolved secret values in memory and persisting only authored channel values.

## Reproduction Assessment

Yes for source-level reproduction: current main's fresh post-core path hands off only plugin result/install records, and the resume path reads the post-doctor config before plugin repair. The linked regression issue supplies concrete package-upgrade steps, but I did not run a live package upgrade in this read-only review.

## Solution Assessment

No, not as written: restoring pre-update channels is the right direction, but serializing `configSnapshot.sourceConfig` is not the narrowest maintainable or secret-safe handoff. The safer shape is a minimized unresolved/authored channel payload plus legacy snapshot fallback.

## Review Findings

Overall correctness: patch is incorrect
