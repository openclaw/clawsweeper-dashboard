## Summary

Keep open: current main still has a real doctor recovery gap. The stale-plugin repair scans and mutates `plugins.allow`, `plugins.entries`, dangling channel config, heartbeat targets, and channel model overrides, but it does not include `plugins.slots.*`, while config validation treats an explicit missing memory slot as a hard error.

## What This Changes

Make `openclaw doctor --fix` repair stale `plugins.slots.*` references consistently with stale plugin allowlist and entry cleanup.

## Best Possible Solution

Extend the doctor stale-plugin cleanup to treat slot references atomically with allow/entries cleanup, using the existing slot defaults when removing a selected stale plugin and warning instead of reporting a complete repair when a slot cannot be made valid.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes from source inspection: a config with `plugins.slots.memory` pointing at a missing plugin will fail validation, and the current doctor stale cleanup never scans or removes that slot reference.

## Solution Assessment

Is this the best way to solve the issue? Yes: the narrow maintainable fix belongs in `stale-plugin-config`/doctor repair, with regression coverage, rather than weakening config validation or adding a new user option.
