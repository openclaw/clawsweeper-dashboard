## Reproduction Assessment

## Solution Assessment

Yes, this appears to be the narrow maintainable fix because it aligns the initial scan with the existing unavailable-lsof classification used by the later polling path. A separate preflight binary lookup would add another resolution path without handling the spawn-error boundary as directly.
