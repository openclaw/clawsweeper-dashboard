## Summary

Keep open. Current main still warns for any initial `lsof` spawn error, while this PR narrowly aligns the initial stale-PID scan with the existing permanent-unavailable handling used by the later polling path.

## What This Changes

The PR makes the Unix initial stale-PID scan silently return for ENOENT/EACCES/EPERM `lsof` spawn errors, shares that classification with port polling, and updates restart tests.

## Best Possible Solution

Merge this focused fix after validation so expected lsof-unavailable states stay quiet while unexpected spawn errors and nonzero lsof statuses remain visible.

Yes. Source inspection on current main shows `findGatewayPidsOnPortSync` warning on any `res.error`, and the existing test pins warning behavior for an ENOENT-like lsof failure.

## Solution Assessment
