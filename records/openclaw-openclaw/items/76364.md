## Summary

Keep open. Current main still warns on the initial Unix `lsof` spawn-error path, and this PR is a narrow implementation for the linked minimal-Linux report with no blocking review findings found.

## What This Changes

The PR makes initial Unix stale-PID scanning return quietly for ENOENT/EACCES/EPERM `lsof` spawn failures, reuses that classification in polling, and updates restart tests.

## Best Possible Solution

Merge this focused fix after targeted restart tests and changed-gate validation so expected missing-or-blocked `lsof` states stay quiet while unexpected failures remain visible.

## Reproduction Assessment

Yes, by source inspection. Current main warns on any `res.error` from the initial `findGatewayPidsOnPortSync` lsof scan, and a missing command produces a spawn error with code ENOENT.

## Solution Assessment

Yes. The PR is the narrow maintainable fix because it shares the same ENOENT/EACCES/EPERM classification already used by the later poll path and preserves warnings for unexpected spawn errors and nonzero lsof statuses.
