## What This Changes

The issue requests changing foreground reply fencing so an older active in-flight reply is not truncated after a newer generation produces visible delivery.

## Best Possible Solution

Keep this issue open while maintainers choose and land the foreground-fence fix that prevents truncation without introducing stale or out-of-order visible delivery.
