## Summary

## What This Changes

The issue requests changing foreground reply cancellation so an older active in-flight generation is not cancelled merely because a newer generation delivered visibly.

## Best Possible Solution

Land one reviewed foreground-fence fix through the existing linked PR path, preserving active in-flight delivery while also addressing the lazy-fence ordering concern tracked in https://github.com/openclaw/openclaw/issues/93858.

## Reproduction Assessment

Yes, at source level. Current main and v2026.6.9 still cancel solely on a newer visible generation, and current freshness coverage expects the older active final to be suppressed.

## Solution Assessment

No, not by itself. The one-line active-generation guard is a plausible mitigation for truncation, but the best final fix should also handle the ordering/interleaving concern raised in https://github.com/openclaw/openclaw/issues/93858.
