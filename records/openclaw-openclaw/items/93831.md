## Summary

Keep open: current main and the latest release still cancel an older foreground reply as soon as a newer generation has visible delivery, and two open closing PRs already target this exact issue.

## What This Changes

The issue requests changing foreground reply fencing so an older active in-flight reply is not cancelled merely because a newer generation delivered visible output.

## Best Possible Solution

Keep this issue open until maintainers land or close one direct closing PR, ideally coordinated with the broader lazy-fence work so truncation and interleaving do not diverge into separate fixes.

Unclear as a final fix: the proposed active-generation guard is a narrow fix for truncation, but the related lazy-fence issue argues it may still allow reply interleaving. The best path is one maintainer-reviewed canonical fix that preserves active delivery and settles the lazy-fence ordering question.
