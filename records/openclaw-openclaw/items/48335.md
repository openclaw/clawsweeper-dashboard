## Summary

Keep open: this draft stacked feature is not implemented on current main and still has source-level prompt-boundary/docs blockers plus missing real behavior proof.

## What This Changes

The branch adds stacked compaction-guard config, detector, and scorer pieces, wires `guardEnabled` into safeguard compaction runtime, and appends guard-specific compaction instructions for high-risk sessions.

## Best Possible Solution

Rebase or replace the stacked series after prerequisites land, with a trusted guard-instruction seam, data-only signal details, updated config docs/help, regression tests, and redacted real compaction-run proof.

## Reproduction Assessment

Yes for the PR defects by source review: PR head routes guard policy through `customInstructions`, while current compaction wraps that channel as untrusted prompt data, and docs/help still say the guard has no runtime effect. This is not an end-to-end current-main reproduction of the broader feature.

## Solution Assessment

No, not as written. The feature direction is plausible, but fixed guard policy should use a trusted instruction path separate from transcript-derived signal data, and docs/help must match the new runtime behavior.
