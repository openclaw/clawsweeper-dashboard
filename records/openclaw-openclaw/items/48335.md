## Summary

Keep open: current main still lacks this guard runtime, but the draft stacked PR is not merge-ready because it routes guard policy through the untrusted custom-instruction path and leaves config docs/help saying the new setting is inert.

## What This Changes

The PR branch adds compaction guard config scaffolding, transcript-tail detection/scoring, and a safeguard compaction hook that conditionally augments compaction instructions for high-risk sessions.

## Best Possible Solution

Rebase the stack onto current `src/agents/pi-hooks`, add a trusted guard-instruction seam that keeps transcript-derived signal details bounded as data, and update config docs/help once `guard.enabled` has runtime effect.

## Reproduction Assessment

Yes. Static review reproduces the PR defects: guard guidance is appended to `customInstructions`, which current main wraps as untrusted prompt data, and the PR docs/help text still says the setting has no runtime effect.

## Solution Assessment

No, not as written. The feature direction is plausible, but the runtime hook should separate trusted fixed guard policy from untrusted transcript-derived evidence and keep documentation aligned with behavior.
