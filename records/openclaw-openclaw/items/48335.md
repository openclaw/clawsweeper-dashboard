## Summary

Keep open. Current main still lacks this loop-aware safeguard compaction hook, and the draft stacked PR has blocking prompt-boundary/docs issues plus missing real behavior proof before maintainers can consider it merge-ready.

## What This Changes

The branch adds compaction guard config/detector/scorer stack pieces and wires `guard.enabled` into safeguard compaction instruction augmentation for high-risk sessions.

## Best Possible Solution

Rebase and land the prerequisite pieces first, then add a trusted guard-instruction seam, keep transcript signal details bounded as data, update the active agents config docs/help, and provide real compaction-run proof.

## Reproduction Assessment

Yes for the PR defects by source review: the PR routes guard-augmented policy through `customInstructions`, which the compaction structure helper wraps as untrusted prompt data, and the docs/help claim no runtime effect while `guard.enabled` changes runtime instructions. This is not an end-to-end current-main bug reproduction for the broader feature.

## Solution Assessment

No, not as written. The feature direction is plausible, but the runtime hook should separate trusted guard policy from transcript-derived evidence and the stack should be refreshed with current docs and proof before merge.
