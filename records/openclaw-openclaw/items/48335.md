## Summary

Keep open. Current main still lacks the loop-aware compaction guard, but this draft stacked PR is not merge-ready because its prerequisite PRs are still open/conflicting and the diff has prompt-boundary and docs mismatches.

## What This Changes

The PR stacks compaction guard config scaffolding, transcript-tail detection/scoring, and runtime instruction augmentation for high-risk safeguard compactions.

## Best Possible Solution

Rebase and land the prerequisite pieces first, then wire the guard through a trusted instruction seam with bounded signal data and update current config docs/help to match behavior.

## Reproduction Assessment

Yes for the PR defects: static source review shows the guard text flows through the untrusted custom-instructions wrapper, and the diff shows docs claiming the guard has no runtime effect while the runtime hook makes it active. The broader feature has no end-to-end current-main reproduction fixture in this PR.

## Solution Assessment

No, not as written. The feature direction is plausible, but the runtime hook should separate trusted guard policy from transcript-derived evidence and the stack should be rebased with docs aligned to current behavior.
