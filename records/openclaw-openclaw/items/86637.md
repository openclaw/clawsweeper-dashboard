## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Require a closed DSML wrapper before recovery:** `src/agents/deepseek-text-filter.ts:54-55`
  - body: This final-flush branch pushes `dsmlCapture` even when no DSML close token was seen. A stream that ends with a closed `<|DSML|invoke>` but no closing `<|DSML|tool_calls>` wrapper would still be parsed and executed, while the reported fix shape only justifies recovery from complete DSML blocks; leave incomplete captures non-executable and add a regression test.
  - confidence: 0.91
- **[P1] Skip DSML recovery when native tool calls arrived:** `src/agents/openai-transport-stream.ts:2740-2742`
  - body: Recovery currently runs after native tool-call blocks are finalized and injects every parsed DSML invoke into `output.content`. If a provider emits native `tool_calls` plus mirrored DSML invoke text, this creates an extra executable tool call; track whether native tool calls arrived and skip DSML recovery in that case, preserving the existing native-tool DSML filtering contract.
  - confidence: 0.89
