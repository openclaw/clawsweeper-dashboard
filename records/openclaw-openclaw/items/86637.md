## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Require a closed DSML wrapper before recovery:** `src/agents/deepseek-text-filter.ts:55`
  - body: `flush()` currently pushes `dsmlCapture` into `capturedBlocks` at stream end even when the outer DSML wrapper never closed. That means a truncated response such as an opened `tool_calls` block containing a closed `invoke` can still be parsed later and executed as a synthetic tool call, even though the related bug report's safe boundary is complete DSML only. Drop incomplete captures or track a closed-block flag before returning recovered calls.
  - confidence: 0.92
- **[P1] Skip DSML recovery when native tool calls arrived:** `src/agents/openai-transport-stream.ts:2740-2742`
  - body: The patch unconditionally injects every recovered DSML call after finishing native tool calls. Current main already has DeepSeek tests for streams that contain DSML text and native `tool_calls` in the same response; if the DSML text contains an `invoke`, this path appends another executable call after the native one and can duplicate side-effecting tools. Treat native tool_calls as authoritative and only recover DSML when no native calls were received, or add proof for a different rule.
  - confidence: 0.9
