## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Require a closed DSML wrapper before recovery:** `src/agents/deepseek-text-filter.ts:55`
  - body: This final-flush branch stores `dsmlCapture` even when no DSML close token was seen. That means an unterminated `<|DSML|tool_calls>` wrapper containing a closed invoke can still be recovered as an executable tool call, which is not the conservative recovery path described by the linked bug. Only capture blocks that closed successfully, or carry an explicit complete/incomplete state and ignore incomplete captures.
  - confidence: 0.91
- **[P1] Skip DSML recovery when native tool calls arrived:** `src/agents/openai-transport-stream.ts:2741-2742`
  - body: Recovery runs after native `tool_calls` have already been added to `output.content`, and it injects every recovered DSML call without checking for existing toolCall blocks. If a provider chunk carries both native tool_calls and DSML text, this appends extra synthetic calls to the same assistant turn and can duplicate or conflict with the native tool execution. Preserve the native path by recovering DSML only when no native tool calls were produced, with a regression in the stream tests.
  - confidence: 0.9
