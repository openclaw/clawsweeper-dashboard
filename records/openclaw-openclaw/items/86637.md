## Best Possible Solution

Keep the recovery idea, but gate it to complete fallback-only DSML tool blocks, preserve native tool_calls as authoritative, add transport-level regression coverage, and require redacted live patched-runtime proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Require a closed DSML wrapper before recovery:** `src/agents/deepseek-text-filter.ts:53-55`
  - body: This final-flush branch stores `dsmlCapture` even when the outer `<|DSML|tool_calls>` block never closed. A truncated provider stream like `<|DSML|tool_calls><|DSML|invoke ...></|DSML|invoke>` would now execute the inner invoke even though the wrapper was incomplete; recovery should only record blocks seen through a matching DSML close token, with a regression test for an unterminated wrapper.
  - confidence: 0.9
- **[P1] Skip DSML recovery when native tool calls arrived:** `src/agents/openai-transport-stream.ts:2740-2742`
  - body: The recovery loop runs after all native `choiceDelta.tool_calls` have already populated `output.content`, but it does not check whether native tool calls were seen. If DeepSeek sends both native tool_calls and DSML invoke text in the same response, OpenClaw will execute both the native and synthetic calls; track native tool-call presence or dedupe/suppress recovery so the existing native path remains authoritative.
  - confidence: 0.84
