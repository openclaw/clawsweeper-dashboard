## Summary

Keep open for maintainer review: current main and the latest release still lack the DSML recovery-buffer cap, and this focused patch appears to fix the source-reproducible unbounded-buffer path with regression coverage.

## What This Changes

The PR adds a UTF-8 byte-counted 256,000-byte cap to DeepSeek DSML tool-call recovery buffering and adds oversized ASCII and multibyte regression tests.

## Best Possible Solution

Land this PR or an equivalent byte-counted recoverer cap after maintainers accept the oversized-DSML recovery cutoff and merge checks remain clean.

## Reproduction Assessment

Yes. Source inspection on current main shows an opened DeepSeek DSML block stays in the recoverer buffer until a close token or final flush, with no size guard before this PR.

## Solution Assessment

Yes. The recoverer owns the retained buffer, and a UTF-8 byte cap matching the nearby stream caps is narrower than moving this to provider routing, downstream filtering, or channel behavior.
