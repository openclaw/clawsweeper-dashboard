## What This Changes

The PR adds a 256,000-byte UTF-8-counted DeepSeek DSML recovery-buffer cap and tests for oversized ASCII and multibyte DSML blocks.

## Best Possible Solution

Land this PR or an equivalent UTF-8 byte-counted recoverer cap after maintainers accept the 256 KB oversized-DSML behavior.

## Reproduction Assessment

Yes. Source inspection on current main shows an opened DSML block remains buffered across chunks until a close token or final flush, with no size cap before then.

## Solution Assessment

Yes, pending maintainer acceptance of the threshold. The recoverer is the narrow owner of the unbounded buffer, and a UTF-8 byte cap matching sibling stream caps is more maintainable than downstream-only filtering.
