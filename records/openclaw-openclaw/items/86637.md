## Summary

Keep open: the DSML recovery is not on current main and addresses a real DeepSeek/Foundry tool-call failure, but this branch still has security-boundary and availability blockers plus only source/simulation proof instead of patched-runtime or live stream proof.

## What This Changes

The branch extends the DeepSeek text filter to capture DSML markup, parse invokes into synthetic tool calls, inject recovered calls into the OpenAI completions stream when native tool calls are absent, and adds parser/filter tests.

## Best Possible Solution

Land one canonical, bounded DSML recovery implementation that only promotes complete executable tool-call wrappers, preserves native-tool precedence, includes regression coverage, and has redacted patched-runtime or live DeepSeek/Foundry output.

## Solution Assessment

No, not yet: the recovery direction is plausible, but the branch should first retain wrapper kind, enforce a capture size limit, and provide real patched-stream or live provider proof.
