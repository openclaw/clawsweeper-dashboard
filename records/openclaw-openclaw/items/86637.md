## Summary

This PR is still useful but not merge-ready: it targets a real provider-controlled DSML buffering availability problem, but the added cap only runs when no close token is present, so an oversized complete block in one delta still bypasses the guard before parsing.

## What This Changes

Adds a 256 KB DeepSeek DSML recovery-buffer cap in `src/agents/openai-transport-stream.ts` and a focused oversized-DSML regression test.

## Best Possible Solution

Enforce one UTF-8 byte cap before both incomplete buffering and complete DSML block parsing, preserve normal below-limit recovery, and cover split-close plus same-delta oversized cases with real runtime proof.

## Reproduction Assessment

Yes. Source inspection shows current main retains DSML content until a close token or final flush, and the PR test demonstrates the split-close oversized path; the same-delta cap bypass is source-provable from the PR head.

## Solution Assessment

No. The PR is in the right layer, but the current condition is too narrow because it only caps no-close buffering and does not use the sibling UTF-8 byte measurement.
