## Summary

Keep open: this is a focused bug-fix PR for an uncapped provider-controlled DeepSeek DSML recovery buffer that still exists on current main and the latest release; the patch adds a sibling-aligned byte cap and regression tests, with maintainer acceptance needed for the oversized-tool-call recovery tradeoff.

## What This Changes

Adds a 256,000-byte cap to DeepSeek DSML tool-call recovery buffering and tests oversized ASCII and multibyte DSML blocks.

## Best Possible Solution

Land this PR or an equivalent UTF-8 byte-counted recoverer cap once maintainers accept the 256 KB oversized-DSML behavior.

## Reproduction Assessment

Yes. Source inspection on current main shows the DSML recoverer retains an opened block across chunks and only clears it on close-token discovery or final flush, with no size cap before this PR.

## Solution Assessment

Yes. The recoverer is the narrow owner of the unbounded buffer, and a UTF-8 byte cap matching sibling stream caps is more maintainable than bounding only downstream filtering.
