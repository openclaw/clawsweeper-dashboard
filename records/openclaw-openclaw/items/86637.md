## Summary

This PR targets a real DeepSeek/Foundry tool-call failure, but it should stay open because the branch still promotes provider text into executable tool calls without preserving the DSML wrapper kind, without a DSML capture cap, and without patched-runtime or live tool-execution proof.

## What This Changes

The PR extends the DeepSeek text filter to capture and parse DSML tool-call markup, injects recovered synthetic tool calls into the OpenAI completions stream when native tool calls are absent, and adds parser/filter regression coverage.

## Best Possible Solution

Land one canonical DSML recovery implementation that only promotes complete executable DSML call wrappers, preserves native tool-call precedence, bounds retained hidden text, and includes patched-runtime or live Foundry proof of exactly one tool execution.
