## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Require a closed DSML wrapper before recovery:** `src/agents/deepseek-text-filter.ts:55`
  - body: `flush()` currently stores `dsmlCapture` even when the stream ended before any closing DSML wrapper. If a response is truncated after `<...tool_calls>` but after a closed `<...invoke>`, `parseDsmlToolCalls()` will still synthesize and execute that tool call, contrary to the linked issue's expected complete-block guard. Please only add captured blocks when a close token was observed and cover the incomplete-wrapper case in tests.
  - confidence: 0.9
- **[P1] Skip recovery after native tool calls:** `src/agents/openai-transport-stream.ts:2740-2742`
  - body: Recovered DSML calls are injected after the native `choiceDelta.tool_calls` path has already populated `output.content`. Existing DeepSeek tests allow DSML text to coexist with native tool calls as filter-only content; with this patch, an `<invoke>` in that filtered text would add a second synthetic tool call and can duplicate or shadow execution. Please skip DSML recovery when any native toolCall block was produced, with a regression test for native-plus-DSML invoke content.
  - confidence: 0.88
