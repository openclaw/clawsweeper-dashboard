## Summary

Keep this PR open: it targets a real agent-stream availability bug, but the current patch only caps the no-close-token path and still lets a complete oversized DSML block be parsed when the close token arrives in the same delta.

## What This Changes

The PR adds a 256 KB DeepSeek DSML recovery-buffer cap in `src/agents/openai-transport-stream.ts` and a regression test for an oversized split-close DSML block.

## Best Possible Solution

Enforce one UTF-8 byte cap before both incomplete buffering and complete DSML block parsing, preserve normal below-limit recovery, and add split-close plus same-delta tests with redacted runtime or agent proof.

## Reproduction Assessment

Yes. Source inspection shows current main retains DSML content until a close token or final flush, and the PR test demonstrates the split-close oversized path; the same-delta cap bypass is also source-provable from the PR head.

## Solution Assessment

No. The PR is in the right layer, but the current condition is too narrow because it only caps no-close buffering and does not use the sibling UTF-8 byte measurement.
