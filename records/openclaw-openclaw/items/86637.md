## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Keep error DSML blocks non-executable:** `src/agents/deepseek-text-filter.ts:44`
  - body: The filter already treats `tool_use_error` as a DSML wrapper, but the new capture path stores every closed DSML body and `recoveredToolCalls()` parses invokes from all captured blocks. A provider error wrapper that contains invoke-shaped markup would be promoted into a synthetic toolCall after flush, so the filter needs to preserve the wrapper kind and recover only executable tool-call wrappers, with a regression test for `tool_use_error` staying non-executable.
  - confidence: 0.91
- **[P2] Bound DSML capture before retaining hidden text:** `src/agents/deepseek-text-filter.ts:51`
  - body: Native tool-call arguments in this stream are capped at 256 kB, but the new `dsmlCapture` path appends hidden provider text into memory without an equivalent byte limit before storing and parsing it. A very large DSML block can now grow retained memory and regex work outside the existing stream safeguards; add a cap and regression coverage for oversized DSML recovery.
  - confidence: 0.86
