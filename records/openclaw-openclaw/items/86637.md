## Summary

Keep open for maintainer review: current main and v2026.6.10 still lack the DSML recovery-buffer cap, and this focused PR is the linked candidate fix with sufficient terminal proof and no blocking code findings.

## What This Changes

Adds a 256,000-byte UTF-8 cap to DeepSeek DSML recovery buffering and regression coverage for oversized ASCII and multibyte payloads.

## Best Possible Solution

Land this PR or an equivalent byte-counted recoverer cap after maintainers accept the 256 KB oversized-DSML behavior.

## Reproduction Assessment

Yes. Source inspection on current main shows the DSML recoverer retains an opened block across chunks and only clears it on close-token discovery or final flush, with no size cap before this PR.

## Solution Assessment

Yes. The recoverer is the narrow owner of the unbounded buffer, and a UTF-8 byte cap matching sibling stream caps is cleaner than trying to bound only the downstream text filter.
