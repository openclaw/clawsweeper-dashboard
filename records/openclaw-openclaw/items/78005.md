## Summary

This PR should stay open, but the current patch is not ready to merge: it adds a plausible deep status diagnostic while still producing false positives for independently versioned plugins, reading the wrong install ledger in daemon/profile splits, and carrying a stale generated schema file. It also still lacks contributor-provided real behavior proof for the changed `status --deep` output.

## What This Changes

The PR adds a deep-only `openclaw [gateway] status --deep` diagnostic for npm/ClawHub plugin versions that differ from the gateway version, plus tests and a changelog entry.

## Best Possible Solution

Keep the diagnostic concept, but implement it against the official/gateway-versioned plugin compatibility boundary, read install records from daemon state, rebase away deleted generated output, and add redacted real `status --deep` proof before merge.

## Reproduction Assessment

Yes, for the review findings by source inspection: the PR helper reports a normal ClawHub plugin at `0.1.0` as drift from gateway `2026.5.4`, and the gather path reads install records without daemon env. The original real-world drift case was not rerun because this review is read-only.

## Solution Assessment

No. `status --deep` is a plausible surface, but this patch needs a narrower official compatibility check, daemon-state record lookup, rebase cleanup, and real behavior proof before it is the maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Limit drift checks to gateway-versioned plugins:** `src/plugins/plugin-version-drift.ts:82`
  - body: The helper treats every npm/ClawHub install as if it must match the gateway version. ClawHub and npm also host independently versioned third-party plugins, so this will produce persistent false warnings for normal plugins such as a ClawHub package at `0.1.0`. Filter to official OpenClaw external plugins or another explicit compatibility signal before reporting drift.
  - confidence: 0.91
- **[P2] Read install records from the daemon state:** `src/cli/daemon-cli/status.gather.ts:506`
  - body: Deep status resolves daemon/service context elsewhere, but this lookup calls the installed-record reader without passing that environment. If the gateway service uses a different state/profile than the CLI shell, `status --deep` can read the wrong `plugins/installs.json` and either miss real drift or report drift for the wrong install set.
  - confidence: 0.88
- **[P2] Drop the deleted generated schema snapshot:** `src/config/schema.base.generated.ts:29438`
  - body: Current main removed `src/config/schema.base.generated.ts` when base config schema moved away from the generated snapshot. Keeping this version bump makes the branch stale and risks reintroducing retired generated output during conflict resolution.
  - confidence: 0.86
