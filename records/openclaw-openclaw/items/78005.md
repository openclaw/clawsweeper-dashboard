## Summary

Keep open; the deep-status diagnostic is useful and the previous running-gateway/service-env issues appear addressed, but the current patch can still produce false drift warnings for ordinary npm/ClawHub plugins and for install records disabled by the existing plugin activation policy.

The PR adds a deep-only `status` diagnostic that reads plugin install records, compares npm/ClawHub plugin versions to the running Gateway version, prints drift warnings, and adds focused unit coverage.

## Best Possible Solution

Keep the deep-only diagnostic, but restrict it to trusted official external installs with a lockstep version contract and gate it through the existing effective plugin activation policy.

## Reproduction Assessment

Yes, source-reproducible: an ordinary npm/ClawHub install with version `1.2.3` or an install record disabled by `plugins.enabled=false`, `plugins.deny`, or restrictive `plugins.allow` will still reach the new drift comparison because the helper only checks source and `plugins.entries.<id>.enabled`. I did not run tests in this read-only review.

## Solution Assessment

No. `status --deep` is a good surface for this, but the implementation should reuse the official external install contract and effective plugin activation policy instead of a local source/enabled check.
