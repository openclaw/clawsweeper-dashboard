## Summary

Keep open: the diagnostic idea is plausible, but this PR still has correctness blockers, lacks required real behavior proof, and is stale against current main’s removed generated schema file.

## What This Changes

The PR adds a deep-only status diagnostic, helper, tests, printed output, and changelog entry for npm/ClawHub plugin versions that differ from the gateway version.

## Best Possible Solution

Keep the diagnostic concept, but gate it to official/gateway-versioned external plugins, read daemon-scoped install records, drop stale generated output, and require redacted real `status --deep` proof.

## Reproduction Assessment

Yes, for the review findings by source inspection: the helper reports a normal ClawHub plugin at `0.1.0` as drift, and the gather path calls the install-record loader without daemon env. The original real-world version-mismatch case was not rerun in this read-only review.

## Solution Assessment

No. `status --deep` is a reasonable surface, but this branch needs a narrower official compatibility check, daemon-state ledger lookup, and stale-file cleanup before it is the maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Limit drift checks to gateway-versioned plugins:** `src/plugins/plugin-version-drift.ts:82`
  - body: The helper treats every npm/ClawHub install as if it must match the gateway version. ClawHub and npm can host independently versioned third-party plugins, so this will produce persistent false warnings for normal plugins such as a ClawHub package at `0.1.0`; filter to official OpenClaw external plugins or another explicit compatibility signal before reporting drift.
  - confidence: 0.91
- **[P2] Read install records from the daemon state:** `src/cli/daemon-cli/status.gather.ts:506`
  - body: Deep status resolves daemon/service context elsewhere, but this lookup calls the installed-record reader without passing that environment. If the gateway service uses a different state/profile than the CLI shell, `status --deep` can read the wrong `plugins/installs.json` and either miss real drift or report drift for the wrong install set.
  - confidence: 0.88
- **[P2] Drop the deleted generated schema snapshot:** `src/config/schema.base.generated.ts:29438`
  - body: Current main removed `src/config/schema.base.generated.ts` when base config schema moved to runtime computation. Keeping this version bump makes the branch stale and risks reintroducing retired generated output during conflict resolution.
  - confidence: 0.86
