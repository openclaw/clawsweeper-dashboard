## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Show auth mismatch errors instead of reconnecting forever:** `ui/src/ui/app-gateway.ts:376-377`
  - body: This path clears `lastError` and `lastErrorCode` for all recoverable closes after a previous connection, including exhausted `AUTH_TOKEN_MISMATCH` cases. Operators can be left in a reconnecting state with no credential-fix guidance while chat remains unavailable.
  - confidence: 0.86
- **[P2] Mark first-seen successful tool results complete:** `ui/src/ui/app-tool-stream.ts:573-574`
  - body: When the first observed event for a tool is a successful `result`, the new entry is initialized as `running` and never reaches the existing-entry branch that marks it complete. Reconnect or sequence-gap recovery can therefore display a completed tool as running indefinitely.
  - confidence: 0.88
- **[P2] Preserve attachment context when retrying prompts:** `ui/src/ui/chat/grouped-render.ts:1201`
  - body: The retry action forwards only the extracted markdown text, so retrying a user message with images or other attachments silently drops that context. That changes the prompt semantics for multimodal turns.
  - confidence: 0.84
