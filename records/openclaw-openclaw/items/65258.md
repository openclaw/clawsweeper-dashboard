## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Show exhausted auth mismatches instead of reconnecting forever:** `ui/src/ui/app-gateway.ts:376-388`
  - body: When `AUTH_TOKEN_MISMATCH` closes after the browser client has already used its device-token retry budget, the client intentionally stops scheduling reconnects. This branch treats that close as recoverable after any prior successful connection, clears `lastError`/`lastErrorCode`, and returns, leaving the UI stuck in reconnecting state without the credential error operators need to fix.
  - confidence: 0.89
- **[P2] Mark first result-only tool events complete:** `ui/src/ui/app-tool-stream.ts:573-575`
  - body: If the first event seen for a tool call is `phase === "result"` with `isError` false, the new entry is initialized as running and never enters the existing-entry branch that flips it to complete. Reconnect or sequence-gap recovery can therefore show a finished tool as running indefinitely.
  - confidence: 0.86
- **[P2] Preserve retry attachments or hide retry:** `ui/src/ui/chat/grouped-render.ts:1194-1197`
  - body: The retry action forwards only the extracted markdown text. For user turns that included files or images, clicking retry silently drops the attachments and sends a different prompt, so multimodal retry can produce incorrect behavior.
  - confidence: 0.85
- **[P2] Route preserved OpenAI IDs through collision handling:** `src/agents/tool-call-id.ts:453-456`
  - body: The `preserveId` early return bypasses the occurrence-aware resolver. If replay history contains repeated canonical `callId|fc_*` IDs, assistant tool calls and matching results keep duplicate raw IDs, reintroducing the provider rejection this sanitizer normally prevents.
  - confidence: 0.82
- **[P2] Do not label every shutdown close as a restart:** `ui/src/ui/app-gateway.ts:382-388`
  - body: `pendingShutdownMessage` is set for both restart and non-restart shutdown events, but the close handler uses its presence to show `Gateway restarting.` and suppress the disconnected error. A normal shutdown after a prior connection can therefore tell operators to wait for a restart that will not happen.
  - confidence: 0.81
- **[P2] Honor edited spaces when session metadata exists:** `ui/src/ui/chat-spaces.ts:43-47`
  - body: `setChatSpace` stores a user override, but `resolveChatSpace` always returns the backend `sessionSpace` first. For sessions that already have space metadata, the field appears editable but snaps back on the next render and cannot change grouping locally.
  - confidence: 0.8
- **[P2] Save chat-space edits before rerenders:** `ui/src/ui/app-render.helpers.ts:195-199`
  - body: The space input is controlled with `.value` but only persists on `change`, so any live rerender before blur or Enter reapplies the old value and discards the operator's in-progress edit. Persist on input or keep draft state for the field.
  - confidence: 0.78
- **[P3] Use attachment kind in preview labels:** `ui/src/ui/views/chat.ts:684-690`
  - body: The new context rail labels every attachment as `Image N` even after computing the MIME-derived kind label. Audio, video, and document attachments will have incorrect visible labels and accessibility text.
  - confidence: 0.77
- **[P3] Add the required changelog entry:** `ui/src/ui/app.ts:146-154`
  - body: This PR adds user-facing Control UI chat, reconnect, approval, and artifact behavior but does not update `CHANGELOG.md`. Repository policy requires a changelog entry for user-facing feature/fix/perf changes before merge.
  - confidence: 0.9
