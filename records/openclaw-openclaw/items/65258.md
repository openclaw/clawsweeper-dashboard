## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Show exhausted auth mismatches instead of reconnecting forever:** `ui/src/ui/app-gateway.ts:376-388`
  - body: When AUTH_TOKEN_MISMATCH closes after the browser client has used its device-token retry budget, this branch still clears the credential error and shows reconnecting after any prior successful connection. Operators lose the actionable auth failure and chat remains unavailable.
  - confidence: 0.89
- **[P2] Mark first result-only tool events complete:** `ui/src/ui/app-tool-stream.ts:573-575`
  - body: If the first event seen for a tool call is a successful result, the new entry is initialized as running and never enters the branch that marks results complete. Reconnect or sequence-gap recovery can therefore show a finished tool as running indefinitely.
  - confidence: 0.86
- **[P2] Preserve retry attachments or hide retry:** `ui/src/ui/chat/grouped-render.ts:1194-1197`
  - body: The retry action forwards only markdown text, so retrying a user turn with files or images silently resends a different text-only prompt. Multimodal retries should either preserve attachments or not offer retry for attachment-bearing turns.
  - confidence: 0.85
- **[P2] Route preserved OpenAI IDs through collision handling:** `src/agents/tool-call-id.ts:453-456`
  - body: The preserveId early return bypasses the occurrence-aware resolver. Replayed duplicate canonical callId|fc_* IDs can therefore keep duplicate assistant tool-call IDs and matching results, reintroducing provider rejection risk.
  - confidence: 0.82
- **[P2] Distinguish shutdown from restart in reconnect banner:** `ui/src/ui/app-gateway.ts:382-388`
  - body: pendingShutdownMessage is set for restart and non-restart shutdown events, but the close handler uses its presence as a restart signal. A normal shutdown can tell operators the gateway is restarting and suppress the disconnected error.
  - confidence: 0.81
- **[P2] Honor edited spaces when metadata exists:** `ui/src/ui/chat-spaces.ts:43-47`
  - body: setChatSpace stores a user override, but resolveChatSpace returns backend sessionSpace first. Sessions with existing space metadata appear editable, then snap back on rerender and cannot be regrouped locally.
  - confidence: 0.8
- **[P2] Save chat-space edits before rerenders:** `ui/src/ui/app-render.helpers.ts:195-199`
  - body: The space input is controlled with .value but persists only on change. A live rerender before blur or Enter reapplies the old value and discards the operator's in-progress edit.
  - confidence: 0.78
- **[P3] Use attachment kind in preview labels:** `ui/src/ui/views/chat.ts:684-690`
  - body: The context rail computes the MIME-derived kind label but still labels every attachment as Image N. Audio, video, and document attachments get incorrect visible and accessibility text.
  - confidence: 0.77
