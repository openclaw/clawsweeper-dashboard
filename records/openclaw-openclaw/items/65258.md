## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Show exhausted auth mismatches instead of reconnecting forever:** `ui/src/ui/app-gateway.ts:375-377`
  - body: When AUTH_TOKEN_MISMATCH closes after the browser client has already used its device-token retry budget, the client intentionally stops scheduling reconnects. This branch clears lastError/lastErrorCode and leaves the dashboard in reconnecting state, hiding the credential error operators need to fix.
  - confidence: 0.89
- **[P2] Mark first result-only tool events complete:** `ui/src/ui/app-tool-stream.ts:573`
  - body: If the first event seen for a tool call is a successful phase=result event, the new entry is initialized as running and never reaches the existing-entry branch that marks results complete. Reconnect or sequence-gap recovery can therefore show a finished tool as running indefinitely.
  - confidence: 0.86
- **[P2] Preserve attachments when retrying prompts:** `ui/src/ui/chat/grouped-render.ts:1201`
  - body: The retry action forwards only the extracted markdown text. For user turns that included files or images, retry silently drops the attachments and sends a different prompt.
  - confidence: 0.85
- **[P2] Route preserved OpenAI IDs through collision handling:** `src/agents/tool-call-id.ts:456-458`
  - body: The preserveId early return bypasses resolveAssistantId. Replayed duplicate canonical callId|fc_* IDs can therefore keep duplicate raw IDs, reintroducing provider rejections that the occurrence-aware resolver is meant to prevent.
  - confidence: 0.82
- **[P2] Distinguish shutdown from restart in reconnect banners:** `ui/src/ui/app-gateway.ts:382-383`
  - body: pendingShutdownMessage is set for both restart and non-restart shutdown events, but the close handler treats any prior shutdown message as a restart and suppresses the disconnected error. A normal shutdown can therefore tell operators the gateway is restarting when it is not.
  - confidence: 0.81
- **[P2] Honor edited spaces when session metadata exists:** `ui/src/ui/chat-spaces.ts:47-50`
  - body: setChatSpace stores a user override, but resolveChatSpace returns backend sessionSpace first. Sessions that already have space metadata appear editable but snap back on the next render.
  - confidence: 0.8
- **[P2] Save chat-space edits before rerenders:** `ui/src/ui/app-render.helpers.ts:195-201`
  - body: The space input is controlled with .value but only persists on change. Any live rerender before blur or Enter reapplies the old value and discards the operator's in-progress edit.
  - confidence: 0.78
- **[P3] Use attachment kind in preview labels:** `ui/src/ui/views/chat.ts:701-702`
  - body: The context rail computes a MIME-derived kind label but still names every attachment Image N. Audio, video, and document attachments get incorrect visible labels and accessibility text.
  - confidence: 0.77
