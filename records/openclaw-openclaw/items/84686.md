## What This Changes

The PR threads a skip-plugin-validation option into runtime config loading for remote TUI startup and moves context-window cache warmup from agents/context module evaluation into EmbeddedTuiBackend.start().

## Best Possible Solution

Land the bounded remote-only skip if maintainers accept deferred plugin validation for TUI clients and have enough cold-start evidence that remote mode becomes responsive while embedded mode still fully validates.

## Reproduction Assessment

## Solution Assessment

Mostly yes: the patch uses an existing plugin-validation skip mode at the TUI boundary and moves warmup to the embedded runtime path. The remaining question is compatibility/product fit because remote startup validation errors are intentionally deferred.
