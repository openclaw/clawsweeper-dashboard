## What This Changes

The PR threads a skip-plugin-validation option through runtime config loading for non-local TUI startup and moves context-window cache warmup from agents/context module evaluation into EmbeddedTuiBackend.start().

## Best Possible Solution

Land the bounded remote-only skip if maintainers accept deferred plugin validation for TUI clients and have enough cold-start evidence that remote mode becomes responsive while embedded mode still fully validates.

## Reproduction Assessment

## Solution Assessment

Mostly yes: the patch uses an existing skip mode at the TUI boundary and moves warmup to the embedded runtime path. The remaining question is product/compatibility fit because remote startup no longer surfaces plugin-specific config validation errors immediately.
