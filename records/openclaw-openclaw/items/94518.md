## Summary

Keep open: current main does not prove the reported DeepSeek cache collapse is fixed, and the related prompt-cache-key PR was closed after maintainer live proof showed that field is not DeepSeek's cache contract. Source inspection also weakens the proposed root cause because OpenAI-compatible completions strip the internal cache-boundary marker before sending the system prompt, so this needs provider-owner live prefix-stability investigation rather than cleanup closure.

## What This Changes

The issue asks OpenClaw to restore DeepSeek prompt-cache hit rates after the 2026.6.x upgrade by disabling boundary-aware cache shaping for DeepSeek or applying a provider-specific cache-prefix fix.

## Best Possible Solution
