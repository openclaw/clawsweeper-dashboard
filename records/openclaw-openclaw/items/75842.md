## Summary

Keep open. Current main still intentionally accepts `memory/.dreams/session-corpus/*.txt` as short-term recall input and has no store-level retention or creation-time pruning contract, while existing docs/tests make transcript ingestion an intended memory-core behavior that should not be removed without maintainer direction.

## What This Changes

The issue requests bounded short-term recall storage and prevention of generated session-corpus artifacts from repeatedly feeding memory-core dreaming.

## Best Possible Solution

Define a memory-core source-class and retention contract that keeps intentional redacted transcript ingestion useful, prevents generated dreaming artifacts from self-reinforcing indefinitely, and adds stable creation metadata for pruning.

## Reproduction Assessment

Yes, by source inspection: session transcript ingestion appends `session-corpus` lines and records them into `short-term-recall.json`, while the store write path lacks retention. I did not run the Vitest path because this read-only shell does not have `pnpm`.

## Solution Assessment

Unclear. Retention plus stable source metadata is the maintainable direction, but simply rejecting all `session-corpus` entries would remove behavior that current docs and tests still describe as intentional.
