## Summary

Keep open. Current main still treats `memory/.dreams/session-corpus/*` as short-term recall input and writes those entries without store-level retention or `firstRecordedAt`; the remaining fix needs maintainer direction because session transcript ingestion is documented and tested behavior.

## What This Changes

The issue requests bounding memory-core short-term recall growth by preventing generated session-corpus artifacts from feeding recall, adding retention, and storing stable creation metadata.

## Best Possible Solution

Define and document a memory-core short-term recall retention/source-class contract that keeps intentional transcript ingestion bounded, prevents generated dreaming artifacts from feeding themselves, and preserves stable creation metadata for pruning.

## Reproduction Assessment

Yes. Source inspection on current main shows session-corpus batches are appended and then recorded into `short-term-recall.json`, while the store write path has no retention pass.

## Solution Assessment

Unclear. Retention plus stable creation metadata is the maintainable direction, but excluding all `session-corpus` paths alone conflicts with documented and tested transcript ingestion.
