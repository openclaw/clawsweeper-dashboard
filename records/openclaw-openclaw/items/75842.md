## Summary

Keep open. Current main still records `memory/.dreams/session-corpus/*.txt` into the short-term recall store and only filters obvious contaminated dreaming narrative snippets; it does not add a store-level retention prune or the requested `firstRecordedAt` field.

## What This Changes

The issue requests bounding `memory/.dreams/short-term-recall.json` by excluding generated session-corpus artifacts, adding recall-store retention, and preserving stable creation metadata.

## Best Possible Solution

Define a memory-core retention and source-class contract that bounds the recall store, preserves stable creation metadata, and separates generated transcript-corpus signals from organic recall where needed.

## Reproduction Assessment

Yes. Source inspection on current main shows session-corpus batches are appended and immediately recorded into `short-term-recall.json`, and tests still assert session-corpus paths are valid candidates.

## Solution Assessment

Unclear. Excluding session-corpus entries is a plausible guard, but the narrowest maintainable fix likely needs retention plus source classification so the intentional transcript-ingestion feature is not accidentally removed.
