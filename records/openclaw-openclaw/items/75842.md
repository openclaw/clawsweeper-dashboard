## Summary

Keep open. Current main still records session-corpus transcript batches into the short-term recall store, accepts those paths as promotion candidates, and has no store-level pruning contract, while docs/tests show transcript ingestion is intentional enough to need maintainer policy before changing it.

## What This Changes

Requests that memory-core stop unbounded short-term recall growth from dreaming session-corpus entries by excluding generated corpus sources and/or adding retention metadata.

## Best Possible Solution

Define and document a short-term recall source-class and retention contract that bounds generated transcript-derived state while preserving intentional transcript ingestion and migration-safe creation metadata.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: run dreaming with eligible session transcripts, and the appended session-corpus results are passed into `recordShortTermRecalls` while no persisted-store pruning occurs.

## Solution Assessment

Unclear. Retention plus stable creation metadata is the maintainable direction, but excluding all session-corpus paths alone conflicts with documented and tested transcript ingestion.
