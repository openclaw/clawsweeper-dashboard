## Summary

Keep open. Current main still records newly collected `memory/.dreams/session-corpus/*` batches into `short-term-recall.json`, and changing that path needs maintainer judgment because redacted session transcript ingestion is documented and covered by tests.

## What This Changes

The issue requests bounding memory-core short-term recall growth by preventing session-corpus artifacts from bloating `short-term-recall.json`, potentially through source exclusion, retention, and stable creation metadata.

## Best Possible Solution

Define and document a memory-core source-class and retention contract that keeps intentional transcript ingestion bounded, prevents generated dreaming artifacts from feeding unbounded state, and preserves stable creation metadata for pruning.

## Reproduction Assessment

Yes. Source inspection on current main shows session-corpus batches are appended and recorded into the short-term recall store, and tests assert those paths can rank as candidates; I did not run a live workspace reproduction in this read-only review.

## Solution Assessment

Unclear. Retention plus stable source metadata is the maintainable direction, but excluding all session-corpus paths alone would change documented and tested transcript-ingestion behavior.
