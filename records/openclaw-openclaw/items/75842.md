## Summary

Keep open. Current main still treats `memory/.dreams/session-corpus/*` as short-term recall input, writes those generated corpus paths into the recall store, and only uses `maxAgeDays` as a ranking gate rather than a store-retention pass.

## What This Changes

The issue requests bounding memory-core short-term recall growth by preventing generated session-corpus artifacts from recursively accumulating in `short-term-recall.json` and adding durable retention metadata.

## Best Possible Solution

Define a memory-core short-term recall retention and source-class contract that bounds generated session-corpus entries, preserves stable creation/source metadata, and keeps intentional transcript ingestion documented.

## Reproduction Assessment

Yes, by source inspection on current main. The path from session transcript ingestion to `appendSessionCorpusLines` to `recordShortTermRecalls` is direct, and the write path has no store-level retention.

## Solution Assessment

Unclear. Excluding all session-corpus paths alone conflicts with current docs/tests for redacted transcript ingestion, so the safer fix is a deliberate source-class/retention contract with updated tests and docs.
