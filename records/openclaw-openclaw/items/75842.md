## Summary

Keep open. Current main still accepts `memory/.dreams/session-corpus/*` as short-term recall input and records it into `short-term-recall.json`; the right fix needs maintainer direction because transcript ingestion is documented and tested behavior, while retention/source-class policy is not yet defined.

## What This Changes

The issue asks memory-core to stop or bound short-term recall growth caused by dreaming storing session-corpus artifacts as recall candidates.

## Best Possible Solution

Define and document a memory-core source-class and retention contract that keeps intentional transcript ingestion bounded, prevents generated dreaming artifacts from feeding themselves, and preserves stable creation metadata for pruning.

## Reproduction Assessment

Yes. Source inspection on current main shows session-corpus batches are appended and then recorded into `short-term-recall.json`, while the recall store write path has no retention pruning pass.

## Solution Assessment

Unclear. Excluding all session-corpus paths is not proven to be the best fix because redacted transcript ingestion is documented and tested; a bounded source-class/retention contract is the safer maintainable direction.
