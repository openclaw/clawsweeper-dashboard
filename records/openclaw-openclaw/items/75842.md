## Summary

Keep open. Current main still treats session-corpus files as short-term recall input and has no store-level retention or stable creation timestamp, while adjacent fixes only cover narrative self-ingestion and repair flows.

## What This Changes

Request to prevent `memory/.dreams/short-term-recall.json` from growing through dreaming session-corpus artifacts and to add a bounded retention/creation-metadata path.

## Best Possible Solution

Define a memory-core short-term recall source and retention contract that keeps intentional transcript ingestion bounded, preserves stable creation metadata, and prunes existing store entries safely.

## Reproduction Assessment

Yes. Source inspection on current main shows session-corpus batches are appended and recorded into `short-term-recall.json`, and the store write path has no retention pruning.

## Solution Assessment

Unclear. Excluding all session-corpus paths alone conflicts with documented and tested transcript ingestion, so the maintainable fix is a product-backed retention/provenance contract rather than a mechanical path filter.
