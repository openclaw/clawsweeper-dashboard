## Summary

Keep open. Current main still records `memory/.dreams/session-corpus/*` results into `short-term-recall.json` without a store-retention contract, and the remaining fix needs maintainer direction because session transcript ingestion is documented and tested behavior.

## What This Changes

Bound memory-core short-term recall so generated session-corpus artifacts do not cause unbounded dreaming recall-store growth.

## Best Possible Solution

Define a memory-core short-term recall source-class and retention contract that bounds generated session-corpus growth while preserving intentional transcript-derived recall signals.

## Reproduction Assessment

Yes, from source inspection. Current main appends session-corpus batches, records them as daily short-term recall signals, and rewrites `short-term-recall.json` without age, size, or source-class pruning.

## Solution Assessment

Unclear. Retention plus stable creation/source metadata looks maintainable, but simply excluding all `session-corpus` paths would conflict with documented and tested transcript ingestion.
