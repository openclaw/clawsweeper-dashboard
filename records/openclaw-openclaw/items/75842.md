## Summary

Keep open. Current main still accepts `memory/.dreams/session-corpus/*` as short-term memory, records collected session-corpus batches into `short-term-recall.json`, and has no store-level retention or `firstRecordedAt`; the remaining fix needs a maintainer decision because session transcript ingestion is documented and tested behavior.

## What This Changes

Request to stop memory-core dreaming from letting session-corpus entries cause unbounded `memory/.dreams/short-term-recall.json` growth, with retention and creation-time metadata suggested.

## Best Possible Solution

Define and document a memory-core short-term recall retention/source-class contract that keeps intentional transcript ingestion bounded, prevents generated dreaming artifacts from feeding themselves, and preserves stable creation metadata for pruning.

## Reproduction Assessment

Yes. Source inspection on current main shows session-corpus batches are appended and then recorded into the short-term recall store, and existing tests assert those paths can rank as candidates.

## Solution Assessment

Unclear. Retention plus stable creation metadata is the maintainable direction, but excluding all session-corpus paths alone conflicts with the documented and tested transcript-ingestion feature.
