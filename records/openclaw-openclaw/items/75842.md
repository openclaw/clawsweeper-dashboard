## Summary

Keep open. Current main still has the reported source path: session-corpus files remain short-term recall inputs, and the newer self-ingestion repair path does not add bounded store retention or stable insertion metadata.

## What This Changes

The issue asks memory-core dreaming to keep `short-term-recall.json` bounded by preventing generated session-corpus artifacts from feeding recall, adding store retention, and stamping creation metadata.

## Best Possible Solution

Define a source/provenance and retention contract that preserves useful redacted transcript signals while pruning generated or stale recall-store entries with stable creation metadata.

## Reproduction Assessment

Yes, by source inspection rather than a live run. Current main still appends session-corpus batches and records them into `short-term-recall.json`, while the store write path has no source-class or age pruning.

## Solution Assessment

Unclear. A blanket session-corpus exclusion is not the best proven fix because session transcript ingestion through that path is documented and tested; a provenance-aware retention design is safer.
