## Summary

Keep open. Current main mitigates some contaminated dreaming-narrative artifacts through doctor/repair paths, but the reported short-term recall growth path remains: session-corpus files are still valid short-term recall sources, session transcript ingestion still records them into `short-term-recall.json`, and there is no store-level retention or stable `firstRecordedAt` metadata.

## What This Changes

The issue requests preventing memory-core dreaming from feeding generated session-corpus artifacts back into short-term recall and adding a bounded retention/creation-metadata contract.

## Best Possible Solution

Define a memory-core short-term recall source/retention contract that keeps intentional transcript signals bounded, prevents generated dreaming artifacts from feeding themselves, and preserves stable creation metadata for pruning and repair.

## Reproduction Assessment

Yes. Source inspection on current main shows the session transcript ingestion path appends session-corpus lines and records them into `short-term-recall.json`, while the store write path has no retention pass.

## Solution Assessment

Unclear. Excluding all `session-corpus` paths is not the narrowest maintainable fix because transcript ingestion is documented and tested; a source-class/provenance plus retention contract is safer.
