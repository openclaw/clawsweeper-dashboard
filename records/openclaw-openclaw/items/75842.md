## Summary

Keep open. Current main still treats generated session-corpus files as short-term recall inputs and rewrites the recall store without a retention prune; the newer repair path only detects a narrower narrative self-ingestion artifact, not the unbounded `short-term-recall.json` growth reported here.

## What This Changes

The issue requests bounding memory-core short-term recall growth by excluding generated dreaming corpus artifacts, adding store retention, and recording stable creation metadata.

## Best Possible Solution

Define a memory-core source-class and retention contract for short-term recall, then update ingestion, ranking, doctor/docs, and tests so generated corpus artifacts cannot grow the store indefinitely.

## Reproduction Assessment

Yes. Source inspection on current main shows session-corpus batches are appended and recorded into `short-term-recall.json`, and current tests assert those generated paths remain eligible recall candidates.

## Solution Assessment

Unclear. Retention metadata and pruning are the right direction, but excluding all session-corpus paths alone would conflict with the current documented transcript-ingestion behavior.
