## Summary

Keep open. Current main still records `memory/.dreams/session-corpus/*` entries into short-term recall and has no store-level retention path, but current docs and tests also make redacted session transcript ingestion intentional, so this needs maintainer direction rather than cleanup closure.

## What This Changes

The issue requests bounded short-term recall storage by avoiding generated session-corpus feedback, adding retention, and preserving stable creation metadata.

## Best Possible Solution

Define a bounded memory-core short-term recall retention and provenance contract that preserves stable creation metadata, prunes stale entries separately from deep ranking age, and prevents generated dreaming artifacts from feeding durable candidates while keeping intentional transcript ingestion.

## Reproduction Assessment

Yes. Source inspection shows session-corpus batches are appended and recorded into `short-term-recall.json`, and the write path has no pruning step for age, size, or source class.

## Solution Assessment

Unclear. Excluding all `session-corpus` paths is not the best standalone fix because redacted session transcript ingestion is documented and tested; retention plus provenance/source-class handling is the safer direction.
