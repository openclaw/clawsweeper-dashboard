## Summary

Keep open. Current main still records `memory/.dreams/session-corpus/*` batches into `short-term-recall.json`, still treats those paths as short-term recall candidates, and still lacks a store-level retention or creation-time contract; the fix needs maintainer judgment because session transcript ingestion is documented and tested behavior.

## What This Changes

The issue asks OpenClaw to stop `short-term-recall.json` from growing through dreaming session-corpus self-ingestion and to add a bounded retention/creation-time contract.

## Best Possible Solution

Define a memory-core source-class and retention contract that preserves intentional transcript ingestion, prevents derived dreaming artifacts from recursively seeding recall, records stable creation metadata, and prunes bounded short-term state with matching docs and tests.

## Reproduction Assessment

Yes. Source inspection on current main shows session-corpus batches are appended and then recorded into the short-term recall store, while ranking and doctor paths still accept those paths as candidates.

## Solution Assessment

Unclear. Excluding every session-corpus path is not safely the best fix because transcript ingestion is documented and tested; a source-class plus retention contract is the narrower maintainable direction.
