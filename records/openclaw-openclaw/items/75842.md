## Summary

Keep open. Current main still persists generated `memory/.dreams/session-corpus/*.txt` entries into `short-term-recall.json`, lacks a store-level retention prune and `firstRecordedAt`, and the remaining fix changes documented/tested memory-core dreaming behavior.

## What This Changes

Prevent memory-core dreaming from unboundedly growing `short-term-recall.json` with generated session-corpus recall candidates and add retention-friendly metadata.

## Best Possible Solution

Define a memory-core retention and source-class contract that bounds `short-term-recall.json`, preserves stable creation metadata, and separates generated transcript-corpus signals from organic recall where appropriate.

## Reproduction Assessment

Yes. Source inspection on current main shows session-corpus batches are appended and then recorded into the short-term recall store, and current tests assert those paths rank as candidates.

## Solution Assessment

Unclear. Retention plus creation metadata is the right direction, but excluding all session-corpus paths alone conflicts with the documented and tested transcript-ingestion feature.
