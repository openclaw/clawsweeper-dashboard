## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Sanitize the bridge environment before spawning:** `extensions/claude/src/app-server/client.ts:82-85`
  - body: `appServer.env` comes from plugin config, but this path merges it directly over `process.env` before spawning `openclaw-claude-bridge`. That bypasses the host-exec sanitizer used elsewhere to reject dangerous inherited and override keys such as path/loading hooks, so a workspace/plugin config can influence a credentialed subprocess. Build the env with `sanitizeHostExecEnv({ baseEnv: process.env, overrides: this.opts.env ?? null })` and add focused coverage.
  - confidence: 0.93
- **[P1] Do not bypass release age for the contributor bridge:** `pnpm-workspace.yaml:12`
  - body: This unversioned exclusion lets every future `@zeroaltitude/openclaw-claude-bridge` publish bypass the workspace `minimumReleaseAge` gate, and that package is the executable subprocess that handles Anthropic credentials and tool execution. Remove the exemption or make it a recorded maintainer/secops decision after the bridge is republished or otherwise owned under an approved scope.
  - confidence: 0.9
- **[P1] Register claude in the middleware runtime normalizer:** `src/plugins/agent-tool-result-middleware-types.ts:5`
  - body: The public type now accepts `"claude"`, and the Claude bridge calls `createAgentToolResultMiddlewareRunner({ runtime: "claude" })`, but `AGENT_TOOL_RESULT_MIDDLEWARE_RUNTIMES` still contains only `openclaw` and `codex`. Plugins declaring `runtimes: ["claude"]` will normalize to an empty list and never load, so the new API surface is broken until the runtime list and tests are updated.
  - confidence: 0.94
- **[P2] Forward source-reply telemetry to the runner:** `extensions/claude/src/app-server/run-attempt.ts:401-407`
  - body: `dynamic-tools.ts` collects `messagingToolSourceReplyPayloads` when the message tool returns an internal source reply, but the attempt result only copies the regular sent-text/media/target telemetry. In `message_tool_only` mode, the runner later reads `attempt.messagingToolSourceReplyPayloads`, so structured source replies are dropped unless this field is copied alongside the other bridge telemetry.
  - confidence: 0.88
