- **[P1] Preserve derived metadata snapshot reuse:** `src/plugins/current-plugin-metadata-snapshot.ts:30`
  - body: This flips derived registry snapshots from reusable to cleared.   - confidence: 0.88
- **[P1] Sanitize the bridge spawn environment:** `extensions/claude/src/app-server/client.ts:82-85`
  - body: `appServer.env` is plugin/user config, but the bridge merges it straight into `process.env` before spawning the credential- and tool-executing subprocess. That bypasses OpenClaw's host env sanitizer and can allow dangerous overrides such as command-resolution or Node execution knobs into the child process. Route overrides through the host exec env sanitizer or an exported plugin-SDK equivalent before `spawn`.
  - confidence: 0.94
- **[P1] Move the bridge runtime under accepted ownership:** `extensions/claude/package.json:10-11`
  - body: The official `@openclaw/claude` package would install and execute `@zeroaltitude/openclaw-claude-bridge`, while the PR body says republishing under OpenClaw ownership is still out of scope. Because that package owns Anthropic credentials, network I/O, and tool execution, bundling it as an official plugin needs accepted ownership or an external-plugin path before merge.
  - confidence: 0.9
- **[P1] Version-bound the release-age exception:** `pnpm-workspace.yaml:12`
  - body: Adding the bare package name to `minimumReleaseAgeExclude` means every future publish of this contributor-scoped bridge package bypasses the workspace quarantine without another reviewable diff.
