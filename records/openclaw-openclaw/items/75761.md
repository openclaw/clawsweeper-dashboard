## Summary

Keep this PR open. The lifecycle change is plugin-local: Active Memory still runs in `before_prompt_build` before the main reply, but this PR adds an earlier terminal branch after `memory_search` so empty or unavailable recall can skip hidden context injection and let the main reply proceed instead of waiting for the embedded subagent timeout.

## What This Changes

The PR changes Active Memory runtime, tests, and changelog to fast-fail terminal Memory Search outputs, classify timeout boilerplate as no recall, and treat `timeoutMs` as the visible recall budget.

## Best Possible Solution

Land a conflict-free plugin-local fix after maintainer alignment on hard timeout semantics, keeping QMD recall-quality work in #75860 and setup-grace policy work separate.

## Reproduction Assessment

Yes. On current main, a mocked zero-hit `memory_search` tool result cannot end Active Memory early because the plugin does not observe formatted tool output, and the setup-grace path can keep the outer recall watchdog open past `timeoutMs`.

## Solution Assessment

Yes for the fast-fail and timeout-boilerplate parts: the PR uses an existing embedded-runner callback seam inside the owner plugin. The hard timeout policy is the part needing maintainer confirmation because current docs and related timeout work still describe setup grace as extra setup budget.
