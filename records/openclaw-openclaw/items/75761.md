## Summary

Keep open. Current main still lacks the early terminal Memory Search observation and timeout-boilerplate handling this PR adds; the patch is plausibly plugin-local, but the hard-timeout versus setup-grace policy and the reported dirty/failing branch state need maintainer handling before merge.

## What This Changes

This PR updates the Active Memory plugin, tests, and changelog to fast-fail terminal Memory Search recalls, discard timeout boilerplate, and treat the configured recall timeout as the user-visible hard budget.

## Best Possible Solution

Land a rebased Active Memory plugin-local fix that observes terminal Memory Search output through existing embedded-runner callbacks, preserves non-terminal Memory Get behavior, and documents or accepts the chosen timeout-budget policy while keeping QMD quality work in #75860.

## Reproduction Assessment

Yes. On current main, a mocked zero-hit or unavailable memory_search result cannot end Active Memory early because the runEmbeddedPiAgent call does not enable tool-output callbacks, and the outer watchdog is visibly timeoutMs plus setupGraceTimeoutMs.

## Solution Assessment

Mostly yes. The fast-fail and timeout-boilerplate parts use the narrow owner-plugin seam already exposed by the embedded runner; the hard timeout policy is the unresolved part because current main still models setup grace as extra watchdog budget.
