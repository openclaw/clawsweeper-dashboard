## Summary

Keep this PR open. Current main still lacks early observation of formatted Memory Search output and still extends the recall watchdog with setup grace, while the PR is a plugin-local implementation with focused tests and no discrete blocking review finding.

## What This Changes

The PR updates Active Memory runtime, tests, and changelog to fast-fail empty or unavailable Memory Search output, classify timeout boilerplate as timeout, and enforce `config.timeoutMs` as the visible recall budget.

## Best Possible Solution

Land this plugin-local fix after maintainer alignment on hard timeout semantics and exact-head validation, while keeping QMD recall-quality changes in #75860.

## Reproduction Assessment

Yes. On current main, Active Memory does not observe formatted `memory_search` tool output during recall and waits for the embedded run or extended watchdog instead.

## Solution Assessment

Yes, with a policy caveat. The patch uses an existing embedded-runner callback seam and keeps the behavior in the owner plugin, but maintainers should confirm that `config.timeoutMs` is the hard user-visible budget.
