## Summary

Current main still lacks the terminal Memory Search fast-fail and configured-budget timeout behavior, and the PR is a focused Active Memory plugin fix with matching tests and changelog coverage; the remaining blocker is maintainer review plus resolving the exact-head `checks-fast-protocol` failure before merge.

## What This Changes

The PR updates the Active Memory runtime, tests, and changelog to fast-fail terminal `memory_search` outputs, avoid treating `memory_get` misses as terminal, classify timeout boilerplate as timeout/no recall, and enforce `config.timeoutMs` as the visible recall budget.

## Best Possible Solution

Land the narrow plugin-owned fix after maintainer review and green exact-head CI, keeping the behavior inside `extensions/active-memory` and using the existing embedded `onToolResult` callback rather than adding a broader core API.

## Solution Assessment

Yes, with the CI caveat. The proposed approach is the narrow maintainable fix for this PR because it observes formatted Memory Search output inside the Active Memory plugin, preserves unrelated Memory Get misses, and keeps the embedded-runner contract unchanged.
