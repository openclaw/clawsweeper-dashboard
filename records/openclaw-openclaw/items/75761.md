## Summary

Keep this PR open. The latest force-pushed diff fixes the earlier changelog attribution blocker, current main still lacks the Active Memory fast-fail and hard-timeout behavior, and the branch is still draft rather than cleanup-closeable.

## What This Changes

This draft PR updates the Active Memory plugin, its tests, and the changelog to fast-fail zero-hit or unavailable Memory Search output, avoid treating Memory Get misses as terminal, classify embedded timeout boilerplate as timeout, and enforce the configured recall timeout as the user-visible deadline.

## Best Possible Solution

Finish the PR readiness loop and land the owner-plugin fix after targeted Active Memory tests and the changed gate pass, keeping the behavior in `extensions/active-memory` and using the existing embedded tool-output callback instead of adding a broader core API.

## Solution Assessment

Yes. The approach is narrow and maintainable: it observes formatted Memory Search output inside the Active Memory plugin, preserves unrelated Memory Get misses, and avoids expanding the embedded-runner contract.
