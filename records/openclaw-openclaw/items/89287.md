## Summary

Keep open: target-aware completion accounting is a useful fix direction, but this patch rejects valid topic-encoded thread evidence and still needs real external-route proof or an explicit maintainer proof override.

## What This Changes

The PR adds target metadata to outbound delivery payload outcomes and makes subagent/ACP completion delivery accounting validate message-tool and automatic delivery evidence against provider/account/thread targets.

## Best Possible Solution

Keep the target-aware accounting, but preserve the shared matcher's topic/implicit-thread semantics, document the new target metadata, and require redacted live route proof or an explicit maintainer proof override before merge.

## Reproduction Assessment

Yes from source inspection: current main accepts topic-encoded thread evidence, while the PR's exact matcher requires a separate `threadId`. I did not run the tests in this read-only review.

## Solution Assessment

No: target-aware accounting is the right layer, but this implementation is too narrow until it preserves canonical topic/implicit-thread matching and documents the new JSON field.
